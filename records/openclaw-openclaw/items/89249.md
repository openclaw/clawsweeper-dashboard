## Summary

Keep open: current main and the latest release still have the Control UI picker mismatch where raw pages can be filtered down to no visible chat rows, and the proof-positive UI fix remains open for maintainer review.

## What This Changes

The issue reports that the Control UI chat session picker becomes effectively unnavigable when many spawned subagent sessions dominate paginated `sessions.list` results.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/89323 for the UI pagination/count fix, while keeping optional subagent visibility controls in https://github.com/openclaw/openclaw/issues/89774 separate.

## Reproduction Assessment

Yes at source level: current main hides subagent and spawned rows after raw `sessions.list` pagination and displays the raw total count, so a hidden-only page can leave the visible picker unchanged. I did not run a live Control UI reproduction because this review is read-only.

## Solution Assessment

Yes for the open UI PR path: fixing the picker to advance through hidden-only pages and count visible rows addresses the remaining bug without changing the Gateway protocol. The CLI-only nextOffset PR is useful parity work but not the best standalone fix for this issue.
