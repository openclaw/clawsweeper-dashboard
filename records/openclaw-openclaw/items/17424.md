## Summary

Keep open: current main still does not implement OpenAI-compatible configurable memory-search dimensions. The existing public field is `outputDimensionality`, but OpenAI inline and batch embedding requests still omit `dimensions`, and the exact `memorySearch.dimensions` key requested here is not accepted by the strict schema.

## What This Changes

The issue requests configurable embedding dimensions for memory search defaults and per-agent overrides, passed through as OpenAI-compatible `dimensions` in embedding API calls.

## Best Possible Solution

Settle the public memory-search dimensionality contract, then wire the selected value through supported OpenAI-compatible inline and batch embedding requests with cache-key/reindex separation, docs, schema/type/help updates, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence current-main gap: the requested `memorySearch.dimensions` key is absent and OpenAI memory embedding requests do not send dimensions.

## Solution Assessment

No, current main is not a complete solution for the requested behavior. The narrowest maintainable path likely reuses or deliberately aliases `outputDimensionality`, then adds OpenAI-compatible request pass-through plus cache and reindex separation.
