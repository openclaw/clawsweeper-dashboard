## Summary

Keep open. Current main has adjacent `outputDimensionality` support for some embedding providers and the gateway `/v1/embeddings` endpoint, but the requested memory-search `dimensions` config key and OpenAI inline/batch `dimensions` pass-through are still missing.

## What This Changes

Add a configurable memory-search embedding dimension setting for defaults and per-agent overrides, then pass it through to OpenAI-compatible embedding requests.

## Best Possible Solution

Define one public memory-search dimension contract, normalize defaults and per-agent overrides into provider options, teach the OpenAI inline and batch paths to send `dimensions` when configured, and make that value part of cache/reindex identity with focused docs and tests.

## Reproduction Assessment

Yes. For this feature request, source inspection is enough to reproduce the gap: the strict schema lacks `dimensions`, the resolver only handles `outputDimensionality`, and OpenAI inline/batch request bodies omit `dimensions`.

## Solution Assessment

No, current main is not the best available solution because its adjacent `outputDimensionality` support does not reach the requested OpenAI-compatible memory-search path. The safer fix is to reuse the existing provider option internally while deliberately exposing or aliasing the public config field.
