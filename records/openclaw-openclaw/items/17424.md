## Summary

Keep open. Current main still has only adjacent `outputDimensionality` plumbing; it does not accept the requested `dimensions` memory-search config key or pass a dimension value through OpenAI inline and batch embedding requests, and recent maintainer discussion reached the same conclusion.

## What This Changes

Request support for configuring OpenAI-compatible memory-search embedding dimensions at defaults and per-agent scope.

## Best Possible Solution

Expose and normalize one memory-search dimension setting, pass it through OpenAI inline and batch embeddings, and include it in cache/reindex identity with docs and focused tests.

## Reproduction Assessment

Yes. Source inspection on current main is enough to reproduce the gap: the strict memory-search schema lacks `dimensions`, the resolver only merges `outputDimensionality`, and OpenAI inline/batch request bodies omit a dimension field.

## Solution Assessment

No. Current main's adjacent `outputDimensionality` support does not solve the requested OpenAI-compatible memory-search path; the narrow fix is to reuse that internal option while deliberately exposing or aliasing the requested public config field.
