## Summary

Keep open: current main still lacks the requested `memorySearch.dimensions` config contract and the OpenAI memory embedding inline and batch paths do not send a `dimensions` request field. Existing `outputDimensionality` support is wired through shared memory provider options, but OpenAI does not consume it or include it in batch cache identity.

## What This Changes

Add configurable embedding dimensions for memory search defaults and per-agent overrides, passed through to OpenAI-compatible embedding requests.

## Best Possible Solution

Define the public memory-search dimensionality contract, then wire the selected value through supported OpenAI-compatible inline and batch embeddings with cache/reindex separation, docs, and focused regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence current-main gap: the exact `memorySearch.dimensions` key is absent and OpenAI memory embedding requests omit `dimensions`.

## Solution Assessment

No, current main is not a complete solution for the requested behavior. The narrowest maintainable path likely reuses or aliases `outputDimensionality` with provider/model capability checks, then updates OpenAI inline and batch paths plus cache/reindex identity.
