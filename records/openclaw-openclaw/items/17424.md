## Summary

Keep open: current main still does not provide the requested OpenAI-compatible memory-search dimensions behavior. The existing config path resolves `outputDimensionality`, but the OpenAI inline and batch embedding request bodies do not send a `dimensions` field, and the exact `memorySearch.dimensions` key from the issue is not part of the strict schema.

## What This Changes

The issue requests configurable embedding dimensions for memory search via default and per-agent `memorySearch` config, with OpenAI-compatible pass-through.

## Best Possible Solution

Keep this open and settle the public config contract, then wire the selected dimensionality through supported OpenAI-compatible inline and batch embedding requests with cache identity, reindex behavior, docs, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence current-main gap: OpenAI memory embedding requests omit dimensions and the exact `memorySearch.dimensions` key is not accepted.

## Solution Assessment

No, current main is not the complete solution for the issue. The narrowest maintainable path likely reuses or aliases the existing `outputDimensionality` contract, then adds OpenAI-compatible request pass-through and cache/reindex separation.
