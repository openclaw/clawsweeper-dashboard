## Summary

Keep open: current main still lacks the requested `memorySearch.dimensions` config key and the OpenAI memory embedding inline and batch paths still do not send a `dimensions` request field. Existing `outputDimensionality` support is adjacent, but the public config name and provider pass-through policy still need maintainer judgment.

## What This Changes

Add configurable embedding dimensions for memory search defaults and per-agent overrides, passing the selected value through to OpenAI-compatible embedding requests.

## Best Possible Solution

Define one public memory-search dimensionality contract, then wire it through compatible providers with cache/reindex separation, docs, schema/help updates, and focused regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that the requested config key and OpenAI-compatible request pass-through are still absent on current main.

## Solution Assessment

No: current main is not a complete solution for the requested behavior. The narrowest maintainable path likely reuses or aliases `outputDimensionality` with provider/model capability checks, then updates OpenAI inline and batch paths plus cache/reindex identity.
