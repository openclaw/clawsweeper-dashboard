## Summary

Keep open. Current main still lacks the heartbeat cooldown state, non-interval wake gating, and overdue-timer recovery this PR is trying to add, while the PR branch remains a focused but not-yet-correct implementation because default no-argument plugin/system heartbeat wakes would still be throttled inside the configured interval.

## What This Changes

The PR adds heartbeat last-run cooldown state, scheduler gating/tests, overdue interval recovery, and a changelog entry to throttle activity-driven non-interval heartbeat wake bursts while preserving selected explicit wake paths.

## Best Possible Solution

Land a corrected heartbeat scheduler change that preserves explicit default/no-reason wake-now semantics while throttling ordinary burst sources and retaining the overdue interval recovery tests.

## Reproduction Assessment

Yes. On the PR branch, run one heartbeat and then call api.runtime.system.requestHeartbeatNow() or requestHeartbeatNow({ coalesceMs: 0 }) inside the configured interval; the omitted reason is classified as other and the PR predicate skips the wake as too-soon before runOnce.

## Solution Assessment

No, not as-is. The cooldown and overdue recovery are the right narrow direction, but the predicate should exempt the default requested/no-reason explicit wake or normalize it to manual semantics before merge.
