## Summary

Keep open. Current main still does not implement localized or rate-limited tool summaries, so this PR is still a live implementation candidate, but the proposed patch still has a concrete ACP throttling bug and is missing the required changelog entry for a user-facing config feature.

## What This Changes

The PR adds `agents.defaults.toolSummaries` locale/minInterval config, localizes embedded and ACP tool summary labels plus common exec hints, updates config schema/docs, and adds focused tests.

## Best Possible Solution

Land a rebased version after fixing ACP interval accounting against actual visible delivery time, adding the required changelog entry, and keeping config types, Zod schema, generated schema, docs, embedded delivery, ACP delivery, and redaction tests aligned.

## Reproduction Assessment

Yes. Current-main absence is reproducible with `rg` and source inspection; the remaining PR bug can be reproduced by configuring ACP `minIntervalMs`, making `flush` or `deliver` slow, and observing that the next summary can be allowed immediately after the previous summary actually becomes visible.

## Solution Assessment

No, not as-is. The overall direction is the narrow maintainable place for the feature, but the ACP throttle should record the timestamp after actual delivery or queueing semantics are settled, and the user-facing config addition needs a changelog entry before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Base ACP throttling on actual delivery time:** `src/auto-reply/reply/acp-projector.ts:505`
  - body: `emittedAt` is captured before `await flush(true)` and `await params.deliver(...)`, then stored as `lastToolSummarySentAt` after delivery. If flushing or delivery is slow, the next tool event can pass the interval check immediately after the prior summary becomes visible, so `minIntervalMs` no longer guarantees spacing between visible ACP summaries. Record the timestamp after the awaited delivery path, or otherwise base the check on the actual send/queue time.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `src/config/zod-schema.agent-defaults.ts:172-180`
  - body: This adds a new user-facing `agents.defaults.toolSummaries` config surface and documents it, but `CHANGELOG.md` is not updated. OpenClaw policy requires user-facing feature/config changes to add an Unreleased changelog entry, so release notes would miss this new operator-facing setting.
  - confidence: 0.9
