## Summary

Keep open. Current main still selects the heartbeat.model override for heartbeat execution, but session_status only receives session/config context and builds the displayed model from the resolved session entry/defaults, so the transient heartbeat run model is still not represented for sessionKey="current".

## What This Changes

Request that session_status inside a heartbeat report the active heartbeat.model per-run override instead of the agent/session default.

## Best Possible Solution

Thread the selected current-run provider/model into the session_status tool context for semantic-current lookups only, while leaving explicit other-session status and session persistence unchanged.

## Reproduction Assessment

Yes, at source level: configure heartbeat.model, let the heartbeat call session_status with sessionKey="current", and the current tool path has no field that can carry the transient selected heartbeat provider/model. I did not live-run the heartbeat in this read-only pass.

## Solution Assessment

Yes. The narrow maintainable fix is to pass the selected runtime model through existing tool construction/run context for semantic-current status only; persisting heartbeat model state would risk regressing the related heartbeat model leak.
