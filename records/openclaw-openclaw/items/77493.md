## Summary

Keep open. Current main still selects heartbeat-scoped model and thinking values for execution, but session_status for semantic current is built from stored session/default state and lacks a transient current-run status context.

## What This Changes

session_status should report heartbeat-scoped runtime overrides for the current heartbeat run instead of stored/default session values.

## Best Possible Solution

Thread the selected current-run status context into session_status for semantic-current lookups only, while keeping explicit other-session lookups and persisted session state unchanged.

## Reproduction Assessment

Yes, at source level: configure heartbeat.model and have the heartbeat call session_status({"sessionKey":"current"}); the run selects the heartbeat model, but the tool has no runtime override fields and reads session/default state. The May 7 reporter comment adds live evidence for both model and thinking mismatch.

## Solution Assessment

Yes. The narrow maintainable solution is to pass transient current-run model/thinking context through the existing tool-construction path for semantic current only; persisting heartbeat state or changing explicit session lookups would be riskier.
