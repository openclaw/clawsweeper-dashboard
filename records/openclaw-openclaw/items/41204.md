## Summary

Keep open: current main does not implement the queue-cap part, and this PR is not mergeable as-is because it regresses subagent cleanup and installer validation behavior that current main protects.

## What This Changes

The PR adds a session-store lock queue cap, subagent orphan-run sweep cleanup, shared installer package/formula validation, and three related tests.

## Best Possible Solution

Keep current main's installer-specific validation and guarded subagent sweep behavior, and replace this with a narrow queue-cap change in store-writer.ts plus deterministic tests if maintainers want that hardening.

## Reproduction Assessment

Yes for the PR regressions: the PR diff and current-main source show the missing archive guard, lost delete retry behavior, overly broad installer regex, and slow queue test path. The queue exhaustion concern is source-plausible through the uncapped pending array, but no live stress run was performed in this read-only review.

## Solution Assessment

No. The proposed mixed patch is not the narrowest maintainable solution because it replaces existing installer and subagent contracts with broader logic that breaks valid installs and cleanup behavior; a focused current-main queue-cap PR would be safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Restore the archive guard before standard cleanup:** `src/agents/subagent-registry.ts:784-785`
  - body: After the orphan branch, entries that are not due for archive can fall into the standard cleanup block because the previous `!entry.archiveAtMs || entry.archiveAtMs > now` guard is gone. The next sweep can delete active or recently ended runs and call `sessions.delete` early.
  - confidence: 0.95
- **[P2] Keep archive cleanup on the retrying path:** `src/agents/subagent-registry.ts:728-746`
  - body: The new due-archive branch removes the run and attachments before `sessions.delete` succeeds, then swallows delete failures. A transient gateway error can lose the retry and context-engine notification path that current main preserves.
  - confidence: 0.9
- **[P2] Use installer-specific safe specs:** `src/agents/skills-install.ts:128`
  - body: The shared package-name regex is applied to node, go, and uv installs, but it rejects valid metadata that current main accepts, including uv extras/version pins, Go `+incompatible`, and npm build metadata or tilde names.
  - confidence: 0.89
- **[P2] Avoid the serial 100-second lock queue test:** `src/config/sessions/store.lock-queue.test.ts:21-23`
  - body: This test queues up to 1001 accepted lock tasks that each sleep for 100 ms, and `Promise.allSettled` waits for the serial drain. Under normal Vitest timeouts this is expected to fail or add a very long test, so the proof needs a deterministic blocked-lock setup.
  - confidence: 0.78
