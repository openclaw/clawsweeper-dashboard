## Summary

Keep this PR open, but it is not merge-ready: the reported download-filename bug is real on current main and latest release, yet the patch applies `attachment` to the shared assistant-media route too broadly and still needs browser-level proof for the save-dialog and preview boundary.

## What This Changes

The PR adds a basename-derived `Content-Disposition: attachment` header to successful Control UI assistant-media responses in `src/gateway/control-ui.ts`.

## Best Possible Solution

Gate `Content-Disposition` to document or non-inline MIME responses, use a safe RFC 5987/6266 header builder with a readable ASCII fallback, add focused gateway/UI tests, and require redacted browser proof.

## Reproduction Assessment

Yes at source level: current main and v2026.6.8 serve assistant-media without `Content-Disposition` while document links use that route. I did not run a live browser save-dialog reproduction in this read-only review.

## Solution Assessment

No as submitted. The patch fixes the filename symptom by forcing `attachment` on the whole route, but the best fix should target document/non-inline responses and preserve preview behavior for image, audio, and video.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Keep previewable media inline:** `src/gateway/control-ui.ts:606-610`
  - body: This endpoint is also used for local image/audio/video preview URLs and image/video open links in WebChat. Setting every successful response to `attachment` changes that shared route from inline/openable media into a download hint, so gate this header to document or non-inline MIME responses instead.
  - confidence: 0.9
- **[P2] Encode filename* with RFC 5987-safe characters:** `src/gateway/control-ui.ts:610`
  - body: `encodeURIComponent` leaves characters such as apostrophes, parentheses, and `*` unescaped, but those are delimiter-sensitive in the extended `filename*` value. Use a dedicated RFC 5987 encoder and a readable ASCII fallback before interpolating the header.
  - confidence: 0.82
