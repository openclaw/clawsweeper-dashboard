## Summary

This PR should stay open for repair rather than cleanup-close: the reported filename problem is real on current main, but the patch applies `attachment` to a mixed media endpoint and can regress existing inline image/audio/video previews.

## What This Changes

The PR adds a basename-derived `Content-Disposition: attachment` header with `filename` and `filename*` parameters to every successful Control UI assistant-media response.

## Best Possible Solution

Preserve inline handling for previewable media while adding download filenames for document/non-preview responses and covering both paths with gateway/UI tests.

## Reproduction Assessment

Yes at source level: current main streams assistant-media without `Content-Disposition`, and the UI routes document links through that endpoint. I did not run a live browser save-dialog repro in this read-only review.

## Solution Assessment

No: this is a plausible symptom fix, but the best fix should target document/non-preview downloads and preserve inline image/audio/video behavior.
