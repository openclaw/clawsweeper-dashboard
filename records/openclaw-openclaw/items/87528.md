## Summary

This PR addresses a real webchat/gateway filename bug that is still present on current main and v2026.6.8, but it is not merge-ready because it applies `Content-Disposition: attachment` to the shared assistant-media route and can break existing inline image/audio/video previews.

## What This Changes

The PR adds a basename-derived `Content-Disposition: attachment` header with `filename` and `filename*` to successful Control UI assistant-media responses.

## Best Possible Solution

Gate the attachment header to document or otherwise non-inline MIME responses, use a standards-safe filename helper, add focused gateway/UI coverage, and prove the Control UI browser behavior.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.8 stream assistant media without Content-Disposition, and the PR proof shows the added header from a live gateway. I did not run a browser save-dialog repro in this read-only review.

## Solution Assessment

No. The header is added at the right server route but too broadly; the best fix should preserve inline media previews and add focused coverage/proof for both the download and preview paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep previewable media inline:** `src/gateway/control-ui.ts:609-610`
  - body: This endpoint is also used as the `src` for local image, audio, and video previews in webchat. Setting every successful response to `attachment` can make existing previews download instead of rendering; gate this header to document/non-inline responses or another explicit download path.
  - confidence: 0.9
- **[P2] Encode filename* with RFC 5987-safe characters:** `src/gateway/control-ui.ts:610`
  - body: `encodeURIComponent` leaves characters such as single quotes, parentheses, and `*` unescaped, which are not safe in the extended `filename*` value syntax. Use a helper that tightens the percent-encoding and supplies a readable ASCII fallback before interpolating the header.
  - confidence: 0.82
