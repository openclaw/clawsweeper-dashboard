Full review comments:

- **[P1] Keep Canvas command defaults plugin-owned:** `src/gateway/node-command-policy.ts:120`
  - body: `extensions/canvas/index.ts` already registers the Canvas node-invoke policy for default platforms, and `docs/refactor/canvas.md` says those defaults were moved out of hardcoded gateway lists. Adding `CANVAS_COMMANDS` to `PLATFORM_DEFAULTS.macos` makes raw `node.invoke` allow Canvas commands even when the Canvas plugin is disabled or denied, creating a second source of truth and bypassing the plugin boundary. Please keep this in the Canvas plugin policy instead.
  - confidence: 0.91
- **[P1] Move canvas.eval danger handling into the Canvas policy:** `src/gateway/node-command-policy.ts:78`
  - body: Adding `canvas.eval` to `DEFAULT_DANGEROUS_NODE_COMMANDS` does not filter active plugin defaults: `resolveNodeCommandAllowlist` still adds `listDefaultPluginNodeCommands`, and the Canvas plugin currently registers `canvas.eval` in a non-dangerous default policy. With the plugin loaded, this PR still allows `canvas.eval` by default while the new test only covers an empty registry; split or mark the Canvas plugin policy as dangerous if eval must be opt-in.
  - confidence: 0.87
