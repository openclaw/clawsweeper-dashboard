## What This Changes

The issue requests a non-plaintext way to configure `hooks.token`, such as `${ENV_VAR}` substitution, `tokenFile`, or SecretRef support.

## Best Possible Solution

Use the existing `${HOOKS_TOKEN}` substitution for `hooks.token`; keep any future `tokenFile` or SecretRef expansion as a separate product/config design only if maintainers want another supported secret surface.

## Solution Assessment
