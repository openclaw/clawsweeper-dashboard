Keep open. Current main has useful Google Meet hardening, but it still does not satisfy the requested reliable headless/listen-first participant model, and the maintainer discussion explicitly says the latest diagnostic fix does not close the broader request.

## Best Possible Solution

Keep this as Google Meet plugin product work: define the supported identity/admission model, implement a listen-first or headless-capable participant path that can prove real transcript/audio movement before reporting success, document the setup constraints, and validate it with a live Google Meet smoke path.
