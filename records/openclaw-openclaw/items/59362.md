## Summary

Keep open: current main still routes normal Windows host exec through a PowerShell `-Command` wrapper, while the merged shell-env `windowsHide` fix only addressed a separate startup probe path.

## What This Changes

The issue asks OpenClaw to stop flashing a visible Windows console window when the exec tool spawns commands.

## Best Possible Solution

Design a Windows exec invocation contract that direct-spawns safely parsed simple executable calls with hidden windows, while falling back to PowerShell for shell syntax, builtins, wrappers, and approval semantics.

## Reproduction Assessment

Yes from source inspection: run OpenClaw on Windows and trigger a normal gateway host exec call, and current main deterministically builds a PowerShell `-Command` child process for that path. I did not run live Windows visual proof in this review.

## Solution Assessment

Unclear. Direct-spawning simple executable invocations is plausible, but the maintainable fix needs an explicit fallback boundary for shell features, `.cmd` wrappers, PowerShell cmdlets, and approval policy.
