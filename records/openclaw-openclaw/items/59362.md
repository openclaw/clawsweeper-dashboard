## Summary

Keep open. Current main still routes non-sandbox Windows host exec through a PowerShell `-Command` wrapper, so the reported wrapper-window flash is not covered by the older `windowsHide: true` focus-steal fix or the open Microsoft tracker.

## What This Changes

The issue asks OpenClaw to avoid visible Windows console flashes when the exec tool spawns commands, likely by direct-spawning simple executable invocations instead of always wrapping them in PowerShell.

## Best Possible Solution

Keep the issue open until Windows exec can avoid visible wrapper windows for simple executable invocations while retaining a well-tested PowerShell fallback for shell features.

## Reproduction Assessment

Yes. Run OpenClaw on Windows and trigger a normal host exec call; current main deterministically builds a PowerShell `-Command` child process for that path, matching the reporter's root-cause reproduction.

## Solution Assessment

Unclear. Direct-spawning simple commands is likely the right direction, but the maintainable fix needs an explicit design for parsing, approvals, `.cmd` wrappers, shell syntax, and fallback behavior.
