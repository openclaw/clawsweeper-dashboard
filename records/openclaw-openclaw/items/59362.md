## Summary

Keep open: current main still routes normal non-sandbox Windows gateway exec through a PowerShell `-Command` wrapper, so the reported visible wrapper-window flash is not resolved by the current `windowsHide` coverage or by the separate shell-env fix.

## What This Changes

The issue asks OpenClaw to stop Windows `exec` tool calls from flashing a console window when commands are spawned.

## Best Possible Solution

Keep the issue open until Windows gateway exec can avoid visible wrapper windows for simple executable invocations while preserving the tested PowerShell fallback and approval semantics.

## Reproduction Assessment

Yes. Run OpenClaw on Windows and trigger a normal gateway host `exec` call; current main deterministically builds a PowerShell `-Command` child process for that path, matching the reported reproduction.

## Solution Assessment

Unclear. Direct-spawning simple commands is likely the right direction, but the maintainable fix needs an explicit discriminator for shell features, PowerShell cmdlets, `.cmd` wrappers, and approval policy.
