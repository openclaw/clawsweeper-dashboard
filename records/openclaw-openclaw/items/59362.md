## Summary

Keep open: current main still routes normal Windows gateway exec through the PowerShell `-Command` wrapper, so the core console-flash report is not fixed by the separate `execLoginShellEnvZero` patch or adjacent `windowsHide` work.

## What This Changes

The issue requests eliminating visible Windows console flashes from gateway exec tool calls, especially the PowerShell wrapper used for spawned commands.

## Best Possible Solution

Design a Windows exec invocation plan that direct-spawns safely parsed simple executable calls with `windowsHide`, while falling back to the current PowerShell path for shell syntax, builtins, wrappers, and approval semantics.

## Reproduction Assessment

Yes from source inspection: run OpenClaw on Windows and trigger a normal gateway host `exec` call, and current main deterministically builds a PowerShell `-Command` child process for that path. I did not run live Windows visual proof in this review.

## Solution Assessment

Unclear. Direct-spawning simple executable invocations is likely the right direction, but the maintainable fix needs an explicit fallback boundary for shell features, `.cmd` wrappers, PowerShell cmdlets, and approval policy.
