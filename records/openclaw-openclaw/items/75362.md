## What This Changes

The issue reports an OpenAI GPT-5.4 regression where parameter-free MCP tool schemas are rejected instead of normalized before OpenAI transport serialization.

## Best Possible Solution

Keep this issue open until PR #75401 or an equivalent central normalizer fix lands with regression coverage for undefined, null, and non-object top-level `properties` values.

Yes. A high-confidence source reproduction is `{ type: "object", properties: undefined }` or `properties: null` through `normalizeToolParameters` or `normalizeOpenAIStrictToolParameters`, where current main leaves the schema unnormalized.

## Solution Assessment

Yes in direction, but not yet on main. The narrow maintainable fix belongs in the shared schema normalizer rather than Flux-specific or transport-only special cases.
