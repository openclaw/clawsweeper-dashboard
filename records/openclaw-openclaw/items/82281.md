## What This Changes

The issue requests a Lobster docs update that enumerates injected workflow step env vars and clarifies that prior step output is accessed through `stdin:` or `$step` references, not `LOBSTER_STEP_*` env vars.

## Best Possible Solution

Land a corrected docs-only change through https://github.com/openclaw/openclaw/pull/82379 or a replacement that documents `LOBSTER_ARGS_JSON` and `LOBSTER_ARG_<NAME>` while warning that step output is not exposed through env vars.

## Reproduction Assessment

Yes. Source inspection reproduces the gap: current docs omit the injected env-var contract while the wrapper only forwards `process.env` and the pinned Lobster runtime injects args env vars, not step-output env vars.

## Solution Assessment

Yes. A targeted docs clarification is the narrowest maintainable fix; adding env-var-based step output would be a new runtime feature and is not needed for this report.
