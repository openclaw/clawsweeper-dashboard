## Summary

Keep open: current main still reads Feishu single messages and thread history without requesting full interactive-card content, so the parser can still receive only fallback card text. The direct fix PR is closed unmerged, and the remaining work is a narrow Feishu plugin repair with an SDK/API-contract check before implementation.

## What This Changes

The issue requests that Feishu message read and thread-history reads return actual interactive-card content instead of `[Interactive Card]`.

## Best Possible Solution

Land a narrow Feishu-plugin read/history fix that requests full card JSON through a verified SDK-safe or raw request adapter, preserves parser fallbacks, adds request-parameter regression tests, and keeps broader parser work coordinated with https://github.com/openclaw/openclaw/issues/60380.

## Solution Assessment
