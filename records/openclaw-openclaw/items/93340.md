## Summary

This PR is still necessary: current main still feeds inspected account summaries into plugin status snapshot hooks, while the typed status contract and SMS status builder expect resolved accounts. The patch looks correct, but it remains compatibility-sensitive because it changes the account shape delivered to channel plugin status hooks.

## What This Changes

The PR changes channel status snapshot construction so plugin status builders receive resolved runtime accounts and adds focused regression coverage.

## Best Possible Solution

Land the core status-path fix after maintainer confirmation of the resolved-account status-hook contract; avoid an SMS-specific workaround and keep read-only inspection semantics documented.
