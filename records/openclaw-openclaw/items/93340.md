## Summary

Keep open: the SMS bug is real, but the PR as written would bypass the documented read-only inspection path and can make status commands resolve credential-bearing accounts where they should preserve configured-unavailable metadata.

## What This Changes

The PR changes channel account status snapshots so plugin status builders receive resolved accounts, and adds regression coverage for that account-source contract.

## Best Possible Solution

Revise the shared status snapshot path so SMS can use resolved account fields when credential resolution is safe, while read-only status commands preserve inspectAccount configured-unavailable behavior and avoid exposing raw secrets.

Yes. Source inspection shows current main can feed SMS buildAccountSnapshot the inspectAccount summary instead of resolved Twilio fields, and the PR body includes redacted before/after live JSON for `openclaw channels status --channel sms --json`.

## Solution Assessment

No, not as written. The shared status path is the right layer for the SMS bug, but the unconditional resolveAccount branch bypasses the documented read-only inspection contract and needs a safer contract-preserving shape.
