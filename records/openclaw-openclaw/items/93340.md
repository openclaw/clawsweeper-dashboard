## Summary

Keep open: the SMS configured-status symptom is real, but the shared resolve-first change is not safe to merge because it bypasses the read-only inspection boundary for sibling plugins that intentionally avoid resolving SecretRef-backed credentials in status commands.

## What This Changes

The branch changes shared channel status snapshot account-source selection to call `resolveAccount` when a plugin status builder exists and adds a regression test for an SMS-like inspected-versus-resolved account shape mismatch.

## Best Possible Solution

Keep the shared status helper inspect-first, then repair SMS at its inspect/status projection boundary so configured SMS accounts report correctly without changing the global plugin status contract.

Yes. Source inspection shows current main can pass SMS's safe inspected account summary into a status builder that expects resolved Twilio fields, and the PR body includes redacted before/after live JSON for `openclaw channels status --channel sms --json`.

## Solution Assessment

No. The PR fixes the SMS symptom, but changing the shared helper to resolve runtime accounts for every status builder is not the narrowest maintainable fix.
