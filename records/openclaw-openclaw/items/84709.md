- **[P1] Treat empty CLI allowlists as enforced policy:** `src/cron/isolated-agent/run-executor.ts:207`
  - body: `toolsAllow: []` is still a defined runtime allowlist, and the CLI preparation layer fails closed for any defined `toolsAllow` because it cannot enforce runtime restrictions. This preflight rejects only non-empty finite lists, so an explicit no-tools CLI cron can still reach `runCliAgent` without `toolsAllow` and run with the CLI backend's native tools.
  - confidence: 0.88
- **[P2] Narrow missing-tool summary matching:** `src/cron/isolated-agent/helpers.ts:45-53`
  - body: The new classifier treats broad substrings such as `tool unavailable` and `unable to run required` in successful summaries as fatal without structured missing-tool metadata. A successful ops report that discusses an external unavailable tool or a historical unmet requirement can now be recorded as a cron failure, so this should use structured failure signals or a narrower sentinel.
  - confidence: 0.78
