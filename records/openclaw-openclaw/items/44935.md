## Summary

Keep open: current main has sticky per-session fallback behavior plus inline cooldown probes, but not the requested background health-probe recovery loop that marks the primary healthy and routes only new work back to it.

## What This Changes

Request a configurable health-probe-based model fallback recovery mechanism that keeps active fallback sessions sticky while using a recovered primary model for new sessions, subagents, and scheduled work.

## Best Possible Solution

Design a core fallback-recovery state machine that reuses existing model/auth probe contracts, records recovered primary availability for future routing, preserves active fallback sessions, and coordinates recovery notices with https://github.com/openclaw/openclaw/issues/44941.

## Reproduction Assessment

Not applicable as a normal bug reproduction because this is a feature request involving outage timing and policy. Source inspection gives high confidence that current main lacks the requested background recovery loop, durable recovered-primary routing state, configurable interval, and recovery notification surface.

## Solution Assessment

Unclear until maintainer design is done. The health-probe direction is plausible, but the maintainable fix should add an explicit recovery state and routing contract rather than clearing sticky session overrides or blindly switching active sessions back.
