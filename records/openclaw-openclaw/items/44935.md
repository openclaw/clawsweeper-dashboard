## Summary

Keep open: current main has partial request-time cooldown probe behavior, but it does not implement the requested background health-probe recovery loop, configurable interval, or default recovery notification policy.

## What This Changes

Request health-probe-based auto-recovery from fallback models to the primary for new sessions, subagents, and scheduled work, while keeping existing sessions sticky and notifying users on recovery.

## Best Possible Solution

Define a product-approved fallback recovery state machine that probes primaries without switching active sessions, routes new work to a recovered primary, and coordinates recovery notices with https://github.com/openclaw/openclaw/issues/44941.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that current main only has request-time cooldown probes and sticky auto fallback overrides, not the requested configurable recovery loop.

## Solution Assessment

Unclear until maintainer product design is done. The existing cooldown-probe and fallback-state machinery is a useful base, but adding background probing, new config, and default recovery notifications is not a narrow bug fix.
