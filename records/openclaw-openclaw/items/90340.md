## Summary

Keep open: current main and the latest release still return a bare `agents.defaults.compaction.model` override as the literal model under the active provider, and two open fix PRs already target the remaining work. This issue should stay open until a canonical fix PR, likely https://github.com/openclaw/openclaw/pull/90885 if maintainers accept that direction, lands or is closed.

## What This Changes

The issue asks OpenClaw to resolve `agents.defaults.compaction.model` aliases through the normal model alias resolver before embedded auto-compaction dispatch.

## Best Possible Solution

Land one canonical narrow fix that reuses the shared model alias resolver for compaction overrides, preserves literal configured model-id precedence, and then close this issue as implemented after merge.

## Reproduction Assessment

Yes at source level: a config with `compaction.model: "gpt54mini"` and an alias entry for `openai/gpt-5.4-mini` will currently produce provider `openai` and model `gpt54mini` from `resolveEmbeddedCompactionTarget`. I did not run a live long-session compaction in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is to call the existing alias/model-ref resolver in the compaction target helper while preserving current literal-model and auth-profile behavior. The open PR at https://github.com/openclaw/openclaw/pull/90885 appears to follow that shape, but the issue should remain open until a fix merges.
