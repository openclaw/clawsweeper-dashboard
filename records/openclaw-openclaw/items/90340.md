## Summary

Keep open: current main and v2026.6.8 still treat a bare compaction model override as a literal model id, while open fix PRs already target the remaining alias-resolution bug.

## What This Changes

The issue reports that embedded auto-compaction sends an alias from `agents.defaults.compaction.model` as a literal provider/model instead of resolving it through configured model aliases.

## Best Possible Solution

Land one canonical narrow fix, likely https://github.com/openclaw/openclaw/pull/90885 if maintainers accept that direction, then close this issue after the merged code is on main.

## Reproduction Assessment

Yes at source level: a bare `compaction.model` alias reaches the return branch as `model: override` without calling the shared alias resolver. I did not run a live long-session auto-compaction in this read-only review.

## Solution Assessment

Yes: the best fix is to reuse the existing shared model alias resolver in `resolveEmbeddedCompactionTarget` while preserving current literal model-id and auth-profile behavior. The linked open PRs are implementation candidates, so the issue should remain open until one merges or maintainers close both.
