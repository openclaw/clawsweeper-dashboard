Keep open: current main still lacks the requested hard token flush threshold, priority-preserving compaction strategy, and memory-search cache TTL. ## What This Changes

Add memory durability controls for a hard memory-flush token threshold, priority-preserving compaction, and memory-search embedding cache TTL.

## Best Possible Solution

Approve the memory durability design first, likely splitting hard flush semantics, priority preservation, and cache expiration into focused schema, runtime, docs, and test changes.

## Solution Assessment

Unclear: the requested controls are plausible, but bundling all three into one public config change is not yet proven to be the narrowest maintainable path. A safer direction is to decide hard flush semantics, priority preservation, and cache expiration separately.
