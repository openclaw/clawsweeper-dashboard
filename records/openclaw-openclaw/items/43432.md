## Summary

Keep open: current main still lacks the requested hard token flush threshold, priority-preserving compaction strategy, and memory-search cache TTL. The related memory-flush issues cover narrower parts of the durability problem, but none supersedes the full three-part public config and runtime policy request.

## What This Changes

Add `memoryFlush.hardThresholdTokens`, `compaction.strategy: "preserve-priority"`, and `memorySearch.cache.ttlSeconds` under `agents.defaults` to improve memory durability at scale.

## Best Possible Solution

Approve the memory durability design first, likely splitting hard flush semantics, priority preservation, and cache expiration into focused schema/runtime/docs/test changes.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that the proposed keys are absent from current main, but the reported 32-agent durability failure was not live-reproduced.

## Solution Assessment

Unclear: the requested controls are plausible, but adding all three as one bundled public config change is not yet proven to be the narrowest maintainable path. A safer path is to decide hard flush semantics, priority preservation, and cache expiration separately.
