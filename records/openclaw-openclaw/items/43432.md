## Summary

Keep open: current main still lacks the three requested memory durability controls, and adding them would be new config/API/runtime policy rather than conservative cleanup. The related open issues cover adjacent memory bounding and progressive flush work, but they do not cover the full cache TTL plus priority-preserving compaction request.

## What This Changes

Add hard token-based memory flush, priority-preserving compaction, and embedding cache TTL configuration under `agents.defaults`.

## Best Possible Solution

Split this into a maintainer-approved memory durability design, then implement the accepted schema, docs, runtime behavior, and focused regression coverage for each control.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that the proposed config keys are absent from current main, but the large-scale memory-loss scenario was not reproduced.

## Solution Assessment

Unclear: the requested controls are plausible, but shipping all three as one bundled config change is not the narrowest maintainable path. A safer solution is to split and approve hard flush, priority preservation, and cache TTL semantics separately.
