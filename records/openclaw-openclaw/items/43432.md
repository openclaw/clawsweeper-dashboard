## Summary

Keep open: current `main` still lacks the requested token hard-flush config, priority-preserving compaction strategy, and embedding cache TTL; adding them would be new public config and runtime policy, not conservative cleanup.

## What This Changes

Add `memoryFlush.hardThresholdTokens`, `compaction.strategy: "preserve-priority"`, and `memorySearch.cache.ttlSeconds` under agent defaults for stronger memory durability at scale.

## Best Possible Solution

Approve a scoped memory durability design, likely splitting hard token flush, priority-preserving compaction, and cache expiration into separately documented schema/runtime changes with focused tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that the proposed config keys are absent from current `main`, but the reported 32-agent memory-loss scenario was not live-reproduced.

## Solution Assessment

Unclear: the requested controls are plausible, but shipping all three as one bundled config change is not yet proven to be the narrowest maintainable path. A safer direction is to approve hard flush semantics, priority preservation, and cache expiration separately.
