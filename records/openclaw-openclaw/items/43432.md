## Summary

Keep open: current main still lacks the three requested memory durability controls, and adding them would be new public config plus runtime policy rather than conservative cleanup. ## What This Changes

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

Unclear: the requested controls are plausible, but shipping all three as one bundled config change is not the narrowest maintainable path. A safer direction is to approve hard flush semantics, priority preservation, and cache expiration separately.
