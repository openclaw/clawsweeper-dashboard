## Summary

Keep this PR open for normal review and CI: current main still has the root-only contract lookup, while the proposed diff is a narrow fix with no blocking functional or security findings from source review.

## What This Changes

The PR adds a `dist/` fallback for external channel secret contract resolution, covers it with an external channel fixture test, and adds a changelog fix entry.

## Best Possible Solution

Land this narrow loader, regression test, and changelog patch after CI or an equivalent targeted gate confirms external channel contract loading from npm package `dist` artifacts.

## Reproduction Assessment

Yes from source inspection. On current main, a plugin record whose root only contains `dist/secret-contract-api.*` makes `resolvePluginContractApiPath` return null, so Discord-style SecretRef assignments are not collected for an npm-installed channel plugin.

## Solution Assessment

Yes. Searching `dist/` after existing root-level candidates is the narrowest maintainable fix because it matches the package build layout while preserving root precedence and the existing plugin-root boundary validation.
