## Summary

Keep open for maintainer review: current main still has the `/acp spawn` cwd inheritance gap, and this PR is a focused fix with matching regression tests, sufficient live proof, green CI, and no blocking findings.

## What This Changes

The PR exports the ACP spawn cwd guard, applies target-agent workspace inheritance in `/acp spawn`, and adds command-path regression tests for inherited and missing workspaces.

## Best Possible Solution

Land the focused command-path fix after maintainer merge gates, preserving explicit `--cwd` precedence and the existing missing-workspace fallback contract.

## Reproduction Assessment

Yes. Source inspection shows `parseSpawnInput` leaves `cwd` undefined unless `--cwd` is supplied, and current `handleAcpSpawnAction` passes that undefined value directly to ACP initialization despite the documented target-workspace inheritance contract.

## Solution Assessment

Yes. Reusing `resolveSpawnedWorkspaceInheritance` plus `resolveRuntimeCwdForAcpSpawn` in the command path is the narrowest maintainable fix and avoids a second cwd policy.
