## Summary

Keep open: the underlying `/acp spawn` cwd inheritance bug is real and the branch is focused, but it is not merge-ready because it skips the existing direct-ACP missing-workspace/access-error guard and lacks required after-fix real behavior proof.

## What This Changes

The PR changes the `/acp spawn` slash-command path to resolve the target agent workspace when `--cwd` is omitted and adds a focused mock regression test.

## Best Possible Solution

Extend the slash-command `/acp spawn` path to share the direct ACP runtime-cwd contract: explicit `--cwd` wins, inherited target workspaces are used only when accessible, missing inherited workspaces fall back to backend default, and access errors are surfaced with focused tests.

## Reproduction Assessment

Yes from source inspection: `parseSpawnInput` leaves `cwd` undefined unless `--cwd` is supplied, and `handleAcpSpawnAction` passes that undefined value to runtime initialization. I did not run a live gateway/ACP repro in this read-only review.

## Solution Assessment

No as written: resolving the target workspace is the right direction, but the slash-command path should also preserve the existing direct ACP runtime-cwd hardening for missing inherited workspaces and access failures.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

- **[P2] Preserve inherited workspace fallback before runtime init:** `src/auto-reply/reply/commands-acp/lifecycle.ts:536`
