## Summary

Keep open: current main still has the `/acp spawn` command path passing only the explicit CLI cwd, while the docs require target-agent workspace inheritance. The PR appears to be the narrow command-path fix, has focused regression coverage, supplied live proof, and no blocking review finding.

## What This Changes

The PR exports the ACP spawn cwd resolver, applies target-agent workspace inheritance to the `/acp spawn` command path, and adds regression tests for inherited and missing workspaces.

## Best Possible Solution

Land the command-path cwd inheritance fix after normal maintainer merge gates, preserving explicit `--cwd` precedence and the existing missing-path fallback contract.

## Reproduction Assessment

Yes, source-reproducible with high confidence: current main leaves `spawn.cwd` undefined unless `--cwd` is supplied, then passes it directly into ACP initialization despite the documented target-workspace inheritance contract. I did not run a live repro during this read-only review, but the code path, docs, PR regression tests, and contributor live output align.

## Solution Assessment

Yes. Reusing `resolveSpawnedWorkspaceInheritance` and the existing ACP cwd guard in the command path is the narrowest maintainable fix; it avoids inventing a second cwd policy and keeps explicit-cwd and missing-workspace behavior consistent with direct ACP spawn.
