## Summary

Keep open. The PR addresses a real source-level gap in `/acp spawn`, but the patch should not merge as-is because it bypasses the existing inherited-cwd missing-path fallback and lacks contributor real behavior proof.

## What This Changes

The PR resolves the target agent workspace for `/acp spawn` when `--cwd` is omitted and adds a focused regression test for that command path.

## Best Possible Solution

Land a narrow command-path fix that shares the existing ACP cwd resolution guard, preserves explicit `--cwd`, keeps missing inherited workspaces falling back to backend defaults, and includes regression coverage plus real behavior proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main parses cwd only from `--cwd` and passes `spawn.cwd` to `initializeSession`, so `/acp spawn codex` without `--cwd` cannot supply the configured target workspace.

## Solution Assessment

No. The direction is right, but the patch should reuse or extract the existing `resolveRuntimeCwdForAcpSpawn` behavior before passing inherited cwd into runtime initialization.
