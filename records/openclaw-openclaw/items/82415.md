## Summary

Keep open: current main still misses the documented `/acp spawn` cwd inheritance path, and this PR appears to be the focused fix with sufficient live proof and no blocking findings.

## What This Changes

The PR exports the ACP spawn cwd guard, applies target-agent workspace inheritance to `/acp spawn`, and adds regression tests for inherited and missing workspaces.

## Best Possible Solution

Land the command-path cwd inheritance fix after normal maintainer merge gates, preserving explicit `--cwd` precedence and the existing missing-path fallback contract.

## Reproduction Assessment

Yes. Source inspection shows `parseSpawnInput` leaves `cwd` undefined unless `--cwd` is supplied, and current `handleAcpSpawnAction` passes that undefined value to ACP initialization despite the documented target-workspace inheritance contract.

## Solution Assessment

Yes. Reusing `resolveSpawnedWorkspaceInheritance` and `resolveRuntimeCwdForAcpSpawn` in the command path is the narrowest maintainable fix and keeps command behavior aligned with direct ACP spawn.
