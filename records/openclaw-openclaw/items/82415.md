## Summary

Keep open: this targets a real `/acp spawn` command-path cwd bug, but the patch would bypass the existing missing-workspace fallback and the external PR still lacks required real behavior proof.

## What This Changes

The branch resolves `/acp spawn` cwd through `resolveSpawnedWorkspaceInheritance` and adds a command test for target-agent workspace inheritance.

## Best Possible Solution

Share the direct ACP cwd resolution guard with the `/acp spawn` command path, cover both existing and missing target workspace cases, then land after redacted real behavior proof is added.

## Reproduction Assessment

Yes. Source inspection shows `parseSpawnInput` only sets cwd from `--cwd` and `handleAcpSpawnAction` passes that optional value directly to ACP initialization, so `/acp spawn codex` without `--cwd` cannot supply the configured target workspace.

## Solution Assessment

No. The direction is right, but the patch should reuse or extract the existing `resolveRuntimeCwdForAcpSpawn` behavior so command-path inheritance preserves explicit cwd, missing-path fallback, and non-missing access errors.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

This slash path now passes the inherited config path straight to `initializeSession`, so stale agent workspace configs that currently stay on the backend default path can fail or diverge from `spawnAcpDirect` instead of matching it.
