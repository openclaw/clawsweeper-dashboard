## What This Changes

The PR changes the `/acp spawn` command path to resolve target-agent workspace inheritance before ACP runtime initialization and adds command-path regression tests.

## Best Possible Solution

Land the command-path cwd inheritance fix after the contributor adds redacted real behavior proof and switches the new temp-directory test to the platform temp directory.

## Reproduction Assessment

Yes, at source level. Current main parses `cwd` only from `--cwd` and then passes that optional value directly into ACP session initialization, so `/acp spawn <agent>` without `--cwd` cannot supply the target agent workspace despite the documented contract.

## Solution Assessment

Mostly yes. Reusing the existing direct ACP cwd guard and workspace inheritance helper is the narrow maintainable fix; the remaining issues are proof and a small portable-test cleanup, not a different design.
