The issue requests custom HTTP headers for CLI remote Gateway WebSocket connections via repeatable CLI flags, an environment variable, and `gateway.remote.headers` config.

## Best Possible Solution

Define one secret-safe Gateway header contract resolved beside remote URL/auth/TLS, forward it through `packages/gateway-client`, and document/test CLI, env, config, redaction, and precedence behavior together.
