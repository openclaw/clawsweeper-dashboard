## Best Possible Solution

## Reproduction Assessment

Not applicable for the feature request itself. The blocking edge is source-reproducible: with `followUsers` configured and an existing session in the same channel, the branch marks the guild follow-owned and can later leave that pre-existing session.

## Solution Assessment

No, not yet. The implementation follows existing Discord voice manager patterns, but it should preserve autoJoin/manual session ownership and still needs real Discord proof plus maintainer acceptance of the new config policy.
