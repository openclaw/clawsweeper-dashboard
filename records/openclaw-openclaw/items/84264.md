## Best Possible Solution

Preserve follow ownership across every internal Discord voice session rebuild/rejoin path, add a DAVE recovery regression for followed-user disconnect, then leave the opt-in voice-follow product decision to maintainers.

## Reproduction Assessment

Yes for the blocking PR finding: source inspection shows `recoverFromDecryptFailures()` still calls `leave()` and `join()` without `preserveFollowState` after this PR made plain `leave()` clear follow maps. This is not a current-main bug because follow-users is introduced by this PR.

## Solution Assessment

No: the feature direction is plausible and well covered, but the implementation must preserve follow ownership through DAVE recovery before merge. The safer path is a narrow code/test repair plus maintainer acceptance of the opt-in voice-follow behavior.
