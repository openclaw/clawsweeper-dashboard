## Summary

Keep open: the PR is useful and has live Discord proof, but the head still drops follow ownership during DAVE receive recovery, so a followed-user session can remain connected after the followed user disconnects.

## What This Changes

The PR adds Discord voice follow-users config, docs, metadata, and manager logic/tests for automatic join, move, leave, startup reconciliation, allowed-channel restrictions, and handoff.

## Best Possible Solution

Preserve follow ownership through every internal voice session rebuild that should remain follow-owned, add the recovery regression, then let maintainers accept or decline the opt-in voice-follow policy.

## Reproduction Assessment

Yes for the blocking PR finding: source inspection shows leave() clears follow maps by default and recoverFromDecryptFailures() calls it before rejoining without preserving follow state. This is not a current-main bug reproduction because follow-users is new in this PR.

## Solution Assessment

No, not yet: the feature direction is plausible, but the implementation needs to preserve follow ownership across recovery rebuilds before it is the narrow maintainable solution. The safer path is a focused code fix plus regression coverage for that recovery sequence.
