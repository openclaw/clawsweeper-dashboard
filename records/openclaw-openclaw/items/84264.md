## Summary

Keep open: this is a useful Discord voice feature with sufficient live recording proof, but the PR head still loses follow ownership during DAVE decrypt recovery because `leave` clears follow state and the recovery rejoin does not preserve it.

## What This Changes

This PR adds opt-in Discord voice `followUsers` configuration, follow/reconciliation voice-session logic, generated config metadata, docs, and e2e coverage.

## Best Possible Solution

Preserve followed-session ownership across every internal Discord voice rebuild/rejoin path, add a focused DAVE recovery regression, and then let maintainers decide whether the opt-in follow-users config is the desired product surface.

## Reproduction Assessment

Yes for the blocking review finding by source inspection: on the PR head, `recoverFromDecryptFailures` calls `leave` and `join` without preserving follow state while `leave` clears that state. I did not run tests in this read-only review.

## Solution Assessment

No, not yet: the feature direction is plausible, but the current patch preserves follow ownership for only some rebuild paths. The narrow maintainable fix is to carry follow ownership through DAVE recovery too and cover that combined path in e2e tests.
