## Summary

Keep open: the feature is useful and the supplied recording proof is sufficient for the common join/move/leave path, but the PR still drops follow ownership during DAVE receive recovery, so a followed voice session can remain connected after the followed user disconnects.

## What This Changes

Adds opt-in Discord voice `followUsers` configuration, Discord voice manager follow/reconcile logic, generated config metadata, docs, and e2e coverage.

## Best Possible Solution

Preserve followed-session ownership across every internal voice rebuild/rejoin path, add a focused DAVE recovery regression, and then let maintainers decide whether the new opt-in follow-users config is the desired product surface.

## Reproduction Assessment

Yes for the blocking review finding by source inspection: the PR head makes `leave` clear follow state by default, but DAVE recovery still calls `leave` and `join` without preserving follow state. I did not run tests in this read-only review.

## Solution Assessment

No: the overall feature direction is plausible, but the current patch only preserves ownership for some rebuild paths. The maintainable fix is to carry follow ownership through DAVE recovery too and cover that combined path in e2e tests.
