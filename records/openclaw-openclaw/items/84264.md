## Summary

Keep open: the Discord voice follow feature now has live proof and fixes the earlier handoff/reconciliation blockers, but source review found a remaining recovery path that can drop follow ownership and leave the bot connected after the followed user disconnects.

## What This Changes

The PR adds Discord `voice.followUsers` and `followUsersEnabled` config, docs, metadata, and voice-manager logic/tests for automatic join, move, leave, startup reconciliation, allowed channels, and handoff.

## Best Possible Solution

Preserve follow ownership through every internal session rebuild that should remain follow-owned, add an e2e regression for recovery rejoin followed by user disconnect, then let maintainers decide whether the new opt-in Discord voice policy belongs in the channel plugin.

## Reproduction Assessment

Yes for the blocking PR finding: source inspection shows `leave()` clears follow maps by default and `recoverFromDecryptFailures()` calls it without preserving follow state before rejoining. This is not a current-main bug reproduction because the follow-users feature is new in this PR.

## Solution Assessment

No, not yet: the feature direction is plausible, but the implementation needs to preserve follow ownership across recovery rebuilds before it is the narrow maintainable solution. The safer path is a focused code fix plus regression coverage for that recovery sequence.
