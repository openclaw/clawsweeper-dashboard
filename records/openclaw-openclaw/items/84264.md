## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.81

Full review comments:

- **[P2] Include startup guilds in follow reconciliation:** `extensions/discord/src/voice/manager.ts:918-923`
  - body: `runFollowedUsersReconcile` bails out whenever `resolveFollowGuildIds()` is empty, but the new docs show a valid `followUsers` config with no `guilds`, `autoJoin`, or `allowedChannels`. In that setup the startup check never calls `getGuildVoiceState`, so a configured user who is already in voice when the gateway starts will not be followed until they generate a later voice-state event. Use READY/known guild data or require and document a guild source, and add a regression for the documented config shape.
  - confidence: 0.87
- **[P2] Preserve other followed users before leaving:** `extensions/discord/src/voice/manager.ts:817-821`
  - body: When one followed user disconnects, this calls `leave`, and `leave` clears all followed-user channel state for the guild. If another configured followed user is still tracked in a different voice channel, the bot disconnects and forgets that remaining target instead of moving back to them. Keep the remaining tracked user/channel or choose another allowed follow target before clearing guild state.
  - confidence: 0.82
