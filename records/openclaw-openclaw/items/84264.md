Full review comments:

- **[P2] Preserve a different followed user before leaving:** `extensions/discord/src/voice/manager.ts:828-832`
  - body: When one followed user disconnects, this path leaves if no remaining followed user is in the current channel. If another configured user is still tracked in a different allowed channel in the same guild, `leave()` clears that guild’s follow state, so the bot stops following the remaining user instead of handing off to that user’s channel.
  - confidence: 0.86
- **[P2] Bound follow-user reconciliation polling:** `extensions/discord/src/voice/manager.ts:944-947`
  - body: The interval reconciliation does one REST voice-state lookup per followed user per resolved guild every 10 seconds. With several configured users and guilds this becomes a persistent N*M Discord REST loop with no cap or backoff; please make reconciliation bounded or event-led so an opt-in config cannot create rate-limit pressure.
  - confidence: 0.78
