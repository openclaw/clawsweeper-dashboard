## Best Possible Solution

Preserve follow ownership through every internal session rebuild path, especially DAVE recovery, keep the feature opt-in and bounded by allowed voice channels, then land after conflict resolution and focused Discord voice proof.

Yes, source-reproducible for the review finding: a follow-owned session that hits recoverFromDecryptFailures() calls leave()/join() without preserveFollowState, clearing the maps the disconnect path relies on. I did not run tests because this review is read-only.

## Solution Assessment
