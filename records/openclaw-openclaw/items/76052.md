## Summary

Keep open. The PR fixes a real restart-recovery gap for topic-suffixed transcript locks, but the exact-path rewrite drops a current-main fallback for same-directory stale generated sessionFile metadata, and the user-facing fix still needs a changelog entry.

## What This Changes

The PR changes main-session restart recovery to match cleaned transcript locks by normalized transcript lock paths and adds regression tests for topic-suffixed, relative, unrelated, and stale sessionFile cases.

## Best Possible Solution

Keep the exact lock-path approach for topic transcripts, but build the accepted lock-path set from the same stale-metadata fallback rules used by transcript readers and add the missing changelog entry.

## Reproduction Assessment

Yes. The PR’s main bug is reproducible by a running main entry with sessionId SESSION, sessionFile SESSION-topic-THREAD.jsonl, and a cleaned lock for that topic file; current main will not mark it. The PR regression is reproducible by a current sessionId with a stale generated sessionFile for another session id under the same sessions directory and a cleaned lock for the current sessionId transcript.

## Solution Assessment

No. Path comparison is the right direction, but this implementation should compare against both the valid persisted transcript lock and the canonical sessionId fallback/stale candidate instead of relying on resolveSessionFilePath alone.
