## Summary

Keep this open. Current main still runs Nextcloud Talk `requireMention` gating against the raw webhook body, while the active open fix PR https://github.com/openclaw/openclaw/pull/79397 covers this exact structured mention path but has not merged.

## What This Changes

The issue requests Nextcloud Talk rich-object mention placeholders such as `{mention-user1}` be resolved so `requireMention: true` works in group rooms.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/79397 with a plugin-local parser that uses structured Nextcloud Talk identifiers for mention gating while preserving plain-text fallback behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to send a signed Nextcloud Talk Create webhook for an allowlisted group room with `requireMention: true` and `object.content` containing structured JSON like `{"message":"hi {mention-user1}","parameters":{...}}`; current main passes that raw JSON string to mention matching.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Nextcloud Talk plugin: parse the structured body locally, compare explicit user mention identifiers against the configured bot identity, and keep the existing fallback for plain text.
