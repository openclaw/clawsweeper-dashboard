## Summary

Keep this open: current main still feeds raw Nextcloud Talk webhook content into mention gating, and the exact structured mention fix is covered by an active open PR that has not merged.

## What This Changes

The issue asks the Nextcloud Talk plugin to resolve structured `{mention-userN}` webhook placeholders against bot identity before `requireMention` gating.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/79397 with a plugin-local structured payload parser that uses stable Nextcloud Talk mention identifiers while preserving plain-text fallback behavior.

## Reproduction Assessment

Yes. A high-confidence source-level path is a signed Nextcloud Talk Create webhook for an allowlisted group room with `requireMention: true` and `object.content` containing structured JSON such as `{"message":"hi {mention-user1}","parameters":{...}}`; current main passes that raw JSON string to mention matching.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Nextcloud Talk plugin: decode structured webhook bodies locally, compare stable mention identifiers against the configured bot account, and keep the existing plain-text fallback.
