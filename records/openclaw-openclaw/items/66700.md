## Summary

Keep this open. Current main still treats Nextcloud Talk webhook content as opaque text and runs group mention gating against that raw string, so `{mention-userN}` rich-object placeholders cannot reliably satisfy `requireMention: true`; the related rich-object attachment PR is open and does not close this mention-gating bug.

## What This Changes

The issue asks the bundled Nextcloud Talk plugin to resolve Nextcloud rich-object mention placeholders before applying room mention detection.

## Best Possible Solution

Add Nextcloud Talk plugin-local rich-object parsing that preserves ordinary text behavior, resolves placeholder display text, carries explicit mention facts or resolvable parameter metadata into inbound handling, and makes the room mention gate accept placeholders that identify the configured bot while rejecting placeholders for other users.

## Reproduction Assessment

Yes. A source-level reproduction is clear: send a signed `Create` webhook for an allowlisted group room with `object.content` set to a JSON string like `{"message":"hi {mention-user1}","parameters":{...}}` and `requireMention: true`; current main checks the raw JSON string with regex mention patterns and drops it as unmentioned.

## Solution Assessment

Yes. The maintainable fix belongs inside `extensions/nextcloud-talk`: parse Nextcloud rich-object content and feed resolved text or explicit mention evidence into the existing mention-gating contract, instead of disabling `requireMention` or adding core special cases.
