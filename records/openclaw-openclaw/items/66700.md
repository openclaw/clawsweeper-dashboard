## Summary

Keep this open. Current main and the latest release still parse Nextcloud Talk webhook `object.content` as an opaque string and run group mention gating against that raw body, so rich-object placeholders such as `{mention-user1}` can still fail `requireMention: true`; the related rich-object attachment PR is useful adjacent work but is open and does not close this mention-gating bug.

## What This Changes

The issue requests Nextcloud Talk rich-object mention placeholders to satisfy the plugin's group `requireMention` gate when they refer to the bot.

## Best Possible Solution

Implement plugin-local rich-object parsing in `extensions/nextcloud-talk`, preserve plain-text fallback behavior, carry explicit/resolved mention facts into the inbound gate, and add regression coverage for bot mentions and non-bot placeholders.

## Reproduction Assessment

Yes. A source-level reproduction is clear: send a signed Create webhook for an allowlisted group room with `requireMention: true` and `object.content` set to rich-object JSON containing `hi {mention-user1}`; current main forwards that raw JSON string to regex mention matching and can drop the message as unmentioned.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Nextcloud Talk plugin using the existing runtime mention helpers, not in core and not by disabling `requireMention`.
