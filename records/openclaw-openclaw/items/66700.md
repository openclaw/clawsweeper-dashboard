## Summary

Keep this open: current main and the latest release still pass raw Nextcloud Talk `object.content` into mention gating, while the active structured-mention fix is open and unmerged.

## What This Changes

The issue asks Nextcloud Talk to resolve structured `{mention-userN}` webhook placeholders against the bot account before `requireMention` gating.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/79397 with a plugin-local structured payload parser that preserves plain-text fallback and coordinates with the rich-object media parser work.

## Reproduction Assessment

Yes. A high-confidence source path is a signed Nextcloud Talk `Create` webhook for an allowlisted group room with `requireMention: true` and `object.content` containing structured JSON such as `{"message":"hi {mention-user1}","parameters":{...}}`; current main feeds that raw string into mention matching.

## Solution Assessment

Yes. The maintainable fix belongs in the Nextcloud Talk plugin: decode structured webhook bodies locally, compare stable mention identifiers against the configured bot account, and keep the existing fallback behavior.
