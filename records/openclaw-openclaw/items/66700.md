## Summary

Keep this open: current main and the latest release still gate Nextcloud Talk mentions against the raw structured webhook body, and the exact bug is covered by an open but conflicting implementation PR that needs maintainer review rather than issue cleanup.

## What This Changes

The issue reports that Nextcloud Talk group rooms with `requireMention: true` miss bot mentions because webhook text contains `{mention-user1}` placeholders inside a structured JSON body instead of literal usernames.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/79397 with a plugin-local structured webhook parser, coordinating with https://github.com/openclaw/openclaw/pull/49869 so Nextcloud Talk uses one maintainable rich-object parsing path.

## Reproduction Assessment

Yes. A high-confidence source path is a signed Nextcloud Talk `Create` webhook for an allowlisted room with `requireMention: true` and structured `object.content` containing `{mention-user1}`; current main feeds that raw string into mention matching.

## Solution Assessment

Yes, the maintainable fix belongs in the Nextcloud Talk plugin: parse the structured body, compare stable mention identifiers against the configured bot account, preserve plain-text fallback, and coordinate the shared rich-object parser boundary.
