## Summary

Keep this open: current main and the latest release still feed the raw Nextcloud Talk structured webhook body into mention gating, and the exact fix path is already covered by an open implementation PR that needs maintainer review rather than issue cleanup.

## What This Changes

The issue reports that Nextcloud Talk group-room mention gating fails because webhook bodies contain structured `{mention-user1}` placeholders instead of literal bot mention text.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/79397 with a plugin-local structured webhook parser that preserves plain-text fallback and coordinates with the rich-object media parser work.

## Reproduction Assessment

Yes. A high-confidence source path is a signed Nextcloud Talk `Create` webhook for an allowlisted group room with `requireMention: true` and structured `object.content` containing a `{mention-user1}` placeholder; current main feeds that raw JSON string into mention matching.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Nextcloud Talk plugin: decode Talk's structured body locally, compare stable mention identifiers against the configured bot account, and keep existing plain-text fallback behavior.
