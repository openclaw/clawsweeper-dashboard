## Summary

Keep open: current `main` and the latest release still have no Discord trusted-webhook allowlist or webhook-to-user mapping, and the requested behavior is a new security-sensitive Discord plugin feature rather than a stale bug report.

## What This Changes

Add a Discord `trustedWebhooks` guild config that lets selected webhook-origin messages trigger agent responses without enabling all bot-authored messages.

## Best Possible Solution

If maintainers accept the feature, implement it inside the Discord plugin with typed config/schema/docs, default-deny scoped trust, mapped-user validation against existing access policy, and focused preflight tests for existing bot safeguards.

Unclear until maintainers choose the identity and trust model. A Discord-plugin allowlist is plausible, but `mapToUser` needs explicit product and security guardrails before implementation.
