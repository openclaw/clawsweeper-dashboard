## Summary

Keep open: current main and the latest shipped tag still lack a Discord trusted-webhook allowlist or webhook-to-user mapping, and the existing `allowBots` modes do not cover the requested unmentioned, allowlisted webhook automation path. ## What This Changes

Requested feature: add a Discord trusted-webhook allowlist, optionally mapping trusted webhook messages to a configured user, so selected webhook-origin messages can trigger agent responses without enabling all bots.

## Best Possible Solution

If maintainers accept the feature, implement it inside the Discord plugin with typed config/schema/docs, default-deny scoped trust, mapped-user validation against existing access policy, and focused preflight tests for existing bot safeguards.
