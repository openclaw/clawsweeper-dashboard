## Summary

Keep open: current main still has only global or mention-based Discord bot gating and no guild-level `trustedWebhooks` config, while the requested `mapToUser` behavior is a new security-sensitive identity surface that needs maintainer product/security review.

## What This Changes

Request adds a guild-level Discord `trustedWebhooks` allowlist with optional user mapping so selected Discord webhook messages can trigger agent responses.

## Best Possible Solution

If accepted, implement this inside the Discord plugin with typed config/schema/docs/tests, default-deny webhook trust, constrained user mapping through existing access checks, and explicit loop/echo safeguards.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives high confidence that the requested config and trust branch do not exist, and current webhook-origin bot messages are governed by `allowBots` and PluralKit behavior.

## Solution Assessment

Unclear until maintainers choose the identity model. A Discord-plugin allowlist is plausible, but `mapToUser` and its interaction with bot-loop and webhook-echo guards need explicit product/security guardrails first.

## Security Review

Status: needs_attention

  - body: `mapToUser` would let webhook-origin messages inherit a configured user's permissions or conversation context, so an implementation should require explicit guild/account allowlist membership and avoid bypassing existing access policy.
  - confidence: 0.84
- **[medium] Scope webhook trust around bot safeguards:** `extensions/discord/src/monitor/message-handler.preflight.ts:143`
  - body: A webhook allowlist must deliberately interact with PluralKit resolution, bound-thread webhook echo suppression, global `allowBots`, and bot-loop protection so it does not reintroduce loops or unintended inbound turns.
  - confidence: 0.82

  - body: Discord webhook IDs can identify a webhook in gateway events, but the trust decision should still be scoped to the expected guild/channel/account context rather than treated as a general authentication secret.
  - confidence: 0.73
