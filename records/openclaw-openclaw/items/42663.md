## Summary

Keep open: current main still only has global Discord `allowBots` handling plus the PluralKit special case, while the requested per-webhook trust model would add a new Discord config and identity-policy surface.

## What This Changes

Add a Discord guild-level `trustedWebhooks` allowlist so selected webhook-authored messages can trigger agent responses without enabling all bot-authored messages.

## Best Possible Solution

If accepted, implement this inside the Discord plugin with schema/types/docs/tests, preserving the default bot-message drop and making webhook trust explicit and auditable.

## Reproduction Assessment

Not applicable as a strict bug reproduction. Source and tests give a high-confidence current-main path: non-PluralKit webhook messages are bot-authored and are dropped unless `allowBots` is enabled globally or mention-gated.

## Solution Assessment

Unclear until maintainer product/security review settles the identity model. A Discord plugin-local allowlist is plausible, but the `mapToUser` semantics need a safer explicit design before implementation.
