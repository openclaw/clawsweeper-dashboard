## Summary

Close: current main already defaults `openclaw logs` text timestamps to local time, keeps UTC available through `--utc`, and the merged fix shipped after https://github.com/openclaw/openclaw/pull/85387.

## What This Changes

The issue asks for `openclaw logs` to render timestamps in the local timezone instead of UTC by default.

## Best Possible Solution

Keep the shipped logs behavior: local timestamps by default with `--utc` for operators or scripts that need UTC, and open a new issue only for any separate current-release console or Docker case that still fails.

## Reproduction Assessment

Yes for the pre-fix behavior: the report gives a concrete non-UTC setup, and current tests model non-UTC output with `TZ=America/New_York`. Current main no longer reproduces the UTC-default behavior because the CLI now defaults to local output.

## Solution Assessment

Yes. The merged solution changes the display default at the CLI formatting boundary, preserves `--local-time`, and adds `--utc` for callers that need the old UTC rendering.
