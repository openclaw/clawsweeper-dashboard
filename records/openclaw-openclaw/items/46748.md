## Summary

Keep open. Current main still makes local timestamp rendering opt-in for human `openclaw logs` output, and open PR #47365 already targets this issue with closing syntax.

## What This Changes

Request that human-readable `openclaw logs` timestamps display in the system local timezone by default instead of UTC.

## Best Possible Solution

Review and land PR #47365 or a narrow replacement that defaults human-readable logs to local time, preserves JSON/raw tooling stability, and provides an explicit UTC opt-out.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: run `openclaw logs --plain` without `--local-time` and without a valid `TZ`; current main leaves `localTime` false and formats the timestamp with `toISOString()`.

## Solution Assessment

Yes for the direction, with maintainer review needed on the exact flag shape. Defaulting only human-readable log output to local time is the narrow path, while raw JSON/tooling output should remain stable.
