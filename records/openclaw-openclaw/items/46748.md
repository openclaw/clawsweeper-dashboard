## Summary

Keep open. Current main and the latest release still default human `openclaw logs` output to UTC unless `--local-time` or a valid `TZ` is present, and an open closing PR already covers the requested default-change path.

## What This Changes

Make `openclaw logs` display local-time timestamps by default instead of UTC, while preserving an explicit UTC path for users and tooling.

## Best Possible Solution

Land one coordinated logs CLI update that defaults human-readable text output to local time, keeps raw JSON stable, preserves `--local-time` as an accepted compatibility alias, and provides an explicit UTC opt-out.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: run `openclaw logs --plain` without `--local-time` and without a valid `TZ`; current main leaves `localTime` false and formats with `toISOString()`.

## Solution Assessment

Yes for the requested direction, but it is compatibility-sensitive. The parser already prefers the local `time` field, so the maintainable fix is to change the human formatter default while preserving raw JSON/tooling output and an explicit UTC escape hatch.
