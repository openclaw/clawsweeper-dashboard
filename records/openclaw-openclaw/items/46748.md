## Summary

Keep this issue open: current main and v2026.5.12 still render human-readable `openclaw logs` timestamps as UTC by default unless local-time mode is explicitly enabled, and https://github.com/openclaw/openclaw/pull/47365 is already the active closing PR.

## What This Changes

The issue requests that `openclaw logs` display timestamps in the system local timezone by default instead of UTC.

## Best Possible Solution

Review, land, or replace https://github.com/openclaw/openclaw/pull/47365 as a focused logs CLI change that defaults human-readable output to local time, preserves a documented UTC opt-out, updates tests/docs, and handles console logging separately if needed.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.5.12` parse the local-offset `time` field but render human-readable output as UTC unless `--local-time` or a valid `TZ` enables local mode.

## Solution Assessment

Yes for keeping this issue open. The active closing PR or a logs-only replacement is the right path, but the default change, UTC opt-out, docs/tests, and proof need maintainer review before this issue can close.
