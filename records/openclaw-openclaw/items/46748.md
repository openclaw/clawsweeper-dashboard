## Summary

Current `main` and the latest release already make `openclaw logs` render local-time timestamps by default, preserve `--local-time`, and provide `--utc` for UTC output, so this issue can close as implemented.

## What This Changes

The issue asks for `openclaw logs` to display local-time timestamps by default instead of UTC timestamps.

## Best Possible Solution

Keep the shipped local-time default with `--utc` as the explicit opt-out; any further timezone preference knob should be tracked separately as a product decision.

## Reproduction Assessment

Yes. The source path and tests show the historical UTC/default-local mismatch and current `main` now covers the expected local-time default; no live command was run because this review was read-only.

## Solution Assessment

Yes. The shipped implementation is the narrow fix: CLI display defaults to local time, `--local-time` remains accepted, and `--utc` preserves the old UTC view for operators who need it.
