## Summary

Keep this issue open: current main and the latest release still default human-readable `openclaw logs` output to UTC unless local-time mode is explicitly enabled, and https://github.com/openclaw/openclaw/pull/47365 is already the active closing PR.

## What This Changes

The issue asks for human-readable `openclaw logs` timestamps to default to local time instead of UTC when local-offset log fields are present.

## Best Possible Solution

Land or replace the linked logs-only PR so default human-readable logs use local time, UTC opt-out behavior is documented and tested, and console logging scope is handled separately if needed.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.5.12` parse the local-offset `time` field but render human-readable output as UTC unless `--local-time` or a valid `TZ` enables local mode.

## Solution Assessment

Yes for keeping this issue open. The active closing PR is the right implementation path, but the changed default and UTC opt-out contract should be reviewed before landing.
