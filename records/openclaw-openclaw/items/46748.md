## Summary

Current main and v2026.6.8 already render `openclaw logs` timestamps in the local timezone by default, keep `--local-time` accepted, and provide `--utc` for UTC output; the parser also prefers the top-level `time` field before `_meta.date`, which solves the central report.

## What This Changes

The issue asks for `openclaw logs` to display log timestamps in the user's local timezone instead of UTC by default.

## Best Possible Solution

Keep the shipped v2026.6.8 local-default logs behavior with `--utc` as the explicit UTC opt-in.
