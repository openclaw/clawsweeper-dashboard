## Summary

Keep open. Current main and the latest release still keep local-time rendering opt-in for human `openclaw logs` output, and two open PRs with closing syntax are already paired to the issue, so maintainers should review or consolidate those instead of closing it now.

## What This Changes

Request that `openclaw logs` display local-time timestamps by default instead of UTC when showing human-readable log output.

## Best Possible Solution

Review and consolidate #46782 and #47365, then land one focused fix that makes human-readable `openclaw logs` output local time by default while preserving raw JSON/tooling output, existing `--local-time` compatibility, and a clear UTC opt-out.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: run `openclaw logs --plain` without `--local-time` and without a valid `TZ`; current main leaves local-time mode false and formats through the UTC `toISOString()` path.

## Solution Assessment

Yes. Defaulting human-readable log output to local time with an explicit UTC opt-out is the narrow maintainable direction; JSON output and log file structure should stay stable for tooling.
