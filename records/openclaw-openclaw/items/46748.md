## Summary

Close: the local-time default for human-readable `openclaw logs` output is implemented on current `main` and shipped in `v2026.6.10`, with `--utc` retained as the explicit UTC opt-in.

## What This Changes

The issue reports that `openclaw logs` rendered UTC timestamps by default and asks for local-time display by default with a UTC opt-out.

## Best Possible Solution

Keep the shipped local-default logs behavior, keep `--utc` for operators who need UTC output, and route separate Control UI timezone configuration work to https://github.com/openclaw/openclaw/issues/90314.

## Reproduction Assessment

Yes, as a historical/source reproduction: `v2026.3.13-1` defaulted `formatLogTimestamp` to UTC unless `--local-time` or a valid `TZ` was set, while current source and tests show local time by default. I did not run a live gateway command in this read-only cleanup review.

## Solution Assessment

Yes. The shipped solution is narrower than adding a new config surface: it changes the human-readable logs CLI default, preserves explicit UTC via `--utc`, and aligns tests and docs.
