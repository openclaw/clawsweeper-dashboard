## Summary

Current main and the latest tagged release already implement the requested behavior: `openclaw logs` renders local-time timestamps by default, reads the top-level local `time` field before falling back to `_meta.date`, and keeps an explicit UTC opt-in.

## What This Changes

The issue requests that `openclaw logs` display log timestamps in the system local timezone instead of defaulting to UTC.

## Best Possible Solution

Keep the shipped local-default CLI behavior with `--utc` as the explicit opt-in for UTC; broader Control UI timezone configuration remains separate at https://github.com/openclaw/openclaw/issues/90314.

## Reproduction Assessment

No failing current-main reproduction remains. Source inspection and adjacent tests show the CLI now defaults to local timestamp rendering and only emits UTC when `--utc` is used.

## Solution Assessment

Yes. The current solution fixes the existing CLI path directly, preserves the compatibility spelling, and avoids adding a new timezone config surface for this bounded display bug.
