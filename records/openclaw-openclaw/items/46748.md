## Summary

Keep open: current `main` and latest release `v2026.5.12` still render human-readable `openclaw logs` timestamps as UTC by default unless `--local-time` or a valid `TZ` enables local rendering, and an open closing PR still targets this issue.

## What This Changes

The issue asks for `openclaw logs` to show local-time timestamps by default instead of UTC.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/47365 with a focused logs CLI change that defaults human-readable output to local time, documents and tests the UTC opt-out, and keeps the raw `_meta.date` mismatch on https://github.com/openclaw/openclaw/issues/40220 separate.

## Reproduction Assessment

Yes, at source level. Current main parses the local-offset `time` field, but `formatLogTimestamp` returns UTC unless `--local-time` or a valid `TZ` enables local rendering.

## Solution Assessment

Yes for keeping this issue open. The active closing PR is the right implementation path to review, but the changed default and UTC opt-out contract need maintainer acceptance before the issue closes.
