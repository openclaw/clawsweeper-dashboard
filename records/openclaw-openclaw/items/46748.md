## Summary

Keep open: current main and v2026.5.22 still render `openclaw logs` timestamps as UTC by default unless `--local-time` or a valid `TZ` enables local rendering, and the latest user comment confirms that remains observable without `TZ`. The remaining work is a maintainer decision about changing a shipped CLI output default and preserving a clear UTC opt-out.

## What This Changes

Request that `openclaw logs` render timestamps in the system local timezone by default instead of UTC, with a documented UTC opt-out path.

## Best Possible Solution

Make a maintainer-approved logs CLI contract change that defaults human-readable output to local time, preserves a documented UTC opt-out, updates focused tests and docs, and leaves https://github.com/openclaw/openclaw/issues/40220 to track `_meta.date` versus `time` consistency.

## Reproduction Assessment

Yes, at source level. Current main parses the local-offset `time` field, but `formatLogTimestamp` returns UTC unless local-time mode is enabled by `--local-time` or a valid `TZ`.

## Solution Assessment

Unclear until maintainers choose the CLI timestamp contract. The code change is small, but default-local output plus a UTC opt-out needs maintainer acceptance because it changes shipped human and script-visible output.
