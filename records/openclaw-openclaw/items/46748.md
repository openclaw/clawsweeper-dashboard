## Summary

Current main already defaults `openclaw logs` text output to local timestamps, keeps `--utc` as the explicit UTC opt-out, and this shipped in v2026.5.26.

## What This Changes

The issue asks for `openclaw logs` to render human-readable timestamps in the local timezone instead of showing UTC by default.

## Best Possible Solution

Keep the shipped local-default CLI rendering with `--utc` for operators who need UTC, and treat any future config-driven log timezone control as a separate product request.

## Reproduction Assessment

No failing current-main reproduction remains. Source and regression tests show `openclaw logs --plain` now emits local-offset timestamps by default and uses UTC only when `--utc` is passed.

## Solution Assessment

Yes. Defaulting human-readable CLI logs to local time while preserving an explicit `--utc` opt-out fixes the reported confusion without adding a new config surface.
