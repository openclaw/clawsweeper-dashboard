## Summary

Keep open. Current main still defaults human `openclaw logs` output to UTC unless `--local-time` or a valid `TZ` is present, and open PR #47365 is already paired to this issue with closing syntax.

## What This Changes

The issue requests that `openclaw logs` show local timezone timestamps by default instead of UTC.

## Best Possible Solution

Review and land PR #47365 or a narrow replacement that makes human-readable CLI logs local by default while preserving raw JSON/tooling output and an explicit UTC opt-out.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: run `openclaw logs --plain` without `--local-time` and without a valid `TZ`; current main leaves `localTime` false and formats with `toISOString()`.

## Solution Assessment

Yes for the direction, with review needed on the exact flag shape. Defaulting only human-readable logs display to local time is the narrow maintainable path; raw log records and JSON output should stay stable for tooling.
