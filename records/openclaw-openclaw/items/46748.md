## Summary

Keep open. Current main and the latest release still render human `openclaw logs` output as UTC unless `--local-time` is passed or `TZ` is valid, and open closing PRs #46782 and #47365 are already paired to this issue.

## What This Changes

The issue requests that `openclaw logs` render local-time timestamps by default instead of UTC `Z` timestamps.

## Best Possible Solution

Land one focused CLI fix that makes human-readable `openclaw logs` output local time by default with an explicit UTC opt-out, while preserving raw JSON/tooling output.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: run `openclaw logs --plain` without `--local-time` and without a valid `TZ`; current main leaves `localTime` false and formats through `toISOString()`.

## Solution Assessment

Yes. Defaulting only the human-readable logs display to local time is the narrow maintainable direction; raw log fields and JSON output should stay stable for tooling.
