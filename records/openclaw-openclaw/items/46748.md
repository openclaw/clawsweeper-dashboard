## Summary

Keep open: current main and the latest release still render `openclaw logs` timestamps as UTC by default unless `--local-time` or a valid `TZ` enables local rendering. The parser already prefers the local-offset `time` field, so the remaining decision is whether human CLI output should default to local time and how to preserve an explicit UTC path.

## What This Changes

The issue asks for `openclaw logs` to display local timezone timestamps by default instead of UTC for non-UTC users.

## Best Possible Solution

Make a maintainer-approved logs CLI contract change that defaults human-readable output to local time, preserves a documented UTC opt-out, updates tests/docs, and leaves https://github.com/openclaw/openclaw/issues/40220 to track `_meta.date` versus `time` consistency.

## Reproduction Assessment

Yes, at source level. Current main parses the local-offset `time` field, but `formatLogTimestamp` returns UTC unless local-time mode is enabled by `--local-time` or a valid `TZ`.

## Solution Assessment

Yes for keeping this issue open. The requested default change is small in code, but the UTC opt-out and compatibility contract need maintainer acceptance before implementation.
