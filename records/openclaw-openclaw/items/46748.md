## Summary

Keep open: current `main` and latest release still render `openclaw logs` timestamps as UTC by default unless `--local-time` or a valid `TZ` enables local rendering, and an open closing PR already owns the default-change proposal.

## What This Changes

The issue requests that `openclaw logs` display local timezone timestamps by default instead of UTC timestamps.

## Best Possible Solution

Review, land, or replace https://github.com/openclaw/openclaw/pull/47365 with a focused logs CLI change that defaults human-readable output to local time, documents and tests the UTC opt-out, and keeps https://github.com/openclaw/openclaw/issues/40220 separate.

## Reproduction Assessment

Yes, at source level. Current `main` parses the local-offset `time` field, but `formatLogTimestamp` returns UTC unless local-time mode is enabled by `--local-time` or a valid `TZ`.

## Solution Assessment

Yes for keeping this issue open. The active closing PR is the right implementation path to review, while the default change and UTC opt-out contract still need maintainer acceptance.
