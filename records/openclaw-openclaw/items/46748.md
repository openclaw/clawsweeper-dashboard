## Summary

Keep open: current main and the latest release still render `openclaw logs` timestamps as UTC by default unless `--local-time` or a valid `TZ` enables local rendering, and an open closing PR already owns the proposed default change.

## What This Changes

The issue asks for `openclaw logs` to render local timezone timestamps by default instead of UTC.

## Best Possible Solution

Review, land, or replace https://github.com/openclaw/openclaw/pull/47365 with a focused logs CLI change that defaults human-readable output to local time, documents and tests the UTC opt-out, and keeps https://github.com/openclaw/openclaw/issues/40220 separate.

## Reproduction Assessment

Yes, at source level. Current main parses the local-offset `time` field, but `formatLogTimestamp` returns UTC unless local-time mode is enabled by `--local-time` or a valid `TZ`.

## Solution Assessment

Yes for keeping this issue open. The active closing PR is the right implementation path to review, while the shipped CLI default change and UTC opt-out contract still need maintainer acceptance.
