## Summary

Keep open: current main still contains raw Feishu media fallback text, and an open linked fix PR should be reviewed before this issue closes.

## What This Changes

The issue asks to sanitize Feishu media fallback text so failed uploads or fetches do not expose local paths, file URLs, relative references, loopback/private URLs, or DNS-private media URLs to recipients.

## Best Possible Solution

Land the linked Feishu fix, or an equivalent narrow Feishu-owned fallback sanitizer that preserves captions while suppressing unsafe media references, then close this issue after merge.

## Reproduction Assessment

Yes. Source inspection shows `sendMediaFeishu` can reject unsafe or failed media loads, and Feishu fallback code currently inserts the raw `mediaUrl` into user-visible text after that failure.

## Solution Assessment

Yes. The maintainable fix is to keep successful upload behavior intact while centralizing Feishu fallback text sanitization across outbound and reply-dispatcher paths.
