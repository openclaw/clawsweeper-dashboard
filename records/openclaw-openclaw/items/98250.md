## Summary

Keep open: current main and v2026.6.11 still expose raw Feishu media references in fallback text, and a linked open PR already owns the fix path.

## What This Changes

The issue requests sanitizing Feishu media failure fallback text so local/private media references are not sent to recipients while safe public URL fallbacks and captions are preserved.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/98251 or an equivalent Feishu-owned fallback sanitizer that preserves captions, allows only SSRF-validated public http(s) fallback links, and suppresses local/private references; close this issue after merge.

## Reproduction Assessment

Yes. Source inspection on current main shows media send failures can reach fallback code that interpolates raw `mediaUrl` into Feishu text/comment replies after `loadWebMedia` rejects blocked or local media.

## Solution Assessment

Yes. The narrowest maintainable end state is a Feishu-owned fallback sanitizer reused across outbound and reply-dispatcher paths, not a core media loader change or new config.
