## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Scope route leases beyond raw runId:** `src/tasks/task-route-lease.ts:140-146`
  - body: Current main deliberately scopes same-runId task operations by runtime and session facts, but this PR stores and resolves route leases with `ON CONFLICT(run_id)` and `getActiveTaskRouteLease(runId)`. A second active task with the same raw runId can replace or expose another task's requester origin, so include the task-registry scope tuple or narrow this to a cron-only identity and add cross-scope collision coverage.
  - confidence: 0.93
- **[P2] Delete route leases with task cleanup:** `src/tasks/task-route-lease.ts:324-336`
  - body: The new GC helper only marks active expired rows as expired, while settled, retired, expired, and orphaned route rows stay in `task_route_leases`; current task deletion only removes `task_delivery_state` and `task_runs`. Add a bounded production deletion owner, preferably in the task cleanup transaction, with regression coverage.
  - confidence: 0.9
- **[P3] Clear settled_at when re-acquiring a lease:** `src/tasks/task-route-lease.ts:140-146`
  - body: Re-acquire makes a terminal lease active again, but the conflict update does not clear `settled_at`, so callers can receive an active lease carrying stale terminal metadata. Add `settled_at: null` to the conflict update and assert the field is absent after re-acquire.
  - confidence: 0.84
