## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Scope route leases beyond raw runId:** `src/tasks/task-route-lease.ts:140-146`
  - body: Current main deliberately allows the same raw runId to exist in multiple task scopes. This PR keys, reads, updates, and settles `task_route_leases` by `run_id` alone, so one runtime/session scope can overwrite or consume another scope's requester origin and deliver completion output to the wrong target. Rebase onto the scoped lease API or include the same runtime/scopeKind/ownerKey/childSessionKey tuple here with cross-scope coverage.
  - confidence: 0.92
- **[P2] Delete route leases when tasks are pruned:** `src/tasks/task-route-lease.ts:324-336`
  - body: The added GC helper only marks active expired rows as expired; settled, retired, expired, and orphaned rows are never hard-deleted, and current task cleanup deletes only `task_delivery_state` and `task_runs`. Add a task-delete cascade or another bounded production cleanup path so durable requester-route rows do not accumulate indefinitely.
  - confidence: 0.86
- **[P3] Clear settled_at on lease re-acquire:** `src/tasks/task-route-lease.ts:140-146`
  - body: The conflict update can make a settled row active again, but it does not clear the old `settled_at` value. Reset `settled_at` to null during re-acquire and add a regression assertion for active-after-reacquire leases so callers do not see stale terminal metadata on an active lease.
  - confidence: 0.82
