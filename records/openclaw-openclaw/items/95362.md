## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Scope route leases beyond raw runId:** `src/tasks/task-route-lease.ts:140-146`
  - body: Current main deliberately allows the same raw runId to exist in multiple task scopes. This PR keys, reads, updates, and settles task-route leases by run_id alone, so one runtime/session scope can overwrite or consume another scope's requester origin and deliver completion output to the wrong target. Rebase onto the scoped lease API or include the same runtime/scopeKind/ownerKey/childSessionKey tuple here with cross-scope coverage.
  - confidence: 0.92
- **[P2] Delete route leases when tasks are pruned:** `src/tasks/task-route-lease.ts:324-336`
  - body: The added GC helper only marks active expired rows as expired; settled, retired, expired, and orphaned rows are never hard-deleted when task records are pruned. Because createRunningTaskRun writes a lease for each eligible detached task, normal task cleanup can leave durable requester-origin rows in shared SQLite indefinitely. Add task-delete cascade cleanup through the lease module and cover it with the real task deletion path.
  - confidence: 0.88
- **[P3] Clear settled_at on lease re-acquire:** `src/tasks/task-route-lease.ts:140-146`
  - body: Re-acquiring an existing runId updates status back to active but does not clear settled_at, so getActiveTaskRouteLease can return an active lease carrying stale terminal metadata from a prior lifecycle. Include settled_at: null in the conflict update and pin the shape with a regression test.
  - confidence: 0.82
