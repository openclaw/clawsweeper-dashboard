Overall correctness: patch is incorrect

Full review comments:

- **[P1] Scope route leases beyond raw runId:** `src/tasks/task-route-lease.ts:140-146`
  - body: The new durable lease stores requester_origin_json and every acquire/get/update/settle path keys only by raw run_id, but current main allows the same runId across runtime/session scopes and suppresses unscoped matches when scope tuples differ. A second task with the same runId can overwrite or expose another requester origin, so key and look up the lease by the task scope tuple or keep this lease cron-only with collision coverage.
  - confidence: 0.94
- **[P2] Delete route leases with task cleanup:** `src/tasks/task-route-lease.ts:324-336`
  - body: The GC helper only marks active expired rows, while settled, retired, expired, and task-deleted rows remain in the new durable table. Wire lease deletion into task cleanup or add a bounded retention owner so requester route state does not accumulate indefinitely.
  - confidence: 0.9
- **[P3] Clear settled_at when re-acquiring a lease:** `src/tasks/task-route-lease.ts:140-146`
  - body: Re-acquire sets status back to active but leaves a prior settled_at value intact, so getActiveTaskRouteLease can return an active lease with stale terminal metadata. Clear settled_at in the conflict update so a re-acquire starts a fresh lifecycle.
  - confidence: 0.84
