## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Scope route leases beyond raw runId:** `src/tasks/task-route-lease.ts:140-146`
  - body: Current main intentionally scopes same-runId task operations by runtime/session facts, but this PR stores and resolves route leases with only `run_id`. A second active task with the same raw runId can replace or expose another task's requester origin, so include the task-registry scope tuple or narrow this to a cron-only scoped lease before merge.
  - confidence: 0.94
- **[P2] Delete route leases with task cleanup:** `src/state/openclaw-state-schema.sql:1148-1153`
  - body: The new table explicitly avoids a task_runs foreign key, but the branch does not add a corresponding task-deletion cleanup path; `expireStaleTaskRouteLeases` only marks active expired rows. Settled, retired, expired, and orphaned rows can accumulate indefinitely unless the registry delete path or settle path owns deletion.
  - confidence: 0.9
- **[P3] Clear settled_at when re-acquiring leases:** `src/tasks/task-route-lease.ts:140-146`
  - body: The conflict update reactivates an existing row by setting status and expiry, but it does not reset `settled_at`. A re-acquired active lease can therefore carry stale terminal metadata from a prior lifecycle, which makes the persisted row internally inconsistent.
  - confidence: 0.82
