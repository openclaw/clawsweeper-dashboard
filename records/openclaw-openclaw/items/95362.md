## Summary

The PR adds a SQLite task-route lease table/API plus cron acquire, settle, resolver, manual runId, GC, repro-script, and test wiring for isolated cron completion delivery.

## Best Possible Solution

## Solution Assessment

No as written; a route lease is a plausible owner-boundary fix, but the generic persistent identity must match task-registry scope or be narrowed to an unambiguous cron-only key before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Scope route leases beyond raw runId:** `src/tasks/task-route-lease.ts:140-146`
  - body: Current main deliberately supports multiple task records sharing a runId when runtime/requester scope differs. This PR stores and resolves task-route leases by run_id alone, so a second active task with the same raw runId can replace or expose another task's requester origin and send a completion to the wrong target. Include the task-registry scope facts, or make this a strictly cron-only identity, and add cross-scope collision coverage.
  - confidence: 0.91
- **[P2] Add a deletion owner for route leases:** `src/tasks/task-route-lease.ts:324-336`
  - body: The new GC helper only marks active expired rows as expired, while settled, retired, expired, and orphaned leases stay in task_route_leases and current task cleanup only deletes task_delivery_state plus task_runs. Add task-prune deletion or another bounded production sweep with regression coverage so durable route rows do not accumulate indefinitely.
  - confidence: 0.87
- **[P3] Clear settled_at on lease re-acquire:** `src/tasks/task-route-lease.ts:140-146`
  - body: The conflict update can reactivate a previously settled row by setting status back to active, but it leaves the old settled_at value in place. Reset settled_at to null during re-acquire and assert the active-after-reacquire shape in the lease tests.
  - confidence: 0.82
- **[P3] Remove the dead task-completion docs reference:** `src/state/openclaw-state-schema.sql:1146`
  - body: The schema comment points readers to docs/concepts/task-completion.md, but that page is not present on current main or this PR. Add the referenced doc in the stack or replace the pointer with the invariant near the table definition.
  - confidence: 0.86
