## Summary

Keep open: current main still has the same verified-listener path that can miss a foreground `openclaw-gateway` child and then print the service-disabled message. The active fix path is the open, conflicting PR at https://github.com/openclaw/openclaw/pull/72224 rather than closing this issue early.

## What This Changes

`openclaw gateway stop` should terminate a foreground-launched gateway instead of only reporting that the service registration is disabled.

## Best Possible Solution

Land the existing PR after rebase and review, or land a narrower equivalent that recognizes bare `openclaw-gateway`, adds Linux listener discovery beyond `lsof`, preserves verified signaling, and covers stop with regression tests.

## Reproduction Assessment

Yes. The reporter provided concrete foreground/no-service-manager commands and retest output, and current source still has the verifier and lsof-only paths that make the stop fallback return null and print `Gateway service disabled.`

## Solution Assessment

Yes. The best fix direction is a narrow verifier/listener-discovery change that keeps SIGTERM guarded by gateway process verification; a message-only change would leave container users manually killing a valid gateway process.
