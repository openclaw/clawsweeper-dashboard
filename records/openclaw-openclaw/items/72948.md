## Summary

Keep open. Current main still has the unmanaged stop fallback, but the argv verifier rejects a bare `openclaw-gateway` process before the `allowGatewayBinary` branch can match, so `gateway stop` can still fall through to the misleading service-disabled path; open PR #72224 remains the implementation candidate.

## What This Changes

Make `openclaw gateway stop` terminate a verified foreground/unmanaged Gateway process instead of reporting only that the service manager registration is disabled.

## Best Possible Solution

Land #72224 after maintainer review and rebase, or land a narrower equivalent that keeps strict process verification while recognizing a bare `openclaw-gateway` listener and adds regression coverage plus a changelog entry.

## Reproduction Assessment

Yes. Start a foreground Gateway such as `openclaw gateway run --dev --port 38789`, run `openclaw gateway stop`, and current source shows a bare `openclaw-gateway` argv can leave no verified PID and fall through to `Gateway service disabled.`.

## Solution Assessment

Yes. A narrow verified unmanaged-process fallback fix is the maintainable direction because it preserves the safety check before sending SIGTERM; a message-only change would leave sandbox/container users manually killing a valid Gateway process.
