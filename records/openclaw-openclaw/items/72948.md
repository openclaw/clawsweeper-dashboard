## Summary

Keep open: current main still has the reported foreground-gateway stop gap, and #72224 is the open implementation candidate for the same verifier/listener-discovery path.

## What This Changes

`openclaw gateway stop` should stop a directly launched foreground gateway instead of falling through to service-manager-only disabled messaging.

## Best Possible Solution

Land #72224 after rebase and maintainer review, or land a narrower equivalent that recognizes bare `openclaw-gateway`, adds Linux listener discovery beyond `lsof`, and covers stop/restart with regression tests and a changelog entry.

## Reproduction Assessment

Yes. The reporter supplied concrete foreground-Gateway repro and retest details, and current-main source still shows the same verifier/listener path can return null and print `Gateway service disabled.`

## Solution Assessment

Yes. A narrow verifier/listener-discovery fix is the best direction because it preserves the safety check before SIGTERM; a message-only change would leave container users manually killing a valid Gateway process.
