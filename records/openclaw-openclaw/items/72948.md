## Summary

Keep open: current main still has the foreground/no-service-manager stop gap. The active implementation path is the open, conflicting PR at https://github.com/openclaw/openclaw/pull/72224, so the issue should remain open until that PR or a narrower current-main port lands.

## What This Changes

The issue asks for `openclaw gateway stop` to stop a directly launched foreground gateway in no-service-manager environments instead of reporting `Gateway service disabled` while leaving the process alive.

## Best Possible Solution

Land the existing PR after rebase and review, or land a narrower equivalent that recognizes bare `openclaw-gateway`, adds Linux listener discovery beyond `lsof`, preserves verified signaling, and covers stop with regression tests.

## Reproduction Assessment

Yes. The reporter provided concrete foreground/no-service-manager commands and retest output, and current source still has verifier and lsof-only paths that make the stop fallback return null and print `Gateway service disabled`; I did not rerun the container smoke in this read-only pass.

## Solution Assessment

Yes. The best fix direction is a narrow verifier/listener-discovery change that keeps SIGTERM guarded by gateway process verification; a message-only change would leave container users manually killing a valid gateway process.
