## Summary

Keep open. Current main still lacks centralized gateway tool-call audit logging, and the linked feature issue was closed as superseded by this PR, but the branch is not merge-ready because it misses denied attempts and the current-main RPC direct-invoke path.

## What This Changes

The PR adds a gateway tool-call audit JSONL helper, threads audit context through direct Gateway tool invocation and OpenResponses embedded tool execution, and updates focused tests plus shared guardrail tests.

## Best Possible Solution

Rebase onto current main and place gateway tool-call audit in the shared direct invoke helper plus embedded tool wrapper so allowed and denied HTTP/RPC/OpenResponses attempts are recorded with redacted args and status/reason metadata; leave approval-decision query work to https://github.com/openclaw/openclaw/pull/72769.

## Reproduction Assessment

Yes. Source inspection shows PR head returns or throws before audit append on denied direct and embedded calls, while current main routes direct HTTP and RPC calls through invokeGatewayTool with no audit context.

## Solution Assessment

No. The feature direction is useful, but the implementation should be rebased into the current shared invoke helper and should record denied outcomes as first-class audit records.
