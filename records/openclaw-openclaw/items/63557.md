## Summary

Keep open. The feature is useful, but the PR is not merge-ready because it adds a security-sensitive audit sink that misses denied tool attempts and still targets the old HTTP-only direct invoke shape while current main routes HTTP and SDK/RPC direct invokes through a shared helper.

## What This Changes

Adds a gateway tool-call JSONL audit helper, threads audit context through direct `/tools/invoke` and OpenResponses embedded tool execution, and updates focused tests plus unrelated shared test guardrails.

## Best Possible Solution

Land a rebased gateway-owned audit trail in `src/gateway/tools-invoke-shared.ts` and the embedded tool hook path that records allowed and denied outcomes with redacted args/status metadata; keep approval-decision query work in https://github.com/openclaw/openclaw/pull/72769 separate.

## Reproduction Assessment

Yes, source-reproducible. Current main has no gateway audit sink, and the PR diff appends only after allowed hook outcomes while direct invokes now flow through a shared helper used by HTTP and RPC.

## Solution Assessment

No. The direction is useful, but the implementation should be rebased onto the shared invoke helper and should write success and denied audit records with status/reason metadata.
