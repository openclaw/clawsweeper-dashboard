## Summary

Keep open: current main still lacks gateway-level tool-call audit logging, and the linked feature issue was explicitly closed as superseded by this PR. The branch is not mergeable as-is because it misses current RPC/shared-invoke and denied-attempt paths.

## What This Changes

This PR adds a redacted gateway tool-call JSONL audit helper, threads audit context through gateway and agent paths, and adds focused gateway audit tests plus unrelated guardrail-test updates.

## Best Possible Solution

Keep the audit feature on this PR, but centralize one redacted, outcome-aware writer at the shared gateway invoke boundary and embedded tool wrapper without adding query UI, retention, signing, or config policy in this change.

## Reproduction Assessment

Yes. Source inspection against current main shows HTTP and RPC direct invokes share invokeGatewayTool, while the PR writes only in the old HTTP path and appends after blocked embedded outcomes have already exited.

## Solution Assessment

No. The JSONL helper and redaction direction are reasonable, but the implementation should move to the current shared helper and record allowed, denied, and failed outcomes consistently.
