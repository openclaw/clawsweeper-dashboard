## Summary

Keep open: current main still lacks gateway tool-call audit logging, and this PR remains the canonical implementation path, but the branch is not mergeable as proposed because it misses denied attempts and the current shared RPC/direct invoke path.

## What This Changes

The branch adds a gateway tool-call JSONL audit sink with argument redaction, threads audit context through OpenResponses and embedded tools, and appends direct `/tools/invoke` audit rows with focused tests plus unrelated guardrail-test updates.

## Best Possible Solution

Land a rebased audit implementation that records redacted allowed and denied HTTP, RPC, and OpenResponses tool attempts from the shared direct invoke helper and embedded tool wrapper, while leaving approval-decision query work to https://github.com/openclaw/openclaw/pull/72769.

## Solution Assessment

No. The audit direction is useful, but the maintainable fix now needs the shared direct invoke helper and embedded wrapper to emit outcome-aware records for both allowed and denied attempts.
