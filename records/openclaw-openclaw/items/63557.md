## Summary

Keep this PR open. Current main still lacks the requested gateway tool-call audit sink, and the PR is useful, but it is not merge-ready because the proposed audit writes only cover allowed tool executions and omit denied gateway-originated attempts.

## What This Changes

The PR adds a gateway tool-call audit JSONL helper, threads audit context through direct `/tools/invoke` and OpenResponses-originated agent execution, updates focused gateway tests, and includes several unrelated shared test guardrail updates.

## Best Possible Solution

Land a gateway-owned audit implementation that records both allowed and denied gateway-originated tool attempts with redacted arguments, stable correlation metadata, denial status/reason, focused coverage for `/tools/invoke` and OpenResponses, and minimal unrelated branch noise.

## Reproduction Assessment

Yes. The skipped-audit behavior is reproducible by code inspection of the PR diff: both proposed append sites are placed after blocked-call exits, so a before-tool hook, approval denial, or loop block returns without writing the audit record.

## Solution Assessment

No. The proposed direction is close, but the best maintainable fix needs the audit helper called from the denied branches as well, ideally with explicit result/status and reason fields rather than only recording successful executions.
