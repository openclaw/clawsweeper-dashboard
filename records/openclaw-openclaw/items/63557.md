## Summary

Keep this PR open: current main still lacks the requested gateway tool-call audit trail, but the branch is stale against the current shared direct-invoke path and misses denied attempts in both direct and embedded execution.

## What This Changes

The PR adds an append-only gateway tool-call audit JSONL sink, threads audit context through direct tools invocation and OpenResponses embedded tool execution, and updates focused tests plus several unrelated guardrail expectations.

## Best Possible Solution

Rebase onto current main and implement gateway tool-call auditing in the shared direct invoke helper plus the embedded tool wrapper so allowed and denied HTTP, RPC, and OpenResponses attempts are recorded with redacted args and status/reason metadata; leave approval-decision querying to https://github.com/openclaw/openclaw/pull/72769.

## Reproduction Assessment

Yes. Source inspection of PR head shows blocked direct and embedded calls return or throw before the audit append, and current main shows both HTTP and RPC direct invokes now use a shared helper with no audit path.

## Solution Assessment

No. The feature direction is useful, but the implementation should be rebased into the shared invoke helper and should record denied outcomes instead of only successful HTTP/OpenResponses tool execution.
