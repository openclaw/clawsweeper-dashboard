## Summary

Keep open: current main still lacks the requested gateway tool-call audit sink, and the linked issue was closed as superseded by this PR rather than shipped. The branch is not merge-ready because it misses current main’s shared direct-invoke RPC path and denied-attempt paths for a security-relevant audit trail.

## What This Changes

The branch adds a redacted `gateway-tool-audit.jsonl` helper, threads gateway audit context through OpenResponses and agent tool execution, appends direct `/tools/invoke` audit records, and also carries unrelated memory-wiki and boundary-test updates.

## Best Possible Solution

Rebase and split the branch, then centralize one outcome-aware redacted audit write at the shared direct-invoke helper and embedded hook boundary without adding query UI, retention, signing, or new config policy in this PR.

## Reproduction Assessment

Yes for the PR review findings via source inspection. Current main routes both HTTP and RPC direct invokes through `invokeGatewayTool`, while the branch appends only in the old HTTP handler and places embedded appends after blocked paths have already exited.

## Solution Assessment

No. The JSONL helper and redaction direction are reasonable, but the write belongs at the current shared helper and embedded outcome boundary so allowed, denied, RPC, and HTTP attempts are covered consistently.
