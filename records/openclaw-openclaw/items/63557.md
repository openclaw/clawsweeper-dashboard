## Summary

Keep open: current main still lacks the requested gateway tool-call audit sink, and this PR remains the canonical implementation path, but the branch is not merge-ready because the proposed writes skip denied attempts and are stale relative to current main's shared tool-invoke helper.

## What This Changes

The PR adds a `gateway-tool-audit.jsonl` helper, threads audit context through direct `/tools/invoke` and OpenResponses embedded tool execution, and adds focused audit tests plus unrelated shared-test fixes.

## Best Possible Solution

Land a gateway-owned audit trail on the current shared tool invocation and embedded hook paths, recording allowed and denied outcomes with redacted args/status metadata and focused tests; keep approval-decision query work in https://github.com/openclaw/openclaw/pull/72769 separate.

## Reproduction Assessment

Yes. Static source inspection shows the PR append sites are after blocked-return paths, and current main's shared helper returns blocked outcomes without any audit sink.

## Solution Assessment

No. The direction is useful, but the implementation should be rebased onto current `invokeGatewayTool` and should write success and denied audit records with status/reason metadata from the shared paths.
