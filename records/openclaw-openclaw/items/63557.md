## Summary

Keep open: current main still lacks the requested gateway tool-call audit sink, but this PR is not mergeable as-is because it writes at the old HTTP-only boundary, misses current RPC/direct shared-invoke coverage, misses denied attempts, and carries unrelated test churn.

## What This Changes

The PR adds a redacted `gateway-tool-audit.jsonl` writer, threads gateway audit context through OpenResponses and embedded tool execution, and updates direct `/tools/invoke` audit tests plus unrelated guardrail tests.

## Best Possible Solution

Rebase and split the branch, then centralize one outcome-aware redacted audit write in the shared direct-invoke helper and embedded hook boundary without adding query UI, retention, signing, or new config policy in this PR.

## Reproduction Assessment

Yes for the PR review findings via source inspection. Current main routes both HTTP and RPC direct invokes through `invokeGatewayTool`, while the PR appends only in the old HTTP handler and places embedded appends after blocked paths have already exited.

## Solution Assessment

No. The JSONL helper and redaction direction are reasonable, but the write belongs at the current shared helper and embedded outcome boundary so allowed, denied, RPC, and HTTP attempts are covered consistently.
