## Summary

Keep open: current main still lacks a gateway tool-call audit sink, and this PR is useful but should not merge as-is because it skips denied tool attempts and is stale against the current shared tool-invoke path.

## What This Changes

The branch adds a gateway tool-call audit JSONL helper, threads audit context through direct and OpenResponses tool execution, adds focused tests, and carries shared CI guardrail test updates.

## Best Possible Solution

Land a rebased gateway-owned audit implementation on the current shared invoke and hook paths that records allowed and denied outcomes with redacted args, status/reason metadata, focused tests, and a changelog entry.

## Reproduction Assessment

Yes. Static inspection is enough: current main has blocked direct and embedded paths, while the PR's audit append sites run only after those blocked exits.

## Solution Assessment

No. The direction is right, but the implementation should audit denied outcomes before returning and should be retargeted to the current shared tool invocation path rather than the older HTTP-only path.
