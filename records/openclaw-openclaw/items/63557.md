## Summary

Keep open: current main still lacks the centralized gateway tool-call audit sink, and this PR is the canonical implementation path, but the patch is not ready because it misses denied attempts and does not fit the current shared HTTP/RPC tool-invoke path.

## What This Changes

The branch adds a `gateway-tool-audit.jsonl` helper, threads gateway audit context through direct `/tools/invoke` and OpenResponses embedded tool execution, and adds focused tests plus unrelated shared-test fixes.

## Best Possible Solution

Land a rebased gateway-owned audit trail in `src/gateway/tools-invoke-shared.ts` and the embedded tool hook path that records allowed and denied outcomes with redacted args/status metadata; keep approval-decision query work in https://github.com/openclaw/openclaw/pull/72769 separate.

## Reproduction Assessment

Yes, source-reproducible. Static inspection shows the PR append sites run only after allowed outcomes, while current main routes both HTTP and RPC direct invokes through a shared helper with no audit sink.

## Solution Assessment

No. The direction is useful, but the implementation should be rebased onto the current shared invoke helper and should write success and denied audit records with status/reason metadata.
