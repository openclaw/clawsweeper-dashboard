## Summary

Keep open: current main still lacks the requested gateway tool-call audit trail, and this PR remains useful but needs a rebase plus security-sensitive coverage fixes before merge.

## What This Changes

The PR adds a `gateway-tool-audit.jsonl` helper, threads audit context through direct `/tools/invoke` and OpenResponses tool execution, and adds focused tests plus unrelated shared test guardrail updates.

## Best Possible Solution

Land a rebased gateway-owned audit trail in `src/gateway/tools-invoke-shared.ts` and the embedded tool hook path that records allowed and denied outcomes with redacted args and status metadata; keep approval-decision query work in https://github.com/openclaw/openclaw/pull/72769 separate.

## Reproduction Assessment

Yes. Source inspection of current main shows no gateway tool-call audit sink, and PR-head inspection shows the proposed append points miss denied outcomes and current-main RPC direct invokes.

## Solution Assessment

No. The feature direction is useful, but the implementation should be rebased onto the shared invoke helper and should write success and denied audit records with status and reason metadata.
