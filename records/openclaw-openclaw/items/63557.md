## Summary

Keep open: current main still lacks this gateway tool-call audit sink, and the closed request at https://github.com/openclaw/openclaw/issues/13131 was explicitly superseded by this PR rather than shipped. The direction is useful, with prior discussion noting Pinchy’s external WebSocket-bridge audit implementation, but the patch is not merge-ready after current main’s shared `tools.invoke` refactor because it misses RPC and denied-attempt coverage.

## What This Changes

The branch adds a redacted `gateway-tool-audit.jsonl` helper, threads audit context through OpenResponses and agent tool execution, appends direct `/tools/invoke` audit records, and carries unrelated memory-wiki/shared guardrail test updates.

## Best Possible Solution

Rebase and split the branch, then centralize one outcome-aware redacted audit write at the shared direct-invoke helper and embedded hook boundary so HTTP, RPC, allowed, and denied attempts are covered without adding retention, query UI, signing, or new config policy in this PR.

## Reproduction Assessment

Yes for the PR review findings via source inspection. Current main routes both HTTP and RPC direct invokes through `invokeGatewayTool`, while this branch appends only in the HTTP handler and places embedded appends after blocked paths have already exited.

## Solution Assessment

No. The JSONL helper and redaction direction are reasonable, but the write belongs at the current shared helper and embedded outcome boundary so allowed, denied, RPC, and HTTP attempts are covered consistently.
