Keep open: current main and v2026.6.10 still create the reported backup temp archive and staging directory, then clean them only during normal in-process unwinding. The linked fix PR is the right current path, but it is still open, so this issue should not close before that fix or an equivalent lands.

## What This Changes

`backup create` should sweep stale backup staging directories and output-adjacent temp archives left by hard-killed prior runs.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/95600 or a narrow equivalent that covers both stale staging directories and output-adjacent temp archives, then close this issue as implemented.

## Reproduction Assessment

Yes, at source level: current main creates the reported temp archive and staging directory and removes both only in an in-process `finally`. I did not run a destructive hard-kill reproduction in this read-only review.

## Solution Assessment

Yes, a startup sweep guarded by tight matching and owner markers is the best bounded way to handle hard-kill cleanup that JavaScript `finally` cannot run. The issue should stay open until the linked implementation or an equivalent merges.
