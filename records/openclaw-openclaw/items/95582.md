The open backup cleanup PR is useful context, but it is conflicting and only targets output-adjacent `.tmp` archives, not stale `openclaw-backup-` staging directories.

## What This Changes

The issue requests startup cleanup for stale `backup create` temp archives and staging directories left after a hard-killed backup run.

## Best Possible Solution

Keep this issue open until backup cleanup safely covers both stale output-adjacent archives and stale `openclaw-backup-` staging directories, either by extending the open cleanup PR or replacing it with a complete focused fix.
