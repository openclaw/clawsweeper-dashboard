## What This Changes

The PR changes Chutes bundled-plugin OAuth userinfo and token-exchange success JSON parsing to use the shared bounded `readProviderJsonResponse` helper.

## Best Possible Solution

Land this narrow plugin-boundary hardening after maintainer acceptance of the shared Chutes OAuth JSON cap, while reviewing https://github.com/openclaw/openclaw/pull/96777 separately for the deprecated core helper.

## Reproduction Assessment

Yes from source inspection and contributor terminal proof: current main buffers two Chutes plugin OAuth success bodies with `response.json()`, while the PR proof shows the bounded helper stopping a 32 MiB server response at about 16 MiB. I did not run the proof locally because this review is read-only.

## Solution Assessment

Yes. Reusing `readProviderJsonResponse` through the plugin SDK is the narrowest maintainable fix because the shared helper owns the byte cap, stream cancellation, and labeled parse errors; a Chutes-local reader would duplicate SDK behavior.
