## Summary

Keep this PR open: current main and the latest release still use unbounded Chutes plugin OAuth success-body `response.json()` reads, while the PR routes those reads through the existing bounded provider JSON helper with focused regression coverage and sufficient terminal proof. ## What This Changes

The PR replaces the Chutes plugin OAuth userinfo and token-exchange success JSON reads with `readProviderJsonResponse` and adds an oversized token-exchange regression test.

## Best Possible Solution

Land this narrow plugin-path hardening after maintainer acceptance of the shared Chutes OAuth JSON cap, while reviewing the core-helper companion separately.

## Reproduction Assessment

## Solution Assessment

Yes. Reusing `readProviderJsonResponse` through the plugin SDK facade is the narrowest maintainable fix because the shared helper owns the byte cap, cancellation, and labeled parse errors; a Chutes-local helper would duplicate policy.
