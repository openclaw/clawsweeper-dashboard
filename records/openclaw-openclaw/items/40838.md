## Summary

Keep open. Current main and latest v2026.5.22 still expose Telegram sender id/name/username to agent context but do not carry Telegram `msg.from.is_bot`; the existing shared `SenderFacts.isBot` field makes this a narrow additive context-plumbing task rather than a broad routing redesign.

## What This Changes

The issue requests exposing Telegram sender metadata, especially bot status, to agent-visible inbound context for classification and routing decisions.

## Best Possible Solution

Add an optional current-sender bot boolean through `SenderFacts.isBot`, finalized `MsgContext`, and untrusted sender metadata, populated from Telegram `msg.from?.is_bot` without changing requireMention, auth, or bot-routing policy.
