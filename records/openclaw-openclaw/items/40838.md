Keep open. Current main and the latest release still expose Telegram sender id/name/username and source modality, but not Telegram `msg.from.is_bot` as agent-visible sender bot-status metadata.

## What This Changes

Expose additional Telegram sender/message facts, especially `msg.from.is_bot`, to the agent-visible context so agents can classify bot versus human senders.

## Best Possible Solution

Add a narrow additive projection for Telegram `msg.from.is_bot` through `SenderFacts`, finalized `MsgContext`, and inbound sender metadata, with focused tests and no broader Telegram metadata expansion.

## Reproduction Assessment

Yes. Source inspection on current main shows Telegram has access to `msg.from.is_bot`, `SenderFacts` already has `isBot`, but the Telegram builder, finalized context, and prompt metadata do not expose it.

## Solution Assessment

Yes. The best fix is a narrow additive projection through the existing sender/context seams; closing this or folding it into the broader bot-targeting PR would leave the exact agent-visible field unresolved.
