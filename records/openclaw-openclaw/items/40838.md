## What This Changes

The issue requests exposing Telegram sender metadata, especially bot status from `msg.from.is_bot`, in the agent context so agents can distinguish Telegram bot and human senders.

## Best Possible Solution

Keep this issue as the canonical request and resolve it through the linked Telegram PR or a maintainer-updated replacement that carries sender bot status from `msg.from.is_bot` through `SenderFacts`, finalized context, prompt metadata, and focused tests.

## Reproduction Assessment

Yes, at source level: current main has `SenderFacts.isBot` and Telegram exposes `msg.from.is_bot`, but Telegram does not populate that fact and the shared context/prompt projection has no bot-status field. I did not run a live Telegram repro in this read-only review.

## Solution Assessment

Yes, if solved as a narrow channel-plus-shared-projection change. A one-line Telegram sender-facts patch is not enough because agents still need the finalized context and prompt metadata to expose the bot-status value.
