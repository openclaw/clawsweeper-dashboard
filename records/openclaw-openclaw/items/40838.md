## Summary

Keep open. Current main still has Telegram sender data and a shared `SenderFacts.isBot` contract, but the Telegram inbound payload, finalized `MsgContext`, and rendered sender metadata only expose sender id/name/username; the related bot-to-bot issue and open bot-targeting PR are adjacent, not replacements for this focused metadata gap.

## What This Changes

The issue requests exposing Telegram sender metadata such as `msg.from.is_bot` to agent-visible context so agents can classify message sources.

## Best Possible Solution

Expose a bounded sender bot boolean from `msg.from?.is_bot` through `SenderFacts`, `MsgContext`, and inbound metadata, while leaving broader Telegram bot-to-bot routing policy to https://github.com/openclaw/openclaw/issues/79077.
