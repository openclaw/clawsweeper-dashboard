## Summary

Keep open. Current main and v2026.5.19 still have Telegram sender data and the shared `SenderFacts.isBot` contract, but the Telegram context payload, `MsgContext`, and rendered inbound metadata do not expose bot classification to agents.

## What This Changes

Expose Telegram sender fields, especially `msg.from.is_bot`, in agent-visible context so agents can classify sender type.

## Best Possible Solution

Add a bounded sender-bot boolean from Telegram through `SenderFacts`, `MsgContext`, and inbound metadata with focused regression coverage and docs, while leaving broader Telegram bot-to-bot policy to the separate issue.

## Reproduction Assessment

Yes for the source-level gap: Telegram and grammY expose `message.from.is_bot`, `SenderFacts.isBot` exists, and current main does not carry it into agent-visible context or inbound metadata. This is not a runtime crash reproduction; it is an additive metadata verification path.

## Solution Assessment

Yes. The narrow maintainable fix is to plumb the existing sender fact into finalized context and rendered metadata, not to expose raw Telegram payloads or implement the broader bot-to-bot policy here.
