## Summary

Current main and the latest release still lack a `Content-Disposition` filename policy for Control UI assistant-media documents, so the reported WebChat download bug remains valid. The issue should stay open because the direct fix is an open draft PR with closing syntax, not a merged implementation.

## What This Changes

The issue asks for WebChat assistant-media document links to download with `Content-Disposition: attachment`, preserve the original filename, and keep previewable images inline.

## Best Possible Solution

Review, repair, or replace the linked fix so document-like assistant media downloads with safe filenames while image/audio/video previews remain inline.

## Reproduction Assessment

Yes, at source level: current main streams assistant-media files without `Content-Disposition`, and the WebChat document link does not provide a `download` filename hint. I did not run a live browser save-dialog reproduction in this read-only review.

## Solution Assessment

Yes, the best fix is in the existing Control UI assistant-media route and its focused tests: set safe attachment filenames for document-like responses while preserving inline behavior for previewable media.
