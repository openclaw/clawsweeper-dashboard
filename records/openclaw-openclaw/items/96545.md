## Summary

Keep open: current main and the latest release still serve WebChat assistant-media documents without a download/filename disposition, and the overlapping fixes are still open rather than landed.

## What This Changes

The issue requests WebChat assistant-media document downloads to use attachment disposition, preserve the original filename, use correct MIME types, and keep images inline.

## Best Possible Solution

Review or replace the strongest current-main-compatible fix so non-inline assistant-media documents download with safe filenames while previewable media stays inline.

## Reproduction Assessment

Yes, at source level: current main streams assistant-media files without `Content-Disposition`, and the WebChat document anchor lacks a `download` attribute. I did not run a live browser save-dialog reproduction in this read-only review.

## Solution Assessment

Yes, the narrow maintainable solution is to set safe attachment filenames for non-inline document responses, keep images/audio/video inline, and cover both the gateway response and WebChat link rendering with focused tests.
