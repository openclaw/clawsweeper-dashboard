## What This Changes

WebChat assistant-media document downloads should use safe attachment disposition and preserve the original filename instead of opening through the generic assistant-media URL.

## Best Possible Solution

Review and land or replace the current draft fixing PR so non-inline assistant-media documents download with safe filenames while previewable media stays inline.

## Solution Assessment

Yes, the narrow maintainable fix is to add safe attachment disposition for non-inline document responses while preserving inline behavior for previewable media, then cover the gateway response and WebChat link rendering with focused tests.
