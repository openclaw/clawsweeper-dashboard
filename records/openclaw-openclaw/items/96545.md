## Summary

Keep open: current main and the latest release still serve Control UI assistant-media documents without an attachment filename disposition, and https://github.com/openclaw/openclaw/pull/96551 is an open draft candidate that uses closing syntax for this issue.

## What This Changes

WebChat assistant-media document downloads should use attachment disposition and preserve the original filename while previewable media stays inline.

## Best Possible Solution

Review, land, or replace https://github.com/openclaw/openclaw/pull/96551 with a current-main-compatible fix that adds safe attachment filenames for non-inline documents while preserving inline previews for previewable media.

## Reproduction Assessment

Yes. At source level, current main serves assistant-media files without a Content-Disposition header and renders document links without a download attribute; I did not run a live browser save-dialog reproduction in this read-only review.

## Solution Assessment

Yes. The best path is the narrow existing-route fix: set safe attachment filenames for non-inline document responses, keep previewable media inline, and cover the Gateway response plus WebChat link rendering with focused tests.
