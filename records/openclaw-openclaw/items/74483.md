## Summary

Keep open. The PR is still an active draft implementation of generic Plugin SDK follow-up seams that current main does not provide, but the current head is not mergeable as-is because tagged unscheduling can be invoked from a stale plugin API object after registry replacement.

## What This Changes

The PR adds additive Plugin SDK and Gateway surfaces for host runtime reads, derived tool paths, attachment hints, Control UI input-suppression descriptors, tagged scheduled-turn cleanup, session-entry slot projection, session actions, QA boundary cleanup, generated Swift clients, and contract tests.

## Best Possible Solution

Keep the draft available for author and maintainer review, fix the tagged-unscheduling liveness guard, preserve the generic plugin-boundary direction, then rebase after #73384 and rerun the SDK, protocol, generated-client, lint, type, and changed-lane validation before marking it ready.

## Reproduction Assessment

Yes for the review finding: retain a plugin API object from a retired registry, load a replacement registry with tagged scheduled turns for the same plugin/session, then call unscheduleSessionTurnsByTag on the stale API object. The PR head reaches cron.list and cron.remove without the live-plugin guard.

## Solution Assessment

No, this is not the best final form as-is. The additive SDK direction fits the existing plugin boundary, but tagged unscheduling should use the same shouldCommitWorkflowSideEffect guard pattern as the sibling workflow APIs before it performs scheduler side effects.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Gate tag unscheduling on live plugin state:** `src/plugins/registry.ts:2506-2513`
  - body: `scheduleSessionTurn`, `sendSessionAttachment`, and `emitAgentEvent` all stop stale API objects with `shouldCommitWorkflowSideEffect`, but `unscheduleSessionTurnsByTag` skips that guard before reaching `cron.list` and `cron.remove`. A plugin API object retained from a retired registry can remove current-generation tagged jobs for the same plugin/session after reload; pass the guard into the helper or check it before listing/removing cron jobs.
  - confidence: 0.88
