## Summary

Keep open; the patch fixes marker propagation and extends filtering to deliverable selectors, but it still preserves reasoning-only fallback payloads as user-visible cron output, leaving a privacy-sensitive residual leak in the reviewed announce path.

## What This Changes

The PR propagates `isReasoning` from embedded-runner payload construction, filters reasoning in cron summary/text/deliverable selectors, adds focused tests, and updates the changelog.

## Best Possible Solution

Land one canonical cron announce fix that propagates `isReasoning` and treats reasoning as non-user-visible across summary, output, delivery, synthesized fallback, first-text fallback, and post-error success detection, with focused regression coverage and the changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: reasoning payloads can be selected as fallback cron output and direct delivery sends nonempty selected payloads; the PR head also adds tests that intentionally preserve the reasoning-only fallback.

## Solution Assessment

No as-is. Marker propagation and primary-pass filtering are the right direction, but reasoning payloads should not be deliverable fallback content for cron announce channels that lack a dedicated reasoning lane.

## Review Findings

Overall correctness: patch is incorrect

- **[P2] Suppress reasoning-only cron fallback payloads:** `src/cron/isolated-agent/helpers.test.ts:169-173`
