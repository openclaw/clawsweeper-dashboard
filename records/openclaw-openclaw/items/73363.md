## Summary

Keep open: the PR fixes marker propagation and mixed-payload filtering, but still preserves reasoning-only fallbacks that can be delivered, lacks after-fix real Matrix/Feishu cron proof, and overlaps the broader open fix in https://github.com/openclaw/openclaw/pull/40534.

## What This Changes

The branch propagates `isReasoning` from embedded-runner payload construction, filters reasoning in cron summary/text/deliverable pickers, adds focused tests, and appends a changelog entry.

## Best Possible Solution

Land one canonical fix that preserves `isReasoning` through embedded payloads and treats reasoning as non-user-visible across cron summary, output, delivery, synthesized fallback, first-text fallback, and post-error recovery, with real Matrix/Feishu cron proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main drops the marker and cron selectors choose text using content and `isError` checks only. The PR head also adds tests that intentionally preserve reasoning-only fallback selection.

## Solution Assessment

No as-is. Propagating `isReasoning` and filtering primary selections is the right direction, but the patch should make reasoning non-user-visible in fallback summary/output/delivery and post-error recovery, or adopt the broader shape from https://github.com/openclaw/openclaw/pull/40534.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Suppress reasoning-only cron fallbacks:** `src/cron/isolated-agent/helpers.test.ts:169-173`
  - body: The new reasoning-only fallback expectations keep `{ isReasoning: true }` text selectable when no visible payload exists. `resolveCronPayloadOutcome()` can then synthesize that text or return it as `deliveryPayloads`, and direct cron delivery sends nonempty payloads while Feishu formats reasoning payload text, so reasoning-only cron runs can still expose model thinking. Please make reasoning non-deliverable in fallback passes instead of treating it like error text.
  - confidence: 0.86
- **[P2] Exclude reasoning from post-error recovery:** `src/cron/isolated-agent/helpers.ts:291-296`
  - body: After this PR starts preserving `isReasoning`, this recovery check still treats later reasoning text as successful non-error output. An error followed only by thinking can clear the fatal error path and report the cron run as ok, so the before/after-error success checks need to ignore `isReasoning` payloads too.
  - confidence: 0.82
