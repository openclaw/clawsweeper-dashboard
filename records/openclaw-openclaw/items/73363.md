## Summary

Keep open. The PR is the right direction, but it is not merge-ready because the current diff still intentionally returns reasoning-only payloads in fallback paths, which can remain user-visible on cron announce delivery for channels without a downstream `isReasoning` guard such as Feishu.

## What This Changes

The PR propagates `isReasoning` from embedded-runner payload construction, filters reasoning payloads in cron summary/text/deliverable pickers on the primary pass, adds tests, and updates the changelog.

## Best Possible Solution

Land one canonical cron announce fix that carries `isReasoning` and treats reasoning as non-user-visible across summary, output, delivery, first-text fallback, and post-error success detection while preserving fallback behavior for real error payloads.

## Reproduction Assessment

Yes. A static reproduction is high-confidence: the PR head adds tests that expect reasoning-only payloads to be returned, and current `resolveCronPayloadOutcome()` can turn selected or synthesized nonempty text into direct delivery payloads.

## Solution Assessment

No as-is. Marker propagation and primary-pass filtering are the right direction, but fallback and recovery paths should exclude reasoning; #40534 shows the broader fix shape.
