## Summary

Keep open: the branch moves in the right direction by preserving the reasoning marker, but it still leaves reasoning-only fallback delivery and post-error recovery paths unsafe, and it lacks required after-fix Matrix/Feishu cron proof.

## What This Changes

The PR propagates `isReasoning` from embedded runner payload construction and filters reasoning payloads from cron announce summary/text/delivery picker primary passes with focused tests and a changelog entry.

## Best Possible Solution

Land one canonical fix that preserves `isReasoning` through embedded payloads and treats reasoning as non-user-visible across cron summary, output, delivery payloads, synthesized fallback, raw first-text fallback, and post-error recovery, with redacted Matrix/Feishu cron proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main creates reasoning text internally, drops the marker before returned payloads, and cron announce selectors/recovery checks choose text using content and `isError` checks only.

## Solution Assessment

No as-is. Propagating `isReasoning` and filtering primary selectors is the right direction, but the patch should also make reasoning non-deliverable in fallback paths, skip raw first-text fallback reasoning, and exclude reasoning from post-error recovery.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Suppress reasoning-only cron fallbacks:** `src/cron/isolated-agent/helpers.test.ts:169-173`
  - body: The new fallback expectations keep `{ isReasoning: true }` text selectable when no visible payload exists. `resolveCronPayloadOutcome()` can then synthesize or deliver that text, and Feishu formats reasoning payload text, so reasoning-only cron runs can still expose model thinking; please make reasoning non-deliverable in fallback passes and skip the raw first-text fallback for reasoning.
  - confidence: 0.87
- **[P2] Exclude reasoning from post-error recovery:** `src/cron/isolated-agent/helpers.ts:291-296`
  - body: After this PR starts preserving `isReasoning`, the recovery checks still treat later reasoning text as successful non-error output. An error followed only by thinking can clear the fatal error path and report the cron run as ok, so the before/after-error success checks need to ignore `isReasoning` payloads too.
  - confidence: 0.84
