## Summary

Keep open, but do not merge as-is. The branch fixes the marker propagation gap and now filters mixed reasoning payloads from the primary cron summary/output/delivery path, including the Feishu gap raised by stainlu, but it still preserves reasoning-only fallback delivery and overlaps the stricter open fix in #40534.

## What This Changes

The branch propagates `isReasoning` from embedded-runner payload construction, filters reasoning payloads from cron summary/output/delivery primary selection, adds regression tests, and records a changelog fix for cron announce leaks.

## Best Possible Solution

Land one canonical cron announce fix that propagates `isReasoning` and treats reasoning payloads as non-deliverable and non-summary in fallback paths, including first-text fallback and error-recovery success checks, unless maintainers explicitly decide that reasoning-only cron output may be shown.

## Reproduction Assessment

Yes. A static reproduction is high-confidence: current main creates reasoning payloads, drops the marker before returning them, and cron announce selection can send selected payloads directly. On this branch, the added reasoning-only fallback tests still prove a remaining user-visible leak path.

## Solution Assessment

No as-is. Marker propagation and primary-pass filtering are the right direction, but the reasoning-only fallback contradicts the suppress-reasoning contract for channels without a dedicated reasoning lane; the safer alternative is the stricter end-to-end filtering modeled by #40534.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P2] Avoid delivering reasoning-only cron payloads:** `src/cron/isolated-agent/helpers.ts:229-236`
  - body: The new filters skip `isReasoning` only in the primary pass, but the fallback path still returns a reasoning payload when every payload is reasoning, and the added tests codify that behavior. `resolveCronPayloadOutcome()` can then expose that text through `summary`, `outputText`, or `deliveryPayloads`, and direct cron delivery sends nonempty payloads to channels like Feishu that do not have a downstream `isReasoning` guard.
