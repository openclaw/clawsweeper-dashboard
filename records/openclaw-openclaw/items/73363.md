## Summary

Keep open. The PR fixes the important marker-propagation gap and now filters reasoning from the mixed-payload cron paths, but it still preserves reasoning-only fallback delivery, which leaves a privacy-sensitive version of the reported cron announce leak unresolved. The open overlapping PR #40534 tracks the stricter end-to-end behavior, so maintainers should pick one canonical branch before merge.

## What This Changes

The PR propagates `isReasoning` from embedded-runner payload construction, skips reasoning blocks in cron summary/output/deliverable primary selection, adds focused tests, and updates the changelog.

## Best Possible Solution

Land one canonical cron announce fix that propagates `isReasoning` and treats reasoning payloads as non-user-visible across summary selection, output selection, deliverable selection, first-text fallback, and post-error success detection, while preserving error fallback behavior only for real error payloads.

## Reproduction Assessment

Yes. A high-confidence static reproduction is enough: after this PR, a payload list containing only `{ text: "thinking", isReasoning: true }` is still expected by the added tests to return from summary/output/deliverable pickers, and cron direct delivery sends nonempty selected payloads.

## Solution Assessment

No as-is. The marker propagation and mixed-payload primary filtering are the right direction, but the reasoning-only fallback should be removed or explicitly approved, and the related first-text/error-recovery paths should be aligned with #40534 before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Avoid delivering reasoning-only cron payloads:** `src/cron/isolated-agent/helpers.ts:229-234`
  - body: The PR filters reasoning out of the successful deliverable set, but when every payload is reasoning it falls back through `pickLastDeliverablePayload()` and returns that reasoning text; the new tests codify the same fallback for summary/output. Cron direct delivery sends selected payloads, and Feishu has no downstream `isReasoning` guard on this path, so a reasoning-only cron run can still leak thinking content. Please make reasoning payloads non-deliverable in fallback paths too, and align the first-text/error-recovery checks with that policy.
  - confidence: 0.86
