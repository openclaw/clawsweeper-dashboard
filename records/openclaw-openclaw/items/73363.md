## Summary

Keep open. The PR fixes the main marker-propagation gap and the mixed visible/reasoning delivery case, but it still codifies reasoning-only cron payloads as deliverable and leaves adjacent fallback/status checks treating reasoning text as user-visible success, so it is not ready to merge as the privacy fix for #73186.

## What This Changes

The PR propagates `isReasoning` from embedded-runner payload construction, filters reasoning payloads in cron summary/output/deliverable primary selection, and adds focused tests plus a changelog entry.

## Best Possible Solution

Land one canonical cron-announce fix that propagates `isReasoning` and treats reasoning as non-user-visible across summary, output, delivery, fallback, first-text, and post-error success detection while preserving real error-payload fallbacks.

## Reproduction Assessment

Yes. A static reproduction is high confidence: on the PR head, a payload list containing only `{ text: "thinking aloud", isReasoning: true }` is expected to be returned by `pickDeliverablePayloads()`, and direct cron delivery sends selected nonempty payloads without a Feishu-side reasoning guard.

## Solution Assessment

No as-is. Marker propagation and primary-pass filtering are the right direction, but reasoning-only fallback and adjacent success/fallback checks need to align with the policy that reasoning is not user-visible output.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Suppress reasoning-only cron fallbacks:** `src/cron/isolated-agent/helpers.test.ts:169-173`

Please make reasoning payloads non-deliverable in fallback paths too, and align first-text/error-recovery checks with that policy.
