## Summary

Keep open. The PR fixes the marker propagation and primary-pass picker filtering, but it still preserves reasoning-only fallback delivery, which leaves a privacy-sensitive cron announce leak path and overlaps with the fuller open fix in #40534.

## What This Changes

The PR propagates `isReasoning` from embedded-runner payloads, filters reasoning in cron summary/text/delivery pickers, adds regression tests, and updates the changelog.

## Best Possible Solution

Land one canonical cron announce fix that propagates `isReasoning` and treats reasoning as non-user-visible across summary, output, delivery, synthesized fallback, first-text fallback, and post-error success detection.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: reasoning payloads can be selected as fallback cron output/delivery, and direct delivery sends selected nonempty payloads to channels such as Feishu that lack a downstream `isReasoning` guard.

## Solution Assessment

No as-is. Marker propagation and primary-pass filtering are the right direction, but reasoning payloads should not be deliverable fallback content for cron announce channels that lack a dedicated reasoning lane.

## Review Findings

Overall correctness: patch is incorrect

- **[P2] Suppress reasoning-only cron fallback payloads:** `src/cron/isolated-agent/helpers.test.ts:168-174`
