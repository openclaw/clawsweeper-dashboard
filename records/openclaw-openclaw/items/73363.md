## Summary

Keep open: the PR fixes the core marker propagation and primary picker paths, but it still permits reasoning-only fallback delivery and lets reasoning text satisfy post-error recovery checks; it also lacks contributor-supplied real Matrix/Feishu cron proof.

## What This Changes

The PR propagates `isReasoning` from embedded-runner payloads and updates cron isolated-agent summary/text/delivery pickers, tests, and changelog to prefer non-reasoning payloads.

## Best Possible Solution

Land one canonical current-main fix that carries `isReasoning` through embedded payloads and treats reasoning as non-user-visible across cron summary, output, delivery, fallback, and post-error recovery, with redacted Matrix or Feishu cron proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main creates reasoning payloads, then loses or ignores `isReasoning`, and PR head still allows reasoning-only fallback and recovery paths to treat thinking text as visible output.

## Solution Assessment

No as-is. Propagating `isReasoning` and filtering primary selectors is the right direction, but the maintainable fix must also make reasoning non-deliverable in fallback paths and exclude it from post-error recovery.

## Review Findings

Overall correctness: patch is incorrect

- **[P2] Suppress reasoning-only cron fallbacks:** `src/cron/isolated-agent/helpers.test.ts:169-173`
  - body: The new expectation keeps a lone `{ isReasoning: true }` payload selectable. `resolveCronPayloadOutcome()` can then synthesize or deliver that text through direct Matrix/Feishu cron delivery, so thinking-only runs can still expose internal reasoning; please keep reasoning non-deliverable in fallback passes too.

  - confidence: 0.84
