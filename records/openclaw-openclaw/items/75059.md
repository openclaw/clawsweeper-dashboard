## Summary

Keep this PR open, but it still needs changes before merge: the retry boundary is useful, but the current branch breaks `--json` retry output, retries some plain-message auth/policy close errors, and lacks the required changelog entry.

## What This Changes

The PR adds bounded exponential-backoff retry handling for transient `openclaw logs --follow` Gateway transport failures, focused CLI tests, and a docs note.

## Best Possible Solution

Land one focused CLI fix that retries only transient follow-mode Gateway disconnects, preserves fatal auth/pairing behavior for typed and plain close errors, keeps `--json` output line-delimited JSON, and includes docs plus a changelog entry.

## Reproduction Assessment

Yes. Current main has a source-level reproduction: an explicit `--url` follow fetch that raises a closed `GatewayTransportError` reaches the catch path and exits; the PR-introduced regressions can be reproduced with `--json` plus a retryable close and with a plain `Error('gateway closed (4001): unauthorized')`.

## Solution Assessment

No, not as-is. The CLI polling loop is the right fix boundary, but the implementation must preserve JSON-mode notices and classify fatal close codes in the plain-error path before it is merge-ready.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Parse plain close codes before retrying:** `src/cli/logs-cli.ts:168`
  - body: The non-`GatewayTransportError` branch treats every `gateway closed (` message as transient. Current code notes pending GatewayClient request failures can arrive as plain `Error`s, so a post-handshake `gateway closed (4001...)` or non-pairing `1008` failure will now wait through the retry loop instead of failing immediately as intended. Parse the close code in the plain-message path and keep 1008/4xxx fatal.
  - confidence: 0.82
- **[P2] Emit retry notices as JSON in --json mode:** `src/cli/logs-cli.ts:343-350`
  - body: The new retry branch always writes a styled text warning to stderr. In `openclaw logs --json`, existing errors and notices are JSON records and the docs advertise line-delimited JSON events, so this plain text breaks machine consumers during a recoverable disconnect. Branch on `jsonMode` and emit a `notice` JSON record instead.
  - confidence: 0.88
- **[P3] Add the required changelog entry:** `src/cli/logs-cli.ts:339`
  - body: This is a user-facing CLI fix, but the PR diff only changes docs, the logs CLI, and tests. Add a single-line active `CHANGELOG.md` Fixes entry for the reconnect behavior with appropriate human credit before merge.
  - confidence: 0.93
