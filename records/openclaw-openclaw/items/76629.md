## Summary

Keep open. Current main has partial runtime-context and delivery sanitizers, but provider-bound replay still lacks the broader transient assistant-message pruning requested here, so standalone `NO_REPLY`, stream-error fallback text, and copied OpenClaw runtime-context/event assistant text can remain in replay history.

## What This Changes

The issue requests pruning OpenClaw-owned internal/control/no-op messages from provider-bound replay context.

## Best Possible Solution

Add a generic embedded-runner replay-hygiene helper that removes only OpenClaw-owned nonsemantic artifacts before provider conversion, with regression coverage and a separate doctor/repair path for already-persisted polluted transcripts.
