## Best Possible Solution

Add a generic replay-boundary sanitizer in the embedded runner that removes only OpenClaw-owned non-semantic control artifacts before provider conversion, with focused regression coverage and optional doctor repair for already-polluted transcripts.

## Repair Work Prompt

Fix #76629 by adding provider-bound replay hygiene for OpenClaw-owned transient/internal artifacts in the embedded runner. Implement a generic helper near `normalizeAssistantReplayContent` / `sanitizeSessionHistory` that removes only non-semantic replay messages before provider conversion: `openclaw.runtime-context` custom messages if present, exact silent assistant payloads such as `NO_REPLY` or the single-key JSON silent envelope, the stream-error fallback placeholder, and assistant text that consists of copied OpenClaw runtime-context/runtime-event scaffolding. Preserve user messages and substantive assistant replies, including text that discusses `NO_REPLY`; preserve provider turn-shape and run existing tool-pairing repair after pruning. Add focused regression tests for replay history and the attempt path, update transcript hygiene docs and CHANGELOG if the behavior is user-facing. Do not add Feishu/Lark-specific logic, broad startup migrations, or a config flag unless maintainers explicitly request one; keep any stored transcript cleanup as a separate doctor/repair path.
