## Summary

Keep open. Current main still has only partial runtime-context cleanup: it strips `openclaw.runtime-context` custom messages and some metadata-only assistant content, but provider-bound replay does not yet prune standalone silent/failure/runtime-context assistant artifacts before provider conversion.

## What This Changes

The issue requests provider-bound replay hygiene that prunes OpenClaw-owned internal runtime-context messages, silent assistant sentinels, empty failure placeholders, and copied runtime-context scaffolding before model context replay.

## Best Possible Solution

Add a small generic replay-hygiene helper in the embedded-runner replay pipeline, with regression tests for each OpenClaw-owned transient artifact and a separate doctor/repair path for already-persisted polluted transcripts.

## Reproduction Assessment

Yes. A source-level reproduction is clear: build session history with standalone assistant `NO_REPLY`, `[assistant turn failed before producing content]`, or copied runtime-context preface text, and current main has no replay pruning step that removes those assistant rows before provider conversion.

## Solution Assessment

Yes. The narrow maintainable fix is generic replay-boundary hygiene in `replay-history.ts`; stored transcript cleanup should be handled separately through doctor/repair rather than by broad startup migration.
