## Summary

Keep open. Current main has partial runtime-context and delivery sanitizers, but provider-bound replay still lacks the broader transient assistant-artifact pruning requested here, so stale standalone `NO_REPLY`, stream-failure sentinel text, or copied runtime-context/event assistant rows can still survive into the next model context.

## What This Changes

The issue asks for provider-bound replay to prune OpenClaw-owned internal/no-op assistant and custom control artifacts before model context construction.

## Best Possible Solution

Add a narrow embedded-runner replay-hygiene pass before provider conversion/provider hooks, and keep any persisted legacy cleanup in a separate doctor/repair path rather than a startup migration.
