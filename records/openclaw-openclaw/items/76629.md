## Summary

Keep open. Current main has partial runtime-context protection, but the provider-bound replay pipeline still lacks the broader invariant requested here for silent/no-op assistant turns, failed-placeholder text, and copied runtime-context assistant scaffolding.

## What This Changes

Request to prune internal runtime-context, silent/no-reply, failed-placeholder, and copied control assistant messages from provider-bound replay context, with optional repair for polluted stored transcripts.

## Best Possible Solution

Add a generic replay-hygiene helper in the embedded runner that removes only OpenClaw-owned non-semantic control artifacts before provider conversion, with focused regression coverage and a separate doctor/session repair only if stored transcript cleanup is needed.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: a replay history containing assistant NO_REPLY text, the failure placeholder, or copied runtime-context scaffolding flows through normalizeAssistantReplayContent without a transient filter. I did not run a live Lark/custom-proxy session because the current-main code path is direct enough to establish the gap.

## Solution Assessment

Yes, the requested direction is the right narrow fix if implemented as a generic replay-boundary sanitizer rather than a channel/provider-specific patch. Stored transcript repair should be treated as an optional doctor path, not a startup mutation.
