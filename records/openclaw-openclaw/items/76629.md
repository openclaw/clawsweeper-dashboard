## Summary

Keep open. Current main has partial runtime-context protection, but the replay pipeline still preserves the reported assistant-side no-op/control artifacts before the next model turn.

## What This Changes

The issue asks OpenClaw to prune internal runtime/control messages, silent assistant sentinels, failed-turn placeholders, and stale runtime-context scaffolding from provider-bound replay history.

## Best Possible Solution

Add a generic embedded-runner replay-hygiene helper that removes only OpenClaw-owned non-semantic control artifacts before provider conversion, with focused regression tests and any stored transcript cleanup handled as an explicit doctor repair path.
