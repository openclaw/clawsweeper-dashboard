## Summary

Keep open. Current main still has only partial replay hygiene: it strips some runtime metadata and trailing stream-error sentinels, but provider-bound replay can still carry nonsemantic assistant artifacts or preserved runtime-context custom messages that match the reported failure class.

## What This Changes

The issue asks OpenClaw to prune internal runtime/control and no-visible-reply transcript artifacts before building provider replay context.

## Best Possible Solution

Add a focused embedded-runner replay-hygiene pass that drops only OpenClaw-owned nonsemantic artifacts before provider conversion, with a separate doctor/repair path for already-persisted polluted transcripts.

## Reproduction Assessment

Yes, source-level. Construct a session history with standalone assistant `NO_REPLY`, a mid-history `[assistant turn failed before producing content]`, or copied runtime-context assistant scaffolding, then pass it through the current replay sanitizer and LLM-boundary normalization; current source lacks a broad transient-artifact pruning step.

## Solution Assessment

Yes for the replay-hygiene portion. The narrow maintainable fix is a central filter before provider conversion; transcript-file doctor cleanup should be separate, and current-turn runtime-context semantics should be preserved unless explicitly changed by owners.
