## Repair Work Prompt

Fix the provider-bound replay hygiene issue at https://github.com/openclaw/openclaw/issues/76629. Add a generic core helper in the embedded runner replay path that removes only OpenClaw-owned nonsemantic artifacts from the in-memory provider-bound replay copy: openclaw.runtime-context custom messages where still present, assistant turns whose visible text is exactly the silent reply payload (`NO_REPLY` or the existing single-key silent envelope), assistant turns whose visible text is exactly STREAM_ERROR_FALLBACK_TEXT, and stale copied assistant scaffolding beginning with the known OpenClaw runtime-context or runtime-event preface. Keep the detector exact and conservative so legitimate user-authored or substantive assistant discussion of these strings is preserved. Wire it before provider replay conversion/hook context, add focused tests for string and block assistant content, and do not add a channel/provider-specific workaround or a config flag. Update transcript-hygiene docs and CHANGELOG.md if behavior changes are user-facing. Treat persisted doctor cleanup as a separate optional repair path unless the fix remains narrow and well covered.
