## Summary

Keep open. Current main has partial runtime-context protections, but the provider-bound replay path still lacks a generic filter for assistant-side silent sentinels, failure placeholders, and copied runtime-context/event scaffolding before model context construction.

## What This Changes

Request provider-bound replay pruning for hidden runtime-context custom messages, silent/no-reply assistant sentinels, failed-placeholder turns, and copied runtime-context/event scaffolding.

## Best Possible Solution

Add a generic replay-boundary hygiene helper for OpenClaw-owned non-semantic artifacts, with focused regression coverage; keep stored transcript repair as a separate doctor/session repair path if needed.

## Reproduction Assessment

Yes, at source level. A replay history containing assistant NO_REPLY text, the failure placeholder, or copied runtime-context scaffolding flows through normalizeAssistantReplayContent and into later provider-bound activeSession history without a transient filter.

## Solution Assessment

Yes, the narrow maintainable fix is a generic replay-boundary sanitizer rather than a Lark/provider-specific patch. A doctor repair can be considered separately for already-polluted stored transcripts.

## Repair Work Prompt

Fix provider-bound replay hygiene for this issue. Add a generic helper in the embedded runner replay path that removes only OpenClaw-owned non-semantic artifacts before provider conversion: openclaw.runtime-context custom messages, assistant visible text that is an exact silent reply payload such as NO_REPLY or {"action":"NO_REPLY"}, the standalone failure placeholder "[assistant turn failed before producing content]", and assistant text that is only/copied OpenClaw runtime-context or runtime-event scaffolding. Keep the fix generic in src/agents/pi-embedded-runner/replay-history.ts and/or the LLM-boundary normalization in run/attempt.ts; do not add Lark/provider-specific policy, do not mutate startup/load-time config, and do not weaken tool-call/result ordering. Add regression coverage for replay pruning and for preserving legitimate visible assistant replies/tool blocks. Update docs/reference/transcript-hygiene.md and CHANGELOG.md if behavior changes are user-facing. Treat stored transcript cleanup as a separate doctor/session repair only if needed.
