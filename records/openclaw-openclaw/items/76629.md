## Summary

Keep open. Current main still has only partial runtime-context protection: provider-bound replay can carry silent/no-op assistant rows, the failure-placeholder text, and copied runtime-context scaffolding through the central replay sanitizer into provider hooks.

## What This Changes

The issue requests core replay hygiene that prunes OpenClaw-owned internal/no-op control artifacts before provider context is built.

## Best Possible Solution

Add a generic replay-boundary sanitizer in the embedded runner that removes only OpenClaw-owned non-semantic control artifacts before provider conversion, with focused regression coverage and optional stored-transcript repair tracked separately if needed.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: a replay history containing assistant NO_REPLY text, the failure placeholder, or copied runtime-context scaffolding flows through normalizeAssistantReplayContent and sanitizeSessionHistory without a transient filter before provider hooks.

## Solution Assessment

Yes. The best fix is a narrow generic replay-boundary sanitizer, not a Feishu/Lark or provider-specific workaround; any persisted-session cleanup should be a separate doctor/repair path.
