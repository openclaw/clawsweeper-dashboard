## Summary

Keep open. Current main has partial runtime-context protections, but the embedded replay pipeline still lacks a generic transient-message pruning step for provider-bound history, so exact silent assistant sentinels, stream-error placeholders, copied runtime-context/event assistant prefaces, and runtime-context custom rows can survive into replay.

## What This Changes

Request core replay hygiene that removes internal runtime-context and no-op assistant artifacts from provider-bound session history, with optional repair for already polluted transcripts.

## Best Possible Solution

Add a generic embedded-runner replay hygiene helper that removes only OpenClaw-owned nonsemantic artifacts before provider conversion, with focused regression tests; handle any persisted transcript cleanup through a conservative doctor/repair path or a separate follow-up.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: construct replay history with assistant NO_REPLY text, the stream-error fallback placeholder, copied runtime-context/event preface text, or an openclaw.runtime-context custom row, and current main has no generic pass that removes those rows before provider replay. I did not run a live Lark/custom-proxy session in this read-only review.

## Solution Assessment

Yes, the narrow maintainable direction is a core replay-boundary sanitizer rather than a channel/provider-specific patch. A config flag is not required for the default invariant; stored transcript cleanup can be separate if it broadens the change.
