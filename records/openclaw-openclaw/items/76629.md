## What This Changes

## Best Possible Solution

Keep this open and add a central embedded-runner replay filter that removes only OpenClaw-owned nonsemantic artifacts before provider conversion, with persisted transcript cleanup handled through doctor/repair.

## Repair Work Prompt

Fix the provider-bound replay hygiene bug for https://github.com/openclaw/openclaw/issues/76629. Add a central embedded-runner replay filter before provider conversion that drops only OpenClaw-owned nonsemantic artifacts: `openclaw.runtime-context` custom replay messages, exact silent assistant payloads via the existing silent-token helper, the stream-error fallback text when it is a persisted assistant artifact, and assistant text whose visible content is only or starts with the modern OpenClaw runtime-context/event preface. Do not drop user messages or substantive assistant messages that merely discuss `NO_REPLY` or runtime-context text. Add focused regression coverage around `sanitizeSessionHistory`/`normalizeMessagesForLlmBoundary` and the replay-history helper. Update `docs/reference/transcript-hygiene.md` and `CHANGELOG.md` for the user-facing behavior change. Do not add a config flag unless maintainers explicitly request one; the default should be safe replay hygiene.
