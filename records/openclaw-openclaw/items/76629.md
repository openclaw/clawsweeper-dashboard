## Summary

Keep open. Current main has partial runtime-context and outbound sanitization, but the provider-bound replay path still lacks a generic pass that drops standalone silent/control assistant artifacts before the next model call.

## What This Changes

Provider-bound transcript replay should prune OpenClaw-owned internal, no-op, and stale control assistant messages before building model context.

## Best Possible Solution

Add a generic embedded-runner replay-hygiene helper that removes only OpenClaw-owned nonsemantic artifacts before provider conversion, with focused regression coverage and a separate doctor/repair path for already-persisted polluted transcripts.

## Solution Assessment

Yes. The narrow maintainable fix is core replay-boundary hygiene in `sanitizeSessionHistory` or the adjacent LLM-boundary normalization, while persisted transcript cleanup should stay in a separate doctor/repair path.
