Keep open. Current main still has partial protection only: runtime-context custom entries and heartbeat pairs are filtered, but provider-bound replay can still retain standalone silent assistant sentinels, synthesized failure-placeholder text, and copied runtime-context assistant scaffolding.

## What This Changes

The issue asks for provider-bound replay to prune OpenClaw-owned internal/no-op transcript artifacts before building model context.

## Best Possible Solution

Add a generic replay-boundary hygiene helper that removes only OpenClaw-owned non-semantic artifacts before provider conversion, then cover the exact reported message shapes with focused regression tests.
