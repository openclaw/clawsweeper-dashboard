## Summary

Keep open. Current main has partial runtime-context and outbound display sanitizers, but the provider-bound replay path still lacks a generic filter for standalone silent assistant payloads, stream-error fallback text, and copied runtime-context/event assistant scaffolding.

## What This Changes

Ensure provider-bound replay removes OpenClaw internal, runtime, and no-op control messages before building model context.

## Best Possible Solution

Add a generic embedded-runner replay hygiene pass that drops only OpenClaw-owned nonsemantic artifacts before provider hooks, with focused regression coverage and a separate transcript repair path for polluted stored sessions if needed.

Yes. ## Solution Assessment

Yes. The narrow maintainable fix is generic replay-boundary hygiene in the embedded runner; outbound sanitizer work such as #75755 is complementary and stored transcript cleanup should be handled separately.
