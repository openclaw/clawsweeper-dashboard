## Summary

Keep open. Current main still has only partial runtime-context and delivery cleanup; provider-bound replay does not prune the broader set of stale assistant/control artifacts described here.

## What This Changes

Provider-bound replay should prune internal runtime-context custom entries, silent/no-reply assistant sentinels, stream-error placeholders, and copied runtime-context/event scaffolding from session history.

## Best Possible Solution

Add a generic embedded-runner replay-hygiene helper that removes only OpenClaw-owned nonsemantic artifacts before provider conversion, with persisted transcript cleanup handled by a separate doctor/repair path.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: build a session history containing standalone assistant `NO_REPLY`, `[assistant turn failed before producing content]`, or copied runtime-context preface text, and current main has no replay-pruning step that removes those assistant rows before provider conversion.

## Solution Assessment

Yes. The narrow maintainable fix is core replay-boundary hygiene in the embedded runner; stored transcript repair should be separate so replay behavior improves without rewriting history unexpectedly.
