## Summary

Keep open: current main still forwards opencode-go block-boundary events without refreshing the stalled-stream idle timer, so the reported false abort remains a valid provider-wrapper bug rather than a duplicate of the already-merged missing-termination fix.

## What This Changes

The issue asks for the opencode-go stalled-stream watchdog to count block-boundary stream events as liveness so live streams are not aborted between token deltas.

## Best Possible Solution

Patch the provider-owned opencode-go stream wrapper so real post-progress boundary events refresh idle liveness, while retaining true-stall abort behavior and focused regression coverage.

## Reproduction Assessment

Yes, at source level. Current main forwards boundary events but only re-arms the idle timer for delta events, which matches the supplied failing timing sequence; I did not execute the repro test in this read-only review.

Yes. The narrowest maintainable fix belongs in extensions/opencode-go/stream-termination.ts with a regression test, not in global cron, session recovery, fallback policy, or non-opencode-go providers.
