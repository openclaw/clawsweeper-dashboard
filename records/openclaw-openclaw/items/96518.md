## Repair Work Prompt

Fix the opencode-go stalled-stream watchdog liveness gap in `extensions/opencode-go/stream-termination.ts`. Current main only treats `text_delta`, `thinking_delta`, and `toolcall_delta` as provider progress; after provider progress has begun, real block-boundary events such as `text_end`, `thinking_end`, `toolcall_start`, and `toolcall_end` should refresh idle liveness so a live stream is not aborted and a later `done` is not dropped. Preserve the existing first-event contract from https://github.com/openclaw/openclaw/pull/93965: synthetic preamble events before the first real provider delta must not shorten the longer first-event window. Add focused regression coverage in `extensions/opencode-go/stream-termination.test.ts` for a delta, boundary-event-only gaps under `idleTimeoutMs`, and a final `done` with no stalled error. Do not change global cron/session recovery, fallback policy, config/schema, non-opencode-go providers, or CHANGELOG.md.
