## Summary

Keep open: current main still has the delta-only opencode-go stream liveness predicate, and https://github.com/openclaw/openclaw/pull/96812 is an open unmerged closing PR rather than a merged fix.

## What This Changes

The issue reports that opencode-go block-boundary SSE events are forwarded but do not refresh the stalled-stream idle timer, allowing live streams to be aborted between token deltas.

## Best Possible Solution

Keep this issue as the canonical report until a focused opencode-go wrapper fix lands, preserving the longer first-event window while refreshing liveness for real post-progress boundary events.

## Reproduction Assessment

Yes. Current source inspection shows boundary events are forwarded but do not re-arm the timer, matching the included failing timing test; I did not run the test because this review is read-only.

## Solution Assessment

Yes. The right fix belongs in `extensions/opencode-go/stream-termination.ts`, not global cron or downstream stuck recovery; it should add regression coverage for post-delta boundary liveness and preserve first-event timing semantics.
