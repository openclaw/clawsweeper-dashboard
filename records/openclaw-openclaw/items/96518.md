## Label Justifications

## Summary

Keep open: current main still has the delta-only opencode-go stream liveness predicate, and the active fix PR is not merged or cleared, so the central user problem remains necessary to track.

## What This Changes

The issue reports that the opencode-go stalled-stream watchdog refreshes its idle timer only on token deltas, so live block-boundary events can be forwarded while the upstream request is later aborted and the final answer is dropped.

## Best Possible Solution

Land a narrow provider-local fix that refreshes idle liveness on post-delta provider-owned block-boundary events while preserving the longer first-event window before real provider progress.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: after a delta, boundary events are forwarded but do not refresh the idle timer, so the timer can abort before a later `done`.

## Solution Assessment

Yes. The best fix is a narrow change in `extensions/opencode-go/stream-termination.ts` plus regression coverage; global cron, fallback, and downstream stuck-session recovery are adjacent layers, not the failing predicate.
