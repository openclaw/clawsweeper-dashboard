## Summary

Keep open: current main still has the opencode-go delta-only idle re-arm predicate, and an open PR now owns the exact fix path but has not merged yet.

## What This Changes

The issue asks for the opencode-go stalled-stream watchdog to treat post-delta block-boundary SSE events as liveness so live streams are not aborted before their final `done` event.

## Best Possible Solution

Land a provider-owned wrapper fix that refreshes idle liveness for real post-progress boundary events while preserving the first-event timer contract, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes, at source level. Current main forwards boundary events but only re-arms idle liveness for deltas, matching the supplied failing wrapper timing sequence.

## Solution Assessment

Yes, the best layer is the opencode-go provider-owned stream wrapper. The fix should not move into global cron, fallback policy, or downstream stuck recovery, and it must preserve the first-event timeout invariant.
