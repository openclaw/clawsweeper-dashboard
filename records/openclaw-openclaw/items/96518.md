## Summary

Keep open: current main still has the opencode-go delta-only idle re-arm behavior, and the issue is now paired with an open closing PR that should be reviewed, fixed, or replaced before this canonical bug is closed.

## What This Changes

The issue reports that opencode-go block-boundary stream events do not re-arm the stalled-stream idle timer, so live provider streams can be aborted and completed answers dropped.

## Best Possible Solution

Keep this issue as the canonical bug and resolve it by landing a corrected opencode-go provider-wrapper PR that preserves the first-event window while refreshing idle liveness after real provider progress.

## Reproduction Assessment

Yes, at source level. Current main forwards boundary events but only re-arms idle liveness for delta events, matching the supplied failing timing sequence; I did not run tests in this read-only review.
