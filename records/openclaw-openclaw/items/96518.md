## What This Changes

The issue reports that opencode-go stream block-boundary events are forwarded but do not refresh the stalled-stream idle timer, allowing a live completion to be replaced by a stalled error.

## Best Possible Solution

Fix the provider-owned opencode-go wrapper so post-delta block-boundary events refresh idle liveness while preserving the longer first-event timer, with focused regression coverage and runtime proof.

## Reproduction Assessment

Yes, at source level. Current main forwards boundary events but only re-arms idle liveness for deltas, and the issue includes a focused failing Vitest sequence.

## Solution Assessment

Yes, the provider-owned opencode-go wrapper is the right layer. The repair should avoid treating pre-delta synthetic or boundary events as first provider progress.
