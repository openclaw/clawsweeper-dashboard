## Summary

Keep this open. Current main still only maps OpenClaw YOLO exec policy to Claude `bypassPermissions` and only allowlists OpenClaw MCP tools, so `exec-approvals.json` allowlist policy is not bridged into Claude Code internal Bash execution; no canonical duplicate or merged fix was found.

## What This Changes

The issue requests forwarding OpenClaw exec allowlist policy into Claude CLI backend sessions so Claude Code internal exec matches native OpenClaw exec enforcement.

## Best Possible Solution

Keep the issue open for maintainer security/design review, then implement a Claude CLI-owned permission bridge for fresh, resumed, and live stdio sessions or fail closed with clear diagnostics when equivalent enforcement cannot be provided.

Yes. The issue provides concrete live steps, and current main statically reproduces the missing bridge because the Claude CLI backend is launched without reading or evaluating `exec-approvals.json` for Claude internal Bash calls.

## Solution Assessment

Unclear. Forwarding policy is the right end state, but the safest implementation needs maintainer review against Claude Code's supported allow/disallow, permission-prompt, and hook semantics plus fail-closed behavior.
