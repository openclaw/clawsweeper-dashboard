## Summary

Keep open. Current main still does not forward OpenClaw `exec-approvals.json` allowlists into Claude CLI internal Bash execution; the implementation maps only OpenClaw YOLO exec policy to Claude `bypassPermissions`, leaving allowlist-mode Claude sessions on Claude Code's own permission path.

## What This Changes

Request to make `claude-cli` backend sessions enforce the same OpenClaw exec allowlist policy as native OpenClaw exec.

## Best Possible Solution

Add an Anthropic plugin-owned permission bridge for fresh, resumed, and live Claude CLI sessions, or fail closed with clear diagnostics when equivalent allowlist semantics cannot be represented.

## Reproduction Assessment

Yes, by source inspection and the issue's provided live steps. Current main keeps Claude permission mode unset for allowlist policy while native OpenClaw exec evaluates `exec-approvals.json`; I did not run a live Claude Code session in this read-only review.

## Solution Assessment

Unclear. Bridging or failing closed is the right end state, but the exact mapping to Claude Code permission rules or the stdio prompt tool needs maintainer security design first.
