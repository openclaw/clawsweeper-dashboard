## Summary

Keep open. Current main still does not bridge OpenClaw `exec-approvals.json` allowlist policy into Claude CLI native tool execution; it only maps fully permissive YOLO policy to Claude `bypassPermissions`, while the native OpenClaw exec path is the one that evaluates the allowlist.

## What This Changes

The issue requests forwarding OpenClaw exec allowlist policy into `claude-cli` backend sessions so Claude native exec decisions match OpenClaw native exec enforcement.

## Best Possible Solution

Add an Anthropic plugin-owned permission bridge for fresh, resumed, and live Claude CLI sessions that maps OpenClaw exec policy to Claude native tool decisions, or fail closed with clear diagnostics when equivalent allowlist semantics cannot be represented safely.

## Reproduction Assessment

Yes. Source inspection on current main shows `allowlist` policy leaves Claude CLI on Claude's native permission path while native OpenClaw exec evaluates `exec-approvals.json`; a live Claude-authenticated session was not rerun in this read-only review.

## Solution Assessment

Unclear. Bridging or failing closed is the right end state, but maintainers need to choose the Claude native tool semantics before a narrow implementation is safe.
