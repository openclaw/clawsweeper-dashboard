## Summary

Keep this open. Current main still maps only OpenClaw YOLO exec policy to Claude `bypassPermissions` and leaves allowlist-mode Claude CLI runs on Claude Code's own internal permission behavior, so the reported `exec-approvals.json` bridge remains missing.

## What This Changes

Request that OpenClaw forward per-agent exec allowlist policy into `claude-cli` backend sessions so Claude Code internal exec obeys the same allowlist as native OpenClaw exec.

## Best Possible Solution

Implement an Anthropic plugin-owned permission bridge for fresh, resumed, and live Claude CLI sessions, or fail closed with clear diagnostics when OpenClaw allowlist semantics cannot be represented safely.

## Reproduction Assessment

Yes. The issue gives concrete live steps, and current main statically reproduces the missing bridge because Claude CLI launch and live-session code do not read or evaluate `exec-approvals.json` for Claude internal Bash execution.

## Solution Assessment

Unclear. Forwarding the policy is the right end state, but the exact implementation should be chosen after maintainer security/design review of Claude Code permission hooks, generated allow/deny rules, and fail-closed behavior.
