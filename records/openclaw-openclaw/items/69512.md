## Summary

Keep open. Current main still only maps OpenClaw YOLO exec policy to Claude CLI `bypassPermissions`; allowlist-mode Claude sessions do not receive `exec-approvals.json` rules, and the related open control-request PR does not implement allowlisted native command execution.

## What This Changes

Request forwarding per-agent `exec-approvals.json` allowlists into Claude CLI backend sessions so Claude native exec decisions match OpenClaw allowlist policy.

## Best Possible Solution

Add an Anthropic plugin-owned permission bridge for fresh, resumed, and live Claude CLI sessions that maps OpenClaw exec policy to Claude native tool decisions, or fail closed with clear diagnostics when equivalent allowlist semantics cannot be represented safely.

## Reproduction Assessment

Yes, at source level. Current main omits Claude permission bridging for allowlist policy while native OpenClaw exec still evaluates `exec-approvals.json` and denies allowlist misses.

## Solution Assessment

Unclear. The requested parity is the right end state, but maintainers need to choose the Claude native-tool semantics: command-by-command allowlist bridge, deny/fail-closed, or a documented Claude permission-control surface.
