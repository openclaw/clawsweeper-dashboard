## Summary

Keep open: the maintainer PR fixes a real doctor migration repeat/noise problem, but the new npm install-record identity shortcut can misclassify conflicting legacy records and archive the legacy source without warning.

## What This Changes

The PR changes doctor legacy state migrations to import non-conflicting plugin-state sidecar rows, drop expired sidecar rows, and let richer SQLite plugin install records supersede covered legacy JSON records.

## Best Possible Solution

Keep the idempotent doctor migration fixes, but require a proven non-empty npm package-name match before treating differing same-version install records as covered.

## Reproduction Assessment

Yes from source inspection: install records are loose legacy records, parseRegistryNpmSpec can return null, and the PR's equality check can treat two undefined npm names as the same package. I did not run the PR tests in this read-only review.

## Solution Assessment

No: the overall repair direction is good, but the install-index shortcut needs a stricter identity predicate before it is the best fix.
