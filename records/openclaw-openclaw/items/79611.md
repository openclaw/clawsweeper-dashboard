## Summary

Keep open: current main still does not provide runtime provider failover for active-memory, while the remaining p95/latency-budget behavior is a new telemetry contract rather than a narrow bug fix.

## What This Changes

The issue asks for active-memory to use runtime provider failover and to surface structured latency-budget or timeout signals instead of dropping recall silently.

## Best Possible Solution

Define the active-memory fallback and telemetry contract, then reuse the existing model-fallback/auth-profile machinery and add focused status, docs, and regression coverage for the chosen behavior.

## Reproduction Assessment

Yes for the no-failover path by source inspection: active-memory resolves one model and calls runEmbeddedPiAgent directly, so provider cooldown or structured failure cannot consult runWithModelFallback. The latency-budget warning is source-confirmed absent, but exact p95 behavior needs design.

## Solution Assessment

Unclear until maintainers choose the contract. The narrowest maintainable implementation would likely adapt existing runWithModelFallback/auth-profile cooldown semantics rather than adding plugin-specific provider discovery.
