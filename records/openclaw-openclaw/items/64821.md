## Summary

Current `main` still appears vulnerable to noisy stdout on non-JSON single-value exec secret providers, and I did not find a landed fix that would conservatively justify closing #64821.

## Evidence

- **Plain exec fallback still returns the entire stdout buffer:** `parseExecValues()` trims all stdout and, when `jsonOnly` is false for a single requested id, falls back to returning that full trimmed string if JSON parsing fails. There is no last-line extraction or rejection of prefixed log noise, so a `logs + value` payload would still be accepted as the secret/config string.
  - file: [src/secrets/resolve.ts:562](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/src/secrets/resolve.ts#L562)
