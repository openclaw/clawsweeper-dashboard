Full review comments:

- **[P2] Preserve quoted spans that close before suffix text:** `packages/agent-core/src/harness/prompt-template-arguments.ts:34`
  - body: Current main treats single quotes like simple shell grouping, so `parseCommandArgs("'foo bar'baz next")` produces `["foo barbaz", "next"]`. In this PR, `firstWhitespaceSpanCloser` only accepts a closing apostrophe when `closesConcatSpan` says the next char is whitespace/end, so the quote after `bar` is ignored when it is followed by `baz`; the same input is split as literal quote text instead. That is unrelated to prose apostrophes and changes an existing exported prompt-template argument form, so please preserve it or explicitly document and test the compatibility break.
