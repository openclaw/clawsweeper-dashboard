## What This Changes

The PR changes the shared prompt-template slash-command argument parser so prose apostrophes are preserved while shell-style quoted spans still group arguments.

## Best Possible Solution

Land the shared parser fix after maintainers confirm the new prose-aware single-quote grammar is the intended compatibility tradeoff.

## Reproduction Assessment

Yes from source: current main unconditionally enters quote mode on apostrophes in parseCommandArgs, and those parsed args feed prompt-template expansion before model-bound text is sent. The PR body also supplies before/after live output through loadPromptTemplates and expandPromptTemplate.

## Solution Assessment

Yes, with maintainer sign-off: the shared parser is the right layer because both src/agents prompt-template expansion and agent-core harness formatting use it, and the sibling native-command parser is a different surface. The compatibility-sensitive part is the exact exported single-quote grammar.
