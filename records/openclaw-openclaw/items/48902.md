## Summary

Keep this PR open. The branch is useful but stale: current main already covers some theme/search/base palette localization, while slash-command descriptions and overview attention strings still remain English on main and the PR needs a careful rebase plus i18n-pipeline cleanup before it is mergeable.

## What This Changes

The PR branch adds Control UI locale keys and rendering changes for theme mode labels, topbar search accessibility text, command palette labels/descriptions, and overview attention messages.

## Best Possible Solution

Rebase the PR onto current main, keep the already-shipped theme/search/base palette localization, add the remaining slash-command and overview-attention localization through the current Control UI i18n source/generator flow, preserve existing palette test exports, and add focused tests for locale changes on those surfaces.

## Reproduction Assessment

Yes. A high-confidence static reproduction is available: current main still renders slash command descriptions from `command.description` and still builds several overview attention titles/descriptions from literal English strings; switching to a non-English locale would leave those strings in English.

## Solution Assessment

No, not in the current branch shape. The direction is right, but the best maintainable solution is to rebase onto current main's newer i18n keys/tests, use the generated locale pipeline, and avoid leaving the theme toggle aria prefix in English.
