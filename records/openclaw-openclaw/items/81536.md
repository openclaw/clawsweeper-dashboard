## Summary

Keep this PR open, but do not merge it as-is: it addresses a real CLI JSON stdout pollution path, yet the implementation permanently reroutes console output whenever any argv token looks like `--json`, including cases that current CLI contracts intentionally treat as parse-only or pass-through arguments.

## What This Changes

The PR changes `runCli` to call `routeLogsToStderr()` immediately after console capture when `--json` appears in argv, before lazy plugin command registration runs.

## Best Possible Solution

Keep the PR open and adjust the fix so early stderr routing is limited to the lazy plugin-registration window or otherwise uses command-aware JSON-output semantics, then add targeted regression coverage.

## Reproduction Assessment

Yes. The source path shows plugin command registration happens before `parseAsync`/`preAction`, and the PR body gives a concrete `openclaw memory search ... --json` command with before/after-style terminal evidence.

## Solution Assessment

No. The direction is right, but the current patch is broader than the existing JSON-output contract; a safer fix should keep plugin-loader diagnostics off stdout without permanently rerouting parse-only or pass-through `--json` invocations.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep early stderr routing scoped to JSON output mode:** `src/cli/run-main.ts:647-648`
  - body: This calls `routeLogsToStderr()` permanently as soon as any argv token looks like `--json`. That is broader than the existing `preAction` contract: `config set` marks `--json` as a parse-only alias for `--strict-json`, and the current tests assert that it must not route logs to stderr. With this patch, `openclaw config set gateway.auth.mode '"local"' --json` would send its normal `runtime.log` success output to stderr. Please scope the early flip to the plugin-registration window or to commands that are actually JSON-output mode, while preserving the existing `--` terminator behavior.
  - confidence: 0.9
