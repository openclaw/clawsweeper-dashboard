## Summary

Keep open. Current main still treats any memory-flush compaction `phase: "end"` event as completed, while the event producer and adjacent follow-up consumer require `completed === true`; the latest shipped release still has the same loose memory-flush consumer behavior and no canonical PR or issue owns this exact replay path.

## What This Changes

Prevent a user message from being redelivered after memory flush and hook-triggered compaction.

## Best Possible Solution

Gate memory-flush session rotation and queued-followup refresh on `completed === true` compaction end events, with regression coverage for missing, false, aborted, and legitimate completed events.

## Reproduction Assessment

Yes for a high-confidence source-level path: a memory-flush run that receives `stream: "compaction"` with `phase: "end"` and `completed` false or missing still rotates and refreshes session state on current main. A full live replay would need the reporter's redacted transcript excerpt.

## Solution Assessment

Yes. The narrow maintainable fix is to honor the existing `completed === true` compaction contract in the memory-flush consumer instead of adding prompt-content dedupe that could suppress intentional repeated messages.
