## Summary

Keep open. Current `main` still matches the report: the memory-flush compaction callback treats any `phase: "end"` as a completed rotation, even though the compaction event already exposes `completed`, and active-run queue dedupe can still allow same-content redelivery under a new message id.

## What This Changes

The issue reports duplicate delivery of the same user message after a memory flush and hook-timed compaction sequence.

## Best Possible Solution

Add a focused idempotency guard at the memory-flush compaction/session-rotation and queued follow-up boundary, honoring only completed compaction events while preserving legitimate repeated prompts.

## Reproduction Assessment

Yes for a synthetic regression: the issue provides timing, ids, hook markers, same-content/new-id behavior, and the memory-flush plus compaction sequence. Exact live reproduction would need a redacted transcript excerpt from the reporter.

## Solution Assessment

Yes, a narrow runtime fix is the best direction. The safer alternative to global prompt dedupe is a session-boundary guard that uses the existing compaction `completed` contract and replay context.
