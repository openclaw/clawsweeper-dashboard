## Summary

Keep open. Current main still has the loose memory-flush compaction-end handling described in the prior review, and adjacent code already exposes and uses a stricter completed-compaction contract that this path ignores.

## What This Changes

The issue asks OpenClaw to stop re-injecting a prior user message after memory flush and hook-triggered compaction.

## Best Possible Solution

Add a focused idempotency guard at the memory-flush compaction/session-rotation and queued follow-up boundary, honoring only completed compaction events while preserving legitimate repeated prompts.

## Reproduction Assessment

Yes for a synthetic source-level regression path: a memory-flush run that receives phase "end" with completed false or missing still rotates/refreshes session state on current main. Exact live reproduction would need a redacted excerpt of the reporter-local transcript.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing completed=true compaction contract in the memory-flush path rather than adding broad prompt-content dedupe that could block intentional repeated prompts.
