## Summary

Keep open. Current main still has adjacent safeguards, but the reported hook-timed memory-flush plus compaction duplicate delivery is not covered: the memory-flush path still treats any compaction end event as completion, normal follow-up handling uses stricter completion semantics, and active-run queue dedupe remains message-id based while the report used a new second id.

## What This Changes

Prevent a user message from being delivered a second time after memory flush and hook-triggered compaction/session rotation.

## Best Possible Solution

Add a narrow idempotency guard at the memory-flush, compaction/session-rotation, and follow-up queue boundary so maintenance or hook-triggered compaction cannot turn the prior visible user prompt into a fresh user turn, while preserving legitimate repeated user messages.

## Reproduction Assessment

Yes for a synthetic regression: the report provides enough timing, ids, and state markers to model one visible user prompt, memory flush, hook-originated compaction/session rotation, and same-content re-delivery. Exact live reproduction would need a redacted trajectory or transcript excerpt from the reporter.

## Solution Assessment

Yes, a focused runtime fix is the best direction. The safest implementation should honor the existing completed compaction event contract and add an idempotency key around maintenance/session-boundary replay rather than applying broad prompt suppression.

## Repair Work Prompt

Fix the duplicate delivery reported in https://github.com/openclaw/openclaw/issues/74370. The observable bug is that after a pre-compaction memory flush and hook-originated compaction/session rotation, the previous visible user prompt can be re-delivered as a fresh user turn under a new message id. Keep the fix narrow: honor the existing compaction event contract (`phase: "end"` plus `completed === true`) in the memory-flush path, and add an idempotency guard around maintenance/session-boundary replay or queued follow-up refresh only if needed. Do not switch normal active-run queueing to broad prompt-based dedupe in a way that would drop legitimate repeated user sends. Add focused regression coverage for an incomplete/aborted compaction end during memory flush and for the reported same-content/new-id replay across memory flush plus compaction/session rotation. Include a user-facing CHANGELOG.md fix entry with reporter credit. Do not change dependency versions, plugin APIs, or unrelated compaction policy.
