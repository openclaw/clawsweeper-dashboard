## Summary

Keep open. Current main still treats any memory-flush compaction `phase: "end"` event as a successful compaction, while the compaction event producer and adjacent follow-up runner require `completed === true`; no merged PR or canonical duplicate owns this exact replay bug.

## What This Changes

The issue reports that a single user message can be redelivered after memory flush and hook-originated compaction.

## Best Possible Solution

Gate memory-flush session rotation and queued-followup refresh on `completed === true` compaction end events, with regression coverage for incomplete, aborted, and missing-completion events while preserving legitimate repeated prompts.

## Reproduction Assessment

Yes for a source-level synthetic path: a memory-flush run that receives `stream: "compaction"` with `phase: "end"` and `completed` false or missing still rotates and refreshes session state on current main. A full live replay would need the reporter's redacted transcript excerpt.

## Solution Assessment

Yes. The narrow maintainable fix is to honor the existing `completed === true` compaction contract in the memory-flush consumer instead of adding prompt-content dedupe that could block intentional repeated messages.

## Repair Work Prompt

Fix the memory-flush duplicate replay path reported in https://github.com/openclaw/openclaw/issues/74370. In `runMemoryFlushIfNeeded`, only treat a compaction end event as completed when `evt.stream === "compaction"`, `phase === "end"`, and `evt.data.completed === true`; do not rotate the session, increment compaction count, or refresh queued followups for missing, false, aborted, or incomplete compaction end events. Add focused regression coverage in the memory-flush helper tests for missing `completed`, `completed: false`, and `completed: true`, and keep legitimate repeated user prompts deliverable by avoiding broad prompt-content dedupe. Check related https://github.com/openclaw/openclaw/pull/67706 so any new overflow/timeout recovery end events still follow the same completion contract. Include a changelog entry if the fix is user-visible under current OpenClaw policy.
