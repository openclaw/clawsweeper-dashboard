## Summary

Keep open. Current main still rotates/refreshes memory-flush session state on any compaction `phase: "end"`, even though adjacent producer and consumer code already distinguish successful compaction with `completed === true`.

## What This Changes

The issue asks OpenClaw to stop redelivering the same user message after memory flush and hook-originated compaction complete in the same session.

## Best Possible Solution

Honor only successful compaction events at the memory-flush session-rotation boundary, add regression coverage for missing/false `completed`, and avoid broad prompt-content dedupe that could block intentional repeated prompts.

## Reproduction Assessment

Yes, for a source-level reproduction path: current main rotates and refreshes queued follow-up state when a memory-flush run receives compaction `phase: "end"` without `completed: true`. The exact live transcript is unavailable, but the synthetic path matches the reported timing boundary.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing `completed === true` compaction contract in the memory-flush path and update the helper tests, rather than adding global same-prompt dedupe.
