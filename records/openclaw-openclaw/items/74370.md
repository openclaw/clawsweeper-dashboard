Fix openclaw/openclaw#74370. The observable bug is duplicate delivery of the same user prompt after memory flush plus hook-timed compaction, where the second delivery has a new message id. Keep the fix narrow in the auto-reply memory flush, compaction event, and queued follow-up/session-rotation boundary. At minimum, make the memory-flush compaction completion guard honor evt.data.completed === true instead of any phase=end event, and add regression coverage for an aborted or hook-timed compaction end that must not rotate/refresh queued followups into re-delivery.
