## Summary

Keep open. Current main still treats any memory-flush compaction `phase: "end"` event as successful, while the compaction event contract and adjacent runner path require `completed === true`; no merged PR or canonical duplicate appears to own this exact replay bug yet.

## What This Changes

The issue reports that a single user prompt can be delivered a second time after memory flush and hook-originated compaction.

## Best Possible Solution

Gate memory-flush session rotation and queued-followup refresh on `completed === true` compaction events, with focused regression coverage that preserves legitimate repeated prompts.

## Reproduction Assessment

Yes for a source-level synthetic path: a memory-flush run that receives `stream: "compaction"` with `phase: "end"` and `completed` false or missing still rotates/refreshes session state on current main. A full live replay would need the reporter's redacted transcript excerpt.

## Solution Assessment

Yes. The narrow maintainable fix is to honor the existing `completed === true` compaction contract in the memory-flush consumer instead of adding broad prompt-content dedupe that could block intentional repeated messages.
