## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/74370. Observable bug: after a memory flush plus hook-originated compaction/session rotation, the previous visible user prompt can be delivered again under a new message id. Keep the change narrow: make the memory-flush compaction callback honor `evt.data.completed === true`, and add an idempotency guard at the memory-flush/session-rotation/follow-up queue boundary so maintenance or hook-triggered compaction cannot replay the last visible user prompt as a fresh user turn. Do not add broad global prompt suppression, do not block legitimate repeated user messages, and do not move owner-specific behavior into core unless a generic seam is required. Add focused regression coverage for a same-content/new-id replay after memory flush plus compaction, and update `CHANGELOG.md` because this is a user-visible bug fix.
