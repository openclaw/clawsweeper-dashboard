## Summary

Keep open. The fix direction is right, but this patch does not appear to catch the current production Anthropic transport failure shape, so the linked cache-miss bug can still surface instead of retrying with stripped thinking blocks.

## What This Changes

The PR wraps the embedded-runner stream function with the existing Anthropic thinking-block recovery helper and adds focused tests for the retry gate.

## Best Possible Solution

Keep the one-shot retry approach, but make it handle the actual Anthropic transport/event-stream failure path before user-visible content is emitted and cover that path with a transport-level regression test.

## Reproduction Assessment

No live reproduction. The linked issue has a concrete provider log and current-main source shows the preserved-thinking replay path, but I did not run a live Anthropic cache-miss session.

## Solution Assessment

No. The requested direction is narrow and appropriate, but this implementation only demonstrates rejection-style recovery and misses the current transport stream/error-event path.
