## Summary

## What This Changes

The issue reports that session-store saves eagerly collect maintenance preserve keys, causing ordinary writes to hit subagent registry reads before maintenance needs protected session keys.

## Best Possible Solution

Land a narrow rebased fix in the current applyFileBackedSessionStoreMaintenance/applyEnforcedMaintenance boundary with focused regression coverage and redacted workload or profiler proof, then close this issue as implemented.

## Reproduction Assessment

Yes. Source inspection shows current main still collects preserve providers from enforced save-side maintenance before prune, cap, or disk-budget work proves provider keys are needed; no live profiler replay was run.

## Solution Assessment

Yes in direction. Deferring preserve-key collection until maintenance needs protected keys is the narrow maintainable fix, but the issue should stay open while the linked candidate PR is reviewed, landed, or replaced.
