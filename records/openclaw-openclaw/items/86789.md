## Summary

Keep open: current main and the latest release still collect session-maintenance preserve keys on save-side enforced maintenance before pruning, capping, or disk-budget work has proven it needs provider state. The linked open pull request remains the active candidate fix, but it is still unmerged and conflicting with current main.

## What This Changes

Defer save-side session-maintenance preserve-provider scans until stale pruning, cap maintenance, or disk-budget enforcement actually needs protected keys.

## Best Possible Solution

Rebase and review https://github.com/openclaw/openclaw/pull/86795, or land an equivalent narrow fix in the current session maintenance operation, then close this issue as implemented.

## Solution Assessment
