## Summary

Keep open, but do not merge as-is: the PR mixes Android APK artifact automation with a fork-maintenance issue writer, has a definite shell bug in the new upstream-watch workflow, and touches signing/release/issue-writing automation while official Android APK distribution remains a maintainer policy item tracked by #9443.

## What This Changes

The PR adds a manual/push Android APK artifact workflow, a scheduled fork upstream-watch workflow that opens or comments on issues, and Android README instructions for GitHub APK builds.

## Best Possible Solution

Keep #9443 as the canonical official Android APK distribution tracker; if maintainers want this PR, split out fork-only automation, fix the workflow bug, and align APK artifacts with an approved signing/distribution policy and docs.

## Reproduction Assessment

Yes for the review finding: the added upstream-watch shell script has a statically visible invalid here-doc terminator, which will fail when the fork-behind branch is executed. For the APK release paths, reproduction is incomplete because only the third-party debug Gradle task was reported as run.

## Solution Assessment

No. The narrow maintainable path is a maintainer-approved Android distribution workflow tied to #9443; this PR adds partial fork-oriented automation plus signing and issue-writing surfaces before that policy is settled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Close the heredoc before updating the issue:** `.github/workflows/upstream-watch.yml:79`
  - body: The closing `EOF` for this `cat <<EOF` block is indented, so Bash will not recognize it as the here-doc terminator.
