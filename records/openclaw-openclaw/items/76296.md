## Summary

Keep this PR open but not merge-ready: it adds useful Android APK build direction, yet the patch has a definite shell bug, lacks the required user-facing changelog entry, and changes APK signing/artifact distribution plus issue-writing workflow automation while the canonical Android APK release request remains open.

## What This Changes

The PR adds a manually runnable Android APK artifact workflow, a scheduled fork upstream-watch workflow that writes issues, and Android APK build instructions for forks.

## Best Possible Solution

Align any official APK artifact workflow with #9443, keep fork-only update automation out of the canonical repo or split it, and require validated play/third-party debug and release paths plus changelog/docs after release approval.

## Reproduction Assessment

Yes for the blocking review bug: the added upstream-watch here-doc terminator is indented and a matching `bash -n` probe fails. APK release behavior is not fully reproduced because the PR only reports the third-party debug Gradle task.

## Solution Assessment

No. The maintainable direction is a maintainer-approved Android distribution workflow tied to the canonical APK release tracker, not a mixed fork helper plus signing/artifact workflow before that policy is settled.
