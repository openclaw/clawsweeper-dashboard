Full review comments:

- **[P1] Install the SDK platform the app compiles against:** `.github/workflows/android-apk-build.yml:64`
  - body: Current main sets `compileSdk = 37` and the existing Android CI installs `platforms;android-37.0`, but this workflow installs only `platforms;android-36`. Align the workflow SDK install with the current Android build contract or the APK workflow can fail after merge.
  - confidence: 0.9
- **[P1] Keep signing secrets out of the job-wide environment:** `.github/workflows/android-apk-build.yml:43-48`
  - body: The Android signing passwords are exported as job-level environment variables, making them available to checkout, setup actions, Gradle setup, and every shell step. Move these secrets into only the release-signing and release-build steps that need them.
  - confidence: 0.88
- **[P1] Gate APK artifact publishing before enabling main pushes:** `.github/workflows/android-apk-build.yml:20-29`
  - body: The docs describe fork setup, but this workflow also runs on pushes to `main`, where fallback inputs publish a thirdParty debug APK artifact from the canonical repository. Remove the push trigger, gate it to forks, or get explicit maintainer approval for official APK distribution.
  - confidence: 0.87
- **[P1] Keep fork upkeep issue automation out of the canonical repo:** `.github/workflows/upstream-watch.yml:5-10`
  - body: The upstream-watch workflow grants `issues: write` and runs on a schedule without a repository guard, even though its purpose is fork maintenance. Add a canonical-repo guard or keep this workflow out of OpenClaw core.
  - confidence: 0.84
