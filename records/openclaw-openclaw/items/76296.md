## Summary

Keep open for maintainer review. The PR is a real attempt at the Android APK distribution request, but it currently has a reproducible workflow shell bug, lacks after-fix workflow proof, and touches Android signing/artifact distribution plus scheduled issue-writing automation that needs explicit release/security policy review.

## What This Changes

The PR adds a manually dispatched Android APK artifact workflow for Play/third-party debug and release variants, a scheduled upstream-watch issue workflow, and fork-oriented APK build instructions.

## Best Possible Solution

Keep APK distribution work tied to the canonical Android release tracker, split fork-maintenance automation out unless maintainers explicitly want it, and merge only a hardened workflow with approved signing/provenance policy and real workflow-run proof.

## Reproduction Assessment

Yes for the blocking review bug: a shell syntax probe matching the indented here-doc terminator fails with an unterminated here-doc. The broader APK release behavior is not reproduced because the PR only reports a local Gradle debug build, not a workflow run.

## Solution Assessment

No as-is. The branch mixes APK artifact/signing automation with fork issue automation before the release policy in https://github.com/openclaw/openclaw/issues/9443 is settled, and it still needs a shell fix plus real workflow proof.
