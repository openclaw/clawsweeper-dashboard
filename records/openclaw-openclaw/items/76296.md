Full review comments:

- **[P1] Scope signing secrets to the release steps:** `.github/workflows/android-apk-build.yml:43-46`
  - body: These signing passwords are set on the whole job, so every setup action and debug build step receives credentials that only release signing needs. Move the secrets into the release-signing/build steps so third-party setup actions and non-release runs cannot read them from the environment.
  - confidence: 0.91
- **[P1] Avoid templating dispatch inputs into bash:** `.github/workflows/android-apk-build.yml:127-128`
  - body: The repository's security-scm-fast run already fails here with zizmor template-injection findings. Pass the workflow inputs through step env or GitHub outputs and validate them before the shell consumes them, instead of expanding `${{ github.event.inputs.* }}` directly inside the run block.
  - confidence: 0.95
- **[P2] Install the app's compile SDK platform:** `.github/workflows/android-apk-build.yml:67`
  - body: Current Android source uses `compileSdk = 37`, and the existing Android CI installs `platforms;android-37.0`. Installing only `platforms;android-36` leaves fresh runners without the required compile platform, so the proposed APK build can fail before uploading artifacts.
  - confidence: 0.86
