Full review comments:

- **[P1] Install the SDK platform the app compiles against:** `.github/workflows/android-apk-build.yml:64`
  - body: Current main sets `compileSdk = 37`, but this workflow installs only `platforms;android-36`, so the APK workflow is not aligned with the current Android build contract and can fail after merge. Match the SDK setup used by the existing Android CI lane.
  - confidence: 0.9
- **[P1] Keep signing secrets out of the job-wide environment:** `.github/workflows/android-apk-build.yml:43-48`
  - body: The Android signing passwords are exported as job-level environment variables, which makes them available to checkout, setup actions, Gradle setup, and every shell step in the job. Move these secrets into only the release-signing/build steps that need them.
  - confidence: 0.88
- **[P1] Gate APK artifact publishing before enabling main pushes:** `.github/workflows/android-apk-build.yml:20-29`
  - body: The docs describe fork setup, but this workflow also runs on pushes to `main`, where the fallback inputs publish a thirdParty debug APK artifact from the canonical repository. Remove the push trigger, gate it to forks, or get explicit maintainer approval for official APK distribution.
  - confidence: 0.87
- **[P1] Keep fork upkeep issue automation out of the canonical repo:** `.github/workflows/upstream-watch.yml:5-10`
  - body: The upstream-watch workflow grants `issues: write` and runs on a schedule without a repository guard, even though its purpose is fork maintenance. Add a canonical-repo guard or keep this workflow out of OpenClaw core.
  - confidence: 0.84
