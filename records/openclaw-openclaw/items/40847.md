## Summary

Keep open. The PR is aimed at a real diagnostic gap, but the current patch leaves key zero-output image paths silent and bypasses the normal OpenClaw warning logger.

## What This Changes

The PR adds warning diagnostics in `src/media-understanding/runner.ts` when selected image attachments produce no media-understanding outputs.

## Best Possible Solution

Implement diagnostics at the runner decision points: warn before the no-provider early return, warn when every selected-image attempt is skipped or empty, preserve failure context, use `logWarn`, add focused regression tests, and keep successful routing unchanged.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: selected images with no resolved entries return at `src/media-understanding/runner.ts:970`, and selected images whose providers return empty output become skipped attempts at `src/media-understanding/runner.ts:823`.

## Solution Assessment

No. The requested direction is narrow and maintainable, but this patch places one warning after an early return, misses the all-skipped zero-output path, bypasses `logWarn`, and omits the required changelog entry.
