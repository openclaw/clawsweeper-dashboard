## Summary

Keep open: current main still has the diagnostic gap, but this PR is not merge-ready because one new warning branch is unreachable and the all-skipped zero-output path still stays silent.

## What This Changes

The PR adds warning diagnostics in `src/media-understanding/runner.ts` when selected image attachments produce no media-understanding outputs.

## Best Possible Solution

Implement diagnostics at the runner decision points: warn before the no-provider early return, warn for all-skipped empty-output attempts, keep successful/native-vision routing unchanged, and cover it with focused regression tests plus changelog.

## Reproduction Assessment

Yes. Source inspection shows selected images with no resolved entries return at `src/media-understanding/runner.ts:970`, while empty provider output becomes skipped attempts at `src/media-understanding/runner.ts:823` and bypasses the failed-only warning at `src/media-understanding/runner.ts:1014`.

## Solution Assessment

No. The requested diagnostic direction is narrow, but this patch places one warning after an early return, misses the all-skipped path, and bypasses the runner’s normal `logWarn` surface.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P2] Move the no-provider warning before returning:** `src/media-understanding/runner.ts:817-821`
  - body: When no image providers resolve, `runCapability` returns at the existing `resolvedEntries.length === 0` guard before this aggregate block runs. This means the new no-provider diagnostic never fires for the scenario it is meant to explain.
  - confidence: 0.94
- **[P2] Warn when every image attempt is skipped:** `src/media-understanding/runner.ts:801-821`
  - body: Providers that return empty output are recorded as `outcome: "skipped"`. If every attempt is skipped, `failedAttempts` is empty and `allAttempts` is non-empty, so this block emits no warning for a zero-output image run.
  - confidence: 0.92
- **[P3] Route diagnostics through logWarn:** `src/media-understanding/runner.ts:814-821`
  - body: The existing runner warning path uses `logWarn`, which reaches OpenClaw’s logging surface. Calling `console.warn` directly can make these diagnostics invisible to operators relying on the normal logger.
  - confidence: 0.82
- **[P3] Add the required changelog entry:** `src/media-understanding/runner.ts:801`
  - body: This changes user-visible diagnostics for image-understanding failures, but the branch only changes the runner. OpenClaw policy requires a `CHANGELOG.md` entry for user-facing feature or fix behavior.
  - confidence: 0.87
