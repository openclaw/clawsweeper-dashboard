## Summary

Keep open. The requested image-understanding diagnostic is still useful and not fully implemented on current main, but this PR’s patch misses the no-provider and all-skipped zero-output paths, bypasses the existing logger, and lacks the required changelog entry.

## What This Changes

The PR adds warning diagnostics in `src/media-understanding/runner.ts` when selected image attachments produce no media-understanding outputs.

## Best Possible Solution

Implement diagnostics at the actual runner decision points: warn before the no-provider early return, warn when selected image attempts all skip or return empty output, preserve failed-attempt context, use `logWarn`, add focused regression tests, and keep successful routing/model selection unchanged.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: selected images with no resolved provider entries return at `src/media-understanding/runner.ts:970`, and selected images whose providers return empty output become skipped attempts at `src/media-understanding/runner.ts:823` without a warning-level log.

## Solution Assessment

No. The requested direction is narrow and maintainable, but this patch places one warning after an early return, misses the all-skipped zero-output path, uses `console.warn` instead of `logWarn`, and omits the required changelog entry.
