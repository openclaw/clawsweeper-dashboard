## Summary

Keep open: current main has explicit `MEDIA:` delivery, message-tool attachment parameters, and scoped artifact downloads, but it does not implement automatic cross-channel hosting or attachment of arbitrary local path text. That remaining behavior is a new product and security design, not a narrow bug cleanup.

## What This Changes

Add smarter file sharing so agent-created local files can be hosted by OpenClaw or automatically attached in the active channel instead of only surfaced as local paths.

## Best Possible Solution

Design a secure artifact-sharing layer that converts explicit agent file outputs into scoped, size-limited channel attachments or short-lived links through the existing media, message-tool, and artifact seams.

## Reproduction Assessment

Not applicable: this is a feature request, not a report of broken documented behavior. Source and docs show explicit media mechanisms exist, but no current contract promises automatic sharing of arbitrary local path text.

## Solution Assessment

Unclear: the requested direction is useful, but implementing it as a broad text parser would be risky. The maintainable path needs explicit product and security policy for allowed roots, storage, TTLs, channel upload/link fallback, and user visibility.
