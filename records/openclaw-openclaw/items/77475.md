## Summary

Keep open: current main has explicit media attachment, message-tool, and transcript artifact surfaces, but automatic cross-channel sharing/hosting of arbitrary agent-mentioned local paths is still an unimplemented product and security design.

## What This Changes

The issue requests smarter file sharing so agent-created local files can be automatically hosted or attached through the active channel instead of only exposing a local filesystem path.

## Best Possible Solution

Design a secure artifact-sharing layer that exposes explicit agent-produced outputs as scoped, size-limited per-channel attachments or short-lived authenticated links using the existing media, message-tool, and artifact seams.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken documented behavior report. Source inspection shows explicit media and artifact paths exist, but no current-main contract promises automatic sharing of arbitrary local path text.

## Solution Assessment

Unclear: the requested direction is useful, but implementing it as a quick path parser would be risky. The maintainable path needs explicit product and security policy for allowed roots, storage, TTLs, and channel fallback behavior.
