## Summary

Keep open: the PR targets a real source-reproducible gateway diagnostic gap, but it is not merge-ready because the patch can report a missing-plugin install hint for policy-blocked official external channel owners and the supplied proof is unit/check output only.

## What This Changes

The PR changes the gateway web-login unavailable-provider path to include official external plugin repair hints for explicitly configured channels and adds start/wait regression tests.

## Best Possible Solution

Keep the gateway web-login hint, but derive install guidance only from the existing missing-owner helper and cover the policy-blocked case so diagnostics match the plugin activation state.

## Reproduction Assessment

Yes. Source inspection reproduces the current-main path: a configured WhatsApp channel with no loaded QR-capable channel plugin makes `web.login.start` or `web.login.wait` return the generic unavailable-provider error.

## Solution Assessment

No, not as written. The gateway handler is the right narrow surface, but the fallback to a generic catalog install hint bypasses the existing missing-owner contract; use only the missing-owner helper for this diagnostic.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep install hints limited to missing owners:** `src/gateway/server-methods/web.ts:48-51`
  - body: This fallback runs when `resolveMissingOfficialExternalChannelPluginRepairHint` returns `null`, including cases where the official external channel owner exists but is disabled, untrusted, or blocked by plugin policy. Please remove the `resolveOfficialExternalPluginRepairHint(channelId)` fallback here and add coverage for a policy-blocked owner.
