## Best Possible Solution

Keep the provider-safe projection, but scan raw serialized schemas for unsupported runtime diagnostics before returning normalized provider output, with a focused regression test for mixed composition branches.

## Reproduction Assessment

Yes, source-reproducible: on the PR head, projectRuntimeToolInputSchema normalizes the parsed schema before findDynamicSchemaKeywordViolations runs, and filterRuntimeCompatibleTools uses that same projection. ## Solution Assessment

No, not yet: provider projection is the right layer, but raw runtime diagnostics should run before provider normalization. The safer fix is to preserve raw inspection and return the normalized schema only for provider/model output.
