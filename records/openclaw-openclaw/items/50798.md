## Summary

Keep open. Current main still lacks a supported visible coordinator-to-ACP-bound Discord thread delivery path that bypasses main channel-session metadata; the remaining work is an API/product/security decision, not cleanup.

## What This Changes

Request a visible coordinator-to-ACP-thread send path, such as `sessions_send visible`, `message proxyOnly`, or an ACP-bound thread exclusion rule, without creating main Discord channel-session stubs.

## Best Possible Solution

Add a narrow, explicit, authorized visible proxy-delivery contract for ACP-bound threads that bypasses main-session metadata only for that call and preserves normal routing/mirroring elsewhere.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction of the gap: the public schemas lack `visible`/`proxyOnly`, `sessions_send` injects internally, and normal visible sends still record outbound session metadata.

## Solution Assessment

Unclear. The requested direction is plausible, but maintainers need to choose the public contract and authorization semantics; a narrow authorized proxy-delivery path is safer than a blanket ACP-bound thread exclusion.
