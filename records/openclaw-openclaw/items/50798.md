## Summary

Keep open. Current main still lacks a supported visible coordinator-to-ACP-bound Discord thread delivery path that bypasses main Discord channel-session metadata creation; this is a feature/API design item rather than conservative cleanup.

## What This Changes

The issue requests visible coordinator-to-ACP Discord thread messaging without creating `agent:main:discord:channel:<thread_id>` main-session stubs.

## Best Possible Solution

Add a narrow, explicit, authorized visible proxy-delivery contract for ACP-bound Discord threads that bypasses main-session metadata only for that call, preserves normal routing elsewhere, and documents the supported coordination workflow.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction of the feature gap: public schemas reject `visible`/`proxyOnly`, `sessions_send` injects internally, and normal visible sends still write outbound session metadata.

## Solution Assessment

Unclear. The requested direction is plausible, but maintainers need to choose the public contract and authorization semantics; a narrow authorized proxy-delivery path is safer than a blanket ACP-bound thread exclusion.
