## Summary

Keep open: current main and latest release still lack the requested visible coordinator-to-ACP Discord thread send path. The merged bound-thread ACP completion work is adjacent, but it does not add `sessions_send({ visible: true })`, `message({ proxyOnly: true })`, or any authorized bypass of main Discord channel-session metadata creation.

## What This Changes

The issue requests visible coordinator-to-ACP Discord thread messaging without creating or routing through a main Discord channel session.

## Best Possible Solution

Add a narrow, explicit visible proxy-delivery contract after maintainer API and security review, preserving normal session metadata elsewhere while bypassing main-session route creation only for the authorized ACP-bound thread send path.

## Reproduction Assessment

Yes at source level for the missing API: `sessions_send` has no visible flag and `message`/gateway send have no proxy-only bypass. No live high-confidence Discord scale reproduction was run for the reported stub accumulation and Discord overflow.

## Solution Assessment

Unclear on exact API shape. The outcome is useful, but maintainers need to choose between a `sessions_send` visible mode, a `message` proxy-only mode, or an ACP-bound exclusion rule because each changes authorization, transcript, and loop behavior.
