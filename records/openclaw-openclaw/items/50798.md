## Summary

Keep open. Current main still lacks a supported visible coordinator-to-ACP-thread send path that bypasses main Discord outbound session metadata; this is a public API and security-design decision, not conservative cleanup.

## What This Changes

The request asks for visible agent-to-agent delivery into ACP-bound Discord threads without creating derived main Discord channel-session stubs.

## Best Possible Solution

Add a narrow, explicitly authorized visible proxy-delivery contract for ACP-bound threads, with metadata bypass limited to that call and normal outbound session recording preserved elsewhere.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction of the missing contract: sessions_send cannot accept visible:true and injects internally, while message/gateway send has no proxyOnly field and normal outbound sends still record session metadata.

## Solution Assessment

Unclear. Current main is not sufficient, and the requested options require maintainer API and security selection; an explicit authorized proxy-delivery contract is safer than a blanket ACP-bound-thread exclusion rule.
