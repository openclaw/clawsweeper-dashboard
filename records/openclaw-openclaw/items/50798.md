## Summary

Keep open: current main fixes adjacent sessions_send announce/thread delivery, but it still lacks the requested supported visible coordinator-to-ACP-bound Discord thread delivery path that bypasses main Discord outbound session metadata creation.

## What This Changes

Request a supported way for a coordinator agent to send visible instructions into an ACP-bound Discord thread without creating a main Discord channel session for that thread.

## Best Possible Solution

Choose and implement a narrow explicit visible proxy-delivery contract with authorization checks, scoped outbound metadata bypass, normal mirroring preserved elsewhere, focused tests, and live Discord ACP proof.

## Reproduction Assessment

Yes for the source-level gap: current main lacks visible/proxyOnly fields, sessions_send injects internally, and normal visible message sends still persist outbound session metadata. I did not run a live dual-account Discord ACP reproduction in this read-only review.

## Solution Assessment

Unclear until maintainers choose the API and security contract; a narrow explicit proxy-delivery API is safer than a blanket ACP-bound-thread exclusion because normal routing metadata has legitimate uses.
