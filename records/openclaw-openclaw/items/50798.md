## Summary

Keep open: current main covers adjacent ACP reply/completion routing, but it still does not provide a visible coordinator-to-worker instruction path or a proxy-only Discord send that avoids main-session creation.

## What This Changes

The issue requests visible coordinator-to-ACP-worker messaging in Discord thread-bound sessions without creating a separate main Discord thread session.

## Best Possible Solution

Maintainers should choose one narrow visible-delivery contract for ACP-bound conversations, then require tests proving the send is visible, does not create an extra main Discord thread session, and preserves bot-loop/security boundaries.

## Reproduction Assessment

No live reproduction was run, but source inspection shows the gap: sessions_send is internal/non-delivered input and explicit outbound sends still create or mirror an outbound session route. The reported scale and Discord 503 consequences would need live Discord ACP proof.

## Solution Assessment

No single proposed option is clearly best yet. Option A changes sessions_send, option B changes Gateway/message send semantics, and option C changes Discord/ACP routing policy, so maintainers should choose the API and trust boundary first.
