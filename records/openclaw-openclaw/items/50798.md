## Summary

Keep open. Current main and latest release still lack a supported visible coordinator-to-ACP Discord thread delivery path that bypasses main Discord channel-session metadata; the related fixed issue at https://github.com/openclaw/openclaw/issues/46645 only covered worker reply announce mirroring, not the initiating visible proxy instruction contract.

## What This Changes

The issue requests visible coordinator-to-ACP Discord thread messages without creating main-agent Discord channel-session metadata.

## Best Possible Solution

Add a narrow, explicit, authorized visible proxy-delivery contract for ACP-bound Discord threads that bypasses main-session metadata only for that call and preserves normal outbound session recording elsewhere.

## Reproduction Assessment

Yes for the feature gap by source inspection: current schemas reject `visible`/`proxyOnly`-style fields, `sessions_send` injects internally with `deliver:false`, and normal outbound message sends still record session metadata. I did not run a live dual-Discord-account reproduction.

## Solution Assessment

Unclear. The proposed options are plausible, but maintainers should choose the API and authorization semantics; a narrow authorized proxy-delivery path is safer than a blanket ACP-bound-thread exclusion.
