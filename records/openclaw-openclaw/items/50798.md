## Summary

Keep open. Current main still lacks a supported visible coordinator-to-ACP-thread send path that bypasses main-agent Discord outbound session metadata creation; the shipped A2A work covers internal injection plus later visible announce/follow-up delivery, not the initiating visible proxy message requested here.

## What This Changes

The issue requests a supported way for a coordinator agent to post visible instructions into an ACP-bound Discord thread without creating a main-agent Discord channel session entry.

## Best Possible Solution

Add a narrow, explicit visible proxy-delivery contract after maintainer API and security review. It should post the coordinator-authored instruction through an authorized Discord account into the ACP-bound thread, bypass only the derived main outbound session entry for that explicit call, preserve normal metadata writes elsewhere, and document the supported ACP coordination workflow.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: sessions_send cannot accept visible:true and internally calls the target with deliver:false, while normal Discord send/message routing derives an outbound session route and records session metadata before delivery.

## Solution Assessment

Unclear. Current main is not sufficient, and the reporter's three options need maintainer selection; a narrow explicit proxy-delivery contract is safer than a broad ACP-bound-thread exclusion rule that could affect normal routing metadata.
