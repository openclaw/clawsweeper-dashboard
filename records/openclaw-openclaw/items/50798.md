## Best Possible Solution

Add a narrow explicit visible proxy-delivery contract after maintainer API/security review, preserving normal session mirroring everywhere else and proving authorized ACP-bound Discord delivery without main-channel session metadata for that path.

## Solution Assessment

Unclear until maintainers choose the public contract. A narrow explicit proxy-delivery API is safer than a blanket ACP-bound-thread exclusion because normal thread transcript mirroring and reply attribution should remain intact.
