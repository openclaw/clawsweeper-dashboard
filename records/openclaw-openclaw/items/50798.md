## Summary

Keep open. Current main still does not provide a supported visible coordinator-to-ACP-thread send path that avoids main Discord outbound session metadata creation; the shipped A2A path covers internal injection plus later announce/follow-up delivery, not the visible initiating proxy message requested here.

## What This Changes

The issue requests a visible agent-to-agent instruction path into an ACP thread-bound Discord session without creating a route-polluting main Discord channel session.

## Best Possible Solution

Add a narrow, explicit visible proxy-delivery contract after maintainer API and security review. It should post the coordinator-authored instruction through an authorized Discord account into the ACP-bound thread, bypass only the derived main outbound session entry for that explicit call, preserve normal metadata writes elsewhere, and document the supported ACP coordination workflow.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: sessions_send cannot accept visible:true and internally calls the target with deliver:false, while normal Discord/gateway send paths still derive outbound session metadata and have no proxy-only bypass.

## Solution Assessment

Unclear. The current implementation is not sufficient, and the reporter's options need maintainer selection; an explicit proxy-delivery contract looks safer than a broad ACP-bound-thread exclusion rule.
