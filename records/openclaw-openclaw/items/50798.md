## Summary

Keep open. Current main still lacks a supported visible coordinator-to-ACP-bound Discord thread delivery contract that bypasses main Discord channel-session metadata creation; related fixes only cover reply/announce routing after an internal send, and the remaining API/security boundary needs maintainer direction.

## What This Changes

The issue requests visible coordinator-to-ACP-thread messaging without creating `agent:main:discord:channel:<thread_id>` session state.

## Best Possible Solution

Add a narrow explicit visible proxy-delivery contract after maintainer API and security review, bypassing main-session metadata only for that authorized ACP-bound delivery path while preserving normal session mirroring elsewhere.

## Reproduction Assessment

Yes for the source-level contract gap: current main lacks `visible`/`proxyOnly`, `sessions_send` injects internally, and visible message sends still persist outbound session metadata. No live dual-account Discord reproduction was run.

## Solution Assessment

Unclear until maintainers choose the public contract. A narrow explicit proxy-delivery API is safer than a blanket ACP-bound-thread exclusion because normal thread transcript mirroring and reply attribution should remain intact.
