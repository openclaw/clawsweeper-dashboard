Keep open. ## What This Changes

Request a visible coordinator-to-ACP Discord thread delivery contract that avoids creating main Discord channel-session metadata.

## Best Possible Solution

Add a narrow, explicit, authorized visible proxy-delivery contract for ACP-bound Discord threads that bypasses main-session metadata only for that call, preserves normal outbound session recording elsewhere, and documents the supported coordination workflow.

Unclear. The proposed options are plausible, but maintainers should choose the API and authorization semantics; a narrow authorized proxy-delivery path is safer than a blanket ACP-bound-thread exclusion.
