## Summary

Current main still lacks the requested supported visible coordinator-to-ACP Discord-thread delivery path that avoids creating main-agent Discord channel-session metadata. The related worker-reply mirroring path has been fixed, but the public `sessions.send` schema, agent tool schema, gateway handler, and outbound mirroring path still do not expose `visible`/`proxyOnly` semantics or a bypass for outbound session metadata creation.

## What This Changes

Request a visible coordinator-to-ACP Discord thread delivery path that does not create main-agent Discord channel-session metadata.

## Best Possible Solution

Add a narrow explicit visible proxy-delivery contract after maintainer API/security review, preserving normal session mirroring for ordinary outbound sends.

## Reproduction Assessment

Yes from source for the contract gap: current main lacks `visible` or `proxyOnly` parameters and still records outbound visible send session metadata, but no live Discord reproduction was run.

## Solution Assessment

Unclear until maintainers choose the API and security semantics; a narrow explicit visible proxy-send contract is safer than a blanket ACP-bound thread exclusion.
