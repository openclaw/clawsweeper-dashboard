## Summary

Keep open. Current main still lacks a supported visible coordinator-to-ACP-bound Discord thread delivery path that bypasses main Discord channel-session metadata; the related announce-thread fix only covers visible follow-up/reply delivery after an internal `sessions_send`.

## What This Changes

Request a supported way for a coordinator agent to post visible instructions into an ACP-bound Discord thread without creating a main Discord channel session.

## Best Possible Solution

Add a narrow, explicit, authorized visible proxy-delivery contract for ACP-bound Discord threads that bypasses main-session metadata only for that call, preserves normal routing elsewhere, and documents the workflow.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction of the feature gap: current schemas reject `visible`/`proxyOnly`, `sessions_send` injects internally with `deliver: false`, and normal visible sends still write outbound session metadata.

## Solution Assessment

Unclear. The requested directions are plausible, but maintainers should choose the public API and authorization semantics; a narrow authorized proxy-delivery path is safer than a blanket ACP-bound thread exclusion.
