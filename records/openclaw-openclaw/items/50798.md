## Summary

Keep open. Current main still lacks a supported visible coordinator-to-ACP-thread send path that bypasses derived main Discord outbound session metadata; the implemented A2A path only delivers a later announce/follow-up after an internal send.

## What This Changes

Request a visible ACP thread-bound messaging path for coordinator agents that does not create `agent:main:discord:channel:<thread_id>` session entries.

## Best Possible Solution

Add a narrow, explicit visible proxy-delivery contract after maintainer API and security review, scoped to authorized ACP-bound thread delivery and preserving normal outbound metadata writes elsewhere.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction of the gap: `sessions_send` cannot accept `visible:true` and injects internally, while normal Discord `send`/message delivery still records outbound session metadata.

## Solution Assessment

Unclear. Current main is not sufficient, and the reporter's options need maintainer selection; a narrow explicit proxy-delivery contract looks safer than a blanket ACP-bound-thread exclusion rule.
