## Summary

Keep open. Current main still lacks a supported visible coordinator-to-ACP-bound Discord thread delivery path that bypasses main Discord channel-session metadata; the related A2A announce work only surfaces follow-up/reply output after an internal send.

## What This Changes

The issue requests a supported way for a coordinator agent to post visible messages into an ACP-bound Discord thread without creating a main Discord channel session.

## Best Possible Solution

Add a narrow, explicit, authorized visible proxy-delivery contract for ACP-bound Discord threads that bypasses main-session metadata only for that call, preserves normal routing elsewhere, and documents the supported coordination workflow.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction of the feature gap: current schemas reject visible/proxyOnly, sessions_send injects internally with deliver: false, and normal visible sends still write outbound session metadata.

## Solution Assessment

Unclear. The requested direction is plausible, but maintainers need to choose the public contract and authorization semantics; a narrow authorized proxy-delivery path is safer than a blanket ACP-bound thread exclusion.
