## Summary

Keep open. Current main still does not provide a supported visible coordinator-to-ACP-thread send path that bypasses main Discord channel-session metadata, and the remaining choice is a public API/security decision rather than cleanup.

## What This Changes

Request a supported way for coordinator agents to post visible instructions into ACP-bound Discord threads without creating main-agent Discord channel-session entries.

## Best Possible Solution

Add a narrow, explicit visible proxy-delivery contract for authorized ACP-bound thread delivery, while preserving normal outbound session metadata writes everywhere else.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction of the gap: sessions_send has no visible option and injects internally, while normal send/message delivery still records outbound session metadata.

## Solution Assessment

Unclear. The current implementation does not solve the request, and the reporter's options need maintainer selection; a narrow authorized proxy-delivery contract looks safer than a blanket ACP-bound thread exclusion.
