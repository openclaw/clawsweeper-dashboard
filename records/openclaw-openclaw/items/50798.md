## Summary

Keep open: current main has partial ACP bound-thread routing and a merged Discord terminal-completion fix, but I did not find a supported visible coordinator-to-ACP thread send path such as `sessions_send({ visible: true })` or `message({ proxyOnly: true })` that avoids creating or using a main Discord channel-session route.

## What This Changes

The issue requests a supported visible agent-to-agent delivery path into ACP thread-bound Discord sessions without creating `agent:main:discord:channel:<thread_id>` session metadata.

## Best Possible Solution

Maintainers should choose one supported core/API shape for visible ACP-bound thread injection, reuse the existing session-binding owner path, and prove that it avoids main-session route pollution while preserving authorization, transcript, and loop-safety guarantees.
