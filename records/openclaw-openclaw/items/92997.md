Ensure follow-up messages in Discord-bound persistent ACP sessions deliver generated replies back to the same Discord thread.

## Best Possible Solution

## Solution Assessment

Mostly yes: the exact fix should narrow the parent-owned ACP suppression gate when the turn originates from the child session's own bound Discord context. The existing open fix candidate is the best current path, but it needs maintainer refresh/review before this issue can close.
