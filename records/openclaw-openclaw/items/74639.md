## Summary

Keep open. Current main still has the source shape described in the report: phase reports/events can be written independently of Dream Diary appends, cron narratives are detached and best-effort, and backfill still uses date-only headings with legacy marker normalization that can prepend a new managed diary section.

## What This Changes

The issue reports memory-core dreaming phase reports continuing while `DREAMS.md` stops receiving new diary entries, plus inconsistent `rem-backfill` and rollback diary structure.

## Best Possible Solution

Memory-core should make cron Dream Diary append failures observable or retryable, keep marker repair and rollback idempotent for legacy `DREAMS.md`, and standardize the backfill heading format with regression coverage.

## Reproduction Assessment

Partly yes. Source inspection and existing tests show the backfill date-only heading and legacy marker-prefix behavior; the multi-day cron stale-diary path still needs live cron/subagent proof or a focused integration test.

## Solution Assessment

No. Current main does not solve the whole report; the maintainable path is to coordinate diary recovery hardening with #70332 and cron/report reliability with #72546 or a focused replacement.
