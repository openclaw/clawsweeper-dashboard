## Summary

Keep open. Current main still permits successful dreaming phase report/event writes to diverge from best-effort detached Dream Diary appends, and the reported `rem-backfill` date-only heading plus rollback/marker-normalization symptoms still match the current memory-core implementation; the existing ClawSweeper discussion correctly links the overlapping open #72546 and #70332 context.

## What This Changes

The issue asks memory-core dreaming to keep `DREAMS.md` advancing when phase artifacts are produced and to make `memory rem-backfill`/rollback preserve consistent Dream Diary structure and heading format.

## Best Possible Solution

Keep this open until memory-core makes Dream Diary recovery idempotent for legacy or partially marked `DREAMS.md` files, aligns backfill heading format with the intended public diary format, and makes cron narrative failures visible or retryable enough that successful phase reports do not imply a healthy diary append.
