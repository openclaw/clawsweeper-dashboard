## Summary

Keep open. Current main still allows phase report/event writes to succeed independently of best-effort detached Dream Diary appends, and the reported backfill date-only heading plus rollback marker-normalization behavior still matches current source and tests.

## What This Changes

The issue reports that memory-core dreaming phase artifacts can continue while `DREAMS.md` stops advancing, and that `rem-backfill`/rollback can leave Dream Diary entries structurally inconsistent.

## Best Possible Solution

Keep this open until memory-core makes Dream Diary recovery idempotent for legacy or partially marked `DREAMS.md` files, aligns backfill headings with the intended public diary format, and makes detached cron narrative failures visible or retryable enough that phase report success cannot silently imply a healthy diary append.

## Reproduction Assessment

Partly yes. The backfill date-only heading and rollback marker-normalization paths are statically reproducible from current source and tests; the exact multi-day cron stale-diary symptom still needs live cron/subagent logs or a focused integration test.

## Solution Assessment

No patch is attached, and current main is not sufficient. The narrow maintainable direction is a scoped memory-core fix coordinated with #70332 and #72546 rather than closing this as duplicate or implemented.
