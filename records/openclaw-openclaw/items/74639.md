## Summary

Keep open. Current main still has the reported split behavior: phase report/event writes are synchronous, while cron Dream Diary prose is detached best-effort work, and the backfill/rollback symptoms still match current marker normalization plus date-only backfill formatting.

## What This Changes

The issue asks memory-core dreaming to keep DREAMS.md advancing when phase artifacts advance, and to keep rem-backfill/rollback output structurally consistent with normal Dream Diary entries.

## Best Possible Solution

Keep this open and harden the memory-core Dream Diary path so legacy or partially marked DREAMS.md files are migrated idempotently, backfill headings match the intended public diary format, and cron narrative failures are visible or retryable without implying that phase report success means diary success.

## Reproduction Assessment

Partly yes. The rollback/header and date-only backfill symptoms have a high-confidence static reproduction from current source and tests; the multi-day cron stale-diary symptom is plausible from current detached narrative behavior but still needs live cron/subagent logs for the exact failure mode.

## Solution Assessment

No proposed patch is attached, and current main is not sufficient. The safest solution should be scoped alongside open PR #70332 and related issue #72546 so the fix does not reintroduce cron stalls while improving diary recovery and observability.
