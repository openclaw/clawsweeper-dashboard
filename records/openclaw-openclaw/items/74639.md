## Summary

Keep open. Current main still permits dreaming phase reports/events to be written while cron Dream Diary narratives run detached and best-effort, and the backfill path still emits date-only diary headings while marker normalization can prepend a new managed diary block around legacy content; related #70332 and #72546 remain open.

## What This Changes

The issue asks memory-core dreaming to keep `DREAMS.md` advancing with phase artifacts and make `rem-backfill`/rollback preserve a well-formed, timestamp-consistent Dream Diary.

## Best Possible Solution

Harden memory-core so cron phase success makes diary append failures observable or retryable, backfill headings are intentionally consistent with the chosen diary format, and rollback is idempotent for legacy or partially marked `DREAMS.md` files.

## Reproduction Assessment

Partly yes. The backfill date-only heading and legacy marker-prefix behavior are reproducible from current source and tests, while the multi-day cron stale-diary path still needs live cron/subagent evidence or a focused integration test.

## Solution Assessment

No. Current main does not solve the whole report; the narrow maintainable path is to coordinate the diary recovery work with #70332 and the cron/artifact reliability work with #72546, or split one focused replacement after owner review.
