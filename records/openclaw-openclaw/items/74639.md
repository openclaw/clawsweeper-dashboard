## Summary

Keep open. Current main still allows phase reports and completion events to be written independently from best-effort detached Dream Diary appends, and the reported backfill/rollback symptoms match current date formatting and diary marker normalization paths.

## What This Changes

The issue asks for memory-core dreaming to keep `DREAMS.md` advancing when phase artifacts are produced and to make `memory rem-backfill`/rollback preserve a consistent Dream Diary structure and heading format.

## Best Possible Solution

Keep this open until memory-core makes Dream Diary recovery idempotent for legacy or partially marked `DREAMS.md` files, aligns backfill heading format with the intended public diary format, and makes cron narrative failures visible or retryable enough that successful phase reports do not imply a healthy diary append.

## Reproduction Assessment

Partly yes. The backfill date-only heading and rollback/header normalization are statically reproducible from current code and tests; the exact multi-day cron stale-diary symptom needs live cron/subagent logs or a targeted integration reproduction.

## Solution Assessment

No patch is attached, and current main is not sufficient. The safest direction is a scoped memory-core fix coordinated with #70332 and #72546 rather than closing this as a duplicate or implemented item.
