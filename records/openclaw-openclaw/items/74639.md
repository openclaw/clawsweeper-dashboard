## Summary

Keep open. Current main still has independent dreaming phase report/event writes and best-effort detached Dream Diary appends for cron, while the recovery path still formats backfill headings as date-only; the related diary and cron hardening work is still open.

## What This Changes

The issue asks memory-core to keep `DREAMS.md` advancing when dreaming phase reports are written and to make `memory rem-backfill`/rollback preserve a well-formed Dream Diary.

## Best Possible Solution

Make Dream Diary append success observable or retryable when phase reports succeed, and make `rem-backfill`/rollback idempotently normalize legacy managed diary sections with an intentional heading policy.

## Reproduction Assessment

Partly yes. Current source and tests reproduce the backfill date-only heading, and source inspection shows cron reports/events can complete while detached diary appends are best-effort; the multi-day stale diary still needs live cron or integration proof.

## Solution Assessment

Not applicable; there is no proposed patch. The maintainable path should align diary recovery work with #70332 and cron/artifact reliability work with #72546 before changing diary heading or rollback semantics.
