## Summary

Keep open. Current main still allows phase report/event writes to succeed independently of detached Dream Diary appends, and the reported `rem-backfill` date-only heading plus rollback marker-normalization symptoms still match the memory-core implementation.

## What This Changes

The issue asks memory-core dreaming to keep `DREAMS.md` advancing when phase artifacts are produced and to make `memory rem-backfill` and rollback preserve consistent Dream Diary structure and heading format.

## Best Possible Solution

Keep this open until memory-core makes Dream Diary recovery idempotent for legacy or partially marked `DREAMS.md` files, aligns backfill heading format with the intended diary format, and makes cron narrative failures visible or retryable enough that successful phase reports do not imply a healthy diary append.

## Reproduction Assessment

Partly yes. The backfill date-only heading and rollback/header normalization paths are statically reproducible from current code and tests; the exact multi-day cron stale-diary symptom needs live cron/subagent logs or a targeted integration reproduction.
