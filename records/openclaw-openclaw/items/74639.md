## Summary

Keep open. Current main still has source-level paths where dreaming reports/events can be written while Dream Diary narrative append remains detached and best-effort, and the public backfill path still intentionally writes date-only diary headings and can prefix a managed diary block around legacy content.

## What This Changes

The issue reports that memory-core dreaming can keep writing phase artifacts while `DREAMS.md` stops advancing, and that `memory rem-backfill`/rollback can leave inconsistent Dream Diary formatting.

## Best Possible Solution

Harden memory-core so cron-visible phase success cannot silently leave `DREAMS.md` stale, and make backfill/rollback idempotent for legacy or partially marked diary files with an explicitly agreed heading format.

## Reproduction Assessment

Partly yes. The backfill date-only heading and legacy marker-prefix behavior are clear from current source and tests; the multi-day cron stale-diary path still needs live cron/subagent evidence or a focused integration test.

## Solution Assessment

No. Current main does not fully solve the report; the maintainable path is to align diary recovery with #70332 and cron/artifact reliability with #72546, then land one coordinated memory-core fix.
