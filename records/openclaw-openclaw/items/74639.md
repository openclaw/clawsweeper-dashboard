## Summary

Keep open. Current main still lets cron phase/report writes succeed independently of detached, best-effort Dream Diary appends, and the backfill path still writes date-only diary headings while related diary-hardening and cron-artifact reports remain open.

## What This Changes

The issue requests fixing memory-core Dream Diary drift where cron phase reports continue while `DREAMS.md` stops updating, plus backfill/rollback diary-format inconsistencies.

## Best Possible Solution

Harden memory-core so cron phase success makes diary append failures observable or retryable, backfill headings match normal diary structure, and rollback is idempotent for legacy or partially marked `DREAMS.md` files.

## Reproduction Assessment

Partly yes. The backfill date-only heading is statically reproducible from current source and tests; the multi-day stale-diary cron path is plausible from current detached best-effort flow but still needs live cron/subagent proof or an integration test.

## Solution Assessment

No current main implementation solves the whole issue. The narrow maintainable path is to coordinate diary recovery with #70332 and cron/artifact reliability with #72546, or split a focused replacement if those do not cover this report.
