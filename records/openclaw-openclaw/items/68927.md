## Summary

Keep open. Current main intentionally shows friendly catalog alias/name labels in the Control UI chat model picker, so this PR is a live product-policy change rather than obsolete. The branch is not ready as-is because it changes the shared model-option label contract without updating the direct helper tests, and it still needs maintainer UX confirmation before replacing the recently shipped friendly-label behavior.

## Best Possible Solution
