## Summary

Keep open. The PR is useful source work, but it is not ready to merge because current main still has a lookup-backed friendly-label contract and the proposed helper change would leave existing helper tests contradictory while also making the exact-ref UX a maintainer product decision.

## What This Changes

The PR changes the Control UI chat model picker to display exact provider/model refs for option and default labels instead of lookup-backed friendly catalog labels, and updates select-state tests for that behavior.

## Best Possible Solution

Choose the Control UI label policy explicitly. If exact visible refs are desired, make the helper API internally consistent by removing lookup plumbing, updating both helper and select-state tests, and adding the required user-facing changelog entry; if friendly catalog labels remain intended, close this PR by product decision and keep the current implementation.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to apply the PR diff and run the focused UI tests: `ui/src/ui/chat-model-ref.test.ts` still expects friendly labels from `buildChatModelOption`, while the PR changes the helper to return exact refs as labels.

## Solution Assessment

No as-is. The exact-ref direction is narrow enough technically, but the proposed patch changes a shared helper contract without updating its tests, leaves dead lookup plumbing, and needs maintainer confirmation before replacing the current friendly-label UX.
