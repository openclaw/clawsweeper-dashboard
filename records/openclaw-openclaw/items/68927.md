## Summary

Keep open. Current main intentionally uses catalog alias/name labels for the Control UI chat model picker, while this PR proposes changing the visible label policy back to exact provider/model refs; that is a product/UX choice rather than conservative cleanup, and the branch still has a concrete helper-test mismatch before it could merge.

## What This Changes

The PR changes Control UI chat model option and default labels from catalog-friendly alias/name displays to exact qualified provider/model refs and updates select-state tests for that behavior.

## Best Possible Solution

Choose the Control UI model-label policy explicitly. If exact refs are desired, make the helper API internally consistent, update both helper and select-state tests, and add the user-facing changelog entry; if friendly catalog labels remain intended, close this PR by product decision and keep the current #61382 behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to apply the PR diff and run `pnpm test ui/src/ui/chat-model-ref.test.ts ui/src/ui/chat-model-select-state.test.ts`; `chat-model-ref.test.ts` still expects friendly labels from `buildChatModelOption` while the PR changes that helper to return exact refs as labels.

## Solution Assessment

No as-is. The exact-ref direction is technically narrow, but the patch changes a shared helper contract without updating its direct tests, leaves dead lookup plumbing, and needs maintainer confirmation before replacing the current friendly-label UX.
