## Summary

Keep open: this PR is a real, focused Control UI proposal, but it intentionally reverses the current friendly alias/name picker policy from #61382 and still needs maintainer UX confirmation plus mechanical cleanup before merge.

## What This Changes

The PR changes the Control UI chat model picker helpers and select-state tests so visible model option/default labels use exact provider/model refs instead of catalog alias/name labels.

## Best Possible Solution

Choose the Control UI label policy explicitly. If exact refs are desired, make the helper API and tests consistently exact-ref based, remove the unused display-lookup plumbing, and add a changelog entry; if friendly catalog labels remain intended, close this PR by product decision and keep the current implementation.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to apply the PR diff and run `pnpm test ui/src/ui/chat-model-ref.test.ts`; the PR changes `buildChatModelOptionFromLookup` to return exact refs while the unchanged helper tests still expect friendly alias/name labels.

## Solution Assessment

No as-is. The implementation is technically narrow, but it changes a shared label contract without updating all tests, leaves now-unused lookup plumbing, and needs maintainer confirmation before replacing the current friendly-label UX.
