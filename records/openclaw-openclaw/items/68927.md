## Summary

Keep open. Current main intentionally uses catalog aliases/names for Control UI chat model labels, and this PR is an active product-policy change that still needs maintainer UX confirmation plus a blocking test update before merge.

## What This Changes

The PR changes the Control UI chat model picker to display exact provider/model refs for option and default labels, with select-state tests updated for that label behavior.

## Best Possible Solution

Choose the Control UI label policy explicitly. If exact visible refs are the desired UX, update all affected helper and select-state tests together and remove the unused display-lookup plumbing; if friendly catalog labels remain intended, close this PR by product decision and keep the current alias/name behavior.

## Reproduction Assessment

Yes for the behavioral and test-level delta. Current main helper tests assert friendly catalog labels, while the PR changes the shared helper to return exact refs, so `pnpm test ui/src/ui/chat-model-ref.test.ts ui/src/ui/chat-model-select-state.test.ts` is the focused reproduction path.

## Solution Assessment

No as-is. The implementation is narrow, but it changes the shared helper contract without updating all affected tests and leaves unused lookup plumbing, so the safer path is to finish the test/API cleanup after maintainers confirm the exact-ref UX direction.
