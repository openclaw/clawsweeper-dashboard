## Summary

Keep open: the PR addresses the linked installer ordering bug in the Node path, but it narrows Homebrew setup to failed Node checks even though macOS Git installation still depends on `brew`, creating a regression for supported-Node/no-Homebrew/missing-Git installs. The supplied behavior proof is also stub-only rather than a real macOS installer run.

## What This Changes

The PR moves `install_homebrew` into the `if ! check_node` branch in `scripts/install.sh` and adds installer call-order regression tests.

## Best Possible Solution

Make Homebrew setup conditional on actual macOS brew consumers: skip it only when Node and Git are already usable, but ensure it runs before `install_node` or `install_git` needs `brew`, with focused installer tests and any necessary docs alignment.

## Reproduction Assessment

Yes, at source level: current main calls `install_homebrew` before Node detection, and the PR diff moves it behind `check_node`. The newly introduced Git-path regression is also source-visible because `install_git` still calls `brew install git` on macOS.

## Solution Assessment

No. The PR is the right general direction for existing-Node installs, but the maintainable fix must preserve Homebrew setup for every macOS path that invokes `brew`, especially the Git installer path.
