## Summary

This PR widens the MCP server `transport` schema and exported config type to include `"stdio"` and adds schema coverage for command-bearing stdio plus unsupported transport values.

## Best Possible Solution

Land a schema/type/test fix that accepts explicit stdio only for command-bearing MCP servers while keeping URL transports limited to `sse` and `streamable-http`.

## Solution Assessment

No as-is. Schema/type alignment is the right layer, but the patch needs a command-bound validation refinement so it does not introduce a new validator/runtime mismatch for URL-only stdio.
