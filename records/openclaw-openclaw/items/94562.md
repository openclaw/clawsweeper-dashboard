## Summary

Keep open: the PR is a focused fix for a valid Workboard CLI/API archive-visibility bug, but merging it changes default CLI list output, so maintainers should explicitly accept that compatibility risk before landing.

## What This Changes

The PR filters archived Workboard cards from `openclaw workboard list` by default, adds `--include-archived`, and updates CLI docs plus regression coverage.

## Best Possible Solution

Land the CLI-layer fix if maintainers accept the default-output compatibility change, with merge or release-note context telling existing scripts to use `--include-archived` when archived cards are required.

## Reproduction Assessment

Yes. Source inspection shows current main and latest `v2026.6.8` list archived cards in the terminal CLI by default, while the tool and slash-command list paths hide them; I did not run a live CLI repro in this read-only review.

## Solution Assessment

Yes, with a maintainer compatibility call. The CLI presentation layer is the right boundary because `WorkboardStore.list` remains unfiltered shared data access, and `--include-archived` provides an explicit old-output path.
