## Best Possible Solution

Land the CLI-layer fix if maintainers accept the default-output compatibility change, with merge or release-note context telling existing scripts to use `--include-archived` when archived cards are required.

Yes, with one maintainer decision. The CLI presentation layer is the right fix boundary because `WorkboardStore.list` remains the shared unfiltered data access layer, but maintainers must accept the shipped default-output compatibility change.
