## Summary

## What This Changes

The PR filters archived Workboard cards from `openclaw workboard list` by default, adds `--include-archived`, updates CLI docs, and adds CLI regression coverage.

## Best Possible Solution

Land the CLI-layer parity fix if maintainers accept the default-output compatibility change, with merge or release-note context telling scripts to use `--include-archived`; otherwise preserve the old default output and add an explicit hide-archived mode.

## Reproduction Assessment

Yes, source inspection is enough: current main and latest `v2026.6.8` list archived cards in the terminal CLI by default while the tool, slash command, and UI hide them unless explicitly requested. I did not run a live CLI reproduction in this read-only review.

## Solution Assessment

Yes, with a maintainer compatibility call. The CLI presentation layer is the right boundary because `WorkboardStore.list` remains shared unfiltered data access, and `--include-archived` provides an explicit old-output path.
