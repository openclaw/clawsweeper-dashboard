## Summary

The requested zero-LLM cron execution is now implemented and shipped: command jobs run directly on the Gateway host while retaining cron history, bounded output, failure accounting, and normal delivery modes.

## What This Changes

Add first-class command cron jobs that execute without an isolated model turn while preserving cron observability and channel delivery.

## Best Possible Solution

Keep the shipped command payload as the canonical zero-LLM cron path and track narrower enhancements, such as richer Control UI authoring, separately if needed.
