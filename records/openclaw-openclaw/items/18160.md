## What This Changes

The issue requests a zero-LLM cron payload that runs scheduled commands directly, captures stdout/stderr, and optionally delivers output through OpenClaw channels.

## Best Possible Solution

Keep the shipped command cron implementation as the core solution, and track narrower remaining requests such as static-message cron or direct tool invocation in their own issues.

## Solution Assessment

Yes. The shipped command payload is the maintainable solution: protocol and CLI store exact argv, runtime executes without an agent/model turn, and cron history/delivery reuse the existing scheduler pipeline; the exact payload name differs from the requested exec spelling but solves the user problem.
