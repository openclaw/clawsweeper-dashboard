## Summary

Current main and the latest stable release now provide first-class command cron jobs, which resolves the requested zero-LLM scheduled script execution and delivery path.

## What This Changes

Request to add a direct cron execution payload so scheduled scripts can run without an agent/model turn and deliver captured output through OpenClaw.

## Best Possible Solution

Keep the shipped command payload implementation as the canonical zero-LLM cron path and track distinct follow-ups, such as richer Control UI authoring, separately.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a feature request; current source, docs, and release history show the requested capability now exists, while the issue comments document the former agent-only limitation.

## Solution Assessment

Is this the best way to solve the issue? Yes. A typed argv-based command payload with bounded process execution, cron history, failure accounting, delivery semantics, and an operator-admin boundary is a maintainable core solution for this request.
