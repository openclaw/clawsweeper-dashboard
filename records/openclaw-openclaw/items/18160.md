## Summary

Keep open: current main still has no zero-LLM cron exec payload, and the issue remains the canonical core feature/security-design tracker with recent production datapoints and a related implementation PR closed unmerged for owner/security review.

## What This Changes

Requests a cron payload and CLI/UI/API path that runs scheduled shell commands directly without an LLM turn while preserving OpenClaw cron delivery and observability.

## Best Possible Solution

Keep this as the canonical core tracker and design a hardened additive cron exec contract across protocol/schema, validation, runtime dispatch, run history, delivery, CLI, Control UI, docs, tests, and host-command security policy.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a new cron capability request. Source inspection of current main and the latest release shows the requested direct exec path is absent rather than a broken existing contract.

## Solution Assessment

Unclear until maintainers settle the API and security policy. A native cron exec payload is a plausible seam, but the maintainable fix needs coordinated Gateway, CLI/UI, delivery, observability, and secops follow-through rather than a narrow child-process branch alone.
