## Summary

The central user problem is resolved: current main and the latest release provide first-class command cron jobs that bypass agent/model execution while retaining cron history, failure accounting, output controls, and delivery semantics.

## What This Changes

Request direct, zero-LLM command execution for scheduled cron jobs with captured output and channel delivery.

## Best Possible Solution

Keep the shipped typed command-job implementation as the canonical zero-LLM cron path and use focused follow-up issues only for distinct remaining enhancements such as richer Control UI authoring.

## Reproduction Assessment

Not applicable as a feature request; source history and production data clearly establish the former agent-only limitation, while the merged PR provides deployed after-fix command-job evidence.

## Solution Assessment

Yes. A typed argv-based command payload with a dedicated bounded process runner, operator-admin authorization, scheduler accounting, migration safeguards, and existing delivery semantics is the narrowest maintainable solution.
