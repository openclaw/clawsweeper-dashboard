## Summary

Keep open: current main still lacks a zero-LLM cron exec payload, and the discussion has consolidated multiple production datapoints plus a closed unmerged implementation attempt under this issue.

## What This Changes

Add a direct cron exec payload or CLI mode that runs scheduled shell commands, captures stdout/stderr, and optionally delivers output without starting an agent/LLM turn.

## Best Possible Solution

Keep this as the canonical core tracker and land an additive, owner-reviewed cron exec payload covering protocol/schema, validation, runtime dispatch, run history, delivery, CLI, Control UI, docs, tests, and host-command policy.

## Reproduction Assessment

Not applicable for a feature request. Source inspection on current main gives high confidence that cron only supports systemEvent and agentTurn payloads, so the requested exec path is absent rather than a broken existing contract.

## Solution Assessment

Yes in direction, but not automation-ready. A native cron exec payload is the maintainable core seam for preserving OpenClaw observability and delivery, but the persisted host-command policy needs explicit owner/secops approval first.
