## Summary

Keep open: current main still lacks a zero-LLM cron exec/script payload, this issue is the canonical tracker for several duplicate reports, and the remaining work needs maintainer/security design rather than cleanup closure.

## What This Changes

The issue requests a direct cron execution mode, such as `payload.kind="exec"` or `--exec`, that runs scheduled scripts without an LLM turn and can deliver captured output through OpenClaw channels.

## Best Possible Solution

Keep this as the canonical tracker until maintainers choose a secure no-LLM cron execution contract and update protocol, validation, runtime, run history, delivery, CLI, Control UI, docs, and tests together.

## Reproduction Assessment

Not applicable for a feature request. Source inspection on current main gives high confidence that cron cannot express a direct exec/script payload and still accepts only `systemEvent` or `agentTurn`.

## Solution Assessment

Unclear until maintainers settle the core-versus-plugin API and security policy. A native cron exec payload is plausible, but the closed PRs show a simple child-process branch is not enough without protocol, UI/CLI, delivery, observability, and secops follow-through.
