## Summary

Keep open: current main and the latest release still only support `systemEvent` and `agentTurn` cron payloads, while the requested zero-LLM scheduled command mode remains a real core feature and security design item.

## What This Changes

Add a direct cron exec payload or CLI/API mode that runs scheduled shell commands without an LLM turn and delivers captured output through OpenClaw cron.

## Best Possible Solution

Keep this as the canonical core tracker and design a hardened additive cron exec contract across protocol/schema, validation, runtime dispatch, run history, delivery, CLI, Control UI, docs, tests, and host-command security policy.

## Reproduction Assessment

Not applicable for a feature request. Source inspection on current main and `v2026.5.12` gives high confidence that cron supports only `systemEvent` and `agentTurn`, so the requested direct exec path is absent rather than a broken existing contract.

## Solution Assessment

Unclear until maintainers settle the API and security policy. A native cron exec payload is a plausible seam, but the maintainable solution needs coordinated Gateway, UI/CLI, delivery, observability, and secops follow-through rather than a narrow child-process branch alone.

## Security Review

Status: needs_attention

Summary: The request is security-sensitive because it would add persisted scheduled host-command execution outside a live agent turn.

  - confidence: 0.9
