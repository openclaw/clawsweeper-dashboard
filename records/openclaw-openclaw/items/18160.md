## Summary

Keep open: current main still has no zero-LLM cron exec payload, and this remains the canonical core tracker after the related implementation PR was closed unmerged for security and validation concerns.

## What This Changes

Add a direct cron exec payload and creation path that runs scheduled shell commands without starting an agent turn while preserving OpenClaw run history and delivery.

## Best Possible Solution

Keep this as the canonical core tracker and land an additive, owner/secops-approved cron exec payload covering protocol/schema, validation, scheduler dispatch, run history, delivery, CLI, Control UI, docs, tests, and host-command policy.

## Reproduction Assessment

Not applicable for a feature request. Source inspection on current main gives high confidence that cron only supports systemEvent and agentTurn payloads, so the requested exec path is absent rather than a broken existing contract.

## Solution Assessment

Yes in direction, but not automation-ready. A native cron exec payload is the maintainable core seam for preserving OpenClaw observability and delivery, but the stored host-command execution policy needs explicit owner/secops design first.
