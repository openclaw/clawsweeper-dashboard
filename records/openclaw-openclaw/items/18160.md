## Summary

Keep this issue open as the canonical core tracker: current main still only supports `systemEvent` and `agentTurn` cron payloads, while the related direct-exec PR is closed unmerged and the recent discussion adds production data rather than resolving the request.

## What This Changes

Add a zero-LLM cron execution mode that runs a shell command on schedule, captures output, and uses OpenClaw cron history and delivery.

## Best Possible Solution

Land an additive, owner-reviewed `exec` cron payload design across types, protocol, validation, runtime dispatch, run history, retry/failure accounting, delivery, CLI/UI/docs, and tests, with secops-approved host-command policy.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-main check that the requested zero-LLM cron payload is still absent.

## Solution Assessment

Unclear until maintainers settle the security/product policy. An additive `payload.kind="exec"` is the likely seam, but the final design needs explicit host-command approval, sandboxing, output, and delivery semantics.
