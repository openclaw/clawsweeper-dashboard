## Summary

Keep open. Current main still lacks a zero-LLM cron exec payload, and the closed PR plus external shim workaround do not replace OpenClaw's core scheduler, run history, retry accounting, and delivery path.

## What This Changes

Add a direct cron execution payload, such as payload.kind="exec", so scheduled scripts can run without an agentTurn/LLM session and optionally deliver command output.

## Best Possible Solution

Keep this as the core cron tracker and land an owner-reviewed additive command payload that updates the typed contract, Gateway schema, validation, scheduler dispatch, run history, delivery, CLI, Control UI, docs, and tests with a documented host-command security model.

## Reproduction Assessment

Yes for the reported missing capability: on current main, payload.kind="exec" is outside the cron types and Gateway schema, isolated-like validation rejects non-agentTurn payloads, and detached execution only calls the isolated agent runner. I did not run a mutating live Gateway cron create path because this was a read-only review.

## Solution Assessment

Yes for the product direction, but not as an unsupervised quick patch. Core cron is the right seam because plugins or the linked shim cannot own OpenClaw's persisted scheduler, run history, retry accounting, and channel delivery contract.
