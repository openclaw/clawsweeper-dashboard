## Summary

Keep open. Current main still lacks a zero-LLM cron exec payload, and the closed implementation PR plus external shim workaround do not replace OpenClaw's core scheduler, run history, retry accounting, and delivery path.

## What This Changes

Request an additive cron payload mode that executes a host command on schedule, captures output, and can deliver it without starting an LLM agent turn.

## Best Possible Solution

Keep this as the core cron tracker and land an owner-reviewed additive command payload that updates the typed contract, Gateway schema, validation, scheduler dispatch, run history, delivery, CLI, Control UI, docs, and tests with a documented host-command security model.

## Reproduction Assessment

Yes. Submit or validate a cron job with `payload.kind="exec"` on current main: the Gateway schema rejects it, job validation requires `agentTurn` for isolated-like targets, and the detached runtime has no direct exec branch.

## Solution Assessment

Yes for the product direction, but not as an unsupervised quick patch. Core cron is the right seam because plugins or the linked shim cannot own OpenClaw's persisted scheduler, run history, retry accounting, and channel delivery contract.
