## Summary

Keep open: current main still has no zero-LLM cron exec payload, the related implementation PR is closed unmerged, and recent comments treat this issue as the canonical tracker for the remaining core/security design.

## What This Changes

Add a cron payload mode that runs scheduled shell commands directly, captures stdout/stderr, and delivers or records output without starting an LLM agent turn.

## Best Possible Solution

Keep this as the canonical core design tracker and land an additive owner/secops-approved cron exec payload with protocol, validation, runtime, delivery, CLI, UI, docs, tests, and host-command policy together.

## Reproduction Assessment

Not applicable for a feature request. Source inspection on current main gives high confidence that cron only supports systemEvent and agentTurn, so the requested direct exec path is absent rather than a broken existing contract.

## Solution Assessment

Unclear until maintainers settle the core API and security policy. A native cron exec payload is a plausible maintainable seam, but it should be owner/secops-designed rather than auto-implemented from the stale branch.

## Security Review

Status: needs_attention

Summary: This feature would add persisted scheduled host-command execution outside a live agent turn, so the trust boundary needs explicit owner/secops review.
