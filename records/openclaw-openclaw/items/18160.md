## Summary

Keep open: current main still lacks a zero-LLM cron exec payload, and the related implementation PR is closed unmerged while the discussion has consolidated multiple production datapoints and duplicates here.

## What This Changes

Add a cron payload/CLI mode that runs a scheduled shell command directly, captures stdout/stderr, and delivers output through OpenClaw without starting an LLM agent turn.

## Best Possible Solution

Keep this as the canonical core tracker and land an additive, owner-reviewed cron exec payload covering protocol/schema, validation, runtime dispatch, run history, delivery, CLI, Control UI, docs, tests, and host-command policy.

## Reproduction Assessment

Not applicable for a feature request. Source inspection gives high confidence that current main only supports systemEvent and agentTurn cron payloads, so the requested exec path is absent rather than a failing existing contract.

## Solution Assessment

Yes in direction, but not as an automated cleanup patch yet. A native cron exec payload is the maintainable core seam for preserving OpenClaw observability and delivery, but the host-command security policy needs explicit owner approval first.
