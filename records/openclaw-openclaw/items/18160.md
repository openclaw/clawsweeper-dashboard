## Summary

Keep open: current main still lacks a zero-LLM cron exec payload, and this issue remains the canonical core tracker with production datapoints plus a closed unmerged implementation attempt.

## What This Changes

Add a direct cron execution mode that runs scheduled shell commands without an LLM turn while preserving OpenClaw run tracking and delivery.

## Best Possible Solution

Keep this as the canonical core tracker and land an additive, owner-reviewed cron exec payload covering protocol/schema, validation, scheduler dispatch, run history, delivery, CLI, Control UI, docs, tests, and host-command policy.

## Reproduction Assessment

Not applicable for a feature request. Source inspection on current main gives high confidence that cron only supports systemEvent and agentTurn payloads, so the requested exec path is absent rather than a broken existing contract.

## Solution Assessment

Yes in direction, but not automation-ready. A native cron exec payload is the maintainable core seam for preserving OpenClaw observability and delivery, but persisted host-command policy needs explicit owner/secops approval first.
