## Summary

Current main and v2026.6.6 provide first-class command cron jobs that run without an agent/model turn, capture bounded output, retain cron history and failure accounting, and use normal delivery modes; the merged implementation resolves the central request.

## What This Changes

The issue requests a zero-LLM cron payload that directly executes scheduled commands and delivers captured output through OpenClaw.

## Best Possible Solution

Keep the shipped typed command-job implementation as the canonical deterministic cron path, including its bounded execution, migration safeguards, observability, and standard delivery semantics.

## Reproduction Assessment

Yes for the original limitation: historical source and discussion establish that cron required agent-mediated payloads, and current source proves the new direct command path removes that dependency.

## Solution Assessment

Yes. A typed scheduler-owned command payload is the narrowest maintainable core solution because it preserves OpenClaw inventory, history, failure accounting, and delivery unlike system-cron or hook workarounds.
