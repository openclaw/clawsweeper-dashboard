## Summary

Keep open: current main and the latest release still leave Docker sandbox `pidsLimit` unset when not configured, and this external PR is useful but not merge-ready because the opt-out/schema/docs/test contract and real Docker proof are still missing.

## What This Changes

The PR adds a `DEFAULT_SANDBOX_PIDS_LIMIT = 1024` constant and uses it as the Docker sandbox `pidsLimit` fallback when no agent or global value is configured.

## Best Possible Solution

Approve the PID cap and explicit opt-out contract, then land this PR or a narrow replacement with aligned schema, types, docs, focused tests, and maintainer-owned changelog while leaving broader CPU and memory defaults to https://github.com/openclaw/openclaw/issues/60827.
