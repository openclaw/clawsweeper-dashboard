## Summary

Keep open: the PR is a useful sandbox hardening candidate, but it is not merge-ready because it changes the default PID behavior for existing Docker sandboxes without an aligned unlimited opt-out, upgrade proof, or after-fix real Docker behavior proof.

## What This Changes

Adds a `DEFAULT_SANDBOX_PIDS_LIMIT` of 1024 and uses it as the fallback when Docker sandbox config has no global or per-agent `pidsLimit`.

## Best Possible Solution

Land a narrow PID-cap hardening fix only after maintainers approve the default and opt-out semantics, align schema/types/docs/tests, and receive real Docker proof for fresh-install and override behavior.

## Reproduction Assessment

Yes. A source-level check shows current main resolves unset Docker sandbox `pidsLimit` to undefined and then emits no `--pids-limit`; the PR changes that path to 1024.

## Solution Assessment

No. The resolver fallback is the right narrow layer, but the PR needs approved opt-out and upgrade semantics plus docs/tests/proof before it is the best fix.
