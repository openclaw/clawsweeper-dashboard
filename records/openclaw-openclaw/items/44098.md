## Summary

Keep open: current main and the latest release still leave Docker sandbox PID limits unset by default, and this PR is the live focused fix path, but it needs an explicit default/opt-out contract plus real Docker behavior proof before merge.

## What This Changes

The PR adds a `DEFAULT_SANDBOX_PIDS_LIMIT` constant and uses it as the fallback `pidsLimit` when resolving Docker sandbox config.

## Best Possible Solution

Land this PR or a narrow replacement after approving the PID cap and opt-out semantics, aligning schema/types/docs/tests/changelog, and leaving broader CPU/memory defaults to https://github.com/openclaw/openclaw/issues/60827.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: resolve Docker sandbox config with no global or per-agent `pidsLimit`, then build Docker create args; current main leaves the value undefined and emits no `--pids-limit`. I did not run a Docker container because this review is read-only.

## Solution Assessment

No as-is. The resolver fallback is the right narrow implementation point, but the PR needs an approved default/opt-out contract and aligned schema, docs, and tests before it is the maintainable fix.
