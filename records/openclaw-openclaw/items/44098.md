## Summary

Keep open: the PR addresses a real current Docker sandbox hardening gap, but the branch is not ready because it changes the unset PID-limit contract without a validated opt-out path and still lacks contributor-provided real Docker behavior proof.

## What This Changes

The PR adds a `DEFAULT_SANDBOX_PIDS_LIMIT` constant and uses it as the fallback Docker sandbox `pidsLimit` when agent/global config omits the field.

## Best Possible Solution

Land this or a narrow replacement after approving the PID cap and opt-out contract, then update resolver, schema/types, docs, tests, and changelog while leaving broader CPU/memory defaults to https://github.com/openclaw/openclaw/issues/60827.

## Reproduction Assessment

Yes. A safe source-level reproduction is to resolve Docker sandbox config with no global or per-agent `pidsLimit`, then pass that config to `buildSandboxCreateArgs`; current main leaves the value undefined and emits no `--pids-limit`.

## Solution Assessment

No as submitted. The resolver fallback is the right narrow implementation point, but the patch needs an approved opt-out contract plus schema, docs, tests, and real behavior proof before it is the best maintainable fix.
