## Summary

Keep open: the missing default PID cap is still present on current main and the latest release, and this PR targets the right resolver path, but it is not merge-ready without an approved opt-out contract, schema/docs/tests alignment, and real Docker behavior proof.

## What This Changes

The PR adds a `DEFAULT_SANDBOX_PIDS_LIMIT` constant and uses it as the fallback Docker sandbox `pidsLimit` when neither agent nor global config sets one.

## Best Possible Solution

Approve the PID-cap default and opt-out semantics, then land a narrow sandbox hardening change that updates resolver constants, schema/types, docs, focused tests, changelog, and real Docker proof while leaving broader CPU/memory defaults to https://github.com/openclaw/openclaw/issues/60827.
