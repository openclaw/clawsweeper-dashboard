## Summary

Keep open: current main and the latest release still lack a default sandbox PID cap, and this PR is a plausible narrow hardening patch, but it changes a security-sensitive default without a validated opt-out contract, aligned schema/docs/tests, or real Docker behavior proof.

The PR adds a sandbox PID-limit default constant and uses it as the Docker sandbox `pidsLimit` fallback when agent/global config omits a value.

## Best Possible Solution
