## Summary

Keep open: current main and the latest release still do not apply a default Docker sandbox PID limit, but this PR changes sandbox default behavior without an approved opt-out/schema/docs/test contract and has no real behavior proof.

## What This Changes

The branch adds a 1024 default PID-limit constant for Docker sandbox containers and uses it when global and per-agent `pidsLimit` are unset.

## Best Possible Solution

Approve the Docker sandbox PID cap and opt-out contract, then land this PR or a replacement that updates resolver behavior, schema/types, docs/help, tests, and changelog while coordinating broader resource defaults separately.

## Reproduction Assessment

Yes. Source inspection is enough: resolving Docker sandbox config with no global or per-agent `pidsLimit` leaves it undefined, and Docker create args then omit `--pids-limit`.

## Solution Assessment

No, not as submitted. The resolver fallback is the right implementation point, but the default value and validated opt-out contract need maintainer approval and schema/docs/tests follow-through before this is the best maintainable fix.
