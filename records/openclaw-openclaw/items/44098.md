## Summary

Keep open: this PR targets a real current-main Docker sandbox hardening gap, but the patch changes the default PID-limit policy without aligning the opt-out/schema/docs/tests contract and still lacks required real Docker behavior proof.

## What This Changes

The PR adds a `DEFAULT_SANDBOX_PIDS_LIMIT` constant and uses it as the Docker sandbox `pidsLimit` fallback when no agent or global value is configured.

## Best Possible Solution

Land this or a narrow replacement only after maintainers approve the PID cap and opt-out contract, then update resolver, schema/types, docs, tests, and changelog together.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: resolve Docker sandbox config without agent/global `pidsLimit`, then build Docker create args; current main leaves the value undefined and emits no `--pids-limit`. I did not run a live fork-bomb or Docker container test during this read-only review.

## Solution Assessment

No as submitted. The resolver fallback is the right narrow implementation point, but the patch needs an approved opt-out contract plus schema/docs/test follow-through before it is the best maintainable fix.
