## Summary

Keep open: current main and the latest release still omit a Docker sandbox PID cap when `pidsLimit` is unset, and this PR is a plausible focused implementation candidate. As submitted, it still needs a validated opt-out contract, schema/docs/tests follow-through, contributor real Docker behavior proof, and maintainer approval of the default cap.

## What This Changes

The PR adds a `DEFAULT_SANDBOX_PIDS_LIMIT` constant and uses it as the fallback in `resolveSandboxDockerConfig` when no per-agent or global Docker `pidsLimit` is configured.

## Best Possible Solution

Land this PR or a narrow replacement after approving the PID cap and opt-out contract, then update resolver constants, schema/types, docs, focused tests, and changelog while leaving broader CPU and memory defaults to https://github.com/openclaw/openclaw/issues/60827.

## Solution Assessment

No as submitted. The resolver fallback is the right narrow implementation point, but the patch needs an approved opt-out contract plus schema, docs, tests, and real behavior proof before it is the best maintainable fix.
