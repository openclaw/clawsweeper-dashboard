## Summary

Keep open: this is a plausible sandbox hardening PR, but it is not merge-ready because the default PID cap and explicit opt-out contract still need maintainer approval, schema/docs/tests follow-through, and real Docker behavior proof.

## What This Changes

The PR adds a `DEFAULT_SANDBOX_PIDS_LIMIT` constant and uses it as the fallback in `resolveSandboxDockerConfig` when per-agent and global Docker `pidsLimit` are omitted.

## Best Possible Solution

Land this or a narrow replacement after approving the PID cap and opt-out contract, then update resolver, schema/types, docs, tests, and changelog while leaving broader CPU and memory defaults to https://github.com/openclaw/openclaw/issues/60827.

## Reproduction Assessment

Yes. A safe source-level reproduction is to resolve Docker sandbox config with no global or per-agent `pidsLimit`, then pass that config to `buildSandboxCreateArgs`; current main leaves the value undefined and emits no `--pids-limit`.

## Solution Assessment

No as submitted. The resolver fallback is the right narrow implementation point, but the patch needs an approved opt-out contract plus schema, docs, tests, and real behavior proof before it is the best maintainable fix.
