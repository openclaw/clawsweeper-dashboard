## Summary

Keep this PR open, but it is not safe to merge as-is. The branch still contains unique macOS work, while the current PR head preserves previously reported blocking regressions, adds direct image-generation behavior outside the gateway/session lifecycle, stores a provider key in app preferences, and lacks real behavior proof from the contributor.

## What This Changes

This PR changes the macOS chat surface from the menu bar popover toward a resizable window with drag-and-drop, composer resizing, Talk Mode handling, toolbar/settings controls, and additional direct Gemini image-generation UI/runtime paths.

## Best Possible Solution

Split this into a narrow macOS window/Talk Mode/drag-and-drop PR with real runtime proof, and handle direct image generation separately through the existing provider, credential, gateway, session-history, and cancellation contracts.

## Reproduction Assessment

Yes for the code-review findings by source inspection: the PR-head shared chat view applies the resize layout to every platform, the direct image path returns before durable transport, and abort only targets gateway run IDs. I did not run the macOS or iOS apps during this read-only review.

## Solution Assessment

No. A smaller owner-reviewed PR should keep the macOS UI fix separate from the new direct Google image-generation, credential, persistence, and cross-platform shared Chat UI changes.

- **[P1] Gate composer resizing to macOS:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatView.swift:73`
  - body: `OpenClawChatView` is shared by the iOS chat sheet, but this branch installs the new `GeometryReader`, resize handle, and fixed 150pt composer height for every platform.   - body: The light and dark assistant bubble branches now both return the same dark color, while assistant text still uses `.labelColor`. In light appearance that makes standard assistant messages low-contrast; keep a light-mode bubble color or switch the text color with the bubble.
  - confidence: 0.89
- **[P2] Do not emit close visibility twice:** `apps/macos/Sources/OpenClaw/WebChatManager.swift:68`
  - body: `controller.close()` already invokes `onVisibilityChanged?(false)` in `WebChatSwiftUIWindowController.close()`, which is wired to `onPanelVisibilityChanged`. Calling `onPanelVisibilityChanged?(false)` again here double-fires the status highlight/suppression update when closing an existing chat window; the same applies to `closeWindow()`.
  - confidence: 0.9
- **[P2] Persist direct image messages before returning:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:570`
  - body: The direct image path appends the prompt and generated image only to local `messages` and returns before the normal `transport.sendMessage` path. Because refresh/history reloads from the gateway, direct image-generation messages disappear after a reload or session refresh unless they are durably recorded first.
  - confidence: 0.87
- **[P2] Wire Stop to the local image request:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:698`
  - body: This adds a fake pending run id for direct image generation so the Stop button appears, but `performAbort()` only calls `transport.abortRun(...)`. That cannot cancel the local `URLSession` request, so Stop remains ineffective until the Google request finishes or times out.
  - confidence: 0.86
