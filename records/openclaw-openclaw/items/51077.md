## Summary

## What This Changes

This PR changes the macOS chat popover toward a windowed UI, adds drag-and-drop/resizable composer/Talk Mode settings, and adds direct Gemini image-generation UI/runtime/metadata handling.

## Best Possible Solution

Split the macOS window, Talk Mode startup, drag-and-drop, and composer-resize work into a narrow owner-reviewed PR with macOS and iOS proof, and route image generation through the existing provider/gateway/auth-profile contracts.

## Reproduction Assessment

Yes for the review findings by source inspection: PR-head shared ChatView affects iOS, direct image generation bypasses transport/history/abort paths, and credentials bypass auth profiles. I did not run the macOS or iOS apps during this read-only review.

## Solution Assessment

No. The maintainable path is a smaller macOS-focused change, with image-generation UI/runtime handled separately through existing provider capability, credential, history, cancellation, and security-policy contracts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Make the image bridge witness internal:** `apps/macos/Sources/OpenClaw/WebChatSwiftUI.swift:1240`
  - body: `GoogleImageGenService` is an internal app type, but this extension declares its protocol witness as `public`. Swift does not allow a public member on an internal enclosing type, so the macOS target will fail to compile; drop `public` from the witness or intentionally make the type public.
  - confidence: 0.86
- **[P1] Gate composer resizing to macOS:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatView.swift:73-128`
  - body: `OpenClawChatView` is also used by the iOS ChatSheet, but this installs the GeometryReader split layout, fixed 150pt composer region, and drag handle for every platform. That regresses iOS by showing a desktop resize affordance and reserving extra composer space in the native sheet.
  - confidence: 0.94
- **[P1] Restore readable assistant bubbles in light mode:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatTheme.swift:24`
  - body: Both light and dark appearance branches now return the same dark assistant bubble color while assistant text still uses the normal label color. In light mode this makes standard assistant messages low contrast; keep a light bubble color or switch the text color with the bubble.
  - confidence: 0.9
- **[P1] Restrict direct image routing to supported Google models:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:567-570`
  - body: `performSend` bypasses the gateway whenever the selected model ID contains image or imagen, but the direct handler is Google-specific. Provider-qualified image models from another provider would be sent to Google's endpoint instead of the configured provider path.
  - confidence: 0.88
- **[P2] Persist direct image runs before returning:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:793-796`
  - body: The direct image branch appends prompt and generated-image messages only to local UI state and returns before `transport.sendMessage`. Since refresh/history reloads from gateway data, those messages disappear after reload unless the run is durably recorded.
  - confidence: 0.87
- **[P2] Wire Stop to the local image request:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:698-699`
  - body: The direct image path inserts a fake `imagegen` pending run so the Stop button appears, but `performAbort` only calls `transport.abortRun`. That cannot cancel the local URLSession request, so Stop remains ineffective until the Google request completes or times out.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: Needs attention because the diff adds app-side Gemini credential storage and direct URLSession calls outside provider/gateway auth and network controls.

Concerns:

- **[medium] Keep provider keys in auth profiles:** `apps/macos/Sources/OpenClaw/GeneralSettings.swift:11`
  - body: `GeneralSettings` binds the Gemini API key to AppStorage/UserDefaults, and `GoogleImageGenService` also reads env/.env directly from the macOS app. Current docs place model API keys in auth profiles, so this creates a separate credential surface outside the normal auth/probe/rotation path.

The direct path bypasses those controls.
  - confidence: 0.88
