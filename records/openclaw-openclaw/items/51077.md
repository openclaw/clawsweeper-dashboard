## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Route image generation through the gateway contract:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:567-570`
  - body: This branch bypasses `transport.sendMessage` for any model id containing `image` and sends it directly to `GoogleImageGenService`. Current main already has registered Google/fal/OpenAI-style image-generation providers with auth-profile resolution, so this creates a second provider path that ignores configured providers, fallback, run lifecycle, cancellation, and persistence; use the existing gateway/image-generation provider contract instead.
  - confidence: 0.94
- **[P1] Store Gemini credentials in auth profiles:** `apps/macos/Sources/OpenClaw/GeneralSettings.swift:11-12`
  - body: Adding a Gemini API key with `@AppStorage` puts provider credentials in macOS UserDefaults instead of the documented `auth-profiles.json`/SecretRef credential surface. That bypasses existing secret lifecycle, profile selection, audits, and credential removal behavior; the native UI should consume credentials through the gateway/provider auth path.
  - confidence: 0.91
- **[P1] Gate the resizable composer layout to macOS:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatView.swift:73-128`
  - body: `OpenClawChatView` is shared with iOS, and the PR replaces its body with a `GeometryReader` split pane plus a 150pt composer and drag handle for every platform. This regresses the iOS ChatSheet layout; keep the old VStack layout on non-macOS or guard the new resize UI with `#if os(macOS)`.
  - confidence: 0.93
- **[P2] Persist direct image messages before returning:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:793-796`
  - body: The direct image path appends user and assistant messages only to `self.messages` and then returns without writing through the transport or refreshing persisted history. A reload or history reconciliation will drop generated-image prompts/results, so either remove the direct path or persist these messages through the gateway session before completing.
  - confidence: 0.88
- **[P2] Remove duplicate close visibility callbacks:** `apps/macos/Sources/OpenClaw/WebChatManager.swift:67-69`
  - body: `controller.close()` already invokes `onVisibilityChanged?(false)` in `WebChatSwiftUIWindowController.close()`, which is wired back to `onPanelVisibilityChanged`. Calling `onPanelVisibilityChanged?(false)` again here can double-fire the status highlight/suppression update on close.
  - confidence: 0.86
- **[P2] Restore the light-mode assistant bubble color:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatTheme.swift:22-24`
  - body: Both branches of `resolvedAssistantBubbleColor` now return the same dark color, while light-mode assistant text still resolves as dark label text. Standard assistant messages become low-contrast in light appearance; keep a light bubble color or switch the text color with the appearance.
  - confidence: 0.84
