## Summary

The PR addresses real macOS chat pain points, but the current branch is not mergeable: source inspection shows shared iOS layout regressions, stale macOS window visibility state, light-mode contrast regression, attachment-limit mismatch, missing real behavior proof, and a new API-key storage concern.

## What This Changes

The PR switches macOS status-item chat to a closable window, adds drag-and-drop/composer resizing/Talk Mode settings, and also adds direct Gemini image generation plus shared chat UI/theme changes.

## Best Possible Solution

Split the work into narrow PRs: land the macOS window/Talk Mode fixes with platform-gated shared UI changes and proof, and route image generation/credential storage through the existing provider or gateway credential design before adding UI controls.

## Reproduction Assessment

Yes for the review findings by source inspection: iOS uses the shared OpenClawChatView, while the PR applies the resizable composer layout to that shared view without a macOS guard. The macOS window-state and attachment-limit issues are also traceable from current main source plus the PR head.

## Solution Assessment

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Gate composer resize layout to macOS:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatView.swift:73-128`
  - body: This shared view is also used by the iOS ChatSheet, but the new GeometryReader split layout and 150pt composer height run on every platform. That makes iOS inherit a fixed composer region and drag handle for a macOS-only interaction, reducing the message viewport and regressing the previous auto-sized mobile layout.
  - confidence: 0.94
- **[P2] Propagate native window close visibility:** `apps/macos/Sources/OpenClaw/MenuBar.swift:186`
  - body: Switching the status-item left click to the closable NSWindow means users can close the chat with the title-bar button. That native close path bypasses WebChatSwiftUIWindowController.close(), so onPanelVisibilityChanged is not fired and the status highlight/HUD suppression state can remain stale.
  - confidence: 0.86
- **[P2] Restore light-mode assistant bubble contrast:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatTheme.swift:23-24`
  - body: The light and dark assistant bubble colors are now the same dark gray, while assistant text still uses .labelColor. In light appearance, .labelColor is dark text on a dark bubble, making standard assistant messages hard to read.
  - confidence: 0.91
- **[P2] Keep gateway-bound images under the gateway cap:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:1359-1360`
  - body: The composer now accepts images up to 20 MB, but the gateway image path still rejects images above MAX_IMAGE_BYTES, currently 6 MB. For normal non-direct chat sends, users can attach a 6-20 MB image that passes UI preflight and then fails deterministically at send time.
  - confidence: 0.87
