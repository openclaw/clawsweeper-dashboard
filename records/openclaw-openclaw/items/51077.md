## Summary

Keep open: the PR addresses real macOS chat pain points, but the current branch is broad, has blocking cross-platform regressions, and introduces direct image-generation credential/runtime paths that need owner review.

## What This Changes

The branch replaces the macOS menu-bar chat panel with an NSWindow, adds toolbar/settings, drag-and-drop, composer resizing, and Talk Mode startup changes, and also adds direct Gemini image-generation UI/runtime and metadata handling.

## Best Possible Solution

Split the macOS window, Talk Mode startup, drag-and-drop, and composer-resize work into a narrow owner-reviewed PR with macOS and iOS proof, and handle image generation separately through the provider, gateway, credential, session-history, and cancellation contracts.

## Reproduction Assessment

Yes for the PR review findings by source inspection: the PR-head diff applies desktop resize layout to shared iOS/macOS chat UI, double-emits window visibility, and bypasses gateway transport for image models. I did not run the macOS or iOS apps during this read-only review.

## Solution Assessment

No. The maintainable path is a smaller macOS-focused PR for the window/Talk/drag-drop fixes, with direct image generation reviewed separately against the existing provider, credential, gateway, persistence, and cancellation contracts.

## Security Review

Status: needs_attention

Summary: The diff introduces app-side Gemini credential storage and direct provider calls outside the existing auth-profile and gateway boundary.

Concerns:

- **[medium] Store provider keys in auth profiles:** `apps/macos/Sources/OpenClaw/GeneralSettings.swift:11`
  - body: GeneralSettings binds the Gemini API key with @AppStorage, which persists it in app preferences. OpenClaw's documented model-credential path is auth profiles, so provider keys should stay in that credential flow rather than a new app preference.
  - confidence: 0.9
- **[medium] Keep provider calls behind the gateway boundary:** `apps/macos/Sources/OpenClaw/GoogleImageGenService.swift:21`
  - body: GoogleImageGenService reads UserDefaults, environment variables, and ~/.openclaw/.env directly, then calls Google's API from the macOS app. That bypasses provider routing, auth profile selection, session persistence, and gateway cancellation policy.
  - confidence: 0.88
- **[low] Remove fixed /tmp provider debug logging:** `apps/macos/Sources/OpenClaw/GoogleImageGenService.swift:123`
  - body: The direct service writes provider response bodies and routing diagnostics to a fixed /tmp/openclaw-debug.log path. A fixed temp log can expose local diagnostic data and should be removed or replaced with OSLog privacy controls.
  - confidence: 0.78
