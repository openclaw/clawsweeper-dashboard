## Summary

Keep open. The PR is still a substantive external feature branch rather than a cleanup-close candidate, but the current patch is not mergeable as-is because it mixes macOS chat window fixes with shared Chat UI regressions, direct Gemini image generation, credential handling, and missing real behavior proof.

## What This Changes

The PR replaces the macOS menu-bar chat panel with a resizable window, adds drag-and-drop and composer resizing, defers Talk Mode startup, adds settings/toolbar controls, and introduces direct Gemini image-generation UI/runtime paths.

## Best Possible Solution

Split the macOS window, Talk Mode startup, drag-and-drop, and composer-resize work into a narrow owner-reviewed PR with real macOS and iOS proof, and handle image generation separately through the provider, gateway, credential, session-history, and cancellation contracts.

## Reproduction Assessment

Yes for the code-review findings by source inspection: the PR-head diff applies a desktop resize layout to the shared iOS/macOS chat view, double-emits visibility close callbacks, and bypasses the gateway transport for image models. I did not run the macOS or iOS apps during this read-only review.

## Solution Assessment

No. The maintainable path is a smaller macOS-focused PR for the window/Talk/drag-drop fixes, with direct image generation reviewed separately against the existing provider, credential, gateway, persistence, and cancellation contracts.

- **[P1] Gate composer resizing to macOS:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatView.swift:73`
  - body: `OpenClawChatView` is also used by the iOS chat sheet, but this installs the new `GeometryReader`, fixed 150pt composer region, and desktop resize handle for every platform.   - confidence: 0.94

  - body: Both appearance branches now return the same dark assistant bubble color while assistant text still uses the normal label color. In light appearance this makes standard assistant messages low-contrast; keep a light bubble color or switch the text color with the bubble.
  - confidence: 0.89
- **[P1] Restrict direct image routing to supported Google models:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:570`
  - body: `performSend` bypasses the gateway whenever the selected model ID contains `image`, but the direct handler is Google-specific. Provider-qualified image models from another provider, such as an OpenAI image model, would be sent to Google's endpoint instead of the configured provider path.
  - confidence: 0.86
- **[P1] Keep gateway attachment preflight at 5 MB:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:1361`
  - body: The shared attachment preflight is raised to 20 MB, but gateway image handling still rejects images above the 5,000,000 byte limit. Normal LLM chats and platforms without the direct image handler would accept 5-20 MB images in the UI and then fail at send time.
  - confidence: 0.87
- **[P2] Do not emit close visibility twice:** `apps/macos/Sources/OpenClaw/WebChatManager.swift:68`
  - body: `controller.close()` already invokes `onVisibilityChanged?(false)`, which is wired to `onPanelVisibilityChanged`. Calling `onPanelVisibilityChanged?(false)` again in the close path double-fires menu-bar highlight and suppression updates.
  - confidence: 0.9
- **[P2] Persist direct image runs before returning:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:571`
  - body: The direct image path appends the prompt and generated images only to local UI state and returns before `transport.sendMessage`. Since refresh/history reloads from gateway data, those image-generation messages disappear after a reload unless the run is durably recorded.
  - confidence: 0.87
- **[P2] Wire Stop to the local image request:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:699`
  - body: The direct image path inserts a fake `imagegen` pending run so the Stop button appears, but `performAbort()` only calls `transport.abortRun(...)`. That cannot cancel the local `URLSession` request, so Stop remains ineffective until the Google request completes or times out.
  - confidence: 0.86
