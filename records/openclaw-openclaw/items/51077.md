## Summary

Keep open. The PR contains useful macOS chat-window work, but it is not mergeable as-is because it mixes a broad macOS UI change with shared iOS chat regressions and direct Gemini credential/network handling outside the existing provider runtime.

## What This Changes

The branch switches macOS menu bar chat toward a persistent resizable NSWindow, adds toolbar/settings controls, drag-and-drop and composer resizing, and also adds direct Gemini image-generation UI, service, and metadata code.

## Best Possible Solution

Keep the useful macOS persistent-window direction, but split or rebase it into a narrower PR that keeps resize behavior macOS-only, reconciles already-shipped drag/drop and Talk Mode overlap, and routes image generation through the existing provider/gateway SecretRef surfaces.

## Reproduction Assessment

Yes. A high-confidence source-level path exists: the PR modifies shared OpenClawChatView used by iOS, adds duplicate visibility callbacks where the controller already emits close state, and bypasses transport.sendMessage for direct image generation.

## Solution Assessment

No. The persistent-window direction is plausible, but this branch is not the best implementation because it combines unrelated provider/security work with macOS UI changes and places macOS-only resizing in a shared iOS chat view.

## Security Review

Status: needs_attention

Summary: Security review needs attention because the branch adds direct Google credential and network handling outside OpenClaw's provider auth/runtime boundary.

Concerns:

- **[medium] Move Gemini credentials out of UserDefaults and URL queries:** `apps/macos/Sources/OpenClaw/GoogleImageGenService.swift:154`
  - body: The PR stores the Gemini API key through AppStorage/UserDefaults and sends it through the direct Google request path instead of the existing provider auth store, SecretRef, and gateway-mediated credential model.   - confidence: 0.86
- **[medium] Avoid bypassing provider runtime policy for image generation:** `apps/macos/Sources/OpenClaw/WebChatSwiftUI.swift:233`
  - body: Wiring GoogleImageGenService directly into the macOS chat view bypasses the existing image_generate provider runtime, routing, auth resolution, and transport history semantics. Provider-specific network code should stay in the provider plugin/runtime path unless maintainers explicitly approve a new app-side trust boundary.
  - confidence: 0.84
