  - body: `performSend` bypasses the gateway whenever the selected model id contains `image`, but the handler is a Google-only client.   - confidence: 0.94
- **[P1] Persist direct image runs before returning:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:567-571`
  - body: The direct image path appends user and assistant messages to `self.messages` and returns before `transport.sendMessage`, so refresh or session reload rebuilds history from the gateway without those generated image messages. Route through durable session/task storage or explicitly persist before bypassing transport.
  - confidence: 0.93
- **[P1] Keep Gemini credentials out of UserDefaults:** `apps/macos/Sources/OpenClaw/GeneralSettings.swift:10-12`
  - body: The new Gemini API key field stores a raw provider key in `@AppStorage`/UserDefaults and the service reads it directly. Provider credentials should go through the existing auth-profile/SecretRef runtime so they can be resolved, rotated, audited, and kept out of app defaults.
  - confidence: 0.9
- **[P1] Gate the resizable composer layout to macOS:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatView.swift:73-131`
  - body: `OpenClawChatView` is shared with iOS, but the new split-pane GeometryReader and fixed composer height are applied unconditionally. That adds a desktop resize affordance and fixed composer region to `ChatSheet`, reducing the message viewport and regressing the existing iOS layout.
  - confidence: 0.92
- **[P2] Keep attachment preflight aligned with the gateway:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:1359-1361`
  - body: This raises the native client image limit to 20 MB while the gateway still rejects oversized image attachments before model dispatch. Non-direct sends can now accept a 5-20 MB image in the composer and then fail at send time; keep the 5 MB preflight or branch the limit by a proven transport/model capability.
  - confidence: 0.89
- **[P2] Remove duplicate visibility callbacks on close:** `apps/macos/Sources/OpenClaw/WebChatManager.swift:63-75`
  - body: `controller.close()` already invokes the window controller visibility callback, which is wired to `onPanelVisibilityChanged`. Calling `onPanelVisibilityChanged?(false)` again from `toggleWindow` and `closeWindow` can double-toggle status highlight state during close.
  - confidence: 0.86
