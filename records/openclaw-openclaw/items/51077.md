## Summary

Keep open. The PR addresses real macOS companion pain points, but the current branch is not safe to merge: source review found shared iOS UI regressions, duplicate visibility callbacks, light-mode contrast regressions, direct image-generation/provider boundary issues, a security concern around Gemini key storage, and no after-fix real behavior proof.

## What This Changes

The branch replaces macOS status-item chat panel behavior with a window and toolbar, adds file drag-and-drop and a resizable composer, defers Talk Mode startup, and adds shared chat UI/theme plus direct Gemini image-generation settings and metadata handling.

## Best Possible Solution

Split this into a narrow macOS windowed-chat/drag-drop/Talk Mode fix, keep shared UI changes platform-guarded, and either remove the direct Gemini work or route it through the established provider/gateway auth contracts before adding proof.

## Reproduction Assessment

Yes for source review: the diff clearly applies macOS-specific composer resizing to the shared view used by iOS and duplicates callbacks that the current close path already emits. I did not run a GUI build or live macOS/iOS repro in this read-only review.

## Solution Assessment

No. A narrow macOS-only implementation with real behavior proof is the maintainable path; the current branch also introduces direct provider/auth/image-generation behavior that should be reviewed or split separately.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Gate the resize layout to macOS:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatView.swift:72`
  - body: `OpenClawChatView` is shared with the iOS chat sheet, but this unconditionally replaces the old flexible VStack with a fixed 150-point composer and a drag handle. That regresses iOS by reserving dead composer space and showing a desktop resize affordance on mobile; keep the new split-pane layout behind `#if os(macOS)` and preserve the existing non-macOS layout.
  - confidence: 0.91
- **[P2] Remove the duplicate close visibility callback:** `apps/macos/Sources/OpenClaw/WebChatManager.swift:68`
  - body: `controller.close()` already invokes `WebChatSwiftUIWindowController.close()`, which emits `onVisibilityChanged(false)` and is wired back to `onPanelVisibilityChanged`. Calling `onPanelVisibilityChanged?(false)` again on this close path can double-fire status-item state updates and leave the highlight/suppression state wrong.
  - confidence: 0.88
- **[P1] Keep light-mode assistant bubbles readable:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatTheme.swift:24`
  - body: The light and dark assistant bubble colors are now the same dark gray while assistant text still uses the normal label color. In light appearance that makes standard assistant messages low-contrast; keep a light-mode bubble color or adjust the text color with the appearance.
  - confidence: 0.86
- **[P1] Restrict direct image routing to the Google image models:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:568`
  - body: This bypasses the gateway whenever the selected model ID contains `image` or `imagen`, but the handler is specifically `GoogleImageGenService`. Existing or future non-Google image models such as provider-qualified OpenAI image models would be sent to Google's endpoint and fail instead of using their configured provider route.
  - confidence: 0.84
- **[P2] Persist direct image messages before returning:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:682`
  - body: The direct image-generation path appends the user and assistant messages only to local `self.messages` and then returns without writing through the transport/session history. A refresh or session reload rebuilds from gateway state, so generated image prompts/results can disappear from the conversation.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The diff introduces a new provider secret path and direct external API client that need security review before merge.

Concerns:

- **[medium] Avoid storing provider API keys in UserDefaults:** `apps/macos/Sources/OpenClaw/GeneralSettings.swift:10`
  - body: The new Gemini key field is backed by `@AppStorage("openclaw.geminiAPIKey")`, and the service reads it directly from UserDefaults. Provider secrets should go through the established credential/auth storage path rather than plain app preferences.
  - confidence: 0.82

  - body: `GoogleImageGenService` sends requests directly from the macOS app and manually reads UserDefaults/env/.env credentials. That bypasses gateway/provider policy, redaction, audit, and configured routing assumptions introduced elsewhere in the app.
  - confidence: 0.76
