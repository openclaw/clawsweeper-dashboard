## Summary

Keep open. The PR is not obsolete on current main, but it still has blocking shared-UI regressions, duplicate macOS window visibility callbacks, and security-sensitive direct provider work that needs maintainer review before merge.

## What This Changes

The PR changes macOS chat toward a persistent resizable NSWindow with toolbar/settings controls, drag-and-drop and composer resizing, while also adding direct Gemini image-generation UI, service, and metadata code.

## Best Possible Solution

Split and rebase the useful macOS window work into a narrower PR, keep any composer resizing macOS-only, preserve already-shipped drag/drop and Talk crash fixes, and route image generation through the existing provider/runtime and SecretRef surfaces.

## Reproduction Assessment

Yes. A high-confidence source-level path exists: the PR modifies shared OpenClawChatView used by iOS, adds duplicate visibility callbacks where controller.close already emits false, and routes image-generation sends outside the current provider transport/runtime path.

## Solution Assessment

No. The persistent-window direction is plausible, but this branch is not the best implementation because it combines unrelated provider/security work with macOS UI changes and places macOS-only resizing in a shared iOS chat view.

  - body: OpenClawChatView is shared with iOS, so replacing the flexible VStack with a fixed-height GeometryReader composer and resize handle on all platforms regresses the iOS chat sheet. Keep the old layout for non-macOS or guard the split layout with #if os(macOS).

  - body: controller.close() already reaches onPanelVisibilityChanged through WebChatSwiftUIWindowController.close(). Calling the callback again can double-fire the status-item visibility state.
  - confidence: 0.9
- **[P2] Remove the extra close callback in closeWindow:** `apps/macos/Sources/OpenClaw/WebChatManager.swift:113`
  - body: closeWindow() repeats the same visibility notification after windowController?.close(), while the controller already emits the false visibility event.
  - confidence: 0.9
- **[P1] Restore a light-mode assistant bubble:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatTheme.swift:24`
  - body: The changed assistant bubble color resolves to the same dark value in light and dark appearances while assistant text remains the normal label color, making light-mode messages low contrast.
  - confidence: 0.82
- **[P1] Route image generation through the provider runtime:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:567-570`
  - body: The direct image-generation branch bypasses transport.sendMessage and localizes results in UI state, so generated image turns are not durable gateway history and non-Google image model IDs can be misrouted.
  - confidence: 0.88
- **[P2] Keep attachment preflight aligned with gateway limits:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:1359-1361`
  - body: Raising the shared composer image limit to 20 MB lets normal gateway-bound sends accept files above the gateway image cap, so users can attach files that are guaranteed to fail later.
  - confidence: 0.86
- **[P2] Wire direct image generation to real cancellation:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:698-699`
  - body: The direct path exposes Stop by inserting a fake pending run ID, but performAbort only calls transport.abortRun and cannot cancel the local URLSession request.
  - confidence: 0.84

  - body: The resize handle pushes NSCursor on hover without tracking a matching pop on repeated hover or disappearance, which can leave the global cursor stuck after the chat window closes.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: Security review needs attention because the PR adds direct Google credential storage and network handling outside OpenClaw's provider auth/runtime boundary.

Concerns:

- **[medium] Keep Gemini credentials out of app UserDefaults:** `apps/macos/Sources/OpenClaw/GeneralSettings.swift:10`
  - body: The PR stores a Gemini API key with AppStorage/UserDefaults and then reads it from a direct Google client, bypassing the existing auth-profile and SecretRef credential model used by provider runtimes.
  - confidence: 0.86
- **[medium] Avoid bypassing provider runtime policy:** `apps/macos/Sources/OpenClaw/WebChatSwiftUI.swift:233`
  - body: Wiring GoogleImageGenService directly into the macOS chat view bypasses the existing image_generate provider runtime, routing, auth resolution, and history semantics; provider-specific network code should stay in the provider/plugin path unless maintainers approve a new app-side trust boundary.
  - confidence: 0.84
