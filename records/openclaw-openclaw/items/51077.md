Full review comments:

- **[P1] Route image generation through the gateway contract:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:564-566`
  - body: The PR bypasses `transport.sendMessage` whenever an image-like model is selected, even though current OpenClaw already exposes cross-provider image generation through `image_generate` with provider routing, auth profiles, fallback, task records, and message delivery. Duplicating that path in the macOS app will drift from the supported provider contract and break non-Google/image-provider setups.
  - confidence: 0.9
- **[P1] Do not store provider secrets in app defaults:** `apps/macos/Sources/OpenClaw/GeneralSettings.swift:11-12`
  - body: Adding `openclaw.geminiAPIKey` to `@AppStorage` stores a model-provider API key in the app defaults path instead of the existing auth-profile/SecretRef credential system. Provider credentials should stay in the gateway credential model or an approved secure storage path, not a new macOS-only settings key.
  - confidence: 0.88
- **[P2] Preserve durable history for direct image runs:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:794-797`
  - body: `performDirectImageGen` appends the user and assistant image messages directly to `self.messages` and returns without persisting through the transport/session history path. After refresh or session reload, history is rebuilt from the gateway and these generated-image turns can disappear.
  - confidence: 0.86
- **[P1] Gate the resize layout to macOS:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatView.swift:71-73`
  - body: `OpenClawChatView` is also used by `apps/ios/Sources/Chat/ChatSheet.swift`, but the new `GeometryReader` split layout and fixed composer height are applied to every platform. Keep the existing VStack layout for non-macOS or guard the resize affordance with `#if os(macOS)` so iOS does not get a desktop resize handle and constrained composer.
  - confidence: 0.9
- **[P1] Reconcile menu-bar left click with current main:** `apps/macos/Sources/OpenClaw/MenuBar.swift:186`
  - body: Current main opens the dashboard from the menu-bar left click, while this stale-base diff changes that path to `toggleWindow(sessionKey:)`. Rebase over main and either preserve dashboard-first behavior or get an explicit maintainer decision before replacing it.
  - confidence: 0.84
- **[P2] Avoid double visibility callbacks on window close:** `apps/macos/Sources/OpenClaw/WebChatManager.swift:67-68`
  - body: `controller.close()` already fires the controller's `onVisibilityChanged(false)` callback wired to `onPanelVisibilityChanged`, so the explicit callback here fires close visibility twice and can desynchronize status-item highlight state.
  - confidence: 0.82
- **[P2] Keep assistant bubbles readable in light mode:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatTheme.swift:23-24`
  - body: The PR returns the same dark assistant bubble color for light and dark appearances while the assistant text still follows the system label color. In light mode that makes dark text render on a dark bubble and can make standard assistant messages unreadable.
  - confidence: 0.78
