## Summary

Keep open: the macOS windowed-chat work is plausibly useful, but current main has not absorbed the central left-click/windowed-chat behavior and this branch is not mergeable as-is because source inspection shows blocking shared-UI, build, provider-routing, persistence, and security-boundary regressions plus missing real behavior proof.

## What This Changes

The PR replaces the macOS menu bar chat popover with a windowed chat UI, adds drag-and-drop/resizable composer/Talk Mode settings and startup handling, and adds direct Gemini image-generation UI/runtime/metadata handling.

## Best Possible Solution

Split the macOS window/Talk Mode/drag-and-drop/composer-resize work into a narrow owner-reviewed PR with macOS and iOS proof, and route image-generation UX through the existing provider/gateway/auth-profile contracts instead of a macOS-only direct client.

## Reproduction Assessment

Yes for the review findings by source inspection: PR-head shared ChatView affects iOS, the image path bypasses transport/history/abort/provider controls, and the visibility callback and Swift access-level issues are visible in the diff. I did not run the apps in this read-only review.

## Solution Assessment

No. The maintainable direction is a smaller macOS-focused change plus a separate provider-backed image-generation design that reuses existing auth, routing, history, timeout, cancellation, and security-policy contracts.

Full review comments:

- **[P1] Make the image bridge witness internal:** `apps/macos/Sources/OpenClaw/WebChatSwiftUI.swift:1240`
  - body: `GoogleImageGenService` is an internal app type, but this extension declares its protocol witness as `public`. Swift does not allow a public member on an internal enclosing type, so the macOS target will fail to compile; drop `public` from the witness or intentionally make the type public.
  - confidence: 0.86
- **[P1] Gate composer resizing to macOS:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatView.swift:73-128`
  - body: `OpenClawChatView` is also used by the iOS chat sheet, but this installs the GeometryReader split layout, fixed 150pt composer region, and drag handle for every platform. That regresses iOS by showing a desktop resize affordance and reserving extra composer space in the native sheet.
  - confidence: 0.94
- **[P1] Restore readable assistant bubbles in light mode:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatTheme.swift:23-24`
  - body: Both light and dark appearance branches now return the same dark assistant bubble color while assistant text still uses the normal label color. In light mode this makes standard assistant messages low contrast; keep a light bubble color or switch the text color with the bubble.
  - confidence: 0.9
- **[P1] Restrict direct image routing to supported Google models:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:567-570`
  - body: `performSend` bypasses the gateway whenever the selected model ID contains `image` or `imagen`, but the direct handler is Google-specific. Provider-qualified image models from another provider would be sent to Google's endpoint instead of the configured provider path.
  - confidence: 0.88
- **[P2] Persist direct image runs before returning:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:793-796`
  - body: The direct image branch appends prompt and generated-image messages only to local UI state and returns before `transport.sendMessage`. Since refresh/history reloads from gateway data, those messages disappear after reload unless the run is durably recorded.
  - confidence: 0.87
- **[P2] Wire Stop to the local image request:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:698-699`
  - body: The direct image path inserts a fake `imagegen` pending run so the Stop button appears, but `performAbort` only calls `transport.abortRun`. That cannot cancel the local URLSession request, so Stop remains ineffective until the Google request completes or times out.
  - confidence: 0.86
- **[P2] Keep gateway attachment preflight at 5 MB:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:1359-1360`
  - body: This raises the shared client-side image limit to 20 MB, while the gateway chat attachment path still rejects images above 5 MB. Non-direct sends can now accept a 5-20 MB file in the composer and then fail with a guaranteed server-side validation error.
  - confidence: 0.88
- **[P2] Avoid double visibility callbacks on window close:** `apps/macos/Sources/OpenClaw/WebChatManager.swift:67-68`
  - body: `controller.close()` already invokes `onVisibilityChanged(false)`, which is wired to `onPanelVisibilityChanged`; the explicit call here fires a second false callback and can desynchronize the status-item highlight state on close.
  - confidence: 0.86
