## Summary

Keep open: current main does not implement the PR’s intended persistent left-click chat window, and the branch still has blocking shared-UI, lifecycle, and provider-boundary issues that need maintainer review before merge.

## What This Changes

The PR changes the macOS companion chat from a menu-bar popover toward a resizable NSWindow, adds toolbar/settings controls, drag-and-drop and composer resizing, and adds direct Gemini image-generation UI/service code.

## Best Possible Solution

Split or rebase this into a focused macOS window/chat PR, keep resize behavior macOS-only, preserve current shared chat and provider contracts, route image generation through the existing image_generate provider pipeline or defer it to a separate reviewed change, and add the required changelog/tests.

## Reproduction Assessment

Yes. A high-confidence source-level path exists from the PR diff and current main: the branch modifies shared OpenClawChatView used by iOS, adds duplicate close callbacks where the controller already emits close state, and bypasses transport.sendMessage for app-side image generation.

## Solution Assessment

No. The persistent-window direction is plausible, but this branch is not the best implementation because it combines unrelated provider/security-sensitive work with macOS UI changes and places macOS-only resizing in a shared iOS chat view.

## Security Review

Status: needs_attention

Summary: The diff introduces concrete credential and provider-runtime boundary concerns by adding app-side Google API key storage and direct network calls for image generation.

Concerns:

- **[medium] Move Gemini credentials out of UserDefaults and URL queries:** `apps/macos/Sources/OpenClaw/GeneralSettings.swift:11`
  - body: The PR stores the Gemini API key through AppStorage/UserDefaults and sends it through an app-side Google request path instead of the existing provider auth/profile model. That broadens where provider secrets live and can expose keys through preferences or request logging.
  - confidence: 0.86
- **[medium] Avoid bypassing provider runtime policy for image generation:** `apps/macos/Sources/OpenClaw/WebChatSwiftUI.swift:233`
  - body: Wiring GoogleImageGenService directly into the macOS chat view bypasses the documented image_generate provider runtime, routing, auth resolution, and transport history semantics. Provider-specific network code should stay in the provider path unless maintainers approve a new app-side trust boundary.
  - confidence: 0.84
