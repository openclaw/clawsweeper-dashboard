## Label Justifications

- `P1`: The issue documents reproducible multi-hour agent-lane outages that require a user reset or gateway restart.
- `impact:session-state`: The embedded-run registry and command-lane task can disagree after abort, leaving the session stuck active.
- `impact:message-loss`: User messages remain queued and unanswered while recovery repeatedly reports success without freeing the lane.
- `impact:crash-loop`: The affected runtime path hangs indefinitely and observed recovery requires a process-level restart.

## Summary

Current main has an unreleased recovery rewrite that improves stale-state and active-session targeting, but the reported abort-without-lane-release state still has a concrete source path and is owned by open closing PR https://github.com/openclaw/openclaw/pull/92303; this issue should remain open until that PR is reviewed, proven in a real setup, and either landed or replaced.

## What This Changes

Make stuck-session recovery actually free a wedged command lane after abort, avoid stale session targets, report ineffective recovery, improve idle-queue observability, and optionally expose recovery thresholds.

## Best Possible Solution

Land a narrow recovery fix that clears the lane only when the same pre-abort task remains active after abort/drain, verifies that the lane can accept queued work before reporting success, and split genuinely separate observability, reset-binding, or configuration requests into focused follow-ups.

## Reproduction Assessment

Yes, at source level with high confidence: an abort can report drained while the same command-lane task remains active, and current release logic returns released=0 when neither queue counter is positive. A live current-main reproduction was not run in this read-only review.

## Solution Assessment

Partly: the linked PR's pre-existing-active-task check is the narrowest plausible fix for the primary release defect, but it needs real behavior proof and should not implicitly claim to solve the report's distinct target-binding, observability, and configuration requests.
