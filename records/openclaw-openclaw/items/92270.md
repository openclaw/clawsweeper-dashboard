## What This Changes

The issue reports that stuck-session recovery logs successful aborts without releasing the session lane, leaving channels wedged until restart or reset.

## Best Possible Solution

Review and either land or replace https://github.com/openclaw/openclaw/pull/92303 for the primary lane-release fix, then split or track the ghost-target and empty-queue diagnostics follow-ups if they still reproduce.

## Reproduction Assessment

Yes, at source level: current main can log an aborted recovery with `released=0` because a successful abort+drain does not reset the lane for post-abort activeCount alone. I did not run the full macOS gateway production scenario in this read-only review.

## Solution Assessment

Unclear until the linked PR is reviewed: resetting the lane when abort leaves the same active task blocked is a plausible narrow fix for the primary failure, but maintainers still need to decide whether ghost-target binding and empty-queue visibility are in scope there or separate follow-ups.
