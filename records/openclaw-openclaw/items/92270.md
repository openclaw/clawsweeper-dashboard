## What This Changes

The issue reports that stuck-session recovery can log `status=aborted` with `released=0` while the session lane remains wedged until restart or `/reset`.

## Best Possible Solution

## Reproduction Assessment

Yes. Source inspection shows current main can still return `status=aborted` with `released=0` after successful abort/drain when the lane remains blocked by the same active task and no queued-work predicate fires; the reporter also supplied production ledger context for that signature.
