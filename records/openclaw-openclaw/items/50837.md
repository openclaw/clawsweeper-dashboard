## What This Changes

The PR adds `publicMode` agent/default configuration, prompt and skills redaction, owner-trust header handling for OpenAI-compatible HTTP ingress, docs, and targeted tests.

## Best Possible Solution

Rebase onto current main and land a security-reviewed public-facing agent mode with default-only inheritance, stream/non-stream parity, documented scope precedence, and redacted real gateway proof.

## Reproduction Assessment

Yes for the PR defects by source inspection of the PR head: default-only `agents.defaults.publicMode` is skipped when no agent entry exists, and streamed `/v1/responses` omits `trustedProxies`. There is no current-main bug reproduction because `publicMode` is a new feature absent from main.

## Solution Assessment

No, not yet. The feature direction is plausible, but the patch needs to fit the current compat HTTP scope model, fix default-only inheritance and streaming parity, and include real gateway proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Honor default-only public mode:** `src/agents/agent-scope.ts:155-156`
  - body: `resolveAgentPublicMode()` delegates to `resolveAgentConfig()`, which returns `undefined` when the target agent has no `agents.list` entry. A config with only `agents.defaults.publicMode: true` therefore skips prompt redaction and non-owner ingress defaults despite enabling public mode.
  - confidence: 0.91
- **[P2] Pass trusted proxies in streamed responses:** `src/gateway/openresponses-http.ts:758`
  - body: The streamed `/v1/responses` path calls `runResponsesAgentCommand()` without `trustedProxies`, while the non-stream branch passes it. Trusted upstreams sending `x-openclaw-sender-is-owner: true` are downgraded only for streamed requests, creating a stream/non-stream behavior mismatch.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The diff changes gateway owner/non-owner semantics and still has concrete trust-boundary issues that should block merge until addressed.

Concerns:

- **[medium] Default-only publicMode is not enforced:** `src/agents/agent-scope.ts:155`
  - body: A config with only `agents.defaults.publicMode: true` and no matching `agents.list` entry resolves to false, leaving prompt metadata and owner defaults exposed despite the configured public mode.
  - confidence: 0.91
- **[medium] Streamed responses ignore trusted-proxy owner assertions:** `src/gateway/openresponses-http.ts:758`
  - body: The streamed OpenResponses path omits `trustedProxies`, so trusted upstream owner assertions cannot be honored there while non-stream requests behave differently.
  - confidence: 0.88
- **[medium] Owner header needs scope-contract reconciliation:** `src/gateway/http-utils.ts:32`
  - body: Current main treats shared-secret OpenAI-compatible HTTP auth as owner/operator access and uses trusted `x-openclaw-scopes` for narrower identity-bearing modes. Adding a separate owner header needs an explicit precedence rule to avoid weakening or confusing that boundary.
  - confidence: 0.78
