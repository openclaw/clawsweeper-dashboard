## Summary

Keep this PR open: current main still lacks the requested public-facing agent mode, but the branch is not ready because two source-review defects remain and the owner-trust design needs maintainer security review against the current compat HTTP scope contract.

## What This Changes

The PR adds `publicMode` agent/default config, prompt and skills redaction, owner-trust header handling for OpenAI-compatible HTTP ingress, docs, and targeted tests.

## Best Possible Solution

Integrate public-facing agent mode as an explicit documented config feature aligned with the current compat HTTP scope model, with default-only inheritance, streaming parity, security review, regression tests, and real gateway proof.

## Reproduction Assessment

Yes for the PR defects: source inspection of the PR head shows default-only `publicMode` returns false through `resolveAgentConfig()`, and the streamed OpenResponses path omits `trustedProxies`. There is no current-main bug reproduction because this is a new feature request.

## Solution Assessment

No, not yet. The direction is useful, but the patch should first reconcile the new owner header with current main's compat HTTP scope contract and fix the default-only and streaming gaps.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Honor default-only public mode:** `src/agents/agent-scope.ts:155-156`
  - body: `resolveAgentPublicMode()` delegates to `resolveAgentConfig()`, which returns `undefined` when the target agent has no `agents.list` entry. A single-agent config with only `agents.defaults.publicMode: true` therefore skips prompt redaction and non-owner ingress defaults even though the operator enabled the default.
  - confidence: 0.91
- **[P2] Pass trusted proxies to streamed responses:** `src/gateway/openresponses-http.ts:758`
  - body: The streamed `/v1/responses` branch calls `runResponsesAgentCommand()` without `trustedProxies`, while the non-stream branch passes it. Trusted upstreams sending `x-openclaw-sender-is-owner: true` are downgraded only in streaming mode, so owner-only behavior diverges by response mode.
  - confidence: 0.88

## Security Review
