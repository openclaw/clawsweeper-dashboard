## What This Changes

The PR adds public-mode agent/default config, prompt and skills redaction, owner-trust header handling for OpenAI-compatible HTTP ingress, docs, and targeted tests.

## Best Possible Solution

Integrate public-facing agent mode as an explicit, documented config feature aligned with the current compat HTTP scope model, with default inheritance, streaming parity, regression tests, and a changelog entry.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Honor defaults when no agent entry exists:** `src/agents/agent-scope.ts:153-154`
  - body: `resolveAgentPublicMode()` delegates to `resolveAgentConfig()`, which returns `undefined` when the requested agent is not listed. A config with only `agents.defaults.publicMode: true` will therefore evaluate as false and skip the prompt and ingress protections the operator enabled.
  - confidence: 0.9
- **[P2] Pass trusted proxies to streamed responses:** `src/gateway/openresponses-http.ts:755`
  - body: The streamed `/v1/responses` path calls `runResponsesAgentCommand()` with `publicMode` but without `trustedProxies`. Since `x-openclaw-sender-is-owner: true` only works for trusted upstreams, trusted callers are downgraded to non-owner only in streaming mode.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `docs/gateway/openai-http-api.md:31`
  - body: This is a user-facing gateway/config feature, but the PR diff does not include `CHANGELOG.md`. The new publicMode and owner-header behavior would otherwise be missing from release notes.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The PR changes gateway owner/non-owner semantics and still has trust-boundary issues that should be resolved before merge.

Concerns:

- **[medium] Default-only publicMode is not enforced:** `src/agents/agent-scope.ts:153`
  - body: Operators can set `agents.defaults.publicMode: true` without listing the default agent, but the PR resolves through `resolveAgentConfig()` and returns false when no entry exists. That can leave runtime and workspace metadata exposed despite the public-mode default.
  - confidence: 0.9
- **[medium] Streamed responses ignore trusted-proxy owner assertions:** `src/gateway/openresponses-http.ts:755`
  - body: The streamed OpenResponses call omits `trustedProxies`, so the new owner header cannot be honored for trusted upstreams in that path. This creates a trust-boundary inconsistency between stream and non-stream requests.
  - confidence: 0.86
- **[medium] Owner header needs reconciliation with scopes:** `src/gateway/http-utils.ts:24`
  - body: Current main derives compat HTTP owner semantics from shared-secret auth and `x-openclaw-scopes` / `operator.admin`. Adding a separate owner header without a clear precedence rule risks confusing deployments and weakening the documented gateway boundary.
  - confidence: 0.78
