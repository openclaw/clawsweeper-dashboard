## Summary

Keep this PR open. Current main does not implement `publicMode`, but the branch still has source-reviewable defects in default inheritance and streamed OpenResponses trusted-proxy handling, plus it lacks required real gateway behavior proof for a security-sensitive trust-boundary change.

## What This Changes

The PR adds `publicMode` agent/default config, prompt and skills redaction, HTTP owner-trust header handling for OpenAI-compatible ingress, docs, and targeted tests.

## Best Possible Solution

Rebase the work onto current main and integrate public-facing agent mode as an explicit security-reviewed config feature aligned with the compat HTTP scope contract, with default-only inheritance, streaming parity, regression coverage, and redacted real gateway proof.

## Reproduction Assessment

Yes for the PR defects: source inspection of the PR head shows default-only `agents.defaults.publicMode` is skipped when no agent entry exists, and streamed OpenResponses omits `trustedProxies`. There is no current-main bug reproduction because `publicMode` is a new feature absent from main.

## Solution Assessment

No, not yet. The direction is useful, but the patch should first fit the current compat HTTP scope model, fix the default-only and streaming gaps, and provide real gateway proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Honor default-only public mode:** `src/agents/agent-scope.ts:155-156`
  - body: `resolveAgentPublicMode()` delegates to `resolveAgentConfig()`, which returns `undefined` before reading `agents.defaults.publicMode` when the target agent has no `agents.list` entry. A default-only single-agent config therefore skips prompt redaction and non-owner ingress defaults even though the operator enabled public mode.
  - confidence: 0.91
- **[P2] Pass trusted proxies to streamed responses:** `src/gateway/openresponses-http.ts:758`
  - body: The streamed `/v1/responses` path calls `runResponsesAgentCommand()` without `trustedProxies`, while the non-stream path passes it. Trusted upstreams sending `x-openclaw-sender-is-owner: true` are downgraded only in streaming mode, so owner-only behavior diverges by response mode.
  - confidence: 0.88

## Security Review

Status: needs_attention
