## What This Changes

## Best Possible Solution

Rebase onto current main and implement public-facing agent mode as a security-reviewed config/API contract with default-only inheritance, streaming parity, documented scope precedence, and redacted real gateway proof.

## Solution Assessment

No, not yet. The feature direction is plausible, but the patch needs to fit the current compat HTTP scope model, fix default-only inheritance and streaming parity, and include real gateway proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Honor default-only public mode:** `src/agents/agent-scope.ts:155-156`
  - body: `resolveAgentPublicMode()` delegates to `resolveAgentConfig()`, which returns `undefined` when the target agent has no `agents.list` entry. A config with only `agents.defaults.publicMode: true` therefore skips prompt redaction and non-owner ingress defaults despite enabling public mode.
  - confidence: 0.91
- **[P2] Pass trusted proxies in streamed responses:** `src/gateway/openresponses-http.ts:758`
  - body: The streamed `/v1/responses` path calls `runResponsesAgentCommand()` with `publicMode` but without `trustedProxies`, while the non-stream path passes it. Trusted upstreams sending `x-openclaw-sender-is-owner: true` are downgraded only for streamed requests, creating a stream/non-stream behavior mismatch.
  - confidence: 0.88
