  - body: Current main already derives compat HTTP owner semantics from `x-openclaw-scopes` and `AuthorizedGatewayHttpRequest`, with shared-secret auth documented as full operator access.   - confidence: 0.91

  - body: The streamed `/v1/responses` path calls `runResponsesAgentCommand` without `trustedProxies`, so trusted upstream requests with `x-openclaw-sender-is-owner: true` are downgraded only when `stream: true`. That creates a stream/non-stream authorization mismatch for owner-only capabilities.
  - confidence: 0.87
