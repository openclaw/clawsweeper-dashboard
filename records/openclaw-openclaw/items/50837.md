## Summary

Keep open: current main does not implement this feature, and the PR head still has blocking default-inheritance and streaming-parity defects plus missing real gateway proof and a trust-boundary review gap.

## What This Changes

Adds per-agent/default `publicMode` config that redacts prompt/runtime details and changes OpenAI-compatible HTTP owner semantics via `x-openclaw-sender-is-owner`, with docs and tests.

## Best Possible Solution

Rebase onto current main and land a security-reviewed public-facing agent mode that honors defaults, keeps stream/non-stream behavior aligned, fits the compat HTTP scope model, and includes real gateway proof.

## Reproduction Assessment

Yes for the PR defects by source inspection of the PR head: default-only `agents.defaults.publicMode` is skipped when no agent entry exists, and streamed `/v1/responses` omits `trustedProxies`. There is no current-main bug reproduction because `publicMode` is a new feature absent from main.

## Solution Assessment

No, not yet. The direction is plausible, but the patch needs default-only inheritance, streaming parity, explicit reconciliation with current HTTP auth semantics, and real gateway proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Honor default-only public mode:** `src/agents/agent-scope.ts:155-156`
  - body: `resolveAgentPublicMode()` delegates to `resolveAgentConfig()`, which returns `undefined` when the target agent has no `agents.list` entry. A config with only `agents.defaults.publicMode: true` therefore skips prompt redaction and non-owner ingress defaults despite enabling public mode.
  - confidence: 0.91
- **[P2] Pass trusted proxies in streamed responses:** `src/gateway/openresponses-http.ts:748-759`
  - body: The streamed `/v1/responses` path calls `runResponsesAgentCommand()` without `trustedProxies`, while the non-stream branch passes it. Trusted upstreams sending `x-openclaw-sender-is-owner: true` are downgraded only for streamed requests, creating a stream/non-stream behavior mismatch.
  - confidence: 0.88
