## Summary

Keep open: the underlying Control UI duplicate-final path is still source-reproducible on current main, but this branch is not mergeable as-is because it lacks inspectable real behavior proof, misses related terminal append paths, and mixes unrelated runtime/security-sensitive changes into the UI fix.

## What This Changes

The branch adds runId-based terminal chat-event deduplication in the Control UI and also changes agent auth, Discord monitor/config files, sandbox tool policy, gaxios compatibility, and AGENTS guidance.

## Best Possible Solution

Land a narrow UI-only fix that records processed terminal runIds for both own-run and other-run final append paths, preserves that state across same-session history reloads, clears it only on real session/agent/history resets, and adds focused regression coverage.

## Reproduction Assessment

Yes, by source inspection. On current main, a renderable own-run final appends a message and clears chatRunId, renderable finals do not force history reload, and a duplicate same-session final can pass through and append again.

## Solution Assessment

No. RunId-based terminal dedupe is the right direction, but this PR misses other-run final appends, clears dedupe state during same-session history reloads, and includes unrelated security-sensitive changes outside the UI bug boundary.
