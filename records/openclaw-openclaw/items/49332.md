## Summary

Keep open: the duplicate-final bug is still source-reproducible on current main, but this PR is not mergeable as-is because it misses terminal paths, clears dedupe state too aggressively, lacks real behavior proof, and bundles unrelated security-sensitive/runtime changes.

## What This Changes

The branch adds Control UI processed-runId deduplication for terminal chat events and also includes unrelated Discord monitor, Google Vertex/gaxios, sandbox policy/config, and AGENTS changes.

## Best Possible Solution

Land a narrow UI-only fix that records processed terminal runIds for both own-run and other-run final append paths, preserves the set across same-session history reloads, clears it only on real session/agent/history resets, and adds focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: call `handleChatEvent` with a renderable own-run `final`, observe `chatRunId` clear, then call it again with the same session/run payload and the handler can append the assistant message a second time.

## Solution Assessment

No. RunId-based terminal dedupe is the right direction, but this branch misses other-run final appends, clears dedupe state during same-session history reloads, and mixes unrelated security-sensitive changes into a UI bug fix.
