## Summary

Keep open. Current `main` still has the two source-level gaps described by the report: JSON embedded fallback can reach the local agent command without preloading the ACPX plugin service, and the Gateway agent RPC still publishes the raw agent result without a descendant-drain reconciliation step that would repair stale `sessions_yield` metadata.

## What This Changes

The issue requests source fixes for explicit `openclaw agent --json` ACP runs so embedded fallback can see the ACPX backend and final JSON reflects completed descendant work after `sessions_yield`.

## Best Possible Solution

Patch the source path with a narrow, policy-aware ACPX backend activation seam for explicit embedded ACP runs, then add final-result reconciliation for `agent --json` so proven descendant completion updates final assistant text, `livenessState`, and `replayInvalid` before JSON is emitted.

## Reproduction Assessment

Yes. A focused source-level reproduction is available by exercising `openclaw agent --json` through embedded fallback or local execution with an ACP session before ACPX is registered in that process, then using an agent turn that yields through `sessions_yield` while descendant ACP work completes.

## Solution Assessment

Yes. The requested direction is the right maintainable shape, provided the backend fix is narrow and policy-aware rather than broadly preloading all plugins for remote JSON, and the metadata fix uses existing run-wait/assistant-reply helpers or an equivalent generic Gateway seam.
