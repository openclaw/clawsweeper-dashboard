## Summary

Keep open. Current main still has the reported source-level gaps: JSON agent fallback can enter the embedded ACP path without locally activating the ACPX backend, and the Gateway agent RPC publishes the raw agent result without descendant-drain reconciliation for stale yielded metadata.

## What This Changes

The issue requests source fixes and regression coverage for explicit `openclaw agent --json` ACP runs that lose ACPX backend visibility or return stale final JSON after `sessions_yield`.

## Best Possible Solution

Add a narrow ACP runtime activation/reconciliation path for explicit embedded ACP runs, then reconcile Gateway `agent --json` final metadata after yielded descendant work drains before emitting the terminal response.

## Reproduction Assessment

Yes. The issue includes a concrete CLI command, and current source still has a focused reproduction path: force `agent --json` into embedded ACP fallback without local ACPX registration, then exercise a yielded descendant run whose terminal JSON remains stale.

## Solution Assessment

Yes. The maintainable fix should be narrow: activate only the configured ACP runtime backend for explicit embedded ACP, and add generic final-result reconciliation in the Gateway agent RPC instead of broad plugin preloading or ACPX-specific core policy.
