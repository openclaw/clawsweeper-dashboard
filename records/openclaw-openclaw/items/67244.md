## Summary

Keep open. Current main still has source-level evidence for both reported gaps: explicit JSON agent fallback can reach the embedded ACP path without loading the ACPX backend in that process, and Gateway agent completion still publishes the raw agent result without descendant-drain reconciliation for stale `sessions_yield` metadata.

## What This Changes

The issue asks OpenClaw to upstream source fixes for explicit ACP `agent --json` backend activation and stale final JSON metadata after `sessions_yield`.

## Best Possible Solution

Add a narrow ACPX backend activation seam for explicit embedded ACP runs, then reconcile Gateway `agent --json` output after yielded descendant completion so final assistant text and liveness metadata are accurate before JSON is emitted.

## Reproduction Assessment

Yes, at source level. `agent --json` skips plugin preload, fallback can call local `agentCommand`, the ACP registry is process-local, and ACP manager still requires a backend without ensuring ACPX registration in that process; the final Gateway response path also returns the raw result without descendant-drain reconciliation.

## Solution Assessment

Yes. The narrow maintainable fix is to activate ACPX only for the explicit embedded ACP path and use an existing or generic wait/read seam to repair final JSON metadata after yielded descendant work drains, without broad plugin preloading or a new config knob.
