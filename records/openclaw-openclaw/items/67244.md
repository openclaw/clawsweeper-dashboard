## Summary

Keep open. Current main still has source-level evidence for both reported failures: `openclaw agent --json` can reach the embedded ACP path without loading the ACPX backend in that process, and the Gateway final response still publishes the raw agent result without reconciling stale `sessions_yield` descendant metadata.

## What This Changes

The issue asks to patch explicit `openclaw agent --json` ACP runs so embedded backend registration is visible in the explicit-run process and final JSON is reconciled after `sessions_yield` descendant completion.

## Best Possible Solution

Add a narrow ACPX backend activation seam for explicit embedded ACP runs, then reconcile Gateway `agent --json` output after yielded descendant completion so final assistant text and liveness metadata are accurate before JSON is emitted.

## Reproduction Assessment

Yes, at source level. `agent --json` skips plugin preload, embedded fallback can call local `agentCommand`, the ACP registry is process-local, and the Gateway final response path returns raw result metadata without descendant-drain reconciliation.

## Solution Assessment

Yes. The narrow maintainable fix is to activate ACPX only for the explicit embedded ACP path and reuse or add a generic descendant-drain reconciliation seam for final JSON metadata, without broad plugin preloading or a new config knob.
