## Summary

Keep open. Current main still has source-level paths matching both reported failures: JSON `agent` skips plugin preload before embedded fallback, ACP backend lookup is process-local/direct, and Gateway publishes the raw agent result without descendant-drain reconciliation before the final JSON response.

## What This Changes

Make explicit `openclaw agent --json` ACP runs see the embedded ACPX backend and return completed final JSON after `sessions_yield` descendant completion.

## Best Possible Solution

Add a narrow source-level ACPX backend activation path for explicit embedded ACP execution, then reconcile Gateway `agent --json` output after yielded descendants drain.

## Reproduction Assessment

Yes, at source level. Current main still has a JSON agent path that can enter embedded fallback without plugin backend preload, and Gateway final response emission still wraps the raw agent result rather than a descendant-drained completed state.

## Solution Assessment

Yes. The narrow maintainable fix is to activate ACPX only where explicit embedded ACP execution needs a local backend and to clean up terminal JSON metadata after descendant completion, without broad plugin preloading or a new config knob.
