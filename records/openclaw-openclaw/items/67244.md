## What This Changes

The issue reports two bugs in explicit `openclaw agent --json` ACP runs: missing local embedded ACPX backend registration and stale final JSON metadata after yielded descendant work completes.

## Best Possible Solution

Patch source-level ACPX activation for explicit embedded ACP execution, then implement terminal JSON reconciliation after yielded descendants drain in the shared Gateway/agent lifecycle contract rather than as a dist-only hotfix.

## Reproduction Assessment

Yes, at source level. Current main still has a JSON agent path that skips ACPX plugin preload before embedded fallback, a process-local ACP backend registry, and a Gateway final response path that returns raw agent metadata without descendant-drained reconciliation.

## Solution Assessment

Partly. Lazy ACPX activation is the narrow source fix for the backend visibility failure; the final JSON cleanup should be aligned with the related foreground descendant lifecycle contract before adding a new `completed` liveness state.
