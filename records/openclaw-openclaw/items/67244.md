## Summary

Keep open. Current main has partial ACPX readiness improvements, but source inspection still shows unresolved paths for JSON ACP runs: embedded fallback can enter local agent execution without the JSON command preloading plugin backends, and Gateway final JSON still publishes the raw agent result before any descendant-drain reconciliation for `sessions_yield` completion metadata.

## What This Changes

The issue requests source-level fixes for explicit `openclaw agent --json` ACP runs so the embedded ACPX backend is visible and final JSON metadata is reconciled after yielded descendant work completes.

## Best Possible Solution

Add a narrow source-level ACPX backend activation path for explicit embedded ACP execution, then reconcile Gateway `agent --json` output after yielded descendant completion before emitting the final JSON payload.

## Reproduction Assessment

Yes, at source level. Current main still has a JSON agent path that can enter embedded fallback without plugin backend preload, and the Gateway final response returns the raw yielded result instead of a descendant-drained completed state.

## Solution Assessment

Yes. The maintainable fix is narrow: activate ACPX only where the explicit embedded ACP path needs it and add or reuse a descendant-drain reconciliation seam for terminal JSON metadata, without broad plugin preloading or a new config option.
