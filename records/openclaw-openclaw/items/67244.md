## Summary

Keep open. Current main still has source-level evidence for both reported gaps: JSON agent fallback can enter the embedded ACP path without loading the ACPX backend in that process, and Gateway agent completion still returns the raw agent result without descendant-drain reconciliation for stale `sessions_yield` metadata.

## What This Changes

The issue requests source fixes for explicit `agent --json` ACP runs so embedded ACPX backend lookup works locally and final JSON reflects descendant completion after `sessions_yield`.

## Best Possible Solution

Add a narrow ACPX backend activation seam for explicit embedded ACP runs, then reconcile Gateway `agent --json` output after yielded descendant completion so final assistant text and liveness metadata are accurate before JSON is emitted.

## Reproduction Assessment

Yes, at source level. `agent --json` skips plugin preload, fallback can call local `agentCommand`, the ACP registry is process-local, and the ACP manager still requires a backend without first ensuring ACPX registration in that process.

## Solution Assessment

Yes. The reporter's proposed direction is the narrow maintainable fix if it avoids broad plugin preloading for all JSON agent runs and uses an existing or generic Gateway wait/read seam for final-result reconciliation.
