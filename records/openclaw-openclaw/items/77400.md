## Summary

Keep open: current main still has schema-only doctor cleanup that can remove active auth-profile data, and #77482 is an open closing PR rather than a merged fix.

## What This Changes

Prevent `openclaw doctor --fix` from deleting active auth-profile data for providers still configured with API keys or used in model fallbacks.

## Best Possible Solution

Land a narrow doctor repair guard with regression coverage so active auth-profile data is preserved and visibly warned about during `doctor --fix`.

## Reproduction Assessment

Yes, from source inspection: current main runs schema-only unknown-key cleanup in `doctor --fix`, `auth.profiles.*` is strict, and runtime auth selection uses configured profiles for provider credential resolution. I did not run a live fixture in this read-only pass.

## Solution Assessment

Yes: preserving active auth-profile fields during unknown-key cleanup, keyed by provider credentials, fallback references, or explicit auth order, is the narrow maintainable fix. Closing should wait for #77482 or an equivalent merged change with regression tests.
