## Summary

Keep open: this is a fresh, detailed regression report, and current main still has the WebChat path that routes configured image-model turns as inline images with a one-turn model override, while the reported providers fail that format.

## What This Changes

WebChat image uploads with a non-vision primary model fail when routed through the configured image model, producing provider format errors instead of usable visual context.

## Best Possible Solution

Repair WebChat text-only image routing to use a provider-compatible media-understanding path, or gate inline image overrides to providers proven compatible, with regression coverage and live provider proof.

## Reproduction Assessment

Yes, with caveat: the reporter provides detailed live steps and logs, and current source shows the matching inline WebChat image-model override path. I did not run the live provider flow in this read-only review.

## Solution Assessment

Unclear until the provider payload mismatch is validated. The narrowest likely fix is to keep WebChat text-only image routing on the compatible media-understanding/tool path used by the working direct flow, or add a provider capability gate for inline overrides.
