## Label Justifications

- `P3`: The request is useful observability work for an unsupported external-injection workflow and needs product/security direction before implementation.
- `impact:message-loss`: The reported symptom is a user turn that reaches the transcript/model but is not visible in the live TUI chat log.
- `impact:session-state`: The live TUI state can diverge from the durable session transcript for externally injected turns.
- `impact:security`: The proposed wake-injection event and external user-turn flow touch operator read scopes and potentially sensitive session content.

## Summary

Keep open: current main still lacks a live TUI path for transcript-backed external user turns, but the requested wake-specific event depends on the broader external injection contract and should not be handled as a narrow automatic bug fix.

## What This Changes

Render externally injected user turns live in the TUI when those turns have been accepted into the active session transcript.

## Best Possible Solution

Decide the supported external user-turn injection contract with https://github.com/openclaw/openclaw/issues/83809, land or replace the TUI session-message subscription work, then render accepted external user turns through the chosen scoped event with dedupe, provenance labeling, and regression coverage.

## Reproduction Assessment

Partly. Source inspection gives high confidence for the TUI event gap because current main does not subscribe to or dispatch session.message or wake.injected events, but the exact wake-socket path is not a supported current-main feature and was not live-reproduced.

## Solution Assessment

Unclear. A wake-specific event may be useful, but the narrower maintainable path may be to consume the existing session.message transcript stream once maintainers decide the external injection contract.
