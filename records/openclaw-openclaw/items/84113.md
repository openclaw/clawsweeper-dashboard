## Summary

Keep open: current main still does not render live transcript/user-turn events in the bundled TUI, and the wake-specific path depends on the open external-injection product/security decision at https://github.com/openclaw/openclaw/issues/83809 plus the existing TUI live-transcript fix path at https://github.com/openclaw/openclaw/pull/73402.

## What This Changes

The issue requests live TUI rendering, dedupe, and source labeling for externally injected user turns that already reach the active conversation transcript.

## Best Possible Solution

Settle the supported external-injection contract with https://github.com/openclaw/openclaw/issues/83809, then render accepted external user turns through the existing `session.message` live transcript path or add a narrowly scoped provenance event only if that contract cannot carry the needed source label.

## Reproduction Assessment

Yes for the source-level TUI gap: current main has no `session.message` or `wake.injected` dispatch path in the bundled TUI, while history reload can render stored user messages. The full wake-socket scenario is not high-confidence because that injection surface is currently a local/proposed API.

## Solution Assessment

Unclear; the proposed `wake.injected` event may be useful, but the existing scoped `session.message` transcript event is likely the narrower maintainable UI path if maintainers accept external user-turn injection at all.
