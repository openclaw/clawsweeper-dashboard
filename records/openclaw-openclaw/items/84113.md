## Label Justifications

- `impact:message-loss`: The reported user turn reaches transcript/model state but is omitted from the live TUI chat log.
- `impact:session-state`: The live TUI display can diverge from the durable conversation transcript for externally injected turns.

## Summary

Keep open: current main still lacks a TUI live-render path for externally accepted user-turn transcript inserts, and the supported external injection contract remains tied to the open product/security discussion at https://github.com/openclaw/openclaw/issues/83809 plus the generic TUI transcript-sync work at https://github.com/openclaw/openclaw/pull/73402.

## What This Changes

The issue requests that externally injected user turns be broadcast and rendered live in the TUI with active-session filtering, dedupe, and source labeling.

## Best Possible Solution

Settle the supported external live-session injection contract in https://github.com/openclaw/openclaw/issues/83809, then render accepted external user turns in TUI through the chosen scoped transcript/event path, preferably reusing `session.message` if it is sufficient, with provenance labeling and regression coverage.

## Reproduction Assessment

No for the full wake-socket scenario: that injection surface is local/proposed rather than supported on current main. Source inspection does show the UI gap: current TUI neither subscribes to `session.message` nor handles a `wake.injected` event live.

## Solution Assessment

Unclear: a new `wake.injected` event may be useful, but the existing scoped `session.message` path and the open TUI transcript-subscription PR look like the narrower maintainable surface if maintainers accept external user-turn injection.
