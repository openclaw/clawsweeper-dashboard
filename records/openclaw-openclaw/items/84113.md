## Summary

Keep open: current main and the latest release still lack a live TUI path for externally injected user turns, but the right fix depends on an unresolved external-injection/event/security contract rather than a narrow automatic bug fix.

## What This Changes

Request to add a supported event and render path so OpenClaw TUI shows accepted externally injected user turns that already reach the transcript and model.

## Best Possible Solution

Choose the supported external-injection contract first, preferably reusing `session.message` if it is sufficient; then render accepted active-session user turns in TUI with provenance labeling, dedupe, and regression coverage.

## Reproduction Assessment

Yes for the source-level UI gap: current TUI has no `wake.injected` or `session.message` live render path, while `addUser` is only used for local sends/history. No for a full wake-socket reproduction, because that injection surface is local/proposed rather than a supported current-main API.

## Solution Assessment

Unclear; the proposed wake-specific event may be useful, but the existing subscribed `session.message` transcript event may be the narrower maintainable surface if maintainers accept external user-turn injection at all.
