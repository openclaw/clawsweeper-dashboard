## Summary

Keep open: current main still lacks the TUI live transcript subscription/dispatch path, and the wake-specific part depends on the open external-injection contract discussion rather than a narrow automatic fix.

## What This Changes

Request that externally injected user turns which reach the active transcript also appear live in openclaw-tui, with scoped delivery, dedupe, and provenance labeling.

## Best Possible Solution

Land or replace the scoped TUI session.message subscription work, then decide the supported external-injection contract in https://github.com/openclaw/openclaw/issues/83809 and add a wake-specific event only if the transcript stream cannot carry the accepted user turn safely.

## Solution Assessment

Unclear. The existing session.message subscription contract is likely the narrowest rendering path, while a new wake.injected event should wait for the external-injection API and security decision.
