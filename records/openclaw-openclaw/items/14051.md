## Summary

Keep open. Current main still lacks activity-based heartbeat idle-timeout behavior: the public config/schema/docs expose periodic cadence, active-hours, busy-lane deferral, delivery, timeout, light-context, isolation, and reasoning controls, while the runner remains interval/wake driven and does not gate on recent allowed inbound user activity.

## What This Changes

The issue requests an activity-based heartbeat idle timeout so heartbeats pause after a configured period without allowed inbound user activity.

## Best Possible Solution

Add a maintainer-approved heartbeat activity-window config through exported types, strict schema, config docs/inventory, and runtime gating on recent allowed inbound user/channel activity while preserving existing interval, active-hours, busy-lane retry, delivery, lightContext, isolatedSession, and per-agent merge behavior.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence the requested capability is absent: the strict heartbeat schema omits the proposed key and the runner schedules by interval/wake without lastInteractionAt gating.

## Solution Assessment

Unclear. Activity-based gating is a plausible direction, but the config name, allowed-activity definition, and session/channel boundary need maintainer approval before implementation.
