## Summary

Keep open: current main and v2026.5.20 still lack a heartbeat idle-timeout config/schema/docs path and the heartbeat runner remains interval/wake driven; related discussion treats this as the canonical tracker, and the open cron-only idle PR does not solve heartbeat behavior.

## What This Changes

The issue requests a heartbeat config option that starts heartbeats after allowed inbound activity and pauses them after a configurable idle window.

## Best Possible Solution

Keep this as the canonical heartbeat idle-timeout tracker and add a maintainer-approved activity/idle config contract plus runtime gating at the correct agent/session/channel boundary, coordinated with cron idle semantics and existing heartbeat cadence, active-hours, wake, busy-deferral, delivery, lightContext, isolatedSession, and per-agent merge behavior.

## Reproduction Assessment

Yes, source-reproducible for the missing capability: current main and v2026.5.20 reject the proposed idle-timeout key through strict heartbeat schema and have no recent-activity gate in the heartbeat runner or scheduler. This is a feature request rather than a failing existing contract.

## Solution Assessment

Unclear. Activity-based gating is a plausible end state, but maintainers should decide the public config shape, units, defaults, and allowed-activity boundary before implementation.
