## Summary

Keep open. Current main still lacks the requested heartbeat idle-timeout/activity gate: heartbeat types, strict schema, docs, and runner scheduling remain interval/wake based and do not check recent allowed inbound activity; #59567 was already closed as a duplicate of this canonical tracker.

## What This Changes

Add a configurable heartbeat idle timeout so heartbeats start after user activity and pause after a period of inactivity.

## Best Possible Solution

Keep this as the canonical tracker and implement a maintainer-approved heartbeat activity/idle contract across config types, strict schema, docs, config-honor inventory, runner scheduling/execution, and focused regression tests.

## Reproduction Assessment

Yes for the missing requested behavior: static inspection on current main confirms the proposed config is rejected by the strict schema and the heartbeat runner has no recent-activity idle gate. This is a feature-gap reproduction rather than a broken existing-contract bug.

## Solution Assessment

Unclear. The direction is useful, but the exact config name, activity source, and scheduler semantics need maintainer choice before implementation; reusing lastInteractionAt may help, but should not silently conflate session expiry with heartbeat policy.
