## Summary

Keep open. Current main still lacks a heartbeat activity-window or idle-timeout config and runtime gate, and the live discussion plus the closed duplicate keep this as the canonical product tracker rather than something to close.

## What This Changes

Add a heartbeat option that starts after real user activity and pauses after an inactivity window.

## Best Possible Solution

Define an additive heartbeat activity-window contract, then wire exported types, strict schema, config docs/inventory, and runner or scheduler gating while preserving cadence, active-hours, wake retry, busy-lane, isolated-session, and delivery behavior.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence the requested feature is absent because strict config rejects the proposed idle fields and the runner/scheduler do not gate heartbeats on recent allowed inbound activity.

## Solution Assessment

Unclear. Activity-based gating is plausible, but maintainers should choose the public config name, defaults, allowed-sender semantics, and whether the gate belongs in scheduler state or run preflight.
