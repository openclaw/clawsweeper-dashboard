## Summary

Keep open. Current main still lacks activity-based heartbeat idle-timeout behavior, and the live labels/comments already route this as a maintainer product decision rather than a stale duplicate.

## What This Changes

Add a heartbeat idle-timeout option so periodic heartbeats start after user activity and pause after a configurable window of inactivity.

## Best Possible Solution

Keep this as the canonical tracker and add a maintainer-approved heartbeat activity/idle config contract, with schema/help/docs/config-honor coverage and runtime gating that preserves existing interval, active-hours, busy retry, wake, lightContext, isolatedSession, delivery, and per-agent merge behavior.

## Reproduction Assessment

Yes, source-reproducible. Current main rejects the proposed idle-timeout key through the strict heartbeat schema and has no recent-activity gate in the runner or scheduler path.

## Solution Assessment

Unclear. Activity-based gating is a plausible end state, but the exact public config and activity boundary need maintainer direction before implementation.
