## Summary

Keep this issue open because current main still lacks an activity-based heartbeat idle timeout: heartbeat config remains interval/active-hours/busy-lane based, and the runner does not gate execution on recent allowed inbound activity.

## What This Changes

Add a heartbeat idle-timeout option so periodic heartbeats start after user activity and pause after an inactivity window.

## Best Possible Solution

Add a documented heartbeat activity/idle contract across exported types, strict schema, generated config metadata, docs, runtime scheduling/execution, and regression tests, while preserving existing interval, active-hours, wake, busy-lane, delivery, and per-agent merge behavior.

## Reproduction Assessment

Not applicable. This is a feature request rather than a broken existing contract, and current-main source inspection shows no idle-timeout key or recent-activity gate to reproduce as a failure.

## Solution Assessment

Unclear for automation. The requested direction is plausible, but the exact config semantics and activity boundary need maintainer API/product judgment before a narrow implementation is safe.
