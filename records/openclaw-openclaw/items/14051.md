## Summary

Keep open. Current main still lacks activity-based heartbeat idle-timeout behavior: the config, schema, docs, and config-honor inventory expose periodic cadence, active-hours, busy-lane deferral, delivery, timeout, light-context, isolation, and reasoning controls, while the runner remains interval/wake driven and the discussion keeps this as the canonical tracker with #59567 closed as a duplicate.

## What This Changes

The issue requests a heartbeat idle-timeout option that starts heartbeats after allowed user activity and pauses them after a configured inactivity window.

## Best Possible Solution

Add a maintainer-approved heartbeat activity-window config through exported types, strict schema, config labels/help/generated docs, docs, and config-honor inventory, then gate scheduling or execution on recent allowed inbound user activity while preserving existing interval, active-hours, busy-lane retry, delivery, lightContext, isolatedSession, and per-agent merge behavior.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence the requested capability is absent: the strict heartbeat schema omits the proposed key and the runner schedules by interval/wake without lastInteractionAt gating.

## Solution Assessment

Unclear. Activity-based gating is a plausible direction, but the exact config name and activity boundary need maintainer approval before implementation.
