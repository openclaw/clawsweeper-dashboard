Keep open: current main and the latest release still lack activity-based heartbeat idle-timeout behavior, and the open heartbeat PRs only address the narrower missing-HEARTBEAT.md token-burn path. ## What This Changes

The issue requests a heartbeat option that starts periodic heartbeats after user activity and pauses them after a configured inactivity window.

## Best Possible Solution

Keep this as the canonical tracker, choose the public heartbeat activity/idle contract, then gate scheduled heartbeats on recent allowed inbound activity while preserving existing active-hours, wake, busy-deferral, isolated-session, and delivery behavior.

## Reproduction Assessment
