## Summary

Current main still has no activity-expiry gate for heartbeats, and no merged related item owns the missing behavior. The request remains useful, but it needs maintainer judgment because it adds public config and must define whether activity is session-scoped, agent-scoped, or channel/account-scoped.

## What This Changes

Add an optional heartbeat idle window so heartbeats pause after recent allowed inbound user activity expires and resume when activity returns.

## Best Possible Solution

Add one opt-in heartbeat inactivity duration using the canonical interactive timestamp for the resolved heartbeat session, with explicit semantics for missing activity and tests proving pause, resume, per-agent isolation, and unchanged scheduler behavior.

## Reproduction Assessment

Yes at source level: current main's heartbeat config, runner, tests, and docs expose no inactivity-window gate, while session interaction timestamps are available elsewhere. This is a verified feature gap, not a failing existing contract.

## Solution Assessment

Yes in product intent, but the proposed shape is not yet the best fully specified solution. The implementation should reuse resolved-session interaction state rather than introduce a parallel activity ledger, and maintainers must define missing-activity and cross-channel semantics.
