## Summary

Keep open. Current main and the latest release still lack heartbeat activity/idle-timeout config and runtime gating, and the live comments/labels already route this as the canonical maintainer product decision rather than a duplicate or stale report.

## What This Changes

Request to add a configurable heartbeat idle window so periodic heartbeats pause after inactivity and resume after allowed inbound user activity.

## Best Possible Solution

Keep this as the canonical heartbeat idle-timeout tracker and add a maintainer-approved config contract plus runtime gating that preserves existing interval, active-hours, busy retry, wake, lightContext, isolatedSession, delivery, and per-agent merge behavior.

## Reproduction Assessment

Yes, source-reproducible for the missing capability: current main rejects the proposed idle-timeout key through the strict heartbeat schema and has no recent-activity gate in the runner or scheduler. This is a feature request rather than a failing existing contract.

## Solution Assessment

Unclear. Activity-based gating is a plausible end state, but the exact public config name, units, defaults, and activity boundary need maintainer direction before implementation.
