## What This Changes

Request an activity-based heartbeat option that starts after user activity and pauses after an idle window such as `idleTimeoutMinutes`.

## Best Possible Solution

Keep this as the canonical heartbeat idle-timeout tracker and implement a maintainer-approved config/schema/docs/runtime gate using canonical recent user/channel activity while preserving existing heartbeat interval, active-hours, busy-lane, delivery, isolated-session, and wake semantics.

## Solution Assessment
