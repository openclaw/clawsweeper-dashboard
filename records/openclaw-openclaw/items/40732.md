## Summary

Keep open: current main still uses the raw textarea behind a reveal-before-edit gate, so the requested editor is not implemented. This PR is useful source work, but it is not mergeable as-is because it conflicts with current raw-secret redaction behavior, has JSON5/i18n/changelog gaps, and overlaps #40009 and #75466 without being cleanly superseded.

## What This Changes

The PR replaces the Control UI raw config textarea with a CodeMirror-based Lit editor, adding line/column status, JSON5 path breadcrumbs, styling, locale strings, dependencies, and browser tests.

## Best Possible Solution

Land one rebased Control UI JSON5 editor that preserves raw secret reveal-before-mount behavior, uses the generated i18n pipeline, handles valid JSON5 keys, includes the required changelog entry, and then closes or supersedes overlapping raw-editor threads.

## Reproduction Assessment

Yes. A source-level reproduction is clear: current main keeps raw secrets out of the DOM before reveal, while the PR hunk mounts the raw value directly; the escaped-key and locale issues follow from the PR parser/token rules and current locale registry/generation policy.

## Solution Assessment

No. CodeMirror is a reasonable direction for the raw editor, but this branch is not the best mergeable solution until it is rebased around current redaction, generated locales, JSON5 identifier handling, changelog policy, and the overlapping implementation decision.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve the reveal gate before mounting raw config:** `ui/src/ui/views/config.ts:801`
  - body: Current main keeps sensitive raw config text out of the DOM until the reveal button is clicked. This replacement binds `props.raw` directly into the new editor, so rebasing this shape would expose secrets before explicit reveal instead of preserving the existing redaction wrapper.
  - confidence: 0.9
- **[P2] Parse escaped JSON5 identifier keys:** `ui/src/ui/components/config-raw-editor-path.ts:39-40`
  - body: JSON5 object keys can be ECMAScript IdentifierName values such as `\u0066oo`. The resolver starts bare tokens only from `isIdentifierStart` or number punctuation, so a leading backslash is skipped and breadcrumbs can show the wrong path for valid JSON5 configs.
  - confidence: 0.86
- **[P2] Regenerate all shipped locale bundles:** `ui/src/i18n/locales/en.ts:127-134`
  - body: Current main ships 18 lazy non-English Control UI locales and treats those bundles plus `.i18n` metadata as generated output. Adding these strings to English and only the older locale subset by hand leaves shipped locale structures out of sync with the generator contract.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `ui/src/ui/views/config.ts:799-805`
  - body: This is a user-facing Control UI feature, but the PR has no `CHANGELOG.md` update. Repo policy expects a single-line entry under the active `### Changes` section with non-bot contributor credit when known.
  - confidence: 0.82
