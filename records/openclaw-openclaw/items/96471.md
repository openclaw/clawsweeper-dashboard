## Best Possible Solution

Ship the closed `default|full` selector only if maintainers want a supported shortcut for default-disabled doctor lint checks; otherwise keep `--only <check-id>` as the sole opt-in path.

Yes, if maintainers accept the product surface: parsing a closed `default|full` value at the CLI boundary and passing it to the existing lint runner is the narrowest implementation.
