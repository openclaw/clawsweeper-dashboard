## Summary

Keep open. Current main still lacks a per-agent restart lifecycle surface: the CLI, Gateway agents RPC handlers, protocol schema, session reset path, and docs expose agent management plus session-key reset, but not an agent-id-scoped soft/hard restart.

## What This Changes

The issue requests `openclaw agents restart <agent-id>` or an equivalent per-agent restart command without restarting the whole Gateway.

## Best Possible Solution

Add an additive, admin-scoped `agents.restart` Gateway lifecycle API plus `openclaw agents restart <agent-id>`, reusing session cleanup where appropriate and explicitly specifying soft/hard provider and channel behavior.

## Reproduction Assessment

Yes for the missing capability: current source and docs inspection show no agent-id restart command, RPC, protocol schema, or docs entry. The production runaway-agent scenario is a feature use case rather than a deterministic failing test.

## Solution Assessment

Unclear because there is no proposed patch. The requested direction belongs in core Gateway lifecycle control rather than ClawHub/plugin work because no current plugin seam can restart another agent's provider/session lifecycle safely.
