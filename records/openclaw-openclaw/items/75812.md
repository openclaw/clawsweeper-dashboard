## Summary

Keep open. Current main still lacks a per-agent restart CLI, Gateway RPC, protocol schema, or docs surface, and the existing reset machinery is scoped to a session key rather than agent-wide lifecycle control.

## What This Changes

Request adds `openclaw agents restart <agent-id>` or an equivalent per-agent restart surface without restarting the whole Gateway.

## Best Possible Solution

Add an additive, admin-scoped `agents.restart` Gateway lifecycle API plus CLI wrapper, with documented soft/hard semantics, health results, lock behavior, and targeted tests.

## Reproduction Assessment

Yes for the missing capability: source and docs inspection on current main show no per-agent restart command, RPC handler, protocol schema, method scope, or docs entry. The reported runaway-agent scenario is a feature use case rather than a deterministic failing test.

## Solution Assessment

Unclear because there is no patch to evaluate. The requested direction belongs in core Gateway lifecycle control, but maintainers should settle the RPC contract, authz, soft/hard behavior, and cleanup guarantees first.
