## Summary

Keep open. Current main still lacks a per-agent restart surface in the CLI, Gateway RPC handler list, protocol schemas, method registry, scopes, and docs; the existing reset path is session-key scoped rather than agent-id lifecycle control.

## What This Changes

Request adds a CLI/Gateway command to restart one agent's session/provider, with soft and hard modes, instead of restarting the whole gateway.

## Best Possible Solution

Add an additive, admin-scoped agents.restart Gateway lifecycle API plus openclaw agents restart <agent-id>, reusing session cleanup where appropriate and explicitly specifying soft/hard provider, channel, lock, and health behavior.

## Reproduction Assessment

Yes for the missing capability: current source and docs inspection show no agent-id restart command, RPC method, protocol schema, method scope, or docs entry. The production runaway-agent scenario is a feature use case rather than a deterministic failing test.

## Solution Assessment

Unclear because there is no proposed patch. The requested direction belongs in core Gateway lifecycle control rather than ClawHub/plugin work, but the soft/hard contract needs maintainer design before implementation.
