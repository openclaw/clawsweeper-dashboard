## Summary

Keep open. Current main still lacks the requested per-agent restart surface: `openclaw agents` has no restart subcommand, the Gateway `agents.*` RPC/protocol/SDK surfaces have no restart method, and the existing reset machinery is session-key scoped rather than agent-wide lifecycle control.

## What This Changes

Request a core CLI/Gateway lifecycle command to restart one agent's session/provider state without restarting the full Gateway.

## Best Possible Solution

Add an additive, admin-scoped core Gateway lifecycle API plus `openclaw agents restart <agent-id>`, with soft semantics built from existing session cleanup and a separately specified hard provider/channel reinit path.

## Reproduction Assessment

Yes for the missing capability: source and docs inspection show no per-agent restart command, RPC, protocol schema, SDK helper, or docs entry, while `sessions.reset` requires a session key. The runaway production scenario is a feature use case rather than a deterministic failing test.

## Solution Assessment

Unclear because there is no proposed patch. The requested direction belongs in core Gateway lifecycle control rather than a plugin, but maintainers should settle the additive RPC, admin scope, soft/hard semantics, health result, and lock behavior before implementation.
