## Summary

Keep open. Current main still has no per-agent restart command, Gateway RPC, or protocol contract; the existing reset and abort paths are session-key scoped while docs expose only full Gateway service restart.

## What This Changes

Request an `openclaw agents restart <agent-id>` capability to restart one agent session/provider without bouncing the whole Gateway.

## Best Possible Solution

Add an additive admin-scoped `agents.restart` Gateway method plus `openclaw agents restart <agent-id>`, with soft session/provider cleanup and separately specified hard channel/provider reinitialization semantics.

## Reproduction Assessment

Yes for the missing capability: source and docs inspection of CLI, Gateway handlers/protocol, method scopes, session reset/abort, and docs shows no `agents restart` or agent-id lifecycle RPC. The runaway production scenario is a feature use case rather than a deterministic failing test.

## Solution Assessment

Unclear because there is no proposed patch. The requested direction likely belongs in core Gateway lifecycle control, but maintainers should settle the API, auth scope, soft/hard semantics, health result, and lock behavior before implementation.
