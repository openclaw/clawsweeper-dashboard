## Summary

Keep open. Current main has session-level reset and background-task cancellation primitives, but it does not expose the requested per-agent restart command or a defined hard restart path that reinitializes only one agent's provider/transport without bouncing the gateway.

## What This Changes

Request to add `openclaw agents restart <agent-id>` or an equivalent control-plane surface for restarting one configured agent's session/provider without restarting the whole gateway.

## Best Possible Solution

Add a documented admin-scope agent restart surface that maps a soft restart onto the existing session reset cleanup for the selected agent, defines a hard restart through explicit provider/channel lifecycle hooks, and verifies one-agent isolation without requiring a full gateway restart.

## Reproduction Assessment

Yes. The gap is reproducible by inspecting the current command registration and docs: `openclaw agents` has no `restart` subcommand, while the available reset primitive is only the lower-level `sessions.reset` RPC keyed by session.

## Solution Assessment

Unclear. Wrapping `sessions.reset` could be a good soft-restart foundation, but the full proposal includes hard provider/transport restart, exit codes, lock handling, and process reaping that need a maintainer-owned contract rather than a quick CLI alias.
