## Summary

Keep open. Current main still lacks an agent-id-scoped restart lifecycle surface, and the related open restart/follow-up work addresses whole-gateway restart continuation rather than a per-agent provider/session/channel restart command.

## What This Changes

The issue requests a per-agent restart command such as `openclaw agents restart <agent-id>` with soft and hard restart behavior.

## Best Possible Solution

Design an additive admin-scoped Gateway lifecycle API plus `openclaw agents restart <agent-id>` that restarts only the named agent and clearly defines soft/hard provider, session, lock, channel, process cleanup, and health behavior.

## Reproduction Assessment

Yes for the missing capability: current source and docs show no agent-id-scoped restart command, Gateway method, protocol schema, handler, or scope entry. The runaway-provider recovery scenario is a feature use case rather than a deterministic current-main bug reproduction.

## Solution Assessment

Unclear because there is no patch yet. The requested direction likely belongs in core Gateway lifecycle control rather than ClawHub/plugin work, but soft/hard semantics need maintainer design before implementation.
