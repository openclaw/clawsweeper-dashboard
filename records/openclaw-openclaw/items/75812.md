## Summary

Keep open. Current main still lacks a per-agent restart lifecycle surface: the CLI, Gateway method registry, agents RPC handlers, protocol schema, session reset path, and CLI docs expose agent management plus session-key reset, but not an agent-id-scoped soft/hard restart. The prior ClawSweeper comment reached the same conclusion, and the related restart/lock issues do not supersede this feature gap.

## What This Changes

Add `openclaw agents restart <agent-id>` or an equivalent surface to restart one agent's session/provider without restarting the whole Gateway.

## Best Possible Solution

Design and implement an additive, admin-scoped `agents.restart` Gateway lifecycle API plus `openclaw agents restart <agent-id>`, reusing session cleanup where appropriate and specifying soft/hard provider and channel behavior.

## Reproduction Assessment

Yes for the missing capability: current source and docs inspection show no agent-id restart command, RPC, protocol schema, method registry entry, or docs entry. The runaway-provider scenario is a feature use case rather than a deterministic failing test.

## Solution Assessment

Unclear because there is no proposed patch. The requested direction belongs in core Gateway lifecycle control rather than ClawHub/plugin work because current plugin/skill surfaces cannot safely restart another agent's provider/session/channel lifecycle.
