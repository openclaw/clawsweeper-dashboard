## Summary

Keep open. Current main still lacks a per-agent restart CLI or Gateway RPC, and the existing restart/reset paths are either whole-Gateway lifecycle control or session-key-scoped cleanup. The related restart and lock reports add useful context but do not supersede this feature request.

## What This Changes

Add a per-agent restart command or equivalent Gateway-backed surface that restarts one configured agent's session/provider lifecycle without restarting the whole Gateway.

## Best Possible Solution

Add a core, admin-scoped Gateway lifecycle API and CLI command for one configured agent. The soft path should reuse existing session abort/reset cleanup across that agent's active sessions, while any hard provider or channel reinitialization should be explicitly specified, permission-gated, documented, and covered by focused Gateway/session/runtime tests plus a changelog entry.

## Reproduction Assessment

Yes for the missing capability: source and docs inspection show no `agents restart` command or `agents.restart` RPC, and the existing reset operation requires a session key. The production wedged-provider scenario is a feature use case rather than a deterministic failing test in the repository.

## Solution Assessment

Unclear because there is no proposed patch. The requested direction belongs in core Gateway lifecycle control rather than ClawHub, but maintainers should settle the soft/hard contract and admin authorization before implementation.
