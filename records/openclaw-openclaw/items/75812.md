## Summary

Keep open. Current main still lacks an agent-id-scoped restart lifecycle surface; the CLI, Gateway method registry, agents protocol schema, agents handlers, method scopes, and docs expose agent management plus session-key reset/abort, but not soft or hard restart for one agent.

## What This Changes

Request adding `openclaw agents restart <agent-id>` with soft/hard behavior to restart one agent without bouncing the whole Gateway.

## Best Possible Solution

Design an additive, admin-scoped Gateway lifecycle API plus `openclaw agents restart <agent-id>` that restarts only the named agent and clearly defines soft/hard provider, session, lock, channel, and health behavior.

## Reproduction Assessment

Yes for the missing capability: current source and docs show no agent-id restart command, Gateway method, protocol schema, handler, scope entry, or docs entry. The runaway-provider recovery scenario is a feature use case rather than a deterministic current-main bug reproduction.

## Solution Assessment

Unclear because there is no patch yet. The requested direction belongs in core Gateway lifecycle control rather than ClawHub/plugin work because current plugin surfaces cannot safely restart another agent's provider/session/channel lifecycle.
