## Summary

Keep open. Current main still lacks the requested per-agent restart surface: there is no `openclaw agents restart` CLI command, no `agents.restart` Gateway RPC/protocol contract, and the existing reset/abort machinery is session-key scoped rather than agent-wide lifecycle control.

## What This Changes

Request to add `openclaw agents restart <agent-id>` or an equivalent per-agent soft/hard restart command without restarting the whole Gateway.

## Best Possible Solution

Add an additive, admin-scoped `agents.restart` Gateway lifecycle API plus `openclaw agents restart <agent-id>`, reusing session cleanup where appropriate and explicitly specifying soft/hard provider and channel behavior.

## Reproduction Assessment

Yes for the missing capability: current source and docs inspection show no agent-id restart command, RPC, protocol schema, SDK/UI helper, or docs entry. The production runaway scenario is a feature use case rather than a deterministic failing test.

## Solution Assessment

Unclear because there is no proposed patch. The direction belongs in core Gateway lifecycle control, but maintainers should settle the RPC shape, authorization, lock handling, soft/hard semantics, and health result before implementation.
