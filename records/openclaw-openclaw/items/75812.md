## Summary

Keep open. Current main still lacks the requested per-agent restart surface: `openclaw agents` only exposes list/add/bind/unbind/delete/set-identity, the Gateway `agents.*` RPC set has no restart method, and the existing reset/abort machinery is session-key based rather than agent-wide provider/session lifecycle control. The related restart and lock reports add useful context, but they do not supersede this narrower feature request.

## What This Changes

Add a command such as `openclaw agents restart <agent-id>` to reset or restart one configured agent without restarting the whole Gateway.

## Best Possible Solution

Add a core, admin-scoped Gateway lifecycle API and CLI command for one configured agent. The soft path should build on existing `sessions.abort`/`sessions.reset` cleanup for that agent's active sessions, while any hard provider/channel reinitialization should be explicitly designed, documented, permission-gated, and covered by focused Gateway/session/runtime tests plus a changelog entry.

## Reproduction Assessment

Yes for the missing capability: source and docs inspection show no `agents restart` command or `agents.restart` RPC, and the existing reset operation requires a session key. The production recovery scenario itself is a feature use case rather than a failing test case.

## Solution Assessment

Unclear because there is no proposed patch. The requested direction belongs in core/Gateway rather than ClawHub because it mutates agent runtime/session/provider lifecycle state, but maintainers should settle the soft/hard contract before implementation.
