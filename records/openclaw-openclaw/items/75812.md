## Summary

Keep open. Current main still lacks an agent-id-scoped restart lifecycle surface; existing CLI, Gateway, protocol, SDK, docs, and tests cover agent CRUD plus session-key reset/abort, but not restarting one named agent's provider/session/channel lifecycle.

## What This Changes

Request to add `openclaw agents restart <agent-id>` or an equivalent surface for restarting one agent without restarting the full Gateway.

## Best Possible Solution

Design an additive admin-scoped Gateway lifecycle API plus CLI/SDK/docs support that restarts only the named agent and explicitly defines provider, session, lock, channel, and health behavior.

## Reproduction Assessment

Yes for the missing capability: current source and docs show no `agents restart` command, `agents.restart` Gateway method, protocol schema, handler, SDK wrapper, or docs entry. The runaway-provider recovery scenario is a feature use case rather than a deterministic current-main bug reproduction.

## Solution Assessment

Unclear because there is no patch yet. The likely maintainable direction is a small additive core Gateway lifecycle design rather than ClawHub/plugin work, because current plugin surfaces cannot safely restart another agent's provider/session/channel lifecycle.
