## Summary

Keep this open. Current main and the latest release only rotate failed sessions when the transcript is missing; they still preserve and reuse failed main sessions when a transcript exists, and the active recovery path is the open terminal-session PR at https://github.com/openclaw/openclaw/pull/89045 rather than a merged fix.

## What This Changes

The issue asks OpenClaw to stop leaving a failed `agent:main:main` session bound so the next TUI launch can recover instead of crashing.

## Best Possible Solution

Define one terminal main-session recovery policy and land it through the shared gateway/session paths, coordinating with https://github.com/openclaw/openclaw/pull/89045 so TUI launches and visible turns recover without losing intended transcript continuity.

## Reproduction Assessment

No, not for the exact TUI crash. Source and test evidence do show the relevant current-main state: failed sessions with existing transcripts remain reusable and bound to `agent:main:main`.

## Solution Assessment

Unclear. The missing-transcript rotation already landed, while the broader recovery policy needs maintainer choice because the active PR preserves session id and transcript continuity instead of always rotating.
