## Summary

Keep this open. Current main has partial guards for failed sessions, but it still intentionally reuses `status: "failed"` main sessions when their transcript exists, so the requested failed-session rotation/clearance behavior is not fully resolved; the adjacent open terminal-session recovery work at https://github.com/openclaw/openclaw/pull/89045 should be reviewed before choosing a separate fix path.

## What This Changes

The issue asks OpenClaw to stop keeping a failed main session bound to `agent:main:main` so the next TUI launch or turn starts from a safe session instead of rehydrating failed state.

## Best Possible Solution

Define one terminal main-session policy: visible TUI/user turns should clear or rotate failed main-session state without crashing or dropping messages, preserving transcript continuity only when maintainers explicitly want that behavior.

## Reproduction Assessment

No high-confidence end-to-end TUI crash reproduction was established. Source and test evidence do show current main still keeps failed sessions reusable when their transcript exists, while only rotating the missing-transcript case.

## Solution Assessment

Unclear as implemented today: current main's partial missing-transcript rotation is narrower than the requested behavior, and current tests intentionally preserve failed-session reuse with existing transcripts. The safer next step is maintainer policy review alongside https://github.com/openclaw/openclaw/pull/89045 before changing rotation semantics.
