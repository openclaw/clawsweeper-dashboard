## Summary

Keep this open. Current main and v2026.6.8 still reuse failed main sessions when an existing transcript is present, while the active recovery work at https://github.com/openclaw/openclaw/pull/89045 is still open and does not fully settle the TUI launch crash versus rotate-or-clear policy.

## What This Changes

The issue asks OpenClaw to stop leaving a failed main session bound as the active TUI session after provider failure or timeout.

## Best Possible Solution

Choose one shared terminal-session recovery policy and land it through the gateway, auto-reply, restart-recovery, and TUI history paths, coordinating with https://github.com/openclaw/openclaw/pull/89045.

## Solution Assessment

Unclear. The current open recovery PR fixes the shared session/admission layer without rotating session ids, but maintainers still need to decide whether this issue should rotate failed TUI main sessions or clear terminal state in place.
