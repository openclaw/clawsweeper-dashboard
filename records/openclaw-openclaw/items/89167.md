## Summary

Keep this open. Current main and v2026.6.8 still allow failed main sessions with an existing transcript to remain reusable, and the active recovery PR is still open while the TUI rotate-versus-clear policy remains unsettled.

## What This Changes

The issue asks OpenClaw to stop leaving failed `agent:main:main` sessions bound in a way that can crash the next TUI launch.

## Best Possible Solution

Adopt one shared terminal-session recovery policy that prevents the TUI from rehydrating a failed main session, coordinate it with https://github.com/openclaw/openclaw/pull/89045, and add focused TUI/session regression proof for the chosen behavior.

## Reproduction Assessment

No, not for the exact TUI crash. Source and current tests do show that a failed main session with an existing transcript remains reusable and that TUI history loading does not filter failed entries.

## Solution Assessment

Unclear. The linked recovery PR fixes shared session/admission behavior by clearing terminal state in place, but maintainers still need to decide whether this TUI case should rotate, unbind, or preserve transcript continuity.
