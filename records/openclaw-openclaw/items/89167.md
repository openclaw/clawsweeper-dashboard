## Summary

Keep open: current main and v2026.6.8 still intentionally reuse a failed main session when its transcript exists, and the overlapping recovery PR is open while the TUI rotate-versus-clear policy remains undecided.

## What This Changes

The issue asks OpenClaw to stop binding a failed `agent:main:main` session so the next TUI launch rotates or recovers instead of rehydrating the failed session.

## Best Possible Solution

Adopt one shared terminal-session recovery policy for main and channel sessions, coordinate it with https://github.com/openclaw/openclaw/pull/89045, and add focused TUI/session regression proof for the chosen behavior.

## Reproduction Assessment

No, not for the exact TUI crash. Source and current tests do show that a failed main session with an existing transcript remains reusable and that TUI history loading does not filter failed entries.

## Solution Assessment

Unclear. The open recovery PR fixes related terminal-session behavior by clearing failed state in place, but maintainers still need to decide whether this TUI case should rotate, unbind, or preserve transcript continuity.
