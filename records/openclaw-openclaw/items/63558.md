## What This Changes

The issue requests an agent selector in the Control UI Dreaming tab so users can switch Dreaming status and diary views between agent workspaces.

## Best Possible Solution

Yes. Source inspection on current main shows the Dreaming tab lacks selector props/rendering, tab refresh calls unscoped loaders, the controller sends empty doctor.memory.* params, and the gateway resolves the default agent.

## Solution Assessment

Yes in direction, but not yet on main. The narrow maintainable fix is to finish the linked PR or an equivalent implementation that makes selected agent identity the single source of truth for Dreaming status, diary, and actions.
