## What This Changes

Explicit agent runs should resolve configured model aliases before capability and thinking validation.

## Best Possible Solution

Resolve bare explicit model overrides through the existing alias-aware resolver before visibility, harness, auth-profile, and thinking validation, with provider-wildcard regression coverage; keep multiple aliases as separate product/schema work.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure an alias under `agents.defaults.models`, include a matching provider wildcard such as `openai/*`, then run `openclaw agent --model <alias> --thinking xhigh`; current main parses the alias before using the alias index.

## Solution Assessment

Yes for the central bug. The narrow maintainable fix is to reuse the existing alias-aware resolver before capability validation and add a regression test; the multiple-alias config shape should remain separate.
