## What This Changes

Explicit agent runs should resolve configured model aliases before thinking and capability validation.

## Best Possible Solution

Review and land the narrow resolver reuse in https://github.com/openclaw/openclaw/pull/84333, while keeping the proposed multi-alias config shape as separate product work.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure an alias under `agents.defaults.models` with a matching provider wildcard such as `openai/*`, then run an explicit agent command using the alias and a thinking level supported only by the alias target.

## Solution Assessment

Yes for the central bug. Reusing the existing alias-aware resolver before visibility, harness, auth-profile, and thinking validation is the narrow maintainable fix; supporting `aliases[]` should remain a separate config decision.
