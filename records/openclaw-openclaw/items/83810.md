## What This Changes

Explicit agent runs should resolve configured model aliases before validating thinking and model capability support.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/84333; the central fix should reuse the existing alias-aware resolver before validation, while multiple-alias config support stays separate.

## Reproduction Assessment

Yes. Source inspection shows a configured alias can be parsed as a default-provider literal before thinking validation when a provider wildcard allows that literal key.

## Solution Assessment

Yes. Reusing `buildModelAliasIndex` and `resolveModelRefFromString` in the explicit override path is the narrow fix; the proposed `aliases[]` config shape should remain a separate follow-up.
