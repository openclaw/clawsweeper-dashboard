## Summary

Keep open: the broken-IPv6 transport problem is plausible, but this PR should not merge as-is because the value-only timeout guard can overwrite an explicit Node runtime policy, the branch conflicts with current main's refactored guarded-fetch/Undici network surface, and the external PR still lacks inspectable after-fix real behavior proof.

## What This Changes

The PR raises Node's default autoSelectFamily attempt timeout to 300 ms during `ensureGlobalUndiciStreamTimeouts()` and adds unit coverage for default and non-default timeout values.

## Best Possible Solution

Keep any remaining broken-IPv6 fallback in the owned transport or guarded-fetch policy for the affected call paths, while preserving explicit Node runtime options unless maintainers intentionally choose a process-wide default change.

## Reproduction Assessment

Yes for the PR-introduced policy regression: Node reports `250` for the untouched default and for explicit CLI or `NODE_OPTIONS` timeout settings, matching the PR's value-only guard. No live broken-IPv6 provider outage was reproduced in this read-only review.

## Solution Assessment

No. The direction addresses a plausible fallback gap, but this patch is not the narrowest maintainable fix because it can overwrite explicit operator timeout policy and does not yet fit current main's per-dispatcher guarded-fetch structure.
