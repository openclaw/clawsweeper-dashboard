## Best Possible Solution

Normalize local OpenAI-compatible streaming usage at the transport boundary so llama.cpp timing metadata and aggregate totals feed the existing promptTokens/session total path without adding a new config knob.

## Reproduction Assessment

No live repro was run, but the source path is clear enough to reproduce with a focused fixture: feed the OpenAI-completions stream parser a local-backend final chunk that uses llama.cpp timings or aggregate-only totals and the session context total will not reflect the real prompt snapshot.

## Solution Assessment
