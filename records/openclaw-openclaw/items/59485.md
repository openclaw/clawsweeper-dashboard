## Summary

Keep this PR open, but do not merge it as-is. The branch is stale and dirty against current main, and the patch still does not add a typed, end-to-end Paperclip runtime-env contract from runner params through attempt tool creation into exec child-process env construction.

## What This Changes

The PR adds Paperclip runtime auth env helpers and tests, attempts to pass Paperclip env values through embedded runner and exec tool defaults, and refactors several fetch test casts into a shared helper.

## Best Possible Solution

Decide the product boundary first. If Paperclip env injection is obsolete, close this PR as superseded by the provider runtime-auth path; if it is still required, rebuild it as a typed end-to-end contract that avoids Paperclip-specific core coupling where possible and includes attempt-boundary plus exec env tests.

## Reproduction Assessment

Yes. Static inspection reproduces the failure path: the PR uses runner and exec-default fields absent from current main types, and current exec env construction never consumes the proposed Paperclip env default.

## Solution Assessment

No. This is not the best current fix because it adds Paperclip-specific fields at only part of the pipeline; the safer direction is either the existing generic provider runtime-auth mechanism or a fully typed, generic exec env contract with explicit security review.
