- **[P1] Resolve proxy hostnames before accepting them:** `src/agents/provider-request-config.ts:194-197`
  - body: The new sanitizer only checks the literal proxy hostname. The explicit-proxy dispatcher later hands the URL to Undici ProxyAgent, which resolves the proxy host at connect time, so `proxy.attacker.example` can still resolve to 169.254.169.254 or loopback and bypass the intended SSRF block. Resolve and validate/pin the proxy host before returning the URL or before creating the dispatcher.
  - confidence: 0.88

A model-created poll can therefore ping `@everyone`, users, roles, or channels through its question/options despite this hardening. Apply the outbound sanitizer to poll fields or enforce it inside the poll normalization/send path.
