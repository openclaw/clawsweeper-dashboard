## Summary

Keep open. Current main still lets a slow Feishu bot-identity probe block Feishu account transport startup, the focused fix PR is closed unmerged, and the related Feishu SDK timeout report is adjacent diagnostics rather than a full superseding fix for this Windows startup delay.

## What This Changes

The issue asks OpenClaw to avoid a 30-second Feishu API timeout delaying Windows Gateway startup when Feishu is enabled.

## Best Possible Solution

Fix this in the Feishu plugin by making startup bot identity lookup bounded or asynchronous, starting account transport promptly, and recovering bot identity in the background without weakening group mention gating.

## Reproduction Assessment

Yes, source-reproducible: configure Feishu in WebSocket mode and make the startup bot identity probe hang until its timeout; current main awaits that path before transport starts. I did not establish a live Windows credential repro in this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix is Feishu-owned startup decoupling with regression coverage. A core Gateway timeout change would be broader and would not preserve the Feishu-specific bot identity and mention-gating contract as directly.

## Repair Work Prompt

Fix the Feishu startup delay reported in https://github.com/openclaw/openclaw/issues/63257. Current main awaits the Feishu bot identity probe before account transport starts, so a slow @larksuiteoapi/node-sdk/Axios call can block Gateway startup for the 30,000 ms startup probe timeout. Implement the fix inside the Feishu plugin, not core Gateway: make startup use a startup-safe, cancellable or asynchronous bot identity path so WebSocket/webhook transport starts promptly, then recover botOpenId/botName in the background. Preserve strict requireMention/group gating while bot identity is unknown. Use #74187 only as prior art because it is closed unmerged, and account for the related diagnostics in #72721 without trying to solve every SDK/Axios network issue in this PR. Add focused regression coverage proving a hanging or slow startup probe does not block account transport. Do not add a new user config option, do not change dependency pins or @larksuiteoapi/node-sdk versions, and do not move Feishu-owned behavior into core. Add a CHANGELOG.md entry because this is a user-facing startup fix.
