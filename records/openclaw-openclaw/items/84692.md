## Summary

Keep open: current main still lets ordinary onboarding writes bypass the size-drop clobber guard, and there is no focused regression proving reruns preserve both `agents.list` and top-level `bindings`.

## What This Changes

The issue asks for onboarding reruns to preserve or explicitly reject destructive changes to persisted `agents.list` and `bindings`.

## Best Possible Solution

Ordinary onboarding should merge-preserve multi-agent routing state and fail closed on destructive config shrink, with explicit reset/import carve-outs and regression coverage for `agents.list` plus `bindings`.

No high-confidence end-to-end reproduction was run because this review is read-only. Source inspection does show the relevant safety failure: ordinary onboarding paths pass `allowConfigSizeDrop: true`, so a size-drop payload is logged but not rejected.

## Solution Assessment

Yes for the primary issue: preserving multi-agent state and narrowing the size-drop bypass is the maintainable fix. The dangling-binding validation suggestion should be compatible or split if it would newly fail existing configs at startup.
