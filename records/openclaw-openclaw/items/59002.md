## Summary

Keep open. Current main still has a credible Gmail setup/session-policy regression, and the closest open repair at https://github.com/openclaw/openclaw/pull/72001 is partial and still needs maintainer policy review before it can resolve the reporter's no-session/no-notification path.

## What This Changes

Restore Gmail hook dispatch so setup-created Gmail watches create routable hook sessions and can deliver inbound-mail notifications to Signal.

## Best Possible Solution

Revise the Gmail setup path to emit a complete, intentional session policy for the effective mapping it installs, then repair or replace https://github.com/openclaw/openclaw/pull/72001 with tests that cover startup validation and runtime `/hooks/gmail` dispatch.

## Reproduction Assessment

Yes. There is a high-confidence source-level reproduction path: setup installs the templated Gmail preset without the policy that current gateway startup and mapped dispatch require; live Gmail/Tailscale/Signal was not exercised.

## Solution Assessment

No landed solution exists. The current open PR seeds prefixes only, which may pass the startup gate but does not by itself satisfy the runtime `mapping-templated` `allowRequestSessionKey` gate or settle the intended Gmail session policy.
