## Summary

Keep this open: current main still has a source-reproducible Gmail hook/session-policy gap, and the related fixes are either closed incomplete or still open for maintainer sequencing rather than merged into main.

## What This Changes

The issue reports that Gmail Pub/Sub hook delivery stopped creating per-email `hook:gmail:*` sessions and stopped sending downstream Signal notifications after upgrading from 2026.3.28 to 2026.3.31.

## Best Possible Solution

Define one supported Gmail hook delivery and session-routing path, then update setup/doctor/docs/tests so upgraded Gmail users either get a complete safe push config or an explicitly accepted pull-mode migration without breaking existing channel delivery.

## Reproduction Assessment

Yes from source inspection, but not from a live repro in this review. Current main combines a templated Gmail preset with a default-deny session-key gate, while setup still does not emit the complete opt-in policy needed for that preset.

## Solution Assessment

Unclear until maintainers choose the security/UX path. A prefix-only repair is not enough; the best fix must decide between complete explicit Gmail session-routing opt-in, a static-session default, doctor migration, or the separately reviewed pull delivery path.
