## Best Possible Solution

## Reproduction Assessment

No live high-confidence failing reproduction was run in this read-only review. The source path is clear: current main calls `loadHistory` through `maybeRefreshHistoryForRun` before finalizing non-local displayable chat finals, matching the reported stale-history race.

## Solution Assessment

No, not as written. The intended no-refresh fix is narrow and maintainable for displayable finals, but the implementation should preserve the current reconciliation path for non-displayable message finals or explicitly prove that backend contract changed.
