Keep open: current main still has the schema/validator mismatch, and the open closing PR at https://github.com/openclaw/openclaw/pull/83267 is the right place to finish the fix before this issue closes.

## What This Changes

The issue reports that documented `models.providers.<id>.timeoutSeconds` overlays for built-in providers fail startup validation unless `baseUrl` and `models` are also provided.

## Best Possible Solution

Land a bounded fix, likely through https://github.com/openclaw/openclaw/pull/83267 after review blockers are addressed, that accepts timeout-only overlays for bundled providers while keeping custom-provider declarations, exported types, docs, and regression tests aligned.

## Reproduction Assessment

Yes. Source inspection shows a `models.providers.openai` object containing only `timeoutSeconds` reaches `ModelProviderSchema`, where `baseUrl` and `models` are still required; I did not run a gateway restart in this read-only review.

## Solution Assessment

Yes, accepting bundled-provider overlays is the best direction if the implementation derives or covers real bundled provider IDs and keeps exported config types honest. The fallback is to document that the field only works inside full custom-provider declarations, but that would remove the documented workaround rather than fixing it.
