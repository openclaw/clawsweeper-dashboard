## What This Changes

The issue asks OpenClaw to make `models.providers.<id>.timeoutSeconds` work as the documented standalone per-provider timeout for built-in providers, or to correct the schema/docs if that is not supported.

## Best Possible Solution

Accept narrow timeout-only overlays for known built-in providers while preserving strict custom-provider validation, then update schema/help/docs and add focused regression coverage.

## Reproduction Assessment

Yes by source inspection: a `models.providers.openai` object with only `timeoutSeconds` enters `ModelProviderSchema`, where `baseUrl` and `models` are required before runtime provider overlay resolution can use the timeout.

## Solution Assessment

Yes: aligning validation with the already documented and runtime-consumed provider timeout overlay is the narrow maintainable fix, provided custom provider declarations remain strict.
