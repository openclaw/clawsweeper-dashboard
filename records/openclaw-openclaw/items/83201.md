The issue asks OpenClaw to make `models.providers.<built-in>.timeoutSeconds` usable without redeclaring `baseUrl` and `models`, or to correct the docs/schema if that config shape is not supported.

## Best Possible Solution

Land or revise the existing fixing PR so built-in provider timeout overlays validate while unknown custom providers still require `baseUrl` and `models`, with regression coverage for validation and model resolution.

## Solution Assessment
