## What This Changes

The issue asks OpenClaw to preserve approval-time channel routing fields when exec approval followup messages resume an agent session after `/approve`.

## Best Possible Solution

Keep this as a focused core bug fix: preserve the full approval-time route through exec followup session resume for external channel plugins, with regression coverage for that fallback path.

## Reproduction Assessment

Yes, at source level: an exec followup with a session key, an external channel id that is not considered deliverable/gateway in this process, and `turnSourceTo` set will pass the raw channel but drop `to` and `accountId`. I did not run a live Lansenger setup.

## Solution Assessment

No current-main solution is sufficient yet. The narrow fix should keep the full approval-time route together through `sendExecApprovalFollowup` and the gateway `agent` resume path instead of relying on webchat or session history fallback.
