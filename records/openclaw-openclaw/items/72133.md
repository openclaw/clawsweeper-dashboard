## Summary

Keep open. Current main still limits the rich token/cache/cost/model/context metadata to the Control UI; channel footers remain partial, Swift chat decodes usage without rendering it, and Android chat still drops usage during history parsing.

## What This Changes

Request to show the Control UI's per-message token/cache/cost/model/context metadata in native iOS/Android chat and optional channel reply footers.

## Best Possible Solution

Add a shared compact assistant-usage presenter fed by existing run/Gateway metadata, use it for responseUsage footers and assistant-only native mobile footers, and keep cost/context display gated by reliable metadata.

## Reproduction Assessment

Not applicable as a feature request, but yes for source verification: Control UI renders the rich metadata while channel and mobile code paths remain missing or partial on current main.

## Solution Assessment

Yes. Extending the existing responseUsage and mobile chat renderers with a shared formatter is the narrowest maintainable direction; per-channel bespoke calculations would be riskier.
