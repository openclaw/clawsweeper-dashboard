## Summary

Keep open. Current main still does not satisfy the requested cross-surface visibility: the rich metadata exists in Control UI, channel replies expose only a partial optional usage footer, Swift decodes usage without rendering it, and Android does not retain usage metadata when parsing chat history.

## What This Changes

Request per-message token, cache, cost, model, and context-window metadata in native mobile chat views and optional channel reply footers.

## Best Possible Solution

Add one shared compact usage presentation for assistant replies, reuse it from the existing responseUsage channel footer and native Swift/Kotlin chat renderers, preserve Android usage metadata during history parsing, and gate model/context percentage on reliable model plus prompt-context metadata.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a feature request, but the gap is high-confidence by source inspection across Control UI, channel footer, Swift chat, and Android chat history parsing.

## Solution Assessment

Is this the best way to solve the issue? Yes: extending the existing /usage responseUsage footer and mobile chat renderers is the narrowest maintainable path, with context percentage derived from existing prompt/context data instead of inventing per-channel implementations.
