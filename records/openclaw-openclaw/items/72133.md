## Summary

Keep open. Current main still confines the rich per-message token/cache/cost/model/context display to Control UI; channel footers remain token/cache/cost-only, Swift mobile decodes usage without rendering it, and Android chat drops usage metadata entirely.

## What This Changes

Request to show Control UI-style per-assistant-message token/cache/cost/model/context metadata in native mobile chat and optional channel reply footers.

## Best Possible Solution

Add a shared compact assistant-usage presenter and wire it through channel responseUsage plus native mobile chat, preserving existing cost visibility rules and showing context percentage only when reliable context-window data is available.

## Reproduction Assessment

Not applicable as a feature request, but yes for source verification: current main shows the rich metadata in Control UI while the channel and native mobile paths are missing or partial.

## Solution Assessment

Yes. Extending the existing responseUsage and native chat render paths with a shared formatter is the narrowest maintainable direction; per-surface bespoke calculations would be harder to keep consistent.
