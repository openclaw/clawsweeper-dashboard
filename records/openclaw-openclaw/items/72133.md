## Summary

Keep open. Current main still renders the rich token/cache/cost/model/context metadata only in the Control UI; channel `responseUsage` footers remain token/cache/cost-only, Swift mobile chat decodes usage without rendering it, and Android chat drops usage during parsing.

## What This Changes

Expose the Control UI's per-assistant-message token/cache/cost/model/context metadata in native mobile chat and optional channel reply footers.

## Best Possible Solution

Add a shared compact assistant-usage presenter, wire it into channel responseUsage and native mobile chat, hide cost unless existing auth/pricing rules allow it, and compute context percentage only from reliable prompt/context metadata.

## Reproduction Assessment

Not applicable as a feature request, but source inspection gives a high-confidence verification path: the web UI renders the rich metadata, while channel and mobile code paths remain missing or partial.

## Solution Assessment

Yes. Extending the existing responseUsage footer and mobile chat renderers is the narrowest maintainable path; it should avoid per-channel bespoke calculations and reuse the existing Gateway usage metadata.
