## Summary

Keep open: current main now covers much of the channel visibility request through `/usage full` and `messages.usageTemplate`, but native iOS/Android chat still does not render per-message usage/context metadata, so the remaining work needs maintainer UX/product direction rather than stale cleanup.

## What This Changes

Request to show Control UI-style per-message token, cost, context, and model metadata in native iOS/Android chat and optional channel reply footers.

## Best Possible Solution

Reuse the existing normalized usage and `/usage full` contract, add compact iOS/Android per-message footers after the UX/defaults are chosen, and keep any channel changes aligned with `messages.usageTemplate` rather than adding a parallel footer system.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence current-main verification path: iOS decodes usage but does not render it, and Android chat drops usage before rendering.

## Solution Assessment

Unclear until maintainers choose the UX/default behavior. The maintainable direction is to reuse existing `responseUsage` and usage-template plumbing rather than create another channel/mobile metadata system.
