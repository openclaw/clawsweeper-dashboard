## Summary

Keep open. Current main still has the requested rich per-message token/cache/cost/model/context display only in the Control UI, while channel response footers omit model and context percentage and native mobile chat surfaces do not render the decoded usage metadata. The existing ClawSweeper comment correctly identified the same gap, and current source at e0cc374b confirms it remains actionable.

## What This Changes

Request to extend the Control UI per-message token, cache, cost, model, and context-fill metadata to native iOS/Android chat views and optional channel reply footers.

## Best Possible Solution

Add a shared compact usage presentation for assistant replies, reuse it from the existing responseUsage channel footer and native Swift/Kotlin chat renderers, preserve Android usage metadata through history parsing, and show model/context percentage only when reliable model and prompt-context metadata are available.

## Reproduction Assessment

Not applicable as a feature request, but the gap is high-confidence by source inspection. Control UI renders the metadata, channel usage formatting omits model/context, Swift does not render decoded usage, and Android does not retain/render usage metadata.

## Solution Assessment

Yes. Extending the existing /usage responseUsage footer plus the native chat renderers is the narrowest maintainable path; context percentage should be gated on reliable run or history metadata instead of adding per-channel bespoke calculations.
