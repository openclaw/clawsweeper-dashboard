## Summary

Keep open. Current main still limits the rich per-message token/cache/cost/model/context metadata to the Control UI; channel footers remain partial, Swift mobile decodes usage without rendering it, and Android chat does not carry usage metadata.

## What This Changes

Request per-assistant-message token, cost, model, cache, and context-fill metadata in native mobile chat and optional messaging-channel reply footers.

## Best Possible Solution

Add one shared compact assistant-usage presenter and wire it into channel responseUsage plus native mobile chat, preserving existing cost visibility rules and computing context percent only from reliable prompt/context metadata.

## Reproduction Assessment

Not applicable as a feature request, but yes for source verification: current main shows the rich metadata in Control UI while the channel and mobile paths are missing or partial.

## Solution Assessment

Yes. Extending the existing responseUsage and native chat render paths with a shared formatter is the narrowest maintainable direction; per-channel bespoke calculations would be harder to keep consistent.
