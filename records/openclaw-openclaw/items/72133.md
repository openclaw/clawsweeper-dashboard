## Summary

Keep open. Current main still only partially covers the request: Control UI has the rich metadata, channel replies expose a narrower opt-in usage footer, Swift chat decodes usage without rendering a footer, and Android chat does not preserve usage/model metadata.

## What This Changes

Request to surface Control UI-style per-message token, cost, model, and context metadata in native mobile chat and messaging-channel replies.

## Best Possible Solution

Define a shared compact usage metadata contract/presenter, extend existing responseUsage/footer plumbing for channels, and render the same decoded usage/model/context data in Swift and Android with explicit privacy defaults.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence verification path: Control UI has the metadata, while channel and native mobile surfaces remain partial or missing.

## Solution Assessment

Unclear until maintainers choose the cross-surface UX and privacy defaults. The narrow maintainable path is to extend existing responseUsage and mobile transcript usage plumbing rather than adding separate per-channel calculations.
