## Summary

Keep open. Current main still has the complete per-message token/cache/cost/model/context display only in the Control UI; channel replies have an opt-in usage footer but omit model and context percentage, Swift chat models decode usage without rendering it, and Android chat parsing drops usage metadata entirely.

## What This Changes

Request to surface per-message token, cost, model, and context metadata in native mobile chat views and optional channel reply footers.

## Best Possible Solution

Add a shared compact usage presentation for assistant replies, reuse it from the existing responseUsage channel footer and native Swift/Kotlin chat renderers, preserve Android usage metadata during history parsing, and show model/context percentage only when reliable model and prompt-context metadata are available.

## Reproduction Assessment

Not applicable as a feature request, but the gap is high-confidence by source inspection: Control UI renders the metadata, channel usage formatting omits model/context, Swift does not render decoded usage, and Android does not retain usage.

## Solution Assessment

Yes. Extending the existing /usage responseUsage footer and native chat renderers is the narrowest maintainable path, with context percentage gated on reliable run or history metadata instead of per-channel bespoke implementations.
