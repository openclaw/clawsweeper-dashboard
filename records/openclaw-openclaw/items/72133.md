## Summary

Keep open. Current main still only has the full per-message token/cache/cost/model/context metadata in Control UI; channel replies have a partial opt-in usage footer, and native Swift/Android chat views do not render the requested footer.

## What This Changes

Request to surface Control UI-style per-message token, cost, model, and context metadata in native mobile chat and optionally in channel reply footers.

## Best Possible Solution

Define a shared compact metadata presenter and feed it from existing usage/model/context data in channel footers plus Swift and Android chat, preserving opt-in channel behavior and cost visibility rules.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence verification path: Control UI renders the metadata, while channel and native mobile surfaces remain partial or missing.

## Solution Assessment

Unclear until maintainers choose the cross-surface UX and privacy defaults. The narrowest maintainable path is to extend the existing responseUsage/native chat plumbing rather than adding per-channel calculations.
