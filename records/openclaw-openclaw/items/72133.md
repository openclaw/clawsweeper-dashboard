## Summary

Keep open. Current main still has the rich Control UI metadata, and there is partial usage plumbing in native Swift, but channel footers and native mobile chat views do not yet expose the requested per-message context/model/cost footer.

## What This Changes

Request to show Control UI-style per-message token, cache, cost, context-percentage, and model metadata in native mobile chat views and optional channel reply footers.

## Best Possible Solution

Keep the channel-footer work aligned with https://github.com/openclaw/openclaw/issues/40215 and https://github.com/openclaw/openclaw/pull/69618, then add a shared compact metadata presenter and native Swift/Android plumbing once maintainers choose the cross-surface UX defaults.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence verification path: Control UI renders the requested metadata, while channel footers and native mobile rendering remain partial or missing.

## Solution Assessment

Unclear until maintainers choose the UX and privacy defaults. The narrowest maintainable direction is to reuse existing usage/model/context plumbing and the responseUsage mode, while adding native mobile presenter support instead of per-channel bespoke calculations.
