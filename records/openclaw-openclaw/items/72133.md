## Summary

Keep open. Current main still provides the rich per-message token/cache/cost/model/context metadata only in the Control UI; channel footers remain partial, Swift mobile chat decodes usage without rendering it, and Android chat still drops usage while parsing history.

## What This Changes

Request to show the Control UI's per-message token/cache/cost/model/context metadata in native iOS/Android chat and optional channel reply footers.

## Best Possible Solution

Add a shared compact assistant-usage presenter fed by existing run/Gateway metadata, render it behind existing `/usage` modes for channel replies and as an assistant-only native mobile footer, and derive context percent only when prompt/context data is reliable.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence verification path: the web UI renders the rich metadata, while channel and mobile code paths remain missing or partial on current main.

## Solution Assessment

Yes. A narrow shared formatter plus mobile renderers is the maintainable path; per-channel bespoke calculations or exposing cost without the current auth gate would be the less safe direction.
