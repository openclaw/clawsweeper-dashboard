## Summary

Keep open: current main partly covers channel reply footers through `/usage full` and usage templates, but native iOS/Android chat still does not render per-message usage/context metadata and the remaining default/visibility behavior needs maintainer scoping.

## What This Changes

Request to expose Control UI per-message token, cost, context, cache, and model metadata in native iOS/Android chat and optional channel reply footers.

## Best Possible Solution

Reuse the existing `responseUsage` and usage-template contract, define a compact native iOS/Android footer UX, and align any remaining channel behavior with `/usage full` instead of adding a parallel metadata system.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes at the source level: channel `/usage` support exists, iOS decodes usage without rendering it, and Android chat drops usage before rendering; this is a feature gap rather than a failing bug reproduction.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose native footer placement and visibility defaults; the maintainable direction is to reuse existing `responseUsage` and `messages.usageTemplate` plumbing rather than create a second metadata system.
