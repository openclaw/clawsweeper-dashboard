## Summary

Keep open. Current main still implements the rich token/cache/cost/model/context display only in the Control UI; channel footers remain a simpler `/usage` line and native mobile chat either does not render decoded usage or drops it during parsing. The earlier ClawSweeper comment matches the current source, so this remains actionable.

## What This Changes

Request to show Control UI-style per-message token, cache, cost, model, and context metadata in iOS/Android chat and optionally in channel reply footers.

## Best Possible Solution

Add a shared compact usage presentation and render assistant-only footers in channel `responseUsage` replies plus native mobile chat, with model/context shown only when reliable and cost hidden where auth/pricing is unavailable.

## Reproduction Assessment

Not applicable as a feature request, but source inspection gives a high-confidence verification path. The Control UI renders and tests rich metadata, while channel and native mobile code paths remain missing or partial.

## Solution Assessment

Yes. Extending the existing `/usage` footer and native chat renderers is the narrowest maintainable solution; it should avoid per-channel bespoke calculations and gate context percentage on existing prompt/context metadata.
