## Summary

Keep open. Current main still keeps the rich per-message token/cache/cost/model/context metadata in the Control UI; channel footers are partial, Swift mobile decodes usage without rendering it, and Android chat drops usage metadata.

## What This Changes

The issue requests Control UI-style per-message usage metadata in native mobile chat and optional channel reply footers.

## Best Possible Solution

Add a compact usage metadata presenter for assistant replies and feed it from existing usage/model/context fields across channel, Swift, and Android render paths, with regression coverage and a changelog entry.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence verification path: Control UI renders the requested metadata while channel and native mobile paths are missing or partial.

## Solution Assessment

Yes. Reusing the existing responseUsage mode and mobile chat message plumbing is the narrow maintainable path; per-channel bespoke calculations would be harder to keep consistent.

## Repair Work Prompt

Implement https://github.com/openclaw/openclaw/issues/72133. Add per-message assistant usage metadata to native mobile chat and richer optional channel reply footers. Reuse the Control UI semantics where practical: input/output tokens, cache read/write, estimated cost only when the current auth/cost rules allow it, model label, and context percent only when prompt/context-window data is reliable. Extend src/auto-reply/reply/agent-runner-usage-line.ts and the runReplyAgent responseUsage call rather than adding per-channel bespoke formatters. Use the existing responseUsage modes for channel toggling unless a new setting is demonstrably required. Add a compact Swift renderer for OpenClawChatMessage.usage, and extend Android ChatMessage parsing/model/rendering to carry and display usage. Add focused tests and a user-facing CHANGELOG.md entry. Do not change provider pricing/catalog contracts, do not expose estimated cost for OAuth-only auth, and do not touch unrelated channel behavior.
