## Summary

Keep open. Current main still only partially satisfies the request: Control UI has the rich metadata, channel replies have a narrower opt-in usage footer, Swift chat carries usage without rendering it, and Android chat does not carry usage metadata.

## What This Changes

The issue requests Control UI-style per-message token, cost, model, cache, and context metadata in native mobile chat and optional channel reply footers.

## Best Possible Solution

Define a shared compact metadata presenter/data contract and reuse it from Control UI, channel responseUsage, Swift chat, and Android chat with existing cost privacy rules and graceful omissions.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence verification path: Control UI has the metadata, while channel and native mobile surfaces remain partial or missing.

## Solution Assessment

Unclear until maintainers choose the cross-surface UX and privacy defaults. The narrow maintainable path is to extend existing responseUsage and transcript usage plumbing rather than adding separate per-channel calculations.
