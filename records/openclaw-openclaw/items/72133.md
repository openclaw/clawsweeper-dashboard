## Summary

Keep open. Current main still limits the full per-message token/cache/cost/model/context display to the Control UI; channel footers expose only partial usage data, Swift decodes usage without rendering it, and Android drops usage during chat history parsing.

## What This Changes

Request to surface Control UI-style per-message token, cache, cost, model, and context metadata in native mobile chat and optional channel reply footers.

## Best Possible Solution

Add a compact shared usage presentation for assistant replies, reuse it from the existing responseUsage channel footer and native chat renderers, and gate model/context percentage display on reliable metadata instead of channel-specific hacks.

## Reproduction Assessment

Not applicable as a feature request, but the gap is high-confidence by source inspection: Control UI renders the metadata, channel usage formatting omits model/context, Swift does not render decoded usage, and Android does not retain usage.

## Solution Assessment

Yes. Extending the existing /usage responseUsage path plus the native Swift/Kotlin chat renderers is the narrowest maintainable solution; a new core API does not appear necessary.
