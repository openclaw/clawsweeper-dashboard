## Summary

Keep open. Current main still implements the rich per-message metadata in Control UI, but channel replies only have a partial `/usage` footer and the native Swift/Android chat views do not render the requested usage/context footer.

## What This Changes

Request per-assistant-message token, cache, cost, model, and context-fill metadata in native mobile chat and channel reply surfaces.

## Best Possible Solution

Define one compact metadata presenter and feed it from existing usage/model/context data across channel footer, Swift chat, and Android chat, with graceful omissions for unavailable cost or context fields.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence verification path: Control UI renders the metadata, while channel and native mobile paths remain partial or missing.

## Solution Assessment

Unclear until maintainers choose the cross-surface UX. The narrow technical path is to reuse existing usage/model/context plumbing and the current responseUsage mode instead of adding per-channel calculations.
