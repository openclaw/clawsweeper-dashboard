## Summary

Keep open. Current main still has the rich metadata in Control UI, but channel replies only expose the existing token/cost `/usage` footer, Swift chat decodes usage without rendering a message footer, and Android chat has no message usage fields; this is a cross-surface feature request that needs maintainer UX/privacy judgment.

## What This Changes

Request to show per-message token, cache, cost, context, and model metadata in iOS/Android chat and optional channel reply footers.

## Best Possible Solution

Define a shared compact metadata presenter/data contract and reuse it from Control UI, channel `responseUsage`, Swift chat, and Android chat with existing cost privacy rules and graceful omissions.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence verification path: Control UI has the metadata, while channel and native mobile surfaces remain partial or missing.

## Solution Assessment

Unclear until maintainers choose the cross-surface UX. The narrow maintainable path is to extend existing `responseUsage` and transcript usage plumbing rather than adding separate per-channel calculations.
