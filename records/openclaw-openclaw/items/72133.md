## Summary

Keep open. Current main still limits the rich token/cache/cost/model/context display to the Control UI; channel replies have a simpler token/cache/cost footer, Swift mobile decodes usage without rendering it, and Android chat drops usage while parsing history.

## What This Changes

Request to show Control UI-style per-message token/cache/cost/model/context metadata in native mobile chat and optional channel reply footers.

## Best Possible Solution

Add a shared compact assistant usage footer for channels and mobile that shows input/output/cache/model/context percent when reliable, hides cost without API-key/pricing, and uses the existing /usage toggle for channel replies.

## Reproduction Assessment

Not applicable as a runtime bug, but source inspection gives a high-confidence verification path. The web renderer and tests show the rich metadata path, while the channel formatter and mobile parsers/renderers show the missing or partial surfaces.

## Solution Assessment

Yes. Reusing the existing responseUsage mode and native chat renderers is the narrowest maintainable direction; add only additive payload fields if model/context data is not already available to mobile clients.
