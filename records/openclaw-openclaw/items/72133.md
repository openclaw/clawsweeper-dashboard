## Summary

Keep open. Current main still has the complete per-message token/cache/cost/model/context display only in the Control UI; channel replies have an opt-in usage footer but still omit model and context percentage, iOS decodes and preserves usage without rendering it, and Android chat messages still drop usage metadata entirely.

## What This Changes

Request to surface Control UI-style per-message token, cache, cost, model, and context metadata in native mobile chat and channel replies.

## Best Possible Solution

Add a shared compact usage presentation for assistant replies, reuse it from the existing responseUsage channel footer and the native Swift/Kotlin chat renderers, preserve Android usage metadata during history parsing, and show model/context percentage only when reliable metadata is available.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a feature request, but the implementation gap is high-confidence by source inspection across the Control UI, channel footer, iOS/Swift chat, and Android chat paths.

## Solution Assessment

Is this the best way to solve the issue? Yes: extending the existing /usage footer and native chat message renderers is the narrowest maintainable path, with context percentage gated on trustworthy run or history metadata.
