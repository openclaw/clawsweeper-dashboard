## Summary

Keep open: current main now covers the channel footer/default portion through `messages.responseUsage` and the `/usage full` usage-bar contract, but the native mobile per-message rendering request is still not implemented in iOS or Android chat bubbles.

## What This Changes

Request to show Control UI-style per-message token, cost, model, cache, and context metadata in native mobile chat and optional channel reply footers.

## Best Possible Solution

Keep the current channel footer contract and add a compact native iOS/Android assistant-message metadata view that reuses existing message usage/session context data without creating another footer configuration surface.

## Reproduction Assessment

Yes, at source level. Current main shows channel footer/default support, but iOS chat bubbles ignore decoded message usage and Android chat messages do not carry per-message usage into the bubble renderer.

## Solution Assessment

Unclear until maintainers choose the mobile UX. The channel side should stay on the current `messages.responseUsage`/usage-template path, while the remaining mobile work should render compact metadata from existing usage/session data.
