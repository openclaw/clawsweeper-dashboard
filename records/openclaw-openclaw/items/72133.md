## Summary

Keep open. Current main still only has the complete per-message token/cache/cost/model/context display in the Control UI; channel replies have an opt-in usage footer but not model/context percentage, iOS decodes and preserves usage without rendering it, and Android drops usage metadata while parsing chat history.

## What This Changes

Request to expose Control UI-style per-message token, cost, cache, model, and context-window metadata in native mobile chat views and optional channel reply footers.

## Best Possible Solution

Add a shared compact usage presentation for assistant replies, reuse it from the existing responseUsage channel footer and the Swift/Kotlin chat renderers, preserve Android usage metadata during history parsing, and show model/context percentage only when reliable model and prompt-context metadata are available.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a feature request, but the gap is high-confidence by source inspection: Control UI renders the metadata, channel usage formatting omits model/context, iOS does not render decoded usage, and Android does not retain usage.

## Solution Assessment

Is this the best way to solve the issue? Yes: extending the existing /usage responseUsage footer and native chat renderers is the narrowest maintainable path, with context percentage gated on reliable run or history metadata instead of inventing per-channel implementations.
