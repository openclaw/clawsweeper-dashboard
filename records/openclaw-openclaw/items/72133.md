## Summary

Keep open. Current main still only has the full per-message token/cache/cost/model/context display in the Control UI; the channel footer is opt-in but partial, Swift decodes usage without rendering it, and Android chat history still drops usage metadata before the UI can show it.

## What This Changes

Request to show per-message token/cost/cache/model/context metadata in native mobile chat views and optional messaging-channel reply footers.

## Best Possible Solution

Add a shared compact usage presentation for assistant replies, extend the existing /usage responseUsage footer with reliable model/context snapshot inputs, reuse the formatting in Swift and Android chat renderers, preserve Android usage during history parsing, and update docs/changelog for the new user-visible surfaces.

## Reproduction Assessment

Not applicable as a feature request, but the gap is high-confidence by source inspection: Control UI renders the metadata, channel usage formatting omits model/context, Swift does not render decoded usage, and Android does not retain usage.

## Solution Assessment

Yes. Extending the existing responseUsage footer and native chat renderers is the narrowest maintainable solution; context percentage should be gated on reliable prompt/context-window metadata rather than inferred from provider totals.
