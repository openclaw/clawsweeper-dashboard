## Summary

Keep open. Current main still only partially satisfies the request: Control UI has the rich message metadata, channel replies have an opt-in usage footer without model/context percent, Swift decodes usage without rendering a footer, and Android chat does not carry usage metadata.

## What This Changes

The issue requests per-message token, cost, cache, model, and context metadata in native mobile chat views and optional channel reply footers.

## Best Possible Solution

Define a shared compact metadata presenter/data contract and reuse it from Control UI, channel responseUsage, Swift chat, and Android chat with existing cost privacy rules and graceful omissions.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence verification path: Control UI has the metadata, while channels and native mobile surfaces remain partial or missing.

## Solution Assessment

Unclear until maintainers choose the cross-surface UX and privacy defaults. The narrow maintainable path is to extend existing responseUsage and transcript usage plumbing rather than adding separate per-channel calculations.
