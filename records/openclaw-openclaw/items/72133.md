## Summary

Keep open. Current main still only fully exposes the rich per-message token/cache/cost/model/context display in the Control UI; channel replies have an opt-in usage footer but it omits model and context percentage, Swift mobile decodes usage without rendering it, and Android does not retain usage metadata in its chat model.

## What This Changes

Request per-assistant-message token, cache, cost, model, and context-fill metadata in native mobile chat views and optional channel reply footers.

## Best Possible Solution

Add a shared compact usage presentation that can be reused by the responseUsage footer and native chat renderers, preserve Android usage metadata through history parsing, render Swift/Android assistant usage footers, and show model/context percentage only when reliable metadata is available.

## Reproduction Assessment

Not applicable as a feature request, but the gap is high-confidence by source inspection. The Control UI renders the requested metadata, while channel footers omit model/context and native mobile chat surfaces do not render the decoded usage metadata.

## Solution Assessment

Yes. Extending the existing opt-in responseUsage footer and native chat renderers is the narrowest maintainable path; a separate per-channel calculation layer would duplicate session/runtime metadata handling.
