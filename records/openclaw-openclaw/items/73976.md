## Summary

Keep this PR open, but not as-is: current main still lacks provider-backed per-keyword hybrid FTS fallback, while the PR currently regresses no-provider FTS-only recall by sharing the new six-term cap with that path.

## What This Changes

The PR adds bounded per-keyword FTS fallback for hybrid memory search, re-enables hybrid keyword tests, adds a fan-out bound test, and updates the changelog.

## Best Possible Solution

Keep the bounded provider-backed hybrid fallback, preserve the existing no-provider FTS-only recall behavior, and add regression coverage for that boundary before landing.

## Reproduction Assessment

Yes, by source inspection. Current main searches every extracted keyword in no-provider FTS-only mode, while PR head routes that path through a helper that slices fallback terms to six, so a no-provider query whose only indexed hit is term seven or later would lose recall.

## Solution Assessment

No, not as-is. The helper and cap are a good shape for provider-backed hybrid fan-out, but FTS-only mode should keep an uncapped fallback unless maintainers intentionally choose to reduce recall and document/test that change.
