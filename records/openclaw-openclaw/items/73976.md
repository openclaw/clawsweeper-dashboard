## Summary

Keep open. Current main still lacks the provider-backed per-keyword FTS fallback this PR is trying to add, but the patch still needs a narrow repair because it also routes no-provider FTS-only search through the capped helper and can reduce recall compared with current main.

## What This Changes

The PR adds a capped per-keyword FTS fallback for memory-core hybrid search, re-enables hybrid keyword tests, adds a fan-out bound test, and updates the changelog.

## Best Possible Solution

Keep the bounded provider-backed hybrid FTS fallback, preserve current no-provider FTS-only recall, add regression coverage for that boundary, then rebase and validate the PR.

## Reproduction Assessment

Yes, by source inspection. Current main searches every extracted keyword in no-provider FTS-only mode, while PR head routes that path through a helper that slices fallback terms to six, so a no-provider query whose only indexed hit is later than the cap can be lost.

## Solution Assessment

No, not as-is. The helper and cap are a good shape for provider-backed hybrid fan-out, but the no-provider FTS-only path should keep its uncapped fallback unless maintainers intentionally choose a recall reduction and add coverage for it.
