## Summary

Keep open. Current main still lacks provider-backed hybrid per-keyword FTS fallback, and this PR is the canonical replacement for #39555, but the branch still caps FTS-only fallback recall and needs a small repair before merge.

## What This Changes

The PR adds a shared bounded per-keyword FTS fallback for memory search, re-enables two hybrid keyword tests, adds a fan-out bound test, and adds a changelog entry.

## Best Possible Solution

Land a rebased version that caps only provider-backed hybrid FTS fan-out while preserving FTS-only all-keyword recall, current failure diagnostics, changelog credit, and focused tests.

## Reproduction Assessment

Yes. In no-provider mode, index content matching only the seventh-or-later extracted term in a long query; current main's all-keyword FTS-only fallback can find it, while this PR's six-term shared helper drops it.

## Solution Assessment

No, not as-is. The helper is a good shape for bounded hybrid fallback, but sharing its cap with no-provider FTS-only mode is not the narrowest maintainable fix.
