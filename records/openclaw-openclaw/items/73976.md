## Summary

Keep open. Current main still lacks the provider-backed hybrid per-keyword FTS fallback this PR targets, but the PR applies the new six-term cap to the no-provider FTS-only fallback path and can drop existing lexical recall for long queries.

## What This Changes

The PR adds a bounded per-keyword FTS fallback for provider-backed hybrid memory search, re-enables hybrid keyword tests, adds a fan-out bound test, and records the fix in the changelog.

## Best Possible Solution

Keep the bounded provider-backed hybrid fallback, but preserve or explicitly test the no-provider FTS-only recall behavior before merging.

## Reproduction Assessment

Yes. Force the embedding provider unavailable, index content matching only the seventh-or-later extracted keyword in a long query, and compare current main's all-keyword FTS-only fallback with the PR's six-term cap.

## Solution Assessment

No, not as-is. The shared helper is a good narrow shape for hybrid fallback, but applying its cap to FTS-only mode changes an existing lexical-only recovery path without coverage.
