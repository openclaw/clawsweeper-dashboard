## Summary

Keep open. The PR addresses a real hybrid memory-search recall gap, but the current diff also routes FTS-only no-provider search through the new capped fallback helper, which can drop lexical matches after the sixth extracted term where current main still searches every extracted keyword.

## What This Changes

The PR adds bounded per-keyword FTS fallback for provider-backed hybrid memory search, re-enables related memory-core tests, adds a fan-out bound regression test, and updates the changelog.

## Best Possible Solution

Keep the capped per-keyword fallback for provider-backed hybrid search, while preserving or explicitly replacing the existing FTS-only no-provider all-keyword recall behavior with regression coverage.

## Reproduction Assessment

Yes. Force the embedding provider unavailable, index content matching only the seventh-or-later extracted keyword in a long query whose full AND query misses, then compare current main's all-keyword FTS-only fallback with this PR's six-term cap.

## Solution Assessment

No, not as-is. The helper extraction is a maintainable shape for hybrid mode, but applying the same cap to FTS-only search changes an existing lexical recovery path without coverage.
