## Summary

Keep open. The PR is still useful source work for the hybrid memory-search bug, but the current diff introduces a blocking FTS-only recall regression by applying the new six-term fallback cap to the no-provider path.

## What This Changes

The PR adds a bounded per-keyword FTS fallback helper for memory search, re-enables hybrid keyword regression tests, adds a fan-out bound test, and updates the changelog.

## Best Possible Solution

Keep the bounded provider-backed hybrid fallback, but preserve no-provider FTS-only all-keyword recall or add an explicit, tested FTS-only fallback strategy before merging.

## Reproduction Assessment

Yes. Force the embedding provider unavailable, index content that matches only the seventh-or-later extracted keyword in a long query, and compare current main's all-keyword FTS-only fallback with the PR's six-term cap.

## Solution Assessment

No, not as-is. The helper is a good narrow shape for hybrid fan-out, but sharing it with the FTS-only branch applies a hybrid performance cap to a path with no vector fallback.
