## Summary

Keep open: current main still lacks the requested provider-backed hybrid per-keyword FTS fallback, but this PR is not mergeable as-is because it also caps the no-provider FTS-only fallback and would lose existing recall for later keywords.

## What This Changes

The PR adds a bounded per-keyword FTS fallback helper for memory-core hybrid search, re-enables hybrid keyword tests, adds a fan-out bound test, and records the fix in the changelog.

## Best Possible Solution

Keep the bounded provider-backed hybrid fallback, preserve the existing uncapped FTS-only fallback recall path, and add regression coverage for a no-provider long query whose only hit is after the hybrid cap.

## Reproduction Assessment

Yes. Source comparison shows a no-provider FTS-only query that misses the full AND query currently searches all extracted keywords, while this PR caps that same path at six through searchKeywordWithFallback; a match only on a seventh-or-later keyword would be lost.

## Solution Assessment

No, not as-is. The shared helper is a good shape for provider-backed hybrid fan-out, but the cap should stay out of the no-provider FTS-only path unless maintainers intentionally choose to reduce recall and document that change.
