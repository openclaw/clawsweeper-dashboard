## Summary

Keep open. The PR fixes the current-main provider-backed hybrid recall gap, but it also routes FTS-only search through the new six-term fallback cap, which can drop later keyword hits in the no-provider path that current main still searches.

## What This Changes

The PR refactors memory-core keyword fallback into a capped helper shared by FTS-only and provider-backed hybrid search, re-enables/adds regression tests, and adds a changelog entry.

## Best Possible Solution

Keep the bounded provider-backed hybrid fallback, but preserve or explicitly test the no-provider FTS-only recall behavior before merging.

## Reproduction Assessment

Yes. Force the embedding provider unavailable, index content matching only the seventh-or-later keyword in a long query, and compare current main's all-keyword FTS-only fallback with the PR's six-term cap.

## Solution Assessment

No, not as-is. The shared helper is a good narrow shape for hybrid fallback, but applying its cap to FTS-only search changes an existing lexical-only recovery path without coverage.
