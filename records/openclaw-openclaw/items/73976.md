## Summary

Keep open: current main still lacks the requested provider-backed hybrid per-keyword fallback, and this PR still introduces a blocking FTS-only recall regression by applying the six-term cap to the no-provider path.

## What This Changes

The PR adds bounded per-keyword FTS fallback for provider-backed hybrid memory search, re-enables hybrid keyword tests, adds a fan-out bound test, and records the fix in the changelog.

## Best Possible Solution

Keep the bounded provider-backed hybrid fallback, but preserve the existing FTS-only fallback recall path and add regression coverage for that boundary before merging.

## Reproduction Assessment

Yes. Source comparison shows current main searches every extracted keyword in no-provider FTS-only mode, while the PR routes that path through a helper that slices fallback terms to six; a long query whose only indexed match is the seventh-or-later term would lose recall.

## Solution Assessment

No, not as-is. The helper is a good narrow shape for provider-backed hybrid fan-out, but the cap should stay hybrid-only unless maintainers explicitly choose to reduce FTS-only recall and cover that behavior.
