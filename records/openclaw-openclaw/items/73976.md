## Summary

Keep open: the PR addresses a real hybrid memory-search recall gap, but the patch also applies the new six-term fallback cap to the no-provider FTS-only path, which regresses current-main recall for long lexical queries with matches after the sixth extracted keyword.

## What This Changes

The PR adds a capped per-keyword FTS fallback helper for memory-core hybrid search, re-enables hybrid keyword tests, adds a fallback bound test, and adds a changelog entry.

## Best Possible Solution

Merge a revised patch that keeps the provider-backed hybrid fallback bounded while preserving or explicitly deciding the no-provider FTS-only recall contract, with targeted tests for both paths.

## Reproduction Assessment

Yes. Force the embedding provider unavailable, index content that matches only the seventh-or-later extracted term in a long query, then compare current main's all-keyword FTS-only fallback with the PR's six-term capped helper.

## Solution Assessment

No, not as-is. The helper is a narrow shape for bounded hybrid fallback, but sharing it with FTS-only search changes an existing lexical recovery path that has no vector fallback.
