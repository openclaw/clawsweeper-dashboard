## Summary

Keep open. Current main still lacks the provider-backed per-keyword hybrid FTS fallback, but this PR has a blocking P2 regression because the new six-term cap also changes the no-provider FTS-only recovery path, and the branch currently reports merge conflicts.

## What This Changes

The PR adds a capped per-keyword FTS fallback helper for memory-core hybrid search, re-enables hybrid keyword tests, adds a fan-out bound test, and records the fix in the changelog.

## Best Possible Solution

Keep the bounded provider-backed hybrid per-keyword fallback, preserve existing FTS-only recall behavior or explicitly test the chosen no-provider tradeoff, then refresh the branch before merge.

## Reproduction Assessment

Yes. Force the embedding provider unavailable, index content matching only the seventh-or-later extracted keyword in a long query, and compare current main's all-keyword FTS-only fallback with the PR's six-term cap.

## Solution Assessment

No, not as-is. The shared helper is a good narrow shape for hybrid fallback, but applying its cap to FTS-only search changes an existing lexical-only recovery path without coverage.

## Review Findings

Overall correctness: patch is incorrect

- **[P2] Preserve FTS-only fallback recall:** `extensions/memory-core/src/memory/manager.ts:379-386`
