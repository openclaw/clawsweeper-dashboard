## Summary

Keep open. Current main still has the nested workspace default while docs still describe peer-level workspaces, and open implementation PRs #71941 and #71899 already cover the fix; maintainers should land or supersede one of those rather than closing this issue or queuing duplicate work.

## What This Changes

The issue asks for the interactive `agents add <id>` wizard to prefill a peer-level `~/.openclaw/workspace-<id>` directory instead of nesting under the main workspace.

## Best Possible Solution

Land one narrow existing PR, or an equivalent maintainer patch, that changes the new-agent wizard default to the documented peer-level workspace while preserving explicit per-agent workspaces, existing legacy paths, and intentional custom shared bases, with focused tests and a changelog entry.

## Reproduction Assessment

Yes. The deterministic source path is enough to reproduce: setup can persist `agents.defaults.workspace`, `agents add` uses `resolveAgentWorkspaceDir(cfg, agentId)` for the prompt default, and the resolver joins non-default ids under that fallback.

## Solution Assessment

Yes, if the fix is scoped to the new-agent wizard default or otherwise preserves explicit and custom shared-root workspace configurations. A broad runtime resolver change needs maintainer approval because it can move existing agents that rely on the #59789 fallback contract.
