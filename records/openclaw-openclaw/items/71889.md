## Summary

Keep open. Current main still uses the add-agent resolver output as the interactive workspace prompt default, and that resolver still nests non-default agents under `agents.defaults.workspace` when setup has persisted the main workspace; linked open PRs are the right path rather than closing this before a merge.

## What This Changes

The issue requests changing the interactive `openclaw agents add <id>` wizard default from nested `workspace/<id>` to the documented peer-level `workspace-<id>` layout.

## Best Possible Solution

Land one narrow existing PR, or an equivalent maintainer patch, that changes the new-agent interactive default to a peer-level workspace path while preserving explicit and legacy configured workspace paths; keep runtime migration behavior conservative and add focused wizard/resolver regression coverage plus the required changelog entry.

## Reproduction Assessment

Yes. The source path is deterministic: setup can persist `agents.defaults.workspace`, the add wizard uses `resolveAgentWorkspaceDir(cfg, agentId)` as the prompt default, and the resolver joins that fallback with the non-default agent id.

## Solution Assessment

Yes. The requested peer-level wizard default is the narrow maintainable direction, provided it is scoped to new-agent defaults and does not silently relocate existing agents with nested or explicit workspace paths.
