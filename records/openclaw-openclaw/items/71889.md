## Summary

Keep open. Current main still uses the runtime workspace resolver as the interactive `agents add` prompt default, and that resolver still appends non-default agent ids under `agents.defaults.workspace`; the related open PRs are implementation candidates and should be reviewed or consolidated before this issue closes.

## What This Changes

The issue asks for the interactive `openclaw agents add <id>` wizard to pre-fill a peer-level workspace path like `~/.openclaw/workspace-<id>` instead of nesting under `~/.openclaw/workspace/<id>`.

## Best Possible Solution

Land one narrow canonical fix that changes the new-agent wizard default to a peer-level workspace while preserving explicit and legacy configured workspace paths, with focused resolver/wizard tests and a user-facing changelog entry.

## Reproduction Assessment

Yes. The source path is deterministic on current main: setup persists `agents.defaults.workspace`, `agents add` uses `resolveAgentWorkspaceDir(cfg, agentId)` as the prompt default, and the resolver appends the non-default id under that fallback.

## Solution Assessment

Yes, the peer-level wizard default is the narrow maintainable direction, provided the implementation is scoped so existing configured workspaces and intentional shared-base layouts are not silently moved.
