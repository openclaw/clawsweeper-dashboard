## Summary

Keep open. Current main still seeds the `agents add` wizard from `resolveAgentWorkspaceDir`, which returns `<agents.defaults.workspace>/<id>` for a non-default agent when setup has stored the main workspace there; #71941 is an open, unmerged PR with closing syntax for this issue, so this should remain open until that or an equivalent fix lands.

## What This Changes

The issue asks for `openclaw agents add <id>` to prefill the documented peer-level workspace path `~/.openclaw/workspace-<id>` instead of nesting under the main workspace.

## Best Possible Solution

Land #71941 or an equivalent narrow fix that changes only the new-agent wizard/default suggestion while preserving legacy/custom runtime workspace resolution.

## Reproduction Assessment

Yes. With setup-created `agents.defaults.workspace` set to the main workspace and a new non-default agent id, `resolveAgentWorkspaceDir` returns `<workspace>/<id>` and the wizard uses that exact value as the prompt default.

## Solution Assessment

Yes. A narrow wizard/default-helper fix is the best maintainable path; changing the runtime resolver broadly could relocate existing agents or break intentionally shared workspace-base configurations.
