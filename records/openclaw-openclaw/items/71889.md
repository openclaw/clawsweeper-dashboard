## Summary

Keep open. Current main still has the reported wizard default path behavior, and an open PR with closing syntax plus another overlapping open PR already cover the fix, so this issue should remain open until one implementation lands or is superseded.

## What This Changes

The issue requests that `openclaw agents add <id>` pre-fill the workspace prompt with the documented peer-level `workspace-<id>` path instead of nested `workspace/<id>`.

## Best Possible Solution

Land a narrow wizard/default-helper fix that makes brand-new agents suggest peer-level workspaces while preserving explicit workspaces and the existing runtime shared-base fallback contract.

## Reproduction Assessment

Yes. With setup-created `agents.defaults.workspace` and a new non-default agent id, current main deterministically resolves the wizard default to `<fallback>/<id>` before showing the prompt.

## Solution Assessment

Yes. The safest direction is the PR #71941 style fix scoped to new-agent prompt defaults; broadly changing runtime resolution would risk moving existing agents that rely on the shared-base fallback.
