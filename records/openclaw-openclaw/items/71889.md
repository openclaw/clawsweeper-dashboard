## Summary

Keep open. Current main still seeds the interactive `agents add` workspace prompt from the resolver path that nests non-default agents under `agents.defaults.workspace`, while the docs still describe peer-level `workspace-<agentId>` paths; multiple open PRs already target the fix, so this should not be closed before one lands.

## What This Changes

Fix the interactive `agents add` wizard so the workspace prompt defaults to the documented peer-level `~/.openclaw/workspace-<agentId>` path instead of nesting under the main workspace.

## Best Possible Solution

Land one narrow fix that changes the new-agent interactive default to a peer-level workspace, preserves explicit and legacy configured workspace paths, adds focused resolver/wizard regression coverage and a changelog entry, then let the linked PR close this issue.

## Reproduction Assessment

Yes. The source path is deterministic: setup can persist `agents.defaults.workspace`, the add wizard uses `resolveAgentWorkspaceDir(cfg, agentId)` as the prompt default, and the resolver joins that fallback with the non-default agent id.

## Solution Assessment

Yes. The requested peer-level wizard default is the narrow maintainable direction as long as existing explicit or intentionally shared workspace configurations are not silently relocated.
