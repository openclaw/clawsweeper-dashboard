## Summary

Keep open. Current main still seeds the `agents add` wizard from the runtime workspace resolver, which nests non-default agents under `agents.defaults.workspace`; #71941 has closing syntax for this issue and #71899 also covers the same fix area, so this should remain open until one implementation lands.

## What This Changes

Request that `agents add` pre-fill new agents with peer-level `workspace-<id>` directories instead of nested `workspace/<id>` directories.

## Best Possible Solution

Land #71941 or an equivalent narrow fix that changes the new-agent wizard default while preserving legacy/custom runtime workspace resolution.

## Reproduction Assessment

Yes. With `agents.defaults.workspace` set to the setup-created main workspace and a new non-default agent id, the current resolver returns `<workspace>/<id>` and the wizard uses that value as the prompt default.

## Solution Assessment

Yes. A narrow wizard/default-helper fix is the maintainable path; changing the runtime resolver broadly could relocate existing agents or break intentionally shared workspace-base configurations.
