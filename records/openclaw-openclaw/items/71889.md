## Summary

Keep open. Current `main` still pre-fills the interactive add-agent workspace prompt from the resolver path that nests non-default agents under `agents.defaults.workspace`, while the public docs still describe peer-level `workspace-<agentId>` paths; multiple open PRs are already linked as fix candidates, so this issue should remain open until one lands.

## What This Changes

The issue requests that interactive `openclaw agents add <id>` pre-fill a peer-level `~/.openclaw/workspace-<id>` workspace instead of nesting the new workspace under the main workspace.

## Best Possible Solution

Land one narrow fix from the existing PR set that changes the interactive default for newly added agents to a documented peer-level workspace path, preserves explicit and legacy workspace paths, adds focused wizard/resolver regression coverage, and includes the required user-facing changelog entry.

## Reproduction Assessment

Yes. The source path is deterministic: setup persists `agents.defaults.workspace`, the add wizard uses `resolveAgentWorkspaceDir(cfg, agentId)` as the prompt default, and the resolver joins that fallback with the non-default agent id.

## Solution Assessment

Yes. The requested peer-level wizard default is the narrow maintainable direction, but the safest implementation should avoid silently migrating existing agents that already have explicit or legacy nested workspace paths.
