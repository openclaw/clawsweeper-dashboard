## What This Changes

The issue reports that Telegram context-overflow auto-compaction retries re-submit the same external Telegram message as a fresh user turn, causing duplicate transcript entries and stale replies.

## Best Possible Solution

Land a narrow fix that makes channel-backed overflow-compaction retry continue from the compacted transcript instead of re-submitting the same inbound prompt and external message id, then let the issue close through the linked PR.
