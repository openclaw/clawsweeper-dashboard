## Summary

Keep open: the report is valid against current main, and two open PRs already reference it with closing syntax. Current main still retries the normal assistant-error overflow compaction path with the original prompt/currentMessageId unless the narrower mid-turn precheck flag is present, so the issue should remain open until one of the linked fixes lands.

## What This Changes

Telegram channel sessions can replay the same external inbound message after context-overflow auto-compaction retry, causing stale assistant output after newer messages arrive.

## Best Possible Solution

Land one canonical fix that makes channel-backed compaction retries idempotent for already-submitted external message ids, with regression coverage for assistantError overflow and late stale-output delivery behavior.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to run a channel-backed turn with `currentMessageId`, force an assistantError context overflow after prompt submission, let compaction succeed, and inspect that current main can retry with the same prompt and external message id.

## Solution Assessment

Yes. The narrow maintainable fix is an idempotent retry/continuation guard around compaction retry, with a stale-output fence as defense in depth if newer channel messages arrive before the old run finishes.
