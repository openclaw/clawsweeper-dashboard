## What This Changes

The issue requests a deterministic `sessions_spawn` and/or `agents.defaults.subagents` option to suppress sub-agent completion announcements without relying on exact model-emitted sentinel tokens.

## Reproduction Assessment

Yes at source level. Current main exposes only exact-token/public-output suppression plus an internal completion flag, with no supported `sessions_spawn` or strict-config policy for deterministic suppression.

## Solution Assessment

No as a separate tracker. The best path is to use the older canonical issue for the same suppression policy and coordinate any broader routing or channel policy there and in the related routing issues.
