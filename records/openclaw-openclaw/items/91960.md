## Label Justifications

- `P2`: The missing supported control can repeatedly pollute parent agent context, but its blast radius is limited to sub-agent workflows that require fire-and-forget behavior.
- `impact:session-state`: The reported failure injects unwanted child completion content into the parent transcript and working context.

## Summary

Current main still lacks a supported spawn argument for deterministic suppression: the registry already has a durable internal no-completion-message path, but the public tool schema and documentation expose only model-output sentinel suppression. The issue is therefore still necessary, although the proposed global config surface is not yet justified.

## What This Changes

Requests a spawn-time or default policy that deterministically prevents native sub-agent completion announcements from entering the parent context.

## Best Possible Solution

Add one typed per-spawn completion-announcement policy backed by the existing durable run state, document native and ACP semantics, preserve announcement as the default, and defer any global config default until a separate use case proves it necessary.

## Reproduction Assessment

Yes. Current main provides a high-confidence source reproduction: the supported path relies on exact child-output tokens, while the deterministic internal completion-message gate is absent from the `sessions_spawn` schema.

## Solution Assessment

Yes for a spawn-time policy, but not for adding both a parallel boolean and a global default. The narrowest maintainable solution is one typed public option that reuses the registry’s existing durable delivery state.
