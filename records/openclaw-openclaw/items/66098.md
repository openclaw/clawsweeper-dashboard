## Summary

Keep open: the underlying Web UI/Gateway mismatch is still real on current main, but this PR should not merge as-is because it misses the global reasoning-default precedence, can use stale cached UI state, and only provides test-only proof.

## What This Changes

The PR adds effective thinking/reasoning values to `chat.history`, stores them in Control UI chat state, uses them for the default thinking label and reasoning visibility, and adds focused Gateway/UI tests.

## Best Possible Solution

Keep the current session-row thinking-default implementation and land a rebased or successor fix that exposes effective reasoning defaults through the canonical Gateway/UI contract while matching runtime precedence and avoiding stale history-cache priority.

## Reproduction Assessment

Yes. By source inspection, a fresh Web UI chat session with `agents.defaults.reasoningDefault` or `agents.list[].reasoningDefault` and no persisted `reasoningLevel` still runs with the runtime default while `chat.history` and `renderChat` do not expose or render that effective reasoning state.

## Solution Assessment

No. The PR targets the right bug, but it is not the best current fix because it misses the global reasoning-default contract and lets history-derived UI caches outrank fresher session/model state.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Match history reasoning defaults to runtime:** `src/gateway/server-methods/chat.ts:1664-1665`
  - body: The new `chat.history` reasoning calculation falls back from the persisted row to `resolveAgentConfig(...).reasoningDefault`, but that helper only returns the per-agent value. Runtime also honors `agents.defaults.reasoningDefault`, so fresh sessions using the global default would still report the wrong reasoning state to the UI.
  - confidence: 0.91
- **[P2] Refresh cached thinking defaults after model changes:** `ui/src/ui/app-render.helpers.ts:679-680`
  - body: The default label now prefers `state.chatThinkingLevel`, but that value is refreshed only by `chat.history`. Changing the model through the chat header updates session/default data without reloading history, so the label can keep showing the previous model's default until a later history refresh.
  - confidence: 0.88
- **[P2] Prefer fresh session reasoning over cached history state:** `ui/src/ui/views/chat.ts:1123`
  - body: Reasoning visibility now prioritizes `props.reasoningLevel`, which is loaded from history, ahead of the live session row. A `sessions.patch` or `sessions.changed` update can therefore leave reasoning blocks shown or hidden according to stale history state until history is reloaded.
  - confidence: 0.9
