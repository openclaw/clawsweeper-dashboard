## Summary

## What This Changes

The PR adds effective thinking/reasoning fields to `chat.history`, stores effective reasoning in Control UI chat state, and uses those values for chat header labels and reasoning visibility with gateway/UI tests.

## Best Possible Solution

Land a rebased or successor patch that derives effective reasoning from the shared runtime/default contract, keeps session rows as the freshest UI source after updates, and includes focused Gateway/UI tests plus real Web UI proof.

## Reproduction Assessment

Yes, by source inspection. Configure reasoning defaults, load a fresh Web UI chat without a persisted `reasoningLevel`, and current main omits effective reasoning from `chat.history` while `renderChat` falls back to `off`.

## Solution Assessment

No. The PR targets the right seam, but the current implementation is not the best mergeable solution because its reasoning precedence diverges from runtime and its UI caches can outrank fresher session state.
