## Summary

Keep open: the underlying Web UI/Gateway mismatch is still real on current main, but this PR is not merge-ready because it misses the runtime-equivalent reasoning default path, can show stale cached defaults, and lacks real after-fix behavior proof.

## What This Changes

The PR extends Gateway `chat.history` plus Control UI chat state, rendering, and tests to carry effective thinking/reasoning defaults for fresh sessions.

## Best Possible Solution

Land a rebased or successor fix that returns effective `reasoningLevel` through `chat.history`, matches runtime reasoning precedence, keeps fresh session-row updates authoritative, and preserves the current session-row thinking-default implementation.

## Reproduction Assessment

Yes. Source inspection shows current main still returns no effective `reasoningLevel` from `chat.history` and the chat view falls back to persisted row reasoning or `off`, while runtime already honors configured reasoning defaults.

## Solution Assessment

No. The PR targets the right bug, but it is not the best current fix because it misses the global reasoning-default contract and lets history-derived caches outrank fresher session/model state.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Match history reasoning defaults to runtime:** `src/gateway/server-methods/chat.ts:1667-1668`
  - body: The new `chat.history` reasoning calculation falls back only to the persisted row or per-agent config. Runtime also honors `agents.defaults.reasoningDefault`, so fresh sessions using the global default would still report the wrong reasoning state to the UI.
  - confidence: 0.91
- **[P2] Refresh cached thinking defaults after model changes:** `ui/src/ui/app-render.helpers.ts:682-683`
  - body: The default label now prefers `state.chatThinkingLevel`, but that value is refreshed only by `chat.history`. Changing the model through the chat header can leave the label showing the previous model’s default until a later history reload.
  - confidence: 0.86
- **[P2] Prefer fresh session reasoning over cached history state:** `ui/src/ui/views/chat.ts:1126`
  - body: Reasoning visibility now prioritizes `props.reasoningLevel` from history ahead of the live session row. A `sessions.patch` or `sessions.changed` update can therefore leave reasoning blocks shown or hidden according to stale history state until history is reloaded.
  - confidence: 0.9
