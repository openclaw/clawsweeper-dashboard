The PR adds effective thinking/reasoning values to `chat.history`, stores them in Web UI chat state, and uses them for the chat header default label and reasoning block visibility.

## Best Possible Solution

Land a rebased or successor patch that derives effective reasoning from the same runtime precedence/gating, preserves session-row thinking defaults as the fresh UI source, invalidates or refreshes effective chat state on session/model changes, and includes focused Gateway/UI tests plus real Web UI proof.
