## Summary

Keep this PR open because current main still does not expose or render effective reasoning defaults for fresh Web UI chat sessions, but the PR branch is not merge-ready. Current main already solved the thinking-default label path through session rows, while this PR's history-derived cache approach can go stale and its reasoning resolver misses the global `agents.defaults.reasoningDefault` runtime contract.

## What This Changes

The PR adds effective thinking and reasoning values to `chat.history`, stores them in Control UI chat state, uses them for the chat default label and reasoning visibility, and adds focused Gateway and UI tests.

## Best Possible Solution

Keep the merged session-row thinking-default path, then add a backward-compatible authoritative effective reasoning default contract that mirrors runtime precedence and is consumed from fresh session/default data.

## Reproduction Assessment

Yes. A fresh Web UI chat session with `agents.defaults.reasoningDefault` or `agents.list[].reasoningDefault` and no persisted `reasoningLevel` can be traced through current main: runtime resolves the configured default, while `chat.history` and `renderChat` do not expose or use it for reasoning visibility.

## Solution Assessment

No. The PR targets the right bug, but the proposed solution is not the best current fix because it misses the global reasoning default contract and puts history-derived cache values ahead of fresher session state.
