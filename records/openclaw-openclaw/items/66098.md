## Summary

Keep this PR open: current main still has the Web UI reasoning-default display gap, and this branch is a focused implementation candidate, but it is conflicting, lacks contributor real-behavior proof, and still has correctness issues already identified in review.

## What This Changes

The PR adds effective thinking/reasoning defaults to `chat.history` and threads those values through Control UI state, rendering, and tests for new chat sessions.

## Best Possible Solution

Land a rebased or successor fix that resolves effective reasoning with the same precedence and gating as runtime, avoids stale cached UI state, preserves the current session-row thinking-default implementation, and includes focused Gateway/UI tests plus real behavior proof.

## Reproduction Assessment

Yes, by source inspection: configure a reasoning default, open a fresh Web UI chat with no persisted `reasoningLevel`, and current main has `chat.history` omit the effective value while `renderChat` falls back to `off`. I did not run a live reproduction in this read-only review, so this is source-reproducible rather than executed.

## Solution Assessment

No. The PR targets the right bug, but the current branch is not the best mergeable solution because its reasoning precedence diverges from runtime and its UI caches can outrank fresher session state.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Match history reasoning defaults to runtime:** `src/gateway/server-methods/chat.ts:1667-1677`
  - body: The new history reasoning calculation only checks the session row and per-agent config before applying model defaulting. Runtime also honors `agents.defaults.reasoningDefault` and skips model-default reasoning when thinking was explicitly disabled, so fresh sessions can still report a different reasoning state than the agent actually uses.
  - confidence: 0.92
- **[P2] Refresh cached thinking defaults after model changes:** `ui/src/ui/app-render.helpers.ts:679-682`
  - body: The default label now prefers `state.chatThinkingLevel`, but that value is refreshed by `chat.history`, not by the model-switch path. Changing the model via the chat header can leave the thinking selector showing the previous effective default until history reloads.
  - confidence: 0.86
- **[P2] Prefer fresh session reasoning over cached history state:** `ui/src/ui/views/chat.ts:1123`
  - body: Reasoning visibility now prioritizes the history-derived `props.reasoningLevel` ahead of the live session row. A later session patch or session-change event can therefore leave reasoning blocks shown or hidden according to stale history state until history is reloaded.
  - confidence: 0.9
