Overall correctness: patch is incorrect

Full review comments:

- **[P2] Include the global reasoning default:** `src/gateway/server-methods/chat.ts:1667-1668`
  - body: This only checks the session override and per-agent `reasoningDefault`, but the documented/current runtime precedence also includes `agents.defaults.reasoningDefault`. Users who set only the global default would still see the Web UI fall back to `off` or model fallback instead of the effective value.
  - confidence: 0.88
- **[P2] Prefer fresh session reasoning over stale history cache:** `ui/src/ui/views/chat.ts:1126`
  - body: `props.reasoningLevel` is only refreshed by `chat.history`, while `sessions.patch` and session refreshes can update `activeSession.reasoningLevel` independently. Prioritizing the cached value can leave reasoning blocks shown or hidden after a setting change until history reloads.
  - confidence: 0.86
- **[P2] Invalidate cached thinking defaults after model changes:** `ui/src/ui/app-render.helpers.ts:680-682`
  - body: The default label uses `state.chatThinkingLevel` when there is no persisted override, but model switching patches the session and refreshes session rows without reloading history or clearing this cached value. After a model change, the header can keep showing the previous model/session default.
  - confidence: 0.82
