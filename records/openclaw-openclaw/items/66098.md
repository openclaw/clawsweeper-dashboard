## Summary

Keep open: the PR targets a real current-main Web UI/Gateway mismatch, but the branch is stale/conflicting, lacks real behavior proof, and its current diff misses parts of the runtime default contract and UI freshness path.

## What This Changes

The PR extends Gateway `chat.history` and Control UI chat state, rendering, and tests to carry effective thinking/reasoning defaults for fresh sessions.

## Best Possible Solution

Land a rebased or successor fix that exposes effective `reasoningLevel` through `chat.history` with the same precedence and gating as runtime, wires Control UI state without stale history outranking fresh session rows, and preserves the current session-row thinking-default implementation.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure a reasoning default, open a fresh Web UI session with no persisted `reasoningLevel`, then `chat.history` omits the effective reasoning state and `renderChat` falls back to `off`.

## Solution Assessment

No. The PR addresses the right bug, but this branch is not the best mergeable form because it misses the global reasoning-default fallback and lets history-derived UI caches outrank fresher session/model state.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Match history reasoning defaults to runtime:** `src/gateway/server-methods/chat.ts:1667-1668`
  - body: The new `chat.history` reasoning calculation falls back only to the persisted row, per-agent config, or model default. Runtime also honors `agents.defaults.reasoningDefault`, so fresh sessions using the global default would still report the wrong reasoning state to the UI.
  - confidence: 0.91
- **[P2] Refresh cached thinking defaults after model changes:** `ui/src/ui/app-render.helpers.ts:682-683`
  - body: The default label now prefers `state.chatThinkingLevel`, but that value is refreshed only by `chat.history`. Changing the model through the chat header can leave the label showing the previous model's default until a later history reload.
  - confidence: 0.86
- **[P2] Prefer fresh session reasoning over cached history state:** `ui/src/ui/views/chat.ts:1126`
  - body: Reasoning visibility now prioritizes `props.reasoningLevel` from history ahead of the live session row. A `sessions.patch` or `sessions.changed` update can therefore leave reasoning blocks shown or hidden according to stale history state until history is reloaded.
  - confidence: 0.9
