Full review comments:

- **[P2] Include the global reasoning default:** `src/gateway/server-methods/chat.ts:1667-1668`
  - body: This computes the effective reasoning level from only the persisted session override or per-agent default. Current docs and schema also support `agents.defaults.reasoningDefault`, so users who set only the global default would still get the wrong Web UI reasoning state for a fresh session.
  - confidence: 0.88
- **[P2] Prefer fresh session reasoning over stale history cache:** `ui/src/ui/views/chat.ts:1126`
  - body: `props.reasoningLevel` is only refreshed by `chat.history`, while `sessions.patch` and session refreshes can update `activeSession.reasoningLevel` independently. Prioritizing the cached value can leave reasoning blocks shown or hidden after a session setting change until history reloads.
  - confidence: 0.86
- **[P2] Invalidate cached thinking defaults after model changes:** `ui/src/ui/app-render.helpers.ts:682-683`
  - body: The default label uses `state.chatThinkingLevel` whenever there is no persisted override, but `switchChatModel()` patches the model and refreshes session rows without reloading history or clearing that cached value. After a model change, the header can keep showing the previous model/session default.
  - confidence: 0.82
