## Summary

Keep open. Current main still lacks the PR’s central replay-boundary behavior, and recent discussion adds live reproduction evidence, but the branch is now stale against the provider replay-policy architecture and needs a current-main rework before it can merge safely.

## What This Changes

The PR adds OpenAI-compatible completions thinking-signature sanitization, history-aware signature preservation, wrapped-Mistral strict9 tool-call ID handling, and related regression tests.

## Best Possible Solution

Rework the fix on current main’s provider replay-policy architecture: add a replay-boundary sanitizer in the current replay-history/helper path, preserve compatible provider-native thinking signatures until the replay target is known, strip only fields that openai-completions cannot replay, include current OpenAI-compatible fields such as reasoning_details, and add a scoped OpenRouter/proxy-provider Mistral strict9 policy with focused tests.

## Reproduction Assessment

Yes. Current source inspection gives a high-confidence static reproduction path: signed assistant thinking blocks can pass through sanitizeSessionHistory to openai-completions without a boundary sanitizer, and the PR discussion includes a live 2026-04-28 fallback failure on OpenClaw 2026.4.25 with OpenAI rejecting missing reasoning state.

## Solution Assessment

No, not as currently proposed. The core direction is right, but the patch is based on old files and misses current replay-policy hooks and current OpenAI-compatible signatures; the safer fix is a rebased provider-policy implementation with scoped proxy-provider strict9 handling.
