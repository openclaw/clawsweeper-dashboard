## Summary

Keep open. The reported replay-boundary bug still appears valid, but this PR is stale against current main’s provider-owned replay architecture and would not land cleanly or fix the current replay path without rework.

## What This Changes

The PR adds OpenAI-compatible completions thinking-signature sanitization, history-aware signature preservation, wrapped-Mistral strict9 detection, and focused regression tests.

## Best Possible Solution

Rework the fix on current main’s replay-history and provider replay-policy surfaces: preserve provider-native signatures until the replay target is known, strip only openai-completions-incompatible thinkingSignature values at that boundary, include current OpenAI-compatible signatures such as reasoning_details, and scope wrapped-Mistral strict9 behavior to OpenRouter/proxy-provider policy rather than a global table.

## Reproduction Assessment

Yes. Current source inspection shows openai-completions replay lacks a boundary sanitizer for provider-native thinkingSignature values, and the PR discussion includes a live 2026-04-28/2026-04-29 fallback failure on OpenClaw 2026.4.25 with OpenAI rejecting missing reasoning state.

## Solution Assessment

No, not as currently proposed. The direction is right, but the patch wires the sanitizer into obsolete files and omits current reasoning_details replay metadata; the safer solution is a rebased provider-policy implementation with focused tests.
