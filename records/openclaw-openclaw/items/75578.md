## Summary

Keep open. Current main still lacks the proposed Plugin SDK/Gateway session-action seam, so this PR is not obsolete, but the latest head has two narrow issues that should be fixed before merge: plugin-emitted event session keys are forwarded without normalization, and a new test fixture contains a secret-looking literal that can trip scanners.

## What This Changes

The PR adds Plugin SDK session-action registration and plugin-attributed event emission, stores session actions in the plugin registry, exposes `plugins.sessionAction` through the Gateway protocol, updates Swift protocol models, adds contract tests, and documents the feature in the changelog.

## Best Possible Solution

Land this as the focused generic session-action SDK/Gateway seam after fixing the two latest-head issues, preserving the additive protocol shape and keeping broader workflow attachments, scheduled turns, and fixture coverage in their separate follow-up PRs.

## Reproduction Assessment

Not applicable for the original feature request because this is an implementation PR, not a bug report. The review findings are high-confidence from static inspection: `emitPluginAgentEvent({ sessionKey: " agent:main:main " })` would emit the raw spaced key, and the secret-looking fixture literal is present at the cited line.

## Solution Assessment

No, not as-is. The overall generic seam is the right implementation direction, but the patch should normalize plugin-emitted event `sessionKey` the same way the new Gateway dispatch does and remove or allowlist the scanner-looking fixture before maintainer merge review.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Normalize emitted event session keys:** `src/plugins/agent-event-emission.ts:75`
  - body: `runId` and `stream` are trimmed before emission, but `sessionKey` is forwarded raw. A plugin that passes leading or trailing whitespace will emit events under a different session key than the normalized session-action dispatch path, creating duplicate or unfindable session-scoped events downstream. Normalize with `normalizeOptionalString` and include the key only when non-empty.
  - confidence: 0.91
- **[P3] Avoid secret-like fixture literals:** `src/plugins/contracts/session-actions.contract.test.ts:341`
  - body: The new test fixture throws `secret-token-123` without an allowlist pragma or clearly non-secret placeholder. Even though it is synthetic, this can trigger secret-scanning noise in CI or repository alerts; replace it with a clearly fake value or add the repo's allowlist pragma.
  - confidence: 0.86
