## What This Changes

The PR adds plugin session-action registration, plugin-attributed agent-event emission, `plugins.sessionAction` Gateway dispatch with scoped authorization, protocol/Swift model updates, tests, and a changelog entry.

## Best Possible Solution

Land the additive SDK/Gateway seam after reverting the shared JSON blob schema to `Type.Unknown()`, rerunning protocol/API checks, and completing maintainer contract review.

## Reproduction Assessment

Not applicable as a feature PR. Source inspection of current main confirms the requested API and Gateway method are not present today.

## Solution Assessment

Mostly yes, but not merge-ready as-is. The registry-owned session action design with dispatch-time scope enforcement fits the plugin boundary; the shared protocol JSON schema should stay as plain `Type.Unknown()` while runtime checks enforce JSON compatibility.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Keep plugin JSON protocol fields as Unknown:** `src/gateway/protocol/schema/plugins.ts:4-9`
  - body: `PluginJsonValueSchema` is used for unvalidated JSON blobs across plugin protocol fields, and runtime code already applies `isPluginJsonValue` where JSON compatibility matters. Wrapping `Type.Unknown()` in `Type.Cyclic(...)` makes generated protocol schemas carry `$defs`/`$ref` machinery for no validation benefit, which is inconsistent with the repo's other unknown payload fields and risks downstream draft-07/codegen compatibility. Please revert this back to plain `Type.Unknown()`.
  - confidence: 0.82
