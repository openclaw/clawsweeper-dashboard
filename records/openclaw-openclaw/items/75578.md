## Summary

Keep open. Current main does not yet implement the proposed session-action Plugin SDK/Gateway seam, and this draft PR is the active fresh-main slice replacing the closed consolidated stack. The diff is directionally aligned with the merged host-hook foundation, but it should not merge as-is because the new plugin `emitAgentEvent` API allows non-bundled plugins to emit host-owned event streams that Gateway clients already interpret as live agent output.

## What This Changes

The PR adds public Plugin SDK session-action and plugin event-emission APIs, registry storage/validation, Gateway `plugins.sessionAction` dispatch, protocol schemas/generated Swift models, focused tests, and a changelog entry.

## Best Possible Solution

Keep the generic session-action Gateway seam in this PR, but tighten `emitAgentEvent` so non-bundled plugins cannot publish host-owned event streams; then run the exact-head protocol/API/test gates and land the coordinated docs once the public contract is settled.

## Reproduction Assessment

Not applicable as a feature PR rather than a bug report. There is a high-confidence comparison path: current main lacks `registerSessionAction` and `plugins.sessionAction`, while PR head adds those surfaces and focused contract coverage; the review finding is reproducible by following `emitAgentEvent(stream: "assistant")` into Gateway chat-event handling.

## Solution Assessment

No as-is. The session-action dispatch design is a reasonable generic seam, but the event-emission portion should reserve built-in host streams for bundled/trusted emitters or require plugin-namespaced custom streams for external plugins.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Reserve host-owned streams for bundled emitters:** `src/plugins/agent-event-emission.ts:44-45`
  - body: This only blocks workspace plugins from `lifecycle`/`model`, but the existing event bus treats built-in streams like `assistant`, `tool`, `approval`, and `error` as host-owned run events. In particular, Gateway turns an `assistant` event with `data.text` into live chat output for the run/session, so a workspace plugin using this public SDK can spoof agent output if it supplies a runId/sessionKey. Please reserve all built-in streams for bundled plugins or require external plugins to emit plugin-namespaced/custom streams only.
  - confidence: 0.87

## Security Review

That broadens the public plugin SDK from plugin-attributed workflow events into host-event spoofing unless the allowed stream set is narrowed.
