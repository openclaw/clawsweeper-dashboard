## Best Possible Solution

## Reproduction Assessment

Not applicable as a user bug because this is a feature PR. The review finding is source-reproducible: PR head wraps `PluginJsonValueSchema` in `Type.Cyclic`, and TypeBox source confirms that emits `$defs`/`$ref` rather than the plain opaque schema used on current main.

## Solution Assessment

No, not with the current schema wrapper. The registry-owned SDK registration and Gateway dispatch shape fit the existing plugin boundary, but the opaque payload schema should stay `Type.Unknown()` while runtime JSON compatibility remains enforced in handlers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Keep plugin JSON payload schemas opaque:** `src/gateway/protocol/schema/plugins.ts:4-9`
  - body: This changes the shared opaque payload schema from `Type.Unknown()` to a TypeBox cyclic wrapper. `scripts/protocol-gen.ts` publishes draft-07 JSON Schema, but TypeBox `Cyclic` emits `$defs` plus a relative `$ref`; that does not add validation and can break downstream draft-07 validators/codegen resolving `PluginJsonValue` for session action payload/result fields. Please keep this as plain `Type.Unknown()` and rely on the existing runtime `isPluginJsonValue` checks.
  - confidence: 0.87
