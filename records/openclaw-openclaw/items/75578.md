## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Normalize emitted session keys:** `src/plugins/agent-event-emission.ts:75`
  - body: `emitPluginAgentEvent` normalizes `runId` and `stream`, but forwards `params.event.sessionKey` raw. A plugin that passes a value with surrounding whitespace will emit events under a different session key than the session-action dispatch path uses, so downstream subscribers can fail to correlate the event with the intended session. Normalize the session key with `normalizeOptionalString` and include it only when non-empty.
  - confidence: 0.89
- **[P3] Export the event emission types:** `src/plugin-sdk/plugin-entry.ts:135-150`
  - body: `emitAgentEvent` is now part of `OpenClawPluginApi`, and `src/plugins/types.ts` exports `PluginAgentEventEmitParams` / `PluginAgentEventEmitResult`, but the standard SDK barrels omit those new public types. Plugin authors importing from `openclaw/plugin-sdk/plugin-entry` or `/core` cannot name the method input/output types without reaching through a lower-level path.
  - confidence: 0.78
