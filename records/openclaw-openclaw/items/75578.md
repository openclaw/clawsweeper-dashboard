## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Derive session-action scopes without local registry state:** `src/gateway/method-scopes.ts:283-287`
  - body: `resolveSessionActionLeastPrivilegeScopes` only finds action scopes from this process's active plugin registry and otherwise falls back to `operator.write`. Out-of-process clients using the least-privilege helpers can have `pluginId`/`actionId` params but no local registry entry, so actions that require `operator.approvals` still connect with a write-scoped token and then fail dispatch with `missing scope`. Pass or discover the action-declared scopes from a caller-available source, or use a dynamic scope strategy that cannot under-scope these actions.
  - confidence: 0.87
