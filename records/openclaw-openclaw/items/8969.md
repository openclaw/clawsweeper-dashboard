## Summary

Keep open: current main and v2026.6.6 cover the first request as `openclaw skills check`, but status still lacks general manifest-registry diagnostics and the non-tool skill command path still rewrites requests without an operator-visible diagnostic.

## What This Changes

The issue requests three developer/operator diagnostics improvements: a skill validation CLI, plugin registry diagnostics in status output, and a warning when skill commands rewrite prompts for model-directed execution.

## Best Possible Solution

Keep the shipped `openclaw skills check`, then add sanitized manifest diagnostics through the canonical status text/JSON/all result path and add a deliberately scoped diagnostic for non-tool skill prompt rewriting, preferably as two focused maintainer-reviewed changes.

## Reproduction Assessment

Yes, at source level: current main exposes no general manifest-registry diagnostics field in status, and the non-tool skill branch rewrites the agent request without an operator-visible signal. This is an enhancement gap rather than a failing existing contract.

## Solution Assessment

Partly: extending the canonical status result and JSON/text renderers is the right shape for registry diagnostics, but the prompt-rewrite warning needs a maintainer decision on whether it belongs in inline text, logs, traces, or another diagnostic surface.
