## Summary

Keep open: current main covers the skill-validation part as `openclaw skills check`, but status still exposes plugin compatibility rather than manifest-registry diagnostics, and the non-tool skill-command path still rewrites the prompt silently. The prior issue discussion reached the same keep-open conclusion, and no searched related item fully supersedes the remaining umbrella work.

## What This Changes

Request to add three developer-experience surfaces: skill requirement validation, plugin/skill registry diagnostics in status, and a warning when skill commands rewrite prompts.

## Best Possible Solution

Preserve `openclaw skills check`, then add manifest-registry diagnostics to text, JSON, and all-status surfaces plus a deliberate operator-visible signal for model-directed skill-command invocation, with focused tests and docs.

## Reproduction Assessment

Not applicable as a bug reproduction: this is an enhancement request. Source inspection on current main clearly shows one requested surface exists while the status diagnostics and prompt-rewrite warning surfaces are absent.

## Solution Assessment

No as a close decision: current main only partially satisfies the requested quick wins. The maintainable path is to scope or split the two remaining surfaces, then implement them without broadening status/plugin runtime loading unexpectedly.
