## Summary

Keep open: the skill-debugging quick win is present as `openclaw skills check`, but current main still does not surface manifest-registry diagnostics through status and still rewrites non-tool skill commands without an operator-visible warning.

## What This Changes

The issue requests three developer-experience improvements: skill validation, plugin registry diagnostics in status, and a warning when skill commands are converted into model-directed prompt rewrites.

## Best Possible Solution

Keep the shipped `openclaw skills check` behavior, then split or implement the remaining status diagnostics and prompt-rewrite warning surfaces with explicit output contracts, focused tests, and docs.

Not applicable as a bug reproduction: this is an enhancement request. Source inspection on current main gives high confidence that `skills check` exists while status registry diagnostics and prompt-rewrite warnings remain absent.
