## Summary

Keep open. Current main and the latest shipped release cover the skill validation request as `openclaw skills check`, but source inspection still shows no manifest-registry diagnostics in status output and no operator-visible notice when a non-tool skill command rewrites the prompt.

## What This Changes

Request three developer-experience improvements: a skill requirements check command, plugin registry diagnostics in status, and a warning for prompt-rewrite skill command execution.

## Best Possible Solution

Preserve `openclaw skills check`, then add registry diagnostics to status text/JSON/all surfaces and an explicit operator-visible signal for model-directed skill command rewrites with focused tests and docs.

## Reproduction Assessment

Not applicable as a bug reproduction: this is an enhancement request. Source inspection gives high confidence about the current feature gap because the status result carries only plugin compatibility and the prompt-rewrite branch mutates the request without a warning surface.

## Solution Assessment

Unclear as one umbrella item. The requested direction is maintainable, but the best path is either to split the two remaining surfaces or implement them together after maintainers choose the status JSON shape and prompt-rewrite warning surface.
