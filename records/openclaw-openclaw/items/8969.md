## What This Changes

The issue requests three developer/operator diagnostics: skill requirement validation, plugin/skill registry diagnostics in status, and a warning when skill commands rewrite prompts for model-directed execution.

## Best Possible Solution

Keep the shipped `openclaw skills check` behavior, then deliberately add sanitized manifest-registry diagnostics to status surfaces and choose one operator-visible diagnostic for non-tool skill prompt rewrites.

## Reproduction Assessment

Yes, at source level: current main exposes skill readiness through `openclaw skills check`, but status lacks a general manifest-registry diagnostics field and the non-tool skill command branch rewrites the agent request without an operator-visible signal.

## Solution Assessment

Partly: the status diagnostics belong in the canonical status result/rendering path, but the prompt-rewrite warning needs maintainer choice on whether it is inline text, logs, trace/debug output, or another diagnostic surface.
