## Summary

Keep open: current main covers the skills validation request as `openclaw skills check`, but the status registry-diagnostics surface and the operator-visible skill prompt-rewrite warning are still missing.

## What This Changes

The issue requests three additive developer/operator UX improvements: a skills validation CLI, plugin registry diagnostics in status output, and a warning when skill commands rewrite the prompt for model-directed execution.

## Best Possible Solution

Preserve the shipped `openclaw skills check` path, then implement or split the two remaining operator surfaces with focused status text/JSON/all coverage, prompt-rewrite branch coverage, and docs updates.

## Reproduction Assessment

Yes, for the remaining gaps by source inspection: current main has `skills check`, while status paths do not expose manifest-registry diagnostics and the non-tool skill command branch rewrites the prompt without a warning. This is an enhancement rather than a failing existing contract.

## Solution Assessment

Yes, with a split-design caveat. The maintainable path is to keep the shipped skills command and handle the remaining status and prompt-rewrite visibility requests as focused operator-facing changes after maintainers choose the warning surface.
