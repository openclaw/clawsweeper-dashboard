Keep open: current main and v2026.6.8 ship the skill validation part as `openclaw skills check`, but the requested status manifest diagnostics and prompt-rewrite warning are still missing.

## What This Changes

Request three developer/operator diagnostics: a skill validation CLI command, plugin registry diagnostics in status output, and a warning when skill commands rewrite prompts.

## Best Possible Solution

Keep `openclaw skills check`, then add sanitized manifest diagnostics to the canonical status text/JSON/all surfaces and choose a deliberate operator-visible signal for non-tool skill prompt rewrites, with focused tests and docs.

## Reproduction Assessment

Yes at source level, but this is an enhancement gap rather than a broken existing contract. Current main exposes no manifest-registry diagnostics in status, and the non-tool skill branch rewrites the agent prompt without an operator-visible signal.

## Solution Assessment

Partly. Status diagnostics belong in the canonical status result/rendering path, while the prompt-rewrite warning needs maintainer choice on whether it should be inline text, logs, trace/debug output, or another diagnostic surface.
