## Summary

Keep open: current main and v2026.6.6 implement the skill-check part as `openclaw skills check`, but the two remaining operator diagnostics are still missing from status output and the non-tool skill-command prompt rewrite path.

## What This Changes

The issue requests three developer/operator ergonomics improvements: a skill validation command, plugin registry diagnostics in status, and a warning when skill commands rewrite prompts for model-directed execution.

## Best Possible Solution

Keep `openclaw skills check` as the shipped answer for skill validation, then add sanitized manifest diagnostics to the canonical status text/JSON/all surfaces and choose a deliberate operator-visible diagnostic for non-tool skill prompt rewrites.

## Reproduction Assessment

Yes, at source level: current main exposes no general manifest-registry diagnostics field in status, and the non-tool skill branch rewrites the agent request without an operator-visible signal. This is an enhancement gap rather than a broken existing contract.

## Solution Assessment

Partly: the status diagnostics belong in the canonical status result/rendering path, but the prompt-rewrite warning needs maintainer choice on whether it is inline text, logs, trace/debug output, or another diagnostic surface.
