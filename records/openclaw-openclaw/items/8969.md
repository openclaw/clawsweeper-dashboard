## What This Changes

The issue requests three developer/operator experience improvements: skill validation CLI diagnostics, plugin registry diagnostics in status output, and a warning when a skill command rewrites the prompt.

## Best Possible Solution

Preserve the shipped `openclaw skills check` path, then implement or split the remaining two operator surfaces with focused status text/JSON/all coverage, prompt-rewrite branch coverage, and docs updates after the warning surface is chosen.

## Reproduction Assessment

Yes, for the remaining source-level gaps: current main has `skills check`, while status paths do not expose manifest-registry diagnostics and the non-tool skill command branch rewrites the prompt without a warning. This is an enhancement rather than a failing existing contract.

## Solution Assessment

Yes, with a split-design caveat. The maintainable path is to keep the shipped skills command and handle the remaining status and prompt-rewrite visibility requests as focused operator-facing changes after maintainers choose the warning surface.
