## Summary

Keep open: current main still does not expose the requested structured provenance, internal-event, or pre-prompt-boundary metadata to `ContextEngine.assemble()`, and the remaining work is a new context-engine/plugin contract decision rather than stale cleanup.

## What This Changes

The issue requests optional `ContextEngine.assemble()` parameters for the current turn boundary, per-message `InputProvenance`, and per-message `AgentInternalEvent` annotations.

## Best Possible Solution

Design and document a backward-compatible context-engine metadata contract that exposes only the runtime signals plugins need while preserving hidden runtime-context filtering and plugin trust boundaries.

## Reproduction Assessment

Not applicable; this is an RFC for a new plugin/API contract. Source inspection gives high confidence that current main does not expose the requested assemble metadata, but there is no broken existing documented behavior to reproduce as a bug.

## Solution Assessment

Unclear; optional assemble metadata is a plausible direction, but the final shape should be settled by ContextEngine owners because it spans core contracts, embedded and Codex runner plumbing, runtime-context filtering, SDK exports, and plugin trust boundaries.
