## Summary

Keep open: current main still lacks the requested structured assemble metadata, and the newer ContextClaw packet reinforces this as an active ContextEngine API design question rather than cleanup.

## What This Changes

Request to add optional structured turn-boundary, input-provenance, and internal-event metadata to `ContextEngine.assemble()` for plugin context engines.

## Best Possible Solution

Design and document a backward-compatible `ContextEngine.assemble()` metadata contract that gives engines safe turn-boundary and provenance signals without leaking hidden runtime context.

## Reproduction Assessment

Not applicable; this is a ContextEngine API RFC, not a reported failure of existing documented behavior. Source inspection confirms the current assemble path does not expose `prePromptMessageCount`, `InputProvenance`, or `internalEvents`.

## Solution Assessment

Unclear; optional metadata fields are a plausible direction, but maintainers should settle the safest contract shape because the change spans core, embedded runner, Codex harness, runtime filtering, and plugin trust boundaries.
