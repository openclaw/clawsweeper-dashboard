## Summary

Keep open: current main still lacks the requested structured boundary/provenance/event metadata in `ContextEngine.assemble()`, and the linked lossless-claw and ContextClaw packets make this an active context-engine API-shape discussion rather than cleanup.

## What This Changes

Request to add optional `prePromptMessageCount`, per-message `inputProvenance`, and per-message `internalEvents` metadata to `ContextEngine.assemble()`.

## Best Possible Solution

Settle a backward-compatible context-engine metadata contract that exposes the minimum structured turn-boundary and provenance signals needed without leaking unnecessary runtime internals.
