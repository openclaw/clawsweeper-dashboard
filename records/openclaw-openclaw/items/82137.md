## Summary

Keep open: current main still does not expose the requested structured provenance/event/boundary metadata to `ContextEngine.assemble()`, and the lossless-claw plus ContextClaw packets are useful downstream evidence for a real API-shape decision rather than stale cleanup.

## What This Changes

Expose turn-boundary, per-message `InputProvenance`, and per-message `AgentInternalEvent` signals to `ContextEngine.assemble()` so context-engine plugins do not infer runtime-only inputs from text shape.

## Best Possible Solution

Design and document a backward-compatible context-engine metadata contract that gives engines structured live-turn and provenance signals while preserving hidden runtime-context filtering and plugin trust boundaries.

## Reproduction Assessment

Not applicable; this is an RFC for a new plugin/context-engine API contract. Source inspection gives a high-confidence contract gap, but there is no broken existing documented behavior to reproduce as a bug.

## Solution Assessment

Unclear; optional metadata fields are plausible, but the final shape should be decided by context-engine owners because it spans core, embedded runner, Codex harness, runtime-context filtering, and plugin trust boundaries.
