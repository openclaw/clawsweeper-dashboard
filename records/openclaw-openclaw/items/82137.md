## Summary

Keep open: current main still lacks the requested structured provenance, internal-event, and turn-boundary metadata on `ContextEngine.assemble()`, and this is a new core/plugin API contract decision rather than conservative cleanup.

## What This Changes

The issue requests optional `prePromptMessageCount`, per-message `InputProvenance`, and per-message `AgentInternalEvent` metadata on `ContextEngine.assemble()` for durable context engines.

## Best Possible Solution

Design and document a backward-compatible context-engine metadata contract that exposes only stable provenance and turn-boundary signals while preserving hidden runtime-context filtering and plugin trust boundaries.

## Reproduction Assessment

Not applicable; this is an RFC for a new plugin/API contract, not a report of broken existing documented behavior. Source inspection confirms current main lacks the requested assemble fields.

## Solution Assessment

Unclear; optional assemble metadata is plausible, but the exact fields, type exports, and visibility rules should be settled by context-engine/runtime owners before implementation.
