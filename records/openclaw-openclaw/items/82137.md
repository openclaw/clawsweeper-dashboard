## Summary

Keep open: current main still does not expose the requested structured turn-boundary, provenance, or internal-event metadata to `ContextEngine.assemble()`, and the newer ContextClaw packet adds useful downstream evidence rather than making the RFC obsolete.

## What This Changes

The issue asks OpenClaw to add optional `prePromptMessageCount`, per-message `InputProvenance`, and per-message `AgentInternalEvent` metadata to `ContextEngine.assemble()`.

## Best Possible Solution

Design one backward-compatible context-engine metadata contract that gives engines structured turn-boundary and provenance/event signals while preserving hidden runtime-context filtering and plugin trust boundaries.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an RFC for a new core/plugin API contract. Source inspection gives high confidence that current main lacks the requested assemble metadata.

## Solution Assessment

Unclear. Optional metadata on `assemble()` is a plausible direction, but the final shape should be decided with context-engine owners because it spans core runner calls, the Codex harness, hidden runtime-context filtering, and plugin trust boundaries.
