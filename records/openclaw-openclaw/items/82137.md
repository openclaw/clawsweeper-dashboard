## Summary

Keep open: current main and v2026.5.18 still omit structured assemble metadata, while the related hook and per-call assembly threads are adjacent rather than a canonical replacement. The lossless-claw and ContextClaw context adds useful downstream evidence, but the remaining decision is a core ContextEngine/API contract review.

## What This Changes

This issue proposes adding optional prePromptMessageCount, inputProvenance, and internalEvents metadata to ContextEngine.assemble().

## Best Possible Solution

Design one backward-compatible ContextEngine metadata contract that gives engines structured turn-boundary and provenance signals while preserving runtime-context filtering, plugin trust boundaries, docs, SDK exports, and tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an RFC for a new plugin/API contract. Source inspection gives high confidence that current main and v2026.5.18 do not expose the requested assemble metadata.

## Solution Assessment

Unclear. Optional fields are a plausible approach, but maintainers should choose the final shape alongside the per-call assembly and before_assemble/after_assemble discussions so the public contract stays coherent.
