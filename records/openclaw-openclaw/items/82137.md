## Summary

Keep open: current main and v2026.6.10 still do not expose the requested live-turn boundary, per-message input provenance, or internal-event metadata to `ContextEngine.assemble()`. The remaining decision is a public context-engine/plugin SDK API shape, not stale cleanup or an already-implemented fix.

## What This Changes

Request to add optional `prePromptMessageCount`, per-message `InputProvenance`, and per-message `AgentInternalEvent` metadata to `ContextEngine.assemble()`.

## Best Possible Solution

Design one backward-compatible context-engine metadata contract that defines safe SDK type exposure, preserves runtime-context filtering, and covers embedded plus Codex harness assemble paths with focused tests and docs.

## Reproduction Assessment

Not applicable as a bug reproduction: this is an RFC for a new plugin/API contract. Source inspection confirms the requested assemble fields are absent from current main and v2026.6.10.

## Solution Assessment

Unclear: optional metadata fields are a plausible direction, but the best public API shape requires maintainer review because it spans plugin SDK exposure, runtime-context filtering, embedded runner, and Codex harness behavior.
