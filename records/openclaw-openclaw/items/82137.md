## Summary

Keep open: current main and latest `v2026.6.6` still omit the requested provenance, internal-event, and pre-prompt boundary fields from `ContextEngine.assemble()`, while source confirms those signals exist elsewhere. The remaining work is a core/plugin API contract decision, not stale cleanup.

## What This Changes

The issue asks to expose optional pre-prompt boundary, `InputProvenance`, and `AgentInternalEvent` metadata to `ContextEngine.assemble()`.

## Best Possible Solution

Design and document a backward-compatible context-engine metadata contract that preserves runtime-context filtering, defines safe SDK type exposure, and covers both embedded and Codex harness paths.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an RFC for a new plugin/API contract. Source inspection confirms current main lacks the requested assemble fields, but no broken existing documented behavior is being reproduced here.

## Solution Assessment

Unclear; optional metadata fields are a plausible direction, but the best API shape requires maintainer design review because it spans plugin SDK exposure, hidden runtime-context filtering, embedded runner, and Codex harness behavior.
