## Summary

Keep open: current main still does not expose the requested structured prompt-boundary, provenance, or internal-event metadata to `ContextEngine.assemble()`, and the remaining work is a new core/plugin SDK contract decision with active downstream input rather than stale cleanup.

## What This Changes

RFC requests optional `prePromptMessageCount`, per-message `InputProvenance`, and per-message `AgentInternalEvent` metadata for `ContextEngine.assemble()`.

## Best Possible Solution

Design and document a backward-compatible context-engine metadata contract that gives engines structured turn-boundary and provenance signals while preserving OpenClaw's hidden runtime-context and plugin trust boundaries.

## Reproduction Assessment

Not applicable; this is an RFC for a new context-engine/plugin SDK contract. Source inspection confirms current main lacks the requested assemble fields, but there is no broken existing documented behavior to reproduce as a bug.

## Solution Assessment

Unclear; optional additive metadata is a plausible direction, but maintainers should settle the contract shape because it spans core assemble types, embedded runner and Codex harness call sites, runtime-context filtering, and plugin trust boundaries.
