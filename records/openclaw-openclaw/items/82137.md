## Summary

Keep open: current main and v2026.6.8 still omit the requested `prePromptMessageCount`, per-message `inputProvenance`, and per-message `internalEvents` from `ContextEngine.assemble()`. The remaining work is a plugin/context-engine API contract decision, not stale cleanup or an already-implemented request.

## What This Changes

This RFC requests a backward-compatible expansion of `ContextEngine.assemble()` so context-engine plugins can see live-turn boundaries and structured runtime provenance/event metadata.

## Best Possible Solution

Design and document one backward-compatible context-engine metadata contract that defines safe SDK type exposure, preserves runtime-context filtering, and covers both embedded and Codex harness paths.

## Reproduction Assessment

Not applicable as a bug reproduction: this is an RFC for a new plugin/API contract. Source inspection confirms the requested assemble fields are absent from current main and the latest release.

## Solution Assessment

Unclear: optional metadata fields are a plausible direction, but the best public API shape requires maintainer review because it spans plugin SDK exposure, runtime-context filtering, embedded runner, and Codex harness behavior.
