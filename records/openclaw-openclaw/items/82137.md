## Summary

Keep open: current main and v2026.6.10 still do not expose the requested live-turn boundary, per-message provenance, or internal-event metadata to `ContextEngine.assemble()`, and the remaining work is a public plugin/context-engine API decision rather than stale cleanup.

## What This Changes

The issue proposes adding optional `prePromptMessageCount`, per-message `InputProvenance`, and per-message `AgentInternalEvent` metadata to `ContextEngine.assemble()`.

## Best Possible Solution

Define one backward-compatible context-engine metadata contract covering SDK type exposure, embedded runner wiring, Codex harness wiring, docs, and focused lifecycle tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is an RFC for a new plugin/API contract. Source and release inspection confirm the requested assemble fields are absent from current main and v2026.6.10.

## Solution Assessment

Unclear: optional metadata fields are a plausible direction, but the best public API shape requires maintainer review because it spans plugin SDK exposure, runtime-context filtering, embedded runner behavior, and Codex harness behavior.
