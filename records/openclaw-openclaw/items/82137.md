## Summary

Keep open: current main still does not expose the requested turn-boundary, input-provenance, or internal-event metadata to `ContextEngine.assemble()`, and the newer ContextClaw packet plus the lossless-claw workaround show active downstream demand. This is a new public context-engine/API contract decision, not stale cleanup or a narrow reproduced bug fix.

## What This Changes

The issue requests optional `ContextEngine.assemble()` metadata for pre-prompt boundary, per-message input provenance, and per-message internal events.

## Best Possible Solution

Design and document a backward-compatible context-engine metadata contract that gives engines structured turn-boundary and provenance signals while preserving runtime privacy and hidden-context filtering.

## Reproduction Assessment

Not applicable; this is an RFC for a new plugin/API contract rather than a broken documented behavior. Source inspection confirms the requested fields are absent from current `assemble()` while the underlying signals exist elsewhere.

## Solution Assessment

Unclear; optional metadata fields are a plausible direction, but the exact public shape should be settled by ContextEngine owners because it spans PI runner, Codex harness, plugin SDK exports, and runtime trust boundaries.
