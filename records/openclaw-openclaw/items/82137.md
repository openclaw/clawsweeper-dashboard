## Summary

Keep open. Current main and the latest release still omit the requested `assemble` metadata, while the provenance/internal-event signals do exist internally; the remaining work is a new public ContextEngine/SDK contract that needs maintainer API and security review rather than cleanup closure.

## What This Changes

The issue requests optional `ContextEngine.assemble` metadata for pre-prompt boundaries, input provenance, and internal events so external context engines can avoid text-sniffing runtime-injected messages.

## Best Possible Solution

Define a narrow additive `assemble` contract with documented trust/redaction semantics, typed SDK exports, PI and Codex parity, feature-detectable optional fields, and focused lifecycle tests; keep cadence and compaction changes in their existing related lanes.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an RFC for a new plugin-facing contract. The contract gap is source-verified on current main and `v2026.5.12` by comparing `assemble`, `afterTurn`, and the existing provenance/internal-event call sites.

## Solution Assessment

Unclear as a final design. Optional fields are a plausible narrow direction, but maintainers need to decide the public event/provenance shape, alignment semantics, SDK exports, and security boundary before implementation.
