## Summary

Keep open: current main and the latest release still do not expose turn-boundary, input-provenance, or internal-event metadata through `ContextEngine.assemble()`. The downstream lossless-claw and ContextClaw context make the RFC useful, but this is a new plugin/API contract with trust-boundary choices rather than conservative cleanup.

## What This Changes

The RFC asks OpenClaw to add optional `prePromptMessageCount`, per-message `InputProvenance`, and per-message `AgentInternalEvent` metadata to `ContextEngine.assemble()`.

## Best Possible Solution

Design one backward-compatible ContextEngine metadata contract with documented redaction semantics, then thread it through PI and Codex assemble paths with focused SDK/docs/tests.

## Reproduction Assessment

Not applicable; this is an RFC for a new plugin/API contract, not a reported regression in existing documented behavior. Source inspection does confirm current main lacks the requested assemble metadata fields.

## Solution Assessment

Unclear as the final implementation shape. Optional assemble metadata is plausible, but maintainers should choose the stable SDK surface, redaction rules, and PI/Codex parity before implementation.
