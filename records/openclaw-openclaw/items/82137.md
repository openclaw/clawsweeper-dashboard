## Summary

Keep open: current main and the latest release still do not expose live-turn boundary, per-message provenance, or internal-event metadata to `ContextEngine.assemble()`. The remaining work is a public plugin/context-engine API design decision, with live downstream plugin context from ContextClaw/lossless-claw supporting the need.

## What This Changes

The issue requests optional `ContextEngine.assemble()` parameters for `prePromptMessageCount`, aligned `InputProvenance`, and aligned `AgentInternalEvent` metadata so context engines can avoid text-sniffing volatile runtime inputs.

## Best Possible Solution

Design one backward-compatible context-engine metadata contract that defines stable SDK exposure, preserves runtime-context filtering semantics, and covers both embedded and OpenClaw Codex harness assemble paths.

## Reproduction Assessment

Not applicable as a bug reproduction: this is an RFC for a new plugin/API contract. Source inspection confirms the requested assemble fields are absent from current main and v2026.6.10.

## Solution Assessment

Unclear: optional metadata fields are a plausible direction, but the best public API shape requires maintainer review because it spans plugin SDK exposure, runtime-context filtering, embedded runner, and OpenClaw Codex harness behavior.
