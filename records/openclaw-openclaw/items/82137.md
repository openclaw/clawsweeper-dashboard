## Summary

Keep open: current main still does not expose structured turn-boundary, provenance, or internal-event metadata to `ContextEngine.assemble()`. The issue is active RFC/API work with concrete downstream context from lossless-claw and ContextClaw, not conservative cleanup to close.

## What This Changes

The issue proposes adding optional `prePromptMessageCount`, `inputProvenance`, and `internalEvents` metadata to `ContextEngine.assemble()` for context-engine plugins.

## Best Possible Solution

Settle a documented, backward-compatible assemble metadata contract that exposes the minimum turn-boundary and provenance signal plugins need without leaking unnecessary runtime internals, then update PI, Codex, SDK exports, docs, and regression tests together.

## Reproduction Assessment
