## What This Changes

The issue reports that plugin text substitutions can rewrite legacy brand strings inside tool-call paths, sending file operations to the wrong workspace.

## Best Possible Solution

Prove the transform owner in a current install, then constrain plugin text transforms away from filesystem and tool-argument surfaces or define a safer explicit contract, with regression coverage and live proof; use https://github.com/mpz4life/openclaw/pull/2 as review input rather than a reason to close this issue.

## Reproduction Assessment

## Solution Assessment

Unclear. Disabling transforms on tool-bearing surfaces or making them path-aware may be the right fix, but the API semantics and Codex/native-tool boundary need maintainer review before choosing the permanent shape.
