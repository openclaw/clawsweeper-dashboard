## Summary

Keep open. Current main still lacks the requested legacy `channels.imessage` auto-acknowledgment or native typing-feedback path before agent dispatch, and BlueBubbles typing support is separate recommended-channel coverage rather than an implementation of this legacy iMessage request.

## What This Changes

Add configurable channel-layer auto-acknowledgment for legacy iMessage before agent processing, including custom text and DM-only behavior.

## Best Possible Solution

Keep this open until owners implement or explicitly decline legacy `channels.imessage` feedback; if implemented, keep it in `extensions/imessage` with typed config/schema/docs, post-gating pre-dispatch feedback, `dmOnly` handling, and echo suppression coverage.

## Solution Assessment
