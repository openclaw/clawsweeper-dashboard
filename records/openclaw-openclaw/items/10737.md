## Summary

Keep open. Current main still has no legacy `channels.imessage` auto-acknowledgment or native typing-feedback path before agent dispatch, and the existing BlueBubbles typing support is a separate recommended iMessage route rather than an implementation of this legacy channel request.

## What This Changes

Request channel-level automatic acknowledgment or typing feedback for legacy iMessage before agent dispatch.

## Best Possible Solution

Keep this open until maintainers implement or explicitly decline legacy `channels.imessage` feedback; an implementation should add typed config/schema/docs, send feedback after access and mention gating but before dispatch, honor `dmOnly`, and cover echo suppression.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection does give high-confidence proof that current main lacks the requested legacy iMessage config key and pre-dispatch feedback path.

## Solution Assessment

Unclear. The narrow implementation belongs in `extensions/imessage`, but the best user-facing behavior still needs owner choice between native upstream typing support, a text fallback, or formally steering legacy users to BlueBubbles.
