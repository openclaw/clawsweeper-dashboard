## Summary

Keep open. Current main still has no legacy `channels.imessage` auto-acknowledgment or native typing-feedback path before agent dispatch, and the latest maintainer-authored discussion kept this request separate from BlueBubbles, which only covers the recommended `channels.bluebubbles` route.

## What This Changes

Add immediate pre-dispatch acknowledgment or typing feedback for inbound messages on the legacy iMessage channel.

## Best Possible Solution

Maintainers should either explicitly decline legacy `channels.imessage` feedback or add a default-off typed config/schema/docs surface that sends feedback after access and mention gating but before dispatch, with echo-cache coverage for any text fallback.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a feature request. The current-main verification path is high-confidence: the strict iMessage config schema has no `autoAck` field, and the legacy monitor has no pre-dispatch feedback call.

## Solution Assessment

Unclear. The requested text auto-ack is implementable, but the best final shape needs maintainer choice between declining legacy-channel work, using a guarded text fallback, or validating whether any current `imsg` native typing API is usable.
