## Summary

Keep open. Current main still lacks a legacy `channels.imessage` pre-dispatch auto-acknowledgment or typing-feedback path, BlueBubbles typing support is separate recommended-channel coverage, and recent maintainer context already kept this request open pending owner direction.

## What This Changes

Request adds a legacy `channels.imessage.autoAck` option that sends an immediate acknowledgment or typing-like signal before handing inbound iMessage traffic to the agent.

## Best Possible Solution

Keep this open until channel owners implement or explicitly decline legacy `channels.imessage` feedback; if implemented, keep it in `extensions/imessage` with typed config/schema/docs, post-gating pre-dispatch feedback, `dmOnly` handling, and echo suppression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high-confidence proof that current main lacks the requested legacy iMessage config key and pre-dispatch feedback path.

## Solution Assessment

Unclear. The implementation boundary is `extensions/imessage`, but the best user-facing behavior requires owner choice between upstream native typing, a text acknowledgment fallback, or steering legacy users to BlueBubbles.
