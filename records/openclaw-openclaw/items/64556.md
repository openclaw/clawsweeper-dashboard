## Summary

Keep open. Current main still accepts hook mapping `agentId` and `sessionKey` but drops them on the `action: "wake"` path before dispatch, and the linked closing PR is still open/conflict-marked rather than merged.

## What This Changes

Webhook wake mappings should honor configured `agentId` and `sessionKey` instead of routing every wake event to the default main session.

## Best Possible Solution

Review, rebase, and land https://github.com/openclaw/openclaw/pull/70268 or an equivalent narrow fix that threads wake mapping targets through the existing hook session-policy path with focused regressions.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: configure a wake mapping with `agentId` or `sessionKey`, then the mapping builder and mapped wake branch drop those fields before `dispatchWakeHook` queues into the main session.

## Solution Assessment

Yes. Threading wake targets through the existing hook session-policy path is the narrow maintainable direction; the open closing PR should be repaired or replaced rather than racing it with a separate issue closure.
