## Summary

Keep open. Current main still accepts hook mapping `agentId` and `sessionKey` but drops them on the mapped `wake` path, and https://github.com/openclaw/openclaw/pull/70268 is still open with closing syntax rather than merged.

## What This Changes

The issue asks OpenClaw to stop silently ignoring `hooks.mappings[].agentId` and `sessionKey` for `action: "wake"` and route wake events to the configured agent/session.

## Best Possible Solution

Land a rebased narrow fix, likely https://github.com/openclaw/openclaw/pull/70268 or an equivalent replacement, that threads mapped wake targets through the existing hook session-policy path with focused regressions.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: configure a mapped wake hook with `agentId` or `sessionKey`, then the mapping builder and mapped wake branch drop those fields before dispatching to the main session.

## Solution Assessment

Yes for the reported mapped wake bug. Threading `agentId` and `sessionKey` through the existing hook session-policy routing is the narrow maintainable solution; the issue should stay open until that lands.
