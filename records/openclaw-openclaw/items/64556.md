## Summary

Keep open. Current main still drops wake-mapping routing before dispatch, and an open closing PR already targets the bug, so this issue should stay open until https://github.com/openclaw/openclaw/pull/70268 or an equivalent fix lands.

## What This Changes

Webhook wake mappings should honor configured `agentId` and `sessionKey` routing instead of accepting those fields and dispatching to the default main session.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/70268 or an equivalent narrow fix that threads wake mapping `agentId` and `sessionKey` through the existing hook session-policy path, with focused regressions and docs/changelog coverage.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: configure a wake mapping with `agentId` or `sessionKey`, then `buildActionFromMapping` and the request handler drop those fields before `dispatchWakeHook` enqueues into the main session.

## Solution Assessment

Yes. Reusing the existing hook target-agent and session-key policy path is the narrowest maintainable solution; the active closing PR should be reviewed, repaired, or replaced rather than racing it with a separate branch.
