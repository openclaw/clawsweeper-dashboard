## Summary

Keep open. Current main still accepts hook mapping `agentId` and `sessionKey` while the wake mapping path drops both before dispatching to the main session, and the open fixing PR still references this issue with closing syntax rather than having merged.

## What This Changes

The issue reports that wake-mode webhook mappings accept `agentId` and `sessionKey` but silently ignore them and wake the default main session.

## Best Possible Solution

Review, rebase, and land https://github.com/openclaw/openclaw/pull/70268 or an equivalent narrow fix that threads mapped wake targets through the existing hook session-policy path with focused regressions.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction path: configure a wake mapping with `agentId` or `sessionKey`, then the mapping builder and mapped wake branch drop those fields before `dispatchWakeHook` queues into the main session.

## Solution Assessment

Yes. Threading mapped wake targets through the existing hook target-agent and session-policy path is the narrow maintainable fix; the open closing PR should be repaired or replaced rather than closing this issue first.
