## Summary

Keep open: current main and the latest release still drop the configured wake-hook routing fields, and the open closing PR is the right implementation candidate but is not merged.

## What This Changes

The issue asks for `action: "wake"` hook mappings to honor configured `agentId` and `sessionKey` instead of silently routing every wake to the default main session.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/70268 with a narrow fix that threads wake hook routing through the existing hook session policy and regression tests, then close this issue as implemented.

## Reproduction Assessment

Yes, from source inspection: current main accepts mapping `agentId`/`sessionKey`, but the wake action, request handler, and dispatcher only carry `text`/`mode` and resolve the main session. I did not run a live gateway curl repro in this read-only review.

## Solution Assessment

Yes, the maintainable fix is to wire the existing routing fields through the wake path using the same hook session-policy checks used for agent hooks. The open linked PR is the right work item, but it needs rebase/conflict resolution and proof before merge.
