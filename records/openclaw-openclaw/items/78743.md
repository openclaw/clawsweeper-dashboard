Keep open: the force-pushed harness-layer direction matches the source-level bug, but the PR is not merge-ready because the sanitizer can still return validator-invalid details and the external after-fix real behavior proof is still missing.

## What This Changes

The PR adds a harness-level tool-result middleware fast path, sanitizes incoming `details` before registered middleware runs, adds focused middleware tests, and records a changelog entry.

## Best Possible Solution

Keep the fix at the harness boundary, make the sanitized incoming `details` provably satisfy `isValidMiddlewareDetails` before middleware sees them, add key/depth regression coverage, and require redacted after-fix live proof before merge.

## Reproduction Assessment
