## Summary

Keep open: the underlying bug is source-reproducible and not superseded, but this PR still has a sanitizer correctness blocker and the external after-fix proof is insufficient for merge.

## What This Changes

The PR fast-paths agent tool-result middleware when no handler exists, sanitizes incoming result details before middleware runs, adds harness regression tests, isolates two provider-registry test modules, and adds a changelog entry.

## Best Possible Solution

Keep the fix at the harness boundary, but make the sanitizer produce details that satisfy the existing byte, key, and depth caps, add regressions for those caps, and then supply redacted tokenjuice-enabled runtime proof.

## Reproduction Assessment

Yes for the source-level path: current main validates the unmodified result after a registered no-op middleware, and tokenjuice returns undefined when it leaves a result unchanged. I did not establish a live gateway or Discord reproduction in this read-only review.

## Solution Assessment

No, not yet. The harness boundary is the right place to fix this bug class, but the current sanitizer must mirror the validator's key and depth caps before the patch is safe to merge.
