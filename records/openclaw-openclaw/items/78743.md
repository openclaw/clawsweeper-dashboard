## Summary

Keep open: the PR addresses a real source-level bug in the tool-result middleware path, but the current patch still allows key-count or depth-invalid sanitized details to fail closed, and the external PR still lacks after-fix real runtime proof for this exact patch.

## What This Changes

The PR changes the agent harness tool-result middleware runner to fast-path empty handler lists, sanitize incoming result details before middleware handlers run, add harness regression tests, and update the changelog.

## Best Possible Solution

Keep the fix at the harness boundary, make sanitized incoming details always satisfy `isValidMiddlewareDetails` including key/depth caps, add focused regressions for those caps, then provide redacted after-fix runtime proof.

## Reproduction Assessment

Yes for the source-level path: current main validates the unmodified result after a registered no-op middleware, and tokenjuice returns `undefined` when it leaves a result unchanged. I did not establish a fresh live Discord roundtrip in this read-only review.

## Solution Assessment

No, not yet. The harness boundary is the narrow maintainable place to fix this bug class, but the sanitizer needs to mirror the validator's key and depth caps before the patch is safe to merge.
