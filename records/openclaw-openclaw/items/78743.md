## What This Changes

The PR adds an empty-handler fast path for tool-result middleware, sanitizes incoming tool-result `details` before registered middleware runs, adds harness tests, and updates the changelog.

## Best Possible Solution

Keep the fix at the harness boundary, make sanitized incoming `details` always satisfy the existing validator caps, add key/depth regression coverage, and then provide redacted after-fix runtime proof.

## Reproduction Assessment

Yes for the source-level path: current main validates the unmodified result after a registered no-op middleware, and tokenjuice returns `undefined` when it leaves a result unchanged. I did not establish a fresh live Discord roundtrip in this read-only review.
