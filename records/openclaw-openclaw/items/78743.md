## Summary

Keep open: the bug is source-reproducible and the PR targets the right harness boundary, but the current sanitizer still misses existing key/depth validation caps and the external after-fix runtime proof gate is not met.

## What This Changes

The PR adds a no-middleware fast path, sanitizes incoming tool-result details before middleware handlers run, adds harness regressions, isolates two provider-registry test mocks, and updates the changelog.

## Best Possible Solution

Keep the fix at the harness boundary, make the sanitizer always produce details accepted by the existing validator caps, and attach redacted tokenjuice-enabled runtime proof before merge.

## Reproduction Assessment

Yes for a source-level reproduction: current main validates the original result after a registered no-op middleware, and tokenjuice returns undefined when unchanged. I did not run a live gateway/channel repro in this read-only review.

## Solution Assessment

No, not yet. The harness boundary is the right place to solve the bug class, but the sanitizer needs to mirror the validator's key and depth caps before the patch is safe to merge.
