## Summary

Keep open: the harness-level fix is the right direction and the source-level regression is clear, but the latest patch still lets JSON-safe incoming details that exceed the existing key/depth caps pass through the sanitizer and then fail closed, and the external PR proof still lacks after-fix real runtime evidence from this exact patch.

## What This Changes

The PR adds a harness fast path for empty tool-result middleware lists, sanitizes incoming tool-result details before registered middleware runs, adds focused regression tests, and updates the changelog.

## Best Possible Solution

Keep the fix at the harness boundary, but make sanitized incoming details always satisfy the existing validator or collapse to a bounded marker, preserve strict middleware-output validation, and add redacted exact-patch runtime proof.

## Reproduction Assessment

Yes for the source-level path: current main validates the unmodified result after a registered no-op middleware, and tokenjuice returns undefined when it leaves a result unchanged. I did not run a live Discord roundtrip in this read-only review.

## Solution Assessment

No, not yet. The harness boundary is the narrow maintainable place to fix this bug class, but the sanitizer must mirror the validator's key and depth caps before this patch is safe to merge.
