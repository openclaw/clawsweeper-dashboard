## Summary

Keep open: the reported harness bug is source-reproducible and the fix belongs at this boundary, but the current patch still lets JSON-safe details that exceed the validator's key/depth caps fall through to the failure sentinel, and external after-fix real behavior proof is still mock-only.

## What This Changes

The branch changes the agent harness tool-result middleware runner to fast-path empty handler lists, sanitize incoming result details before middleware handlers run, add harness regression tests, and update the changelog.

## Best Possible Solution

Keep the fix at the harness boundary, but make sanitized incoming details always satisfy the validator's byte, key, and depth caps, add focused regressions for those caps, then provide redacted tokenjuice-enabled runtime proof before merge.

## Reproduction Assessment

Yes for source-level reproduction: current main validates the unchanged result after any registered no-op middleware, and tokenjuice returns `undefined` when it leaves a result unchanged. I did not establish a live Discord or gateway roundtrip in this read-only review.

## Solution Assessment

No, not yet. The harness boundary is the narrow maintainable place to fix this bug class, but the sanitizer needs to mirror the validator's key and depth caps before the patch is safe to merge.
