## Summary

Keep this PR open: the harness-level fix addresses a real source-level regression, but the current sanitizer can still return details that fail the existing key/depth validator caps, and the external after-fix real-behavior proof is not yet sufficient.

## What This Changes

The PR adds a no-handler fast path for agent tool-result middleware, sanitizes incoming tool-result details before middleware runs, adds harness regression tests, and updates the changelog.

## Best Possible Solution

Keep the fix at the harness boundary, but make the sanitizer produce a projection that satisfies all existing validator caps, add key/depth regressions, and attach redacted after-fix runtime proof from a tokenjuice-enabled setup.

## Reproduction Assessment

Yes for the source-level path: current main validates the original result after a registered no-op middleware, and tokenjuice returns undefined when unchanged. I did not establish a live gateway reproduction in this read-only review.

## Solution Assessment

No, not yet. The harness boundary is the right place to solve the bug class, but the sanitizer needs to preserve the validator's key and depth caps before this patch is safe to merge.
