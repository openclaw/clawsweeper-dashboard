## Summary

I found no blocking patch findings; the remaining action is ordinary maintainer review and merge decision.

## What This Changes

The PR changes provider usage loading to prefer an explicit fetch, otherwise use the existing environment proxy-aware fetch helper before falling back to global fetch, and adds regression coverage plus a small unused MCP type removal.

## Best Possible Solution

Land this loader-level fix, or an equally narrow replacement, so provider usage preserves explicit fetch precedence and otherwise honors existing proxy env settings before using global fetch.

## Reproduction Assessment

Yes from source: current main and v2026.6.8 resolve provider usage with plain fetch, then pass that fetch into the OpenAI/Codex usage request. I did not run a live proxy-only OpenClaw status command in this read-only review.

## Solution Assessment

Yes. The provider-usage loader is the narrow owner of fetchFn fan-out; duplicating proxy handling inside the Codex hook or adding a new config knob would be broader than needed.
