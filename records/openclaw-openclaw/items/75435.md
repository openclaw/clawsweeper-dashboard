Keep this PR open. Current main still has the unguarded shared-memory `.search(...)` call, while the PR is a narrow defensive fallback with targeted regression coverage, a changelog entry, and no blocking review findings; the linked issue should close only after this PR merges.

## What This Changes

The PR adds a guarded shared-memory search helper in memory-wiki, query/tool regressions for malformed or throwing shared managers, and a changelog fix entry for the linked bug.

## Best Possible Solution

Land this PR after maintainer review so memory-wiki treats optional shared recall failures as empty memory results while preserving local wiki output; handle adjacent shared-memory session scoping work separately if that behavior still needs a follow-up.
