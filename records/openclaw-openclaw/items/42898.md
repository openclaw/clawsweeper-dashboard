## Summary

Keep open: current main still has the flat-action classifier bug, and this PR targets the right code path, but the branch still overmatches unsupported gateway actions and lacks real gateway or Telegram behavior proof.

## What This Changes

The branch updates agent tool mutation classification so gateway actions with read-only dotted leaf verbs are treated as non-mutating and adds focused gateway, cron, and canvas assertions.

## Best Possible Solution

Classify non-mutating gateway calls only when the full action is declared for the agent gateway tool and read-scoped, keep unknown and write actions fail-closed, and land with focused regression tests plus redacted live gateway or Telegram proof.

## Reproduction Assessment

Yes at source level: current main compares the full gateway action string against a flat read-only verb set, while config.schema.lookup is a read-scoped gateway method and that mutation state feeds the visible warning path. I did not run a live Telegram or gateway repro in this read-only review.

## Solution Assessment

No for this exact patch: the classifier path is the right place to fix the bug, but leaf-only matching should be constrained to full declared/read-scoped gateway tool actions and backed by real behavior proof.
