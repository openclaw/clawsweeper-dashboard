## Summary

Keep open: current main still has the reported classifier bug and this PR is useful, but the patch should narrow the gateway fallback and still needs contributor-supplied real behavior proof before merge.

## What This Changes

The PR changes `src/agents/tool-mutation.ts` so dotted gateway read-only actions are classified from their leaf verb and adds regression coverage in `src/agents/tool-mutation.test.ts` for gateway plus cron/canvas fail-closed cases.

## Best Possible Solution

Classify only recognized gateway read-only actions or method-scope entries as non-mutating, keep unsupported gateway/cron/canvas actions fail-closed, then land with focused regression tests and redacted real gateway/Telegram proof.

## Reproduction Assessment

Yes, source-level. Current main classifies `gateway` action `config.schema.lookup` as mutating even though the gateway scope table marks that method read-only and the lookup miss handler returns the reported structured error.

## Solution Assessment

No, not quite as written. The classifier-level fix is the right owner boundary, but the gateway leaf fallback should be restricted to known read-only gateway actions or scope metadata instead of any dotted action ending in a read-like verb.

## Review Findings
