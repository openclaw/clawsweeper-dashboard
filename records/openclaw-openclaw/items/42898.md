## Summary

Keep open. Current main still has the source-level bug, but this PR is not merge-ready because it lacks after-fix real behavior proof and the proposed gateway leaf fallback can suppress warnings for unsupported dotted read-like actions.

## What This Changes

The PR updates `src/agents/tool-mutation.ts` to classify dotted gateway actions by their leaf read-only verb and adds `src/agents/tool-mutation.test.ts` coverage for gateway plus cron/canvas fail-closed cases.

## Best Possible Solution

Classify only declared or read-scoped gateway methods as non-mutating, keep unsupported gateway/cron/canvas actions fail-closed, then land with focused regression tests and redacted gateway/Telegram proof.

## Reproduction Assessment

Yes. Source inspection shows current main classifies `gateway` action `config.schema.lookup` as mutating even though the gateway scope table marks that method read-only and the lookup miss handler returns the reported structured error.

## Solution Assessment

No, not quite as written. The classifier-level owner boundary is right, but the gateway fallback should match known gateway methods or scope metadata rather than any dotted action ending in a read-like leaf verb.
