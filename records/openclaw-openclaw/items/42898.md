## Summary

Keep open: the underlying bug is still source-reproducible on current main, but this PR still needs real behavior proof and its gateway leaf-verb fallback overmatches unsupported gateway actions that should stay fail-closed.

## What This Changes

The PR changes `isMutatingToolCall` so dotted gateway actions are checked by their leaf verb while keeping cron/canvas flat-action behavior, and adds focused classifier tests.

## Best Possible Solution

Classify only declared or read-scoped gateway actions such as `config.get` and `config.schema.lookup` as non-mutating, keep unknown and write-like gateway actions fail-closed, then land with focused tests and redacted real gateway or Telegram proof.

## Reproduction Assessment

Yes, at source level. Current main returns the reported `config schema path not found` lookup error while classifying `gateway` `config.schema.lookup` as mutating, and the payload warning policy consumes that mutation state for visible warnings.

## Solution Assessment

No, not as written. The PR should match declared or read-scoped gateway actions instead of trusting any unsupported dotted action whose leaf looks read-only.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9
