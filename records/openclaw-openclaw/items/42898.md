## Summary

Keep open. The PR targets a real current-main bug, but the head still overmatches unsupported gateway actions by leaf verb and lacks real gateway or Telegram behavior proof, so it should be revised and proven before merge.

## What This Changes

The branch changes agent tool mutation classification so dotted gateway actions with read-only leaf verbs are treated as non-mutating and adds focused mutation-helper tests.

## Best Possible Solution

Classify non-mutating gateway calls only for declared read-scoped full gateway actions, keep unknown and write-like actions fail-closed, and land with regression tests plus redacted live gateway or Telegram proof.

## Reproduction Assessment

Yes, at source level: current main checks the full action config.schema.lookup against a flat read-only verb set, the gateway descriptor marks it operator.read, and mutatingAction feeds the visible warning policy. I did not run a live Telegram or gateway repro in this read-only review.

## Solution Assessment

No for this exact patch: the classifier is the right boundary, but leaf-only gateway matching should be constrained to declared read-scoped gateway actions and backed by real behavior proof.
