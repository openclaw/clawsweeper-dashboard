## Summary

Keep open. The underlying bug is valid and source-reproducible on current main, but this PR still overmatches unsupported gateway actions by read-like leaf verb and lacks required after-fix real behavior proof.

## What This Changes

The PR changes the agent tool mutation classifier so dotted gateway actions are checked by leaf verb, and adds focused regression tests for gateway, cron, and canvas action classification.

## Best Possible Solution

Classify only declared or read-scoped gateway actions such as `config.get` and `config.schema.lookup` as non-mutating, keep unknown and write-like gateway actions fail-closed, then land with focused regression coverage and redacted real behavior proof.

## Reproduction Assessment

Yes, at source level. Current main treats `gateway` `config.schema.lookup` as mutating even though the gateway descriptor marks it read-only, and the payload warning policy consumes that mutation state for visible failed-tool warnings.

## Solution Assessment

No, not as written. The PR should match declared or read-scoped gateway actions instead of trusting any unsupported dotted action whose leaf looks read-only.

## Review Findings

Overall correctness: patch is incorrect
