## Summary

Keep open. Current main still has the linked classifier bug, but this PR is not merge-ready because the gateway leaf fallback also downgrades unsupported read-like gateway actions and the PR lacks real Telegram/gateway behavior proof.

## What This Changes

The PR changes the agent tool mutation classifier and tests so dotted gateway actions such as `config.schema.lookup` are classified by read-only leaf verbs while cron and canvas stay flat-action fail-closed.

## Best Possible Solution

Classify only declared or read-scoped gateway actions as non-mutating, keep unknown and write-like gateway actions fail-closed, and land with focused tests plus redacted real gateway or Telegram proof.

## Reproduction Assessment

Yes, at source level. Current main returns the reported missing-path error, classifies the dotted `config.schema.lookup` gateway action as mutating, and feeds that mutation state into the visible warning payload path.

## Solution Assessment

No, not as written. The fix should classify declared/read-scoped gateway actions rather than trusting any unsupported dotted action whose final segment looks read-only.
