## Summary

Keep this PR open for maintainer review. Current main still misclassifies dotted gateway read actions as mutating, while the latest provided PR diff addresses the bug narrowly and the prior review comments about cron/canvas fail-closed behavior have been handled.

## What This Changes

The PR changes `src/agents/tool-mutation.ts` to treat gateway dotted read leaf verbs as non-mutating while leaving cron/canvas fail-closed, and adds regression tests in `src/agents/tool-mutation.test.ts`.

## Best Possible Solution

Land this classifier fix or an equivalent exact gateway read-action implementation with regression coverage, then coordinate with #42962 and close #42856 only after the user-visible gateway lookup warning behavior ships.

## Reproduction Assessment

Yes. A high-confidence unit-level reproduction exists on current main: `isMutatingToolCall("gateway", { action: "config.schema.lookup" })` reaches the flat `READ_ONLY_ACTIONS` check and returns mutating even though the gateway method scope marks that method read-only.

## Solution Assessment

Yes. The latest provided PR diff is a narrow maintainable fix for the classifier bug and preserves cron/canvas fail-closed behavior; an exact gateway read-method whitelist would be stricter but is a maintainer preference, not a clear correctness blocker.
