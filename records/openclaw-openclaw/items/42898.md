## Summary

Keep open: the underlying bug is source-provable, but this branch still loosens the gateway classifier for unsupported dotted actions and lacks contributor after-fix real behavior proof.

## What This Changes

The PR updates the agent tool mutation classifier and tests so dotted gateway actions with read-only leaf verbs, especially `config.schema.lookup`, are treated as non-mutating while cron/canvas stay fail-closed.

## Best Possible Solution

Classify only declared or read-scoped gateway actions as non-mutating, keep unsupported gateway and flat cron/canvas actions fail-closed, then land with focused regression coverage and redacted real gateway or Telegram proof.

## Reproduction Assessment

Yes, at source level. Current `main` classifies `gateway` `config.schema.lookup` as mutating because it checks only the full action string against a flat read-only set, while the gateway descriptor marks that method `operator.read` and the warning policy consumes the mutation bit.

## Solution Assessment

No, not as written. The classifier boundary is right, but the PR should match declared or read-scoped gateway methods instead of trusting any read-like leaf segment.
