## Summary

Keep open: current main still has the reported classifier bug, but the PR head still overmatches unsupported dotted gateway actions and lacks after-fix real behavior proof.

## What This Changes

The PR updates the agent mutation classifier and tests so dotted gateway read-only actions such as `config.schema.lookup` are treated as non-mutating while cron and canvas stay fail-closed.

## Best Possible Solution

Classify only declared or read-scoped gateway methods as non-mutating, keep unsupported gateway/cron/canvas actions fail-closed, and land with focused regression coverage plus redacted gateway or Telegram proof.

## Reproduction Assessment

Yes, at source level. Current main returns `config schema path not found` for a missing `config.schema.lookup` path, classifies the dotted gateway action as mutating, and the failed-tool path can render mutating errors as warnings.

## Solution Assessment

No, not quite as written. The classifier boundary is right, but the PR should match declared or read-scoped gateway methods instead of accepting any dotted gateway action whose leaf is read-like.

## Review Findings
