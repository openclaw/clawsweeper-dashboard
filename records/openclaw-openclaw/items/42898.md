## Summary

Keep open. The PR targets a valid user-visible bug and fixes the current main classifier directionally, but it still needs real behavior proof and the patch should fail closed for unsupported dotted gateway actions instead of trusting any read-only leaf verb.

## What This Changes

The PR updates the agent tool-mutation classifier and tests so dotted gateway actions with read-only leaf verbs are treated as non-mutating while cron and canvas unsupported actions remain fail-closed.

## Best Possible Solution

Constrain gateway mutation classification to explicit supported read-only gateway actions or gateway scope semantics, keep unsupported gateway actions fail-closed, add focused regression coverage, and provide redacted runtime or Telegram proof.

## Reproduction Assessment

Yes. Source inspection shows current main classifies `gateway` action `config.schema.lookup` as mutating even though the gateway scope table marks it read-only and the handler can return the reported `config schema path not found` error.

## Solution Assessment

No, not as written. A classifier-level fix is the right direction, but the current leaf-verb fallback should be narrowed so invalid gateway actions still fail closed, and the contributor still needs real after-fix proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Fail closed for unknown gateway actions:** `src/agents/tool-mutation.ts:137`
  - body: The gateway fallback now treats any dotted action ending in a read-only verb, or `lookup`, as non-mutating. The gateway tool only supports a small explicit action enum and throws on unknown actions, so an invalid call such as `something.lookup` can fail execution while bypassing the mutating warning path when the assistant also replies. Please classify the known read-only gateway actions explicitly, or use the gateway scope contract, so unsupported actions remain fail-closed.
  - confidence: 0.86
