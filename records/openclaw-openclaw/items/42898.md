## Summary

## What This Changes

The PR updates `src/agents/tool-mutation.ts` and tests so gateway compound read-only actions with leaf verbs like `lookup`, `get`, and `list` are non-mutating while cron/canvas dotted or unsupported actions stay fail-closed.

## Best Possible Solution

Land this classifier-level fix after contributor-supplied real behavior proof, keeping cron/canvas fail-closed and focused regression coverage.

## Reproduction Assessment

Yes, at source level. Current main classifies `gateway` action `config.schema.lookup` as mutating while the gateway scope contract marks that method read-only, and the lookup handler returns the reported `config schema path not found` error for misses.

## Solution Assessment

Yes. Updating the mutation classifier is the narrow maintainable fix because that state directly feeds warning rendering, and the PR’s final version keeps cron/canvas unsupported dotted actions fail-closed.
