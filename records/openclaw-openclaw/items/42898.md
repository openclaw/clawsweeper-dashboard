## Reproduction Assessment

Yes, at source level: current main classifies `gateway` plus `config.schema.lookup` as mutating because it checks the full dotted action against a flat read-only verb set. I did not run a live Telegram or gateway repro in this read-only review.

## Solution Assessment

No for this exact patch: the classifier is the right boundary, but matching only the dotted leaf verb is too broad. The safer fix is to use the gateway method scope contract or an exact read-only action set so unknown actions remain fail-closed.
