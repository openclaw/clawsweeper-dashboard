## Summary

Keep open. The PR targets a real source-reproducible bug, but the current patch still overmatches unsupported gateway actions and lacks real behavior proof, so it should not merge as-is.

## What This Changes

The PR changes agent tool mutation classification so dotted gateway actions with read-only leaf verbs are treated as non-mutating, while cron and canvas dotted actions remain fail-closed.

## Best Possible Solution

Classify only declared or read-scoped gateway actions as non-mutating, keep unknown and write-like gateway actions fail-closed, and land with focused tests plus redacted real gateway or Telegram proof.

## Reproduction Assessment

Yes, at source level. Current main classifies config.schema.lookup as mutating even though the gateway descriptor marks it operator.read, and that mutation state feeds the visible failed-tool warning path.

## Solution Assessment

No for this exact patch. The right fix is still the classifier path, but it should use declared or read-scoped gateway actions instead of trusting any read-like dotted leaf.
