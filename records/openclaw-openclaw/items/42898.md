## Summary

Keep open. The underlying bug is real and this PR is the active linked fix, but the current diff still classifies unknown dotted gateway actions by leaf verb and lacks real gateway or Telegram behavior proof, so it should be revised and proven before merge.

## What This Changes

The PR changes the agent tool mutation classifier to treat gateway dotted actions with read-only leaf verbs as non-mutating and adds focused tests for gateway, cron, and canvas classification.

## Best Possible Solution

Classify gateway calls as non-mutating only when the full action is a declared read-scoped gateway method, keep unknown actions fail-closed, and add redacted real behavior proof for the user-visible warning path.

## Reproduction Assessment

Yes, at source level. Current main checks the full `config.schema.lookup` action against a flat read-only verb set, the gateway descriptor marks that method read-only, and `mutatingAction` feeds the visible warning policy.

## Solution Assessment

No for this exact patch. The classifier is the right boundary, but the maintainable fix should key off declared read-scoped gateway actions instead of leaf verbs and should include real gateway or Telegram proof.
