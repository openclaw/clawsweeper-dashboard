## Summary

Keep open. The reported bug is still present on current main and this PR targets the right classifier path, but the current diff overmatches unsupported gateway actions and still lacks after-fix real behavior proof.

## What This Changes

The PR changes the agent mutation classifier and tests so dotted gateway actions are checked by their leaf verb, making `config.schema.lookup` non-mutating while keeping cron/canvas dotted actions fail-closed.

## Best Possible Solution

Classify only accepted or declared read-scoped gateway actions such as `config.get` and `config.schema.lookup` as non-mutating, keep unknown/write-like gateway actions fail-closed, then land with focused regression coverage and redacted real behavior proof.

## Reproduction Assessment

Yes, at source level. Current main treats `gateway` `config.schema.lookup` as mutating even though the gateway descriptor marks it read-only, and the warning policy consumes `mutatingAction` for user-facing failed-tool warnings.

## Solution Assessment

No, not as written. The PR should match accepted or read-scoped gateway actions instead of trusting any read-like dotted leaf segment.
