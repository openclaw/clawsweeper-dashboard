## Summary

Keep open. This PR targets a real current-main bug, but the branch is not merge-ready because it classifies unsupported dotted gateway actions by leaf verb and still lacks real gateway or Telegram behavior proof.

## What This Changes

The PR changes the agent tool mutation classifier to treat gateway dotted actions with read-like leaf verbs as non-mutating and adds focused mutation tests.

## Best Possible Solution

Classify gateway actions from the exact supported read-scope contract, or from a small exact read-only gateway action set, so unknown actions fail closed before adding regression coverage and redacted real gateway or Telegram proof.

## Reproduction Assessment

Yes, at source level: current main checks `config.schema.lookup` against a flat read-only verb set and therefore marks it mutating, while the gateway descriptor marks the method as `operator.read`. I did not run live Telegram or gateway proof in this read-only pass.

## Solution Assessment

No for this patch: the classifier is the right boundary, but matching by leaf verb is broader than the gateway action contract and lets unsupported dotted actions fail open. Use exact read-scoped gateway action knowledge and leave unknowns mutating.
