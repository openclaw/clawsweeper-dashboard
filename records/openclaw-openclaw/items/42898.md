## What This Changes

The PR changes `isMutatingToolCall` to classify gateway dotted actions by their leaf verb while keeping cron/canvas flat actions fail-closed, and adds focused mutation-helper tests.

## Best Possible Solution

Replace leaf-verb matching with exact gateway read-method or allowed-action classification, keep unknown gateway actions mutating/fail-closed, and require redacted real gateway or Telegram proof before merge.

## Reproduction Assessment

Yes, at source level. Current main marks `gateway` plus `config.schema.lookup` as mutating because it checks the full dotted action against a flat read-only set, and that flag feeds the visible failed-tool warning policy.

## Solution Assessment

No for this patch. The classifier is the right boundary, but the current leaf-only gateway fallback is broader than the gateway method-scope contract and should be narrowed before merge.
