## Summary

Keep open: the PR targets a real source-reproducible bug, but the latest patch still overmatches unsupported gateway actions by leaf verb and lacks after-fix real behavior proof for the user-visible warning flow.

## What This Changes

The PR changes the agent tool mutation classifier and tests so dotted gateway actions are classified by read-only leaf verbs while cron and canvas remain flat-action fail-closed.

## Best Possible Solution

Classify only declared or read-scoped gateway actions as non-mutating, keep unknown and write-like gateway actions fail-closed, then land with focused tests plus redacted real gateway or Telegram proof.

## Reproduction Assessment

Yes, at source level. Current main classifies `gateway` `config.schema.lookup` as mutating even though the gateway descriptor marks it read-only, and the warning policy consumes that mutation state for visible failure warnings.

## Solution Assessment

No, not as written. The fix should match declared or read-scoped gateway actions rather than trusting any unsupported dotted action whose final segment looks read-only.
