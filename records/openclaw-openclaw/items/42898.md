## Summary

Keep open. Current main still has the reported classifier bug, and this PR targets the right area, but the patch still overmatches unsupported dotted gateway actions and lacks required real behavior proof.

## What This Changes

The PR updates `src/agents/tool-mutation.ts` and its tests so dotted gateway actions such as `config.schema.lookup` are treated as non-mutating while cron and canvas remain fail-closed.

## Best Possible Solution

Classify only declared or read-scoped gateway tool actions as non-mutating, keep unsupported gateway/cron/canvas actions fail-closed, and land with focused regression coverage plus redacted gateway or Telegram proof.

## Reproduction Assessment

Yes, at source level. A `gateway` call with `action: "config.schema.lookup"` and a missing path returns `config schema path not found`, but current main classifies that dotted action as mutating before warning rendering.

## Solution Assessment

No, not quite as written. The classifier-level fix is the right boundary, but the PR should match known gateway actions or scope metadata instead of accepting any dotted action with a read-like leaf verb.

## Review Findings

Overall correctness: patch is incorrect
