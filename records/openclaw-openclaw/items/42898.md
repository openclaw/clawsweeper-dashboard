## Summary

Keep open. The PR targets a real source-level bug, but the current patch still overmatches unsupported gateway actions ending in read-like verbs and lacks after-fix real behavior proof for the visible warning flow.

## What This Changes

The PR changes the agent tool mutation classifier and tests so gateway dotted read-only actions such as `config.schema.lookup` are treated as non-mutating while cron/canvas remain fail-closed.

## Best Possible Solution

Classify only declared or read-scoped gateway methods as non-mutating, keep unsupported gateway and flat cron/canvas actions fail-closed, and land with focused regression coverage plus redacted real gateway or Telegram proof.

## Reproduction Assessment

Yes, at source level. Current main classifies `gateway` `config.schema.lookup` as mutating even though the gateway descriptor marks it read-only, and that mutation state reaches the warning-rendering path.

## Solution Assessment

No, not as written. The classifier boundary is right, but the PR should match declared or read-scoped gateway methods instead of trusting any read-like leaf segment.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

That failed execution would lose the warning path; match declared or read-scoped gateway actions instead of any read-like leaf.
