## Summary

Keep open. The reported bug is still source-reproducible on current main and this PR targets the right classifier, but the PR head still overmatches unsupported gateway actions by leaf verb and lacks after-fix real gateway or Telegram proof.

## What This Changes

The PR updates `src/agents/tool-mutation.ts` and regression tests so dotted gateway actions are classified by their leaf verb, making `config.schema.lookup` non-mutating while keeping cron/canvas dotted actions fail-closed.

## Best Possible Solution

Classify only accepted or declared read-scoped gateway actions such as `config.get` and `config.schema.lookup` as non-mutating, keep unknown/write-like gateway actions fail-closed, then land with focused regression coverage and redacted real behavior proof.

## Reproduction Assessment

Yes, at source level. Current main treats `gateway` `config.schema.lookup` as mutating even though the gateway descriptor marks it read-only and the warning policy consumes `mutatingAction`.

## Solution Assessment

No, not as written. The PR should match accepted or read-scoped gateway actions instead of trusting any read-like dotted leaf segment.
