## Summary

Keep open: current main covers much of the model-attribution request through `messages.responsePrefix` templates, but it still lacks any config or runtime approval gate before fallback execution, and recent discussion confirms that approval gate is the remaining core gap.

## What This Changes

Request opt-in confirmation before model fallback plus optional visible provider/model attribution in replies.

## Best Possible Solution

Keep this as the canonical core design tracker for per-agent fallback approval, use `messages.responsePrefix` for message attribution, and leave usage-footer-specific display to https://github.com/openclaw/openclaw/issues/65477.

## Reproduction Assessment

Not applicable. This is an enhancement request; source inspection on current main shows no fallback approval config field or approval hook, while response-prefix attribution support exists.

## Solution Assessment

Unclear as an implementation path. The attribution portion already has a maintainable prefix mechanism, but fallback approval needs maintainer decisions on schema, UX, timeout, and denial semantics before a narrow patch is appropriate.
