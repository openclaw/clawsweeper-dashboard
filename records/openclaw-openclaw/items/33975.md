## Summary

Keep open: current main covers most of the model-attribution need through `messages.responsePrefix` templates, but the requested opt-in approval gate before model fallback is still absent and requires maintainer product/config decisions.

## What This Changes

Request optional configuration to ask before switching to a fallback model and surface the actual provider/model in outbound replies.

## Best Possible Solution

Keep this as the canonical core design tracker for opt-in per-agent fallback approval; use `messages.responsePrefix` for reply attribution and leave usage-footer placement to https://github.com/openclaw/openclaw/issues/65477.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that response-prefix attribution exists while fallback approval has no schema field, runtime type, or fallback-loop hook.

## Solution Assessment

Unclear as an implementation path. The attribution portion already has a maintainable prefix mechanism, but fallback approval needs maintainer decisions on schema shape, channel prompt routing, timeout, denial, and proceed/fail semantics before a narrow patch is safe.
