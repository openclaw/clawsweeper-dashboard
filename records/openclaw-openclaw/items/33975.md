## Summary

Keep open: current main covers much of the model-attribution request through `messages.responsePrefix` template variables, but the fallback-approval gate is still absent and recent discussion confirms that remains the core gap.

## What This Changes

Requests opt-in approval before switching to a fallback model plus optional model/provider attribution in outbound messages.

## Best Possible Solution

Keep this as the core design tracker for per-agent fallback approval, while using `messages.responsePrefix` for model display and leaving usage-footer-specific display to #65477.

## Reproduction Assessment

Not applicable. This is an enhancement request; source inspection shows current main lacks an approval gate while existing response-prefix support covers much of the attribution side.

## Solution Assessment

Unclear as an implementation path. The attribution portion already has a maintainable prefix mechanism, but fallback approval needs maintainer decisions on config shape and UX before a narrow patch is appropriate.
