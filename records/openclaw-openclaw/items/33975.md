## Summary

Keep open: current main covers much of the message attribution request through `messages.responsePrefix` templates and fallback notices, but it still lacks an opt-in approval gate before automatic model fallback.

## What This Changes

Requests opt-in confirmation before switching to a fallback model and optional provider/model attribution in outbound messages.

## Best Possible Solution

Keep this as the canonical core tracker for opt-in per-agent fallback approval, preserve default automatic fallback, use response-prefix templates for general attribution, and leave usage-footer placement to https://github.com/openclaw/openclaw/issues/65477.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing contract. Source inspection gives high confidence that attribution exists through response-prefix templates while fallback approval has no config field, runtime type, or fallback-loop approval hook on current main.

## Solution Assessment

Unclear until maintainers decide the public API and cross-channel approval semantics. The narrow maintainable path is an opt-in per-agent fallback approval policy that preserves default automatic fallback behavior and reuses existing approval delivery where appropriate.
