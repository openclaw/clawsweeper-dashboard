## Summary

Keep open. Current main covers the model-attribution half through `messages.responsePrefix` templates, but the requested approval gate before using a fallback model is still absent and needs maintainer product/config decisions.

## What This Changes

Requests opt-in confirmation before model fallback and optional model/provider attribution in outgoing messages.

## Best Possible Solution

Keep this as the canonical core design tracker for opt-in per-agent fallback approval; use `messages.responsePrefix` for reply attribution and leave usage-footer placement to https://github.com/openclaw/openclaw/issues/65477.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that response-prefix attribution exists while fallback approval has no schema field, runtime type, or fallback-loop hook.

## Solution Assessment

Unclear as an implementation path. The attribution portion already has a maintainable prefix mechanism, but fallback approval needs maintainer decisions on schema, channel prompting, timeout, denial, and proceed/fail semantics before a narrow patch is safe.
