## Summary

Keep open. Current main and the latest release cover the model-attribution half through `messages.responsePrefix` template variables, but the requested opt-in approval gate before executing a fallback model is still absent and requires maintainer product/API direction.

## What This Changes

Request an opt-in model fallback approval mode plus visible model attribution in outgoing messages.

## Best Possible Solution

Keep this as the canonical core tracker for opt-in per-agent fallback approval, while using `messages.responsePrefix` for current message attribution and https://github.com/openclaw/openclaw/issues/65477 for usage-footer placement.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that attribution is available through `messages.responsePrefix`, while fallback approval has no config field, runtime type, or approval hook on current main or the latest release.

## Solution Assessment

Unclear until maintainers decide the public config and cross-channel approval semantics. If accepted, the narrow maintainable path is an opt-in per-agent fallback approval policy that preserves default automatic fallback behavior and reuses existing approval delivery where appropriate.
