## Summary

Keep open. Current main covers most model attribution needs through `messages.responsePrefix` templates and related UI/display work, but the requested opt-in approval gate before model fallback is still absent and needs maintainer product/config decisions.

## What This Changes

The issue requests opt-in confirmation before switching to fallback models plus optional visible model attribution on outgoing replies.

## Best Possible Solution

Keep this as the canonical core tracker for opt-in per-agent fallback approval; use `messages.responsePrefix` for reply attribution and leave usage-footer placement to https://github.com/openclaw/openclaw/issues/65477.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that attribution can be handled with `messages.responsePrefix`, while fallback approval has no config field, runtime type, or fallback-loop approval hook on current main.

## Solution Assessment

Unclear until maintainers decide the public config and cross-channel approval semantics. The maintainable path is a narrow opt-in fallback approval policy after timeout, denial, and delivery behavior are agreed.
