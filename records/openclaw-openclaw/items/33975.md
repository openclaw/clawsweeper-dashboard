## Summary

Keep open. Current main covers the model-attribution half through `messages.responsePrefix` template variables, but it still lacks an opt-in approval gate before fallback model execution, and the latest discussion confirms that is the remaining requested behavior.

## What This Changes

Request optional user approval before switching to fallback models and optional visible model attribution on responses.

## Best Possible Solution

Keep this as the canonical fallback-approval tracker, preserve `messages.responsePrefix` as the current attribution path, and add a documented opt-in per-agent approval policy only after maintainers settle the lifecycle semantics.

## Reproduction Assessment

Not applicable as a feature request. The source-level verification is high confidence: current schema/type/runtime paths have no fallback approval policy, while attribution is already available through response-prefix templates.

## Solution Assessment

Unclear. A per-agent opt-in flag looks like the right boundary, but the exact approval UI, timeout behavior, and fallback execution policy need maintainer product review before implementation.
