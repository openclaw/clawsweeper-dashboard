## Summary

Keep open. Current main covers the model-attribution half through configurable response-prefix templates, but the fallback-approval half remains absent and needs a public config and cross-channel confirmation decision.

## What This Changes

The issue requests an opt-in approval prompt before model fallback plus optional model attribution in delivered messages.

## Best Possible Solution

Keep this as the canonical core tracker for opt-in per-agent fallback approval, while using `messages.responsePrefix` for current message-level attribution and `https://github.com/openclaw/openclaw/issues/65477` for usage-footer placement.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that attribution is available through `messages.responsePrefix`, while fallback approval has no config field, runtime type, or approval hook on current main.

## Solution Assessment

Unclear until maintainers decide the public config and cross-channel approval semantics. If accepted, the narrow maintainable path is an opt-in per-agent fallback approval policy that reuses existing approval delivery where appropriate without changing default automatic fallback behavior.
