## Summary

Keep open: current main covers model visibility through `messages.responsePrefix` template variables and fallback notices, but it still lacks the requested opt-in approval gate before executing a fallback model.

## What This Changes

The issue requests opt-in fallback approval before model switching plus visible model attribution in outbound messages.

## Best Possible Solution

Keep the existing response-prefix attribution path, and only add fallback approval after maintainers define the per-agent config shape plus approval delivery, timeout, and resume semantics.

## Reproduction Assessment

Not applicable as a bug reproduction, but the source path is clear. Current schema and fallback runtime show attribution support exists while confirm-before-fallback behavior is absent.

## Solution Assessment

Unclear until maintainers decide the product shape. The attribution half is best handled by `messages.responsePrefix`; the fallback approval half needs a deliberate core design rather than a narrow automatic fix.
