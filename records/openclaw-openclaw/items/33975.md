## Summary

Keep open. Current main covers visible model attribution through response-prefix templates and fallback notices, but it still lacks an opt-in approval gate before automatic model fallback; related searches show this remains the canonical tracker for that remaining product decision.

## What This Changes

The issue requests opt-in approval before model fallback plus optional provider/model attribution in outgoing messages.

## Best Possible Solution

Keep this as the canonical core tracker for opt-in per-agent fallback approval, preserve default automatic fallback, use existing response-prefix templates for general attribution, and leave usage-footer placement to https://github.com/openclaw/openclaw/issues/65477.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing contract. Source inspection gives high confidence that attribution exists through response-prefix templates while fallback approval has no config field, runtime type, or fallback-loop approval hook on current main.

## Solution Assessment

Unclear until maintainers decide the public API and cross-channel approval semantics. The narrow maintainable path is an opt-in per-agent fallback approval policy that preserves default automatic fallback behavior and reuses existing approval delivery where appropriate.
