## Summary

Keep open: current main and the latest release cover much of the model-attribution need through `messages.responsePrefix` template variables, but they still lack an opt-in approval checkpoint before executing fallback model candidates.

## What This Changes

The issue requests an optional approval mode before model fallback plus visible provider/model attribution in outbound messages.

## Best Possible Solution

Decide the fallback-approval product shape first; if accepted, add a current-schema per-agent approval mode in the model fallback owner with docs and focused fallback-loop coverage, while using `messages.responsePrefix` for attribution today.

## Reproduction Assessment

Not applicable as a bug repro, but source inspection gives a high-confidence way to verify the missing capability: current schema and fallback execution lack an approval-before-fallback field or checkpoint.

## Solution Assessment

Unclear: a per-agent opt-in approval flag is plausible, but the best solution needs maintainer choice on approval owner, channel UX, timeout policy, and resume semantics before implementation.
