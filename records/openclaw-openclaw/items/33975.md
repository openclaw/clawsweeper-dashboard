## Summary

Keep open: current main covers much of the model-attribution request through `messages.responsePrefix` template variables and usage footer model fields, but it still lacks an opt-in approval checkpoint before fallback model execution.

## What This Changes

Request an optional fallback-approval mode before switching models plus optional model attribution in user-visible replies.

## Best Possible Solution

Decide the fallback-approval product/API shape first; if accepted, add a current-schema per-agent approval mode in the model fallback owner with docs and focused fallback-loop coverage, while using `messages.responsePrefix` for attribution today.

## Reproduction Assessment

Not applicable as a bug repro, but source inspection gives a high-confidence verification path: current schema and fallback execution lack an approval-before-fallback field or checkpoint.

## Solution Assessment

Unclear: a per-agent opt-in approval flag is plausible, but the best solution needs maintainer choice on approval routing, timeout policy, resume semantics, and whether the extra config surface is justified.
