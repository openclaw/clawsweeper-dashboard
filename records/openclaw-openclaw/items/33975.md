## Summary

Keep open. Current main covers the model-attribution side through `messages.responsePrefix` template variables and fallback notices, but it still lacks an opt-in approval gate before executing a fallback model, which requires public config and cross-channel behavior decisions.

## What This Changes

Request adds opt-in approval before model fallback and optional model attribution in outbound messages.

## Best Possible Solution

Keep this as the canonical core tracker for opt-in per-agent fallback approval, while using `messages.responsePrefix` for current model attribution and https://github.com/openclaw/openclaw/issues/65477 for usage-footer placement.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection gives high confidence that attribution exists through `messages.responsePrefix`, while fallback approval has no config field, runtime type, or fallback-loop hook on current main or `v2026.5.12`.

## Solution Assessment

Unclear until maintainers decide the public API and cross-channel approval semantics. If accepted, the narrow maintainable path is an opt-in per-agent fallback approval policy that preserves default automatic fallback behavior and reuses existing approval delivery where appropriate.
