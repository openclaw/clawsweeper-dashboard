## Summary

Current OpenClaw does not define an `overwatch-daemon.cjs` or `engage` CLI contract; core already provides background task tracking, audit, maintenance, cron, and Gateway-owned sweeps, while the requested Facebook engagement workflow is optional service-specific automation that belongs as a ClawHub plugin or skill.

## What This Changes

The issue asks to restore an installed `overwatch-daemon.cjs` and `engage` workflow for autonomous Facebook engagement, stalled-task cleanup, and guardrail enforcement.

## Best Possible Solution

Keep core task management on the documented `openclaw tasks`, cron, and Gateway maintenance paths; build any Facebook `engage` or Overwatch-style workflow as an external ClawHub plugin or skill unless a concrete missing plugin API is identified.

## Reproduction Assessment

Not applicable as a bug reproduction: the reported daemon and `engage` command are not current OpenClaw source or docs contracts. Source inspection instead confirms the supported task-management paths are `openclaw tasks`, cron, and Gateway maintenance.

## Solution Assessment

No, restoring a standalone core Overwatch daemon is not the best way to solve this. The maintainable path is to use existing core task/cron primitives and publish service-specific engagement behavior through ClawHub unless a focused plugin API gap is proven.
