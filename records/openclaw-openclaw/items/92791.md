## Summary

Keep open: the PR targets the right Agent Teams lifecycle bug and includes useful proof, but the new terminal-result predicate can still treat visible pre-tool assistant text from a `toolUse` turn as a completed child result.

## What This Changes

The PR adds a strict child-transcript terminal-result reader and uses it in the subagent sweeper lost-context branch, with unit and registry tests for terminal output, tool-only output, and `sessions_yield` waits.

## Best Possible Solution

Keep the lost-context recovery, but make the terminal predicate reject any latest assistant turn that still represents tool use or pending continuation unless a later terminal no-tool-call assistant result exists.

## Reproduction Assessment

Yes, source-level: the lost-context sweeper now reads child transcript history and recovers as ok when the new predicate returns text. A normal transcript shape with visible assistant text plus a non-yield tool call would pass that predicate even though current incomplete-turn code treats it as non-terminal.

## Solution Assessment

No. The recovery shape is close, but the predicate must distinguish displayable pre-tool progress from a genuine terminal child result before this is the best fix.
