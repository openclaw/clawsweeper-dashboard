## Summary

The PR is a plausible fix for the Matrix case-loss bug, but it should not merge yet because the new legacy-alias cleanup can delete a separate valid Matrix session whose room ID differs only by case. The existing bot review flagged the same concrete defect, and source inspection confirms `persistResolvedSessionEntry` deletes every collected `legacyKey`.

## What This Changes

The PR adds a case-preserving session-key registry for Matrix channel/group and Signal group peers, updates session construction/store lookup/gateway explicit-key handling, and adds session-key regression tests.

## Best Possible Solution

Keep the generic case-preservation direction, but make legacy cleanup distinguish true pre-fix folded aliases from separate exact Matrix room sessions and add a regression that updates one of two case-distinct rooms without deleting the other.

## Reproduction Assessment

Yes: source inspection gives a high-confidence reproduction path for the PR defect by placing two Matrix session keys in the store whose room IDs differ only by case, then updating the mixed-case key so the folded sibling is collected in `legacyKeys` and deleted during persistence.

## Solution Assessment

No: the case-preservation registry is a reasonable direction, but this patch is not the best final form until legacy alias collapse stops treating separate case-distinct Matrix room IDs as deletable aliases.
