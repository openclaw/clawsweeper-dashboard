## Summary

Keep open: current main still has a source-level identity mismatch where Copilot token exchange omits `Copilot-Integration-Id` while downstream Copilot requests and catalog refreshes identify as `vscode-chat`.

## What This Changes

The issue reports GitHub Copilot image-tool requests failing with a 400 model-availability error because the minted Copilot API token appears scoped to `copilot-language-server` instead of `vscode-chat`.

## Best Possible Solution

Keep the repair in the GitHub Copilot provider/auth boundary: mint Copilot API tokens with the same `vscode-chat` identity used for downstream requests, refresh stale affected cache entries if needed, and add focused regression coverage.

## Reproduction Assessment

Yes for source-level reproduction: current main exchanges the Copilot token without `Copilot-Integration-Id` while runtime Copilot requests add `vscode-chat`. I did not run a live Copilot image-tool request in this read-only review.

## Solution Assessment

Yes: aligning the token exchange identity is the narrowest maintainable first fix. Model pre-detection or fallback selection should remain secondary unless live validation proves entitlement still differs after the token identity is corrected.
