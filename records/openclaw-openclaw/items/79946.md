## Summary

Keep open: current main still has the reported Copilot auth mismatch, and https://github.com/openclaw/openclaw/pull/79976 is an open closing PR rather than a merged fix.

## What This Changes

GitHub Copilot image/tool requests should mint Copilot API tokens with the same `vscode-chat` integration identity used by downstream requests so image-capable model calls do not fail with a 400.

## Best Possible Solution

Land a narrow GitHub Copilot provider-auth fix, likely via https://github.com/openclaw/openclaw/pull/79976 after review, that adds `Copilot-Integration-Id: vscode-chat` to token exchange, updates regression coverage, and validates live/cached-token behavior.

## Reproduction Assessment

Yes for source-level reproduction: current main mints the Copilot API token without `Copilot-Integration-Id` while later Copilot model and runtime requests send `vscode-chat`. I did not run a live Copilot image/tool request in this review.

## Solution Assessment

Yes: fixing the token exchange identity is the narrowest maintainable repair in the provider-auth boundary. Model preselection should stay a fallback only if live validation proves token identity is not enough.
