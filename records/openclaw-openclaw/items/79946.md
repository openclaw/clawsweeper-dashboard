## Summary

Keep open: current `main` still shows a source-level mismatch where Copilot runtime requests identify as `vscode-chat`, but the token exchange that mints the Copilot API token does not send that integration id.

## What This Changes

Make GitHub Copilot image-tool requests avoid the reported model/integrator mismatch by aligning the token exchange identity with the downstream `vscode-chat` Copilot request identity or by preflighting unsupported models.

## Best Possible Solution

Keep this in the GitHub Copilot provider/auth boundary: mint Copilot API tokens with the same `vscode-chat` identity used for downstream requests, add regression coverage, and only add model preselection fallback if live validation still proves model entitlement can differ.

## Reproduction Assessment

Yes for source-level reproduction: current main exchanges the Copilot token without `Copilot-Integration-Id` while runtime Copilot requests add `vscode-chat`. I did not run a live Copilot image-tool request in this read-only review.

## Solution Assessment

Yes: fixing the token exchange identity is the narrowest maintainable first repair. Pre-detecting model availability should remain a fallback only if live validation proves the token identity cannot solve the mismatch.
