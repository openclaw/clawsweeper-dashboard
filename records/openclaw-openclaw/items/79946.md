## Summary

Keep open: current main still has the reported Copilot token/request identity mismatch, and https://github.com/openclaw/openclaw/pull/79976 is an open closing PR rather than a merged fix.

## What This Changes

GitHub Copilot image-tool requests should either succeed with `github-copilot/gemini-3.1-pro-preview` or avoid sending a request with a Copilot token scoped to the wrong integrator.

## Best Possible Solution

Review and land the linked provider-auth fix, with test coverage for the token-exchange headers and redacted live proof that the token exchange or image-model request no longer reports the wrong integrator.

## Reproduction Assessment

Yes for source-level reproduction: current main omits `Copilot-Integration-Id` during `/v2/token` exchange while later Copilot requests send `vscode-chat`, and the reporter supplied the corresponding 400 error. I did not rerun a live Copilot subscription flow here.

## Solution Assessment

Yes: fixing the token exchange identity is the narrowest maintainable first repair. Model pre-detection should stay a fallback only if live validation proves the header-aligned token still cannot access the model.
