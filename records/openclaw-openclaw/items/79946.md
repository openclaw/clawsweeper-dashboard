## Summary

Keep open: current main still has the reported Copilot identity mismatch at the token-exchange boundary, and the linked fixing PR is open but unmerged.

## What This Changes

The issue reports that GitHub Copilot image-tool requests fail with a 400 because the Copilot API token appears scoped as copilot-language-server while runtime requests identify as vscode-chat.

## Best Possible Solution

Review and land a focused GitHub Copilot provider-auth fix through https://github.com/openclaw/openclaw/pull/79976 or a replacement that adds the token header, updates regression coverage, and includes redacted live or dependency-contract proof.

## Reproduction Assessment

Yes for source-level reproduction: current main omits `Copilot-Integration-Id` from the Copilot `/v2/token` exchange while later Copilot model and runtime requests send `vscode-chat`. I did not run a live Copilot image-tool request in this read-only review.

## Solution Assessment

Yes: the narrowest maintainable fix is in the GitHub Copilot token exchange so token minting uses the same integration identity as the downstream model/runtime calls. Model preselection should remain a fallback only if live validation proves entitlement can still differ after the auth fix.
