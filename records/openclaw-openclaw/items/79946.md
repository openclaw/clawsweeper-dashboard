## Summary

Keep open: current main still mints the short-lived Copilot API token without `Copilot-Integration-Id`, while downstream Copilot runtime and catalog calls identify as `vscode-chat`; no linked PR or canonical duplicate owns this token-exchange mismatch.

## What This Changes

The issue requests fixing GitHub Copilot image/tool requests that fail with a Copilot integrator/model-availability 400 despite runtime requests setting `Copilot-Integration-Id: vscode-chat`.

## Best Possible Solution

Fix the GitHub Copilot provider/auth boundary so token minting uses the same `vscode-chat` identity as runtime requests, with cache handling and regression coverage; keep Gemini endpoint routing in its existing separate issue.
