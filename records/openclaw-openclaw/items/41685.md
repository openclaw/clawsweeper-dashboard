## Summary

Keep open. Current main partially mitigates deleted-agent targeting, but the Control UI listing and stale URL/storage restore paths still allow unregistered agent namespaces to remain visible or selected, and the remaining fix crosses a documented Gateway/ACP discovery boundary.

## What This Changes

The issue requests that Control UI stop displaying or restoring sessions from agents no longer registered in `openclaw.json`.

## Best Possible Solution

Keep broad Gateway/ACP discovery available for non-UI consumers, but make Control UI default to configured-agent sessions and normalize stale `agent:<id>:*` URL/storage selections back to the current default when the agent is no longer configured.

## Reproduction Assessment

Yes for the remaining source path, though not with a live browser run. With only `main` configured and a disk-only agent session store present, current main still feeds discovered stores into `sessions.list`, and the UI renders returned rows while preserving explicit stale session keys.

## Solution Assessment

Unclear until a maintainer chooses the UI/API boundary. The narrow maintainable direction is a Control UI configured-agent filter or additive configured-only Gateway mode, not removing disk discovery from the shared Gateway resolver.
