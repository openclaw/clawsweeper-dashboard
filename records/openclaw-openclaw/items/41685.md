## Summary

Keep open. Current main partially mitigates deleted-agent targeting, but Control UI still consumes broad Gateway session discovery and can preserve explicit unconfigured `agent:<id>:*` selections, so the visible UI boundary remains unresolved.

## What This Changes

Request that Control UI stop displaying or restoring sessions owned by unregistered or deleted agents.

## Best Possible Solution

Keep broad Gateway/ACP discovery available, but give Control UI a configured-agent boundary and normalize stale URL/storage `agent:<id>:*` selections to the current default unless a maintainer explicitly chooses broad visibility.

## Reproduction Assessment

Yes for the remaining source path, though I did not run a live UI flow. With only `main` configured and a disk-only `agents/codex/sessions/sessions.json`, current source feeds that store into `sessions.list` and the UI renders the returned row.

## Solution Assessment

Unclear until a maintainer chooses the UI/API boundary. The safest direction is an additive configured-agent UI/listing mode or UI-side filter, not removing disk discovery from the shared Gateway store resolver.
