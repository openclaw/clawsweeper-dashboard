## Summary

Keep open. Current main still lets Control UI consume broad Gateway session lists that include disk-discovered unregistered-agent stores, and URL/storage selection still preserves explicit stale agent-scoped keys; the merged related work only mitigated send/resolve targeting for deleted agents.

## What This Changes

The issue asks for Control UI session listing and restored session selection to exclude sessions whose agent is no longer registered.

## Best Possible Solution

Keep broad Gateway/ACP discovery available for consumers that need it, but make the default Control UI list and restored chat selection validate against configured agents or an additive configured-only session-list mode.

## Reproduction Assessment

Yes, source-reproducible with high confidence: a disk-only `agents/<id>/sessions/sessions.json` for an unconfigured agent is included by the combined Gateway store and then rendered by the Control UI `sessions.list` path. I did not run a live UI reproduction because this was a read-only review.

## Solution Assessment

Unclear. Removing disk discovery from `sessions.list` would conflict with current docs/tests, so the safer direction is a maintainer-chosen UI/default boundary or additive RPC mode plus restore-path validation.
