## Summary

Keep open. Current main still lacks the OpenRouter thinking-profile opt-in needed to accept `xhigh`, and PR #74839 is open with closing syntax for this issue, so the issue should remain tracked until that PR is reviewed, landed, or closed.

## What This Changes

OpenRouter should advertise and accept `xhigh` thinking for OpenRouter-routed DeepSeek V4 Pro and Flash models that OpenRouter documents as xhigh-capable.

## Best Possible Solution

Add a provider-owned OpenRouter thinking profile for the documented xhigh-capable model scope, with focused regression tests and a user-facing changelog entry.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: `openrouter/deepseek/deepseek-v4-pro` falls through to the base thinking profile because OpenRouter has no provider profile hook and its dynamic model metadata does not carry effort-level support.

## Solution Assessment

Yes, with scope review. A provider-owned OpenRouter `resolveThinkingProfile` is the narrowest maintainable fix because current core already delegates model-specific thinking levels to provider hooks.
