## Summary

Keep open. Current main still lacks an OpenRouter thinking profile for DeepSeek V4 xhigh, and open PR #74839 uses closing syntax for this issue, so cleanup should wait for that PR to be reviewed, revised, landed, or closed.

## What This Changes

OpenRouter DeepSeek V4 model refs should accept `/think xhigh` instead of falling back to base thinking levels and rejecting it.

## Best Possible Solution

Review and land or revise PR #74839 as an OpenRouter-owned thinking-profile fix with focused positive/negative coverage and a user-facing changelog entry.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: with `openrouter/deepseek/deepseek-v4-pro` or `openrouter/deepseek/deepseek-v4-flash`, `/think xhigh` resolves through the base OpenRouter profile and is rejected before the stream wrapper can send the request.

## Solution Assessment

Yes, with scope review. A provider-owned OpenRouter `resolveThinkingProfile` is the narrowest maintainable direction, but it should advertise `xhigh` only where OpenRouter documents or exposes effort support.
