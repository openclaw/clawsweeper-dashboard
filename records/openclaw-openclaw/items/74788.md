## Summary

Keep open. Current main still lacks the OpenRouter thinking-profile hook needed to accept `xhigh` for OpenRouter DeepSeek V4 refs, and open PR #74839 references this issue with closing syntax, so the issue should track that PR until it lands or is closed.

## What This Changes

The issue requests that the OpenRouter plugin advertise `xhigh` thinking support for `openrouter/deepseek/deepseek-v4-pro` and `openrouter/deepseek/deepseek-v4-flash`.

## Best Possible Solution

Land a provider-owned OpenRouter `resolveThinkingProfile` for the documented DeepSeek V4 xhigh-capable refs, with focused OpenRouter regression tests and a user-facing changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `openrouter/deepseek/deepseek-v4-pro` falls through to the base thinking profile because OpenRouter has no provider thinking hook and its dynamic metadata does not carry per-effort support.

## Solution Assessment

Yes. A provider-owned OpenRouter `resolveThinkingProfile` is the narrowest maintainable fix because core already delegates model-specific thinking levels to provider hooks, and the transport wrapper already maps accepted `xhigh` requests correctly.
