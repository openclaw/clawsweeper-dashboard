## Summary

Keep open. Current main still lacks an OpenRouter-owned thinking profile or legacy xhigh hook, so OpenRouter DeepSeek V4 models still fall back to the base thinking levels unless catalog compat explicitly opts in. The issue is also paired with open PR #74839, so it should remain open until that PR lands, is revised, or is replaced.

## What This Changes

Request OpenRouter to advertise `/think xhigh` for OpenRouter-hosted DeepSeek V4 Pro and Flash models so the directive is accepted before transport.

## Best Possible Solution

Land a narrow OpenRouter provider policy, preferably `resolveThinkingProfile`, for the known DeepSeek V4 OpenRouter model ids with focused provider tests and a user-facing changelog entry; keep the model-specific policy out of core fallback logic.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: selecting `openrouter/deepseek/deepseek-v4-pro` or `openrouter/deepseek/deepseek-v4-flash` with `/think xhigh` reaches thinking validation, where current main has neither an OpenRouter profile hook nor compat effort metadata to add `xhigh`.

## Solution Assessment

Yes for the requested direction. A provider-owned thinking profile is the narrowest maintainable fix, but current main does not yet contain it and open PR #74839 should be reviewed for exact model scope before this issue is closed.
