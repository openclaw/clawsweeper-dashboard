## Summary

Keep open. Current main still lacks an OpenRouter thinking-profile hook for the reported DeepSeek V4 routes, and open PR #74839 references this issue with closing syntax, so the issue should wait for that PR to be reviewed, landed, or closed.

## What This Changes

Expose xhigh thinking support for OpenRouter DeepSeek V4 Pro and Flash so `/think xhigh` is accepted instead of falling back to base levels.

## Best Possible Solution

Review and land #74839, or a narrow equivalent OpenRouter-owned `resolveThinkingProfile` fix with focused tests and a user-facing changelog entry, then close this issue after merge.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: selecting `openrouter/deepseek/deepseek-v4-pro` or `openrouter/deepseek/deepseek-v4-flash` leaves `/think xhigh` on the base OpenRouter profile because current main has no provider hook or effort-level catalog metadata for those refs.

## Solution Assessment

Yes. A provider-owned OpenRouter `resolveThinkingProfile` is the narrowest maintainable solution because plugin docs assign model-specific thinking levels to provider profiles, and the existing transport path already maps accepted `xhigh` requests to DeepSeek max reasoning.
