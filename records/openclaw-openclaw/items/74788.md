## Summary

Keep open. Current main still has the reported OpenRouter thinking-profile gap, and open PR #74839 uses closing syntax for this issue, so this should remain open until that PR is reviewed, revised, landed, or closed.

## What This Changes

Allow OpenRouter DeepSeek V4 Pro and Flash model refs to accept `/think xhigh` by advertising xhigh-capable thinking levels in the OpenRouter provider profile.

## Best Possible Solution

Review, narrow if needed, and land PR #74839 or an equivalent OpenRouter-owned `resolveThinkingProfile` fix with focused provider tests and a user-facing changelog entry.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: selecting `openrouter/deepseek/deepseek-v4-pro` or `openrouter/deepseek/deepseek-v4-flash` and sending `/think xhigh` resolves to base levels because the OpenRouter provider has no profile hook and dynamic capabilities store only boolean reasoning.

## Solution Assessment

Yes, with scope review. A provider-owned OpenRouter `resolveThinkingProfile` is the narrowest maintainable direction, but it should advertise `xhigh` only for models where OpenRouter docs or capability metadata prove that effort is supported.
