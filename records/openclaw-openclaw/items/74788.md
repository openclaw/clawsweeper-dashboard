Keep open. Current `main` still has the reported OpenRouter thinking-profile gap, and an open PR already uses closing syntax for this issue, so the issue should remain open until that PR is reviewed, revised, landed, or closed.

## What This Changes

The issue requests that the OpenRouter provider advertise `xhigh` thinking support for DeepSeek V4 OpenRouter models so `/think xhigh` is accepted.

## Best Possible Solution

Land, revise, or replace PR #74839 with an OpenRouter-owned thinking-profile fix for the known DeepSeek V4 OpenRouter model IDs, plus focused positive/negative provider tests and a user-facing changelog entry. ## Reproduction Assessment

Yes. Yes, with scope review. A provider-owned OpenRouter `resolveThinkingProfile` is the narrowest maintainable direction, but the open PR should be checked so it only advertises `xhigh` where OpenRouter actually supports that effort.
