## Summary

Keep open. Current main still lacks an OpenRouter-owned thinking profile or legacy xhigh hook, so OpenRouter DeepSeek V4 model refs still fall back to the base thinking levels; open PR #74839 explicitly closes this issue and should be reviewed, landed, revised, or replaced before the issue is closed.

## What This Changes

The issue requests that the OpenRouter plugin advertise `/think xhigh` support for OpenRouter DeepSeek V4 models so those refs are not rejected during thinking-level validation.

## Best Possible Solution

Keep this issue open until #74839 or a replacement lands a narrow OpenRouter `resolveThinkingProfile` for the supported DeepSeek V4 OpenRouter model ids, with focused positive/negative coverage and a user-facing changelog entry. The direct DeepSeek implementation should stay separate from OpenRouter proxy policy.

## Reproduction Assessment

Yes. The source-level path is high-confidence: selecting `openrouter/deepseek/deepseek-v4-pro` or `openrouter/deepseek/deepseek-v4-flash` and sending `/think xhigh` reaches validation, where current main has no OpenRouter provider profile, compat effort metadata, or legacy xhigh hook to add `xhigh`.

## Solution Assessment

Yes. A provider-owned OpenRouter thinking profile is the narrowest maintainable fix because the docs and resolver make exact model thinking levels provider-owned; #74839 is the active candidate to review rather than closing this issue early.
