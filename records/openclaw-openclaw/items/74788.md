## Summary

Keep open. Current main still lacks the OpenRouter provider thinking-profile opt-in needed to accept `xhigh`, and open PR #74839 uses closing syntax for this issue, so the issue should remain tracked until that PR is reviewed, landed, or closed.

## What This Changes

Requests that the OpenRouter plugin advertise `xhigh` thinking support for DeepSeek V4 models so `/think xhigh` is accepted.

## Best Possible Solution

Land or replace PR #74839 with an OpenRouter-owned `resolveThinkingProfile` for documented xhigh-capable models, focused regression tests, and a user-facing changelog entry.

## Reproduction Assessment

Yes. Source inspection shows `openrouter/deepseek/deepseek-v4-pro` falls through to the base thinking profile because OpenRouter has no thinking-profile hook and its dynamic metadata cannot carry per-effort support.

## Solution Assessment

Yes, with scope review. A provider-owned OpenRouter `resolveThinkingProfile` is the narrowest maintainable fix because current core already delegates model-specific thinking levels to provider hooks.
