## Summary

Keep open. Current main still lacks an OpenRouter thinking-profile hook, so `openrouter/deepseek/deepseek-v4-pro` falls back to base thinking levels; open PR #74839 already references this issue with closing syntax and should remain the implementation path.

## What This Changes

The issue asks OpenRouter to advertise supported `xhigh` thinking for DeepSeek V4 OpenRouter model refs so `/think xhigh` is accepted before request construction.

## Best Possible Solution

Review and land or replace PR #74839 with a provider-owned OpenRouter thinking profile scoped to documented xhigh-capable models, plus regression coverage and changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: OpenRouter has no thinking-profile hook, its dynamic catalog lacks effort-level metadata, and the resolver therefore rejects `xhigh` for the named model before the DeepSeek V4 transport wrapper runs.

## Solution Assessment

Yes, with scope review. A provider-owned OpenRouter `resolveThinkingProfile` is the narrowest maintainable fix because current core already delegates model-specific thinking levels to provider hooks.
