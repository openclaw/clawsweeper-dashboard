## Summary

Keep open. Current main still lacks an OpenRouter-owned thinking profile or legacy xhigh hook, so the reported `/think xhigh` rejection path remains valid; open PR #74839 is the active closing implementation path and should be reviewed before this issue closes.

## What This Changes

Request OpenRouter to advertise `xhigh` thinking support for DeepSeek V4 Pro and Flash so `/think xhigh` is accepted instead of rejected.

## Best Possible Solution

Land a narrow OpenRouter-owned `resolveThinkingProfile` or equivalent provider hook that exposes `xhigh` for the supported DeepSeek V4 OpenRouter refs, keeps unsupported OpenRouter routes excluded, adds focused positive and negative coverage, and includes a user-facing changelog entry when shipped.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: select `openrouter/deepseek/deepseek-v4-pro` or `openrouter/deepseek/deepseek-v4-flash` and send `/think xhigh`; current main validates against provider/catalog thinking levels before transport, and OpenRouter currently contributes no `xhigh` support.

## Solution Assessment

Yes. A provider-owned OpenRouter thinking profile is the narrow maintainable fix because core already exposes the profile seam and direct DeepSeek already uses that pattern; changing core fallback behavior would be broader than necessary.
