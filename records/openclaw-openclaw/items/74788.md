## Summary

Keep open. Current main still lacks an OpenRouter-owned thinking profile or legacy xhigh hook, so `/think xhigh` for OpenRouter DeepSeek V4 models still falls back to the base thinking levels; open PR #74839 explicitly closes this issue and should be reviewed, revised, or landed before the issue is closed.

## What This Changes

Allow OpenRouter DeepSeek V4 Pro and Flash model refs to advertise and accept `/think xhigh` through provider-owned thinking support.

## Best Possible Solution

Land, revise, or replace #74839 with a narrow OpenRouter provider-profile fix for the known DeepSeek V4 OpenRouter model ids, plus focused positive/negative coverage and a user-facing changelog entry. Keep DeepSeek-specific policy out of core fallback logic.

## Reproduction Assessment

Yes. The source-level path is high-confidence: selecting `openrouter/deepseek/deepseek-v4-pro` or `openrouter/deepseek/deepseek-v4-flash` and sending `/think xhigh` reaches thinking validation, where current main has no OpenRouter provider profile, compat effort metadata, or legacy xhigh hook to add `xhigh`.

## Solution Assessment

Yes. A provider-owned `resolveThinkingProfile` or equivalent OpenRouter policy hook is the narrowest maintainable fix because the docs and runtime resolver make model-specific thinking level sets provider-owned.
