## Summary

Keep open. Current main still has the reported OpenRouter thinking-profile gap, and an open PR already references this issue with closing syntax, so the issue should remain open until that PR is reviewed, revised, landed, or closed.

## What This Changes

The issue asks OpenRouter to advertise `xhigh` thinking for DeepSeek V4 Pro and Flash model refs so `/think xhigh` is accepted before transport.

## Best Possible Solution

Land, revise, or replace PR #74839 with an OpenRouter-owned `resolveThinkingProfile` for the known DeepSeek V4 OpenRouter model IDs, with focused positive/negative provider tests and a user-facing changelog entry. Keep the policy in the OpenRouter plugin instead of adding DeepSeek-specific fallback logic to core.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: select `openrouter/deepseek/deepseek-v4-pro` or `openrouter/deepseek/deepseek-v4-flash`, send `/think xhigh`, and current main resolves OpenRouter to the base thinking levels because there is no OpenRouter provider profile, compat effort metadata, or legacy xhigh hook.

## Solution Assessment

Yes, with scope review. A provider-owned OpenRouter `resolveThinkingProfile` is the narrowest maintainable direction, but the PR should be checked so it does not over-advertise `xhigh` for unrelated OpenRouter reasoning models.
