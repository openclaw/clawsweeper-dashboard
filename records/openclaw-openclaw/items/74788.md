## Summary

Keep open. Current main still lacks the OpenRouter provider thinking-profile hook needed to accept `xhigh`, and open PR #74839 already references this issue with closing syntax, so the issue should wait for that PR to be reviewed, landed, or closed.

## What This Changes

The issue requests that the OpenRouter provider expose `xhigh` thinking for DeepSeek V4 OpenRouter models so `/think xhigh` is accepted before transport.

## Best Possible Solution

Review and land or close PR #74839 as the canonical OpenRouter-owned thinking-profile fix, with focused regression coverage and a user-facing changelog entry.

## Reproduction Assessment

Yes. The high-confidence reproduction is source-level: on current main, OpenRouter `xhigh` validation falls through to the base thinking profile because no OpenRouter profile hook or effort-level compat metadata is present.

## Solution Assessment

Yes, with scope review. A provider-owned OpenRouter `resolveThinkingProfile` is the narrowest maintainable fix because core already delegates model-specific thinking levels to provider hooks.
