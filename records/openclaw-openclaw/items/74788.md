## Summary

Keep open. Current main still lacks an OpenRouter-owned thinking profile or legacy xhigh hook, so the reported `/think xhigh` rejection path remains valid; open PR #74839 references this issue with closing syntax and should be reviewed, revised, or landed before cleanup.

## What This Changes

Expose `xhigh` thinking support for OpenRouter DeepSeek V4 Pro and Flash model refs so `/think xhigh` is accepted before transport.

## Best Possible Solution

Land a narrow OpenRouter-owned `resolveThinkingProfile`, or equivalent revised provider hook, that exposes `xhigh` only where OpenRouter actually supports it, with focused OpenRouter provider tests and a user-facing changelog entry if the behavior ships.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: selecting `openrouter/deepseek/deepseek-v4-pro` or `openrouter/deepseek/deepseek-v4-flash` with `/think xhigh` reaches thinking validation, but current main has neither an OpenRouter profile/hook nor compat effort metadata to add `xhigh`.

## Solution Assessment

Yes. A provider-owned thinking profile is the narrowest maintainable fix because current docs and resolver behavior make plugins responsible for model-specific thinking level sets; the linked open PR is the right path to review before closing this issue.
