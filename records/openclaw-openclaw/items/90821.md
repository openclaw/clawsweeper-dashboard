## What This Changes

The PR registers queued manual context-engine `/compact` operations as abortable active embedded runs, forwards abort signals, adds a drain-timeout guard, and extends focused compact tests.

## Best Possible Solution

Land this only after redacted live gateway proof shows manual context-engine `/compact` in progress, `/stop` aborting it, and active-run state clearing, while keeping Codex-native pending compaction cancellation out of this fix's scope.

## Reproduction Assessment

Yes, source-level: current main starts manual queued context-engine compaction without registering it as an active embedded run, while `/stop` aborts active embedded or reply runs. I did not run a live gateway reproduction in this read-only review.

## Solution Assessment

Yes, this is the right narrow layer for OpenClaw-owned context-engine compaction because registration happens inside the queued task and avoids replacing an existing active run. Command-entry registration would race queue ordering, and Codex-native cancellation needs a separate `turn/interrupt(threadId, turnId)` owner.
