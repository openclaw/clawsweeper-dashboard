## Summary

Keep this PR open: the patch is a focused and likely correct fix for the manual `/compact` cancelability gap, but it still needs redacted live gateway proof before merge because the changed behavior is a real `/compact` plus `/stop` active-run lifecycle, not just isolated function calls or unit tests.

## What This Changes

The PR registers queued manual context-engine compaction as a temporary abortable embedded run, forwards aborts into the compaction signal, adds drain-timeout handling, and extends focused compact/abort tests.

## Best Possible Solution

Land the narrow active-run lifecycle fix only after redacted live gateway proof shows manual context-engine `/compact` in progress, `/stop` aborting it, and active-run state clearing, while keeping Codex-native pending compaction cancellation out of this PR's scope.

## Reproduction Assessment

Yes, source-level: current main aborts any existing compaction-capable run before invoking manual compaction, but the queued context-engine compaction itself is not registered as an active run for `/stop`. I did not establish a live gateway failing run in this read-only review.

## Solution Assessment

Yes, this appears to be the best narrow fix for OpenClaw-owned context-engine compaction because registration happens inside the queued task and avoids replacing an existing active run. A command-entry registration would be more race-prone, and Codex-native app-server cancellation needs a separate turn-lifecycle owner.
