## Summary

Keep open: current main and the latest release still provide adjacent session labels, parent linkage, thread/topic routing, and UI search primitives, but not the requested first-class assistant/persona topic-session family model or explicit family-level memory handoff boundary.

## What This Changes

Request an explicit topic-session family model where one assistant/persona owns multiple named isolated topic lanes with controlled shared durable memory or handoff behavior.

## Best Possible Solution

Keep this as the focused design tracker, have maintainers decide the family data model and memory/security boundary first, then split accepted work into protocol/store, chat commands, UI lifecycle, memory handoff/docs, and routing tests.
