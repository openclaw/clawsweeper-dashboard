## Summary

Keep open: current main and v2026.6.8 still provide adjacent primitives such as labels, parentSessionKey lineage, thread/topic routing, and session listing, but not the requested first-class assistant/persona topic-session family model or explicit family-level memory handoff boundary. ## What This Changes

Requested first-class topic-session families so one assistant can keep multiple named, isolated context lanes with explicit shared-memory and handoff rules.

Keep this as the focused design tracker, decide the family data model and memory/security boundary first, then split accepted work into protocol/store, chat commands, UI, memory handoff/docs, and routing tests.
