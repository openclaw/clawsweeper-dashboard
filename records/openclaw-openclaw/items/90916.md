Request a first-class session-family model where one assistant has multiple named topic lanes with isolated recent context and explicit shared durable memory.

## Best Possible Solution

Stage a core session-family design: define explicit family/topic metadata and privacy rules first, then land narrow protocol/store, UI/chat command, channel-routing, and memory/docs slices with isolated defaults.
