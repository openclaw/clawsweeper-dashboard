## Summary

Keep open: current main and the latest release still have session labels, parent links, topic/thread routing, `/focus`, and memory metadata, but not the requested first-class topic-session family contract or explicit cross-topic durable-memory handoff boundary.

## What This Changes

The issue requests a first-class topic-session family model so one assistant/persona can own multiple named, isolated topic lanes with explicit family-level durable memory and cross-topic handoff rules.

## Best Possible Solution

Keep this as a maintainer RFC/scoping tracker, decide the family data model and memory/security contract first, then split implementation into protocol/store, chat lifecycle, UI/TUI lifecycle, memory handoff/docs, and routing tests.
