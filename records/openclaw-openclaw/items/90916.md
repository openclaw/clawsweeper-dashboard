## Summary

Keep open: current main and the latest release still have session labels, parent linkage, channel/thread/topic routing, UI session search, and durable memory primitives, but not a first-class assistant/persona topic-session family model with isolated topic transcripts and explicit family-level memory handoff gates.

## What This Changes

Request a topic-session family model where one assistant/persona has multiple named isolated context lanes while sharing durable memory only through explicit family-level rules.

## Best Possible Solution

Keep this as a focused maintainer product/security design tracker, then split accepted work into protocol/store metadata, chat/UI lifecycle, explicit memory handoff receipts, privacy docs, and routing tests.
