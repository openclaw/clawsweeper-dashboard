## Summary

Keep open: current main and v2026.6.8 have session labels, parent linkage, thread/topic routing, per-topic channel configuration, and memory primitives, but not the requested first-class topic-session family model or explicit family-level memory handoff boundary. The related open issues overlap on UI, shared cross-channel sessions, or broader multi-session architecture, but none supersede this narrower product/security scoping request.

## What This Changes

Request a first-class topic-session family model so one assistant/persona can own multiple named, transcript-isolated topic lanes with explicit durable-memory sharing and handoff rules.

## Best Possible Solution

Keep this issue as the scoped design tracker, decide the family data model and memory/security boundary first, then split implementation into protocol/store, chat commands, UI, memory handoff/docs, and routing tests if maintainers accept the direction.
