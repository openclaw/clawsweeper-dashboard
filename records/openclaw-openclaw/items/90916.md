## Summary

Keep open: current main and v2026.6.9 still have labels, parent session linkage, topic/thread routing, session listing, and UI search primitives, but not the requested first-class assistant/persona topic-session family model or explicit family-level memory handoff boundary.

## What This Changes

The issue requests a first-class topic-session family model so one assistant/persona can own multiple named, transcript-isolated topic lanes with explicit durable-memory sharing and handoff rules.

## Best Possible Solution

Keep this as the focused design tracker, decide the family data model and memory/security boundary first, then split accepted work into protocol/store, chat commands, UI lifecycle, memory handoff/docs, and routing tests.
