## Summary

Keep open: current main has session labels, parent linkage, `/new`, `/name`, `/focus`, topic/thread routing, and memory primitives, but it still lacks the requested first-class topic-session family model and explicit cross-topic memory handoff boundary.

## What This Changes

Request adds a first-class session-family model where one assistant/persona can own multiple named, transcript-isolated topic lanes with explicit shared durable-memory rules.

## Best Possible Solution

Keep this as a maintainer RFC/scoping tracker, decide the family data model and memory/security boundary first, then split implementation into protocol/store, chat lifecycle, UI, memory handoff/docs, and routing tests.
