## Summary

Keep open: current main and v2026.6.11 still provide adjacent session labels, parent links, channel/thread/topic isolation, and /new, /name, /focus commands, but not a first-class assistant/persona topic-session family model with named lanes and explicit durable-memory handoff. The related open issues cover UI management, selected shared sessions, and a broader multi-session RFC; none supersedes this narrower family-state request.

## What This Changes

The issue requests a first-class topic-session family model so one assistant can use multiple named, transcript-isolated context lanes while sharing durable memory only through explicit rules.

## Best Possible Solution

Keep this as the focused design tracker, have maintainers decide the family data model and memory/security boundary first, then split accepted work into protocol/store, chat commands, UI lifecycle, memory handoff/docs, and routing tests.
