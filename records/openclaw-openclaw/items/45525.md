## Summary

Keep open: the scanner gap is worth addressing, but this diff is not ready to merge because the first-token regex approach leaves dynamic-load bypasses and adds critical false positives on an install-blocking security path.

## What This Changes

The PR adds critical skill-scanner line rules for non-literal `require()` and `import()` calls in `src/security/skill-scanner.ts`.

## Best Possible Solution

Replace the first-token line regexes with an argument-aware classifier or small tokenizer, then cover variables, concatenation, comments, static and interpolated templates, multiline calls, and install-blocking behavior with regression tests.

## Reproduction Assessment

Yes. A read-only probe using the PR regexes reproduced the missed concatenated and comment-separated dynamic calls, plus static-template and commented-literal false positives.

## Solution Assessment

No. A single first-token line regex is not the best fix for a security-sensitive module-load classifier; a tokenizer or parser-backed helper would be narrower and less likely to fail closed on static imports.

## Review Findings

Full review comments:

- **[P1] Classify the full module specifier:** `src/security/skill-scanner.ts:181-188`
  - body: The regex only checks the first non-whitespace character after `(`, so `require("./" + name)` and `import("./" + name)` avoid the critical finding even though the module target is resolved dynamically at runtime.
  - confidence: 0.93
- **[P1] Handle comments before the call parenthesis:** `src/security/skill-scanner.ts:181-188`
  - body: JavaScript permits comments between the callee and `(`, but these patterns only allow whitespace, so `require/*x*/(name)` and the matching `import` form bypass the intended critical rule.
  - confidence: 0.9
- **[P2] Skip commented literal import specifiers:** `src/security/skill-scanner.ts:188`
  - body: A literal import with a leading comment, such as `import(/* webpackChunkName: "x" */ "./mod")`, starts with `/` after the parenthesis and is reported as critical even though the module specifier is static.
  - confidence: 0.87
- **[P2] Treat static templates as constants:** `src/security/skill-scanner.ts:181-188`
  - body: The patterns exclude only single and double quotes, so `import(`./fixed.js`)` and the matching `require` form are classified as critical even when the template contains no interpolation.
  - confidence: 0.88
