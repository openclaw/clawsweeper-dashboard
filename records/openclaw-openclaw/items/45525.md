## Summary

Keep open: the hardening goal is valid, but the current patch is not merge-ready because it lacks real behavior proof and the regex-only rules introduce both scanner bypasses and critical false positives on an install-blocking path.

## What This Changes

The PR adds two critical skill-scanner line rules intended to flag dynamic `require()` and `import()` calls with non-literal arguments.

## Best Possible Solution

Replace the two first-character regexes with a tested module-load classifier or parser-backed scanner path that flags runtime-resolved specifiers, permits static literals, and proves install-blocking behavior before merge.

## Reproduction Assessment

Yes for the proposed patch defects: `scanSource` applies the added regexes to raw source lines, so bypasses and false positives are source-reproducible from the rule shape without running the full install flow.

## Solution Assessment

No. The scanner should cover dynamic module loads, but first-character regexes are too brittle for critical install-blocking policy; a tested expression classifier is the safer solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Detect concatenated module specifiers:** `src/security/skill-scanner.ts:178-185`
  - body: The rule treats any single- or double-quote-starting argument as static, so `require("./" + name)` and `import("./" + name)` evade the critical finding even though the module target is resolved at runtime.
  - confidence: 0.91
- **[P1] Handle comments before module-load parens:** `src/security/skill-scanner.ts:178-185`
  - body: JavaScript permits comments between the callee and `(`, but these regexes only allow whitespace, so `require/*x*/(target)` and `import/*x*/(target)` bypass the added critical checks.
  - confidence: 0.88
- **[P2] Skip commented literal import forms:** `src/security/skill-scanner.ts:185`
  - body: The import rule can flag static forms like `import(/* webpackChunkName: "x" */ "./mod")` because it only checks the first character after `(`, turning benign literal imports into critical install blockers.
  - confidence: 0.86
- **[P2] Treat static template literals as static:** `src/security/skill-scanner.ts:178-185`
  - body: Because backticks are not excluded, fixed template literals such as `import(`./fixed.js`)` are classified as dynamic even without interpolation, which can block safe installs.
  - confidence: 0.88
