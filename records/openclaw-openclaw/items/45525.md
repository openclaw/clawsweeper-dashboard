## Summary

Keep this PR open, but do not merge it as-is. Current main still lacks dynamic `require()`/`import()` detection, while the PR’s first-token regex classifier leaves concrete dynamic-load bypasses and introduces critical false positives in a fail-closed install scanner.

## What This Changes

This PR adds two critical `src/security/skill-scanner.ts` line rules for non-literal `require()` and `import()` calls.

## Best Possible Solution

Replace the first-token regexes with an argument-aware module specifier classifier, then cover variables, concatenation, comments, static and interpolated templates, multiline calls, commented literal imports, install blocking behavior, and the required changelog entry.

## Reproduction Assessment

Yes. `scanSource` applies each line rule regex directly per line, and a read-only probe against the PR patterns reproduced both missed dynamic-load cases and critical false positives.

## Solution Assessment

No. A first-token line regex is not the best fix for a security-sensitive module-load classifier; a small tokenizer or parser-backed helper would be narrower and more maintainable.

  - body: The regex only checks the first non-whitespace character after `(`, so `require("./" + name)` and `import("./" + name)` avoid the critical finding even though the module target is resolved dynamically at runtime.
  - confidence: 0.93
- **[P1] Handle comments before the call paren:** `src/security/skill-scanner.ts:181-188`
  - body: JavaScript permits comments between the callee and `(`, but these patterns only allow whitespace, so `require/*x*/(name)` and the matching `import` form bypass the intended critical rule.
  - confidence: 0.9

  - body: A literal import with a leading comment, such as `import(/* webpackChunkName: "x" */ "./mod")`, starts with `/` after the paren and is reported as critical even though the module specifier is static.
  - confidence: 0.87
- **[P2] Treat static templates as constants:** `src/security/skill-scanner.ts:181-188`
  - body: The patterns exclude only single and double quotes, so `import(`./fixed.js`)` and the matching `require` form are classified as critical even when the template contains no interpolation.
  - confidence: 0.88
