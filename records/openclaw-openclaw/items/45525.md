## Summary

Keep open: current main still lacks the requested dynamic module-load scanner, but the proposed regex classifier is not safe to merge because it leaves dynamic-load bypasses and adds critical false positives in a scanner that can block installs.

## What This Changes

The PR adds two critical `LINE_RULES` in `src/security/skill-scanner.ts` to flag `require()` and dynamic `import()` calls when the first argument does not start with a single or double quote.

## Best Possible Solution

Implement an argument-aware module specifier classifier in `src/security/skill-scanner.ts`, then cover variables, concatenation, comments, static and interpolated templates, multiline calls, and commented literal imports in focused scanner tests plus the required changelog entry.

## Reproduction Assessment

Yes. A read-only probe against the exact PR regexes reproduces the missed dynamic forms and false-positive static forms, and current `scanSource` confirms those regexes would be applied line-by-line.

## Solution Assessment

No. A first-character line regex is not the best fix for a security-sensitive module-load classifier; a small tokenizer or parser-backed helper is the narrower maintainable path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Classify the full module specifier:** `src/security/skill-scanner.ts:181-188`
  - body: The regex only checks the first non-whitespace character after `(`, so `require("./" + name)` and `import("./" + name)` avoid the critical finding even though the module target is resolved dynamically at runtime.
  - confidence: 0.93
- **[P1] Detect calls separated by comments:** `src/security/skill-scanner.ts:181-188`
  - body: JavaScript permits comments between the callee and `(`, but these patterns only allow whitespace, so `require/*x*/(name)` and the matching `import` form bypass the intended critical rule.
  - confidence: 0.9
- **[P2] Skip commented literal import specifiers:** `src/security/skill-scanner.ts:188`
  - body: A literal import with a leading comment, such as `import(/* webpackChunkName: "x" */ "./mod")`, starts with `/` after the paren and is reported as critical even though the module specifier is static.
  - confidence: 0.87
- **[P2] Treat static template literals as constants:** `src/security/skill-scanner.ts:181-188`
  - body: The patterns exclude only single and double quotes, so `import(`./fixed.js`)` and the matching `require` form are classified as critical even when the template contains no interpolation.
  - confidence: 0.88
- **[P3] Add the required changelog entry:** `src/security/skill-scanner.ts:181`
  - body: This `fix` changes user-facing install and audit scanner behavior, but the PR only changes the scanner source. Add a `CHANGELOG.md` entry for the new dynamic-load detection before merge.
  - confidence: 0.82
