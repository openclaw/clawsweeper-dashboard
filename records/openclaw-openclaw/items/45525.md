## Summary

Keep open: current main still lacks dynamic require/import scanner rules, but this PR is not mergeable as-is because source inspection and prior review show critical-rule bypasses, install-blocking false positives, and missing real behavior proof.

## What This Changes

Adds two critical `LINE_RULES` in `src/security/skill-scanner.ts` to flag `require()` and `import()` calls whose first argument does not start with single or double quotes.

## Best Possible Solution

Implement a tested module-load expression classifier in the scanner that flags runtime-resolved specifiers while allowing static literals, then provide focused install-scan proof.

## Reproduction Assessment

Yes for the proposed patch defects: `scanSource` applies each `LineRule` regex to raw source lines, and the PR patterns only inspect the first token after the module-load paren. The bypasses and false positives are source-reproducible without a full install run.

## Solution Assessment

No. The hardening direction belongs in the scanner, but first-character regexes are not maintainable enough for critical install-blocking policy; a tested classifier or parser-backed path is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Detect concatenated module specifiers:** `src/security/skill-scanner.ts:178-185`
  - body: The rule treats any quote-starting argument as static, so `require("./" + name)` and `import("./" + name)` evade the critical finding even though the target is resolved at runtime.
  - confidence: 0.9
- **[P1] Handle comments before module-load parens:** `src/security/skill-scanner.ts:178-185`
  - body: JavaScript permits comments between the callee and `(`, but these regexes only allow whitespace, so `require/*x*/(target)` and `import/*x*/(target)` bypass the added critical checks.
  - confidence: 0.86
- **[P2] Skip static literal import forms:** `src/security/skill-scanner.ts:185`
  - body: The import rule can flag static forms such as `import(/* webpackChunkName: "x" */ "./mod")` because it only checks the first character after `(`, turning literal imports into critical install blockers.
  - confidence: 0.84
- **[P2] Treat static template literals as static:** `src/security/skill-scanner.ts:178-185`
  - body: Because only single and double quotes are excluded, fixed template literals such as `import(`./fixed.js`)` are classified as dynamic even without interpolation.
  - confidence: 0.88
