## Summary

Keep open: current main still lacks dynamic module-load detection, but this PR is not merge-ready because the regex approach misses runtime-resolved specifiers, creates critical false positives in the install-blocking scan path, and still lacks after-fix real behavior proof.

## What This Changes

Adds critical `dynamic-require` and `dynamic-import` line rules to `src/security/skill-scanner.ts` for `require()` and `import()` calls whose first argument is not single- or double-quoted.

## Best Possible Solution

Replace the two first-character regexes with a tested module-load expression classifier that flags runtime-resolved specifiers, permits static literals, and has focused install-scan proof before merge.

## Reproduction Assessment

Yes for the proposed patch defects: `scanSource` applies the added regexes to raw source lines, so the bypasses and false positives are source-reproducible without running the full install flow.

## Solution Assessment

No. The hardening belongs in the scanner, but first-character regexes are too brittle for critical install-blocking policy; a tested expression classifier is the safer solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Detect concatenated module specifiers:** `src/security/skill-scanner.ts:178-185`
  - body: The rule treats any single- or double-quote-starting argument as static, so `require("./" + name)` and `import("./" + name)` evade the critical finding even though the module target is resolved at runtime.
  - confidence: 0.91
- **[P1] Handle comments before module-load parens:** `src/security/skill-scanner.ts:178-185`
  - body: JavaScript permits comments between the callee and `(`, but these regexes only allow whitespace, so `require/*x*/(target)` and `import/*x*/(target)` bypass the added critical checks.
  - confidence: 0.88
- **[P2] Skip commented literal import forms:** `src/security/skill-scanner.ts:185`
  - body: The import rule can flag static forms like `import(/* webpackChunkName: "x" */ "./mod")` because it only checks the first character after `(`, turning benign literal imports into critical install blockers.
  - confidence: 0.86
- **[P2] Treat static template literals as static:** `src/security/skill-scanner.ts:178-185`
  - body: Because backticks are not excluded, fixed template literals such as `import(`./fixed.js`)` are classified as dynamic even without interpolation, which can block safe installs.
  - confidence: 0.88
