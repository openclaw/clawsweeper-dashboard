## Summary

Keep this PR open: the requested hardening is still useful and current main does not add dynamic require/import rules, but the proposed regexes remain too easy to bypass and can also block safe static template-literal imports; the PR also lacks after-fix real behavior proof.

## What This Changes

The PR adds two critical `LINE_RULES` in `src/security/skill-scanner.ts` for non-literal `require()` and `import()` calls.

## Best Possible Solution

Add tested dynamic-load detection that distinguishes literal specifiers from runtime expressions and covers concatenation, interpolated templates, and JavaScript comments in both audit and install paths.

## Reproduction Assessment

Yes. From source inspection, the PR regexes only inspect a narrow raw-line shape, so examples like `require("./" + userInput)`, `require/*x*/(userInput)`, and static backtick literals have predictable bypass or false-positive behavior.

## Solution Assessment

No. Adding two raw line regexes is not the best security-gate solution unless they are backed by focused parsing/normalization and regression tests for literal, concatenated, template, and comment forms.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Detect concatenated dynamic specifiers:** `src/security/skill-scanner.ts:178-185`
  - body: The new rules treat any call whose first argument token is a quote as safe, so `require("./" + userInput)` and `import("./" + name)` evade the critical finding even though the module target is computed at runtime. This leaves the intended dynamic-load gate bypassable.
  - confidence: 0.93
- **[P1] Handle comments around dynamic-load calls:** `src/security/skill-scanner.ts:178-185`
  - body: The regex only allows whitespace between the callee and `(`, so JavaScript forms like `require/*x*/(userInput)` bypass the new critical rule while still executing as dynamic module loads. The same shape affects `import`, so a malicious skill can avoid this check with comment trivia.
  - confidence: 0.88
- **[P2] Do not block static template-literal specifiers:** `src/security/skill-scanner.ts:178-185`
  - body: The lookahead excludes only single and double quotes, so constant backtick specifiers such as `import(`./fixed-module`)` are reported as critical even when there is no interpolation. Critical findings block installs, so safe plugins or skills can be rejected.
  - confidence: 0.9
