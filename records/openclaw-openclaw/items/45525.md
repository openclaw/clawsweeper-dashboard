## Summary

Keep open: the hardening goal is valid and not implemented on current main, but this PR still has install-blocking scanner correctness/security gaps and lacks contributor real-behavior proof.

## What This Changes

The PR adds critical `dynamic-require` and `dynamic-import` line-rule regexes to `src/security/skill-scanner.ts`.

## Best Possible Solution

Implement dynamic module-load detection with a tested lightweight JavaScript load-expression classifier that distinguishes runtime expressions from static literals before adding install-blocking critical findings.

## Reproduction Assessment

Yes for the proposed patch defects: source inspection shows the PR patterns only inspect the first non-whitespace argument character while the scanner runs raw one-line regexes. Current main also lacks the requested dynamic-load rules.

## Solution Assessment

No. The hardening direction is useful, but two first-character line regexes are not the narrowest maintainable solution for an install-blocking security scanner.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Detect concatenated module specifiers:** `src/security/skill-scanner.ts:178-185`
  - body: The new patterns skip calls whose first argument starts with a quote, so `require("./" + name)` and `import("./" + name)` still evade the critical rule even though the module target is resolved at runtime.
  - confidence: 0.9
- **[P1] Handle comments before dynamic-load parens:** `src/security/skill-scanner.ts:178-185`
  - body: JavaScript permits comments between the callee and `(`, but these regexes only allow whitespace there. Calls like `require/*x*/(target)` or `import/*x*/(target)` bypass the new checks.
  - confidence: 0.86
- **[P2] Skip commented static literal imports:** `src/security/skill-scanner.ts:185`
  - body: The rule checks only the first character after `(`, so `import(/* webpackChunkName: "x" */ "./mod")` is reported as dynamic even though the specifier is a static literal.
  - confidence: 0.84
- **[P2] Treat static template literals as static:** `src/security/skill-scanner.ts:178-185`
  - body: Backtick specifiers without interpolation, such as `import(`./fixed-module`)`, are static, but the new patterns classify them as critical dynamic loads.
  - confidence: 0.88
