## Summary

Keep open: current main does not already contain dynamic require/import scanning, and the proposed classifier changes a blocking security gate while leaving reproducible bypasses and false-positive blocks. Because this touches CODEOWNERS-protected security code, it needs maintainer/secops review before merge.

## What This Changes

The PR adds critical line rules to `src/security/skill-scanner.ts` for `require()` and `import()` calls whose first argument does not start with a single or double quote.

## Best Possible Solution

Replace the first-token regex rules with a module-specifier-aware classifier and focused scanner tests covering variables, concatenation, comments, static and interpolated templates, multiline calls, and install/audit blocking behavior, then add the required changelog entry.

## Reproduction Assessment

Yes. `scanSource` applies line rules directly, and a read-only probe of the PR regexes reproduces the missed dynamic forms and critical false positives.

## Solution Assessment

No. A first-character line regex is not the best solution for a security-sensitive module-load classifier; a small tokenizer or parser-backed helper would be more maintainable and less brittle.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Classify the full module specifier:** `src/security/skill-scanner.ts:181-188`
  - body: The regex only checks the first non-whitespace character after `(`, so `require("./" + name)` and `import("./" + name)` avoid the critical finding even though the module target is resolved dynamically at runtime.
  - confidence: 0.94
- **[P1] Detect comment-separated dynamic load calls:** `src/security/skill-scanner.ts:181-188`
  - body: JavaScript permits comments between the callee and `(`, but these patterns only allow whitespace, so `require/*x*/(name)` and the matching `import` form bypass the intended critical rule.
  - confidence: 0.91
- **[P2] Treat static template literals as constants:** `src/security/skill-scanner.ts:181-188`
  - body: The patterns exclude only single and double quotes, so `import(`./fixed.js`)` and the matching `require` form are classified as critical even when the template contains no interpolation.
  - confidence: 0.89
- **[P2] Skip commented literal import specifiers:** `src/security/skill-scanner.ts:188`
  - body: A literal import with a leading comment, such as `import(/* webpackChunkName: "x" */ "./mod")`, starts with `/` after the paren and is reported as critical even though the module specifier is static.
  - confidence: 0.88
- **[P3] Add the required changelog entry:** `src/security/skill-scanner.ts:181`
  - body: This `fix` changes user-facing install and audit scanner behavior, but the PR only changes the scanner source. Add a `CHANGELOG.md` entry for the new dynamic-load detection before merge.
  - confidence: 0.82
