## Summary

Current main already routes Ollama thinking menus through the provider-owned profile, so reasoning-capable Ollama models expose `off, low, medium, high, max` instead of the base hardcoded list with `minimal`; the fix is on main after the latest release tag.

## What This Changes

Ollama `/think` should keep model-specific levels throughout a session instead of falling back to the base list with `minimal`.

## Best Possible Solution

Keep the current main implementation and ship a release containing `5e9258d0a81b9209656675a1672afcd9604b557a`; users on `v2026.5.4` need a newer build for this fix.

## Reproduction Assessment

No current-main failing reproduction was established; the report was intermittent and I did not run Ollama Cloud. Source inspection matches the fallback described in the report and shows current main now resolves Ollama's provider profile before the client falls back to generic levels.

## Solution Assessment

Yes. The current fix keeps the behavior provider-owned, feeds the same profile into gateway rows and client `/think` handling, and avoids adding an Ollama-specific core default.
