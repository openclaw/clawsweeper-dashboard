## Summary

Current main already carries the Ollama provider-policy fix that prevents `/think` menus from falling back to the generic `off, minimal, low, medium, high` list when the full plugin runtime is not active; the fix is on main after v2026.5.4 and is recorded in the unreleased changelog.

## What This Changes

The issue reports that Ollama per-model thinking levels can be replaced mid-session by OpenClaw's static thinking defaults.

## Best Possible Solution

Keep the current main provider-policy implementation and ship it in the next release so Ollama thinking menus do not depend on runtime plugin activation.

## Reproduction Assessment

Yes, at source level: the reported menu text is generated from `resolveCommandArgMenu`, which formerly could fall through to `BASE_THINKING_LEVELS`; current main routes Ollama through the bundled provider-policy profile instead.

## Solution Assessment

Yes: the current fix is the narrow maintainable path because it keeps Ollama-owned thinking levels in the plugin policy surface without hardcoding Ollama behavior into core.
