## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Quote plugin IDs in command hints:** `src/plugins/loader-provenance.ts:260`
  - body: This warning is emitted for untracked local plugin code, and manifest IDs are only trimmed/nonempty in `loadPluginManifest`. A plugin can choose an ID containing shell metacharacters, so embedding it directly in `openclaw plugins inspect ${plugin.id}` turns the trust warning into a copyable command with untrusted text; shell-quote the argument or keep the `<id>` placeholder.
  - confidence: 0.83
- **[P3] Avoid truncated allowlist snippets:** `src/plugins/loader-provenance.ts:233`
  - body: When more than six plugins are auto-discovered, this snippet contains only the previewed IDs while the warning also says there are `+N more`. If a user pastes the snippet, `plugins.allow` becomes restrictive and can disable the omitted plugins, so include all IDs or make the truncated example explicit.
  - confidence: 0.74
