## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Require a trusted bw binary path:** `scripts/bw-exec-resolver.mjs:29`
  - body: The wrapper is executed through OpenClaw's validated absolute exec-provider command, but it then runs `bw` via PATH while the docs tell users to pass `BW_SESSION` and `PATH`. Any earlier `bw` on PATH receives the vault session and bypasses the current command/trustedDirs checks, so the wrapper should take and validate an absolute Bitwarden CLI path or otherwise avoid PATH lookup before forwarding secrets.
  - confidence: 0.91
- **[P2] Ship the bundled resolver in the package:** `docs/gateway/secrets.md:339`
  - body: The docs advertise `scripts/bw-exec-resolver.mjs` as a bundled wrapper, but the package `files` list only includes selected scripts and not this new resolver or declaration. npm installs would not contain the documented command path, so either package/install the resolver under an included path or avoid documenting it as bundled.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: The proposed resolver forwards Bitwarden vault credentials to a PATH-resolved `bw` binary, bypassing OpenClaw's exec-provider path validation.

Concerns:

- **[high] PATH-resolved Bitwarden CLI receives vault credentials:** `scripts/bw-exec-resolver.mjs:29`
  - body: OpenClaw validates the wrapper script path, but the wrapper invokes `bw` by name and the docs pass PATH plus BW_SESSION into that process. A malicious or unexpected earlier PATH entry could run with vault credentials, so the wrapper needs an absolute validated Bitwarden CLI path or equivalent trust check.
  - confidence: 0.91
