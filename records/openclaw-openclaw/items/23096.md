## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Validate the bw binary before spawning it:** `scripts/bw-exec-resolver.mjs:28-30`
  - body: The core exec provider validates only the configured wrapper path, but this wrapper then runs `bw` by name while the docs pass `PATH` through. That lets an unexpected binary earlier on PATH receive `BW_SESSION` and the requested secret ids, so require an absolute configured `bw` path and validate it before spawning.
  - confidence: 0.91
- **[P2] Ship the resolver before documenting it as bundled:** `docs/gateway/secrets.md:339`
  - body: The docs call `scripts/bw-exec-resolver.mjs` a bundled wrapper, but `package.json` only ships a small allowlist of scripts and does not include this new file. Users installing OpenClaw from the package will not have the documented resolver path, so either package/build it or move the example to an external/plugin-owned script.
  - confidence: 0.86
- **[P2] Redact Bitwarden CLI errors before returning them:** `scripts/bw-exec-resolver.mjs:126-129`
  - body: `runBw` wraps raw stderr/err.message and `resolveSecrets` returns that string in protocol errors. Current SecretRef resolution includes per-id error messages in activation failures, so Bitwarden vault, account, or item details from the CLI can leak into logs; map failures to bounded redacted messages instead.
  - confidence: 0.83
