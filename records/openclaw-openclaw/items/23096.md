## Summary

Keep this PR open, but do not merge it as-is. Current main still lacks Bitwarden/Vaultwarden support, while the PR introduces a security-sensitive wrapper that executes the real `bw` binary via `PATH` and needs maintainer direction between a hardened exec-wrapper path and the planned secret-provider plugin seam.

## What This Changes

The PR branch adds Bitwarden/Vaultwarden SecretRef documentation, a protocol-v1 Bitwarden CLI exec resolver script with TypeScript declarations, and tests for parsing, field extraction, grouping, and child-process resolution.

## Best Possible Solution

Land Bitwarden/Vaultwarden support only after the route is explicit: preferably as a secrets-bitwarden plugin once the secret-provider seam exists, or as a hardened exec-wrapper/docs integration that validates an absolute Bitwarden binary path and preserves redaction and tests.

## Reproduction Assessment

Not applicable as a feature PR rather than a bug report. For the blocking finding, yes: configure the wrapper as documented, put a different `bw` earlier in `PATH`, and the wrapper will execute that binary with the Bitwarden credential environment.

## Solution Assessment

No, not as currently proposed. The exec-wrapper route can work only after binary selection is hardened, and maintainers still need to decide whether the plugin seam is the narrower long-term home.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Resolve the Bitwarden CLI without PATH search:** `scripts/bw-exec-resolver.mjs:29`
  - body: The configured exec provider validates only this wrapper script, but the wrapper then calls `execFile("bw", ...)` while the docs pass `PATH` with Bitwarden credential env. A writable or overridden PATH can make OpenClaw run a different `bw` binary with those credentials, bypassing the exec-provider path hardening. Require an absolute validated Bitwarden binary path, such as a checked `BW_BIN`, before spawning it.
  - confidence: 0.86

## Security Review
