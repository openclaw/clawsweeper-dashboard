- **[P1] Resolve the Bitwarden CLI from a trusted absolute path:** `scripts/bw-exec-resolver.mjs:29`
  - body: The configured exec provider only validates the wrapper path; this line then launches `bw` through the inherited `PATH`, and the docs tell users to pass `PATH` into the wrapper.   - confidence: 0.91

For a secret-provider wrapper, map CLI failures to bounded sanitized messages so account, endpoint, item, or vault context from `bw` stderr is not propagated into logs or user-visible errors.
