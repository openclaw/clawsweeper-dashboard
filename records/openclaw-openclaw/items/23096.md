## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Resolve the Bitwarden CLI without PATH lookup:** `scripts/bw-exec-resolver.mjs:29-30`
  - body: The exec provider validates the configured wrapper path, but the wrapper then runs `bw` by name. Because the docs pass `PATH` through to this secret-reading process, a contaminated PATH can route vault access to an unintended binary while still bypassing OpenClaw's command-path checks; accept and validate an absolute Bitwarden binary path instead.
  - confidence: 0.87
- **[P2] Sanitize CLI errors before returning protocol failures:** `scripts/bw-exec-resolver.mjs:126-129`
  - body: `bw` stderr and child-process messages are copied into per-id protocol errors, and core includes those messages in SecretRef resolution failures. For a vault resolver, map known failure cases to redacted messages instead of forwarding raw CLI diagnostics into OpenClaw logs/operator errors.
  - confidence: 0.78
- **[P3] Add the required changelog entry:** `docs/gateway/secrets.md:288`
  - body: This adds a user-facing secrets integration, but the diff has no `CHANGELOG.md` entry. OpenClaw policy requires one for user-facing `feat` changes; do not force a forbidden bot or maintainer `Thanks @...` attribution if no human attribution is appropriate.
  - confidence: 0.91

## Security Review

Status: needs_attention

Summary: The diff adds a secret-reading command wrapper and needs hardening around CLI binary resolution and error redaction before merge.

Concerns:

- **[medium] Unvalidated bw binary lookup:** `scripts/bw-exec-resolver.mjs:29`
  - body: The wrapper executes `bw` through PATH even though OpenClaw's exec-provider hardening only validates the configured wrapper path. This can send vault reads to an unintended binary if the inherited PATH is unsafe.
  - confidence: 0.88
- **[medium] Raw CLI diagnostics in SecretRef errors:** `scripts/bw-exec-resolver.mjs:128`
  - body: The wrapper serializes raw `err.message` from the Bitwarden CLI into protocol errors, which core can surface in resolution failures. Vendor stderr should be mapped to sanitized, actionable messages.
  - confidence: 0.78
