## Summary

Keep this PR open: current main still does not provide Bitwarden/Vaultwarden SecretRef support, and maintainer discussion treats this as the focused candidate, but the branch still needs live proof plus security/package fixes before merge.

## What This Changes

The PR branch adds Bitwarden/Vaultwarden SecretRef documentation, a protocol-v1 Bitwarden CLI exec resolver script, declarations, and tests for batch secret resolution.

## Best Possible Solution

Land a refreshed Bitwarden/Vaultwarden integration as either a hardened packaged exec resolver coordinated with the existing Bitwarden Secrets resolver work, or as a secret-provider plugin once the plugin source seam is accepted.

## Reproduction Assessment

Not applicable: this is a feature PR, not a bug report. Static review does give a high-confidence path for the blocking patch issues: the wrapper resolves bw via PATH, package files omit the documented script, and the PR body only shows mocked verification.

## Solution Assessment

No, not in its current form. The existing exec transport can support the use case, but this branch needs trusted nested CLI resolution, packaging or an external install path, redacted live proof, and a maintainer route decision.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Resolve the nested bw binary from a trusted path:** `scripts/bw-exec-resolver.mjs:29`
  - body: The configured OpenClaw exec command is the wrapper, but the wrapper then starts `bw` by name while the docs pass `PATH` and Bitwarden session env.   - confidence: 0.88

  - confidence: 0.84
