## Summary

Keep open: current main still lacks Bitwarden/Vaultwarden SecretRef support, and the open Bitwarden Secrets and plugin-seam PRs do not fully supersede this branch. The PR is not merge-ready because the wrapper resolves the nested `bw` binary through `PATH` while handling vault credentials, and the proof remains mocked/unit-only.

## What This Changes

The PR branch adds Bitwarden/Vaultwarden SecretRef documentation plus a protocol-v1 Bitwarden CLI exec resolver script, declarations, and tests for multi-secret batch resolution.

## Best Possible Solution

Choose the intended integration route, then land either a trusted-binary-hardened exec wrapper or a plugin-seam Bitwarden provider with redacted live vault proof and focused tests.

## Reproduction Assessment

Yes for the blocking finding: source inspection shows the documented wrapper passes `PATH` and `BW_SESSION` while the wrapper runs `execFile("bw", ...)`, so a PATH-precedence binary can be reasoned about without a live vault. The feature itself still lacks live Bitwarden/Vaultwarden behavior proof.

## Solution Assessment

No, not as-is. The branch should either be ported to the plugin seam once that route is chosen, or the exec wrapper must validate an explicit trusted `bw` binary path before merge.
