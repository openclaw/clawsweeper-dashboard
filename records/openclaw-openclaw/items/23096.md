## Summary

Keep open: current main still lacks Bitwarden/Vaultwarden support, prior discussion keeps this as a focused candidate, and the PR still needs real Bitwarden proof plus security hardening and a maintainer route decision before merge.

## What This Changes

The PR adds Bitwarden/Vaultwarden SecretRef docs, a protocol-v1 `bw` CLI exec resolver, type declarations, and tests for batched secret resolution.

## Best Possible Solution

Land either a hardened exec-wrapper/docs integration or a plugin-shaped Bitwarden provider, with trusted binary selection, redacted errors, refreshed tests, and redacted live vault proof.

## Reproduction Assessment

No for the user-facing integration: the PR body and tests show mocked/unit verification, and no live Bitwarden/Vaultwarden run is documented. The two security blockers are source-reproducible from the PR head.

## Solution Assessment

No, not as currently proposed. The direction is useful, but this patch needs trusted nested-binary selection, redacted resolver errors, real behavior proof, and a maintainer choice between exec-wrapper and plugin-seam routes before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Require a trusted bw binary path:** `scripts/bw-exec-resolver.mjs:29`
  - body: The wrapper is launched by the hardened exec provider, but it then spawns `bw` by PATH lookup while the docs pass `PATH` through. That nested lookup bypasses OpenClaw's absolute-command/trusted-dir checks, so require an absolute `BW_BIN` or validate the resolved binary path before spawning it.
  - confidence: 0.89
- **[P2] Redact CLI stderr before returning resolver errors:** `scripts/bw-exec-resolver.mjs:38`
  - body: `runBw` turns raw Bitwarden stderr into an Error and `resolveSecrets` serializes that message into protocol errors, which core includes in per-id SecretRef errors. Map expected failures to fixed redacted messages so account, vault, or secret-adjacent stderr cannot leak through diagnostics.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The diff introduces a secret-manager exec wrapper with concrete nested command-path trust and error-redaction issues.

Concerns:

- **[medium] Nested bw lookup bypasses exec hardening:** `scripts/bw-exec-resolver.mjs:29`
  - body: The wrapper calls `execFile("bw", ...)`, so the Bitwarden binary is selected from PATH inside the wrapper instead of being validated by OpenClaw's absolute command and trusted-directory checks.
  - confidence: 0.89
- **[medium] Raw CLI stderr reaches SecretRef errors:** `scripts/bw-exec-resolver.mjs:38`
  - body: Nonzero `bw` exits are converted from raw stderr into protocol errors, which core includes in per-id SecretRef resolution failures. A secret resolver should return redacted fixed messages for expected failure modes.
  - confidence: 0.84
