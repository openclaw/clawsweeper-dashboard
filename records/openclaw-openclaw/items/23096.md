## Summary

Keep open. The Bitwarden/Vaultwarden integration remains useful and is not on current main, but this patch still has blocking command-execution/redaction issues, lacks real behavior proof, and needs maintainer route selection before merge.

## What This Changes

The branch adds Bitwarden/Vaultwarden SecretRef docs, a protocol-v1 `bw` CLI exec resolver script with declarations, and tests for batched secret resolution.

## Best Possible Solution

Land either a hardened Bitwarden exec-wrapper/docs integration or a plugin-shaped Bitwarden provider after the maintainer route is chosen, with trusted binary selection, redacted errors, targeted tests, and live proof.

## Reproduction Assessment

Yes for the PR review blockers: source inspection shows the wrapper spawns `bw` from `PATH` and returns raw CLI stderr through protocol errors. This is feature work, so there is no current-main bug reproduction path.

## Solution Assessment

No, not as-is. The integration direction is useful, but this patch needs command-execution hardening, redacted error mapping, live proof, and a maintainer decision on exec-wrapper versus plugin-provider shape.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Require a trusted Bitwarden binary path:** `scripts/bw-exec-resolver.mjs:29-30`
  - body: The wrapper is launched through OpenClaw's hardened exec provider, but it then spawns `bw` by searching `PATH`. The documented multi-secret config passes `PATH`, so the actual secret-manager binary bypasses the absolute-command and trusted-dir checks that protect exec SecretRefs; require an absolute `BW_BIN` or equivalent validated path before spawning the nested CLI.
  - confidence: 0.89
- **[P2] Redact CLI stderr before returning resolver errors:** `scripts/bw-exec-resolver.mjs:38`
  - body: `runBw` rejects with raw stderr, and `resolveSecrets` serializes that message into SecretRef protocol errors. Core can surface per-id exec errors during SecretRef resolution, so arbitrary Bitwarden CLI stderr can leak account, vault, or secret-adjacent data into diagnostics; map expected failures to fixed redacted messages instead.
  - confidence: 0.84

## Security Review

Status: needs_attention
