## Security Review

Status: needs_attention

Summary: The diff introduces a nested secrets CLI execution path that currently resolves the Bitwarden binary through PATH instead of an absolute validated path.

Concerns:

- **[medium] Nested bw execution bypasses core path validation:** `scripts/bw-exec-resolver.mjs:27`
  - body: OpenClaw's exec SecretRef provider validates the configured command path, but this wrapper then runs `execFile("bw", ...)` inside that validated process. Because the docs pass PATH through, a different binary can be selected at runtime without the trusted-dir and symlink checks applied to the outer command.
  - confidence: 0.88
