Full review comments:

- **[P1] Resolve Bitwarden through a validated absolute binary path:** `scripts/bw-exec-resolver.mjs:29`
  - body: The core exec provider validates only the wrapper script path, but this wrapper then runs `bw` from `PATH`; the docs also pass `PATH` into the wrapper. That lets the actual secret-handling binary bypass the existing absolute-command/trustedDirs boundary. Take an absolute `bw` path or validated `BW_BIN` and apply the same path checks before spawning it.
  - confidence: 0.91
- **[P2] Redact Bitwarden CLI errors before returning them:** `scripts/bw-exec-resolver.mjs:38`
  - body: `stderr` is copied into an `Error` and then returned as protocol error text for each requested id. Current SecretRef resolution includes provider error messages in operator-facing failures, so raw CLI output can leak account, server, item, or unexpected secret-adjacent context. Map known failure modes to bounded messages before returning them.
  - confidence: 0.82
