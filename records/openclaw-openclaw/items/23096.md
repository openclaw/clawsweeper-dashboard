## Security Review

Status: needs_attention

Summary: The PR introduces a command-execution path that resolves the nested Bitwarden CLI from PATH while passing secret-bearing environment variables.

Concerns:

- **[medium] PATH-resolved Bitwarden CLI bypasses exec hardening:** `scripts/bw-exec-resolver.mjs:29`
  - body: OpenClaw validates the configured exec provider command path, but this wrapper starts a second executable with `execFile("bw", ...)`; combined with docs passing `PATH`, the actual Bitwarden CLI binary is selected outside OpenClaw's trusted-dir checks and receives env such as `BW_SESSION`.
  - confidence: 0.88
