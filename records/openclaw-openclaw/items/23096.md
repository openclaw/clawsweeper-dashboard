## Security Review

Status: needs_attention

Summary: The PR introduces a command-execution path that bypasses OpenClaw's exec command hardening by resolving the nested Bitwarden CLI from PATH.

Concerns:

- **[medium] PATH-resolved Bitwarden CLI bypasses exec hardening:** `scripts/bw-exec-resolver.mjs:29`
  - body: OpenClaw validates the configured exec provider command path, but this wrapper starts a second executable with `execFile("bw", ...)`; combined with docs passing `PATH`, the actual Bitwarden CLI binary is selected outside OpenClaw's trusted-dir checks and receives secret-bearing env such as `BW_SESSION`.
  - confidence: 0.87
