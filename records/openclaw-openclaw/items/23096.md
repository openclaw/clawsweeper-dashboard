## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Pin the Bitwarden CLI instead of searching PATH:** `scripts/bw-exec-resolver.mjs:29`
  - body: `resolveExecRefs` validates only the configured wrapper command path before running it, but the wrapper then invokes `"bw"` through PATH while the docs pass PATH into the child. That lets an unexpected PATH entry choose the binary that receives `BW_SESSION` and other credential env vars, bypassing the exec provider's absolute-command/trusted-dir checks. Require a configured absolute Bitwarden binary path, validate it, or avoid the wrapper path for this case.
  - confidence: 0.91
- **[P2] Ship the wrapper before documenting it as bundled:** `docs/gateway/secrets.md:339-347`
  - body: This docs block tells users to run the bundled `scripts/bw-exec-resolver.mjs`, but `package.json` only publishes selected scripts and does not include this new file or its declaration. Users installing OpenClaw from the package will not have the documented wrapper path, so either package/move the resolver to an installed location or do not present it as bundled.
  - confidence: 0.89
