## Summary

Keep this PR open. Current main still does not include Bitwarden/Vaultwarden SecretRef support, and the branch remains the focused candidate, but it needs maintainer route selection, two security fixes, and real behavior proof before merge.

## What This Changes

The PR adds Bitwarden/Vaultwarden SecretRef docs, a protocol-v1 `bw` CLI exec resolver script, TypeScript declarations, and tests for batched secret resolution.

## Best Possible Solution

Choose the integration route explicitly, then land either a plugin-shaped Bitwarden provider on the pluggable SecretRef seam or a hardened exec resolver with trusted binary selection, redacted errors, docs, tests, and live proof.

## Reproduction Assessment

No. The PR body and comments only show mocked/unit verification and explicitly say no live Bitwarden vault was verified; the security blockers are source-reproducible in `scripts/bw-exec-resolver.mjs`.

## Solution Assessment

No, not as currently proposed. The branch is useful, but the maintainable path needs a maintainer decision between the plugin seam and a hardened exec wrapper, plus fixes for command-path trust and error redaction.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Require a trusted bw binary path:** `scripts/bw-exec-resolver.mjs:29-30`
  - body: The wrapper is launched through OpenClaw's hardened exec provider, but it then spawns `bw` by searching `PATH`. The documented multi-secret config passes `PATH`, so the actual secret-manager binary bypasses the absolute-command/trusted-dir checks that protect exec SecretRefs; require an absolute `BW_BIN` or equivalent validated path before spawning the nested CLI.
  - confidence: 0.89
- **[P2] Redact CLI stderr before returning resolver errors:** `scripts/bw-exec-resolver.mjs:38`
  - body: `runBw` wraps raw `stderr`, and `resolveSecrets` serializes that message into SecretRef protocol errors. Core surfaces per-id exec errors, so arbitrary Bitwarden CLI stderr can leak account, vault, or secret-adjacent data into OpenClaw diagnostics; map failures to fixed redacted messages instead.
  - confidence: 0.83

## Security Review

Status: needs_attention

Summary: Needs attention because the diff introduces a secret-manager exec wrapper that resolves `bw` from PATH and returns raw CLI stderr through SecretRef errors.

Concerns:

- **[medium] Nested bw lookup bypasses exec hardening:** `scripts/bw-exec-resolver.mjs:29`
  - body: The wrapper calls `execFile("bw", ...)`, so the Bitwarden binary is selected from `PATH` inside the wrapper instead of being validated by OpenClaw's absolute command and trusted-directory checks.
  - confidence: 0.89
- **[medium] Raw CLI stderr reaches SecretRef errors:** `scripts/bw-exec-resolver.mjs:38`
  - body: Nonzero `bw` exits are converted from raw stderr into protocol errors, which can be surfaced by OpenClaw during SecretRef resolution. A secret resolver should return redacted fixed messages for expected failure modes.
  - confidence: 0.83
