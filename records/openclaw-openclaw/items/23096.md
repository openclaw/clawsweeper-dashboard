The PR adds Bitwarden/Vaultwarden SecretRef docs, a protocol-v1 Bitwarden CLI exec resolver script with declarations, and tests for parsing, field extraction, batching, and child-process behavior.

## Best Possible Solution

Keep this as the focused Bitwarden/Vaultwarden candidate, but merge only after maintainers choose the architecture and the chosen path is packaged, documented, hardened, and validated against the current SecretRef baseline.

## Solution Assessment

No. The proposed direction is useful, but as written it is not yet the best mergeable solution because the bundled script is not packaged for npm installs and the wrapper reintroduces PATH-based CLI resolution inside a security-sensitive SecretRef flow.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.78

Full review comments:

- **[P2] Package the bundled resolver before documenting it:** `docs/gateway/secrets.md:339`
  - body: The docs tell users to use the bundled `scripts/bw-exec-resolver.mjs`, but the root package only includes a small whitelist of scripts and this PR does not add the new resolver there. Users following the public docs from an npm install would not have the documented file, so either package the resolver or document this as source-checkout-only.
  - confidence: 0.88
- **[P2] Resolve the Bitwarden CLI through a trusted path:** `scripts/bw-exec-resolver.mjs:28`
  - body: The wrapper runs `execFile("bw", ...)` while the docs pass `PATH` into the provider environment. That bypasses the exec provider's absolute-command and `trustedDirs` checks for the second-stage Bitwarden process, so a bad PATH entry could run the wrong binary with `BW_SESSION`; accept an absolute `BW_BIN`/argv path or validate the resolved executable before spawning.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The PR is additive but security-sensitive because the resolver handles secrets and spawns a second process using PATH-based CLI resolution.

Concerns:

- **[medium] Second-stage CLI bypasses trusted command policy:** `scripts/bw-exec-resolver.mjs:28`
  - body: Core exec providers require absolute command paths and trusted-dir validation, but the new wrapper executes `bw` by PATH after receiving `PATH` and Bitwarden credentials in the child environment. This can run an unintended binary and expose secrets to it.
  - confidence: 0.82
