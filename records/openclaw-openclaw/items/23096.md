## Summary

Keep this PR open, but do not merge it as-is. Current main still has no Bitwarden/Vaultwarden support, and this branch remains the focused candidate, but the proposed wrapper bypasses OpenClaw's exec-provider command hardening by resolving the real `bw` binary through `PATH` while receiving Bitwarden credential environment.

## What This Changes

The PR adds Bitwarden/Vaultwarden SecretRef documentation, a protocol-v1 Bitwarden CLI exec resolver script with TypeScript declarations, and tests for parsing, field extraction, grouping, and child-process batch resolution.

## Best Possible Solution

Land Bitwarden/Vaultwarden support only after the route is explicit: preferably as a `secrets-bitwarden` plugin once #72548 provides the secret-provider seam, or as a deliberately hardened exec-wrapper/docs integration that requires an absolute validated Bitwarden binary path, preserves error redaction, updates the changelog, and coordinates with #29110.

## Reproduction Assessment

Not applicable as a feature PR rather than a bug report. For the blocking finding, yes: configure the documented wrapper with `PATH` passed through, put a different `bw` earlier in `PATH`, and the wrapper will execute that binary with the Bitwarden credential environment.

## Solution Assessment

No, not as currently proposed. The exec-wrapper approach can work only after binary selection is hardened, and maintainers still need to decide whether the plugin seam is the narrower long-term home.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Resolve the Bitwarden CLI without PATH search:** `scripts/bw-exec-resolver.mjs:28`
  - body: The configured exec provider validates only this wrapper script, but the wrapper then calls `execFile("bw", ...)` while the docs pass `PATH` along with Bitwarden credential env. A writable or overridden `PATH` can make OpenClaw run a different `bw` binary with those credentials, bypassing the exec-provider path hardening. Require an absolute validated Bitwarden binary path, such as a checked `BW_BIN`, before spawning it.
