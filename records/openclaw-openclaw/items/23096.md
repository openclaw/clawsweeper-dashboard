## Summary

Keep open: current main still does not ship Bitwarden/Vaultwarden SecretRef support, the plugin SecretRef seam is still unmerged, and this PR still has a security-sensitive PATH-selected CLI execution issue that should block merge.

## What This Changes

The PR branch adds Bitwarden/Vaultwarden SecretRef docs, a protocol-v1 Bitwarden CLI exec resolver script, TypeScript declarations, and resolver tests.

## Best Possible Solution

Keep this PR open until maintainers choose the integration route; if the exec-wrapper route is kept, require an absolute validated Bitwarden binary path such as a checked BW_BIN before spawning it.

## Reproduction Assessment

Not applicable for the feature request itself. For the blocking security finding, yes: configure the documented wrapper, put another bw earlier in PATH, and the wrapper will execute that binary with BW_SESSION passed through.

## Solution Assessment

No, not as currently proposed. The exec-wrapper approach can be viable only after binary selection is hardened, and maintainers still need to decide whether the plugin seam is the longer-term home.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Resolve the Bitwarden CLI without PATH search:** `scripts/bw-exec-resolver.mjs:29`
  - body: The configured exec provider validates only the wrapper script, but the wrapper then calls `execFile("bw", ...)` while the docs pass `PATH` with Bitwarden credential env. A writable or overridden PATH can make OpenClaw run a different `bw` binary with those credentials, bypassing the exec-provider command path checks. Require an absolute validated Bitwarden binary path before spawning it.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff introduces a command-execution wrapper that spawns a PATH-selected Bitwarden CLI with secret-bearing environment.

Concerns:

- **[medium] PATH-resolved Bitwarden binary bypasses exec hardening:** `scripts/bw-exec-resolver.mjs:29`
  - body: OpenClaw validates the configured wrapper command, but the wrapper invokes `bw` by name and the docs pass `PATH` along with Bitwarden credentials. That lets the real secrets-reading binary be selected outside OpenClaw's command path validation and trustedDirs checks.
  - confidence: 0.86
