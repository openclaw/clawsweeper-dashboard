## Summary

Keep open. The PR has the protected `maintainer` label and the latest head still has blocking gateway registry regressions: hidden core method collisions can crash startup/reload, and core descriptor scope metadata is over-normalized.

## What This Changes

The PR adds a gateway method descriptor registry and routes gateway request dispatch, advertised method listing, plugin/channel metadata, and WebSocket handling through descriptor-backed metadata.

## Best Possible Solution

Keep the descriptor-registry direction, but preserve legacy gateway loading contracts: reject plugin collisions against the full core handler set before registry construction, and keep core descriptor scopes aligned with legacy method-scope metadata.

## Reproduction Assessment

Yes. From source inspection, current main allows plugin collision checks against the advertised method list while hidden core handlers exist, and the PR then rejects duplicate registry names; the scope bug is reproducible by checking the registry scope for `config.get`.

## Solution Assessment

No as written. The registry abstraction is a reasonable maintainability direction, but the patch needs owner-aware scope normalization and full-core collision validation before it can safely replace the legacy dispatch path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Reject hidden core method collisions before registry build:** `src/gateway/server.impl.ts:1102`
  - body: `buildAttachedGatewayMethodRegistry` combines full core descriptors with plugin descriptors, but plugin registration still only guards against the advertised base method list. A plugin can register a hidden core name such as `config.openFile` or `chat.inject` and then make gateway startup or reload throw `gateway method already registered`, so the collision needs to be rejected before registry construction.
  - confidence: 0.91
- **[P2] Preserve declared scopes for core descriptors:** `src/gateway/methods/registry.ts:47`
  - body: `normalizeDescriptor` runs plugin reserved-prefix normalization for every descriptor, including core descriptors. That changes metadata such as `config.get` from its declared read scope to admin because `config.*` is plugin-reserved, so the registry scope view no longer matches the legacy method-scope contract.
  - confidence: 0.87
