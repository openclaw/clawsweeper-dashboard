## Summary

Keep open: latest head still contains two source-reproducible gateway registry regressions from the prior review. Real behavior proof is sufficient and the security pass found no separate supply-chain concern, but the PR should not merge until the registry fixes land.

## What This Changes

The PR adds a gateway method descriptor registry and routes gateway dispatch, advertised method listing, plugin inspect metadata, and WhatsApp web-login declarations through descriptor-backed metadata.

## Best Possible Solution

Keep the registry refactor, but make descriptor normalization owner-aware and validate plugin method names against the full core descriptor set before registry construction.

## Reproduction Assessment

Yes. Source inspection of PR head shows a hidden core method such as `config.openFile` can pass the advertised-list plugin guard and collide during registry build, and `config.get` scope metadata is still coerced by plugin-only policy.

## Solution Assessment

No as written. The registry direction is maintainable, but it must preserve legacy plugin collision handling and core least-privilege metadata before replacing the current dispatch path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Reject hidden core collisions before building the registry:** `src/gateway/server.impl.ts:1111`
  - body: `buildAttachedGatewayMethodRegistry` registers full core descriptors, including hidden handlers, before plugin descriptors. Plugin loading still validates against the advertised method list, so a plugin can register a hidden core name such as `config.openFile` and then make attach/reload throw `gateway method already registered`; validate plugin names against the full core descriptor set before registry construction.
  - confidence: 0.91
- **[P2] Preserve declared scope for non-plugin descriptors:** `src/gateway/methods/registry.ts:33`
  - body: `normalizeDescriptor` applies the plugin reserved-prefix policy to every descriptor, including core descriptors. That turns declared core metadata such as `config.get` from `operator.read` into `operator.admin`, so registry scope views no longer match the core least-privilege contract; only plugin-owned descriptors should get plugin namespace coercion.
  - confidence: 0.88
