## Summary

Keep open: the descriptor-registry direction is reasonable and the PR body has sufficient live gateway proof, but current PR head still has two source-reproducible gateway registry regressions that should block merge. The protected `maintainer` label also rules out cleanup closing.

## What This Changes

The PR adds a gateway method descriptor registry and routes gateway dispatch, advertised method listing, plugin inspect metadata, and WhatsApp web-login declarations through descriptor-backed method metadata.

## Best Possible Solution

Keep the registry refactor, but make descriptor normalization owner-aware and validate plugin method names against the full core descriptor set before registry construction.

## Reproduction Assessment

Yes. Source inspection of PR head shows a plugin method named `config.openFile` can pass the advertised-list plugin guard and then collide with a hidden core descriptor during registry build; it also shows `config.get` registry metadata is coerced from read to admin.

## Solution Assessment

No as written. The descriptor registry is a maintainable direction, but it must preserve legacy collision handling and core scope metadata before replacing the current dispatch/metadata path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Reject hidden core collisions before building the registry:** `src/gateway/server.impl.ts:1111`
  - body: `buildAttachedGatewayMethodRegistry` now registers full core descriptors, including hidden handlers, before plugin descriptors. Plugin loading still validates against the advertised method list, so a plugin can register a hidden core name such as `config.openFile` or `chat.inject` and make gateway attach/reload throw `gateway method already registered`; validate plugin names against the full core descriptor set before registry construction.
  - confidence: 0.91
- **[P2] Preserve declared scope for non-plugin descriptors:** `src/gateway/methods/registry.ts:33`
  - body: `normalizeDescriptor` runs `normalizePluginGatewayMethodScope` for every descriptor, including core descriptors. That plugin-only reserved-prefix rule turns core metadata such as `config.get` from `operator.read` into `operator.admin`, so registry scope views no longer match the declared core least-privilege contract.
  - confidence: 0.88
