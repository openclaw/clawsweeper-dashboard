## Summary

Keep this active PR open: it has a protected `maintainer` label and the latest head fixes the earlier registry-normalization findings, but the current diff downgrades persistent exec-approval policy RPCs from admin scope to approval scope.

## What This Changes

The PR adds a descriptor-backed gateway method registry and migrates gateway dispatch, advertised method listing, plugin/channel declarations, and related tests to descriptor metadata.

## Best Possible Solution

Keep the descriptor registry refactor, but preserve the existing admin-only contract for `exec.approvals.*` policy RPCs and add a regression test that approvals-only clients cannot call the policy setters.

## Reproduction Assessment

Yes. Source inspection shows current main resolves `exec.approvals.*` through the reserved admin prefix, while the PR head declares those policy RPCs as `operator.approvals`.

## Solution Assessment

No. The registry design is a good direction, but this implementation should carry over the current admin scope for persistent exec-approval policy RPCs while keeping one-shot `exec.approval.*` methods on `operator.approvals`.
