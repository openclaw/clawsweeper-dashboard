## Summary

This PR should stay open because it is a protected maintainer-labeled gateway refactor and the current branch still has source-proven registry regressions in the descriptor normalization and hidden-core collision path. The PR direction is useful, but it needs targeted fixes before merge.

## What This Changes

The PR adds a gateway method descriptor registry, routes gateway dispatch and advertised-method metadata through descriptor-backed registries, updates plugin/channel gateway method metadata, and refreshes related tests and prompt snapshots.

## Best Possible Solution

Keep the descriptor registry refactor, but make descriptor normalization owner-aware and validate plugin gateway method names against the full core descriptor set before registry construction.

## Reproduction Assessment

Yes for the review findings. Source inspection of the PR diff shows `config.get` is declared read-scope but normalized through plugin reserved-prefix policy, and hidden core descriptors can collide after plugin validation because the validation base is still the advertised method list.

## Solution Assessment

No as written. The descriptor registry is a maintainable direction, but the PR needs owner-aware scope normalization and full core-name collision validation before it can safely replace the current gateway metadata paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Reject hidden core collisions before registry build:** `src/gateway/server.impl.ts:1111`
  - body: `buildAttachedGatewayMethodRegistry` registers hidden core descriptors before plugin descriptors, but plugin loading still validates names against the advertised method list. A plugin can register a hidden core name such as `config.openFile`, pass the plugin guard, and then crash attach/reload when the registry throws `gateway method already registered`; validate plugin names against the full core descriptor set before registry construction.
  - confidence: 0.91
- **[P2] Preserve declared scope for non-plugin descriptors:** `src/gateway/methods/registry.ts:33`
  - body: `normalizeDescriptor` applies plugin reserved-prefix policy to every non-node/non-dynamic descriptor. Core descriptors such as `config.get` and `exec.approvals.get` are therefore coerced to `operator.admin`, so registry scope views no longer match the declared least-privilege metadata; only plugin-owned descriptors should get plugin namespace coercion.
  - confidence: 0.89
