## Summary

Keep open: the registry refactor is useful, but the current PR head still has two source-backed gateway correctness regressions and carries the protected maintainer label.

## What This Changes

The PR adds a gateway method descriptor registry and routes gateway dispatch, advertised method listing, plugin metadata, inspect/status output, and WhatsApp web-login declarations through descriptor-backed method metadata.

## Best Possible Solution

Keep the descriptor-registry direction, but preserve legacy gateway contracts by validating plugin names against the full core descriptor set and applying reserved-prefix scope coercion only to plugin-owned descriptors.

## Reproduction Assessment

Yes. From source inspection, register a plugin method named config.openFile and the current advertised-list guard permits it, then the PR registry build throws on the duplicate hidden core descriptor; building the registry for config.get also shows the scope metadata is coerced from read to admin.

## Solution Assessment

No as written. The registry abstraction is a reasonable maintainability direction, but it needs owner-aware scope normalization and full-core collision validation before replacing the legacy dispatch/metadata path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Reject hidden core collisions before building the registry:** `src/gateway/server.impl.ts:1111`
  - body: `buildAttachedGatewayMethodRegistry` now merges full core descriptors with plugin descriptors, but plugin registration still only rejects names from the advertised gateway method list. A plugin can register a hidden core name such as `config.openFile` or `chat.inject` and make gateway startup or reload throw `gateway method already registered`, so reject those names before registry construction.
  - confidence: 0.91
- **[P2] Preserve declared scope for non-plugin descriptors:** `src/gateway/methods/registry.ts:33`
  - body: `normalizeDescriptor` applies plugin reserved-prefix normalization to every descriptor, including core descriptors. That changes declared core metadata such as `config.get` from `operator.read` to `operator.admin`, so registry scope views no longer match the legacy least-privilege method contract.
  - confidence: 0.88
