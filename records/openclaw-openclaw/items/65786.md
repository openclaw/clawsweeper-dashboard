## Summary

Keep open: current main and the latest release still reject the reported Feishu config shape before runtime because Feishu keeps `streaming` boolean-only and still omits `contextVisibility` from the strict schema while the runtime reads that field.

## What This Changes

The issue reports that upgrading to OpenClaw 2026.4.11 leaves `channels.feishu` invalid, and `openclaw doctor --fix` does not repair legacy Feishu config fields.

## Best Possible Solution

Add Feishu-owned schema/manifest/docs support for runtime-supported `contextVisibility`, plus a doctor compatibility migration for legacy scalar Feishu `streaming` values and focused regression tests.

## Reproduction Assessment

Yes, source-reproducible. The provided config contains `streaming: "progress"` and `contextVisibility: "all"`, which current strict Feishu validation rejects even though Feishu runtime reads context visibility.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Feishu plugin schema/metadata and doctor compatibility path rather than a generic startup migration or a global relaxation of channel validation.
