## Summary

Keep open: current main and the latest release still reject the reported Feishu config shape before runtime because Feishu keeps `streaming` boolean-only and still omits `contextVisibility` from the strict schema, while the runtime reads that field. The related open Feishu upgrade issue and PR cover different stale keys, so they do not supersede this specific report.

## What This Changes

The issue asks OpenClaw to repair an upgraded Feishu config where `channels.feishu.streaming: "progress"` and `channels.feishu.contextVisibility: "all"` make strict validation fail and `openclaw doctor --fix` does not help.

## Best Possible Solution

Add Feishu-owned schema, manifest, docs, and regression-test support for runtime-supported `contextVisibility`, plus a doctor compatibility migration for legacy Feishu scalar `streaming` values before strict validation runs.

## Reproduction Assessment

Yes. The provided config is source-reproducible on current main: `streaming: "progress"` fails the Feishu boolean schema, `contextVisibility` is an additional property, and validation reports that under `channels.feishu` before runtime.

## Solution Assessment

Yes. The narrow maintainable direction is a Feishu-owned schema/manifest alignment plus doctor compatibility migration, not a global relaxation of channel validation or a startup-time core migration.
