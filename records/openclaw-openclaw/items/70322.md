## Summary

Keep this open. Current main still routes `github-copilot/claude-opus-4.7` through the Anthropic Messages adaptive-thinking path, maps low/minimal thinking to `output_config.effort: "low"`, and records non-timeout format failures into auth-profile failure state; open PR #70584 is the active closing implementation candidate and has not merged.

## What This Changes

The issue reports that `github-copilot/claude-opus-4.7` rejects low/minimal adaptive `output_config.effort` values and that the resulting format failure can put the Copilot auth profile into cooldown.

## Best Possible Solution

Keep this issue paired with PR #70584 until maintainers land that PR or an equivalent targeted clamp for `github-copilot/claude-opus-4.7`; then handle the cooldown-poisoning policy as a separate focused provider/auth decision if it remains unfixed.

Yes. Current main can be statically traced from a low/minimal thinking level on Copilot Opus 4.7 to an Anthropic Messages adaptive payload with `output_config.effort: "low"`; the reporter and PR context provide the live Copilot 400 evidence for that exact payload.

## Solution Assessment

Yes for the immediate request rejection: a provider/model-specific clamp at the Anthropic Messages transport boundary is the narrowest maintainable fix. The auth cooldown behavior is adjacent and likely needs a separate policy change rather than being bundled into the effort clamp without owner review.
