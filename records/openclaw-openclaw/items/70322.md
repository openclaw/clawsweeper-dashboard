## Summary

Keep open. Current main still sends `output_config.effort: "low"` for low/minimal adaptive-thinking Opus 4.7 requests, and the GitHub Copilot wrapper does not normalize that payload; the auth-profile cooldown side effect appears addressed on current main, but the rejected request shape remains.

## What This Changes

The issue asks OpenClaw to stop sending unsupported `output_config.effort` values to `github-copilot/claude-opus-4.7` and to avoid provider-wide auth-profile cooldown cascades from those schema rejections.

## Best Possible Solution

Revise or replace the linked implementation with provider-aware Copilot Opus 4.7 adaptive-effort compatibility, with regression coverage for low/minimal and the related high/xhigh evidence while preserving direct Anthropic behavior.

## Reproduction Assessment

Yes. A `github-copilot/claude-opus-4.7` Anthropic Messages request with low/minimal reasoning can be traced on current main to `output_config.effort: "low"`, and the issue includes live Copilot `invalid_reasoning_effort` logs.

## Solution Assessment

Yes for provider-aware transport-boundary normalization or a supported-effort capability table. A model-only clamp is not the best complete fix because direct Anthropic Opus 4.7 behavior should remain intact and related evidence reports broader high/xhigh failures.
