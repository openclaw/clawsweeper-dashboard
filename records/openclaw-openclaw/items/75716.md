## Summary

Keep open. Current main still violates the documented Google Chat SecretRef precedence contract, and the latest PR diff is a narrow plugin-owned fix with regression coverage and no concrete review findings.

## What This Changes

The PR updates Google Chat service-account resolution to fail on unresolved sibling serviceAccountRef credentials, removes resolved sibling refs from runtime snapshots, and adds regression coverage plus a changelog entry.

## Best Possible Solution

Land the Google Chat-owned resolver and secret-contract fix after maintainer review and CI, without changing shared SecretRef runtime semantics.

## Reproduction Assessment

Yes. On current main, a Google Chat config with both inline serviceAccount and unresolved serviceAccountRef reproduces the fallback because inline parsing returns before the ref guard; the PR adds a targeted regression for that shape.

## Solution Assessment

Yes. The proposed approach is the narrow maintainable fix: make unresolved sibling refs authoritative in the account resolver and remove the sibling marker only after the Google Chat secret contract resolves it into the runtime snapshot.
