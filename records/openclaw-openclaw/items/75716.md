## Summary

Keep open. Current main still has the documented Google Chat SecretRef precedence bug, and the PR appears to be a narrow, owner-local fix with regression coverage and a changelog entry; it should proceed through normal maintainer review and CI rather than cleanup closure.

## What This Changes

The PR changes Google Chat credential resolution so unresolved `serviceAccountRef` takes precedence over inline `serviceAccount`, removes resolved sibling refs from runtime snapshots, adds regression tests, and adds a changelog entry.

## Best Possible Solution

Land the Google Chat-owned fix after CI or targeted test proof, keeping the precedence repair in the plugin resolver and secret contract without changing shared SecretRef runtime semantics.

## Reproduction Assessment

Yes. Source inspection on current main shows a high-confidence reproduction: configure Google Chat with both inline `serviceAccount` and unresolved sibling `serviceAccountRef`, then call `resolveGoogleChatAccount`; the resolver returns inline credentials before checking the ref.

## Solution Assessment

Yes. The PR is the narrow maintainable solution: it makes unresolved sibling refs authoritative in the Google Chat resolver and only scrubs the sibling marker after the Google Chat secret contract has resolved it into the runtime snapshot.
