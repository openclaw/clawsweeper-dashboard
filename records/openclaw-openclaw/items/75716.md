## What This Changes

The PR changes Google Chat service-account resolution to fail on unresolved `serviceAccountRef`, scrubs resolved sibling refs from runtime snapshots, adds regression coverage, and adds a changelog entry.

## Best Possible Solution

Land the Google Chat-owned fix after CI or targeted test proof, keeping the precedence repair in the plugin resolver/secret contract rather than changing shared SecretRef runtime semantics.

## Reproduction Assessment

Yes. On current main, `resolveGoogleChatAccount` with both inline `serviceAccount` and unresolved `serviceAccountRef` reproduces the bug because inline parsing returns before the ref guard; the PR adds a direct regression test for that shape.

## Solution Assessment

Yes. The latest approach is the narrow maintainable fix: make unresolved sibling refs authoritative in the account resolver, and scrub the sibling marker only when the Google Chat secret contract has resolved it into the runtime snapshot.
