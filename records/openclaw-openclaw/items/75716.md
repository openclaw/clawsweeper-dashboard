## Summary

Keep open. Current main still has the Google Chat credential-precedence bug, and this PR is a focused plugin-owned fix with no blocking review findings; it should proceed through normal maintainer review and CI.

## What This Changes

The PR changes Google Chat service-account resolution so unresolved `serviceAccountRef` values take precedence over legacy inline `serviceAccount`, removes resolved sibling refs from runtime snapshots, and adds regression coverage plus a changelog entry.

## Best Possible Solution

Land the Google Chat-owned resolver and secret-contract fix after maintainer review and CI, without changing shared SecretRef runtime semantics.

## Reproduction Assessment

Yes. On current main, a Google Chat config with both inline `serviceAccount` and unresolved `serviceAccountRef` follows the inline return path before the ref guard; the PR adds a targeted regression for that shape.

## Solution Assessment

Yes. The patch is the narrow maintainable fix: make unresolved sibling refs authoritative in the Google Chat resolver and clear the sibling marker only after runtime SecretRef resolution succeeds.
