## Summary

Keep this PR open for maintainer review: the patch is a focused fix for documented Google Chat SecretRef precedence, but it intentionally changes shipped credential fallback behavior for stale mixed configs, so CI and contributor proof do not settle the compatibility decision.

## What This Changes

The PR reorders Google Chat service-account resolution to reject unresolved `serviceAccountRef` before inline credentials, removes resolved sibling refs from runtime snapshots, adds regression tests, and adds a changelog entry.

## Best Possible Solution

Land the plugin-owned precedence fix if maintainers accept documented fail-closed SecretRef behavior, or add a focused doctor/audit diagnostic first if they want a smoother upgrade path for stale mixed Google Chat credential configs.

## Reproduction Assessment

Yes. Current `main` and `v2026.6.8` parse inline `serviceAccount` before checking `serviceAccountRef`, while the docs say the ref takes precedence; I did not run tests because this review is read-only.

## Solution Assessment

Yes for the code location, subject to maintainer compatibility acceptance. The PR changes the plugin-owned credential resolver and materialized SecretRef snapshot path directly instead of adding a second policy layer.
