## What This Changes

The PR adds a Windows-only install-plan guard that ignores `OPENCLAW_WRAPPER` when it resolves to the generated `gateway.cmd` task script, extracts the shared task-script path helper, and adds regression coverage.

## Best Possible Solution

Land the narrow install-plan guard after redacted real Windows proof shows the self-wrapper state is cleared, regenerated `gateway.cmd` launches the OpenClaw entrypoint directly, and gateway health succeeds.

## Reproduction Assessment

## Solution Assessment

Yes. The proposed guard is the narrow maintainable boundary because it clears only the generated Windows task script self-wrapper before program arguments and service env are built, while preserving valid wrappers.
