## Summary

Keep open: current main and the latest release still lack an accepted `requestLabels` provider config contract and do not inject top-level labels into Google or Vertex generateContent payloads. The recent reseller-billing comment adds a concrete production Vertex use case, while the related open Vertex PRs address ADC/provider routing rather than billing labels.

## What This Changes

The issue requests configurable Google/Vertex request labels so operators can attach billing attribution metadata to outbound Gemini API calls.

## Best Possible Solution

Add a documented `google-vertex` request-label option with GCP label validation and provider-owned injection into Vertex streaming and non-streaming content requests, while leaving direct Gemini unchanged unless upstream support is proven.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a provider feature request. Source inspection gives a high-confidence verification path: current main has no accepted `requestLabels` config field and no Google/Vertex payload label-injection path.

## Solution Assessment

Yes, with the scope tightened to Vertex: provider-owned validation and payload injection is the maintainable direction. Maintainers still need to approve the public config shape and static-versus-runtime label policy before implementation.
