## Summary

Keep open: current main and v2026.6.6 still lack a `requestLabels` or equivalent Google/Vertex billing-label contract, while the official Vertex documentation supports labels for `generateContent` and `streamGenerateContent`. This remains a concrete provider feature request that needs maintainer product/config approval rather than conservative cleanup. ([docs.cloud.google.com](https://docs.cloud.google.com/gemini-enterprise-agent-platform/models/capabilities/add-labels-to-api-calls))

## What This Changes

The issue requests configurable labels on outbound Google Gemini or Vertex `generateContent` requests so operators can attribute GCP billing costs by deployment or workload.

## Best Possible Solution

Keep this as the canonical label-attribution request and, after maintainer approval, add a documented `google-vertex` request-label contract with validation, provider-owned payload injection, and focused tests/docs.

## Reproduction Assessment

Yes for source-level verification: current main has no `requestLabels` schema/type field and the Google/Vertex request builder emits no `labels` payload. This is a feature absence rather than a broken existing behavior repro.

## Solution Assessment

Yes, with scope tightened to Vertex. A provider-owned label config with Google label validation is the maintainable direction, but maintainers still need to decide the public config shape and static-versus-runtime label policy.
