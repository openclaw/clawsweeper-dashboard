## Summary

Keep open: current main still has Google Vertex transport support but no accepted or injected configurable billing labels. This is a provider-owned enhancement, not a broken existing OpenClaw contract.

## What This Changes

The issue requests configurable labels on Google/Gemini API requests so GCP billing can be grouped by deployment or workload.

## Best Possible Solution

Add a Google-provider owned, Vertex-only labels option with GCP label validation, request-body tests, docs for the chosen config path, and no generic core behavior for all providers.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a bug reproduction because this is a feature request; source inspection confirms current main rejects the proposed top-level key and never adds labels to Google request bodies.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers pick the public config contract; the maintainable direction is a narrow Google provider change scoped to Vertex labels with validation and docs.
