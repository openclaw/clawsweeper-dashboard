Keep open: current main and the latest release still lack any `requestLabels` or equivalent Google/Vertex billing-label contract, while the request remains concrete and backed by current Vertex API label support. This needs maintainer product/config approval rather than conservative cleanup.

The issue requests configurable Google/Vertex request labels for Gemini API calls so operators can attribute GCP billing by deployment, environment, agent, skill, or workload.

## Best Possible Solution

Keep this as the canonical request and, after maintainer approval, add a Vertex-only Google plugin label contract with validation, payload injection, focused transport/schema tests, and public config docs.
