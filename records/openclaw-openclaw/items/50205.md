## Summary

Keep open: current main and v2026.6.10 still lack a Google/Vertex request-label config contract or payload injection, while the issue comments add Vertex-only API context and a concrete reseller-billing use case that make this a real product/config decision.

## What This Changes

Request configurable request labels on outbound Google/Vertex Gemini generateContent calls so GCP billing can attribute OpenClaw spend by deployment or workload.

## Best Possible Solution

Keep this issue open as the canonical request and, after maintainer approval, add a Vertex-only Google plugin label contract with validation, payload injection, focused schema/transport tests, and docs.

## Reproduction Assessment

Yes. Source inspection shows the proposed requestLabels key is not accepted by the strict provider schema and the Google/Vertex request builder emits no labels field; I did not run a live Vertex request in this read-only review.

## Solution Assessment

Unclear until maintainers choose the contract. A provider-owned, Vertex-only label field is the likely maintainable path; a broad generic request-body escape hatch would be a riskier fit for the repo's config-surface policy.
