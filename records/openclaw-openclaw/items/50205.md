## What This Changes

The issue asks OpenClaw to let operators configure Gemini/Vertex request labels for GCP billing attribution.

## Best Possible Solution

Add a documented `google-vertex` request-label option with GCP label validation and provider-owned injection into both `generateContent` and `streamGenerateContent`, leaving non-Vertex Gemini unchanged unless upstream support is proven.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence verification path: current main has no `requestLabels` schema field and no Google payload label-injection path.

## Solution Assessment

Yes, with scope tightened to `google-vertex`: provider-owned validation and payload injection is the maintainable direction. Maintainers still need to approve the exact config shape and any automatic runtime labels before implementation.
