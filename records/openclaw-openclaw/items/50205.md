## Summary

Keep open: current main and the latest release still lack an accepted `requestLabels` provider config contract and do not inject top-level labels into Google or Vertex `generateContent` payloads. The related Vertex ADC/routing work improves the provider path but does not cover billing-label attribution.

## What This Changes

Requested configurable Gemini or Vertex request labels so GCP operators can attribute billing by deployment, environment, agent, or workload.

## Best Possible Solution

Add a documented `google-vertex` request-label option with GCP label validation and provider-owned injection into Vertex streaming and non-streaming content requests, leaving direct Gemini unchanged unless upstream support is proven.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a provider feature request. Source inspection gives a high-confidence verification path: current main rejects `requestLabels` in provider config and has no Google/Vertex payload label-injection path.

## Solution Assessment

Yes, with the scope tightened to Vertex. Provider-owned validation and payload injection is the maintainable direction, but maintainers still need to approve the public config shape and static-versus-runtime label policy.
