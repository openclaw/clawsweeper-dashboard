## Summary

Keep open: current main still lacks a configurable request-label contract and does not inject labels into Google/Gemini request bodies; no duplicate or merged fix was found, so this remains a narrow provider enhancement needing maintainer approval.

## What This Changes

Request configurable static labels on outbound Gemini or Vertex generateContent calls so GCP billing can attribute costs by deployment or workload.

## Best Possible Solution

Add a documented `google-vertex` request-label option with GCP label validation, provider-owned payload injection, focused config and transport tests, and a changelog entry after the config shape is approved.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection shows current main lacks a `requestLabels` config field and the Google request builder still has no labels injection path.

## Solution Assessment

Yes, with scope tightened to `google-vertex`: provider-owned validation and payload injection is the maintainable direction, while a generic core label mechanism or non-Vertex Gemini injection needs upstream API proof first.
