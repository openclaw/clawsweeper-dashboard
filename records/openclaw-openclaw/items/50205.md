Keep open: current main still does not accept or inject Google request labels, and the request is a narrow Google Vertex provider enhancement rather than a broken existing contract.

## What This Changes

Request to let Google/Gemini provider configuration attach static request labels to outbound generateContent calls for GCP billing attribution.

## Best Possible Solution

Implement a Vertex-only Google provider request-label option through the existing provider params/config path, with validation, regression tests, docs, and no label injection for direct Gemini API calls.

Yes. The narrow maintainable direction is provider-owned Vertex support, preferably documented as a Google provider runtime param, rather than generic core behavior for every model provider.

Add a documented provider runtime parameter, preferably `models.providers.google-vertex.params.requestLabels`, accepting a record of static string labels for GCP billing attribution. Validate or reject invalid label keys/values according to Google label constraints, pass the labels through the existing extra-params/provider transport path as needed, and inject `labels` only into Google Vertex generateContent/streamGenerateContent request bodies.
