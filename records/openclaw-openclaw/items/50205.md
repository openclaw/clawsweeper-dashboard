## Best Possible Solution

Add a documented Vertex-only Google request-label option with GCP label validation, provider-owned payload injection, focused tests, and a changelog entry.

## Solution Assessment

Yes, with scope tightened to `google-vertex`: provider-owned validation and payload injection is the narrow maintainable direction. A generic core label mechanism or non-Vertex Gemini injection would be a less safe contract without upstream API support.
