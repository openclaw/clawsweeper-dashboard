## Summary

Keep open: current main and the latest release still lack a `requestLabels` provider config contract and the Google/Vertex request builder does not inject billing labels. The related Vertex ADC/onboarding work improves the provider path but does not cover request-label attribution, so this remains a maintainer product/config decision.

## What This Changes

Request to add configurable request labels to outbound Google/Gemini or Vertex `generateContent` calls so GCP billing can attribute costs by deployment or workload.

## Best Possible Solution

Keep this issue as the canonical label-attribution request and, after maintainer approval, add a documented `google-vertex` request-label config with validation, provider-owned payload injection, and focused tests/docs; leave direct Gemini unchanged unless upstream support is proven.

## Reproduction Assessment

Yes for source-level verification: current main has no `requestLabels` schema/type field and the Google request builder emits no `labels` payload. This is a feature absence rather than a broken existing behavior repro.

## Solution Assessment

Yes, with the scope tightened to Vertex. A provider-owned label config with Google label validation is the maintainable direction, but maintainers still need to decide the public config shape and static-versus-runtime label policy.
