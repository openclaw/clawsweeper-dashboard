## Summary

Keep open: current main and the latest release still lack a supported provider config key or Google transport payload path for Gemini/Vertex billing labels, while the discussion and Google docs show a real Vertex billing-attribution use case.

## What This Changes

Requested support for configurable labels on Google/Vertex Gemini generateContent requests for GCP billing attribution.

## Best Possible Solution

Define a maintainer-approved Google/Vertex billing-label contract, validate label keys and values against Google’s documented constraints, and inject labels only on API routes where Google documents request labels.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: current and latest-release schema/transport paths have no requestLabels config key or generated request-body labels field.

## Solution Assessment

Unclear. Google’s documented labels request field is the right upstream primitive for Vertex, but OpenClaw still needs a maintainer decision on config naming, validation, and whether this is Vertex-only or provider-param driven.
