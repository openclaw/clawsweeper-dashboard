## Best Possible Solution

Add a documented `google-vertex`-only request-label option with GCP label validation, provider-owned payload injection, focused tests, and a changelog entry.

## Solution Assessment

Yes, if scoped to `google-vertex`: provider-owned validation and payload injection is the narrow maintainable direction. A generic core label mechanism or non-Vertex Gemini injection would be less safe without an upstream API contract.
