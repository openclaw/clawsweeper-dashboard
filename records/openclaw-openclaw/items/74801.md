## Summary

Keep open. Current main bounds OpenClaw-managed file logs, but Linux systemd service output is still surfaced through journald and neither the generated unit nor docs bound journal storage.

## What This Changes

The issue asks OpenClaw to address Linux systemd installs where file-log rotation does not limit journald storage growth.

## Best Possible Solution

Document journald retention as a separate Linux/systemd log-storage surface with SystemMaxUse or RuntimeMaxUse and manual journalctl vacuum options, while leaving automatic vacuuming or output rerouting for explicit maintainer review.

## Reproduction Assessment

Yes for the actionable gap: current main shows systemd service logs are surfaced through journalctl while file-log rotation and docs do not cover journald retention. A full disk-growth reproduction would require a sustained Linux systemd user-service run and journalctl --disk-usage on that host.

## Solution Assessment

Yes for a docs-first repair: it is the narrowest maintainable fix that corrects operator expectations without changing host-wide journal policy. Automatic vacuuming or redirecting systemd output should be evaluated separately.
