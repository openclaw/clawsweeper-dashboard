## Summary

Keep open. Current main still treats Linux service logs as journald while the documented rotation settings cover only OpenClaw-managed file logs, and there is still no journald retention or vacuum guidance.

## What This Changes

Request that Linux/systemd logging guidance or service behavior account for journald growth instead of only rotating the OpenClaw file log.

## Best Possible Solution

Document journald as a separate Linux/systemd log-storage surface, including retention settings and manual vacuum commands, while leaving automatic vacuuming or output rerouting out of the narrow fix.

## Reproduction Assessment

Yes for the actionable gap: current main shows Linux service logs are surfaced through `journalctl`, while file-log rotation and docs do not cover journald retention. A full storage-growth reproduction would require running a Linux systemd user service long enough to measure `journalctl --disk-usage`.

## Solution Assessment

Yes for a docs-first repair: it satisfies the issue's documented guidance option without editing host-wide journal policy. Automatic journal vacuuming or redirecting service output should be handled only after explicit maintainer product review.
