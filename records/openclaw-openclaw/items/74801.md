## Summary

Keep open. Current main still rotates OpenClaw file logs only, while Linux systemd service guidance and runtime hints rely on journald without documenting retention or adding a generated unit policy that bounds journal storage.

## What This Changes

The issue asks OpenClaw to address Linux systemd log growth by adding journald retention/vacuum guidance or an equivalent service/file-log behavior change.

## Best Possible Solution

Document journald as a separate Linux/systemd log-storage surface with `SystemMaxUse`/`RuntimeMaxUse` and manual `journalctl --vacuum-*` guidance, leaving automatic vacuuming or output routing for a separate maintainer decision.

## Reproduction Assessment

Yes for the actionable gap: current main can be inspected to show systemd logs are surfaced through journald while file-log rotation and docs do not cover journal retention. A full disk-growth reproduction would require a sustained Linux systemd user-service run and `journalctl --disk-usage`.

## Solution Assessment

Yes for a docs-first repair. It is the narrowest maintainable fix for the misleading storage expectation without changing host-wide journal policy; automatic vacuuming or service output changes need separate product/ops review.
