## Summary

Keep open. Current main rotates OpenClaw-managed file logs, but Linux systemd service output is still surfaced through journald and the generated unit plus user-facing docs do not bound journal storage or explain retention knobs.

## What This Changes

The issue asks OpenClaw to address Linux systemd/journald log growth because file-log rotation does not constrain journal storage.

## Best Possible Solution

Document journald retention as a separate Linux/systemd log-storage surface, including `SystemMaxUse=`/`RuntimeMaxUse=` and manual `journalctl --vacuum-*` options, while leaving automatic vacuuming or output rerouting for maintainer ops review.

## Reproduction Assessment

Yes for the actionable gap: current main can be inspected to show Linux service logs are surfaced through `journalctl` while file-log rotation and docs do not cover journald retention. A full disk-growth reproduction would require a sustained Linux systemd user-service run and `journalctl --disk-usage`.

## Solution Assessment

Yes for a docs-first repair; it is the narrowest maintainable fix because OpenClaw should not silently change host-wide journal policy or service output routing without explicit maintainer review.
