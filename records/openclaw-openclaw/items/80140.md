Full review comments:

- **[P1] Send WATCHDOG=1 instead of --watchdog:** `src/infra/systemd-notify.ts:79`
  - body: `systemd-notify` on Ubuntu systemd 255 does not support `--watchdog`; it exits nonzero, and this helper disables all later notifications after the first failure. A regenerated unit with `WatchdogSec` would then receive no keep-alive pings and can be killed/restarted even though the gateway is healthy. Send `WATCHDOG=1` through the supported variable form and update the test to assert that contract.
  - confidence: 0.94
- **[P1] Do not enable default watchdog before startup can ping:** `src/daemon/systemd-unit.ts:80`
  - body: Adding `WatchdogSec=90s` to the default unit while keeping implicit `Type=simple` starts the watchdog deadline before OpenClaw reaches post-ready maintenance and sends its first ping. Slow plugin-heavy starts can be killed and restarted in a loop; make this opt-in or move notifications to a startup-safe notify path with upgrade tests.
  - confidence: 0.87
