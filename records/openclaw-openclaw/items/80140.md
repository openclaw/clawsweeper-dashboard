  - body: The new watchdog path calls `systemd-notify --watchdog`, but the local systemd-notify contract exposes watchdog pings as the `WATCHDOG=1` variable rather than a `--watchdog` option.   - confidence: 0.92
- **[P1] Do not enable default watchdog before startup can ping:** `src/daemon/systemd-unit.ts:80`
  - body: Adding `WatchdogSec=90s` to every generated unit while keeping the service on the existing simple startup path changes supervision for regenerated installs. If gateway startup or notification delivery takes too long, systemd can kill an otherwise valid service before the first heartbeat, so this needs an opt-in/default contract or live upgrade proof before merge.
  - confidence: 0.82
