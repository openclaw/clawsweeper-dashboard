## Summary

Keep open. The PR addresses a real current-main gap in restart continuation visibility, but the submitted timeout fix still misses coalesced pending restarts and the PR still needs a shared timeout constant plus the required user-facing changelog entry.

## What This Changes

The PR forces restart-sentinel continuation dispatches to request automatic source reply delivery, adds a 30s deferral timeout to explicit restart callers, and extends restart continuation and deferral regression tests.

## Best Possible Solution

Revise the PR so restart continuations explicitly request automatic source delivery, explicit restart timeout policy is exported from the restart module, pending restart state stores and updates the effective per-request timeout during coalescing, focused tests cover that coalesced timeout path, and Unreleased Fixes records the user-facing change.

## Reproduction Assessment

Yes. Current main can be checked by dispatching a restart continuation for a Telegram-style group/topic session and observing no requested automatic source delivery, then by scheduling a no-timeout pending restart with `gateway.reload.deferralTimeoutMs: 0` and coalescing a later explicit restart that should impose a finite timeout.

## Solution Assessment

No as submitted. The automatic source-delivery override is the right narrow direction, but the deferral timeout must survive coalescing and the duplicated 30s constants should be centralized before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Propagate deferral timeouts through coalesced restarts:** `src/infra/restart.ts:739-746`
  - body: The new timeout is read from `opts` captured by the first scheduled restart. If a later explicit restart with `deferralTimeoutMs` coalesces onto an already pending restart that had no per-request timeout, the pending callback still sees the old undefined value and can wait indefinitely when runtime config disables the timeout. Store the effective timeout in pending restart state and update it on coalescing.
  - confidence: 0.9

  - body: The PR defines the same 30s explicit restart timeout separately in the gateway tool and chat restart command paths. Export one named constant from the restart module and import it at both call sites so the two explicit restart surfaces cannot drift.
