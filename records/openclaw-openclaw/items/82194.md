## Summary

Keep open: the implementation is close to the right Telegram-local boundary, but it still misses coalesced direct-message turns and the external PR only supplies local test/build output instead of real Telegram proof for a reply-context behavior.

## What This Changes

The PR adds Telegram cache session markers and uses them to skip same-session direct-chat context while preserving reply targets and group/supergroup context.

## Best Possible Solution

Keep the session-key cache approach, but mark every cached Telegram message represented by a successful direct session turn, preserve reply targets and group context, and prove it with focused regression tests plus redacted live Telegram evidence.

## Reproduction Assessment

Yes, at source level: current main records incoming Telegram messages, selects recent cached messages into `Conversation context`, and sends that as current-turn context alongside the session transcript. I did not run a live Telegram setup, so this is source-reproducible rather than fully reproduced.

## Solution Assessment

No, not yet. Session-bound cache markers are a maintainable direction, but the patch should mark all source message IDs in a coalesced turn and still needs real Telegram proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Mark all coalesced source messages:** `extensions/telegram/src/bot-message.ts:185-194`
  - body: The new marker is derived only from `ctxPayload.MessageSid`, but debounced messages and text fragments are combined into one session turn with `messageIdOverride` set to the last Telegram message id. Earlier cached message ids from that same transcript turn remain without `sessionKeys`, so the next direct prompt can still include them as ambient context and duplicate text already in the session. Track and mark every source message id represented by the turn, not only the final `MessageSid`.
  - confidence: 0.78
