## Summary

Keep open: the PR addresses a valid Telegram direct-chat context duplication bug and adds useful focused tests, but the current patch still misses combined/buffered direct turns and the supplied proof is synthetic harness output rather than real Telegram behavior proof.

## What This Changes

The PR adds Telegram message-cache session markers, uses the active session key to dedupe direct-chat ambient context while preserving reply targets, and adds focused Telegram cache/bot regression tests.

## Best Possible Solution

Mark every cached Telegram source message represented by a successfully dispatched direct turn with the active session key, preserve reply targets and group/supergroup ambient context, add buffered-turn regression coverage, and attach real Telegram proof before merge.

## Reproduction Assessment

Yes, at source level: current main builds recent Telegram cached messages into untrusted current-turn context while the active session transcript also carries the dispatched turn. I did not run a live Telegram bot in this read-only review.

## Solution Assessment

No, not as currently written: session-key dedupe is the right direction, but this patch marks only one MessageSid per dispatched turn. Combined debounce/text-fragment turns need all represented cached message ids marked, with regression coverage.
