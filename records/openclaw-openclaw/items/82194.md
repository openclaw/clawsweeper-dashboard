## Summary

Keep open: the Telegram session-key dedupe direction is sound, but the latest PR head still marks only one message id for coalesced turns, so debounced or text-fragment direct chats can still re-add earlier same-session messages as ambient context; the external real-behavior proof is also only local test/build output.

## What This Changes

The branch adds Telegram message-cache session keys, resolves direct-chat prompt context after the session key is known, marks a dispatched Telegram message as session-bound, and adds cache/runtime regression tests for direct-chat context dedupe.

## Best Possible Solution

Keep the Telegram-local session-key cache approach, but mark every source message id represented by a successful direct session turn and require redacted live or equivalent Telegram proof before merge.

## Reproduction Assessment

Yes, at source level: current main records inbound Telegram messages, selects recent cached messages into `Conversation context`, and passes that current-turn context alongside the session transcript. I did not run live Telegram, so this is source-reproducible rather than fully reproduced.

## Solution Assessment

No, not yet. Session-bound cache markers are a maintainable fix direction, but the patch should mark all source message ids in coalesced turns and supply real Telegram proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Mark every coalesced source message:** `extensions/telegram/src/bot-message.ts:185-194`
  - body: The new marker is derived only from `ctxPayload.MessageSid`, but debounced messages and text fragments are combined into one session turn with `messageIdOverride` set to the last Telegram message id.
