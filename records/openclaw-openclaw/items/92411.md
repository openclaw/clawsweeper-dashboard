## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Validate availability group arrays before auditing plugin tools:** `src/tools/availability.ts:76-80`
  - body: `isToolAvailabilityExpression` only checks for a `kind`, `allOf`, or `anyOf` key. After this PR, a JS plugin can return `availability: { allOf: "not-array" }`, the descriptor capture will preserve it, and the new registration audit will reach `expression.allOf.flatMap(...)` during `resolvePluginTools`, throwing instead of warning. Please validate that group fields are arrays, preferably recursively, and cover this through descriptor capture or `resolvePluginTools`.
  - confidence: 0.92
- **[P2] Document or confine the new plugin availability API:** `src/plugin-sdk/index.ts:58`
  - body: This export makes `ToolAvailabilityExpression`/`ToolAvailabilitySignal` part of the public plugin SDK while `AnyAgentTool` also grows an `availability` field. That lets third-party plugins depend on an internal planner shape without matching docs or a contract test. If this is the intended API, document and lock it; otherwise keep the audit path internal to captured descriptors.
  - confidence: 0.78
