## Best Possible Solution

Surface `unsupported-signal` diagnostics once at the descriptor-registration or planner-integration boundary that actually owns availability metadata, with a regression through `resolvePluginTools` or the gateway catalog path.

## Reproduction Assessment

Yes at source level: current main returns `unsupported-signal` diagnostics in `ToolPlan.hidden` for empty groups, and the proposed production audit still cannot see those groups because captured plugin descriptors omit `availability`. I did not run tests because this was a read-only review.

## Solution Assessment

No; this is not the best fix yet because it audits descriptors after the availability data has been dropped. The safer fix is to report from the boundary that owns the malformed descriptor data and prove that real path.
