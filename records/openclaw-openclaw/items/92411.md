## Summary

Keep this PR open, but not merge-ready: the lint issue from the prior review is fixed, yet the production audit still consumes cached plugin descriptors that no longer carry `availability`, so it can add an exported planner callback without surfacing the gateway-startup warning requested by the linked issue.

## What This Changes

The PR adds an optional `onHiddenDiagnostic` callback to the tool planner, a plugin descriptor audit during `resolvePluginTools`, and regression tests for hidden availability diagnostics.

## Best Possible Solution

Surface `unsupported-signal` diagnostics once from the descriptor-registration or planner-integration boundary that still retains availability metadata, keep the evaluator pure, and only merge the callback if maintainers want that exported contract.

## Reproduction Assessment

Yes, source-reproducible: current main returns `unsupported-signal` for empty availability groups and only stores it in `ToolPlan.hidden`. The PR's production audit can also be checked from source because it consumes captured plugin descriptors that omit `availability`; I did not run tests in this read-only review.

## Solution Assessment

No; the planner callback works for synthetic descriptors, but the production plugin path is not fed descriptors that still contain malformed availability. The safer fix is to report from the availability-bearing registration boundary or make the callback a deliberate maintainer-approved contract.
