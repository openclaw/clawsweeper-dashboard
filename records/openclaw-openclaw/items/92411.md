## Summary

This PR does not yet fix the reported registration-time observability gap: it adds an exported callback but no production registration or startup caller, and the callback’s current semantics also classify expected runtime unavailability as authoring errors. Likely related people: Vincent Koc introduced the current planner and availability contracts in commit `60e818f563d809d1b0e20769fb4fdb21ab8526e3` (high confidence).

## What This Changes

Adds an optional `buildToolPlan` callback that receives every diagnostic for hidden tools, with focused tests for malformed availability groups and unsupported expressions.

## Best Possible Solution

Validate and surface malformed availability descriptors at the canonical registration/startup owner through a typed authoring-error channel, while retaining expected runtime unavailability only in `ToolPlan.hidden`.

## Reproduction Assessment

Yes, at source level: empty `allOf` or `anyOf` groups produce `unsupported-signal`, the planner hides the descriptor, and repository search shows no production planner caller that surfaces the diagnostic.

## Solution Assessment

No. An unconsumed optional callback leaves the reported startup behavior unchanged, while exposing every runtime-unavailability diagnostic is broader than the malformed-descriptor problem.
