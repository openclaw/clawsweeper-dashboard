## Summary

Current main still leaves the reported Slack DM gap open: Slack accepts and documents `dmHistoryLimit`, but the Slack monitor does not consume a DM-specific limit and still gates prompt history on the room/group path that excludes 1:1 `im` conversations. The closed related #67020 overlaps, while #7359 is broader Slack cross-session visibility work and should not replace this narrower DM history hydration bug.

## What This Changes

The issue requests that Slack 1:1 DM sessions honor `channels.slack.dmHistoryLimit` by hydrating recent DM conversation history when a fresh DM session starts.

## Best Possible Solution

Implement this inside `extensions/slack`: resolve an effective DM history limit from `channels.slack.dms.<user>.historyLimit` before `channels.slack.dmHistoryLimit`, fetch recent `conversations.history` only for fresh top-level Slack `im` sessions, exclude the current inbound event, format prior user and bot messages through the existing Slack inbound envelope/history contract, and populate both prompt body context and `InboundHistory`. Keep #7359's broader cross-session channel visibility out of this fix.

## Reproduction Assessment

Yes. There is a high-confidence source-level reproduction path: configure Slack `dmHistoryLimit > 0`, start a fresh 1:1 DM session, and current main will not fetch or inject Slack DM history because the Slack monitor has no `dmHistoryLimit` consumer and `channel_type: "im"` fails the `isRoomish` history gates.

## Solution Assessment

Yes. A focused Slack plugin fix is the narrowest maintainable direction; the existing generic transcript trimming helper does not hydrate Slack platform history for new DM sessions, and a broad session-visibility change would solve the wrong problem.
