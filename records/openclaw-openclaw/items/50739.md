## Summary

Keep open. Current `main` still routes `openclaw system event --mode now` through the queued wake and heartbeat path, and the busy-session-lane guard still deliberately defers delivery with `requests-in-flight`; the KeepMyClaw snapshot and quota-prealert comments are useful mitigation context, but neither provides immediate in-session/TUI alert delivery.

## What This Changes

The issue requests a high-priority or bypass-queue mode for `openclaw system event --mode now` so urgent in-session alerts can appear immediately even when the session lane is congested.

## Best Possible Solution

Define a first-class urgent system alert contract that gives operators immediate UI/TUI visibility during degraded provider incidents, while handling any transcript or prompt mutation through a separately ordered and trust-reviewed path shared by the CLI, Gateway wake protocol, heartbeat/cron delivery, and tests.

## Reproduction Assessment

Yes. The source-level reproduction path is high-confidence: `system event --mode now` sends a `wake`, cron enqueues the event and requests heartbeat, and heartbeat returns `requests-in-flight` when the resolved session lane is busy; the busy-lane regression test covers that behavior.

## Solution Assessment

Unclear/no for the raw `--bypass-queue` shape. Immediate alert delivery is valid, but bypassing directly into trusted prompt-visible system events is not obviously the safest maintainable solution; a separate operator-alert surface plus ordered transcript handling is safer.

## Risks / Open Questions

- A prompt-visible bypass could race an active streaming turn or reorder transcript state; current heartbeat code explicitly skips busy session lanes to avoid that.
- Wake params currently allow unknown metadata, so callers can send a `priority` field that validates but is silently ignored by the gateway handler.
- Because queued system events become trusted `System:` prompt lines by default, urgent delivery needs spoofing, trust, and prompt-injection review before it mutates the transcript mid-turn.
