## Summary

Keep open. Current main and v2026.5.20 still enqueue `system event --mode now` through the normal wake plus heartbeat path, and busy active reply/session lanes still skip with `requests-in-flight`; related concurrency and audience-work items do not provide the requested immediate operator-visible alert contract.

## What This Changes

Requests a high-priority or bypass-queue mode for `openclaw system event --mode now` so urgent system alerts surface immediately during session-lane congestion.

## Best Possible Solution

Define an explicit urgent system-alert contract that delivers immediate operator-visible notifications while keeping transcript or prompt mutation on a separately ordered, trust-reviewed path.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: the CLI and wake RPC only send normal wake fields, `wake()` enqueues a system event, and heartbeat returns `requests-in-flight` while the target reply run or session lane is busy; no live provider-outage incident was run in this read-only pass.

## Solution Assessment

Unclear. The proposed bypass flag addresses the alert-delay symptom, but the safer maintainable path is likely an immediate operator-visible alert surface plus separately ordered transcript insertion.
