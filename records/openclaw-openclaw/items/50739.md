## Summary

Keep open. Current main and the latest release still route `openclaw system event --mode now` through queued wake/heartbeat delivery and still skip busy session lanes; the discussion adds useful recovery and pre-failure alert context, but the remaining fix needs maintainer product/safety direction rather than stale cleanup.

## What This Changes

Request a high-priority or bypass mode for `openclaw system event --mode now` so urgent in-session alerts do not wait behind congested session-lane work.

## Best Possible Solution

Define a core urgent-alert contract that makes incident notifications immediately operator-visible, then add prompt-visible transcript injection only if maintainers approve ordered and trust-reviewed semantics.
