## Summary

Keep open. Current main and the latest release still implement `openclaw system event --mode now` as queued system-event plus heartbeat wake delivery, with no typed priority or bypass contract; the KeepMyClaw and quota pre-warning comments are useful adjacent mitigation context but do not provide immediate in-session alert delivery.

## What This Changes

The issue requests a high-priority or bypass-queue mode for system events so urgent in-session alerts can surface despite a congested session lane.

## Best Possible Solution

Define a typed urgent-alert contract that makes incident notifications immediately operator-visible, and only add prompt-visible transcript injection through a separately ordered, trust-reviewed session path.

## Reproduction Assessment

Yes, at source level: current main has no priority or bypass CLI/protocol field, and mode now still enqueues before heartbeat delivery. I did not run a live provider-outage reproduction in this read-only review.

## Solution Assessment

No, not as a standalone flag yet. The maintainable solution needs a maintainer decision between immediate operator-visible notification, prompt-visible transcript injection, or both.
