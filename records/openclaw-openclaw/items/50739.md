## Summary

Keep open. Current main still has no priority or bypass mode for `openclaw system event --mode now`: the CLI sends only wake text/mode/session target, wake enqueues a system event, and heartbeat delivery still skips when the main or target session lane is busy. The KeepMyClaw snapshot and pre-failure quota comments are useful recovery/early-warning context, but they do not satisfy the requested immediate in-session alert path.

## What This Changes

The issue requests a high-priority or bypass-queue mode for `openclaw system event --mode now` so urgent in-session alerts are not delayed behind saturated session lanes.

## Best Possible Solution

Define an explicit urgent system-alert contract that delivers immediate operator-visible notifications, with separate ordering and trust rules for any transcript or prompt mutation.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: wake enqueues a system event and requests heartbeat, while heartbeat skips delivery when the main or target session lane is busy. I did not run a live provider-outage/TUI reproduction in this read-only pass.

## Solution Assessment

Unclear. The proposed bypass flag addresses the alert-delay symptom, but the safer maintainable path is likely an immediate operator-visible alert surface plus a separately ordered transcript insertion path.
