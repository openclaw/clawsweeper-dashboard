## Summary

Keep open. Current main still routes `openclaw system event --mode now` through queued system-event plus heartbeat wake delivery, and the heartbeat path deliberately skips busy main or session lanes, so the requested immediate reliable in-session alert behavior remains unimplemented.

## What This Changes

The issue requests a high-priority or bypass-queue mode for `openclaw system event --mode now` so urgent system alerts can appear immediately during congested LLM/session lanes.

## Best Possible Solution

Define a first-class urgent system alert contract that gives operators immediate TUI/UI visibility during degraded provider incidents, while handling prompt or transcript mutation through a separately ordered and trust-reviewed path across the CLI, Gateway wake protocol, heartbeat delivery, docs, and regression tests.

## Reproduction Assessment

Yes. The source-level path is high-confidence: `system event --mode now` sends `wake`, cron enqueues a system event and requests heartbeat, and heartbeat returns `requests-in-flight` when the main or resolved session lane is busy; the existing tests cover those pieces.

## Solution Assessment

No/unclear for the raw bypass flag as proposed. Immediate alert delivery is valid, but directly bypassing into trusted prompt-visible system events is not the safest maintainable shape without a separate operator-alert surface and explicit transcript-ordering semantics.

## Risks / Open Questions

- A prompt-visible bypass could race an active streaming turn or reorder transcript-visible state; the current heartbeat guard explicitly avoids busy session lanes for that reason.
- System events are trusted prompt-visible `System:` lines by default, so an urgent bypass needs clear spoofing, ownership, and prompt-injection semantics before it mutates an active prompt or transcript.
- Wake params currently accept unknown metadata while the Gateway handler ignores it, so a caller can send `priority` today and get validation success without any priority behavior.
- The product shape is unresolved: the immediate operator alert surface may need to be separate from prompt-visible transcript injection.
