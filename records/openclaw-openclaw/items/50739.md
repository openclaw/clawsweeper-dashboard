## Summary

Keep open. Current main still routes `openclaw system event --mode now` through queued system-event plus heartbeat wake delivery, and the existing busy-lane guard deliberately defers delivery instead of bypassing an active session lane.

## What This Changes

Add a high-priority or bypass-queue mode to `openclaw system event` so urgent alerts can appear during congested sessions.

## Best Possible Solution

Define a high-priority system alert contract that delivers immediate operator-visible notifications, while keeping transcript or prompt mutation separately ordered and trust-reviewed.

## Reproduction Assessment

Yes, from source and focused tests: enqueue a system event, make the target session lane busy, and heartbeat delivery returns `requests-in-flight` instead of delivering immediately. I did not run a live gateway/TUI repro in this read-only sweep.

## Solution Assessment

Unclear. A direct prompt-stream bypass is not obviously the safest fix; the maintainable path likely needs an explicit immediate alert surface plus separate ordering rules for any transcript-visible `System:` line.
