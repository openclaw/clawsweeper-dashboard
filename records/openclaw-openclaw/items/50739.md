## Summary

Keep open. Current main and the latest release still route `openclaw system event --mode now` through queued system-event plus heartbeat delivery, with no priority or bypass contract; the remaining work needs maintainer direction because direct prompt-visible injection can disturb active session ordering.

## What This Changes

Request a high-priority or bypass-queue mode for `openclaw system event --mode now` so urgent alerts can reach an active session during provider-lane congestion.

## Best Possible Solution

Define a core urgent-alert contract that makes incident notifications immediately operator-visible, and only add prompt-visible transcript injection through a separately ordered, trust-reviewed session path.

## Reproduction Assessment

Yes, at source level: current main has no priority/bypass CLI or wake contract, `mode: now` enqueues then wakes heartbeat, and heartbeat skips busy session lanes. I did not run a live provider-outage reproduction in this read-only review.

## Solution Assessment

No, not as a standalone flag yet. The maintainable path is to decide whether OpenClaw needs immediate operator-visible notification, prompt-visible transcript injection, or both before adding a typed CLI/protocol surface.
