## Summary

Keep open. Current main and v2026.5.20 still route `system event --mode now` through the queued system-event plus heartbeat path, and the requested immediate operator-visible alert contract is not implemented; the KeepMyClaw and quota-threshold comments add useful operational context but do not replace this core delivery decision.

## What This Changes

The issue requests a high-priority or queue-bypassing `openclaw system event` mode so urgent in-session alerts appear during congested provider-failure lanes.

## Best Possible Solution

Define an explicit urgent system-alert contract that delivers immediate operator-visible notifications while keeping any transcript or prompt mutation on a separately ordered, trust-reviewed path.

Yes. ## Solution Assessment

Unclear. The proposed bypass flag addresses the alert-delay symptom, but the safer maintainable path is likely an immediate operator-visible alert surface plus separately ordered transcript insertion.

- Adjacent scheduler and system-event issues indicate this should be coordinated with broader non-preemption and system-event audience decisions rather than treated as a narrow flag-only repair.
