## Summary

Keep open. Current main and v2026.6.6 still have no priority or bypass contract for `openclaw system event --mode now`; the event is queued and then delivered through heartbeat, which intentionally skips busy lanes to protect active session ordering.

## What This Changes

The issue requests a high-priority or bypass-queue mode for `openclaw system event --mode now` so urgent alerts can reach an active session during provider-lane congestion.

## Best Possible Solution

Define a core urgent-alert contract that makes incident notifications immediately operator-visible, then add a separately ordered and trust-reviewed path only if prompt-visible transcript injection is required.

## Reproduction Assessment

Yes, at source level: current main has no priority/bypass CLI or wake contract, `mode now` enqueues then wakes heartbeat, and heartbeat skips busy main or session lanes. I did not run a live provider-outage saturated-lane reproduction in this read-only pass.

## Solution Assessment

No, not as a narrow flag alone. The maintainable fix needs a maintainer decision about immediate operator-visible notification versus prompt-visible transcript injection before implementation starts.
