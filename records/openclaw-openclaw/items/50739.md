## Summary

Keep open. Current main still lacks a priority or bypass mode for `openclaw system event --mode now`; it enqueues prompt-visible system events and relies on heartbeat delivery, while busy main or session lanes still defer the wake. The KeepMyClaw and quota-trigger comments are useful operational context, but they do not satisfy the requested immediate in-session alert path.

## What This Changes

The issue requests a high-priority or bypass-queue mode for `openclaw system event --mode now` so urgent in-session alerts are not delayed behind congested LLM session lanes.

## Best Possible Solution

Define a first-class urgent operator alert contract that gives immediate TUI/UI visibility during degraded-provider incidents, while routing transcript or prompt mutation through separately ordered, trust-reviewed semantics.

## Reproduction Assessment

Yes. The source-level reproduction path is high-confidence: CLI sends only `{ mode, text }`, the gateway wake handler passes only those fields, cron wake enqueues a system event plus heartbeat wake, and heartbeat returns `requests-in-flight` while main or resolved session lanes are busy.

## Solution Assessment

No/unclear for the raw bypass flag as proposed. Immediate alert delivery is valid, but directly bypassing into trusted prompt-visible system events is not the narrowest safe shape without explicit ordering, spoofing, and active-turn semantics.
