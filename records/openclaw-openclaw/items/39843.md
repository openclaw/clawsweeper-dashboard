## Summary

Keep open: current main and the latest release still leave Google Chat's real typing placeholder tied to normal reply delivery, while exact `NO_REPLY` can finish without a delivery call that consumes that placeholder. The linked fork PR is an open candidate fix, not an implemented fix.

## What This Changes

The issue asks Google Chat to clear or consume its visible typing placeholder when an agent reacts and then finishes with `NO_REPLY` instead of sending text.

## Best Possible Solution

Review, land, or replace the narrow Google Chat placeholder cleanup so silent no-delivery turns delete the placeholder without requiring a dummy visible message.

## Reproduction Assessment

Yes by source inspection: Google Chat creates a placeholder before dispatch, exact `NO_REPLY` can skip queued delivery, and the Google Chat call site does not wire a cleanup callback that deletes the placeholder. I did not run a live Google Chat workspace reproduction in this read-only review.

## Solution Assessment

Yes, the best fix is a Google Chat-owned cleanup hook around the existing placeholder lifecycle; sending a dummy user-visible message would be the wrong layer and worse UX.
