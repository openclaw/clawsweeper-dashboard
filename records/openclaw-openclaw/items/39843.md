## Summary

Keep open. Current main still has the Google Chat silent-turn typing cleanup gap, and open PR #39891 references this issue with closing syntax, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Google Chat should clear its message-backed typing indicator when a reaction-only turn ends with `NO_REPLY`.

## Best Possible Solution

Land or replace #39891 with a Google Chat-owned cleanup that deletes any remaining `typingMessageName` after silent dispatch completion and adds monitor-level regression coverage.

## Reproduction Assessment

Yes. Current main has a deterministic source path: Google Chat creates a typing message, exact `NO_REPLY` can skip dispatcher delivery, and no Google Chat cleanup hook clears the stored message.

## Solution Assessment

Yes. The narrow maintainable fix is in the Google Chat monitor lifecycle, either by wiring the existing typing cleanup hook or by deleting any still-held typing message after silent dispatch completion.
