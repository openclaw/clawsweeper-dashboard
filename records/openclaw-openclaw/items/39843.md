## Summary

Keep open: current main and v2026.6.10 still leave Google Chat's real typing placeholder tied to normal reply delivery, while a silent NO_REPLY turn can finish without invoking that delivery callback. An open fork PR is a candidate fix, not an implemented fix.

## What This Changes

The issue reports that Google Chat leaves the visible typing placeholder behind when an agent reacts with an emoji and returns NO_REPLY.

## Best Possible Solution

Review, land, or replace the open candidate fix so Google Chat deletes or consumes the typing placeholder whenever an inbound turn completes without normal reply delivery.

## Reproduction Assessment

Yes, source-level: Google Chat creates `typingMessageName`, exact silent replies can finish without invoking delivery, and the call site does not provide an `onTypingCleanup` callback that deletes the placeholder. I did not run a live Google Chat workspace reproduction.

## Solution Assessment

Yes, the narrow maintainable fix is Google Chat-owned cleanup through the existing typing lifecycle, not sending a dummy visible message. The open fork PR appears to target that exact boundary but still needs maintainer review and proof before this issue can close.
