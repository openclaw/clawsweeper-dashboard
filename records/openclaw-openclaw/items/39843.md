## Summary

Keep open. Current main still has the Google Chat silent-turn typing cleanup gap, and open PR #39891 references this issue with closing syntax, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that Google Chat's message-based typing indicator remains after an emoji reaction plus an exact `NO_REPLY` silent turn.

## Best Possible Solution

Land a Google Chat-owned cleanup that deletes any remaining `typingMessageName` after silent dispatch completion and adds monitor-level regression coverage for reaction-only exact `NO_REPLY`.

## Reproduction Assessment

Yes. Current main still has a deterministic source path: Google Chat creates a typing message, exact `NO_REPLY` can skip dispatcher delivery, and no Google Chat cleanup hook clears the stored message.

## Solution Assessment

Yes. The narrow maintainable solution is to fix the Google Chat monitor lifecycle, either by wiring the existing typing cleanup hook or by deleting any still-held typing message after silent dispatch completion.
