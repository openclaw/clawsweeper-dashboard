Keep open. Current main still has the Google Chat silent-turn cleanup gap, and an open PR with closing syntax is already the implementation candidate for this issue.

## What This Changes

Google Chat should clear its message-backed typing placeholder when an agent reacts with emoji and then returns exact `NO_REPLY`.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/39891 with Google Chat-owned post-turn cleanup for any remaining `typingMessageName`, plus focused regression coverage and redacted real Google Chat proof.

## Reproduction Assessment

Yes, source-level. Current main creates and stores a Google Chat typing message, exact `NO_REPLY` can skip delivery, and the monitor has no fallback cleanup after the turn when delivery is skipped.

## Solution Assessment

Yes. The narrow maintainable fix is to delete the remaining Google Chat typing message in the plugin lifecycle after silent completion, not to send a dummy reply or add core Google Chat policy.
