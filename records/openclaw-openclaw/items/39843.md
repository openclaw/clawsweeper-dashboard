## What This Changes

Google Chat should clear its message-backed typing placeholder when an agent reacts with an emoji and then returns `NO_REPLY`.

## Best Possible Solution

## Reproduction Assessment

Yes, source-level. Current main creates and stores a Google Chat typing message, exact `NO_REPLY` can return before delivery, and the monitor has no post-turn fallback cleanup if `typingMessageName` remains set.

## Solution Assessment

Yes. The narrow maintainable solution is Google Chat-owned cleanup after the active `core.channel.turn.run` lifecycle, not a dummy reply or core Google Chat policy change.
