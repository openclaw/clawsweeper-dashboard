## Summary

Keep open. Current main still has the Google Chat silent-turn cleanup gap, and an open PR with closing syntax is already the implementation candidate for this issue.

## What This Changes

The issue reports that Google Chat leaves its message-backed typing indicator visible when an agent reacts with an emoji and then returns `NO_REPLY`.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/39891 with Google Chat-owned cleanup for any remaining `typingMessageName` after a silent turn, plus focused regression coverage on the current turn-kernel path.

## Reproduction Assessment

Yes, source-level. Current main creates and stores a Google Chat typing message, exact `NO_REPLY` can skip delivery, and the monitor has no cleanup after the turn when delivery is skipped.

## Solution Assessment

Yes. The narrow maintainable solution is to delete the remaining Google Chat typing message in the plugin lifecycle after silent completion, not to send a dummy reply or add core Google Chat policy.
