## Summary

Keep open: current main and v2026.5.22 still create a Google Chat message-backed typing placeholder, skip delivery for exact NO_REPLY, and have no post-turn cleanup for the still-owned placeholder. The target-repo fix PRs are closed; the remaining repair is narrow, with an open fork PR available as prior art but not merged into OpenClaw main.

## What This Changes

Google Chat should clear its message-backed typing placeholder when an agent reacts and then returns NO_REPLY without a text reply.

## Best Possible Solution

Add a Google Chat-owned post-turn cleanup that deletes any still-owned typingMessageName after silent turns, while preserving normal delivery edits and avoiding dummy replies or new config.

## Reproduction Assessment

Yes, at source level: current main sends and stores typingMessageName before core.channel.turn.run, exact NO_REPLY returns before delivery, and the Google Chat monitor exits without deleting a still-owned placeholder. I did not run a live Google Chat Workspace scenario in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is Google Chat lifecycle cleanup after the current turn run; sending dummy replies or adding a config option would be a noisier workaround.
