## Summary

Keep open. Current main still lacks Google Chat cleanup for silent reaction turns, and open PR #39891 is already the implementation candidate referenced with closing syntax.

## What This Changes

Google Chat should clear its message-backed typing indicator when a reaction-only turn ends with exact `NO_REPLY`.

## Best Possible Solution

Land or replace #39891 with a Google Chat-owned cleanup that deletes or wires cleanup for any remaining `typingMessageName` after silent dispatch, with regression coverage around monitor lifecycle.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Google Chat creates and stores a typing message, exact `NO_REPLY` returns before delivery, and this channel does not wire `onTypingCleanup`.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Google Chat monitor lifecycle by clearing the stored typing message on silent completion rather than sending a placeholder reply.
