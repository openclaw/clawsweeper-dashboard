## Summary

Keep open. Current main still has the Google Chat silent-turn cleanup gap, and https://github.com/openclaw/openclaw/pull/39891 is an open implementation candidate that references this issue with closing syntax.

## What This Changes

Ensure Google Chat clears its message-backed typing indicator when an agent reacts with an emoji and returns `NO_REPLY` without sending text.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/39891 with Google Chat-owned cleanup for any remaining `typingMessageName` after silent dispatch, plus regression coverage around the current `turn.run` path.

## Reproduction Assessment

Yes, source-level. Current main starts a Google Chat message-mode typing placeholder, exact `NO_REPLY` can return before delivery, and the monitor has no cleanup after the turn when delivery is skipped.

## Solution Assessment

Yes. The narrow maintainable solution is to clear the stored Google Chat typing message on silent completion in the plugin lifecycle, not to send a placeholder reply or add core policy.
