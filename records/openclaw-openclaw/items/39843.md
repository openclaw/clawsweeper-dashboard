## Summary

Keep open. Current main still has the Google Chat silent-turn cleanup gap, and https://github.com/openclaw/openclaw/pull/39891 is an open implementation candidate that references this issue with closing syntax.

## What This Changes

Google Chat should clear its message-backed typing indicator when a reaction-only turn ends with exact NO_REPLY.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/39891 with Google Chat-owned cleanup after the current turn-kernel path so a remaining typing message is deleted after silent dispatch.

## Reproduction Assessment

Yes, source-level: current main creates a Google Chat typing message, exact NO_REPLY can skip delivery, and the monitor has no post-turn cleanup for a still-set typingMessageName. I did not run a live Google Chat Workspace reproduction in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to delete any remaining Google Chat typing message after silent turn completion while preserving normal delivery-owned update/delete behavior; the issue should stay open until the linked fix lands or is replaced.
