## Summary

Keep open. Current main still has the Google Chat silent-reply typing cleanup gap, and open PR #39891 specifically references this issue with closing syntax, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks Google Chat to clear its typing placeholder when an agent creates an emoji reaction and then returns an exact `NO_REPLY` with no text response.

## Best Possible Solution

Land #39891 after review, or apply an equivalent Google Chat-owned lifecycle fix, so a leftover `typingMessageName` is deleted when dispatch completes without a delivery callback and a monitor-level regression test covers reaction-only exact `NO_REPLY`.

## Reproduction Assessment

Yes. The source-level path is deterministic on current main: Google Chat creates a real typing message, exact `NO_REPLY` is normalized away before delivery, and no Google Chat cleanup callback deletes the leftover message.

## Solution Assessment

Yes. The narrowest maintainable solution is to keep silent-reply semantics in core and fix the Google Chat monitor lifecycle by wiring cleanup or deleting any still-held `typingMessageName` after dispatch completes without delivery.
