## Summary

Current main has moved past the v2026.4.29 failure mode: the generated `plugin-runtime-deps` tree is now treated as legacy cleanup state, bundled plugin runtime code is loaded from a `dist-runtime` overlay that forwards to the canonical root `dist/` graph, and `json5` remains declared as a root runtime dependency. The proof commit is on main only; no release tag contains it yet.

## What This Changes

The issue requests that bundled plugin runtime dependency resolution honor `json5`'s package entrypoint, or generate explicit `json5/lib/index.js` imports, so channels do not fail before replying on v2026.4.29.

## Best Possible Solution

Keep the current unified bundled runtime graph and legacy dependency cleanup behavior, then ship it in the next release so users affected on v2026.4.29 receive the fix without adding a `json5`-specific import rewrite.

## Reproduction Assessment

Yes. The report has a concrete v2026.4.29 cache path and failing module list; current-main verification is source-based, showing that this `plugin-runtime-deps` path is now legacy-only and bundled runtime imports resolve through the root dist graph instead.

## Solution Assessment

Yes. Removing the generated runtime-deps mirror from the active bundled runtime path is broader and more maintainable than special-casing `json5/lib/index.js`, because it fixes the package-entrypoint class of failures for root-owned dist imports.
