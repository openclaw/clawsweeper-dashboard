## Summary

Current main already resolves the reported v2026.4.29 failure mode by no longer treating user-state `plugin-runtime-deps` as the bundled plugin runtime path, loading bundled channel code from the package bundled roots, and guarding root-owned dist imports against missing package dependencies. This is the same root cause as related #75821, which was already reviewed as implemented on main; no release tag in this checkout contains the proof commit yet.

## What This Changes

The issue asks OpenClaw's plugin runtime dependency path to resolve bare `json5` imports through the package entrypoint, or to generate imports for `json5/lib/index.js`, so bundled agents/channels do not fail before replying.

## Best Possible Solution

Keep the current main design: bundled plugins should load from the packaged OpenClaw graph, old user-state runtime-deps roots should remain cleanup-only legacy state, and the root dist dependency guard should prevent future bare-package imports from shipping without matching package dependencies.

## Reproduction Assessment

Yes. The report includes a concrete v2026.4.29 runtime cache path, failing generated files, and the exact `json5@2.2.3` entrypoint mismatch. I did not run the live channel probe in this read-only review, but the failure maps directly to the old staging surface now removed on main.

## Solution Assessment

Yes. The current main approach is the narrow maintainable fix because it removes the brittle staged runtime-deps resolution path and lets `json5` resolve through the normal installed package dependency graph instead of adding a one-off `json5/lib/index.js` workaround.
