## Summary

Keep open. Current `main` still hardcodes short-term promotion writes to `workspaceDir/MEMORY.md`; the merged compaction and contamination fixes reduce growth/noise but do not add a configurable promotion target or automatic search-index contract for alternate files.

## What This Changes

The issue requests a memory-core configuration option to write promoted short-term memories to a configurable Markdown target instead of always using `MEMORY.md`.

## Best Possible Solution

Define a memory-core-owned promotion-target resolver shared by manual and managed promotion, default it to `MEMORY.md`, constrain safe workspace-relative Markdown/date-token targets, and align schema, docs, indexing, and tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction, but yes for current behavior: source inspection confirms both manual and managed promotion paths still write to `workspaceDir/MEMORY.md` with no destination override.

## Solution Assessment

Yes directionally, but not as an automated bug fix. A shared configurable resolver is the narrow maintainable direction, while maintainers still need to decide the public config, path safety, date-token, and indexing contract.
