## Summary

Keep open. Current main still writes short-term promotions to `workspaceDir/MEMORY.md`; the merged compaction and contamination fixes reduce growth/noise but do not add a configurable promotion target or search-index contract for alternate files.

## What This Changes

Add a memory-core config option that lets short-term promotion write to a configurable/date-templated Markdown target while keeping promoted content searchable.

## Best Possible Solution

Define a memory-core-owned promotion-target resolver shared by manual and managed promotion, default it to `MEMORY.md`, constrain safe workspace-relative Markdown/date-token targets, and align schema, docs, indexing, and tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction, but yes for current behavior: source inspection confirms both manual and managed promotion paths still write to `workspaceDir/MEMORY.md` with no destination override.

## Solution Assessment

Yes directionally. A shared configurable resolver is the narrowest maintainable direction, but maintainers need to decide the public config, path safety, date-token, and indexing contract first.
