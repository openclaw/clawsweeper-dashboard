## Summary

Keep open. Current main still routes both manual and managed short-term promotions to `workspaceDir/MEMORY.md`; the merged budget compaction work bounds growth but does not provide a configurable promotion target or search-indexing contract for alternate files.

## What This Changes

Add a memory-core configuration option for where short-term promotions are written instead of always appending them to `MEMORY.md`.

## Best Possible Solution

Define a memory-core-owned promotion-target resolver shared by manual promote and managed dreaming, default it to `MEMORY.md`, constrain it to safe workspace-relative Markdown targets, document any date tokens, and ensure the resolved target remains searchable.

## Reproduction Assessment

Yes. Source inspection on current main shows the manual and managed apply paths both reach `applyShortTermPromotions`, which still resolves and writes `workspaceDir/MEMORY.md` with no destination override.

## Solution Assessment

Yes, directionally. The requested config/resolver is the narrowest maintainable way to separate promotion storage from curated bootstrap memory, but the exact path template and indexing rules need maintainer API judgment.
