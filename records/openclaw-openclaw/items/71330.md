## Summary

Keep open. Current main and the latest release still write deep short-term promotions to `MEMORY.md`; merged compaction and contamination fixes reduce the worst symptoms but do not add the requested configurable promotion target or settle alternate-file search behavior.

## What This Changes

The issue requests a configurable target file for memory-core short-term promotions instead of always writing promoted memories into `MEMORY.md`.

## Best Possible Solution

Maintainers should choose the promotion-destination contract first; if accepted, implement one shared safe resolver with default `MEMORY.md`, approved date tokens, schema/docs alignment, and memory-search coverage for alternate targets.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: current main and `v2026.6.10` still resolve deep promotion writes to `workspaceDir/MEMORY.md`, and the manifest exposes no target-path config.

## Solution Assessment

Unclear. A configurable path would address the report, but the best permanent shape needs maintainer judgment because new plugin config and alternate-file indexing are compatibility-sensitive.
