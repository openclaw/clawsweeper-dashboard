## Summary

Keep open. Current main and v2026.6.10 still write short-term promotions to `MEMORY.md` and expose no promotion target config; merged compaction and contamination fixes mitigate related symptoms but do not satisfy the requested separate target or the lifecycle concerns raised in discussion.

## What This Changes

Request to add memory-core configuration/routing so short-term promotions can be written to a configurable file instead of hardcoded `MEMORY.md`.

## Best Possible Solution

Maintainers should define a constrained promotion-routing contract first, then implement one safe resolver with default `MEMORY.md`, approved target lanes or routing rules, schema/docs alignment, indexing coverage, and coordination with lifecycle curation in https://github.com/openclaw/openclaw/issues/87660.
