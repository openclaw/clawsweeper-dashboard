## Summary

Keep open. Current main still hardcodes short-term promotion writes to `workspaceDir/MEMORY.md`; the merged compaction and contamination fixes reduce growth/leakage, but they do not add a configurable target file or an indexing contract for alternate promotion files.

## What This Changes

The issue requests a configurable memory-core promotion target path so promoted short-term memories can be written somewhere other than `MEMORY.md`.

## Best Possible Solution

Define a memory-core-owned promotion-target resolver shared by manual promote and managed dreaming, default it to `MEMORY.md`, constrain it to safe workspace-relative Markdown targets, document supported date tokens, and ensure the resolved file is searchable.

## Reproduction Assessment

Yes. Source inspection on current main shows both manual and managed promotion paths reach `applyShortTermPromotions`, which still resolves and writes `workspaceDir/MEMORY.md` with no destination override.

## Solution Assessment

Yes, directionally. A configurable resolver is the narrowest maintainable direction for separating promoted metadata from curated bootstrap memory, but the exact config key, path-template rules, and indexing behavior need maintainer API judgment.
