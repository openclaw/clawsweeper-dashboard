## Summary

Keep open. Current main still hardcodes short-term promotion writes to `MEMORY.md`; the recent compaction and contamination fixes reduce growth/noise but do not add a configurable promotion target or define how alternate files enter memory search.

## What This Changes

Request a memory-core config option for writing promoted short-term memories to a configurable Markdown target, including date-token paths and searchability.

## Best Possible Solution

Add a shared memory-core promotion-target resolver, default it to `MEMORY.md`, validate safe Markdown/date-token targets, and wire schema, docs, indexing, CLI, managed dreaming, and regression tests together.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not as a strict bug reproduction, but yes for current behavior: source inspection confirms manual and managed promotion paths still write to `workspaceDir/MEMORY.md` with no destination override.

## Solution Assessment

Is this the best way to solve the issue? Yes directionally, but the maintainable version is a shared resolver plus schema/docs/indexing contract, not a one-off path string in one caller.
