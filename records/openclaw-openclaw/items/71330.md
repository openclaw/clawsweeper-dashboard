## Summary

Keep open: current main and the latest release still hardcode short-term promotion writes to `MEMORY.md`; the merged compaction and contamination fixes reduce symptoms but do not provide the requested configurable destination.

## What This Changes

Request adding a memory-core promotion-target config with templated paths so short-term promotions can write somewhere other than `MEMORY.md` while remaining searchable.

## Best Possible Solution

Decide the promotion-target contract first, then implement one shared resolver used by managed dreaming and CLI promotion with safe paths, default `MEMORY.md`, search/indexing semantics, docs, schema, and regression coverage.

## Reproduction Assessment

Yes for current behavior, but not as a strict bug reproduction: source inspection confirms manual and managed short-term promotion still write only to `workspaceDir/MEMORY.md` with no destination override.

## Solution Assessment

Unclear. A configurable target is plausible, but the best maintainable fix needs a shared resolver and indexing/search contract rather than adding a one-off path string to one caller.
