## Summary

Keep open. Current main still routes short-term promotion writes to `workspaceDir/MEMORY.md`; the recent compaction and contamination fixes reduce growth/noise but do not add a configurable destination, date-token handling, or search-index contract for alternate promotion files.

## What This Changes

Request to add a memory-core configuration option for the short-term promotion target file, defaulting to `MEMORY.md` while allowing alternate or date-templated Markdown paths.

## Best Possible Solution

Define a memory-core-owned promotion-target resolver shared by manual and managed promotion, default it to `MEMORY.md`, constrain safe Markdown targets, and update schema, docs, indexing, and regression tests together.

## Reproduction Assessment

Not applicable as a strict bug reproduction. The current behavior is source-reproducible: manual and managed promotion both call the same writer, and that writer hardcodes `MEMORY.md`.

## Solution Assessment

Yes directionally. A shared configurable resolver is the narrowest maintainable solution, but maintainers need to settle the user-facing config, path safety, date-token, and indexing contract first.
