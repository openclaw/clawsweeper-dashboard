## Summary

Keep open. Current main still writes short-term promotions to `MEMORY.md`; the merged compaction and contamination fixes reduce growth and noise, but they do not add a configurable promotion target or an indexing contract for alternate files.

## What This Changes

Request adds a memory-core config option to route short-term promotion writes from hardcoded `MEMORY.md` to a configured Markdown target, including date-token paths and searchability.

## Best Possible Solution

Define a memory-core-owned promotion-target resolver shared by manual and managed promotion, default it to `MEMORY.md`, constrain safe workspace-relative Markdown/date-token targets, and align schema, docs, indexing, and tests.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for current behavior by source inspection: `applyShortTermPromotions` still sets `workspaceDir/MEMORY.md`, and both CLI and managed dreaming callers pass no target override.

## Solution Assessment

Is this the best way to solve the issue? Yes directionally, but maintainers need to decide the public config, path safety, date-token, and indexing contract before implementation.
