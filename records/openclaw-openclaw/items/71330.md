## Summary

Keep open. Current main and v2026.5.18 still route short-term promotion writes to `MEMORY.md` with no configurable destination; the merged compaction/contamination fixes and the open archive-first PR reduce related symptoms but do not settle this public config/indexing contract.

## What This Changes

Add a memory-core configuration option for where promoted short-term memories are written, with a backward-compatible `MEMORY.md` default and searchable alternate targets.

## Best Possible Solution

Define a memory-core-owned promotion destination policy shared by manual and managed promotion, default it to current `MEMORY.md` behavior, constrain safe Markdown/date-token targets, and align schema, docs, indexing, events, and tests with the chosen design.

## Reproduction Assessment

Yes for current behavior, but not as a bug reproduction: source, docs, tests, and the latest release tag all show promotion still writes to `MEMORY.md` with no destination override.

## Solution Assessment

Yes directionally, but not as an automated bug fix. A configurable resolver is maintainable only after maintainers choose the path safety, date-template, archive-vs-target, and indexing contract.
