## Summary

Keep open: current main still preserves nonblank provider thinking signatures across the compaction retained-tail/successor path, while the replay sanitizer only strips missing or blank signatures, so the reported Anthropic rejection remains actionable.

## What This Changes

The issue asks OpenClaw to prevent stale Anthropic extended-thinking signatures from being replayed after session compaction.

## Best Possible Solution

Sanitize provider-bound signed thinking on retained pre-compaction assistant turns at the compaction/replay boundary, with successor transcript and sanitize-history regression coverage while preserving valid post-compaction signatures.

## Reproduction Assessment

Yes at source level: current main still replays retained pre-compaction assistant messages after a compaction summary and preserves nonblank thinking signatures through to Anthropic transport. I did not run a live Anthropic reproduction or inspect the reporter's private JSONL specimen.

## Solution Assessment

No current implementation is not the best end state: the existing sanitizer only handles missing or blank signatures and does not know compaction boundaries. The narrow maintainable fix should strip or downgrade signatures only for retained pre-compaction signed-thinking turns and prove both rotated and non-rotated compaction paths.
