## Summary

Keep open: the PR targets a plausible embedded transcript regression, but the patch broadens the CLI transcript mirror to all embedded Pi runs and would duplicate Pi-owned JSONL turns instead of only filling a missing assistant tail; it also lacks after-fix real behavior proof.

## What This Changes

The PR widens `agent-command` transcript persistence and post-turn compaction from CLI-only to CLI-or-embedded, adds a focused unit test, and adds a changelog entry for #77823.

## Best Possible Solution

Revise this into a transcript-aware embedded fix that appends only a demonstrably missing current-turn assistant entry, leaves Pi-owned user/assistant writes intact, and proves the result with a real webchat or Feishu-style embedded run.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Source inspection gives a high-confidence gate path: embedded results carry `runner: "embedded"` while current `agent-command` only runs the CLI mirror for `runner: "cli"`; no live webchat/Feishu reproduction was provided or run.

## Solution Assessment

Is this the best way to solve the issue? No; making every embedded run use `persistCliTurnTranscript` is too broad because Pi already owns normal embedded transcript writes, so a safer fix must check the transcript/current attempt and only append genuinely missing assistant text.
