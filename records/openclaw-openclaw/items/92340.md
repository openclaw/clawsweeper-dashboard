## Summary

This useful Feishu feature has convincing live proof, but it is not merge-ready: subscribed installations can gain autonomous meeting-join turns without an OpenClaw opt-in, and pairing-required inviters are routed through an invalid synthetic chat target. The linked issue should remain open while this PR receives owner direction and a focused routing repair.

## What This Changes

The PR registers Feishu VC meeting-invite events, converts them into synthetic direct-message agent turns requesting an immediate join, adds regression tests, and documents the required event subscription.

## Best Possible Solution

Add an explicit owner-approved, default-off Feishu setting for automatic invite joins unless maintainers formally accept event subscription as sufficient authorization, then represent synthetic DM routing with a canonical direct-recipient contract and test both pairing-required and admitted flows through the real ingress behavior.

## Solution Assessment

No. Reusing canonical Feishu admission and reply behavior is the right boundary, but fabricating a user route as chat_id does not satisfy the whole ingress contract, and unconditional activation leaves a merge-critical authorization decision unresolved.
