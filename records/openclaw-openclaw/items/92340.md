## Summary

Keep open: current main and v2026.6.6 still lack Feishu VC invite handling, but this PR is not merge-ready because the synthetic invite path can still send default pairing challenges to `chat:<inviter id>` instead of a valid user target.

## What This Changes

Adds Feishu VC meeting invite handling in the bundled Feishu plugin, converts valid invites into synthetic direct-message turns, updates focused tests, and documents the required event subscription.

## Best Possible Solution

Keep the implementation in the bundled Feishu plugin, repair the pairing target path with regression coverage, then merge only after maintainers accept event subscription as the opt-in boundary or add a default-off Feishu gate.

## Solution Assessment

No, not yet. Reusing the Feishu DM ingress and reply path is the right owner boundary, but the current head is not the best fix until the pairing-required path targets the inviter as a user/DM recipient and maintainers decide the auto-join opt-in model.
