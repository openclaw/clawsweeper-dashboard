## Summary

Keep this PR open, but do not merge it in its current shape. The Feishu capability is useful and the supplied live proof is credible, yet the patch creates a second implementation of the channel’s policy, routing, binding, session, and reply lifecycle; it also needs an explicit maintainer decision on whether Feishu event subscription alone authorizes automatic meeting joins.

## What This Changes

Adds Feishu VC meeting-invite handling that creates an inbound inviter turn, requests an immediate meeting join, preserves direct-message reply routing, and documents the required event subscription.

## Best Possible Solution

Add a narrow Feishu VC-event adapter that parses and deduplicates the event, then feeds a shared canonical Feishu inbound-turn lifecycle; before landing, explicitly choose and document whether event subscription is sufficient opt-in or whether automatic joining requires a default-off Feishu setting.

## Reproduction Assessment

Yes at source level. Current main has no VC invite event registration, and the PR provides a concrete live Feishu invitation and successful after-fix join path, although this review did not independently rerun the credentialed live scenario.

## Solution Assessment

No. Keeping the event adapter in the Feishu plugin is the correct owner boundary, but copying the complete inbound lifecycle is broader and more drift-prone than introducing a reusable canonical Feishu ingress seam.
