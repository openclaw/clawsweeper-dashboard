## Summary

The PR targets the correct cron finalization gap, but the current head publishes the session context snapshot as `usage.total`, which can omit completion tokens and understate downstream OTel metrics. The linked issue should remain open while this PR is corrected and supplied with real subscriber/exporter proof.

## What This Changes

The PR adds trusted `model.usage` diagnostic emission for nonzero isolated cron runs, including trace, session, context, cost, and duration metadata, plus positive and zero-usage tests.

## Best Possible Solution

Emit the canonical billed aggregate in `usage.total`, keep prompt/context occupancy in `context.used`, assert that distinction with asymmetric usage data, and demonstrate one redacted real scheduled event at the subscriber or exporter boundary.

## Reproduction Assessment

Yes. Current main has a high-confidence source reproduction: isolated cron finalization records nonzero usage without the canonical `model.usage` emission, corroborated by the production report against v2026.6.5.

## Solution Assessment

No, not in the current form. Emitting from cron finalization is the narrowest maintainable fix, but the billed total must not reuse the session context snapshot and the subscriber path still needs real proof.
