## Summary

Keep open. This protected-label PR is not implemented on current main, and the updated head still has a blocking policy bug: evidence-type labels such as `Terminal transcript` can make test/CI-only output pass the new external proof gate.

## What This Changes

The PR adds a real-behavior-proof PR-template section, a required `pull_request_target` check, Barnacle proof labels/actions, contributor/changelog docs, and tests for external PR proof enforcement.

## Best Possible Solution

Land a maintainer-approved gate after tightening the classifier so evidence-type labels do not count without actual non-test live evidence, while preserving the trusted-base workflow design.

## Reproduction Assessment

Yes. Source inspection of the PR head gives a high-confidence path: a `Terminal transcript:` or `console output` label sets `hasRealEvidence`, so `pnpm test` or CI-only proof is not classified as mock-only.

## Solution Assessment

No. The workflow shape is reasonable, but the classifier should reject mock/test/lint/typecheck/CI-only content before accepting evidence labels, or require actual live evidence payload beyond those labels.

## Review Findings

Overall correctness: patch is incorrect

That contradicts the new template and docs, which say tests and CI are supplemental only.
