## Summary

Keep open. Current main does not already implement this gate, the PR has the protected `maintainer` label, and the latest PR head still has a policy bug that can pass mock-only test/CI transcripts when they are labeled as terminal or console output.

## What This Changes

The PR adds a Real behavior proof PR-template section, a `pull_request_target` workflow/check, Barnacle labels/actions, contributor and changelog docs, and tests for requiring external PRs to provide after-fix live evidence.

## Best Possible Solution

Land a maintainer-approved version of the gate after tightening the classifier so test/mock/lint/typecheck/CI-only output is rejected even when wrapped in accepted evidence labels, while keeping the trusted base-checkout workflow design.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a Real behavior proof section whose evidence says `Terminal transcript:` and then shows `pnpm test` or CI output sets `hasRealEvidence`, suppresses `hasMockOnlyEvidence`, and passes despite the stated supplemental-only contract.

## Solution Assessment

No. The workflow shape is reasonable, but the current classifier should reject mock-only content before accepting evidence-type labels or require actual non-test live evidence beyond labels like `terminal transcript` or `console output`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Reject labeled test transcripts before passing proof:** `scripts/github/real-behavior-proof-policy.mjs:60`

That contradicts the new template and docs, which say tests and CI are supplemental only.
