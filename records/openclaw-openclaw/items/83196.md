## Repair Work Prompt

Fix the current issue by adding safe info-level Telegram outbound success logging in extensions/telegram/src/send.ts for the modern durable sendMessageTelegram() path. Log after successful text sends once sendChunkedText returns a message id, and after successful media sends once mediaMessageId is resolved; include accountId, chatId, messageId, operation/media kind, thread id, reply-to id, silent flag, and chunk count when available. Do not log reply text, captions, media URLs, local file paths, sender names/usernames, chat titles, bot tokens, apiRoot, raw request params, quote text, or raw error bodies. Add focused regression coverage in extensions/telegram/src/send.test.ts for text and media success logs and redaction/non-leak assertions. Treat https://github.com/openclaw/openclaw/issues/68770 and https://github.com/openclaw/openclaw/pull/68836 as related media-only context, but do not rely on that PR because it does not cover the durable text sendMessageTelegram path. Changelog is expected only if maintainers treat the log line as user-facing release content.
