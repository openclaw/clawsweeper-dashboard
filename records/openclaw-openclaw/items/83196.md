## What This Changes

Telegram durable replies should emit a journal-facing outbound delivery success line after Telegram accepts a text or media send.

## Best Possible Solution

Keep the merged send-boundary logging implementation and close this issue as fixed by the shipped beta release and current main.

## Reproduction Assessment

No on current main: the source-level gap no longer reproduces because `sendMessageTelegram()` now logs after Telegram message-id resolution for text and media sends. The prior failure mode is covered by focused tests in `extensions/telegram/src/send.test.ts`.

## Solution Assessment

Yes. Logging at the Telegram send boundary after a platform message id is available is the narrow maintainable fix, and the tests verify that message bodies, captions, media URLs, and file names stay out of the log line.
