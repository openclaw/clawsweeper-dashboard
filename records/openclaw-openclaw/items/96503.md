Keep open: this PR is a narrow, likely-correct fix for a current shipped OpenAI-compatible streaming regression, but the external-PR real behavior proof gate is not met because the PR body only shows mocked/focused test output.

## What This Changes

The PR updates the OpenAI SDK guarded fetch path to sniff `application/json` streaming responses, relabel already-SSE bodies as `text/event-stream`, and add regression coverage for the double-prefix case.

## Best Possible Solution

Land the narrow transport fix after real behavior proof and required checks are complete, preserving the existing true-JSON fallback and avoiding a new config option.

## Reproduction Assessment

Yes, at source level. Current main wraps JSON-labeled streaming bodies as SSE while the OpenAI SDK parses one stripped SSE `data` payload as JSON, so an already-SSE body can reach `JSON.parse` as `data: {...}`; I did not run a live gateway in this read-only review.

## Solution Assessment

Yes, this is the narrow best fix shape. Sniffing and relabeling only JSON-labeled already-SSE bodies preserves the existing JSON fallback and is safer than adding the requested strict-mode config option.
