## What This Changes

The PR changes OpenAI-compatible streaming response normalization to sniff JSON-labeled streamed bodies and relabel already-SSE bodies before the JSON-to-SSE fallback can re-prefix them.

## Best Possible Solution

Land this narrow transport fix after current-head validation and hot-path maintainer review, preserving true JSON fallback behavior and avoiding a new config option.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.10 can feed an already-SSE `application/json` body into the JSON-to-SSE fallback, producing `data: data: ...` before the OpenAI SDK parses it.

## Solution Assessment

Yes. Sniffing and relabeling only JSON-labeled already-SSE bodies in `provider-transport-fetch` is the narrowest maintainable fix and is safer than adding the requested strict-mode config option.
