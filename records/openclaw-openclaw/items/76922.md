Keep open: the PR is a focused, still-useful fix for a source-reproducible hooks CLI hang on current main, and I did not find a blocking correctness or security issue in the proposed patch.

## What This Changes

The PR adds a success-path one-shot exit for `hooks list`, `hooks info`, `hooks check`, `hooks enable`, `hooks disable`, and bare `hooks`, plus guard coverage and a changelog entry.

## Best Possible Solution

Land this PR or the same narrow patch after the branch is mergeable and exact-head CI passes, preserving relay/install/update behavior outside the one-shot exit path.

## Reproduction Assessment

Yes, source-reproducible. Current main routes the affected hooks commands through a success path with no exit, and hooks inspection can load plugin modules and invoke `register()`, so an open handle can keep the CLI alive after output.

## Solution Assessment

Yes. The proposed fix is the narrow maintainable path because it exits only after successful one-shot hooks commands, keeps relay/install/update behavior unchanged, and no longer disables the fix under plain `NODE_ENV=test`.
