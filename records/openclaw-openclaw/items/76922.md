## What This Changes

This PR adds a one-shot hooks CLI runner that forces successful hooks list/info/check/enable/disable and bare hooks commands to exit outside Vitest, adds guard coverage, and records the fix in the changelog.

## Best Possible Solution

Resolve the branch conflict, rerun exact-head CI, and land this focused one-shot hooks CLI exit fix without broadening it to relay, install, update, or plugin lifecycle behavior.

## Reproduction Assessment

Yes, source-reproducible. Current main routes the affected hooks commands through a successful return path with no exit, while hooks inspection can load plugin runtime registration, so a plugin that leaves a timer/socket open can keep the CLI alive after output.

## Solution Assessment

Yes. The PR is the narrow maintainable fix because it forces exit only after successful one-shot hooks commands, keeps relay/install/update outside the change, and no longer disables the fix merely under NODE_ENV=test.
