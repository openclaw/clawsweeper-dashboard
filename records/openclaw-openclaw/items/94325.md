## Summary

Keep this PR open, but it is not merge-ready: the intended Form-to-Raw scroll reset is shadowed by the existing render-local helper, so the new callback does not reset `.config-content` and CI reports the added helper as unused/no-shadow.

## What This Changes

The PR adds a Settings config mode-change scroll reset so the Raw JSON textarea should be visible immediately after switching from Form to Raw mode.

## Best Possible Solution

Use one non-shadowed helper that can reset the current `.config-content` on both section changes and Form/Raw mode changes, then add focused browser coverage and refresh real behavior proof.

## Reproduction Assessment

Yes. Source inspection and CI show the current PR callback resolves to the existing render-local `resetContentScroll` with no event target, so it returns without finding `.config-content`; I did not run a local browser repro because this review is read-only.

## Solution Assessment

No. Resetting scroll on mode change is the right fix shape, but this implementation is shadowed; the safer fix is to reuse or rename the existing helper and cover the Form-to-Raw scroll transition in `config.browser.test.ts`.
