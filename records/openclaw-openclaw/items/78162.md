## Summary

Keep open: current main has a source-reproducible path where provider auth can change `agents.defaults.model.primary` even when the user did not pass `--set-default`. The central bug is narrow enough to fix without implementing the broader tier-labeling idea.

## What This Changes

The issue asks OpenClaw to preserve the user’s active default model when adding or refreshing providers, while still making newly available premium models selectable.

## Best Possible Solution

Preserve the existing default model unless the user passes `--set-default` or makes an explicit model selection, while still merging provider catalog entries, aliases, auth profile config, and provider settings.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: provider API-key auth can return a config patch that changes `agents.defaults.model.primary`, and `models auth login` applies that patch before checking `--set-default`. I did not run a live provider login because this review is read-only.

## Solution Assessment

Yes, the narrowest maintainable fix is to restore or preserve the prior default around provider auth config patches when `setDefault` is false. Tier labels and confirmation UI can remain separate follow-up work.
