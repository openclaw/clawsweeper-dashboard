## Summary

Keep open for normal merge review: current main and v2026.6.1 still drop `/reset` args in the Control UI path, while this patch targets the right boundary and the updated screenshots now provide sufficient real UI/Gateway proof.

## What This Changes

This PR preserves `/reset` slash-command arguments in Control UI dispatch, excludes `/reset soft ...` from the hard-reset confirmation path, and adds a focused regression test.

## Best Possible Solution

Land the narrow Control UI fix after normal CI and maintainer review so `/reset soft ...` reaches the existing Gateway parser unchanged and destructive-reset confirmation remains limited to hard reset forms.

## Reproduction Assessment

Yes. Source inspection of current main shows `parseSlashCommand` preserves `/reset soft ...` args, but `dispatchSlashCommand` drops them by sending a hardcoded `/reset`, so the Gateway cannot see the soft reset token.

## Solution Assessment

Yes. The patch fixes the Control UI local command boundary and confirmation detector to match the existing Gateway soft-reset token contract, which is narrower than changing Gateway reset semantics or adding a new command path.
