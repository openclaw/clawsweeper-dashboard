## Summary

Keep open. Current main still trims delivered trailing assistant turns from the durable session JSONL, and that matches the reported Telegram/WebChat transcript loss rather than an already-fixed or stale behavior.

## What This Changes

The issue asks session-file repair to preserve delivered trailing assistant transcript entries and leave provider-specific assistant-prefill stripping to outbound model-request construction.

## Best Possible Solution

Preserve durable delivered assistant replies in `repairSessionFileIfNeeded`, keep only true file-corruption repairs there, and rely on provider-specific outbound sanitizers for assistant-prefill constraints.

## Reproduction Assessment

Yes. A session JSONL ending with a normal non-tool-call assistant message exercises the current trim loop, and the existing test asserts that the assistant entry is deleted from disk.

## Solution Assessment

Yes, with one guard: moving this behavior out of durable repair is the maintainable direction, but the fix should verify strict-provider outbound replay still strips unsupported assistant-prefill turns.
