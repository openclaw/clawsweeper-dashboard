## Summary

Keep open: current main and v2026.6.8 still have explicit auth order, cooldown-aware ordering, and implicit last-used/session rotation, but no selectable primary-first or failover-only Codex/OpenAI auth-profile policy. This remains a compatibility-sensitive product/config decision, with related but non-duplicative open routing and cooldown issues.

## What This Changes

The issue requests an explicit primary-first Codex/OpenAI auth-profile mode for users with one preferred primary account and cold backup accounts.

## Best Possible Solution

Define one compatibility-safe OpenAI/Codex auth-profile policy model around the existing resolver and session override path, with explicit docs and reliable cooldown persistence semantics.

## Reproduction Assessment

Yes at source level: current main and v2026.6.8 have no auth-profile policy field and still implement explicit order plus cooldown/last-used/session rotation. I did not run a live multi-account Codex setup in this read-only review.

## Solution Assessment

Unclear until maintainers choose semantics; a policy enum may be appropriate, but it must preserve compatibility for explicit order, session auto-selection, cooldown sorting, and legacy OpenAI/Codex migration.
