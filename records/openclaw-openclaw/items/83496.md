## Summary

Keep open: current main and the latest release have explicit auth order, cooldowns, round-robin ordering, and session auto-rotation, but no requested primary-first/last-good/load-balance selection mode. Related issues cover broader pool routing, UI exposure, stale profile recovery, and rotation caps, but none is a canonical duplicate for this Codex cold-backup policy.

## What This Changes

The issue requests a Codex auth-profile selection policy so a stable primary Pro account is tried before cold backups unless real failure requires fallback.

## Best Possible Solution

Add a documented auth profile selection policy in the auth-routing layer, preserving current behavior by default unless maintainers choose a migration, with schema/help/docs and focused tests for primary-first fallback and recovery.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config-policy request. Source inspection gives high confidence current main lacks the requested policy because schema and docs expose only order/cooldown controls while the resolver uses round-robin and session rotation.

## Solution Assessment

Unclear until maintainers choose the product semantics. A narrow policy enum may be the right shape, but it should preserve compatibility and define how fallback sessions return to the primary profile.
