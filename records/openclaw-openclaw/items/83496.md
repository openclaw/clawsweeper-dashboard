## Summary

Keep open: current main supports explicit auth order, cooldown rotation, session-pinned overrides, and documented round-robin selection, but it still lacks the requested primary-first/last-good/load-balance auth-profile policy mode. The linked and searched related issues cover stale recovery, rotation caps, UI visibility, or broader provider-pool routing, but none cleanly replaces this Codex cold-backup policy request.

## What This Changes

Request an explicit Codex auth-profile selection mode so a stable primary Pro profile is tried before backup profiles unless it fails.

## Best Possible Solution

Add an explicit, documented auth-profile selection policy in the auth-routing/config contract, preserving current behavior by default and covering primary-first fallback and recovery with focused tests.

## Reproduction Assessment

No high-confidence bug repro is needed because this is a feature/config-policy request. Source inspection gives high confidence that current main lacks the requested policy knob and currently documents round-robin/session-sticky behavior.

## Solution Assessment

Unclear until maintainers choose the product semantics. A narrow policy enum could fit, but it should preserve compatibility for current round-robin users and define how failover returns to the primary after recovery.
