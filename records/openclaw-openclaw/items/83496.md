## Summary

The linked reports cover adjacent stale-profile, rotation, and UI visibility problems, but none is a canonical duplicate for this cold-backup account policy.

## What This Changes

Request an explicit Codex auth-profile selection policy that prefers a stable primary account and uses backup accounts only after failure.

## Best Possible Solution

Add a documented auth profile selection policy in the auth-routing layer, preserving current behavior unless maintainers intentionally choose a migration path, with schema/help/docs and focused tests for primary-first plus fallback recovery.

## Solution Assessment

Unclear until maintainers choose the product semantics. A narrow policy enum could be a good fit, but it should preserve compatibility for existing round-robin users and define how session auto overrides return to the primary.
