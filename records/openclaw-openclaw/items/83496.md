## Summary

## What This Changes

The issue requests an explicit Codex/OpenAI auth-profile routing mode, such as primary-first or failover-only, for multi-account setups with a stable primary account and backup profiles.

## Best Possible Solution

Define a backward-compatible auth-profile routing policy for canonical OpenAI/Codex profiles, reusing existing auth order and cooldown state where possible, then implement the chosen CLI/docs/UI surface with upgrade-safe defaults.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.10 have explicit order, cooldown movement, and round-robin behavior, but no primary-first or failover-only selector. I did not run a live multi-account Codex setup, so this is source-reproducible rather than fully reproduced.

## Solution Assessment
