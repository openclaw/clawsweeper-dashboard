## Summary

Keep open: current main supports explicit auth order, cooldown rotation, and session/round-robin selection, but it does not expose the requested primary-first/last-good/load-balance auth-profile policy mode. The linked issues cover stale profile recovery, rotation caps, and UI visibility, while this issue remains the narrower cold-backup account policy request.

## What This Changes

Request adds explicit Codex/OpenAI auth-profile selection modes so stable primary Pro accounts can stay first while backup accounts remain cold failover.

## Best Possible Solution

Add a documented auth-profile selection policy in the auth-routing layer, preserving current behavior by default unless maintainers explicitly choose a migration, with schema/help/docs and focused tests for primary-first plus fallback recovery.

## Reproduction Assessment

No high-confidence bug reproduction is needed because this is a feature/config-policy request. Source inspection confirms current main has explicit order, round-robin/session rotation, and cooldown fallback, but no primary-first policy mode.

## Solution Assessment

Unclear until maintainers choose the product semantics. A narrow policy enum may be the right shape, but it should preserve compatibility for existing round-robin users and define how auto-selected sessions return to the primary after fallback recovery.
