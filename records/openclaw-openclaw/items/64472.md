## Summary

Keep open. Current main still routes fixed workspace bootstrap files through the workspace-root boundary reader and still records guarded-read failures as missing; this issue is also paired with open fix PRs, including a maintainer-labeled scoped replacement that needs security review rather than a new cleanup close.

## What This Changes

The issue requests loading workspace bootstrap file symlinks that resolve into sibling `~/.openclaw/` directories instead of rendering them as `[MISSING]`.

## Best Possible Solution

Land one scoped workspace-bootstrap fix, preferably along the #73401 shape, that follows explicit fixed top-level bootstrap symlinks to safe regular files while keeping shared boundary-path behavior and negative security tests intact.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a fixed `AGENTS.md` symlink resolving outside the workspace root reaches `openBoundaryFile`, fails the canonical boundary check, and is converted to a missing bootstrap entry; I did not create a filesystem fixture because this review is read-only.

## Solution Assessment

No, not as broadly proposed in the issue. The safer path is a scoped fixed-bootstrap reader exception with negative tests, rather than relaxing the shared `boundary-path` primitive for all callers.
