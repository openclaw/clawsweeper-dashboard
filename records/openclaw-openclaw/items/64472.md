## Summary

Keep open. Current main still routes fixed workspace bootstrap files through the workspace-root safe-open path and converts failed reads into missing entries, while the linked fix PRs are still open and the scoped maintainer replacement needs security review.

## What This Changes

Allow workspace bootstrap files such as `AGENTS.md` to load when they are symlinks to sibling directories under `~/.openclaw/` instead of being rendered as `[MISSING]`.

## Best Possible Solution

Land one scoped fixed-bootstrap symlink implementation that preserves the shared boundary-path contract, rejects unsafe targets, and includes negative security tests for hardlinks, non-regular files, dangling links, oversized files, credential/state roots, and extra/glob bootstrap paths.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an `AGENTS.md` symlink resolving outside the workspace root reaches `openRootFile`, fails the current root boundary contract, and is converted to a missing bootstrap entry.

## Solution Assessment

No, not as broadly proposed in the issue. The safer solution is a scoped fixed-bootstrap reader exception with targeted negative tests, not relaxing the shared boundary primitive for all callers.
