## Summary

Keep open. Current main and the latest shipped release still route fixed workspace bootstrap reads through the workspace-root boundary and render failed reads as missing, while this issue is paired with open implementation PRs that need maintainer/security resolution before closure.

## What This Changes

The issue requests support for workspace bootstrap files that are symlinks into sibling directories under `~/.openclaw` instead of treating them as missing.

## Best Possible Solution

Keep this issue open until a reviewed scoped fixed-bootstrap symlink reader, or an explicit unsupported-policy decision, lands and the related reports can be consolidated.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a fixed bootstrap symlink resolving outside the workspace reaches `openRootFile` under the workspace root, fails the boundary contract, is recorded as missing, and is rendered as `[MISSING] Expected at: ...`; no filesystem fixture was run because this review is read-only.

## Solution Assessment

No. The issue's proposed global `~/.openclaw` fallback in shared boundary checks is wider than the bootstrap symptom; the narrower maintainable path is a scoped fixed-bootstrap reader exception such as https://github.com/openclaw/openclaw/pull/73401 or an explicit policy decision that external bootstrap symlinks are unsupported.
