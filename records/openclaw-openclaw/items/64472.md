## Summary

Keep open. Current main still reads fixed workspace bootstrap files through the workspace-root file boundary and converts guarded-read failures into missing bootstrap entries; this issue is also tied to open implementation PRs, including an open closing PR from the reporter and a maintainer-labeled scoped replacement that needs trust-boundary review.

## What This Changes

The issue requests support for workspace bootstrap file symlinks that point from a workspace into sibling subdirectories under the user’s OpenClaw data directory.

## Best Possible Solution

Keep this open until a reviewed scoped fixed-bootstrap symlink implementation, such as https://github.com/openclaw/openclaw/pull/73401, lands or is explicitly superseded; avoid relaxing the shared fs-safe boundary primitive globally.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a fixed `AGENTS.md` symlink resolving outside the workspace root reaches `openRootFile`, fails the root boundary contract, and is recorded as a missing bootstrap entry; I did not create a filesystem fixture because this review is read-only.

## Solution Assessment

No, not as broadly proposed in the issue body. The safer solution is a scoped fixed-bootstrap reader exception with negative security coverage, not a general OpenClaw-state-directory fallback in shared boundary checks.
