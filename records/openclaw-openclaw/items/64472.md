## Summary

Keep open. Current main still sends fixed workspace bootstrap files through the workspace-root guarded reader and converts rejected reads into missing prompt entries, while open linked PRs still own the implementation path.

## What This Changes

Workspace bootstrap files symlinked from a workspace into sibling directories under `~/.openclaw/` should load instead of rendering as missing.

## Best Possible Solution

Land PR https://github.com/openclaw/openclaw/pull/73401 or an equivalent scoped fixed-bootstrap reader implementation with negative security coverage, then close this issue as implemented.

## Reproduction Assessment

Yes. Source and dependency inspection give a high-confidence path: an `AGENTS.md` symlink resolving outside the workspace root reaches `openRootFile`, fails the root-boundary contract, and is converted into a missing bootstrap entry.

## Solution Assessment

No, not as broadly proposed in this issue. The safer maintainable solution is a scoped fixed-bootstrap reader exception with tests for unsafe targets, not relaxing the shared boundary primitive for all callers.
