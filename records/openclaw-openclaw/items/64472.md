## Summary

Keep open. Current main and the latest release still read fixed workspace bootstrap files through the workspace-root boundary and mark guarded-read failures as missing; the issue is also paired with open implementation PRs, including a reporter PR with closing syntax and a maintainer-labeled scoped replacement that still needs trust-boundary review.

## What This Changes

The issue asks OpenClaw to load fixed workspace bootstrap files when those files are symlinks into a sibling directory under `~/.openclaw`.

## Best Possible Solution

Keep this issue open until a reviewed scoped bootstrap-reader fix, such as https://github.com/openclaw/openclaw/pull/73401 or an explicit successor, lands or maintainers decide the external-symlink trust policy is not planned.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a fixed `AGENTS.md` symlink resolving outside the workspace root reaches `openRootFile`, fails the root boundary contract, is recorded as missing by `loadWorkspaceBootstrapFiles`, and is rendered as `[MISSING] Expected at: ...`; I did not create a filesystem fixture because this review is read-only.

## Solution Assessment

No, not as broadly proposed in the issue body. A global `~/.openclaw` fallback in shared boundary checks is wider than needed; the safer maintainable direction is a scoped fixed-bootstrap reader exception or explicit policy with negative security coverage.
