## Summary

Keep open. Current main still routes fixed workspace bootstrap files through the workspace-root boundary and still marks guarded-read failures as missing; the related open PRs need maintainer security review before this issue can be resolved.

## What This Changes

Allow workspace bootstrap files such as AGENTS.md to be symlinks to sibling directories under `~/.openclaw/` and still load into startup prompt context.

## Best Possible Solution

Land one scoped fix for explicit fixed workspace bootstrap files that follows readable symlinks to regular targets while preserving rejection of hardlinks, dangling symlinks, non-regular files, oversized files, extra/glob paths, and arbitrary shared-boundary relaxation unless maintainers explicitly approve that trust model.

## Reproduction Assessment

Yes. The source path is high-confidence: a workspace `AGENTS.md` symlink resolving to a sibling `~/.openclaw/` target is outside `resolvedDir`, so the boundary resolver rejects it and `loadWorkspaceBootstrapFiles` records it as missing; I did not create a fixture because this review is read-only.

## Solution Assessment

No, not as broadly proposed in the issue. A targeted workspace-bootstrap reader exception with negative security tests, like the maintainer replacement path describes, is safer than relaxing the shared `boundary-path` primitive for all callers.
