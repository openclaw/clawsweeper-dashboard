## Summary

Keep open. Current main still lacks a universal pre-filter raw inbound plugin hook, the linked WhatsApp-only implementation PR is closed unmerged, and existing message hooks run after admission/gating paths rather than before them.

## What This Changes

Add an observe-only plugin hook and optional channel config/sink for raw inbound messages before allowlist, sender, and mention filtering.

## Best Possible Solution

Define an explicit opt-in, observe-only, privacy-documented SDK/channel contract, then wire participating channels with ordering tests before adding any sink or retention surface.

## Solution Assessment

Unclear whether the exact hook name or configurable sink is the right API. A maintainer-approved, opt-in, privacy-aware SDK/channel contract is the narrowest maintainable path before implementation.
