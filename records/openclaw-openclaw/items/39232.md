## Summary

Keep open. Current main still lacks a universal pre-filter raw inbound Plugin SDK hook; existing message hooks are downstream of channel admission, the WhatsApp-only implementation at https://github.com/openclaw/openclaw/pull/39240 was closed unmerged, and the newer archive-plugin report at https://github.com/openclaw/openclaw/issues/47167 overlaps but does not fully settle the before-allowlist/before-sender-policy contract requested here.

## What This Changes

Add an observe-only raw inbound plugin hook, and possibly a channel-level observation sink, that fires before allowlist, sender-policy, and mention gates.

## Best Possible Solution

Define an explicit opt-in, observe-only, privacy-documented SDK/channel contract for early inbound visibility, then coordinate it with the archive-plugin and Telegram passive-observation threads before adding any retention sink.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-main verification path: no raw/pre-filter hook is registered, and representative WhatsApp gates still drop messages before downstream observation surfaces.

## Solution Assessment

Unclear whether the exact `on_raw_inbound` name or sink option is the best API. The maintainable path is a maintainer-approved, opt-in, privacy-aware SDK/channel contract rather than a silent global observer or a WhatsApp-only private hook.
