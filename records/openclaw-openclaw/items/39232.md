## Summary

Keep open. Current main still lacks a universal pre-filter raw inbound plugin hook, and the existing WhatsApp `message_received` surface is opt-in and emitted only after access control and group activation gates; the related WhatsApp-only PR was closed unmerged.

## What This Changes

Request a pre-filter, observe-only plugin hook plus optional channel config for passive inbound message observation before allowlist, sender-policy, and mention gates.

## Best Possible Solution

Define an explicit opt-in, observe-only, privacy-documented SDK/channel contract for pre-filter inbound observation, then wire participating channels with ordering tests before exposing any sink config.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence current-main verification path. Source inspection shows no registered raw/pre-filter hook, and WhatsApp still drops or skips messages before downstream `message_received` emission.

## Solution Assessment

Unclear whether the exact proposed `on_raw_inbound` name is the best API. The maintainable solution is a deliberately named, opt-in, privacy-aware SDK/channel contract rather than a silent global observer or a WhatsApp-only private hook.
