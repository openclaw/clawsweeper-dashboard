## Summary

Keep open. Current main still lacks a universal pre-filter raw inbound plugin hook, and the existing WhatsApp `message_received` surface remains opt-in and downstream of access-control and group activation gates; the linked WhatsApp-only PR was closed unmerged.

## What This Changes

The issue requests an observe-only pre-filter inbound plugin hook, plus optional channel observation configuration, so plugins can passively inspect incoming channel messages before routing gates.

## Best Possible Solution

Define an explicit opt-in, observe-only, privacy-documented SDK/channel contract for pre-filter inbound observation, then wire participating channels with ordering tests before adding any sink config.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence current-main verification path. Source inspection shows no registered raw/pre-filter hook, and WhatsApp still drops or skips messages before downstream `message_received` emission.

## Solution Assessment

Unclear whether the exact proposed `on_raw_inbound` name is the best API. The maintainable solution is a deliberately named, opt-in, privacy-aware SDK/channel contract rather than a silent global observer or a WhatsApp-only private hook.
