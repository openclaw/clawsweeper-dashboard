## Summary

Keep open. Current main still lacks a universal pre-filter raw inbound plugin hook; the existing `message_received` path is a downstream, opt-in observation surface, and the related WhatsApp-only PR was closed unmerged.

## What This Changes

Request an observe-only raw inbound plugin hook, optionally with channel observe/sink config, that fires before allowlists, sender policy, and mention gating.

## Best Possible Solution

Define a versioned, explicit opt-in, observe-only raw inbound SDK/channel contract with privacy docs and ordering tests, then wire participating channels deliberately instead of adding a WhatsApp-only private sink.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence current-main verification path. Source inspection shows no raw/pre-filter hook, and WhatsApp still applies access control and group gating before the existing `message_received` emission.

## Solution Assessment

Unclear whether the exact proposed `on_raw_inbound` API is the best shape. The maintainable path is an explicitly opt-in, privacy-documented SDK/channel contract rather than a silent global observer or a WhatsApp-only private hook.
