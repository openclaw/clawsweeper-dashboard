## Summary

Keep open. Current main still lacks a universal pre-filter raw inbound plugin hook, the existing typed hook surfaces are downstream of channel admission, and the linked WhatsApp-only implementation PR was closed unmerged rather than superseding the request.

## What This Changes

Request an observe-only raw inbound plugin hook and optional channel observe/sink config that fires before allowlist, sender policy, and mention gates.

## Best Possible Solution

Define an explicit opt-in, observe-only, privacy-documented SDK/channel contract, then wire participating channels with ordering tests before adding any sink config.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-main verification path: no raw/pre-filter hook is registered, and WhatsApp still drops messages before downstream observation surfaces.

## Solution Assessment

Unclear whether the exact `on_raw_inbound` name or sink option is the best API. The maintainable path is a maintainer-approved, opt-in, privacy-aware SDK/channel contract rather than a silent global observer or WhatsApp-only private hook.
