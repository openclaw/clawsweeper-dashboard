## Summary

Keep open. The PR is a plausible direction for the linked macOS Canvas auth bug, but it is not merge-ready because it resolves through the app-wide gateway snapshot instead of the node-session capability source/refresh path, logs capability-bearing URLs publicly, and still lacks real macOS proof.

## What This Changes

The PR adds macOS Canvas logic and tests to resolve `/__openclaw__/canvas/*` and `/__openclaw__/a2ui/*` targets against a cached gateway canvas capability URL before loading them in WKWebView.

## Best Possible Solution

Resolve node-invoked hosted Canvas/A2UI paths from the `GatewayNodeSession` canvas surface URL with refresh support, preserve arbitrary external/local targets, and redact capability-bearing diagnostics before asking for real macOS proof.

## Reproduction Assessment

Yes from source, but not from live macOS execution here. Current main sends node Canvas targets into WKWebView without capability rewriting, while docs and node-session code require scoped plugin surface URLs for protected hosted Canvas paths.

## Solution Assessment

No. Capability URL rewriting is the right direction, but this PR wires it to the app-wide gateway snapshot instead of the node-session capability source and refresh path, and it logs the credential-bearing URL.

Full review comments:

- **[P1] Resolve node Canvas paths from the node session:** `apps/macos/Sources/OpenClaw/CanvasManager.swift:161`
  - body: `canvas.present` and `canvas.navigate` arrive through `MacNodeRuntime`, whose production coordinator already injects `GatewayNodeSession.currentCanvasHostUrl()` and `refreshCanvasHostUrl()`. Caching `pluginsurfaceurls.canvas` from `GatewayConnection.shared` in `CanvasManager` can be nil, stale, or from the app-wide operator connection, so the remote-node path can still fall back to the unauthenticated load or an expired capability.
  - confidence: 0.88
- **[P1] Redact capability URLs before logging:** `apps/macos/Sources/OpenClaw/CanvasWindowController.swift:220`
  - body: The resolved URL includes the `/__openclaw__/cap/<token>` capability segment, and logging `resolved.absoluteString` with `privacy: .public` exposes a scoped route credential in local diagnostics. Log a redacted value or mark the sensitive segment private before loading.
  - confidence: 0.9
