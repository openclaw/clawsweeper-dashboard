## Summary

Keep open. This is a focused macOS Canvas auth bugfix candidate, but the patch is not merge-ready because it resolves node-invoked Canvas paths from an app-wide gateway snapshot instead of the existing node-session capability source/refresh path, logs a capability-bearing URL publicly, and still lacks redacted real macOS behavior proof.

## What This Changes

The PR caches `pluginsurfaceurls.canvas` in macOS CanvasManager, resolves Canvas/A2UI targets in CanvasWindowController, and adds resolver smoke coverage.

## Best Possible Solution

Resolve hosted Canvas/A2UI targets from MacNodeRuntime's node-session current/refresh capability URL path, preserve local and external loads, redact diagnostics, and require redacted real macOS proof before merge.

## Reproduction Assessment

Yes, from source inspection rather than live macOS execution. Current main sends node Canvas targets into WKWebView without capability rewriting, while gateway docs and node-session code require scoped plugin surface URLs for protected hosted Canvas paths.

## Solution Assessment

No. Capability URL rewriting is the right direction, but this PR wires it to the app-wide gateway snapshot instead of the node-session capability source and refresh path, and it logs the credential-bearing URL.
