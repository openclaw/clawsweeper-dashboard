Keep open: this PR targets a real macOS remote Canvas auth bug, but the current diff uses the app-wide gateway snapshot instead of the node-session capability refresh path, logs a capability-bearing URL publicly, and still lacks real macOS behavior proof.

## What This Changes

The branch caches `pluginsurfaceurls.canvas` in macOS CanvasManager, resolves Canvas/A2UI targets in CanvasWindowController, and adds resolver smoke-test coverage.

## Best Possible Solution

Resolve node-invoked hosted Canvas/A2UI paths through the GatewayNodeSession current/refresh capability URL path, preserve external/local targets, redact capability-bearing diagnostics, and add redacted real macOS proof.

## Reproduction Assessment

## Solution Assessment

No. Capability URL rewriting is the right direction, but this PR wires it to the app-wide gateway snapshot instead of the node-session capability source and refresh path, and it logs the credential-bearing URL.

Full review comments:

- **[P1] Resolve node Canvas paths from the node session:** `apps/macos/Sources/OpenClaw/CanvasManager.swift:161`
  - body: `canvas.present` and `canvas.navigate` arrive through `MacNodeRuntime`, whose production coordinator already injects `GatewayNodeSession.currentCanvasHostUrl()` and `refreshCanvasHostUrl()`. Caching `pluginsurfaceurls.canvas` from `GatewayConnection.shared` in `CanvasManager` can be nil, stale, or from the app-wide operator connection, so the remote-node path can still fall back to the unauthenticated load or an expired capability.
  - confidence: 0.88
