## Summary

Keep open. The PR targets a real macOS Canvas auth gap, but the patch appears to wire hosted-path resolution to the app-wide gateway snapshot instead of the node session that receives `canvas.present`, and it adds public logging of capability-bearing URLs.

## What This Changes

The PR adds macOS Canvas logic to resolve `/__openclaw__/canvas/*` and `/__openclaw__/a2ui/*` targets against a cached gateway canvas capability URL, with new smoke-test coverage.

## Best Possible Solution

Resolve node-invoked hosted Canvas/A2UI paths from the `GatewayNodeSession` canvas surface URL, refresh that capability when needed, keep arbitrary external/local targets unchanged, and redact capability-bearing diagnostics.

## Reproduction Assessment

Yes from source, but not from live macOS execution here. Current main sends node `canvas.present` targets into CanvasManager unchanged, while the gateway contract requires scoped plugin surface capability URLs for protected hosted Canvas paths.

## Solution Assessment

No. Capability URL rewriting is the right direction, but this implementation appears to use the app-wide gateway snapshot instead of the node-session capability source and does not refresh expiring capability URLs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Resolve node canvas paths from the node session:** `apps/macos/Sources/OpenClaw/CanvasManager.swift:158`
  - body: `canvas.present` and `canvas.navigate` are invoked through `MacNodeRuntime`, which already receives `GatewayNodeSession.currentCanvasHostUrl()` and `refreshCanvasHostUrl()`. Caching `pluginsurfaceurls.canvas` from `GatewayConnection.shared` in `CanvasManager` can be nil, stale, or from a different app-wide connection, so the remote-node path can still fall back to the local canvas scheme or an expired capability and reproduce the 401. Resolve hosted canvas/a2ui targets using the node-session surface URL and refresh hook before handing the target to CanvasManager.
  - confidence: 0.88
- **[P1] Redact capability URLs before logging:** `apps/macos/Sources/OpenClaw/CanvasWindowController.swift:219`
  - body: The resolved URL includes the `/__openclaw__/cap/<token>` capability segment, and logging it with `privacy: .public` exposes a route credential in local diagnostics. Log a redacted origin/path or mark the sensitive part private instead of emitting the full capability URL.
  - confidence: 0.9
