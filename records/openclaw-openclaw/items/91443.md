## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Resolve hosted canvas paths from the node session:** `apps/macos/Sources/OpenClaw/CanvasManager.swift:158`
  - body: `canvas.present` and `canvas.navigate` are invoked through `MacNodeRuntime`, whose coordinator already injects `GatewayNodeSession.currentCanvasHostUrl()` and `refreshCanvasHostUrl()`. Caching `pluginsurfaceurls.canvas` from `GatewayConnection.shared` in `CanvasManager` can leave remote-node loads on a nil, stale, or app-wide capability URL, so the reported 401 can remain for the node path this PR is meant to fix.
  - confidence: 0.88
- **[P1] Redact capability-bearing URLs before logging:** `apps/macos/Sources/OpenClaw/CanvasWindowController.swift:219-220`
  - body: The resolved URL includes the `/__openclaw__/cap/<token>` capability segment, and logging it with `privacy: .public` exposes a route credential in local diagnostics. Log a redacted origin/path or mark the sensitive portion private before emitting this new resolved-load diagnostic.
  - confidence: 0.86
