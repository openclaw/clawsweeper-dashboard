## Summary

Keep open. Current main and v2026.6.10 still enqueue WhatsApp connected/disconnected lifecycle notices into prompt-visible system events, and the requested threshold, verbosity, or surface-aware policy remains an unimplemented product/config decision.

## What This Changes

The issue requests suppressing, throttling, or surface-filtering transient channel connection status system events so short reconnects do not clutter unrelated active sessions or agent context.

## Best Possible Solution

Implement a tested WhatsApp-owned lifecycle notification policy with default-preserving behavior, short-outage coalescing, and docs/config coverage; generalize only if multiple channel plugins need the same contract.
