## Summary

Keep open. Current main and v2026.6.6 still enqueue WhatsApp connected/disconnected lifecycle notices into the resolved session and drain retained queue entries into prompt-visible `System:` lines; the older banner-filter PR was closed as superseded by this issue, not merged.

## What This Changes

The issue requests suppression, throttling, or surface-aware filtering for transient channel connection status system events that clutter active sessions.

## Best Possible Solution

Implement a tested WhatsApp-owned lifecycle notification policy with reconnect coalescing or a threshold, then promote a generic SDK/helper contract only if more channel plugins need the same behavior.

## Reproduction Assessment

Yes, source-reproducible. Current main still enqueues WhatsApp lifecycle notices and the shared prompt drain formats retained queue entries as `System:` lines.

## Solution Assessment

Unclear until maintainers choose the user-facing policy. The older shared prefix filter is too broad; the safer direction is a WhatsApp-owned lifecycle policy that preserves terminal outage context.
