## Summary

Keep open. Current main and v2026.6.10 still enqueue WhatsApp connected/disconnected lifecycle notices into prompt-visible system events, and the remaining fix is a product/config policy decision rather than an implemented or superseded item.

## What This Changes

Request to suppress, throttle, or surface-filter transient channel connection lifecycle system events, especially short WhatsApp reconnect notices.

## Best Possible Solution

Maintainers should choose a default-compatible WhatsApp-owned lifecycle notification policy that coalesces short reconnects, preserves terminal operator-action notices, and documents any config surface before generalizing it.

## Reproduction Assessment

## Solution Assessment

Unclear until maintainers choose the product policy. A shared prefix filter is too broad because it can hide terminal disconnect context; a channel-owned lifecycle policy with explicit defaults is safer.
