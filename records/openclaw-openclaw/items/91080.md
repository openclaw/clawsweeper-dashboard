## Summary

Keep this PR open: it contains a useful gateway startup diagnostic and now has real terminal proof for the watchdog module, but the patch still cancels the watchdog too early for multi-bind startup and adds an undocumented default-on env surface.

## What This Changes

The PR adds a one-shot gateway startup watchdog that writes active diagnostic phases to stderr when HTTP listen does not complete before a configurable timeout.

## Best Possible Solution

Cancel the watchdog after the full HTTP listen phase completes, add multi-bind regression coverage, and document the env knob with its default and disable behavior.

## Reproduction Assessment

Yes for the patch defect by source inspection: current main can bind multiple loopback hosts, while the PR disarms the watchdog after the first per-host listen callback. I did not run a full gateway hang reproduction under this read-only review.

## Solution Assessment

No; the watchdog direction is useful, but the cancel point should match the full `http.listen` completion boundary and the new env surface should be documented.

Full review comments:

- **[P2] Cancel only after the full listen phase completes:** `src/gateway/server/http-listen.ts:41`
  - body: `resolveGatewayListenHosts("127.0.0.1")` can return both `127.0.0.1` and `::1`, and `startListening()` waits through every bind before `http.bound`. Canceling the watchdog inside each per-host `listenGatewayHttpServer()` disarms it after the first socket, so a hang in a later bind is silent again. Move cancellation to the full `startListening()` completion boundary and cover the multi-host case.
  - confidence: 0.87
- **[P2] Document the new watchdog env var:** `src/logging/gateway-startup-watchdog.ts:20`
  - body: This adds the public `OPENCLAW_STARTUP_WATCHDOG_MS` knob and a default-on 60s startup behavior, but the gateway profiling docs only list the existing startup/restart/diagnostics env vars. Please document the default, `0` disable behavior, and relation to startup trace so operators know how to use and disable it.
  - confidence: 0.82
