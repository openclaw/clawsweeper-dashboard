## Summary

Keep open: the diagnostic idea is useful, but this patch cancels the watchdog before the full multi-host listen phase completes, adds an undocumented env/default surface, and lacks after-fix real behavior proof.

## What This Changes

The PR adds a gateway startup watchdog that emits the active diagnostic phase stack to stderr when HTTP listen has not completed before a configurable timeout.

## Best Possible Solution

Cancel the watchdog at the full http.listen completion boundary, document the env knob with its default and disable value, and require redacted live stderr/log proof before merge.

## Reproduction Assessment

Yes for the patch defect: source shows non-Windows loopback can bind multiple hosts, while the PR cancels on the first per-host listening event before startListening completes. I did not run a live hang scenario.

## Solution Assessment

No; the direction is useful, but the current helper-level cancellation is too early for the gateway listen lifecycle, and the new env surface needs docs plus real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Cancel only after the full listen phase completes:** `src/gateway/server/http-listen.ts:41`
  - body: `resolveGatewayListenHosts("127.0.0.1")` can produce both `127.0.0.1` and `::1`, and `startListening()` waits through every bind before `http.bound`. Canceling the watchdog inside each per-host `listenGatewayHttpServer()` call disarms it after the first socket, so a hang in the remaining `http.listen` work goes back to silent startup. Move cancellation to the full `startListening()` completion boundary and cover the multi-host case.
  - confidence: 0.87
- **[P2] Document the new watchdog env var:** `src/logging/gateway-startup-watchdog.ts:20`
  - body: This adds the public `OPENCLAW_STARTUP_WATCHDOG_MS` knob and a default-on 60s startup behavior, but the gateway profiling docs only list the existing startup/restart/diagnostics env vars. Please document the default, `0` disable behavior, and relation to startup trace so operators know how to use and disable it.
  - confidence: 0.82
