## Summary

## What This Changes

Adds `prompt_tokens`/`completion_tokens` fallbacks to CLI JSON/JSONL usage parsing and a JSONL regression test for llama.cpp-shaped usage.

## Best Possible Solution

Route CLI JSON/JSONL usage through `normalizeUsage()` with a small adapter that preserves Gemini `stats.cached` semantics, keep focused parser coverage, and require redacted real CLI-backed proof or a maintainer proof override.

## Reproduction Assessment

Yes, at source level: a CLI JSON/JSONL payload with `prompt_tokens` and `completion_tokens` reaches `toCliUsage()`, while current main only reads input/output aliases there. I did not run a live llama.cpp or CLI-backed provider setup in this read-only review.

## Solution Assessment

No: the direction is right, but adding another alias list is not the best fix because `normalizeUsage()` already owns these provider usage aliases. A narrow CLI adapter to the shared normalizer is the safer maintainable path.
