## What This Changes

## Best Possible Solution

Route CLI JSON/JSONL usage through the shared `normalizeUsage()` contract, keep focused regression coverage, and require redacted real CLI-backed proof or a maintainer proof override before merge.

## Reproduction Assessment

## Solution Assessment

No: the direction is useful, but adding a second alias chain is not the best fix when `normalizeUsage()` already centralizes this contract. A narrow adapter from CLI usage payloads into the shared normalizer is the safer maintenance path.
