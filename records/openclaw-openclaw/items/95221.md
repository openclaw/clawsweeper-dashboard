The PR adds an EADDRNOTAVAIL guard to Telegram fetch fallback classification, logs a local-socket diagnostic, and adds a direct fetch regression test.

## Best Possible Solution

Fix both the direct fetch classifier and the probe forced-fallback boundary so EADDRNOTAVAIL never promotes Telegram IP rotation, then add focused regression coverage and redacted real runtime/log proof or an explicit maintainer proof override.
