## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Keep unclosed reasoning hidden after visible text:** `src/shared/text/reasoning-tags.ts:341-342`
  - body: `flush()` now emits `droppedThinking` for any unclosed reasoning span. For a malformed stream like `visible <think>private`, the filter has already delivered `visible ` and then appends `private` at end, while the existing sanitizer only recovers unclosed content when there was no visible prefix; this reintroduces the visible and persisted reasoning leak this PR is meant to stop.
  - confidence: 0.93
- **[P1] Preserve orphan close boundaries for downstream stripping:** `src/shared/text/reasoning-tags.ts:325-330`
  - body: Outside a reasoning span, an orphan close tag is consumed here without applying the current sanitizer's orphan-close rule. With split chunks like `private </thi` + `nk>answer`, the prefix can already be emitted as visible text, then the close marker is dropped, so delivery-time stripping no longer has the boundary needed to suppress the private prefix.
  - confidence: 0.9
