## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep unclosed reasoning hidden after visible text:** `src/shared/text/reasoning-tags.ts:341-342`
  - body: `flush()` currently emits `droppedThinking` whenever the stream ends inside a reasoning span. For chunks like `Visible <think>secret`, the filter has already emitted `Visible `, then this flush emits `secret`, while the existing strict sanitizer contract keeps that trailing unclosed reasoning hidden unless the whole output would otherwise be empty. That can still leak chain-of-thought into channel-visible text and persisted session parts.
  - confidence: 0.93
- **[P1] Preserve orphan close boundaries for downstream sanitizing:** `src/shared/text/reasoning-tags.ts:325-331`
  - body: When a `</think>` appears without a prior open tag, the existing sanitizer uses that boundary to drop the reasoning prefix and keep only the final answer. The streaming filter can emit the prefix before seeing the close tag, then removes the close tag here, leaving downstream delivery/history sanitizers no marker to repair the text. Either match the strict sanitizer behavior or leave the orphan boundary intact for the final sanitizer.
  - confidence: 0.86
