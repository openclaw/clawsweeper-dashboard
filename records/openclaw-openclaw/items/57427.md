## Summary

Close as superseded by the upstream OpenShell fix. The observable failure was OpenShell proxy placeholder resolution for generic provider credentials in URL query parameters, and NVIDIA/OpenShell now tracks and fixes that exact behavior; OpenClaw should not work around it by broadly forwarding host dotenv secrets into sandbox exec.

## What This Changes

Request that OpenClaw make `.env`/generic OpenShell provider credentials resolve inside OpenShell-backed sandbox exec so `openshell:resolve:env:*` placeholders do not reach outbound curl URLs.

## Best Possible Solution

Keep OpenClaw's sandbox env isolation intact for this issue and rely on the updated OpenShell proxy credential resolver; handle broader sandbox env forwarding only through a separately approved OpenClaw contract.

## Reproduction Assessment

Yes for the historical setup: source inspection shows OpenClaw does not inject host/global dotenv values into sandbox exec, and upstream OpenShell 0.0.16 left generic-provider query placeholders unresolved. The unresolved-placeholder failure itself is now superseded by the upstream OpenShell fix.

## Solution Assessment

Yes. For this specific failure, the narrow maintainable solution is the upstream OpenShell proxy rewrite fix rather than OpenClaw forwarding resolved host secrets into every sandbox exec environment.
