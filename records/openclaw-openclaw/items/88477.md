Full review comments:

- **[P1] Persist project IDs in session patches:** `src/gateway/server-methods/sessions.ts:1118`
  - body: `sessions.create` and `projects.sessions.attach` now pass `projectId`, and list filtering depends on `entry.projectId`, but `applySessionsPatchToStore` never handles that field. As a result, attaching or creating a project chat reports success without storing the association, so the project session list remains empty and detach cannot clear anything.
  - confidence: 0.96
- **[P2] Regenerate locale bundles for the new tabs:** `ui/src/i18n/locales/en.ts:243-244`
  - body: This adds `projects` and `snesStudio` only to the English locale. The shipped-locale test flattens every locale and expects it to exactly match English, so the UI i18n lane will fail until the generated locale bundles and metadata are refreshed.
  - confidence: 0.91
- **[P2] Bound uploaded resource size before decoding:** `src/projects/store.ts:699`
  - body: `contentBase64` is accepted without a schema max length and decoded with `Buffer.from` before the store checks `MAX_RESOURCE_BYTES`. A large authenticated Gateway request can allocate memory first and only then return the size error, so reject by encoded length before decoding or add a protocol limit.
  - confidence: 0.88
- **[P2] Remove backup restore scripts from this dashboard PR:** `work/scripts/backup-sync-github.sh:4`
  - body: The PR adds operator-specific backup automation with hardcoded `/Users/openclaw` paths and private backup repository push/clone behavior. That is unrelated to Projects/SNES dashboard work and changes sensitive operational behavior that needs a separate owner review if it should land.
  - confidence: 0.9
