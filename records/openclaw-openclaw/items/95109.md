## Summary

Keep open: current main and latest release still run Tavily's strict SecretRef normalization before the documented TAVILY_API_KEY environment fallback, and the live linked fix PRs remain open rather than merged.

## What This Changes

The issue reports that Tavily search/extract fail when a configured env SecretRef is unresolved before Tavily can fall back to TAVILY_API_KEY from the gateway process environment.

## Best Possible Solution

Land one guarded Tavily-local inspect-mode resolver that only falls back to ambient TAVILY_API_KEY for the matching permitted env SecretRef, with regression coverage and runtime proof, then close this issue as implemented.

## Reproduction Assessment

Yes, at source level. Current main still routes Tavily configured apiKey through strict SecretRef normalization before the documented TAVILY_API_KEY fallback, so an unresolved configured SecretRef can throw before fallback.

## Solution Assessment

Yes on the fix boundary: the Tavily plugin resolver is the right place to repair this. The safest implementation should use inspect-mode SecretRef resolution and preserve explicit nonmatching SecretRef boundaries instead of broad catch-and-fallback behavior.
