Yes. Source inspection shows an env SecretRef at `plugins.entries.tavily.config.webSearch.apiKey` reaches strict `normalizeResolvedSecretInputString` before the `process.env.TAVILY_API_KEY` fallback; the live command was not run in this read-only review.

## Solution Assessment

Yes. A narrow plugin-owned resolver fix that uses inspect-mode SecretRef handling with guarded env fallback is the maintainable path; core SecretInput strict behavior should remain intact.
