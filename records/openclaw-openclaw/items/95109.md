The issue reports that Tavily runtime search/extract tools throw on an unresolved env SecretRef before reaching the `TAVILY_API_KEY` process-env fallback.

## Best Possible Solution

Land one focused guarded Tavily resolver fix that preserves literal config and only falls through to the matching env fallback when the configured ref is unavailable.
