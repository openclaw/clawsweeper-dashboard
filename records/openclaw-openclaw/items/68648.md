## Summary

Keep open: the PR targets a real async exec followup timeout path, but the exact diff would return after the accepted ack and skip direct fallback if the resumed agent run fails afterward.

## What This Changes

The PR changes exec approval completion followups to resolve after the gateway accepts the resumed agent request and updates focused tests to expect accepted-only semantics.

## Best Possible Solution

Keep the useful timeout fix direction, but revise it so followups avoid a full-turn wait while still detecting terminal startup or dispatch failures and sending direct fallback when needed.

## Reproduction Assessment

Yes, at source level. A focused test can mock an accepted gateway response followed by a terminal error, and current gateway/client source shows `expectFinal: false` resolves on the accepted frame before fallback can run.

## Solution Assessment

No. Accepted-only semantics fixes the timeout symptom, but the maintainable fix needs a bounded observer such as `agent.wait` or equivalent logic that preserves direct fallback on post-accept failures.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve fallback on post-accept followup failures:** `src/agents/bash-tools.exec-approval-followup.ts:235`

The gateway can still emit a terminal error after deferred dispatch starts, so post-accept setup or dispatch failures now skip `sendDirectFollowupFallback` and can drop the exec completion followup.
