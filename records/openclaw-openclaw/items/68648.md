## Summary

Current main already shipped the useful fix in a safer form: exec approval followups no longer wait on the initial `agent` RPC final result, but accepted runs are followed with `agent.wait` so later terminal failures still reach the fallback path.

## What This Changes

This PR changes exec approval followup session resumes from final-result waiting to accepted-only waiting and updates focused assertions.

## Best Possible Solution

Keep the v2026.5.18 implementation from https://github.com/openclaw/openclaw/pull/82717 and leave the broader dispatch-failure visibility work to https://github.com/openclaw/openclaw/issues/78738.

## Solution Assessment

Yes for current main, not for this branch. The shipped replacement is the safer fix because it avoids the initial final-result wait while still observing post-accept failures before deciding whether fallback delivery is needed.
