## Summary

Keep this PR open, but do not merge it as-is. The accepted-only wait fixes the reported freeze direction, yet the current diff returns after the accepted frame and can skip the existing direct fallback when the gateway later reports a terminal dispatch failure.

## What This Changes

The PR changes exec approval followup session resumes to wait only for gateway acceptance instead of the final resumed turn, with focused regression tests.

## Best Possible Solution

Keep the accepted-only direction, but preserve the existing direct fallback when a resumed followup run fails shortly after acceptance.

## Reproduction Assessment

Yes, at source level. Current main waits with `expectFinal: true`, and the gateway client/server contract shows that `expectFinal: false` resolves on accepted before deferred dispatch can still fail.

## Solution Assessment

No. Accepted-only waiting is the right direction for the freeze, but this patch is incomplete because it drops fallback behavior for terminal failures emitted after the accepted ack.
