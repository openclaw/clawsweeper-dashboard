## Summary

Keep open. The PR fixes a real long-wait symptom, but the current diff would also skip the existing direct fallback when a resumed agent run is accepted and then fails during deferred setup or dispatch.

## What This Changes

The PR changes exec approval followup agent resumes to use `{ expectFinal: false }` and updates focused tests to assert accepted-only behavior.

## Best Possible Solution

Keep the accepted-only resume path, capture the accepted run id, and use a bounded terminal observer so immediate setup or dispatch failures still route through the direct followup fallback.

## Reproduction Assessment

Yes, at source level. With `{ expectFinal: false }`, the client resolves on the accepted frame while the gateway can still emit a terminal error after deferred dispatch, so the existing fallback can be skipped.

## Solution Assessment

No. Accepted-only waiting is the right direction for the timeout symptom, but the patch should preserve fallback behavior for post-accept terminal failures.
