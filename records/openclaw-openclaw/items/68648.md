## Summary

Keep open. The PR addresses a valid async exec followup timeout path, but the current diff is incomplete because accepted-only waiting can skip the existing direct fallback when the resumed gateway run fails after the accepted ack.

## What This Changes

The PR switches exec approval followup session resumes from final-result waiting to accepted-only waiting and updates focused tests.

## Best Possible Solution

Keep the accepted-only resume direction, but preserve a bounded failure/fallback path for post-accept terminal errors without reintroducing a full final-turn wait.

## Reproduction Assessment

Yes, at source level. Current main waits for final results, while the PR’s accepted-only change would resolve on the accepted frame even though the gateway can still emit a later terminal error.

## Solution Assessment

No. Accepted-only waiting is the right direction for the freeze, but this patch should also preserve fallback or visible failure handling for terminal errors emitted after acceptance.
