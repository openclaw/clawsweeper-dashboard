## Summary

Keep open: the accepted-only resume direction addresses the reported timeout/freeze class, but this patch currently bypasses fallback handling for terminal errors emitted after the accepted ack and still lacks inspectable real behavior proof.

## What This Changes

The PR changes exec approval followup session resumes to call the gateway with `expectFinal: false` and updates focused tests to assert accepted-only behavior.

## Best Possible Solution

Keep the accepted-only resume direction only if the followup path also preserves a bounded post-accept failure or fallback signal, with regression coverage and inspectable real runtime proof.

## Reproduction Assessment

Yes, at source level. Current main waits for final results, while the proposed accepted-only change would resolve on the accepted frame even though the gateway can still emit a later terminal error for the same run.

## Solution Assessment

No. Switching to accepted-only waiting is a plausible direction for the freeze, but this patch is not complete because deferred terminal errors no longer reach the existing fallback path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve fallback after accepted followup failures:** `src/agents/bash-tools.exec-approval-followup.ts:235`

That skips `sendDirectFollowupFallback`, so an exec completion followup can be dropped when the gateway emits a terminal error after acceptance.
