## Summary

Keep this PR open, but the current patch should not merge as-is: it fixes a real timeout symptom, yet it also returns success on the accepted frame before post-accept dispatch failures can trigger the existing direct fallback.

## What This Changes

The PR changes exec approval followup session resumes to wait for gateway acceptance instead of the final resumed agent result, with focused test updates.

## Best Possible Solution

Keep accepted-only session resume, capture the accepted run id, and use a short terminal observer so immediate post-accept errors still trigger direct fallback while pending or timeout remains non-blocking.

## Reproduction Assessment

Yes, at source level. A focused test can mock an accepted `agent` response followed by a terminal run error, and the current client/gateway source shows `{ expectFinal: false }` resolves before that later error can set `sessionError`.

## Solution Assessment

No. Accepted-only waiting fixes the long-wait symptom, but returning success immediately is not the best fix because it loses the existing fallback for terminal post-accept failures.
