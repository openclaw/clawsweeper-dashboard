## Summary

Keep open: the timeout fix is useful, but this exact patch can now report success after only the accepted ack and skip the existing direct fallback if the resumed run later fails during deferred dispatch.

## What This Changes

The PR changes exec approval followup session resumes to accepted-only waiting and updates focused tests to expect `{ expectFinal: false }`.

## Best Possible Solution

Keep the non-blocking accepted path, but add a bounded run observer that routes terminal post-accept failures through the existing direct fallback.

## Reproduction Assessment

Yes, at source level. A focused test can mock an accepted `agent` response followed by a terminal run error, and current gateway/client source shows `{ expectFinal: false }` resolves before that terminal error can set `sessionError`.

## Solution Assessment

No. Accepted-only semantics fixes the long wait symptom, but the patch should preserve direct fallback on terminal setup or dispatch failures after acceptance.

## Review Findings

Overall correctness: patch is incorrect
