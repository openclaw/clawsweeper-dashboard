## Summary

Keep open: the PR targets a real exec-followup timeout, but the current patch treats the accepted ack as complete success and skips the existing direct fallback for terminal failures that can arrive after acceptance.

## What This Changes

The PR changes exec approval followup session resumes from final-result waits to accepted-only waits and updates focused tests to expect `{ expectFinal: false }`.

## Best Possible Solution

Keep the non-blocking accepted path, capture the accepted run id, and use a bounded terminal observer so setup or dispatch failures still route through the existing direct fallback without reintroducing a full final-turn wait.

## Reproduction Assessment

Yes, at source level. The client resolves accepted frames immediately with `{ expectFinal: false }`, while the gateway can emit a terminal error after the accepted ack from deferred dispatch.

## Solution Assessment

No. Accepted-only waiting addresses the long-wait symptom, but the patch should also preserve the fallback path for post-accept terminal failures.
