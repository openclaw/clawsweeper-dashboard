## Summary

Keep this PR open, but it still needs a narrow repair before merge: the accepted-only wait fixes the long followup timeout symptom, but the current patch can skip the existing direct fallback when the accepted run later fails during deferred setup or dispatch.

## What This Changes

The PR changes exec approval followup session resumes from final-result waits to accepted-only waits and adds focused regression coverage.

## Best Possible Solution

Keep the accepted-only resume path, capture the accepted run id, and add a bounded terminal observer so post-accept setup or dispatch failures still use the existing direct fallback.

## Reproduction Assessment

Yes, at source level. With `{ expectFinal: false }`, the client resolves the accepted frame immediately, while the gateway can still emit a terminal error for the same run after deferred dispatch.

## Solution Assessment

No. The proposed one-line wait change fixes the timeout symptom, but it should also preserve fallback delivery for terminal failures emitted after the accepted ack.
