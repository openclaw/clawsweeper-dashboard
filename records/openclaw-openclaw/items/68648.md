## Summary

Keep open: the PR targets a real exec approval followup timeout path, but the current diff is incomplete because accepted-only resolution can skip the existing direct fallback when the resumed run fails after the accepted ack.

## What This Changes

The PR changes exec approval followup session resumes to use accepted-only gateway waits and updates focused tests to assert `expectFinal: false`.

## Best Possible Solution

Keep the accepted-only direction, but capture enough accepted-run state to observe short post-accept terminal failures and invoke the existing direct fallback when the resumed run fails.

## Reproduction Assessment

Yes, at source level. Current main waits with `expectFinal: true`, and the gateway client/server contracts show that `expectFinal: false` resolves on accepted before deferred setup or dispatch can still fail.

## Solution Assessment

No. Accepted-only waiting is the right direction for the freeze, but this patch is incomplete because it drops fallback behavior for terminal failures emitted after the accepted ack.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

The gateway can still emit a terminal error for the same run from deferred setup or dispatch, so the existing direct fallback is skipped and the exec completion followup can be dropped.
