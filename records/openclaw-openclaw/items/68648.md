## Summary

Keep open: the timeout fix is useful, but the patch can now report success after the initial accepted frame and skip the existing direct fallback if the gateway later emits a terminal error for that same followup run.

## What This Changes

The branch switches exec approval followup session resumes from final-result waiting to accepted-only waiting and updates focused tests.

## Best Possible Solution

Keep the accepted-only resume path, then observe the accepted `runId` with a short bounded wait and route terminal post-accept failures through the existing direct fallback.

## Reproduction Assessment

Yes, at source level. Current gateway/client code resolves accepted-only calls before terminal run errors, and a focused mock test can cover an accepted `agent` response followed by an `agent.wait` error outcome.

## Solution Assessment

No. Accepted-only waiting addresses the 60s blocking symptom, but the patch should also preserve fallback behavior for setup or dispatch failures that arrive after acceptance.

## Review Findings

Overall correctness: patch is incorrect
