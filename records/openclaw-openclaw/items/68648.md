## Summary

Keep open: the PR targets a real exec-followup timeout problem, but the exact diff is unsafe because accepted-only gateway resolution can skip the direct fallback when the resumed agent run fails after the accepted ack.

## What This Changes

The PR switches exec approval followup agent resumes from waiting for the final result to accepted-only behavior and updates focused tests.

## Best Possible Solution

Use a bounded, non-blocking completion observer, such as `agent.wait` after acceptance, so session followups avoid the 60s full-turn wait while still falling back on terminal failures.

## Reproduction Assessment

Yes, at source level. A focused test can mock an accepted gateway response followed by a terminal error, and current gateway/client source shows `expectFinal: false` resolves on the accepted frame before fallback can run.

## Solution Assessment

No. Accepted-only semantics addresses the timeout symptom, but the maintainable fix needs a bounded observer or equivalent path that preserves direct fallback on post-accept failures.

## Review Findings

Overall correctness: patch is incorrect
