## Summary

Keep open: the accepted-only direction is plausible for the timeout symptom, but this PR currently drops the existing fallback path for failures that happen after the gateway has already accepted the resumed agent run, and the PR still lacks inspectable real behavior proof.

## What This Changes

The PR changes exec approval followup session resumes to use accepted-only gateway waits and updates focused tests to expect `expectFinal: false`.

## Best Possible Solution

Keep the accepted-only wait only if the followup path also preserves a bounded post-accept failure or fallback signal, with regression coverage and inspectable runtime proof.

## Reproduction Assessment

Yes, at source level. Current main waits for final results, while the PR would resolve on the accepted frame even though the gateway can still emit a later terminal error for the same run.

## Solution Assessment

No. Switching to accepted-only waiting addresses the timeout symptom, but this diff is incomplete because deferred terminal errors no longer reach the fallback path.
