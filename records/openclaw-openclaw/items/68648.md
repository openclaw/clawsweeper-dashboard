## Summary

Keep this PR open, but it still needs a code change before merge: the timeout fix is useful, yet the exact diff can skip the existing direct fallback when the resumed agent run fails after the accepted ack.

## What This Changes

The PR changes exec approval followup session resumes to wait for gateway `agent` acceptance instead of the final resumed turn, with focused tests updated for accepted-only behavior.

## Best Possible Solution

Keep the accepted-only resume ack, then add a bounded post-accept run observer that routes terminal setup or dispatch failures through the existing direct fallback.

## Reproduction Assessment

Yes, at source level. A focused test can mock an accepted `agent` response with a run id followed by a terminal run error, and current gateway client/server source shows accepted-only resolution happens before deferred dispatch errors are observable.

## Solution Assessment

No, not as written. Accepted-only waiting is the right direction for the timeout symptom, but the patch should preserve direct fallback for terminal post-accept setup or dispatch failures.

## Review Findings
