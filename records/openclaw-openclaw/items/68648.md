## Summary

Keep open because the branch addresses a real timeout symptom, but the current patch has a blocking regression: accepted-only success can return before post-accept agent errors trigger the existing direct fallback.

## What This Changes

The PR changes exec approval followup session resumes to accepted-only waiting and updates focused tests to expect `{ expectFinal: false }`.

## Best Possible Solution

Keep the non-blocking accepted path, but add a bounded run observer that routes terminal post-accept failures through the existing direct fallback.

## Reproduction Assessment

Yes, at source level. A focused test can mock an accepted `agent` response followed by a terminal run error, and current gateway/client source shows `{ expectFinal: false }` resolves before that terminal error can set `sessionError`.

## Solution Assessment

No. Accepted-only semantics fixes the long wait symptom, but the patch should preserve direct fallback on terminal setup/dispatch failures after acceptance.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve fallback on post-accept followup failures:** `src/agents/bash-tools.exec-approval-followup.ts:235`

The gateway can still emit a terminal error for the same run from deferred setup/dispatch, so the existing direct fallback is skipped and the exec completion followup can be dropped.
