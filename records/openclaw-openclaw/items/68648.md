## Summary

Keep open: this PR targets a real source-proven exec followup timeout path, but the proposed accepted-only wait would skip the existing fallback when the gateway emits a terminal failure after the accepted ack.

## What This Changes

The branch changes exec approval followup session resumes from `expectFinal: true` to `expectFinal: false` and updates focused tests to assert accepted-only behavior.

## Best Possible Solution

Preserve the non-blocking accepted path, capture the returned run id, and use a short bounded run observer so immediate terminal setup or dispatch failures still route through the existing direct fallback.

## Reproduction Assessment

Yes, at source level. Current main still waits for the final session result, and the PR's accepted-only change can be checked against the gateway client/server contracts that resolve accepted frames before deferred dispatch can fail.

## Solution Assessment

No. Accepted-only waiting addresses the long-wait symptom, but the patch should also preserve fallback behavior for terminal failures emitted after the accepted ack.
