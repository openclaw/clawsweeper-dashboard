## Summary

This PR is a focused fix for an exec approval followup timeout, but the current diff still introduces a correctness regression: it treats the initial accepted frame as final success and can skip the existing direct fallback if the resumed agent run later fails. The prior ClawSweeper and inline review identified the same issue, and current source still supports that finding.

## What This Changes

The PR changes exec approval followup session resumes from final-result waiting to accepted-only waiting and updates focused tests to expect `{ expectFinal: false }`.

## Best Possible Solution

Keep the non-blocking accepted path, but add a bounded run observer that routes terminal post-accept failures through the existing direct fallback.

## Reproduction Assessment

Yes, at source level. A focused test can mock an accepted `agent` response followed by a terminal run error, and current gateway/client source shows `{ expectFinal: false }` resolves before that terminal error can set `sessionError`.

## Solution Assessment

No. Accepted-only waiting fixes the long wait symptom, but the patch should preserve direct fallback on terminal setup or dispatch failures after acceptance.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve fallback on post-accept followup failures:** `src/agents/bash-tools.exec-approval-followup.ts:235`

The gateway can still emit a terminal error for the same run from deferred setup or dispatch, so the existing direct fallback is skipped and the exec completion followup can be dropped.
