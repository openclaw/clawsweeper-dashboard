## Summary

Keep this PR open, but it is not merge-ready: current main does not implement the requested rate-limit headroom output, while the PR still has a compile blocker, route-parser gap, missing real-provider proof, and a credentialed raw-fetch design issue.

## What This Changes

The PR adds an opt-in `--rate-limits` flag, rate-limit header parsing/types, RPM/TPM probe table columns, JSON `rateLimit` output, and parser/formatter tests for `models status --probe`.

## Best Possible Solution

Expose rate-limit headroom through the existing model probe/provider transport and auth contracts, wire the flag through both routed and Commander CLI paths, and require redacted real-provider proof for table plus JSON output before merge.

## Reproduction Assessment

Yes for the PR defects: source inspection shows unresolved helper references, the route-first parser drops the new flag, and the follow-up request uses raw provider fetch. The linked request itself is a new feature, not a broken current-main behavior.

## Solution Assessment

No. The feature direction is useful, but this implementation should reuse OpenClaw's routed CLI, provider transport, and auth contracts instead of adding a parallel credentialed fetch path.
