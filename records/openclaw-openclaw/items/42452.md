## Summary

Keep open: current main still does not expose provider rate-limit headroom, and this PR remains the canonical implementation thread, but it cannot merge as-is because it lacks real behavior proof and bypasses existing CLI, auth, and provider-transport contracts.

## What This Changes

The PR adds an opt-in `--rate-limits` flag to `models status --probe`, parses provider rate-limit headers, and displays RPM/TPM headroom in CLI and JSON probe output.

## Best Possible Solution

Expose rate-limit headroom through the existing model probe/provider transport and auth contracts, carry the flag through both routed and Commander CLI paths, and prove table plus JSON output with redacted real provider output.

## Reproduction Assessment

Yes for the PR defects: source inspection shows the route-first parser drops the new flag, the follow-up request uses raw fetch, and custom auth/config lookup can diverge from the main probe path. The underlying request is a new feature, not a broken current-main behavior.

## Solution Assessment

No. The feature direction is useful, but this patch should be reworked to reuse OpenClaw's routed CLI, credential, and provider-transport contracts instead of adding a parallel request path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Forward `--rate-limits` through the routed parser:** `src/cli/models-cli.ts:91`
  - body: Normal startup can route `models status` through `parseModelsStatusRouteArgs` before Commander. This patch adds the flag only to the Commander path, so routed `openclaw models status --probe --rate-limits` reaches `modelsStatusCommand` without `rateLimits` and produces no RPM/TPM output.
  - confidence: 0.9
- **[P2] Use the provider transport for header probes:** `src/commands/models/list.probe.ts:617-622`
  - body: `probeRateLimits` posts credentials with raw `fetch` to locally constructed URLs. That bypasses configured request headers/auth overrides, proxy/TLS/private-network policy, guarded fetch behavior, local service leases, transport timeouts, and diagnostics used by the successful main probe.
  - confidence: 0.88
- **[P2] Resolve custom provider configs by normalized key:** `src/commands/models/list.probe.ts:570-575`
  - body: `resolveCustomBaseUrl` checks only exact and already-normalized keys. A config stored under a key like `OpenAI` can make the main probe use the custom endpoint while the rate-limit follow-up falls back to the default host and reports `-`.
  - confidence: 0.84
- **[P2] Resolve probe credentials through the usable auth path:** `src/commands/models/list.probe.ts:558-564`
  - body: The follow-up resolver calls `resolveEnvApiKey` without the same config/workspace context and falls back to raw `getCustomProviderApiKey`, which can return env SecretRef marker names instead of resolved values. A target can pass the main probe but return no rate-limit data or send the marker as a key.
  - confidence: 0.84
