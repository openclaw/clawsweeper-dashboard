## Summary

Keep open. This PR targets a valid Matrix feature and builds on the merged structured approval metadata pattern, but the current head is still not mergeable: normal Matrix delivery will not reach the new presentation hooks, Matrix extra content is forwarded too broadly, media reply fanout regresses single-use reply semantics, and docs/changelog/dependency metadata need alignment.

## What This Changes

Adds Matrix outbound MessagePresentation rendering with `com.openclaw.presentation` event content, text fallback, outbound tests/docs, and a root Matrix SDK mirror dependency check.

## Best Possible Solution

Land a repaired Matrix plugin implementation that attaches validated `com.openclaw.presentation` through the registered outbound facade with text fallback, correct reply fanout, discoverable docs, changelog coverage, and synced package metadata.

## Reproduction Assessment

Yes. Source inspection shows core delivery depends on facade-level `renderPresentation` and `sendPayload`, while the PR head only adds those hooks to the Matrix runtime adapter; the remaining media reply and extra-content issues are also visible directly in the PR source.

## Solution Assessment

No. The feature direction matches the existing MessagePresentation and Matrix approval metadata contracts, but the implementation should use the registered facade/forwarder path, whitelist the Matrix metadata key, reuse reply fanout helpers, and align docs/changelog/package state before merge.
