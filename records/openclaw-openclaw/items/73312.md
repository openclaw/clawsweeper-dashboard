## Summary

Keep open. The PR is directionally useful and builds on the merged Matrix approval metadata work, but the current patch is not mergeable because normal Matrix delivery still cannot see the new hooks, the Matrix extra-content path is too broad, docs/changelog/package-lock work remains, and GitHub reports the branch as dirty.

## What This Changes

The PR adds Matrix MessagePresentation rendering and sendPayload handling, outbound tests, a Matrix presentation docs page, and a root matrix-js-sdk runtime mirror dependency/check.

## Best Possible Solution

Revise this PR after rebase so the registered Matrix outbound facade exposes presentation/sendPayload hooks, Matrix metadata is constrained to validated presentation content, and docs navigation, changelog, lockfile, and dependency versions are aligned.

## Reproduction Assessment

Yes. A high-confidence source trace shows core delivery calls renderPresentation/sendPayload only from the registered outbound adapter, while the current Matrix facade delegates only sendText, sendMedia, and sendPoll and this PR does not change that facade.

## Solution Assessment

No. The metadata format and fallback direction match the existing Matrix approval metadata pattern, but the current implementation is not the narrowest mergeable fix until the facade path, extra-content validation, docs/changelog, and dependency mirror are corrected.
