## Summary

Keep open: the Matrix metadata direction matches the existing structured approval pattern, but source inspection shows normal Matrix delivery still cannot reach the new payload hooks and the patch still forwards unvalidated Matrix event content.

## What This Changes

The branch adds Matrix outbound MessagePresentation rendering, `com.openclaw.presentation` extra content, targeted outbound tests, docs, and a root Matrix SDK mirror check.

## Best Possible Solution

Keep the feature direction, but merge only after the Matrix facade forwards presentation payload delivery, Matrix extra content is narrowed to validated presentation metadata, docs/changelog/deps are synchronized, and redacted real Matrix event or client proof is added.

## Reproduction Assessment

Yes. Source inspection shows core delivery reads `renderPresentation` and `sendPayload` from the registered Matrix channel facade, while the PR only adds those hooks to the runtime outbound adapter.

## Solution Assessment

No. The metadata format and fallback approach are consistent with the existing Matrix approval metadata pattern, but this patch is not the narrowest maintainable solution until facade forwarding, event-content validation, docs/changelog/dependency alignment, and real behavior proof are handled.
