## Summary

This PR is useful but not merge-ready: the Matrix metadata direction matches the existing custom Matrix event-content pattern, but the current diff still misses the registered facade path and leaves event-content, reply fanout, docs, changelog, and dependency alignment issues.

## What This Changes

The PR adds Matrix MessagePresentation rendering and docs, attaches `com.openclaw.presentation` metadata to outbound Matrix payload sends, expands Matrix outbound tests, and mirrors `matrix-js-sdk` at the root package level.

## Best Possible Solution

Keep the Matrix metadata design, wire the new outbound hooks through the registered Matrix facade, narrow Matrix extra content to validated presentation metadata, preserve single-use reply fanout semantics, and finish docs, changelog, and dependency alignment before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: normal Matrix delivery uses the registered channel facade, and current main forwards only text, media, and poll methods there while the PR adds the new hooks only to `matrixOutbound`.

## Solution Assessment

No. The metadata format and fallback approach are consistent with the existing Matrix approval metadata pattern, but this patch is not the best mergeable form until facade wiring, content narrowing, reply fanout, docs, changelog, and package alignment are fixed.
