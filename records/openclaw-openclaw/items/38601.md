The issue remains valid and is paired with an open fork PR, so it should stay open pending that implementation path.

## What This Changes

Add a user-visible warning or error when sandbox execution ignores an explicit `pty: true` request.

## Best Possible Solution

Refresh or adopt the focused warning patch against the current exec pipeline, add a regression test proving the warning appears in sandbox output, and document the current limitation without weakening sandbox isolation.

Yes, at source level with high confidence: select sandbox execution and pass `pty: true`; current main forces `usePty` to false and adds no corresponding warning. A live runtime reproduction was not run because this review was required to remain read-only.

## Solution Assessment

Yes in direction: an informational warning through the existing warnings path is the narrowest maintainable fix and preserves sandbox behavior. The patch should be refreshed onto the current refactored module and include focused output coverage.
