## Summary

Keep open: current main and the latest release still turn sandboxed `pty: true` into `usePty=false` without a sandbox-specific warning. The same-repo fix attempts are closed unmerged, while the open fork PR is a candidate for maintainer review rather than proof that the issue is fixed.

## What This Changes

The issue requests warning or error feedback when `exec` is called with `pty: true` while sandbox mode forces PTY off.

## Best Possible Solution

Keep PTY disabled under sandbox unless maintainers intentionally choose broader sandbox PTY support, but emit a clear warning through the existing exec warning path and document the sandbox limitation.

## Reproduction Assessment

Yes from source inspection: current main accepts `pty` but computes `usePty` as false whenever sandbox is active, and I did not find a sandbox-specific warning before `runExecProcess`. I did not run a live sandbox repro because this review is read-only.

## Solution Assessment

Yes for a warning-only repair: it is the narrowest maintainable fix for this issue because it explains the ignored explicit request without weakening the sandbox boundary. Turning this into a hard error or enabling sandbox PTY would be a compatibility/security choice for maintainers.
