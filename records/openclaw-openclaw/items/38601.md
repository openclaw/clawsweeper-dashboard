## Summary

Keep open: current main and v2026.6.10 still suppress PTY for sandboxed exec without a sandbox-specific warning. The prior same-repo fix attempts are closed unmerged, and the open fork PR is a candidate fix rather than proof that the issue is resolved.

## What This Changes

The issue asks OpenClaw to warn or error when an exec call explicitly requests `pty: true` but sandbox routing disables PTY.

## Best Possible Solution

Keep sandbox PTY disabled unless maintainers intentionally approve broader sandbox PTY support, but emit a clear warning through the existing exec warning path and document the limitation.

## Reproduction Assessment

Yes from source inspection. Current main still disables PTY with `params.pty === true && !sandbox`, and no inspected warning branch covers that exact downgrade path.

## Solution Assessment

Yes for a warning-only repair. Adding an informational warning is the narrowest maintainable fix because it explains the ignored explicit request without weakening the sandbox boundary; throwing or enabling sandbox PTY would need maintainer product/security approval.
