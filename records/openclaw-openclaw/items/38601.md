## Summary

Keep open: current main and v2026.6.10 still disable `pty: true` when sandbox execution is active without adding a sandbox-specific warning, and the only open closing fix path is an external fork PR that still needs maintainer review or replacement.

## What This Changes

The issue requests a warning or error when `exec` is called with `pty: true` in a sandboxed session and PTY is disabled.

## Best Possible Solution

Keep PTY disabled under sandbox unless maintainers intentionally approve broader sandbox PTY support, but emit a clear warning through the existing exec warning path and document the sandbox limitation.

## Reproduction Assessment

Yes, by source inspection: current main accepts `pty` but computes `usePty=false` whenever sandbox execution is active, and no sandbox-specific warning branch appears before `runExecProcess`. I did not run a live sandbox command because this review is read-only.

## Solution Assessment

Yes for a warning-only repair: it explains the ignored explicit request without weakening the sandbox boundary. Turning sandboxed PTY into a hard error or enabling PTY inside sandbox would be a broader compatibility and security decision.
