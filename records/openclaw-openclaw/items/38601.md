## Summary

Keep open: current main and v2026.6.6 still force `pty:true` off under sandbox without adding a sandbox-specific warning, and the same-repo fix attempts are closed unmerged while the remaining open closing PR is in a fork.

## What This Changes

Request to warn users when `exec` is called with `pty:true` but sandbox execution forces PTY off, and to clarify that limitation in docs.

## Best Possible Solution

Keep sandbox PTY disabled, emit a clear warning through the existing exec warning channel when `pty:true` is requested under sandbox, cover that path with a focused test, and update the exec docs.

## Reproduction Assessment

Yes, at source level. A sandboxed `exec` call with `pty:true` computes `usePty=false`, passes that into execution, and never pushes a sandbox-specific warning before result rendering.

## Solution Assessment

Yes. Adding an informational warning through the existing `warnings` path is the narrowest maintainable fix; throwing would be more disruptive, and a global PTY default is tracked separately as a feature direction.
