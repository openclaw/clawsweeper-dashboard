## Summary

Keep open: current main and v2026.6.5 still force explicit PTY requests off under sandbox without adding a warning, and the open fork PR at https://github.com/BingqingLyu/openclaw/pull/102 remains the active implementation path rather than a reason to close this issue early.

## What This Changes

The issue requests user-visible feedback when `exec` receives `pty: true` but sandbox execution prevents PTY use.

## Best Possible Solution

Refresh or adopt the focused warning patch in the current exec pipeline, add regression coverage for sandbox `pty: true`, and document the current limitation without weakening sandbox isolation.

## Reproduction Assessment

Yes, at source level with high confidence: sandbox execution plus `pty: true` reaches `usePty = false` with no corresponding warning added. I did not run a live exec command because this review was constrained to read-only inspection.

## Solution Assessment

Yes, the best fix direction is an informational warning through the existing `warnings` path, because it preserves sandbox behavior while explaining the ignored explicit request. Throwing would be a larger compatibility change and is not needed to solve the reported silence.
