Keep open: current main and the latest release still suppress a requested PTY under sandbox without adding a sandbox-specific warning, and the same-repo fix attempts are closed unmerged. The remaining work is narrow, but it sits on a security-sensitive exec/sandbox boundary and should get maintainer review before repair.

## What This Changes

The issue asks for `exec` to warn or error when `pty:true` is requested but sandbox mode disables PTY.

## Best Possible Solution

Keep sandbox PTY disabled, emit a clear warning through the existing exec warning channel when `pty:true` is requested under sandbox, cover that path with a focused test, and update the exec docs.

## Reproduction Assessment

Yes, at source level. A sandboxed exec call with `pty:true` computes `usePty=false`, passes that into execution, and current source has no sandbox-specific warning branch.

## Solution Assessment

Yes. Adding an informational warning through the existing `warnings` path is the narrowest maintainable fix; throwing would be more disruptive, and a global PTY default is tracked separately as a feature direction.
