## Summary

Keep open: current main and the latest release still disable PTY under sandbox by computing PTY from `params.pty === true && !sandbox` without adding a user-visible warning, and the linked warning-fix attempts are not merged into the target repository.

## What This Changes

The issue requests a warning or error when an exec call explicitly sets `pty: true` but sandbox mode prevents PTY use.

## Best Possible Solution

Keep sandbox PTY disabled, emit a clear exec warning when `pty: true` is requested under sandbox, add focused regression coverage, and update the exec docs caveat.

## Reproduction Assessment

Yes, at source level. Current main has a direct source path where `pty: true` is ignored under sandbox and no warning is pushed before the result is rendered.

## Solution Assessment

Yes for a warning-only fix. Throwing would be a stronger compatibility change, and enabling PTY in sandbox should remain out of scope unless maintainers make a separate security decision.
