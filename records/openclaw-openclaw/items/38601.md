## Summary

Keep this open. Current main still suppresses an explicit `pty:true` request when exec resolves to sandbox without adding a sandbox-specific warning, and https://github.com/openclaw/openclaw/pull/39349 remains an open closing PR for the narrow fix.

## What This Changes

The issue asks exec to warn or error when `pty:true` is requested but sandbox routing disables PTY.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/39349 or an equivalent narrow patch that emits a sandbox PTY warning, adds focused regression coverage, and documents the limitation without enabling PTY inside sandbox execution.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with a sandbox runtime selected by `host=auto` or `host=sandbox`, pass `pty:true`; current main computes `usePty` as false and has no sandbox-specific warning branch.

## Solution Assessment

Yes. A warning plus focused docs/test coverage is the narrowest maintainable fix because it makes the ignored explicit request visible without changing the sandbox security boundary.
