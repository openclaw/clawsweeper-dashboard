## Summary

Current main still silently suppresses an explicit PTY request in sandbox mode. The issue should remain open because the linked implementation PR is open and unmerged, and the sandbox-boundary impact requires maintainer review rather than a competing automated fix.

## What This Changes

Warn users when sandbox execution suppresses an explicitly requested PTY.

## Best Possible Solution

Preserve the sandbox PTY prohibition, add a clear warning through the canonical exec warnings channel when `pty: true` is denied, cover sandbox and non-sandbox calls at the tool boundary, and document the limitation in the public exec page.

## Reproduction Assessment

Yes at source level with high confidence: current main explicitly combines `pty: true` with `!sandbox` and has no warning on the sandbox-denied branch. This read-only review did not execute a live sandbox session.

## Solution Assessment

Yes in direction: using the existing warning channel is the narrowest maintainable fix and avoids weakening the sandbox or converting a compatible command into a hard failure. The open patch should be rebased onto the current pipeline and paired with tool-level regression coverage.
