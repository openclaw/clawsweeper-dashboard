## Summary

## What This Changes

The issue asks OpenClaw to warn or error when `exec` receives `pty: true` while sandbox mode prevents PTY execution.

## Best Possible Solution

Keep sandbox PTY policy explicit: preserve the sandbox boundary, add a clear warning or intentional error when explicit PTY cannot be honored, and cover the behavior in focused tests plus the exec docs.

## Reproduction Assessment

Yes. Source inspection shows that a sandboxed exec call with `pty: true` reaches `usePty=false` without adding a sandbox-specific warning; I did not run a live sandbox command because this review is read-only.

## Solution Assessment
