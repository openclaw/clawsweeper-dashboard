## Summary

## What This Changes

The issue requests a user-visible warning or error when the exec tool receives `pty: true` but sandbox mode prevents PTY execution.

## Best Possible Solution

Keep this issue open until a narrow fix merges that preserves sandbox PTY disablement, emits a clear exec warning when an explicit PTY request is ignored, adds focused regression coverage, and updates the public exec docs.

## Reproduction Assessment

Yes by source inspection. Current `main` accepts `pty`, computes `usePty` as false whenever sandboxing is active, and the checked runtime/docs path does not surface a sandbox-specific warning for that explicit request.

## Solution Assessment

Yes for a warning-only repair. Adding an informational warning through the existing exec warnings path is the narrowest maintainable solution for this issue; turning it into a hard error or enabling sandbox PTY would be a separate compatibility/security decision.
