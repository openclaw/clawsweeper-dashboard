This should stay open until that PR, or an equivalent narrow fix, lands.

## What This Changes

Request a warning or error when exec `pty:true` is ignored because sandbox execution disables PTY.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/39349 or an equivalent warning-only patch with focused regression coverage and real sandbox `pty:true` proof, while keeping sandbox PTY disabled.

## Reproduction Assessment

Yes by source inspection, not by live command. Current main and v2026.5.7 compute `usePty` as `params.pty === true && !sandbox`, so `pty:true` becomes false under sandbox and no sandbox-specific warning branch is present.

## Solution Assessment

Yes, the warning-only direction is the narrowest maintainable fix because it explains the ignored explicit request without changing the sandbox security boundary. The linked PR still needs proof and coverage before the issue should close.
