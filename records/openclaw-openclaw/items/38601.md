## Summary

Keep open: current main still suppresses an explicit `pty:true` request when exec resolves to sandbox without a sandbox-specific warning, and an open closing PR already owns the narrow fix path.

## What This Changes

Warn users when exec `pty:true` is ignored because the selected exec host is sandboxed.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/39349 or an equivalent narrow patch that emits the sandbox PTY warning, adds focused coverage, and documents the limitation without enabling PTY inside sandbox execution.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: with a sandbox runtime selected by `host=auto` or `host=sandbox`, pass `pty:true`; current main computes `usePty` as false and has no sandbox-specific warning branch.

## Solution Assessment

Yes. A warning plus focused docs/test coverage is the narrowest maintainable fix because it makes the ignored explicit request visible without changing the sandbox security boundary.
