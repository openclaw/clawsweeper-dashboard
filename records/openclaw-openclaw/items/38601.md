## Summary

Keep open: current main and the latest release still disable sandboxed `pty: true` by computing `usePty=false`, and I did not find a sandbox-specific warning before execution. The closed same-repo PRs were not merged, and the open fork PR is a candidate to review rather than a reason to close this issue.

## What This Changes

The issue requests warning or error feedback when `pty: true` is requested for the exec tool while sandbox mode disables PTY.

## Best Possible Solution

Keep PTY disabled under sandbox unless maintainers choose broader support, but emit a clear warning through the existing exec warning path and document the sandbox limitation.

## Reproduction Assessment

Yes from source inspection: current main accepts `pty` but computes `usePty` as false whenever sandbox is active, and I did not find a sandbox-specific warning before `runExecProcess`. I did not run a live sandbox repro because this review is read-only.

## Solution Assessment

Yes: a warning-only repair is the narrowest maintainable fix for this issue because it explains the ignored explicit request without weakening the sandbox boundary. Turning this into a hard error would be a compatibility-sensitive behavior change and should be a maintainer choice.
