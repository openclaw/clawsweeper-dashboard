## Summary

Keep open: current main and v2026.6.10 still force sandboxed `pty: true` to `usePty=false` without a sandbox-specific warning. The same-repo fix attempts are closed unmerged, while the open fork PR is only a candidate for maintainer review, not proof that this is fixed.

## What This Changes

The issue requests a warning or error when the exec tool receives `pty: true` but sandbox mode prevents PTY execution.

## Best Possible Solution

Keep PTY disabled under sandbox unless maintainers intentionally broaden sandbox PTY support, but emit a clear warning through the existing exec warning path and document the sandbox limitation.

## Reproduction Assessment

Yes. From source inspection, current main accepts `pty` but computes `usePty` as false whenever sandbox is active, and no sandbox-specific warning branch was found before `runExecProcess`; I did not run a live sandbox repro because this review is read-only.

## Solution Assessment

Yes for a warning-only repair. It is the narrowest maintainable fix because it explains the ignored explicit request without weakening the sandbox boundary; enabling sandbox PTY or throwing a hard error would be a broader compatibility/security choice.
