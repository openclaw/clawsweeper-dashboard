## What This Changes

The PR adds WhatsApp `messages.update` handling in `extensions/whatsapp/src/inbound/monitor.ts` to update buffered edited messages, notify on late edits, and ignore revocations.

## Best Possible Solution

Rebase onto current main and implement this inside the current WhatsApp inbound state model with separate buffered, in-flight, and processed edit states, full message-key scoping, focused tests, and redacted live WhatsApp proof; leave broader cancel/reprocess semantics to https://github.com/openclaw/openclaw/issues/55691.

I did not run live WhatsApp proof in this read-only review.

## Solution Assessment

No. The feature direction is plausible, but the current PR needs a current-main rebase, an explicit in-flight state, full-key scoping, and maintainer agreement on late-edit/revoke semantics before it is the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Track in-flight edits separately from buffered edits:** `extensions/whatsapp/src/inbound/monitor.ts:167`
  - body: `pendingMessageIds` is only cleared in the `finally` after `await options.onMessage(messageToDeliver)`, so an edit that arrives while the agent is already processing still takes the buffered path and only mutates `pending.body`. For batched flushes the delivered body has already been computed, so the agent can process stale text with no late-edit notice; move entries out of the buffered map before calling `onMessage` and handle that in-flight state explicitly.
  - confidence: 0.91
- **[P2] Scope edit state by the full WhatsApp key:** `extensions/whatsapp/src/inbound/monitor.ts:615`
  - body: Baileys sends `messages.update` with a full `WAMessageKey`, and current WhatsApp dedupe keys by account, `remoteJid`, and message id. This PR tracks pending and processed edits by `msgId` only, so a collision across chats can mutate or notify for the wrong original message; include at least the remote JID with the id before trusting the state hit.
  - confidence: 0.78
