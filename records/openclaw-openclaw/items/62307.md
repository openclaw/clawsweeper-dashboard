## Summary

Keep open. ## What This Changes

The branch modifies `extensions/whatsapp/src/inbound/monitor.ts` to listen for Baileys `messages.update`, replace buffered edited message bodies, notify on late edits, and ignore revocations.

## Best Possible Solution

Land a current-main WhatsApp plugin implementation that models buffered, in-flight, and processed message IDs separately, preserves current debounce/dedupe/close-drain behavior, adds focused timing/access-control tests, and includes redacted live WhatsApp proof for buffered edits, late edits, and revocations.

## Reproduction Assessment

Yes for the PR defect by source inspection: enqueue a debounced WhatsApp message, let `onFlush` compute and pass `messageToDeliver`, keep `onMessage` unresolved, then emit `messages.update`; the handler still sees the ID in `pendingMessageIds` even though mutating it no longer reliably changes the delivered payload. I did not run a live WhatsApp test in this read-only review.

## Solution Assessment

No, not as currently implemented. The feature belongs in the WhatsApp plugin, but the maintainable fix needs a current-main rebase plus distinct buffered, in-flight, and processed edit states rather than treating every pending ID as still safely mutable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Track in-flight edits separately:** `extensions/whatsapp/src/inbound/monitor.ts:615-619`
  - body: When `onFlush` is already awaiting `options.onMessage(messageToDeliver)`, the payload has been handed off to the agent, but this branch still leaves each ID in `pendingMessageIds`. A later `messages.update` event hits the buffered path and mutates `pending.body`; for batched messages the combined body was already computed, and for single messages the agent may already have read the original, so the user gets no late-edit notice while stale text can still be processed. Please move flushed IDs out of the buffered map before delivery and track an explicit in-flight state for the late-edit path.
  - confidence: 0.88
