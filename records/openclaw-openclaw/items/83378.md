## Summary

Keep open: the patch is a focused CLI bug fix, but it only rejects empty gateway text and still lets non-empty gateway error payloads return success, which can leave the linked usage-limit exit-code failure unresolved.

## What This Changes

The PR adds a gateway `model.run` empty-text check in `src/cli/capability-cli.ts` and a focused CLI regression test for empty gateway payloads.

## Best Possible Solution

Reject both empty gateway payloads and gateway payloads marked `isError: true`, while keeping the local transport behavior and CLI routing unchanged.

## Reproduction Assessment

Partially yes. Current-main source shows empty gateway payloads return success today, but the original ChatGPT-Team usage-limit path still needs live proof and the PR proof exercises a no-provider empty-payload path instead.

## Solution Assessment

No. The patch is the right general area, but the maintainable fix should also preserve and reject gateway `isError` payloads so provider failures with explanatory text do not exit 0.
