## Summary

Keep open. Current main still lets malformed per-agent session-store placeholder entries flow through `sessions.list`, and the only linked fix attempt is closed without merging.

## What This Changes

The issue asks for `sessions.list` to return only real session rows and filter store-level placeholder keys such as `agent:<id>:sessions`.

## Best Possible Solution

Filter non-listable per-agent store placeholder rows in the Gateway session listing path and add regression coverage for a placeholder key alongside a valid session.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a raw `sessions` key in an agent store canonicalizes to `agent:<id>:sessions`, passes current list filtering, and is emitted with `updatedAt: null` and no `sessionId`.

## Solution Assessment

Yes. A server-side listability guard before `buildGatewaySessionRow` is the narrowest maintainable repair; Control UI-only filtering would leave the RPC contract leaking malformed rows.
