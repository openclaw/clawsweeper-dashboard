## Summary

Keep open. Current main still has a direct source path for a malformed per-agent store placeholder to pass through `sessions.list`, and the only linked fix attempt was closed unmerged.

## What This Changes

The issue reports that `sessions.list` returns a phantom store-key row with no `sessionId` and `updatedAt: null`.

## Best Possible Solution

Add a server-side listability guard so only valid session entries with real `sessionId` and finite `updatedAt` are serialized, with regression coverage for both utility and Gateway RPC paths.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a raw `sessions` entry in a per-agent store canonicalizes to `agent:<id>:sessions`, passes current filters, and serializes with `updatedAt: null` and no `sessionId`.

## Solution Assessment

Yes. Filtering invalid entries on the server before row construction is the narrowest maintainable fix; Control UI filtering would leave the RPC contract leaking malformed rows.
