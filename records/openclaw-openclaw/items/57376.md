## Summary

Keep open. Current main and the latest tagged release still have a source path where a malformed per-agent store placeholder can flow into `sessions.list`, and the linked fix attempt at https://github.com/openclaw/openclaw/pull/57419 was closed unmerged.

## What This Changes

The issue reports that `sessions.list` can return a store-level placeholder row instead of only real sessions.

## Best Possible Solution

Add a server-side listability guard so only valid session entries with a non-empty `sessionId` and finite `updatedAt` are selected and serialized, with focused regression coverage for the utility and Gateway RPC paths.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a raw `sessions` object in a per-agent store canonicalizes to `agent:<id>:sessions`, passes current list filters, and serializes with `updatedAt: null` and no `sessionId`.

## Solution Assessment

Yes. Filtering invalid entries on the server before row construction is the narrowest maintainable fix; Control UI filtering would leave the RPC contract leaking malformed rows.
