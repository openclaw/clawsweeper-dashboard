## Summary

Keep open. Current main still has the reported source path: Gateway combines raw per-agent store entries, the session-list filter does not require real session metadata, and row construction can emit a malformed row; the linked fix PR is closed unmerged.

## What This Changes

The issue asks for `sessions.list` to stop returning store-level placeholder rows like `agent:<id>:sessions` with `updatedAt: null` and no `sessionId`.

## Best Possible Solution

Filter non-listable session-store placeholder rows in the Gateway session listing path and cover both utility and RPC behavior with focused regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a raw `sessions` entry in a per-agent store canonicalizes to `agent:<id>:sessions`, passes current filtering, and is serialized with `updatedAt: null` and no `sessionId`.

## Solution Assessment

Yes. A server-side listability guard before row construction is the narrowest maintainable fix; Control UI-only filtering would leave the RPC contract leaking malformed rows.
