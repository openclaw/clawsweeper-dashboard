## Summary

Keep open. Current main and the latest shipped tag still allow a malformed per-agent store placeholder to reach sessions.list row construction, and the only linked fix PR is closed unmerged.

## What This Changes

sessions.list should exclude store-level placeholder entries that have no real session identity or activity.

## Best Possible Solution

Add a Gateway-side listability guard before row construction so persisted placeholders without real session identity or activity are excluded from sessions.list, with regression coverage for both the utility path and RPC path.

## Reproduction Assessment

Yes, source-reproducible. A per-agent store row keyed sessions can be canonicalized to agent:<id>:sessions, accepted by the current list filters for that agent, and serialized as a row without sessionId.

## Solution Assessment

Yes. A server-side guard before row construction is the narrow maintainable fix; UI-only filtering would leave the RPC contract malformed, and filtering only null updatedAt would miss normalized placeholder variants.
