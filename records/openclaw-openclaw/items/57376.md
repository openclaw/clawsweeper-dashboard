## Summary

Keep open. Current main still has the source path that lets malformed session-store placeholder rows reach `sessions.list`, and the only linked implementation PR is closed without merging.

## What This Changes

`sessions.list` should omit store-level placeholder entries that have no real `sessionId` and no activity timestamp.

## Best Possible Solution

Add a server-side listable-session guard before row construction, with regression coverage for a placeholder plus a real session for the same agent.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a per-agent store containing a raw `sessions` object entry canonicalizes it to `agent:<id>:sessions`, list filtering keeps it, and row serialization emits `updatedAt: null` without `sessionId`.

## Solution Assessment

Yes. The narrow maintainable fix is server-side filtering or normalization before `buildGatewaySessionRow`; Control UI filtering would leave the Gateway API leaking malformed rows.
