## Summary

Keep open. Current main still allows a malformed per-agent store placeholder to flow through `sessions.list` as a row with `updatedAt: null` and no `sessionId`, and the directly linked fix PR is closed without merging.

## What This Changes

The issue reports that `sessions.list` includes an agent store placeholder key such as `agent:sean:sessions` instead of returning only real session rows.

## Best Possible Solution

Add a server-side validity guard before session row construction so non-session store records are excluded, and cover the placeholder-plus-real-session case in Gateway session-list tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a per-agent store containing a raw `sessions` object entry plus a real session canonicalizes the placeholder to `agent:<id>:sessions`, keeps it through list filtering, and serializes it with `updatedAt: null` and no `sessionId`.

## Solution Assessment

Yes. The narrow maintainable fix is server-side filtering or normalization before `buildGatewaySessionRow`; client-side Control UI filtering would leave the Gateway API leaking malformed rows.
