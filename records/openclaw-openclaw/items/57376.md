## Summary

Keep open. Current main still lets malformed session-store placeholder rows reach `sessions.list`, and the only directly linked implementation PR is closed without merging.

## What This Changes

`sessions.list` should exclude store-level placeholder rows such as `agent:<id>:sessions` that lack a real `sessionId` and activity timestamp.

## Best Possible Solution

Add a server-side validity guard before row construction so malformed non-session store rows are excluded, with regression coverage for a placeholder plus a real session for the same agent.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: a per-agent store containing a raw `sessions` object entry plus a real session canonicalizes the placeholder to `agent:<id>:sessions`, keeps it through list filtering, and serializes it with `updatedAt: null` and no `sessionId`. I did not run a test because this review is read-only.

## Solution Assessment

Yes. The narrow maintainable fix is server-side filtering or normalization before `buildGatewaySessionRow`; client-side Control UI filtering would leave the API contract leaking malformed rows.
