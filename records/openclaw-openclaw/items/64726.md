## Summary

Keep open. Batch embedding and exact-hash cache reuse are already present on current main and in the latest release, but the remaining fuzzy invalidation, hit/miss/latency metrics, cache warming policy, gradual model migration, and vector storage changes are still meaningful design work.

## What This Changes

The issue requests a smarter embedding cache with batched embedding calls, fuzzy invalidation, cache warming, cache metrics, gradual model migration, and vector storage optimization.

## Best Possible Solution

Keep this as a memory-cache design umbrella and split the remaining work into staged changes for metrics, fuzzy reuse policy, warming behavior, model/index migration, and storage optimization, coordinating provider/model migration with https://github.com/openclaw/openclaw/issues/63990.
