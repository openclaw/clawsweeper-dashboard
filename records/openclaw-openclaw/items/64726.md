## Summary

Keep open. Current main partially satisfies the request with batch embedding, exact-hash cache reuse, entry counting, and max-entry pruning, but source and docs still show no fuzzy cache reuse, hit/miss/latency metrics, cache warming policy, gradual model migration, or compressed vector storage.

## What This Changes

The issue requests embedding-cache improvements covering batch embedding, fuzzy invalidation/reuse, warming, metrics, model/provider migration, and vector storage optimization.

## Best Possible Solution

Keep the shipped batch/exact-cache behavior, then split the remainder into staged changes: metrics first, opt-in fuzzy reuse with safety tests, explicit warming policy, and schema-backed migration/storage updates.

Not applicable as a single bug reproduction; this is a feature and performance request. The remaining gaps are high-confidence from source inspection of exact-hash lookup, status fields, schema, and reindex decisions on current main.
