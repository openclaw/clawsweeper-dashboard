## Summary

Keep open. Current main covers batch embedding and exact-hash cache reuse, but fuzzy reuse, hit/miss/latency metrics, warming, gradual model migration, and compressed vector storage remain meaningful feature work needing maintainer design.

## What This Changes

Improve the memory embedding cache with batch embedding, smarter invalidation, warming, metrics, model-migration support, and storage pruning/compression.

## Best Possible Solution

Keep this as a design-backed memory-cache umbrella, then split it into staged work: metrics first, opt-in fuzzy reuse with correctness tests, explicit warming policy, and schema-backed migration/storage changes.
