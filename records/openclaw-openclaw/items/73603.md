## Summary

Keep open. Current main has in-process plan update events, subagent/task lifecycle tracking, diagnostics, and trajectory export surfaces, but it still does not provide the requested default-off durable plan/session audit stream with stable plan identifiers, spawn/yield correlation, and exactly one terminal close/abort record. This is also not a clean ClawHub close because the missing piece is a stable core event/API contract and persistence/redaction policy, not only an external auditor implementation.

## What This Changes

The issue requests a default-off config flag that emits plan and session lifecycle audit events to a durable JSONL event stream.

## Best Possible Solution

Define an additive experimental observability contract with stable plan/run identifiers, open/update records, child spawn/yield/end correlation, and exactly one terminal close/abort record. Keep it disabled by default, document schema/redaction/retention guarantees, wire config/help/docs, and add focused tests for disabled behavior plus enabled lifecycle completeness.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: the requested config key and durable JSONL sink are absent, plan events are update-only, and subagent/task lifecycle state is keyed by runId/session rather than a stable plan id.

## Solution Assessment

Unclear. A default-off JSONL stream is plausible, but maintainers should first decide the stable schema, privacy model, file/sink ownership, and whether this belongs in core diagnostics, task registry, trajectory export, or a plugin/API extension.
