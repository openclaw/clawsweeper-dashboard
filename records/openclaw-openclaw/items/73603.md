## Summary

Keep open. Current main has in-process plan, lifecycle, tool, trajectory, and plugin-adjacent event surfaces, but it still lacks the requested stable, redacted plan/session lifecycle audit sink with plan correlation and exactly-once close semantics.

## What This Changes

The issue requests an opt-in plan/session lifecycle event stream, proposed as `events.emit.plan` writing correlated plan, spawn, yield, and close records to `/var/log/openclaw-events.jsonl`.

## Best Possible Solution

Define a maintainer-approved, redacted plan/session lifecycle contract and persistence owner first, preferably by extending existing agent event subscriptions, diagnostics, trajectory, or plugin API seams before adding a global JSONL sink.

## Reproduction Assessment

Yes, at source level. Current main has in-process plan/lifecycle/tool events, trajectory exports, and plugin event subscriptions, but focused searches found no `events.emit.plan` config, `/var/log/openclaw-events.jsonl` writer, or `closes_plan` audit contract.

## Solution Assessment

No for the exact hard-coded `/var/log` plus new config shape without maintainer design. The safer path is to choose a redacted lifecycle schema and owner across the existing diagnostics, trajectory, and plugin-event surfaces before implementing persistence.
