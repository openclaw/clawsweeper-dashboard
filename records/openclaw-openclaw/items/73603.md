## Summary

Keep open. Current main still lacks the requested durable, default-off plan/session audit stream; the existing implementation has an in-process plan update stream, run-scoped lifecycle/task tracking, plugin event subscriptions, and trajectory export, but not stable plan IDs, spawn/yield plan correlation, terminal close/abort records, or the requested JSONL sink.

## What This Changes

Request a default-off `events.emit.plan` configuration path that writes plan and session lifecycle records to `/var/log/openclaw-events.jsonl` for external task-completion auditors.

## Best Possible Solution

Define an additive observability contract before implementation: stable plan/run identifiers, plan open/update records, subagent spawn/yield/end correlation, one terminal close/abort record, default-off configuration, redaction/file-permission/retention guarantees, public docs, and focused tests for both disabled and enabled behavior.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: the requested config keys and durable JSONL sink are absent, and the current lifecycle surfaces are update-only or run/task/session scoped rather than plan-correlated.

## Solution Assessment

Unclear. A default-off JSONL audit stream is plausible, but maintainers should first choose the durable schema, privacy model, file ownership, retention behavior, and whether the sink itself belongs in core or a plugin built on a new stable event contract.
