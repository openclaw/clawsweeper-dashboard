## Summary

Keep open. Current main has adjacent trajectory export, in-memory agent plan events, and plugin event subscriptions, but it still lacks the requested durable plan/session lifecycle audit stream with stable plan IDs, close correlation, and `/var/log/openclaw-events.jsonl` emission.

## What This Changes

The issue requests an opt-in `events.emit.plan` configuration that writes plan and session lifecycle events to a canonical JSONL audit stream.

## Best Possible Solution

Define a maintainer-approved lifecycle telemetry contract covering schema stability, privacy/redaction, storage and retention, plugin/core ownership, and tests for plan open/close plus subagent spawn/yield correlation.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Partially: source inspection on current main shows the requested config and canonical plan JSONL sink are absent, but I did not run a live gateway against `/var/log/openclaw-events.jsonl`.

## Solution Assessment

Is this the best way to solve the issue? Unclear: the operator need is valid, but a direct `/var/log` JSONL flag should wait for a maintainer decision on telemetry ownership, schema, privacy, and whether this belongs in core or a plugin/API extension.
