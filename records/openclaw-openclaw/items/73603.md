## Summary

Keep open. Current main still lacks the requested opt-in durable plan/session lifecycle stream to `/var/log/openclaw-events.jsonl`; current plan events, task ledger, and trajectory export are adjacent surfaces but do not provide the stable plan IDs, close records, config flag, or canonical JSONL sink requested here.

## What This Changes

The issue requests a default-off `events.emit.plan` option that writes plan/session lifecycle events to `/var/log/openclaw-events.jsonl` for operator audit tooling.

## Best Possible Solution

Define an additive experimental observability contract with stable plan/run identifiers, terminal outcomes, redaction guarantees, permissions/retention rules, docs, and focused disabled/enabled coverage.

## Reproduction Assessment

Not applicable as a broken-behavior reproduction; this is a feature/API request. Source inspection on current main gives high-confidence proof that the requested flag, stable plan IDs, terminal close records, and canonical JSONL sink are absent.

## Solution Assessment

Unclear. A default-off JSONL stream is plausible, but maintainers should first decide schema stability, privacy boundaries, sink ownership, and whether to build this on the task ledger, trajectory capture, plugin events, or a separate audit sink.
