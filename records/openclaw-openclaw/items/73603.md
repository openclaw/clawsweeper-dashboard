## Summary

Keep open. Current main still lacks the requested opt-in durable plan/session lifecycle stream to `/var/log/openclaw-events.jsonl`; the existing plan surfaces are update-only in-process events, and trajectory capture remains a separate per-session diagnostic sidecar.

## What This Changes

Request a default-off `events.emit.plan` config that writes plan/session lifecycle records to the canonical JSONL audit stream.

## Best Possible Solution

Define an additive experimental observability contract with stable plan/run identifiers, open/update/spawn/yield/end records, redaction guarantees, file permission and retention rules, config/docs coverage, and focused disabled/enabled tests.

## Reproduction Assessment

Yes for the capability gap: static inspection of current main shows the requested config key and durable JSONL sink are absent, while current plan events are update-only in-process records.

## Solution Assessment

Unclear. A default-off JSONL stream is plausible, but maintainers should first decide schema stability, privacy boundaries, sink ownership, and whether this belongs on trajectory capture, plugin events, or a separate audit sink.
