## Summary

Keep open. Current main has process-local plan/lifecycle/tool events and per-session trajectory capture, but it still lacks the requested stable, redacted plan/session lifecycle audit sink with plan correlation and exactly-once close semantics.

## What This Changes

The issue requests an opt-in plan/session lifecycle audit event stream written to `/var/log/openclaw-events.jsonl` with plan IDs, spawn/yield correlation, and terminal close semantics.

## Best Possible Solution

Define a maintainer-approved, redacted lifecycle audit contract and persistence owner, preferably by extending existing agent event, diagnostics, trajectory, or plugin/SDK seams instead of adding a hard-coded global JSONL sink.

## Reproduction Assessment

## Solution Assessment

No for the exact hard-coded `/var/log` plus new config shape as written. The safer solution is a maintainer-chosen event schema, redaction model, and persistence owner across existing diagnostics, trajectory, and plugin-event seams.
