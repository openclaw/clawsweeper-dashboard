## Summary

Keep open. Current main has adjacent diagnostics, tool execution events, trajectory sidecars, and plugin exporter surfaces, but it still does not provide the requested stable plan/session lifecycle audit contract with plan ids, spawn/yield correlation, and exactly-one close or timeout semantics.

## What This Changes

The issue requests an opt-in plan/session lifecycle event stream, written to a JSONL audit sink, so operators can detect abandoned Maestro plans and subagent work.

## Best Possible Solution

Define a maintainer-approved, redacted plan/session lifecycle event contract first, then decide whether persistence belongs in core diagnostics, trajectory, or an approved plugin/exporter sink.

## Solution Assessment

No, not as the exact hardcoded `/var/log` plus new config shape without maintainer design. The safer path is to add the missing redacted lifecycle contract to the approved diagnostics, trajectory, or plugin exporter surface before choosing a persistence sink.
