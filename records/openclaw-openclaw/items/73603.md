## Summary

Keep open. Current main still lacks the requested opt-in durable plan/session lifecycle stream to `/var/log/openclaw-events.jsonl`; existing plan events are update-only in-process records, and making them durable needs maintainer schema, privacy, retention, and sink-ownership decisions.

## What This Changes

Add a default-off `events.emit.plan` config flag that writes plan and session lifecycle audit records to `/var/log/openclaw-events.jsonl`.

## Best Possible Solution

Define an additive experimental audit contract with stable correlation IDs, explicit terminal states, redaction guarantees, file permission and retention rules, docs, and disabled/enabled regression coverage before wiring durable emission.

## Reproduction Assessment

Yes for the capability gap: source inspection of current main shows no requested config key or canonical durable event sink, while current plan events are update-only in-process records. This is a feature gap rather than a broken existing contract.

## Solution Assessment

Unclear. A default-off JSONL stream is plausible, but maintainers should first decide schema stability, privacy boundaries, sink ownership, and whether this belongs in trajectory capture, plugin events, or a separate audit sink.
