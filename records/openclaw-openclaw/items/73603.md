## Summary

Keep open. Current main still lacks the requested default-off durable plan/session audit stream; existing plan signals are in-process and update-only, while current lifecycle/task tracking is keyed by run and session rather than a stable plan correlation contract. The issue includes concrete operator use cases and a workaround, so this is not stale, duplicate, or safely movable to ClawHub without a new core/plugin observability contract.

## What This Changes

Request a default-off config flag that emits plan and session lifecycle records with plan/run correlation to `/var/log/openclaw-events.jsonl`.

## Best Possible Solution

Define an additive experimental observability contract with stable plan/run identifiers, explicit open/update/spawn/yield/terminal records, default-off config, redaction and retention rules, portable sink ownership, docs/config schema updates, and focused tests for disabled behavior plus enabled lifecycle completeness.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: the requested config keys and sink markers are absent, plan events are update-only, `update_plan` has no plan id, and lifecycle/task state is scoped by run/session rather than plan identity.

## Solution Assessment

Unclear. A default-off JSONL stream is a plausible direction, but the best implementation needs maintainer decisions about schema stability, privacy/redaction, file location and permissions, and whether the public surface belongs in diagnostics, trajectory export, task registry, or plugin API.
