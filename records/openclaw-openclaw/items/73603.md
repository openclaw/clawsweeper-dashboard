## Summary

Keep open. Current main still does not provide the requested default-off durable plan/session audit stream to `/var/log/openclaw-events.jsonl`; the existing plan event is an in-process update notification, and trajectory capture is per-session support telemetry rather than the canonical audit sink described in the report.

## What This Changes

Add an opt-in OpenClaw event stream that emits plan and session lifecycle records to `/var/log/openclaw-events.jsonl` for task-completion auditing.

## Best Possible Solution

Define an additive experimental observability contract with stable run/plan identifiers, open/update/spawn/yield/end records, redaction guarantees, file permissions/rotation, config/docs coverage, and focused disabled/enabled tests.

## Reproduction Assessment

Yes. Static inspection of current main reproduces the gap: the requested config key and JSONL sink are absent, plan events are update-only, and trajectory capture is per-session sidecar telemetry rather than `/var/log/openclaw-events.jsonl`.

## Solution Assessment

Unclear. A default-off JSONL stream is plausible, but the maintainable fix needs maintainer decisions on schema stability, privacy, sink ownership, and whether to build on agent events, trajectory capture, or plugin subscriptions.
