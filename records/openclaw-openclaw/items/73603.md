## Summary

Keep open. Current main still lacks the requested opt-in durable plan/session lifecycle stream to `/var/log/openclaw-events.jsonl`; the closest surfaces are update-only in-process plan events and separate per-session trajectory sidecars, so this remains a real feature/API gap that needs maintainer design.

## What This Changes

The issue requests a default-off config flag that emits plan/session lifecycle records with stable correlation ids to `/var/log/openclaw-events.jsonl`.

## Best Possible Solution

Define an additive experimental observability contract with stable plan/run identifiers, open/update/spawn/yield/end records, redaction guarantees, file permission and retention rules, config/docs coverage, and focused disabled/enabled tests.

## Reproduction Assessment

Not applicable as a broken-behavior reproduction; this is a feature/API request. Source inspection does give high-confidence proof of the capability gap: the requested config flag, stable plan ids, terminal close records, and canonical JSONL sink are absent on current main.

## Solution Assessment

Unclear. A default-off JSONL stream is plausible, but maintainers should first decide schema stability, privacy boundaries, sink ownership, and whether this belongs in trajectory capture, plugin events, task/subagent registries, or a separate audit sink.
