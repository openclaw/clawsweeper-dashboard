## Summary

Keep open. Current main still lacks the requested durable, default-off plan/session audit stream and the existing lifecycle surfaces are run/task/session scoped rather than a stable plan correlation contract.

## What This Changes

The issue requests a config-gated event stream that writes plan and session lifecycle records to `/var/log/openclaw-events.jsonl` for task-completion auditing.

## Best Possible Solution

Define an additive observability contract with stable plan/run identifiers, explicit plan open/update records, subagent spawn/yield/end correlation, and exactly one terminal close/abort record. Keep it disabled by default, document schema and redaction guarantees, wire config/help/docs, and add focused coverage for disabled behavior plus enabled lifecycle completeness.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: the requested config keys and durable JSONL sink are absent, and the current code only exposes update-only plan events plus run-scoped lifecycle/task tracking.

## Solution Assessment

Unclear. A default-off durable JSONL stream is plausible, but maintainers should choose the schema, privacy model, sink ownership, and core-versus-plugin/API boundary before implementation.
