## Summary

Keep open. Current main has in-process plan update events, plugin event subscriptions, and run/session/task lifecycle tracking, but it still lacks the requested default-off durable JSONL plan/session audit stream with stable plan identifiers, spawn/yield correlation, and terminal close/abort records.

## What This Changes

Requests a default-off `events.emit.plan` config that writes plan and subagent lifecycle records with plan/run correlation to `/var/log/openclaw-events.jsonl`.

## Best Possible Solution

Define an additive experimental observability contract that is disabled by default, emits stable plan/run identifiers plus open/update/spawn/yield/end/abort records, documents redaction and retention behavior, wires config/help/docs, and adds focused tests for disabled behavior and enabled lifecycle completeness.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: the requested config key and durable JSONL sink are absent, plan events are update-only, and subagent/task lifecycle state is keyed by run/session rather than a stable plan id.

## Solution Assessment

Unclear. A default-off JSONL stream is plausible, but maintainers should first choose the stable schema, privacy model, sink ownership, and core-versus-plugin boundary before implementation.
