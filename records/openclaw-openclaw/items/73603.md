## Summary

Keep open. Current main and the latest release still lack the requested durable plan/session audit stream, `/var/log/openclaw-events.jsonl` emission, stable plan IDs, and close-correlation contract; the existing trajectory, diagnostics, plugin event, and in-process plan event surfaces are adjacent but not equivalent.

## What This Changes

Request an opt-in config flag that emits plan/session lifecycle events to a durable JSONL event sink for task-completion audits.

## Best Possible Solution

Define one maintainer-approved lifecycle telemetry contract with stable plan/session IDs, close semantics, redaction/retention policy, storage/sink ownership, plugin/API exposure, and tests for plan update, spawn, yield, timeout/abort, and final close correlation.

## Reproduction Assessment

## Solution Assessment

Unclear: the operator need is valid, but the proposed direct JSONL flag is not the best confirmed shape until maintainers decide telemetry ownership, storage, security, schema stability, and relation to trajectory/diagnostics/process-event surfaces.
