## Summary

Keep open. Current main still does not provide the requested default-off durable plan/session lifecycle stream to `/var/log/openclaw-events.jsonl`; the existing plan events are in-process/update-only, subagent tracking is run-scoped, and trajectory capture is per-session support telemetry rather than the canonical operator audit sink requested here.

## What This Changes

The issue requests an opt-in `events.emit.plan` configuration path that emits stable plan and session lifecycle records to `/var/log/openclaw-events.jsonl`.

## Best Possible Solution

Define an additive experimental observability contract with stable plan/run identifiers, open/update/spawn/yield/end records, redaction guarantees, file permission and retention rules, config/docs coverage, and focused disabled/enabled tests.

## Reproduction Assessment

Yes. Static inspection of current main reproduces the gap: the requested config key and JSONL sink are absent, plan events are update-only, and subagent lifecycle records are run-scoped rather than plan-scoped.

## Solution Assessment

Unclear. A default-off JSONL stream is plausible, but the maintainable fix needs maintainer decisions on schema stability, privacy, sink ownership, and whether to build on trajectory capture, plugin event APIs, or a broader run-trace contract.
