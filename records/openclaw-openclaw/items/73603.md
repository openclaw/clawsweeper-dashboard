## Summary

Keep open. Current main still lacks the requested opt-in durable plan/session lifecycle stream: there is no `events.emit.plan` config key or `/var/log/openclaw-events.jsonl` writer, plan events are update-only in-process records, and trajectory capture remains a separate per-session diagnostic sidecar.

## What This Changes

Request an opt-in `events.emit.plan` flag that writes plan/session lifecycle events to `/var/log/openclaw-events.jsonl`.

## Best Possible Solution

Define an additive experimental observability contract with stable plan/run identifiers, open/update/spawn/yield/end records, redaction guarantees, file permissions, retention rules, config/docs coverage, and disabled/enabled tests.

## Reproduction Assessment

Yes for the capability gap: source inspection shows the requested config key and durable JSONL sink are absent, while current plan events are update-only in-process records.

## Solution Assessment

Unclear. A default-off JSONL stream is plausible, but maintainers should first decide schema stability, privacy boundaries, sink ownership, and whether this belongs with trajectory capture, plugin hooks, or a new audit sink.
