## Summary

Keep open. Current main has adjacent trajectory capture, in-memory agent events, and subagent lifecycle hooks, but it still lacks the requested durable plan/session audit stream with plan ids, close correlation, and `/var/log/openclaw-events.jsonl` emission.

The issue requests an opt-in config flag that emits plan and session lifecycle events with plan correlation into the canonical JSONL event sink for external audit tools.

## Best Possible Solution

Define a maintainer-approved lifecycle telemetry contract first, including owner boundary, schema stability, privacy/redaction, retention/storage sink, opt-in config, and tests for plan open/close plus subagent spawn/yield correlation.

## Solution Assessment

Is this the best way to solve the issue? Unclear pending maintainer design; the operator need is valid, but a direct `/var/log/openclaw-events.jsonl` flag must be weighed against trajectory capture, in-memory agent events, plugin hooks, and storage/privacy policy.
