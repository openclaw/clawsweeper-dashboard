## Summary

Keep open. Current main has adjacent diagnostics, agent/session broadcasts, trajectory exports, and plugin exporter surfaces, but it still does not provide the requested stable plan/session lifecycle audit contract with plan ids, spawn/yield correlation, and exactly-one close or timeout semantics.

## What This Changes

Request a new opt-in plan/session lifecycle event stream to audit unfinished Maestro plans and subagent transitions.

## Best Possible Solution

Define a maintainer-approved, redacted plan/session lifecycle event contract first, then decide whether persistence belongs in diagnostics, trajectory export, a plugin sink, or another named operator audit artifact.

## Reproduction Assessment

Yes by source inspection: current main exposes adjacent diagnostics/session events but not the requested stable plan-id close audit stream. A live run is not needed to see that the public contract and fields are absent.

## Solution Assessment

No for the exact hardcoded /var/log plus new config shape without maintainer design. The maintainable path is to choose the public schema, redaction model, config surface, and persistence owner first.
