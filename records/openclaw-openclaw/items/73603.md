## Summary

Keep open. Current main has adjacent per-session trajectory capture, in-memory agent events, and typed subagent plugin hooks, but it does not provide the requested stable plan/session audit stream with plan identifiers, close correlation, or `/var/log/openclaw-events.jsonl` emission; the remaining work needs maintainer API, storage, and privacy design rather than cleanup closure.

## What This Changes

Request a new opt-in event contract that emits plan and session lifecycle transitions to a canonical JSONL audit sink for task-completion auditing.

## Best Possible Solution

Define one canonical lifecycle telemetry contract for plan, run, and subagent transitions, preferably reusing trajectory or plugin/diagnostic event seams, with explicit schema, retention, privacy, and upgrade semantics before adding operator-facing config.

## Reproduction Assessment

Not applicable as a bug reproduction: this requests a new lifecycle audit capability. Source inspection gives high-confidence gap evidence because current main lacks the requested `plan_id`/`closes_plan`/canonical event-sink contract.

## Solution Assessment

Unclear pending maintainer design. A direct `/var/log/openclaw-events.jsonl` flag is not obviously the best shape because existing trajectory, diagnostic, agent-event, and plugin-hook surfaces overlap and repository policy discourages new persistent JSONL state without a named contract.
