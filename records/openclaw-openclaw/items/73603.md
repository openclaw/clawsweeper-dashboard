## Summary

Keep open. Current main still does not provide the requested default-off durable plan/session lifecycle stream to `/var/log/openclaw-events.jsonl`; the existing plan events are update-only in-process events, and trajectory export is a separate per-session diagnostic sidecar.

## What This Changes

Request a default-off `events.emit.plan` config that writes plan/session lifecycle records to the canonical `/var/log/openclaw-events.jsonl` audit stream.

## Best Possible Solution

Define an additive experimental observability contract with stable plan/run identifiers, open/update/spawn/yield/end records, redaction guarantees, file permission and retention rules, config/docs coverage, and focused disabled/enabled tests.

## Reproduction Assessment

Yes. Static inspection of current main reproduces the gap: the requested config key and JSONL sink are absent, plan events are update-only, and existing trajectory capture is per-session sidecar telemetry rather than `/var/log/openclaw-events.jsonl`.

## Solution Assessment

Unclear. A default-off JSONL stream is plausible, but maintainers should first decide schema stability, privacy boundaries, sink ownership, and whether to build on trajectory capture or plugin/runtime events.
