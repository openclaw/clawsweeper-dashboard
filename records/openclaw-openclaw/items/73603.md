## Summary

Keep open. Current main still lacks the requested default-off durable plan/session audit stream to `/var/log/openclaw-events.jsonl`; existing plan events are update-only in-process events, and trajectory capture is a per-session sidecar/debug export rather than the requested canonical audit sink.

## What This Changes

Request a default-off `events.emit.plan` config flag that writes plan/session lifecycle records with stable plan and subagent correlation to a JSONL event sink.

## Best Possible Solution

Define an additive experimental observability contract with stable plan/run identifiers, open/update/spawn/yield/end records, redaction guarantees, file permission and retention rules, config/docs coverage, and focused disabled/enabled tests.

## Reproduction Assessment

Yes. Static inspection of current main reproduces the gap: the requested config key and JSONL sink are absent, plan events are update-only, and existing trajectory capture is per-session sidecar telemetry rather than `/var/log/openclaw-events.jsonl`.

## Solution Assessment

Unclear. A default-off JSONL stream is plausible, but the maintainable fix needs maintainer decisions on schema stability, privacy, sink ownership, and whether to build on trajectory capture, plugin event APIs, or broader run-trace work.
