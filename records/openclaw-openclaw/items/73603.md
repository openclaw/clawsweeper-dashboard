## Summary

Keep open. Current main has in-process plan update events and run/session/task lifecycle tracking, but it still lacks the requested default-off durable plan/session audit stream with stable plan identifiers and spawn/yield/terminal correlation.

## What This Changes

Add a default-off configuration option that emits durable plan/session lifecycle events with plan correlation into `/var/log/openclaw-events.jsonl`.

## Best Possible Solution

Define an additive experimental observability contract with stable plan/run identifiers, explicit open/update records, child spawn/yield/end correlation, and exactly one terminal close/abort record. Keep it disabled by default, document schema and redaction guarantees, wire config/help/docs, and add focused tests for disabled behavior plus enabled lifecycle completeness.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: the requested config key and durable JSONL sink are absent, plan events are update-only, and task/subagent lifecycle state is keyed by runId/session rather than a stable plan id.

## Solution Assessment

Unclear. A default-off JSONL stream is plausible, but maintainers should first decide the stable schema, privacy model, file/sink ownership, and whether this belongs in core diagnostics, the task ledger, trajectory export, or a plugin/API extension.
