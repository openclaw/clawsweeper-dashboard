## Summary

Keep open. Current main still lacks the requested durable, default-off plan/session lifecycle audit stream: the existing implementation exposes in-process plan updates and run-scoped lifecycle/task events, but there is no `events.emit.plan` config, no stable `plan_id`/`plan_ref`/`closes_plan` contract, and no canonical `/var/log/openclaw-events.jsonl` writer for these events. The request is valid, but it needs maintainer design for schema, redaction, sink ownership, permissions, and retention before an automated fix PR is safe.

## What This Changes

Request to add a default-off configuration flag that emits plan/session lifecycle records with plan, spawn, yield, and close correlation to `/var/log/openclaw-events.jsonl`.

## Best Possible Solution

Define an additive observability contract with stable plan/run identifiers, explicit plan open/update records, subagent spawn/yield/end correlation, and exactly one terminal close/abort record. Keep it disabled by default, document schema and redaction guarantees, wire config/help/docs, and add focused tests for disabled behavior plus enabled lifecycle completeness.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: targeted source/docs searches and the current plan/lifecycle code show the requested config keys, durable JSONL writer, stable plan identifiers, and terminal plan close correlation are absent.

## Solution Assessment

Unclear. A default-off durable JSONL stream is plausible, but maintainers should first choose the schema, redaction/access model, sink ownership, and whether this belongs in core, the task ledger, trajectory export, or plugin/runtime APIs.
