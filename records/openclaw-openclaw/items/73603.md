## Summary

Keep open. Current main still does not provide the requested default-off durable plan/session lifecycle stream, and adding it would require a new config surface plus maintainer decisions about schema stability, redaction, file ownership, permissions, and retention.

## What This Changes

Request adding a default-off `events.emit.plan` flag that writes plan/session lifecycle records to `/var/log/openclaw-events.jsonl` for auditing abandoned Maestro work.

## Best Possible Solution

Define an additive experimental observability contract with stable run/plan identifiers, explicit start/update/spawn/yield/end or abort records, documented privacy guarantees, file ownership/permissions, retention behavior, config/docs coverage, and disabled/enabled regression tests.

## Reproduction Assessment

Yes for the capability gap: static inspection of current main shows plan updates and session lifecycle concepts exist, but the requested config key, stable plan identifiers, terminal plan events, and durable `/var/log/openclaw-events.jsonl` emission path are absent.

## Solution Assessment

Unclear. A default-off JSONL audit stream is plausible, but maintainers should first decide whether this belongs in core, trajectory capture, plugin event subscriptions, or a new event-sink API with explicit redaction and retention rules.
