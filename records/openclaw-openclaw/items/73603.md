## Summary

Keep open. Current main still lacks the requested default-off durable plan/session audit stream; existing plan events are in-process and update-only, while lifecycle/task surfaces are run/session-scoped rather than a stable plan correlation contract.

## What This Changes

Request to add a default-off config flag that emits plan and session lifecycle events, including plan open/spawn/yield/close correlation, to a durable JSONL event stream.

## Best Possible Solution

Define an additive experimental observability contract with stable plan/run identifiers, explicit plan open/update records, child spawn/yield/end correlation, and exactly one terminal close/abort record. Keep it disabled by default, document schema and redaction guarantees, wire config/help/docs, and add focused tests for disabled behavior plus enabled lifecycle completeness.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: the requested config keys and durable JSONL sink are absent, AgentPlanEventData only supports update events, and task/session lifecycle state is keyed by runId/sessionKey rather than a plan id.

## Solution Assessment

Unclear. A default-off JSONL stream is plausible, but maintainers should first decide the stable schema, privacy model, file/sink ownership, and whether the surface belongs in core diagnostics, the task ledger, trajectory export, or a plugin/API extension.
