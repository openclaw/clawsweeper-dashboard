## Summary

Keep open. Current main still reuses the same startup config snapshot across in-process gateway restart iterations, and an open fixing PR already references this issue with closing syntax, so the issue should remain open until that PR lands or is closed.

## What This Changes

The issue reports that in-process gateway restarts reuse a stale startup config snapshot and can overwrite user-written `openclaw.json` changes made after the initial gateway boot.

## Best Possible Solution

Land or close the linked fixing PR after review; the intended end state is first-start-only snapshot reuse with later in-process restart iterations reading the current config from disk.

## Reproduction Assessment

Yes. There is a high-confidence source-level reproduction path: write config after the initial gateway pre-read, trigger an in-process restart, and the next loop iteration still injects the original snapshot instead of rereading disk.

## Solution Assessment

Yes. The narrow first-start-only use of `startupConfigSnapshotRead` is the best maintainable fix because it preserves the cold-start optimization while restoring normal disk reads for restart iterations.
