## Summary

Keep open. Current main still reuses the pre-read startup snapshot across restart-loop iterations, and two open fix PRs already reference this issue with closing syntax, so the issue should stay open until one fix lands.

## What This Changes

Ensure in-process gateway restarts reread the current `openclaw.json` instead of replaying a stale pre-loop startup snapshot and dropping post-boot config writes.

## Best Possible Solution

Land one of the open fix PRs, or replace them with an equivalent patch, so the pre-read snapshot is consumed only for the first start and restart iterations force a fresh disk read with regression coverage.

## Reproduction Assessment

Yes. The source path is high-confidence: `gateway run` pre-reads a snapshot, the in-process loop calls the same start callback again, and server startup honors the injected snapshot instead of rereading disk.

## Solution Assessment

Yes. The first-start-only snapshot consumption proposed here is the narrowest maintainable fix because it preserves the cold-start optimization while restoring normal disk reads for restart iterations.
