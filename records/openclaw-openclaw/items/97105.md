## Best Possible Solution

Land the node-host diagnostic and fail-closed crash fix after normal CI and maintainer review, while keeping the omitted-cwd behavior and macOS app exec-host follow-up separate.

## Solution Assessment

Yes. The node host is the right layer because it is where the remote cwd is interpreted; the patch preserves fail-closed behavior instead of dropping or rewriting an explicit cwd, and gateway-side validation cannot see the node filesystem.
