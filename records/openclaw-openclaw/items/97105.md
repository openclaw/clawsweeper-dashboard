## Summary

Keep open. The PR addresses a still-present node-host non-approval path where an explicit bad remote cwd reaches `spawn()`, producing a misleading shell ENOENT or a synchronous ENOTDIR throw; current main only solves the omitted-cwd and approval-gated cases.

## What This Changes

The branch adds node-host spawn cwd error clarification, catches synchronous spawn throws as failed run results, and adds focused helper and real-spawn coverage in `src/node-host/invoke.test.ts`.

## Best Possible Solution

Land the node-host diagnostic and fail-closed crash fix after normal CI and maintainer review, while keeping the shipped omitted-cwd behavior and any macOS app exec-host follow-up separate.

## Reproduction Assessment

Yes. Current main forwards explicit node cwd values into `runCommand`, and a local Node probe reproduced the raw missing-cwd ENOENT and cwd-as-file synchronous ENOTDIR behavior that the PR handles.

## Solution Assessment

Yes. The node host is the right layer because it is where the remote cwd is interpreted; the fix preserves fail-closed behavior instead of dropping or rewriting an explicit cwd and avoids duplicating gateway-side validation that cannot see the node filesystem.
