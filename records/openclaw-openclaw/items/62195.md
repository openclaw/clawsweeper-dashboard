## Summary

Keep open: current main still lacks the preHook feature, but this PR is not merge-ready because it is missing contributor-side real behavior proof and still allows relative or PATH-resolved preHook executables in a new scheduled host-execution path.

## What This Changes

The PR adds an optional cron and heartbeat preHook executable gate across runtime execution, protocol/config/tool schemas, Swift models, docs, and tests.

## Best Possible Solution

Land a hardened pre-run gate or explicitly supersede it through the broader cron lifecycle hooks design, with absolute executable validation, clear host-exec approval and resource boundaries, aligned schemas/docs/tests, and real proceed/skip/error proof.

## Reproduction Assessment

Yes for the PR-introduced issue: PR-head source stores any non-empty preHook.file and passes it directly to execFile with shell:false. This is not a current-main bug reproduction because current main does not contain the preHook feature.

## Solution Assessment

No. The feature direction is plausible, but this PR should first enforce the documented absolute-path contract at ingress and spawn time, then settle the scheduled host-execution policy and proof gap.
