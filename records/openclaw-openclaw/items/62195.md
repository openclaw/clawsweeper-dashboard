## Summary

Keep open: current main still lacks the requested cron/heartbeat preHook feature, and this branch is a real implementation candidate, but it has a blocking executable-path security issue plus proof and product/security gaps before merge.

## What This Changes

The branch adds an optional cron and heartbeat preHook executable gate with config/protocol/tool schemas, runtime integration, Swift models, docs, and focused tests.

## Best Possible Solution

Choose one hardened cron/heartbeat pre-run or lifecycle hook contract, enforce absolute or otherwise pinned executable paths and host-exec boundaries, align schemas/docs/tests, and require real proceed/skip/error proof before merge.

## Reproduction Assessment

Yes for the PR-introduced review finding: PR-head source stores any non-empty preHook.file and passes it directly to execFile. This is not a current-main bug reproduction because current main does not contain the preHook feature.

## Solution Assessment

No for the patch as written. The feature direction is plausible, but the mergeable path needs absolute-path validation at ingress and spawn time plus a maintainer/security decision on scheduled host execution and its overlap with the broader lifecycle-hooks work.

## Review Findings
