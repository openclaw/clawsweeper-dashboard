## Summary

Keep open: current main does not implement this preHook capability, but the PR is not merge-ready because the executable path hardening is still incomplete, real cron/heartbeat proof is missing, the branch is conflicting, and the work overlaps the broader lifecycle-hooks track.

## What This Changes

The branch adds an optional cron/heartbeat preHook executable gate across config, protocol and tool schemas, runtime execution, Swift models, docs, generated baselines, and focused tests.

## Best Possible Solution

Either harden and prove a narrow trusted-admin preHook contract, or fold the useful schema/test/runtime pieces into the broader cron lifecycle-hooks design after a maintainer security/product decision.

## Reproduction Assessment

Yes for the PR-introduced finding: PR-head source stores any non-empty preHook.file and passes it directly to execFile. This is not a current-main bug reproduction because current main does not contain the preHook feature.

## Solution Assessment

No in its current form: the narrow preHook idea may be viable, but the implementation should enforce pinned executable paths, document the trusted-admin/resource model, provide real behavior proof, and resolve whether this belongs in the broader lifecycle-hooks design.

- **[P1] Require absolute or pinned preHook executables:** `src/cron/pre-hook.ts:66-70`
