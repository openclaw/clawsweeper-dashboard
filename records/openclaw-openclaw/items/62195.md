## Summary

Keep open: current main does not implement preHook, and the latest PR head is not mergeable because it still accepts non-absolute executable paths despite documenting an absolute-path contract and lacks live behavior proof.

## What This Changes

The PR adds optional cron and heartbeat `preHook` executable gates across runtime execution, cron protocol/config/tool schemas, Swift models, docs, and tests.

## Best Possible Solution

Land a hardened additive preHook contract only after absolute executable validation is enforced at every ingress and spawn boundary, the admin/security policy is explicit, and real cron/heartbeat behavior proof is added.

## Reproduction Assessment

Yes for the PR-introduced issue: PR-head source shows any non-empty `preHook.file` is persisted and then passed to `execFile` with `shell:false`. This is not a current-main bug reproduction because current main does not contain preHook.

## Solution Assessment

No. The feature direction is plausible, but the PR should first enforce the documented absolute-path contract at schemas, normalization, and spawn time, then settle the scheduled host-execution policy boundary.

## Review Findings
