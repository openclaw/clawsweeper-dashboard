## What This Changes

The PR adds optional cron and heartbeat preHook executable gates across runtime execution, protocol/config/tool schemas, Swift models, docs, and tests.

## Best Possible Solution

Land a hardened, aligned pre-run gate only after absolute executable validation is enforced, the scheduled host-execution policy is explicit, and real proceed/skip/error behavior proof is added.

## Reproduction Assessment

## Solution Assessment

No. The feature direction is plausible, but this PR should first enforce the documented absolute-path contract at ingress and spawn time, then settle the scheduled host-execution policy and proof gap.
