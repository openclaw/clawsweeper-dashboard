## Summary

This should close as superseded by the prior MiniMax image-routing decision: current OpenClaw supports MiniMax image understanding through the plugin-owned `MiniMax-VL-01` path, while forcing MiniMax chat models such as M2.5/M2.7 onto the image path is the same unsupported configuration already reviewed with live MiniMax validation.

## What This Changes

The issue asks OpenClaw to make MiniMax image description work for URL inputs and a `minimax-cn/MiniMax-M2.5` image model configuration.

## Best Possible Solution

Keep MiniMax image understanding on `minimax/MiniMax-VL-01`; track CLI URL input or future MiniMax M2.5 chat-model vision support only as separate feature/API-compatibility requests with upstream proof.

## Reproduction Assessment

Yes for the source-level and linked live-validation path: forcing a MiniMax chat model into image understanding matches the already validated `chat content is empty` configuration failure at https://github.com/openclaw/openclaw/issues/81695. Yes. Closing as superseded is the narrowest maintainable outcome because current supported routing is `MiniMax-VL-01`, while adding M2.5/M2.7 chat-model vision or HTTP URL support to the CLI would be a new compatibility/product decision.
