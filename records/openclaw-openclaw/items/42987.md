- **[P1] Emit playbook signals from cron finalizers:** `src/cron/service/timer.ts:1071-1072`
  - body: The hook is added only in executeJob, but current scheduler, startup catch-up, and manual run flows execute through executeJobCoreWithTimeout and finalize through applyOutcomeToStoredJob/applyJobResult. In practice, real cron failures and recoveries would not populate this memory.

  - body: This stores signal.error directly in the persistent playbook JSON. Cron errors can include private identifiers, provider details, stack traces, user text, or credentials, so the memory layer needs a redacted and bounded diagnostic summary instead of raw error retention.
  - confidence: 0.84

  - body: The recovery branch rebuilds the failure signature from only previousError. If the original failure was classified by errorKind and the text does not match the regex heuristics, the recovery is written under a different signature or dropped.
  - confidence: 0.86
