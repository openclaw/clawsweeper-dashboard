## Summary

Keep this PR open, but it is not merge-ready: the proposed hook is wired to a helper path that real scheduled, startup catch-up, and manual cron runs bypass, recovery classification loses `errorKind`, and the external PR still lacks real cron runtime proof.

## What This Changes

The branch adds a cron procedural playbook memory JSON store, gateway signal wiring, targeted tests, and a draft autonomy roadmap documentation page.

## Best Possible Solution

Move playbook signal emission into the canonical cron outcome/finalization flow for scheduled, startup catch-up, and manual runs; persist enough classifier state to match recoveries; cover those paths with focused tests; then add redacted real cron proof.

## Reproduction Assessment

Yes, from source inspection. The PR hook is in `executeJob`, while current scheduled, startup catch-up, and manual cron runs use `executeJobCoreWithTimeout` plus finalizers; I did not run a live cron setup.

## Solution Assessment

No. The maintainable solution is to record from the shared cron outcome/finalization path and carry the prior failure classifier through recovery, rather than attaching side effects to a helper path that production runs bypass.
