Full review comments:

- **[P1] Record signals from cron finalization paths:** `src/cron/service/timer.ts:1071-1072`
  - body: The new hook is only in executeJob, but current scheduled ticks, startup catch-up, and manual cron run finalize through executeJobCoreWithTimeout plus applyOutcomeToStoredJob/applyJobResult.   - confidence: 0.91
- **[P1] Redact stored cron errors:** `src/cron/procedural-playbook-memory-v0.ts:236`
  - body: This persists signal.error directly into the playbook JSON state. Cron errors can contain secrets, private targets, user content, stack traces, or provider details, so the memory layer needs a redacted and bounded representation before it writes durable state.
  - confidence: 0.84
- **[P2] Preserve classifier metadata for recoveries:** `src/cron/service/timer.ts:1083-1090`
  - body: Recovery signals are rebuilt from lastError without the original errorKind. When the prior failure was classified by errorKind and the text does not match the regex fallback, the recovery gets a different signature and is silently dropped.
  - confidence: 0.82
