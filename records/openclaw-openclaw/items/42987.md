  - body: The new hook is only in executeJob, but scheduled ticks, startup catch-up, and manual cron runs execute through executeJobCoreWithTimeout and finalize through applyOutcomeToStoredJob or applyJobResult.   - body: The recovery branch rebuilds the failure signature from previousError because CronJobState does not store the prior errorKind. If the failed run was classified by errorKind and the text does not match a regex, recovery tracking targets a different signature and is silently dropped.
  - confidence: 0.87
- **[P2] Redact stored playbook errors before saving:** `src/cron/procedural-playbook-memory-v0.ts:236`
  - body: The memory layer persists signal.error verbatim as lastError. Cron errors can include provider URLs, identifiers, stack traces, user content, or tokens, so the value needs redaction and a length bound before it is written to persistent JSON state.
  - confidence: 0.84
