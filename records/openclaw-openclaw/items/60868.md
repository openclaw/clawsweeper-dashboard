## Summary

Keep open: current main still lacks the central chain-aware overflow detection, but the latest PR head remains incorrect because a raw nested overflow message can be selected and logged, and the external contributor has not provided after-fix real behavior proof.

## What This Changes

The PR adds a `.error`/`.cause` chain message collector and uses it in agent model fallback, embedded overflow compaction, and auto-reply failure classification with focused tests and small adjacent Discord, Gateway, MiniMax, and lane-test edits.

## Best Possible Solution

Revise the PR so raw chain messages are used only for predicates, a parallel redacted value is logged or shown, billing/rate-limit and structured code precedence stay covered, and unrelated edits are split or explicitly accepted by a maintainer.

## Reproduction Assessment

Yes, at source level: on the latest PR head, a `promptError` with a nested raw `.error.message` containing a bearer token and an overflow phrase can be selected from the raw candidate list and then logged. I did not run a runtime test because this was a read-only review.

## Solution Assessment

No, not yet. Chain-wide classification is the right direction, but the embedded-runner path must keep raw strings classifier-only and use redacted text for diagnostics before this is a maintainable fix.
