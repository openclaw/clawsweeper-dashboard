## Summary

Keep this PR open: current main still lacks the proposed chain-aware overflow handling, but the latest PR head keeps a blocking security regression by letting raw nested prompt-error messages become logged and stored overflow diagnostics, and the external PR still lacks real after-fix behavior proof.

## What This Changes

The branch adds error-chain message collection and uses it for context-overflow classification in model fallback, embedded-runner overflow compaction, and auto-reply failure classification, with related tests plus small Discord, Gateway, MiniMax, and changed-lane edits.

## Best Possible Solution

Revise the PR so raw chain candidates are used only for internal classification while all logged, persisted, and user-visible diagnostics use redacted display text; keep the chain-wide tests and add proof from a real run.

## Reproduction Assessment

Yes, at source level. A promptError with outer "request failed" and a nested .error or .cause message containing both an Authorization bearer token and a context-overflow phrase reaches the PR's raw candidate path and can become the surfaced overflow errorText.

## Solution Assessment

No. The direction is useful, but the implementation still conflates raw classification candidates with display diagnostics; the safer solution is to keep paired raw/redacted candidates or otherwise guarantee redaction before logging or metadata storage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep raw chain messages out of overflow diagnostics:** `src/agents/pi-embedded-runner/run.ts:1296`
  - body: `collectErrorChainMessages(promptError, { redact: false })` can select a raw nested provider message as `errorText`; that same text is logged and stored in run metadata. If the nested error includes an Authorization header, API key, or request metadata plus an overflow phrase, it bypasses the existing `formatErrorMessage()` redaction boundary, so raw candidates should stay classifier-only and surfaced text should be redacted.
  - confidence: 0.93
