## Summary

Keep open: current main still lacks chain-aware overflow detection in the affected runtime paths, but the latest PR head still lets raw nested error messages reach embedded-runner overflow diagnostics and has no real after-fix behavior proof.

## What This Changes

The PR adds a chain-aware error message collector for agent failover/model fallback, embedded overflow compaction, and auto-reply classification, with regression tests plus small Discord/Gateway trusted-event and changed-lane test updates.

## Best Possible Solution

Keep the chain-wide classifier work, but separate raw internal matching candidates from redacted display text for logs, failure metadata, Control UI output, and user-facing replies.

## Reproduction Assessment

Yes. A source-level repro for the blocking finding is a promptError shaped like Object.assign(new Error("request failed"), { error: { message: "Authorization: Bearer ... context length exceeded" } }); the PR selects the raw nested message and logs it in the overflow diagnostic.

## Solution Assessment

No. The product direction is right, but the implementation should keep raw chain messages classifier-only and surface only redacted strings through the existing error-formatting boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep raw chain candidates out of overflow diagnostics:** `src/agents/pi-embedded-runner/run.ts:1296`
  - body: `collectErrorChainMessages(promptError, { redact: false })` is added to the candidate list, and the matched segment becomes `errorText`, which is logged as `error=${errorText.slice(0, 200)}` and reused in overflow failure metadata. If a transport `.error.message` contains an Authorization header, API key, or request metadata plus an overflow phrase, this bypasses the existing `formatErrorMessage()` redaction boundary; match on raw text internally and surface a redacted string.
  - confidence: 0.93

## Security Review

Status: needs_attention

Summary: The diff still introduces a concrete secret-redaction regression in the embedded-runner overflow diagnostic path.

Concerns:

- **[high] Raw nested error messages can leak to logs:** `src/agents/pi-embedded-runner/run.ts:1296`
  - body: The embedded runner opts into raw chain messages for prompt-error overflow detection, then propagates the matching raw segment as diagnostic text. Provider and transport errors can contain bearer tokens, API keys, or request metadata in nested `.error.message` fields, so the raw value should stay classifier-only or be redacted before logging.
  - confidence: 0.92
