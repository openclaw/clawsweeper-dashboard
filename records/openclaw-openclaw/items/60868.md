## Summary

Keep open: current main still lacks the proposed chain-wide overflow detection, but the latest PR head has a blocking raw-error diagnostic leak and still lacks external real behavior proof.

## What This Changes

The PR adds an error-chain message collector for `.error` and `.cause`, applies it to agent model fallback, embedded overflow compaction, and auto-reply classification, and includes focused tests plus small adjacent Discord, Gateway, MiniMax, and changed-lane edits.

## Best Possible Solution

Revise this PR so raw chain messages are used only for predicates, redacted companion text is used for logs and user-visible diagnostics, existing billing/rate-limit precedence stays covered, and the contributor adds real after-fix proof.

## Reproduction Assessment

Yes, source-level: the PR head selects raw nested overflow text in `src/agents/pi-embedded-runner/run.ts` and logs that same selected value in the context-overflow diagnostic. I did not run tests because this review was read-only.

## Solution Assessment

No, not yet. Chain-wide overflow detection is the right direction, but the implementation must keep raw classifier candidates separate from redacted diagnostic and reply text before it is safe to merge.
