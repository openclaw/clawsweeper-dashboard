## Summary

Keep open: the PR targets a real gap that current main still does not fully cover, but the latest head still has a blocking security issue because raw nested error-chain text can be selected as embedded-runner overflow diagnostics and then logged or stored in metadata.

## What This Changes

The PR adds a shared error-chain message collector and uses `.error`/`.cause` chain messages for agent model fallback, embedded overflow compaction, and auto-reply failure classification, with focused tests plus small Discord/Gateway/changed-lanes edits.

## Best Possible Solution

Revise the PR so raw chain candidates are used only for classification while logs, metadata, Control UI text, and replies use redacted display text; then resolve conflicts and add real behavior proof.

## Reproduction Assessment

Yes. By source inspection of the PR head, an `Error("request failed")` with a custom `.error.message` containing an Authorization token plus `context length exceeded` can be selected as raw `errorText` and then logged/stored by the embedded-runner overflow path.

## Solution Assessment

No. Chain-wide overflow classification is the right direction, but the current PR should keep raw candidates internal and surface only redacted diagnostics before it is mergeable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Redact selected overflow diagnostics before surfacing:** `src/agents/pi-embedded-runner/run.ts:1296-1302`
  - body: `collectErrorChainMessages(promptError, { redact: false })` can supply the selected `errorText`, and that same value is later logged and stored in run metadata. Keep raw nested messages for classifier checks only, then return a redacted display string for diagnostics so provider errors containing Authorization headers or API keys cannot leak.

## Security Review

Status: needs_attention

Summary: The PR still has a concrete secret-redaction regression in embedded-runner overflow diagnostics.

Concerns:

- **[high] Raw nested error messages can leak to logs:** `src/agents/pi-embedded-runner/run.ts:1296`

Provider and transport errors can include bearer tokens, API keys, or request metadata in nested `.error`/`.cause` messages, so raw candidates should not be surfaced.
  - confidence: 0.93
