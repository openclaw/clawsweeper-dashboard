## Summary

Keep open: current main still lacks the chain-wide context-overflow handling, and this PR remains useful, but the latest diff is not safe to merge because the embedded-runner overflow path can surface raw nested error-chain messages into logs and run metadata.

## What This Changes

The PR adds a shared error-chain message collector and applies it to agent model fallback, embedded overflow compaction, and auto-reply failure classification, with focused tests plus small Discord/Gateway trusted-event and changed-lane updates.

## Best Possible Solution

Revise the branch so chain-wide classification can inspect .error and .cause candidates, while logs, metadata, Control UI text, and replies use redacted display text; then resolve current-main conflicts and split unrelated churn if maintainers want a narrower PR.

## Reproduction Assessment

Yes. A promptError with an outer “request failed” message and a nested .error.message containing an Authorization bearer token plus “context length exceeded” reaches the PR’s raw overflow candidate path and can be selected as the diagnostic errorText.

## Solution Assessment

No. The product direction is right, but the current implementation still lets raw nested chain messages feed a surfaced overflow diagnostic instead of keeping raw values internal to classification.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep raw chain messages out of overflow diagnostics:** `src/agents/pi-embedded-runner/run.ts:1297`
  - body: `collectErrorChainMessages(promptError, { redact: false })` feeds the selected `errorText`, and the existing overflow path logs `errorText.slice(0, 200)` and stores it in `meta.error.message`. A nested provider error with an Authorization header plus an overflow phrase would bypass `formatErrorMessage()` redaction, so raw candidates should stay classifier-only and surfaced text should be redacted.
  - confidence: 0.93

## Security Review

Status: needs_attention

Summary: The diff still introduces a concrete secret-redaction regression in embedded-runner overflow diagnostics.

Concerns:

- **[high] Raw nested error messages can leak to logs:** `src/agents/pi-embedded-runner/run.ts:1297`
  - body: The embedded-runner overflow branch opts into raw chain messages and can propagate the matching raw segment into logs and run metadata. Provider or transport errors can include bearer tokens, API keys, or request metadata in nested .error/.cause messages, so raw candidates should not be surfaced.
  - confidence: 0.91
