## Summary

Keep open: current main still does not implement the PR’s chain-aware overflow handling, but the latest PR head still has a blocking security issue because embedded-runner overflow diagnostics can log raw nested error messages. The external-contributor real behavior proof gate is also still unmet, so this needs contributor revision and maintainer review rather than cleanup close.

## What This Changes

The PR adds a `.error`/`.cause` message collector and applies chain-aware context-overflow classification to model fallback, embedded overflow compaction, and auto-reply failure handling, with focused tests plus small adjacent Discord, Gateway, MiniMax, and changed-lane edits.

## Best Possible Solution

Revise the PR so raw chain candidates are used only for predicates, redacted counterparts are used for logs and user-visible text, and regression coverage includes a nested `.error.message` with a secret plus an overflow phrase.

## Reproduction Assessment

Yes, at source level on the latest PR head: a `promptError` whose outer message is `request failed` and whose nested `.error.message` contains `Authorization: Bearer ... context length exceeded` is selected from the raw candidate list and then logged. No runtime test was run because this was a read-only review.

## Solution Assessment

No, not yet. Chain-wide classification is the right direction, but the implementation must keep raw nested messages classifier-only and log or surface only redacted text.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Redact chain messages before logging overflow diagnostics:** `src/agents/pi-embedded-runner/run.ts:1296`
  - body: `collectErrorChainMessages(promptError, { redact: false })` feeds the candidate list, and when a custom nested `.error.message` is the first overflow-matching segment it becomes `contextOverflowError.text` and is logged as `error=...`. A transport wrapper can include `Authorization: Bearer ... context length exceeded`, so keep raw strings only for matching and log a redacted parallel value.
  - confidence: 0.92
