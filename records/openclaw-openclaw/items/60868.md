## Summary

Keep open. Current main still lacks the custom `.error`/full-chain overflow detection this PR is trying to add, but the latest PR head has a security-sensitive blocker because raw nested messages can still be selected for embedded-runner diagnostics.

## What This Changes

The PR adds chain-aware error message collection for agent failover, model fallback, embedded overflow compaction, and auto-reply classification, with related tests plus small Discord/Gateway trusted-event and changed-lane test updates.

## Best Possible Solution

Revise the branch so raw chain candidates are used only for internal classification, while diagnostics, logs, stored failure text, Control UI output, and user-facing replies use redacted display text; keep the chain-wide overflow tests and split unrelated churn if maintainers prefer a narrower PR.

## Reproduction Assessment

Yes. A prompt error shaped like `Object.assign(new Error("request failed"), { error: { message: "Authorization: Bearer sk-test-super-secret-token-that-should-never-leak context length exceeded" } })` reaches the PR's raw chain candidate path and can be selected as the overflow diagnostic text.

## Solution Assessment

No. The product direction is right, but this is not the best implementation until raw nested messages stay classifier-only and every surfaced error string goes through the existing redaction boundary or an equivalent redacted display path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep raw chain messages out of overflow diagnostics:** `src/agents/pi-embedded-runner/run.ts:1296`
  - body: `collectErrorChainMessages(promptError, { redact: false })` feeds the selected overflow `errorText`, which is later logged in the context-overflow diagnostic. For custom `.error.message` branches that `formatErrorMessage()` does not traverse, a nested provider or transport error containing an Authorization header, API key, or request metadata can bypass redaction; use raw candidates only for matching and surface a redacted string.
  - confidence: 0.92

## Security Review

Status: needs_attention

Summary: The PR still has a concrete secret-redaction regression in the embedded-runner overflow diagnostic path.

Concerns:

- **[high] Raw nested error messages can leak to logs:** `src/agents/pi-embedded-runner/run.ts:1296`
  - body: The embedded runner opts into raw chain messages for prompt-error overflow detection, then propagates the matching raw segment as diagnostic text. Provider and transport errors can contain bearer tokens, API keys, or request metadata in nested `.error.message` fields, so the raw value should stay classifier-only or be redacted before logging.
  - confidence: 0.9
