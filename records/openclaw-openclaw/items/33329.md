## Summary

Keep open. Current main implements the per-provider discovery gates, and the closed companion PR correctly noted that the code moved upstream, but the public docs still do not document the Hugging Face and Ollama opt-out paths and the later global-disable request needs maintainer scoping.

## What This Changes

The issue asks for documented config controls that suppress implicit startup discovery for Bedrock, Copilot, Hugging Face, and Ollama, plus a later request for a global model-discovery opt-out.

## Best Possible Solution

Document the existing plugin-owned discovery toggles for Hugging Face and Ollama, keep Bedrock/Copilot aligned, and split the global discovery kill switch into a separate product decision if maintainers want that surface.

## Reproduction Assessment

Yes. Source inspection shows the runtime toggles and tests exist, while the Hugging Face and Ollama public provider docs still do not show the corresponding plugin-owned opt-out paths.

## Solution Assessment

No for closing now. The maintainable path is to document the existing plugin-owned toggles first, not add more legacy `models.*Discovery` keys or silently fold in a global kill switch without maintainer scoping.
