## Summary

Keep open. Current main implements per-provider discovery toggles and documents Bedrock, but public docs still omit the non-Bedrock toggle paths and there is no typed global model-discovery disable control, so the request is only partially addressed.

## What This Changes

The issue asks to document existing Bedrock discovery config and add documented opt-out controls for Copilot, Hugging Face, Ollama, plus a later-requested global discovery disable flag.

## Best Possible Solution

Document the plugin-owned discovery toggles publicly, then add an additive global model-discovery disable control with explicit precedence or split that global switch into a separate design-tracked item.

## Reproduction Assessment

Yes. Static inspection on current main shows the per-provider toggles exist in runtime and manifests, but `docs/gateway/config-tools.md` only documents Bedrock and `ModelsConfigSchema` has no global discovery-disable field.

## Solution Assessment

Partially. The plugin-owned per-provider toggles are the right boundary, but a global kill switch needs a maintainer-chosen public key and precedence relative to those plugin toggles before coding.
