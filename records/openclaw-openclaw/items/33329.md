## Summary

Keep open. Current main has provider-owned discovery toggles for the originally named providers and newer Bedrock Mantle coverage, but Hugging Face and Ollama public docs still do not show the opt-out path, there is still no global model-discovery disable key, and replace-mode discovery behavior is still handled by an open follow-up PR.

## What This Changes

The issue requests documentation and config controls to disable implicit model-provider discovery network calls, with later discussion asking for a global disable switch.

## Best Possible Solution

Document every plugin-owned discovery opt-out, land or replace the focused replace-mode fix, and define one global discovery-disable contract with explicit precedence over per-plugin settings.

## Reproduction Assessment

Yes. Static source and docs inspection on current main shows provider-level toggles exist, but the Hugging Face and Ollama docs gap, missing global disable key, and replace-mode implicit resolver call remain visible.

## Solution Assessment

Partially. Plugin-owned discovery.enabled settings are the right boundary for provider-specific opt-outs, but a global disable switch and its precedence need maintainer product judgment before a narrow implementation is safe.
