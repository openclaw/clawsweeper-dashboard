## Summary

Keep open. Current main has runtime per-provider discovery toggles for GitHub Copilot, Hugging Face, and Ollama, and Bedrock discovery is documented, but public docs still do not document the non-Bedrock toggle paths and there is no typed global model-discovery disable control.

## What This Changes

The issue requests documentation for implicit model discovery settings plus config toggles for Copilot, Hugging Face, and Ollama, with later discussion asking for a global disable-all model discovery flag.

## Best Possible Solution

Finish the public docs for the shipped plugin-owned discovery paths and legacy aliases, then add an additive typed global discovery disable control with focused tests or split that global kill switch into a narrower follow-up if maintainers want to decide the config shape separately.

## Reproduction Assessment

Yes. Static inspection against current main reproduces the remaining gap: runtime per-provider toggles are present and tested, while docs searches do not find the non-Bedrock toggle paths and the model config type has no global discovery-disable field.

## Solution Assessment

Partially. The plugin-owned per-provider toggles are the right owner boundary for provider-specific discovery, but the issue is not fully solved until those paths are documented and the requested global disable behavior is either implemented or explicitly split into its own tracked design decision.
