## Summary

Keep open. Current main has plugin-owned discovery toggles for Bedrock, GitHub Copilot, Hugging Face, and Ollama, but Hugging Face/Ollama public docs still do not show their disable paths and the requested global model-discovery disable control is not implemented.

## What This Changes

Requests documentation and config controls to suppress implicit startup model-discovery network calls for Bedrock, Copilot, Hugging Face, Ollama, and all model discovery globally.

## Best Possible Solution

Document every plugin-owned discovery knob, then add a maintainer-approved global model-discovery disable control with clear precedence over per-plugin settings.

## Reproduction Assessment

Yes. Static source inspection on current main proves the provider-level toggles exist, the docs gap remains for Hugging Face/Ollama, and the strict models schema has no global discovery-disable key.

## Solution Assessment

Partially. Plugin-owned discovery.enabled settings are the right per-provider boundary, but the global kill switch needs a core config contract and precedence policy before implementation is maintainable.
