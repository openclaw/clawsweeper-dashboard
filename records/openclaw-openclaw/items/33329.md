## Summary

Keep open. Current main has provider-owned discovery toggles for Bedrock, GitHub Copilot, Hugging Face, and Ollama, but the public docs still only document Bedrock discovery and there is still no global model-discovery disable control.

## What This Changes

The issue requests documentation for implicit model discovery and config toggles to disable startup discovery for Bedrock, GitHub Copilot, Hugging Face, Ollama, and all model discovery globally.

## Best Possible Solution

Publicly document the plugin-owned discovery toggles, then add or explicitly split a global model-discovery disable control once naming and override rules are settled.

## Reproduction Assessment

Yes. Static inspection on current main shows the provider-level toggles exist, while public docs omit the non-Bedrock toggle paths and the strict ModelsConfigSchema has no global discovery-disable field.

## Solution Assessment

Partially. Provider-owned toggles are the right boundary for per-plugin discovery, but the requested global kill switch needs a maintainer-chosen key and precedence model before a narrow implementation is safe.
