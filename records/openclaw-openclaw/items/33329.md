## Summary

Current main and shipped releases satisfy the issue’s central request: Bedrock, GitHub Copilot, Hugging Face, and Ollama have provider-owned discovery controls, focused tests cover disabling discovery, and Bedrock’s complete discovery configuration is documented. The later suggestion for one global switch is a broader product-policy addition rather than a reason to keep this provider-specific request open.

## What This Changes

Add documented controls to suppress implicit startup model discovery triggered by ambient credentials for Bedrock, GitHub Copilot, Hugging Face, and Ollama.

## Best Possible Solution

Keep the shipped provider-owned controls as the canonical design; if maintainers want one cross-provider master switch, track that separately as an explicit product and configuration decision.

## Reproduction Assessment

Yes. Source inspection and focused tests show the original ambient discovery paths and verify that `discovery.enabled: false` suppresses them on current main.

## Solution Assessment

Yes. Provider-owned plugin configuration is narrower and more maintainable than parallel core `models.*Discovery` keys, and it preserves each provider’s ownership boundary.
