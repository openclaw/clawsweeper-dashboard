## Summary

Keep open. Current main implements provider-owned discovery toggles for the originally listed providers, but the public docs still do not fully document every disable path, the requested global model-discovery kill switch is not implemented, and a related Mantle discovery gate remains tracked separately.

## What This Changes

The issue asks OpenClaw to document implicit model-discovery controls and add config toggles so startup discovery can be disabled independently of ambient environment variables.

## Best Possible Solution

Document every plugin-owned discovery knob, resolve the separate Mantle gate through its canonical issue/PR, and add a maintainer-approved global model-discovery disable control with clear precedence over per-plugin settings.

## Reproduction Assessment

Yes. Static source and docs inspection on current main proves the provider-level toggles exist, the Hugging Face/Ollama docs gap remains, and the strict models schema has no global discovery-disable key.

## Solution Assessment

Partially. Plugin-owned discovery.enabled settings are the right per-provider boundary, but the global kill switch needs a core config contract and precedence policy before implementation is maintainable.
