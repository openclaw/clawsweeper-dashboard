## Summary

Keep open. Current main has the original provider-level runtime toggles and Bedrock documentation, but the public docs still omit several disable paths, there is no global model-discovery disable key, and related replace-mode and Bedrock Mantle discovery gaps remain open in separate canonical items.

## What This Changes

The issue asks OpenClaw to document existing discovery controls and add config toggles for startup model discovery paths such as Bedrock, Copilot, Hugging Face, Ollama, plus a later-requested global discovery disable control.

## Best Possible Solution

Document every plugin-owned discovery knob, land or replace the separate Mantle and replace-mode fixes, and have maintainers choose a clear global discovery-disable contract with precedence over per-plugin settings.

## Reproduction Assessment

Yes. Static source and docs inspection on current main proves the provider-level toggles exist, but the public docs gap, lack of a global discovery-disable key, replace-mode implicit resolver call, and Mantle missing gate remain visible in source.

## Solution Assessment

Partially. Plugin-owned `discovery.enabled` settings are the right provider boundary, but the global disable control and precedence with `models.mode`, per-plugin toggles, and existing discovery flows need maintainer product judgment before a narrow implementation is safe.
