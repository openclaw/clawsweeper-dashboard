## Summary

Keep open. Current main has per-provider plugin discovery toggles for GitHub Copilot, Hugging Face, and Ollama, and Bedrock discovery is documented, but public docs still omit the non-Bedrock toggle paths and there is no typed global model-discovery disable control.

## What This Changes

The issue requests documentation for implicit model discovery settings plus config toggles for Copilot, Hugging Face, and Ollama, with later discussion asking for a global disable-all model discovery flag.

## Best Possible Solution

Document the plugin-owned discovery toggles publicly, then either add an additive global model-discovery disable control with clear precedence or split that global kill switch into its own tracked design item.

## Reproduction Assessment

Yes. Static inspection against current main reproduces the remaining gap: per-provider toggles exist in runtime and manifests, but public docs omit the non-Bedrock paths and the typed config/schema has no global discovery-disable field.

## Solution Assessment

Partially. The plugin-owned per-provider toggles are the right owner boundary, but the issue is not fully solved until those paths are documented and the global disable behavior is either implemented or explicitly split into a follow-up.
