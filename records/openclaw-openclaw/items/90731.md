The PR fixes a reproduced gateway capability mismatch at the correct generic model-catalog boundary: exact allowlisted dynamic rows are hydrated through existing provider hooks, richer metadata is used before rejecting thinking levels, and failed hydration remains usable but is excluded from persistent/shared gateway caches. ## What This Changes

The branch hydrates incomplete exact allowlisted dynamic model rows during full catalog loads, uses the resulting metadata for agent and session thinking validation, and prevents partial catalog results from entering stable caches.

## Best Possible Solution

Land the generic exact-ref hydration design after maintainers explicitly accept provider I/O on full metadata loads; keep read-only browsing side-effect-free and evolve partial-catalog completeness into a typed shared contract if this pattern gains more consumers.
