## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Cap MCP App resources before decoding and writing:** `src/agents/mcp-ui-resource.ts:49-53`
  - body: `readResource` can return arbitrarily large `text` or base64 `blob` content, and this path decodes it and writes the resulting HTML into a canvas document without any host-side byte cap. A configured or compromised MCP server can therefore force large memory allocations and persistent state-dir writes when apps are enabled; reject oversized resource payloads before base64 decode and before `createCanvasDocument`.
  - confidence: 0.87
- **[P1] Enforce MCP App CSP for rendered HTML:** `ui/src/ui/components/mcp-app-view.ts:249-250`
  - body: The iframe only gets sandbox plus `navigate-to`/`form-action` restrictions, but MCP Apps require the host to enforce a restrictive default CSP and allow only declared domains. With scripts enabled and no effective `default-src`/`connect-src`/resource policy, MCP-provided app HTML can load or exfiltrate to undeclared origins, so this needs a real sandbox/CSP serving path before merge.
  - confidence: 0.91
