## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Stop persisting MCP session data in canvas metadata:** `src/agents/mcp-ui-resource.ts:96-99`
  - body: `buildMcpAppCanvasJson` serializes the view metadata together with raw `toolInput` and full `toolResult`, and that view metadata already includes `sessionKey`. This puts a bearer-like session capability and user/tool payloads into stored chat/canvas data; keep the session key in live UI context and persist only minimal redacted display metadata.
  - confidence: 0.92
- **[P1] Bound MCP App resources before decoding or writing:** `src/agents/mcp-ui-resource.ts:49-53`
  - body: This path accepts MCP resource text or base64 blobs and immediately decodes/writes the HTML canvas document without raw or decoded byte limits. A configured MCP server can therefore exhaust memory or disk when MCP Apps are enabled; add explicit raw and decoded size caps before base64 decode and canvas persistence.
  - confidence: 0.9
- **[P1] Enforce MCP Apps resource CSP metadata:** `ui/src/ui/components/mcp-app-view.ts:249-250`
  - body: The dependency contract exposes `_meta.ui.csp`/permissions for MCP App resources, but the host drops that metadata and hardcodes only `navigate-to`/`form-action` on the iframe. Carry the resource metadata into the manifest and build the iframe CSP/allow policy from it, including secure defaults for omitted domains.
  - confidence: 0.86
