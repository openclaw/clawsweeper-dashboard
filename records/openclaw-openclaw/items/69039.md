## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Cap MCP App resources before decoding and writing:** `src/agents/mcp-ui-resource.ts:49-53`
  - body: When MCP Apps are enabled, this accepts any resource `text` string or base64 `blob`, decodes it, and passes the full HTML into `createCanvasDocument`, whose HTML path writes it to disk. The MCP SDK schemas only constrain these fields as strings, so a configured server can return an oversized UI resource and exhaust memory or disk before the host rejects it; add byte caps for both text and blob resources with focused tests.
  - confidence: 0.92
- **[P2] Render disabled MCP App previews through the canvas frame:** `ui/src/ui/chat/tool-cards.ts:478-479`
  - body: This early return hides any persisted canvas preview tagged with `mcpApp` whenever `mcp.apps.enabled` is false. The PR body says disabled apps should fall back to existing canvas/embed rendering, and current main renders assistant canvas previews through `renderPreviewFrame`, so disabling the flag or reloading history makes the preview disappear instead.
  - confidence: 0.89
