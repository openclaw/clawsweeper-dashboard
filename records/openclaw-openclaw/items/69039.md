## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep bridge capabilities out of stored canvas metadata:** `src/agents/mcp-ui-resource.ts:55`
  - body: `fetchMcpAppView` copies `runtime.sessionKey` into `mcpApp`, and `buildMcpAppCanvasJson` serializes that metadata together with tool input/result into assistant canvas JSON. That turns a live bridge capability and potentially sensitive MCP payloads into persisted chat/rendered content, so the bridge should bind authorization from active server/UI state or a short-lived server-side handle instead.
  - confidence: 0.9
- **[P1] Revalidate runtimes before serving app bridge calls:** `src/gateway/server-methods/mcp-app-proxy.ts:29-31`
  - body: `resolveRuntime` returns `manager.getExisting(sessionId)` before loading current config or checking the current MCP fingerprint. An already-open app view can therefore keep calling a runtime after `mcp.apps.enabled` is disabled or a server is removed/rotated, bypassing the current-main runtime revalidation contract.
  - confidence: 0.88
- **[P1] Reuse canonical session IDs for restored app runtimes:** `src/gateway/server-methods/mcp-app-proxy.ts:38-40`
  - body: When the session-key mapping is missing, the proxy creates a runtime under `mcp-app:${sessionKey}`. Normal session cleanup and subsequent chat turns use the canonical session id, so restored app views can create a second MCP runtime and leave MCP processes outside the expected lifecycle.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The diff adds a security-sensitive MCP HTML bridge and has concrete concerns around persisted bridge capability data, stale runtime access, and unscoped app resource reads.

Concerns:

- **[high] Persisted app bridge capability:** `src/agents/mcp-ui-resource.ts:55`
  - body: `runtime.sessionKey` is copied into canvas metadata, making a live bridge capability persist in assistant-rendered content rather than staying bound to active UI/server state.
  - confidence: 0.9
- **[medium] Sensitive tool payloads stored in canvas JSON:** `src/agents/mcp-ui-resource.ts:87`
  - body: `toolInput` and `toolResult` are copied into the persisted canvas JSON payload; MCP tool arguments/results can contain user, workspace, or service data that needs an explicit redaction or retention policy.
  - confidence: 0.84
- **[high] Stale runtime can preserve old MCP access:** `src/gateway/server-methods/mcp-app-proxy.ts:29`
  - body: The app proxy returns an existing runtime without current config revalidation, so disabled apps or removed/rotated MCP servers can remain callable through an already-open bridge.
  - confidence: 0.88
- **[medium] Unscoped resource reads need policy review:** `src/gateway/server-methods/mcp-app-proxy.ts:106`
  - body: `mcp.readResource` accepts any URI for the selected server once apps are enabled, instead of constraining reads to the resource that produced the app view or another maintainer-approved resource policy.
  - confidence: 0.72

  - body: The PR decodes text/blob UI resources into memory and persists them as canvas documents without an explicit size limit at the MCP app boundary.
