## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Stop persisting MCP session data in canvas metadata:** `src/agents/mcp-ui-resource.ts:96-99`
  - body: `mcpApp` is written into chat/canvas JSON, so spreading `view.mcpApp` here carries the serialized `sessionKey`, and these lines add raw tool input/result payloads. Keep session capability and sensitive tool data in memory or behind a server-side authorization handle instead of storing them in persisted chat/canvas/export material.
  - confidence: 0.9
- **[P1] Bound MCP App resources before decoding or writing:** `src/agents/mcp-ui-resource.ts:49-54`
  - body: A server can return arbitrary-size text or base64 HTML here, and the code decodes then writes it as a canvas document without any raw or decoded byte cap. Add strict limits before base64 decode and before persistence so enabling MCP Apps cannot exhaust memory or disk.
  - confidence: 0.88
- **[P1] Enforce MCP Apps resource CSP metadata:** `ui/src/ui/components/mcp-app-view.ts:249-250`
  - body: The iframe gets only a static navigation/forms CSP, but the MCP Apps spec requires hosts to enforce resource-declared `connectDomains`, `resourceDomains`, `frameDomains`, and restrictive defaults. Parse and persist the resource CSP metadata, then apply it to the hosted app document/iframe before exposing this host surface.
  - confidence: 0.86
