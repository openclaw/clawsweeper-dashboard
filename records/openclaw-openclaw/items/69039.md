## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Cap MCP App resources before decoding or writing HTML:** `src/agents/mcp-ui-resource.ts:49-54`
  - body: An enabled MCP server can return arbitrary `text` or base64 `blob` contents here, and the patch decodes the entire value before persisting it as canvas HTML. Add a fixed host-side byte limit before blob decoding and before `createCanvasDocument`, with regression coverage for oversized text and blob resources, so a buggy server cannot exhaust memory or disk during a tool call.
  - confidence: 0.89
- **[P2] Render disabled MCP Apps through the existing iframe fallback:** `ui/src/ui/chat/tool-cards.ts:478-480`
  - body: The PR says disabled MCP Apps should fall back to normal rendering, but this early return drops any preview carrying `mcpApp` before the existing `renderPreviewFrame` fallback is reached. Persisted or restored app views can disappear after `mcp.apps.enabled` is turned off; let the later branch choose the bridged element only when enabled and otherwise render the hosted iframe without the bridge.
  - confidence: 0.82

## Security Review

Status: needs_attention
