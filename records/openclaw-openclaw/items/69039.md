## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Stop persisting MCP session and tool payload data:** `src/chat/canvas-render.ts:23-29`
  - body: The MCP app preview metadata is durable chat/canvas state, but this PR stores `sessionKey`, `toolInput`, and full `toolResult` in it. That turns session identifiers and potentially sensitive tool payloads into history/log/export/preview data; keep only non-sensitive app identity here and source session context from in-memory authorized UI/gateway state.
  - confidence: 0.92
- **[P1] Bound MCP App resources before decoding or writing:** `src/agents/mcp-ui-resource.ts:49-54`
  - body: A configured MCP server can return arbitrarily large `text` or base64 `blob` content, and this path decodes and writes it as a canvas document without a raw or decoded byte cap. Add limits before base64 decode and before canvas persistence so enabling MCP Apps cannot exhaust memory or disk.
  - confidence: 0.9
- **[P1] Enforce MCP Apps resource CSP metadata:** `ui/src/ui/components/mcp-app-view.ts:249-250`
  - body: The ext-apps contract exposes resource CSP and sandbox permission metadata with secure defaults, but the iframe host ignores those fields and applies only a hardcoded `navigate-to`/`form-action` CSP while granting bridge capabilities. Parse the resource metadata and translate it into iframe CSP/permissions so app HTML gets only the network, resource, frame, and permission surface it requested and the host approved.
  - confidence: 0.86
