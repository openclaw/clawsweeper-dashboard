## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Stop persisting MCP session and tool payload data:** `src/agents/mcp-ui-resource.ts:96-99`
  - body: `mcpApp` includes the live session metadata from the canvas view and then appends the full tool input/result before serializing the canvas JSON. That data becomes chat/tool metadata, so secrets or session-scoped bridge data can outlive the live UI and appear in logs, history, or exports; keep the bridge session in UI memory and persist only minimal non-sensitive display metadata.
  - confidence: 0.93
- **[P1] Bound MCP App resources before decode and write:** `src/agents/mcp-ui-resource.ts:49-53`
  - body: A server-provided blob is decoded straight into a UTF-8 string and then written as a canvas document with no MCP Apps-specific byte cap. A trusted but buggy or compromised MCP server can return a very large resource and force avoidable memory/disk growth before any UI sandbox boundary is reached.
  - confidence: 0.9
- **[P1] Apply resource CSP and permissions metadata:** `ui/src/ui/components/mcp-app-view.ts:249-250`
  - body: The iframe is created with a fixed CSP that only blocks navigation and form actions, while `@modelcontextprotocol/ext-apps` defines resource CSP and permissions metadata with secure defaults. Without applying that metadata, apps can make undeclared network/resource requests or fail in ways the server explicitly declared.
  - confidence: 0.88
