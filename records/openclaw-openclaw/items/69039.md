Full review comments:

- **[P1] Cap MCP App resources before decode and persistence:** `src/agents/mcp-ui-resource.ts:49-54`
  - body: When `mcp.apps.enabled` is true, a configured MCP server can return arbitrary `text` or base64 `blob` content here. The PR decodes it and persists it as canvas HTML without a byte cap, so a broken or hostile trusted server can exhaust memory or disk; reject oversized resources before decode/write and cover the limit in tests.
  - confidence: 0.91
- **[P1] Enforce MCP Apps CSP through the sandbox proxy:** `ui/src/ui/components/mcp-app-view.ts:249-250`
  - body: The MCP Apps host contract requires web hosts to load raw app HTML through a different-origin sandbox proxy and enforce CSP from `_meta.ui.csp` with restrictive defaults.   - confidence: 0.88
