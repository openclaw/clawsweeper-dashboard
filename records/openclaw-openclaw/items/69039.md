## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Keep MCP App payload metadata out of stored canvas JSON:** `src/agents/mcp-ui-resource.ts:96-99`
  - body: `buildMcpAppCanvasJson` writes the session routing key plus raw tool input/result into the canvas block that is then preserved in chat/canvas metadata. That makes restored/exported messages carry bridge routing and potentially sensitive tool payloads beyond the live iframe lifetime; keep the session key in UI context and store only non-sensitive view metadata, or add an explicit redacted persistence contract.
  - confidence: 0.88
- **[P1] Revalidate MCP bridge runtimes against current config:** `src/gateway/server-methods/mcp-app-proxy.ts:29-31`
  - body: `resolveRuntime` returns a cached runtime as soon as a session mapping exists, so it skips the current config check before serving app bridge calls. If an operator disables `mcp.apps.enabled` or removes/rotates a server while an app view is open, the old runtime can continue handling `mcp.*` calls; verify the cached runtime against the latest config or route through a current-config owner path before use.
  - confidence: 0.86
- **[P1] Bind restored MCP Apps to canonical session runtime ownership:** `src/gateway/server-methods/mcp-app-proxy.ts:38-40`
  - body: When no mapping exists, restored app calls create a new runtime under `mcp-app:${sessionKey}`. Later agent turns use their normal session id and cleanup/dispose paths target that id, so the restored app can run a second MCP process/config snapshot outside the canonical session lifecycle; use the canonical session identity or a single owner-managed restore path.
  - confidence: 0.84
- **[P2] Cap MCP App resource size before decoding and persisting:** `src/agents/mcp-ui-resource.ts:49-53`
  - body: The UI resource fetch decodes text/blob into a string and immediately writes it to a canvas document with no byte limit. A configured server returning a huge MCP App resource can burn memory/disk in the agent path; enforce a host limit before base64 decode and before `createCanvasDocument`, with a rejection test.
  - confidence: 0.8
