## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Stop persisting MCP session and tool payload data:** `src/agents/mcp-ui-resource.ts:91-97`
  - body: `params.view.mcpApp` already carries `sessionKey`, and this adds `toolInput` plus the full `toolResult` to the canvas JSON that is stored and later rendered from chat/canvas state. That can retain session routing data and sensitive tool payloads in history, logs, exports, or restored previews; keep the session key in live UI context and persist only a redacted/minimal app view descriptor.
  - confidence: 0.93
- **[P1] Bound MCP App resources before decoding or writing:** `src/agents/mcp-ui-resource.ts:48-69`
  - body: A server-provided `blob` is decoded with `Buffer.from(..., "base64")` and arbitrary HTML is then written through `createCanvasDocument` with no byte cap. A trusted but buggy or compromised MCP server can exhaust memory or grow the state canvas store during a normal enabled tool call, so reject oversized text/base64 before decode and before persistence.
  - confidence: 0.9
- **[P1] Enforce MCP App resource CSP and permissions:** `ui/src/ui/components/mcp-app-view.ts:240-242`
  - body: The MCP Apps dependency exposes resource metadata for CSP and permissions, but the host discards that metadata and renders every app iframe with a fixed `navigate-to`/`form-action` string and no permission-policy `allow` mapping. This weakens the browser boundary for server-provided HTML; parse the resource `_meta`, apply a strict default policy, and map requested permissions explicitly.
  - confidence: 0.84
- **[P2] Keep restored app runtimes under canonical session ownership:** `src/gateway/server-methods/mcp-app-proxy.ts:33-39`
  - body: When the session-key mapping is missing, `resolveRuntime` invents a `mcp-app:${sessionKey}` runtime id. Restored app views can then start MCP processes outside the normal run/session id that later turns and cleanup own, making runtime disposal and config changes harder to reason about; route restored views through a canonical session id or a dedicated lifecycle owner with tests.
  - confidence: 0.78
