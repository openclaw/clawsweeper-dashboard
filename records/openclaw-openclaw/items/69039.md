## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep MCP App payload metadata out of stored canvas JSON:** `src/agents/mcp-ui-resource.ts:96-99`
  - body: `buildMcpAppCanvasJson` writes `view.mcpApp` plus raw tool input/result into the canvas JSON that is preserved through chat/canvas metadata. Since `view.mcpApp` can include the session routing key, restored/exported messages can carry bridge routing and sensitive tool payloads beyond the live iframe lifetime; keep the session context live-only and store only non-sensitive view metadata or an explicit redacted contract.
  - confidence: 0.89
- **[P2] Bind restored MCP Apps to canonical session ownership:** `src/gateway/server-methods/mcp-app-proxy.ts:34`
  - body: When there is no existing session mapping, the bridge creates a runtime under `mcp-app:${sessionKey}`. Later normal agent turns can bind the same session key to a different session id, leaving the synthetic MCP runtime outside canonical session cleanup until idle eviction; route restored views through a single owner-managed session identity or adopt/dispose the synthetic runtime.
  - confidence: 0.85
- **[P2] Cap MCP App resources before decoding and writing:** `src/agents/mcp-ui-resource.ts:49-53`
  - body: The resource fetch accepts arbitrary text or base64 from the MCP server and passes the decoded HTML to `createCanvasDocument` with no byte cap. A configured or compromised server can exhaust memory or disk through the app view path; enforce a host limit before base64 decode and before persistence, with a rejection test.
  - confidence: 0.83
- **[P2] Render disabled MCP Apps through the canvas fallback:** `ui/src/ui/chat/tool-cards.ts:478-479`
  - body: When a stored canvas preview has `mcpApp` metadata but `mcpAppsEnabled` is false, this returns `nothing`, so existing app canvases disappear instead of rendering as the normal sandboxed iframe fallback promised by the PR. Use the existing `renderPreviewFrame` path when the bridge is disabled.
  - confidence: 0.82
