## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Cap MCP App resources before decoding and writing:** `src/agents/mcp-ui-resource.ts:49-53`
  - body: `readResource` can return arbitrarily large `text` or base64 `blob` content, and this path decodes it and writes the resulting HTML into a canvas document with no host byte cap. A configured or compromised MCP server can force large memory allocation and persistent state-dir writes when apps are enabled; reject oversized payloads before decode and before `createCanvasDocument`.
  - confidence: 0.94
- **[P1] Enforce MCP App CSP for rendered HTML:** `ui/src/ui/components/mcp-app-view.ts:249-250`
  - body: The iframe only gets the sandbox plus `navigate-to`/`form-action`, but the MCP Apps spec requires a restrictive default CSP and declared-domain allowlist. With scripts enabled, app HTML can load or connect to undeclared origins through the Control UI/canvas host path; serve MCP app HTML through a per-resource CSP path or sandbox proxy before merge.
  - confidence: 0.95
