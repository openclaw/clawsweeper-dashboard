## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Keep MCP session capability out of persisted app metadata:** `src/agents/mcp-ui-resource.ts:66`
  - body: This stores the runtime sessionKey in the canvas manifest, and the same MCP app JSON later carries full toolInput/toolResult. That durable chat/canvas metadata is then used by the UI/gateway proxy as the session selector, so logs, exports, or restored message state can retain a capability plus sensitive tool payloads. Keep the session binding server-side or in-memory and persist only redacted display metadata.
  - confidence: 0.86
- **[P1] Enforce the MCP app CSP before rendering HTML:** `ui/src/ui/components/mcp-app-view.ts:250`
  - body: The iframe only sets a narrow csp attribute, while the canvas host serves the stored MCP app HTML without a Content-Security-Policy header. The ext-apps contract says hosts enforce declared connect/resource/frame domains and default omitted domains closed, so enabled apps can currently load arbitrary browser network and resource origins. Persist the resource CSP and emit enforced headers or an equivalent policy before rendering.
  - confidence: 0.8
- **[P2] Bound MCP UI resource bytes before decoding and writing:** `src/agents/mcp-ui-resource.ts:53`
  - body: A configured MCP server can return an arbitrary-size text body or base64 blob here; the code decodes it into memory and writes it into the canvas document without a byte cap. A buggy or hostile server can exhaust process memory or state disk just by serving a huge app resource. Reject oversized text/blob payloads before decode and before createCanvasDocument writes them.
  - confidence: 0.88
- **[P2] Require the requested MCP UI resource URI exactly:** `src/agents/mcp-ui-resource.ts:36`
  - body: When the readResource response lacks the requested URI, this falls back to contents[0] and can render a different HTML resource than the tool advertised. That breaks the tool-to-resource binding later used for app-view authorization. Reject responses that do not contain an exact uiResourceUri match.
  - confidence: 0.82
