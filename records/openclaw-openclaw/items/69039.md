## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Cap MCP App resources before decode and persist:** `src/agents/mcp-ui-resource.ts:49-53`
  - body: `fetchMcpAppView` decodes server-provided `text` or base64 `blob` directly into a string and then passes the full HTML into canvas persistence. With `mcp.apps.enabled=true`, a configured MCP server can return an arbitrarily large UI resource and drive memory or disk exhaustion, so this needs a host-side byte cap before decode and before writing the canvas document.
  - confidence: 0.92
- **[P2] Render disabled MCP App previews through the iframe fallback:** `ui/src/ui/chat/tool-cards.ts:478-479`
  - body: This early return drops the whole persisted preview when MCP Apps are disabled, so a saved canvas URL renders as nothing instead of using the existing `renderPreviewFrame` fallback below. That contradicts the default-off fallback behavior described by the PR and breaks restored chats for users who later disable the feature.
  - confidence: 0.88
