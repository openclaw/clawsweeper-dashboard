Overall correctness: patch is incorrect

Full review comments:

- **[P1] Cap MCP App resources before decode and persistence:** `src/agents/mcp-ui-resource.ts:49-54`
  - body: When `mcp.apps.enabled` is true, a configured MCP server can return arbitrary `text` or base64 `blob` content here. The PR decodes it and persists it as canvas HTML without a byte cap, so a broken or hostile trusted server can exhaust memory or disk; reject oversized resources before decode/write and test the limit.
  - confidence: 0.91
- **[P1] Enforce MCP Apps CSP for rendered HTML:** `ui/src/ui/components/mcp-app-view.ts:249-250`
  - body: The iframe loads the persisted app page directly and only sets `navigate-to`/`form-action`, but the MCP Apps web-host contract requires a sandbox proxy plus restrictive default and declared-domain CSP enforcement. Without that, enabled app HTML can load or connect to undeclared origins through the Control UI host path.
  - confidence: 0.88
