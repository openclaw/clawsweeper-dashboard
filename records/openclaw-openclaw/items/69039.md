## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Stop persisting MCP session and tool payload data:** `src/agents/mcp-ui-resource.ts:96-99`
  - body: `buildMcpAppCanvasJson` serializes the view metadata plus full `toolInput` and `toolResult`; because the view metadata already includes `runtime.sessionKey`, this stores a live session-linked capability and sensitive tool payloads in chat/canvas data instead of keeping them in live UI context.
  - confidence: 0.9
- **[P1] Bound MCP App resources before decoding or writing:** `src/agents/mcp-ui-resource.ts:49-54`
  - body: The resource body is accepted as arbitrary text or base64 and decoded before any explicit byte cap, then written as a canvas HTML entrypoint. The MCP SDK schema only promises strings here, so a server can drive large memory/disk writes when MCP Apps are enabled.
  - confidence: 0.88
- **[P1] Apply resource CSP and iframe permissions:** `ui/src/ui/components/mcp-app-view.ts:249-250`
  - body: The iframe is created with a fixed sandbox/CSP and no `allow` mapping, but MCP Apps resources carry `_meta.ui` CSP and permissions metadata and the dependency exposes `buildAllowAttribute`. Ignoring that contract can either block valid apps or under-enforce server-declared browser policy.
  - confidence: 0.86
- **[P2] Keep restored app runtimes under canonical sessions:** `src/gateway/server-methods/mcp-app-proxy.ts:31-42`
  - body: When a session key is not mapped after restart, the proxy creates a synthetic `mcp-app:${sessionKey}` runtime. That lets restored views own MCP transports outside the normal run/session id lifecycle, so cleanup and session association are no longer the same path as live agent runs.
  - confidence: 0.78

## Risks / Open Questions

- Merging the current branch can persist live MCP session keys and full tool input/result payloads into canvas manifests and chat metadata, which expands the sensitive-data surface beyond the live UI bridge.
- Enabled MCP Apps can cause OpenClaw to decode and write server-provided HTML without an explicit byte cap, creating a resource-exhaustion path for trusted but buggy or compromised MCP servers.
- The iframe host ignores dependency-declared resource CSP and permissions metadata, so green CI would not prove the browser boundary matches the MCP Apps contract.
- Restored app views can recreate MCP runtimes under synthetic session ids when the canonical session mapping is gone, which risks stale or mis-associated session state after restart.
- The branch is draft and currently merge-conflicting, so maintainer review should wait for an updated branch after the security/session fixes.
