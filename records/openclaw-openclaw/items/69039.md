## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Cap MCP App resources before decoding and writing:** `src/agents/mcp-ui-resource.ts:49-53`
  - body: `fetchMcpAppView` accepts MCP-server `text` or base64 `blob` content, decodes it into a string, and passes it to canvas persistence without any byte limit. With `mcp.apps.enabled=true`, a configured server can return a huge app resource and force OpenClaw to allocate memory and write it to disk; add a strict host-side cap before base64 decode and before `createCanvasDocument`, with oversized text/blob tests.
  - confidence: 0.91
- **[P1] Render disabled MCP App previews through the canvas frame:** `ui/src/ui/chat/tool-cards.ts:478-480`
  - body: This early return hides every persisted canvas preview that carries `mcpApp` metadata when the feature is disabled. The PR says disabled MCP Apps should fall back to the existing canvas/embed path, and current main renders canvas previews as ordinary iframes, so disabling the option or reopening history silently removes assistant content instead of showing the saved canvas document.
  - confidence: 0.88
