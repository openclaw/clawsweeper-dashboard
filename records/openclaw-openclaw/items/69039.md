Overall confidence: 0.8

Full review comments:

- **[P1] Keep bridge credentials out of persisted canvas metadata:** `src/agents/mcp-ui-resource.ts:55-56`
  - body: This stores `sessionKey` in the MCP app metadata that is serialized into the canvas/chat payload and later replayed into the UI, alongside tool input and result data. A bridge/session capability should not be persisted with assistant content; use a short-lived runtime lookup or another non-stored handle so transcripts do not retain callable app credentials and sensitive tool payloads.
  - confidence: 0.88
- **[P1] Revalidate MCP app runtimes before proxying bridge calls:** `src/gateway/server-methods/mcp-app-proxy.ts:29-38`
  - body: Returning `manager.getExisting(sessionId)` before checking the latest MCP config lets an open app keep using removed or rotated MCP servers, and the fallback `mcp-app:${sessionKey}` runtime is not tied to the normal run cleanup identity. Bridge calls need to resolve through the current session/runtime contract and revalidate the current config before tools/resources stay callable.
  - confidence: 0.84
- **[P1] Move the implementation onto the current agent-bundle runtime:** `src/agents/pi-bundle-mcp-runtime.ts:1`
  - body: This PR still adds MCP Apps behavior to the old `pi-bundle-*` runtime files, but current main owns this area under `agent-bundle-*`. Landing the stale surface would either miss the canonical runtime path or require a conflicted merge, so the feature needs to be rebased and reviewed against the current runtime/module split.
  - confidence: 0.79
- **[P2] Enforce or reject MCP Apps resource CSP before rendering HTML:** `ui/src/ui/components/mcp-app-view.ts:95`
  - body: The MCP Apps dependency contract treats resource CSP metadata as host-enforced security input, but this path renders app HTML through the generic iframe bridge without a reviewed policy for declared domains, network access, or rejected metadata. The host should either enforce the declared CSP/resource policy or explicitly reject unsupported app resources before exposing the bridge.
  - confidence: 0.76
