## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Stop serializing MCP session keys and raw payloads:** `src/agents/mcp-ui-resource.ts:96-99`
  - body: `mcpApp` is written into chat/canvas JSON with the persisted view metadata, so the spread can carry `sessionKey` and these lines add raw `toolInput`/`toolResult`. That turns a runtime/session capability and potentially sensitive tool data into stored chat/canvas/export/log material instead of keeping it in the in-memory UI context or a narrow server-side authorization token.
  - confidence: 0.9
- **[P1] Bound MCP App resource size before decode and persistence:** `src/agents/mcp-ui-resource.ts:49-54`
  - body: A server can return an arbitrary-size text resource or base64 blob, and this path decodes then persists it as a canvas document without any raw or decoded byte cap. Because MCP resource contents have no type-level size bound, enabling MCP Apps lets a buggy or hostile trusted server exhaust memory or disk during tool materialization.
  - confidence: 0.87
- **[P1] Enforce resource-declared MCP Apps CSP:** `ui/src/ui/components/mcp-app-view.ts:249-250`
  - body: The iframe CSP only blocks navigation/forms, while the MCP Apps contract puts connect/resource/frame/base URI policy on the UI resource metadata with secure defaults when omitted. Ignoring that metadata gives enabled app HTML broader browser fetch/embed/resource capability than the server-declared policy and host security model allow.
  - confidence: 0.84
