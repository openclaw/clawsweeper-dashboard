## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Do not persist MCP app bridge credentials:** `src/agents/mcp-ui-resource.ts:96-100`
  - body: This JSON is persisted with the chat/canvas preview, but it spreads `view.mcpApp`, which already includes `runtime.sessionKey`, and then adds the full tool input/result. That session key is later sent back to Gateway for `mcp.*` bridge calls, so transcripts, canvas manifests, exports, or same-origin UI readers can get a reusable bridge capability plus sensitive tool payloads. Keep the bridge binding server-side or in live UI context and persist only non-sensitive view identity.
  - confidence: 0.9
- **[P2] Bound MCP app HTML before materializing it:** `src/agents/mcp-ui-resource.ts:49-53`
  - body: The new path accepts arbitrary `text` or base64 `blob` content from an MCP resource, decodes it into a string, and then writes it as a canvas document. A trusted but buggy or compromised MCP server can force large memory allocation and disk writes; reject oversized UI resources before decoding/persisting and add a focused limit test.
  - confidence: 0.86
- **[P2] Render disabled MCP app previews through the fallback:** `ui/src/ui/chat/tool-cards.ts:478-480`
  - body: When `mcp.apps.enabled` is false, this returns `nothing` for any stored MCP app canvas preview. Current main renders assistant-message canvas previews as ordinary sandboxed iframes, and the PR body promises that fallback when MCP Apps are disabled; with this branch, turning the option off or reloading history makes those views disappear. Disable only the app bridge and fall back to `renderPreviewFrame`.
  - confidence: 0.93
- **[P2] Constrain app-initiated resource reads:** `src/gateway/server-methods/mcp-app-proxy.ts:203`
  - body: After a view is authorized, `mcp.readResource` forwards any app-supplied URI to the server. The dependency expects hosts to mediate `resources/read`, so OpenClaw should bind reads to listed/app-visible resources or the rendered UI resource scope; otherwise enabled app HTML can ask the user's MCP session to read unrelated resources from the same server.
  - confidence: 0.78
