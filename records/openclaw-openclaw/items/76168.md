## Summary

Keep open. Current main still performs strict configured-channel plugin loading in the message CLI before the action's normal error/exit wrapper, so a `PluginLoadFailureError` can bypass the tested `exit(1)` path; the related gateway-delegation PR remains open and does not prove this plugin-load failure is fixed.

## What This Changes

Make `openclaw message send` exit cleanly or load bundled runtime deps correctly when Telegram/plugin loading fails, instead of leaving an `openclaw-message` child spinning.

## Best Possible Solution

Keep this open until message CLI plugin-registry failures become a bounded, observable nonzero exit, and avoid local Telegram runtime loading for gateway-owned sends if the delegation path is available.

## Reproduction Assessment

No: the report has detailed production logs and process evidence, and current-main source shows a plugin-load exception path before the clean exit wrapper, but I did not establish a deterministic local Telegram/runtime-deps repro.

## Solution Assessment

Yes for the narrow first repair: route `ensurePluginRegistryLoaded` failures through the same nonzero exit/finalization path as message delivery failures, while coordinating any broader gateway fast path with #75478 and timeout policy with #75895.

## Repair Work Prompt

Fix the message CLI plugin-load failure path for https://github.com/openclaw/openclaw/issues/76168. Current main calls `ensurePluginRegistryLoaded(...)` in `src/cli/program/message/helpers.ts` before `runCommandWithRuntime`, so strict `PluginLoadFailureError` exceptions can bypass the message action's `exit(1)` handling and finalization. Make `openclaw message send` fail fast with a clear nonzero exit when configured-channel plugin loading fails, without leaving an `openclaw-message` process alive. Keep the scope to CLI message action/plugin-load failure handling; do not add a new timeout flag/config or broad runtime-deps policy. Coordinate with the open gateway-delegation work in #75478/#75477 so a Telegram gateway fast path is not duplicated.
