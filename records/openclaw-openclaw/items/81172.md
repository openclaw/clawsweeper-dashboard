## Summary

Keep open: the report is current, specific, and consistent with current memory-core search paths that still run long memory search work inside the gateway request path. The available evidence is strong enough to route the issue, but not enough to close or auto-repair without live reproduction and a maintainer decision on timeout/isolation semantics.

## What This Changes

Prevent memory-core `memory_search` from blocking the event loop during Discord DMs so the gateway stays responsive and slow searches fail gracefully.

## Best Possible Solution

Define a bounded, non-blocking `memory_search` execution contract in memory-core, isolate or avoid synchronous search/indexing work on the gateway event loop, and add regression proof that a slow memory search cannot starve channel I/O.

## Solution Assessment
