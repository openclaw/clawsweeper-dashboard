## Summary

Keep this PR open. The PR targets a real fallback/replay transcript cleanup gap, but the latest head still makes the new helper too broad: it strips any trailing empty assistant turn, while current main explicitly preserves nonzero-usage clean silent replies as valid transcript state.

## What This Changes

The PR adds a shared trailing-empty-assistant transcript repair helper, applies it before fallback/replay and BTW snapshots, and updates regression tests, transcript-hygiene docs, and the changelog.

## Best Possible Solution

Keep the in-memory fallback tail cleanup at the embedded-runner boundary, but narrow it to the existing replay contract: remove failed error/sentinel tails and zero-usage empty-stop false successes while preserving nonzero-usage clean silent replies and other non-error empty assistant turns.

## Reproduction Assessment

Yes. A focused helper test with `[user, assistant stop content: [] usage: nonzero]` reproduces the regression: the PR strips the assistant tail, while current-main replay-history coverage preserves that shape as a valid silent reply.

## Solution Assessment

No. The shared helper and call sites are the right shape, but the predicate should reuse or mirror the zero-usage/error/sentinel contract instead of dropping every empty assistant tail.
