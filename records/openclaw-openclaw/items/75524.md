## Summary

This PR should stay open: the updated branch fixes the prior silent-reply regression, but the new sentinel predicate can still drop a trailing assistant turn that contains real content, and the external-PR real behavior proof gate is still failing.

## What This Changes

The branch adds a transcript repair helper, wires it into embedded-runner fallback/replay submission, exports the zero-token usage predicate, and updates tests, docs, and the changelog.

## Best Possible Solution

Keep the in-memory fallback cleanup only after narrowing it to the same sentinel-only, error, and zero-usage empty-stop contract as current replay normalization, then add real runtime fallback proof before merge.

## Reproduction Assessment

Yes for the review finding: source inspection gives a high-confidence reproduction with a trailing assistant content array that includes the stream-error sentinel plus a real content block, because the PR's Array.some() predicate strips the whole turn. The original provider failure is source-supported by current replay-tail fixes and related reports, but this review did not run a live LiteLLM/Vertex fallback.

## Solution Assessment

No. The embedded-runner boundary is reasonable, but the helper should reuse or mirror current main's narrower droppable-tail rules instead of stripping any assistant turn that merely contains the sentinel, and the PR still needs real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Require sentinel-only content before stripping:** `src/agents/session-transcript-repair.ts:393-403`
  - body: `containsStreamErrorSentinel` returns true when any block equals the fallback sentinel, and `stripTrailingEmptyAssistantTurn` removes the whole trailing assistant before checking whether the rest of the content is usable. That can delete a real assistant turn that includes the sentinel string alongside actual text or tool content; the existing replay trim on main requires the sentinel-only shape/provenance for this reason. Please make this predicate require the whole content to be the synthetic empty/error shape and add mixed-content coverage.
