## Summary

Keep this PR open, but do not merge it as-is. Current main still has the fallback transcript-tail gap, and the PR targets the right boundary, but the new helper is broader than the stated failed-tail repair and would also remove clean silent assistant replies that existing current-main contracts preserve.

## What This Changes

The PR adds a shared trailing-empty-assistant transcript repair helper, calls it in embedded-runner fallback/replay paths, aligns BTW snapshots, and adds regression tests, transcript-hygiene docs, and a changelog entry.

## Best Possible Solution

Keep the in-memory fallback cleanup, but narrow the predicate to failed or false-success fallback tails: error/aborted turns, zero-usage empty stops, or the stream-error sentinel. Preserve current-main valid silent replies and add a regression test for a nonzero-usage clean stop assistant tail before landing with green exact-head checks.

## Reproduction Assessment

Yes. A focused unit test passing a `[user, assistant stop content: [] usage: nonzero]` transcript to the new helper would reproduce the regression: the PR strips it, while current replay-history coverage preserves that shape as a legitimate silent reply.

## Solution Assessment

No. The shared helper and embedded-runner boundary are the right shape, but the current predicate is not the narrowest maintainable fix because it removes valid clean silent assistant history in addition to failed fallback tails.
