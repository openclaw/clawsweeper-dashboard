## Best Possible Solution

Keep the in-memory fallback cleanup, but make its sentinel predicate mirror current main's exact sentinel-only plus synthetic-provenance guard and add mixed-content and real-sentinel preservation tests.

No. The embedded-runner cleanup seam is reasonable, but this implementation should require exact sentinel-only synthetic provenance before deleting a trailing assistant turn.
