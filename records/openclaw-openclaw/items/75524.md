## Summary

Keep open. The bug boundary is credible and not fully implemented on current main, but this PR is not merge-ready because it strips valid silent assistant replies that current replay and incomplete-turn contracts deliberately preserve.

## What This Changes

The PR adds a shared helper that removes trailing empty/error assistant messages from embedded-runner in-memory replay before fallback/prompt submission, updates BTW snapshots, docs, changelog, and focused tests.

## Best Possible Solution

Keep the in-memory fallback cleanup, but narrow the predicate to failed error/sentinel tails and zero-usage empty-stop false successes while preserving allowed silent replies, then land with focused regression tests plus the existing docs/changelog coverage.

## Reproduction Assessment

Yes. A focused helper test with a trailing assistant `stop` turn using `content: []` plus nonzero usage, or a blank-text clean assistant turn allowed as silent, reproduces the regression because the PR strips it while current-main replay and incomplete-turn tests preserve it.

## Solution Assessment

No. The helper and embedded-runner boundary are plausible, but the predicate is broader than the current transcript contract; reusing the existing error/sentinel and zero-usage-empty-stop boundary is the safer maintainable fix.
