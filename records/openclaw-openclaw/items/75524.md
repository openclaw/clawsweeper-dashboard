## Summary

Keep open. The latest head still has a blocking predicate bug: it strips valid clean silent assistant tails, while current main explicitly preserves those turns unless they are error or zero-usage false-success stops.

## What This Changes

This PR adds a shared transcript helper to strip trailing empty assistant turns before fallback/replay, wires it into embedded-runner prompt submission and BTW snapshots, and adds tests, docs, and a changelog entry.

## Best Possible Solution

Keep the in-memory fallback cleanup, but narrow it to failed error/sentinel tails and zero-usage empty-stop false successes, preserving clean silent replies and adding focused regression coverage.

## Reproduction Assessment

Yes. A focused helper test with a trailing assistant stop turn using content: [] plus nonzero usage, or blank text content allowed as silent, reproduces the regression because the PR strips it while current-main replay and incomplete-turn tests preserve it.

## Solution Assessment

No. The helper and embedded-runner boundary are a reasonable shape, but the predicate should mirror the current error/zero-usage/sentinel contract instead of treating every empty assistant tail as failed.
