## Summary

Keep open. The PR targets a real fallback transcript hygiene gap, but the latest head still strips every trailing `content: []` assistant turn and conflicts with the current silent-reply contract on main.

## Best Possible Solution

Keep the embedded-runner cleanup shape, but narrow the predicate to failed error/sentinel tails and zero-usage empty-stop false successes while preserving clean silent replies.
