## Summary

Keep this PR open. The in-memory fallback transcript gap still exists on current main and the PR mostly addresses the right boundary, but the latest head still leaves the active-run /btw snapshot built from the pre-strip replay array, so side questions can inherit the failed assistant tail that the prompt path just removed.

## What This Changes

The PR adds a shared trailing-empty-assistant transcript repair helper, applies it before fallback and after replay normalization in embedded attempts, and updates regression tests, transcript-hygiene docs, and the changelog.

## Best Possible Solution

The fallback prompt path and active-run /btw snapshot should both derive from the same tail-cleaned in-memory message array, while leaving the persisted session-file repair from #75606 as the complementary on-disk cleanup.

## Reproduction Assessment

Yes. Current main can normalize an empty/error assistant replay turn into the stream-error sentinel immediately before prompt submission and has no later tail strip before activeSession.prompt; the PR's helper tests cover the relevant transcript shapes.

## Solution Assessment

Yes, with one narrow consistency follow-up. A shared helper at the embedded-runner boundary and after replay normalization is the maintainable fix, but the active-run snapshot should use activeSession.messages or replaySafeMessages after the strip.
