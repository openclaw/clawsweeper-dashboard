## Summary

The requested core config override should close because channel labels are already a plugin-owned metadata surface, and the concrete WeChat label problem has an external plugin PR that changes the plugin metadata directly. Adding a generic `channels.<id>.label` core override would create a second source of truth for a display concern that the plugin/ClawHub package should own.

## What This Changes

Request to let channel plugins expose a user-configurable `channels.<channelId>.label`/display name for session and UI display.

## Best Possible Solution

Keep channel display labels owned by plugin metadata and have the WeChat package publish the friendly label through its own release/ClawHub listing rather than adding a generic core override.

## Reproduction Assessment

Not applicable: this is a feature request, and source inspection confirms the requested core override does not currently exist as a supported contract. The concrete WeChat label mismatch is reproducible from the linked external plugin patch context rather than an OpenClaw core bug.

## Solution Assessment

No: adding a core `channels.<id>.label` override is not the best boundary because plugin metadata and `channelConfigs` already own channel labels. The narrower maintainable path is to fix the external WeChat plugin metadata and ClawHub/catalog entry.
