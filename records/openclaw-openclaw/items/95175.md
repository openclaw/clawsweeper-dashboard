## Summary

## What This Changes

The PR propagates an initiating Gateway device id into chat-triggered exec approval requests, binds reviewer-device visibility for trusted approval runtimes, and handles iOS foreground approval events with focused tests.

## Best Possible Solution

Thread the reviewer device id through every OpenClaw tool construction surface, including Codex app-server dynamic tools, add focused regression coverage for that sibling runtime, and keep the trusted-runtime plus operator.approvals gate as the boundary model.
