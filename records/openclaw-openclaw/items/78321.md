## Summary

Close as implemented: current main and the v2026.5.2+ release line no longer match the reported 2026.4.21 failure because Feishu is externalized from the root package, owns its SDK dependency, and disabled plugin paths now short-circuit before full runtime import.

## What This Changes

The issue asks for vanilla installs to stop loading disabled Feishu before its runtime dependency is present, and for `openclaw plugins disable feishu` to work instead of requiring manual config edits.

## Best Possible Solution

Keep the shipped externalized Feishu/package-loader design; users on 2026.4.21 should upgrade to v2026.5.2 or later, while https://github.com/openclaw/openclaw/pull/76974 can continue only as warning UX polish.

## Reproduction Assessment

No high-confidence current-main failure path remains: ordinary disabled plugins short-circuit before runtime import, Feishu is no longer bundled into the root package, and the setup entry is tested not to import `@larksuiteoapi/node-sdk`. The report is credible for 2026.4.21, but it no longer matches the current source or v2026.5.2+ package boundary.

## Solution Assessment

Yes, the current solution is maintainable: externalize Feishu so its package owns the SDK dependency, keep setup discovery lightweight, and honor disabled state before full runtime loading. Mirroring the SDK back into the root package is no longer the best direction for the current plugin model.
