## Summary

Current main and shipped releases no longer expose the reported half-bundled Feishu state: Feishu is excluded from the core npm package, installable as `@openclaw/feishu`, and carries its own Lark SDK dependency.

## What This Changes

The issue reports that the bundled Feishu plugin in OpenClaw 2026.4.21 can crash on a missing `@larksuiteoapi/node-sdk` dependency and that `openclaw plugins disable feishu` imports the broken plugin before writing the disable flag.

## Best Possible Solution

Keep Feishu as an external official plugin with plugin-local dependencies and keep plugin lifecycle commands on metadata/config paths before runtime imports.

## Reproduction Assessment

Yes. The old 2026.4.21 failure path is high-confidence from this report plus related #70613 logs showing config/gateway loading `dist/extensions/feishu/client-*.js` before `@larksuiteoapi/node-sdk` was available; current main no longer ships that bundled Feishu dist path.

## Solution Assessment

Yes. The shipped external-plugin packaging path is the narrow maintainable solution because Feishu owns the Lark SDK dependency and core package installs no longer carry a partial Feishu runtime; loader and CLI paths also preserve disable-before-runtime behavior.
