## Summary

Keep open. The PR targets a valid current-main bug, but the current patch would hide existing actionable diagnostics, adds an unsafe warning-preview path, and lacks inspectable after-fix real behavior proof for an external PR.

## What This Changes

The PR changes `formatAssistantErrorText` so unrecognized assistant errors return generic user-facing copy instead of raw or truncated error text.

## Best Possible Solution

Refresh this PR or replace it with a narrow fallback hardening that only genericizes truly unclassified assistant errors, preserves current diagnostic carve-outs, and logs redacted, sanitized previews after safe truncation.

## Reproduction Assessment

Yes. By source inspection, current main returns raw or truncated unclassified assistant error text from `formatAssistantErrorText`, and `buildEmbeddedRunPayloads` can deliver that text as an `isError` reply.

## Solution Assessment

No. The direction is appropriate, but this patch is not the best final fix because its unconditional fallback hides current actionable diagnostics and its new warning preview is not safely redacted and sanitized before logging.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve actionable diagnostics before the fallback:** `src/agents/pi-embedded-helpers/errors.ts:659`
  - body: The new generic catch-all also catches plain provider/config diagnostics that current main keeps actionable, including proxy/TLS unsupported text and non-streaming JSON parse/request-validation errors. Add explicit classifiers or narrow the generic branch so those recovery clues do not become `/new` guidance.
  - confidence: 0.87
- **[P2] Redact and sanitize the preview before truncating:** `src/agents/pi-embedded-helpers/errors.ts:657-658`
  - body: The preview slices `raw` before redaction and only strips CR/LF, so a token near the 500-character boundary can miss default redaction patterns and ANSI/control bytes can still reach logs. Redact full or overlapped text first, truncate after, and pass the preview through `sanitizeForLog`.
  - confidence: 0.86
