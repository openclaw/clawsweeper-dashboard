## Summary

Keep open: the bug is source-reproducible on current main, but this external PR is still blocked by missing inspectable live proof, a conflicting branch, and remaining correctness/security review findings.

## What This Changes

The PR changes `formatAssistantErrorText` so unrecognized assistant errors return generic user-facing copy while logging a redacted preview, with formatter regression tests for the fallback.

## Best Possible Solution

Refresh or replace this with narrow fallback hardening that genericizes only truly unclassified assistant errors, preserves actionable diagnostics, logs redacted and sanitized previews, and includes redacted live channel proof.

## Reproduction Assessment

Yes, by source inspection. An assistant message with `stopReason: "error"` and an unclassified `errorMessage` flows through `formatAssistantErrorText`, then `buildEmbeddedRunPayloads` can push that text as a channel-visible `isError` reply.

## Solution Assessment

No. The direction is right, but this patch should preserve current diagnostic carve-outs, redact and pass log previews through `sanitizeForLog` before truncating, refresh against main, and provide inspectable real behavior proof before merge.

## Review Findings
