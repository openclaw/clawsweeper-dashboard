## Summary

Keep open. The reported raw-error user surface is still source-reproducible on current main, but this PR still needs contributor real behavior proof and has blocking correctness/security issues in the proposed fallback.

## What This Changes

The PR changes `formatAssistantErrorText` to replace the final raw unrecognized-error fallback with generic chat copy, add targeted carve-outs, log a redacted preview, and add formatter regression tests.

## Best Possible Solution

Refresh this PR or replace it with a narrow current-main patch that genericizes only truly unclassified user-facing assistant errors, preserves actionable diagnostics, logs redacted and sanitized previews, and includes redacted live channel proof.

## Reproduction Assessment

Yes, by source inspection. An assistant message with `stopReason: "error"` and an unclassified `errorMessage` can flow through `formatAssistantErrorText` unchanged and then be pushed as an `isError` reply.

## Solution Assessment

No. The direction is right, but the patch should preserve current diagnostic carve-outs, redact and sanitize before truncating log previews, and provide inspectable real behavior proof before merge.
