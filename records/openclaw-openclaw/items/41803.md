## Summary

Keep open: the PR addresses a source-reproducible raw assistant-error leak, but the current patch still needs inspectable real behavior proof plus fixes for diagnostic preservation and log-safe preview handling before merge.

## What This Changes

The PR changes `formatAssistantErrorText` so unrecognized assistant errors log a preview and return generic user-facing copy, with tests for orphaned tool-call and long raw errors.

## Best Possible Solution

Refresh this PR or replace it with a narrow fallback hardening that genericizes only truly unclassified assistant errors, preserves actionable diagnostics, logs redacted and sanitized previews, and includes redacted live proof.

## Reproduction Assessment

Yes by source inspection. Current main can send an assistant message with `stopReason: "error"` through `formatAssistantErrorText`, return an unclassified raw `errorMessage`, and deliver it as an `isError` reply.

## Solution Assessment

No. The direction is right, but the patch should preserve current diagnostic carve-outs and log an already redacted, sanitized preview before truncation; it also needs inspectable real behavior proof.
