## Summary

Keep open: the user-facing raw-error leak is still real on current main, but this PR should not merge yet because the new warning preview remains log-unsafe and the external PR lacks inspectable real behavior proof.

## What This Changes

The PR changes `formatAssistantErrorText` to return generic user copy for unrecognized assistant errors, preserve several known actionable error cases, log a redacted preview, and add formatter catch-all tests.

## Best Possible Solution

Keep the generic user-facing fallback, but redact and log-sanitize before truncation, preserve actionable known error branches, update affected lifecycle expectations, and require redacted real channel proof before merge.

## Reproduction Assessment

Yes. By source inspection, an assistant message with `stopReason: "error"` and an unclassified `errorMessage` can return raw text from `formatAssistantErrorText` and then be pushed as an `isError` delivery payload.

## Solution Assessment

No. The generic user-copy direction is the narrow fix, but the PR should redact and sanitize the warning preview before truncation and provide inspectable after-fix channel proof before normal merge review continues.
