## What This Changes

The PR changes `formatAssistantErrorText` so the final unrecognized-error fallback logs a redacted preview, returns a generic `/new` message, and adds formatter tests for orphaned tool-call and long raw errors.

## Best Possible Solution

Refresh or replace this with a current-main formatter/payload fix that genericizes truly internal unclassified provider errors, preserves actionable diagnostics, logs only redacted and sanitized previews, and includes redacted live channel proof.

## Solution Assessment

No. The direction is useful, but the patch needs to be refreshed against current main, preserve existing diagnostic carve-outs, fix log redaction/sanitization order, and provide inspectable real behavior proof before merge.
