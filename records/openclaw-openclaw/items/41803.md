## Summary

Keep open. The reported raw-error leak remains source-reproducible on current main, but this external PR is not ready to merge because it lacks inspectable real behavior proof and still has correctness/security blockers in the fallback/logging path.

## What This Changes

The PR changes `formatAssistantErrorText` so the unrecognized-error fallback logs a redacted preview, returns a generic `/new` message, and adds formatter tests for orphaned tool-call and long raw errors.

## Best Possible Solution

Refresh or replace this with a narrow current-main formatter fallback that genericizes truly internal unclassified provider errors, preserves actionable diagnostics, logs only redacted and sanitized previews, and includes redacted live channel proof.

## Reproduction Assessment

Yes, source-level. An assistant message with `stopReason: "error"` and an unclassified `errorMessage` flows through `formatAssistantErrorText`, and `buildEmbeddedRunPayloads` can push the returned text as a channel-visible `isError` reply.

## Solution Assessment

No. The direction is useful, but this patch should preserve current diagnostic carve-outs, redact and pass log previews through `sanitizeForLog` before truncating, refresh against main, and provide inspectable real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

  - body: Current main keeps non-transport proxy/config text and non-streaming JSON/request-validation diagnostics actionable. This final generic fallback would replace those strings with the `/new` copy unless the branch adds explicit classifiers or carve-outs before returning the generic message.

  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff adds a warning log for suppressed raw errors, but the preview is truncated before redaction and does not use the existing log sanitizer.

Concerns:

- **[medium] Preview truncates before redaction:** `src/agents/pi-embedded-helpers/errors.ts:657`
  - body: `redactSensitiveText(raw.slice(0, 500))` can leave a secret fragment unredacted when a quoted, JSON, header, or token-like value is cut before the pattern can match the complete sensitive field.
  - confidence: 0.82
- **[low] Preview leaves non-CRLF control characters:** `src/agents/pi-embedded-helpers/errors.ts:657`
  - body: The preview only replaces CR/LF, so tabs, ANSI escapes, DEL, and other control characters can still be interpolated into warning logs instead of going through `sanitizeForLog`.
  - confidence: 0.86
