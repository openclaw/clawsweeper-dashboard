## Summary

Keep open for revision rather than cleanup. Current main still has the source-reproducible raw assistant-error fallback this PR targets, but the proposed patch still hides existing actionable diagnostics, needs log-safety hardening, and lacks inspectable real behavior proof.

## What This Changes

The PR changes `formatAssistantErrorText` to replace the final raw unrecognized-error fallback with generic user-facing copy, add a few targeted carve-outs, log a redacted preview, and add formatter regression tests.

## Best Possible Solution

Refresh this PR or replace it with a narrow fallback hardening that genericizes only truly unclassified assistant errors, preserves actionable diagnostics, logs redacted and sanitized previews, and includes redacted live proof.

## Reproduction Assessment

Yes. By source inspection, an assistant message with `stopReason: "error"` and an unclassified `errorMessage` can flow through `formatAssistantErrorText` unchanged and then be pushed as an `isError` reply.

## Solution Assessment

No. The direction is right, but this patch should preserve current diagnostic carve-outs, redact and sanitize before truncating log previews, and provide inspectable real behavior proof before merge.
