  - body: The attempt result no longer includes `replayMetadata` or `itemLifecycle`, but the runner type and downstream completion/retry paths still depend on those fields. This can fail post-processing after a model call and breaks the current attempt-result contract.
  - confidence: 0.9

  - body: This now resolves bootstrap context unconditionally, bypassing the configured `continuation-skip` path that avoids reinjecting workspace context after a completed bootstrap turn. Continuation sessions can regain large context every turn and lose the intended overflow protection.
  - confidence: 0.88

  - body: The wrapper fetches a stored provider API key and passes it to `providerStreamFn`, which is provider/plugin-controlled code. That weakens the credential boundary and can disclose a secret to code that should only receive credentials through the approved resolver path.
  - confidence: 0.86
- **[P1] Preserve prompt contribution parameters:** `src/agents/system-prompt.ts:235`
  - body: Removing `includeMemorySection` and `promptContribution` from `buildAgentSystemPrompt` leaves sibling callers/tests that still pass those options and drops provider-owned prompt and memory-section controls. Keep the parameters or update every caller and preserve the contract deliberately.
  - confidence: 0.84
- **[P2] Normalize runtime capabilities before rendering:** `src/agents/system-prompt.ts:715-718`
  - body: The runtime line now joins capability strings without the existing normalization helper. Capability values can contain prompt-breaking characters, so this should keep the normalized capability-id path before rendering into system prompt text.
  - confidence: 0.8
