Full review comments:

- **[P1] Keep the prompt API contract with current callers:** `src/agents/system-prompt.ts:189-228`
  - body: `buildAgentSystemPrompt` no longer accepts `includeMemorySection` or `promptContribution`, but this branch still passes both from the embedded prompt wrapper and memory tests. This will fail type checking and also drops the context-engine/provider prompt controls those parameters carry.
  - confidence: 0.93
- **[P1] Return lifecycle metadata required by runner post-processing:** `src/agents/pi-embedded-runner/run/attempt.ts:1938-1965`
  - body: The attempt result returned here omits `replayMetadata` and `itemLifecycle`, while same-branch incomplete-turn logic dereferences them directly. Normal runs can therefore fail in post-processing or lose retry/replay safety decisions after the model call completes.
  - confidence: 0.9
- **[P1] Honor contextInjection before resolving bootstrap files:** `src/agents/pi-embedded-runner/run/attempt.ts:385-394`
  - body: This unconditionally resolves bootstrap context, bypassing the documented `agents.defaults.contextInjection` modes such as `continuation-skip` and `never`. Existing continuation sessions can start reinjecting workspace files every turn, increasing prompt size and defeating the skip behavior.
  - confidence: 0.88
- **[P1] Do not pass stored provider keys into plugin stream functions:** `src/agents/pi-embedded-runner/run/attempt.ts:242-247`
  - body: When a provider stream function is registered, this wrapper reads the stored provider API key and passes it in the options object to plugin-controlled stream code. That widens credential exposure beyond the central resolver and should be removed or replaced with an explicit trusted credential path.
  - confidence: 0.9
- **[P1] Prefer the resolved per-run credential for WebSocket transport:** `src/agents/pi-embedded-runner/run/attempt.ts:950-952`
  - body: The WebSocket path reads `authStorage.getApiKey(params.provider)` and ignores the already resolved per-run credential. Auth-profile rotation or ephemeral overrides can then connect the WebSocket transport with the wrong provider key.
  - confidence: 0.86
- **[P2] Preserve per-agent heartbeat prompt configuration:** `src/agents/pi-embedded-runner/run/attempt.ts:678-683`
  - body: Heartbeat prompt injection is now decided without config/agent identifiers and resolves only the default heartbeat prompt. Per-agent heartbeat overrides and flags such as disabling the system-prompt section can be ignored for embedded runs.
  - confidence: 0.84
- **[P2] Normalize runtime capabilities before rendering them:** `src/agents/system-prompt.ts:372-374`
  - body: The branch trims capability strings and then joins them directly into the runtime line. Restore strict capability normalization before prompt rendering so malformed capability values cannot inject line breaks or prompt text.
  - confidence: 0.8
- **[P2] Use runtime capability detection for native approvals:** `src/agents/system-prompt.ts:176-187`
  - body: The approval guidance hardcodes Discord, Slack, Telegram, and WebChat as native-approval channels, but current channel support is capability-driven. Channels with native approval capability outside that list will receive the wrong plain-chat `/approve` guidance.
  - confidence: 0.78
