  - confidence: 0.94

  - body: This now always calls `resolveBootstrapContextForRun`, while the existing path skips workspace bootstrap/context injection after a completed bootstrap turn when `agents.defaults.contextInjection` is `continuation-skip`. Users who opted into that compatibility path would get repeated workspace context injection again, with the prompt bloat and continuation behavior the setting is meant to avoid.
  - confidence: 0.9
- **[P1] Pass resolved runtime credentials into stream setup:** `src/agents/pi-embedded-runner/run/attempt.ts:950-967`
  - body: The WebSocket path fetches directly from `authStorage`, and the new inline `resolveEmbeddedAgentStreamFn` no longer accepts `params.resolvedApiKey`. That drops the per-run resolved credential override used for auth profiles or rotated credentials, so a provider or WebSocket call can run under the wrong key.
  - confidence: 0.9
- **[P1] Keep provider prompt and memory-section parameters:** `src/agents/system-prompt.ts:189-235`
  - body: `buildAgentSystemPrompt` no longer accepts `includeMemorySection` or `promptContribution`, but the embedded system-prompt wrapper still passes both and current runner code depends on them for context-engine memory gating and provider-owned prompt sections. This is both a compile-time mismatch and a behavior regression for provider/runtime prompt customization.
  - confidence: 0.91
- **[P2] Normalize runtime capabilities before rendering:** `src/agents/system-prompt.ts:700-718`
  - body: The previous `normalizePromptCapabilityIds` call was removed, so capability strings are joined directly into the Runtime line. If a capability source contains a newline or delimiter, it can break the one-line runtime format and inject unintended prompt text; restore the normalization before rendering.
