## Summary

Keep open: the PR targets a real duplicate-Off UI bug, but it is not merge-ready because the unconditional filter removes the explicit Off override for non-off defaults and the external PR still lacks after-fix running UI proof.

## What This Changes

The PR exports and changes the chat thinking option helper to filter normalized off/none entries, disables the chat thinking select when no options remain, and adds helper-level Vitest coverage.

## Best Possible Solution

Keep the duplicate-Off fix, but suppress explicit Off only when the inherited/default row already represents Off, preserving explicit Off for non-off defaults and adding render-level coverage for both cases.

## Reproduction Assessment

Yes, at source level: current main renders the inherited empty-value thinking row plus normalized dynamic options, and the linked issue supplies visual evidence from OpenClaw 2026.5.18. I did not run the UI because this review is read-only.

## Solution Assessment

No; the PR fixes the default-off duplicate by filtering Off unconditionally, but the narrower maintainable solution is to filter Off only when the inherited row already represents Off.

Full review comments:

- **[P1] Preserve explicit Off for non-off defaults:** `ui/src/ui/chat/session-controls.ts:293-294`
