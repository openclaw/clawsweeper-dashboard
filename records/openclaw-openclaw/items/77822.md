## What This Changes

The issue reports that repeated plugin registration can split closure state between hook handlers and tool handlers, breaking stateful plugins such as @mem0/openclaw-mem0.

## Best Possible Solution

Land a narrow resolver fix that uses the live registry for stateful hook-plus-tool plugins, with regression coverage and real plugin or gateway proof.
