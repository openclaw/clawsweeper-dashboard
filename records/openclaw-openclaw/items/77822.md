## Summary

Keep this issue open: current main and the latest release still have the cached descriptor tool path before live registry materialization, and the open closing PR is the active implementation candidate rather than a reason to close the issue early.

## What This Changes

The issue reports that repeated plugin `register()` calls split hook and tool closure state, breaking @mem0/openclaw-mem0 session and memory behavior.

## Best Possible Solution

Review and either land or explicitly supersede https://github.com/openclaw/openclaw/pull/78075; if it is rejected, document the supported per-run state-sharing contract for hook-plus-tool plugins.

## Reproduction Assessment

Yes at source level: current main can return cached descriptor-backed tools before the live registry path, while hooks are stored on registry entries. I did not run a live mem0 gateway reproduction in this read-only review.

## Solution Assessment

Unclear pending maintainer API-contract judgment. The linked resolver fix is narrow for preserving same-registration closure state, while an explicit state API pattern remains the safer alternative if closure sharing should not be a plugin contract.
