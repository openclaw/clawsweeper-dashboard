## Summary

Keep open: current main still has the cached descriptor-backed plugin tool path, and the open implementation candidate at https://github.com/openclaw/openclaw/pull/78075 uses closing syntax for this issue but has not landed.

## What This Changes

The issue asks OpenClaw to stop splitting plugin hook and tool closure state across repeated `register()` calls for `@mem0/openclaw-mem0`.

## Best Possible Solution

Review, validate, and land or supersede https://github.com/openclaw/openclaw/pull/78075 so hook handlers and tool factories for a live hook+tool plugin use the same active registry closure without adding mem0-specific core policy.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection shows a plausible current-main path where cached descriptor wrappers are used before the live runtime registry, matching the reported hook/tool closure split.

## Solution Assessment

Yes for the current direction: keeping this issue open for the active resolver PR is the narrowest maintainable path. The fix should preserve live hook/tool closure state rather than guaranteeing `register()` runs only once or adding a broad shared-state API by default.
