## Summary

Keep open: current main still appends Anthropic `input_json_delta` fragments to `partialJson` and parses them through the `parseStreamingJson` dependency path, which is `JSON.parse`-backed for complete JSON and can round snowflake-sized bare integers before tool execution.

## What This Changes

Preserve unsafe integer literals in Anthropic streamed `tool_use` input arguments before they are parsed for tool execution.

## Best Possible Solution

Add a core Anthropic streamed-tool-input parser guard before any `JSON.parse`-backed path can coerce unsafe integer literals, with focused SSE regression coverage and no vendored dependency monkey-patch.

## Reproduction Assessment

Yes. Feed a mocked Anthropic SSE stream with a `tool_use` block and `input_json_delta` containing `{\"to\":1481220477346119781}` through `createAnthropicMessagesTransportStreamFn`; the current path reaches `parseStreamingJson`, and a direct Node probe confirms the same literal rounds under `JSON.parse`.

## Solution Assessment

Yes in direction. The narrow maintainable fix is at the Anthropic streamed argument parse boundary before number coercion; post-processing already-rounded values or patching vendored parser code would be more fragile.
