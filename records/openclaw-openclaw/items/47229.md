## Summary

Keep open: current main still parses Anthropic streamed `input_json_delta` tool arguments through a plain `JSON.parse` path, so unsafe integer IDs can still be rounded before OpenClaw records the tool call.

## What This Changes

Preserve unsafe integer literals in Anthropic streamed `tool_use.input` arguments instead of rounding them during JSON parsing.

## Best Possible Solution

Add a core unsafe-integer-preserving JSON parser for streamed tool arguments, use it in the Anthropic `input_json_delta` path, and cover unsafe and safe integer cases without importing plugin-private Ollama code.

## Reproduction Assessment

Yes, at source level: current main appends Anthropic `input_json_delta` fragments and parses the completed JSON through a dependency path that calls `JSON.parse`. A focused primitive check shows that operation rounds the reported snowflake literal.

## Solution Assessment

No for the original SDK monkey-patch suggestion, because current main uses a custom Anthropic SSE client for the normal transport. The narrow maintainable fix is to parse `block.partialJson` with an unsafe-integer-preserving core helper before assigning `block.arguments`.
