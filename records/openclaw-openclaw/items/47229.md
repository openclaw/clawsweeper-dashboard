## Summary

Keep open: current main still routes Anthropic streamed tool input through `parseStreamingJson()` before tool execution, and the pinned parser path uses `JSON.parse`, so bare unsafe integer IDs can still be rounded.

## What This Changes

The issue asks OpenClaw to preserve large bare integer literals in Anthropic Messages streamed `tool_use.input` arguments, matching the existing Ollama unsafe-integer guard.

## Best Possible Solution

Add an unsafe-integer-preserving parser guard at the Anthropic streamed tool-input boundary, with focused regression coverage and a changelog fix entry.

## Reproduction Assessment

Yes, by source inspection and a direct `JSON.parse` probe. A mocked Anthropic SSE stream with `input_json_delta` containing `{"to":1481220477346119781}` should flow through `parseStreamingJson(block.partialJson)` and round the value before tool execution.

## Solution Assessment

Yes. The narrow maintainable fix is to preserve unsafe integer literals before parsing `block.partialJson`; post-processing already-rounded numbers or monkey-patching vendored dependency code would be more fragile.
