## Summary

Keep this open: current main still parses Anthropic streamed `tool_use` `input_json_delta` buffers through a JSON.parse-backed path before tool execution, so bare snowflake-sized IDs can still be rounded.

## What This Changes

Fix Anthropic Messages streaming so `tool_use.input` large integer literals are preserved instead of being coerced to imprecise JavaScript numbers.

## Best Possible Solution

Add a core Anthropic streamed-tool-input parser guard before any JSON.parse-backed path can coerce unsafe integer literals, with focused SSE regression coverage and no vendored dependency monkey-patch.

## Reproduction Assessment

Yes. A mocked Anthropic SSE stream with a `tool_use` block and `input_json_delta` containing `{"to":1481220477346119781}` should reproduce the current parser path, and a direct Node probe confirms the same literal is rounded by `JSON.parse`.

## Solution Assessment

Yes in direction: the narrow maintainable fix is at the Anthropic streamed argument parse boundary before number coercion. Post-processing already-rounded values or patching vendored parser code would be more fragile.
