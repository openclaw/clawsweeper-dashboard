## Summary

Keep open: current main still parses Anthropic streamed `input_json_delta` tool input through `parseStreamingJson()`, and the pinned parser path still uses `JSON.parse`, so bare unsafe integer tool arguments can still be rounded before tool execution.

## What This Changes

Preserve large bare integer literals in Anthropic Messages streamed `tool_use.input` arguments before they reach tool execution.

## Best Possible Solution

Add a shared/core unsafe-integer-preserving JSON parser at the Anthropic streamed tool-input boundary and cover it with a mocked Anthropic SSE regression, without monkey-patching dependencies or importing from the Ollama plugin.

## Reproduction Assessment

Yes. A mocked Anthropic SSE stream with `input_json_delta` containing `{"to":1481220477346119781}` should flow through `parseStreamingJson(block.partialJson)`, and the dependency source plus direct `JSON.parse` probe show that value rounds before tool execution.

## Solution Assessment

Yes. The narrow maintainable fix is to preserve unsafe integer literals before parsing `block.partialJson`; dependency monkey-patching or post-parse repair of already-rounded numbers would be more fragile.
