## Summary

Keep open: current main still parses Anthropic streamed `input_json_delta` tool input through `parseStreamingJson()`, whose complete and fallback parser paths use `JSON.parse`, so bare unsafe integer tool arguments can still be rounded before tool execution.

## What This Changes

The issue asks OpenClaw to preserve large integer literals in Anthropic Messages `tool_use` input instead of rounding them during streaming JSON parsing.

## Best Possible Solution

Add a core unsafe-integer-preserving parser at the Anthropic streamed tool-input boundary and cover it with a mocked SSE regression, reusing the Ollama behavior without importing from the Ollama plugin.

## Reproduction Assessment

Yes, by source inspection and a direct `JSON.parse` probe. A mocked Anthropic SSE stream with `input_json_delta` containing `{"to":1481220477346119781}` should flow through `parseStreamingJson(block.partialJson)` and round the value before tool execution.

## Solution Assessment

Yes. The narrow maintainable fix is to preserve unsafe integer literals before parsing `block.partialJson`; monkey-patching dependencies or trying to repair already-rounded numbers would be more fragile.

Keep safe integers as numbers and leave string literals unchanged. Add a `CHANGELOG.md` fix entry with valid human attribution if known.
