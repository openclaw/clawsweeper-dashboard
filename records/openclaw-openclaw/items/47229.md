## Summary

Keep open: current main still parses Anthropic streamed `tool_use` input through a `JSON.parse`-backed path before tool execution, so large bare integer IDs can still be rounded. The earlier ClawSweeper review remains accurate against the current source.

## What This Changes

Preserve Anthropic streamed `tool_use` input arguments containing unsafe integer literals instead of allowing JSON parsing to round them.

## Best Possible Solution

Add a core-side unsafe-integer-preserving parser guard at the Anthropic streamed tool-input boundary, with focused SSE regression coverage and a changelog fix entry.

## Reproduction Assessment

Yes, by source inspection. A mocked Anthropic SSE stream with a `tool_use` block and `input_json_delta` containing `{"to":1481220477346119781}` should flow into `parseStreamingJson(block.partialJson)`, and a direct Node probe confirms that literal rounds under `JSON.parse`.

## Solution Assessment

Yes in direction. The narrow maintainable fix is to preserve unsafe integer literals before parsing `block.partialJson`; post-processing already-rounded numbers or monkey-patching vendored dependency code would be more fragile.
