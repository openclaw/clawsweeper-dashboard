## Summary

Keep open: current main still routes Anthropic streamed `tool_use` `input_json_delta` buffers through `parseStreamingJson`, which calls plain `JSON.parse` for complete JSON and can round unsafe integer IDs before OpenClaw records tool arguments. An open draft PR already references this issue with closing syntax, so the issue should stay open until that PR lands or is replaced.

## What This Changes

Preserve unsafe integer literals in Anthropic streamed `tool_use` input arguments instead of letting JSON parsing round them.

## Best Possible Solution

Land or replace the linked PR with a shared core unsafe-integer-preserving parser applied to the Anthropic tool-call argument parse points, plus focused regression coverage and changed-check proof.

## Reproduction Assessment

Yes, at source level: current main accumulates Anthropic `input_json_delta` into `partialJson` and parses it with `parseStreamingJson`, while the pinned parser calls plain `JSON.parse` for complete JSON. I did not run live Anthropic/Discord proof in this read-only review.

## Solution Assessment

Yes, the linked direction is the narrowest maintainable fix: add a shared core unsafe-integer parser and call it at the Anthropic assignment points. Monkey-patching vendored code or trying to repair already-rounded numbers later would be riskier.
