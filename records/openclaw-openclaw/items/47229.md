## Summary

Keep open: current main no longer uses the Anthropic SDK MessageStream path, but the replacement Anthropic Messages transport still accumulates `input_json_delta` into `partialJson` and parses it through a `JSON.parse`-backed dependency path before tool execution.

## What This Changes

Preserve snowflake-sized bare integer literals in Anthropic Messages streamed `tool_use` input instead of letting JSON parsing round them before tools run.

## Best Possible Solution

Add a core parser guard at the Anthropic streamed tool-input boundary that quotes unsafe integer literals before any `JSON.parse`-backed path sees them, with focused SSE regression coverage and a changelog entry.

## Reproduction Assessment

Yes. A mocked Anthropic SSE stream with a `tool_use` block and `input_json_delta` containing `{"to":1481220477346119781}` should reach `parseStreamingJson`, and a direct Node probe confirms that literal rounds under `JSON.parse`.

## Solution Assessment

Yes in direction. The narrow maintainable fix is to preserve unsafe integer literals before parsing `block.partialJson`; post-processing already-rounded numbers or monkey-patching vendored dependency code would be more fragile.
