## Summary

Keep open: current main still routes Anthropic streamed `input_json_delta` tool arguments through `parseStreamingJson`, whose pinned dependency path uses plain `JSON.parse` for complete JSON and can round unsafe integer IDs before OpenClaw records the tool call.

## What This Changes

The issue asks OpenClaw to preserve unsafe integer literals in Anthropic streamed `tool_use.input` JSON arguments.

## Best Possible Solution

Add a core unsafe-integer-preserving JSON parser for streamed tool arguments, use it on the Anthropic partial and final tool-call argument paths, and cover unsafe and safe integer cases without importing plugin-private Ollama code.

## Reproduction Assessment

Yes, at source level: current main appends Anthropic `input_json_delta` fragments and parses the completed buffer through a dependency path that calls `JSON.parse`. A focused primitive check shows that operation rounds the reported snowflake literal.

## Solution Assessment

Yes for a narrow parser guard, but not for monkey-patching the SDK dependency. The maintainable fix is a core helper that preserves unsafe integer literals before parsing streamed tool-call JSON and is applied at the Anthropic argument assignment points.
