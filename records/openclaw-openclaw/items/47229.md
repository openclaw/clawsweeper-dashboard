## Summary

Keep open: current main still routes Anthropic Messages streamed tool-use input through a JSON.parse-backed parser before tool execution, so bare unsafe integer IDs can still be rounded.

## What This Changes

Preserve large bare integer literals in Anthropic Messages streamed `tool_use` input arguments before OpenClaw executes the tool call.

## Best Possible Solution

Add unsafe-integer-preserving parsing at the core Anthropic streamed `partialJson` boundary, with regression coverage for a large Discord-style ID and no provider or vendored-dependency behavior change.

## Reproduction Assessment

Yes, by source inspection. A mocked Anthropic SSE stream with `content_block_start` for `tool_use` and an `input_json_delta` like `{"to":1481220477346119781}` reaches `parseStreamingJson(block.partialJson)`, whose pinned dependency path uses `JSON.parse`.

## Solution Assessment

Yes in direction. The narrowest maintainable fix is to guard the raw `partialJson` string before parsing at the Anthropic transport boundary; monkey-patching vendored parsers or post-processing already-rounded numbers would be more fragile.
