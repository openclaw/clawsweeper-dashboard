## Summary

Keep open: current main still parses Anthropic streamed `tool_use` input through a plain JSON parse path, so unsafe integer literals can be rounded before OpenClaw receives the tool arguments.

## What This Changes

The issue asks OpenClaw to preserve large integer tool arguments in Anthropic streamed `tool_use` input, matching the earlier Ollama precision guard.

## Best Possible Solution

Add an Anthropic-stream-safe parser that preserves unsafe integer literals before assigning `block.arguments`, share the logic without depending on Ollama plugin internals, and cover unsafe and safe integer cases in focused Anthropic transport tests.

## Reproduction Assessment

Yes, at source level. Current main appends `input_json_delta` to `partialJson` and parses it through `parseStreamingJson`, whose complete-object path calls `JSON.parse`; a Node primitive check rounds the reported literal.

## Solution Assessment

Yes for keeping this open and routing a narrow fix; the original SDK monkey-patch is no longer the best current-main approach. The maintainable fix is to parse Anthropic tool input with an unsafe-integer-preserving parser before assigning final arguments.
