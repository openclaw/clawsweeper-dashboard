## Summary

Keep open: current main still parses Anthropic streamed `input_json_delta` tool-call buffers through the pinned `parseStreamingJson` path, which uses plain `JSON.parse` for complete JSON and can round unsafe integer IDs before OpenClaw records arguments.

## What This Changes

The issue requests preserving unsafe integer literals in Anthropic `tool_use` input JSON so IDs such as Discord snowflakes are not silently rounded.

## Best Possible Solution

Add a core unsafe-integer-preserving JSON parser for streamed tool-call arguments, apply it to Anthropic partial and final argument parsing, and cover safe, unsafe, nested, and malformed JSON cases without importing Ollama plugin-private code.

## Reproduction Assessment

Yes, at source level: current main routes Anthropic `input_json_delta` buffers through a pinned parser that uses `JSON.parse` for complete JSON. A focused primitive check confirms that exact parser operation rounds the reported unsafe integer literal.

## Solution Assessment

Yes for a narrow parser guard, but not by monkey-patching the dependency. The maintainable path is a shared core helper applied at the transport argument assignment points, with regression coverage in the Anthropic stream tests.
