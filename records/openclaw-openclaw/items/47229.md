## Summary

Keep this open. Current main still parses Anthropic streamed tool-use `input_json_delta` buffers through `parseStreamingJson`, and the pinned parser path ultimately uses `JSON.parse`, so bare Discord snowflake-sized integers can still be rounded before tool execution.

## What This Changes

The issue requests an Anthropic Messages streamed tool-use parser guard that preserves unsafe integer literals in tool `input` arguments instead of rounding them through JavaScript numbers.

## Best Possible Solution

Add a core Anthropic transport parse helper that quotes unsafe integer literals before `parseStreamingJson` or an equivalent complete-JSON parse path can coerce them, with focused SSE regression coverage for `input_json_delta` tool-use arguments. Keep the existing Ollama plugin behavior intact and avoid monkey-patching dependency internals.

## Reproduction Assessment

Yes. A mocked Anthropic SSE stream with a `tool_use` block whose `input_json_delta` contains `{"to":1481220477346119781}` should exercise the current parser path, and a direct Node probe confirms the same numeric literal is rounded by `JSON.parse`.

## Solution Assessment

Yes. The narrow maintainable fix is at the Anthropic streamed tool-argument parse boundary, before the raw JSON buffer is converted to JavaScript numbers; post-processing already-rounded values or patching vendored dependencies would be more fragile.
