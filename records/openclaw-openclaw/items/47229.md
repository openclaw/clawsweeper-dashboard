## Summary

Keep open: current main still routes Anthropic streamed `input_json_delta` tool-call buffers through `parseStreamingJson`, which uses plain `JSON.parse` for complete JSON and can round unsafe integer IDs. There is now an open draft fix PR linked with closing syntax, so this issue should stay open until that PR or an equivalent fix merges.

## What This Changes

The issue asks OpenClaw to preserve large integer literals in Anthropic streamed `tool_use` input before recording tool-call arguments.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/83063, or an equivalent core parser guard with focused regression coverage, then close this issue as implemented after merge.

## Reproduction Assessment

Yes, at source level: current main routes Anthropic streamed tool-use argument buffers through `parseStreamingJson`, and that dependency uses `JSON.parse` for complete JSON. A focused Node primitive check confirms the reported unsafe integer class rounds before OpenClaw could preserve it.

## Solution Assessment

Yes, the narrow maintainable fix is a core unsafe-integer-preserving JSON helper applied at the Anthropic transport argument assignment points, with regression coverage. Monkey-patching the dependency is less maintainable, and the current linked PR follows the better direction but still needs proof and validation.
