## Summary

Keep open. Current main still has the Synology Chat webhook-level 120-second `Promise.race` watchdog, so a longer configured agent runtime can still be preempted before `channel.turn.run` finishes.

## What This Changes

The issue asks for Synology Chat webhook delivery to honor `agents.defaults.timeoutSeconds` instead of failing agent responses after a hardcoded 120 seconds.

## Best Possible Solution

Fix this in the Synology Chat plugin by removing the independent 120-second delivery watchdog or deriving it from the resolved configured runtime timeout, with regression coverage for replies that take longer than 120 seconds.

## Reproduction Assessment

Yes. A focused unit test can make `deliver` remain pending beyond 120 seconds, advance fake timers, and observe the current handler logging/sending the timeout error before a larger configured agent timeout could complete.

## Solution Assessment

Yes. The narrowest maintainable fix is plugin-owned: let the existing channel turn runtime budget control agent execution rather than changing core timeout defaults or channel contracts.

## Repair Work Prompt

Fix the Synology Chat webhook timeout bug reported in https://github.com/openclaw/openclaw/issues/59966. The observable bug is that `extensions/synology-chat/src/webhook-handler.ts` races `params.deliver(...)` against a hardcoded 120-second `Agent response timeout (120s)` timer, so `agents.defaults.timeoutSeconds` values above 120 seconds are ignored for this plugin path. Keep the fix plugin-owned: remove the independent watchdog or derive any guard from the resolved configured agent runtime timeout; do not change core timeout defaults or channel contracts. Preserve the immediate Synology webhook ACK behavior and existing body/read security guards. Add a focused regression test in `extensions/synology-chat/src/webhook-handler.test.ts` that demonstrates a delivery pending beyond 120 seconds is not failed by the webhook layer when the configured agent runtime allows it. Check related refs #38360 and #39745 for prior context, but do not copy unrelated timer-cleanup changes. Add a `CHANGELOG.md` fix entry because this is user-facing. Validate with `pnpm test extensions/synology-chat/src/webhook-handler.test.ts` and `pnpm check:changed` under the repo’s normal Testbox policy.
