## Summary

Keep this PR open: the planner approach is useful and the protected `maintainer` label requires explicit handling, but the latest head still has a narrow fallback bug that can preserve a stale managed `openclaw` host peer when npm planning fails.

## What This Changes

The PR replaces installed-tree managed peer discovery with npm lockfile planning, adds fallback/rollback handling, preserves active root-managed OpenClaw runtimes, updates managed npm tests, and adds a changelog entry.

## Best Possible Solution

Fix the fallback host-peer filtering with focused regression coverage, then land this PR after maintainer review and use it to supersede https://github.com/openclaw/openclaw/pull/81632 if the folded active-host guard remains included.

## Reproduction Assessment

Yes. Source inspection of PR head shows the fallback map copies prior managed peers without excluding `openclaw`; a manifest with `dependencies.openclaw`, `openclaw.managedPeerDependencies: ["openclaw"]`, and any nonzero planner result would preserve the host peer instead of pruning it.

## Solution Assessment

No. Using npm's lockfile planner is a good direction, but the fallback needs to apply the same host-peer exclusion invariant before this is the narrowest maintainable fix.
