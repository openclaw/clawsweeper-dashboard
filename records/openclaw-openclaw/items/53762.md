## Summary

Keep open: this is a real Telegram partial-stream message-loss fix path, but the current PR still skips the existing preview-finalized hook contract and lacks the required real Telegram proof before merge.

## What This Changes

This PR adds a Telegram dispatch flush for buffered final-answer text before the split-segment early return, plus a mocked regression test and changelog entry.

## Best Possible Solution

Update this PR or land an equivalent Telegram patch that drains buffered final answers while routing preview-finalized deliveries through the existing hook path, preserving current-main dispatcher behavior and adding redacted real Telegram proof.

## Reproduction Assessment

Yes, from source inspection. Current main can note a reasoning hint, skip reasoning delivery, buffer the final answer, and then return from the segmented final branch without calling takeBufferedFinalAnswer(); I did not run tests because this review was read-only.

## Solution Assessment

No, not as currently proposed. The missing flush is the right narrow direction, but the flushed delivery result must be returned or emitted through the same preview-finalized hook path as normal final segment delivery.
