## Summary

Keep open: current main still has the skipped-reasoning buffered-answer path, but this PR is not mergeable as-is because the new flush bypasses the preview-finalized hook and the external PR still lacks real Telegram proof.

## What This Changes

This PR adds a Telegram dispatch flush for buffered final-answer text before the segmented final-delivery early return, plus a regression test and changelog entry.

## Best Possible Solution

Land this PR or an equivalent Telegram-owned patch after the buffered flush returns or emits its delivery result through `emitPreviewFinalizedHook()`, preserves current segmented-return side effects, and includes redacted real Telegram proof.

## Reproduction Assessment

Yes, from source inspection. Current main can note a reasoning hint, skip reasoning delivery, buffer the final answer, and then return from the segmented final branch without calling `takeBufferedFinalAnswer()`.

## Solution Assessment

No, not as currently proposed. The missing flush is the right narrow direction, but the flushed `LaneDeliveryResult` must follow the same preview-finalized hook path as normal final segment delivery.

## Review Findings
