## Summary

Keep open: this is a plausible narrow fix for a real Telegram partial-streaming message-loss path, but the proposed patch still skips the preview-finalized hook/transcript path and lacks required real Telegram proof before merge.

## What This Changes

The PR adds a Telegram dispatcher flush for buffered final-answer text before a segmented final-delivery early return, a skipped-reasoning regression test, and a changelog entry.

## Best Possible Solution

Land this PR or an equivalent Telegram-owned patch after the buffered flush returns or emits the `LaneDeliveryResult` through the same preview-finalized hook path, adds focused coverage, and includes redacted real Telegram proof.

## Reproduction Assessment

Yes. Source inspection on current main shows a final mixed reasoning/text payload can leave reasoning hinted, buffer the answer, then return from the segmented branch without draining it when reasoning delivery is skipped; I did not run a live Telegram repro in this read-only review.

## Solution Assessment

No, not as currently proposed. Adding the missing flush is the right narrow direction, but the implementation must preserve preview-finalized hook/transcript emission and provide real Telegram proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Emit the hook when flushing buffered final answers:** `extensions/telegram/src/bot-message-dispatch.ts:702`
  - body: The new final-segment early return calls `flushBufferedFinalAnswer()` and exits, but that helper discards the `deliverLaneText()` result. When the buffered answer finalizes an existing preview, Telegram shows the answer while `emitPreviewFinalizedHook(...)` and the transcript mirror are skipped, unlike normal final segment delivery. Capture the result and route preview-finalized deliveries through the hook before resetting state.
  - confidence: 0.9
