## Summary

Keep this PR open: it targets a real Telegram partial-stream message-loss path, but the patch needs one correctness fix because the new buffered final-answer flush can bypass the internal `message:sent` hook when it finalizes a preview.

## What This Changes

This PR adds a Telegram dispatcher flush for buffered final-answer text before the segmented final-delivery early return, plus a regression test and changelog entry.

## Best Possible Solution

Land this PR or a replacement after the buffered final-answer flush preserves preview-finalized hook emission and the branch is refreshed against current main.

## Reproduction Assessment

Yes, by source inspection. Current main can buffer a final answer while reasoning remains `hinted` and then return from the segmented final branch without draining that buffer; this read-only review did not run the focused test.

## Solution Assessment

No, not as currently proposed. Adding the missing flush is the right narrow direction, but the flush must emit or return the `LaneDeliveryResult` through the same preview-finalized hook path as normal final segment delivery.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Emit hooks for flushed buffered final answers:** `extensions/telegram/src/bot-message-dispatch.ts:703`
  - body: The new segmented-final branch calls `flushBufferedFinalAnswer()` and exits, but that helper discards the `deliverLaneText()` result. When the buffered answer finalizes an existing preview, Telegram shows the answer but `emitPreviewFinalizedHook(...)` is skipped, so `sessionKeyForInternalHooks` flows miss the internal `message:sent` hook unlike normal final segment delivery. Route the flush result through the hook before resetting state.
  - confidence: 0.88
