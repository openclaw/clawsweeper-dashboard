## Summary

Keep open. The Telegram message-loss path is source-reproducible, but this PR is not ready as-is because the new flush path bypasses the preview-finalized hook/transcript contract and the external PR still lacks real Telegram proof.

## What This Changes

This PR adds a Telegram final-delivery buffered-answer flush, a regression test for skipped reasoning after a tool boundary, and a changelog entry.

## Best Possible Solution

Land this PR or an equivalent Telegram patch after the buffered-answer flush preserves preview-finalized hook/transcript handling, keeps the fix scoped to Telegram dispatch, and includes focused regression coverage plus redacted live Telegram proof.

## Reproduction Assessment

Yes, from source inspection. Current main can hint reasoning, skip reasoning delivery, buffer the final answer, and then return from the segmented final branch without draining that buffer.

## Solution Assessment

No, not as currently proposed. The missing flush is the right narrow direction, but the flushed delivery result must be routed through the same preview-finalized hook path as ordinary final segment delivery.
