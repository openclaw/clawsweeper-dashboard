## Summary

Keep open: current main still has the Telegram segmented-final early return without the missing buffer flush, so the PR targets a valid message-loss bug, but the proposed patch still misses the existing preview-finalized `message:sent` hook path.

## What This Changes

This PR adds a Telegram final-answer buffer flush before segmented final delivery returns, adds a skipped-reasoning regression test, and adds a changelog fix entry.

## Best Possible Solution

Land this PR or an equivalent Telegram-owned patch after the buffered final-answer flush preserves `LaneDeliveryResult` hook emission and keeps the fix scoped to Telegram dispatch, focused tests, and the user-facing changelog entry.

## Reproduction Assessment

Yes, by source inspection. Current main can buffer a final answer while reasoning remains `hinted` and then return from the segmented final branch without draining that buffer; I did not run tests because this was a read-only review.

## Solution Assessment

No, not as currently proposed. Adding the missing flush is the right narrow direction, but the flush must return or emit the `LaneDeliveryResult` through the same preview-finalized hook path as normal final segment delivery.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Emit the hook when flushing buffered final answers:** `extensions/telegram/src/bot-message-dispatch.ts:703`
  - body: The new segmented-final early return calls `flushBufferedFinalAnswer()` and exits, but that helper discards the `deliverLaneText()` result. Capture or return the result and pass preview-finalized deliveries through the hook.
