## Summary

Keep open. The PR targets a real Telegram partial-streaming message-loss edge that is still source-reproducible on current main, but the branch is not merge-ready because the new flush path drops the preview-finalized hook result and the external PR still lacks required live Telegram proof.

## What This Changes

This PR adds a Telegram dispatcher flush for buffered final-answer text before the segmented final-delivery early return, plus a skipped-reasoning regression test and changelog entry.

## Best Possible Solution

Update this PR or land an equivalent Telegram-scoped patch that drains buffered final answers on skipped reasoning, routes the `LaneDeliveryResult` through preview-finalized hook emission, preserves current segmented-delivery behavior, keeps focused tests and changelog coverage, and adds redacted live Telegram proof.

## Reproduction Assessment

Yes. Source inspection shows current main can buffer a final answer while reasoning remains hinted, skip the reasoning-delivered flush when delivery returns `skipped`, and return from the segmented branch without draining that buffer.

## Solution Assessment

No, not as currently proposed. Adding the missing flush is the right narrow direction, but the flush must preserve or emit the `LaneDeliveryResult` through the same preview-finalized hook path as normal final segment delivery.
