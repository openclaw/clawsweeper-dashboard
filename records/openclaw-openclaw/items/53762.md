Keep open: current main still has the skipped-reasoning early-return hole, so the PR is useful, but it should not merge as-is because the new flush path bypasses existing preview-finalized hook/transcript handling and still lacks real Telegram proof.

## What This Changes

The PR adds a Telegram final-payload buffered-answer flush before the segmented-delivery early return, plus a focused regression test and changelog entry.

## Best Possible Solution

Land this PR or an equivalent Telegram-owned patch after the buffered flush preserves the preview-finalized hook path and redacted live Telegram proof shows final text delivery after a reasoning/tool boundary.

## Reproduction Assessment

Yes, from source inspection. Current main can buffer final answer text while reasoning remains hinted, then return from the segmented final branch without draining that buffer when reasoning delivery is skipped.

## Solution Assessment

No, not as currently proposed. The missing flush is the right narrow fix, but the flushed delivery result must be routed through `emitPreviewFinalizedHook()` and the PR still needs redacted real Telegram proof.
