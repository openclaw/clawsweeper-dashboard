## Summary

Keep open: this is a real, source-reproducible Telegram message-loss bug, but the current PR still bypasses the preview-finalized hook path and lacks real Telegram proof before merge.

## What This Changes

This PR adds a Telegram dispatch flush for buffered final-answer text before the segmented final-delivery early return, plus a focused regression test and changelog entry.

## Best Possible Solution

Land this PR or an equivalent Telegram-owned patch after the buffered flush returns or emits the delivery result through `emitPreviewFinalizedHook()`, preserves current-main dispatch behavior, and includes redacted live Telegram proof.

## Reproduction Assessment

Yes, from source inspection. Current main can buffer final answer text while reasoning remains hinted, skip reasoning delivery, and return from the segmented final branch without draining that buffer.

## Solution Assessment

No, not as currently proposed. The missing flush is the right narrow direction, but the flushed delivery result must follow the same preview-finalized hook path as normal final segment delivery.
