## Summary

Keep open: current main still lacks the skipped-reasoning buffered-final flush, but this PR is not merge-ready because the new flush path would bypass the existing preview-finalized `message:sent` and transcript hook contract.

## What This Changes

The PR adds a Telegram segmented-final flush for buffered final-answer text, a focused regression test, and a changelog entry.

## Best Possible Solution

Land this PR or an equivalent Telegram-owned patch after the buffered flush returns or emits its `LaneDeliveryResult` through the preview-finalized hook path and includes redacted real Telegram proof.

## Reproduction Assessment

Yes, from source inspection. Current main can note a reasoning hint, skip reasoning delivery, buffer the final answer, and then return from the segmented final branch without calling `takeBufferedFinalAnswer()`.

## Solution Assessment

No, not as currently proposed. The missing flush is the right narrow direction, but the flushed delivery result must follow the same preview-finalized hook path as normal final segment delivery.
