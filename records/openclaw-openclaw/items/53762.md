## Summary

Keep open: current main still has the skipped-reasoning buffer-drain gap, and this PR is a plausible narrow fix, but the proposed flush path drops the preview-finalized hook result and lacks real Telegram proof.

## What This Changes

This PR adds a Telegram dispatcher flush for buffered final-answer text before the segmented final-delivery early return, plus a skipped-reasoning regression test and changelog entry.

## Best Possible Solution

Land an updated Telegram-owned patch that drains buffered final answers on skipped reasoning while preserving preview-finalized hook emission, current compaction/block-media handling, focused tests, changelog, and live Telegram proof.

## Reproduction Assessment

Yes. Source inspection shows current main can buffer a final answer while reasoning remains hinted, skip the reasoning-delivered flush when delivery returns skipped, and return from the segmented branch without draining that buffer.

## Solution Assessment

No, not as currently proposed. Adding the missing flush is the right narrow direction, but the flush must preserve or emit the `LaneDeliveryResult` through the same preview-finalized hook path as normal final segment delivery.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Emit the hook when flushing buffered final answers:** `extensions/telegram/src/bot-message-dispatch.ts:703`
  - body: The new segmented-final branch calls `flushBufferedFinalAnswer()` and returns, but that helper discards the `deliverLaneText()` result. When the buffered answer finalizes an existing preview, Telegram shows the answer while `emitPreviewFinalizedHook(...)` is skipped, so `sessionKeyForInternalHooks` flows miss the internal `message:sent` hook unlike normal final segment delivery. Return the result or emit the hook before resetting state.
