## Summary

This PR is aimed at a real Telegram partial-streaming message-loss edge case, but it should stay open because the proposed flush still drops the preview-finalized hook/transcript result and the PR lacks live Telegram after-fix proof.

## What This Changes

The PR adds a Telegram dispatcher flush for buffered final-answer text before the segmented final-delivery early return, adds a skipped-reasoning regression test, and adds a changelog entry.

## Best Possible Solution

Land this PR or an equivalent Telegram-owned patch after the buffered flush preserves the LaneDeliveryResult hook path and the PR includes redacted live Telegram proof for the partial-streaming reasoning-plus-final-text flow.

## Reproduction Assessment

Yes. Source inspection on current main shows a final mixed reasoning/text payload can leave reasoning hinted, buffer the answer, and return from the segmented branch without draining it when reasoning delivery is skipped; I did not run a live Telegram bot in this read-only review.

## Solution Assessment

No, not as currently proposed. Adding the missing flush is the right narrow direction, but the implementation should route the flushed delivery result through the same preview-finalized hook path as normal final segment delivery.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Emit the hook when flushing buffered final answers:** `extensions/telegram/src/bot-message-dispatch.ts:702`
  - body: The new final-segment early return calls `flushBufferedFinalAnswer()` and exits, but that helper discards the `deliverLaneText()` result. When the buffered answer finalizes an existing preview, Telegram shows the answer while the internal `message:sent` hook and transcript mirror are skipped, unlike normal final segment delivery. Capture the result and route preview-finalized deliveries through the hook before resetting state.
  - confidence: 0.88
