## Summary

Keep open: current main still lacks the proposed Telegram final-answer flush, but this PR is not merge-ready because the added flush path skips preview-finalized hook emission and the PR still has only synthetic proof.

## What This Changes

The PR adds a Telegram buffered final-answer flush before a segmented final-delivery early return, plus a skipped-reasoning regression test and changelog entry.

## Best Possible Solution

Land this PR or an equivalent Telegram-owned patch after the buffered flush returns or emits the `LaneDeliveryResult` through the same preview-finalized hook path, preserves current-main segmented delivery behavior, adds focused hook coverage, and includes redacted real Telegram proof.

## Reproduction Assessment

Yes. Source inspection shows current main can buffer a final answer while reasoning remains hinted, skip the reasoning-delivered flush when delivery returns skipped, and return from the segmented branch without draining that buffer.

## Solution Assessment

No, not as currently proposed. Adding the missing flush is the right narrow direction, but the implementation must preserve preview-finalized hook emission and provide real Telegram proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

- **[P2] Emit the hook when flushing buffered final answers:** `extensions/telegram/src/bot-message-dispatch.ts:703`

Capture the result and route preview-finalized deliveries through the hook before resetting state.
