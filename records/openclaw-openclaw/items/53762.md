## Summary

Keep open. The PR addresses a real Telegram partial-streaming message-loss path that is still present on current main, but the patch is not merge-ready because the new flush path can skip the internal message:sent hook for preview-finalized buffered answers; the source branch is also currently dirty against main.

## What This Changes

This PR adds a Telegram dispatcher flush for buffered final-answer text before the segmented final-delivery early return, plus a focused regression test and changelog entry.

## Best Possible Solution

Land this PR or a narrow replacement after rebasing onto current main and changing flushBufferedFinalAnswer() to return or emit the LaneDeliveryResult through the same preview-finalized hook path as normal final segment delivery, with focused regression coverage for both the skipped-reasoning answer delivery and hook emission.

## Reproduction Assessment

Yes. The PR body gives a concrete two-turn Telegram partial-streaming path, and current main statically shows the answer can be buffered while reasoning remains hinted, then lost when the segmented final branch returns after a skipped reasoning delivery.

## Solution Assessment

No, not as currently proposed. Adding the missing flush is the right narrow fix for the dropped answer, but the implementation must preserve the existing preview-finalized message:sent hook contract.
