## Summary

This is a focused fix for a real systemd-managed update bug, but the latest patch is not merge-ready because the transient service only forwards the sentinel/update env and the supplied proof does not show an actual OpenClaw update under a live user systemd service.

## What This Changes

The PR passes detected supervisor context into managed update handoff and routes systemd-supervised update helpers through systemd-run with transient-unit tests and failure handling.

## Best Possible Solution

Keep the cgroup escape, preserve the full sanitized helper environment for systemd handoffs, and prove an end-to-end update on a user systemd service before merging.

## Reproduction Assessment

Yes at source/report level: current main starts the helper as a detached child from a systemd unit rendered with KillMode=control-group, and the linked issue reports handoff files without handoff.log or version change. I did not run a live systemd repro in this read-only review.

## Solution Assessment

No: systemd-run is the right direction, but this branch uses a service unit with only two forwarded env vars and no live user-systemd proof. The safer path is a real scope-preserving handoff or explicit sanitized env forwarding plus profiled/custom-state coverage.
