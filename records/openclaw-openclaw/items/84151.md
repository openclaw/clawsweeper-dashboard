## Summary

Keep this PR open: it targets a valid P1 systemd update-handoff bug, but the latest patch forwards the entire gateway environment through systemd-run arguments and still lacks end-to-end user-systemd OpenClaw update proof.

## What This Changes

The PR passes detected supervisor context into managed update handoff and routes systemd-supervised helpers through a systemd-run transient service, with tests for the systemd path and failure handling.

## Best Possible Solution

Keep the systemd cgroup escape, but transfer only the needed helper environment through a secret-safe mechanism and prove a real user-systemd update end to end before merging.

## Reproduction Assessment

Yes at source/report level: current main starts the helper as a detached child while the documented/generated systemd unit uses KillMode=control-group, and the linked report includes a live Linux/systemd failure. I did not run a live systemd update in this read-only review.

## Solution Assessment

No: the cgroup-escape direction is right, but forwarding the full environment through systemd-run argv is not the safest implementation and the PR still needs real user-systemd update proof.
