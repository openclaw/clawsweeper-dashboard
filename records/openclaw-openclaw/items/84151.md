## What This Changes

The branch routes systemd-managed update handoffs through `systemd-run --user --scope`, changes generated systemd restart and marker behavior, adds helper failure recovery/tests, and adds two proof scripts under `src/`.

## Best Possible Solution

Keep the scoped systemd handoff approach, but preserve shipped-unit exit compatibility, restart failures with argv-based `systemctl`, remove the unrelated proof scripts, and require current-head user-systemd proof before merge.

## Reproduction Assessment

Yes at source/report level: current main combines a detached handoff helper with generated systemd `KillMode=control-group`, and the linked issue plus PR proof show why detached children remain killable by the service cgroup. I did not run a packaged OpenClaw update in this read-only review.

## Solution Assessment

No for the current PR head: `systemd-run --user --scope` is the right direction, but the implementation still breaks old-unit compatibility, shells an unvalidated unit name, and includes unrelated production source files.

Full review comments:

- **[P1] Keep systemd update exits compatible with shipped units:** `src/cli/gateway-cli/run-loop.ts:227`
  - body: Existing generated units on current main and the latest release only include `RestartPreventExitStatus=78`, but this branch makes the update restart path exit 80. Users upgrading with an existing unit will be restarted by `Restart=always`, so the helper no longer has the quiet update window this fix relies on; use the existing 78 path or include a tested migration/proof for old and new units.
  - confidence: 0.93
- **[P1] Start systemd with argv instead of shell text:** `src/gateway/server-methods/update-managed-service-handoff.ts:173`
  - body: `params.systemdUnit` is derived from `OPENCLAW_SYSTEMD_UNIT` and is concatenated into an `execSync` shell command. A value with shell metacharacters can execute extra commands on helper failure; use `spawnSync`/`execFileSync` with `['--user', 'start', params.systemdUnit]` or strictly validate the unit name before use.
  - confidence: 0.9
- **[P1] Remove the unrelated proof scripts from production source:** `src/proof-typecheck.ts:12-13`
  - body: The added `src/proof-*` files are unrelated to the systemd update fix and are included by the production `src/**/*` typecheck. `proof-typecheck.ts` passes `requesterSenderId` and `senderIsOwner` to `listChannelAgentTools`, but this PR does not change that signature, so the branch is poised to fail core typechecking and should drop these files or move real coverage into the relevant test suite.
  - confidence: 0.95
