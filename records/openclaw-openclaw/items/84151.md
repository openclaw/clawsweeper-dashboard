## What This Changes

The PR routes systemd-supervised update handoff helpers through `systemd-run --user --scope`, adds exit-78 gateway handoff handling, marker/recovery logic, and focused tests.

## Best Possible Solution

Land the scoped systemd handoff once maintainers accept the upgrade-safety boundary, while preserving the existing non-systemd detached handoff path and exit-78 compatibility for shipped units.

## Reproduction Assessment

Yes at source/proof level: current main spawns a detached helper while generated systemd units use `KillMode=control-group`, and the linked issue plus PR proof show why that helper can be killed before it runs. I did not run a live systemd update in this read-only review.

## Solution Assessment

Yes at code-shape level: a transient systemd scope is the right ownership boundary for a helper that must outlive the gateway service cgroup. The remaining question is maintainer acceptance of upgrade-safety risk, especially full version swap and failure recovery.
