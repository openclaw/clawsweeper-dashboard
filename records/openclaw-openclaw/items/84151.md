## Summary

Keep this PR open: the systemd cgroup escape is the right bug-fix direction, but the current head still has blocking upgrade, build, and security issues, and the posted proof no longer cleanly matches the branch state.

## What This Changes

The PR routes systemd-managed update handoffs through `systemd-run --user --scope`, changes generated systemd restart behavior and marker handling, adds helper failure recovery/tests, and currently includes two unrelated proof scripts under `src/`.

## Best Possible Solution

Keep the scoped systemd handoff, use an upgrade-compatible restart-prevented exit or explicit migration, restart failures with argv-based `systemctl`, remove the unrelated proof scripts, and then post PR-head user-systemd proof for success plus failure recovery.

## Reproduction Assessment

Yes at source/report level: current main combines a detached handoff helper with generated systemd `KillMode=control-group`, and the linked issue plus PR proof show that detached children remain killable by the service cgroup. I did not run a packaged OpenClaw update in this read-only review.

## Solution Assessment

No for the current PR head: `systemd-run --user --scope` is the right core direction, but the implementation still breaks old-unit compatibility, shells an unvalidated unit name, and includes unrelated source files that should not merge.
