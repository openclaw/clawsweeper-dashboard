- `P1`: The PR targets a broken in-product update path for recommended Linux systemd-supervised installs, which blocks users from upgrading through the Control UI.

## Summary

Keep open: the branch targets a real P1 systemd update-handoff bug, but the current patch still has a blocking transient-service environment contract issue, CI failures, and insufficient live systemd proof.

## What This Changes

The PR passes supervisor detection into the managed update handoff and adds a systemd-specific `systemd-run` transient-unit launch path with mocked coverage for systemd, non-systemd, and failure cases.

## Best Possible Solution

Keep the systemd cgroup-escape direction, but make the transient unit explicitly carry the update/sentinel environment, use an exec-aware readiness contract, preserve the non-systemd path, and prove the end-to-end update on a Linux user-systemd host.

## Reproduction Assessment

Yes at source level: current main combines a plain detached helper spawn with a systemd unit that keeps children in the service cgroup, and the linked Linux report provides concrete no-`handoff.log` evidence. I did not run a live user-systemd upgrade in this read-only review.

## Solution Assessment

No, not yet: the systemd-run direction is the right shape, but this patch needs an explicit transient-unit env/readiness contract and live systemd proof before it is the safest fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Pass sentinel env into the transient unit:** `src/gateway/server-methods/update-managed-service-handoff.ts:335-345`
  - body: `systemd-run` service mode runs the helper in a clean service environment, but the update command only reads the control-plane metadata path from `OPENCLAW_CONTROL_PLANE_UPDATE_SENTINEL_META`. Passing `env` to the `systemd-run` client process here does not set that variable on the transient unit, so the update can finish without rewriting the final restart sentinel or continuation.
  - confidence: 0.88
- **[P2] Make systemd-run wait through helper exec:** `src/gateway/server-methods/update-managed-service-handoff.ts:335-345`
  - body: The default transient service start can succeed once the start job is accepted, before the helper process has actually exec'd. Add `--service-type=exec` or an equivalent readiness contract so `result.status === "started"` does not mask an immediate helper exec failure.
  - confidence: 0.78
- **[P2] Fix the failing spawn mock type:** `src/gateway/server-methods/update-managed-service-handoff.test.ts:363`
  - body: `check-test-types` currently fails because the hoisted spawn mock infers `pid` as `number`, while this new failure case returns `pid: undefined`. Type the mock return more broadly or omit `pid` in the override so the PR can pass the test type gate.
  - confidence: 0.97
