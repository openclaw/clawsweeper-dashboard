## Summary

Keep open: current main still has the systemd cgroup handoff failure shape, and this PR is a focused fix for the linked Linux/systemd update bug. I found no blocking code-review finding in the proposed patch, but the systemd lifecycle still has compatibility and availability risk that maintainers should explicitly accept or prove before merge.

## What This Changes

The PR routes systemd-supervised managed update handoffs through `systemd-run --user --scope`, preserves update exit code 78, adds marker/recovery handling, and expands systemd/update handoff tests.

## Best Possible Solution

Land the scoped systemd handoff after exact-head packaged update and failure-recovery proof, while preserving the existing non-systemd detached handoff path.

## Reproduction Assessment

Yes at source/proof level: current main spawns a detached helper while generated systemd units use `KillMode=control-group`, and the linked issue plus PR proof show why that helper can be killed before it runs. I did not run a live systemd update in this read-only review.

## Solution Assessment

Yes at code-shape level: a transient systemd scope is the right ownership boundary for a helper that must outlive the gateway service cgroup. The remaining question is upgrade-safety proof for the exact PR head, especially failure recovery.
