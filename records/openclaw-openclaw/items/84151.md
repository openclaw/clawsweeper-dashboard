## Summary

## What This Changes

The PR routes systemd-supervised managed update handoffs through `systemd-run --user --scope`, exits with status 78 for update restarts, adds marker/recovery handling, and expands daemon/gateway tests.

## Best Possible Solution

Land the scoped systemd handoff after the CI blockers are fixed and redacted current-head Linux systemd proof shows both successful update handoff and failed-update recovery behavior.

## Reproduction Assessment

Yes at source/proof level: current main starts the helper as a detached child while generated systemd units use `KillMode=control-group`, and the linked issue plus PR proof show that this cgroup relationship kills the helper. I did not run a live systemd update in this read-only review.

## Solution Assessment

Yes at direction level: launching the helper in a transient user scope is the right boundary for escaping the gateway unit while keeping `KillMode=control-group`. The proposed patch still needs CI fixes and fuller current-head update/recovery proof before it is the best mergeable solution.
