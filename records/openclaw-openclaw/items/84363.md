## Summary

Keep open: the PR targets a plausible launchd restart-loop problem, but the plist change conflicts with current macOS supervised restart paths that intentionally exit with code 0 and rely on KeepAlive to relaunch the gateway.

## What This Changes

This PR changes the generated launchd LaunchAgent KeepAlive policy to `SuccessfulExit=false`, updates the launchd audit check/message, and adjusts the plist shape test.

## Best Possible Solution

Fix the launchd loop without breaking intentional supervised restarts, either by updating the restart handoff before switching to `SuccessfulExit=false` or by keeping the current KeepAlive policy until a live-tested replacement is ready.

## Reproduction Assessment

Yes for the PR-introduced regression by source inspection: current launchd-supervised restart paths exit with code 0 and depend on KeepAlive relaunch. The original reported loop was not live-reproduced here.

## Solution Assessment

No. The plist-only fix is not the best solution because it treats all clean exits as non-restartable while current code uses clean exits for intentional launchd-supervised restarts.
