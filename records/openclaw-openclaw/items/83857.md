## Summary

Keep open: current main still does not give Crestodian a generic media or tool-invocation route, while the normal agent, infer, and gateway media surfaces do exist. The remaining question is a product/API boundary decision for Crestodian rather than a high-confidence cleanup close.

## What This Changes

The issue requests a Crestodian REPL route for xAI-backed image generation and TTS so direct `/tool image_generate`, natural-language image requests, and `/tts` can invoke existing media provider/tool surfaces.

## Best Possible Solution

Choose the Crestodian contract explicitly: either add a generic handoff or media/tool route that reuses existing agent/gateway provider contracts, or document Crestodian as setup-only and guide media requests to `talk to agent`.

## Reproduction Assessment

Yes, from source inspection: Crestodian routes input through its typed operation parser and assistant allow-list, which exclude generic `/tool`, `image_generate`, and `tts` commands while the xAI media surfaces exist elsewhere. I did not run a live xAI OAuth setup.

## Solution Assessment

Unclear as filed: exposing media from Crestodian may be useful, but the maintainable fix should reuse existing agent/gateway tool contracts and needs a maintainer decision on whether Crestodian should execute media actions at all.
