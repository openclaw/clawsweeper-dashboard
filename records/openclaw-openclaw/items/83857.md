## Summary

Keep open: current main and the latest release still do not expose media or generic tool invocation through Crestodian. The underlying `image_generate`, `tts`, and `tools.invoke` surfaces exist elsewhere, but Crestodian remains a closed setup/helper command surface, so the remaining work needs a maintainer product-boundary decision.

## What This Changes

The issue requests that Crestodian REPL route xAI image generation and TTS requests to available enabled media tools when the same provider setup works through `openclaw infer`.

## Best Possible Solution

Define a provider-neutral Crestodian behavior for media requests: either reuse existing agent/Gateway media tools through an approved contract, or explicitly hand users to the normal agent TUI and document that boundary.

## Reproduction Assessment

Yes, at source level. Current main routes Crestodian input through a closed operation set that does not include `image_generate`, `tts`, `/tool`, or `tools.invoke`; I did not run a live xAI credentialed repro.

## Solution Assessment

Unclear. Direct media execution inside Crestodian is plausible, but a cleaner end state may be generic Gateway tool invocation, automatic handoff to the normal agent TUI, or clearer setup-only documentation.
