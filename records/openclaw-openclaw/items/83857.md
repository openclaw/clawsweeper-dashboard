## Summary

Keep open: current main still has the Crestodian-specific media routing gap. The normal agent and gateway media surfaces expose `image_generate`, `tts`, and `tools.invoke`, but Crestodian remains a setup/helper command surface that does not parse or plan those commands.

## What This Changes

The issue requests that Crestodian route xAI-backed image generation and TTS requests through available media tools when those providers work through `openclaw infer`.

## Best Possible Solution

Define a provider-neutral Crestodian media/tool contract that reuses existing agent or gateway media tools, or explicitly route media work to the normal agent TUI.

## Reproduction Assessment

Yes. Current-main source inspection shows the requested capability is missing: Crestodian resolves input only to its closed setup/helper operation set, while `image_generate` and `tts` live on the normal agent and gateway tool surfaces.

## Solution Assessment

Unclear. Exposing media in Crestodian is plausible, but the best fix needs maintainer choice between direct generic gateway tool invocation, automatic handoff to the normal agent TUI, or documenting Crestodian as setup-only.
