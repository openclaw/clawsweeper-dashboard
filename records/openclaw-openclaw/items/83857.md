## Summary

The report describes a real, source-backed Crestodian routing gap: the REPL command allow-list only exposes text-focused commands and plugin/admin actions, while image generation and TTS are implemented through separate gateway/infer surfaces, so valid xAI media providers are not reachable from Crestodian today.

## What This Changes

The issue requests that Crestodian route direct and natural-language image generation and text-to-speech requests to the enabled media tools/providers instead of falling back to overview or plugin search.

## Best Possible Solution

Add a generic Crestodian media/tool routing path that reuses the existing provider and gateway contracts for image generation and TTS, with direct-command and natural-language coverage, instead of hard-coding xAI-specific behavior.

## Reproduction Assessment

Yes from source inspection: `/tool image_generate ...` reaches the Crestodian command allow-list path, and image/TTS commands are absent there while the infer media commands exist elsewhere. I did not run a live xAI OAuth setup in this read-only review.

## Solution Assessment

Unclear as filed: exposing the capability in Crestodian is the right direction, but the maintainable fix should reuse gateway/tool contracts and needs maintainer choice on the REPL command surface rather than copying infer command implementations.
