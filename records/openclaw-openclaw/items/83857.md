## Summary

Keep open: current main still has the Crestodian-specific routing gap. The normal agent, gateway, and infer media surfaces exist, but Crestodian remains a closed setup/helper command surface that does not parse or plan `image_generate`, `/tts`, or generic tool invocation.

## What This Changes

Crestodian should route xAI-backed image generation and text-to-speech requests from the REPL instead of only supporting those providers through `openclaw infer`.

## Best Possible Solution

Define a provider-neutral Crestodian media/tool contract that reuses existing agent or gateway media tools, or explicitly route users to the normal agent TUI for media work instead of adding xAI-specific branches.

## Reproduction Assessment

Yes. Current-main source inspection shows Crestodian messages resolve only to the closed setup/helper operation set, while `image_generate` and `tts` exist only on the normal agent/tool surfaces.

## Solution Assessment

Unclear. Exposing media in Crestodian is reasonable, but the best fix needs maintainer choice between direct generic gateway tool invocation, automatic handoff to the normal agent TUI, or documenting Crestodian as setup-only.
