## Summary

Keep open: current main still has a source-backed Crestodian routing gap. Image generation and TTS exist through the normal agent/gateway/infer paths, but Crestodian's parser and planner remain limited to setup, repair, model, agent, plugin, config, and gateway operations, so the reported media requests are not implemented there.

## What This Changes

Request Crestodian REPL support for routing image generation and TTS requests to enabled xAI-backed media tools/providers the way `openclaw infer` can.

## Best Possible Solution

Define a generic Crestodian media/tool routing contract that reuses existing agent/gateway media tools, or explicitly document that Crestodian must hand users to the normal agent TUI for media work; avoid xAI-specific routing.

## Reproduction Assessment

Yes, from current-main source inspection: Crestodian text resolves only to the closed setup/repair operation set and the planner is constrained to that same non-media vocabulary. I did not perform a live xAI OAuth run in this read-only review.

## Solution Assessment

Unclear as a concrete implementation path: exposing the capability is reasonable, but the best fix needs maintainer choice on whether Crestodian should invoke gateway tools directly or hand off to the normal agent TUI.
