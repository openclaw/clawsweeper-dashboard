## Summary

Keep open for normal PR handling: the linked Telegram message-loss regression is valid, current main still lets transcript-only OpenClaw assistant rows satisfy the latest-assistant reader, and this patch is a focused fix with no blocking code findings.

## What This Changes

The PR makes the session transcript latest-assistant reader skip OpenClaw `delivery-mirror` and `gateway-injected` assistant rows, with regression coverage for the mirror filter and provider boundary.

## Best Possible Solution

Land the narrow shared transcript-reader filter, preserving transcript mirror writes while preventing transcript-only OpenClaw rows from becoming user-visible latest assistant answers.

## Reproduction Assessment

## Solution Assessment

Yes. Filtering transcript-only OpenClaw assistant rows at the shared latest-assistant reader is the narrowest maintainable fix and keeps the provider boundary explicit so non-OpenClaw models with similar names still work.
