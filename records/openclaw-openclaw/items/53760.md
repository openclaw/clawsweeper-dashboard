## Summary

Keep this open. Current main still has no enforced DM-level sender tool-policy path: the active resolver is group/channel-scoped, direct sessions do not produce trusted group ids, Signal only exposes `toolsBySender` under group config, and Telegram's direct `toolsBySender` config exists in schema/types but is not reached by the enforced policy resolver.

## What This Changes

Request DM-level `toolsBySender` so allowlisted direct-message senders can receive enforced per-sender tool restrictions on Signal, Telegram, and similar channels.

## Best Possible Solution

Add a generic direct-message sender tool-policy contract, resolve trusted DM sender identity for Signal and Telegram, wire that resolver into both `createOpenClawCodingTools` and `applyFinalEffectiveToolPolicy`, align config/schema/docs for direct-message `tools` and `toolsBySender`, and add regression tests proving one allowlisted DM sender can retain tools while another is denied.
