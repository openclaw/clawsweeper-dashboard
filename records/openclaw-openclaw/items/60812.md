## Summary

Keep open. Current main still exposes failed delivery state, but Discord, Slack, Signal, WhatsApp, and the LINE typed-routing slice do not send the user a fallback reply when final dispatch fails; the closed LINE-specific report points back to this item as the canonical tracker.

## What This Changes

The issue asks for Discord, Slack, Signal, and WhatsApp to send user-visible fallback messages when agent reply dispatch fails, matching Telegram’s fallback behavior.

## Best Possible Solution

Add a small plugin-safe fallback selection/delivery helper, then wire Discord, Slack, Signal, WhatsApp, and LINE’s typed-routing slice through it with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: force final reply delivery or dispatch to fail in Discord, Slack, Signal, or WhatsApp and current main logs, reacts, throws, or returns without sending fallback text.

## Solution Assessment

Yes. A shared plugin-safe helper plus channel-specific wiring is the narrowest maintainable fix; copying fallback selection into every channel or deep-importing plugin internals would create drift.
