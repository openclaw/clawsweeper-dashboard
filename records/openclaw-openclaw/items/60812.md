## Summary

Keep open. Current main still records or reacts to dispatch failures in the affected channels, but Discord, Slack, Signal, and WhatsApp do not send the requested user-visible fallback message; LINE remains the closed duplicate slice for typed routing.

## What This Changes

Request typed user-visible delivery fallback for Discord, Slack, Signal, WhatsApp, and the related LINE path when inbound reply dispatch fails or produces no visible response.

## Best Possible Solution

Add a small plugin-safe delivery-fallback helper or SDK/runtime seam, then wire Discord, Slack, Signal, WhatsApp, and LINE through it with per-channel regression tests that preserve abort, stop, streaming cleanup, and silent-success behavior.

## Reproduction Assessment

Yes, from source inspection: force reply delivery or dispatch to fail in any affected channel and the current path logs, rethrows, sets reactions, or returns without sending fallback text. I did not run tests or live channel flows because this review was required to keep the checkout read-only.

## Solution Assessment

Yes. A shared plugin-safe helper plus channel-specific wiring is the narrowest maintainable fix; deep imports from bundled plugins into core or duplicated per-channel error-copy logic would be the weaker path.
