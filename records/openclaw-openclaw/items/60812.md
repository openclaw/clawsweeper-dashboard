## Summary

Keep open. Current main still has Telegram and LINE fallback behavior, but Discord, Slack, Signal, and WhatsApp continue to log, react, throw, or return when dispatch/final delivery fails without sending a user-visible fallback message.

## What This Changes

The issue requests user-visible dispatch failure fallback parity for Discord, Slack, Signal, and WhatsApp, matching Telegram’s fallback behavior.

## Best Possible Solution

Add a small plugin-safe delivery-fallback helper or SDK/runtime seam, then wire Discord, Slack, Signal, WhatsApp, and the LINE typed-routing slice through it with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: force reply delivery or dispatch to fail in Discord, Slack, Signal, or WhatsApp and current main logs, reacts, throws, or returns without sending fallback text.

## Solution Assessment

Yes. A shared plugin-safe helper plus channel-specific wiring is the narrowest maintainable repair; duplicating channel-local fallback selection or deep-importing plugin internals would be weaker.
