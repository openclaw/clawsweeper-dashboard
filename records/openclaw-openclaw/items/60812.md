## Summary

Keep open. Current main still has Telegram/LINE-style user-visible fallback behavior only in some paths; Discord, Slack, Signal, and WhatsApp still log, react, rethrow, or return without sending the requested fallback message when reply dispatch fails or produces no visible response.

## What This Changes

The issue requests user-visible fallback messages for Discord, Slack, Signal, and WhatsApp when reply dispatch fails or leaves no visible reply.

## Best Possible Solution

Add a small plugin-safe delivery-fallback helper or SDK seam, then wire Discord, Slack, Signal, WhatsApp, and the existing LINE fallback through it with regression coverage that preserves abort, stop, streaming cleanup, and silent-success behavior.

## Reproduction Assessment

Yes, from source inspection: force reply dispatch or delivery to fail in Discord, Slack, Signal, or WhatsApp and current main logs, rethrows, reacts, or returns without sending fallback text. I did not run live channel flows or tests because this review is read-only.

## Solution Assessment

Yes. A shared plugin-safe helper plus channel-specific wiring is the narrowest maintainable fix; duplicating error-message selection in each plugin would be harder to keep aligned and easier to regress.
