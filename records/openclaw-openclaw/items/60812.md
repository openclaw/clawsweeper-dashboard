## Summary

Keep open. Current main still has Telegram and LINE fallback behavior, but Discord, Slack, Signal, and WhatsApp do not send the requested user-visible fallback when reply dispatch or final delivery fails.

## What This Changes

The issue asks OpenClaw to add typed, user-visible reply-delivery failure fallbacks for Discord, Slack, Signal, and WhatsApp, with LINE typed-routing parity folded into the same tracker.

## Best Possible Solution

Add a small plugin-safe delivery-fallback helper or SDK/runtime seam, then wire Discord, Slack, Signal, WhatsApp, and the LINE typed-routing slice through it with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: force reply delivery or dispatch to fail in Discord, Slack, Signal, or WhatsApp and current main logs, throws, sets reactions, or returns without sending fallback text.

## Solution Assessment

Yes. A shared plugin-safe helper plus channel-specific wiring is the narrowest maintainable repair; duplicating channel-local error-copy logic or deep-importing core internals from plugins would be weaker.
