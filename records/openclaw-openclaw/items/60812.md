## Summary

Keep open. Current main still leaves the requested cross-channel fallback parity incomplete: Discord, Slack, Signal, WhatsApp, and LINE still log, rethrow, return, or use a hardcoded fallback instead of the typed user-visible fallback pattern.

## What This Changes

Request to add shared typed dispatch-failure fallback handling for Discord, Slack, Signal, WhatsApp, and LINE parity with Telegram.

## Best Possible Solution

Add a small plugin-safe fallback helper and wire the affected channel dispatch sites to send exactly one typed fallback only when no visible reply was delivered.

## Reproduction Assessment

Yes. By source inspection, forcing the affected channel deliverer or dispatch call to fail reaches log/rethrow/return paths without fallback text outside Telegram and LINE's generic catch; no live channel run was performed in this read-only review.

## Solution Assessment

Yes. A shared helper exported through the supported plugin SDK or channel-message runtime, plus channel-specific delivery-state wiring, is the narrowest maintainable fix; deep imports or duplicated per-channel error strings would be weaker.
