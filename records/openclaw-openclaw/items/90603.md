## Summary

Keep open: the PR is directionally useful for non-`main` default agents, but as written it can widen secrets/model discovery outside the caller-provided state directory and still lacks real behavior proof.

## What This Changes

The PR changes two secrets discovery helpers to seed paths from `resolveDefaultAgentId(config)`, removes the `agentId === "main"` early exits, and adds default-agent path tests.

## Best Possible Solution

Keep the default-agent direction, but resolve fallback agent paths from the caller's `stateDir` or env, preserve custom `agentDir` handling, and require redacted real behavior proof before merge.

## Reproduction Assessment

Yes, source-level. With an empty config or a configured `main` agent and a caller-supplied `stateDir` that differs from `process.env`, the patched loop falls through to `resolveAgentDir` and can add a process-global agent store to the scan.

## Solution Assessment

No as written. The maintainable fix is to use the configured default-agent identity while keeping path resolution owned by the explicit scan `stateDir` or env boundary.
