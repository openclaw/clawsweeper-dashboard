## Summary

Keep open: current main still has the hardcoded-main secrets discovery bug, and this PR is useful but not merge-ready because custom default agents with `agentDir` still scan state-dir auth/model stores alongside the configured custom directory.

## What This Changes

The PR changes secrets auth-profile and models.json discovery to seed from the configured default agent id, handles configured agent dirs in the scan loops, and adds regression tests for non-main default-agent path discovery.

## Best Possible Solution

Make the default auth/model seeds use the same custom-agentDir-aware path decision as configured agents, add default-custom regression coverage, and leave https://github.com/openclaw/openclaw/issues/90573 open unless the remaining surfaces are handled.

## Solution Assessment

No. The helper-level direction is right, but the best fix must make the default seed share the custom-dir-aware decision and avoid closing the broader issue unless the remaining surfaces are covered.
