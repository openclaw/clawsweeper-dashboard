## Summary

Keep open: current main and v2026.6.9 still have the original hard-coded `main` secrets discovery behavior, but this branch is not merge-ready because the default-agent seeds still scan stale paths for custom `agentDir` setups and the branch is dirty against current main.

## What This Changes

The PR updates secrets auth-profile and models.json path discovery to seed from `resolveDefaultAgentId(config)`, scopes configured agents without `agentDir` to the passed `stateDir`, and adds default-agent regression tests.

## Best Possible Solution

Make default auth/model discovery share one custom-dir-aware path decision, preserve the no-config implicit-main fallback only where it is intentional, refresh the branch against current main, and keep the broader issue open unless the remaining audit/session surfaces are covered.

## Reproduction Assessment

Yes, source-level. Current main and v2026.6.9 still hard-code `main` in the secrets auth/model scan helpers, and PR head still constructs default custom-agentDir seeds under `stateDir`.

## Solution Assessment

No. The helper-level direction is right, but the implementation should use one custom-dir-aware default path decision and avoid closing the broader issue unless the remaining surfaces are fixed.
