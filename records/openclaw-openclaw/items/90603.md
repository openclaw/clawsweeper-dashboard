## Summary

Keep open: the PR targets a real current-main and latest-release hardcoded-default-agent bug, but the branch still has a blocking stateDir-boundary defect and no after-fix real behavior proof.

## What This Changes

The PR changes two secrets discovery helpers to seed auth/model paths from the configured default agent ID, removes the `main` early exits, and adds default-agent path unit tests.

## Best Possible Solution

Keep the default-agent direction, but resolve fallback agent paths against the caller-provided stateDir/env, preserve custom `agentDir` support, and add negative tests proving no ambient process-state path is scanned.

## Reproduction Assessment

Yes, source-level. With an empty config or configured `main` agent and a caller-supplied stateDir that differs from `process.env`, the patched loop falls through to `resolveAgentDir(config, agentId)` and can add an ambient process-state path.

## Solution Assessment

No as written. The narrow maintainable fix is to use configured default-agent identity while keeping path resolution owned by the explicit scan stateDir/env boundary, not by ambient `process.env`.
