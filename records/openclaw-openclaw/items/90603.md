## Summary

Keep open: the linked default-agent issue is still real on current main, but this PR is not merge-ready because the removed `main` early exits let fallback scans call `resolveAgentDir()` with ambient `process.env`, and the PR still lacks after-fix real behavior proof.

## What This Changes

The branch updates secrets auth/model path discovery to use the configured default agent ID, removes hardcoded `main` loop branches, and adds default-agent path tests.

## Best Possible Solution

Keep the configured-default-agent direction, but resolve fallback/default scan paths through the caller-provided stateDir or env boundary while preserving configured custom `agentDir` support.

## Reproduction Assessment

Yes, source-level. With empty config or a configured `main` agent and a caller-supplied stateDir that differs from `process.env`, the patched loop falls through to `resolveAgentDir()` and can add an ambient process-state path.

## Solution Assessment

No as written. The best fix is to use configured default-agent identity while keeping path resolution owned by the explicit scan stateDir/env boundary, not ambient `process.env`.
