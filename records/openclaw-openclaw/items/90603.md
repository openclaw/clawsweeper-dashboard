## Summary

Keep open: current main still has the reported hardcoded default-agent path issue, but this branch is not merge-ready because the removed `main` branches can make scans fall through to ambient process-global agent paths and the external real-behavior proof gate is still unsatisfied.

## What This Changes

This PR changes two secrets path discovery helpers to seed configured default-agent paths, removes their `main` special cases, and adds focused unit tests.

## Best Possible Solution

Keep the PR open, preserve the configured-default direction, and resolve fallback agent paths against the explicit scan stateDir/env while still honoring configured custom `agentDir` entries.

## Reproduction Assessment

Yes, source-level. With an empty config or a configured `main` agent and a caller-supplied stateDir that differs from process env, the patched loop calls `resolveAgentDir(config, agentId)` without that boundary and can add an ambient process-state path.

## Solution Assessment

No as written. The narrow maintainable fix is to use configured default-agent identity while keeping fallback path resolution owned by the explicit scan stateDir/env boundary.
