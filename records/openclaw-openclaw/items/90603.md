## Summary

## What This Changes

The PR changes two secrets discovery helpers to seed auth/model paths from the configured default agent ID, removes `main` early exits, and adds default-agent path unit tests.

## Best Possible Solution

Keep the configured-default-agent direction, but preserve the explicit scan stateDir/env boundary for implicit/default agent paths, support configured custom `agentDir`, and add exact-path regression tests plus redacted real behavior proof before merge.

## Reproduction Assessment

Yes, source-level. `runSecretsAudit` passes an explicit `stateDir`, while the proposed `main` fallthrough calls `resolveAgentDir` without passing that scan env/stateDir, so it can resolve from ambient `process.env`.

## Solution Assessment

No as written. The direction is right, but the implementation should preserve scanner-owned stateDir/env resolution instead of delegating implicit paths to ambient process state.
