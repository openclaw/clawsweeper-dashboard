## Summary

Keep open: the PR addresses a real default-agent bug, but the current patch introduces a stateDir isolation regression in the same secrets discovery helpers and still lacks after-fix real behavior proof.

## What This Changes

The PR changes two secrets discovery helpers to seed paths from `resolveDefaultAgentId(config)`, removes the `agentId === "main"` early exits, and adds default-agent path tests.

## Best Possible Solution

Keep the default-agent direction, but resolve fallback agent paths against the caller-provided stateDir/env, preserve custom `agentDir` support, and add negative tests proving no ambient process-state path is scanned.

## Reproduction Assessment

Yes, source-level. With an empty config or a configured `main` agent and a caller-supplied stateDir that differs from `process.env`, the patched loop falls through to `resolveAgentDir(config, agentId)` and can add an ambient process-state path.

## Solution Assessment

No as written. The narrow maintainable fix is to use configured default-agent identity while keeping path resolution owned by the explicit scan stateDir/env boundary, not by ambient `process.env`.
