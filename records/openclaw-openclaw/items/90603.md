## Summary

Keep open: current main and v2026.6.6 still contain the reported hard-coded secrets discovery paths, and this PR is a useful narrow direction, but the patch is not merge-ready because removing the main early exits can add ambient process-global auth/model stores outside the caller-provided stateDir and the contributor has not supplied real behavior proof.

## What This Changes

The PR changes two secrets discovery helpers to seed paths from the configured default agent ID, removes main-specific early exits, and adds unit tests for default-agent path inclusion and deduplication.

## Best Possible Solution

Keep the default-agent direction, but resolve fallback/default paths against the caller-provided stateDir or env while preserving configured custom agentDir support, then add exact-path negative tests and real behavior proof.

## Reproduction Assessment

Yes, source-level. With empty config or a configured main agent and a caller-supplied stateDir that differs from process.env, the patched loop falls through to resolveAgentDir(config, agentId) and can add an ambient process-state path.

## Solution Assessment

No as written. The narrow maintainable fix is to use the configured default-agent identity while keeping path resolution owned by the explicit scan stateDir/env boundary rather than ambient process.env.
