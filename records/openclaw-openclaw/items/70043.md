## Summary

Keep open: current main still reproduces the reported unset-to-`minimal` Telegram reaction guidance path, and the broader open reaction-guidance enhancement #9835 does not replace this defaulting/stability bug.

## What This Changes

Investigate whether unset Telegram reaction configuration can still produce `MINIMAL` reaction guidance in GPT-backed Telegram system prompts.

## Best Possible Solution

Define a stable Telegram reaction default contract, then align runtime resolution, prompt injection, docs, generated config metadata, and regression tests with that contract.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: leave Telegram `reactionLevel` unset, resolve Telegram reaction guidance, and build the embedded system prompt to reach the exact `MINIMAL mode` text.

## Solution Assessment

No. The current implementation is internally consistent but not the best user-facing solution because a prompt-affecting default can surface as an apparent silent mode switch for existing unset sessions.
