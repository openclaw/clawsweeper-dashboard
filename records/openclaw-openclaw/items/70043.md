## Summary

Keep open: current main and the latest release still resolve an unset Telegram `reactionLevel` to `minimal`, pass that effective value into GPT agent prompt composition, and emit the reported `MINIMAL mode` text without a config write. The related Telegram reactions feature item at https://github.com/openclaw/openclaw/issues/9835 overlaps the policy area but does not supersede this narrower default/session-stability question.

## What This Changes

The issue asks for upgrade-safe Telegram reaction prompt behavior so GPT Telegram sessions do not appear to switch to `MINIMAL` reaction guidance without an explicit user config change.

## Best Possible Solution

Maintainers should define one Telegram reaction default and session-stability contract for existing and fresh sessions, then update the resolver, prompt path, docs, and focused tests with doctor or migration handling only if the config contract changes.

## Reproduction Assessment

Yes. Source inspection on current main is enough: an unset Telegram `reactionLevel` resolves to `minimal`, the Telegram adapter exposes that as reaction guidance, and the shared prompt builder emits the exact `MINIMAL mode` line.

## Solution Assessment

Unclear. The current implementation matches documented fresh-install defaults, but the best fix depends on a maintainer decision about existing-session prompt stability versus the documented Telegram default.
