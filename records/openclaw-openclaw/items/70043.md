## Summary

Keep open: current main still has the unset Telegram `reactionLevel` path that resolves to `minimal` and injects the exact reported prompt text. The related Telegram reaction guidance enhancement is adjacent, but it does not supersede this defaulting/stability regression.

## What This Changes

The issue asks OpenClaw to prevent unset GPT Telegram sessions from unexpectedly receiving `MINIMAL` reaction prompt guidance, or to make that default explicit and stable.

## Best Possible Solution

Define the stable Telegram reaction default contract for existing unset configs, then align runtime resolution, prompt injection, docs, generated config metadata, and focused regression tests.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: leave Telegram `reactionLevel` unset, resolve Telegram reaction guidance, and build the embedded system prompt to reach the exact `MINIMAL mode` text.

## Solution Assessment

No. The current implementation is internally traceable but not a safe final answer because defaulting and prompt semantics for existing unset Telegram installs need maintainer policy before code changes.
