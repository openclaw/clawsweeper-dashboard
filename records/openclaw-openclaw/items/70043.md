## Summary

Keep open: current main and the latest release still resolve unset Telegram `reactionLevel` to `minimal`, forward that into agent prompt composition, and can emit the reported MINIMAL guidance. The behavior is source-reproducible, but changing it needs a maintainer product decision because current docs and tests also encode `minimal` as the unset default.

## What This Changes

The issue asks OpenClaw to avoid or clarify prompt-visible Telegram MINIMAL reaction guidance appearing in GPT-backed sessions when the user did not explicitly configure reaction settings.

## Best Possible Solution

Maintainers should define the unset Telegram reactionLevel compatibility contract, then align the resolver, prompt adapter, docs, schema metadata, and focused tests; if shipped semantics change, include doctor or migration guidance.

## Reproduction Assessment

Yes, at source level. With Telegram reactionLevel unset, current main resolves the level to minimal, forwards Telegram reaction guidance into embedded prompt composition, and emits the reported MINIMAL mode text.

## Solution Assessment

Unclear. The current implementation matches current docs and focused tests, so the best fix depends on a maintainer decision about existing-session prompt stability versus the documented Telegram default.
