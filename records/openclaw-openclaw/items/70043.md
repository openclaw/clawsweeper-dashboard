## Summary

Keep open: current main and v2026.6.6 still resolve an unset Telegram `reactionLevel` to `minimal`, pass that value into agent prompt composition, and emit the reported `MINIMAL mode` text without any config write. The broader Telegram reactions feature at https://github.com/openclaw/openclaw/issues/9835 is still open, but it does not replace this narrower default/session-stability question.

## What This Changes

The issue reports that GPT-backed Telegram sessions can start showing `MINIMAL` reaction guidance in the live prompt despite no explicit Telegram reaction-level config write.

## Best Possible Solution

Maintainers should define whether the unset `minimal` default may affect existing Telegram sessions, then make one compatibility-aware resolver, prompt, docs, and test update with doctor or migration handling if the config contract changes.

## Reproduction Assessment

Yes. Source inspection on current main shows unset Telegram `reactionLevel` resolves to `minimal`, the Telegram prompt adapter forwards that as reaction guidance, and the prompt builder emits the exact `MINIMAL mode` text; I did not run a live Telegram GPT session in this read-only review.

## Solution Assessment

Unclear. The current implementation matches docs and focused tests, but the best fix depends on a maintainer decision about upgraded-session prompt stability versus the documented Telegram default.
