## Summary

Keep open: current main still resolves an unset Telegram `reactionLevel` to `minimal`, forwards that into Telegram prompt guidance, and can emit the reported `MINIMAL mode` prompt line without a config write. The remaining work is a maintainer product/API decision about unset Telegram reaction defaults and prompt stability, not an already-fixed cleanup item.

## What This Changes

The issue asks OpenClaw to stop unset Telegram reaction defaults from surfacing as unexpected `MINIMAL` reaction guidance in GPT-backed Telegram sessions, or to define a stable prompt/default contract.

## Best Possible Solution

Define the stable unset Telegram reaction default and prompt-stability contract, then align the resolver, prompt adapter, system prompt text, docs/config metadata, and focused regression tests around that contract.

## Reproduction Assessment

Yes, by source inspection. Leave `channels.telegram.reactionLevel` unset, resolve Telegram channel reaction guidance, and build the embedded system prompt to reach the exact reported `Reactions are enabled for Telegram in MINIMAL mode.` text.

## Solution Assessment

Unclear until maintainers choose the product contract. Current behavior is internally consistent with docs and tests, so changing it safely requires a deliberate default/prompt policy decision rather than a narrow mechanical bug fix.
