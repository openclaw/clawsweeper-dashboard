## Summary

Keep open: current main and the latest release still resolve an unset Telegram `reactionLevel` to `minimal`, pass that effective value into GPT agent prompt composition, and emit the reported `MINIMAL mode` text without any config write. The related Telegram reactions feature issue is still open, but it does not replace this narrower default/session-stability question.

## What This Changes

The issue asks OpenClaw to stop GPT-backed Telegram sessions from surfacing `MINIMAL` reaction prompt guidance as an apparent silent setting change when no reaction config was written.

## Best Possible Solution

Maintainers should define whether the unset `minimal` default may affect existing Telegram sessions, then make one compatibility-aware resolver/prompt/docs/test change with a doctor or migration path if the config contract changes.

## Reproduction Assessment

Yes. Source inspection on current main shows unset Telegram `reactionLevel` resolves to `minimal`, the Telegram prompt adapter forwards that as reaction guidance, and the prompt builder emits the exact `MINIMAL mode` text; I did not run a live Telegram GPT session in this read-only review.

## Solution Assessment

Unclear. The current implementation matches docs and tests, but the best fix depends on a maintainer decision about upgraded-session prompt stability versus the documented Telegram default.
