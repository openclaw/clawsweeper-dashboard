## Summary

Keep open: current main still defaults unset Telegram `reactionLevel` to `minimal`, forwards that into Telegram prompt guidance, and can emit the reported `MINIMAL mode` line without a config write. The remaining question is a Telegram default and prompt-stability product contract, not an obsolete or already-fixed report.

## What This Changes

The issue asks OpenClaw to avoid or clarify apparent silent `MINIMAL` Telegram reaction guidance when GPT-backed Telegram sessions have no explicit reaction-level config.

## Best Possible Solution

Define the stable Telegram reaction default contract for existing unset configs, then align runtime resolution, prompt text, docs/config metadata, and focused regression tests around that contract.

## Reproduction Assessment

Yes, by source inspection. Leave Telegram `reactionLevel` unset, resolve Telegram channel reaction guidance, and build the embedded system prompt to reach the exact reported `Reactions are enabled for Telegram in MINIMAL mode.` text.

## Solution Assessment

Unclear until maintainers decide the product contract. Current behavior is coherent with docs and tests, so changing it safely requires a policy decision for existing unset Telegram configs rather than a narrow mechanical bug fix.
