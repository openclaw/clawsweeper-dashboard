## Summary

Keep open: current main still resolves an unset Telegram `reactionLevel` to `minimal`, passes that into Telegram prompt guidance, and emits the reported `MINIMAL mode` prompt text; this remains a product/default-contract question rather than an obsolete report.

## What This Changes

Request to prevent Telegram GPT sessions from surfacing `MINIMAL` reaction guidance when `reactionLevel` is unset, or to define a stable default and prompt contract for that case.

## Best Possible Solution

Define the stable Telegram reaction default contract for existing unset configs, then align runtime resolution, prompt text, docs, generated config metadata, and focused regression tests around that contract.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: leave Telegram `reactionLevel` unset, resolve Telegram reaction guidance, and build the embedded system prompt to reach the exact reported `MINIMAL mode` text.

## Solution Assessment

Unclear until maintainers decide the product contract. The current behavior is coherent with docs and tests, but changing it safely requires a policy choice for existing unset Telegram configs rather than a narrow mechanical bug fix.
