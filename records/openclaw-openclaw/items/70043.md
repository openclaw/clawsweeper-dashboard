## Summary

Keep open: current main still resolves an unset Telegram reaction level to `minimal`, forwards that into Telegram prompt guidance, and can emit the reported `MINIMAL mode` prompt text. The remaining decision is a Telegram default/prompt-stability contract question, not an obsolete report.

## What This Changes

The issue asks OpenClaw to avoid or explicitly stabilize prompt-affecting Telegram `minimal` reaction guidance for GPT Telegram sessions when no reaction setting was configured by the user.

## Best Possible Solution

Define the stable Telegram reaction default contract for existing unset configs, then align runtime resolution, prompt text, docs/config metadata, and focused regression tests around that contract.

## Reproduction Assessment

Yes, by source inspection. Leave Telegram `reactionLevel` unset, resolve Telegram reaction guidance, and build the embedded system prompt to reach the exact reported `Reactions are enabled for Telegram in MINIMAL mode.` text.

## Solution Assessment

Unclear until maintainers decide the product contract. Current behavior is coherent with docs and tests, so changing it safely requires a policy decision for existing unset Telegram configs rather than a narrow mechanical bug fix.
