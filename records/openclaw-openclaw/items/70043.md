## Summary

Keep open: current main still resolves an unset Telegram `reactionLevel` to `minimal` and injects the reported `MINIMAL mode` prompt text, so this remains a real defaulting and prompt-stability question rather than an obsolete report.

## What This Changes

The issue asks OpenClaw to avoid an apparent silent switch to Telegram reaction guidance `MINIMAL` for GPT Telegram sessions when no reaction-level setting was explicitly changed.

## Best Possible Solution

Define the stable Telegram reaction default contract for existing unset configs, then align runtime resolution, prompt text, docs, generated config metadata, and focused regression tests around that contract.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: leave Telegram `reactionLevel` unset, resolve Telegram reaction guidance, and build the embedded system prompt to reach the exact reported `MINIMAL mode` text.

## Solution Assessment

No. The current behavior is traceable and tested, but deciding whether unset Telegram configs should keep injecting `minimal` guidance for existing GPT sessions needs maintainer product judgment before a safe code change.
