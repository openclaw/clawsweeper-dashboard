## Summary

Current main still has the reported behavior: unset Telegram `reactionLevel` resolves to `minimal`, becomes Telegram prompt reaction guidance, and reaches the exact `MINIMAL mode` system prompt text during embedded runs. The related open enhancement #9835 is broader reaction-guidance work and does not supersede this defaulting/stability bug.

## What This Changes

Make GPT-backed Telegram sessions avoid appearing to switch to `MINIMAL` reaction guidance when the user has not explicitly configured Telegram reaction behavior.

## Best Possible Solution

Make Telegram reaction prompt behavior explicit and stable: choose the intended unset/default contract, add any needed doctor or migration path for existing installs, and keep runtime code, tests, generated config metadata, TypeScript comments, and public Telegram docs aligned.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: use Telegram config with no `reactionLevel`, resolve Telegram reaction guidance, and build the embedded system prompt; the source path and existing tests reach the exact reported `MINIMAL mode` text.

## Solution Assessment

No. The current implementation is internally explainable, but it is not the best user-facing solution because a prompt-affecting default can surface as a silent effective mode change; the safer fix needs maintainer direction on default semantics and migration.
