## Summary

## What This Changes

The issue asks OpenClaw to prevent unset Telegram reaction guidance from appearing like a spontaneous MINIMAL-mode switch in GPT-backed sessions.

## Best Possible Solution

Maintainers should decide the Telegram reaction default compatibility contract, then update one canonical resolver, prompt behavior, docs, and focused tests, with doctor or migration handling if the effective config contract changes.

Yes. Source inspection on current main and v2026.6.8 shows unset Telegram `reactionLevel` resolves to `minimal`, the Telegram prompt adapter forwards that value, and the prompt builder emits the exact reported MINIMAL text.

## Solution Assessment

Unclear. The current implementation matches docs and tests, but the best fix depends on a maintainer decision about upgraded-session prompt stability versus the documented Telegram default.
