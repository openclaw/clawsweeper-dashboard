## Summary

Keep open: current main and the latest release still resolve unset Telegram reaction guidance to `minimal` and inject the reported `MINIMAL mode` prompt text. The source path is clear, but changing it needs a maintainer decision about documented defaults versus upgraded-session prompt stability.

## What This Changes

The issue reports that GPT-backed Telegram sessions can receive `MINIMAL` reaction prompt guidance even though the user did not explicitly configure a Telegram reaction level.

## Best Possible Solution

Maintainers should define the upgrade-safe Telegram reaction default and session-prompt contract, then make a narrow prompt/defaulting change with tests for unset, explicit `minimal`, `ack`, `off`, and upgraded-session behavior.

## Reproduction Assessment

Yes. Source inspection shows unset Telegram `reactionLevel` resolves to `minimal`, Telegram exposes that as reaction guidance, and the prompt builder emits the exact `MINIMAL mode` text; I did not run a live Telegram GPT session in this read-only review.

## Solution Assessment

Unclear. The current implementation matches docs and tests, but the user-visible session change is source-reproducible; maintainers need to decide whether unset `reactionLevel` should inject prompt guidance for upgraded sessions.
