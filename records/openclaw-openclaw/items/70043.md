## Summary

Keep open: current main still resolves an unset Telegram `reactionLevel` to `minimal`, forwards that effective value into agent prompt composition, and emits the reported `MINIMAL mode` text. The related Telegram reactions feature issue remains open, but it does not replace this narrower compatibility/session-stability question.

## What This Changes

The issue asks OpenClaw to avoid or clarify an apparent silent switch to Telegram `MINIMAL` reaction prompt guidance in GPT-backed sessions when `reactionLevel` was never explicitly configured.

## Best Possible Solution

Maintainers should decide the unset/default compatibility contract, then align the Telegram resolver, prompt injection, docs, and tests around that single behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows unset Telegram `reactionLevel` resolves to `minimal`, Telegram forwards that as reaction guidance, and the agent prompt builder emits the exact reported text; I did not run a live Telegram GPT session in this read-only review.

## Solution Assessment

Unclear. The current implementation matches current docs and tests, but the best fix depends on a maintainer decision about upgraded-session prompt stability versus the documented Telegram default.
