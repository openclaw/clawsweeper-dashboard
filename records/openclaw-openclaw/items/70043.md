## Summary

Keep open: current `main` and the latest release still resolve unset Telegram `reactionLevel` to `minimal`, forward that into Telegram prompt guidance, and render the reported `MINIMAL mode` prompt text without a config write. The remaining work is a maintainer product/API decision about existing-session prompt stability, not an already-implemented fix or a stale missing-info report.

## What This Changes

The issue asks OpenClaw to prevent unset Telegram reaction defaults from surfacing as an apparent runtime switch to `MINIMAL` prompt guidance in GPT Telegram sessions.

## Best Possible Solution

Maintainers should decide the Telegram reaction default contract for upgraded sessions, then implement that as a narrow compatibility-aware prompt/defaulting change with regression coverage for unset, explicit `minimal`, and disabled reaction levels.

## Reproduction Assessment

Yes, by source inspection: an unset Telegram `reactionLevel` resolves to `minimal`, Telegram exposes that as agent reaction guidance, and the prompt builder emits the exact `MINIMAL mode` line. I did not run a live Telegram GPT session in this read-only review.

## Solution Assessment

Unclear: the current implementation is internally consistent with docs and tests, but it does not answer whether upgraded existing sessions should receive prompt-affecting defaults without explicit opt-in. A maintainer product/API decision should come before an implementation PR.
