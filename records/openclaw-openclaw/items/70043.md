## Summary

Keep open: current main and latest release v2026.6.8 still resolve an unset Telegram reactionLevel to minimal, pass that into agent prompt composition, and emit the reported MINIMAL mode text. The broader Telegram reactions feature at https://github.com/openclaw/openclaw/issues/9835 overlaps but does not supersede this narrower existing-session default stability question.

## What This Changes

The issue asks OpenClaw to stop unset Telegram reaction guidance from surfacing as an apparent MINIMAL mode switch in GPT-backed Telegram sessions.

## Best Possible Solution

Maintainers should define the unset Telegram reactionLevel compatibility contract, then update the resolver, prompt behavior, docs, and focused tests together with doctor or migration handling if existing config semantics change.
