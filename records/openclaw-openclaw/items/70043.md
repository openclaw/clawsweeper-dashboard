## Summary

Keep open: current main and v2026.6.10 still resolve an unset Telegram reactionLevel to minimal, forward it into embedded prompt composition, and render the reported MINIMAL guidance; changing that behavior is a maintainer compatibility decision because docs and tests also encode the minimal default.

## What This Changes

The issue reports that GPT-backed Telegram sessions can show MINIMAL reaction guidance in the live prompt even when the user did not explicitly configure Telegram reaction settings.

## Best Possible Solution

Maintainers should define the unset Telegram reactionLevel compatibility contract, then align the resolver, prompt adapter, docs/types/schema metadata, and focused tests; if shipped config semantics change, add doctor or migration guidance.

## Reproduction Assessment

Yes, at source level. With Telegram reactionLevel unset, current main resolves the level to minimal, forwards Telegram reaction guidance into embedded prompt composition, and emits the reported MINIMAL mode text.

## Solution Assessment

Unclear. The current implementation matches current docs and focused tests, so the best fix depends on a maintainer decision about existing-session prompt stability versus the documented Telegram default.
