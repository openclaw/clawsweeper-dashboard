## Summary

Keep this PR open, but it is not merge-ready. Current main still lacks Telegram forum-topic deletion, and the PR is useful source work; however the current PR head still accepts non-decimal strict-integer strings on destructive delete paths, and the branch predates the current Telegram plugin runtime layout.

## What This Changes

The PR branch adds Telegram forum-topic deletion through `delete`/`topic-delete`, a `deleteForumTopicTelegram` transport helper, `topicId` schema support, docs, and focused tests.

## Best Possible Solution

Refresh the feature onto the current Telegram plugin layout, keep the deletion behavior behind the existing delete-action gate, validate destructive IDs as decimal-only integers before coercion, and land focused docs, tests, and changelog coverage while leaving #11414 as the broader forum-topic lifecycle umbrella.

## Reproduction Assessment

Yes. Without live Telegram, the remaining parser bug is reproducible by passing `0x10` or `1e2` through the PR-head `readNumberParam(..., { strict: true, strictInteger: true })` path; JavaScript converts both to finite integers before the new check.

## Solution Assessment

No. The feature direction is narrow and belongs in the Telegram plugin, but the current patch is not the best mergeable solution until it is rebased onto current main and the strict-integer parser validates raw decimal syntax before `Number(...)`.
