## Summary

Keep open for focused follow-up: Telegram forum-topic deletion is still missing on current main, but this PR is not mergeable as-is because it needs to be ported to the current Telegram plugin runtime, hardened for destructive ID parsing, and backed by real Telegram proof.

## What This Changes

The PR adds Telegram forum topic deletion through `topic-delete`/`deleteForumTopic`, a `deleteForumTopicTelegram` transport helper, message-tool schema/docs updates, and focused tests.

## Best Possible Solution

Port or replace the useful deletion behavior in the current `extensions/telegram` runtime, align it with the canonical forum-topic lifecycle tracker, enforce decimal-only ID validation, and require redacted live Telegram proof before merge.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature PR. Current-main source inspection gives high-confidence proof that Telegram exposes topic create/edit only and has no `topic-delete` or `deleteForumTopic` runtime path.

## Solution Assessment

No. The feature direction is reasonable, but this branch needs to target the current Telegram plugin runtime, harden destructive ID validation, and fit the canonical lifecycle tracker before it is a maintainable solution.
