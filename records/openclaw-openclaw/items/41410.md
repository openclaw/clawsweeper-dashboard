## Summary

The PR adds Telegram forum-topic deletion through `topic-delete`/`deleteForumTopic`, a transport helper, message-tool/docs updates, and focused tests.

## Best Possible Solution

Port the deletion behavior into the current Telegram plugin runtime, align it with the canonical forum-topic lifecycle tracker, validate destructive IDs as base-10 integers, and require redacted live Telegram proof.

## Solution Assessment

No. The feature direction is reasonable, but this branch needs to target the current extension runtime, harden destructive ID validation, and supply real Telegram proof before it is the maintainable solution.
