## Summary

Keep open: current main still lacks Telegram forum-topic deletion, and the PR is useful but not merge-ready because its new strict integer mode still accepts non-decimal literals for destructive IDs and it lacks the required changelog entry.

## What This Changes

The PR adds Telegram forum topic deletion through `deleteForumTopic`/`topic-delete`, keeps explicit topic IDs on `delete`, updates transport/schema/docs, and adds focused tests.

## Best Possible Solution

Refresh or replace the PR on current `extensions/telegram`, keep deletion behind the existing delete-action gate, require raw decimal-only IDs before destructive calls, add the changelog entry, and leave #11414 as the broader lifecycle tracker.

## Reproduction Assessment

Yes for the PR review finding: source inspection shows `strictInteger` validates after numeric coercion, and a focused Node check confirms `0x10` and `1e2` become integer values. No live Telegram account is needed for this parser path.

## Solution Assessment

No. The feature direction fits the Telegram plugin, but the patch should not merge until strict integer parsing rejects raw non-decimal strings and the required user-facing changelog entry is added.
