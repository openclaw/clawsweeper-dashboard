## Summary

Keep open: current main still lacks Telegram forum-topic deletion, but this branch is not mergeable as-is because it conflicts with the current Telegram plugin runtime layout, still has a destructive ID parsing concern, and lacks real Telegram proof.

## What This Changes

The PR adds a Telegram forum-topic delete action, transport helper, shared action/spec updates, docs, and focused unit tests.

## Best Possible Solution

Port or replace the useful deletion behavior in the current `extensions/telegram` runtime, enforce decimal-only IDs, align it with https://github.com/openclaw/openclaw/issues/11414, and require redacted live Telegram proof before merge.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature PR. Source inspection on current main confirms Telegram exposes topic create/edit only and has no `topic-delete` or `deleteForumTopic` runtime path.

## Solution Assessment

No. The feature direction is valid, but this branch needs to be ported to the current Telegram plugin runtime, hardened for decimal-only destructive IDs, and backed by live Telegram proof.

  - body: The new `deleteForumTopic` handler is added under a core Telegram action file that no longer exists on current main. Please move this behavior into `extensions/telegram/src/action-runtime.ts` and the current action-discovery path instead of reviving the stale runtime layout.
