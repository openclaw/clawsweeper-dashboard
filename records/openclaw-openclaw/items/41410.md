## Summary

Keep open: current main still lacks Telegram forum-topic deletion, but this PR is not merge-ready because it is stale against the current Telegram plugin runtime, lacks live Telegram proof, and leaves a destructive ID parsing flaw.

## What This Changes

The PR adds Telegram forum-topic deletion through `topic-delete`/`deleteForumTopic`, a transport helper, shared action metadata, docs, parsing hardening, and focused tests.

## Best Possible Solution

Port topic deletion into the current Telegram plugin runtime beside create/edit, enforce decimal-only IDs for destructive paths, align docs/tests with current files, and attach live Telegram proof before merge.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature PR. Source inspection and the PR diff give high-confidence evidence for current-main behavior and the parser flaw, but no live after-fix Telegram proof is present.

## Solution Assessment

No. The feature direction matches Telegram and grammY APIs, but the current branch needs to be ported to the current runtime, reject non-decimal ID literals, and provide live Telegram proof before it is the narrow maintainable fix.
