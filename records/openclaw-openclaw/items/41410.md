## What This Changes

The PR adds Telegram forum-topic deletion through topic-delete/delete action routing, a deleteForumTopic transport helper, shared action/schema/docs updates, and unit tests.

## Best Possible Solution

Port a fixed deletion path into the current extensions/telegram runtime as part of the canonical forum-topic lifecycle work, with decimal-only ID validation, current action wiring, docs/tests/config alignment, and live Telegram proof.

## Reproduction Assessment

Yes for the PR finding by source inspection: strictInteger checks Number.isInteger after Number(trimmed), so strings like 0x10 or 1e2 still pass. For the requested feature, source inspection of current main is enough to show topic-delete/deleteForumTopic is absent.

## Solution Assessment

No, not as-is. The direction is useful, but this branch is stale, lacks live proof, and leaves malformed non-decimal IDs accepted on destructive paths; a current-main port or replacement is safer.
