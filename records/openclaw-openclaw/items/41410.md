## Summary

Keep open. This is a real Telegram feature PR, but current main still lacks forum topic deletion, the branch is conflicting and stale against the current plugin runtime, a destructive ID-validation issue remains, and the external PR has no real Telegram proof.

## What This Changes

The branch adds Telegram `topic-delete`/`deleteForumTopic` support, a transport helper, docs/tests, a `topicId` tool alias, and stricter ID parsing for delete paths.

## Best Possible Solution

Refresh this into the current Telegram plugin runtime with decimal-only explicit ID parsing, focused docs/config/runtime coverage, and live Telegram proof before merge, while keeping the broader lifecycle scope aligned with https://github.com/openclaw/openclaw/issues/11414.

## Reproduction Assessment

Yes for the review blockers: current-main source inspection shows no `topic-delete`/`deleteForumTopic` runtime path, and PR-head `readNumberParam` accepts non-decimal numeric literals before `strictInteger` validation. A live Telegram account is not needed to reproduce those code-path blockers.

## Solution Assessment

No. The feature belongs in the Telegram plugin, but this branch needs to be ported to `extensions/telegram/src/action-runtime.ts`, hardened with decimal-only ID parsing, and backed by real Telegram proof before merge.

  - body: The adapter dispatches `deleteForumTopic`, but the branch adds that handler in legacy `src/agents/tools/telegram-actions.ts`; current main routes Telegram actions through `extensions/telegram/src/action-runtime.ts`, which is absent from the PR head and has no `deleteForumTopic` branch.   - confidence: 0.9

  - confidence: 0.88
