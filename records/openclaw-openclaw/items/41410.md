## Summary

Keep open. The feature is still missing on current main and the PR contains useful Telegram topic-delete prior art, but it is not merge-ready because it targets a stale action-runtime layout, still has a destructive ID-parsing issue, and lacks required live Telegram proof.

## What This Changes

The PR adds Telegram `topic-delete`/`deleteForumTopic` routing, a transport helper, message-tool schema and docs updates, and focused tests for deleting forum topics.

## Best Possible Solution

Port or replace the useful deletion behavior in the current `extensions/telegram` runtime, enforce decimal-only IDs, align it with the canonical Telegram lifecycle tracker, and require redacted live Telegram proof before merge.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature PR. Current-main source inspection gives high-confidence proof that Telegram exposes topic create/edit only and has no `topic-delete` or `deleteForumTopic` runtime path.

## Solution Assessment

No. The feature direction is valid, but this branch needs to be ported to the current Telegram plugin runtime, hardened for decimal-only destructive IDs, and backed by live Telegram proof.
