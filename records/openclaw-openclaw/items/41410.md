## Summary

Keep this PR open. Current main still lacks Telegram forum-topic deletion in the message action surface, so the contribution remains relevant, but the proposed patch is not ready to merge because its new strict integer parser still accepts non-decimal numeric strings for destructive delete paths.

## What This Changes

The PR adds Telegram forum-topic deletion through `topic-delete` and legacy `delete` routing, a `deleteForumTopicTelegram` transport helper, stricter ID parsing, docs, and targeted tests.

## Best Possible Solution

Keep the focused deletion support, but update it on current main with explicit `topicId`/`threadId` requirements, `deleteMessage` gating, decimal-only integer validation for destructive IDs, Telegram docs, and targeted action/runtime/transport tests. Leave #11414 as the broader forum-topic lifecycle umbrella.

## Reproduction Assessment

Yes. Source inspection on current main shows the missing `topic-delete` action map entry, runtime branch, transport helper, docs entry, and tests; the remaining PR defect can be reproduced by passing non-decimal strings such as `0x10` or `1e2` through the new `strictInteger` delete paths.

## Solution Assessment

No, not as currently proposed. The direction is narrow and mostly aligned with the Telegram plugin boundary, but the parser must reject non-decimal integer strings before the destructive delete path is safe to merge.

## Review Findings

Overall correctness: patch is incorrect

- **[P2] Reject non-decimal strict integer inputs:** `src/agents/tools/common.ts:183`
