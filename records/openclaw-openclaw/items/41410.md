## Summary

Keep open. The feature is useful and still missing on current main, but this PR needs real Telegram behavior proof before merge and the patch still has a destructive-ID validation bug for strict integer parsing.

## What This Changes

The PR adds Telegram forum topic deletion via `topic-delete` and legacy `delete` topic-id routing, plus a `deleteForumTopicTelegram` transport helper, docs, and tests.

## Best Possible Solution

Port the topic-delete support onto the current Telegram plugin runtime, reject non-base-10 ID strings before destructive calls, and validate it with focused tests plus live Telegram proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a new Telegram capability. Source inspection shows current main has topic create/edit but no topic-delete/deleteForumTopic path.

## Solution Assessment

No, not as-is. The direction is maintainable, but the branch needs a current-main port, decimal-only ID validation, and live Telegram proof before it is the right merge path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Reject non-decimal strict integers before deleting topics:** `src/agents/tools/common.ts:151-152`
