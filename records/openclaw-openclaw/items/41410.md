## Summary

Keep open: current main still lacks Telegram forum-topic deletion support, and this PR addresses a real feature gap, but the patch is not merge-ready because its strict integer parser still accepts non-decimal ID literals on destructive delete paths and the required changelog entry is missing.

## What This Changes

The PR adds Telegram `topic-delete`/`deleteForumTopic` routing, transport support, schema/docs updates, and focused tests while preserving explicit topic-ID compatibility for `delete`.

## Best Possible Solution

Land a refreshed Telegram plugin change that adds topic deletion behind the existing delete-action gate, requires decimal-only explicit IDs, includes focused tests/docs/changelog coverage, and leaves #11414 as the broader topic-lifecycle tracker.

## Reproduction Assessment

Yes for the remaining review bug: source inspection of the PR-head parser shows `Number("0x10")` and `Number("1e2")` pass before integer validation. No live Telegram account is needed to validate that path.

## Solution Assessment

No. The feature belongs in the Telegram plugin, but this patch should not merge until strict integer parsing is decimal-only and the user-facing changelog entry is added.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Reject non-decimal strict integer inputs:** `src/agents/tools/common.ts:154-156`
  - body: The new `strictInteger` mode still validates only after `Number(trimmed)`, so inputs like `0x10` and `1e2` become valid integers and can reach destructive Telegram delete paths. Validate the raw string as a base-10 integer before coercion.
