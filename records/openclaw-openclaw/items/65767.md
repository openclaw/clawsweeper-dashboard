## What This Changes

The PR backfills `owner_key` from legacy `owner_session_key`, rebuilds `flow_runs` without the legacy column, and adds regression tests for legacy-only and hybrid schemas.

## Best Possible Solution

Land one canonical TaskFlow SQLite migration that preserves rows, normalizes `owner_key`, removes the legacy `owner_session_key` constraint, recreates current indexes, and keeps regression coverage for upgraded legacy and hybrid databases.

## Solution Assessment

Yes. Rebuilding the table inside the TaskFlow SQLite migration layer is the narrow maintainable fix; changing the webhook request contract would contradict the bound-session docs and would not repair upgraded databases.
