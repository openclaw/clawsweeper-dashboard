## What This Changes

This PR refactors TaskFlow SQLite schema migration to backfill `owner_key` from `owner_session_key`, rebuild `flow_runs` without the legacy column, and add migration regression tests.

## Best Possible Solution

Land one canonical TaskFlow SQLite migration that preserves existing rows, normalizes `owner_key`, removes the legacy `owner_session_key` constraint, recreates indexes, and keeps regression coverage for legacy and hybrid upgraded databases.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a legacy `flow_runs` table with `owner_session_key TEXT NOT NULL` can survive current `ensureSchema()`, then managed-flow persistence inserts only `owner_key`.

## Solution Assessment

Yes. Rebuilding the TaskFlow SQLite table in the migration layer is the narrow maintainable fix; maintainers should choose one canonical implementation rather than changing the webhook request owner contract.
