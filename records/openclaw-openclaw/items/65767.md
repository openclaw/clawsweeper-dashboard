## Summary

Keep open. The PR addresses a real TaskFlow SQLite migration gap that is still present on current main and the latest release, but it needs contributor after-fix real behavior proof and maintainer consolidation with the overlapping open migration PR.

## What This Changes

The PR rebuilds legacy `flow_runs` tables to migrate `owner_session_key` data into `owner_key`, drops the legacy column, and adds TaskFlow SQLite migration tests.

## Best Possible Solution

Land one canonical TaskFlow SQLite migration that preserves rows, normalizes `owner_key`, removes the legacy `owner_session_key` constraint, recreates indexes, and keeps regression coverage for legacy and hybrid upgraded schemas.

Yes. Source inspection shows a legacy `flow_runs` table with `owner_session_key TEXT NOT NULL` can survive current `ensureSchema()`, while managed-flow persistence inserts only `owner_key`; the linked issue supplies the Docker/webhook curl and log path.

## Solution Assessment
