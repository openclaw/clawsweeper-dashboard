## What This Changes

The issue reports that memory indexing can permanently stall when markdown chunking splits an emoji surrogate pair and the embedding path sends the resulting lone surrogate to a provider.

## Best Possible Solution

Review and land or replace the linked PR with a refreshed patch that guards the coarse chunk boundary and sanitizes every provider-bound text field across inline, structured, and remote batch embedding paths.

Yes, at source level. Current main's coarse split can cut 39 ASCII characters plus a flower emoji at a 40-code-unit boundary into lone high and low surrogate chunks before embedding.

## Solution Assessment

Yes in direction. The maintainable fix is the linked chunker boundary guard plus embed-boundary sanitization, but it should cover inline text, structured text parts, and remote batch chunks before the issue closes.
