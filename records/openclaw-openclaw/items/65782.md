## Summary

Keep open. Current main still has the coarse UTF-16 chunk split and unsanitized embedding handoff, and the paired open PR with closing syntax has not merged, so this issue should remain open until that PR or a narrow replacement lands.

## What This Changes

Memory indexing can stall when `chunkMarkdown` splits an emoji or supplementary-plane character into lone UTF-16 surrogate halves and the embedding path sends that text unsanitized.

## Best Possible Solution

Land or replace the linked PR with a current-main patch that guards the coarse chunk boundary and sanitizes top-level plus structured embedding text before provider calls.

## Reproduction Assessment

Yes. The current coarse split can be reproduced at source level with `maxChars=40` and `39` ASCII characters followed by `🌸`, which creates lone high and low surrogate chunks before provider embedding.

## Solution Assessment

Yes in direction. The narrow maintainable solution is the linked chunker boundary guard plus embed-boundary sanitizer, with structured text-part coverage included before merge.
