## Summary

Keep open. Current main still has the reported failure shape: the outer `chunkMarkdown` pass slices by fixed UTF-16 code-unit width, the embedding path forwards text without a lone-surrogate sanitizer, and open PR #65783 is already paired to this issue with closing syntax.

## What This Changes

Fix memory indexing so chunking cannot split emoji or supplementary-plane surrogate pairs and embedding calls cannot send lone surrogates to providers.

## Best Possible Solution

Land or replace PR #65783 with the same narrow two-layer fix: preserve surrogate pairs in the outer coarse chunk boundary, sanitize lone surrogates at the embedding boundary, and keep focused regression tests for both paths.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: with a 10-token budget, `maxChars` is 40, and a mostly-Latin line of 39 ASCII characters followed by an emoji splits between `\uD83C` and `\uDF38` in the current outer loop.

## Solution Assessment

Yes. The best fix direction is the linked PR's root-cause guard plus embed-boundary sanitizer; that is narrower and more robust than only expanding retry rules or only fixing one chunking path.
