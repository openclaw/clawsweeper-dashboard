## Summary

Keep open. Current main still has the reported coarse chunker surrogate-pair split and raw embedding handoff, and open PR #65783 is already paired to this issue with closing syntax, so this should remain open until that PR or a narrow replacement lands.

## What This Changes

Fix memory indexing stalls caused by UTF-16 surrogate pairs being split during memory chunking and then sent unsanitized to embedding providers.

## Best Possible Solution

Land or replace #65783 with the same two-layer fix: preserve surrogate pairs at the coarse chunk boundary, sanitize lone surrogates before embedding provider calls, and keep focused regression tests for both paths.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: with `tokens=10`, `maxChars` is 40, and `39 ASCII + flower emoji + 39 ASCII` makes the current outer loop split between `\uD83C` and `\uDF38`.

## Solution Assessment

Yes. The proposed fix direction is the narrowest maintainable solution because it removes the chunker root cause and adds an embed-boundary guard; retry-policy changes alone would still send invalid payloads.
