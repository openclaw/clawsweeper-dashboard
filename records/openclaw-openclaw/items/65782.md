## Summary

Keep open. Current main still has the reported chunker and embedding-boundary failure shape, and open PR #65783 is already paired to this issue with closing syntax, so the issue should stay open until that PR or an equivalent replacement lands.

## What This Changes

The issue reports that memory indexing can stall permanently when `chunkMarkdown` splits an emoji or supplementary-plane character across a UTF-16 surrogate-pair boundary and the embedding path sends the resulting lone surrogate to a provider.

## Best Possible Solution

Land or replace #65783 with the same two-layer fix: avoid surrogate-pair cuts in the coarse chunk boundary, sanitize lone surrogates before embedding provider calls, and keep focused regression tests for both paths.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: with `tokens=10`, `maxChars` is 40, and `39 ASCII + 🌸 + 39 ASCII` makes the current coarse loop split between `\uD83C` and `\uDF38`.

## Solution Assessment

Yes. The linked fix direction is the narrowest maintainable solution because it removes the chunker root cause and adds an embed-boundary guard against future invalid strings; retry-policy changes alone would still send invalid payloads.
