## Summary

Keep open. Current main still has the reported coarse chunker surrogate-pair split and unsanitized embedding handoff, and the paired open PR with closing syntax is still unmerged/conflicting, so this issue should remain open until that PR or a narrow replacement lands.

## What This Changes

The issue reports that memory indexing can stall when the markdown chunker splits an emoji surrogate pair and the embedding path forwards a lone surrogate to the provider.

## Best Possible Solution

Land or replace the paired PR with a current-main patch that preserves surrogate pairs at the coarse chunk boundary, sanitizes top-level and structured embedding text before provider calls, and leaves broader retry/checkpointing work to the existing follow-up issues.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: with `maxChars=40`, `39` ASCII characters followed by the flower emoji makes the current coarse slice boundary land between `\uD83C` and `\uDF38` before any guard runs.

## Solution Assessment

Yes in direction. The narrow maintainable fix is the two-layer chunker plus embed-boundary sanitizer already targeted by the paired PR, but the final patch should also cover structured text parts and be ported to current main.
