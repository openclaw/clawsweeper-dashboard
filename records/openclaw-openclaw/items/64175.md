## Summary

Keep open. Current main still drops OpenRouter's `supported_parameters` tool-support signal and treats dynamic OpenRouter models with missing `compat.supportsTools` as tool-capable, so the reported Sonar Deep Research primary model can still receive normal agent tools; the related open PR only addresses fallback filtering.

## What This Changes

The issue asks OpenClaw to avoid tool-capable execution for OpenRouter models such as `perplexity/sonar-deep-research` when OpenRouter metadata does not advertise `tools`.

## Best Possible Solution

Preserve OpenRouter `supported_parameters.includes("tools")` in runtime capabilities and map known toolless dynamic OpenRouter models to `compat.supportsTools: false`, with focused regression tests and a changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level path is clear: current OpenRouter metadata lacks `tools` for the reported model, current main drops that signal, and embedded runs default absent tool compat to enabled.

## Solution Assessment

Yes. The narrowest maintainable fix is to feed OpenRouter tool metadata into the existing `compat.supportsTools` and `supportsModelTools()` gate; adding a new non-tool agent mode would be broader than needed.
