## Summary

Keep this PR open. The runtime change is focused and appears to preserve the allowlist/safe-bin checks, but it changes documented exec-approval behavior and the public exec docs still say redirections are unsupported, so the docs need to be updated before merge.

## What This Changes

The PR exempts exact stdout/stderr-to-/dev/null and fd 1/2 duplication redirections from POSIX exec authorization planner blocking and adds behavioral tests for allowed and blocked redirect forms.

## Best Possible Solution

Keep the runtime exception narrow, update the public exec approval docs to describe the supported harmless subset and blocked file-touching forms, then let exact-head CI/security checks gate merge.

## Reproduction Assessment

Yes. Source inspection shows current main treats every shell `redirect` risk as unanalyzable, and `requiresExecApproval` prompts on analysis misses in allowlist/on-miss mode.

## Solution Assessment

Mostly yes for the runtime layer: the planner is the right place to distinguish analyzable harmless redirects before allowlist evaluation. The proposed solution is incomplete until the public exec approval docs are updated to match the new supported subset.
