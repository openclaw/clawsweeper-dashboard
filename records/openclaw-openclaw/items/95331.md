## Best Possible Solution

Update the advanced exec approval docs to match the exact harmless redirect subset, then merge only if maintainers accept the no-prompt policy change and exact-head checks pass.

## Reproduction Assessment

Yes, source-reproducible: current main treats redirect risks as unanalyzable and the allowlist/on-miss caller prompts when analysisOk is false. I did not run tests because this review is read-only.

## Solution Assessment

Mostly yes for the runtime layer: the planner is the right place to distinguish analyzable stream-only redirects before allowlist evaluation. It is not the best complete merge state until the advanced docs are updated and maintainers accept the policy relaxation.
