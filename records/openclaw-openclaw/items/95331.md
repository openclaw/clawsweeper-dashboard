Land this implementation only if maintainers accept the narrow no-prompt redirect policy; otherwise keep the current fail-closed redirect behavior.

## Solution Assessment

Yes if maintainers accept the policy: the planner is the narrowest layer to distinguish stream-only redirects before existing per-segment allowlist checks run. The safer alternative is to keep all redirects fail-closed.
