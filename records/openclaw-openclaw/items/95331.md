## What This Changes

The PR changes POSIX exec authorization so exact stream-only redirections to `/dev/null` or between fd 1/2 no longer force approval when the command segments otherwise satisfy allowlist or safe-bin checks, and updates matching tests and docs.

## Best Possible Solution

Keep redirections fail-closed unless a maintainer explicitly sponsors this narrow no-prompt policy; if sponsored, land a refreshed maintainer-backed PR with the existing tests and docs alignment.
