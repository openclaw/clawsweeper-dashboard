## Summary

Keep open: current main and the latest release still lack a durable memory-index retention setting or an age-pruning path for old daily memory files. The request is useful, but it adds public config and deletes indexed rows, so it needs maintainer direction before implementation.

## What This Changes

The issue requests a configurable memory-index retention window that limits dated memory files while exempting evergreen memory sources.

## Best Possible Solution

Define a maintainer-approved retention contract for durable memory indexes, then implement it through memorySearch config/schema/docs, sync pruning, and regression coverage for dated notes, evergreen files, and extraPaths.

## Reproduction Assessment

Yes for the source-level gap: current main has no memorySearch retention config and the sync path only prunes indexed paths that disappear. I did not recreate the reporter's 80-day database because the remaining request is for a new bounded-retention capability, not a broken existing contract.

## Solution Assessment

Unclear until maintainers choose the product contract. A default-off retention setting may be right, but the best fix needs explicit decisions on config location, date basis, evergreen exemptions, extraPaths, and pruning semantics.
