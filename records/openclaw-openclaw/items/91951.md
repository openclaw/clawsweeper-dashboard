## Summary

Keep open: current main and the latest release still lack a memory-file age-retention window, while the current indexer discovers all memory files and prunes rows only when a source path disappears. The request is useful, but it adds public config and pruning semantics that need maintainer product review before implementation.

## What This Changes

The issue requests an opt-in memory-search retention setting that limits indexed daily memory files and prunes old chunks while preserving evergreen memory paths.

## Best Possible Solution

Define one documented memory-core retention contract, then implement it in the canonical discovery and reindex cleanup path with schema, help text, docs, and regression coverage while preserving unset config as current behavior.
