## Summary

Keep open: current main still indexes every discovered builtin memory file and only removes rows whose paths disappear, so an opt-in bounded indexing window is still missing.

## What This Changes

Request to add an opt-in memory-search retention setting that skips and prunes old daily memory files while preserving evergreen memory files.

## Best Possible Solution

Define a memory-plugin-owned retention contract for dated daily notes, then make sync/reindex atomically exclude and delete aged-out builtin rows while keeping default behavior unchanged when unset.

## Reproduction Assessment

Yes at source level: current main indexes all discovered builtin memory files and only removes rows for paths that disappear. The reporter's exact database size was not independently reproduced in this read-only review.

## Solution Assessment

Unclear as proposed: the retention behavior belongs in the existing sync/reindex lifecycle, but maintainers still need to choose the public config name, age source, exemptions, and interaction with temporal decay.
