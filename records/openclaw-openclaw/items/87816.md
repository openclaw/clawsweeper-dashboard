## Summary

Keep open. Current main and v2026.6.8 implement the VoiceDesign half through the merged Xiaomi PR, but VoiceClone remains unlisted, undocumented, and without a maintainer-confirmed sample-audio config contract.

## What This Changes

The issue requests Xiaomi TTS support for MiMo voicedesign and voiceclone models, including request-body handling for models that do not use preset voices.

## Best Possible Solution

Keep this issue open as the canonical VoiceClone follow-up, preserve the shipped VoiceDesign behavior, and add provider-local VoiceClone support only after maintainers choose the sample-audio config and validation contract.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.8 do not list or special-case `mimo-v2.5-tts-voiceclone`, while they do list and special-case VoiceDesign. I did not run a live Xiaomi VoiceClone request because this read-only review has no credentialed sample-audio setup.

## Solution Assessment

No for the issue's exact omit-voice sketch. The maintainable fix is a provider-local VoiceClone contract for sample audio, docs, tests, and live proof, not a generic branch that drops `audio.voice` for both custom-voice models.
