## Summary

Keep open. Current main and the latest release still advertise only the two existing Xiaomi TTS models and always send `audio.voice`; the request remains valid, but the proposed fix needs adjustment because Xiaomi's current docs require `audio.voice` for VoiceClone data URIs while VoiceDesign omits preset voice selection.

## What This Changes

Request to add Xiaomi MiMo VoiceDesign and VoiceClone TTS model support and handle their model-specific Xiaomi audio payloads.

## Best Possible Solution

Decide the public Xiaomi TTS config shape, then implement provider-local VoiceDesign and VoiceClone model-mode support in `extensions/xiaomi` with docs and focused request-payload tests.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not as a bug reproduction, but yes at source level: current main lacks the requested model metadata and always serializes `audio.voice`, while Xiaomi's docs require model-specific payloads.

## Solution Assessment

Is this the best way to solve the issue? No for the exact suggested patch: VoiceDesign should omit preset `voice`, but VoiceClone should keep `audio.voice` for a base64 data URI; maintainers should first settle the public config wording/schema.
