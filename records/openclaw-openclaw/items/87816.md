## Summary

Keep open. The linked merged work and current release now cover Xiaomi MiMo VoiceDesign, but current main and v2026.6.6 still do not list or document `mimo-v2.5-tts-voiceclone`, and upstream Xiaomi docs show VoiceClone needs a different sample-audio contract than the issue's omit-voice sketch.

## What This Changes

Request support for Xiaomi MiMo VoiceDesign and VoiceClone TTS model modes in the Xiaomi speech provider, including model listing and model-specific audio payload handling.

## Best Possible Solution

Keep this issue open as the canonical VoiceClone follow-up, then implement provider-local VoiceClone support after the public sample-audio config contract is decided, with docs and request-payload tests.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.6 list and test VoiceDesign but not `mimo-v2.5-tts-voiceclone`, and the docs omit VoiceClone. I did not run a live Xiaomi request because this read-only review has no credentialed live setup.

## Solution Assessment

No for closing or for the exact suggested patch. The best path is to preserve the shipped VoiceDesign behavior and settle the VoiceClone sample-audio config shape before adding the model and payload tests.
