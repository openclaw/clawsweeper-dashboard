## Summary

Keep open: current main does not implement the requested env var, but this PR is not correct as written because it forces Jiti filesystem caching on when the OpenClaw override is absent and it leaves loader cache reuse blind to the effective cache setting. The external PR also still lacks after-fix real behavior proof.

## What This Changes

The PR adds an `OPENCLAW_JITI_CACHE_DIR` fsCache override to the shared plugin loader, SDK root alias, and Matrix runtime Jiti paths, plus disables `fsCache` in one git-path regression fixture.

## Best Possible Solution

Land a narrow version that only passes `fsCache` for a trimmed non-empty OpenClaw override, preserves Jiti env/default behavior otherwise, scopes cached loaders by the effective cache option, adds focused tests, and documents the accepted env surface after maintainer agreement.

## Reproduction Assessment

Yes. The PR defect is source-reproducible: with `OPENCLAW_JITI_CACHE_DIR` unset and `JITI_FS_CACHE=false`, the diff still passes explicit `fsCache: true`, while Jiti v2.7.0 applies user options after env-derived defaults.

## Solution Assessment

No. The requested override is plausible, but the current patch should omit `fsCache` unless the OpenClaw-specific override is non-empty, isolate cache reuse by the effective value, and be rebased onto the current Matrix wrapper shape.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Preserve Jiti fsCache defaults when unset:** `src/plugins/sdk-alias.ts:455-458`
  - body: `resolveJitiFsCache()` falls back to `true`, and the duplicated root-alias/Matrix call sites use the same pattern, so OpenClaw now passes an explicit `fsCache` option even when the new env var is absent or blank. Jiti resolves `JITI_FS_CACHE`/`JITI_CACHE` first and then applies user options, so this regresses deployments that disable filesystem cache writes; omit `fsCache` unless the OpenClaw override is non-empty and apply the same rule to every duplicated loader path.
  - confidence: 0.94
- **[P2] Scope loader cache keys by fsCache:** `src/plugins/sdk-alias.ts:466`
  - body: This change makes shared loader options depend on an env-controlled cache directory, but the module loader cache key still only includes `tryNative` and aliases. A test or long-lived process that changes the effective cache setting can reuse a Jiti loader created with a different cache path or cache-disabled state; include the effective `fsCache` value or otherwise prevent cross-setting reuse.
  - confidence: 0.84
