## Summary

Keep this PR open, but request changes before merge. Current main still lacks `OPENCLAW_JITI_CACHE_DIR`, and prior review comments correctly identify that this implementation changes Jiti cache env semantics, misses cache isolation/docs, and is now stale against the current Matrix wrapper.

## What This Changes

The PR adds an `OPENCLAW_JITI_CACHE_DIR` fsCache override to shared plugin Jiti options, SDK root alias, Matrix runtime wrapper, and disables fsCache in one git-path regression fixture.

## Best Possible Solution

Land a narrow current-main implementation that only supplies a trimmed OpenClaw cache directory when non-empty, otherwise leaving Jiti defaults/env controls untouched, with cache isolation, focused tests, and changelog/docs coverage.

## Reproduction Assessment

Yes. Apply the PR, leave `OPENCLAW_JITI_CACHE_DIR` unset, set `JITI_FS_CACHE=false`, and the new fallback still passes explicit `fsCache: true`, overriding Jiti 2.6.1's env-derived default.

## Solution Assessment

No. The requested override is reasonable, but the patch should omit `fsCache` unless the OpenClaw-specific override is set, and it should be rebased to current main's loader structure.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve Jiti fsCache defaults when unset:** `src/plugins/sdk-alias.ts:457-458`
  - body: When `OPENCLAW_JITI_CACHE_DIR` is absent or blank, `resolveJitiFsCache()` returns `true`, so OpenClaw now passes an explicit `fsCache` option. Jiti resolves `JITI_FS_CACHE`/`JITI_CACHE` before applying user options, so this regresses deployments that already disable filesystem cache writes through Jiti env vars. Omit `fsCache` when there is no OpenClaw override, and mirror that behavior in direct loaders.
  - confidence: 0.94
- **[P2] Include fsCache in loader cache isolation:** `src/plugins/sdk-alias.ts:466`
  - body: The Jiti options become dependent on an environment-controlled cache directory, but the loader cache key still only covers `tryNative` and aliases. Tests or long-lived processes that change the effective cache setting can reuse a loader created with a different `fsCache`; include the effective value in the cache key/config cache or otherwise scope reuse.
  - confidence: 0.84
- **[P2] Document the new operator env var:** `src/plugins/sdk-alias.ts:455`
  - body: This introduces a user-facing environment variable but does not update the changelog or operator-facing env docs. Add an Unreleased `### Changes` entry and document how the OpenClaw override interacts with Jiti's own cache env vars.
  - confidence: 0.82
