## Best Possible Solution

If maintainers accept the env surface, implement it as an opt-in override that omits `fsCache` when unset or blank, scopes loader reuse by the effective cache setting, updates current loader paths, adds focused tests/docs, and requires redacted real runtime proof.

## Solution Assessment

No. The override is plausible, but this implementation should preserve upstream Jiti defaults when unset, scope loader reuse by the effective cache option, rebase away stale Matrix assumptions, and get maintainer acceptance for the new env surface.
