## Summary

Keep open: the reported ACP timeout/control bug is real and this branch contains useful fixes, but it is not mergeable as-is because after-fix real OpenClaw ACP proof is still absent and the new detail/logging paths can expose unredacted backend-controlled data.

## What This Changes

The PR removes automatic ACP timeout config-option replay, routes `/acp timeout` through runtime options, adds fail-soft handling for unsupported ACP controls, forwards acpx `getCapabilities({ handle })`, updates ACP docs/changelog, and bumps the acpx dependency to `0.7.0`.

## Best Possible Solution

Land a consolidated ACP control-handling fix for https://github.com/openclaw/openclaw/issues/78546 that stops auto-injected timeout controls from breaking narrow backends, keeps explicit unsupported user setters strict where appropriate, preserves redacted diagnostics, and includes live Claude or Pi ACP proof.

## Reproduction Assessment

Yes. Current main still emits `timeout` as an ACP config option and fail-fast rejects it when not advertised, and the linked live report shows the same `ACP_BACKEND_UNSUPPORTED_CONTROL` path on `2026.5.8`; I did not run the live Claude ACP smoke locally.

## Solution Assessment

No, not as-is. The timeout-handling direction is valid, but the branch needs redaction fixes for new detail/log paths, a careful rebase over landed ACP error/acpx changes, and after-fix real OpenClaw ACP proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Redact RPC details before returning error text:** `src/acp/runtime/error-text.ts:43`
  - body: `formatAcpRuntimeErrorText` is used in ACP command and dispatch replies, and this now returns `data.details` from backend JSON-RPC errors directly. Those details are backend-controlled and can include private paths, prompts, endpoints, or token-shaped strings, bypassing the redacted ACP error-chain formatter that landed for these surfaces. Please run the detail through the existing redaction path before appending it.
  - confidence: 0.88
- **[P2] Stop logging raw config option values:** `src/acp/control-plane/manager.runtime-controls.ts:155`
  - body: This verbose warning logs the raw rejected config option `value`. Generic ACP options and `backendExtras` are backend-defined, so a rejected value can contain private tokens, endpoints, or local paths and would be persisted in verbose logs. Log the key/control only, or redact the value before rendering it.
  - confidence: 0.84
