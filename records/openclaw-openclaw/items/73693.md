## Summary

Keep open: current `main` still contains the ACP timeout-control failure path, and this draft PR is a plausible fix, but it needs changes, maintainer scope judgment, and after-fix real behavior proof before merge.

## What This Changes

The PR removes automatic ACP timeout config mirroring, routes `/acp timeout` through local runtime options, adds JSON-RPC error-detail helpers, makes unsupported ACP controls fail-soft, forwards acpx capability input, and updates tests/docs/changelog.

## Best Possible Solution

Land a focused ACP timeout fix that keeps manager-owned timeout enforcement, avoids sending unsupported timeout config, keeps safety-related controls visible or fail-closed, and includes targeted tests plus a live Claude ACP smoke.

## Reproduction Assessment

Yes, source inspection plus linked live diagnostics provide a high-confidence reproduction path: current `main` still emits `timeout` as ACP config and applies it before the turn, while https://github.com/openclaw/openclaw/issues/78546#issuecomment-4414010521 shows a live Claude ACP failure on that key. I did not run the live harness in this read-only review.

## Solution Assessment

No, not as submitted. Removing timeout mirroring and routing `/acp timeout` locally is the right direction, but the broad fail-soft behavior and dependency wording need correction and the branch needs after-fix real behavior proof.
