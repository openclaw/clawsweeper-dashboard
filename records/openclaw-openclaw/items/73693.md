## Summary

Keep open. Current main still has the ACP timeout/config-control failure this PR targets, so the work is not obsolete, but the patch is not merge-ready because its changelog claims `acpx@0.6.2`-backed advertised-key reconciliation even though this PR does not bump the dependency and that version is not published/pinned here.

## What This Changes

The PR removes ACP timeout backend mirroring, reroutes ACP timeout commands to local runtime options, adds JSON-RPC detail extraction plus fail-soft unsupported-control handling, forwards acpx capability input, and updates ACP docs/tests/changelog.

## Best Possible Solution

Correct the changelog to describe the behavior that actually ships with the pinned dependency, then finish review/validation of this focused ACP fix or land a clean superseding PR that preserves the same runtime boundary and test coverage.

## Reproduction Assessment

Yes. Current main provides a source-level reproduction path: ACP timeout becomes a backend `timeout` config key and `/acp timeout` calls the same backend config path, while the PR discussion includes a direct `claude-agent-acp` JSON-RPC rejection for that unsupported key.

## Solution Assessment

Mostly yes. Keeping timeout enforcement local and fail-softing only classified unsupported backend controls is a narrow maintainable fix, but the submitted changelog is not the best solution because it promises a dependency-backed `acpx@0.6.2` behavior this PR does not ship.
