## Summary

Keep open. The PR targets a real current-main ACP/acpx failure and the runtime approach is mostly sound, but the submitted changelog still claims an `acpx@0.6.2` bump and dependency-backed reconciliation that this PR does not ship.

## What This Changes

This PR removes ACP timeout backend config mirroring, routes ACP timeout commands through local runtime options, adds JSON-RPC ACP error-detail extraction and fail-soft unsupported-control handling, forwards acpx capability input, and updates tests, docs, and changelog.

## Best Possible Solution

Correct the changelog dependency wording, then land this focused ACP fix or a clean superseding PR with the targeted ACP/acpx tests and a live smoke confirming timeout stays local and unsupported backend controls no longer collapse turns into opaque errors.

## Reproduction Assessment

Yes. Current main provides a high-confidence source-level reproduction path: timeout runtime options are converted into a backend `timeout` config key and applied before the turn, while the PR body and related #74225 include a direct `claude-agent-acp` rejection for that unsupported key.

## Solution Assessment

Mostly yes for runtime behavior. The narrow maintainable path is to keep timeout local, fail-soft only classified unsupported controls, forward handle-aware capabilities, and reword the changelog so it does not claim the future `acpx@0.6.2` bump shipped here.
