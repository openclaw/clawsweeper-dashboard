Full review comments:

- **[P2] Redact backend detail before reply text:** `src/acp/runtime/error-text.ts:49-53`
  - body: `formatAcpRuntimeErrorText` is used by ACP command/dispatch reply surfaces, but this new detail line renders JSON-RPC `data.details` from the backend without passing through redaction. A backend error containing a token, credential-shaped value, or private path could be posted back to chat; redact the summary before building `detailLine` and add a regression test.
  - confidence: 0.86
- **[P2] Stop logging raw ACP config values:** `src/acp/control-plane/manager.runtime-controls.ts:155`
  - body: The new verbose warning includes `value=${value}` for rejected config options. Because generic `/acp set` and backendExtras accept arbitrary strings, a rejected secret-like value would be written raw to gateway verbose logs; omit the value or redact it before logging.
  - confidence: 0.84
- **[P2] Correct stale acpx version wording:** `CHANGELOG.md:21`
  - body: The changelog says advertised-key reconciliation is plumbed through the bumped `acpx@0.6.2`, but current main already pins `acpx` 0.7.0 and this PR does not change package metadata or lockfiles. Reword this as handle/capability forwarding on the existing dependency state so release notes do not claim a dependency bump from this PR.
  - confidence: 0.9
