## Summary

Keep open: current main still has the Telegram target-resolution gap, and the linked closing PR #77156 is open rather than merged.

## What This Changes

The issue reports that Telegram group/forum-topic replies routed through the agent message tool reject numeric topic targets that the CLI accepts.

## Best Possible Solution

Review and land a narrow fix like PR #77156 so shared message-tool target resolution accepts Telegram numeric group/topic ids without requiring the Telegram plugin hook to be active.

## Reproduction Assessment

Yes, with source-level confidence: the issue provides exact dry-run inputs and current main shows those targets can miss `looksLikeTargetId` fallback and end at `unknownTargetError`. I did not execute the failing runtime path in this read-only pass.

## Solution Assessment

Yes: the narrow maintainable fix is to align shared id-like detection with the Telegram numeric topic grammar already supported by the plugin. Because PR #77156 uses closing syntax and is still open, this issue should remain open until that PR lands or is replaced.
