## Summary

Keep open: this is a valid current Telegram delivery bug, not a stale or duplicate cleanup. The reported v2026.5.2 failure matches current source boundaries: group/topic replies can require `message(action=send)`, Telegram owns numeric topic grammar, and the shared message-tool target resolver can still fall through to `Unknown target` when that plugin grammar is not available on the agent path.

## What This Changes

Make agent `message(action=send)` accept the same Telegram numeric group/topic targets that `openclaw message send` accepts.

## Best Possible Solution

Keep this open and fix the shared message-action target resolution or Telegram target artifact so agent and CLI delivery use the same plugin-owned Telegram grammar, with regression coverage for topic targets and current-source defaults.

## Reproduction Assessment

Yes, with source-level confidence: the issue provides exact `message(action=send)` dry-run inputs and current source shows the resolver path that can emit the same `Unknown target` error. I did not run the failing path in this read-only review, so this is source-reproducible rather than fully reproduced.

## Solution Assessment
