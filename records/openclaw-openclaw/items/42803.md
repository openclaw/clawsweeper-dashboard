- `impact:session-state`: The affected commands stop, inspect, or replace active agent session state from Feishu.

## Summary

Keep open: current main and the latest release only partially address the Feishu fast-path regression. `/stop` now uses the Feishu control lane, but `/status` and `/new` still resolve to the normal same-chat key and can wait behind active same-chat work.

## What This Changes

The issue requests restoring Feishu text command fast-path behavior so `/stop`, `/new`, and `/status` bypass active-run queueing.

## Best Possible Solution

Review or replace the linked Feishu fix with a current-main patch that routes safe immediate controls and reset commands onto the right non-blocking lane, with focused active-run concurrency tests.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.6.8` still route `/status` and `/new` to the Feishu base chat key, and same-key tasks run FIFO behind active same-chat work.

## Solution Assessment

Unclear as a final implementation path. A separate Feishu lane is the right mechanism, but maintainers still need to choose the safe command whitelist and `/new` ordering semantics before landing the linked approach.
