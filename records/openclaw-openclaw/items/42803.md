Keep open. ## What This Changes

The issue reports that Feishu text commands such as `/stop`, `/new`, and `/status` queue behind active agent runs instead of using the documented command-only fast path.

## Best Possible Solution

Review, land, or replace the linked Feishu fix with a focused queue-classification change that covers the accepted command-only fast paths, preserves normal-message ordering, and adds authorization-aware regression coverage.

## Reproduction Assessment

Yes, as a source-level reproduction. Current main still awaits the Feishu sequential queue, and the current key policy leaves `/status` and `/new` on the base chat key while only abort-like text and `/btw` move out of band.

## Solution Assessment

Unclear for the linked direct-dispatch approach as written. The maintainable direction is to extend Feishu queue-key classification for the accepted command-only set with authorization and ordering tests after maintainers confirm the final scope.
