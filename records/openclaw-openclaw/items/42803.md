- `impact:session-state`: The affected commands control session state and status, including aborting or starting sessions during an active run.

## Summary

Keep open: current main and the latest release only partially address the Feishu active-run command regression. `/stop` has an out-of-band Feishu lane now, but `/status` and `/new` still resolve to the normal same-chat sequential key and can wait behind an active Feishu dispatch.

## What This Changes

The issue asks for Feishu text commands such as `/stop`, `/new`, and `/status` to bypass the same-chat queue and execute immediately during an active agent run.

## Best Possible Solution

Consolidate on one Feishu command-lane fix that preserves normal same-chat ordering while making documented authorized command-only controls reach command handling without waiting behind an active run.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: an active Feishu same-chat task occupies the base sequential key, and current main maps `/status` and `/new` to that same base key instead of a bypass lane. I did not run a live Feishu tenant repro in this read-only review.

## Solution Assessment

Unclear as a final implementation path: the issue is valid, but the best fix needs a maintainer decision on which text commands are safe to bypass and how to avoid fragmenting partial PRs. A single Feishu command-lane policy with focused regression coverage is safer than landing overlapping partial fixes.
