## Summary

Keep open. Current main still only partially addresses the regression: abort-like Feishu text such as `/stop` is routed out of the base lane, but `/status` and `/new` still resolve to the normal per-chat queue despite the documented command-only fast path, and an open linked PR is still the active fix candidate.

## What This Changes

Restore Feishu text slash-command fast-path behavior so `/stop`, `/new`, and `/status` execute immediately during an active agent run instead of waiting behind the per-chat queue.

## Best Possible Solution

Extend Feishu's sequential-key policy to route the intended authorized command-only fast paths out of the per-chat lane, with focused tests for `/stop`, `/new`, `/status`, authorization, debounce cleanup, and normal-message ordering.

## Reproduction Assessment

Yes, as a source-level reproduction. Current main awaits the Feishu sequential queue, and the current key policy leaves `/status` and `/new` on the base chat key while only abort-like text and `/btw` move out of band.

## Solution Assessment

Unclear for the linked direct-dispatch PR as written. The safer maintainable direction is to extend the Feishu sequential-key classification with authorization-aware queue/debounce tests rather than broadly bypassing dispatch before the command scope is reviewed.
