## Summary

Keep open. Current main still has the reported Comfy local-mode SSRF gap, and the linked fix PR was closed unmerged rather than landing a current-main fix.

## What This Changes

Allow the Comfy plugin's local or explicit `allowPrivateNetwork` mode to work when `baseUrl` is a DNS hostname that resolves to a private IP.

## Best Possible Solution

Patch the Comfy-owned network policy to build a hostname-scoped private-network policy for the configured `baseUrl` when local mode or explicit `allowPrivateNetwork` is active, with focused regression coverage and no core-wide SSRF default change.

## Reproduction Assessment

Yes, source-reproducible on current main: configure Comfy local mode with a DNS `baseUrl` resolving to RFC1918, and the policy builder returns `{}` before strict fetch blocks the resolved private IP. I did not run the live LAN setup during this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix is in the Comfy policy builder: keep the configured hostname allowlist and allow private DNS answers only for that opted-in endpoint. A shared SSRF default change or new config knob would be broader than needed.
