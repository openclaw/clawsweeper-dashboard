## Summary

Keep open: current main and the latest release still keep the Comfy DNS-hostname gate before building the private-network SSRF policy, so the reported local-DNS path remains source-reproducible. The direct fix PR was closed unmerged and the open xAI PR is adjacent, not a Comfy fix.

## What This Changes

The issue reports Comfy local mode blocking `baseUrl` DNS hostnames that resolve to private LAN IPs even when private-network access is explicitly enabled.

## Best Possible Solution

Keep this open for a narrow Comfy plugin fix that allows explicit local/private-network DNS hostnames through an origin-scoped SSRF policy, with regression coverage and maintainer/security review.

## Reproduction Assessment

Yes. Source inspection shows a DNS hostname such as `comfy.local.example.com` leaves `resolveComfyNetworkPolicy` with no policy even when local mode has private-network intent, and the shared fetch guard then rejects private DNS answers.

## Solution Assessment

Yes as a direction: the best fix is narrow and plugin-owned, reusing an existing scoped SSRF policy shape rather than adding a config key or broad bypass. The final patch still needs maintainer/security review because it relaxes an SSRF allow path.
