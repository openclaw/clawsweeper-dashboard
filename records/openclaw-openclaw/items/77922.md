## Summary

Keep open: current main and latest v2026.6.6 still gate Comfy private-network policy on `isPrivateOrLoopbackHost`, so DNS hostnames that resolve to private IPs remain source-reproducible. The closed unmerged related PR is useful context, but it is not a safe superseding fix.

## What This Changes

Allow the Comfy plugin's local/private-network mode to reach a configured DNS `baseUrl` that resolves to a private IP without tripping the SSRF guard.

## Best Possible Solution

Update the Comfy plugin to use the existing origin-scoped SSRF policy for the configured local/private `baseUrl`, with regression coverage for private-DNS success, different-port blocking, and metadata/link-local rebinding blocking.

## Reproduction Assessment

Yes from source inspection: a DNS hostname such as `comfy.local.example.com` leaves `resolveComfyNetworkPolicy` with no policy even when local mode has `allowPrivateNetwork` true. I did not run a live DNS/Comfy repro in this read-only cleanup pass.

## Solution Assessment

Yes: the best fix appears narrow, owned by the Comfy plugin, and should reuse the existing plugin SDK exact-origin SSRF policy instead of adding a new config option or broad bypass.
