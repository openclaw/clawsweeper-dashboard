## Summary

Keep open. Current main still rejects or ignores per-agent typing settings: the strict per-agent schema lacks the two keys, the per-agent resolver does not carry them, and reply plus heartbeat typing still read only defaults/session config.

## What This Changes

Request per-agent `typingMode` and `typingIntervalSeconds` overrides in `agents.list[]`, overriding `agents.defaults` for the selected agent.

## Best Possible Solution

Add the two fields to the core per-agent config contract, resolve them as `session` over selected agent over defaults, document the precedence, and cover reply plus heartbeat typing behavior with focused tests.

## Reproduction Assessment

Yes. A static reproduction is high confidence: parsing an `agents.list[]` entry with `typingMode` reaches strict `AgentEntrySchema`, which does not define that key, matching the reported unrecognized-key failure.

## Solution Assessment

Yes. This belongs in core config/runtime because typing indicators are already core agent/session behavior; extending the existing per-agent contract is narrower than a plugin workaround.
