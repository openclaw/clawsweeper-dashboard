## Summary

Keep open. Current main still rejects or ignores per-agent typing settings: the strict per-agent schema and resolver do not include `typingMode` or `typingIntervalSeconds`, reply typing reads only `agents.defaults` plus `session`, heartbeat typing does the same, and docs/tests only cover the defaults/session surface.

## What This Changes

Request to allow `agents.list[]` entries to set `typingMode` and `typingIntervalSeconds` as per-agent overrides of `agents.defaults`.

## Best Possible Solution

Add a narrow core config/runtime change: accept the two typing fields in `agents.list[]`, carry them through the per-agent resolver, apply them for the selected agent in reply and heartbeat typing resolution with documented precedence, regenerate schema/docs, and add focused config plus runtime tests.

## Reproduction Assessment

Yes. A static reproduction is high confidence: a config with `agents.list[0].typingMode` reaches strict `AgentEntrySchema`, which does not define that key, matching the reported unrecognized-key failure; runtime inspection also shows the value would not be applied even if parsing allowed it.

## Solution Assessment

Yes. The requested direction is the narrowest maintainable fix because typing indicators are already core agent/session behavior, and the missing piece is extending the existing per-agent config contract rather than moving this to a plugin or workaround.
