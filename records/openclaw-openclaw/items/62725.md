## Summary

Keep open. Current main still rejects the requested per-agent typing keys because the strict per-agent schema and public type omit them, and the reply plus heartbeat typing paths still resolve typing mode/cadence from session config or `agents.defaults` rather than the selected `agents.list[]` entry.

## What This Changes

The issue requests `typingMode` and `typingIntervalSeconds` overrides on `agents.list[]` entries, with per-agent values taking precedence over `agents.defaults`.

## Best Possible Solution

Add the fields to the per-agent schema, public types, resolved config, docs, and generated config help, then apply precedence as session over selected agent over defaults in both reply and heartbeat typing paths.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: strict `AgentEntrySchema` omits `typingMode` and `typingIntervalSeconds`, so those keys in `agents.list[]` are rejected before runtime.

## Solution Assessment

Yes. Extending the existing per-agent config/resolution path is the narrow maintainable solution because typing indicators already live in core session/default agent config rather than a channel-specific plugin surface.
