## Summary

Keep open. Current main still rejects the requested per-agent typing keys in the strict `agents.list[]` schema, and the reply plus heartbeat typing paths still read only session/default typing config; no closing PR or canonical duplicate for this specific per-agent mode/cadence request was found.

## What This Changes

Allow `typingMode` and `typingIntervalSeconds` to be configured on individual `agents.list[]` entries, overriding `agents.defaults` when set.

## Best Possible Solution

Add the fields to the per-agent schema/types/resolver, apply precedence as session over selected agent over defaults, and cover both reply and heartbeat typing paths with docs and focused tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `agents.list[]` entries are validated by strict `AgentEntrySchema`, which still omits `typingMode` and `typingIntervalSeconds`, so the reported unrecognized-key failure follows directly.

## Solution Assessment

Yes. Extending the existing core per-agent config and resolution path is the narrow maintainable solution because typing indicators already live in core session/default agent config rather than in a plugin-only surface.
