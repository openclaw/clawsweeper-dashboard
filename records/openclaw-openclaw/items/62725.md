## Summary

Keep open. Current main and the latest shipped release still limit `typingMode` and `typingIntervalSeconds` to `agents.defaults` or `session`, while the strict `agents.list[]` schema rejects the requested per-agent keys; the related TTL PR at https://github.com/openclaw/openclaw/pull/72009 does not supersede this per-agent mode/cadence request.

## What This Changes

Requests `typingMode` and `typingIntervalSeconds` overrides on `agents.list[]` entries, taking precedence over `agents.defaults` for the selected agent.

## Best Possible Solution

Add per-agent typing fields to the schema/types/resolved config, apply precedence as session over selected agent over defaults, and cover reply plus heartbeat typing paths with docs and focused tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `agents.list[]` entries are validated by strict `AgentEntrySchema`, which still omits `typingMode` and `typingIntervalSeconds`.

## Solution Assessment

Yes. Extending the existing core per-agent config and resolution path is the narrow maintainable solution because typing indicators already live in core session/default agent config rather than a plugin-only surface.
