## Summary

Keep open. Current main still exposes `typingMode` and `typingIntervalSeconds` only through session/default agent config; `agents.list[]` remains strict without those keys, and reply plus heartbeat typing paths do not resolve per-agent values.

## What This Changes

Allow `agents.list[]` entries to override `typingMode` and `typingIntervalSeconds` independently of global typing defaults.

## Best Possible Solution

Add the two fields to the core per-agent config contract, resolve typing precedence as session over selected agent over defaults, and update docs plus focused regression tests for reply and heartbeat typing.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: parsing an `agents.list[]` entry with `typingMode` reaches strict `AgentEntrySchema`, which lacks that key, matching the reported unrecognized-key failure.

## Solution Assessment

Yes. Extending the existing core per-agent config/resolution path is the narrow maintainable fix because typing indicators are already core session/default agent behavior, not a plugin-only concern.
