## Summary

Keep open. Current main still rejects the requested per-agent keys and only resolves typing mode/cadence from `session` or `agents.defaults`; the adjacent TTL PR is related typing-config work but does not cover per-agent mode/cadence.

## What This Changes

The issue requests allowing `typingMode` and `typingIntervalSeconds` on individual `agents.list[]` entries with per-agent precedence over `agents.defaults`.

## Best Possible Solution

Add the two fields to the per-agent config contract, resolve precedence as session over selected agent over defaults, and document/test both reply and heartbeat typing paths.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: parsing an `agents.list[]` entry with `typingMode` still reaches strict `AgentEntrySchema`, which lacks that key and should reject it as unrecognized.

## Solution Assessment

Yes. Extending the existing core per-agent config and resolved-agent path is the narrow maintainable solution because typing indicators are already core session/default agent behavior, not plugin-only behavior.
