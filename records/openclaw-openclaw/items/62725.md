## Summary

Keep open. Current main still accepts typing mode and cadence only at `session` and `agents.defaults`; `agents.list[]` remains strict without these fields, and reply plus heartbeat typing still read defaults/session rather than selected-agent config.

## What This Changes

Request adding per-agent `typingMode` and `typingIntervalSeconds` overrides under `agents.list[]` for selected agents.

## Best Possible Solution

Add the fields to the per-agent config contract, resolve precedence as session over selected agent over defaults, and cover schema, docs, reply typing, and heartbeat typing with focused tests.

Yes. Source inspection gives a high-confidence path: `AgentEntrySchema` is strict and lacks `typingMode`, so an `agents.list[]` entry with that key still matches the reported unrecognized-key failure.

## Solution Assessment

Yes. Extending the existing per-agent config/resolution path is the narrow maintainable solution because typing mode and cadence are already core session/default behavior, not a plugin-only concern.
