## Summary

Keep open. Current main still only accepts typing indicator mode/cadence at `session` and `agents.defaults`; strict `agents.list[]` schema, generated types, agent resolution, reply typing, and heartbeat typing still lack per-agent `typingMode` and `typingIntervalSeconds` support.

## What This Changes

Allow `typingMode` and `typingIntervalSeconds` on individual `agents.list[]` entries as overrides over `agents.defaults`.

## Best Possible Solution

Add the two fields to the per-agent config contract, resolve precedence as session over selected agent over defaults, and document/test both reply and heartbeat typing paths.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: parsing an `agents.list[]` entry with `typingMode` still reaches strict `AgentEntrySchema`, which lacks that key and therefore matches the reported unrecognized-key failure.

## Solution Assessment

Yes. Extending the existing core per-agent config/resolution path is the narrow maintainable fix because typing indicators are already session/default agent behavior in core, not a plugin-only concern.
