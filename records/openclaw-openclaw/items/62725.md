## Summary

Keep open. Current main still accepts typing indicator mode and cadence at `session` and `agents.defaults`, but `agents.list[]` remains strict and does not expose or resolve per-agent `typingMode` or `typingIntervalSeconds`.

## What This Changes

Request per-agent `typingMode` and `typingIntervalSeconds` overrides on `agents.list[]`, with per-agent values overriding `agents.defaults`.

## Best Possible Solution

Add the two fields to the per-agent config contract, resolve precedence as `session` over selected agent over `agents.defaults`, and document/test both reply and heartbeat typing behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `agents.list[]` entries still parse through strict `AgentEntrySchema`, which lacks `typingMode` and `typingIntervalSeconds`.

## Solution Assessment

Yes, if maintainers approve the expanded config surface. Extending the existing per-agent schema/resolver and using it in reply plus heartbeat typing paths is the narrow maintainable direction.
