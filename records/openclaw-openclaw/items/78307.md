## Summary

Keep open: current main has the coarse run data, but the Dashboard Sessions table still does not surface a visible live/status indicator and the visible `Active` filter label still reads like liveness rather than a lookback window.

## What This Changes

Request to distinguish live/running sessions from recently updated sessions in the Dashboard Sessions view and clarify the `Active N` lookback filter.

## Best Possible Solution

Render a compact status/live badge from existing `hasActiveRun`/`status` row fields in the Sessions table and rename the lookback control to updated-within wording, leaving richer heartbeat semantics to #57933/#39127.

## Reproduction Assessment

Yes. Source inspection on current main shows `sessions.list` rows expose `status`/`hasActiveRun`, while the Sessions table omits a visible status/live column and the visible filter label remains `Active`.

## Solution Assessment

Yes for a narrow first pass. The maintainable path is to project existing row fields into the Sessions UI and clarify the filter copy, without adding a new Gateway activity protocol in this issue.
