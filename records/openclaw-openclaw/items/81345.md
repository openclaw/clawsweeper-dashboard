## Summary

Keep open. Current main still has a source-visible path where subagent completion can be recorded as ok and announced as completed while the announced result is taken from raw tool output rather than a final assistant/yield response.

## What This Changes

The issue asks OpenClaw to flag subagent runs with no final assistant/yield text as incomplete or failed and to stop using trailing tool output as the completion result.

## Best Possible Solution

Propagate the existing terminal liveness/no-output signal into subagent run outcomes and announce formatting, and emit an explicit no-result/incomplete result when no final assistant or sessions_yield output exists.

## Reproduction Assessment

Yes, at source level: an empty assistant result with trailing tool output currently reaches the announce fallback tests, and subagent lifecycle completion still flattens normal end events to ok. I did not run a live gpt-5.5-pro reproduction.

## Solution Assessment

Yes, the maintainable fix is to use existing liveness/replay-invalid/final-text signals in the subagent registry and announce formatter; the optional new max-tool-iteration config should not be required for the first fix.
