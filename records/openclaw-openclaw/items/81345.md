## Summary

Keep open: current main still has a source-reproducible path where an `ok` subagent completion with no assistant text can announce `completed successfully` and use latest tool output as the result.

## What This Changes

The issue asks OpenClaw to fail or mark incomplete subagent runs that produce no synthesized text/yield output and to stop using trailing tool output as the announce result.

## Best Possible Solution

Add a focused subagent finalization/output guard that treats zero-text, no-yield, zero-output-token runs as failed or incomplete, avoids raw tool-output results, and updates the affected tests/docs.

## Reproduction Assessment

Yes, at source level: mock `agent.wait` as `ok` and `chat.history` as empty assistant text plus a toolResult, and current main will announce success with tool output. I did not run tests because this review is read-only.

## Solution Assessment

Yes, a narrow fix in subagent completion classification and announce output selection is the maintainable path; the optional max-tool-iteration guard can be a follow-up if needed.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/81345 on current main. In the subagent runtime/announce path, do not treat a terminal subagent run that produced no assistant text/yield output and zero output tokens as a clean `completed successfully` result, and do not use trailing tool/toolResult output as the child `Result` for that case. Keep the change narrowly scoped to subagent completion classification/output selection and the OpenAI Responses stop/usage edge if needed; do not add a new config option or broad max-iteration feature unless the existing code already has the seam. Update current tests that expect tool-output fallback, add a regression case for `agent.wait` ok + assistant-empty/toolResult-only history, and update subagent/task docs if the result contract changes. Preserve normal assistant-text completion, explicit `sessions_yield`, timeout partial-progress, and terminal-error no-stale-output behavior.
