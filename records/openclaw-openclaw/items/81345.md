## Summary

Keep open. Current main still has source-level paths where a subagent completion with no synthesized assistant result can be marked successful, and result capture can fall back to generic tool-result text rather than an explicit child result.

## What This Changes

The issue reports a subagent run with no text/yield output and zero usage being announced as successful, with the result populated from the last exec output.

## Best Possible Solution

Add a guarded subagent completion path that treats no synthesized result plus zero usage as failed or unknown, and never uses generic tool output as the child result.

## Reproduction Assessment

Yes, at source level. Current main can select tool/toolResult text as subagent output and can render a non-error/non-timeout wait as completed successfully without checking for a real final assistant result; the exact gpt-5.5-pro live loop was not rerun.

## Solution Assessment

Yes, if kept narrow. The maintainable fix is in subagent completion capture/finalization and regression tests, not a new configurable max-tool-iterations feature in this pass.
