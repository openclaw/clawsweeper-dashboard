## Solution Assessment

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Register auto-recall unconditionally:** `extensions/memory-core/src/auto-recall.ts:79-81`
  - body: The registrar returns before registering `before_prompt_build` when startup config has no autoRecall enabled. Since the handler reads live config per turn, enabling `agents.defaults.memorySearch.autoRecall` later has no effect until restart.
  - confidence: 0.88
- **[P2] Use runtime.config.current for live config:** `extensions/memory-core/src/auto-recall.ts:89`
  - body: Bundled plugin production code calls the deprecated `api.runtime.config.loadConfig()` API. The plugin runtime docs and config boundary guard require `api.runtime.config.current()` or passed config, so this will fail the deprecated-config guard.
  - confidence: 0.86
- **[P2] Warn from effective recall availability:** `extensions/memory-core/src/auto-recall.ts:59-61`
  - body: This check suppresses the recall-only warning from raw `autoRecall.enabled`, but `resolveMemorySearchConfig` can still return null when memory search is disabled or an agent override disables recall. Those agents lose bootstrap memory without the warning this guard is meant to provide.
  - confidence: 0.82
- **[P2] Let channel plugins normalize thread delivery:** `src/utils/delivery-context.ts:116-126`
  - body: This fallback now runs before `messaging.resolveDeliveryTarget` for Telegram, Slack, and Mattermost thread children. Telegram topic delivery depends on the plugin resolver to normalize the chat and topic ids, so the early return can route with raw conversation ids.
  - confidence: 0.87
