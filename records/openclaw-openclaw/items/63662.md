## Summary

Keep this PR open: the feature is still useful and current main does not implement the central memoryInjection/autoRecall behavior, but the branch still has concrete P2 regressions, is merge-conflicting, mixes unrelated churn, and lacks external after-fix real behavior proof.

## What This Changes

The branch adds `agents.defaults.memoryInjection`, `agents.defaults.memorySearch.autoRecall`, a memory-core `before_prompt_build` recall hook, docs/schema/tests, and unrelated delivery-context, QA, and script-guard changes.

## Best Possible Solution

Land a narrow memory-core/config/docs implementation that preserves plugin-owned delivery normalization, uses live runtime config snapshots safely, warns from resolved recall availability, and is backed by real behavior proof.

## Reproduction Assessment

Yes for the PR defects: source inspection shows startup-only hook registration, deprecated config API use, warning checks based on raw config rather than resolved recall availability, and delivery resolver bypass. I did not run runtime tests under the read-only review constraint.

## Solution Assessment

No. The feature direction is useful, but this patch should be narrowed to the memory feature, repaired for live config and warning semantics, and stripped of unrelated delivery/script/QA churn before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Register auto-recall before checking startup config:** `extensions/memory-core/src/auto-recall.ts:80-82`
  - body: The registrar returns before installing the `before_prompt_build` hook when startup config has autoRecall disabled. Since the handler is intended to read current config each turn, enabling `agents.defaults.memorySearch.autoRecall` later will have no effect until restart.
  - confidence: 0.88
- **[P2] Use the supported runtime config snapshot:** `extensions/memory-core/src/auto-recall.ts:90`
  - body: Bundled plugin code calls the deprecated `api.runtime.config.loadConfig()` compatibility API. The plugin runtime contract and config boundary guard require `api.runtime.config.current()` or passed config, so this will fail the bundled-plugin config guard.
  - confidence: 0.87
- **[P2] Let channel plugins normalize thread delivery:** `src/utils/delivery-context.ts:116-126`
  - body: This early fallback runs before `messaging.resolveDeliveryTarget` for Slack, Mattermost, and Telegram thread children. Telegram topic delivery depends on the plugin resolver to normalize targets and thread IDs, so the fallback can route with raw conversation IDs.
  - confidence: 0.87
- **[P2] Warn from resolved auto-recall availability:** `extensions/memory-core/src/auto-recall.ts:63-69`
  - body: The warning check treats raw `autoRecall.enabled` as sufficient, but `resolveMemorySearchConfig` can still return null when memory search is disabled or an agent override disables recall. In `recall-only` mode, those agents lose automatic memory context without the warning this guard is meant to provide.
  - confidence: 0.8
