## What This Changes

The PR adds `agents.defaults.memoryInjection`, `agents.defaults.memorySearch.autoRecall`, a memory-core prompt hook, docs/schema/tests, and some unrelated delivery, QA, and script-guard changes.

## Best Possible Solution

Land a narrower memory-only implementation after settling the config contract, fixing live-config/effective-warning behavior, preserving channel plugin delivery normalization, and adding redacted real OpenClaw proof.

## Reproduction Assessment

Yes for the PR defects: source inspection shows the startup-only hook registration, deprecated config call, ineffective warning cases, and delivery resolver bypass. The requested memory mode itself is a feature, not a bug reproduction.

## Solution Assessment

No. The feature direction is useful, but this patch should be narrowed and repaired before merge so memory-core follows live config contracts and channel delivery remains plugin-normalized.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Register auto-recall unconditionally:** `extensions/memory-core/src/auto-recall.ts:80-81`
  - body: The registrar returns before installing `before_prompt_build` when startup config has no autoRecall enabled. Since the handler reads live config per turn, enabling autoRecall later has no effect until restart.
  - confidence: 0.88
- **[P2] Use runtime.config.current for live config:** `extensions/memory-core/src/auto-recall.ts:90`
  - body: Bundled plugin production code calls deprecated `api.runtime.config.loadConfig()`. The plugin runtime docs and config boundary guard require passed config or `api.runtime.config.current()`, so this will fail the bundled-plugin config contract.
  - confidence: 0.86
- **[P2] Warn from effective recall availability:** `extensions/memory-core/src/auto-recall.ts:63-69`
  - body: This check suppresses the recall-only warning from raw `autoRecall.enabled`, but `resolveMemorySearchConfig` can still return null when memory search is disabled or an agent override disables recall. Those agents lose bootstrap memory without the warning this guard is meant to provide.
  - confidence: 0.84
- **[P2] Let channel plugins normalize thread delivery:** `src/utils/delivery-context.ts:118-126`
  - body: This fallback now runs before `messaging.resolveDeliveryTarget` for Telegram, Slack, and Mattermost thread children. Telegram topic delivery depends on the plugin resolver to normalize chat and topic ids, so the early return can route with raw conversation ids.
  - confidence: 0.87
