## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Register auto-recall before startup checks:** `extensions/memory-core/src/auto-recall.ts:80-82`
  - body: The registrar returns before `api.on("before_prompt_build", ...)` when startup config has no autoRecall enabled. Since the hook reads live config per turn, enabling autoRecall later has no effect until restart.
  - confidence: 0.88
- **[P2] Use runtime config current for auto-recall:** `extensions/memory-core/src/auto-recall.ts:90`
  - body: Bundled plugin production code calls deprecated `api.runtime.config.loadConfig()`. The plugin runtime docs and config boundary guard require `api.runtime.config.current()` or passed config, so this will fail the deprecated-config guard.
  - confidence: 0.86
- **[P2] Let channel plugins resolve thread delivery first:** `src/utils/delivery-context.ts:116-126`
  - body: This fallback now runs before `messaging.resolveDeliveryTarget` for Telegram, Slack, and Mattermost thread children. Telegram topic delivery depends on the plugin resolver to normalize the target and thread id, so the early return can route with raw conversation ids.
  - confidence: 0.87
- **[P2] Warn from resolved auto-recall availability:** `extensions/memory-core/src/auto-recall.ts:59-61`
  - body: The warning check treats raw `autoRecall.enabled` as enough, but `resolveMemorySearchConfig` can still return `null` when memory search is disabled or an agent override disables autoRecall. In `recall-only` mode those agents lose automatic memory context without the intended warning.
  - confidence: 0.8
- **[P2] Add the required changelog entry:** `src/config/types.agent-defaults.ts:172-173`
  - body: This is a user-facing feature config/docs change, but the PR diff does not include `CHANGELOG.md`. Repo policy requires a single-line changelog entry for user-facing feature changes, with contributor credit where appropriate.
  - confidence: 0.82
- **[P3] Drop unrelated guard allowlist changes:** `scripts/check-channel-agnostic-boundaries.mjs:62-65`
  - body: The memory feature branch adds allowlist entries to architecture and random-tmp guards for unrelated files. That can mask future guard failures and should be removed or split into a separately justified change.
  - confidence: 0.76
