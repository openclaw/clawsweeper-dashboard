## Summary

Keep open: the feature is not implemented on current main and there is real operator demand, but this external PR lacks after-fix real behavior proof and introduces source-reproducible P2 defects in auto-recall registration, live config usage, recall-only warnings, and Telegram delivery normalization.

## What This Changes

Adds `agents.defaults.memoryInjection`, `agents.defaults.memorySearch.autoRecall`, a memory-core `before_prompt_build` auto-recall hook, docs/schema/tests, and unrelated delivery, QA, and script-guard updates.

## Best Possible Solution

Land a narrower memory-only implementation with schema/docs/tests, an unconditional live-config-safe auto-recall hook, effective per-agent warnings, and preserved channel plugin delivery normalization.

## Reproduction Assessment

Yes for the PR defects: source inspection shows the startup-only hook registration, deprecated config call, ineffective warning cases, and delivery resolver bypass. The requested feature itself is not a bug reproduction; current main lacks the new config surface.

## Solution Assessment

No. The feature direction is useful, but this patch should be narrowed and repaired before merge so memory-core follows live config contracts and channel delivery remains plugin-normalized.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Register auto-recall unconditionally:** `extensions/memory-core/src/auto-recall.ts:80-81`
  - body: The registrar returns before registering `before_prompt_build` when startup config has no autoRecall enabled. Since the handler is supposed to read live config per turn, enabling `agents.defaults.memorySearch.autoRecall` later has no effect until restart.
  - confidence: 0.88
- **[P2] Use runtime.config.current for live config:** `extensions/memory-core/src/auto-recall.ts:90`
  - body: Bundled plugin production code calls the deprecated `api.runtime.config.loadConfig()` API. The plugin runtime docs and config boundary guard require `api.runtime.config.current()` or passed config, so this will fail the deprecated-config guard.
  - confidence: 0.86
- **[P2] Warn from effective recall availability:** `extensions/memory-core/src/auto-recall.ts:63-69`
  - body: This check suppresses the recall-only warning from raw `autoRecall.enabled`, but `resolveMemorySearchConfig` can still return null when memory search is disabled or an agent override disables recall. Those agents lose bootstrap memory without the warning this guard is meant to provide.
  - confidence: 0.83
- **[P2] Let channel plugins normalize thread delivery:** `src/utils/delivery-context.ts:118-126`
  - body: This fallback now runs before `messaging.resolveDeliveryTarget` for Telegram, Slack, and Mattermost thread children. Telegram topic delivery depends on the plugin resolver to normalize the chat and topic ids, so the early return can route with raw conversation ids.
  - confidence: 0.87
