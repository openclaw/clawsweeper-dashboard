## Summary

Keep open: current main still lacks the memoryInjection and memory-core autoRecall config surface, and the PR addresses real user demand, but the patch is not merge-ready because it has source-proven P2 regressions, no real behavior proof, and a broad dirty branch.

## What This Changes

The branch adds agents.defaults.memoryInjection, agents.defaults.memorySearch.autoRecall, a memory-core before_prompt_build auto-recall hook, docs/schema/tests, and unrelated delivery, QA, and script-guard changes.

## Best Possible Solution

Land a narrower memory-focused implementation with schema/docs/tests, live-config-safe hook registration, effective-config warnings, preserved channel delivery normalization, and maintainer-owned changelog handling.

## Reproduction Assessment

Yes for the PR defects: source inspection shows the startup-only hook registration, deprecated config API call, raw-flag warning bypass, and delivery resolver bypass. The requested memoryInjection/autoRecall capability itself is a feature request, not a current-main bug reproduction.

## Solution Assessment

No. The feature direction is useful, but this patch should be narrowed and repaired before merge so memory-core follows live config contracts and channel delivery remains plugin-normalized.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Register auto-recall hook unconditionally:** `extensions/memory-core/src/auto-recall.ts:80-81`
  - body: The registrar returns before registering `before_prompt_build` when startup config has no autoRecall enabled. Since the handler is meant to read live config per turn, enabling `agents.defaults.memorySearch.autoRecall` later has no effect until restart.
  - confidence: 0.88
- **[P2] Use runtime.config.current for live config:** `extensions/memory-core/src/auto-recall.ts:90`
  - body: Bundled plugin production code calls the deprecated `api.runtime.config.loadConfig()` API. The plugin runtime docs and config boundary guard require `api.runtime.config.current()` or passed config, so this will fail the deprecated-config guard.
  - confidence: 0.86
- **[P2] Warn from effective recall availability:** `extensions/memory-core/src/auto-recall.ts:63-69`
  - body: This suppresses the recall-only warning from raw `autoRecall.enabled`, but `resolveMemorySearchConfig` can still return null when memory search is disabled or an agent override disables recall. Those agents lose bootstrap memory without the warning this guard is meant to provide.
  - confidence: 0.83
- **[P2] Let channel plugins normalize thread delivery:** `src/utils/delivery-context.ts:119-126`
  - body: This fallback now runs before `messaging.resolveDeliveryTarget` for Telegram, Slack, and Mattermost thread children. Telegram topic delivery depends on the plugin resolver to normalize chat and topic ids, so the early return can route with raw conversation ids.
  - confidence: 0.87
