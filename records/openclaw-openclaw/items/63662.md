## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Register auto-recall even when initially disabled:** `extensions/memory-core/src/auto-recall.ts:80-81`
  - body: `registerMemoryAutoRecall` exits without registering `before_prompt_build` when startup config has autoRecall off. Since the handler reads runtime config each turn, enabling autoRecall later cannot take effect until restart; the sibling memory-lancedb hook stays registered but inert for this reason.
  - confidence: 0.91
- **[P2] Warn from effective recall availability:** `extensions/memory-core/src/auto-recall.ts:63-64`
  - body: This raw defaults check suppresses the recall-only warning whenever default autoRecall is true, even if `memorySearch.enabled` is false or an agent override disables autoRecall. In those cases `resolveMemorySearchConfig` returns null/disabled, so recall-only removes `MEMORY.md` and the agent gets no automatic memory context without a warning.
  - confidence: 0.9
- **[P2] Let channel plugins resolve thread delivery:** `src/utils/delivery-context.ts:118-126`
  - body: This new early return runs before the channel plugin `resolveDeliveryTarget` hook. For Telegram topics, the plugin parses the canonical topic conversation and strips it to `{to: chatId, threadId: topicId}`; returning the raw `conversationId` here can target the wrong thread shape and bypass Telegram-owned normalization.
  - confidence: 0.88
- **[P2] Remove the channel-boundary allowlist drive-by:** `scripts/check-channel-agnostic-boundaries.mjs:64-67`
  - body: Adding hard-coded exceptions for `src/agents/acp-spawn.ts` is unrelated to configurable memory recall and makes the channel-agnostic boundary guard stop reporting those violations. This should be fixed or split with owner review rather than hidden in this feature PR.
  - confidence: 0.84
- **[P2] Do not broaden the tmpdir guard allowlist here:** `scripts/check-no-random-messaging-tmp.mjs:18-31`
  - body: This expands the messaging tmpdir guard allowlist by a dozen unrelated paths, so future green CI would no longer prove the same random tmpdir boundary. The memory feature should not land with unrelated guard bypasses.
  - confidence: 0.82
