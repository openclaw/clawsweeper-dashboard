## Summary

Keep open. Current main still lacks the proposed `memoryInjection` and `memorySearch.autoRecall` contract, but this PR should not merge as-is because the auto-recall hook, live config API use, delivery normalization change, warning logic, and changelog coverage need repair.

## What This Changes

The branch adds `agents.defaults.memoryInjection`, `agents.defaults.memorySearch.autoRecall`, a memory-core `before_prompt_build` recall hook, schema/docs/tests, and also touches delivery-context, QA, and script guard files.

## Best Possible Solution

Land a narrow memory-only implementation with schema/docs/tests/changelog, live-config-safe auto-recall registration, resolved warning checks, and no unrelated delivery or guard allowlist changes.

## Reproduction Assessment

Yes for the PR defects. Source review shows the startup-only hook registration, deprecated config call, resolver bypass, and missing changelog; no runtime test was executed under the read-only constraint.

## Solution Assessment

No. The feature direction is reasonable, but this patch needs a narrower implementation that preserves channel plugin normalization, uses supported runtime config snapshots, warns from resolved recall availability, and includes the required changelog entry.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Register auto-recall before startup checks:** `extensions/memory-core/src/auto-recall.ts:80-82`
  - body: The registrar returns before `api.on("before_prompt_build", ...)` when startup config has no autoRecall enabled. Since the handler is supposed to read live config each turn, enabling `agents.defaults.memorySearch.autoRecall` later has no effect until restart.
  - confidence: 0.88
- **[P2] Use runtime.config.current for live config:** `extensions/memory-core/src/auto-recall.ts:90`
  - body: Bundled plugin production code calls the deprecated `api.runtime.config.loadConfig()` API. The plugin runtime docs and config boundary guard require `api.runtime.config.current()` or passed config, so this will fail the deprecated-config guard.
  - confidence: 0.86
- **[P2] Let channel plugins normalize thread delivery:** `src/utils/delivery-context.ts:116-126`
  - body: This fallback now runs before `messaging.resolveDeliveryTarget` for Telegram, Slack, and Mattermost thread children. Telegram topic delivery depends on the plugin resolver to normalize the target and thread id, so the early return can route with raw conversation ids.
  - confidence: 0.87
- **[P2] Warn from resolved recall availability:** `extensions/memory-core/src/auto-recall.ts:59-61`
  - body: The warning check treats any raw `autoRecall.enabled` as enough, but `resolveMemorySearchConfig` can still return `null` when memory search is disabled or when an agent override disables autoRecall. In `recall-only` mode those agents lose automatic memory context without the warning this guard is meant to provide.
  - confidence: 0.8
- **[P2] Add the required changelog entry:** `src/config/types.agent-defaults.ts:172-173`
  - body: This is a user-facing `feat` config/docs change, but the PR file list does not include `CHANGELOG.md`. Repo policy requires a single-line changelog entry for user-facing feature changes, with contributor credit where appropriate.
  - confidence: 0.82
