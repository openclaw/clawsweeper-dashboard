## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Register auto-recall unconditionally:** `extensions/memory-core/src/auto-recall.ts:80-81`
  - body: The registrar returns before installing `before_prompt_build` when startup config has no autoRecall enabled. Because the handler is supposed to use live config, enabling `autoRecall` later has no effect until restart.
  - confidence: 0.89
- **[P2] Use the live runtime config snapshot:** `extensions/memory-core/src/auto-recall.ts:90`
  - body: The bundled plugin hook calls `api.runtime.config.loadConfig()` on each turn. Current plugin runtime docs mark that compatibility helper deprecated for bundled plugins, so the hook should use the current runtime snapshot path.
  - confidence: 0.86
- **[P2] Warn from effective recall availability:** `extensions/memory-core/src/auto-recall.ts:63-69`
  - body: The recall-only warning checks raw `autoRecall.enabled` flags, but effective recall can still be disabled by `memorySearch.enabled: false` or agent overrides. Those agents can lose bootstrap memory without the warning this guard is meant to provide.
  - confidence: 0.84
- **[P2] Let channel plugins normalize thread delivery:** `src/utils/delivery-context.ts:118-126`
  - body: The new early return for Slack, Mattermost, and Telegram thread children runs before `messaging.resolveDeliveryTarget`. Telegram topic delivery depends on plugin normalization, so this can route with raw conversation ids instead of the normalized target.
  - confidence: 0.88
