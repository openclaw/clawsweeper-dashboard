## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Register auto-recall unconditionally:** `extensions/memory-core/src/auto-recall.ts:80-81`
  - body: The registrar returns before installing the before_prompt_build hook when startup config has no autoRecall enabled. Because the handler reads live config per turn, enabling autoRecall later cannot work until restart.
  - confidence: 0.89
- **[P2] Use the current runtime config snapshot:** `extensions/memory-core/src/auto-recall.ts:90`
  - body: This bundled plugin hook calls the deprecated runtime.config.loadConfig helper. Current plugin runtime docs require bundled long-lived handlers to use passed config or runtime.config.current, and boundary guards are intended to catch this usage.
  - confidence: 0.86
- **[P2] Warn from effective recall availability:** `extensions/memory-core/src/auto-recall.ts:63-69`
  - body: The recall-only warning checks raw autoRecall flags, but effective recall can still be disabled by memorySearch.enabled: false or agent overrides. Those agents can lose bootstrap memory without the warning this guard is meant to provide.
  - confidence: 0.84
- **[P2] Let channel plugins normalize thread delivery:** `src/utils/delivery-context.ts:118-126`
  - body: The new early return for Telegram thread children runs before messaging.resolveDeliveryTarget. Telegram topic delivery depends on plugin normalization, so this can route with raw conversation ids instead of the normalized target.
  - confidence: 0.88
