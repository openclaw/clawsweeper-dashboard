## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Register auto-recall unconditionally:** `extensions/memory-core/src/auto-recall.ts:80-81`
  - body: The registrar returns before installing `before_prompt_build` when startup config has no autoRecall enabled. Since the handler is meant to honor live config, toggling autoRecall later cannot work until restart.
  - confidence: 0.89
- **[P2] Use the live runtime config snapshot:** `extensions/memory-core/src/auto-recall.ts:90`
  - body: Bundled plugin production code calls deprecated `api.runtime.config.loadConfig()`. The plugin runtime docs and config boundary guard require passed config or `api.runtime.config.current()`, so this will fail the bundled-plugin config contract.
  - confidence: 0.87
- **[P2] Warn from effective recall availability:** `extensions/memory-core/src/auto-recall.ts:63-69`
  - body: This check suppresses warnings from raw `autoRecall` flags, but `resolveMemorySearchConfig` can still return null when memory search is disabled or an agent override disables recall. In `recall-only` mode those agents lose bootstrap memory with no warning.
  - confidence: 0.84
- **[P2] Let channel plugins normalize thread delivery:** `src/utils/delivery-context.ts:118-126`
  - body: The new early return runs before `messaging.resolveDeliveryTarget` for Telegram, Slack, and Mattermost thread children. Telegram topic delivery depends on plugin normalization, so this can route with raw conversation ids.
  - confidence: 0.88
