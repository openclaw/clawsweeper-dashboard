## What This Changes

The PR changes session write-lock stale recovery to use fs-safe's `remove-if-unchanged` stale cleanup callback and adds a regression test for changed lock files during recovery.

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

Yes. Moving deletion to fs-safe's observed-snapshot `remove-if-unchanged` contract is narrower than adding another session-wrapper shim and keeps stale-owner policy in the existing session lock code.
