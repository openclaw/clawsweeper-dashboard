## Summary

Keep this PR open: the Beeper-native Matrix streaming feature is not on current main, but the branch is still not merge-ready because it lacks real Beeper proof and introduces source-visible authorization and quiet-mode regressions.

## What This Changes

Adds Beeper-native Matrix streaming events, Beeper approval reaction keys, Matrix SDK to-device plumbing, tests, docs, and a changelog entry.

## Best Possible Solution

Keep Beeper-native streaming in the Matrix plugin only with room-authorized subscribers, preserved quiet-mode semantics, focused regression tests, and real Beeper/Beeper Desktop proof.

## Reproduction Assessment

Yes for the review findings from source inspection: the subscribe path registers event.sender without room authorization, and quiet mode routes through a controller that sends a normal placeholder message. I did not run a live Beeper client because this read-only review found no real runtime proof in the PR.

## Solution Assessment

No. Native Beeper streaming may be useful, but this patch is not the maintainable solution until subscriber authorization, quiet-mode behavior, and live Beeper proof are handled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Authenticate Beeper stream subscribers before sending deltas:** `extensions/matrix/src/matrix/beeper-stream.ts:249-258`
  - body: `handleToDeviceSubscribe` accepts any matching to-device subscribe event and registers `event.sender`/`device_id` as a subscriber, but later deltas can include reasoning, command output, patches, and approval state. A to-device event is account-scoped, so verify the sender is joined or otherwise authorized for `roomId` before adding it.
  - confidence: 0.9
- **[P2] Preserve quiet mode for the initial Beeper stream event:** `extensions/matrix/src/matrix/beeper-stream.ts:187-193`
  - body: Beeper native streaming is enabled for every non-off Matrix streaming mode, including `streaming: "quiet"`, but the initial stream placeholder is sent as a normal text message without the quiet draft path's notice/no-mention options. That can notify recipients before the finalized reply, regressing the documented quiet behavior.
  - confidence: 0.86
