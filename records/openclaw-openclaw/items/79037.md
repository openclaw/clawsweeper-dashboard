## Summary

Keep open. This focused Matrix feature is not ready to merge because it lacks real behavior proof and the diff introduces a to-device subscription authorization gap plus a quiet-streaming regression.

## What This Changes

The PR adds a Beeper-specific native stream controller to the Matrix plugin, wires it into Matrix reply dispatch, accepts Beeper approval reaction keys, and updates Matrix docs/changelog/tests.

## Best Possible Solution

Keep this in the Matrix plugin only if maintainers want Beeper-specific behavior, with room-authorized subscribers, preserved Matrix streaming mode semantics, focused tests, and real Beeper/Beeper Desktop proof before merge.

## Reproduction Assessment

Yes for the review findings from source inspection: the subscribe path registers event.sender without a room authorization check, and quiet mode routes into a controller that sends a normal placeholder message. I did not run a live Beeper client because this was a read-only review and the PR lacks real behavior proof.

## Solution Assessment

No. Native Beeper streaming may be a useful direction, but the current implementation needs subscriber authorization, quiet-mode preservation, and live Beeper proof before it is the maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Authenticate Beeper stream subscribers before sending deltas:** `extensions/matrix/src/matrix/beeper-stream.ts:249-258`
  - body: `handleToDeviceSubscribe` accepts any matching to-device subscribe event and then `publishPartToSubscribers` sends thinking, command output, patches, and approval state to `event.sender`/`device_id`. To-device events are account-scoped, not proof that the sender can read the room, so this should verify the subscriber is joined/authorized for `roomId` before adding it.
  - confidence: 0.9
- **[P2] Preserve quiet mode for the initial Beeper stream event:** `extensions/matrix/src/matrix/beeper-stream.ts:187-193`
  - body: The handler enables `beeperStream` for every non-off Beeper streaming mode, including `streaming: "quiet"`, but this send creates a normal Matrix text placeholder without the notice/no-mention options used by quiet draft previews. That can notify Beeper/Matrix recipients before the finalized reply, regressing the documented quiet mode behavior.
  - confidence: 0.86
