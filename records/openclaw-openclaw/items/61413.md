Add an explicit transparent-proxy-compatible trusted path that preserves hostname/private-IP policy checks before dispatch, documents the security contract, and includes guarded cause-chain diagnostics plus real proxy proof.

## Solution Assessment

No. The proposed default-fetch fallback is broader than the current guard contract; a safer solution would keep policy validation, make transparent-proxy behavior explicit, and guard cause serialization.
