## Summary

Keep open for maintainer review: current main still has the session-delivery blind-replay gap, and this PR is a focused fix, but merging it intentionally changes ambiguous crash-window recovery from replay to fail-safe refusal.

## What This Changes

The PR wires session-delivery recovery to the existing shared SQLite `recovery_state` column, adds restart-continuation delivery hooks, and adds crash/retry regression coverage.

## Best Possible Solution

Land this bounded recovery-state fix if maintainers accept fail-safe refusal of ambiguous restarted turns, and track adapter-side reconciliation as a separate multi-adapter follow-up.

## Reproduction Assessment

Yes, from source inspection: enqueue an `agentTurn`, let `deliver` succeed without completing `ackSessionDelivery`, then run recovery again and current main re-delivers the same pending entry. I did not run tests in this read-only review.

## Solution Assessment

Yes, this appears to be the best bounded fix short of a larger adapter reconciliation API. It reuses the existing SQLite recovery column and mirrors the outbound no-blind-replay contract while preserving pre-send and busy-deferred retries.
