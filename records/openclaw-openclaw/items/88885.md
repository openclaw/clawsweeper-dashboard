## Summary

Keep open: this PR is a focused fix candidate with sufficient real behavior proof, but it marks `send_attempt_started` before `dispatchAssembledChannelTurn` finishes its pre-dispatch recording phase, so a pre-dispatch failure can move a turn to `failed/` even though no reply could have been sent.

## What This Changes

This PR adds session-delivery recovery hooks and storage helpers that persist a `send_attempt_started` marker, refuse recovered marked entries, clear busy-deferred markers, and add regression coverage.

## Best Possible Solution

Use the existing SQLite `recovery_state` column, but place the marker at the actual dispatch-start boundary after successful inbound recording and before non-idempotent work, then merge only with explicit acceptance of fail-safe replay refusal.

## Reproduction Assessment

Yes. Source inspection and the PR's supplied proof give a high-confidence path: enqueue an `agentTurn`, let delivery succeed without completing ack, then run recovery again and observe replay on current main.

## Solution Assessment

No, not yet. The recovery-state approach is the right bounded solution, but the marker must move after pre-dispatch recording succeeds so pre-dispatch failures remain retryable.
