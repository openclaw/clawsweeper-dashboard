## Summary

Keep open for maintainer review. Current main still replays an unacked session-delivery entry with no durable send marker, while this PR wires the existing SQLite recovery_state path at the agent-turn boundary and adds regressions that address the earlier marker-clearing concern; I found no blocking code defect, but merge still needs maintainer acceptance of the intentional fail-safe crash-window tradeoff.

## What This Changes

The PR adds session-delivery recovery hooks/state handling that marks agent-turn delivery start, refuses recovered marked entries, clears busy-deferred markers, and adds focused recovery tests.

## Best Possible Solution

Land this bounded recovery-state fix if maintainers accept fail-safe refusal of ambiguous restarted turns, and track adapter-side reconciliation as a separate multi-adapter follow-up.
