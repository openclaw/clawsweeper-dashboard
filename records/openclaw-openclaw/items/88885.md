## Summary

Keep open for maintainer review: the patch appears to address a valid crash-recovery duplicate-turn bug, but it intentionally changes ambiguous session-delivery recovery from replay to fail-safe failed/ and needs explicit maintainer acceptance before merge.

## What This Changes

The PR wires session-delivery recovery into the existing SQLite recovery_state marker, adds send-boundary hooks in restart-continuation delivery, and adds focused recovery regression tests.

## Best Possible Solution

Land this bounded recovery-state fix if maintainers accept fail-safe refusal of ambiguous restarted turns, and track adapter-side reconciliation as a separate multi-adapter follow-up.

## Reproduction Assessment

Yes. Source inspection plus the supplied live-output proof show current main can deliver then fail before ack with no durable marker, and the PR's tests model crash-before-ack, recovered-marker, sent-before-error, and retryable pre-send paths.

## Solution Assessment

Yes, if maintainers accept the tradeoff. This is the best bounded fix short of a larger adapter reconciliation API because it reuses the existing SQLite recovery_state column and mirrors the outbound queue's no-blind-replay contract.
