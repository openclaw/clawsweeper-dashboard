## Summary

Keep open: the recovery-state direction is useful and well-scoped, but the patch still has a pre-dispatch marker-write failure edge that can classify a turn as already run before `dispatchAssembledChannelTurn` starts.

## What This Changes

The branch wires session-delivery recovery to the shared SQLite `recovery_state` marker, adds restart-sentinel delivery hooks, and expands recovery regression tests.

## Best Possible Solution

Fix the pre-dispatch marker-write ordering and add a regression for that edge, then land the bounded recovery-state approach if maintainers accept the fail-safe crash-window semantics.

## Reproduction Assessment

Yes. Source inspection and the supplied proof show the current-main path: a queued `agentTurn` can complete `deliver(entry)` but remain pending if `ackSessionDelivery` is interrupted, so the next recovery pass re-delivers it.

## Solution Assessment

No, not as written. Reusing the shared `recovery_state` marker is the right bounded fix, but the hook must not mark the in-process attempt as started until the durable marker write succeeds before dispatch.
