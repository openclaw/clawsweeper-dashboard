## Summary

Keep open: the PR is still the focused candidate for the linked session-delivery blind-replay bug, but the current marker boundary can fail-safe a retryable record/pre-dispatch failure before any reply could have been sent.

## What This Changes

The PR wires a session-delivery recovery marker into SQLite-backed restart-continuation delivery so recovered already-started agent turns are not blindly replayed after a crash.

## Best Possible Solution

Use the existing afterRecord boundary after inbound recording succeeds and before dispatch starts, add a regression for record/pre-dispatch failure, then have maintainers explicitly accept fail-safe refusal or pause for adapter reconciliation design.

## Reproduction Assessment

Yes. Source inspection and the supplied terminal proof give a high-confidence path: enqueue an agentTurn, let delivery succeed without completing ackSessionDelivery, then run recovery again and observe current main replay the pending entry.

## Solution Assessment

No, not yet. Reusing the shared SQLite recovery_state column is the right bounded direction, but this patch sets the marker before the kernel’s inbound-recording boundary, so the marker must move to afterRecord before the solution is the best fix.
