## Summary

Keep open: current main and the latest release still deliver then ack session-delivery entries without a durable send marker, while this PR is a focused fix that remains useful but needs maintainer acceptance of the intentional fail-safe recovery tradeoff before merge.

## What This Changes

The branch wires session-delivery recovery_state marking/refusal hooks through the restart-continuation drain and adds regression coverage for crash-before-ack, recovered-marker, sent-before-error, pre-send failure, and busy-deferred paths.

## Best Possible Solution

Land this bounded recovery-state fix if maintainers accept fail-safe refusal of ambiguous restarted turns; otherwise pause for a larger adapter reconciliation design.

## Reproduction Assessment

Yes. Source inspection shows current main delivers then acks with no marker, and the PR body supplies a before/after terminal probe for the crash-before-ack recovery path; I did not run tests in this read-only review.

## Solution Assessment

Yes, this appears to be the best bounded fix short of adding a larger adapter reconciliation API. It reuses the existing shared SQLite recovery_state column and mirrors the outbound queue's no-blind-replay invariant while preserving pre-send and busy-deferred retries.
