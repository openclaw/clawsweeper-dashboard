## Summary

Keep open: current main still has the session-delivery blind-replay gap, and this PR is the active focused fix with sufficient proof and no concrete line-level defect found; maintainer review should decide the intentional fail-safe recovery tradeoff before merge.

## What This Changes

The PR wires session-delivery recovery to the existing SQLite recovery_state marker, adds send-boundary hooks in restart-continuation delivery, and expands session recovery tests.

## Best Possible Solution

Land this bounded recovery-state fix if maintainers accept fail-safe refusal of ambiguous restarted turns, and track adapter-side reconciliation as a separate multi-adapter follow-up.

## Reproduction Assessment

Yes. Source inspection shows current main delivers then acks with no session recovery marker, and the linked issue plus PR proof model a crash-before-ack restart where the same pending agentTurn is delivered twice.

## Solution Assessment

Yes, this appears to be the best bounded fix short of a broader adapter reconciliation API. It reuses the existing SQLite recovery_state column and mirrors the outbound no-blind-replay contract while preserving pre-send and busy-deferred retries.
