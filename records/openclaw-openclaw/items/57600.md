## Summary

Keep this PR open. Current main still lacks the proposed read-tool binary-document fallback, the linked issue #35406 and broader same-author PR #57612 are both still open, and the PR diff has a concrete blocking regression: it re-reads the entire target file into memory to sniff MIME after the paged read path returns.

## What This Changes

The PR adds a read wrapper option that re-reads the requested file, detects PDF and Office MIME types, and replaces raw text output with a structured fallback message, with one DOCX regression test.

## Best Possible Solution

Keep one canonical document-read path: either land a bounded fallback guard for PDF/Office formats or fold the safe fallback into the broader native document extraction work, with MIME sniffing limited to a small prefix and covered for both host and sandbox reads.

## Reproduction Assessment

Yes for the PR regression: the diff shows every non-empty text read can call loadBuffer(filePath), and the supplied host/sandbox callbacks read the full file into a Buffer before detectMime. A runtime reproduction was not run because this review is read-only.

## Solution Assessment

No. The fallback behavior is a useful narrow step, but the implementation should use bounded sniffing or read-operation metadata instead of a second whole-file read, and maintainers should decide how it relates to #57612 before landing.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Bound the MIME sniff read:** `src/agents/pi-tools.read.ts:326-327`
  - body: The new normalization awaits a full Buffer from readBufferForToolPath before calling detectMime, and both supplied callbacks satisfy that by reading the entire file. A large PDF/Office document will now be loaded completely after the paged read path already returned, bypassing the wrapper's adaptive maxBytes cap and making read(path) able to spike memory on the files this guard targets. Please sniff a bounded prefix or expose bounded MIME metadata from the read operations instead.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff introduces a local resource-exhaustion concern by adding an unbounded whole-file read for MIME sniffing; it does not add dependencies, workflow changes, or package-publishing changes.

  - body: The proposed readBufferForToolPath contract requires a full Buffer, and the host/sandbox implementations read the full target file before MIME detection. A user-controlled read of a very large document can consume memory and CPU outside the existing adaptive read cap.
