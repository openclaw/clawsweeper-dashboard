## Summary

## What This Changes

The PR adds PDF/Office MIME detection to the agent `read` wrapper, returns fallback text for those documents, wires host/sandbox buffer reads, and adds a DOCX fallback regression test.

## Best Possible Solution

Land one bounded document-read path that prevents raw binary token waste, using extension or bounded-prefix detection and coordinating DOCX extraction through the document-extractor contract.

## Reproduction Assessment

Yes, by source inspection. Current main only normalizes image reads after the upstream read path and the sandbox MIME hook returns image MIME types only, so PDF/Office documents still fall through to non-image text handling; I did not run a live read in this read-only review.

## Solution Assessment

No, not as-is. The warning fallback is useful, but it should use extension or bounded-prefix detection and be coordinated with the open parser-backed extraction work.
