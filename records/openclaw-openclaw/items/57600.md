## Summary

Keep open: the underlying document-read problem is real and still unresolved on current main, but this PR is not merge-ready because it lacks real behavior proof and introduces an unbounded full-file MIME sniffing path.

## What This Changes

The branch adds PDF/Office MIME detection to the agent read wrapper, returns fallback text for detected binary documents, wires host and sandbox buffer reads, and adds a DOCX fallback regression test.

## Best Possible Solution

Land one bounded document-read path that prevents binary token waste without full-file sniffing, coordinated with the document-extractor/PDF/DOCX boundary and backed by real read proof.

## Reproduction Assessment

Yes, by source inspection. Current main only special-cases image handling in read, while the proposed fix path and its unbounded loadBuffer call are visible directly in the PR diff; I did not run a live binary read because this review is read-only.

## Solution Assessment

No, not as-is. The fallback direction is useful, but the implementation should sniff a bounded prefix or use extension/already-bounded MIME metadata, and parser-backed extraction should be coordinated with the open document-extraction work.
