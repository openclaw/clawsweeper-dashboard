## Summary

Keep open. The useful fallback is not on current main, and this external draft PR still has a P2 resource concern, missing real behavior proof, and overlap with the separate DOCX extraction PR.

## What This Changes

The branch adds PDF/Office MIME detection to the read wrapper, a structured fallback message, host/sandbox buffer readers, and DOCX fallback regression coverage.

## Best Possible Solution

Use one canonical document-read path with bounded MIME sniffing or bounded extraction metadata, preserving read limits and coordinating the fallback with the DOCX extraction design.

## Reproduction Assessment

Yes for the review finding. Source inspection of the PR diff shows normalizeBinaryDocumentReadResult awaits a full readBufferForToolPath result before MIME detection, and both injected readers return full buffers.

## Solution Assessment

No. The fallback direction is useful, but this implementation should sniff only a bounded prefix or pass bounded MIME metadata through the existing read operation, with maintainer coordination alongside https://github.com/openclaw/openclaw/pull/57612.
