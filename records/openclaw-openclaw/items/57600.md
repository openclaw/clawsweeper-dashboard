## Summary

Keep open. Current main still has the reported binary-document read behavior, but this external draft is not mergeable as-is because its MIME guard performs an unbounded full-file read and the PR still lacks real after-fix behavior proof.

## What This Changes

The PR adds a PDF/Office MIME guard to the `read` wrapper, wires host and sandbox buffer reads for detection, and adds a DOCX fallback regression test.

## Best Possible Solution

Land one bounded, maintainer-approved document-read path that prevents binary token waste without bypassing read caps, with parser-backed extraction coordinated through the document-extractor/plugin contract.

## Reproduction Assessment

Yes by source inspection: current main's wrapper still lets non-image `read` results flow through the upstream UTF-8 text path. I did not run a live file read because this cleanup review is read-only.

## Solution Assessment

No, not as-is. The fallback direction is useful, but this implementation should sniff a bounded prefix or reuse bounded MIME metadata, and the parser-backed path should be coordinated with the open DOCX extraction work.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Bound MIME sniffing reads:** `src/agents/pi-tools.read.ts:333`
  - body: The new normalization awaits a full buffer from `readBufferForToolPath` before MIME detection, and the host implementation reads the full target file. Large PDFs or Office documents can therefore bypass the wrapper's adaptive read cap and spike memory after the normal read path already returned; sniff a bounded prefix or pass bounded MIME metadata through the read operation instead.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: Needs attention: the patch adds full-buffer MIME sniffing of local documents outside existing read limits.

  - body: `readBufferForToolPath` returns a full `Buffer`, and the host implementation uses `fs.readFile` before MIME detection. A user-controlled read of a very large document can consume memory outside the existing adaptive read limit.
