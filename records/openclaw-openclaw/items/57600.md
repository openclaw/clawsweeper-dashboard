## Summary

Keep open: current main still lacks a generic PDF/Office read fallback, but this draft PR is not merge-ready because it conflicts with main, lacks real behavior proof, and adds an unbounded full-file sniff path.

## What This Changes

The PR adds MIME-based PDF/Office detection to the read wrapper so matching documents return a structured fallback message, with host/sandbox wiring and a DOCX regression test.

## Best Possible Solution

Land one bounded document-read path that prevents binary token waste without full-file sniffing, coordinated with the document-extractor/PDF/DOCX boundary and backed by real read proof.

## Reproduction Assessment

Yes, by source inspection. Current main only special-cases image handling in read, while the proposed fix path and its unbounded loadBuffer call are visible directly in the PR diff; I did not run a live binary read because this review is read-only.

## Solution Assessment

No, not as-is. The fallback direction is useful, but the implementation should sniff a bounded prefix or reuse bounded MIME metadata, and parser-backed extraction should be coordinated with the open document-extraction work.
