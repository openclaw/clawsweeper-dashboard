## What This Changes

The issue reports 2026.5.19 Telegram/Codex harness stalls where turns stop after active work starts and session rows remain `running` while `hasActiveRun` is false.

## Best Possible Solution

Enforce a shared recovery/lifecycle invariant: when stuck embedded or model work is abort-drained, force-cleared, or lane-released, persisted session rows and `sessions.changed` should become terminal or accurately remain active, with Telegram/Codex proof before closing this issue.
