## Summary

## What This Changes

This PR packs npm GitHub/git package update specs into a temporary tarball before staged global npm install, with related CLI help, docs, changelog, and update tests changed.

## Best Possible Solution

Keep the pack-before-install design, but make the detector match npm's hosted GitHub source URL contract and cover aliased no-`.git` HTTP(S) forms before merge.

## Reproduction Assessment

Yes at source level: npm-package-arg and hosted-git-info classify hosted GitHub HTTPS URLs without `.git` as git specs, while this PR's detector does not pack them. I did not run a full live `openclaw update` reproduction in this read-only review.

## Solution Assessment

No, not yet. Prepacking git source specs is the right repair, but the implementation needs the no-`.git` hosted GitHub HTTPS detector/test case or an explicit maintainer decision to reject that form.
