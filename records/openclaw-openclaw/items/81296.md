## Summary

Keep open: the core prepack approach and supplied terminal proof are useful, but the PR still misses npm-supported GitHub shorthand and SCP-style git specs, so some promised git update targets would bypass the pack step and lose the current fail-closed behavior.

## What This Changes

This PR pre-packs npm GitHub/git update specs into a temporary tarball before the staged global npm install, removes early rejection for OpenClaw source specs, and updates CLI docs, tests, and changelog text.

## Best Possible Solution

Keep the pack-before-install design, but classify every npm-supported git package spec form that OpenClaw accepts before removing the early source-target rejection, with focused regression tests for the missed forms.

## Reproduction Assessment

Yes at source level: current main rejects OpenClaw source package targets before update, and the PR body proves the `github:` pack/install sequence succeeds. The new blocker is also source-reproducible from the regex versus npm's documented package-spec forms.

## Solution Assessment

No, not yet. Prepacking git source specs is the right repair, but the detector must cover npm's GitHub shorthand and SCP-style SSH forms or explicitly keep those forms rejected before merge.
