## Summary

Keep this PR open, but do not merge it as-is: the Slack bug is source-reproducible, yet the patch changes the shared mention-gating contract for every caller and can suppress non-Slack messages that current docs and tests say should pass when mention detection is unavailable. It also lacks after-fix real Slack behavior proof.

## What This Changes

The PR makes required-mention Slack channels fail closed when explicit mention detection is unavailable, adds Slack log context, and updates mention-gating regression tests.

## Best Possible Solution

Preserve the shared mention-gating default, add a Slack-scoped fail-closed branch for the `auth.test`/bot-id unavailable path while preserving `app_mention` and authorized command bypasses, add focused tests, and include redacted real Slack proof.

## Reproduction Assessment

Yes, from source: when Slack `auth.test` fails, `botUserId` stays empty, `canDetectMention` can be false, and current main only skips required-mention messages when detection is possible. I did not run a live Slack reproduction in this read-only review.

## Solution Assessment

No. The Slack fail-closed direction is reasonable, but changing the shared helper is too broad; the safer fix is a Slack-local fail-closed branch or an explicit policy mode with docs/tests for every affected caller.
