## Summary

Keep open for maintainer review. Current main still restores legacy top-level routed session keys verbatim, and the PR is a focused fix with regression coverage, but it intentionally changes persisted session-state compatibility for older browser storage.

## What This Changes

The PR changes Control UI settings restore so legacy top-level routed session keys fall back to main while scoped per-gateway routed selections are preserved.

## Best Possible Solution

Land the storage-restore fix only after maintainers explicitly accept the legacy top-level routed-session compatibility boundary and keep the scoped-key preservation tests.

## Reproduction Assessment

## Solution Assessment

Yes as an implementation shape: save already writes scoped `sessionsByGateway`, and the PR preserves that current shape while sanitizing only legacy top-level restore input. The unresolved part is whether maintainers accept resetting those older top-level routed fields to `main`.
