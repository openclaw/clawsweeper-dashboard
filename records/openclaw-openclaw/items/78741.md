## Summary

Keep open. The PR targets a valid Control UI restore bug, but the current diff is incomplete because its sanitizer misses routed direct-session key shapes that current routing can persist, and the external real-behavior proof is still only test/check output.

## What This Changes

Adds Control UI storage restore sanitization for persisted routed direct/group session keys and storage tests expecting those restores to fall back to `main`.

## Best Possible Solution

Use storage-layer restore sanitization backed by the shared routed-session key contract so all persisted routed direct/group keys reset to main by default while explicit URL or session-picker selections still work.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: seed `sessionsByGateway` with a routed key, call `loadSettings()`, and the value flows into startup settings before `applySessionDefaults()` preserves it as user-selected.

## Solution Assessment

No. Resetting persisted routed sessions during storage restore is the right boundary, but this implementation should cover all current routed direct/group key shapes or reuse shared parsing instead of the narrower regex.
