## Summary

This PR should stay open: it targets the right Control UI restore layer, but the current sanitizer still misses valid colon-bearing routed topic/thread session keys and the supplied proof is unit/jsdom output rather than a real browser restore check.

## What This Changes

The PR changes Control UI settings restore to sanitize persisted routed direct/group session keys back to `main` and adds storage tests for several routed key shapes.

## Best Possible Solution

Use a contract-aware routed-session detector that covers all direct/group session-key shapes, preserve non-routed agent sessions, and require browser reload proof plus maintainer acceptance of the reset-to-main restore policy.

## Reproduction Assessment

Yes for source-level reproduction: seed `openclaw.control.settings.v1:<gateway>` with a routed key in `sessionsByGateway`, and current main returns it unchanged from `loadSettings()`. I did not run a live browser repro in this read-only review.

## Solution Assessment

No: the storage restore layer is the right boundary, but the current matcher is too shape-specific and misses existing routed topic/thread keys. A safer fix should derive the routed-session classification from the session-key contract or cover all direct/group suffix forms explicitly.
