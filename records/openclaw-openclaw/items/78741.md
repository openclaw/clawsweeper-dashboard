## What This Changes

This PR adds a Control UI settings restore sanitizer that resets legacy top-level routed session keys to `main` while preserving scoped `sessionsByGateway` selections, plus regression coverage.

## Best Possible Solution

Land this only after maintainers accept that legacy top-level routed selections are stale restore state while scoped `sessionsByGateway` remains the durable persisted-selection path.

## Reproduction Assessment

## Solution Assessment

Yes, if maintainers accept the compatibility boundary. The storage restore layer is the narrowest place to normalize stale persisted selections, and the latest patch avoids resetting explicit scoped `sessionsByGateway` choices.
