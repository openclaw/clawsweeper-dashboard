## Summary

Keep open: the fix is useful, but the restore sanitizer is still too broad because it resets canonical group/channel session keys to `main` before the UI knows whether the session is stale or valid.

## What This Changes

The PR adds a Control UI settings restore sanitizer for routed direct/dm/group session-key shapes and regression tests expecting those restored keys to fall back to `main`.

## Best Possible Solution

Preserve canonical group/channel/topic session restores and narrow any reset to keys proven stale or to a maintainer-approved legacy subset with focused upgrade tests.

## Reproduction Assessment

Yes from source: a selected group session is persisted by `resetChatStateForSessionSwitch`, then the PR classifier marks the same key shape historical and `loadSettings` restores `main`. I did not execute tests because this review was required to keep the checkout read-only.

## Solution Assessment

No. The current patch is a plausible mitigation for stale direct/thread keys, but the best fix needs to avoid discarding valid group/channel/topic session keys before session availability is known.
