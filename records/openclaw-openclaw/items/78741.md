## Summary

Keep this PR open: it targets a real Control UI restore bug, but the proposed detector misses supported routed direct-session key shapes and the external proof is still test-only.

## What This Changes

The PR adds Control UI settings-load sanitization for persisted routed direct/group session keys and storage tests that expect those restored selections to fall back to main.

## Best Possible Solution

Broaden the storage-layer routed-session reset to all current direct/group key shapes or a shared parser, keep explicit URL/session-picker selections intact, add focused storage coverage and a changelog entry, then supply real Control UI reload proof.

## Reproduction Assessment

Yes, from source inspection. Seeding Control UI local settings with a routed `sessionsByGateway` key exercises the current-main restore path before gateway defaults, without needing backend routing changes.

## Solution Assessment

No. Resetting at storage restore is the right layer, but this PR’s regex is narrower than the current routed direct-session key contract and should be replaced with broader/shared session-key detection.
