## Summary

Keep open: the linked bug is source-reproducible and the PR targets the right storage restore layer, but the current detector misses valid routed direct-session key shapes and the external proof is only test/check output.

## What This Changes

The PR adds Control UI settings-load sanitization for restored routed direct/group session keys and storage tests expecting those persisted selections to fall back to main.

## Best Possible Solution

Use storage-layer restore sanitization backed by the shared routed-session key contract so all persisted direct/group routed keys reset to main by default while explicit URL or session-picker selections remain intact.

## Reproduction Assessment

Yes. Seeding scoped Control UI local settings with a routed sessionsByGateway key flows through loadSettings() on current main and is then preserved by applySessionDefaults() as a user-selected session.

## Solution Assessment

No. Resetting persisted routed sessions during storage restore is the right layer, but this implementation should cover all current routed direct/group key shapes or reuse a shared classifier instead of the narrower regex.
