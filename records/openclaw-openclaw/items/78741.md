## Summary

Keep open: this PR targets a valid current Control UI restore bug, but the patch is not ready because the routed-session detector misses current direct-message key shapes and the external real behavior proof is still test-only.

## What This Changes

The branch adds Control UI settings-load sanitization that resets persisted routed direct/group session selections to main, plus storage tests for two restore cases.

## Best Possible Solution

Use storage-layer restore sanitization backed by the routed-session key contract or shared parsing, preserve explicit URL/session-picker selections, and keep the linked issue open until a complete fix lands.

## Reproduction Assessment

Yes. Seeding scoped Control UI local settings with a routed sessionsByGateway entry such as agent:main:feishu:direct:ou_xxx flows through loadSettings(), and applySessionDefaults() preserves non-alias session selections on current main; no tests were run because this review is read-only.

## Solution Assessment

No. Resetting persisted routed sessions during storage restore is the right boundary, but this implementation should cover all current routed direct/group key shapes or reuse a shared parser instead of the narrower regex.
