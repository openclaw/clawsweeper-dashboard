## What This Changes

The PR adds a Control UI storage sanitizer that resets legacy top-level routed session keys to `main` on restore while preserving scoped `sessionsByGateway` selections, plus focused storage regression tests.

## Best Possible Solution

Land this only if maintainers accept that legacy top-level peer-routed session keys are unsupported restore targets; otherwise narrow the predicate or move the rule into a shared session-key policy.

## Reproduction Assessment

Yes. Current main source restores top-level `parsed.sessionKey` when no scoped entry exists, so a stored legacy routed key can become the initial Control UI session on reload; I did not execute tests because this review is read-only.

## Solution Assessment

Yes, if maintainers accept the restore policy. The PR fixes the browser-local legacy storage boundary without changing gateway routing or live session switching; the safer alternative is narrowing the classifier if those legacy peer keys remain supported.
