## Best Possible Solution

Normalize historical routed session keys consistently at the Control UI restore boundary for persisted local state, while preserving explicit URL/session selections or a maintainer-approved scoped-session contract with proof.

## Reproduction Assessment

Yes, from source inspection: seed the current scoped settings key with a routed `sessionsByGateway` selection and `loadSettings` returns it as `sessionKey`; app initialization then uses that restored value before gateway defaults preserve it as a selected session.

## Solution Assessment

No. Sanitizing only legacy top-level keys is too narrow because current `saveSettings` stores session selections under `sessionsByGateway`; the safer fix is to sanitize restored persisted routed selections consistently unless an explicit session request or maintainer-approved contract says otherwise.
