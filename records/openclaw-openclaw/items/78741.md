## Summary

Keep this PR open: it fixes the reported stale restore case, but the current patch can also reset valid user-selected routed sessions to `main` on reload before the gateway can prove they are stale.

## What This Changes

The PR changes Control UI settings restore so persisted direct, group, DM, per-peer, and threaded routed session keys are sanitized back to `main`.

## Best Possible Solution

Keep the stale historical restore fix, but preserve explicit routed session selections or defer cleanup until gateway session state can distinguish missing stale rows from valid selected rows.

## Solution Assessment

No. The PR is a plausible mitigation for stale historical keys, but it is not the best fix while it cannot distinguish stale restored keys from valid explicit routed session selections.
