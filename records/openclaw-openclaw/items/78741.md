## What This Changes

The PR updates Control UI settings restore to reset legacy top-level routed session selections to `main` while preserving scoped `sessionsByGateway` selections, with regression coverage for both paths.

## Best Possible Solution

Land this as the UI restore fix only after maintainers accept that legacy top-level routed selections are historical state and `sessionsByGateway` is the durable restore source for explicit routed session choices.

## Reproduction Assessment

## Solution Assessment

Yes, with maintainer acceptance: the PR is an acceptable targeted mitigation because it sanitizes only the legacy top-level restore path while preserving scoped `sessionsByGateway` selections. The safer alternative, if maintainers still support those legacy top-level keys, is to preserve them and add a more explicit restore/default contract.
