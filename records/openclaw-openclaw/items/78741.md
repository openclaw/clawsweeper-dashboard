The PR adds a Control UI settings-restore sanitizer that resets persisted historical routed direct/group session keys to `main` and adds storage tests for several routed key shapes.

## Best Possible Solution

Use a contract-aware storage restore sanitizer that covers all routed direct/group session-key shapes, preserves non-routed agent sessions, and lands with redacted browser reload proof plus maintainer acceptance of the reset-to-main policy.

## Reproduction Assessment

Yes for source-level reproduction: seed `openclaw.control.settings.v1:<gateway>` / `sessionsByGateway` with a routed key and current `loadSettings()` returns it unchanged. I did not run tests or a live browser repro in this read-only review.

## Solution Assessment

No. The storage restore layer is the right boundary, but the current matcher is too segment-count-specific and should cover the shared routed-session key contract before merge.
