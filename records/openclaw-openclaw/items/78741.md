## Summary

Keep open: the reported stale routed-session restore problem is real, but this PR fixes it with a string-shape classifier that also treats valid routed session keys as historical before gateway-backed session state is available.

## What This Changes

The PR changes Control UI storage restore behavior to normalize historical routed session keys back to `main` and adds storage regression coverage.

## Best Possible Solution

Preserve valid routed session selections by default and only fall back to `main` after authoritative gateway/session-list validation proves the saved key is stale.

## Reproduction Assessment

Yes. Source inspection shows current main restores saved scoped session keys directly, and the PR classifier marks canonical routed keys as historical before gateway sessions are loaded.

## Solution Assessment

No. The persistence boundary is plausible, but a local string-shape fallback is too broad because current UI display and delivery-route code treat those routed key shapes as valid sessions.
