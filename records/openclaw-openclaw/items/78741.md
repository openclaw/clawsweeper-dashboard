The PR sanitizes historical routed top-level Control UI session selections to `main`, preserves scoped `sessionsByGateway` selections, and adds storage plus browser E2E regression coverage.

## Best Possible Solution

Land this restore-layer fix after maintainers explicitly accept the legacy top-level routed-session reset policy, while preserving scoped selections and backend routing semantics.

## Solution Assessment

Mostly yes. Storage restore is the right boundary because it hydrates browser-local settings before `OpenClawApp.sessionKey` initializes; the safer alternative is only needed if maintainers want a narrower classifier or a different restore policy.
