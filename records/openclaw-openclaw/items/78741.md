## Summary

Keep open. The PR targets the right Control UI storage layer, but the routed-session detector misses supported direct-session key shapes, and the external-PR proof is only unit/type/lint output rather than a real Control UI restore run.

## What This Changes

The PR adds Control UI storage sanitization and two storage tests so persisted `agent:*:*:(direct|group):*` selections restore to `main`.

## Best Possible Solution

Keep the storage-layer fallback, but define routed-session detection against all current session-key shapes or a shared parser/helper, add coverage for per-peer and per-account direct keys, add the required changelog entry, and preserve explicit URL/session-picker selections.

## Reproduction Assessment

Yes from source inspection. Current main restores scoped `sessionsByGateway` selections before gateway defaults, and seeding that storage with a routed key exercises the reported path without changing backend routing.

## Solution Assessment

No. The storage-layer reset is the right direction, but the PR regex is too narrow for supported routed direct-session keys and should be broadened or delegated to shared routing/session-key logic.
