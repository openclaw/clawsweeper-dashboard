## Summary

Keep open. The PR targets the right Control UI restore boundary, but the current detector can still reset valid non-routed session keys and the supplied proof is jsdom/unit-test output rather than a real browser reload.

## What This Changes

The PR adds Control UI storage sanitization that resets persisted routed direct/group session keys to `main` during settings restore, plus storage tests for several routed key shapes.

## Best Possible Solution

Match only the canonical routed peer-session shapes generated by `buildAgentPeerSessionKey`, preserve other valid agent session keys, and merge after browser restore proof plus maintainer acceptance of the reset policy.

## Reproduction Assessment

Yes for source-level reproduction: current main restores scoped session keys unchanged and gateway defaults preserve non-alias keys as user-selected. I did not run browser proof because this review is read-only.

## Solution Assessment

No: storage restore is the right layer, but the regex should be narrowed to exact routed peer key shapes and backed by preservation tests plus real browser reload proof.
