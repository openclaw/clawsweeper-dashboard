## Summary

Keep this PR open for maintainer review: current main still restores legacy top-level routed Control UI session keys, and this branch is a bounded storage-layer fix with focused regression coverage. There are no blocking code findings from this read-only review, but the PR intentionally changes browser-local session restore compatibility for legacy direct/dm/group-shaped keys.

## What This Changes

The PR updates Control UI storage restore to sanitize legacy top-level routed session keys back to `main` while preserving routed selections stored in `sessionsByGateway`, and adds focused storage regression cases.

## Best Possible Solution

Land only after maintainers explicitly accept that legacy top-level routed session keys should no longer restore, while preserving current scoped `sessionsByGateway` selections for explicit routed session picks.

## Reproduction Assessment

Yes. Current main deterministically loads top-level `parsed.sessionKey` when no scoped session selection exists, so a persisted legacy routed key can become the restored Control UI session on reload; I did not execute tests because this was a read-only review.

## Solution Assessment

Yes, this is the best bounded fix shape: it changes the legacy restore boundary in `ui/src/ui/storage.ts` without touching gateway routing, heartbeat targeting, or live session switching. The remaining question is maintainer acceptance of the legacy compatibility change, not a better code layer.
