## Summary

Keep open. The PR targets a real Control UI restore bug, but the detector still misses valid routed direct/group session keys with longer topic/thread/sender suffixes, and the supplied proof is unit-test output rather than a real Control UI reload.

## What This Changes

The PR adds a Control UI storage restore sanitizer that resets persisted routed direct/group session selections to main and extends storage tests for several routed key shapes.

## Best Possible Solution

Keep the storage-layer fix, but base the detector on the shared routed session-key contract or cover all direct/group keys with topic/thread/sender suffixes, while preserving explicit session selection and adding real Control UI restore proof.

## Reproduction Assessment

Yes. Source inspection shows current main restores sessionsByGateway values directly and applySessionDefaults preserves non-alias sessions; in the PR, keys longer than seven segments are not sanitized, so seeding one of the documented topic/sender shapes would still restore it.

## Solution Assessment

No as written. The storage layer is the right boundary, but the detector should cover the shared routed-session key contract and the PR still needs real browser/localStorage reload proof.
