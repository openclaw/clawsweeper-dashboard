## What This Changes

The PR changes Control UI settings restore so legacy top-level routed session keys fall back to `main`, preserves scoped `sessionsByGateway` selections, and adds storage plus browser E2E regression tests.

## Best Possible Solution

Land the restore-layer fix after maintainers explicitly accept resetting legacy top-level routed selections to `main` while preserving scoped selections and backend routing semantics.

## Solution Assessment

Mostly yes. Storage restore is the right boundary because the app initializes `sessionKey` from `loadSettings()` before gateway connection; the remaining question is maintainer acceptance of the legacy reset policy.
