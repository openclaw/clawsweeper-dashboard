## Summary

Keep open: the PR addresses a real stale Control UI restore report, but the current shape-only sanitizer would also discard valid channel-routed session selections, so it is not merge-ready without a narrower restore policy.

## What This Changes

The PR adds Control UI settings-restore logic that normalizes historical routed session keys back to `main` and expands storage regression coverage for routed-session restore cases.

## Best Possible Solution

Preserve valid routed session selections by default, and only fall back to `main` after gateway-backed session/default validation proves the saved key is stale, with regression tests for both stale and valid routed keys.

## Reproduction Assessment

Yes. Source inspection shows the added predicate returns true for canonical routed session keys, and `loadSettings()` seeds `OpenClawApp.sessionKey` from that sanitized value before gateway session rows are available.

## Solution Assessment

No. The fix is in the right persistence area, but a local string-shape classifier is the wrong boundary because current UI, docs, and route-preservation tests treat those same shapes as valid sessions.
