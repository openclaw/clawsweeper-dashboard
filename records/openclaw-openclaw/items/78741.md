## Summary

Keep open: this fixes a real Control UI restore bug in the right area, but the current matcher can reset non-peer persisted agent sessions and the supplied proof is still unit/jsdom-only.

## What This Changes

The PR adds Control UI settings-load sanitization that resets persisted direct/group routed session keys to `main` and expands storage tests for routed key shapes.

## Best Possible Solution

Constrain the detector to exact `buildAgentPeerSessionKey` routed peer shapes, preserve non-routed agent sessions with tests, and merge only after redacted browser reload proof plus maintainer acceptance of the reset policy.

## Reproduction Assessment

Yes for source-level reproduction: seed `openclaw.control.settings.v1` / `sessionsByGateway` with a routed key such as `agent:main:feishu:direct:ou_xxx`, call `loadSettings()`, and current main restores it unchanged. I did not run tests or browser proof because this review is read-only.

## Solution Assessment

No: the storage layer is the right boundary, but this implementation should use an exact routed peer-session detector and preservation coverage before it is the safest fix.
