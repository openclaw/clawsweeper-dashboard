Keep open. ## What This Changes

The issue requests a multi-step RuntimePlan and embedded-runner finalization roadmap covering baseline docs, native Harness V2 adoption, runner file splits, neutral embedded-runner naming, final docs, and smoke verification.

## Best Possible Solution

Keep the RFC open while maintainers either retire the roadmap explicitly or split the remaining work into narrow follow-ups. If continuing, start with one bounded structural slice such as model/auth-plan extraction or native V2 registry adoption, then handle naming/docs and smoke verification separately.
