## Summary

Keep open. The reported Chat Completions bug is valid, but this PR fixes it at the shared reasoning resolver and would also disable `gpt-5.4-mini` reasoning for Responses and Codex Responses callers.

## What This Changes

The PR adds a changelog entry, helper-level coverage, and a `gpt-5.4-mini` matcher that returns no supported reasoning efforts from the shared OpenAI reasoning resolver.

## Best Possible Solution

Keep `gpt-5.4-mini` reasoning enabled for Responses paths and suppress only the invalid Chat Completions `reasoning_effort` field, with transport-level regression coverage.

## Reproduction Assessment

Yes. The linked issue gives concrete Telegram/custom OpenAI-compatible steps and a 400 response, and source inspection shows current main resolves `gpt-5.4-mini` to supported efforts before assigning Chat Completions `reasoning_effort`.

## Solution Assessment

No. The current PR suppresses reasoning in the shared resolver, but the narrow maintainable fix is at the Chat Completions payload boundary or in an API-scoped resolver.
