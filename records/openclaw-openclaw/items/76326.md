## Summary

Keep this PR open but do not merge it as-is: it targets a real Chat Completions bug, but the patch disables `gpt-5.4-mini` reasoning through the shared resolver used by Responses as well.

## What This Changes

The PR adds a `gpt-5.4-mini` exclusion to the shared OpenAI reasoning-effort resolver, adds helper tests for mini variants, and records the fix in the changelog.

## Best Possible Solution

Keep the shared resolver/model catalog reasoning-capable for Responses and suppress only the invalid Chat Completions `reasoning_effort` field for `gpt-5.4-mini`, with transport-level regression coverage for both endpoints.

Yes. The linked issue gives a concrete `openai-completions` route and 400 response, and current main source shows that route resolves `gpt-5.4-mini` reasoning and emits top-level `reasoning_effort`.

## Solution Assessment

No. The PR suppresses reasoning in a shared model-level resolver, while the narrow maintainable fix belongs at the Chat Completions payload boundary or in an endpoint-scoped capability resolver.
