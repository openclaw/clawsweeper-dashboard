## Summary

Keep open. The latest PR branch fixes the initial matcher-order problem, but the new exclusion is still placed in a shared resolver used by both Chat Completions and Responses, so it can regress the Responses path that should keep `gpt-5.4-mini` reasoning enabled.

## What This Changes

The PR adds a changelog entry, a `gpt-5.4-mini` exclusion in `resolveOpenAISupportedReasoningEfforts`, and helper-level regression coverage.

## Best Possible Solution

Keep `gpt-5.4-mini` reasoning enabled for shared and Responses paths, and suppress only the invalid Chat Completions payload field with a payload-builder regression test.

## Reproduction Assessment

Yes. The linked issue gives a concrete Telegram/OpenAI-compatible route and 400 response, and the static code path shows current main sends `reasoning_effort` for `gpt-5.4-mini` on the Chat Completions builder.

## Solution Assessment

No. The PR's matcher order is now reachable, but the helper-level exclusion is broader than the reported `/v1/chat/completions` failure; a transport-scoped suppression is safer.
