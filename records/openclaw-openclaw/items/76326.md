## Summary

Keep open. The PR is narrower than the earlier related attempt and includes a changelog entry, but the new guard is still placed after the broad GPT-5.2+ matcher, so it does not fix the reported `gpt-5.4-mini` chat-completions path; the shared-resolver approach is also broader than the stated `/v1/chat/completions` incompatibility.

## What This Changes

The PR adds a changelog entry and attempts to make `resolveOpenAISupportedReasoningEfforts` return no supported efforts for `gpt-5.4-mini`.

## Best Possible Solution

Replace the resolver-wide exclusion with a chat-completions payload guard or compat rule for native OpenAI `gpt-5.4-mini` function-tool requests, then add focused regression coverage while keeping the changelog entry.

## Reproduction Assessment

Yes. Static inspection gives a high-confidence reproduction: `gpt-5.4-mini` matches the broad GPT-5.2+ resolver branch, and the chat-completions builder can still combine function tools with `reasoning_effort`; #76176 supplies the matching 400 error text.

## Solution Assessment

No. This PR is not the best fix as submitted because the new branch is unreachable and, if made reachable, would disable reasoning in shared non-chat paths; the safer solution is scoped to the failing chat-completions payload shape.
