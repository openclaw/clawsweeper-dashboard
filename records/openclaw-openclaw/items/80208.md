## Summary

Keep open. The import gap is real, but this patch promotes a durable cross-session transcript writer as public SDK without the authority/docs contract already called out by the related feature request and SDK boundary rules.

## What This Changes

The PR adds `openclaw/plugin-sdk/transcript.runtime` to package exports and SDK entrypoints, re-exporting existing assistant transcript append helpers.

## Best Possible Solution

Define a constrained, documented transcript-append SDK contract with explicit authority, assistant-attributed default writes, idempotency guarantees, transcript-update behavior, and export/API checks.

## Reproduction Assessment

Yes for the import gap by source inspection: current main lacks the package export and SDK entrypoint while the internal helper exists. I did not run a package import because this is a read-only review.

## Solution Assessment

No. A raw re-export is the shortest mechanical path, but durable cross-session transcript writes should be exposed only through a constrained, documented, authority-gated SDK/runtime contract.

## Security Review

Status: needs_attention
