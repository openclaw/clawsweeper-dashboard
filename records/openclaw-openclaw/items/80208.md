## Summary

Keep open. Current main still lacks the requested public Plugin SDK transcript export, but this PR promotes durable cross-session transcript writes as a raw public subpath without a settled authority/attribution contract, and the supplied proof does not yet show the actual built package specifier resolving.

## What This Changes

This PR adds a public `openclaw/plugin-sdk/transcript.runtime` subpath, wires it into SDK entrypoints and package exports, and re-exports existing session transcript append helpers and types.

## Best Possible Solution

Define a constrained, documented transcript append contract with explicit authority, attribution, idempotency, update-event semantics, docs, and package/API export coverage before exposing a stable SDK seam.

## Reproduction Assessment

Yes, by source inspection: current main has the internal helper but lacks the `transcript.runtime` package export and SDK entrypoint. The PR's before evidence also matches the expected published-package `ERR_MODULE_NOT_FOUND`, but I did not run a local package import because this is a read-only review.

## Solution Assessment

No. A raw subpath re-export is mechanically small, but the maintainable solution should be a capability-gated or otherwise explicitly authorized Plugin SDK/runtime contract, with documentation for scope and attribution.

## Security Review

Status: needs_attention

Summary: The patch exposes durable transcript writes to third-party plugin code without a concrete authority, consent, or attribution model.

Concerns:

- **[medium] Gate durable transcript writes:** `src/plugin-sdk/transcript.runtime.ts:7`
  - body: The new public subpath would let arbitrary plugin code call helpers that persist assistant content into caller-selected session transcripts. That needs an explicit trusted capability, operator opt-in, or equivalent authority boundary before it becomes supported SDK API.
  - confidence: 0.89
- **[medium] Constrain exact-message attribution:** `src/plugin-sdk/transcript.runtime.ts:8`
  - body: `appendExactAssistantMessageToSessionTranscript` accepts a full assistant message object, so public access would let plugins set provider/model/timestamp-like metadata in durable transcripts unless the contract narrows or documents attribution and replay semantics.
  - confidence: 0.84
