## Summary

Keep open. The import gap is real, but this PR promotes a durable cross-session transcript writer to a public SDK subpath without a documented authority model, and the supplied proof does not yet exercise the package subpath that the PR changes.

## What This Changes

The PR adds `openclaw/plugin-sdk/transcript.runtime` to package exports and the SDK entrypoint list, re-exporting existing assistant transcript append helpers.

## Best Possible Solution

Define a constrained, documented transcript-append SDK/runtime contract with explicit operator authority, assistant attribution, idempotency, transcript-update behavior, and contract coverage.

## Reproduction Assessment

Yes for the import gap by source inspection: current main omits `transcript.runtime` from both `package.json` exports and `scripts/lib/plugin-sdk-entrypoints.json` while the internal helper exists. This is a feature/API gap rather than a broken documented behavior path.

## Solution Assessment

No. A raw package subpath re-export is mechanically small, but durable transcript writes should be exposed through a constrained and documented runtime/capability contract with authority and audit semantics.
