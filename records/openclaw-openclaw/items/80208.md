## Summary

Keep open for maintainer Plugin SDK/security review. Current main does have the import gap this PR targets, but the patch promotes a durable cross-session transcript writer as a raw public SDK subpath without a settled authority contract, docs, or sufficient package-subpath proof.

## What This Changes

The PR adds `openclaw/plugin-sdk/transcript.runtime`, re-exporting existing assistant transcript append helpers and wiring the subpath into SDK entrypoints and `package.json` exports.

## Best Possible Solution

Define a constrained, documented Plugin SDK/runtime transcript-append contract with explicit authority semantics, assistant-attributed writes, idempotency/update-event behavior, export/API checks, and focused tests tied to https://github.com/openclaw/openclaw/issues/73274.

## Reproduction Assessment

Yes for the import gap by source inspection: current main omits `transcript.runtime` from `package.json` exports and `scripts/lib/plugin-sdk-entrypoints.json` while the internal append helper exists. This is a feature/API exposure request rather than a broken documented behavior.

## Solution Assessment

No. A raw subpath re-export is mechanically small, but durable transcript writes should go through a constrained runtime/capability contract with documented authority and attribution semantics.

## Security Review

Status: needs_attention

Summary: The patch introduces a public durable transcript-write SDK surface without a settled authority, consent, or audit model.

Concerns:

- **[medium] Gate durable transcript writes:** `src/plugin-sdk/transcript.runtime.ts:7`
  - body: The new subpath would make transcript append helpers available to arbitrary plugin code as a supported import. Because the helper writes persistent transcript content for caller-supplied sessions, it needs an explicit trusted capability, operator opt-in, or equivalent authority model before becoming stable SDK surface.
  - confidence: 0.88
- **[medium] Avoid public exact-message spoofing semantics:** `src/plugin-sdk/transcript.runtime.ts:8`
  - body: `appendExactAssistantMessageToSessionTranscript` accepts a full assistant message object. Even with the role check, public exact-message writes need attribution and replay semantics before arbitrary plugins can set provider/model/timestamp-like metadata in persistent transcripts.
  - confidence: 0.82
