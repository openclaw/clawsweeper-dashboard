## Summary

Keep open. Current main still lacks the requested Plugin SDK subpath, but this PR promotes durable cross-session transcript writes as public API and still needs maintainer Plugin SDK/security judgment plus stronger package-resolution proof before merge.

## What This Changes

The branch adds `openclaw/plugin-sdk/transcript.runtime` to package exports and SDK entrypoints and re-exports existing assistant transcript append helpers from a new Plugin SDK subpath.

## Best Possible Solution

Define a constrained, documented transcript-append SDK/runtime contract with explicit authority, attribution, idempotency, and update semantics, then align exports, API baseline, docs, and focused checks.

## Reproduction Assessment

Yes by source inspection: current main has the internal transcript append helpers but no `openclaw/plugin-sdk/transcript.runtime` export or entrypoint. This is an API exposure gap, not a broken documented behavior.

## Solution Assessment

No. The raw subpath export is mechanically small, but durable transcript writes into caller-selected sessions should be stabilized only after the authority, attribution, and documentation contract is settled.

## Security Review

Status: needs_attention

Summary: The PR introduces a public durable transcript-write SDK surface without a settled authority, consent, or attribution model.

Concerns:

- **[medium] Gate durable transcript writes:** `src/plugin-sdk/transcript.runtime.ts:7`
  - body: The new subpath would make transcript append helpers available as supported API to arbitrary plugin code. Because the helper writes persistent transcript content for caller-supplied sessions, it needs an explicit trusted capability, operator opt-in, or equivalent authority model before becoming stable SDK surface.
  - confidence: 0.88
- **[medium] Define exact-message attribution semantics:** `src/plugin-sdk/transcript.runtime.ts:8`
  - body: `appendExactAssistantMessageToSessionTranscript` accepts a full assistant message object. Public exact-message writes need attribution and replay semantics before plugins can set provider, model, timestamp-like, or replay-sensitive metadata in persistent transcripts.
  - confidence: 0.82
