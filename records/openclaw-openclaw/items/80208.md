## Summary

Keep open: the SDK import gap is real, but this PR makes a durable cross-session transcript writer a public SDK subpath without a documented authority/audit contract, and the related API design request remains open at https://github.com/openclaw/openclaw/issues/73274.

## What This Changes

The PR adds a package export, SDK entrypoint, and new `src/plugin-sdk/transcript.runtime.ts` file that re-exports existing assistant transcript append helpers.

## Best Possible Solution

Define a constrained, documented transcript-append SDK contract with explicit authority semantics, assistant-attributed writes, idempotency guarantees, update-event behavior, and contract checks.

## Reproduction Assessment

Yes for the import gap by source inspection: current main omits `transcript.runtime` from both `package.json` exports and `scripts/lib/plugin-sdk-entrypoints.json` while the internal helper exists. I did not run a package import because this review is read-only.

## Solution Assessment

No. A raw re-export is the shortest resolution fix, but durable cross-session transcript writes should go through a constrained, documented Plugin SDK/runtime contract with explicit authority and audit semantics.
