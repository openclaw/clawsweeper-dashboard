## Summary

Keep open. The import gap is real, but this PR promotes a durable transcript-write helper to the public Plugin SDK without the authority, attribution, documentation, and API-surface follow-through that this contract needs.

## What This Changes

The PR adds a new `openclaw/plugin-sdk/transcript.runtime` subpath that re-exports existing assistant transcript append helpers and wires it into SDK entrypoint/package exports.

## Best Possible Solution

Define a constrained, documented transcript-append SDK/runtime contract with explicit authority semantics, assistant-attributed default writes, idempotency and update-event behavior, package export checks, and follow-through for https://github.com/openclaw/openclaw/issues/73274.

## Reproduction Assessment

Yes for the import gap by source inspection: current `main` omits `transcript.runtime` from both the package exports and SDK entrypoint list while the internal append helper exists. This is an API exposure request rather than a broken documented behavior bug.

## Solution Assessment

No. A raw re-export is the shortest path, but durable cross-session transcript writes should be exposed through a constrained and documented SDK/runtime contract with explicit authority and attribution semantics.
