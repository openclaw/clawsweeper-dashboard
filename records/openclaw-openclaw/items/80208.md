## Summary

Keep open. The PR addresses a real Plugin SDK exposure gap and the related plugin use case, but it promotes durable cross-session transcript writes as a raw public subpath before the authority, attribution, and documentation contract is settled.

## What This Changes

The PR adds a public `openclaw/plugin-sdk/transcript.runtime` subpath by creating a Plugin SDK re-export file and adding it to the SDK entrypoint list and package exports.

## Best Possible Solution

Define a constrained, documented Plugin SDK/runtime transcript append contract with explicit authority, attribution, idempotency, update-event semantics, and focused contract tests before exposing this as stable public API.

## Reproduction Assessment

Yes by source inspection: current main contains the internal transcript append helper but lacks both the `package.json` export and SDK entrypoint for `transcript.runtime`. This is an API exposure request, not a broken documented behavior.

## Solution Assessment

No. The raw re-export is mechanically consistent with SDK subpath wiring, but the safer maintainable fix is a capability-gated or explicitly documented runtime contract for durable transcript writes.
