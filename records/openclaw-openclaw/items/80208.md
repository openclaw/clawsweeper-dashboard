## Summary

Keep open: the import gap is real, but this PR promotes durable transcript writes into public Plugin SDK surface without a settled authority, attribution, and documentation contract, and the contributor proof still does not show the patched package subpath resolving.

## What This Changes

This PR adds a public `openclaw/plugin-sdk/transcript.runtime` subpath that re-exports existing session transcript append helpers and wires it into SDK entrypoints and package exports.

## Best Possible Solution

Define a constrained, documented Plugin SDK/runtime transcript-append contract with explicit authority, attribution, idempotency, and transcript-update semantics, then expose it through the chosen stable SDK seam.

## Reproduction Assessment

Yes by source inspection: current main has the internal helper, but `package.json` exports and `scripts/lib/plugin-sdk-entrypoints.json` do not include `transcript.runtime`. This is an API exposure request, not a broken documented behavior.

## Solution Assessment

No. A raw re-export is mechanically small, but the durable cross-session transcript write surface should be authority-gated or explicitly documented before becoming stable Plugin SDK API.

## Security Review

Status: needs_attention

Summary: The patch introduces a public durable transcript-write SDK surface without an explicit authority, consent, or attribution model.

Concerns:

- **[medium] Gate durable transcript writes:** `src/plugin-sdk/transcript.runtime.ts:7`
  - body: The new subpath would make transcript append helpers available as supported API to arbitrary plugin code. Because the helper writes persistent transcript content for caller-supplied sessions, it needs an explicit trusted capability, operator opt-in, or equivalent authority model before becoming stable SDK surface.
