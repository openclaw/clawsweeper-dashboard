## Summary

Keep open. The PR addresses a real Plugin SDK import gap, but it promotes durable cross-session transcript writers to public SDK surface before the authority, attribution, and documentation contract is settled; the related feature request remains open.

## What This Changes

The PR adds an `openclaw/plugin-sdk/transcript.runtime` package export, registers it as an SDK entrypoint, and re-exports existing session transcript append helpers.

## Best Possible Solution

Define a constrained, documented Plugin SDK/runtime transcript-append contract with explicit authority, assistant attribution, idempotency/update-event semantics, export/API checks, and focused tests tied to the open feature request.

## Reproduction Assessment

Yes, for the import gap by source inspection: current main lacks the package export and SDK entrypoint while the internal append helper exists. This is a feature/API exposure request rather than broken documented behavior.

## Solution Assessment

No. The raw subpath re-export is mechanically small, but the safer maintainable solution is a constrained runtime/capability contract with documented authority and attribution semantics.

## Security Review

Status: needs_attention

Summary: The patch promotes durable cross-session transcript writes to public SDK surface without explicit authority, consent, or audit semantics.

Concerns:

- **[medium] Gate durable transcript writes:** `src/plugin-sdk/transcript.runtime.ts:7`
  - body: The new subpath would make transcript append helpers available to arbitrary plugin code as a supported import, allowing persistent writes to caller-selected session transcripts without a settled capability or operator opt-in model.
  - confidence: 0.88
