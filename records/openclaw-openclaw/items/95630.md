## Summary

Keep open: current main and v2026.6.9 still lack a channel-turn no-settlement deadline, while adjacent open reports cover restart-recovery and cron-side fallout rather than replacing this focused channel-dispatch work. Because the requested timeout knob, default, user-visible abort, and restart-recovery semantics are new public behavior, this needs maintainer product review before any fix PR is safe.

## What This Changes

Request adding a configurable wall-clock no-settlement deadline for inbound channel agent turns, with visible timeout handling and no restart-recovery marking.

## Best Possible Solution

Design a shared channel-turn deadline in the core channel/reply dispatch path, with approved config/schema/docs, visible timeout delivery, and tests proving text/tool settlement cancels the timer without marking restart recovery.
