## Summary

Keep open: current main still routes inbound channel turns through the shared dispatch path without a no-settlement wall-clock deadline, and the requested timeout/default/config semantics need maintainer direction before implementation.

## What This Changes

Request a configurable wall-clock no-settlement timeout for inbound channel-lane agent turns, with visible abort messaging and no restart-recovery marking.

## Best Possible Solution

Design a shared core channel-turn no-settlement deadline with maintainer-approved config/defaults, visible timeout delivery, and tests proving text/tool settlement cancels the timer without marking restart recovery.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Partly: source inspection confirms current main lacks the requested channel-turn no-settlement deadline, but this review did not live-reproduce the reported 559-second Telegram/Claude hang.

## Solution Assessment

Is this the best way to solve the issue? Unclear: a shared dispatcher-level deadline is plausible because providers converge through the same dispatch path, but the timeout default, config surface, settlement definition, and restart-recovery semantics need maintainer direction first.
