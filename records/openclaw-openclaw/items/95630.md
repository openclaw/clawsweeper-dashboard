## Label Justifications

- `P1`: The report describes live channel turns hanging for minutes and cascading into gateway restart-loop behavior on a released build.
- `impact:session-state`: The requested timeout semantics explicitly depend on preventing restart-recovery session marking after a channel timeout.

- `impact:crash-loop`: The report ties the unbounded channel turn to supervisor restarts and gateway restart-loop behavior.

## Summary

Keep open: current main and the latest release still lack a channel-turn no-settlement deadline, and the request targets a real stability gap, but the timeout knob, default, settlement semantics, and restart-recovery behavior need maintainer product review before implementation.

## What This Changes

The issue requests a configurable wall-clock timeout for channel-dispatched agent turns that do not settle with assistant text or a tool call.

## Best Possible Solution

Design a shared channel-turn no-settlement deadline in the core channel turn or reply dispatch path after maintainers approve the config/default contract, visible timeout response, and restart-recovery semantics.
