## Summary

The issue should stay open: current main still lacks a shared channel-turn no-settlement deadline, and the linked event-loop/channel outage reports show a real maintainer-visible stability problem even though the requested config/default semantics need product review.

## What This Changes

The issue requests a dispatcher-level wall-clock timeout for channel-lane agent turns that make no user-visible settlement, with configurable defaults and non-crashing timeout handling.

## Best Possible Solution

Keep the issue open for a shared channel-turn deadline design that reuses reply abort/settlement hooks, emits a visible timeout response, avoids marking restart-aborted session state, and includes config/schema/docs/tests only after maintainer direction on the public knob and default.

Partly: source inspection shows current main has no shared channel-turn no-settlement timeout, but I did not recreate the reported long live Telegram/Claude hang.

## Solution Assessment

Unclear: the proposed shared deadline is plausible, but the exact config surface, default, settlement definition, and session-recovery semantics need maintainer product and compatibility review.
