## Label Justifications

- `impact:message-loss`: A no-settlement channel turn can leave Telegram or other channel users without a visible reply until manual or supervisor recovery.

## Summary

Keep open: current main still has no dispatcher-level channel-turn no-settlement deadline, and the requested default/config/abort semantics need maintainer product direction before a fix PR is safe.

## What This Changes

The issue requests a configurable wall-clock no-settlement timeout for inbound channel agent turns, with user notification and no restart-recovery marking on timeout.

## Best Possible Solution

Design a shared channel-turn deadline in the core channel/reply dispatch path, with approved config/schema/docs and tests proving text/tool settlement cancels the timer, users get a visible timeout reply, and restart recovery is not marked.

## Reproduction Assessment

Partly: source inspection confirms current main lacks the requested channel-turn no-settlement deadline, but this read-only review did not live-reproduce the reported 559-second Telegram/Claude hang.

## Solution Assessment

Unclear: a shared dispatcher-level deadline is a plausible narrow mitigation, but the default duration, public config shape, settlement definition, and restart-recovery semantics need maintainer direction first.
