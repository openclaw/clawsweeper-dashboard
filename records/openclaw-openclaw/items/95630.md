## Summary

Keep open: current main still does not provide a dispatcher-level no-settlement deadline for channel agent turns, and the proposed timeout/default/config/session-recovery semantics need maintainer direction before a safe fix PR is possible.

## What This Changes

The issue requests a configurable wall-clock deadline for inbound channel agent turns that aborts no-settlement hangs, notifies the channel user, and avoids marking the session for restart recovery.

## Best Possible Solution

Define a maintainer-approved shared channel-turn timeout contract, then implement it in the common inbound dispatch/reply path with config/schema/docs, structured cancellation, visible timeout reply, restart-recovery semantics, and live channel proof.

## Reproduction Assessment

Partly. Source inspection shows no dispatcher-level no-settlement deadline on current main, but this read-only review did not live-reproduce the reported 559-second Telegram/Claude hang.

## Solution Assessment

Unclear. A shared dispatcher-level deadline is a plausible narrow mitigation, but the default duration, public config shape, settlement definition, and restart-recovery behavior need maintainer direction before it is the best fix.
