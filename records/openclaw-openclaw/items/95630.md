## Label Justifications

- `impact:message-loss`: A no-settlement channel turn can leave Telegram or other channel users without a visible reply until manual or supervisor recovery.
- `impact:crash-loop`: The report ties the channel hang to event-loop starvation and supervisor restart loops that can exhaust service restarts.

## Summary

Keep open: current main still lacks a dispatcher-level channel-turn no-settlement deadline, and the remaining work needs maintainer direction on the public config/default, settlement definition, cancellation behavior, and restart-recovery contract.

## What This Changes

The issue requests a configurable wall-clock no-settlement timeout for channel-dispatched agent turns, with clean cancellation, a visible channel reply, and no restart-recovery marking.

## Best Possible Solution

Design a shared channel-turn no-settlement deadline in the core channel/reply dispatch path, with approved config/schema/docs and tests proving text or tool settlement cancels the timer, timeout sends a visible channel reply, and the session is not marked for restart recovery.

## Reproduction Assessment

Partly: source inspection confirms current main lacks the requested channel-turn no-settlement deadline, but this review did not live-reproduce the reported 559-second Telegram/Claude hang.

## Solution Assessment

Unclear: a shared dispatcher-level deadline is a plausible narrow mitigation, but the default duration, public config shape, settlement definition, and restart-recovery contract need maintainer direction first.
