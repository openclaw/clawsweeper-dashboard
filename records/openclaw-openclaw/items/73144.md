## Summary

Keep open. Current main only partially improves the reported model-switch flow: unmanaged gateway restart now has a verified SIGUSR1 fallback and generated models registries merge configured providers, but agent-driven provider setup, allowlist diagnostics, and active-session default remapping remain unresolved design/security work.

## What This Changes

The issue requests a smoother provider/model switch flow covering agent `config.patch` provider additions, global versus agent `models.json` diagnostics, allowlist handling, gateway restart, and active-session model behavior.

## Best Possible Solution

Split this into explicit maintainer-approved pieces: a safe provider setup path, doctor/status checks for provider and allowlist mismatches, and clear session switch/reset UX rather than silent active-session remapping.

## Reproduction Assessment

Unclear. Static current-main review confirms several individual behaviors, but the issue lacks the exact patch payload, registry state, and live switch sequence needed for a high-confidence end-to-end reproduction.

## Solution Assessment

Unclear. Automatically allowing agent `config.patch` writes to `models.providers`, auto-expanding allowlists, or silently remapping active sessions is not clearly the safest path; explicit setup commands plus diagnostics and visible session switching look more maintainable.
