## Summary

Keep open. Current main has materially improved the Qwen/DeepSeek path, model-listing behavior, and unmanaged gateway restart handling, but the remaining provider-write, diagnostics, allowlist, and active-session semantics are still product and security design work rather than a completed fix.

## What This Changes

The issue requests a smoother Qwen-to-DeepSeek model switch flow covering provider setup, dual model config diagnostics, allowlist/default behavior, gateway restart, and active-session model updates.

## Best Possible Solution

Split this into maintainer-approved pieces: explicit provider auth/setup UX, doctor/status diagnostics for provider and model-list mismatches, and visible session switch/reset semantics without broadening agent writes to credential-bearing config paths.

## Reproduction Assessment

No high-confidence end-to-end reproduction is established. Static source inspection confirms several current behaviors, including the protected agent config.patch path and explicit session_status model switching, but the original live config and session sequence are missing.

## Solution Assessment

Unclear. Broadly allowing config.patch to add providers or silently remapping active sessions is not obviously the safest solution; explicit setup, diagnostics, and visible session controls look more maintainable.
