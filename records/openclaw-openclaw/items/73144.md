## Summary

Keep open. Current main has fixed or documented several adjacent parts of the DeepSeek/model-switch path, including DeepSeek catalog docs, models.json generation from configured providers, default-model allowlist upserts, and unmanaged gateway restart fallback, but the central five-part UX report is still not fully resolved: the agent-facing config.patch path still blocks provider writes, configured model views intentionally preserve explicit allowlist semantics, and active sessions still require an explicit session model change rather than automatic remapping after config reload.

## What This Changes

The issue requests a smoother model-switching flow for moving from qwen3.6-plus to deepseek-v4-pro, covering provider setup, global versus agent models.json drift, model allowlists, gateway restart behavior, and active-session model updates.

## Best Possible Solution

Keep the already-landed DeepSeek/default-model/restart improvements, then split the remaining work into maintainer-owned changes: a safe custom-provider setup flow, targeted models status or doctor diagnostics for provider/allowlist drift, and an explicit session switch/reset UX rather than silent active-session remapping.

## Reproduction Assessment

Partially: source inspection gives a high-confidence way to confirm the protected config.patch block, allowlist behavior, and explicit session switch path on current main. A full replay still needs the reporter's exact openclaw.json, agent models.json, and config.patch payload.

## Solution Assessment

Unclear: the requested automatic provider writes, allowlist expansion, and active-session remapping are not obviously the safest implementation. A narrower CLI/status/doctor design with explicit session switching is likely more maintainable.
