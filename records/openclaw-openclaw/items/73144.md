## Summary

Keep open. Current main has partial improvements for models.json merge behavior and unmanaged Gateway restart, but the central model-switch UX bundle remains unresolved: the agent-facing config mutation guard still blocks provider writes, there is no first-class custom provider add command, configured allowlists still intentionally hide non-allowlisted provider models, and active sessions still require explicit per-session model switching rather than automatic remapping after a default-model config reload.

## What This Changes

The issue asks to improve the model-switching setup flow when moving from a Qwen provider to DeepSeek, covering provider config writes, dual model config visibility, allowlist mismatch diagnostics, Gateway restart behavior, and active-session model updates.

## Best Possible Solution

Split the remaining work into maintainer-approved pieces: a safe custom-provider setup path, diagnostics that explain provider/models.json/allowlist mismatches, and explicit current-session switch/reset guidance, while keeping provider endpoint and credential mutation behind a reviewed trust boundary.

## Reproduction Assessment

Unclear. The report includes enough environment and observed errors for a plausible manual reproduction, and static current-main inspection confirms several remaining behaviors, but a high-confidence end-to-end reproduction still needs the reporter's exact patch payload and config state.

## Solution Assessment

Unclear. Automatically allowing provider writes, expanding every configured provider into the allowlist, or silently remapping active sessions is not clearly the safest fix; a narrower CLI/status/doctor flow with explicit session switching looks more maintainable.
