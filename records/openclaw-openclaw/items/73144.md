## Summary

Keep open. Current main has adjacent fixes and documentation for DeepSeek V4, direct config writes, default-model allowlist upserts, and unmanaged Gateway restart, but the central model-switch UX request is not fully implemented: agent-facing config writes still block provider changes, configured model views still let `agents.defaults.models` win over provider catalogs, and active sessions still require explicit model switching.

## What This Changes

Request to improve the UX for switching from `qwen/qwen3.6-plus` to `deepseek/deepseek-v4-pro` across provider config writes, model allowlists, Gateway restart, and active sessions.

## Best Possible Solution

Split this into maintainer-scoped UX work: keep the bundled DeepSeek path, add focused doctor/status warnings for provider entries hidden by agent allowlists or per-agent model state, design an explicit `openclaw models add/set` custom-provider flow if desired, and keep active-session switching explicit unless maintainers approve a clear reset/remap policy.

## Reproduction Assessment

Partially. The report includes concrete versions, setup, model refs, and symptoms, and current source/docs confirm the remaining behaviors by inspection; the exact patch payload and config files are missing for a full five-step replay.

## Solution Assessment

Unclear. Automatic allowlist expansion and active-session remapping may conflict with intentional scoping and trust boundaries, while explicit CLI/status/doctor flows look like the narrower maintainable direction.
