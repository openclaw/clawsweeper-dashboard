Keep open. Current main partially improves the flow, but the remaining request still spans credential-bearing provider setup, allowlist diagnostics/policy, and active-session model semantics that need maintainer design rather than cleanup closure.

## What This Changes

The issue asks to improve the model-switch/provider setup experience after switching from qwen3.6-plus to deepseek-v4-pro, covering provider writes, per-agent model config, allowlists, gateway restart, and active-session model changes.

## Best Possible Solution

Split this into maintainer-approved pieces: a safe operator-approved provider setup path, doctor/status diagnostics for provider and allowlist mismatches, and explicit active-session switch/reset UX instead of silent remapping on config reload.

## Reproduction Assessment

Unclear. Static current-main checks confirm the individual mechanics, but the report lacks the exact config.patch payload, models.json state, and live switch sequence needed for a high-confidence end-to-end reproduction.

## Solution Assessment

Unclear. Automatically allowing provider writes, auto-expanding allowlists, or silently remapping active sessions is not obviously the safest design; explicit setup and diagnostics look more maintainable.
