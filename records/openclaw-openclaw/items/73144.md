## Summary

Keep open. Current main partially improves the model-switch flow, but first-class provider setup, allowlist diagnostics, and active-session default-change semantics remain unresolved product and security-policy work.

## What This Changes

The issue requests a smoother model-provider switch path covering provider creation, global versus agent model config, allowlist behavior, gateway restart, and active-session model refresh.

## Best Possible Solution

Split this into explicit maintainer-approved improvements: a safe provider setup path, doctor/status diagnostics for provider and allowlist mismatches, and clear session-switch or reset semantics.

## Reproduction Assessment

No high-confidence end-to-end reproduction is established. Source inspection confirms several individual behaviors on current main, but the report lacks the exact config patch, models.json state, and live session sequence needed for a full repro.

## Solution Assessment

Unclear. Broadly allowing provider writes, automatically expanding allowlists, or silently remapping active sessions is not clearly the safest path; explicit setup and diagnostics look more maintainable.
