## Summary

Keep open. Current main has partial improvements around model config visibility, models.json merge behavior, and unmanaged gateway restart, but the central UX bundle is still unresolved: the agent-facing gateway tool still blocks provider config writes, there is no first-class provider-add command, allowlist semantics still intentionally hide non-allowlisted provider models, and active-session model changes still require an explicit per-session switch/reset path.

## What This Changes

The issue asks for a smoother provider/model switch flow after moving from `bailian/qwen3.6-plus` to `deepseek/deepseek-v4-pro`, covering provider config patching, global versus agent model config, allowlist warnings, gateway restart behavior, and active-session model refresh.

## Best Possible Solution

Split the remaining work into maintainer-approved pieces: a safe custom-provider setup path, diagnostics that explain provider/models.json/allowlist mismatches, and explicit current-session switch/reset guidance rather than silently remapping active sessions on default-model config reload.

## Reproduction Assessment

Unclear. The issue includes enough environment and model detail for a plausible manual reproduction, and static current-main inspection confirms several remaining behaviors, but a high-confidence end-to-end reproduction still needs the reporter's exact patch payload and config state.

## Solution Assessment

Unclear. Automatically allowing provider writes, expanding all configured providers into the allowlist, or silently remapping active sessions may not be the safest design; a narrower CLI/status/doctor flow with explicit session switching looks more maintainable.
