## Summary

Keep open. Current main only partially improves the model-switch flow: unmanaged Gateway restart is now covered and models.json generation merges configured providers, but agent-facing config.patch still blocks provider writes, there is no first-class custom provider add command, allowlist behavior still hides non-allowlisted configured provider models, and active sessions still need an explicit model override/reset.

## What This Changes

Improve the model-switching UX when moving from Qwen to DeepSeek across provider setup, agent models.json, allowlist visibility, Gateway restart, and active-session model changes.

## Best Possible Solution

Split the work into maintainer-approved pieces: a safe custom-provider setup path, diagnostics for provider/models.json/allowlist mismatches, and explicit current-session switch/reset UX instead of silent session remapping.

## Reproduction Assessment

Unclear. The report includes enough environment and model detail for a plausible manual reproduction, and static current-main checks confirm several remaining behaviors, but the exact config.patch payload and agent models.json state are not available here.

## Solution Assessment

Unclear. Automatically allowing provider writes, expanding allowlists from every configured provider, or silently changing active-session models is not obviously the safest design; a narrower CLI/doctor/status flow plus explicit session switching looks more maintainable.
