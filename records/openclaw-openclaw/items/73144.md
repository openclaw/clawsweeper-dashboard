## Summary

Keep open. Current main only partially improves the reported model-switch flow: unmanaged Gateway restart now has a fallback path and models.json generation merges configured providers, but agent-facing config.patch still blocks provider writes, there is no first-class custom provider add command, allowlisted model views still hide configured provider models, and active sessions still keep their persisted runtime/override model until explicitly changed.

## What This Changes

The issue asks to smooth model-provider switching by allowing provider setup through config tooling, warning on provider/models.json/allowlist mismatches, fixing restart behavior, and handling active-session model changes after default-model reloads.

## Best Possible Solution

Split the work into maintainer-approved pieces: an explicit safe provider setup path, doctor/status diagnostics for provider/models.json/allowlist mismatches, and clear session switch/reset UX instead of silent active-session remapping.

## Reproduction Assessment

Unclear. Static current-main checks reproduce or confirm several remaining behaviors, but the report does not include the exact config.patch payload or agent models.json state needed for a high-confidence end-to-end qwen-to-deepseek reproduction.

## Solution Assessment

Unclear. Automatically allowing provider writes, auto-expanding allowlists, or silently changing active sessions is not obviously the safest design; explicit setup, diagnostics, and session-switch commands look more maintainable.
