## Summary

Keep open. Current main only partially addresses the reported flow: unmanaged gateway restart has a fallback and generated models registries merge configured providers, but provider setup, allowlist diagnostics, and active-session model semantics still need maintainer design.

## What This Changes

The issue requests a smoother qwen-to-DeepSeek model-switch flow across provider config, per-agent models registry reconciliation, allowlist handling, gateway restart, and active-session model updates.

## Best Possible Solution

Split this into explicit maintainer-approved pieces: a safe provider setup path, doctor/status checks for provider and allowlist mismatches, and clear session switch/reset UX rather than silent active-session remapping.

## Reproduction Assessment

Unclear. Static current-main checks confirm several individual behaviors, but the report does not include the exact config patch payload, model registry state, and live switch sequence needed for a high-confidence end-to-end reproduction.

## Solution Assessment

Unclear. Automatically allowing provider writes, auto-expanding allowlists, or silently remapping active sessions is not obviously the safest design; explicit setup, diagnostics, and user-visible session switching look more maintainable.
