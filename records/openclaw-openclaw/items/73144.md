## Summary

Keep open. Current main has improved several adjacent pieces of the DeepSeek/model-switching path, but it still does not resolve the core UX bundle: the agent-facing gateway config tool still blocks `models.providers.*` writes, there is no first-class `openclaw models add <provider>` command, configured model views still intentionally preserve `agents.defaults.models` allowlist semantics, and active sessions still require an explicit session model switch or reset.

## What This Changes

The issue requests a smoother switch from `bailian/qwen3.6-plus` to `deepseek/deepseek-v4-pro`, covering provider config writes, dual global/agent model config visibility, allowlist diagnostics, gateway restart behavior, and active-session model refresh.

## Best Possible Solution

Keep the already-landed DeepSeek/default-model/restart improvements, then split the remaining UX into maintainer-approved pieces: a safe custom-provider setup path, status or doctor diagnostics for provider/models.json/allowlist mismatches, and an explicit current-session switch or reset flow instead of silent active-session remapping.

## Reproduction Assessment

Unclear. The report gives concrete environment and model details, and static current-main inspection confirms several remaining behaviors, but high-confidence end-to-end reproduction still needs the exact patch payload and the reporter's global plus agent model config state.

## Solution Assessment

Unclear. The requested automatic all-provider allowlist expansion and active-session remapping are not clearly the safest design; a narrower CLI/status/doctor flow with explicit session switching or reset looks more maintainable.
