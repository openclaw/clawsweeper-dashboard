## Summary

Keep open. Current main has fixed or documented several adjacent parts of the DeepSeek/model-switch path, but the core five-part UX report is not fully resolved: the agent-facing gateway config tool still blocks provider writes, the normal configured model view still preserves explicit allowlist semantics, and active sessions still require an explicit per-session model switch.

## What This Changes

The issue requests a smoother switch from Qwen to DeepSeek covering custom provider addition, global versus agent model config visibility, allowlist/default-model consistency, unmanaged gateway restart, and active-session model refresh behavior.

## Best Possible Solution

Keep the already-landed DeepSeek/default-model/restart improvements, then split the remaining UX into explicit maintainer-owned changes: a safe custom-provider setup flow, targeted models status or doctor diagnostics for provider/allowlist mismatches, and a clear explicit session switch/reset path rather than silent remapping of running sessions.

## Reproduction Assessment

Unclear: the report gives concrete models, version, OS, setup, and symptoms, and current source confirms several remaining behaviors by inspection. A full replay still needs the exact patch payload and the user's openclaw.json plus agent models.json state.

## Solution Assessment

Unclear: automatic all-provider allowlist expansion and silent active-session remapping may conflict with intentional scoping and session stability. A narrower CLI/status/doctor design with explicit session switching looks safer and more maintainable.
