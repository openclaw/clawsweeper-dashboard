## Summary

Keep open. Current main only partially addresses the reported switch flow: unmanaged Gateway restart has a fallback and generated `models.json` now merges configured providers, but agent-facing `config.patch` still cannot write `models.providers.*`, there is no `openclaw models add <provider>` command, the configured model view still honors `agents.defaults.models` as an allowlist, and active sessions keep or require explicit session model state instead of silently adopting a changed default.

## What This Changes

The issue requests a smoother model-switch flow for adding a DeepSeek provider, reconciling global and per-agent model registries, diagnosing allowlist mismatches, restarting unmanaged gateways, and updating active sessions after default model changes.

## Best Possible Solution

Split this into maintainer-approved work: an explicit safe provider setup path, doctor/status diagnostics for provider/models.json/allowlist mismatches, and clear session switch/reset UX instead of silent active-session remapping.

## Reproduction Assessment

Unclear. Static current-main checks confirm several individual behaviors, but the issue does not include the exact config patch payload, model registry state, and live switch sequence needed for a high-confidence end-to-end reproduction.

## Solution Assessment

Unclear. Automatically allowing provider writes, auto-expanding allowlists, or silently remapping active sessions is not obviously the safest solution; explicit setup, diagnostics, and session reset/switch commands look more maintainable.
