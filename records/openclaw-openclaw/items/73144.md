## Summary

Keep open. Current main partially improves model catalog generation and documentation, but this broad model-switch workflow still includes unresolved provider-write policy, allowlist/diagnostic UX, managed-service restart expectations, and the stale-session subcase tracked by a narrower open issue and PR.

## What This Changes

The issue asks for a smoother default-model switch workflow covering provider addition, global/agent model catalog alignment, allowlist mismatch diagnostics, gateway restart behavior, and active-session model refresh semantics.

## Best Possible Solution

Adopt an owner-approved model setup path that separates explicit provider/auth registration from agent-facing config.patch, adds model status or doctor diagnostics for provider and allowlist mismatches, documents managed-restart expectations, and lands the narrower stale-session fix through the linked subissue/PR.

## Reproduction Assessment

No, not for the whole five-part flow. Source inspection confirms several current-main behaviors, and the stale-session subcase has a clearer reproduction path in https://github.com/openclaw/openclaw/issues/77322, but the original full qwen-to-deepseek sequence was not live-replayed here.

## Solution Assessment

Unclear. Broad provider writes through config.patch and automatic allowlist/session remapping are not obviously the safest fixes; explicit provider/auth setup plus targeted diagnostics and the narrow session-cache fix is the more maintainable direction.
