## Summary

Keep open. Current main only partially improves this model-switch flow: unmanaged Gateway restart has a fallback path and models.json generation defaults to merge, but agent-facing gateway config.patch still blocks provider writes, there is no first-class custom provider add command, configured model views still prefer the allowlist, and session-scoped model selections remain explicit rather than silently following a changed default.

## What This Changes

Improve the UX for switching from qwen to DeepSeek by reducing manual provider setup, allowlist, restart, and active-session model friction.

## Best Possible Solution

Split this into maintainer-approved pieces: an explicit safe provider setup path, doctor/status diagnostics for provider/models.json/allowlist mismatches, and clear session switch/reset UX instead of silent active-session remapping.

## Reproduction Assessment

Unclear. Static current-main checks confirm several remaining behaviors, but the issue lacks the exact patch payload, agent models.json state, and provider credentials needed for a high-confidence end-to-end reproduction.

## Solution Assessment

Unclear. Allowing agent config.patch to write provider credentials, auto-expanding allowlists, or silently remapping active sessions is not obviously the safest path; explicit setup and diagnostics look more maintainable.
