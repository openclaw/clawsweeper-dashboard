## Summary

Keep open. Current main has improved adjacent DeepSeek setup, provider catalog generation, default-model upserts, and unmanaged gateway restart handling, but the five-part UX report is not fully resolved: the agent-facing gateway config tool still blocks provider writes, configured model views still preserve explicit allowlist semantics, and active sessions still require an explicit per-session model switch/reset.

## What This Changes

The issue asks to reduce friction when switching from a Bailian Qwen default model to DeepSeek by improving provider setup, config/model-list diagnostics, allowlist handling, gateway restart behavior, and active-session model switching.

## Best Possible Solution

Keep the landed DeepSeek/provider-catalog and restart improvements, then split the remaining UX into maintainer-owned changes: a safe custom-provider setup command or guided auth path, targeted models/doctor diagnostics for provider and allowlist mismatches, and an explicit session switch/reset workflow rather than silently remapping running sessions.

## Reproduction Assessment

Unclear: the report gives concrete models, version, OS, setup, and symptoms, and current source confirms several remaining behaviors by inspection. A high-confidence live replay still needs the exact patch payload and the reporter's openclaw.json plus agent models.json state.

## Solution Assessment

Unclear: allowing all provider writes through the agent-facing gateway tool, auto-expanding every provider model into the allowlist, and silently remapping active sessions may conflict with intentional scoping and session stability. A narrower CLI/status/doctor design with explicit session switching looks safer.
