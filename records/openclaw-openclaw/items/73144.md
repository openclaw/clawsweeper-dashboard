## Summary

Keep open. Current main has safer adjacent paths for this flow, including DeepSeek onboarding, `openclaw models set` allowlist updates, models.json merge planning, and unmanaged gateway restart fallback, but the reported agent-driven provider setup, stale registry diagnostics, and active-session default-change semantics still need maintainer product/security direction.

## What This Changes

The issue requests lower-friction model-provider setup, diagnostics for global versus agent model configuration mismatch, safer allowlist behavior, restart reliability, and clearer active-session model switching after default changes.

## Best Possible Solution

Keep this open and split the remaining work into an explicit trusted provider setup path, doctor/status diagnostics for provider and allowlist mismatches, and a user-visible session switch/reset flow for existing sessions.

## Reproduction Assessment

Unclear. Static current-main inspection verifies the relevant guard, allowlist, restart, and session-status behaviors, but the report lacks the exact payload, registry state, and live sequence needed to reproduce the full five-step flow end to end.

## Solution Assessment

Unclear. The broad suggestions to allow provider writes, auto-expand allowlists, or silently remap active sessions are not obviously the safest design; explicit CLI/onboarding setup plus diagnostics and explicit session switching look more maintainable.
