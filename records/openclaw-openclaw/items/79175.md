Full review comments:

- **[P2] Restore beacon index selection:** `src/commands/onboard-remote.ts:91`
  - body: The select option value is now the beacon object, but the handler only accepts non-manual strings and parses them as indexes. Choosing a discovered gateway therefore never sets `selectedBeacon`, so the direct/SSH connection branch below becomes unreachable and users fall through to the manual/default URL prompt. Keep string index values or handle the selected beacon object directly.
  - confidence: 0.96
- **[P2] Use Lit interpolation for translated attributes:** `ui/src/ui/chat/run-controls.ts:28-29`
  - body: These attributes use JSX-style `{t(...)}` syntax inside a Lit template, so the browser receives literal `{t("...")}` text instead of translated labels. The same pattern appears on the attach-file button in `chat.ts`; use `title=${t(...)}` / `aria-label=${t(...)}` for dynamic attributes.
  - confidence: 0.97
- **[P2] Localize welcome suggestions instead of hard-coding Chinese:** `ui/src/ui/chat/chat-welcome.ts:19-23`
  - body: `WELCOME_SUGGESTIONS` is rendered for every locale, so English and all non-Chinese users now see and send Chinese starter prompts from the empty chat screen. Keep English source strings and route the suggestions through the translation map, or otherwise select them by locale.
  - confidence: 0.96
- **[P2] Remove duplicate welcome translation keys:** `ui/src/i18n/locales/en.ts:1052`
  - body: The `welcome` object declares `startTalk` twice here, and the zh-CN locale adds the same duplicate. TypeScript/strict UI checks reject duplicate object-literal properties, so the UI type/build lane will fail until one copy is removed.
  - confidence: 0.86
