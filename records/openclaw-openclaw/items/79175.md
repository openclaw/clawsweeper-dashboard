  - body: This changes discovered gateway option values from string indexes to beacon objects, but the handler still only assigns `selectedBeacon` for non-manual string selections.   - body: `chat.welcome.startTalk` is defined twice in the locale object. TypeScript rejects duplicate object-literal properties, so keep one entry in each changed locale map before this can pass UI type/build checks.
  - confidence: 0.96
- **[P2] Keep welcome suggestions locale-neutral:** `ui/src/ui/chat/chat-welcome.ts:20-23`
  - body: `WELCOME_SUGGESTIONS` is now a Chinese literal array, so English and every other locale would render Chinese quick prompts. Keep the source strings English or move the suggestions into the locale map before translating them.
  - confidence: 0.97
- **[P2] Use Lit expression bindings for translated attributes:** `ui/src/ui/chat/run-controls.ts:28-29`
  - body: In Lit templates, `title={t(...)}` and `aria-label={t(...)}` are literal attribute text, not expression bindings. Use `${...}` bindings here and in the matching changed chat toolbar attributes so tooltips and assistive labels localize correctly.
  - confidence: 0.95
- **[P2] Preserve assistant-specific chat placeholders:** `ui/src/ui/views/chat.ts:811`
  - body: The current placeholder includes the active assistant name, but this replacement hard-codes the generic locale string to OpenClaw. Keep the assistant name as a translation parameter so multi-agent chats do not lose their context.
  - confidence: 0.92
