  - body: This changes discovered gateway option values to beacon objects, but the handler only sets `selectedBeacon` for string selections. The prompter returns `option.value` unchanged, so choosing a discovered gateway leaves `selectedBeacon` null and falls through to manual URL entry.
  - confidence: 0.98

  - body: The PR adds `startTalk` twice inside `chat.welcome`. TypeScript rejects duplicate object-literal properties in `.ts` files, so keep one entry in each locale map before this can pass the UI type/build checks.
  - confidence: 0.94
- **[P2] Keep welcome suggestions locale-neutral:** `ui/src/ui/chat/chat-welcome.ts:20-23`
  - body: `WELCOME_SUGGESTIONS` is now a Chinese literal array, so English and all other locales would show Chinese quick prompts. Keep English source strings or move the suggestions into the locale map before translating them.
  - confidence: 0.96
- **[P2] Use Lit bindings for translated attributes:** `ui/src/ui/chat/run-controls.ts:28-29`
  - body: In Lit templates, `title={t(...)}` and `aria-label={t(...)}` render as literal attribute text, not translated values. Use `${...}` bindings here and for the matching chat toolbar attributes so tooltips and assistive labels localize correctly.
  - confidence: 0.95
- **[P2] Restore the removed changelog entry:** `CHANGELOG.md:9`
  - body: The PR replaces the existing Active Memory unreleased bullet from current main instead of adding this i18n entry beside it. Merging as-is would drop another user-facing release note.
  - confidence: 0.96

  - body: The changelog says the zh-TW glossary expands from 19 to 116 entries, but the PR-head JSON contains 105 entries. Correct the count or remove it so the release note does not overstate the shipped scope.
