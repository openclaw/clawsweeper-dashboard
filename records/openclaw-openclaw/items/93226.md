## Best Possible Solution

Repair status and expiry selection together in both rollups, add focused mixed-profile coverage, and keep the related refresh-path PR as the separate cache-refresh fix if maintainers want both changes.

## Reproduction Assessment

Yes at source level: current main prioritizes expired OAuth profiles before ok in both provider rollups, while the linked report describes usable runtime auth alongside stale expired inventory. I did not run a live Control UI reproduction.

## Solution Assessment

No as currently written. Preferring usable OAuth may be the right direction, but the PR must select status and expiry from the same credential set and needs maintainer acceptance for the changed auth-provider status contract.
