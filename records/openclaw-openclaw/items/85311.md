## Summary

## What This Changes

The PR prepends the agent default model provider's auth profile order when resolving an auto-selected session auth profile override.

## Best Possible Solution

Fix stale auto fallback model and auth state together in the normal session/model selection and inheritance paths, with regression coverage and redacted real runtime proof.

## Reproduction Assessment

Yes, source-reproducible. The PR defect follows from the diff plus current callers: the resolver can return an agent-primary auth profile while the run provider/model remain on the fallback; I did not run a live MiniMax/DeepSeek setup in this read-only pass.

## Solution Assessment

No. Reordering provider-specific auth profiles inside an auth-only helper is not the best fix; the selected provider/model and auth profile need to recover together in session/model selection.
