## Summary

## What This Changes

The branch prepends the agent default model provider's auth profile order when `resolveSessionAuthProfileOverride` handles an auto-selected session auth override.

## Best Possible Solution

Recover stale auto fallback provider, model, display, and auth state together in the session/model selection layer, keep the auth resolver provider-scoped, and require focused regression coverage plus redacted real behavior proof.

## Reproduction Assessment

Yes, source-reproducible. The diff can make the resolver return an agent-primary auth profile while the caller continues with the already-selected fallback provider/model; I did not run a live MiniMax/DeepSeek setup in this read-only review.

## Solution Assessment

No. Reordering provider-specific auth profiles inside an auth-only helper is not the best fix; stale provider, model, display, and auth state need to recover together in the session/model selection path.
