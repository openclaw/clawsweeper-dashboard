## Summary

Keep open, but not merge-ready: the linked bug is real, yet the auth-only fix can decouple credentials from the active provider/model and still lacks real behavior proof.

## What This Changes

The PR prepends the agent default model provider's auth profile order when auto session auth override resolution handles a fallback provider.

## Best Possible Solution

Recover stale auto fallback provider, model, display, and auth state together in the session/model selection layer, keep the auth resolver provider-scoped, and require focused regression coverage plus redacted real behavior proof.

## Reproduction Assessment

Yes, source-reproducible. `get-reply-run.ts` passes the already-selected provider into `resolveSessionAuthProfileOverride` and receives only an auth profile back, so the PR's new cross-provider order can return an incompatible profile without changing the run provider/model.

## Solution Assessment

No. The PR is a plausible symptom fix, but not the best owner-boundary fix because provider, model, display, and auth need to recover together in session/model selection instead of broadening a provider-scoped auth helper.
