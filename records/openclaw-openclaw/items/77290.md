## Summary

Keep open: the PR is a useful docs-only cleanup, but two added comments contradict the current plugin entrypoint and loader/registry ownership boundaries.

## What This Changes

The PR adds module-level JSDoc and explanatory comments to `src/plugins/api-builder.ts` and `src/plugins/registry.ts`.

## Best Possible Solution

Revise the comments so they describe `register(api)` with legacy `activate(api)` compatibility, and frame `registry.ts` as the registration/API/state hub while leaving module loading and activation with the loader.
