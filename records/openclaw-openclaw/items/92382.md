## Summary

The streaming split fixes the original blank-Webchat symptom, but the latest head still does not clear the browser’s first-attempt draft when a finalize hook requests revision. Gateway clears only its internal chat buffer; the replace-only `agent` event is ignored by Control UI assistant handling, and no `chat` replacement delta is emitted because the event has no text. The PR also still needs inspectable real behavior proof for the hook-revision path.

## What This Changes

The PR streams gateway assistant deltas immediately while finalize hooks are active, continues deferring channel partial replies, and emits a replace-only assistant event when a hook requests revision.

## Best Possible Solution

Keep immediate gateway streaming and deferred channel previews, but project a replace-only `chat` delta that clears active Webchat state before the retry, with runner-to-Gateway-to-Control-UI regression coverage and live revision proof.

## Reproduction Assessment

Yes. Source inspection provides a high-confidence reproduction: a finalize hook can reject a streamed first attempt and continue another pass with the same run, while the current replacement event does not clear Control UI chat state.

## Solution Assessment

No, not yet. Splitting gateway streaming from channel partial replies is the correct boundary, but the revision reset must use the established client-visible chat replacement path instead of only clearing Gateway state and broadcasting an ignored assistant agent event.
