## Summary

This PR fixes a real current-main regression, but the patch is not merge-ready because it makes rejected pre-final text visible to webchat before `before_agent_finalize` can revise or suppress it, and the supplied proof does not demonstrate that risky path.

## What This Changes

The branch changes embedded-agent delivery so gateway assistant stream events emit immediately while channel partial replies remain buffered until the `before_agent_finalize` terminal gate resolves.

## Best Possible Solution

Introduce explicit replaceable preview semantics for gateway chat—either clear/replace streamed text when the terminal gate revises or suppresses it, or narrow immediate streaming to a path that can guarantee retraction—while keeping durable channel delivery gated.

## Reproduction Assessment

Yes. Current main deterministically buffers assistant stream events whenever `onBeforeTerminalDelivery` is configured, matching the reported blank-until-terminal webchat behavior.

## Solution Assessment

No. Splitting gateway and channel delivery is directionally sensible, but the proposed implementation exposes rejected text without adding the replacement semantics required by the hook contract.
