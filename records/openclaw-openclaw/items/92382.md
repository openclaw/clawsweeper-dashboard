## Summary

This is a valid webchat streaming regression, but the patch exposes provisional assistant text before `before_agent_finalize` can revise or suppress it; the existing chat replacement path should reconcile those outcomes before merge, and inspectable real behavior proof is still missing.

## What This Changes

The PR emits gateway assistant stream events immediately during `before_agent_finalize` while continuing to defer channel partial replies and terminal block replies.

## Best Possible Solution

Stream provisional gateway text, then use the existing replacement-delta semantics to clear or replace it when finalize revises or suppresses an attempt, with agent, gateway, and Control UI coverage.

## Reproduction Assessment

Yes. Current main's source and focused harness show that registering the finalize gate buffers gateway assistant events until terminal resolution, which explains the reported frozen-then-burst webchat behavior.

## Solution Assessment

No, not yet. Splitting gateway and channel delivery is directionally correct, but the established replacement-delta path should reconcile provisional text for revise and suppress outcomes rather than knowingly leaving rejected output visible.
