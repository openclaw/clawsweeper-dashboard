## Summary

The latest head fixes the earlier subscription-state lifetime mistake, but the revision reset still appears server-local: Webchat clients that already rendered the rejected attempt may not receive an empty replacement before the retry. The original streaming bug is credible and the split boundary is directionally sound, so this PR should remain open for a focused projection repair and stronger real behavior proof.

## What This Changes

The PR streams gateway assistant deltas immediately when `before_agent_finalize` hooks are active, keeps channel partial replies deferred, and emits an assistant replacement event when a hook requests revision.

## Best Possible Solution

Keep gateway text streaming immediate and channel partial replies gated, but make hook revision emit a documented client-visible empty chat replacement that clears both gateway and Webchat state before the next attempt, with production-shaped two-attempt coverage.

## Reproduction Assessment

Yes at source level: current main gates assistant stream events whenever the finalize hook enables deferred terminal delivery, matching the reported blank-then-burst Webchat behavior. The contributor reports a manual smoke test, but no inspectable artifact establishes a high-confidence real reproduction or the revision case.

## Solution Assessment

No, not yet. Separating ephemeral gateway streaming from channel partial replies is the right ownership direction, but the current empty reset must be projected to the browser and tested through the actual retry/Gateway/UI composition.
