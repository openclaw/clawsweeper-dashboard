## Summary

Keep open: the reported webchat streaming regression is credible, but this patch bypasses the intentional pre-delivery hook boundary and can expose an answer that a finalize hook subsequently suppresses or revises; inspectable real-behavior proof is also still missing.

## What This Changes

The PR makes gateway assistant deltas stream before `before_agent_finalize` resolves while continuing to defer channel partial replies and terminal block replies.

## Best Possible Solution

Preserve the finalize hook's pre-delivery guarantee and restore progressive webchat output through a distinct provisional/retractable preview event, or merge a contract change only after explicit maintainer approval, documentation, compatibility tests, and real proof for continue, suppress, and revise outcomes.

## Reproduction Assessment

Yes at source level: installing `onBeforeTerminalDelivery` currently causes all assistant gateway events to be queued until the terminal decision, preventing progressive webchat output. The contributor's reported timing and live environment were not independently reproduced.

## Solution Assessment

No. The patch restores streaming by emitting normal assistant events before a decision that is specifically intended to suppress or revise delivery; a provisional and retractable stream is the safer maintainable boundary.
