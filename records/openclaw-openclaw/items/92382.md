## Summary

Current main has a source-reproducible regression: merely registering a `before_agent_finalize` hook defers all gateway assistant events, so Control UI text arrives only after finalization. The PR’s split matches the existing architecture—Codex and embedded runs produce assistant deltas before completion, gateway chat treats them as incremental preview snapshots, and durable block/channel delivery remains gated—but the external contributor has not supplied inspectable after-fix real behavior evidence, so it should remain open pending proof rather than close or enter automated repair.

## What This Changes

The PR emits embedded-agent gateway assistant stream events immediately while continuing to defer channel partial replies and block replies until the `before_agent_finalize` terminal gate resolves.

## Best Possible Solution

Keep gateway assistant deltas as immediate replaceable previews, continue gating durable block and channel partial delivery, and require a redacted Control UI recording or equivalent live artifact showing incremental streaming plus the final result after a hook-driven revision.

## Reproduction Assessment

Yes from source inspection: on current main, registering the terminal callback causes every assistant gateway event to be queued until finalization, which removes incremental Control UI streaming. I did not run the user’s live environment because this review is read-only.

## Solution Assessment

Yes, conditionally: separating ephemeral gateway previews from durable channel and block delivery is the narrowest owner-aligned fix and matches both gateway and Codex event lifecycles. Inspectable live proof and explicit maintainer acceptance of preview disclosure are still required before merge.
