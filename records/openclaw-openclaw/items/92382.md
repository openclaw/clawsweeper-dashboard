## Summary

The original streaming regression is real, but this patch introduces a blocking revision-path bug: provisional gateway text from an answer rejected by `before_agent_finalize` remains in the same client-run buffer and is merged with the revised answer. The PR also lacks inspectable real-environment evidence for accept, revise, and suppress outcomes.

## What This Changes

The PR emits gateway assistant events immediately while continuing to defer channel partial replies and terminal block replies until `before_agent_finalize` resolves.

## Best Possible Solution

Introduce an explicit gateway and Control UI reset/replacement boundary for provisional text when a finalize hook revises or suppresses an attempt, while preserving immediate streaming for accepted attempts and deferred channel partial replies.

## Reproduction Assessment

Yes. Current main unambiguously queues all assistant gateway events whenever terminal delivery is gated, reproducing the reported lack of incremental webchat output from source; the proposed same-run revision regression is likewise source-reproducible.

## Solution Assessment

No. Separating gateway and channel delivery is directionally appropriate, but immediate gateway emission is unsafe without explicit replacement or retraction semantics for revise and suppress outcomes.
