## Summary

Keep open. Current main still has a caller-side 10s Gateway timeout around a final-result send RPC, and WhatsApp still lacks the adapter reconciler needed to turn an ambiguous timed-out send into a later delivered status.

## What This Changes

The issue reports WhatsApp dispatch returning an unknown timeout outcome even though the outbound WhatsApp message is later delivered.

## Best Possible Solution

Define a maintainer-approved deferred WhatsApp delivery-status contract, then implement adapter-backed reconciliation or an explicit queued/accepted outcome with live WhatsApp proof.

## Reproduction Assessment

Yes for a source-level path: current main still combines a 10s gateway transport timeout, a final-result send RPC, and WhatsApp lacking the unknown-send reconciler that core requires for later reconciliation. Live WhatsApp timing was not reproduced here.

## Solution Assessment

Unclear until maintainers choose the delivery-status contract. A WhatsApp adapter reconciler or explicit queued outcome looks safer than only increasing the gateway timeout.
