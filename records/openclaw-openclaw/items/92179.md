Full review comments:

- **[P1] Keep verbose-off from rendering tool progress:** `src/tui/tui-event-handlers.ts:738-742`
  - body: The existing TUI contract still treats `/verbose off` as hiding tool UI: the command clears tools and history only rebuilds them when verbose is not off. This new gate ignores `state.sessionInfo.verboseLevel` for displayable bash/apply_patch progress, so users with verbose off will see live tool rows that then disappear after a history reload. Please preserve the setting or add an explicit maintainer-approved progress mode with matching tests/docs.
  - confidence: 0.89
- **[P2] Clear assistant separators with hidden tool rows:** `src/tui/components/chat-log.ts:113-118`
  - body: `addAssistantSeparatorIfNeeded()` adds a non-tool separator after a tool row, but `/verbose off` still calls `clearTools()`, which only removes components in `toolById`. After a tool followed by an assistant reply, hiding tools leaves a stray `回答` separator in the chat log. Track and remove these separators with the tool UI or avoid adding them in a path that can be hidden independently.
  - confidence: 0.84
