## Summary

Keep this PR open, but it is not merge-ready: the TUI progress idea is useful, yet the patch bypasses `/verbose off`, adds non-English user-facing labels, does not consume the real command-output event stream, and lacks after-fix real TUI proof.

## What This Changes

The branch changes TUI chat-log/tool rendering to classify selected tool calls as control-plane progress, surface output with redaction/fence stripping, and separate progress rows from final assistant replies.

## Best Possible Solution

Preserve `/verbose off` semantics or add an explicit mode, render actual typed `command_output` events, keep TUI labels in English, and require real terminal proof before merge.

## Reproduction Assessment

Yes, source-reproducible: a verbose-off active run with a matched bash command reaches `startTool`, and current live command output is emitted on `command_output`, which the TUI handler does not consume. I did not run tests because this review is read-only.

## Solution Assessment

No. The current diff is not the best fix because it classifies shell command strings in the renderer and bypasses existing verbosity semantics; a safer path is to consume typed item/command-output events while preserving user preferences.
