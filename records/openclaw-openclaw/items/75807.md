## Summary

Keep this PR open. The snapshot direction is useful, but the latest exact head still has a P2 regression in channel auto-enable CI, and the snapshot generator still cannot remove stale generated artifacts that its check path rejects.

## What This Changes

This PR adds generator-backed Codex happy-path prompt snapshots, scripts/docs/tests, a Codex prompt-snapshot test API, TTS silence-guidance cleanup, and related plugin metadata/package-boundary fixes.

## Best Possible Solution

Keep the snapshot approach, but preserve externalized bundled channel metadata for auto-enable decisions, make snapshot generation self-cleaning, and rerun the prompt snapshot plus changed-lane checks before merge.

## Reproduction Assessment

Yes. The latest exact-head GitHub check annotations reproduce the channel auto-enable regression in `src/config/plugin-auto-enable.channels.test.ts`, and source inspection shows the stale-file generator gap.

## Solution Assessment

No, not as-is. Generator-backed prompt snapshots are a maintainable solution, but the current implementation regresses channel metadata behavior and leaves the documented regeneration command unable to clean stale artifacts.
