## Summary

Keep this open. Current main and the latest release still write resolved host-specific default workspace paths into config, and the open closing PR is the right active fix path rather than a reason to close the issue before merge.

## What This Changes

The issue reports that OpenClaw can persist a host-specific default workspace path such as `/Users/wangtao/.openclaw` instead of a portable `~/.openclaw` workspace.

## Best Possible Solution

Land a focused write-time canonicalization that stores recognized default workspaces as `~/.openclaw/workspace`, preserves explicit custom absolute paths, and keeps runtime workspace resolution unchanged.

## Reproduction Assessment

Yes, at source level. Current main resolves the default workspace to a host-specific path and setup/onboarding/configure write paths persist that value without canonicalizing it back to the documented portable default.

## Solution Assessment

Yes. The narrow maintainable fix is write-time canonicalization for recognized default workspace values, with custom absolute workspaces preserved and regression coverage for setup/onboard/configure persistence.
