## Summary

Keep this open. Current main and the latest release still persist resolved default workspace paths into config, while the linked closing PR is open and currently conflicting rather than merged.

## What This Changes

The issue asks OpenClaw to stop baking host-specific absolute default workspace paths into `agents.defaults.workspace`.

## Best Possible Solution

Land or replace the linked closing PR with a narrow write-boundary fix that stores recognized default workspaces as portable `~/.openclaw/...` values while preserving explicit custom absolute workspaces.

## Reproduction Assessment

Yes. Source inspection shows current main resolves default workspace inputs and persists the resolved string into `agents.defaults.workspace`; I did not run the interactive setup flow because this review is read-only.

## Solution Assessment

Yes. The narrow maintainable fix is to canonicalize only recognized default workspace paths at config-write boundaries while preserving explicit custom absolute workspaces; the linked PR targets that boundary but must be updated and merged first.
