## Summary

Keep this open. Current main still does not contain a literal `wangtao` hardcode, but the actionable bug remains: default workspace values are resolved to host-absolute paths before being persisted into config, and the directly related #56810 fix PR is closed and unmerged.

## What This Changes

The issue requests that OpenClaw stop persisting or using a host-specific `/Users/wangtao/.openclaw` workspace path and use the current user's default `~/.openclaw` workspace instead.

## Best Possible Solution

Land a focused config-serialization fix that stores canonical default workspaces as portable `~/.openclaw/workspace` or profile-aware `~/.openclaw/workspace-<profile>` values, while continuing to resolve absolute paths only at runtime and leaving explicit custom paths unchanged.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: follow the default setup/onboard/configure path and inspect the written `agents.defaults.workspace`; current code resolves the default before persisting it.

## Solution Assessment

Yes. The narrowest maintainable fix is to canonicalize only default workspace values at config write time, since runtime already supports `~` through `resolveUserPath()` and does not need a broader migration or prompt/runtime rewrite.
