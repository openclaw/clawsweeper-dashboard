## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/79872 or an equivalent narrow repair that canonicalizes recognized default workspace paths to portable `~/.openclaw/...` values at config-write boundaries while preserving explicit custom absolute workspaces.
