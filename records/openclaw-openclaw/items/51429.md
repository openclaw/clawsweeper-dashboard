## Summary

Keep this open. Current main still has no literal `wangtao` hardcode, but the actionable config portability bug remains: default workspace values are resolved to host-absolute paths before being persisted, and the directly related fix PR was closed unmerged.

## What This Changes

The issue asks OpenClaw to stop persisting or reporting another host's absolute default workspace path and keep the default workspace portable as `~/.openclaw/workspace`.

## Best Possible Solution

Land a focused config-serialization fix that stores canonical default workspaces as portable `~/.openclaw/workspace` or profile-aware `~/.openclaw/workspace-<profile>` values, while resolving to absolute paths only at runtime and leaving custom paths unchanged.

Add or update focused regression tests and a user-facing `CHANGELOG.md` fix entry.
