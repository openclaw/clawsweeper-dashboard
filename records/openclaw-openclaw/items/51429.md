## Summary

Keep this open. Current main still has no literal `wangtao` hardcode, but the actionable config-portability bug remains: default workspace values are resolved to host-absolute paths before being persisted, and the directly related fix PR #56810 was closed unmerged.

## What This Changes

The issue reports OpenClaw creating or reporting `/Users/wangtao/.openclaw` instead of the user's expected `~/.openclaw` workspace after onboarding.

## Best Possible Solution

Add a focused config-serialization helper that stores canonical default workspaces as portable `~/.openclaw/workspace` or profile-specific variants while preserving custom paths and resolving to absolute paths only at runtime.

## Solution Assessment
