## Summary

Keep this open. Current main has no literal `wangtao` hardcode, but the actionable bug remains: default workspace values are resolved to host-absolute paths before being persisted, while docs describe the default config value as portable `~/.openclaw/workspace`.

## What This Changes

The issue reports that onboarding/config persisted a host-specific absolute workspace path, making OpenClaw appear to use `/Users/wangtao/.openclaw` instead of a portable default workspace.

## Best Possible Solution

Add a focused config serialization helper that stores recognized default workspaces as portable `~/.openclaw/workspace` or profile-aware `~/.openclaw/workspace-<profile>`, resolves to absolute paths only at runtime, leaves custom paths unchanged, and adds regression coverage for setup/onboard/configure flows.

## Reproduction Assessment

Yes, at source level: follow setup, onboarding, migration import, or configure with the default workspace and inspect `agents.defaults.workspace`; current main passes a resolved absolute path into config persistence. I did not run a live onboarding command because this review is read-only.

## Solution Assessment

Yes. The narrowest maintainable fix is write-time canonicalization for recognized default workspace paths, with runtime path resolution unchanged and no new config option or core API.
