## Summary

Keep this open. Current main still has the source-level config serialization bug, and an open closing PR now targets the fix, so the issue should wait for that PR to be reviewed and landed rather than be closed early.

## What This Changes

The issue requests fixing config writes that bake machine-specific default workspace paths into `agents.defaults.workspace` instead of preserving a portable default workspace value.

## Best Possible Solution

Land a focused write-time canonicalization that stores recognized default workspaces as portable `~/.openclaw/workspace` values, preserves custom paths, and keeps runtime path resolution unchanged.

## Reproduction Assessment

Yes, at source level: the default workspace resolves to a host-specific path and setup/onboarding/configure write that value into config. I did not run a live onboarding flow because this review is read-only.

## Solution Assessment

Yes. The narrowest maintainable fix is write-time canonicalization for recognized default workspace paths, and the open closing PR follows that direction pending proof and normal review.
