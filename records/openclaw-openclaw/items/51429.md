- `impact:session-state`: The persisted workspace path controls the agent working directory and can make session/context state point at the wrong host-specific location.

## Summary

Keep this open. Current main and the latest release still persist resolved host-specific default workspace paths into config, and an open closing PR already targets the narrow portability fix.

## What This Changes

The issue asks OpenClaw to stop saving the default workspace as a machine-specific absolute path that can later appear as a hardcoded foreign home directory.

## Best Possible Solution

Land a focused write-time canonicalization that stores recognized default workspaces as `~/.openclaw/workspace`, preserves explicit custom absolute paths, and keeps runtime workspace resolution unchanged.

## Reproduction Assessment

Yes, at source level. Current main resolves the default workspace to a host-specific absolute path and setup/onboarding/configure write paths persist that value without canonicalizing it back to the documented portable default.

## Solution Assessment

Yes. The narrow maintainable fix is write-time canonicalization for recognized default workspace values, with custom absolute workspaces preserved and regression tests for setup/onboard/configure persistence.
