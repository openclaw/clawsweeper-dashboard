## Summary

## What This Changes

The PR adds TLS fingerprint selection to the shared operator approvals Gateway client and focused tests for local WSS, env override, and CLI override cases.

## Best Possible Solution

Land a rebased shared approval-client TLS fix that mirrors callGateway precedence, preserves local fallback TLS, guards remote pinning to actual remote or env-targeted cases, and includes focused regression coverage plus real runtime proof.

## Reproduction Assessment

Yes, source-level. Current main routes approval resolution through createOperatorApprovalsGatewayClient without tlsFingerprint, while GatewayClient requires that option for pinned local self-signed WSS handling; I did not run a live gateway reproduction in this read-only review.

## Solution Assessment

No, not as currently written. The direction is right, but the PR should guard remote fingerprint use on an actual remote target or env override, preserve local fallback behavior, and be rebased before merge.
