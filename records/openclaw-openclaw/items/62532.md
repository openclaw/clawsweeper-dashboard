## Summary

Keep open: the underlying operator-approval TLS bug still exists on current main, but this PR head applies remote TLS pinning too broadly and only has mocked/unit proof, so it should not merge as-is.

## What This Changes

The PR adds TLS fingerprint selection and focused tests for the shared operator approvals Gateway client.

## Best Possible Solution

Land a rebased shared approval-client fix that passes the local runtime fingerprint for local WSS and the remote fingerprint only for real remote/env cases, with coverage for the remote-missing local fallback.

## Reproduction Assessment

Yes, at source level. Current main routes approval resolution through `createOperatorApprovalsGatewayClient`, which omits `tlsFingerprint`, while `GatewayClient` requires that option to accept a pinned local self-signed WSS certificate.

## Solution Assessment

No, not as currently written. The direction is right, but the PR should guard remote fingerprint use on an actual remote target or env override, preserve local fallback behavior, and be reconciled with current main before merge.
