Request reliable Telegram forum topic-title to topic_id resolution or catalog/list support so agents can route messages by human-readable topic names.

## Best Possible Solution

## Solution Assessment

Unclear until maintainers choose the product/API surface. The current reverse topic-name cache is useful, but it is not the requested routing-by-title resolver and should not be stretched into silent best-effort sends without ambiguity rules.
