## Summary

Keep open. Current main still provides only partial Telegram forum-topic coverage: agents can see the current topic's numeric ID and observed name, but there is still no topic-name-to-topic_id resolver, topic catalog, or list action for routing by title.

## What This Changes

Request first-class Telegram forum topic name-to-topic_id resolution or a topic catalog/list surface for agent routing.

## Best Possible Solution

Add a Telegram-owned observed topic catalog or resolver with explicit ambiguity and staleness handling, plus a safe clarification path when a title cannot be resolved.

## Reproduction Assessment

Yes, as a source-level capability gap rather than a broken existing contract. Current main exposes only current-topic topic_id/topic_name and numeric thread targeting, while source inspection shows no topic catalog, list action, or name-to-ID resolver.

## Solution Assessment

No, the current partial topic-name cache is not the full solution. The maintainable path needs a deliberate Telegram channel/tool design for observed topic names, ambiguity, stale data, and safe fallback behavior.
