## Summary

Keep open. Current main partially mitigates the problem through the merged topic-name cache from https://github.com/openclaw/openclaw/pull/65973, but it still lacks the requested forum topic catalog, list action, or name-to-topic-id resolver for deterministic routing by human-readable topic name.

## What This Changes

The issue requests first-class Telegram forum topic-name to topic-id resolution for agent routing.

## Best Possible Solution

Add a Telegram-owned observed topic catalog surface with clear partial-cache semantics, exposed either in agent context or as a list/resolve action, without pretending the Bot API can enumerate every topic on demand.

## Reproduction Assessment

No as a strict bug reproduction; this is a missing feature/API surface. Source inspection on current main shows current-topic topic_id/topic_name exposure and numeric topic targets, but no forum_topics map, list action, or name-to-id resolver.

## Solution Assessment

No, the current partial topic-name cache is not the full solution for this issue. The narrow maintainable path is a deliberate Telegram channel/API design around the existing observed cache, including how agents handle unknown or stale topic names.
