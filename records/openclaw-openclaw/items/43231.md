## Summary

Keep open. Current main and v2026.6.8 expose the current Telegram topic id and a cached id-to-name value, but they still do not expose a reliable topic-name-to-topic-id catalog or list action for routing to another forum topic by human-readable name.

## What This Changes

The issue requests first-class Telegram forum topic-name to topic_id resolution for agent routing, via a forum_topics context map, a topic-list message action, or a channel metadata endpoint.

## Best Possible Solution

Define a Telegram-owned topic catalog capability that reuses the existing cached chatId:threadId-to-name store and exposes a scoped list or metadata surface to agents/tools, with numeric topic targets kept as the reliable fallback.

## Reproduction Assessment

Yes for source-level verification: current main exposes only current topic_id/topic_name metadata and numeric topic targets, while Telegram actions lack a topic-list or name-resolution surface. I did not run a live Telegram forum because this read-only review has no forum credentials.

## Solution Assessment

Unclear until maintainers choose the public API shape. A Telegram-plugin catalog/list surface built on the existing cache is the narrowest likely direction, while the merged reverse-mapping PR only partially addresses the request.
