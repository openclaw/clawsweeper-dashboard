## Summary

Keep open. Current main has useful partial coverage from the merged Telegram topic-name cache, but it still exposes only current-topic `topic_id`/`topic_name` and numeric thread targeting, not a reliable topic-name-to-topic_id resolver or topic catalog/list surface for agent routing.

## What This Changes

Request first-class Telegram forum topic name to topic_id resolution, topic listing, or topic catalog metadata for agent routing.

## Best Possible Solution

Design a Telegram-owned observed topic catalog or resolver with explicit partial-cache semantics, then expose it through the chosen agent/tool surface without requiring manual topic maps.

## Reproduction Assessment

Yes as a source-level capability gap, not as a strict broken-contract bug. Current main exposes current-topic `topic_id`/`topic_name` and numeric thread targeting, while source inspection shows no topic catalog, list action, or name-to-ID resolver.

## Solution Assessment

No, the current partial topic-name cache is not the full solution for this issue. The maintainable path needs a deliberate Telegram channel/API design for observed topic names, ambiguity, stale data, and safe fallback behavior.
