## Summary

Keep open. Current main has useful Telegram forum topic-name learning and persistence, but it still only exposes the current topic name next to the current numeric topic id; it does not provide a forum topic catalog, a topic-name resolver, or name-based outbound routing to another topic.

## What This Changes

Request first-class Telegram forum support to resolve human-readable topic names to numeric topic IDs for reliable agent routing.

## Best Possible Solution

Implement the remaining capability in the bundled Telegram plugin: reuse the persisted topic-name cache, expose an agent-visible observed-topic catalog or thread/topic resolve action, and allow outbound sends only on exact unique non-closed matches with clarification fallback for ambiguity or missing data.

## Reproduction Assessment

Yes. The code-level reproduction is high-confidence: ask an agent to route to another Telegram forum topic by name, and current main has no topic-name resolver or list action while target parsing remains numeric-only.

## Solution Assessment

Yes, the best direction is a narrow Telegram-owned implementation built on the existing persisted topic-name cache. It should avoid fuzzy matching and should not claim authoritative coverage beyond observed or persisted topic metadata.
