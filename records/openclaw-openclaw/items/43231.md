## Summary

Keep open. Current main has useful reverse topic-name context and persistence, but it still does not expose a deterministic Telegram forum topic catalog, list action, or topic-name-to-topic_id resolver for agent routing.

## What This Changes

The issue requests first-class Telegram forum topic title to topic_id resolution so agents can route messages by human-readable topic name.

## Best Possible Solution

Add a Telegram-owned observed topic catalog or resolver with safe unknown/ambiguous-name behavior and expose it through the chosen agent/tool surface without requiring manual topic maps.

## Reproduction Assessment

No live Telegram reproduction was run, but source inspection gives a high-confidence source reproduction: current main exposes only current-topic topic_id/topic_name and numeric topic targets, with no forum_topics map or topic title resolver.

## Solution Assessment

No, the existing topic-name cache is only a partial solution. The maintainable fix needs a deliberate Telegram channel/API design around observed topic data and safe fallback semantics.
