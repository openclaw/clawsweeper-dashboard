## Summary

Keep open. Current main has useful Telegram forum topic-name learning and persistence, but it still exposes only the current topic id/name pair and has no topic catalog, name resolver, or name-based outbound routing for another topic.

## What This Changes

Request first-class Telegram forum topic-name to topic_id resolution or catalog support so agents can route messages by human-readable forum topic name.

## Best Possible Solution

Implement the remaining capability inside the bundled Telegram plugin: expose an observed-topic catalog or resolve action backed by the persisted topic-name cache, and allow routing by name only for exact unique non-closed matches with cache-miss and ambiguity fallbacks.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: ask an agent in a Telegram forum group to route to another topic by name, and current main has no topic-name resolver or topic-list action while outbound target parsing and threadId params are numeric-only.

## Solution Assessment

Yes. The narrow maintainable direction is Telegram-owned support built on the existing persisted topic-name cache, avoiding fuzzy matching and not claiming complete coverage beyond observed or persisted topic metadata.
