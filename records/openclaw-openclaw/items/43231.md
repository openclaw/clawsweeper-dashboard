## What This Changes

The issue asks for first-class Telegram forum topic title to topic ID resolution for agent routing.

## Best Possible Solution

Add a Telegram-owned observed topic catalog or resolver with explicit freshness, ambiguity, closed-topic, permission, and no-send-on-uncertain-match semantics exposed through the maintainer-chosen tool, context, or metadata surface.

## Reproduction Assessment

Yes, as a source-level capability gap. Current main exposes current-topic `topic_id` and `topic_name` plus numeric thread targeting, while the code has no topic catalog, list action, or name-to-ID resolver.

## Solution Assessment

Unclear until maintainers choose the product/API surface. The existing reverse topic-name cache is useful, but it should not be stretched into silent best-effort sends without explicit ambiguity and freshness rules.
