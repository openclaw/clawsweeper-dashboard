## Summary

Keep open. Current main has useful partial coverage from the merged topic-name cache, including persisted `topic_id -> topic_name` context, but it still does not provide the requested topic catalog, list action, or deterministic topic-name-to-topic_id resolver for Telegram forum routing.

## What This Changes

The issue requests first-class Telegram forum support for resolving human-readable topic names to numeric topic IDs for agent message routing.

## Best Possible Solution

Add a Telegram-owned observed topic catalog or resolver with clear partial-cache semantics, then expose it through the chosen agent/tool surface without requiring manual topic maps.

## Reproduction Assessment

No as a strict bug reproduction; yes as a source-level capability gap. Current main exposes current-topic `topic_id`/`topic_name` and numeric thread targeting, but source inspection shows no topic catalog, list action, or name-to-ID resolver.

## Solution Assessment

No, the current partial topic-name cache is not the full solution for this issue. The maintainable path is a deliberate Telegram channel/API design around the observed cache, including stale, unknown, duplicate, and renamed topic behavior.
