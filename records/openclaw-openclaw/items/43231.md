## Summary

Keep open. Current main has numeric Telegram topic routing plus current-topic topic_id/topic_name exposure, but it still lacks a first-class title-to-topic_id resolver, topic catalog, or topic-list action for routing by human-readable forum topic name.

## What This Changes

The issue requests reliable Telegram forum topic-name to topic_id resolution for routing agent messages by human-readable topic title.

## Best Possible Solution

Add a Telegram-owned observed topic catalog or title resolver with explicit staleness and ambiguity behavior, exposed through the maintainer-chosen tool, context, or channel metadata surface.

## Reproduction Assessment

Yes, as a source-level capability gap. Current main exposes only current-topic topic_id/topic_name plus numeric thread targeting, while the code has no topic catalog, list action, or name-to-ID resolver.

## Solution Assessment

Unclear until maintainers choose the product/API surface. The current reverse topic-name cache is useful, but it is not the requested routing-by-title resolver and should not be stretched into silent best-effort sends without ambiguity rules.
