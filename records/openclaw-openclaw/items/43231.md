## Summary

Keep open. Current main has useful observed Telegram topic ID-to-name context and persisted topic-name cache, but it still does not expose a deterministic topic-title-to-topic-ID resolver, forum-topic catalog, or topic-list action that an agent can use for routing by human-readable title.

## What This Changes

The issue requests first-class Telegram forum topic title-to-topic-ID resolution or a topic catalog/list surface for agent routing.

## Best Possible Solution

Add a Telegram-owned observed topic catalog or title resolver with explicit freshness, ambiguity, and fail-safe behavior, exposed through the maintainer-chosen tool, context, or channel metadata surface.

## Reproduction Assessment

Yes, as a source-level capability gap. Current main exposes current-topic topic_id/topic_name and numeric thread targeting, while the code has no forum-topic catalog, list action, or title-to-ID resolver.

## Solution Assessment

Unclear until maintainers choose the API/product surface. The current observed topic-name cache is useful, but it should not be stretched into silent best-effort title sends without ambiguity and staleness rules.
