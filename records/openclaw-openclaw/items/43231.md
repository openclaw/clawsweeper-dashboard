## Summary

Keep open. Current main has useful Telegram forum topic-name learning and persistence, but it still exposes only the current topic id/name pair and does not provide a topic catalog, name resolver, or name-based outbound routing for another topic.

## What This Changes

The issue requests first-class Telegram forum topic name to topic id resolution or a topic catalog so agents can route messages by human-readable topic name.

## Best Possible Solution

Implement the remaining capability inside the bundled Telegram plugin: expose an observed-topic catalog or resolve action backed by the persisted topic-name cache, and allow routing by name only for exact unique non-closed matches with ambiguity and cache-miss fallback.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: ask an agent in a Telegram forum group to route to another topic by name, and current main has no topic-name resolver or topic-list action while outbound target parsing is numeric-only.

## Solution Assessment

Yes, the narrow maintainable direction is Telegram-owned support built on the existing persisted topic-name cache. It should avoid fuzzy matching and should not claim complete coverage beyond observed or persisted topic metadata.
