## Summary

This should stay open: current main and v2026.6.6 have useful partial work for numeric Telegram topic routing and ID-to-name metadata, but they still do not expose a reliable topic-name-to-topic-ID resolver or agent-callable topic catalog.

## What This Changes

The issue asks for reliable Telegram forum topic-name to topic_id resolution so agents can route messages by human-readable forum topic names.

## Best Possible Solution

Add a Telegram-owned observed-topic catalog and resolver action, likely aligned with the existing thread-list/message-action surface, that returns numeric IDs plus lifecycle state and rejects unknown or ambiguous title matches.

## Reproduction Assessment

Yes, at source level: current main accepts numeric thread IDs and exposes observed topic names, but no public Telegram action or metadata API accepts a human-readable topic title and returns a matching topic ID. I did not run a live Telegram forum reproduction in this read-only review.

## Solution Assessment

No to injecting a potentially incomplete topic map into every session by default. The narrower maintainable solution is an on-demand Telegram-owned observed-topic catalog/resolver with explicit unknown, duplicate, renamed, and closed-topic semantics.
