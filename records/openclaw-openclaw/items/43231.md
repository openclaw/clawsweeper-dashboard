## Summary

Keep open. The merged topic-name cache from https://github.com/openclaw/openclaw/pull/65973 is useful partial coverage, but current main still lacks the requested forum topic catalog, list action, or topic-name-to-topic_id resolver for deterministic Telegram forum routing.

## What This Changes

The issue requests deterministic Telegram forum topic-name to topic_id resolution for agent routing, via a topic catalog, list action, or metadata endpoint.

## Best Possible Solution

Add a Telegram-owned observed topic catalog or resolver with clear partial-cache and unknown-name behavior, then expose it through the chosen agent/tool surface without requiring manual topic maps.

## Reproduction Assessment

No as a strict bug reproduction; this is a missing Telegram feature/API surface. Source inspection on current main shows current-topic topic_id/topic_name exposure and numeric topic targets, but no forum_topics map, list action, or title resolver.

## Solution Assessment

No, the current partial topic-name cache is not the full solution for this issue. The maintainable path is a deliberate Telegram channel/API design around the observed cache, including stale, unknown, and duplicate topic-name behavior.
