## Summary

Keep this open: current main and the latest release still only expose observed Telegram forum topic names alongside numeric topic IDs; they do not provide a reliable name-to-ID resolver, topic catalog action, or agent-facing lookup contract.

## What This Changes

The issue requests reliable Telegram forum topic-name to topic-id resolution for agent routing.

## Best Possible Solution

Design a Telegram-owned topic catalog or resolver contract that exposes observed topic IDs and names to agents while preserving numeric threadId as the delivery authority.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: ask an agent to route to another Telegram forum topic by name when current actions and docs require numeric messageThreadId/threadId and no topic catalog action exists.

## Solution Assessment

No current solution is complete. The merged topic-name cache is useful ID-to-name context, but the best fix needs a maintainer-approved resolver or catalog API that handles stale and ambiguous names safely.
