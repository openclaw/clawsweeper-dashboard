The issue requests first-class Telegram forum topic name-to-topic ID resolution for routing agent messages to a named topic.

## Best Possible Solution

Add a Telegram-owned observed topic catalog or title resolver with explicit freshness, ambiguity, and safe-failure semantics, exposed through the maintainer-chosen tool, context, or metadata surface.

## Reproduction Assessment

## Solution Assessment

Unclear until maintainers choose the API/product surface. The existing observed topic-name cache is useful, but it should not be stretched into silent best-effort title sends without ambiguity and staleness rules.
