## Summary

Keep open. Current main and the latest shipped tag still provide only partial Telegram forum-topic support: agents can see the current numeric topic ID and an observed current topic name, but there is still no name-to-topic_id resolver, topic-list action, or forum topic catalog for routing by title.

## What This Changes

The issue asks for reliable Telegram forum topic-title to topic_id resolution so agents can route messages by human-readable topic name.

## Best Possible Solution

Add a Telegram-owned observed topic catalog or resolver with explicit ambiguity, staleness, and safe-clarification behavior.

## Reproduction Assessment

Yes, as a source-level capability gap rather than a broken existing contract. Current main exposes only current-topic topic_id/topic_name plus numeric thread targeting, while source inspection shows no topic catalog, list action, or name-to-ID resolver.

## Solution Assessment

No, the current partial topic-name cache is not enough to solve routing by title. The maintainable path needs a deliberate Telegram channel/tool design for observed topic names, ambiguity, stale data, and safe fallback behavior.
