## Summary

This remains necessary: current main supports numeric Telegram topic routing and persists names for observed topic IDs, but it exposes no agent-callable topic listing or title-to-ID resolution operation. The merged work in https://github.com/openclaw/openclaw/pull/65973 is valuable partial progress, while Telegram’s Bot API discovery limits and duplicate-name behavior require a maintainer product decision before implementation.

## What This Changes

Request reliable resolution of Telegram forum topic names to numeric topic IDs for agent-directed message routing.

## Best Possible Solution

Add an on-demand Telegram-owned action that lists and resolves persisted observed topics, returns numeric IDs and lifecycle state, rejects ambiguous matches, and clearly reports that arbitrary Bot API topic discovery is unavailable.

## Reproduction Assessment

Yes. Current main has numeric topic targeting and ID-to-name storage, but no public action or API accepts a human-readable title and returns the corresponding topic ID.

## Solution Assessment

No to injecting a potentially large and incomplete catalog into every session. The narrowest maintainable direction is an on-demand Telegram action over the existing persistent observed-topic store, with explicit unknown and ambiguous results.
