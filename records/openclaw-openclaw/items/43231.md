## Summary

Keep open. Current main and v2026.5.20 expose numeric Telegram topic targeting plus current-topic ID-to-name context, but they still do not expose a deterministic topic-title-to-topic-ID resolver, topic catalog, or topic-list action for agent routing.

## What This Changes

The issue requests first-class Telegram forum topic title-to-topic-ID resolution or catalog/list metadata so agents can route by human-readable topic name.

## Best Possible Solution

Add a Telegram-owned observed topic catalog or title resolver with explicit freshness, duplicate-title, closed-topic, and safe-failure semantics, exposed through the maintainer-chosen tool, context, or metadata surface.

## Reproduction Assessment

Yes, as a source-level capability gap. Current main exposes current-topic `topic_id`/`topic_name` and numeric thread targeting, while the code has no forum-topic catalog, list action, or title-to-ID resolver.

## Solution Assessment

Unclear until maintainers choose the API/product surface. The existing observed topic-name cache is useful, but it should not be stretched into silent best-effort title sends without ambiguity and staleness rules.
