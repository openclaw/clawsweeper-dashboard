## Summary

Keep this open: current main and the latest release expose observed Telegram forum topic names in agent context, but they still do not provide a reliable topic-name-to-topic-ID resolver, topic catalog action, or documented agent-facing lookup path.

## What This Changes

The issue requests first-class Telegram forum topic name-to-topic ID resolution so agents can route messages by human-readable topic names instead of asking users for numeric IDs.

## Best Possible Solution

Design a Telegram-owned topic catalog/resolver contract that exposes observed topic IDs and names to agents while preserving numeric threadId routing as the delivery authority.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: ask an agent to route by Telegram forum topic name when only the current code's numeric threadId/target fields are available, and there is no resolver/catalog action to satisfy that request.

## Solution Assessment

No current solution is complete; the merged topic-name cache is useful but only solves ID-to-name context, while the best fix needs a maintainer-approved resolver or catalog API that handles stale and ambiguous names.
