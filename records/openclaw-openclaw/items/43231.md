## Summary

Keep this open: current `main` and `v2026.6.6` partially solve the reverse direction by exposing observed topic names, but they still do not expose a reliable topic-name-to-topic-ID resolver or agent-callable topic catalog.

## What This Changes

Request to add reliable Telegram forum topic-name to topic_id resolution, or an agent-visible topic catalog, for routing messages by human-readable topic name.

## Best Possible Solution

Add a Telegram-owned observed-topic catalog/resolver action that returns numeric IDs with lifecycle state and rejects unknown or ambiguous title matches.

## Reproduction Assessment

Yes, at source level: current `main` and `v2026.6.6` accept numeric thread IDs and expose observed topic names, but no public Telegram action or metadata API accepts a human-readable topic title and returns a matching topic ID. I did not run a live Telegram forum reproduction in this read-only review.

## Solution Assessment

No to injecting an incomplete map into every session by default. The narrower maintainable path is an on-demand Telegram-owned observed-topic catalog/resolver with explicit unknown, duplicate, renamed, and closed-topic semantics.
