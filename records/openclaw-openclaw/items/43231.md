## Summary

Keep open: current main and the latest release expose observed Telegram topic names alongside topic IDs and support numeric topic routing, but they still do not provide a first-class topic-name to topic_id catalog, lookup, or list tool for agent routing.

## What This Changes

Expose reliable Telegram forum topic title-to-topic_id resolution or a topic catalog/list surface so agents can route by human-readable topic names.

## Best Possible Solution

Add a Telegram-owned tool/context surface that exposes an explicitly partial observed `topic_id <-> title` catalog from the persisted topic-name cache, with docs/tests that state it is not authoritative Bot API enumeration.

## Reproduction Assessment

Yes, as a source-level reproduction: current main only exposes the current observed `topic_id`/`topic_name` pair and only accepts numeric topic targets for sends. I did not run a live Telegram forum, but the missing name-to-ID catalog/list surface is clear from source, docs, and the related PR discussion.

## Solution Assessment

Unclear pending maintainer choice: the issue’s proposed directions are reasonable, but the best implementation must choose explicit semantics for a partial observed cache because the Bot API does not provide authoritative topic-name lookup.
