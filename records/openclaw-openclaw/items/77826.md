Keep open. Current main still has the source-level stripped-refresh path that can prepare empty web-tool metadata and publish it over active runtime state, and the linked candidate fix is still an open draft PR with blocking review findings and missing real behavior proof.

## What This Changes

Reports that config writes can make runtime secrets refresh overwrite active web-tool metadata with empty metadata, dropping web and plugin tools in fresh sessions.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/97096 or a replacement that carries explicit stripped-refresh provenance, preserves only ambiguous fast-path refreshes, and proves explicit clears still work.

## Reproduction Assessment

Yes for source-level reproduction: current main can prepare empty web-tool metadata from a no-web/no-SecretRef source config and activation installs it unconditionally. I did not run a live gateway reproduction in this read-only review.

## Solution Assessment

Yes, the maintainable fix belongs at the secrets runtime refresh/activation boundary. The implementation still needs explicit provenance so stripped partial refreshes are preserved without making authoritative clears sticky.
