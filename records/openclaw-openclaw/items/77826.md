## Summary

Keep open: current main still has the source-level failure path, and the live issue is already linked to open fix PRs with closing references. Those PRs should be reviewed, landed, or superseded before this issue closes.

## What This Changes

The issue reports that config-write refreshes can replace populated plugin web-tool runtime metadata with an empty fast-path snapshot, causing web and plugin tools to disappear in fresh sessions.

## Best Possible Solution

Land one runtime-boundary fix that preserves active web-tool metadata only for stripped refreshes while keeping explicit removals authoritative, then let the merged PR close this issue.

## Reproduction Assessment

Yes. At source level, activate a snapshot with configured web or plugin web metadata, then refresh with a source config lacking `tools.web` and plugin `webSearch`/`webFetch`; current main prepares empty fast-path metadata and activates it.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the secrets runtime refresh/activation boundary, and the open linked fix PRs should be reviewed rather than closing this issue early.
