## Label Justifications

- `P1`: The issue reports a regression that can remove web, memory, media, and plugin tools from subsequent agent sessions after ordinary config writes.
- `impact:session-state`: The failure is active runtime metadata state drift across config refreshes, so fresh sessions can see a different tool surface than the resolved gateway state.
- `impact:auth-provider`: The affected metadata carries runtime web-search/web-fetch provider selection and SecretRef-derived provider state, which can be lost after refresh.

## Summary

Keep open: current main and the latest release still have the source-level path where a stripped config-write refresh can take the secrets fast path and replace populated web-tool metadata with an empty metadata object. The closed linked PRs attempted the right area but did not merge, and the open related plugin-tool issue is adjacent rather than a safe canonical replacement for this config-refresh bug.

## What This Changes

The issue asks OpenClaw to preserve plugin/web-tool runtime metadata when config writes refresh the secrets runtime from a stripped source config.

## Best Possible Solution

Add a durable runtime-boundary preservation guard with regression coverage: ambiguous fast-path stripped refreshes should not clear populated active web-tool metadata, while full resolver results and explicit empty web/plugin containers remain authoritative clears.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes at source level: activate populated runtime web-tool metadata, then process a config write whose `nextSourceConfig` omits web/plugin containers; the refresh can take the fast path and activation installs empty metadata. I did not run a live gateway session in this read-only review.

## Solution Assessment

Is this the best way to solve the issue? Yes: the maintainable fix belongs at the secrets runtime snapshot boundary, preserving only ambiguous fast-path refreshes and leaving explicit clears/full resolver results authoritative.
