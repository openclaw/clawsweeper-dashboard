## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/77826. Current main can clobber active runtime web-tool metadata after a config write refresh whose source config omits web/plugin web-tool containers: `prepareSecretsRuntimeSnapshot` takes the fast path and returns empty `webTools`, and activation unconditionally installs that empty metadata. Implement a narrow runtime-boundary fix, preferably by carrying fast-path provenance through `PreparedSecretsRuntimeSnapshot` and preserving existing active web-tool metadata only when a fast-path refresh is ambiguous because the incoming source config lacks explicit `tools.web` or `plugins.entries` web-tool containers. Keep explicit empty web/plugin containers authoritative so deliberate provider removals clear metadata. Do not add config options, runtime fallbacks for old config shapes, or CHANGELOG.md edits. Use the closed unmerged attempts https://github.com/openclaw/openclaw/pull/77859, https://github.com/openclaw/openclaw/pull/78067, and https://github.com/openclaw/openclaw/pull/78268 as context, but re-check the current source and avoid the stale-credential/second-stripped-refresh pitfalls noted there. Add focused regression coverage for single and repeated stripped refresh preservation, full-resolver no-web clear, and explicit empty-container clear.
