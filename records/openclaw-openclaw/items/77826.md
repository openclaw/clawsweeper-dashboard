## What This Changes

The issue reports that config writes with stripped source config can cause the runtime secrets fast path to drop active plugin web-tool metadata, making web tools disappear in later sessions.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/97096 or a narrow replacement that distinguishes stripped refreshes from authoritative no-web config and proves preservation, explicit clear, and unrelated plugin-entry refresh cases.

## Reproduction Assessment

## Solution Assessment

Yes, the maintainable fix belongs at the secrets runtime refresh and activation boundary. The current candidate PR still needs to prove the distinction between stripped partial refreshes and authoritative clears before it can close this issue.
