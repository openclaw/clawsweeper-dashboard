## Summary

Keep open. Current main and v2026.6.8 still let a no-web/no-SecretRef refresh prepare empty web-tool metadata and activate it unconditionally; normal config writes are partly mitigated by canonical-source rereads, but the runtime-level stripped-refresh invariant is not implemented.

## What This Changes

The issue reports that config writes or stripped config refreshes can drop active plugin/web-tool metadata from later agent tool lists.

## Best Possible Solution

Add a focused secrets-runtime guard that preserves populated web-tool metadata only for ambiguous stripped fast-path refreshes, while keeping full-resolver runs and explicit empty web/plugin containers authoritative.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only review. Source inspection does show a clear current-main path where a no-web/no-SecretRef refresh prepares empty web-tool metadata and activation installs it unconditionally.

## Solution Assessment

No, current main is only a partial mitigation. The better fix is at the secrets runtime preparation/activation boundary so stripped refreshes cannot clobber active metadata while explicit clears still work.
