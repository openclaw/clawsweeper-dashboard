## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Protect all Markdown fences before extracting math:** `ui/src/ui/katex-renderer.ts:161`
  - body: `extractMathBlocks` only special-cases exactly ``` fences before scanning for `$...$`, but markdown-it treats both backtick and tilde fences of length 3+ as code blocks. With `ui.mathRendering: "katex"`, a message like `~~~\n$x^2$\n~~~` is placeholdered before markdown-it and then restored as KaTeX inside `<code>`, changing code that currently renders literally. Please make the extraction fence handling follow markdown-it/CommonMark fence rules, or tokenize before replacement, and add regression tests for tilde and long fences.
  - confidence: 0.91
- **[P3] Retry failed KaTeX stylesheet loads:** `ui/src/ui/katex-renderer.ts:437-443`
  - body: `cssLoaded` flips true before the `<link>` stylesheet has actually loaded, and the code only resets it when the `?url` import rejects. In the browser, the asset URL import can succeed while the stylesheet request fails, leaving opted-in math permanently unstyled until reload. Attach load/error handlers or keep a pending state so the next render can retry failed link loads.
  - confidence: 0.82
