Overall confidence: 0.93

Full review comments:

- **[P1] Pass mathRendering through the chat pipeline:** `ui/src/ui/views/chat.ts:174`
  - body: `ChatProps` now exposes `mathRendering`, but the PR head never supplies it from the app/config side and `renderChat()` never forwards it into `renderMessageGroup()`. Since `grouped-render.ts` only switches to the KaTeX renderer when `opts.mathRendering === "katex"`, setting `ui.mathRendering` will still render ordinary markdown.
  - confidence: 0.96
- **[P1] Sanitize rendered KaTeX before unsafeHTML:** `ui/src/ui/markdown.ts:822`
  - body: This restores KaTeX HTML after DOMPurify has already sanitized the markdown placeholders, so the new KaTeX tag/attribute allowlists and hook do not protect the rendered math before it reaches `unsafeHTML`. Rendered math should be sanitized before insertion, or the PR needs an explicit safe-output contract with hostile-input coverage.
  - confidence: 0.88
- **[P1] Update pnpm-lock for the new dependency:** `ui/package.json:17`
  - body: The PR adds `katex` to `ui/package.json`, but the PR diff does not add the corresponding `pnpm-lock.yaml` importer/package entries. OpenClaw CI and packaging use frozen-lockfile installs, so this can fail install and also leaves the resolved dependency graph unreviewed.
  - confidence: 0.92
- **[P2] Do not cache the unloaded-KaTeX fallback:** `ui/src/ui/katex-renderer.ts:286-291`
  - body: On the first render after `preloadKatex()` starts, `restoreMathBlocksSync()` emits fallback `<code>` when the module is not loaded, and `toSanitizedMarkdownHtmlWithKatex()` then caches that result under the `katex` cache key. Existing messages can remain fallback-rendered even after the dynamic import completes unless the cache is bypassed/invalidated or a rerender is requested after load.
  - confidence: 0.82
