## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Move KaTeX preload out of the render path:** `ui/src/ui/views/chat.ts:1071-1074`
  - body: When math rendering is enabled, `renderChat` runs on every Lit render and calls `preloadKatex().then(...)`. After KaTeX is loaded, `preloadKatex()` returns an already-resolved promise, so this clears the markdown cache and calls `requestUpdate()` after every render, causing a continuous update loop for the opt-in feature. Move this to a stateful lifecycle/effect path that only runs when the setting transitions to KaTeX or when the module first resolves.
  - confidence: 0.93
- **[P1] Remove the unrelated root lockfile dependency:** `pnpm-lock.yaml:307-309`
  - body: The PR lockfile adds `tldts` to the root importer, but the PR does not change root `package.json` and this KaTeX feature only adds `katex` under `ui/package.json`. That leaves the lockfile inconsistent with the manifest and introduces an unrelated supply-chain delta that frozen installs and dependency review should not have to accept.
  - confidence: 0.9
- **[P2] Cap KaTeX sizes for untrusted chat math:** `ui/src/ui/katex-renderer.ts:312-317`
  - body: This renders user/model-supplied chat math with KaTeX but does not set `maxSize`; KaTeX documents the default as `Infinity`, allowing inputs like `\rule{500em}{500em}` to create arbitrarily large elements. Add a bounded `maxSize` suitable for chat and cover the hostile-size case so opt-in math rendering cannot be used to make the Control UI unusable.
  - confidence: 0.86
