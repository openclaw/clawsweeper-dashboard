## Summary

Keep open: the attach recovery direction and supplied live output are solid, but current PR head still has a blocking child-adapter null-stdin regression in inherited-stdin mode.

## What This Changes

The branch adds `process attach`, input-wait metadata and hints, docs/changelog coverage, and child/PTY stdin writable-state tracking for interactive background session recovery.

## Best Possible Solution

Land the attach recovery feature after guarding nullable child stdin and adding inherited/default stdin regression coverage.

## Reproduction Assessment

Yes for the review finding: PR head reads `child.stdin` before checking it, and a real Node inherited-stdin spawn returns `child.stdin === null`. The underlying attach scenario is supported by the contributor's copied live output rather than rerun in this read-only review.

## Solution Assessment

No, not as-is: the attach direction is narrow and aligned with the maintainer request, but the child adapter must preserve nullable inherited-stdin behavior. The safer repair is to guard `child.stdin` before reading stream state and cover the inherited/default stdin path in tests.
