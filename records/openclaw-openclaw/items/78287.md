## Summary

Keep open: the current head fixes the earlier lint blockers and has green CI plus sufficient live proof for the attach flow, but the PR introduces a nullable-stdin crash in the child adapter path that should block merge.

## What This Changes

Adds `process attach`, input-wait metadata and hints, stdin writable-state tracking for child/PTY adapters, docs, tests, and a changelog entry for interactive background process recovery.

## Best Possible Solution

Land the attach recovery feature after preserving nullable stdin behavior in the child adapter and adding focused regression coverage for the inherited-stdin path.

## Reproduction Assessment

Yes for the merge blocker: Node was verified to return `child.stdin === null` for inherited stdin, and the PR head reads `child.stdin.destroyed` before checking for a stream. The original attach scenario is supported by the PR's copied live output rather than rerun locally in this read-only review.

## Solution Assessment

No, not as-is: the attach direction is narrow and aligned with the linked maintainer request, but the stdin state tracking must preserve the existing nullable child-stdin contract. The safer fix is to guard `child.stdin` before reading stream state and cover `stdinMode: inherit` or omitted `stdinMode` in tests.
