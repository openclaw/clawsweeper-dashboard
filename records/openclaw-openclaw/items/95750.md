## Summary

Keep open: current main still has no persistent cross-boot retry budget for main-session restart recovery, and open fix PRs now own the implementation path.

## What This Changes

The issue requests a durable retry ceiling and quarantine path for wedged main-session restart recovery so a persisted `abortedLastRun` row cannot death-loop the gateway across reboots.

## Best Possible Solution

Land one reviewed implementation that adds durable main-session recovery budget/quarantine state, preserves the transcript, resets the budget on successful progress, and keeps trigger-layer fixes separate.

## Reproduction Assessment

Yes, at source level: seed a running main-session row with `abortedLastRun: true`, make recovery fail or wedge again, and a fresh process will treat the row as eligible because no durable attempt ceiling is stored. I did not run the full Telegram/systemd production loop.

## Solution Assessment

Yes, a durable main-session retry budget with quarantine is the narrowest maintainable survivability fix; a new config knob is optional and should not be required for the first repair.
