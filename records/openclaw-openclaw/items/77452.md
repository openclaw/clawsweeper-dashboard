## Summary

Keep open: current main still prefers the built-in channel alias over the claiming plugin id in the no-preferOver fallthrough, while this PR directly fixes that path with focused coverage and a changelog entry.

## What This Changes

The PR changes plugin auto-enable channel fallback to use a claiming plugin's manifest id instead of the normalized built-in channel alias, with a regression test and changelog entry.

## Best Possible Solution

Land this PR or an equivalent fix after CI, preserving the built-in fallback only when no manifest claims the configured channel.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure a built-in/catalog channel id with a claiming manifest plugin whose id differs and has no preferOver, then current main collects the alias before the plugin id.

## Solution Assessment

Yes. Dropping builtInId only from the claimed-plugin fallthrough is the narrowest maintainable fix because the no-claims branch already handles built-in channel fallback.
