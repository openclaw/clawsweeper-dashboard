## Summary

Keep open for normal maintainer review: current main still has the alias-before-plugin-id bug, and this PR is a narrow, source-supported fix with focused regression coverage and changelog text.

## What This Changes

The PR changes plugin auto-enable channel selection to prefer a claiming plugin manifest id over the built-in channel alias when no preferOver is declared.

## Best Possible Solution

Land this PR or an equivalent narrow fix after CI, preserving built-in fallback when no plugin claims a channel while using manifest plugin ids whenever a plugin does claim that channel.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a configured built-in channel with a claiming manifest plugin id and no preferOver reaches the fallthrough that currently returns the channel alias before the plugin id.

## Solution Assessment

Yes. Dropping `builtInId` only from the claimed-plugin fallthrough is the narrowest maintainable solution because the no-claims branch already handles built-in channel fallback and the preferOver branch remains unchanged.
