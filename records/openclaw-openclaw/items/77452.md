## Summary

Keep open: current main still has the reported alias-preference bug, and this PR is a narrow fix with focused regression coverage and a changelog entry. I found no blocking review findings.

## What This Changes

The PR removes the built-in channel id from the claimed-plugin fallthrough, adds a regression test for a manifest plugin id that differs from a built-in channel alias, and records the user-facing fix in the changelog.

## Best Possible Solution

Land this fix or an equivalent one after CI, preserving built-in fallback when no plugin claims a channel while using manifest plugin ids whenever a plugin does claim that channel.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a configured built-in channel with a claiming manifest plugin id and no preferOver reaches the current fallthrough and returns the channel alias instead of the manifest id.

## Solution Assessment

Yes. Dropping `builtInId` only from the claimed-plugin fallthrough is the narrowest maintainable fix because the no-claims branch already handles built-in channel fallback.
