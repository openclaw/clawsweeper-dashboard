## Review Findings

Full review comments:

- **[P1] Do not delete temp indexes that may still be active:** `extensions/memory-core/src/memory/manager-sync-ops.ts:2083`
  - body: This sweep removes every matching `.tmp-*` or `.backup-*` base file during an ordinary sync without an age, owner, lock, or active-operation check. A gateway and `openclaw memory index --force`, or two gateway processes sharing state, can overlap; one process can therefore unlink the other process's live reindex database. Keep orphan recovery in the atomic-reindex owner and delete only artifacts that are demonstrably abandoned, including their WAL/SHM sidecars through the existing retry helpers.
  - confidence: 0.94
- **[P2] Prove checkpointing without allowing WAL replay:** `extensions/memory-core/src/memory/manager.wal-checkpoint.test.ts:88-93`
  - body: Opening the database through SQLite and selecting the row does not prove `wal_checkpoint(TRUNCATE)` copied it into the main database file: the new connection can discover and read the existing WAL. This test would also pass without the production checkpoint in the normal valid-WAL case. Use a crash/restart fixture that removes or separates the WAL before inspection, inspect a copied main file without its sidecars, or otherwise assert checkpoint state directly.
  - confidence: 0.93
- **[P3] Remove the PR body artifact from the repository:** `PR_BODY.md:1`
  - body: `PR_BODY.md` duplicates the GitHub PR description and is unrelated to the runtime fix. Landing it would leave contributor workflow debris at the repository root; remove it from the branch.
