## Summary

## What This Changes

The PR adds a `resolve_exec_env` typed plugin hook, docs, tests, and exec-tool wiring so plugins can contribute filtered environment variables to gateway, sandbox, and node exec runs.

## Best Possible Solution

Land the additive hook only after priority collisions, env filtering, docs/tests, and redacted real exec proof are aligned; keep sender identity payload work with the linked context PR and feature request.

## Reproduction Assessment

Not applicable as a feature PR. Source inspection confirms current main lacks the hook, and the branch behavior is currently backed by unit tests rather than a reproduced bug path.

## Solution Assessment

Mostly yes, a dedicated plugin-owned exec-env hook after env sanitization fits the plugin boundary. The implementation should preserve high-priority values on duplicate keys and needs real runtime proof before merge.
