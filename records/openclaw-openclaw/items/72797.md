## Summary

Keep open. Current main still lacks this hook, and the PR remains blocked by exec env filtering bypass, missing node forwarding, and absent real behavior proof rather than being a cleanup-close candidate.

## What This Changes

The PR adds a `resolve_exec_env` plugin hook, hook runner types/tests, and exec-time merging of plugin-provided environment variables.

## Best Possible Solution

Ship an additive exec-env SDK contract that reuses reserved-key validation, forwards audited values consistently for gateway/sandbox/node, documents the plugin/privacy contract, and aligns with https://github.com/openclaw/openclaw/issues/73343.

## Reproduction Assessment

Yes for the PR defects: source inspection shows hook output would be merged after host env sanitizer enforcement, and node execution still builds system.run env from params.env. The requested hook itself is a new capability, so bug-style reproduction is not applicable.

## Solution Assessment

No, not as currently patched. A dedicated hook is a plausible direction, but the implementation must preserve the exec env security contract and route the same audited values through node execution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve host env filtering for hook output:** `src/agents/bash-tools.exec.ts:1627`
  - body: The hook output is assigned after `sanitizeHostExecEnvWithDiagnostics` rejects blocked request env keys, so a plugin can reintroduce `PATH`, loader variables, proxy/TLS overrides, or runtime-control keys for host execution. Run hook output through the same reserved-key policy, or an explicit trusted-plugin policy, before merging it into the process env.
  - confidence: 0.91
- **[P2] Forward hook env to node execution:** `src/agents/bash-tools.exec.ts:1627`
  - body: For `host=node`, this merge only mutates the local `env` object. The node path still builds the `system.run` payload env from `requestedEnv: params.env`, so variables returned by `resolve_exec_env` will not be sent to node execution.
  - confidence: 0.87
