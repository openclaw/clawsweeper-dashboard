## Summary

Keep open. Current main still lacks the proposed hook, and the PR remains a useful feature candidate, but the patch is not merge-ready because hook output can bypass the existing host exec env security boundary, node execution would not receive hook-provided env, and contributor real behavior proof is still missing.

## What This Changes

The PR adds a typed `resolve_exec_env` plugin hook, exposes it through the hook runner, calls it during exec environment construction, and adds focused hook tests.

## Best Possible Solution

Land a revised additive SDK seam that validates or explicitly gates plugin-provided exec env, forwards the final audited env consistently to gateway, sandbox, and node execution, and documents privacy plus reserved-key semantics alongside the sender-context work.

## Reproduction Assessment

Yes. Source inspection of the PR diff and current main shows hook output is assigned after host env sanitization, and the node path builds the `system.run` payload from `params.env` rather than the mutated local `env`.

## Solution Assessment

No, not as currently patched. A dedicated hook is a reasonable direction, but the implementation needs to preserve the exec env security contract and route the audited hook env through every host path before it is the narrowest maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve host env filtering for hook output:** `src/agents/bash-tools.exec.ts:1627`
  - body: The hook output is assigned to `env` after `sanitizeHostExecEnvWithDiagnostics` has already rejected reserved host keys, so a plugin can reintroduce `PATH`, loader variables, proxy/TLS overrides, or other blocked env pivots for gateway execution. Please apply the same reserved-key policy, or add an explicit trusted-plugin gate, before merging hook output into the execution environment.
  - confidence: 0.91
- **[P2] Forward hook env to node execution:** `src/agents/bash-tools.exec.ts:1627`
  - body: For `host=node`, this merge only changes the local `env` object. The node path builds the `system.run` target env from `requestedEnv: params.env`, so variables returned by `resolve_exec_env` never reach the node host. Please route the sanitized hook env through the node request path and add node coverage.
  - confidence: 0.87
