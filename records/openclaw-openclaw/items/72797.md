## Summary

Keep open: this PR adds a useful public SDK hook that is not on current main, but it still has a blocking duplicate-key priority issue and lacks required real behavior proof from a real setup.

## What This Changes

The branch adds a public `resolve_exec_env` plugin hook, exec-tool integration, reserved-key filtering, docs, and tests for plugin-contributed exec environment variables.

## Best Possible Solution

Keep the dedicated plugin-owned exec env seam, preserve priority-aware duplicate-key behavior, retain reserved-key filtering, document the final semantics, and require redacted runtime proof that plugin env values reach the intended exec host.

## Reproduction Assessment

Not applicable; this is a feature PR rather than a bug report. Source inspection confirms current main lacks the hook, while the proposed runtime behavior is only supported by tests and PR text so far.

## Solution Assessment

Mostly yes; a dedicated plugin hook after exec env construction fits the plugin-agnostic boundary better than hardcoding channel env vars in core. The implementation should align duplicate-key handling with hook priority semantics and add real runtime proof before merge.
