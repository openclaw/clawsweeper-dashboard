## Summary

Keep open. The feature direction fits the plugin boundary, but this PR is not merge-ready because it still lacks real behavior proof and the current head has blocking source-review issues around hook priority semantics and gateway approval visibility for plugin-provided exec env.

## What This Changes

The PR adds a public `resolve_exec_env` plugin hook, wires filtered hook output into exec environment construction, and adds hook docs plus focused tests.

## Best Possible Solution

Keep the dedicated plugin-owned exec env seam, but bind filtered plugin env into gateway approval/audit inputs, preserve priority-aware collision behavior, document the final semantics, and require redacted runtime proof before maintainer SDK/security review.

## Reproduction Assessment

Not applicable as a current-main bug reproduction; this is a feature PR. The review findings are source-reproducible from PR head: the merge function overwrites earlier hook results, and gateway approvals receive `params.env` rather than merged `pluginEnv`.

## Solution Assessment

Mostly yes, but not yet. A dedicated plugin hook after exec env construction is the right boundary, but the implementation needs priority-safe merging, approval/audit visibility, and real runtime proof before it is the maintainable solution.
