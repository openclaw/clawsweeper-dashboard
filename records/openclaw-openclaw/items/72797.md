## Summary

Keep open. The PR adds a plausible SDK seam, but current main does not already implement it and the branch still has two concrete blockers: hook-provided env can bypass the existing host exec env boundary, and `host=node` would not receive hook values.

## What This Changes

The PR adds a `resolve_exec_env` plugin hook, a hook-runner method, unit tests, and exec-time merging of plugin-provided environment variables.

## Best Possible Solution

Ship an additive exec-env SDK seam that preserves the audited host/node/sandbox env boundary, documents the plugin trust and privacy contract, and includes real plugin/channel exec proof.

## Reproduction Assessment

Yes for the PR defects: source inspection shows hook output is merged after host env sanitization, and node execution still builds the remote env from `params.env`. The requested hook itself is a new capability, so bug-style reproduction is not applicable.

## Solution Assessment

No, not as currently patched. A dedicated hook is a plausible direction, but it must preserve or explicitly redefine the trusted env policy and route the final audited env through node execution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve host env filtering for hook output:** `src/agents/bash-tools.exec.ts:1543`
  - body: The hook result is merged after `sanitizeHostExecEnvWithDiagnostics` rejects blocked request env keys. For host execution, this lets a plugin reintroduce values such as `PATH`, loader/runtime options, proxy/TLS overrides, or credential/config pivots that the current code and docs reject; run hook output through the same policy or add an explicit trusted-plugin policy before merging it.
  - confidence: 0.91
- **[P2] Forward hook env to node execution:** `src/agents/bash-tools.exec.ts:1543`
  - body: For `host=node`, this mutates the local `env` object, but the remote `system.run` target is built from `params.requestedEnv`, and the call site still passes `requestedEnv: params.env`. Hook-provided variables will be absent from node execution, so the new hook silently fails for node hosts.
  - confidence: 0.88
