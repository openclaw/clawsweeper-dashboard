## Summary

Keep open. The PR adds a plausible plugin-owned exec-env seam that current main does not have, but it still lacks contributor real behavior proof and has merge-blocking SDK contract/priority-semantics issues.

## What This Changes

The PR adds a `resolve_exec_env` plugin hook, exec-tool integration, docs, and hook tests so plugins can contribute filtered environment variables to exec runs.

## Best Possible Solution

Keep the hook as an additive plugin SDK seam, update the SDK API baseline, settle duplicate-key priority semantics, and require redacted real runtime proof that hook-provided env reaches exec.

## Reproduction Assessment

Not applicable as a feature PR. Source inspection confirms current main lacks this hook; the proposed behavior is currently supported by code/tests in the branch, not by a current-main bug reproduction.

## Solution Assessment

Mostly yes, a dedicated plugin-owned hook after env construction fits the plugin-agnostic boundary. It is not merge-ready until the public SDK baseline and priority collision semantics are corrected and real behavior proof is supplied.
