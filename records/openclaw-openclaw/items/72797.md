## Summary

This PR should stay open: the feature is not on current main and the proposed hook fits the plugin-owned env-injection direction, but the current branch still has a merge-blocking hook-priority collision bug and lacks required real behavior proof for an external PR.

## What This Changes

Adds a `resolve_exec_env` plugin hook, docs, and tests so plugins can contribute filtered environment variables to exec tool runs.

## Best Possible Solution

Keep the hook as a plugin-owned exec env seam, preserve high-priority hook values on duplicate keys, retain reserved-key filtering, and require real runtime proof that env reaches the intended exec hosts.

## Reproduction Assessment

Not applicable as a feature PR. Source inspection confirms current main lacks this hook, while the proposed behavior is currently demonstrated only by tests rather than a real runtime setup.

## Solution Assessment

Mostly yes, a dedicated plugin hook after sanitized env construction matches the plugin-agnostic boundary. The implementation should align duplicate-key handling with existing high-priority hook semantics before merge.
