## Summary

Keep open. Current main and v2026.6.10 still provide model selection, fallback editing, and generic schema-driven config editing, but not the requested simple Dashboard Models/Providers flow for provider add/edit/remove, credential mode selection, connection testing, or role assignment.

## What This Changes

Request to add a Control UI/Dashboard Models/Providers screen for managing multiple model providers and model roles without raw JSON editing or rerunning onboarding.

## Best Possible Solution

Design a scoped Models/Providers Control UI screen that reuses config.get/config.set/config.apply, baseHash protection, SecretRef redaction, and the current models.providers.<id>.models[] contract, with provider probing kept behind backend safety boundaries.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing behavior. Source inspection confirms current main has adjacent model/config controls but no dedicated provider-management flow to reproduce as a failing path.

## Solution Assessment

Unclear until maintainers scope the UX and security boundaries. The maintainable direction is to reuse the existing config controller and current provider schema, but the issue sketch should not be copied literally because current config names and provider contracts differ.
