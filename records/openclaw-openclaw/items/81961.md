Keep open. This is a new Dashboard provider-management feature, not a fixed bug or duplicate: current main has generic config editing, model selection, and configured-model listing, but no Add/Edit/Test/Remove provider flow or role-assignment wizard.

The issue requests a Dashboard / Control UI flow for adding, editing, testing, removing, and choosing multiple model providers without manual JSON edits.

Design a scoped Control UI provider-management screen that reuses the existing config.set/config.apply, model catalog, model auth status, SecretRef redaction, and provider config contracts instead of inventing a parallel schema.
