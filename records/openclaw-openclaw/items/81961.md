## Summary

Keep open. Current main still has generic config editing and model selection, but it does not provide the requested Dashboard provider-management workflow for add/edit/test/remove, multi-model entry, credential mode, or role assignment.

## What This Changes

The issue requests a Control UI Models/Providers screen for managing multiple model providers after onboarding without hand-editing JSON.

## Best Possible Solution

Design a scoped Models/Providers Control UI screen that reuses config.set/config.apply, the current models.providers.<id>.models[] schema, SecretRef redaction, Gateway-authoritative model visibility, and SSRF-safe backend provider testing.
