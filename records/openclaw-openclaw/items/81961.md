## Summary

Keep open. Current main still lacks a dedicated Dashboard provider-management workflow for add/edit/test/remove and role assignment, and this is fresh UI/product work with credential and provider-test semantics rather than conservative cleanup.

## What This Changes

The issue requests a Dashboard Models/Providers flow for adding, editing, testing, removing, and assigning roles for multiple model providers after OpenClaw is running.

## Best Possible Solution

Design a scoped Models/Providers Control UI screen that reuses config.set/config.apply, SecretRef redaction, and the current models.providers.<id>.models[] contract, with provider connection testing handled through an SSRF-safe backend path.
