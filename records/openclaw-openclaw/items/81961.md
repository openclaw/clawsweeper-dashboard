## Summary

Keep open. Current main still has model selection, fallback editing, and generic schema-driven config editing, but not the requested Dashboard flow for provider add/edit/remove, credential mode selection, connection testing, or role assignment; the overlapping Dashboard PR is closed unmerged and the open onboarding issue covers a different surface.

## What This Changes

Request a simple Control UI Models/Providers screen for managing multiple model providers after OpenClaw is already running.

## Best Possible Solution

Design a scoped Models/Providers Control UI screen that reuses config.get/config.set/config.apply, baseHash protection, SecretRef redaction, and the current models.providers.<id>.models[] contract, with provider probing kept behind backend safety boundaries.
