## Summary

Keep open. Current main still lacks a dedicated Dashboard provider-management workflow, and the remaining work is product/UI scope with credential and provider-test security semantics rather than conservative cleanup.

## What This Changes

The issue requests a Dashboard/Control UI flow for adding, editing, testing, removing, and assigning roles for multiple model providers after OpenClaw is running.

## Best Possible Solution

Design a scoped Models/Providers Control UI screen that reuses config.set/config.apply, SecretRef redaction, and the current models.providers.<id>.models[] contract, with provider connection testing handled through an SSRF-safe backend path.

## Reproduction Assessment

Not applicable; this is a feature request, not a bug report. Source inspection shows current main has model selection and generic config editing, but no dedicated Dashboard provider add/edit/test/remove flow.

## Solution Assessment

Unclear pending maintainer scoping. Reusing the existing config controller and current provider schema is the maintainable direction; copying the issue sketch directly would not be the best implementation.
