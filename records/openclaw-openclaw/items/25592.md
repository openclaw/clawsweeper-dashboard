## Summary

Keep open. Current main has important commentary/final-answer safeguards and group/channel tool-only defaults, but direct/source chats still default to automatic visible replies and unphased assistant text can still be flushed before tool execution.

## What This Changes

The issue requests preventing assistant text emitted before or between tool calls from being delivered as visible messages to messaging channels.

## Best Possible Solution

Keep this as the canonical channel-leak bug, review the two relevant open OpenClaw PRs, and make a maintainer delivery-policy decision with regression coverage for direct/source chats and unphased pre-tool/inter-tool text.

## Reproduction Assessment

Yes. A high-confidence source/test reproduction exists: default direct/source chats can still resolve to automatic delivery, unphased assistant text enters the block reply buffer, and a current test asserts that buffered text flushes before tool execution.

## Solution Assessment

No. Current main is a partial mitigation; the remaining fix needs maintainer choice around direct/source visible-reply policy and how to treat ordinary unphased assistant text before or between tool calls.
