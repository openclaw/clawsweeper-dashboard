## Summary

Keep open: current main has important commentary/phase safeguards, but the central cross-channel leak remains source-reproducible through automatic source reply delivery, unphased assistant text handling, and cron announce payload selection. Recent duplicate and production comments continue to route narrower variants back to this issue as the canonical tracker.

## What This Changes

The issue asks OpenClaw to keep intermediate agent text between tool calls private instead of delivering it as visible channel messages.

## Best Possible Solution

Keep this issue open as the canonical bug while maintainers choose a generic final-only/message-tool delivery boundary, preserve intentional compatibility where needed, and add regression coverage for embedded source replies plus cron announce reasoning payloads.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction model for the remaining broad class: automatic source delivery can still emit unphased assistant text, and cron announce payload selection can still select reasoning-like text. A live channel repro was not run in this read-only review.

## Solution Assessment

No, current main is not the complete fix; it is a set of partial phase/commentary safeguards. The best fix is a generic delivery-boundary policy with focused regression tests, not channel-specific filters or prompt discipline alone.
