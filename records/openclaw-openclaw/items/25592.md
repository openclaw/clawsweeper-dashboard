## Summary

Keep open. Current main has meaningful mitigations for commentary-phase text and group/channel defaults, but the broader leak is not fully resolved because default direct chats can still auto-deliver assistant text and unphased assistant chunks remain eligible for visible block/final delivery on automatic paths.

## What This Changes

Prevent intermediate assistant text produced before or between tool calls from being delivered to messaging channels unless it is a final or explicit outbound reply.

## Best Possible Solution

Keep this as the canonical channel-leak bug, preserve the existing phase safeguards, and choose a direct-chat/source policy that prevents unphased pre-tool and inter-tool text from leaving automatic delivery paths with regression coverage.

## Reproduction Assessment

Yes. A high-confidence source/test reproduction exists: direct chats still resolve to automatic delivery by default, and current source still buffers unphased assistant text for visible delivery on automatic paths.

## Solution Assessment

No. Current main is a partial mitigation; the remaining fix requires a maintainer delivery-policy decision for direct/source chats and ordinary unphased text.
