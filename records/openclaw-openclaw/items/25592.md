## Summary

Keep open. Current main and the latest shipped tag still allow automatic visible delivery paths where ordinary unphased assistant text can be buffered and flushed to messaging channels; the merged commentary-phase fixes are important partial mitigation but do not close the broader leak.

## What This Changes

The issue requests that assistant text produced before or between tool calls remain internal instead of being delivered as visible Slack, Telegram, iMessage, WhatsApp, or other channel messages.

## Best Possible Solution

Keep this issue as the canonical channel-leak bug and land a maintainer-approved final-only, explicit-send, or equivalent delivery policy that preserves existing phase safeguards, includes upgrade-safe defaults or migration guidance, and adds focused regression plus live channel proof.

No. Current main contains important partial mitigations for commentary-phase text, but the complete fix still needs a maintainer policy decision for final-only or explicit-send behavior and upgrade-safe defaults.
