## Summary

Keep open. Current main has partial commentary/final-answer safeguards, but direct/source chats can still use automatic visible replies and unphased assistant text is still buffered and flushed around tool execution; open PRs also explicitly target this issue.

## What This Changes

Prevent assistant text produced before or between tool calls from being delivered as visible messages to Slack, Telegram, WhatsApp, iMessage, and similar channels.

## Best Possible Solution

Keep this as the canonical channel-leak bug, review https://github.com/openclaw/openclaw/pull/46559 and https://github.com/openclaw/openclaw/pull/51156, then choose a direct/source delivery policy with regression coverage for unphased text before and between tool calls.

## Reproduction Assessment

Yes. A high-confidence source/test reproduction exists: direct/source chats can still default to automatic delivery, unphased assistant text enters the block reply buffer, and current tests assert that buffered text flushes before tool execution.

## Solution Assessment

No. Current main is a partial mitigation; the remaining fix needs maintainer choice around direct/source visible-reply policy and how to treat ordinary unphased assistant text before or between tool calls.
