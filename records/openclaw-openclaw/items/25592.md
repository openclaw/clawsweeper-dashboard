## Summary

Keep open. Current main has partial safeguards for commentary-phase and group/channel delivery, but source inspection still shows direct/source automatic delivery plus unphased assistant text buffering that can flush before tool execution; two relevant closing PRs remain open.

## What This Changes

The issue asks OpenClaw to prevent intermediate assistant text between tool calls from being delivered as visible messages in Slack, Telegram, iMessage, WhatsApp, and similar channels.

## Best Possible Solution

Keep this as the canonical channel-leak bug, review or supersede the two open implementation PRs, and land a maintainer-approved final-only or explicit-send policy with regression coverage for unphased pre-tool and inter-tool text.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: direct/source chats can still default to automatic delivery, unphased assistant text enters the block buffer, and a current test asserts that pre-tool buffered text is delivered when tool execution starts.

## Solution Assessment

No. Current main is only a partial solution; closing now would leave the automatic direct/source delivery path and unphased pre-tool text policy unresolved, with open closing PRs still awaiting maintainer disposition.
