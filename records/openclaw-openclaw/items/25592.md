## Label Justifications

- `P1`: Fresh production reports show recurring internal-output leakage into real customer and operator conversations across multiple channels.
- `impact:security`: Leaked reasoning, metadata, and raw tool arguments can disclose private customer or operational context to unintended channel recipients.
- `impact:message-loss`: Intermediate content is duplicated or misrouted as standalone outbound messages instead of one intentional final response.

## Summary

Keep this issue open as the canonical cross-channel bug. Current main contains valuable commentary-phase safeguards, but it still intentionally auto-delivers unphased assistant text and flushes buffered pre-tool text, while recent production reports identify unresolved interactive and cron variants; open implementation PRs should be reviewed rather than creating another competing fix.

## What This Changes

Prevent intermediate assistant narration, reasoning, placeholders, and tool-call-adjacent text from becoming user-visible messaging-channel output.

## Best Possible Solution

Classify terminal user-visible output once in the agent runtime, preserve that closed classification through reply payloads, and enforce it consistently before interactive, streaming, cron, subagent, and channel delivery; pair this with an explicit compatibility decision and upgrade coverage for direct chats that currently rely on automatic bare-text replies.

## Reproduction Assessment

Yes, at source level with high confidence. Current-main tests deliberately flush unphased pre-tool text, current cron selection preserves multiple successful payloads, and detailed June 2026 reports describe matching failures on shipped builds; this read-only review did not run a live transport reproduction.

## Solution Assessment

No existing proposal is yet the complete best solution. Explicit-send semantics offer the strongest boundary but require a direct-chat compatibility decision, while verbose gating, final-tag prompting, or provider-specific phase filters alone do not cover the full interactive and autonomous delivery surface.
