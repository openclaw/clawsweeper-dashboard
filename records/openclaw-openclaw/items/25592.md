## Label Justifications

- `P1`: The report remains a high-priority broken channel workflow because internal assistant text can appear in real Slack, Telegram, Discord, iMessage, and cron-delivered conversations.
- `impact:security`: Leaked reasoning, tool arguments, raw routing fragments, or customer context can expose sensitive operational information to channel recipients.
- `impact:message-loss`: The issue is about channel delivery misrouting extra internal messages as visible user-facing replies.

## Summary

Keep open: current main and the latest stable release still have source-reproducible delivery paths where unphased assistant text can be treated as visible channel output, and recent reports add Discord, Telegram, and cron announce variants after several partial fixes. The linked implementation PRs are still open, unmerged, and conflicting, so they are candidates for maintainer review rather than proof that this issue is resolved.

## What This Changes

The issue asks OpenClaw to stop routing assistant text produced before or between tool calls to messaging channels unless it is an intentional final response or explicit message-tool delivery.

## Best Possible Solution

Keep this as the canonical delivery-boundary bug: choose a compatibility-safe invariant for direct chats and cron/announce delivery, preserve explicit message-tool and transient progress-preview semantics, and add regression coverage for unphased pre-tool, inter-tool, reasoning, and yield/abort payloads across generic and channel paths.

## Reproduction Assessment

Yes at source level: current tests deliberately flush buffered unphased text before tool execution, and cron helper selection still preserves successful text payloads without carrying isReasoning. I did not run a live channel repro in this read-only pass.

## Solution Assessment

Yes to keeping this issue open as the canonical tracker; no single existing PR is yet the complete best fix. The best repair needs a maintainer-selected delivery invariant rather than a channel-specific sanitizer or one-off prompt workaround.
