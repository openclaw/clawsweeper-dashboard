## Summary

Keep open. Current main has important partial safeguards for commentary/final_answer phases and message-tool-only group/channel delivery, but the broader bug remains source-reproducible for automatic direct/source delivery and unphased assistant text before tool calls; there are also open implementation PRs that still need maintainer disposition.

## What This Changes

The issue asks OpenClaw to prevent assistant text emitted before or between tool calls from being delivered as standalone user-visible messages to Slack, Telegram, iMessage, WhatsApp, and similar channels.

## Best Possible Solution

Keep this as the canonical channel-leak bug, review the open targeted PRs, and choose a direct/source-safe final-only or explicit-send policy with regression coverage for unphased assistant text before and between tool calls.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: direct chats still default to automatic delivery without a global or harness override, unphased assistant text is buffered, and current tests assert that buffered pre-tool text is delivered when tool execution starts.

## Solution Assessment

No. Closing now would rely on partial commentary-phase and group/channel fixes while leaving the direct/source automatic delivery and unphased pre-tool text policy unresolved; the maintainable path is a maintainer-selected delivery policy plus focused regression tests.
