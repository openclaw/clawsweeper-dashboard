## Summary

Keep open. Current main and v2026.6.5 still have automatic delivery paths where unphased assistant text can become visible channel output, and recent Discord/Telegram reports plus open linked implementation candidates show the central bug is not resolved.

## What This Changes

The issue reports that assistant text emitted before or between tool calls is routed as visible Slack, Telegram, Discord, or similar channel messages instead of remaining internal until an intentional final/source reply.

## Best Possible Solution

Keep this as the canonical product bug until maintainers choose and land a cross-delivery policy that suppresses internal/inter-tool text while preserving intentional message-tool and final delivery, with regression coverage for interactive and cron paths.

## Reproduction Assessment

Yes. I did not run a live channel repro in this read-only review, but current source plus recent Discord and Telegram reports give a high-confidence reproduction path: automatic source delivery plus unphased assistant text before or between tool calls can produce visible channel output.

## Solution Assessment

Unclear as a final policy. The requested explicit-send direction would solve the leak class but has backward-compatibility impact, so maintainers should choose the source-reply policy before landing the final cross-path fix.
