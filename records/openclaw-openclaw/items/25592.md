## Summary

Keep open: current main and the latest release still preserve automatic source delivery plus pre-tool block flushing for ordinary unphased assistant text, and the linked fix candidates are open or partial rather than merged fixes.

## What This Changes

The issue reports intermediate assistant text between tool calls leaking as user-visible messaging-channel replies instead of only delivering the final intended response.

## Best Possible Solution

Keep this issue open as the canonical delivery bug and land a maintainer-approved final-only or explicit-delivery policy with regression coverage for direct chats, group/channel chats, streaming/block delivery, message-tool turns, and cron announce delivery.

## Reproduction Assessment

Yes from source: current tests still prove buffered pre-tool assistant text is flushed as a block reply, and automatic delivery remains the default for direct chats unless configured otherwise. I did not run a live channel repro in this read-only review.

## Solution Assessment

No, current main only has partial safeguards such as commentary-phase and message-tool-only suppression. The best fix needs a maintainer product decision because making direct chats or all source replies final-only/message-tool-only has upgrade and compatibility impact.
