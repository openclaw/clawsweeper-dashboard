## Summary

Keep open. Current main and the latest shipped tag still have an automatic delivery path where unphased assistant text can enter the block reply buffer and be flushed to channels before tool execution; the commentary-phase fixes are useful partial hardening but do not close the broader leak.

## What This Changes

The issue asks OpenClaw to prevent intermediate assistant text between tool calls from being delivered as visible messages in Slack, Telegram, WhatsApp, iMessage, and similar channels.

## Best Possible Solution

Keep this as the canonical channel-leak bug and land a maintainer-approved final-only, explicit-send, or equivalent delivery policy that preserves phase safeguards, includes upgrade-safe defaults or migration guidance, and adds focused regression plus live channel proof.

## Reproduction Assessment

Yes. At source level, current main still appends unphased assistant text to the block reply buffer, a current test expects pre-tool buffered text to be delivered, and automatic visible delivery remains available for source replies.

## Solution Assessment

No. Current main contains important partial mitigations for commentary-phase text, but the complete fix still needs a maintainer policy decision for final-only or explicit-send behavior and upgrade-safe defaults.
