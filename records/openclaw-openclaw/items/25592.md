## Summary

Keep open. Current main has meaningful partial mitigations and a global opt-in for message-tool-only delivery, but direct chats still default to automatic delivery and unphased assistant text can still be buffered and emitted as visible output, matching the newest Telegram/Slack reports and the open closing PRs.

## What This Changes

The issue requests preventing intermediate assistant/tool-call narration from being delivered to messaging channels, leaving only final or explicitly sent user-facing replies visible.

## Best Possible Solution

Keep this as the canonical channel-leak bug, choose a direct-chat-safe delivery policy, then land or close the linked implementation PRs with regression coverage for unphased pre-tool and inter-tool text in streaming and final payload assembly.

## Reproduction Assessment

Yes. A high-confidence source/test reproduction exists: direct chats still resolve to automatic delivery by default, and current tests still prove unphased assistant text can emit a visible block reply and populate assistantTexts.

## Solution Assessment

No. Current main is a partial solution with useful commentary-phase and configurable message-tool-only protections, but it does not fully solve the reported default direct-chat and ordinary unphased intermediate-text leak class.
