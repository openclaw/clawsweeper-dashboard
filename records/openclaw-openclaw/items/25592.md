## Summary

Keep open. Current main and the latest v2026.6.5 release still preserve automatic source-reply paths that can deliver unphased assistant text, the cron announce helper still selects deliverable payloads without carrying reasoning metadata, and the issue has fresh June 2026 production reports plus two open linked implementation PRs.

## What This Changes

The issue asks OpenClaw to stop delivering assistant text emitted before or between tool calls as visible messages in Slack, Telegram, Discord, iMessage, WhatsApp, and other messaging channels.

## Best Possible Solution

Keep this issue as the canonical product/security bug, review the two open linked PRs, and choose a direct-chat and cron-safe delivery policy that preserves commentary-phase safeguards while suppressing unphased inter-tool text and reasoning payloads from user-facing channels.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: direct chats still default to automatic delivery, unphased assistant chunks are appended to the block-reply buffer, and a current test asserts buffered pre-tool text is delivered before tool execution.

## Solution Assessment

Unclear. A narrow suppression patch is plausible, but the best permanent fix needs maintainer policy on direct-chat compatibility and cron announce delivery while preserving existing message-tool-only and commentary-phase safeguards.
