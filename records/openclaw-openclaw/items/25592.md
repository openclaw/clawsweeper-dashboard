## Summary

Keep open. Current main has meaningful mitigations for commentary-phase text and group/channel defaults, but the broader leak is still live because default direct/source chats can auto-deliver assistant text and unphased assistant chunks remain eligible for visible block/final delivery on automatic paths.

## What This Changes

The issue asks OpenClaw to prevent intermediate assistant text between tool calls from being delivered to Slack, Telegram, iMessage, WhatsApp, and similar messaging channels.

## Best Possible Solution

Keep this as the canonical channel-leak bug, preserve existing phase safeguards, and choose a direct/source-chat policy that prevents unphased pre-tool and inter-tool text from leaving automatic delivery paths with regression coverage.

## Reproduction Assessment

Yes. A high-confidence source/test reproduction exists: default direct/source chats can still resolve to automatic delivery, and unphased assistant text is buffered and flushed before tool execution on the block-reply path.

## Solution Assessment

No. Current main is a partial mitigation; the remaining fix requires a maintainer delivery-policy choice for direct/source chats and ordinary unphased assistant text.
