## Summary

The current checkout confirms the issue remains necessary: tests preserve automatic source-reply delivery, commentary suppression depends on structured phase metadata, and fresh production reports identify an additional cron reasoning-delivery gap. The two closing PRs are still open, conflicting, unreviewed fork branches and therefore are implementation candidates rather than a safe canonical resolution.

## What This Changes

Prevent intermediate assistant narration, reasoning, or tool-envelope text from being delivered as visible channel messages during multi-tool turns.

## Best Possible Solution

Define a structured final/user-visible delivery invariant shared by interactive, cron, and subagent paths, preserve reasoning/finality metadata through payload selection, and introduce any direct-chat default change with explicit compatibility and upgrade coverage rather than provider- or regex-specific suppression.

## Reproduction Assessment

Yes, at source level. Current main explicitly defaults ordinary source replies to automatic delivery and suppresses commentary only when phase metadata is present; fresh reports identify matching interactive and cron delivery paths, although this read-only review did not run a live channel reproduction.

## Solution Assessment

No single linked proposal is the best complete solution. The verbose-off stream guard and optional final-tag prompt each cover only part of the surface; the durable fix should preserve structured finality/reasoning metadata across all delivery paths and separately settle direct-chat compatibility.
