## Summary

Keep open: current main has real partial safeguards for commentary-phase and message-tool-only paths, but source and tests still leave automatic direct/source replies, unphased pre-tool block flushing, and cron announce payload selection as plausible leak paths. Recent production comments and the open partial PR show this remains the canonical cross-channel tracker rather than something safely closable.

## What This Changes

The issue asks OpenClaw to stop routing intermediate assistant text between tool calls to messaging channels as user-visible messages.

## Best Possible Solution

Keep this issue open as the canonical tracker until maintainers choose and implement a compatible final-only or explicit-send policy with regression tests for direct chats, group/topic channels, and cron/announce delivery.

## Reproduction Assessment

Yes, with medium confidence from source: direct chats can still use automatic source delivery, unphased assistant text can be buffered and flushed before tool execution, and cron announce payload selection does not preserve reasoning metadata. I did not run a live messaging-channel reproduction in this read-only pass.

## Solution Assessment

No. Current main has useful partial safeguards, but the remaining fix needs a maintainer decision because making direct chats or all channel turns final-only/message-tool-only by default is compatibility-sensitive for existing bare-text reply workflows.
