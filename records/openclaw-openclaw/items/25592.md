## Summary

Keep open. Current main contains useful partial safeguards, but the central product bug remains: direct chats still default to automatic source delivery, unphased assistant text can still enter block/final reply assembly, and the issue context includes recent shipped-stable reproductions plus open implementation PRs using closing syntax.

## What This Changes

The issue requests that intermediate assistant text produced before or between tool calls remain internal and not be routed as visible Slack, Telegram, iMessage, WhatsApp, or similar channel messages.

## Best Possible Solution

Keep this issue as the canonical channel-leak bug. Maintainers should choose a direct-chat-safe delivery policy, such as message-tool-only for more source chats, configurable final-tag enforcement, or another final-only guard, then add regression coverage for unphased assistant text before and between tool calls in streaming and final reply assembly.

## Reproduction Assessment

Yes. A high-confidence source/test reproduction exists: direct chats resolve to automatic delivery, and current tests still assert that unphased assistant text can emit a visible block reply and populate assistantTexts.

## Solution Assessment

No. Current main is a partial mitigation for commentary-phase and group/channel cases, but it does not yet solve the direct-chat and unphased-text paths described in the newest reports.
