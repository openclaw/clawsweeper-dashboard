## Summary

Keep open. Current main and the latest shipped tag still preserve the observable leak path: unphased assistant text can enter the block reply buffer, current tests expect pre-tool buffered text to flush, and automatic source reply delivery remains reachable unless a stricter visible-reply mode is selected.

## What This Changes

The issue asks OpenClaw to stop sending intermediate assistant text between tool calls to external messaging channels unless it is an intentional final or explicit message-tool reply.

## Best Possible Solution

Keep this as the canonical channel-leak bug and land a maintainer-approved final-only, explicit-send, or equivalent delivery policy with upgrade-safe defaults plus regression and live channel proof.

## Reproduction Assessment

## Solution Assessment

No. Current main contains useful partial mitigations for commentary-phase text, but the complete fix still needs a maintainer policy decision for final-only or explicit-send behavior and upgrade-safe defaults.
