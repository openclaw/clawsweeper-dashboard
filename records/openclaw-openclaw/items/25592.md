## Summary

Keep open. Current main and the latest shipped tag still have automatic visible delivery paths where ordinary unphased assistant text can be buffered and flushed to channels; commentary-phase suppression is a useful partial fix, but it does not resolve the broader inter-tool text leak.

## What This Changes

The issue requests that assistant text produced before or between tool calls stay internal instead of being delivered as visible Slack, Telegram, iMessage, WhatsApp, or similar channel messages.

## Best Possible Solution

Keep this as the canonical channel-leak bug and land a maintainer-approved final-only, explicit-send, or equivalent delivery policy with regression coverage and live Slack/Telegram/WhatsApp/iMessage proof.

## Reproduction Assessment

Yes, at source level. Current main still lets unphased assistant chunks enter the block reply buffer, a current test asserts pre-tool buffered text is delivered, and automatic visible delivery remains the default for unresolved source-chat paths.

## Solution Assessment

No. Current main contains important partial mitigations for commentary-phase text, but the complete fix still needs a maintainer policy decision for final-only or explicit-send behavior and upgrade-safe defaults.
