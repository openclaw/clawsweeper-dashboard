## Summary

Keep open. Current main has useful partial mitigations, but direct/source chats still default to automatic visible delivery and ordinary unphased assistant text can still be buffered and flushed before tool execution; the open implementation PRs should be reviewed rather than closing this issue early.

## What This Changes

The issue asks OpenClaw to stop intermediate assistant text between tool calls from being delivered as user-visible messages in Slack, Telegram, iMessage, WhatsApp, and similar channels.

## Best Possible Solution

Keep this as the canonical channel-leak bug, review the open subset PRs, and make a maintainer delivery-policy decision with regression coverage for direct/source chats and unphased pre-tool/inter-tool assistant text.

## Reproduction Assessment

Yes. A high-confidence source/test reproduction exists: direct/source chats can resolve to automatic delivery, unphased assistant text enters the block-reply buffer, and current coverage asserts that buffered text flushes before tool execution.

## Solution Assessment

No. Current main is a partial mitigation; the complete fix needs a maintainer choice around direct/source visible-reply policy, final-only semantics, or a configurable enforcement path.
