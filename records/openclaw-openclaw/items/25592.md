## Summary

Keep open. Current main has meaningful partial mitigations, but the broad leak class is still not fully resolved: direct chats still default to automatic delivery, unphased assistant text is still buffered and deliverable on automatic paths, and final payload assembly still treats every non-empty assistant text as a reply item.

## What This Changes

The issue asks OpenClaw to prevent intermediate assistant text between tool calls from being delivered to messaging channels, allowing only intentional final replies or explicit message-tool sends.

## Best Possible Solution

Keep this as the canonical channel-leak bug, preserve the existing commentary-phase safeguards, and choose a direct-chat-safe final-only or message-tool-only policy with regression coverage for unphased pre-tool and inter-tool text.

## Reproduction Assessment

Yes. A high-confidence source/test reproduction exists: direct chats still resolve to automatic delivery by default, and current source still buffers unphased assistant text for visible block/final delivery paths.

## Solution Assessment

No. Current main is a partial solution; the remaining fix needs a maintainer delivery-policy decision for direct chats and ordinary unphased text before changing default behavior.
