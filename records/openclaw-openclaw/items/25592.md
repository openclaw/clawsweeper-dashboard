## Summary

Keep open. Current main has useful commentary/final-answer safeguards and Codex/direct-chat mitigations, but ordinary unphased assistant text can still be buffered and delivered on automatic source-reply paths.

## What This Changes

The issue requests that assistant text produced before or between tool calls stay internal instead of being posted to Slack, Telegram, iMessage, WhatsApp, or similar channels.

## Best Possible Solution

Keep this as the canonical channel-leak bug, review the two relevant open PRs, and make a maintainer delivery-policy decision with regression coverage for direct/source chats and unphased pre-tool/inter-tool text.

## Reproduction Assessment

Yes. A high-confidence source/test reproduction exists: direct/source chats can resolve to automatic delivery, unphased assistant text enters the block reply buffer, and current coverage asserts that buffered text flushes before tool execution.

## Solution Assessment

No. Current main is a partial mitigation; the complete fix needs a maintainer choice around direct/source visible-reply policy and how to treat ordinary unphased assistant text before or between tool calls.
