## Summary

Keep open. Current main still has a source-reproducible leak path for automatic direct/source delivery: unphased assistant text can be buffered and flushed before tool execution, while direct chats still default to automatic visible replies; the issue also has active recent repro/triage comments and open implementation PRs with closing syntax.

## What This Changes

The issue asks OpenClaw to prevent intermediate assistant text between tool calls from being delivered as visible messaging-channel replies.

## Best Possible Solution

Keep this as the canonical channel-leak bug and land a direct/source-safe final-only or explicit-send policy with regression coverage for unphased assistant text before and between tool calls.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: direct/source chats can still resolve to automatic delivery, unphased assistant text is buffered, and current tests assert that buffered text is delivered when a tool starts.

## Solution Assessment

No. Current main is a partial mitigation; the best path is to review the open layer-specific PRs and make the remaining direct/source delivery policy decision before closing this issue.
