## Summary

Keep open. Current main still has a source-reproducible automatic direct/source delivery path where unphased assistant text can be buffered and delivered before or between tool calls, while the two real closing implementation PRs are still open and need maintainer disposition.

## What This Changes

The issue asks OpenClaw to prevent assistant text emitted before or between tool calls from being routed to messaging channels unless it is an intentional final or explicit sent response.

## Best Possible Solution

Keep this as the canonical channel-leak bug, review the two open closing PRs, and choose a direct/source-safe final-only or explicit-send policy with regression coverage for unphased assistant text before and between tool calls.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: direct chats can still default to automatic delivery, unphased assistant text is buffered, and current tests assert a buffered pre-tool chunk is delivered when tool execution starts.

## Solution Assessment

No. Current main contains important partial fixes for commentary-phase output, but closing now would leave the direct/source automatic delivery and unphased pre-tool text policy unresolved.
