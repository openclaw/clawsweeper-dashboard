## Summary

Keep open. Current main has important partial mitigations, but direct/source chats can still resolve to automatic visible delivery and unphased assistant text can still be buffered and flushed before tool execution; two open implementation PRs also still use closing syntax for this issue.

## What This Changes

The issue asks OpenClaw to stop routing intermediate assistant text emitted before or between tool calls to Slack, Telegram, iMessage, WhatsApp, and other messaging channels.

## Best Possible Solution

Keep this as the canonical channel-leak bug, review the two live implementation PRs, and choose a direct/source delivery policy or final-only guard with regression coverage for unphased assistant text before and between tool calls.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: direct/source chats can resolve to automatic delivery, unphased assistant text is buffered, and current tests assert that the buffered text is delivered before tool execution starts.

## Solution Assessment

No. Current main is a partial mitigation; the safest path is to review the open layer-specific PRs and make the remaining direct/source visible-reply policy decision before closing this issue.
