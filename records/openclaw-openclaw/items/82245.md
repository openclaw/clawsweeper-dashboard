## Summary

Keep this PR open: it addresses a real source-reproducible bug, but the patch can let a broad `agents.defaults.models["provider/*"]` runtime override mask an exact configured provider-model runtime, and the external PR proof is currently only unit/seam test output rather than real behavior proof.

## What This Changes

The PR updates model runtime policy resolution and tests so `agents.defaults.models` provider wildcard entries can supply `agentRuntime` during harness selection.

## Best Possible Solution

Keep the wildcard runtime behavior, but resolve exact provider/model runtime policy before applying provider wildcard fallback, add regression coverage for that mixed-precedence case, and refresh proof with a real runtime-selection command or log.

## Reproduction Assessment

Yes, at source level: current main only exact-matches `agents.defaults.models` runtime entries, so `vllm/*` does not apply to `vllm/qwen-local`. I did not run tests because this review was required to keep the checkout read-only.

## Solution Assessment

No, not yet; wildcard runtime fallback is the right direction, but the implementation should preserve exact provider/model runtime precedence before a broad wildcard fallback. The PR also needs real behavior proof beyond targeted tests before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.76
