## Summary

Keep open: the code change is a focused fix for a source-reproducible runtime-policy gap, but the external PR still needs after-fix real behavior proof and a small docs alignment before merge.

## What This Changes

The PR changes agent runtime policy resolution so `agents.defaults.models` and per-agent model-map `provider/*` entries can supply an `agentRuntime` fallback after exact model entries and before provider-level policy.

## Best Possible Solution

Land the narrow resolver fix after the contributor adds real runtime-policy proof and the docs mention `provider/*` runtime policy with the same exact-over-wildcard precedence the tests lock in.

## Reproduction Assessment

Yes. Source inspection on current main shows a concrete path: `vllm/*` reaches `modelKeyMatches`, which requires the key's model segment to equal the concrete model id, so the `agentRuntime` entry is skipped.

## Solution Assessment

Yes for the code direction. Exact `provider/model` first, then matching `provider/*`, then provider-level `agentRuntime` is the narrowest maintainable ordering, but proof and docs still need follow-through.
