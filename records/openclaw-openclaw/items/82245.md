## Summary

Keep open: current main still has the wildcard runtime-policy bug, but this PR head should not merge yet because wildcard fallback can outrank an exact configured provider-model runtime and the supplied proof is test-only.

## What This Changes

The PR adds provider-wildcard matching to `resolveModelRuntimePolicy` plus resolver and harness-selection regression tests for wildcard `agentRuntime` entries.

## Best Possible Solution

Resolve exact model policies across agent/default maps and configured provider models before applying provider-wildcard fallback, then prove the path with targeted tests and a redacted real runtime-selection command or log.

## Reproduction Assessment

Yes. Source inspection on current main shows `provider/*` keys are routed through exact model-key matching, so `vllm/*` does not apply to `vllm/qwen-local` during runtime-policy resolution.

## Solution Assessment

No, not yet. Wildcard fallback is the right direction, but the PR should preserve exact configured provider-model runtime precedence and add real behavior proof before merge.
