## What This Changes

The issue requests preventing Gateway OOM/restart loops from isolated cron jobs exceeding Node/V8 heap, either by reducing cron memory pressure, throttling catch-up work, or supporting heap-size service startup configuration.

## Best Possible Solution

Keep the shipped cron lifecycle cleanup, replay throttling, diagnostics, and wrapper-based heap override path; reopen only with new evidence that current v2026.5.22+ still OOMs under the same cron workload.

## Reproduction Assessment

## Solution Assessment

Yes. The current solution fixes the memory-retention root cause, keeps catch-up execution bounded, and preserves the safer wrapper-based heap override instead of adding broad raw `NODE_OPTIONS` service passthrough.
