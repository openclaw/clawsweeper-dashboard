## Summary

Keep this open. Current main has meaningful cron catch-up, cron concurrency, isolated-session cleanup, and session-store bloat mitigations, but it still lacks the durable Gateway heap-headroom/config behavior that would address the reported Node V8 heap ceiling.

## What This Changes

The issue reports Gateway heap exhaustion during isolated cron workloads and asks for lower memory use, a default Node heap limit, or throttled cron catch-up.

## Best Possible Solution

Land a reviewed Gateway service heap-headroom path, likely a revised #42853 or equivalent, while keeping the existing cron/session pressure mitigations in place.

## Reproduction Assessment

Unclear for the full OOM workload: this review did not run the WSL2 cron scenario. Yes for the remaining source-level behavior: current main has no generated Gateway heap option, no nodeMaxOldSpaceMb config field, and no Node --max-old-space-size insertion.

## Solution Assessment
