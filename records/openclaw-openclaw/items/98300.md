## Summary

Keep open: current main has runtime Tool Search directory-mode schema deferral, but it still does not expose the requested public plugin SDK API for name-and-description-only tool registration with lazy schema and handler hydration.

## What This Changes

The issue requests a public plugin API for deferred model-facing tool registration where only name and description are always visible and full schema/handler loading happens on demand.

## Best Possible Solution

Use the existing Tool Search directory mode and agent-core deferred resolver as the implementation baseline, then add one narrow public plugin SDK contract only if maintainers accept this API gap.
