## Summary

Keep open: current main and v2026.6.11 already provide Tool Search directory-mode schema deferral for plugin/runtime catalogs, but the plugin SDK still lacks the requested public name-and-description-only registration API with a schema resolver.

## What This Changes

Request for an additive plugin API that advertises model-facing tool names and descriptions up front while loading full input schemas only when selected.

## Best Possible Solution

Use the shipped Tool Search directory mode as the baseline and have maintainers decide whether a narrow plugin SDK schema-resolver contract is still needed for truly dynamic plugin catalogs.

## Reproduction Assessment

Not applicable; this is an additive plugin API request rather than a report of broken existing behavior. Source review confirms the requested API surface is absent while related runtime directory-mode behavior exists.

## Solution Assessment

Unclear until maintainers choose the API direction. The best implementation, if accepted, should reuse the existing Tool Search catalog and agent-core deferred resolver instead of creating a second tool-discovery path.
