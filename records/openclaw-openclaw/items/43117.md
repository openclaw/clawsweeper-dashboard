Keep open. The issue requests multimodal Gemini Embedding 2 memory search for conversation attachments and media, mixed-modal retrieval, and configurable embedding output dimensions.

## Best Possible Solution

Extend the existing structured multimodal memory foundation only after maintainers define a public PDF/video modality contract and a separate opt-in retained-media memory policy with deletion and provider-upload limits.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source review gives high confidence that current main supports Gemini dimensions plus image/audio `extraPaths`, while PDF/video modalities and retained chat-attachment indexing are absent.

## Solution Assessment

Unclear until maintainers settle the media-retention and provider-upload contract. The existing structured embedding path is the right base, but broad attachment indexing should not be added without explicit opt-in, deletion semantics, and live provider contract proof.
