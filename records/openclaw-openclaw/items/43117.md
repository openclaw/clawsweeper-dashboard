## Summary

Keep open. Current main covers the shipped subset of Gemini output dimensions plus opt-in image/audio files under `memorySearch.extraPaths`, but PDF/video modalities and remembered chat-attachment indexing are still absent; the closed video/PDF PR at https://github.com/openclaw/openclaw/pull/45256 is useful prior art, not a merged fix.

## What This Changes

Request to extend `memorySearch` so Gemini Embedding 2 can index multimodal files and remembered attachments with output-dimensionality control and mixed-modal retrieval.

## Best Possible Solution

Keep the shipped image/audio `extraPaths` foundation, then add PDF/video and remembered-media indexing only behind an explicit public modality and retained-media policy with deletion, retention, and provider-upload limits.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main supports Gemini dimensions plus image/audio `extraPaths`, while PDF/video modalities and retained chat-attachment indexing are absent.

## Solution Assessment

Unclear until maintainers settle the media-retention and provider-upload contract. The existing structured embedding path is the right base, but broad attachment indexing should not be added without explicit opt-in, deletion semantics, and live provider contract proof.
