## Summary

Keep open. Current main and the latest shipped release implement Gemini Embedding 2 output dimensions plus opt-in image/audio indexing from `memorySearch.extraPaths`, but PDF/video modalities and remembered chat-attachment indexing remain missing and require product/security review.

## What This Changes

Request to add Gemini Embedding 2 multimodal memory search for media and attachments, configurable embedding dimensions, and cross-modal retrieval.

## Best Possible Solution

Keep the shipped image/audio `extraPaths` foundation, then split the remaining work into a deliberate PDF/video modality contract and a separate opt-in retained-attachment memory design with deletion, audit, and provider-upload limits.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main supports only Gemini dimensions plus image/audio `extraPaths`, while PDF/video and retained attachment indexing remain absent.

## Solution Assessment

Unclear until maintainers settle the media-retention and provider-upload contract. The existing structured embedding path is the right base, but automatically indexing inbound attachments would not be the safest product shape without explicit opt-in and deletion semantics.
