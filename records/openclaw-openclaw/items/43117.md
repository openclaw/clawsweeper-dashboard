## Summary

Keep open. Current main covers the `outputDimensionality` request and opt-in Gemini image/audio indexing from `memorySearch.extraPaths`, but the issue also asks for PDF/video support and remembered chat-attachment indexing, which are still missing and need product/security decisions.

## What This Changes

Request support for Gemini Embedding 2 multimodal memory search, including media embeddings, configurable dimensionality, and cross-modal retrieval for stored history and attachments.

## Best Possible Solution

Keep the current opt-in image/audio `extraPaths` foundation, then deliberately add PDF/video file indexing and a separate opt-in retained-attachment memory contract with clear media retention and remote-upload limits.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main supports only the dimensionality and image/audio `extraPaths` subset while PDF/video and session attachment indexing remain unimplemented.

## Solution Assessment

Unclear until maintainers settle privacy and provider scope. The existing structured embedding path is the right base, but automatically indexing inbound attachments would not be the safest product shape without explicit opt-in and deletion semantics.
