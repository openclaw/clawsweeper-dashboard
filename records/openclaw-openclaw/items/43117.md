## Summary

Keep open. Current main and v2026.6.6 cover output dimensionality plus opt-in Gemini image/audio files under memorySearch.extraPaths, but the requested PDF/video modalities and remembered inbound-attachment indexing path remain outside the public memory contract.

## What This Changes

Request Gemini Embedding 2 multimodal memory search for conversation attachments/media, mixed-modal retrieval, and output-dimensionality configuration.

## Best Possible Solution

Keep this as the canonical tracker: preserve the shipped image/audio extraPaths foundation, refresh the Gemini Embedding 2 contract, and design PDF/video plus remembered-attachment indexing as explicit opt-in memory/security behavior.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.6 only admit image/audio files under memorySearch.extraPaths, while PDF/video and remembered inbound attachments are not accepted by the modality/config contract.

## Solution Assessment

No, the current implementation is only a partial solution. The maintainable path is to extend the existing structured-input foundation after a deliberate modality, provider-limit, and retained-attachment security design.
