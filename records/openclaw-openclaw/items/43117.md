## Summary

Keep open. Current main covers a useful Gemini Embedding 2 subset, but the requested PDF/video modalities and remembered chat-attachment memory path are still missing and need explicit media retention/provider-upload policy.

## What This Changes

Request support for Gemini Embedding 2 multimodal memory search, including media/attachment embeddings, output dimensionality, and cross-modal retrieval.

## Best Possible Solution

Keep the shipped image/audio `extraPaths` foundation, then add explicit PDF/video modality support and a separate opt-in retained-attachment memory contract with clear consent, deletion, and upload limits.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that output dimensions and image/audio `extraPaths` indexing exist while PDF/video and session attachment indexing remain absent.

## Solution Assessment

Unclear until maintainers settle privacy and provider scope. The existing structured embedding path is a good base, but silently indexing every inbound attachment would not be the safest product shape.
