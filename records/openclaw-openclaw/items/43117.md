## Summary

Keep open. Current main has the Gemini Embedding 2/output-dimensionality base and opt-in image/audio indexing from `memorySearch.extraPaths`, but the requested PDF/video modalities and retained chat-attachment indexing path are still not implemented.

## What This Changes

Request to extend `memorySearch` so Gemini Embedding 2 can embed conversation media and attachments across text, image, video, audio, and PDF content with configurable output dimensionality.

## Best Possible Solution

Keep the shipped Gemini image/audio `extraPaths` foundation, then explicitly design PDF/video modality support and a separate opt-in retained-attachment memory contract before implementation.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Current-main source verification is high-confidence: output dimensions and image/audio `extraPaths` indexing exist, while PDF/video and session attachment indexing are absent.

## Solution Assessment

Unclear until maintainers settle privacy and scope. The existing structured embedding path is the right base, but the remaining work should add explicit media contracts rather than silently indexing every inbound attachment.
