## Summary

Keep open: current main and the latest release cover Gemini Embedding 2 dimensions plus opt-in image/audio indexing from `memorySearch.extraPaths`, but PDF/video modalities and retained chat-attachment indexing are still outside the shipped memory contract.

## What This Changes

The issue asks `memorySearch` to use Gemini Embedding 2 for cross-modal media indexing/search and configurable embedding dimensions.

## Best Possible Solution

Keep the shipped image/audio `extraPaths` foundation, then define an explicit PDF/video modality contract and a separate opt-in retained-attachment ingestion path with deletion, consent, provider-upload, and result-display limits.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main and `v2026.5.22` support Gemini dimensions plus image/audio `extraPaths`, while PDF/video and retained chat-attachment indexing remain absent.

## Solution Assessment

Unclear until maintainers settle the media-retention and provider-upload contract. The existing structured embedding path is the right base, but broad attachment indexing should be explicit opt-in with deletion semantics and current Gemini API proof.
