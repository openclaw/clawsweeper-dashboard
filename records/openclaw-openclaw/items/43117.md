## Summary

Keep open: current main implements Gemini Embedding 2 dimensionality plus opt-in image/audio indexing from `memorySearch.extraPaths`, but PDF/video modalities and remembered chat-attachment ingestion are still absent and need maintainer product/security decisions.

## What This Changes

The issue asks `memorySearch` to use Gemini Embedding 2 multimodal embeddings for stored media and attachments, including configurable output dimensions and mixed-modal retrieval.

## Best Possible Solution

Keep the shipped image/audio `extraPaths` foundation, then add explicit PDF/video modality support and a separate opt-in retained-attachment ingestion path with consent, deletion, provider-upload, and result-display limits.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main supports Gemini dimensions plus image/audio `extraPaths`, while PDF/video and retained chat-attachment indexing remain absent.

## Solution Assessment

Unclear until maintainers settle the media-retention and provider-upload contract. The existing structured embedding path is the right base, but broad attachment indexing should be explicit opt-in with deletion semantics and current Gemini API proof.
