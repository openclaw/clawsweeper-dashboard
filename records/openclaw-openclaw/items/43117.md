## Summary

Keep open. Current main implements Gemini Embedding 2, output dimensions, and opt-in image/audio indexing from `memorySearch.extraPaths`, but it still does not index session/chat attachments and video/PDF remain outside the shipped multimodal memory scope.

## What This Changes

Request support for Gemini Embedding 2 multimodal `memorySearch`, including media/attachment indexing, configurable output dimensions, and cross-modal retrieval.

## Best Possible Solution

Keep the shipped Gemini/`extraPaths` base, then add a documented opt-in path that converts eligible transcript media into existing structured `EmbeddingInput` chunks and deliberately scopes any video/PDF support.

## Reproduction Assessment

Yes. Source inspection shows a media-only transcript message is skipped because session indexing only collects string/text blocks, while eligible `extraPaths` image/audio files can become structured inline-data embedding chunks.

## Solution Assessment

Unclear until maintainers settle session-media privacy and scope. The current provider and `extraPaths` implementation is the right base; the remaining work should bridge transcript media into that structured input path rather than reworking the Gemini provider.
