## Summary

Keep open. Current main implements Gemini Embedding 2 output dimensions and opt-in image/audio indexing from `memorySearch.extraPaths`, but the requested PDF/video modalities and remembered chat-attachment indexing are still missing and need product/security review.

## What This Changes

The issue requests multimodal memory search with Gemini Embedding 2, including media attachments, mixed-modal retrieval, and configurable embedding dimensions.

## Best Possible Solution

Keep the shipped image/audio `extraPaths` foundation, then split the remaining work into a deliberate PDF/video modality contract and a separate opt-in retained-attachment memory design.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main supports only Gemini dimensions plus image/audio `extraPaths`, while PDF/video and retained attachment indexing remain absent.

## Solution Assessment

Unclear until maintainers settle the media-retention and provider-upload contract. The existing structured embedding path is the right base, but automatically indexing inbound attachments would not be the safest product shape without explicit opt-in and deletion semantics.
