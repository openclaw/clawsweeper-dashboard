## Summary

Keep open. Current main implements the Gemini Embedding 2 text/vector-dimension path and opt-in image/audio indexing from `memorySearch.extraPaths`, but the request still includes PDF/video support and remembered chat-attachment indexing, which are not implemented and need product/security review.

## What This Changes

Request support for multimodal `memorySearch` with Gemini Embedding 2, including media embeddings, mixed-modal retrieval, and configurable output dimensionality.

## Best Possible Solution

Keep the current image/audio `extraPaths` foundation, then split the remaining work into a deliberate PDF/video modality contract and a separate opt-in retained-attachment memory design with clear provider-upload and deletion rules.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main supports only the shipped Gemini dimensions plus image/audio `extraPaths` subset while PDF/video and retained attachment indexing remain absent.

## Solution Assessment

Unclear until maintainers settle the media-retention and provider-upload contract. The existing structured embedding path is the right base, but automatically indexing inbound attachments would not be the safest product shape without explicit opt-in and deletion semantics.
