- `P2`: This is a normal-priority memory-search feature gap with useful partial support already shipped and limited blast radius to opt-in multimodal memory users.
- `impact:session-state`: The issue affects what media can become searchable memory context and therefore what later `memory_search` can retrieve.

## Summary

Keep open: current main and the latest release already cover the useful image/audio `extraPaths` subset and `outputDimensionality`, but the requested PDF/video modalities and remembered inbound-attachment indexing contract remain missing and need maintainer product/security direction.

## What This Changes

The issue requests Gemini Embedding 2 multimodal memory search for text, images, video, audio, PDFs, mixed-modal queries, remembered attachments, and configurable output dimensionality.

## Best Possible Solution

Keep the shipped image/audio `extraPaths` foundation, then split or design the remaining PDF/video modality contract and explicit remembered-attachment ingestion path with provider limits, consent/retention rules, docs, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction, but yes at source level: current main and v2026.6.6 only expose image/audio multimodal indexing under `extraPaths`, while PDF/video and remembered inbound attachments are not accepted by the modality/config/session-indexing contract.

## Solution Assessment

No automatic fix is appropriate yet. The current implementation is a safe partial foundation, but the remaining work needs a maintainer-approved media retention and provider-format contract before code changes.
