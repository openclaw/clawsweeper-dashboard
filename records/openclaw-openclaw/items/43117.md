## Summary

Keep open. Current main implements Gemini Embedding 2, output dimensionality, and opt-in image/audio indexing from `memorySearch.extraPaths`, but the requested PDF/video modalities and remembered chat-attachment indexing path are still missing.

## What This Changes

The issue requests multimodal Gemini Embedding 2 memory search for images, chat attachments, video, audio, PDFs, configurable output dimensionality, and cross-modal retrieval.

## Best Possible Solution

Keep the shipped image/audio `extraPaths` foundation, then design PDF/video modality support and a separate explicit retained-attachment memory contract before implementation.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that output dimensions and image/audio `extraPaths` indexing exist while PDF/video and session attachment indexing remain absent.

## Solution Assessment

Unclear until maintainers settle privacy and scope. The existing structured embedding path is a good base, but the remaining work should add explicit media contracts rather than silently indexing every inbound attachment.
