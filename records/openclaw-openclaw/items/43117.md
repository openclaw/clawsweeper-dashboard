## Summary

Keep open: current main and v2026.6.10 already cover `outputDimensionality` plus opt-in Gemini image/audio indexing from `memorySearch.extraPaths`, but the requested PDF/video modalities and remembered inbound-attachment indexing path are still missing and require maintainer product/security direction.

## What This Changes

The issue requests Gemini Embedding 2 multimodal memorySearch support for media embeddings, dimensionality control, mixed-modal retrieval, and attachments remembered from conversation history.

## Best Possible Solution

Keep the shipped image/audio `extraPaths` foundation, then split or explicitly approve the remaining work into a PDF/video modality contract and a retained-attachment ingestion path with privacy, retention, provider-limit, and documentation coverage.

## Reproduction Assessment

Yes, at source level: current main accepts only image/audio multimodal memory under `extraPaths`, while schema, classifier, docs, and session indexing exclude PDF/video and inbound media attachments. I did not run a live Gemini account because this read-only review has no provider credentials.

## Solution Assessment

No automatic fix is appropriate yet. The existing image/audio implementation is a safe partial foundation, but the remaining work needs maintainer-approved media retention, modality, and provider-format contracts before code changes.
