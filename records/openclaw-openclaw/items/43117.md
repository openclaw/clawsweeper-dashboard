## Summary

Keep open. Current main implements Gemini Embedding 2 output dimensions and opt-in image/audio indexing from `memorySearch.extraPaths`, but it still does not cover session/chat attachment indexing or the requested video/PDF memory modalities.

## What This Changes

Request support for Gemini Embedding 2 multimodal memory search, including attachment/media indexing, configurable output dimensionality, and cross-modal retrieval.

## Best Possible Solution

Keep the shipped Gemini/`extraPaths` foundation, then design explicit public contracts for retained session attachments, PDF/video modalities, size/MIME limits, provider uploads, deletion, and result presentation.

## Reproduction Assessment

Not applicable as a bug reproduction. Current-main source verification is high-confidence: image/audio `extraPaths` support exists, while session attachment indexing and PDF/video modalities are absent from code, schema, docs, and tests.

## Solution Assessment

Unclear until maintainers settle privacy and scope. The current structured `EmbeddingInput` and Gemini provider implementation are the right base, but the remaining work should bridge opted-in retained media into that path rather than silently indexing every inbound attachment.
