## Summary

Keep open. Current main covers `outputDimensionality` plus opt-in Gemini image/audio indexing from `memorySearch.extraPaths`, but the requested PDF/video modalities and remembered chat-attachment indexing path are still missing and need product/security decisions.

## What This Changes

Request multimodal Gemini Embedding 2 memorySearch support for media/attachments, output dimensionality, and cross-modal retrieval.

## Best Possible Solution

Keep the shipped image/audio `extraPaths` foundation, update the Gemini model contract deliberately, then add PDF/video extraPaths support and a separate opt-in retained-attachment memory contract with clear consent, deletion, and upload limits.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main supports output dimensions and image/audio extraPaths indexing while PDF/video and session attachment indexing remain absent.

## Solution Assessment

Unclear until maintainers settle privacy and provider scope. The existing structured embedding path is a good base, but silently indexing inbound attachments would not be the safest product shape.
