## Summary

Keep open: current main and the latest release cover `outputDimensionality` plus opt-in image/audio indexing from `memorySearch.extraPaths`, but PDF/video modalities and remembered inbound-attachment indexing remain missing and need maintainer product/security direction.

## What This Changes

The issue requests Gemini Embedding 2 multimodal memory search support for media and attachments, mixed-modal retrieval, and configurable embedding dimensionality.

## Best Possible Solution

Keep the shipped image/audio `extraPaths` foundation, then add a maintainer-approved PDF/video modality contract and a separate explicit retained-attachment ingestion path with privacy, retention, provider-limit, and documentation coverage.

## Reproduction Assessment

Yes, at source level: current main accepts only image/audio multimodal memory under `extraPaths`, while schema, classifier, docs, and session indexing exclude PDF/video and inbound media attachments. I did not run a live Gemini account because this read-only review has no provider credentials.

No automatic fix is appropriate yet. The existing image/audio implementation is a safe partial foundation, but the remaining work needs a maintainer-approved media retention and provider-format contract before code changes.
