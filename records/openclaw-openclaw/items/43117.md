## Summary

Keep open. Current main and the latest release ship the useful Gemini Embedding 2 subset for output dimensionality plus image/audio files under `memorySearch.extraPaths`, but the requested PDF/video modalities and remembered inbound-attachment indexing path are still missing and need product/security decisions.

## What This Changes

The issue requests Gemini Embedding 2 multimodal memorySearch support for media/attachments, output dimensionality, and cross-modal retrieval.

## Best Possible Solution

Keep this as the canonical tracker for the remaining work: preserve the shipped image/audio extraPaths foundation, add a deliberate PDF/video modality contract only after provider-limit review, and design remembered-attachment indexing as an explicit opt-in retention/security flow.

## Reproduction Assessment

Yes, at source level: current main and `v2026.6.6` only support image/audio files under `memorySearch.extraPaths`, while PDF/video and remembered inbound attachments are not admitted by the modality/config contract.

## Solution Assessment

No, the current implementation is only a partial solution to this issue. The maintainable path is to keep the existing structured-input foundation, then separately design PDF/video modality support and safe retained-attachment ingestion.
