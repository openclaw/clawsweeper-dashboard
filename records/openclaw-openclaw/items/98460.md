## Summary

Keep open: current main still restores the selected Control UI session on startup, and no existing cold-start or dormant-start setting/query parameter was found. The request is clear and aligned with frontend ergonomics, but it is a new user preference and needs maintainer product approval before implementation.

## What This Changes

Requested a Control UI preference or query parameter that loads the chat dormant instead of restoring the last session on page refresh.

## Best Possible Solution

If maintainers accept the product direction, add a narrow browser-local Control UI startup-mode preference that preserves auto-restore as the default and skips the startup history restore only for the opted-in browser/session.

## Reproduction Assessment

Not applicable; this is a feature request for an opt-out from documented auto-restore behavior, not a failing current contract.
