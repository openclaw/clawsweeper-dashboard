## Summary

Keep open: current main has partial media-preview infrastructure, but the WebUI still does not convert `media://inbound/...` image references into the existing authenticated assistant-media proxy URL, so the reported inline-preview gap remains actionable.

## What This Changes

The issue requests inline WebUI previews for inbound channel image attachments that are stored and surfaced as `media://inbound/...` media references.

## Best Possible Solution

Route `media://inbound` image references through the existing authenticated `__openclaw__/assistant-media` path in the Control UI renderer, with focused coverage for the combined renderer-plus-media-reference case.

## Reproduction Assessment

Yes, source-reproducible: `media://inbound/*.png` is classified as an image path but is not considered proxyable, so the renderer leaves a browser-unloadable `media://` URL instead of using the gateway media route. I did not run a live WebUI reproduction in this read-only pass.

## Solution Assessment

Yes, the best fix is a narrow shared WebUI rendering repair that reuses the existing assistant-media route and ticket flow. Adding a new public `/media` file-serving route would duplicate security-sensitive gateway behavior already present in current main.

## Repair Work Prompt

Fix the Control UI inbound image preview gap reported at https://github.com/openclaw/openclaw/issues/89591. Current main already has `/__openclaw__/assistant-media` and gateway tests for `media://inbound/<id>`, but `ui/src/ui/chat/grouped-render.ts` does not treat `media://inbound/...` image references as proxyable, so the WebUI can leave an unloadable `media://` image URL. Reuse the existing assistant-media route, metadata check, auth token, and media-ticket flow; do not add a new public `/media` route or broaden arbitrary local file serving. Preserve existing filesystem-root checks for local paths, preserve non-image document/audio/video attachment behavior, and add focused renderer coverage for `MediaPath` / `MediaPaths` with `media://inbound/*.png` or `.jpg`. Do not edit `CHANGELOG.md`.
