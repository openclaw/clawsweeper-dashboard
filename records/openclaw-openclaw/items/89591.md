## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/90172, reusing the existing authenticated `__openclaw__/assistant-media` route for canonical inbound refs instead of adding a new public media route.

## Solution Assessment

Yes, the best fix shape is to teach the WebUI renderer to recognize canonical inbound media refs and route them through the existing authenticated, ticketed assistant-media path.
