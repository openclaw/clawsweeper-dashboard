## Summary

Keep open: current main has adjacent config safety, Nix whole-config immutability, and Policy-plugin conformance, but it does not implement a startup-loaded immutable operator policy overlay or locked-policy write rejection. The issue is maintainer-owned, security-sensitive config/product work, so it should stay open for maintainer direction rather than close as implemented or queue an automated fix.

## What This Changes

Add an immutable operator-owned policy/defaults layer that loads on startup, takes deterministic precedence over mutable config where applicable, is visible in status/doctor, and blocks writes that weaken locked policy.

## Best Possible Solution

Define the maintainer-approved core policy contract first, then implement one bounded config/policy overlay path with startup loading, deterministic precedence, status/doctor visibility, conflict rejection for config writes, docs, and upgrade-focused tests.
