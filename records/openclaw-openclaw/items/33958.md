## Summary

Keep open: this is a maintainer-authored, protected security/product feature request, and current main has adjacent policy-conformance and Nix immutable-config pieces but not the requested immutable startup policy overlay with precedence, status/doctor visibility, and write-lock enforcement.

## What This Changes

The issue requests an immutable operator-owned defaults/policy layer loaded at startup, merged above mutable config, exposed read-only in status/doctor, and protected from agent/runtime writes.

## Best Possible Solution

Have maintainers decide the permanent security boundary first, then either extend the Policy plugin into startup enforcement with explicit precedence, status/doctor output, write preflight, tests, and docs, or document that immutable deployment policy remains Nix/config-owner managed rather than a new runtime overlay.

## Solution Assessment

A maintainer design decision should choose whether this belongs in core config loading, the bundled Policy plugin, or deployment-managed config ownership.
