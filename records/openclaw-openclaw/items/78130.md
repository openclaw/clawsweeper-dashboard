## Summary

This PR should stay open, but it is not merge-ready: it targets a real memory-core session-corpus contamination path, yet the patch filters grounded candidate staging instead of the actual session-transcript recall ingestion path, conflicts with current dreaming-phase expectations, and still lacks external-contributor real behavior proof. Related hardening has landed in https://github.com/openclaw/openclaw/pull/68774, but the canonical session-corpus pollution issue remains open at https://github.com/openclaw/openclaw/issues/77831.

## What This Changes

The PR adds an `isSessionCorpusPath()` helper, skips session-corpus paths during grounded candidate staging and promotion ranking, and adds focused short-term-promotion tests.

## Best Possible Solution

Revise this branch or a replacement so session-corpus artifacts can support internal dreaming analysis without entering durable promotion or memory-search pollution, update the dreaming-phase expectations accordingly, and verify against a workspace containing session-corpus data.

## Reproduction Assessment

Yes, at source level: current main writes session-corpus files, feeds those memory results into `recordShortTermRecalls()`, and the short-term path predicate accepts session-corpus paths. I did not run a live multi-day dreaming workspace reproduction in this read-only review.

## Solution Assessment

No. The intended path-based exclusion is plausible, but this patch misses the actual recall ingestion path named in the PR body and leaves current dreaming-phase tests with a contradictory session-corpus ranking contract.
