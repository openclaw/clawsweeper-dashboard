## Summary

Keep open: the PR targets a valid memory-core bug, but the current patch is incomplete and still needs contributor real behavior proof. Current main and docs support keeping dreaming artifacts out of durable promotion, yet this diff misses the actual session transcript recall ingestion path and conflicts with existing dreaming-phase expectations.

## What This Changes

The PR adds a session-corpus path helper, skips those paths during grounded candidate staging and ranking, and adds focused short-term-promotion tests.

## Best Possible Solution

Define session-corpus as internal dreaming evidence, block it from durable promotion and memory/search pollution at the right ingestion/ranking boundary, update dreaming-phase tests/docs to match, and verify on a workspace containing corpus data.

## Solution Assessment

No. The path-helper and ranker direction is plausible, but this patch guards grounded staging instead of the actual session transcript recall ingestion path and leaves current dreaming-phase tests contradictory.
