## Summary

Keep open. Current main has partial queue-timeout, lane-snapshot, and diagnostic-recovery plumbing, but the reported Discord cron/heartbeat session-lane stall is not fully resolved: active-looking session lanes are preserved, heartbeat still skips busy target lanes, and no supported lane-status command exists.

## What This Changes

Request to fix intermittent Discord session-lane stalls by verifying terminal run paths release their lane, adding stale-lane recovery, and exposing lane diagnostics.

## Best Possible Solution

Add liveness-aware core recovery that can distinguish stale bookkeeping from real active work, abort or release only after that policy is satisfied, cover the Discord cron/heartbeat delivery shape with focused tests, and expose read-only lane snapshots through a supported diagnostic command.

## Reproduction Assessment

No. The reporter provided credible long-run observations and current tests can model blocked lanes, but there is still no deterministic current-main reproduction for the exact Discord cron/heartbeat stale-session race.

## Solution Assessment

No. Unconditional lane reset is not the best fix because it can disrupt valid work; the maintainable path is a maintainer-approved liveness and abort policy plus diagnostics.
