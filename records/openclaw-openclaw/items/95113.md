## Summary

Keep open: the original helper-level cleanup hardening is useful, but the current head now also makes aged orphan cleanup rethrow deletion failures. Because `runInPlaceReindex` calls that sweep before building the shadow DB, a locked stale sidecar can now abort forced memory reindexing instead of remaining a best-effort cleanup failure.

## What This Changes

The PR changes memory-core shadow database cleanup to keep iterating after sidecar delete failures, rethrow the first post-reindex cleanup error, and route aged orphan cleanup through that helper with tests.

## Best Possible Solution

Keep `removeMemoryDatabaseFiles` collecting and rethrowing for the post-reindex warning path, but keep `cleanupAgedMemoryReindexTempFiles` best-effort while still attempting all eligible sidecars.

## Reproduction Assessment

Yes. Source inspection shows current main swallows aged cleanup deletion errors, while PR head throws them and `runInPlaceReindex` calls that path before reindex work starts.

## Solution Assessment

No. The helper-level post-reindex cleanup fix is the right narrow layer, but making the aged-orphan preflight cleanup fatal is not the best solution because that sweep should remain best-effort.
