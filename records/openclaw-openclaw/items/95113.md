## What This Changes

The PR changes memory-core shadow database cleanup to attempt every `.memory-reindex-*` SQLite sidecar before rethrowing the first direct cleanup error, while keeping aged-orphan cleanup best-effort and adding regression tests for both paths.

## Best Possible Solution

Land the narrow helper-level cleanup hardening if maintainer review accepts the helper-level proof, keeping aged orphan cleanup best-effort and direct post-run cleanup warning semantics unchanged.

## Reproduction Assessment

Yes, from source inspection: current main stops direct sidecar cleanup on the first `fs.rmSync` error, while the PR head continues through all suffixes and the PR body shows a real filesystem proof for that helper. I did not execute the Windows forced-reindex scenario in this read-only review.

## Solution Assessment

Yes, the current shape is the narrow maintainable fix: share the all-sidecar cleanup helper, rethrow only for the post-run warning path, and keep aged preflight cleanup best-effort. A broader reindex redesign would duplicate the database-first work already merged through https://github.com/openclaw/openclaw/pull/94646.
