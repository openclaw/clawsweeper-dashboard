## Summary

Keep open: the sender metadata addition is plausible and has terminal proof, but the latest diff still routes provenance `sourceChannel` through `Provider`/`Surface`, which are runtime policy inputs rather than prompt-only metadata.

## What This Changes

The PR adds senderId, senderName, and senderRole to input provenance and uses operator-admin `systemInputProvenance` to set `chat.send` sender identity plus `Provider`/`Surface` source-channel context.

## Best Possible Solution

Carry source identity through a prompt/provenance-only contract, keep `MsgContext` routing and policy fields on the internal chat surface unless maintainers explicitly choose otherwise, and align the authorship model with https://github.com/openclaw/openclaw/issues/68353.

## Reproduction Assessment

Not applicable for the feature request itself. For the patch defect, yes: source inspection shows the PR writes provenance `sourceChannel` into `Provider`/`Surface`, and current main uses those fields in runtime policy paths.

## Solution Assessment

No. The sender override is narrow, but overloading `Provider`/`Surface` is not the best fix because those fields affect policy and channel behavior; use dedicated prompt/provenance metadata or remove that override.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep sourceChannel out of Provider and Surface:** `src/gateway/server-methods/chat.ts:2260-2264`
  - body: The latest patch still assigns `systemInputProvenance.sourceChannel` to `Provider` and `Surface`, but those fields are consumed by runtime policy paths, not just prompt metadata. For example, group policy falls back to `ctx.Provider ?? ctx.Surface` when `OriginatingChannel` is absent, and directive handling also uses the provider key. An operator bridge sending a channel-like source would make an internal `chat.send` turn run under that channel's policy even though delivery routing was fixed; keep these fields on `INTERNAL_MESSAGE_CHANNEL` and carry the source channel through provenance or a prompt-only field.
  - confidence: 0.88
