## Summary

Keep open. The PR is a recent, focused implementation that is not on current main, but it changes the public gateway provenance/identity contract and overlaps with open author-attribution design work, so cleanup close is not appropriate.

## What This Changes

This PR extends InputProvenance with senderId, senderName, and senderRole, then uses operator-admin systemInputProvenance in chat.send to set sender identity and source-channel Provider/Surface metadata.

## Best Possible Solution

Decide one additive gateway attribution contract, then land it with schema, runtime, trust semantics, docs, and focused tests aligned across provenance and chat history/event attribution.

## Reproduction Assessment

Not applicable for the feature request itself. Source inspection confirms current main lacks the new sender provenance fields, while the PR body and follow-up comment provide after-fix terminal proof for the branch behavior.

## Solution Assessment

Unclear as a product direction. The patch is mechanically narrow after the OriginatingChannel correction, but maintainers should choose whether trusted per-message identity belongs in systemInputProvenance or the first-class author contract before merge.
