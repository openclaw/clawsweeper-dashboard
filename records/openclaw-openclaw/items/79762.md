## Summary

Keep open for maintainer protocol review. The latest patch is mechanically narrow and has sufficient terminal proof, but it changes public gateway identity semantics and overlaps the open first-class author-attribution design thread.

## What This Changes

The PR adds senderId, senderName, and senderRole to InputProvenance, normalizes them, and lets operator.admin chat.send calls use provenance sender fields plus sourceChannel in MsgContext.

## Best Possible Solution

Align this PR with https://github.com/openclaw/openclaw/issues/68353, decide the trusted identity contract, then land the chosen additive gateway protocol with docs and focused tests.

## Reproduction Assessment

Not applicable for a feature PR. Source inspection shows current main lacks these provenance sender fields, and the PR supplies terminal proof for the new operator.admin gateway path.

## Solution Assessment

Unclear as a product direction. The patch is narrow, but maintainers should decide whether this belongs in trusted provenance or a first-class author contract before merge.
