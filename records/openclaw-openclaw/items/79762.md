## What This Changes

Adds optional senderId, senderName, and senderRole to InputProvenance and uses operator-admin systemInputProvenance in chat.send to override MsgContext sender identity plus Provider/Surface metadata.

## Best Possible Solution

Choose one additive gateway attribution contract, then land schema, runtime behavior, trust semantics, tests, docs, and client/projection alignment consistently across provenance, prompts, history, and events.

## Reproduction Assessment

Not applicable for a feature PR. Source inspection confirms current main lacks these sender provenance fields, and the PR body/comment provide after-fix terminal proof for the proposed branch behavior.

## Solution Assessment

Unclear as a product direction. The patch is mechanically narrow, but maintainers should decide whether trusted per-message identity belongs in systemInputProvenance or in a broader first-class author/requester identity contract before merge.
