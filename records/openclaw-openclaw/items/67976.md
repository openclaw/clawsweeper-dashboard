## Summary

Keep open. Current main still does not support the reported `volcengine-plan/doubao-seed-2.0-pro` imageModel path: the explicit imageModel config is still routed into media understanding, but the reported model is absent from the bundled Volcengine catalog, the built-in `volcengine-plan` rows remain text-only, and no Volcengine media-understanding provider contract is declared. Related #55688 is useful context, but it is still an open partial PR and does not cover the exact model/endpoint report.

## What This Changes

The issue requests fixing or documenting Volcengine Ark coding imageModel failures for `volcengine-plan/doubao-seed-2.0-pro` when image input is sent through OpenClaw.

## Best Possible Solution

Keep this open until maintainers live-verify the exact Ark coding image endpoint and model ID. Then either add Volcengine-owned image-capable catalog/media-understanding support with regression coverage, or document `volcengine-plan` imageModel as unsupported and direct users to a supported image-capable `volcengine/*` model or a custom provider config with image input metadata.

## Reproduction Assessment

Unclear for the provider-side 404 without live Volcengine credentials. Source inspection gives a high-confidence current-main reproduction of the unsupported OpenClaw path: the configured imageModel is selected, but the reported model is absent and bundled `volcengine-plan` models are text-only.

## Solution Assessment

Unclear. A catalog-only change is not proven to be the best fix for this exact report; the safer path is to verify the endpoint/model first, then support it in the Volcengine-owned surface or document it unsupported.
