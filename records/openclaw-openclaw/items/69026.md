## Summary

Keep open. Current main still uses the 30s TUI streaming watchdog default and still emits the normal watchdog warning, so this PR is not implemented or superseded; the diff is narrow, but it still needs changelog coverage and a maintainer UX decision on silent resets.

## What This Changes

The PR raises the TUI streaming watchdog default from 30s to 120s, suppresses the normal user-visible watchdog warning, and updates watchdog tests to expect silent resets.

## Best Possible Solution

Choose the intended TUI watchdog UX in core, then land a tested implementation with changelog coverage: either a softer/deduped warning, a configurable threshold, or an explicit decision that silent resets are preferred.

## Reproduction Assessment

Yes from source inspection. Current fake-timer tests arm the watchdog with a 5s timeout, advance past the window, and assert both the idle reset and the `streaming watchdog` system message.

## Solution Assessment

Unclear. The patch is narrow, but complete suppression is a maintainer UX choice because it removes diagnostics for genuine silent drops; softer deduped feedback or a config-backed threshold may be safer.
