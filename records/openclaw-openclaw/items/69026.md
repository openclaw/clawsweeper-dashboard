## Summary

Keep this PR open for maintainer follow-up. Current main still has the 30s default and the normal watchdog timeout still emits the user-visible warning, so the PR is not implemented or obsolete; however, newer main work fixed adjacent tool/reconnect false-idle cases and leaves a real product choice between silent reset, softer copy, or configurable behavior.

## What This Changes

The PR raises the TUI streaming watchdog default from 30s to 120s, suppresses the watchdog system message, and updates watchdog tests to expect silent internal resets.

## Best Possible Solution

Rebase the PR onto current main, preserve the merged reconnect and tool/lifecycle watchdog recovery behavior, and make a maintainer product decision on whether the user-facing outcome should be silent reset, softer warning copy, duplicate suppression, or a configurable watchdog setting.
