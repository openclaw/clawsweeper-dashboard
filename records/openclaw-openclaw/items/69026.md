## Summary

Keep open. The PR is not merge-ready because it would remove the visible stuck-run notice entirely, while current main has already softened the alarming wording and separate open work tracks threshold configurability and notice dismissal.

## What This Changes

The PR raises the TUI streaming watchdog default from 30s to 120s, suppresses the user-visible watchdog notice, and updates fake-timer tests to expect silent resets.

## Best Possible Solution

Preserve a visible but low-anxiety watchdog notice, make stale notices dismissible or tune the timeout through the canonical TUI/config work, and keep the shipped proof-of-life and reconnect behavior intact.

## Reproduction Assessment

Yes. Source inspection and existing fake-timer tests reproduce the current watchdog path by sending a delta, advancing past `streamingWatchdogMs`, and observing the idle reset plus system notice.

## Solution Assessment

No. Complete suppression is not the best current direction because main already replaced the alarming backend wording with softer guidance; a dismissible notice or the configurable-threshold work in https://github.com/openclaw/openclaw/issues/68596 is safer.
