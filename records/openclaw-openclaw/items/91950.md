## Summary

The report is valid, but this patch is not safe to merge as written: deleting all whitespace changes native URL path/query semantics before caching, provider fallback, redirects, and SSRF enforcement, and the supplied proof does not exercise a real after-fix `web_fetch` call.

## What This Changes

The PR strips leading, trailing, and internal whitespace from the `web_fetch` URL argument before URL parsing and fetch execution.

## Best Possible Solution

Normalize only accidental whitespace between an HTTP(S) scheme delimiter and its authority, preserve native WHATWG path/query encoding and strict rejection elsewhere, and prove the corrected URL through the real guarded `web_fetch` path.
