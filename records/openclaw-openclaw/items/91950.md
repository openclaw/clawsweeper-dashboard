## Summary

The issue is valid, but this PR is not safe to merge as written: deleting every whitespace character changes valid WHATWG URL semantics and can silently fetch a different path or query value. The linked issue should remain open while this PR is narrowed and given real web_fetch runtime proof.

## What This Changes

The PR strips leading, trailing, and internal whitespace from the web_fetch URL argument before parsing, caching, provider fallback, and guarded fetching.

## Best Possible Solution

Add a narrow URL-repair helper that fixes only the proven malformed scheme/authority whitespace, preserves normal URL percent-encoding elsewhere, and has focused execution tests plus redacted live web_fetch proof.

## Reproduction Assessment

Yes, at source level with high confidence: current main rejects `https:// docs.openclaw.ai`, while the platform parser accepts path and query spaces by percent-encoding them. A live model-generated reproduction was not performed.

## Solution Assessment

No. Repairing the reported scheme typo is appropriate, but deleting all whitespace is not the narrowest maintainable fix because it changes valid path and query meaning before security checks.
