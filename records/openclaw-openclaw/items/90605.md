## Risks / Open Questions

- Direct OCI/ORAS support would add a new executable artifact distribution path for skills and plugins, so provenance, pinning, scanning, install-policy behavior, and offline update semantics need design before implementation.
- The proposed registry login command would introduce credential storage and operator workflow questions that current ClawHub/npm/git/local installers do not settle.
- Workspace-template artifacts may contain private memory, bootstrap instructions, and executable-adjacent files, so maintainers need to decide whether this belongs in core, ClawHub, or a documented external ORAS pull-to-local workflow.
