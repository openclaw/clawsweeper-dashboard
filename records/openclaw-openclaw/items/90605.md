## What This Changes

The issue requests first-class OCI/ORAS artifact distribution and `openclaw registry` / `oci://` install flows for skills, plugins, and workspace templates.

## Best Possible Solution

Have maintainers decide whether OCI belongs as a ClawHub artifact backend, a documented ORAS pull-to-local workflow, or a new core installer surface before adding registry credentials and `oci://` commands.

## Reproduction Assessment

Not applicable; this is a feature request rather than a report of broken existing behavior. Source and docs checks can verify that current install surfaces lack OCI/ORAS, but there is no failing current-main path to reproduce.

## Solution Assessment

Unclear; direct core `registry login` and `oci://` installers are not clearly the best shape under the current ClawHub-first plugin/skill vision. A ClawHub artifact backend or documented ORAS pull-to-local path may cover disconnected setups with less core credential surface.
