## Summary

Keep open: current main and the latest release still route plaintext local tailnet dashboard links to the tailnet IP while Gateway listener setup only adds extra loopback coverage for loopback binds. ## What This Changes

The issue asks for local plaintext tailnet dashboard access to use loopback, with matching loopback listener coverage for specific non-loopback Gateway binds.

## Best Possible Solution

Land or replace the linked fix with a narrow Gateway/dashboard change that adds loopback listener coverage for local non-loopback binds, prefers loopback only for plaintext local dashboard URLs, preserves secure TLS tailnet URLs, and updates any affected docs/tests.
