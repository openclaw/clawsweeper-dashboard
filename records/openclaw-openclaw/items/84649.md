## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Preserve derived snapshot file freshness:** `src/plugins/plugin-metadata-snapshot.ts:773`
  - body: This stores derived snapshot memo entries with the lookup-time `registryState`, whose watched files come from the persisted index. Current main rebuilds that state from `result.snapshot.index` so edits to a derived plugin's manifest/package/source files invalidate the memo in the same process; the PR also changes the regression test to only rewrite the installed index. Long-running TUI/gateway processes can therefore keep stale derived metadata until index rewrite or restart.
  - confidence: 0.9
- **[P1] Include setup CLI backend owners in provider lookup:** `src/plugins/providers.ts:521`
  - body: The new manifest-only owner lookup checks top-level `plugin.cliBackends` but not `plugin.setup?.cliBackends`. The current no-registry path delegates to the shared contribution helper, which includes setup CLI backends, and docs call `setup.cliBackends` the descriptor-first setup/control-plane lookup surface. Because this PR makes provider paths supply/load a manifest registry more often, setup-only backend owners can stop being activated for explicit backend refs.
  - confidence: 0.84
