## Summary

Keep this open: current main still has source-provable Ollama paths that send truthy `think` at the top level, and an open PR with closing syntax is already paired to this issue. The remaining decision is to review, revise, or land that PR rather than close the issue before a fix merges.

## What This Changes

The issue asks OpenClaw to avoid Ollama 400s by no longer sending `think` as a top-level `/api/chat` field for non-thinking models.

## Best Possible Solution

Review https://github.com/openclaw/openclaw/pull/80334 and either land a corrected fix or replace it with a capability-aware guard that prevents non-thinking models from receiving truthy top-level `think` while preserving the upstream top-level field for thinking-capable models.

## Reproduction Assessment

Yes, from source for the explicit native Ollama params/runtime path: current main can emit truthy top-level `think`, and upstream Ollama rejects truthy `req.Think` for non-thinking models. The exact default-thinking route still needs live/config confirmation because current validation can clamp unsupported levels to `off`.

## Solution Assessment

Unclear as proposed: moving all Ollama thinking control into `options.think` is not the safest demonstrated fix because upstream Ollama models `think` as a top-level chat field. A narrower capability-aware guard is the more maintainable direction unless live upstream proof validates the alternate wire shape.
