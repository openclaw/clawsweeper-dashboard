## Summary

Keep open: current main still has Ollama paths that can emit a truthy top-level `think`, and the linked fix PR was closed unmerged. The requested blanket move to `options.think` conflicts with the current Ollama/OpenClaw documented contract, so this needs maintainer direction before a code change.

## What This Changes

Route Ollama `think` into `options` or otherwise prevent top-level truthy `think` from reaching non-thinking Ollama models.

## Best Possible Solution

Keep top-level Ollama `think` for reasoning-capable native models, and add a capability-aware guard or validation path so non-reasoning models cannot receive truthy top-level `think` unless upstream support for another contract is proven.

## Reproduction Assessment

No, not for the exact `thinkingDefault: "low"` steps without a live Ollama run; current source suggests that route is normally validated or remapped. The underlying top-level truthy `think` problem is source-reproducible through explicit Ollama model params, and upstream source shows why non-thinking models reject it.

## Solution Assessment

No. Moving all native Ollama thinking control to `options.think` is not the narrowest maintainable solution because upstream and OpenClaw docs currently model `think` as top-level; a guarded top-level path for supported models is safer.
