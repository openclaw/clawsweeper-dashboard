## Summary

Keep open: current main and the latest release still send native Ollama thinking control as top-level `think`, and there is still a source-provable path where explicit truthy `params.think` or `params.thinking` reaches the top-level request. The requested blanket move to `options.think` conflicts with current OpenClaw docs/tests and upstream Ollama's request contract, so this needs maintainer direction rather than cleanup closure.

## What This Changes

The issue requests moving Ollama `think` from the top-level `/api/chat` payload into `options.think` to avoid 400 responses from non-thinking models.

## Best Possible Solution

Preserve top-level Ollama `think` for reasoning-capable native models, then decide whether OpenClaw should guard, warn, or reject truthy explicit thinking params for models known not to support thinking.

## Reproduction Assessment

Yes, for a narrowed source path: current main can put a truthy configured `params.think` or `params.thinking` on the top-level Ollama chat payload, and upstream Ollama rejects truthy `req.Think` for non-thinking models. The exact `thinkingDefault: "low"` route still needs live confirmation because current OpenClaw also has supported-level validation.

## Solution Assessment

No, not as suggested: moving all native Ollama thinking control to `options.think` conflicts with current OpenClaw docs/tests and upstream Ollama source. A safer solution is capability-aware guarding or diagnostics while preserving top-level `think` for supported models.
