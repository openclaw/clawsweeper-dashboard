## Summary

Keep this open: current main still has source-proven top-level truthy Ollama `think` paths, and an open linked PR tracks a proposed fix, but the proposed wire-shape change needs maintainer review and live Ollama proof.

## What This Changes

The issue asks OpenClaw to stop sending truthy Ollama `think` at the top level for non-thinking native chat models, preferably by routing it through `options`.

## Best Possible Solution

Keep the issue open and resolve it through https://github.com/openclaw/openclaw/pull/80334 or a replacement that preserves top-level `think` for reasoning-capable models while preventing truthy top-level `think` for non-reasoning models.

## Reproduction Assessment

Yes, partially from source: current main can emit a truthy top-level native Ollama `think` from explicit model params, and upstream Ollama rejects truthy `req.Think` for non-thinking models. The reporter's `thinkingDefault` route still needs live/config confirmation because current OpenClaw also validates supported thinking levels.

## Solution Assessment

No, not as written: moving all native Ollama thinking control into `options.think` is not proven by the upstream contract. A narrower capability-aware guard, or live proof that `options.think` is the intended Ollama path, is safer.
