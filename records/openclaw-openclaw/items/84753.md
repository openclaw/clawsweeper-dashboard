## Summary

Keep open. Current main already carries resolved `label`/`displayName` through parts of the Gateway and Control UI, but `openclaw status` and local session listing still render raw keys, and the requested cross-channel/configurable labeling policy needs maintainer product direction.

## What This Changes

Request to show channel user display names instead of opaque peer/session IDs across the Control UI, `openclaw status`, and `sessions_list`, possibly via a configurable session label format.

## Best Possible Solution

Define one session display-name precedence at the Gateway/session metadata boundary, preserve explicit labels and raw keys, then extend status/CLI/session-list surfaces to consume it with targeted coverage while folding Feishu-specific fallbacks into the same plan or their canonical follow-ups.

## Reproduction Assessment

Yes, source-reproducible rather than live-reproduced: current main has session label/displayName fields, but the `openclaw status` and local sessions projection/formatting paths never read them. The Control UI/Gateway path is only partially addressed.

## Solution Assessment

Unclear. A new per-channel `sessionLabelFormat` may be broader than needed; the safer maintainable direction is to settle one cross-channel precedence/sanitization policy and reuse existing `label`/`displayName` metadata before adding config.
