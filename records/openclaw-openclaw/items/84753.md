## Summary

Keep open: current main has partial label/display-name plumbing, but the requested operator-facing behavior is still not consistently implemented across `openclaw status`, local session tables, and channel-derived labels, and the remaining design needs product/API judgment.

## What This Changes

The issue requests that session lists show channel display names instead of raw peer IDs, potentially through a cross-channel `sessionLabelFormat` or existing sender-name resolution.

## Best Possible Solution

Define one generic session display-label contract that all listing surfaces consume, with raw peer IDs retained as fallback and channel-specific name resolution owned by the relevant channel/plugin.

## Reproduction Assessment

Yes, at source level: current main shows labels/display names in some gateway/UI paths while `openclaw status` and local session-table paths still project and render raw keys. I did not run a live Feishu, Discord, Telegram, or WhatsApp session, so this is not runtime-reproduced.

## Solution Assessment

Unclear: a `sessionLabelFormat` option may be useful, but the safer direction is first to standardize the display-label contract and fallback policy across listing surfaces before adding per-channel formatting knobs.
