## Summary

Keep open: current main has partial session label/display-name plumbing, but the requested operator-facing behavior is not consistently implemented across the CLI status/session tables and there is no cross-channel label-format policy yet.

## What This Changes

The issue requests display-name-based labels for channel sessions instead of raw peer IDs in the Control UI, `openclaw status`, and `sessions_list` output.

## Best Possible Solution

Define a generic persisted session display-label contract that all listing surfaces consume consistently, with raw peer IDs preserved as fallback and per-channel name resolution kept in the owning channel/plugin.

## Reproduction Assessment

Yes at source level: the CLI session/status paths still render raw session keys while only some Gateway/UI paths consume label/display-name metadata. I did not run a live Feishu/Discord/Telegram session, so this is source-reproducible rather than runtime-reproduced.

## Solution Assessment

Unclear: a configurable `sessionLabelFormat` may be useful, but the safer first design is to standardize persisted session display labels and make all list surfaces consume the same field with raw-ID fallback.
