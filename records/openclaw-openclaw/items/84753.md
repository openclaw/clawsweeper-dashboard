## Summary

Keep open: current main still lacks a shared display-name contract across Gateway sessions.list, CLI/status output, the Control UI session surfaces, and the model-facing sessions_list tool. The related open PRs cover narrower account-name or UI-truncation slices, but they do not resolve the full cross-surface behavior requested here.

## What This Changes

The issue asks OpenClaw to show friendly session display names instead of opaque user IDs across session lists, status/UI surfaces, and the sessions_list tool, potentially with configurable label formatting or channel-specific mappings.

## Best Possible Solution

Define one shared Gateway session display/search contract, reuse existing label/displayName/account metadata where appropriate, then have CLI, Control UI, and sessions_list consume that contract; add new config only if maintainers explicitly choose that product surface.

## Reproduction Assessment

Yes. Source inspection shows direct sessions without explicit label/displayName still fall back to origin labels or raw keys in the shared Gateway projection, and CLI/tool consumers forward those values.

## Solution Assessment

Unclear for the proposed config-first shape. The better path is a shared Gateway display/search contract with consistent consumers, while custom mappings or format config need maintainer product approval.
