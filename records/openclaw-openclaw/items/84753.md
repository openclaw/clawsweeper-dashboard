## Summary

Keep open: current main has partial label/displayName plumbing, but the requested cross-channel operator-facing behavior is not consistently implemented across status, CLI/TUI, and session-list consumers, and the broad label-format/name-resolution policy still needs maintainer judgment.

## What This Changes

Request to show human-readable channel display names instead of raw peer IDs in session list, status, and sessions_list surfaces, potentially through a configurable session label format.

## Best Possible Solution

Define a generic persisted session display-label contract consumed by all listing surfaces, with raw peer IDs preserved as fallback and channel-specific name resolution kept in the owning channel/plugin.

## Reproduction Assessment

Yes at source level: current main stores and partly exposes label/displayName metadata, but status/CLI/TUI paths still render raw session keys and there is no cross-channel label-format policy. I did not run a live Feishu, Discord, Telegram, or WhatsApp session.

## Solution Assessment

Unclear: a configurable sessionLabelFormat may be useful, but the safer direction is first to standardize a persisted display-label contract with explicit raw-ID fallback and privacy rules.
