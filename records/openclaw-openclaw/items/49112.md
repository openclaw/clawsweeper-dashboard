## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Do not age out live gateway locks:** `src/infra/gateway-lock.ts:221-227`
  - body: The inspector marks every lock older than `staleMs` as stale even when the recorded PID is alive. A healthy gateway can hold its lock longer than 30 seconds, so `doctor --fix` could remove an active singleton guard and allow a second gateway to start.
  - confidence: 0.94
- **[P2] Reuse the gateway owner-status contract:** `src/infra/gateway-lock.ts:204`
  - body: `inspectGatewayLock` reduces liveness to `isPidAlive`, while startup also checks port reachability, Linux start time, and gateway argv shape. A recycled PID or unrelated process can look healthy to doctor even when startup would reclaim the stale lock.
  - confidence: 0.91
- **[P2] Register the check in doctor health contributions:** `src/commands/doctor.ts:223`
  - body: Current main routes doctor checks through `runDoctorHealthContributions`. Adding the helper to the old command body is stale after rebase and will not be part of the active doctor pipeline.
  - confidence: 0.88
- **[P3] Render unknown PID status as unknown:** `src/commands/doctor-gateway-locks.ts:27`
  - body: `GatewayLockInspection` allows `pidAlive` to be `null`, but the formatter treats any falsy value as dead. Unknown owner status should not be displayed as a confirmed-dead process.
  - confidence: 0.78
