Full review comments:

- **[P1] Preserve live gateway locks during repair:** `src/infra/gateway-lock.ts:221-227`
  - body: `inspectGatewayLock()` treats age alone as stale and removes the file in repair mode even when the recorded PID is alive. Gateway locks are long-lived, so `openclaw doctor --fix` could remove an active singleton guard and allow a second gateway to start.
  - confidence: 0.94
- **[P2] Reuse the gateway owner-status contract:** `src/infra/gateway-lock.ts:205`
  - body: The inspector reduces owner detection to `isPidAlive(pid)`, while gateway startup also checks port reachability, Linux start time, and gateway argv shape. A recycled PID or unrelated process can look healthy to doctor even when startup would reclaim the stale lock.
  - confidence: 0.91
- **[P2] Register the check in doctor health contributions:** `src/commands/doctor.ts:223`
  - body: Current main routes doctor checks through `runDoctorHealthContributions()`. Adding this helper to the old command body is stale after rebase and will not put the gateway-lock check into the active doctor pipeline.
  - confidence: 0.88
