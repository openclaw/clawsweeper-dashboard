## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Do not age out live gateway locks:** `src/infra/gateway-lock.ts:221-227`
  - body: The inspector marks any lock older than `staleMs` as stale and removes stale locks in repair mode, even when the recorded PID is alive. Healthy gateway locks normally live past 30 seconds; current startup only uses age-based cleanup when the owner is not confirmed alive, so this can delete an active singleton guard.
  - confidence: 0.94
- **[P2] Reuse the gateway owner-status contract:** `src/infra/gateway-lock.ts:204`
  - body: This reduces liveness to `isPidAlive(pid)`, while gateway startup also checks port reachability, Linux start time, and gateway argv shape. A recycled PID or unrelated process can look healthy to doctor even when startup would reclaim the stale lock.
  - confidence: 0.91
- **[P2] Register the check in doctor health contributions:** `src/commands/doctor.ts:223`
  - body: Current main routes doctor checks through `runDoctorHealthContributions()`. Adding the helper to the old command body is stale after rebase and will not put the gateway-lock check into the active doctor pipeline.
  - confidence: 0.88
- **[P3] Render unknown PID status as unknown:** `src/commands/doctor-gateway-locks.ts:27`
  - body: `GatewayLockInspection` allows `pidAlive` to be `null`, but the formatter treats any falsy value as `dead`. Unknown owner status should not be displayed as a confirmed-dead process.
  - confidence: 0.78
