Full review comments:

- **[P1] Preserve live gateway locks during repair:** `src/infra/gateway-lock.ts:225-231`
  - body: `inspectGatewayLock()` treats age alone as stale and then removes any stale lock in repair mode. Healthy gateway locks can be older than 30 seconds, so `openclaw doctor --fix` could delete an active singleton guard and allow a second gateway to start.
  - confidence: 0.94
- **[P2] Reuse the gateway owner-status contract:** `src/infra/gateway-lock.ts:208`
  - body: The inspector reduces owner detection to `isPidAlive(pid)`, while gateway startup also checks port reachability, Linux start time, and gateway argv shape. A recycled PID or unrelated process can look healthy to doctor even when startup would reclaim the stale lock.
  - confidence: 0.91
- **[P2] Register the check in doctor health contributions:** `src/commands/doctor.ts:226`
  - body: Current main routes doctor checks through `runDoctorHealthContributions()`. Adding this helper to the old command body is stale after rebase and will not put the gateway-lock check into the active doctor pipeline.
  - confidence: 0.88
