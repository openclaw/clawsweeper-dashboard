## Summary

Keep open: current main still lacks a gateway-lock doctor check, and this useful PR is not merge-ready because its inspector can delete active gateway locks, diverges from runtime owner-status semantics, and lacks after-fix real behavior proof.

## What This Changes

The PR adds a gateway-lock doctor helper, inspector, tests, fast-path mock, and `doctor` wiring to report and remove stale gateway lock files with `--fix`.

## Best Possible Solution

Rebase the PR onto current main, register a gateway-lock doctor health contribution, reuse the runtime owner-status contract, repair only dead/unknown/malformed stale locks under `--fix`, and add focused coverage plus real CLI proof.

## Reproduction Assessment

Yes. Source inspection is high-confidence: the PR marks any lock older than `staleMs` stale even when `pidAlive` is true, while current runtime acquisition only age-reclaims when owner status is not alive.

## Solution Assessment

No. The requested direction is useful, but the current diff should not merge until it reuses runtime owner-status semantics, avoids deleting live locks by age alone, and is wired into the current doctor contribution flow.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Avoid deleting live gateway locks by age alone:** `src/infra/gateway-lock.ts:225-227`
  - body: `inspectGatewayLock()` adds the `age` stale reason regardless of whether the recorded PID is alive. A healthy gateway can hold its lock for far longer than 30 seconds, so `openclaw doctor --fix` can remove an active gateway lock and allow a second gateway to start; mirror `acquireGatewayLock()` and only age-reclaim when the owner is not confirmed alive.
  - confidence: 0.95
- **[P2] Reuse the runtime gateway owner-status check:** `src/infra/gateway-lock.ts:208`
  - body: The inspector uses `isPidAlive(pid)` instead of the runtime `resolveGatewayOwnerStatus()` path. On Linux, a crashed gateway whose PID is reused by another process will be reported alive, so doctor will hide the stale lock that startup would reclaim.
  - confidence: 0.9
