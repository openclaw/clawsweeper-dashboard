## Summary

Keep this PR open. Current main still lacks doctor inspection and repair for gateway lock files, so the linked product gap remains, but the submitted patch is not safe to merge because its repair path can remove a confirmed-live gateway lock and its inspection semantics are weaker than gateway startup lock acquisition.

## What This Changes

This PR adds a gateway-lock inspection helper, a new doctor note module and tests, a doctor fast-path mock, and direct wiring from `doctorCommand` so `openclaw doctor --fix` can remove stale gateway locks.

## Best Possible Solution

Add gateway-lock doctor health as a current doctor contribution, backed by the same owner-status semantics used by gateway startup. Repair should remove only dead owners or unknown owners that satisfy the existing stale heuristic, and the docs should move Fly/manual cleanup guidance toward `openclaw doctor --fix` once covered by tests.

## Reproduction Assessment

Yes. Current-main absence is reproducible by inspecting the doctor contribution registry and docs; the PR bug is reproducible from the diff by creating a lock with a live PID and old `createdAt`, then calling the new inspector with `removeStale: true`, which removes it because age alone marks it stale.

## Solution Assessment

No. The feature direction is right, but this patch is not the best solution because it reimplements weaker lock ownership logic and targets an older doctor flow shape; the safer fix is to share or expose the gateway owner-status helper and register it through the current contribution pipeline.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Do not age out confirmed-live gateway locks:** `src/infra/gateway-lock.ts:222-228`
  - body: The new inspector treats `ageMs > staleMs` as stale even when `pidAlive` is true, and `doctor --fix` then removes the lock. A healthy gateway lock normally lives longer than 30 seconds, so this can delete the active lock and allow a second gateway instance to start.
  - confidence: 0.94
- **[P2] Reuse gateway owner status for inspection:** `src/infra/gateway-lock.ts:203-205`
  - body: `inspectGatewayLock` only calls `isPidAlive`, while `acquireGatewayLock` also validates the port, Linux start time, and process argv. If a PID is recycled or belongs to another process, doctor will report `stale=no` even though startup would reclaim the stale lock.
  - confidence: 0.91
- **[P3] Render unknown PID status accurately:** `src/commands/doctor-gateway-locks.ts:25-27`
  - body: `pidAlive` is typed as `boolean | null`, but the formatter renders any falsy value as `dead`. If inspection cannot determine owner status, doctor should show `unknown` rather than telling operators the owner is dead.
  - confidence: 0.78
