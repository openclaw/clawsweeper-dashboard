## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Do not age out live gateway locks:** `src/infra/gateway-lock.ts:222-228`
  - body: The inspector marks a lock stale solely because `ageMs > staleMs`, and `doctor --fix` removes stale locks. Healthy gateways can hold the lock longer than 30 seconds, so this can delete an active lock and allow a second gateway to start.
  - confidence: 0.94
- **[P2] Reuse gateway owner status for inspection:** `src/infra/gateway-lock.ts:203-205`
  - body: `inspectGatewayLock` only checks `isPidAlive`, while startup also checks port reachability, Linux start time, and process argv. Recycled PIDs or unrelated processes can therefore look healthy to doctor even though startup would reclaim the lock.
  - confidence: 0.91
- **[P2] Register the check through doctor contributions:** `src/commands/doctor.ts:223-224`
  - body: Current main runs doctor health through `runDoctorHealthContributions`, but this patch wires the gateway-lock check into the old command body. Add it to the contribution list so it runs in the active doctor pipeline after rebase.
  - confidence: 0.88
- **[P3] Render unknown PID status accurately:** `src/commands/doctor-gateway-locks.ts:25-27`
  - body: `pidAlive` is typed as `boolean | null`, but the formatter renders any falsy value as `dead`. Unknown owner status should display as unknown rather than implying a confirmed-dead process.
  - confidence: 0.78
- **[P3] Add the required changelog entry:** `src/commands/doctor-gateway-locks.ts:55-57`
  - body: This adds user-facing doctor diagnostics and repair behavior, but the PR does not update `CHANGELOG.md`. OpenClaw policy requires user-facing fix, feat, and perf changes to include a changelog entry.
  - confidence: 0.86
