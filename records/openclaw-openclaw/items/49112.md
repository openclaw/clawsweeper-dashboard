## Summary

Keep this PR open, but it needs changes before merge: current main still lacks gateway-lock doctor coverage, while the PR’s inspector can delete healthy long-running gateway locks and does not reuse the gateway startup owner-status contract.

## What This Changes

The PR adds a gateway-lock inspection helper, doctor output/repair wiring, fast-path mocks, and tests so `openclaw doctor --fix` can report and remove stale gateway lock files.

## Best Possible Solution

Implement gateway-lock doctor health as a registered doctor contribution backed by the same owner-status semantics as gateway startup, with focused tests, docs, and changelog coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows doctor delegates to `runDoctorHealthContributions`, which registers session-lock health but no gateway-lock contribution; the PR defects are also visible in the diff.

## Solution Assessment

No. The feature direction is right, but the patch should reuse current startup owner-status logic and register through the current doctor contribution pipeline before it is mergeable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Do not age out confirmed-live gateway locks:** `src/infra/gateway-lock.ts:222-228`
  - body: The inspector marks any lock older than `staleMs` as stale, and `doctor --fix` then removes it. Healthy gateway locks routinely live longer than 30 seconds, so this can delete an active gateway lock and allow a second gateway to start.
  - confidence: 0.94
- **[P2] Reuse gateway owner status during inspection:** `src/infra/gateway-lock.ts:203-205`
  - body: `inspectGatewayLock` only calls `isPidAlive`, while gateway startup also checks the port, Linux start time, and process argv. Recycled PIDs or unrelated processes can therefore look healthy to doctor even when startup would reclaim the lock.
  - confidence: 0.91
- **[P2] Register the check through doctor contributions:** `src/commands/doctor.ts:223-224`
  - body: Current main runs doctor health through `runDoctorHealthContributions`, but this patch wires the new check into the old command body. Move the gateway-lock check into `src/flows/doctor-health-contributions.ts` so it runs in the active doctor pipeline after rebase.
  - confidence: 0.88
- **[P3] Render unknown PID status accurately:** `src/commands/doctor-gateway-locks.ts:25-27`
  - body: `pidAlive` is typed as `boolean | null`, but the formatter renders any falsy value as `dead`. If owner status is unknown, doctor should say `unknown` instead of implying a confirmed-dead process.
  - confidence: 0.78
- **[P3] Add the required changelog entry:** `src/commands/doctor-gateway-locks.ts:55-57`
  - body: This adds user-facing doctor diagnostics and repair behavior, but the PR does not update `CHANGELOG.md`. OpenClaw policy requires user-facing fix, feat, and perf changes to include a changelog entry.
  - confidence: 0.86
