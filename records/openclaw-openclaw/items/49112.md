## Summary

Keep open: current main still lacks gateway-lock doctor coverage, but this PR cannot merge as-is because the proposed inspector can remove confirmed-live locks, misses the existing owner-status checks, and is wired into an outdated doctor path.

## What This Changes

The PR adds a gateway lock inspector, doctor output and repair helper, doctor wiring, fast-path mocks, and tests so `openclaw doctor` can report and remove stale gateway lock files.

## Best Possible Solution

Land a rebased implementation that adds gateway-lock health as a doctor contribution backed by the same owner-status semantics as gateway startup, with focused tests plus docs and changelog coverage.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes. Source inspection shows current main has no gateway-lock doctor contribution, and the PR defects are provable by comparing its inspector with the current gateway owner-status contract.

## Solution Assessment

Is this the best way to solve the issue? No. The feature direction is right, but the implementation should reuse the existing owner-status logic and register through `src/flows/doctor-health-contributions.ts` before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Do not age out confirmed-live gateway locks:** `src/infra/gateway-lock.ts:222-228`
  - body: The inspector marks a lock stale when `ageMs > staleMs` even if the PID is alive, and `doctor --fix` then removes it. Healthy gateway locks can live longer than 30 seconds, so this can delete an active gateway's lock and allow a duplicate gateway to start.
  - confidence: 0.94
- **[P2] Reuse gateway owner status for inspection:** `src/infra/gateway-lock.ts:203-205`
  - body: `inspectGatewayLock` only checks `isPidAlive`, while startup also checks the port, Linux start time, and process argv. A recycled PID or unrelated process can therefore look healthy to doctor even though startup would reclaim the lock.
  - confidence: 0.91
- **[P2] Register the check through doctor contributions:** `src/commands/doctor.ts:223-224`
  - body: Current main runs doctor health via `runDoctorHealthContributions`, but this patch wires the check into the old command body. Move the gateway-lock check into `src/flows/doctor-health-contributions.ts` so it runs in the active pipeline after rebase.
  - confidence: 0.88
- **[P3] Render unknown PID status accurately:** `src/commands/doctor-gateway-locks.ts:25-27`
  - body: `pidAlive` is typed as `boolean | null`, but the formatter renders any falsy value as `dead`. Unknown owner status should not be shown as a confirmed-dead process.
  - confidence: 0.78
- **[P3] Add the required changelog entry:** `src/commands/doctor-gateway-locks.ts:55-57`
  - body: This adds user-facing doctor diagnostics and repair behavior without a `CHANGELOG.md` entry. OpenClaw policy requires user-facing fix, feat, and perf changes to include one.
  - confidence: 0.86
