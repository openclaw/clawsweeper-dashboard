## Summary

Keep this PR open, but it needs changes before merge: the feature is still missing from current main, while the proposed patch can remove live gateway locks, uses weaker owner detection than gateway startup, and is wired through a stale doctor path.

## What This Changes

The PR adds gateway-lock inspection, doctor output, `--fix` removal, fast-path mocks, and tests so `openclaw doctor` can report and repair stale gateway lock files.

## Best Possible Solution

Implement gateway-lock doctor health as a registered doctor contribution backed by the same owner-status logic as gateway startup, with docs, changelog, and regression coverage for live, stale, and unknown-owner cases.

## Reproduction Assessment

Yes. Source inspection of current main shows there is no gateway-lock doctor contribution, and the PR defects are visible from the provided diff against the existing gateway-lock owner-status contract.

## Solution Assessment

No. The feature direction is maintainable, but the patch should share startup owner-status semantics and register through `src/flows/doctor-health-contributions.ts` before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Do not age out confirmed-live gateway locks:** `src/infra/gateway-lock.ts:222-228`
  - body: `inspectGatewayLock` treats `ageMs > staleMs` as stale regardless of owner status, so `openclaw doctor --fix` can delete a healthy gateway lock after the default 30 seconds and allow a second gateway to start.
  - confidence: 0.94
- **[P2] Reuse gateway owner status for inspection:** `src/infra/gateway-lock.ts:203-205`
  - body: The inspector only calls `isPidAlive`, while startup also probes the port and validates Linux start time or process argv. Recycled PIDs or unrelated processes can therefore look healthy to doctor even though startup would reclaim the lock.
  - confidence: 0.91
- **[P2] Register the check through doctor contributions:** `src/commands/doctor.ts:223-224`
  - body: Current main runs doctor health checks through `runDoctorHealthContributions`, but this patch wires the check into the old command body. Add the gateway-lock health check to the contribution list so it runs in the active pipeline after rebase.
  - confidence: 0.88
- **[P3] Render unknown PID status accurately:** `src/commands/doctor-gateway-locks.ts:25-27`
  - body: `pidAlive` is typed as `boolean | null`, but the formatter renders any falsy value as `dead`. Unknown owner status should be shown as `unknown` rather than implying a confirmed-dead process.
  - confidence: 0.78
- **[P3] Add the required changelog entry:** `src/commands/doctor-gateway-locks.ts:55-57`
  - body: This adds user-facing doctor diagnostics and repair behavior, but the PR does not update `CHANGELOG.md`. OpenClaw policy requires user-facing fix, feat, and perf changes to include a changelog entry.
  - confidence: 0.86
