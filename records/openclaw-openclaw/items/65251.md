## Summary

Keep open: current main still has static bundle-declared LSP support, but the requested workspace auto-detection, package-manager heuristics, PATH discovery, install/fix flow, update notices, override model, and dashboard UX remain unimplemented and need maintainer design.

## What This Changes

Request zero-configuration LSP server detection and configuration from project files, package managers, installed servers, optional auto-install, overrides, updates, and diagnostics dashboard UX.

## Best Possible Solution

Keep this as a maintainer design item for a scoped LSP detection/lifecycle API, explicit install authority, documented overrides, status diagnostics, and plugin or bundle ownership for language-specific defaults.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature proposal. The current-main gap is source-verifiable from the bundle-backed LSP loader, runtime, docs, and absence of a dynamic detection or install surface.

## Solution Assessment

No direct all-in-one core auto-installer should be assumed as the best fix. The maintainable path needs product/API/security design for detection, install permission, overrides, diagnostics, and plugin-boundary ownership before implementation.
