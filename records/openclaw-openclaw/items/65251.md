## Summary

Keep open. Current main still only supports bundle-declared LSP defaults plus a stdio runtime, while the request asks for workspace language scanning, PATH discovery, optional install/fix flows, overrides, update notices, and dashboard UX.

## What This Changes

Request automatic workspace LSP detection and configuration, with optional server installation, status diagnostics, overrides, and update notices.

## Best Possible Solution

Keep this as a maintainer design item for a scoped LSP detection API, explicit install authority, documented overrides, status diagnostics, and plugin or bundle extension points for language-specific behavior.

## Reproduction Assessment

Not applicable as a bug reproduction. The current-main gap is verifiable from source and docs: LSP support is loaded from enabled bundle configuration and spawned only from configured server entries.

## Solution Assessment

No direct core auto-installer should be assumed as the best fix yet. The maintainable path needs product, plugin API, UX, and security design before implementation.
