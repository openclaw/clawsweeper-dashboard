## Summary

Keep open: current main still supports only static bundle-declared LSP defaults and runtime startup for configured stdio servers; workspace detection, PATH discovery, install/fix flows, overrides, update notices, and dashboard/status UX remain broad product/API/security work.

## What This Changes

Request automatic LSP server detection, configuration, optional installation, overrides, status diagnostics, and one-click repair for common languages.

## Best Possible Solution

Keep this as a maintainer design item for a scoped LSP detection/lifecycle API, explicit install authority, documented overrides, diagnostics/status events, and plugin or bundle ownership for language defaults.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature proposal. The current-main gap is source-verifiable from the bundle-backed loader, runtime, docs, and absence of a standalone LSP detection surface.

## Solution Assessment

No direct core auto-installer should be assumed as the best fix. The maintainable path needs product/API/security design for detection, install permission, overrides, diagnostics, and plugin-boundary ownership before implementation.
