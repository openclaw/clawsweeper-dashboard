## Summary

Keep open. Current main has bundle-declared LSP defaults and a stdio runtime, but it does not implement the requested workspace language scan, package-manager heuristics, PATH discovery, opt-in install/fix flow, update notices, override model, or dashboard UX.

## What This Changes

This issue requests zero-configuration LSP server detection and setup from project files, installed tools, package-manager clues, and editor configuration.

## Best Possible Solution

Keep this as a maintainer design item for a scoped LSP detection API, explicit install authority, documented overrides, status diagnostics, and a plugin or bundle extension point for language-specific behavior.

## Reproduction Assessment

Not applicable as a bug reproduction. The current-main gap is verifiable from source and docs: LSP support is loaded from enabled bundle configuration and spawned only from configured server entries.

## Solution Assessment

No direct core-only auto-installer should be assumed as the best fix. The maintainable path needs product/API/security design for detection, install permission, overrides, diagnostics, and plugin-boundary ownership before implementation.
