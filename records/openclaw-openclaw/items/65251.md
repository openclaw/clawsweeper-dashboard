## Summary

Keep open: current main still only supports static bundle-declared LSP defaults and runtime startup for configured servers; the requested workspace detection, PATH discovery, install/fix flow, override model, update notices, and dashboard remain broader product/API/security work.

## What This Changes

The issue requests zero-configuration LSP server detection, discovery, optional installation, smart configuration, and diagnostics for projects.

## Best Possible Solution

Keep this as a maintainer design item for a scoped LSP detection/lifecycle API, explicit install authority, documented overrides, status diagnostics, and plugin or bundle ownership for language-specific defaults.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature proposal. The current-main gap is source-verifiable from the bundle-backed loader, runtime, docs, and absence of a standalone LSP detection surface.

## Solution Assessment

No direct core auto-installer should be assumed as the best fix. The maintainable path needs product/API/security design for detection, install permission, overrides, diagnostics, and plugin-boundary ownership before implementation.
