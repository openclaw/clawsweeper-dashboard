## Summary

Keep open: current main still provides static bundle-declared LSP defaults and configured stdio server startup, but not workspace/file/PATH auto-detection, install/update flows, overrides, or status/dashboard UX. The related diagnostics proposal is still open at https://github.com/openclaw/openclaw/issues/65252, and the earlier auto-detection PR at https://github.com/openclaw/openclaw/pull/15057 was closed unmerged rather than becoming the canonical implementation.

## What This Changes

The issue requests automatic LSP server detection, setup recommendations, optional installation, configuration, and status UX for zero-configuration language support.

## Best Possible Solution

Keep this as a maintainer design item for a scoped LSP detection/lifecycle API, explicit install authority, documented overrides, diagnostics/status events, and clear plugin or bundle ownership for language defaults.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature proposal. The current-main gap is source-verifiable from the bundle-backed loader/runtime, docs, lack of src/lsp detection surface, and the open related diagnostics proposal.

## Solution Assessment

No, not as a direct unscoped core auto-installer. The maintainable path needs product, API, and security design for detection, install permission, overrides, diagnostics, and plugin-boundary ownership before implementation.
