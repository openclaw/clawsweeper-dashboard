## Summary

Keep open. Current main has static bundle-declared LSP defaults and a stdio runtime, but it still lacks the requested workspace language scan, package-manager/project-file detection, PATH server discovery, opt-in install/fix flow, override semantics, update notices, and dashboard UX.

## What This Changes

Request zero-configuration LSP setup through project detection, installed-server discovery, optional installation/repair, smart configuration, user overrides, update notices, and diagnostics UI.

## Best Possible Solution

Keep this as a product/API design item for explicit, safe workspace detection and installed-server discovery, with opt-in installation/repair, documented overrides, LSP status diagnostics, and a generic plugin or bundle extension point where language-specific behavior belongs outside core.

## Reproduction Assessment

Not applicable as a bug reproduction. The current-main gap is verifiable by source and docs: LSP support is loaded from enabled bundle configuration and spawned only from configured server entries.

## Solution Assessment

No direct core-only implementation should be assumed as the best fix. A maintainable solution needs scoped design for detection, install authority, overrides, diagnostics, and plugin-boundary ownership before code work starts.
