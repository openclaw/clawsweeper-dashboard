## Summary

Keep open: current main has several targeted lifecycle drains and the Feishu cleanup fix has landed, but OpenClaw still does not have the requested shared ResourceManager-style shutdown/cancellation/drain protocol. The remaining work is a maintainer API and ownership decision, not stale cleanup.

## What This Changes

The issue requests a shared async resource lifecycle/shutdown protocol so retry and cleanup paths do not dispose resources while work is still in flight.

## Best Possible Solution

Keep this as the canonical design tracker until maintainers choose a minimal lifecycle primitive, then migrate only proven high-risk retry, shutdown, channel/plugin, and delivery paths with focused contract tests.

## Reproduction Assessment

No, not as a single current-main bug repro. This is an architecture/API proposal built from related bugs; the motivating examples have source evidence and targeted fixes, but the ResourceManager request itself needs design review.

## Solution Assessment

Unclear. A shared lifecycle primitive is plausible, but the best solution needs maintainer agreement on the owner boundary and should reuse existing drain, abort, host-cleanup, and lifecycle-generation primitives where they already fit.
