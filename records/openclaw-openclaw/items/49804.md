## Summary

Keep open. Current main has several scoped lifecycle mechanisms and tactical fixes, but no standard async ResourceManager-style shutdown protocol across agents, channel plugins, gateway delivery recovery, and plugin resources; this is broad architecture work that needs maintainer API/product direction.

## What This Changes

Request to standardize async resource lifecycle management with a shared ResourceManager-like shutdown protocol for retry and cleanup paths.

## Best Possible Solution

Make an explicit maintainer design decision: keep fixing narrow user bugs in their owning paths, and only add a shared lifecycle/resource API after defining the public SDK and shutdown semantics it must guarantee.

## Reproduction Assessment

Not applicable as a broad architecture cleanup request. Source inspection does confirm current main lacks the requested shared ResourceManager-style contract while related concrete bugs are tracked separately.

## Solution Assessment

Unclear. A shared lifecycle manager may be the right end state, but the current request is broader than one safe code repair and needs a maintainer-owned SDK/gateway/agent design decision first.
