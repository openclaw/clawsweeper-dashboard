## Summary

Keep open: current main has several targeted lifecycle/drain fixes and opt-in helpers, but it still does not provide the requested standard cross-surface async resource lifecycle protocol; the remaining decision crosses plugin SDK, gateway shutdown, channel delivery, and agent/session ownership boundaries.

## What This Changes

Request a standard ResourceManager-like shutdown, drain, or cancellation pattern for async operations that can outlive cleanup across retry, channel, and gateway paths.

## Best Possible Solution

Keep this as a maintainer design tracker until the team chooses a minimal lifecycle primitive, then migrate only proven high-risk paths with contract tests instead of adding a broad abstraction by default.

## Reproduction Assessment

Not applicable as one bug reproduction: this is an architecture/API proposal. The motivating examples have separate source or live evidence, but the ResourceManager request itself needs design review rather than one current-main failing path.

## Solution Assessment

Unclear: a ResourceManager-style primitive is plausible, but the best fix is a maintainer-approved minimal lifecycle contract that reuses existing drain and abort primitives where they already fit.
