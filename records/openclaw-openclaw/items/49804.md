## Summary

Keep open: current main has targeted lifecycle fixes for several motivating bugs, but it does not have a standard ResourceManager-style shutdown/cancellation/drain protocol, and the remaining shape crosses agent cleanup, gateway shutdown, outbound delivery, and plugin SDK boundaries that need maintainer design judgment.

## What This Changes

The issue asks for a shared ResourceManager-style lifecycle protocol so async retries, cleanup, and resource disposal do not leak state or drop messages.

## Best Possible Solution

Keep this as a maintainer design tracker until the team chooses a minimal lifecycle primitive, then migrate only proven high-risk paths with contract tests instead of adding a broad abstraction by default.

## Reproduction Assessment

Not applicable as a single bug repro: this issue is an architecture/API proposal. The motivating examples have separate source or live evidence, but the ResourceManager request itself needs design review rather than one current-main failing path.

## Solution Assessment

Unclear. The ResourceManager direction is plausible, but the safest solution is a maintainer-approved minimal lifecycle contract that reuses existing drain primitives where they already fit.
