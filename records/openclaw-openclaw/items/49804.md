- `impact:session-state`: The motivating failures include transcript/context loss and lifecycle state drift when cleanup outruns async work.

## Summary

Keep open: current main has targeted lifecycle drains and abort helpers, including a landed Feishu cleanup fix, but it still does not implement the requested standard ResourceManager-style shutdown/cancellation/drain protocol across retry, gateway shutdown, channel/plugin lifecycle, and outbound delivery paths.

## What This Changes

The issue requests a standard ResourceManager-style async lifecycle protocol for cleanup, retry, shutdown, and pending work boundaries.

## Best Possible Solution

Keep this as the canonical maintainer design tracker until the team chooses a minimal lifecycle primitive, then migrate only proven high-risk retry, shutdown, channel/plugin, and delivery paths with focused contract tests.

## Reproduction Assessment

No, not as a single current-main bug repro. This issue is an architecture/API proposal built from related bugs; the motivating examples have source evidence and targeted fixes, but the ResourceManager request itself needs design review.

## Solution Assessment

Unclear. A shared lifecycle primitive is plausible, but the best solution needs maintainer agreement on the owner boundary and should reuse existing drain, abort, and lifecycle-generation primitives where they already fit.
