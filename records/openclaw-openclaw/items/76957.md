## Summary

Current main still routes typed Control UI `/new` through `new-session` and `sessions.create`, and that create path records parent linkage without emitting `command:new`; the documented hook contract and session-memory behavior therefore remain broken until the linked fix PR lands.

## What This Changes

Restore Control UI typed `/new` hook delivery so parent-session `command:new`, session-memory, and lifecycle hooks run before switching to the new session.

## Best Possible Solution

Land a focused fix that makes parent-linked `sessions.create` preserve the Control UI fresh-session UX while emitting the expected parent-session hook/lifecycle events with regression coverage.

## Reproduction Assessment

Yes, by source inspection: typed Control UI `/new` reaches `sessions.create`, and that current-main branch does not emit `command:new`. I did not run a live hook scenario, so this is source-reproducible rather than fully reproduced.

## Solution Assessment

Yes, the narrow maintainable direction is to fix the parent-linked `sessions.create` path rather than reverting the Control UI fresh-session behavior. PR #77004 should be reviewed against that boundary and the lifecycle-hook expectations.
