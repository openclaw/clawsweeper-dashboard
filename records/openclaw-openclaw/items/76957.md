## Summary

Current main still has the reported regression: Control UI `/new` goes through `sessions.create`, and that path records the parent session but does not fire the documented `command:new` hook. Keep this open because PR #77004 is already open with `Fixes #76957` and should be reviewed or landed rather than closing the issue early.

## What This Changes

The issue reports that Control UI typed `/new` creates a new dashboard session without firing `command:new` hooks for the previous session.

## Best Possible Solution

Review and land PR #77004, or an equivalent narrow gateway fix, so parent-backed `sessions.create` preserves the fresh Control UI session UX while emitting the hook lifecycle against the outgoing parent session with regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: type `/new` in Control UI, follow `onSlashAction("new-session")` to `sessions.create`, and observe that current main stores the parent session but never emits `command:new`.

## Solution Assessment

Yes. Fixing the gateway-side parent-session create path is the narrowest maintainable direction because it preserves the fresh dashboard-session behavior from the earlier UI change while restoring the documented hook contract.
