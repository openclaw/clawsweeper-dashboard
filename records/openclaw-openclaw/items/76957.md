## Summary

This should stay open because current `main` still has the reported Control UI `/new` path and an open fix PR, #77004, is already paired with the issue. Source inspection confirms the path creates a dashboard session without going through the gateway reset hook emitter.

## What This Changes

The issue requests restoring hook-visible Control UI typed `/new` behavior so `command:new`, session-memory, and related reset lifecycle hooks run against the previous session.

## Best Possible Solution

Review and land a gateway-side fix that preserves fresh dashboard session creation while emitting the same parent-session lifecycle hooks that `/new` users and bundled hooks rely on.

## Reproduction Assessment

Yes, from source inspection: Control UI `/new` reaches `sessions.create` with a parent session, and that method does not emit the documented `command:new` hook. A live hook run was not performed because this review is read-only.

## Solution Assessment

Yes, the narrow maintainable direction is to fix the gateway session-creation path used by Control UI `/new`, with regression coverage for parent-session hook emission. The already-open #77004 is the right item to review rather than opening a second repair lane.
