## Summary

Keep open: current main still routes Control UI typed `/new` through `sessions.create`, and that path does not emit the documented `command:new` hook. A linked fix PR exists, so this should be resolved by reviewing and landing that implementation or an equivalent one.

## What This Changes

Typing `/new` in Control UI should fire `command:new` hooks for the previous session when creating a fresh dashboard session.

## Best Possible Solution

Review, rebase, and land #77004 or an equivalent narrow fix that emits the parent-session hook/lifecycle events from `sessions.create` with regression coverage.

## Reproduction Assessment

Yes from source inspection: Control UI typed `/new` reaches `sessions.create` with a `parentSessionKey`, and current `sessions.create` does not emit the documented `command:new` hook. I did not run a live Gateway hook test in this read-only sweep.

## Solution Assessment

Yes, the narrow maintainable direction is to restore parent-session hook emission in the `sessions.create` path without undoing the fresh-dashboard-session behavior from #69599. The linked PR follows that direction but still needs normal review/rebase before this issue can close.
