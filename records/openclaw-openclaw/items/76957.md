## Summary

Keep open: current `main` still routes Control UI typed `/new` through `sessions.create` without emitting the documented `command:new` hook, and the focused fix in #77004 is open but not merged.

## What This Changes

The issue reports a regression where Control UI typed `/new` creates a new session through `sessions.create` but skips `command:new` hooks for the previous session.

## Best Possible Solution

Land #77004 or an equivalent server-side fix that emits the existing parent-session hook contract from `sessions.create` with regression coverage, then close the issue after merge.

## Reproduction Assessment

Yes, by source inspection: Control UI `/new` calls `new-session`, that path sends `sessions.create` with `parentSessionKey`, and current `sessions.create` never emits `command:new`. A live hook-enabled UI run was not executed because this review was read-only.

## Solution Assessment

Yes. The narrow maintainable boundary is the server `sessions.create` parent-session path, so the Control UI can keep the fresh-session behavior from #69599 while restoring the existing hook contract.
