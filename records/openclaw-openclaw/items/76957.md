## Summary

Keep open: current main still routes Control UI typed `/new` through `sessions.create` without the documented `command:new` hook path, and the closing fix PR #77376 is open but unmerged.

## What This Changes

The issue reports that Control UI typed `/new` no longer fires `command:new`, `before_reset`, or session lifecycle hooks after the fresh dashboard-session routing change.

## Best Possible Solution

Review and land PR #77376, or an equivalent explicit `emitCommandHooks` opt-in fix, then close this after merge verification.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: typed `/new` reaches `sessions.create`, and current main has no hook emission in that path while `/reset` does emit hooks.

## Solution Assessment

Yes. The active PR’s explicit `emitCommandHooks` opt-in is the narrow maintainable direction because it restores the Control UI `/new` hook contract without making every SDK `parentSessionKey` create look like a webchat command.
