## Summary

Keep open. Current main still has the Windows PowerShell stale-import regex bug, and the PR fixes that path, but the latest branch also changes maintainer merge/prep helpers and introduces a co-author lookup fallback that can silently drop human contributor trailers on lookup failure.

## What This Changes

The PR updates the Windows Parallels npm update PowerShell stale-import regex and regression test, adds a changelog entry, and also changes PR/changelog maintainer scripts, OpenGrep Bash compatibility, and media-generation test env isolation.

## Best Possible Solution

Keep the regex fix and generated-output regression coverage, but first fix or split the maintainer-flow helper so bot/app authors are skipped intentionally while eligible human contributor lookup failures still fail loudly.

## Reproduction Assessment

Yes. Current main emits a PowerShell regex that a focused pwsh -match probe rejects with `Unrecognized escape sequence \o`, while the doubled-backslash pattern used by the PR matches the intended Windows stale dist path.

## Solution Assessment

Mostly yes for the reported smoke failure: fixing the generated PowerShell regex at the producer boundary with generated-output coverage is the narrow repair. The added maintainer-flow co-author helper should not conflate intentionally skipped bot/app authors with failed lookups for eligible humans.
