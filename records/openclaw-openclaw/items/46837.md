## Summary

Keep this PR open, but it still needs changes before merge: the diagnostic is useful and current main still lacks the missing-extraPaths warning, but the proposed path resolution does not match the builtin memory runtime contract.

## What This Changes

The PR adds an `openclaw doctor` warning and unit coverage for configured `memorySearch.extraPaths` entries that do not exist.

## Best Possible Solution

Add the doctor diagnostic on current main using the same extraPaths normalization as the active memory runtime, with focused coverage for stale absolute and workspace-relative paths.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with builtin memory `extraPaths: ["~/notes"]`, `$HOME/notes` present, and `<workspace>/~/notes` absent, this PR can report healthy while the runtime still looks under the workspace.

## Solution Assessment

No. The diagnostic is the right direction, but the implementation should reuse or mirror `normalizeExtraMemoryPaths` for builtin memory instead of adding a separate tilde-expansion contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9
