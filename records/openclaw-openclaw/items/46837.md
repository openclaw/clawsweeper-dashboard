## What This Changes

The PR adds an `openclaw doctor` warning and tests for configured `memorySearch.extraPaths` entries that resolve to missing files or directories.

## Best Possible Solution

Add the doctor warning by reusing or centralizing `extraPaths` normalization, and keep `~` behavior consistent with the runtime and https://github.com/openclaw/openclaw/issues/58026.

Yes for the PR defect. Source inspection gives a high-confidence path: with `extraPaths: ["~/notes"]`, `$HOME/notes` present, and `<workspace>/~/notes` absent, this branch can suppress the new doctor warning while current builtin indexing still looks under the workspace.

## Solution Assessment

No. The diagnostic is the right direction, but it should share the runtime normalization contract or fold in the canonical tilde runtime fix instead of resolving paths independently.
