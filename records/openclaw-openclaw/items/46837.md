## What This Changes

The PR adds an `openclaw doctor` memory-search warning and tests for configured `memorySearch.extraPaths` entries that do not exist.

## Best Possible Solution

Add the diagnostic on current main using the same `extraPaths` normalization as the memory runtime, cover stale absolute and workspace-relative paths in tests, and add the required changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with builtin memory `extraPaths: ["~/notes"]`, `$HOME/notes` present, and `<workspace>/~/notes` absent, this PR suppresses the warning while runtime normalization still indexes the workspace-relative path.

## Solution Assessment

No. The diagnostic is the right direction, but the implementation should reuse or mirror `normalizeExtraMemoryPaths` instead of creating a separate tilde-expansion contract.
