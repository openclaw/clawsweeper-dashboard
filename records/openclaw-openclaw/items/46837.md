## Summary

Keep open: current main still lacks the proposed `openclaw doctor` missing-`extraPaths` warning, but the PR branch is not ready because its tilde-path handling diverges from the current builtin memory runtime and it still lacks real behavior proof.

## What This Changes

The PR adds a doctor warning and unit tests for configured `memorySearch.extraPaths` entries that resolve to missing files or directories.

## Best Possible Solution

Add the doctor warning by reusing or centralizing the canonical `extraPaths` normalization, and resolve the tilde contract consistently with the open runtime issue at https://github.com/openclaw/openclaw/issues/58026.

## Reproduction Assessment

Yes for the PR defect. Source inspection shows the PR suppresses the new warning for `~/...` when a home-expanded path exists, while current builtin memory still treats the same value as workspace-relative.

## Solution Assessment

No. The diagnostic is useful, but it should share the runtime normalizer or fix the canonical normalizer instead of resolving `extraPaths` independently in doctor.

## Review Findings
