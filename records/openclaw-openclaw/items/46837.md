## Summary

## What This Changes

The PR adds an `openclaw doctor` warning and tests for configured `memorySearch.extraPaths` entries that do not exist.

## Best Possible Solution

Add the diagnostic against the same normalized paths the active memory runtime uses, rebase on current main, add the required changelog entry, and validate the touched doctor and memory-host tests.

## Reproduction Assessment

Yes. With builtin memory configured as `extraPaths: ["~/notes"]`, `$HOME/notes` present, and `<workspace>/~/notes` absent, this PR can report healthy while runtime normalization indexes nothing.

## Solution Assessment

No. The diagnostic is useful, but this implementation is not the best fix because it creates a separate path-resolution contract instead of reusing or mirroring `normalizeExtraMemoryPaths`.
