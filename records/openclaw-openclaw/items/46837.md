## Summary

Keep open: current main still lacks the requested missing-extraPaths doctor warning, but this branch should not merge as-is because the new helper resolves `~` differently from the active memory runtime and the external PR lacks after-fix real behavior proof.

## What This Changes

The PR adds an `openclaw doctor` warning and unit tests for configured `memorySearch.extraPaths` entries that point to missing files or directories.

## Best Possible Solution

Add the doctor diagnostic on current main by reusing or mirroring `normalizeExtraMemoryPaths`, with focused coverage for missing absolute paths, workspace-relative paths, and `~` as workspace-relative input.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with builtin memory `extraPaths: ["~/notes"]`, `$HOME/notes` present, and `<workspace>/~/notes` absent, this PR can suppress the warning while the runtime still resolves the configured path under the workspace.

## Solution Assessment

No. The diagnostic is the right direction, but the implementation should match the existing memory runtime normalization instead of adding a separate home-relative `~` contract.
