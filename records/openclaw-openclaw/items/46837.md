## Summary

## What This Changes

The branch adds an `openclaw doctor` diagnostic and tests that warn when configured `memorySearch.extraPaths` entries resolve to missing paths.

## Best Possible Solution

Add the warning by sharing or centralizing canonical `extraPaths` normalization so doctor, builtin list/read, QMD behavior, tests, and docs agree on the path contract.

## Reproduction Assessment

Yes for the PR defect by source inspection: a `~/notes` entry can be treated as existing by the new doctor helper while builtin `normalizeExtraMemoryPaths()` still resolves it under the workspace for list/read. I did not run a live doctor command in this read-only review.

## Solution Assessment

No. The warning is useful, but the maintainable fix is to reuse or fix the canonical runtime normalizer instead of adding doctor-only path semantics that can diverge from builtin memory.
