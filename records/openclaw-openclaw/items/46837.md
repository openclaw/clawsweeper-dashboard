## Summary

Keep open: the diagnostic is useful, but the branch adds a doctor-only path check that disagrees with the memory runtime and it still lacks real behavior proof, so it should not merge as-is.

## What This Changes

The PR adds an `openclaw doctor` warning and tests for configured `agents.defaults.memorySearch.extraPaths` entries that point to missing paths.

## Best Possible Solution

Add the doctor warning by reusing or centralizing the canonical `extraPaths` normalization, and resolve the home-prefix contract consistently with https://github.com/openclaw/openclaw/issues/58026.

## Reproduction Assessment

Yes for the PR defect by source inspection: the PR helper can resolve a home-prefixed path differently from builtin `normalizeExtraMemoryPaths()`, and both list/read runtime paths still use that normalizer. I did not run a live doctor command in this read-only review.

## Solution Assessment

No. The warning is useful, but the maintainable fix is to share the runtime normalizer or fix the canonical normalizer first instead of adding doctor-only path semantics.
