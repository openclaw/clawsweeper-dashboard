## Summary

Keep open: current main still lacks the missing-extraPaths doctor warning, but this PR needs real behavior proof and a path-normalization fix because its tilde handling can diverge from the builtin memory runtime.

## What This Changes

The branch adds an `openclaw doctor` memory-search warning and tests for configured `memorySearch.extraPaths` entries that resolve to missing files or directories.

## Best Possible Solution

Add the doctor warning by reusing or centralizing the canonical `extraPaths` normalization, and resolve the tilde contract consistently with https://github.com/openclaw/openclaw/issues/58026.

## Reproduction Assessment

Yes for the PR defect. Source inspection shows `extraPaths: ["~/notes"]` can be treated as home-relative by the new doctor helper while builtin memory still resolves the same value under the workspace.

## Solution Assessment

No. The diagnostic is useful, but the PR should share the runtime normalizer or fix the canonical normalizer instead of resolving `extraPaths` independently in doctor.
