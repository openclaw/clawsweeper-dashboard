## Summary

Keep open: the PR addresses a useful doctor diagnostic, but the branch is dirty, lacks real behavior proof, and introduces a doctor-only `extraPaths` resolver that can disagree with the runtime memory path contract.

## What This Changes

The PR adds an `openclaw doctor` warning for configured memory search `extraPaths` entries that do not exist.

## Best Possible Solution

Centralize `extraPaths` resolution so doctor, builtin list/read behavior, QMD collection handling, docs, and tests use the same path contract, then prove the diagnostic with a real doctor run.

## Reproduction Assessment

Yes for the PR defect by source inspection: the proposed helper expands `~/notes` with `resolveUserPath()`, while current builtin memory runtime resolves every non-absolute `extraPaths` value relative to the workspace.

## Solution Assessment

No. The warning should share the runtime `extraPaths` normalization contract instead of introducing a doctor-only resolver that can report health differently from actual indexing.
