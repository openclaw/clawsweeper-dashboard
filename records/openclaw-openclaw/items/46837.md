## Summary

This PR should stay open: current main still lacks the requested doctor warning, but the proposed implementation has a P2 path-contract bug for `~` entries and the external contributor has not provided after-fix real behavior proof.

## What This Changes

The PR adds an `openclaw doctor` warning and unit coverage for configured memory search `extraPaths` entries that do not exist.

## Best Possible Solution

Add the doctor diagnostic on current main using the same `extraPaths` normalization as the active memory runtime, with focused coverage for missing absolute and workspace-relative paths.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with builtin memory `extraPaths: ["~/notes"]`, `$HOME/notes` present, and `<workspace>/~/notes` absent, this PR can report healthy while runtime indexing still checks the workspace-relative path.

## Solution Assessment

No. The diagnostic is the right direction, but the implementation should reuse or mirror `normalizeExtraMemoryPaths` for builtin memory rather than adding a separate tilde-expansion contract.
