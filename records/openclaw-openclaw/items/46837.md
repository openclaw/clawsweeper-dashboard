## Summary

Keep open. Current main still lacks the requested missing-extraPaths doctor warning, but this PR should not merge as-is because its tilde handling can make doctor report healthy while builtin memory resolves the same entry under the workspace and indexes nothing. The branch is also dirty against current main, so it needs revision and rebase before validation.

## What This Changes

The PR adds an `openclaw doctor` memory-search warning for missing configured `memorySearch.extraPaths`, plus unit tests for missing, existing, and tilde-prefixed entries.

## Best Possible Solution

Revise the diagnostic so it reuses or exactly mirrors the active backend path resolver. For builtin memory, check the same normalized workspace-relative extraPaths that indexing and memory reads use; keep any QMD-specific behavior aligned with QMD collection resolution, then rebase and validate the touched doctor and memory-host tests.

## Reproduction Assessment

Yes. For builtin memory with `extraPaths: ["~/notes"]`, `$HOME/notes` present, and `<workspace>/~/notes` absent, this PR's doctor check can suppress the warning while runtime normalization still looks under the workspace and indexes nothing.

## Solution Assessment

No. The requested doctor warning is useful, but the current implementation is not the best fix because it creates a path-resolution contract different from builtin runtime and docs; the safer path is to reuse or mirror `normalizeExtraMemoryPaths` for builtin checks.
