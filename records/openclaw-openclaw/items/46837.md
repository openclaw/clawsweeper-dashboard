## Summary

Keep open: the diagnostic is useful and current main still lacks a missing-extraPaths doctor warning, but this PR needs contributor proof and a code refresh because its doctor-only `~/...` handling can diverge from the builtin memory runtime.

## What This Changes

The branch adds an `openclaw doctor` check and unit tests that warn when configured `agents.defaults.memorySearch.extraPaths` entries do not exist.

## Best Possible Solution

Add the doctor warning through shared `extraPaths` normalization, alongside or after the runtime tilde fix tracked at https://github.com/openclaw/openclaw/issues/58026, then verify it with a real doctor run.

## Solution Assessment

No. The warning should reuse or centralize canonical `extraPaths` normalization so doctor, builtin list/read, QMD behavior, docs, and tests agree.
