## Summary

Keep open: current main still lacks this doctor warning, but the PR is not mergeable as-is because its doctor-only path resolution can disagree with builtin memory indexing/reading, and the external PR still lacks real `openclaw doctor` behavior proof.

## What This Changes

The PR adds an `openclaw doctor` diagnostic and tests that warn when configured `agents.defaults.memorySearch.extraPaths` entries resolve to paths that do not exist.

## Best Possible Solution

Add the warning by reusing or centralizing canonical `extraPaths` normalization, resolve the home-prefix contract tracked in https://github.com/openclaw/openclaw/issues/58026, and then show a real doctor run.

## Reproduction Assessment

Yes for the PR defect by source inspection: a `~/notes` entry can be treated as existing by the new doctor helper while builtin `normalizeExtraMemoryPaths()` still resolves it under the workspace for list/read. I did not run a live doctor command in this read-only review.

## Solution Assessment

No. The warning is useful, but the maintainable fix is to share or fix the canonical runtime normalizer instead of adding doctor-only path semantics that can diverge from builtin memory.
