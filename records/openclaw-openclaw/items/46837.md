The PR adds an `openclaw doctor` warning and tests for configured `memorySearch.extraPaths` entries that do not exist.

## Best Possible Solution

Add the doctor diagnostic on current main using the same `extraPaths` normalization as memory runtime, cover stale absolute and workspace-relative paths, and add the active changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with the PR patch, `extraPaths: ["~/notes"]`, `$HOME/notes` present, and `<workspace>/~/notes` absent, doctor can report healthy while runtime memory lookup still resolves the path under the workspace.

## Solution Assessment

No. The diagnostic is useful, but the implementation should reuse or mirror runtime `extraPaths` normalization instead of adding a separate tilde-expansion contract, and it needs the required changelog entry.
