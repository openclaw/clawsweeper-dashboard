## Summary

Keep open. The report still describes a current docs-platform gap: current main has generated locale navigation but no durable path-preserving Mintlify language-switch handler, the live docs still do not serve the proposed standalone docs-root assets, and the related fix PRs were closed unmerged because the deployment hook was not proven.

## What This Changes

Preserve the current docs page path when switching languages in the Mintlify docs language picker.

## Best Possible Solution

Keep this as the canonical tracker until maintainers either get Mintlify to preserve page paths upstream or add a supported publish/config integration that derives locale mappings from GENERATED_LOCALES and is verified in a Mintlify preview or live deployment.

## Reproduction Assessment

Yes. The issue gives precise live-docs steps and a Puppeteer DOM inspection path; live checks confirm both affected page URLs exist and the standalone script assets remain unavailable.

## Solution Assessment

Unclear. The rejected standalone docs-root JavaScript approach is not the best maintainable fix until Mintlify custom-script deployment is verified; the safer path is an upstream Mintlify fix or a publish-pipeline hook tied to the generated locale source of truth.
