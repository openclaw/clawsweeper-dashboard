## Summary

Keep open. Current main still has no durable path-preserving locale switcher or publish hook, live docs-root custom script paths still return 404, and the related closed PRs show the standalone-script approach is not deployable without maintainer/docs-platform follow-up.

## What This Changes

Request that the Mintlify docs language switcher preserve the current page path when changing locales instead of navigating to the target locale homepage.

## Best Possible Solution

Keep this as the canonical tracker until maintainers either get Mintlify to preserve locale-switch paths upstream or wire a supported publish-pipeline hook that derives locale mappings from GENERATED_LOCALES and is verified in a Mintlify preview or live deployment.

## Reproduction Assessment

Yes. The issue body and existing ClawSweeper comment provide a concrete browser replay, and current live checks still show both equivalent pages exist while docs-root custom script URLs return 404.

## Solution Assessment

Unclear. The requested behavior is correct, but the closed standalone docs-root JavaScript approach is not the best fix unless maintainers first prove how Mintlify serves and configures that hook for the publish repo.
