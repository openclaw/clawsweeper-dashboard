## Summary

Keep open. Current main still has no durable repo-side language path preservation, live docs still reproduce the path loss, and the closed standalone-script PRs did not prove a supported Mintlify deployment hook.

## What This Changes

Request that the Mintlify docs language dropdown preserve the current page path when switching locales.

## Best Possible Solution

Keep this as the canonical tracker until maintainers either get Mintlify to preserve locale-switch paths upstream or wire a supported publish/config hook derived from `GENERATED_LOCALES` and verify it in a Mintlify preview or live deployment.

## Reproduction Assessment

Yes. A live browser check reproduces the English `/install/docker` to Chinese switch landing on `/zh-CN`, while both the source and equivalent localized page return 200.

## Solution Assessment

Unclear. The requested behavior is valid, but the prior standalone docs-root JavaScript approach is not the best fix unless the Mintlify deployment hook is proven; a platform fix or publish-pipeline integration is safer.
