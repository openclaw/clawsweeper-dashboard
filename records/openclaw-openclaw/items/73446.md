## Summary

Keep open. The docs language switcher still loses the page path on live docs, current main has no durable path-preserving locale switcher or proven custom-script hook, and the earlier standalone-script PRs were closed because that approach is unserved and drift-prone.

## What This Changes

The issue asks for docs.openclaw.ai language switching to preserve the current page path instead of navigating to the target locale homepage.

## Best Possible Solution

Keep this as the canonical tracker until maintainers either get Mintlify to preserve locale-switch paths upstream or wire a supported publish/config hook derived from `GENERATED_LOCALES` and verified in a Mintlify preview or live deployment.

## Reproduction Assessment

Yes. A live Chromium replay on May 14, 2026 opened `/install/docker`, selected Simplified Chinese in the Mintlify language menu, and landed on `/zh-CN` even though `/zh-CN/install/docker` returns 200.

## Solution Assessment

Unclear. The requested behavior is correct, but the prior standalone docs-root JavaScript approach is not the best proven fix unless OpenClaw wires and verifies a supported Mintlify asset hook; an upstream Mintlify fix or publish-pipeline integration is safer.
