## Summary

Keep open. Current main still lacks a durable path-preserving Mintlify locale switcher, live docs still do not serve standalone docs-root scripts, and the related standalone-script PRs were closed unmerged because the supported deployment hook and generated locale source of truth were not proven.

## What This Changes

The issue asks for the Mintlify docs language switcher to preserve the current page path when switching between English and localized docs.

## Best Possible Solution

Keep this as the canonical tracker until maintainers either get Mintlify to preserve locale-switch paths upstream or wire a supported docs publish/config hook that derives locale mappings from GENERATED_LOCALES and is validated in a Mintlify preview or live deployment.
