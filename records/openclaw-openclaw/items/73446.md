## Summary

Keep open. Current `main` still has no durable path-preserving locale switcher, and a live Chromium replay against the hosted docs still navigates from the Docker page to the Chinese homepage instead of the equivalent localized page.

## What This Changes

The issue asks for the hosted Mintlify docs language switcher to preserve the current page path when switching between English and localized docs pages.

## Best Possible Solution

Keep this as the canonical tracker until maintainers either get Mintlify to preserve locale-switch paths upstream or wire a supported publish/config hook that derives locale mappings from `GENERATED_LOCALES` and is validated in a Mintlify preview or live deployment.

## Reproduction Assessment

Yes. A live Chromium replay against `docs.openclaw.ai/install/docker` reproduces the issue by switching to Simplified Chinese and landing on `/zh-CN` instead of `/zh-CN/install/docker`.

## Solution Assessment

Unclear. The requested behavior is right, but the earlier standalone docs-root JavaScript approach is not the best path unless OpenClaw proves how Mintlify serves and configures that hook; upstream Mintlify behavior or publish-pipeline integration is safer.
