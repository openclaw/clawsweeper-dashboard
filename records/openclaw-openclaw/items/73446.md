## Summary

Keep open. Current main still has no durable repo-side fix for preserving the docs page path through Mintlify's language switcher, and the live docs deployment still does not serve standalone docs-root JavaScript assets that the closed attempted fixes depended on.

## What This Changes

The issue asks for the Mintlify docs language switcher to preserve the current page path when switching locales, such as `/install/docker` to `/zh-CN/install/docker` instead of the locale homepage.

## Best Possible Solution

Keep this as the canonical tracker until maintainers either get Mintlify to preserve locale-switch paths upstream or wire a supported docs publish/config hook through the sync pipeline, deriving locale mappings from `GENERATED_LOCALES` and verifying the result in a Mintlify preview or live deployment.

## Reproduction Assessment

Yes. The issue gives precise live-docs steps, #34021 reports the same user-visible behavior, and current live checks confirm both target pages exist while the standalone asset paths proposed by prior fixes are not served.

## Solution Assessment

Unclear. The requested behavior is correct, but the standalone docs-root JavaScript approach in #73415 and #73436 is not proven maintainable until the Mintlify deployment hook is confirmed; a supported upstream or publish-pipeline integration is safer.
