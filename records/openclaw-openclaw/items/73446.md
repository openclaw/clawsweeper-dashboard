## Summary

Keep open. Current main still lacks a durable path-preserving Mintlify locale switcher, live docs still do not serve standalone docs-root scripts, and the related standalone-script PRs were closed unmerged because the supported deployment hook and generated locale source of truth were not proven.

## What This Changes

The issue asks for the Mintlify docs language switcher to preserve the current page path when switching between English and localized docs pages.

## Best Possible Solution

Keep this as the canonical tracker until maintainers either get Mintlify to preserve locale-switch paths upstream or wire a supported docs publish/config hook through the sync pipeline, deriving mappings from `GENERATED_LOCALES` and validating in a Mintlify preview or live deployment.

## Solution Assessment

Unclear. The requested behavior is valid, but the closed standalone docs-root JavaScript approach is not maintainable unless the Mintlify deployment hook is proven; the safer path is an upstream Mintlify fix or publish-pipeline integration using the generated locale source of truth.
