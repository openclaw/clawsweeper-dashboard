## Summary

Keep open. Current main still has no durable path-preserving docs locale switcher or proven Mintlify custom-script deployment hook, and the related standalone-script PRs were closed unmerged because that approach was not validated against the docs publish pipeline.

## What This Changes

Request that the Mintlify docs language switcher preserve the current page path when switching between English and localized docs pages.

## Best Possible Solution

Keep this as the canonical tracker until maintainers either get Mintlify to preserve locale-switch paths upstream or wire a supported publish/config hook that derives locale mappings from GENERATED_LOCALES and is validated in preview or live docs.

## Reproduction Assessment

Yes. The issue gives exact live-docs steps and DOM details, and live checks confirm both equivalent pages exist while docs-root script assets are not served; browser dropdown replay is still needed as final fix validation.

## Solution Assessment

Unclear. The requested behavior is valid, but the closed standalone docs-root JavaScript approach is not maintainable unless Mintlify serves it and the mapping is generated from the publish-sync locale source of truth.
