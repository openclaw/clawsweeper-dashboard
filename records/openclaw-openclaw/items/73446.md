## Summary

Keep open. Current main still lacks a durable path-preserving Mintlify locale switcher, the live docs still do not serve standalone docs-root assets, and the related standalone-script PRs were closed unmerged because the deployment hook and locale source of truth were not proven.

## What This Changes

The issue requests that the Mintlify docs language switcher preserve the current page path when switching between English and localized documentation pages.

## Best Possible Solution

Keep this as the canonical tracker until maintainers either get Mintlify to preserve locale-switch paths upstream or wire a supported docs publish/config hook through the sync pipeline, deriving locale mappings from `GENERATED_LOCALES` and validating the behavior in a Mintlify preview or live deployment.

## Reproduction Assessment

Yes. The issue gives exact live-docs steps and DOM details, the older #34021 report describes the same behavior, and live checks confirm the English and Chinese target pages exist while the proposed standalone asset path is not served.

## Solution Assessment

Unclear. The requested behavior is valid, but the closed standalone docs-root JavaScript approach is not the best maintainable fix unless the Mintlify deployment hook is proven; the safer path is an upstream Mintlify fix or a publish-pipeline integration using the generated locale source of truth.
