## Summary

Close as cannot_reproduce: the reported Mintlify dropdown behavior no longer matches the live docs site. ## What This Changes

Report that the docs language switcher navigates to the target locale homepage instead of the equivalent page path.

## Best Possible Solution

Keep the current hosted docs language-picker behavior and route any future regression through the docs publish/rendering path rather than an unreferenced root JavaScript file.

## Reproduction Assessment

No. A current live docs probe shows the language option for Simplified Chinese points to `/zh-CN/install/docker`, so the original path-loss reproduction no longer fails.

## Solution Assessment

Yes. The best current solution is the hosted docs picker preserving equivalent page paths through generated locale navigation, without merging the earlier standalone root JS workaround.
