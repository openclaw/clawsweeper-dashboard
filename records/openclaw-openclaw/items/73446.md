## Summary

Keep open. Current main still lacks a durable path-preserving Mintlify locale switcher, live docs still do not serve docs-root scripts directly, and the related standalone-script PRs were closed unmerged because the supported deployment hook and generated locale source of truth were not proven.

## What This Changes

The issue requests preserving the current docs page path when switching languages in the Mintlify docs locale dropdown.

## Best Possible Solution

Keep this as the canonical tracker until maintainers either get Mintlify to preserve locale-switch paths upstream or wire a supported publish/config hook that derives locale mappings from GENERATED_LOCALES and is validated in a Mintlify preview or live deployment.

## Reproduction Assessment

Yes. The issue provides exact live-docs steps and DOM findings, and live checks confirm both equivalent pages exist while docs-root script paths are not served directly.

## Solution Assessment

Unclear. The requested behavior is valid, but the closed standalone JavaScript approach is not maintainable unless the Mintlify deployment hook is proven; an upstream Mintlify fix or publish-pipeline integration would be safer.
