## Summary

Keep open. Current main still has no durable implementation for preserving docs paths across Mintlify locale switches, and live checks still support the report plus the maintainer concern that standalone docs-root scripts are not being served.

## What This Changes

The issue asks for the docs language switcher to keep the current page path when moving between English and localized docs pages.

## Best Possible Solution

Keep this as the canonical tracker until maintainers either get Mintlify to preserve locale-switch paths upstream or wire a supported docs publish/config hook through the sync pipeline using GENERATED_LOCALES.

## Reproduction Assessment

Yes. The issue gives exact live-docs steps and DOM details, the older #34021 report describes the same behavior, and live checks confirm both equivalent pages exist while standalone docs-root script assets are not served.

## Solution Assessment

Unclear. The requested behavior is valid, but the prior standalone JavaScript approach is not maintainable unless a Mintlify deployment hook is proven; the safer solution is upstream Mintlify behavior or publish-pipeline integration.
