## Summary

Keep open. Current main still lacks general cron job tags/categories across persistence, closed Gateway schemas, service filtering, create/update paths, agent tool/CLI/UI surfaces, tests, and docs; the open Reminders-tab issue depends on this metadata, while the plugin scheduled-turn tag cleanup is a narrow name-prefix cleanup seam rather than a public cron tags contract.

## What This Changes

Add optional freeform cron job tags/categories that persist, appear in cron list/UI output, and support filtering/grouping.

## Best Possible Solution

Add a backward-compatible end-to-end cron tags contract with normalized persisted `tags?: string[]`, Gateway add/update/list validation and filtering, service filtering, agent-tool/CLI/UI display and filter controls, focused tests, docs, and a changelog entry.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a runtime bug, but yes as a feature gap from source inspection: current main has no persisted `tags` field or tag filter in the cron model, Gateway schema, service list path, agent tool, CLI, or UI state.

## Solution Assessment

Is this the best way to solve the issue? Yes for the base feature: an additive `tags?: string[]` contract is more maintainable than name-prefix parsing, while plugin scheduled-turn tag cleanup should remain a separate namespaced helper.
