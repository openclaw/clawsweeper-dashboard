## Summary

Keep open. Current main and the latest release still lack a persisted public cron tags/categories contract; the existing cron surfaces support name, description, agent, enabled state, schedule/status filters, and text search, but not tag metadata or tag filtering.

## What This Changes

Request adds optional freeform cron job tags/categories that persist and can be shown or filtered in CLI, API, and UI surfaces.

## Best Possible Solution

Add an additive cron metadata contract with normalized tag storage, typed filtering in service/protocol/agent tool/CLI/UI, and leave bulk tag operations for a separate maintainer decision.

## Reproduction Assessment

Yes as a source-level feature gap, not as a runtime bug: current main has cron storage, protocol, CLI, and UI filters but no persisted tags field or tag filter path.

## Solution Assessment

Yes at the direction level: normalized optional tags are cleaner than name prefixes, but the exact public metadata, filter, export, and bulk-operation contract needs maintainer product/protocol alignment.
