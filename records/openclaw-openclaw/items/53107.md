## Summary

Keep this issue open. Current main still lacks a cron job tags/categories contract: persisted jobs, Gateway validation/list filtering, service create/update, agent tool schemas, CLI output/options, Control UI filters/forms, and public cron docs all remain name/description/status/schedule oriented. The related cron reports in the provided context cover history, hooks, catch-up, and broader management work, but none supersede the freeform job tag request.

## What This Changes

Add optional freeform tags to cron jobs for organization and filtering across persistence, Gateway/API, CLI, UI, exports, and docs, with bulk tag actions as a nice-to-have.

## Best Possible Solution

Ship an additive cron tags contract: `tags?: string[]` on persisted jobs, create/update/list Gateway schemas with tag filtering, CLI add/edit/list/show support, Control UI create/edit/list/filter controls, agent cron tool support, focused tests, and public docs. Keep bulk tag actions separate until the base metadata and filtering behavior is reviewed.
