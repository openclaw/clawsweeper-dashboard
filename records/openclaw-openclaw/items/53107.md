## Summary

Keep open. Current main still lacks cron tags across persistence, Gateway schemas, service filtering, create/update paths, agent tool, CLI, UI, docs, and tests; the linked Reminders-tab issue remains open, and a related opaque-metadata PR was closed as needing owner-reviewed protocol design.

## What This Changes

Request to add optional freeform tags/categories to cron jobs for persistence, list output, UI/API filtering, and possible tag-based bulk enable/disable.

## Best Possible Solution

Add a backward-compatible end-to-end cron tags contract with normalized persisted `tags?: string[]`, Gateway validation and tag filtering, service filtering, agent-tool/CLI/UI display and controls, focused tests, docs, and a changelog entry; defer bulk tag operations unless accepted as part of the design.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a runtime bug, but yes as a feature gap from source inspection: current main has no persisted cron tags or tag filter in the model, Gateway schema, service list path, agent tool, CLI, or UI.

## Solution Assessment

Is this the best way to solve the issue? Yes at the direction level: additive tags are safer than name-prefix parsing, but the exact public cron API/UI contract needs maintainer design review before implementation.
