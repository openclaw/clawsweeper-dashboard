## Summary

This PR targets the correct Anthropic request boundary, but it leaves one effective-disabled state unhandled and its supplied proof covers only a mocked payload rather than the reported persistent-session workflow.

## What This Changes

The PR passes the current thinking setting into Anthropic message conversion, converts readable historical thinking blocks to text and drops redacted blocks when thinking is explicitly disabled, and adds a captured-payload regression test.

## Best Possible Solution

Derive one effective Anthropic thinking mode before payload conversion, then omit incompatible prior-turn thinking for every disabled request while preserving mandatory-thinking models and any blocks required by an active tool-use turn.

## Reproduction Assessment

Yes at the source level: a request can replay signed same-model thinking while omitting the current Anthropic thinking configuration, matching the linked persistent-session sequence. No live current-main Anthropic call was independently reproduced.

## Solution Assessment

No, not yet. Request-boundary normalization is the right design, but it must use the effective request mode rather than only `thinkingEnabled === false`.
