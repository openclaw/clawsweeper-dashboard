## What This Changes

The PR replaces duplicated Windows/macOS/iOS manual checkout steps with one YAML-anchored platform checkout step and retries timeout-style fetch failures up to three times.

## Best Possible Solution

Land the shared retry wrapper after maintainer acceptance of actionlint, guard coverage, and exact-head CI, or require a small simulated 124/137 timeout proof if deterministic retry-branch coverage is mandatory.

## Reproduction Assessment

No deterministic current-main reproducer was established. The linked GitHub Actions attempt failed in Checkout and the same run passed on rerun, while current main exits after one timeout-style fetch failure.

## Solution Assessment

Yes, this is the best scoped fix I found: it preserves the non-GNU manual watchdog needed by platform jobs and mirrors an existing sibling retry pattern. Duplicating four separate checkout scripts or switching platform jobs to GNU timeout would be less maintainable or less portable.
