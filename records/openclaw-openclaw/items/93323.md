## Summary

Keep open: current main and the latest release still contain the non-terminal direct-announce branch that maps completion handoff pending to a failed delivery, while the split PR that claimed to close this issue was closed unmerged and the remaining open PRs are only candidate fixes.

## What This Changes

Fix subagent completion announcement delivery so pending direct-announce gateway responses are not treated as failed and dropped across spawn depths.

## Best Possible Solution

Land a maintainer-approved subagent announce delivery fix so accepted, started, and in_flight direct announces keep or durably deliver completion instead of exhausting the retry budget, with focused regression coverage and real gateway proof.

## Reproduction Assessment

Yes at source level, but not as a live current-main run in this review. Current main still maps a non-terminal direct completion handoff to completion_handoff_pending, and the registry retry cap can then give up after three failed attempts.

## Solution Assessment

No current-main solution is present yet. The maintainable fix should change the subagent announce delivery/retry contract for non-terminal direct announces, then prove it against the gateway path rather than closing from an unmerged candidate PR.
