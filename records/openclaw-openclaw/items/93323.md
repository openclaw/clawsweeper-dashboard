## What This Changes

The issue asks OpenClaw to stop dropping subagent completion announcements when a direct announce returns a non-terminal pending status under load.

## Best Possible Solution

Land one canonical, proof-backed fix that keeps non-terminal completion handoffs from consuming the short give-up budget and preserves requester wake/session-state semantics under real gateway load.

## Solution Assessment

Unclear as a final implementation choice. The fix belongs in subagent announce delivery/retry handling, but maintainers still need to choose between uniform pending success, durable queueing, or retry/wait semantics and require real gateway proof.
