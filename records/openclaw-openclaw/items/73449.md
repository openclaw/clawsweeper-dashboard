## Summary

Keep this PR open. Current main still lacks the Codex app-server normalized tool event bridge, and the PR is aimed at the right surface, but the latest head still emits unsanitized tool args/results through the agent event bus and is missing the required user-facing changelog entry.

## What This Changes

The PR adds Codex app-server projector support for normalized `stream: "tool"` start, update, and result events, plus projector tests for command events, result deduplication, and non-full-output redaction of bulky Codex payloads.

## Best Possible Solution

Keep the Codex tool-stream bridge, but route start args, streamed partial results, and final result payloads through a shared SDK-safe sanitizer that matches the Pi tool stream, add regression tests with secret-bearing command/MCP/dynamic/image/full-output payloads, and add a single-line Unreleased/Fixes changelog entry with appropriate contributor credit.

## Reproduction Assessment

Yes. Instantiate `CodexAppServerEventProjector` with `onAgentEvent`, send Codex command/MCP/dynamic notifications containing secret-like args or output, and inspect the emitted `stream: "tool"` events on the PR head; current main also has a high-confidence bug reproduction because the same path emits no normalized Codex tool events at all.

## Solution Assessment

No, not as-is. The PR targets the narrow correct projector surface, but the safest maintainable solution must preserve the producer-side redaction guarantee already present in the Pi tool stream and restore the required changelog coverage.
