## Summary

Keep open. Current main still has the all-or-nothing doctor migration path, and the linked fixing PR is open and unmerged, so this issue should remain the tracker until that PR lands or is replaced.

## What This Changes

The issue reports that `openclaw doctor --fix` drops safe legacy config migrations when the migrated config still has an unrelated plugin/provider validation error.

## Best Possible Solution

Land a narrow doctor-flow fix that makes safe legacy migrations independently committable and preserves unresolved validation errors for follow-up, with regression coverage for a migrated config that still has a missing provider/plugin issue.

## Reproduction Assessment

Yes. From source inspection, a migrated config that still fails plugin-aware validation returns `config: null`, so the caller keeps the unmigrated candidate; I did not run a live `doctor --fix` repro because this review is read-only.

## Solution Assessment

Yes. The narrow maintainable direction is to persist only known-safe legacy migration changes independently, while still reporting unrelated validation errors and avoiding a general invalid-config write bypass.
