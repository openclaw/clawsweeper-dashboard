## Summary

Keep open. Current main still matches the reported all-or-nothing doctor migration path, and #76800 is an open, unmerged closing PR, so this issue should continue tracking the bug until that implementation lands or is closed.

## What This Changes

The issue asks for `openclaw doctor --fix` to persist safe legacy config migrations even when unrelated plugin/provider validation errors remain.

## Best Possible Solution

Doctor should persist safe legacy migration deltas independently and continue reporting unresolved plugin/provider validation issues without swallowing the migration.

## Reproduction Assessment

Yes. Source inspection on current main shows `migrateLegacyConfig` returns `config: null` after post-migration plugin validation fails, and the caller then keeps the unmigrated state before the later write can fail on the unrelated issue.

## Solution Assessment

Yes. The narrow maintainable fix is to make legacy migrations independently committable while preserving validation feedback for remaining issues; #76800 is the current implementation candidate to review.
