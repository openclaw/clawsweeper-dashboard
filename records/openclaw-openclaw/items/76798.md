Keep open. Current main still has the all-or-nothing doctor migration path, and the open closing PR #76800 should be reviewed or merged before this issue is resolved.

## What This Changes

Fix `doctor --fix` so safe legacy config migrations are retained and persisted even when unrelated plugin/provider validation errors remain.

## Best Possible Solution

Keep the issue open and review #76800 against the full persistence path so safe legacy migrations can be committed independently without hiding unresolved provider/plugin errors.

## Reproduction Assessment

Yes. Source inspection on current main shows a migrated config that still fails plugin-aware validation returns `config: null`, the caller keeps the unmigrated candidate, and the later write path rejects the remaining unrelated validation issue.

## Solution Assessment

Yes, the requested direction is the right boundary: decouple safe legacy migration persistence from unrelated plugin/provider validation. The implementation should verify the final write path can persist only the safe migration delta rather than merely carrying an invalid migrated candidate forward.
