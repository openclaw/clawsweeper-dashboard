Full review comments:

- **[P1] Remove the stray cron test braces:** `src/cron/service.jobs.test.ts:521-522`
  - body: The PR head has two extra closing lines after the new providers patch test, before the next `it(...)`. That leaves the test file syntactically invalid and explains the cron-service validation failure.
  - confidence: 0.98
- **[P1] Add providers to the public cron protocol schema:** `src/cron/types.ts:260`
  - body: `CronAgentTurnPayload` now accepts `payload.providers`, but `cron.add` and `cron.update` validate through the gateway-protocol schema, whose agent-turn payload still has `additionalProperties: false` and no `providers` field. Public RPC callers will reject the new cron header override before it reaches the service.
  - confidence: 0.94
- **[P1] Normalize cron provider payloads before merging:** `src/cron/isolated-agent/run.ts:564-567`
  - body: This merge copies `payload.providers` into `agents.list[].providers`, so a raw persisted or direct cron payload shaped with `request.auth`, `proxy`, `tls`, or `allowPrivateNetwork` can reach full provider transport policy instead of a cron headers-only boundary. Add a dedicated headers-only normalizer before the job override enters agent config.
  - confidence: 0.9
- **[P1] Persist cron provider payloads through the store:** `src/cron/types.ts:260`
  - body: Adding `providers` to the persisted cron payload type is not enough because the SQLite payload codec enumerates agent-turn fields and currently serializes model, fallbacks, thinking, timeout, lightContext, and toolsAllow only. Saved jobs would lose `payload.providers` after a store round trip.
  - confidence: 0.88
- **[P2] Forward overrides through static catalog fallback:** `src/agents/embedded-agent-runner/model.ts:1567-1577`
  - body: `resolveModelAsync` forwards `agentId` and `jobProviderRequest` through explicit and dynamic resolution, but the bundled static catalog fallback calls `applyConfiguredProviderOverrides` without those values. Runs resolved through that fallback will silently drop the new per-agent or per-job request headers.
  - confidence: 0.84
