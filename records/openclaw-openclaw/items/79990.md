## Summary

Keep this PR open: it is a plausible feature direction, but the branch still leaves the public cron RPC/schema contract out of sync and introduces persisted job-scoped headers without an approved validation/redaction policy.

## What This Changes

The PR adds per-agent provider request overrides, job-scoped cron provider headers, global-to-agent-to-job precedence, cron payload patch persistence, and focused resolver/service tests.

## Best Possible Solution

Align the additive agent config and cron protocol contracts, validators, generated/client surfaces, docs, and persisted-header policy, then prove the full Gateway cron create/update/run path against a mock provider before merge.

## Reproduction Assessment

Yes for the PR findings by source inspection: the branch adds an internal cron payload providers field while current main's closed Gateway cron schema does not accept it, and the branch includes an unused import under a noUnusedLocals core typecheck. These are proposed-patch failure paths, not current-main bugs.

## Solution Assessment

No. The per-agent override direction may be viable, but this branch is not the narrowest merge-ready solution until the public cron protocol boundary, generated/client surfaces, docs, and persisted-header security policy are updated together.
