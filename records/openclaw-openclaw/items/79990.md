## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Restrict cron provider payloads before merging:** `src/cron/isolated-agent/run.ts:544-546`
  - body: This copies the mutable cron payload provider object into agent provider config, but the gateway/tool normalization paths preserve unknown payload fields and the schema allows additional properties. A caller can therefore send `request.auth`, `request.proxy`, `request.tls`, or `allowPrivateNetwork` in `payload.providers` and have the normal full provider-request sanitizer apply it, despite the PR's stated headers-only cron boundary. Sanitize or rebuild this object from `request.headers` only before merging it into agent config, and add a regression test for rejected or stripped transport keys.
  - confidence: 0.91
- **[P1] Persist cron provider overrides in the store:** `src/cron/types.ts:249-250`
  - body: Adding `providers` to `CronPayload` is not enough for scheduled jobs: the SQLite payload codec still binds and reconstructs only the existing explicit payload columns. A job created or patched with `payload.providers` will appear to work in memory, then lose the provider headers after the cron store is saved and reloaded. Add a persisted JSON column/codec or other canonical storage path plus reload coverage before exposing this field.
  - confidence: 0.9
