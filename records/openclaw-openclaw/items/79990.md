## Summary

Keep open: the feature is useful, but this PR still leaves the advertised cron job header path unreachable through the public Gateway schema and needs a defined policy for secret-bearing persisted cron headers.

## What This Changes

Adds agent-level provider request override config, cron payload provider header overrides, runtime merge plumbing, and tests across provider request, PI model resolution, cron service, plus a gateway session-send test cleanup.

## Best Possible Solution

Land a version that updates the public cron protocol schema/client/docs, defines safe cron header storage and redaction, and keeps agent-level overrides in the provider-request plumbing with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection of PR head shows `payload.providers` added to internal cron payload types while the public cron agent-turn payload schema remains closed without `providers`; `cron.add` and `cron.update` validate through that schema before saving.

## Solution Assessment

No. The agent-level provider request plumbing is a plausible direction, but the cron job portion needs the public Gateway contract and a secret-handling policy before this is safe to merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Add providers to the cron protocol schema:** `src/cron/types.ts:205`
  - body: This adds `payload.providers` to the internal cron payload type and service merge path, but `cronAgentTurnPayloadSchema` still has no `providers` field and is closed with `additionalProperties: false`.   - confidence: 0.87
