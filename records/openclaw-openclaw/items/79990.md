## Summary

Keep open: the PR has a useful direction and sufficient copied live-output proof, but it still has blocking correctness gaps in the cron gateway protocol and resolved-agent model path, plus unrelated generated Canvas hash drift.

## What This Changes

The branch adds agent-level and cron-job-level provider request overrides, merges global/agent/job precedence into model resolution, and adds provider-request and cron payload persistence tests.

## Best Possible Solution

Carry the resolved agent id into model resolution, expose the cron providers shape through the gateway protocol and generated surfaces, remove unrelated generated drift, then let maintainers review the new config-surface policy.

## Reproduction Assessment

Yes for the review blockers by source inspection: the cron protocol schema is closed and lacks providers, and runEmbeddedPiAgent resolves an effective agent id but the PR passes only params.agentId into model resolution.

## Solution Assessment

No. The direction is plausible, but the current patch is not the narrowest complete solution until it aligns the public cron protocol, uses the resolved agent id, and removes unrelated generated output.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Expose providers in the cron protocol schema:** `src/cron/types.ts:167-168`
  - body: Adding `CronPayload.providers` only to the internal TS type/service path leaves the gateway protocol closed: `cronAgentTurnPayloadSchema` still uses `additionalProperties: false` and has no `providers` member, so `cron.add`/`cron.update` requests carrying the new field are rejected before persistence. Add the provider request schema to `src/gateway/protocol/schema/cron.ts` and regenerate the protocol artifacts.
  - confidence: 0.9
- **[P2] Use the resolved agent id for model requests:** `src/agents/pi-embedded-runner/run.ts:529`
  - body: These `resolveModelAsync` calls run after `resolveRunWorkspaceDir` has already resolved an agent from explicit input, session key, or the default config. Passing only `params.agentId` means default-agent and session-key-selected runs leave `options.agentId` undefined, so their `agents.list[].providers` overrides are skipped. Pass `workspaceResolution.agentId` to both model-resolution calls and cover that path.
  - confidence: 0.88
- **[P3] Drop the unrelated Canvas bundle hash change:** `extensions/canvas/src/host/a2ui/.bundle.hash:1`
  - body: This provider-request PR still changes the generated A2UI bundle hash even though it does not change the Canvas bundle; repo guidance says that hash is generated and should be ignored unless `pnpm canvas:a2ui:bundle` is intentionally run and committed separately. Remove this file from the branch to avoid a stale generated-artifact mismatch.
  - confidence: 0.86
