- **[P2] Add providers to the cron protocol schema:** `src/cron/types.ts:196-198`
  - body: The PR adds `payload.providers` to internal cron types and service merge code, but the public cron agentTurn payload schema remains closed with `additionalProperties: false` and no `providers` field. `cron.add` and `cron.update` validate through that schema before persistence, so supported Gateway calls carrying job-level headers are rejected before the new code can run.
  - confidence: 0.95
- **[P2] Persist provider headers through cron row reconstruction:** `src/cron/service/jobs.ts:872`
  - body: `buildPayloadFromPatch` now accepts `providers`, but cron runtime jobs are persisted and reloaded through `bindPayloadColumns`/`payloadFromRow`, whose normalized column set does not include provider overrides. A job can therefore lose its job-level headers after a Gateway restart, causing scheduled provider requests to silently run without the configured headers.
  - confidence: 0.9
- **[P2] Define cron header redaction before readback:** `src/cron/types.ts:196-198`
  - body: `payload.providers.*.request.headers` accepts arbitrary persisted header names and SecretInput values, while cron read methods return job objects/pages. Before this public payload field is accepted, credential-like headers such as `Authorization` or `Cookie` need a redaction, denylist, or SecretRef-only policy so cron readers do not get raw secrets.
  - confidence: 0.86
