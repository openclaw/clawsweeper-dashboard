Review and land https://github.com/openclaw/openclaw/pull/68304 or an equivalent narrow Telegram-owned guard with regression coverage, then track any remaining live-stream flash separately.

Yes. Source inspection on current main shows a direct `sendMessageTelegram(..., "NO_REPLY", ...)` has no exact silent-token early return before the Telegram `api.sendMessage` path, while Slack has that guard.

## Solution Assessment
