## What This Changes

Add a Telegram `sendMessageTelegram` guard that suppresses an exact `NO_REPLY` sentinel before calling the Telegram API when there is no media or inline keyboard.

## Best Possible Solution

Review and land #68304 or an equivalent narrow Telegram send-level guard with focused regression coverage; keep broader stream-buffering changes as a separate follow-up if still needed.

Yes. Source inspection shows a direct `sendMessageTelegram(..., "NO_REPLY", ...)` has no exact silent-token early return before the Telegram API send path, while Slack has that guard.

## Solution Assessment

Yes for the defensive-coding gap: the narrow maintainable fix is to add the Telegram send-level exact-sentinel guard with regression coverage, matching Slack without changing global streaming behavior.
