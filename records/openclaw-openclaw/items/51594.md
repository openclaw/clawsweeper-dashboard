## Summary

Keep this PR open: it addresses a real current-main cron stagger bug with concrete production impact, but the branch still has a correctness blocker in comma-list parsing and lacks required after-fix real behavior proof.

## What This Changes

The PR adds a minute-field helper and tests so cron top-of-hour stagger detection treats selected step, list, range, range-step, and wildcard minute fields as including minute zero.

## Best Possible Solution

Land a Croner-compatible minute-zero helper with recursive comma-list parsing, no unsupported numeric-prefix step coverage, persisted-path regression tests, and redacted after-fix proof, then close the linked bug once it reaches main.

## Reproduction Assessment

Yes. Source inspection on current main shows resolveDefaultCronStaggerMs still reaches literal minute-field equality checks, and the PR discussion includes concrete production impact for step cron expressions.

## Solution Assessment

No. The PR targets the right helper, but the current patch should not merge until comma-separated entries recurse through range/step syntax and unsupported Croner numeric-prefix step support is removed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Recurse through comma-separated minute entries:** `src/cron/stagger.ts:28`
  - body: `minuteFieldIncludesZero` returns from the comma branch before checking range or step syntax inside each list item. Croner parses list entries recursively, so expressions like `0-5,30 * * * *` and `*/15,45 * * * *` fire at minute zero but this helper returns false, leaving those jobs without the default stagger this PR is meant to add.
  - confidence: 0.94
- **[P3] Remove unsupported numeric-prefix step support:** `src/cron/stagger.ts:23`
  - body: OpenClaw constructs Croner with default options, and Croner 10 rejects numeric-prefix stepping such as `0/15` unless `sloppyRanges` is enabled. This helper and its new test should not lock in `0/15` as supported OpenClaw cron syntax; use Croner-valid forms such as `*/15` or `0-59/15` instead.
  - confidence: 0.86
