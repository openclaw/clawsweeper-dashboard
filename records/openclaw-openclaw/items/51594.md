## Summary

Keep open. Current main still has the reported strict minute-field check, and this PR is useful but not merge-ready because the proposed helper misses Croner-valid composite list entries and adds unsupported numeric-prefix step semantics.

## What This Changes

The PR adds a minute-field helper and tests so top-of-hour cron stagger detection treats step, list, range, range-step, and wildcard minute fields as including minute zero.

## Best Possible Solution

Update and land a narrow cron-stagger helper fix that mirrors Croner-compatible minute-field semantics for both 5-field and 6-field schedules, extends regression coverage through persisted default-stagger paths, adds the required changelog entry, and then lets the linked bug close from the merged fix.

## Reproduction Assessment

Yes. Current main can be reproduced by source inspection and a read-only helper probe: cron expressions such as */15 * * * *, 0,30 * * * *, and 0-5 * * * * return false even though they fire at minute zero.

## Solution Assessment

No. The PR targets the right narrow helper, but it should recurse through comma-separated entries and avoid recognizing numeric-prefix stepping that OpenClaw's Croner configuration rejects.

  - body: `minuteFieldIncludesZero` returns from the comma branch before checking range or step syntax inside each list item.
