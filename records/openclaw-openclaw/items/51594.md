## Summary

Keep open: current main and the latest release still have the strict minute-field check, but this PR cannot merge yet because the proposed helper still diverges from Croner semantics and external real behavior proof is missing.

## What This Changes

This PR adds a minute-field helper and tests so cron top-of-hour stagger detection treats step, list, range, range-step, and wildcard minute fields as firing at minute zero.

## Best Possible Solution

Land a Croner-compatible minute-zero helper with recursive comma-list parsing, unsupported numeric-prefix step coverage removed, focused persisted-path regression coverage, and contributor real behavior proof before merge.

## Reproduction Assessment

Yes. Source inspection on origin/main shows `resolveDefaultCronStaggerMs` still reaches literal minute-field equality checks, so expressions such as `*/15 * * * *` and `0,30 * * * *` take the no-default-stagger path even though Croner treats them as minute-zero schedules.

## Solution Assessment

No. The PR changes the right helper, but it needs recursive comma parsing and Croner-compatible step syntax before it is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Recurse through comma-separated minute entries:** `src/cron/stagger.ts:28`
  - body: `minuteFieldIncludesZero` returns from the comma branch before checking range or step syntax inside each list item. Croner parses list entries recursively, so expressions like `0-5,30 * * * *` and `*/15,45 * * * *` fire at minute zero but this helper still returns false, leaving those jobs without the default stagger this PR is meant to add.
  - confidence: 0.94
- **[P3] Drop unsupported numeric-prefix step handling:** `src/cron/stagger.ts:23`
  - body: OpenClaw pins Croner 10.0.1 with default options, and Croner rejects numeric-prefix stepping such as `0/15` unless sloppy range parsing is enabled. The helper and tests should not lock in unsupported OpenClaw cron syntax; use Croner-valid forms such as `*/15` or `0-59/15` instead.
  - confidence: 0.86
