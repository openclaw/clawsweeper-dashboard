## Summary

Keep open: this PR targets a real cron default-stagger bug that is still present on current main and the latest release, but the branch still has Croner-semantics gaps and lacks required real behavior proof before merge.

## What This Changes

This PR adds a minute-field parser in `src/cron/stagger.ts` and helper tests so default top-of-hour staggering applies to step, list, range, range-step, and wildcard minute fields.

## Best Possible Solution

Land a Croner-compatible minute-zero helper with recursive list parsing, Croner-valid step coverage, and focused regression coverage across helper, create-job, normalization, and doctor migration paths.

## Reproduction Assessment

Yes. Source inspection on current main shows `resolveDefaultCronStaggerMs` still reaches literal minute-field equality checks, so Croner-valid minute-zero expressions such as `*/15 * * * *` and `0,30 * * * *` take the no-default-stagger path.

## Solution Assessment

No. The PR changes the right helper, but it should follow Croner semantics by recursively parsing comma entries and avoiding unsupported `0/N` step syntax before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Recurse through comma-separated minute entries:** `src/cron/stagger.ts:28-29`
  - body: `minuteFieldIncludesZero` returns from the comma branch before parsing range or step syntax inside each list item. Croner parses comma entries recursively, so expressions like `0-5,30 * * * *` and `*/15,45 * * * *` fire at minute zero but this helper returns false, leaving those jobs without the default stagger this PR is meant to add.
  - confidence: 0.94
- **[P3] Remove unsupported numeric-prefix step coverage:** `src/cron/stagger.ts:23-24`
  - body: OpenClaw uses Croner with default options, and Croner 10 rejects numeric-prefix stepping such as `0/15` unless sloppy range parsing is enabled. This helper and its new test should not lock in `0/15` as supported OpenClaw cron syntax; use Croner-valid forms such as `*/15` or `0-59/15` instead.
  - confidence: 0.86
