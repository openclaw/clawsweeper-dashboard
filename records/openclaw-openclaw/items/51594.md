## Summary

Keep open: current main still has the strict minute-field equality bug, but this PR is not ready because the helper diverges from Croner semantics and the external contributor has not posted required after-fix real behavior proof.

## What This Changes

The PR adds minute-field parsing to cron top-of-hour stagger detection for steps, lists, ranges, range-steps, and wildcards, plus helper-level regression assertions.

## Best Possible Solution

Land a Croner-compatible minute-zero helper with recursive comma-list parsing, unsupported numeric-prefix step coverage removed, focused regression coverage for the persisted default-stagger paths, and contributor real behavior proof before merge.

## Reproduction Assessment

Yes. Source inspection on current main shows resolveDefaultCronStaggerMs still reaches literal minute-field equality checks, so expressions such as */15 * * * * and 0,30 * * * * take the no-default-stagger path even though Croner treats them as minute-zero schedules.

## Solution Assessment

No. The PR changes the right helper, but it needs recursive comma parsing and Croner-compatible step syntax before it is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Recurse through comma-separated minute entries:** `src/cron/stagger.ts:28`
  - body: minuteFieldIncludesZero returns from the comma branch before checking range or step syntax inside each list item. Croner parses list entries recursively, so expressions like 0-5,30 * * * * and */15,45 * * * * fire at minute zero but this helper still returns false, leaving those jobs without the default stagger this PR is meant to add.
  - confidence: 0.94
- **[P3] Remove unsupported numeric-prefix step coverage:** `src/cron/stagger.ts:23`
  - body: OpenClaw constructs Croner with default options, and Croner 10 rejects numeric-prefix stepping such as 0/15 unless sloppyRanges is enabled. This helper and its new test should not lock in 0/15 as supported OpenClaw cron syntax; use Croner-valid forms such as */15 or 0-59/15 instead.
  - confidence: 0.86
