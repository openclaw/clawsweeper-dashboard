## Summary

Keep this open: current main and v2026.6.8 still hydrate bot-authored Telegram reply-chain media, and current tests assert that behavior in at least one allowed-context case, so this needs maintainer contract review rather than cleanup closure.

## What This Changes

The issue reports that Telegram reply-media resolution treats media from a bot-authored reply target as inbound user attachment media.

## Best Possible Solution

Decide the Telegram reply-context contract explicitly, then update the reply-chain media resolver and regression tests so bot-authored media is either intentionally preserved as context or intentionally excluded from inbound attachments.

## Reproduction Assessment

Yes from source inspection: current main and v2026.6.8 can fetch media from bot-authored reply-chain nodes when the node has inbound media and passes visibility checks. I did not run live Telegram or tests because this review was required to keep the checkout read-only.

## Solution Assessment

Unclear: the requested self-authored skip is a plausible narrow fix, but current tests and docs preserve reply-chain media hydration, so maintainers need to decide the intended contract before implementation.
