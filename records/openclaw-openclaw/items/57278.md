## Summary

Keep this open for maintainer contract review: current main and the latest release still hydrate Telegram reply-chain media from bot-authored messages, but focused tests and the closed follow-up discussion show that behavior may be intentional context rather than a straightforward bug.

## What This Changes

The issue asks Telegram reply-media resolution to skip bot-authored replied-to photos so prior bot media is not attached to the next inbound user turn.

## Best Possible Solution

Make the Telegram reply-chain media contract explicit, then either close this as intended behavior or land a narrow fix with regression coverage for both bot-authored and user-authored reply media.

## Reproduction Assessment

Yes from source inspection: current main can fetch media from bot-authored reply-chain nodes when they have inbound media and pass visibility checks. I did not run live Telegram or tests because this review is read-only.

## Solution Assessment

Unclear: a self-authored skip is a plausible narrow change, but current tests and docs preserve bot-authored reply-chain media as context. Maintainers should choose the intended contract before implementation.
