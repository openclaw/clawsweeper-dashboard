## Summary

Keep open: current main still resolves Telegram reply-target media without checking whether the replied-to message was authored by the bot, and open PR #57298 uses closing syntax for this issue but has not merged.

## What This Changes

The issue asks Telegram reply-media resolution to skip bot-authored reply targets so a user text reply to a bot-sent photo does not re-inject the bot's own image as inbound media.

## Best Possible Solution

Land or replace PR #57298 with a runtime guard before getFile plus regression coverage for user replies to bot-sent photos.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a user text message with reply_to_message.photo from ctx.me reaches resolveReplyMediaForMessage, which downloads the reply media before any self-authored reply-target check.

## Solution Assessment

Yes. The narrow maintainable fix is to return [] for bot-authored reply targets before resolving the file id or calling getFile, while preserving reply media from users and other bots.
