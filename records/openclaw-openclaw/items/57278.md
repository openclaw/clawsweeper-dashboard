## Summary

Keep this open: current main and v2026.6.6 still fetch media from bot-authored Telegram reply-chain nodes, the adjacent test currently expects that fetch, and the only open linked closing PR is conflicting against an older implementation shape.

## What This Changes

The issue requests that Telegram reply-media resolution skip media from bot-authored replied-to messages so generated bot media is not injected as inbound user attachment context.

## Best Possible Solution

Add a current-main guard in resolveReplyMediaForChain that skips media fetches for self-authored source nodes while preserving reply-chain text/context, then update regression coverage and require real Telegram proof.

## Reproduction Assessment

Yes from source: current main still resolves and fetches media from each reply-chain sourceMessage when it has inbound media, and the adjacent test asserts fetching a bot-authored generated photo. I did not run a live Telegram repro in this read-only sweep.

## Solution Assessment

Unclear pending maintainer decision: the narrow guard is the likely best code shape, but the existing test contract currently preserves the opposite behavior and should be intentionally changed or confirmed.
