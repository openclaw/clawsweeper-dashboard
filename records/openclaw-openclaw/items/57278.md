## Summary

Keep open: current main still fetches Telegram reply-chain media without checking whether the replied-to chain node was authored by the bot, and the open unmerged closing PR at https://github.com/openclaw/openclaw/pull/57298 already tracks the fix path.

## What This Changes

Prevent Telegram replies to OpenClaw bot-sent photos from re-injecting that bot media as inbound user attachments.

## Best Possible Solution

Refresh or replace the linked PR with a numeric bot-id guard in resolveReplyMediaForChain before bot.api.getFile, while preserving reply media from users and other bots and adding focused regression coverage plus redacted live Telegram proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a Telegram text update whose reply_to_message.photo is authored by ctx.me enters the reply chain and reaches bot.api.getFile before any self-authored reply-target check.

## Solution Assessment

Yes. The narrow maintainable fix is to skip only reply-chain media whose numeric sender id matches the current bot id before fetching the file, then keep user and other-bot reply media working.
