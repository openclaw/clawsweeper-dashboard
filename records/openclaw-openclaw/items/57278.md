## Summary

Keep open: current main still has the source-level Telegram reply-media path that can fetch media from a bot-authored reply target, and open linked PRs are already carrying the proposed fix path.

## What This Changes

The issue asks OpenClaw to stop Telegram reply-media resolution from downloading a bot-authored replied photo and injecting it as inbound user media.

## Best Possible Solution

Land one refreshed Telegram fix that skips only self-authored reply-chain media before `bot.api.getFile`, preserves user-authored reply media, updates the focused contract tests, and includes redacted after-fix Telegram proof.

## Reproduction Assessment

Yes, via source inspection. A Telegram text update whose reply-chain node contains a bot-authored photo can reach `bot.api.getFile(replyFileId)` in `resolveReplyMediaForChain` before any nested sender-id guard.

## Solution Assessment

Unclear until maintainers pin the reply-chain media policy. The narrow code fix is a numeric bot-id guard before the active fetch, but current coverage expects bot-authored reply targets to hydrate media, so tests and proof must be updated deliberately.
