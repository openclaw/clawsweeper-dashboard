## Summary

Keep open: current main still fetches Telegram reply-chain media without checking whether the replied-to media was authored by the bot, and the open closing PR at https://github.com/openclaw/openclaw/pull/57298 remains the implementation path.

## What This Changes

Fix Telegram reply-media handling so a user reply to the bot's own photo does not re-ingest that bot-sent photo as inbound user media.

## Best Possible Solution

Land or replace the linked PR with a numeric bot-id guard in resolveReplyMediaForChain before bot.api.getFile, preserving user and other-bot reply media and adding focused regression coverage plus redacted live Telegram proof.

## Reproduction Assessment

Yes, via source inspection. A Telegram text update with reply_to_message.photo from ctx.me can be normalized into a reply-chain node and reach bot.api.getFile in resolveReplyMediaForChain before any nested sender-id check.

## Solution Assessment

Yes. The narrow maintainable fix is to skip only reply-chain media whose sourceMessage.from.id matches the numeric current bot id before fetching, rather than disabling reply media more broadly.
