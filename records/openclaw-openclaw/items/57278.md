## Summary

Keep open: current main still fetches Telegram reply-chain media without checking whether the replied-to media was authored by the bot, and the open closing PR remains the right implementation path but is conflicting and proof-pending.

## What This Changes

The issue reports that Telegram reply handling treats a bot-sent photo in `reply_to_message` as a new inbound user attachment.

## Best Possible Solution

Refresh and land the linked PR with a numeric bot-id guard inside `resolveReplyMediaForChain` before `bot.api.getFile`, preserving user and other-bot reply media with focused regression coverage and redacted live Telegram proof.

## Reproduction Assessment

Yes, via source inspection. A Telegram text update whose `reply_to_message.photo` was authored by `ctx.me` can be normalized into a reply-chain node and reach `bot.api.getFile` in `resolveReplyMediaForChain` before any nested sender-id check.

## Solution Assessment

Yes. The narrow maintainable fix is to skip only reply-chain media whose source message sender id matches the numeric current bot id before fetching, rather than disabling reply media broadly.
