## Summary

Keep open for maintainer policy review: the reported Telegram reply-media behavior is source-reproducible, but current main and the latest shipped release also contain focused coverage that intentionally hydrates bot-authored reply-target media, so a blanket self-authored skip would change an asserted reply-chain contract rather than fix a clearly broken one.

## What This Changes

The issue requests that Telegram reply-media resolution skip bot-authored replied-to media so the agent does not receive its own prior photo as inbound context.

## Best Possible Solution

Keep the shipped reply-chain hydration behavior unless maintainers choose a narrower Telegram policy; any change should update the focused bot- and user-authored reply-media tests and include redacted live Telegram proof.

## Reproduction Assessment

Yes. Source inspection shows reply-chain media from a bot-authored Telegram message reaches `bot.api.getFile` and then the inbound media context, and current tests intentionally cover that path.

## Solution Assessment

Unclear. The proposed blanket guard is narrow, but it conflicts with the existing shipped contract that hydrates bot-authored reply targets for follow-up context; maintainers need to choose the intended policy first.
