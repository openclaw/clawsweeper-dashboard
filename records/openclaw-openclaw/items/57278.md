## Summary

Keep open for maintainer policy review: current main and the latest release intentionally hydrate bot-authored Telegram reply-chain media, so the requested self-authored skip would change shipped reply-context semantics rather than fix a clearly-defined bug.

## What This Changes

The issue asks Telegram to skip downloading and attaching media from bot-authored replied-to messages when building inbound reply-media context.

## Best Possible Solution

Choose and document the intended Telegram bot-authored reply-media policy; if maintainers want filtering, implement it narrowly in the active reply-chain resolver with regression coverage for both bot- and user-authored media plus redacted live Telegram proof.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: `resolveReplyMediaForChain` fetches media for any media-bearing reply-chain node, and the focused test constructs a bot-authored photo reply target that is fetched and attached. I did not run a live Telegram repro in this read-only pass.

## Solution Assessment

Unclear. The proposed blanket skip is mechanically small, but it conflicts with current shipped tests and changelog-described reply-chain semantics, so maintainers need to decide the intended policy first.
