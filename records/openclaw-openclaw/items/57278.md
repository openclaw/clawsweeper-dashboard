The issue asks Telegram reply-media resolution to skip media from bot-authored replied-to messages so generated media is not reattached as inbound context.

## Best Possible Solution

Maintainers should either keep the current bot-authored reply-chain hydration contract or explicitly change it in resolveReplyMediaForChain with focused tests for bot- and user-authored media plus real Telegram proof.

## Reproduction Assessment
