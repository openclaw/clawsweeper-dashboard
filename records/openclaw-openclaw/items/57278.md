## Summary

Keep this open for maintainer review: current main still has the Telegram reply-chain media hydration path, current tests intentionally hydrate bot-authored reply ancestors when visibility allows it, and an open closing PR still references this issue but is conflicting and targets older resolver names.

## What This Changes

The issue asks Telegram to avoid downloading and attaching media from bot-authored replied-to messages as inbound context when a user replies to a bot photo.

## Best Possible Solution

Make the Telegram reply-chain media contract explicit; if bot-authored media should be skipped, land a current-main guard with tests for bot- and user-authored reply nodes, otherwise close this as intended behavior with the current test contract cited.

## Reproduction Assessment

Yes, at source level. Current main can hydrate media from bot-authored reply-chain nodes when visibility allows it, and current tests explicitly exercise that bot-authored generated-photo path.

## Solution Assessment

Unclear. A self-authored guard is a plausible narrow code change, but the existing Telegram reply-chain tests expect the opposite behavior, so maintainers need to choose the contract before this is the best fix.
