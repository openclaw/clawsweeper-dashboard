## Summary

Keep this issue open. Current main still lacks a self-authored reply-target guard in Telegram reply media resolution, and open PR #57298 is already the explicit implementation candidate.

## What This Changes

Telegram should avoid treating media from a bot-authored replied-to message as inbound user media when a user replies with text.

## Best Possible Solution

Land a focused Telegram runtime fix that returns no reply media when the replied-to Telegram message was authored by ctx.me, with a regression test proving a user text reply to a bot-sent photo does not call getFile or attach that media.

## Reproduction Assessment

Yes. The source path is clear: a text message with reply_to_message.photo reaches resolveReplyMediaForMessage, passes hasInboundMedia, and calls bot.api.getFile without checking replyMessage.from.id against ctx.me.id.

## Solution Assessment

Yes. The narrow maintainable fix is to add the self-authored reply-target guard before file-id resolution/download, then keep the existing non-bot reply-media behavior covered by tests.
