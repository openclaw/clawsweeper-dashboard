## Summary

Keep open: current main still resolves Telegram reply-target media without checking whether the replied-to message was authored by the bot, and open PR #57298 remains the linked implementation candidate.

## What This Changes

Telegram should avoid treating bot-authored reply-target media as inbound user media when a user replies to a bot-sent photo.

## Best Possible Solution

Review and land or replace PR #57298 with a current-main guard in Telegram reply-media resolution plus regression coverage for user text replies to bot-sent media.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path: a user message with reply_to_message.photo from ctx.me reaches resolveReplyMediaForMessage, which resolves the reply file id and calls getFile before any reply-target author check.

## Solution Assessment

Yes. The narrow maintainable fix is to return [] for bot-authored reply targets before resolving the file id or calling getFile; PR #57298 is the existing implementation candidate but should be checked against the current handler split.
