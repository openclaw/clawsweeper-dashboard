## Summary

Keep this open for maintainer review: current main and the latest release still hydrate Telegram reply-chain media from bot-authored messages, but existing tests and a closed follow-up PR show that may be intentional context rather than a straightforward bug.

## What This Changes

The issue asks Telegram reply-media handling to skip media from bot-authored replied-to messages so the agent does not receive its own generated media as inbound user attachments.

## Best Possible Solution

Make the Telegram reply-chain media contract explicit, then either close this as intended behavior or land a narrow fix with regression coverage for both bot-authored and user-authored reply media.

## Reproduction Assessment

Yes from source inspection: current main can fetch media from bot-authored reply-chain nodes when they have inbound media and pass visibility checks. I did not run a live Telegram repro because this review is read-only.

## Solution Assessment

Unclear: adding a self-authored guard is a plausible narrow fix, but current Telegram tests preserve bot-authored reply-chain media hydration in an allowed context. Maintainers should choose the intended contract before implementation.
