## What This Changes

The issue reports a Web UI regression where image-heavy chat sessions can show chat/sidebar inline images at abnormally large sizes.

## Best Possible Solution

Land or adopt a narrow fix that makes sidebar markdown image caps match main chat image caps, with browser proof; only pursue virtual-scroll work if oversized images still reproduce afterward.

## Reproduction Assessment

No high-confidence live browser reproduction was established. Source inspection does show the likely path: markdown images render as `.markdown-inline-image`, main chat caps that class tightly, and sidebar markdown still allows a wider/taller image cap.

## Solution Assessment

Yes for the source-visible mismatch: matching the sidebar `.markdown-inline-image` constraints to the existing main-chat constraints is the narrowest maintainable fix. A broader virtual-scroll repair should wait until the problem remains reproducible after CSS parity lands.
