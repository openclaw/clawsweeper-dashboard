## Summary

Keep open. Current main still leaves a source-visible oversized-image path in Control UI sidebar markdown, and open unmerged PR #46947 already references this issue with closing syntax, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Control UI web chat images should remain consistently bounded during longer conversations instead of growing too large.

## Best Possible Solution

Review and land PR #46947 or an equivalent Control UI CSS change that aligns sidebar markdown image caps with main chat, then do a focused image-heavy visual check for main chat and sidebar markdown.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No for the full extended-session main-chat symptom because no live browser proof or screenshot is available. Source inspection does show a narrower current-main path where `.sidebar-markdown .markdown-inline-image` can render base64 markdown images at 100% sidebar width and 420px height.

## Solution Assessment

Is this the best way to solve the issue? Yes for the known sidebar path: matching the sidebar inline-image cap to the existing main chat cap is narrow and maintainable. It does not by itself prove any separate virtual-scroll growth path is fixed.
