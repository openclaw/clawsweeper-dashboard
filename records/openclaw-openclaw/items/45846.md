## What This Changes

The issue reports a Web UI regression where chat images become abnormally large during longer conversations with image messages.

## Best Possible Solution

Land, adopt, or replace the linked sidebar CSS parity fix so sidebar markdown images use the same reasonable bounds as main chat markdown images.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not fully: source inspection shows the likely sidebar markdown image sizing mismatch, but this read-only pass did not run a long Windows Web UI chat with repeated image messages.

## Solution Assessment

Is this the best way to solve the issue? Yes for the source-visible mismatch: matching the sidebar .markdown-inline-image cap to the existing main-chat cap is the narrowest maintainable fix, while virtual-scroll work should wait for proof that the issue remains after CSS parity lands.
