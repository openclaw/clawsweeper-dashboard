## Summary

Keep open: the change targets a real Anthropic thinking recovery gap, but the proposed stream shape can leak two `start` events into downstream consumers and leave an extra partial assistant turn in session state.

## What This Changes

The PR changes Anthropic thinking recovery so a control-plane `start` event no longer prevents retry before real content is emitted.

## Best Possible Solution

Retry after a control-plane `start` without exposing two `start` events to agent-loop consumers; buffer, suppress, or replace the first partial start before forwarding the recovered stream.

## Reproduction Assessment

Yes from source: the PR's own expected stream is `start`, retry `start`, `done`, and `agentLoop` treats each `start` as a new assistant message. I did not run tests because this is a read-only review.

## Solution Assessment

No. Allowing retry after `start` is the right direction, but the implementation should not forward two `start` events into agent-loop consumers.
