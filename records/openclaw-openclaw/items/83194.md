## Summary

Keep this PR open. The cleanup goal is useful, but the patch should not land as-is because Telegram live and Telegram desktop proof comments can share the same `github-actions[bot]` `eyes` reaction, so a per-workflow cleanup can remove the only in-progress marker while a sibling Mantis run is still active; the PR also lacks after-fix real workflow proof and carries the protected `maintainer` label.

## What This Changes

The branch adds end-of-workflow issue-comment cleanup jobs to four Mantis GitHub Actions workflows to remove the workflow bot's `eyes` reaction from trigger comments.

## Best Possible Solution

Use one owner for the Mantis command reaction lifecycle, or have cleanup verify no sibling Mantis workflow from the same trigger comment is still queued or running before deleting the shared bot reaction.

## Reproduction Assessment

Yes, by source inspection: current main has overlapping Telegram Mantis issue-comment triggers, and the PR deletes all `github-actions[bot]` `eyes` reactions for a comment from each workflow's cleanup job. I did not run a live Actions scenario in this read-only review.

## Solution Assessment

No, the per-workflow cleanup is too narrow for a shared comment reaction because sibling Mantis workflows can be active from the same Telegram command. A safer fix would coordinate cleanup across the workflows or disambiguate the triggers before deleting the reaction.
