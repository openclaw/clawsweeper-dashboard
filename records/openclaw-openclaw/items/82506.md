## Summary

Keep open: the latest head still has two blocking P2 issues. The approval-delivery fix is useful and has terminal proof, but the PR still regresses iOS no-route fallback and weakens unrelated cron persisted-payload validation.

## What This Changes

The PR changes gateway exec/plugin approval routing to distinguish delivered versus attempted approval dispatches and adds tests plus a few supporting test-helper changes.

## Best Possible Solution

Keep the richer delivery-result contract, make iOS delivery distinguish no eligible route from attempted failed delivery, and remove the unrelated cron validation change.

## Reproduction Assessment

Yes. The linked report has concrete approval-dispatch failure steps and the PR body supplies terminal proof for the all-target failure path; the remaining PR regressions are source-reproducible from the iOS false no-route contract and the new attempted gate.

## Solution Assessment

No. Surfacing explicit approval delivery failures is the right direction, but this implementation still conflates iOS no-route with failed attempted delivery and changes cron validation outside the gateway approval fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve iOS no-route fallback:** `src/gateway/server-methods/exec-approval.ts:401`
  - body: Mapping every iOS `handleRequested` result to `{ attempted: true }` treats the current `false` no-route result as an explicit failed delivery. Because `approval-shared` now skips turn-source fallback once `attempted` is true, approvals that could fall back to the originating chat route can now expire as `no-approval-route`; return richer iOS state or only set attempted after a real push attempt.
  - confidence: 0.91
- **[P2] Keep persisted cron payload validation strict:** `src/cron/persisted-shape.ts:67`
  - body: This drops the trim-empty check for persisted `agentTurn.message`, so whitespace-only messages are no longer rejected at load. That is unrelated production behavior for a gateway approval PR and can let malformed persisted jobs into the scheduler path; revert it or handle it in a targeted cron fix with coverage.
  - confidence: 0.84
