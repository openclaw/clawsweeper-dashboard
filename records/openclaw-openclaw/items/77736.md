## What This Changes

The PR makes explicit `tools.web.search.provider` values authoritative for agent `web_search` routing, ahead of stale runtime metadata, and reloads plugin web-search providers when the explicit provider is absent from the active runtime registry.

## Best Possible Solution

Keep the narrow routing fix direction, but require real behavior proof, add a focused regression test and a one-line Unreleased changelog entry, then land after the normal changed-file gates pass.

## Reproduction Assessment

Yes, from source inspection: current main lets runtime metadata drive provider-discovery mode before config and the agent wrapper does not pass the explicit configured provider as `providerId`. I did not run a live custom-plugin OpenClaw scenario in this read-only review.

## Solution Assessment

Mostly yes: making the configured provider id authoritative and reloading plugin providers only when an explicit provider is missing is the narrow maintainable fix. It still needs real behavior proof, regression coverage, and changelog before merge.
