## Summary

Keep open: current main and v2026.6.8 still lack the requested `first-progress` and `startup-failed` lifecycle contract, and the open candidate fix is still a maintainer/product-review path rather than a merged implementation.

Request to add subagent `first-progress` and typed `startup-failed` lifecycle signaling so startup aborts do not look like indefinitely running child sessions.
