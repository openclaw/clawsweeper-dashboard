## Label Justifications

- `P1`: The reported silent subagent startup failure affects a core agent workflow and can leave users unable to distinguish an active child run from a hung one.

## Summary

Keep open: current main still exposes embedded run lifecycle as start/end/error only, so the requested first-progress and startup-failed signals are not implemented; the related implementation is still an open, conflicting PR rather than merged behavior.

## What This Changes

The issue asks OpenClaw to add explicit embedded subagent startup progress and startup-failure lifecycle signals so aborted child runs do not look like silent hangs.

## Best Possible Solution

Refresh and review https://github.com/openclaw/openclaw/pull/77415, or replace it with an equivalent narrow PR that emits first-progress and terminal startup-failed signals and updates the gateway, SDK, docs, and regression tests together.

## Reproduction Assessment

Yes, at source level: current main only emits and projects lifecycle start/end/error, while the requested first-progress and startup-failed phases are absent from the emitter, session projection, SDK normalizer, and docs. I did not run a live stalled subagent, so the live corpus count remains reporter evidence rather than independently reproduced runtime proof.

## Solution Assessment

Yes, conditionally: an additive first-progress/startup-failed lifecycle path is the narrow owner-boundary fix because progress is observed in the embedded subscription handlers and terminal startup failure must be projected through gateway/session/SDK surfaces. The existing open PR is the right implementation candidate but needs conflict resolution and maintainer review before this issue can close.
