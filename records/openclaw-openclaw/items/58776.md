## Summary

Keep open: current main and the latest release still map aborted lifecycle ends to timeout-shaped state and do not emit the requested first-progress/startup-failed lifecycle phases. The open candidate fix is relevant, but it is still unmerged, conflicting, and labeled for product/merge-risk review, so this issue remains necessary.

## What This Changes

The issue requests explicit sub-agent startup lifecycle signals so aborted startup runs do not appear as silent long-running hangs.

## Best Possible Solution

Review, land, or replace the canonical lifecycle fix so embedded child runs emit first progress, report pre-progress aborts as startup failures, and keep agent.wait/session projections consistent with that terminal state.
