## Summary

Keep open: current main still only emits and projects lifecycle `start`, `end`, and `error`, so the requested `first-progress` and `startup-failed` signals are not implemented; the related implementation PR is still open, conflicting, and unmerged.

## What This Changes

The issue requests additive subagent lifecycle signals for first progress and typed startup failure so aborted startup runs do not appear hung.

## Best Possible Solution

Refresh and review https://github.com/openclaw/openclaw/pull/77415, or replace it with an equivalent narrow PR that updates the lifecycle emitter, registry/wait/session projections, SDK normalization, docs, and regression tests together.

## Solution Assessment
