## Summary

Keep open: current main and the latest release still lack the requested `first-progress` / `startup-failed` lifecycle phases, and aborted lifecycle ends still flow through timeout-shaped state. The open candidate fix remains unmerged and dirty, so this issue is still necessary.

## What This Changes

The issue requests additive sub-agent lifecycle signals so parents can distinguish first progress from startup failures instead of treating aborted startup as a silent hang.

## Best Possible Solution

Review, refresh, or replace the open lifecycle PR so embedded child runs emit first progress, report pre-progress aborts as startup failures, and keep subagent, agent.wait, session, and SDK projections consistent.
