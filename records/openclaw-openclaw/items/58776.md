## Summary

Keep open. Current main still has the reported lifecycle gap: the affected consumers only handle start/end/error and aborted lifecycle ends still collapse to timeout, while the successor implementation remains open and unmerged.

## What This Changes

The issue requests additive sub-agent lifecycle signals so parents can distinguish first progress from startup failures instead of treating silent aborted starts as hangs.

## Best Possible Solution

Land a consolidated lifecycle fix, likely through https://github.com/openclaw/openclaw/pull/77415 or a replacement, that emits and preserves first-progress/startup-failed through embedded runs, registry, gateway wait/session state, docs, and regression coverage.

## Reproduction Assessment

Yes, at source level. Current main handles only start/end/error in the affected lifecycle consumers and maps end+aborted to timeout, though I did not replay the reporter's live corpus fixture.

## Solution Assessment

Yes, the additive first-progress/startup-failed direction is the narrow maintainable fix. The issue should stay open until the open successor PR or a replacement implementation merges.
