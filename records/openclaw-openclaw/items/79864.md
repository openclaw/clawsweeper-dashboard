## Summary

This PR should remain open: the code direction looks sound after the latest security fix, but it has the protected `maintainer` label and still lacks OpenClaw after-fix delivery proof with an `imsg` build that actually advertises `send-rich --file`.

## What This Changes

The PR hydrates iMessage reply attachments through the outbound media resolver, feature-detects `imsg send-rich --file`, stages buffers for `sendRichMessage`, and adds focused handler and runner tests.

## Best Possible Solution

Land after maintainer review once the upstream imsg capability is available or otherwise verified, with redacted live OpenClaw proof of threaded attachment delivery through the iMessage channel.

I did not run a live macOS/iMessage reproduction in this read-only review.

## Solution Assessment
