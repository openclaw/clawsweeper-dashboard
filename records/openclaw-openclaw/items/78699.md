## What This Changes

Report a macOS LaunchAgent update/restart race where launchd respawns the Gateway before OpenClaw's own restart step, causing a transient port 18789 conflict and UI hang.

## Best Possible Solution

Make macOS update restarts idempotent and serialized: avoid a second restart when launchd has already brought up the expected Gateway, and otherwise wait for the old listener to clear before starting the replacement.
