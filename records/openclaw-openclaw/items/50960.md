## Summary

Keep open: this PR targets a real skill exec env regression, but it is not merge-ready because it broadens sensitive skill env admission into process-wide exec environments without selected-skill scoping or duplicate-key handling, and it still lacks real behavior proof.

## What This Changes

The PR allowlists sensitive-pattern keys from configured skill env blocks, moves active skill env tracking into shared state, adjusts skill eligibility checks, and adds focused unit tests.

## Best Possible Solution

Land or replace this with a scoped env boundary that lets intended skill-owned subprocesses receive explicit configured env values while generic exec strips unrelated active skill secrets and duplicate sensitive keys fail or are isolated.

## Reproduction Assessment

Yes, source-backed. Current main stages skills.entries.*.env keys but only allowlists metadata-declared sensitive keys, and host exec inherits process.env after that sanitizer path.

## Solution Assessment

No. Allowlisting explicit configured keys is directionally useful, but this branch admits secrets too broadly and leaves duplicate sensitive-key conflicts; scoped admission plus generic exec stripping is the safer solution.

Full review comments:

- **[P1] Gate sensitive env admission to selected skills:** `src/agents/skills/env-overrides.ts:163-165`
  - body: The added allowlist loop admits every staged `skills.entries.*.env` key before no-snapshot runs narrow the catalog to eligible or prompted skills. Disabled, bundled-blocked, or unprompted skills can therefore put `*_TOKEN` or `*_PASSWORD` values into `process.env`, and host exec children inherit that environment.
  - confidence: 0.86
- **[P2] Reject duplicate sensitive env conflicts:** `src/agents/skills/env-overrides.ts:163-165`
  - body: `acquireActiveSkillEnvKey()` keeps the first active value for a repeated key and only increments a count. After this PR admits arbitrary configured sensitive names, two skills with the same token/password env name can silently share the first skill's credential instead of failing or isolating the conflict.
  - confidence: 0.78
