Full review comments:

- **[P1] Restrict sensitive env admission to selected skills:** `src/agents/skills/env-overrides.ts:163`
  - body: `applySkillEnvOverrides` runs against the loaded skill catalog before prompt filtering, so allowlisting every configured pending override lets disabled, blocked, or otherwise unprompted skills inject `*_TOKEN`/`*_PASSWORD` values into `process.env`. Generic host `exec` then inherits from that process env, which broadens secret exposure compared with current main where these sensitive names were still blocked.
  - confidence: 0.88
- **[P2] Handle duplicate sensitive env keys deliberately:** `src/agents/skills/env-overrides.ts:163`
  - body: The active env registry is one value per key, so widening sensitive-key admission means two configured skills that use the same name such as `GITHUB_TOKEN` can silently share the first active credential for the whole run. The fix needs scoped or conflict-aware handling instead of admitting all configured sensitive keys into the process-wide map.
  - confidence: 0.8
