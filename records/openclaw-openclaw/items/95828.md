## Summary

Keep open: current main and v2026.6.10 still hardcode Twilio REST traffic to US1 with no `twilio.region` config, and the open linked PR is the active implementation candidate that should close this only after merge.

## What This Changes

Request support for a closed `twilio.region` voice-call configuration so Twilio Calls API operations can use regional REST hostnames for IE1/AU1 while preserving US1 by default.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/95832; keep this issue as the canonical product record until a Twilio region implementation reaches main.

## Solution Assessment

Yes, pending maintainer acceptance: the linked region-only mapping is the narrow maintainable shape because it avoids arbitrary base URLs and keeps the SSRF allowlist exact. This issue should remain open until that PR is reviewed and either merged or replaced.
