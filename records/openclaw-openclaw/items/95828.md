## What This Changes

The issue requests a voice-call Twilio configuration option that lets users select a Twilio processing Region instead of always using the default US1 REST API endpoint.

## Best Possible Solution

Review and land the linked region-only PR if maintainers accept the closed Twilio Region contract; otherwise keep this issue as the product-decision record for the voice-call plugin.

## Reproduction Assessment

Yes, at source level: current main hardcodes Twilio REST requests and SSRF allowlists to api.twilio.com, and the Twilio config schema has no region field. I did not run a credentialed current-main regional call failure in this read-only review.

## Solution Assessment

Yes as a proposed direction: a closed region-to-hostname mapping in the voice-call plugin is narrower than an arbitrary base URL or user-exposed edge/region pair. The open PR is the right place to prove and review that contract before closing the issue.
