## Summary

Keep open. Current main and v2026.5.19 still default Chrome MCP existing-session startup to `npx -y chrome-devtools-mcp@latest`, and cold-cache status/profile/tab paths can still create temporary Chrome MCP sessions; the merged timeout and `pageReady` work is useful mitigation but does not fully resolve the reported hang/orphan-process concern.

## What This Changes

The issue asks OpenClaw to fix Gateway browser-control hangs caused by spawning `chrome-devtools-mcp` through `npx` for Chrome MCP existing-session profiles.

## Best Possible Solution

Define and implement the browser plugin contract for passive Chrome MCP probes, subprocess cleanup, and the default launch command, with live macOS existing-session proof that passive status/tabs/profile listing do not spawn new `npx` children while explicit start still works.

## Reproduction Assessment

## Solution Assessment

No current merged fix fully solves the issue. The best path needs maintainer agreement on passive no-spawn behavior, process-tree cleanup, and whether the default `npx@latest` wrapper remains acceptable.
