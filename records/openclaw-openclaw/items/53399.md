## Summary

Keep open. Current main has useful timeout/readiness improvements, but cold-cache Chrome MCP status/profile/tab paths still spawn the default `npx chrome-devtools-mcp@latest` stdio transport, so the reported spawn/orphan failure is not fully implemented or superseded.

## What This Changes

Fix browser control server hangs and orphan process buildup when the Gateway spawns `npx chrome-devtools-mcp@latest` for existing-session browser control.

## Best Possible Solution

Scope a browser-plugin fix that keeps passive probes non-spawning on cold cache, bounds and cleans up failed stdio subprocess trees, and deliberately decides whether the default Chrome MCP launch should remain `npx@latest`, be pinned/resolved, or require explicit `mcpCommand` configuration.

## Reproduction Assessment

Partly: the current-main source path is clear for cold-cache status, profile listing, and tab listing to spawn `npx chrome-devtools-mcp@latest`. The exact macOS/Node 25 external hang was not live-reproduced in this read-only pass.

## Solution Assessment

No current patch is sufficient. The merged timeout/readiness fixes are useful mitigations, but they do not remove passive cold-cache spawning or prove orphan cleanup for the default `npx` wrapper path.
