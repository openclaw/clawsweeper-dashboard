## Summary

Keep open: current main and the latest release still have the scoped `sessions.describe` gap, and https://github.com/openclaw/openclaw/pull/92970 is an open linked closing PR rather than a merged fix.

## What This Changes

The issue reports that Gateway `sessions.describe` does not preserve a requested `agentId`, so a `global` session lookup can resolve against the default store agent instead of the caller-scoped agent.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/92970 with a narrow fix that adds optional `agentId` to `sessions.describe`, threads it through the current helper, and covers scoped plus omitted-agent behavior.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: the schema cannot accept `agentId`, and the handler does not pass scope into a helper that already supports scoped `global` resolution. I did not run a live multi-agent Gateway repro in this read-only review.

## Solution Assessment

Yes, the best fix is the narrow protocol-plus-handler threading path, not a new session lookup mechanism. The issue should stay open until the linked implementation merges or is replaced.
