## Summary

Keep open: current main and the latest release still do not let `sessions.describe` carry an explicit `agentId`, and the open linked fix PR has not merged yet.

## What This Changes

Request preserving an explicit `agentId` scope through Gateway `sessions.describe` so `global` session lookups resolve against the caller's agent instead of the default store agent.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/92964 with an optional `agentId` protocol field, handler threading into the existing scoped store-target helper, and regression tests for default-agent and non-default-agent `global` lookups.

## Reproduction Assessment

Yes from source inspection: current main's `sessions.describe` schema cannot accept `agentId`, while the handler calls the scoped-capable target helper without options. I did not run a live multi-agent Gateway repro because this review is read-only.

## Solution Assessment

Yes: optional protocol schema support plus handler threading into the existing scoped store-target helper is the narrowest maintainable fix. The linked PR should be reviewed for the missing schema/test pieces before it becomes the closing implementation.
