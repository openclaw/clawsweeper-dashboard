## Summary

Keep open: current main and the latest release still lack an agentId-aware sessions.describe contract, and https://github.com/openclaw/openclaw/pull/92970 is the open closing implementation path.

## What This Changes

The issue reports that Gateway sessions.describe drops the caller's agentId and resolves global session lookups through the default agent store.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/92970 with optional agentId schema support, handler threading into the existing scoped store-target path, and focused regression coverage for non-default global lookups.

## Reproduction Assessment

Yes. Current main source shows the describe schema rejects agentId and the handler loads entries without caller scope; I did not run a live multi-agent Gateway repro because this review is read-only.

## Solution Assessment

Yes. Optional protocol support plus forwarding into the existing scoped session target helper is the narrowest maintainable fix, and sibling session handlers already follow that pattern.
