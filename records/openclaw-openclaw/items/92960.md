## Summary

Keep open: current main and the latest release still reject or drop scoped `agentId` for `sessions.describe`, and the open closing PR has not merged yet.

## What This Changes

The issue requests that Gateway `sessions.describe` preserve an explicit caller `agentId` when resolving `global` session keys.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/92970 with optional `agentId` in `sessions.describe`, handler threading into scoped session target resolution, and focused regression proof.

## Reproduction Assessment

Yes. Source inspection shows current main rejects `agentId` on `sessions.describe` and then resolves the target without agent scope, while the underlying store target helper already supports scoped `global` lookup.

## Solution Assessment

Yes, with the open PR still needing review. The narrow maintainable fix is to add an optional protocol field and thread it into the existing scoped session target path rather than adding a new store or caller workaround.
