## Summary

Keep open: current main and v2026.6.6 still omit `agentId` from `sessions.describe` validation and handler resolution, and the linked closing PRs are still open. https://github.com/openclaw/openclaw/pull/93077 looks like the strongest fix path, but it has maintainer-requested compatibility follow-up before this can be closed.

## What This Changes

The issue asks Gateway `sessions.describe` to preserve the caller's `agentId` so `global` session lookup resolves against the requested agent instead of the default store agent.

## Best Possible Solution

Land one current-main fix that adds optional `agentId` to the describe schema, threads it through the current handler without breaking deleted-main alias compatibility, and closes this after merge.

## Reproduction Assessment

Yes. Source inspection shows current main rejects `agentId` on `sessions.describe` and then resolves `global` without passing agent scope, while the store helper already supports scoped global lookup.

## Solution Assessment

Yes. Optional protocol schema support plus handler threading into the existing scoped store helper is the narrow maintainable fix; the linked implementation still needs the maintainer-requested compatibility repair before landing.
