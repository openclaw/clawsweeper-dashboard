The issue asks for Gateway `sessions.describe` and higher-level describe calls to preserve a requested `agentId` when resolving global sessions.

## Reproduction Assessment

## Solution Assessment

Yes for the direction, but not as an issue close yet. Adding an optional protocol field and threading resolved agent scope into the existing store-target helper is the narrow fix; the open PR should preserve deleted-main alias compatibility before landing.
