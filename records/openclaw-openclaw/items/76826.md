## Summary

Keep open: current main renders aggregate cleanup counts and a per-session dry-run action table, but it still does not provide the requested per-label kept/pruned breakdown. The existing ClawSweeper comment reached the same conclusion, and I found no linked merged PR or duplicate that supersedes this request.

## What This Changes

Add a per-label kept/pruned breakdown to the text output for `openclaw sessions cleanup --dry-run`.

## Best Possible Solution

Add a deterministic text-mode label summary after the existing action table, using `entry.label` first and a stable fallback for unlabeled sessions, while leaving cleanup policy and JSON output semantics unchanged unless maintainers explicitly choose to extend JSON too.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken documented behavior. Source, docs, and tests confirm current main has aggregate counts and an action table but no per-label summary.

## Solution Assessment

Yes; the narrowest maintainable solution is to extend the CLI dry-run renderer and tests using the existing preview result and `SessionEntry.label`, without changing retention decisions or adding configuration.
