## Repair Work Prompt

Implement the requested `openclaw sessions cleanup --dry-run` text output enhancement. Add a deterministic summary after the existing per-session action table that groups sessions by `SessionEntry.label` when present and by a stable unlabeled/session-key fallback otherwise, showing kept and pruned counts per group plus totals. Treat `prune-missing`, `prune-stale`, `cap-overflow`, and `evict-budget` as pruned; `keep` remains kept. Do not change cleanup decisions, retention settings, Gateway behavior, or JSON output unless tests/docs reveal an existing contract requiring it. Add focused coverage in `src/commands/sessions-cleanup.test.ts`, update `docs/cli/sessions.md`, and add a user-facing `CHANGELOG.md` entry without guessing contributor attribution.
