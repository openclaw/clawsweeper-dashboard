## Summary

Keep open: current main renders aggregate cleanup counts and a per-session dry-run action table, but no per-label kept/pruned breakdown. The request is a narrow CLI UX enhancement, and the existing `SessionEntry.label` field makes the implementation boundary clear.

## What This Changes

Add a `openclaw sessions cleanup --dry-run` summary grouped by session label with kept and pruned counts.

## Best Possible Solution

Add a deterministic text-mode dry-run label summary using `entry.label` first and a stable key-derived fallback, with tests, docs, and a changelog entry, without changing retention decisions or write behavior.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken documented behavior. Source and docs confirm current main has aggregate dry-run counts and the action table, but not the requested label breakdown.

## Solution Assessment

Yes; the narrowest maintainable path is to extend the CLI dry-run renderer and tests, deriving counts from the existing preview result instead of changing cleanup policy or adding configuration.
