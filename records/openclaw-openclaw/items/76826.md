## Summary

Keep open: current main still prints aggregate cleanup counts and a per-session dry-run action table, but it does not print the requested per-label kept/pruned summary. The issue is also paired with open closing PR #77021, so it should remain open until that PR or an equivalent change merges.

## What This Changes

Add a per-label kept/pruned summary to the text output of `openclaw sessions cleanup --dry-run`.

## Best Possible Solution

Land #77021 or an equivalent narrow CLI change that adds a deterministic per-label text summary from preview data while leaving retention decisions and JSON semantics unchanged unless maintainers explicitly choose to extend them.

## Reproduction Assessment

Not applicable; this is a feature request, not a broken documented behavior. Source, tests, and docs confirm current main has aggregate counts and an action table but no per-label summary.

## Solution Assessment

Yes; the narrowest maintainable approach is a renderer-level summary derived from `previewResults.beforeStore` and cleanup action sets. The linked PR should be reviewed for deterministic output, docs, changelog, and focused tests before landing.
