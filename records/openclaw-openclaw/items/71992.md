## Summary

Keep open. Current main still has the Control UI terminal-event ordering that can expose both committed history and live stream content, and open PR #72004 already targets this issue but has not merged.

## What This Changes

The report asks for Control UI WebChat to render each assistant turn once by clearing active stream state before appending terminal assistant messages.

## Best Possible Solution

Land #72004 or an equivalent narrow Control UI patch that snapshots and clears active stream/run state before mutating chatMessages, with regression coverage for the intermediate ordering.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an active Control UI chat run with non-null chatStream receiving a final or aborted event appends to chatMessages before clearing chatStream, while buildChatItems can render both history and the live stream.

## Solution Assessment

Yes for the narrow client-render race. Clearing or snapshotting stream state before appending terminal messages is the smallest maintainable fix; broader transcript dedupe should stay in a separate path.
