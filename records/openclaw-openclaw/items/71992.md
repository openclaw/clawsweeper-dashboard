## What This Changes

The issue asks for Control UI terminal chat events to clear live stream/run state before committing final or aborted assistant messages so assistant replies render once.

## Best Possible Solution

Land a focused WebChat controller/test fix that snapshots fallback stream text, clears active stream/run state before terminal message commits, and leaves broader persisted transcript/idempotency cleanup to https://github.com/openclaw/openclaw/issues/69208.

## Reproduction Assessment

Yes, source-level. Dispatch an active-run final or aborted event while chatStream is non-null; current main writes chatMessages before clearing the stream, and the render path can receive both values.

## Solution Assessment

Yes. Reordering the existing terminal-state mutations after snapshotting fallback stream text is the narrowest maintainable fix; backend transcript dedupe should stay on the broader tracker.
