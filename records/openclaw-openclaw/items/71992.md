## Summary

Keep open. Current main still has the Control UI terminal-event ordering that this report identifies, the focused fix PR was closed unmerged, and the broader transcript-duplication umbrella should remain separate from this narrow WebChat render race.

## What This Changes

The issue requests fixing Control UI WebChat duplicate assistant replies by clearing live stream/run state before appending final or aborted assistant messages.

## Best Possible Solution

Land a narrow current-main replacement for the closed focused PR, preserving fallback semantics while clearing live stream/run state before terminal assistant-message commits; keep persisted transcript dedupe under the umbrella tracker.

## Reproduction Assessment

Yes, source-level: dispatch an owned `final` or `aborted` chat event while `chatStream` is non-null. Current main appends to `chatMessages` before clearing the stream, and the renderer can render committed messages plus the live stream item.

## Solution Assessment

Yes. Snapshotting any fallback stream text, clearing `chatStream`/`chatRunId`/`chatStreamStartedAt`, then appending the terminal assistant message is the narrowest maintainable fix for this client render race.
