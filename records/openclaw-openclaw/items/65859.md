## Summary

Keep open. Current main and the latest release already cover the broad WebChat audio picker/playback path, but the requested audio upload support still has a narrow classifier gap for MIME-positive audio files whose filenames use video-container extensions such as .webm or .mp4.

## What This Changes

The issue requests audio MIME support in the WebChat upload picker and inline playback for received audio attachments.

## Best Possible Solution

Keep the shipped audio picker and inline player, then fix the WebChat attachment classifier to accept explicit audio/* MIME files before extension-based video rejection while preserving video/* rejection.

## Reproduction Assessment

Yes. A source-level reproduction is File { name: "clip.webm", type: "audio/webm" } through isSupportedChatAttachmentFile, which currently returns false because the .webm extension is rejected after the video/* MIME check.

## Solution Assessment

Yes for the remaining gap. The narrow maintainable fix belongs in the WebChat attachment support helper plus focused picker/drop tests, without changing gateway media policy or enabling video uploads.
