## Summary

Keep open. Current main and the latest release cover the broad WebChat audio picker/playback path, but the requested `audio/webm` upload case still fails because the WebChat attachment helper rejects `.webm` filenames after only checking for `video/*` MIME types.

## What This Changes

Add WebChat audio upload filtering and inline audio playback for received audio attachments, including common formats such as MP3, M4A, Ogg, WAV, and WebM.

## Best Possible Solution

Keep the shipped `audio/*` picker and inline player, then fix the WebChat attachment helper so `audio/*` MIME files are allowed before extension-based video rejection, with tests proving `audio/webm` uploads are accepted while `video/webm` remains rejected.

## Reproduction Assessment

Yes. A source-level reproduction is direct: `File { name: "clip.webm", type: "audio/webm" }` reaches `isSupportedChatAttachmentFile`, passes the `video/*` guard, then fails the `.webm` filename regex used by both picker and drop handling.

## Solution Assessment

Yes for the remaining gap. The narrow maintainable fix is in `ui/src/ui/chat/attachment-support.ts`, plus focused WebChat picker tests; this avoids changing gateway media policy or enabling video uploads.
