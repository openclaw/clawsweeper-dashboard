## Summary

Keep open. Current main already has broad WebChat audio picker support and inline audio playback, but the upload classifier still rejects explicit audio files when their filename uses video-container extensions such as .webm or .mp4.

## What This Changes

Request adds WebChat audio MIME upload support and inline audio controls for received audio attachments.

## Best Possible Solution

Keep the shipped picker/player behavior and update the WebChat attachment classifier to honor explicit `audio/*` MIME before extension-based video-container rejection.

## Reproduction Assessment

Yes at source level. `File { name: "clip.webm", type: "audio/webm" }` follows the current `isSupportedChatAttachmentFile` predicate and returns false because the extension blacklist runs after only a `video/*` MIME check.

## Solution Assessment

Yes. The narrow maintainable fix belongs in `ui/src/ui/chat/attachment-support.ts` with focused picker/drop coverage, without changing gateway media policy or enabling video uploads.
