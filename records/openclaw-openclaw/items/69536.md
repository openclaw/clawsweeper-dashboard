## Summary

Keep open. Current main still has the reported mismatch: the session-memory hook writes `YYYY-MM-DD-slug.md`, while the memory-core CLI, live dreaming ingestion, grounded seeding, rem-harness scan, and Gateway Dream Diary backfill still accept only exact `YYYY-MM-DD.md` files; the startup-context fix only covers first-turn preloading.

## What This Changes

The issue asks OpenClaw to align session-memory slugged files with memory-core dreaming/backfill discovery so session artifacts can participate in promotion and grounded replay.

## Best Possible Solution

Add a shared dated-memory filename parser that accepts both canonical `YYYY-MM-DD.md` and slugged `YYYY-MM-DD-*.md` session-memory artifacts, then reuse it across memory-core discovery, live ingestion, historical seeding, rem-harness, and Gateway Dream Diary backfill.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: a file like `memory/2026-04-21-ai-assistant-config.md` is still produced by the hook writer but fails every exact date-only matcher used by the promotion/backfill readers inspected above.

## Solution Assessment

Yes, a reader-side discovery fix is the best narrow solution. It is backward-compatible for existing slugged files, while changing only the hook writer would leave already-created session artifacts invisible.
