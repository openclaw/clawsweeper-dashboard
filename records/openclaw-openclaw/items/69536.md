## Summary

Keep open. Current main still has the same cross-system mismatch: the session-memory hook writes date-plus-slug files, while memory-core dreaming, rem-backfill, REM harness, and Gateway Dream Diary backfill continue to use exact date-only matchers.

## What This Changes

The issue reports that session-memory creates `memory/YYYY-MM-DD-slug.md` files that are not ingested by dreaming and backfill readers expecting `memory/YYYY-MM-DD.md`.

## Best Possible Solution

Add a shared dated-memory filename parser that accepts canonical `YYYY-MM-DD.md` plus bounded `YYYY-MM-DD-*.md` session-memory artifacts, then reuse it across memory-core discovery, live ingestion, historical seeding, REM harness, and Gateway backfill while preserving plugin/core boundaries.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: create `memory/2026-04-21-ai-assistant-config.md` or any hook-style `YYYY-MM-DD-slug.md`, then `openclaw memory rem-backfill --path ./memory` and the exact date-only matchers skip it or report no `YYYY-MM-DD.md` files.

## Solution Assessment

Yes. A reader-side parser fix is the narrowest maintainable solution because it preserves existing hook output and already-created artifacts; changing only the writer or adding a config knob would leave historical slugged files invisible.
