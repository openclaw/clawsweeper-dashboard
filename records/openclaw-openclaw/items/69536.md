## Summary

Keep open. Current main still has the core mismatch: the bundled session-memory hook writes date-plus-slug files, while memory-core daily ingestion, rem-backfill, REM harness, and Gateway Dream Diary backfill still use exact `YYYY-MM-DD.md` matchers.

## What This Changes

The issue requests aligning session-memory hook artifacts with memory-core/dreaming daily-note ingestion so slugged memory files are not ignored.

## Best Possible Solution

Add a shared dated-memory filename parser that accepts canonical daily files plus bounded date-prefixed session-memory artifacts, then reuse it across memory-core discovery, live ingestion, historical seeding, REM harness, and Gateway backfill while preserving current hook output.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: create `memory/2026-04-21-ai-assistant-config.md` or another `YYYY-MM-DD-slug.md` file, then `openclaw memory rem-backfill --path ./memory`; the exact date-only matchers skip it.

## Solution Assessment

Yes. A reader-side parser fix is the narrowest maintainable solution because it preserves existing hook output and historical artifacts; changing only the writer or adding a config knob would leave already-created slugged files invisible.
