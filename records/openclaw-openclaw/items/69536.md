## Summary

Keep open. Current main still has the reported mismatch: the session-memory hook writes dated files with a slug, while memory-core historical discovery, live daily ingestion, grounded seeding, REM harness scanning, and Gateway Dream Diary backfill still accept only exact `YYYY-MM-DD.md` daily files. The related open memory issue is about configurable daily-note locations, not this slugged-file ingestion bug.

## What This Changes

The issue asks OpenClaw to align session-memory hook filenames with dreaming/backfill discovery so hook-created memory files can be ingested and promoted.

## Best Possible Solution

Add a shared dated-memory filename parser that accepts canonical `YYYY-MM-DD.md` and bounded `YYYY-MM-DD-*.md` session-memory artifacts, then reuse it across memory-core discovery, live ingestion, historical seeding, REM harness, and Gateway Dream Diary backfill while keeping non-daily Markdown excluded.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a hook-created file like `memory/2026-04-21-ai-assistant-config.md` matches the writer’s date-plus-slug format but fails every exact `YYYY-MM-DD.md` matcher used by rem-backfill, live daily ingestion, grounded seeding, REM harness, and Gateway backfill.

## Solution Assessment

Yes. A reader-side parser fix is the narrowest maintainable solution because it preserves existing hook behavior and makes already-created slugged artifacts visible; changing only the hook writer would leave historical files invisible, and a new config option is not needed for this bug.
