## What This Changes

The issue reports that the session-memory hook writes `memory/YYYY-MM-DD-<slug>.md` while memory-core/dreaming consumers only discover `memory/YYYY-MM-DD.md` daily files.

## Best Possible Solution

Land or refresh the focused linked PR so memory readers accept bounded `YYYY-MM-DD-<slug>.md` session-memory artifacts while keeping day extraction deterministic and unrelated markdown excluded.

## Reproduction Assessment

Yes. Source inspection on current main shows a concrete path: create a hook-style `memory/2026-04-21-ai-assistant-config.md`, run `openclaw memory rem-backfill --path ./memory`, and the exact date-only matchers skip it before ingestion.

## Solution Assessment

Yes. A reader-side matcher fix is the narrowest maintainable direction because it preserves existing hook output and historical slugged artifacts; the issue should remain open while the linked PR is reviewed or replaced.
