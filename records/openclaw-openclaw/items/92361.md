## Summary

Keep open, but treat this as an unreleased developer-diagnostics design gap rather than a shipped regression: current main deliberately returns an `unsupported-signal` diagnostic for empty groups, while no production caller currently consumes or logs the hidden-plan diagnostics, and the contract does not yet specify warning versus rejection behavior.

## What This Changes

Surface malformed empty `allOf` and `anyOf` availability groups to tool authors during descriptor registration instead of leaving the diagnostic only in the hidden tool plan.

## Best Possible Solution

Define malformed availability expressions as a structural descriptor-validation result, surface them once from the eventual descriptor registration or gateway startup owner, and retain the existing hidden-plan diagnostic as defensive runtime output.

## Reproduction Assessment

Yes at source level. Empty groups deterministically become `unsupported-signal` hidden-plan diagnostics, and repository-wide consumer searches show that no current production path reports those diagnostics.

## Solution Assessment

Unclear. Registration-time reporting is preferable to evaluator logging, but maintainers must first choose whether malformed descriptors should warn, fail registration, or remain diagnostics consumed by the future planner owner.
