## Label Justifications

- `impact:other`: Malformed tool descriptors can silently disappear from plans, causing meaningful debugging impact for plugin and core tool authors.

## Summary

Current main still computes malformed empty-group diagnostics but only retains them in the hidden plan, and the two unmerged closing PRs propose materially different contracts. Keep the issue open while maintainers select and land the narrow owner-boundary fix; the discussion correctly discourages a third competing implementation.

## What This Changes

Surface empty `allOf`/`anyOf` availability groups as actionable descriptor-authoring diagnostics instead of silently hiding the affected tool.

## Best Possible Solution

Integrate the planner at the actual descriptor-registration owner and emit only malformed-descriptor diagnostics once through that owner’s logger or diagnostic sink, while keeping availability evaluation pure and preserving normal hidden-tool semantics.

## Reproduction Assessment

Yes. Current main deterministically returns `unsupported-signal` for an empty group, and `buildToolPlan` retains that diagnostic only in `ToolPlan.hidden`; source inspection provides a high-confidence reproduction even though the planner has no production caller yet.

## Solution Assessment

No, neither proposed local mechanism is yet proven as the best complete fix. The maintainable solution is to preserve the pure evaluator and surface only authoring errors once at a real descriptor-registration or planner-integration owner, with startup-level proof.
