## Summary

## What This Changes

Empty allOf/anyOf tool availability groups should emit an actionable warning-level authoring diagnostic at descriptor registration instead of only hiding the tool in the planner.

## Best Possible Solution

Land a callback or registration-boundary diagnostic path that keeps the evaluator and planner pure, warns once during plugin/tool descriptor registration, and includes focused planner and registration coverage plus SDK docs if availability remains public.

## Reproduction Assessment

Yes, from source inspection. Current main returns unsupported-signal for empty allOf/anyOf groups and buildToolPlan only stores those diagnostics in hidden without logging or callback surfacing.

## Solution Assessment

Yes, the best current path is the active registration-boundary callback/audit PR rather than adding console.warn side effects inside the pure evaluator or planner.
