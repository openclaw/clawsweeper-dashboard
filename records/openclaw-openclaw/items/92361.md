## Summary

Current main still leaves malformed empty availability-group diagnostics observable only through `ToolPlan.hidden`. The issue should remain open while maintainers choose between the two linked PRs and define the lifecycle-owned, one-time diagnostic boundary.

## What This Changes

Add an observable, one-time diagnostic for malformed empty tool-availability groups instead of leaving the authoring error only in the hidden tool plan.

## Best Possible Solution

Keep evaluation and planning pure, distinguish malformed descriptor diagnostics from normal runtime unavailability, and emit malformed diagnostics once from the actual descriptor-registration or activation owner with focused integration coverage.

## Reproduction Assessment

Yes. Source inspection provides a deterministic path: an empty group returns `unsupported-signal`, `buildToolPlan` puts the descriptor in `hidden`, and no production caller consumes that result.

## Solution Assessment

No linked implementation is yet proven to be the best complete fix: evaluator logging violates the pure boundary, while an unwired planner callback does not create startup visibility. The actual descriptor lifecycle owner should surface only malformed-authoring diagnostics once.
