## Summary

The report identifies a real current-main diagnostic gap, and the issue should remain open because a clean open PR explicitly owns the fix. That PR still needs review of whether warnings should occur in the evaluator or once at descriptor registration/startup.

## What This Changes

Surface an actionable authoring warning when empty `allOf` or `anyOf` tool-availability groups currently cause a descriptor to be hidden silently.

## Best Possible Solution

Review and land the linked PR after placing malformed-descriptor reporting at the narrowest one-time registration or startup boundary, while preserving structured hidden-plan diagnostics and covering both empty group forms.

## Reproduction Assessment

Yes. Constructing a descriptor with `availability: { anyOf: [] }` or `{ allOf: [] }` and passing it to `buildToolPlan` follows a direct current-main path into `hidden` with no logging; source and existing tests establish this with high confidence.

## Solution Assessment

The requested warning is the right outcome, but the best implementation should report the authoring error once through the canonical registration/startup logger rather than potentially adding repeated side effects to the pure evaluator.
