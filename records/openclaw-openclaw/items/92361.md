Keep open: current main and the latest release still preserve empty `allOf`/`anyOf` diagnostics only inside the hidden tool plan, and an open linked PR is already the closing fix candidate.

## What This Changes

The issue asks OpenClaw to surface malformed empty `allOf`/`anyOf` tool availability diagnostics instead of silently hiding the descriptor.

## Best Possible Solution

Land one owner-boundary fix that surfaces `unsupported-signal` authoring diagnostics once at descriptor registration or planner integration while keeping `evaluateExpression` and `evaluateToolAvailability` pure, then close this issue.

## Reproduction Assessment

Yes at source level: a descriptor with `availability: { anyOf: [] }` or `{ allOf: [] }` reaches `unsupported-signal`, and `buildToolPlan` only stores that diagnostic in `hidden`. I did not run tests because this review is read-only.

## Solution Assessment

The issue's direction is sound, but the exact fix still needs owner-boundary judgment: emitting from the pure evaluator or adding an unused observer is weaker than reporting once from descriptor registration or planner integration.
