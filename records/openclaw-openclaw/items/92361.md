## What This Changes

Add warning-level reporting for malformed empty tool-availability groups beyond the existing structured hidden-plan diagnostic.

## Best Possible Solution

Define one lifecycle owner for malformed descriptor validation, emit a deduplicated author warning there when production registration is wired, and retain structured hidden-plan diagnostics without repeated evaluator side effects.

## Reproduction Assessment

Yes, current source deterministically reproduces an empty-group descriptor becoming hidden with an `unsupported-signal` diagnostic. The stronger claim that the diagnostic is lost is not reproduced because it remains available in `ToolPlan.hidden`.

## Solution Assessment

Unclear: additional author-facing reporting may help, but evaluator-level `console.warn` is not proven to be the best boundary; a one-time registration or startup validator is safer if the planner gains a production lifecycle owner.
