## Summary

Keep open: current main still lacks configured-model-derived non-live discovery scoping, but this PR is not merge-ready because it can under-scope unresolved or omitted configured model providers and still lacks after-fix real behavior proof.

## What This Changes

The PR scopes non-live implicit provider discovery from configured agent model/provider hints and adds focused provider-discovery filter tests.

## Best Possible Solution

Land one reconciled discovery-scope implementation that uses the full configured-model ref set plus explicit providers, preserves safe fallback for unresolved hints, keeps current live/providerDiscoveryProviderIds behavior, and includes focused regression coverage with real startup proof.

## Reproduction Assessment

Yes. Source inspection on current main shows a non-live config with configured model refs but no providerDiscoveryProviderIds still reaches onlyPluginIds: undefined, so the broad discovery path remains reproducible from code without running a startup benchmark.

## Solution Assessment

No, not as-is. The owner-resolution direction is right, but the mergeable fix should reuse the central configured-model-ref collector or cover the same paths, preserve unresolved provider hints or broad fallback, fix the fallback test, and include real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Preserve unresolved provider hints in the discovery scope:** `src/agents/models-config.providers.implicit.ts:201-214`
  - body: This drops any configured provider/model provider whose owner lookup returns no plugin ids. With a mixed config, such as one resolvable model plus one custom-provider model, the result is a non-empty onlyPluginIds list that excludes the custom provider; the existing provider-id path keeps unknown ids as plugin-id fallbacks. Add the unresolved provider id or fall back to broad discovery when any configured hint cannot be resolved.
  - confidence: 0.86
- **[P2] Collect every configured model ref before narrowing discovery:** `src/agents/models-config.providers.implicit.ts:156-177`
  - body: The local collector covers only a subset of configured model slots and misses paths current main already treats as model refs, including heartbeat, compaction, channel overrides, hooks, tools.subagents, message TTS, and Discord voice. If another hint resolves, discovery becomes scoped and those omitted providers will not be discovered; reuse the central model-ref collector or cover the same paths.
  - confidence: 0.82
- **[P3] Exercise the unresolved model-owner fallback:** `src/agents/models-config.providers.live-filter.test.ts:38-47`
  - body: The broad-fallback test passes config: {}, so there are no model refs and resolveModelOwners is never called. Add an unresolved configured model ref so the test covers the intended model-owners-return-empty fallback branch.
  - confidence: 0.92
