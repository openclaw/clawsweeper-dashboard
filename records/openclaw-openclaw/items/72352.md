## Summary

Keep open. The optimization target is still source-reproducible on current main, but this PR is not safe as submitted because it can under-scope mixed provider configs, duplicates an incomplete model-ref collector, has a weak fallback test, and lacks after-fix real behavior proof.

## What This Changes

The PR narrows non-live implicit provider discovery by deriving plugin scopes from configured agent model/provider references and adds focused discovery-filter tests.

## Best Possible Solution

Land one reconciled provider-discovery scoping change that reuses the central model-ref collector, preserves unresolved provider/plugin fallback semantics and current discovery-scope seams, adds focused tests, and includes redacted real startup proof.

## Reproduction Assessment

Yes, at source level. Current main only passes explicit providerDiscoveryProviderIds into the discovery filter and otherwise returns undefined in non-live mode, so configured agent model refs alone do not scope discovery.

## Solution Assessment

No, not as submitted. The direction is maintainable, but the branch needs to preserve unresolved provider fallbacks, reuse current main's central model-ref collector, repair the fallback test, reconcile with current discovery seams, and add real behavior proof.
