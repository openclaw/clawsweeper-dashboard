Full review comments:

- **[P2] Preserve unresolved provider hints in the discovery scope:** `src/agents/models-config.providers.implicit.ts:201-214`
  - body: This drops any configured provider/model provider whose owner lookup returns no plugin ids. With a mixed config, such as one resolvable model plus one custom-provider model, the result is a non-empty onlyPluginIds list that excludes the custom provider; the existing provider-id path keeps unknown ids as plugin-id fallbacks. Add the unresolved provider id or fall back to broad discovery when any configured hint cannot be resolved.
  - confidence: 0.86
- **[P2] Collect every configured model ref before narrowing discovery:** `src/agents/models-config.providers.implicit.ts:156-177`
  - body: The local collector covers only a subset of configured model slots and misses paths current main already treats as model refs, including heartbeat, compaction, channel overrides, hooks, tools.subagents, message TTS, and Discord voice. If another hint resolves, discovery becomes scoped and those omitted providers will not be discovered; reuse the central model-ref collector or cover the same paths.
  - confidence: 0.82
- **[P3] Exercise the unresolved model-owner fallback:** `src/agents/models-config.providers.live-filter.test.ts:38-47`
  - body: The broad-fallback test passes an empty config, so there are no model refs and resolveModelOwners is never called. Add an unresolved configured model ref so this covers the intended model-owners-return-empty fallback branch.
  - confidence: 0.92
