## Summary

Keep open. Current main still does not scope non-live provider discovery from configured agent model refs, but this PR is not merge-ready because it can exclude unresolved configured providers, duplicates an incomplete model-ref collector, has a weak fallback test, conflicts with newer discovery-scope work, and lacks required real behavior proof.

## What This Changes

The branch narrows non-live implicit provider discovery by collecting configured agent model/provider refs, resolving owning plugin ids, and adding focused discovery-filter tests.

## Best Possible Solution

Land one reconciled provider-discovery scoping change that uses current main's central model-ref collector and metadata owner seams, keeps unresolved hints safe, preserves live and startup provider filters, and includes focused tests plus redacted real startup proof.

## Reproduction Assessment

Yes, at source level. Current main only passes providerDiscoveryProviderIds into resolveProviderDiscoveryFilter and otherwise returns undefined in non-live mode, so configured agent model refs alone do not scope provider discovery.

## Solution Assessment

No, not as-is. The direction is maintainable, but the mergeable solution needs to preserve unresolved provider fallbacks, reuse the central model-ref collector, fix the fallback test, reconcile with current main, and include real behavior proof.

If any collected hint resolves, discovery becomes scoped and providers referenced only by those omitted paths are not discovered; reuse src/config/model-refs.ts instead of maintaining this partial copy.
