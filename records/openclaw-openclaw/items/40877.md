## Summary

This PR is still useful but not ready to close or merge: current main still has the reported iOS permission-snapshot warning path, and the PR improves the Speech side, but the PR head still leaves a CoreLocation main-thread probe and a scheduler-dependent test helper. The PR also needs contributor-provided real behavior proof before merge.

## What This Changes

The PR makes iOS gateway permission snapshots async for Speech authorization, reuses a stored CLLocationManager, adds gateway reconnect/refresh race guards, and expands GatewayConnectionController tests.

## Best Possible Solution

Land a tightened version that routes location permission through delegate-managed app state without main-thread CoreLocation probes, keeps the intended Location Services semantics, uses deterministic async test waits, and includes redacted real runtime proof.

Yes for the source-level path: current main routes gateway option creation through currentPermissions(), which directly probes Speech and CoreLocation authorization APIs from the iOS gateway controller path. I did not reproduce the live Xcode/CoreLocation warning in this read-only review.

## Solution Assessment

No, not as currently proposed. Moving Speech authorization off the main actor and guarding reconnect races are useful, but the PR should finish the CoreLocation warning fix and remove scheduler-dependent test waits before merge.
