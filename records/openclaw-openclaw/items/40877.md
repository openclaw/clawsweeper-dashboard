## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Remove the remaining CoreLocation main-thread probe:** `apps/ios/Sources/Gateway/GatewayConnectionController.swift:985`
  - body: `currentPermissions()` still calls `CLLocationManager.locationServicesEnabled()` in the permission snapshot. Apple documents globally disabled Location Services as `.denied` and says this call is not required, so this leaves a warning-prone call in the exact path the PR is meant to quiet. ([developer.apple.com](https://developer.apple.com/documentation/corelocation/cllocationmanager/1423648-locationservicesenabled?changes=_2.&utm_source=openai))
  - confidence: 0.86
- **[P2] Wait for startAutoConnect completion deterministically:** `apps/ios/Sources/Gateway/GatewayConnectionController.swift:1115-1123`
  - body: `_test_startAutoConnect()` still stops after 100 `Task.yield()` iterations while `startAutoConnect` awaits async permission collection. On slower schedulers it can return before `activeGatewayConnectConfig` is applied, so the new assertions can fail nondeterministically; use a deadline or explicit condition helper.
  - confidence: 0.82
- **[P3] Replace the refresh test's fixed-yield drain:** `apps/ios/Tests/GatewayConnectionControllerTests.swift:289-292`
  - body: The refresh race test gives the unstructured refresh task only 200 yields before asserting. Slow runners can assert before the refresh commits, so wait on an explicit condition or deadline instead.
  - confidence: 0.74
