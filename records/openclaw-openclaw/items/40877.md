## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Avoid the remaining CoreLocation main-thread probe:** `apps/ios/Sources/Gateway/GatewayConnectionController.swift:983-985`
  - body: currentPermissions() still calls CLLocationManager.locationServicesEnabled() while building gateway permissions. Apple documents denied authorization as covering globally disabled Location Services and says this call is optional when an app needs separate global availability, so leaving it here keeps a warning-prone probe in the path this PR is trying to quiet. ([developer.apple.com](https://developer.apple.com/documentation/corelocation/clauthorizationstatus/denied?utm_source=openai))
  - confidence: 0.86
- **[P2] Wait for the connect task by condition, not yields:** `apps/ios/Sources/Gateway/GatewayConnectionController.swift:1115-1123`
  - body: _test_startAutoConnect() still returns after 100 Task.yield() attempts even though startAutoConnect() now awaits permission work before applying the config. On slower schedulers this helper can return before the config is applied, making the new reconnect assertions flaky; wait on an explicit condition with a deadline instead.
  - confidence: 0.82
- **[P3] Make the refresh regression test wait for commit:** `apps/ios/Tests/GatewayConnectionControllerTests.swift:289-292`
  - body: refreshActiveGatewayRegistrationPreservesLatestAuthAcrossAwait only yields 200 times before checking credentials. If the refresh task has not applied yet, the test can pass without exercising the stale-overwrite path it is meant to guard; wait on a visible completion or state change with a timeout.
  - confidence: 0.75
