## Summary

Keep open: the PR is useful but still leaves one CoreLocation main-thread probe in the permission snapshot path and one scheduler-dependent async test helper, so it needs narrow repair before merge.

## What This Changes

This PR makes iOS gateway permission snapshot generation async for Speech authorization, reuses a CLLocationManager instance, adds reconnect/refresh race guards, and expands GatewayConnectionController tests.

## Best Possible Solution

Land this PR or a tightened replacement after removing the remaining CoreLocation main-thread probe and replacing fixed-yield async waits with deterministic completion or timeout-based helpers.

## Reproduction Assessment

Yes for the source-level path: current main routes gateway option creation through currentPermissions(), which directly probes Speech and CoreLocation authorization APIs on the @MainActor controller path. I did not run simulator or device console logging in this read-only review.

## Solution Assessment

No, not as currently proposed. Moving Speech authorization off the main actor and guarding reconnect races are useful, but the PR should finish the CoreLocation warning fix and make the new async waits deterministic before merge.
