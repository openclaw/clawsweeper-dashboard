## Summary

Keep open: the PR targets a real iOS permission-snapshot warning path, but the latest head still leaves one CoreLocation probe in place, has a flaky async test wait, and lacks visible after-fix runtime proof from the contributor.

## What This Changes

The PR makes iOS gateway permission snapshots async for Speech authorization, reuses a stored CLLocationManager, adds reconnect/refresh race guards, and expands GatewayConnectionController tests.

## Best Possible Solution

Land a tightened iOS controller patch that moves Speech off-main, removes the remaining warning-prone CoreLocation probe while preserving Location Services semantics, waits deterministically in tests, and includes redacted iOS runtime proof.

## Reproduction Assessment

Yes for a source-level path: current main routes gateway option creation through currentPermissions(), which probes Speech and CoreLocation authorization APIs directly. I did not reproduce the live Xcode/CoreLocation warning in this read-only review.

## Solution Assessment

No, not as currently proposed. Moving Speech authorization off the main actor is useful, but the branch should finish the CoreLocation warning fix and replace the scheduler-dependent test wait before merge.
