## Summary

Keep open: the PR is useful and scoped to the right iOS gateway area, but the latest head still leaves one CoreLocation warning-prone call in the permission snapshot path and one new async test helper can return before the connect task commits.

## What This Changes

The PR makes iOS gateway permission snapshots async for Speech authorization, reuses a CLLocationManager instance, adds reconnect/refresh race guards, and expands GatewayConnectionController tests.

## Best Possible Solution

Finish this PR by removing all CoreLocation warning-prone permission probes from currentPermissions and replacing fixed-yield async test waits with deadline or completion-based waits, then run the focused iOS lint/parse/build and console checks.

## Reproduction Assessment

Yes for the source-level path: current main routes gateway option creation through currentPermissions(), which directly probes Speech and CoreLocation authorization APIs on the @MainActor controller path. I did not run simulator or device console logging in this read-only review.

## Solution Assessment

No, not as currently proposed. Moving Speech authorization off the main actor and guarding reconnect races are useful, but the PR should finish the CoreLocation warning fix and make the new async waits deterministic before merge.
