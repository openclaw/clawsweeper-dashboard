## Summary

Keep open: current main still has the iOS permission snapshot warning path, but this PR still leaves a CoreLocation main-thread probe in that path, has a scheduler-dependent async test wait, and lacks viewable after-fix real behavior proof.

## What This Changes

The PR makes iOS gateway permission snapshots async for Speech authorization, reuses a stored CLLocationManager, adds gateway reconnect/refresh race guards, and expands GatewayConnectionController tests.

## Best Possible Solution

Land a tightened iOS controller patch that moves Speech off-main, removes warning-prone CoreLocation probes while preserving Location Services semantics, waits deterministically in tests, and includes redacted device or simulator proof.

## Reproduction Assessment

Yes for a source-level reproduction path: current main routes gateway option creation through currentPermissions(), which directly probes Speech and CoreLocation authorization APIs. I did not reproduce the live Xcode warning in this read-only review.

## Solution Assessment

No, not as currently proposed. Moving Speech authorization off the main actor and guarding reconnect races are useful, but the branch should finish the CoreLocation warning fix and remove the scheduler-dependent test wait before merge.
