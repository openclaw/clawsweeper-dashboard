## Summary

Keep open: current main still has the iOS permission-snapshot warning path, but this PR still leaves a CoreLocation class-method probe in that path, keeps one scheduler-dependent async test helper, and lacks contributor-provided real behavior proof.

## What This Changes

The PR makes iOS gateway permission snapshots async for Speech authorization, reuses a stored CLLocationManager, adds gateway reconnect/refresh race guards, and expands GatewayConnectionController tests.

## Best Possible Solution

Land a tightened iOS controller fix that removes warning-prone CoreLocation and Speech probes from permission snapshots without changing permission semantics, uses deterministic async test waits, and includes redacted runtime proof from device or simulator.

Yes for the source-level path: current main routes gateway option creation through currentPermissions(), which directly probes Speech and CoreLocation APIs. I did not live-reproduce the Xcode/CoreLocation runtime warning in this read-only review.

## Solution Assessment

No, not as currently proposed. The branch improves the Speech probe and several async reconnect races, but it should remove the remaining CoreLocation warning path and replace the fixed-yield wait before merge.
