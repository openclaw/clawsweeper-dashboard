## Summary

Keep open: current main still has the warning-prone iOS permission snapshot path, and this PR remains a plausible fix, but the latest head still leaves a CoreLocation main-thread probe and lacks contributor-provided real runtime proof.

## What This Changes

The PR makes iOS gateway permission snapshots async for Speech authorization, reuses a stored CLLocationManager, adds gateway reconnect/refresh race guards, and expands GatewayConnectionController tests.

## Best Possible Solution

Land a tightened version that routes location permission through delegate-managed state or a verified equivalent without main-thread CoreLocation class probes, keeps Speech off-main, uses deterministic async test waits, and includes redacted runtime proof.

## Reproduction Assessment

Yes for the source-level path: current main routes gateway option creation through currentPermissions(), which directly probes Speech and CoreLocation authorization APIs from the iOS gateway controller path. I did not reproduce the live Xcode/CoreLocation warning in this read-only review.

## Solution Assessment

No, not as currently proposed. Moving Speech authorization off the main actor and adding reconnect guards are useful, but the PR should finish the CoreLocation warning fix, remove the scheduler-dependent helper wait, and add real behavior proof before merge.
