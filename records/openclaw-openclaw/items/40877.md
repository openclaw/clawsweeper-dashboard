## What This Changes

The PR updates iOS gateway permission snapshot handling around CoreLocation and Speech authorization, adds reconnect/refresh race guards, and expands GatewayConnectionController tests.

## Best Possible Solution

Land a tightened iOS fix that moves Speech authorization off the main actor, routes location permission through delegate-managed state without warning-prone CoreLocation probes, keeps the intended Location Services semantics, uses deterministic async test waits, and includes redacted real runtime proof.

## Reproduction Assessment

## Solution Assessment

No, not as currently proposed. The Speech off-main change and reconnect guards are useful, but the PR should remove the remaining CoreLocation main-thread probe and make the async test wait deterministic before merge.
