## Summary

Keep open: this PR is useful and narrow, but PR head still leaves a CoreLocation main-thread permission probe in the targeted warning path and one async test helper remains scheduler-dependent.

## What This Changes

The PR makes iOS gateway permission snapshots async for Speech authorization, reuses a stored location manager, adds async reconnect/refresh race guards, and expands GatewayConnectionController tests.

## Best Possible Solution

Land this PR or a tightened replacement after removing the remaining CoreLocation main-thread probe, making async connect test waits deterministic, and adding the required changelog entry if the fix is treated as user-facing.

## Reproduction Assessment

Yes for the source-level path: current main routes gateway option creation through currentPermissions(), which directly probes Speech and CoreLocation authorization APIs on the @MainActor controller path. I did not run simulator or device console logging in this read-only review.

## Solution Assessment

No, not as currently proposed. Moving Speech authorization off the main actor and guarding reconnect races are useful, but the PR should finish the CoreLocation warning fix and remove scheduler-dependent test waits before merge.
