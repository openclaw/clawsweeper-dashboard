## Summary

Keep this PR open: it is the canonical open fix for the iOS permission-snapshot warning work, but the current branch still leaves a CoreLocation main-thread probe, retains one scheduler-dependent test wait, and lacks contributor-provided real behavior proof.

## What This Changes

The PR makes iOS gateway permission snapshots async for Speech authorization, reuses a stored CLLocationManager, adds gateway reconnect/refresh race guards, and expands GatewayConnectionController tests.

## Best Possible Solution

Land a tightened iOS controller patch that moves Speech off-main, removes warning-prone CoreLocation probes while preserving global Location Services semantics, uses deterministic async test waits, and includes redacted device or simulator proof.

## Reproduction Assessment

Yes for a source-level reproduction path: current main routes gateway option creation through currentPermissions(), which directly probes Speech and CoreLocation authorization APIs. I did not reproduce the live Xcode/CoreLocation warning in this read-only review.

## Solution Assessment

No, not as currently proposed. Moving Speech authorization off the main actor and guarding reconnect races are useful, but the branch should finish the CoreLocation warning fix and remove the scheduler-dependent test wait before merge.
