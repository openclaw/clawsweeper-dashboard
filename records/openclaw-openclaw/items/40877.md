## Summary

Keep open: the PR is useful and has already addressed several review threads, but the current head still leaves one CoreLocation main-thread probe in the permission snapshot path and keeps a scheduler-dependent wait in the new async auto-connect test helper.

## What This Changes

The branch makes iOS gateway connect-option construction async, moves Speech authorization probing to a detached task, reuses a CLLocationManager, adds reconnect/refresh race guards, and expands GatewayConnectionController tests.

## Best Possible Solution

Land this PR or a tightened replacement after removing or rerouting the remaining CoreLocation main-thread probe and replacing fixed-yield async waits with deterministic completion or deadline-based helpers.

## Reproduction Assessment

Yes for the source-level path: current main routes gateway option creation through currentPermissions(), which directly probes Speech and CoreLocation authorization APIs. I did not run an iOS simulator or device console check in this read-only review.

## Solution Assessment

No, not as currently proposed. Moving Speech authorization off the main actor and adding reconnect guards are useful, but the PR should complete the CoreLocation warning fix and make the new async waits deterministic before merge.
