## Summary

Keep open: this PR addresses part of the reported iOS permission warning path, but the latest head still leaves a CoreLocation main-thread probe in `currentPermissions()` and adds a scheduler-dependent async test helper, so it is not ready to merge as-is.

## What This Changes

The PR makes iOS gateway permission snapshots async for Speech authorization, reuses a stored location manager, adds async gateway reconnect/refresh race guards, and expands `GatewayConnectionController` tests.

## Best Possible Solution

Land this PR or a tightened replacement after removing the remaining CoreLocation warning source, making the async test wait deterministic, and adding the required changelog entry.

Yes for the source-level path: current main routes gateway option creation through `currentPermissions()`, which directly probes Speech and CoreLocation authorization APIs on the `@MainActor` controller path. I did not run simulator or device console logging in this read-only review.

## Solution Assessment

No, not as currently proposed. Moving Speech authorization off the main actor and guarding reconnect races are useful, but the PR should finish the CoreLocation warning fix and remove scheduler-dependent test waits before merge.
