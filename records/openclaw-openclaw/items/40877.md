## Summary

Keep open. Current main still has the warning-prone iOS permission snapshot path, so this PR is not obsolete, but the latest head still leaves a CoreLocation main-thread probe in that path and adds a scheduler-dependent async test helper that can flake before merge.

## What This Changes

The PR makes iOS gateway connect-option creation async, moves Speech authorization status probing into a detached utility task, reuses a stored CLLocationManager, adds async gateway refresh/reconnect guards, and expands GatewayConnectionController tests.

## Best Possible Solution

Land this PR or a tightened equivalent after completing the CoreLocation portion of the permission snapshot fix and making the async auto-connect test wait deterministic, while preserving the async Speech probe and focused reconnect race guards.

## Reproduction Assessment

Yes for the source-level issue and PR blockers: current main routes makeConnectOptions() through currentPermissions() with Speech/CoreLocation authorization probes on the main-actor controller path, and PR head still shows the remaining CoreLocation probe plus the fixed-yield wait. The exact console warning symptom still needs simulator or device verification.

## Solution Assessment

No, not as currently proposed. Moving Speech authorization off the main actor is the right direction, but the patch should remove or reroute the remaining CoreLocation services-enabled probe and replace the fixed-yield test wait before it is the narrow maintainable fix.
