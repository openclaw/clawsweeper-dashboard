## Summary

Keep open. Current main partially satisfies the request through SearXNG/web_search and the shared SSRF guard, but web_fetch, media URL inputs that reuse web_fetch policy, and Gateway OpenResponses input_file/input_image URL fetches still lack a broad private-network opt-in. The remaining work is security-sensitive and overlaps open partial PR #74790, so this needs maintainer security/product review rather than cleanup closure.

## What This Changes

Add private/local network access support for web_fetch, web_search/image tool URL fetches, and Gateway URL-fetch paths, ideally through an explicit allowPrivateNetwork-style opt-in.

## Best Possible Solution

Keep this as the broader tracker until maintainers choose a default-off private-network policy shape for web_fetch/media references and a separate documented Gateway OpenResponses URL-input policy, with tests for direct private hosts, DNS-to-private, redirects, allowlists, cache isolation, and docs/changelog updates.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: the requested web_fetch private-network config key is rejected by strict schema/runtime surfaces, and Gateway input_file/input_image URL fetches explicitly pass allowPrivateNetwork: false.

## Solution Assessment

Unclear as proposed. A default-off, explicitly named opt-in is the maintainable direction, but automatically enabling private-network access in local Gateway mode and widening Gateway URL-input behavior needs maintainer security/product review.
