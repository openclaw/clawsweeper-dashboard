## Summary

Keep open: current main and v2026.5.18 still lack the requested broad private-network opt-in for arbitrary `web_fetch`, image/media, and gateway-local behavior. Related work now covers SearXNG/private web-search and model-provider cases, but the remaining SSRF config/API boundary still needs maintainer security and product approval.

## What This Changes

Request a config/API path that lets web tools and local gateway deployments opt into private-network access for self-hosted and local-first infrastructure.

## Best Possible Solution

Keep this as the maintainer-owned broad SSRF policy discussion, and route the narrower web_fetch implementation through https://github.com/openclaw/openclaw/issues/39604 and https://github.com/openclaw/openclaw/pull/39630 once maintainers approve the security shape.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection gives high confidence that current main still blocks arbitrary private `web_fetch` targets while preserving only narrow fake-IP exceptions.

## Solution Assessment

Unclear for the exact proposed automatic local-mode behavior. The safer path is a maintainer-approved, default-off nested SSRF policy with explicit metadata/link-local, redirect, cache, docs, and test coverage.
