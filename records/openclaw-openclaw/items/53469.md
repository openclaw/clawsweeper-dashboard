## Summary

Keep open: current main and the latest release still lack the requested broad private-network opt-in for arbitrary web_fetch, remote media/image fetches, and gateway local-mode policy. SearXNG web_search and model-provider private endpoints are already covered, but the remaining behavior is a security-sensitive product/API decision with an older canonical web_fetch thread and an open stale implementation candidate still under review.

## What This Changes

The issue requests opt-in private/local network access for web tools and gateway/local deployments, including web_fetch, web_search, image/media fetches, and an automatic local gateway mode default.

## Best Possible Solution

Approve one default-off SSRF policy contract for web_fetch and inherited remote media fetches, explicitly decide whether gateway local mode should ever widen it automatically, and preserve the already-shipped SearXNG and provider-request scoped behavior.

## Reproduction Assessment

Yes, at source level for the config gap. Current main’s strict web_fetch schema and runtime policy forwarding show arbitrary private-network web_fetch/media access is still blocked, while SearXNG and provider private endpoint paths are separate scoped implementations.

## Solution Assessment

Unclear for the requested automatic local-mode behavior. The safer maintainable solution is a maintainer-approved, default-off SSRF policy surface with explicit threat-model coverage rather than an unconditional local gateway default.
