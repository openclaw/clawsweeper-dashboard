## Summary

Keep open: current main and v2026.5.18 still lack the requested broad private-network opt-in for arbitrary `web_fetch` and media/gateway fetch paths. SearXNG search and model-provider work cover parts of the use case, but the remaining SSRF policy surface is security-sensitive and already has open implementation candidates needing maintainer review.

## What This Changes

The issue requests opt-in private/local network access for web tools and gateway/local deployments so agents can reach self-hosted or internal services.

## Best Possible Solution

Keep this open until maintainers approve and land a default-off, documented private-network SSRF policy covering `web_fetch`, media/image fetches, gateway/local scope, redirects, cache partitioning, and metadata/link-local behavior.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source inspection gives high confidence in the current-main gap: strict schema/runtime paths still lack a broad private-network `web_fetch` policy while tests and docs preserve blocking.

## Solution Assessment

No for the issue's proposed automatic local-mode default as-is. The safer direction is a maintainer-approved, default-off SSRF policy contract with explicit metadata/link-local, redirect, cache, media, and gateway semantics.
