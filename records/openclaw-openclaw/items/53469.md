## Summary

Keep open: current main still lacks the requested broad private-network opt-in for arbitrary web_fetch, media/image remote fetches, and gateway/local-mode policy, while narrower SearXNG search and model-provider private endpoint support are already covered. ## What This Changes

Request opt-in private/local network access for web_fetch, web_search, image/media tools, and gateway-local deployments.

## Best Possible Solution

Approve one default-off SSRF policy contract for web tools and gateway/local deployments that explicitly covers metadata/link-local handling, redirects, DNS rebinding, cache isolation, allowlists, audit/docs, and tests while preserving the already-shipped SearXNG and provider-request scoped behavior.

Yes, at source level for the config gap. Current main’s strict web_fetch schema and runtime policy forwarding show arbitrary private-network web_fetch/media access is still blocked, while SearXNG and provider private endpoint paths are separate scoped implementations.

## Solution Assessment

Unclear for the exact automatic local-mode behavior. The safer maintainable solution is a maintainer-approved, default-off SSRF policy surface with explicit threat-model coverage rather than an unconditional local gateway default.
