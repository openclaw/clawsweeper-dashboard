## Summary

Keep open. Current main still lacks the requested broad private-network opt-in for arbitrary web_fetch, image/media, and gateway URL-fetch paths; SearXNG self-hosted web_search is partially covered, but related web_fetch work remains open at https://github.com/openclaw/openclaw/issues/39604 and needs maintainer security/API judgment.

## What This Changes

The issue requests a default-off private-network opt-in for web_fetch, web_search, image/media tools, and gateway URL fetches, with local gateway deployments auto-enabling it.

## Best Possible Solution

Define one maintainer-approved SSRF policy contract that keeps defaults strict, preserves current fake-IP and self-hosted SearXNG behavior, documents the operator trust boundary, and then apply it consistently only to the approved tool and gateway surfaces.

## Reproduction Assessment

Not applicable as a bug repro because this is a feature request. Source inspection confirms current main preserves strict private/internal blocking for arbitrary web_fetch, media/image, and gateway URL fetches while only targeted self-hosted web_search paths are allowed.

## Solution Assessment

Unclear as proposed. The capability is valid, but the blanket local-mode auto-enable and top-level allowPrivateNetwork shape need maintainer security/product decisions; the related nested ssrfPolicy direction appears safer.
