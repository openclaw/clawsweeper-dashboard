- `impact:message-loss`: Unbounded identity-emoji fallback matching can misroute ordinary group-room messages into an agent reply workflow when that emoji appears in normal text.

## Summary

Keep open: current main still derives shared fallback mention regexes from `identity.emoji`, Matrix consumes those regexes for room mention checks, and the remaining work is a new config/default surface that needs maintainer product direction.

## What This Changes

Requested a Matrix/shared group-chat configuration switch to stop `identity.emoji` from being used as an automatic mention trigger.

## Best Possible Solution

Add a shared, backward-compatible derived-mention option that disables only identity-emoji fallback matching while preserving native platform mentions, name-derived fallback matching, and explicit configured regex patterns.

## Reproduction Assessment

Yes, source-reproducible: configure an agent with `identity.emoji` and no explicit mention patterns, then `buildMentionRegexes()` derives that emoji regex and Matrix passes it into `resolveMentions()`. I did not run a live Matrix room reproduction.

## Solution Assessment

No for a Matrix-only switch; the better solution is a shared backward-compatible derived-mention option because the fallback derivation is cross-channel. The exact config shape still needs maintainer product approval.
