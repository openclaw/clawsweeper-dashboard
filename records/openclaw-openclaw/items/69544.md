## Summary

Keep open. Current main still derives fallback mention regexes from `identity.emoji`, Matrix still consumes those shared regexes for room mention gating, and the shared group-chat config has no emoji-only opt-out. The adjacent #70864 is related but the discussion says it does not cover this behavior, while the focused #74211 was closed unmerged.

## What This Changes

The issue requests a configuration option to prevent Matrix/shared group-chat fallback mention matching from treating an agent identity emoji as a mention trigger.

## Best Possible Solution

Add a backward-compatible shared group-chat setting with global and per-agent precedence that defaults to current behavior but can exclude only `identity.emoji` from identity-derived fallback regexes. Native Matrix mentions, name-derived fallback matching, and explicit `groupChat.mentionPatterns` should remain unchanged, with schema/types/help/docs and focused tests updated together.

## Reproduction Assessment

Yes. Configure an agent with `identity.emoji`, omit agent-level and global `groupChat.mentionPatterns`, call `buildMentionRegexes`, and Matrix `resolveMentions` will mark any message containing that emoji as mentioned through the shared regex fallback path.

## Solution Assessment

The Matrix-only switch from the title is not the best final shape because the fallback derivation is shared across channel runtimes. A shared global/per-agent control over identity-derived fallback patterns is the narrowest maintainable solution, while #70864's scoped mention-pattern policy is adjacent rather than sufficient.
