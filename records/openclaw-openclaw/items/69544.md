## Label Justifications

- `P2`: The issue affects false group-chat agent wakeups in shared rooms, but it is a bounded behavior/config improvement rather than an emergency.
- `impact:message-loss`: The requested change concerns mention detection that can misroute shared-room messages to the wrong agent workflow.

## Summary

Keep open: current main and the latest release still derive fallback mention regexes from `identity.emoji` with no config/schema opt-out, while the merged scoped mention-pattern policy explicitly left emoji-derived fallback matching out of scope and the direct fix PR was closed unmerged.

## What This Changes

Add configuration to let Matrix/shared group-chat mention derivation opt out of treating an agent's identity emoji as a fallback mention trigger.

## Best Possible Solution

Add a shared, backward-compatible current-schema option that excludes only `identity.emoji`-derived fallback regexes while preserving native mentions, `identity.name` derivation, and explicit `groupChat.mentionPatterns`, with schema/help/docs and focused mention tests.

## Reproduction Assessment

Yes. Source inspection shows `identity.emoji` is compiled into fallback mention regexes when explicit mention patterns are absent, and Matrix uses those regexes to decide `wasMentioned`; no live Matrix scenario was run in this read-only pass.

## Solution Assessment

Unclear until maintainers choose the config surface. A shared opt-out is the best-looking direction because the fallback derivation is shared, while Matrix-only policy would leave sibling channels inconsistent.
