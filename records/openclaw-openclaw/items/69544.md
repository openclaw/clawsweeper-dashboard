## Summary

Keep open. Current main still derives fallback mention regexes from `identity.emoji`, Matrix still uses those regexes for room mention gating, and there is no global/per-agent emoji-only opt-out in config or docs; the adjacent scoped policy PR is open but does not cover this behavior, while the focused opt-out PR was closed unmerged.

## What This Changes

Request a Matrix/shared group-chat config switch to opt out of `identity.emoji`-derived mention fallback triggers.

## Best Possible Solution

Add a backward-compatible shared group-chat setting with global and per-agent precedence that defaults to current emoji fallback behavior but can exclude only `identity.emoji` from derived fallback regexes while preserving native mentions, name-derived matching, and explicit `mentionPatterns`.

## Reproduction Assessment

Yes, by source inspection. Configure an agent with `identity.emoji` and no agent-level or global `groupChat.mentionPatterns`; `buildMentionRegexes` includes the emoji regex, and Matrix `resolveMentions` treats matching text as `wasMentioned`.

## Solution Assessment

No for a Matrix-only switch. Because fallback derivation is shared, the narrow maintainable solution is a shared global/per-agent control over identity-derived emoji fallback patterns, coordinated with https://github.com/openclaw/openclaw/pull/70864.
