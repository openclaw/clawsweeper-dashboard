## Summary

Keep open. Current main and the latest release still include `identity.emoji` in fallback mention regexes, Matrix still treats those regex hits as mentions, and there is no current config surface to disable only the emoji-derived fallback; the adjacent scoped policy PR is related but does not implement this opt-out, while the focused opt-out PR was closed unmerged.

## What This Changes

Add a Matrix-visible, preferably shared, configuration switch so agent `identity.emoji` does not have to act as a fallback mention trigger.

## Best Possible Solution

Add a backward-compatible shared group-chat derived-pattern control, with global and per-agent precedence, that defaults to current behavior but can exclude only `identity.emoji` while preserving native mentions, name-derived fallback matching, and explicit `mentionPatterns`.

## Reproduction Assessment

Yes, by source inspection. Configure an agent with `identity.emoji` and no explicit agent-level or global `groupChat.mentionPatterns`; `buildMentionRegexes` includes the emoji regex and Matrix `resolveMentions` treats matching text as `wasMentioned`.

## Solution Assessment

No for a Matrix-only switch. Because fallback derivation is shared, the more maintainable solution is a shared global/per-agent control over identity-derived emoji fallback patterns, coordinated with the open scoped mention-pattern policy work.
