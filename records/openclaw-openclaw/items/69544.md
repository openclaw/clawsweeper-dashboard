## Label Justifications

- `P2`: The issue describes unwanted group-channel agent replies with limited blast radius and a clear but product-sensitive config shape.
- `impact:message-loss`: The affected behavior can misroute group-channel messages by treating ordinary emoji usage as an agent mention.

## Summary

Keep open: current main and the latest release still derive fallback mention regexes from `identity.emoji`, Matrix still uses that shared helper for mention checks, and the merged related policy PR explicitly left emoji-derived fallback opt-out out of scope.

## What This Changes

Requested a configurable way to opt out of identity emoji-derived mention triggers for Matrix/shared group chat mentions.

## Best Possible Solution

Choose a shared, backward-compatible config shape for identity-derived fallback patterns, then implement the emoji opt-out in the shared mention derivation path with focused schema, docs, and cross-channel helper tests.

## Reproduction Assessment

Yes. The source path is clear: `buildMentionRegexes` falls back to `deriveMentionPatterns`, which includes `identity.emoji`, and Matrix passes those regexes into room mention resolution; I did not run a live Matrix room repro in this read-only review.

## Solution Assessment

Unclear until maintainers choose the config contract. The best-supported direction is a shared opt-out for identity-derived emoji fallback patterns, not a Matrix-only switch, because the fallback derivation is shared.
