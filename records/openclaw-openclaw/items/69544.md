## Summary

Keep open. Current main and the latest stable tag still derive fallback mention regexes from `identity.emoji`, Matrix still feeds those shared regexes into mention detection, and the related discussion shows the remaining work is a shared opt-out config surface rather than a Matrix-only cleanup.

## What This Changes

The issue requests a configurable opt-out so an agent's `identity.emoji` does not automatically act as a group-chat mention trigger, especially in Matrix rooms.

## Best Possible Solution

Add a backward-compatible shared derived mention-pattern control that defaults to current behavior and can exclude only the `identity.emoji` fallback globally or per agent while preserving native mentions, name-derived matching, and explicit `groupChat.mentionPatterns`.

## Reproduction Assessment

Yes, by source inspection. Configure an agent with `identity.emoji` and no explicit agent-level or global `groupChat.mentionPatterns`; the shared fallback includes the emoji regex and Matrix treats a regex text hit as `wasMentioned`.

## Solution Assessment

No for a Matrix-only switch. The maintainable fix is a shared global/per-agent derived-pattern control because the emoji fallback is produced by shared group-chat mention code before Matrix consumes it.
