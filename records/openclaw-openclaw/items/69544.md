## Label Justifications

- `P2`: The issue affects false group-chat agent wakeups in shared rooms, but it is a bounded config/product improvement rather than an emergency.
- `impact:message-loss`: The reported mention matching can misroute shared-room messages into the wrong agent workflow when an emoji appears in normal chat text.

## Summary

Keep open: current main and v2026.6.8 still derive a fallback mention regex from `identity.emoji` with no opt-out. The merged scoped mention-pattern policy is adjacent but explicitly left this emoji-derived fallback out of scope, and the exact opt-out PR is closed unmerged.

## What This Changes

Request a config-controlled opt-out for `identity.emoji`-derived fallback mention patterns so Matrix/shared group chats do not wake agents on any matching emoji.

## Best Possible Solution

Adopt a shared, backward-compatible derived-mention option that defaults to current behavior and disables only the identity-emoji fallback while preserving native mentions, name fallback, and explicit `groupChat.mentionPatterns`.

## Reproduction Assessment

Yes. Current source shows `identity.emoji` is converted directly into a fallback regex, and Matrix uses the shared mention helper for room mention checks.

## Solution Assessment

No for a Matrix-only switch; the best fix is shared because the fallback derivation is shared across channels that call `buildMentionRegexes`. A maintainer should choose the exact config shape before implementation.
