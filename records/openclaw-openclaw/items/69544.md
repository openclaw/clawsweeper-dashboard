## Summary

Keep open: current main and the latest release still derive fallback mention regexes from `identity.emoji` with no opt-out, while the merged scoped mention-pattern policy explicitly left this behavior out of scope and the direct opt-out PR is closed unmerged.

## What This Changes

The issue requests a configurable way to disable identity emoji-derived mention fallback matching, initially observed in Matrix group chats.

## Best Possible Solution

Add a shared, backward-compatible derived-mention config shape that can disable only identity emoji fallback matching while preserving native platform mentions, name-derived fallback, and explicit configured regexes.

## Reproduction Assessment

Yes from source, but not from a live Matrix room. Current `deriveMentionPatterns()` still turns `identity.emoji` into an unbounded regex and Matrix consumes the shared regexes for mention checks.

## Solution Assessment

No for a Matrix-only switch. The best direction is a shared, backward-compatible derived-mention option because the fallback derivation is cross-channel and explicit `groupChat.mentionPatterns` already provide a heavier workaround.
