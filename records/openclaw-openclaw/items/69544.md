## Summary

Keep open. Current main and the latest release still derive fallback mention regexes from `identity.emoji`, Matrix still treats those regex matches as mentions, and the repository has no config key that disables only the emoji-derived fallback; the focused fix PR is closed unmerged and the adjacent scoped mention-policy PR explicitly does not cover this opt-out.

## What This Changes

The issue requests a Matrix/shared configuration knob to disable identity emoji-derived fallback mention patterns while preserving name-derived and explicit mention matching.

## Best Possible Solution

Add a backward-compatible shared derived-mention-pattern control with global and per-agent precedence, defaulting to current behavior while excluding only the `identity.emoji` fallback when disabled.

## Reproduction Assessment

Yes, by source inspection. Configure an agent with `identity.emoji` and no explicit agent-level or global `groupChat.mentionPatterns`; `buildMentionRegexes` includes the emoji regex and Matrix `resolveMentions` treats a text match as `wasMentioned`.

## Solution Assessment

No for a Matrix-only switch. Because fallback derivation is shared, the maintainable direction is a shared global/per-agent control that preserves native mentions, name-derived matching, and explicit configured regexes.
