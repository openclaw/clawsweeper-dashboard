## Summary

Keep open: current main still derives fallback mention regexes from `identity.emoji`, Matrix still consumes those shared regexes for room mention checks, and the exact shared config surface still needs maintainer product approval.

## What This Changes

The issue requests a configuration switch to prevent an agent `identity.emoji` from becoming an implicit group-chat mention trigger.

## Best Possible Solution

Add a shared, backward-compatible derived-mention option that disables only identity-emoji fallback matching while preserving native platform mentions, name-derived fallback matching, and explicit configured regex patterns.

## Reproduction Assessment

Yes from source inspection, though not from a live Matrix room. `deriveMentionPatterns()` still turns `identity.emoji` into an unbounded regex, and Matrix passes those shared regexes into room mention checks.

## Solution Assessment

No for a Matrix-only switch; the better solution is a shared, backward-compatible derived-mention option because the fallback derivation is cross-channel. The exact config shape still needs maintainer product approval.
