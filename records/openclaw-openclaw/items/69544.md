## Summary

Keep open. Current main and the latest release still derive fallback mention regexes from `identity.emoji`, Matrix still consumes those shared regexes, and the merged scoped mention-policy work explicitly did not add the requested emoji-only opt-out.

## What This Changes

The issue requests a configurable opt-out so identity emojis do not automatically become fallback group-chat mention triggers.

## Best Possible Solution

Add a backward-compatible shared derived-mention-pattern control that can disable only identity-emoji fallback while preserving native mentions, name-derived fallback, and explicit configured `mentionPatterns`.

## Reproduction Assessment

Yes. Source inspection shows `buildMentionRegexes()` falls back to `deriveMentionPatterns()`, which includes `identity.emoji` as a regex whenever no explicit agent/global `mentionPatterns` are configured.

## Solution Assessment

No final API choice yet. A Matrix-only switch would be the wrong layer because the emoji fallback is produced in shared group-chat mention code before Matrix consumes it.
