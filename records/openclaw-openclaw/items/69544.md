## Label Justifications

- `P2`: False group-chat wakeups in shared Matrix rooms are a normal-priority channel behavior improvement with limited blast radius.
- `impact:message-loss`: The reported emoji fallback can route ordinary shared-room messages to the wrong agent workflow when a configured emoji appears in normal text.

## Summary

Keep open: current main and the latest release still derive fallback mention regexes from `identity.emoji` with no opt-out. The merged scoped mention-pattern policy is adjacent but explicitly left this behavior out of scope, and the exact opt-out PR is closed unmerged, so the remaining work is a maintainer product/config decision.

## What This Changes

Request to add Matrix/shared configuration for disabling `identity.emoji`-derived fallback mention matching while preserving native mentions and explicit mention patterns.

## Best Possible Solution

Adopt a shared, backward-compatible derived-mention option that defaults to current behavior and disables only the identity-emoji fallback while preserving native mentions, name fallback, and explicit `groupChat.mentionPatterns`.

## Reproduction Assessment

Yes. Source inspection on current main shows `identity.emoji` is compiled into a fallback regex and Matrix uses the shared mention helper for room mention checks; I did not run a live Matrix room reproduction.

## Solution Assessment

No for a Matrix-only switch. The maintainable solution is a shared, backward-compatible derived-mention config decision because the fallback derivation is shared across channels that call `buildMentionRegexes`.
