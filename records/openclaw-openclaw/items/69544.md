## Summary

Keep open. Current main still derives fallback mention regexes from `identity.emoji`, Matrix still treats those regex matches as `wasMentioned`, and there is no current config/docs surface to disable only emoji-derived fallback mentions; the adjacent scoped mention-policy PR is open but does not close this behavior, while the focused opt-out PR was closed unmerged.

## What This Changes

The issue requests a configurable way to prevent an agent's identity emoji from being treated as a fallback mention trigger in Matrix/group chat handling.

## Best Possible Solution

Add a backward-compatible shared group-chat setting, with global and per-agent precedence, that defaults to current behavior but can exclude only the `identity.emoji` fallback while preserving native mentions, name-derived matching, and explicit `mentionPatterns`.

## Reproduction Assessment

Yes, by source inspection. Configure an agent with `identity.emoji` and no explicit agent-level or global `groupChat.mentionPatterns`; `buildMentionRegexes` includes the emoji regex and Matrix `resolveMentions` treats matching text as `wasMentioned`.

## Solution Assessment

No for a Matrix-only switch. Because fallback derivation is shared, the maintainable solution is a shared global/per-agent control over identity-derived emoji fallback patterns, coordinated with the open scoped mention-pattern policy work.
