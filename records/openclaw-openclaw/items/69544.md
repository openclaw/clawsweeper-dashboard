## Summary

Keep open: current main and the latest release still derive fallback mention regexes from `identity.emoji` with no opt-out, while the merged scoped mention-pattern policy explicitly left this behavior out of scope and the direct opt-out pull request is closed unmerged.

## What This Changes

The issue requests a Matrix/shared configuration option to opt out of identity emoji-derived mention matching in group chats.

## Best Possible Solution

Add a shared, backward-compatible derived-mention option that can disable only identity emoji fallback matching while preserving native platform mentions, name-derived fallback matching, and explicit configured regexes.

## Reproduction Assessment

Yes from source, but not from a live Matrix room. Current `deriveMentionPatterns()` still turns `identity.emoji` into an unbounded regex, and Matrix consumes those shared regexes for mention checks.

## Solution Assessment

No for a Matrix-only switch. The best direction is a shared, backward-compatible derived-mention option because the fallback derivation is cross-channel and explicit `groupChat.mentionPatterns` already provide a heavier workaround.
