## Summary

Keep open. Current main still derives fallback mention regexes from `identity.emoji`, Matrix still treats those regex hits as mentions, and neither the current config surface nor the latest release exposes an emoji-only opt-out; the scoped mention-policy PR at https://github.com/openclaw/openclaw/pull/70864 is adjacent but explicitly does not cover this fallback behavior, while the focused opt-out PR at https://github.com/openclaw/openclaw/pull/74211 was closed unmerged.

## What This Changes

Request a configuration switch to disable identity-emoji-derived mention matching while preserving the existing default behavior.

## Best Possible Solution

Add a backward-compatible shared group-chat derived-mention setting with global and per-agent precedence that can exclude only the emoji fallback while preserving native mentions, name-derived matching, and explicit `mentionPatterns`.

## Reproduction Assessment

Yes, by source inspection. Configure an agent with `identity.emoji` and no explicit agent-level or global `groupChat.mentionPatterns`; `buildMentionRegexes` includes the emoji regex and Matrix `resolveMentions` treats matching text as `wasMentioned`.

## Solution Assessment

No for a Matrix-only switch. Because fallback mention derivation is shared, the maintainable solution is a shared global/per-agent control over identity-derived emoji patterns, coordinated with the scoped mention-policy work.
