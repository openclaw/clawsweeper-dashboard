## Summary

Keep open. Current main still derives fallback mention regexes from `identity.emoji`, Matrix uses those shared regexes for room gating, and there is still no emoji-only opt-out; #70864 is adjacent but open and does not add this control, while #74211 was closed unmerged.

## What This Changes

Add a configuration option so users can disable identity-emoji-derived mention triggers without removing the agent emoji or replacing all mention patterns.

## Best Possible Solution

Add a backward-compatible shared group-chat setting with global and per-agent precedence that defaults to current behavior but can exclude only `identity.emoji` from identity-derived fallback regexes, while leaving native Matrix mentions, name-derived fallback matching, and explicit `groupChat.mentionPatterns` unchanged.

## Reproduction Assessment

Yes. Configure an agent with `identity.emoji` and no agent-level or global `groupChat.mentionPatterns`; `buildMentionRegexes` includes the emoji regex, and Matrix `resolveMentions` treats any room message containing that emoji as mentioned.

## Solution Assessment

No for a Matrix-only switch; the fallback derivation is shared across channels. A shared global/per-agent control over identity-derived fallback patterns is the narrowest maintainable solution, and #70864's scoped configured-pattern policy is adjacent rather than sufficient.
