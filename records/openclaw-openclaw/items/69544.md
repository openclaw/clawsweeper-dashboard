## Summary

Keep open. Current main still derives fallback mention regexes from `identity.emoji`, Matrix still feeds those shared regexes into room mention detection, and the config/docs surface has no emoji-only opt-out; #70864 is adjacent but open and intentionally does not cover this behavior, while #74211 was closed unmerged.

## What This Changes

Request a Matrix-visible/shared config switch that lets users disable `identity.emoji` as an automatically derived group-chat mention trigger while preserving current defaults.

## Best Possible Solution

Add a backward-compatible shared group-chat setting with global and per-agent precedence that excludes only the `identity.emoji`-derived fallback regex while leaving name-derived fallback matching, native Matrix mentions, and explicit `groupChat.mentionPatterns` unchanged.

## Reproduction Assessment

Yes. Configure an agent with `identity.emoji` and no agent-level or global `groupChat.mentionPatterns`; `buildMentionRegexes` includes the emoji regex, and Matrix `resolveMentions` treats room text containing that emoji as mentioned.

## Solution Assessment

No for a Matrix-only switch. The behavior comes from the shared mention derivation path, so the maintainable solution is a shared global/per-agent control rather than a Matrix-specific channel key.
