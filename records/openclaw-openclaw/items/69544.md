- `impact:message-loss`: Unbounded identity-emoji fallback matching can misroute ordinary group-room messages into an agent reply workflow when the emoji appears in normal text.

## Summary

Keep open: current main and the latest release still derive fallback mention regexes from `identity.emoji` with no emoji-only opt-out, and the related scoped mention-pattern PR explicitly left this behavior out of scope while the direct opt-out PR closed unmerged.

## What This Changes

Requests a configurable way to disable Agent `identity.emoji` as a derived mention trigger, especially for Matrix group rooms where ordinary emoji text can wake an agent.

## Best Possible Solution

Add a shared, backward-compatible derived-mention option that disables only identity-emoji fallback matching while preserving native platform mentions, name-derived fallback matching, and explicit configured regex patterns.

## Reproduction Assessment

Yes from source, but not from a live Matrix room. Current `deriveMentionPatterns()` still turns `identity.emoji` into an unbounded regex, and Matrix consumes those shared regexes for mention checks.

## Solution Assessment

No for a Matrix-only switch; the better direction is a shared, backward-compatible derived-mention option because the fallback derivation is cross-channel. The exact config shape still needs maintainer product approval.
