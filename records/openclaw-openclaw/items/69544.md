## Summary

Keep open. Current main and the latest release still derive a fallback mention regex from `identity.emoji`, Matrix still treats regex matches as mentions, and there is no config key that disables only the emoji-derived fallback. The adjacent scoped mention-pattern policy PR is related but does not implement this opt-out, while the focused opt-out PR was closed unmerged.

## What This Changes

The issue requests a Matrix or shared configuration switch to prevent `identity.emoji` from becoming an automatic group-chat mention trigger.

## Best Possible Solution

Add a backward-compatible shared derived-mention-pattern control with global and per-agent precedence, defaulting to current behavior while allowing only the `identity.emoji` fallback to be excluded.

## Reproduction Assessment

Yes, by source inspection. Configure an agent with `identity.emoji` and no explicit agent-level or global `groupChat.mentionPatterns`; `buildMentionRegexes` includes the emoji regex and Matrix `resolveMentions` treats matching text as `wasMentioned`.

## Solution Assessment

No for a Matrix-only switch. Because the fallback derivation is shared, the maintainable solution is a shared global/per-agent control that preserves native mentions, name-derived matching, and explicit configured regexes.
