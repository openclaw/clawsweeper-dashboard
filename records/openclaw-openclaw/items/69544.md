## Summary

Keep open. Current main and v2026.5.12 still derive mention regexes from `identity.emoji`, Matrix still treats those regex matches as mentions, and there is no config key that disables only the emoji-derived fallback; the focused fix PR was closed unmerged and the open scoped mention-policy PR does not implement this opt-out.

## What This Changes

Request to add a Matrix-visible/shared config opt-out so an agent's `identity.emoji` no longer automatically acts as a mention trigger.

## Best Possible Solution

Add a backward-compatible shared derived-mention-pattern control with global and per-agent precedence, defaulting to current behavior while excluding only the `identity.emoji` fallback when disabled.

## Reproduction Assessment

Yes, by source inspection. Configure an agent with `identity.emoji` and no explicit agent-level or global `groupChat.mentionPatterns`; `buildMentionRegexes` includes the emoji regex and Matrix `resolveMentions` treats matching text as `wasMentioned`.

## Solution Assessment

No for a Matrix-only switch. Because fallback derivation is shared, the maintainable direction is a shared global/per-agent control that preserves native mentions, name-derived matching, and explicit configured regexes.
