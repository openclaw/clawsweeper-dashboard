## Summary

Keep open. Current main still includes `identity.emoji` in shared fallback mention regexes, Matrix still consumes those regexes for room mention gating, and neither the current config/docs surface nor the latest release provide an emoji-only opt-out; the open scoped mention-policy PR is adjacent, while the focused opt-out PR was closed unmerged.

## What This Changes

Request a config-controlled way to disable identity emoji-derived mention fallback, especially for Matrix rooms with multiple agents.

## Best Possible Solution

Add a backward-compatible shared group-chat setting with global and per-agent precedence that defaults to current behavior but can exclude only `identity.emoji` from identity-derived fallback regexes, while leaving native mentions, name-derived fallback matching, and explicit `groupChat.mentionPatterns` unchanged.

## Reproduction Assessment

Yes, by source inspection. Configure an agent with `identity.emoji` and no agent-level or global `groupChat.mentionPatterns`; `buildMentionRegexes` includes the emoji regex, and Matrix `resolveMentions` treats matching text as `wasMentioned`.

## Solution Assessment

No for a Matrix-only switch. The fallback derivation is shared, so the narrow maintainable solution is a shared global/per-agent control over identity-derived emoji fallback patterns, coordinated with the open scoped mention-policy work.
