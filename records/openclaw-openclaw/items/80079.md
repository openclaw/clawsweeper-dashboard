## Summary

Keep open: current main still contains the reported ACP regression in source and tests. The spawn path can patch a child session before ACP metadata is persisted, and the ACP manager still treats an output-bearing stream that closes without terminal `done` as a failed turn despite the documented thread-bound delivery contract.

The issue asks to make Codex ACP Discord thread spawns persist ACP metadata before the first turn, tolerate output-bearing stream closure without terminal `done`, and deliver child replies back to the bound Discord thread.
