## Summary

Keep open: current main still contains source-level paths matching the reported ACP regression, and no linked or searched PR has merged a complete fix for this issue. The existing ClawSweeper comment remains consistent with the current checkout: the child session is patched before ACP metadata is persisted, and output-bearing ACP streams without a terminal event still end in ACP_TURN_FAILED.

## What This Changes

The issue requests a regression fix so Codex ACP Discord thread sessions initialize ACP metadata before the first spawned turn, tolerate output-bearing stream closure without a terminal done event, and post child replies back to the originating Discord thread.

## Best Possible Solution

Repair the ACP spawn metadata ordering and thread-bound completion/delivery paths with focused regression coverage, preserving the existing Discord thread-bound delivery contract without adding a new config option.

## Solution Assessment

Yes. The narrowest maintainable fix is in the existing ACP spawn/control-plane and delivery paths, with tests that distinguish no-output child exits from output-bearing closure and verify Discord-bound reply delivery.
