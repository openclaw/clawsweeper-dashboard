## What This Changes

The issue asks for Codex ACP Discord thread spawns to persist ACP metadata before first dispatch, tolerate output-bearing streams that close without `done`, and deliver child replies back to the originating bound Discord thread.

## Best Possible Solution

Repair the ACP owner path so metadata is persisted before first dispatch, output-bearing stream closure does not lose the reply, and bound Discord thread delivery is covered by focused regression and live proof.
