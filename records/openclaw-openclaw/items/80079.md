Keep open: current main still matches the reported ACP spawn and turn-completion regression, and an open closing PR already targets this issue but has not merged.

## What This Changes

The issue reports that Codex ACP Discord thread spawns can fail the first turn or keep child replies in the ACP transcript instead of posting them back to the bound Discord thread.

## Best Possible Solution

Keep this issue open until the linked fix PR is reviewed, proven with redacted real Discord/Codex ACP behavior, and either landed or replaced by a narrower maintainer fix.

## Reproduction Assessment

Yes, source-level: current main patches the child session before ACP metadata initialization and rejects output-bearing streams without a terminal `done` event. I did not live-run the Discord WSL2 scenario in this read-only review.

## Solution Assessment

Yes, the narrow fix direction is to persist/refresh ACP metadata before the first child turn, accept meaningful output-only stream completion, and keep the documented thread-bound delivery path covered by regression plus real behavior proof.
