Keep open. The PR targets a real session-lock integrity gap, but current main still lacks an owner callback/write guard for watchdog lock loss, the branch is dirty against current main, real behavior proof is missing, and the diff still has a cleanup-order bug.

## What This Changes

The PR adds session lock force-release callbacks, a lock-ownership helper, a guarded SessionManager write path, and tests to prevent writes after a watchdog releases the session lock.

## Best Possible Solution
