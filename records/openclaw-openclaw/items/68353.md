## Summary

Keep open. Current main and v2026.6.8 still lack a first-class `chat.send.author` field, durable author metadata on chat-send user transcript rows, and `authorId`/`authorName` history projection; the related open provenance PR overlaps but is conflicting and does not cover this public attribution contract.

## What This Changes

Add a first-class per-message author object to `chat.send` and project that author identity through stored chat history and session message events for multi-user shared sessions.

## Best Possible Solution

Define one additive gateway authorship contract covering schema validation, trust semantics, transcript storage, `chat.history` and `session.message` projection, UI typing, docs, and focused tests before implementation.

## Reproduction Assessment

Yes, as a source-level capability check. Current main and v2026.6.8 reject unknown `chat.send.author` because the schema is closed, and the inspected runtime/storage/projection paths have no first-class author metadata.

## Solution Assessment

Unclear. An optional `author` object is plausible, but maintainers should decide whether it is display metadata, operator-trusted provenance, or attested room identity before making it authoritative in history and events.
