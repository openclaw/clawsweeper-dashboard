## Summary

Keep open. Current main still lacks the requested first-class `chat.send.author` protocol field, durable `authorId`/`authorName` storage, and `chat.history`/`session.message` projection; the related open provenance PR covers only an operator-admin subset and does not supersede the broader shared-session attribution request.

## What This Changes

Add first-class per-message author attribution to `chat.send` so shared-session messages can round-trip user identity through history and events.

## Best Possible Solution

Define one additive gateway authorship contract covering validation, storage, history/events, UI typing, docs, tests, and explicit display-only versus attested identity semantics.

## Reproduction Assessment

Yes, for the implementation gap: source inspection on current main shows `chat.send` rejects an `author` field and has no durable `authorId`/`authorName` projection path. This is a new protocol capability rather than a broken existing contract.

## Solution Assessment

Unclear. An additive `author` object is plausible, but maintainers should first decide whether author values are client-declared display metadata, operator-admin provenance, or attested room identity.
