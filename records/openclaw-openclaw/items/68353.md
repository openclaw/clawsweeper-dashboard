## Summary

Keep open: current main and v2026.6.10 still do not provide first-class `chat.send.author` input, durable per-message author fields, or `chat.history`/`session.message` author projection; the related open PR is partial and conflicting, so this still needs maintainer product and security review.

## What This Changes

Request to add validated per-message author attribution to `chat.send` for multi-user shared sessions and surface it through chat history/events.

## Best Possible Solution

Define one additive Gateway authorship contract covering validation, trust semantics, transcript storage, `chat.history`/`session.message` projection, UI/client typing, docs, and focused tests.

## Reproduction Assessment

Yes. Source inspection on current main shows `ChatSendParamsSchema` rejects unknown author data, and the runtime, storage, projection, and UI send paths lack first-class author fields.

## Solution Assessment

Unclear until maintainers decide trust semantics. The optional author object is plausible, but it should not become authoritative without a display-only, operator-trusted, or attested-identity decision.
