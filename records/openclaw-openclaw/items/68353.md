## Summary

Keep open. Current main and v2026.6.6 still lack a first-class `chat.send.author` protocol field, durable author metadata on chat-send user turns, and `authorId`/`authorName` projection in history/events; the related open token/provenance work is adjacent rather than a merged replacement.

## What This Changes

The issue requests an optional `chat.send.author` object whose values persist on user turns and project as `senderLabel`, `authorId`, and `authorName` in shared-session history and events.

## Best Possible Solution

Define one additive gateway authorship contract covering schema, validation, trust semantics, transcript storage, history/event projection, UI/client typing, docs, and tests before implementation.

## Reproduction Assessment

Yes, as a source-level capability check. Current main has no accepted `chat.send.author` field, no durable author fields on chat-send user turns, and no `authorId`/`authorName` history or `session.message` projection path.

## Solution Assessment

Unclear. An optional `author` object is plausible, but maintainers should decide whether it is display metadata, operator-trusted provenance, or attested room identity before making it authoritative in visible history.
