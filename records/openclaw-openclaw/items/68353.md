## Summary

Keep open. Current main still lacks a first-class `author` field on `chat.send`, has no storage/projection path for `authorId` or `authorName`, and the related open provenance PR is a narrower operator-admin subset rather than a superseding implementation.

## What This Changes

Add an optional per-message `author` object to `chat.send` and round it through history/events as `senderLabel`, `authorId`, and `authorName`.

## Best Possible Solution

Define an additive gateway protocol contract for per-message authorship that covers validation, storage, `chat.history` and `session.message` projection, UI/client typing, docs, tests, and explicit display-only versus attested identity semantics.

## Reproduction Assessment

Yes. Source inspection is enough: `chat.send` has a closed schema with no `author`, and the runtime, transcript, history, and event projection paths have no field that can round-trip `authorId` or `authorName`.

## Solution Assessment

Unclear. An additive `author` object is plausible, but letting it override `senderLabel` should wait for maintainer agreement on whether the values are display-only, operator-provided, or attested identity.
