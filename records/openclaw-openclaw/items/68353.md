## Summary

Keep open: current main and v2026.6.10 still lack a first-class `chat.send.author` payload, durable author fields on chat-send user turns, and `authorId`/`authorName` projection; the overlapping PR #79762 is open, conflicting, and only partial.

## What This Changes

Add first-class per-message author attribution to `chat.send` and surface it through `chat.history` and `session.message` for multi-user shared sessions.

## Best Possible Solution

Define one additive gateway authorship contract covering schema, trust semantics, transcript storage, `chat.history` and `session.message` projection, UI/client typing, docs, and focused tests.

## Reproduction Assessment

Yes, source-reproducible: the closed `ChatSendParamsSchema` rejects unknown author data, and the runtime, storage, projection, and UI send paths lack author fields.

## Solution Assessment

Unclear until maintainers decide trust semantics; the optional author object is plausible, but it should not become authoritative without a display-only, operator-trusted, or attested identity decision.
