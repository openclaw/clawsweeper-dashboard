## Summary

Keep open: current main still omits allow-only official external plugin ids from the one-time release install repair, and the linked open PR is the correct vehicle to finish this issue.

## What This Changes

The issue asks `doctor --fix` to preserve and install allow-only official external plugins such as `lobster` during the v2026.5.2 migration instead of letting stale cleanup remove them.

## Best Possible Solution

Review and land a focused fix like #77193 that adds only official external catalog matches from `plugins.allow` to the release repair set before stale cleanup runs.

## Reproduction Assessment

Yes, by source inspection: an allow-only official plugin id is not collected for release install repair, then stale cleanup can classify and remove that missing allow entry when discovery is healthy. I did not run a live doctor reproduction because this review is read-only.

## Solution Assessment

Yes, the linked PR direction is the narrowest maintainable fix: include only official external catalog entries from `plugins.allow` in the one-time release migration, while preserving normal behavior that arbitrary allow-only ids are not auto-installed.
