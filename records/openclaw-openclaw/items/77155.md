## Summary

Keep open: current main still omits allow-only plugin ids from the one-time release repair, and open PR #77193 is the linked implementation candidate, so closing this issue before a fix lands would be premature.

## What This Changes

Running `openclaw doctor --fix` after the v2026.5.2 migration can remove official external plugins that appear only in `plugins.allow`, such as `lobster`, instead of installing and preserving them.

## Best Possible Solution

Land a narrow fix that adds only official external catalog matches from `plugins.allow` to the release repair set before stale cleanup runs, while keeping arbitrary allow-only ids excluded.

## Reproduction Assessment

Yes, by source inspection for the allow-only case: current main excludes `plugins.allow` from the release repair collector while stale cleanup removes missing allow and entry refs. I did not run doctor because this was a read-only review.

## Solution Assessment

Yes, the linked PR direction is the narrowest maintainable fix: include allow-only ids only when they resolve to official external catalog entries, not arbitrary allowlist values.
