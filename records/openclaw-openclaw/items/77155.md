## Summary

Keep open: current main still excludes allow-only official external plugin ids from the release install repair, so stale cleanup can remove lobster from restrictive allowlists before it is installed; material `plugins.entries` are now protected, but the allow-only path remains actionable.

## What This Changes

`openclaw doctor --fix` should preserve and install official externalized plugins such as lobster when an older config only lists them in `plugins.allow`.

## Best Possible Solution

Add a release-migration-only collector for `plugins.allow` ids that resolve in the official external catalog, while preserving ordinary allow-only exclusion in normal missing-install repair.

## Reproduction Assessment

Yes, by source inspection: a config whose only lobster reference is `plugins.allow` will not feed `lobster` into the release install repair, and stale cleanup can then remove the missing allowlist id. I did not live-run doctor because this was a read-only review.

## Solution Assessment

Yes, the narrowest maintainable fix is to include only official external catalog matches from `plugins.allow` in the one-time release migration set, without changing the normal rule that arbitrary allowed-but-unused plugins are not installed.
