## Summary

Keep open: current main still omits allow-only official external plugin ids from the 2026.5.2 release install repair, and the linked open PR is the right place to finish the fix before this issue closes.

## What This Changes

The issue reports that `doctor --fix` can remove allow-only official external plugins during the 2026.5.2 migration instead of installing and preserving them.

## Best Possible Solution

Land #77193 or an equivalent narrow fix that adds official external `plugins.allow` ids to the release repair set before stale cleanup, while keeping arbitrary unofficial allow-only ids excluded.

## Reproduction Assessment

Yes, by source inspection: current main still excludes allow-only ids from the release repair collector while stale cleanup removes missing allow refs. I did not live-run doctor because this was a read-only review.

## Solution Assessment

Yes, the linked direction is the narrow maintainable fix: include only official external catalog matches from `plugins.allow` in the release repair set and keep normal arbitrary allow-only ids out of missing-install repair.
