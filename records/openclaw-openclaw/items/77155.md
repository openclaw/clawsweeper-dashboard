## Summary

Keep open: current main still excludes allow-only plugin ids from the release repair set, and stale cleanup can remove those refs afterward. PR #77193 is an open closing PR for the same bug, so this issue should wait for that PR to land or be replaced.

## What This Changes

The issue asks `doctor --fix` to preserve and install allow-only official external plugins such as `lobster` during the v2026.5.2 configured-plugin migration.

## Best Possible Solution

Review and land PR #77193 or an equivalent narrow patch that adds official external catalog matches from `plugins.allow` to the release repair set while keeping arbitrary allow-only ids excluded.

## Reproduction Assessment

Yes, there is a high-confidence source reproduction: the release collector has no `plugins.allow` pass, while stale repair removes missing allow refs when discovery is healthy. I did not live-run `doctor --fix` because this review is read-only.

## Solution Assessment

Yes, the linked approach is the narrowest maintainable fix: include allow-only ids only when the official external catalog resolves them, avoiding new installs for arbitrary allowlist entries.
