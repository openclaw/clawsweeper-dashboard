## Summary

Keep open: current main still excludes official external plugins that are present only in `plugins.allow` from the v2026.5.2 release install repair, and stale cleanup can remove those missing allow/entry references afterward. The existing ClawSweeper comment correctly identified the remaining allow-only gap, while current main appears to cover material `plugins.entries` usage.

## What This Changes

Preserve and install official external plugins such as `lobster` when they are listed only in `plugins.allow` during the one-time v2026.5.2 doctor migration.

## Best Possible Solution

Teach the one-time release repair to include `plugins.allow` ids only when they resolve to official external catalog entries, while preserving the normal rule that arbitrary allow-only ids are not treated as install usage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `plugins.allow: ["lobster"]` with no installed record is not included in the release install set, then stale cleanup can remove the missing allow reference when discovery is healthy.

## Solution Assessment

Yes. The narrowest maintainable fix is an official-external-catalog exception in the release migration collector, not a broad change to normal allowlist semantics.
