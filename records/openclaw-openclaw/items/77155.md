## Summary

Keep open: current main still excludes allow-only official external plugin ids from the release repair set, and open PR #77193 is the linked implementation candidate.

## What This Changes

The issue asks `doctor --fix` to preserve and install official external plugins such as `lobster` when they are listed only in `plugins.allow` during the 2026.5.2 migration.

## Best Possible Solution

Review and land PR #77193, or an equivalent narrow fix, so only official external catalog matches from `plugins.allow` join the release repair set while arbitrary allow-only ids remain excluded.

## Reproduction Assessment

Yes. Current-main source shows `collectReleaseConfiguredPluginIds` does not read `plugins.allow`, the tests assert allow-only ids are excluded, and stale cleanup removes missing allow ids when plugin discovery is healthy.

## Solution Assessment

Yes. The narrow maintainable solution is to include only official external catalog matches from `plugins.allow` in the one-time release repair path, preserving the existing behavior for arbitrary allow-only entries.
