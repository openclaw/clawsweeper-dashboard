## Summary

Keep open: current main still omits allow-only official external plugin ids from the one-time release repair, and the linked fix PR is open rather than merged.

## What This Changes

The issue asks `doctor --fix` to preserve allow-only official external plugins such as `lobster` during the 2026.5.2 plugin externalization repair.

## Best Possible Solution

Land or otherwise supersede PR #77193 with regression coverage that official external allow-only ids are included in the release repair while unofficial allow-only ids remain excluded.

## Reproduction Assessment

Yes, by source inspection against current main: the release collector never reads `plugins.allow`, normal missing-install repair excludes allow-only ids, and stale cleanup removes missing allow refs. I did not live-run `doctor --fix` because this review is read-only.

## Solution Assessment

Yes, the narrow fix is to add only official external catalog matches from `plugins.allow` to the one-time release repair set. That preserves the existing rule that arbitrary allow-only ids are not installed by normal repair.
