## Summary

Keep open for maintainer review. Current main still lacks a command-level hard timeout for local and embedded fallback agent runs, while this PR adds a focused implementation with regression coverage and supplied terminal proof; I found no blocking correctness or security finding, but the compatibility and availability tradeoff is maintainer-owned.

## What This Changes

The PR adds a CLI hard timeout for finite local `openclaw agent --local` and embedded fallback runs, documents the exit-124 behavior, and adds focused CLI/gateway timeout tests.

## Best Possible Solution

Land this as the no-completion hard-timeout fix only if maintainers accept finite local-agent timeouts as a CLI process boundary, while keeping `--timeout 0` as the documented escape hatch and reviewing the companion post-success cleanup PR separately.

## Reproduction Assessment

Yes, source-reproducible: current main directly awaits local and embedded fallback `agentCommand` calls without a command-level timeout race, so a wedged embedded run can keep the CLI alive. I did not run a live reproduction in this read-only review, but the PR body supplies terminal before/after proof.

## Solution Assessment

Yes, conditionally: the patch is a narrow fix on the CLI local/embedded boundary, preserves signal precedence and `--timeout 0`, and covers startup plus fallback paths. The unresolved question is product policy, not a mechanical repair: whether maintainers accept hard exit as the default for finite timeouts.
