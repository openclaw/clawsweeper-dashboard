## What This Changes

This issue reports that `openclaw tui --session ... --deliver` loads Telegram session history but does not live-tail inbound messages or agent responses until manual refresh or relaunch.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/96252 for the narrow TUI `session.message` consumer, while keeping broader bidirectional mirror policy in https://github.com/openclaw/openclaw/issues/40678.

## Solution Assessment

Yes for the narrow bug: consuming the existing Gateway `session.message` contract in TUI is the right layer. The broader bidirectional mirror idea is a separate product/security decision tracked outside this narrow fix.
