## What This Changes

The issue asks for `openclaw tui --session ... --deliver` to show inbound Telegram messages and agent replies in real time without relaunching or manually reloading history.

## Best Possible Solution

Land or close the viable same-root TUI fix, preferably by consuming existing Gateway `session.message` transcript events without changing channel delivery semantics; keep broader bidirectional mirroring as separate product work.

## Solution Assessment
