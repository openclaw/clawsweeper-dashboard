## Summary

## What This Changes

The issue requests real-time TUI live-tailing for `openclaw tui --session ... --deliver`, including inbound Telegram messages and agent replies without manual refresh.

## Best Possible Solution

Keep the remaining implementation and review discussion on https://github.com/openclaw/openclaw/issues/38829 and https://github.com/openclaw/openclaw/pull/73402, and treat bidirectional channel mirroring as separate product work.

## Reproduction Assessment

Yes. Source inspection on current main shows the TUI loads history but has no active-session transcript subscription or `session.message` handler, so cross-surface messages can remain invisible until history reloads.

## Solution Assessment

Yes. Closing this duplicate while keeping the canonical issue and PR open is the cleanest maintainer path; the implementation should consume the existing scoped `session.message` contract rather than broaden `chat` events.
