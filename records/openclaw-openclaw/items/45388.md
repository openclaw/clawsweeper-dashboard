## Summary

Keep open: current main and the latest release still expose the gateway live transcript API, but the TUI does not subscribe to it or dispatch `session.message` events, so the reported live-tail gap remains source-reproducible. The issue is already tied to open fix work, especially https://github.com/openclaw/openclaw/pull/73402 plus an external closing PR, so it should not be closed before a fix lands.

## What This Changes

The issue requests that `openclaw tui --session ... --deliver` live-tail inbound channel messages and agent replies instead of requiring manual refresh or relaunch.

## Best Possible Solution

Review, update, and land or replace https://github.com/openclaw/openclaw/pull/73402 as the scoped same-session TUI transcript subscription fix; keep broader bidirectional mirror behavior on https://github.com/openclaw/openclaw/issues/40678.

## Reproduction Assessment

Yes. Source inspection on current main shows the TUI loads history and handles gateway chat events, but has no `sessions.messages.subscribe` client method or `session.message` dispatch/handler; the reporter's Ctrl+T/manual-refresh observation matches that path.

## Solution Assessment

Yes for the central bug. The narrow maintainable fix is to wire TUI session mode to the existing `sessions.messages.subscribe` / `session.message` gateway contract, while bidirectional channel mirroring is separate product work.
