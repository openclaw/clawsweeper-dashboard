## Label Justifications

- `P2`: The report describes a bounded TUI live-update bug that affects session visibility without making the core runtime unusable.
- `impact:session-state`: The issue is about the TUI not keeping its displayed session transcript state in sync with live session activity.

## Summary

Keep open: current main and v2026.6.6 still have the Gateway transcript subscription contract, but the bundled TUI does not subscribe to or route live `session.message` updates. The remaining work is already represented by open implementation candidates, so this issue should stay open until one lands or is replaced.

## What This Changes

The issue asks for `openclaw tui --session ... --deliver` to live-tail inbound Telegram messages and agent replies instead of only refreshing history on relaunch or `Ctrl+T`.

## Best Possible Solution

Land or replace the narrow TUI active-session transcript subscription fix, with session filtering and dedupe, while keeping cross-channel bidirectional mirroring in its separate product/security track.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.6 expose Gateway live transcript subscriptions, but the TUI has no subscription method, no client request path, and no `session.message` event routing. I did not run a live Telegram/TUI reproduction in this read-only review.

## Solution Assessment

Yes for the narrow bug: consuming the existing per-session Gateway transcript subscription in the TUI is the owner-boundary fix. The reporter's bidirectional mirroring note belongs to the broader cross-channel visibility issue, not this bug fix.
