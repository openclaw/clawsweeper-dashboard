## Summary

Keep open: this member-authored PR is still the active fix candidate for the killed-subagent task-row bug, and the runtime patch looks coherent, but the docs still need to reflect the new maintenance-inferred `lost` to same-run `succeeded` exception before merge.

## What This Changes

The branch defers killed subagent task-row finalization to task maintenance, marks killed-run delivery as `not_applicable`, adds terminal subagent-run lookup, allows maintenance-inferred `lost` to upgrade to `succeeded`, and adds regression coverage.

## Best Possible Solution

Land the maintenance-owned repair after documenting the `lost` to `succeeded` exception and maintainer acceptance of the bounded session-state tradeoff.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes at source level: current main persists killed subagent terminal state without task-row finalization, while maintenance lacks terminal-run reconciliation; I did not run a live gateway race in this read-only review.

## Solution Assessment

Is this the best way to solve the issue? Yes, with the docs update and maintainer acceptance: maintenance-owned finalization avoids writing `failed` in the kill path before a concurrent successful lifecycle completion can arrive.
