## Summary

Keep open: the branch is directionally fixing the killed-subagent task finalization race, but the new lost-to-succeeded transition can leave lost-derived error text and lost cleanup retention on a task that now says it succeeded.

## What This Changes

This PR defers killed subagent task-row finalization to task maintenance, marks killed-run delivery as not_applicable, adds subagent-run lookup during maintenance, allows lost-to-succeeded run-scoped updates, and adds regression tests.

## Best Possible Solution

Make the lost-to-succeeded upgrade fully replace the lost projection, including clearing lost-only error state and resetting or recomputing cleanupAfter for succeeded retention.

## Reproduction Assessment

Yes. Source inspection gives a focused reproduction: mark a subagent task lost with an error, then call the new allowed run-scoped succeeded update and inspect that error and cleanupAfter remain from the lost projection.

## Solution Assessment

No. Deferring killed-run status finalization to maintenance is a good owner-boundary fix, but the late-success repair must also normalize the terminal fields it is overriding.
