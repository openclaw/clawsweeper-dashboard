## Summary

This PR should stay open because it improves gateway scheduled maintenance but still misses the documented standalone operator repair path: `openclaw tasks maintenance --apply` does not run with an authoritative runtime and therefore will not use the new terminal subagent lookup.

## What This Changes

The PR defers killed subagent task-row finalization to maintenance, marks killed-run delivery as not_applicable, allows lost-to-succeeded run-scoped updates, and adds focused regression tests.

## Best Possible Solution

Keep the kill-vs-complete deferral, but make task maintenance read authoritative persisted subagent-run terminal state for the standalone repair path as well as gateway sweeps.

## Reproduction Assessment

Yes from source: current main can persist a killed subagent run with endedAt while the task row remains running, and maintenance still sees a backing session entry. I did not run tests because this review is read-only.

## Solution Assessment

No. The PR is a good partial gateway-sweep fix, but it is not the best complete fix until standalone `openclaw tasks maintenance --apply` can use durable terminal subagent-run state too.
