## What This Changes

The PR changes killed subagent task reconciliation so maintenance reads terminal subagent run state, marks matching subagent/CLI task rows lost, and allows a late same-run completion to upgrade maintenance-inferred lost to succeeded.

## Best Possible Solution

Land the maintenance-owned repair after documenting the lost-to-succeeded exception and maintainer acceptance of the bounded session-state tradeoff.

## Reproduction Assessment

Yes from source inspection: current main persists killed subagent terminal state without task-row finalization, and maintenance lacks a terminal subagent-run lookup. I did not run a live gateway reproduction in this read-only review.

## Solution Assessment

Yes, with a docs update and maintainer acceptance: maintenance-owned finalization avoids writing failed in the kill path before a concurrent successful lifecycle completion can arrive. A direct kill-path failTaskRunByRunId repair is simpler but preserves the race described by the PR.
