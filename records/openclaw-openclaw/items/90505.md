## What This Changes

The PR changes subagent kill handling and task maintenance so killed subagent task rows are finalized by maintenance while allowing late same-run completion to upgrade maintenance-inferred `lost` state to `succeeded`.

## Best Possible Solution

Land the maintenance-owned repair after documenting the `lost` to `succeeded` exception and maintainer acceptance of the bounded session-state tradeoff.

## Reproduction Assessment

Yes at source level: current main persists killed subagent terminal state without task-row finalization, while this PR adds maintenance reconciliation for the exact terminal-run shape. I did not run a live gateway race in this read-only review.

## Solution Assessment

Yes, with the docs update and maintainer acceptance: maintenance-owned finalization avoids writing `failed` in the kill path before a concurrent successful lifecycle completion can arrive. A direct kill-path failure write is simpler but keeps the race described by the PR.
