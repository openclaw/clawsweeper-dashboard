## Summary

Keep open: the race-avoidance shape is useful, but the latest diff still only lets authoritative in-process maintenance see terminal subagent state, so the operator maintenance path described in the linked report remains incomplete.

## What This Changes

The PR defers killed subagent task-row finalization to task maintenance, marks killed-run delivery as not_applicable, allows lost-to-succeeded run-scoped updates, and adds focused regression tests.

## Best Possible Solution

Use the durable merged subagent-run state as maintenance's terminal-run signal, preserve the lost -> succeeded race guard, and add CLI maintenance coverage before merge.

## Reproduction Assessment

Yes at source level: current main persists terminal killed subagent run state without updating the mirrored task row, and maintenance can retain that row when a stale child-session entry remains. I did not run a live gateway reproduction in this read-only review.

## Solution Assessment

No, not yet. The race fix is the right direction, but the maintenance lookup should use durable/restored subagent run state so the reported CLI maintenance path is covered.
