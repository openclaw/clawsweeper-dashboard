## Summary

Keep open: this PR fixes the prior lost-to-succeeded blocker and is aimed at the right task/subagent lifecycle surface, but the new maintenance lookup still depends on process-local subagent state. That leaves the operator CLI maintenance path and first post-restart sweeps able to miss persisted terminal subagent runs, which is part of the linked bug report's cleanup path.

## What This Changes

The PR marks killed subagent task delivery as not_applicable, lets maintenance infer terminal task status from subagent run endedAt, and allows a late successful lifecycle COMPLETE to override maintenance-inferred lost status.

## Best Possible Solution

Back the maintenance endedAt lookup with durable/restored subagent run state and cover the CLI maintenance plus post-restart paths, while keeping the lost -> succeeded race guard.

## Reproduction Assessment

Yes. Current main source shows killed subagent runs persist terminal endedAt while task maintenance lacks a subagent-run lookup and retains subagent tasks when the stale session entry remains.

## Solution Assessment

No, not yet. The parent-row race fix is the right shape, but the maintenance lookup should use durable/restored subagent run state so the reported CLI maintenance and restart paths are actually covered.
