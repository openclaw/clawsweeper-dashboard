## Summary

Keep open: this is a focused member-authored bug-fix PR for the linked stuck subagent task-row issue, but it still needs maintainer acceptance of the session-state reconciliation change and a small docs alignment before merge.

## What This Changes

The branch changes killed subagent task reconciliation so maintenance reads terminal subagent run state, marks matching subagent/CLI task rows lost, and lets a late same-run completion upgrade maintenance-inferred lost to succeeded.

## Best Possible Solution

Land the maintenance-owned repair after documenting the lost-to-succeeded exception and maintainer acceptance of the bounded session-state tradeoff.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes at source level: current main persists killed subagent terminal state without task-row finalization, and maintenance lacks terminal-run reconciliation. I did not run a live gateway kill race in this read-only review.

## Solution Assessment

Is this the best way to solve the issue? Yes, with the docs update and maintainer acceptance: maintenance-owned finalization avoids writing failed in the kill path before a concurrent successful lifecycle completion can arrive. A direct kill-path failTaskRunByRunId repair is simpler but preserves the race described by the PR.
