## What This Changes

The branch changes embedded-agent block chunking, block-reply delivery trimming, coalescer joining, and regression tests to carry paragraph separators across non-final streamed block deliveries.

## Best Possible Solution

Land one consolidated shared chunker/subscriber/coalescer design that preserves paragraph boundaries for both in-budget and maxChars split cases without over-limit chunks, backed by real delivered-behavior proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes, source-level: current main consumes paragraph separators at the chunk boundary, and PR head still reproduces the same loss for an exact/near-maxChars paragraph boundary such as `abcdefghij\n\nRest` with `maxChars: 10`.

## Solution Assessment

Is this the best way to solve the issue? No. The shared chunker/subscriber/coalescer layer is the right place, but the text-only implementation is partial until overflow paragraph boundaries are represented without exceeding maxChars or corrupting coalesced seams.
