## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve exact-boundary paragraph separators within maxChars:** `src/agents/embedded-agent-block-chunker.ts:295-296`
  - body: When a paragraph break starts exactly at `maxChars`, `withSeparator.length <= maxChars` is false, so the raw chunk is emitted without `\n\n` and the following newline run is skipped. For input like `abcdefghij\n\nRest` with `maxChars: 10`, reconstruction still becomes `abcdefghijRest`; split earlier or carry the separator in a bounded follow-up chunk.
  - confidence: 0.93
- **[P2] Do not shorten carried paragraph separators with newline joiners:** `src/auto-reply/reply/block-reply-coalescer.ts:41`
  - body: For `breakPreference: "newline"`, the coalescer joiner is `\n`. If a chunk already carries `\n\n`, this helper strips it and rejoins with one newline, so two chunks coalesced in one flush lose the paragraph boundary the chunker preserved.
  - confidence: 0.86
