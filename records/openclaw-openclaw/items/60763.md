## Summary

Keep this PR open. Current main still does not expose `expiresAtMs` or `bindSession` on the Discord component spec/parser/builder surface, so the feature is not obsolete, but the proposed patch should not merge as-is because it changes Discord interaction authorization and registry lifetime without a safe policy.

## What This Changes

The PR adds optional Discord component `expiresAtMs` and `bindSession` fields to the component spec, parser, message builder, message-tool schema, and component tests.

## Best Possible Solution

Keep the PR as an implementation candidate, but update it for the current split files and require explicit safe semantics: bounded expiry, documented unbound routing, authorization for shared components, and focused Discord component tests/docs before merge.

## Reproduction Assessment

Yes. The feature gap is reproducible by source inspection on current main: the spec, parser, builder, and current tool surface lack both fields. The security findings are also traceable through the dispatch fallback and permissive missing-allowlist path; no runtime test was run because this was a read-only review.

## Solution Assessment

No. The direction is plausible for cron-owned Discord components, but the current patch is not the best mergeable form until it enforces a bounded expiry policy, protects or explicitly models unbound shared interactions, and rebases onto the current file layout.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Require an authorization principal for unbound components:** `extensions/discord/src/components.ts:788-790`
  - body: When `bindSession=false` clears the stored route context, any channel-authorized user can use a component unless `allowedUsers` is set, because the current interaction path allows missing component allowlists. Please require an allowlist/originating user or make shared-channel use an explicit guarded mode before dispatching callbacks through the channel route.
  - confidence: 0.88
- **[P2] Bound caller-provided component expiry:** `extensions/discord/src/components.ts:802`
  - body: `expiresAtMs` is accepted as any finite timestamp and then stored as `expiresAt`, bypassing the registry's 30-minute default TTL for far-future values. Clamp or reject values outside the intended retention window before registering entries and modals.
  - confidence: 0.84

## Security Review

Status: needs_attention

Concerns:

  - body: Clearing session, agent, and account route overrides lets callbacks run through the channel-resolved route while `ensureComponentUserAllowed` allows missing component allowlists. This can make privileged or user-specific buttons/forms usable by other authorized channel members unless the PR requires an origin user, allowlist, or explicit shared-mode guard.
