## Summary

Keep open: current main and the latest release still pass an empty Codex model-input history array into `before_prompt_build`, while the hook contract and in-tree memory plugins expect loaded session messages. The open closing PR at https://github.com/openclaw/openclaw/pull/89004 owns the narrow repair, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Codex app-server should pass loaded session messages to `before_prompt_build` without re-injecting mirrored history into Codex model input.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/89004 or an equivalent hook/model-input split, then close this issue as implemented.

## Reproduction Assessment

Yes. Source inspection on current main shows session history is loaded and then replaced with `[]` for `before_prompt_build`, and the adjacent test asserts that empty payload after adding a prior assistant message.

## Solution Assessment

Yes. The narrowest maintainable fix is to split hook-observation messages from Codex model-input history; the open linked PR implements that shape directly.
