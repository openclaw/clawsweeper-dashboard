## Summary

Keep open: current main and the latest release still route Codex `before_prompt_build.messages` through the empty Codex model-input history array. The remaining fix is already owned by the clean open closing PR at https://github.com/openclaw/openclaw/pull/89004, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Codex app-server should pass loaded session messages to the `before_prompt_build` hook while keeping mirrored history out of Codex model input.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/89004 or an equivalent narrow split that restores hook-observation history without reintroducing mirrored history into Codex model input.

## Reproduction Assessment

Yes. Source inspection on current main shows loaded `historyMessages` are replaced by an empty `codexModelInputHistoryMessages` array for `before_prompt_build`, and the adjacent test asserts that empty hook payload after a prior assistant message is appended.

## Solution Assessment

Yes. The best fix is the linked hook/model-input split: pass cloned loaded session messages to `before_prompt_build`, while keeping Codex `turn/start`, diagnostics, and `llm_input.historyMessages` on the empty native-thread history path.
