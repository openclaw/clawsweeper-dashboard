## Summary

Keep this issue open: current main and v2026.6.6 still pass the empty Codex model-input history array into `before_prompt_build`, while an open linked PR already owns the narrow fix path.

## What This Changes

The issue requests restoring Codex `before_prompt_build.messages` so plugins can observe loaded session history without re-injecting mirrored OpenClaw history into Codex model input.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/89004 or an equivalent narrow hook/model-input split, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes. Source inspection shows current main loads session history and then passes the empty Codex model-input array to `before_prompt_build`; a user comment also reports the same behavior on v2026.6.6 with `active-memory`.

## Solution Assessment

Yes. The best fix is to separate hook-observation messages from Codex native model-input history; the open linked PR implements that boundary without changing the Codex `turn/start` history path.
