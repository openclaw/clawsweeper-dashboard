## Summary

Keep open: current main and the latest release still pass the empty Codex model-input history array into `before_prompt_build.messages`, while the hook contract and in-tree memory plugins expect loaded session messages. The open, mergeable fix PR at https://github.com/openclaw/openclaw/pull/89004 uses closing syntax and owns the narrow repair, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports a Codex app-server regression where `before_prompt_build` receives `messages: []` after session history is loaded, breaking history-sensitive plugin hooks.

## Best Possible Solution

Land the focused hook/model-input split in the open fix PR or an equivalent patch, then close this issue as implemented.

## Reproduction Assessment

Yes. Source inspection shows current main loads session history and then passes `[]` into `before_prompt_build`, the adjacent test asserts that behavior, and a June 14 comment confirms the failure on v2026.6.6 with active-memory.

## Solution Assessment

Yes. The best fix is to split hook-observation messages from Codex model-input history, which is exactly the direction of the open fix PR.
