## Summary

Keep open: current main and v2026.6.6 still route Codex `before_prompt_build.messages` through the empty model-input history array, while the hook contract and sibling runtime paths expect loaded session messages. The open fix PR at https://github.com/openclaw/openclaw/pull/89004 uses closing syntax and has sufficient proof, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Restore the Codex app-server `before_prompt_build` hook payload so plugins observe loaded session history while Codex model input remains history-free.

## Best Possible Solution

Land the focused hook/model-input split in the open fix PR or an equivalent patch, then close this issue as implemented.

## Reproduction Assessment

Yes. Source inspection shows current main loads session history and then passes `[]` into `before_prompt_build`, and the adjacent test asserts that behavior; a June 14 comment also confirms the failure on v2026.6.6 with active-memory.

## Solution Assessment

Yes. The best fix is to split hook-observation messages from Codex model-input history, which is exactly the direction of the open fix PR.
