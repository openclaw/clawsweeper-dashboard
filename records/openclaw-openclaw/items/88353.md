## Label Justifications

- `P2`: This is a normal-priority Codex plugin hook regression with a bounded affected surface and an open fix PR.
- `impact:session-state`: The issue is about resumed Codex session history being hidden from `before_prompt_build` consumers that depend on prior messages.

## Summary

Keep open: current main and v2026.6.6 still pass the empty Codex model-input history array into `before_prompt_build`, even though the hook contract and sibling runner treat `messages` as loaded session history. The remaining fix is already represented by open PR https://github.com/openclaw/openclaw/pull/89004, so this issue should wait for that PR to land or for maintainers to choose a different hook contract.

## What This Changes

Restore the Codex app-server `before_prompt_build.messages` hook payload so history-sensitive plugins can observe loaded session messages while Codex native model input remains history-free.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/89004’s hook/model-input split, or explicitly change the hook docs, types, and plugin expectations if maintainers reject history-bearing Codex hook payloads.

## Reproduction Assessment

Yes. Source inspection shows current main loads `historyMessages` but passes `codexModelInputHistoryMessages = []` into the hook, and the adjacent test appends a prior assistant message then asserts the hook receives `[]`; v2026.6.6 contains the same path.

## Solution Assessment

Yes. Splitting hook-observation messages from Codex model-input history is the narrowest fix because it restores the documented plugin hook contract without re-injecting mirrored OpenClaw history into Codex `turn/start` input.
