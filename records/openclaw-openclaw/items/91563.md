## Summary

Keep open: current main and the latest release still let memory-core deep promotion satisfy minUniqueQueries with recall-day diversity, and no merged PR owns the exact unique-query contract. The code path is source-reproducible, but current tests intentionally preserve dreaming-only promotion, so maintainers should settle the policy before an automated fix rewrites it.

## What This Changes

The issue asks memory-core deep promotion to enforce minUniqueQueries with distinct recall query evidence instead of recall-day diversity.

## Best Possible Solution

Make the promotion contract explicit by keeping recallDays as consolidation scoring only or documenting it as intentional, then align rank/apply, docs, and regression tests.

## Reproduction Assessment

Yes for the narrow gate behavior: current main stores query hashes and recall days separately, then lets recallDays length satisfy minUniqueQueries in rank and apply. I did not replay the reporter's full 9-agent fleet.

## Solution Assessment

Unclear. The suggested code change is mechanically narrow, but existing tests intentionally allow dreaming-only daily signals to clear default gates, so maintainers should decide the promotion policy first.
