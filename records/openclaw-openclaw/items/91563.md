## Summary

## What This Changes

The issue asks memory-core deep promotion to make `minUniqueQueries` count genuine distinct query evidence instead of using recall-day diversity as a fallback gate signal.

## Best Possible Solution

Maintainers should define whether `minUniqueQueries` means organic distinct query hashes or broader cross-day context diversity, then align rank/apply gates, docs, and regression tests around that contract.

## Reproduction Assessment

Yes, at source level: current main stores query hashes and recall days separately, then lets recall-day length satisfy the `minUniqueQueries` gate in both rank and apply. I did not replay the full fleet workload from the report.

## Solution Assessment

Unclear. The suggested code change is narrow, but it conflicts with current comments and tests that intentionally let repeated dreaming-only daily signals clear default promotion gates, so maintainers should settle the policy first.
