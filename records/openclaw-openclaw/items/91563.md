## Summary

Keep open: current main still allows memory-core deep promotion to satisfy the minUniqueQueries gate from recall-day diversity, and that conflicts with the documented distinct-query wording. The behavior is source-reproducible, but current tests intentionally preserve a dreaming-only promotion path, so maintainers should decide the desired policy before an automated fix rewrites it.

## What This Changes

The issue reports that memory-core dreaming deep promotion can satisfy the minUniqueQueries gate by counting distinct recall days rather than distinct recall query strings.

## Best Possible Solution

Make the minUniqueQueries contract explicit: either keep day diversity as an intentional gate substitute and update docs, or change rank/apply so recallDays only affects consolidation scoring while query hashes satisfy the query-diversity gate.

## Reproduction Assessment

Yes for the narrow gate behavior: current main records daily dreaming signals and the rank/apply gates allow recallDays length to satisfy minUniqueQueries. A full multi-agent fleet reproduction was not attempted in this read-only review.

## Solution Assessment

Unclear. The suggested code change is narrow, but existing tests intentionally preserve a dreaming-only daily-signal promotion path, so the best fix depends on maintainer policy for dreaming-only promotion.
