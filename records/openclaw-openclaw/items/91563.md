## Summary

Keep open: current main and v2026.6.6 still let memory-core deep promotion satisfy `minUniqueQueries` with recall-day diversity, and no merged or open PR owns this exact query-diversity contract. The source path is reproducible, but current tests intentionally preserve dreaming-only promotion, so maintainers need to settle the policy before an automated fix rewrites it.

## What This Changes

The issue asks memory-core deep promotion to stop counting recall-day diversity as satisfying `minUniqueQueries` and keep day diversity as ranking/consolidation signal only.

## Best Possible Solution

Make the promotion contract explicit, then align rank/apply, docs, and tests so `minUniqueQueries` either means real distinct query hashes or is deliberately renamed/documented as broader context diversity.

## Reproduction Assessment

Yes for the narrow gate behavior: current main rank and apply both let `recallDays.length` satisfy `minUniqueQueries`, and daily dreaming signals can feed that store. I did not replay the reporter's full 9-agent fleet.

## Solution Assessment

Unclear. The suggested code change is mechanically narrow, but current tests intentionally allow dreaming-only daily signals to clear default gates, so maintainers should decide the promotion policy first.
