## Summary

Keep open. Current main still turns the no-provider media-understanding path into missing text and then emits the generic path-based `No transcript/description returned` errors; the issue also has three open, unmerged PRs with closing syntax, so it should remain open until a canonical fix lands.

## What This Changes

The issue asks for `infer audio transcribe` and `infer image describe` to report missing media-provider configuration instead of generic empty transcript/description errors when no provider is configured.

## Best Possible Solution

Land one canonical fix that preserves the existing empty-result errors when a provider actually runs, but carries missing-provider/no-attempt state far enough for `infer audio transcribe`, `infer image describe`, and likely `infer image describe-many` to emit actionable configuration errors with regression tests and a user-facing changelog entry.

## Reproduction Assessment

Yes. The issue body gives deterministic CLI steps with no configured media provider, and current source verifies the path: no resolved providers return a skipped/no-output decision, the runtime collapses that to undefined text, and the CLI throws the generic path-based error.

## Solution Assessment

Yes for the solution direction. The narrow maintainable fix is to distinguish missing-provider/no-attempt from provider-ran-but-empty-output at the media runtime or CLI boundary; choosing among the linked PR implementations is maintainer review work.
