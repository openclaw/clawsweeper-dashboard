## What This Changes

The issue requests clearer `infer image describe`, `infer image describe-many`, and `infer audio transcribe` errors when no media-understanding provider is configured.

## Best Possible Solution

Keep the current main implementation and let the already-superseded PRs be closed or redirected to this fix rather than adding another competing patch.

Yes. The issue body gives deterministic no-provider CLI steps, and current tests now model that skipped/no-attempt decision for image describe, image describe-many, and audio transcribe.

## Solution Assessment

Yes. The current implementation is the narrow maintainable fix because it preserves the existing provider-ran-empty-output errors while carrying missing-provider state far enough for the CLI to emit actionable configuration guidance.
