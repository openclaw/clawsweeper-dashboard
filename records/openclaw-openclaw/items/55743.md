## Summary

Keep open. Current main still renders assistant shell snippets as sanitized markdown code blocks with a Copy-only button, while the existing approval UI resolves runtime-created approval records; the request remains a security-sensitive new Control UI feature rather than an already-implemented bug fix.

## What This Changes

The issue requests Control UI command cards with Copy and explicit Approve and Run actions for assistant-suggested terminal commands.

## Best Possible Solution

Add command cards only from explicit structured command metadata, bind exact command/argv, cwd, host/node, session, requester, and approval identity to the existing exec approval/systemRunPlan machinery, refuse arbitrary markdown execution, stream approved output back into chat, and document the security model.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The current-main gap is source-verifiable: assistant shell snippets render as Copy-only markdown, while approval controls resolve queued runtime approval records.

## Solution Assessment

Unclear until maintainers define the product and security contract. The safest implementation direction is structured command metadata that reuses existing approval and node systemRunPlan binding, not execution of arbitrary markdown code blocks.
