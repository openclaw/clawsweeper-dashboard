## Summary

Keep open: current main and v2026.6.8 support copy-only assistant code blocks and existing event-driven exec approval modals, but they still do not provide a structured Control UI command proposal card with an Approve and Run action for assistant-suggested terminal commands.

## What This Changes

Request a Control UI command card that can copy or explicitly approve and run assistant-suggested shell commands with command text, host/cwd, and risk metadata.

## Best Possible Solution

Design a structured command-proposal path that reuses the existing exec approval and systemRunPlan contract, rendering run actions only for explicit command proposals with exact command, cwd, host, session, and risk metadata.

## Reproduction Assessment

Yes for source behavior: current main shows assistant code blocks only support copy, while command execution approvals are driven by explicit gateway approval events. There is no current-main or latest-release bridge from suggested shell text to an approved run action.

## Solution Assessment

Unclear until maintainers choose the product and security boundary. The safer solution is a structured proposal-to-approval contract, not a direct run button attached to arbitrary rendered markdown.
