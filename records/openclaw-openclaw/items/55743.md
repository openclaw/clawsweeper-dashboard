## Summary

Keep open: current main and v2026.6.10 still render assistant shell snippets as copy-only markdown and only show approve/run controls for Gateway-created exec approval events, so the requested Control UI command proposal card remains unimplemented and needs product/security design.

## What This Changes

Request a Control UI command card for assistant-suggested shell commands with Copy, Approve and Run, host/cwd/risk metadata, and explicit approval safeguards.

## Best Possible Solution

Design a structured command-proposal contract that reuses existing exec approval and `systemRunPlan` safeguards rather than adding a run button to arbitrary markdown code blocks.

## Reproduction Assessment

Yes. Source inspection shows assistant code blocks render only `.code-block-copy`, while exec approval actions are only created from `exec.approval.requested` Gateway events.

## Solution Assessment

Unclear as requested until maintainers choose the product/security boundary; a structured proposal-to-approval path is safer than inferring executable commands from arbitrary markdown.
