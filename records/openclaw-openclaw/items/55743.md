## Summary

Keep open: current main and the latest release still provide copy-only assistant code blocks plus event-driven exec approval modals, but no structured Control UI command proposal card with an Approve and Run action.

## What This Changes

The issue requests a Control UI command card for assistant-suggested shell commands with Copy, Approve and Run, and safety context.

## Best Possible Solution

Design a structured command-proposal path that reuses existing exec approval and `systemRunPlan` contracts rather than adding a run button to arbitrary markdown code blocks.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main renders assistant command blocks with copy-only chrome while approval actions are only driven by queued gateway approval events.

## Solution Assessment

Unclear as requested until maintainers choose the product and security boundary. The safer direction is a structured proposal-to-approval contract with exact command, cwd, host, session, and risk metadata, not a direct run button attached to arbitrary rendered markdown.
