## Summary

Keep open. Current main still renders assistant shell snippets as generic markdown with Copy-only code blocks, while the approval UI only resolves queued runtime approval records; this remains a new security-sensitive Control UI feature, not conservative cleanup.

## What This Changes

The issue requests Control UI command cards with Copy and Approve and Run actions for assistant-suggested shell commands.

## Best Possible Solution

Add command cards only from explicit structured command metadata, bind exact command/argv, cwd, host/node, session, and approval identity to the existing exec approval and systemRunPlan machinery, refuse unstructured markdown execution, stream approved output back into chat, and document the security model.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The current-main gap is source-verifiable: assistant shell snippets render as Copy-only markdown while approval controls only resolve queued runtime approval records.

## Solution Assessment

Unclear until maintainers define the product and security contract. The safest implementation direction is structured command metadata that reuses existing approval and node systemRunPlan binding, not execution of arbitrary markdown code blocks.
