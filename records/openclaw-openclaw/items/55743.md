## Summary

Keep open: current main and v2026.6.10 still have copy-only assistant code blocks plus event-driven exec approval modals, but no structured Control UI command-proposal card with an Approve and Run action. This remains useful but security-sensitive product work, not conservative closeout material.

## What This Changes

The issue requests Control UI command cards for assistant-suggested terminal commands, including Copy, Approve and Run, Cancel, host/cwd metadata, and risk labels.

## Best Possible Solution

Design a structured command-proposal path that reuses existing exec approval and `systemRunPlan` contracts instead of adding a run button to arbitrary markdown code blocks.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main renders assistant command blocks with copy-only chrome while approval actions are only driven by queued Gateway approval events.

## Solution Assessment

Unclear as requested until maintainers choose the product and security boundary; the safer direction is a structured proposal-to-approval contract, not a direct run button attached to arbitrary rendered markdown.
