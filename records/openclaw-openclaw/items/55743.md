## Summary

Keep this open. Current main still renders assistant shell snippets as sanitized markdown with Copy-only code blocks, while the existing approval UI only resolves runtime-created approval records; this remains a new security-sensitive product feature rather than cleanup that can be closed.

## What This Changes

Request to add Control UI command cards with Copy and user-approved execution for assistant-suggested shell commands.

## Best Possible Solution

Add command cards only from explicit structured command metadata, bind them to the existing exec approval and systemRunPlan machinery, refuse unstructured markdown execution, stream approved output back into chat, and document the security model.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The current-main gap is source-verifiable: assistant shell snippets render as Copy-only markdown while approval controls only resolve queued runtime approval records.

## Solution Assessment

Unclear until maintainers define the product and security contract. The safest implementation direction is structured command metadata that reuses existing approval and node systemRunPlan binding, not execution of arbitrary markdown code blocks.
