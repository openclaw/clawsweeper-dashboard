## Summary

Keep open. Current main and the latest release support code-block copying and explicit exec approval modals, but they still do not provide a structured Control UI command proposal card with an Approve and Run action for assistant-suggested shell commands.

## What This Changes

Request a Control UI command card for assistant-suggested terminal commands with Copy, Approve and Run, Cancel, metadata, and risk classification.

## Best Possible Solution

Design a structured command-proposal path that reuses the existing exec approval and systemRunPlan contract, rendering run actions only for explicit command proposals rather than scraped markdown code blocks.

## Reproduction Assessment

Yes for source behavior: current main shows assistant code blocks only support copy, while actual command execution approvals are driven by explicit gateway approval events. There is no current-main bridge from suggested shell text to an approved run action.

## Solution Assessment

Unclear until maintainers choose the product and security boundary. The safer direction is a structured command proposal feeding the existing approval contract, not a direct run button on arbitrary rendered markdown.
