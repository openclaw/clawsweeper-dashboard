## Summary

Keep this open. Current main still treats assistant shell snippets as sanitized markdown with Copy-only code blocks, while the existing Control UI approval path only resolves approval records created by the runtime; the requested action is a new security-sensitive product feature, not a stale bug cleanup candidate.

## What This Changes

The issue requests Control UI command cards for assistant-suggested shell commands with Copy, Approve and Run, Cancel, risk labels, and exact host/cwd display.

## Best Possible Solution

Add command cards only from explicit structured command metadata, bind them to the existing exec approval/systemRunPlan machinery, stream results back into chat, and document the security model.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The current-main gap is source-verifiable: assistant shell snippets render as Copy-only markdown while approval controls only resolve queued runtime approval records.

## Solution Assessment

Unclear until maintainers define the product and security contract. The safe direction is structured command metadata that reuses existing approval and node systemRunPlan binding, not execution of arbitrary markdown code blocks.
