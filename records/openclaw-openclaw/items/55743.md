## Summary

Keep open: current main and the latest release still only provide copy chrome for assistant-rendered command blocks plus event-driven exec approval modals; they do not provide a structured Control UI command proposal card with an Approve and Run action.

## What This Changes

The issue requests a Control UI command card for assistant-suggested shell commands with Copy, Approve and Run, cancellation, risk labels, and explicit execution safeguards.

## Best Possible Solution

Design a structured command-proposal path that reuses the existing exec approval and systemRunPlan contracts, and render run actions only for explicit proposals carrying exact command, cwd, host, session, and risk metadata.

## Reproduction Assessment

Yes for source-level reproduction: assistant code blocks render copy-only chrome, while execution approvals appear only from gateway approval events. I did not run the browser UI because this review is read-only, but current source and existing tests establish the missing path.

## Solution Assessment

Unclear until maintainers choose the product and security boundary. The safer direction is a structured proposal-to-approval contract, not a direct run button attached to arbitrary rendered markdown.
