## Summary

Keep open: current main still has activity-based idle and tool-call loop guards, but no semantic text/thinking repetition breaker or context rollback. The open text-repetition PR is a plausible implementation candidate, not a merged fix, and prior review found blockers plus missing real behavior proof.

## What This Changes

Add protection against active streaming text or thinking repetition loops, including loop abort and session-history cleanup after a poisoned turn.

## Best Possible Solution

Land a maintainer-approved stream repetition guard, likely by repairing or replacing https://github.com/openclaw/openclaw/pull/39961, that detects text/thinking loops, aborts without flushing repeated buffers, and defines transcript cleanup semantics; keep generation-timeout config work in https://github.com/openclaw/openclaw/issues/8724 separate.

## Reproduction Assessment

Yes for the source-level gap: the idle watchdog resets around each streamed chunk, assistant text/thinking streaming has no repetition detector, and current main has no text-repetition guard. No live Gemini incident replay was run.

## Solution Assessment

Unclear as a concrete implementation: the proposed semantic breaker and rollback direction is appropriate, but the thresholds/defaults, buffer suppression, schema/docs, real behavior proof, and transcript sanitization policy need maintainer acceptance before merge.
