## Summary

Keep open. Current main still lacks a semantic text/thinking repetition breaker and loop-aborted replay quarantine: the idle watchdog is still chunk-activity based, assistant streaming has no repeated-content detector, and PR #39961 remains the open implementation candidate.

## What This Changes

Request to add a semantic streaming repetition breaker plus context sanitization so continuous repetitive model output cannot evade idle timeout or poison later turns.

## Best Possible Solution

Review, land, adapt, or replace PR #39961 with a core assistant-stream guard that detects repeated text/thinking while chunks are still arriving, aborts with explicit loop metadata, suppresses repetitive final drains, and excludes loop-aborted assistant output from provider replay while preserving transcript visibility.

## Reproduction Assessment

Yes. A stream that yields repeated text or thinking chunks faster than the idle timeout will keep streamWithIdleTimeout active, and current main has no semantic detector in the assistant stream path to stop that loop.

## Solution Assessment

Yes in direction, but not yet implemented on main. The maintainable boundary is a core assistant-stream circuit breaker plus replay quarantine; existing tool-loop detection cannot solve pure streamed text/thinking loops.
