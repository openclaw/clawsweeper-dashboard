## Summary

Keep open. Current main still lacks a semantic text/thinking repetition breaker and loop-aborted replay quarantine; the idle watchdog remains chunk-activity based, and the open related PR #39961 is still the implementation candidate rather than a merged fix.

## What This Changes

The issue requests a semantic repetition breaker and context sanitization for streaming assistant loops that keep producing tokens before the idle watchdog fires.

## Best Possible Solution

Review, land, adapt, or replace PR #39961 with a core assistant-stream guard that detects repeated text/thinking, aborts with explicit loop metadata, suppresses repetitive final drains, and excludes loop-aborted assistant output from provider replay while preserving transcript visibility.

## Reproduction Assessment

Yes. A source-level reproduction is clear: a stream that yields repeated text or thinking chunks faster than the idle timeout will keep streamWithIdleTimeout active, and current main has no semantic detector in the assistant stream path to stop it.

## Solution Assessment

Yes in direction, but it is not implemented on main. The narrow maintainable boundary is a core assistant-stream circuit breaker plus replay quarantine; existing tool-loop detection cannot solve pure streamed text/thinking loops.
