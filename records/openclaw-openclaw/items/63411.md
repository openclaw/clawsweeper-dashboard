## Summary

Keep open. Current main still has an activity-based idle watchdog and tool/post-compaction loop guards, but it does not include a semantic streamed text/thinking repetition breaker or a loop-aborted replay quarantine; the focused implementation candidate remains open at https://github.com/openclaw/openclaw/pull/39961.

## What This Changes

Request a semantic streaming repetition breaker and session-history sanitization for runaway assistant text/thinking loops that keep producing tokens before the idle timeout.

## Best Possible Solution

Land or replace the open stream-guard work with a conservative core assistant-stream detector that aborts repeated text/thinking, suppresses buffered repeat drains, and prevents loop-aborted assistant output from being replayed into future provider context.

## Reproduction Assessment

Yes, by source inspection. A stream that yields repeated text or thinking chunks faster than the idle timeout will keep streamWithIdleTimeout active, and the assistant message handler currently has no semantic repetition check before buffering streamed content.

## Solution Assessment

Yes in direction, but it is not implemented on main. The maintainable fix is a core assistant-stream guard plus replay quarantine; tuning the existing tool-loop detector would not catch pure text/thinking loops.
