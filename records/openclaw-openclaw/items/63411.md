## Summary

Keep open. Current main still lacks the requested semantic text/thinking repetition breaker and loop-aborted replay quarantine: the idle watchdog remains chunk-activity based, assistant text/thinking stream handling has no semantic loop detector, and the related open PR is an implementation candidate rather than shipped behavior.

## What This Changes

Request to add a semantic streaming repetition breaker plus context sanitization so continuous repetitive model output cannot evade idle timeout or poison later turns.

## Best Possible Solution

Review, land, adapt, or replace the related open PR with a core assistant-stream guard that detects repeated text/thinking while chunks are still arriving, aborts with explicit loop metadata, suppresses buffered repetitive final drains, and excludes loop-aborted assistant output from future provider replay while keeping transcript visibility intact.

## Reproduction Assessment

Yes. A stream that yields repeated text or thinking chunks faster than the idle timeout will keep streamWithIdleTimeout active, and current main has no semantic detector in the assistant stream path to stop that loop.

## Solution Assessment

Yes in direction, but not yet implemented on main. The maintainable boundary is a core assistant-stream circuit breaker plus replay quarantine; existing tool-loop detection cannot solve pure streamed text/thinking loops.
