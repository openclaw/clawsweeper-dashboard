## Summary

Keep open. Current main still has a chunk-activity idle watchdog and tool-call/post-compaction guards, but no semantic text/thinking repetition breaker or loop-aborted replay quarantine; the focused implementation candidate at https://github.com/openclaw/openclaw/pull/39961 is still open.

## What This Changes

Request a streaming text/thinking repetition breaker plus context sanitization so active-but-repetitive model output is aborted before poisoning session history.

## Best Possible Solution

Land or replace the open stream-guard implementation with a conservative core assistant-stream detector that aborts repeated text/thinking, marks the abort, suppresses buffered repeat drains, and excludes loop-aborted assistant output from provider replay.

## Reproduction Assessment

Yes, by source inspection. A stream that yields repeated text or thinking chunks faster than the idle timeout will keep streamWithIdleTimeout active, and handleMessageUpdate currently appends those chunks without a semantic repetition check.

## Solution Assessment

Yes in direction, but not implemented on main. The maintainable fix is a core assistant-stream guard plus replay quarantine; tuning the existing tool-loop detector cannot catch pure text/thinking loops.
