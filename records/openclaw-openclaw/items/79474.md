## What This Changes

The PR passes the requested `agentId` into `loadCombinedSessionStoreForGateway` for `sessions.resolve` sessionId and label lookups and adds scoped/unscoped regression coverage.

## Best Possible Solution

Add a single-line Unreleased changelog entry, then land this narrow resolver fix while tracking any broader session-store OOM work separately on https://github.com/openclaw/openclaw/issues/51264 if needed.

## Reproduction Assessment

Yes at source level: current main calls the all-agent combined loader for `sessions.resolve` sessionId and label paths even when `agentId` is present. I did not run a heap-OOM live scenario because this review is read-only and the crash depends on production-sized stores.

## Solution Assessment

Yes, this is the narrowest maintainable fix: use the existing `agentId` option on the combined-store loader where `sessions.resolve` already has that runtime fact. The tests preserve unscoped cross-agent behavior, so this avoids adding a cache or changing the public protocol.
