## Summary

Keep open: the PR addresses a real cron empty-output bug, but current head still leaves embedded fallback repair candidates unconstrained after a CLI-origin empty-output repair.

## What This Changes

The PR adds a one-pass cron final-reply repair path, fail-loud empty-output handling, CLI bundled-MCP suppression/session reuse, sender ownership propagation, descendant follow-up handling, and focused regression tests.

## Best Possible Solution

Keep the one-pass repair design, but make repair constraints runtime-specific for every fallback candidate so CLI repair suppresses bundled MCP while embedded repair remains tool-disabled.

## Reproduction Assessment

Yes. Current main's silent empty-output outcome is source-reproducible in direct cron delivery, and the PR-head fallback defect is source-reproducible from runWithModelFallback reusing one repair options object across candidates.

## Solution Assessment

No, not yet. The PR is the right overall shape, but the repair mode must choose constraints per fallback candidate instead of deriving one shared option set from the initial repair provider.

## Review Findings

Overall confidence: 0.86

Full review comments:

- **[P2] Constrain embedded repair fallbacks:** `src/cron/isolated-agent/run-executor.ts:575-577`
