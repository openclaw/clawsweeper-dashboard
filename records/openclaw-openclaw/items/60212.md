## Summary

Keep open: the PR is useful and includes sufficient live repair proof, but the current head still leaves an empty-sanitized-output path when fresh or active descendants exist, so the fail-loud guarantee is incomplete.

## What This Changes

The PR adds a cron empty-output repair pass, CLI bundled-MCP suppression and session reuse for repair, sender-scope regression tests, SDK subpath docs, and a changelog entry.

## Best Possible Solution

Keep the one-pass repair design, but make empty-output finalization cover descendant cases by recovering descendant output, performing the repair, or recording the same explicit cron error.

## Reproduction Assessment

Yes, by source inspection against PR head and current main. Empty sanitized output with a fresh or active descendant skips repair, and current delivery dispatch exits before descendant follow-up when `synthesizedText` is absent.

## Solution Assessment

No, not yet. The proposed repair is narrow for no-descendant runs, but it needs to cover the descendant-empty-output branch instead of leaving that path to finish without recovered text or an explicit error.
