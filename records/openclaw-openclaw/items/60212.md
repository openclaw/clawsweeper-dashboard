## Summary

Keep open; the PR’s central cron repair direction is useful, but the latest head still leaves a blocking fallback edge where a CLI repair can fall through to an embedded runtime without the intended no-tools constraint.

## What This Changes

Adds a cron empty-output repair pass, fail-loud delivery handling, CLI bundled-MCP suppression/session reuse, sender ownership coverage, descendant-output handling, tests, and a changelog entry.

## Best Possible Solution

Keep the one-pass repair design, but make repair constraints runtime-specific inside the fallback candidate callback so embedded repair candidates are tool-disabled while CLI candidates resume the captured session and only suppress bundled MCP.

## Reproduction Assessment

Yes. Source inspection on current main shows empty normalized delivery payloads can finish ok/not-delivered, and the PR-head fallback edge is source-reproducible through runWithModelFallback reusing one repair options object across candidates.

## Solution Assessment

No, not quite. The PR is the right shape for the reported cron bug, but the repair options need to be selected per execution provider so embedded fallback remains no-tools while CLI keeps session reuse and bundled-MCP suppression.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep fallback repair candidates constrained:** `src/cron/isolated-agent/run-executor.ts:543-544`
  - body: When a CLI cron run produces empty output, this builds one repair options object with `disableTools: undefined` and passes it through `runWithModelFallback`. If that CLI repair attempt falls back to an embedded provider, the embedded retry also receives `disableTools: undefined`, so the constrained finalization turn can call OpenClaw tools despite the repair prompt forbidding tools. Decide `disableTools` inside the `run` callback from the current `executionProvider` so embedded fallback candidates stay tool-disabled while CLI candidates only suppress bundled MCP.
  - confidence: 0.86
