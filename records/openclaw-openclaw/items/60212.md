## Summary

Needs changes before merge. The PR’s empty-output repair direction is useful and the supplied live proof is sufficient, but the current head still computes repair tool constraints once before model fallback, so a CLI-origin repair can fall through to an embedded candidate with OpenClaw tools enabled despite the repair prompt forbidding tool use.

## What This Changes

The PR adds cron empty-sanitized-output repair, fail-loud delivery handling, CLI bundled-MCP suppression, CLI session reuse, sender-ownership coverage, and descendant-output recovery tests.

## Best Possible Solution

Keep the repair/fail-loud design, but derive repair constraints inside the per-candidate fallback callback so each resolved runtime gets the correct tool boundary.

## Reproduction Assessment

Yes. Current main’s empty synthesized-text path is source-reproducible in delivery finalization, and the PR-head blocker is source-reproducible from shared repair options being reused across fallback candidates.

## Solution Assessment

No, not yet. The proposed repair shape is maintainable, but repair constraints must be calculated per fallback candidate after resolving the execution provider.
