## Summary

Keep open: the cron empty-output bug is valid on current main, but this PR still needs a current-main port and must preserve CLI session context during the repair retry.

## What This Changes

The PR adds a cron no-tools repair retry for empty sanitized final replies, explicit delivery errors when repair still fails, CLI no-tools handling, regression tests, and a changelog entry.

## Best Possible Solution

Port the repair to current main as one no-tools retry for unintended empty sanitized cron finals, preserve CLI session context or restate the original task safely, fail explicitly only after repair still has no deliverable text, and leave intentional `NO_REPLY` plus https://github.com/openclaw/openclaw/issues/60204 unchanged.

## Reproduction Assessment

Yes. Source inspection on current main shows empty payloads can resolve with no `synthesizedText`, and text delivery returns `null` instead of repairing or failing that state.

## Solution Assessment

No, not as proposed. The repair direction is narrow, but the PR changes CLI session reuse inputs and needs to be ported to the current executor split before it is the maintainable fix.
