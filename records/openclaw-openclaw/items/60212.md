## Summary

Keep open: the PR addresses a real cron silent-output gap, but the current head still has a merge-blocking repair-path bug because it computes repair tool constraints from the first empty-output provider and reuses them across all fallback candidates.

## What This Changes

The PR adds cron empty-output repair, fail-loud delivery handling, CLI bundled-MCP suppression and session reuse, descendant-output recovery, sender-ownership coverage, and regression tests.

## Best Possible Solution

Keep the repair and fail-loud design, but derive embedded no-tools versus CLI bundled-MCP constraints inside the per-candidate fallback callback and cover the mixed-runtime fallback case.

## Reproduction Assessment

Yes. Current main’s empty synthesizedText path is source-reproducible in delivery finalization, and the PR-head blocker is source-reproducible from the shared repair options object used across fallback candidates.

## Solution Assessment

No, not yet. The direction is maintainable, but repair constraints must be selected after each candidate's execution provider is known.
