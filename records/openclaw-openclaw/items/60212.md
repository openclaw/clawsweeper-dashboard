## Summary

Keep open: the PR addresses a real source-reproducible cron empty-output bug and has sufficient live repair proof, but the latest head still lets thread/structured direct delivery bypass the new descendant fallback and fail-loud path.

## What This Changes

The PR adds cron empty-output repair, CLI repair constraints/session reuse, shared silent-reply normalization, fail-loud delivery handling, and focused regression tests.

## Best Possible Solution

Move descendant fallback and no-deliverable-text error handling ahead of the direct/thread versus text-delivery split, then add a thread or structured-delivery regression for the empty-descendant-output case.

## Reproduction Assessment

Yes. The original main-branch bug is source-reproducible from the missing empty-final-output repair path, and the PR-head blocker is source-reproducible when emptyOutputHadFreshDescendants is true and delivery uses thread or structured direct mode.

## Solution Assessment

No, not yet. The repair design is narrow and maintainable, but the delivery-side contract must apply before direct/thread delivery as well as text delivery.
