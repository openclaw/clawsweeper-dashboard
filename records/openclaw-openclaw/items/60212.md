## Summary

This PR should stay open because the current head still has a concrete repair-path bug: CLI-origin empty-output repair options are computed once and can be reused by an embedded fallback, allowing tools during a no-tools repair prompt.

## What This Changes

The PR adds one constrained cron empty-output repair pass, CLI repair session reuse with bundled MCP disabled, explicit empty-output failure handling, descendant-output delivery handling, and focused regression tests.

## Best Possible Solution

Keep the one-pass repair design, but compute repair constraints inside the fallback callback so embedded candidates always get disableTools=true and CLI candidates only disable bundled MCP while reusing the matching CLI session.

## Reproduction Assessment

Yes. The original silent empty-output behavior and the PR-head mixed-runtime fallback blocker are both source-reproducible from the delivery dispatch, run-executor, and model-fallback control flow.

## Solution Assessment

No, not yet. The proposed repair shape is maintainable, but the implementation must make repair constraints runtime-specific before merge.
