## Best Possible Solution

Keep the one-pass repair design, but preserve fresh-descendant presence through final delivery so completed descendants with no recovered text trigger the same explicit error or repair outcome, with a focused regression test.

Yes, by source inspection: an empty final output with a fresh descendant already settled and no fallback text makes the repair gate skip the repair, while delivery-dispatch no longer knows a descendant existed and falls through to silent delivery handling.

## Solution Assessment

No, not yet. The repair approach is narrow for ordinary and active-descendant runs, but it needs to carry the completed-descendant/no-output case into the same fail-loud path before merge.
