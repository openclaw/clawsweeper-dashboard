## Summary

Keep open. The PR addresses a real cron empty-output failure and includes useful live proof, but current head still chooses repair constraints before model fallback, so a CLI-origin repair can fall through to an embedded candidate with tools enabled.

## What This Changes

Adds cron empty-output repair, CLI bundled-MCP suppression/session reuse, descendant-output fail-loud handling, and focused cron/CLI regression tests.

## Best Possible Solution

Keep the one-pass repair design, but choose repair constraints per fallback candidate runtime inside the fallback callback.

## Reproduction Assessment

Yes. Current main's delivery path can source-reproduce the silent ok/not-delivered empty-output outcome, and the PR-head fallback edge is source-reproducible through runWithModelFallback reusing one repair options object across candidates.

## Solution Assessment

No, not quite. The PR is the right overall fix shape, but the repair mode must be runtime-specific so embedded fallback candidates remain tool-disabled while CLI candidates resume the captured CLI session and suppress only bundled MCP.
