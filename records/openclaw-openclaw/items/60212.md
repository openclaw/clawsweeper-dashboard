## Summary

Keep open. The PR addresses a valid cron empty-output bug and includes sufficient live proof, but current head still has a merge-blocking tool-boundary issue when a CLI-origin repair falls back to embedded execution.

## What This Changes

The PR adds a cron empty-output repair pass, explicit fail-loud handling, CLI bundled-MCP suppression/session reuse, sender ownership propagation, descendant follow-up handling, and regression tests.

## Best Possible Solution

Keep the one-pass repair flow, but compute repair constraints per fallback candidate so embedded repair always disables tools and CLI repair disables only OpenClaw bundled MCP while reusing the matching CLI session.

## Reproduction Assessment

Yes. Current main's silent empty-output behavior is source-reproducible from delivery dispatch returning silent completion when normalized payloads are empty, and the PR-head blocker is source-reproducible from fallback reusing one repair option set across candidate runtimes.

## Solution Assessment

No, not yet. The repair design is the right shape, but the implementation needs candidate-specific runtime constraints before it is safe to merge.
