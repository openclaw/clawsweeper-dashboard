## Summary

Keep open; the current PR head still chooses empty-output repair tool constraints once from the provider that produced the empty reply, but model fallback can switch the repair attempt between CLI and embedded runtimes.

## What This Changes

The PR adds cron empty-sanitized-output repair/fail-loud handling, CLI bundled-MCP suppression and session reuse, descendant-output recovery, sender-ownership coverage, and focused regression tests.

## Best Possible Solution

Keep the cron repair/fail-loud behavior, but derive embedded no-tools and CLI bundled-MCP constraints inside each fallback candidate after resolving that candidate's execution provider.

## Reproduction Assessment

Yes. The current-main bug is source-reproducible from empty synthesized text returning null in delivery finalization, and the PR-head blocker is source-reproducible from runWithModelFallback invoking per-candidate callbacks while repair options are shared.

## Solution Assessment

No, not yet. The direction is narrow and maintainable, but repair constraint selection needs to move into the per-candidate execution path after the runtime provider is known.
