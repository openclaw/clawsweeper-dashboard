## Summary

Keep open: this PR addresses a real cron empty-output bug and includes sufficient live proof, but the current head still shares repair constraints across every model-fallback candidate, leaving a mixed CLI-to-embedded repair able to regain OpenClaw tools despite the no-tools repair prompt.

## What This Changes

The PR adds a cron empty-sanitized-output repair pass, CLI repair mode with bundled MCP disabled, explicit empty-output failures, descendant-output handling, and focused cron/CLI regression tests.

## Best Possible Solution

Keep the one-pass repair design, but compute repair constraints after each fallback candidate resolves its runtime so embedded candidates always get disableTools=true and CLI candidates only disable bundled MCP while reusing the matching CLI session.

## Reproduction Assessment

Yes. Current main's empty synthesizedText path is source-reproducible in finalizeTextDelivery, and the PR-head mixed-fallback blocker is source-reproducible from runWithModelFallback invoking the callback per candidate while repair options are computed once.

## Solution Assessment

No, not yet. The proposed repair shape is maintainable, but repair constraints need to be runtime-specific inside the fallback callback before this is the safest fix.
