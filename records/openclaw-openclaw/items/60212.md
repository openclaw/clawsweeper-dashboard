Keep open: the PR still has a source-reproducible repair-path blocker. The cron repair retry chooses tool constraints from the provider that produced the empty reply, but fallback execution is candidate-scoped, so a CLI-origin repair can fall back into embedded execution without `disableTools`.

## What This Changes

Yes. The current-main bug is source-reproducible where delivery finalization returns null for absent synthesized text, and the PR-head blocker is source-reproducible from shared repair options crossing candidate-scoped fallback execution.
