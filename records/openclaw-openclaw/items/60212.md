## Summary

Keep open: current main still has the empty-sanitized-output cron failure shape, but this PR is not merge-ready because the CLI repair retry can lose session context and the external PR lacks real behavior proof.

## What This Changes

The PR adds a no-tools cron repair retry for empty sanitized final output, fail-loud delivery handling, CLI disableTools MCP gating, regression tests, and a changelog entry.

## Best Possible Solution

Port one no-tools empty-output repair retry into the current cron executor/delivery split, preserve CLI session context or restate the original task safely, and fail explicitly only after repair still has no deliverable text.

## Reproduction Assessment

Yes. Source inspection shows empty payloads can resolve without synthesizedText, executeCronRun only retries likely interim acknowledgement text, and finalizeTextDelivery returns null when synthesizedText is missing.

## Solution Assessment

No, not as proposed. The repair direction is narrow, but the branch should preserve the existing CLI session binding or safely include the original task instead of changing the session-reuse prompt hash for the retry.
