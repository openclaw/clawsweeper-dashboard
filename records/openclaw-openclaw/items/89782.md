## Summary

Keep this PR open: it targets a real plugin callback dispatch bug that is still present on current main, but the proposed guard can turn a present malformed `pluginRoot` into an unscoped registration and misroute binding-resolution callbacks.

## What This Changes

The PR adds guarded enumeration of conversation-binding resolved callback registrations and a regression test for an unreadable registration before a healthy callback.

## Best Possible Solution

Keep the registry-read guard, but skip registrations whose present `pluginRoot` metadata is malformed, and add a focused regression proving a bad same-plugin root does not receive another root's callback.

## Reproduction Assessment

Yes. Source inspection shows current main reads callback registration metadata before the per-handler try/catch, and the PR body reports a focused red Vitest where an unreadable registration prevents a healthy callback from firing.

## Solution Assessment

No, not as written. The fix is in the right layer, but it should skip present malformed `pluginRoot` values instead of converting them into unscoped callback registrations.
