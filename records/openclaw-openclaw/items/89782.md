## Summary

Keep open: this PR fixes a real current-main callback dispatch bug, but the current patch still lets malformed same-plugin `pluginRoot` metadata become an unscoped callback registration.

## What This Changes

The PR adds a guarded conversation-binding resolved-callback enumerator and a regression test for an unreadable registration before a healthy callback.

## Best Possible Solution

Keep the registry-read guard, but only dispatch sanitized registrations; skip present malformed `pluginId`, `pluginRoot`, or handler metadata and add a focused malformed-root regression before merge.

## Reproduction Assessment

Yes. Current main reads callback registration metadata before the per-handler try/catch, and the PR body reports a red focused Vitest where a throwing registration prevents the healthy callback from firing; I did not run tests because this review is read-only.

## Solution Assessment

No, not as written. The fix is in the right layer, but the helper should skip present malformed `pluginRoot` values instead of treating them as unscoped registrations.
