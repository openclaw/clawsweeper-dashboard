## Summary

Keep open: this maintainer-labeled draft PR addresses a real plugin callback dispatch bug, but the current patch still has a narrow routing defect where malformed present `pluginRoot` metadata is treated as an unscoped callback registration.

## What This Changes

The PR adds a guarded conversation-binding resolved-callback registry reader and a regression test for an unreadable callback registration before a healthy callback.

## Best Possible Solution

Sanitize registrations before dispatch: skip invalid `pluginId`, present invalid `pluginRoot`, and non-function handlers while preserving truly unscoped entries, with a focused malformed-root regression test.

## Reproduction Assessment

Yes. Current main reads callback registration metadata before the per-handler try/catch, and the PR body reports a red focused Vitest for an unreadable sibling registration; this read-only review did not run tests.

## Solution Assessment

No, not as written. The fix is in the right layer, but the new helper should skip present malformed `pluginRoot` values instead of treating them as unscoped registrations.
