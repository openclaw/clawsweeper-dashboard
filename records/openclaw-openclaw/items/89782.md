## Summary

Keep open: the callback-dispatch bug is real on current main, but this PR is not merge-ready because the new guard treats a present non-string `pluginRoot` as unscoped instead of malformed, which can misdeliver binding-resolution callbacks.

## What This Changes

The PR adds guarded enumeration for conversation-binding resolved callback registrations and a regression test for an unreadable registration before a healthy callback.

## Best Possible Solution

Keep the registry-read guard, but only dispatch sanitized registrations whose `pluginId`, optional string `pluginRoot`, and handler satisfy the registry contract, with a regression for present malformed root metadata.

## Reproduction Assessment

Yes. Source inspection shows current main reads callback registration metadata before the per-handler try/catch, and the PR body reports a focused red Vitest where an unreadable registration prevents a healthy callback from firing.

## Solution Assessment

No, not as written. The fix is in the right layer, but it should skip present malformed `pluginRoot` values instead of converting them into unscoped callback registrations.
