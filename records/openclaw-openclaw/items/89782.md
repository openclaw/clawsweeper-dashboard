## Summary

Keep this PR open, but it still needs the existing callback-routing blocker fixed before merge. Current main still has the unguarded registry metadata read, while the PR’s helper would treat a malformed present `pluginRoot` as unscoped and could deliver resolved callbacks to the wrong plugin root.

## What This Changes

This PR adds a guarded conversation-binding resolved-callback enumerator and a regression test for an unreadable registration before a healthy Codex callback.

## Best Possible Solution

Keep the registry-read guard, but skip malformed callback registrations before dispatch while preserving legitimate missing-root unscoped entries, then rerun the focused and full conversation-binding tests.

## Reproduction Assessment

Yes. Current main’s source reads callback registration metadata before the per-handler try/catch, and the PR body reports a red focused Vitest where an unreadable sibling prevents the healthy callback from firing; I did not run tests in this read-only review.

## Solution Assessment

No, not as written. The fix is in the right owner layer, but the sanitizer should skip present malformed `pluginRoot` metadata instead of converting it into an unscoped callback registration.
