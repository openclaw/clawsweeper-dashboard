## Summary

Keep open: the central bugfix is still useful because current main still reads callback registration metadata before the per-handler guard, but this branch still has a routing defect where a present non-string `pluginRoot` is treated as an unscoped callback instead of being skipped.

## What This Changes

The PR guards conversation-binding resolved callback registry reads and adds a regression test for an unreadable registration preceding a healthy callback.

## Best Possible Solution

Keep the registry-read guard, but dispatch only sanitized callback registrations whose `pluginId`, optional `pluginRoot`, and handler satisfy the registry contract; malformed present root metadata should be skipped before merge.

## Reproduction Assessment

Yes, by source and PR-body proof. Current main reads callback registration metadata before the per-handler try/catch, and the PR body reports a red focused Vitest where a throwing registration prevented the healthy callback from firing.

## Solution Assessment

No, not as written. The fix is in the right layer, but the helper should skip present non-string `pluginRoot` values instead of treating them as unscoped registrations.
