## Summary

Keep open: this is a valid bug fix in the right area, but the current helper treats malformed present pluginRoot metadata as unscoped and can misdeliver binding-resolution callbacks; the maintainer-labeled draft state also rules out cleanup close.

## What This Changes

The PR adds guarded enumeration for conversation-binding resolved callback registrations and a regression test for an unreadable registration before a healthy callback.

## Best Possible Solution

Keep the registry-read guard, but distinguish truly absent roots from present malformed roots, skip malformed callback registrations, and cover that routing boundary with a focused regression before merge.

## Reproduction Assessment

Yes, source-reproducible. Current main reads callback registration metadata before the per-handler try/catch, so a throwing pluginId or pluginRoot accessor before a healthy handler can abort callback dispatch.

## Solution Assessment

No, not as written. The fix belongs in the callback dispatch layer, but present malformed pluginRoot metadata should be skipped rather than converted into an unscoped callback registration.
