- **[P2] Keep same-priority FIFO on sequence order:** `src/process/command-queue.ts:307-308`
  - body: `pickNextIndex` now breaks ties on `enqueuedAt` for every same-effective-priority entry, replacing the existing sequence-based FIFO guarantee. If `Date.now()` moves backward between enqueues, or preserved queue state has non-monotonic timestamps, a later foreground/normal/background entry can drain before an earlier one. Use `sequence` as the FIFO tie-breaker after effective priority; `enqueuedAt` should only decide whether an entry is starvation-promoted.
