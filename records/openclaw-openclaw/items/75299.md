## What This Changes

The PR adds Low/Normal/High command queue priorities, priority-aware lane selection with starvation promotion, an optional enqueue priority field, and queue tests.

## Best Possible Solution

Normalize preserved or malformed priorities to `CommandPriority.Normal`, cover the preserved singleton queue path, then merge only after contributor proof, green checks, and maintainer approval of the scheduling contract.

## Reproduction Assessment

Yes. Current main source shows FIFO-only draining and no enqueue priority option, and the PR-branch regression is source-reproducible from preserved singleton queue entries created before the new `priority` field existed.

## Solution Assessment

No, not yet. The in-place priority scan is a reasonable direction, but the branch should normalize missing or non-finite priorities and add regression coverage before this becomes a safe core queue contract.
