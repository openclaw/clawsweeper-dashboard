## Summary

Keep this PR open for changes: the starvation-guard idea is focused and useful, but the current comparator still lets aged foreground work outrank older aged background work, so it does not actually prevent starvation under sustained foreground backlog.

## What This Changes

The PR adds a 30-second dequeue-time starvation promotion for priority command queues and a regression test for aged background work running before fresh foreground work.

## Best Possible Solution

Implement starvation as a true fairness tier that selects starved entries by age or FIFO above static priority, then align this branch with the broader queue-priority policy work before landing.

## Reproduction Assessment

Yes, at source level. Current main drains the statically priority-sorted queue head, and the PR comparator still lets aged foreground entries outrank older aged background entries; I did not run a live gateway contention repro in this read-only review.

## Solution Assessment

No, not as written. Dequeue-time aging is the right surface, but promoted entries need to outrank static priority by age/FIFO or be folded into the broader scheduling policy in https://github.com/openclaw/openclaw/pull/80199.
