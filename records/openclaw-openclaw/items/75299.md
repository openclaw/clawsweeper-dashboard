## What This Changes

The PR adds a 30-second starvation promotion threshold to the command queue, scans queued entries at dequeue time, and adds a fake-timer regression test.

## Best Possible Solution

Land one command-queue starvation policy that preserves FIFO within promoted work and is backed by current-head contention proof, either in this focused PR or in the canonical scheduler PR.

Yes, at source level: current main inserts queued work by static priority and drains the head entry, so a steady stream of foreground entries can keep an older background entry behind. I did not establish a live contention reproduction in this read-only review.
