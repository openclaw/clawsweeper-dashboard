## Summary

Keep this PR open but not merge-ready: the branch is focused, but the current comparator still lets aged foreground backlog outrank older aged background work, and the supplied proof shows compiled code plus gateway startup rather than the claimed queue-order behavior.

## What This Changes

The branch exports a 30-second starvation-promotion constant, changes command-lane dequeue to pick an effective-priority index instead of shifting the queue head, and adds a focused queue regression test.

## Best Possible Solution

Implement starvation as a separate fairness tier where entries past the threshold beat all non-starved entries and are selected by oldest enqueue order, then reconcile the policy with https://github.com/openclaw/openclaw/pull/80199.

## Reproduction Assessment

Yes, at source level. The PR ranks starved entries as static priority plus 1000, so an aged foreground backlog still beats an older aged background entry; I did not run a live gateway contention repro in this read-only review.

## Solution Assessment

No, not as written. Dequeue-time aging is the right surface, but promoted entries need to outrank static priority by age or FIFO and the policy should be reconciled with the related scheduler work.
