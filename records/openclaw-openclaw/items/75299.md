## What This Changes

The branch adds a 30-second starvation promotion threshold to command queue dequeue selection, exports the threshold constant, and adds a focused regression test.

## Best Possible Solution

Implement dequeue-time aging so the oldest starved work can outrank static priority buckets, keep current queue diagnostics and legacy-state coverage intact, and coordinate the result with the related priority branch before merge.

## Reproduction Assessment

Yes at source level: current main has static priority insertion and head-only dequeue, while user/manual and background triggers map to different priorities. I did not run a live gateway repro in this read-only review.

## Solution Assessment

No, not as-is. Dequeue-time aging is the right surface, but the comparator should make the oldest starved work win across priority buckets and the branch needs green CI plus real behavior proof.
