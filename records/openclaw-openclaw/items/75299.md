## Summary

Keep this PR open: the starvation-guard idea is relevant, but the current comparator can still starve older background work under an aged foreground backlog, and the updated proof only shows compilation/startup rather than the changed queue behavior.

## What This Changes

The PR adds a 30-second starvation promotion constant, selects queue entries by computed priority at dequeue time, and adds a focused regression test for aged background work versus a fresh foreground entry.

## Best Possible Solution

Implement one queue fairness contract that makes starved entries compete by age/FIFO above static priorities, with sustained-backlog coverage and alignment with the broader priority branch.

## Reproduction Assessment

Yes at source level: current main inserts queued work by static priority and drains the head, so lower-priority work can remain behind a sustained stream of higher-priority entries. I did not run a live contention reproduction in this read-only review.

## Solution Assessment

No, not as-is. Dequeue-time aging is the right surface, but the proposed comparator keeps static priority inside the promoted tier and misses the sustained-backlog starvation case.

Overall confidence: 0.9
