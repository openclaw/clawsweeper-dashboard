## Summary

Keep this PR open, but it is not merge-ready: the starvation guard is aimed at a real queue fairness gap, yet the current comparator can still starve lower-priority work behind an aged foreground backlog and the supplied terminal proof does not demonstrate the changed queue behavior.

## What This Changes

The PR adds a 30-second command-queue starvation promotion constant, dequeue-time effective-priority scan, and a regression test for aged background work beating fresh foreground work.

## Best Possible Solution

Implement fairness so starved entries are selected by age/FIFO above static priorities, add sustained-backlog coverage, and align this branch with https://github.com/openclaw/openclaw/pull/80199 before landing.

## Reproduction Assessment

Yes at source level: current main inserts by static priority and drains the head of the queue, so lower-priority work can remain behind sustained foreground work. I did not run a live gateway reproduction in this read-only review.

## Solution Assessment

No, not as-is. Dequeue-time aging is the right surface, but promoted entries need to outrank static priority by age/FIFO or be folded into the broader policy in https://github.com/openclaw/openclaw/pull/80199.
