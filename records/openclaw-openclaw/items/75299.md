- **[P2] Preserve FIFO with sequence after effective priority:** `src/process/command-queue.ts:307-308`
  - body: `pickNextIndex()` still prefers smaller `enqueuedAt` before `sequence` for entries with the same effective priority. That replaces the existing sequence-based FIFO guarantee, so a clock rollback or preserved queue state with non-monotonic timestamps can let a later foreground, normal, or background entry drain before an earlier one. Use `sequence` as the tie-breaker after effective priority; `enqueuedAt` should only decide starvation eligibility.
