## Summary

Keep this PR open for maintainer review: current main still lacks command queue priorities, but this branch is not merge-ready because it lacks real behavior proof, has a concrete preserved-entry priority bug, and overlaps with the broader open implementation at https://github.com/openclaw/openclaw/pull/80199.

## What This Changes

The PR adds Low/Normal/High command queue priorities, priority-based lane dequeueing with starvation promotion, an optional enqueue priority field, and focused queue tests.

## Best Possible Solution

Adopt one canonical priority-scheduling design that normalizes preserved queue entries, wires real foreground/background callers, and updates the queue docs and regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows FIFO dequeueing with `state.queue.shift()` and no priority field in enqueue options; the PR-branch regression is source-reproducible from preserved singleton entries without a priority value.

## Solution Assessment

No, not as-is. The in-place priority scan is a plausible direction, but this branch should normalize missing or malformed priorities and then be weighed against the broader related implementation before merge.

Normalize missing or non-finite priorities to Normal before comparing and cover this preserved-entry path.
