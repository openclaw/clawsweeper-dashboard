## Summary

Keep open: the PR is a useful narrow scheduler guard, but it is not merge-ready because the promoted score still lets aged foreground work outrank older aged background work, and the supplied terminal proof does not exercise real queue contention.

## What This Changes

The branch adds a 30-second starvation promotion constant, dequeue-time priority scanning, and a regression test for an aged background entry beating a fresh foreground entry.

## Best Possible Solution

Implement starvation as a separate aged tier where aged entries beat non-aged entries and drain by enqueue age/FIFO, then reconcile that policy with https://github.com/openclaw/openclaw/pull/80199 before merge.

## Reproduction Assessment

Yes, at source level. The PR ranks starved entries as static priority plus 1000, so aged foreground work still beats older aged background work; no live contention run was performed in this read-only review.

## Solution Assessment

No, not as written. Dequeue-time aging is the right surface, but promoted entries need a distinct fairness tier ordered by enqueue age or FIFO, and maintainers should reconcile this with the broader scheduler branch.
