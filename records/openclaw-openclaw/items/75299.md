## Summary

Keep open: current main still has no dequeue-time starvation guard, but this PR’s aging rule does not fully prevent starvation when foreground backlog entries are also aged, and the supplied proof is still test-only.

## What This Changes

The PR adds a 30-second starvation promotion check to command queue dequeue selection, plus a focused test and changelog entry.

## Best Possible Solution

Implement dequeue-time aging so the oldest promoted lower-priority work eventually wins, preserve current queue diagnostics, add a regression with aged foreground backlog, and coordinate the result with https://github.com/openclaw/openclaw/pull/80199.

## Reproduction Assessment

Yes, at source level: current main inserts by static priority and drains the queue head, while user/manual session work is foreground and maintenance-style work is background. I did not run a live gateway reproduction in this read-only review.

## Solution Assessment

No, not as-is. Dequeue-time aging is the right surface, but this comparator still lets aged foreground work outrank aged background work indefinitely under a sustained foreground backlog.
