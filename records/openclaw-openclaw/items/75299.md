## Summary

Keep open: this PR targets a real command-queue starvation gap, but the proposed comparator still lets aged foreground backlog outrank older aged background work, and the supplied proof does not exercise sustained contention.

## What This Changes

The PR adds a 30-second starvation promotion constant, dequeue-time priority scanning, and a regression test for aged background work in the command queue.

## Best Possible Solution

Implement starvation as a separate aged tier where aged entries beat non-aged entries and drain by enqueue age or FIFO, then reconcile that policy with https://github.com/openclaw/openclaw/pull/80199 before merge.

## Reproduction Assessment

Yes, at source level. The PR ranks starved entries as static priority plus 1000, so aged foreground work still beats older aged background work under sustained backlog.

## Solution Assessment

No, not as written. Dequeue-time aging is the right surface, but promoted entries need a distinct fairness tier ordered by enqueue age or FIFO, and maintainers should reconcile this with the broader scheduler branch.
