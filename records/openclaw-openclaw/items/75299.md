## Summary

Keep this PR open, but it is not merge-ready: the branch targets a real queue starvation gap, yet the effective-priority comparator still lets aged foreground backlog outrank older aged background work, and the supplied terminal proof does not demonstrate the contention behavior it claims to fix.

## What This Changes

The branch exports a 30-second starvation threshold, changes command-lane dequeue to scan for an effective-priority entry, and adds a regression test for an aged background task ahead of a fresh foreground task.

## Best Possible Solution

Implement starvation as an explicit aged tier where starved entries beat non-starved entries and the aged tier is ordered by oldest enqueue order, then reconcile that policy with the broader scheduler PR.

## Reproduction Assessment

Yes, at source level. The PR ranks starved entries as static priority plus 1000, so aged foreground work still outranks older aged background work; I did not run a live gateway contention repro in this read-only review.

## Solution Assessment

No, not as written. Dequeue-time aging is the right surface, but promoted entries need a separate fairness tier ordered by age or FIFO, and the policy should be reconciled with the related scheduler PR.
