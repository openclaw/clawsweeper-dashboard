## Best Possible Solution

Rebase on current main, implement fair dequeue aging where old lower-priority work eventually beats aged foreground backlog, preserve queue diagnostics, add sustained-backlog regression coverage, and coordinate with the overlapping scheduling PR.

Yes, at source level. Current main priority-sorts queued entries and drains the head, while embedded user/manual work is foreground and maintenance work is background; I did not run a live gateway reproduction in this read-only review.

## Solution Assessment

No, not as-is. Dequeue-time aging is the right surface, but the comparator must handle aged foreground backlog and the branch must be rebased without losing current queue diagnostics.
