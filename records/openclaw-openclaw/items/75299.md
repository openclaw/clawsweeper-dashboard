## Summary

Keep open: the starvation guard is useful and focused, but the patch still has a deterministic FIFO bug for promoted same-millisecond entries, the real behavior proof does not show the changed contention behavior, and https://github.com/openclaw/openclaw/pull/80199 overlaps the same queue-policy decision.

## What This Changes

The PR adds a 30-second starvation promotion constant, dequeue-time effective-priority selection, and a regression test for aged lower-priority command queue work.

## Best Possible Solution

Preserve deterministic FIFO for promoted entries, then have maintainers choose whether this standalone guard or the overlapping scheduler branch owns the starvation policy.

## Reproduction Assessment

Yes at source level: current main inserts queued work by static priority and drains the head entry, so lower-priority work can remain behind a steady stream of foreground entries. I did not run a live contention scenario in this read-only review.

## Solution Assessment

No, not yet: the guard is plausible, but the implementation needs `sequence` as the final FIFO tie-breaker and maintainer acceptance of the 30-second ordering tradeoff.
