## What This Changes

The PR adds Low/Normal/High command queue priorities, priority-based dequeueing with starvation promotion, an optional enqueue priority field, and focused queue tests.

## Best Possible Solution

Converge on one priority-scheduling PR that normalizes legacy entries, wires real foreground/background callers if the issue is meant to be fixed end-to-end, documents the semantics, and includes real behavior proof.

Yes. Source inspection shows current main has FIFO dequeueing and no priority option, and the PR-branch regression is source-reproducible with preserved singleton entries that lack a priority value.

## Solution Assessment

No, not as-is. The in-place scan is plausible, but this branch should normalize missing or malformed priorities and be reconciled with the broader related implementation before merge.
