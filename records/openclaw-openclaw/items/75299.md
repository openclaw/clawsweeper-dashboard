## Summary

## What This Changes

This branch adds a 30-second queue aging threshold, scans for an effective dequeue priority, exports the threshold constant, and adds command-queue ordering tests.

## Best Possible Solution

Land a maintainer-approved starvation policy that either truly bounds low-priority starvation or updates the contract, tests, and proof to make foreground dominance explicit.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: hold a lane busy, enqueue background work, keep adding fresh foreground work, and the PR’s one-tier promotion keeps the aged background entry below foreground.

## Solution Assessment

No. One-tier promotion may be a valid policy, but it is not the best implementation for a PR that claims low-priority work cannot be starved by higher-priority enqueues.
