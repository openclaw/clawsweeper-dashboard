## Summary

Keep this PR open for maintainer review, but do not merge it as-is: current main still lacks command-queue priority support, the branch has a preserved-entry priority normalization bug, real behavior proof is missing, and the overlapping implementation at https://github.com/openclaw/openclaw/pull/80199 needs to be reconciled.

## What This Changes

The branch adds Low/Normal/High priority support to the in-process command queue, priority-based dequeueing with a starvation guard, an optional enqueue priority field, and focused queue tests.

## Best Possible Solution

Adopt one canonical priority-queue implementation that normalizes legacy entries, wires foreground/background callers where the user-visible delay occurs, updates the queue contract/docs, and closes the linked issue only after real behavior proof.

## Reproduction Assessment

Yes. Current main source shows FIFO `state.queue.shift()` and no priority option, and PR-branch source shows preserved entries can compare with an undefined priority.

## Solution Assessment

No, not as-is. The in-place scan is a plausible direction, but this branch should normalize missing or malformed priorities and be reconciled with https://github.com/openclaw/openclaw/pull/80199 before merge.
