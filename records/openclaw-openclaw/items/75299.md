## Summary

Keep open: current main and v2026.5.28 still have priority insertion with FIFO dequeue and no starvation promotion, so the proposed behavior is not already implemented. The patch looks like a plausible focused fix, but merging it needs maintainer scheduling-policy review because it changes default session-lane ordering and overlaps the broader open queue-priority work at https://github.com/openclaw/openclaw/pull/80199.

## What This Changes

The PR adds a fixed 30-second starvation-promotion threshold to the priority command queue dequeue path and covers the aged-background-before-fresh-foreground case with a focused test.

## Best Possible Solution

Land one maintainer-approved command-queue starvation policy with tests and docs aligned, either by accepting this focused 30-second guard or folding the behavior into the broader priority-policy PR.

## Reproduction Assessment

Yes, source-reproducible. Current main inserts by static priority and drains with state.queue.shift(), while user/manual work is foreground and cron/maintenance work is background, so a steady foreground stream can keep older lower-priority work queued.

## Solution Assessment

Unclear as final product policy. The implementation is a narrow queue-layer fix, but the fixed 30-second threshold and overlap with https://github.com/openclaw/openclaw/pull/80199 need maintainer choice before calling it the best solution.
