## Summary

Keep open: the PR is focused, but the starvation comparator still lets aged foreground work outrank older aged background work, so it does not fully solve the stated starvation case. The supplied proof shows compiled code and gateway startup, not real queue behavior under sustained contention, and the remaining scheduling policy overlaps the broader queue-priority work in https://github.com/openclaw/openclaw/pull/80199.

## What This Changes

The PR adds a 30-second dequeue-time starvation promotion constant, selects queued command entries by effective priority, and adds a regression test for aged background work beating fresh foreground work.

## Best Possible Solution

Implement starvation as a true fairness tier that selects starved entries by age or FIFO above static priority, then align this branch with the broader queue-priority policy before landing.

## Reproduction Assessment

Yes, at source level. Current main drains the statically sorted queue head, and the PR comparator still ranks aged foreground above older aged background; I did not run a live gateway contention repro in this read-only review.

## Solution Assessment

No, not as written. Dequeue-time aging is the right surface, but promoted entries need to outrank static priority by age or FIFO, and the policy should be reconciled with the related queue-priority PR.
