## Summary

Keep open, but this PR is not merge-ready: the proposed aging score still lets aged foreground work outrank older aged background work, the supplied proof does not exercise sustained contention, and maintainers need to reconcile this queue policy with the related scheduler PR.

## What This Changes

The branch adds a 30-second starvation promotion constant, dequeue-time priority scanning, and a regression test for aged background work beating a fresh foreground entry.

## Best Possible Solution

Implement starvation as a distinct aged tier where aged entries beat non-aged entries and drain by enqueue age or FIFO, then reconcile that policy with https://github.com/openclaw/openclaw/pull/80199 before merge.

## Reproduction Assessment

Yes, at source level. The PR ranks starved entries as static priority plus 1000, so aged foreground work still beats older aged background work; no live contention run was performed in this read-only review.

## Solution Assessment

No, not as written. Dequeue-time aging is the right surface, but promoted entries need a distinct fairness tier ordered by enqueue age or FIFO, and maintainers should reconcile this with the broader scheduler branch.
