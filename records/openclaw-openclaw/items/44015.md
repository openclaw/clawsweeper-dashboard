## Summary

Keep open: current `main` still has no command-level `chat.send` byte cap, so the PR is not obsolete, but the proposed patch needs real behavior proof and a revision for the normalized-byte enforcement gap before merge.

## What This Changes

The PR adds a 512 KiB gateway `chat.send` message cap in server sanitization, mirrors it as schema metadata, adds focused sanitizer tests, and documents the protocol limit.

## Best Possible Solution

Land a revised gateway hardening change that defines the raw-vs-normalized byte policy, documents schema character semantics, covers both cases in tests, and includes redacted real gateway/client proof.

## Reproduction Assessment

Yes for the PR-introduced defect: source inspection and a focused byte-count probe show decomposed Unicode can be over 512 KiB on input and under 512 KiB after NFC normalization. This is not a current-main bug reproduction because current main has no command-level `chat.send` byte cap.

## Solution Assessment

No. The hardening direction is useful, but the patch should check the intended byte representation before accepting the message and should make clear that schema `maxLength` is a character constraint rather than byte enforcement.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Measure raw chat.send bytes before normalization:** `src/gateway/server-methods/chat.ts:235-236`
  - body: The new cap checks `Buffer.byteLength(normalized, "utf8")` after `message.normalize("NFC")`. A decomposed payload such as `"e\u0301".repeat(200000)` is about 600 KB on input but about 400 KB after NFC, so it can pass despite the documented 512 KiB inbound limit; check raw message bytes first or define a normalized-size policy.
  - confidence: 0.9
- **[P3] Clarify schema maxLength character semantics:** `src/gateway/protocol/schema/logs-chat.ts:35-39`
  - body: `Type.String({ maxLength })` is a JSON Schema character-count constraint, not a UTF-8 byte limit. Multi-byte messages can exceed 512 KiB and still pass schema validation before the sanitizer sees them, so this schema/comment should not read as byte-level enforcement.
  - confidence: 0.84
