Yes. Source inspection gives a high-confidence path: call `readConfiguredLogTail` once, append more than `maxBytes` while the prior cursor remains within the file, and current `main` sets both `truncated` and `reset`, causing both CLI notices.

## Solution Assessment

Yes for the code direction. Removing `reset = true` only from the fast-growth branch is the narrow maintainable fix because true rotation remains represented by `cursor > size`, while skipped bytes continue to use the existing `truncated` flag.
