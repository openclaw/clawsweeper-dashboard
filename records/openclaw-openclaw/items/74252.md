## What This Changes

The PR narrows log-tail `reset` semantics, adds optional `skippedBytes` metadata for skipped tail windows, re-anchors Control UI log entries on skipped bytes, refreshes the generated Swift protocol model, and adds log-tail/CLI/UI tests.

## Best Possible Solution

Land the PR if maintainers accept optional `skippedBytes` as the log-tail gap signal; otherwise request a reset-only variant or a separate protocol follow-up.

## Reproduction Assessment

## Solution Assessment

Yes, with maintainer acceptance of the protocol expansion. Narrowing `reset` to cursor invalidation and using `skippedBytes` for skipped windows is cleaner than overloading the rotation flag for UI re-anchoring.
