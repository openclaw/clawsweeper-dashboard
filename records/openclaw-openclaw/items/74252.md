## Summary

Keep open: the false CLI rotation notice is real and the supplied proof is strong, but this patch fixes it by changing the existing `logs.tail.reset` meaning that current Control UI and external clients can use as a non-contiguous-window re-anchor signal.

## What This Changes

The PR removes `reset = true` from the fast-growth log-tail branch and adds log-tail plus CLI tests for truncated-only and true-rotation behavior.

## Best Possible Solution

Preserve `logs.tail.reset` as the re-anchor/non-contiguous-window signal and make rotation-specific CLI output a separate backwards-compatible condition.

## Reproduction Assessment

Yes. Current main source and the supplied terminal proof establish that fast log growth currently sets `truncated: true` and `reset: true`; I did not execute a local repro because this review is read-only.

## Solution Assessment

No. Removing `reset` fixes the CLI message by deleting the shared re-anchor signal; the better fix is to preserve that signal and make rotation-specific presentation explicit.
