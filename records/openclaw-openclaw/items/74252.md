## Summary

Keep open: the core log-tail fix is well targeted and the new terminal proof covers the before/after behavior, but the PR still changes the `logs.tail` reset contract in a way that leaves the current Control UI appending across skipped fast-growth gaps.

## What This Changes

This PR narrows `readLogSlice.reset` to file-shrink rotations, adds fast-growth and rotation regression tests, and adds a changelog entry.

## Best Possible Solution

Keep the false rotation notice fix, but update the Control UI or `logs.tail` contract so skipped fast-growth chunks still have an explicit discontinuity path before merge.

## Reproduction Assessment

Yes. Source inspection on current main shows fast growth sets both `truncated` and `reset`, and the CLI maps `reset` to the false rotation notice; the contributor's terminal proof also demonstrates the before/after boolean behavior.

## Solution Assessment

No. The one-line core change is the right direction for the CLI message, but the PR should also preserve or explicitly update the existing Control UI discontinuity handling for skipped chunks.
