## Summary

Keep open. Current main still reports a fast-growth log tail as both truncated and reset, while this PR narrowly fixes that behavior, adds focused regression coverage, and includes the required changelog entry; I found no blocking review findings.

## What This Changes

The PR removes the fast-growth `reset = true` path in log tailing, adds regression tests for fast growth and real rotation, and adds an Unreleased changelog entry.

## Best Possible Solution

Land the narrow reset/truncated semantic split with the new tests and changelog once the branch is mergeable and CI is green.

## Reproduction Assessment

Yes. On current main, read the log once, append more than `maxBytes` while the previous cursor is still valid, then read again with that cursor; `readLogSlice` sets both `truncated` and `reset`, causing the CLI rotation notice.

## Solution Assessment

Yes. Removing `reset` only from the fast-growth branch is the narrowest maintainable fix because true file shrink still reports reset, skipped bytes still report truncation, and the payload shape stays unchanged.
