## Summary

Keep open: the core fix is well targeted and the contributor supplied useful before/after terminal proof, but the patch changes the `logs.tail` reset contract without updating the Control UI consumer that currently uses `payload.reset` to re-anchor the displayed log buffer.

## What This Changes

This PR removes `reset = true` from the fast-growth log-tail path, adds log-tail regression tests for fast growth and true shrink/rotation, and adds a changelog entry.

## Best Possible Solution

Keep the rotation notice honest while preserving an explicit consumer contract for skipped log-tail gaps, either by updating Control UI to re-anchor on truncated fast-growth payloads or by adding a separate gap/re-anchor signal.

Yes. Current-main source shows the fast-growth branch returns `truncated: true` and `reset: true`, and the PR comments include a concrete temp-file harness proving the before/after behavior; I did not run tests because this review is read-only.

## Solution Assessment

No, not yet. Narrowing `reset` in `readLogSlice` fixes the CLI notice, but the best fix also updates consumers that used `reset` as a re-anchor signal for skipped log windows.
