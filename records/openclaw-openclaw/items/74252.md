## Best Possible Solution

## Reproduction Assessment

Yes. Current main and `v2026.6.10` show the fast-growth branch setting `reset = true`, and the CLI maps `reset` directly to the file-rotated notice; the contributor also supplied terminal proof with a real on-disk log file.

## Solution Assessment

Yes. Narrowing `reset` to cursor invalidation and using additive `skippedBytes` for skipped-window UI re-anchoring is the clean owner-boundary fix; keeping `reset` for skipped windows would preserve the false rotation signal.
