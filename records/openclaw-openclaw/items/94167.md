## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve config env vars during deferred substitution:** `src/agents/mcp-transport.ts:152-155`
  - body: This reads from `process.env` at request time, but gateway startup reads config with `isolateEnv: true` and applies `env.vars` only to the cloned env used for the snapshot. A current config with `env.vars.MCP_TOKEN` and `mcp.servers.remote.headers.Authorization: "Bearer ${MCP_TOKEN}"` resolves today, but after this change the placeholder is preserved and the later transport lookup cannot see the config-provided value. The deferred resolver needs to carry the merged runtime env source, not only `process.env`.
  - confidence: 0.9
- **[P2] Handle MCP server names with dots:** `src/config/env-substitution.ts:181-182`
  - body: The wildcard matcher treats `*` as a single dot-delimited segment, but MCP server names are arbitrary object keys and the docs explicitly normalize non-alphanumeric names for tool prefixes. A key such as `"corp.github"` produces a path like `mcp.servers.corp.github.headers.Authorization`, so this ignore rule misses it and the header is still substituted at startup. Match the MCP object path structurally instead of with a dot-joined regex.
  - confidence: 0.78
