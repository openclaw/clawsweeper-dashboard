## Summary

Keep open. Current main still preserves the kickstart-first macOS LaunchAgent restart path, and the related supervision-state work references this issue without implementing the requested stop, wait, then start ordering.

## What This Changes

Request a two-phase macOS LaunchAgent restart that stops the service, waits for the previous process to exit, then bootstraps and starts the new service instance.

## Best Possible Solution

Make an explicit maintainer decision on the macOS LaunchAgent restart contract; if accepted, implement the ordering in the launchd owner module while preserving detached self-restart and supervision recovery paths.

## Reproduction Assessment

Unclear for a high-confidence live reproduction because the failure depends on launchd and port timing. The acceptance gap is source-reproducible: current main still starts with `kickstart -k` before any explicit bootout and previous-process wait sequence.

## Solution Assessment

Unclear pending maintainer judgment. The proposed two-phase sequence directly targets the race, but current tests intentionally protect kickstart-first behavior and recent launchd fixes show bootout/bootstrap recovery has regressed before.
