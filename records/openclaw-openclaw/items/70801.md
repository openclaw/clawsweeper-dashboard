## Summary

Keep open. Current main still uses the kickstart-first macOS LaunchAgent restart path, and the merged supervision-state work at https://github.com/openclaw/openclaw/pull/72616 only references this issue as adjacent context rather than changing normal restart ordering.

## What This Changes

Request two-phase macOS LaunchAgent restart ordering: stop the old job, wait for the previous process/port to clear, then bootstrap and start the service.

## Best Possible Solution

Make a maintainer launchd-contract decision; if accepted, implement the stop/wait/start ordering in `src/daemon/launchd.ts` while preserving detached handoff and supervision recovery behavior.

## Reproduction Assessment

No for a high-confidence live reproduction; this review did not exercise macOS launchd timing. The source path is reproducible by ordering inspection because normal restart still reaches `kickstart -k` before any explicit bootout or previous-process wait.

## Solution Assessment
