## Summary

Keep open: the PR targets the right memory-core bug and includes useful terminal proof, but current head still treats contentful list-wrapped headings without a colon as placeholders, which would suppress legitimate short-term memory promotions.

## What This Changes

The PR adds a memory-core guard for markdown placeholder short-term promotion snippets across store load, candidate recording, ranking, relocation, apply, rehydration, and focused regression tests.

## Best Possible Solution

Keep the shared promotion guard, but broaden heading payload detection so contentful headings without colons remain promotable and add a focused regression test for that case before merge.

## Reproduction Assessment

Yes, source-level. Current main permits markdown skeleton snippets through promotion gates, and PR head shows a concrete over-filter path for contentful list-wrapped headings without colons.

## Solution Assessment

No, not as currently written. The shared guard is the right direction, but the colon-only inline-heading test is too narrow; the safer fix is to treat headings with meaningful text as contentful and cover that case.
