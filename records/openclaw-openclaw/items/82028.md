## Summary

Keep open: the PR is on the right memory-core boundary, but it still leaves one normalized heading/empty-marker skeleton shape promotable and the external PR proof remains test-only.

## What This Changes

The PR adds a shared memory-core predicate to reject markdown placeholder short-term promotion snippets across store load, recording, ranking, relocation, apply, and rehydration paths, with focused regression tests.

## Best Possible Solution

Tighten the shared predicate to reject normalized single-line heading/empty-marker skeletons, keep the boundary guards and regression coverage, then land after real behavior proof and green checks.

## Reproduction Assessment

Yes. Current main has a source-level path where non-empty markdown placeholders can enter, rank, relocate, and apply, and the linked issue provides a concrete skeleton shape plus sanitized affected-workspace cleanup evidence.

## Solution Assessment

No as submitted. The shared boundary guard is the right maintainable direction, but it should also catch normalized 'heading - heading' skeletons and the PR still needs real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Reject normalized heading skeletons without trailing markers:** `extensions/memory-core/src/short-term-promotion.ts:354-360`
  - body: `normalizeStore` sees persisted snippets after whitespace has already been collapsed, so a skeleton range like `## Tagesnotizen\n-\n\n## Entscheidungen` becomes `## Tagesnotizen - ## Entscheidungen`. This branch treats that as a plain heading and never marks the embedded empty bullet as a placeholder, leaving a stale stored skeleton able to rank and apply. Please detect marker separators between headings before accepting the snippet.
