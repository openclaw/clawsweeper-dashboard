## What This Changes

Adds a shared memory-core predicate to filter markdown placeholder short-term promotion snippets at store load, recording, ranking, relocation, apply, and rehydration, with regression tests.

## Best Possible Solution

Extend the shared guard to reject normalized heading-marker-heading skeletons, keep the boundary checks and regression coverage, then land after non-test real behavior proof and green required checks.

## Reproduction Assessment

Yes, as a source-level path: current main accepts non-empty markdown placeholders through existing empty/dreaming-contamination gates, and the linked issue provides a concrete daily-note skeleton plus sanitized cleanup evidence.

## Solution Assessment

No as submitted. The shared guard at each promotion boundary is the right maintainable direction, but it must also reject normalized heading-marker-heading stale snippets and the PR still needs non-test real behavior proof.

Full review comments:

- **[P2] Reject normalized heading-marker skeletons:** `extensions/memory-core/src/short-term-promotion.ts:354-360`
