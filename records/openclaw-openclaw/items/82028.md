## Summary

## What This Changes

The PR adds a memory-core short-term promotion guard that filters markdown placeholder or skeleton snippets across store normalization, candidate recording, ranking, applying, and tests.

## Best Possible Solution

Keep the shared unpromotable-snippet guard, broaden heading payload detection so real heading text does not require a colon, add predicate and promotion-path coverage for non-colon heading snippets, and refresh real temp-workspace proof on the current head.

## Reproduction Assessment

Yes, source-level. On the current PR head, a snippet like '- ## Decision move backups to S3' strips to a heading, fails the colon-only payload check, and is returned as unpromotable before record/rank/apply paths can promote it.

## Solution Assessment

No, not as currently written. The shared predicate is the right boundary, but the colon-only heading heuristic is too narrow; preserving durable heading text without requiring punctuation is the safer maintainable fix.
