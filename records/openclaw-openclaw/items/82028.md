## Summary

Keep open: the PR addresses a real placeholder-promotion bug and has sufficient filesystem proof, but the current PR-head guard still suppresses valid non-colon markdown heading memories across memory-core promotion paths.

## What This Changes

The PR adds a shared memory-core guard for markdown skeleton snippets during short-term recall loading, recording, ranking, applying, and rehydration, plus focused regression tests.

## Best Possible Solution

Keep the shared placeholder guard, but make heading payload detection accept meaningful heading text without requiring `:`, and cover both plain and list-wrapped non-colon headings in predicate plus promotion-path tests.

## Reproduction Assessment

Yes. Source inspection of PR head shows `- ## Decision move backups to S3` is stripped to a heading without `:`, marked unpromotable, and then rejected by the record/rank/apply gates that now call the helper.

## Solution Assessment

No. The shared guard is the right shape for the reported placeholder bug, but colon-only heading payload detection is too narrow; the safer fix is meaningful-content detection that preserves existing non-colon heading memories.
