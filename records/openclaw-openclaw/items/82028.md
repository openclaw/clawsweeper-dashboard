## Summary

Keep open for normal maintainer review: current main still allows markdown marker/skeleton snippets through memory-core short-term promotion paths, and this PR is a focused fix with regression coverage and sufficient terminal proof.

## What This Changes

The PR adds a shared memory-core predicate that rejects markdown marker/skeleton snippets during short-term recall store load, candidate recording, ranking, relocation, direct apply, and rehydration, plus regression tests.

## Best Possible Solution

Land this focused guard and regression coverage, then close https://github.com/openclaw/openclaw/issues/80582 once the fix is on main; keep the separate empty REM placeholder work tracked separately.

## Reproduction Assessment

Yes, as a source-level path. Current main treats non-empty markdown placeholders as ordinary snippets after normalization and can carry them through ranking, relocation, and apply; the linked issue also provides sanitized real-workspace cleanup evidence.

## Solution Assessment

Yes. A shared unpromotable-snippet predicate at every short-term promotion boundary is the narrow maintainable fix for the reported placeholder leak, and the PR avoids adding a new config or product mode.
