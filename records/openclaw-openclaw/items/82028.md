## Summary

Keep open: this is a plausible focused fix for the linked memory bug, but the current PR still misses a normalized marker-heading skeleton shape and the posted proof is fixture-test-only rather than after-fix real behavior proof.

## What This Changes

Adds a shared memory-core guard for unpromotable markdown placeholder snippets across short-term promotion paths, plus regression tests for placeholder recording, ranking, applying, and ordinary bullet preservation.

## Best Possible Solution

Land a shared unpromotable-snippet guard across store load, recording, ranking, relocation, apply, and rehydration once it handles normalized skeleton variants and has real memory-workspace proof.

## Reproduction Assessment

Yes, at source level. Current main can store, rank, relocate, and apply non-empty placeholder snippets because the existing filters only reject empty or contaminated dreaming snippets, and the linked issue gives the daily-note skeleton shape.

## Solution Assessment

Not yet. The shared predicate at the promotion boundaries is the right maintainable shape, but it needs to reject marker-prefixed normalized heading skeletons before the PR is correct.
