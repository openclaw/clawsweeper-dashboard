## Summary

Keep open: the Slack catalog opt-in matches the existing channel recovery shape, but the Brave half does not actually unblock the Brave error described in the PR body because current installer recovery still rejects `tools.web.search.provider` validation issues. The external PR also needs after-fix real behavior proof before merge.

## What This Changes

This PR adds `allowInvalidConfigRecovery: true` to the official Brave plugin and Slack channel catalog install entries.

## Best Possible Solution

Extend the narrow recovery path to cover same-plugin official external web-search provider failures, with regression coverage, while keeping unrelated invalid configs fail-closed.

## Reproduction Assessment

Yes, at source level: unavailable Brave validates as `tools.web.search.provider`, while installer invalid-config recovery only accepts channel and plugin-load issue shapes. I did not run a live 2026.5.12 upgrade/install scenario in this read-only review.

## Solution Assessment

No. Adding the catalog flag is sufficient for Slack-style `channels.slack` recovery, but Brave also needs the installer recovery predicate and tests to recognize the missing official web-search provider case, or it should be handled by the broader doctor-gate fix tracked in https://github.com/openclaw/openclaw/issues/82301.
