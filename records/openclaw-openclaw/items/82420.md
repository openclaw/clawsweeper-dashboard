## Summary

Keep open: this is a plausible narrow catalog repair for Slack, but the Brave half is incomplete because current install recovery still rejects the reported `tools.web.search.provider` validation issue, and the external PR also lacks after-fix real behavior proof.

## What This Changes

Adds `allowInvalidConfigRecovery: true` to the Brave official external plugin catalog entry and the Slack official external channel catalog entry.

## Best Possible Solution

Keep the narrow catalog opt-ins only if the Brave web-search validation path is also admitted through the existing recovery predicate and covered by focused regression tests.

## Reproduction Assessment

Yes, at source level: current validation can produce a `tools.web.search.provider` issue for a missing Brave provider, and install recovery rejects that issue path even after catalog opt-in. I did not run the live 2026.5.12 upgrade host in this read-only review.

## Solution Assessment

No: the Slack catalog opt-in matches the existing channel recovery shape, but the Brave fix needs the recovery predicate to recognize the matching web-search provider validation issue. A broader doctor-gate fix should remain with https://github.com/openclaw/openclaw/issues/82301.
