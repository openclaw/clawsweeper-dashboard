## Summary

Keep open: the PR’s useful UX goal is clear, but the current one-line shared-helper change is not safe to merge because it changes creation-only semantics, conflicts with current tests/docs, and lacks a bounded resend policy for live pairing codes.

## What This Changes

The PR changes `issuePairingChallenge` to send a pairing reply whenever the store returns a code, including existing pending requests, instead of only when a request is newly created.

## Best Possible Solution

Define a maintainer-approved bounded resend policy that preserves creation-only callback/result semantics, then update the shared helper, affected channel expectations, docs, and tests together.

## Reproduction Assessment

Yes for the PR-introduced behavior: with `upsertPairingRequest` returning `{ code: "ABCD", created: false }`, the changed guard falls through into reply sending, `onCreated`, and a `{ created: true, code }` result. The inspected Telegram screenshot also shows repeated `/start` replies after the patch.

## Solution Assessment

No. The one-line guard change is not the best solution because it changes shared callback/result semantics and lacks an explicit resend bound or channel-specific policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P2] Preserve creation state when resending codes:** `src/pairing/pairing-challenge.ts:31`
  - body: When the store returns an existing `{ code, created: false }`, this guard now sends a reply, invokes `onCreated`, and returns `{ created: true, code }`. A resend path should keep creation-only callbacks and returned state tied to actual new requests.
  - confidence: 0.95
- **[P2] Bound repeated pairing-code replies:** `src/pairing/pairing-challenge.ts:31`
  - body: This resends the same pending pairing code on every repeat sender message until expiry across the shared helper, while current docs and tests limit sends to new requests. Add a maintainer-approved cooldown, cap, or channel-specific policy before changing this shared path.
  - confidence: 0.9
