## Summary

Keep open: the requested UX may be valid, but this patch is not mergeable because it changes a shared pairing contract without preserving creation-only callbacks, cooldown/security policy, or tests/docs.

## What This Changes

This PR changes the shared pairing challenge guard so an existing pending request with a nonempty code sends another pairing reply instead of returning early.

## Best Possible Solution

Model repeat delivery as an explicit pairing reminder path that preserves creation-only signals, applies cooldown/suppression policy, and updates contract tests, affected channel tests, docs, and changelog together.

## Reproduction Assessment

Yes. Mock `upsertPairingRequest` to return `{ code: "ABCD", created: false }`: current main returns `{ created: false }` without sending, while this PR falls through to callback/reply delivery and a created-style result.

## Solution Assessment

No. The one-line guard change conflates request creation with reminder delivery; the safer design is a named reminder contract with preserved creation semantics and abuse controls.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Preserve creation-only side effects:** `src/pairing/pairing-challenge.ts:31`
  - body: Changing this guard to `!code` lets repeat pending requests returning `{ code, created: false }` fall through to `onCreated`, `sendPairingReply`, and a `{ created: true, code }` result. Channel callers use those signals for first-request logging or follow-up work, so repeated `/start` messages would be reported as new pairing requests.
  - confidence: 0.93
- **[P2] Keep resend policy explicit:** `src/pairing/pairing-challenge.ts:31`
  - body: This flips the shared repeat-pending behavior for all pairing consumers, but current tests and security docs still say repeated DMs do not resend codes. Matrix's reminder path has explicit cooldown semantics, so this needs a named reminder contract and matching tests/docs rather than falling through the creation path.
  - confidence: 0.9
