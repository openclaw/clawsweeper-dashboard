## Summary

Keep open for maintainer decision: the repeat-code UX is plausible, but this patch conflicts with the documented and tested one-message-per-new-request contract and changes shared creation side effects across channels.

## What This Changes

The PR changes the shared DM pairing challenge guard so an existing non-empty pairing code is replied to again instead of returning when the request was not newly created.

## Best Possible Solution

If repeat sends are accepted, add an explicit resend branch that preserves `created: false`, gates `onCreated` to new requests, updates docs/tests, and defines resend throttling expectations.

## Reproduction Assessment

Yes for the proposed patch regression: source inspection shows `{ code, created: false }` would fall through to reply and return `created: true`. There is no current-main bug reproduction for the requested behavior because current docs/tests intentionally do not resend until a new request is created.

## Solution Assessment

No. Replacing the guard alone is not the best solution because it changes shared creation semantics and documented security behavior without a resend contract, tests, docs, or cooldown policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P2] Preserve creation state when resending codes:** `src/pairing/pairing-challenge.ts:31`
  - body: When `upsertPairingRequest` returns `{ code, created: false }`, this guard now falls through to `onCreated`, sends a reply, and returns `{ created: true, code }`. A resend should not look like a newly created request or repeat creation side effects; keep `created` false and gate `onCreated` to new requests.
  - confidence: 0.94
- **[P2] Update the shared resend contract:** `src/pairing/pairing-challenge.ts:31`
  - body: Current docs and tests say already-pending requests are not sent another pairing message. If repeat sends are intended, the shared helper needs updated tests, docs, channel expectations, and abuse limits rather than changing the guard alone.
  - confidence: 0.88
