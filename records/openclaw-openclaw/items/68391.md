## Summary

Keep open. Current main still accepts the inherited `models --agent <id>` option on the `set` and `set-image` write paths, but those paths ignore the agent scope and update global model defaults. The linked fix PR #68399 is closed and unmerged, so the bug remains a narrow, source-backed repair candidate.

## What This Changes

The issue asks for `openclaw models --agent <id> set <model>` to avoid silently updating `agents.defaults.model.primary` when the user requested an agent-scoped model change.

## Best Possible Solution

Add an explicit guard for `--agent` on `models set` and `models set-image` unless the CLI intentionally implements per-agent writes, while preserving existing `status`, `list`, and `auth` agent-scoped behavior and adding focused regression tests.

## Reproduction Assessment

Yes. The source path is enough to reproduce with high confidence: `models --agent shuri set ...` is accepted by the parent command, the `set` action ignores the inherited option, and the command writes through the global-default updater.

## Solution Assessment

Yes, the narrowest maintainable fix is to reject `--agent` on these write commands with a clear error until per-agent writes are deliberately designed. Full per-agent mutation can be a follow-up, but it should not be implied by a parent option that currently updates global defaults.
