Full review comments:

- **[P1] Preserve existing Vertex model registrations during onboarding:** `extensions/google/provider-contract-api.ts:119-121`
  - body: This configPatch writes a fresh one-item `models.providers["google-vertex"].models` array. `applyProviderAuthConfigPatch` deep-merges objects but replaces arrays, so re-running this auth flow on an existing Vertex setup drops any user-registered Vertex models and metadata. Build the patch from the existing provider config or use an additive helper so onboarding only adds the default when it is missing.
  - confidence: 0.9
- **[P2] Scope provider auto-registration to valid provider configs:** `src/plugins/provider-model-primary.ts:99-117`
  - body: `applyPrimaryModel` is a generic helper used by manual/default-model flows, but this new branch creates `models.providers.<provider>.models` for any slash-containing model. For an unknown custom provider, current schema rejects that provider unless it also has `baseUrl`, so a manual `custom/model` selection can now write invalid config. Limit this to existing providers or known bundled overlays, or keep the google-vertex registration local to the google provider setup path.
  - confidence: 0.84
