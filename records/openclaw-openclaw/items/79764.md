## Summary

Keep open: the context-loss path is source-reproducible, but this PR changes raw pre-compaction transcript reseeding into the default and reverses an explicit safety test, so it needs maintainer policy/security review plus real behavior proof before merge.

## What This Changes

The branch changes CLI session reseeding to return a bounded raw message tail for sessions with no compaction and updates the unit test to expect a raw-tail history prompt.

## Best Possible Solution

Preserve the explicit transcript safety boundary unless maintainers choose otherwise; the maintainable path is a documented opt-in or other reviewed reset-time recovery design with schema, tests, and live proof.

## Reproduction Assessment

Yes from source: current main returns no reseed messages for uncompacted sessions, and the fresh-session prompt path then has no rendered history. I did not live-run the idle-reset scenario in this read-only review.

## Solution Assessment

No: the unconditional fallback is narrow, but it is not the safest maintainable fix because it reverses an explicit no-raw-pre-compaction safety behavior. A reviewed opt-in or reset-time recovery design is safer.
