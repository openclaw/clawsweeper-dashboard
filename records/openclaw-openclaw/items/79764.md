## Summary

Keep open for maintainer/contributor follow-up: the reported context-loss path is real, but this PR changes a deliberate raw-transcript safety boundary and is not landable as the default behavior.

## What This Changes

The PR makes uncompacted CLI session reseeding fall back to a bounded raw message tail and updates the no-compaction unit test to expect those raw messages.

## Best Possible Solution

Keep the no-raw default and land a documented CLI backend opt-in, or a later summarize-on-invalidation design, with schema coverage and live redacted resume proof.

## Reproduction Assessment

Yes from source: current main returns no reseed messages for uncompacted sessions, and the fresh-session path only gets a historyPrompt when those messages render. I did not live-run the idle-reset scenario in this read-only review.

## Solution Assessment

No as written: the patch is narrow, but it makes raw pre-compaction transcript reseeding the default. The safer maintainer-backed path is an explicit backend opt-in or summary-based recovery.
