## Summary

Keep open: the reported no-compaction reseed gap is source-reproducible, but this PR changes the default transcript safety boundary by sending raw uncompacted history into fresh CLI sessions and still lacks real after-fix behavior proof.

## What This Changes

The PR changes CLI session reseeding so uncompacted sessions return a bounded raw message tail and updates the unit test to expect that raw-tail history.

## Best Possible Solution

Preserve the default no-raw safety boundary unless maintainers approve a documented opt-in or reset-time recovery design with schema, tests, and live proof.

## Reproduction Assessment

Yes from source: current main returns no reseed messages for uncompacted sessions, and the fresh-session prompt path then has no rendered history. I did not live-run the idle-reset scenario in this read-only review.

## Solution Assessment

No: the unconditional fallback is narrow, but it reverses an explicit no-raw-pre-compaction safety behavior. A reviewed opt-in flag or compaction-based recovery design is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Keep raw reseed behind an explicit opt-in:** `src/agents/cli-runner/session-history.ts:200-206`
  - body: This makes every uncompacted transcript feed its raw message tail into `historyPrompt` for fresh CLI sessions. Keep the default empty behavior unless maintainers approve a documented opt-in policy.
