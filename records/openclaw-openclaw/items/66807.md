## Summary

Current main and the shipped v2026.5.2 release already implement the requested Slack target-peer account routing through a safer shared resolver; this branch is superseded and would be unsafe to merge as-is because it can select unintended credential-bound Slack accounts.

## What This Changes

The PR adds a peer-id scan in `runMessageAction` to select a Slack account for outbound sends when the target matches a configured binding peer.

## Best Possible Solution

Keep the released resolver-based implementation and close this superseded branch instead of merging its raw binding scan.

## Reproduction Assessment

Yes for the historical bug: the PR body gives a concrete two-account Slack reproduction, and current source plus regression coverage show the old fallback was replaced by target-aware account resolution. I did not rerun a live Slack workspace repro during this read-only review.

## Solution Assessment

Yes for current main, no for this branch. The shipped resolver is route-only, agent-scoped, peer-aware, and peer-kind aware; this branch's raw peer scan is not the narrowest maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve agent scoping when matching peer bindings:** `src/infra/outbound/message-action-runner.ts:866`
  - body: This fast path assigns `accountId` from the first binding that matches only channel plus `peer.id`, before the agent-keyed fallback can run. If two Slack bots are intentionally bound to the same peer for different agents, the later agent can be forced onto the first binding's account and fail with the same kind of cross-account routing error.
  - confidence: 0.88
- **[P1] Filter route bindings before selecting peer accounts:** `src/infra/outbound/message-action-runner.ts:849-851`
  - body: Scanning raw `cfg.bindings` includes ACP bindings, while the existing routing helpers use `listRouteBindings` to exclude them. An ACP binding with the same channel and peer can therefore select an account that was not intended for outbound route sends.
  - confidence: 0.84
