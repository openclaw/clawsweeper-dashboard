## Summary

Keep open: the gateway inflight race is valid and the PR is pointed in the right direction, but this head still needs contributor real behavior proof and a rebase/fix for current-main message action media-root forwarding before merge.

## What This Changes

The PR rewrites gateway send, message.action, and poll idempotency handling to use synchronous cached/inflight/ready state, register work before async resolution, and add a concurrent message.action regression test.

## Best Possible Solution

Rebase on current main, keep the synchronous inflight claim for send/message.action/poll, preserve message action mediaLocalRoots, and merge only after real behavior proof is added.

## Reproduction Assessment

Yes, from source inspection. Current main can check inflight before async work is registered for send/message.action, and poll has no inflight registration at all.

## Solution Assessment

Mostly yes, but not as-is. The cached/inflight/ready shape is the narrow maintainable fix, but the branch needs to preserve current-main media roots and provide real behavior proof before merge.
