## Summary

Keep open: current main still has the gateway inflight deduplication race for `send`/`message.action` and no inflight path for `poll`, while this PR’s latest code direction appears to address the race. The merge blocker is process/proof, not a clear code defect: this external PR still lacks after-fix real behavior proof and the Real behavior proof check is failing.

## What This Changes

The PR changes gateway `send`, `message.action`, and `poll` idempotency handling to use synchronous cached/inflight/ready state, register shared work before async resolution, and add a concurrent `message.action` regression test.

## Best Possible Solution

Land a focused gateway fix that synchronously claims `send`, `message.action`, and `poll` inflight work before async resolution, with contributor-provided real behavior proof and focused regression coverage for the changed handlers.

## Reproduction Assessment

Yes, from source inspection. On current main, `message.action`/`send` can await channel resolution before inflight registration and `poll` has no inflight registration, so same-key concurrent requests can race; I did not run a live repro in this read-only review.

## Solution Assessment

Yes for the code direction. The cached/inflight/ready split is the narrow maintainable fix for this race, but the PR still needs after-fix real behavior proof before merge.
