## Summary

Keep open. Current main still has the gateway inflight race, and this PR is a narrow, useful fix for send, message.action, and poll; the only concrete blocker I found is the missing required changelog entry for a user-facing gateway fix.

## What This Changes

The PR rewrites gateway send, message.action, and poll idempotency handling to distinguish cached, inflight, and ready states, register shared work before async channel resolution, and add concurrent regression coverage.

## Best Possible Solution

Gateway outbound methods should share a synchronously claimed inflight promise for send, message.action, and poll while preserving existing authorization, validation, and channel behavior, with the user-facing fix documented in the Unreleased changelog before merge.

## Reproduction Assessment

Yes. Two concurrent gateway requests with the same idempotencyKey can both miss the inflight map on current main before async channel resolution registers work, and poll has no inflight registration at all.

## Solution Assessment

Yes. The PR direction is the narrow maintainable fix: synchronously decide cached/inflight/ready, register the shared work before yielding, and cover concurrent message.action and poll behavior; it still needs the changelog entry before it is mergeable.
