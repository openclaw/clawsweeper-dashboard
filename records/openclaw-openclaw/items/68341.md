## Summary

Keep this PR open. Current main still has the gateway inflight dedupe race the PR targets, and the PR is the active implementation lane; it needs maintainer review and a required changelog entry before merge rather than cleanup closure.

## What This Changes

The PR rewrites gateway send, message.action, and poll dedupe handling to distinguish cached, inflight, and ready states, moves async channel work into shared promises, and adds concurrent regression tests for message.action and poll.

## Best Possible Solution

Land a narrow version of this fix that synchronously claims the inflight promise before the first await for send, message.action, and poll; preserves existing authorization/channel/session behavior; includes focused concurrent regression coverage; and adds the required changelog entry.

## Reproduction Assessment

Yes. The race is reproducible by issuing two concurrent gateway requests with the same idempotencyKey: on main, both send/message.action callers can pass the inflight miss before later async channel resolution registers work, and poll has no inflight registration at all.

## Solution Assessment

Yes, the PR direction is the right narrow fix: move async resolution into the shared work promise and handle cached/inflight/ready states consistently. It still needs changelog and validation before it is the best mergeable form.
