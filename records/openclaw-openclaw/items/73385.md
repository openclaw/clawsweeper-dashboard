## What This Changes

The PR changes the old `src/agents/pi-tools.ts` write-tool gate so sandboxed `write`, `edit`, and `apply_patch` only appear for `workspaceAccess: "rw"`, adds old pi-tools tests for `none`/`ro`/`rw`/binds, and adds a changelog entry.

## Best Possible Solution

Port the `workspaceAccess === "rw"` tool-surface gate and tests to current `src/agents/agent-tools.ts`, remove the changelog edit, and leave writable-`none` or writable-bind semantics to the canonical sandbox policy issues.
