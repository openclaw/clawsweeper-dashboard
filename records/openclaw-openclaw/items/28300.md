## Summary

Keep open. Current main still provides the narrower Claw/Knot/Dash theme model plus one browser-local tweakcn import slot, while the requested preset gallery, hue generator, gradient controls, grouped editor, and per-variable reset studio remain absent.

## What This Changes

The issue requests a Control UI theme customization system with curated preset themes, a custom Theme Studio, generated color variables, gradient controls, per-variable overrides, persistence, and i18n.

## Best Possible Solution

Define the Control UI theming product contract first, then implement the chosen model consistently across theme types, CSS token families, browser-local persistence, Appearance and Quick Settings UI, English source copy plus generated locales, public docs, and focused UI/storage regression tests.

## Reproduction Assessment

Not applicable as a feature request. Current-main source and docs provide a high-confidence verification path: the UI exposes one tweakcn import slot and the requested preset gallery/studio surfaces are absent.

## Solution Assessment

Unclear until maintainers choose the product direction. The current tweakcn import slot is a maintainable partial answer for custom colors, but it is not the full built-in preset and Theme Studio system requested here.
