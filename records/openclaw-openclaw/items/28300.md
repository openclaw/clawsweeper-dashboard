## Summary

Keep open. Current main has a narrower Control UI theming model with Claw, Knot, Dash, and one browser-local tweakcn import slot; it does not implement the requested preset gallery, hue generator, gradient controls, grouped editor, or per-variable override/reset studio, and the remaining direction is product/design work rather than conservative cleanup.

## What This Changes

Request a Control UI theme customization system with six preset themes plus a custom Theme Studio for hue generation, mode controls, gradients, and per-variable overrides.

## Best Possible Solution

Define the intended Control UI theming model first, then implement the chosen model consistently across the theme contract, CSS token families, browser-local persistence, Appearance and Quick Settings UI, generated/i18n copy, public docs, and focused regression tests.

## Reproduction Assessment

Not applicable as a feature request. Current-main source, docs, and focused tests provide a high-confidence verification path: the UI exposes one tweakcn import slot and the requested preset gallery/studio surfaces are absent.

## Solution Assessment

Unclear until maintainers choose the product direction. The current tweakcn import slot is a maintainable partial answer for custom colors, but it is not the full built-in preset and Theme Studio system requested here.
