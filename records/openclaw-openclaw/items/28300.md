## Summary

Keep open. Current main contains the narrower approved Control UI theming model with Claw, Knot, Dash, and one browser-local tweakcn import slot, but it still lacks the requested preset gallery, hue-based studio, per-variable editor/reset flow, and OpenClaw-side persistence discussed later.

## What This Changes

Add Control UI preset themes plus a Custom Theme Studio with generated color variables, per-variable overrides, persistence, and i18n.

## Best Possible Solution

Set an explicit Control UI theming policy, then implement the chosen scope consistently through theme types, CSS tokens, Appearance and Quick Settings, storage, docs/i18n, and focused UI tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs checks on current main show only Claw/Knot/Dash plus one browser-local tweakcn import slot, not the requested studio, preset gallery, or broader persistence model.

## Solution Assessment

Unclear until maintainers choose the product direction. The current tweakcn slot is the approved smaller solution, while a built-in gallery/studio and OpenClaw-side persistence would intentionally expand the theming and settings model.
