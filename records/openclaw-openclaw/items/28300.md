## Summary

Keep open. Current main implements the narrower approved Control UI theming model with Claw, Knot, Dash, and one browser-local tweakcn import slot; the requested preset gallery, hue-based Theme Studio, per-variable override/reset editor, and OpenClaw-side persistence remain absent and need maintainer product/design direction.

## What This Changes

Add preset Control UI theme families and a Custom Theme Studio with hue generation, gradients, per-variable overrides, and persisted theme settings.

## Best Possible Solution

Decide the intended Control UI theming scope first, then implement the approved model consistently across theme types, CSS tokens, Appearance and Quick Settings, storage, docs/i18n, and focused UI tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs checks on current main show only Claw/Knot/Dash plus one browser-local tweakcn import slot, not the requested studio, preset gallery, or broader persistence model.

## Solution Assessment

Unclear until maintainers choose the product direction. The current tweakcn import slot is the approved narrower solution, while a built-in gallery/studio and OpenClaw-side persistence would intentionally expand the theming and settings model.
