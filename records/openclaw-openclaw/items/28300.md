## Summary

Keep open. Current main ships a narrower Claw/Knot/Dash plus browser-local tweakcn import model, while the requested preset gallery, interactive Theme Studio, hue controls, per-variable overrides, and OpenClaw-side cross-browser persistence remain unimplemented and need maintainer product/design direction.

## What This Changes

Add a Control UI theme customization system with preset themes, a custom theme studio, live previews, per-variable overrides, and persisted user-selected themes.

## Best Possible Solution

Define the intended Control UI theming model first, then implement the chosen scope consistently across theme types, CSS tokens, Appearance and Quick Settings, persistence, docs, i18n, and focused UI tests.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a feature request. Source, docs, tests, and live issue state give high-confidence verification that the requested gallery, studio, editor controls, and cross-browser persistence are absent.

## Solution Assessment

Unclear until maintainers choose product direction. The current tweakcn import slot is an approved narrower solution, while a full built-in studio or gateway-persisted theme library would deliberately expand the theming model.
