## Summary

Keep open. Current main implements a narrower Control UI theming model with Claw, Knot, Dash, and one browser-local tweakcn import slot; it still lacks the requested preset gallery, hue generator, gradient controls, grouped editor, and per-variable override/reset studio, so this remains product/design work rather than conservative cleanup.

## What This Changes

The issue requests a Control UI theme customization system with curated presets, hue-generated custom themes, gradient controls, and per-variable overrides.

## Best Possible Solution

Define the intended Control UI theming model first, then implement the chosen scope consistently across the theme contract, CSS token families, browser-local persistence, Appearance and Quick Settings UI, generated/i18n copy, public docs, and focused regression tests.

## Reproduction Assessment

Not applicable as a feature request. Current-main source, docs, and focused tests provide a high-confidence verification path: the UI exposes one tweakcn import slot and the requested preset gallery/studio surfaces are absent.

## Solution Assessment

Unclear until maintainers choose the product direction. The current tweakcn import slot is a maintainable partial answer for custom colors, but it is not the full built-in preset and Theme Studio system requested here.
