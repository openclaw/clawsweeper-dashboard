## Summary

Keep open. Current main implements a narrower Control UI theming model with Claw, Knot, Dash, and one browser-local tweakcn import slot; it does not implement the requested preset gallery, hue generator, gradient controls, grouped editor, or per-variable override/reset studio, and the remaining work needs maintainer product/design direction.

## What This Changes

The issue requests built-in Control UI theme presets plus a Custom Theme Studio with hue-based generation, dark/light controls, gradient accents, per-variable overrides, persistence, and i18n.

## Best Possible Solution

Define the intended Control UI theming model first, then implement the approved model consistently across the theme contract, CSS token families, browser-local persistence, Appearance and Quick Settings UI, generated/i18n copy, public docs, and focused UI/storage regression tests.
