## Summary

Keep open. Current main has a narrower Control UI theme surface with Claw, Knot, Dash, and one browser-local tweakcn import slot, but the requested preset gallery, hue generator, gradient controls, grouped editor, and per-variable override/reset studio are still absent and require product/design direction.

## What This Changes

The issue requests a broader Control UI theme customization system with six presets, a hue-driven Custom Theme Studio, gradient controls, local persistence, and i18n updates.

## Best Possible Solution

Define the intended Control UI theming model first, then implement the chosen scope consistently across theme contracts, CSS token families, browser-local persistence, Appearance and Quick Settings UI, docs, i18n generation, and focused regression tests.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a feature request. Source and docs inspection on current main provides a high-confidence verification path showing that the requested preset gallery and Theme Studio are absent.

## Solution Assessment

Unclear until maintainers choose the product direction. The current tweakcn import slot is a maintainable partial answer for user colors, but it is not the full built-in preset and per-variable studio system requested here.
