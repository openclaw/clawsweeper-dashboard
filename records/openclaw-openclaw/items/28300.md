## Summary

Keep open. Current main implements a narrower Control UI theme model with built-in Claw/Knot/Dash families plus one browser-local tweakcn import slot, but it does not provide the requested preset gallery, hue generator, gradient controls, grouped editor, or per-variable override/reset studio.

## What This Changes

The issue requests a Control UI theming system with six preset themes, a Custom Theme Studio, generated and overrideable colors, mode and gradient controls, persistence, and i18n.

## Best Possible Solution

Define the intended Control UI theming model first, then implement the chosen scope consistently across theme contracts, CSS token families, browser-local persistence, Appearance and Quick Settings UI, docs, i18n, and focused regression tests.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection on current main provides a high-confidence verification path showing that the requested preset gallery and Theme Studio are absent.

## Solution Assessment

Unclear until maintainers choose the product direction. The current tweakcn import slot is a maintainable partial answer for user colors, but it is not the full built-in preset and per-variable studio system requested here.
