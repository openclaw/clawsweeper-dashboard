## Summary

Keep open: current main and v2026.6.5 still provide a narrower Control UI theme model with Claw, Knot, Dash, and one browser-local tweakcn import slot, not the requested preset gallery, in-app theme studio, per-token overrides, reset controls, or synchronized persistence. ## What This Changes

Requests a Control UI theme customization system with curated presets, a custom theme studio, generated CSS variables, per-variable overrides, and persisted settings.

## Best Possible Solution

Define the supported Control UI theming model first, then implement the chosen scope consistently across theme types, CSS variables, Appearance and Quick Settings UI, storage, docs, i18n, and focused regression tests.

## Reproduction Assessment

Not applicable: this requests new theming capabilities beyond the current documented theme contract rather than reporting broken existing behavior. Current source, tests, and docs establish the existing built-in, imported-theme, and browser-local boundaries.

## Solution Assessment

No, not as one all-encompassing automated change: the proposed studio is plausible but bundles several independent UI, token, preset, and persistence decisions. A maintainer-approved smaller increment, such as an interactive picker/import improvement plus an explicit persistence contract, would be safer.
