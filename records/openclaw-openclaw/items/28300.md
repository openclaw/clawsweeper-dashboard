## Summary

Current main and v2026.6.6 ship a narrower theme system—Claw, Knot, Dash, and one browser-local tweakcn import slot—so the requested preset gallery, in-app theme studio, per-token overrides, and cross-browser persistence remain materially unimplemented. The request fits Control UI ergonomics, but maintainers must decide whether to extend the tweakcn-backed flow or make OpenClaw own another theme-authoring and synchronized-preference system.

## What This Changes

Add curated Control UI theme presets and an in-app custom theme studio with generated colors, per-token overrides, previews, and persistent selection.

## Best Possible Solution

Keep one canonical Appearance architecture: reuse tweakcn for authoring, decide whether OpenClaw should add interactive preview and a curated accessible preset layer, and design synchronized preference storage explicitly before moving theme state out of browser-local storage.

## Reproduction Assessment

Not applicable: source, docs, and v2026.6.6 clearly establish the current narrower behavior, while this issue requests new preview, editing, preset, and synchronization capabilities rather than reporting a defect.

## Solution Assessment

No: the complete studio proposed here is not yet proven to be the narrowest maintainable solution because current main delegates theme authoring to tweakcn; interactive preview, curated presets, and synchronized persistence should be evaluated as separable product decisions.
