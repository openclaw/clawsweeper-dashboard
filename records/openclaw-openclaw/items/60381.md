## Summary

Keep open. Current main already implements the requested gated `evaluate` action, but the `force` click option is still absent from the browser action schema, request type, normalizer, CLI/docs, and both managed Playwright and existing-session click paths; the May 12 issue comment gives clear guardrails that force clicks should remain explicit and auditable.

## What This Changes

The issue requests an explicit `force` option for browser clicks and an `evaluate` escape hatch for browser actions.

## Best Possible Solution

Keep the existing gated `evaluate` implementation, and add an opt-in `force?: boolean` click path through the browser plugin contract, normalizer, managed Playwright forwarding, existing-session behavior, CLI/docs/skill guidance, and focused regression tests.

## Reproduction Assessment

Yes, by source inspection: submit `/act` with `{ "kind": "click", "ref": "1", "force": true }`; current main has no schema, type, normalizer, CLI, or Playwright/Chrome-MCP forwarding path for that field. I did not run tests because this review is read-only.

## Solution Assessment

Yes. The narrow maintainable solution is an additive opt-in click field that forwards to Playwright `locator.click`/`dblclick` with `force: true` while preserving evaluate gating, navigation guards, and guidance that normal clicks remain the default.
