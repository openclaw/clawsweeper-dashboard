## Summary

Keep open: current main and v2026.6.10 already expose browser `act:evaluate`, including the newer statement-body evaluate path, but the requested force-click parameter is still absent from the browser tool schema, normalized action type, route normalization, CLI surface, tests, and Playwright click execution path.

## What This Changes

Request to add an explicit browser click force option and expose browser evaluate as a JavaScript escape hatch for difficult web UI automation.

## Best Possible Solution

Keep the existing gated `evaluate` implementation and add `force` only if maintainers approve it as an explicit last-resort click option with schema/types/normalization/runtime/CLI/docs/tests and clear safety guidance.

## Reproduction Assessment

Yes at source level for the remaining part: current click requests have no `force` field and click execution never passes `force` to Playwright. I did not drive a live overlay-intercepted page in this read-only review.

## Solution Assessment

Unclear as a product/API decision: a narrow force field is mechanically plausible, but the best solution should first settle explicit last-resort guidance because normal click fidelity remains important.
