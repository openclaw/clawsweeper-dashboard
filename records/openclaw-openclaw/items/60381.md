## Summary

Keep open: current main already has the requested gated `evaluate` browser action, but the `force` click option is still missing from the browser action contract, normalizer, CLI, docs, and Playwright forwarding path. The FSB prior-art comments are useful guidance for keeping force click explicit and auditable rather than default behavior.

## What This Changes

The issue requests an additive browser automation escape hatch: `force?: boolean` for click actions and a page/ref JavaScript `evaluate` action.

## Best Possible Solution

Keep the shipped gated evaluate implementation and add explicit opt-in `force?: boolean` click support through the browser schema, request type, normalizer, CLI, docs/skill guidance, tests, and managed Playwright execution path, with clear existing-session behavior.

## Reproduction Assessment

Yes. Source inspection shows a `/act` click payload with `force: true` has no schema, type, normalizer, CLI, or Playwright forwarding path on current main; I did not run a live browser repro because this review is read-only.

## Solution Assessment

Yes for the remaining work. The narrow maintainable path is an additive opt-in click field that forwards to Playwright while preserving default actionability checks and existing evaluate gating; statement-body evaluate should stay in its separate follow-up.
