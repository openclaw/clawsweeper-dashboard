## Summary

Keep open. Current main implements and gates the evaluate half, but the requested Playwright click `force` option still has no schema, type, normalization, executor, CLI, or docs path, so the remaining work is a narrow browser plugin enhancement.

## What This Changes

Add a Playwright-backed `force` option for browser click actions and expose an evaluate action for arbitrary page JavaScript.

## Best Possible Solution

Keep the existing evaluate implementation and gate, then add only `force?: boolean` through the browser act contract, request normalization, Playwright execution, CLI/docs, and focused tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to submit `browser act` with `kind: "click"` and `force: true`; current main has no schema/type/normalizer field for it and never forwards it to Playwright.

## Solution Assessment

Yes for the remaining force-click work. A narrow additive `force?: boolean` on click actions is the maintainable path; using evaluate as the workaround would overuse arbitrary JavaScript and lose Playwright click semantics.
