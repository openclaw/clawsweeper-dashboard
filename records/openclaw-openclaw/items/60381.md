## Summary

Keep open. The evaluate half is already implemented and gated, which also covers the related closed JavaScript-execution request, but current main still does not expose or forward Playwright click `force`, so the remaining work is a narrow browser plugin enhancement.

## What This Changes

The issue requests an additive browser click `force` option and an `evaluate` action for page-context JavaScript execution.

## Best Possible Solution

Keep the existing evaluate implementation and gate as-is, then add only the remaining force-click capability through the browser plugin act contract, request normalization, Playwright execution, CLI/docs, and focused tests.

## Reproduction Assessment

Yes. A high-confidence reproduction path is source-level: submit `browser act` with `kind: "click"` and `force: true`; current main has no schema/type/normalizer field for it and never forwards it to Playwright.

## Solution Assessment

Yes. A narrow additive `force?: boolean` on click actions is the maintainable fix; using `evaluate` as the workaround would broaden reliance on arbitrary JavaScript and would not preserve Playwright click semantics.

Add `force?: boolean` for `kind: "click"` through the browser tool schema, `BrowserActRequest` type, `normalizeActRequest`, `clickViaPlaywright` opts, and the `locator.click` / `locator.dblclick` calls.
