## Summary

Keep open. Current `main` and latest release already expose `evaluate`, but the requested opt-in `force` click path is still missing from the browser action schema, request type, normalizer, Playwright forwarding, CLI/docs, and existing-session behavior.

## What This Changes

Add an explicit browser click `force` option and expose a browser action for page/ref JavaScript evaluation.

## Best Possible Solution

Keep the gated evaluate implementation and add an opt-in `force?: boolean` click contract that threads through schema, types, normalization, Playwright execution, CLI/docs/skill guidance, and focused regression tests.

## Reproduction Assessment

Yes, by source inspection: submit `/act` with `{ "kind": "click", "ref": "1", "force": true }`; current `main` has no schema/type/normalizer field and no Playwright forwarding path for that option. I did not run tests because this was a read-only review.

## Solution Assessment

Yes. The narrow maintainable solution is an additive, default-false click field that forwards to Playwright click/double-click options while keeping evaluate gated and documenting force as an audited fallback.
