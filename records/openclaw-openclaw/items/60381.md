## Summary

Keep open: current main implements the gated browser `act:evaluate` path, but the requested `force?: boolean` click option is still missing from the browser action schema, request type, normalizer, CLI, docs, and Playwright forwarding path.

## What This Changes

The issue requests adding an explicit force option for browser click actions and exposing a browser evaluate action for page-context JavaScript.

## Best Possible Solution

Keep the current gated evaluate implementation and add an explicit opt-in `force?: boolean` click path through the browser plugin contract, CLI, docs, browser-automation skill guidance, and focused tests.

## Reproduction Assessment

Yes. By source inspection, a `/act` click payload with `force: true` has no schema field, typed request field, normalizer field, CLI option, or Playwright forwarding path on current main.

## Solution Assessment

Yes. The maintainable path is an additive opt-in click field that forwards to Playwright click options while preserving evaluate gating and documenting force as a fallback after normal click recovery paths.
