## Summary

Keep open. Current main already has the requested gated `evaluate` action, but the `force` click option is still missing from the browser act schema, request type, normalizer, CLI surface, and Playwright click forwarding path; the May 12 comment also adds useful guidance that force clicks should stay explicit and auditable.

## What This Changes

Requests a `force` option for browser click actions and an `evaluate` action for running page JavaScript.

## Best Possible Solution

Keep the shipped gated evaluate implementation, and add an opt-in `force?: boolean` click path through the browser plugin contract with explicit fallback guidance and focused regression coverage.

## Reproduction Assessment

Yes, by source inspection: submit `/act` with `{ "kind": "click", "ref": "1", "force": true }`; current main has no schema/type/normalizer field and no Playwright forwarding path for that option. I did not run tests because this was a read-only review.

## Solution Assessment

Yes. The narrow maintainable path is an additive opt-in click field that forwards to Playwright click options while preserving evaluate gating and documenting force as a fallback after ordinary click failure.
