## Summary

Keep open. Current main already implements and gates the evaluate half, but the requested opt-in force-click parameter is still absent from the browser act contract, normalizer, CLI/docs, and Playwright click forwarding path.

## What This Changes

The issue requests browser act support for `click.force` plus an exposed browser `evaluate` action.

## Best Possible Solution

Keep the shipped `evaluate` implementation and add the remaining opt-in force-click path through the browser plugin contract, managed Playwright execution, CLI/docs/skill guidance, and focused tests, with explicit existing-session behavior.

## Reproduction Assessment

Yes, by source inspection: submit `browser act` click with `force: true`; current main has no schema/type/normalizer field and no Playwright forwarding path for that option.

## Solution Assessment

Yes. The narrow maintainable path is an additive `force?: boolean` on click actions, keeping evaluate as-is and documenting force as an explicit fallback rather than a default interaction mode.
