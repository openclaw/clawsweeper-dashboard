## Summary

Keep open: the browser tool now exposes `evaluate` on current main and in the latest release, but the requested opt-in `force` click flag is still absent from the tool schema, normalized action type, route normalizer, CLI, docs, and Playwright click forwarding path. The FSB comments are useful prior art for keeping forced clicks explicit and auditable instead of making them the default.

## What This Changes

Requests an additive browser `click` force flag and a browser `evaluate` action for modern frontend automation escape hatches.

## Best Possible Solution

Add a narrow browser-plugin force-click feature that preserves normal clicks as the default, forwards `force: true` only when explicitly requested, documents the fallback guidance, and leaves `evaluate` unchanged.

## Reproduction Assessment

Yes at source level: current main exposes `evaluate`, but the browser click schema, normalized request type, route normalizer, CLI command, docs, and `locator.click` call path have no `force` parameter. I did not run a live browser because this review is read-only.

## Solution Assessment

Yes for the remaining work: the narrowest maintainable fix is to add an explicit `force?: boolean` click field through the browser plugin action pipeline and forward it to Playwright. `evaluate` should not be reworked because it is already implemented and shipped.
