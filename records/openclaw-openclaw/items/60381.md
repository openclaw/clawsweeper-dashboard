## Summary

Keep open. Current main implements and gates the `evaluate` half, but the requested opt-in `force` click parameter is still absent from the browser act schema, request type, normalizer, CLI/docs, and Playwright click forwarding path.

## What This Changes

The issue requests an opt-in `force` parameter for browser click actions and an `evaluate` browser action for JavaScript execution.

## Best Possible Solution

Keep the shipped `evaluate` implementation and add the remaining opt-in force-click path through the browser plugin contract, managed Playwright execution, CLI/docs, and focused regression coverage, with explicit existing-session behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: submit browser act click with `force: true`; current main has no schema/type/normalizer field for it and no Playwright forwarding path.

## Solution Assessment

Yes. A narrow additive `force?: boolean` on click actions is the maintainable remaining fix; using `evaluate` as the workaround would broaden arbitrary JavaScript reliance and would not preserve Playwright click semantics.
