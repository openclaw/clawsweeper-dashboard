## Summary

Keep open. Current main implements and gates the evaluate half, but the requested force-click parameter is still absent from the browser action contract, normalizer, managed Playwright path, existing-session path, CLI, and docs; the latest comment also adds an explicit audit/normal-click-first constraint that needs maintainer judgment.

## What This Changes

The issue requests an opt-in force parameter for browser click actions and an evaluate action for browser-page JavaScript execution.

## Best Possible Solution

Keep the shipped evaluate action and add force-click as an explicit audited fallback through the browser action contract, managed Playwright path, existing-session handling, CLI/docs, and focused tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: send `browser act` click with `force: true`; current main has no schema/type/normalizer field that preserves or forwards it to Playwright.

## Solution Assessment

Yes in direction, but the exact implementation needs maintainer judgment. A narrow opt-in click field is maintainable, provided it records why normal click failed and does not silently become the default path.
