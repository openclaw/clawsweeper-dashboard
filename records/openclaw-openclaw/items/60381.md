## Summary

Keep open. Current main already has the requested evaluate action, but the force-click half is still missing from the browser act schema, request type, normalizer, CLI/docs, and Playwright click forwarding path; the latest human comment also adds useful guidance that force clicks should be explicit and auditable, not default behavior.

## What This Changes

The issue requests an additive browser click `force` option plus an `evaluate` browser action for page-context JavaScript execution.

## Best Possible Solution

Keep the shipped gated evaluate implementation, and add an opt-in `force?: boolean` click path through the browser plugin contract, normalizer, Playwright forwarding, CLI/docs/skill guidance, and focused regression tests.

## Reproduction Assessment

Yes, by source inspection: submit `/act` with `{ "kind": "click", "ref": "1", "force": true }`; current main has no schema/type/normalizer field and no Playwright forwarding path for that option. I did not run tests because this was a read-only review.

## Solution Assessment

Yes. The narrow maintainable path is an additive opt-in click field that forwards to Playwright `locator.click`/`dblclick` while preserving evaluate gating and documenting force as a fallback after ordinary click failure.
