## Summary

Keep open: current main and the latest release already expose browser `evaluate`, but the requested opt-in `force` click flag is still absent from the browser action schema, normalized request type, route normalizer, CLI/docs surface, and Playwright click forwarding path. The FSB comments are useful prior art for keeping forced clicks explicit and auditable instead of making them the default.

## What This Changes

Request to add an explicit `force?: boolean` option for browser click actions and expose page JavaScript evaluation through the browser tool.

## Best Possible Solution

Add only the remaining explicit force-click capability through the browser plugin action pipeline, keep normal clicks as the default, document/audit the fallback guidance, and leave the already-shipped evaluate path unchanged.

## Reproduction Assessment

Yes, at source level: current main exposes `evaluate`, but the browser click schema, normalized request type, route normalizer, CLI/docs surface, and `locator.click` call path have no `force` parameter. I did not run a live browser because this review is read-only.

## Solution Assessment

Yes for the remaining work: the narrowest maintainable fix is an explicit `force?: boolean` click field carried through the browser plugin pipeline and forwarded to Playwright only when requested. Reworking `evaluate` is not the best path because it is already implemented and shipped.
