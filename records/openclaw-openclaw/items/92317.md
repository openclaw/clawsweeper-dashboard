- `impact:session-state`: The report concerns incomplete visibility of prior assistant messages and tool results within the current session view.

## Summary

Current main still exposes only a bounded recent-history tail to WebChat: the Control UI requests 200 messages and the gateway caps the request at 1,000, with additional byte and text projection limits. There is no cursor or prepend path for older messages, so the report remains necessary; resolving it safely requires maintainers to define a bounded pagination and scroll-anchoring contract rather than merely increasing limits.

## What This Changes

Allow users to retrieve older assistant replies and tool outputs from the active WebChat transcript beyond the current capped history tail.

## Best Possible Solution

Add a bounded backward cursor to `chat.history` with page metadata, then make Control UI prepend older projected messages on top-scroll or an explicit load action while preserving scroll position and all existing payload safety caps.

## Reproduction Assessment

Yes. Current-main source deterministically shows that WebChat requests a recent 200-message tail and has no cursor, offset, or load-older path; gateway tests confirm additional truncation boundaries, although no browser run was performed.

## Solution Assessment

No, a simple limit increase is not the best solution because it preserves the same failure mode while increasing payload and rendering costs. Bounded backward pagination with retained scroll position is the narrowest maintainable direction, subject to a maintainer UX and API decision.
