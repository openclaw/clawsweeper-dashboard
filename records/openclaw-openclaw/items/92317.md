## Summary

Keep open: current main and the latest release still bound WebChat to a recent history window, while the active pagination fix has not merged and still needs maintainer review.

## What This Changes

The issue asks WebChat/Control UI to make older assistant replies and tool outputs reachable in long conversations by retaining or loading more scrollback.

## Best Possible Solution

Land or replace the active bounded-pagination implementation so WebChat can load older transcript pages without increasing unbounded history payloads.

## Reproduction Assessment

Yes, at source level: current WebChat requests only 100 messages, the renderer caps visible history at 100/240k chars, and current `chat.history` has no older-page cursor. I did not run a live browser session in this read-only review.

## Solution Assessment

Yes, bounded pagination is the best direction; simply raising the limit would keep the same failure mode and increase payload/render cost. The active PR is the current implementation path, but it still needs maintainer review before this issue can close.
