## Summary

Current main still exposes only a bounded recent WebChat history window, while the matching pagination PR remains open, conflicted, mixed with unrelated silent-reply changes, and missing real-browser proof; this issue should remain the canonical user problem until maintainers repair, replace, or close that implementation path.

## What This Changes

Add paginated WebChat scrollback so users can retrieve older assistant replies and tool outputs from long sessions.

## Best Possible Solution

Choose one canonical bounded cursor contract, then land a focused WebChat paging implementation with stable transcript identity, strict non-overlapping cursors, deduplication, scroll-anchor preservation, and real-browser proof on a long tool-heavy session.

## Reproduction Assessment

Yes, from source with high confidence: current WebChat requests a bounded 100-message history page and can render a passive hidden-history notice without a paging action. No live browser reproduction was performed during this read-only review.

## Solution Assessment

No, the currently proposed mixed-scope PR is not yet the best landing shape. Pagination is the right product direction, but maintainers should first choose the canonical HTTP or WebSocket contract and isolate it from unrelated silent-reply changes.
