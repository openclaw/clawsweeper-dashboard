## Summary

Keep open: current main and v2026.6.6 still expose only a bounded recent WebChat history window, and the active pagination PR has not merged and still needs maintainer review.

## What This Changes

WebChat should let users reach older assistant replies and tool outputs in long conversations instead of stopping at a small recent-history window.

## Best Possible Solution

Land or replace the active bounded `chat.history`/WebChat older-history pagination implementation, preserving payload budgets and scroll position instead of raising the fixed limit.

## Reproduction Assessment

Yes at source level: WebChat asks for 100 recent messages, the UI can expand only within the loaded/renderable window, and `chat.history` has no older-page cursor on current main. I did not run a live browser reproduction in this read-only review.

## Solution Assessment

Yes on direction: bounded pagination is the maintainable fix shape because raising the limit preserves the same failure mode and increases payload/render cost. The exact PR/API contract still needs maintainer selection because multiple open PRs touch the same surface.
