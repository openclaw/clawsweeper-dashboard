## Summary

Keep open: current main and the latest release still expose only a bounded recent WebChat history window, while the active pagination implementation remains an open maintainer-labeled PR.

## What This Changes

The issue requests that WebChat make older assistant replies, tool outputs, and conversation context accessible when users scroll upward in long chats.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/87111 or a maintainer-approved bounded cursor-pagination replacement that lets WebChat load older transcript pages while preserving payload and render caps.

## Reproduction Assessment

Yes at source level: WebChat requests only a 100-message page, the WebSocket `chat.history` schema has no older-page cursor, and current tests still encode the hidden-history notice. I did not run a live browser session in this read-only review.

## Solution Assessment

Yes directionally: bounded cursor pagination with a WebChat load-older control is more maintainable than raising the limit because it preserves payload caps. The current implementation path is the open PR, which still needs maintainer review before this issue can close.
