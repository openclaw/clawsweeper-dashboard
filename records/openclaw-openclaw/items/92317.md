## Summary

Keep open as the canonical WebChat scrollback item: current main and the latest release still expose only a bounded recent history window, and the active fix path is an open pagination PR that still needs maintainer handling.

## What This Changes

The issue requests that WebChat / Control UI retain or load enough long-conversation history for users to scroll back to earlier assistant replies and tool outputs.

## Best Possible Solution

Keep this issue open as the canonical report and resolve it through https://github.com/openclaw/openclaw/pull/87111 or a maintainer-approved bounded pagination replacement that preserves payload and render safety caps.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.6 request only the latest 100 chat.history entries, render a bounded tail, and expose no chat.history cursor or older-page load path for WebChat scrolling.

## Solution Assessment

Yes for the current direction, but not as a blind limit increase. Bounded cursor/page-based loading, as in the active WebChat pagination PR, is the maintainable solution because it preserves the existing payload and render safety goals.
