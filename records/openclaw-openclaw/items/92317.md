## Summary

Keep open: current main and v2026.6.8 still expose only a bounded recent WebChat history window, while the active pagination fix remains open and needs maintainer review.

## What This Changes

The issue requests WebChat scrollback that can retain or load older assistant replies, tool outputs, and prior context in long Control UI conversations.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/87111 or a maintainer-approved bounded cursor-pagination replacement that lets WebChat load older transcript pages while preserving payload and render caps.

## Reproduction Assessment

Yes at source level: WebChat requests only a 100-message history page, the WebSocket `chat.history` schema has no older-page cursor, and current tests still encode the hidden-history notice. I did not run a live browser session in this read-only review.

## Solution Assessment

Yes directionally: bounded cursor pagination with a WebChat load-older control is better than raising the limit because it preserves payload caps. The current implementation path is the open PR, which still needs maintainer review before this issue can close.
