## What This Changes

Request WebChat to load or page older conversation history so earlier assistant replies and tool outputs remain accessible in long sessions.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/87111 or a maintainer-approved bounded cursor-pagination replacement that lets WebChat load older transcript pages while preserving payload and render caps.

Yes, at source level: current main requests one fixed bounded chat.history page, the WebSocket schema has no older-page cursor, and renderer tests still expect a passive hidden-count notice. I did not run a live browser transcript in this read-only review.

## Solution Assessment

Yes directionally: bounded cursor pagination with a WebChat load-older control is more maintainable than raising limits because it preserves payload and render caps. The current path is the open PR, which still needs maintainer review and merge readiness before this issue closes.
