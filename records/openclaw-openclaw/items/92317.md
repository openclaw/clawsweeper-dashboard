## Summary

Keep open: current main and v2026.6.6 still bound WebChat to a recent history window with no WebSocket older-page cursor, while the active bounded-pagination PR is open, conflicting, and still needs maintainer session-state review.

## What This Changes

The issue requests that WebChat retain or load enough current-session history to make older assistant replies and tool outputs accessible when scrolling upward.

## Best Possible Solution

Land or replace the active bounded-pagination implementation so WebChat can load older transcript pages without unbounded history payloads or a parallel history source of truth.

## Reproduction Assessment

Yes, at source level: current WebChat requests only a 100-message bounded history page, the WebSocket chat.history schema has no older-page cursor, and tests assert hidden older history. I did not run a live browser reproduction in this read-only review.

## Solution Assessment

Yes, bounded pagination or an explicit load-older flow is the best fix direction; raising the fixed cap would increase payload and render cost while preserving the same failure mode. The current implementation path is the open pagination PR, but it needs maintainer review and conflict repair before this issue can close.
