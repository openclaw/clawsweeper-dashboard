## Summary

Keep open: current main and the latest release still expose only a bounded recent WebChat history window, while the active pagination fix is open, conflicting, and maintainer-labeled for review.

## What This Changes

Request that WebChat/Control UI retain or page older current-session assistant replies and tool outputs so long conversations remain reviewable from the chat scrollback.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/87111 with an additive bounded chat.history pagination and load-older implementation that preserves payload caps and scroll position.

## Reproduction Assessment

Yes. Source inspection shows WebChat requests only 100 history messages, the renderer caps the visible tail, and the WebSocket chat.history schema has no older-page cursor on current main.

## Solution Assessment

Yes in direction, not yet in the repository. Bounded server-side pagination plus a WebChat load-older control is the maintainable solution; raising the fixed limit would keep the same failure mode and increase payload/render cost.
