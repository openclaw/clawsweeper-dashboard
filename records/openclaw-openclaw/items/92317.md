## Summary

Keep open: current main and v2026.6.6 still bound WebChat to a recent history window with no older-page cursor, while the active pagination fix is still open and needs maintainer review.

## What This Changes

Request that WebChat retain or load enough scrollback to revisit older assistant replies and tool outputs in long conversations.

## Best Possible Solution

Land or replace the open bounded-pagination implementation so WebChat can load older transcript pages while preserving current payload caps and active-branch history rules.

## Reproduction Assessment

Yes, at source level: current main still requests only 100 WebChat history messages and the WebSocket history schema has no cursor for older-page loading. I did not run a live browser repro in this read-only review.

## Solution Assessment

Yes on direction: bounded pagination is the maintainable shape; raising the fixed limit would increase payload/render cost while preserving the same failure mode. The current concrete path is the open pagination PR, which still needs maintainer review.
