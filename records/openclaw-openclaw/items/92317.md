## Summary

Keep open: current main and v2026.6.6 still use bounded WebChat history with only a passive hidden-history notice, while the canonical pagination PR remains open, conflicting, and under maintainer review.

## What This Changes

The issue requests WebChat scrollback pagination or equivalent load-older behavior so long conversations keep older assistant replies and tool outputs accessible.

## Best Possible Solution

Land a repaired canonical pagination implementation, or replace it with a narrower maintainer-approved design that loads older WebChat history pages while keeping live WebSocket streaming and bounded rendering intact.

## Reproduction Assessment

Yes, at source level: current main requests only a bounded 100-message WebChat history window, caps rendered history, and has no older-page cursor for the WebSocket history method. I did not run a live browser transcript repro in this read-only review.

## Solution Assessment

No, the current repository state is not yet the best solution because the active pagination PR has not merged and still needs maintainer review. Bounded pagination is the right direction; simply raising limits would keep the same failure mode and increase payload/render cost.
