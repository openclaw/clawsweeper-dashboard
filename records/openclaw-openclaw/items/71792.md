## Summary

Keep open. Current main still leaves automatic daily/idle rollover on the session lifecycle path while bundled session-memory only listens to command reset/new events, and this PR is a plausible focused fix; it still needs real behavior proof and maintainer selection against the overlapping implementation at https://github.com/openclaw/openclaw/pull/61675.

## What This Changes

The PR emits synthetic internal `command:reset` hooks for automatic daily/idle session rollovers, adds daily/idle/fresh-session tests, and updates memory hook docs plus changelog entries.

## Best Possible Solution

Land this or a maintainer-approved replacement that persists session-memory exactly once for automatic daily and idle rollovers while preserving explicit reset behavior and existing session lifecycle hooks.

## Reproduction Assessment

Yes, by source inspection. Seed a stale daily or idle session and call `initSessionState` on current main: it rolls the session and emits lifecycle hooks, while bundled session-memory remains gated to `command:new` and `command:reset`.

## Solution Assessment

Mostly yes. Reusing the existing internal `command:reset` hook is a narrow fix for bundled session-memory, but maintainers still need to choose it over the broader before_reset/session-hook shape proposed in https://github.com/openclaw/openclaw/pull/61675.
