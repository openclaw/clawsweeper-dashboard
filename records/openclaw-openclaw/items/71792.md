## Summary

Keep open. Current main still leaves automatic daily/idle rollovers on the session lifecycle path while bundled session-memory only listens to command hooks; this PR is a plausible narrow fix, but it still needs contributor real-behavior proof and maintainer selection against the overlapping implementation at https://github.com/openclaw/openclaw/pull/61675.

## What This Changes

The branch emits a synthetic `command:reset` internal hook with `commandSource` set to `auto:daily` or `auto:idle` during automatic stale rollover, adds daily/idle/fresh-session coverage, and updates session-memory docs and changelog text.

## Best Possible Solution

Land this or one maintainer-approved replacement that persists bundled session-memory exactly once for automatic daily and idle rollovers while preserving manual reset behavior, transcript archival, and existing session lifecycle hooks.

## Reproduction Assessment

Yes, by source inspection. Seed an existing session that becomes stale by daily or idle policy and call `initSessionState`; current main starts a new session and lifecycle hooks while bundled session-memory remains limited to `command:new` and `command:reset`.

## Solution Assessment

Mostly yes. Reusing the existing internal `command:reset` event is narrow for bundled session-memory, but maintainers should deliberately choose it over the broader `before_reset` and command-action shape proposed in https://github.com/openclaw/openclaw/pull/61675.
