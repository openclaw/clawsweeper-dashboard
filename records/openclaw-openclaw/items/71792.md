## What This Changes

The PR emits a synthetic internal `command:reset` hook for automatic daily/idle stale session rollovers, adds daily/idle/fresh-session tests, and updates session-memory docs plus changelog entries.

## Best Possible Solution

Land this or a maintainer-approved replacement that persists session-memory exactly once for automatic daily and idle rollovers while preserving manual reset behavior and the existing session lifecycle hooks.

## Reproduction Assessment

Yes, by source inspection. Seed a stale daily or idle session and call `initSessionState` on current main: it creates a new session and emits lifecycle hooks, while bundled session-memory remains gated to `command:new` and `command:reset`.

## Solution Assessment

Mostly yes. Reusing the existing internal `command:reset` event is narrow for bundled session-memory, but maintainers need to choose it over the broader before_reset/session-hook contract proposed in https://github.com/openclaw/openclaw/pull/61675.
