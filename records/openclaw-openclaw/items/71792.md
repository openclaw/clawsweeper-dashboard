## Summary

Keep this PR open: it targets a real current-main session-memory gap and has focused coverage, but it still needs contributor real-behavior proof and one docs correction before merge readiness.

## What This Changes

The PR emits a synthetic internal `command:reset` hook for automatic daily/idle session rollover, adds daily/idle/fresh-session tests, and updates session-memory docs plus changelog text.

## Best Possible Solution

Land one maintainer-approved implementation that persists session-memory exactly once for automatic daily and idle rollovers, preserves manual reset behavior, and documents the actual timestamp-vs-LLM-slug filename contract.

## Reproduction Assessment

Yes, by source inspection. Seed a stale daily or idle session and call `initSessionState` on current main: it creates a new session and lifecycle hooks while bundled session-memory remains gated to `command:new` and `command:reset`.

## Solution Assessment

Unclear as merge-ready. The code path is narrow and plausible, but the docs need alignment with current filename behavior and maintainers should choose it deliberately against the overlapping before_reset approach in https://github.com/openclaw/openclaw/pull/61675.
