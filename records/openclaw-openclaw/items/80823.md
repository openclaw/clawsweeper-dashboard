## Summary

Keep this PR open: the linked CLI hint bug is still real on current main and the branch is useful, but the proposed helper can under-warn for plugin-owned restart-required config paths because standalone config commands do not load plugin reload registrations.

## What This Changes

This PR changes `config set`, `config patch`, and `config unset` success text to derive restart or hot-reload hints from the gateway reload planner and adds CLI regression tests.

## Best Possible Solution

Land after the CLI hint helper either uses gateway-equivalent plugin reload metadata or conservatively keeps restart hints for plugin-owned paths, with focused regression coverage and a refresh onto current main.

## Reproduction Assessment

Yes. Source inspection shows current main always prints the restart hint, while the PR’s planner call can miss plugin restart prefixes because config commands do not load the active plugin registry.

## Solution Assessment

No, not yet. Delegating to shared reload metadata is the right direction for core paths, but the PR needs a conservative plugin-owned path fallback or equivalent plugin reload metadata before merge.
