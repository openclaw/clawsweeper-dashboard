## Summary

Keep this PR open, but the current patch is not merge-ready: it reuses the reload planner from the actual config diff, which is the right layer, but it ignores `gateway.reload.mode` and can tell existing `off` or `restart` mode users that hot-path changes apply without a gateway restart.

## What This Changes

The PR changes `config set`, `config patch`, and direct `config unset` success text to derive restart, hot-reload, or no-restart hints from the actual config diff and gateway reload planner.

## Best Possible Solution

Keep the actual-diff planner integration, but make the CLI hint mode-aware before merge; leave the broader machine-readable post-apply status contract to https://github.com/openclaw/openclaw/issues/83851.

## Reproduction Assessment

Yes. Source inspection shows PR head maps hot reload-plan results to `without restarting` while current gateway reload code gates actual apply behavior through `gateway.reload.mode`.

## Solution Assessment

No, not yet. Reusing actual config diffs and `buildGatewayReloadPlan()` is the right maintainable layer, but the hint must also account for `resolveGatewayReloadSettings()` before it is the best fix.
