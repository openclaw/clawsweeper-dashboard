## Summary

This is a plausible narrow fix for the misleading CLI restart hint and the PR body includes sufficient terminal proof, but the current patch is not merge-ready because it introduces two concrete CI blockers in the changed files.

## What This Changes

The PR changes `config set` and `config unset` success hints for active `agents.list[...]` mutations and adds focused config CLI tests.

## Best Possible Solution

Keep the narrow hint-classification approach, fix the mechanical CI blockers, then let maintainers validate the dormant `agentRuntime.id` restart policy before landing.

## Reproduction Assessment

Yes. Current main source unconditionally prints the restart hint for config writes, while the gateway reload plan and tests show `agents.list` changes are hot reloads rather than gateway restarts.

## Solution Assessment

Mostly yes. A post-mutation classifier is a maintainable way to narrow the CLI wording, but this patch first needs its test fixture and lint violations fixed.
