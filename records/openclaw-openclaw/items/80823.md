## What This Changes

The PR adds config CLI hint selection and tests so hot-loadable active `agents.list` mutations get a next-invocation hint while restart-required paths keep the gateway restart hint.

## Best Possible Solution

Land the narrow CLI hint change after maintainer approval, preserving the conservative restart hint for non-agent, `agentRuntime`-touching, dormant-runtime, and mixed-batch mutations.

## Reproduction Assessment

Yes. Current main source always appends the restart hint on config set/patch/unset success paths while `agents.list` is hot-reload scoped, and the linked issue plus PR body include terminal proof of active-agent behavior.

## Solution Assessment

Yes. The PR changes only post-write success-message hint selection, computes the decision from the post-mutation config, and defaults to the existing restart hint when it cannot positively classify the operation as hot-loadable.
