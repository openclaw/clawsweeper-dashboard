## What This Changes

The PR adds config CLI hint selection and tests so hot-loadable active-agent config mutations get a next-invocation hint while restart-required paths keep the gateway restart hint.

## Best Possible Solution

Land the narrow CLI hint change after normal maintainer approval, preserving the restart hint for non-agent, mixed-batch, dormant-runtime, and `agentRuntime`-touching mutations.

## Reproduction Assessment

Yes. Current main source still appends the restart hint unconditionally for config set/patch/unset success paths, while the linked issue and PR body show active-agent model changes hot-load on later invocation without a gateway restart.

## Solution Assessment

Yes. The PR changes only success-message hint selection after computing the post-mutation config and defaults back to the existing restart hint unless every operation is positively classified as hot-loadable.
