The PR changes `config set`, `config patch`, and direct `config unset` success text to derive restart or hot-apply hints from actual config diffs and the gateway reload planner, with focused CLI and gateway tests.

## Best Possible Solution

Land this narrow planner-backed CLI hint fix, or a maintainer-approved equivalent, while leaving the broader machine-readable apply-status contract to https://github.com/openclaw/openclaw/issues/83851.

## Solution Assessment

Yes. Reusing `buildGatewayReloadPlan()` from the CLI is the narrowest maintainable fix compared with adding a suppression flag or duplicating per-agent heuristics, and the PR preserves restart-backed hints for broad and plugin-owned cases that remain unsafe to downgrade.
