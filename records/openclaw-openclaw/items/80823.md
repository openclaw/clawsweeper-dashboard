## Summary

Keep open: the PR fixes the stale CLI hint for agent/model config paths and includes strong terminal proof, but the new helper can under-warn plugin-owned restart-required config paths because the CLI does not install the active plugin reload registry used by the gateway.

## What This Changes

The PR changes `config set`, `config patch`, and `config unset` success text to derive restart/hot-reload hints from `buildGatewayReloadPlan()` and adds CLI regression tests for hot, restart-required, mixed, and legacy runtime paths.

## Best Possible Solution

Land after the CLI hint path either uses the same plugin reload metadata as the gateway or conservatively keeps restart hints for plugin-owned paths, with regression coverage for both hot agent/model paths and restart-required plugin paths.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main hard-codes restart hints for config writes while the gateway planner classifies agent/model paths as hot; the PR also supplies real terminal output for the target CLI behavior. The remaining review finding is source-reproducible from the planner's active-registry dependency plus Canvas's restart prefix.

## Solution Assessment

No, not quite: delegating to the shared planner is the right direction for core hot-loadable paths, but the CLI must not use a planner state that lacks plugin reload registrations. The safest repair is to use gateway-equivalent metadata or fall back to restart for plugin-owned paths.

Full review comments:

- **[P1] Keep plugin restart prefixes conservative in CLI hints:** `src/cli/config-cli.ts:960`
  - body: This helper calls `buildGatewayReloadPlan()` from the config CLI, but plugin restart prefixes come from the active plugin registry. That would make `openclaw config set plugins.entries.canvas...` print that the change applies without restarting even though Canvas declares that prefix restart-required. Please load/pass equivalent reload metadata or conservatively keep restart hints for plugin-owned paths, with a regression test for this case.
