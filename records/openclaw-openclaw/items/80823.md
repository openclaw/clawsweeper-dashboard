## Summary

Keep open: current main still has the misleading unconditional restart hint, while this PR is a narrow tested fix with sufficient terminal proof and no blocking review findings.

## What This Changes

This PR changes the config CLI and tests so hot-loadable active-agent config mutations get a next-invocation hint while restart-required paths keep the gateway-restart hint.

## Best Possible Solution

Land the focused CLI hint change after maintainer approval, keeping the restart hint as the conservative fallback for non-agent, `agentRuntime`-touching, dormant-runtime, and mixed-batch mutations.

## Reproduction Assessment

Yes. Source inspection shows current main unconditionally emits the restart hint on config set/patch/unset success paths, while the reload plan and linked terminal proof show active `agents.list` model changes can apply on the next invocation.

## Solution Assessment

Yes. The PR computes the hint from the post-mutation config, keeps write semantics unchanged, and defaults to the existing restart wording unless every touched path is positively classified as hot-loadable.
