## Summary

Keep open: this is a focused fix with useful proof, but the merge result now conflicts with current main's runtime-policy contract by treating legacy whole-agent `agents.list[].agentRuntime.id` as restart-required even though current docs and runtime selection say that key is ignored.

## What This Changes

The PR adds config CLI hint-selection logic and tests so hot-loadable `agents.list` mutations print a next-invocation hint while restart-required paths keep the gateway restart hint.

## Best Possible Solution

Align the hint classifier and tests with current provider/model-scoped runtime policy, or get an explicit maintainer decision that legacy whole-agent runtime keys should still force restart guidance despite current docs.

## Reproduction Assessment

Yes from source, not local execution: current main unconditionally logs the restart hint after config mutations, and the linked issue plus PR body show the misleading active-agent case in a real CLI profile.

## Solution Assessment

No as written: the narrow hint split is the right direction, but the classifier should not encode legacy whole-agent `agentRuntime` as restart-required while current main says runtime selection ignores it.
