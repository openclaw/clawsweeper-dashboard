## Summary

Keep open: the branch is now focused and useful, but it should not merge yet because it plans CLI apply hints from requested paths rather than actual changed paths, which can understate restart requirements for broad config writes.

## What This Changes

The PR changes `config set`, `config patch`, and `config unset` success text to derive gateway restart/hot-reload hints from `buildGatewayReloadPlan()`, with focused CLI and gateway reload tests.

## Best Possible Solution

Compute the CLI apply hint from the actual changed dotted paths produced by the write, preserving conservative restart wording whenever a broad parent write contains restart-owned nested config.

## Reproduction Assessment

Yes. Source inspection shows current main hard-codes generic restart hints for config writes while the planner/docs classify agent/model paths as hot, and the PR-introduced broad-parent false-hot case is also source-reproducible from the requested-path planner input.

## Solution Assessment

No for the current patch shape. Reusing the shared reload planner is the right direction, but the helper must feed it actual changed paths or stay conservative for broad parents that can contain restart-owned children.
