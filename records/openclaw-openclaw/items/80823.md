## Summary

Keep open: this PR addresses a real CLI restart-hint bug that current main still has, but the patch is not merge-ready because the new hot/no-restart hint ignores the configured reload mode and can mislead existing off-mode setups.

## What This Changes

The PR changes CLI config mutation success messages to derive restart/no-restart hints from gateway reload planning and adds CLI/gateway regression tests.

## Best Possible Solution

Keep the planner-based hint direction, but make the CLI hint account for reload mode so path classification does not overstate actual apply behavior for existing off/restart-mode configurations.

## Reproduction Assessment

Yes. Source inspection on current main shows generic restart hints in the CLI write paths while the gateway planner and docs classify agent/model config as hot-loadable; the PR-specific reload-mode gap is also source-reproducible from the path-only helper and existing off-mode contract.

## Solution Assessment

No. Reusing `buildGatewayReloadPlan()` is the right direction, but this implementation should also account for reload mode before claiming a change will apply without a gateway restart.
