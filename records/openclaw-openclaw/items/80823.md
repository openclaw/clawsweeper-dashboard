## Summary

Keep open for maintainer review: current main still has the misleading generic CLI restart hint, this PR is the active fix candidate, and source review found no blocking patch defect, but the change is compatibility-sensitive because CLI restart guidance affects operator decisions.

## What This Changes

The PR changes `config set`, `config patch`, and `config unset` success hints to derive restart, hot-apply, or no-restart text from actual changed config paths and the gateway reload planner, with CLI and gateway regression tests.

## Best Possible Solution

Land the planner-backed CLI hint implementation after maintainer accepts the operator-guidance compatibility risk and checks remain clean, while leaving the broader machine-readable apply-status contract to https://github.com/openclaw/openclaw/issues/83851.

## Reproduction Assessment

Yes. Current main's `runConfigOperations` and direct `runConfigUnset` still print the generic restart hint, while the reload planner and docs classify the reported agent/model paths as hot-applying.

## Solution Assessment

Yes. Reusing actual config diffs plus `buildGatewayReloadPlan()` is the best narrow fix; an agentRuntime-specific CLI heuristic or suppression flag would duplicate reload policy and drift from gateway behavior.
