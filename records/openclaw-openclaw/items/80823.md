## Summary

Keep open: the PR is the active fix candidate for the linked CLI restart-hint bug, but the latest patch still leaves direct `config unset` using requested-root hint planning, which can under-warn for broad unsets that remove restart-owned nested config.

## What This Changes

The PR changes CLI config set, patch, and unset success text to derive gateway restart hints from reload planning, with CLI and gateway regression tests for hot-loadable and restart-required config paths.

## Best Possible Solution

Have every post-write CLI hint, including direct `config unset`, derive from actual before/after changed paths; leave the broader machine-readable apply-status contract to https://github.com/openclaw/openclaw/issues/83851.

## Reproduction Assessment

Yes. Source inspection shows the new direct `config unset` hint uses only the requested path, while existing reload rules require nested actual paths to distinguish hot roots from restart-owned children; I did not run tests because this was a read-only review.

## Solution Assessment

No. Reusing the reload planner is the right fix direction, but direct `config unset` needs the same actual before/after diff planning already added for set and patch.
