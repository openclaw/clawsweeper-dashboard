## Summary

Keep open: the PR addresses a real CLI hint bug, but the new helper can still soften the restart hint for whole agent-entry mutations that implicitly remove or replace `agentRuntime`. That needs a narrow fix before merge.

## What This Changes

The PR adds config CLI hint selection and tests so active `agents.list[N]` mutations use a next-invocation hint while restart-required paths keep the gateway restart hint.

## Best Possible Solution

Land this after narrowing the helper to only positively classify descendant active-agent config paths that cannot implicitly change `agentRuntime`, with regression coverage for whole entry replacement/removal.

## Reproduction Assessment

Yes. Source inspection shows current main always appends the restart hint on config set/patch/unset success paths, while the linked issue and PR body include terminal proof that active-agent model changes can apply on the next invocation without a gateway restart.

## Solution Assessment

No. The intended direction is right, but the proposed helper needs a conservative guard for whole `agents.list[N]` parent mutations that can implicitly change `agentRuntime` without naming that segment.

## Review Findings
