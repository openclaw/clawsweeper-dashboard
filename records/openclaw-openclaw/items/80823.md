## What This Changes

The PR adds config CLI hint-selection logic and tests so hot-loadable `agents.list` mutations get a next-invocation hint while restart-required paths keep the gateway restart hint.

## Best Possible Solution

Land the narrow CLI hint change after normal maintainer approval, preserving the conservative restart hint for non-agent, `agentRuntime`-touching, dormant-runtime, and mixed-batch mutations.

## Reproduction Assessment

Yes. Source inspection shows current main always appends the restart hint on config set/patch/unset success paths while `agents.list` is already hot-reload scoped, and the linked issue plus PR body include terminal proof of the active-agent behavior.

## Solution Assessment

Yes. The PR changes only success-message hint selection after computing the post-mutation config, keeps write semantics unchanged, and defaults to the existing restart hint when it cannot positively classify the operation as hot-loadable.
