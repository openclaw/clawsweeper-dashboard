## Summary

Keep open: the PR fixes the misleading agent/model restart hint direction, but the current helper can misclassify plugin-owned restart-required config paths because config commands do not load plugin reload metadata before calling the planner.

## What This Changes

The PR changes `config set`, `config patch`, and `config unset` success text to use `buildGatewayReloadPlan()` and adds CLI tests for hot, restart-required, and mixed config writes.

## Best Possible Solution

Keep the shared planner approach, but make CLI hints conservative or metadata-aware for plugin-owned paths and prove a restart-owned plugin path with the real planner before merge.

## Solution Assessment

No, not yet. Reusing the shared planner is the right direction for core paths, but the PR needs conservative handling or metadata loading for plugin-owned restart prefixes before this is the best mergeable fix.
