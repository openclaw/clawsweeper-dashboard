## Summary

Keep open. The underlying CLI warning bug is valid and the PR includes real terminal proof, but the current patch should not merge because the new unset test uses an unsupported `model.fallback` field instead of the typed `fallbacks` field, and the legacy `agents.list[].agentRuntime` policy still needs maintainer confirmation against current runtime docs.

## What This Changes

The PR changes CLI config write hints so active `agents.list[...]` mutations can say they apply on next agent invocation while restart-required paths keep the gateway restart hint.

## Best Possible Solution

Land a narrow CLI hint fix that follows the current gateway reload and runtime-policy contracts, with tests covering real `model.primary` and `model.fallbacks` paths plus restart-required non-agent or runtime-policy mutations.

## Reproduction Assessment

Yes. Current main source unconditionally prints the gateway restart hint for config set/unset, and the linked issue plus reload-plan code show active `agents.list` model changes can take effect without a gateway restart.

## Solution Assessment

No, not quite. The helper shape is a reasonable narrow fix, but the patch's new unset test uses the unsupported singular `fallback` field and the legacy `agentRuntime` restart distinction should be reconciled with current runtime-policy docs before merge.
