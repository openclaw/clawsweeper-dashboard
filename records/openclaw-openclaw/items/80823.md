## What This Changes

The PR adds config CLI hint selection for set/patch/unset so active-agent config mutations under agents.list can show a next-invocation hint while restart-required paths keep the gateway restart hint.

## Best Possible Solution

Land the narrow CLI hint change after maintainer approval, preserving conservative restart wording for anything not positively classified as hot-loadable.

## Reproduction Assessment

Yes. Source inspection shows current main unconditionally appends the restart hint on config set/patch/unset success paths while `agents.list` is already hot-reload scoped, and the linked issue plus PR body include terminal proof of the active-agent behavior.

## Solution Assessment

Yes. The PR changes only post-write success-message hint selection based on the post-mutation config, keeps write semantics unchanged, and defaults to the existing restart hint when it cannot positively classify an operation as hot-loadable.
