## Best Possible Solution

## Reproduction Assessment

Yes. Source inspection shows current main still hardcodes restart hints in `config set`, `config patch`, and direct `config unset`, while the planner and docs classify agent/model paths as hot-applying.

## Solution Assessment

Yes. Reusing `buildGatewayReloadPlan()` is the narrow maintainable fix because it avoids a duplicate CLI classifier, and the plugin-entry guard avoids over-promising when plugin reload metadata is not available to the CLI.
