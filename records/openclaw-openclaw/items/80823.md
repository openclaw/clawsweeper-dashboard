## Summary

Keep open: the PR is useful and the shared reload-planner direction is right, but the latest diff still can under-warn restart-required plugin-owned config paths when the standalone config CLI has not loaded plugin reload metadata.

## What This Changes

The PR changes `config set`, `config patch`, and `config unset` success messages to choose restart/no-restart hints from `buildGatewayReloadPlan()` and adds CLI regression tests for core agent, model, and gateway paths.

## Best Possible Solution

Land after plugin-owned config paths either get gateway-equivalent reload metadata in the CLI path or conservatively keep restart hints, with a focused regression test for a restart-required plugin path.

## Reproduction Assessment

Yes. Source inspection shows current `main` always prints the generic restart hint, while the PR’s planner call can miss plugin restart prefixes because standalone config commands do not load the active plugin registry.

## Solution Assessment

No, not yet. Delegating to shared reload metadata is the right shape for core paths, but the PR needs a conservative plugin-owned fallback or equivalent plugin reload metadata before merge.
