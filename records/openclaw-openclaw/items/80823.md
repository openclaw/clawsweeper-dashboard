## Best Possible Solution

Land the planner-backed CLI hint implementation after maintainer review if the current-head proof and targeted checks remain clean, while leaving the broader machine-readable apply-status contract to https://github.com/openclaw/openclaw/issues/83851.

## Reproduction Assessment

Yes. Source inspection on current main shows config set/patch and direct unset still print generic restart guidance, while the reload planner and docs classify the target agent/model paths as hot-applying.

## Solution Assessment

Yes. Reusing actual config diffs plus `buildGatewayReloadPlan()` is the best narrow fix; a CLI-only agentRuntime heuristic or suppression flag would duplicate reload policy and drift from gateway behavior.
