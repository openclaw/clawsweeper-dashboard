## What This Changes

The PR changes `config set`, `config patch`, and `config unset` success text to derive restart/no-restart hints from the gateway reload planner and adds CLI/gateway regression tests.

## Best Possible Solution

Keep the narrow CLI hint fix, but compute the hint from actual post-mutation changed paths or stay conservative for broad parent writes before merge.

## Reproduction Assessment

Yes. Source inspection shows current main still emits generic restart hints, and the PR-introduced broad-parent false-hot case follows from planning `models` instead of `models.pricing.enabled`.

## Solution Assessment

No for the current patch shape. Reusing the reload planner is the right direction, but the helper must feed it actual changed paths or stay conservative for broad parents that can contain restart-owned children.
