## Summary

Keep open. The PR adds a useful native detached plugin subagent primitive, but it is not mergeable yet because the patch misses plugin-owned session cleanup semantics and still has rebase/generated/changelog drift against current main.

## What This Changes

The PR adds optional `runtime.subagent.spawnDetached(...)` types and SDK exports, gateway wiring to `spawnSubagentDirect`, registry late binding, docs, and runtime/gateway/registry tests.

## Best Possible Solution

Rebase onto current main, preserve plugin ownership through the native detached spawn path, move the stale docs/mock edits to current files, regenerate the SDK API baseline, add a changelog entry, and then validate the public plugin runtime surface.

## Reproduction Assessment

Not applicable as a feature PR, but the cleanup regression has a high-confidence static reproduction path: spawn via `spawnDetached`, then call `deleteSession` from the same plugin; current delete ownership checks require a stored `pluginOwnerId` that the PR's native spawn path does not persist.

## Solution Assessment

No as-is. Reusing `spawnSubagentDirect` is the right narrow primitive, but the proposed implementation needs ownership propagation and current-main housekeeping before it is the maintainable solution.
