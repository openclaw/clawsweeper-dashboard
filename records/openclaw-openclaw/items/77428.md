## Summary

Keep open: current main still treats the already-loaded repair branch as success only after an unconditional `launchctl kickstart`, so the reported running-service interruption is still plausible and narrowly fixable.

## What This Changes

Request that macOS LaunchAgent bootstrap repair treat an already-loaded, running gateway as success instead of kickstarting it.

## Best Possible Solution

Gate the already-loaded branch on `readLaunchAgentRuntime()`: return `already-loaded` when runtime is running, kickstart only for stopped or unknown states, and cover both paths with regression tests plus a changelog entry.

## Reproduction Assessment

Yes, source-reproducible on current main: mock `bootstrap` as already-loaded while the fake LaunchAgent is running and the current unit harness shows `repairLaunchAgentBootstrap()` still calls `kickstart`. I did not run a live macOS launchd reproduction in this read-only review.

## Solution Assessment

Yes, the suggested runtime check is the narrowest maintainable fix because the helper already has a `readLaunchAgentRuntime()` state probe. The safer boundary is to skip kickstart only for `status: "running"` and preserve current recovery for stopped, unknown, or unreadable states.

## Repair Work Prompt

Fix the macOS LaunchAgent bootstrap repair already-loaded branch reported in https://github.com/openclaw/openclaw/issues/77428. In `src/daemon/launchd.ts`, when `launchctl bootstrap` reports an already-loaded job, inspect runtime state with `readLaunchAgentRuntime(env)` or an equivalent local helper. If the runtime status is `running`, return `{ ok: true, status: "already-loaded" }` without `launchctl kickstart`. Preserve the existing kickstart fallback for already-loaded services that are stopped, unknown, or whose runtime probe fails, and preserve genuine bootstrap/kickstart failure results. Update `src/daemon/launchd.test.ts` to cover already-loaded+running with no kickstart and already-loaded+not-running with kickstart; adjust nearby recovery tests only if the behavior changes their observable contract. Add an Unreleased Fixes changelog entry because this changes user-facing macOS repair behavior. Do not change unrelated `src/infra/restart.ts` restart handoff behavior unless a test proves it shares this repair path.
