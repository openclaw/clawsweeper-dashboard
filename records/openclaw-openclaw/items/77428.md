## Summary

Keep open. Current main still sends the already-loaded LaunchAgent repair branch to `launchctl kickstart`, and the existing tests lock in that behavior instead of checking whether the loaded service is already running.

## What This Changes

The issue requests that macOS LaunchAgent repair return success without kickstarting when `launchctl bootstrap` reports an already-loaded gateway that is already running.

## Best Possible Solution

Gate the already-loaded branch on `readLaunchAgentRuntime`: return `already-loaded` without kickstart when the service is running, preserve kickstart for stopped or unknown states, and add regression coverage plus a changelog entry.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: make `bootstrap` return exit 130 or `already exists in domain` while the mocked service is running, and current main proceeds to `kickstart`; I did not run a live macOS launchd reproduction.

## Solution Assessment

Yes. The suggested runtime check is the narrowest maintainable fix because it reuses the existing launchd runtime reader and keeps the current recovery nudge for already-loaded services that are not running.
