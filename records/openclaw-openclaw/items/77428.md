## Summary

Keep open. Current `main` still takes the already-loaded LaunchAgent repair branch to `launchctl kickstart`, and the existing tests explicitly expect that nudge instead of checking whether the already-loaded service is already running.

## What This Changes

The issue asks `repairLaunchAgentBootstrap()` to avoid kickstarting an already-loaded macOS gateway LaunchAgent when runtime state shows the service is already running.

## Best Possible Solution

Gate the already-loaded branch on `readLaunchAgentRuntime`: return `already-loaded` without kickstart when the service is running, and preserve the kickstart fallback for stopped, unknown, or unreadable runtime states.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: make `bootstrap` return an already-loaded failure while the mocked service is running, and current `repairLaunchAgentBootstrap` reaches the `kickstart` call instead of reading runtime state.

## Solution Assessment

Yes. Using the existing `readLaunchAgentRuntime` helper is the narrowest maintainable fix because it avoids disrupting healthy running services while keeping the current recovery nudge for non-running or uncertain already-loaded services.

Add or update focused tests in `src/daemon/launchd.test.ts` for already-loaded/running skips kickstart, already-loaded/stopped still kickstarts, runtime read failure still kickstarts, and existing kickstart failure reporting remains typed.
