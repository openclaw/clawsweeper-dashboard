## Summary

Keep open. Current main still takes the reported already-loaded branch to a `launchctl kickstart`, and the existing unit tests explicitly lock in that behavior rather than checking whether the loaded service is already running.

## What This Changes

Avoid kickstarting the macOS Gateway LaunchAgent during bootstrap repair when `bootstrap` reports the service is already loaded and runtime state is already running.

## Best Possible Solution

Gate the already-loaded branch on `readLaunchAgentRuntime`: return `already-loaded` without kickstart when the service is running, but preserve kickstart for loaded-but-stopped or unknown/unhealthy states, with regression tests and a changelog entry.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: make `bootstrap` return an already-loaded failure while the mocked service is running, and current tests expect `repairLaunchAgentBootstrap` to call `kickstart`. I did not run a live macOS launchd reproduction in this read-only review.

## Solution Assessment

Yes, the suggested runtime check is the narrowest maintainable fix because it uses the existing `readLaunchAgentRuntime` helper and preserves the current kickstart fallback when an already-loaded service is not running.
