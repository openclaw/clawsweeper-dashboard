## Summary

Keep open. Current main still routes the macOS LaunchAgent already-loaded repair branch to `launchctl kickstart`, and the existing unit coverage explicitly asserts that behavior instead of checking whether the loaded service is already running.

## What This Changes

Request that macOS LaunchAgent repair skip `kickstart` when `bootstrap` reports already-loaded and the service runtime is already running.

## Best Possible Solution

Gate the already-loaded branch on `readLaunchAgentRuntime`: return `already-loaded` without kickstart when the service is running, otherwise keep the existing kickstart fallback, with regression tests and a changelog entry.

## Reproduction Assessment

Yes, by source-level reproduction. On current main, make `launchctl bootstrap` return exit 130 or an `already exists in domain` error while the mocked service is running; `repairLaunchAgentBootstrap` still calls `kickstart`, and current tests assert that call.

## Solution Assessment

Yes. The suggested runtime check is the narrowest maintainable fix because it uses the existing `readLaunchAgentRuntime` seam and keeps the repair kickstart only for non-running or unhealthy already-loaded services.

## Repair Work Prompt

Fix the macOS LaunchAgent repair bug reported in https://github.com/openclaw/openclaw/issues/77428. In `repairLaunchAgentBootstrap`, when `launchctl bootstrap` fails only because the service is already loaded, call the existing `readLaunchAgentRuntime(env)` helper and return `{ ok: true, status: "already-loaded" }` without `kickstart` if the runtime is already `running`. Preserve the current `kickstart` fallback for already-loaded services that are stopped, unknown, or whose runtime check fails. Add focused regression coverage in `src/daemon/launchd.test.ts` for the running already-loaded case and keep coverage for stopped/unhealthy already-loaded kickstart behavior. Check whether `src/cli/daemon-cli/launchd-recovery.test.ts` or `src/commands/doctor-gateway-daemon-flow.test.ts` need expectation updates. Add a single-line user-facing `CHANGELOG.md` fix entry with appropriate reporter credit; do not change restart semantics, service labels, config keys, or launchd plist generation.
