## Summary

Keep open. Current main still sends the already-loaded LaunchAgent repair branch to `launchctl kickstart`, and the unit tests explicitly lock in that behavior even though runtime-state inspection already exists.

## What This Changes

The issue requests that macOS LaunchAgent bootstrap repair skip `kickstart` when `bootstrap` only reports an already-loaded service that is already running.

## Best Possible Solution

Gate the already-loaded repair branch on `readLaunchAgentRuntime`: return success without kickstart when the service is running, while preserving kickstart for stopped or unknown already-loaded states.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: make `bootstrap` return an already-loaded failure while the mocked service is running, and current tests expect `repairLaunchAgentBootstrap` to call `kickstart`. I did not run a live macOS launchd reproduction in this read-only review.

## Solution Assessment

Yes, the suggested runtime check is the narrowest maintainable fix because it uses the existing `readLaunchAgentRuntime` helper and preserves the existing kickstart fallback for non-running or unhealthy already-loaded services.
