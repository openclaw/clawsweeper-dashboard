## Summary

The reported `replies=0` behavior matches OpenClaw's documented group/channel visible-reply policy: Feishu topic turns are still group turns, and without `messages.groupChat.visibleReplies: "automatic"`, normal assistant finals remain private unless the agent uses `message(action=send)`.

## What This Changes

Feishu `group_topic` mention turns are expected to send the generated answer back to the same topic instead of completing with `replies=0`.

## Best Possible Solution

Keep the current tool-only group visible-reply policy and have this setup use `messages.groupChat.visibleReplies: "automatic"` or a model/runtime that reliably calls `message(action=send)` for visible Feishu topic output.

## Reproduction Assessment

Yes, at source level: Feishu topic turns are group turns, and current group/channel visible replies default to `message_tool_only` when the override is absent. I did not run a live Feishu tenant, but the source and docs explain the observed `replies=0` path.

## Solution Assessment

Yes. The maintainable solution is to use the existing visible-reply configuration or message tool behavior, not change Feishu topic routing code.
