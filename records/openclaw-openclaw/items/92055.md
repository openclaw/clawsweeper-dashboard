## Summary

The PR addresses a valid state-relative image hydration bug at the correct shared cache boundary, but the state-first `existsSync` decision introduces a concrete fallback regression: an existing directory, unusable file, or check/open race under the state directory can suppress a cwd-relative attachment that current main would read. The supplied real-path proof is convincing, and the existing allowlist plus canonical safe-open checks remain intact.

## What This Changes

The PR makes relative inbound attachments prefer an allowed file under the OpenClaw state directory before the existing cwd-relative fallback and adds cache and current-turn image hydration tests.

## Best Possible Solution

Validate state and cwd candidates in order through the cache’s existing regular-file and safe-open path, falling back to cwd for missing or unusable state candidates while continuing to fail closed for lexical or canonical root-policy violations.

## Reproduction Assessment

## Solution Assessment

No, not as written. The shared cache is the correct fix boundary, but candidate selection must preserve the existing cwd fallback when the state candidate is missing or unusable rather than committing to it after a separate existence probe.
