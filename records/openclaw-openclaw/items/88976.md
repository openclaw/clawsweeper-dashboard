## Summary

## What This Changes

The PR adds `supportsPromptCacheKey` to the bundled Mistral transport compat patch, updates normalization reuse checks, and extends Mistral API tests for that flag.

## Best Possible Solution

Land a Mistral provider fix that sends only documented `prompt_cache_key` unless a separate long-retention capability is true, updates Mistral cache-read pricing/tests, and includes redacted live Mistral evidence.

## Reproduction Assessment

Yes, from source inspection: current main gates `prompt_cache_key` on `compat.supportsPromptCacheKey`, and this PR's new flag also reaches the existing long-retention branch. No live Mistral request was run in this read-only review.

## Solution Assessment

No. Provider-owned compat is the right direction, but the current flag alone overloads key support with 24h retention and leaves pricing/proof gaps; the safer solution is to separate long-retention capability and update catalog coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Gate Mistral's long-retention field:** `extensions/mistral/api.ts:17`
  - body: `supportsPromptCacheKey: true` also makes `buildOpenAICompletionsParams` add `prompt_cache_retention: "24h"` whenever `cacheRetention === "long"`. Mistral's OpenAPI `ChatCompletionRequest` lists `prompt_cache_key`, has `additionalProperties: false`, and does not define `prompt_cache_retention`, so existing Mistral configs with long cache retention can start failing instead of only sending the supported cache key. Split key support from long-retention support before enabling this for Mistral.
  - confidence: 0.89
- **[P2] Price cached Mistral tokens before enabling them:** `extensions/mistral/api.ts:17`
  - body: The linked request and Mistral API schema say cached tokens are billed at 10% of input, but the bundled Mistral catalog still sets `cacheRead: 0` for every listed chat model. Once this flag lets cached-token usage appear, OpenClaw cost estimates will treat those tokens as free unless the catalog and model-definition expectations are updated.
  - confidence: 0.82
