## What This Changes

## Best Possible Solution

Land a version that keeps the compat split but aligns Mistral `cacheRead` pricing and regression coverage with the prompt-cache enablement before exposing cache hits.

## Reproduction Assessment

Yes, from source inspection rather than a live Mistral run. Current main lacks the Mistral `supportsPromptCacheKey` flag while the transport gates `prompt_cache_key` on compat support.

## Solution Assessment

No, not yet. The cache-key versus long-retention split is the right layer, but the best fix also updates Mistral cached-read pricing and coverage before enabling the flag.
