## Summary

Keep open: current main does not implement `neverReply`, and the latest PR head still has blocking consistency issues plus missing after-fix real behavior proof.

## What This Changes

The PR adds a `neverReply` channel/default/account config and per-channel group-message branches to suppress replies while optionally recording pending history.

## Best Possible Solution

Choose one listen-only contract, then make schema, access gating, pending-history write/read/clear behavior, docs, and live proof consistent across the supported channels.

## Reproduction Assessment

Yes. Source inspection of the PR head shows concrete mismatches in Matrix access/history ordering, IRC config acceptance, Slack file-only pending history, and Telegram scoped config semantics.

## Solution Assessment

No. The user need is real, but this branch is not yet the narrowest maintainable solution because the core-config approach remains inconsistent and the plugin-hook alternative needs maintainer direction.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Move Matrix neverReply after access and history recording:** `extensions/matrix/src/matrix/monitor/handler.ts:706-709`
  - body: This branch returns before `resolveMatrixMonitorAccessState` runs and before the pending-history text path is reached. With `neverReply` enabled, Matrix can consume room traffic without the promised live access ordering or context accumulation.
  - confidence: 0.9
- **[P2] Accept IRC neverReply in the strict schema:** `extensions/irc/src/inbound.ts:157-162`
  - body: This runtime branch reads `channels.irc.neverReply`, but the strict IRC account/channel schema still has no `neverReply` field. Users who configure IRC listen-only mode can have the config rejected before this code runs.
  - confidence: 0.88
- **[P2] Preserve Slack file-only messages in silent history:** `extensions/slack/src/monitor/message-handler/prepare.ts:495-501`
  - body: The neverReply path records only trimmed `message.text`, so file-only room or group-DM messages are dropped from passive context. The mention-skip path below already builds a file placeholder and should be matched here.
  - confidence: 0.86
- **[P2] Remove or honor Telegram scoped neverReply:** `src/config/zod-schema.providers-core.ts:134-151`
  - body: The schema accepts `neverReply` on Telegram groups and topics, but `resolveNeverReply` only checks account, channel, and defaults. A valid-looking group/topic setting therefore has no effect.
  - confidence: 0.84
