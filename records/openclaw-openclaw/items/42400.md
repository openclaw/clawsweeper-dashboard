## Summary

Keep open: current main has no implemented replacement, and the PR still has source-visible correctness blockers around pending-history behavior and config scope.

## What This Changes

This PR adds a `neverReply` channel config/default resolver, schema and SDK exports, docs, and per-channel group-message gates intended to suppress group replies while preserving context where available.

## Best Possible Solution

Maintainers should decide whether listen-only group observation belongs in core config or a plugin hook, then make the chosen path consistent across resolver scope, schemas, channel history recording, and history draining.

## Reproduction Assessment

Yes. Source inspection of the supplied PR diff and current-main channel paths shows the Discord, LINE, and Mattermost `neverReply` branches can return before existing pending-history helpers run.

## Solution Assessment

No. The user need is valid, but this patch is not yet the narrowest maintainable solution because it exposes config semantics the channel handlers and resolver do not reliably honor.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Record Discord history before returning for neverReply:** `extensions/discord/src/monitor/message-handler.preflight.ts:394-401`
  - body: This branch returns before `historyEntry` is built and before the existing `recordPendingHistoryEntryIfEnabled` paths run, so Discord suppresses the reply but drops the group message instead of preserving it as pending context. Move the check after the history entry is available and record it with the same key used by mention-skip paths.
  - confidence: 0.88
- **[P2] Store LINE group events before denying neverReply:** `extensions/line/src/bot-handlers.ts:320-323`
  - body: Returning `denied` here exits before the existing LINE group-history record path in `handleMessageEvent`, so `channels.line.neverReply` loses the passive context the feature promises. Record the pending history entry with the same key/body fallback as the mention-skip path before returning.
  - confidence: 0.86
- **[P2] Move Mattermost neverReply after the history route:** `extensions/mattermost/src/mattermost/monitor.ts:1359-1366`
  - body: This check runs before Mattermost computes the route `historyKey` and before the local `recordPendingHistory` helper exists, so no suppressed group post reaches `channelHistories` despite the log claiming it was stored. Compute the route/history key first and call the existing helper before returning.
  - confidence: 0.87
- **[P2] Remove inert Telegram group neverReply schema:** `src/config/zod-schema.providers-core.ts:121-136`
  - body: The schema accepts `neverReply` on Telegram group/topic entries, but `resolveNeverReply` only receives channel and account scope, so group-specific values validate without taking effect. Either thread group scope through the resolver or remove these fields.
  - confidence: 0.83
