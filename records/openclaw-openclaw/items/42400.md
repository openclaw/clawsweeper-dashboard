## Summary

Keep this PR open. The listen-only use case is valid, but the current branch still has blocking consistency issues across channel config, access gating, and pending-history behavior, and it also lacks after-fix real behavior proof from a real channel/setup.

## What This Changes

The PR adds a `neverReply` channel/default/account config, schema/help/docs/SDK export, and per-channel group-message branches intended to suppress replies while optionally preserving pending context.

## Best Possible Solution

Choose either a core listen-only config or a plugin-hook design, then make one consistent contract across schemas, access gating, history recording, history draining, docs, and live proof for the affected channels.

## Reproduction Assessment

Yes. Source inspection of the PR head shows concrete mismatches: Matrix returns before access/history, IRC accepts no channel/account `neverReply` config, and Slack loses file-only pending context.

## Solution Assessment

No. The user need is real, but this branch is not yet the narrowest maintainable solution because the new config semantics are not consistently honored across channel schemas and handlers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Move Matrix neverReply after access and history recording:** `extensions/matrix/src/matrix/monitor/handler.ts:712-715`
  - body: This branch runs before Matrix resolves group allowlists/access state and before the existing `roomHistoryTracker.recordPending` path used for skipped room messages. With `neverReply` enabled, Matrix can silently consume room traffic without the promised access-control ordering or pending context.
  - confidence: 0.9
- **[P2] Add IRC neverReply to the strict config schemas:** `extensions/irc/src/inbound.ts:157-162`
  - body: This runtime branch checks `channels.irc.neverReply`, but the strict IRC account schemas still do not declare `neverReply`, so channel/account-level IRC settings are rejected even though the feature claims per-channel support. Add the field to the IRC schemas or remove the IRC-specific branch/docs.
  - confidence: 0.88
- **[P2] Preserve Slack file-only messages in neverReply history:** `extensions/slack/src/monitor/message-handler/prepare.ts:481-487`
  - body: The new branch records history only when `message.text` is non-empty, so file-only Slack room or group-DM messages are dropped from passive context. The mention-skip path directly below already uses a file placeholder fallback; the neverReply path should use the same pending body.
  - confidence: 0.86
- **[P2] Remove or honor Telegram group neverReply fields:** `src/config/zod-schema.providers-core.ts:121-138`
  - body: The schema accepts `neverReply` on Telegram group/topic entries, but the resolver only checks account, channel, and defaults, and the handler does not pass group/topic scope. Users can set a valid-looking group/topic value that has no effect.
  - confidence: 0.84
