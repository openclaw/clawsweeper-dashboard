## Summary

Keep open. Current main still lets claude-cli stream-json result usage become the CLI session context snapshot, and three open PRs already reference this issue as the fix path, so the issue should close only after a repair merges.

## What This Changes

The issue reports claude-cli stream-json cache-read accounting inflating session token snapshots and triggering failed compaction stalls across channels.

## Best Possible Solution

Land a focused claude-cli stream-json accounting fix that preserves the latest deduped assistant-step usage for session context snapshots while keeping cumulative result usage only for aggregate run accounting.

## Reproduction Assessment

Yes. The current source path can be reproduced with a focused claude stream-json fixture containing assistant usage events followed by a cumulative result usage record; I did not run the live claude-cli binary here.

## Solution Assessment

Yes for the accounting layer. The narrow maintainable fix is to preserve latest assistant-step usage for the context snapshot instead of feeding summed result usage into `lastCallUsage`; broader compaction ownership should remain separate if real context pressure remains after that fix.
