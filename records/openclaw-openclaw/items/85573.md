## Summary

Keep open. Current main has adjacent context-pressure work and now surfaces some CLI compaction failures, but the claude-cli stream-json accounting path still lets final `result.usage` become the fresh context snapshot; the parser does not preserve the latest assistant usage that the report says carries the per-call cache-read value.

## What This Changes

The issue requests fixing claude-cli stream-json cache-read accounting so summed tool-subcall cache reads do not poison session token totals or trigger futile compaction.

## Best Possible Solution

Keep this open and land a focused claude-cli stream-json accounting fix that carries the latest deduped assistant-call usage into the session context snapshot while preserving aggregate usage only where aggregate run/cost accounting is intended.

## Solution Assessment

Yes for the accounting layer: the narrow maintainable fix is to preserve latest assistant-call usage for the context snapshot instead of feeding summed result usage into `lastCallUsage`. Broader claude-cli compaction ownership should be a separate maintainer choice only if true context pressure remains.
