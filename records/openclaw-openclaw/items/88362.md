## What This Changes

The issue reports that WhatsApp inbound images in OpenClaw v2026.5.27 can be read once via `media://inbound/<id>` but fail on a later read with a media ID resolution error.

## Best Possible Solution

Add a focused regression for a WhatsApp-saved inbound image flowing into the agent prompt as `media://inbound/<basename>` and being read more than once, then fix only the path that emits, remaps, or deletes a non-persistent media ID.
