## Summary

Current main and the shipped v2026.5.6 package already strip `Sender (untrusted metadata)` fenced JSON blocks from user-visible chat history, with both gateway projection and Control UI defense-in-depth plus regression coverage for the exact `openclaw-control-ui` sender block.

## What This Changes

The issue asks for Control UI/WebChat message rendering to stop showing internal `Sender (untrusted metadata)` JSON envelope blocks as chat messages.

## Best Possible Solution

Keep the shared gateway projection and UI defense-in-depth; any future report should narrow to a cache, stale asset, or raw-event surface if the block still appears after update refresh.

## Reproduction Assessment

No, not as a current-main failure. Source, tests, docs, and the v2026.5.6 package artifact show the reported sender JSON envelope is stripped from chat history and metadata-only user rows are hidden.

## Solution Assessment

Yes. The existing shared sanitizer plus display projection is the narrowest maintainable fix because it covers Control UI, WebChat, and other transcript consumers without a Control UI-only special case.
