## What This Changes

The issue asks OpenClaw to avoid or warn about a paid/high-cost Anthropic default during Windows QuickStart before the first automated terminal chat runs.

## Best Possible Solution

Make the Anthropic first-run policy explicit before the first hatch: choose a safer default, add a clear billing/credit warning or preflight, or document why the Opus default remains intentional while ensuring billing failures read as provider setup failures rather than install failures.
