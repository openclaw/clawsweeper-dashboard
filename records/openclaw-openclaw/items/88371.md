## Summary

Current main and the latest release still do not provide a clear pre-hatch Anthropic billing/credit warning, and the API-key path still applies an Opus primary model even though Sonnet is now added to the allowlist. This remains useful but policy-sensitive setup work, so it should stay open for maintainer review.

## What This Changes

The issue requests safer Windows QuickStart handling when Anthropic onboarding selects a paid/high-cost default model for the first chat.

## Best Possible Solution

Make the Anthropic first-run policy explicit: either default API-key QuickStart to the intended safer model, show a billing-risk warning before terminal hatch, or add a provider-owned preflight/error path that distinguishes install success from provider billing failure.

## Reproduction Assessment

Partly. Source inspection shows current main can still choose an Anthropic Opus primary model and terminal hatch can immediately send the bootstrap message, but the exact Windows low-credit account failure was not live-reproduced.

## Solution Assessment

No, not yet. The maintainable fix depends on a maintainer product choice between changing the provider-owned default, adding a pre-hatch billing warning/preflight, or improving first-chat billing failure copy across setup surfaces.
