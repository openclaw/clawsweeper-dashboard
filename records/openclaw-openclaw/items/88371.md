## Summary

Keep open: current main and v2026.6.8 still default Anthropic setup/API-key paths to an Opus model and do not provide a clear pre-hatch billing/credit warning, although main now has partial post-failure billing copy and a Sonnet allowlist mitigation. This remains aligned first-run UX work, but changing Anthropic defaults or warning semantics needs maintainer product direction rather than conservative cleanup closure.

## What This Changes

The issue requests safer Windows QuickStart Anthropic first-run behavior, such as avoiding a high-cost default model, warning about required API credits before terminal hatch, or making first-chat billing failures clearly non-install failures.

## Best Possible Solution

Make the Anthropic first-run policy explicit: either choose a safer default for API-key QuickStart, add a clear pre-hatch billing/credit warning or preflight, or document why Opus remains the default while ensuring billing failures are framed as provider setup failures rather than install failures.

## Reproduction Assessment

Partly. Source inspection shows current main can still select an Anthropic Opus primary model and immediately launch the first terminal hatch message without a billing-credit warning, but the exact Windows low-credit Anthropic account failure was not live-reproduced.

## Solution Assessment

No, not yet. The requested direction is valid, but the best fix depends on a maintainer product choice between changing the Anthropic default, adding pre-hatch billing/preflight copy, improving first-chat billing copy, or combining those approaches.
