## Summary

Keep open: current main and the latest release still put Anthropic API/setup-token onboarding on an Opus default, and the QuickStart terminal hatch still sends the first message without a pre-hatch billing or credit warning. The remaining work is valid first-run UX, but the fix needs maintainer product direction on default model versus warning/preflight policy.

## What This Changes

Request a Windows QuickStart Anthropic first-run experience that avoids an unwarned paid Opus default or clearly warns/preflights billing before the first hatch.

## Best Possible Solution

Make the Anthropic first-run policy explicit before the first hatch with a maintainer-approved safer default, billing/credit warning or preflight, and provider-billing failure wording that does not look like an install failure.

## Reproduction Assessment

No high-confidence live reproduction was run in this read-only review. Source inspection confirms current main and v2026.6.8 still use an Anthropic Opus default path and the first-hatch copy lacks a pre-hatch billing or credit warning.

## Solution Assessment

No, current main is not the final solution. The best fix needs a maintainer choice between changing the Anthropic default, adding pre-hatch billing/preflight copy, improving first-chat billing copy, or combining those approaches.
