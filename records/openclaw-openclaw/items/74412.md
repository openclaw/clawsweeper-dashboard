## Summary

Current main supersedes this PR: cron delivery is now resolved from preserved current or stored delivery context, and Discord DM turns store `user:<id>` as the routable target, so the old raw session-key inference path this branch patches no longer exists.

## What This Changes

The PR prefixes Discord `direct`/`dm` session-key inferred cron delivery targets with `user:`, adds four cron-tool tests, and adds a changelog entry.

## Best Possible Solution

Keep the current context-based cron delivery implementation and close this stale branch instead of reintroducing provider target reconstruction from session-key fragments.

## Reproduction Assessment

No for current main. The old PR base had a source-visible bare session-key fallback, but current main resolves cron delivery from preserved current or stored delivery context and has coverage that raw session-key fragments do not become targets without that context.

## Solution Assessment

Yes for current main. The context-based fix is narrower and safer than prefixing reconstructed Discord session-key fragments because it preserves provider-owned target syntax and avoids rebuilding case-sensitive or ambiguous targets from session keys.
