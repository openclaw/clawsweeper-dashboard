## Summary

Keep open: the prompt direction is useful, but the patch still applies `presentation.blocks` guidance to all automatic group prompts despite channel capability gating, and the supplied proof is tests/snapshots rather than live Telegram behavior.

## What This Changes

The PR changes group auto-reply prompt wording and related tests/fixtures/snapshots to allow inline-button `message(action=send)` calls in group prompts while keeping plain-text replies automatic.

## Best Possible Solution

Keep the inline-button exception, but inject it only for presentation-capable group prompts or Telegram-specific contexts, then land with redacted live Telegram proof showing rendered group inline buttons and a silent final turn.

## Reproduction Assessment

No live reproduction was established, but the source path is clear: current main forbids same-group `message` sends while Telegram docs/runtime support presentation-backed inline buttons. The sibling-channel regression is also source-reproducible from WhatsApp lacking the advertised `presentation` capability.

## Solution Assessment

No, not as written. A prompt exception is a narrow fix for Telegram, but it should be capability-gated instead of added to every automatic group prompt.
