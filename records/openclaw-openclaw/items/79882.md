## Summary

Keep open: current main still has session-scoped bundled MCP runtime caching, while this PR adds a useful opt-in shared runtime mode that is not implemented elsewhere and needs maintainer approval for the new isolation/config policy before merge.

The PR adds an opt-in `mcp.runtimeScope` setting for shared bundled MCP runtimes keyed by workspace and MCP config fingerprint, with config/schema/docs updates and runtime lifecycle tests.

## Best Possible Solution

If maintainers accept the opt-in isolation tradeoff, rebase and land the shared runtime mode with `session` as the default while preserving current MCP config fields and validating the exact head.
