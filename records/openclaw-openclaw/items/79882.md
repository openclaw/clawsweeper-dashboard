The PR adds `mcp.runtimeScope: "session" | "shared"` so bundled MCP runtimes can optionally be reused across sessions by workspace and MCP config fingerprint, with docs, schema/help labels, and runtime tests.

## Best Possible Solution

If maintainers accept the opt-in isolation tradeoff, land this after rebase with exact-head validation while keeping `session` as the default and preserving the single-tenant caveat in docs.
