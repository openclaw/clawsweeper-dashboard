## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Recompute shared TTLs when a session detaches:** `src/agents/agent-bundle-mcp-runtime.ts:1184-1187`
  - body: Shared runtime TTLs only ever increase, with `0` treated as unbounded. When a long-TTL or `0`-TTL session detaches while other sessions remain, this early return leaves its TTL on the runtime, so the remaining sessions may never get idle-evicted as configured. Track TTL per attached session and recompute the runtime TTL on detach.
  - confidence: 0.88
- **[P2] Use an opaque shared runtime key:** `src/agents/agent-bundle-mcp-runtime.ts:493-496`
  - body: `buildSharedRuntimeKey` embeds the raw workspace path in the runtime id, and shared scope passes that id as `runtime.sessionId` for logs/errors. That can expose local workspace paths and conflicts with the canonical issue's diagnostics requirement. Hash or otherwise opaque the key and keep any debug metadata separately redacted.
  - confidence: 0.84
- **[P2] Warn config readers about shared MCP isolation:** `docs/gateway/configuration-reference.md:179-184`
  - body: The primary configuration reference introduces `"shared"` without saying it is for trusted single-tenant deployments and trades away per-session MCP client/child-process isolation. Add that warning here, matching the CLI docs, so operators see the boundary before enabling the new config.
  - confidence: 0.86
