## What This Changes

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Recompute shared TTL after session detach:** `src/agents/pi-bundle-mcp-runtime.ts:688-690`
  - body: `idleTtlMsByRuntimeKey` keeps the max or zero TTL ever seen for a shared runtime, but `disposeSession` detaches one session and returns while other sessions remain without recalculating it. If a `sessionIdleTtlMs: 0` or long-TTL session leaves, the remaining sessions inherit that unbounded or long TTL until another `getOrCreate` call, so idle sweep can keep MCP children alive after the remaining sessions would allow eviction. Track per-session TTLs or recompute the runtime TTL when a session detaches.
  - confidence: 0.86
