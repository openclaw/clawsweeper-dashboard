## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Recompute shared runtime TTL after detach:** `src/agents/agent-bundle-mcp-runtime.ts:1184-1187`
  - body: When a shared runtime has one attached session with `sessionIdleTtlMs: 0` and another with a finite TTL, `0` correctly wins while both are attached. But `disposeSession` detaches the unbounded session and returns while other sessions remain without recalculating `idleTtlMsByRuntimeKey`, so the finite-TTL session inherits unbounded lifetime until it also detaches. Track TTL per attached session or recompute the runtime TTL after detach.
  - confidence: 0.86
- **[P2] Warn config readers about shared MCP isolation:** `docs/gateway/configuration-reference.md:179-184`
  - body: The primary configuration reference introduces `"shared"` without saying it is for trusted single-tenant deployments or that MCP clients, child processes, server-side state, and failure state can cross sessions. Add that warning here so operators see the isolation boundary before enabling the new config.
  - confidence: 0.85
