Full review comments:

- **[P2] Recompute shared runtime TTL after detach:** `src/agents/agent-bundle-mcp-runtime.ts:1184-1187`
  - body: When a shared runtime has one attached session with `sessionIdleTtlMs: 0` and another with a finite TTL, detaching the unbounded session leaves `idleTtlMsByRuntimeKey` at `0` because this branch returns while other sessions remain attached. The finite-TTL session then inherits unbounded lifetime until final detach; track per-session TTLs or recompute the runtime TTL from the remaining attachments before returning.
  - confidence: 0.86
- **[P2] Warn config readers about shared MCP isolation:** `docs/gateway/configuration-reference.md:179-184`
  - body: The primary configuration reference introduces `"shared"` without saying it is for trusted single-tenant deployments or that MCP clients, child processes, catalog state, server-side state, and failure/backoff state can cross sessions. Add that warning here so operators see the isolation boundary before enabling the new config.
  - confidence: 0.85
