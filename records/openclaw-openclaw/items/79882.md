## Summary

Keep open: this is a recent, plausible opt-in performance feature with focused tests and sufficient real behavior proof, but it adds a new config policy that trades MCP runtime isolation for reuse and needs maintainer judgment plus a rebase before merge.

## What This Changes

The PR adds `mcp.runtimeScope: "session" | "shared"` so bundled MCP runtimes can optionally be shared by workspace and MCP config fingerprint, with matching config schema, docs, and runtime tests.

## Best Possible Solution

If maintainers accept the opt-in isolation tradeoff, rebase the branch, rerun the focused runtime/config tests and changed gate, then land the feature with `session` kept as the default and the single-tenant caveat preserved.

## Reproduction Assessment

Not applicable as a bug reproduction: this PR is an opt-in performance feature. Current-main source inspection confirms the existing runtime is session-keyed, and the PR body plus inspected screenshot provide after-fix performance evidence for the proposed shared mode.

## Solution Assessment
