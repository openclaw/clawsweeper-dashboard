## Summary

Keep open: the PR is a plausible opt-in fix direction for duplicated bundled MCP runtimes, and the new Docker benchmark satisfies the real-behavior proof gate, but the patch introduces a shared-runtime TTL regression that should block merge.

## What This Changes

The PR adds an opt-in `mcp.runtimeScope: "session" | "shared"` config option, refactors bundled MCP runtime caching to share by workspace/config fingerprint, and updates schema/help labels plus runtime tests.

## Best Possible Solution

Fix shared-runtime TTL aggregation so `0` remains a disabling value while any attached session requires it, add focused coverage for mixed TTL attachments/detaches, update public MCP/config docs, then have maintainers decide the isolation policy before landing.

## Reproduction Assessment

Yes for the review finding: source inspection gives a high-confidence path by attaching shared sessions with mixed `sessionIdleTtlMs` values, including `0`, then sweeping after the positive TTL. Not applicable for the PR's main feature claim beyond the supplied Docker benchmark evidence.

## Solution Assessment

No as currently written: the shared-runtime approach is directionally aligned with https://github.com/openclaw/openclaw/issues/75621, but the TTL merge must preserve the existing disabled-eviction contract and the isolation/default policy still needs maintainer approval.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve disabled idle TTL when sharing runtimes:** `src/agents/pi-bundle-mcp-runtime.ts:457-458`
  - body: `sessionIdleTtlMs: 0` currently means idle eviction is disabled, but shared runtimes merge attached sessions with a numeric max. Since `0` is lower than any positive TTL, a runtime with an attached zero-TTL session can still be assigned a positive TTL and later swept, or a later positive-TTL attach can re-enable eviction. Track per-session TTLs and make `0` dominate while any attached session uses it, then recompute on detach.
  - confidence: 0.91
