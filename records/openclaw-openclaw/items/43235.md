## Summary

Keep open. Current main and the latest release still route default inbound agent turns through the process-wide `main` lane, and the open fixing PR with closing syntax has not merged.

## What This Changes

Request support for per-agent or agent-group command lanes so long-running agents do not starve other agents on the shared main lane.

## Best Possible Solution

Review, fix, and land or replace https://github.com/openclaw/openclaw/pull/73991 with backward-compatible, opt-in, server-owned command-lane config plus schema, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows default inbound runs still enter the shared `main` lane, and the Feishu comment provides concrete multi-bot `lane=main` wait logs; I did not run a live load test.

## Solution Assessment

Yes, the requested direction is a maintainable core queue/config solution, but it is not implemented yet. The active PR or a replacement should preserve shared `main` by default and make lane choice server-controlled.
