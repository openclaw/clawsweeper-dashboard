## Summary

Keep open. Current main still routes default inbound agent turns through a per-session lane and then the shared process-wide `main` lane, and open PR #73991 targets this issue with closing syntax but has not merged.

## What This Changes

Request opt-in per-agent or per-agent-group command lanes/concurrency so long-running agents do not starve other agents in multi-agent deployments.

## Best Possible Solution

Review, rebase, and land or replace PR #73991 with a backward-compatible opt-in queue/config change that preserves shared `main` by default and updates schema, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows default inbound runs still serialize per session and then enter shared `main`, and the Feishu comment provides concrete multi-bot `lane=main` wait logs; I did not run a live Feishu load test.

## Solution Assessment

Yes, the requested direction is the narrow maintainable one: an opt-in core queue/config path rather than a Feishu-specific workaround or separate gateway deployment. The paired PR matches that direction but remains open.
