## Summary

Keep open. Current main still routes default inbound agent turns through the shared process-wide `main` lane, and open PR #73991 targets this issue with closing syntax but has not merged.

## What This Changes

The issue requests opt-in per-agent command lanes so long-running work from one agent does not consume the shared main lane for every other agent.

## Best Possible Solution

Review, rebase, and land or replace PR #73991 with a backward-compatible opt-in queue/config change that preserves shared `main` by default and updates schema, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows default inbound runs still serialize per session and then enter shared `main`, and the Feishu comment provides concrete multi-bot `lane=main` wait logs; I did not run a live Feishu load test.

## Solution Assessment

Yes. An opt-in core queue/config path is the narrow maintainable direction versus a Feishu-specific workaround or separate gateway deployment, but the linked implementation PR remains open.
