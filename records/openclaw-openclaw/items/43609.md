## What This Changes

The PR adds Grok web_search null guards, wraps internal timeout AbortErrors as plain errors, raises the Grok default timeout to 60 seconds, and also changes status-command loading/model-alias handling.

## Best Possible Solution

Port only the Grok/xAI web_search fixes into the current xAI plugin runtime: harden the shared Responses parser, convert provider-internal timeout aborts into structured tool errors while preserving cause, keep explicit timeout overrides, use a Grok-specific 60s default when unset, and add focused regression coverage. Leave the status-command changes out of the repair.
