## Summary

Keep open. Current main still renders Telegram fenced code blocks with a static `<pre><code>` marker, the formatter test still asserts output without a language class, and the latest release has the same behavior; the WebChat copy-button path is separate and does not change Telegram Bot API HTML.

## What This Changes

Request to preserve fenced code language identifiers in Telegram HTML as `class="language-X"` so Telegram clients can show native code-block copy support.

## Best Possible Solution

Carry sanitized optional fence-language metadata through the Telegram render/chunk path, emit `class="language-X"` for fenced blocks, and preserve existing escaping, file-reference wrapping, and chunk-limit behavior.
