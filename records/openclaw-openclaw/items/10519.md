## Summary

Keep open. Current main and the latest tagged release still drop fenced-code language metadata before Telegram HTML rendering, and the related WebChat copy-button work is a separate UI path rather than a Telegram Bot API output fix.

## What This Changes

The request is to preserve Markdown fenced-code language identifiers in Telegram HTML code blocks so Telegram can show native code-copy affordances.

## Best Possible Solution

Preserve a sanitized first fence-language token through the Markdown IR/render-aware chunking path and emit a safe Telegram code class only for fenced blocks.

## Reproduction Assessment

Yes, at source level: current main still renders a js fence as <pre><code> without a class and drops markdown-it token.info before Telegram rendering. I did not run Vitest or a live Telegram send in this read-only review.

## Solution Assessment

Yes. Carrying a sanitized fence-language field through the existing shared Markdown IR and render-aware chunking contract is the cleanest maintainable fix; a Telegram-local fence-aware renderer is a fallback only if maintainers reject widening the IR shape.
