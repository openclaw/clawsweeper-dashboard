## Summary

Keep open: current main has the optimistic-tail history reload mitigation, but it does not fully cover the reported WebChat cluster because stream segments are still rendered as independent bubbles around tool calls and passive same-session runs still are not adopted into `chatRunId`.

## What This Changes

The issue reports WebChat assistant output briefly rendering and then disappearing or collapsing after tool-heavy runs because history reload, run tracking, and stream segmentation can drift from the live transcript.

## Best Possible Solution

Keep this issue open as the canonical report for Tim's environment and land a focused WebChat fix that preserves the optimistic-history safeguards while adding session-scoped run adoption and deduplicated or unified stream rendering around tool calls.

## Reproduction Assessment

No, not as a live current-main browser reproduction in this pass. Source inspection gives a credible reproduction path because stream segments and live streams are still rendered independently and same-session deltas still do not adopt an idle `chatRunId`.

## Solution Assessment

Unclear as a complete current-main solution: the optimistic-tail history reload fix is the right direction for the stale snapshot race, but the remaining maintainable fix should add focused regression coverage for stream segment deduplication/unified rendering and passive run adoption without undoing the existing safeguards.
