## Summary

Keep open: the patch still puts billing diagnostics into replay-visible assistant content while replay cleanup only trims the canonical stream-error sentinel, and the PR still lacks after-fix real behavior proof.

## What This Changes

The PR changes stream-error message construction to render billing/credits hints in assistant content and adds an OpenRouter insufficient-credits regression test.

## Best Possible Solution

Keep stream-error content on the canonical sentinel, surface billing failures through errorMessage/formatAssistantErrorText/final payload rendering, and add replay coverage that billing stream-error turns are not sent back to providers.

## Reproduction Assessment

Yes. Source inspection shows the PR creates a non-sentinel billing assistant content turn, while current replay normalization only drops empty or STREAM_ERROR_FALLBACK_TEXT trailing error turns.

## Solution Assessment

No. The safer maintainable path is to keep replay-visible content on the sentinel and improve the existing errorMessage-based user-facing rendering path instead of placing billing diagnostics in assistant content.
