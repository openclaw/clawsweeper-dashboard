## Summary

Keep open: current `main` still serves Control UI/WebChat static files as raw buffers, and the request is a narrow performance feature rather than an already-fixed bug or stale cleanup item.

## What This Changes

The issue requests gzip or Brotli compression for gateway-served webchat static assets when clients send `Accept-Encoding`.

## Best Possible Solution

Implement negotiated gzip/Brotli compression for Control UI static text assets in `src/gateway/control-ui.ts`, preserving existing path-safety checks and adding focused HTTP tests.

## Reproduction Assessment

Yes. Source inspection of current `main` shows `/assets/*` requests reach `serveResolvedFile`, which writes the raw buffer without any `Accept-Encoding` or `Content-Encoding` handling; I did not run a live curl because this review is read-only.

## Solution Assessment

Yes. The narrow maintainable solution is gateway-owned compression negotiation on the existing Control UI static-file response path, with `Vary: Accept-Encoding`, a compressible MIME allowlist, and regression tests for GET and HEAD behavior.
