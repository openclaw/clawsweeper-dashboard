## Summary

This is a valid, narrow performance feature request rather than a stale or already-implemented bug: current main serves Control UI/WebChat static assets as raw buffers and has no gateway-side `Accept-Encoding` negotiation.

## What This Changes

The issue requests gzip or brotli compression for static WebChat/Control UI assets served by the gateway.

## Best Possible Solution

Add targeted compression negotiation to the Control UI static asset helpers, compressing only text-like assets and HTML while preserving auth, CSP, cache, HEAD, and path-safety behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows static files and `index.html` are served directly from raw buffers/strings, and the gateway has no `Accept-Encoding` or `Content-Encoding` handling; I did not start a live gateway in this read-only review.

## Solution Assessment

Yes, if maintainers want built-in compression. The narrowest maintainable fix is in `src/gateway/control-ui.ts`, not gateway-wide API compression or plugin routing.
