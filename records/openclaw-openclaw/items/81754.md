## What This Changes

Request gzip or Brotli compression for Gateway-served Control UI static assets when clients send `Accept-Encoding`.

## Best Possible Solution

Land a safe version of https://github.com/openclaw/openclaw/pull/81794, or an equivalent narrow PR, that serves negotiated gzip/Brotli using cached, async, or streamed compression while preserving current routing, CSP, and file-safety boundaries.

## Reproduction Assessment

Yes. Current main's Control UI static-file helpers send raw file and HTML bodies without `Accept-Encoding` negotiation, and the issue provides a curl path that observes the missing `Content-Encoding` header.

## Solution Assessment

Mostly yes on owner boundary: the Gateway Control UI handler is the right place to negotiate static asset encodings. The final implementation should avoid synchronous per-request zlib and use cached, async, or streamed compression before this issue closes.
