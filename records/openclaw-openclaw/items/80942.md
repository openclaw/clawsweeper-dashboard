## Summary

Keep open: current main and the latest release still have a static Control UI browser title, and the dynamic-title work is now owned by the open implementation candidate at https://github.com/openclaw/openclaw/pull/90577 rather than solved on main.

## What This Changes

The issue asks for the Control UI browser tab title to update dynamically from the active agent or session so multi-agent tabs can be distinguished.

## Best Possible Solution

Adopt one Control UI title resolver that uses safe explicit session or agent metadata, honors any accepted static configured-title precedence, and covers switching, metadata refresh, fallback reset, and unsafe identifier filtering.

## Reproduction Assessment

Yes for the observable current baseline: current main and `v2026.6.10` still show the static `OpenClaw Control` title, and source inspection found no lifecycle `document.title` writer. This is a feature request rather than a broken existing contract.

## Solution Assessment

Unclear as a product decision. The active session-title PR is a plausible implementation path, but the final solution should avoid competing static and dynamic title writers and preserve safe metadata filtering.
