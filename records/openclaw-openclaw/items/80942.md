## Summary

Keep open: current main still uses a static Control UI browser title and has no dynamic `document.title` updater. The related dynamic-title attempts are closed unmerged, while the static `gateway.controlUi.title` PR is still open, so this remains a live UI feature/product decision rather than a cleanup close.

## What This Changes

The issue requests dynamically setting the Control UI browser tab title from the active agent name or id, with any configured static gateway title taking precedence when present.

## Best Possible Solution

Decide the Control UI title policy, then implement a small UI-side title synchronizer that uses the active agent identity/session fallback and composes cleanly with any accepted static gateway title config.

## Reproduction Assessment

## Solution Assessment
