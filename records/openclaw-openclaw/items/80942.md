## Summary

Keep open: current main still uses a static Control UI browser title and has no dynamic `document.title` synchronizer. The issue is also paired with the open same-author implementation at https://github.com/openclaw/openclaw/pull/80944, so it should stay open until that PR is reviewed and either merged or closed.

## What This Changes

Request dynamic Control UI browser-tab titles that use the active agent identity, with graceful fallback behavior and precedence for any configured static gateway title.

## Best Possible Solution

Review and either land or close https://github.com/openclaw/openclaw/pull/80944 after deciding how dynamic per-agent titles should compose with the static gateway title proposal.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms the current baseline: the HTML title is static and there is no current-main lifecycle updater for `document.title`.

## Solution Assessment

Unclear until maintainers decide the title policy. A small UI-side title synchronizer is plausible, but it should be reviewed against the static gateway title proposal and the fallback behavior called out in the issue discussion.
