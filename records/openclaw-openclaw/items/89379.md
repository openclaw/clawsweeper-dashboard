## Summary

Keep open: the direct Gemini fix direction is sound, but the branch also flips the shared Google hook used by Gemini CLI to native mode without proof or sibling coverage, and the external PR still lacks live after-fix Gemini behavior proof.

## Best Possible Solution

Make native reasoning mode conditional for native Google Gemini transports while preserving tagged/final enforcement for Gemini CLI unless maintainers intentionally migrate it with live proof and tests.

No as written. Moving direct Gemini API reasoning output to native is the right layer, but the shared hook must avoid changing Gemini CLI unless that sibling provider is separately proven safe.
