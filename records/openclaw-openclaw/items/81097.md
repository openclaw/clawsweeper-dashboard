## Summary

Keep open: the PR targets a real heartbeat/Telegram bug, but the current patch still marks ordinary trailing-whitespace replies as silent and the supplied proof does not exercise live Telegram delivery.

## What This Changes

The PR strips a final standalone `notify=false` marker from heartbeat fallback replies, passes `silent` into durable delivery, updates Telegram heartbeat test-shim coverage, and adds a changelog note.

## Best Possible Solution

Keep the fix at the heartbeat fallback normalization boundary, but only set `silent` when a standalone final `notify=false` marker is actually consumed, add negative whitespace coverage, and provide redacted live Telegram proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current main normalizes fallback heartbeat text without consuming `notify=false` and sends it through durable delivery without `silent`. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

No as written. The fix belongs in this boundary, but marker detection must compare the replacement before trimming ordinary whitespace, and the transport-visible behavior still needs live Telegram proof.
