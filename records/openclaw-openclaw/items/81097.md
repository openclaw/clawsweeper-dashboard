## Summary

Keep open: the PR targets a valid heartbeat/Telegram leak, but the current diff still has a silent false-positive bug and the supplied proof is not live Telegram proof for the transport-visible behavior.

## What This Changes

The PR strips trailing standalone `notify=false` markers from heartbeat fallback replies, forwards `silent` through durable delivery, updates heartbeat Telegram test-shim coverage, and adds a changelog entry.

## Best Possible Solution

Fix the helper so `silent` is set only when a standalone trailing marker actually matched, add a trailing-whitespace no-marker regression case, and provide redacted live Telegram proof for the visible delivery path.

## Reproduction Assessment

Yes for source-level reproduction: current main has a fallback heartbeat path that preserves a trailing `notify=false` as text and sends without `silent`, and the PR false positive is directly visible from the helper logic. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

No as written. The normalization boundary is the right place to fix the leak, but the helper must distinguish marker removal from ordinary trailing whitespace and the PR still needs live Telegram proof.

Overall correctness: patch is incorrect
