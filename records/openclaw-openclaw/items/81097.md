## Summary

Keep open: this is a plausible targeted fix for the linked Telegram heartbeat bug, but the patch currently marks ordinary trailing-whitespace replies as silent and the posted proof still does not exercise live Telegram delivery.

## What This Changes

The PR strips a final standalone `notify=false` marker from heartbeat fallback replies, passes `silent` into durable delivery, updates heartbeat Telegram test-shim coverage, and adds a changelog entry.

## Best Possible Solution

Keep the fix at the heartbeat fallback normalization boundary, but only set `silent` when the marker replacement actually matched and then prove the result with a redacted live Telegram run.

## Reproduction Assessment

Yes. Source inspection shows current main can deliver fallback heartbeat text containing a trailing standalone `notify=false` without converting it into `silent` delivery metadata.

## Solution Assessment

No as written. The owner boundary is right, but the helper should compare marker removal before trimming ordinary whitespace, and the PR still needs live Telegram proof for the visible transport behavior.

## Review Findings

Overall correctness: patch is incorrect
