## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Refresh the exec baseline before enabling the guard:** `scripts/check-no-unjustified-exec.mjs:18-23`
  - body: The new check scans all non-test `src/**`, but `EXEC_IMPORT_ALLOWLIST` only includes `src/agents/cli-credentials.ts`. Current main also has `execSync` in `src/agents/sessions/resolve-config-value.ts` and a default `child_process` import in `src/infra/sqlite-wal.ts`, both of which this checker flags, so the fail-closed boundary shard will break after a rebase unless those surfaces are rewritten or explicitly justified.
  - confidence: 0.88
- **[P2] Parse imports without spanning semicolonless statements:** `scripts/check-no-unjustified-exec.mjs:87-88`
  - body: `CHILD_PROCESS_IMPORT_STMT_RE` can start at an earlier semicolonless import and consume through a later `child_process` import, causing `classifyImportSpecifier` to inspect the wrong binding list and miss a new `exec` or `execSync` import. Please use a real parser or add bounded statement handling plus a semicolonless regression test.
  - confidence: 0.9
- **[P3] Ignore comments and strings before counting shell true:** `scripts/check-no-unjustified-exec.mjs:168`
  - body: `SHELL_TRUE_RE` runs over raw source text, so a comment or string mentioning `shell: true` in a non-allowlisted file will fail CI even when no shell option exists. Strip comments/strings or parse object expressions before making this a fail-closed guard.
  - confidence: 0.78
- **[P3] Remove stale large-file allowlist entries:** `scripts/check-no-large-files.mjs:18-24`
  - body: Current main has no tracked files over the 3 MiB threshold in this checkout, but the new allowlist exempts two paths that are no longer tracked. Leaving those exceptions means the guard would silently permit reintroducing large files at those paths without a fresh review.
  - confidence: 0.74
