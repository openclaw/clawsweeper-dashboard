Full review comments:

- **[P1] Baseline current exec surfaces before enabling the guard:** `scripts/check-no-unjustified-exec.mjs:18-22`
  - body: The proposed fail-closed guard only allowlists `src/agents/cli-credentials.ts`, but current main also has non-test risky import shapes in `src/agents/sessions/resolve-config-value.ts`, `src/commands/doctor/shared/legacy-oauth-sidecar.ts`, and `src/infra/sqlite-wal.ts`. A rebase that keeps this list will either fail the new check or force unreviewed allowlist churn, so migrate or explicitly justify those surfaces with coverage before wiring it into CI.
  - confidence: 0.91
- **[P1] Parse imports without semicolon-sensitive spans:** `scripts/check-no-unjustified-exec.mjs:87-88`
  - body: The import regex can start at an unrelated semicolonless import and continue through the next `node:child_process` import, causing the classifier to inspect the wrong binding block and miss `exec` or `execSync`. Use a parser or an import-declaration scanner that cannot consume across declarations, and add a semicolonless regression test.
  - confidence: 0.9
- **[P2] Ignore comments and strings when scanning shell true:** `scripts/check-no-unjustified-exec.mjs:168-172`
  - body: `findShellTrueViolations` scans raw file text, so a comment or error string mentioning `shell: true` in any non-allowlisted `src/**` file can fail the new CI guard even when no shell option exists. Strip comments/strings or parse object literals before making this fail closed.
  - confidence: 0.82
