## Summary

Keep open. Current main still lacks the proposed hygiene checks, but the PR is not ready to merge because the new exec-surface guard can miss a valid risky import pattern and can also fail on non-code `shell: true` text.

## What This Changes

The PR generalizes Swift-kit `.gitignore` entries and adds large-file plus unjustified-exec boundary checks with package-script wiring and tests.

## Best Possible Solution

Land a revised version of this hygiene work that keeps the narrow CI scope, uses parser-backed or otherwise statement-safe import detection, ignores comments and string literals for `shell: true`, and keeps the large-file allowlist limited to files that currently exceed the configured threshold.

## Reproduction Assessment

Yes. Current-main absence is reproducible by inspecting `.gitignore`, `package.json`, and `scripts/run-additional-boundary-checks.mjs`; the blocking PR bug is reproducible with a semicolonless unrelated import immediately before an `exec` import from `node:child_process`.

## Solution Assessment

No. The direction is maintainable, but the proposed exec guard is not the best mergeable solution until it scans valid TypeScript import statements safely and avoids raw-text false positives for `shell: true`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Detect semicolonless child_process imports:** `scripts/check-no-unjustified-exec.mjs:88`
  - body: The import matcher can start at an earlier semicolonless import and consume through the risky `node:child_process` import. In that case the classifier only sees the first brace block, so a valid `import { exec } from "node:child_process"` can enter `src/**` without a violation.
  - confidence: 0.94
- **[P3] Ignore comments and strings before counting shell: true:** `scripts/check-no-unjustified-exec.mjs:168-171`
  - body: `findShellTrueViolations` scans raw file text, so a comment or string literal containing `shell: true` in a non-allowlisted source file would fail CI even though no shell option is used. Strip or parse non-code text before counting matches.
  - confidence: 0.9
- **[P3] Remove under-threshold files from the large-file allowlist:** `scripts/check-no-large-files.mjs:23`
  - body: `canvas-cool.png` is currently below the 3 MiB default threshold, so allowlisting it lets that path grow past the limit later without tripping the guard. Keep the allowlist to files that actually exceed the threshold today.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff adds a security-oriented exec-surface guard, but the current import scanner can miss a new risky `exec` import in valid TypeScript.

Concerns:

- **[medium] Exec import guard can be bypassed:** `scripts/check-no-unjustified-exec.mjs:88`
  - body: Because `CHILD_PROCESS_IMPORT_STMT_RE` uses `[^;]*?` as the statement boundary, a semicolonless import before a `node:child_process` import can be consumed as one match and the risky binding can be missed. That weakens the guard's protection against new shell execution surfaces.
  - confidence: 0.94
