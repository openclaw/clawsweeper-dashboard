## Summary

Keep open. Current main still reads post-compaction context from the runner process cwd, while this PR makes the narrow code change to use the queued run workspace and adds focused regression coverage; no blocking review findings or security concerns were found.

## What This Changes

The PR changes the auto-reply runner to pass `followupRun.run.workspaceDir` into the post-compaction AGENTS.md refresh, adds a cwd-differs-from-workspace regression test, and records the user-facing fix in `CHANGELOG.md`.

## Best Possible Solution

Merge this PR after normal maintainer review and CI, then let the linked bug report close through the PR's closing reference once the fix lands.

## Reproduction Assessment

Yes. On current main, a source-level reproduction is to trigger the `autoCompactionCount > 0` path with a `sessionKey` while `process.cwd()` differs from `followupRun.run.workspaceDir`, then observe that `readPostCompactionContext` receives cwd instead of the queued workspace.

## Solution Assessment

Yes. Passing the required queued `workspaceDir` is the narrowest maintainable fix, matches the adjacent preflight compaction path, and avoids a cwd fallback that would preserve the bug when the run contract already supplies a workspace.
