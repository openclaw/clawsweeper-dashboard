## Summary

Keep open: current main still treats ordinary concurrent session-file appends during a released embedded prompt lock as a fatal takeover, and the new parallel sub-agent reproduction is useful even though it overlaps existing session-takeover reports.

## What This Changes

The issue reports that parallel embedded sub-agent/tool workflows can hit `EmbeddedAttemptSessionTakeoverError` when the session JSONL changes while an embedded prompt lock is released.

## Best Possible Solution

Keep the takeover fence strict for true external mutations while adding a narrow, tested ownership or queue path for legitimate OpenClaw-owned sub-agent/completion writes.

## Reproduction Assessment

Yes, source-reproducible at medium confidence: current main still throws on generic session-file changes while the embedded prompt lock is released, and upstream Codex has real parallel spawn/wait flows. The exact live 3+ sub-agent timing was not run in this read-only review.

## Solution Assessment

Unclear. A soft checksum check is plausible, but the safer maintainable direction is to classify or queue only OpenClaw-owned concurrent writes so real session takeovers still fail closed.
