## Summary

Keep open: this is a focused, source-backed TUI bug fix, and current main still has the one-run boolean optimistic marker that can leave burst runs untagged. I found no blocking correctness issue in the patch, but the required CI state should be made green before merge.

## What This Changes

The PR changes TUI optimistic-send state from a boolean to a counter, binds first-sighted burst run IDs as local, and adds burst/overlap regression tests plus a changelog entry.

## Best Possible Solution

Land the TUI counter and first-sighting local-binding fix with its focused regression tests once required checks and normal maintainer review are complete.

## Reproduction Assessment

Yes. Source inspection on current main shows burst sends can leave later run IDs untagged, and https://github.com/NVIDIA/NemoClaw/issues/3145 provides matching live Ubuntu TUI symptoms.

## Solution Assessment

Yes. The counter plus first-sighting binding is the narrowest maintainable TUI-only fix and avoids changing gateway protocol, queueing, or history-reload policy.
