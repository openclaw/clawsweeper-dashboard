## Summary

Keep open: current main still has the single boolean optimistic-send marker and no-active-run-only binding, while this PR applies a narrow TUI fix with regression coverage and sufficient terminal proof. I found no blocking correctness or security finding in the patch.

## What This Changes

The PR changes TUI optimistic-send tracking from a boolean to a counter, binds first-sighted pending gateway runs as local, adds burst/overlap regression tests, and updates the changelog.

## Best Possible Solution

Land the TUI counter and first-sighting local-binding fix with the regression tests and changelog once required checks and maintainer review are complete.

## Reproduction Assessment

Yes. A high-confidence source path exists: queue several normal TUI sends before gateway chat events, and current main only lets the first event consume the boolean local marker while later finals can reach the non-local `loadHistory()` path; https://github.com/NVIDIA/NemoClaw/issues/3145 provides the matching live transcript symptom.

## Solution Assessment

Yes. The counter plus first-sighting binding is the narrowest maintainable TUI-only fix for the broken optimistic-send contract and avoids protocol, queueing, or history-reload policy changes.
