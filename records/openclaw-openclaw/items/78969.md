## Summary

Keep open: current main still has a source-reproducible TUI burst-send race, and this PR is the focused fix path with regression coverage and sufficient terminal proof. I found no blocking correctness or security issue, but the branch is currently dirty/conflicting and needs a rebase plus fresh required checks before merge.

## What This Changes

The PR changes TUI optimistic-send tracking from a boolean to a numeric pending counter, binds first-seen non-BTW gateway runs as local, adds burst/overlap regression tests, and updates the changelog.

## Best Possible Solution

Land the TUI-only optimistic-send binding fix with regression coverage once it is rebased cleanly and required checks are green.

## Reproduction Assessment

Yes. The source path is high-confidence: queue several normal TUI sends before gateway chat events, and current main only lets the first event consume the boolean local marker while later finals can reach the non-local `loadHistory()` path; the NemoClaw report supplies the matching live transcript symptom.

## Solution Assessment

Yes. The counter plus first-sighting local-binding is the narrowest maintainable TUI-only fix for the broken optimistic-send contract and avoids protocol, queueing, or history-reload policy changes.
