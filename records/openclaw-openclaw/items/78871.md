## Summary

This can close because the reported failing package/path is outside this repository: current OpenClaw ships `@openclaw/feishu`, and its Feishu command replies flow through the same render-mode-aware reply dispatcher as normal assistant replies.

## What This Changes

Have Feishu system command replies such as `/status` honor `channels.feishu.renderMode = "card"` with a plain-text fallback.

## Best Possible Solution

Keep the in-repo `@openclaw/feishu` dispatcher path as-is; fixes for `@larksuite/openclaw-lark` should be made in that plugin, or users should migrate to current `@openclaw/feishu`.

## Reproduction Assessment

No. The reported external package and plain-text dispatch log do not exist in current main, while current Feishu command replies flow through a render-mode-aware dispatcher; I did not run a live Feishu account.

## Solution Assessment

Not applicable as a core patch. The narrow maintainable path is to keep OpenClaw's bundled plugin on the shared dispatcher and fix the external `openclaw-lark` command path in its own plugin if that package still bypasses cards.
