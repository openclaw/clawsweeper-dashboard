## Summary

Keep this PR open: current main still leaves meaningful one-shot bundled MCP cleanup gaps, and the related source PR has credible leak evidence, but this patch should not merge as-is because its session runtime manager can still race when replacing a stale in-flight runtime.

## What This Changes

This PR threads one-shot bundled MCP cleanup through isolated cron, isolated heartbeat, queued follow-up, and reply-runner paths, hardens session MCP runtime disposal, and adds regression tests plus a changelog entry.

## Best Possible Solution

Revise this PR or create a narrow replacement that keeps the one-shot cleanup boundary, preserves the #68450 credit and changelog entry, adds a post-await recheck or equivalent per-session single-flight guard in the runtime manager, and validates the touched MCP, cron, heartbeat, and follow-up paths.

## Reproduction Assessment

Yes. The source PR provides a high-confidence live A/B reproduction for the underlying leak, current main source inspection shows the missing cleanup paths, and the PR race can be reproduced by interleaving another same-session getOrCreate while the stale in-flight replacement path is awaiting the old promise.

## Solution Assessment

No as-is. The cleanup direction is the narrow maintainable fix, but the runtime-manager replacement path must re-read createInFlight and runtimesBySessionId after awaiting the stale promise before creating another runtime.

## Review Findings

Overall correctness: patch is incorrect
