## Summary

Keep open: the cleanup target is valid and not fully implemented on current main, but the branch still leaves a same-session in-flight runtime race in the manager code it edits.

## What This Changes

The branch propagates one-shot bundled MCP cleanup through isolated cron CLI, isolated heartbeat, queued follow-up, and reply-runner paths, hardens session runtime disposal, and adds tests plus a changelog entry.

## Best Possible Solution

Keep the narrow one-shot cleanup boundary, repair the runtime manager with a post-await state recheck or per-session single-flight guard, and merge after the focused regression tests plus changed gate pass.

## Reproduction Assessment

Yes. The leak is source-reproducible from current main’s missing cleanup propagation and isolated rollover early return, and #68450 provides live A/B connection accumulation evidence.

## Solution Assessment

No as-is. The cleanup direction is the narrow maintainable fix, but the runtime manager must re-read or serialize same-session state after awaiting stale in-flight disposal before it creates a replacement.
