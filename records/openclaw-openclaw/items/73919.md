## Summary

Keep open. Current main still has the isolated one-shot bundled MCP cleanup gaps this PR targets, and the related source PR provides useful live A/B evidence, but the PR head still has a P2 same-session runtime replacement race that should block merge.

## What This Changes

The PR propagates bundled MCP cleanup to isolated cron, isolated heartbeat, and queued follow-up one-shot runs, adjusts session MCP runtime disposal, adds regression tests, and updates the changelog.

## Best Possible Solution

Keep the focused one-shot cleanup boundary, but re-read or serialize same-session runtime state after awaiting stale in-flight disposal before installing a replacement runtime.

## Reproduction Assessment

Yes. The leak is source-reproducible on current main from the missing cleanup propagation and isolated cron rollover paths, and #68450 adds live A/B evidence for connection accumulation under isolated cron and heartbeat load.

## Solution Assessment

No as-is. The PR is the right narrow direction, but the runtime manager needs a post-await recheck or equivalent per-session single-flight guard before the patch is mergeable.
