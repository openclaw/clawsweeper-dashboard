## Summary

Keep open. Current main still lacks several one-shot bundled MCP cleanup paths, and this PR remains useful, but the PR head is merge-conflicting and its session-runtime replacement path can still create or return an untracked duplicate runtime.

## What This Changes

The PR propagates bundled MCP run-end cleanup through isolated cron CLI, isolated heartbeat, and queued follow-up runs, removes the isolated cron rollover skip, adjusts session-runtime disposal guards, and adds regression tests plus a changelog entry.

## Best Possible Solution

Refresh the focused cleanup patch against current main, add a post-await same-session recheck or serialization in createSessionMcpRuntimeManager, and keep the isolated cron/heartbeat/follow-up regression coverage without broad unrelated plugin changes.

## Reproduction Assessment

Yes, at source level. Current main still omits cleanup propagation in isolated cron CLI, isolated heartbeat, cron rollover, and queued follow-up paths, and the source PR includes live A/B connection-count evidence for the leak.

## Solution Assessment

No as-is. The cleanup direction is the narrow maintainable fix, but the runtime-manager replacement path needs a post-await state recheck or serialization and the PR branch must be refreshed before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Recheck same-session state before creating a replacement runtime:** `src/agents/pi-bundle-mcp-runtime.ts:507-533`

Re-read createInFlight/runtimesBySessionId after the awaits or serialize replacement creation.
