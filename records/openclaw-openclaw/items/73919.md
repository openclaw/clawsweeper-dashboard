## Summary

Keep open. The PR targets a real bundled MCP lifecycle leak that is still visible on current main for isolated cron rollover, CLI cron, isolated heartbeat, and queued follow-up paths. The patch is useful and narrow, but it still has a same-session runtime-manager race in the different-fingerprint getOrCreate path, so it needs one focused repair before merge.

## What This Changes

This PR adds one-shot bundled MCP cleanup propagation for isolated cron, isolated heartbeat, and queued follow-up runs, hardens session MCP runtime disposal guards, adds regression tests, and adds a changelog entry crediting the source PR.

## Best Possible Solution

Keep the one-shot cleanup boundary and credit from #68450, but add a post-await recheck or equivalent per-session single-flight guard in the runtime manager before creating a replacement runtime. The final patch should prove isolated cron rollover, CLI cron, isolated heartbeat, queued follow-up, reply-runner propagation, cleanup-error isolation, and the same-session race.

## Reproduction Assessment

Yes. The linked source PR provides live A/B reproduction for the underlying leak, and current-main source inspection confirms the missing cleanup paths. The PR-specific race can be reproduced by interleaving another same-session getOrCreate while the different-fingerprint path awaits the stale in-flight promise.

## Solution Assessment

No as-is. The cleanup direction is the narrow maintainable solution, but the runtime-manager replacement path must re-read the session runtime and in-flight maps after awaiting the stale promise before it creates and publishes another runtime.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Recheck the session before creating a replacement runtime:** `src/agents/pi-bundle-mcp-runtime.ts:509-516`
