## Summary

Keep open. Current main still leaves several isolated one-shot bundled MCP cleanup paths uncovered, and #68450 provides credible leak reproduction, but this PR is not merge-ready because its runtime-manager change can overwrite or leak a newer same-session runtime under concurrent replacement.

## What This Changes

The PR propagates bundled MCP cleanup through isolated cron, heartbeat, queued follow-up, and reply-runner paths, removes the isolated cron rollover skip, hardens session runtime disposal, and adds tests plus a changelog entry.

## Best Possible Solution

Keep the focused one-shot cleanup direction, add a post-await session recheck or equivalent per-session single-flight guard before runtime replacement, and preserve the targeted regression coverage and #68450 attribution.

## Reproduction Assessment

Yes. #68450 gives a high-confidence live A/B reproduction for the leak, current-main source inspection shows the missing cleanup propagation paths, and the PR race can be reproduced by interleaving another same-session getOrCreate while the stale in-flight replacement path awaits.

## Solution Assessment

No as-is. The cleanup boundary is the narrow maintainable fix, but the runtime manager must re-read or serialize session state after awaiting stale in-flight creation before creating and publishing a replacement runtime.

## Review Findings
