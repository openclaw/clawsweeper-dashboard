## Summary

Keep open. The MCP one-shot cleanup problem is real and still visible on current main for heartbeat, queued follow-up, cron CLI, and isolated cron rollover paths, but this PR still has a session runtime replacement race that should be fixed before merge.

## What This Changes

The PR propagates one-shot bundled MCP runtime cleanup through isolated cron, isolated heartbeat, queued follow-up, and reply-runner paths, hardens session runtime disposal, and adds regression tests plus a changelog entry.

## Best Possible Solution

Land the narrow one-shot cleanup after adding a post-await session recheck or equivalent per-session single-flight guard, preserving focused coverage for isolated cron, heartbeat, follow-up, and reply-runner cleanup.

## Reproduction Assessment

Yes. #68450 provides a high-confidence live A/B reproduction for the leak, current-main source inspection shows the missing cleanup paths, and the PR race can be reproduced by interleaving another same-session getOrCreate while the stale in-flight replacement path awaits.

## Solution Assessment

No as-is. The cleanup boundary is the right narrow direction, but the runtime manager must re-read or serialize session state after awaiting stale in-flight creation before it creates or stores a replacement runtime.
