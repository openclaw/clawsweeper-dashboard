## Summary

Keep open. The PR is focused on a real one-shot bundled MCP runtime leak that current main still only partially covers, but the proposed runtime-manager change still has a same-session replacement race that should block merge.

## What This Changes

The PR adds one-shot bundled MCP cleanup propagation for isolated cron, isolated heartbeat, queued follow-up, and reply-runner paths, hardens session MCP runtime disposal, and adds tests plus a changelog entry.

## Best Possible Solution

Keep the narrow cleanup direction, add a post-await session recheck or equivalent per-session single-flight guard before runtime replacement, and preserve focused regression coverage for isolated cron, heartbeat, follow-up, and reply-runner cleanup.

## Reproduction Assessment

Yes. #68450 gives a high-confidence live A/B reproduction for the leak, current-main source inspection shows the missing cleanup paths, and the PR race can be reproduced by interleaving another same-session getOrCreate while the stale in-flight replacement path awaits.

## Solution Assessment

No as-is. The one-shot cleanup boundary is the narrow maintainable fix, but the runtime manager must re-read or serialize session state after awaiting stale in-flight creation before replacement.
