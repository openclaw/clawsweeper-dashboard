## Summary

Keep open: the PR addresses a real source-reproducible bundled MCP runtime leak and preserves the useful #68450 direction, but it still has a same-session runtime-manager race that should block merge until repaired.

## What This Changes

The PR propagates one-shot bundled MCP cleanup through isolated cron, isolated heartbeat, queued follow-up, and reply-runner paths, hardens session runtime disposal, and adds tests plus a changelog entry.

## Best Possible Solution

Keep the narrow one-shot cleanup boundary, but add a post-await session recheck or equivalent per-session single-flight guard before creating a replacement runtime, with regression coverage for the interleaving.

## Reproduction Assessment

Yes. Source inspection of current main shows the missing cleanup propagation in isolated CLI cron, isolated heartbeat, queued follow-up, and isolated cron rollover paths, and #68450 supplies live A/B accumulation evidence.

## Solution Assessment

No as-is. The cleanup placement is the narrow maintainable direction, but the runtime manager must re-read or serialize same-session state after awaiting a stale in-flight runtime before it creates or stores a replacement.
