## Summary

Keep open. The underlying bundled MCP leak is still present on current main, and this PR is the right narrow cleanup direction, but the proposed runtime-manager change still leaves a same-session replacement race that can orphan a newer runtime.

## What This Changes

The PR propagates one-shot bundled MCP cleanup through isolated cron, isolated heartbeat, queued follow-up, and reply-runner paths, hardens session MCP runtime disposal, and adds tests plus a changelog entry.

## Best Possible Solution

Keep the narrow one-shot cleanup boundary, but add a post-await session recheck or equivalent per-session single-flight guard before creating a replacement runtime, with regression coverage for the interleaving.

## Reproduction Assessment

Yes. The underlying leak is source-reproducible on current main from the missing cleanup propagation paths, and #68450 provides live A/B evidence for isolated cron and heartbeat accumulation.

## Solution Assessment

No as-is. The PR's cleanup direction is the narrow maintainable fix, but the runtime manager must re-read or serialize same-session state after awaiting a stale in-flight runtime before installing a replacement.
