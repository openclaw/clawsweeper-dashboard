## Summary

Keep open: the cleanup direction is valid and still addresses current-main leak paths, but the PR leaves a same-session post-await interleaving in the runtime manager that can create and store a second replacement runtime.

## What This Changes

The PR adds one-shot bundled MCP cleanup propagation for isolated cron, isolated heartbeat, and queued follow-up runs, updates session MCP runtime disposal guards, adds tests, and records a changelog fix.

## Best Possible Solution

Keep the focused one-shot cleanup boundary, but make session runtime replacement recheck or serialize same-session state after awaits before landing.

## Reproduction Assessment

Yes, from source inspection rather than a live run in this read-only review. Current main lacks cleanup propagation in several one-shot paths, and the linked source PR provides live A/B evidence of connection growth versus a flat fixed branch.

## Solution Assessment

No as-is. The PR's cleanup direction is the right narrow fix, but the runtime manager needs a post-await recheck or per-session single-flight guard before creating a replacement runtime.
