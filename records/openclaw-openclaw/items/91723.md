## Summary

Keep open: current main and v2026.6.5 still use the Teams SDK stream-final path implicated by the report, while the linked draft PR remains open and its author’s production check disproved the original failed-stream fallback theory.

## What This Changes

Prevent Microsoft Teams partial streaming from leaving both a streamed preview and a duplicate final message for replies over 4,000 characters.

## Best Possible Solution

Land a narrow Teams-plugin repair for the proven SDK preview/final collapse mechanism that renders one canonical reply for both short and over-4,000-character DMs, preserves final-message affordances, and keeps failed-stream fallback state scoped to the affected segment.

This read-only review did not have a live Teams tenant to independently execute it.

## Solution Assessment

No for the original prefix-trimming proposal: its production check still duplicated without entering the failed-stream fallback. The best fix must target the demonstrated SDK preview/final activity path and prove that final metadata remains intact.
