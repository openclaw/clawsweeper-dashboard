## What This Changes

The issue reports that Microsoft Teams partial streaming persists both the streamed preview and a final or chunked fallback message for replies longer than about 4000 characters after the Teams SDK rebase.

## Best Possible Solution

Keep this issue open until the linked draft is reworked or replaced by a focused msteams streaming fix with regression coverage and redacted real Teams long-reply proof.

## Reproduction Assessment

Yes, source-reproducible but not live-reproduced in this review. Current main still stores and can return the full pending final payload after streamed text was emitted, and the latest linked PR discussion confirms the real Teams duplicate remains unresolved.

## Solution Assessment

No current-main solution is in place. The likely best fix is a narrow msteams controller or SDK-finalization repair that prevents duplicate visible delivery while preserving final metadata, not disabling Teams streaming globally.
