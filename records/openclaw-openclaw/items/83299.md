## Summary

Keep open: the isolated restart benchmark tooling is plausible and builds on recent gateway restart diagnostics, but the patch currently breaks its advertised machine-readable JSON mode and still needs stronger after-fix proof of a real restart benchmark run.

## What This Changes

The PR adds a `pnpm test:restart:gateway` package script plus a new Gateway restart benchmark script and focused tests for CLI parsing, trace parsing, restart intent writing, probes, and fixture generation.

## Best Possible Solution

Keep the benchmark isolated under `scripts/`, but make machine-readable output parseable, add a regression test for `--json`, and supply a real Gateway restart benchmark run before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: run a completing case with `--json`, and the progress `console.log()` calls in `runCase()` will be written before the final JSON payload.

## Solution Assessment

No, not yet. The isolated script approach matches existing benchmark tooling, but progress logs should move to stderr or be suppressed in JSON mode, and the branch needs real benchmark proof.
