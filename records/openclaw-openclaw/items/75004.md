## Summary

Keep open: this automerge-opted maintainer PR still has a narrow correctness blocker. The exact head exposes Tree-sitter byte offsets as `SourceSpan` indexes while tests and likely consumers treat them as JavaScript string indexes, so non-ASCII command sources will produce incorrect spans.

## What This Changes

The PR branch adds a Tree-sitter-backed internal shell command explainer under `src/infra`, parser dependencies and build-policy wiring, focused tests, an index export, and a changelog entry.

## Best Possible Solution

Land the explainer after `SourceSpan` coordinates are Unicode-safe and exact-head validation passes, keeping it internal until approval UI integration is intentionally added.

## Reproduction Assessment

Yes for the PR blocker: exact-head source inspection plus the `web-tree-sitter` type contract show byte indexes being exposed where tests consume string indexes. I did not run tests because this review was read-only.

## Solution Assessment

No, not as submitted. Tree-sitter is a reasonable parser base, but the span coordinate contract needs byte-to-string-index conversion or an explicitly byte-based public contract with matching callers and tests.
