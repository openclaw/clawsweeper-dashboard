## Summary

Keep open. Current main does not already contain the shell command explainer, and the PR is a draft maintainer-labeled member PR that still has a concrete resource-cleanup bug in the latest head: `explainShellCommand` parses a Tree-sitter tree and returns derived data without deleting the tree.

## What This Changes

The PR adds a Tree-sitter-backed internal shell command explainer under `src/infra/command-explainer`, focused tests, a changelog entry, and runtime dependency/lockfile entries for `web-tree-sitter` and `tree-sitter-bash`.

## Best Possible Solution

Keep the Tree-sitter explainer direction, but make tree ownership explicit by freeing parsed trees with `try/finally` before returning derived explanation data; then rerun the focused explainer test and changed gate before wiring it into approval UI or allowlist decisions.

## Reproduction Assessment

Yes for the review finding: the latest diff shows `explainShellCommand` creates a `web-tree-sitter` `Tree` and returns without deleting it, while the dependency types document `Tree.delete()`. I did not execute tests because this was a read-only review.

## Solution Assessment

No, not yet. A Tree-sitter-backed explainer is a reasonable implementation direction, but the current patch should release parsed trees before the code is used in long-lived command-review flows.
