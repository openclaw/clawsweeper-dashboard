## Summary

## What This Changes

The PR replaces the Anthropic provider's hardcoded Claude CLI user-agent version with a cached runtime `claude --version` lookup in `src/llm/providers/anthropic.ts`.

## Best Possible Solution

Use one tested Claude CLI version resolver across both Anthropic OAuth builders, avoid shell-string execution or tie the probe to the configured Claude CLI command, and require redacted live OAuth proof before merge.

## Reproduction Assessment

Yes by source inspection: current main and `v2026.6.8` send `claude-cli/2.1.75` in both Anthropic OAuth builders, and the linked issue reports the live `authentication_error`. I did not live-reproduce Anthropic's rejection without reporter credentials.

## Solution Assessment

No; the proposed fix is too narrow because the managed Anthropic Messages transport still uses the stale constant, and the runtime `execSync` approach needs security and ownership hardening.
