## Summary

Keep open: the PR targets a real P1 Anthropic OAuth bug, but the current patch can throw during provider/transport import when Claude Code is only available as OpenClaw's supported external CLI package, and the added resolver test still contradicts the implementation's validation rule.

## What This Changes

This PR adds a shared Claude Code version resolver, switches the legacy Anthropic provider and managed Anthropic transport OAuth user-agent headers to that shared value, and adds resolver tests.

## Best Possible Solution

Use one Claude Code version source for both Anthropic OAuth builders that is lazy, compatible with OpenClaw's supported external CLI packaging, covered by focused tests, and backed by redacted patched OpenClaw runtime or network proof.

## Reproduction Assessment

## Solution Assessment

No, not yet. Sharing one resolver across both headers is the right direction, but this patch resolves an undeclared external CLI package at module load and still has a validation/test mismatch.
