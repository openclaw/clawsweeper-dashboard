## Summary

## What This Changes

The branch replaces the two hardcoded Anthropic OAuth `claude-cli/2.1.75` user-agent constants with a shared request-time helper that reads `@anthropic-ai/claude-code/package.json`, plus resolver unit tests.

## Best Possible Solution

Land one canonical fix that removes the stale `2.1.75` constants in both OAuth builders while sourcing the Claude version from a supported runtime or dependency contract and proving both OpenClaw paths.

Yes for the source-level failure: current main and v2026.6.9 still build both Anthropic OAuth user-agent headers from `2.1.75`. I did not run a credentialed Anthropic OAuth request, so the upstream rejection is not live-reproduced here.

## Solution Assessment

No. Sharing a call-time helper across both headers is the right direction, but using undeclared Node package metadata as the only version source is not the safest owner boundary for supported CLI and token layouts.
