## Summary

Keep open. Current main has Feishu broadcast handling for platform-delivered inbound group messages, but it still does not implement the requested send-time synthetic fan-out for bot-authored Feishu group sends; related #46776 overlaps the Feishu multi-agent problem space but does not fully replace this request.

## What This Changes

Request an opt-in Feishu send-time synthetic broadcast so bot-authored group messages are delivered to other registered OpenClaw Feishu bots with loop protection.

## Best Possible Solution

Keep this as a Feishu plugin-owned design item; if approved, add opt-in post-send fan-out only after successful group sends, mark injected turns as synthetic bot-originated, enforce dedupe and consecutive-bot-turn limits, and document the config/tests.

## Reproduction Assessment

Yes, at source level. The reporter describes a concrete multi-bot Feishu group setup, and current main’s outbound Feishu send paths return after direct delivery with no synthetic peer-event injection.

## Solution Assessment

Unclear. Send-time fan-out is a plausible compensation for Feishu’s platform behavior, but the maintainable shape depends on maintainer decisions about metadata, loop safety, dedupe, and overlap with existing inbound broadcast groups.
