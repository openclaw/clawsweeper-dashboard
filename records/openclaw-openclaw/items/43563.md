## Summary

Keep open. Current main has Feishu broadcast dispatch for platform-delivered inbound group events, but it still does not implement the requested send-time synthetic fan-out for bot-authored Feishu group messages; that remaining work adds user-facing behavior, config, and loop-safety policy.

## What This Changes

The issue requests opt-in Feishu send-time synthetic fan-out so peer bot accounts in the same group receive bot-authored group messages.

## Best Possible Solution

Keep this as Feishu plugin-owned design work and, if approved, add opt-in post-send fan-out after successful group sends with synthetic-origin metadata, peer registry/dedupe, loop limits, docs, and regression tests.

## Reproduction Assessment

Yes, at source level. Current main only fan-outs platform-delivered inbound Feishu group events, while outbound Feishu send paths return direct delivery results without injecting peer synthetic events.

## Solution Assessment

Unclear. Send-time fan-out is plausible, but the narrowest maintainable solution needs maintainer decisions on synthetic event metadata, loop limits, dedupe, config shape, and overlap with existing top-level broadcast behavior.
