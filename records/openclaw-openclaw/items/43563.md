## Summary

Keep open. Current main supports Feishu broadcast dispatch for platform-delivered inbound group events, but it still lacks the requested send-time synthetic fan-out for bot-authored Feishu group messages.

## What This Changes

Request Feishu to broadcast synthetic bot-originated group message events to other registered Feishu bots for Discord-like multi-bot collaboration.

## Best Possible Solution

Keep this as a Feishu plugin-owned design item and, if approved, add opt-in post-send fan-out only after successful group sends with synthetic-origin metadata, dedupe, loop limits, docs, and regression tests.

## Reproduction Assessment

Yes, at source level for the missing behavior. The reporter gives a concrete multi-bot Feishu group setup, and current main's outbound Feishu send paths return after direct delivery without peer-event injection.

## Solution Assessment

Unclear. Send-time fan-out is plausible, but the maintainable solution depends on maintainer decisions about synthetic event metadata, loop safety, dedupe, config shape, and overlap with existing inbound broadcast groups.
