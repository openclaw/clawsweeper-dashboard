## Summary

Close as ClawHub scope. The requested Feishu bot-to-bot workaround is useful, but current main still lacks a built-in send-time synthetic fan-out contract, and related maintainer cleanup has already routed this optional protocol-style workflow to ClawHub rather than a new bundled OpenClaw config surface.

## What This Changes

The issue requests opt-in Feishu cross-bot broadcast for bot-authored group messages, including loop protection and a new `channels.feishu.crossBotBroadcast` config.

## Best Possible Solution

Keep bot-to-bot Feishu relay protocols in ClawHub/optional plugin space, and only reopen core work for a specific missing extension API needed by an external implementation.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request for a platform workaround. Source inspection confirms current main has inbound Feishu broadcast support but no send-time synthetic cross-bot relay or requested config.

## Solution Assessment

Yes for cleanup: the maintainable path is ClawHub/plugin-scope now, because optional protocol work already exists outside core and related Feishu bot-to-bot reports were routed there. A core change should wait for a concrete missing API rather than adding a Feishu-specific policy surface here.
