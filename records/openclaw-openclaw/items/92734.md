## Summary

Keep open: the report matches current source and latest-release behavior, and the linked open PR is the active fix path. This issue should close only after that PR lands or maintainers choose a different fix.

## What This Changes

The issue reports Signal media captions are truncated to one character because the shared Markdown chunker treats an Infinity limit as the fallback limit of 1.

## Best Possible Solution

Land a narrow shared Markdown chunker fix that treats positive Infinity as unbounded while preserving NaN and finite-limit behavior, with regression coverage and Signal caption proof.

## Reproduction Assessment

Yes. Source inspection shows the Signal media path passes Number.POSITIVE_INFINITY into the shared chunker, where current main normalizes non-finite limits to 1; I did not run a live Signal send in this read-only review.

## Solution Assessment

Yes. The best fix is in the shared Markdown chunker rather than a Signal-only special case, because the bad behavior comes from the shared limit normalization contract.
