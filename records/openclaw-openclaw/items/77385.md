## Summary

Keep this PR open: the stale-binding cleanup is useful, but timeout/unknown recovery is still retried outside the CLI runner after a failed `agent_end`, and the changelog entry still needs contributor credit.

## What This Changes

The PR clears stale CLI session bindings for all `FailoverError` reasons, retries recoverable reasons with a fresh CLI session, rethrows non-recoverable reasons after clearing, adds CLI attempt tests, and updates `CHANGELOG.md`.

## Best Possible Solution

Land stale-session cleanup for all resumed-session `FailoverError`s while keeping recoverable retry inside a hook-aware CLI runner seam and preserving clear-and-rethrow behavior for non-recoverable reasons.

## Reproduction Assessment

Yes, by source inspection. Current main only clears stored CLI bindings for `session_expired`, and the PR’s timeout/unknown retry runs after `runPreparedCliAgent` has already emitted a failed `agent_end`.

## Solution Assessment

No. Clearing the stale binding is the right store-boundary fix, but recoverable fresh-session retry should happen before the CLI runner emits the final failure hook.

## Review Findings
