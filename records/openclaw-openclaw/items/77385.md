## Summary

This PR is still useful and addresses a real stale-session cleanup bug, but the updated patch still retries timeout/unknown failures after the real CLI runner has already emitted a failed `agent_end` hook, so it can report a failed run that later succeeds.

## What This Changes

The PR changes CLI FailoverError handling to clear stored CLI session bindings for any resumed-session FailoverError, retry selected recoverable reasons, add regression tests, and update the changelog.

## Best Possible Solution

Land a hook-aware stale-session recovery that clears persisted bindings for all resumed-session FailoverErrors, retries recoverable ones before failed `agent_end` emission, and keeps final hook state truthful.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main only clears stored CLI bindings for `session_expired`, and the PR's timeout/unknown retry runs after `cli-runner.ts` has emitted a failed `agent_end` event.

## Solution Assessment

No, not as written. Clearing in the caller is the right store boundary for non-recoverable errors, but recoverable retry needs to happen at or before the CLI runner hook boundary with regression coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88
