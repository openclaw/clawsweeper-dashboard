## Summary

The PR targets a real stale CLI session bug, but it is not merge-ready because timeout/unknown retries still happen after `runPreparedCliAgent` has already emitted a failed `agent_end` hook, so a recovered run can report both failure and success to hook consumers.

## What This Changes

The PR widens CLI FailoverError handling in `attempt-execution` to clear stored CLI session bindings for all resumed-session failures, retry selected recoverable reasons, add mocked attempt-level tests, and update the changelog.

## Best Possible Solution

Land a hook-aware stale-session recovery that clears persisted bindings for all resumed-session FailoverErrors, retries recoverable failures before any failed `agent_end` is emitted, and keeps non-recoverable errors as clear-then-rethrow.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main only clears stored CLI bindings for `session_expired`, and this PR's timeout/unknown retry would run after `cli-runner.ts` has emitted a failed `agent_end` event.

## Solution Assessment

No, not as written. Clearing the binding in the caller is the right store boundary, but recoverable retries need to move under the CLI runner hook boundary or otherwise prove that recovered retries emit only the final successful end event.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Move retry before failed agent_end hook:** `src/agents/command/attempt-execution.ts:535-542`
  - body: The updated patch retries `timeout` and `unknown` from `attempt-execution` after `runCliAgent` returns. In the real CLI runner, non-`session_expired` FailoverErrors already emit `agent_end` with `success: false` before throwing, so a successful fresh retry from here reports a failed run and then returns success. Keep the stale binding cleanup, but move the recoverable retry before failed hook emission or add hook-aware coverage proving recovered retries emit only the final successful end event.
  - confidence: 0.88
