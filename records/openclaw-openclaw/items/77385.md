## Summary

Keep open: the PR addresses a real stale CLI-session cleanup gap, but the recoverable timeout/unknown retry is still outside the CLI runner hook boundary and can emit a failed final hook before returning a successful retry.

## What This Changes

The PR broadens stale CLI session cleanup in `attempt-execution`, retries `session_expired`/`timeout`/`unknown` with a fresh CLI session, rethrows non-recoverable reasons after clearing, adds focused tests, and updates the changelog.

## Best Possible Solution

Land hook-aware stale-session recovery that clears persisted bindings for every resumed-session FailoverError, retries recoverable reasons before failed `agent_end` emission, and rethrows non-recoverable reasons after clearing.

## Reproduction Assessment

Yes. Source inspection shows current main only clears stored CLI bindings for `session_expired`, and the PR’s timeout/unknown retry would run after `runPreparedCliAgent` has already emitted a failed `agent_end`.

## Solution Assessment

No, not as written. Clearing in `attempt-execution` is the right store boundary for non-recoverable errors, but recoverable retry should happen inside the CLI runner hook boundary with hook-aware regression coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Move recoverable retry before failed agent_end hooks:** `src/agents/command/attempt-execution.ts:535-543`
  - body: For `timeout` and `unknown`, `runCliAgent` reaches the non-`session_expired` FailoverError branch in `runPreparedCliAgent`, which emits `agent_end` with `success: false` before throwing. This new outer retry can then call `runCliAgent` again and return success, so hook consumers observe a failed final state followed by a success for one turn. Keep the stale-binding cleanup, but move recoverable retry inside the CLI runner hook boundary or add hook-aware coverage proving only the final end event is emitted.
  - confidence: 0.88
