## What This Changes

The PR replaces six Google OAuth project/token JSON parses with `readProviderJsonResponse` and adds three OAuth oversized-body regression tests.

## Best Possible Solution

Move the byte cap to the Google OAuth guarded fetch/read boundary, preserve the existing proxy and SSRF release behavior, and prove oversized streamed plus normal OAuth paths.

## Reproduction Assessment

Yes. Source inspection shows every changed OAuth reader receives a `Response` from `fetchWithTimeout`, and that helper already drains the upstream body with `arrayBuffer()` before the new cap can run.

## Solution Assessment

No. `readProviderJsonResponse` is the right primitive, but the best fix applies the limit before or inside the OAuth fetch helper so the original stream is capped and cancelled.
