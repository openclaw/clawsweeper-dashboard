## What This Changes

The PR adds optional `agentRuntime` metadata to Gateway `models.list`, documents/regenerates protocol types, and uses it in WebUI chat model picker labels.

## Best Possible Solution

Land after owners accept the optional `ModelChoice.agentRuntime` contract and WebUI-first scope, or expand native Swift display before merge if native picker parity is required for this surface.

## Reproduction Assessment

Yes for the source-level gap: current main's `ModelChoiceSchema` has no `agentRuntime` field, so model picker clients cannot receive runtime display metadata from `models.list`. I did not run a failing current-main UI scenario because this is an additive feature PR rather than a narrow bug fix.

## Solution Assessment

Yes, with maintainer acceptance: putting runtime metadata on `models.list` and consuming it in the WebUI label helper is the narrowest durable way to preserve canonical model refs while showing runtime identity. The unsettled part is product/API scope, especially whether native Swift picker parity is required before merge.
