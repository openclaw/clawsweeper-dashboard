## Summary

Keep open for normal maintainer review: current main does not expose model runtime metadata through `models.list` or WebUI picker labels, and this PR adds that additive surface with focused Gateway/UI proof. I found no blocking patch defect.

## What This Changes

The branch adds optional `agentRuntime` metadata to `models.list` rows, propagates it through protocol/UI/Swift types, computes metadata from existing runtime policy, and appends runtime labels in the WebUI model picker.

## Best Possible Solution

Land an additive Gateway/UI implementation that preserves canonical `provider/model` values while exposing runtime metadata, after maintainer review of the public protocol shape and normal CI gates.

## Reproduction Assessment

Yes for the behavior gap: current main's `ModelChoiceSchema` and UI model catalog type have no model-level runtime field, so the picker cannot show `openai/gpt-5.5` plus Codex runtime from `models.list`. I did not run a live current-main Gateway repro in this read-only review.

## Solution Assessment

Yes, this is a maintainable fix: it computes metadata at the Gateway model-list boundary from the existing runtime resolver and lets the picker display it without changing canonical model refs. Alias generation or config rewrites would create a second source of truth.
