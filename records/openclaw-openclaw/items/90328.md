## Summary

Keep open: current main still does not expose runtime metadata in `models.list` or WebUI picker labels, and the PR is a coherent implementation with sufficient real Gateway/WebUI proof. The remaining blocker is maintainer acceptance of the new public Gateway response field and whether WebUI-first scope is enough while native Swift chat still ignores the metadata.

## What This Changes

The PR adds optional Gateway `models.list` `agentRuntime` metadata, documents/regenerates the protocol surface, and renders non-default runtime labels in the WebUI chat model picker without changing canonical model ids.

## Best Possible Solution

Land a narrowly documented Gateway metadata contract and WebUI rendering if owners accept WebUI-first scope, or extend the native mapping/display in the same PR if the public model picker promise includes native chat.

## Reproduction Assessment

Yes from source inspection: current `models.list` does not include runtime metadata and current WebUI label helpers cannot render a runtime suffix. I did not run the app because this review is read-only, but the missing path is clear in current main.

## Solution Assessment

Yes for the WebUI/Gateway scope: the PR reuses the existing runtime metadata resolver instead of inventing a parallel source of truth, preserves canonical model ids, and documents the additive field. The best-fix uncertainty is scope, because native Swift chat currently maps the same model choices without using the runtime label.
