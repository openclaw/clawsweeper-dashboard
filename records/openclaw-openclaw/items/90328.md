## Best Possible Solution

Land the additive runtime metadata field only after maintainers explicitly accept the public Gateway protocol/client compatibility tradeoff; otherwise keep runtime distinction in existing alias/config surfaces until a broader protocol design is chosen.

## Reproduction Assessment

## Solution Assessment

Yes, if maintainers accept the public protocol field: the PR reuses existing runtime policy and label helpers instead of reintroducing `openai-codex` routes or relying on manual aliases. The safer alternative is to keep aliases only, but that would not solve automatic runtime observability.
