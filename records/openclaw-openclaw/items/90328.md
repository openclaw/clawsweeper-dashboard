## Best Possible Solution

Keep the additive `models.list`/picker design, fix nested model-id runtime resolution, add focused coverage, and provide redacted Gateway or WebUI real behavior proof before merge.

## Reproduction Assessment

No live reproduction was run in this read-only review. The nested-model metadata defect is source-reproducible from the new helper passing `entry.id` into a resolver that treats slash-prefixed model ids as provider-qualified refs.

## Solution Assessment

No; `models.list` plus the picker suffix is the right layer, but the branch should qualify nested model IDs and include real Gateway/UI proof before it is the best fix.
