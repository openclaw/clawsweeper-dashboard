## Summary

Keep open: the Gateway/UI approach looks well-scoped and the supplied proof is useful, but the generated OpenClawKit Swift initializer turns the additive optional protocol field into a source-compatibility break until the generator defaults `agentRuntime` to `nil`.

## What This Changes

The branch adds optional `agentRuntime` metadata to `models.list`, propagates it through protocol/native/UI types, and appends material runtime labels to WebUI model picker options.

## Best Possible Solution

Keep the additive Gateway/UI runtime metadata, but update the Swift protocol generator so `ModelChoice.agentRuntime` defaults to `nil` in the public initializer and rerun protocol/UI/Gateway validation before merge.

## Reproduction Assessment

Yes for the PR defect: source inspection shows the generated `ModelChoice` initializer requires `agentruntime`, and the generator only defaults `ModelChoice.available`. I did not run tests because this was a read-only review.

## Solution Assessment

No as-is: the Gateway/UI owner boundary is appropriate, but the native protocol generator should preserve Swift source compatibility for the additive field before merge.
