## What This Changes

The PR adds optional `agentRuntime` metadata to `models.list`, propagates it through generated/client types, documents the protocol field, and appends non-default runtime labels in the WebUI model picker.

## Best Possible Solution

Land the additive models.list metadata and WebUI label path after normal CI/maintainer review, keeping canonical `openai/*` model refs and treating `agentRuntime` as optional display metadata.

## Reproduction Assessment

Yes at source level: current main has the canonical OpenAI/Codex runtime split but the current `ModelChoice` and picker label paths do not expose runtime metadata, matching the ambiguity reported in #90420 and #84032.

## Solution Assessment

Yes: the PR uses existing runtime-policy and label helpers, adds an optional protocol field, and avoids route aliases or config changes, which is the narrowest maintainable fix for the picker observability gap.
