## Summary

Current main and the latest release do not expose runtime metadata on `models.list`, and this PR is a focused implementation with sufficient current-head proof and no blocking code findings; the remaining blocker is maintainer acceptance of the additive public Gateway/Swift client surface.

## What This Changes

The PR adds optional `agentRuntime` metadata to Gateway model choices, propagates it through generated Swift/UI types, and renders non-default runtime labels in the WebUI model picker without changing canonical model refs.

## Best Possible Solution

Land only after maintainers accept the optional Gateway protocol and Swift `ModelChoice` surface, while keeping model refs stable and runtime metadata optional.

## Solution Assessment

Yes, this is a good narrow implementation if maintainers want this public observability surface: Gateway already owns `models.list`, UI already formats catalog labels, and the runtime-policy resolver is reused instead of duplicating provider/runtime rules. Alias-only or status-only alternatives would leave the picker ambiguous or split the source of truth.
