## Best Possible Solution

Land the optional runtime metadata only after maintainers accept the `models.list` compatibility risk and exact-head protocol/UI/Gateway gates are green.

Yes, this is the best narrow shape: expose runtime metadata separately from the canonical model ref and let the picker render it, rather than changing model values or reviving legacy `openai-codex/*` routes. The remaining decision is compatibility acceptance for the additive Gateway response field.
