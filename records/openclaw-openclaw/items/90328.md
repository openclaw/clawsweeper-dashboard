## What This Changes

The PR adds optional `agentRuntime` metadata to Gateway `models.list` model choices, threads it through generated/Swift/UI types, and appends runtime labels in the WebUI model picker.

## Best Possible Solution

Land the optional protocol field only after maintainer acceptance of the additive `models.list` contract, keeping the canonical selected model ref stable while surfacing runtime metadata through generated clients and the WebUI picker.

## Reproduction Assessment

Yes. Current main has no `ModelChoice.agentRuntime` field, and the existing runtime resolver can source Codex runtime metadata that the model picker cannot currently display from `models.list`.

## Solution Assessment

Yes, subject to maintainer protocol acceptance. Reusing `resolveModelAgentRuntimeMetadata` at the Gateway model-list boundary is the narrowest maintainable way to expose the same runtime fact that session and agent surfaces already compute.
