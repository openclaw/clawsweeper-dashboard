## What This Changes

The PR adds optional `agentRuntime` metadata to `models.list` model choices, derives it from configured or implicit runtime policy, renders runtime suffixes in WebUI model-picker labels, and updates focused Gateway/UI/Swift protocol coverage.

## Best Possible Solution

Land after maintainers explicitly accept the additive `ModelChoice.agentRuntime` contract and the WebUI-first rendering scope; add native picker parity in this PR only if that scope is required before merge.

## Reproduction Assessment

## Solution Assessment

Yes, with a scope decision. Adding derived runtime metadata at the Gateway `models.list` boundary and using it in WebUI labels is a maintainable owner-boundary fix; the remaining question is whether native picker parity must be included before merge.
