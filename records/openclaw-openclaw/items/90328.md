## What This Changes

Adds optional `agentRuntime` metadata to `models.list` model choices and uses it to append runtime labels in the WebUI model picker while preserving canonical model values.

## Best Possible Solution

Adopt `models.list.agentRuntime` as the canonical Gateway-to-client runtime-label metadata only after maintainer acceptance of the public contract, while leaving deeper session migration and route-drift fixes to their linked issues.

## Reproduction Assessment

Yes from source and supplied proof: current main has no `agentRuntime` on model choices, and the picker label path only has catalog/provider/alias data. I did not run the app or tests in this read-only review.

## Solution Assessment

Yes, if maintainers accept the API contract: Gateway already owns runtime-policy resolution, and the WebUI picker is catalog-driven, so surfacing metadata through `models.list` is cleaner than duplicating runtime inference in the UI. A UI-only workaround would be more brittle.
