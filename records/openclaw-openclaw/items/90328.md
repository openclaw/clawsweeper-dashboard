## Summary

Keep open. The PR is pointed at the right gateway/UI boundary, but it still drops the common implicit OpenAI Codex runtime case and hides explicit OpenClaw runtime overrides, so it does not yet solve the linked model-picker ambiguity.

## What This Changes

The branch adds optional `agentRuntime` metadata to `models.list` model choices for explicitly configured runtimes and appends non-default runtime labels in the WebUI model picker.

## Best Possible Solution

Expose effective runtime metadata for material runtime differences, including implicit OpenAI Codex and explicit OpenClaw opt-outs, then prove the WebUI picker with a real gateway setup.

## Reproduction Assessment

Yes, by source inspection: official OpenAI implicit runtime resolves to Codex, while the PR helper drops all implicit runtime metadata and the UI suppresses explicit OpenClaw labels. I did not live-run the WebUI picker.

## Solution Assessment

No. Additive `models.list` metadata is the right layer, but the current shape is too narrow because it misses the implicit Codex route and hides explicit OpenClaw opt-outs.
