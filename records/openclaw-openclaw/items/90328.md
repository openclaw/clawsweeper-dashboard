## Summary

Keep open. The PR is a useful additive gateway/UI observability change, but it still has a source-reproducible runtime metadata bug for provider-native nested model IDs, and the contributor proof is still weaker than the changed Gateway/Web UI surface.

## What This Changes

The PR adds optional `agentRuntime` metadata to `models.list` model choices, displays runtime suffixes in Web UI model picker labels, updates protocol/Swift/UI types, and adds focused regressions.

## Best Possible Solution

Keep the additive `agentRuntime` protocol/UI shape, but resolve runtime metadata using the same provider-qualified model identity used by picker values and add a nested-model regression before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a catalog row like `provider: "openrouter", id: "google/gemma-4-26b-a4b-it"` is resolved as `model: entry.id`, while current runtime policy requires the provider-qualified key to match `openrouter/google/gemma-4-26b-a4b-it`.

## Solution Assessment

No. The PR is the right layer for exposing picker metadata, but the helper should use the provider-qualified model identity for policy lookup and prove that slash-containing catalog IDs keep explicit runtime labels.
