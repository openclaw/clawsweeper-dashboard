## What This Changes

The issue asks for forced macOS gateway installs to remove embedded LaunchAgent gateway tokens and for `openclaw channels list` to succeed when auth providers exist but no chat channels are configured.

## Best Possible Solution

Keep the shipped service-metadata repair and land a focused CLI/provider-usage change that treats usage enrichment as optional for `channels list` in both human and JSON output.

## Reproduction Assessment

Yes. The report gives concrete CLI output, and current source gives a deterministic unit path: configure auth profiles with no chat channel accounts, have provider usage fetching reject with `TypeError: fetch failed`, then run `channelsListCommand` with default usage enabled.

## Solution Assessment

Yes for the remaining work. The narrowest maintainable fix is to make per-provider usage loading non-fatal for this listing command or at the provider-usage aggregation boundary, without reopening the already-fixed service metadata behavior.
