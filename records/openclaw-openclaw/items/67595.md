The issue asks for gateway service installation to stop embedding `OPENCLAW_GATEWAY_TOKEN` in LaunchAgent metadata and for `openclaw channels list` to succeed when no chat channels are configured.

## Best Possible Solution

Keep the shipped env-file LaunchAgent handling and the simplified channel-only listing; provider usage should stay on `openclaw status` and `openclaw models list`.

No. Current source and regression tests show the original failing paths are gone: LaunchAgent env values are file-backed, and `channels list` no longer loads provider usage or renders auth providers.

## Solution Assessment

Yes. The current implementation fixes the two reported failures at the right ownership boundary without adding a new option: service metadata avoids plist token embedding, and provider usage is no longer part of channel listing.
