## What This Changes

## Best Possible Solution

Keep the shipped service-metadata repair and land a focused CLI/provider-usage change that makes usage enrichment optional and non-fatal for `channels list` in both human and JSON output.

## Solution Assessment

Yes. The narrow maintainable fix is to convert per-provider usage fetch failures into optional usage error snapshots or otherwise suppress them for this listing command, while leaving the LaunchAgent service-token repair path unchanged.
