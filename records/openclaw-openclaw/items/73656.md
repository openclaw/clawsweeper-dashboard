## Summary

Keep open. Current main still has the reported visible-transcript behavior, but that behavior is now an explicit single-source prompt contract from the #72019/#72053 work, so this should not be closed or mechanically reversed without maintainer prompt-contract direction.

## What This Changes

The issue requests making the first visible subagent user message include the delegated task text as a transcript-visible fallback while keeping the task in the subagent system prompt.

## Best Possible Solution

Define the subagent task visibility contract explicitly: either add an audit/transcript diagnostic that exposes assignment text without duplicating model-visible input, or intentionally move/copy the task into the visible kickoff message and update source, tests, docs, and changelog together.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the symptom: the first visible user message omits the delegated task, while the task is rendered under ## Your Role in the subagent system prompt; I did not run tests because this review is read-only.

## Solution Assessment

Unclear. The suggested local patch is narrow, but it conflicts with the shipped single-source task contract from #72019/#72053, so a maintainer should choose between visible duplication, moving the task, or a transcript-only diagnostic.
