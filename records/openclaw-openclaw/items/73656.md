## Summary

Keep open. Current main still has the reported visible-transcript behavior, and source plus tests show it is the current intentional #72019 token-dedupe contract, so this needs maintainer judgment rather than automated cleanup.

## What This Changes

The issue requests that native subagent spawns include the delegated task text in the first visible user message as an auditable fallback while still carrying the task in the system prompt.

## Best Possible Solution

Define an explicit subagent task visibility contract: either add a transcript/audit diagnostic that exposes enough assignment context without duplicating model input, or deliberately move/copy the task into the visible kickoff message with updated tests, docs, and changelog.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the symptom: the first visible user message omits the delegated task, while buildSubagentSystemPrompt renders the task under ## Your Role and tests assert that split.

## Solution Assessment

Unclear. The suggested patch is narrow, but it changes an intentional token-dedupe contract and a transcript visibility boundary, so maintainers should choose the product/security direction before implementation.
