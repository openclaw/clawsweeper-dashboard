## Summary

Keep open. Current main still intentionally omits the delegated task from the first visible subagent user message, and source, tests, changelog, and related discussion show that this is the #72019/#72053 task-dedupe contract rather than an already-fixed omission. The report is valid, but changing it needs maintainer product/security judgment because the requested fallback changes visible transcript contents and first-request token cost.

## What This Changes

Request to include the delegated task text in the first visible native subagent user message while retaining the task-bearing system prompt.

## Best Possible Solution

Define an explicit subagent task-visibility contract: either add a visible, bounded/redacted kickoff diagnostic, add safer transcript metadata that confirms the assignment without duplicating full content, or keep the current single-source system-prompt model and improve diagnostics elsewhere. The chosen direction should update the prompt builder, spawn tests, docs, and changelog together.

## Reproduction Assessment

Yes. Source inspection against current main reproduces the symptom: the first visible user message omits the delegated task, while `buildSubagentSystemPrompt` carries the task under `## Your Role` and tests assert that single-sourcing behavior.

## Solution Assessment

Unclear. Passing `task` into `buildSubagentInitialUserMessage` is mechanically narrow, but it is not clearly the best product/security solution until maintainers choose between visible duplication, a bounded/redacted diagnostic, or preserving the token-dedupe contract.
