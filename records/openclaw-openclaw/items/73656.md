## Summary

Keep open. Current main still has the reported visible-transcript behavior, and that behavior is now an explicit prompt-placement contract from the #72019/#72053 work, so changing it needs maintainer prompt/transcript judgment rather than cleanup closure.

## What This Changes

The issue requests including the delegated subagent task text in the first visible subagent user message while keeping the system-prompt task block as the canonical instruction source.

## Best Possible Solution

Define the subagent task visibility contract explicitly: either add a transcript/audit diagnostic that exposes assignment text without duplicating model-visible input, or intentionally copy or move the task into the visible kickoff message and update source, tests, docs, and changelog together.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the symptom: the first visible user message omits the delegated task, while the task is rendered under ## Your Role in the subagent system prompt; I did not run tests because this review is read-only.

## Solution Assessment

Unclear. The suggested local patch is narrow, but it conflicts with the shipped #72019/#72053 single-source task placement, so a maintainer should choose between visible duplication, moving the task, or a transcript-only diagnostic.
