## Summary

Keep open. Current main still has the reported visible-transcript behavior, and the source, tests, and changelog show it is an intentional task-dedupe contract from #72019/#72053 rather than an already-fixed omission. The report is valid, but changing it needs maintainer judgment because the requested visible fallback trades off auditability against first-request token cost and exposing delegated task text in channel transcripts.

## What This Changes

The issue requests including the delegated subagent task text in the first visible subagent user message as an audit/debug fallback while keeping the system-prompt `## Your Role` task block.

## Best Possible Solution

Define the subagent task visibility contract explicitly: either add a visible kickoff/audit diagnostic with any needed redaction or length policy, or keep the model input single-sourced in the system prompt and expose enough safe transcript metadata for diagnosis. Whichever direction maintainers choose should update the prompt builder, spawn tests, docs, and changelog together.

## Reproduction Assessment

Yes. Source inspection against current main reproduces the symptom: the first visible user message omits the delegated task, while the task is rendered under `## Your Role` in the subagent system prompt and tests assert that single-sourcing behavior.

## Solution Assessment

Unclear. The suggested code shape is narrow, but it is not clearly the best fix until maintainers choose between visible duplication, moving the task out of the hidden system prompt, or adding a transcript-only diagnostic that preserves the token-dedupe contract.
