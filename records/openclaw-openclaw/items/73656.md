## Summary

Keep open. Current main still has the reported visible-transcript behavior, and the current source, tests, and changelog intentionally enforce single-sourcing the delegated task in the subagent system prompt for the #72019 token-cost fix, so this needs maintainer prompt-contract direction rather than cleanup closure.

## What This Changes

Request that native subagent spawns include the delegated task text in the first visible user message while keeping the system prompt task block.

## Best Possible Solution

Define the subagent task visibility contract explicitly: either add a transcript/audit diagnostic that exposes assignment text without duplicating model-visible input, or intentionally move/copy the task into the kickoff message with updated source, tests, docs, changelog, and privacy/token rationale.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the symptom: the first visible user message omits the delegated task, while the task is rendered under ## Your Role in the subagent system prompt; existing tests assert that behavior.

## Solution Assessment

Unclear. The suggested local patch is narrow, but it reverses the shipped #72019 single-source prompt contract, so a maintainer should choose between visible duplication, moving the task, or a transcript-only diagnostic.
