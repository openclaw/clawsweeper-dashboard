## Summary

Keep open. Current main still has the reported visible kickoff shape, but that behavior is an intentional token-dedup transcript contract from #72019/#72053 and is enforced by tests/changelog, so changing it needs maintainer product judgment rather than conservative cleanup.

## What This Changes

The issue requests adding the delegated task text to the first visible subagent user message as a transcript-visible fail-safe.

## Best Possible Solution

Define an explicit subagent task-visibility contract, then implement the chosen visible diagnostic, sanitized fallback, or metadata path while preserving the intended token behavior and fixing #73624 separately.

## Reproduction Assessment

Yes for the visible-message symptom: source inspection shows the first visible subagent message has no task input while the task is carried in `extraSystemPrompt`, and current tests assert that separation. I did not run tests because this review was read-only and the source path is sufficient.

## Solution Assessment

Unclear. Passing `task` into `buildSubagentInitialUserMessage` is mechanically small, but it reverses a deliberate current contract; a safer direction is a maintainer-approved visibility design, possibly sanitized or metadata-based, alongside the separate #73624 fix.
