## Summary

Keep open. Current main still reproduces the visible-transcript symptom, and the opposite behavior is deliberately enforced for the #72019 token-deduplication contract, so this needs maintainer product review rather than conservative cleanup.

## What This Changes

The issue requests that native subagent spawns include the delegated task in the first visible kickoff message as an auditable fallback.

## Best Possible Solution

Define the subagent task visibility contract explicitly, then implement a maintainer-approved visible diagnostic, sanitized fallback, metadata path, or full task duplication with aligned prompt-builder tests, spawn tests, docs if needed, and changelog.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the symptom: the first visible subagent message omits the delegated task while the system prompt carries it under `## Your Role`, and tests enforce that separation.

## Solution Assessment

Unclear. Passing `task` into `buildSubagentInitialUserMessage` is narrow, but the best solution depends on maintainer choice between visible duplication, sanitized diagnostics, metadata, and the separate #73624 task-loss fix.
