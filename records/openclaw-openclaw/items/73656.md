## Summary

Keep open. Current main still has the reported visible-transcript behavior, and source, tests, and changelog show that the omission is now an intentional token-dedupe prompt contract rather than an accidental missing parameter.

## What This Changes

The issue requests that native subagent spawns include the delegated task in the first visible user message as an auditable fallback while keeping the canonical task in the system prompt.

## Best Possible Solution

Define the subagent task visibility contract explicitly: either add a transcript/audit diagnostic that exposes assignment text without duplicating model input, or intentionally copy or move the task into the visible kickoff message and update source, tests, docs, and changelog together.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the symptom: the first visible user message omits the delegated task, while the task is rendered under `## Your Role` in the subagent system prompt; tests currently assert that behavior.

## Solution Assessment

Unclear. The suggested patch is narrow, but it reverses the shipped single-source task placement from #72019/#72053, so a maintainer should choose between visible duplication, moving the task, or a transcript-only diagnostic.
