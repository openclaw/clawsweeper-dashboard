## Summary

Keep open. Current main still intentionally omits the delegated task from the visible subagent kickoff message while carrying it in the subagent system prompt; the issue is valid, but changing that contract needs maintainer product/security judgment rather than conservative cleanup.

## What This Changes

Request to include the delegated subagent task text in the first visible subagent user message as an auditable fallback while keeping the system-prompt task block.

## Best Possible Solution

Define the subagent task visibility contract explicitly, then implement a maintainer-approved visible diagnostic, sanitized fallback, or metadata path with aligned prompt-builder tests, spawn tests, docs if needed, and changelog.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the reported symptom: the visible initial subagent message omits the delegated task while the system prompt carries it under `## Your Role`, and tests enforce that separation.

## Solution Assessment

Unclear. Passing `task` into `buildSubagentInitialUserMessage` is narrow, but the best solution depends on maintainer choice between visible duplication, sanitized transcript diagnostics, metadata, and the separate #73624 task-loss fix.
