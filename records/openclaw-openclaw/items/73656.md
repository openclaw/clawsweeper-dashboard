## Summary

Keep open. Current main still reproduces the visible bootstrap-only subagent start, but that behavior is an intentional task single-sourcing contract from the #72019/#72053 work, so changing it needs maintainer product and privacy judgment rather than conservative cleanup.

## What This Changes

The issue requests making the delegated task visible in the first subagent user message as an auditable fallback while keeping the system-prompt task block.

## Best Possible Solution

Define the subagent task visibility contract explicitly, then implement either a sanitized visible diagnostic/fallback or transcript metadata path while preserving the #72019 token behavior and handling #73624 separately.

## Reproduction Assessment

Yes. Source inspection on current main is enough: the initial user-message builder does not accept `task`, the spawn path sends the task in `extraSystemPrompt`, and tests assert the task token is absent from the visible first message.

## Solution Assessment

Unclear. Passing `task` into the visible kickoff is a narrow patch, but it is not clearly the best solution until maintainers choose between full visible duplication, sanitized diagnostics, transcript metadata, and the separate hidden-task-loss repair.
