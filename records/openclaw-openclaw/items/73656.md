## Summary

Keep open. Current main still has the reported visible-transcript behavior, and the source, tests, and changelog show that omission is an intentional prompt token-dedupe contract rather than an accidental missing parameter.

## What This Changes

Request that native subagent initial visible user messages include the delegated task text as an auditable fallback while retaining the task in the subagent system prompt.

## Best Possible Solution

Define the subagent task visibility contract explicitly: either add a transcript/audit diagnostic that exposes enough assignment text without duplicating model input, or intentionally copy or move the task into the visible kickoff message and update source, tests, docs, and changelog together.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the symptom: the first visible user message omits the delegated task, while the task is rendered under ## Your Role in the subagent system prompt and tests assert that placement.

## Solution Assessment

Unclear. The suggested patch is narrow, but it changes the current token-dedupe contract, so maintainers should choose between visible duplication, moving the task, or a transcript-only diagnostic before implementation.
