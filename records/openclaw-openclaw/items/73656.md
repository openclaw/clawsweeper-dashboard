## Summary

Keep open. Current main still reproduces the visible bootstrap-only subagent start, but source comments, tests, and changelog show this is an intentional #72019/#72053 task single-sourcing contract, so changing it needs maintainer product/privacy judgment rather than conservative cleanup.

## What This Changes

Request to include delegated subagent task text in the first visible subagent user message as a diagnostic fallback.

## Best Possible Solution

Define the subagent task visibility contract explicitly, then implement either a safe visible diagnostic/fallback or transcript metadata path while updating prompt builders, tests, docs if needed, and the changelog together.

## Reproduction Assessment

Yes. Source inspection on current main is enough to reproduce the symptom: the first visible user message omits the delegated task while `## Your Role` in `extraSystemPrompt` carries it, and tests assert that separation.

## Solution Assessment

Unclear. Passing `task` into `buildSubagentInitialUserMessage` is narrow, but it is not clearly the best fix until maintainers choose between visible duplication, sanitized diagnostics, metadata, and the separate #73624 repair.
