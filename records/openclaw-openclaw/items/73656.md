## Summary

Keep open. Current main still has the reported visible-transcript behavior: the first visible subagent user message omits the delegated task and points to `## Your Role` instead. The issue is valid, but the requested change would reverse or qualify the intentional #72019/#72053 task-deduping contract, so it needs maintainer product/security judgment rather than conservative auto-close or an unreviewed repair lane.

## What This Changes

The issue asks for the first visible subagent user message to include the delegated task text as an auditable fallback while retaining the task in the subagent system prompt.

## Best Possible Solution

Define the subagent task visibility contract explicitly: either add a visible kickoff/audit diagnostic with a clear redaction or length policy, or keep task text single-sourced in the system prompt and expose safe transcript metadata for diagnosis. The chosen direction should update the prompt builder, spawn tests, docs, and changelog together.

## Reproduction Assessment

Yes. Source inspection against current main is a high-confidence reproduction path: the visible initial user message omits the delegated task, while `buildSubagentSystemPrompt` carries it under `## Your Role` and tests assert that single-sourcing behavior.

## Solution Assessment

Unclear. Including the full task in the first visible message is a narrow code change, but it is not clearly the best fix until maintainers choose between visible duplication, redacted transcript diagnostics, or another audit surface that preserves the token-dedupe contract.
