## Summary

Keep open. Current main still has the reported visible-transcript behavior, and the omission is now an explicit shipped prompt/token contract rather than an accidental missing parameter; changing it needs maintainer judgment about transcript visibility, fallback behavior, and token cost.

## What This Changes

Request that native subagent spawns include the delegated task text in the first visible user message as an audit/debug fallback while keeping the system-prompt task block.

## Best Possible Solution

Define the subagent task visibility contract explicitly. If maintainers want visible auditability, add a designed visible assignment surface, possibly redacted or transcript-only, or intentionally copy/move the task into the kickoff message, then update the prompt builders, spawn tests, docs, and changelog together.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the issue: the first visible user message has no task parameter or task text, while the task is rendered under `## Your Role` in `extraSystemPrompt`, and tests assert that split.

## Solution Assessment

Unclear. The suggested patch is narrow, but it reverses the shipped #72019/#72053 single-source task placement; a transcript-only diagnostic or explicit maintainer-approved duplication may be safer depending on the desired prompt contract.
