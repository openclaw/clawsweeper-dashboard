## Summary

Keep open. Current main still reproduces the visible bootstrap-only subagent start, but the source, tests, and changelog show that behavior is an intentional #72019/#72053 task single-sourcing contract, so changing it needs maintainer product/security judgment rather than conservative cleanup.

## What This Changes

The issue requests adding the delegated task text to the first visible subagent user message as an auditable fallback while keeping the canonical task in the system prompt.

## Best Possible Solution

Define the subagent task visibility contract explicitly, then implement either a safe visible diagnostic/fallback or transcript metadata path while updating prompt builders, tests, docs, and changelog together.

## Reproduction Assessment

Yes. Source inspection on current main is enough to reproduce the symptom: the first visible user message omits the delegated task, while `## Your Role` in `extraSystemPrompt` carries it and tests assert that separation.

## Solution Assessment

Unclear. Passing `task` into `buildSubagentInitialUserMessage` is narrow, but it is not clearly the best fix until maintainers choose between visible duplication, sanitized diagnostics, metadata, and the separate #73624 hidden-task-loss repair.
