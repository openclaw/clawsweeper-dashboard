## Summary

Keep open. Current main still has the reported visible-transcript behavior, but it is tied to the intentional #72019/#72053 task-dedupe contract, so changing it needs maintainer product/security judgment rather than conservative cleanup.

## What This Changes

The issue asks for delegated subagent task text to appear in the first visible subagent user message as an auditable fallback instead of only in the hidden system prompt.

## Best Possible Solution

Define the subagent task visibility contract explicitly, then implement either a safe visible diagnostic/fallback or a transcript metadata path while updating the prompt builders, tests, docs, and changelog together.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the symptom: the first visible user message omits the delegated task, while the task is rendered under ## Your Role and tests assert that single-sourcing behavior.

## Solution Assessment

Unclear. Passing task into buildSubagentInitialUserMessage is narrow, but it is not clearly the best fix until maintainers choose between visible duplication, sanitized diagnostics, or preserving system-prompt-only task delivery.
