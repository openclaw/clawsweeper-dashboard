## Summary

Keep open. Current main still has the reported visible kickoff shape, but that shape is also a deliberate shipped #72019 token-dedup contract enforced by tests, so changing it is a prompt/transcript product decision rather than conservative cleanup.

## What This Changes

The issue requests that native subagent startup include the delegated task text in the first visible user message as an auditable fallback while retaining the system-prompt task.

## Best Possible Solution

Define an explicit subagent task visibility contract, then implement the chosen visible diagnostic, sanitized fallback, or metadata path while preserving the token-dedup behavior and fixing #73624 separately.

## Reproduction Assessment

Yes for the reported visible-message symptom: source inspection shows the first visible subagent message has no task input while the system prompt carries the task, and current tests assert that separation. I did not run tests because this was a read-only review and the source path is sufficient.

## Solution Assessment

Unclear. Passing `task` into `buildSubagentInitialUserMessage` is mechanically small, but the best fix depends on maintainer choice between verbatim visible duplication, sanitized transcript diagnostics, metadata, and the separate #73624 repair.
