## Summary

Keep open. Current main still has the reported visible kickoff shape, but that shape is a deliberate shipped #72019 token-deduplication contract enforced by tests and changelog, so changing it needs maintainer product/privacy judgment rather than conservative cleanup.

## What This Changes

Request that the first visible subagent kickoff message include the delegated task as a diagnostic fallback while keeping the task in the system prompt.

## Best Possible Solution

Define an explicit subagent task visibility contract, then implement the chosen verbatim, sanitized, or metadata-based diagnostic path without regressing #72019 and while fixing #73624 directly.

## Reproduction Assessment

Yes for the visible-message symptom: source inspection shows the first visible user message does not take the delegated task, while the spawn path sends the task-bearing prompt as extraSystemPrompt. I did not run tests because this review is read-only, but current tests assert the same separation.

## Solution Assessment

Unclear. Passing task into buildSubagentInitialUserMessage is mechanically small, but the best maintainable fix depends on maintainer choice between verbatim visible duplication, sanitized diagnostics, transcript metadata, and the separate #73624 repair.
