## Summary

Keep open. Current main still resolves Telegram account tokens before disabled filtering in both action discovery and gateway runtime snapshots, so a disabled account with a non-env SecretRef can still throw during the five-minute health-monitor snapshot path.

## What This Changes

The issue requests that Telegram exec-SecretRef bot tokens on disabled accounts not be resolved during gateway snapshot or discovery paths.

## Best Possible Solution

Keep the fix Telegram-owned: return a non-resolving disabled-account snapshot or filter disabled raw account config before strict token reads, then add focused regression coverage for disabled non-env SecretRefs in runtime snapshot and action discovery paths.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a disabled Telegram account with an exec/file botToken reaches resolveTelegramToken through getRuntimeSnapshot or listEnabledTelegramAccounts before disabled filtering; no live gateway run was performed in this read-only review.

## Solution Assessment

Yes. The narrowest maintainable solution is to avoid strict Telegram token resolution for disabled accounts in discovery/snapshot code while preserving strict resolution for enabled active runtime paths.
