## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Move the pre-send hook after the final send decision:** `src/infra/outbound/deliver.ts:1639-1644`
  - body: The new `message:sending` event is scheduled with `fireAndForgetHook` before `applyMessageSendingHook` runs. If a plugin later cancels or rewrites the payload, user hooks will have already observed a message that is not the final delivered content, and async handlers are not guaranteed to finish before delivery. Move this notification after the final plugin decision or make the event explicitly observation-only rather than a pre-delivery guardrail.
  - confidence: 0.9
- **[P2] Guard the reply payload before reading metadata:** `src/infra/heartbeat-runner.ts:1889`
  - body: `resolveHeartbeatReplyPayload` can return `undefined`, but `getReplyPayloadMetadata` accepts an `object`. The new call passes `replyPayload` before narrowing, so the strict TypeScript lanes should reject this heartbeat privacy change. Check `replyPayload` before calling the helper.
  - confidence: 0.84
