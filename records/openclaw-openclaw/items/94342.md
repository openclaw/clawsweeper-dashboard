Full review comments:

- **[P1] Do not fire pre-send hooks before final delivery decisions:** `src/infra/outbound/deliver.ts:1636-1644`
  - body: The new internal `message:sending` notification is scheduled with `fireAndForgetHook` before `applyMessageSendingHook`, so async user hooks can complete after delivery and can observe content that plugin hooks later rewrite or cancel. Move it to the final deliverable payload decision point and await it if this is meant to be a pre-send event, or document it as observation-only.
  - confidence: 0.9
- **[P1] Narrow metadata lookup to defined reply payloads:** `src/infra/heartbeat-runner.ts:1888`
  - body: `replyPayload` is `ReplyPayload | undefined` here, but `getReplyPayloadMetadata` accepts an `object`. This makes the heartbeat change fail strict TypeScript checking on the empty-payload path; only call the helper after `replyPayload` is truthy, or add a narrow helper that accepts `undefined`.
  - confidence: 0.9
