## Summary

The PR is useful and fixes the update-path completion regression, but it still changes the shared completion installer from best-effort to throwing while wizard and doctor callers remain uncaught; that can make optional shell-completion repair abort onboarding or doctor before merge.

## What This Changes

The branch makes JSON nodes-list output JSON-only, turns completion install failures into thrown errors, catches those failures during update completion refresh, and makes the CLI startup build helper throw on spawn errors.

## Best Possible Solution

Keep the stricter direct completion-install failure behavior, but preserve best-effort semantics for optional update, wizard, and doctor repair flows with focused regression tests before merge.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path for the PR regression: make installCompletion fail during wizard or doctor completion setup and the uncaught await propagates out of the flow.

## Solution Assessment

No; the direct CLI failure handling is a reasonable fix, but the PR needs the same best-effort boundary preserved for wizard and doctor that it now preserves for update.
