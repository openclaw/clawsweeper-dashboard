## Summary

Keep this draft PR open. The direction matches the reported iMessage tapback bug and the PR includes useful live-smoke proof, but the current patch appears to break default `own` notification matching for bot-authored messages whose cached iMessage ids include the existing `p:0/` prefix.

## What This Changes

The PR routes iMessage inbound tapbacks into untrusted reaction system events, adds `reactionNotifications` config/docs/schema coverage, and records tool-sent iMessage outputs as bot-authored targets.

## Best Possible Solution

Keep the reaction-system-event approach, but normalize iMessage ids consistently across tapback targets, echo cache, and reply cache so both `p:0/<guid>` and bare GUID bridge shapes identify bot-authored messages.

## Reproduction Assessment

Yes, from source inspection rather than a live run. Current main formats `is_reaction` payloads as normal inbound text and has no reaction decision branch, matching the reported reply-loop path.

## Solution Assessment

No, not as currently written. The proposed direction is maintainable, but stripping `p:0/` before own-mode cache lookup is narrower than the existing id contract and should be replaced with consistent id matching or dual-key lookup.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Check both prefixed and bare tapback target ids:** `extensions/imessage/src/monitor/inbound-processing.ts:83`
  - body: The new target normalization strips `p:0/` before own-mode lookup, but current iMessage send and echo-cache code preserve prefixed ids such as `p:0/imsg-1` as the platform message id. When a bot-authored message was cached with that prefixed id, a tapback target normalized to the bare GUID will miss both `echoCache.has()` and `isKnownFromMeIMessageMessageId()`, so the default `reactionNotifications: "own"` path drops the reaction instead of enqueueing it. Please normalize the cached ids the same way or try both prefixed and bare target ids.
  - confidence: 0.84
