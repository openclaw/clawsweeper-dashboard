## What This Changes

The PR routes iMessage tapback payloads through authorized untrusted reaction system events, records tool-sent bridge messages as bot-authored reply-cache entries, and adds reaction notification config, docs, changelog, and tests.

## Best Possible Solution

Keep this event-routing approach and land it after normal maintainer handling if CI and branch hygiene are acceptable; it should become the canonical fix for the linked iMessage tapback bug.

## Reproduction Assessment

## Solution Assessment

Yes. Classifying tapbacks before dispatch, preserving sender authorization, and enqueueing untrusted reaction system events is the maintainable fix, and the latest head now wires the reply-cache ownership lookup correctly.
