## What This Changes

The issue requests that model-generated message body aliases such as `SendMessage`, `content`, or `text` be normalized to canonical `message` before outbound send validation.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84102 or an equivalent central send-path fix that normalizes body aliases only as fallback text and preserves canonical `message` precedence.

## Solution Assessment

Yes: central pre-validation normalization in the shared send path is the narrow maintainable fix, as long as explicit `message` remains authoritative and aliases are fallback-only.
