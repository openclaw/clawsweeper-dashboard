- `impact:message-loss`: The failure occurs before outbound delivery, causing status or completion messages to be absent from the Discord channel.

## Summary

Keep open: current main and the latest release still validate outbound send bodies only from canonical `message`, while the linked fixing PR remains open and explicitly closes this issue when merged.

## What This Changes

The issue asks OpenClaw to normalize model-generated send-body aliases such as `SendMessage`, `content`, or `text` into canonical `message` before outbound message validation.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84102, or replace it with an equivalent central send-body alias normalization fix that keeps canonical `message` authoritative and aliases fallback-only.

## Reproduction Assessment

Yes. By source inspection, a send payload with only `SendMessage` bypasses current normalization and reaches `readStringParam(actionParams, "message", { required: true })` as missing; I did not run a live Discord replay.

## Solution Assessment

Yes. Central pre-validation alias normalization in the outbound send path is the narrow maintainable fix, and this issue should stay open while the linked closing PR is reviewed.
