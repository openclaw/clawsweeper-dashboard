## What This Changes

The issue requests immediate delivery of intermediate tool-kind text in the QQBot/Telegram gateway when partial streaming is configured.

## Best Possible Solution

Add a QQBot-local progress delivery path for text-only tool progress when streaming is active, with regression coverage, while preserving media fallback and official C2C streaming behavior.

## Reproduction Assessment

Yes, source-level. A text-only `{ kind: "tool" }` delivery followed by a `{ kind: "block" }` delivery through QQBot outbound dispatch buffers the text, then the block reply clears the fallback timer before it can send.

## Solution Assessment

Yes. The narrow maintainable fix is in the QQBot outbound/progress boundary; adding a new config option or globally reclassifying all tool results would be broader and riskier.
