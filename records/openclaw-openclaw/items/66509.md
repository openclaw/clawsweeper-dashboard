## Summary

Keep open: current main and the latest release still source-reproduce the QQBot tool-progress drop, and the linked open fix PR should own resolution before this issue closes.

## What This Changes

The issue asks for QQBot/Telegram gateway text-only tool progress to deliver under partial streaming instead of being buffered and discarded before the final block response.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/85258; after it merges, close this issue as implemented by that commit or release.

## Reproduction Assessment

Yes, source-level. A text-only tool delivery followed by a block delivery buffers the progress text and then clears the fallback timer before it can be sent; I did not run live QQBot network proof in this read-only review.

## Solution Assessment

Yes. The narrow maintainable path is a QQBot-local progress delivery fix with regression coverage, while adding a new global config option or reclassifying all tool deliveries would be broader than needed.
