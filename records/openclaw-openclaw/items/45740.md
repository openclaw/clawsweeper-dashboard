## Summary

Keep open. Current main and the latest release still route GitHub issue body text into gh-issues child work without a first-class untrusted-data handoff, and the related tool-scoping work only covers one layer of the hardening story.

## What This Changes

The issue requests runtime-backed hardening so the gh-issues skill does not pass untrusted GitHub issue and review text into tool-enabled sub-agent prompts as ordinary instructions.

## Best Possible Solution

Keep this issue open until gh-issues uses an approved runtime-backed untrusted-content handoff plus an explicit child capability policy for unattended workers without breaking completion notifications.

## Reproduction Assessment

Yes for source-level reproduction: current main still requires gh-issues worker prompts to include issue body text, and sessions_spawn lacks a workflow-supplied untrusted-content or per-spawn tool-scope handoff. I did not execute a runtime exploit after the model receives that content.

## Solution Assessment

No, a SKILL-only fence is not the best fix because the prior template-only PR was closed for delimiter-bypass and notification/tool-policy regressions. The maintainable direction is runtime/API support for untrusted content plus explicit child capability policy.
