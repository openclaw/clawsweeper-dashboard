## Summary

Keep open: source and discussion evidence show the hardening request remains unresolved on current main; the gh-issues workflow still requires issue body content in worker prompts and prior SKILL-only attempts were rejected as incomplete.

## What This Changes

The issue asks OpenClaw to prevent gh-issues from passing raw GitHub issue and review text directly into tool-enabled child-agent prompts.

## Best Possible Solution

Keep this open until maintainers choose and implement a runtime-backed untrusted-content handoff for gh-issues, with explicit child capability policy for unattended workers and regression tests for hostile issue/comment content.

## Reproduction Assessment

Yes for source-level reproduction: current main and v2026.5.20 still require gh-issues worker prompts to include issue body text, and sessions_spawn has no workflow-supplied per-spawn tool-scope parameter. I did not run a live exploit after model ingestion.

## Solution Assessment

No, a SKILL-only fence is not the best complete solution because prior review found delimiter-bypass and notification/tool-policy tradeoffs. The maintainable fix is runtime/API support for untrusted content plus an explicit child capability policy.
