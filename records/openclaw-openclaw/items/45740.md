## Summary

Keep open: current main and v2026.5.22 still require gh-issues worker prompts to carry issue body text into spawned subagent task text, and the existing untrusted-content helpers are not wired into this workflow. The open toolsAllow work covers only the child tool-policy layer, not the raw external-content handoff that this issue reports.

## What This Changes

Harden the gh-issues automation so untrusted GitHub issue and review text is not passed raw into sub-agent prompts, especially for unattended cron workers.

## Best Possible Solution

Keep this issue open until maintainers land a runtime-backed untrusted-content handoff or structured summary path for gh-issues, composed with an explicit child capability policy for unattended workers.

## Reproduction Assessment

Yes for source-level reproduction: current main fetches GitHub issue bodies, requires gh-issues worker prompts to include body text, and passes sessions_spawn task text into the child [Subagent Task] message. I did not run a live model exploit.

## Solution Assessment

No for a SKILL-only warning: the prior attempt was bypassable and could break notifications. The maintainable path is runtime/API support that treats external issue/comment text as data plus an explicit child capability policy.
