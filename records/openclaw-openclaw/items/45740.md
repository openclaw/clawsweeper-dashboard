## Summary

Keep open. Current main still interpolates raw GitHub issue bodies and review-comment JSON into `gh-issues` child-agent prompts, and the remaining fix needs maintainer direction because it spans untrusted-content handling, sandboxing, and child tool-scope policy.

## What This Changes

Harden `gh-issues` so untrusted GitHub issue bodies and review comments are treated as data before spawning tool-enabled child agents.

## Best Possible Solution

Define a maintainer-approved untrusted external-content boundary for `gh-issues` child task construction, then pair it with explicit unattended sandbox/tool-scope policy and regression coverage for hostile issue/comment content.

## Reproduction Assessment

Yes for the source-level hardening gap: current main still substitutes issue bodies and review-comment content directly into child task prompt strings. No runtime exploit reproduction was established for a security-boundary bypass after the model receives that content.

## Solution Assessment

Unclear because no merged fix exists. Prompt warnings would be a partial defensive layer, but the narrower maintainable direction is structured untrusted-data handling plus a maintainer decision on sandbox and tool scope for unattended child agents.
