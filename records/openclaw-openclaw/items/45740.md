## Summary

Keep open: current main and v2026.5.22 still route gh-issues issue body text into worker prompt material, while the remaining fix requires maintainer security/product decisions around runtime untrusted-data framing and child tool-policy API behavior.

## What This Changes

The issue asks OpenClaw to stop passing untrusted GitHub issue and review-comment text directly into gh-issues spawned sub-agent prompts without isolation or sanitization.

## Best Possible Solution

Land a maintainer-approved runtime untrusted-content handoff for gh-issues and compose it with a child capability policy, reusing https://github.com/openclaw/openclaw/pull/78441 if its public contract is approved.

## Reproduction Assessment

Yes for source-level reproduction: current main and v2026.5.22 still require gh-issues worker prompts to include issue body text, and native subagent startup places delegated task text into the child prompt. I did not run a live model exploit.

## Solution Assessment

No for a SKILL-only warning: the prior prompt-only attempt was bypassable and could break notifications. The maintainable direction is runtime/API support for treating external content as data plus an explicit child capability policy.
