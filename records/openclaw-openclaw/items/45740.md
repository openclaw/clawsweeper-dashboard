## Summary

Keep open. Current main and v2026.5.12 still embed raw GitHub issue bodies and review-comment JSON into `gh-issues` child-agent prompts, while the prior SKILL-only fix was closed after review found a fence bypass and a notification regression.

## What This Changes

The issue asks OpenClaw to harden `gh-issues` so untrusted GitHub issue bodies and review comments are not passed raw into tool-enabled sub-agent prompts.

## Best Possible Solution

Define a runtime-backed untrusted-data handoff for `gh-issues`, then pair it with an explicit child sandbox/tool-scope policy and hostile-content regression coverage.

## Reproduction Assessment

Yes for source-level reproduction: current main and v2026.5.12 still substitute `{body}` and `{json_array_of_actionable_comments}` directly into child task prompt strings. I did not establish a runtime exploit showing a boundary bypass after the model receives that content.

## Solution Assessment

Unclear until maintainers choose the runtime/API contract. A SKILL.md-only fence is not the best fix because the prior attempt was bypassable and regressed notification behavior; the maintainable path is runtime untrusted-data handling plus explicit child capability policy.
