## Summary

Keep open. Current main and the latest release still place raw GitHub issue bodies and review-comment JSON into `gh-issues` child-agent prompts, and the earlier prompt-only PR was closed after review found an XML-fence bypass and a `--notify-channel` regression.

## What This Changes

The issue asks OpenClaw to stop feeding raw `gh-issues` GitHub issue and review-comment bodies into tool-enabled child-agent prompts and add a safer untrusted-content or capability boundary.

## Best Possible Solution

Define a maintainer-approved untrusted-data handoff for `gh-issues` child prompts, reuse or extend existing wrapping/sanitization where appropriate, and pair it with explicit unattended child sandbox/tool-scope policy plus hostile-content regression coverage.

## Reproduction Assessment

Yes for source-level reproduction: current main still substitutes `{body}` and `{json_array_of_actionable_comments}` directly into child task prompt strings. I did not establish a runtime PoC showing a security-boundary bypass after the model receives that content.

## Solution Assessment

Unclear because no merged fix exists and the prompt-only patch was reviewed as bypassable and regressive. The safer direction is runtime-backed untrusted-data handling plus a maintainer decision on sandbox and per-spawn tool scope, not another SKILL.md-only fence.
