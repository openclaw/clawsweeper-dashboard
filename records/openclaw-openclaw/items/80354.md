## What This Changes

The branch disables the embedded idle watchdog for non-off thinking runs, adds timeout tests, moves Ollama native `think` into `options.think`, and adds extra fields to `sessions --json` rows.

## Best Possible Solution

Land a narrow embedded idle-timeout bypass with focused tests and live Gemini proof, keep Ollama `think` at the request root, and split the sessions JSON expansion into its own PR if still wanted.

## Reproduction Assessment

Yes from source and linked logs, but not from a live rerun in this review. Current main applies the default idle watchdog to cloud model runs without considering non-off thinking, and the linked bug records repeated roughly 125s Gemini failures with `thinking=medium`.

## Solution Assessment

No as submitted. The embedded timeout bypass is a plausible narrow fix, but moving Ollama `think` into `options.think` conflicts with the documented/current contract and the sessions JSON change should be split from this bug fix.
