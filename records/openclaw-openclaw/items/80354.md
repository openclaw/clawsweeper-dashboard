## Summary

Keep this PR open, but it should not merge as submitted: the embedded idle-timeout bypass targets a real linked timeout path, while the branch also changes Ollama request semantics in a way that conflicts with the current contract and still lacks proof that the live Gemini timeout path improves.

## What This Changes

The branch adds a thinking-enabled idle-watchdog bypass for embedded runs, updates timeout tests, moves native Ollama `think` into `options.think`, and expands `sessions --json` row metadata.

## Best Possible Solution

Land a narrow embedded idle-timeout fix with focused tests and relevant live proof, keep native Ollama `think` at the request root, and split the sessions JSON expansion into separate work if still desired.

## Reproduction Assessment

Yes, from source and linked logs, but not from a live rerun in this review. Current main applies the default cloud idle watchdog without checking non-off thinking, and the linked report records repeated roughly 125s Gemini failures with `thinking=medium`.

## Solution Assessment

No as submitted. The embedded timeout bypass is a plausible narrow fix, but the Ollama `options.think` change conflicts with the current docs/tests and upstream API shape, and the sessions JSON change should be split from this bug fix.
