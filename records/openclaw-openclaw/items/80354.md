## Summary

This PR should stay open: the embedded idle-timeout change is a plausible narrow fix for the linked Gemini thinking timeout, but the branch still mixes in an Ollama wire-shape regression and unrelated sessions JSON work. The supplied terminal proof is useful helper-level evidence, but it does not demonstrate the live embedded/Gemini behavior that failed for users.

## What This Changes

The branch disables the embedded LLM idle watchdog for non-off thinking runs, adds resolver tests, moves native Ollama thinking parameters into `options.think`, and exposes additional `sessions --json` row fields.

## Best Possible Solution

Narrow this PR to the embedded thinking-aware idle-timeout bypass with focused tests and relevant live provider proof, while keeping the Ollama and sessions changes in their separate review threads.

## Reproduction Assessment

Yes, from source and linked logs, but not from a live rerun in this review. Current main applies the default cloud idle watchdog without checking non-off thinking, and the linked bug records repeated roughly 125s Gemini failures with `thinking=medium`.

## Solution Assessment

No as submitted. The embedded timeout bypass is plausible, but the PR also moves Ollama `think` into an unsupported request location and bundles unrelated sessions JSON work; the narrow maintainable solution is to split those out and prove the live embedded path.
