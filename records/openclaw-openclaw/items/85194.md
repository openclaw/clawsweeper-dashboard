## Summary

Keep open. The PR targets the right source-reproducible denial loop, but it changes the documented node `exec.denied` delivery contract without updating the protocol docs, and it appears to leave unused approval runtime handoffs behind for denied elevated followups.

## What This Changes

The PR suppresses denied exec approval follow-up agent turns, drops remote `exec.denied` node events before enqueue/heartbeat, and adds Chinese stop phrases to abort detection tests and primitives.

## Best Possible Solution

Land the narrow denial-loop repair after documenting the new `exec.denied` contract and avoiding unused runtime handoff state for terminal denied results.

## Reproduction Assessment

Yes, at source level. Current main still routes ordinary denied exec approval results into agent follow-up work, wakes sessions on node `exec.denied`, and lacks the reported `停下来` / `暂停` abort triggers.

## Solution Assessment

Mostly yes. The terminal denial direction is the narrow maintainable fix, but the protocol docs and unused runtime-handoff state need follow-through before this is the best merge shape.
