## Summary

Keep open: this is a useful candidate for the QMD lexical-mode embedInterval issue, but it is not merge-ready because it changes a documented config contract, still mishandles the zero-interval edge, has a lint failure, and lacks real QMD runtime proof.

## What This Changes

The PR adds a resolved QMD `embedIntervalConfigured` flag, uses it to let configured `memory.qmd.update.embedInterval` bypass lexical search-mode embed gates, and updates focused QMD config and manager tests.

## Best Possible Solution

If maintainers accept explicit positive `embedInterval` as a lexical-mode override, gate on a positive parsed interval, update docs/help/tests, and require redacted real QMD proof; otherwise keep lexical mode embedding-free and direct automatic embedding users to `query` or `vsearch`.

## Reproduction Assessment

Yes, by source inspection. Current main resolves `embedInterval`, but `shouldRunEmbed()` and `shouldScheduleEmbedTimer()` return in lexical `search` mode before the interval can affect embedding.

## Solution Assessment

No as submitted. Carrying explicit interval intent through resolved config is the right boundary, but the patch still needs a positive-interval gate, docs/help alignment, lint cleanup, maintainer acceptance, and real runtime proof.
