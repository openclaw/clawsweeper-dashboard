The PR adds an internal resolved QMD `embedIntervalConfigured` flag, uses it to bypass lexical-mode embed gates when `memory.qmd.update.embedInterval` is set, and updates focused QMD config and manager tests.

## Best Possible Solution

If maintainers accept the override, require a positive parsed explicit interval, align docs/help/tests with the new lexical contract, and land only with redacted real QMD runtime proof; otherwise keep lexical mode embedding-free and direct automatic embedding users to `query` or `vsearch`.
