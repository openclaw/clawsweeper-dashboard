## Summary

Keep open. The branch is close to the right shape, but it still needs maintainer agreement on the lexical-mode embedding contract and has two merge blockers: blank or whitespace `embedInterval` is treated as configured, and the public docs still promise lexical mode does no embedding maintenance.

## What This Changes

The PR adds an internal `embedIntervalConfigured` flag in resolved QMD config and uses it to let lexical `searchMode: "search"` run and schedule QMD embedding when `memory.qmd.update.embedInterval` is present, with focused tests.

## Best Possible Solution

If maintainers accept explicit opt-in embedding for lexical mode, land a version that tracks a valid configured interval, updates docs/tests for the new contract, and includes real QMD runtime proof; otherwise keep lexical mode embedding-free and direct automatic embedding users to `query` or `vsearch`.

## Reproduction Assessment

Yes, by source inspection. Current main resolves `embedInterval`, but the QMD manager returns false for lexical `searchMode: "search"` before checking the interval; I did not run a live QMD process in this read-only review.

## Solution Assessment

No as submitted. Carrying explicit interval intent through resolved config is the right boundary, but the patch must treat blank values consistently, align docs/tests, and wait for maintainer acceptance of the lexical-mode behavior change.
