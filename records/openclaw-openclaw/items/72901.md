## Summary

Keep this PR open, but do not merge it as-is. The Slack-owned fix targets a real current-main bug, but the PR head still does not match the core session lifecycle contract for legacy session entries, lacks the required changelog entry, and is currently dirty against main.

## What This Changes

This PR changes Slack thread-context preparation to load thread history for stale daily/idle thread sessions, passes config into the resolver, reexports session helpers through the Slack runtime barrel, and adds regression tests.

## Best Possible Solution

Land the Slack fix after the prefetch gate reuses the same lifecycle timestamp resolution as `initSessionState`, adds a legacy `sessionFile` header regression test, adds the changelog bullet, and rebases cleanly onto current main.

## Reproduction Assessment

Yes. The original bug is reproducible by a Slack thread session that crosses a daily or idle reset boundary while retaining a stored timestamp; the remaining PR gap is reproducible with a legacy session row whose `updatedAt` is fresh, `sessionStartedAt` is absent, and the JSONL session header predates the reset boundary.

## Solution Assessment

No, not yet. The PR direction is the right narrow owner-module fix, but the maintainable solution is to share or expose the lifecycle resolver so Slack's prefetch decision and core session initialization make the same freshness decision.
