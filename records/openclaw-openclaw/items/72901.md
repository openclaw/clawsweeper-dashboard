## Summary

Keep this open. The stale-session fix is pointed at the right Slack-owned path, but the branch still diverges from the session lifecycle contract for legacy session entries and lacks the required changelog entry, so it should not merge as-is.

## What This Changes

The PR changes Slack thread-context preparation to load thread history for stale daily/idle thread sessions, passes config into the resolver, reexports session-store freshness helpers, and adds regression tests.

## Best Possible Solution

Land the Slack-owned stale-session fix after the prefetch gate shares `initSessionState`'s lifecycle timestamp resolution, covers the legacy `sessionFile` header case, and adds the Unreleased changelog bullet.

## Reproduction Assessment

Yes. The original bug reproduces when a Slack thread session crosses a daily or idle reset while retaining a stored `updatedAt`; the remaining PR gap reproduces with a legacy entry whose `updatedAt` is fresh, `sessionStartedAt` is absent, and the JSONL session header predates the reset boundary.

## Solution Assessment

No, not yet. The direction is the narrow maintainable fix, but the implementation should route freshness through the same lifecycle resolver as session initialization so current and legacy session rows agree.
