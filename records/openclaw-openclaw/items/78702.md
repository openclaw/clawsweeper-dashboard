## Summary

Keep open. Current main has partial mitigations for externalized channel plugins, but it still does not fully satisfy the reported post-upgrade behavior: Feishu is excluded from the core package, startup/status remains warning/repair-oriented, and the merged related PR only fixes the onboard recovery path.

## What This Changes

The issue asks OpenClaw upgrades from bundled Feishu to external `@openclaw/feishu` to either reinstall the plugin automatically or surface a hard, actionable offline error.

## Best Possible Solution

Add a maintainer-approved upgrade recovery path for configured official external channels, with Feishu showing an unmistakable offline state and exact repair command unless automatic repair is explicitly chosen.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction shape: keep `channels.feishu` or `plugins.allow: ["feishu"]` configured while the external package is absent from the core install. I did not live-run the full old-to-new npm upgrade in this read-only checkout.

## Solution Assessment

Unclear. The current onboard fix is useful but not the best complete solution for this issue because it only helps after selecting the channel in setup; the upgrade/status behavior still needs a maintainer choice between automatic repair and a hard visible failure.
