## Summary

Keep open. Current main exposes hooks and partial mitigations, but it does not have the requested native outbound finalizer, per-channel finalizer config, turn-level coalescing flush semantics, or LLM cleanup pass; the remaining direction overlaps several open delivery/API trackers and needs maintainer product/API judgment.

## What This Changes

The issue requests a native outbound message finalizer that coalesces and strips scratchpad/process narration before replies reach messaging channels.

## Best Possible Solution

Define the product boundary first: either keep this as a ClawHub code plugin supported by stronger hook/coalescing APIs, or approve a narrow core finalizer design that composes with the existing outbound policy, visible-reply, and audience-boundary work.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this is a feature request for new native behavior. Source inspection does give high confidence that current main lacks the requested native finalizer, and the issue/commentary provide production anecdotes rather than a current-main runnable repro.

## Solution Assessment
