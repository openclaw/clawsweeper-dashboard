## Summary

This is a well-scoped operator need, but the requested LLM post-filter/coalescer is optional runtime policy that fits OpenClaw’s plugin and ClawHub boundary rather than core; current main already exposes the hook surfaces needed to build and distribute it outside core.

## What This Changes

Requests a native outbound message finalizer in OpenClaw core to coalesce and clean outbound messaging replies before channel delivery.

## Best Possible Solution

Keep the outbound finalizer as a code plugin published through ClawHub, and open a narrower plugin API issue only if a specific missing hook field or lifecycle guarantee blocks that plugin.

## Reproduction Assessment

Not applicable; this is a feature request for native core behavior, not a report of broken current behavior. Source review confirms the relevant existing plugin hook surfaces rather than a failing reproduction path.

## Solution Assessment

No for native core: the narrower maintainable path is a ClawHub/plugin implementation using message_sending, message_received, agent_end, and before_agent_finalize, with any missing hook contract filed separately as a focused plugin API gap.
