## Best Possible Solution

Fix the shared Copilot plugin auth helper, or a narrow plugin-facing SDK seam it uses, so catalog and embeddings honor ordered eligible profiles while preserving env-token precedence, discovery disablement, and explicit auth-order exclusions.

## Reproduction Assessment

## Solution Assessment

No. The bug is real, but this PR solves it with a core fallback after plugin discovery; the safer implementation is to make the shared Copilot plugin auth helper or SDK seam honor `auth.order` while preserving disablement and explicit exclusions.
