## Summary

Keep open: the linked bug remains valid, but this PR’s core fallback bypasses plugin-owned discovery semantics and still lacks inspectable real-behavior proof.

## What This Changes

The PR adds a core implicit GitHub Copilot fallback plus regression tests so generated models config follows `auth.order` when selecting stored Copilot profiles.

## Best Possible Solution

Move order-aware Copilot token selection into the plugin-owned helper or SDK auth seam while preserving env-token precedence, discovery disablement, explicit `auth.order` exclusions, and coverage for catalog discovery plus embeddings.

## Reproduction Assessment

Yes, by source inspection: configure multiple stored `github-copilot` token profiles, set `auth.order.github-copilot` to prefer a non-first profile, omit env tokens, then run implicit Copilot discovery or Copilot embeddings; current main reaches `profileIds[0]`.

## Solution Assessment

No. The bug is real, but this PR adds a core fallback after plugin discovery; the narrower maintainable fix belongs in the GitHub Copilot plugin helper or an SDK auth helper it uses.
