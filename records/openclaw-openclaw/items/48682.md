## Summary

Keep this PR open, but do not merge it as-is. The underlying Copilot auth.order bug still exists on current main, while this patch is stale against the current provider split and introduces credential-use regressions around plugin discovery disablement and explicit auth.order exclusions.

## What This Changes

The PR adds a core implicit GitHub Copilot provider resolver and expands models-config tests for auth.order-driven Copilot profile selection and fallback behavior.

## Best Possible Solution

Move the fix into the GitHub Copilot plugin auth/catalog path, using the existing order-aware provider-auth SDK seam or a plugin-local helper that delegates to it while preserving env-token precedence, explicit provider config, plugin discovery disablement, and documented auth.order exclusion semantics.

## Reproduction Assessment

Yes. Configure two stored github-copilot token profiles, set auth.order.github-copilot to prefer the non-first profile, and run implicit Copilot discovery or Copilot embeddings with no env token; current main reaches profileIds[0] in resolveFirstGithubToken.

## Solution Assessment

No. The requested behavior is valid, but this PR solves it in stale core provider code and bypasses plugin-owned discovery controls; the narrower maintainable solution is in extensions/github-copilot auth/catalog code using the provider-auth order seam.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Respect disabled Copilot discovery:** `src/agents/models-config.providers.ts:803-811`
  - body: The added core fallback injects github-copilot whenever plugin discovery did not return it, but intentional disablement also produces that absence. With Copilot discovery disabled, this can re-add the provider and trigger token exchange despite the user's opt-out.
  - confidence: 0.86

  - body: The fallback path can try raw stored profiles after configured ordered candidates are exhausted. The auth semantics doc says profiles omitted from explicit auth.order are excluded, so this can select a credential the operator intentionally left out.
  - confidence: 0.78
