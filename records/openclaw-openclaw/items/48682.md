## Summary

Keep open. The underlying bug is still present on current main, but this PR should not merge as-is because it adds a core fallback that can override credential-selection controls, and the external contributor has not posted inspectable after-fix behavior proof.

## What This Changes

The branch adds core-side implicit GitHub Copilot discovery fallback logic and regression tests so stored profile selection can follow `auth.order`.

## Best Possible Solution

Move order-aware token selection into `extensions/github-copilot/auth.ts` using the provider-auth SDK seam, preserve env-token precedence and `auth.order` exclusions, and cover both catalog discovery and embeddings.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure multiple stored `github-copilot` token profiles, set `auth.order.github-copilot` to a non-first profile, omit env tokens, then run implicit discovery or Copilot embeddings; current main reaches `profileIds[0]` in `resolveFirstGithubToken`.

## Solution Assessment

No. The bug is real, but this PR adds a core fallback after plugin discovery; the narrower maintainable fix is in the GitHub Copilot plugin-owned auth helper or SDK auth seam, preserving disablement and order exclusions.
