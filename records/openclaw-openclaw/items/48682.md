## Summary

Keep open. The reported bug is still source-reproducible on current main, but this PR’s current diff is not safe to merge because it adds a core Copilot fallback that can bypass plugin disablement and explicit `auth.order` exclusions.

## What This Changes

The PR adds core-side implicit GitHub Copilot discovery fallback logic and regression tests so stored Copilot profile selection follows `auth.order`.

## Best Possible Solution

Rework the fix inside the GitHub Copilot plugin token helper using the order-aware provider-auth seam, with focused coverage for catalog discovery, embeddings, disabled discovery, and excluded profiles.

## Reproduction Assessment

Yes. Source inspection on current main shows `resolveFirstGithubToken` reads `profileIds[0]`, and the catalog plus embeddings paths call that helper after the documented `auth.order` contract says explicit order controls profile selection.

## Solution Assessment

No. The requested behavior is valid, but this PR solves it with a core fallback after plugin discovery; the narrower maintainable fix belongs in the GitHub Copilot plugin-owned auth helper.
