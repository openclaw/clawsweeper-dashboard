## Summary

Keep open. The underlying auth-order bug is still present on current main, but this PR is not ready because it adds a core fallback that can bypass plugin discovery controls, can fall back outside explicit auth.order, is now conflicting, and still lacks inspectable real behavior proof.

## What This Changes

The PR adds a core implicit GitHub Copilot provider fallback plus regression tests so stored Copilot profile selection follows auth.order during implicit models config generation.

## Best Possible Solution

Move order-aware Copilot token selection into the GitHub Copilot plugin helper or a plugin-facing SDK auth helper, preserving env-token precedence, discovery disablement, explicit auth.order exclusions, and coverage for catalog discovery plus embeddings.

## Reproduction Assessment

Yes. By source inspection, configure multiple stored github-copilot token profiles, set auth.order.github-copilot to prefer a non-first profile, omit env tokens, and current main reaches profileIds[0] in resolveFirstGithubToken for catalog discovery and embeddings.

## Solution Assessment

No. The bug is real, but this PR adds a core fallback after plugin discovery; the narrower maintainable fix belongs in the GitHub Copilot plugin helper or an SDK auth helper it uses.
