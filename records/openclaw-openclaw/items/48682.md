## Summary

Keep open: the linked bug is still valid on current main, but this PR is conflicting and the proposed core fallback introduces credential-selection boundary regressions that should not merge as-is.

## What This Changes

This PR adds a core models-config fallback that selects implicit `github-copilot` profiles using `auth.order` and expands models-config tests for ordered and unresolved fallback cases.

## Best Possible Solution

Move ordered Copilot token selection into the shared plugin auth helper or a narrow SDK auth seam used by catalog and embeddings, while preserving env precedence, discovery disablement, and explicit `auth.order` exclusions.

## Reproduction Assessment

Yes. By source inspection on current main, multiple stored `github-copilot` profiles with `auth.order.github-copilot` preferring a non-first profile still reach `resolveFirstGithubToken`, which selects `profileIds[0]` when env tokens are absent.

## Solution Assessment

No. The bug is real, but this PR solves it with a core post-discovery fallback; the maintainable fix belongs in the shared Copilot auth helper or SDK auth seam and must preserve discovery disablement plus auth-order exclusions.
