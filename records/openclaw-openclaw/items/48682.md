## Summary

The source path confirms the reported bug is real, but this PR should not merge as-is: it adds a core fallback that can bypass Copilot plugin discovery disablement and can retry credentials explicitly excluded by auth.order, while the branch is also conflicting and still lacks inspectable real behavior proof.

## What This Changes

The PR changes implicit GitHub Copilot models discovery to select Copilot auth profiles using auth.order and adds regression tests for ordered and unresolved profile fallback cases.

## Best Possible Solution

Fix the shared Copilot plugin auth helper, or a narrow plugin-facing SDK seam it uses, so catalog and embeddings honor ordered eligible profiles while preserving env-token precedence, discovery disablement, and explicit auth-order exclusions.

## Reproduction Assessment

Yes by source inspection. On current main, configure multiple stored github-copilot token profiles, set auth.order.github-copilot to prefer a non-first profile, omit env tokens, and the catalog path reaches resolveFirstGithubToken, which selects profileIds[0].

## Solution Assessment

No. The bug is real, but the PR solves it with a core fallback after plugin discovery; the maintainable fix belongs in the Copilot plugin helper or SDK auth seam and must preserve disablement plus auth-order exclusions.
