## Summary

Keep open: this PR targets a real source-reproducible Copilot auth-order bug, but the current patch still has blocking credential-selection regressions, lacks inspectable real behavior proof, and is merge-conflicting against current main.

## What This Changes

The branch adds a core fallback that builds an implicit github-copilot provider from ordered auth profiles and expands models-config tests for ordered and unresolved Copilot profile fallback paths.

## Best Possible Solution

Fix the Copilot plugin auth helper or plugin-facing SDK seam so catalog and embeddings both use ordered, eligible profiles while preserving env precedence, discovery disablement, and explicit auth.order exclusions.

## Reproduction Assessment

Yes. By source inspection on current main, configure multiple stored github-copilot token profiles, set auth.order.github-copilot to prefer a non-first profile, omit env tokens, and the catalog path reaches resolveFirstGithubToken, which selects profileIds[0].

## Solution Assessment

No. The bug is real, but this PR solves it with a core fallback after plugin discovery; the narrower maintainable fix belongs in the Copilot plugin helper or SDK auth seam and must preserve disablement plus auth-order exclusions.
