## Summary

Keep open. Current main still uses a native textarea for the Control UI Raw config surface, has textarea-specific tests/styles, and does not include a structured editor dependency or component. The provided related PRs are open implementation candidates, not merged behavior.

## What This Changes

The issue requests replacing the Raw config textarea with a structured JSON editor that supports syntax highlighting, folding, formatting, validation, and unchanged submit/save behavior.

## Best Possible Solution

Keep the issue open and converge the open Raw config editor work into one reviewed Control UI implementation that adds folding/highlighting/validation/formatting while preserving raw availability, sensitive redaction/reveal, reset-to-saved, raw diff, `onRawChange`, `config.set`, `config.apply`, and `baseHash` semantics.
