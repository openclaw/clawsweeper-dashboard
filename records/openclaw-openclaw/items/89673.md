## What This Changes

The branch removes `hold-exceeded` from report-only stale lock reasons, adds a live-owner `maxHoldMs` reclaim regression test, and updates `maxHoldMs` help/docs text.

## Best Possible Solution

Land this only after maintainers explicitly accept `maxHoldMs` as the hard cross-process reclaim deadline for unchanged live-holder session locks.

## Reproduction Assessment

Yes. Current main appends `hold-exceeded` when lock age exceeds the holder-recorded `maxHoldMs`, then refuses removal because `hold-exceeded` is still report-only; I did not reproduce the overnight gateway hang live.

## Solution Assessment

Yes, if maintainers accept the policy: this is the narrow owner-boundary fix using the existing fs-safe unchanged-snapshot removal contract. If `maxHoldMs` must stay watchdog-only for existing users, the safer solution is an explicit opt-in reclaim policy.
