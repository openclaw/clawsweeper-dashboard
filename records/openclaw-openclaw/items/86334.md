## What This Changes

The issue asks WebChat to preserve explicit Markdown links using custom app URL schemes such as obsidian:// and things:// so they render as clickable anchors.

## Best Possible Solution

Review and resolve https://github.com/openclaw/openclaw/pull/86335 with a maintainer-approved custom-scheme policy, then close this issue after the fix merges or after maintainers decide custom app protocols should stay inert.
