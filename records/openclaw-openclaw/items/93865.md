Full review comments:

- **[P1] Gate backfill on real recovery state:** `extensions/mattermost/src/mattermost/monitor.ts:1495`
  - body: The shared turn kernel clears pending history after successful dispatch, so `existing.length === 0` is also true for normal follow-up messages after an agent reply. This can repeatedly fetch and inject the full server thread during active conversations; gate this on a first-sighting or recovery signal instead of any empty local window.
  - confidence: 0.89
- **[P1] Preserve disabled group history:** `extensions/mattermost/src/mattermost/monitor.ts:1512`
  - body: When `historyLimit` is `0`, `entries.slice(-historyLimit)` becomes `entries.slice(0)`, and this direct map write stores the fetched thread even though the existing history helpers treat nonpositive limits as disabled. Guard the backfill on `historyLimit > 0` or route storage through the history window contract.
  - confidence: 0.92
- **[P2] Bound the pre-dispatch thread work:** `extensions/mattermost/src/mattermost/monitor.ts:1497-1502`
  - body: This awaited fetch runs before inbound dispatch, then resolves user info for every returned thread post before trimming to the configured limit. Add a short abortable timeout and slice candidate posts before serial sender lookups so a slow or large Mattermost thread cannot stall the recovery turn.
  - confidence: 0.86
- **[P3] Remove the unrelated CodeGraph ignore file:** `.codegraph/.gitignore:1`
  - body: This branch is a focused Mattermost runtime fix, but it also adds repository metadata for a local CodeGraph directory. Drop this unrelated file so the PR only carries the bug fix and its tests.
  - confidence: 0.8
