## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Gate backfill to real recovery windows:** `extensions/mattermost/src/mattermost/monitor.ts:1496-1498`
  - body: This condition fires whenever the local history window is empty, but the shared turn kernel clears group pending history after successful dispatch. An ordinary follow-up in an active Mattermost thread can therefore await the server thread fetch and inject full server history even though this is not restart/session-clear recovery. Track a recovery/session/first-sighting signal before fetching.
  - confidence: 0.9
- **[P2] Move backfill behind admission gates:** `extensions/mattermost/src/mattermost/monitor.ts:1496-1498`
  - body: The new block runs before `evaluateMattermostMentionGate`, so messages later dropped for onchar or missing mention can still fetch from Mattermost and seed thread state. Move the backfill after those drop checks, or prove ignored thread traffic should do this work.
  - confidence: 0.88
- **[P2] Trim before resolving backfilled senders:** `extensions/mattermost/src/mattermost/monitor.ts:1510-1515`
  - body: The thread endpoint can return the full thread, but this loop awaits `resolveUserInfo` for every returned post and only then stores the last `historyLimit` entries. Select the effective history-window posts before resolving senders so long cold threads do not fan out into unnecessary lookups before dispatch.
  - confidence: 0.86
