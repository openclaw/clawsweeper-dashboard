## Summary

Keep open. Current main and the latest release still keep the two Telegram sticker e2e cases quarantined, so the PR is still useful for the #50185 coverage gap, but the proposed SDK module mock is too broad as written and the two old skip comments remain stale.

## What This Changes

The PR adds a direct `openclaw/plugin-sdk/agent-runtime` mock to the Telegram media e2e harness and changes two Telegram sticker e2e tests from `it.skip` to `it`.

## Best Possible Solution

Keep this PR as the active implementation candidate for #50185, but change the new SDK mock to preserve existing `agent-runtime` exports, remove the stale skip comments, and then validate the Telegram sticker e2e file plus changed Telegram test lanes before merging.

## Reproduction Assessment

Yes. The missing coverage is reproducible by inspection on current main and `v2026.4.29`: both target cases are still `it.skip(...)`, and the PR diff changes those exact calls to `it`.

## Solution Assessment

No, not as written. Re-enabling the tests is the right direction, but the direct SDK mock should spread the real module exports or stay on a narrower wrapper seam so unrelated Telegram imports keep working.
