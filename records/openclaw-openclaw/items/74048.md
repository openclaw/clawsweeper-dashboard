## Summary

Keep open. The PR targets a real Claude ACP failure, but the patch changes the existing ACP capability contract and can silently drop requested runtime controls, including permission policy settings, instead of preserving current main behavior.

## What This Changes

The PR changes `applyManagerRuntimeControls` to forward ACP config options only when `configOptionKeys` is non-empty, filtering to advertised keys and skipping all config options when no keys are advertised.

## Best Possible Solution

Preserve the current permissive meaning of missing `configOptionKeys`, add an explicit strict capability/reporting path for strict adapters such as Claude ACP, and cover the manager plus ACPX behavior with focused tests and a changelog entry.

## Reproduction Assessment

Yes. Applying this patch would make the existing manager fixture that omits `configOptionKeys` skip persisted runtime options, while the linked #73875 report gives concrete live Claude ACP steps for the original strict-adapter failure.

## Solution Assessment

No. The patch solves the strict-backend symptom by changing a shared capability contract and dropping requested controls; the safer path is to distinguish unknown/permissive capability reporting from an explicit strict key list.
