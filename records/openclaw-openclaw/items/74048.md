## Summary

Keep open. The related bug looks valid, but this PR is not merge-ready because it changes the current ACP runtime capability contract and removes caller visibility for unsupported advertised config keys.

## What This Changes

The PR changes ACP manager runtime-control application to send session config options only when `configOptionKeys` is non-empty and to forward only keys in that advertised set.

## Best Possible Solution

Keep the PR open but require a narrower fix that prevents Claude ACP from receiving unsupported generic config keys while preserving explicit error visibility and the documented `configOptionKeys` semantics, with regression coverage and an Unreleased changelog entry.

## Reproduction Assessment

Yes. The linked bug gives concrete Claude ACP reproduction steps, and current main statically shows persisted runtime options become `model`, `thinking`, `approval_policy`, and `timeout` calls when no strict key list is advertised.

## Solution Assessment

No. The PR points at the right failure mode, but the implementation is not the narrowest maintainable solution because it changes generic core behavior instead of proving whether the acpx/Claude adapter can advertise or translate the strict key set.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve unadvertised-key capability semantics:** `src/acp/control-plane/manager.runtime-controls.ts:99-108`
  - body: This branch gates all `setConfigOption` calls on `advertisedKeys.size > 0`. Current `AcpRuntimeCapabilities` says empty or undefined `configOptionKeys` means the backend accepts keys without a strict list, and the existing manager test uses that exact capability shape while expecting runtime options to be sent. With this patch, those config-capable backends stop receiving every persisted option.
  - confidence: 0.94
- **[P2] Keep surfacing unsupported advertised options:** `src/acp/control-plane/manager.runtime-controls.ts:99-100`
  - body: For backends that do advertise keys, `filter()` silently drops requested options that are not in `configOptionKeys` instead of raising `ACP_BACKEND_UNSUPPORTED_CONTROL`. A stored `approval_policy` or timeout can then appear accepted while being ignored, which regresses the current feedback path for misconfigured runtime controls.
  - confidence: 0.87
- **[P3] Add the required changelog entry:** `src/acp/control-plane/manager.runtime-controls.ts:99-108`
  - body: This is a user-facing ACP session fix, but the branch only changes runtime code. OpenClaw policy requires user-facing fixes to add an Unreleased `CHANGELOG.md` entry before merge.
  - confidence: 0.82
