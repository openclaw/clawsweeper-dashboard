## Summary

Keep open: this PR targets a real ACP/acpx failure, but the patch changes the existing capability contract so permissive backends lose all persisted runtime controls and advertised-key backends silently ignore unsupported controls.

## What This Changes

This PR updates `src/acp/control-plane/manager.runtime-controls.ts` so `applyManagerRuntimeControls` only calls `setConfigOption` for keys listed in `configOptionKeys` and sends no config options when no keys are advertised.

## Best Possible Solution

Revise or replace the patch with a capability-aware ACP/acpx fix that preserves undefined `configOptionKeys` as permissive, derives an explicit strict key list or adapter-specific mapping for Claude ACP, keeps unsupported controls visible, and adds focused tests plus a changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence patch regression path: existing fixtures omit `configOptionKeys` and expect persisted config options to be sent, while this PR would skip them; linked #73875 and #76564 provide concrete live Claude ACP failure reports for the original bug.

## Solution Assessment

No. Skipping every option when `configOptionKeys` is undefined fixes the strict-backend symptom by changing the established optional-capability meaning; the safer solution is an explicit strict capability/key-mapping path for Claude ACP/acpx while preserving caller feedback.
