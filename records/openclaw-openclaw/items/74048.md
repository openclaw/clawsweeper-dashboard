## Summary

Keep open: this PR is a focused attempt at a real ACP bug, but the current patch is not safe to merge because it changes the documented no-key capability contract and lacks after-fix real behavior proof. The linked root problem is now partly addressed on main by the advertised-key aliasing work, while the remaining timeout/control failure is still tracked by the open canonical issue and overlapping fix PRs.

## What This Changes

The PR modifies ACP runtime-control setup so `session/set_config_option` calls are skipped entirely when the backend has no advertised `configOptionKeys`.

## Best Possible Solution

Preserve the permissive no-key contract, keep the current advertised-key alias mapping, and fix the remaining unsupported auto-injected controls by skipping or fail-softing optional runtime controls such as timeout while keeping explicit user `set_config_option` errors strict; require live Claude ACP proof before merge.

## Reproduction Assessment

Yes for the PR regression: current source and tests show no-key ACP runtimes are expected to receive persisted config options, while this PR skips them. The original Claude ACP failure also has linked live diagnostics in https://github.com/openclaw/openclaw/issues/78546, but this review did not run a live ACP session.

## Solution Assessment

No. Skipping all config options when keys are absent violates the current optional-capability contract; the maintainable fix is advertised-key/status-derived mapping plus narrow skip or fail-soft handling for unsupported auto-injected controls.
