## Summary

Keep open. The PR targets the real ACP failure reported in #73875, but the patch changes the strict advertised-key path from an explicit unsupported-control error to silent filtering, and it conflicts with the current runtime capability contract for missing `configOptionKeys`.

## What This Changes

The PR changes ACP manager runtime-control setup to send `session/set_config_option` only for advertised `configOptionKeys`, and to send no config options when a backend does not advertise a key list.

## Best Possible Solution

Land a tested ACP runtime-control fix that avoids sending generic OpenClaw options to strict adapters like Claude ACP, while preserving explicit failure or visible warning when a backend advertises a strict key list and a requested security or correctness control is unsupported. Clarify the missing-key contract in types/tests, and keep adapter-specific key mapping in the ACP/acpx boundary.

## Reproduction Assessment

Yes. A focused reproduction is a runtime that supports `session/set_config_option`, returns no `configOptionKeys`, and has persisted `model`, `thinking`, `approval_policy`, or `timeout`; current main will send all of them. For the PR finding, advertise a strict key list that excludes `approval_policy` and the patch silently drops it.

## Solution Assessment

No. The PR addresses the right failure area, but it is not the best fix because it silently filters advertised-key mismatches and contradicts the documented undefined-key contract. A safer fix should handle the no-key-list case deliberately while keeping unsupported strict-list controls visible.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve errors for advertised-key mismatches:** `src/acp/control-plane/manager.runtime-controls.ts:99-108`
  - body: When a backend advertises `configOptionKeys`, current main throws `ACP_BACKEND_UNSUPPORTED_CONTROL` for a requested key outside that list. This patch filters those keys out, so controls such as `approval_policy` or `timeout` can be ignored while the session continues as if they applied. Keep the explicit failure or add a visible warning for advertised strict lists, and handle the no-key-list case separately.
  - confidence: 0.92

  - confidence: 0.8
