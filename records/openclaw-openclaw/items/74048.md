## Summary

Keep open: the linked Claude ACP bug is real, but this PR changes the existing ACP capability contract and can silently ignore requested runtime controls, including approval policy, before maintainer-owned semantics are settled.

## What This Changes

This PR changes `applyManagerRuntimeControls` to send ACP config options only when `configOptionKeys` is non-empty, and to skip all config options when no keys are advertised.

## Best Possible Solution

Revise this PR or replace it with a capability-aware fix that preserves the current permissive meaning of missing `configOptionKeys`, adds or derives an explicit strict key contract for Claude ACP, preserves visibility for unsupported permission controls, and includes focused tests plus a changelog entry.

## Reproduction Assessment

Yes. The existing manager test fixture covers the permissive no-`configOptionKeys` path and would lose all persisted config-option calls under this patch; linked issue #73875 also gives concrete live Claude ACP reproduction steps for the original failure.

## Solution Assessment

No. The patch fixes the strict-backend symptom by redefining missing `configOptionKeys` and silently dropping unsupported advertised-key options; the safer solution is to distinguish unknown/permissive capability reporting from a strict supported-key list.
