## Summary

Keep open: the underlying ACP bug is real, with #73875 and #76564 both pointing at OpenClaw-managed ACP failures, but this PR changes the documented missing-`configOptionKeys` contract and can silently drop requested runtime controls.

## What This Changes

The PR changes `applyManagerRuntimeControls` to call `setConfigOption` only when `configOptionKeys` is non-empty, filtering to advertised keys and skipping all config options when no keys are advertised.

## Best Possible Solution

Revise or replace the patch with a capability-aware fix that preserves the existing missing-key contract, handles strict Claude/ACPX unsupported keys explicitly, keeps unsupported requested controls visible, and adds focused tests plus a changelog entry.

## Reproduction Assessment

Yes, source-reproducible. The PR patch would skip `setConfigOption` when `configOptionKeys` is undefined, while the current manager test fixture uses that undefined-key capability shape and expects persisted runtime options to be applied.

## Solution Assessment

No. The patch fixes the strict-backend symptom by redefining missing `configOptionKeys` and silently filtering unsupported advertised-key options; a narrower fix should distinguish permissive unknown capabilities from a strict supported-key list.
