## Summary

Keep open: this PR targets a real ACP unsupported-config bug, but the current patch is not safe because it turns optional/no-list config capability semantics into skip-all behavior and still lacks after-fix real behavior proof.

## What This Changes

The PR changes ACP runtime-control setup so `session/set_config_option` is only called when advertised config keys are present, and all config options are skipped when no keys are advertised.

## Best Possible Solution

Preserve permissive no-list semantics, derive strict backend keys from capabilities or status, map canonical options to advertised aliases, and handle unsupported timeout controls narrowly with live Claude ACP proof.

## Reproduction Assessment

Yes for the PR regression: source inspection shows the no-`configOptionKeys` path currently expects persisted config options to be sent, while this patch would skip them. The original Claude ACP failure also has linked live diagnostics, but this read-only review did not run a live ACP session.

## Solution Assessment

No. Skipping all config options when keys are absent fixes one strict-backend symptom by violating the current optional-capability contract; the narrower fix is status/capability-derived key mapping plus focused unsupported-timeout handling.

## Review Findings

Overall correctness: patch is incorrect
