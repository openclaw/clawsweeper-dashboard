## Summary

Keep open: this PR targets a real ACP failure tracked by #73875 and reinforced by #76564, but the current diff changes the documented missing-`configOptionKeys` contract and can silently ignore requested runtime controls.

## What This Changes

This PR changes `applyManagerRuntimeControls` to send ACP config options only when a backend advertises a non-empty `configOptionKeys` list, filtering out all other keys.

## Best Possible Solution

Revise or replace this PR with a capability-aware ACP/acpx fix that preserves permissive missing-key semantics, exposes strict keys only when known, keeps unsupported requested controls visible, and adds focused tests plus a changelog entry.

## Reproduction Assessment

Yes. Source inspection shows the patch would stop the existing no-`configOptionKeys` manager test fixture from receiving persisted config options, and #73875/#76564 provide concrete managed Claude ACP failure reports.

## Solution Assessment

No. The patch fixes the strict-backend symptom by redefining missing `configOptionKeys` and silently dropping unsupported advertised-key options; the safer path is an explicit strict capability contract for backends that actually expose supported keys.
