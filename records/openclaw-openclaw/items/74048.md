## Summary

Keep open: the PR addresses a real ACP/Claude failure, but the current patch changes the ACP capability contract and can silently drop requested runtime controls such as `approval_policy`.

## What This Changes

The branch changes ACP runtime-control setup so `session/set_config_option` is sent only for keys listed in `configOptionKeys`, and sends no config options when no keys are advertised.

## Best Possible Solution

Revise or replace the patch with a capability-aware ACP/acpx fix that preserves missing-`configOptionKeys` permissive semantics, keeps unsupported requested controls visible, handles Claude’s strict config keys without sending generic options, adds regression tests, and includes a changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path for the PR regression: a no-`configOptionKeys` runtime would stop receiving persisted config-option calls, contradicting current tests and type comments; #73875 and #76564 describe the original Claude failure.

## Solution Assessment

No. The patch fixes the strict-backend symptom by redefining missing `configOptionKeys` and silently filtering unsupported options; the safer path is to preserve the current contract and add a targeted strict-backend/acpx handling path.
