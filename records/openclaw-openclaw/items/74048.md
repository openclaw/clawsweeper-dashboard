## Summary

Keep open: the underlying ACP bug is valid, but this PR still changes missing `configOptionKeys` from the current permissive contract to skip-all behavior, which would drop persisted runtime controls. It also needs after-fix real behavior proof before merge.

## What This Changes

This PR changes ACP runtime-control setup to send `session/set_config_option` only when advertised config keys are present, and to skip all config options when none are advertised.

## Best Possible Solution

Keep the capability-aware current-main direction: preserve permissive no-list semantics, derive strict backend keys from capabilities or status, map canonical options to advertised aliases, and require live Claude ACP proof before merging a fix.

## Reproduction Assessment

Yes for the PR regression: source inspection shows the no-`configOptionKeys` fixture currently expects persisted config options to be sent, while this PR would skip them. The original Claude ACP failure also has linked live diagnostics in https://github.com/openclaw/openclaw/issues/78546, but this read-only review did not run a live ACP session.

## Solution Assessment

No. Skipping all config options when keys are absent fixes one strict-backend symptom by violating the current optional-capability contract; the narrower solution is status/capability-derived key mapping while preserving permissive runtimes.
