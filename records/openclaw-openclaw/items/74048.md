## Summary

Close as superseded: the linked Claude ACP config-key bug was partly resolved on main by a merged canonical mapping fix, and the remaining timeout/unsupported-control behavior is now tracked by a newer canonical issue and stronger open fix candidates; this branch would also change the current permissive `configOptionKeys` contract by skipping all controls when no strict key list is advertised.

## What This Changes

The branch changes ACP runtime-control setup to send `session/set_config_option` only when `configOptionKeys` is non-empty, skipping all config options when no keys are advertised.

## Best Possible Solution

Keep the merged main alias-mapping implementation, review the remaining timeout/unsupported-control fix on https://github.com/openclaw/openclaw/issues/78546 and its focused candidate PRs, and avoid merging this branch's skip-all capability semantics.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path for the PR regression: current tests and types treat missing `configOptionKeys` as permissive, while the PR's new branch skips every persisted config option in that state.

## Solution Assessment

No. The safer path is the current main alias/status-key mapping plus a focused timeout/unsupported-control fix; this PR solves the symptom by redefining missing capability keys in a way that would break existing permissive backends.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Preserve permissive config-option capabilities:** `src/acp/control-plane/manager.runtime-controls.ts:155-158`
  - body: `configOptionKeys` is optional and current tests rely on empty or undefined meaning the backend accepts config options without a strict list. This branch turns that state into a no-op, so persisted model, thinking, approval policy, timeout, and backend extras are never applied for existing permissive backends.
  - confidence: 0.94

## Security Review

Status: needs_attention

  - body: When `advertisedKeys.size === 0`, the patch skips all config options, including `approval_policy`; a backend that supports the control but does not advertise a strict list would continue under defaults while OpenClaw behaves as if persisted runtime controls were applied.
