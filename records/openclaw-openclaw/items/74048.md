## Summary

Keep open: this targets a real ACP unsupported-control bug, but the patch violates the current ACP capability contract by treating missing advertised keys as no supported keys, which drops persisted controls for permissive runtimes and still lacks real behavior proof.

## What This Changes

The PR changes ACP runtime-control setup to skip all `session/set_config_option` calls when the backend does not advertise config option keys.

## Best Possible Solution

Preserve permissive no-list semantics, keep advertised/status-derived alias mapping, and fix the timeout/control failure by skipping or fail-softing unsupported auto-injected controls while keeping explicit user config writes strict.

## Reproduction Assessment

Yes for the PR regression: source inspection shows current main treats missing `configOptionKeys` as permissive and tests expect persisted config options to be sent, while this PR skips them. The original Claude ACP failure also has linked live diagnostics in https://github.com/openclaw/openclaw/issues/78546, but this read-only review did not run a live ACP session.

## Solution Assessment

No. Skipping all config options when keys are absent is broader than the ACP contract allows; the narrower maintainable fix is to filter/fail-soft unsupported auto-injected options while preserving permissive runtimes and strict explicit user writes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve permissive config-option semantics:** `src/acp/control-plane/manager.runtime-controls.ts:153-158`
  - body: `configOptionKeys` is optional, and current main treats empty or undefined as a permissive backend with no strict advertised list. This branch skips every persisted model, thinking, approval-policy, timeout, and backend-extra option in that case, so permissive ACP runtimes lose configured controls before a turn.
  - confidence: 0.93
