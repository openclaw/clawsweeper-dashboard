## Summary

Keep open; this tackles a real ACP/Claude failure, but the current diff is not merge-ready because it changes the documented ACP capability contract and can hide unsupported approval controls.

## What This Changes

The PR changes ACP runtime-control setup to send `session/set_config_option` only for advertised `configOptionKeys`, and to send no config options when no keys are advertised.

## Best Possible Solution

Land a capability-aware ACP fix that preserves the existing permissive meaning of missing `configOptionKeys`, exposes strict backend key support explicitly, keeps unsupported controls visible, adds regression tests, and includes the required changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path for the PR regression: under this patch, the existing no-`configOptionKeys` manager fixture would stop receiving persisted model, thinking, approval_policy, and timeout config-option calls.

## Solution Assessment

No. The patch fixes the strict-backend symptom by redefining missing `configOptionKeys` and silently filtering unsupported advertised-key options; a safer fix should preserve the current contract and add an explicit strict capability path for Claude/acpx.
