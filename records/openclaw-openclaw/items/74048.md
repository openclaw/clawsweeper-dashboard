## Summary

Keep open: the linked ACP/Claude failure looks real, but this PR conflicts with the current ACP capability contract, would regress permissive backends, and can silently ignore requested permission controls.

## What This Changes

The branch changes `applyManagerRuntimeControls` so ACP config options are sent only when `configOptionKeys` is non-empty and filtered to advertised keys.

## Best Possible Solution

Revise or replace this with a capability-aware fix that preserves the permissive missing-key contract, makes strict ACPX/Claude support explicit, keeps unsupported controls visible, and adds focused tests plus a changelog entry.

## Reproduction Assessment

Yes. Source inspection shows this patch would skip every persisted config option for the existing no-`configOptionKeys` manager fixture; linked #73875 gives live Claude ACP reproduction steps for the original failure.

## Solution Assessment

No. The patch fixes the strict-backend symptom by changing the meaning of missing `configOptionKeys` and by silently filtering requested controls; a safer solution should distinguish permissive unknown capability reporting from strict advertised-key support.
