## Summary

Keep open: the underlying ACP bug is real and has corroborating reports, but this PR is not mergeable as written because it changes the documented missing-`configOptionKeys` contract and can silently ignore requested runtime and permission controls.

## What This Changes

The PR changes `applyManagerRuntimeControls` to send ACP config options only when keys are advertised, and to send none when no key list is advertised.

## Best Possible Solution

Revise or replace this PR with a capability-aware fix that preserves empty/undefined `configOptionKeys` semantics, handles Claude ACP strict keys through the ACPX/adapter boundary, keeps unsupported permission controls visible, and adds focused tests plus a changelog entry.

## Reproduction Assessment

Yes. Source inspection shows the patch would stop all persisted config-option calls for the current default no-`configOptionKeys` fixture, and #73875 provides concrete live Claude ACP reproduction steps for the original failure.

## Solution Assessment

No. The patch fixes the strict-backend symptom by redefining missing `configOptionKeys` and silently dropping requested controls; the narrower maintainable path is to preserve the current contract and make strict backend capabilities explicit.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve missing configOptionKeys semantics:** `src/acp/control-plane/manager.runtime-controls.ts:99`
  - body: `configOptionKeys` is documented as optional, with empty or undefined meaning the backend accepts keys without a strict advertised list. Moving every `setConfigOption` call under `advertisedKeys.size > 0` means permissive ACP backends no longer receive persisted model, thinking, `approval_policy`, timeout, or backend-extra updates before turns.
  - confidence: 0.93
- **[P2] Keep unsupported controls visible:** `src/acp/control-plane/manager.runtime-controls.ts:100-108`
  - body: For backends that do advertise a strict key list, current main throws `ACP_BACKEND_UNSUPPORTED_CONTROL` when a requested option is absent. Filtering the list silently lets requested controls such as `approval_policy` become no-ops while the session continues as if configuration succeeded.
  - confidence: 0.9
- **[P3] Add the required changelog entry:** `src/acp/control-plane/manager.runtime-controls.ts:99`
  - body: This is a user-facing ACP bug fix and the PR body says it closes #73875, but the branch only changes the runtime-control source file. Repo policy requires a `CHANGELOG.md` entry for user-facing fix changes.
  - confidence: 0.9
- **[P3] Remove the unused runtime error import:** `src/acp/control-plane/manager.runtime-controls.ts:1`
  - body: After removing the unsupported-key throw, `AcpRuntimeError` no longer has a use in this module. Drop it from the import so the patch does not leave dead code for lint/type hygiene checks.
  - confidence: 0.82
