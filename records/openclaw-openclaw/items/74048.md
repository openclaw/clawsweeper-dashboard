## Summary

Keep open: this PR targets a real ACP unsupported-config bug, but the current patch drops every persisted config option when no key list is advertised, which conflicts with OpenClaw's current permissive capability contract and can silently disable controls. It also still lacks contributor real-behavior proof, while narrower overlapping ACP timeout/config-control PRs are available for maintainer comparison.

## What This Changes

The PR changes `applyManagerRuntimeControls` to skip sending all runtime config options when the backend does not advertise `configOptionKeys`.

## Best Possible Solution

Keep the permissive no-key contract, map canonical options to backend-advertised aliases from capabilities or status, skip or tolerate only unsupported optional auto-injected controls, and require live ACP proof before merge.

## Reproduction Assessment

Yes for the PR regression: source inspection shows current main treats missing `configOptionKeys` as permissive and tests expect persisted config controls to be sent, while this PR would skip them all. I did not run a live ACP session in this read-only review.

## Solution Assessment

No. The patch fixes one strict-backend symptom by changing the no-key capability contract; the narrower maintainable solution is advertised-key alias mapping plus scoped timeout/control fallback behavior.
