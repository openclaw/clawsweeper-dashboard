## Summary

Keep open: this PR targets a real Claude ACP failure, but the patch is not mergeable because it breaks the documented optional-capability contract and can silently ignore requested runtime controls, including approval policy.

## What This Changes

The PR changes ACP runtime-control application to filter config options to advertised keys and send none when the backend reports no `configOptionKeys`.

## Best Possible Solution

Implement a capability-aware ACPX/ACP fix that preserves missing `configOptionKeys` as permissive, derives or maps strict per-backend keys for Claude ACP, keeps unsupported requested controls visible, and adds focused regression coverage plus a changelog entry.

## Reproduction Assessment

Yes. Source inspection shows the patch would skip all persisted config options for the existing no-`configOptionKeys` test fixture, while linked bug #73875 provides concrete live Claude ACP steps for the original strict-adapter failure.

## Solution Assessment

No. The patch fixes the strict-backend symptom by changing the meaning of missing `configOptionKeys` and by silently dropping unsupported requested controls; a narrower fix should add strict capability discovery or mapping without changing the existing permissive contract.
