## Summary

Keep open: current main and the latest release still ignore `QQBOT_DATA_DIR`, and the open linked implementation PR remains the active fix path.

## What This Changes

The issue requests that QQBot honor `QQBOT_DATA_DIR` instead of always using `~/.openclaw/qqbot` for diagnostics and persisted state.

## Best Possible Solution

Review, repair, land, or replace https://github.com/openclaw/openclaw/pull/78253 with a focused QQBot helper fix that validates `QQBOT_DATA_DIR`, aligns diagnostics, and adds regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main is high-confidence: `getQQBotDataPath()` always joins `getHomeDir()`, `.openclaw`, and `qqbot`, while diagnostics and state stores call that helper, so `QQBOT_DATA_DIR` cannot affect the directory.

## Solution Assessment

Yes in direction. Centralizing the override in the QQBot data-path helper with focused tests is the narrow maintainable fix, but this issue should remain open until the linked PR lands or is replaced.
