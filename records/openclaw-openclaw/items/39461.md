## Summary

Keep open: current main and the latest release still ignore `QQBOT_DATA_DIR`, and the open linked fix PR with closing syntax is the active implementation path rather than a reason to close this issue early.

## What This Changes

QQBot should honor `QQBOT_DATA_DIR` for its data directory instead of always using `~/.openclaw/qqbot`.

## Best Possible Solution

Review, repair, land, or replace https://github.com/openclaw/openclaw/pull/78253 with a centralized QQBot data-path helper that trims and validates `QQBOT_DATA_DIR`, aligns diagnostics with the effective path, and covers session and credential backup paths with regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: set `QQBOT_DATA_DIR`, restart the Gateway, and diagnostics plus persistence still resolve through `getHomeDir()/.openclaw/qqbot` on current main.

## Solution Assessment

Yes in direction. Centralizing the override in the QQBot data-path helper is the narrow maintainable fix, but this issue should remain open until the linked PR lands or is replaced.
