## Summary

Keep open: current main and the latest release still ignore the advertised QQBOT_DATA_DIR override, and the open linked fix PR should be reviewed or replaced rather than closing this issue early.

## What This Changes

QQBot should honor QQBOT_DATA_DIR as the effective data root instead of always using the home-directory .openclaw/qqbot path.

## Best Possible Solution

Review, repair, land, or replace https://github.com/openclaw/openclaw/pull/78253 with a centralized QQBot data-path helper that trims and validates QQBOT_DATA_DIR, aligns diagnostics with the effective path, and covers session plus credential backup paths with regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: set QQBOT_DATA_DIR, restart the Gateway, and diagnostics plus persistence still resolve through getHomeDir()/.openclaw/qqbot on current main.

## Solution Assessment

Yes in direction. Centralizing the override in the QQBot data-path helper is the narrow maintainable fix, but this issue should remain open until the linked PR lands or is replaced.
