## Summary

Keep open. Current `main` still advertises `QQBOT_DATA_DIR` in QQBot diagnostics but the central QQBot data-path helper never reads it, so the reported Windows path override bug remains valid and narrow.

## What This Changes

Honor `QQBOT_DATA_DIR` as the QQBot persistent data directory instead of always using `~/.openclaw/qqbot`.

## Best Possible Solution

Implement the override in the QQBot-owned persistent-storage helper, using a trimmed non-empty `QQBOT_DATA_DIR` as the root while preserving the default `~/.openclaw/qqbot` behavior and existing media-path boundaries.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: set `QQBOT_DATA_DIR`, then `runDiagnostics()` and QQBot persistence still call `getQQBotDataPath()`/`getQQBotDataDir()`, which always return paths under the home directory.

## Solution Assessment

Yes. The narrow maintainable fix is plugin-owned: centralize env handling in `getQQBotDataPath()`/`getQQBotDataDir()` and update diagnostics, docs, metadata, tests, and changelog to match the already-advertised variable.
