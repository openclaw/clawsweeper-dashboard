## Summary

Keep open: current main and the latest release still hard-code the QQBot data root while diagnostics tell affected Windows users to use QQBOT_DATA_DIR, and an open linked PR is the right path to finish the fix.

## What This Changes

QQBot should honor the advertised QQBOT_DATA_DIR override instead of resolving data, sessions, and credential backups under the default user-home path.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/78253 with a centralized QQBot data-path helper that validates QQBOT_DATA_DIR, aligns diagnostics with the effective path, and covers session plus credential backup paths with regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: set QQBOT_DATA_DIR, restart the Gateway, and diagnostics plus QQBot persistence still resolve under getHomeDir()/.openclaw/qqbot on current main.

## Solution Assessment

Yes in direction. Centralizing the override in the QQBot data-path helper is the narrow maintainable fix, but this issue should remain open until the linked PR lands or is replaced.
