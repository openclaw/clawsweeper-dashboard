## Summary

Keep open: current main and the latest release still ignore `QQBOT_DATA_DIR`, and the linked open implementation PR with closing syntax remains the active fix path.

## What This Changes

The issue requests QQBot to honor `QQBOT_DATA_DIR` instead of always resolving data storage under the default OpenClaw home directory.

## Best Possible Solution

Review, repair, land, or replace https://github.com/openclaw/openclaw/pull/78253 with a small QQBot helper fix that validates the override, aligns diagnostics, and adds focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main is high-confidence: setting `QQBOT_DATA_DIR` cannot affect diagnostics, sessions, known users, ref indexes, or credential backups because all of those paths still route through the hard-coded `getHomeDir()/.openclaw/qqbot` helper.

## Solution Assessment

Yes in direction. Centralizing the override in the QQBot data-path helper is the narrow maintainable fix, but this issue should stay open until the linked PR is corrected, proven, and merged or superseded.
