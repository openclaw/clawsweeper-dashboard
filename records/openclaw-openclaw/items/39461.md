## Summary

Keep open. Current main still advertises `QQBOT_DATA_DIR` in QQBot diagnostics, but the central QQBot data-path helper always returns `~/.openclaw/qqbot`, so the reported Windows override bug remains valid and narrow.

## What This Changes

The issue requests that the QQBot plugin honor `QQBOT_DATA_DIR` for its data directory instead of always using the default home-directory path.

## Best Possible Solution

Implement the override in the QQBot-owned storage helper, using a trimmed non-empty `QQBOT_DATA_DIR` as the data root while preserving default `~/.openclaw/qqbot` behavior and existing media-path boundaries.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: set `QQBOT_DATA_DIR`, then `runDiagnostics()` and QQBot persistence still call `getQQBotDataDir()`/`getQQBotDataPath()`, which always return the home-directory root.

## Solution Assessment

Yes. The narrowest maintainable fix is plugin-owned: centralize env handling in `getQQBotDataPath()`/`getQQBotDataDir()` and update diagnostics, docs, metadata, tests, and changelog to match the already-advertised variable.
