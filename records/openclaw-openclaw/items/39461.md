## Summary

## What This Changes

The issue requests QQBot to honor `QQBOT_DATA_DIR` for its data directory instead of always resolving under the user's home `.openclaw/qqbot` path.

## Best Possible Solution

Land or replace the linked QQBot helper PR with a small env override in the central path helper, regression coverage for data and media paths, diagnostics aligned to the effective path, and redacted real Gateway proof.

Yes. Source inspection on current main is high-confidence: set `QQBOT_DATA_DIR` and run QQBot diagnostics or persist session state; the helpers still ignore `process.env` and resolve under `getHomeDir()/.openclaw/qqbot`.

## Solution Assessment

Yes in direction. Centralizing the override in the QQBot data-path helper is the narrow maintainable fix, but the issue should remain open until the linked PR is corrected, proven, and merged or superseded.
