## Summary

Keep open. Current main still does not honor `QQBOT_DATA_DIR`: the QQBot persistent-storage helper always derives `~/.openclaw/qqbot`, diagnostics still recommends an override that is not implemented, and the docs/metadata/tests have not been updated for the variable.

## What This Changes

The issue requests QQBot persistent storage to use `QQBOT_DATA_DIR` when set instead of always using the default user-home path.

## Best Possible Solution

Implement `QQBOT_DATA_DIR` in the QQBot plugin’s central persistent-storage helper, preserving the existing default path, subpath joining, and lazy directory creation; then align diagnostics, docs, plugin metadata, focused tests, and changelog credit so the recommended override is real and discoverable.
