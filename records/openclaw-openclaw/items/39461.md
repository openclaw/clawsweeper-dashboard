- `impact:auth-provider`: QQBot credential backup and recovery for appId/clientSecret depend on the affected path helper.

## Summary

Keep open: current main and the latest release still ignore `QQBOT_DATA_DIR`, and the open linked PR with closing syntax remains the active fix path rather than a reason to close the issue early.

## What This Changes

The issue requests QQBot to honor `QQBOT_DATA_DIR` so diagnostics, sessions, media-adjacent state, and credential backups use the configured data directory instead of `~/.openclaw/qqbot`.

## Best Possible Solution

Review, repair, land, or replace https://github.com/openclaw/openclaw/pull/78253 with a centralized QQBot data-path helper that trims and validates `QQBOT_DATA_DIR`, aligns diagnostics with the effective path, and covers session and credential backup paths with regression tests.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: set `QQBOT_DATA_DIR`, restart the Gateway, and diagnostics plus persistence still resolve through `getHomeDir()/.openclaw/qqbot`.

## Solution Assessment

Yes in direction. Centralizing the override in the QQBot data-path helper is the narrow maintainable fix, but this issue should remain open until the linked PR lands or is replaced.
