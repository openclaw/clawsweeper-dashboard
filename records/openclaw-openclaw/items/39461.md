## Summary

Keep open: current main and the latest release still ignore `QQBOT_DATA_DIR`, while the open linked implementation PR remains the active fix path for this issue.

## What This Changes

The issue requests that QQBot use the `QQBOT_DATA_DIR` environment variable instead of always resolving data paths under `~/.openclaw/qqbot`.

## Best Possible Solution

Review, repair, land, or replace https://github.com/openclaw/openclaw/pull/78253 with a narrow QQBot helper fix that validates the override, aligns diagnostics, and adds focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main is high-confidence: setting `QQBOT_DATA_DIR` cannot affect diagnostics, sessions, ref data, known users, or credential backups because those paths still route through the hard-coded `getHomeDir()/.openclaw/qqbot` helper.

## Solution Assessment

Yes in direction. Centralizing `QQBOT_DATA_DIR` handling in the QQBot data-path helper is the narrow maintainable fix, but this issue should stay open until the linked PR is corrected, proven, and merged or superseded.
