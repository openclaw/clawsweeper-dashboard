## What This Changes

QQBot should honor `QQBOT_DATA_DIR` as the base data directory instead of always using `~/.openclaw/qqbot` for diagnostics and persisted state.

## Best Possible Solution

Land a corrected QQBot helper change that trims and validates the env override, aligns diagnostics/docs/tests, and includes redacted real Gateway or QQBot setup proof.

Yes. Source inspection gives a high-confidence path: set `QQBOT_DATA_DIR`, then `runDiagnostics()` and persisted QQBot state still flow through helpers that always resolve under `getHomeDir()/.openclaw/qqbot`.

## Solution Assessment

Yes in direction, but not complete through the current open PR yet. Centralizing the override in the QQBot data helper is the maintainable fix, with diagnostics and tests updated alongside it.
