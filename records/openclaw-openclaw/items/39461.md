## Summary

Keep open: current main and v2026.5.22 still ignore `QQBOT_DATA_DIR` while QQBot diagnostics tell affected Windows users to set it, and the open linked PR is the right implementation path.

## What This Changes

The issue asks QQBot to honor `QQBOT_DATA_DIR` for its data root instead of always using `~/.openclaw/qqbot`.

## Best Possible Solution

Review, repair, and land or replace https://github.com/openclaw/openclaw/pull/78253 with a centralized QQBot data-path helper that validates `QQBOT_DATA_DIR`, aligns diagnostics with the effective path, and covers session plus credential backup paths with regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: set `QQBOT_DATA_DIR`, restart the Gateway, then diagnostics and QQBot persistence still resolve under `getHomeDir()/.openclaw/qqbot` on current main.

## Solution Assessment

Yes in direction. Centralizing a validated override in the QQBot data-path helper is the narrow maintainable fix, but this issue should remain open until the linked PR lands or is replaced.
