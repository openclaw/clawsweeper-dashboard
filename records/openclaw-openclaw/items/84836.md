- **[P1] Redact URLs before emitting the diagnostic event:** `src/utils/fetch-timeout.ts:99`
  - body: `sanitizeTimeoutLogUrl` only removes credentials, query, and hash; it leaves path segments intact, and Telegram Bot API tokens live in paths like `/bot<token>/getMe`. Because `emitDiagnosticEvent` delivers this payload to diagnostic/plugin listeners before the normal log redaction path, a delayed Telegram timeout can expose the bot token through the new `url` field. Redact with the existing sensitive-text redactor before adding `url`, or omit URL from this event and keep only the low-cardinality operation/timing fields.
  - confidence: 0.9
