## Summary

Keep open: the merged Gemini CLI auth bridge fixes a related backend execution path, but current `main` still routes the reported `infer model run --local` command through local simple completion, so #88748 is not enough proof that this exact API-key failure is fixed.

## What This Changes

The issue reports that `openclaw infer model run --local --model google-gemini-cli/gemini-3-flash-preview` rejects a Gemini API key that works through Gemini REST and the upstream `gemini` CLI.

## Best Possible Solution

Keep #85610 open until maintainers choose and implement the local `google-gemini-cli/*` model-run contract, with focused coverage and a redacted live key proof for the selected behavior.

## Reproduction Assessment

Yes, at source level: the reported command enters local simple completion, while the merged #88748 fix stages credentials only for the Gemini CLI backend subprocess. I did not run a live Gemini credential repro in this read-only review.

## Solution Assessment

No: closing against #88748 would treat a related backend fix as covering a different local inference transport. The safer solution is to fix or explicitly define `google-gemini-cli/*` behavior for local `model run`.
