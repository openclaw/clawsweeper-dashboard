## What This Changes

The issue reports that Android realtime Talk plays assistant audio through Bluetooth earbuds but records user speech from the phone microphone instead of the connected headset microphone.

## Best Possible Solution

Implement Android-owned audio-route handling for realtime Talk: prefer and verify a connected headset input or communication route when available, clear it on stop, and add regression plus real-device proof without changing Gateway/provider contracts.

## Reproduction Assessment

Yes. The reporter supplied high-confidence device steps with ADB route output, and current main still has the matching unrouted `AudioRecord` capture path; I did not rerun the Bluetooth hardware scenario locally.

## Solution Assessment

Unclear. Automatic Bluetooth communication-device routing looks like the likely repair, but an Android owner should choose whether the fix also needs visible route selection or new permission/audio-mode behavior.
