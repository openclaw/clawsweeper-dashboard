Fix Android realtime Talk so connected Bluetooth headset microphones are used for capture, or so users can see/select the active input route.

## Best Possible Solution

Fix the Android Talk capture owner path so realtime Talk selects and restores the appropriate Bluetooth communication/input route when available, reports the active route when useful, and is validated on a real Android Bluetooth headset setup.
