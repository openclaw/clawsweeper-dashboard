## Summary

Keep open: the PR is useful security hardening, but the current branch still has a fail-open path for partial non-owner exec defaults and lacks real behavior proof.

## What This Changes

The branch passes sender ownership into auto-reply exec override resolution, denies non-owner senders with no exec overrides, ignores inline/session exec fields for non-owner senders, and adds resolver tests.

## Best Possible Solution

Revise the PR so non-owner auto-reply ignores user-controlled inline/session exec fields and only grants exec when operator policy is complete; otherwise force security=deny and ask=always, coordinated with https://github.com/openclaw/openclaw/pull/64719 and verified with real runtime proof.

## Reproduction Assessment

Yes. A source-level repro is to evaluate the PR-head resolver with senderIsOwner=false and agentExecDefaults containing only host or node, then follow bash exec's current fallback where missing security and ask become full and off.

## Solution Assessment

No. The direction is right, but returning partial operator defaults unchanged is not a safe command-execution boundary; the safer fix is to require complete safe policy fields or fill missing policy with deny/always for non-owner senders.
