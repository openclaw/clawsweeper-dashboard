## Summary

Keep open: the PR targets a real command-execution hardening gap on current main, but the branch still fails open for partial non-owner agent exec defaults and only provides unit-level proof.

## What This Changes

The branch adds sender ownership to auto-reply exec override resolution, passes it from reply directives, and tests deny defaults plus ignored inline/session exec overrides for non-owner senders.

## Best Possible Solution

Land a coordinated security fix that ignores user-controlled inline/session exec fields for non-owner auto-reply and fails closed unless explicit operator policy supplies complete safe exec permissions.

## Reproduction Assessment

Yes. A source-level reproduction is to evaluate the PR-head resolver with senderIsOwner=false and partial agentExecDefaults, then follow bash exec's current fallback where missing security and ask become full and off.

## Solution Assessment

No. The direction is right, but returning partial operator defaults unchanged is not a safe command-execution boundary; the safer fix is to require complete policy fields or fill missing policy with deny/always for non-owner senders.
