## Summary

Keep this PR open. The hardening direction is valid, but the current diff still has a command-execution fail-open path for partial agent exec defaults, and the final auto-reply tool boundary overlaps the related open security PR.

## What This Changes

The PR adds sender ownership to auto-reply exec override resolution, forwards the command owner flag, and adds unit coverage for non-owner exec deny defaults and ignored inline/session overrides.

## Best Possible Solution

Revise the non-owner auto-reply exec boundary so user-controlled inline/session exec fields are ignored and missing non-owner policy fields fail closed, then coordinate the final tool-access model with the related channel dangerous-tool PR before merge.

## Reproduction Assessment

Yes. A focused source-level reproduction is to call the PR version of resolveReplyExecOverrides with senderIsOwner false and agentExecDefaults containing only host or node, then follow the downstream exec defaults to security=full and ask=off.

## Solution Assessment

No. The direction is right, but returning partial operator defaults as-is is not the narrowest safe fix; the resolver should fail closed for missing non-owner security/ask or require a fully explicit safe policy.
