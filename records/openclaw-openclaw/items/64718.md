## Summary

Keep open: current main still lacks sender-aware auto-reply exec override resolution, but this PR head is not safe to merge because partial non-owner agent exec defaults can still fall through to permissive exec policy, and the contributor has only provided unit-level proof.

## What This Changes

The PR adds sender ownership to auto-reply exec override resolution, forwards the owner flag from reply directives, and expands resolver unit coverage.

## Best Possible Solution

Revise the PR so non-owner auto-reply ignores user-controlled inline/session exec fields and either fills missing operator policy with `security=deny` and `ask=always` or requires an explicit safe operator policy, coordinated with https://github.com/openclaw/openclaw/pull/64719.

## Reproduction Assessment

Yes. A focused source-level reproduction is to evaluate the PR-head resolver with `senderIsOwner: false` and `agentExecDefaults` containing only `host` or `node`, then follow current exec defaults resolution where missing `security` and `ask` become `full` and `off`.

## Solution Assessment

No. Ignoring inline/session exec values for non-owner senders is the right direction, but returning partial operator defaults unchanged is not a safe command-execution boundary.
