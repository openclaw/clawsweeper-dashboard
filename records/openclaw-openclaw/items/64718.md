## Summary

Keep open. Current main still lacks sender-aware auto-reply exec defaulting, but this PR remains unsafe as written because partial non-owner agent exec defaults can still fall through to permissive host exec defaults.

## What This Changes

The PR adds `senderIsOwner` to auto-reply exec override resolution, passes the command owner flag at the call site, and adds resolver tests for non-owner deny defaults and ignored inline/session exec overrides.

## Best Possible Solution

Revise the PR so non-owner auto-reply ignores user-controlled inline/session exec fields and only grants exec when operator policy is complete; otherwise fill or force `security=deny` and `ask=always`, then coordinate the final boundary with https://github.com/openclaw/openclaw/pull/64719.

## Reproduction Assessment

Yes, at source level. Evaluate the PR-head resolver with `senderIsOwner=false` and `agentExecDefaults` containing only `host` or `node`, then follow current exec fallback where missing `security` and `ask` resolve to `full` and `off`.

## Solution Assessment

No. The direction is right, but returning partial operator defaults unchanged is not a safe command-execution boundary; the safer fix is to require complete safe policy or fill missing fields with deny/always for non-owner senders.
