## Summary

Keep open: current main still lacks sender-aware auto-reply exec defaults, and the PR remains security-sensitive because its latest diff can fail open when non-owner agent exec defaults are partial.

## What This Changes

The PR adds sender ownership to auto-reply exec override resolution, forwards `command.senderIsOwner` at the directive call site, and adds unit coverage for non-owner exec defaults plus ignored inline/session overrides.

## Best Possible Solution

Revise the non-owner auto-reply exec boundary so user-controlled inline/session exec fields are ignored and missing non-owner policy fields fail closed, then align the final tool-access model with #64719.

## Reproduction Assessment

Yes. A focused unit reproduction is to call the PR version of `resolveReplyExecOverrides` with `senderIsOwner: false` and `agentExecDefaults` containing only `host` or `node`, then confirm downstream exec creation can still fall back to `security=full` and `ask=off`.

## Solution Assessment

No. The direction is right, but returning partial operator defaults as-is is not the narrowest safe fix; the resolver should merge in `security=deny` and `ask=always` for missing non-owner fields or require a fully explicit safe policy.
