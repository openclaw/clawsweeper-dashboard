## Summary

Keep open: current main still lacks sender-aware exec override handling for auto-reply, but this PR can still fail open for non-owner senders when operator exec defaults are partial, and it needs security review plus real behavior proof before merge.

## What This Changes

The PR adds sender ownership to auto-reply exec override resolution, passes the owner flag at the reply-directives call site, and adds resolver unit tests.

## Best Possible Solution

Revise the PR so non-owner auto-reply ignores user-controlled inline/session exec fields and fails closed when operator exec policy is absent or partial, coordinated with https://github.com/openclaw/openclaw/pull/64719.

## Reproduction Assessment

Yes. A focused source-level reproduction is to evaluate the PR-head resolver with `senderIsOwner: false` and `agentExecDefaults` containing only `host` or `node`, then follow current exec defaults where missing `security` and `ask` become `full` and `off`.

## Solution Assessment

No. Ignoring inline/session exec values for non-owner senders is the right direction, but returning partial operator defaults unchanged is not a safe command-execution boundary.
