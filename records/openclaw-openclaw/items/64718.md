## Summary

Keep open: current main still lacks sender-aware auto-reply exec override handling, but this PR still has a blocking fail-open security case and only unit-level proof for a live channel behavior change.

## What This Changes

The PR adds sender ownership to auto-reply exec override resolution, forwards the command owner flag, and adds resolver unit tests for non-owner exec denial and ignored inline/session overrides.

## Best Possible Solution

Revise the PR so non-owner auto-reply ignores user-controlled inline/session exec fields and fails closed unless operator policy explicitly supplies safe `security` and `ask`, then coordinate the final boundary with https://github.com/openclaw/openclaw/pull/64719.

## Reproduction Assessment

Yes, source-level. Evaluate the PR-head resolver with `senderIsOwner: false` and partial `agentExecDefaults` containing only `host` or `node`, then follow downstream exec defaults where missing `security` and `ask` become `full` and `off`.

## Solution Assessment

No. Ignoring inline/session exec values for non-owner senders is the right direction, but returning partial operator defaults unchanged is not a safe command-execution boundary.
