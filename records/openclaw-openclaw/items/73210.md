## Summary

Keep open. Current main still lacks the requested auto-TTS voice-only/caption delivery policy and the prompt hint still describes TTS mechanics without speech-first writing guidance.

## What This Changes

Requests a new auto-TTS text-delivery mode for voice-only or caption-only replies plus stronger speech-oriented system-prompt guidance.

## Best Possible Solution

Add a backward-compatible generic auto-TTS text-delivery policy that defaults to current behavior, suppresses visible text only after successful voice-style audio delivery, preserves text fallback and non-voice attachment context, and adds concise speech-first prompt guidance with docs and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: enable `messages.tts.auto: "always"` for a normal final text reply, and the auto-TTS success path returns audio while preserving visible `text` for voice-capable channel delivery.

## Solution Assessment

Yes. The narrow maintainable solution is a generic speech-core/config delivery policy plus prompt-hint update; per-channel suppression would duplicate policy, while the broader spoken-rewrite pipeline from #22028 is larger than this request.
