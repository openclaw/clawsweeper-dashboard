## Summary

Keep open. Current main still lacks the requested generic auto-TTS voice-only/text-delivery policy and speech-first prompt guidance; some channel-level voice suppression already exists, but it does not provide the requested configurable auto-TTS behavior.

## What This Changes

Request adds an auto-TTS voice-only/caption text-delivery mode and stronger system prompt guidance for spoken, conversational output.

## Best Possible Solution

Add a backward-compatible `messages.tts` text-delivery policy that defaults to current behavior, suppresses visible text only after successful native voice-style audio delivery, preserves text fallback on TTS failure or non-voice attachments, and adds concise speech-first prompt guidance.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: enable auto-TTS for a normal final reply on a voice-capable channel and the success path preserves visible `text` while adding generated audio.

## Solution Assessment

Yes. The narrow maintainable direction is a generic speech-core/config policy plus prompt-hint update; pushing suppression separately into each channel would duplicate policy, while reviving the full #22028 spoken-rewrite pipeline would be broader than this issue.
