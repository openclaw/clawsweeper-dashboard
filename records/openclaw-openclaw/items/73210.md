## Summary

Keep open. Current main still lacks a generic auto-TTS text-delivery policy and the requested speech-first prompt guidance; Feishu now suppresses visible text for native voice media by default, but Telegram, WhatsApp, Matrix, config, commands, and docs still do not expose the requested user-selectable voice-only/caption/preserve behavior.

## What This Changes

Request a configurable auto-TTS voice-only text-delivery mode plus stronger system-prompt guidance for natural spoken output.

## Best Possible Solution

Add a backward-compatible shared auto-TTS text-delivery policy plus a concise speech-first prompt hint; channel plugins should honor the policy only after successful native voice delivery and preserve text fallback for failures or non-voice attachments.

## Reproduction Assessment

Yes. Source inspection on current main shows the generic auto-TTS path keeps visible text while adding audio, and the prompt/config/command/docs surfaces still lack the requested policy and speech guidance.

## Solution Assessment

Unclear for the exact proposed option shape. The maintainable direction is a generic delivery policy plus prompt hint, but maintainers should choose how it unifies suppress, caption, separate, and preserve-text requests before implementation.
