## Summary

Keep open. Current main and v2026.5.12 still lack a shared auto-TTS text-delivery policy and speech-first prompt guidance; related open issues cover adjacent Telegram, Feishu, and provider-hint work but do not fully supersede this combined request.

## What This Changes

Request a voice-only or caption-only auto-TTS delivery mode plus stronger system-prompt guidance for natural spoken output.

## Best Possible Solution

Define a backward-compatible shared auto-TTS text-delivery policy and concise speech-first prompt hint, then document channel fallback semantics and keep existing text fallback when synthesis or native voice delivery fails.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source and latest-release inspection give high confidence that the requested delivery policy and speech-first prompt guidance are absent today.

## Solution Assessment

Unclear for the exact proposed option shape. The maintainable direction is a generic delivery policy plus prompt hint, but maintainers should choose how suppress, caption, separate, preserve-text, and failure fallback modes compose across channels.
