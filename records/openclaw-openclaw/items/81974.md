Full review comments:

- **[P1] Validate secrets policy rule shapes:** `extensions/policy/src/doctor/register.ts:1209`
  - body: The new secrets checks read `secrets.denySources` with `readStringList` and the boolean rules with `readPolicyBoolean`, but the policy shape pass never validates the new `secrets` block. Malformed values such as `denySources: "exec"` or `allowInsecureProviders: "false"` can be ignored and produce a clean result, so add fail-closed `policy-jsonc-invalid` validation and regressions for these rules.
  - confidence: 0.9
- **[P2] Reject blank auth metadata entries:** `extensions/policy/src/doctor/register.ts:902-908`
  - body: `auth.profiles.requireMetadata` accepts blank strings because the unsupported-key check is skipped when `entry.trim() === ""`; the effective requirement set later filters the blank out. This silently weakens the authored policy, so match the tool metadata validation and reject blank entries.
  - confidence: 0.88
- **[P2] Validate auth mode allowlists:** `extensions/policy/src/doctor/register.ts:1296`
  - body: `auth.profiles.allowModes` is read through `readStringList` without any shape validation, so a malformed allowlist can disable auth-mode enforcement instead of reporting invalid policy. Add the same fail-closed validation used for other policy allow/deny arrays.
  - confidence: 0.86
