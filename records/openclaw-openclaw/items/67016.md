## Summary

Keep open. Current main and v2026.5.18 still expose only partial Dreams status: next-run timing and memory counts are visible, but the requested last-run timestamp, run result, processed trace count, and diary write/workspace summary are not surfaced in the Dreams UI.

## What This Changes

The issue asks for the Dreams UI to show run metadata such as last run time, item counts, diary writes, outcome, and next scheduled cycle.

## Best Possible Solution

Add a memory-core-owned Dreams run summary, expose it through the Dreams status API, normalize it in the Control UI controller, render it in the Dreams tab, and document how multi-workspace partial success is summarized.

## Reproduction Assessment

Yes. Source inspection on current main shows the Dreams status path carries counts, entries, and next-run phase data, but not the requested last-run, outcome, processed-count, or diary-write summary fields.

## Solution Assessment

Yes directionally, but the maintainable solution should be a memory-core-owned run summary rather than deriving everything from raw cron state or grepping logs.
