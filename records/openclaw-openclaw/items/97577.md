## Best Possible Solution

Fix the detector to aggregate stable no-progress outcomes per tool/args pattern across the scoped session tail, keep changing outcomes as progress, add detector and before-tool-call proof, and land it against the linked canonical issue.

## Reproduction Assessment

Yes, source-level. Current main computes the global breaker from the current tool/args no-progress streak, and the PR diff shows the proposed replacement still resets when adjacent completed calls have different result hashes.

## Solution Assessment

No. The detector is the right layer, but the best fix should count stable no-progress per tool/args pattern across the session tail instead of requiring every consecutive call to share one result hash.
