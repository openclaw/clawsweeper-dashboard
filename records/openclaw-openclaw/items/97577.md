## What This Changes

The branch adds a session-wide no-progress streak helper, rewires `globalCircuitBreakerThreshold` to use it, and adds detector tests for cross-tool same-result blocking.

## Best Possible Solution

Fix the detector to aggregate stable no-progress outcomes per tool/args pattern across the run-scoped session tail, with focused detector coverage and redacted runtime or before-tool-call proof.

## Reproduction Assessment

Yes, source-level. Current main uses a per-tool/args no-progress streak for the global breaker, and the PR helper still breaks on different adjacent result hashes; no live Gateway replay was run in this read-only review.

## Solution Assessment

No. The detector is the right layer, but the proposed aggregation is too narrow because it requires every adjacent call to share one result hash instead of counting stable per-pattern no-progress outcomes across the session tail.
