The PR changes `globalCircuitBreakerThreshold` to use a new session-wide result-hash streak helper and adds detector tests for same-outcome cross-tool streaks.

## Best Possible Solution

Fix the detector to aggregate stable no-progress per tool/args pattern across the scoped session tail, keep changing outcomes as progress, add focused detector/runtime-boundary proof, and then land it against https://github.com/openclaw/openclaw/issues/79252.

## Solution Assessment

No. The detector is the right layer, but the best fix should count stable no-progress per tool/args pattern across the session tail instead of requiring every consecutive call to share one result hash.
