## Summary

Keep open. The underlying subagent exec-heartbeat bug is source-reproducible on current main, but this PR head still cannot merge because the new test references a removed heartbeat mock, its enqueue assertion does not match the current payload shape, the cron-run suppression still needs narrower proof, and the external PR has no after-fix real behavior evidence.

## What This Changes

The PR guards exec-event heartbeat requests for subagent and cron-run session keys in the bash exec runtime, adds one regression test, and adds a changelog entry.

## Best Possible Solution

Land a corrected producer-side guard for subagent session keys with passing regression coverage, and handle cron-owned exec wakes only with cron-specific proof or a narrower follow-up.

## Reproduction Assessment

Yes, source-reproducible. Current main still requests exec-event heartbeats from both exec completion call sites, and forced subagent session keys resolve to the main session; I did not run a live WebChat/subagent scenario in this read-only review.

## Solution Assessment

No for the current patch. A producer-side subagent guard is the narrow maintainable direction, but this PR must use the current heartbeat mock, assert the current system-event payload, and either prove or narrow the cron-run behavior before merge.
