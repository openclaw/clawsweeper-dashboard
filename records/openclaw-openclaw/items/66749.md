## Summary

Keep open. The source-level subagent bug is valid, but this patch still has blocking issues: the cron-run guard conflicts with current main's cron delivery routing, the new assertion does not match the queued event payload, and the contributor's latest evidence does not show after-fix behavior in a patched setup.

## What This Changes

The PR guards exec-event heartbeat scheduling for subagent and cron-run session keys in the bash exec runtime, adds a subagent regression test, and adds a changelog entry.

## Best Possible Solution

Rebase the PR onto current main, keep the subagent producer-side skip, handle cron-run sessions through the existing delivery-mode/remap contract or a separately reviewed cron design, fix the regression assertion, and add redacted after-fix runtime proof.

## Reproduction Assessment

Yes, at source level. Current main still requests exec-event heartbeats for subagent session keys, and heartbeat resolution routes forced subagent keys back to the main session; I did not run a live patched WebChat/subagent setup in this read-only review.

## Solution Assessment

No for the current patch. The subagent guard is the narrow maintainable direction, but the cron-run guard bypasses current main's delivery-mode routing and the new test does not match the actual event payload.
