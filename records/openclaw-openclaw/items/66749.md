## Summary

Keep open. The source-level subagent bug is real and the producer-side guard is the right direction, but the latest PR head still has a blocking stale mock reference, a mismatched system-event assertion, an unsafe cron-run suppression under current heartbeat resolution, and no after-fix real behavior proof.

## What This Changes

The PR guards exec completion heartbeat wakes for subagent and cron-run session keys, adds an exec-system-event regression test, and adds a changelog entry.

## Best Possible Solution

Land a corrected producer-side subagent guard, either keep cron-run wakes intact or add a cron-specific resolver/test proving the new behavior, fix the regression test, and require redacted real behavior proof before merge.

## Reproduction Assessment

Yes, source-reproducible. Current main unconditionally requests exec-event heartbeats from the exec completion producers, and forced subagent keys resolve to the main session; I did not run a live WebChat/subagent scenario in this read-only review.

## Solution Assessment

No for the current PR head. The subagent guard is a narrow maintainable fix, but the cron-run guard is not supported by current heartbeat resolution, the added test is broken, and real behavior proof is missing.
