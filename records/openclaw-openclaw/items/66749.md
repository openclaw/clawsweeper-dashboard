## Summary

Keep open. Current main still requests exec-event heartbeat wakes for background exec completions, while the latest PR diff narrowly suppresses those wakes for subagent and cron-run session keys without dropping the system event itself.

## What This Changes

The PR guards exec-event heartbeat wakes in `maybeNotifyOnExit` and `emitExecSystemEvent` for subagent and cron-run session keys, adds subagent regression coverage, and records the fix in the changelog.

## Best Possible Solution

Land the narrow producer-side guard after normal PR CI and review, and keep broader event-audience/transcript leakage design work in #69492.

## Reproduction Assessment

Yes. On current main, calling `emitExecSystemEvent` or completing a background exec with `sessionKey: "agent:main:subagent:abc-123"` enqueues the event and requests an exec-event heartbeat; heartbeat resolution maps forced subagent keys to the main session.

## Solution Assessment

Yes. Guarding the two heartbeat producer call sites with existing session-key predicates is the smallest maintainable fix and preserves exec system-event storage for the owning session.
