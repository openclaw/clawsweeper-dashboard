## Summary

Keep open: the subagent-side producer guard is a plausible narrow fix, but the latest PR also suppresses cron-run exec wakes in a way current heartbeat routing does not support, its new test is still broken, and the external PR has no after-fix real behavior proof.

## What This Changes

The PR guards exec-event heartbeat requests for subagent and cron-run session keys in the bash exec runtime, adds one emitExecSystemEvent regression test, and adds a changelog entry.

## Best Possible Solution

Land a corrected producer-side guard for subagent session keys, keep cron-run wake handling in a cron-specific routing/remap path, add passing regression coverage, and leave broader event-audience work to https://github.com/openclaw/openclaw/issues/69492.

## Reproduction Assessment

Yes, from source inspection. Current main unconditionally requests an exec-event heartbeat after enqueueing exec system events, and forced subagent session keys resolve back to the main session; I did not run a live WebChat/subagent scenario in this read-only review.

## Solution Assessment

No for the current patch as submitted. The subagent guard is the narrow maintainable direction, but the cron-run guard is not supported by current heartbeat resolution and the added test still fails against the current mock and payload shape.

## Review Findings

Overall correctness: patch is incorrect

  - body: The new guard also suppresses cron-run session keys, but current `resolveHeartbeatSession` only remaps subagent keys; cron-run keys pass through as forced session keys and heartbeat preflight peeks that queue.   - confidence: 0.88

  - confidence: 0.97
