## What This Changes

Request to enable native server-side web search for GitHub Copilot GPT models that use the OpenAI Responses API, without requiring a separate managed search provider.

## Best Possible Solution

Land a provider-owned Copilot native-search path that preserves existing `web_search` allow/deny policy gates, has focused tests/docs, and makes the hosted-search default an explicit maintainer-approved behavior.

## Reproduction Assessment

Partly: source inspection gives a high-confidence way to reproduce the current-main gap because the Copilot Responses wrapper never injects native `web_search`. The external Copilot API support itself was not freshly live-tested here and rests on the linked PR's copied live output.

## Solution Assessment

Unclear as a product default. A provider-local wrapper fits the OpenClaw owner boundary, but the final solution needs policy-gate preservation and maintainer acceptance of the default hosted-search routing.
