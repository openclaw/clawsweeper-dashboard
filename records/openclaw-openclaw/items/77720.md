## Summary

Keep open: current main still has individual lost-context recovery and explicit cascade-kill controls, but I did not find a path that automatically settles active descendants when an intermediate parent subagent dies abruptly.

## What This Changes

The issue reports that abrupt death of a parent subagent leaves depth-2 child sessions/tasks stuck as stale running work with no termination or orphan cleanup signal.

## Best Possible Solution

Add one maintainer-approved lifecycle reconciliation path that settles active descendants and task state when a parent subagent dies, reusing existing cascade and maintenance primitives instead of adding a separate stale-task workaround.

## Reproduction Assessment

Yes, at source level: current main has per-run lost-context cleanup and explicit cascade kill, but no inspected path that triggers descendant termination or orphan marking when an intermediate parent dies. I did not perform a live gateway kill/OOM reproduction.

## Solution Assessment

No current implementation is sufficient: the narrowest maintainable direction is to connect parent-death detection to existing descendant and task reconciliation primitives after maintainers choose terminate-vs-orphan semantics.
