## What This Changes

The issue requests lifecycle cleanup so depth-2 subagent sessions are terminated or automatically orphan-reconciled when their depth-1 parent dies mid-run.

## Best Possible Solution

Implement one canonical lifecycle reconciliation path that, when a parent subagent becomes terminal or lost unexpectedly, settles or cascades active descendants and updates subagent task and TaskFlow state so audit no longer reports stale_running indefinitely.
