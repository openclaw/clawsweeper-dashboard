## Summary

Keep open: current main has partial lifecycle safeguards, but I did not find a source path that automatically settles or terminates active descendants when a parent subagent dies abruptly; related open P1 lifecycle reports show this stale-running family is still active.

## What This Changes

The issue reports that abrupt death of a depth-1 subagent can leave depth-2 child sessions stale-running with no termination signal or automatic cleanup.

## Best Possible Solution

Implement one canonical lifecycle reconciliation path that, when a parent subagent becomes terminal or lost unexpectedly, settles or marks active descendants and synchronizes subagent task and TaskFlow state without adding a new config knob.

## Reproduction Assessment

Yes at source level, but not by live run: the report has concrete production evidence, and current main still shows separate per-run sweeper, restart orphan recovery, explicit cascade kill, and task-maintenance cleanup paths rather than a proven parent-death descendant reconciliation path.

## Solution Assessment

No implemented-on-main close is justified; the narrow maintainable solution is a bounded lifecycle/task reconciliation rule in the existing subagent and task registries, coordinated with adjacent open lifecycle reports.
