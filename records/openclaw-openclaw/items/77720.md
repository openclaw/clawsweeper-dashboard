## Summary

Keep open: current main has explicit stop/kill cascade paths and restart-aborted orphan recovery, but the inspected lifecycle and task-maintenance paths still do not show a general parent-death hook that recursively terminates or marks depth-2 descendants when a depth-1 subagent dies outside those explicit paths.

## What This Changes

The issue requests lifecycle cleanup for subagent descendants when an intermediate parent subagent dies abruptly.

## Best Possible Solution

Add narrow lifecycle reconciliation so descendants of a terminal or lost subagent parent are recursively cancelled, resumed, or marked lost, with depth-2 regression coverage.

## Reproduction Assessment

Yes, from source inspection and the production `stale_running` evidence rather than a live run: explicit stop/kill paths cascade, while ordinary parent terminal/lost handling does not connect a dead depth-1 parent to still-active depth-2 descendant records.

## Solution Assessment

No, current main is only a partial solution because it covers explicit control paths and marker-based restart recovery. The narrow maintainable fix is to hook parent terminal/lost reconciliation into existing subagent registry/control cleanup without implementing the broader chain-tracking request.
