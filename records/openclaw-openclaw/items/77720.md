## Summary

Keep open: current main covers explicit cascade kill and individual lost-run cleanup, but I did not find automatic descendant termination or orphan marking when an intermediate parent subagent dies. ## What This Changes

The issue requests reliable cleanup or orphan handling for descendant subagent sessions when an intermediate parent subagent dies abruptly.

## Best Possible Solution

Add one maintainer-approved lifecycle reconciliation path that settles active descendants and task state when a parent subagent dies, reusing existing cascade kill, orphan recovery, and task maintenance primitives instead of adding another ad hoc stale-task workaround.

## Reproduction Assessment

Yes at source level: the issue gives concrete depth-2 steps and production stale-running task evidence, and current source shows the recovery pieces do not automatically mark or settle descendants when the parent subagent is only lost. I did not perform a live gateway kill/OOM reproduction.

No current implementation is sufficient. The narrow maintainable solution is to connect parent-death detection to existing descendant kill/orphan and task reconciliation primitives after maintainers choose terminate-vs-orphan/resume semantics.
