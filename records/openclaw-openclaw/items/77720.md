## Summary

Keep open: current main has useful stale-run and restart-orphan recovery, but it still does not reconcile descendants from the parent-terminal path when an intermediate subagent dies; the related stale-task PR addresses delayed task maintenance, not immediate parent-death lifecycle propagation.

## What This Changes

The issue reports depth-2 subagent sessions remaining stale-running when their depth-1 parent dies unexpectedly.

## Best Possible Solution

Add focused subagent lifecycle reconciliation so descendants of a terminal or lost parent are recursively cancelled, marked lost/orphaned, or otherwise made cleanup-eligible without implementing the broader session-chain feature.

## Reproduction Assessment

Yes, source-level: explicit kill paths cascade, but ordinary parent-loss handling does not traverse descendants, and task maintenance can still treat a stale non-wedged child session row as backing state. I did not run a live SIGUSR1/OOM gateway repro in this read-only checkout.

## Solution Assessment

No current implementation is not enough: stale-run sweeping and orphan recovery reduce symptoms, but the narrow maintainable fix is parent-terminal descendant reconciliation with depth-2 regression coverage. The open stale-task PR is complementary cleanup, not the core lifecycle fix.
