## Summary

Keep open: current main still lacks a parent-terminal lifecycle path that recursively terminates or marks depth-2 descendants when an intermediate subagent dies, and the related open stale-task PR only covers delayed task maintenance cleanup rather than immediate descendant lifecycle propagation.

## What This Changes

The issue requests lifecycle cleanup for child subagent sessions when their parent subagent dies mid-run.

## Best Possible Solution

Add focused subagent lifecycle reconciliation so descendants of a terminal or lost parent are recursively cancelled, marked lost/orphaned, or otherwise made eligible for cleanup without implementing the broader session-chain feature.

## Reproduction Assessment

Yes, source-level. Explicit stop/kill paths cascade, but ordinary parent death is not wired into descendant lifecycle cleanup, and task maintenance can still treat a stale child session row as backing state.

## Solution Assessment

No. Current main and the open stale-task PR reduce some stale-running symptoms, but the narrow maintainable fix is parent-death descendant reconciliation with regression coverage for depth-2 sessions.
