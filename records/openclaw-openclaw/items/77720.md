## Summary

Keep open: current main has partial stale-run recovery and task-maintenance cleanup, but I did not find a parent-death path that automatically terminates or marks all active descendant subagents when an intermediate parent dies abruptly.

## What This Changes

The issue asks OpenClaw to settle or terminate descendant subagent sessions when an intermediate parent session dies abruptly so task/session state does not remain stale-running.

## Best Possible Solution

Add one canonical parent-death lifecycle reconciliation path that settles active descendants and synchronizes subagent task and TaskFlow state, reusing existing subagent registry and task-maintenance primitives.

## Reproduction Assessment

Source-reproducible, but not live-reproduced: the report gives concrete production examples and current source shows no parent-death descendant cascade, but this review did not kill a real depth-1 parent in a running setup.

## Solution Assessment

No implemented-on-main close is justified; the maintainable fix should be a bounded lifecycle reconciliation rule in the existing subagent/task registries rather than a separate ad hoc cleanup path.
