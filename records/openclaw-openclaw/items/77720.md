## Summary

Keep open: current main has cascade handling for explicit stop/kill paths and restart-aborted recovery, but source inspection still shows no general parent-terminal lifecycle hook that recursively terminates or marks depth-2 descendants when a depth-1 subagent dies outside those paths.

## What This Changes

The issue requests reliable cleanup or orphan marking for nested subagent sessions when their parent subagent dies mid-run.

## Best Possible Solution

Add narrow lifecycle reconciliation so descendants of a terminal or lost subagent parent are recursively cancelled, resumed, or marked lost without implementing the broader chain-tracking feature request.

## Reproduction Assessment

Yes, from source inspection rather than a live run: explicit stop/kill paths cascade, but ordinary parent terminal handling and task reconciliation do not connect a dead parent to still-present descendant session rows. The issue also includes concrete production `stale_running` task evidence.

## Solution Assessment

No, current main is not yet the best solution because it only covers explicit stop/kill and marker-based restart recovery. A narrower maintainable fix is to hook parent subagent terminal/lost reconciliation into the existing registry/control paths and add regression coverage for depth-2 descendants.
