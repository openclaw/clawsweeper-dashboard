## Summary

Keep open. Current main still has the restart/offline-edit invalidation gap for existing persisted skills snapshots: disabled skills are filtered correctly only when a snapshot is rebuilt, while the reuse predicate still accepts a persisted `version: 0` snapshot when the restarted process also reports version `0`; draft #71497 targets the same root cause but is unmerged.

## What This Changes

The issue asks OpenClaw to make `skills.entries.<skill>.enabled: false` invalidate stale persisted skills snapshots for existing long-lived sessions, including offline config edits followed by gateway restart.

## Best Possible Solution

Add restart-stable snapshot invalidation based on the effective inputs that make a skills snapshot valid, ideally a deterministic fingerprint over effective skills config, agent skill filter, and resolved skill roots or entries. If maintainers choose the narrower #71497 version-seeding repair, it should include focused coverage proving an already-snapshotted skill is removed after an offline `enabled:false` config edit and restart.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is an existing session with `skillsSnapshot.version: 0`, an offline `skills.entries.<name>.enabled=false` edit, then gateway restart and another turn on the same session; current main computes version `0` again and reuses the stale snapshot.

## Solution Assessment

Yes for the issue's durable inputs-fingerprint direction, because it directly models snapshot validity across restarts and offline edits. The narrower #71497 version-seeding approach may be acceptable if it proves this disabled-skill path and maintainers accept the smaller scope.
