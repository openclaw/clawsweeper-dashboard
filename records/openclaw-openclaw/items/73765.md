## What This Changes

Refresh existing session skills snapshots when a skill is disabled in config while OpenClaw was offline or restarted, so disabled skills leave the model's available-skills prompt without requiring /new.

## Best Possible Solution

Keep the remaining code, regression coverage, and changelog work in https://github.com/openclaw/openclaw/pull/71497, or an equivalent shared durable snapshot-freshness fix, with this report's disabled-skill reproduction preserved as context.
