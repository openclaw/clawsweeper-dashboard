## Summary

Keep open. Current main still has the offline-edit-after-restart stale snapshot gap: the filter rejects disabled skills when rebuilding, but persisted version-0 snapshots can be reused when the process-local skills version is also 0.

## What This Changes

Existing long-lived sessions should rebuild cached skill snapshots so `skills.entries.<name>.enabled:false` removes disabled skills after offline config edits and restarts.

## Best Possible Solution

Add restart-stable snapshot validity based on the effective skills config and skill roots, with regression coverage for disabled skills being removed from a persisted session after restart.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a persisted session with `skillsSnapshot.version: 0`, an offline `skills.entries.<name>.enabled:false` edit, process restart, and another turn while `getSkillsSnapshotVersion()` is again 0.

## Solution Assessment

Yes. Fixing shared snapshot freshness is the right boundary because `shouldIncludeSkill` already rejects `enabled:false`; changing the filter would not address stale persisted prompts.
