## Summary

Keep open. Current main still lacks restart-stable invalidation for offline `skills.entries.*.enabled=false` edits: persisted snapshots store only a numeric version, the refresh state still starts each process at `0`, and the reuse predicates do not compare the effective skills config or skill-root inputs.

## What This Changes

The issue asks OpenClaw to refresh existing session skills snapshots after an offline skills config change so disabled skills disappear without requiring `/new`.

## Best Possible Solution

Add restart-stable snapshot invalidation based on the effective inputs that make a skills snapshot valid, with regression coverage for an offline `enabled:false` edit removing an already-snapshotted skill after restart.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a persisted session with `skillsSnapshot.version: 0`, an offline `skills.entries.<name>.enabled=false` edit, a gateway restart, and another turn on the same session while `getSkillsSnapshotVersion()` is again `0`.

## Solution Assessment

Yes for the issue's durable inputs-fingerprint direction: comparing a stable skills config and skill-root fingerprint directly models snapshot validity across restarts. A narrower version-seeding patch is only sufficient if it explicitly covers this disabled-skill case in both agent-command and auto-reply paths.
