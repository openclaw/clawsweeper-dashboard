## Summary

Keep open. Current main still has the offline-edit-after-restart stale skills snapshot gap, and the open maintainer draft PR that targets this shared path has not merged.

## What This Changes

Existing long-lived sessions should stop advertising skills disabled via `skills.entries.*.enabled:false` after offline config edits and gateway or CLI restarts.

## Best Possible Solution

Land restart-stable skills snapshot invalidation in the shared refresh-state/snapshot-validity path, with regression coverage proving a persisted `version:0` session drops a skill disabled while OpenClaw was offline.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a persisted existing session with `skillsSnapshot.version:0`, an offline `skills.entries.<name>.enabled:false` edit, process restart, and another turn where `getSkillsSnapshotVersion()` is again `0`.

## Solution Assessment

Yes on the shared invalidation direction. A durable input fingerprint would be the most precise fix, while the open version-seeding PR is a narrow acceptable path if it explicitly covers disabled skills after restart.
