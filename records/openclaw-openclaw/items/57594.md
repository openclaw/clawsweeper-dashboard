## What This Changes

The issue reports that Zalo proactive CLI sends reject valid non-numeric Bot API chat_id values before delivery.

## Best Possible Solution

Review and land the open Zalo-local fix PR if maintainers accept its opaque chat_id behavior, then close this issue after merge provenance exists.

## Reproduction Assessment

Yes. Source inspection on current main shows Zalo direct target recognition uses `isNumericTargetId`, whose three-or-more-digits regex rejects the reported hex-like chat_id values before the resolver reaches the direct-send path.

## Solution Assessment

Yes. A Zalo-local target predicate is the right owner boundary because Zalo owns its upstream chat_id grammar and the shared numeric helper should remain available for channels that really distinguish numeric IDs from handles.
