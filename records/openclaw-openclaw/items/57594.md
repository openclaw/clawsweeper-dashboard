Yes. Source inspection shows a nonnumeric Zalo target reaches the Zalo resolver with `looksLikeId` still set to `isNumericTargetId`, which rejects non-digit chat_id values before send resolution.

## Solution Assessment

Yes. A Zalo-local target predicate is the narrowest maintainable fix boundary; shared outbound heuristics should stay unchanged and this issue should remain open until the linked PR lands or is replaced.
