## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/57594. Zalo proactive outbound sends reject valid non-numeric Bot API `chat_id` values because `extensions/zalo/src/channel.ts` still uses the numeric-only `isNumericTargetId` as `targetResolver.looksLikeId`. Keep the fix plugin-local: do not change core target resolution, add config, or edit CHANGELOG.md. Add a Zalo target-id predicate that checks the normalized target value passed by core, or strips `zalo:`/`zl:` prefixes itself, and accepts current numeric IDs plus hex-like non-numeric chat IDs matching the report examples while continuing to reject empty, too-short, or non-hex random strings unless upstream proof supports a broader contract. Add focused tests for bare hex, uppercase hex, whitespace, numeric IDs, and provider-prefixed `zalo:<hex>` / `zl:<hex>` targets, plus a resolution-path assertion that a valid hex target resolves instead of producing Unknown target. Review closed https://github.com/openclaw/openclaw/pull/90859 for useful context, but avoid its raw-only prefix gap.
