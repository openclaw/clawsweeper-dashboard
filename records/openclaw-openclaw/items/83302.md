## Summary

Keep open: current main and the latest release still flatten Telegram forum-topic origin targets in inbound, audio preflight, and mention-skip contexts while preserving thread metadata, so this remains a narrow real Telegram routing bug.

## What This Changes

The issue requests preserving topic-qualified Telegram `OriginatingTo` values for forum-topic inbound/follow-up delivery and adding non-General topic regression coverage.

## Best Possible Solution

Reuse the existing Telegram routing-target helper for inbound origin metadata across normal inbound, audio preflight, and mention-skip hook paths, with focused non-General forum-topic regression tests.

## Reproduction Assessment

Yes, at source level: a Telegram forum-group inbound context with a non-General `message_thread_id` carries `MessageThreadId` while setting `OriginatingTo` to the flat chat target. I did not run a live Telegram repro in this read-only pass.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing `buildTelegramRoutingTarget` contract on affected inbound-context paths instead of adding new config or routing policy.
