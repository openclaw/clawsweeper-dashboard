## Summary

Keep open: current main and v2026.5.12 still flatten Telegram forum-topic origin targets on the normal inbound context, audio preflight context, and mention-skip hook context even when thread metadata is available. Existing helper, docs, and upstream type contracts support preserving the non-General topic in the routing target, and adjacent open Telegram topic PRs do not cover these inbound context builders.

## What This Changes

Preserve topic-qualified Telegram `OriginatingTo` values for inbound forum-topic contexts when thread metadata is known, with focused regression coverage.

## Best Possible Solution

Reuse `buildTelegramRoutingTarget` for normal inbound, audio preflight, and mention-skip hook origin metadata, preserving General-topic behavior and adding non-General forum-topic regressions.

## Reproduction Assessment

Yes, at source level: a current-main Telegram forum inbound context can carry `MessageThreadId` while setting `OriginatingTo` to the flat `telegram:<chatId>` target. I did not run a live Telegram forum repro in this read-only pass.

## Solution Assessment

Yes: the narrow maintainable fix is to reuse the existing Telegram routing-target helper on the affected inbound-context builders rather than adding a new config or routing policy.
