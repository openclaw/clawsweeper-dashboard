- **[P1] Validate permission responses before reporting success:** `extensions/feishu/src/bitable.ts:393-402`
  - body: `permissionMember.create()` can return a nonzero Feishu response without throwing, but this code ignores the returned response and sets `requester_permission_added = true` immediately after `await`.   - confidence: 0.91

  - body: This submits every trusted requester as `member_type: "openid"`, but Feishu inbound parsing can populate the same runtime requester field from a `user_id` fallback when `open_id` is missing. Please preserve or derive the ID kind so those valid deliveries either grant with `userid` or report an accurate skip/error instead of silently missing the requester grant.
  - confidence: 0.87
