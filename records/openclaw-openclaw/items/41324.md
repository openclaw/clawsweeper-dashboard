## Summary

Keep open: current main still lacks the requested Bitable requester grant, but the PR head is not mergeable because it can report failed permission API responses as success, can send a Feishu `user_id` as `openid`, omits the new auto-share path from the Feishu security audit, and has no redacted real Feishu proof.

## What This Changes

This PR adds `grant_to_requester` support to `feishu_bitable_create_app`, reads the trusted Feishu requester context, grants edit access with `drive.permissionMember.create`, and returns requester permission status fields.

## Best Possible Solution

Land a corrected Bitable requester grant that validates Feishu API response codes, handles requester ID type safely, updates security audit and regression coverage, and includes redacted real Feishu proof.

## Reproduction Assessment

Yes for the review findings from source inspection: make `permissionMember.create` resolve with a nonzero `code`, or process a Feishu event where `open_id` is absent and `user_id` is used as the requester sender. The PR path would report success or submit the wrong member type without needing a live Feishu run to see the source-level fault.

## Solution Assessment

No. Mirroring `feishu_doc` is the right direction, but the current diff needs response validation, ID-type-safe requester handling, audit/test updates, and real behavior proof before it is the narrow maintainable fix.

  - body: `permissionMember.create()` resolves to a Feishu response with `code` and `msg`, and existing permission code treats nonzero `code` as failure. This branch sets `requester_permission_added` after any resolved response, so missing scopes or invalid members can be reported as a successful grant.
  - confidence: 0.91
- **[P2] Match requester member_type to the ID format:** `extensions/feishu/src/bitable.ts:397-398`

  - confidence: 0.86

  - body: Adding `grant_to_requester` creates a new automatic Bitable sharing path, but the Feishu security audit still warns only for doc creation grants. Operators relying on that audit would miss the new auto-share surface.
  - confidence: 0.78
