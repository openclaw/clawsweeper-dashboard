## Summary

Keep open: this PR addresses a plausible current-main compaction bug, but it should not merge yet because the estimator undercounts mixed latest-usage plus post-usage content and the external PR only provides mock/test proof.

## What This Changes

The branch changes preflight compaction's stale transcript token estimator to prefer usage, message, and trailing-byte estimates over whole transcript byte size, and adds a focused regression test.

## Best Possible Solution

Revise the estimator to exclude pre-usage transcript metadata while adding post-usage active content to the latest usage snapshot, then add changelog coverage and real runtime proof before merge.

## Reproduction Assessment

Yes for the code-level regression: current main can source-reproduce metadata byte pressure through the stale transcript fallback. A live MiniMax/Feishu/memory-lancedb-pro reproduction was not established.

## Solution Assessment

No. The direction is narrow and likely right, but this implementation undercounts combined usage plus trailing content and still needs the required changelog and real behavior proof.
