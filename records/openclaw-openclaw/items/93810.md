## Summary

Keep this PR open: it is a focused bug fix, but the runtime change is too broad because it disables startup future-slot repair for every job in the pass, not only protected overflow deferrals, and the external-PR real behavior proof is still test/mock-only.

## What This Changes

The branch adds a regression test for startup overflow cron catch-up deferrals and changes `start()` to disable future cron next-run repair during its post-catch-up maintenance pass.

## Best Possible Solution

Scope the exemption so protected overflow deferrals survive while ordinary startup stale-future repair remains active, then add real behavior proof for the startup recovery path.

## Reproduction Assessment

Yes, source-level: current `start()` runs a second maintenance pass after `runMissedJobs`, and the PR changes that pass to disable future-slot repair for all cron jobs. I did not run tests because this is a read-only review.

## Solution Assessment

No. Preserving overflow deferrals is the right goal, but disabling startup future-slot repair globally is broader than the bug requires and weakens the merged stale-future repair path.
