## Summary

The PR is still necessary: current main still has Tavily's `web-search-contract-api` returning a provider whose `createTool` is `null`, while the patch routes that artifact to the executable provider and adds focused regression coverage. ## What This Changes

The branch re-exports Tavily's executable web-search provider from the contract artifact, adds a Tavily regression test for the contract provider's executable `web_search` tool, and makes a small lint cleanup in an E2E fixture helper.

## Best Possible Solution

Land a narrow Tavily-owned fix that keeps the public contract artifact and runtime provider from drifting, with focused regression coverage for the explicit provider path.

## Reproduction Assessment

Yes, this is the clean owner-boundary fix: Tavily already has one executable provider implementation, and the PR makes the contract artifact reuse it instead of maintaining a second metadata-only path. Changing core provider resolution to special-case Tavily would be broader and more brittle.
