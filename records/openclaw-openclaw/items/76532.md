## Summary

Keep open. Current main still uses strict configured model-id equality in `resolveContextWindowInfo`, and #76572 is an open unmerged closing PR, so this issue should stay open until that fix lands or is replaced.

## What This Changes

Make context-window resolution treat bare and provider-scoped model IDs as equivalent under the selected provider so configured `contextTokens` caps are honored.

## Best Possible Solution

Land a narrow provider-aware matcher in `src/agents/context-window-guard.ts`, add focused regression coverage in `src/agents/context-window-guard.test.ts`, and let #76572 close the issue if review passes.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: call `resolveContextWindowInfo` with provider `github-copilot`, model ID `github-copilot/claude-opus-4.7-1m-internal`, a bare configured model entry, and runtime `modelContextWindow: 128000`; the strict matcher misses the configured cap.

## Solution Assessment

Yes. Provider-aware bare/scoped ID equivalence inside the resolver is the narrow maintainable fix, provided it stays constrained to the selected provider and gets regression coverage.
