## Summary

Keep this open because current main still contains the strict model-id equality described in the report, and an open PR with explicit closing syntax is already proposed for the same bug. The issue should stay open until that PR is reviewed and merged or replaced.

## What This Changes

The issue reports that configured per-model context caps are ignored when a provider-scoped runtime model id is compared against a bare configured model id.

## Best Possible Solution

Review and land PR #76572, or replace it with an equivalent provider-aware matcher plus focused regression coverage in the context-window guard.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: pass provider `github-copilot`, runtime model id `github-copilot/<id>`, and a bare configured model id into `resolveContextWindowInfo`; the current strict matcher misses the configured `contextTokens` and falls back.

## Solution Assessment

Yes. The narrow maintainable fix is provider-aware bare/scoped id equivalence in `resolveContextWindowInfo`, constrained to the selected provider and covered by a regression test.
