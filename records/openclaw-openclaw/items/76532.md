## Summary

Keep open as a valid narrow bug. Current main still uses strict configured-model id equality in the context-window guard, and the linked implementation PR #76572 is open and unmerged, so this issue should remain open until that PR lands or is replaced.

## What This Changes

Make context-window resolution treat bare and provider-scoped model ids as equivalent for the selected provider so configured `contextTokens` overrides are not dropped.

## Best Possible Solution

Land a narrow provider-aware matcher plus regression coverage in the context-window guard, then let the linked implementation PR close this issue after merge.

## Reproduction Assessment

Yes. Source inspection reproduces the failure path: a provider-scoped `modelId` reaches `resolveContextWindowInfo`, the configured bare id misses the strict lookup, and the runtime fallback value is persisted as the session context cap.

## Solution Assessment

Yes. The narrow maintainable fix is provider-aware bare/scoped id equivalence inside the selected provider's configured model lookup, with focused tests for both mismatch directions.
