## Summary

Keep open as a valid narrow bug: current main still uses strict model-id equality in the context-window guard, and the linked fix PR is open rather than merged.

## What This Changes

Treat configured bare model ids and provider-scoped runtime model ids as equivalent in the context-window guard so explicit contextTokens are preserved.

## Best Possible Solution

Land or revise #76572 so resolveContextWindowInfo matches bare and provider-scoped ids only within the selected provider, with focused regression tests for both directions.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: call resolveContextWindowInfo with provider github-copilot, a provider-scoped modelId, a bare configured model row with contextTokens, and a 128000 runtime contextWindow; the current strict match misses the configured row.

## Solution Assessment

Yes. The narrow maintainable fix is provider-aware bare/scoped id equivalence in resolveContextWindowInfo, constrained to the selected provider and covered by a regression test.
