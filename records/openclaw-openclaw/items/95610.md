## Summary

Keep open: current main and v2026.6.10 still mutate the `message` tool description for `message_tool_only` turns, so the central OpenAI prefix-cache churn report is not implemented or obsolete. The system-prompt side has been partly moved below the cache boundary, but the remaining tool-schema churn is enough to keep this as real follow-up work.

## What This Changes

The issue asks OpenClaw to stop turn-scoped `message_tool_only` system-prompt and message-tool-schema changes from defeating OpenAI automatic prompt caching.

## Best Possible Solution

Keep this issue open until maintainers either move the turn-scoped visible-reply directive out of stable system/tool schema bytes for OpenAI paths or prove current main's remaining schema churn no longer affects cache reuse.

## Reproduction Assessment

No high-confidence live current-main reproduction was established in this review. Source inspection does show the reported mode-sensitive `message` tool schema path still exists, and the reporter provided live 2026.6.9 telemetry tying it to cold OpenAI turns.

## Solution Assessment

No, current main is not yet the complete fix. It partially improves prompt placement below the cache boundary, but the `message` tool schema still changes with `message_tool_only`, so the narrow maintainable solution should stabilize tool schemas and put turn-specific delivery guidance in late-bound turn context.
