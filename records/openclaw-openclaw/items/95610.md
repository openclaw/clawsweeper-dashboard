## Summary

Keep open: current main and the latest release still contain the reported `message_tool_only` churn paths in both the message tool schema and system-prompt messaging section, and no related open PR fixes this specific within-TTL OpenAI prefix-stability issue.

## What This Changes

The issue asks OpenClaw to stop per-turn `message_tool_only` delivery guidance from changing OpenAI cached prompt prefixes and tool schemas.

## Best Possible Solution

Preserve `message_tool_only` delivery correctness while making the message tool schema byte-stable and moving per-turn delivery guidance out of OpenAI's cached prefix, with focused tests and live or replayed cache evidence.

## Reproduction Assessment

Yes at source level: current main still varies the message tool description and system-prompt messaging section when `message_tool_only` flips. I did not run a live current-main OpenAI cache probe in this read-only review.

## Solution Assessment

Unclear as a concrete patch: the requested stable-schema and below-prefix direction is right, but maintainers need to choose the durable prompt/delivery contract so visible replies remain reliable.
