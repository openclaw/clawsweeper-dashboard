## Summary

Keep open: current main and v2026.6.9 still contain the reported OpenAI/Codex prompt-cache churn surfaces. Source and tests show `message_tool_only` turns mutate the `message` tool description, and OpenAI/Codex transports strip the internal cache boundary before sending stable plus dynamic system-prompt text as one request instruction/system payload.

## What This Changes

The issue requests stable OpenAI/Codex prompt prefixes by removing per-turn message-tool schema text and keeping volatile runtime/system context out of OpenAI's automatic-cache prefix.

## Best Possible Solution

Keep the `message` tool schema byte-stable, move per-turn visible-reply/runtime guidance into non-prefix turn context for OpenAI/Codex paths, and add focused cache-stability tests plus live OpenAI cache proof.

## Solution Assessment
