## What This Changes

The issue reports that `sessions.list` can become O(rows) slow under concurrent agent/cron activity because per-row plugin metadata lookup reads a globally mutated active plugin-registry workspace directory.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/90819, or an equivalent request-scoped workspace pin with regression coverage, then close this issue after merge proof on main.

## Reproduction Assessment

## Solution Assessment

Yes: a request-scoped workspace pin is the narrowest maintainable fix because increasing cache size does not help changing keys, and removing the row-loop yield would trade this bug for gateway responsiveness risk.
