## Summary

Keep open: current main and the latest release still lack the request-scoped workspace pin, and the same reporter has an open paired PR targeting this exact regression with supplied proof. The issue should remain the tracker until that PR, or an equivalent fix, lands.

## What This Changes

Stabilize plugin workspace metadata lookup during `sessions.list` row building so concurrent agent or cron registry mutations do not cause per-row metadata scans.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/90819, or an equivalent request-scoped workspace pin with regression coverage, then close this issue after merge proof.

## Reproduction Assessment

Yes for source-level reproduction: current main still has the async `sessions.list` yield window, global workspaceDir lookup, and workspaceDir-keyed metadata memo. I did not run the live multi-agent gateway workload in this read-only pass.

## Solution Assessment

Yes: a request-scoped workspace pin at the session-list row batch/workspace reader boundary is the narrow maintainable fix. Increasing cache size would not fix changing keys, and removing the row-yield would regress gateway responsiveness.
