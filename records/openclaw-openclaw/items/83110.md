## Summary

Keep open: current source still contains the half-timeout pre-execution formula and the issue is paired with the same-author open fix PR plus an open configurable-watchdog PR. The exact silent-tool diagnosis is not proven from source because execution milestones clear the pre-execution watchdog, but the broader no-execution-milestone watchdog path is real enough that this should be resolved with the linked PRs rather than auto-closed.

## What This Changes

Issue reports that cron's pre-execution watchdog can abort silent long-running shell-tool cron jobs before the configured job timeout.

## Best Possible Solution

Keep the early-stall watchdog split, but make slow healthy cron startup and missing execution milestones explicit through the linked watchdog/config work plus regression coverage for the real no-output path.

## Reproduction Assessment

Partly. Source shows a no-execution-milestone pre-execution abort can happen before the full job timeout, but I did not establish that a silent tool after process/tool execution starts remains under this watchdog because those milestones clear it.

## Solution Assessment

Unclear. Removing the /2 divisor alone is not the best complete fix on current main because the 60s cap still dominates, and a safer solution should prove whether the missing piece is a cap/config change or a missing execution milestone.
