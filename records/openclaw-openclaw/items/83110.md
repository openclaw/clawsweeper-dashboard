## What This Changes

The issue asks to stop computing the isolated-agent pre-execution watchdog from half of `timeoutSeconds` for silent long-running tool executions.

## Best Possible Solution

Resolve the paired formula and watchdog-config PRs together: either close the formula change with evidence that execution milestones already cover silent runs, or land a tested policy/config change for any remaining pre-execution startup gap.

## Reproduction Assessment

No high-confidence failing reproduction on current main. Source and release-tag inspection show `process_spawned` and `tool_execution_started` clear the pre-execution watchdog before the silent period, while the remaining plausible failure is a startup/cap timing path rather than the reported silent-tool path.

## Solution Assessment

Unclear. Removing `/2` alone is not the best proven fix because the current cap still dominates some cases and docs intentionally keep pre-execution stalls separate; the paired PRs need maintainer review and regression proof.
