## Summary

Keep open. Current main still contains the half-timeout pre-execution watchdog formula, and this issue is paired with the same author's open implementation PR plus the complementary configurable-watchdog PR, so it needs maintainer review rather than cleanup closure.

## What This Changes

The issue asks cron to stop deriving the pre-execution watchdog ceiling from half of the configured job timeout for long silent executions.

## Best Possible Solution

Resolve this through the linked cron watchdog PRs with regression coverage that preserves fast pre-execution stall detection while proving long healthy CLI or embedded runs are governed by the job timeout.

## Reproduction Assessment

No high-confidence current-main reproduction for the exact silent-tool path. Source confirms the /2 formula exists, but current execution milestones should clear the pre-execution watchdog once tool or process execution starts, so a phase-specific repro log is still needed.

## Solution Assessment

No, not as a one-line formula-only change. The maintainable fix should account for the 60s cap, CLI and embedded phase emission, documented fail-fast behavior, and the open configurable-watchdog work.
