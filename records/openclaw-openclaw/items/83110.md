## Summary

Keep open: current main still contains the half-timeout pre-execution formula, and the same author has an open paired PR proposing the exact formula change. Source inspection also shows the stated silent embedded-tool path should clear the watchdog after `tool_execution_started`, so the remaining decision needs PR review with phase-specific proof rather than closing this issue independently.

## What This Changes

The issue requests changing the isolated cron pre-execution watchdog ceiling from half of `timeoutSeconds` to the full job timeout for long silent command executions.

## Best Possible Solution

Review https://github.com/openclaw/openclaw/pull/83111 with phase-specific runtime evidence, then land a focused watchdog fix or close both items with proof that the failing path is already governed by the job timeout.

## Reproduction Assessment

Unclear. Removing the `/2` divisor is a narrow change, but the current 60s cap and execution-milestone semantics mean the best fix may also need watchdog configurability or a missing phase event rather than formula-only cleanup.
