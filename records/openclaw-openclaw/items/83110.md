## Summary

Keep open: this issue is paired with the same-author companion PR https://github.com/openclaw/openclaw/pull/83111 and overlaps the active watchdog configurability PR https://github.com/openclaw/openclaw/pull/82223, while current source shows the stated silent-tool-after-execution path should clear the pre-execution watchdog once execution milestones fire.

## What This Changes

The issue requests changing the cron isolated-agent pre-execution watchdog so silent tool execution is not bounded by half of the configured job timeout.

## Best Possible Solution

Keep this issue open until the companion watchdog PRs are reviewed; if a current-main failure remains, land one focused fix with regression coverage for the exact phase that leaves the pre-execution watchdog armed.

## Reproduction Assessment

No. Current source and regression coverage show embedded tool, CLI process, model, and other execution milestones clear the pre-execution watchdog, so the stated silent-after-execution path is not reproduced from source alone.

## Solution Assessment

No. Changing the formula to the full job timeout may be part of the discussion, but the safer maintainer path is to resolve the companion PRs with phase-specific proof and tests without weakening early stall detection unnecessarily.
