## Summary

## What This Changes

The issue asks for cron's pre-execution watchdog to stop aborting silent but active isolated agent/tool runs before their configured `timeoutSeconds`.

## Best Possible Solution

Keep this issue open until the linked PR or a canonical watchdog fix lands; the final fix should preserve fast pre-execution stall detection while ensuring active CLI/embedded execution is bounded by the job timeout, with regression coverage for silent long-running commands.

## Reproduction Assessment

No, not for the full silent-tool scenario with high confidence. Source confirms the half-timeout formula and a waiting-for-execution abort path, but current main also clears the watchdog on `tool_execution_started` and `process_spawned`, and I did not run the reported rclone cron job.

## Solution Assessment

No, the one-line formula change proposed in the paired PR is not sufficient on current main because the 60s cap still dominates the reported 600s repro. The safer fix is to identify the missing execution milestone or combine formula/cap changes with focused regression coverage.
