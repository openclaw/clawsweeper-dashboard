## Summary

Close as cannot reproduce: current main and the reported v2026.5.12 release clear the pre-execution watchdog once the run reaches model, process, or tool execution, so the reported silent-tool path should be governed by the job timeout rather than the half-timeout pre-execution formula.

## What This Changes

The issue asks cron to stop deriving the pre-execution watchdog from half of the job timeout so long silent tool commands can use the full configured timeout.

## Best Possible Solution

Keep the existing split: pre-model/pre-process stalls can fail early, while started model/process/tool execution should remain bounded by the job timeout; any remaining rclone failure should be re-filed with phase-specific run output or handled through the active watchdog tuning discussion.

## Reproduction Assessment

No. Source and regression coverage show that a silent tool run after model/process/tool execution starts clears the pre-execution watchdog, so the reported kill-at-half-timeout path does not reproduce from the current or shipped code path.

## Solution Assessment

No. Changing the pre-execution watchdog to the full job timeout would weaken the documented early-stall detector; if a real rclone abort remains, the narrower fix is to identify the last recorded phase and patch the missing execution milestone or watchdog tuning path.
