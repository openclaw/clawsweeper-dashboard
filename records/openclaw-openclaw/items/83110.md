## Summary

Keep open for now: current source review does not support the reported silent-tool half-timeout path, but the same author has an open paired PR for this exact change, so the issue and PR should be resolved together rather than closing only this issue.

## What This Changes

The issue reports that isolated cron's pre-execution watchdog halves the effective timeout for silent long-running tool executions.

## Best Possible Solution

Resolve this with the paired PR review: close both if the current source diagnosis holds, or patch the missing phase emission/classification if logs show the run never reaches an execution milestone.

## Reproduction Assessment

No. Current source, docs, shipped-tag inspection, and regression coverage show silent work after process/tool/model execution starts is governed by the job timeout rather than the pre-execution half-timeout formula.

## Solution Assessment

No for the proposed formula change as-is. Removing the divisor weakens the early-stall watchdog; the narrower maintainable fix would be to patch whichever execution milestone is missing if phase logs prove one is missing.
