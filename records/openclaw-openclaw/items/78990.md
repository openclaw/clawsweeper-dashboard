## Summary

Keep open: current main still defaults Matrix table handling to `code`, but the desired default is a Matrix product/UX choice rather than a pure source-level fix.

## What This Changes

The issue asks Matrix outbound markdown tables to render as bullets or structured HTML instead of fenced code blocks by default.

## Best Possible Solution

Choose the Matrix table default explicitly in the plugin-owned `messaging.defaultMarkdownTableMode` path, then add Matrix config/send/formatter regression coverage and a user-facing changelog entry.

## Reproduction Assessment

Yes. The current default path is source-reproducible: Matrix has no plugin table default, the shared resolver returns `code`, and Matrix send conversion runs before HTML formatting.

## Solution Assessment

Unclear until a maintainer chooses the default. A current-main fix should be plugin-owned; the older unmerged PR's core hard-code is no longer the best shape, and `md.enable("table")` appears redundant with markdown-it's default preset.
