## Summary

Keep open: current main still sends the compaction preflight through embedded harness selection before any CLI runtime alias guard, so a model-scoped `claude-cli` runtime can still be treated as a harness id. An open fixing PR already references this issue, so the issue should stay open until that PR lands or is replaced.

## What This Changes

The issue reports that over-threshold Claude CLI runtime sessions throw `MissingAgentHarnessError("claude-cli")` during compaction preflight instead of continuing to the normal runtime path.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/84878 with a CLI runtime alias bypass in `maybeCompactAgentHarnessSession`, preserving context-engine compaction and focused regression coverage.

## Reproduction Assessment

## Solution Assessment

Yes: the best fix is the narrow CLI-runtime alias bypass before embedded harness selection, while letting the existing context-engine compaction path continue. The open fixing PR is the right review target unless maintainers choose a broader shared runtime-boundary refactor.
