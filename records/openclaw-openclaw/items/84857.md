## Summary

Current main still has the reported CLI-runtime versus embedded-harness mismatch in compaction preflight, and an open PR already references this issue with closing syntax. Keep this issue open until one canonical fix lands.

## What This Changes

The issue reports that compaction preflight treats model-scoped `claude-cli` as an embedded harness and throws before dispatch for over-threshold sessions.

## Best Possible Solution

Review and land one canonical fix, likely https://github.com/openclaw/openclaw/pull/84878 or https://github.com/openclaw/openclaw/pull/84966, then close this issue and supersede the duplicate PR path.

## Reproduction Assessment

Yes at source level. Current main still resolves explicit runtimes through `selectAgentHarness`, and `claude-cli` is a CLI runtime alias rather than a registered harness id, so the reported throw follows from the inspected path.

## Solution Assessment

Yes. The narrow maintainable fix is to make compaction preflight respect CLI runtime aliases and continue through the existing non-harness compaction path instead of treating `claude-cli` as a harness id.
