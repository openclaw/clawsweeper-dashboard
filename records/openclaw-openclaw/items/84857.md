- `impact:message-loss`: Affected inbound channel turns can receive an error reply instead of the model response.

- `impact:auth-provider`: The root cause is runtime routing for a Claude CLI auth/backend policy being treated as an embedded harness id.

## Summary

Keep open. Current main still sends the compaction harness preflight through embedded harness selection before checking CLI runtime aliases, so a model-scoped `claude-cli` runtime can still hit the reported `MissingAgentHarnessError` surface even though normal main dispatch now routes CLI turns through `runCliAgent`.

## What This Changes

The issue asks for CLI-runtime compaction preflight to bypass embedded agent harness lookup for model-scoped `agentRuntime.id: "claude-cli"` sessions.

## Best Possible Solution

Add a narrow CLI-runtime alias guard before embedded harness selection in the compaction preflight, with regression coverage proving `claude-cli` skips harness lookup while real plugin harness compaction still works.

## Reproduction Assessment

Yes for a source-level path: current main still calls embedded harness selection from `maybeCompactAgentHarnessSession` and would throw for a model-scoped `claude-cli` runtime with no matching harness. I did not run a live over-threshold channel turn in this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix is to branch CLI runtime aliases out of the embedded harness compaction preflight before `selectAgentHarness`. The existing normal dispatch and CLI transcript compaction paths should stay intact.

## Repair Work Prompt

Fix the CLI-runtime compaction preflight bug reported in https://github.com/openclaw/openclaw/issues/84857. In current main, `maybeCompactAgentHarnessSession` calls `selectAgentHarness` before checking whether the resolved runtime is a CLI alias such as `claude-cli`, so model-scoped Claude CLI runtime policy can throw `MissingAgentHarnessError` on compaction. Add a bounded guard in the harness/queued compaction path so CLI runtime aliases skip embedded harness lookup or delegate to the existing CLI/context-engine compaction path, while preserving plugin harness compaction and PI behavior. Add regression coverage for a canonical Anthropic model with `agentRuntime.id: "claude-cli"` proving no `MissingAgentHarnessError`, plus a control proving plugin harness compaction still returns its existing unsupported/compact result. Update `CHANGELOG.md` because this is user-visible. Do not change model runtime config semantics, provider selection defaults, or the existing main dispatch `runCliAgent` path.
