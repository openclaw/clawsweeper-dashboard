## Summary

Keep open: current main still routes compaction preflight through embedded harness selection before recognizing CLI runtime aliases, and the linked fixing PR is open rather than merged.

## What This Changes

Fix compaction preflight so model-scoped `claude-cli` runtime sessions over threshold do not pass `claude-cli` to embedded harness selection and fail before dispatch.

## Best Possible Solution

Land a focused CLI-runtime alias bypass in `maybeCompactAgentHarnessSession` that lets queued context-engine compaction continue, with regression coverage for model-scoped `claude-cli` runtime policy.

## Reproduction Assessment

Yes at source level: current main resolves model-scoped `claude-cli` runtime policy and then calls `selectAgentHarness`, which throws for unmatched explicit runtimes. A user also reported the same symptom live on OpenClaw 2026.5.18 Telegram and cron sessions.

## Solution Assessment

Yes: the narrow maintainable fix is to make compaction preflight respect existing CLI runtime alias semantics and continue into the non-harness compaction path. The linked open PR implements that shape, but the issue should stay open until it merges or is replaced.
