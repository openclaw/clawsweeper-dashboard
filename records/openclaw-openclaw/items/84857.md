## Label Justifications

- `P1`: The issue describes an active regression that blocks channel turns for Claude CLI runtime sessions once compaction preflight crosses threshold.
- `impact:message-loss`: The reported failure replaces the expected assistant response with the missing-harness warning on inbound channel turns.
- `impact:session-state`: The failure prevents compaction and leaves long-running sessions growing toward overflow until users reset conversation state.
- `impact:auth-provider`: The affected path is the Claude CLI runtime/auth selection created by model-scoped runtime policy.

## Summary

The report should stay open because current main still has the same CLI-runtime versus embedded-harness mismatch in the compaction preflight path. The normal dispatch path now recognizes model-scoped `claude-cli` runtime policy, but the preflight helper still calls the agent-harness selector unconditionally, so the reported `MissingAgentHarnessError("claude-cli")` remains source-reproducible.

## What This Changes

The issue asks OpenClaw to handle context-engine compaction preflight correctly for Claude CLI runtime sessions configured with model-scoped `agentRuntime.id: "claude-cli"`.

## Best Possible Solution

Bypass embedded harness compaction for CLI runtime aliases while allowing the active context engine to compact normally, then add focused regression coverage for model-scoped `claude-cli` runtime policy.

## Reproduction Assessment

Yes at source level: current main still resolves the model-scoped `claude-cli` runtime through `selectAgentHarness` during compaction preflight, and that selector still throws for non-harness explicit runtimes. I did not execute a live over-threshold Claude CLI session in this read-only review.

## Solution Assessment

Yes, the requested direction is the narrow maintainable fix: make the compaction preflight respect CLI runtime aliases instead of treating them as embedded harness ids. The implementation should preserve context-engine compaction rather than skipping the whole compaction operation for CLI sessions.
