## Summary

Current main already fixes the reported compaction preflight failure by making harness selection treat provider-owned CLI runtime aliases such as `claude-cli` as PI passthrough placeholders instead of throwing `MissingAgentHarnessError`. The fix is merged on main via https://github.com/openclaw/openclaw/pull/85631; the earlier open fix PR is now superseded.

## What This Changes

The issue reports that compaction preflight treats a model-scoped `claude-cli` runtime as an embedded harness and throws `MissingAgentHarnessError` once sessions cross the compaction threshold.

## Best Possible Solution

Keep the merged CLI-runtime passthrough in harness selection; no further core change is needed for this reported compaction preflight failure.

## Reproduction Assessment

No current-main failure path remains from source inspection: `claude-cli` for the `anthropic` provider now routes through the CLI alias passthrough before the missing-harness throw. The pre-fix parent of the merge commit shows the old throwing branch, which matches the reported symptom.

## Solution Assessment

Yes. Fixing the shared harness selection seam is narrower and more durable than patching only the compaction caller, because it also protects other non-dispatch callers while preserving fail-closed behavior for unknown runtimes.
