## Summary

Keep open: current main still has the source-level startup ordering that can send plugin-loader diagnostics to stdout before JSON-mode stderr routing is enabled, and an open closing PR already targets the narrow fix.

## What This Changes

The issue asks for non-routed plugin-provided `openclaw ... --json` commands to keep stdout parseable by routing early plugin-loader diagnostics to stderr before lazy plugin CLI registration.

## Best Possible Solution

Review and land the narrow `runCli` startup-order fix so JSON-mode stderr routing is enabled before lazy plugin CLI registration, while preserving existing routed-command and non-JSON behavior.

## Reproduction Assessment

Yes. The reported `openclaw memory search "BRAIN autonomy" --json` path is source-reproducible on current main because lazy plugin registration can emit loader debug lines before commander preAction enables JSON-mode stderr routing.

## Solution Assessment

Yes. The best path is the existing narrow fix in https://github.com/openclaw/openclaw/pull/81536, which moves the already-established JSON diagnostic routing earlier for the non-routed commander path instead of changing plugin logging policy globally.
