## Summary

Keep this issue open: current main still has the source-level ordering that can leak plugin loader diagnostics to stdout before `preAction`, and the issue is already paired with an open fixing PR that uses closing syntax but has unresolved review feedback.

## What This Changes

The issue requests that non-routed plugin-backed `openclaw <command> --json` invocations keep stdout parseable by routing early plugin-loader diagnostics to stderr before lazy command registration.

## Best Possible Solution

Land a focused fix through `https://github.com/openclaw/openclaw/pull/81536` or a replacement that scopes early stderr routing to actual JSON-output plugin-registration work, then close this issue as implemented.

## Reproduction Assessment

Yes. The `openclaw memory search ... --json` path is source-reproducible because lazy plugin registration runs before `preAction`, while plugin-loader debug output still uses stdout unless stderr routing has already been forced.

## Solution Assessment

Yes on the issue direction, but not yet as merged code. The maintainable fix is to route early plugin-loader diagnostics off stdout for real JSON-output commands without broadening parse-only or pass-through `--json` behavior.
