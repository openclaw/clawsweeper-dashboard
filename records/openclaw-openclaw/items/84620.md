## Summary

Keep open: the PR targets a real outbound-delivery regression, but the current patch is not merge-ready because it can break the existing setup-shell bootstrap path and still lacks contributor-supplied real behavior proof.

## What This Changes

The PR changes outbound adapter lookup to stop active-registry fallback when a pinned channel entry exists, preserves active-registry lookup for channels absent from the pinned registry, and updates channel pinning tests.

## Best Possible Solution

Preserve stable pinned lookup for fully loaded channel entries, repin or explicitly route through the runtime registry loaded by bootstrap for setup-shell entries, keep dynamic-channel fallback, and add redacted Discord proof after the patch.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: pin a setup-only channel entry with no outbound, set the active registry to a runtime entry with outbound, then call `loadChannelOutboundAdapter` or `deliverOutboundPayloads`; current main has focused tests for that path.

## Solution Assessment

No. The PR addresses the unstable active-registry fallback, but the current shape removes an existing bootstrap compatibility path instead of making bootstrap return or repin the runtime registry it loads.
