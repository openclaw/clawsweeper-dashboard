## Summary

Keep open: the fix direction is valid, but the updated branch still breaks the setup-shell outbound handoff and CI now reproduces that failure; the PR also still lacks artifact-backed real behavior proof.

## What This Changes

The PR changes channel outbound registry lookup to avoid active-registry fallback when a pinned channel entry exists, while keeping active lookup for channels absent from the pinned registry, and updates channel pinning tests.

## Best Possible Solution

Land a version that removes request-time dependence on arbitrary active registry replacements, explicitly preserves setup-shell bootstrap handoff, keeps dynamic channel support, and includes regression tests plus redacted Discord proof.

## Reproduction Assessment

Yes for the PR regression: the failed CI shard at head `be794a455bc3b3ec349beaf308620939b7065a84` reproduces `Outbound not configured for channel: matrix` in the setup-shell outbound test. For the linked Discord beta issue, source inspection matches the unstable active-registry fallback, but I did not run a live Discord setup.

## Solution Assessment

No, not as-is. The maintainable fix should keep pinned registry lookup stable for normal outbound sends while making bootstrap explicitly repin or return the loaded runtime registry for setup-only entries.
