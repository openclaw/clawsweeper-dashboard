## Summary

Keep open: current main does not implement Anthropic native web search, and this PR still has blocking functional, policy, and artifact-hygiene issues before it can merge.

## What This Changes

The PR adds an Anthropic native web-search provider, a payload-injection stream wrapper, unit tests, legacy bundled registry wiring, and two root runtime log artifacts.

## Best Possible Solution

Rebuild the feature on current main inside the Anthropic plugin using a manifest-declared `webSearchProviders` public artifact, explicit opt-in activation tied to web-search config, propagated provider settings/credentials, docs/changelog/tests, and no runtime artifacts.

## Reproduction Assessment

Yes. The blocking PR issues are source-reproducible from the diff and current-main source: unconditional wrapper installation, missing config propagation, stale registry wiring, and committed log files.

## Solution Assessment

No. The direction is plausible, but this patch is not the best solution because it bypasses current manifest-first discovery and OpenClaw's configured web-search policy boundaries.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Gate native search before wrapping Anthropic streams:** `extensions/anthropic/index.ts:390-394`
  - body: `wrapStreamFn` is installed for every Anthropic request, and the wrapper injects a server `web_search_*` tool whenever the model is direct Anthropic. That ignores `tools.web.search.enabled` and `tools.web.search.provider`, so disabled web search or another selected provider can still trigger Anthropic server-side searches and costs.
  - confidence: 0.94
- **[P1] Pass resolved Anthropic search config into the wrapper:** `extensions/anthropic/index.ts:392-393`
  - body: The wrapper is constructed with only `ctx.streamFn`, so `buildAnthropicWebSearchServerTool()` always sees `searchConfig` as undefined. Runtime settings such as `toolVersion`, domain filters, `maxUses`, and `userLocation` are silently ignored.
  - confidence: 0.91
- **[P2] Register through the manifest web-search contract:** `src/bundled-web-search-registry.ts:1`
  - body: Adding Anthropic to the legacy bundled registry does not match current main, where bundled web-search providers are discovered from `contracts.webSearchProviders` plus public artifacts/runtime entries. Without updating the manifest and artifact path, this provider will not be exposed correctly after rebase.
  - confidence: 0.89
- **[P2] Persist or reuse configured Anthropic credentials:** `extensions/anthropic/src/anthropic-web-search-provider.ts:149-154`
  - body: This provider declares a web-search credential path but `setCredentialValue` is a no-op and `getCredentialValue` only reads `process.env.ANTHROPIC_API_KEY`. Configure/onboarding flows can accept a key without saving or later resolving it, leaving the provider unavailable unless the env var was already set.
  - confidence: 0.84
- **[P2] Remove committed runtime log artifacts:** `openclaw-2026-03-18.log:1`
  - body: The two root `openclaw-*.log` files are unrelated to the provider implementation and include local runtime metadata such as run IDs, installed paths, timestamps, and debug/error messages. They should not be part of the PR.
  - confidence: 0.9
