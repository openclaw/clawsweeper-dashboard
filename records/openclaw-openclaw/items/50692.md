## Summary

Keep open. Current main does not implement Anthropic native web search, and this PR still has blocking correctness and security-policy issues around unconditional server-side search injection, missing config propagation, stale web-search discovery wiring, credential persistence, and committed runtime logs.

## What This Changes

The PR adds an Anthropic native web-search provider, a stream wrapper that injects Anthropic server tools, unit tests, legacy bundled registry wiring, and two root log artifacts.

## Best Possible Solution

Rebuild the feature on current main inside the Anthropic plugin with a manifest/public-artifact webSearchProviders contract, explicit activation policy, config propagation, credential handling, docs/changelog/tests, and no unrelated artifacts.

## Reproduction Assessment

Yes. The blocking issues are source-reproducible from the PR diff and current-main code: the wrapper is unconditionally installed, config is not passed into it, the registry file is stale, and the log artifacts are in the PR file list.

## Solution Assessment

No. The direction is plausible, but this patch is not the best solution because it bypasses current manifest-first discovery and does not preserve OpenClaw's web-search policy/config boundaries.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Gate native search before wrapping Anthropic streams:** `extensions/anthropic/index.ts:390-394`
  - body: `wrapStreamFn` is installed for every Anthropic request, and the wrapper injects a server `web_search_*` tool whenever the model is direct Anthropic. That ignores `tools.web.search.enabled` and `tools.web.search.provider`, so users who disabled web search or selected another backend can still trigger Anthropic server-side searches and costs.
  - confidence: 0.94
- **[P1] Pass resolved Anthropic search config into the wrapper:** `extensions/anthropic/index.ts:392-393`
  - body: The wrapper is constructed with only `ctx.streamFn`, so `buildAnthropicWebSearchServerTool()` always sees `searchConfig` as undefined. Runtime settings such as `toolVersion`, domain filters, `maxUses`, and `userLocation` are therefore ignored even though the provider accepts and tests them.
  - confidence: 0.91
- **[P2] Register through the manifest web-search contract:** `src/bundled-web-search-registry.ts:1`
  - body: Adding Anthropic to the legacy bundled registry does not match current main, where bundled web-search providers are discovered from `contracts.webSearchProviders` plus public artifacts/runtime entries. Without updating the Anthropic manifest and public artifact path, this provider will not be exposed correctly after rebase.
  - confidence: 0.89
- **[P2] Persist or reuse configured Anthropic credentials:** `extensions/anthropic/src/anthropic-web-search-provider.ts:149-154`
  - body: This provider declares a web-search credential path but `setCredentialValue` is a no-op and `getCredentialValue` only reads `process.env.ANTHROPIC_API_KEY`. Configure/onboarding flows can accept a key without saving or later resolving it, leaving the provider unavailable unless the env var was already set.
  - confidence: 0.84
- **[P2] Remove committed runtime log artifacts:** `openclaw-2026-03-18.log:1`
  - body: The two root `openclaw-*.log` files are unrelated to the provider implementation and include local runtime metadata such as run IDs, absolute installed paths, timestamps, and debug/error messages. They should not be part of the PR.
  - confidence: 0.9
