## Summary

Keep this PR open for maintainer review: current main still does not provide an Anthropic native web-search provider, but this branch has blocking policy, config, discovery, and artifact issues before it can merge.

## What This Changes

The branch adds an Anthropic native web-search provider and stream wrapper tests, wires it into Anthropic provider/web-search registration, and includes two root runtime log files.

## Best Possible Solution

Rebuild the feature on current main inside the Anthropic plugin with a manifest/public-artifact `webSearchProviders` contract, explicit opt-in gating, config and credential propagation, docs/changelog/tests, and no runtime artifacts.

## Reproduction Assessment

Yes. The blocking issues are source-reproducible from the PR diff and current main: the wrapper is installed outside the configured web-search gate, config is not passed into it, the registry target is obsolete, and log artifacts are in the file list.

## Solution Assessment

No. The direction is plausible, but this patch is not the best way to solve it because it bypasses current manifest-first discovery and does not preserve OpenClaw's web-search policy, credential, and domain-filter boundaries.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Gate native search before wrapping Anthropic streams:** `extensions/anthropic/index.ts:390-394`
  - body: `wrapStreamFn` is installed for every Anthropic request, and the wrapper injects a `web_search_*` server tool whenever the model is direct Anthropic. That ignores `tools.web.search.enabled` and `tools.web.search.provider`, so users who disabled web search or selected another backend can still trigger Anthropic server-side searches and costs.
  - confidence: 0.94
- **[P1] Pass resolved Anthropic search config into the wrapper:** `extensions/anthropic/index.ts:392-393`
  - body: The wrapper is constructed with only `ctx.streamFn`, so `buildAnthropicWebSearchServerTool()` always sees `searchConfig` as undefined. Runtime settings such as `toolVersion`, domain filters, `maxUses`, and `userLocation` are silently ignored.
  - confidence: 0.91
- **[P2] Register through the manifest web-search contract:** `src/bundled-web-search-registry.ts:1`
  - body: Adding Anthropic to `src/bundled-web-search-registry.ts` does not match current main, where bundled web-search providers are discovered from manifest `contracts.webSearchProviders` plus public artifacts. Without updating the Anthropic manifest and artifact path, the provider will not be exposed correctly after rebase.
  - confidence: 0.89
- **[P2] Persist or reuse configured Anthropic credentials:** `extensions/anthropic/src/anthropic-web-search-provider.ts:149-154`
  - body: This provider declares a credential path but only reads `process.env.ANTHROPIC_API_KEY`, while `setCredentialValue` is a no-op and no configured-credential helper is provided. Setup can accept a key without saving or later resolving it, leaving the provider unavailable unless the env var was already set.
  - confidence: 0.84
- **[P2] Remove committed runtime log artifacts:** `openclaw-2026-03-18.log:1`
  - body: The root `openclaw-*.log` files are unrelated to the provider implementation and include local runtime metadata such as run IDs, installed paths, timestamps, and channel/session details. They should not be part of this PR.
  - confidence: 0.9
