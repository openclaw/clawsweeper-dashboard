Full review comments:

- **[P1] Gate native search before wrapping Anthropic streams:** `extensions/anthropic/index.ts:393-395`
  - body: `wrapStreamFn` is installed for every Anthropic request, and the wrapper injects a server `web_search_*` tool whenever the model looks like direct Anthropic traffic. That ignores `tools.web.search.enabled` and `tools.web.search.provider`, so disabled web search or another selected backend can still trigger Anthropic-hosted searches and costs.
  - confidence: 0.95
- **[P1] Pass resolved search config into the wrapper:** `extensions/anthropic/index.ts:395`
  - body: The wrapper is constructed with only `ctx.streamFn`, so `buildAnthropicWebSearchServerTool()` always sees `searchConfig` as undefined. Runtime settings such as `toolVersion`, domain filters, `maxUses`, and `userLocation` are silently ignored, including domain restrictions users may rely on for policy.
  - confidence: 0.93
- **[P2] Register through the manifest web-search contract:** `src/bundled-web-search-registry.ts:1`
  - body: Current main discovers bundled web-search providers through manifest `webSearchProviders` and public artifacts, while this PR adds Anthropic to `src/bundled-web-search-registry.ts`, a file absent from current main. After rebase, this does not expose the provider through the current plugin discovery boundary.
  - confidence: 0.9
- **[P2] Use the current pi package imports:** `extensions/anthropic/src/anthropic-native-search-wrapper.ts:1-2`
  - body: Current main depends on `@earendil-works/pi-agent-core` and `@earendil-works/pi-ai`, but this new file imports the old `@mariozechner` package names. On current main, module resolution will fail unless the imports are updated.
  - confidence: 0.9
- **[P2] Persist or reuse configured Anthropic credentials:** `extensions/anthropic/src/anthropic-web-search-provider.ts:142-147`
  - body: This provider declares a web-search credential path, but `setCredentialValue` is a no-op and `getCredentialValue` only reads `process.env.ANTHROPIC_API_KEY`. Configure flows and stored Anthropic SecretRefs can therefore leave the web-search provider looking unconfigured unless the environment variable is already set.
  - confidence: 0.82
- **[P2] Reject mutually exclusive domain filters:** `extensions/anthropic/src/anthropic-web-search-provider.ts:81-85`
  - body: `buildAnthropicWebSearchServerTool()` emits both `allowed_domains` and `blocked_domains` when both config arrays are present, and the PR body's sample config includes both. Anthropic's server-tool contract allows either `allowed_domains` or `blocked_domains`, not both in one request, so OpenClaw should validate or reject this before the API call. ([platform.claude.com](https://platform.claude.com/docs/en/agents-and-tools/tool-use/server-tools))
  - confidence: 0.86
