## Summary

Keep open. The feature request is valid and paired with the still-open canonical issue, but this PR is not ready to merge because the latest diff injects Anthropic server-side search outside the configured web-search policy, drops configured search constraints, misses current main's manifest-first provider registration path, includes committed runtime logs, and lacks real Anthropic behavior proof.

## What This Changes

The PR adds an Anthropic native web-search provider and stream wrapper for injecting Anthropic server-side `web_search` tools, registers it in Anthropic/bundled web-search paths, adds unit tests, and includes two runtime log files.

## Best Possible Solution

Rebase or replace this on current main using the existing provider-owned native-search pattern: manifest-declare the Anthropic web-search provider/config, gate injection on active web-search policy, pass and validate Anthropic settings, handle server result/citation blocks, remove logs, update docs/tests, and add redacted live proof.

## Reproduction Assessment

Yes for the review findings by source inspection: the PR diff shows unconditional wrapper registration and missing config propagation, and current main shows manifest-first provider discovery. I did not run tests or live Anthropic calls because this was a read-only review.

## Solution Assessment

No. The feature direction is plausible, but this implementation is not the narrow maintainable solution until it follows the existing native-search gating pattern, current manifest contracts, Anthropic's domain-filter contract, and real behavior proof requirements.

Full review comments:

- **[P1] Gate native search on configured web-search policy:** `extensions/anthropic/index.ts:390-394`
  - body: The provider wrapper is installed for every Anthropic request, and the wrapper injects a server `web_search` tool for direct Anthropic payloads without checking `tools.web.search.enabled` or whether `tools.web.search.provider` is `anthropic`. That can send paid server-side searches when users disabled web search or selected another provider, so only compose/inject this when the Anthropic web-search provider is active.
  - confidence: 0.94

  - body: `createAnthropicNativeSearchStreamWrapper` is constructed with only `ctx.streamFn`, so the server tool is always built from undefined/default search config. Configured `allowedDomains`, `blockedDomains`, `maxUses`, `userLocation`, and tool version are ignored, including policy-sensitive domain constraints.
  - confidence: 0.93

  - body: Current main discovers bundled web-search providers from `contracts.webSearchProviders`; runtime registration alone, plus the stale bundled registry edit, will not make `provider: "anthropic"` a manifest-owned provider on current main. Add the Anthropic manifest contract/schema/UI hints and remove the obsolete registry wiring.
  - confidence: 0.86

  - body: The builder emits both `allowed_domains` and `blocked_domains` when both arrays are configured, but Anthropic rejects requests that include both. Validate this as a config error or choose one policy before adding the server tool.
  - confidence: 0.88
