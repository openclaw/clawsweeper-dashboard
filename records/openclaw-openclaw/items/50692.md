## Summary

Keep open. Current main still lacks the requested Anthropic native web-search provider, but this PR is not mergeable as-is because the branch is conflicting and source inspection shows policy-gating, config propagation, manifest-discovery, dependency-import, credential, proof, and artifact issues.

## What This Changes

The PR adds an Anthropic native web-search provider, a stream wrapper for Anthropic server-side web_search tools, provider wiring, unit tests, and two root runtime log files.

## Best Possible Solution

Rework this as a current-main Anthropic plugin change with manifest-declared webSearchProviders, policy-gated server-tool injection only when Anthropic native search is enabled and selected, propagated config/credentials, server result/citation handling, docs/tests, and no runtime artifacts.

## Reproduction Assessment

Yes for the blocking PR issues: source inspection shows unconditional stream wrapping, missing config propagation, stale discovery wiring, stale pi imports, credential persistence gaps, and committed runtime logs. The underlying request is a feature gap rather than a broken existing behavior reproduction.

## Solution Assessment

No. The direction is plausible, but this branch should be rebuilt around current manifest-first plugin discovery and explicit web-search/server-tool policy instead of an always-on provider stream wrapper.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Gate native search before wrapping Anthropic streams:** `extensions/anthropic/index.ts:393-395`
  - body: `wrapStreamFn` is installed for every Anthropic request, and the wrapper injects a server `web_search_*` tool whenever the model looks like direct Anthropic traffic. That ignores `tools.web.search.enabled` and `tools.web.search.provider`, so disabled web search or another selected backend can still trigger Anthropic-hosted searches and costs.
  - confidence: 0.94
- **[P1] Pass resolved search config into the wrapper:** `extensions/anthropic/index.ts:395`
  - body: The wrapper is constructed with only `ctx.streamFn`, so `buildAnthropicWebSearchServerTool()` always sees `searchConfig` as undefined. Runtime settings such as `toolVersion`, domain filters, `maxUses`, and `userLocation` are silently ignored, including domain restrictions users may rely on for policy.
  - confidence: 0.91
- **[P2] Register through the manifest web-search contract:** `src/bundled-web-search-registry.ts:1`
  - body: Current main discovers bundled web-search providers through manifest `webSearchProviders` and public artifacts, while this PR adds Anthropic to `src/bundled-web-search-registry.ts`, a stale file absent from current main. After rebase, this does not expose the provider through the current plugin discovery boundary.
  - confidence: 0.89
- **[P2] Update the renamed pi package imports:** `extensions/anthropic/src/anthropic-native-search-wrapper.ts:1-2`
  - body: Current main uses `@earendil-works/pi-agent-core` and `@earendil-works/pi-ai`, but this new file imports the old `@mariozechner` package names. On current main, module resolution will fail unless the imports are updated.
  - confidence: 0.9
- **[P2] Persist or reuse configured Anthropic credentials:** `extensions/anthropic/src/anthropic-web-search-provider.ts:142-147`
  - body: This provider declares a web-search credential path, but `setCredentialValue` is a no-op and `getCredentialValue` only reads `process.env.ANTHROPIC_API_KEY`. Configure flows can accept a key without saving or later resolving it unless the environment variable was already set.
  - confidence: 0.81
- **[P2] Remove committed runtime log artifacts:** `openclaw-2026-03-18.log:1`
  - body: The root `openclaw-*.log` files are unrelated to the provider implementation and include local runtime metadata such as run IDs, installed paths, timestamps, and debug/error messages. They should not be part of this PR.
  - confidence: 0.9
