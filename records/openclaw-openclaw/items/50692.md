## Summary

Keep open. Current main does not implement Anthropic native web search, and this PR is still a plausible contribution toward #49949, but the branch is not mergeable as-is and the diff introduces blocking functional and security-sensitive defects around unconditional server-side search injection, lost configuration, stale registry wiring, and committed runtime logs.

## What This Changes

The PR adds an Anthropic native web-search provider, a stream wrapper that injects Anthropic `web_search_*` server tools, provider registration, tests, legacy bundled web-search registry wiring, and two root log files.

## Best Possible Solution

Rebuild this on current main inside the Anthropic plugin with an explicit manifest/public-artifact `webSearchProviders` contract, activation logic modeled after the existing Codex native-search gate, config propagation for Anthropic tool options, managed-tool suppression only while native search is actually active, docs/changelog/tests for disabled and alternate-provider cases, and no unrelated artifacts.

## Reproduction Assessment

Yes. The blocking issues are reproducible by code inspection of the PR diff against current main: the wrapper is installed unconditionally, no search config is passed into it, the branch edits a removed registry path, and the added logs are visible in the PR file list.

## Solution Assessment

No. The feature direction is plausible, but this patch is not the narrowest maintainable solution because it bypasses current manifest-first discovery, ignores user web-search policy/config, loses configured credentials, and commits unrelated runtime logs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Gate native search before wrapping Anthropic streams:** `extensions/anthropic/index.ts:390-394`
  - body: `wrapStreamFn` is installed for every Anthropic request, and the wrapper injects a server `web_search_*` tool whenever the model is direct Anthropic. That ignores `tools.web.search.enabled` and `tools.web.search.provider`, so users who disabled web search or selected another backend can still trigger Anthropic server-side searches and costs.
  - confidence: 0.94
- **[P1] Pass resolved Anthropic search config into the wrapper:** `extensions/anthropic/index.ts:392-393`
  - body: The wrapper is constructed with only `ctx.streamFn`, so `buildAnthropicWebSearchServerTool()` always sees `searchConfig` as undefined. Runtime settings such as `toolVersion`, domain filters, `maxUses`, and `userLocation` are therefore ignored even though the provider accepts and tests them.
  - confidence: 0.91
- **[P2] Register through the manifest web-search contract:** `src/bundled-web-search-registry.ts:1`
  - body: Adding Anthropic to the legacy bundled registry does not match current main, where bundled web-search providers are discovered from `contracts.webSearchProviders` plus public artifacts/runtime entries. Without updating the Anthropic manifest and public artifact path, this provider will not be exposed correctly after rebase.
  - confidence: 0.89
- **[P2] Persist configured Anthropic credentials:** `extensions/anthropic/src/anthropic-web-search-provider.ts:152-154`
  - body: This provider declares a credential path but has no configured credential setter and `setCredentialValue` is a no-op. Configure/onboarding flows that call `setCredentialValue(...)` will accept a pasted key but not save it, leaving the provider unconfigured unless `ANTHROPIC_API_KEY` was already set.
  - confidence: 0.85
- **[P2] Remove committed runtime log artifacts:** `openclaw-2026-03-18.log:1`
  - body: The two root `openclaw-*.log` files are unrelated to the provider implementation and include local runtime metadata such as run IDs, absolute installed paths, timestamps, and debug/error messages. They should not be part of the PR.
  - confidence: 0.9
