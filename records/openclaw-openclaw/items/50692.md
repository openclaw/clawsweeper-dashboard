## Summary

Keep open, but the PR is not merge-ready. It addresses a real Anthropic native web-search feature gap, yet the current branch injects Anthropic server-side search too broadly, drops important configuration, misses manifest-first provider metadata, and still lacks real behavior proof.

## What This Changes

The PR adds an Anthropic native web-search provider, a stream wrapper that injects Anthropic server-side web_search tools, bundled registry wiring, unit tests, and two root OpenClaw log files.

## Best Possible Solution

Rework the feature in the current Anthropic plugin boundary: declare manifest/config ownership, compose with existing Anthropic stream wrappers, activate only when Anthropic web_search is enabled and selected, suppress the managed web_search tool when native search is active, pass policy config into the server tool, remove generated logs, update docs/tests, and provide redacted real behavior proof.

## Reproduction Assessment

Yes for the review findings via source inspection. The PR head unconditionally installs the wrapper without web-search selection checks, passes no search config into it, and current main shows the managed web_search/native-tool arbitration path this needs to follow.

## Solution Assessment

No. The current diff is not the best maintainable solution because it bypasses manifest-first metadata and configured tool policy; the safer path is a rebase plus a config-gated Anthropic plugin implementation modeled on existing native-tool activation patterns.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Gate Anthropic server search on configured web search:** `extensions/anthropic/index.ts:393-395`
  - body: This hook wraps every direct Anthropic request, so the PR will inject Anthropic's server-side web_search tool even when `tools.web.search.enabled` is false or a different provider is selected. That can trigger unexpected outbound searches/costs and also leaves the managed `web_search` tool exposed instead of suppressing it when native search is active.
  - confidence: 0.95
- **[P1] Pass Anthropic search policy into the wrapper:** `extensions/anthropic/index.ts:395`
  - body: The wrapper is constructed with only `ctx.streamFn`, so the injected server tool always uses defaults. User-configured `allowedDomains`, `blockedDomains`, `maxUses`, `userLocation`, and `toolVersion` are silently ignored, including domain restrictions a user may rely on for policy or compliance.
  - confidence: 0.94
- **[P2] Declare Anthropic web search in the manifest:** `src/bundled-web-search-registry.ts:62-66`
  - body: Adding Anthropic to the bundled web-search registry is not enough on current main: bundled web-provider discovery and contract tests are manifest-first, but the Anthropic manifest still has no `webSearchProviders` contract or config schema for this provider. Setup/discovery will drift and the loader contract expects the registry to match manifest-owned web-search IDs.
  - confidence: 0.9
- **[P2] Persist credentials collected by search setup:** `extensions/anthropic/src/anthropic-web-search-provider.ts:146-148`
  - body: The setup flow calls `setCredentialValue` for providers without `setConfiguredCredentialValue`; this implementation makes that method a no-op while `getCredentialValue` only reads `ANTHROPIC_API_KEY`. A user can choose Anthropic in web-search setup, paste a key, and finish with nothing saved for the next run.
  - confidence: 0.86
- **[P2] Remove the committed OpenClaw logs:** `openclaw-2026-03-18.log:1`
  - body: The PR adds root runtime log files that are not fixtures or source. These generated logs add unrelated churn and can expose local execution details, so they should be removed from the branch before merge.
  - confidence: 0.88

## Security Review
