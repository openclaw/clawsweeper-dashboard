## Summary

Keep open. Current main and latest release still do not add ambient gateway proxy variables or NODE_OPTIONS to MCP stdio child env, and the linked open PR remains the active implementation path but needs rebase/proof and an explicit NODE_OPTIONS policy decision.

## What This Changes

MCP stdio child processes should receive the gateway proxy/NODE_OPTIONS context needed in proxy-only sandbox environments, or the supported configuration path should be explicit.

## Best Possible Solution

Land or replace the linked PR with a refreshed stdio launch-boundary fix that propagates safe ambient proxy defaults, preserves configured-env filtering, and documents/tests the accepted NODE_OPTIONS inheritance policy.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: start the gateway with HTTPS_PROXY or NODE_OPTIONS, configure a stdio MCP server without explicit env, and current main spawns the child with only the MCP SDK allowlist plus configured env.

## Solution Assessment

Unclear for the full requested scope. Propagating safe proxy defaults at the stdio resolver boundary is the maintainable direction, but ambient NODE_OPTIONS inheritance needs an explicit security/product policy.
