## Summary

Keep open. Current main still resolves stdio MCP env from the configured server block only and then spawns with the MCP SDK safe default env plus that explicit env, so ambient gateway proxy variables and NODE_OPTIONS are not propagated. The issue is also paired with open PR #65574 using closing syntax, so it should remain open until that PR or an equivalent fix lands.

## What This Changes

Request MCP stdio child processes to inherit or auto-forward gateway proxy env and NODE_OPTIONS so proxy-only sandbox MCP servers can reach external APIs.

## Best Possible Solution

Land or replace the linked MCP stdio fix with narrow proxy-env propagation, explicit NODE_OPTIONS policy, precedence tests, and docs that match the chosen startup-safety contract.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: run the gateway with HTTP_PROXY/HTTPS_PROXY or NODE_OPTIONS, configure a stdio MCP server without explicit env, and current main resolves only raw server env before spawning with SDK defaults that exclude those keys.

## Solution Assessment

Unclear for the full requested scope. Auto-forwarding safe proxy env vars is a narrow maintainable fix, but blindly inheriting NODE_OPTIONS would bypass the current startup-control filter and needs an explicit security decision.

## Security Review

Status: needs_attention

Summary: The requested NODE_OPTIONS inheritance is security-sensitive because it can execute parent startup hooks inside MCP stdio children.

Concerns:

- **[medium] Decide NODE_OPTIONS inheritance explicitly:** `src/infra/host-env-security-policy.json:3`
  - body: NODE_OPTIONS is currently blocked as an interpreter-startup control key, and MCP stdio env filtering uses that policy. Any automatic inheritance exception should be narrowly scoped, documented, and covered by tests rather than bundled with general proxy env propagation.
  - confidence: 0.9
- **[medium] Avoid broad ambient env copying:** `src/agents/mcp-stdio-transport.ts:50`
  - body: Proxy URLs can contain credentials and MCP stdio servers may be third-party processes. A fix should propagate only the intended proxy-related keys with clear precedence instead of copying the gateway environment wholesale.
  - confidence: 0.82
