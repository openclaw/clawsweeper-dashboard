## Summary

Keep open. Current main still passes only configured MCP stdio env into the transport, the transport merges that with the MCP SDK allowlist that omits proxy variables and NODE_OPTIONS, and open PR #65574 already targets this issue with closing syntax.

## What This Changes

MCP stdio child processes should receive the proxy-related environment needed in proxy-only deployments, with an explicit policy for NODE_OPTIONS.

## Best Possible Solution

Review, land, or replace #65574 with a current-main fix that propagates safe proxy env defaults with precedence tests and documents the deliberate NODE_OPTIONS policy.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: start the gateway with HTTP_PROXY, HTTPS_PROXY, or NODE_OPTIONS, configure a stdio MCP server without explicit env, and current main resolves no ambient env before spawning with SDK defaults that omit those keys.

## Solution Assessment

Unclear for the full requested scope. Safe proxy env propagation is likely the narrow fix, but NODE_OPTIONS should not be inherited blindly because current startup-safety filtering treats it as dangerous.
