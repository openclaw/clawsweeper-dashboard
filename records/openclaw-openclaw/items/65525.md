Keep open. Current main and the latest release still do not add ambient proxy variables or NODE_OPTIONS to MCP stdio child env, and the linked open fix PR remains the active path but needs rebase/proof and an explicit NODE_OPTIONS policy decision.

## What This Changes

The issue reports that MCP stdio server children lose gateway proxy and NODE_OPTIONS environment, breaking proxy-only sandbox deployments.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/65574 with a refreshed stdio launch-boundary fix that propagates safe proxy defaults, preserves user-env precedence and filtering, and documents/tests the NODE_OPTIONS policy.

Yes. Source inspection gives a high-confidence path: start the gateway with HTTPS_PROXY or NODE_OPTIONS, configure a stdio MCP server without explicit env, and current main spawns with only the MCP SDK allowlist plus configured env.

## Solution Assessment

Unclear for the full requested scope. Propagating safe proxy env defaults at the stdio resolver boundary is maintainable, but NODE_OPTIONS inheritance needs an explicit security/product decision and the linked PR must preserve the env filter.
