## Summary

Keep open: current main still blocks Slack room/group text control commands before elevated authorization can evaluate `tools.elevated.allowFrom.slack`, and the existing elevated fallback hook is not registered by Slack.

## What This Changes

The issue asks for Slack elevated allowFrom to authorize `/elevated` in room/group messages, or for a documented Slack fallback equivalent to the Discord elevated allowFrom fallback.

## Best Possible Solution

Adopt one explicit Slack contract for elevated commands, then cover Slack room/group `/elevated` with focused tests and keep Discord sibling fallback behavior consistent.

Yes from source: on current main, Slack room/group `/elevated` can be blocked at the command gate before `resolveElevatedPermissions` reads `tools.elevated.allowFrom.slack`. I did not run a live Slack socket-mode repro in this read-only review.
