## Summary

Keep open: current main still drops Slack room/group text control commands before elevated authorization can evaluate tools.elevated.allowFrom.slack, and no open or merged PR appears to close this Slack-specific ingress gap.

## What This Changes

The issue asks for Slack /elevated text commands to honor tools.elevated.allowFrom.slack, or for an equivalent documented Slack fallback, instead of being silently blocked by the command gate.

## Best Possible Solution

Implement an elevated-only Slack authorization path that lets room/group /elevated reach resolveElevatedPermissions, with regression coverage and docs clarifying Slack fallback or prerequisite behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a Slack room/group message containing /elevated from a sender listed only in tools.elevated.allowFrom.slack returns null at the Slack ingress command gate before elevated permissions run.

## Solution Assessment

Unclear until maintainers choose the authorization seam. The safer direction is an elevated-specific ingress/fallback path, not broadening the normal Slack command gate for every control command.
