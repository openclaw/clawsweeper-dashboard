## Summary

Keep open: current main and v2026.6.6 still drop Slack room/group `/elevated` text commands before `tools.elevated.allowFrom.slack` can be evaluated, and the fix changes a security-sensitive command authorization boundary.

## What This Changes

The issue requests that Slack `tools.elevated.allowFrom.slack` either satisfy the `/elevated` command gate or have an implemented, documented fallback path.

## Best Possible Solution

Adopt a maintainer-approved Slack elevated authorization model that only changes `/elevated` admission, preserves the elevated security boundary, and adds focused Slack tests plus docs, with the related Discord fallback considered explicitly.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path for a Slack room/group text command from a sender listed only in `tools.elevated.allowFrom.slack`; the message is dropped at ingress before elevated permissions run.

## Solution Assessment

Unclear until maintainers choose the security model. A reply-stage fallback alone is too late for Slack room/group text commands, while broadening command authorization must be directive-scoped to avoid widening other command access.
