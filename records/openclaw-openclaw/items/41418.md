## Summary

Keep open: current main still lacks a global agent dry-run mode, the linked branch is useful but has not landed as a PR, and the remaining work needs request-scoped execution semantics rather than cleanup closure.

## What This Changes

The issue requests a global agent `--dry-run` mode that previews all agent tool calls without executing them.

## Best Possible Solution

Define a request-scoped agent dry-run contract, then implement CLI, Gateway, embedded fallback, adapter preview behavior, docs, changelog, and focused regression coverage in coordination with https://github.com/openclaw/openclaw/issues/13617 and https://github.com/openclaw/openclaw/issues/66672.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main confirms `openclaw agent` has no `--dry-run` option and the agent tool paths still execute or delegate tools rather than returning preview payloads.

## Solution Assessment

Unclear. Intercepting near the tool adapter is plausible, but a process-global CLI flag alone is not the best maintainable solution until Gateway, local fallback, hooks, client tools, and delivery semantics are request-scoped.
