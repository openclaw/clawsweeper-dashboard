## Summary

Keep open: current main still lacks a global agent tool dry-run mode, existing dry-run support is scoped to other commands or per-tool behavior, and the requested change is a cross-cutting CLI/tool-execution feature that needs maintainer design before implementation.

## What This Changes

Request to add `openclaw agent --dry-run` so agent tool calls return structured previews instead of executing.

## Best Possible Solution

Define a request-scoped agent dry-run contract, then implement the CLI flag, Gateway/local propagation, adapter behavior, docs, changelog, and focused regression tests around the settled semantics.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms current main has no `openclaw agent --dry-run` flag and the tool adapter still calls real tool execution.

## Solution Assessment

Unclear. Intercepting near `toToolDefinitions()` is a plausible seam, but the proposed process-global approach and Gateway/client-tool behavior need request-scoped maintainer design before it is clearly the best solution.
