## Summary

Keep this open. Current main still has only scoped dry-run behavior; `openclaw agent` has no `--dry-run` flag, the Gateway and embedded agent option contracts do not carry dry-run state, and the Pi tool adapter still executes managed tools or delegates client tools instead of returning structured dry-run previews.

## What This Changes

Request to add a global `openclaw agent --dry-run` mode that previews all agent tool calls without executing them.

## Best Possible Solution

Implement this as core agent execution plumbing: add a request-scoped dry-run option from the CLI through Gateway and embedded agent options, intercept managed and client tool definitions after policy hooks, return structured preview results, log redacted previews to stderr, and document the safety limits clearly.

## Reproduction Assessment

Not applicable as a feature request, but the missing behavior is high-confidence from static verification. The CLI, protocol schema, embedded options, docs, and tool adapter all lack the requested `dryRun` path on current main.

## Solution Assessment

No, not exactly as proposed. The central adapter is the right execution funnel, but a process-global `setDryRun()`/`isDryRun()` pattern is not the safest fit for Gateway and fallback paths; request-scoped options threaded into the adapter would be more maintainable.

## Repair Work Prompt

Implement request-scoped global dry-run support for `openclaw agent` as tracked by https://github.com/openclaw/openclaw/issues/41418. Add `--dry-run` to `openclaw agent`, thread `dryRun` through `AgentCliOpts`, Gateway `AgentParamsSchema`, Gateway server-method dispatch, `AgentCommandOpts`, and the Pi tool-definition adapter paths. Intercept managed tools and client tools after before-tool-call hooks/policy blocks, return structured tool results with `status: "dry_run"`, `tool`, `would_execute: true`, and redacted/safe params, and log redacted previews to stderr without writing to stdout when `--json` is active. Do not implement this via process-global mutable state; keep it per request/run so Gateway and embedded fallback remain concurrency-safe. Do not change unrelated command-specific dry-run behavior for doctor, cron, update, message, backup, config, reset, sessions, or `/tools/invoke`. Update `docs/cli/agent.md` and add a user-facing CHANGELOG entry crediting `Thanks @itsarbit`. Add focused regression coverage for managed tool interception, client tool interception, normal execution, before-tool-call blocking precedence, CLI option plumbing, Gateway schema/dispatch plumbing, and JSON/stderr behavior where practical.
