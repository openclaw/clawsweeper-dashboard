## Summary

Keep open. Current main has scoped dry-run support and plugin hook blocking, but no global `openclaw agent --dry-run` flag or universal tool-preview result path; this is a new cross-cutting safety/UX mode that needs maintainer product and security agreement before implementation.

## What This Changes

Add a global `openclaw agent --dry-run` mode that exposes tools but intercepts execution and returns structured previews instead of side effects.

## Best Possible Solution

Define a core-owned agent dry-run contract first, then thread it through CLI/Gateway/local execution and the universal tool adapters with bounded preview payloads after existing policy hooks.

## Reproduction Assessment

Not applicable. This is a feature request, and source/docs inspection shows the requested flag and global interception behavior are not present on current main.

## Solution Assessment

Unclear. The universal adapter is a plausible seam, but the exact semantics need maintainer agreement because this changes the behavior of every tool execution path rather than fixing a defined bug.
