## Summary

Keep open: current main still lacks a global agent dry-run mode, and the remaining work is a cross-cutting agent execution feature that needs maintainer design rather than cleanup closure.

## What This Changes

Add a global `openclaw agent --dry-run` mode that previews tool calls instead of executing local, plugin, channel, or client-hosted tools.

## Best Possible Solution

Define a request-scoped agent dry-run contract, then implement CLI, Gateway, local fallback propagation, adapter preview behavior, docs, changelog, and focused regression coverage coordinated with https://github.com/openclaw/openclaw/issues/13617.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main confirms `openclaw agent` has no `--dry-run` option and the adapter paths still execute or delegate tools instead of returning dry-run previews.

## Solution Assessment

Unclear. Intercepting near the tool adapter is plausible, but the proposed process-global CLI flag is not clearly the best solution until request-scoped Gateway, embedded fallback, hooks, and client-tool semantics are settled.
