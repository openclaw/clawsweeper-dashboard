## Summary

Keep open: current main still lacks a global agent tool dry-run mode, and adding one is a cross-cutting tool-execution safety feature that needs maintainer design before implementation.

## What This Changes

The issue requests an `openclaw agent --dry-run` mode that previews all agent tool calls instead of executing them.

## Best Possible Solution

Define a request-scoped agent tool dry-run contract, then implement the CLI, tool-adapter, docs, changelog, and regression tests once maintainers settle scope and preview semantics.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms current main lacks the agent flag and still executes tools directly through the adapter.

## Solution Assessment

Unclear. Intercepting near `toToolDefinitions()` is a plausible seam for embedded tools, but the process-global approach and Gateway/client-tool behavior need design before it is clearly the best solution.
