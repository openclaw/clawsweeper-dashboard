## Summary

Keep this issue open: current main and the latest release still lack a global agent dry-run mode, and the related dry-run/workflow issues cover narrower or broader product areas rather than superseding this concrete agent tool-execution request.

## What This Changes

The issue requests a global `openclaw agent --dry-run` mode that lets the model see tools while returning preview payloads instead of executing real tool calls.

## Best Possible Solution

Define an explicit request-scoped agent execution-mode contract, then implement dry-run at the canonical tool execution and delegation boundaries with coverage for built-in, plugin/channel, client-hosted, gateway, and hook-order behavior.

## Reproduction Assessment

Yes for the current source state: source inspection shows `openclaw agent` has no `--dry-run` flag and the shared adapter still calls real tool execution. This is a feature gap rather than a failing bug reproduction.

## Solution Assessment

Unclear. Intercepting near the shared adapter is plausible, but the best solution needs a maintainer-approved, request-scoped execution-mode contract that covers gateway/client/plugin paths before a fix PR starts.
