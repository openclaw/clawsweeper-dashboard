## Risks / Open Questions

- Returning a generic synthetic success can cause later model steps to assume files, processes, messages, or remote state changed when they did not.
- A process-global flag would not be a safe foundation for gateway concurrency, nested runs, or other request-sharing environments.
- Intercepting only the primary adapter may leave client, nested code-mode, remote, plugin, or alternate execution paths outside the advertised guarantee.
- Tool parameters may contain secrets or private data, so preview logging needs the same redaction and output-safety rules as existing tool logging.
- A separate dry-run mechanism could drift from the broader Ask/Plan/Modify and trusted-tool-policy architecture.
