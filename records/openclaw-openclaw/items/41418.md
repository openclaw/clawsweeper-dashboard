## Summary

Keep open: current main and the latest release still lack a global agent tool-execution dry-run, and the related dry-run/workflow items cover narrower command scopes or broader workflow modes rather than replacing this product and security decision.

## What This Changes

Request adds a global `openclaw agent --dry-run` mode that previews all agent tool calls instead of executing built-in, plugin, channel, and client tools.

## Best Possible Solution

Define an additive, request-scoped global agent dry-run contract across CLI, Gateway/embedded runs, built-in/plugin/client tools, hook ordering, output shape, logging, docs, and regression tests before implementation.

## Reproduction Assessment

Yes. Source inspection shows current main has no `openclaw agent --dry-run` flag or docs entry, and the shared adapter/core tool paths still execute or delegate tools; this is feature-gap proof rather than a failing bug reproduction.

## Solution Assessment

Unclear. A global dry-run capability fits the safety goal, but the proposed process-global adapter flag is not yet the best design; a request-scoped contract across CLI, Gateway, embedded, plugin, and client-tool paths is safer.
