## Summary

Keep open: current main still lacks a global agent dry-run mode, and the request is a cross-cutting CLI/Gateway/tool-execution feature that needs maintainer design rather than cleanup closure.

## What This Changes

Add an `openclaw agent --dry-run` mode that lets the model plan tool calls while returning structured previews instead of executing tools.

## Best Possible Solution

Define a request-scoped agent dry-run contract, then implement CLI/Gateway/local propagation, adapter behavior, docs, changelog, and focused regression coverage around the settled semantics.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main shows no `openclaw agent --dry-run` option and the adapter still executes or delegates tools rather than returning dry-run previews.

## Solution Assessment

Unclear. Intercepting near the tool adapter is a plausible implementation seam, but the process-global approach in the proposal is not clearly the best solution until request scoping, Gateway behavior, hooks, and client-tool semantics are settled.
