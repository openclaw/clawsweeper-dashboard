## Summary

Keep this issue open: current main and v2026.6.6 still lack a global agent dry-run mode, and the overlapping dry-run/control-mode issues do not supersede this narrower agent tool-execution request.

## What This Changes

Request a global `openclaw agent --dry-run` mode that lets the model plan tool calls while returning structured previews instead of executing side effects.

## Best Possible Solution

Define a request-scoped agent execution-mode contract, then implement preview behavior at the canonical tool execution and delegation boundaries with tests for built-in, plugin/channel, client-hosted, Gateway, and hook-order paths.

## Reproduction Assessment

Yes, source-reproducible: current main has no `openclaw agent --dry-run` option and the shared adapter still calls `tool.execute(...)`. I did not run a live agent turn because the feature gap is visible from current source and docs.

## Solution Assessment

Unclear: adapter-level interception is plausible, but the proposed process-global CLI flag is not the best proven boundary until request-scoped Gateway, client-tool, hook, and preview semantics are approved.
