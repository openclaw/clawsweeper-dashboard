Add a global agent dry-run/preview mode that prevents tool execution and returns structured would-execute results.

## Best Possible Solution

Define a request-scoped agent tool-preview contract first, then implement CLI and protocol plumbing, adapter/delegated-tool behavior, docs, changelog, and regression coverage across Gateway and embedded execution.

Not applicable as a feature request. Source inspection on current main confirms there is no `openclaw agent --dry-run` option or request-scoped dry-run parameter in the agent CLI, Gateway protocol, embedded runner params, or tool adapter path.

## Solution Assessment

Unclear. Intercepting near the tool adapter is plausible, but a process-global CLI-only flag is not the narrowest maintainable solution for Gateway, embedded fallback, plugin hooks, client-hosted tools, and delivery semantics.
