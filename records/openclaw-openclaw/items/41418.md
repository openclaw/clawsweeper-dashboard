## Summary

Keep open: current main and the latest release still lack a global agent dry-run mode, and the requested no-side-effect guarantee crosses tool execution, Gateway, embedded fallback, hooks, delegated client tools, and delivery semantics. This is a feature/security-boundary design item, not a conservative cleanup close.

## What This Changes

The issue requests a global `openclaw agent --dry-run` mode that previews every agent tool call without executing it.

## Best Possible Solution

Define a request-scoped agent tool-preview contract first, then add CLI, Gateway protocol, SDK, embedded runner, adapter/client-tool, hook-ordering, delivery, docs, changelog, and regression coverage around that contract.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and release inspection give a high-confidence absence check: there is no `openclaw agent --dry-run` option, request/protocol/SDK dry-run field, or adapter dry-run interception path on current main or v2026.5.12.

## Solution Assessment

Unclear. Intercepting near the shared adapter is plausible, but the proposed process-global CLI-only shape is not the safest maintainable solution for Gateway, embedded fallback, hooks, delegated client tools, and delivery semantics.
