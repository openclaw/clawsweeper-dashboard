## Summary

Keep this issue open: current main and the latest release still lack a global agent dry-run mode, and the related dry-run/control-mode issues overlap without superseding this narrower agent tool-execution request.

## What This Changes

The issue requests a global `openclaw agent --dry-run` mode that keeps tools visible to the model but returns structured previews instead of executing tool calls.

## Best Possible Solution

Define a request-scoped agent execution-mode contract, then implement preview behavior at the canonical tool execution and delegation boundaries with coverage for built-in, plugin/channel, client-hosted, Gateway, and hook-order paths.

## Reproduction Assessment

Yes for the current source state: `openclaw agent` has no `--dry-run` flag and the shared adapter still executes tools. This is a source-proven feature gap rather than a failing bug reproduction.

## Solution Assessment

Unclear. Intercepting near the shared adapter is plausible, but the best solution needs a maintainer-approved request-scoped execution-mode contract that covers Gateway, client-hosted, plugin/channel, and hook behavior.
