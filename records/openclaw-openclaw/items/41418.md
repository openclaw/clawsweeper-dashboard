## Summary

Keep open: current main and the latest release still lack a global agent dry-run mode, and the no-side-effect guarantee needs maintainer product and security design across Gateway, embedded execution, hooks, delegated client tools, and reply delivery.

## What This Changes

The issue requests a global `openclaw agent --dry-run` mode that previews agent tool calls instead of executing them.

## Best Possible Solution

Define a request-scoped agent tool-preview contract first, then add CLI/protocol plumbing, adapter and client-tool behavior, docs, changelog, and regression coverage across Gateway and embedded execution.

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection on current main and `v2026.5.12` give high-confidence absence proof: no `openclaw agent --dry-run` option, request/protocol field, or adapter preview branch exists.

## Solution Assessment

Unclear. Intercepting near the tool adapter is plausible for OpenClaw-owned tools, but the proposed CLI-only process-global shape is not sufficient across Gateway, embedded fallback, hooks, delegated client tools, and reply delivery.
