## Summary

Keep open: current main still lacks a global agent dry-run mode, related dry-run threads are either narrower or broader rather than replacements, and the request needs maintainer design review for request-scoped no-side-effect semantics.

## What This Changes

Add a global `openclaw agent --dry-run` mode that previews tool calls instead of executing them.

## Best Possible Solution

Define a request-scoped agent dry-run contract, then implement CLI, Gateway protocol, embedded fallback, adapter preview behavior, docs, changelog, and focused regression coverage in alignment with the broader safe-mode work.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main confirms `openclaw agent` has no `--dry-run` option and the agent tool paths still execute or delegate tools rather than returning preview payloads.

## Solution Assessment

Unclear. Intercepting near the tool adapter is plausible, but a process-global CLI flag alone is not the best maintainable solution until Gateway, fallback, hooks, client tools, and delivery semantics are request-scoped.
