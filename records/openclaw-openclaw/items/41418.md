## Best Possible Solution

Define an additive, request-scoped global agent dry-run contract covering CLI, Gateway/embedded runs, built-in/plugin/client tools, before-tool-call ordering, output shape, logging, docs, and tests before implementation.

## Reproduction Assessment

## Solution Assessment

Unclear: the global dry-run capability fits the safety goal, but the proposed process-global adapter flag is not yet the best design. A request-scoped contract across CLI, Gateway, embedded, plugin, and client-tool paths is the safer maintainable shape.
