## Summary

Keep open. Current main and the latest release still drop the existing Telegram `markdown.tables` setting on the outbound chunker path, even though the table renderer and resolver already support the requested mode.

## What This Changes

The issue reports that Telegram outbound chunking ignores configured Markdown table conversion and sends raw pipe tables during chunked or streaming delivery.

## Best Possible Solution

Thread `cfg` and `accountId` through the existing outbound chunker context, have the Telegram chunker resolve the current table mode, and add focused channel/account tests for table conversion through chunked delivery.

## Reproduction Assessment

Yes, at source level. Current main shows the Telegram chunker and generic outbound planner never pass the existing `markdown.tables` config into `markdownToTelegramChunks`, so configured table conversion cannot affect this path.

## Solution Assessment

Yes. The existing resolver and renderer already implement the behavior; the maintainable fix is config propagation through the adapter chunker context, not a new Telegram option or a separate formatter.
