## Summary

Keep open: current main and the latest shipped release still drop Telegram table-mode config on the outbound chunker path, even though the docs and resolver support per-channel/per-account `markdown.tables`. The repair is narrow and source-reproducible.

## What This Changes

The issue asks Telegram outbound chunking and reply-payload sends to honor configured `markdown.tables` table mode for chunked Markdown-to-HTML delivery.

## Best Possible Solution

Implement an additive TypeScript-source fix that passes cfg/accountId through outbound chunking and applies resolved tableMode to Telegram Markdown-to-HTML chunk rendering, with regression tests for direct text and reply-payload sends.

## Reproduction Assessment

Yes by source inspection: the documented `markdown.tables` setting resolves through cfg/accountId, but the Telegram outbound chunker path never receives or forwards that context. I did not run a live Telegram scenario in this read-only review.

## Solution Assessment

Mostly yes: the issue's proposed data flow is the right fix, but it should be implemented in TypeScript source and the shared optional chunker context rather than by editing compiled output filenames.
