## What This Changes

The issue requests that Telegram outbound Markdown chunking honor the existing `channels.telegram.markdown.tables` setting during streamed or chunked sends.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/85098, or an equivalent TypeScript-source fix that forwards `cfg`/`accountId` through chunking and passes resolved `tableMode` into Telegram Markdown HTML chunks.

## Reproduction Assessment

Yes, by source inspection: the documented `markdown.tables` setting resolves from `cfg`/`accountId`, but the Telegram outbound chunker and payload paths never receive or forward that context. I did not run a live Telegram scenario in this read-only review.

## Solution Assessment

Yes, the maintainable fix is to thread optional config/account context through the existing chunker seam and forward `tableMode` to the Telegram formatter. The already-open linked PR follows that direction and should be reviewed rather than closing this issue early.
