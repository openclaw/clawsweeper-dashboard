## What This Changes

The issue asks for Telegram outbound chunking and streaming to honor per-channel/per-account `markdown.tables` instead of emitting raw Markdown table separators.

## Best Possible Solution

Review and land or close the linked PR; the durable fix should thread cfg/accountId through chunker context and pass resolved tableMode into Telegram Markdown-to-HTML chunks with focused regression coverage.

## Reproduction Assessment

Yes by source inspection: the documented `markdown.tables` setting resolves through cfg/accountId, but current Telegram outbound chunking only receives formatting and never forwards tableMode. I did not run a live Telegram scenario in this read-only review.

## Solution Assessment

Mostly yes: the issue's proposed data flow is the narrow maintainable fix, but it should be implemented in TypeScript source and the shared optional chunker context rather than compiled output filenames.
