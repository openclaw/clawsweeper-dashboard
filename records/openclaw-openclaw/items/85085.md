## Summary

Keep open: current main still ignores the documented Telegram `markdown.tables` setting on the outbound chunking path because the chunker and formatter do not carry or apply the cfg/account context needed to resolve table mode.

## What This Changes

The issue requests that Telegram outbound chunking and streaming honor per-channel or per-account `markdown.tables` configuration when converting Markdown tables for Telegram delivery.

## Best Possible Solution

Pass optional cfg/accountId through the shared outbound chunker context, resolve Telegram table mode at the adapter boundary, forward it through `markdownToTelegramHtmlChunks`, and add regression tests for direct outbound planning plus reply-payload sends.

## Reproduction Assessment

Yes by source inspection: the documented `markdown.tables` setting resolves through cfg/accountId, but the Telegram outbound chunker and reply-payload paths only pass formatting context and the HTML chunk wrapper drops tableMode. I did not run a live Telegram scenario in this read-only review.

## Solution Assessment

Yes, with one implementation caveat: the proposed data flow is the narrow maintainable fix, but it should be implemented in TypeScript source and the shared optional chunker context rather than by editing compiled output filenames.
