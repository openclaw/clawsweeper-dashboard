## Summary

Keep open: current main still lacks the stuck session-lane timeout, and this PR is the active implementation attempt, but the latest head still has ACP serialization and runtime-cache lifecycle blockers that should prevent merge pending maintainer design review.

## What This Changes

The PR adds configurable ACP session-lane task timeouts, threads them through manager/config schema/help surfaces, and adds regression coverage for hung lane tasks.

## Best Possible Solution

Keep the session-lane deadline direction, but implement it so deadlines cannot release same-session work before terminal bookkeeping is durable or the runtime handle is quarantined; preserve cache accounting until close is confirmed and cover initialize, turn, and idle-close timeout paths with focused tests.

## Reproduction Assessment

Yes. Source inspection shows a never-settling `runtime.ensureSession()` can run inside `withSessionActor()`, and `KeyedAsyncQueue` only advances the same-key lane when that promise settles; a manager test with a hung ensureSession can reproduce the stuck lane.

## Solution Assessment

No. The PR unblocks the lane by rejecting the queue task before the underlying ACP operation has stopped, so the narrow maintainable fix needs cancellation, stale fencing, terminal bookkeeping, and runtime-handle quarantine rather than early release alone.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve terminal bookkeeping on lane timeout:** `src/acp/control-plane/manager.core.ts:923-925`
  - body: When the lane timeout marks the operation stale, a prompt turn that already created a background task and set the session to `running` returns from this catch before `recordTurnCompletion()`, `markBackgroundTaskTerminal()`, and the error-state write run. The caller sees a timeout while durable task/session state can remain running, so later status/recovery paths inherit stale state.
  - confidence: 0.91
- **[P1] Quarantine handles while idle close is pending:** `src/acp/control-plane/manager.core.ts:1947-1948`
  - body: Wrapping idle eviction in the lane timeout lets this task reject while `runtime.close()` keeps running. Because the cache entry remains visible until close resolves, later same-session work can reuse or re-close a handle that is already terminating; add a closing/quarantine state or keep the lane held until the handle is retired.
  - confidence: 0.88
