## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve existing ACP turn timeouts:** `src/acp/control-plane/manager.core.ts:2252`
  - body: `runTurn()` also enters `withSessionActor()`, so resolving `taskTimeoutMs` inside the helper puts every ACP prompt turn under the new 600000 ms default. Existing turns previously followed `agents.defaults.timeoutSeconds` (48 hours by default), and the docs describe this setting as covering initialize/status/cancel/close serialization, so upgrading can break legitimate long-running ACP turns unless the timeout is separate, opt-in, or derived from the turn timeout.
  - confidence: 0.87
- **[P1] Keep stale close failures counted:** `src/acp/control-plane/manager.core.ts:2495`
  - body: When a timed-out close later rejects, the stale-operation branch clears the `closing` cache entry. That drops the runtime from `RuntimeCache.size()` even though close failed and the process/session may still be alive, so `maxConcurrentSessions` can admit new sessions while the old runtime is still consuming resources; keep or restore the counted entry until closure is confirmed or explicitly retired.
  - confidence: 0.84
