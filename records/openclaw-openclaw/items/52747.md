## Summary

Keep open: the underlying stuck-lane bug is valid on current main, but this PR should not merge as-is because its timeout path can leave task/session state running and can reuse runtime handles while close is pending.

## What This Changes

The PR adds a configurable ACP session-lane task timeout, wires it through manager/config schema/help surfaces, and adds regression coverage for hung lane tasks.

## Best Possible Solution

Keep the session-lane deadline direction, but redesign it so timeout reporting cannot release same-session work without cooperative cancellation or stale fencing that preserves terminal state, and quarantine runtime handles until close is confirmed.

## Reproduction Assessment

Yes. Source inspection shows current main can await runtime.ensureSession() inside withSessionActor(), and KeyedAsyncQueue advances same-key work only when that promise settles; I did not execute tests in this read-only review.

## Solution Assessment

No. The PR's timer race settles the queue promise before the underlying ACP operation necessarily stops, and the added stale fencing/cache handling leaves terminal state and runtime lifecycle races unresolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve terminal state on lane timeout:** `src/acp/control-plane/manager.core.ts:923-925`
  - body: When a lane timeout fires after a prompt turn has created a background task and set the session running, this stale-operation return skips recordTurnCompletion(), markBackgroundTaskTerminal(), and the session error-state write. The caller receives a timeout, but durable task/session state can remain running.
  - confidence: 0.91
- **[P1] Quarantine handles during idle close:** `src/acp/control-plane/manager.core.ts:1927-1930`
  - body: Idle eviction now runs close() under the lane timeout while the cached handle stays visible until close resolves. If the timeout releases the lane first, later same-session work can reuse or re-close a runtime that is already terminating; keep the lane held or mark the handle closing until it is retired.
  - confidence: 0.88

  - body: This adds a user-facing ACP fix and new config key, but the PR does not update CHANGELOG.md. Add an active-version changelog entry for the session-lane timeout behavior before merge.
