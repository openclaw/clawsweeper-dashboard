## Summary

Keep open: the PR addresses a real ACP session-lane wedge, but the current diff still has a blocking runtime accounting bug when a timed-out close later rejects.

## What This Changes

The PR adds configurable ACP session-lane task timeouts with cooperative abort/stale fencing, runtime-cache closing accounting, ACPX signal plumbing, config/docs/schema updates, and ACP manager regression tests.

## Best Possible Solution

Keep the session-lane deadline design, but keep timed-out closing runtimes counted until close succeeds or a confirmed termination/retry path retires them, with rejection-path regression coverage.

## Reproduction Assessment

Yes. Current main's same-key queue waits for a never-settling task and initializeSession awaits ensureSession without a lane deadline, while the PR body supplies terminal proof for the stuck-before and unblocked-after path.

## Solution Assessment

No. The timeout, abort, and stale-fencing direction is maintainable, but the stale close rejection path clears counted runtime state without proof of closure and should be fixed before merge.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Keep rejected stale closes counted:** `src/acp/control-plane/manager.core.ts:2495`
