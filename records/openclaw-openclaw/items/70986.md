## Summary

Keep open: current main still extracts Memory Wiki links from raw Markdown and validates only path-shaped targets, while the open closing PR is not merged and has review blockers.

## What This Changes

The issue asks for Memory Wiki lint to ignore wikilink-shaped text in Markdown non-link regions and resolve Obsidian links by frontmatter title, aliases, and normalized target forms.

## Best Possible Solution

Land a single Memory Wiki-local fix that uses delimiter-aware Markdown masking for extraction and one normalized title/alias/path resolver target set, then close this issue after the fix merges and ships.

## Reproduction Assessment

Yes. Source inspection on current main shows raw Markdown reaches `extractWikiLinks`, and the resolver target set lacks title or alias keys; the issue discussion also includes multiple live vault reproductions.

## Solution Assessment

Yes on direction, but not yet landed. A generic Markdown-aware extractor plus normalized metadata-backed resolver keys is the maintainable fix; tag-specific filtering would leave fenced code and title-link false positives behind.
