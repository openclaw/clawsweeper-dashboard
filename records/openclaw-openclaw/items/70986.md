## What This Changes

The issue requests Memory Wiki lint fixes so Markdown code/comment/frontmatter text is not parsed as wikilinks and title-style Obsidian links resolve against page metadata.

## Best Possible Solution

Land a Memory Wiki-local Markdown-aware extractor fix and coordinate the resolver/title normalization half through the active resolver PRs or a single canonical replacement, then close this issue once both behaviors are on `main` and shipped.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: raw `[[...]]` in fenced or inline code still reaches `extractWikiLinks`, and title-style targets still miss because lint indexes only path-shaped target keys.

## Solution Assessment

Yes. A generic Markdown-aware extractor plus normalized metadata-backed resolver keys is the narrow maintainable fix; tag-specific reply filtering or content rewrites would leave fenced shell/code examples and title links broken.
