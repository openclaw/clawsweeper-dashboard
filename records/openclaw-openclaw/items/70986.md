Keep open: current `main` and latest release still expose the two central Memory Wiki lint bugs. ## What This Changes

The issue requests fixing Memory Wiki lint false positives by making wikilink extraction Markdown-aware and resolving wikilinks against frontmatter titles or normalized aliases.

## Best Possible Solution

Land a Memory Wiki-local fix that strips or scans Markdown structure before extracting wikilinks and uses metadata-backed normalized target keys for lint resolution, coordinating the resolver half with the active title-normalization PRs.

## Reproduction Assessment

Yes, at source level. Raw `[[...]]` inside fenced or inline code still reaches `extractWikiLinks`, and title-style targets still miss because `collectBrokenLinkIssues` indexes only path-shaped keys.

## Solution Assessment
