Keep open: current main still has the two reported memory-wiki lint failure modes, with only the narrower `.md` relative-target case addressed. The issue comments add independent live-vault evidence, and the related open PRs cover resolver/tag-specific slices rather than a merged end-to-end fix.

## What This Changes

The issue asks for `openclaw wiki lint` to ignore Markdown/code literals when extracting wikilinks and to resolve links against frontmatter titles or normalized page metadata.

## Best Possible Solution

Land a memory-wiki-local fix that combines Markdown-aware wikilink extraction with normalized title/alias/path target resolution, coordinating the resolver half with the active related PRs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: raw `[[...]]` in code/prose still reaches `extractWikiLinks`, and title-style targets still miss `collectBrokenLinkIssues` because only path-shaped keys are indexed.

## Solution Assessment

Yes. A generic Markdown-aware extraction pass plus normalized metadata-based resolver is the narrow maintainable fix; tag-specific reply filtering would leave bash/code literals and title links broken.

Fix the Markdown-aware wikilink extraction part of https://github.com/openclaw/openclaw/issues/70986. In `extensions/memory-wiki/src/markdown.ts`, make `extractWikiLinks` skip YAML frontmatter, fenced code blocks using backticks or tildes, inline code spans, HTML comments, and the existing managed Related block before applying Obsidian/Markdown link regexes. Add focused regression coverage proving `[[reply_to_current]]` and bash `[[ ... ]]` inside fenced or inline code do not become `broken-wikilink` targets, while real Obsidian wikilinks still extract. Do not add tag-specific allowlists, do not change bridge rendering, and do not duplicate title/alias resolver work from https://github.com/openclaw/openclaw/pull/80915 or https://github.com/openclaw/openclaw/pull/73722 unless those branches have merged and the implementation must adapt. Add a `CHANGELOG.md` Fixes entry for the user-visible lint behavior change.
