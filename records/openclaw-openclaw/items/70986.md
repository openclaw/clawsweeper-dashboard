## What This Changes

The issue asks Memory Wiki lint to ignore non-link Markdown `[[...]]` literals and resolve wikilinks against frontmatter titles or normalized target keys.

## Best Possible Solution

Land a Memory Wiki-local fix that makes link extraction Markdown-aware and adds regression coverage, while reviewing or consolidating the open resolver-normalization PRs for title and alias targets.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: raw `[[...]]` in code-like Markdown still reaches `extractWikiLinks`, and title-style targets still miss because lint indexes only path-shaped keys.

## Solution Assessment

Yes. A generic Markdown-aware extraction pass plus metadata-backed target normalization is the narrow maintainable fix; tag-specific reply filtering would leave fenced shell/code literals and title links broken.
