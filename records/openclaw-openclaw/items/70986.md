## What This Changes

The issue requests Markdown-aware Memory Wiki wikilink extraction and resolver support for frontmatter-title Obsidian links.

## Best Possible Solution

Land a Memory Wiki-local fix that strips or scans Markdown structure before link extraction and adds normalized title/alias/path resolver keys, coordinating the resolver half with the active related PRs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: raw `[[...]]` in code or inline prose still reaches `extractWikiLinks`, and title-style targets still miss lint because `collectBrokenLinkIssues` indexes only path-shaped keys.

## Solution Assessment

Yes. A generic Markdown-aware extraction pass plus metadata-backed target normalization is the narrow maintainable fix; tag-specific reply filtering would leave fenced bash/code literals and title links broken.
