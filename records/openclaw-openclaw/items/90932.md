## Summary

Keep open: this PR targets a real Control UI bug, but the current patch still lets an explicit scoped avatar clear fall back to the shipped legacy flat key, and the supplied proof is helper-level jsdom output rather than a real Control UI browser run. ## What This Changes

The branch scopes Control UI local assistant avatar localStorage reads and writes by agent ID, threads the agent ID through render/bootstrap/identity callers, adds legacy flat-key fallback, and updates focused tests.

## Best Possible Solution

Land one canonical per-agent assistant-avatar storage fix that preserves legacy flat values on upgrade while blocking fallback after an explicit scoped clear, with live Control UI proof for upload, switch, clear, and reload/bootstrap.

## Reproduction Assessment

## Solution Assessment

No, not as currently patched. Agent-scoped storage is the right layer, but this branch needs explicit scoped-clear semantics, live browser proof, and canonical-branch selection before it is the best merge path.
