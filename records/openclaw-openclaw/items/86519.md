Keep open: the Telegram-specific mitigation in https://github.com/openclaw/openclaw/pull/87657 does not cover the broader core preview/final reply duplication described in this issue, and current main plus the latest release still preserve final payloads after partial previews.

The issue reports that long streamed agent replies are delivered multiple times across Telegram, WhatsApp, and WebChat after the 2026.5.20 update.

## Best Possible Solution

Restore an approved core or shared delivery invariant that suppresses only final text already delivered via partial preview, while preserving genuinely new final text, media, block-streaming fallback, and channel-specific finalization behavior.
