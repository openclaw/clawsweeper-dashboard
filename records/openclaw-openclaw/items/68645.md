## Summary

Keep this PR open. The reported cron task-ledger bug is plausible and the PR body includes useful production hot-patch evidence, but the branch is draft/conflicting and the proposed implementation is based on older timer contracts, leaves the new public timeout field partly unwired, and changes existing no-timeout semantics.

## What This Changes

The PR adds a cron agent-turn wall-clock timeout, timeout-path task-ledger closing, and regression tests for stale cron task cleanup.

## Best Possible Solution

Rebase onto current main, layer the wall-clock ceiling into the existing phase-aware timeout cleanup flow, wire the field through cron schemas/tools/docs/update merging, and either preserve or explicitly decide the no-timeout semantics.

## Reproduction Assessment

Yes, source-reproducible: current main has no cron runTimeoutSeconds path, and the proposed fake-timer isolated-runner scenario exercises the missing wall-clock ceiling when a runner ignores abort. The PR body also reports production stale/lost task counts and post-hot-patch improvement.

## Solution Assessment

No, not as currently proposed. The direction is useful, but the branch needs a current-main rebase, full public contract wiring, and a maintainer decision on default timeout semantics before it is the narrow maintainable fix.
