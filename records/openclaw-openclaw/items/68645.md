## Summary

Keep this PR open: the reported cron timeout/task-ledger problem is real and the direction is useful, but the draft branch is stale against current main and would need a careful rebase plus full cron contract wiring before merge.

## What This Changes

The PR adds a cron `agentTurn` `runTimeoutSeconds` wall-clock ceiling, proactively closes timed-out task-ledger rows, and adds timer regression tests.

## Best Possible Solution

Rebase onto current main, layer the wall-clock ceiling into the existing timeout cleanup flow, settle the default policy, and wire the field through schemas, cron tool docs, update merging, and focused tests.

## Reproduction Assessment

Yes, source-reproducible: current main has no cron `runTimeoutSeconds` path, while the PR body gives production stale/lost task counts and after-hot-patch improvement. A fake-timer isolated runner that ignores abort would exercise the missing wall-clock kill path without needing live credentials.

## Solution Assessment

No, not as currently proposed. The branch needs to preserve current timeout cleanup semantics and wire the new field through the full public cron contract before this is the maintainable fix.
