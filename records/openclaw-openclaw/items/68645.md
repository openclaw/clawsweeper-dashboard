## Summary

Keep this PR open: it targets a credible cron lifecycle gap with production context, but the branch is not ready because it is based on older timeout code and does not wire the new public cron timeout field through the full contract surface.

## What This Changes

The PR adds cron agentTurn wall-clock timeout handling, proactive task-ledger timeout cleanup, and timer regression tests.

## Best Possible Solution

Rebase onto current main, integrate the wall-clock ceiling into the existing watchdog and cleanup flow, and wire the cron contract through schemas, tools, docs, update merging, and focused regression tests.

## Reproduction Assessment

Yes, source-reproducible: current main has no cron runTimeoutSeconds path, and the PR's unresponsive fake runner models the missing wall-clock ceiling when an isolated run ignores abort. The PR body also reports production stale-running rows clearing after the hot patch.

## Solution Assessment

No, not as currently proposed. The direction is useful, but this patch must preserve current main's phase-aware timeout cleanup and wire the new public cron field end to end before merge.
