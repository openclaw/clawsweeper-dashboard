## Summary

Keep open: this is a plausible cron lifecycle bug fix with useful production context and supportive related comments, but the draft branch is merge-conflicting and currently drops newer main timeout cleanup behavior while adding an incompletely wired public timeout field.

## What This Changes

The PR adds cron agentTurn `runTimeoutSeconds` wall-clock timeout handling, proactive task-ledger finalization on timer errors, normalization/type support, and regression tests.

## Best Possible Solution

Rebase the PR onto current main, integrate wall-clock run cancellation into the existing phase-aware watchdog and cleanup flow, wire the public field through schema/tool/docs/update normalization, and add focused regression coverage for both ledger closure and the public contract.

## Reproduction Assessment

Yes. The source path is clear: current main has no `runTimeoutSeconds` cron field, and the PR's unresponsive-runner test models the missing guard while the PR body reports production stale-running rows clearing after the hot patch.

## Solution Assessment

No, not as currently proposed. The direction is useful, but the patch must preserve current main's timeout cleanup path and wire the new public cron option end to end before it is mergeable.
