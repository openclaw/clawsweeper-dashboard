## What This Changes

Prevent agent responses during inter-session communication from creating repeated cross-session message loops visible in webchat.

## Best Possible Solution

## Solution Assessment

No, a webchat-specific provenance filter or new per-agent config is not the best fix; reverse callbacks should be prevented or deduplicated in the core sessions_send orchestration owner while preserving intended bounded replies.
