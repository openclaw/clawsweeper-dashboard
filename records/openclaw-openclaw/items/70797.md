## Summary

Keep open. Current main still has the reported leak class: bare OpenAI-style `tool_calls` JSON is not stripped by the shared assistant-visible sanitizer, and the Telegram partial-preview path can still forward the resulting text; the two directly linked repair PRs are closed unmerged and no open exact fixing PR was found.

## What This Changes

The issue reports that raw internal `tool_calls` JSON can appear in Telegram-visible replies during background `exec` followed by `process poll` flows.

## Best Possible Solution

Add a shared, streaming-aware assistant-visible guard for bare OpenAI-style `tool_calls` JSON, apply it before both final visible delivery and partial preview callbacks, and cover complete, incomplete, inline, fenced, and ordinary JSON cases with focused regressions.

## Reproduction Assessment

Yes. The live race is not fully minimized, but the source-level reproduction is high-confidence: a bare `{"tool_calls":[...]}` payload bypasses the current sanitizer guard and can flow through `handleMessageUpdate()` into Telegram `onPartialReply`.

## Solution Assessment

Yes. A narrow shared sanitizer plus preview-path guard is the best maintainable fix; provider-specific tinyrouter workarounds or attempting to recover execution from downgraded assistant text would be broader and less reliable.
