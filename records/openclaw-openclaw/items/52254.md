## Summary

Keep open: current main still has untranslated English display copy in the named Control UI surfaces even though zh-CN lazy loading and partial tab localization are present. Related open localization PRs cover Skills and Nodes slices, but they are not merged and do not fully supersede this issue.

## What This Changes

The issue requests full zh-CN localization of internal Control UI content for Agents, Skills, Channels, Sessions, Nodes, and Debug after selecting Simplified Chinese.

## Best Possible Solution

Land or consolidate focused tab display-copy i18n changes through the existing en.ts plus generated zh-CN pipeline, while preserving raw identifiers and debug payloads.

## Reproduction Assessment

Yes. Source inspection on current main shows zh-CN is registered and partially populated, while Skills, Nodes, and exec approvals still render hardcoded English display strings that would remain English after selecting Simplified Chinese.

## Solution Assessment

Yes, for the actionable portion. The maintainable fix is to add locale keys and route display chrome through t(), not to translate raw gateway identifiers, command names, debug payloads, or generated data in place.
