## Summary

Keep open: current main has lazy-loaded zh-CN locale support and some translated tab namespaces, but several modules named in the report still render hard-coded English strings, especially Skills and Nodes.

## What This Changes

Request that Control UI lazy-loaded tab contents display zh-CN translations instead of English in Agents, Skills, Channels, Sessions, Nodes, and Debug.

## Best Possible Solution

Coordinate the open Control UI i18n PRs, move remaining user-facing raw strings from the affected tab views into en.ts, regenerate supported locale bundles through scripts/control-ui-i18n.ts, and keep intentional English terms documented in the glossary or raw-copy baseline.

## Reproduction Assessment

Yes by source inspection: zh-CN lazy loading is wired, but current-main Skills and Nodes views still render hard-coded English strings that bypass the locale bundle. I did not run a live browser session in this read-only review.

## Solution Assessment

No, closing is not the best outcome because the requested behavior is only partially implemented. The maintainable path is a scoped i18n cleanup coordinated with the existing partial Control UI localization PRs.
