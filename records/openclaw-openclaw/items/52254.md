Keep open. Current main still has observable zh-CN localization gaps in the reported lazy Control UI surfaces: several views render user-facing English literals directly, and the raw-copy baseline currently accepts those strings instead of forcing them through the i18n source map.

## Best Possible Solution

Move remaining user-facing copy in the reported lazy tabs into ui/src/i18n/locales/en.ts, replace render-site literals with t(...), regenerate locale outputs and metadata through the Control UI i18n pipeline, and leave only intentional product/protocol identifiers in the raw-copy baseline.
