Keep open: current main and the latest shipped release still have hardcoded English display copy in the Control UI Skills and Nodes/exec approvals surfaces after zh-CN is selected. Several related localization PRs cover slices of this issue, but they are open and partial, so they are implementation candidates rather than cleanup-close evidence.

## What This Changes

The issue requests complete zh-CN localization for module-internal Control UI copy in Agents, Skills, Channels, Sessions, Nodes, and Debug after selecting Simplified Chinese.

## Best Possible Solution

Land or consolidate focused display-copy i18n changes through the existing en.ts plus generated locale pipeline, while preserving raw identifiers, brands, command names, and debug payloads.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.5.19 shows that selecting zh-CN still leaves Skills, Nodes, and exec approvals display copy rendered from hardcoded English strings.

## Solution Assessment

Yes for keeping this open. The maintainable fix is to add display-copy locale keys and regenerate locale outputs, not to hand-edit zh-CN generated files or translate raw runtime identifiers in place.
