## Summary

Keep this issue open. Current main has improved parts of the reported surface, especially Sessions and Debug, but the observable zh-CN problem remains because several reported lazy Control UI areas still render English literals directly or ship generated zh-CN entries that remain English.

## What This Changes

The issue requests complete Simplified Chinese localization for Control UI lazy-loaded tabs whose inner UI still appears in English.

## Best Possible Solution

Move the remaining user-facing copy in the reported lazy Control UI tabs and their child panels into ui/src/i18n/locales/en.ts, remove those strings from the raw-copy baseline where they are not intentional, regenerate locale bundles through the Control UI i18n pipeline, and add focused coverage that exercises zh-CN or catches raw user-facing literals in the affected views.

## Solution Assessment
