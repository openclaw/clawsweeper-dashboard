## Summary

Keep open: current main still routes reasoning-only and empty-response retries through generic visible-answer continuation prompts without a latest-inbound-turn anchor, and the production confirmation narrows the failure to the embedded runner retry path rather than Telegram delivery ordering. The broader related tracker remains open, but this issue is the more focused repair boundary for retry anchoring.

## What This Changes

Ensure reasoning-only, empty-response, and tool-use continuations answer the latest inbound user message instead of older nearby intent.

## Best Possible Solution

Add a generic embedded-runner latest-turn anchor through reasoning-only and empty-response retry construction, with regression coverage that prevents a retry continuation from answering an older nearby user turn.
