## Summary

Keep open: current main still has generic reasoning-only and empty-response visible-answer retry prompts without a latest-message/run-intent anchor, and the production confirmation narrows this to the embedded-runner retry path rather than Telegram delivery ordering. ## What This Changes

The issue reports stale assistant replies after reasoning-only or tool retry continuations, where the final answer can target an older nearby user message instead of the latest inbound turn.

## Best Possible Solution

Add a generic embedded-runner latest-turn anchor through reasoning-only and empty-response retry construction, with regression coverage that prevents retry continuations from answering an older nearby user turn.
