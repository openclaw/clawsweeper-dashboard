## Summary

Keep open: current main still shows the generic reasoning-only and empty-response visible-answer retry path without a latest-turn anchor, and the production confirmation narrows this beyond a stale Telegram delivery-order report.

## What This Changes

The issue reports that Telegram direct-chat replies can answer an older user intent after a tool-use or reasoning-only continuation even though the latest inbound message is already in session history.

## Best Possible Solution

Add a generic embedded-runner latest-turn anchor through reasoning-only and empty-response retry prompt construction, backed by a regression that prevents a replay-safe tool/retry continuation from answering an older nearby user turn.

## Reproduction Assessment

No deterministic live reproduction was established in this review. The report has a concrete Telegram sequence, a production confirmation, and current source still exposes the generic retry path that can plausibly lose the latest-user anchor.

## Solution Assessment
