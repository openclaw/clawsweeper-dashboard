Keep open: current main still has generic visible-answer retry instructions without a latest-turn anchor, and the production confirmation narrows this to the embedded-runner retry path rather than Telegram delivery ordering.

## What This Changes

The issue reports that Telegram direct-chat final replies can answer an older user intent after reasoning-only or empty-response continuation retries instead of the latest inbound message.

## Best Possible Solution

Add a generic embedded-runner latest-turn anchor through reasoning-only and empty-response retry construction, with regression coverage that prevents a retry continuation from answering an older nearby user turn.
