## Summary

Keep open: current main and v2026.5.7 still use generic visible-answer continuation text for reasoning-only/empty retries, and I found no latest-inbound anchor or regression that would rule out the confirmed stale-intent path.

## What This Changes

The issue asks OpenClaw to prevent reasoning-only or tool-continuation retries from producing a visible final answer for an older user message instead of the latest inbound turn.

## Best Possible Solution

Add a core embedded-runner latest-turn anchor for visible-answer retry construction and final delivery, backed by a regression that prevents older nearby intent from being answered after a tool/reasoning-only continuation.

## Reproduction Assessment

No. The production reports and current source give a credible source-reproducible path, but I did not establish a deterministic current-main Telegram replay or failing test.

## Solution Assessment

Yes in direction. The maintainable fix belongs in the generic embedded-runner retry/session boundary, not in Telegram ordering alone, and it should preserve replay safety while anchoring the retry to the latest inbound turn.
