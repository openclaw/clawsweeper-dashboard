## Summary

Keep open. Current main still uses fence-only safe-break logic in the shared block streaming chunker, so markdown tables can still be split before channel-level table conversion; the linked open PRs are implementation candidates, not shipped behavior.

## What This Changes

Prevent block streaming from splitting markdown tables across separate channel messages.

## Best Possible Solution

Land a narrow shared chunker fix that adds table span detection and table-safe guards for paragraph, newline, whitespace, and max-length break paths, plus the #66605 integration test, then close this as implemented.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to use EmbeddedBlockChunker with flushOnParagraph and append `Intro\n\n| A | B |\n|---|---|\n| 1 | 2 |\n\nOutro`; current main will treat the table-adjacent blank lines as safe because only fence spans are checked.

## Solution Assessment

No current-main solution is present. The maintainable fix belongs in the shared block chunker/markdown span layer; channel-level table conversion is too late because chunk boundaries have already been selected.
