## Summary

Keep open. Current main still has fence-only safe-break handling in the shared block streaming chunker, and the open related PRs are implementation candidates rather than shipped behavior.

## What This Changes

Requests table-aware block streaming chunk boundaries so markdown tables stay intact instead of being split across separate channel messages.

## Best Possible Solution

Land one narrow shared chunker and markdown-span implementation with unit coverage plus the #66605 integration case, then close this issue after the fix merges.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to instantiate EmbeddedBlockChunker with flushOnParagraph, append `Intro\n\n| A | B |\n|---|---|\n| 1 | 2 |\n\nOutro`, and observe that current main treats the table-adjacent blank lines as safe because only fence spans are checked.

## Solution Assessment

Yes. The best fix direction is table-span safe-break logic in the shared block chunker; channel-level convertMarkdownTables handling is too late because the chunker has already selected message boundaries.
