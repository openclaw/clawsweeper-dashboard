## Summary

Keep open. Current main still has fence-only safe-break handling in the shared block streaming chunker, with no markdown table span parser or table-safe boundary guard; related open PRs are implementation candidates rather than shipped behavior.

## What This Changes

The issue requests that block streaming keep markdown tables intact across streaming channel messages instead of splitting on paragraph breaks around or inside tables.

## Best Possible Solution

Land a narrow shared chunker/table-span fix with the #66605 integration coverage, reconciling #66344 and #73981 while preserving useful credit from #66568.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to instantiate EmbeddedBlockChunker with flushOnParagraph and append text like `Intro\n\n| A | B |\n|---|---|\n| 1 | 2 |\n\nOutro`; current main treats table-adjacent blank lines as safe because only fence spans are checked.

## Solution Assessment

No current-main solution is present. The maintainable fix is table-span safe-break logic in the shared block chunker plus regression tests; channel-level convertMarkdownTables handling is too late because the chunker has already chosen message boundaries.
