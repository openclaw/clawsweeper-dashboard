## Summary

Keep open. Current main and the latest release still use fence-only safe-break handling in the shared block streaming chunker, table-specific integration coverage is absent, and the open fix path is already represented by https://github.com/openclaw/openclaw/pull/73981.

## What This Changes

Block streaming should keep markdown tables intact across streaming message chunks instead of splitting them at paragraph boundaries.

## Best Possible Solution

Land one shared block-streaming fix that detects protected markdown table spans before choosing chunk boundaries, adds the subscribeEmbeddedPiSession table integration case tracked by https://github.com/openclaw/openclaw/issues/66605, and preserves credit from https://github.com/openclaw/openclaw/pull/66568.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to feed `Intro\n\n| A | B |\n|---|---|\n| 1 | 2 |\n\nOutro` through EmbeddedBlockChunker with flushOnParagraph enabled; current main only checks fence safety, so the table-adjacent paragraph breaks are not protected.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the shared block chunker/table-span layer because channel-level markdown table conversion runs after the streaming chunker has already chosen message boundaries.
