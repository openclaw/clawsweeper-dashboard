## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Restore the public memory-core SDK subpath:** `src/plugin-sdk/entrypoints.ts:61`
  - body: This deletes `openclaw/plugin-sdk/memory-core-host-embedding-registry` from the published package surface. That subpath is currently listed as a public plugin-owned SDK entrypoint, so existing third-party plugins importing it will fail to resolve after upgrade; keep a re-export shim with a deprecation path or get explicit maintainer approval for the breaking removal.
  - confidence: 0.92
- **[P1] Keep legacy cron jobs waking when enabled is missing:** `src/cron/service/jobs.ts:693`
  - body: `nextWakeAtMs` now checks `j.enabled` directly, so persisted jobs that predate an explicit `enabled` field are filtered out and will not schedule a wake even though `isJobEnabled` still defines missing as enabled. The removed test covered exactly this upgrade path; use `isJobEnabled(j)` here and keep the regression coverage.
  - confidence: 0.95
- **[P1] Preserve failure-destination clear sentinels:** `src/cron/store/delivery-codec.ts:35-38`
  - body: The SQLite codec now writes `null` for explicit undefined failure-destination fields, which loses the own-property clear signal used to opt out of inherited global failure destinations. After save/load, those jobs can inherit and send failure notifications to a destination the user explicitly cleared, so keep the sentinel encoding or add a tested migration-safe replacement.
  - confidence: 0.9
- **[P1] Make peer relinking atomic:** `src/plugins/plugin-peer-link.ts:257-258`
  - body: When a copied `node_modules/openclaw` directory is present, this removes it before attempting the replacement symlink. If symlink creation is denied or fails, the catch only reports `skipped` and leaves the plugin without the package it previously had; move/restore the directory or link through a temporary path before replacing it.
  - confidence: 0.93
