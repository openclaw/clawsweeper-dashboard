## Summary

Keep open. Current main still drops Telegram URL-only presentation buttons in the Telegram plugin, while related PR #74176 is open but unmerged and broadens the behavior into Mini App `web_app` semantics that need maintainer review.

## What This Changes

Telegram URL inline keyboard buttons should render from shared presentation buttons instead of being silently dropped.

## Best Possible Solution

Fix this in the Telegram plugin by carrying shared `url` buttons through the Telegram button type, interactive mapper, and inline keyboard serializer, with explicit tests for URL-only and value-plus-url precedence; avoid reintroducing raw `buttons` on the generic message tool.

## Reproduction Assessment

Yes. A URL-only presentation button is preserved by the shared payload layer, then dropped by `buildTelegramInteractiveButtons` because it has no `value`, and any URL-shaped Telegram button would also be filtered out by `buildInlineKeyboard` because it lacks `callback_data`.

## Solution Assessment

No, the one-line filter change suggested in the issue is not sufficient. The safer fix needs the Telegram inline button type and mapper to represent `url` or a chosen Telegram action field, then serialize exactly one Bot API action field per button.
