## Summary

Keep open. Current main still drops Telegram URL-only presentation buttons, and the issue is now paired with open closing PR #76264, so it should remain open until that PR is reviewed and either landed or replaced.

## What This Changes

Request to preserve Telegram inline keyboard URL buttons from shared presentation instead of treating Telegram buttons as callback-only.

## Best Possible Solution

Land a Telegram-plugin fix that carries `url` through the Telegram button type, interactive mapper, and inline keyboard serializer with focused tests, while keeping the generic message-tool surface centered on shared `presentation` unless maintainers explicitly expand it.

## Reproduction Assessment

Yes. A URL-only presentation button is preserved by the shared payload layer, then dropped by `buildTelegramInteractiveButtons` because it has no `value`; any URL-shaped Telegram button would also be filtered out by `buildInlineKeyboard` because it lacks `callback_data`.

## Solution Assessment

No, the issue's one-line filter change is not sufficient by itself because current main cannot produce URL Telegram buttons from shared presentation. The narrow maintainable fix needs the Telegram button type, mapper, serializer, and tests to agree on `url` versus `callback_data` precedence.
