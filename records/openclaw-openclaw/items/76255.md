## Summary

Keep open. Current main still drops URL-only Telegram presentation buttons, and the linked closing PR #76264 is open and unmerged, so the issue should remain open until that fix or an equivalent narrow fix lands.

## What This Changes

Send Telegram inline keyboard link buttons from shared message presentation instead of dropping URL-only buttons.

## Best Possible Solution

Land a narrow Telegram plugin fix that carries `url` buttons through the Telegram button type, mapper, inline keyboard serializer, and regression tests; avoid expanding deprecated raw button schema unless a maintainer deliberately approves it.

## Reproduction Assessment

Yes. A source-only reproduction is clear on current main: a presentation button with `label` and `url` is preserved by shared normalization, then dropped by Telegram because the mapper requires `value` and the serializer requires `callback_data`.

## Solution Assessment

No for the issue's serializer-only suggested fix; that would still miss the earlier Telegram mapper drop. The maintainable fix is to update the Telegram button type, mapper, serializer, and focused tests, then land #76264 if it matches that boundary.
