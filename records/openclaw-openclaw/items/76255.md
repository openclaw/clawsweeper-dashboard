## Summary

Keep open. Current main still drops URL-only Telegram presentation buttons, and #76264 is open and unmerged, so this issue should remain open until that narrow fix or an equivalent one lands.

## What This Changes

The issue asks Telegram to preserve shared presentation URL inline keyboard buttons instead of dropping them when no callback data is present.

## Best Possible Solution

Carry `url` through the Telegram button type, shared-button mapper, inline keyboard serializer, and focused regression tests, then land #76264 or an equivalent narrow fix.

## Reproduction Assessment

Yes. A source-only path on current main is high-confidence: a `{ label, url }` presentation button is accepted and preserved by shared presentation code, then Telegram drops it because the mapper requires `value` and the serializer requires `callback_data`.

## Solution Assessment

No for the issue's serializer-only suggested fix. The maintainable fix must update the Telegram button type, mapper, serializer, and regression coverage; raw message-tool button expansion should stay separate.
