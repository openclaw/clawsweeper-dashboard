## Summary

Keep open. Current main contains useful partial work for model fallback visibility, status output, and cron delivery validation, but it still does not satisfy the production-operations requests for startup primary-model reachability, a compact agent-readable runtime diagnostics surface, and cron add delivery-target preflight/dry-run before persistence.

## What This Changes

The issue requests production hardening around startup model health checks, agent-readable runtime diagnostics, stricter task execution discipline, cron add validation/dry-run behavior, and clearer error propagation.

## Best Possible Solution

Keep this open as the production execution-discipline tracker, or split it into scoped follow-ups for startup provider reachability/degraded mode, a safe structured runtime diagnostics RPC/tool, and cron add dry-run/delivery-target preflight.

## Reproduction Assessment

Partially yes. Current source gives a high-confidence reproduction of the missing gates: startup prewarm is explicitly non-blocking, diagnostics lacks the requested structured model/fallback/error fields, and cron add does not resolve the announce target before persistence.

## Solution Assessment

No. Treating this as one automated fix is not the best path because the remaining work crosses startup policy, a security-sensitive diagnostic contract, and cron create semantics; maintainer scoping or split follow-ups are safer.
