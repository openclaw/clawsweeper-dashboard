## Summary

Keep open. Current main has partial observability and cron improvements, but it still lacks blocking/degraded startup primary-model reachability, a compact agent-readable runtime model/fallback/last-error diagnostics contract, and cron add dry-run or concrete delivery-target preflight before persistence.

## What This Changes

The issue asks for production safeguards around startup model health, fallback visibility, agent self-diagnostics, cron creation validation/dry-run, and clearer pre-context errors.

## Best Possible Solution

Define or split the remaining contracts into scoped follow-ups for startup degraded-mode/provider reachability, a safe structured runtime diagnostics RPC/tool, and cron add dry-run plus delivery-target preflight.

## Reproduction Assessment

Partially yes. Source inspection on current main shows the missing gates directly: nonblocking metadata-only startup prewarm, diagnostics.stability only, and cron add persistence without concrete target preflight.

## Solution Assessment

No as a single automated fix. The maintainable path is to scope the product contracts first, because the requested behavior crosses startup policy, sandbox-safe diagnostics, and cron create semantics.
