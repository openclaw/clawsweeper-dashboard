## Summary

Keep open: current main still has a plausible dashboard/runtime mismatch, because MCP tools are discovered during embedded agent run materialization but not by the Control UI catalog/effective inventory path that feeds the reported tool count.

## What This Changes

The issue requests that configured remote MCP tools be discovered with `tools/list` and shown in the Agent tools dashboard after a successful MCP handshake.

## Best Possible Solution

Define a single MCP availability contract for the Tools panel: either include session bundle-MCP discovery with status/errors, or explicitly show configured MCP servers separately from runtime-discovered tools.

## Reproduction Assessment

Yes for the dashboard symptom by source inspection: the UI uses `tools.catalog` and `tools.effective`, and that path does not materialize configured MCP servers or call `tools/list`. The exact remote SSE setup still lacks the documented `mcp.servers` config and logs.

## Solution Assessment

Unclear. The narrowest maintainable solution depends on whether maintainers want `tools.effective` to perform live remote MCP discovery or want a UI/docs distinction between configured MCP servers and runtime materialization.
