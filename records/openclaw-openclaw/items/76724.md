## Summary

Keep open: current main still has runtime MCP discovery during embedded agent turns, but the Control UI `tools.effective` inventory path does not materialize configured MCP tools, so the reported dashboard symptom is plausible. The reporter's `workspace/config/mcporter.json` path is not the documented OpenClaw MCP registry, so their exact setup also needs config/log confirmation.

## What This Changes

The issue reports that a configured SSE MCP server initializes successfully but its tools are not discovered or shown in the Agent Tools dashboard.

## Best Possible Solution

Define one clear contract: either include bundle-MCP discovery in the session-scoped Tools panel with caching and redacted errors, or route MCP visibility to a dedicated diagnostics/status surface and update the dashboard/docs accordingly.

## Reproduction Assessment

Yes for the dashboard symptom by source inspection: `tools.effective` does not materialize bundle-MCP tools, while agent turns do. No for the reporter's exact Flask SSE setup because the issue lacks the actual `mcp.servers` config and logs.

## Solution Assessment

Unclear. The narrow fix is likely in `tools.effective` or the Tools panel, but maintainers should first decide whether live remote MCP discovery belongs in that endpoint or in the broader MCP diagnostics/status surface tracked by #70309.
