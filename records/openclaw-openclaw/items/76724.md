## Summary

Keep open: current main has runtime MCP discovery for agent turns, but the Control UI tool inventory path still does not materialize configured bundle-MCP servers, so the reported dashboard behavior remains plausible. The exact reporter setup also needs config confirmation because `mcporter.json` is not the documented OpenClaw MCP registry path.

## What This Changes

The issue reports that an SSE MCP server initializes successfully but its tools are not listed in the Agent tools dashboard or requested via `tools/list`.

## Best Possible Solution

Define one MCP availability contract for the Tools panel: either materialize and show discovered `bundle-mcp` tools with status/errors, or show configured MCP servers separately and document that discovery happens during agent-run materialization.

## Reproduction Assessment

Yes for the dashboard symptom by source inspection: `tools.effective` builds from `createOpenClawCodingTools(...)`, while bundle-MCP discovery is only materialized in agent-run code. The exact SSE server setup is not fully reproduced because the issue lacks the OpenClaw `mcp.servers` config and logs.

## Solution Assessment

Unclear. The narrowest maintainable solution depends on whether maintainers want live remote MCP discovery in `tools.effective`, or a UI/docs change that separates configured MCP servers from runtime-discovered tools.
