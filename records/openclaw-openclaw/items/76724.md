## Summary

Keep open: current main still has a real gap between embedded-agent MCP discovery and the Control UI inventory path, so the reported dashboard behavior is plausible from source inspection. The exact SSE setup still needs confirmation because the issue mentions `mcporter.json`, not the documented OpenClaw `mcp.servers` registry.

## What This Changes

The issue asks for configured SSE MCP server tools to be discovered after a successful initialize handshake and shown in the agent Tools dashboard.

## Best Possible Solution

Define a single MCP visibility contract: either have the Tools panel materialize and report bundle-MCP tools with status/errors, or show configured MCP servers separately and document that discovery happens only during agent runtime materialization.

## Reproduction Assessment

Yes for the dashboard symptom by source inspection: `tools.effective` feeds Available Right Now but does not materialize bundle-MCP, while embedded agent attempts do. The exact custom SSE setup is not fully reproduced because the issue lacks the canonical `mcp.servers` config and logs.

## Solution Assessment

Unclear. The narrowest maintainable fix depends on whether maintainers want `tools.effective` to make live remote MCP calls, reuse a session MCP catalog, or keep the dashboard static and add explicit MCP server status/config guidance.
