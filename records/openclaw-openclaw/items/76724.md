## Summary

Keep open: current main still has the dashboard/API gap described in the prior ClawSweeper review. Runtime MCP discovery calls `listTools()` during embedded Pi agent turns, but `tools.effective` still builds the Tools panel inventory from `createOpenClawCodingTools()` only, so configured MCP tools can remain absent from the Agent Tools dashboard even when the MCP handshake succeeds.

## What This Changes

The issue asks OpenClaw to discover and display remote SSE MCP tools in the Agent Tools dashboard after a successful MCP initialize handshake.

## Best Possible Solution

Make tools.effective and the Tools panel include discovered bundle-MCP tools from the existing session MCP runtime/catalog path, with clear unavailable/error status, while keeping mcp.servers as the supported config source.

## Reproduction Assessment

Yes for the dashboard/API gap from source inspection: tools.effective does not materialize bundle-MCP tools or call tools/list, while embedded Pi agent runs do. The reporter's exact remote SSE server setup was not live-reproduced because the issue lacks supported mcp.servers config and logs.

## Solution Assessment

Yes: the narrow maintainable fix is to reuse the existing session MCP runtime/catalog and final bundled-tool policy in tools.effective, not add a mcporter.json convention or a separate dashboard-only MCP client.
