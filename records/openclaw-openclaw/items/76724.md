## Summary

Keep open: current main has runtime MCP discovery for embedded agent turns, but the Control UI tool inventory path does not materialize configured MCP servers, so the reported dashboard behavior is plausible from source inspection. The report also references `mcporter.json`, which is not the documented OpenClaw MCP registry, so the exact live SSE setup still needs config/log confirmation.

## What This Changes

Report that a configured remote SSE MCP server initializes successfully but its tools are not discovered or shown in the agent tools dashboard.

## Best Possible Solution

Define one clear MCP availability contract: either make the Tools panel show discovered `bundle-mcp` tools with status/errors, or explicitly surface configured MCP servers separately and document that remote tool discovery happens during agent runtime materialization.

## Reproduction Assessment

Yes for the dashboard symptom by source inspection: the UI uses `tools.catalog`/`tools.effective`, and that effective-inventory path does not materialize configured MCP servers or call tools/list. The exact reported SSE server setup is not fully reproduced because the issue lacks the OpenClaw `mcp.servers` config and logs.

## Solution Assessment

Unclear. The narrowest maintainable fix depends on whether maintainers want live MCP discovery in `tools.effective` despite remote-call cost, or a UI/docs change that separates configured MCP servers from runtime-discovered tools.
