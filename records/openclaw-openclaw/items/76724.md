## Summary

Keep open: current main still has a source-reproducible gap between the bundle-MCP runtime that discovers tools for agent runs and the `tools.effective` inventory that feeds the Agent Tools dashboard. The reporter's `mcporter.json` path is not the documented MCP registry, but the dashboard/API path still does not materialize configured MCP tools.

## What This Changes

The issue reports that an SSE MCP server initializes successfully, but its tools are not requested with `tools/list` or shown in the Agent Tools dashboard.

## Best Possible Solution

Make `tools.effective` and the Agent Tools panel include discovered `mcp.servers` bundle-MCP tools by reusing the existing session MCP runtime/materializer, while keeping `mcp.servers` as the supported config contract.

## Reproduction Assessment

Unclear for the exact reporter environment because the SSE server and UI were not live-reproduced. The dashboard/API gap is source-reproducible: `tools.effective` does not call the bundle-MCP discovery/materialization path that performs `tools/list`.

## Solution Assessment

Yes: the narrow maintainable fix is to share the existing session bundle-MCP runtime/materializer with the `tools.effective` inventory path, not add a new `mcporter.json` convention or a separate dashboard-only MCP client.
