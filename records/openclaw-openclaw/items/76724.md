## Summary

Keep open: current main still supports runtime MCP discovery during embedded agent turns, but the Control UI inventory path does not materialize configured bundle-MCP servers, so the reported dashboard symptom remains plausible. The exact SSE setup still needs maintainer/config confirmation because the report names `mcporter.json` rather than the documented `mcp.servers` registry.

## What This Changes

The issue reports that an SSE MCP server initializes successfully but its tools never appear in the Agent Tools dashboard or trigger a `tools/list` request.

## Best Possible Solution

Define one MCP dashboard contract: either surface configured remote MCP servers with discovered tool/status data in the Tools panel, or clearly separate runtime-only MCP discovery from configuration guidance.

## Reproduction Assessment

Yes for the dashboard symptom by source inspection: `tools.effective` uses `resolveEffectiveToolInventory`, which does not materialize bundle-MCP tools, while actual agent turns do. The exact SSE server report is not fully reproduced because the issue lacks the OpenClaw `mcp.servers` config and logs.

## Solution Assessment

Unclear. The narrowest maintainable fix depends on whether maintainers want live MCP discovery in `tools.effective`, or a UI/docs change that says dashboard reload does not perform remote `tools/list`.
