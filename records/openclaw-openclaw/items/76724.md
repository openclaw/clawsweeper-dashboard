## Summary

Keep open: current main still has a source-visible gap between bundle-MCP discovery and the `tools.effective` inventory that feeds the Agent Tools panel, so the dashboard can stay at the built-in count even though runtime MCP discovery exists.

## What This Changes

The issue asks for SSE MCP server tools to be discovered with `tools/list` and shown in the Agent Tools dashboard after a successful MCP handshake.

## Best Possible Solution

Make `tools.effective` and the Agent Tools panel include discovered bundle-MCP tools from supported `mcp.servers` config, reusing the existing session MCP runtime and policy rules rather than adding a new config convention.

## Reproduction Assessment

No live high-confidence reproduction was run, but the source path is reproducible: bundle-MCP discovery calls `listTools` during agent runs while `tools.effective` does not materialize or query that runtime.

## Solution Assessment

Yes: the narrow maintainable fix is to share the existing session MCP runtime/materialization and final bundled-tool policy with the tools-effective inventory path, not build a separate dashboard-only MCP client.
