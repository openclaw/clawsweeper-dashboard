## Summary

Keep open: current main still has a source-reproducible gap where configured MCP tools can be materialized for agent runs, but the Agent Tools inventory path does not discover or list them.

## What This Changes

The issue reports that an SSE MCP server initializes successfully, but its tools are not discovered or shown in the Agent Tools dashboard.

## Best Possible Solution

Make tools.effective and the Agent Tools panel include discovered bundle-MCP tools through the existing session MCP runtime/materialization path, with clear status/error handling and mcp.servers kept as the supported config source.

## Reproduction Assessment

Yes, from source inspection: the agent run path calls the bundle-MCP runtime and tools/list, while the tools.effective dashboard path does not. The exact remote SSE server was not live-tested.

## Solution Assessment

Yes: the narrow maintainable fix is to reuse the existing session MCP runtime, materializer, naming, and final policy behavior in the tools.effective inventory path rather than add a second MCP discovery implementation.
