## Summary

Keep open: current main still has a source-visible gap between configured bundle-MCP discovery and the Agent Tools inventory path, so the dashboard can remain at the built-in tool count even though runtime MCP connection/discovery exists elsewhere.

## What This Changes

The issue asks for configured SSE MCP server tools to be discovered with `tools/list` and shown in the Agent Tools dashboard after a successful initialize handshake.

## Best Possible Solution

Make `tools.effective` and the Agent Tools panel include discovered configured bundle-MCP tools through the existing session MCP runtime/materialization and policy pipeline, while keeping `mcp.servers` as the supported config source.

## Reproduction Assessment

Yes, source-level: a configured `mcp.servers` catalog can be materialized during an agent run, but the current `tools.effective` path never calls that runtime and therefore cannot send `tools/list` or include those tools in the panel. The exact remote SSE deployment was not live-reproduced because the issue lacks logs and effective OpenClaw config.

## Solution Assessment

Yes: the narrow maintainable fix is to reuse the existing session bundle-MCP runtime/materializer and final tool-policy behavior in the effective-inventory path, not add a new `mcporter.json` convention or dashboard-only MCP implementation.
