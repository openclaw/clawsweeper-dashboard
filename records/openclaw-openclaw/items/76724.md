## Summary

Keep open: current main still has a source-proven gap where the Agent Tools dashboard and `/tools` inventory call `resolveEffectiveToolInventory(...)`, but that resolver only builds core/plugin/channel tools and never invokes the configured MCP `tools/list` materialization path. ## What This Changes

The issue reports that configured MCP server tools are not discovered or shown in the Agent Tools inventory after a successful MCP initialize handshake.

## Best Possible Solution

Implement a focused inventory fix that reuses the existing session MCP runtime, materialization, final policy filtering, name-collision handling, cache/error behavior, and lease disposal for both tools.effective and /tools; keep final provider-body serialization proof tracked with https://github.com/openclaw/openclaw/issues/80909.
