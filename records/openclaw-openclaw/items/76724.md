## Summary

Keep open: current main still has a source-visible gap between runtime MCP discovery and the Agent Tools inventory path, so configured MCP tools can be initialized for agent use while the dashboard remains at the built-in tool count.

## What This Changes

The issue requests that SSE MCP tools be discovered with `tools/list` and shown in the Agent Tools dashboard after a successful MCP initialize handshake.

## Best Possible Solution

Make `tools.effective` and the Agent Tools panel include discovered bundle-MCP tools from the existing session MCP runtime, while keeping `mcp.servers` as the supported config source and surfacing bounded status/errors.

## Reproduction Assessment

Yes, from source inspection: `tools.effective` feeds the Tools panel but never materializes the session MCP runtime that performs `client.listTools`. I did not live-run the reporter's SSE server, so this is source-reproducible rather than fully reproduced.

## Solution Assessment

Yes: the narrow maintainable fix is to reuse the existing bundle-MCP runtime/materialization and final tool-policy semantics in the `tools.effective` path. It should not add a new `mcporter.json` convention or a separate dashboard-only MCP implementation.
