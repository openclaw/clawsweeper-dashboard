## Summary

Keep open: current main still has a source-backed gap between bundle-MCP discovery and the Agent Tools inventory, so a configured MCP server can initialize while the dashboard stays at the built-in effective-tool count.

## What This Changes

The issue reports that an SSE MCP server initializes successfully, but the Agent Tools dashboard never sends `tools/list` or shows the server's tools.

## Best Possible Solution

Make `tools.effective` and the Tools panel include discovered `mcp.servers` bundle-MCP tools using the existing session MCP runtime/materialization and final tool policy, while keeping `mcp.servers` as the supported config source.

Yes, source-level: configure `mcp.servers` for a session and call `tools.effective`; current main's resolver never reaches the bundle-MCP `listTools` path. I did not live-reproduce the reporter's exact Flask SSE setup.

## Solution Assessment
