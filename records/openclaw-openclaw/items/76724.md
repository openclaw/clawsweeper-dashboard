## Summary

Keep open: current main still has a source-proven gap between configured MCP discovery/materialization and the Agent Tools dashboard inventory. The open provider-body report at https://github.com/openclaw/openclaw/issues/80909 is related but broader, and the open inventory PR at https://github.com/openclaw/openclaw/pull/70876 only targets `/tools` and remains incomplete for this dashboard path.

## What This Changes

The reporter wants configured SSE MCP server tools to be discovered after initialize and shown in the Agent Tools dashboard instead of the dashboard staying at the built-in tool count.

## Best Possible Solution

Have `tools.effective` and `/tools` reuse the existing session MCP materialization plus final policy path, show configured MCP tools in the Agent Tools UI, and keep provider-body serialization work tracked separately in https://github.com/openclaw/openclaw/issues/80909.
