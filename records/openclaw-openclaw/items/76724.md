## Summary

Keep open: current `main` still has runtime bundle-MCP discovery for embedded agent turns, but the Control UI `tools.effective` inventory path does not materialize configured MCP servers, so the reported dashboard behavior remains plausible. The exact SSE setup still needs config/log confirmation because the report mentions `mcporter.json`, which is not the documented OpenClaw MCP registry.

## What This Changes

Remote MCP tools configured over SSE do not appear in the Agent tools dashboard after a successful initialize handshake, and reload does not trigger `tools/list`.

## Best Possible Solution

Define one clear MCP availability contract: either make `tools.effective` surface discovered `bundle-mcp` tools with status/errors, or make the Tools panel show configured MCP servers separately and document that remote tool discovery happens only during agent runtime materialization.

## Reproduction Assessment

Unclear for the full live SSE report. The dashboard symptom is source-reproducible because `tools.effective` does not materialize configured MCP servers, but the report does not include a documented `mcp.servers` entry or gateway logs for the exact setup.

## Solution Assessment

Unclear. The narrowest maintainable fix depends on whether maintainers want live MCP discovery in the `tools.effective` dashboard path or a UI/docs split between configured MCP servers and runtime-discovered tools.
