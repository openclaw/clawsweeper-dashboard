## Summary

Keep open: current main still has a source-proven Agent Tools dashboard gap for configured MCP tools. The run path can materialize MCP tools, but `tools.effective` still builds only the synchronous OpenClaw coding-tool inventory and never reaches the session MCP runtime that performs `tools/list`.

## What This Changes

The issue asks for configured SSE MCP server tools to be discovered and shown in the Agent Tools dashboard after a successful MCP initialize handshake.

## Best Possible Solution

Make `tools.effective` reuse the session MCP materialization and final policy path used by embedded Pi runs so configured MCP tools appear in the Agent Tools dashboard, while keeping provider-body serialization tracked separately if it remains distinct.

## Reproduction Assessment

Yes, from source for the dashboard path: `tools.effective` calls a synchronous inventory resolver that builds `createOpenClawCodingTools(...)` output and never reaches the MCP runtime that performs `client.listTools()`. I did not live-run the reporter's SSE server.

## Solution Assessment

Yes, the narrow maintainable fix is to route effective inventory through the same configured-MCP materialization and policy path used by embedded Pi runs. Closing this as implemented or fully duplicated would leave the dashboard-specific `tools/list` gap untracked.
