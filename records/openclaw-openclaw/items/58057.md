## What This Changes

The issue requests dynamic identity resolution for DM and group allowlists, including dynamic policy modes, a resolver or authorization hook, and unknown-sender handling.

## Best Possible Solution

Define an additive generic dynamic authorization or access-group contract with explicit fail-closed semantics, then wire it through iMessage using the shared ingress resolver while preserving static defaults and documenting the new behavior.
