## Summary

Keep open: current main and v2026.6.10 still lack a generic dynamic sender-authorization mode for iMessage or HTTP-backed identity lookup. The merged access-groups work is useful partial progress, but this remains a security-sensitive product/API decision rather than cleanup-close material.

## What This Changes

The issue requests dynamic DM and group authorization so channel sender identity can be resolved at message time instead of maintained only in static allowlist config.

## Best Possible Solution

Define a maintainer-approved shared dynamic authorization seam, likely as an access-group or channel-ingress resolver contract, with current safe defaults preserved and iMessage able to plug in identity lookup without ad hoc core policy.

## Reproduction Assessment

Yes. Source inspection shows current main and v2026.6.10 have no dynamic dmPolicy/groupPolicy value, and iMessage sender admission is still resolved from allowlists/access groups before dispatch.

## Solution Assessment

Unclear. The need is valid, but HTTP resolver, pre-allowlist hook, unknown-sender routing, and dynamic access-group membership are different security/API shapes that require maintainer selection.
