## Summary

Keep open: current main and latest release still do not provide a generic dynamic DM/group sender authorization mode for iMessage or an HTTP-backed identity resolver. The Discord access-groups work is useful partial progress, but this remains a security-sensitive product/API decision rather than cleanup-close material.

## What This Changes

Requested support for dynamic sender authorization in DM/group allowlists through a resolver or pre-allowlist hook, plus optional unknown-sender routing.

## Best Possible Solution

Define a maintainer-approved shared dynamic authorization seam, likely through access groups or channel ingress, with safe defaults preserved and iMessage able to plug in identity lookup without ad hoc core policy.

## Reproduction Assessment

Yes for source-level absence: current config and ingress policy unions only accept static policy values, and iMessage drops blocked senders before dispatch. This proves the requested feature is missing, not that current documented behavior is broken.

## Solution Assessment

Unclear. The need is valid, but HTTP resolver, pre-allowlist hook, unknown-sender routing, and dynamic access-group membership are different security/API shapes that need maintainer selection.
