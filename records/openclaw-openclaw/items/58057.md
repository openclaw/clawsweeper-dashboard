## Summary

Keep open. Current main still lacks the requested dynamic DM/group policy modes, resolver config, pre-allowlist authorization hook, and iMessage/Supabase identity resolution; this is a new authorization feature that needs maintainer product and security design.

## What This Changes

Request adding dynamic DM/group sender authorization that can resolve identities at runtime via an HTTP resolver or authorization hook, including unknown-sender routing for iMessage-style deployments.

## Best Possible Solution

Define an additive generic channel authorization or dynamic access-group contract with explicit fail-closed behavior, keep static allowlist/pairing defaults, then wire it through iMessage with docs and focused ingress tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection at current main shows the requested dynamic policy modes, resolver config, and pre-allowlist authorization hook are absent.

## Solution Assessment

Unclear. The proposed HTTP resolver and authorization hook are plausible, but the safer maintainable path is a generic channel authorization or dynamic access-group contract with explicit fail-closed security semantics before adding iMessage-specific config.
