## Summary

Keep open for human review: current main still lacks derived model refs, but this PR adds a new model-profile config contract and is not merge-ready because routed dynamic preparation, context-params semantics, and real behavior proof are unresolved.

## What This Changes

The PR adds `routeTo` and `contextTokens` to `agents.defaults.models`, routes model resolution through a configured upstream ref, updates generated schema, and adds focused unit tests.

## Best Possible Solution

Define one backward-compatible selectable model-profile routing contract, then update schema/help/docs, picker/listing, runtime resolution, dynamic preparation, params/context budgeting, sessions, fallbacks, and tests together.

## Reproduction Assessment

Yes for the PR blockers by source inspection: the PR routes inside `resolveModelWithRegistry` after `resolveModelAsync` prepares dynamic models, and `resolveContextTokensForModel` reads params from the original ref while checking the routed ref. The requested `routeTo` capability itself is a feature/config-contract proposal, not a broken current-main behavior reproduction.

## Solution Assessment

No, not as-is. `routeTo` may be a reasonable direction, but the patch needs maintainer agreement on model-profile semantics plus fixes for routed async preparation and explicit params inheritance or non-inheritance.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Prepare routed providers before dynamic resolution:** `src/agents/pi-embedded-runner/model.ts:619-628`
  - body: `resolveModelAsync` still calls `prepareProviderDynamicModel` with the original selected provider/model before this routed resolver runs. If `routeTo` points at another provider whose model is only available after provider preparation, the routed provider is never prepared and the request can still fall through to `Unknown model` even though the upstream target is valid.
  - confidence: 0.91
- **[P2] Define params inheritance for routed refs:** `src/agents/context.ts:408-409`
  - body: This reads `params` from the original derived ref while checking Anthropic 1M support against the routed upstream ref. If the upstream entry has `params.context1m` and the derived entry omits both `params` and `contextTokens`, the budget falls through; either inherit the upstream params or explicitly document and test non-inheritance.
  - confidence: 0.86
