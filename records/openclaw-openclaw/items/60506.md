## Summary

## What This Changes

The PR adds `routeTo` and `contextTokens` to `agents.defaults.models` entries, routes embedded model resolution through the configured target, updates generated config schema, and adds focused unit tests.

## Best Possible Solution

Define one backward-compatible model-profile/API-id routing contract, then implement it through schema/help/docs, picker/listing, async provider preparation, runtime resolution, context budgeting, session state, and focused upgrade/runtime proof.

## Reproduction Assessment

Yes for the PR blockers by source inspection: current main prepares dynamic models before the PR's routed resolver would run, and the PR reads params from the derived ref while checking context support against the routed ref. The requested `routeTo` capability itself is a feature/config contract, not a broken current-main behavior reproduction.

## Solution Assessment

No, not as-is. `routeTo` may be a viable direction, but the patch needs maintainer agreement on model-profile semantics plus fixes for routed dynamic preparation and explicit params/context inheritance behavior.
