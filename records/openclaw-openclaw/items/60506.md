## Best Possible Solution

Define one backward-compatible selectable model-profile contract, then update schema/help/docs, model listing and picker behavior, async provider preparation, params/context budgeting, sessions, fallbacks, and regression proof together.

## Solution Assessment

No, not as-is. A routed selectable model profile may be viable, but this branch needs maintainer agreement on the public contract plus fixes for async provider preparation, context-budget semantics, documentation, and real behavior proof.

  - confidence: 0.88
