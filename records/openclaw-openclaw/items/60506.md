## Summary

Keep open. The feature is not implemented on current main, and the PR is a plausible implementation candidate, but the proposed patch still has blocking runtime semantics issues around routed dynamic-provider preparation and undefined context-param inheritance.

## What This Changes

The PR adds `routeTo` and `contextTokens` to `agents.defaults.models`, routes model resolution through the configured upstream ref, updates the generated config schema, and adds focused unit coverage for derived model refs.

## Best Possible Solution

Keep this PR open as the implementation candidate, but update it so routed refs are resolved before async provider preparation and then reused consistently for preparation, final resolution, error reporting, context budgeting, schema/help, and picker/list behavior. The final contract should either inherit upstream params intentionally or document and test that params are not inherited.

## Reproduction Assessment

Yes. The PR findings have high-confidence static repro paths: configure a derived ref that routes to a dynamic-only target provider to hit the unprepared-provider path, and configure upstream `params.context1m` without derived `params`/`contextTokens` to observe the context-budget fallthrough.

## Solution Assessment

No, not as-is. A generic `routeTo` field is a reasonable direction, but routing only inside `resolveModelWithRegistry` is too late for async dynamic preparation, and the params inheritance behavior is not yet defined or covered.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Prepare routed providers before dynamic resolution:** `src/agents/pi-embedded-runner/model.ts:616-626`
  - body: `resolveModelWithRegistry` rewrites `provider/modelId` via `routeTo`, but `resolveModelAsync` still prepares dynamic models using the original provider/model before it calls into that resolver. If a derived ref routes across providers to a model that is only available after the target provider's prepare hook runs, OpenClaw prepares the wrong provider and can still return `Unknown model`. Compute the routed ref before the async prepare step and use the same effective provider/model for prepare and resolution.
  - confidence: 0.91
- **[P2] Define context params inheritance for routed refs:** `src/agents/context.ts:402-410`
  - body: The new context path reads `params` from the original derived ref, then checks the routed upstream ref for Anthropic 1M support. A derived ref that omits `params` and relies on the upstream entry's `params.context1m` will fall through to the smaller/default budget even though execution routes to the 1M-capable upstream model. Either fall back to routed-ref params or document that params are not inherited and add coverage for that case.
  - confidence: 0.86
