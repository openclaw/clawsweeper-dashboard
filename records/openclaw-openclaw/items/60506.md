## Summary

Keep open. Current main still does not implement the PR's `agents.defaults.models` `routeTo` contract, and the PR remains a plausible implementation candidate, but it has source-inspected runtime/config-semantics blockers plus no real behavior proof.

## What This Changes

The PR adds `routeTo` and `contextTokens` to `agents.defaults.models`, routes model resolution through the configured upstream ref, updates generated schema, and adds focused tests.

## Best Possible Solution

Define one backward-compatible selectable model-profile routing contract, then update schema/help/docs, picker/listing, runtime resolution, dynamic preparation, params, context budgeting, sessions, fallbacks, and tests together.

## Reproduction Assessment

Yes for the PR blockers by source inspection: dynamic model preparation still happens before the PR's routed resolver, and the params/context-budget asymmetry is visible in the PR diff. The requested `routeTo` capability itself is a feature/config-contract proposal, not a broken existing behavior reproduction.

## Solution Assessment

No, not as-is. `routeTo` may be a reasonable direction, but the patch needs maintainer agreement on model-profile semantics plus fixes for routed async preparation and explicit params inheritance or non-inheritance.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Prepare routed providers before dynamic resolution:** `src/agents/pi-embedded-runner/model.ts:616-633`
  - body: `resolveModelWithRegistry` rewrites to the routed provider/model, but `resolveModelAsync` still calls `prepareProviderDynamicModel` with the original ref before this resolver runs. A cross-provider `routeTo` to a dynamic-only target can still fail as an unknown model because the routed provider was never prepared.
  - confidence: 0.91
- **[P2] Define params inheritance for routed refs:** `src/agents/context.ts:407-410`
  - body: This reads `params` from the original derived ref while checking Anthropic 1M support against the routed upstream ref. If the upstream entry has `params.context1m` and the derived entry omits both `params` and `contextTokens`, the budget falls through; either inherit or explicitly document and test non-inheritance.
  - confidence: 0.86
