## Summary

Keep open. Current main still does not implement derived agent model refs, and the proposed patch is a useful but not merge-ready public config/runtime change because routed refs are not applied consistently across async dynamic-provider preparation and context-budget semantics remain underspecified.

## What This Changes

The PR adds `routeTo` and per-entry `contextTokens` to `agents.defaults.models`, updates config schemas, routes model execution through the configured upstream ref, and adds focused unit coverage.

## Best Possible Solution

Keep this PR open as an implementation candidate, but update it so routed-model resolution is computed once and applied consistently to sync resolution, async provider preparation, runtime execution, context budgeting, schema/help/docs, picker/list behavior, tests, and changelog. The final contract should explicitly document params inheritance and include coverage for same-provider routing, cross-provider dynamic routing, cycles, aliases, and context budgets.

## Reproduction Assessment

Yes. The current-main gap is reproducible by validating a config with `agents.defaults.models.*.routeTo` or per-entry `contextTokens`, which the strict schema rejects; the PR defects have static repro paths through cross-provider `routeTo` dynamic preparation and derived refs that omit both `params` and explicit `contextTokens`.

## Solution Assessment

No, not as-is. A generic `routeTo` field is a reasonable direction, but this patch should route before both async preparation and final resolution, define or document params inheritance, and add the required release-note surface before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Prepare routed providers before dynamic resolution:** `src/agents/pi-embedded-runner/model.ts:616-626`
  - body: `resolveModelWithRegistry` rewrites `provider/modelId` via `routeTo`, but `resolveModelAsync` still prepares dynamic models using the original provider/model before it calls into that resolver. If a derived ref routes across providers to a model that is only available after the target provider's prepare hook runs, OpenClaw prepares the wrong provider and can still return `Unknown model`. Compute the routed ref before the async prepare step and use the same effective provider/model for prepare and resolution.
  - confidence: 0.91
- **[P2] Define context params inheritance for routed refs:** `src/agents/context.ts:402-410`
  - body: The new context path reads `params` from the original derived ref, then checks the routed upstream ref for Anthropic 1M support. A derived ref that omits `params` and relies on the upstream entry's `params.context1m` will fall through to the smaller/default budget even though execution routes to the 1M-capable upstream model. Either fall back to routed-ref params or document that params are not inherited and add coverage for that case.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `src/config/types.agent-defaults.ts:12-14`
  - body: This introduces public `agents.defaults.models` config fields and runtime behavior, but the PR does not touch `CHANGELOG.md`. OpenClaw requires user-facing `feat` changes to add an active-version changelog entry, so this config API would otherwise ship without release notes.
  - confidence: 0.88
