## Summary

Keep open. Current main still does not implement `agents.defaults.models.*.routeTo`, so the PR is not obsolete, but the patch is not merge-ready because routed dynamic provider preparation, routed context parameter semantics, and real behavior proof are still unresolved.

## What This Changes

The PR adds `routeTo` and `contextTokens` to `agents.defaults.models`, routes model registry resolution through configured upstream refs, updates config schema, and adds focused unit tests.

## Best Possible Solution

Define one backward-compatible selectable model-profile routing contract, then update schema/help/docs, picker/listing, runtime dynamic preparation, params/context budgeting, sessions, fallbacks, and tests together.

## Reproduction Assessment

Yes for the PR blockers by source inspection: the patch routes inside `resolveModelWithRegistry` after `resolveModelAsync` prepares dynamic models, and context budgeting mixes original-ref params with routed-ref model checks. The requested `routeTo` capability itself is a new config-contract feature, not a broken current-main behavior reproduction.

## Solution Assessment

No, not as-is. `routeTo` may be a reasonable direction, but the patch needs maintainer agreement on model-profile semantics plus fixes for routed async preparation and explicit params inheritance or non-inheritance.
