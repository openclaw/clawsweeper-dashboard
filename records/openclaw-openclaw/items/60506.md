## Summary

Keep open. Current main still lacks derived `agents.defaults.models` routing, but this PR has blocking runtime/context semantics and a missing changelog entry, so it should not merge as-is.

## What This Changes

The PR adds `routeTo` and per-entry `contextTokens` under `agents.defaults.models`, routes runtime model resolution through the configured upstream ref, regenerates config schema, and adds focused unit tests.

## Best Possible Solution

Land a rebased version that computes the effective routed ref once and applies it consistently to schema/help/docs, picker/listing, async preparation, sync resolution, runtime execution, context budgeting, aliases, cycles, and changelog.

## Reproduction Assessment

Yes. Source inspection shows current main rejects `agents.defaults.models.*.routeTo` and entry-level `contextTokens`; the PR defects are also traceable through the async prepare and context-budgeting paths without needing a live provider.

## Solution Assessment

No, not as-is. A generic `routeTo` field is a plausible direction, but the patch must route before dynamic provider preparation, define params inheritance, and add release-note/docs coverage before it is the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Prepare routed providers before dynamic resolution:** `src/agents/pi-embedded-runner/model.ts:616-626`
  - body: `resolveModelWithRegistry` rewrites `provider/modelId` via `routeTo`, but `resolveModelAsync` still prepares dynamic models using the original provider/model before it calls this resolver. If a derived ref routes across providers to a model that only appears after the target provider's prepare hook runs, OpenClaw prepares the wrong provider and can still return `Unknown model`. Compute the routed ref before the async prepare step and use the same effective provider/model for prepare and resolution.
  - confidence: 0.91
- **[P2] Define context params inheritance for routed refs:** `src/agents/context.ts:402-410`
  - body: The new context path reads `params` from the original derived ref, then checks the routed upstream ref for Anthropic 1M support. A derived ref that omits `params` and relies on the upstream entry's `params.context1m` falls through to the smaller/default budget even though execution routes to the 1M-capable upstream model. Either fall back to routed-ref params or document that params are not inherited and cover that case.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `src/config/types.agent-defaults.ts:12-14`
  - body: This introduces public `agents.defaults.models` config fields and runtime behavior, but the PR does not update `CHANGELOG.md`. OpenClaw requires user-facing `feat` changes to add an active-version changelog entry, so this config API would otherwise ship without release notes.
  - confidence: 0.88
