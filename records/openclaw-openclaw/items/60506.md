## Summary

Keep open, but do not merge as-is: current main still lacks the requested derived-model routing surface, while this PR adds a new public config contract with unresolved routing semantics, blocking correctness issues, merge conflicts, and missing real behavior proof.

## What This Changes

This PR adds `routeTo` and entry-level `contextTokens` to `agents.defaults.models`, routes model resolution through the configured upstream ref, updates generated config schema, and adds focused tests.

## Best Possible Solution

Define a backward-compatible selectable model-profile routing contract, then rebase a complete patch that applies the effective routed ref before async preparation and consistently across schema/help/docs, picker/listing, runtime resolution, params, context budgeting, sessions, fallbacks, and tests.

## Reproduction Assessment

Yes for the PR blockers by source inspection: `resolveModelAsync` prepares the original ref before `resolveModelWithRegistry`, while the PR routes inside the later resolver; the params/context-budget asymmetry is also visible in the diff. The requested `routeTo` capability itself is a feature/config-contract proposal, not a broken existing behavior reproduction.

## Solution Assessment

No, not as-is. `routeTo` is a plausible direction, but the patch needs maintainer agreement on model-profile semantics, routed async preparation, and explicit params inheritance or non-inheritance before it is the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Prepare routed providers before dynamic resolution:** `src/agents/pi-embedded-runner/model.ts:616-633`
  - body: `resolveModelWithRegistry` now rewrites to the routed provider/model, but `resolveModelAsync` still calls `prepareProviderDynamicModel` with the original ref before this resolver runs. A cross-provider `routeTo` to a dynamic-only target can still fail as an unknown model because the routed provider was never prepared.
  - confidence: 0.91
- **[P2] Define params inheritance for routed refs:** `src/agents/context.ts:407-410`
  - body: This path reads `params` from the original derived ref while checking Anthropic 1M support against the routed upstream ref. If the upstream entry has `params.context1m` and the derived entry omits both `params` and `contextTokens`, the budget falls through; either inherit or explicitly document and test non-inheritance.
  - confidence: 0.86
