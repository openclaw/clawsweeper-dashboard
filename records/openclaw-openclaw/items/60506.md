## Summary

Keep open. Current main still lacks derived `agents.defaults.models.*.routeTo` support, and this PR is not merge-ready because routed dynamic-provider preparation and routed context-parameter semantics are not settled.

## What This Changes

The PR adds `routeTo` and per-entry `contextTokens` to `agents.defaults.models`, routes model resolution through a configured upstream ref, updates generated schema, and adds focused tests.

## Best Possible Solution

Land a rebased implementation that defines `routeTo` semantics once and applies the effective routed ref consistently across schema/help/docs, picker/listing, async preparation, runtime resolution, context budgeting, aliases, cycle handling, and changelog.

## Reproduction Assessment

Yes. Source inspection is enough: current main rejects the new config fields, and the PR defects follow from the unchanged async prepare path plus the modified context-budgeting path.

## Solution Assessment

No, not as-is. A generic `routeTo` field is plausible, but this patch should route before dynamic provider preparation, define or document params inheritance, and add release-note coverage before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Prepare routed providers before dynamic resolution:** `src/agents/pi-embedded-runner/model.ts:616-626`
  - body: `resolveModelWithRegistry` rewrites `provider/modelId` via `routeTo`, but `resolveModelAsync` still prepares dynamic models using the original provider/model before calling this resolver. If a derived ref routes across providers to a model that only appears after the target provider's prepare hook runs, OpenClaw prepares the wrong provider and can still return `Unknown model`. Compute the effective routed ref before the async prepare step and use it for both prepare and resolution.
  - confidence: 0.91
- **[P2] Define context params inheritance for routed refs:** `src/agents/context.ts:402-410`
  - body: The new context path reads `params` from the original derived ref, then checks the routed upstream ref for Anthropic 1M support. A derived ref that omits `params` and relies on the upstream entry's `params.context1m` falls through to the smaller/default budget even though execution routes to the 1M-capable upstream model. Either fall back to routed-ref params or document that params are not inherited and cover that case.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `src/config/types.agent-defaults.ts:12-14`
  - body: This introduces public `agents.defaults.models` config fields and runtime behavior, but the PR does not update `CHANGELOG.md`. OpenClaw requires user-facing `feat` changes to add an active-version changelog entry, so this config API would otherwise ship without release notes.
  - confidence: 0.88
