## What This Changes

This PR adds `routeTo` and per-entry `contextTokens` to `agents.defaults.models`, routes model registry resolution through the configured upstream ref, updates generated config schema, and adds focused tests.

## Best Possible Solution

Define one backward-compatible selectable model-profile contract, then update schema/help/docs, picker/listing, async provider preparation, params/context budgeting, session persistence, and proof together.

## Reproduction Assessment

Yes for the PR blockers by source inspection: routing is added after async provider preparation, and the context-token scan bypasses current main's explicit-provider guard. The requested `routeTo` capability itself is a feature/config-contract proposal, not a broken current-main behavior reproduction.

## Solution Assessment

No, not as-is. `routeTo` may be a viable direction, but the branch needs maintainer agreement on model-profile semantics plus fixes for async routed-provider preparation and context-budget semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Prepare the routed provider before dynamic lookup:** `src/agents/pi-embedded-runner/model.ts:616-626`
  - body: `resolveModelAsync` still calls `prepareProviderDynamicModel` with the original selected provider/model before this routed resolver runs. When `routeTo` targets a provider whose model is only available after provider preparation, the routed provider is never prepared and the request can still fall through to `Unknown model` even though the upstream target is valid.
  - confidence: 0.92
- **[P2] Keep inferred-provider calls out of config-token scans:** `src/agents/context.ts:391-399`
  - body: This new lookup runs before the existing explicit-provider guard. For model-only callers with a slash-containing model string, the inferred provider can be wrong, so `agents.defaults.models[provider/model].contextTokens` can be applied to the wrong runtime provider and misreport the session/context budget.
  - confidence: 0.88
- **[P2] Define routed-ref parameter inheritance:** `src/agents/context.ts:408-409`
  - body: `resolveContextTokensForModel` reads `params` from the original derived ref while checking 1M support against the routed upstream ref. If params are meant to inherit from the upstream, this under-budgets; if they are not meant to inherit, the public config contract needs docs and regression coverage.
  - confidence: 0.8
