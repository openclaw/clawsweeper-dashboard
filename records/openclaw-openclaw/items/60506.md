## Summary

Keep open. Current main does not implement derived model refs under `agents.defaults.models`, and the PR is not mergeable as-is because routed refs are applied after async dynamic-provider preparation and the context-parameter inheritance semantics are still unresolved.

## What This Changes

The PR adds `routeTo` and per-entry `contextTokens` to `agents.defaults.models`, routes model resolution through configured upstream refs, updates generated config schema, and adds focused unit tests.

## Best Possible Solution

Land a rebased implementation that computes one effective routed model ref and applies it consistently to schema/help/docs, picker/listing, alias selection, async preparation, runtime resolution, context budgeting, tests, and changelog.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes. Source inspection is enough here: current main lacks the proposed config fields, and the PR defects follow directly from the async prepare and context-budgeting paths without needing a live provider.

## Solution Assessment

Is this the best way to solve the issue? No, not as-is. A generic `routeTo` field is a plausible direction, but it needs consistent effective-ref handling and explicit parameter-inheritance semantics before it is maintainable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Prepare the routed provider before dynamic resolution:** `src/agents/pi-embedded-runner/model.ts:616-626`
  - body: `resolveModelWithRegistry` rewrites `provider/modelId` via `routeTo`, but `resolveModelAsync` still prepares dynamic models using the original provider/model before it calls this resolver. If a derived ref routes to a provider whose model appears only after that provider's prepare hook runs, OpenClaw prepares the wrong provider and can still return `Unknown model`. Compute the effective routed ref before the async prepare step and use it for both prepare and resolution.
  - confidence: 0.91
- **[P2] Define context-param inheritance for routed refs:** `src/agents/context.ts:402-410`
  - body: The new context path reads `params` from the original derived ref, then checks the routed upstream ref for Anthropic 1M support. A derived ref that omits `params` but routes to an upstream entry with `params.context1m` falls through to a smaller/default budget. Either inherit/fall back to routed-ref params or document and test that params are not inherited.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `src/config/types.agent-defaults.ts:12-14`
  - body: This introduces public `agents.defaults.models` config fields and runtime behavior, but the PR does not update `CHANGELOG.md`. User-facing feature/config changes need an active-version changelog entry so the new API does not ship silently.
  - confidence: 0.88
