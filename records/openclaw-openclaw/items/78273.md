## Summary

Keep open. Current main still has a credible silent data-loss path where CLI replies are delivered after transcript persistence fails, and the related reports/PRs are adjacent rather than a terminal fix for this OpenClaw mirror stall.

## What This Changes

The issue reports that OpenClaw's mirrored JSONL session transcript can stop appending mid-session for Claude CLI-backed Discord sessions while the Claude CLI transcript continues updating.

## Best Possible Solution

Make Claude CLI transcript mirroring durable in the existing OpenClaw persistence path, with detection or recovery for failed/stalled writes and focused regression coverage for the chosen failure mode.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only review. Source inspection shows a reproducible class of failure: make persistCliTurnTranscript or the append lock fail after a successful CLI result, and the reply path logs a warning while the OpenClaw transcript stops advancing.

## Solution Assessment

No proposed patch is present. The narrowest maintainable fix is in CLI turn transcript persistence/recovery, not in status formatting or only in readers that bypass the OpenClaw mirror.
