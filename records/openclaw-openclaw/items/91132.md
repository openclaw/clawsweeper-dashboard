- **[P1] Route new timing phases to the originating connection:** `src/gateway/server-runtime-subscriptions.ts:80`
  - body: Current `chat.send_timing` phases are sent with `broadcastToConnIds(..., new Set([connId]))` so only the originating operator UI receives per-run timing.   - body: The new payload sets both `ackToPhaseMs` and `receivedToPhaseMs` from `receivedAtMs`, so `ackToPhaseMs` includes pre-ACK work and no longer means what existing timing consumers expect. Carry the original `ackedAtMs` from `ChatRunTiming` and compute the two fields separately.
  - confidence: 0.86

  - body: This test manually calls `broadcastToConnIds` with `first-output` and `completed` payloads after `chat.send` returns, so it can pass even if the production delta/final handlers never emit the new phases or emit them to the wrong recipients. Drive the agent event handler path that produces the real frames.
