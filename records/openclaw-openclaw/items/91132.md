  - body: Existing `chat.send_timing` emissions target the sending Control UI connection through `broadcastToConnIds`; this callback uses broad `broadcast`, so unrelated READ-scope clients can receive timing for runs they did not start.   - confidence: 0.9

  - body: The new test manually calls `broadcastToConnIds` with the expected `first-output` and `completed` payloads, so it would pass even if the production agent delta/final handler never emitted those events. Drive `createAgentEventHandler` or a live-style gateway path instead.
