## Summary

Keep open: current main has better diagnostics and doctor/update repair for configured-but-missing external channel plugins, but the Discord quick setup page still omits the required `@openclaw/discord` install step. The open linked docs PR at https://github.com/openclaw/openclaw/pull/78090 directly closes this issue, so the issue should wait for that PR to land, be refreshed, or be replaced.

## What This Changes

The issue reports a Discord regression where Docker/package installs with `channels.discord` configured no longer load Discord after the plugin was externalized from the core package.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/78090 so the Discord setup page requires installing `@openclaw/discord` before gateway start, while keeping the existing doctor/update repair and channel-list diagnostic paths.
