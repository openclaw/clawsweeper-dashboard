## Summary

Keep open: the reporter confirmed Discord loads again after installing/enabling the external Discord plugin and restarting, but current main still has the Discord quick setup docs gap and the open closing PR has not landed.

## What This Changes

The issue reports that Docker/package users who configure `channels.discord` no longer get Discord loaded after Discord stopped being bundled by default in 2026.5.x.

## Best Possible Solution

Land or revise https://github.com/openclaw/openclaw/pull/78090 so Discord setup docs state the external plugin install, enable/restart, and doctor recovery path while preserving the external-plugin boundary.

## Reproduction Assessment

Yes at source level: current docs still allow a config-only Discord setup while the package excludes Discord and the official catalog requires `@openclaw/discord`. I did not establish a live Docker or bare-metal failure in this read-only pass.

## Solution Assessment
