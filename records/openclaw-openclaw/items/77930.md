## Summary

Keep open: current main still treats Discord as an official external plugin, and the report is plausible for a Docker/config-only setup where `@openclaw/discord` was not installed. The remaining question is whether this should be handled as support/docs around the externalized plugin transition or as a startup/repair regression.

## What This Changes

The issue reports that a configured Discord channel is missing from the plugin list in Docker 2026.5.4 after working in earlier releases.

## Best Possible Solution

Make the Discord external-plugin transition unambiguous for Docker/config-only setups and ensure configured Discord either reaches a clear install/doctor path or is installed through the supported setup flow.

## Reproduction Assessment

Unclear for the reporter's full Docker environment: source inspection shows a config-only setup without installed `@openclaw/discord` will omit Discord, and the reported plugin list matches that path. I did not establish a live current-main Docker failure or confirm whether their external plugin install path failed.

## Solution Assessment

Unclear; the repo already has `channels add` and doctor repair paths for official external plugins, so silently installing on Gateway startup would need maintainer product judgment. A narrower maintainable path is to verify the setup flow, then improve docs/diagnostics or repair only the proven failed path.
