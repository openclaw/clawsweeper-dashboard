## What This Changes

The issue reports a 2026.5.x Discord setup regression where Docker/package users can configure `channels.discord` but the Discord plugin is absent from the loaded plugin list unless `@openclaw/discord` is installed and enabled.

## Best Possible Solution

Keep the external-plugin runtime and doctor behavior, then land or replace the Discord docs correction so setup explicitly installs `@openclaw/discord` before gateway startup.

## Reproduction Assessment

Yes, at source level: current main and `v2026.6.6` omit the Discord plugin install step from the setup page while package/catalog sources show Discord is external and installed through `@openclaw/discord`. The reporter also confirmed install/enable/restart restored Discord loading.

## Solution Assessment

Yes. The linked docs-only fix is the narrow maintainable remaining solution because install, enable-by-default, channel-list hinting, and doctor repair behavior already exist in current main.
