Keep open: current main and v2026.5.12 still make `workspaceAccess: "none"` read-only at the managed `/workspace` mount and both filesystem bridges, while the active fix PR has not landed and still needs complete parity and proof.

## What This Changes

The issue requests that `workspaceAccess: "none"` keep the agent workspace hidden while leaving the isolated sandbox `/workspace` writable.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/46300 with a complete, tested fix that makes only `ro` read-only across Docker/browser mounts and local/remote bridges while preserving host isolation.
