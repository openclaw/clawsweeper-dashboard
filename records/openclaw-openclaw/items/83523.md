## Summary

Keep this PR open: the source-level bug is valid and the patch uses the right existing hook helper, but the current diff still starts tracking the rotated new session without ending or forgetting the replaced session, and the supplied proof is source/mock-only rather than a real gateway/plugin run.

## What This Changes

The PR imports the gateway session-start hook emitter into `agent.send` auto-rotation, passes the previous session ID as `resumedFrom`, and adds mocked regression coverage for caller-supplied session ID rotation.

## Best Possible Solution

Make `agent.send` rotation emit a paired old-session end or otherwise forget the replaced session before starting the new session with `resumedFrom`, then prove the path with focused regression coverage and a redacted real gateway/plugin run.

## Reproduction Assessment

Yes, at source level: current main rotates `agent.send` sessions and emits `sessions.changed reason="create"` without the session-start helper. I did not establish a live gateway/plugin reproduction in this read-only review.

## Solution Assessment

No. Adding the start hook is the right direction, but the PR should also close or forget the replaced session using the existing lifecycle pattern and then provide real gateway/plugin proof.
