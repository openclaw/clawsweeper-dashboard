## Summary

This PR is close, but it should stay open: the hook wiring is the right shape, yet the latest diff still collapses idle-expiry rotations into `session_end.reason: "daily"` and the supplied proof is source/mocked validation rather than a real gateway/plugin run.

## What This Changes

The PR adds paired `session_end` and `session_start` lifecycle hook emission to `agent.send` session auto-rotation and adds regression coverage for caller-supplied sessionId rotation.

## Best Possible Solution

Derive the end-hook reason from the actual freshness cause, including `idle` versus `daily`, preserve the end-before-start pairing, and prove the path with redacted gateway/plugin output.

## Reproduction Assessment

Yes, at source level: current main rotates `agent.send` sessions and emits `sessions.changed reason="create"` without the lifecycle hook path. The remaining PR defect is also source-reproducible because reset policy supports idle expiry while the new reason fallback sends `daily`.

## Solution Assessment

No, not yet. Reusing the existing hook helpers is the right shape, but the PR should distinguish idle from daily rotations and then provide real gateway/plugin proof before merge.
