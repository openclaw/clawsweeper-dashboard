This PR adds `session_end` and `session_start` lifecycle hook emission to `agent.send` session auto-rotation and adds focused coverage for caller-supplied `sessionId` rotation.

## Best Possible Solution

Reuse the existing lifecycle hook helpers, derive the reason from the earliest expired reset threshold, cover caller-supplied/daily/idle cases, and provide redacted gateway/plugin output.

## Reproduction Assessment

Yes, at source level: current `main` rotates `agent.send` sessions and emits `sessions.changed reason="create"` without the lifecycle hook path. The remaining PR defect is also source-reproducible from reset policy code and docs when both daily and idle thresholds are expired.

## Solution Assessment

No, not yet. Reusing the existing hook helpers is the right shape, but the PR should choose the actual earliest expired reset cause and then provide real gateway/plugin proof before merge.
