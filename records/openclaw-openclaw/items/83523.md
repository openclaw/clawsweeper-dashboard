## Summary

Keep open: the PR targets a real gateway session lifecycle gap, but the current patch is not merge-ready because it starts tracking the replacement session without pairing the previous session lifecycle transition, and the supplied proof is still source/test-only rather than a real gateway/plugin run.

## What This Changes

The PR wires the existing `session_start` hook helper into `agent.send` auto-rotation and adds an agent handler regression test for caller-supplied session ID rotation.

## Best Possible Solution

Update the branch to mirror the existing lifecycle transition contract: end or forget the previous session when the ID changes, emit the new `session_start` with `resumedFrom`, and attach real gateway/plugin proof for explicit and daily auto-rotation.

## Reproduction Assessment

Yes, source-level: current main rotates `agent.send` sessions and emits `sessions.changed reason="create"` without the lifecycle hook helper. I did not run a live gateway/plugin reproduction in this read-only review.

## Solution Assessment

No as-is: reusing the existing helper is the right direction, but the rotation transition should also close or forget the previous session and pass `resumedFrom` before this is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Pair auto-rotation with the previous session end:** `src/gateway/server-methods/agent.ts:1544-1551`
  - body: This new start hook records the replacement session in the shutdown tracker, but when `isNewSession` replaces an existing `sessionEntry.sessionId`, the old ID is never ended or forgotten and the new start has no `resumedFrom`. The existing reset/session-update lifecycle paths end the old session and link the new one, so plugins can otherwise see a start for the replacement while the previous session is finalized later as a stale shutdown session. Please emit the previous `session_end`/tracker cleanup before this start and cover both hooks in the regression test.
  - confidence: 0.78
