## Summary

Keep open: the lifecycle-hook fix is on the right path and the latest test-mock blocker was addressed, but the PR now hard-codes `session_end.reason` to `daily` for every `agent.send` rotation even though the branch also rotates for idle-stale sessions, caller-supplied replacement IDs, and other non-daily causes. The PR still only supplies source probes and mocked/local validation, not a real gateway/plugin observation of the hook delivery.

## What This Changes

The PR adds paired `session_end` and `session_start` hook emission to `agent.send` session auto-rotation and adds a mocked regression test for caller-supplied session ID rotation.

## Best Possible Solution

Keep the helper-based lifecycle pairing, but derive the end-hook reason from the actual rotation cause and cover daily, idle, and explicit replacement cases before proving the path with a redacted real gateway/plugin run.

## Reproduction Assessment

Yes, at source level: current main rotates `agent.send` sessions and emits `sessions.changed reason="create"` without invoking the lifecycle hook path. I did not run a live gateway/plugin reproduction in this read-only review.

## Solution Assessment

No, not yet: reusing the existing lifecycle hook helpers is the right shape, but the current PR hard-codes non-daily rotations as `daily`. The safer fix is to carry an explicit rotation cause into the hook payload and test the distinct branches.
