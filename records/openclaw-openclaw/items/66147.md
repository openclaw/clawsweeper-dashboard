## Summary

Keep open: the feature is not implemented on current main, but this PR is still unsafe to merge because the proposed wrapper changes only the model argument after primary-model transport, auth, tool availability, transcript policy, and wrappers have already been selected.

## What This Changes

The PR adds a `model.tool` config field plus resolver/schema tests and a Pi embedded runner stream wrapper for routing tool-continuation turns to a configured alternate model.

## Best Possible Solution

Decide the core-vs-plugin API shape first; if it stays in core, the effective model should drive provider/model resolution, tool capability, transport, credentials, transcript policy, wrappers, diagnostics, fallback, context budgeting, config docs, and validation.

## Reproduction Assessment

Yes for the PR blockers by source inspection: the diff swaps only the `Model` argument after the primary-bound model-call pipeline is prepared, and the PR body still leaves the real Ollama plus Anthropic integration test unchecked.

## Solution Assessment

No. The wrapper is too narrow for the advertised cross-provider workflow; a maintainable fix needs an effective-model pipeline or explicit routing contract rather than a late stream-function model swap.
