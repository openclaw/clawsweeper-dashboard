## Summary

Keep open: current main still lacks split model routing, and the linked API request remains broader than this branch, but this PR should not merge as-is because the proposed late stream wrapper crosses provider/auth boundaries, misses the first tool-emitting turn, and lacks real Ollama plus Anthropic proof.

## What This Changes

The PR adds `model.tool` config/schema/resolver support plus a Pi embedded runner stream wrapper intended to route tool-continuation turns to a configured alternate model.

## Best Possible Solution

Keep the feature under maintainer architecture/security review; if it stays in core, the effective model should own the whole model-call pipeline, and if it moves to plugins, https://github.com/openclaw/openclaw/issues/13225 should define the redirect contract first.

## Reproduction Assessment

Yes for the PR blockers by source inspection: the branch adds a late stream wrapper after current main has already chosen primary-model tool support, transport, auth, and provider wrappers. The advertised real Ollama plus Anthropic path is not reproduced because the PR body leaves that integration run unchecked.

## Solution Assessment

No. The late `Model` argument swap is too narrow for the advertised cross-provider workflow; the maintainable solution needs an effective-model routing contract or a plugin redirect API that owns transport, auth, policy, fallbacks, diagnostics, docs, and upgrade behavior together.
