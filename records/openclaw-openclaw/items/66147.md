## Summary

Keep open, but not merge-ready: source inspection confirms current main still lacks split tool-model routing, while this PR's late stream wrapper does not route the full provider/auth/tool pipeline and lacks real cross-provider proof.

## What This Changes

The branch adds `agents.*.model.tool` config/schema/resolution plus a Pi embedded runner stream wrapper and tests to route tool-continuation calls to an alternate model.

## Best Possible Solution

Keep the PR open for maintainer architecture/security review; if this belongs in core, route by an effective model that owns transport, auth, policy, fallbacks, diagnostics, docs, and upgrade behavior together.

## Reproduction Assessment

Yes for the PR blockers by source inspection: current main builds tool enablement, provider transport, auth, transcript policy, and wrappers from `params.model`, while the PR swaps only the stream call model after that setup. The PR body does not provide the promised real Ollama plus Anthropic run.

## Solution Assessment

No. The late wrapper is too narrow for the advertised cross-provider workflow; the maintainable solution is an effective-model routing contract or plugin API that owns the whole model-call pipeline.
