## Summary

Keep open: current main still lacks split tool-turn model routing, but this PR is not mergeable as-is because it adds a public config option and a late model swap that does not control first tool-call selection, transport/auth, provider wrappers, or generated config docs.

## What This Changes

The PR adds `agents.*.model.tool` config/resolution plus a Pi embedded runner stream wrapper and tests to route tool-continuation calls to a configured alternate model.

## Best Possible Solution

Keep the feature open for maintainer architecture/security review; if it stays in core, the effective model should drive the full model-call pipeline rather than a late `Model` argument swap.

## Reproduction Assessment

Yes for the PR blockers by source inspection: the proposed wrapper is added after primary provider/session setup, while current main builds tool enablement, transport, auth, and provider wrappers from `params.model`. The PR body also leaves the real Ollama plus Anthropic integration run unchecked.

## Solution Assessment

No. The late stream wrapper is too narrow for the advertised cross-provider workflow; the safer direction is an effective-model routing contract that owns transport, auth, policy, fallbacks, diagnostics, and docs together.

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Route tool turns through the effective transport:** `src/agents/pi-embedded-runner/run/attempt.ts:1287-1293`
  - body: The wrapper swaps only the `model` argument after the session stream function has already been resolved for the primary provider/model. In an Ollama primary plus Anthropic tool setup, tool-continuation calls can still use primary-bound transport and credential injection instead of the configured tool provider.
  - confidence: 0.92
- **[P1] Choose the tool model before first tool use:** `src/agents/pi-embedded-runner/split-model-stream.ts:44-50`
  - body: Routing activates only when the last message is a tool result, but the first assistant turn has to emit the tool call before any tool result exists. If the local primary model is the one that cannot reliably use tools, the configured tool model is never reached for the main advertised workflow.
  - confidence: 0.9

  - body: The new wrapper and tests import `@mariozechner/pi-ai` and `@mariozechner/pi-agent-core`, but current main depends on the renamed `@earendil-works/pi-*` packages. These imports will fail after rebasing onto current main.
  - confidence: 0.88
- **[P2] Refresh the generated config contract:** `src/config/zod-schema.agent-model.ts:8`
  - body: Adding `tool` to the strict agent model schema changes the public config surface, but the generated config baseline/docs artifacts are not updated. The config docs check would drift and users would not get the new contract documented.
  - confidence: 0.78
