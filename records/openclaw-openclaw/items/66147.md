## Summary

Keep open: this is a plausible feature direction, but the branch is not merge-ready because it adds provider/auth-sensitive model routing with missing real behavior proof and clear source-level P1 defects.

## What This Changes

The PR adds a public `model.tool` config field and a Pi embedded runner stream wrapper that tries to route tool-continuation turns to a configured alternate model.

## Best Possible Solution

Decide the routing contract first; if it belongs in core, make the effective primary/tool model drive tool enablement, transport, auth, provider wrappers, fallbacks, diagnostics, and context budgeting end to end.

## Reproduction Assessment

Yes for the PR blockers by source inspection: the branch swaps the model only after current main has prepared primary-model tool support, transport, auth, and provider wrappers. The advertised real Ollama plus Anthropic workflow is still not proven by a live run.

## Solution Assessment

No. The current patch is a late stream wrapper, but the maintainable fix is an effective-model pipeline or approved plugin/runtime routing contract that owns provider, auth, fallback, diagnostics, and session semantics.
