## Summary

Keep open: this is a useful feature proposal, but the branch is still an implementation candidate with source-level P1 routing defects, missing real cross-provider proof, merge conflicts, and overlap with the open model/session routing design thread at https://github.com/openclaw/openclaw/issues/13225.

## What This Changes

The PR adds `model.tool` config/schema/resolver support and a Pi embedded runner stream wrapper intended to route tool-continuation turns to a configured alternate model.

## Best Possible Solution

If maintainers want split routing in core, define an effective-model call pipeline where the selected primary/tool model owns tool enablement, transport, auth, provider wrappers, transcript policy, fallback, diagnostics, and context-budget behavior; otherwise continue the design in https://github.com/openclaw/openclaw/issues/13225.

## Solution Assessment

No, not as currently implemented. A narrow maintainable solution needs the effective model to drive the whole model-call pipeline, not only the `Model` argument passed into a primary-bound stream function.
