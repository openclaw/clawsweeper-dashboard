## Summary

Keep this PR open, but it is not merge-ready: the branch adds a useful split-model-routing idea, yet the proposed wrapper swaps only the `Model` argument after the primary model has already determined tool availability, transport, auth, provider-specific wrappers, transcript policy, and fallback behavior.

## What This Changes

The PR adds a `model.tool` configuration field plus Pi embedded runner wiring intended to route tool-continuation turns to a separate model from normal chat turns.

## Best Possible Solution

Either continue the API/security design in https://github.com/openclaw/openclaw/issues/13225, or rework this branch so the selected effective model owns the whole model-call pipeline: tool enablement, transport, auth profile/API key, provider wrappers, transcript policy, fallback, diagnostics, and context budgeting.

## Reproduction Assessment

Yes for the PR blockers by source inspection: current main and the PR head show tool enablement, stream transport, auth, provider wrappers, and transcript policy keyed to the primary model. The advertised real Ollama plus Anthropic path is not reproduced because the PR body leaves that integration run unchecked.

## Solution Assessment

No. Adding a late stream wrapper is not the best fix for cross-provider split routing; the effective model must own the full model-call pipeline, or maintainers should first settle the plugin/runtime routing contract in the open design issue.
