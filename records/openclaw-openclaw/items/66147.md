## Summary

Keep open: this is a plausible feature PR, but the patch is not mergeable as-is because it adds a core model-routing config while only swapping the model object after primary-bound tool, auth, transport, and wrapper setup have already been selected.

## What This Changes

The PR adds a `model.tool` agent config value, config resolver/schema updates, and Pi embedded runner stream routing/tests to send tool-continuation turns to a configured alternate model.

## Best Possible Solution

Keep this PR open for maintainer architecture review; if split routing belongs in core, implement a per-call effective-model pipeline that drives tools, transport, auth, provider wrappers, diagnostics, docs, generated config artifacts, and validation together, otherwise continue the plugin/API design in https://github.com/openclaw/openclaw/issues/13225.

## Reproduction Assessment

Yes. Static inspection against current main and the PR diff gives a high-confidence reproduction path for the blockers: the patch only swaps the model object after primary-bound tool enablement, auth, stream transport, transcript policy, and wrappers are selected.

## Solution Assessment

No. The wrapper is too narrow for the stated cross-provider use case; the maintainable solution is either a first-class per-call effective-model pipeline or an explicit plugin/API routing contract with provider, auth, audit, and failure semantics.
