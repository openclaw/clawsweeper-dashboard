## Summary

Keep open: current main still does not implement split tool-turn model routing, and this PR remains an unmerged feature/config change with blocking correctness, security, and proof gaps around provider/auth routing.

## What This Changes

The PR adds a `model.tool` agent config field plus resolver/schema support and a Pi embedded runner stream wrapper intended to route tool-continuation turns to a separately configured model.

## Best Possible Solution

Keep the PR open for maintainer architecture review; if split routing belongs in core, make the effective model drive the full call pipeline, otherwise continue the plugin/runtime API design in https://github.com/openclaw/openclaw/issues/13225.

## Reproduction Assessment

Yes for the PR blockers. Static inspection of the current main pipeline and the PR diff shows the patch swaps only the `Model` argument after primary-bound tool enablement, stream transport, auth, transcript policy, and wrappers are selected.

## Solution Assessment

No. The proposed wrapper is too narrow for the advertised cross-provider use case; the maintainable fix needs either a first-class per-call effective-model pipeline or an explicit plugin/runtime routing contract with provider, auth, audit, and failure semantics.
