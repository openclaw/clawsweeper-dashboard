## Summary

Keep open: current main does not implement split model routing, and the PR still contains blocking correctness, security, and proof gaps for the advertised cross-provider use case.

## What This Changes

The PR adds `model.tool` config, resolver/schema support, and a Pi embedded runner stream wrapper to route tool-continuation LLM calls to a configured alternate model.

## Best Possible Solution

Keep this PR open for maintainer architecture/security review; if split routing belongs in core, the effective model should drive the whole model-call pipeline, otherwise the routing API design should continue in https://github.com/openclaw/openclaw/issues/13225.

## Reproduction Assessment

Yes for the PR blockers: static inspection of current main and the PR diff shows the patch swaps only the `Model` argument after primary-bound tool availability, transport, auth, transcript policy, and wrappers are selected. No live Ollama plus Anthropic after-fix proof is supplied.

## Solution Assessment

No. The proposed wrapper is too narrow for the advertised cross-provider use case; a maintainable implementation needs a first-class per-call effective-model pipeline or an explicit plugin/runtime routing contract with provider, auth, audit, fallback, and failure semantics.
