## What This Changes

The PR changes cron isolated-agent finalization to use `agentMeta.lastCallUsage` for the stored session `totalTokens` snapshot while preserving aggregate counters and adding focused Vitest coverage.

## Best Possible Solution

Land the localized cron writer fix after refreshed checks, keeping aggregate counters and cost on cumulative usage while using final-call usage only for the context snapshot.

## Reproduction Assessment

Yes, at source level: current main passes accumulated cron run usage into `deriveSessionTotalTokens`, so a multi-call run with smaller `lastCallUsage.input` will overstate the stored context snapshot. I did not run tests because this review is read-only.

## Solution Assessment

Yes. The writer-side change is the narrow maintainable fix because the embedded runner already exposes final-call usage and sibling session writers use it for context snapshots while leaving aggregate usage for telemetry and cost.
