## Summary

Keep open: current main still lacks native ClawHub skill integrity verification, but this PR is not mergeable yet because the proposed security signal can fail open and the external contributor has not provided real behavior proof.

## What This Changes

This branch adds SHA-256 lockfile metadata to ClawHub skill installs, skill integrity verifier helpers, security audit findings for missing or mismatched hashes, and a proposal doc.

## Best Possible Solution

Land a rebased, security-reviewed implementation that records stable trusted skill integrity metadata, reports mismatch, missing-hash, and verifier-error states, hashes runtime-relevant contents except explicit OpenClaw metadata, and aligns with the canonical skill-signature tracker.

## Reproduction Assessment

Yes for the PR defects: source inspection of the PR head shows verifier error results are ignored and hidden files are excluded from the digest used by the new verification path. I did not run a mutating runtime repro because this review is read-only.

## Solution Assessment

No. The install/audit direction is plausible, but this is not the best mergeable form until it handles verifier errors, hashes runtime-relevant hidden files, rebases onto current main, and gets maintainer direction on the trust semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Report verifier errors in audit findings:** `src/security/audit-extra.async.ts:1350-1358`
  - body: `verifySkillIntegrity` can return `status: "error"` for read/hash failures, but this collector only records missing hashes and mismatches. A tampered or unreadable skill can therefore make verification fail open while `security audit` stays clean; emit an audit finding for error results.
  - confidence: 0.9
- **[P2] Hash hidden runtime files during integrity verification:** `src/agents/skills-clawhub.ts:526`
  - body: The verifier uses `computeSkillFingerprint`, whose traversal skips dot-prefixed files and directories. If a skill uses hidden runtime files, post-install tampering there still verifies as `ok`; exclude only explicit OpenClaw metadata such as `.clawhub`.
  - confidence: 0.86
