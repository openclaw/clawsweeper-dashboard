## Summary

Keep open. Current main does not implement ClawHub skill integrity lock metadata or an audit collector, so this PR is not obsolete, but the patch is not merge-ready because the new security signal can fail open on verifier errors and misses hidden skill contents. The touched `src/security` surface also needs maintainer/security review before merge.

## What This Changes

The PR adds a skill-integrity proposal doc and wires ClawHub skill installs/security audit to store SHA-256 hashes in `.clawhub/lock.json` and report missing or mismatched skill hashes.

## Best Possible Solution

Land a rebased, security-reviewed implementation that records stable full-content hashes for ClawHub-installed skills, reports mismatch, missing-hash, and verifier-error states through the current audit runtime, hashes all runtime-relevant files except explicit OpenClaw metadata, and aligns tests/docs with the final audit-versus-loader scope.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: make `verifySkillIntegrity` return `status: "error"` and the PR's audit collector emits no finding, or mutate a hidden runtime file after install and the digest remains unchanged because dot entries are skipped. I did not run a mutating repro because this review is read-only.

## Solution Assessment

No. The ClawHub install/audit surface is a plausible place for this hardening, but this patch should not merge until it handles verifier errors, hashes hidden runtime files deliberately, reconciles the docs/schema/load-time scope, and adds targeted coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Report integrity verification errors:** `src/security/audit-extra.async.ts:1350-1358`
  - body: `verifySkillIntegrity` returns `status: "error"` when hashing or reading a skill fails, but the audit collector drops that status entirely. A tampered or unreadable installed skill can therefore make integrity verification fail open and still produce a clean `security audit` result; add a finding for error results instead of ignoring them.
  - confidence: 0.9
- **[P2] Hash hidden skill files:** `src/agents/skills-clawhub.ts:320`
  - body: The new install hash uses `computeSkillFingerprint`, which skips every dotfile and dot directory. Since the same helper is used during verification, post-install changes to hidden runtime files inside a skill will not trigger `skills.integrity.hash_mismatch`; exclude only OpenClaw metadata such as `.clawhub` instead of all hidden entries.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The PR adds no new dependency or workflow execution surface, but the new security check has concrete fail-open integrity gaps that need attention before merge.

Concerns:

- **[medium] Integrity errors are silently ignored:** `src/security/audit-extra.async.ts:1350`
  - body: Hash/read failures are represented as `status: "error"`, but the audit collector never converts them into findings, so compromised or unreadable skill contents can avoid the intended tamper warning.
