## Summary

Keep open. Current main still lacks native skill integrity verification, and this PR is a plausible security-hardening direction, but the current branch has fail-open audit gaps, missing real behavior proof, and unresolved trust-model alignment with the canonical skill-signature tracker.

## What This Changes

Adds SHA-256 hashes to ClawHub skill lockfile entries, skill integrity verifier helpers, security audit findings for missing or mismatched hashes, and a proposal doc.

## Best Possible Solution

Land a rebased, security-reviewed implementation that records stable trusted skill integrity metadata, reports mismatch, missing-hash, and verifier-error states, hashes runtime-relevant contents except explicit OpenClaw metadata, and aligns with https://github.com/openclaw/openclaw/issues/12507.

## Reproduction Assessment

Yes for the PR defects: source inspection of the PR head shows verifier error results are ignored and hidden files are excluded from the digest used by the new verification path. I did not run a mutating runtime repro because this review is read-only.

## Solution Assessment

No. The install/audit direction is plausible, but this is not the best mergeable form until verifier errors become findings, hidden runtime files are covered, the branch is rebased, real behavior proof is added, and maintainers decide the trust-model scope.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P2] Report verifier errors in audit findings:** `src/security/audit-extra.async.ts:1350-1358`
  - body: `verifySkillIntegrity` can return `status: "error"` for read or hash failures, but this collector only emits findings for missing hashes and mismatches. A compromised or unreadable skill can therefore make verification fail open while `security audit` stays clean; emit an audit finding for error results.
  - confidence: 0.9
- **[P2] Hash hidden runtime files during integrity verification:** `src/agents/skills-clawhub.ts:526`
  - body: The new verifier calls `computeSkillFingerprint`, whose traversal skips dot-prefixed files and directories. If a skill uses hidden runtime files, post-install tampering there still verifies as `ok`; exclude only explicit OpenClaw metadata such as `.clawhub`.
  - confidence: 0.86
