## Summary

Keep this PR open, but do not merge it as-is: current main still lacks the proposed skill lockfile integrity/audit behavior, while the PR head has concrete security-signal gaps, no real behavior proof, and is conflicting against current main.

## What This Changes

The PR records SHA-256 fingerprints for ClawHub skill installs in the lockfile, adds verifier helpers and audit findings, and adds a skill-integrity proposal doc.

## Best Possible Solution

Land a rebased, security-reviewed implementation aligned with https://github.com/openclaw/openclaw/issues/12507 that records trusted skill integrity metadata, reports mismatch, missing-hash, and verifier-error states, hashes runtime-relevant contents except explicit OpenClaw metadata, and documents loader policy.

## Reproduction Assessment

Yes for the PR defects: PR-head source inspection shows verifier error results are ignored and dot-prefixed contents are excluded from the digest used by verification. I did not run a mutating install/audit repro because this review is read-only.

## Solution Assessment

No. The install/audit direction is plausible, but this is not the best mergeable form until verifier errors become findings, runtime-relevant hidden contents are covered, the branch is rebased, real behavior proof is added, and maintainers settle the trust-model scope.
