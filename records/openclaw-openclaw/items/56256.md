## Summary

Keep open. Current main does not implement ClawHub skill integrity hashes or `skills.integrity.*` audit findings, and the PR is not merge-ready because the proposed security check still has fail-open gaps and is stale against the current audit/ClawHub skill code.

## What This Changes

The PR branch adds a skill-integrity proposal doc, stores optional SHA-256 metadata in ClawHub skill lockfile entries, exposes verification helpers, and wires new findings into security audit.

## Best Possible Solution

Land a rebased, security-reviewed implementation that records stable full-content hashes for ClawHub-installed skills, reports mismatch, missing-hash, and verifier-error states, deliberately excludes only OpenClaw-owned metadata, and aligns tests, docs, changelog, and audit-versus-loader scope with maintainer policy.

## Reproduction Assessment

Yes. Source inspection of PR head gives a high-confidence reproduction for the review findings: `status: "error"` results are dropped by the collector, and hidden files are omitted from the hash used for both install and verification.

## Solution Assessment

No. The ClawHub install/audit direction is plausible, but the current patch should not merge until it fails closed on verifier errors, hashes runtime-relevant hidden files, rebases onto current main, and resolves audit-only versus loader-time security policy.

Full review comments:

- **[P2] Report integrity verification errors:** `src/security/audit-extra.async.ts:1350-1358`
  - body: `verifySkillIntegrity` can return `status: "error"`, but this collector only emits findings for missing hashes and mismatches. If hashing or reading a skill fails after tampering, `security audit` can report clean results; convert error results into an audit finding instead of dropping them.
  - confidence: 0.9
- **[P2] Hash hidden skill contents:** `src/agents/skills-clawhub.ts:320`
  - body: The recorded and verification hashes use `computeSkillFingerprint`, which skips every dotfile and dot directory. Hidden runtime files inside a skill can be modified after install without changing the integrity result; exclude only explicit OpenClaw metadata such as `.clawhub`.
  - confidence: 0.86
