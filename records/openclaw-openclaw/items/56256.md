## Summary

Keep open. Current main does not implement skill lockfile hashes or `security audit` skill-integrity findings, and the PR is not merge-ready because the proposed security signal still fails open on verifier errors and misses hidden runtime files.

## What This Changes

The PR adds a skill-integrity proposal doc, stores SHA-256 metadata for ClawHub skill installs, and wires new skill hash findings into `openclaw security audit`.

## Best Possible Solution

Land a rebased, security-reviewed implementation that records stable full-content hashes for ClawHub-installed skills, reports mismatch, missing-hash, and verifier-error states, hashes runtime-relevant files except explicit OpenClaw metadata, and aligns tests/docs/changelog with the final audit-versus-loader scope.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction for the PR defects: `status: "error"` results are dropped by the collector, and hidden skill files are omitted from the digest used for both install and verification. I did not run a mutating repro because this review is read-only.

## Solution Assessment

No. The ClawHub install/audit surface is plausible, but the current patch should not merge until it fails closed on verifier errors, deliberately covers hidden runtime files, rebases onto current main, and settles the loader-versus-audit security scope.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Report integrity verification errors:** `src/security/audit-extra.async.ts:1350-1358`
  - body: `verifySkillIntegrity` can return `status: "error"`, but this collector only emits findings for missing hashes and mismatches. If hashing or reading a skill fails after tampering, `security audit` can report clean results; convert error results into an audit finding instead of dropping them.
  - confidence: 0.9
- **[P2] Hash hidden skill files:** `src/agents/skills-clawhub.ts:320`
  - body: The recorded and verification hashes use `computeSkillFingerprint`, which skips every dotfile and dot directory. Hidden runtime files inside a skill can be modified after install without changing the integrity result; exclude only explicit OpenClaw metadata such as `.clawhub`.
  - confidence: 0.86
