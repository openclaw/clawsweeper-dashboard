## Summary

Keep open. The PR is a plausible security-hardening direction, but current main still lacks skill integrity verification, the PR has concrete fail-open gaps, and the remaining loader/audit trust model needs maintainer security review before merge.

## What This Changes

Adds SHA-256 metadata to ClawHub skill installs, exposes skill integrity verification helpers, reports missing/mismatched hashes in `openclaw security audit`, and adds a proposal doc.

## Best Possible Solution

Land a rebased, security-reviewed implementation that records stable full-content hashes for ClawHub-installed skills, reports mismatch, missing-hash, and verifier-error states, deliberately excludes only OpenClaw metadata, and aligns with the canonical skill-signature trust model.

## Reproduction Assessment

Yes for the PR defects: source inspection of the PR head shows verifier `error` results are ignored and dot-prefixed skill contents are excluded from the digest. I did not run a mutating repro because this review is read-only.

## Solution Assessment

No. The install/audit direction is plausible, but this is not the best mergeable form until it handles verifier errors, hashes runtime-relevant hidden files except explicit metadata, rebases onto current main, and gets maintainer direction on audit-only versus loader-time policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Report verifier errors in audit findings:** `src/security/audit-extra.async.ts:1350-1358`
  - body: `verifySkillIntegrity` can return `status: "error"` for read/hash failures, but this collector only records missing hashes and mismatches. A tampered or unreadable skill can therefore make verification fail open while `security audit` stays clean; emit an audit finding for error results.
  - confidence: 0.9
- **[P2] Hash runtime-relevant hidden skill files:** `src/agents/skills-clawhub.ts:320`
  - body: The new recorded hash uses `computeSkillFingerprint`, which skips every dot-prefixed file or directory. If a skill uses a hidden runtime file, post-install tampering of that file will still verify as `ok`; exclude only explicit OpenClaw metadata such as `.clawhub`.
  - confidence: 0.86
