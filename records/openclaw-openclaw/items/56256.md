## Summary

Keep open. Current main still does not implement native skill integrity/signature verification, and this security-sensitive PR is not merge-ready because the proposed audit signal fails open in concrete cases and lacks contributor real behavior proof.

## What This Changes

The PR adds SHA-256 fingerprints to ClawHub skill lockfiles, verifier helpers, two security audit findings, and a skill-integrity proposal document.

## Best Possible Solution

Land a rebased, security-reviewed implementation aligned with https://github.com/openclaw/openclaw/issues/12507 that records trusted skill integrity metadata, reports mismatch, missing-hash, and verifier-error states, hashes runtime-relevant contents except explicit OpenClaw metadata, and defines loader policy.

## Reproduction Assessment

Yes for the PR defects: PR-head source inspection shows verifier error results are ignored and dot-prefixed contents are excluded from the digest used by verification. I did not run a mutating install/audit repro because this review is read-only.

## Solution Assessment

No. The audit/hash direction is useful, but this is not the best mergeable form until verifier errors become findings, runtime-relevant hidden contents are covered, the branch is rebased, real behavior proof is added, and maintainers settle the trust-model scope.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Report integrity verifier errors:** `src/security/audit-extra.async.ts:1350-1358`
  - body: `verifySkillIntegrity` can return status `error` for read or hash failures, but this collector only emits findings for missing hashes and mismatches. A compromised or unreadable skill can therefore make verification fail open while `security audit` stays clean; emit an audit finding for error results.
  - confidence: 0.9
- **[P2] Hash hidden skill files:** `src/agents/skills-clawhub.ts:486-487`
  - body: Integrity verification here depends on `computeSkillFingerprint`, but that traversal skips every dot-prefixed file and directory. If a skill uses hidden runtime files, post-install tampering there still verifies as ok; exclude only explicit OpenClaw metadata such as `.clawhub`.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The proposed security signal has concrete fail-open integrity gaps and does not yet resolve the broader skill trust model.

Concerns:

- **[medium] Integrity errors are silently ignored:** `src/security/audit-extra.async.ts:1350`
  - body: Hash/read failures are represented as status `error`, but the audit collector never turns them into findings, so compromised or unreadable skill contents can avoid the intended warning.
  - confidence: 0.9
- **[medium] Hidden skill contents are outside the digest:** `src/agents/skills-clawhub.ts:486`
  - body: Install and verification hashes skip dot-prefixed files and directories, so hidden runtime files can change after install without triggering a mismatch.
  - confidence: 0.86
- **[medium] Local hashes do not authenticate publisher content:** `src/agents/skills-clawhub.ts:320`
  - body: The PR records a digest computed from already-installed bytes; without trusted registry digest/signature metadata, this cannot prove publisher provenance or detect malicious content served at install time.
  - confidence: 0.78
