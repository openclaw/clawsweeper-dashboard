## Summary

Keep open. Current main still lacks this skill integrity feature, but the PR is not mergeable as-is because the new security signal has fail-open gaps, missing real behavior proof, and unresolved skill trust-model scope tied to the canonical tracker.

## What This Changes

The branch adds SHA-256 hashes to ClawHub skill lockfile entries, verifier helpers, security audit findings for missing or mismatched hashes, and a proposal document.

## Best Possible Solution

Land a rebased, security-reviewed implementation that records stable trusted skill integrity metadata, reports mismatch, missing-hash, and verifier-error states, hashes runtime-relevant contents except explicit OpenClaw metadata, and aligns with https://github.com/openclaw/openclaw/issues/12507.

## Reproduction Assessment

Yes for the PR defects: PR-head source inspection shows verifier error results are ignored and dot-prefixed contents are excluded from the digest used by verification. I did not run a mutating install/audit repro because this review is read-only.

## Solution Assessment

No. The install/audit direction is plausible, but this is not the best mergeable form until verifier errors become findings, runtime-relevant hidden contents are covered, the branch is rebased, real behavior proof is added, and maintainers settle the trust-model scope.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Report verifier errors in audit findings:** `src/security/audit-extra.async.ts:1350-1358`
  - body: `verifySkillIntegrity` can return `status: "error"` for read or hash failures, but this collector only emits findings for missing hashes and mismatches. A compromised or unreadable skill can therefore make verification fail open while `security audit` stays clean; emit an audit finding for error results.
  - confidence: 0.9
- **[P2] Hash hidden runtime files during integrity verification:** `src/agents/skills-clawhub.ts:526`
  - body: The new verifier calls `computeSkillFingerprint`, whose traversal skips dot-prefixed files and directories. If a skill uses hidden runtime files, post-install tampering there still verifies as `ok`; exclude only explicit OpenClaw metadata such as `.clawhub`.
  - confidence: 0.86
