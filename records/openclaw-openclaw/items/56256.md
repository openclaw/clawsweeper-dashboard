## Summary

Keep open. Current main still lacks ClawHub skill hash storage and `skills.integrity.*` audit findings, so the PR is not obsolete. The branch is relevant to the ClawHub skill supply-chain reports, and the earlier `skillRootDir` review bug was fixed in the PR head, but the patch remains security-sensitive and not merge-ready because verifier errors fail open, hidden skill files are excluded from the integrity signal, and the branch is stale against the current audit module layout.

## What This Changes

This PR adds a skill-integrity proposal doc and wires ClawHub skill installs/security audit to record SHA-256 hashes in `.clawhub/lock.json` and report missing or mismatched skill hashes.

## Best Possible Solution

Land a rebased, security-reviewed implementation that records stable full-content hashes for ClawHub-installed skills, reports mismatch/missing/error states through the current audit runtime, hashes all runtime-relevant files except explicit OpenClaw metadata, and adds focused unit coverage plus docs that match the final scope.

Yes. ## Solution Assessment

No. The core ClawHub install/audit surface is a plausible place for this hardening, but this patch should not merge until it reports verifier errors, handles hidden files deliberately, aligns schema/docs, adds tests, and rebases onto current main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Report integrity verification errors:** `src/security/audit-extra.async.ts:1350-1358`
  - body: `verifySkillIntegrity` returns `status: "error"` when hashing or reading a skill fails, but the audit collector drops that status entirely. A tampered or unreadable installed skill can therefore make integrity verification fail open and still produce a clean `security audit` result; add a finding for error results instead of ignoring them.
  - confidence: 0.9
- **[P2] Hash hidden skill files:** `src/agents/skills-clawhub.ts:320`
  - body: The new install hash uses `computeSkillFingerprint`, which skips every dotfile and dot directory. Since the same helper is used during verification, post-install changes to hidden runtime files inside a skill will not trigger `skills.integrity.hash_mismatch`; exclude only OpenClaw metadata such as `.clawhub` instead of all hidden entries.
  - confidence: 0.86

## Security Review
