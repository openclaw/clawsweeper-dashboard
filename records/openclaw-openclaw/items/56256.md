## Summary

Keep open: current main does not implement skill-integrity verification, and this PR is still useful security hardening work, but the proposed patch should not merge while verifier errors fail open, hidden skill contents are omitted from the digest, and the branch is stale against current main.

## What This Changes

The PR records SHA-256 metadata for ClawHub skill installs, adds verification helpers and security audit findings, and documents a skill-integrity proposal.

## Best Possible Solution

Land a rebased, security-reviewed implementation that records stable full-content hashes for ClawHub-installed skills, reports mismatch, missing-hash, and verifier-error states, hashes runtime-relevant files except explicit OpenClaw metadata, and aligns tests/docs/changelog with the final audit-versus-loader scope.

## Reproduction Assessment

Yes for the PR defects: source inspection shows `status: "error"` results are dropped by the collector, and the digest helper skips dot-prefixed skill files. I did not run a mutating repro because this review is read-only.

## Solution Assessment

No. The install/audit direction is plausible, but this is not the best mergeable form until it fails closed on verifier errors, deliberately covers hidden runtime files, rebases onto current main, and settles the loader-versus-audit scope.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Report verifier errors in audit findings:** `src/security/audit-extra.async.ts:1350-1358`
  - body: `verifySkillIntegrity` returns `status: "error"` for read/hash failures, but this collector only handles missing hashes and mismatches. A tampered or unreadable skill can therefore make integrity verification fail open while `security audit` stays clean; emit a finding for error results.
  - confidence: 0.9
- **[P2] Hash runtime-relevant hidden skill files:** `src/agents/skills-clawhub.ts:320`
  - body: The PR records hashes with `computeSkillFingerprint`, which skips every dot-prefixed file or directory. If a skill uses a hidden runtime file, post-install tampering of that file will not change the recorded/verifier digest; exclude only explicit OpenClaw metadata such as `.clawhub`.
  - confidence: 0.86
