## Summary

Keep open: current main does not already provide skill integrity verification, but this security hardening PR is not mergeable as-is because the proposed audit signal can fail open and the branch is dirty against current main.

## What This Changes

The PR adds SHA-256 metadata to ClawHub skill lockfile entries, new skill integrity verifier helpers, security audit findings, and a proposal document.

## Best Possible Solution

Land a rebased, security-reviewed implementation that records stable ClawHub skill content integrity, reports mismatch, missing-hash, and verifier-error states, hashes runtime-relevant contents except explicit OpenClaw metadata, and aligns with the canonical SIE/loader discussion.

## Reproduction Assessment

Yes for the PR defects: source inspection at the PR head shows verifier `error` results are ignored by the audit collector, and the digest helper skips dot-prefixed files while the new verifier relies on it. I did not run a mutating reproduction because this review is read-only.

## Solution Assessment

No. The install-hash plus audit approach is plausible, but this is not the best mergeable form until it rebases, fails closed on verifier errors, deliberately covers hidden runtime files, and settles the audit-only versus loader-gate scope.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Report verifier errors in audit findings:** `src/security/audit-extra.async.ts:1350-1358`
  - body: `verifySkillIntegrity` returns `status: "error"` for read or hash failures, but this collector only records missing hashes and mismatches. A compromised or unreadable skill can therefore make verification fail open while `security audit` stays clean; emit an audit finding for error results.
  - confidence: 0.9
- **[P2] Hash hidden skill contents deliberately:** `src/agents/skills-clawhub.ts:526`
  - body: The new verifier depends on `computeSkillFingerprint`, which skips every dot-prefixed file or directory. If a skill uses hidden runtime files, post-install tampering there will still verify as `ok`; hash all runtime-relevant skill contents and exclude only explicit OpenClaw metadata.
  - confidence: 0.86
