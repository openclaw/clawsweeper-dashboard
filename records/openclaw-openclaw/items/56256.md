Keep open. Current main does not implement native skill integrity metadata or audit findings, but this security-sensitive PR is not mergeable as-is because it lacks real behavior proof and still has fail-open integrity gaps that overlap the broader canonical skill trust-model work.

## What This Changes

The PR adds SHA-256 metadata for ClawHub skill installs, verifier helpers, security-audit findings for missing or mismatched hashes, and a skill-integrity proposal document.

## Best Possible Solution

Rebase and land a security-reviewed implementation aligned with https://github.com/openclaw/openclaw/issues/12507 that records trusted skill integrity metadata, reports mismatch, missing-hash, and verifier-error states, hashes runtime-relevant contents except explicit OpenClaw metadata, and defines loader policy.

## Solution Assessment

No. The install/audit direction is plausible, but this is not the best mergeable form until verifier errors become findings, runtime-relevant hidden contents are covered, the branch is rebased, real behavior proof is added, and maintainers settle the trust-model scope.
