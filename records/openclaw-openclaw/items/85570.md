## Summary

Keep open: the PR targets a real current-main media tool auth gap, but the proposed patch still leaves generation provider readiness inconsistent for providers with `isConfigured` hooks, and the protected `maintainer` label requires explicit maintainer handling.

## What This Changes

The PR adds a config-aware tool-auth helper and threads config/workspace auth context through image, PDF, video, music, and image-generation model preflight paths.

## Best Possible Solution

Make config-backed provider API keys participate in the same readiness decision as generation providers with `isConfigured` hooks, then cover that exact provider shape with a regression test.

## Reproduction Assessment

Yes, by source inspection: current main checks env/profile auth in tool preflight while config-backed custom provider API keys are resolved elsewhere, and generation providers with `isConfigured` hooks can still return false before the new helper runs.

## Solution Assessment

No. The shared helper is the right direction for fallback auth checks, but the generation provider path should also handle providers with `isConfigured` hooks instead of only the no-hook fallback.
