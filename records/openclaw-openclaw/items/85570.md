## Summary

Keep open. This is a maintainer-labeled MEMBER PR, so it is not a cleanup-close candidate, and the patch still appears to miss generation providers whose `isConfigured` hook returns false before the new config-aware helper can run.

## What This Changes

The PR adds a config-aware tool auth helper and routes image, PDF, and media-generation model preflight through it, with focused regression tests for config-backed provider `apiKey`.

## Best Possible Solution

Make config-backed provider `apiKey` augment generation-provider readiness even when a provider has `isConfigured`, then cover that path with a regression using an `isConfigured` provider shape.

## Reproduction Assessment

Yes, by source inspection: current main checks env/profile auth in the tool preflight path while config-backed `models.providers.*.apiKey` is resolved elsewhere. I did not run tests because this review was constrained to read-only inspection.

## Solution Assessment

No. The helper is the right direction for image/PDF paths, but the generation path should also handle providers with `isConfigured` hooks instead of only the fallback branch.

## Review Findings

Full review comments:

- **[P2] Honor config auth before returning false from `isConfigured`:** `src/agents/tools/media-tool-shared.ts:219`
