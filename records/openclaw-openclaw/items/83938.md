## Summary

Keep open: the PR addresses the right backup hardlink path, but it still lacks contributor real behavior proof and leaves `backup verify` accepting Link targets that normalize under the archive root but are absent from the archive.

## What This Changes

The branch disables node-tar hardlink emission for backup creation, adds hardlink linkpath validation to backup verify, adds regression tests, and updates the changelog.

## Best Possible Solution

Land the hardlink dereference fix only after backup verify also rejects missing Link targets and the PR includes redacted real backup/verify/extract proof.

## Reproduction Assessment

Yes. The source-level path is high confidence: create a backup from a tree with hardlinked files where tar emits a Link entry with a traversal-bearing `linkpath`; current main remaps only `entry.path` and verify ignores `linkpath`.

## Solution Assessment

No, not fully. Dereferencing through tar's link cache is a narrow writer-side fix, but the verifier should also reject hardlink targets that are not present in the archive and the PR needs real after-fix restore proof.
