## Summary

Keep open: the new real-behavior proof satisfies the contributor proof gate and the hardlink-dereference approach matches node-tar's linkCache contract, but the verifier change is still incomplete because it checks Link targets for path safety without confirming the target exists in the archive.

## What This Changes

This PR disables node-tar hardlink emission during backup creation, adds backup verification checks for unsafe hardlink targets, adds regression tests, and updates the changelog.

## Best Possible Solution

Land the dereference fix with a complete verifier check that rejects unsafe and missing hardlink targets, while accepting the documented storage tradeoff for restore-compatible backup archives.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create a backup from hardlinked files, current main lets node-tar emit Link entries via linkCache while remapping only entry.path, and backup verify ignores entry.linkpath; I did not rerun macOS bsdtar in this read-only review.

## Solution Assessment

No, not yet. Dereferencing newly created archives is the right narrow fix, but the verifier side should also reject root-contained Link targets that are missing from the archive before this is merge-ready.
