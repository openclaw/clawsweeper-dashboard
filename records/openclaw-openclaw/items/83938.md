## Summary

Keep open. The PR targets the right backup restore-safety gap, but it still needs after-fix real behavior proof from a real backup/restore path, and the verifier change should also reject hardlink targets that are safe-looking but absent from the archive.

## What This Changes

The branch disables tar hardlink emission during backup creation, records Link entry metadata during backup verification, rejects unsafe hardlink targets, adds backup regression tests, and updates the changelog.

## Best Possible Solution

Land a backup fix that dereferences newly created archives and makes backup verify reject unsafe or missing hardlink targets, with redacted real backup/restore proof before merge.

## Reproduction Assessment

Yes. The source-level path is high confidence: create a backup from a tree with hardlinked files where tar emits a Link entry with a traversal-bearing linkpath; current main remaps entry.path and verify ignores linkpath, though I did not run macOS bsdtar in this read-only review.

## Solution Assessment

No, not fully. Dereferencing via tar's linkCache is the right narrow writer-side approach, but the verifier should also require hardlink targets to exist in the archive and the PR needs real after-fix restore proof.
