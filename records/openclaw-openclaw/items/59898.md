## Summary

Keep this PR open: the branch targets a real prompt/report bug, but the latest head still leaves explicit no-tool sessions with leaked skills diagnostics and provider tool guidance that contradicts the no-tools contract.

## What This Changes

The PR changes agent system prompt/report generation, embedded runner plumbing, hosted client-tool normalization, OpenResponses handling, and tests for explicit empty effective tool lists.

## Best Possible Solution

Keep the no-tools direction, but make the rendered prompt, provider overlays, and prompt report all consume the same final effective tool/skills state.

## Reproduction Assessment

Yes. The PR body gives a concrete live CLI repro with tools denied, and source inspection shows both current main and the PR head still have prompt/report paths that can be checked directly.

## Solution Assessment

No. The current patch targets the right boundary, but it is not yet the best fix because no-tool prompts still include skill catalogs/reporting and provider tool guidance.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Suppress skills in explicit no-tool prompts:** `src/agents/system-prompt.ts:837-846`
  - body: When `toolNames: []` and `explicitEmptyToolListMeansNoTools` are active, this still builds the skills section in `catalog-only` mode and later emits `## Skills (mandatory)` plus the catalog. That contradicts the no-tools prompt and can still leak skill-only guidance into sessions that should be chat-only.
  - confidence: 0.93
- **[P2] Report skills from the rendered prompt:** `src/agents/system-prompt-report.ts:149-174`
  - body: `buildSystemPromptReport` still parses `params.skillsPrompt` and reports `promptChars` from the raw input. If the prompt builder or provider transform omits the Skills section, `/context` will report skill bytes and entries the model did not receive.
  - confidence: 0.9
- **[P2] Gate provider tool guidance in no-tool prompts:** `src/agents/system-prompt.ts:944-968`
  - body: Explicit no-tool prompts still apply provider prompt overrides and stable prefixes. For OpenAI GPT-5, that reintroduces tool-oriented `tool_discipline` and heartbeat guidance even though the same prompt says no tools are available.
  - confidence: 0.84
