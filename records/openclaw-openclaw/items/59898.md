## Summary

Keep open: the reported prompt/report mismatch is still valid on current main and this PR has useful live proof, but the branch still leaves contradictory no-tool behavior in the rendered skills block, prompt report, and provider-owned GPT-5 tool guidance.

## What This Changes

The PR changes agent system-prompt and prompt-report handling for explicit no-tool and hosted client-tool sessions, plus client tool validation in the OpenResponses path.

## Best Possible Solution

Land a focused repair that makes explicit no-tool sessions render a true no-tool prompt and report, while leaving broader effective-tool diagnostics to https://github.com/openclaw/openclaw/issues/44484 and payload stripping to https://github.com/openclaw/openclaw/pull/70790.

## Reproduction Assessment

Yes, source-reproducible. The PR body gives a concrete live CLI no-tools repro, and current main source shows the empty-tool fallback catalog plus raw skills-reporting path that explain the failure; I did not execute the live CLI repro in this read-only review.

## Solution Assessment

No. The direction is maintainable, but the current branch still leaks the skills catalog, reports raw skills that may not be rendered, and injects provider-owned tool guidance in explicit no-tool sessions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Suppress skills for explicit no-tool prompts:** `src/agents/system-prompt.ts:837-846`
  - body: When `toolNames: []` and `explicitEmptyToolListMeansNoTools` are active, this still builds `skillsSection` in `catalog-only` mode and later inserts it into the prompt. That leaves `## Skills (mandatory)` and the raw catalog in a session that says no tools are available, so the model still sees skill-selection content this PR is meant to remove.
  - confidence: 0.94
- **[P2] Derive skills stats from the rendered prompt:** `src/agents/system-prompt-report.ts:149`
  - body: `buildSystemPromptReport` still parses `params.skillsPrompt`, and the returned `promptChars` also uses the raw input length. If prompt construction or provider transforms remove the Skills section, `/context` can still report skill bytes and entries that were never sent to the model.
  - confidence: 0.93
- **[P2] Skip tool-driving provider prefixes without tools:** `src/agents/system-prompt.ts:966-967`
  - body: The explicit no-tools path still injects `providerStablePrefix` unconditionally. For GPT-5/OpenAI sessions that prefix includes tool-discipline instructions, so a prompt can still combine `No tools are available` with provider-owned guidance to prefer or continue tool calls.
  - confidence: 0.86
