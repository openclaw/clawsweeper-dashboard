## Summary

Keep open: current main still has the reported prompt/report mismatch, and PR head is aimed at the right bug but still leaves explicit no-tool sessions with skills, report, and provider-prefix contradictions.

## What This Changes

The PR changes agent system-prompt and prompt-report handling for explicit empty tool lists, hosted client tools, and OpenResponses client-tool validation.

## Best Possible Solution

Land a focused repair that makes explicit no-tool sessions render and report a true no-tool prompt, while leaving broader effective-tool diagnostics to https://github.com/openclaw/openclaw/issues/44484 and payload stripping to https://github.com/openclaw/openclaw/pull/70790.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current main falls back to a standard tool catalog and reports raw skills when the effective tool list is empty. The PR body also supplies copied before/after live output for the same no-tools CLI flow.

## Solution Assessment

No. The direction is maintainable, but this PR head still leaks skills, reports raw skills that may not be rendered, and injects provider-owned tool-discipline text into explicit no-tool sessions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Suppress skills for explicit no-tool prompts:** `src/agents/system-prompt.ts:837-846`
  - body: When `toolNames: []` and `explicitEmptyToolListMeansNoTools` are active, this still builds `skillsSection` from any non-empty `skillsPrompt` and later inserts it into the rendered prompt. That leaves `## Skills (mandatory)` and the catalog in a session that says no tools are available, so the model still sees skill-selection content this PR is meant to remove.
  - confidence: 0.94
- **[P2] Derive skills stats from the rendered prompt:** `src/agents/system-prompt-report.ts:149`
  - body: `buildSystemPromptReport` still parses `params.skillsPrompt`, and `promptChars` still uses the raw input length. If prompt construction or provider transforms remove the Skills section, `/context` can still report skill bytes and entries that were never sent to the model.
  - confidence: 0.93
- **[P2] Gate provider stable prefixes without tools:** `src/agents/system-prompt.ts:965-967`
  - body: The explicit no-tools path still injects `providerStablePrefix` unconditionally. For GPT-5/OpenAI sessions that prefix includes tool-discipline instructions, so a prompt can still combine `No tools are available` with provider-owned guidance to prefer or continue tool calls.
  - confidence: 0.86
