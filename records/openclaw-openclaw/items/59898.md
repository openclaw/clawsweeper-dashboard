## Best Possible Solution

Repair and rebase this branch, or replace it with a narrow current-main branch, so explicit no-tool prompts omit tool-only skills/provider guidance and prompt reports derive skills from rendered prompt content.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Suppress skills in explicit no-tool prompts:** `src/agents/system-prompt.ts:837-846`
  - body: When `toolNames` is explicitly empty, this still builds and inserts a Skills section from any non-empty `skillsPrompt`. That leaves a skill catalog in a prompt that says no tools are available, which is the leakage the PR body says should be removed.
  - confidence: 0.94
- **[P2] Report only rendered skills content:** `src/agents/system-prompt-report.ts:149`
  - body: `buildSystemPromptReport` still parses and counts the raw `params.skillsPrompt`. If prompt construction or a provider transform omits the Skills section, `/context` can report skill bytes and entries that were never sent to the model.
  - confidence: 0.93
- **[P2] Gate provider stable prefixes without tools:** `src/agents/system-prompt.ts:965-967`
  - body: The explicit no-tools path still injects `providerStablePrefix` unconditionally. The built-in GPT-5 stable prefix contains tool-discipline instructions, so no-tool sessions can still be told to continue with unavailable tool calls.
  - confidence: 0.88
