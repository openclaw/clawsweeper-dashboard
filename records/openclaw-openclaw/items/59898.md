## Summary

Keep this PR open: the reported explicit no-tools prompt bug is real and current main still needs a fix, but the current branch still leaves tool-only guidance and inaccurate skill reporting in the no-tools path.

## What This Changes

This PR changes agent system-prompt construction, prompt-report metadata, embedded-runner prompt plumbing, OpenResponses client-tool validation, and focused tests for explicit empty-tool and hosted-client-tool sessions.

## Best Possible Solution

Repair this branch so explicit no-tool sessions omit all skills and tool-only provider guidance, while prompt reports derive skill stats from the rendered prompt and hosted-client-tool sessions remain truthful.

## Reproduction Assessment

Yes. The PR body gives a concrete no-tools `openclaw agent --local` reproduction with before/after prompt-report output, and source inspection corroborates the remaining prompt/report paths; I did not run it in this read-only review.

## Solution Assessment

No, not at the current PR head. The direction is the right narrow owner-boundary fix, but the implementation still leaks skills/provider tool guidance and reports raw skills input, so it needs a focused repair before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Suppress skills for explicit no-tool prompts:** `src/agents/system-prompt.ts:837-846`
  - body: When `toolNames` is explicitly empty, `buildSkillsSection` still runs for any non-empty `skillsPrompt` and inserts a Skills section into the final prompt. That leaves a skills catalog in a prompt that says no tools are available, which is the leak this PR is meant to remove.
  - confidence: 0.94
- **[P2] Report skills from the rendered prompt:** `src/agents/system-prompt-report.ts:149-174`
  - body: `buildSystemPromptReport` still parses and measures `params.skillsPrompt` instead of the skills block actually present in `systemPrompt`. If prompt construction or provider transforms omit the Skills section, `/context` can show skills that were never sent to the model.
  - confidence: 0.93
- **[P2] Gate provider stable prefixes without tools:** `src/agents/system-prompt.ts:965-967`
  - body: The explicit no-tools path gates execution-bias overrides, but still injects `providerStablePrefix` unconditionally. The GPT-5 stable prefix contains tool-discipline text, so no-tool sessions can still receive instructions to prefer tool evidence or make additional tool calls.
  - confidence: 0.88
