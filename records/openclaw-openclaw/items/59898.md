## Summary

Keep open: this PR targets a real prompt/report consistency bug and includes sufficient live-output proof, but the current head still leaks skills in explicit no-tool prompts, reports raw skill input, and can inject GPT-5 tool-discipline guidance when no tools are available.

## What This Changes

The PR changes agent system-prompt construction, prompt-report metadata, embedded-runner client-tool context, OpenResponses client-tool validation, and focused tests for explicit no-tool and hosted-client-tool sessions.

## Best Possible Solution

Repair this branch so explicit no-tool sessions omit skills and all tool-only provider guidance, prompt reports measure only the rendered prompt, and hosted-client-tool sessions remain truthful.

## Reproduction Assessment

Yes. The PR body gives a concrete `openclaw agent --local` no-tools reproduction with before/after prompt-report output, and source inspection shows the current-main raw skills/fallback paths still exist; this read-only review did not execute the repro.

## Solution Assessment

No, not at the current PR head. The direction is the right narrow fix, but the implementation still renders skills, reports raw skills input, and injects provider stable-prefix tool guidance in no-tool sessions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Suppress skills for explicit no-tool prompts:** `src/agents/system-prompt.ts:837-846`
  - body: When `toolNames` is explicitly empty, this still builds `skillsSection` from any non-empty `skillsPrompt` and later appends it. That means a prompt can say no tools are available while still leaking the skills catalog, which is the core behavior this PR is meant to remove.
  - confidence: 0.94
- **[P2] Report skills from the rendered prompt:** `src/agents/system-prompt-report.ts:149`
  - body: `buildSystemPromptReport` still parses raw `params.skillsPrompt` and reports its byte count. If prompt construction or provider transforms omit the Skills section, `/context` can show skill names and prompt bytes that were never sent to the model.
  - confidence: 0.93
- **[P2] Gate provider stable prefixes without tools:** `src/agents/system-prompt.ts:965-967`
  - body: The explicit no-tools path gates the execution-bias override, but still injects `providerStablePrefix` unconditionally. The GPT-5 stable prefix contains tool-discipline text, so no-tool sessions can still receive instructions to prefer tool evidence and continue tool calls.
  - confidence: 0.88
