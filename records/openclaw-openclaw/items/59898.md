## Summary

Keep open: this addresses a real prompt/report bug and the PR body has sufficient live proof, but the current PR head still leaves explicit no-tool sessions with leaked skills metadata and provider tool-oriented prompt text.

## What This Changes

The PR changes agent system-prompt/report construction, embedded-runner hosted-client-tool plumbing, and OpenResponses client-tool validation for explicit empty-tool and hosted-tool-only sessions.

## Best Possible Solution

Land a narrow prompt/report repair that treats explicit empty effective tools as chat-only, derives report skills from rendered prompt content, and keeps hosted-client-tool guidance distinct from OpenClaw built-in tool guidance.

## Reproduction Assessment

Yes. The PR body gives a concrete live CLI before/after repro, and source inspection shows current main and PR head can still disagree between effective no-tool state, rendered skills, and reported skills.

## Solution Assessment

No, not at the current PR head. The direction is right, but the implementation still needs to suppress rendered skills, report only rendered skills, and avoid tool-driving provider stable prefixes in explicit no-tool sessions.
