Repair this PR or replace it with a narrow branch that makes explicit no-tool sessions omit tool-only skills/provider guidance and reports only rendered prompt contents, while leaving payload stripping to https://github.com/openclaw/openclaw/pull/70790 and broader tool diagnostics to https://github.com/openclaw/openclaw/issues/44484.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Suppress skills in explicit no-tool prompts:** `src/agents/system-prompt.ts:837-846`
  - body: When `toolNames` is explicitly empty, this still builds `skillsSection` from any non-empty `skillsPrompt` and later inserts it into the prompt. That leaves the skills catalog in a prompt that says no tools are available, so the model still receives skill-selection guidance this PR is meant to remove.
  - confidence: 0.94
- **[P2] Report only rendered skills content:** `src/agents/system-prompt-report.ts:149`
  - body: `buildSystemPromptReport` still parses and counts `params.skillsPrompt` directly. If prompt construction or a provider transform omits the Skills section, `/context` can report skill bytes and entries that were never sent to the model.
  - confidence: 0.93
- **[P2] Skip provider stable prefixes without tools:** `src/agents/system-prompt.ts:965-967`
  - body: The explicit no-tools path still injects `providerStablePrefix` unconditionally. Provider prefixes can include tool-discipline guidance, so the final prompt can still combine no-tools wording with provider-owned instructions to use tools.
  - confidence: 0.88
