## Summary

Keep open. Current main still has the prompt/report mismatch for explicit no-tool sessions, and this PR is the closest focused fix, but PR head still leaks skills catalog guidance and raw skills diagnostics, so it should not merge until those blockers are fixed; https://github.com/openclaw/openclaw/pull/70790 covers payload stripping rather than this prompt/report surface.

## What This Changes

The PR changes agent system-prompt construction, prompt reporting, embedded-runner plumbing, and OpenResponses client-tool handling so explicit no-tool and hosted-client-tool-only sessions are advertised consistently.

## Best Possible Solution

Keep the focused no-tools prompt/report fix, but suppress skill catalog/tool-only prompt content for empty effective tool sets and derive skills diagnostics from the final rendered prompt; leave provider payload stripping to the separate payload-focused work.

## Reproduction Assessment

Yes. The PR body gives a concrete live CLI repro using a deny-all tools agent and `/context detail`, and current-main source inspection shows both the fallback tool catalog and raw skills-report paths still exist.

## Solution Assessment

No. The direction is right, but this branch still needs narrow source/test repairs so explicit no-tool prompts do not include skills catalogs and prompt reports match the rendered prompt.
