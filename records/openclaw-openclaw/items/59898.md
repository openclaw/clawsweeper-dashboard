## Summary

Keep open: current main still has the prompt/report inconsistency, and the PR head is still not mergeable as-is because explicit no-tool prompts can still render skills, report raw skill bytes, and include provider tool-discipline guidance.

## What This Changes

The PR updates agent system-prompt construction, prompt-report accounting, embedded-runner client-tool plumbing, OpenResponses client-tool validation, and focused tests for explicit no-tool and hosted-client-tool sessions.

## Best Possible Solution

Repair and rebase the branch so explicit no-tool sessions omit all skill and tool-only provider guidance, while prompt reports measure only what was actually rendered and hosted-client-tool sessions stay truthful.

## Reproduction Assessment

Yes. The PR body gives a concrete `openclaw agent --local` no-tools reproduction with before/after prompt-report output, and current-main source inspection confirms the raw skills/report paths remain; I did not run the live repro in this read-only review.

## Solution Assessment

No, not at the current PR head. The direction is the right narrow fix, but the implementation still needs to suppress skills, report rendered skills, and gate provider stable-prefix tool guidance for explicit no-tool sessions.
