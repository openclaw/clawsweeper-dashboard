## Summary

Keep open. Current main has not implemented the central explicit no-tools prompt/report behavior, and the current PR head still has blocking prompt/report defects that would preserve the reported mismatch.

## What This Changes

The PR changes agent system-prompt construction, prompt-report metadata, embedded-runner client-tool plumbing, OpenResponses client-tool validation, and focused tests for explicit no-tool and hosted-client-tool sessions.

## Best Possible Solution

Repair and rebase the branch so explicit no-tool sessions omit skills and all tool-only provider guidance, prompt reports measure only rendered prompt content, and hosted-client-tool sessions remain truthful.

## Reproduction Assessment

Yes. The PR body gives a concrete `openclaw agent --local` no-tools reproduction with before/after prompt-report output, and source inspection confirms current main still has the raw skills/report and fallback tool-guidance paths; I did not execute the live repro in this read-only review.

## Solution Assessment

No, not at the current PR head. The direction is the right narrow fix, but the implementation still renders skills, reports raw skills input, and injects provider stable-prefix tool guidance in no-tool sessions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Suppress skills for explicit no-tool prompts:** `src/agents/system-prompt.ts:837-846`
  - body: When `toolNames` is explicitly empty, this branch still builds `skillsSection` from a non-empty `skillsPrompt` using `catalog-only` mode. That lets the final prompt contain `## Skills` and `<available_skills>` while also saying no tools are available, preserving the core prompt leak this PR is meant to remove.
  - confidence: 0.94
- **[P2] Report skills from the rendered prompt:** `src/agents/system-prompt-report.ts:149`
  - body: `buildSystemPromptReport` still parses and measures raw `params.skillsPrompt`. If prompt construction or provider transforms omit the Skills section, `/context` can report skill names and prompt bytes that were never sent to the model.
  - confidence: 0.93
- **[P2] Gate provider stable prefixes without tools:** `src/agents/system-prompt.ts:965-967`
  - body: The explicit no-tools path gates the execution-bias override, but still injects `providerStablePrefix` unconditionally. The GPT-5 stable prefix contains tool-discipline text, so no-tool sessions can still receive instructions to prefer tool evidence and continue tool calls.
  - confidence: 0.88
