## Summary

Keep open: the reported bug is valid and the contributor provided sufficient live-output proof, but the current PR head still leaves tool-only skills/reporting/provider guidance in explicit no-tool sessions, so it should not merge as-is.

## What This Changes

The PR changes agent system-prompt/report construction, embedded-runner prompt plumbing, and OpenResponses client-tool handling so explicit empty tool lists and hosted client tools are represented consistently.

## Best Possible Solution

Repair this PR or replace it with a narrow branch so explicit no-tool sessions omit tool-only skills/provider guidance and report only rendered prompt contents, while leaving payload stripping to https://github.com/openclaw/openclaw/pull/70790 and broader tool diagnostics to https://github.com/openclaw/openclaw/issues/44484.

## Reproduction Assessment

Yes. The PR body gives a concrete `openclaw agent --local` no-tools repro with before/after live output, and current main plus PR-head source inspection explains the same failure path.

## Solution Assessment

No, not at the current PR head. The direction is maintainable, but the patch must suppress no-tool skills/provider guidance and derive skills reporting from rendered prompt contents before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Suppress skills in explicit no-tool prompts:** `src/agents/system-prompt.ts:837-846`
  - body: When `toolNames` is explicitly empty, this still builds `skillsSection` from any non-empty `skillsPrompt` and later inserts it into the prompt. That leaves the skills catalog in a prompt that says no tools are available, so the model still receives skill-selection guidance this PR is meant to remove.
  - confidence: 0.94
- **[P2] Derive skills stats from the rendered prompt:** `src/agents/system-prompt-report.ts:149`
  - body: `buildSystemPromptReport` still parses and counts `params.skillsPrompt` directly. If prompt construction or provider transforms omit the Skills section, `/context` can report skill bytes and entries that were never sent to the model.
  - confidence: 0.93
- **[P2] Skip provider stable-prefix guidance without tools:** `src/agents/system-prompt.ts:965-967`
  - body: The explicit no-tools path still injects `providerStablePrefix` unconditionally. For GPT-5/OpenAI sessions that stable prefix includes tool-discipline instructions, so the final prompt can still combine no-tools wording with provider-owned guidance to keep using tools.
  - confidence: 0.88
