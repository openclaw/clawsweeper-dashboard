## Summary

Keep this PR open, but it still needs changes before merge. Current main does not implement the requested no-tools prompt/report behavior, and the PR head still leaks the skills catalog, reports raw skills input, and injects provider tool guidance in explicit no-tool sessions.

## What This Changes

The PR changes agent system-prompt construction, prompt-report metadata, embedded-runner client-tool plumbing, OpenResponses client-tool validation, and focused tests for explicit no-tool and hosted-client-tool sessions.

## Best Possible Solution

Repair and rebase the branch so explicit no-tool sessions omit skills and all tool-only provider guidance, prompt reports measure only the rendered prompt, and hosted-client-tool sessions remain truthful.

## Reproduction Assessment

Yes. The PR body gives a concrete openclaw agent --local no-tools reproduction with before/after prompt-report output, and current-main source inspection confirms the raw skills/report paths still exist; I did not execute the live repro in this read-only review.

## Solution Assessment

No, not at the current PR head. The direction is the right narrow fix, but the implementation still renders skills, reports raw skills input, and injects provider stable-prefix tool guidance in no-tool sessions.
