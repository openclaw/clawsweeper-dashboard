## Summary

Keep open: current main does not implement the explicit no-tools prompt/report fix, and the PR still has concrete P2 defects that keep no-tool sessions from being truthful.

## What This Changes

The branch updates agent system-prompt/report generation, embedded-runner client-tool prompt context, and OpenResponses client-tool validation for explicit empty-tool and hosted-client-tool sessions.

## Best Possible Solution

Repair the branch so explicit no-tool sessions suppress all tool-only prompt sections and skills, report skills from rendered prompt content, and keep hosted-client-tool sessions truthful without broad capability redesign.

## Reproduction Assessment

Yes. The PR body provides a concrete live CLI no-tools repro with copied before/after prompt-report output, and source inspection of main and PR head corroborates the raw-skills and no-tool prompt paths.

## Solution Assessment

No, not at the current PR head. The direction is right, but the patch still leaks skills/provider tool guidance and reports raw skill input, so it needs a focused repair before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Suppress skills for explicit no-tool prompts:** `src/agents/system-prompt.ts:837-846`
  - body: When `toolNames` is explicitly empty, `buildSkillsSection` still runs for any non-empty `skillsPrompt` and inserts a catalog-only Skills section into the final prompt. That leaves a skills catalog in a prompt that says no tools are available, which is the leak this PR is intended to remove.
  - confidence: 0.94
- **[P2] Report skills from the rendered prompt:** `src/agents/system-prompt-report.ts:149`
  - body: `buildSystemPromptReport` still parses `params.skillsPrompt` and reports its raw length. If prompt construction or provider transforms omit the Skills section, `/context` can show skills that were never sent to the model.
  - confidence: 0.93
- **[P2] Gate provider stable prefixes without tools:** `src/agents/system-prompt.ts:965-967`
  - body: The explicit no-tools path still injects `providerStablePrefix` unconditionally. The GPT-5 stable prefix contains tool-discipline text, so no-tool sessions can still receive instructions to prefer tool evidence or make additional tool calls.
  - confidence: 0.88
