- **[P1] Suppress skills for explicit no-tool sessions:** `src/agents/system-prompt.ts:184-191`
  - body: The new no-tool path still lets the `catalog-only` skills branch append the full `skillsPrompt`, and the prompt builder constructs `skillsSection` even when the tool list is explicitly empty. A no-tool run with skills available can therefore still expose skill entries, which is the central behavior this PR is meant to prevent.
  - confidence: 0.93
- **[P1] Port prompt plumbing to the current runner:** `src/agents/pi-embedded-runner/run/attempt.ts:1143`
  - body: This branch wires `clientTools` and no-tool prompt state through `src/agents/pi-embedded-runner`, but current main uses `src/agents/embedded-agent-runner` and GitHub reports the PR as conflicting. Until this is ported to the current runner, the patch cannot merge or prove the runtime path that exists on main.
  - confidence: 0.96
- **[P2] Report rendered skills, not raw skill input:** `src/agents/system-prompt-report.ts:149`
  - body: `buildSystemPromptReport` still derives skill entries from `params.skillsPrompt`, even though this PR can omit or transform the skills section in the rendered prompt. That makes `/context` and prompt-budget diagnostics report skills that were not actually present in explicit no-tool or provider-transformed sessions.
  - confidence: 0.9
- **[P1] Preserve OpenResponses tool-name compatibility:** `src/gateway/openresponses-http.ts:270-281`
  - body: The new normalizer rejects client tool names that the current OpenResponses schema and docs accept as non-empty strings, including case/alias collisions, before the request reaches the agent. That changes existing `/v1/responses` clients from accepted requests to 400s without a documented compatibility contract.
  - confidence: 0.87
