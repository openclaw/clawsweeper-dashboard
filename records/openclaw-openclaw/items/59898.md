Repair this PR so explicit no-tool sessions omit skills and provider tool guidance and /context derives skills from the rendered prompt, while leaving payload stripping to https://github.com/openclaw/openclaw/pull/70790 and broader tool diagnostics to https://github.com/openclaw/openclaw/issues/44484.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P2] Suppress skills for explicit no-tool prompts:** `src/agents/system-prompt.ts:837-846`
  - body: When `toolNames: []` and `explicitEmptyToolListMeansNoTools` are active, this still builds `skillsSection` from a non-empty `skillsPrompt` and later inserts it into the prompt. That leaves the Skills section and raw catalog in a session that says no tools are available, so the model still sees skill-selection content this PR is meant to remove.
  - confidence: 0.94
- **[P2] Derive skills stats from the rendered prompt:** `src/agents/system-prompt-report.ts:149`
  - body: `buildSystemPromptReport` still parses and counts `params.skillsPrompt` directly. If prompt construction or provider transforms remove the Skills section, `/context` can still report skill bytes and entries that were never sent to the model.
  - confidence: 0.93
- **[P2] Skip provider tool guidance without tools:** `src/agents/system-prompt.ts:965-967`
  - body: The explicit no-tools path still injects `providerStablePrefix` unconditionally. For GPT-5/OpenAI sessions that prefix includes tool-discipline instructions, so a prompt can still combine `No tools are available` with provider-owned guidance to keep using tools.
  - confidence: 0.87
