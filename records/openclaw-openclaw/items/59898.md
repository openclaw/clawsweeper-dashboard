## Summary

Keep open: current main still has the explicit-empty-tools prompt/report mismatch, and the PR has useful real repro proof, but the PR head still leaves no-tool prompts and reports internally inconsistent.

## What This Changes

The branch changes agent system prompt/report construction, embedded-runner tool context plumbing, and OpenResponses client-tool normalization for explicit no-tool and hosted-client-tool sessions.

## Best Possible Solution

Repair and rebase this branch, or replace it with a narrow current-main branch, so explicit no-tool prompts omit tool-only skills/provider guidance and prompt reports derive skills from rendered prompt content.

## Reproduction Assessment

Yes, source-reproducible. The PR body gives a concrete live CLI no-tools repro with before/after output, and current main source still renders fallback tool catalog/skills and raw skills reporting when the tool list is empty.

## Solution Assessment

No, not at the current PR head. The direction is maintainable, but the patch still leaks skills/provider tool guidance and reports raw skills, so it needs focused repair before merge.
