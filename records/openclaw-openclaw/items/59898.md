## Summary

Keep open: the PR is a plausible bug fix with sufficient live proof, but the latest head still leaves the central no-tools prompt/report mismatch in place. The remaining blockers are narrow and source-backed, so this should be repaired rather than closed or merged as-is.

## What This Changes

The PR changes agent system-prompt construction, system-prompt reporting, embedded-runner prompt plumbing, and OpenResponses client-tool validation for explicit empty-tool and hosted-client-tool-only sessions.

## Best Possible Solution

Keep the focused no-tools prompt/report fix, but make explicit empty-tool sessions suppress all skill catalog/tool-only prompt content and derive skills diagnostics from the final rendered prompt; leave broader tool visibility to https://github.com/openclaw/openclaw/issues/44484 and payload stripping to https://github.com/openclaw/openclaw/pull/70790.

## Reproduction Assessment

Yes. The PR body gives a concrete live CLI repro using a deny-all tools agent and `/context detail`, and source inspection shows the current-main and PR-head code paths that still create prompt/report disagreement.

## Solution Assessment

No. The direction is the right maintainable boundary, but the current branch still needs narrow source/test repairs so no-tool prompts and prompt reports agree.
