## Summary

This is a valid bug-fix direction with sufficient live-output proof, but the current PR head still leaves two correctness blockers: explicit no-tool prompts can still include the skills catalog, and prompt reports still count raw skills input instead of the rendered prompt.

## What This Changes

The branch changes agent system-prompt construction/reporting, embedded-runner plumbing, and OpenResponses client-tool validation so explicit no-tool and hosted-client-tool-only sessions advertise tool availability more consistently.

## Best Possible Solution

Land a focused prompt/report fix that suppresses all skill catalog content for true no-tool sessions and derives skills diagnostics from the final rendered prompt; leave broader capability visibility to https://github.com/openclaw/openclaw/issues/44484 and payload stripping to https://github.com/openclaw/openclaw/pull/70790.

## Reproduction Assessment

Yes. The PR body gives a concrete live CLI repro using a deny-all tools agent, and source inspection shows current main and PR-head code paths that still create prompt/report disagreement.

## Solution Assessment

No. The direction is the right maintainable boundary, but the current branch still needs narrow repairs so explicit no-tool prompts contain no skills catalog and prompt reports reflect the rendered prompt.
