## Summary

Keep this PR open, but it still needs changes before merge. The branch targets a real prompt/report bug, has useful live proof, and current main does not already implement it; however the current PR head still leaves explicit no-tool prompts able to include skills/tool-oriented provider text and still reports skills from raw input.

## What This Changes

The PR changes agent system-prompt construction, prompt-report metadata, embedded-runner client-tool plumbing, OpenResponses client-tool validation, and regression tests for explicit no-tool and hosted-client-tool sessions.

## Best Possible Solution

Repair this branch so explicit no-tool sessions omit all skills and tool-only provider guidance, prompt reports derive skill stats from rendered prompt content, and hosted-client-tool sessions remain truthful.

## Reproduction Assessment

Yes. The PR body gives a concrete openclaw agent --local no-tools reproduction with before/after prompt-report output, and current-main source inspection shows the same raw skills prompt/report paths still exist; I did not execute the repro in this read-only review.

## Solution Assessment

  - body: When `toolNames` is explicitly empty, this branch still builds and appends `skillsSection` from any non-empty `skillsPrompt`, so the final prompt can contain the skills catalog while also saying no tools are available. That leaves the core leak this PR is meant to remove.

  - confidence: 0.88
