## Summary

Keep open: current main still has the explicit empty-tools prompt/report bug, and this PR is a plausible fix path, but the latest head still leaks skills, reports raw skills that may not be rendered, and can inject provider-owned tool-discipline text into no-tool prompts.

## What This Changes

The PR changes agent system-prompt/report construction, embedded-runner prompt plumbing, OpenResponses client-tool validation, and tests so explicit empty-tool and hosted-tool-only sessions receive truthful prompt/report data.

## Best Possible Solution

Repair this PR so explicit no-tool sessions omit skills and provider tool guidance and `/context` derives skills from the rendered prompt, while leaving payload stripping to https://github.com/openclaw/openclaw/pull/70790 and broader tool diagnostics to https://github.com/openclaw/openclaw/issues/44484.

## Reproduction Assessment

Yes, source-reproducible with high confidence. The PR body gives a concrete no-tools CLI repro and current main source still has the fallback tool catalog plus raw skills-reporting path that explain the failure.

## Solution Assessment

No, not as currently implemented. The direction is the right maintainable fix, but the current PR head still leaves three narrow prompt/report inconsistencies that should be fixed before merge.

Full review comments:

- **[P2] Suppress skills for explicit no-tool prompts:** `src/agents/system-prompt.ts:837-846`
  - body: When `toolNames: []` and `explicitEmptyToolListMeansNoTools` are active, this still builds `skillsSection` from a non-empty `skillsPrompt` and later inserts it into the prompt.   - confidence: 0.94
- **[P2] Derive skills stats from the rendered prompt:** `src/agents/system-prompt-report.ts:149`

  - confidence: 0.93
- **[P2] Skip provider prefixes without tools:** `src/agents/system-prompt.ts:966-967`
  - body: The explicit no-tools path still injects `providerStablePrefix` unconditionally. For GPT-5/OpenAI sessions that prefix includes tool-discipline instructions, so a prompt can still combine `No tools are available` with provider-owned guidance to prefer additional tool work.
  - confidence: 0.87
