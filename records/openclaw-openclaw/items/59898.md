Full review comments:

- **[P1] Keep hosted tools out of native alias normalization:** `src/agents/command/shared-types.ts:50-52`
  - body: `normalizeToolName` is the native OpenClaw tool-policy normalizer, so it folds aliases and case in ways the OpenResponses hosted-tool API has not promised. This makes hosted tools like `Foo`/`foo` or `bash`/`exec` fail as duplicates even though current main forwards them, so keep hosted-tool uniqueness exact after sanitization or split this public API policy for explicit approval.
  - confidence: 0.89
- **[P2] Omit skills from true no-tool prompts:** `src/agents/system-prompt.ts:837-846`
  - body: The explicit no-tool path still builds and spreads `skillsSection`, and the added test expects `## Skills (mandatory)` to remain. That leaks the skills catalog and skill-reading guidance into sessions whose tools were explicitly disabled; gate the skills section out for true no-tool prompts.
  - confidence: 0.94
- **[P2] Report skills from rendered prompt state:** `src/agents/system-prompt-report.ts:149`
  - body: `buildSystemPromptReport` still parses `params.skillsPrompt` directly, so once the renderer correctly omits skills for no-tool sessions, `/context` can continue showing skill bytes and entries that were not in the final prompt. Derive skill stats from the rendered prompt or structured render metadata instead of the raw catalog input.
  - confidence: 0.84
- **[P2] Gate provider tool-discipline text for no-tool sessions:** `src/agents/system-prompt.ts:965-967`
  - body: The no-tools path suppresses `execution_bias` overrides but still injects `providerStablePrefix`. The built-in GPT-5 stable prefix contains `<tool_discipline>` instructions that prefer tool evidence and more tool work before replying, so explicit no-tool sessions can still receive tool-driving text.
  - confidence: 0.86
