## What This Changes

The branch updates agent system-prompt/report construction, embedded-runner hosted-tool plumbing, and OpenResponses client-tool validation for explicit no-tool and hosted-client-tool sessions.

## Best Possible Solution

Land a narrow prompt/report repair that treats explicit empty effective tools as chat-only while preserving omitted-tool legacy behavior and hosted-client-tool guidance.

## Reproduction Assessment

Yes. The PR body gives concrete live CLI before/after prompt-report output, and source inspection confirms current main and PR head can still diverge between effective no-tool state, rendered skills, and reported skills.

## Solution Assessment

No, not at the current PR head. The direction is right, but the implementation must suppress rendered skills, report only rendered skills, and avoid tool-driving provider stable prefixes in explicit no-tool sessions.
