## Summary

Keep open. Current main does not implement this no-tools prompt/report fix, and the latest PR head still leaves explicit no-tool sessions inconsistent in three narrow ways that should block merge.

## What This Changes

The PR changes agent system-prompt construction, prompt-report metadata, embedded-runner client-tool plumbing, OpenResponses client-tool validation, and focused tests for explicit no-tool and hosted-client-tool sessions.

## Best Possible Solution

Repair this branch so explicit no-tool sessions omit skills and all tool-only provider guidance, prompt reports reflect only the rendered prompt, and hosted-client-tool sessions remain truthful.

## Reproduction Assessment

Yes. The PR body gives a concrete `openclaw agent --local` no-tools reproduction with before/after prompt-report output, and current-main source inspection shows the raw skills prompt/report paths still exist; this read-only review did not execute the repro.

## Solution Assessment

No, not at the current PR head. The direction is the right narrow fix, but the implementation still renders skills, reports raw skills input, and injects provider stable-prefix tool guidance in no-tool sessions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Suppress skills for explicit no-tool prompts:** `src/agents/system-prompt.ts:837-846`
  - body: When `toolNames` is explicitly empty, this branch still calls `buildSkillsSection` with any non-empty `skillsPrompt`, so the final prompt can contain the skills catalog while also saying no tools are available. That keeps the core prompt leak this PR is meant to remove.
  - confidence: 0.94
- **[P2] Report skills from the rendered prompt:** `src/agents/system-prompt-report.ts:149`
  - body: `buildSystemPromptReport` still parses and measures raw `params.skillsPrompt`. If prompt construction or provider transforms omit the Skills section, `/context` can report skills and prompt bytes that were never sent to the model.
  - confidence: 0.93
- **[P2] Gate provider stable prefixes without tools:** `src/agents/system-prompt.ts:965-967`
  - body: The explicit no-tools path gates the execution-bias override, but still injects `providerStablePrefix` unconditionally. The GPT-5 stable prefix contains tool-discipline text, so no-tool sessions can still receive instructions to prefer tool evidence and continue tool calls.
  - confidence: 0.88
