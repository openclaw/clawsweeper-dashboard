## Summary

Keep this PR open: it fixes a real prompt/report bug area, but the latest head still leaves two blocking consistency issues in the explicit no-tools path.

## What This Changes

The branch changes agent system prompt/report construction and OpenResponses hosted-client-tool plumbing to distinguish explicit empty tool lists, hosted client tools, and legacy omitted tool lists.

## Best Possible Solution

Keep the no-tools prompt/report direction, but suppress Skills catalog content for explicit empty effective tool sets and derive skill diagnostics from the final rendered prompt or structured render metadata.

## Reproduction Assessment

Yes. The PR body gives a concrete live CLI repro using a tools-deny-all agent, and source inspection shows current main still has the empty-list fallback and raw skills-report paths.

## Solution Assessment

No. The PR targets the right boundary, but the current branch is not ready until no-tool prompt content and report stats are aligned with the same final effective prompt state.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Suppress skill catalogs in explicit no-tool prompts:** `src/agents/system-prompt.ts:837-846`
  - body: When `toolNames: []` and `explicitEmptyToolListMeansNoTools` are active, this still builds `skillsSection` from `skillsPrompt` in `catalog-only` mode and later includes it in the prompt. That leaves `## Skills (mandatory)` and skill catalog guidance in sessions that also say no tools are available, so the model still sees tool-only skill guidance the PR is meant to remove.
  - confidence: 0.91
- **[P2] Derive skill stats from the rendered prompt:** `src/agents/system-prompt-report.ts:149-174`
  - body: `buildSystemPromptReport` still parses `params.skillsPrompt` and reports `promptChars` from the raw input. If the prompt builder or a provider transform removes the Skills section, `/context` will report skill bytes and entries the model did not receive, leaving the diagnostics bug in place.
  - confidence: 0.9
