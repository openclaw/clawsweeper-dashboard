## What This Changes

The PR adds device public-key proof verification to bootstrap-token auth, forwards the verified handshake signature into the verifier, moves bootstrap token binding through pairing approval, and updates gateway/pairing tests.

## Best Possible Solution

Keep the verifier and gateway proof boundary, but preserve fresh setup-code auto-approval and bounded handoff semantics before merge.

Yes. Source inspection is high-confidence: current main expects fresh setup-code bootstrap to auto-approve and return `hello-ok`, while PR head forces bootstrap-token pairing through `PAIRING_REQUIRED` before manual approval.

## Solution Assessment

No. Passing public-key proof into the verifier is the right boundary, but replacing the established setup-code handoff with interactive approval is not the narrowest maintainable fix.
