## Summary

Keep open: current main documents the OpenAI/Codex provider/runtime split and channel picker runtime choices have improved, but the WebUI chat model picker still consumes a `models.list` catalog shape that has no effective runtime or auth-route metadata to render.

## What This Changes

The issue requests clearer GPT/OpenAI Codex model picker labels that distinguish provider/auth route from agent runtime.

## Best Possible Solution

Add an additive model-choice metadata path for effective provider, auth route/profile family, and agent runtime, then update WebUI and relevant picker surfaces to display it without changing canonical provider/model refs or existing aliases.

## Reproduction Assessment

Yes for the central UI/API limitation: source inspection shows the WebUI picker payload and label logic lack runtime/auth-route metadata. The exact latency/auth-path outcome from the reporter's beta install was not live-reproduced.

## Solution Assessment

Unclear as an immediate implementation path: the requested direction is valid, but the maintainable fix should be an additive gateway/catalog metadata design rather than manual alias generation alone.
