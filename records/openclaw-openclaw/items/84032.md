## Summary

Keep open. Current main has fixed some channel/runtime transparency paths, but the WebUI `models.list` catalog still does not carry effective runtime or auth-route metadata for the model picker, so the reported ambiguity remains a real product/API gap.

## What This Changes

The issue asks for the model picker to distinguish provider/auth route and effective agent runtime for GPT/Codex model choices instead of showing an ambiguous model label.

## Best Possible Solution

Add an additive model-choice metadata contract for effective runtime and auth route/display hints, then update WebUI and any picker surfaces to render it while preserving canonical provider/model refs and existing aliases.

## Reproduction Assessment

Yes for the central source-level limitation: current `models.list` and WebUI catalog labels lack effective runtime/auth-route metadata. The reporter's exact beta latency and auth-route selection was not live-reproduced.

## Solution Assessment

Unclear as a final implementation: the requested direction is valid, but the maintainable fix should be an additive gateway/catalog metadata design rather than alias generation alone.
