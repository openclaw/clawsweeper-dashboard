## Summary

Keep open. Current main already documents the OpenAI/Codex provider/runtime split, and some channel surfaces expose runtime separately, but the WebUI chat model catalog path still only has model/provider/alias-style display data and no effective runtime/auth-route metadata for the picker.

## What This Changes

The issue asks the model picker and configuration flow to make GPT/Codex provider/auth route and agent runtime visible instead of collapsing materially different routes into the same apparent model choice.

## Best Possible Solution

Add an additive model-choice metadata path for effective provider, auth route/profile family, and agent runtime, then update WebUI and relevant picker surfaces to display it without changing canonical provider/model refs or existing aliases.

## Reproduction Assessment

Yes for the central UI/API limitation: source inspection shows the model picker payload and label logic lack runtime/auth-route metadata. The exact latency/auth-path outcome from the reporter's beta install was not live-reproduced.

## Solution Assessment

Unclear as an implementation path: the requested direction is valid, but the best fix should be an additive gateway/catalog metadata design rather than manual alias generation alone.
