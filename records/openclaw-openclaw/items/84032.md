## Summary

Keep open: current main has fixed one channel-side runtime readback problem, but the WebUI `models.list` catalog and chat picker still only carry model/provider/alias-style fields, so they cannot reliably display the effective GPT/Codex auth route and agent runtime.

## What This Changes

The issue requests clearer model-picker display for GPT/Codex entries by distinguishing provider, auth route, and agent runtime.

## Best Possible Solution

Add an additive model-choice metadata contract for effective provider, auth route/profile family, and agent runtime, then update WebUI and relevant picker displays without changing canonical model refs or existing aliases.

## Reproduction Assessment

Yes for the source-level limitation: current `models.list`, `ModelChoiceSchema`, and WebUI catalog types lack effective runtime/auth-route metadata. The reporter's exact beta runtime latency path was not live-reproduced.

## Solution Assessment

Unclear as an implementation path: the requested direction is valid, but the best fix needs an additive gateway/catalog metadata design rather than manual alias generation alone.
