## Summary

Keep open: current main documents the OpenAI/Codex provider/runtime split and has related channel/CLI fixes, but the WebUI `models.list` catalog still lacks effective runtime/auth-route metadata for the picker to distinguish those routes.

## What This Changes

The issue requests clearer WebUI/channel model picker display for GPT provider/auth routes and agent runtimes such as OpenAI, OpenAI-Codex OAuth, PI, and Codex.

## Best Possible Solution

Add an additive model-choice metadata contract for effective provider/auth route and agent runtime, then render it in WebUI and relevant picker surfaces without changing canonical provider/model refs or existing aliases.

## Reproduction Assessment

Yes for the source-level limitation: current `models.list`, the WebUI catalog type, and picker label logic lack runtime/auth-route metadata. I did not live-reproduce the reporter's exact beta-install latency or route outcome.

## Solution Assessment

Unclear as an implementation until maintainers choose the public metadata shape. The direction is sound, but manual alias generation alone would be a workaround rather than the best durable fix.
