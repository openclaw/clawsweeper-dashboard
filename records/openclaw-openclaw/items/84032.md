## Summary

Keep open: current main documents and partially exposes the OpenAI/Codex runtime split, but the WebUI `models.list` model-choice contract still lacks effective runtime/auth-route metadata for the picker.

## What This Changes

The issue requests clearer model picker display of GPT provider/auth route and agent runtime so distinct OpenAI, OpenAI-Codex, and Codex runtime routes are not collapsed under the same visible model name.

## Best Possible Solution

Add an additive model-choice metadata path for effective provider, auth route/profile family, and agent runtime, then update WebUI and relevant picker surfaces to display it without changing canonical provider/model refs or existing aliases.

## Reproduction Assessment

Yes for the central UI/API limitation: source inspection shows the `models.list` model-choice payload and WebUI label logic still lack runtime/auth-route metadata. The reporter's exact beta latency/auth route was not live-reproduced.

## Solution Assessment

Unclear as a concrete implementation path: the requested direction is valid, but the maintainable fix should be additive gateway/catalog metadata rather than only forcing exact refs or relying on manual aliases.
