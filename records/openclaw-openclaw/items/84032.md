## Summary

Keep open. Current main still documents provider/model/runtime as separate OpenClaw concepts, but the WebUI `models.list` and chat model picker contract still expose only model/provider/alias-style data, so the picker cannot reliably distinguish OpenAI API, OpenAI Codex OAuth/PI, and Codex runtime routes without a product/protocol decision.

## What This Changes

Request that model picker surfaces distinguish GPT provider/auth route and agent runtime instead of showing ambiguous `gpt-5.5` entries.

## Best Possible Solution

Add additive model-choice metadata for effective provider/auth route/profile family and agent runtime, then update WebUI and picker surfaces to render that metadata without changing canonical provider/model refs or relying on manual aliases.

## Reproduction Assessment

Yes for the central source-level limitation: current `models.list` and WebUI picker types lack runtime/auth-route metadata, and label logic only has alias/name/provider data. No live reproduction was performed for the reporter's exact slower-route runtime outcome.

## Solution Assessment

Unclear as an implementation path: the requested direction is valid, but the maintainable fix should be an additive gateway/catalog metadata design plus picker rendering rather than manual alias generation alone.
