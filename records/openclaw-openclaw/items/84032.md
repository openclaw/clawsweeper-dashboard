## Summary

Keep open. Current main still models picker choices as id/name/provider/alias catalog entries, while provider/auth route and agent runtime are documented as distinct layers but are not carried through the `models.list` model-choice contract for the WebUI picker.

## What This Changes

The issue requests clearer model-picker display and setup affordances for GPT/Codex provider/auth routes and agent runtime choices.

## Best Possible Solution

Add additive model-choice metadata for effective provider/auth route and agent runtime, then update WebUI and relevant picker surfaces to display that metadata without changing canonical provider/model refs or existing aliases.

## Reproduction Assessment

Yes for the central UI/API limitation: source inspection shows the model picker payload and label logic lack runtime/auth-route metadata. The exact latency/auth-path outcome from the reporter's beta install was not live-reproduced.

## Solution Assessment

Unclear as an implementation path: the requested direction is valid, but maintainers should choose the additive gateway/catalog metadata shape before implementation.
