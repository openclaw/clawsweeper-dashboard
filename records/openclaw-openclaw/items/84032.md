## Summary

Keep open. Current main documents and configures the OpenAI/Codex provider/runtime split, but the `models.list` catalog and Control UI picker still do not carry effective runtime or auth-route metadata, so the reported ambiguity remains real.

## What This Changes

The issue asks for GPT/Codex model picker entries to distinguish provider, auth route, and effective agent runtime instead of showing ambiguous duplicate model names.

## Best Possible Solution

Add additive model-choice metadata for effective provider/auth route/profile family and agent runtime, then update WebUI and channel picker surfaces to render it while preserving canonical model refs and existing aliases.

## Reproduction Assessment

Yes for the source-level limitation: the gateway model schema, UI catalog type, and picker label logic lack runtime/auth-route metadata. The exact latency/auth-path outcome from the reporter's beta install was not live-reproduced.

## Solution Assessment

Unclear as a concrete implementation path: the direction is valid, but the best fix should be an additive gateway/catalog metadata design rather than manual alias generation alone.
