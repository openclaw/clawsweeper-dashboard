## Summary

Keep open. Current main documents the OpenAI/Codex provider/runtime split and channel model surfaces have some effective-runtime metadata, but the WebUI `models.list` catalog path still lacks effective runtime/auth-route metadata and the Control UI picker can only label entries from alias/name/provider/id.

## What This Changes

Request that GPT/OpenAI-Codex model picker entries distinguish provider/auth route and agent runtime instead of collapsing materially different routes under labels like `gpt-5.5`.

## Best Possible Solution

Add additive `models.list`/catalog metadata for effective provider, auth route/profile family, agent runtime, and display hints, then update WebUI and relevant picker surfaces to render those facts while preserving canonical refs and existing aliases.

## Reproduction Assessment

Yes for the central API/UI limitation: source inspection shows the WebUI picker payload and label logic lack effective runtime/auth-route metadata. The reporter's exact beta install latency/auth-path outcome was not live-reproduced.

## Solution Assessment

Unclear as an implementation until maintainers choose the protocol and display contract. The narrow maintainable direction is additive model-choice metadata, not manual alias generation alone or a wholesale switch to exact-ref labels.
