## Summary

Keep open. Current main documents the OpenAI/Codex provider/runtime split and exposes agent runtime on some session/agent surfaces, but the `models.list` catalog and Control UI model picker still lack effective runtime/auth-route metadata for distinguishing GPT routes.

## What This Changes

The issue asks for the WebUI/channel model picker to distinguish GPT provider/auth route and agent runtime instead of collapsing distinct OpenAI/Codex routes into the same visible model label.

## Best Possible Solution

Add additive model-choice metadata for effective provider, auth route/profile family, and agent runtime, then update WebUI and relevant picker surfaces to display it while preserving canonical provider/model refs and existing aliases.

## Reproduction Assessment

Yes for the central UI/API limitation: source inspection shows the picker payload and label logic lack runtime/auth-route metadata. The reporter's exact beta latency/auth-path outcome was not live-reproduced.

## Solution Assessment

Unclear as a final implementation path: the requested direction is valid, but the maintainable fix should be an additive gateway/catalog metadata design rather than manual alias generation alone.
