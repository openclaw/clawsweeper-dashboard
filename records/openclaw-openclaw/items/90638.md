## Label Justifications

- `P1`: Slack users can lose the in-channel recovery path for wedged sessions, which is a broken channel workflow with real user impact.
- `impact:session-state`: The affected `/compact`, `/new`, and `/stop` commands mutate or recover session state.
- `impact:message-loss`: The reported Slack DM commands produce no useful command acknowledgement or effect, leaving user recovery messages effectively lost.

## Summary

Keep open: current main still has a Slack DM path where chat admission can be open while text command authorization remains tied to owner allowlists, so `/compact`, `/new`, and `/stop` can remain ineffective despite the documented standalone command contract.

## What This Changes

Requests that Slack DM text `/compact`, `/new`, and `/stop` execute or visibly fail instead of becoming inert plain messages when a session is wedged.

## Best Possible Solution

Restore Slack DM text recovery commands for accepted session senders with visible success or denial, add focused `/compact`, `/new`, and `/stop` Slack DM coverage, and preserve owner-only command boundaries.

## Reproduction Assessment

Yes, by source inspection: Slack DM chat admission can succeed while command authorization stays false for text control commands, and the docs expect standalone slash commands to work. I did not run a live Slack DM reproduction.

## Solution Assessment

Unclear until maintainers choose the exact Slack DM command authorization boundary. The best fix should execute or visibly reject session recovery commands without exposing owner-only commands to unintended senders.
