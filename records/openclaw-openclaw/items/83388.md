## Summary

Keep open. Current `main` still has the documented agent-level code-mode config rejected by the strict per-agent schema, and https://github.com/openclaw/openclaw/pull/83473 is an open closing PR that should be reviewed or closed before this issue is resolved.

## What This Changes

The issue asks OpenClaw to make documented `agents.list[].tools.codeMode` config valid and effective for a single agent, or to correct the docs if per-agent code mode is not intended.

## Best Possible Solution

Land or revise the linked focused fix so agent-scoped code mode is accepted by schema/types and actually controls the active agent's model-visible tool surface.

## Reproduction Assessment

Yes, from current-main source inspection. The documented `agents.list[].tools.codeMode` path flows into a strict agent tools schema that lacks `codeMode`, and runtime code still resolves code mode from root `config.tools.codeMode`.

## Solution Assessment

Yes, the best fix is to honor the documented per-agent contract across schema, exported types, and active-agent runtime resolution. A docs-only rollback would remove a useful documented rollout path, and a schema-only patch would be incomplete.
