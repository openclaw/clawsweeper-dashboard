## Summary

Keep this PR open: the core Kimi/OpenAI-compatible transcript normalization bug is still plausible on current main, but the branch still needs real provider proof and has two merge-blocking correctness/security concerns in the proposed diff.

## What This Changes

This PR adds normalization for function-prefixed or mangled tool names and Kimi-style tool-call IDs in agent transcript repair/sanitization, adds focused tests, and changes explicit node workdir approval routing.

## Best Possible Solution

Land a narrowed transcript/tool-call ID normalization that shares the existing unambiguous tool-name resolver, leaves node exec routing to a separate reviewed change, and includes redacted real provider proof.

## Reproduction Assessment

Yes, source-reproducible. Current main keeps raw assistant/result IDs distinct and only trims transcript tool names, so a transcript split between `functions.exec:0` and `functions exec:0` still follows the broken pairing path; I did not run a live Kimi provider repro.

## Solution Assessment

No. The normalization direction is the right narrow fix, but this merge shape should reuse the existing ambiguity-safe resolver, split the node exec side change, and add real provider proof before merge.
