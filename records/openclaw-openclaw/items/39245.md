## Summary

Keep open: this PR targets a valid source-reproducible Kimi/OpenAI-compatible tool-call pairing bug, but it is not merge-ready because real affected-provider proof is missing and the current patch still has command/tool execution-sensitive blockers.

## What This Changes

The branch normalizes mangled OpenAI-compatible function tool IDs and prefixed tool names in agent transcript repair and tool-call ID sanitization, adds focused tests, and also changes explicit host=node workdir approval routing.

## Best Possible Solution

Land a narrowed normalization fix that reuses the existing ambiguity-safe tool-name resolver, leaves node exec routing to a separate reviewed change, rebases onto current main, and includes redacted real provider proof.

## Reproduction Assessment

Yes, source-reproducible. Current main still pairs sanitizer IDs by raw string and only trims transcript tool names, so canonical `functions.exec:0` and mangled `functions exec:0` remain on the broken path without a live provider run.

## Solution Assessment

No. The normalization direction is right, but the PR should reuse the existing ambiguity-safe resolver, split the node exec routing change, and add real affected-provider proof before merge.
