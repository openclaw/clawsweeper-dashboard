## Summary

Keep open: the Kimi/OpenAI-compatible mangled tool-call problem is real and current main still has the raw-ID pairing gap, but this PR still needs real affected-provider proof and has two merge-blocking correctness/security concerns in replay name normalization and unrelated node execution routing.

## What This Changes

The PR adds `functions ...` tool-call ID/name normalization in agent transcript repair and tool-call ID sanitization, focused tests, and an explicit `host=node` workdir routing change.

## Best Possible Solution

Land a rebased, narrow normalization fix that pairs mangled/canonical Kimi-style IDs, routes prefixed replay names through the ambiguity-safe resolver, and removes or separately proves the node exec routing change.

## Reproduction Assessment

Yes, source-reproducible. Current main pairs sanitizer state by exact raw IDs while the documented Kimi form is `functions.<name>:<index>`, so a provider/proxy that changes the dot to a space will not share the pending key.

## Solution Assessment

No. The normalization direction is useful, but this branch should keep replay name recovery fail-closed through the ambiguity-safe resolver, split or separately prove node routing, and include live affected-provider proof before merge.
