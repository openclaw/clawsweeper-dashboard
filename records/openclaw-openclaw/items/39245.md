## What This Changes

The PR normalizes provider-mangled `functions ...` tool-call IDs and prefixed tool names in agent transcript repair and ID sanitization, adds focused tests, and also changes `host=node` explicit-workdir routing.

## Best Possible Solution

Land a rebased, narrow normalization fix that pairs mangled and canonical Kimi-style IDs, reuses current main's ambiguity-safe tool-name resolver, preserves replay-safe raw IDs, keeps node exec routing separate, and includes redacted live provider proof.

## Reproduction Assessment

Yes, source-reproducible. Current main keys sanitized assistant/result pairing by raw ID and only preserves canonical `functions.<name>:<index>` Kimi IDs, so a `functions exec:0` result does not match a canonical assistant call.

## Solution Assessment

No. The normalization direction is useful, but the current PR should share the ambiguity-safe resolver, remove or split the host=node routing change, rebase over main, and add real affected-provider proof before merge.
