## Summary

## What This Changes

The PR adds transcript-repair and tool-call-ID normalization for OpenAI-compatible provider-mangled tool names/IDs, plus focused tests and a host=node explicit-workdir routing change.

## Best Possible Solution

Land a rebased, narrowed normalization fix that pairs mangled and canonical Kimi-style IDs, uses ambiguity-safe prefixed-name resolution, keeps node exec changes separate, and includes redacted live provider proof.

## Reproduction Assessment

Yes, source-reproducible. Current main keys tool-call ID pairing by raw string and only trims tool names, while the Kimi docs establish `functions.<name>:<index>` IDs; a provider/proxy that changes the dot to a space leaves assistant/result IDs unmatched.

## Solution Assessment

No. The normalization direction is useful, but the current PR should share the ambiguity-safe resolver, remove or split the host=node routing change, and add real affected-provider proof before merge.
