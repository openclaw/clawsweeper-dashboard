## Summary

Keep open. Current main still does not implement the PR's central provider-normalization fix, and the latest PR head appears to address the earlier transcript/tool-call review comments, but it still bundles an unrelated node-host exec approval-path change that needs to be split or explicitly reviewed before merge.

## What This Changes

This PR adds agent transcript and tool-call normalization for provider-prefixed `functions` tool names and IDs, adds focused regression tests, and changes node-host exec handling for explicit workdirs.

## Best Possible Solution

Land or adapt the narrow agent-side normalization fix, including regression coverage for space-prefixed `functions` IDs and prefixed `sessions_spawn` redaction, after splitting the node-host exec workdir approval-path change into a separate explicitly reviewed PR or getting maintainer security approval for that scope.

Yes. ## Solution Assessment

No, not as one PR in its current shape. The normalization approach is the right narrow fix, but the node-host exec approval behavior change is outside the stated scope and should be split or explicitly reviewed.
