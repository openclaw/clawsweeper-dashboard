## Summary

Keep open: current main still lacks the central Kimi-style mangled tool ID/name pairing fix, but this PR is not merge-ready because real behavior proof is missing and the branch still mixes the normalization fix with fail-closed replay and node-exec routing concerns.

## What This Changes

The PR adds provider-prefixed tool-name and tool-call ID normalization in agent transcript repair and ID sanitization, focused tests for mangled Kimi-style IDs, and an explicit `host=node` workdir routing change.

## Best Possible Solution

Land a rebased, narrow normalization fix that pairs mangled/canonical Kimi-style IDs, uses the shared replay-name resolver for prefixed names, keeps tool-result IDs raw where required, and splits or separately proves the node-exec routing change.

## Reproduction Assessment

Yes, source-reproducible. Current main keys sanitizer pending state by exact raw IDs while the documented Kimi form is `functions.<name>:<index>`, so a provider or proxy that changes the separator to a space will not match the same pending entry.

## Solution Assessment

No. The normalization direction is useful, but the branch should be rebased onto current main's shared replay-name resolver, keep node-exec routing separate or separately proved, and include affected-provider proof before merge.
