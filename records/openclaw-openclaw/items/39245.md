## Summary

Keep open. Current main still lacks the Kimi/OpenAI-compatible space-mangled tool ID/name repair, and this remains the canonical open patch, but the current PR head still needs affected-provider real behavior proof and has two blocking scope/security concerns.

## What This Changes

The branch adds functions-prefix tool name and tool-call ID normalization in agent transcript repair and tool-call ID sanitization, focused tests for Kimi-style pairing and sessions_spawn redaction, and a host=node explicit-workdir routing change.

## Best Possible Solution

Land a narrowed normalization fix that pairs Kimi-style mangled and canonical IDs, reuses ambiguity-safe tool-name resolution, leaves node exec routing to a separate proved change, and includes redacted live affected-provider proof.

## Reproduction Assessment

Yes, source-reproducible. Current main uses raw assistant/tool-result IDs for pairing and only trims transcript-repair names, so a provider/proxy that changes `functions.exec:0` to `functions exec:0` leaves the strings unmatched.

## Solution Assessment

No. The normalization direction is useful, but the current branch should use the existing ambiguity-safe resolver semantics, split or separately prove the node exec routing change, and add real affected-provider proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep prefixed replay names fail-closed:** `src/agents/session-transcript-repair.ts:68-72`
  - body: `normalizePrefixedToolName()` strips `functions.` before the transcript-repair allowlist check, so a replayed name like `functions.exec2` can become allowed `exec2` even when current dispatch deliberately treats that shape as ambiguous if both `exec` and `exec2` are available. Reuse the ambiguity-safe resolver semantics or strip only when the suffix resolves unambiguously.
  - confidence: 0.86
- **[P2] Split node workdir routing out of this fix:** `src/agents/bash-tools.exec-host-node-phases.ts:60`
  - body: Adding `workdirExplicit` changes full/off `host=node` commands with explicit cwd from the direct path to the prepare path, but this PR is scoped and proved only for provider-mangled tool IDs/names. Remove this from the normalization PR or move it to a separate node exec change with its own behavior proof.
  - confidence: 0.78
