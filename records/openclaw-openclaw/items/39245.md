## Summary

Keep open. The agent-side normalization is still useful, but this PR is not ready as-is because it bundles an unrelated node-host exec approval-path change and lacks the required changelog entry for a user-visible fix.

## What This Changes

This PR normalizes provider-prefixed or mangled `functions` tool names and IDs in agent transcript repair/tool-call ID sanitization, adds focused regression tests, and also changes node-host exec handling for explicit workdirs.

## Best Possible Solution

Land the narrow transcript/tool-call normalization with regression coverage after removing or splitting the node-host exec approval-path change and adding the required changelog entry; keep any node exec work in a separate explicitly reviewed security/exec PR.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to run the sanitizer/repair paths on `functions.exec:0` versus `functions exec:0`, and on a prefixed `functions.sessions_spawn` call; current main still uses raw ID keys and raw trimmed names in those lower-level helpers.

## Solution Assessment

No, not as one PR in its current shape. The normalization approach is the right narrow fix, but the exec approval-path change is outside the stated scope and the changelog requirement is still unmet.
