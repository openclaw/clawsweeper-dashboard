## Summary

Keep this PR open. Current main still lacks the core functions-space/functions-dot tool-call ID equivalence and persisted transcript prefix stripping/redaction behavior, while the PR's latest head appears to address the earlier ID-normalization review comments; the remaining blocker is maintainer/security review for an unrelated node-host exec approval-path change bundled into the same branch.

## What This Changes

This PR adds agent transcript and tool-call normalization for provider-prefixed `functions` tool names and IDs, adds regression tests for those paths, and changes node-host exec handling so explicit workdirs do not skip the approval prepare phase.

## Best Possible Solution

Land or adapt the narrow agent-side normalization fix that treats only leading `functions ` and `functions.` IDs as equivalent where sanitizer pairing needs it, preserves raw IDs for pending-state and replay-safe thinking paths, and strips `functions.`/`functions ` prefixes before persisted transcript allowlist/redaction and tool-result name normalization. Split or explicitly document and review the node-host exec workdir approval change separately.

## Reproduction Assessment

Yes. A high-confidence reproduction is to run the sanitizer/repair path on an assistant tool call with ID `functions.exec:0` and a tool result with `functions exec:0`; current main's exact raw-ID maps cannot pair those IDs, and prefixed `functions.sessions_spawn` names miss the persisted redaction path.

## Solution Assessment

No, not as a single PR in its current shape. The agent normalization approach is the right narrow fix, but the node-host exec approval change is outside the stated scope and should be split or explicitly reviewed before merge.
