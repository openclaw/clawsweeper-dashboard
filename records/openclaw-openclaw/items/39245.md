## Summary

Keep open: current main still lacks the requested transcript/tool-call ID normalization, but this PR is not merge-ready because it lacks real provider proof and still bundles command-exec routing behavior outside its stated scope.

## What This Changes

This PR normalizes provider-mangled `functions` tool call IDs and tool names in agent transcript repair/sanitization, adds focused tests, and changes explicit `host=node` workdir approval routing.

## Best Possible Solution

Land a narrowed transcript/tool-call ID normalization that reuses the existing unambiguous tool-name policy, keeps node exec routing separate, and includes redacted real provider proof.

## Reproduction Assessment

Yes, source-reproducible. A transcript with assistant/result IDs split between `functions.exec:0` and `functions exec:0` follows the current raw-ID pairing path and current name trimming does not strip `functions` prefixes; I did not run a live Kimi provider repro.

## Solution Assessment

No. The normalization direction is the right narrow fix, but this PR is not the best merge shape until the node exec side change is split, ambiguous prefixed names stay fail-closed, and after-fix provider evidence is added.
