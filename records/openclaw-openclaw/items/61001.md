## Summary

Keep open. Current main still lacks the requested per-file bootstrap selectors by normalized session type or file frontmatter; the shipped behavior is a mix of coarse config knobs, fixed subagent/cron filtering, heartbeat-specific modes, and a global extra-files hook. The related open pull requests are useful adjacent work, but they do not settle this issue's public API shape.

## What This Changes

The issue requests per-bootstrap-file session-type selectors, or equivalent frontmatter, so files like HEARTBEAT.md, TOOLS.md, USER.md, and MEMORY.md are only injected for relevant session kinds.

## Best Possible Solution

Design one additive core bootstrap-selection contract that preserves blanket loading by default, defines stable run/session-kind names, reconciles tier and exact-session extra-file work, and documents/tests direct, group, heartbeat, cron/isolated, and subagent behavior.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection: current main lacks per-file session selectors or bootstrap-file frontmatter, while only adjacent coarse filters and config knobs exist.

## Solution Assessment

Unclear until maintainers choose the public API shape. The requested direction is plausible, but the narrow maintainable path should align agents.defaults config, run/session-kind names, heartbeat behavior, and the related tier/exact-session PRs before implementation.
