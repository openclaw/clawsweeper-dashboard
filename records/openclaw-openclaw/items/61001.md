## Summary

Keep this issue open. Current main still lacks the requested per-file bootstrap selectors by normalized session type or file frontmatter; the existing controls are coarse defaults, hard-coded subagent/cron filtering, heartbeat light context, and a global bootstrap-extra-files hook. The related open PR #74735 is useful adjacent work, but its own scope excludes session-type selectors and a new `agents.defaults` bootstrap policy.

## What This Changes

The issue requests additive per-bootstrap-file loading controls keyed by session type so direct, group, heartbeat or isolated, and subagent runs can receive different boot context while preserving blanket loading by default.

## Best Possible Solution

Design an additive core bootstrap-selection contract that keeps blanket loading as the default, defines stable run/session kind names, applies selection before context files are built, and documents/tests direct, group, heartbeat, cron/isolated, and subagent cases.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection: current schema, loader, tests, and docs lack the requested per-file session-type selectors.

## Solution Assessment

Unclear until maintainers choose the public API shape. The requested direction is plausible, but the narrow maintainable path should align the core config contract with the existing hook surface and adjacent exact-session work in PR #74735.
