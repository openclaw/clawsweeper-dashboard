## Summary

Keep open. Current main still lacks user-configurable per-file bootstrap selectors by normalized session type or bootstrap-file frontmatter; the shipped controls are coarser, and the related exact-session extra-files PR explicitly does not cover this API.

## What This Changes

Request adds an additive policy for loading workspace bootstrap files only for selected session types such as direct, group, heartbeat, isolated, and subagent sessions.

## Best Possible Solution

Define an additive core bootstrap-selection contract that preserves blanket loading by default, names stable run/session kinds, applies selection before context files are built, and documents/tests direct, group, heartbeat, cron/isolated, and subagent behavior.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection provide high-confidence verification that current main lacks per-file session-type selectors or bootstrap frontmatter selection.

## Solution Assessment

Unclear until maintainers choose the public API shape. The requested direction is plausible, but it should align with the existing bootstrap hook surface, run-kind filters, cache behavior, and adjacent exact-session work before implementation.
