## Summary

Keep this issue open. Current main still has only coarse bootstrap controls, hard-coded subagent/cron filtering, and heartbeat/cron lightweight modes; it does not provide per-file selectors by normalized session type or bootstrap-file frontmatter. The open related PR #74735 is adjacent exact-session extra-file work, but its described scope explicitly excludes session-type selectors and a new `agents.defaults` bootstrap policy.

## What This Changes

The issue requests additive per-bootstrap-file loading controls keyed by session type so direct, group, heartbeat, isolated, and subagent runs can receive different boot context while preserving current blanket loading by default.

## Best Possible Solution

Design an additive core bootstrap-selection contract that preserves blanket loading by default, defines stable session/run-kind names, applies selection before context files are built, keeps hook/plugin behavior compatible, and documents/tests the direct, group, heartbeat, cron/isolated, and subagent cases.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection: current schema, loader, tests, and docs lack the requested per-file session-type selectors.

## Solution Assessment

No current implementation is the best full solution yet. The narrow maintainable path is a maintainer-approved core config/API design; exact session-key extra files in PR #74735 may be useful adjacent work but do not solve this issue by themselves.
