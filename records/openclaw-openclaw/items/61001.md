## Summary

Keep open. Current main and the latest shipped release still do not provide per-file bootstrap selectors by normalized session type or bootstrap-file frontmatter; existing behavior is a fixed workspace file list, hard-coded subagent/cron filtering, coarse context modes, and a global extra-files hook. The related open work in https://github.com/openclaw/openclaw/pull/22439 and https://github.com/openclaw/openclaw/pull/74735 is relevant, but neither has landed or settles this issue's public API shape.

## What This Changes

Request an additive config or frontmatter contract that loads each workspace bootstrap file only for selected session types.

## Best Possible Solution

Design one additive bootstrap-selection contract that preserves blanket loading by default, defines stable session-kind names, reconciles tier, exact-session, heartbeat, and extra-file behavior, and documents/tests direct, group, heartbeat, cron, and subagent runs.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and release inspection: current main and v2026.5.18 lack per-file session selectors or bootstrap-file frontmatter while only adjacent coarse controls exist.

## Solution Assessment

Unclear until maintainers choose the public API shape. The requested direction is plausible, but the maintainable path should align agents.defaults config, stable session-kind names, heartbeat behavior, exact-session extra files, and the tiered-loading proposal before implementation.
