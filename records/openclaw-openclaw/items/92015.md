## Summary

Keep open: current main and the latest release still resolve an implicit workspace for non-default agents and run the normal workspace seeding path, including `git init`, without a parent-repository guard or ACP/runtime-managed exception.

## What This Changes

Request to skip implicit workspace scaffolding and nested `git init` for ACP/runtime-managed agents that do not set an explicit workspace.

## Best Possible Solution

Preserve explicit workspace behavior, but treat ACP agents without an explicit workspace as runtime-managed for scaffold/git-init purposes and make workspace `git init` parent-repo-safe with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an ACP-configured agent with no explicit workspace still gets a default workspace resolved and passed to `ensureAgentWorkspace`, which seeds bootstrap files and runs `git init` for a brand-new directory; I did not run the filesystem scenario in this read-only review.

## Solution Assessment

Yes, with a narrow boundary. The maintainable fix is to skip implicit scaffold/git-init only for runtime-managed ACP agents without explicit workspaces, preserve explicit workspace contracts, and add a parent-git guard in the seeder rather than adding a new config knob first.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/92015. Current main resolves a default workspace for non-default agents without `workspace`, then runtime paths call `ensureAgentWorkspace`, which seeds bootstrap files and runs `git init`; ACP agents with `agents.list[].runtime.type="acp"` and no explicit workspace should be treated as runtime-managed for this default workspace scaffold path. Preserve behavior when an agent has an explicit `workspace`, preserve embedded/default-agent workspace behavior, and do not change ACP binding cwd precedence except as required by this invariant. Also harden `ensureGitRepo` so brand-new workspace creation does not run `git init` inside an existing parent Git repository unless a shipped contract explicitly requires it. Add focused tests for ACP no-explicit-workspace behavior, explicit-workspace preservation, and parent-repo `git init` skipping. Do not add a new config/env option unless maintainers explicitly approve it, and do not touch CHANGELOG.md.
