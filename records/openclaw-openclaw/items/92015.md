## Summary

Keep open. Current main still routes non-default agents with `agents.defaults.workspace` into a derived workspace, then both agent-command and auto-reply startup call the bootstrap seeder; the seeder writes templates and runs `git init` without checking parent repositories, while ACP runtime bindings already have their own `runtime.acp.cwd` contract.

## What This Changes

The issue asks OpenClaw to skip default workspace scaffold/git initialization for ACP runtime-managed agents without explicit workspaces and to avoid nested `git init` inside parent repositories.

## Best Possible Solution

Implement a narrow core fix that treats ACP agents with `runtime.type: "acp"` and no explicit `workspace` as runtime-managed for bootstrap purposes, keeps explicit workspaces honored, and skips `git init` when an ancestor repository is already present.

## Reproduction Assessment

Yes, at source level: current main derives the reported default workspace, then calls `ensureAgentWorkspace`, which seeds files and runs `git init` for brand-new directories. I did not execute the filesystem reproduction because this review was read-only.

## Solution Assessment

Yes, if scoped narrowly: the best fix is to skip full bootstrap for ACP runtime-managed agents without explicit workspaces and add an ancestor-repository guard before `git init`. A broad rewrite of all non-default workspace defaults should stay separate from this issue.
