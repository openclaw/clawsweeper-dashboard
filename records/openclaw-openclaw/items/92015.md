## Summary

Keep open. Current main and the latest release still resolve an implicit per-agent workspace for ACP/runtime-managed agents and send it through the normal bootstrap and git-init path; the linked open PR is the active fix candidate but has not merged.

## What This Changes

The issue requests that runtime-managed ACP agents without an explicit workspace skip default workspace scaffolding and avoid nested git initialization.

## Best Possible Solution

Land a narrow core fix, likely through https://github.com/openclaw/openclaw/pull/92939 or a replacement, that skips scaffolding only for implicit runtime-managed workspaces, preserves explicit workspaces, and covers reply, agent-command, and cron setup paths.

Yes, the requested direction is the right layer, but the active PR must preserve explicit runtime-agent workspaces and cover sibling workspace setup paths before this issue can close.
