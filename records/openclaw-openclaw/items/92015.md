## Summary

Keep open. Current main and the latest release still resolve and ensure an implicit workspace for the reported ACP agent shape, and the linked repair PR remains open with review blockers rather than merged as the canonical fix.

## What This Changes

The issue asks OpenClaw to avoid creating, scaffolding, and git-initializing implicit default workspaces for ACP runtime-managed agents whose ACP binding already supplies the effective cwd.

## Best Possible Solution

Land a narrow core fix that skips scaffolding only for implicit runtime-managed workspaces, preserves explicit workspaces, and prevents unintended nested `git init` across ACP-reachable workspace setup paths.

## Reproduction Assessment

Yes. Source inspection shows the resolver computes an implicit workspace, the reply path ensures it, and workspace setup writes bootstrap files plus runs `git init` for brand-new workspaces.

## Solution Assessment

Yes, the requested direction is the right layer, but the implementation should distinguish implicit runtime-managed workspaces from explicit user workspaces and cover sibling entry points.
