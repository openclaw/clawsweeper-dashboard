## Summary

Keep open. Current main still resolves a fallback agent workspace for ACP-configured agents and runs the normal workspace seeder before ACP handling, so the reported bootstrap-file and nested-git side effects remain plausible and source-traceable.

## What This Changes

The issue requests skipping default workspace scaffolding and nested git initialization for runtime-managed ACP agents that omit an explicit workspace.

## Best Possible Solution

Preserve explicit per-agent workspace bootstrapping, but make runtime-managed ACP agents without an explicit workspace skip default workspace scaffolding and nested git initialization; add a focused regression for that exact config shape.

## Reproduction Assessment

Yes, at source level: current main resolves a fallback workspace and calls the seeder before ACP handling, while the seeder writes bootstrap files and initializes git for brand-new workspaces. I did not run the filesystem reproduction in this read-only sweep.

## Solution Assessment

Yes, the best fix is a narrow runtime-managed skip that keeps explicit workspace behavior intact. A broad derived-workspace rename or generic seeder rewrite would create avoidable compatibility risk.
