## Summary

Keep open. Current `main` still resolves and ensures an agent workspace before ACP dispatch, and the shared seeder can write bootstrap files plus run `git init` in a brand-new target without checking whether the target sits under an existing repository.

## What This Changes

The issue requests that ACP/runtime-managed agents without explicit workspaces avoid default workspace scaffolding and nested git initialization.

## Best Possible Solution

Keep this issue open and implement a narrow core hardening fix that skips implicit workspace bootstrap for ACP runtime-managed agents with no explicit workspace, while preventing nested `git init` centrally in the workspace seeder.

## Reproduction Assessment

Yes, by source-level reproduction rather than a mutating filesystem run: `agent-command` resolves and ensures an agent workspace before ACP dispatch, and the seeder can create bootstrap files plus `.git` in a brand-new target without checking parent repositories. I did not run a live repro because this review must keep the checkout read-only.

## Solution Assessment

Yes. The narrow maintainable solution is to guard nested git initialization in `src/agents/workspace.ts` and skip implicit workspace bootstrap for `agents.list[].runtime.type="acp"` agents only when no explicit workspace was configured, without changing normal embedded-agent bootstrap.
