## Summary

Keep open: current main and the latest release still create and seed the computed workspace before running git init, and the open linked implementation PR is not merged yet.

## What This Changes

Request skipping default workspace scaffolding and git init for runtime-managed ACP agents without an explicit workspace, plus safer nested git init behavior.

## Best Possible Solution

Land a focused fix through https://github.com/openclaw/openclaw/pull/92939 or an equivalent branch that skips scaffolding only for runtime-managed agents without an explicit workspace, preserves explicit workspace behavior, and adds caller-level regression coverage.

## Reproduction Assessment

Yes by source inspection: current main resolves a default workspace for the ACP-configured agent and then calls ensureAgentWorkspace, which creates the directory, seeds bootstrap files, and can run git init. I did not run a live gateway reproduction in this read-only review.

## Solution Assessment

Yes on direction: the narrow fix is to skip default scaffolding for runtime-managed agents only when no explicit workspace is configured. The current linked PR needs its explicit-workspace compatibility repair before this issue should close.
