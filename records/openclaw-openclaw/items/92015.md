## Summary

Keep open. Current main and v2026.6.8 still resolve an implicit workspace for ACP/runtime-managed agents and route that workspace through bootstrap file creation plus `git init`; the linked implementation PR is open but not merged and has compatibility blockers to resolve first.

## What This Changes

The issue asks OpenClaw to stop auto-scaffolding and `git init` for implicit runtime-managed ACP agent workspaces, while preserving explicit workspace behavior and avoiding nested git repositories.

## Best Possible Solution

Land a narrow core fix that skips scaffolding only for implicit runtime-managed ACP workspaces, preserves explicit agent workspaces, and prevents or explicitly gates nested `git init` under parent repositories.

## Reproduction Assessment

Yes, by source inspection: current main resolves an implicit workspace for agents without `workspace`, then calls `ensureAgentWorkspace`, which writes bootstrap files and runs `git init` for brand-new workspaces. I did not run the full ACP startup scenario in this read-only review.

## Solution Assessment

Yes, the requested direction is the right layer, but the implementation should distinguish implicit runtime-managed workspaces from explicit user workspaces and cover every ACP-reachable workspace setup path.
