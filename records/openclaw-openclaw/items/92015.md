## Summary

Keep open. Current main and v2026.6.9 still resolve an implicit per-agent workspace for ACP/runtime-managed agents and pass it through the normal bootstrap and git-init path; https://github.com/openclaw/openclaw/pull/92939 is the active open fix candidate but has not merged.

## What This Changes

The issue requests that OpenClaw skip default workspace scaffolding and git initialization for ACP/runtime-managed agents that do not configure an explicit workspace.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/92939 with a narrow core fix that skips scaffolding only for implicit runtime-managed workspaces, preserves explicit workspaces, and covers reply plus sibling workspace setup paths.

## Reproduction Assessment

Yes, at source level. A non-default ACP agent without an explicit workspace still resolves to agents.defaults.workspace/<agentId>, then current reply and agent-command paths ensure that workspace through the bootstrap/git-init helper.

## Solution Assessment

Yes, the requested direction is the right layer, but the fix should be limited to implicit runtime-managed workspaces and must preserve explicit workspace setup plus sibling callers.
