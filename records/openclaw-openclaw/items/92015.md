## Summary

Keep open: current main still resolves a default agent workspace and runs the normal workspace bootstrap path for ACP/runtime-managed agents without an explicit workspace. The active implementation path is https://github.com/openclaw/openclaw/pull/92939, but it is still open and has a compatibility review finding, so this issue remains necessary.

## What This Changes

Request skipping default workspace scaffolding and nested `git init` for runtime-managed ACP agents that do not set an explicit workspace.

## Best Possible Solution

Land a narrow fix, likely by repairing the linked PR, that skips default workspace scaffold and `git init` only for non-embedded runtime agents without an explicit workspace, with reply-path regression coverage for ACP without workspace, ACP with explicit workspace, and embedded/default behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an ACP agent without `workspace` falls through `resolveAgentWorkspaceDir`, then the reply path calls `ensureAgentWorkspace`, which creates bootstrap files and runs `git init` for a brand-new workspace.

## Solution Assessment

Yes, keeping this issue open while the linked PR is repaired is the best current path. The maintainable fix is a narrow runtime/default-workspace predicate, not a broad new config option.
