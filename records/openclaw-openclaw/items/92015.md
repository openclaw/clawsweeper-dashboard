## Summary

Keep this open: current main still has a source-reproducible path where an ACP/runtime-managed configured agent without an explicit workspace resolves a default workspace and the shared bootstrap helper can seed files plus run `git init` there.

## What This Changes

The issue requests skipping default workspace scaffolding and nested git initialization for ACP/runtime-managed agents that omit an explicit workspace.

## Best Possible Solution

Add a narrow runtime-managed-agent workspace policy that skips default workspace materialization for ACP agents without explicit workspace while preserving normal embedded-agent bootstrap behavior and preventing nested git initialization.
