## Summary

Keep open. Current main and v2026.6.8 still resolve and ensure an implicit workspace for the reported ACP agent shape, including bootstrap-file writes and `git init`; the open linked PR is the active candidate but has not merged and still has compatibility/sibling-path blockers.

## What This Changes

The issue requests skipping default workspace scaffolding and nested `git init` for runtime-managed ACP agents that omit an explicit workspace.

## Best Possible Solution

Land a narrow core fix that skips scaffolding only for implicit runtime-managed workspaces, preserves explicit workspaces, and prevents unintended nested `git init` across ACP-reachable workspace setup paths.

## Reproduction Assessment

Yes. Source inspection shows a non-default runtime agent without `workspace` still resolves an implicit default workspace, the reply and command paths ensure that workspace, and the workspace seeder writes bootstrap files plus runs `git init` for brand-new workspaces.

## Solution Assessment

Yes, the requested direction is the right layer. The best fix should distinguish implicit runtime-managed workspaces from explicit user workspaces and centralize or prove the decision across ACP-reachable workspace setup callers.
