## Summary

Keep open: current main and latest v2026.6.6 still resolve and bootstrap an implicit workspace before ACP/runtime-managed turns, and the Git initializer only checks the workspace itself, not parent repositories. The related open pull request at https://github.com/openclaw/openclaw/pull/72807 is only a partial opt-out path and is not a canonical fix for this default-safe behavior.

## What This Changes

The issue requests preventing implicit workspace scaffolding and nested Git initialization for ACP/runtime-managed agents that do not explicitly configure a workspace.

## Best Possible Solution

Preserve explicit OpenClaw workspaces, make ACP/runtime-managed agents with no explicit workspace skip OpenClaw scaffold/bootstrap, and make automatic Git initialization ancestor-aware by default.

## Reproduction Assessment

Yes, from source inspection: an ACP agent without `workspace` still flows through `resolveAgentWorkspaceDir` and `ensureAgentWorkspace`, and that seeder can run `git init` in a brand-new fallback workspace. I did not run the full startup flow because this review had to keep the checkout read-only.

## Solution Assessment

Yes, the best fix is a bounded core hardening change rather than a new broad config knob: skip scaffold for runtime-managed ACP agents without explicit workspaces and make `git init` refuse nested repositories by default. The existing open pull request at https://github.com/openclaw/openclaw/pull/72807 is useful context but only covers an opt-out for one surface.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/92015. Current main resolves a default workspace for `agents.list[]` entries with `runtime.type: "acp"` and no explicit `workspace`, then reply/bootstrap paths call `ensureAgentWorkspace`, which writes bootstrap files and can run `git init` under a parent checkout. Implement a narrow core fix that: (1) skips OpenClaw-managed workspace scaffold/bootstrap for ACP/runtime-managed agents that have no explicit workspace and use ACP `cwd` or binding `cwd`, while preserving explicit workspace behavior and embedded/default agents; (2) prevents automatic `git init` when the workspace is inside an existing parent Git repository, without adding a new config/env knob unless maintainers explicitly request it; (3) adds focused regression coverage for ACP no-workspace/no-scaffold and parent-Git no nested-init behavior; (4) reviews the related open pull request https://github.com/openclaw/openclaw/pull/72807 so the fix does not introduce a competing config-surface path. Do not change ACP runtime session cwd semantics, general workspace template content, or `CHANGELOG.md`.
