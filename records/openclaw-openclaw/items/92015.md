## Summary

Keep open: current main still resolves a default per-agent workspace and the normal reply path still ensures/scaffolds it before runtime dispatch, while the workspace helper still runs `git init` for brand-new workspaces without checking parent repositories.

## What This Changes

Request to harden ACP/runtime-managed agents so omitted workspaces are not auto-scaffolded, and to avoid nested `git init` under an existing repository.

## Best Possible Solution

Define a central runtime-managed workspace policy, keep explicit workspaces working, and make `ensureGitRepo` skip nested repositories unless maintainers intentionally preserve the current default.

## Reproduction Assessment

No high-confidence live reproduction was run in this read-only review. Source inspection shows the reported path is plausible: default workspace resolution feeds `ensureAgentWorkspace`, which seeds files and runs `git init` for a brand-new directory.

## Solution Assessment

Unclear until maintainers confirm the default policy. The requested direction is plausible, but the best fix should centralize runtime-managed workspace detection and nested-git safety rather than adding one-off ACP checks at individual call sites.
