- `impact:other`: The concrete impact is unwanted workspace filesystem mutation and nested git repository creation, which does not fit the narrower owned impact labels.

## Summary

Keep open: current main still derives a default workspace for ACP-configured agents with no explicit workspace, calls the normal bootstrap seeder before ACP turns, and only checks `.git` in the target directory before running `git init`.

## What This Changes

The issue requests hardening so runtime-managed ACP agents without explicit workspaces do not auto-scaffold default workspaces or create nested git repositories.

## Best Possible Solution

Implement a bounded core fix that skips bootstrap/git initialization for ACP runtime agents without explicit workspaces and prevents auto-`git init` under an existing parent repository.

## Reproduction Assessment

Yes at source level, but I did not run a live repro. The current path is `resolveAgentWorkspaceDir` -> `ensureAgentWorkspace` -> `ensureGitRepo`, with ACP turns still entering that path.

## Solution Assessment

Yes, a targeted core fix is the best direction. The existing global `agents.defaults.skipBootstrap` workaround is too broad and does not solve parent-repo `git init` safety.
