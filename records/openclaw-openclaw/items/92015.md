## Summary

Keep open. Current main and the latest release still resolve an implicit workspace for the agent and route normal replies through workspace setup that creates bootstrap files and runs `git init`; the linked repair PR is open but has not merged and was reviewed as needing an explicit-workspace compatibility fix.

## What This Changes

The issue requests skipping default workspace scaffolding and `git init` for runtime-managed ACP agents without an explicit workspace, plus hardening against nested git initialization.

## Best Possible Solution

Land a narrow core fix that skips scaffolding only for implicit runtime-managed workspaces, preserves explicit workspaces, and prevents nested `git init` inside parent repositories or makes that behavior explicitly intentional.

## Reproduction Assessment

Yes, by source inspection: current main resolves an implicit workspace, always calls `ensureAgentWorkspace` on the reply path, and that helper creates bootstrap files and runs `git init` for brand-new workspaces. I did not execute the ACP startup scenario in this read-only sweep.

## Solution Assessment

Yes, the requested direction is the right layer: a core runtime guard should distinguish implicit runtime-managed workspaces from explicit user workspaces. The linked PR is close but should preserve explicit workspace initialization and add focused coverage before landing.
