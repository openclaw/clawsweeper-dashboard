## Summary

Keep open. The metadata-aware heartbeat filter is useful, but the runtime-context fallback still drops configured custom heartbeat prompts, and the external PR has no real behavior proof before merge.

## What This Changes

The branch adds metadata-aware heartbeat pair filtering and a heartbeat-specific runtime-context fallback when the compact heartbeat transcript marker is not present in the effective prompt.

## Best Possible Solution

Keep the stable heartbeat transcript marker and metadata-aware filtering, but make the runtime-context fallback identify heartbeat turns without relying on the default prompt text, with coverage for custom heartbeat prompts and real heartbeat proof.

## Reproduction Assessment

Yes, at source level: with transcriptPrompt `[OpenClaw heartbeat poll]` and effectivePrompt set to a custom `agents.*.heartbeat.prompt` body, the PR-head detector leaves `isHeartbeatTurn` false and returns only the marker. I did not run a live heartbeat in this read-only review.

## Solution Assessment

No. The metadata-aware filter is narrow and useful, but the runtime fallback should use an explicit heartbeat signal or marker-based contract instead of depending on the default HEARTBEAT.md prompt sentence.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91
