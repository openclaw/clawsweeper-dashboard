## Summary

Keep open. The PR addresses a real Slack capabilities bug that current main still has, but the current patch also treats Slack `acceptedScopes` as granted token scopes, which would make the diagnostic output misleading if merged as-is.

## What This Changes

The PR adds Slack `auth.test` as a third scope-discovery fallback, reads Slack response metadata for scopes, adds focused Slack scope tests, and adds a changelog entry for modern bot-token capabilities discovery.

## Best Possible Solution

Land the Slack plugin fix after a narrow correction: keep the legacy methods first, add `auth.test` as the modern-token fallback, return granted scopes only from `response_metadata.scopes`, update the focused tests to reject accepted-scopes-only grant discovery, and keep the changelog wording limited to granted scope reporting.
