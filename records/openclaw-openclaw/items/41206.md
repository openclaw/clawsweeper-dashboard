## Summary

Keep open: current main still documents and uses the flat `jobs.json` plus `jobs-state.json` cron store, while this PR is a partial new backend with correctness and filename-safety blockers before it can merge.

## What This Changes

This PR adds exported directory-store helpers and tests for one JSON file per cron job under `cron/jobs/` while retaining `jobs.json` compatibility.

## Best Possible Solution

Rework the feature as a first-class cron storage backend with safe id-keyed filenames, explicit load/write precedence, state-sidecar preservation, CLI/doctor/docs support, and regression tests.

## Reproduction Assessment

Yes. Static reproduction against the PR diff is high-confidence: a job with id `brew-1` and name `Daily Brew` saves as `daily-brew.json`, while deletion by id compares `daily-brew` to `brew-1` and leaves the file behind.

## Solution Assessment

No. The directory-store direction is plausible, but this patch is not the best implementation because it is not wired into live cron persistence and conflicts with the current split config/state contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Wire the directory store into live persistence:** `src/cron/store.ts:196`
  - body: `loadCronStoreDir` is exported and tested, but current main's scheduler load and persist paths still call the flat `loadCronStore`/`saveCronStore` backend. Creating `cron/jobs/` would not affect Gateway, CLI, doctor, or documented behavior.
  - confidence: 0.97
- **[P1] Key job files by immutable ids:** `src/cron/store.ts:246-247`
  - body: `saveCronJobFile` derives the filename from mutable, non-unique `job.name`. Duplicate names can overwrite each other, and renames can leave multiple files for the same stable job id.
  - confidence: 0.96
- **[P1] Delete job files by persisted ids:** `src/cron/store.ts:271-272`
  - body: `removeCronJobFile` compares the requested id with the filename-derived id, but files written by this patch are name-derived. Normal jobs whose stable id differs from the display name will not be removed.
  - confidence: 0.96
- **[P2] Keep fallback filenames inside jobs:** `src/cron/store.ts:151`
  - body: When slugification produces an empty string, the helper returns raw input. Values containing path separators can escape `cron/jobs/` unless the fallback is safe and the resolved path is checked.
  - confidence: 0.9
- **[P2] Preserve the runtime state sidecar:** `src/cron/store.ts:248`
  - body: Current main keeps runtime fields in `jobs-state.json`, but this writer serializes the full `CronJob` into each per-job config file. If wired in, runtime execution state becomes user-editable config again.
  - confidence: 0.86
- **[P2] Define the symlink policy for job files:** `src/cron/store.ts:211`
  - body: The related GitOps use case depends on symlinkable job definitions, but `entry.isFile()` skips symlinks before parsing. The backend should either support symlinks with containment checks or reject them with a documented diagnostic.
  - confidence: 0.82
