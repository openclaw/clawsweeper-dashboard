## Summary

Keep open: current main and v2026.6.6 cover the older exec-event heartbeat bypass and part of Codex usage-limit reset handling, but the reported combined storm is not fully closed because failed in-flight Codex turns refresh the usage-limit message without marking the auth profile blocked until reset, and broader automated quota/circuit-break policy remains open.

## What This Changes

The issue requests that heartbeat event/retry wakes honor configured per-agent cadence and that hard Codex/provider quota errors stop automated retry storms until the provider reset window.

## Best Possible Solution

Keep this open for a maintainer-scoped fix that preserves the shipped heartbeat wake-intent matrix, extends Codex failed-turn usage-limit handling to block the active auth profile until the reset window, and explicitly decides how automated heartbeat/cron quota storms coordinate with provider cooldown and recovery probing.
