## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep policy check read-only:** `extensions/policy/src/cli.ts:114`
  - body: `policy check` and `policy watch` are documented as read-only, but this path calls `readConfigFileSnapshot()` with the observing default. In this PR snapshot reads write config health state unless callers pass `{ observe: false }`, so audit and CI runs can still mutate local state.
  - confidence: 0.91
- **[P2] Preserve MCP server ID case:** `extensions/policy/src/doctor/register.ts:1103`
  - body: This helper lowercases all policy list entries, but it also backs `mcp.servers` allow/deny matching. OpenClaw preserves configured server keys such as `ProdDocs`, so `deny: ["ProdDocs"]` becomes `proddocs` and no longer matches the configured server.
  - confidence: 0.88
- **[P2] Remove or implement the tool metadata config keys:** `extensions/policy/openclaw.plugin.json:23-29`
  - body: The manifest advertises `requireRisk` and `requireSensitivity` under plugin config, and the plugin reference shows them, but policy evaluation and runtime enforcement only read `policy.jsonc` `tools.requireMetadata`. Operators can enable these config keys and get no tool metadata checks.
  - confidence: 0.87
