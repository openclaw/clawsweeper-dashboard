## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Stop emitting MCP server URLs in policy evidence:** `extensions/policy/src/policy-state.ts:131`
  - body: `mcp.servers.*.url` is treated as sensitive elsewhere because it can contain userinfo or query tokens, but this copies the raw URL into `policy check --json` evidence and the attestable audit payload. That would leak credentials into logs/artifacts; emit only non-secret fields or a redacted URL/host.
  - confidence: 0.92
- **[P1] Report invalid policy JSONC as a failed check:** `extensions/policy/src/doctor/register.ts:238`
  - body: When `policy.jsonc` has parse errors, this branch returns the existing findings array unchanged. For an enabled policy extension that means `policy check`/`doctor --lint` can report clean instead of failing the conformance gate, so convert diagnostics into an error finding.
  - confidence: 0.9
- **[P2] Include agents.defaults.models keys in model evidence:** `extensions/policy/src/policy-state.ts:336`
  - body: The scanner only records string values under `model` or `*Model` keys, but `agents.defaults.models` is a documented allowlist keyed by `provider/model`. A policy denying or allowlisting providers can therefore miss configured allowlist entries unless the provider is also selected as a primary or configured under `models.providers`.
  - confidence: 0.86
