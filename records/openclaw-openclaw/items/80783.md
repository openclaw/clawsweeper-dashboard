## Summary

This PR remains useful but is not merge-ready: the linked proof covers the main model/MCP/network policy path, but MCP allow/deny ids are still lowercased before matching case-preserving `mcp.servers` keys, so mixed-case servers can evade or false-trigger policy findings.

## What This Changes

This PR extends the bundled policy plugin and CLI with model-provider, MCP-server, and private-network conformance evidence, doctor checks, docs, and tests.

## Best Possible Solution

Preserve current MCP server key semantics in policy matching, add mixed-case deny/allow regression tests, then route the policy shape and evidence boundaries to maintainer review.

## Reproduction Assessment

Yes, from source inspection: `readPolicyStringArray` lowercases policy strings while `scanPolicyMcpServers` preserves `mcp.servers` keys and the tests only cover lowercase ids; a mixed-case server name demonstrates the mismatch.

## Solution Assessment

No: the conformance-layer approach is plausible, but this implementation should not merge until MCP ids either preserve authored casing or both policy and config ids use a documented canonicalization contract.
