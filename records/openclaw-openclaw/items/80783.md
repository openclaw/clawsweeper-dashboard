- **[P1] Redact MCP server URLs from policy evidence:** `extensions/policy/src/policy-state.ts:145-147`
  - body: `mcp.servers.*.url` is redacted elsewhere because it can contain userinfo or query tokens, but this copies the raw URL into `policy check --json` evidence and the attestation payload.   - confidence: 0.92
- **[P1] Emit a finding for invalid policy JSONC:** `extensions/policy/src/doctor/register.ts:260-262`
  - body: When `policy.jsonc` has parser errors, this returns the current findings unchanged. For an enabled policy extension, `policy check` and `doctor --lint` can therefore report clean even though the policy artifact cannot be read; convert diagnostics into an error finding.
  - confidence: 0.9
- **[P2] Scan model allowlist keys as model evidence:** `extensions/policy/src/policy-state.ts:383-385`
  - body: The scanner only records string values under `model` or `*Model` keys, but `agents.defaults.models` is a documented allowlist keyed by provider/model refs. Provider policy can miss entries such as `openrouter/*` unless the provider is also configured elsewhere.
  - confidence: 0.86
