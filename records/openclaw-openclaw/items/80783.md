## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep policy check read-only:** `extensions/policy/src/cli.ts:114`
  - body: `policy check` is documented as read-only, but this path calls `readConfigFileSnapshot()` without `{ observe: false }`. The new config snapshot helper defaults to observing snapshots, which writes config health state; use the same non-observing read that `doctor --lint` uses before this becomes an audit or CI command.
  - confidence: 0.9
- **[P2] Preserve MCP server IDs in policy list matching:** `extensions/policy/src/doctor/register.ts:1103`
  - body: `readPolicyStringArray()` lowercases every list entry before matching, but `mcp.servers` names are user-defined config keys that OpenClaw preserves. A policy such as `deny: ["ProdDocs"]` will be normalized to `proddocs` and fail to match `mcp.servers.ProdDocs`, so keep MCP server comparisons case-sensitive and only normalize provider ids where that is the contract.
  - confidence: 0.84
- **[P2] Remove or wire no-op metadata config keys:** `extensions/policy/openclaw.plugin.json:23-29`
  - body: The manifest accepts `plugins.entries.policy.config.requireRisk` and `requireSensitivity`, and the plugin reference shows them, but policy evaluation and runtime enforcement only read `policy.jsonc` `tools.requireMetadata`. Operators can therefore enable these config keys and get no tool metadata checks, so either implement compatibility support or drop the schema/docs entries.
  - confidence: 0.86
