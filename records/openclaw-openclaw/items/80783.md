## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep policy check from observing config state:** `extensions/policy/src/cli.ts:114`
  - body: `policy check` is documented as read-only, but this path still calls `readConfigFileSnapshot()` without `{ observe: false }`. The new config snapshot option defaults to observing snapshots, which writes config health state; `doctor --lint` already opts out, so policy check/watch should use the same non-observing read before this becomes an audit or CI command.
  - confidence: 0.9
- **[P2] Preserve MCP server IDs when matching policy lists:** `extensions/policy/src/doctor/register.ts:1103`
  - body: `readStringList()` lowercases every string before matching, but `mcp.servers` names are user-defined config keys that the MCP commands preserve. A policy such as `deny: ["ProdDocs"]` will be normalized to `proddocs` and fail to match `mcp.servers.ProdDocs`, so denied mixed-case servers can pass and allowed mixed-case servers can be reported as unapproved. Keep MCP server comparisons case-sensitive and only normalize provider ids where that is a defined contract.
  - confidence: 0.84
- **[P2] Remove or wire the no-op metadata config keys:** `extensions/policy/openclaw.plugin.json:23-29`
  - body: The manifest accepts `plugins.entries.policy.config.requireRisk` and `requireSensitivity`, but neither policy evaluation nor runtime enforcement reads those settings; both paths only consume `policy.jsonc` `tools.requireMetadata`. Accepting these keys makes operators think tool metadata checks are enabled when they are not, so either wire compatibility support or drop the schema entries.
  - confidence: 0.86
