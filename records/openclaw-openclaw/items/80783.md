## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep policy check read-only:** `extensions/policy/src/cli.ts:118`
  - body: `policy check` is documented as a read-only audit command, but this path still calls `readConfigFileSnapshot()` with the observing default. Since this PR adds `{ observe: false }` specifically to avoid config-health writes and `doctor --lint` uses it, policy audit and CI runs can still mutate local observation state unless this call passes `{ observe: false }`.
  - confidence: 0.92
- **[P2] Preserve MCP server id case:** `extensions/policy/src/doctor/register.ts:1103`
  - body: This helper lowercases every policy list entry, but it also backs `mcp.servers.allow` and `mcp.servers.deny`. OpenClaw preserves configured MCP server keys, so a policy like `deny: ["ProdDocs"]` becomes `proddocs` and no longer matches `mcp.servers.ProdDocs`.
  - confidence: 0.9
- **[P2] Remove or wire the metadata config keys:** `extensions/policy/openclaw.plugin.json:23-29`
  - body: The manifest advertises `requireRisk` and `requireSensitivity` under plugin config, and the plugin reference shows them, but policy evaluation and runtime enforcement only read `policy.jsonc` `tools.requireMetadata`. Operators can enable these config keys and get no tool metadata checks, so either wire them into evaluation or remove them from the config/docs surface.
  - confidence: 0.88
