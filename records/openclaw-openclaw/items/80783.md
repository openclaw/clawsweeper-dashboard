Full review comments:

- **[P2] Quote MCP server IDs in policy oc paths:** `extensions/policy/src/policy-state.ts:191`
  - body: MCP server names are arbitrary config keys, and existing configs can use names with spaces such as `Outlook Graph`. Interpolating `id` directly produces targets like `oc://openclaw.config/mcp/servers/Outlook Graph`, which the oc-path parser rejects for whitespace, so findings for valid server names have unusable `target`/`ocPath` audit links. Build these paths through the oc-path formatter/quoting helper and add a regression for a spaced server id.
  - confidence: 0.9
- **[P2] Quote model allowlist keys in policy oc paths:** `extensions/policy/src/policy-state.ts:528`
  - body: `agents.defaults.models` keys are model refs like `openrouter/*`, but `escapeRefSegment()` writes `openrouter~1*`; oc-path does not JSON-pointer-decode `~1`, and raw slashes would split the path. This makes model allowlist finding targets point at a non-existent key. Use valid oc-path quoting/formatting for the model-ref segment and test a provider/model key.
  - confidence: 0.88
