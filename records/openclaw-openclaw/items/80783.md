This PR extends the bundled policy plugin, docs, and tests with model-provider, MCP-server, private-network, tool-metadata/runtime-policy checks, plus approval metadata propagation.

## Best Possible Solution

Preserve MCP server key semantics in policy matching, or document and test an explicit canonicalization contract for both policy and config ids before maintainer policy-shape review resumes.

## Reproduction Assessment

Yes. Source inspection shows `readPolicyStringArray` lowercases policy MCP ids while `scanPolicyMcpServers` preserves configured server ids, so a mixed-case server name exercises the mismatch without needing a live MCP runtime.

## Solution Assessment

No. The conformance-layer direction is plausible, but this implementation should not merge until MCP ids are compared under a clear preserved-case or documented-canonicalization contract.
