## Summary

Keep open: the core refactor is directionally aligned, but the PR introduces a P1 account-resolution regression for thread-bound subagent spawns and still lacks live channel proof for the user-visible routing path.

## What This Changes

The PR moves native `thread: true` subagent binding from bundled Discord, Matrix, and Feishu `subagent_spawning` hooks into core session-binding preparation while deprecating the public hook type and updating docs/tests.

## Best Possible Solution

Keep the core-owned binding direction, but resolve omitted channel accounts through the existing plugin `defaultAccountId`/account-list contract before policy and capability checks, add regression coverage for named-account configs, and require redacted live channel proof before merge.

## Solution Assessment

No, not in its current form. Moving binding preparation into core is maintainable, but the implementation must reuse the existing channel default-account contract instead of duplicating raw config fallback logic.
