## Summary

Keep this PR open, but do not merge it as-is: the effective merge still treats synthetic fallback ids as real accounts, can hide unresolved configured-channel diagnostics, and lacks contributor real behavior proof for the Feishu/status path.

## What This Changes

The PR changes channel-summary account handling to skip summary/line emission when it sees no real accounts and adds an empty-account regression test.

## Best Possible Solution

Replace the account-count heuristic with an authored-account/config signal that suppresses only synthetic unconfigured defaults, keeps diagnostics for explicit unresolved configs, and adds Feishu-focused status or restart proof.

## Reproduction Assessment

Yes, source-reproducible: current main and the latest release route Feishu through the shared account helper, which can synthesize `default`, and channel summary then builds and emits a status line. I did not run a live Feishu gateway restart.

## Solution Assessment

No. The PR's `accountIds.length` heuristic is not the narrow maintainable fix because synthetic ids can look real and broad line suppression can hide unresolved configured-channel diagnostics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Distinguish synthetic defaults from real accounts:** `src/infra/channel-summary.ts:167`
  - body: `hasRealAccounts` treats any returned account id as real, but Feishu's shared helper can return a synthetic `default` id when no account map exists. That still lets the status adapter run and push the phantom default line this PR is meant to suppress; use an authored-account/config signal instead of `accountIds.length`.
  - confidence: 0.94
- **[P1] Keep the new regression test runnable:** `src/infra/channel-summary.test.ts:313-314`
  - body: In the GitHub potential merge result, the new test calls `setActivePluginRegistry` and `createTestRegistry`, but those helpers are not imported in the merged file. The focused regression test would throw before exercising the behavior; keep the helper imports through the merge or use the existing `plugins` option instead.
  - confidence: 0.88
- **[P2] Exercise the real Feishu fallback path:** `src/infra/channel-summary.test.ts:320-321`
  - body: The new fixture forces `accountIds: []`, but real Feishu uses `createAccountListHelpers`, which can return `default` for an empty account map. This test can pass while the reported Feishu path still sets `hasRealAccounts` to true, so the regression should cover the synthetic-default helper behavior.
  - confidence: 0.91
- **[P2] Keep unresolved channel configs visible:** `src/infra/channel-summary.ts:224-225`
  - body: Dropping the whole status line when both `accountIds` and `configuredEntries` are empty can hide channels that are configured but cannot list an account id yet, such as unresolved SecretRef-based Nostr config. Suppress only synthetic unconfigured defaults, not all diagnostics for unresolved configured channels.
  - confidence: 0.84
