## Summary

Keep this PR open, but do not merge it as-is: current main and the latest release still have the reported Feishu/default status path, while the PR's guard still treats synthetic default ids as real accounts, the potential merge result leaves the added test broken, and the external PR lacks after-fix real behavior proof.

## What This Changes

The PR gates channel-summary status building and line emission behind a `hasRealAccounts` check and adds a no-accounts regression test for the phantom Feishu default status line.

## Best Possible Solution

Use an authored-account/config signal to suppress only synthetic unconfigured defaults, preserve diagnostics for configured but unresolved accounts, and add Feishu-focused regression plus redacted real status or restart proof.

## Reproduction Assessment

Yes for the source-level path: current main and `v2026.5.12` can synthesize `default` from the shared helper and feed it through Feishu channel-summary/status output. No live Feishu restart was established, so the persistence part remains unverified.

## Solution Assessment

No. The PR's `accountIds.length` heuristic is not the narrow maintainable fix because synthetic ids can look real and broad line suppression can hide unresolved configured-channel diagnostics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Distinguish synthetic defaults from real accounts:** `src/infra/channel-summary.ts:167`
  - body: `hasRealAccounts` treats any returned account id as real, but Feishu's shared helper returns a synthetic `default` id when no account map exists. That still lets the status adapter run and push the phantom default line this PR is meant to suppress; use an authored-account/config signal instead of `accountIds.length`.
  - confidence: 0.94
- **[P1] Keep the new regression test runnable:** `src/infra/channel-summary.test.ts:313-314`
  - body: In the potential merge result, the new test calls `setActivePluginRegistry` and `createTestRegistry`, but those helpers are not imported in the merged file. The focused regression test would throw before exercising the behavior; update the branch to current test patterns or pass the fixture through the existing `plugins` option.
  - confidence: 0.88
- **[P2] Exercise the real Feishu fallback path:** `src/infra/channel-summary.test.ts:320-321`
  - body: The new fixture forces `accountIds: []`, but real Feishu uses `createAccountListHelpers`, which can return `default` for an empty account map. This test can pass while the reported Feishu path still sets `hasRealAccounts` to true, so cover the synthetic-default helper behavior directly.
  - confidence: 0.91
- **[P2] Keep unresolved channel configs visible:** `src/infra/channel-summary.ts:224-225`
  - body: Dropping the whole status line when both `accountIds` and `configuredEntries` are empty can hide channels that are configured but cannot resolve account ids yet, such as SecretRef-managed Nostr config. Suppress synthetic unconfigured defaults without making unresolved configured channels disappear from status output.
  - confidence: 0.84
