## Summary

Keep this PR open but not merge-ready: the patch still treats synthetic helper defaults as real accounts, the new regression test is not runnable on the current target, and the PR has no real Feishu/status proof.

## What This Changes

The PR gates channel-summary status adapter calls and line emission behind a `hasRealAccounts` check and adds a regression test for phantom Feishu default-account status output.

## Best Possible Solution

Suppress only synthetic unconfigured defaults using an authored-account or explicit configuration signal, preserve diagnostics for unresolved configured accounts, and add Feishu-focused regression coverage plus redacted real status proof.

## Reproduction Assessment

Yes for the source-level path: current main can synthesize `default` through the shared helper and feed it into channel summary/status output. I did not establish a live Feishu gateway restart reproduction, so the persistence part remains unverified.

## Solution Assessment

No. The PR's `accountIds.length` heuristic conflates synthetic fallback IDs with real accounts, so the maintainable fix needs an explicit authored/configured-account signal while keeping unresolved configured-channel diagnostics visible.
