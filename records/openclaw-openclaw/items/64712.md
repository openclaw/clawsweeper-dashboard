## Summary

Keep this PR open, but it is not merge-ready: the patch still relies on `accountIds.length`, which is true for the shared helper's synthetic `default` id, and it can also hide existing channel diagnostics. The PR is paired with the still-open Feishu report at https://github.com/openclaw/openclaw/issues/64707 and needs a corrected implementation plus real behavior proof before merge.

## What This Changes

The PR gates channel-summary status building and line emission behind a `hasRealAccounts` check and adds a no-accounts regression test for the phantom Feishu default status line.

## Best Possible Solution

Use an authored-account or explicit configuration signal to suppress only synthetic unconfigured defaults, preserve diagnostics for configured but unresolved accounts, and add Feishu-focused regression coverage plus redacted real status or restart proof.

## Reproduction Assessment

Yes for the source-level path: current main can synthesize `default` through the shared helper and feed it into channel summary/status output. No live Feishu gateway restart was established, so the persistence part remains unverified.

## Solution Assessment

No. The PR's `accountIds.length` heuristic is not the best fix because synthetic ids can look real, and broad line suppression can hide configured but unresolved channel diagnostics.
