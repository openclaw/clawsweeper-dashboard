## Summary

Keep open: current main still has the fallback/default-account status path, and this PR remains incorrect because its guard treats Feishu's synthetic `default` id as real while also hiding unresolved-channel diagnostics; after-fix real behavior proof is still missing.

## What This Changes

The PR changes channel-summary rendering to suppress a synthetic default-account status line when a Feishu-like channel has no configured accounts and adds a regression test.

## Best Possible Solution

Repair or replace this PR with a fix that distinguishes authored or persisted account state from synthetic fallback ids, preserves diagnostics for explicitly configured but unresolved channels, and adds Feishu-focused regression coverage.

## Reproduction Assessment

Yes by source inspection: current main still resolves empty account lists through default-account fallback paths, and Feishu inherits a shared helper that can synthesize `default`. I did not run a live gateway restart in this read-only review.

## Solution Assessment

No. The PR's `accountIds.length > 0 || configuredEntries.length > 0` heuristic is not the narrowest maintainable fix because it cannot tell real accounts from synthetic fallback ids and can hide unresolved configured-channel diagnostics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Distinguish synthetic Feishu defaults from real accounts:** `src/infra/channel-summary.ts:194`
  - body: `hasRealAccounts` is true whenever `accountIds` is non-empty, but Feishu's shared account helper can return `default` when no accounts are configured. In the real Feishu path this still lets the status adapter run and can emit the phantom default line the PR is trying to suppress.
  - confidence: 0.91
- **[P2] Import the registry helpers used by the test:** `src/infra/channel-summary.test.ts:369-370`
  - body: The new test calls `setActivePluginRegistry` and `createTestRegistry`, but this file does not import either symbol and the PR diff adds no import hunk. The regression test will fail before it verifies the behavior.
  - confidence: 0.95
- **[P2] Keep unresolved configured channels visible:** `src/infra/channel-summary.ts:242-246`
  - body: The new guard suppresses the whole status line when `accountIds` and `configuredEntries` are both empty. That can hide channels that are explicitly configured but cannot resolve account IDs yet, such as Nostr configs backed by non-literal SecretRef private keys, turning an actionable status diagnostic into no output.
  - confidence: 0.84
