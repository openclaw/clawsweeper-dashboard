## Summary

Keep open: the PR is a plausible bug-fix attempt, but the current patch still treats Feishu's synthetic `default` id as a real account, adds a regression test that is not runnable in the merge result, and lacks after-fix real behavior proof.

## What This Changes

The PR gates channel-summary status building and line emission on a `hasRealAccounts` heuristic and adds a no-configured-accounts regression test.

## Best Possible Solution

Use an authored-account/config signal that suppresses only synthetic unconfigured defaults, preserves diagnostics for explicit unresolved configs, and adds Feishu-focused restart/status proof.

## Reproduction Assessment

Yes, source-reproducible: current main can synthesize `default` through the shared account helper and feed it into Feishu channel-summary/status paths. I did not establish a live Feishu restart reproduction.

## Solution Assessment

No. The PR's `accountIds.length` heuristic is not the narrow maintainable fix because synthetic ids can look real, and broad line suppression can hide diagnostics for configured but unresolved channels.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Distinguish synthetic defaults from real accounts:** `src/infra/channel-summary.ts:167`
  - body: `hasRealAccounts` treats any returned account id as real, but Feishu's shared helper can synthesize `default` when no account map exists. That still lets the status adapter run and push the phantom default line this PR is meant to suppress; use an authored-account/config signal instead of `accountIds.length`.
  - confidence: 0.94
- **[P1] Keep the regression test runnable:** `src/infra/channel-summary.test.ts:313-314`
  - body: In the GitHub potential merge result, the new test calls `setActivePluginRegistry` and `createTestRegistry`, but those helpers are not imported in this file. The focused regression test would fail before exercising the behavior; keep the imports or use the existing `plugins` option instead.
  - confidence: 0.88
- **[P2] Exercise the real Feishu fallback path:** `src/infra/channel-summary.test.ts:320-321`
  - body: The new fixture forces `accountIds: []`, but real Feishu uses `createAccountListHelpers`, which can return `default` for an empty account map. This test can pass while the reported Feishu path still sets `hasRealAccounts` to true, so the regression should cover the synthetic-default helper behavior.
  - confidence: 0.91
- **[P2] Keep unresolved channel configs visible:** `src/infra/channel-summary.ts:222-225`
  - body: Dropping the whole status line when both `accountIds` and `configuredEntries` are empty can hide channels that have meaningful config but cannot resolve account IDs yet, such as SecretRef-managed Nostr config. Suppress only synthetic unconfigured defaults, not all diagnostics for unresolved configured channels.
  - confidence: 0.84
