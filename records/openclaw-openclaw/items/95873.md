## What This Changes

The issue requests that early failed isolated cron agent runs preserve already-known `session_id`, `provider`, and `model` in `cron_run_logs`.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/95943, then close this issue once the merged fix preserves attribution for timer-built timeout and cancellation rows.

## Reproduction Assessment

Yes. Current-main source shows timer-built timeout and cancellation outcomes can omit attribution even though the execution callbacks already expose session, provider, and model, and the run-log writer only persists what `coreResult` carries.

## Solution Assessment

Yes. The timer owns the synthetic timeout/cancel outcomes, so preserving attribution there is the narrowest maintainable fix; a run-log writer change alone cannot recover metadata it never receives.
