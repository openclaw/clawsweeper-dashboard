Review and land or replace https://github.com/openclaw/openclaw/pull/95943, then close this issue after the merged fix preserves attribution for post-runner timeout and cancel rows.

## Reproduction Assessment

Yes. Current-main source shows timer-owned timeout and cancellation outcomes omit attribution while the run-log path persists only the returned `coreResult` fields.

## Solution Assessment

Yes. Carrying already-known attribution at the timer/outcome boundary is the narrow maintainable fix; changing only the run-log writer would not recover metadata it never receives.
