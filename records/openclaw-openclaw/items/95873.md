## What This Changes

The issue requests preserving `session_id`, `provider`, and `model` in `cron_run_logs` when a cron agent run fails after run identity is already known.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/95943, then close this issue once merged code preserves attribution for timer-built timeout/cancel rows while leaving true pre-runner blanks unchanged.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: the timer has last-known execution identity but returns synthetic timeout/cancel outcomes without provider/model/session, and the run-log writer persists only fields from that outcome.

## Solution Assessment

Yes. Carrying attribution into the timer-built outcomes is the narrowest maintainable fix because changing `appendCronRunLog` cannot recover metadata that never reaches the cron finished event.
