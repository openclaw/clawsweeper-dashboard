## Summary

Keep open: the regression area is plausible, but this PR skips the documented context-engine subagent hook for every isolated spawn, conflicting with current tests/types, and it lacks contributor real behavior proof for the reported Ollama workflow.

## What This Changes

The PR adds an early return from context-engine subagent preparation whenever the resolved spawn context is isolated and adds a test asserting the context engine is not initialized for isolated spawns.

## Best Possible Solution

Fix isolated lightweight payload bloat while preserving normal isolated/fork context-engine lifecycle hooks, or deliberately revise that hook contract with maintainer approval, then add focused tests and redacted live Ollama proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No for the full live Ollama CPU/fallback symptom; source inspection does show current main reaches context-engine preparation after selecting isolated context, and existing tests give a high-confidence source conflict with this PR's broad skip.

## Solution Assessment

Is this the best way to solve the issue? No. The early return is too broad because it bypasses the isolated context-engine hook that current tests, docs, and exported types already model; a safer fix should preserve the hook or intentionally update the contract.
