## Summary

Keep open: the linked regression is source-backed, but this patch is not mergeable because it bypasses the isolated context-engine lifecycle hook for all isolated spawns and still lacks after-fix real behavior proof for the reported Ollama setup.

## What This Changes

The PR adds an early return that skips context-engine subagent preparation for isolated context spawns and adds a regression test asserting the context engine is not resolved or initialized for that path.

## Best Possible Solution

Fix isolated lightweight payload bloat with a narrower no-inherit preparation path that preserves normal isolated and forked context-engine hooks, plus updated tests and live or redacted-log proof for the reported runtime flow.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No for the full live Ollama CPU/fallback symptom; source inspection does show current main passes `lightContext` into native subagent spawning and then still reaches context-engine preparation after isolated context resolution.

## Solution Assessment

Is this the best way to solve the issue? No. The early return is too broad because it skips the isolated `prepareSubagentSpawn` hook that current tests, docs, and exported types already expose; the safer fix should constrain inherited-payload behavior or intentionally revise the contract with owner review.
