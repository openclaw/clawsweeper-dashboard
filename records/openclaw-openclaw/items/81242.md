## Summary

Keep open: the proposed fix targets a plausible regression, but the early return bypasses the context-engine `prepareSubagentSpawn` hook for every isolated spawn and conflicts with current-main tests that intentionally cover isolated preparation. The PR also lacks required real behavior proof for the reported Ollama/runtime symptom.

## What This Changes

The PR adds an early return from `prepareContextEngineSubagentSpawn` for isolated subagent contexts and adds a regression test asserting the context engine is not invoked.

## Best Possible Solution

Preserve or deliberately revise the context-engine lifecycle contract, then fix isolated-mode payload bloat with a no-inherit preparation path and updated existing context-mode tests plus live runtime proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No for the full live Ollama CPU/fallback symptom; source inspection does show current main reaches context-engine preparation after selecting isolated context, matching the reported suspect path.

## Solution Assessment

Is this the best way to solve the issue? No. The early return is too broad because it bypasses the isolated context-engine hook that current tests and exported types already model; a safer fix should preserve the hook or intentionally update the contract with maintainer review.
