## Summary

Keep open: the reported regression appears source-backed, but this PR's broad isolated-mode skip conflicts with the current context-engine lifecycle contract and still lacks after-fix real behavior proof for the Ollama `sessions_spawn` symptom.

## What This Changes

The PR adds an early return from `prepareContextEngineSubagentSpawn` for all isolated subagent contexts and adds a unit test asserting context engines are not invoked for isolated spawns.

## Best Possible Solution

Fix isolated lightweight native spawns without breaking the normal isolated/fork context-engine lifecycle, then add focused regression tests and redacted real-run proof from the affected setup.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No for the full Ollama CPU/fallback symptom; source inspection does show current main reaches context-engine preparation after isolated context resolution while `lightContext` only changes later bootstrap flags.

## Solution Assessment

Is this the best way to solve the issue? No. The early return is too broad because it skips `prepareSubagentSpawn` for every isolated spawn, conflicting with current tests and the context-engine contract; scope the fix to the lightweight payload path or revise the contract deliberately.
