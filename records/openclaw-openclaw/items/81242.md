## Summary

Keep open: the linked regression is source-backed, but this PR’s fix is too broad because it bypasses the context-engine subagent lifecycle hook for every isolated spawn, while current tests, docs, and exported types still model isolated-mode preparation as valid.

## What This Changes

The PR adds an isolated-mode early return in subagent context-engine preparation and a regression test asserting context engines are not initialized or resolved for isolated spawns.

## Best Possible Solution

Fix isolated lightContext native subagent payload bloat without disabling prepareSubagentSpawn for all isolated modes; preserve the existing hook or deliberately update the contract with matching tests, docs, and live runtime proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No for the full live Ollama CPU/fallback symptom, but source inspection gives high confidence that current main reaches context-engine preparation for isolated spawns and that this PR would bypass an existing isolated-mode hook contract.

## Solution Assessment

Is this the best way to solve the issue? No. The early return is too broad; a safer fix should be scoped to the lightweight inherited-payload problem or intentionally revise the context-engine contract with owner review.
