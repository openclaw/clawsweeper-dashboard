## Summary

Keep open; this is a recent security-labeled CI PR with a plausible fix, but the new installer should verify the pinned release asset before executing it.

## What This Changes

The PR adds `scripts/install-opengrep.sh`, switches both OpenGrep workflows to it, and updates local missing-tool guidance.

## Best Possible Solution

Keep the repo-owned installer direction, but pin or verify the v1.19.0 asset before execution and land it through normal security-owner review.

## Reproduction Assessment

Yes. The current workflow runs the upstream installer with `-v v1.19.0`, and the upstream installer validates that version through an `echo | grep -q` pipeline under `pipefail`; the linked failing PR context shows the same install step failing before any scan ran.

## Solution Assessment

No as submitted. Moving to a repo-owned installer is the right fix shape, but it should preserve asset integrity by checking pinned SHA-256 digests or verifying `.cert`/`.sig` before running the downloaded binary.
