## Best Possible Solution

Keep the repo-owned installer direction, but verify the pinned v1.19.0 asset with checked-in SHA-256 values or cosign before chmod/execution, then land through security-owner review.

Yes. Current main uses the upstream v1.19.0 installer path with the pipefail-sensitive version check, and the related failing PR context shows the install step failing before any scan ran.

## Solution Assessment
