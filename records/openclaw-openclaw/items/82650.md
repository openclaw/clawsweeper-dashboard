## Summary

Current `main` still has the reported edge case, so this should stay open as a narrow bug. The helper accepts an explicit `limit: 0`, compares against it, and then uses `slice(-limit)`, which makes zero return the whole log list instead of no entries.

## What This Changes

The issue asks for `loadSessionLogs` to return no entries when callers pass zero, negative, or non-finite limits.

## Best Possible Solution

Normalize the explicit limit inside `loadSessionLogs`, preserving the omitted default and missing-file `null` behavior while returning an empty array for zero, negative, and non-finite explicit limits.

## Reproduction Assessment

Yes from source inspection: an explicit `limit: 0` flows into `sortedLogs.slice(-0)`, and a Node probe confirms that returns the full array. I did not run the full helper through Vitest because this review was read-only.

## Solution Assessment

Yes, the narrow maintainable fix is to validate and normalize `params.limit` in `loadSessionLogs` itself, since both direct helper callers and the Gateway RPC path can reach that function.
