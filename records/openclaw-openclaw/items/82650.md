## Summary

Current main still has the reported limit bug, so this should stay open as a narrow, source-proven fix candidate. Explicit zero reaches `slice(-0)`, and invalid direct helper limits are not normalized before slicing or returning the full sorted log list.

## What This Changes

The issue asks for `loadSessionLogs` to return no logs for explicit zero, negative, and non-finite limits while preserving positive-limit behavior.

## Best Possible Solution

Normalize the explicit limit inside `loadSessionLogs`, preserving the omitted default and missing-transcript `null` behavior while adding regression coverage for zero, negative, non-finite, and positive limits.

## Reproduction Assessment

Yes. Source inspection shows `limit: 0` flows into `sortedLogs.slice(-0)`, and a focused Node probe confirms the expression returns the full array; I did not run the full helper through Vitest in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to validate and normalize `params.limit` in `loadSessionLogs` itself, with focused regression tests in the existing session-cost usage test file.
