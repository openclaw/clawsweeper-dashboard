## Summary

Keep open: current main still has a source-reproducible raw `codex-cli/*` helper path, but current docs and doctor tests treat that prefix as legacy config to repair, so this needs a maintainer runtime-contract decision before code changes.

## What This Changes

The issue asks OpenClaw to make legacy `codex-cli/gpt-5.4` helper and embedded runs resolve and authenticate like the OpenAI Codex path instead of failing during session-memory slug generation.

## Best Possible Solution

Maintainers should choose and document one legacy `codex-cli/*` contract: enforce doctor-only repair with clearer diagnostics, or add narrow helper/runtime normalization with regression coverage.

## Reproduction Assessment

Yes. A raw primary model of `codex-cli/gpt-5.4` still flows from the slug helper into provider-scoped embedded model resolution, where no raw `codex-cli` model owner is guaranteed before `model_not_found`.

## Solution Assessment

Unclear. The closed compatibility PR direction would fix the symptom, but current docs and doctor tests prefer canonical `openai/*` refs plus `agentRuntime.id: "codex"`, so maintainers should choose the contract first.
