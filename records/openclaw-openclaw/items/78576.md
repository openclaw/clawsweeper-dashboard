## Summary

Current main and the shipped release path no longer depend on the bundled-only capability lazy-load route for agent web_search, so the standalone Brave provider problem reported against 2026.5.2 is already implemented/fixed.

## What This Changes

The issue asks for globally installed @openclaw/brave-plugin web_search providers to load at runtime instead of being rejected by a bundled-origin lazy-load filter.

## Best Possible Solution

Close this issue and direct 2026.5.2 users to update to a release at or after 2026.5.3, with v2026.5.6 containing the verified current implementation.

## Reproduction Assessment

Yes for the historical 2026.5.2 report: the issue describes a credible bundled-origin filter failure, and current source shows the agent runtime now takes a different web-provider discovery path. I did not run a live macOS launchd gateway with a Brave key during this read-only review.

## Solution Assessment

Yes. The current solution keeps web_search on the generic runtime web-provider loader with manifest candidate discovery and empty-registry fallback, which is narrower than bundling Brave into core and preserves external plugin support.
