## Summary

Keep open. Current main has partial runtime-deps mitigations, but the lock-removal path still returns false for PID-alive locks before stale-age checks when start-time evidence is unavailable, and gateway startup still does plugin bootstrap/prestaging before HTTP listen, so a runtime-deps stall can keep /healthz unavailable. Open PR #74950 is the relevant follow-up, but it is still unmerged and needs Docker/Unraid validation before this issue can close.

## What This Changes

The issue asks OpenClaw to recover Docker/Unraid gateway startup and doctor --fix behavior after the 2026.4.24-2026.4.26 bundled runtime-deps regressions.

## Best Possible Solution

Keep this issue open until #74950 or an equivalent stale Docker runtime-deps lock fix lands, then validate a Docker/Unraid or focused Docker stale-lock scenario where the gateway binds /healthz and openclaw doctor --fix exits cleanly. Any remaining plugin-loader hot-loop behavior should stay tracked under #73532.
