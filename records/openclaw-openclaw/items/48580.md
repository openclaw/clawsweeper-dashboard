The issue reports that `acpx codex sessions new` can create a Codex session record that remains open even though the spawned process immediately exits because stdin is not a terminal.

## Best Possible Solution

Reconcile ACPX session records with live process and terminal truth at the supported `acpx/runtime` boundary, then land or replace the terminal-truth work with redacted real Codex ACP proof for failed startup and dead-session recovery.

## Solution Assessment

Unclear. The merged lease cleanup is a useful mitigation, but the best fix should prove creation/status terminal truth at the `acpx/runtime` boundary instead of relying only on startup or close-time cleanup.
