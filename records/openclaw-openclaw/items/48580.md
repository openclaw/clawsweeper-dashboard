## Summary

Keep open: the report describes a plausible ACPX session-state drift bug, current main still delegates session creation/status to `acpx/runtime` without OpenClaw-owned terminal-truth artifacts, and the related implementation PR is still open.

## What This Changes

The issue asks ACPX Codex session creation to either satisfy Codex's TTY requirements or mark the session closed/failed when the spawned process exits immediately.

## Best Possible Solution

Land a focused ACPX terminal-truth/status reconciliation fix with regression coverage for early process exit after session creation, keeping shared runtime behavior in the ACPX-owned layer.

No high-confidence live reproduction was established in this read-only checkout. The source path and issue logs make the drift plausible, but the current `acpx/runtime` dependency implementation and macOS Codex ACP process behavior were not available to run here.
