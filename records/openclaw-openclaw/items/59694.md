## What This Changes

The branch imports `isBlockedObjectKey` in `src/shared/config-eval.ts` and returns `undefined` before traversing blocked prototype-key path segments.

## Best Possible Solution

Land the guard with focused `config-eval` regression assertions for `__proto__`, `prototype`, and `constructor`, then require redacted contributor real behavior proof and fresh CI before merge.

## Reproduction Assessment

Yes. Source inspection on current main shows `resolveConfigPath({}, "constructor")` reaches inherited property access before the PR’s guard would return `undefined`.

## Solution Assessment

Yes for the runtime code. Checking each raw path segment with the existing blocked-key helper before property access is the narrowest maintainable fix; the remaining gaps are proof and regression coverage.
