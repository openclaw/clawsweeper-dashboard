## What This Changes

The PR imports `isBlockedObjectKey` into `src/shared/config-eval.ts` and returns `undefined` when `resolveConfigPath` sees a blocked prototype-key path segment.

## Best Possible Solution

Land the inline blocked-key guard with focused `config-eval` regression assertions and redacted contributor proof, then rerun the relevant checks before maintainer landing.

## Reproduction Assessment

Yes, source inspection provides a high-confidence reproduction path. Current `main` indexes raw path segments in `resolveConfigPath`, so a path such as `constructor` can reach inherited prototype data instead of being rejected.

## Solution Assessment

Yes for the runtime code. Because `resolveConfigPath` accepts raw dot-separated strings, checking each segment with the existing blocked-key helper before property access is the narrowest maintainable fix; proof and regression coverage are the remaining gaps.
