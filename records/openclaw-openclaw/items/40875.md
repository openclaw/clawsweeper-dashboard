## Summary

Keep this PR open. Current main still does not document heartbeat-specific model fallback behavior, but the proposed docs text is not correct enough to merge because it says a different-provider heartbeat override errors without trying other models, while current fallback logic can still append the configured primary when no explicit fallback override is supplied.

## What This Changes

The PR adds a changelog entry and two heartbeat documentation bullets describing when `heartbeat.model` uses or skips the standard fallback chain.

Keep runtime code unchanged and add the required changelog attribution.
