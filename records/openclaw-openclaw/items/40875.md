## Summary

Keep open: current main still has a heartbeat.model docs gap, but this PR’s wording is inaccurate because a cross-provider heartbeat override can still retry the configured primary when no explicit fallback override is supplied.

## What This Changes

The PR adds a heartbeat.model fallback-behavior note to the heartbeat docs and an Unreleased changelog entry.

## Best Possible Solution

Add a corrected ParamField note explaining that no override and same-provider overrides follow the standard chain, cross-provider overrides skip unrelated configured fallbacks unless the override is configured, and the configured primary remains a retry target when no explicit fallback override is supplied; coordinate with https://github.com/openclaw/openclaw/pull/69495 if it lands first.

## Reproduction Assessment

Not applicable as a docs-only PR. Source inspection of current main’s fallback resolver plus regression coverage gives a high-confidence verification path for the documented behavior.

## Solution Assessment

No. The docs-only direction is right, but the patch must document the configured-primary retry and target the current ParamField docs structure before it is the best fix.
