## What This Changes

The PR changes Telegram config/status accessors so `allowFrom` and `defaultTo` metadata reads use merged config data without strict-resolving `botToken` SecretRefs.

## Best Possible Solution

Keep the shipped config-only Telegram accessor implementation on main and close this branch; the linked issue can be closed against the shipped fix after PR cleanup.

## Reproduction Assessment

Yes. The linked issue gives a concrete file-backed Telegram `botToken` SecretRef status path, and current regression coverage exercises the same read-only accessor behavior; I did not run a full live `openclaw status` flow in this read-only review.

## Solution Assessment

Yes. The config-only `resolveAccessorAccount` split is the narrow maintainable fix because it keeps status metadata reads away from strict token resolution while preserving strict runtime credential handling.
