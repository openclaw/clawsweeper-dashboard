## Summary

Keep open: this is a supported Discord behavior, and current main does not yet prove the reporter's multi-account installed-plugin path is fixed or invalid. ## What This Changes

The issue reports Discord guild messages being dropped as `no-mention` in a multi-account setup even when `requireMention: false` or an explicit bot mention should allow them.

## Best Possible Solution

Reproduce the reporter's exact multi-account Discord package and config state, then either fix account/guild/channel preflight resolution in the Discord plugin or document the precise config/package repair path if current source behavior is correct.
