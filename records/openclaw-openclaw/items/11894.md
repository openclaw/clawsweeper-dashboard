## Summary

Keep open: current main has transcript-backed cost summaries, live per-turn usageState, and auth-profile routing, but completed-call usage still is not retained or aggregated by the resolved auth profile that paid for the call. The burn0 workaround and the merged usageState hook are useful adjacent work, but they do not provide native durable auth-profile attribution.

## What This Changes

Request native per-call usage/cost reporting that attributes completed model calls to the resolved auth profile and exposes fallback/profile-selection visibility.

## Best Possible Solution

A single canonical completed-call accounting contract should record redacted resolved-profile attribution and feed transcript, status, gateway, UI, and plugin views without credential material.

## Reproduction Assessment

Yes, at source level: current main keeps usage/cost data and auth-profile routing as separate facts, and the completed-call usage contracts do not join them. No live provider repro was run because this read-only review only needed source confirmation of a feature gap.

## Solution Assessment

No current implementation fully solves it. The requested outcome is valid, but the maintainable path is one canonical completed-call attribution contract before deriving transcript, status, gateway, UI, hook, or event views.
