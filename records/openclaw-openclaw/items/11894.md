## Summary

Keep open. Current main and v2026.6.5 still expose completed-call usage and cost by provider/model/session, while resolved auth-profile attribution and per-profile cost buckets remain missing.

## What This Changes

Request native per-call cost tracking with resolved auth-profile attribution and fallback/profile event visibility.

## Best Possible Solution

Add privacy-safe resolved authProfileId metadata at the completed model-call usage seam, aggregate cost by auth profile in usage/status diagnostics, and emit explicit fallback/profile-rotation events without storing credential material.

Not applicable as a bug reproduction; this is a feature request. Source, docs, release, and live issue inspection give a high-confidence verification path showing the requested auth-profile attribution is absent.

## Solution Assessment

Unclear for the exact public API shape, but native additive attribution at the normalized completed-call usage seam is the maintainable direction. External per-request tools and broader usage-ledger issues are useful context, not replacements for OpenClaw's resolved auth-profile awareness.
