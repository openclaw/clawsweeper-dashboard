## Summary

Keep open. Current main has transcript-backed usage and cost summaries plus auth-profile rotation state, but completed model-call accounting is still keyed by provider/model/session rather than the resolved auth profile that paid for each call; the burn0 discussion and https://github.com/openclaw/openclaw/issues/55379 remain external cost-workaround context, not native OpenClaw auth-profile attribution.

## What This Changes

Request native per-call cost tracking that records which auth profile paid for each model call and exposes fallback/profile cost attribution in transcripts, status, or system events.

## Best Possible Solution

Add safe additive resolved authProfileId metadata for completed model calls, aggregate cost by auth profile in usage/status surfaces, and emit clear fallback/profile-rotation events without storing credential material.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives a high-confidence verification path showing current main lacks resolved auth-profile fields in transcript parsing, usage cache/types, aggregates, diagnostics, and status output.

## Solution Assessment

Unclear for the exact public API shape, but the maintainable direction is native additive attribution at the normalized completed model-call usage seam. burn0 can remain an external per-request cost workaround, but it cannot replace OpenClaw's resolved auth-profile awareness.
