## Summary

Keep open. Current main has token/cost summaries and auth-profile rotation/fallback, but completed call usage is still parsed and aggregated by provider/model/session rather than by the resolved auth profile, and the burn0/#55379 discussion remains only an interim cost workaround.

## What This Changes

Request native per-call cost metadata, per-auth-profile cost attribution, and fallback/profile-use visibility for sessions using multiple auth profiles.

## Best Possible Solution

Record safe resolved authProfileId metadata when each model call completes, propagate it through transcript usage/status diagnostics, aggregate costs by profile, and emit a clear profile-rotation/fallback event without exposing secrets.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source inspection: current transcript parsing, usage aggregation, Gateway usage types, status output, and docs expose costs by provider/model/session but not by resolved auth profile.

## Solution Assessment

Unclear for the exact UI/API shape, but native runtime metadata is the right product boundary. A network-layer workaround like burn0/#55379 can help with request cost, but it cannot reliably attribute OpenClaw's selected auth profile.
