## Summary

Keep this open. Current main has transcript-backed cost summaries and the merged live `usageState` hook, but completed-call usage still does not persist or aggregate the resolved auth profile that paid for a call, so the central fallback-attribution request remains unsolved.

## What This Changes

Add native per-call cost tracking that attributes each completed model call to the resolved auth profile, especially when auth-profile fallback or rotation occurs.

## Best Possible Solution

Define one safe completed-model-call attribution contract that records the resolved profile identifier and selection reason, then derive transcript, status, gateway usage, plugin, and event views from that canonical source without exposing credential material.

## Reproduction Assessment

Yes, at source level. Current main has separate per-call usage/cost data and auth-profile routing, but the completed-call usage contracts and aggregators do not join those facts by resolved profile.

## Solution Assessment

Unclear as proposed. The requested outcome is valid, but the best implementation should first establish a canonical safe attribution contract rather than add one-off fields independently to transcript, status, hook, and event surfaces.
