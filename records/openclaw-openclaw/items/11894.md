## Summary

Keep open. Current main still has transcript-backed cost summaries and auth-profile routing, but completed model-call usage is exposed by provider/model/session rather than by the resolved auth profile that paid for the call; the burn0 discussion is useful interim context, not a native OpenClaw attribution solution.

## What This Changes

Request native per-call cost visibility with resolved auth-profile attribution and fallback/profile-rotation indication.

## Best Possible Solution

Record safe resolved authProfileId metadata at completed model-call usage time, aggregate cost by auth profile, and expose additive transcript, Gateway/status/UI, and fallback-event diagnostics without storing credential material.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. The high-confidence verification path is source inspection: current main exposes provider/model/session usage and auth routing, but not completed-call resolved auth-profile attribution.

## Solution Assessment

Unclear for the exact public API and UI shape, but native metadata at the normalized model-call usage seam is the maintainable direction. Network-layer burn0-style tracking can help with per-request cost, but it cannot reliably attribute OpenClaw's selected auth profile.
