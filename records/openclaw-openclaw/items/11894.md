## Summary

Keep open. Current main has transcript-backed per-call usage/cost parsing and auth-profile routing, but completed-call usage is still exposed and aggregated by provider/model/session rather than by the resolved auth profile that paid for the call; the burn0 discussion is a useful workaround, not native attribution.

## What This Changes

Add per-call cost tracking and resolved auth-profile attribution for model calls, including fallback-profile visibility and per-profile cost summaries.

## Best Possible Solution

Record safe resolved authProfileId metadata at completed model-call usage time, aggregate costs by auth profile, and expose additive transcript, Gateway/status/UI, and fallback-event diagnostics without credential material.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The high-confidence verification path is source inspection showing current main exposes provider/model/session usage and auth routing, but not completed-call resolved auth-profile attribution.

## Solution Assessment

Unclear for the exact public API and UI shape, but native safe metadata at the normalized model-call usage seam is the maintainable direction. Network-layer burn0-style tracking can help with per-request costs, but it cannot reliably attribute OpenClaw's selected auth profile.
