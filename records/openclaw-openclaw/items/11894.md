## Summary

Keep open. Current main has transcript-backed cost summaries, Gateway/UI usage aggregation, status cost display, and auth-profile rotation/fallback diagnostics, but completed model-call usage is still not attributed to the resolved auth profile that paid for each call; the burn0 discussion is useful workaround context, not a native replacement.

## What This Changes

Request native per-call cost tracking with resolved auth-profile attribution, per-profile cost summaries, and a fallback/profile-rotation event.

## Best Possible Solution

Record safe resolved authProfileId metadata at the normalized completed-model-call usage seam, aggregate cost by auth profile, and expose additive transcript, Gateway/status/UI, and fallback-event diagnostics without storing secrets.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The high-confidence verification path is source inspection showing current main exposes provider/model/session usage and auth routing, but not completed-call resolved auth-profile attribution.

## Solution Assessment

Unclear for the exact public API and UI shape, but native metadata at the normalized model-call usage seam is the maintainable direction. Network-layer burn0-style tracking can help with per-request costs, but it cannot reliably attribute OpenClaw's selected auth profile.
