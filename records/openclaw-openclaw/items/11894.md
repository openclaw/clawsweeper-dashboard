## Summary

Keep open. Current main and the latest shipped release still expose usage and cost by provider/model/session while auth-profile routing exists separately; completed-call accounting does not persist or aggregate the resolved auth profile that paid for each call.

## What This Changes

Request native per-call cost tracking that attributes completed model calls to the resolved auth profile, including fallback visibility and per-profile cost summaries.

## Best Possible Solution

Add privacy-safe resolved authProfileId metadata at the completed model-call usage seam, aggregate cost by auth profile in usage/status/diagnostic surfaces, and emit clear fallback/profile-rotation events without storing credential material.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and release inspection give a high-confidence verification path showing current main and v2026.5.20 lack resolved auth-profile fields in transcript parsing, usage cache/types, aggregates, and status output.

## Solution Assessment

Unclear for the exact public API shape, but native additive attribution at the normalized completed model-call usage seam is the maintainable direction. External per-request tools and broader usage-ledger requests are useful context, not replacements for OpenClaw's resolved auth-profile awareness.
