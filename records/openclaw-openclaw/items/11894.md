## Summary

Keep open. Current main has transcript-backed token and cost summaries plus auth-profile routing, but completed-call usage remains keyed by provider/model/session and is not attributed to the resolved auth profile that paid for the call; the linked burn0 item is only an external cost workaround.

## What This Changes

Requested native per-call cost tracking with attribution to the resolved auth profile used for each model call.

## Best Possible Solution

Record safe resolved authProfileId metadata for completed model calls, aggregate cost by auth profile, and expose additive transcript, Gateway, status, and UI fallback/rotation signals without persisting secrets.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source inspection: current usage parsing, Gateway aggregates, status output, and docs expose provider/model/session cost data and auth routing, but not resolved auth-profile attribution for completed calls.

## Solution Assessment

Unclear for the exact API and UI shape, but native runtime/transcript metadata is the right product boundary. A network-layer workaround such as burn0 can help with request cost, but it cannot reliably attribute OpenClaw's selected auth profile.
