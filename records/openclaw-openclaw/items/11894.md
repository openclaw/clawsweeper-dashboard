## Summary

Keep open: current main still separates completed-call usage/cost accounting from resolved auth-profile routing, so native per-call profile attribution and per-profile cost summaries remain missing.

## What This Changes

Request native per-call cost tracking with the resolved auth profile used for each model call, especially when auth-profile fallback pays for a call.

## Best Possible Solution

Add one additive completed-call attribution contract that records safe resolved authProfileId plus cost metadata and feeds transcript, status, gateway/API, UI, diagnostics, and plugin views without credential material.

## Reproduction Assessment

Yes. Source inspection shows completed-call usage/cost data and resolved auth-profile routing are separate facts today, and the accounting contracts do not join them; no live provider fallback run is needed to prove the feature gap.

## Solution Assessment

No current implementation fully solves it. The best path is a single safe completed-call attribution contract before deriving transcript, status, gateway/API, UI, hook, or event views; burn0 remains only an interim external cost workaround.
