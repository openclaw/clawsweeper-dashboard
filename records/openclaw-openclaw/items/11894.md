## Summary

Keep open. Current main has substantial adjacent usage/cost summaries, diagnostic model usage events, auth-profile routing, and fallback logging, but it still does not expose completed model-call cost or aggregation by the resolved auth profile that paid for the call.

## What This Changes

The issue requests per-call cost tracking with attribution to the resolved auth profile used for each model call.

## Best Possible Solution

Record safe resolved authProfileId metadata at completed model-call usage time, aggregate cost by auth profile, and expose additive transcript, Gateway/status/UI, and fallback-event diagnostics without storing credential material.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The high-confidence verification path is source inspection showing current main exposes provider/model/session usage and auth routing, but not completed-call resolved auth-profile attribution.

## Solution Assessment

Unclear for the exact public API and UI shape, but native metadata at the normalized model-call usage seam is the maintainable direction. Network-layer burn0-style tracking can help with per-request costs, but it cannot reliably attribute OpenClaw's selected auth profile.
