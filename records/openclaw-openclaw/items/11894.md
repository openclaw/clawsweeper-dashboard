## Summary

Current main provides per-call token/cost data and auth-profile fallback routing, but its completed-call usage pipeline still does not retain or aggregate the resolved profile that paid for each call. The burn0 suggestion is useful for immediate per-request cost visibility, while https://github.com/openclaw/openclaw/issues/13219 overlaps on usage reporting, but neither covers native profile attribution. Likely related people: @steipete and Vincent Koc, based on recent usage and auth-routing history.

## What This Changes

Requests per-model-call cost reporting tied to the resolved authentication profile, including explicit fallback attribution.

## Best Possible Solution

Carry a non-secret resolved profile identifier and closed selection-reason code into the canonical completed model-call usage record, then derive redacted transcript diagnostics, fallback events, and per-profile cost totals from that single record.

## Reproduction Assessment

Yes, at high confidence from current source: parsed completed-call usage and gateway aggregate contracts have no resolved auth-profile dimension. This is a missing capability rather than a failing runtime path.

## Solution Assessment

No, the three proposed options should not become independent implementations; the narrowest maintainable solution is one canonical completed-call attribution record from which summaries and events are derived.
