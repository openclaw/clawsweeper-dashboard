## Summary

Keep open. Current main has transcript-backed cost summaries and auth-profile rotation, but completed model-call usage is still parsed, exposed, and aggregated by provider/model/session rather than by the resolved auth profile that paid for each call.

## What This Changes

Request per-call cost metadata, resolved auth-profile attribution, per-profile cost summaries, and fallback/profile-rotation events for model calls.

## Best Possible Solution

Record safe resolved-auth-profile metadata when each model call completes, expose and aggregate it through transcript usage/status diagnostics, and emit a clear fallback/profile-rotation event without leaking secret material.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source inspection: current transcript parsing, usage aggregation, Gateway usage types, status output, and docs expose costs by provider/model/session but not by resolved auth profile.

## Solution Assessment

Unclear for the exact UI/API shape, but native runtime metadata is the right product boundary. A network-layer workaround like burn0/#55379 can help with request cost, but it cannot reliably attribute OpenClaw's selected auth profile.
