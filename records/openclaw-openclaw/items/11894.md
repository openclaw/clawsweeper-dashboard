## Summary

Keep open. Current main has provider/model/session usage and cost summaries plus auth-profile routing, but completed-call usage metadata is still not attributed to the resolved auth profile that paid for the call; the linked burn0 discussion remains only an external cost workaround.

## What This Changes

Request native per-call cost metadata and resolved auth-profile attribution, including profile-keyed cost summaries and fallback/profile-rotation events.

## Best Possible Solution

Record safe resolved authProfileId metadata for each completed model call, aggregate cost by auth profile, and expose fallback/profile-rotation events through additive transcript, Gateway, status, and UI surfaces without persisting secrets.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source inspection: current usage parsing, Gateway aggregates, status output, and docs expose provider/model/session cost data but not resolved auth-profile attribution.

## Solution Assessment

Unclear for the exact API and UI shape, but native runtime/transcript metadata is the right product boundary. A network-layer workaround such as burn0 can help with request cost, but it cannot reliably attribute OpenClaw's selected auth profile.
