## Summary

Keep open. Current main has transcript-backed token/cost summaries and auth-profile routing, but completed model-call usage is still surfaced by provider/model/session rather than by the resolved auth profile that paid for each call; the burn0 discussion is useful as an external workaround, not a native attribution fix.

## What This Changes

Request per-call cost visibility plus attribution of which auth profile was used, especially when fallback profile rotation occurs.

## Best Possible Solution

Record safe resolved authProfileId metadata for completed model calls, aggregate cost by auth profile, and expose additive transcript, Gateway, status/UI, and fallback-event signals without storing credentials or secret material.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. The high-confidence verification path is source inspection: current main exposes provider/model/session usage and auth routing, but not completed-call resolved auth-profile attribution.

## Solution Assessment

Unclear for the exact API and UI shape, but native metadata at the normalized model-call usage seam is the maintainable direction. Network-layer burn0-style tracking can help with per-request cost, but cannot reliably attribute OpenClaw's selected auth profile.
