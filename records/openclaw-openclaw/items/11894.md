## Summary

Keep open: current main has transcript-backed cost summaries, auth-profile routing, and live per-turn usageState, but completed-call accounting still does not persist or aggregate the resolved auth profile that paid for each call. The burn0 workaround and merged usageState hook are useful adjacent work, but neither replaces native durable auth-profile attribution.

## What This Changes

The issue requests per-call cost tracking with attribution to the resolved auth profile, especially when fallback credentials are used.

## Best Possible Solution

A single canonical completed-call accounting contract should record redacted resolved-profile attribution and feed transcript, status, gateway, UI, and plugin views without credential material.

## Reproduction Assessment

Yes, at source level: current main keeps usage/cost data and auth-profile routing as separate facts, and the completed-call usage contracts do not join them. No live provider repro was run because this read-only review only needed source confirmation of a feature gap.

## Solution Assessment

No current implementation fully solves it. The maintainable path is one canonical completed-call attribution contract before deriving transcript, status, gateway, UI, hook, or event views.
