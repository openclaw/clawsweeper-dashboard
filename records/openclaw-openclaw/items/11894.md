## Summary

Keep open. Current main has usage and cost summaries plus auth-profile routing/fallback machinery, but completed calls are still exposed and aggregated by provider/model/session rather than by the resolved auth profile that paid for each call.

## What This Changes

Add native per-call cost tracking with resolved auth-profile attribution and clearer fallback/profile-rotation visibility.

## Best Possible Solution

Record the resolved authProfileId at model-call completion as safe metadata, expose it through transcript/usage/status or diagnostics, aggregate cost by auth profile, and emit a clear fallback/profile-rotation event.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source inspection: current transcript parsing, usage aggregation, gateway usage types, status output, and docs expose cost by provider/model/session but not by resolved auth profile.

## Solution Assessment

Unclear for the exact UI/API shape, but native runtime metadata is the right product boundary. A network-layer tool like burn0 can help with request cost, but it cannot reliably attribute OpenClaw's selected auth profile.
