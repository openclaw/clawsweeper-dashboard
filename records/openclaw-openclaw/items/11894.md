## Summary

Keep open. Current main has transcript-backed token/cost summaries, durable transcript-derived usage cache, and auth-profile rotation state, but it still does not record or aggregate completed model-call cost by the resolved auth profile that paid for the call.

## What This Changes

The issue requests native per-call cost tracking with resolved auth profile attribution, including fallback visibility and per-profile cost summaries.

## Best Possible Solution

Add safe, additive resolved authProfileId metadata for completed model calls, aggregate cost by profile in usage/status surfaces, and emit clear fallback/profile-rotation events without storing credential material.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The high-confidence verification path is source inspection showing current main lacks resolved auth-profile fields in transcript parsing, usage types, caches, aggregates, and status output.

## Solution Assessment

Unclear for the exact public API shape, but the maintainable direction is native additive attribution at the normalized model-call usage seam. burn0 can remain an external per-request cost workaround, but it cannot replace OpenClaw's resolved auth-profile awareness.
