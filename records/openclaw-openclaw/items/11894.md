## Summary

Keep open. Current main still records and aggregates usage by provider/model/session while auth-profile routing exists separately, so the requested resolved-profile attribution and per-profile cost accounting remain missing.

## What This Changes

Request native per-call cost tracking that records which auth profile paid for each model call and exposes per-profile/fallback attribution in transcript, status, or events.

## Best Possible Solution

Add privacy-safe resolved authProfileId metadata at the completed model-call usage seam, aggregate cost by auth profile in usage/status diagnostics, and emit explicit fallback/profile-rotation events without storing credential material.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and release inspection give a high-confidence verification path showing current main and v2026.5.22 lack resolved auth-profile fields in transcript parsing, usage cache/types, aggregates, and status output.

## Solution Assessment

Unclear for the exact public API shape, but native additive attribution at the normalized completed-call usage seam is the maintainable direction. External per-request tools and broader usage-ledger requests are useful context, not replacements for OpenClaw's resolved auth-profile awareness.
