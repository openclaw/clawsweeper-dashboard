## Summary

Keep open. Current main has transcript-backed token/cost summaries, Gateway usage aggregation, status cost display, and auth-profile routing, but completed model-call usage is still keyed by provider/model/session rather than by the resolved auth profile that paid for each call; the burn0 discussion remains useful workaround context, not a native attribution solution.

## What This Changes

The issue requests per-call model cost tracking with attribution to the resolved auth profile, especially when fallback profiles are used.

## Best Possible Solution

Record safe resolved authProfileId metadata at completed model-call usage time, aggregate cost by auth profile, and expose additive transcript, Gateway/status/UI, and fallback-event diagnostics without storing credential material.

## Reproduction Assessment

No, not as a bug reproduction; this is a feature request. The high-confidence verification path is source inspection showing current main exposes provider/model/session usage and auth routing, but not completed-call resolved auth-profile attribution.

## Solution Assessment

Unclear for the exact public API and UI shape, but native metadata at the normalized model-call usage seam is the maintainable direction. Network-layer burn0-style tracking can help with per-request cost, but it cannot reliably attribute OpenClaw's selected auth profile.
