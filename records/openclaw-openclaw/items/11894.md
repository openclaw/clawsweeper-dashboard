## Summary

Keep open: current main has transcript-backed usage/cost summaries, live per-turn usage state, and auth-profile routing/fallback mechanics, but completed-call accounting still does not persist or aggregate the resolved auth profile that paid for each call. The burn0 discussion is useful as an interim external cost workaround, not a native OpenClaw attribution fix.

## What This Changes

Request native per-call cost visibility that attributes completed model calls to the resolved auth profile, including fallback profile use and per-profile cost summaries or events.

## Best Possible Solution

Add an additive completed-call attribution contract that records safe resolved authProfileId plus cost metadata and feeds transcript, status, gateway/API, UI, diagnostics, and plugin views without credential material.

## Reproduction Assessment

Yes. At source level, current main keeps completed-call usage/cost data and resolved auth-profile routing as separate facts, and the accounting contracts do not join them; no live provider fallback run is needed to prove the feature gap.

## Solution Assessment

No current implementation fully solves it. The maintainable path is one canonical completed-call attribution contract before deriving transcript, status, gateway/API, UI, hook, or event views; burn0 remains only an interim external cost workaround.
