## Summary

Keep open. Current main has transcript-backed token/cost summaries and auth-profile rotation state, but it still does not record completed model-call usage by the resolved auth profile that paid for each call; the burn0 workaround discussed on the issue helps with per-request cost only and does not replace native OpenClaw attribution.

## What This Changes

Request per-call cost metadata plus resolved auth profile attribution and fallback/profile-rotation visibility in transcript, session_status, or diagnostic surfaces.

## Best Possible Solution

Add safe additive resolved authProfileId metadata for completed model calls, aggregate cost by auth profile in usage/status surfaces, and emit clear fallback/profile-rotation events without storing credential material.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The high-confidence verification path is source inspection showing current main lacks resolved auth-profile fields in transcript parsing, usage types, caches, aggregates, and status output.

## Solution Assessment

Unclear for the exact public API shape, but the maintainable direction is native additive attribution at the normalized model-call usage seam. burn0 can remain an external per-request cost workaround, but it cannot replace OpenClaw's resolved auth-profile awareness.
