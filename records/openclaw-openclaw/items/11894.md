## Summary

Keep open. Current main has transcript-backed usage/cost summaries and runtime auth-profile routing, but completed model-call accounting is still persisted and aggregated by provider/model/session rather than by the resolved auth profile that paid for each call; burn0 and https://github.com/openclaw/openclaw/issues/55379 remain external workaround context, not native OpenClaw attribution.

## What This Changes

Request privacy-safe per-call cost visibility with resolved auth-profile attribution and fallback/profile-rotation notification.

## Best Possible Solution

Add privacy-safe resolved authProfileId metadata for completed model calls, aggregate cost by auth profile in usage/status/diagnostic surfaces, and emit clear fallback/profile-rotation events without storing credential material.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives a high-confidence verification path showing current main lacks resolved auth-profile fields in transcript parsing, usage cache/types, aggregates, and status output.

## Solution Assessment

Unclear for the exact public API shape, but native additive attribution at the normalized completed model-call usage seam is the maintainable direction. External per-request tools and broader usage-ledger requests are useful context, not replacements for OpenClaw's resolved auth-profile awareness.
