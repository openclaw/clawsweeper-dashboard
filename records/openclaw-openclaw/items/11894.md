## Summary

Keep open. Current main has transcript-backed usage/cost summaries and auth-profile routing, but completed call usage is still parsed and aggregated without the resolved auth profile that paid for each call; the linked burn0 workaround covers network-level costs, not native OpenClaw profile attribution.

## What This Changes

The issue requests per-call cost tracking with attribution to the resolved auth profile, especially when fallback profiles are used.

## Best Possible Solution

Record safe resolved authProfileId metadata for each completed model call, propagate it through transcript usage/status diagnostics, aggregate costs by profile, and emit a clear profile-rotation/fallback event without exposing secrets.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source inspection: current usage parsing, Gateway aggregates, status output, and docs expose provider/model/session cost data but not resolved auth-profile attribution.

## Solution Assessment

Unclear for the exact UI/API shape, but native runtime/transcript metadata is the right product boundary. A network-layer workaround like burn0 can help with request cost, but it cannot reliably attribute OpenClaw's selected auth profile.
