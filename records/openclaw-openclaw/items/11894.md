## Summary

Keep open. Current main has transcript-backed token/cost summaries and auth-profile rotation, but completed model-call usage is still parsed, stored, and aggregated by provider/model/session rather than by the resolved auth profile that paid for the call.

## What This Changes

Request native per-call cost visibility with resolved auth profile attribution, per-profile cost summaries, and a fallback/profile-rotation event.

## Best Possible Solution

Record safe resolved-auth-profile metadata at model-call completion, expose it through transcript/usage/status or diagnostics, aggregate cost by auth profile, and emit a clear fallback/profile-rotation event.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source inspection: current transcript parsing, usage aggregation, gateway usage types, status output, and docs expose costs by provider/model/session but not by resolved auth profile.

## Solution Assessment

Unclear for the exact UI/API shape, but native runtime metadata is the right product boundary. A network-layer workaround like burn0 can help with request cost, but it cannot reliably attribute OpenClaw's selected auth profile.
