## Summary

Keep open: current main still has token/cost summaries, provider/model aggregation, live usage state, and auth-profile routing, but it does not persist or aggregate the resolved auth profile that paid for each completed model call. The burn0 discussion is useful interim context, not a native OpenClaw attribution fix.

## What This Changes

Requested native per-call cost tracking with resolved auth profile attribution, profile-level cost summaries, and fallback/profile-rotation visibility.

## Best Possible Solution

Add one additive completed-call attribution contract that records safe resolved authProfileId plus cost metadata and derives transcript, status, gateway/API, UI, diagnostics, and plugin views from it without credential material.

## Reproduction Assessment

Yes. At source level, current main keeps completed-call usage/cost data and resolved auth-profile routing as separate facts, and the accounting contracts do not join them; no live provider fallback run is needed to prove the feature gap.

## Solution Assessment

No current implementation fully solves it. The maintainable solution is one canonical completed-call attribution contract before deriving transcript, status, gateway/API, UI, hook, or event views; burn0 remains only an interim external cost workaround.
