## Summary

Keep open. Current main has transcript-backed token/cost summaries and auth-profile routing, but completed model-call usage is still recorded and aggregated by provider/model/session rather than by the resolved auth profile that paid for each call; the linked burn0 discussion is a useful external cost workaround, not native OpenClaw profile attribution.

## What This Changes

The issue requests per-call cost tracking with attribution to the resolved auth profile, especially when automatic fallback/rotation moves a call from a free-tier profile to a paid profile.

## Best Possible Solution

Record safe resolved authProfileId metadata at completed model-call usage time, aggregate cost by auth profile, and expose additive transcript, Gateway usage/status/UI, and fallback-event diagnostics without storing credentials or secret material.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. The high-confidence verification path is source inspection: current main exposes provider/model/session usage and auth routing, but not completed-call resolved auth-profile attribution.

## Solution Assessment

Unclear for the exact API and UI shape, but native metadata at the normalized model-call usage seam is the maintainable direction. Network-layer burn0-style tracking can help with per-request cost, but cannot reliably attribute OpenClaw's selected auth profile.
