## Summary

Keep open. Current main has transcript-backed usage/cost summaries and auth-profile routing, but completed model-call cost is still parsed and aggregated by provider/model/session rather than by the resolved auth profile that paid for each call; the burn0 discussion remains a useful workaround, not native OpenClaw attribution.

## What This Changes

Request native per-call cost reporting that attributes each model call and fallback to the resolved auth profile.

## Best Possible Solution

Record safe resolved authProfileId metadata at completed model-call usage time, aggregate cost by auth profile, and expose additive transcript, Gateway/status/UI, and fallback-event diagnostics without credential material.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The high-confidence verification path is source inspection showing current main exposes provider/model/session usage and auth routing, but not completed-call resolved auth-profile attribution.

## Solution Assessment

Unclear for the exact public API and UI shape, but native safe metadata at the normalized model-call usage seam is the maintainable direction. Network-layer burn0-style tracking can help with per-request costs, but it cannot reliably attribute OpenClaw's selected auth profile.
