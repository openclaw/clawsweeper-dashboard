## Summary

Keep open. Current main has transcript-backed usage/cost summaries and auth-profile rotation state, but completed model-call accounting is still keyed by provider/model/session surfaces rather than the resolved auth profile that paid for each call; the burn0 discussion remains an interim per-request cost workaround, not a native attribution solution.

## What This Changes

Add native per-call cost tracking that attributes completed model calls to the resolved auth profile, including fallback/profile rotation visibility.

## Best Possible Solution

Add safe additive resolved authProfileId metadata for completed model calls, aggregate cost by auth profile in usage/status surfaces, and emit clear fallback/profile-rotation events without storing credential material.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives a high-confidence verification path showing current main lacks resolved auth-profile fields in transcript parsing, usage cache/types, aggregates, and status output.

## Solution Assessment

Unclear for the exact public API shape, but the maintainable direction is native additive attribution at the normalized completed model-call usage seam. burn0 can remain an external per-request cost workaround, but it cannot replace OpenClaw's resolved auth-profile awareness.
