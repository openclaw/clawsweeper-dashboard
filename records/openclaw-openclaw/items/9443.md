## Summary

Keep this issue open: OpenClaw now distributes the official Android app through Google Play, resolving the earlier lack of any official install path, but current main and release v2026.6.5 still do not publish the requested GitHub-release APK. The remaining work is a deliberate product and security decision about whether to support sideloading alongside Google Play.

## What This Changes

Request official prebuilt Android APK assets in OpenClaw releases with a documented installation path.

## Best Possible Solution

Keep Google Play as the documented official channel unless maintainers approve sideloading; if approved, publish one clearly supported signed APK flavor with checksums or attestations, an update and revocation policy, and matching installation documentation.

## Reproduction Assessment

Yes. Current main establishes that Google Play is the official install path and that release tooling is AAB-oriented, while the v2026.6.5 asset list concretely shows that no GitHub-release APK is published.

## Solution Assessment

No, not exactly as proposed. Uploading both debug and release APKs without an approved signing, flavor, provenance, and update policy is not the safest maintainable solution; maintainers should choose one supported signed sideload path first.
