## Summary

The issue remains necessary because OpenClaw’s official Google Play listing solves the earlier lack of an install path, but current main and release v2026.6.5 still do not publish official GitHub-release APKs. The remaining sideload channel is a maintainer product and supply-chain decision, and the open workflow proposal at https://github.com/openclaw/openclaw/pull/76296 should be evaluated rather than creating another implementation.

## What This Changes

Request official prebuilt Android APK downloads through OpenClaw releases.

## Best Possible Solution

Keep Google Play as the documented official channel unless maintainers approve sideloading; if approved, refocus the open PR onto the canonical signed Android release pipeline, publish one supported APK flavor with checksums/provenance, and document installation and updates at https://docs.openclaw.ai/platforms/android.

## Risks / Open Questions

- Official sideloading requires long-term ownership of signing-key custody, artifact provenance, revocation, and update expectations; publishing debug or ambiguously signed builds would weaken user trust.
- A GitHub APK channel can drift from Google Play in version, flavor, permissions, and compatibility unless both channels share a canonical release process.
- The existing open PR uploads temporary workflow artifacts for forks rather than official release assets and mixes in unrelated fork-maintenance automation, so it is not yet a direct landing path.
