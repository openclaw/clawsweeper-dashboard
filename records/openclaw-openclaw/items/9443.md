## Summary

Keep this open: current main now documents Google Play and can locally archive a signed third-party APK, but the latest GitHub release still does not publish an official standalone Android APK, and the open workflow candidate remains blocked on proof, signing, and automation/security concerns.

## What This Changes

Request official prebuilt Android APK downloads in GitHub releases for users who cannot or should not build the Android app locally.

## Best Possible Solution

Keep this issue open as the canonical standalone Android APK distribution tracker until maintainers either land an approved signing/upload path that publishes official GitHub Release or F-Droid APKs, or explicitly choose Google Play as the only official Android distribution channel and document that decision.

## Reproduction Assessment

Yes. This is reproducible from source and release inspection: current docs point to Google Play/source build paths, current tooling only archives a third-party APK locally, and v2026.6.11 release assets do not include an Android APK.

## Solution Assessment

No complete fix is on main yet. The best implementation should reuse the existing Android release signing/archive lane and publish only maintainer-approved signed artifacts, rather than adding a parallel ad hoc signing workflow.
