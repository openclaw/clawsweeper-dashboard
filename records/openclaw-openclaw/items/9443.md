## Summary

Keep this issue open as the canonical official sideload-distribution request. Google Play now solves the mainstream installation gap, but current releases still provide no APK, and supporting a second distribution channel requires an explicit signing, upgrade, and security decision.

## What This Changes

Publish an official signed Android APK through GitHub releases for users who need a sideload installation path.

## Best Possible Solution

Choose whether Google Play is the sole supported Android channel; if official sideloading is supported, publish one signed, upgrade-compatible third-party APK with checksums or attestations and clearly documented update and support expectations.

## Risks / Open Questions

- Supporting GitHub APKs alongside Google Play creates a second signed distribution and update path; certificate continuity, flavor choice, upgrade behavior, revocation, and support expectations remain unresolved.
- The app README still marks release hardening incomplete, so publishing a broadly downloadable APK could expand exposure before the Android owners consider the sideload build ready.
- Signing credentials and artifact provenance would move into release automation, requiring explicit secret custody, least-privilege permissions, checksum or attestation handling, and an owner-approved incident process.
