## Summary

Keep this issue open for an explicit Android release-policy decision. OpenClaw now distributes the official companion app through Google Play, but current main and the latest release still do not provide the requested GitHub-hosted APK; supporting sideloading would add a separate signing, provenance, update, and flavor-support commitment.

## What This Changes

Request official prebuilt Android APK assets through OpenClaw’s release process.

## Best Possible Solution

Choose and document one supported Android distribution policy: either keep Google Play as the sole official channel and state that GitHub APKs are intentionally unsupported, or publish a maintainer-signed APK for a clearly named flavor with checksums, provenance, update expectations, and an official installation link.

## Reproduction Assessment

Yes. Current docs, release tooling, workflow inventory, and the v2026.6.5 asset list establish with high confidence that an official Google Play app exists but no official GitHub APK is published.

## Solution Assessment

Unclear. Automating an APK upload is technically plausible, but it is not the best solution until maintainers choose the supported flavor, signing custody, sideload update policy, and whether Google Play should remain the sole official channel.
