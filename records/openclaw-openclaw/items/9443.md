## Summary

Keep open: current main now documents an official Google Play install path and has Android AAB release tooling, but it still does not publish official prebuilt Android APK assets through GitHub Releases. The open implementation candidate at https://github.com/openclaw/openclaw/pull/76296 is related, but APK distribution still needs maintainer decisions on signing, provenance, flavor scope, and release channel policy.

## What This Changes

Request to publish official prebuilt Android APK downloads, likely through GitHub Actions release automation, so users can install the Android companion app without building from source.

## Best Possible Solution

Keep this as the canonical Android APK distribution tracker until maintainers either approve signed/provenanced APK release assets and update https://docs.openclaw.ai/platforms/android, or explicitly decide that Google Play/source builds are the supported distribution path.

## Solution Assessment

Unclear until maintainers settle Android release policy. A generic APK upload workflow is not the best confirmed solution without signing-key custody, artifact provenance, flavor scope, and distribution-channel approval.
