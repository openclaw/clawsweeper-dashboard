## Summary

Current main still has the reported WhatsApp ack-before-process crash window: accepted inbound messages can be marked read before durable storage or successful handler delivery, so this should stay open as a narrow bundled plugin reliability bug.

## What This Changes

The issue requests a durable or post-processing acknowledgment boundary for inbound WhatsApp messages so a gateway crash cannot silently lose an already-acked message.

## Best Possible Solution

Fix the bundled WhatsApp plugin by moving the read receipt behind durable inbound storage or successful handler delivery and dedupe finalization, with focused tests and docs describing the crash-recovery boundary.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an accepted WhatsApp `notify` upsert reaches `sock.readMessages` before dedupe claim, debounced `onMessage`, and any durable replay boundary.

## Solution Assessment

Yes. The issue's persist-before-ack or delayed-ack direction is the narrow maintainable solution; a broader gateway state snapshot tool is complementary, not a replacement for fixing this plugin-owned ordering.

## Repair Work Prompt

Fix the bundled WhatsApp plugin reliability bug described in the current issue. Current main sends WhatsApp read receipts through `sock.readMessages` before inbound messages are durably stored or successfully delivered to `onMessage`, so a gateway crash can lose a message after WhatsApp considers it read. Keep the fix plugin-owned unless implementation proves a generic channel SDK seam is necessary. Move the ack boundary so accepted inbound messages are either persisted for replay before `readMessages` or `readMessages` is delayed until handler delivery succeeds and inbound dedupe is finalized. Update focused WhatsApp monitor/dedupe tests and user-facing WhatsApp docs if behavior or recovery guarantees change. Add a CHANGELOG entry because this is a user-facing reliability fix. Do not add root-owned dependencies, broad core migrations, or new config unless a maintainer approves that product choice.
