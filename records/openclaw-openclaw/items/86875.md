## Summary

This PR is a focused fix for a real embedded-runner prompt-context bug, but it is not merge-ready because the new runtime-context extractor can leak malformed hidden context into the model prompt and the latest scanner commit still lacks current-head real output proof.

## What This Changes

The PR changes the embedded runner prompt splitter and tests so `before_prompt_build` prepend/append context stays in the model prompt while marked internal runtime-context blocks remain hidden.

## Best Possible Solution

Keep the hook context prompt-local behavior, but make the marked runtime-context splitter match the existing fail-closed delimiter semantics and require current-head proof plus green relevant checks before merge.

## Reproduction Assessment

Yes from source inspection: current main moves `effectivePrompt - transcriptPrompt` into `runtimeContext`, while the docs/types define `prependContext` and `appendContext` as prompt-building hook output. The new malformed-marker leak is also source-reproducible from the PR head return path on `end === -1`; no local tests were run because this review is read-only.

## Solution Assessment

No, not yet: keeping hook context prompt-local is the right direction, but the new delimiter extraction needs to preserve the existing fail-closed hidden-context boundary. The safer solution is to reuse or mirror the existing internal-runtime-context parser and add focused regression coverage for unmatched and malformed standalone markers.
