## Summary

This PR should stay open for maintainer review: the patch is focused and has no blocking correctness findings, but it deliberately changes plugin hook context delivery in embedded prompts, so the compatibility-sensitive behavior needs maintainer acceptance before merge.

## What This Changes

The branch changes embedded-runner prompt splitting so `before_prompt_build` prepend/append context remains in the model prompt when a transcript prompt exists, while marked internal runtime-context blocks continue moving into hidden runtime context, with focused regression tests.

## Best Possible Solution

Land the focused embedded-runner fix if maintainers agree that `before_prompt_build` prepend/append output is model-local prompt context, while preserving the hidden internal runtime-context separation and regression coverage.

## Reproduction Assessment

Yes for source-level reproduction: current main splits `effectivePrompt` against `transcriptPrompt` without a separate model prompt, and the linked issue plus PR regression tests show hook context being diverted out of the model prompt path. I did not run tests because this was a read-only review.

## Solution Assessment

Yes, conditionally: the patch is a narrow implementation that keeps hook additions model-local while still stripping marked internal runtime-context blocks. The remaining question is maintainer acceptance of the compatibility-sensitive hook placement contract.
