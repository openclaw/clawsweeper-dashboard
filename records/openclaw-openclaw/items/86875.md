## Summary

## What This Changes

The PR changes embedded-runner prompt splitting so `before_prompt_build` prepend/append context remains in the model prompt while marked internal runtime-context blocks stay hidden.

## Best Possible Solution

Land this PR after maintainer acceptance of the prompt-local hook semantics, keeping hidden runtime context separated and the helper plus embedded-runner regressions in place.

## Reproduction Assessment

Yes, source-reproducible: current main applies hook context to `effectivePrompt` and then splits against the raw transcript prompt, moving the extra text into runtime context. I did not run tests because this review is read-only, but the source path and existing regression test expectation show the failure mode clearly.

## Solution Assessment

Yes, with maintainer compatibility sign-off: adding a separate model prompt for the split is a narrow fix that preserves hidden internal runtime context while restoring hook context to the model prompt. The safer alternative would only be needed if maintainers want to preserve the old hidden/system placement for existing plugins.
