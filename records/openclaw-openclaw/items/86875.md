## Best Possible Solution

Land the narrow splitter/caller change after CI and maintainer acceptance of the compatibility/security-boundary behavior: hook context should be prompt-local, while internal runtime-context blocks stay hidden and malformed blocks fail closed.

## Solution Assessment

Yes, the PR is a narrow maintainable fix: it preserves the hook-adjusted model prompt before hidden runtime annotation, then lets the splitter separate model-visible hook text from hidden internal context. The added fail-closed and multi-block tests cover the prior security concern without broad rewrites.
