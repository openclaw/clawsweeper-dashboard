## Summary

Keep open: current main and v2026.6.8 still block the requested agent-facing `plugins.bundledDiscovery` patch, and the linked open PR with closing syntax remains the implementation candidate.

## What This Changes

Allow the agent-facing Gateway config mutation guard to set `plugins.bundledDiscovery` only to `allowlist` so agents can apply doctor-recommended hardening.

## Best Possible Solution

Review and land the linked one-way guard PR, or replace it with an equivalent narrow fix that only permits `plugins.bundledDiscovery="allowlist"` while leaving broader trusted-agent config policy to https://github.com/openclaw/openclaw/issues/77439.

## Reproduction Assessment

Yes. Source inspection shows `plugins.bundledDiscovery` is a valid config key, but current main omits it from the agent-facing mutation allowlist, so a patch changing that path is rejected before the Gateway RPC.

## Solution Assessment

Yes, subject to maintainer security approval. The narrowest maintainable shape is a one-way guard at the existing mutation boundary; a broad plugin-config write allowlist belongs in the separate policy discussion.
