Allow the agent-facing gateway `config.patch` tool to set `plugins.bundledDiscovery` to `allowlist` while keeping loosening back to `compat` blocked.

## Best Possible Solution

Review and land the linked one-way guard PR, or replace it with an equivalent narrow fix that only permits the hardening value and keeps broader plugin discovery policy protected.

## Reproduction Assessment

Yes. Source inspection shows `plugins.bundledDiscovery` is a valid config key but absent from the agent-facing mutation allowlist, so a patch changing that path is rejected before the gateway RPC.

## Solution Assessment

Yes, subject to maintainer security-boundary approval. The narrow one-way exception is the maintainable shape; a broad plugin-config allowlist would be the wrong fix.
