## Best Possible Solution

Review and land the linked one-way guard PR, or replace it with an equivalent narrow fix that allows only the hardening transition and keeps broader plugin trust policy protected.

## Solution Assessment

Yes, the linked one-way exception is the narrow maintainable shape: add exactly `plugins.bundledDiscovery` and reject any value other than `"allowlist"`. A broader plugin-config allowlist would be the wrong security boundary.
