## Summary

Keep open: the underlying macOS A2UI wildcard-host problem is still source-reproducible on current main, but this branch is not merge-ready because it targets the old canvas host path, risks dropping scoped surface refresh behavior, maps IPv6 wildcard hosts to IPv4 loopback, and lacks real macOS proof.

## What This Changes

The PR injects a macOS node canvas host provider, rewrites wildcard A2UI hosts to loopback before appending the A2UI path, and adds Swift tests for injected host and IPv4 wildcard handling.

## Best Possible Solution

Normalize wildcard hosts inside the current scoped canvasSurfaceUrl/refreshCanvasSurfaceUrl flow, map 0.0.0.0 to 127.0.0.1 and :: to ::1, add focused Swift coverage, and prove canvas.a2ui.push on macOS.

## Reproduction Assessment

No high-confidence live reproduction was run in this read-only review. Source inspection shows current main still appends a raw scoped canvas surface URL, and the linked report provides the wildcard-host A2UI_HOST_UNAVAILABLE path.

## Solution Assessment

No. The current branch fixes the right class of bug, but the maintainable path is to apply host normalization to the current scoped plugin-surface resolver, preserve refresh, and handle IPv4 and IPv6 loopback separately.
