## Summary

Keep open. The branch targets a plausible macOS A2UI URL bug, but it is not mergeable as-is: it is built on the old `canvasHostUrl` path, leaves IPv6 wildcard handling incorrect, and has no after-fix macOS proof; current main has capability refresh but not this wildcard normalization.

## What This Changes

The PR injects a macOS A2UI canvas-host URL provider, rewrites wildcard hosts before appending the A2UI path, and adds Swift tests for provider injection plus IPv4 wildcard rewriting.

## Best Possible Solution

Rebase the normalization onto current scoped surface URL resolution, preserve capability refresh, map IPv4 and IPv6 wildcards correctly, add focused Swift coverage, and prove the A2UI flow on macOS.

## Reproduction Assessment

No live current-main reproduction was established. Source inspection shows a wildcard plugin surface URL would still be appended verbatim by macOS A2UI URL resolution, while the linked issue was closed under the newer capability-refresh path.

## Solution Assessment

No. The current PR needs to be rebased onto the `canvasSurfaceUrl`/`refreshCanvasSurfaceUrl` path and should map `::` to IPv6 loopback rather than forcing IPv4.
