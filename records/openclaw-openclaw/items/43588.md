## Summary

Keep this PR open, but do not merge it as-is. The requested Slack `allowBots: "mentions"` behavior is still missing on current `main`, while the branch patches the removed `src/slack` runtime path and misses active Slack plugin metadata/config surfaces.

## What This Changes

The PR widens Slack `allowBots` typing/schema/help text and adds mention-gated bot-message handling plus tests in the old `src/slack` runtime path.

## Best Possible Solution

Ship the feature through the active Slack plugin: define a tri-state Slack bot policy across config types, schema, plugin hints, generated metadata, docs, changelog, and `extensions/slack` runtime; then gate bot-authored messages on DM, explicit mention, or allowed implicit thread participation while preserving own-bot filtering, allowlists, owner-presence checks, and `thread.requireExplicitMention`.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current `main`: Slack config validation/types reject `allowBots: "mentions"`, and active Slack message preparation only has a boolean bot gate with no mention-only mode.

## Solution Assessment

No. The feature direction is sound, but this PR is not the best current implementation because it patches the retired `src/slack` path and misses active plugin metadata; the safer solution is a focused port or replacement PR against `extensions/slack` and the public config surfaces.
