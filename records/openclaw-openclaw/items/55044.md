## Summary

Keep this open. Current main still stores explicit `/stop` and natural-language aborts in the same `abortedLastRun` flag that main-session restart recovery consumes, so a running main session with a resumable transcript tail can still receive the synthetic restart-resume turn after an explicit stop.

## What This Changes

The issue asks OpenClaw to honor explicit stop/no-action instructions without later injecting an internal continuation turn.

## Best Possible Solution

Persist explicit user-abort intent separately from restart-interruption recovery state, and allow synthetic main-session resume only for sessions marked by restart recovery.

## Reproduction Assessment

Yes. A high-confidence source-level path exists: persist a running main session with `abortedLastRun: true` from `/stop` or an abort trigger, give it a user/tool-result transcript tail, and `recoverRestartAbortedMainSessions` will issue the synthetic resume `agent` call.

## Solution Assessment

Yes. The narrowest maintainable fix is to split durable restart-recovery state from explicit user-abort state, preserving restart recovery without weakening stop detection.
