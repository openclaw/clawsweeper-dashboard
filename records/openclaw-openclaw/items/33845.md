## Summary

Keep open. The Slack restart-survival bug is still real on current main, but this branch still has blocking filesystem-persistence hardening and test issues, and it now overlaps the merged state-store foundation plus open #58731, so maintainers should choose the canonical Slack implementation path.

## What This Changes

The PR adds JSON-backed persistence for the Slack thread participation cache, expands `sent-thread-cache` tests, and adds an API-barrel comment keeping only production cache functions exported.

## Best Possible Solution

Land one owner-approved Slack restart-survival implementation that preserves the fast in-memory participation check, stores account/channel/thread participation durably for the existing 24-hour policy, preferably uses `api.runtime.state.openKeyedStore` where feasible, and keeps restart, TTL, clear, corrupt-state, cap, split-module, and Slack prepare/dispatch behavior covered by focused tests.

## Reproduction Assessment

Yes. Record Slack participation in a thread, restart the gateway so the process-global cache is empty, then send a non-mention reply in that same thread; current main has no durable state for `prepare.ts` to classify it as `bot_thread_participant`.

## Solution Assessment

No. The PR targets the right user-visible bug, but the current branch is not the best merge shape while `openKeyedStore` exists and while the patch still has non-regular-file, failed-rename cleanup, and isolated-test defects.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Reject non-regular state files before hydration:** `extensions/slack/src/sent-thread-cache.ts:103-110`
  - body: `hydrateFromDisk()` calls `statSync` and then `readFileSync` on the configured state path without checking that the target is a regular file. A symlink, FIFO, or device at the persist path can block Slack message handling or read unintended local content before the JSON parse fallback; guard with `lstat`/`fstat` or move this persistence onto the keyed store before reading.
  - confidence: 0.87
- **[P3] Remove temp files when rename fails:** `extensions/slack/src/sent-thread-cache.ts:217`
  - body: After `writeFileSync` creates the UUID-named temp JSON file, `renameSync` can still throw and the scheduled persist path swallows the error. That leaves Slack thread identifiers in leftover `.tmp` files, so wrap the rename and unlink `tmpPath` on failure.
  - confidence: 0.82
- **[P2] Initialize tempDir before the directory-mode test:** `extensions/slack/src/sent-thread-cache.test.ts:311-312`
  - body: This test builds `path.join(tempDir, ...)` without calling `setup()`, but `tempDir` is only assigned inside that helper. Running this test, or the whole file when it reaches this case, throws before checking the directory mode; call `setup()` or create a fresh temp dir first.
  - confidence: 0.9
