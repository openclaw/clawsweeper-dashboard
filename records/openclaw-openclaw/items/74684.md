## Summary

Keep open. Current main still lacks first-class ACP PID/process correlation in the spawn result, runtime/status contracts, persisted session metadata, and operator docs, and open PR #74756 already references this issue with closing syntax.

## What This Changes

Expose process metadata or PID correlation for ACP sessions created by `sessions_spawn(runtime="acp")` so operators can inspect or terminate stuck child runs safely.

## Best Possible Solution

Land #74756 or a replacement that adds an additive ACP process-metadata contract through spawn, status, and persisted metadata without making raw PID guessing the only control path.

## Reproduction Assessment

Yes. Static inspection of current main is a high-confidence reproduction path because the accepted spawn result, tool boundary, runtime/status contracts, persisted metadata, and docs all omit first-class PID/process correlation.

## Solution Assessment

Unclear. Exposing process correlation is the right direction, but the maintainable fix should distinguish adapter and child-agent process metadata and define unavailable cases rather than only adding a raw optional PID.
