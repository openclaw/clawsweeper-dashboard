## Summary

Keep open. Current main still omits first-class ACP process metadata from the spawn result, runtime/status contracts, and persisted session metadata, and open PR #74756 is already linked as the implementation candidate.

## What This Changes

The issue asks OpenClaw to expose process metadata for ACP sessions spawned through `sessions_spawn` so operators can correlate or terminate running child processes.

## Best Possible Solution

Review and land #74756 or a replacement that adds additive, typed ACP process metadata through spawn, status, and stored metadata, with clear unavailable cases and no reliance on process-table guessing.

## Reproduction Assessment

Yes. Static inspection of current main is a high-confidence reproduction path because the response type, return paths, tool boundary, runtime/status contracts, and persisted metadata all omit process correlation fields.

## Solution Assessment

Unclear until #74756 is reviewed. The direction is right, but maintainers should confirm whether the public contract is a single `pid` or richer adapter/agent process metadata plus unavailable semantics.
