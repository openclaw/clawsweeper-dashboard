## Summary

Keep open. Current main still omits first-class ACP process metadata across the spawn result, tool return, runtime/status contract, and persisted ACP session metadata; #74756 is open and already references this issue with closing syntax, so the issue should stay open for that PR or a replacement.

## What This Changes

Request first-class PID/process correlation for ACP sessions spawned with `sessions_spawn(runtime="acp", ...)`.

## Best Possible Solution

Land #74756 or a replacement that adds an additive ACP process-metadata contract through ACPX, spawn/status surfaces, and persisted session metadata with clear unavailable semantics.

## Reproduction Assessment

Yes. Source inspection of current main is a high-confidence reproduction path because the ACP spawn result, tool boundary, runtime/status contracts, persisted metadata, and docs all omit first-class PID/process correlation.

## Solution Assessment

Unclear. Exposing process correlation is the right direction, but the maintainable fix should distinguish adapter and child-agent process metadata and define unavailable cases rather than only adding a raw optional PID.
