## Summary

Keep open. Current main still lacks the requested TUI reasoning-stream experience: backend/session validation accepts `stream`, but the TUI hides it in command help/completion and has no `thinking` agent-event renderer for live reasoning updates.

## What This Changes

The issue asks for OpenClaw TUI reasoning stream output that visibly updates like OpenAI/Grok thinking progress.

## Best Possible Solution

Land one coordinated reasoning-stream UX: emit `thinking` events for gateway/TUI clients without a channel-only callback dependency, render them in the chosen TUI surface behind reasoning/thinking visibility gates, and align help, docs, and regression tests.

## Reproduction Assessment

Yes, at source level. Current main accepts `reasoningLevel: "stream"`, but TUI command metadata omits `stream`, the no-argument usage still says `on|off`, and the TUI event path has no `thinking` stream renderer.

## Solution Assessment

No for an isolated cleanup patch. The maintainable fix needs a UX/privacy decision and coordination with the open reasoning-stream PRs before final TUI rendering and docs are wired in.
