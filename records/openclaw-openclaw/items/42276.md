## Summary

Current main and v2026.6.5 still do not provide the requested TUI experience: TUI help/completion omits `stream`, gateway thinking events remain callback-gated, and no current TUI handler renders those events as transient live reasoning. Related runtime PRs do not yet resolve the issue: https://github.com/openclaw/openclaw/pull/79687 is still open, while its proposed replacement https://github.com/openclaw/openclaw/pull/87481 contains substantial unrelated branch churn and is not presently a safe canonical landing path.

## What This Changes

Show live reasoning progress in the OpenClaw TUI through continuously updated transient output similar to OpenAI or Grok interfaces.

## Best Possible Solution

Land a clean, privacy-gated gateway thinking-event fix first, align TUI help and completion with the canonical `stream` level, then render one replaceable transient reasoning surface with explicit finalization cleanup and a maintainer-approved footer-versus-chat policy.

## Reproduction Assessment

Yes at source level. Current main and v2026.6.5 retain the callback-dependent gateway emission condition, TUI `on|off` metadata, and no dedicated TUI projection for `stream: "thinking"` events.

## Solution Assessment

No, an isolated footer line rewrite is not the best complete fix because it would leave the gateway emission and privacy contract fragmented. The narrow maintainable solution is one canonical, visibility-gated event path followed by a deliberate transient TUI renderer.
