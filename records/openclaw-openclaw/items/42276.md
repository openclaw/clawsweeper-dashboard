## Summary

Keep open. Current main has partial reasoning-stream plumbing, but the requested TUI experience is still missing: `stream` is documented but not advertised in TUI help/completion, thinking agent events are not rendered by the TUI, and the remaining UX overlaps open Gateway/Web/reasoning-stream PRs that need maintainer direction.

## What This Changes

Request OpenAI/Grok-like visible reasoning-stream updates in the OpenClaw TUI, with discussion of footer/status rendering and a persistent default.

## Best Possible Solution

Land one coordinated, visibility-gated reasoning-stream UX that advertises `stream` in TUI help/completion, emits and routes thinking events consistently, renders the chosen transient TUI surface, and treats persistent defaults as a deliberate config decision.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: `/reasoning stream` is documented but not advertised in TUI command metadata/usage, `handleAgentEvent` ignores thinking events, and embedded stream emission remains callback-gated.

## Solution Assessment

No for an isolated automated patch. The maintainable solution needs a maintainer UX/product decision and coordination with the active Gateway/Web/reasoning-stream work while preserving the thinking visibility gate.
