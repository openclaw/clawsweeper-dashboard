## Summary

Keep open: current main and v2026.6.6 still do not fully satisfy the TUI reasoning-stream request; `/reasoning stream` is documented and partly supported, but TUI help/completions still omit `stream`, TUI agent-event handling does not consume `stream: "thinking"`, and the open related PRs are not merged landing paths yet.

## What This Changes

The issue requests OpenAI/Grok-style live reasoning updates in the OpenClaw TUI, including clearer `/reasoning stream` behavior and possibly transient footer-style updates.

## Best Possible Solution

Land a maintainer-approved current-file fix that exposes `/reasoning stream` in TUI help/completion, consumes live `agent.stream="thinking"` under existing reasoning/thinking visibility gates, and reuses existing reasoning defaults rather than adding config.

## Reproduction Assessment

Yes, by source inspection: current main still advertises `/reasoning` as only on/off in TUI metadata and has no TUI `stream: "thinking"` event branch, while docs and session config already recognize `stream`. I did not run a live TUI session.

## Solution Assessment

Unclear as a product path: the metadata and event-consumption repairs are straightforward, but OpenAI/Grok-style line overwrite or footer rendering needs maintainer UI direction. The safer implementation should use existing `reasoningDefault` config rather than adding another persistent option.
