## Summary

Keep open: current main and the latest release still leave the reported TUI reasoning-stream UX incomplete. The core/session/docs contract accepts `stream`, but TUI help/completions still advertise only `on|off`, TUI agent-event handling has no `thinking` rendering path, and the open producer-side PR at https://github.com/openclaw/openclaw/pull/87481 is not merged and would not by itself settle the TUI rendering policy.

## What This Changes

The issue requests OpenAI/Grok-like live reasoning progress in OpenClaw TUI/WebChat, including visible updates instead of only final reasoning blocks.

## Best Possible Solution

Settle producer-side thinking-event delivery through https://github.com/openclaw/openclaw/pull/87481 or an equivalent narrow fix, then add a maintainer-approved TUI rendering path and align command help, docs, and tests while reusing existing `reasoningDefault` config instead of adding a duplicate knob unless maintainers choose one.

## Reproduction Assessment

Yes, by source inspection: current main accepts and documents `reasoningLevel: "stream"`, while TUI command metadata/help omit `stream`, TUI agent-event handling lacks a `thinking` branch, and callback-less gateway paths cannot emit live thinking from the current producer gate. I did not run a live TUI session in this read-only review.

## Solution Assessment

Unclear. The narrow runtime producer fix in https://github.com/openclaw/openclaw/pull/87481 looks like a partial prerequisite, but the best end-to-end solution still needs maintainer direction for whether TUI renders live thinking in chat, the footer/status line, or a line-overwrite style.
