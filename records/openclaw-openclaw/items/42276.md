## Summary

Keep open. Current main still does not deliver the requested TUI reasoning-stream experience: TUI help/completion omit `stream`, the TUI handler does not render `stream: "thinking"` events, and embedded reasoning stream emission is still callback-gated before the global agent event broadcast. The focused TUI PR was closed unmerged, while newer open PRs cover adjacent gateway/web/commentary pieces rather than the full TUI behavior.

## What This Changes

Request a TUI/OpenAI/Grok-style live reasoning stream for `/reasoning stream`, with later discussion of a persistent default.

## Best Possible Solution

Land one coordinated, visibility-gated reasoning-stream path that emits callback-less thinking events, advertises `stream` in TUI help/completion, and renders transient TUI thinking updates; track persistent defaults separately.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: TUI help/completions omit `stream`, `handleAgentEvent` ignores `stream: "thinking"`, and embedded reasoning stream emission is still callback-gated for callback-less clients.

## Solution Assessment

Yes for the narrow visibility-gated stream direction, but not as an isolated cleanup branch. The maintainable solution should coordinate with the active gateway/web/TUI streaming PRs and keep the persistent-default request separate.
