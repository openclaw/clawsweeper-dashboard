## Summary

Keep open. Current main still has partial reasoning-stream plumbing, but the reported TUI experience is not implemented: the docs advertise `/reasoning stream`, the TUI help/completion still omits it, TUI agent-event handling ignores `stream: "thinking"`, and the core emission path is still callback-gated for Gateway-style clients.

## What This Changes

The issue requests an OpenAI/Grok-like live reasoning stream in the TUI, including visible updating reasoning rather than only final or separate thinking output.

## Best Possible Solution

Land one coordinated reasoning-stream UX that advertises `stream` in TUI help/completion, preserves the thinking visibility gate, emits/routes thinking events consistently, renders the chosen transient TUI surface, and updates docs for the final behavior.

## Reproduction Assessment

Yes, source inspection gives a high-confidence current-main path: `/reasoning stream` is documented, but TUI command metadata/usage omit `stream`, TUI event handling ignores `stream: "thinking"`, and embedded reasoning emission is still callback-gated.

## Solution Assessment

No for an isolated automated patch. The maintainable solution needs a maintainer UX/product decision and coordination with the active Gateway/Web/reasoning-stream PRs while preserving the visibility gate.
