## Summary

Keep open: current main and v2026.6.10 still leave the requested TUI reasoning-stream UX incomplete. The core/session contract accepts `stream` and docs advertise it, but TUI command help/completions still show only `on|off`, the TUI agent-event handler has no `thinking` stream rendering branch, and the open producer-side PR only covers callback-less gateway emission rather than the TUI rendering policy.

## What This Changes

Request OpenAI/Grok-like live reasoning stream or line-rewrite behavior in OpenClaw TUI, with clearer `/reasoning stream` behavior and possibly WebChat visibility.

## Best Possible Solution

Settle the producer-side thinking-event delivery path, then add a maintainer-approved TUI `/reasoning stream` rendering policy with aligned help/completions, docs, and regression coverage.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main accepts and documents `reasoningLevel: "stream"`, while TUI help/usage omit `stream`, TUI agent-event handling lacks a `thinking` branch, and callback-less producer paths still cannot emit live thinking from the current gate. I did not run a live TUI visual proof in this read-only review.

## Solution Assessment

Unclear. The narrow producer fix in https://github.com/openclaw/openclaw/pull/87481 looks like a partial prerequisite, but the best end-to-end solution still needs maintainer direction for whether TUI renders live thinking in chat, the footer/status line, or an overwrite-style transient line.
