## Summary

Keep open. Current main still does not provide the requested TUI reasoning-stream experience: `/reasoning stream` is not advertised in TUI command help/completion, TUI agent events still ignore `stream: "thinking"`, and the embedded agent path remains callback-gated before broadcasting live thinking events for callback-less clients. The earlier focused TUI PR was closed unmerged, while newer open PRs only cover overlapping gateway/web/commentary pieces rather than the full TUI behavior.

## What This Changes

Add OpenAI/Grok-like live reasoning updates in the OpenClaw TUI for `/reasoning stream`, including a transient line/footer-style display instead of only static messages.

## Best Possible Solution

Land one coordinated, visibility-gated reasoning-stream path that emits callback-less thinking events, advertises `stream` in TUI help/completion, and renders transient TUI thinking updates; track persistent default behavior separately.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: TUI help/completions omit `stream`, `handleAgentEvent` ignores `stream: "thinking"`, and embedded reasoning stream emission is still callback-gated for callback-less clients.

## Solution Assessment

Yes for the narrow direction, but not as an isolated cleanup branch. The maintainable solution should coordinate with the active gateway/web/TUI streaming PRs and avoid bundling the separate persistent-default request.
