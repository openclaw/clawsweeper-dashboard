## Summary

Keep open: current main and v2026.6.10 still leave the requested TUI reasoning-stream UX incomplete. The backend/session contract accepts `stream` and docs advertise it, but TUI command help/completions still show only `on|off`, the TUI agent-event handler has no `thinking` stream rendering branch, and the open producer-side PR only covers callback-less gateway emission rather than the TUI rendering policy.

## What This Changes

The issue asks for OpenAI/Grok-style live reasoning updates in the OpenClaw TUI, including clearer `/reasoning stream` behavior and possibly status/footer-style line rewriting.

## Best Possible Solution

Settle the producer-side thinking-event delivery path, then add a maintainer-approved TUI rendering policy for `/reasoning stream` and align TUI help/completions, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection shows current main accepts and documents `reasoningLevel: "stream"`, while TUI command metadata/help omit `stream`, TUI agent-event handling lacks a `thinking` rendering branch, and callback-less producer paths still cannot emit live thinking from the current gate.

## Solution Assessment

Unclear. The narrow producer fix in https://github.com/openclaw/openclaw/pull/87481 looks like a partial prerequisite, but the best end-to-end solution still needs maintainer direction for whether TUI renders live thinking in chat, the footer/status line, or a line-overwrite style.
