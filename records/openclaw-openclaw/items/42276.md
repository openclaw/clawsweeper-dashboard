Keep open. Current main has partial reasoning-stream plumbing, but the TUI still does not expose or render the requested live reasoning stream consistently, and the linked implementation attempts are either closed unmerged or still open for review.

## What This Changes

The issue asks for an OpenAI/Grok-like live reasoning stream in the OpenClaw TUI, including updating or overwriting visible progress/status text while reasoning is in progress.

## Best Possible Solution

Land one coordinated reasoning-stream UX that preserves thinking visibility/privacy gates, renders `thinking` events in the chosen TUI surface, and aligns TUI docs, command help, and tests with the final behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows `/reasoning stream` is documented and accepted by session validation, while TUI command metadata/usage omit `stream` and `handleAgentEvent` has no `thinking` stream renderer.

## Solution Assessment

No for an isolated automated patch. The maintainable solution needs maintainer UX/privacy direction and coordination with the open reasoning-stream PRs before wiring the TUI rendering path.
