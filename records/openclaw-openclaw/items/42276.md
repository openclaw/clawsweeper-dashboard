## Summary

Keep open. Current main and v2026.5.22 still leave the TUI reasoning-stream experience incomplete: the backend accepts `reasoningLevel: "stream"`, but TUI command help/completions still advertise only `on|off`, current gateway chat deltas do not project `stream: "thinking"` into TUI chat rendering, and the TUI agent-event handler ignores thinking events.

## What This Changes

The issue requests OpenAI/Grok-style live reasoning updates in the OpenClaw TUI, including a clearer `/reasoning stream` experience and possibly footer-style progress rewriting.

## Best Possible Solution

Land one coordinated, privacy-gated TUI reasoning-stream UX: expose `stream` in TUI command metadata, render live thinking events in the chosen chat or footer surface, and align tests/docs with the runtime PR that owns gateway emission.

## Reproduction Assessment

Yes, at source level. Current main accepts and documents `reasoningLevel: "stream"`, but the TUI still omits `stream` from command metadata/usage and does not render `agent` thinking events.

## Solution Assessment

No, current main is only a partial foundation. The maintainable fix needs a product/privacy decision for chat versus footer rendering and should coordinate with the open gateway and Control UI reasoning-stream PRs.
