## Summary

Keep open. Current main and the latest release still do not provide the requested TUI reasoning-stream experience: backend/session paths accept and emit reasoning stream events, but the TUI still omits `stream` from command help/completions and has no `thinking` event renderer.

## What This Changes

The issue requests an OpenAI/Grok-like reasoning stream in the OpenClaw TUI that visibly updates reasoning/progress while an agent runs.

## Best Possible Solution

Land one coordinated TUI reasoning-stream UX: expose `stream` in TUI command metadata/usage, render `thinking` agent events in the chosen chat or footer surface behind visibility gates, and align docs plus regression tests.

## Reproduction Assessment

Yes, at source level. Current main accepts and emits `reasoningLevel: "stream"` thinking events, but the TUI command metadata, usage text, and agent-event handler do not expose or render that stream.

## Solution Assessment

No, current main is not enough. The maintainable solution needs a coordinated TUI UX/privacy decision and should account for the already-open reasoning-stream PRs before wiring final rendering behavior.
