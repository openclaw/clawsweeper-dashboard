## Summary

Keep open. Current main still does not provide the requested TUI reasoning-stream experience: `stream` is a valid/documented reasoning level, but TUI help/completion omit it, the TUI event handler ignores `stream: "thinking"`, and embedded thinking-event broadcast is still gated behind an optional callback.

## What This Changes

The issue asks for OpenAI/Grok-style live reasoning updates in the OpenClaw TUI, including transient line-rewriting or footer-style thinking display when `/reasoning stream` is enabled.

## Best Possible Solution

Land one coordinated, visibility-gated reasoning-stream path that advertises `stream` in TUI help/completion, emits gateway thinking events without requiring a channel callback, and renders transient TUI thinking updates; keep persistent-default follow-up separate unless maintainers expand scope.

## Reproduction Assessment

Yes. Source inspection on current main shows a high-confidence path: `stream` is accepted and documented, but the TUI omits it from command metadata/usage, ignores `stream: "thinking"`, and the embedded broadcast path is callback-gated.

## Solution Assessment

No for an isolated automated patch. The maintainable solution should coordinate the active reasoning-stream PRs and preserve the thinking visibility gate while maintainers choose the TUI rendering behavior.
