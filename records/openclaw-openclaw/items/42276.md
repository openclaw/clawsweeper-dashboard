## Summary

Keep open. Current main still has the TUI reasoning-stream gap: backend/session logic accepts `stream` and can emit `stream: "thinking"` events, but the TUI command metadata/help still advertises only `on|off` and the TUI agent-event handler ignores thinking events. The focused TUI PR #42312 was closed unmerged, while open PR #75587 is broader webhook/sink work rather than the TUI rendering fix.

## What This Changes

Request TUI-visible `/reasoning stream` behavior similar to OpenAI/Grok live reasoning updates, including clearer stream-mode behavior in OpenClaw TUI.

## Best Possible Solution

The TUI should advertise the documented `stream` reasoning level and render existing `thinking` stream events for the active run, with persistent defaults handled as a separate follow-up if maintainers want that behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: open TUI help/completions for `/reasoning` and only `on|off` is advertised; feed a known-run `AgentEvent` with `stream: "thinking"` and `handleAgentEvent` falls through without updating chat/status UI.

## Solution Assessment

Yes. The narrow maintainable fix is in the TUI client: expose `stream` in command metadata/usage and consume existing thinking stream events for known runs, instead of adding a broad core sink or persistent config change.
