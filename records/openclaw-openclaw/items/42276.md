## Summary

Keep open. Current main still has the TUI reasoning-stream gap: backend/session code accepts `stream` and can emit `stream: "thinking"` events, but TUI help/completions omit `stream` and the TUI event handler only renders tool/lifecycle agent events; the focused TUI PR #42312 was closed unmerged, while open PR #75587 is adjacent webhook/sink work rather than the TUI rendering fix.

## What This Changes

The issue requests OpenAI/Grok-like live reasoning updates in the OpenClaw TUI, especially `/reasoning stream` rendering with transient or overwritten lines.

## Best Possible Solution

Land a focused TUI repair that exposes `stream` in `/reasoning` help/completions, consumes existing `stream: "thinking"` events for known active runs, clears transient reasoning on terminal events, and aligns TUI/thinking docs plus changelog with the shipped behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: enter `/reasoning` in current TUI and the command help/completions only advertise `on|off`; feed a known-run `AgentEvent` with `stream: "thinking"` and `handleAgentEvent` falls through without rendering or status changes.

## Solution Assessment

Yes. A narrow TUI-client repair is the best path because the gateway/session validation and reasoning event shape already exist; broad core sink/webhook work is unnecessary for the user-visible TUI gap.
