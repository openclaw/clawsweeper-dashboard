## Summary

Keep open. Current main still does not provide the requested TUI reasoning-stream experience: `stream` is a valid reasoning level and documented for TUI, but the TUI command metadata/help still omit it, the TUI agent-event handler does not render `stream: "thinking"`, and embedded thinking-event broadcast is still callback-gated.

## What This Changes

The issue requests an OpenAI/Grok-like live reasoning stream in the OpenClaw TUI, with visible updating thinking text instead of only final or manually toggled thinking output.

## Best Possible Solution

Land one coordinated, visibility-gated reasoning-stream path that advertises `stream` in TUI help/completion, emits callback-less thinking events where appropriate, and renders transient TUI thinking updates; track persistent defaults separately.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: the documented `stream` reasoning value is not surfaced in TUI help/completion, `handleAgentEvent` ignores thinking events, and embedded reasoning-stream broadcast is gated behind an optional callback.

## Solution Assessment

No for an isolated automated patch. The maintainable fix should coordinate the active Gateway/web/TUI reasoning-stream work and preserve the thinking visibility gate while maintainers choose the TUI rendering behavior.
