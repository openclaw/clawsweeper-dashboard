## Summary

Keep open. Current main still leaves the TUI reasoning-stream experience incomplete: `/reasoning stream` is accepted by session validation and partly documented, but TUI help/completions still omit `stream`, the TUI event handler has no `thinking` renderer, and the agent emission path is still callback-gated for gateway/TUI-style clients.

## What This Changes

The issue requests an OpenAI/Grok-like live reasoning stream in the OpenClaw TUI, including visibly updated or overwritten progress lines while a run is thinking.

## Best Possible Solution

Land one coordinated reasoning-stream UX: emit `thinking` events for gateway/TUI clients without the channel-only callback dependency, render them in the chosen TUI surface behind existing visibility gates, and align command help, docs, and regression tests with that behavior.

## Reproduction Assessment

Yes, at source level. Current main accepts `reasoningLevel: "stream"`, but TUI command metadata omits `stream`, the TUI handler has no `thinking` renderer, and the agent stream path can prevent gateway-client thinking events without a channel callback.

## Solution Assessment

No for an isolated cleanup patch. The maintainable path needs a maintainer UX/privacy decision and coordination with the overlapping reasoning-stream PRs before wiring TUI rendering and docs as final behavior.
