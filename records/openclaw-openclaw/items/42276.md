## Summary

Keep open. Current main still has the TUI reasoning-stream gap: the backend/session surface accepts and can emit stream-mode reasoning data, but the TUI command surface omits `stream` and the TUI agent-event handler has no `stream: "thinking"` branch; the earlier focused PR #42312 was closed unmerged, and #75587 is adjacent core sink work rather than a TUI fix.

## What This Changes

Request live OpenAI/Grok-style reasoning updates in the OpenClaw TUI when `/reasoning stream` is enabled.

## Best Possible Solution

Land a focused TUI repair that exposes `stream` in `/reasoning` help/completions, consumes existing `stream: "thinking"` agent events for active runs, clears transient reasoning on terminal events, and aligns the TUI/thinking docs with the shipped behavior.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: current TUI help/completions only advertise `on|off`, and a known-run `AgentEvent` with `stream: "thinking"` falls through `handleAgentEvent` without a chat or status update.

## Solution Assessment

Yes. A narrow TUI-client repair is the best path because the gateway/session validation and agent event shape already exist; changing core stream semantics or landing a broad webhook sink is unnecessary for this issue.
