## Summary

Keep open. Current main still has only partial reasoning-stream support for this request: TUI docs advertise `stream`, session patching accepts it, but the TUI command help/completion omits it, TUI agent-event handling has no `stream: "thinking"` renderer, and gateway-originated reasoning emission is still blocked when no channel callback is present.

## What This Changes

Request to show `/reasoning stream` as live OpenAI/Grok-style rewritten thinking/progress in the OpenClaw TUI, with clearer command support and persistence expectations.

## Best Possible Solution

Land one coordinated reasoning-stream UX that advertises `stream` in TUI help/completion, preserves the thinking visibility gate, emits/routes thinking events consistently, renders the chosen transient TUI surface, and updates docs for the final behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path: `/reasoning stream` is documented and accepted by session validation, but TUI help/completion omit it, TUI event handling ignores `stream: "thinking"`, and gateway emission is callback-gated.

## Solution Assessment

No for an isolated automated patch. The maintainable solution needs a maintainer UX/product decision and coordination with the active Gateway/Web/reasoning-stream PRs while preserving the visibility gate.
