## Summary

Keep open. Current main still does not deliver the requested TUI reasoning-stream experience: `stream` is accepted by session validation, but TUI command help/completion still hides it, the TUI agent-event path has no `thinking` renderer, and the embedded reasoning stream still depends on a channel callback that gateway/TUI clients may not provide.

## What This Changes

Request OpenAI/Grok-style live reasoning/progress output in OpenClaw TUI when `/reasoning stream` is enabled.

## Best Possible Solution

Land one coordinated reasoning-stream UX: emit `thinking` events for gateway/TUI clients without the channel-only callback dependency, render them in the chosen TUI surface behind visibility gates, and align command help, docs, and regression tests.

## Reproduction Assessment

Yes, at source level. Current main accepts `reasoningLevel: "stream"`, but TUI command metadata omits `stream`, the TUI event handler lacks a `thinking` renderer, and the agent stream path can still be blocked by the missing channel callback.

## Solution Assessment

No for an isolated cleanup patch. The best path needs a maintainer UX/privacy decision and coordination with overlapping reasoning-stream PRs before final TUI rendering and docs are wired in.
