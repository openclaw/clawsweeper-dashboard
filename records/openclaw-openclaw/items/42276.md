## Summary

Keep open. Current main still does not deliver the documented `/reasoning stream` experience in TUI: the command UI advertises only `on|off`, TUI agent events ignore `stream: "thinking"`, and the gateway/embedded agent path still has callback gating that prevents callback-less clients from receiving live thinking events.

## What This Changes

The issue asks for OpenClaw TUI reasoning stream updates that visibly show live thinking progress, similar to OpenAI or Grok.

## Best Possible Solution

Land one coordinated fix that keeps reasoning visibility gated, emits callback-less thinking events for `stream` sessions, advertises `stream` in TUI help/completion, and renders live thinking updates in TUI; treat persistent defaults as a separate follow-up.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: TUI help/completions still omit `stream`, `handleAgentEvent` drops `stream: "thinking"`, and the agent stream is still callback-gated for callback-less clients.

## Solution Assessment

Yes for the narrow direction, but it should be coordinated with the active gateway/webchat PRs. The maintainable fix is to preserve reasoning visibility gates while wiring the existing `stream` contract through TUI rather than adding a new persistent default or broad sink feature here.
