## Summary

Keep open: current main still forces Codex native code-mode-only for the unrestricted Codex runtime path, and the newer restricted-allowlist escape hatch does not cover the reported default tool-using turns. The issue has strong source and live released/beta evidence, but the fix direction needs Codex harness owner judgment because the forced mode is documented and tested as intentional.

## What This Changes

The issue asks OpenClaw to stop or complete the forced Codex code-mode-only path so Codex-runtime tool turns can finish instead of stalling channel sessions.

## Best Possible Solution

Make Codex-runtime tool-using turns complete reliably, either by supporting the native code-mode tool contract end to end with live regression proof or by defaulting/overriding away from code-mode-only until that path is proven.

## Solution Assessment

Unclear. Disabling the flags is a validated mitigation in the issue comments, but the forced code-mode-only path is intentional current docs/tests policy, so the maintainable fix should be chosen by Codex harness owners and proven with a real tool-using turn.
