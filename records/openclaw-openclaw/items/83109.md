## Summary

Keep open: current main still defaults unrestricted Codex runtime turns to code-mode-only, and the reported live mitigations show disabling that path restores tool turns while no merged fix or shipped release proof closes the regression.

## What This Changes

The issue asks OpenClaw to stop forced Codex code-mode-only from stalling tool-using Codex runtime turns after upgrade.

## Best Possible Solution

Make Codex-runtime tool-using turns complete reliably, either by implementing the native code-mode dynamic-tool handoff end to end or by disabling forced code-mode-only/defaulting to an overrideable non-code-mode path until live proof and regression coverage exist.
