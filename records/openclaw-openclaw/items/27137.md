## Summary

Keep open: current main still lacks the requested unified ingress authorization contract, and the existing implementation remains split between shared low-level helpers and channel/event-specific authorization paths. The prior maintainer review also kept this as security architecture follow-up, so this is not a cleanup-close candidate.

## What This Changes

The issue requests a unified `authorizeIngress()` gate for cross-channel ingress authorization instead of per-channel and per-event authorization reimplementation.

## Best Possible Solution

Keep the RFC open until secops approves an additive security/SDK contract, then migrate representative bundled channel ingress paths to the shared decision point with adapter-specific identity normalization and regression tests.

## Solution Assessment
