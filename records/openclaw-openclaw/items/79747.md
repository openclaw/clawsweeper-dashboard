## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Keep the shared Control UI auth resolver:** `ui/src/ui/app-render.helpers.ts:59-64`
  - body: This helper now falls back to raw settings/password values and no longer uses the current `resolveControlUiAuthToken` path. That drops `hello.auth.deviceToken` support for paired/Tailscale-style Control UI sessions and loses the CR/LF header sanitization currently applied before authenticated media requests.
  - confidence: 0.86
- **[P2] Preserve queued session-switch state:** `ui/src/ui/app-render.helpers.ts:99`
  - body: The PR clears `chatQueue` on every session switch. Current main saves queued follow-ups per session and restores them when switching back, so using a pinned slot while messages are queued would silently lose those queued messages and also skips the current input/session/avatar reset work on this path.
  - confidence: 0.84
- **[P2] Keep bounded optimistic history refreshes:** `ui/src/ui/controllers/chat.ts:194-199`
  - body: This history refresh path regresses to an unbounded-by-chars `limit: 200` request and direct replacement of `chatMessages`. Current main sends `maxChars`, records the returned `sessionId`, and preserves optimistic tail messages when history briefly lags, so this can reintroduce blank/dropped local messages after sends or switches.
  - confidence: 0.82
