## Summary

Keep open: the feature is recent and the supplied real UI proof is sufficient, but the PR still has a source-reproducible persistence bug in how cross-browser pinned preferences are hydrated.

## What This Changes

This PR adds a Control UI pinned-chats sidebar, local and gateway-backed pinned-session preferences, history snapshot caching for chat switching, protocol/session fields, docs, and tests.

## Best Possible Solution

Keep the pinned-chats direction, but hydrate the gateway-backed preference from an unfiltered main-session lookup before final maintainer UX and release acceptance.

## Reproduction Assessment

Not applicable for the feature request itself. Yes for the blocking finding: source inspection shows saved prefs are hydrated only from the filtered `sessions.list` result, while current defaults and gateway filtering can omit the main session row.

## Solution Assessment

No. The UI and protocol approach is plausible, but the persistence sync should use `sessions.describe` or an equivalent unfiltered lookup for the main preference row instead of depending on the sessions panel filters.
