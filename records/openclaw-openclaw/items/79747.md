## Summary

Keep this PR open: the pinned-chats feature direction is plausible and the supplied browser proof is sufficient, but the latest head still has a source-reproducible alias bug where a saved `main` pin is matched verbatim against canonical `agent:main:main` session rows and can render as missing.

## What This Changes

The PR adds a Control UI pinned-chats sidebar with local and gateway-backed preference sync, history snapshot caching, protocol/session-store fields, docs, and UI/gateway tests.

## Best Possible Solution

Keep the pinned-chats feature path, but normalize pinned session keys through the same main-session alias/default rules before rendering, active checks, and persistence.

## Reproduction Assessment

Yes for the blocking review finding: source inspection shows the PR hydrates `main` as a pinned key, then `resolvePinnedChatEntries` matches that raw key against canonical `sessions.list` rows. Reproduction is not applicable to the feature request itself.

## Solution Assessment

No. The UI/protocol approach is plausible, but the PR should canonicalize pinned preference keys before row matching and persistence instead of carrying alias strings verbatim through the render path.
