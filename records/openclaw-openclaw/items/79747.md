- **[P1] Move pinned patch params to the package protocol schema:** `src/gateway/protocol/schema/sessions.ts:172-176`
  - body: Current main validates `sessions.patch` from `packages/gateway-protocol`, but this branch adds the new fields under the old `src/gateway/protocol` schema path. After rebase, put these fields in the package schema and refresh generated clients or the gateway/client contract will drift.
  - confidence: 0.94
- **[P1] Use selected-agent scoping when creating pinned chats:** `ui/src/ui/app-render.helpers.ts:392-396`
  - body: For a selected global session on a non-default agent, this fallback sends `agentId: "main"`. Pinned parallel chat creation should use the same selected-session helper path as current New Chat so it creates and refreshes under the active agent.
  - confidence: 0.91
- **[P1] Route pinned renames through scoped session patching:** `ui/src/ui/app-render.helpers.ts:468`
  - body: This direct `sessions.patch` call omits the `agentId` that current `patchSession` adds for UI global sessions, so renaming a pinned global row can patch the default/main agent instead of the selected agent. Reuse or extend the scoped patch helper.
  - confidence: 0.88
- **[P1] Reuse the current chat message cache:** `ui/src/ui/controllers/chat.ts:287-295`
  - body: Current main already preserves per-session visible messages through `chatMessagesBySession`; adding a separate WeakMap snapshot cache creates a second source of truth for the same session history state after rebase. Fold pinned switching into the existing cache helpers instead.
  - confidence: 0.86
