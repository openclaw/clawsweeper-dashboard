## Summary

Keep open: current main still has fixed 30000ms completion-cache spawn timeouts, and this PR provides a focused configurable-timeout implementation with real Pi 5 proof. The patch still has a small docs correctness issue and overlaps with another open timeout-policy PR, so it should not be closed as implemented or duplicate cleanup.

## What This Changes

The PR adds `OPENCLAW_COMPLETION_CACHE_WRITE_TIMEOUT_MS`, a shared strict timeout resolver, update/doctor spawn-site usage, resolver tests, docs, and a changelog entry.

## Best Possible Solution

Land one coherent slow completion-cache policy after fixing the doctor wording; either keep this env-var resolver or consolidate it with the fixed-cap/progress approach in https://github.com/openclaw/openclaw/pull/72910.

## Reproduction Assessment

No high-confidence live timeout was reproduced in this read-only pass. Source inspection confirms both current-main spawn sites still use fixed 30000ms timeouts, and related PR context documents real slow environments exceeding that budget.

## Solution Assessment

Mostly yes: the shared strict resolver is a narrow maintainable operator knob while preserving default behavior. The docs/comment wording should not name `openclaw doctor completion`, and maintainers should resolve the overlap with the fixed-timeout/progress PR.
