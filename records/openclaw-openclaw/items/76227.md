The PR adds `OPENCLAW_COMPLETION_CACHE_WRITE_TIMEOUT_MS`, a shared strict timeout resolver, docs, tests, and uses it in update and doctor completion-cache regeneration.

## Best Possible Solution

If maintainers accept the operator knob, land the shared resolver, tests, docs, and changelog together after a fresh green run; otherwise close with an explicit product decision against adding this env var.

## Solution Assessment
