## Summary

Keep open: the core timeout knob is still useful and current main still hard-codes 30 seconds in both affected spawn paths, but the latest branch still carries two concrete merge blockers: unrelated generated baseline hash edits and user-facing docs that name a nonexistent doctor subcommand.

## Best Possible Solution

Land one canonical completion-cache timeout fix, preferably this shared strict resolver after dropping unrelated generated checksum churn and correcting the doctor wording, then close or supersede the overlapping timeout PR.

No, not for an actual timeout failure on current main in this review. Source inspection does give a high-confidence reproduction of the missing configurability: both current spawn sites still hard-code 30000 ms, and the PR body supplies after-fix terminal proof for the new env path.
