## What This Changes

The PR adds `OPENCLAW_COMPLETION_CACHE_WRITE_TIMEOUT_MS`, a shared strict timeout resolver, tests, docs, and changelog text for update and doctor completion-cache writes.

## Best Possible Solution

Land one completion-cache timeout policy; this shared strict env resolver is a good path if maintainers want operator-configurable budgets, otherwise supersede it with the fixed-timeout/progress PR.

## Solution Assessment
