## Summary

Keep open: the timeout override is a useful narrow change with sufficient Pi-class terminal proof, but the current head still includes unrelated generated baseline hash edits and documents a nonexistent `openclaw doctor completion` command.

## What This Changes

The PR adds an `OPENCLAW_COMPLETION_CACHE_WRITE_TIMEOUT_MS` resolver used by the update and doctor shell-completion cache spawn paths, plus docs, tests, changelog, and generated checksum changes.

## Best Possible Solution

Merge the shared strict timeout resolver after dropping unrelated generated hash churn and naming the actual `openclaw doctor` shell-completion check/cache path.

## Reproduction Assessment

No, not for an actual timeout failure on current main. Source inspection does show both spawn sites still hard-code 30000ms, and the PR body supplies after-fix terminal proof for the new env path.

## Solution Assessment

Mostly yes. A shared strict resolver used by both spawn sites is the narrow maintainable shape, but the generated hash churn and nonexistent doctor subcommand wording should be fixed before merge.
