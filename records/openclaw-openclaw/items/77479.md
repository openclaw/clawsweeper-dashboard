## Summary

Keep this PR open: the image-runtime change is aligned with the reported Bedrock `aws-sdk` bug, but the runner-side change still lets no-static-key execution fail before an AWS-backed provider can run.

## What This Changes

The PR changes Bedrock image and media-runner auth resolution to avoid forcing a static API key when provider auth mode is `aws-sdk`.

## Best Possible Solution

Land a narrow fix that allows `aws-sdk`/no-static-key Bedrock image execution and, if the runner surface is kept in scope, gives that path an explicit no-key execution mode instead of passing an empty key list into rotation.

## Reproduction Assessment

Yes. Current main has a high-confidence source reproduction for the linked image bug: `auth.mode === "aws-sdk"` with no `apiKey` reaches `requireApiKey` in `src/media-understanding/image.ts` and throws before pi-ai/AWS SDK credential resolution.

## Solution Assessment

No. The image carve-out matches the existing text-path behavior, but the runner-side change is incomplete because the rotation layer still requires at least one concrete key; the safer solution is an explicit aws-sdk/no-key execution path plus regression coverage using the real key collection behavior.
