## Summary

Keep open: the image-path direction matches current Bedrock aws-sdk behavior, but the media runner change still leaves aws-sdk/no-key execution with an empty rotation list, and the PR needs the required changelog entry.

## What This Changes

The PR lets Bedrock image and media-runner auth skip static API-key enforcement for aws-sdk mode and adds image/runner regression coverage.

## Best Possible Solution

Keep the image aws-sdk carve-out, repair the runner execution path so credentialless aws-sdk auth reaches the provider instead of empty key rotation, and add focused regression coverage plus a changelog entry.

## Reproduction Assessment

Yes, from source inspection: configure amazon-bedrock with auth mode aws-sdk and no static apiKey, then call the image tool; current main resolves missing apiKey and immediately calls requireApiKey, which throws before the AWS SDK credential chain can run.

## Solution Assessment

No, not as proposed: the image change is the right narrow direction, but the runner change must also bypass or adapt API-key rotation for aws-sdk/no-key execution instead of passing an empty key set.
