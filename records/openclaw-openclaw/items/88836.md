## What This Changes

Fix Microsoft Teams attachment replies so channel replies configured for threading stay in the originating thread instead of posting as top-level messages.

## Best Possible Solution

Review and land one narrow Teams plugin fix that forwards the existing thread root through SharePoint and OneDrive attachment sends, then close duplicate attempts after merge.

## Reproduction Assessment

Yes. Source inspection shows a threaded Teams channel file send reaches SharePoint or OneDrive attachment branches that bypass the existing thread-aware send path; I did not run a live Teams tenant reproduction in this read-only review.

## Solution Assessment

Yes. The best fix is the narrow owner-boundary change already attempted by linked PRs: reuse the existing threadActivityId delivery contract for attachment sends while preserving explicit top-level behavior.
