## Label Justifications

- `P2`: This is a clear, bounded Control UI usability bug with limited blast radius.
- `impact:other`: Users receive no visible explanation or in-card status when Debug snapshot refresh is unavailable or in progress.

## Summary

Current main still silently ignores disconnected snapshot refreshes and provides loading feedback only through the button label. The original upstream fix closed unmerged, while the identical open fork PR remains conflicting and is not a viable landing path without revision.

## What This Changes

Show in-card progress during Debug snapshot refresh and explain when refresh cannot run because the gateway is disconnected.

## Best Possible Solution

Pass actual gateway connection state into `renderDebug`, add localized loading and disconnected states inside the Snapshots card, preserve existing snapshot display behavior, and prove the result with focused render coverage plus a real Control UI recording or screenshot sequence.

## Reproduction Assessment

Yes at source level with high confidence: current main silently returns while disconnected and provides loading feedback only in the Refresh button. A live browser reproduction was not run during this read-only review.

## Solution Assessment

Yes in direction, but the best implementation should use real caller-owned connection state and localized copy rather than infer disconnection from empty snapshot payloads.
