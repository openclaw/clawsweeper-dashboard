## Summary

Close: current main and the v2026.5.3-beta.1 tag include the lightweight `sessions.list` row path that removes the reported per-row transcript/model/plugin enrichment from the hot path; v2026.5.2 lacks that path, which matches the report.

## What This Changes

Make `sessions.list` and Control UI session loading cheap enough on Slack-heavy long-lived installs that dashboard calls cannot starve the gateway or degrade Slack Socket Mode responsiveness.

## Best Possible Solution

Keep the lightweight `sessions.list` implementation on main/beta; if saturation persists after v2026.5.3-beta.1 or a later stable release, track a new narrower performance report with diagnostics from that build.

## Reproduction Assessment

Yes, at source level: the v2026.5.2 row builder still did per-row transcript usage fallback and enrichment work that can scale with accumulated sessions, matching the reported large-store CPU path. I did not run a live Slack workload in this read-only review.

## Solution Assessment

Yes: the current fix is in the owning gateway session-list path, makes bulk rows lightweight by default, preserves opt-in preview/title fields, bounds catalog waiting, and avoids adding a new config knob or Slack-specific policy.
