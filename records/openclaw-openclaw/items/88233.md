## Summary

Keeping this open: the related Xiaomi fixes address provider-side reasoning_content handling, but current main still has Feishu-specific paths that intentionally render reasoning previews when enabled and a separate final-text sanitizer that does not strip common thinking tags, matching the reported Feishu display leak as remaining work.

## What This Changes

The issue asks OpenClaw to prevent Xiaomi MiMo thinking or reasoning content from being rendered to Feishu users, including during streamed Feishu card updates.

## Best Possible Solution

Strip accidental thinking/reasoning blocks from normal Feishu-visible text while preserving explicit /reasoning stream previews, with regression tests for streamed card updates and final card/direct delivery.

## Reproduction Assessment

No high-confidence live reproduction was performed, but the reported Feishu behavior is source-reproducible enough to act on because current Feishu card streaming can render explicit reasoning previews and the final sanitizer path does not strip thinking tags.

## Solution Assessment

Yes, a narrow Feishu/display-layer sanitizer fix is the best direction; the merged Xiaomi provider fixes solve API/replay behavior but do not own Feishu card rendering or accidental final text exposure.
