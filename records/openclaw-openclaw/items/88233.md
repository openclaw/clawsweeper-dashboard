## Summary

Issue #88233 should stay open: current main and the latest release still have a Feishu-specific delivery path that can forward unflagged thinking-tag text in final/static replies even though streamed partial snapshots are sanitized and explicit reasoning previews are gated separately.

## What This Changes

The issue asks OpenClaw to stop Feishu replies from exposing internal model thinking/reasoning text to end users.

## Best Possible Solution

Sanitize Feishu final/static card and text delivery with the same strict reasoning-tag stripping used for partial snapshots, preserve opt-in reasoning previews, and validate with Feishu/Xiaomi regression tests plus live channel proof.

## Reproduction Assessment

Yes at source level, but not with a fresh live run: current main still sanitizes only streamed partial snapshots while the final/static Feishu delivery path can forward ordinary text containing reasoning tags, and reporters supplied Feishu/Xiaomi production repros.

## Solution Assessment

Yes, the best fix is in the Feishu rendering layer: provider-specific MiMo fixes such as #81589 address API/replay behavior, but this issue is about channel delivery sanitization while preserving explicit reasoning previews.
