## Summary

Keep open: current main and the latest release still have a Feishu final/static delivery path that can forward unflagged thinking-tag text, while only streamed partial snapshots are stripped and opt-in reasoning previews are gated separately.

## What This Changes

The issue reports Feishu exposing model thinking/reasoning text to users alongside the intended answer.

## Best Possible Solution

Sanitize Feishu final/static card and text delivery with the same strict reasoning-tag stripping used for partial snapshots, preserve opt-in reasoning previews, and validate with Feishu/Xiaomi regression tests plus live channel proof.

## Reproduction Assessment

Yes, source-level: current main strips reasoning tags from Feishu partial snapshots but not from ordinary final/static delivery text. Production Feishu/Xiaomi reports support the same path, but I did not run a live tenant repro.

## Solution Assessment

Yes for the narrow issue: the best fix belongs at the Feishu user-visible delivery boundary, not in Xiaomi provider replay alone. Provider fixes such as https://github.com/openclaw/openclaw/pull/81589 addressed API-level MiMo behavior but do not close this channel-rendering leak.
