## Summary

Keep open: current main and v2026.6.9 still leave Feishu final/static text and card delivery without the reasoning-tag sanitizer used for streamed partial snapshots, so the reported Feishu/Xiaomi reasoning exposure is not fixed by the merged Xiaomi provider work.

## What This Changes

Feishu should stop exposing internal thinking/reasoning text in user-visible replies while still delivering the intended final answer.

## Best Possible Solution

Apply the canonical assistant-visible/reasoning sanitizer to Feishu final/static card and text delivery, add focused dispatcher regression tests, and close only after Feishu/Xiaomi channel proof shows normal replies hide reasoning while /reasoning stream still works.

Yes at source level: current Feishu code sanitizes streamed partial snapshots but not the final/static delivery path, and the issue includes Feishu/Xiaomi production reproduction details. I did not run a fresh live Feishu tenant test.

## Solution Assessment

Yes, the best fix is Feishu delivery sanitization, preferably through the shared canonical visible-text sanitizer; provider-only Xiaomi fixes do not cover the channel rendering path.
