## Summary

Current main and the shipped v2026.5.2 package include Telegram's grammY runtime dependencies at the root package level, so the reported global npm install failure from 2026.4.29 is already fixed. Prior related reports #73140 and #76130 covered the same Telegram runtime-dependency class and were closed after the packaging/runtime path was repaired.

## What This Changes

The report asks for Telegram's grammY runtime dependency to be available after `npm install -g openclaw` so the bundled Telegram channel loads without manual npm installs.

## Best Possible Solution

Keep the shipped v2026.5.2 packaging model; users on 2026.4.29 should upgrade instead of manually installing `grammy` into the global package.

## Reproduction Assessment

Yes for the reported old release: a448042 has Telegram deps only in the plugin package, while current main and v2026.5.2 include them in the root install package. The current-main failure path no longer reproduces from source inspection.

## Solution Assessment

Yes. Mirroring the still-bundled Telegram runtime deps into the root published package is the narrow packaging fix for global npm installs, and the release metadata shows it has shipped.
