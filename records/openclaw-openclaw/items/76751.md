## Summary

Current main already carries Telegram's grammY runtime packages as root package dependencies, which is the dependency tree used by `npm install -g openclaw`, so the reported missing-module path is fixed on main even though I could not prove it has shipped in a tagged release yet.

## What This Changes

The issue asks for the published global npm package to include Telegram's `grammy` runtime dependency so the bundled Telegram channel loads after install.

## Best Possible Solution

Keep the root runtime dependency declaration and installed-package smoke coverage; users on the reported 2026.4.29 build should update once a release containing current main is available.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the reported failure class: Telegram imports `grammy` and `@grammyjs/runner` at runtime, so a package install missing those dependencies would fail exactly as reported; current main no longer has that missing dependency contract.

## Solution Assessment

Is this the best way to solve the issue? Yes: adding the grammY packages to root runtime dependencies is the narrow package-level fix for global npm installs, while the Telegram plugin keeps its owner-local dependency metadata.
