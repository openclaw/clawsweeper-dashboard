## What This Changes

The PR normalizes voice-call trusted-proxy IP comparisons in webhook URL reconstruction and adds Twilio/Plivo regression tests for IPv4-mapped remote addresses.

## Best Possible Solution

Land the focused PR after security-aware maintainer review, preserving the existing forwarded-header gates and leaving the linked issue open until the PR merges.

## Reproduction Assessment

Yes, source-level. Current main literal-matches `trustedProxyIPs.includes(remoteIP)`, so a configured `127.0.0.1` proxy fails when Node reports the same peer as `::ffff:127.0.0.1`; the PR adds Twilio and Plivo regression coverage for that shape.

## Solution Assessment

Yes. Normalizing both configured proxy IPs and the socket remote address inside `reconstructWebhookUrl` is the narrow fix, matches the existing media-stream invariant, and avoids adding a new config surface.
