## What This Changes

The PR normalizes voice-call trusted proxy IP comparisons for webhook URL reconstruction and adds Twilio/Plivo regression tests for IPv4-mapped loopback proxy addresses.

## Best Possible Solution

Voice-call webhook security should treat plain IPv4 and IPv4-mapped forms of configured trusted proxies equivalently while preserving the existing forwarded-header opt-in and Twilio/Plivo regression coverage.

## Reproduction Assessment

Yes, source-reproducible: current main compares trustedProxyIPs to remoteIP with literal includes, so trustedProxyIPs containing 127.0.0.1 misses remoteAddress ::ffff:127.0.0.1. I did not run a live provider callback in this read-only review.

## Solution Assessment

Yes, normalizing both configured proxy IPs and the request remote IP before the existing fromTrustedProxy gate is the narrow maintainable fix and matches the sibling media-stream path. It still needs maintainer acceptance because that gate controls forwarded host/proto trust.
