## What This Changes

This PR normalizes voice-call webhook trusted-proxy IP comparisons so IPv4-mapped socket addresses match plain IPv4 trusted proxy entries, and adds Twilio and Plivo regression tests.

## Best Possible Solution

Land the narrow normalization and regression tests once maintainers accept the security-boundary equivalence and required checks are green; avoid broadening this PR into CIDR support or new forwarding-header policy.

## Reproduction Assessment

Yes, source inspection gives a high-confidence repro: current main compares trustedProxyIPs to remoteIP literally, so the configured 127.0.0.1 case cannot match ::ffff:127.0.0.1. The PR's focused Twilio and Plivo tests encode that exact failing shape.

## Solution Assessment

Yes, this is a narrow maintainable fix: it applies the same normalization shape already used by the voice-call media-stream trusted-proxy path while leaving allowedHosts, trustForwardingHeaders, and trustedProxyIPs gating intact.
