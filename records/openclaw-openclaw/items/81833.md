## What This Changes

Adds an optional `allowConfigSizeDrop` flag to config-mode `skills.update`, forwards it to the config writer, and extends the skills update test coverage for enabled and default-disabled write options.

## Best Possible Solution

Merge a narrow admin-only `skills.update` size-drop opt-in that preserves the other writer guards, documents the new protocol field, and closes the linked issue after maintainer policy approval.

## Reproduction Assessment

Yes. Source inspection shows current main rejects `allowConfigSizeDrop` and writes skill config without `writeOptions`, and the PR body supplies real temp-config output showing the default reject and flagged success path.

## Solution Assessment

Mostly yes. Forwarding the flag through the schema, handler, and existing writer option is the narrow implementation, but the protocol docs should be updated and maintainers should confirm the admin-scope policy before merge.
