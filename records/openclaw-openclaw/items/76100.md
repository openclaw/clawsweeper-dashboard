## Summary

Current main has moved past the v2026.4.29 shared runtime-deps race: bundled gateway startup now prefers built plugin artifacts, the old plugin-runtime-deps roots are documented and cleaned up as legacy state, and the fix is present on main but not yet in a release tag.

## What This Changes

The issue asks OpenClaw to prevent parallel bundled plugin runtime dependency staging from reusing an incomplete shared node_modules tree that leaves Telegram providers missing json5 on cold startup.

## Best Possible Solution

Keep the current main approach: bundled gateway startup should load shipped built artifacts, and legacy runtime-deps state should remain cleanup-only until the fix ships in the next release.

## Reproduction Assessment

Yes. The report gives a concrete v2026.4.29 cold-cache reproduction by deleting the runtime-deps root and starting the gateway; current-main source review shows that startup path no longer creates or reuses that shared runtime-deps tree.

## Solution Assessment

Yes. Preferring built bundled artifacts and removing startup dependency repair is the narrow maintainable fix because it eliminates the partial shared node_modules observation path instead of adding a json5-specific hotfix.
