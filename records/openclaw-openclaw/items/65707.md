## What This Changes

The PR adds prompt-mode plumbing to embedded context-pruning setup and caps cache-ttl `keepLastAssistants` at 1 for minimal prompt mode, with factory tests for minimal and full modes.

## Best Possible Solution

Land a rebased branch that derives pruning policy from the same effective prompt mode used to build the prompt, preserves explicit `keepLastAssistants: 0`, covers subagent, cron, and `toolsAllow` minimal paths, and includes redacted real-behavior proof.

## Reproduction Assessment

Yes for the PR defect by source inspection: the branch passes raw `promptMode` into pruning setup while the runner separately forces `effectivePromptMode` to `minimal` for non-empty `toolsAllow`. I did not run tests because this is a read-only review.

## Solution Assessment

No. The direction is narrow, but the best fix is to pass `effectivePromptMode` or an equivalent prepared signal into pruning setup and add coverage for the `toolsAllow` path before merge.
