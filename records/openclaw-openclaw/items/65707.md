Full review comments:

- **[P1] Preserve explicit pruning retention settings:** `src/agents/pi-embedded-runner/extensions.ts:55`
  - body: This cap runs after `computeEffectiveSettings(raw)`, so a user who set `contextPruning.keepLastAssistants` to 2 or 3 would still be forced down to 1 in minimal mode. The session-pruning docs say explicit pruning values are not overridden, and changing that setting can remove recent tool-result context from existing setups. Please only cap the default when the user did not set `keepLastAssistants`, or get an explicit product decision and docs/tests for the new override behavior.
  - confidence: 0.9
- **[P1] Gate pruning on autonomous runs, not all minimal prompts:** `src/agents/pi-embedded-runner/extensions.ts:55`
  - body: `promptMode` is broader than the subagent/cron case described here. Current main also forces `effectivePromptMode` to `"minimal"` when `toolsAllow` is set for embedded attempts, so this would cap `keepLastAssistants` to 1 for constrained-tool regular runs too and can drop recent tool-result context outside the intended autonomous path. Please gate this on an explicit subagent/cron/autonomous-run signal or prove the broader minimal-mode policy is intended.
  - confidence: 0.84
