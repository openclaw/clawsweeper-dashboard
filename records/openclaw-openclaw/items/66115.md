## Summary

Keep open. Current main still implements and documents `/v1/models` as an agent-target list, while the provider-model catalog exists over Gateway RPC `models.list`; exposing that catalog through HTTP needs maintainer API, auth, and compatibility decisions.

## What This Changes

Expose actual provider model IDs through OpenAI-compatible HTTP model discovery so third-party clients can populate model pickers without reading OpenClaw config files.

## Best Possible Solution

Preserve the documented agent-first `/v1/models` contract and add an additive read-only HTTP provider-model discovery surface backed by `models.list` once maintainers choose the public API and auth semantics.

## Reproduction Assessment

Yes, for the current behavior. Source inspection and e2e coverage show `/v1/models` returns OpenClaw agent targets, while provider-model discovery is available through the separate `models.list` RPC path.

## Solution Assessment

Unclear. Replacing `/v1/models` is not clearly the best fix because docs and tests intentionally make it agent-first; an additive endpoint or explicit opt-in HTTP view is the safer direction after maintainer API review.
