## What This Changes

The branch changes transport stream finalization to throw `output.errorMessage`, adds tests, and also normalizes cron `agentId`/`sessionKey` string `"null"` inputs to `null` with tests.

## Best Possible Solution

Land a scoped transport fix that uses a trimmed non-empty error fallback, proves the intended provider failover/auth behavior, and either splits the cron string-null fix or covers it with explicit review and proof.

## Reproduction Assessment

Yes, by source inspection. Current main stores `output.errorMessage` in the failure helper but the finalizer replaces errored or aborted stream output with a generic throw; the cron normalizer also currently preserves nonblank strings such as `"null"`.

## Solution Assessment

Partly. Threading a non-empty provider error through the finalizer is the right direction, but the patch should retain a blank-message fallback and the cron sentinel behavior should be split or explicitly proved and owned.
