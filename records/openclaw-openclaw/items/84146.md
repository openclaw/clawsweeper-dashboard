## What This Changes

The branch changes transport stream finalization to throw `output.errorMessage` for errored or aborted streams, adds unit coverage, and also normalizes cron `agentId`/`sessionKey` string `"null"` values to `null` with tests.

## Best Possible Solution

Land a scoped transport fix with real provider failure proof, keep the generic fallback for blank or missing messages, and handle the cron string-null sentinel either in its own PR or with explicit coverage for the linked cron issue and compatibility decision.
