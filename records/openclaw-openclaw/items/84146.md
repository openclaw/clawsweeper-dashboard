The PR changes transport stream finalization to throw `output.errorMessage` on errored or aborted streams, adds transport-stream tests, and also normalizes cron `agentId`/`sessionKey` string `"null"` inputs to `null` with tests.

## Best Possible Solution

Land a scoped transport fix that throws non-empty provider error text with a generic fallback, split or explicitly review the cron string-null fix, and attach real provider/cron proof before merge.

## Reproduction Assessment

## Solution Assessment

Partly. Threading `errorMessage` is the right narrow direction, but the patch should keep a non-empty generic fallback and either split or fully prove the cron sentinel behavior before merge.
