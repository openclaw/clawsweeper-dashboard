## Summary

Keep open. The agents fix is source-backed, but the branch is not merge-ready because it also carries the cron commit from https://github.com/openclaw/openclaw/pull/83618, which has an `agentId` compatibility regression, and the proof is still unit-test-only.

## What This Changes

The branch changes transport stream finalization to throw `output.errorMessage` with a generic fallback, adds unit coverage, and also changes cron normalization/tests so string `"null"` clears `agentId` and `sessionKey`.

## Best Possible Solution

Keep the transport error-message fix, but split or repair the cron string-null change so only the intended nullable field behavior changes and real after-fix runtime proof is supplied before merge.

## Reproduction Assessment

Yes by source inspection. Current main hardcodes the transport finalization error despite `failTransportStream` carrying `output.errorMessage`, and the cron path currently preserves non-empty string values including `"null"`.

## Solution Assessment

No as written. The one-line transport fix is the narrow solution, but the bundled cron commit should be split or fixed because applying the string-null sentinel to `agentId` conflicts with the current agent-id contract.

## Review Findings
