## Summary

Keep open: current main still admits non-Slack channel tool/progress delivery, Matrix rooms are classified as `ChatType: "channel"`, and https://github.com/openclaw/openclaw/pull/85069 is an open closing PR for this issue.

## What This Changes

The issue reports that default automatic-delivery Matrix channel rooms receive internal tool/progress events as visible chat messages and asks to suppress them by default.

## Best Possible Solution

Land a shared dispatch fix that suppresses default non-forum channel progress/tool summaries while preserving explicit opt-ins, room-event suppression, Slack non-direct behavior, and media or approval pass-through.

## Reproduction Assessment

Yes for source-level reproduction: Matrix non-DM rooms set `ChatType: "channel"`, and the current shared dispatch predicate allows verbose/tool summaries for that value. I did not run a live Matrix room, so this is source-reproducible rather than live reproduced.

## Solution Assessment

Yes on direction, with maintainer review for the exact exception policy. The maintainable fix belongs in the shared dispatch gate with regression coverage, not in a Matrix-specific workaround or a new config knob.
