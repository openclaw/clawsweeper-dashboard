## Label Justifications

- `P1`: The report describes a reproduced privacy-sensitive regression that can expose internal tool/progress details in a real multi-person Matrix room.
- `impact:security`: Visible tool/progress summaries can include file paths, command names, search terms, and operational context in a shared room.
- `impact:message-loss`: The issue is a channel delivery bug where internal progress messages are routed to a visible chat surface that should stay quiet by default.

## Summary

Keep open: current main and the latest shipped release still match the reported source path, with Matrix rooms classified as `ChatType: "channel"` and the shared verbose/tool-summary gate suppressing Slack non-direct and `group` but not default non-forum `channel` turns.

## What This Changes

Suppress internal verbose tool/progress summaries for default automatic-delivery Matrix channel rooms while preserving explicit `message_tool_only` verbose delivery.

## Best Possible Solution

Move the policy into the shared auto-reply dispatch gate so default non-forum channel rooms suppress internal tool/progress text, while explicit verbose/message-tool-only and forum or authorized-command exceptions are covered by focused tests.

## Reproduction Assessment

Yes for source-level reproduction: Matrix emits `ChatType: "channel"`, and the current shared dispatch predicate allows verbose/tool summaries for that value. This review did not run a live Matrix room, so the structured status is source-reproducible rather than locally reproduced.

## Solution Assessment

Yes on direction, with owner review for the exact exception policy. The maintainable fix belongs in shared dispatch with regression coverage, not in a Matrix-specific workaround or a new config knob.
