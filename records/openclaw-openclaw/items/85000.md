## Summary

Keep open: current main still has the reported default-channel verbose/tool-summary gate behavior, and the open linked PR at https://github.com/openclaw/openclaw/pull/85069 is the right place to resolve this issue before it closes.

## What This Changes

Default-mode Matrix multi-person rooms can receive internal tool/progress events as visible chat messages, and the request is to suppress those messages by default while preserving explicit verbose/message-tool opt-ins.

## Best Possible Solution

Review and land the linked shared-dispatch fix if its opt-in exceptions are correct, then close this issue from the merged PR with Matrix proof and focused regression coverage.

## Reproduction Assessment

Yes for source-level reproduction: current main classifies Matrix rooms as `ChatType: "channel"`, and the shared dispatch predicate still allows verbose/tool-summary delivery for that value. I did not run a live Matrix room, so this is source-reproducible rather than locally reproduced.

## Solution Assessment

Yes on direction, with maintainer review for the exact shared-channel exception policy. The maintainable fix belongs in `src/auto-reply/reply/dispatch-from-config.ts` with focused regression tests, not in a Matrix-only workaround or a new config knob.
