## What This Changes

The issue requests suppressing internal verbose/tool progress messages on default-mode Matrix and other non-Slack channel rooms while preserving final replies and explicit opt-in progress paths.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/85069 or an equivalent shared dispatch fix that suppresses default non-forum channel progress while preserving final replies and explicit opt-in paths.

Yes for source-level reproduction: Matrix emits `ChatType: "channel"`, and current main's shared dispatch predicate allows verbose/tool summaries for that value. I did not run a live Matrix room in this read-only review, so this is source-reproducible rather than locally reproduced.

## Solution Assessment

Yes on direction, but the exact exception policy needs maintainer review. The maintainable fix belongs in the shared auto-reply dispatch gate with focused regression tests, not in a Matrix-only workaround or a new config knob.
