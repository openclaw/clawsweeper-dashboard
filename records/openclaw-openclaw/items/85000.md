## Label Justifications

- `P1`: The issue reports a privacy-sensitive regression that can expose internal tool/progress details in a real multi-person Matrix room.
- `impact:security`: Visible tool/progress summaries can include file paths, command names, search terms, and operational context in a shared room.

## Summary

Keep open: current main and the latest shipped release still allow non-Slack `ChatType: "channel"` turns through the shared verbose/tool-summary gate, and the issue is already paired with an open closing fix PR at https://github.com/openclaw/openclaw/pull/85069.

## What This Changes

Default-mode Matrix multi-person rooms should suppress internal tool/progress events in visible chat while preserving explicit verbose/message-tool opt-ins.

## Best Possible Solution
