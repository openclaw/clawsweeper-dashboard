## Summary

Keep open: current main and v2026.6.8 still turn the reported Claude live-session `out of extra usage` result into a billing failure, fail the active turn, and exclude billing-shaped failures from the fresh CLI-session retry path. The linked auth-cooldown work is related, but it is not a merged fix for the suspected poisoned Claude live subprocess boundary.

## What This Changes

Investigate false Claude CLI live-session `out of extra usage` failures while the same subscription still works through direct `claude --print`.

## Best Possible Solution

Add a maintainer-approved Claude CLI live-session recovery rule that retires a suspected poisoned live subprocess and proves fresh-process retry or next-turn recovery without weakening true billing backoff.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No live high-confidence reproduction was run, but source inspection reproduces the OpenClaw side: a Claude live `is_error` extra-usage result becomes a billing FailoverError, fails the turn, and is excluded from fresh-session retry.

## Solution Assessment

Is this the best way to solve the issue? Unclear. The likely fix belongs at the Claude CLI live-session recovery boundary, but maintainers need to decide how billing-shaped errors should interact with true billing cooldowns and same-auth fallback behavior.
