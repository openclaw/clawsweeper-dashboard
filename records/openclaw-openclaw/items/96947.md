## What This Changes

The issue reports a regression where OpenRouter-routed Anthropic sessions with `cacheRetention: "long"` and `contextPruning: { mode: "cache-ttl" }` stopped writing conversation-level prompt-cache checkpoints after upgrading to v2026.6.10.

## Best Possible Solution

Capture the failing OpenRouter request payload plus `model.completed` usage for a repeated-turn session, then repair the narrow provider-cache path so system, tool, conversation, retention, and cache-ttl pruning behavior remain aligned for OpenRouter Anthropic models.

## Reproduction Assessment

No high-confidence full reproduction yet. The reporter gives detailed production usage and source inspection shows a plausible wrapper-level gap, but the current OpenAI-completions path also has conversation marker logic, so the actual failing runtime path still needs tracing.

## Solution Assessment

Unclear. Adding conversation markers directly to the wrapper is plausible, but current main already has a separate completions-layer marker path, so the best fix should be chosen after confirming which path sends the failing payload.
