## Summary

Keep open: the report is a fresh released OpenRouter Anthropic cache regression with strong production telemetry, and neither current main nor the related OpenRouter session-affinity work proves that conversation-history cache writes are restored.

## What This Changes

The issue reports that after upgrading to v2026.6.10, OpenRouter Anthropic sessions stopped writing conversation-history prompt-cache markers, leaving only the static system prompt cached and increasing long-session cost.

## Best Possible Solution

Capture the outgoing v2026.6.10/current-main OpenRouter Anthropic payload and live usage, then fix the provider-owned cache shaping or sticky-routing path with focused regression coverage and live cacheWrite/cacheRead proof.

## Reproduction Assessment

## Solution Assessment

Unclear. The fix should stay in the OpenRouter/provider cache boundary, but maintainers need to determine whether the best repair is cache_control placement, wrapper ordering, compat metadata, session affinity, or an approved `session_id` path.
