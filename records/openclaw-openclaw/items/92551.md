## What This Changes

The issue asks OpenClaw to stop official Docker images from shipping a stale nested `openclaw` package and `/app/node_modules/.bin/openclaw` shim brought in by bundled extension package pins.

## Best Possible Solution

Land a Docker runtime-assets cleanup that removes the stale top-level `openclaw` link, `.bin` shim, and pnpm virtual-store package before the final `node_modules` copy, with focused Docker proof that all stale host-package material is absent.

Yes. The issue gives a Docker build/run repro, and current source shows the final image copies `node_modules` while extension manifests and the lockfile still resolve `openclaw@2026.5.28`; I did not run the Docker build in this read-only pass.

## Solution Assessment

No for a top-level-only cleanup. The best fix should prune all stale host-package material from `runtime-assets`, including pnpm virtual-store content, before the final image copy.
