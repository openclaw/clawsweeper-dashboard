## Summary

This issue is still necessary: current main and the latest release still copy `/app/node_modules` into the Docker runtime image while bundled plugin manifests and the lockfile still materialize `openclaw@2026.5.28`. The linked fixing PR is open and cleanly targets the problem, but it is unmerged, so the issue should stay open until that fix or an equivalent one lands.

## What This Changes

The issue asks OpenClaw to stop shipping a stale nested `openclaw` package and `.bin/openclaw` shim in the Docker runtime image when bundled plugin manifests pin the published release.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/92590, or replace it with an equivalent narrow Docker image cleanup that removes the stale nested package and shim while preserving runtime resolution to `/app/dist`.

## Reproduction Assessment

## Solution Assessment

Yes, the narrow Docker image cleanup in the linked PR is the best immediate fix for the stale binary footgun. A broader dependency-model cleanup may be useful later, but it is not required before removing the shipped stale package and shim.
