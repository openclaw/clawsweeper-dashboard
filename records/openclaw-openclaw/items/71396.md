## Summary

Keep open. Current main still has the Feishu mention-cascade path, and the latest PR fixes the inherited propagation direction but remains blocked because it removes the stable open-ID/card conversion path needed for explicit mentions; a maintainer has already requested Feishu-owner follow-up.

## What This Changes

The PR removes Feishu inbound `mentionTargets` from reply delivery, rewrites the agent mention hint as context-only with display-name hardening, adds text/card/streaming regression tests, and updates `CHANGELOG.md`.

## Best Possible Solution

Land a Feishu-owned revision that disables unconditional inbound mention propagation while preserving intentional explicit mentions through stable `{openId, name}` context and safe text/card conversion coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows inbound Feishu mentions become `ctx.mentionTargets`, flow into both reply-dispatch paths, and are converted into outbound text/card mention markup.

## Solution Assessment

No. Removing inherited `mentionTargets` from fallback, ack, error, and ordinary replies is the right direction, but the patch also removes the remaining stable conversion path for explicit card/streaming/post mention replies before replacing it safely.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve stable mention IDs in context:** `extensions/feishu/src/bot.ts:316`
  - body: After dispatcher-side mention injection is removed, this hint gives the model only sanitized display names. Post/rich-text inputs can render mentions as `@name` while the stable open ID stays internal, so intentional mention-forward replies cannot reliably notify the intended Feishu user when names are duplicated or changed.
  - confidence: 0.86
- **[P2] Keep card mention conversion for explicit replies:** `extensions/feishu/src/reply-dispatcher.ts:528-535`
  - body: Card sends no longer pass mention metadata, and streaming-card close sends raw combined text. Feishu card mentions need the open-ID backed `<at id=...></at>` form, so explicit mention replies in card mode can render as plain text unless a safe explicit rewrite path replaces the removed automatic prefixing.
  - confidence: 0.84
