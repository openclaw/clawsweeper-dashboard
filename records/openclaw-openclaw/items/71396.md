## Summary

Keep this PR open: it targets a valid Feishu cascade bug that is still present on current main, but the latest branch still needs real Feishu runtime proof and leaves maintainer-flagged explicit mention/card behavior unresolved.

## What This Changes

The PR removes inbound Feishu `mentionTargets` from outbound reply dispatch, changes the agent mention hint to context-only sanitized names, adds Feishu reply regression tests, and updates the changelog.

## Best Possible Solution

Land a Feishu-owner revision that disables inherited mention propagation while preserving explicit mentions through stable `{openId, name}` context and safe card/text conversion, backed by focused tests and redacted runtime proof.

## Reproduction Assessment

Yes, source-reproducible: current main passes inbound `ctx.mentionTargets` into Feishu reply dispatch, and the dispatcher turns them into outbound mentions for text, card, fallback, and streaming close paths. I did not establish a live Feishu tenant reproduction in this read-only review.

## Solution Assessment

No, not yet: removing unconditional propagation is the right direction, but the current PR also removes the stable-ID/card conversion path needed for intentional explicit mentions and still lacks after-fix Feishu runtime proof.
