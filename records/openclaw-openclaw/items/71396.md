## Summary

Keep open: current main still has the Feishu automatic mention-cascade bug, and this PR is the active fix path, but the latest branch still needs the Feishu maintainer pass requested in discussion for explicit mention/card semantics.

## What This Changes

This PR removes Feishu inbound `mentionTargets` from reply dispatch, rewrites the agent prompt as context-only, adds text/card/streaming-card regression tests, and updates the changelog.

## Best Possible Solution

Land a Feishu-owned revision that disables unconditional inbound mention propagation while preserving intentional explicit mentions through stable allowlisted context and safe text/card conversion.

## Reproduction Assessment

Yes. Source inspection on current main shows inbound `mentionTargets` are captured, passed into reply dispatch, and applied to first outbound text/card/fallback chunks plus streaming-card close.

## Solution Assessment

No. Removing inherited mention metadata is necessary, but the branch still drops the only openId-backed card conversion path and leaves stable explicit-mention behavior unresolved.

  - body: After dispatcher-side mention injection is removed, this hint gives the model only sanitized display names.   - confidence: 0.86

Feishu card mentions require the `openId`-backed `<at id=...></at>` form, so explicit mention replies in card mode can render as plain text unless a safe explicit rewrite path replaces the removed automatic prefixing.
  - confidence: 0.84
