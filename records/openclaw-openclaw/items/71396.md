## Summary

Keep open: this PR targets a real Feishu mention-cascade bug still present on current main, but the latest branch still drops explicit mention/card conversion paths and lacks after-fix real Feishu proof.

## What This Changes

The PR removes inbound Feishu `mentionTargets` from reply dispatch, rewrites the Feishu agent prompt to context-only sanitized names, adds Feishu regression tests, and adds a changelog entry.

## Best Possible Solution

Land a Feishu-owned revision that disables inherited mention propagation while preserving intentional explicit mentions through stable `{openId, name}` context and safe text/card conversion, backed by focused tests and redacted runtime proof.

## Reproduction Assessment

Yes, source-reproducible: current main passes `ctx.mentionTargets` into the Feishu reply dispatcher, and the dispatcher forwards them as outbound mentions on first text/card chunks and streaming-card close. I did not establish a live Feishu tenant reproduction.

## Solution Assessment

No, not yet: removing unconditional propagation is the right direction, but the branch still removes stable-ID/card conversion needed for intentional explicit mentions and lacks real after-fix Feishu proof.

  - body: After dispatcher-side mention injection is removed, this hint gives the model only sanitized display names.   - confidence: 0.86

  - confidence: 0.84
