## Summary

Keep open: current main still has the Feishu mention-cascade behavior, and this PR addresses the right narrow bug, but the latest head still needs the Feishu-owner pass already requested in discussion because it removes the stable-ID/card conversion path before adding an explicit-only replacement.

## What This Changes

The PR changes the Feishu plugin to stop passing inbound `mentionTargets` into outbound replies, rewrites mention prompt context, adds Feishu regression tests, and updates the changelog.

## Best Possible Solution

Land a Feishu-owned revision that disables unconditional inherited mention propagation while preserving intentional explicit mentions through stable allowlisted context and safe text/card/streaming conversion, with focused regression tests and the changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: a Feishu message with bot plus other-user mentions creates `ctx.mentionTargets`, both dispatch paths forward them, and first text/card/fallback replies plus streaming-card close convert them into real outbound mention markup.

## Solution Assessment

No. The PR is pointed at the right bug and removes the unconditional cascade, but it is not the best complete fix until it preserves explicit mention behavior with stable IDs and safe card-compatible conversion.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve stable mention IDs in context:** `extensions/feishu/src/bot.ts:315-317`
  - body: After dispatcher-side mention injection is removed, this hint only gives the model sanitized display names. Post/rich-text inputs often render mentions as `@name` while keeping the stable open ID internal, so intentional mention-forward replies cannot reliably notify the intended Feishu user when names collide or change.
  - confidence: 0.86
- **[P2] Keep explicit card mention conversion:** `extensions/feishu/src/reply-dispatcher.ts:598-605`
  - body: Card replies now call `sendStructuredCardFeishu` without mention metadata, and streaming-card close sends raw combined text. Feishu card mentions need the open-ID-backed `<at id=...></at>` form produced by `buildMentionedCardContent`, so intentional mentions can degrade to plain text unless a safe explicit rewrite path replaces the removed auto-prefixing.
  - confidence: 0.84
