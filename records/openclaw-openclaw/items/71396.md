## Summary

Keep this PR open. Current main still has the Feishu mention-cascade path, and the PR removes that path but still needs a Feishu-owner pass because it drops the stable explicit-mention conversion used by card, streaming-card, and post/rich-text flows.

## What This Changes

The PR removes inbound Feishu `mentionTargets` from outbound reply dispatch, rewrites the agent mention prompt as context-only sanitized display names, adds no-auto-mention regression tests, and updates the changelog.

## Best Possible Solution

Land a Feishu-owned revision that disables unconditional inbound `mentionTargets` propagation while preserving intentional explicit mentions through stable `{openId, name}` context and a safe allowlisted conversion path for text/card/streaming delivery.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: inbound Feishu mentions become `ctx.mentionTargets`, bot dispatch passes them to the reply dispatcher, and the dispatcher converts them into outbound Feishu mention markup.

## Solution Assessment

No. Removing inherited mention metadata is necessary, but this patch removes the only stable open-ID conversion hook before adding an explicit-only replacement for card, streaming-card, and post/rich-text mention replies.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve stable mention IDs in context:** `extensions/feishu/src/bot.ts:316`
  - body: After dispatcher-side `mentionTargets` injection is removed, this context hint gives the model only sanitized display names. Post/rich-text inputs render mentions as `@name` while keeping stable open IDs internal, so intentional mention-forward replies cannot reliably notify the right user when names are duplicated or changed.
  - confidence: 0.86
- **[P2] Keep card mention conversion for explicit replies:** `extensions/feishu/src/reply-dispatcher.ts:528-535`
  - body: Card sends now call `sendStructuredCardFeishu` without mention metadata, and streaming-card close sends raw combined text. Feishu card mentions require the `<at id=...></at>` form produced from stable open IDs, so explicit mention replies in card/streaming mode can render as plain text unless an explicit-only rewrite path replaces the removed automatic prefix.
  - confidence: 0.84
