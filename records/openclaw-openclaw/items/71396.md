## Summary

Keep open: current main still has the Feishu mention cascade, and the latest PR head fixes that narrow path but still leaves explicit mention/card semantics unresolved under active maintainer handoff.

## What This Changes

The PR removes Feishu inbound `mentionTargets` from outbound reply dispatch, changes the agent mention hint to context-only text, adds regression tests, and updates `CHANGELOG.md`.

## Best Possible Solution

Land a Feishu-owned revision that disables inherited mention propagation while preserving explicit mentions through allowlisted stable `{openId, name}` context and safe text/card/streaming conversion, with focused tests and the changelog entry kept.

## Reproduction Assessment

Yes. Source inspection on current main shows Feishu `mentionTargets` are extracted, passed into reply dispatch, and converted into outbound mention markup on first text/card/fallback replies and streaming-card close.

## Solution Assessment

No. Removing unconditional propagation is the right direction, but this patch removes the open-ID-backed conversion path before adding the explicit-only replacement needed for card/post/streaming flows.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve stable mention IDs in context:** `extensions/feishu/src/bot.ts:315`
  - body: This hint now exposes only display names after dispatcher-side `mentionTargets` are removed. Post/rich-text inputs render mention text as `@name` while keeping `open_id` internal, so an explicit mention reply cannot reliably notify the intended Feishu user when names collide or change.
  - confidence: 0.86
- **[P2] Keep explicit card mention conversion:** `extensions/feishu/src/reply-dispatcher.ts:595-601`
  - body: Card sends now call `sendStructuredCardFeishu` without mention metadata, and streaming-card close sends raw combined text. Feishu card mentions need the open-ID-backed `<at id=...></at>` conversion from `buildMentionedCardContent`, so explicit mention replies in card or streaming mode can degrade to plain text.
  - confidence: 0.84
