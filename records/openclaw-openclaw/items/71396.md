## Summary

Keep open: the target Feishu cascade bug is still present on current main, and the latest branch still has a maintainer-confirmed gap around explicit mention/card semantics before merge.

## What This Changes

This PR removes Feishu inbound `mentionTargets` from outbound reply dispatch, rewrites the agent mention hint as sanitized context, adds regression tests, and updates the changelog.

## Best Possible Solution

Land a Feishu-owned revision that disables unconditional inherited mention propagation while preserving explicit mentions through stable allowlisted `{openId, name}` context and safe text/card conversion, backed by focused regression tests.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: inbound Feishu mentions become `ctx.mentionTargets`, the bot passes them into reply dispatch, and send helpers convert them into outbound Feishu mention markup.

## Solution Assessment

No. Removing unconditional mention propagation is the right direction, but this patch removes the stable ID/card conversion path without replacing it with an explicit-only mechanism.

  - confidence: 0.86

Feishu card notifications require the `openId`-backed `<at id=...></at>` form, so intentional mention replies in card mode can render as plain text unless a safe explicit-only conversion path replaces the removed automatic prefixing.
  - confidence: 0.84
