## Summary

Keep this PR open. Current main still has the Feishu mention-cascade path the PR targets, but the latest patch still drops stable mention ID/card conversion behavior for explicit mention replies and the discussion already has a Feishu maintainer request for another pass before merge.

## What This Changes

This PR removes inbound Feishu `mentionTargets` from outbound reply dispatch, changes the agent mention hint to sanitized context-only text, adds text/card/streaming regression tests, and updates `CHANGELOG.md`.

## Best Possible Solution

Land a Feishu-owned revision that stops inherited mention propagation for fallback, error, ack, ordinary, card, and streaming replies while preserving intentional explicit mentions through stable open IDs and safe text/card conversion.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: inbound Feishu mentions become `ctx.mentionTargets`, bot dispatch passes them into the reply dispatcher, and the dispatcher/send helpers convert them into outbound Feishu mention markup.

## Solution Assessment

No. Removing inherited mention metadata is necessary, but this patch removes the only stable open-ID/card conversion path before adding an explicit-only replacement for card, streaming-card, and post/rich-text mention replies.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve stable mention IDs in context:** `extensions/feishu/src/bot.ts:316`
  - body: After dispatcher-side `mentionTargets` injection is removed, this hint gives the model only sanitized display names. In post/rich-text inputs the model can see only `@name` while the stable open ID stays internal, so intentional mention-forward replies cannot reliably notify the intended Feishu user when names are duplicated or changed.
  - confidence: 0.86
- **[P2] Keep explicit card mention conversion:** `extensions/feishu/src/reply-dispatcher.ts:528-535`
  - body: Card sends now drop `mentions`, and streaming-card close sends raw combined text. Feishu card mentions require the `openId`-backed `<at id=...></at>` form produced by `buildMentionedCardContent`, so intentional mention replies in card mode can render as plain text and stop notifying users unless an explicit-only rewrite path replaces the removed automatic prefixing.
  - confidence: 0.84
