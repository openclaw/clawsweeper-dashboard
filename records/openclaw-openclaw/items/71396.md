## Summary

Keep open: current main still has the Feishu mention-cascade path, and this PR targets the right bug but still needs real Feishu runtime proof plus a Feishu-owner pass on explicit mention/card semantics before merge.

## What This Changes

The branch removes Feishu `mentionTargets` from outbound reply dispatch, rewrites the agent mention hint as sanitized context, adds Feishu regression tests, and updates the changelog.

## Best Possible Solution

Land a Feishu-owned revision that disables inherited mention propagation while preserving safe explicit mentions through stable `{openId, name}` context and card-compatible conversion.

## Reproduction Assessment

Yes, source-reproducible: current main extracts `ctx.mentionTargets`, passes them into `createFeishuReplyDispatcher`, and the dispatcher forwards them as outbound mention metadata on first replies and streaming-card close. I did not establish a live Feishu runtime reproduction.

## Solution Assessment

No. Removing unconditional propagation is the right direction, but the latest patch also removes the stable-ID/card conversion path needed for intentional explicit mentions; the safer fix keeps propagation off while adding an explicit-only rewrite path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve stable IDs for explicit mentions:** `extensions/feishu/src/bot.ts:338-342`
  - body: After dispatcher-side mention injection is removed, the new context hint only gives the model sanitized display names. In post/rich-text flows the stable open ID is kept internally while rendered text can be only `@name`, so intentional mention-forward replies cannot reliably notify the intended user when names collide or change.
  - confidence: 0.82
- **[P2] Keep explicit card mention conversion:** `extensions/feishu/src/reply-dispatcher.ts:599-605`
  - body: Card sends now call `sendStructuredCardFeishu` without mention metadata, and streaming-card close sends raw combined text. Feishu card mentions still require the open-ID-backed `<at id=...></at>` form produced by the existing conversion helper, so intentional mention replies in card or streaming mode can render as plain text and stop notifying users.
  - confidence: 0.84
