## Summary

Keep this PR open: current main still has the Feishu mention-cascade bug, and the PR is a plausible fix direction, but latest head still lacks real Feishu runtime proof and leaves explicit mention/card behavior unresolved after maintainer review.

## What This Changes

The branch stops passing inbound Feishu `mentionTargets` into outbound Feishu reply dispatch, rewrites the agent mention hint as sanitized context, adds Feishu regression tests, and updates the changelog.

## Best Possible Solution

Land a Feishu-owned revision that disables inherited mention propagation while preserving safe explicit mentions through stable openId/name context and card-compatible conversion, with focused tests and real Feishu proof.

## Reproduction Assessment

Yes, by source inspection: current main extracts `ctx.mentionTargets`, passes them into `createFeishuReplyDispatcher`, and the dispatcher forwards them as outbound mention metadata on first replies and streaming-card close. I did not live-test Feishu.

## Solution Assessment

No. Removing unconditional propagation is the right direction, but latest head still drops stable open IDs and the card conversion path needed for intentional explicit mentions; the safer fix keeps an explicit-only mention rewrite path.
