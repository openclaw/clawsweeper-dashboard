## Summary

Keep this PR open. Current main still has the Feishu mention-cascade behavior, and the PR fixes that narrow path, but the latest branch still removes the open-ID/card conversion path for intentional mentions and the discussion includes an unresolved maintainer handoff for one more Feishu-owner pass.

## What This Changes

The PR removes Feishu inbound `mentionTargets` from outbound reply dispatch, rewrites the agent mention hint as sanitized context-only text, adds regression tests, and updates the changelog.

## Best Possible Solution

Land a Feishu-owned revision that disables unconditional inherited mention propagation while preserving intentional explicit mentions through stable allowlisted `{openId, name}` context and safe text/card/streaming conversion, with focused regression tests and the changelog entry kept.

## Reproduction Assessment

Yes. Source inspection on current main shows Feishu `mentionTargets` are extracted, passed into reply dispatch, and converted into outbound mention markup on first text/card/fallback replies and streaming-card close.

## Solution Assessment

No. Removing unconditional propagation is the right direction, but this patch removes the open-ID-backed conversion path before adding the explicit-only replacement needed for card/post/streaming flows.

  - body: After dispatcher-side mention injection is removed, the model only receives sanitized display names in this hint.   - confidence: 0.86

Feishu card mentions need the open-ID-backed `<at id=...></at>` form produced by `buildMentionedCardContent`, so explicit mention replies in card mode can degrade to plain text unless a safe explicit rewrite path replaces the removed auto-prefixing.
  - confidence: 0.84
