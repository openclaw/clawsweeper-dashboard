## Summary

Keep this PR open: the cascade bug is still present on current main, but the branch remains blocked by active Feishu maintainer feedback about explicit mention/card semantics and lacks required real behavior proof.

## What This Changes

The PR removes Feishu `mentionTargets` from outbound reply dispatch, softens sanitized mention context in the agent prompt, adds Feishu regression tests, and updates the changelog.

## Best Possible Solution

Land a Feishu-owned revision that stops inherited mention propagation while preserving intentional mentions through stable open-ID context and safe allowlisted text/card conversion, backed by focused tests and real Feishu proof.

## Reproduction Assessment

Yes, by source inspection. Current main captures `mentionTargets`, passes them into the dispatcher, and the dispatcher forwards them into text/card/fallback/streaming close sends; no live Feishu run was performed in this read-only review.

## Solution Assessment

No. Removing unconditional propagation is the right direction, but this PR still leaves explicit mention/card semantics unresolved and needs a Feishu owner pass plus real behavior proof.
