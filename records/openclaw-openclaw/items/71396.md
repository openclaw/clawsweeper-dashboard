## Summary

Keep open: this PR fixes the inherited Feishu mention cascade path, but the latest patch still drops stable explicit-mention/card conversion and lacks required real Feishu runtime proof after active maintainer feedback.

## What This Changes

The PR removes `mentionTargets` from Feishu outbound reply dispatch, rewrites mentioned-user prompt context, adds no-auto-mention regression tests, and adds a changelog entry.

## Best Possible Solution

Land a Feishu-owned revision that disables inherited mention propagation while preserving intentional mentions through stable `{openId, name}` context and safe explicit-only text/card/streaming conversion, backed by focused tests and real Feishu proof.

## Reproduction Assessment

Yes, by source inspection against current main: Feishu captures `ctx.mentionTargets`, passes them into `createFeishuReplyDispatcher`, and the dispatcher forwards them into first text/card/fallback sends and streaming-card close. I did not run a live Feishu scenario.

## Solution Assessment

No. Removing unconditional propagation is the right direction, but this patch still makes intentional Feishu mentions unreliable in card, streaming-card, and post/rich-text flows; the safer fix preserves an explicit-only conversion path.

## Review Findings

Overall correctness: patch is incorrect

  - body: After dispatcher-side `mentionTargets` injection is removed, this hint only gives the model sanitized display names.   - confidence: 0.86

Feishu card mentions require open-ID-backed `<at id=...></at>` conversion, so intentional mention replies in card mode can render as plain text and stop notifying users.
  - confidence: 0.84
