## Summary

Keep open: current main still has the Feishu mention-cascade path, and the PR is useful, but the latest branch remains blocked because explicit Feishu mention preservation in card/post/streaming flows is unresolved and a maintainer asked for another pass.

## What This Changes

The PR removes Feishu `mentionTargets` from outbound reply dispatch, rewrites the agent mention prompt as context-only, adds regression tests, and updates the changelog.

## Best Possible Solution

Land a Feishu-owned revision that disables unconditional inherited `mentionTargets` propagation while preserving intentional explicit mentions through stable allowlisted context and safe text/card conversion, with focused regressions and the changelog entry kept.

## Reproduction Assessment

Yes. Source inspection on current main shows incoming Feishu mention targets are captured, passed into reply dispatch, and converted into outbound mention markup on first fallback, text/card, and streaming-card replies.

## Solution Assessment

No. Removing inherited mention propagation is the right direction, but this patch also removes the open-ID-backed conversion path before adding the explicit-only replacement needed for card/post flows.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve stable mention IDs in context:** `extensions/feishu/src/bot.ts:313-315`
  - body: After dispatcher-side mention injection is removed, the model only sees sanitized display names. Post/rich-text inputs render mentions as `@name` while retaining `open_id` internally, so intentional mention-forward replies cannot reliably notify the intended Feishu user when names collide or change.
  - confidence: 0.86
- **[P2] Keep card mention conversion for explicit replies:** `extensions/feishu/src/reply-dispatcher.ts:590-599`
  - body: Card sends now call `sendStructuredCardFeishu` without mention metadata, and streaming-card close sends raw combined text. Feishu card mentions require the open-ID-backed `<at id=...></at>` form, so explicit mention replies in card mode can render as plain text unless a safe explicit rewrite path replaces the removed automatic prefixing.
  - confidence: 0.84
