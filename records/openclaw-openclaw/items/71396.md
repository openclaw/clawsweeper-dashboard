## Summary

Keep this PR open. It targets a real Feishu bug that is still present on current main, but the latest head still removes the stable ID/card conversion path needed for intentional explicit mentions, and a Feishu maintainer has already asked for another pass before prepare/merge.

## What This Changes

This PR removes Feishu inbound mention targets from reply dispatch, changes the agent prompt to context-only sanitized mention names, adds reply-path regression tests, and updates the changelog.

## Best Possible Solution

Land a Feishu-owned revision that disables unconditional inherited mention propagation while preserving intentional explicit mentions through stable allowlisted context and safe text/card conversion coverage.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: inbound Feishu mentions become `ctx.mentionTargets`, bot dispatch passes them into the reply dispatcher, and the dispatcher/send helpers convert them into outbound Feishu mention markup.

## Solution Assessment

No. Removing inherited mention metadata is necessary, but the current patch removes the only stable open-ID/card conversion path before adding an explicit-only replacement for card, streaming-card, and post/rich-text mention replies.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

  - body: After dispatcher-side mention injection is removed, this hint gives the model only sanitized display names. Post/rich-text inputs keep stable open IDs internal while rendering `@name`, so intentional mention-forward replies cannot reliably notify the intended Feishu user when names are duplicated or changed.
  - confidence: 0.86

Feishu card mentions require the open-ID-backed `<at id=...></at>` form, so explicit mention replies in card mode can render as plain text unless a safe explicit rewrite path replaces the removed automatic prefixing.
  - confidence: 0.84
