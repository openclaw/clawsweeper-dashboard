## Summary

Keep open: the branch fixes the unwanted cascade, but it still drops explicit mention support in card, streaming-card, and post flows, and maintainer discussion asked for one more Feishu-owner pass before preparation.

## What This Changes

The PR removes Feishu inbound `mentionTargets` from reply delivery, changes the agent mention hint to context-only sanitized names, adds text/card/streaming regression tests, and updates `CHANGELOG.md`.

## Best Possible Solution

Land a Feishu-owned revision that disables unconditional inbound mention propagation while preserving intentional explicit mentions through stable `{openId, name}` context and a safe allowlisted conversion path, backed by the focused tests and changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: inbound Feishu mentions become `ctx.mentionTargets`, bot dispatch passes them to the reply dispatcher, and the dispatcher converts them into outbound Feishu mention markup.

## Solution Assessment

No. Removing inherited mention metadata is necessary, but this patch removes the only stable open-ID conversion hook before adding an explicit-only replacement for card, streaming-card, and post/rich-text mention replies.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve stable mention IDs in context:** `extensions/feishu/src/bot.ts:313-316`
  - body: After dispatcher-side `mentionTargets` injection is removed, this hint gives the model only sanitized display names. In post/rich-text inputs the model sees `@name` while the stable open ID stays internal, so intentional mention-forward replies cannot reliably notify the right user when names are duplicated or changed.
  - confidence: 0.86
- **[P2] Keep card mention conversion for explicit replies:** `extensions/feishu/src/reply-dispatcher.ts:528-535`
  - body: Card sends now call `sendStructuredCardFeishu` without mention metadata, and streaming-card close sends raw combined text. Feishu card mentions require the `openId`-backed card form from `buildMentionedCardContent`, so explicit mention replies can render as plain text unless an explicit-only rewrite path replaces the removed automatic prefixing.
  - confidence: 0.84
