## Summary

Keep open: the Feishu mention-cascade bug is still present on current main, so this PR is relevant, but the latest patch still drops the stable-ID/card conversion path needed for explicit Feishu mentions in post, card, and streaming-card flows. The PR discussion also has an active Feishu-owner handoff request; CI is green and the security surface looks clear, but the functional review findings remain blocking.

## What This Changes

This PR removes Feishu inbound `mentionTargets` from outbound reply dispatch, changes the agent mention prompt to sanitized context-only display names, adds no-auto-mention regression tests, and updates `CHANGELOG.md`.

## Best Possible Solution

Land a Feishu-owned revision that disables unconditional inherited mention propagation for fallback, error, ack, ordinary, card, and streaming replies while preserving intentional explicit mentions through stable `{openId, name}` context and a safe allowlisted conversion path for text/card delivery, with the focused regression tests and changelog entry retained.

## Reproduction Assessment

Yes. Current-main source inspection gives a high-confidence path: inbound Feishu mentions become `ctx.mentionTargets`, are passed into reply dispatch, and are converted into real Feishu mention markup on first text/card chunks and streaming-card close.

## Solution Assessment

No. Removing inherited mention metadata is necessary for the cascade bug, but the proposed patch drops the only stable open-ID conversion hook before adding an explicit-only replacement for post/card/streaming mention replies.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve stable mention IDs in context:** `extensions/feishu/src/bot.ts:316`
  - body: After dispatcher-side `mentionTargets` injection is removed, this prompt leaves the model only quoted display names. Post/rich-text inputs record stable open IDs internally but render mentions as `@name`, so an intentional mention-forward reply can lose the target ID or hit duplicate names. Include an allowlisted stable target mapping before relying on model-authored mentions.
  - confidence: 0.86
- **[P2] Keep card mention conversion for explicit replies:** `extensions/feishu/src/reply-dispatcher.ts:528-535`
  - body: Card delivery now sends chunks without mention metadata, and streaming close sends raw combined text. Feishu card mentions rely on open IDs converted into `<at id=...></at>`, so explicit mention replies in card/streaming mode can render as plain text and stop notifying users unless an explicit-only rewrite path replaces the removed auto-prefix.
  - confidence: 0.84
