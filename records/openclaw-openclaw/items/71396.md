## Summary

Keep this PR open: the Feishu mention-cascade bug is still present on current main, and the latest PR head is a valid narrow attempt but still drops the open-ID-backed explicit mention conversion path that active maintainer review asked to resolve before preparation.

## What This Changes

The PR removes inbound Feishu `mentionTargets` from outbound reply dispatcher calls, rewrites the agent mention hint as context-only sanitized display names, adds Feishu reply-path regression tests, and updates `CHANGELOG.md`.

## Best Possible Solution

Land a Feishu-owned revision that disables unconditional inherited mention propagation while preserving intentional mentions through stable allowlisted `{openId, name}` context and safe text/card/streaming conversion, backed by focused regression tests and the changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main shows `mentionTargets` are extracted, passed into Feishu reply dispatch, and converted into outbound mention markup for first text/card/fallback replies and streaming-card close; I did not run a live Feishu reproduction.

## Solution Assessment

No. Removing unconditional propagation is the right direction, but the current PR removes the open-ID-backed conversion path before adding the explicit-only replacement needed for card/post/streaming flows.
