## Summary

Keep this PR open. Current main still has the Feishu automatic mention cascade, and the latest branch fixes that narrow cascade but still drops the stable-ID/card conversion path that a maintainer explicitly asked to preserve before preparation.

## What This Changes

The PR removes inbound Feishu `mentionTargets` from outbound reply dispatch, changes the mention prompt to context-only display names, adds Feishu regression tests, and updates the changelog.

## Best Possible Solution

Land a Feishu-owned revision that disables unconditional inbound mention propagation while preserving intentional explicit mentions through stable allowlisted context and safe text/card conversion, with focused regressions and the changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main shows inbound Feishu mentions become `ctx.mentionTargets`, both reply paths forward them to the dispatcher, and the dispatcher/send helpers convert them into outbound Feishu mentions.

## Solution Assessment

No. Removing inherited mention metadata is necessary, but this branch also removes the remaining stable-ID/card conversion path before adding a safe explicit-only replacement.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

  - body: After dispatcher-side mention injection is removed, this hint gives the model only sanitized display names. In post/rich-text inputs the model often sees only `@name` while the stable open ID stays internal, so intentional mention-forward replies cannot reliably notify the intended Feishu user when names are duplicated or changed.
  - confidence: 0.86

Feishu card mentions require the `openId`-backed `<at id=...></at>` form, so explicit mention replies in card mode can render as plain text unless a safe explicit rewrite path replaces the removed automatic prefixing.
  - confidence: 0.84
