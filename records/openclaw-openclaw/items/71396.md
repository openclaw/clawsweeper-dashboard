## Summary

Keep this PR open: current main still has the Feishu mention-cascade bug and the PR is aimed at the right surface, but the latest patch still needs Feishu maintainer handling for explicit mention/card semantics before merge.

## What This Changes

The PR removes Feishu inbound `mentionTargets` from outbound reply dispatch, changes the mention prompt to context-only sanitized names, adds plain/card/streaming regression tests, and updates `CHANGELOG.md`.

## Best Possible Solution

Land a Feishu-owned revision that disables inherited mention propagation for fallback/error/ack/ordinary replies while preserving intentional mentions through allowlisted stable IDs and safe text/card conversion, with focused regression tests and the changelog entry.

## Reproduction Assessment

Yes, by source inspection. Current main extracts Feishu mention targets, passes them into reply dispatch, and converts them into outbound Feishu mention markup for first text/card/fallback replies and streaming-card closeout; I did not run a live Feishu reproduction.

## Solution Assessment

No. Removing unconditional propagation is the right direction, but the patch deletes the existing open-ID-backed conversion path before adding the explicit-only replacement needed for card, streaming-card, and post/rich-text flows.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve stable mention IDs in context:** `extensions/feishu/src/bot.ts:313-316`
  - body: After dispatcher-side mention injection is removed, this hint only gives the model sanitized display names. Post/rich-text parsing renders mentions as `@name` while retaining `open_id` internally, so explicit mention-forward replies can no longer reliably notify the intended Feishu user when names collide or change.
  - confidence: 0.86

Feishu card mentions need the open-ID-backed `<at id=...></at>` conversion from `buildMentionedCardContent`, so explicit mentions can degrade to plain text in card/streaming replies.
  - confidence: 0.84
