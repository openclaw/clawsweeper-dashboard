## Summary

Keep this PR open. Current main still has the Feishu mention-cascade bug the PR targets, but the latest PR head still removes the existing stable-ID/card mention conversion path before adding an explicit-only replacement, and maintainer discussion already asked for one more Feishu-owned pass before preparation.

## What This Changes

This PR removes inherited Feishu `mentionTargets` from reply dispatch, changes the agent mention prompt to sanitized context-only display names, adds no-auto-mention regression tests for text/card/streaming delivery, and adds a changelog entry.

## Best Possible Solution

Land a Feishu-owned revision that disables unconditional inherited mention propagation for fallback, error, ack, ordinary, card, and streaming replies while preserving intentional explicit mentions through stable allowlisted `{openId, name}` context and a safe card/text conversion path, with the focused regression tests and changelog entry retained.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: inbound Feishu mentions become `ctx.mentionTargets`, are passed into reply dispatch, and are converted into real mention markup on first text/card chunks and streaming-card close.

## Solution Assessment

No. Removing inherited mention metadata is necessary for the cascade bug, but the current PR also drops the only stable open-ID conversion hook for explicit card/post mention replies before adding a safe explicit-only replacement.

## Review Findings

Overall correctness: patch is incorrect

- **[P2] Preserve stable mention IDs in context:** `extensions/feishu/src/bot.ts:313-316`
  - body: After dispatcher-side mention injection is removed, explicit mentions depend on the model context, but this added hint exposes only sanitized display names. Post/rich-text inputs keep the stable `open_id` internal while rendering `@name`, so explicit mention-forward replies can no longer reliably notify the intended user when names are duplicated or changed.
  - confidence: 0.86
- **[P2] Keep explicit card mention conversion:** `extensions/feishu/src/reply-dispatcher.ts:528-535`
  - body: Removing `mentions` from card delivery and closing streaming cards with raw text removes the path that rewrites Feishu open IDs into card-compatible `<at id=...></at>` tokens. Explicit mentions in card or streaming-card replies can therefore render as plain text and stop notifying users unless an explicit-only rewrite path replaces the removed auto-prefixing.
  - confidence: 0.84
