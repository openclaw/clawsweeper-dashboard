## Summary

Keep open: current main still has the Feishu mention-cascade behavior, and the PR branch addresses that core bug but still needs a Feishu-owned pass for explicit mention handling in card, streaming-card, and post/rich-text flows before merge.

## What This Changes

The PR removes inbound Feishu `mentionTargets` from outbound reply dispatch, rewrites the agent mention hint as sanitized context-only text, adds focused regression tests, and updates the changelog.

## Best Possible Solution

Land a Feishu-owned revision that disables unconditional inbound mention propagation while preserving intentional explicit mentions through stable allowlisted context and safe text/card/streaming conversion, backed by regression tests and the changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main shows incoming Feishu mention targets are captured, passed into the reply dispatcher, and converted into outbound mention markup on first fallback, text/card, and streaming-card replies.

## Solution Assessment

No. Removing inherited mention propagation is the right direction, but the latest PR also removes the existing open-ID-backed card conversion path before adding a safe explicit-only replacement.
