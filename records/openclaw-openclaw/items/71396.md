## Summary

Keep this PR open: the underlying Feishu mention-cascade bug is still present on current main, but the branch still needs real Feishu runtime proof and Feishu-owner review for explicit mention/card semantics before merge.

## What This Changes

The PR removes inbound Feishu `mentionTargets` from reply-dispatcher delivery, changes the agent prompt to context-only sanitized mention names, adds Feishu regression tests, and updates the changelog.

## Best Possible Solution

Land a Feishu-owned revision that disables inherited mention propagation while preserving intentional explicit mentions through stable target identity and safe card/text conversion, with focused regression tests and redacted runtime proof.

## Reproduction Assessment

Yes, source-reproducible: current main passes `ctx.mentionTargets` into the Feishu reply dispatcher, and the dispatcher forwards them as outbound `mentions` on first replies and streaming-card close. I did not establish a live Feishu runtime reproduction.

## Solution Assessment

No, not yet: removing unconditional propagation is the right direction, but the branch still removes the stable-ID/card conversion path needed for intentional explicit mentions and lacks real after-fix Feishu proof.
