## Summary

Keep open: the PR targets a valid Feishu bug still present on current main, but the latest head remains merge-blocked by missing real Feishu behavior proof and unresolved explicit mention/card semantics.

## What This Changes

The PR removes Feishu inbound `mentionTargets` from reply dispatch, changes the agent prompt hint to sanitized context-only names, adds Feishu regression tests, and updates the changelog.

## Best Possible Solution

Land a Feishu-owned revision that disables unconditional inherited mention propagation while preserving intentional explicit mentions through stable allowlisted IDs and safe text/card conversion, with focused tests and redacted runtime proof.

## Reproduction Assessment

Yes, source-reproducible: current main passes inbound `ctx.mentionTargets` into Feishu reply dispatch, and the dispatcher turns them into outbound mentions for text, card, fallback, media fallback, and streaming-card close paths. I did not establish a live Feishu tenant reproduction in this read-only review.

## Solution Assessment

No, not merge-ready: removing unconditional propagation is the right direction, but this patch also removes the stable-ID/card conversion path needed for intentional explicit mentions and still lacks after-fix Feishu runtime proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve stable mention IDs for explicit replies:** `extensions/feishu/src/bot.ts:340-343`
  - body: After dispatcher-side mention injection is removed, this hint exposes only sanitized display names. Post/rich-text inputs can render mentions as `@name` while the stable open ID stays internal, so an intentional mention-forward reply cannot reliably notify the intended Feishu user when names collide or change.
  - confidence: 0.85
- **[P2] Keep card mention conversion for explicit replies:** `extensions/feishu/src/reply-dispatcher.ts:601-610`
  - body: Card sends now call `sendStructuredCardFeishu` without mention metadata, and streaming-card close sends raw combined text. Feishu card notifications require the open-ID-backed form produced by `buildMentionedCardContent`, so explicit mention-forward replies in card or streaming mode can degrade to plain text.
  - confidence: 0.84
