## Summary

Keep this PR open: it fixes the inherited Feishu mention cascade, but the latest branch still drops the stable-ID/card conversion path needed for intentional explicit mentions, and a Feishu maintainer has already asked for one more owner pass before preparation.

## What This Changes

This PR removes Feishu `mentionTargets` from outbound reply dispatch, changes the agent mention hint to sanitized context-only display names, adds reply-dispatcher and bot-helper regression tests, and updates the changelog.

## Best Possible Solution

Land a Feishu-owned revision that disables unconditional inbound mention propagation while preserving intentional mentions through stable allowlisted `{openId, name}` context and safe text/card conversion, with focused regressions and the existing changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main shows inbound Feishu mentions become `ctx.mentionTargets`, both dispatch paths pass them into the reply dispatcher, and the dispatcher/send helpers convert them into outbound Feishu mention markup.

## Solution Assessment

No. Removing inherited mention metadata is necessary, but this branch removes the remaining stable open-ID/card conversion path before adding a safe explicit-only replacement, matching the maintainer request for another pass.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

  - body: After dispatcher-side mention injection is removed, this hint gives the model only sanitized display names. In post/rich-text inputs the model often sees only `@name` while the stable open ID stays internal, so intentional mention-forward replies cannot reliably notify the intended Feishu user when names are duplicated or changed.
  - confidence: 0.86

Feishu card mentions require the `openId`-backed `<at id=...></at>` form, so explicit mention replies in card mode can render as plain text unless a safe explicit rewrite path replaces the removed automatic prefixing.
  - confidence: 0.84
