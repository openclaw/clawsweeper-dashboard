## Summary

Keep this PR open. It fixes a real Feishu mention-cascade bug that is still present on current main, but the current branch still needs Feishu maintainer review because it removes the remaining card/streaming conversion path before adding a safe explicit-only mention path.

## What This Changes

The PR changes the Feishu plugin to stop forwarding inbound `mentionTargets` into outbound replies, updates the agent prompt to treat mentioned users as context, adds Feishu regression tests, and updates the changelog.

## Best Possible Solution

Land a Feishu-owned revision that disables unconditional inbound mention propagation while preserving intentional explicit mentions through stable allowlisted context and safe text/card conversion, backed by focused regressions and the existing changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main shows incoming Feishu mention targets are captured, forwarded into reply dispatch, and converted into outbound mention markup on first fallback, card, text, and streaming-card replies.

## Solution Assessment

No. Removing inherited mention propagation is the right core direction, but the current patch also removes the only open-ID-backed card/streaming conversion path before adding the explicit-only replacement requested in review.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

- **[P2] Preserve stable mention IDs in context:** `extensions/feishu/src/bot.ts:316`
  - body: After dispatcher-side mention injection is removed, this hint gives the model only sanitized display names. In post/rich-text inputs the model can see only `@name` while the stable open ID stays internal, so intentional mention-forward replies cannot reliably notify the intended Feishu user when names collide or change.
  - confidence: 0.86

  - body: Card sends now omit mention metadata, and streaming-card close sends raw combined text. Feishu card mentions require the open-ID-backed `<at id=...></at>` form, so intentional mention replies in card mode can render as plain text unless a safe explicit rewrite path replaces the removed automatic prefixing.
  - confidence: 0.84
