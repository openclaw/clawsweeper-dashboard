## Summary

Keep open: current main still has the Feishu automatic mention cascade, and this PR is useful but still leaves explicit Feishu mention preservation unresolved for card, streaming-card, and post/rich-text flows.

## What This Changes

The PR removes inbound Feishu `mentionTargets` from outbound reply dispatch, rewrites the agent mention hint as context-only with display-name hardening, adds Feishu regression tests, and updates the changelog.

## Best Possible Solution

Land a Feishu-owned revision that disables unconditional inbound mention propagation while preserving intentional explicit mentions through stable allowlisted context and safe text/card conversion, with focused regressions and the changelog entry kept.

## Reproduction Assessment

Yes. Source inspection on current main shows Feishu mention targets are captured, passed into reply dispatch, and converted into outbound mention markup on first fallback, card, text, and streaming-card replies.

## Solution Assessment

No. Removing inherited mention propagation is the right direction, but the current PR also removes the open-ID-backed conversion path before adding the explicit-only replacement needed for card, streaming-card, and post/rich-text flows.

  - confidence: 0.84
