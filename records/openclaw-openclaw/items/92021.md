## Summary

## What This Changes

The PR removes MCP HTTP transport’s opt-in to unsafe cross-origin redirect replay and adds regression coverage for cross-origin 307 and 302 POST redirects.

## Best Possible Solution

Retain the focused removal of the MCP-only unsafe replay override, then demonstrate with a real redirecting server that the destination receives a bodyless request and confirm one representative MCP transport or OAuth flow still succeeds.

## Reproduction Assessment

Yes. Current main explicitly opts MCP requests into unsafe cross-origin replay, and the shared guard tests and implementation show that a cross-origin 307 or 308 preserves the POST body when this option is enabled.

## Solution Assessment

Yes. Removing the exceptional opt-in restores the canonical shared guard policy, matches the sibling provider transport, and avoids adding a second MCP-specific redirect implementation.
