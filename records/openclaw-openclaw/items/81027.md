- **[P1] Handle dispatcher destroy rejections:** `src/infra/net/ssrf.ts:685`
  - body: Undici 8 types and docs define `Dispatcher.destroy()` as returning `Promise<void>` when no callback is supplied, but this path calls it as fire-and-forget inside abort cleanup. A rejected destroy promise will bypass the sync `try/catch` and can surface as an unhandled rejection; type `destroy` as `() => Promise<void> | void` and attach/await a catch path like `close()` does.
  - confidence: 0.86

  - body: This test only waits for the hanging server request to close after the abort signal, which ordinary fetch abort can already do even if the new hard-kill hook is removed. Assert `destroyDispatcher` or a dispatcher `destroy()` seam directly, and keep coverage for the delayed fallback-survival behavior the PR is meant to protect.
  - confidence: 0.78
