## Summary

Keep open. The hard-kill direction is plausible, but the branch still misses the model request abort signal path, changes the stable run routing key during retries, and includes an unrelated dependency override that needs separate supply-chain review.

## What This Changes

The PR exports `destroyDispatcher`, attaches SSRF fetch dispatcher teardown to abort handling, suffixes embedded retry run IDs, adds a gateway client-disconnect test, and bumps `protobufjs` in the root override and lockfile.

## Best Possible Solution

Implement dispatcher hard-kill in the network primitive using a composed fetch signal, keep `runId` stable for routing, add focused SSRF/fetch-guard regression coverage, and handle `protobufjs` through a separate dependency update or explicit maintainer approval.

## Reproduction Assessment

No high-confidence live reproduction was established by this review. Source inspection gives a credible current-main path: model abort is carried in `RequestInit.signal`, while the guarded fetch layer builds and uses a separate timeout/params signal.

## Solution Assessment

No. The hard-kill approach should compose the caller abort signal with timeout handling and preserve the stable run ID; the dependency override should be split or explicitly approved separately.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Wire teardown to the composed fetch abort signal:** `src/infra/net/fetch-guard.ts:373-375`
  - body: The teardown listener is attached to the guard signal built from `params.signal` and timeout state, but model transport aborts are carried in `RequestInit.signal` and this caller does not pass a top-level signal. Compose `currentInit.signal` with the guard/timeout signal and use that same signal for fetch and dispatcher teardown, or this can still miss the abort path the PR is meant to fix.
  - confidence: 0.88
- **[P1] Keep retries on the original runId:** `src/agents/pi-embedded-runner/run.ts:1339`
  - body: `runId` is registered and waited on as the gateway routing key, while embedded subscriptions emit the supplied value. Suffixing retry attempts with `-retry-N` makes later attempt events look like different unregistered runs, so keep `runId` stable and add a separate attempt identifier if diagnostics need one.
  - confidence: 0.91
