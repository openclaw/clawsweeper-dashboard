## Summary

Keep open: the hard-kill direction is plausible, but the patch still misses the model request abort signal, destabilizes retry event routing by changing `runId`, and includes a root dependency override change that needs security/supply-chain attention.

## What This Changes

The PR exports and uses dispatcher destruction on SSRF-guarded aborts, changes embedded retry attempt run IDs, adds a gateway abort test, and bumps the root `protobufjs` override/lockfile.

## Best Possible Solution

Keep the fix in the guarded network primitive by composing the actual fetch abort signal, preserving `runId` as the stable external routing key, adding direct dispatcher teardown coverage, and splitting the dependency upgrade unless maintainers explicitly want it bundled.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only review. Source inspection shows a credible current-main path through `RequestInit.signal`, guarded fetch timeout signal replacement, and stable `runId` routing that the PR must handle.

## Solution Assessment

No. The hard-kill primitive is plausible, but this implementation should compose the real fetch abort signal, keep `runId` stable, and separate the package override change from the gateway bug fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Compose the fetch abort signal before tearing down:** `src/infra/net/fetch-guard.ts:373-375`
  - body: The teardown listener is attached to the guard's top-level signal, but model transports carry aborts in `RequestInit.signal` and this caller does not pass `params.signal`. Compose `currentInit.signal` with the timeout/guard signal and use that same signal for both fetch and dispatcher teardown, or the abort path this PR is meant to fix can still be missed.
  - confidence: 0.88
- **[P1] Keep retry attempts on the registered runId:** `src/agents/pi-embedded-runner/run.ts:1339`
  - body: `runId` is registered and waited on as the gateway routing key, while embedded subscriptions emit the supplied value. Suffixing retry attempts with `-retry-N` makes later attempt events look like a different unregistered run, so keep `runId` stable and add a separate attempt id if diagnostics need one.
  - confidence: 0.91
