## Summary

Keep open. The socket-teardown direction is plausible and the PR includes real log proof, but the branch introduces blocking runtime-cleanup and run-id routing issues that should be fixed before merge.

## What This Changes

The PR wires guarded fetch aborts to force dispatcher teardown, exports `destroyDispatcher`, changes embedded retry `runId`s, and adds OpenAI HTTP abort coverage.

## Best Possible Solution

Keep the fix focused on per-request guarded-fetch dispatcher teardown, preserve the user-request `runId` across retries, handle async cleanup failures explicitly, and add a seam test that observes abort-time dispatcher destruction.

## Reproduction Assessment

No, not as an independently reproduced current-main failure. Source inspection shows current guarded fetch abort cleanup still goes through graceful dispatcher close, and the PR supplies live logs, but I did not reproduce the delayed Google fallback failure end to end.

## Solution Assessment

No, not as written. The network-layer teardown is a reasonable fix direction, but changing retry `runId`s and fire-and-forgetting async close fallback errors are not the narrowest maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep retry attempts on the original run id:** `src/agents/pi-embedded-runner/run.ts:1339`
  - body: The gateway and agent-event wait paths are keyed by `runId`; appending `-retry-N` makes events from later attempts use an id that clients waiting on the original request do not know. Keep `params.runId` stable and add a separate attempt id if retry disambiguation is needed.
  - confidence: 0.88
- **[P2] Catch async close fallback failures:** `src/infra/net/ssrf.ts:570`
  - body: `close` is typed here as `Promise<void> | void`, but `void candidate.close()` inside this `try` block does not catch asynchronous rejection. For wrappers without `destroy`, an abort can produce an unhandled rejection instead of the intended ignored cleanup failure.
  - confidence: 0.86
