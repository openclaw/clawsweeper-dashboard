- **[P2] Catch destroy rejections during abort cleanup:** `src/infra/net/ssrf.ts:685`
  - body: Undici's `Dispatcher.destroy()` returns a promise when called without a callback, but this helper drops that return value and the surrounding `try/catch` only covers synchronous throws. Since the PR now calls this on the normal abort path, a destroy failure can surface as an unhandled rejection during gateway abort cleanup; handle the returned promise the same way the new close fallback catches `close()` rejections.
  - confidence: 0.82
- **[P2] Assert the hard-kill path in the regression test:** `src/gateway/openai-http.test.ts:2358-2364`
  - body: The new test waits for the hanging server's request to close, which ordinary fetch cancellation can do without the new abort listener or `destroyDispatcher` call. Spy on or inject the dispatcher destroy seam, and await the guarded fetch rejection, so this test fails when the hard-kill hook is removed.
  - confidence: 0.78
