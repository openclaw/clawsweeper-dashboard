## Summary

Keep open: the guarded-fetch teardown direction is supported by current source and Undici’s dispatcher contract, but the branch still changes retry runIds in a way that can break event routing and per-run caches, and the supplied external proof is too weak for this runtime/network claim.

## What This Changes

The PR changes guarded fetch to destroy the active Undici dispatcher on abort, exports a broader dispatcher teardown helper, adds a gateway abort test, and changes embedded retry attempts to use suffixed runIds.

## Best Possible Solution

Keep the guarded-fetch abort teardown, drop the retry runId suffix, and replace the redundant gateway abort test with focused dispatcher-destruction or fallback-survival coverage plus stronger redacted real-runtime proof.

## Reproduction Assessment

No high-confidence runtime reproduction was established. Source inspection shows current main only gracefully closes guarded-fetch dispatchers and Undici supports destroy as the abrupt abort primitive, but the delayed fallback failure was not reproduced here.

## Solution Assessment

No. The network teardown approach is the right boundary, but changing retry attempt runIds is not the narrow fix and the added gateway test should observe dispatcher teardown or fallback survival rather than only client-disconnect abort propagation.
