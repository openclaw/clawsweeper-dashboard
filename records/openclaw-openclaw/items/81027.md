## Summary

Keep this PR open, but it is not ready to merge: the hard-kill dispatcher direction is plausible, while the branch still changes retry event routing and the supplied live proof does not demonstrate the claimed socket-teardown/fallback outcome.

## What This Changes

The PR exports dispatcher destruction, wires guarded fetch to destroy active dispatchers on abort, adds gateway abort coverage, syncs a node-invoke test with main, and suffixes retry attempt runIds.

## Best Possible Solution

Keep the abort hard-kill fix inside the guarded-fetch dispatcher boundary, preserve stable runId routing, and add focused regression coverage that observes dispatcher destruction or fallback survival after abort.

## Reproduction Assessment

No high-confidence runtime reproduction was established. Source inspection shows current main only performs graceful dispatcher close on guarded-fetch release, and the Undici contract supports destroy as the relevant abort primitive, but the full delayed fallback failure was not reproduced.

## Solution Assessment

No. The network teardown approach is directionally right, but changing retry attempt runId is not the narrow fix and the regression test should exercise dispatcher teardown rather than duplicate client-disconnect abort propagation.
