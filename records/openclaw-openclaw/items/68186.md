## Summary

Keep open: current main still has the macOS screen.snapshot validation and payload-bound bug, and this PR is the active fix path, but the branch still has a near-limit false-rejection bug, lacks real macOS node/gateway proof, and is currently conflicting.

## What This Changes

The PR hardens macOS screen.snapshot by rejecting malformed params, sanitizing snapshot errors, adding inline frame-size guards with nodeId plumbing, regression tests, and a changelog entry.

## Best Possible Solution

Repair the coarse raw precheck so in-budget payloads reach the projected guard, add an accepted near-limit regression, then require redacted real macOS node/gateway proof and focused Swift checks before merge.

## Reproduction Assessment

Yes by source inspection. Current main falls back from malformed screen.snapshot params to defaults and emits inline base64 without a size guard; the PR-head false rejection is also shown by the raw precheck arithmetic.

## Solution Assessment

No, not yet. The fail-closed and frame-bounding direction is maintainable, but the raw precheck should not reject frames that the later projection would accept, and the external PR still needs real behavior proof.
