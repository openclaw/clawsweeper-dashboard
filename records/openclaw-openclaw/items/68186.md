## Summary

Keep open. The PR is the active fix path for a real current-main macOS screen.snapshot bug, but the latest head still introduces a near-limit false rejection in the raw payload precheck and only provides test/mock proof rather than real macOS node behavior proof.

## What This Changes

The PR hardens macOS screen.snapshot by rejecting malformed params, classifying snapshot errors, bounding inline result frames with request/node id accounting, adding Swift regression tests, and updating the changelog.

## Best Possible Solution

Keep this PR as the active fix path, adjust the payload guard so near-limit in-budget frames are not rejected before precise sizing, add a focused near-limit regression test, and require redacted real macOS node/gateway proof before merge.

## Reproduction Assessment

Yes by source inspection. Current main still falls back from malformed params to default snapshot params and emits res.data as inline base64 JSON without a size guard; the PR-head regression is also source-reproducible from the raw precheck and 25 MiB frame arithmetic.

## Solution Assessment

No, not yet. The hardening direction is narrow and maintainable, but the raw precheck should use a true base64-only cutoff or be removed so the frame-size calculation and regression tests decide near-limit payloads.
