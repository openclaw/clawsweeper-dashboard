## Summary

Keep open. This PR targets a real current-main macOS screen.snapshot bug and is the active closing fix path, but it still needs real behavior proof and the PR head still over-rejects some near-limit snapshot frames that fit under the gateway payload limit.

## What This Changes

This PR hardens macOS screen.snapshot by rejecting malformed params, sanitizing and classifying snapshot errors, bounding inline result frame sizes with request/node id accounting, plumbing nodeId through the bridge request, adding Swift regression tests, and updating the changelog.

## Best Possible Solution

Keep this PR as the active fix path, repair the near-limit size guard with an accepted in-budget regression case, require redacted real macOS node or gateway proof, then rebase and run focused Swift checks before merge.

## Reproduction Assessment

Yes by source inspection. Current main still falls back from malformed screen.snapshot params to defaults and emits inline base64 payloadJSON without a size guard; the PR-head near-limit regression is also source-reproducible from the raw precheck arithmetic.

## Solution Assessment

No, not yet. The fail-closed and frame-bounding direction is maintainable, but the size guard should be based on the actual serialized frame or a calibrated precheck/reserve, and the external PR still needs real behavior proof.
