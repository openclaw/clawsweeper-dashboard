## Summary

Keep open: current main still has the reported macOS screen.snapshot bug and this PR is the active same-author fix path, but PR head still rejects some in-budget near-limit snapshots and only has mock/test-level behavior proof.

## What This Changes

The PR hardens macOS screen.snapshot by rejecting malformed params, sanitizing and classifying snapshot errors, adding id/nodeId-aware inline frame size checks, plumbing nodeId through bridge invoke requests, adding Swift regression coverage, and updating the changelog.

## Best Possible Solution

Keep this PR as the active fix path, repair the near-limit size guard with an accepted in-budget regression case, require redacted real macOS node or gateway proof, then run focused Swift checks before merge.

## Reproduction Assessment

Yes by source inspection. Current main still falls back from malformed screen.snapshot params to defaults and emits inline base64 payloadJSON without a size guard; the PR-head near-limit regression is also reproducible from the raw precheck arithmetic.

## Solution Assessment

No, not yet. The fail-closed and frame-bounding direction is maintainable, but the raw precheck should not reject frames that the later projected-size check would accept, and the external PR still needs real behavior proof.
