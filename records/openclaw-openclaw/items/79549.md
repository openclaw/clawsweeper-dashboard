## Summary

Keep open: the latest commit addressed the gateway-auth unsetPaths case, but the patch still flips a shared config-write primitive while existing channel remove/logout callers delete config or credential fields without passing unsetPaths, and the PR still lacks real after-fix behavior proof.

## What This Changes

The PR changes config merge-patch generation to preserve absent keys, adds explicit unsetPaths deletion handling in config write paths, forwards gateway auth credential unsets, and adds config persistence regression tests.

## Best Possible Solution

Keep the preservation-by-default goal, but make deletion intent reliable across all existing mutation, channel remove, and logout callers, with focused regressions and redacted real CLI or Docker restart proof.

## Reproduction Assessment

Yes. Source inspection on current main shows omitted target keys become null merge-patch deletes, and the PR’s regression tests describe the partial-write path; I did not run tests because this review is read-only.

## Solution Assessment

No as currently patched. Explicit unsetPaths is a maintainable direction, but the branch only migrates some deletion paths and leaves existing channel remove/logout callers able to preserve stale fields.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve delete intents for existing config callers:** `src/config/io.write-prepare.ts:31`
  - body: Skipping omitted target keys makes every deletion depend on unsetPaths, but existing callers still delete fields and call replaceConfigFile without writeOptions. For example, channel remove uses plugin.config.deleteAccount, and Telegram/Line/Nextcloud logout delete credential fields before replaceConfigFile; under this PR those omitted fields can be restored from the source snapshot, so commands can report deletion while the file keeps old config or secrets.
  - confidence: 0.9

## Security Review

Status: needs_attention

That can leave credentials in openclaw.json after the UI or CLI reports they were removed.
