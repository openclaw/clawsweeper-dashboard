## Summary

Keep open: the PR fixes a real config clobbering bug directionally, but the new omitted-key preservation contract still leaves deletion callers able to restore removed channel/account config unless they pass unsetPaths. It also still lacks required after-fix real behavior proof.

## What This Changes

The PR changes config merge-patch writes to preserve omitted keys by default, adds explicit unsetPaths deletion handling, forwards gateway auth credential unsets, and adds config persistence regression tests.

## Best Possible Solution

Keep the preservation-by-default direction, but make deletion intent reliable either by deriving explicit unsetPaths for full-config mutations or migrating every delete/logout/remove caller with focused regressions and real CLI/Docker proof.

## Reproduction Assessment

Yes from source inspection: current main emits null-delete merge patches for keys omitted from a narrow nextConfig, and the PR's red tests encode the ACP deliveryMode loss. I did not run tests because this review was read-only.

## Solution Assessment

No as currently patched: explicit unsetPaths is a maintainable direction, but the patch must cover channel/account deletion callers or provide a generic deletion tracker before merge.
