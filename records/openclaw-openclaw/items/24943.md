## Summary

Keep open: current main has partial iMessage formatting support for simple emphasis markers, but general Markdown such as headings, links, list markers, and code spans can still reach plain-text iMessage delivery. The broader shared fix remains represented by the open implementation candidate at https://github.com/openclaw/openclaw/pull/56283.

## What This Changes

Request to convert or strip Markdown for iMessage/plain-text sends so recipients do not see raw Markdown markers.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/56283 with a shared outbound Markdown policy that strips only for true plain-text delivery after existing sanitization, preserves Markdown-capable channels, and covers direct sends, monitor replies, captions, and config overrides.

## Reproduction Assessment

Yes, source-level. A reply containing headings, Markdown links, list markers, code spans, or fenced blocks can still pass through current iMessage/plain-text delivery without a general Markdown-to-plain-text pass.

## Solution Assessment

No complete solution is on main yet. The current iMessage typed-run converter is useful but partial; the maintainable solution is the shared outbound policy represented by the open implementation candidate or an equivalent replacement.
