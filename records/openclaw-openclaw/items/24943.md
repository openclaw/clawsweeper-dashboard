## Summary

Keep open: current main and v2026.5.19 still lack a general Markdown-to-plain-text policy for iMessage/plain-text outbound delivery. Current code handles Markdown tables and simple iMessage style runs only, while the shared implementation candidate at https://github.com/openclaw/openclaw/pull/56283 is still open and needs rebase/product direction.

## What This Changes

Request to convert or strip Markdown from iMessage/plain-text outbound replies so raw Markdown markers are not delivered to recipients.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/56283 with a shared outbound Markdown policy that strips only for true plain-text delivery after existing sanitization, preserves renderer-capable channels, and covers direct sends, monitor replies, captions, and account-scoped config.

## Reproduction Assessment

Yes, source-level. A reply containing headings, Markdown links, list markers, code spans, or fenced blocks can still pass through iMessage delivery without a general Markdown-to-plain-text pass; current code only handles tables and simple style markers.

## Solution Assessment

No complete solution is on main yet. The shared-policy direction is the maintainable shape, but it needs rebase or replacement work that preserves renderer-owned channels and settles the default/config policy.
