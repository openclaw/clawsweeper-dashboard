## Summary

Keep open: current main partially strips iMessage emphasis markers and converts Markdown tables, but it still lacks the requested general Markdown-to-plain-text handling for headings, code spans/fences, links, lists, and shared plain-text delivery.

## What This Changes

Request capability-aware outbound handling that strips or converts Markdown before sending replies through iMessage and other non-Markdown channels.

## Best Possible Solution

Review, rebase, or replace the shared outbound text policy so non-Markdown channels get clean plain text while Markdown-capable and lightweight-format channels keep their intended rendering.

## Reproduction Assessment

Yes, for source-level verification. Text containing headings, links, list markers, or backticks still flows through sendMessageIMessage without a general Markdown stripping pass, while only tables and emphasis-style markers are handled.

## Solution Assessment

No complete solution is on main yet. The current iMessage formatting-run support is useful but narrower than the requested capability-aware plain-text policy, and the open shared-delivery approach still needs maintainer review for channel compatibility.
