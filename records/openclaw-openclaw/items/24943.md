## What This Changes

The issue requests converting or stripping Markdown before iMessage sends so raw syntax is not shown in Messages.

## Best Possible Solution

Land a rebased or replacement shared outbound Markdown policy that strips only for true plain-text iMessage delivery after existing sanitization, preserves renderer-owned channels, and covers direct sends, monitor replies, media captions, config behavior, and empty-after-strip cases.

## Solution Assessment

No complete solution is on main yet. The shared-policy direction in https://github.com/openclaw/openclaw/pull/56283 is the right shape, but it needs rebase or replacement work that preserves renderer-owned channels and settles the config/default policy.
