## Summary

Keep open because current main still does not strip general Markdown for iMessage/plain-text delivery, and the referenced open PR remains the active implementation candidate rather than merged code.

## What This Changes

Request outbound iMessage/plain-text channel delivery to strip or convert Markdown markers so recipients see readable plain text.

## Best Possible Solution

Land PR #56283 or an equivalent shared outbound Markdown-to-plain-text normalization stage for non-Markdown-capable channels, with focused coverage for direct iMessage sends, monitor replies, captions, Markdown-capable channel preservation, and empty-after-strip handling.

## Solution Assessment
