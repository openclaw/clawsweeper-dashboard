## Summary

Keep open: current main and v2026.5.12 still lack a general Markdown-to-plain-text pass for iMessage/plain-text delivery, and the shared implementation candidate remains open and conflicting at https://github.com/openclaw/openclaw/pull/56283.

## What This Changes

Request to strip or convert Markdown before iMessage/plain-text channel sends so users do not see raw markers like headings, backticks, links, and list syntax.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/56283 with a shared outbound Markdown policy that strips only for true plain-text delivery after existing sanitization and covers iMessage direct sends, monitor replies, captions, config behavior, and renderer-preservation cases.

## Reproduction Assessment

Yes, source-level. A reply containing headings, Markdown links, list markers, code spans, or fenced blocks can still pass through sendMessageIMessage without a general Markdown-to-plain-text pass; current code only handles tables and simple style markers.

## Solution Assessment

No complete solution is on main yet. The shared-policy direction is the right shape, but it needs rebase or replacement work that preserves renderer-owned channels and settles the default/config policy.
