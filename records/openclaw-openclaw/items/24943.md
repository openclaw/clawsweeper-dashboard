## Summary

Keep open. Current main still lacks general Markdown-to-plain-text stripping for legacy iMessage delivery, and the remaining shared fix is represented by open, unmerged PR #56283 rather than shipped code.

## What This Changes

Request that outbound iMessage replies strip or convert Markdown formatting before sending because iMessage displays raw Markdown characters as plain text.

## Best Possible Solution

Review, land, or replace PR #56283 with an equivalent shared outbound Markdown stripping stage for non-Markdown-capable/plain-text channels, applied after existing HTML/plain-text sanitization and covered by direct iMessage, monitor reply, media caption, config override, Markdown-capable channel preservation, and empty-after-strip tests.
