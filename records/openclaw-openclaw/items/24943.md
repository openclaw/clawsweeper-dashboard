## Summary

Keep open: current main only strips/converts a narrow subset of iMessage Markdown into typed-run formatting, while headings, links, list markers, and code spans can still pass through plain-text delivery. The central shared fix is still represented by the open implementation candidate at https://github.com/openclaw/openclaw/pull/56283.

## What This Changes

The issue requests that iMessage/plain-text delivery convert or strip Markdown before sending bot replies.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/56283 with a shared outbound policy that strips Markdown only for true plain-text delivery after existing sanitization, preserves Markdown-capable channels, and covers direct sends, monitor replies, media captions, and override behavior.

## Reproduction Assessment

Yes, source-level: a reply containing headings, Markdown links, list markers, code spans, or fenced blocks can still pass through current iMessage/plain-text delivery without a general Markdown-to-plain-text pass. I did not run a live iMessage send in this read-only review.

## Solution Assessment

No complete solution is on main yet. The current typed-run converter is useful but partial; the maintainable solution is the shared outbound policy represented by the open implementation candidate or an equivalent replacement.
