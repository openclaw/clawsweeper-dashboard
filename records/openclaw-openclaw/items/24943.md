## Summary

Keep open because current main still lacks general Markdown stripping for iMessage/plain-text delivery, and the active shared implementation candidate remains open rather than merged.

## What This Changes

Request that iMessage delivery convert or strip Markdown markers before sending so recipients do not see raw formatting syntax.

## Best Possible Solution

Review and land PR #56283 or an equivalent shared outbound Markdown-to-plain-text normalization stage for non-Markdown-capable channels, with Markdown-capable channels preserved.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: send text such as `**bold**` through current iMessage delivery and it reaches `sendMessageIMessage` after table conversion and directive cleanup without a general Markdown stripping pass.

## Solution Assessment

Yes for the proposed direction, but it is not implemented on main yet. A shared outbound stage after existing HTML/plain-text cleanup is safer than patching only direct iMessage sends because monitor replies, captions, and other plain-text channels share the same delivery boundary.
