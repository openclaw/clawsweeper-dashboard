## Summary

Keep open: current main only handles a narrow subset of iMessage Markdown by converting tables and extracting simple emphasis/style runs; general headings, links, lists, code/backticks, and shared plain-text-channel policy remain unresolved. The active implementation path is still https://github.com/openclaw/openclaw/pull/56283, which is open and conflicting, so closing this tracker now would be premature.

## What This Changes

Request that OpenClaw strip or convert Markdown to plain text before sending bot replies through the iMessage channel.

## Best Possible Solution

Land a rebased or replacement shared outbound Markdown policy that strips for true plain-text iMessage delivery after existing sanitization, while preserving Markdown-capable and lightweight-format renderers with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection shows headings, links, lists, and code/backticks can still reach sendMessageIMessage without a general Markdown-to-plain-text pass; only tables and simple emphasis-style runs are handled on current main.

## Solution Assessment

No complete solution is on main yet. The linked shared-policy direction is the right shape, but it needs rebase or replacement work that preserves renderer-owned channels and settles the channel-formatting policy.
