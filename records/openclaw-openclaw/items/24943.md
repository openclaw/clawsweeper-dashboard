## Summary

Keep open: current main and v2026.5.19 still lack a general Markdown-to-plain-text pass for iMessage/plain-text outbound delivery. The current iMessage path only handles Markdown tables plus simple bold/italic/underline/strikethrough markers, while the broader shared implementation remains open at https://github.com/openclaw/openclaw/pull/56283.

## What This Changes

Request automatic stripping or conversion of Markdown syntax before sending replies through iMessage/plain-text delivery.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/56283 with a shared outbound Markdown policy that strips only for true plain-text delivery after existing sanitization, preserves renderer-owned channels, and covers direct sends, monitor replies, captions, and account-scoped config.

## Reproduction Assessment

Yes, source-level. A reply containing headings, Markdown links, list markers, code spans, or fenced blocks can still pass through sendMessageIMessage without a general Markdown-to-plain-text pass.

## Solution Assessment

No complete solution is on main yet. The shared-policy direction is the maintainable shape, but it needs rebase or replacement work that preserves renderer-owned channels and settles default/config policy.
