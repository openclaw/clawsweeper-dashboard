## Summary

Keep open. Current main still lets general Markdown markers pass through legacy iMessage delivery, and open PR #56283 remains the canonical implementation candidate rather than merged code.

## What This Changes

Strip or convert Markdown to plain text for iMessage and other plain-text outbound delivery so users do not see raw formatting markers.

## Best Possible Solution

Land PR #56283 or an equivalent shared outbound Markdown stripping stage for non-Markdown-capable channels, with tests for direct iMessage sends, monitor replies, captions, channel preservation, and empty-after-strip handling.

## Reproduction Assessment

Yes. A source-level reproduction is clear: send text like `**bold**` through sendMessageIMessage with a mocked client, and current main passes it through table conversion and directive cleanup without removing Markdown markers before `client.request("send", params)`.

## Solution Assessment

Yes. A shared outbound normalization stage is the best direction because an iMessage-only send.ts patch would miss monitor/caption/shared plain-text delivery behavior and could diverge from other non-Markdown-capable channels.
