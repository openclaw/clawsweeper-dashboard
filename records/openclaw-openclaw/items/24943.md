## What This Changes

The issue requests converting or stripping Markdown formatting before sending iMessage replies over a plain-text transport.

## Best Possible Solution

Review and land PR #56283 or an equivalent shared outbound Markdown-to-plain-text normalization stage for non-Markdown-capable channels.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: send text like `**bold**` through current iMessage delivery, and it reaches `sendMessageIMessage`/client send parameters after table conversion and directive cleanup without general Markdown stripping.

## Solution Assessment

Yes. A shared outbound normalization stage is the best fit because an iMessage-only patch would miss monitor replies, captions, and other non-Markdown-capable channels that use the same plain-text delivery boundary.
