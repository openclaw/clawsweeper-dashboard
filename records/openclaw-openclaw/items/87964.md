Full review comments:

- **[P1] Route structured file paths through the media loader:** `extensions/mattermost/src/mattermost/send.ts:553-558`
  - body: The attachments[].filePath branch calls opts.mediaReadFile directly and uploads those bytes. That skips the shared loadOutboundMediaFromUrl path that applies localRoots and host-read policy, so structured attachment paths can bypass the safety boundary.
  - confidence: 0.91
- **[P2] Fail top-level file path load failures instead of posting text:** `extensions/mattermost/src/channel.ts:279-282`
  - body: Mapping params.path and params.filePath into mediaUrl sends them through the existing mediaUrl fallback. If the local media load fails while the message has text, send.ts can still post text with no file_ids, so the reported false-success behavior remains for that failure case.
  - confidence: 0.86
- **[P2] Normalize every advertised attachment alias:** `extensions/mattermost/src/channel.ts:301-319`
  - body: The shared send schema accepts attachments[].mediaUrl, fileUrl, url, and name, but this mapper only forwards media, path/filePath, buffer, filename, and contentType/mimeType. Those advertised aliases can still be silently omitted from Mattermost uploads.
  - confidence: 0.82
