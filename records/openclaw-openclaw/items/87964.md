Full review comments:

- **[P1] Route attachment file paths through the shared loader:** `extensions/mattermost/src/mattermost/send.ts:550`
  - body: This branch calls `mediaReadFile(att.filePath)` directly for structured attachments. That skips the shared outbound media loader's localRoots check, size cap, MIME/sniffing, and host-read validation used by the existing media path, so a Mattermost tool send can cross the local media boundary differently from every other media upload path. Please load filePath/path/mediaUrl attachments through `loadOutboundMediaFromUrl` with the existing mediaLocalRoots/mediaReadFile options.
  - confidence: 0.88
- **[P1] Restore the imports that send.ts still uses:** `extensions/mattermost/src/mattermost/send.ts:2`
  - body: The new import block only imports `resolveMarkdownTableMode`, but this file still uses `MessageReceipt`, `MessageReceiptPartKind`, `createMessageReceiptFromOutboundResults`, and `requireRuntimeConfig`. As written, the Mattermost extension cannot typecheck/compile; keep the original SDK imports or add the missing imports from their existing subpaths.
  - confidence: 0.97
- **[P1] Preserve the existing Mattermost outbound adapter:** `extensions/mattermost/src/channel.ts:672`
  - body: Replacing `outbound: mattermostOutbound` with a new base/attachedResults object drops `presentationCapabilities`, `renderPresentation`, `sendPayload`, and the deliveryCapabilities that current Mattermost outbound tests rely on for normal reply-payload button delivery. Please extend or reuse `mattermostOutbound` so this attachment fix does not regress existing Mattermost presentation sends.
  - confidence: 0.9
- **[P2] Pass cfg into the new send tests:** `extensions/mattermost/src/mattermost/send.test.ts:713-718`
  - body: The newly added `sendMessageMattermost` tests call the function without the required `cfg` option, while the implementation requires runtime config before resolving the account. Once the missing import is restored, these cases will fail before exercising uploads; add `cfg: TEST_CFG` to the new calls.
  - confidence: 0.86
- **[P2] Handle the advertised attachment aliases:** `extensions/mattermost/src/channel.ts:299-305`
  - body: The shared message tool advertises structured attachment keys like `mediaUrl`, `fileUrl`, `url`, and `name`, but this adapter only reads `media`, `path`/`filePath`, `buffer`, and `filename`. A send with an advertised alias can still drop the intended attachment while returning ok for the text message; normalize all documented aliases or fail loudly for unsupported attachment entries.
  - confidence: 0.8
