## Summary

Keep open. Current main and the latest release still intentionally fail closed unless `gateway.mode=local` is configured or `--allow-unconfigured` is passed; the no-provider WebUI crash and Raw-mode complaint still need concrete reproduction before a narrow fix can be scoped.

## What This Changes

The issue asks OpenClaw to let Gateway and Control UI start before all settings and model providers are configured, and not crash when no provider exists.

## Best Possible Solution

Define a reviewed first-run startup contract: either a loopback-only bootstrap Control UI with explicit security bounds, or keep fail-closed startup and improve setup/diagnostic messaging while fixing separately reproduced UI crashes.

## Reproduction Assessment

Partial. The Gateway guard is source-reproducible from current main and v2026.5.2, but the alleged no-provider WebUI crash and Raw-mode failure lack route, config, log, or browser-console steps.

## Solution Assessment

Unclear. Broadly relaxing startup would change an intentional fail-closed guard, so the safer path is maintainer-approved bootstrap policy plus targeted fixes once the UI claims have reproductions.
