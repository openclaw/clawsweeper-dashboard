## Summary

Keep open. Current main still intentionally fail-closes normal Gateway startup without `gateway.mode=local` while offering `--allow-unconfigured` as a bootstrap escape hatch; the remaining no-provider WebUI crash and Raw-mode claims need focused reproduction before a narrow code fix is safe.

## What This Changes

Requests that Gateway and Control UI start without requiring complete config or model/provider setup, and reports an unspecified no-provider crash plus Raw-mode failure.

## Best Possible Solution

Define a reviewed startup contract: either add a loopback-only bootstrap state that keeps Control UI reachable without provider setup, or keep fail-closed startup and improve setup/repair messaging while fixing separately reproduced UI crashes.

## Reproduction Assessment

Partial. The Gateway policy behavior is source-reproducible from current main: ordinary startup blocks missing `gateway.mode`, while `--allow-unconfigured` starts the bootstrap path; the WebUI no-provider crash and Raw-mode failure are not yet reproducible from the report.

## Solution Assessment

Unclear. The requested broad relaxation is not proven to be the narrowest maintainable fix because it changes an intentional startup guard; a safer path is a maintainer-approved bootstrap contract plus targeted UI fixes for reproduced crashes.
