## Summary

Keep open. Current main still blocks normal Gateway startup when `gateway.mode` is missing, so the central no-config startup request is not implemented; the no-provider WebUI and Raw-mode claims need a narrower reproduction.

## What This Changes

The issue asks for Gateway and WebUI startup to work without all configuration, models, or providers being set up.

## Best Possible Solution

Define a reviewed startup contract: either add a loopback-only bootstrap state that keeps Control UI reachable without providers, or keep fail-closed startup and improve setup/repair messaging plus targeted UI fixes for reproduced crashes.

## Reproduction Assessment

Partial. The Gateway policy portion is reproducible from source and tests: normal startup blocks missing `gateway.mode`, while `--allow-unconfigured` covers a bootstrap path; the no-provider WebUI crash and Raw-mode failure do not yet have enough concrete data for high-confidence reproduction.

## Solution Assessment

Unclear. The requested broad relaxation is not yet the narrowest maintainable solution because it changes an intentional Gateway safety guard; a safer path is a maintainer-approved bootstrap mode plus separate fixes for any reproduced UI crash.
