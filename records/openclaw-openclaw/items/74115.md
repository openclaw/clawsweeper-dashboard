## Summary

Keep open. Current main still intentionally blocks normal Gateway startup without `gateway.mode=local`; the inspected model/config UI paths are defensive, but the no-provider WebUI crash and Raw-mode claims need narrower reproduction before a safe fix can be scoped.

## What This Changes

The issue requests that Gateway and WebUI startup not require full provider/model configuration, and reports crashes or broken Raw mode when provider setup is missing.

## Best Possible Solution

Define a reviewed startup contract: either add a loopback-only bootstrap state that keeps Control UI reachable without providers, or keep fail-closed startup and improve setup/repair messaging plus targeted UI fixes for reproduced crashes.

## Reproduction Assessment

Partial. The Gateway policy portion is reproducible from source and tests: normal startup blocks missing `gateway.mode`, while `--allow-unconfigured` covers a bootstrap path; the no-provider WebUI crash and Raw-mode failure still lack concrete reproduction data.

## Solution Assessment

Unclear. Broadly removing the startup guard is not yet the narrowest maintainable solution because it changes a deliberate safety policy; a maintainer-approved bootstrap mode plus separate fixes for reproduced UI crashes would be safer.
