## Summary

Keep open. Current main still encodes the normal Gateway startup guard as intentional policy, while the no-provider WebUI crash and Raw-mode complaint remain too underspecified for a confident close or narrow automated fix.

## What This Changes

Requests that Gateway and Control UI start without requiring complete settings or configured model providers, and adds that Raw mode is not working.

## Best Possible Solution

Maintainers should define the intended bootstrap contract explicitly: either allow safe loopback Gateway/Control UI startup with no config or provider, or keep the existing guard and make setup/recovery UX clearer. Any verified no-provider Control UI crash or Raw-mode regression should then be split into a focused bug with config, route, logs, and a regression test.

Unclear. The Gateway policy portion is source- and test-reproducible from current main, but the WebUI no-provider crash and Raw-mode failure do not have enough steps, logs, or config state to reproduce with high confidence.

## Solution Assessment
