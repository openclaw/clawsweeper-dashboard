## Summary

Keep open: the branch has useful source-level hardening for message appends, but it still publishes ownership only through the message-persist hook while sibling runtime-owned session entries write the same JSONL through separate paths.

## What This Changes

The branch replaces embedded session-lock message-persist fence refresh with a pre-append fingerprint-gated owned-write publish hook and adds regression coverage for message append interleavings.

## Best Possible Solution

Use one ownership publication contract for every OpenClaw-owned session/transcript write during the released prompt window, or pause this PR in favor of the SQLite/accessor migration that removes the file-fingerprint fence.

## Reproduction Assessment

Yes, source-level. The PR proof demonstrates the message-append trust gate, and current source plus later PR comments show non-message session entries can still advance the same file outside that hook; I did not run a fresh live repro in this read-only review.

## Solution Assessment

No as currently proposed. The pre-append message gate is a useful stopgap, but it is not the best complete fix unless the same ownership contract covers sibling session-entry writes or maintainers explicitly defer the class to the SQLite migration.
