## What This Changes

The PR replaces embedded session-lock post-message unconditional fence refresh with a pre-append fingerprint-gated owned-write publish hook and adds regression tests around that message persistence path.

## Best Possible Solution

Use one ownership publication contract for every OpenClaw-owned session/transcript write during the released prompt window, or explicitly pause this stopgap in favor of the SQLite/accessor migration that removes the file-fingerprint fence.
