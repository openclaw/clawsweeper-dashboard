## Summary

Keep this PR open: it targets a real embedded-session false takeover path, but the redesigned owned-write publish path can still mark a mixed external-plus-pi write as owned and weaken the takeover fence.

## What This Changes

The PR adds `publishOwnedPostMessageWrite()` to the embedded attempt session-lock controller, wires it from `runEmbeddedAttempt`'s `onMessagePersisted` callback, and adds session-lock regression tests.

## Best Possible Solution

Keep the false-positive fix, but record pi-owned writes from a pre-append baseline captured immediately around the actual session-manager append and add a negative regression for external append before the pi append/publish path.

## Reproduction Assessment

Yes. Current source shows `onMessagePersisted` only refreshes the local fence, and the PR body includes a terminal harness showing the false takeover before the patch and the owned-write success after it.

## Solution Assessment

No. The callback is the right area, but recording the post-write fingerprint without proving the pre-append baseline preserves a fail-open hole for external mutations that arrive before the owned append.
