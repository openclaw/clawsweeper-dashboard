## Summary

Keep open: this PR addresses a real Telegram native-command auth gap that is still present on current main and v2026.6.6, and the new Mantis proof now covers the visible Telegram `/status` path. It is still a draft authorization-boundary change, so it needs maintainer acceptance before merge.

## What This Changes

The PR lets Telegram native commands trust a resolved open group/topic policy when `commands.allowFrom.telegram` is unset, keeps configured command allowlists authoritative, and adds open-group regression coverage.

## Best Possible Solution

Land this narrow Telegram-owned fix after maintainers explicitly accept open group policy as native-command authorization, while preserving configured `commands.allowFrom.telegram`, disabled policy, and disallowed group blocks.

## Reproduction Assessment

Yes. Source inspection shows current main and v2026.6.6 can pass Telegram group policy and still hit the later allowlist-only command gate; the Mantis before/after proof also exercises the visible `/status` path.

## Solution Assessment

Yes. The PR is the narrowest maintainable fix I found: reuse the existing Telegram group-policy evaluator, honor scoped overrides, and keep `commands.allowFrom.telegram` authoritative when configured.
