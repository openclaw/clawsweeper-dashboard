## Summary

Keep open; the patch is not merge-ready because the new authorization shortcut still keys off unscoped Telegram group policy, so documented per-group/per-topic open configurations can remain blocked, and the supplied proof is only synthetic harness output.

## What This Changes

The PR adds a Telegram native-command authorization shortcut for open group policy and a group-auth regression test for `/status` without sender allowlists.

## Best Possible Solution

Make native command auth use the same scoped group/topic policy resolution as normal Telegram messages, preserve `commands.allowFrom` as the sole command auth source when configured, and include live Telegram proof for `/status` in the target open-group setup.

## Reproduction Assessment

Yes from source inspection, not from a live Telegram run. The group policy helper only reads nested group/topic overrides when `useTopicAndGroupOverrides` is true; normal messages set true, while native command auth and the PR shortcut rely on the false path.

## Solution Assessment

No. The PR is only a top-level-policy fix; the maintainable fix is to have native command authorization consume the same scoped policy decision normal Telegram messages already use, with regression coverage for global, group, and topic overrides.
