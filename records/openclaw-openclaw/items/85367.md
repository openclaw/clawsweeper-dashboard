Full review comments:

- **[P2] Require a session key before saving running state:** `ui/src/ui/controllers/workboard.ts:913-915`
  - body: `startWorkboardCard` still patches `status: "running"` when `sessions.create` returns no usable `key`. The existing Sessions controller treats a missing key as an error, so this can leave a card running with no linked handoff target.
  - confidence: 0.9
- **[P2] Use lifecycle live detection for card actions:** `ui/src/ui/views/workboard.ts:616`
  - body: `renderCard` sets `live` only from `session.hasActiveRun === true`, while Workboard lifecycle also treats `session.status === "running"` as active. Rows that only report running status hide the live badge and Stop action even though the card lifecycle says the run is active.
  - confidence: 0.88
- **[P2] Offer restart when the linked session is missing:** `ui/src/ui/views/workboard.ts:679-707`
  - body: The action branch checks only whether a card has a stored session key, so a deleted or expired linked session renders Open session and never Start. Since lifecycle already classifies that state as missing, the card needs a relink/restart path instead of a dead session target.
  - confidence: 0.87
- **[P2] Preserve explicit zero positions:** `extensions/workboard/src/store.ts:280-283`
  - body: `normalizePosition(input.position, 0) || ...` discards an intentional `position: 0` and remaps the card to the end of the column. Importers or callers that pin a card to the top cannot persist deterministic ordering.
  - confidence: 0.84
- **[P2] Disable Workboard writes without operator.write:** `ui/src/ui/views/workboard.ts:579`
  - body: The Workboard view receives `canWrite`, but the start/edit/delete/new/drop actions still call write-scoped RPCs when that prop is false. Read-only operators will see actionable controls that reliably fail at the gateway instead of a read-only board state.
  - confidence: 0.86
- **[P2] Gate session capture before wiring the action:** `ui/src/ui/app-render.ts:2113-2123`
  - body: `onAddToWorkboard` is exposed whenever the plugin is enabled, without checking operator.write. That button calls `captureSessionToWorkboard` and then `workboard.cards.create`, which the plugin correctly registers under `operator.write`, so read-only operators get a write-denied click path.
  - confidence: 0.9
- **[P2] Wait for plugin enablement before force-loading Workboard:** `ui/src/ui/app-settings.ts:427-438`
  - body: The Workboard tab refresh starts `loadConfig` and `loadWorkboard({ force: true })` in parallel. For a default-off plugin this can issue a guaranteed failing list call before config proves enablement and mark the board as attempted, leaving same-session enablement dependent on a manual forced refresh.
  - confidence: 0.82
