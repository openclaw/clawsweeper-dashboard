## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Require a session key before marking cards running:** `ui/src/ui/controllers/workboard.ts:756-758`
  - body: `startWorkboardCard` still updates the card to `status: "running"` even when `sessions.create` returns no usable `key`. The existing sessions controller treats a missing key as an error, so this can leave a card running without any linked session or handoff target.
  - confidence: 0.9
- **[P2] Use lifecycle live detection for card actions:** `ui/src/ui/views/workboard.ts:553`
  - body: `renderCard` derives `live` only from `session.hasActiveRun === true`, while `getWorkboardLifecycle` also treats `session.status === "running"` as active. Rows that report running status without `hasActiveRun` show a running lifecycle but hide the live badge and Stop action, leaving operators unable to abort from Workboard.
  - confidence: 0.88
- **[P2] Offer restart when the linked session is missing:** `ui/src/ui/views/workboard.ts:615`
  - body: The action branch checks only `card.sessionKey`, so a card whose linked session was deleted or expired renders Open session and never Start. Since lifecycle already classifies that state as missing, the card should offer a relink/restart path instead of sending users to a dead session key.
  - confidence: 0.87
- **[P2] Preserve explicit zero positions:** `extensions/workboard/src/store.ts:172-175`
  - body: `normalizePosition(input.position, 0) || ...` discards an intentional `position: 0` and remaps the card to the end of the column. Any gateway caller that imports or pins cards with deterministic top ordering cannot persist that ordering.
  - confidence: 0.84
