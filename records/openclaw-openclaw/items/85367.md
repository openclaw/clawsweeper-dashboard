## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Require a created session key before marking cards running:** `ui/src/ui/controllers/workboard.ts:752-759`
  - body: `startWorkboardCard` still updates the card to `status: "running"` even when `sessions.create` returns no usable `key`. That leaves a running card with no linked session and no navigable handoff target while the function returns `null`, so the card state says work is active even though Workboard cannot open it.
  - confidence: 0.9
- **[P2] Preserve explicit zero positions on card creation:** `extensions/workboard/src/store.ts:172-175`
  - body: `normalizePosition(input.position, 0) || ...` treats an intentional `position: 0` as absent and replaces it with the computed tail position. Any importer or caller trying to pin a new card at the top of a column cannot persist that ordering.
  - confidence: 0.87
- **[P2] Use lifecycle live state for card actions:** `ui/src/ui/views/workboard.ts:557`
  - body: The card action state only checks `session.hasActiveRun === true`, while `getWorkboardLifecycle` treats `session.status === "running"` as active too. Session rows that carry `status: "running"` without `hasActiveRun` will render as running but hide the live badge and Stop action.
  - confidence: 0.84
- **[P2] Offer restart when a linked session is missing:** `ui/src/ui/views/workboard.ts:619-647`
  - body: The action branch keys only on `card.sessionKey`, so a card whose linked session was deleted or expired shows Open session for a dead key and never offers Start session. That leaves lifecycle `missing` cards without an in-place way to relink and continue work.
  - confidence: 0.86
- **[P2] Avoid using stale runId after relinking cards:** `ui/src/ui/controllers/workboard.ts:785-788`
  - body: Stopping a card always forwards the persisted `runId` when one exists, but editing can change `sessionKey` without clearing that old run id. In that state `chat.abort` can miss the active run for the new session; clear the run id on relink or retry/fall back to session-level abort when the run-id abort reports no run.
  - confidence: 0.82
