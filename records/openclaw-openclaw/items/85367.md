Full review comments:

- **[P2] Classify the Workboard runtime API barrel:** `scripts/lib/bundled-runtime-sidecar-paths.json:19`
  - body: Adding `dist/extensions/workboard/runtime-api.js` makes the plugin runtime API guardrail discover `extensions/workboard/runtime-api.ts`, but the PR does not add that file to `RUNTIME_API_EXPORT_GUARDS` or the unguarded list. The current CI job `checks-fast-contracts-plugins-a` fails on that exact mismatch, so this needs a guard entry before merge.
  - confidence: 0.96
- **[P2] Require a session key before marking cards running:** `ui/src/ui/controllers/workboard.ts:753-759`
  - body: `startWorkboardCard` still updates the card to `status: "running"` even when `sessions.create` returns no usable key. The existing Sessions controller treats that response as an error, and Workboard should do the same or it can leave a running card with no linked session or handoff target.
  - confidence: 0.89
- **[P2] Preserve explicit zero positions on create:** `extensions/workboard/src/store.ts:172-175`
  - body: `normalizePosition(input.position, 0) || ...` treats an intentional `position: 0` as absent and replaces it with the computed tail position. Any import or caller trying to pin a new card at the top of a column cannot persist that ordering.
  - confidence: 0.9
- **[P2] Use lifecycle live state for card actions:** `ui/src/ui/views/workboard.ts:553`
  - body: The card view only sets `live` from `session.hasActiveRun === true`, while `getWorkboardLifecycle` also treats `session.status === "running"` as active. Session rows can carry `status` without `hasActiveRun`, so those cards render as running but hide the live badge and Stop action.
  - confidence: 0.86
- **[P2] Offer restart when a linked session is missing:** `ui/src/ui/views/workboard.ts:615-642`
  - body: The action branch keys only on `card.sessionKey`, so a card whose linked session was deleted or expired still shows Open session for a dead key and never offers Start session. Use the resolved lifecycle/session presence, not only the persisted key, when deciding whether the card can be restarted or relinked.
  - confidence: 0.84
- **[P2] Disable Workboard writes for read-only operators:** `ui/src/ui/views/workboard.ts:812-820`
  - body: `canWrite` is passed into the Workboard view, but the mutating controls still render enabled; only lifecycle sync checks the flag. Read-only operator sessions are documented as inspect-only, so create/edit/move/start/stop/delete controls and Sessions capture should be hidden or disabled instead of sending writes that the gateway rejects.
  - confidence: 0.82
