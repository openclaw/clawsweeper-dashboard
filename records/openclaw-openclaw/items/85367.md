## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Require a key before marking cards running:** `ui/src/ui/controllers/workboard.ts:752-759`
  - body: `sessions.create` can return a payload without a usable key, and current session code treats that as an error. This path still updates the card to `running` with no `sessionKey`, so the board can show active work with no session to open or sync.
  - confidence: 0.91
- **[P2] Use lifecycle-active state for live actions:** `ui/src/ui/views/workboard.ts:557`
  - body: `getWorkboardLifecycle` treats `session.status === "running"` as active, but the card renderer only checks `hasActiveRun`. Session payloads that report `status: "running"` without `hasActiveRun` hide the live badge and Stop action even though the card lifecycle is running.
  - confidence: 0.88
- **[P2] Let missing linked sessions restart:** `ui/src/ui/views/workboard.ts:619`
  - body: The action branch checks `card.sessionKey` rather than whether a matching session exists. When lifecycle is `missing`, the card still renders only Open session for a dead key and never offers Start session to relink and continue the work.
  - confidence: 0.86
- **[P2] Preserve explicit zero positions:** `extensions/workboard/src/store.ts:172-175`
  - body: Using `normalizePosition(input.position, 0) || tailPosition` discards an explicit `position: 0` and moves the new card to the end of the column. Callers that import or create cards with deterministic top ordering cannot persist that order.
  - confidence: 0.86
