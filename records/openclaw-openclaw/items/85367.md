## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Gate Workboard write controls on canWrite:** `ui/src/ui/views/workboard.ts:861-866`
  - body: `canWrite` is passed into the Workboard view, but the create/edit/start/delete/drag controls still render and call gateway methods registered under `operator.write`. Read-only operators will see actionable controls that predictably fail instead of a read-only board state.
  - confidence: 0.91
- **[P2] Gate session capture on operator write access:** `ui/src/ui/app-render.ts:2164-2174`
  - body: The Sessions tab exposes `onAddToWorkboard` whenever the plugin is enabled, but capture calls `workboard.cards.create`, which requires `operator.write`. This gives read-only operators an Add to Workboard action that can only fail with a write-denied RPC.
  - confidence: 0.9
- **[P2] Treat running status as live for card actions:** `ui/src/ui/views/workboard.ts:618`
  - body: `renderCard` only treats `hasActiveRun === true` as live, while the lifecycle code also treats `session.status === "running"` as active. Rows that report a running status without `hasActiveRun` will hide the live badge and Stop action for an active linked run.
  - confidence: 0.88
- **[P2] Offer restart when a linked session is missing:** `ui/src/ui/views/workboard.ts:683-711`
  - body: The action branch keys only on whether the card has a linked session key. If that session is gone, the card renders Open session for a dead key and never offers Start, so operators cannot continue work from the missing-session state.
  - confidence: 0.84
- **[P2] Preserve explicit zero positions:** `extensions/workboard/src/store.ts:280-283`
  - body: `normalizePosition(input.position, 0) || tailPosition` treats an explicit `position: 0` as absent and moves the new card to the end of the column. API callers cannot pin or import a card at the top even though zero is accepted by the normalizer.
  - confidence: 0.89
- **[P2] Deduplicate captured execution links:** `ui/src/ui/controllers/workboard.ts:604`
  - body: Capture dedupe checks only `card.sessionKey`, but the controller elsewhere treats `card.sessionKey ?? card.execution?.sessionKey` as the canonical link. Cards linked only through execution metadata can be duplicated from Sessions instead of opening the existing card.
  - confidence: 0.86
