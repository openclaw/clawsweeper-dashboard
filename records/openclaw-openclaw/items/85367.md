Full review comments:

- **[P2] Apply label length validation to comma-separated labels:** `extensions/workboard/src/store.ts:91-96`
  - body: `normalizeLabels` enforces the 40-character limit only for array input. A caller using the documented comma-separated string form can persist oversized labels, so validation and rendered card layout depend on the transport shape.
  - confidence: 0.9
- **[P2] Add labels to the dashboard card draft:** `ui/src/ui/controllers/workboard.ts:333-339`
  - body: The Workboard docs and filtering model include labels, but the dashboard draft has no label field and the create request never sends `labels`. Cards created from the UI cannot use the documented label workflow unless users bypass the dashboard.
  - confidence: 0.87
- **[P2] Require a session key before marking a card running:** `ui/src/ui/controllers/workboard.ts:449-456`
  - body: `startWorkboardCard` computes `sessionKey` as nullable but still updates the card to `running` before returning it. If `sessions.create` returns no key, the card becomes running with no handoff target; the existing sessions controller treats that response as an error instead.
  - confidence: 0.86
- **[P3] Move remaining Workboard view copy into i18n:** `ui/src/ui/views/workboard.ts:36-43`
  - body: The branch generated Workboard locale keys, but this view still hard-codes status labels, placeholders, action titles, and empty-state copy. Non-English Control UI users will see mixed English strings and future i18n checks can drift from the source of truth.
  - confidence: 0.82
