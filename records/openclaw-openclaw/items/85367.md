Full review comments:

- **[P2] Gate Workboard write controls on canWrite:** `ui/src/ui/views/workboard.ts:857-864`
  - body: `canWrite` is only passed through to lifecycle sync; the visible create/edit/start/drag/delete controls still call operator.write RPCs. Read-only operators will see actions that are guaranteed to fail instead of a read-only Workboard state.
  - confidence: 0.91
- **[P2] Hide session capture for read-only operators:** `ui/src/ui/app-render.ts:2114-2124`
  - body: `onAddToWorkboard` is wired whenever the plugin is enabled, without checking operator.write access. That button calls `captureSessionToWorkboard` and then `workboard.cards.create`, so read-only Sessions users get a failing write action.
  - confidence: 0.89
- **[P2] Treat running status as live for card actions:** `ui/src/ui/views/workboard.ts:616`
  - body: The card action UI only treats `hasActiveRun === true` as live, but the controller lifecycle and existing session helper also treat `status === "running"` as active. Rows that carry running status without `hasActiveRun` hide the live badge and Stop action.
  - confidence: 0.9
- **[P2] Offer restart when linked session is missing:** `ui/src/ui/views/workboard.ts:679-707`
  - body: The action branch keys off the persisted session key, not whether a matching session exists. A card whose linked session expired enters the missing lifecycle but still only offers Open session to a dead key, with no in-place Start/relink path.
  - confidence: 0.85
- **[P2] Preserve explicit zero positions:** `extensions/workboard/src/store.ts:280-283`
  - body: `normalizePosition(input.position, 0) || tailPosition` treats an intentional `position: 0` as missing and appends the card to the end. Importers or callers trying to pin a card at the top cannot persist that order.
  - confidence: 0.86
