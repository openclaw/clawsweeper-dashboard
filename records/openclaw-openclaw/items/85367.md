## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Require a session key before marking cards running:** `ui/src/ui/controllers/workboard.ts:752-761`
  - body: `sessions.create` can return without a usable `key`, and the existing Sessions controller treats that as an error. This path still updates the Workboard card to `running` without a linked session and then returns `null`, leaving a running card with no navigable handoff target.
  - confidence: 0.89
- **[P2] Treat running session status as live:** `ui/src/ui/views/workboard.ts:557`
  - body: `renderCard` only considers `hasActiveRun === true`, but lifecycle resolution and the shared session helper treat `status: "running"` as active too. Rows that report `status: "running"` without `hasActiveRun` will hide the live badge and Stop action even though the session is running.
  - confidence: 0.86
- **[P2] Let missing linked sessions restart:** `ui/src/ui/views/workboard.ts:619-646`
  - body: The action branch keys only on `card.sessionKey`, so a card whose linked session was deleted or expired still shows only Open session for a dead key. Since lifecycle already marks that state as `missing`, the UI should allow starting/relinking the card instead of trapping it.
  - confidence: 0.84
- **[P2] Preserve explicit zero card positions:** `extensions/workboard/src/store.ts:172-175`
  - body: Using `normalizePosition(input.position, 0) || ...` discards a valid explicit `position: 0` and always appends the card to the end of the column. Importers or callers that pin a card at the top cannot persist that ordering.
  - confidence: 0.88
