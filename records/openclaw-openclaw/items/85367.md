## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Avoid loading Workboard before config says it is enabled:** `ui/src/ui/app-settings.ts:432-437`
  - body: This Workboard tab branch calls `loadWorkboard({ force: true })` in parallel with `loadConfig`, so opening the default-off Workboard tab can issue a guaranteed failing `workboard.cards.list` before plugin enablement is known. Because the controller marks the load attempted, enabling the plugin later in the same session can leave the board empty until a manual refresh.
  - confidence: 0.9
- **[P2] Require a session key before marking cards running:** `ui/src/ui/controllers/workboard.ts:910-915`
  - body: `startWorkboardCard` still updates the card to `status: "running"` even when `sessions.create` returns no usable key. The existing sessions controller treats that response as an error, so Workboard should not persist running state without a navigable session handoff target.
  - confidence: 0.88
- **[P2] Treat running session status as live in card actions:** `ui/src/ui/views/workboard.ts:616`
  - body: `renderCard` only sets `live` from `session.hasActiveRun === true`, while the lifecycle code and existing session helper also treat `session.status === "running"` as active. Rows that report running status without `hasActiveRun` will show running lifecycle but hide the Stop action.
  - confidence: 0.9
- **[P2] Allow restarting cards whose linked session is missing:** `ui/src/ui/views/workboard.ts:679-707`
  - body: The action branch keys off the presence of any stored session key, so a card whose linked session expired still renders only Open session and never shows the start controls. That leaves the missing-session lifecycle with no in-place relink/restart path.
  - confidence: 0.85
- **[P2] Gate Workboard write controls on operator write access:** `ui/src/ui/views/workboard.ts:857-864`
  - body: The Workboard view receives `canWrite`, but create/delete/start/drag actions are still rendered without using it. Since the plugin registers write methods under `operator.write`, read-only operators will see controls that can only fail with write-denied errors instead of a read-only UI state.
  - confidence: 0.86
- **[P2] Preserve explicit zero positions when creating cards:** `extensions/workboard/src/store.ts:280-283`
  - body: `normalizePosition(input.position, 0) || ...` treats a valid `position: 0` as absent and replaces it with the computed tail position. Callers that import or create cards with deterministic top-of-column ordering cannot persist that ordering.
  - confidence: 0.87
