## Summary

Keep open. Current main does not implement the durable session archive/unarchive contract, but this PR still has blocking source-level issues, lacks required real behavior proof, is conflicting, and needs maintainer product review for writing chat excerpts into indexed memory.

## What This Changes

This PR adds durable session archiving through an `archivedAt` session-store field, Control UI archive filters/actions, archive summary files under `memory/`, inbound auto-unarchive, Swift protocol fields, and a Telegram `/set_topic_name` command.

## Best Possible Solution

Split and rebase the work: land a typed, bounded session archive/unarchive contract only after the retention location is decided, preserve current archived-row filtering where useful, and handle `/set_topic_name` separately with live Telegram proof.

## Reproduction Assessment

Yes for the patch blockers: PR-head source shows `SessionEntry.archivedAt?: number` assigned `null`, and archive summary generation calls a synchronous full-transcript reader before yielding. I did not run tests because this review is read-only.

## Solution Assessment

No as-is. The feature direction is plausible, but the patch needs a split/rebase, a typed archive/unarchive contract, bounded summary reads, retention review, and real UI/Telegram proof before merge.
