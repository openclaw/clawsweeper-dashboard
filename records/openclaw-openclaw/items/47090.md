## Best Possible Solution

Split and rebase the work: land a typed, bounded archive/unarchive contract only after retention location is decided, preserve internal session lookup behavior, and handle `/set_topic_name` separately with live Telegram proof.

## Reproduction Assessment

Yes for the patch blockers: PR-head source shows the `archivedAt` type mismatch, the default archive filter affecting internal lookups, and a synchronous full-transcript archive summary read. I did not run tests because this review is read-only.

## Solution Assessment

No as-is. The feature direction is plausible, but the patch needs a typed archive contract, lookup preservation, bounded transcript reads, explicit retention policy, rebase/conflict cleanup, and real UI/Telegram proof before merge.
