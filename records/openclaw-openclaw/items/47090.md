## Summary

Keep open. Current main only has partial archived-row filtering in the Sessions UI; it still lacks the PR's durable archive/unarchive API, archive summary behavior, and Telegram command, while the PR still has correctness, proof, and retention-review blockers.

## What This Changes

Adds durable `archivedAt` session archiving with Control UI archive filters/actions, archive summary files under `memory/`, inbound auto-unarchive, Swift protocol fields, and a Telegram `/set_topic_name` command.

## Best Possible Solution

Split and rebase the work: land a typed, bounded session archive/unarchive contract only after retention location is decided, preserve current archived filters where useful, and handle the Telegram topic command separately with real Telegram proof.

## Reproduction Assessment

Yes for the patch blockers: PR-head source inspection shows a `number | undefined` archive field assigned `null`, and archive summary generation synchronously scans transcripts before yielding. I did not run tests because this review is read-only.

## Solution Assessment

No as-is. The feature direction is plausible, but the patch needs a split/rebase, a typed archive/unarchive contract, bounded summary reads, retention review, and real UI/Telegram proof before merge.
