## Summary

Keep open: the PR still contains useful feature work that is not implemented on current main, but it is not merge-ready because real behavior proof is missing and the archive filter still breaks an ACP exact-session hydration path. The merged Control UI archived-row filter only partially overlaps with this PR and does not provide the `archivedAt` archive/unarchive workflow, summary files, or inbound auto-unarchive behavior.

## What This Changes

The PR adds a user-facing session archive/unarchive flow with a new `archivedAt` session field, archive-summary file writes, Control UI filtering/actions, auto-unarchive on inbound messages, and a Telegram `/set_topic_name` command.

## Best Possible Solution

Land a focused, current-main-based version that preserves exact archived-session lookup for ACP/internal tooling, splits unrelated Telegram topic-label behavior if maintainers want it separately, and includes real UI plus Telegram proof.

## Reproduction Assessment

Yes for the blocking review finding by source inspection: the PR hides `archivedAt` rows by default in `listSessionsFromStore`, while ACP `getGatewaySessionRow` calls `sessions.list` with `search` and no archived opt-in. The end-to-end archive feature itself was not live-verified in this review.

## Solution Assessment

No. The archive direction is plausible, but the implementation should preserve existing exact-session metadata lookups and should provide real UI/Telegram proof before merge.
