## What This Changes

Overall confidence: 0.88

Full review comments:

- **[P1] Clear archivedAt through a typed contract:** `src/auto-reply/reply/session.ts:445`
  - body: `SessionEntry.archivedAt` is declared as `number | undefined`, but this auto-unarchive path assigns `null` directly to a `SessionEntry`. In strict TypeScript this breaks the session-store type contract; use a typed clear path that persists removal without assigning an invalid value.
  - confidence: 0.9
- **[P2] Bound archive transcript reads:** `src/sessions/archive-summary.ts:94`
  - body: `writeSessionArchiveSummary` is async, but `readSessionMessages` uses `fs.readFileSync` before the first await, so the `setImmediate` callback still blocks the gateway event loop on the full transcript. Use an async/bounded reader or skip large transcripts for archive summaries.
  - confidence: 0.86
