## Summary

Keep open. Current main does not implement this PR's durable manual archive/unarchive contract, and the latest PR head still has blocking source-level issues, missing real behavior proof, and an indexed-memory retention concern; the related request at https://github.com/openclaw/openclaw/issues/69271 is open tracking product direction rather than superseding this implementation.

## What This Changes

Adds a durable session `archivedAt` field, archive filtering/actions in the Sessions UI, archive summary files under `memory/`, inbound auto-unarchive, Swift protocol fields, and a Telegram `/set_topic_name` command.

## Best Possible Solution

Split and rebase the work: land a typed, bounded session archive/unarchive contract only after retention location is decided, preserve internal exact-key lookups, and handle `/set_topic_name` separately with live Telegram proof.

## Reproduction Assessment

Yes for the patch blockers: PR-head source shows a `number | undefined` `archivedAt` field assigned `null`, a default archive filter that can hide rows from internal exact-key `sessions.list` lookups, and archive summary generation using a synchronous full-transcript reader. I did not run tests because this review is read-only.

## Solution Assessment

No as-is. The feature direction is plausible, but the patch needs a typed archive/unarchive contract, internal lookup preservation, bounded transcript reads, retention review, and real UI/Telegram proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Clear archivedAt through a typed session contract:** `src/auto-reply/reply/session.ts:445`
  - body: `SessionEntry.archivedAt` is declared as `number | undefined`, but inbound auto-unarchive assigns `null` directly to the session entry. In strict TypeScript this breaks the declared store shape; clear the field through `sessions.patch` or a typed helper that deletes the property before persistence.
  - confidence: 0.93
- **[P2] Keep archived sessions available to internal exact-key lookups:** `src/gateway/session-utils.ts:893-894`
  - body: This new default filter drops `archivedAt` rows from every `sessions.list` call unless the caller opts in. Internal consumers such as ACP still resolve an exact session key through `sessions.list` without an include flag, so archived sessions can lose snapshot/config metadata even though the session still exists.
  - confidence: 0.86
- **[P2] Bound archive summary transcript reads:** `src/sessions/archive-summary.ts:94`
  - body: `writeSessionArchiveSummary` calls `readSessionMessages` before the first await, and that reader performs synchronous full-file transcript reads. Even from the deferred callback, a large archived transcript can block the gateway event loop; use a bounded recent reader or an async transcript path.
  - confidence: 0.86
