The PR adds session archive metadata/list filtering/UI controls, archive summary writing, inbound auto-unarchive behavior, Swift protocol fields, and a Telegram `/set_topic_name` command.

## Best Possible Solution

Land a rebased, focused archive implementation that models archive state consistently, keeps user-facing hiding separate from internal lookups, adds targeted Gateway/UI/ACP coverage, includes the changelog, and either intentionally keeps or splits the Telegram topic-name command.

## Reproduction Assessment

Yes. Source inspection of the PR head shows the strict type mismatch, and the ACP lookup break follows from the new default filter plus the existing `sessions.list({ search: sessionKey })` call without an archive opt-in.

## Solution Assessment

No. The feature direction is plausible, but this branch is not the best mergeable solution until archive clearing is typed or deletion-aware, internal exact lookups include archived rows, and the user-facing change has changelog and regression coverage.
