Keep open. Current main only has a related Control UI archived-row filter and does not implement this PR's `archivedAt` session API, archive-summary flow, or Telegram topic-name command; the PR still contains unique feature work but has merge-blocking correctness, proof, and retention issues.

## What This Changes

The PR adds session archiving metadata/filtering, archive/unarchive UI controls, memory-backed archive summaries, inbound auto-unarchive, Swift/Gateway protocol fields, and a Telegram `/set_topic_name` command.

## Best Possible Solution

Split or rebase the feature around current Sessions UI, implement a typed archive/unarchive contract with bounded summaries and explicit retention semantics, then prove the UI and Telegram flows in a real setup.

## Reproduction Assessment

Yes for the patch blockers: PR-head source inspection shows `null` assigned to a `number | undefined` field and a full synchronous transcript read in the archive-summary path. I did not run tests because this review is read-only.

## Solution Assessment

No as-is. The direction is plausible, but the patch needs a typed archive contract, bounded or async summaries, retention review, rebase against current Sessions UI, and real UI/Telegram proof before merge.
