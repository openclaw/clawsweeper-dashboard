Keep this PR open, but it is not merge-ready: the feature is not already on current main, the PR still has source-level blockers, missing real behavior proof, and a retention/security question around writing chat excerpts into indexed memory.

## What This Changes

The PR adds session archive metadata/filtering, UI archive controls, archive-summary files, inbound auto-unarchive, Gateway/Swift protocol fields, and a Telegram `/set_topic_name` command.

## Best Possible Solution

Rebase or split around current `sessionsShowArchived` UI, implement a typed archive/unarchive contract with bounded or async summaries, settle retention semantics, and prove the UI plus Telegram flows in a real setup.

## Solution Assessment

No as-is. The direction is plausible, but the current patch needs a type-contract fix, bounded summary reads, retention review, rebase against current Sessions UI, and real UI/Telegram proof.
