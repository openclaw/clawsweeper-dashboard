## Summary

Keep open. Current main still lacks the proposed archivedAt/includeArchived archive-unarchive contract, and the PR head still has blocking typed-state, event-loop, retention, and proof issues.

## What This Changes

This PR adds session archivedAt metadata, list filtering, Control UI archive/unarchive controls, memory-backed archive summaries, inbound auto-unarchive, Swift protocol fields, and a Telegram /set_topic_name command.

## Best Possible Solution

Split or rebase the work around current Sessions UI, settle the archive retention contract with https://github.com/openclaw/openclaw/issues/56609 and the row-action UX with https://github.com/openclaw/openclaw/issues/69271, then add bounded summaries and real UI/Telegram proof.

## Reproduction Assessment

Yes for the patch blockers: PR-head source inspection shows a `number | undefined` field is assigned `null`, and archive summary generation performs a synchronous full transcript scan before yielding. I did not run tests because this review is read-only.

## Solution Assessment

No as-is. The direction is plausible, but the patch needs a typed archive/unarchive contract, bounded or async summaries, retention review, a rebase against current Sessions UI, and real UI/Telegram proof before merge.
