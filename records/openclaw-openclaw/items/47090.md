## Summary

Keep open, but not merge-ready: the branch conflicts with current main's newer session-list UI shape, still lacks real behavior proof, and has source-visible type, retention, and availability blockers that need maintainer decisions before review can proceed.

## What This Changes

Adds persisted `archivedAt` session state, sessions list/patch support, Control UI archive controls and filtering, archive summary files under agent memory, auto-unarchive on inbound messages, Swift protocol fields, and a Telegram `/set_topic_name` command.

## Best Possible Solution

Rebase on current main, split Telegram topic naming from session archiving, and implement archive behavior through the settled sessions API/UI with bounded summary handling and an explicit retention policy coordinated with https://github.com/openclaw/openclaw/issues/69271.
