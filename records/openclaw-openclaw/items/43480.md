## What This Changes

The issue asks to stop a macOS VoiceWakeOverlay SwiftUI render loop that can pinwheel the app when attributed transcript changes resize the overlay and update overflow state synchronously.

## Best Possible Solution

Make overlay height measurement side-effect-free or coalesced, update overflow state outside SwiftUI's synchronous render/update cycle with equality guards, and prove the macOS overlay remains responsive.
