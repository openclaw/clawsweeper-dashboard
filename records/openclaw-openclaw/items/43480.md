## What This Changes

The issue reports a macOS VoiceWake overlay SwiftUI render loop that can pinwheel the app when attributed-text changes synchronously resize the overlay and update overflow state.

## Best Possible Solution

Land a focused macOS fix that defers attributed-text resize onto the next main-queue turn, avoids redundant observable overflow writes from measuredHeight, and adds regression coverage for unchanged overflow state.
