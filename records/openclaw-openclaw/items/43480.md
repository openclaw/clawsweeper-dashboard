The issue requests fixing a macOS VoiceWakeOverlay SwiftUI render loop where transcript changes can trigger repeated frame measurement and observed state mutation until the app pinwheels.

## Best Possible Solution

Make overlay height measurement pure or safely coalesced, move overflow-state updates outside SwiftUI's synchronous view-update path with an equality guard, and verify the real macOS overlay no longer pinwheels or emits SwiftUI update-loop warnings.
