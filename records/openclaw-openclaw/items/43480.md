## What This Changes

The issue asks to stop a macOS VoiceWakeOverlay SwiftUI render loop that can pinwheel the app when attributed-text changes synchronously resize the overlay and update overflow state.

## Best Possible Solution

Make overlay height measurement side-effect-free or safely coalesced, update overflow state outside SwiftUI's synchronous render/update cycle with equality guards, and verify the real macOS overlay no longer pinwheels or emits update-loop warnings.

## Solution Assessment

Yes: the best fix is a narrow VoiceWakeOverlay controller/view change that removes synchronous observed-state mutation from the render/update path. The related WorkActivityStore watchdog PR addresses a separate menu-bar display-state problem.
