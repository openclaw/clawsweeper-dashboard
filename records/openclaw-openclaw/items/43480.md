## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/43480 in the macOS VoiceWakeOverlay path. Break the synchronous SwiftUI loop where attributed onChange calls updateWindowFrame, targetFrame calls measuredHeight, and measuredHeight writes model.isOverflowing during view update. Prefer making measurement pure or coalescing the overflow-state mutation outside the synchronous view-update path with a no-op guard; do not solve this via the unrelated WorkActivityStore menubar watchdog path or a broad VoiceSession architecture rewrite. Add focused regression coverage for repeated attributed/overflow updates where feasible, add a CHANGELOG.md fix entry, and stop for maintainer review if the fix requires a new product/config decision. Validate with the focused Swift tests and a real macOS manual overlay check showing no pinwheel and no SwiftUI runtime warnings for state mutation during view update or onChange multiple times per frame.
