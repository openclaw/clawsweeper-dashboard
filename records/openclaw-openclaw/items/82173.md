## Summary

Keep open: this is a useful but security-sensitive new feature request, not implemented on current main, and it needs maintainer threat-model and product-boundary review before any implementation path is safe.

## What This Changes

Add an in-conversation flow that accepts a secret, generates a temporary copy-page URL for WhatsApp or Telegram sharing, and expires it after use or a short time.

## Best Possible Solution

Define the threat model and ownership first; if optional, implement it as a security-reviewed plugin/ClawHub package using existing tool, HTTP route, RPC, and service seams, or extend the plugin API only if a concrete missing seam is found.

Not applicable: this is a new capability request, not a broken existing behavior. Current-main source and docs searches did not find an existing secret-link flow to exercise.

## Solution Assessment

Unclear: a core-native flow is not yet proven to be the best home because OpenClaw policy favors optional plugin delivery, and this feature needs security/product design before implementation.
