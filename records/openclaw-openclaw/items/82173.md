## Summary

Keep open: this is a new security-sensitive feature request, not a stale bug or already-implemented behavior. Current main has SecretRef management for configured credentials, but no temporary secret-link command, unauthenticated copy page, one-time URL store, or WhatsApp/Telegram sharing flow.

## Best Possible Solution

Define a maintainer-reviewed product and threat model first; if accepted, implement a narrow operator-controlled secret-link capability with TTL/one-time access, no plaintext logging, revocation, docs, and real WhatsApp/Telegram in-app-browser proof.
