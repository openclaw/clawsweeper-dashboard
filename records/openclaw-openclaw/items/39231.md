## Summary

Keep open: current main still does not implement the CaMeL feature, so this is not cleanup-closeable, but the PR remains a security-sensitive core feature with blocking false-assurance paths and missing real behavior proof.

## What This Changes

The PR adds an opt-in CaMeL-style prompt-injection defense across agent orchestration, tool-call hooks, config schema, gateway exec approvals, SECURITY.md, and tests.

## Best Possible Solution

Route this through maintainer security/product review, likely alongside https://github.com/openclaw/openclaw/issues/62939, and land only a design that preserves provenance end-to-end, resolves config through one shared path, and synthesizes post-side-effect replies without tool access.

## Reproduction Assessment

Yes. Source inspection of PR head provides high-confidence reproduction paths for the review findings: taint can be laundered through intermediate results, synthesis can use a normal tool-enabled prompt, retry guards can miss partial side effects, and defaults-only hook config can skip enforcement.

## Solution Assessment

No. CaMeL-style provenance may be a useful direction, but this PR is not the best mergeable implementation until the security contract, taint propagation, retry behavior, config resolution, and real behavior proof are resolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve taint dependencies when wrapping tool results:** `src/agents/camel/orchestrator.ts:153`
  - body: This wraps each result from only the callee name, so a tainted value passed through a trusted or no-side-effect transform can become trusted before a later side-effect check. Carry the wrapped argument dependencies into the result provenance instead.
  - confidence: 0.92
- **[P1] Disable tools during post-orchestration synthesis:** `src/agents/pi-embedded-runner/run/attempt.ts:2178-2182`
  - body: After CaMeL has already run side-effect steps, this prompts the normal session again and relies on text telling the model not to re-run tools. Use a no-tool synthesis path or persist completed calls as non-executable context before prompting.
  - confidence: 0.88
- **[P1] Set the replay guard before running side effects:** `src/agents/pi-embedded-runner/run/attempt.ts:2154`
  - body: The outer retry guard is set only after orchestrator.execute returns. If one mutating step succeeds and a later step throws, the retry loop can enter CaMeL again and replay the completed side effect.
  - confidence: 0.84
- **[P2] Merge defaults into hook CaMeL config:** `src/agents/pi-tools.before-tool-call.ts:187-195`
  - body: The embedded runner resolves agents.defaults.camel, agents.camel, and per-agent overrides, but this hook fallback omits defaults. Defaults-only enablement can therefore fall back to the normal tool loop without the configured CaMeL guard.
  - confidence: 0.87
- **[P2] Recheck CaMeL after hook parameter rewrites:** `src/agents/pi-tools.before-tool-call.ts:358`
  - body: CaMeL policy runs before before_tool_call hooks can merge replacement params, so the arguments that actually execute may differ from the approved arguments. Re-run the CaMeL decision after hook rewrites or move the check after final params are known.
  - confidence: 0.78
