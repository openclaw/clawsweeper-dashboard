## Summary

Keep open. Current main does not implement the CaMeL feature, but the PR still has high-confidence security/correctness blockers and missing real behavior proof, so it needs maintainer/security review rather than cleanup close.

## What This Changes

The PR adds an opt-in CaMeL taint/provenance engine, agent runner and before-tool-call integration, config schema, gateway approval plumbing, SECURITY.md docs, and tests.

## Best Possible Solution

Keep this open for maintainer/security review or supersede it with a narrower design aligned with https://github.com/openclaw/openclaw/issues/62939 that preserves provenance end-to-end, resolves CaMeL config through one path, and performs post-side-effect synthesis without tool access.

## Reproduction Assessment

Yes. Source inspection of the PR head gives high-confidence paths for the review findings: intermediate tool results can lose taint, defaults-only config can skip the hook guard, and post-orchestration synthesis can use the normal tool-enabled session.

## Solution Assessment

No. CaMeL-style provenance is directionally useful, but this patch is not the best mergeable implementation until taint propagation, shared config resolution, no-tool synthesis, and maintainer/security review are resolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve taint through intermediate tool results:** `src/agents/camel/orchestrator.ts:153`
  - body: The orchestrator wraps each result from only the callee name, so tainted input passed through a trusted or no-side-effect transform becomes trusted before a later side-effect check. Preserve dependencies from the wrapped arguments when creating the result.
  - confidence: 0.91
- **[P1] Disable tools for the synthesis turn:** `src/agents/pi-embedded-runner/run/attempt.ts:2178-2182`
  - body: After CaMeL executes side effects, this prompts the normal tool-enabled session again and relies on text telling the model not to rerun tools. Use a no-tool synthesis path or persist completed tool calls as non-executable context.
  - confidence: 0.86
- **[P2] Merge defaults into hook CaMeL config:** `src/agents/pi-tools.before-tool-call.ts:187-195`
  - body: The embedded runner resolves agents.defaults.camel, agents.camel, and per-agent overrides, but the before-tool-call fallback omits defaults. Defaults-only enablement can therefore reach the orchestrator path while fallback or direct tool calls skip the guard.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff introduces a core prompt-injection defense, but concrete bypass and false-assurance paths remain.

Concerns:

- **[high] Taint can be laundered through intermediate tools:** `src/agents/camel/orchestrator.ts:153`
  - body: CaMeLOrchestrator wraps tool results using only the executed tool name, and TaintTracker labels non-untrusted tools as trusted, so a tainted value can lose provenance before a later side-effect policy check.
  - confidence: 0.91
- **[high] Post-orchestration tools can replay side effects:** `src/agents/pi-embedded-runner/run/attempt.ts:2178`
  - body: The final synthesis turn uses the normal activeSession.prompt path after side effects already completed, so prompt text rather than enforcement prevents duplicate exec/write/message actions.
  - confidence: 0.86
- **[medium] Defaults-only CaMeL config skips hook fallback:** `src/agents/pi-tools.before-tool-call.ts:187`
  - body: The hook fallback omits agents.defaults.camel even though the runner resolves it, leaving documented defaults-level enablement without the advertised guard in fallback/direct tool paths.
  - confidence: 0.86
