## Summary

Keep open for maintainer/security review. Current main does not implement CaMeL, and the PR branch still has blocking security-control gaps plus no contributor real behavior proof.

## What This Changes

The PR adds an opt-in CaMeL-style prompt-injection defense across agent orchestration, taint tracking, before-tool-call enforcement, config schema, gateway exec approvals, SECURITY.md, and tests.

## Best Possible Solution

Route this through maintainer security/product review alongside https://github.com/openclaw/openclaw/issues/62939 and land only a design that preserves provenance end to end, resolves config through one shared path, prevents replay after side effects, and keeps approval-routing trust internal.

## Reproduction Assessment

Yes for the PR review findings. PR-head source inspection gives high-confidence paths for taint laundering through intermediate results, tool-enabled synthesis after side effects, partial-failure replay, defaults-only hook bypass, and client-supplied approval-forwarding source bypass.

## Solution Assessment

No. CaMeL-style provenance may be a useful direction, but this PR is not a mergeable implementation until the security contract, taint propagation, retry behavior, config resolution, approval-routing trust boundary, and real behavior proof are resolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve taint dependencies when wrapping tool results:** `src/agents/camel/orchestrator.ts:153`
  - body: This wraps each tool result using only the executed tool name, so output from a trusted or no-side-effect intermediate tool can lose the tainted argument provenance that produced it. Carry the wrapped argument dependencies into the result value before any later side-effect check.
  - confidence: 0.92
- **[P1] Disable tools during post-orchestration synthesis:** `src/agents/pi-embedded-runner/run/attempt.ts:2178-2182`
  - body: After CaMeL has already run side-effect steps, this prompts the normal tool-enabled session and relies on prompt text telling the model not to re-run tools. Use a no-tool synthesis path or persist completed calls as non-executable context before prompting.
  - confidence: 0.88
- **[P1] Set the replay guard before side effects can fail:** `src/agents/pi-embedded-runner/run/attempt.ts:2154`
  - body: The retry guard is set only after orchestrator.execute returns. If one mutating step succeeds and a later step throws, the outer retry loop can enter CaMeL again and replay the already-completed side effect.
  - confidence: 0.84
- **[P2] Merge defaults into hook CaMeL config:** `src/agents/pi-tools.before-tool-call.ts:187-195`
  - body: The embedded runner resolves agents.defaults.camel, agents.camel, and per-agent overrides, but this hook fallback omits defaults. Defaults-only enablement can therefore fall back to the normal tool loop without the configured CaMeL guard.
  - confidence: 0.87
- **[P2] Recheck CaMeL after hook parameter rewrites:** `src/agents/pi-tools.before-tool-call.ts:358`
  - body: CaMeL policy runs before before_tool_call hooks can merge replacement params, so the arguments that actually execute may differ from the approved arguments. Re-run the CaMeL decision after hook rewrites or move the check after final params are known.
  - confidence: 0.78
- **[P2] Do not trust client-supplied approval source:** `src/infra/exec-approval-forwarder.ts:73-75`
  - body: The new source field is accepted from exec.approval.request and source="camel" bypasses the forwarding enabled gate. Derive this marker internally for CaMeL requests instead of letting any operator.approvals client force session approval forwarding while it is disabled.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff introduces a core prompt-injection defense, but concrete bypass, replay, and approval-routing false-assurance paths remain.

Concerns:

- **[high] Taint can be laundered through intermediate tools:** `src/agents/camel/orchestrator.ts:153`
  - body: The orchestrator wraps tool results using only the executed tool name while TaintTracker labels non-untrusted tool outputs trusted, so derived values can lose untrusted provenance before a later side-effect policy check.
  - confidence: 0.92
- **[high] Post-orchestration tools can replay side effects:** `src/agents/pi-embedded-runner/run/attempt.ts:2178`
  - body: The final synthesis turn uses the normal activeSession.prompt path after side effects have completed, so enforcement is replaced by prompt text asking the model not to call tools again.
  - confidence: 0.88
- **[high] Partial orchestration failures can replay side effects:** `src/agents/pi-embedded-runner/run/attempt.ts:2154`
  - body: The side-effect replay guard is set after orchestrator.execute returns, so a failure after an earlier mutating step can leave the outer retry loop free to execute the side-effect plan again.
  - confidence: 0.84
- **[medium] Defaults-only enablement can skip hook enforcement:** `src/agents/pi-tools.before-tool-call.ts:187`
  - body: The before-tool-call fallback omits agents.defaults.camel even though the runner resolves it, leaving a documented config placement without the advertised guard in fallback/direct tool paths.
  - confidence: 0.87
- **[medium] Approval forwarding source is client-controlled:** `src/infra/exec-approval-forwarder.ts:73`
  - body: source="camel" bypasses the exec approval forwarding enabled gate, but the gateway accepts source from the public exec.approval.request params rather than deriving it from trusted CaMeL internals.
  - confidence: 0.82
