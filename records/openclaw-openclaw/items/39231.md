## Summary

Keep open. This PR proposes a useful security feature, but current main does not already contain it and the PR head still has source-proven security bypass/false-assurance paths plus no after-fix real behavior proof.

## What This Changes

The PR adds an opt-in CaMeL prompt-injection defense engine, agent runner and before-tool-call integration, config schema, gateway exec approval metadata, SECURITY.md documentation, and tests.

## Best Possible Solution

Keep this open for maintainer/security review or replace it with a narrower prompt-injection boundary design aligned with https://github.com/openclaw/openclaw/issues/62939 that preserves provenance end-to-end, resolves config through one path, and prevents tool-enabled synthesis after side effects run.

## Reproduction Assessment

Yes. Source inspection against the PR head gives high-confidence paths: a tainted value can pass through an intermediate trusted tool, defaults-only config can skip the hook fallback, and post-orchestration synthesis can run with normal tools still enabled.

## Solution Assessment

No. CaMeL-style provenance may be directionally useful, but this patch is not the best mergeable implementation until provenance propagation, shared config resolution, no-tool synthesis, and the security boundary are fixed and reviewed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve taint through intermediate tool results:** `src/agents/camel/orchestrator.ts:153`
  - body: The orchestrator rewraps each result using only the callee name, so a tainted value can pass through a trusted or no-side-effect transform and become trusted before a later side-effect policy check. Derive result provenance from the wrapped input dependencies so taint survives multi-step plans.
  - confidence: 0.91
- **[P1] Disable tools for the synthesis turn:** `src/agents/pi-embedded-runner/run/attempt.ts:2178-2182`
  - body: After CaMeL executes the side-effect plan, this prompts the normal tool-enabled session again and relies on text telling the model not to rerun tools. Synthesize with tools disabled or persist completed tool calls as non-executable context before asking for a summary.
  - confidence: 0.86
- **[P2] Merge defaults into the hook CaMeL config:** `src/agents/pi-tools.before-tool-call.ts:187-195`
  - body: The embedded runner resolves agents.defaults.camel, agents.camel, and per-agent overrides, but the before-tool-call fallback only reads agents.camel and per-agent camel. A deployment enabling CaMeL under defaults can reach fallback or direct tool calls without the advertised hook guard.
  - confidence: 0.86
