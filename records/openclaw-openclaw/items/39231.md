## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve taint when wrapping tool results:** `src/agents/camel/orchestrator.ts:153`
  - body: `wrapToolResult` derives result provenance only from the callee name, so a tainted value can pass through any non-taint-source tool and become trusted before the next side-effect check. Carry dependencies from `wrappedArgs` into the result so taint survives intermediate tool steps.
  - confidence: 0.9
- **[P1] Disable tools during final synthesis:** `src/agents/pi-embedded-runner/run/attempt.ts:2178-2182`
  - body: After orchestrated side effects complete, this calls the normal tool-enabled session again and relies on prompt text to prevent reruns. The model can still call `exec`, `write`, or messaging tools a second time, so the synthesis step must run without tools or persist exact executed tool results before any tool-capable prompt.
  - confidence: 0.88
- **[P1] Set the replay guard before side effects can fail:** `src/agents/pi-embedded-runner/run/attempt.ts:2154`
  - body: `camelSideEffectsExecuted` is set only after `orchestrator.execute()` returns, so a mutating step followed by a later failure leaves the outer retry loop free to rerun already-completed side effects. Flip a durable guard as soon as the first side-effect step starts or completes.
  - confidence: 0.86
- **[P1] Merge defaults.camel into fallback enforcement:** `src/agents/pi-tools.before-tool-call.ts:190-199`
  - body: The schema accepts `agents.defaults.camel` and the main runner merges it, but the before-tool-call CaMeL check only reads `agents.camel` and per-agent overrides. Any fallback or normal-loop path under defaults-only config silently skips the configured protection.
  - confidence: 0.84
- **[P1] Do not trust request payloads for CaMeL forwarding:** `src/infra/exec-approval-forwarder.ts:68-72`
  - body: The PR exposes `source` on `exec.approval.request` and then bypasses the forwarding enabled gate when that payload says `camel`. That trust bit needs to come from an internal CaMeL-only call path, not from a client-supplied protocol field.
  - confidence: 0.82
