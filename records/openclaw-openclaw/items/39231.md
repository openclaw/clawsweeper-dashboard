- **[P1] Preserve taint dependencies when wrapping tool results:** `src/agents/camel/orchestrator.ts:153`
  - body: This wraps each tool result using only the callee name, so a chain like `web_fetch -> trusted transform -> message.send` can relabel tainted data as trusted before the final side-effect check. Carry the wrapped argument dependencies into the result value so taint survives intermediate steps.
  - confidence: 0.92
- **[P1] Disable tools during the synthesis turn:** `src/agents/pi-embedded-runner/run/attempt.ts:2178-2182`
  - body: After CaMeL has executed side-effect steps, this sends the normal tool-enabled session another prompt and relies on text saying not to rerun tools. A mutating prompt can therefore trigger duplicate exec/write/message actions during summary generation; synthesize with a tool-free path or persist executed results before re-prompting.
  - confidence: 0.9
- **[P1] Set the replay guard before side effects can fail:** `src/agents/pi-embedded-runner/run/attempt.ts:2154`
  - body: `camelSideEffectsExecuted` is set only after `orchestrator.execute()` returns. If an earlier mutating step succeeds and a later step throws, the outer retry loop can treat the attempt as replay-safe and run the side effects again.
  - confidence: 0.88
- **[P1] Do not trust client-supplied CaMeL approval source:** `src/gateway/server-methods/exec-approval.ts:151-154`
  - body: The gateway now accepts `source` from the request payload, and the forwarder treats `source === "camel"` as bypassing the forwarding enabled gate. That CaMeL-only bypass needs to be derived from trusted internal state, not a public RPC parameter.
  - confidence: 0.86
