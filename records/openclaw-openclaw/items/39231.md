Overall confidence: 0.92

Full review comments:

- **[P1] Preserve taint dependencies when wrapping results:** `src/agents/camel/orchestrator.ts:153`
  - body: `wrapToolResult` derives provenance only from the tool name and discards the current step's tainted argument dependencies. A plan can pass untrusted data through a read-only or trusted transform and then use the clean-labeled result in a later side-effect call, bypassing the approval check this feature advertises.
  - confidence: 0.91
- **[P1] Disable tools during the synthesis turn:** `src/agents/pi-embedded-runner/run/attempt.ts:2178-2182`
  - body: After the orchestrator has already executed side-effect steps, this prompts the normal tool-enabled session with the original request and only text asking the model not to rerun tools. That can trigger a second `exec`, `write`, or messaging action because the completed plan is not represented as a normal tool transcript.
  - confidence: 0.9
- **[P1] Set the replay guard before side effects can fail:** `src/agents/pi-embedded-runner/run/attempt.ts:2154`
  - body: `camelSideEffectsExecuted` is set only after `orchestrator.execute()` returns successfully. If an earlier step mutates state and a later step throws, the outer retry path can rerun the same side-effect plan because the guard remains false.
  - confidence: 0.89
- **[P1] Do not trust client-supplied CaMeL approval source:** `src/gateway/protocol/schema/exec-approvals.ts:133`
  - body: Adding `source` to the public exec approval request lets the requester mark an approval as `camel`, and the forwarder uses that value to bypass the forwarding enabled gate. The server should derive a CaMeL/internal approval source from trusted state or a separate internal method, not accept it from request payloads.
  - confidence: 0.87
- **[P1] Honor defaults.camel in the fallback hook:** `src/agents/pi-tools.before-tool-call.ts:186-195`
  - body: The main runner accepts `agents.defaults.camel`, but the before-tool-call CaMeL check only merges `agents.camel` and per-agent overrides. A config that enables CaMeL through defaults can therefore fall back to the normal tool loop with hook enforcement disabled.
  - confidence: 0.86
