## Summary

This should stay open for explicit maintainer/security review: current main does not contain the CaMeL engine, but the PR still has concrete security-boundary defects and lacks after-fix real behavior proof.

## What This Changes

The PR adds an opt-in CaMeL prompt-injection defense engine, agent runner and before-tool-call integration, config schema support, gateway exec approval metadata, SECURITY.md documentation, and related tests.

## Best Possible Solution

Either redesign this behind the narrower prompt-injection boundary tracked in https://github.com/openclaw/openclaw/issues/62939, or repair the PR so provenance, config resolution, synthesis without tools, and approval semantics are enforced end to end before a security review.

## Reproduction Assessment

Yes. Source inspection of the PR head gives high-confidence paths for taint laundering through intermediate tools, duplicate side effects during post-orchestration synthesis, and defaults-only configuration skipping the hook fallback.

## Solution Assessment

No. CaMeL-style provenance may be useful, but this patch is not the best mergeable implementation until the security boundary is repaired or narrowed and validated with real runtime proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve taint through intermediate tool results:** `src/agents/camel/orchestrator.ts:153`
  - body: The orchestrator rewraps each tool result using only the callee name, so a tainted value can pass through a trusted or no-side-effect transform and become trusted before a later side-effect policy check. Derive result provenance from the wrapped input dependencies so taint survives multi-step plans.
  - confidence: 0.91
- **[P1] Disable tools for the synthesis turn:** `src/agents/pi-embedded-runner/run/attempt.ts:2178-2182`
  - body: After CaMeL executes the side-effect plan, this prompts the normal tool-enabled session again and relies on text telling the model not to rerun tools. Clean trusted-input actions may be duplicated before the summary; synthesize with tools disabled or persist completed tool calls as non-executable context.
  - confidence: 0.86
- **[P2] Merge defaults into the hook CaMeL config:** `src/agents/pi-tools.before-tool-call.ts:187-195`
  - body: The embedded runner resolves `agents.defaults.camel`, `agents.camel`, and per-agent overrides, but the before-tool-call fallback only reads `agents.camel` and per-agent `camel`. A deployment enabling CaMeL under defaults can parse and reach the orchestrator path while fallback or direct tool calls skip the hook guard.
  - confidence: 0.86
