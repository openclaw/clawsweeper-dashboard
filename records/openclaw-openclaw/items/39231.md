## Summary

Keep open. Current main still does not ship the CaMeL engine/config, but this security-sensitive PR has source-reproducible bypass/replay defects and no real behavior proof, so it should not be closed as stale or implemented; the adjacent prompt-boundary issue is related but not a shipped replacement.

## What This Changes

The PR adds an opt-in CaMeL prompt-injection defense spanning a new `src/agents/camel` engine, agent runner and before-tool-call integration, config schema, gateway approval metadata, docs, and tests.

## Best Possible Solution

Revise or replace the PR with a reviewed prompt-injection boundary that preserves provenance end-to-end, resolves CaMeL config through one shared path, and synthesizes post-side-effect replies without tool access, aligned with https://github.com/openclaw/openclaw/issues/62939.

## Reproduction Assessment

Yes. Source inspection against the PR head gives high-confidence paths: a tainted `web_fetch` result can be laundered through a trusted/read-only intermediate tool, defaults-only CaMeL enablement can miss the fallback hook, and post-orchestration synthesis can use the normal tool-enabled session.

## Solution Assessment

No. CaMeL-style provenance may be directionally useful, but this patch is not the best mergeable implementation until provenance propagation, shared config resolution, no-tool synthesis, and the security boundary are fixed and reviewed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve taint through intermediate results:** `src/agents/camel/orchestrator.ts:153`
  - body: The orchestrator wraps each result using only the callee name, so tainted data can pass through a trusted or no-side-effect transform and become `TrustedTool` before a later side-effect policy check. Derive the result capabilities from the wrapped input dependencies so provenance survives multi-step plans.
  - confidence: 0.91
- **[P1] Disable tools for the synthesis turn:** `src/agents/pi-embedded-runner/run/attempt.ts:2178-2182`
  - body: After CaMeL executes a side-effect plan, this prompts the normal tool-enabled session again and relies on text telling the model not to rerun tools. Clean trusted-input actions can be duplicated before the summary, so synthesize with tools disabled or persist completed tool calls as non-executable context.
  - confidence: 0.86
- **[P2] Merge defaults into the hook CaMeL config:** `src/agents/pi-tools.before-tool-call.ts:187-195`
  - body: The embedded runner resolves `agents.defaults.camel`, `agents.camel`, and per-agent overrides, but the before-tool-call fallback only reads `agents.camel` and per-agent `camel`. A deployment enabling CaMeL under defaults can parse and reach fallback/direct tool paths without the configured guard.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff introduces a security-sensitive prompt-injection defense, but concrete bypass and false-assurance paths remain.

Concerns:

- **[high] Taint can be laundered through intermediate tools:** `src/agents/camel/orchestrator.ts:153`
  - body: `CaMeLOrchestrator` rewraps tool results using only the callee name, and `TaintTracker` labels non-untrusted tools as trusted, which can drop provenance from tainted inputs before a later side-effect policy check.
  - confidence: 0.91
- **[high] Post-orchestration tools can replay side effects:** `src/agents/pi-embedded-runner/run/attempt.ts:2178`
  - body: The synthesis turn runs through the normal tool-enabled session after side effects already completed. Prompt text is not an enforcement boundary, and clean trusted-input actions may be duplicated without a CaMeL taint block.
  - confidence: 0.86
- **[medium] Defaults-only CaMeL config skips hook fallback:** `src/agents/pi-tools.before-tool-call.ts:187`
  - body: The orchestrator path reads `agents.defaults.camel`, but the before-tool-call path does not, so documented defaults-level enablement can leave fallback or direct tool calls without the advertised CaMeL policy check.
