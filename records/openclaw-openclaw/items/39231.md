## Summary

Keep open. The feature direction is useful, but this PR is not merge-ready because source inspection still shows security-boundary bugs in taint propagation, side-effect replay, and approval-source trust, and the PR body still lacks after-fix real behavior proof.

## What This Changes

The PR adds an opt-in CaMeL prompt-injection defense across agent config, a new taint/policy/orchestration layer, embedded-runner and before-tool-call integration, exec approval forwarding, SECURITY.md documentation, and regression tests.

## Best Possible Solution

Land only a maintainer/security-approved prompt-injection boundary that preserves provenance end to end, prevents side-effect replay, derives approval trust inside trusted code, and includes focused tests plus redacted real runtime proof.

## Reproduction Assessment

Yes for the PR blockers by source inspection: the PR head shows the provenance drop, tool-enabled synthesis path, late replay guard, and client-supplied approval-source bypass. I did not run a live CaMeL scenario in this read-only review.

## Solution Assessment

No. CaMeL-style provenance may be a valid direction, but this branch is not the best mergeable form until the security-boundary defects, upgrade behavior, and proof gap are resolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve taint dependencies in tool results:** `src/agents/camel/orchestrator.ts:153`
  - body: The orchestrator wraps each tool result using only the callee name, so output from a trusted or no-side-effect intermediate tool can lose the tainted argument provenance that produced it. Carry the wrapped argument dependencies into the result before later side-effect policy checks.
  - confidence: 0.92
- **[P1] Disable tools for post-orchestration synthesis:** `src/agents/pi-embedded-runner/run/attempt.ts:2178-2182`
  - body: After CaMeL has already executed side-effect steps, this prompts the normal tool-enabled session and relies on prompt text asking the model not to re-run tools. Use a no-tool synthesis path or persist completed calls as non-executable context before prompting.
  - confidence: 0.88
- **[P1] Mark side effects before executing the rest of the plan:** `src/agents/pi-embedded-runner/run/attempt.ts:2154`
  - body: The replay guard is set only after orchestrator.execute returns successfully. If a later plan step throws after an earlier mutating step, the outer retry path can replay already-completed side effects.
  - confidence: 0.86
- **[P1] Do not trust client-supplied CaMeL approval source:** `src/infra/exec-approval-forwarder.ts:71-76`
  - body: The forwarder bypasses the approval forwarding enabled gate when request.source is "camel", but that source is accepted from the request payload. Derive this trust server-side from the CaMeL approval path instead of from caller-supplied RPC params.
  - confidence: 0.82
