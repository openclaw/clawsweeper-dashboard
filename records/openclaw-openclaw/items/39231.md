- **[P1] Preserve taint when wrapping tool results:** `src/agents/camel/orchestrator.ts:153`
  - body: `wrapToolResult` derives provenance only from the executed tool name, so output from a trusted or no-side-effect intermediate tool can drop tainted argument dependencies before a later side-effect check. Carry dependencies from `wrappedArgs` into the result value so taint survives multi-step plans.
  - confidence: 0.92
- **[P1] Disable tools during final synthesis:** `src/agents/pi-embedded-runner/run/attempt.ts:2178-2182`
  - body: After CaMeL has already executed side-effect steps, this prompts the normal tool-enabled session and relies on text telling the model not to re-run tools. Use a no-tool synthesis path or persist completed calls as non-executable context before prompting.
  - confidence: 0.88
- **[P1] Set the replay guard before side effects can fail:** `src/agents/pi-embedded-runner/run/attempt.ts:2154`
  - body: `camelSideEffectsExecuted` is set only after `orchestrator.execute()` returns. If one mutating step succeeds and a later step throws, the outer retry loop can enter CaMeL again and replay already-completed side effects.
  - confidence: 0.84
- **[P2] Merge defaults into hook CaMeL config:** `src/agents/pi-tools.before-tool-call.ts:187-195`
  - body: The embedded runner resolves `agents.defaults.camel`, but the before-tool-call fallback only merges `agents.camel` and per-agent overrides. Defaults-only enablement can fall back to the normal tool loop without the configured CaMeL guard.
  - confidence: 0.87
- **[P2] Re-check CaMeL after hook param rewrites:** `src/agents/pi-tools.before-tool-call.ts:395-397`
  - body: CaMeL policy runs before `before_tool_call` hooks can merge replacement params, but the rewritten params are what the tool executes. Re-run the CaMeL decision after hook rewrites or move the check after final params are known.
  - confidence: 0.78
- **[P2] Derive the approval source inside trusted CaMeL code:** `src/infra/exec-approval-forwarder.ts:71-75`
  - body: The new `source` field is accepted from `exec.approval.request`, and `source: "camel"` bypasses the forwarding enabled gate. Derive this marker inside trusted CaMeL approval code instead of allowing any approval-request client to select the bypass path.
  - confidence: 0.82
