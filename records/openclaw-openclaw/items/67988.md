## Summary

Keep open. Current main still builds the Agents/Control UI model dropdown from configured aliases or raw model ids before catalog display metadata can replace them, and the open closing PR at https://github.com/openclaw/openclaw/pull/75157 has not merged yet.

## What This Changes

The issue requests that the Control UI model dropdown render full catalog display names for Claude Opus 4.7 and Claude Sonnet 4.6 instead of shorthand aliases or raw model keys.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/75157 or an equivalent focused change that uses catalog display metadata for configured Agents model options while preserving intentional custom aliases.

## Reproduction Assessment

Yes. Source inspection on current main shows configured aliases/raw ids are emitted before matching catalog display metadata can replace them, matching the reported `opus` and `sonnet` dropdown labels.

## Solution Assessment

Yes. Reusing the existing catalog display-name lookup in the Agents option builder is the narrow maintainable fix; provider-owned catalog metadata should supply any missing model display name.
