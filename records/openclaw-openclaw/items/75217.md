## Best Possible Solution

Keep the resolver-level fix, but suppress only the default workspace-file load under `skipBootstrap` while preserving hook overrides, filtering, sanitization, schema/help surfaces, generated docs, and changelog coverage.

Yes. Current main is source-reproducible by creating existing workspace bootstrap files, setting `agents.defaults.skipBootstrap: true`, and observing `resolveBootstrapFilesForRun` still loads them; the PR regression is source-reproducible because the new early return exits before hook execution.

## Solution Assessment

No. The shared resolver is the right place to fix the bug, but returning immediately is too broad; start from an empty workspace-file set when `skipBootstrap` is true and continue through the existing hook/filter/sanitize pipeline.
