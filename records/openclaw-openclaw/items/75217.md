## Summary

Keep open. The PR is a plausible narrow fix for the runtime bootstrap bug, but it is not merge-ready because the public config-schema metadata gap remains and the supplied proof is still only a focused test run rather than real CLI/Gateway behavior.

## What This Changes

The PR changes agent bootstrap-file resolution so `agents.defaults.skipBootstrap` skips existing workspace bootstrap files at runtime while preserving `agent:bootstrap` hook-injected files, with docs/JSDoc and resolver tests updated.

## Best Possible Solution

Land the resolver-level workspace-file skip with hook preservation after adding schema label/help metadata for the new contract and real agent/Gateway proof that `systemPromptReport.injectedWorkspaceFiles` drops under `skipBootstrap: true`.

## Reproduction Assessment

Yes. Current-main source inspection shows `resolveBootstrapFilesForRun` still loads workspace bootstrap files without reading `skipBootstrap`, while the creation path already uses the flag.

## Solution Assessment

No, not merge-ready yet. The resolver boundary is the right maintainable shape, but the patch should also update the schema label/help contract and provide real runtime proof instead of test-only output.
