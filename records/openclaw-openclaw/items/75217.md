## Summary

Keep this PR open: the resolver fix is pointed at a real bug and the hook-preserving shape looks right, but the public config schema metadata still does not document `agents.defaults.skipBootstrap`, and the supplied proof is still a focused resolver test rather than a real runtime `systemPromptReport` run.

## What This Changes

The PR changes agent bootstrap-file resolution so `agents.defaults.skipBootstrap` suppresses existing workspace bootstrap files at runtime while preserving `agent:bootstrap` hook-injected files, and updates related docs, JSDoc, and tests.

## Best Possible Solution

Land the resolver-level workspace-file skip with hook preservation after adding matching schema label/help metadata and real `openclaw agent --json` proof that workspace bootstrap files disappear from `systemPromptReport` under `skipBootstrap: true`.

## Reproduction Assessment

Yes for source-level reproduction: current main loads workspace bootstrap files in `resolveBootstrapFilesForRun` without consulting `agents.defaults.skipBootstrap`, while workspace creation already honors the flag. I did not run a live current-main agent turn in this read-only review.

## Solution Assessment

No, not merge-ready yet. The resolver-level boundary is the right maintainable direction, but schema metadata and real runtime proof still need to be completed before merge.
