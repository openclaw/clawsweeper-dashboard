## What This Changes

The PR adds a runtime `skipBootstrap` guard to bootstrap file resolution, updates docs/JSDoc, and adds regression tests for existing workspace bootstrap files.

## Best Possible Solution

Keep the resolver-level fix, but suppress only the automatic workspace-file load under `skipBootstrap` while preserving explicit bootstrap hooks, sanitization, schema/help surfaces, generated config docs, and changelog coverage.

Yes. Current main has a high-confidence source reproduction: existing workspace bootstrap files are loaded by `resolveBootstrapFilesForRun` even when `agents.defaults.skipBootstrap` is true; I did not run a live filesystem test because this review is read-only.

## Solution Assessment

No. The shared resolver is the right place to fix the issue, but returning immediately is too broad; the maintainable path is to start from no default workspace files under `skipBootstrap` and continue through hook overrides and sanitization.
