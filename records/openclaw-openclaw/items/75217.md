## Summary

Keep open. This is a focused and recently reviewed bug-fix PR, and the resolver shape looks right, but it still lacks contributor real behavior proof and leaves the config schema metadata path inconsistent with the new `skipBootstrap` contract.

## What This Changes

The PR makes `agents.defaults.skipBootstrap` suppress existing workspace bootstrap files during runtime bootstrap resolution while still allowing `agent:bootstrap` hook-injected files, with docs/JSDoc and resolver tests updated.

## Best Possible Solution

Land the resolver-level workspace-file skip with hook preservation after adding schema label/help metadata and contributor-supplied real CLI or Gateway proof that the runtime report no longer injects workspace bootstrap files under `skipBootstrap: true`.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main loads workspace bootstrap files in `resolveBootstrapFilesForRun` without reading `agents.defaults.skipBootstrap`, while workspace creation already honors the flag. I did not run a live current-main Gateway reproduction in this read-only pass.

## Solution Assessment

Mostly yes, but not merge-ready. The resolver boundary and hook-preservation shape are the right maintainable fix, but the patch should also update schema label/help metadata and the contributor should provide real after-fix CLI or Gateway proof.
