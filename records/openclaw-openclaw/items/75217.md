The PR adds a `skipBootstrap` guard to runtime bootstrap-file resolution, documents runtime injection behavior, updates the TypeScript config comment, and adds regression tests for existing workspace bootstrap files.

## Best Possible Solution

Keep the resolver-level fix, but make `skipBootstrap` start from an empty default workspace-file list while still running hook overrides and sanitization; then add hook-under-skipBootstrap regression coverage, align the zod/generated schema help, and add the required changelog bullet.

Yes. On current main, create `AGENTS.md`/`SOUL.md`, set `agents.defaults.skipBootstrap: true`, and `resolveBootstrapContextForRun` still returns context files; on the PR branch, enable `bootstrap-extra-files` with `skipBootstrap: true` and the early return prevents hook-added files from appearing.

## Solution Assessment

No. The shared resolver is the right place to fix the original bug, but returning before hook processing is broader than necessary; the safer shape is to suppress default workspace files while preserving explicit hook context and public schema consistency.
