## Summary

Keep open. Current main still has the reported runtime `skipBootstrap` gap, and this PR is directionally useful, but the head is not merge-ready because the resolver guard skips explicit bootstrap hooks, the config schema/help remains stale, and the user-facing fix still needs a changelog entry.

## What This Changes

The PR adds a runtime `skipBootstrap` guard in bootstrap-file resolution, updates agent config docs/JSDoc, and adds regression coverage for existing workspace bootstrap files.

## Best Possible Solution

Keep the resolver-level fix, but suppress only the default workspace-file load under `skipBootstrap` while preserving hook overrides, sanitization, schema/help surfaces, generated config docs, and changelog coverage.

## Reproduction Assessment

Yes. Current main can be reproduced by placing existing bootstrap files in the workspace, setting `agents.defaults.skipBootstrap: true`, and observing that `resolveBootstrapFilesForRun` still loads them; the PR regression is source-reproducible because the new early return exits before hook execution.

## Solution Assessment

No. The shared resolver is the right place to fix the bug, but returning immediately is too broad; start from an empty workspace-file set under `skipBootstrap` and continue through the existing hook/filter/sanitize pipeline.
