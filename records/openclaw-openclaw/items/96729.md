## Best Possible Solution

Make MCP null omission schema-aware before merge, preserve required or meaningful nulls, keep or narrow the validation change deliberately, and require focused regression coverage plus redacted real OpenClaw MCP proof.

## Reproduction Assessment

## Solution Assessment

No. Preserving null through validation is plausible, but unconditional top-level null stripping is too broad; the safer fix is schema-aware optional-null omission or preserving required nullable values by default.
