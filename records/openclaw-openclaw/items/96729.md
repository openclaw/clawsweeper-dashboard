## Summary

Keep open: the updated validation change addresses the likely null-to-empty-string coercion path, but the runtime filter still strips every top-level null before MCP dispatch and can break required nullable tool arguments; the PR also still has only unit-test proof, not real MCP behavior proof.

## What This Changes

The PR changes plain JSON-schema union coercion to prefer exact null matches, strips top-level null object entries before bundle-MCP `callTool`, and adds unit tests for both paths.

## Best Possible Solution

Keep the validation-layer null preservation if maintainers accept the broader union semantics, but make MCP omission schema-aware so only optional null-as-absent fields are dropped, then add validated-path regression coverage and redacted real MCP proof.

## Reproduction Assessment

No high-confidence live reproduction was run in this review. Source and reporter evidence show current main can coerce `null` through a string union branch and then forward validated args to MCP, but there is no real awslabs EKS MCP run or raw wire capture here.

## Solution Assessment

No. The validation-layer fix is plausible for the suspected coercion bug, but the unconditional runtime null filter is too broad; the maintainable fix is schema-aware optional-null omission or preserving required nullable values.
