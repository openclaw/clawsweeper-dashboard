## Summary

Keep open: this PR targets a real MCP nullable-argument bug, but the runtime-side filter drops every top-level `null` after validation, which can break MCP tools where an explicit required nullable value is meaningful; it also still lacks real MCP behavior proof.

## What This Changes

The PR changes plain JSON-schema union coercion to preserve exact `null` matches, strips top-level `null` entries before bundle-MCP `callTool`, and adds unit tests for both paths.

## Best Possible Solution

Keep the validation-layer null preservation if maintainers accept the broader union semantics, but make MCP omission schema-aware so only optional null-as-absent fields are dropped, then require redacted real MCP proof before merge.

## Reproduction Assessment

No high-confidence live reproduction was run in this review. Source and reporter evidence show the suspected null-to-empty-string path and MCP dispatch boundary, but there is no real awslabs EKS MCP run or raw wire capture here.

## Solution Assessment

No. The validation-layer change is plausible, but the unconditional runtime null filter is too broad; the safer fix is schema-aware optional-null omission or preserving required nullable values.
