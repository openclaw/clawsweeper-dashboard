## Summary

Keep open. The PR targets a real current-main gateway auth gap, but it is not mergeable as-is because bootstrap failure accounting can survive successful fallback auth, runnable DoS PoC scripts need maintainer security/disclosure review, and the required changelog entry is missing.

## What This Changes

The branch adds a bootstrap-token rate-limit scope/check in WebSocket connect auth, extends auth-context tests, and adds standalone bootstrap-token DoS PoC scripts under scripts/.

## Best Possible Solution

Land a minimal gateway auth fix that rate-limits failed bootstrap-token verification, defers or resets bootstrap failure accounting after successful fallback auth, covers the behavior with focused tests, adds the changelog entry, and keeps exploit PoC material out of the mergeable source tree unless a maintainer explicitly approves it.

## Reproduction Assessment

Yes. Current main reaches verifyDeviceBootstrapToken for bogus bootstrap-token WebSocket connects before any bootstrap-specific limiter, and that verifier serializes through module-level withLock over disk-backed state.

## Solution Assessment

No, not as-is. The limiter scope is the right direction, but the patch should fix fallback accounting, remove or relocate the runnable PoC scripts, and add the required changelog entry before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Reset bootstrap failures after fallback auth succeeds:** `src/gateway/server/ws-connection/auth-context.ts:197`
  - body: The new bootstrap-token `recordFailure` runs before device-token fallback can succeed. A reconnect that carries a stale `bootstrapToken` plus a valid `deviceToken` will keep incrementing the bootstrap limiter on successful auth, eventually blocking real bootstrap pairing from that IP until lockout expires; defer the failure or reset the bootstrap scope when fallback auth succeeds.
  - confidence: 0.86
- **[P2] Remove the executable DoS PoC scripts:** `scripts/poc-bootstrap-dos-real.mjs:1-16`
  - body: The runtime fix does not need to merge runnable scripts that demonstrate the bootstrap-token stall. Keeping these under `scripts/` publishes attack tooling and duplicated auth logic; prefer focused Vitest coverage or an advisory-only artifact unless maintainers explicitly approve public PoC scripts.
  - confidence: 0.81
- **[P3] Add the required changelog entry:** `src/gateway/server/ws-connection/auth-context.ts:160`
  - body: This changes non-test gateway auth behavior, but the PR does not update `CHANGELOG.md`. Add an Unreleased fix entry referencing this PR and crediting the contributor so the merge gate has the required user-facing release note.
  - confidence: 0.93
