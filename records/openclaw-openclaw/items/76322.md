## Summary

Keep open: the PR addresses the right Gateway bootstrap-token DoS path, and the supplied terminal proof now clears the real-behavior evidence gate, but the patch still only protects cases where an auth rate limiter is already present. Default non-browser deployments without `gateway.auth.rateLimit`, and default loopback-exempt cases, can still enter the mutex-backed verifier.

## What This Changes

The PR adds a bootstrap-token auth rate-limit scope, checks it before bootstrap-token verification, defers bootstrap failure accounting across device-token fallback, and expands auth-context coverage.

## Best Possible Solution

Land one canonical bootstrap-token fix that uses a dedicated always-on limiter before the mutex-backed verifier, covers unconfigured/configured/browser/loopback cases, and leaves broader alerting or sibling DoS work tracked separately.

## Reproduction Assessment

Yes. Source inspection on current main shows a signed device identity plus `auth.bootstrapToken` reaches `verifyDeviceBootstrapToken` before any bootstrap limiter, and that verifier enters `withLock`; the PR also supplies after-fix terminal output for the changed auth-decision path.

## Solution Assessment

No. Reusing the optional auth limiter is too narrow for this mutex-backed pre-auth path; the safer maintainable fix is a bootstrap-token-specific limiter that is created unconditionally and does not inherit the loopback exemption.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Use an always-on limiter before bootstrap verify:** `src/gateway/server/ws-connection/auth-context.ts:168`
  - body: Because this check only runs when `params.rateLimiter` exists, deployments without `gateway.auth.rateLimit` and default loopback-exempt non-browser clients can still enter `verifyBootstrapToken`, which is the mutex-backed path this PR is meant to protect. Please route bootstrap-token verifies through a dedicated limiter that is always constructed and not loopback-exempt.
