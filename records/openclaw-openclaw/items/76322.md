## Summary

Keep open. Current main has the reported bootstrap-token gap, and the PR is pointed at the right gateway auth surface, but the branch should not merge as-is because the new bootstrap failure accounting can penalize successful device-token fallback handshakes and the branch also ships runnable DoS PoC scripts.

## What This Changes

The PR adds a bootstrap-token auth rate-limit scope/check in WebSocket connect auth and adds standalone bootstrap-token DoS proof-of-concept scripts under `scripts/`.

## Best Possible Solution

Land a minimal gateway fix that rate-limits failed bootstrap-token verification without counting failures against handshakes that ultimately authenticate through a valid fallback, add focused auth-context regression tests and a changelog entry, and keep exploit PoC code out of the mergeable source tree unless a maintainer explicitly approves it.

## Reproduction Assessment

Yes. Current main reaches `verifyDeviceBootstrapToken` for bogus bootstrap-token WebSocket connects before any bootstrap-specific limiter, and that verifier serializes through a module-level lock over disk-backed bootstrap state.

## Solution Assessment

No, not as-is. The limiter scope is the right direction, but the patch should reset or defer bootstrap failure accounting across successful fallback auth and remove or relocate the runnable PoC scripts before maintainer merge review.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Reset bootstrap failures after fallback auth succeeds:** `src/gateway/server/ws-connection/auth-context.ts:196`
  - body: The new `recordFailure` runs before the device-token fallback has a chance to succeed. A handshake that carries a stale `bootstrapToken` plus a valid `deviceToken` will still increment the bootstrap-token limiter on every successful reconnect, and after the threshold the same IP is locked out of real bootstrap pairing until the lockout expires. Mirror the shared-secret pattern by resetting the bootstrap-token scope on fallback success, or defer recording until the final auth decision rejects.
  - confidence: 0.86
- **[P2] Remove the executable DoS PoC scripts:** `scripts/poc-bootstrap-dos-real.mjs:1-16`
  - body: The runtime fix does not need to merge runnable scripts that demonstrate the bootstrap-token stall. Keeping those under `scripts/` permanently ships attack tooling and duplicated security-sensitive auth logic; prefer focused Vitest coverage or a private advisory artifact for the PoC details.
  - confidence: 0.79
- **[P3] Add the required changelog entry:** `src/gateway/server/ws-connection/auth-context.ts:159`
  - body: This changes non-test gateway auth behavior, so the PR gate requires a `CHANGELOG.md` entry referencing this PR and crediting the contributor. Without it, the merge gate will fail with the repository's missing-changelog check.
  - confidence: 0.93
