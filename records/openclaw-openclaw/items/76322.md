Overall confidence: 0.88

Full review comments:

- **[P1] Record pending bootstrap failures when fallback is rate-limited:** `src/gateway/server/ws-connection/auth-context.ts:256-258`
  - body: When an invalid bootstrap token is followed by a device-token candidate whose bucket is already locked, `deviceTokenRateLimited` skips this branch entirely, so `pendingBootstrapFailure` is never recorded. An attacker can first lock the device-token scope and then keep sending bogus bootstrap tokens with any device-token candidate; each request still reaches `verifyBootstrapToken` while the bootstrap-token bucket never advances. Flush the pending bootstrap failure before returning from the rate-limited fallback path and cover that case in the tests.
  - confidence: 0.9
