Keep open: current main and the latest release still patch process-global stdout across an awaited config preflight, and an open fixing PR already closes this issue, so the issue should remain open until that PR is merged or rejected.

## What This Changes

The issue asks to stop config preflight suppression from swallowing unrelated concurrent stdout writes and callbacks.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/84439; the end state should suppress only OpenClaw preflight notes while preserving unrelated stdout writes and callbacks.

## Reproduction Assessment

Yes, at source level: call `ensureConfigReady()` with `suppressDoctorStdout: true`, keep the mocked preflight pending, and write to stdout from a concurrent task before the `finally` restore. I did not execute that repro because this review is read-only.

## Solution Assessment

Yes. The maintainable solution is the linked PR's direction: scope suppression to OpenClaw `note()` output instead of mutating `process.stdout.write`, while preserving JSON stdout cleanliness.
