## Summary

Keep open: current main and the latest release still patch process-global stdout across an awaited doctor preflight, and the existing tests cover preflight noise suppression but not unrelated concurrent output loss.

## What This Changes

The issue asks OpenClaw to stop suppressing unrelated concurrent stdout writes while hiding doctor preflight output for JSON-mode CLI startup.

## Best Possible Solution

Replace the global stdout monkey-patch with a scoped doctor-preflight note/output suppression contract, then add a regression proving unrelated concurrent stdout writes and callbacks still flow.

## Reproduction Assessment

Yes, at source level: call `ensureConfigReady()` with `suppressDoctorStdout: true`, keep the mocked preflight pending, and write to stdout from a queued concurrent task before the `finally` restore. I did not execute that repro because this review was read-only.

## Solution Assessment

Yes: the maintainable fix is to scope suppression to doctor preflight notes/output rather than mutating process-global stdout, while retaining the existing JSON stdout contract.
