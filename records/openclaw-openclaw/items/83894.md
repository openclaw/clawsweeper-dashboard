## Summary

Keep open: current main and the latest release still replace process-global stdout across an awaited config preflight, and the linked fix PR is open with closing syntax, so this issue should stay open until that PR lands or is rejected.

## What This Changes

The issue asks OpenClaw to stop suppressing unrelated concurrent stdout writes while config preflight output is hidden for JSON-mode CLI startup.

## Best Possible Solution

Review and land or reject the focused linked PR; if it is rejected, implement the same scoped note-output suppression without mutating `process.stdout.write`.

## Reproduction Assessment

Yes, at source level: call `ensureConfigReady()` with `suppressDoctorStdout: true`, keep the mocked preflight pending, then write to stdout from a queued concurrent task before the `finally` restore. Current main would route that write through the temporary no-op function.

## Solution Assessment

Yes. The maintainable solution is scoped doctor-preflight note suppression, as in the linked PR, rather than a process-global `stdout.write` monkey-patch across an await.
