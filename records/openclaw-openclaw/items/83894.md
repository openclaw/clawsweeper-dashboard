## Summary

Keep open: current main and the latest stable release still replace process-global stdout while awaiting the doctor config preflight, so unrelated concurrent stdout writes can be swallowed before the finally block restores the stream.

## What This Changes

The issue asks OpenClaw to stop suppressing unrelated concurrent stdout output while JSON-mode CLI startup runs the doctor config preflight.

## Best Possible Solution

Replace the process-global stdout monkey-patch with a scoped doctor-preflight output suppression contract, preserving clean JSON stdout while letting unrelated stdout writes and callbacks behave normally.

## Reproduction Assessment

Yes, at source level: call `ensureConfigReady()` with `suppressDoctorStdout: true`, keep `runDoctorConfigPreflight()` pending, and perform an unrelated queued `process.stdout.write` before the `finally` restore. I did not execute that repro because this review was read-only.

## Solution Assessment

Yes: the narrow maintainable fix is to scope suppression to doctor preflight note/output handling rather than mutating `process.stdout.write` across an awaited operation.
