## Summary

Keep open: current main still replaces process-global stdout with a no-op across an awaited doctor preflight in JSON-output startup, and the existing tests only prove doctor noise is hidden, not that unrelated concurrent stdout writes still work.

## What This Changes

The issue reports that CLI config preflight output suppression monkey-patches `process.stdout.write` across an awaited operation, which can suppress unrelated concurrent async output.

## Best Possible Solution

Replace the global stdout monkey-patch with a scoped doctor-preflight note/output suppression contract, then add regression coverage for concurrent stdout writes and write callbacks while keeping JSON stdout parseable.

## Reproduction Assessment

Yes, at source level: call `ensureConfigReady()` with `suppressDoctorStdout: true`, keep the mocked preflight pending, and write to stdout from another queued async task before the `finally` restore. I did not run that repro because this pass was read-only.

## Solution Assessment

Yes: the maintainable fix is to scope suppression to doctor preflight output instead of mutating process-global stdout, while retaining the existing JSON stdout contract.
