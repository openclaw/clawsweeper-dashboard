## Summary

Keep open: current main and the latest release still replace process-global stdout across an awaited config preflight, and the active closing PR has not merged yet.

## What This Changes

The issue reports that config preflight stdout suppression can drop unrelated concurrent async stdout writes and callbacks because it monkey-patches `process.stdout.write` globally.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84439, or an equivalent scoped note-suppression fix, then close this issue after the fix reaches `main`.

## Reproduction Assessment

Yes. Source inspection shows `ensureConfigReady()` can await the doctor preflight while `process.stdout.write` is replaced, so a queued concurrent stdout write during that interval would be swallowed on current main.

## Solution Assessment

Yes. The maintainable fix is to scope suppression to OpenClaw doctor note output, as the linked PR does, while preserving JSON stdout cleanliness.
