## Summary

Keep open: current main still has a source-visible WebChat/Gateway TTS live-delivery gap. The adjacent trusted-local-media fix shipped for https://github.com/openclaw/openclaw/issues/82628 improves the TTS payload trust gate, but the agent-run final chat event is still text-only and the same-author implementation PR remains open.

## What This Changes

Report that WebChat auto-TTS audio is generated and persisted but not included in the live final chat delivery, so the Control UI cannot play it in real time.

## Best Possible Solution

Fix the agent-run Gateway/WebChat final delivery path so the audio content appended to the transcript is also visible to the active Control UI, either in the final chat event or through a deliberate post-final history reload, without duplicating transcript entries.

## Reproduction Assessment

No live WebChat/TTS reproduction was run in this read-only review. Source inspection gives a high-confidence failure path: agent media is appended as a transcript supplement, while the live final event remains text-only and the UI can skip the deferred history reload.

## Solution Assessment

No. The proposed rawFinalPayloads removal targets the non-agent final assembly branch; the maintainable fix should address the agent-run final broadcast or reload semantics that keep the audio out of the active WebChat view.
