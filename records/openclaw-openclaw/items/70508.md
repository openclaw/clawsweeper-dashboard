## Summary

Keep this PR open. Current main still has the Mattermost draft-preview overwrite bug, and this branch is useful source work, but the PR head is not mergeable because it still has callback-order races and a deterministic status-text test contract mismatch.

## What This Changes

The PR changes Mattermost draft streaming to rotate draft preview posts at assistant/reasoning/tool boundaries and adds item-event based emoji/title tool-status preview text.

## Best Possible Solution

Land one focused Mattermost fix on current main that preserves streamed assistant text before assistant, reasoning, and tool boundaries, serializes draft boundary/partial/tool-status updates, and adds targeted Mattermost monitor or draft-stream tests. Keep the existing tool-status text contract unless maintainers explicitly choose the richer item-title UX and update tests/docs as needed.

## Reproduction Assessment

Yes. The current-main source path is enough to reproduce the reported behavior: partial replies edit one Mattermost draft post, assistant/reasoning boundaries only clear local text, and tool starts update that same draft post. The PR's test-contract regression is also deterministically visible from the unchanged draft-stream test expectations on the PR head.

## Solution Assessment

No. The intended boundary rotation is the right direction, but this PR is not the best mergeable form because it queues item guards too late, awaits boundary flushes in fire-and-forget callback paths, and changes status-text output without updating the test contract.
