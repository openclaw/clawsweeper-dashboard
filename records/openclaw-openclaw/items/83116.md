## Summary

Keep open: current main still has unguarded async SIGTERM/SIGUSR1 signal-handler paths, and a rejected lifecycle runtime import escapes before the shutdown or restart request is made. The report is narrow, source-backed, and suitable for a focused fix with regression coverage.

## What This Changes

The issue asks OpenClaw to catch lifecycle runtime import failures in gateway SIGTERM/SIGUSR1 handlers so shutdown or restart still proceeds deterministically.

## Best Possible Solution

Make the gateway signal handlers own their async failure path: log lifecycle import failures, drive deterministic shutdown/recovery fallback, and add focused run-loop regression tests.

## Reproduction Assessment

Yes, there is a high-confidence source-level reproduction path. If loadGatewayLifecycleRuntimeModule() rejects inside either signal listener, the async IIFE rejects before request(...) runs, and the Node event probe confirms that pattern becomes an unhandled rejection after the event emit returns.

## Solution Assessment

Yes, a local catch with logging and a deterministic fallback in the two signal-handler async paths is the narrowest maintainable fix. This should preserve the existing lazy lifecycle boundary and normal SIGUSR1 authorization behavior when the module loads.
