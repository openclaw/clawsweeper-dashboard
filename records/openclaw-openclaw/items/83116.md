## Summary

Keep open: current main still has the unguarded SIGTERM/SIGUSR1 async signal-handler paths, and the open linked PR is the right fix candidate rather than a reason to close the issue early.

## What This Changes

The issue requests `.catch()` handling around gateway SIGTERM/SIGUSR1 fire-and-forget async handlers so lifecycle import failures are logged and do not skip shutdown or restart handling.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/83131, or an equivalent focused run-loop fix with regression coverage, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes for source-level reproduction: current main awaits the lifecycle runtime import inside discarded SIGTERM/SIGUSR1 async IIFEs before `request(...)` runs, and a focused Node probe confirms that rejected shape becomes an unhandled rejection. No full live gateway fault-injection run was performed in this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix is to handle rejection at the two signal-handler async boundaries while preserving the existing lazy lifecycle module and normal SIGUSR1 authorization path. The linked open PR is the current implementation candidate, so the issue should remain open until that fix lands or is replaced.
