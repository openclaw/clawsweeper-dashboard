## Summary

Keep open: current main still has unguarded async SIGTERM/SIGUSR1 gateway signal-handler paths, and two open closing PRs already target the fix. The issue should remain open until one focused fix lands or the linked PRs are closed.

## What This Changes

The issue requests `.catch()` handling for SIGTERM/SIGUSR1 gateway signal-handler async imports so shutdown/restart requests are not skipped if the lifecycle runtime import rejects.

## Best Possible Solution

Land one focused linked fix that catches lifecycle import failures, logs them, preserves SIGUSR1 authorization semantics, and adds regression coverage in the run-loop tests.

## Reproduction Assessment

Yes, there is a high-confidence source-level reproduction path: if `loadGatewayLifecycleRuntimeModule()` rejects inside either signal listener, the async IIFE rejects before `request(...)` runs. I did not run a live gateway fault-injection repro in this read-only pass.

## Solution Assessment

Yes, local error handling in the two signal-handler async paths is the narrow maintainable fix. The linked PRs should be reviewed for the exact SIGUSR1 fallback semantics before landing.
