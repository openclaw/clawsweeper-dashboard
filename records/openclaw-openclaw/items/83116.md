## Summary

Keep open: current main still has the unguarded gateway signal-handler paths, and the issue is already paired with an open closing PR that should be reviewed or replaced before the issue closes.

## What This Changes

The issue reports that gateway SIGTERM and SIGUSR1 handlers discard async lifecycle-runtime import failures without a local catch, which can skip shutdown or restart handling.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/83131 if its fallback semantics are acceptable, or replace it with an equivalent focused fix plus run-loop regression coverage.

## Reproduction Assessment

Yes, at source level. The current signal handlers await a rejectable lazy import inside uncaught fire-and-forget IIFEs before calling request(...), and a Node probe confirms that shape becomes an unhandled rejection.

## Solution Assessment

Yes. A local catch around the two async signal-handler paths, with logging and a deterministic SIGTERM stop fallback while preserving normal SIGUSR1 authorization behavior, is the narrow maintainable fix.
