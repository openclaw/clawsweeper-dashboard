## Summary

Keep open: current main still has the unguarded SIGTERM/SIGUSR1 async signal-handler paths, and the open linked closing PR is the correct fix candidate rather than a reason to close this issue early.

## What This Changes

The issue asks for gateway SIGTERM/SIGUSR1 signal handlers to catch rejected lifecycle-runtime imports so shutdown or restart handling is not skipped.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/83131, or replace it with an equivalent focused run-loop fix, then close this issue after main contains the handled rejection path.

## Reproduction Assessment

Yes for source-level reproduction: current main awaits the lifecycle runtime import inside discarded SIGTERM/SIGUSR1 async IIFEs before `request(...)` runs, so a rejected import skips the shutdown/restart request path. I did not run a live gateway fault-injection repro in this read-only review.

## Solution Assessment

Yes, the narrow maintainable solution is to handle rejection at the two signal-handler async boundaries while preserving the existing lazy lifecycle module and normal SIGUSR1 authorization flow. The linked PR is the current implementation candidate, so this issue should wait for that review instead of spawning parallel repair work.
