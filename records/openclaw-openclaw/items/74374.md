## Summary

Keep open. Current main still has the reporter's disabled-thinking collision path: per-model `thinking: false` is ignored, DeepSeek V4 falls back to enabled thinking, and `extra_body.thinking.type=disabled` is applied after the provider wrapper can add `reasoning_content`; the linked open PRs should be reconciled before closure.

## What This Changes

The issue asks OpenClaw to avoid DeepSeek v4-pro 400 replay failures when disabled-thinking configuration and stored `reasoning_content` disagree across turns.

## Best Possible Solution

Land one cohesive DeepSeek V4 payload contract fix that covers canonical off aliases, boolean/string disabled settings, and `extra_body` collisions with regression tests.

## Reproduction Assessment

Yes. Static reproduction is high confidence: with `deepseek/deepseek-v4-pro`, `params.thinking: false` is ignored, the model default resolves to enabled thinking, the DeepSeek wrapper can add `reasoning_content`, and post-plugin `extra_body.thinking.type=disabled` then overwrites the thinking flag.

## Solution Assessment

No. Current main is only a partial adjacent fix; the narrow maintainable solution should normalize disabled thinking aliases before provider wrapping and make post-merge `extra_body` disabled cleanup deterministic.
