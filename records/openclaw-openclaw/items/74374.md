## Summary

Keep open. Current main has the adjacent DeepSeek V4 enabled-thinking replay fix, but the reporter's disabled-thinking configuration is still not covered because `params.thinking: false` is ignored and `extra_body.thinking` can overwrite the provider wrapper after `reasoning_content` has already been added; the open closing PRs should be reconciled before closure.

## What This Changes

The issue reports DeepSeek v4-pro multi-turn 400s when thinking is disabled via `extra_body` because `reasoning_content` can remain in replayed history.

## Best Possible Solution

Review and land one cohesive DeepSeek V4 fix that normalizes disabled thinking aliases and prevents post-`extra_body` payload collisions, with regression coverage for the reporter's exact config.

## Reproduction Assessment

Yes. Static reproduction is high confidence: with `params.thinking: false` plus `extra_body.thinking.type="disabled"`, current main can resolve DeepSeek V4 thinking to `high`, add replay `reasoning_content`, and then overwrite `thinking` to disabled afterward; the live API failure frequency remains intermittent.

## Solution Assessment

No. Current main is only a partial solution; the narrow maintainable fix is to reconcile #74403 and #74418 or equivalent changes into one tested contract before closing this issue.
