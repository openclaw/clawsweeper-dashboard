## What This Changes

The report asks OpenClaw to prevent DeepSeek V4 Pro disabled-thinking sessions from replaying inconsistent `reasoning_content` history that triggers a 400 on the next turn.

## Best Possible Solution

Land one cohesive DeepSeek V4 fix from #74403/#74418 or an equivalent replacement that normalizes disabled aliases before provider wrapping and cleans replay fields consistently after `extra_body` sets disabled thinking, with regression coverage for the reporter's config.

## Reproduction Assessment

Yes. The static reproduction path is high-confidence: `params.thinking: false` is ignored, DeepSeek V4 falls back to `high`, the wrapper can add `reasoning_content`, and `extra_body.thinking.type="disabled"` can later overwrite the request thinking mode.

## Solution Assessment

No. Current main is not the best solution yet because it covers the enabled-thinking replay case but not the disabled alias and post-`extra_body` collision paths together.
