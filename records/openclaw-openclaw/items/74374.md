## Summary

Keep open. Current main still allows the reported DeepSeek V4 disabled-thinking collision: boolean `params.thinking: false` is not normalized to `off`, the DeepSeek V4 wrapper can still backfill `reasoning_content`, and `extra_body.thinking.type=disabled` is applied afterward without cleaning that payload shape.

## What This Changes

Make DeepSeek V4-pro disabled-thinking multi-turn requests avoid `reasoning_content` replay collisions when `extra_body` disables thinking.

## Best Possible Solution

Land one cohesive DeepSeek V4 payload-contract fix that normalizes disabled thinking aliases and makes disabled `extra_body` collisions deterministically remove or neutralize `reasoning_content`, with focused regression tests.

## Reproduction Assessment

Yes. Static reproduction is high confidence: `params.thinking: false` falls through to the DeepSeek V4 `high` default, the wrapper can add `reasoning_content`, and `extra_body.thinking.type=disabled` then overwrites the payload without cleanup.

## Solution Assessment

No. Current main is not the complete fix, and the two open PRs are partial unless reconciled into one behavior that covers both disabled aliases and post-wrapper `extra_body` collisions.
