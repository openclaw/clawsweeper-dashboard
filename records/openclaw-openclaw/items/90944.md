## Summary

Keep open: the report is a current, source-supported message-delivery bug, and current main does not prove it fixed. The latest main still has the completion direct-text fallback that can mirror raw child output, while the documented contract says the requester/parent session owns user-facing synthesis.

## What This Changes

Reported bug where a `sessions_yield` parent resume reply is recorded in the transcript but not delivered, while the subagent announce direct-text mirror sends the raw child output instead.

## Best Possible Solution

Make the parent/requester completion handoff authoritative: when a yielded parent produces a resume reply, deliver or retry that reply and suppress the raw child `:text-direct` mirror for the same completion.

## Reproduction Assessment

No high-confidence live current-main reproduction was run. The reporter's transcript evidence is detailed, and source inspection shows the same direct-text fallback and completion handoff paths still exist on current main.

## Solution Assessment

No, current behavior is not the best solution because it can fall back to raw child text after a parent handoff lacks visible delivery evidence. The safer repair is to tie announce fallback to requester-delivery proof and avoid direct child-text delivery when a parent resume reply exists for the same completion.
