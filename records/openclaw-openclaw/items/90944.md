## What This Changes

The issue reports that a yielded parent session records its resume reply but the external channel receives the child subagent's raw text-direct announce mirror instead.

## Best Possible Solution

Review and land a narrow agents fix that makes the yielded parent/requester completion authoritative, suppresses child raw text-direct delivery for sessions_yield completions, and preserves cleanup without falsely crediting subagent delivery.

## Reproduction Assessment

Yes, we have a high-confidence source and report reproduction path: spawn a run-mode subagent from an external-channel parent, yield for completion, then compare the parent transcript to the delivered channel message. I did not run a fresh live channel repro in this read-only review, so the structured status is source-reproducible rather than reproduced.

## Solution Assessment

Yes, the best fix is a narrow runtime coordination change in the subagent announce/yield lifecycle, not a channel-specific workaround or new configuration option. The linked open PR targets that boundary, but the issue should stay open until that fix or an equivalent one merges.
