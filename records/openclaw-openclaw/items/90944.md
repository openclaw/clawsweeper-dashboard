## Summary

Keep open: current main and the latest release still retain the raw child `:text-direct` fallback and do not carry a durable sessions_yield completion marker, while https://github.com/openclaw/openclaw/pull/91370 is an open, unmerged closing PR for this exact issue.

## What This Changes

The issue reports that a sessions_yield parent resume reply is recorded in the transcript but not delivered, while the subagent auto-announce direct-text mirror delivers the child’s raw result to the external channel.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/91370 or a replacement that makes the parent/requester completion handoff authoritative, suppresses the child raw `:text-direct` mirror for yielded completions, and proves the parent route delivers the visible reply.

## Reproduction Assessment

Yes at source level: current main still clears the sessions_yield pause marker before announce cleanup and still has the direct-message `:text-direct` fallback from child completion output. I did not run a fresh live Feishu/WhatsApp reproduction in this review.

## Solution Assessment

Yes for the requested direction: the docs make parent/requester delivery authoritative, so the fix should prevent raw child direct-text delivery for yielded completions and rely on the parent route with delivery proof. The current open PR is the right review target rather than opening another repair lane.
