## Summary

Keep open: independent Feishu and WhatsApp evidence shows the intended parent reply being replaced by a raw child `delivery-mirror` message, and current main still contains the matching direct-first fallback. The open closing PR addresses fallback suppression, while a second open PR addresses parent-route preservation; neither has merged, and the newly reported retained-lane condition remains unresolved.

## What This Changes

Fix yielded subagent completion routing so the parent’s resumed response is delivered once, raw child announce text is suppressed, and terminal session ownership is released.

## Best Possible Solution

Treat a yielded completion as one parent-owned transaction: preserve the parent’s external route, deliver or durably retry its resumed reply, suppress the child `:text-direct` mirror, and guarantee that every terminal or recovery path leaves no active lane owner.

## Reproduction Assessment

Yes. Independent Feishu and WhatsApp live traces show the same real parent response followed by a synthetic `:text-direct` child mirror, including a reproduction on v2026.6.5; the observed sequence matches current-main source decisions.

## Solution Assessment

No, unconditional fallback suppression alone is not the complete solution. The narrow maintainable repair must also preserve the parent route, make parent delivery authoritative and retryable, and prove terminal lane-owner cleanup.
