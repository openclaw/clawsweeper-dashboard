## What This Changes

The issue reports that a `sessions_yield` parent resume reply is recorded in the transcript but not delivered, while the subagent `delivery-mirror` `:text-direct` fallback sends raw child output to the external channel.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/91370 or an equivalent narrow fix, then close this issue with source and release proof once yielded completions suppress raw child text-direct delivery and preserve parent reply ownership.

## Reproduction Assessment

Yes, but through source inspection and detailed reporter traces rather than a fresh live run in this read-only pass. Current main still has the direct-first completion path, `:text-direct` mirror fallback, and pauseReason clearing sequence that match the Feishu and WhatsApp reproductions.

## Solution Assessment

Yes, the best fix direction is to make the parent/requester completion handoff authoritative for yielded completions and suppress the child raw-text fallback for that same completion. The open linked PR targets that boundary, but this issue should remain open until that PR lands or maintainers choose a replacement.
