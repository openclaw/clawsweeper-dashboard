## Summary

Keep open: current main and v2026.6.9 still clear the sessions_yield pause marker before announce cleanup and still keep the child `:text-direct` fallback path; the open linked PR is the right fix candidate, not a reason to close the issue before merge.

## What This Changes

The issue reports that a sessions_yield parent resume reply is recorded in the transcript but not delivered, while the subagent auto-announce text-direct mirror sends the child raw result to the user.

## Best Possible Solution

Make the parent/requester completion handoff authoritative for yielded subagent completions, suppress the raw child text-direct mirror for that completion, and close this issue only after the linked fix or an equivalent lands.

## Reproduction Assessment

Yes at source level: current main clears the sessions_yield pause marker before announce cleanup and still has the direct text fallback path that creates `:text-direct` mirrors. I did not run a live Feishu or WhatsApp reproduction in this read-only review.

## Solution Assessment

Yes at the direction level: the fix belongs in the subagent lifecycle-to-announce handoff so yielded completions skip raw child direct-text delivery. The linked PR is the active candidate, but it still needs normal maintainer review before this issue can close.
