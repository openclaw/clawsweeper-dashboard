## Best Possible Solution

Review and land, or replace, https://github.com/openclaw/openclaw/pull/91370 with proof that sessions_yield completions are parent-owned and do not also emit raw child :text-direct mirrors; keep separate stuck-lane recovery work linked if it remains after that fix.

## Solution Assessment

Yes for the issue direction: the parent/requester completion handoff should be authoritative and the raw child :text-direct mirror should be suppressed for yielded completions. The linked fix PR is the current best implementation candidate, pending maintainer review and live/route proof.
