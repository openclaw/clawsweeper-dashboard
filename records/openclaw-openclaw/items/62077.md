## Summary

Keep open: current main still emits full inbound Conversation info and Sender metadata for direct non-webchat channels, including Telegram DMs, and the compact metadata implementation remains open and unmerged.

## What This Changes

Add a configurable minimal/compact inbound metadata mode for direct chats to reduce repeated DM prompt metadata and long-session context bloat.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/72749 or an equivalent backward-compatible opt-in compact direct metadata mode; if that path is abandoned, keep the remaining design consolidated with https://github.com/openclaw/openclaw/issues/72704.

## Reproduction Assessment

Yes for the observable prompt-shape complaint: source inspection shows Telegram DMs become direct telegram context, direct non-webchat turns emit full Conversation/Sender blocks, and get-reply-run prepends that prefix. I did not live-measure the hallucination impact.

## Solution Assessment

Unclear as a final product decision. An opt-in compact direct metadata mode is plausible, but the exact config contract and safety-preserving metadata shape should be reviewed through the linked implementation before this issue closes.
