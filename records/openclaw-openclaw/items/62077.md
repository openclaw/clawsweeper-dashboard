Keep open: current main still emits full inbound Conversation info and Sender metadata for direct non-webchat channels, and the referenced compact metadata work remains an open implementation candidate rather than shipped behavior.

## What This Changes

The issue requests an opt-in compact/minimal inbound metadata mode for direct Telegram chats to reduce repeated prompt metadata in long DM sessions.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/72749 or an equivalent backward-compatible compact direct metadata mode, with tests proving direct DMs can opt in while full untrusted metadata remains available for groups and rich contexts.

## Solution Assessment

Unclear as a final product decision. An opt-in compact direct metadata mode is a plausible maintainable fix, but the exact config contract and safety-preserving metadata shape should be reviewed through the linked implementation before this issue closes.
