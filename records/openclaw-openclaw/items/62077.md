## Summary

Keep open. Current main still emits full untrusted Conversation info and Sender JSON blocks for direct non-webchat channels, including Telegram DMs, and the requested compact/minimal inbound metadata config has not merged; #72749 is the open implementation candidate rather than proof of completion.

## What This Changes

Request a compact or minimal inbound metadata mode for direct chats so repeated Telegram DM metadata does not bloat prompts.

## Best Possible Solution

Review and land #72749 or an equivalent backward-compatible compact direct metadata implementation, including schema/docs and prompt-builder plus sanitizer tests, then close this issue and #72704 after merge or release.

## Reproduction Assessment

Yes. A unit-level call to `buildInboundUserContextPrefix` with `ChatType: "direct"` and `OriginatingChannel: "telegram"` follows the same direct external-channel branch that renders full Conversation info and Sender blocks; `getReplyRun` then prepends that prefix on normal inbound turns.

## Solution Assessment

Unclear until #72749 is reviewed. The compact direct-chat config direction is the narrow maintainable solution, but current main still lacks it and the open PR must be validated before this can be considered solved.
