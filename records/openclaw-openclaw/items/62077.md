## Summary

Keep open. Current main still prepends full `Conversation info` and `Sender` JSON metadata for direct external-channel messages, including Telegram DMs, and there is no shipped or current-main compact/minimal inbound metadata config surface. The related compact metadata work is an open implementation candidate, so this issue should remain open until that PR or an equivalent fix merges.

## What This Changes

Request an optional minimal or compact inbound metadata mode for direct chats, especially Telegram DMs, to avoid repeating full Conversation info and Sender JSON in every user prompt.

## Best Possible Solution

Land a backward-compatible core prompt/config change that emits compact untrusted metadata for direct external-channel chats while preserving full metadata for groups, replies, forwards, locations, structured context, and history; include schema/help/docs updates plus sanitizer and prompt-builder tests, then close this issue and #72704 after merge or release.

## Reproduction Assessment

Yes. A focused unit path using `buildInboundUserContextPrefix({ ChatType: "direct", OriginatingChannel: "telegram", MessageSid, SenderName, SenderId })` will take the direct external-channel branch and produce the full Conversation info and Sender blocks; `getReplyRun` prepends that prefix on each inbound turn.

## Solution Assessment

Unclear until the open PR is reviewed and merged. The requested compact direct-chat mode is a maintainable direction, and #72749 appears to target the narrow fix boundary, but current main does not yet implement it.
