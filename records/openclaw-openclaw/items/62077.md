## Summary

Keep open: current main still emits full inbound Conversation info and Sender metadata for direct non-webchat channels, and the compact direct metadata implementation is still an open PR rather than merged behavior.

## What This Changes

The issue requests an optional minimal inbound metadata mode for direct chats so Telegram DMs do not prepend repeated full Conversation info and Sender JSON blocks on every message.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/72749 or an equivalent backward-compatible compact direct metadata implementation with schema/docs/tests while preserving full untrusted metadata for richer chat contexts.

## Reproduction Assessment

Yes. Source inspection shows Telegram DMs build direct Telegram channel context, buildInboundUserContextPrefix emits full Conversation info and Sender blocks for direct non-webchat channels, and get-reply-run prepends that prefix to normal inbound turns.

## Solution Assessment

Yes as a direction, but not solved yet. A narrow opt-in compact direct-chat metadata mode is maintainable, but the open PR needs review and merge before this issue can close.
