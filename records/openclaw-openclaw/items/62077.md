## Summary

Keep open: current main still emits full Conversation info and Sender blocks for direct non-webchat inbound turns, and the related compact-direct implementation is still an open unmerged PR rather than shipped behavior.

## What This Changes

Request an opt-in minimal or compact inbound metadata mode for direct Telegram/DM chats to reduce repeated prompt metadata.

## Best Possible Solution

Review, repair, and land https://github.com/openclaw/openclaw/pull/72749 or an equivalent backward-compatible compact direct metadata mode that keeps rich contexts on full untrusted metadata.

## Reproduction Assessment

Yes, source-reproducible. Telegram DMs build direct Telegram context, current main includes full Conversation info and Sender blocks for direct non-webchat channels, and get-reply-run prepends that prefix into inbound prompt construction.

## Solution Assessment

Unclear as a final product shape. A narrow opt-in compact direct metadata mode is plausible, but the implementation should preserve attribution/sanitization contracts and needs maintainer review before this issue can be closed.
