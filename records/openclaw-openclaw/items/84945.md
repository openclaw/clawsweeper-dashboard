## What This Changes

The issue reports that gateway chat handling drops resolved LLM idle-timeout `isError` payloads after an agent run has started instead of broadcasting a terminal chat error.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84953, or an equivalent gateway change, so resolved delivered `isError` payloads after agent start emit a terminal chat error and error dedupe state.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: make `dispatchInboundMessage` call `onAgentRunStart` and resolve with a final payload containing `isError: true`; the current post-dispatch branch reaches only transcript update plus ok dedupe. I did not execute a live provider timeout in this read-only review.

## Solution Assessment

Yes, the linked gateway fix is the narrow maintainable path: inspect delivered error payloads in the agent-run post-dispatch branch, broadcast the existing chat error event, preserve transcript update ordering, and store error dedupe state.
