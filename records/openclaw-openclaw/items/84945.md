## What This Changes

The issue asks OpenClaw to broadcast a terminal chat error when an agent run has already started but later resolves with an `isError` LLM idle-timeout payload.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84953 or an equivalent gateway fix that broadcasts resolved `isError` replies and records error dedupe state without duplicating normal transcript entries.

## Reproduction Assessment

Yes. Source inspection shows an agent run can resolve with a delivered `isError` timeout payload, while the current `agentRunStarted` post-dispatch branch only updates the transcript and caches ok.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the gateway post-dispatch path, with regression coverage for terminal chat error broadcast and dedupe replay; the open linked PR follows that shape.
