## What This Changes

The issue asks gateway `chat.send` to broadcast a terminal `state: "error"` event when an agent-run dispatch resolves with delivered replies marked `isError: true`.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/84953, or an equivalent gateway fix, so resolved delivered error payloads after agent start emit `ChatErrorEvent`, cache an error dedupe result, and preserve transcript ordering.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: make `dispatchInboundMessage` call `onAgentRunStart` and then resolve with a final delivered payload containing `isError: true`; current main reaches the `agentRunStarted` post-dispatch branch without `broadcastChatError`.

## Solution Assessment

Yes. The linked PR's direction is the narrow maintainable fix because the gateway post-dispatch branch owns live chat broadcasts and dedupe replay for resolved delivery payloads.
