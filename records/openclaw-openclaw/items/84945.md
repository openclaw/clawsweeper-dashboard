## What This Changes

The issue reports that resolved LLM idle-timeout error payloads after `agentRunStarted` are persisted but not broadcast as terminal chat errors to connected gateway clients.

## Best Possible Solution

Land the focused gateway fix that broadcasts agent-started returned `isError` payloads and records the matching terminal error dedupe state, then let the merged PR close this issue.

## Solution Assessment

Yes, the narrow maintainable solution is in the gateway post-dispatch path: convert returned terminal error payloads into a `chat` error event and consistent failure dedupe state. The open PRs are implementation candidates, so this issue should remain open until one lands.
