## Summary

Keep open: current main still has a source-supported cron early-return path that can record interim text as a successful cron result, and the pi package export side is not fixed or superseded on main.

## What This Changes

The issue reports cron-triggered agent runs in OpenClaw 2026.5.7 returning interim or empty final messages without completing tool-backed work, plus one helper path showing bundled pi package export errors.

## Best Possible Solution

Fix the cron completion path so a still-running background tool/process followed by empty or interim assistant output cannot be recorded as a successful completed cron run, then split the pi package export failure into a dependency/package follow-up if it remains reproducible.

## Reproduction Assessment

Partly: the cron early-return symptom has a high-confidence source path and an independent Linux transcript showing the failure shape. The full macOS/Gmail helper flow and pi package export errors were not live-reproduced in this review.

## Solution Assessment

No close or duplicate path is appropriate. The likely best fix is structural cron liveness detection around still-running tool/process results rather than only adding more text hints; the dependency export side should not be hidden by that cron fix.

## Repair Work Prompt

Fix the cron early-return bug described in https://github.com/openclaw/openclaw/issues/79524. Current main can record an isolated cron run as `ok` when a background exec/process tool is still running, the next assistant turn is empty, and prior interim assistant text remains as the run summary. Prefer a structural guard over broad text-only hint expansion: detect the still-running tool/process + empty/non-final assistant pattern and either continue polling/re-prompt or surface a non-ok incomplete-run error so cron does not kill the background process and report success. Keep the fix scoped to cron/embedded-run completion semantics; do not rename or upgrade the `@mariozechner/pi-*` dependencies in this PR. Add focused regression coverage for the reported Claude/Anthropic-style path with visible interim text plus empty final assistant turn, and preserve existing successful final-summary behavior. Add a user-facing changelog entry because this changes cron reliability behavior.
