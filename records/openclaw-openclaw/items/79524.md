## Repair Work Prompt

Fix the cron early-return regression reported at https://github.com/openclaw/openclaw/issues/79524. Scope the PR to the cron/embedded-runner behavior where a cron agent starts or polls long-running tool/process work, receives a still-running result, then returns an empty assistant turn or interim text that cron records as `status: ok`. Prefer structural detection over broad text matching: if the latest tool/process result indicates work is still running and no final assistant content or delivery occurred, retry/continue or surface a clear incomplete-run error instead of marking success. Keep the fix provider-neutral where safe, but do not make unrelated dependency/package namespace changes for the @mariozechner/pi-* export errors in this PR. Add regression coverage for the two reported shapes: interim summary such as 'Let me wait longer'/'Now let me run the pipeline' and empty assistant content after a `process.poll` or `exec` still-running tool result. Update CHANGELOG.md if the behavior change is user-facing. Stop and ask for maintainer guidance if the implementation requires a new config option or changes dependency versions.
