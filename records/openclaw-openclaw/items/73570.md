## Summary

Keep open. Current main still has the linked hook gap, but this PR is not merge-ready: it emits the new pre-send event for cancelled or blanked payloads, reports stale content after rewrites, and does not reach standalone hooks registered as `message_sending`; after-fix real behavior proof is also absent.

## What This Changes

The branch adds a fire-and-forget internal `message`/`sending` hook in shared outbound delivery and updates outbound delivery tests to expect it.

## Best Possible Solution

Define and document the pre-send internal hook contract, then emit it only for deliverable payloads with effective post-plugin content and compatibility tests for the intended standalone hook key.

## Reproduction Assessment

Yes, at source level. The PR head inserts the new fire-and-forget call before cancel and empty-payload delivery gates, uses the original payload summary, and current internal dispatch only matches type or colon-style keys.

## Solution Assessment

No. Bridging in shared outbound delivery is the right area, but this patch should first settle the event key and semantics, then use the effective deliverable payload after rewrite and cancellation handling.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.96

Full review comments:

- **[P2] Do not notify hooks for messages that will not send:** `src/infra/outbound/deliver.ts:1086`
  - body: The new internal hook block runs before `hookResult.cancelled` and before the later `!effectivePayload` skip. A plugin `message_sending` hook can cancel or blank the payload, so this can emit a pre-send notification for a message that never reaches the channel.
  - confidence: 0.96
- **[P2] Pass rewritten content to internal subscribers:** `src/infra/outbound/deliver.ts:1095`
  - body: This builds the internal event from the original `payloadSummary`, but `applyMessageSendingHook` returns rewritten text in `hookResult.payloadSummary` when a plugin modifies outbound content. Internal subscribers would observe stale pre-rewrite content.
  - confidence: 0.96
- **[P2] Bridge the key that standalone hooks register:** `src/infra/outbound/deliver.ts:1090-1091`
  - body: The linked report registers hook-directory metadata for `message_sending`, but `createInternalHookEvent("message", "sending", ...)` only reaches `message` and `message:sending` listeners because directory hook metadata is registered verbatim. Add compatibility or document the new key before this can fix the reported standalone hook.
  - confidence: 0.93
