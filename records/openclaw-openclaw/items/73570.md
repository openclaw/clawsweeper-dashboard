## Summary

Keep open. Current main still lacks the requested pre-send internal hook bridge, and the linked bug remains coherent, but the PR patch would not yet solve it correctly because it emits for canceled sends, reports stale content after plugin rewrites, and dispatches `message:sending` while the reported standalone hook registers `message_sending`.

## What This Changes

The PR adds a fire-and-forget internal `message:sending` trigger in outbound delivery after plugin `message_sending` processing.

## Best Possible Solution

Amend this into a documented and tested internal pre-send hook surface: emit only for non-canceled sends, pass the effective post-rewrite content, decide whether to support `message_sending` as a compatibility alias or require `message:sending`, and update tests, docs, and the changelog accordingly.

## Reproduction Assessment

Yes. The linked issue gives concrete standalone-hook steps, and source inspection confirms current main runs only the plugin `message_sending` hook while internal hook dispatch only reaches `message` and `message:<action>` registrations.

## Solution Assessment

No. The PR direction is plausible, but this patch is not the best solution yet because it preserves the reported event-key mismatch and introduces canceled-send and stale-content regressions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Do not notify internal hooks for canceled sends:** `src/infra/outbound/deliver.ts:1032`
  - body: The new `fireAndForgetHook` block runs before the `hookResult.cancelled` guard. If a `message_sending` plugin cancels delivery, this still emits a pre-send internal event for a message that will not be sent, so the guard needs to run before the internal hook or be included in the condition.
  - confidence: 0.94
- **[P2] Pass rewritten hook content to internal subscribers:** `src/infra/outbound/deliver.ts:1041`
  - body: This builds the internal event from the original `payloadSummary`, but `applyMessageSendingHook` returns rewritten text in `hookResult.payloadSummary` when a plugin modifies outbound content. Internal subscribers would see stale pre-rewrite content; use the payload summary from `hookResult`.
  - confidence: 0.95
- **[P2] Make the bridged event match user hook registrations:** `src/infra/outbound/deliver.ts:1035-1036`
  - body: The linked report registers standalone hooks for `message_sending`, but `createInternalHookEvent("message", "sending", ...)` dispatches only `message` and `message:sending` because internal hook metadata is registered verbatim. Either add a compatibility path for `message_sending` or update the contract, docs, and tests so users register the supported key.
  - confidence: 0.88

  - body: This is a user-facing hook fix and the branch only changes delivery code. Repo policy requires user-facing fixes to add a single-line `CHANGELOG.md` entry in the active fixes section with appropriate contributor credit.
