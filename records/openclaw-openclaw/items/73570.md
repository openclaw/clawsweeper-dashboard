## Summary

Keep open: current main still lacks the requested hook-directory pre-send bridge, but this PR is not merge-ready because it emits the new event with the wrong contract, ordering, and content, and now double-classifies SDK subpaths.

## What This Changes

The PR adds an internal `message`/`sending` notification during outbound delivery, updates two outbound delivery tests, and adds Matrix/Mattermost to the public plugin-owned SDK entrypoint classification.

## Best Possible Solution

Define the internal pre-send hook key and notification semantics, then emit it only for payloads that will be delivered using effective post-plugin content, with docs/changelog and regression coverage.

## Reproduction Assessment

Yes, at source level. Current main has no internal pre-send bridge, and the latest PR diff still places the new dispatch before cancellation/deliverability checks while using the original payload summary and colon-style key.

## Solution Assessment

No. The PR direction is plausible, but this patch is not the best solution until the event contract is decided and the implementation uses the effective content, correct delivery gating, non-overlapping SDK classification, docs, tests, and changelog coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P2] Do not notify hooks for messages that will not send:** `src/infra/outbound/deliver.ts:1086`
  - body: The new internal hook block runs before the `hookResult.cancelled` guard and before the later `!effectivePayload` skip. A plugin `message_sending` hook can cancel or rewrite text to blank, so this would emit a pre-send notification for a message that is never delivered.
  - confidence: 0.96
- **[P2] Pass rewritten content to internal subscribers:** `src/infra/outbound/deliver.ts:1095`
  - body: This builds the internal event from the original `payloadSummary`, but `applyMessageSendingHook` returns rewritten text in `hookResult.payloadSummary` when a plugin modifies outbound content. Internal subscribers would see stale pre-rewrite content.
  - confidence: 0.96
- **[P2] Make the bridged event match hook registrations:** `src/infra/outbound/deliver.ts:1090-1091`
  - body: The linked report registers standalone hooks for `message_sending`, but `createInternalHookEvent("message", "sending", ...)` only dispatches `message` and `message:sending` because directory hook metadata is registered verbatim. This patch would still miss the reported hook key unless a compatibility path or documented contract change is added.
  - confidence: 0.92
- **[P2] Avoid double-classifying SDK subpaths:** `src/plugin-sdk/entrypoints.ts:30-31`
  - body: `matrix` and `mattermost` are already listed in `supportedBundledFacadeSdkEntrypoints` on current main. Adding them to `publicPluginOwnedSdkEntrypoints` makes the package-contract guard's `classificationOverlaps` nonempty and should be removed or replaced with the intended single classification.
  - confidence: 0.94
- **[P3] Add the required changelog entry:** `src/infra/outbound/deliver.ts:1086`
  - body: This is a user-facing hook behavior fix, but the PR only changes delivery source, tests, and SDK classification. Repo policy requires a single-line `CHANGELOG.md` entry for user-facing fixes.
  - confidence: 0.88
