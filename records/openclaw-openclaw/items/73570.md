## Summary

Keep open: current main still lacks an internal pre-send bridge, but this PR is not ready because the added notification can fire for canceled sends, reports stale content after rewrites, and still does not match the `message_sending` registration key from the linked issue.

## What This Changes

The PR adds a fire-and-forget internal `message`/`sending` hook during outbound delivery and updates outbound delivery tests to expect a pre-send internal hook call.

## Best Possible Solution

Define the standalone pre-send hook contract, then emit it only for non-canceled sends using the effective post-plugin-rewrite content, with regression tests, docs, and a changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main shows only the plugin `message_sending` path runs before delivery, while internal hook dispatch only reaches `message` and colon-style `message:<action>` registrations.

## Solution Assessment

No. The PR direction is plausible, but this patch is not the best solution yet because it preserves the event-key mismatch and introduces canceled-send and stale-content regressions.

## Review Findings

Full review comments:

- **[P2] Do not notify internal hooks for canceled sends:** `src/infra/outbound/deliver.ts:1086`
  - body: The new fire-and-forget block runs before the `hookResult.cancelled` guard. If a plugin `message_sending` hook cancels delivery, this still emits a pre-send internal event for a message that will not be sent; move the guard before this block or include `!hookResult.cancelled` in the condition.
  - confidence: 0.95
- **[P2] Pass rewritten hook content to internal subscribers:** `src/infra/outbound/deliver.ts:1095`
  - body: This builds the internal event from the original `payloadSummary`, but `applyMessageSendingHook` returns rewritten text in `hookResult.payloadSummary` when a plugin modifies outbound content. Internal subscribers would see stale pre-rewrite content; use the payload summary from `hookResult`.
  - confidence: 0.96

  - body: The linked report registers standalone hooks for `message_sending`, but `createInternalHookEvent("message", "sending", ...)` dispatches only `message` and `message:sending` handlers. Add a compatibility path for `message_sending` or update the contract, docs, and tests so users register the supported key.
  - confidence: 0.9
