## Summary

Keep open: the branch targets a real hook-dispatch gap, but it is not merge-ready because it emits before cancellation/empty-payload gates, reports stale content, does not reach the reported `message_sending` hook key, and lacks real behavior proof.

## What This Changes

The PR adds a fire-and-forget `message`/`sending` internal hook during outbound delivery and updates outbound delivery tests to expect the extra dispatch.

## Best Possible Solution

Define and document the pre-delivery internal hook contract, then emit it only for effective deliverable payloads with compatibility coverage for the intended standalone hook key.

## Reproduction Assessment

Yes, at source level. Current main has a plugin-only `message_sending` path and the PR head adds the internal event in a place that still exposes the cancellation, rewrite, and key-matching failures.

## Solution Assessment

No. Bridging shared outbound delivery is the right area, but this patch should first settle the event key and observe-versus-intercept semantics, then use the effective payload after cancellation and empty-payload handling.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.96

Full review comments:

- **[P2] Do not notify hooks for messages that will not send:** `src/infra/outbound/deliver.ts:1086`
  - body: The new internal hook block runs before `hookResult.cancelled` and before the later `!effectivePayload` skip. A plugin `message_sending` hook can cancel or blank the payload, so this can emit a pre-send notification for a message that never reaches the channel.
  - confidence: 0.96
- **[P2] Pass rewritten content to internal subscribers:** `src/infra/outbound/deliver.ts:1095`
  - body: This builds the internal event from the original `payloadSummary`, but `applyMessageSendingHook` returns rewritten text in `hookResult.payloadSummary` when a plugin modifies outbound content. Internal subscribers would observe stale pre-rewrite content.
  - confidence: 0.96
- **[P2] Bridge the key that standalone hooks register:** `src/infra/outbound/deliver.ts:1090-1091`
  - body: The linked report registers hook-directory metadata for `message_sending`, but `createInternalHookEvent("message", "sending", ...)` only reaches `message` and `message:sending` listeners because directory hook metadata is registered verbatim. Add compatibility or document the new key before this fixes the reported standalone hook.
  - confidence: 0.93
