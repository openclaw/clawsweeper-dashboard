## Summary

Keep open. Current main still lacks the requested pre-send internal hook bridge, but this PR is not merge-ready because the latest diff emits the new internal event for canceled sends, passes stale pre-rewrite content, and does not settle the `message_sending` versus `message:sending` registration contract.

## What This Changes

The PR adds a `message`/`sending` internal hook dispatch in shared outbound delivery and updates two outbound delivery tests to expect both pre-send and sent internal hook calls.

## Best Possible Solution

Define the public pre-send internal hook contract, then update shared outbound delivery, regression tests, docs, and changelog so only non-canceled sends emit the effective post-plugin-rewrite content under that contract.

## Reproduction Assessment

Yes. The linked issue gives standalone-hook steps, and current main source shows only the plugin `message_sending` path while internal dispatch reaches only `message` and `message:<action>` registrations.

## Solution Assessment

No. Bridging near `applyMessageSendingHook` is the right seam, but this patch is not the best solution until it handles cancellation, rewritten content, and the public event-key contract.

## Review Findings

Full review comments:

- **[P2] Do not notify internal hooks for canceled sends:** `src/infra/outbound/deliver.ts:1086`
  - body: The new fire-and-forget block runs before the `hookResult.cancelled` guard. If a plugin `message_sending` hook cancels delivery, this still emits a pre-send internal event for a message that will not be sent; move the guard before this block or include `!hookResult.cancelled` in the condition.
  - confidence: 0.95
- **[P2] Pass rewritten hook content to internal subscribers:** `src/infra/outbound/deliver.ts:1095`
  - body: This builds the internal event from the original `payloadSummary`, but `applyMessageSendingHook` returns rewritten text in `hookResult.payloadSummary` when a plugin modifies outbound content. Internal subscribers would see stale pre-rewrite content; use the payload summary from `hookResult`.
  - confidence: 0.96
- **[P2] Make the bridged event match user hook registrations:** `src/infra/outbound/deliver.ts:1090-1091`
  - body: The linked report registers standalone hooks for `message_sending`, but `createInternalHookEvent("message", "sending", ...)` dispatches only `message` and `message:sending` because internal hook metadata is registered verbatim. Either add a compatibility path for `message_sending` or update the contract, docs, and tests so users register the supported key.
  - confidence: 0.9
