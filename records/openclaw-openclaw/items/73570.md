## Summary

Keep open; the PR targets a real hook-dispatch gap, but the current diff still misses the linked standalone-hook registration path and would emit inaccurate pre-send notifications for canceled or rewritten sends.

## What This Changes

The PR adds a `message`/`sending` internal hook dispatch in outbound delivery and updates outbound delivery tests to expect the extra pre-send dispatch.

## Best Possible Solution

Define the supported standalone pre-send hook contract, then emit it only for non-canceled sends with effective post-plugin content and matching tests, docs, and changelog coverage.

## Reproduction Assessment

Yes. The linked #73542 report gives concrete standalone-hook steps, and source inspection confirms directory hooks register event keys verbatim while current outbound delivery only runs the plugin `message_sending` path.

## Solution Assessment

No. The bridge direction is plausible, but this patch is not the best solution until the event key contract is settled and the implementation respects cancellation and rewritten content.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Do not notify internal hooks for canceled sends:** `src/infra/outbound/deliver.ts:1086`
  - body: The new fire-and-forget block runs before the `hookResult.cancelled` guard. If a plugin `message_sending` hook cancels delivery, this still emits a pre-send internal event for a message that will not be sent; guard this block with `!hookResult.cancelled` or move it after the cancel check.
  - confidence: 0.95
- **[P2] Pass rewritten hook content to internal subscribers:** `src/infra/outbound/deliver.ts:1095`
  - body: This builds the internal event from the original `payloadSummary`, but `applyMessageSendingHook` returns rewritten text in `hookResult.payloadSummary` when a plugin modifies outbound content. Internal subscribers would see stale pre-rewrite content; use the payload summary from `hookResult`.
  - confidence: 0.96
- **[P2] Make the bridged event match user hook registrations:** `src/infra/outbound/deliver.ts:1090-1091`
  - body: The linked report registers standalone hooks for `message_sending`, but this emits `message`/`sending`, which `triggerInternalHook` only dispatches to `message` and `message:sending` handlers. Add a compatibility path for `message_sending` or update the public contract, docs, and tests so users register the supported key.
  - confidence: 0.9
- **[P3] Add the required changelog entry:** `src/infra/outbound/deliver.ts:1086`
  - body: This is a user-facing hook behavior change, but the PR only changes delivery source and tests. Add a single-line `CHANGELOG.md` entry in the active fixes section with appropriate human contributor credit.
  - confidence: 0.88
