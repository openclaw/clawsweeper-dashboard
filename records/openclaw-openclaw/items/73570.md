## Summary

Keep open. Current main still lacks a standalone internal pre-send message hook, and this PR is useful source work, but the proposed bridge still misses the reported `message_sending` registration path and would emit misleading events for canceled or rewritten sends.

## What This Changes

The PR adds a fire-and-forget internal `message:sending` trigger in outbound delivery and updates two outbound delivery tests to expect it alongside `message:sent`.

## Best Possible Solution

Define and document the supported internal pre-send message hook contract, then emit it only for non-canceled sends with effective post-rewrite content and focused regression coverage.

## Reproduction Assessment

Yes, by source inspection and the linked issue's steps. Current main has a plugin `message_sending` path but no internal pre-send bridge, and internal hooks registered as `message_sending` will not receive `message:sending` dispatches.

## Solution Assessment

No. The direction is plausible, but this patch is not the best solution because it keeps the event-key mismatch and introduces canceled-send and stale-content notification regressions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Do not notify internal hooks for canceled sends:** `src/infra/outbound/deliver.ts:1086`
  - body: The new internal hook block runs before the `hookResult.cancelled` guard. If a plugin `message_sending` hook cancels delivery, this still emits a pre-send notification for a message that will not be sent, so the guard needs to run before the internal hook or be included in the condition.
  - confidence: 0.94
- **[P2] Pass rewritten hook content to internal subscribers:** `src/infra/outbound/deliver.ts:1095`
  - body: This builds the internal event from the original `payloadSummary`, but `applyMessageSendingHook` returns rewritten text in `hookResult.payloadSummary` when a plugin modifies outbound content. Internal subscribers would see stale pre-rewrite content; use the payload summary from `hookResult`.
  - confidence: 0.95
- **[P2] Make the bridged event match user hook registrations:** `src/infra/outbound/deliver.ts:1090-1091`
  - body: The linked report registers standalone hooks for `message_sending`, but `createInternalHookEvent("message", "sending", ...)` dispatches only `message` and `message:sending` because internal hook metadata is registered verbatim. Either add a compatibility path for `message_sending` or update the contract, docs, and tests so users register the supported key.
  - confidence: 0.88
- **[P3] Add the required changelog entry:** `src/infra/outbound/deliver.ts:1086`
  - body: This is a user-facing hook behavior change and the branch only changes delivery code and tests. Repo policy requires user-facing fixes to add a single-line `CHANGELOG.md` entry in the active fixes section with appropriate contributor credit.
  - confidence: 0.9
