## What This Changes

The branch adds a fire-and-forget internal `message:sending` dispatch in outbound delivery after plugin `message_sending` hooks and rewrites a large section of `src/infra/outbound/deliver.test.ts`.

## Best Possible Solution

Define the standalone pre-send hook contract, add compatibility for the reported key or document the colon key, emit only for effective post-rewrite deliverable payloads, restore unrelated test coverage, and prove it with a redacted hook-directory run.

## Reproduction Assessment

Yes, at source level. Current main runs plugin `message_sending` before delivery, but standalone hook events are registered verbatim and internal dispatch only matches type or `type:action` keys.

## Solution Assessment

No. Shared outbound delivery is the right area, but this patch needs a hook-key contract decision, corrected emit timing, rewritten payload content, and a narrower test diff before it is a maintainable fix.
