## Summary

Keep open: the PR targets a valid hook gap, but the latest diff is not merge-ready because the new internal dispatch fires before cancellation/empty-payload gating, uses stale pre-plugin content, and does not reach standalone hooks registered as `message_sending`.

## What This Changes

The PR adds a fire-and-forget internal `message`/`sending` notification in outbound delivery and updates outbound-delivery tests to expect it.

## Best Possible Solution

Define and document the canonical pre-send internal hook contract, then emit it only for payloads that will actually be delivered using effective post-plugin content, with regression tests and a changelog entry.

## Reproduction Assessment

Yes, at source level. Current main has no internal pre-send bridge, and the latest PR diff inserts a bridge that still runs before delivery gating, uses the original payload summary, and dispatches only the colon-style internal key.

## Solution Assessment

No. Bridging near `applyMessageSendingHook` is the right area, but this patch is not the best solution until the event key and observe-only semantics are decided and the implementation uses the effective deliverable payload.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.96

Full review comments:

- **[P2] Do not notify hooks for messages that will not send:** `src/infra/outbound/deliver.ts:1086`
  - body: The new internal hook block runs before `hookResult.cancelled` and before the later `!effectivePayload` skip. A plugin `message_sending` hook can cancel or rewrite the text to blank, so this emits a pre-send notification for a message that never leaves the system.
  - confidence: 0.96
- **[P2] Pass rewritten content to internal subscribers:** `src/infra/outbound/deliver.ts:1095`
  - body: This builds the internal event from the original `payloadSummary`, but `applyMessageSendingHook` returns rewritten text in `hookResult.payloadSummary` when a plugin modifies outbound content. Internal subscribers would observe stale pre-rewrite content.
  - confidence: 0.96
- **[P2] Bridge the key that standalone hooks register:** `src/infra/outbound/deliver.ts:1090-1091`
  - body: The linked report registers hook-directory metadata for `message_sending`, but `createInternalHookEvent("message", "sending", ...)` only reaches `message` and `message:sending` listeners because directory hook metadata is registered verbatim. This does not fix the reported standalone hook key unless compatibility or docs are added.
  - confidence: 0.93
- **[P3] Add the required changelog entry:** `src/infra/outbound/deliver.ts:1086`
  - body: This is a user-facing hook behavior change, but the PR updates only delivery code and tests. Repo policy requires a single-line `CHANGELOG.md` entry for user-facing fixes.
  - confidence: 0.88
