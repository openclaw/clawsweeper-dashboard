## Summary

Keep open: current main still lacks a pre-send internal hook bridge, but this PR is not merge-ready because the added dispatch misses the linked standalone `message_sending` registration key and would emit incorrect notifications for canceled or rewritten sends.

## What This Changes

This PR adds a fire-and-forget internal `message`/`sending` hook dispatch in outbound delivery and updates outbound delivery tests to expect the additional internal hook call.

## Best Possible Solution

Define the public internal pre-send hook contract, then emit it only for non-canceled sends with the effective post-plugin content and matching regression tests, docs, and changelog coverage.

## Reproduction Assessment

Yes. The linked issue's directory-hook setup is source-reproducible: metadata events are registered verbatim, `triggerInternalHook` dispatches only `message` and `message:sending`, and current outbound delivery never emits an internal pre-send event.

## Solution Assessment

No. The PR direction is plausible, but this patch is not the best solution yet because it leaves the event-key mismatch unresolved and introduces canceled-send and stale-content regressions.
