## Summary

Keep open: the branch targets the right outbound hook area, but latest head still does not satisfy the linked standalone-hook bug because it dispatches the colon key while the reported hook registers the underscore key, emits before suppression decisions, and sends stale pre-rewrite content; contributor real behavior proof is also still missing.

## What This Changes

The branch adds a fire-and-forget internal `message:sending` dispatch in `deliverOutboundPayloadsCore` and rewrites outbound delivery tests to expect both pre-send and post-send internal hook events.

## Best Possible Solution

Define the standalone pre-send hook contract, support the reported key or document an intentional colon-key migration, emit only for effective post-plugin deliverable content, restore unrelated test coverage, and prove it with a redacted standalone hook-directory run.

## Reproduction Assessment

Yes, at source level. Current main only runs the plugin `message_sending` path before outbound delivery, while standalone hook metadata is registered verbatim and internal dispatch only matches `message` or colon-form `message:*` keys.

## Solution Assessment

No. Shared outbound delivery is the right implementation area, but this patch needs the hook-key contract resolved plus corrected cancellation, suppressed-payload, rewritten-content, and proof coverage before it is the best fix.
