## Summary

Keep open. The PR targets a real current-main gap, but the current patch remains unsafe because it fires before the cancel guard, sends stale content after plugin rewrites, and still misses hooks registered with the underscore event named by the linked report.

## What This Changes

The PR adds an internal `message`/`sending` notification during outbound delivery and updates two outbound delivery tests to expect both pre-send and sent internal hook dispatches.

## Best Possible Solution

Define and test the intended internal pre-send hook contract, then emit it only for non-canceled sends with the effective post-plugin-rewrite content and matching docs/changelog coverage.

## Reproduction Assessment

Yes. The linked issue gives concrete standalone-hook steps, and source inspection confirms current `main` only runs the plugin `message_sending` path while internal dispatch only reaches `message` and `message:<action>` registrations.

## Solution Assessment

No. The PR direction is plausible, but this patch is not the best solution yet because it preserves the event-key mismatch and introduces canceled-send and stale-content regressions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Do not notify internal hooks for canceled sends:** `src/infra/outbound/deliver.ts:1086`
  - body: The new fire-and-forget block runs before the `hookResult.cancelled` guard. If a plugin `message_sending` hook cancels delivery, this still emits a pre-send internal event for a message that will not be sent; move the guard before this block or include `!hookResult.cancelled` in the condition.
  - confidence: 0.95
- **[P2] Pass rewritten hook content to internal subscribers:** `src/infra/outbound/deliver.ts:1095`
  - body: This builds the internal event from the original `payloadSummary`, but `applyMessageSendingHook` returns rewritten text in `hookResult.payloadSummary` when a plugin modifies outbound content. Internal subscribers would see stale pre-rewrite content; use the payload summary from `hookResult`.
  - confidence: 0.96
- **[P2] Make the bridged event match user hook registrations:** `src/infra/outbound/deliver.ts:1090-1091`
  - body: The linked report registers standalone hooks for `message_sending`, but `createInternalHookEvent("message", "sending", ...)` dispatches only `message` and `message:sending` because internal hook metadata is registered verbatim. Either add a compatibility path for `message_sending` or update the contract, docs, and tests so users register the supported key.
  - confidence: 0.9
- **[P3] Add the required changelog entry:** `src/infra/outbound/deliver.ts:1086`
  - body: This is a user-facing hook fix, but the PR only changes delivery source and tests. Repo policy requires user-facing fixes to add a single-line `CHANGELOG.md` entry in the active fixes section with appropriate contributor credit.
  - confidence: 0.88
