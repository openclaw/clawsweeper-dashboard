## Summary

Keep open. The linked bug still appears valid on current main, but this PR should not merge as-is because the new internal hook fires for canceled sends, reports stale content after plugin rewrites, and does not satisfy the `message_sending` standalone-hook registration named in the report.

## What This Changes

The PR adds a fire-and-forget internal pre-send message hook trigger in outbound delivery after plugin `message_sending` processing.

## Best Possible Solution

Define a documented and tested pre-send internal hook surface that emits only for non-canceled sends, carries the effective post-rewrite content, and explicitly resolves whether `message_sending` is a compatibility alias or unsupported for standalone hooks.

## Reproduction Assessment

Yes. Current main source shows the plugin-only `message_sending` path, and standalone hooks register metadata events verbatim while internal dispatch reaches only `message` and `message:<action>` listeners.

## Solution Assessment

No. Bridging this lifecycle event may be the right direction, but this patch is not the best implementation until it handles cancellation, rewritten content, event-key compatibility, docs, tests, and changelog together.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Do not notify internal hooks for canceled sends:** `src/infra/outbound/deliver.ts:1032`
  - body: The new `fireAndForgetHook` block runs before the `hookResult.cancelled` guard. If a plugin `message_sending` hook cancels delivery, this still emits a pre-send internal event for a message that will not be sent; move it after the guard or include `!hookResult.cancelled` in the condition.
  - confidence: 0.94
- **[P2] Pass rewritten hook content to internal subscribers:** `src/infra/outbound/deliver.ts:1041`
  - body: This builds the internal event from the original `payloadSummary`, but `applyMessageSendingHook` returns rewritten text in `hookResult.payloadSummary` when a plugin modifies outbound content. Internal subscribers would see stale pre-rewrite content; use the payload summary from `hookResult`.
  - confidence: 0.95
- **[P2] Make the bridged event match standalone registrations:** `src/infra/outbound/deliver.ts:1035-1036`
  - body: The linked report registers standalone hooks for `message_sending`, but `createInternalHookEvent("message", "sending", ...)` dispatches only `message` and `message:sending`. Add a compatibility path for `message_sending` or update the contract, docs, and tests so users register the supported key.
  - confidence: 0.88

  - body: This is a user-facing hook fix and the branch only changes delivery code. Repo policy requires a single-line `CHANGELOG.md` entry in the active fixes section with appropriate contributor credit.
