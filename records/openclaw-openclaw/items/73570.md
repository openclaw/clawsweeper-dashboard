## Summary

Keep open. Current main still lacks an internal pre-send bridge, and this PR is useful source work, but the patch is not mergeable because it can notify hooks for cancelled sends, sends stale pre-rewrite content, and does not settle the public internal hook key that the linked report relies on.

## What This Changes

The PR adds a fire-and-forget internal `message`/`sending` hook dispatch in outbound delivery and updates outbound delivery tests to expect the extra internal hook call.

## Best Possible Solution

Define the internal pre-send hook contract, then emit it only for non-cancelled effective sends with post-plugin-rewrite content, plus docs, changelog, and regression coverage.

## Reproduction Assessment

Yes. The linked issue gives concrete standalone-hook steps, and source inspection confirms current main only runs the plugin `message_sending` path while internal dispatch currently covers `message:sent`, not pre-send.

## Solution Assessment

No. The PR direction is plausible, but this patch is not the best solution yet because it preserves an event-key ambiguity and introduces cancelled-send and stale-content regressions.

  - body: The new fire-and-forget block runs before the `hookResult.cancelled` guard. If a plugin `message_sending` hook cancels delivery, this still emits a pre-send internal event for a message that will not be sent; move the guard before this block or include `!hookResult.cancelled` in the condition.
  - confidence: 0.95
- **[P2] Pass rewritten hook content to internal subscribers:** `src/infra/outbound/deliver.ts:1095`
  - body: This builds the internal event from the original `payloadSummary`, but `applyMessageSendingHook` returns rewritten text in `hookResult.payloadSummary` when a plugin modifies outbound content. Internal subscribers would see stale pre-rewrite content; use the payload summary from the hook result or the later effective payload, depending on the chosen contract.
  - confidence: 0.96

  - body: `createInternalHookEvent("message", "sending", ...)` dispatches only `message` and `message:sending`, while the linked report registers standalone hooks for `message_sending` and the docs do not define a pre-send internal key. Add a compatibility path or update the documented contract and tests so users register the supported key.
  - confidence: 0.9
