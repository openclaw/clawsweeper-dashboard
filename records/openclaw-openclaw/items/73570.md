## Summary

Keep open: current main lacks the requested pre-send internal hook bridge, but this PR still introduces correctness blockers around cancellation, rewritten content, event-key compatibility, and SDK entrypoint classification.

## What This Changes

The PR adds an internal `message`/`sending` dispatch during outbound delivery, updates outbound delivery tests for the extra hook call, and classifies Matrix/Mattermost SDK subpaths as public plugin-owned entrypoints.

## Best Possible Solution

Define and document the pre-send internal hook key and observe-only semantics, then emit it only for deliverable payloads using effective post-plugin content with regression tests and a changelog entry.

## Reproduction Assessment

Yes, at source level. Current main has no internal pre-send bridge, and the latest PR patch shows the remaining failure paths without needing a live gateway run.

## Solution Assessment

No. The PR direction is plausible, but the patch is not the best fix until it gates delivery correctly, uses rewritten content, matches the registered hook key, avoids SDK classification overlap, and documents the chosen contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P2] Do not notify hooks for messages that will not send:** `src/infra/outbound/deliver.ts:1086`
  - body: The new internal hook dispatch runs before the `hookResult.cancelled` guard and before the later `!effectivePayload` skip. A `message_sending` plugin hook can cancel or blank the payload, so this would notify internal subscribers about a send that never happens.
  - confidence: 0.96
- **[P2] Pass rewritten content to internal subscribers:** `src/infra/outbound/deliver.ts:1095`
  - body: This builds the internal event from the original `payloadSummary`, but `applyMessageSendingHook` returns rewritten content in `hookResult.payloadSummary`. Internal subscribers would see stale pre-rewrite content whenever a plugin modifies the outbound message.
  - confidence: 0.96
- **[P2] Make the bridged event match hook registrations:** `src/infra/outbound/deliver.ts:1090-1091`
  - body: The linked report registers standalone hooks for `message_sending`, while `createInternalHookEvent("message", "sending", ...)` only dispatches `message` and `message:sending`. Add the intended compatibility path or document and test the colon-style contract explicitly.
  - confidence: 0.92
- **[P2] Avoid double-classifying SDK subpaths:** `src/plugin-sdk/entrypoints.ts:30-31`
  - body: `matrix` and `mattermost` are already listed in `supportedBundledFacadeSdkEntrypoints` on current main. Adding them to `publicPluginOwnedSdkEntrypoints` makes the package-contract guard's `classificationOverlaps` nonempty.
  - confidence: 0.94
- **[P3] Add the required changelog entry:** `src/infra/outbound/deliver.ts:1086`
  - body: This is a user-facing hook behavior fix, but the PR does not update `CHANGELOG.md`. Repo policy requires a single-line changelog entry for user-facing fixes.
  - confidence: 0.88
