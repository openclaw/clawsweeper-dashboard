## Summary

Keep open. Current main still lacks the internal pre-send hook bridge this PR is trying to add, so the linked bug is real, but the patch has blocking correctness problems around cancelled sends, rewritten content, and the standalone-hook event key contract before it can merge.

## What This Changes

The PR adds an internal `message:sending` hook trigger in `src/infra/outbound/deliver.ts` after outbound `applyMessageSendingHook` processing.

## Best Possible Solution

Amend this into a documented and tested internal pre-send hook surface: emit only after the plugin hook result is known to be non-cancelled, pass the effective post-rewrite hook content, settle the `message:sending` versus `message_sending` compatibility contract, and add focused regression tests, docs, and a changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: current main runs the plugin `message_sending` hook without an internal `message:sending` trigger, and the PR patch can be checked directly for the cancelled-send and stale-content regressions.

## Solution Assessment

No. The direction is plausible, but this patch is not the best solution yet because it emits before the cancellation guard, ignores `hookResult.payloadSummary`, and does not resolve the standalone hook event-key mismatch from the linked report.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Do not notify internal hooks for cancelled sends:** `src/infra/outbound/deliver.ts:1032`
  - body: The new `fireAndForgetHook` block runs before `if (hookResult.cancelled) { continue; }`. If a `message_sending` plugin cancels delivery, this still emits an internal pre-send event for a message that will not be sent, so the guard needs to run before the internal hook or be included in the condition.
  - confidence: 0.94
- **[P2] Pass rewritten hook content to internal subscribers:** `src/infra/outbound/deliver.ts:1041`
  - body: This builds the internal event from the original `payloadSummary`, but `applyMessageSendingHook` returns rewritten text in `hookResult.payloadSummary` when a plugin modifies outbound content. Internal subscribers would see stale pre-rewrite content; use the payload summary from `hookResult`.
  - confidence: 0.95
- **[P2] Make the bridged event match user hook registrations:** `src/infra/outbound/deliver.ts:1035-1036`
  - body: The linked report registers standalone hooks for `message_sending`, but `createInternalHookEvent("message", "sending", ...)` dispatches only `message` and `message:sending` because internal hook metadata is registered verbatim. Either add a compatibility path for `message_sending` or update the contract, docs, and tests so users register the supported key.
  - confidence: 0.88
- **[P3] Add the required changelog entry:** `src/infra/outbound/deliver.ts:1032`
  - body: This is a user-facing hook fix and the branch only changes delivery code. Repo policy requires user-facing fixes to add a single-line `CHANGELOG.md` entry in the active fixes section with appropriate contributor credit.
  - confidence: 0.9
