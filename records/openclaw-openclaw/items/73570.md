## Summary

Keep open: the linked bug is source-reproducible and this PR targets the right outbound path, but the current diff would still miss `message_sending` hook-directory listeners and can emit stale or spurious pre-send events.

## What This Changes

The PR adds a fire-and-forget internal `message`/`sending` dispatch after plugin `message_sending` hooks in outbound delivery and updates delivery tests to expect the extra internal hook call.

## Best Possible Solution

Define and document the pre-delivery standalone hook contract, then emit it only for deliverable effective payloads with tests covering key compatibility, cancellation, rewrites, and blank payloads.

## Reproduction Assessment

Yes, at source level. Current main runs plugin `message_sending` before delivery but only internal `message:sent` is bridged, while hook-directory events are registered verbatim and dispatched only by type or colon key.

## Solution Assessment

No. Shared outbound delivery is the right area, but this patch should settle the event key and observe-only semantics, then emit using the effective post-plugin payload only for messages that will be delivered.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.96

Full review comments:

- **[P2] Do not notify hooks for messages that will not send:** `src/infra/outbound/deliver.ts:1086`
  - body: The new internal hook block runs before `hookResult.cancelled` and before the later `!effectivePayload` skip. A plugin `message_sending` hook can cancel or blank the payload, so this can emit a pre-send notification for a message that never reaches the channel.
  - confidence: 0.96
- **[P2] Pass rewritten content to internal subscribers:** `src/infra/outbound/deliver.ts:1095`
  - body: This builds the internal event from the original `payloadSummary`, but `applyMessageSendingHook` returns rewritten text in `hookResult.payloadSummary` when a plugin modifies outbound content. Internal subscribers would observe stale pre-rewrite content.
  - confidence: 0.96
- **[P2] Bridge the key that standalone hooks register:** `src/infra/outbound/deliver.ts:1090-1091`
  - body: The linked report registers hook-directory metadata for `message_sending`, but `createInternalHookEvent("message", "sending", ...)` only reaches `message` and `message:sending` listeners because directory hook metadata is registered verbatim. Add compatibility or document the new key before this fixes the reported standalone hook.
  - confidence: 0.93
