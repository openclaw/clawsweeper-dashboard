## Summary

Keep open. Current main still lacks an internal pre-send hook bridge, so this PR targets a real gap, but the patch is not mergeable because it would miss the reporter's `message_sending` registration key and emit incorrect events for canceled or rewritten sends.

## What This Changes

The PR adds a fire-and-forget internal `message:sending` trigger in outbound delivery after plugin `message_sending` processing.

## Best Possible Solution

Define and test a documented internal pre-send hook surface that fires only for non-canceled sends, carries the effective post-rewrite content, and explicitly supports or rejects `message_sending` compatibility.

## Reproduction Assessment

Yes. The linked report gives concrete standalone-hook steps, and current source inspection confirms main only invokes plugin `message_sending` while internal dispatch reaches `message` and `message:<action>` registrations.

## Solution Assessment

No. The direction is plausible, but this patch is not the best solution yet because it preserves the registration-key mismatch and introduces canceled-send and stale-content regressions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Dispatch the user-facing hook key that reporters register:** `src/infra/outbound/deliver.ts:1035-1036`
  - body: The linked report registers standalone hooks for `message_sending`, but this creates a `message`/`sending` internal event. `triggerInternalHook` only dispatches `message` and `message:sending`, so those reported hooks would still not fire unless the patch adds a compatibility path or updates the contract and tests.
  - confidence: 0.88
- **[P2] Do not notify internal hooks for canceled sends:** `src/infra/outbound/deliver.ts:1032`
  - body: The new `fireAndForgetHook` block runs before the `hookResult.cancelled` guard. If a `message_sending` plugin cancels delivery, this still emits a pre-send internal event for a message that will not be sent, so the guard needs to run before the internal hook or be included in the condition.
  - confidence: 0.94
- **[P2] Pass rewritten hook content to internal subscribers:** `src/infra/outbound/deliver.ts:1041`
  - body: This builds the internal event from the original `payloadSummary`, but `applyMessageSendingHook` returns rewritten text in `hookResult.payloadSummary` when a plugin modifies outbound content. Internal subscribers would see stale pre-rewrite content; use the payload summary from `hookResult`.
  - confidence: 0.95

  - body: This is a user-facing hook fix and the branch only changes delivery code. Repo policy requires user-facing fixes to add a single-line `CHANGELOG.md` entry in the active fixes section with appropriate contributor credit.
