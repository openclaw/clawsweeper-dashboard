## Summary

Keep open: this PR targets a real hook-directory gap, but the latest diff is not merge-ready because it can notify before cancellation/effective-payload gates, sends stale pre-rewrite content, and emits only the colon-style internal key while the linked report registers `message_sending`.

## What This Changes

This PR adds a fire-and-forget internal `message`/`sending` notification after plugin `message_sending` hooks in outbound delivery and updates outbound-delivery tests to expect both pre-send and post-send internal hook calls.

## Best Possible Solution

Define and document the canonical pre-send internal hook contract, then emit it only for effective deliverable payloads with compatibility for existing hook-directory event names, regression coverage, and a changelog entry.

## Reproduction Assessment

Yes, at source level. Current main has no internal pre-send bridge, and this PR adds one in a way that still runs before delivery gates, uses the original payload summary, and misses the `message_sending` hook-directory key from the linked reproduction.

## Solution Assessment

No. Bridging near the outbound `message_sending` hook is the right area, but the implementation should use effective post-plugin content, avoid unsent payload notifications, and settle the internal event key compatibility before merge.
