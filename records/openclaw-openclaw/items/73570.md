## Summary

Keep open: the outbound delivery hook path is the right area, but this PR is not merge-ready because it does not reach the reported standalone `message_sending` hook key, can emit false or stale pre-send notifications, removes broad unrelated test coverage, and still lacks real behavior proof.

## What This Changes

The PR adds a fire-and-forget internal `message:sending` dispatch in outbound delivery and rewrites broad outbound delivery tests to expect both pre-send and post-send internal hook events.

## Best Possible Solution

Define the standalone pre-send hook contract, add compatibility for the reported underscore key or explicitly document the colon key, emit only for effective post-plugin deliverable content, restore unrelated test coverage, and prove it with a redacted hook-directory run.

## Reproduction Assessment

Yes, at source level. Current main only runs plugin `message_sending` in outbound delivery, while standalone hook metadata is registered verbatim and internal dispatch matches only `message` or `message:sending` keys.

## Solution Assessment

No. Shared outbound delivery is the right implementation area, but this patch needs the hook-key contract resolved, corrected cancellation and rewritten-content semantics, restored focused coverage, and real hook-directory proof before it is the best fix.
