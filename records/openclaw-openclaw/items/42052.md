## Summary

Keep open: the requested backup output-path behavior is still source-reproducible on current main, but this PR edits a resolver that is no longer on the active command path and it lacks required real behavior proof.

## What This Changes

The PR changes backup output-path resolution so missing extensionless `--output` paths are classified as directories while suffixed paths remain explicit archive filenames.

## Best Possible Solution

Move the missing-path heuristic into `src/infra/backup-create.ts`, preserve explicit suffixed filenames, add focused tests, and require redacted real backup proof before merge.

## Reproduction Assessment

Yes, by source inspection on current main: pass a missing extensionless output path to the active backup resolver and it returns the raw path after `fs.stat()` fails, which is then used as `archivePath`. I did not run a filesystem backup because this review is read-only.

## Solution Assessment

No. The heuristic is plausible, but the PR applies it to stale command-local code instead of the active infra resolver and still needs focused tests plus real behavior proof.
