Keep open. The PR addresses a plausible backup output-path bug, but it changes the old command-local resolver while current main uses the infra backup resolver, so the patch would not fix the active `backup create` path and still needs real behavior proof.

## What This Changes

The PR changes `src/commands/backup.ts` so a missing extensionless `--output` path is treated as a destination directory while suffixed paths remain explicit archive filenames.

## Best Possible Solution

Move the missing-path directory/file heuristic into `src/infra/backup-create.ts`, preserve explicit suffixed filenames, add focused regression coverage, and require redacted real backup proof before merge.
