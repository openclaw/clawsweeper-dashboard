## Summary

Keep open: the underlying backup bug is still present on current main, but this PR applies the fix to a stale command-level resolver instead of the active infra resolver.

## What This Changes

The PR changes backup output-path resolution so missing extensionless output paths are treated as destination directories while existing paths or paths with suffixes remain explicit archive filenames.

## Best Possible Solution

Move the output-path heuristic into the active infra backup resolver, preserve explicit suffix filenames, rethrow non-ENOENT stat errors, and cover it with focused regression tests plus a changelog entry.

## Reproduction Assessment

Yes, by source inspection. Calling `backupCreateCommand` or `createBackupArchive` with `output` set to a missing extensionless path reaches `resolveOutputPath`, which catches the missing `fs.stat` result and publishes the archive to that path without appending the generated `.tar.gz` basename.

## Solution Assessment

No. The intended policy is reasonable, but the patch changes `src/commands/backup.ts` while current main resolves backup output paths in `src/infra/backup-create.ts`, so this hunk would not fix the active code path after rebase.
