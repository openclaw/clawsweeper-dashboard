## Summary

Keep open. The missing extensionless output-directory behavior is still present on current main, but this PR changes the old command-local resolver instead of the active backup resolver and still needs real backup-run proof before merge.

## What This Changes

The PR adds ENOENT handling to the backup output resolver in `src/commands/backup.ts` so missing extensionless paths are treated as directories while suffixed paths remain explicit filenames.

## Best Possible Solution

Move the missing-extensionless-output heuristic into `src/infra/backup-create.ts`, preserve explicit suffix filenames and non-ENOENT stat errors, add focused regression coverage, and collect redacted real backup-run proof.

## Reproduction Assessment

Yes, by source inspection. Current main delegates `backup create` to `createBackupArchive`, whose active resolver returns a missing extensionless `--output` path directly instead of appending the generated `.tar.gz` basename.

## Solution Assessment

No. The heuristic is reasonable, but this branch applies it to the stale command-level resolver instead of the active infra resolver used by current main.
