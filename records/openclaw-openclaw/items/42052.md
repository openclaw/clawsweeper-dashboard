## Summary

Keep open. The reported bug is still source-reproducible on current main, but this PR edits a resolver that has since moved out of `src/commands/backup.ts`; it also lacks required after-fix real behavior proof, so it should not merge as-is.

## What This Changes

The PR changes `src/commands/backup.ts` so missing extensionless `backup create --output` paths are treated as destination directories while suffixed paths remain explicit archive filenames.

## Best Possible Solution

Move the missing-path directory/file heuristic into `src/infra/backup-create.ts`, preserve explicit suffixed filenames, add focused regression coverage, and require redacted real backup proof before merge.

## Reproduction Assessment

Yes. Source inspection shows `backupCreateCommand()` calls `createBackupArchive()`, whose active resolver returns a missing extensionless output path directly and then publishes to that exact path; I did not run a filesystem backup because this review is read-only.

## Solution Assessment

No. The intended heuristic is plausible, but this PR changes a stale command-local resolver; the maintainable fix belongs in `src/infra/backup-create.ts` with focused tests and real behavior proof.
