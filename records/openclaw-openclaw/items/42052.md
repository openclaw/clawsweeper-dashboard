## Summary

Keep open. The reported backup output bug still exists on current main, and the PR branch is no longer pointed at the active implementation path: current backup create delegates to the infra resolver, while the PR edits the old command-level resolver surface.

## What This Changes

The branch changes backup output resolution so missing extensionless output paths are treated as destination directories while existing files and paths with suffixes are treated as explicit archive filenames.

## Best Possible Solution

Move the heuristic into the active `src/infra/backup-create.ts` resolver, keep explicit suffixes as filenames, rethrow non-`ENOENT` stat errors, and add focused regression coverage around missing extensionless outputs and explicit filename suffixes.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: call `backupCreateCommand` or `createBackupArchive` with a mocked backup plan and `output` set to a missing extensionless path; current main resolves `archivePath` to that path instead of appending the generated `.tar.gz` basename inside it.

## Solution Assessment

No. The PR's policy direction is reasonable, but the branch edits a stale command-level resolver; the maintainable fix belongs in the active infra resolver with tests for the current write path.
