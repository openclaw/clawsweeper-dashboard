## What This Changes

The branch changes backup output-path resolution so a missing extensionless output is treated as a destination directory while existing paths or suffixed paths remain explicit archive filenames.

## Best Possible Solution

Move the heuristic into src/infra/backup-create.ts, preserve explicit suffix filenames and non-ENOENT stat errors, and add focused regression coverage for missing extensionless output paths.

## Reproduction Assessment

Yes, by source inspection. Current main's backupCreateCommand reaches src/infra/backup-create.ts, where a missing extensionless --output path is returned directly and then used as the final archivePath instead of receiving the generated .tar.gz basename.

## Solution Assessment

No. The intended policy is reasonable, but this branch changes a stale resolver in src/commands/backup.ts instead of the active infra resolver used on current main.
