## Summary

Keep open: current main and v2026.6.8 still have the heartbeat task clock-skew behavior, and this PR is a focused fix with no blocking code findings. It is not merge-ready yet because the external contributor proof is unit-test-only and the branch/checks need refresh against current main.

## What This Changes

The PR changes `isTaskDue` so future `lastRunMs` values are treated as due and adds focused unit tests for heartbeat task due calculation.

## Best Possible Solution

Keep the helper-level clock-skew fix, confirm future task timestamps are not a supported deferral contract, add real heartbeat proof, and refresh validation against current main before merge.

## Reproduction Assessment

Yes, source-reproducible: current main computes `nowMs - lastRunMs >= intervalMs`, so a future `lastRunMs` makes the delta negative and leaves the task not due. I did not run tests because this review is read-only.

## Solution Assessment

Yes, this is the best fix layer: both task selection and task timestamp advancement use `isTaskDue`, while a runner-only check would duplicate the invariant.
