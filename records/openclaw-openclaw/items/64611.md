## What This Changes

This PR changes `isTaskDue` to treat future last-run timestamps as due and adds unit tests for never-run, elapsed, not-elapsed, and clock-skew cases.

## Best Possible Solution

Merge the narrow helper guard with focused `isTaskDue` regression coverage, add a single-line Unreleased Fixes changelog entry with contributor credit, and validate the changed heartbeat surface before landing.

## Reproduction Assessment

Yes. On current main, calling `isTaskDue(now + 5 * 60 * 1000, "30m", now)` returns false because the elapsed delta is negative; the runner uses that helper to filter due heartbeat tasks.

## Solution Assessment

Yes. Updating `isTaskDue` is the narrowest maintainable fix because both due-task filtering and timestamp updates already go through that helper; the safer merge path is to keep this patch, add the missing changelog entry, and rerun validation.
