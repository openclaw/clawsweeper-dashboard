## What This Changes

The PR changes `isTaskDue` to treat future `lastRunMs` values as due and adds focused unit coverage for never-run, elapsed, not-yet-elapsed, and future-timestamp cases.

## Best Possible Solution

Land the focused helper guard and regression coverage after the branch is refreshed and real behavior proof confirms future heartbeat task timestamps recover without disturbing normal intervals.

## Reproduction Assessment

Yes, source-reproducible: current main computes `nowMs - lastRunMs >= intervalMs`, so `lastRunMs > nowMs` returns false and the runner can skip with no due tasks. I did not run tests because this review was read-only.

## Solution Assessment

Yes, this is the best fix location: task selection and timestamp advancement both use the shared `isTaskDue` helper, so a runner-only guard would duplicate the invariant.
