## Summary

Keep open. Current main still has the heartbeat task clock-skew bug, and the PR is a narrow functional fix with focused tests; the remaining concrete cleanup is release-note coverage plus normal validation before merge.

## What This Changes

This PR treats future heartbeat task `lastRunMs` values as due in `isTaskDue` and adds unit coverage for never-run, elapsed, not-elapsed, and clock-skew cases.

## Best Possible Solution

Merge or cherry-pick the future-timestamp guard and focused tests with a single Unreleased/Fixes changelog entry, then validate the heartbeat test and changed gate.

## Reproduction Assessment

Yes. On current main, calling `isTaskDue(now + 5 * 60 * 1000, "30m", now)` reproduces the bug because the helper subtracts a future timestamp and treats the negative elapsed value as not due.

## Solution Assessment

Yes. Updating `isTaskDue` is the narrowest maintainable fix because both due-task filtering and post-run timestamp updates already flow through that helper; the safer merge path is to add changelog coverage and validate.
