## What This Changes

This PR changes `isTaskDue` to treat future `lastRunMs` values as due and adds unit coverage for normal and clock-skew due cases.

## Best Possible Solution

Land the helper-level clock-skew guard with focused `isTaskDue` coverage after the branch is refreshed, current checks pass, and redacted real heartbeat behavior proof is added.

## Reproduction Assessment

Yes for source-level reproduction: current main computes `nowMs - lastRunMs`, so a future `lastRunMs` makes the helper return false and can lead task-mode heartbeat to skip as `no-tasks-due`. I did not run a live NTP or VM-resume scenario in this read-only review.

## Solution Assessment

Yes, the helper-level fix is the best narrow shape because both due-task filtering and timestamp updates share `isTaskDue`. A runner-local clamp would duplicate the timing invariant and leave the helper contract misleading.
