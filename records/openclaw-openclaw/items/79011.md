## Summary

Keep open: current main does not accept the requested `agents.defaults.models.*.contextTokens` shape, and the runtime still treats `agents.defaults.contextTokens` as a global cap over model-level context metadata. This is a valid feature/config-design request, not a conservative cleanup close.

## What This Changes

Request per-model `contextTokens` overrides under `agents.defaults.models`, falling back to the global agent default and then provider model metadata.

## Best Possible Solution

Decide the public config contract first, then either add a documented `agents.defaults.models.*.contextTokens` precedence path with schema/types/tests/docs or clarify the provider-model workaround and global-cap semantics.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection does confirm current main’s behavior: the requested key is absent from schema/types and the existing global cap is applied over model metadata.

## Solution Assessment

Unclear: the exact proposed `agents.defaults.models` shape may be reasonable, but current OpenClaw already keeps per-model runtime caps in `models.providers`, so maintainers should choose the config surface and precedence before implementation.
