## Summary

Keep open: current main does not implement the requested `agents.defaults.models.<model>.contextTokens` field, and the already-supported per-model cap lives under `models.providers.*.models[]`, so adding a second agent-level config surface needs maintainer product judgment.

## What This Changes

The issue requests a per-model `contextTokens` override under `agents.defaults.models` that would override the global `agents.defaults.contextTokens` for matching model refs.

## Best Possible Solution

Choose one canonical config owner for per-model runtime context caps, then align schema, docs, resolver precedence, and regression tests around that single path.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms the requested `agents.defaults.models` field is absent while the existing per-model context cap is supported through provider catalog config.

## Solution Assessment

Unclear. Adding this exact field is plausible, but current docs and code already use `models.providers.*.models[].contextTokens`, so maintainers should decide whether to duplicate that surface or steer users to the existing path.
