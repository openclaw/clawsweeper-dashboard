## Summary

Keep open: current main and v2026.6.8 still do not accept `agents.defaults.models.<model>.contextTokens`, and the existing per-model cap lives under the provider catalog, so this remains a product/config-contract decision rather than implemented behavior.

## What This Changes

The issue requests per-model `contextTokens` overrides under `agents.defaults.models` so users can set different context caps for different active models without changing the global agent cap.

## Best Possible Solution

Choose one canonical owner for per-model runtime context caps, then align schema, docs, resolver precedence, and tests around that single contract.
