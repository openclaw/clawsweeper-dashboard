## Summary

Keep open: current main and the latest release still do not support `agents.defaults.models.*.contextTokens`; the schema only allows allowlist metadata such as `alias`, `params`, `agentRuntime`, and `streaming` there, while runtime still treats `agents.defaults.contextTokens` as a global cap. This remains a public config and precedence decision, not conservative cleanup.

## What This Changes

Request to allow model-specific `contextTokens` under `agents.defaults.models` so different models can have different effective context caps without manually patching the global default.

## Best Possible Solution

Choose one canonical context-budget contract: either add schema, runtime, docs, and tests for `agents.defaults.models.*.contextTokens` with explicit precedence, or document provider/model `contextTokens` as the supported path and close this request by decision.

Not applicable as a feature request. Source inspection is high-confidence that the requested config key is absent and the current global cap behavior remains defined by `Math.min(agentContextTokens, modelContextTokens)`.

## Solution Assessment
