## Summary

Close as superseded: the exact `agents.defaults.models.<model>.contextTokens` field is still absent, but the remaining model-aware context-budget problem is now tracked by the broader open canonical issue with newer runtime evidence, while current code and docs already support provider-catalog per-model `contextTokens`.

## What This Changes

Request to add a per-model `contextTokens` override under `agents.defaults.models` so switching between models with different context windows does not require changing the global agent context cap.

## Best Possible Solution

Track the model-aware context-budget policy in https://github.com/openclaw/openclaw/issues/88404 and keep `models.providers.*.models[].contextTokens` as the documented override unless maintainers choose a new canonical agent-level config surface.

## Solution Assessment

No for the exact standalone request. Adding another `agents.defaults.models` field would create a second context-budget config owner, so the better path is to resolve the model-aware precedence and default policy in the canonical issue.
