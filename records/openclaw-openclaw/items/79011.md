## Summary

Keep open: current main still does not accept `agents.defaults.models.*.contextTokens`, and the remaining work is a public config/precedence design choice rather than conservative cleanup.

## What This Changes

Request support for per-model `contextTokens` overrides under `agents.defaults.models` so users do not have to change the global context cap when switching models.

## Best Possible Solution

Decide the config contract first, then either add a documented `agents.defaults.models.*.contextTokens` precedence path with schema/types/tests/docs or clarify that per-model runtime budgets belong only under `models.providers.*.models`.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection does confirm the requested field is absent from current schema/types and the existing runtime uses the global cap/provider-model metadata path.

## Solution Assessment

Unclear: the proposed shape is plausible, but OpenClaw already has per-provider-model `contextTokens`, so maintainers should choose the config surface and precedence before implementation.
