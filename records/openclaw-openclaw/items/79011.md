## Summary

Keep open: current main still does not accept `agents.defaults.models.*.contextTokens`, and runtime resolution still treats `agents.defaults.contextTokens` as a global cap. The related open passthrough PR covers part of the config-shape problem, but it does not settle the requested per-model-over-global precedence.

## What This Changes

The issue requests a per-model `contextTokens` override under `agents.defaults.models` so different selected models can use different context caps.

## Best Possible Solution

Decide the public config contract first, then either add documented `agents.defaults.models.*.contextTokens` precedence with schema/types/tests/docs or explicitly direct users to the existing `models.providers.*.models.*.contextTokens` contract.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection does confirm that the requested key is absent and the existing global cap still applies over model-level context metadata.

## Solution Assessment

Unclear: the requested shape is plausible, but it overlaps existing provider-model caps and https://github.com/openclaw/openclaw/pull/44475. Maintainers should choose the config surface and precedence before implementation.
