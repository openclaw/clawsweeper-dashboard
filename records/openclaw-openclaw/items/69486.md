## Summary

Keep this PR open. Current main still has the branch-and-reappend rewrite path without on-disk abandoned-entry cleanup, so the change is not obsolete; however, the proposed cleanup is not merge-ready because it only preserves `parentId` ancestry and can delete entries still referenced by surviving metadata such as label targets.

## What This Changes

The PR adds post-rewrite cleanup in `rewriteTranscriptEntriesInSessionFile` that snapshots active branch IDs before and after a rewrite, removes abandoned JSONL rows after a `parentId` ancestry filter, and adds leaf-branch regression tests plus a production-derived fixture.

## Best Possible Solution

Revise the cleanup into a small reference-graph based reconciler at the transcript rewrite boundary: remove only IDs abandoned by the specific rewrite after preserving every surviving core entry reference edge, add regression coverage for labels, compaction keep markers, sibling branches, repeated rewrites, and the production-derived lineage, and keep compaction-chain reader/reset work in #68765 or another narrow follow-up.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a session where a surviving label entry targets a message on the rewritten active path: that target can be in `branchIdsBefore \ branchIdsAfter`, is not reached through the label entry's `parentId` chain, and is therefore deleted while the label survives.

## Solution Assessment

No. The PR’s active-branch diff is the right narrow boundary, but the `parentId`-only integrity filter is not the safest maintainable solution because OpenClaw session entries have non-parent ID references that also need preservation or dependent cleanup.

## Review Findings
