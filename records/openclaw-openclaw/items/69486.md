## Summary

Keep open. The underlying transcript duplication problem still exists on current main and this PR targets the right rewrite boundary, but the proposed cleanup can delete entries that surviving session metadata still references through non-parent IDs, so it needs revision before merge.

## What This Changes

The PR adds post-rewrite cleanup for abandoned transcript JSONL entries after active-branch rewrites, plus leaf-branch and production-derived regression coverage.

## Best Possible Solution

Revise the cleanup into a small reference-aware reconciler at the transcript rewrite boundary: remove only IDs abandoned by the specific rewrite after preserving or consistently removing every surviving session reference edge, add regression coverage for label targets and sibling branches, and keep compaction-chain reader/reset work in #68765 or another focused follow-up.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a session file where a surviving label entry targets a message on the rewritten active path: that target can be in branchIdsBefore minus branchIdsAfter, is not reached through the label entry's parentId chain, and is deleted while the label row survives.

## Solution Assessment

No. The active-branch diff is the right narrow boundary, but parentId-only integrity filtering is not sufficient for OpenClaw session entries because labels and other metadata can hold load-bearing non-parent ID references.
