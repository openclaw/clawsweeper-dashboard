## Summary

Keep open: the current-main delta bug is source-reproducible and the PR is aimed at the right area, but the latest head still needs real behavior proof and introduces blocking regressions around reasoning sanitization and heartbeat reasoning delivery.

## What This Changes

The PR removes core reasoning formatting before delta calculation, carries raw reasoning payloads through Embedded Pi, and adjusts Telegram, Feishu, Discord, message-tool, and auto-reply tests around raw reasoning streams.

## Best Possible Solution

Keep raw text for internal delta math, but preserve or intentionally migrate every outward reasoning contract across heartbeat delivery, message-tool sanitization, channel previews, docs, changelog, and the broader thinking-events work.

## Reproduction Assessment

Yes. Source inspection on current main shows formatted reasoning is used for prefix-based delta calculation, and `formatReasoningMessage` changes the compared string by adding the `Reasoning:` wrapper and underscores.

## Solution Assessment

No. The raw-core delta direction is maintainable, but this PR leaves existing formatted-wrapper consumers behind and should be reconciled with the broader thinking-events contract before merge.
