## Summary

Keep open. Current main and the latest release still use repeated-signal and diversity gates for deep memory promotion, and the test suite explicitly preserves the behavior where a high-scoring one-off recall does not reach `MEMORY.md`; the KinthAI comment adds useful production context for why this remains a real memory-quality gap.

## What This Changes

Request a memory-core promotion path for critically important one-off events that cannot satisfy repeated-recall deep dreaming thresholds.

## Best Possible Solution

Add a guarded high-priority single-event promotion lane in memory-core that preserves default gates while allowing trusted directives, classifier output, or review-approved candidates with tests and docs.

## Reproduction Assessment

Yes. Current source and tests provide a high-confidence reproduction path: one high-scoring recall run through default dreaming promotion leaves `MEMORY.md` empty; no test was run because this review was read-only.

## Solution Assessment

Unclear. The requested direction is valid, but the maintainable fix should be a bounded priority/review lane rather than lowering the global recall and diversity thresholds.
