## Summary

Keep open. Current main still has the reported bundled memory-core behavior: automatic deep promotion requires repeated signal and query/day diversity, and the repo has a regression test asserting that a one-off high-scoring recall does not reach MEMORY.md under default thresholds. The later KinthAI comment reinforces this as a real memory-quality gap rather than an obsolete support report.

## What This Changes

The issue requests memory-core promotion support for critically important single-occurrence events that currently fail recall and diversity thresholds.

## Best Possible Solution

Define and test a durable high-priority single-event lane in memory-core, with narrow bypass semantics for recall and diversity gates and pollution guardrails. Good candidates are an explicit remember directive, a manual priority/review flag, or a critical-feedback classifier/session-end sweep, followed by updates to memory-core tests and the Dreaming docs.

## Reproduction Assessment

Yes. The current code path and the existing dreaming test provide a high-confidence reproduction: record one high-scoring short-term recall, run default dreaming promotion, and MEMORY.md remains empty because the recall and diversity gates do not pass.

## Solution Assessment

Unclear. The requested direction is valid, but the issue lists several materially different designs; the safest fix should add an explicit high-priority lane rather than globally relaxing the current thresholds.
