## Summary

Keep open. Current main and the latest release still enforce repeated-signal and diversity gates, and current tests explicitly keep one-off recalls out of `MEMORY.md`; this is a valid memory-core feature/product gap rather than an already-fixed bug.

## What This Changes

The issue requests a memory-core path for important single-occurrence memories to bypass the default repeated-recall promotion gates.

## Best Possible Solution

Add a guarded high-priority single-event promotion lane in memory-core that preserves existing default thresholds while allowing trusted directives, classifier output, or review-approved candidates to promote with clear tests and docs.

## Solution Assessment

Unclear. The requested direction is valid, but the best fix should be a bounded high-priority lane with review and memory-pollution guardrails rather than relaxing the global promotion thresholds.
