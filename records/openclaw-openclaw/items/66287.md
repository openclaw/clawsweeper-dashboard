## Summary

Keep open. Current main and v2026.5.18 still require repeated signal and diversity gates before Deep Dreaming writes to `MEMORY.md`; the KinthAI comment adds useful production context for one-shot memories, while the related memory issues are adjacent rather than a canonical replacement for this product gap.

## What This Changes

Allow memory-core Dreaming to preserve critically important single-occurrence events without depending only on repeated recall and diversity signals.

## Best Possible Solution

Add a guarded high-priority single-event promotion or review lane in memory-core that preserves default frequency/diversity gates while allowing trusted directives, classifier output, or review-approved candidates with tests and docs.

## Reproduction Assessment

Yes, at source level. Current defaults, promotion gates, docs, latest-release inspection, and an explicit test show that one high-scoring recall under default thresholds leaves `MEMORY.md` empty; I did not run tests because this review is read-only.

## Solution Assessment

Unclear. The requested direction is valid, but the maintainable fix should be a bounded priority or review lane rather than lowering global recall and diversity thresholds.
