## Summary

Keep open. Current main and v2026.5.12 still require repeated signal and diversity gates before deep dreaming writes to `MEMORY.md`, and the current tests explicitly preserve the one-off recall behavior; the KinthAI follow-up adds useful production context, but the related issues are adjacent rather than superseding this product gap.

## What This Changes

Requests a memory-core path for critically important one-off events to bypass repeated-recall promotion gates and reach long-term memory.

## Best Possible Solution

Add a guarded high-priority single-event promotion or review lane in memory-core that preserves default frequency/diversity gates while allowing trusted directives, classifier output, or review-approved candidates with tests and docs.

## Reproduction Assessment

Yes, at source level. Current defaults, promotion gates, docs, and an explicit regression test show that one high-scoring recall under default thresholds leaves `MEMORY.md` empty; I did not run tests because this was a read-only review.

## Solution Assessment

Unclear. The requested direction is valid, but the maintainable fix should be a bounded priority or review lane rather than lowering the global recall and diversity thresholds.
