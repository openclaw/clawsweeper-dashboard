## Summary

Keep open. Current main still has the reported memory-core behavior: default deep promotion requires repeated signal and query/day diversity, and an existing regression test asserts that a one-off high-scoring recall does not reach MEMORY.md. The KinthAI comment reinforces this as a real memory-quality gap; current grounded backfill and CLI threshold overrides are manual/recovery paths, not an unattended singleton-promotion lane.

## What This Changes

The issue requests a way for important single-occurrence memory events to be promoted to long-term memory instead of being blocked by repeated-signal promotion thresholds.

## Best Possible Solution

Define and test a durable high-priority single-event lane in memory-core, with explicit guardrails for directives, review flags, classifiers, or session-end sweeps that can bypass recall/diversity gates only for trusted candidates.

## Reproduction Assessment

Yes. A high-confidence reproduction already exists in the test suite: record one high-scoring short-term recall, run default dreaming promotion, and MEMORY.md remains empty because the recall and diversity gates do not pass.

## Solution Assessment

Unclear. The issue describes a real gap, but the best fix should be a bounded high-priority lane with review/pollution guardrails rather than relaxing the global thresholds for all candidates.
