  - body: Current main and public docs support `openclaw plugins init <id> --name "Display Name"`, but this patch removes `--name` and redefines the positional argument as the display name.   - confidence: 0.95

  - body: The provider scaffold only calls `api.registerProvider(...)`. Current provider docs and `defineSingleProviderPluginEntry` use `registerModelCatalogProvider` for list/help/picker catalog surfaces, so the generated provider can build and validate while still being absent from model browsing surfaces.
  - confidence: 0.87
- **[P1] Avoid unpinned lifecycle installs in PR validation:** `scripts/validate-plugin-init-provider-scaffold.ts:64`
  - body: This script is run by a new pull_request workflow and executes `npm install` inside the generated package. That package uses `clawhub: "latest"`, `openclaw: "latest"`, and semver-ranged dev dependencies without a lockfile or `--ignore-scripts`, so CI will execute registry lifecycle scripts and resolve moving versions outside the repo lockfile.
  - confidence: 0.86
