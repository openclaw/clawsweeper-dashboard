## Summary

Keep open: the PR is directionally right and has terminal proof, but current head still stages local attachments after only string-root filtering plus final-path lstat, leaving a symlink-parent escape before managed-media copying.

## What This Changes

The PR stages local native iMessage inbound attachments into managed inbound media, converts HEIC/HEIF to JPEG with sips, preserves remote raw-path handling, adjusts target-prefix behavior, and adds tests plus a changelog entry.

## Best Possible Solution

Keep the plugin-owned staging and HEIC conversion approach, but require realpath revalidation against the effective local attachment roots before any read, conversion, or managed-media save.

## Reproduction Assessment

Yes, source reproducible: current main forwards allowed native iMessage attachments as raw original_path values while media tools intentionally do not widen local roots from those sources. I did not run a live macOS Gateway in this read-only review.

## Solution Assessment

No, not yet. The owner-boundary staging direction is maintainable, but the patch should canonicalize and re-check local attachment paths against the effective iMessage roots before copying them into managed media.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:
