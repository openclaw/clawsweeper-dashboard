## Summary

Keep this PR open: it addresses a real native iMessage media gap and includes credible terminal proof, but the patch stages local attachments before iMessage access, pairing, mention, echo, and rate-limit decisions have accepted the message.

## What This Changes

The PR adds local native iMessage attachment staging with HEIC/HEIF-to-JPEG conversion, wires staged media into inbound dispatch, adds tests and changelog coverage, and removes a stale Codex test helper.

## Best Possible Solution

Keep the staging/conversion approach, but move local media staging until after the inbound message is accepted for dispatch while preserving pre-decision attachment placeholders and the remote-host raw-path contract.

## Reproduction Assessment

Yes for the reviewed defect: source inspection shows the PR calls staging before `resolveIMessageInboundDecision`, whose current-main path can still return pairing or drop. The original HEIC issue is also source-reproducible because current main forwards raw iMessage attachment paths and MIME types into inbound media context.

## Solution Assessment

No, not as-is. The media staging design is a narrow fit for the bug, but it must run only for messages that survive iMessage access and routing checks, with regression coverage for pairing or blocked senders.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Move staging after inbound access checks:** `extensions/imessage/src/monitor/monitor-provider.ts:264-268`
  - body: The new local staging call runs before `resolveIMessageInboundDecision`, so a sender who is still in default pairing mode, blocked by allowlists, or later dropped for group policy/mention/echo can still make OpenClaw read, convert, and save their attachment into managed media. Please keep the placeholder calculation from the validated attachment metadata, but defer `stageIMessageAttachments` until after the decision is `dispatch` and the rate-limit guard has passed.
  - confidence: 0.92

## Security Review

Status: needs_attention

Summary: The patch introduces a security-sensitive ordering regression by reading/converting/saving local iMessage media before sender access checks accept the message.

Concerns:

- **[medium] Pre-authorization media staging:** `extensions/imessage/src/monitor/monitor-provider.ts:264`
  - body: Local attachments are staged after root filtering but before iMessage pairing, allowlist, group, mention, echo, and rate-limit decisions. That lets dropped or unpaired senders trigger local disk reads/conversions and managed-media writes that should only happen for accepted inbound messages.
