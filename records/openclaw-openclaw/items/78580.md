## Summary

Keep this PR open. It fixes a real iMessage media bug and has sufficient real behavior proof, but the current patch still has two merge-blocking issues: HEIC conversion can hang the monitor path, and the attachment fix still carries an unrelated core routing contract change.

## What This Changes

The PR stages native iMessage inbound attachments into managed inbound media, converts HEIC/HEIF files to JPEG before dispatch, preserves remote-host raw paths, and adds iMessage monitor tests plus a changelog entry.

## Best Possible Solution

Land the plugin-owned attachment staging and HEIC-to-JPEG path after bounding conversion execution and keeping service-prefix grammar inside the iMessage plugin rather than core routing.

## Reproduction Assessment

Yes. Current main is source-reproducible: it forwards valid native iMessage original_path values into the agent media context while media tools only trust default/workspace roots, and the PR discussion adds real v2026.5.12-beta.3 HEIC evidence.

## Solution Assessment

No, not as-is. Staging attachments in the iMessage plugin is the right boundary, but the converter needs bounded process execution and the unrelated core target-prefix behavior change should be removed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Bound the HEIC conversion process:** `extensions/imessage/src/monitor/media-staging.ts:79-91`
  - body: `execFileAsync("sips", ...)` is awaited with no timeout while staging inbound media. Node's `execFile` default timeout is 0, so a stuck conversion can hold the native iMessage monitor before dispatch; use the plugin SDK process helper or explicit timeout/output limits.
  - confidence: 0.88
- **[P2] Keep service prefixes out of core routing:** `src/infra/outbound/channel-target-prefix.ts:6-23`
  - body: Adding iMessage service/chat prefixes to the generic target-kind block makes a plugin-advertised `imessage:` target prefix stop selecting that provider, reversing the existing cron delivery contract. Keep this attachment fix scoped to plugin staging or plugin-owned target parsing.
  - confidence: 0.8
