Keep open: the bug and plugin-owned staging direction are well supported, and the supplied real behavior proof is sufficient, but the latest head still awaits `sips` without a timeout before dispatch. A stalled HEIC conversion can block the native iMessage monitor path.

## Best Possible Solution

## Solution Assessment

No, not as-is. Staging inside the iMessage plugin is the right boundary, but the converter needs explicit timeout/output bounds before this is the narrow maintainable fix.
