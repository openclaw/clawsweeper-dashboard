## Summary

Close as superseded: the same unsupported xAI Grok 4.20 multi-agent endpoint was already handled in a closed canonical discussion, and current main documents the supported Grok 4.20 variants plus the multi-agent limitation.

## What This Changes

Report that selecting `xai/grok-4.20-multi-agent-0309` in a Telegram forum topic with xAI API-key auth causes assistant turns to fail with `400 status code (no body)` and no visible reply.

## Best Possible Solution

Use the supported Grok 4.20 reasoning/non-reasoning or Grok 4.3 models on the bundled provider, and route any native xAI multi-agent endpoint integration through ClawHub/plugin work unless a concrete missing plugin API is identified.

## Reproduction Assessment

No live reproduction was run. The issue logs and the closed canonical xAI multi-agent discussion establish the same unsupported endpoint failure mode, while current source/docs show OpenClaw intentionally keeps multi-agent ids out of the normal bundled xAI path.

## Solution Assessment

Yes, closing this as superseded is the best cleanup path because the core product decision already exists and current docs point users to supported Grok variants. Native multi-agent support would be a separate plugin/ClawHub integration, not a duplicate core issue.
