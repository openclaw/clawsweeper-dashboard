## Best Possible Solution

Reject unsupported xAI multi-agent ids before session override or runner dispatch with a clear chat-facing/provider error, while keeping native Grok multi-agent endpoint support as a separate ClawHub/provider-feature decision.

## Reproduction Assessment

## Solution Assessment

Yes for the narrow validation path: rejecting unsupported xAI multi-agent ids before dispatch is the maintainable fix for this bug. Native Grok multi-agent support would require a separate provider/plugin design because the upstream API is not standard Chat Completions.
