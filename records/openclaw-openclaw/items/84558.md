## Summary

Keep open: the Telegram formatter capability is useful and the visual proof is adequate, but the patch still makes a Telegram-specific expandable blockquote style part of the public MarkdownIR/SDK boundary and needs maintainer direction before merge.

## What This Changes

Adds an opt-in Telegram formatter option that promotes the trailing Markdown blockquote to `<blockquote expandable>`, updates formatter/chunking tests, and extends shared MarkdownIR style ordering.

## Best Possible Solution

Keep expandable final-blockquote promotion in the Telegram-owned formatter/chunker as an opt-in behavior, or deliberately promote it to a documented SDK style with contract tests and coordinated raw HTML sanitizer support.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this adds a new opt-in formatter/API behavior. Source inspection confirms current main has normal Telegram blockquote rendering but no `expandFinalBlockquote` option, and the PR screenshots show the proposed Telegram client behavior.

## Solution Assessment

No, not as written: the formatter capability is useful, but the best boundary is not to add a Telegram-only entity to public MarkdownIR without an explicit SDK contract. A safer path is Telegram-local metadata or a documented, tested public IR expansion coordinated with the sanitizer PR.
