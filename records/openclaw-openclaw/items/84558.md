## Summary

Keep this PR open: the visible Telegram proof is strong and current main does not have this final-blockquote formatter option, but the patch still exports a Telegram-specific style through the shared MarkdownIR/SDK surface without a clear maintainer decision or contract follow-through.

## What This Changes

Adds an `expandFinalBlockquote` Telegram formatter option that renders trailing blockquote spans as `<blockquote expandable>`, adds formatter/chunking tests, and extends the shared MarkdownIR style union.

## Best Possible Solution

Keep expandable final-blockquote promotion in the Telegram-owned formatter path, or intentionally add a documented generic extension seam and coordinate it with the raw HTML sanitizer PR.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this adds a new formatter option/API behavior. Source inspection confirms current main has normal Telegram blockquote rendering but no `expandFinalBlockquote` option, and the PR screenshots show the requested Telegram client behavior.

## Solution Assessment

No, not as proposed. The Telegram behavior is plausible, but exporting a Telegram-specific style through shared MarkdownIR is not the narrowest maintainable solution unless maintainers explicitly accept and document that API expansion.
