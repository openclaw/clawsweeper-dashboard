The PR adds an opt-in Telegram formatter option that renders the trailing Markdown blockquote as `<blockquote expandable>`, updates Telegram formatter chunk tests, and extends shared MarkdownIR/render style handling.

## Best Possible Solution

Keep expandable final-blockquote promotion inside the Telegram formatter/chunker boundary, or deliberately promote it to a documented shared MarkdownIR style with contract coverage and coordination with the raw-HTML sanitizer path.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this PR adds a new opt-in Telegram formatter capability. Source inspection confirms current main renders normal blockquotes but has no `expandFinalBlockquote` option, and the PR body provides after-fix Telegram screenshots.

## Solution Assessment

No: the formatter behavior is reasonable, but the current patch is not the best boundary because it exports a Telegram-specific style through shared MarkdownIR without a deliberate public contract.
