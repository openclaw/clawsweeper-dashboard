## What This Changes

The PR adds an opt-in `expandFinalBlockquote` Telegram formatter option, renders a promoted trailing blockquote as `<blockquote expandable>`, and adds formatter/chunking tests while widening shared MarkdownIR style types.

## Best Possible Solution

Keep expandable blockquote metadata local to the Telegram formatter/chunker, or deliberately promote it as public MarkdownIR with SDK docs and contract tests, while coordinating raw HTML sanitizer support with https://github.com/openclaw/openclaw/pull/84082.

## Reproduction Assessment

Not applicable as a strict bug repro because this PR adds a new opt-in formatter capability. Source inspection confirms current main has no `expandFinalBlockquote` option, and the merge blocker is visible in the PR diff widening `MarkdownStyle`.

## Solution Assessment

No as-is; the generated Telegram formatter behavior is a reasonable surface, but a Telegram-only marker should not enter shared MarkdownIR without explicit SDK contract work. The safer path is a Telegram-local promotion plus tests, with raw HTML sanitizer support handled deliberately alongside the related open PR.
