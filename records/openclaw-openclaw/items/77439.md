## Summary

Keep open: current main still uses a hardcoded fail-closed gateway config write allowlist, and the requested operator-configurable extension would add a new security-sensitive config policy surface.

## What This Changes

Add an opt-in operator-configurable allowlist or preset policy for agent-driven gateway config.apply/config.patch writes while keeping sensitive paths protected.

## Best Possible Solution

Design a first-class typed gateway config-write policy with fail-closed defaults, explicit protected paths that always win, schema/help/docs, and focused guard tests before implementation.
