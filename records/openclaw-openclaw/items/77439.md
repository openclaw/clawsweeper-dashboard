## Summary

Keep open: current main still lacks this capability, and adding it would create a new security-sensitive gateway config policy rather than a narrow cleanup or bug fix.

## What This Changes

Request adds an opt-in operator-configurable allowlist for agent-driven gateway `config.patch`/`config.apply` writes beyond the current hardcoded safe paths.

## Best Possible Solution

Design a first-class, typed gateway config-write policy with fail-closed defaults, explicit always-protected paths, schema help, docs, and focused guard tests.

## Reproduction Assessment

Not applicable; this is an enhancement request, not a bug report. Source inspection of current main confirms the gateway tool still uses a fixed allowlist and no `configWriteAllow` or trusted-agent policy key was found.

Unclear; a configurable allowlist is plausible, but the exact config shape, presets, wildcard rules, and protected denylist need maintainer security/product review before code is written.
