## Summary

Keep open: current main still uses a hardcoded fail-closed gateway config write allowlist, and the requested operator-configurable extension would be a new security-sensitive config surface rather than an already-implemented cleanup.

## What This Changes

The issue requests an opt-in operator config surface to extend trusted-agent `config.patch`/`config.apply` permissions beyond the hardcoded default allowlist.

## Best Possible Solution

Design a first-class, typed gateway config-write policy with fail-closed defaults, protected paths that always win, schema/docs support, and focused guard tests before implementation.
