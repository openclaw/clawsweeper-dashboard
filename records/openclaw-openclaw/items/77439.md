## Summary

Keep open: current main still uses a hardcoded fail-closed gateway config write allowlist, and the requested trusted-agent extension would add a new security-sensitive config/policy surface rather than closeable cleanup.

## What This Changes

Request adds an operator-configurable allowlist or preset policy so trusted agents can extend `gateway config.patch` and `config.apply` write permissions while protected paths remain denied.

## Best Possible Solution

Design a typed gateway config-write policy with fail-closed defaults, explicit operator-owned allowlists, protected denylist precedence, schema/docs help, and focused guard coverage before implementation.
