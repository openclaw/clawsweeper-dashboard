## Summary

Keep open: current main still uses a hardcoded fail-closed gateway config write allowlist, and the requested operator-configurable extension would be a new security-sensitive config surface rather than an already-implemented cleanup.

## What This Changes

Add an operator-configurable allowlist for agent-driven `gateway config.patch` and `gateway config.apply` writes, with safe presets and protected paths.

## Best Possible Solution

Design a first-class, typed gateway config-write policy with fail-closed defaults, explicit protected paths that always win, docs, schema help, and focused guard tests before implementing the operator allowlist.
