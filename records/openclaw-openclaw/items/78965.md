## Summary

Keep open: current main supports Docker, SSH, OpenShell, and a plugin sandbox-backend seam, but no local-user backend; the protected security/maintainer labels and host privilege-switching questions require maintainer design review rather than cleanup.

## What This Changes

The issue requests a local-user sandbox backend that runs each agent's tools under a configured Unix user account.

## Best Possible Solution

Keep this as a maintainer-owned sandbox architecture/security design item, likely starting with a plugin-first prototype unless owners decide the core config/docs/doctor surface should grow a first-class backend.

## Reproduction Assessment

Not applicable: this is a feature/security design request, not a broken existing behavior. Source and docs confirm current main has Docker, SSH, and OpenShell backends but no local-user backend to reproduce.

## Solution Assessment

Unclear: the requested direction is plausible, and current SDK seams make a plugin-backed prototype possible, but deciding `sudo`/`runuser`/helper semantics and trust boundaries is a maintainer security/product decision.
