## Summary

Keep open. Current main still does not enforce the configured Feishu webhook path before processing webhook requests, and PR #54904 remains the open implementation candidate that explicitly closes this issue.

## What This Changes

The issue asks Feishu webhook mode to enforce `channels.feishu.webhookPath` so signed requests on other paths are rejected before dispatch.

## Best Possible Solution

Review and land PR #54904, or an equivalent Feishu plugin patch, that normalizes the configured path, rejects mismatches before body parsing/signature validation/dispatch, preserves query-string acceptance, and adds focused regression coverage.

## Reproduction Assessment

Yes. The source-level path is high-confidence: configure Feishu webhook mode with a custom `webhookPath`, send a correctly signed POST to another path on the same listener, and current `monitorWebhook` has no path comparison before dispatch.

## Solution Assessment

Yes. The narrow maintainable fix is Feishu-owned path enforcement at the webhook ingress boundary before body parsing and signature work; a broader SDK or product change is not required for this bug.
