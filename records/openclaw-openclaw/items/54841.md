## Summary

Keep open. Current main and v2026.5.18 still do not enforce the configured Feishu webhookPath before processing signed webhook payloads, and https://github.com/openclaw/openclaw/pull/54904 is open with closing syntax for the targeted fix.

## What This Changes

Feishu webhook mode should reject requests whose path does not match the configured webhookPath before parsing or dispatching signed payloads.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/54904, or an equivalent Feishu plugin patch, with early path normalization/rejection before body parsing, signature validation, challenge handling, and dispatch.

## Reproduction Assessment

Yes. Source inspection on current main shows monitorWebhook never compares req.url or URL.pathname to webhookPath before body parsing, signature validation, challenge handling, and dispatch.

## Solution Assessment

Yes. The narrow maintainable fix is Feishu-owned early path normalization and mismatch rejection before the existing webhook pipeline; the open linked PR is the right implementation path to review.
