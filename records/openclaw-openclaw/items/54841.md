## What This Changes

The issue requests Feishu webhook-mode route enforcement so only requests to the configured webhookPath are accepted.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/54904, or an equivalent Feishu plugin patch, that rejects mismatched paths before body parsing/signature validation/dispatch and adds focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows monitorWebhook never compares req.url or URL.pathname to webhookPath before body parsing, signature validation, challenge handling, and dispatch.

## Solution Assessment

Yes. The narrow maintainable solution is Feishu-owned early path normalization and mismatch rejection before the existing webhook pipeline; the linked open PR is the current candidate to review.
