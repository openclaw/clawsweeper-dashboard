## Summary

Keep open. Current main still resolves the configured Feishu webhookPath for logging/status/rate-limit identity but does not compare the incoming request URL/pathname before body parsing, signature validation, challenge handling, or dispatcher invocation. PR #54904 is open and unmerged with closing syntax for this bug, so this issue should remain open until that PR or an equivalent Feishu plugin fix lands.

## What This Changes

The issue requests Feishu webhook mode to reject requests whose URL path does not match the configured channels.feishu.webhookPath before parsing, signature validation, or dispatch.

## Best Possible Solution

Review and land PR #54904, or an equivalent Feishu-owned patch, that normalizes the configured webhookPath, compares the incoming URL pathname before body parsing/signature validation/dispatch, preserves query-string acceptance on the configured path, and adds focused regression coverage.
