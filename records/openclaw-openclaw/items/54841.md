## Summary

Keep open. Current main and the latest release still do not enforce the configured Feishu webhook path, and an open unmerged implementation PR already tracks the exact fix with closing syntax.

## What This Changes

The issue reports that Feishu webhook mode accepts signed requests on arbitrary paths instead of enforcing the configured webhookPath.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/54904, or an equivalent Feishu plugin patch, that rejects mismatched request paths before body parsing, signature validation, challenge handling, and dispatch while preserving configured-path requests with query strings.

## Reproduction Assessment

Yes. Source inspection on current main shows `monitorWebhook` reads `webhookPath` but never compares the incoming request path before body parsing, signature validation, challenge handling, or dispatch.

## Solution Assessment

Yes. The narrow maintainable fix is Feishu-owned route enforcement at the webhook ingress boundary before body parsing and signature work; no broader SDK or product change is required.
