## Summary

Keep open. Current main and the latest release still do not enforce the configured Feishu webhookPath before processing signed webhook payloads, and an open implementation PR with closing syntax already tracks the exact fix.

## What This Changes

Feishu webhook mode should reject signed webhook requests whose URL path does not match channels.feishu.webhookPath.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/54904 or an equivalent Feishu plugin patch that normalizes the configured webhookPath, rejects mismatched paths before body parsing/signature validation/dispatch, and adds focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows monitorWebhook never compares req.url or URL.pathname to webhookPath before body parsing, signature validation, challenge handling, and dispatch, so a correctly signed POST to a different path follows the same pipeline.

## Solution Assessment

Yes. The narrow maintainable fix is Feishu-owned early path normalization and rejection before generic guards, body parsing, signature validation, and dispatch; the existing open PR is the right implementation path to review and land.
