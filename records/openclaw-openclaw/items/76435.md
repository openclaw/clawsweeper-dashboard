## Summary

Keep open. Current main still returns OpenAI attribution headers in the browser WebRTC SDP offer path and the UI forwards them to `api.openai.com`; the linked PR #76457 is the active closing fix candidate, so this issue should remain open until that PR lands or is replaced.

## What This Changes

The issue reports that OpenAI Realtime Talk fails because browser SDP POSTs include attribution headers rejected by OpenAI CORS.

## Best Possible Solution

Review and land PR #76457, or a narrow replacement, so the OpenAI provider strips server-only attribution from browser SDP offer headers while preserving Gateway-side attribution.

## Reproduction Assessment

Yes. Current source shows the OpenAI browser session can return `originator`/`version` in `offerHeaders`, the UI forwards those headers to the direct SDP `fetch`, and the prior ClawSweeper comment includes live CORS preflight proof for the endpoint.

## Solution Assessment

Yes. The best fix is provider-local: remove server-only OpenAI attribution from browser-visible `offerHeaders` and update focused provider/UI coverage, leaving the generic WebRTC session contract intact.
