## Summary

Keep this PR open. It targets a real display-default bug, but the current patch can override per-agent `thinkingDefault` in native `/status`, leaves a typed test mock with the wrong `ModelAliasIndex` shape, and still has only synthetic fixture proof for visible Telegram behavior.

## What This Changes

The PR changes native Telegram `/status` and Telegram `/think` menu display paths to hydrate runtime model catalog metadata for default thinking labels, with focused regression tests.

## Best Possible Solution

Keep the focused display-default fix, but preserve session, per-agent, global, per-model, and model-capability precedence consistently, repair the typed test mock, and add redacted live Telegram proof.

## Reproduction Assessment

Yes. Current main is source-reproducible: native `/status` supplies no default thinking resolver, and Telegram `/think` uses only configured catalog rows, so runtime-only reasoning metadata cannot affect those displays.

## Solution Assessment

No, not as submitted. Hydrating runtime catalog metadata is the right direction, but native `/status` must not override explicit per-agent defaults and the new typed test mock must satisfy the existing alias-index contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve per-agent /status thinking defaults:** `src/auto-reply/reply/get-reply-native-slash-fast-path.ts:138-140`
  - body: This computes a non-undefined `resolvedThinkLevel` before `buildStatusReply`, but `buildStatusText` gives that value precedence over `resolveAgentConfig(...).thinkingDefault`. Because the new resolver only receives `cfg/provider/model`, an agent with its own `thinkingDefault` will now show the model-derived default instead of the configured per-agent value. Keep the explicit per-agent/default precedence before falling back to runtime catalog metadata.
  - confidence: 0.9
- **[P2] Return a real ModelAliasIndex in the test mock:** `src/auto-reply/reply/get-reply.fast-path.test.ts:244-248`
  - body: This mock is typed as `resolveDefaultModel`'s return value, whose `aliasIndex` is a `ModelAliasIndex` object with `byAlias` and `byKey`; returning a bare `Map` violates that contract and should fail the core test typecheck. Use the new `emptyAliasIndex()` helper here too.
  - confidence: 0.86
