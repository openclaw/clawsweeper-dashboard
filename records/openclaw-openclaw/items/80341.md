## Summary

Keep open: this PR addresses a real Telegram thinking-default display bug and now has live default-status screenshot proof, but the head still has two blocking review issues: native `/status` can mask per-agent thinking defaults, and one new test mock violates the `ModelAliasIndex` contract behind the failing type-check lane.

## What This Changes

The PR hydrates runtime model catalog metadata for Telegram/native thinking status displays and adds regressions for `/status`, session overrides, and `/think` menu defaults.

## Best Possible Solution

Keep the runtime catalog hydration, but preserve thinking precedence by passing only explicit session overrides as resolved status values and letting the status formatter handle per-agent/global/model defaults; also return a real `ModelAliasIndex` in the test mock.

## Reproduction Assessment

Yes. Current main is source-reproducible: native `/status` supplies no useful thinking default resolver, Telegram `/think` uses only configured catalog rows, and the redacted Telegram screenshots show the default-status path changing from `Think: off` to `Think: medium`.

## Solution Assessment

No, not as submitted. Hydrating runtime catalog metadata is the right direction, but the native status path must preserve per-agent thinking defaults and the typed test mock must satisfy the existing alias-index contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve per-agent /status thinking defaults:** `src/auto-reply/reply/get-reply-native-slash-fast-path.ts:138-140`
  - body: This now passes a model-derived default as `resolvedThinkLevel` whenever the target session has no explicit override. `buildStatusText` gives `resolvedThinkLevel` precedence over `resolveAgentConfig(...).thinkingDefault`, so a configured per-agent `/status` default can be reported as the model default instead. Only pass an explicit session override here, or otherwise preserve the existing per-agent/default precedence before falling back to runtime catalog metadata.
  - confidence: 0.91
- **[P2] Return a real ModelAliasIndex in the mock:** `src/auto-reply/reply/get-reply.fast-path.test.ts:247`
  - body: This mock is typed as `resolveDefaultModel`'s return value, whose `aliasIndex` is a `ModelAliasIndex` object with `byAlias` and `byKey`. Returning a bare `Map` violates that contract and matches the failing `check-test-types` lane; use the `emptyAliasIndex()` helper here too.
  - confidence: 0.9
