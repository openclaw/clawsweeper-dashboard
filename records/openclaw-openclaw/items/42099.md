## Summary

Keep open: current main still warns for distinct config-vs-bundled duplicate plugin roots, and #72409 is the open closing PR for the requested diagnostic change. The remaining question is policy alignment, because docs and tests currently describe that warning as intentional.

## What This Changes

The issue requests suppressing duplicate plugin ID warnings when expected plugin discovery precedence, such as config-selected msteams overriding a bundled plugin, leaves only one active plugin.

## Best Possible Solution

Review #72409 against current manifest-registry behavior and docs, then either land it with matching docs/changelog if cross-origin override warnings should be silent, or close it and keep the documented warning policy.
