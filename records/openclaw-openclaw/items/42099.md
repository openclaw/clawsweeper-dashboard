Keep open: current main still emits the config-vs-bundled duplicate warning for distinct plugin roots, and open PR #72409 is already the closing implementation candidate. The blocker is maintainer policy alignment because the current docs and tests describe this warning as intentional.

The issue requests suppressing duplicate plugin ID warnings when expected cross-origin discovery, such as a config-pinned msteams plugin overriding the bundled plugin, keeps only one active plugin.
