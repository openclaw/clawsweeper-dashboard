## Summary

Keep open: current main and v2026.6.8 already suppress same-physical-root duplicate plugin warnings, but distinct-root config-selected overrides still warn by documented policy and the open replacement PR at https://github.com/openclaw/openclaw/pull/72409 formally owns that remaining diagnostic-policy decision.

## What This Changes

The issue asks the plugin manifest registry to stop logging duplicate plugin ID warnings for expected cross-origin discovery or shadowing, especially for msteams.

## Best Possible Solution

Resolve the linked replacement PR first: either keep the documented config-selected duplicate warning contract and close the PR/issue as covered for same-root false positives, or merge a refreshed PR that deliberately changes the diagnostic policy without regressing installed override suppression.

## Reproduction Assessment

Yes, by source inspection for the remaining behavior: current tests expect a warning for distinct-root config-over-bundled duplicates. The original same-physical-root false positive is not reproducible on current main because identical path/realpath duplicates are already suppressed.

## Solution Assessment

Unclear until maintainers resolve the linked PR. Current main is the narrow best solution for same-physical-root false positives, while silencing distinct-root cross-origin overrides would change the documented diagnostic contract.
