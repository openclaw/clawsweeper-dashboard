## Summary

Close as ClawHub/plugin scope: current main already has sub-agent model selection, local-model provider configuration, local service startup hooks, and provider/plugin extension seams; the remaining hardware-aware installer/benchmark/router is an optional integration and product policy layer that should not be bundled into core.

## What This Changes

The issue requests automatic local inference offload for sub-agents, including hardware-aware selection or installation of frameworks such as llama.cpp, vLLM, Whisper, and ComfyUI.

## Best Possible Solution

Keep OpenClaw core on the existing local-model and sub-agent seams, track the cookbook gap in https://github.com/openclaw/openclaw/issues/51333, and publish any hardware-aware backend chooser/installer as a ClawHub provider/tool plugin.
