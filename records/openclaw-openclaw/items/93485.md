## Summary

Keep open. Current main and the latest release still evaluate stale-pages freshness for concept and synthesis pages, but the requested exclusion/default policy and new plugin config surface need maintainer product review before an automated fix should implement it.

## What This Changes

The issue asks memory-wiki to stop treating durable concept and synthesis pages like ordinary aging pages in the Stale Pages report and to expose stale/aging thresholds as configuration.

## Best Possible Solution

Define the plugin-owned freshness policy for durable wiki pages, then update the stale-pages and lint freshness paths with focused tests and docs; add report configuration only if maintainers want operator-tunable behavior rather than a better default.

## Reproduction Assessment

Yes at source level: current main includes concept and synthesis pages in stale-pages freshness evaluation and has no exclusion or threshold config path. I did not run a vault compile because this review is read-only and compile writes report artifacts.

## Solution Assessment

No, not yet. The proposed configurable exclude list is plausible, but maintainers should first choose whether the best fix is a safer default, durable frontmatter, report-specific settings, or some combination.
