## Summary

Keep open: current main and the latest release still downgrade failed Slack external arg-menu registration to static menus with only verbose logging, and I found no merged/open canonical item that adds the requested operator-visible degraded health signal.

## What This Changes

The issue asks OpenClaw to keep Slack external arg-menu registration fallback non-fatal while making the degraded mode visible to operators.

## Best Possible Solution

Preserve the non-fatal fallback and publish a sanitized Slack-owned degraded status or warning through the existing channel status conventions so operators can see that external arg menus are disabled.

## Reproduction Assessment

Yes, at source level: current main catches app.options() registration failure, falls back to static_select, and the existing Slack slash test already exercises that path. I did not run tests because this review is read-only.

## Solution Assessment

Yes. A narrow Slack plugin-owned status or warning signal using existing setStatus/channel-status conventions is the maintainable path; making startup fatal or adding broad core health plumbing would be heavier than the reported gap.
