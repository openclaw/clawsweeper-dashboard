## Summary

Keep open: current main and v2026.5.12 still downgrade Slack external arg-menu registration failures to static menus with only verbose logging, and no linked PR now owns the requested visible health signal.

## What This Changes

The issue asks OpenClaw to show an operator-visible Slack health signal when external argument-menu registration falls back to static menus.

## Best Possible Solution

Add a Slack plugin-owned warning, startup status field, or diagnostics entry for the external arg-menu degraded mode while preserving the existing non-fatal static-menu fallback.

## Reproduction Assessment

Yes, at source level: the existing Slack slash test makes app.options() throw and verifies static_select fallback. This read-only review did not run tests, but the source path is direct and covered by current tests.

## Solution Assessment

Yes. A narrow Slack plugin change using existing runtime warning/status/diagnostic conventions is the best fit; this should not require a core health redesign, startup failure, or new config option.
