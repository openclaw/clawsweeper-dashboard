## Summary

Keep open. Current main still treats Slack external arg-menu registration failure as non-fatal, but the fallback is only recorded through verbose logging and static menu behavior; there is no visible warning, status detail, doctor signal, or Slack docs guidance for operators.

## What This Changes

The issue asks OpenClaw to keep Slack external arg-menu registration fallback non-fatal while surfacing the degraded mode as a visible warning, health/status signal, or diagnostic.

## Best Possible Solution

Add a plugin-owned Slack status detail and visible warning for external arg-menu registration fallback, preserving static-menu fallback and avoiding core health-model changes unless the existing channel status adapter needs a small generic field pass-through.

## Reproduction Assessment

Yes. The focused reproduction is already represented by a unit test that makes `app.options()` throw during Slack slash registration; current inspection shows that path falls back to `static_select` and only reaches `logVerbose`.

## Solution Assessment

Yes. A narrow Slack plugin fix that threads status/warning publication into slash registration is the best path; it keeps the existing non-fatal behavior and avoids a broad core diagnostics redesign.
