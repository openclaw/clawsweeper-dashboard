## Summary

Keep open. Current main still catches Slack `app.options()` registration failures and preserves the static-menu fallback, but the only operator-facing signal is a verbose log; there is no status, doctor, or docs-visible degraded-mode signal yet.

## What This Changes

Request a visible warning, health/status entry, or diagnostic signal when Slack external arg-menu registration falls back to static menus while keeping startup non-fatal.

## Best Possible Solution

Add plugin-owned Slack runtime status/status-issue output and a visible warning for external arg-menu fallback, while preserving the current non-fatal static-menu behavior and documenting the diagnostic path.

## Reproduction Assessment

Yes, at source level. The existing test harness can make `app.options()` throw during slash registration, and current main clearly falls back to `static_select` while only logging through `logVerbose`.

## Solution Assessment

Yes. A narrow Slack plugin change that threads degraded-mode reporting into slash registration is the best path; it avoids core health-model redesign and keeps provider startup resilient.
