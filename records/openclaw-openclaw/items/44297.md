## Summary

Keep open: current main still downgrades Slack external arg-menu registration failures to static menus with only verbose logging, and the degraded mode is not exposed through status, doctor output, or Slack docs.

## What This Changes

The issue asks Slack to keep the non-fatal external arg-menu fallback while surfacing the degraded mode through a visible warning, health/status signal, or diagnostics.

## Best Possible Solution

Add Slack plugin-owned runtime status/status-issue output plus a normal operator-visible warning for external arg-menu fallback, and document the diagnostic path while preserving static-menu fallback.

## Reproduction Assessment

Yes, at source level. The existing Slack slash test can make app.options() throw, and current main then falls back to static_select while only logging through logVerbose.

## Solution Assessment

Yes. A narrow Slack plugin change using existing status and runtime-warning conventions is the maintainable fix; a core health-model redesign is not needed.
