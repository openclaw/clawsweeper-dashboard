## Summary

Keep open: current main and the latest release still downgrade failed Slack external arg-menu registration to static menus with only verbose logging, and I found no merged or open related item that adds an operator-visible degraded health signal.

## What This Changes

Request an operator-visible warning, status, or diagnostic signal when Slack external arg-menu registration falls back to static menus.

## Best Possible Solution

Preserve the non-fatal fallback and publish a sanitized Slack-owned degraded status or warning through the existing channel status conventions so operators can see that external arg menus are disabled.

## Reproduction Assessment

Yes, at source level: current code catches app.options() registration failure and only calls logVerbose, and the existing Slack slash test already simulates that throw while verifying static_select fallback. I did not run tests because this review is read-only.

## Solution Assessment

Yes. A narrow Slack plugin-owned status or warning signal using existing setStatus/channel-status conventions is the maintainable path; making startup fatal or adding broad core health plumbing would be heavier than the reported gap.
