## Summary

This PR targets a plausible packaged-runtime bug, but it should not land as written: it exposes a test-only hook through a public plugin SDK subpath, relies on scanning arbitrary hashed chunk names instead of restoring a stable dist artifact, and still lacks usable real behavior proof for the changed path.

## What This Changes

The PR adds a hashed-chunk fallback for `provider-auth-runtime`, exports a test-only resolver helper, and adds a unit test for the no-stable-alias path.

## Best Possible Solution

Keep the runtime auth module on a stable package dist path, or create the stable alias the existing resolver expects, and cover that packaged path without expanding the public SDK surface.

## Reproduction Assessment

No high-confidence executed reproduction was established in this read-only review. Source inspection does show a credible current-main failure path: the resolver only checks stable filenames while the runtime auth module is loaded through a lazy runtime chunk.

## Solution Assessment

No. The narrower maintainable fix is to preserve or emit a stable dist entry for the runtime auth module and keep resolver test hooks private, rather than importing the first matching hashed chunk from a package directory.
