## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Redact activity tool arguments before streaming:** `src/agents/pi-embedded-subscribe.handlers.tools.ts:411`
  - body: This serializes raw tool args into activity metadata. Tool args can include secrets, private paths, headers, or command-embedded credentials, and the PR later stores the same metadata in localStorage.
  - confidence: 0.92
- **[P2] Keep activity telemetry ephemeral by default:** `ui/src/ui/activity/activity-controller.ts:247`
  - body: `handleEvent` schedules a localStorage save for every activity event. Since the activity tree can include tool inputs, tool outputs, and subagent result previews, opening the Activity tab would persist sensitive execution data despite the stated ephemeral default.
  - confidence: 0.94
- **[P2] Preserve matching descendants while filtering:** `ui/src/ui/activity/activity-tree.ts:347-349`
  - body: `filterTree` drops nodes whose own kind is not selected and then rebuilds roots only from surviving original roots. A tool-only or subagent-only filter can render an empty tree even when matching descendants exist under a run.
  - confidence: 0.9
- **[P2] Emit a terminal event when subagent startup fails:** `src/agents/subagent-spawn.ts:725-738`
  - body: The PR emits `subagent.start` before the child gateway request, but the catch and registration-failure paths return errors without any matching activity end/error event. Failed spawns can leave a running subagent node until reload or pruning.
  - confidence: 0.86
- **[P2] Sync activity strings across all shipped locales:** `ui/src/i18n/locales/en.ts:32`
  - body: Current main ships 19 locale files, but this PR updates only the older six-locale set. After rebase, the Activity tab strings need to be synced across all shipped locale bundles and locale tests.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: The diff introduces sensitive activity telemetry paths and persists that telemetry locally by default.

Concerns:

- **[medium] Activity metadata is persisted by default:** `ui/src/ui/activity/activity-controller.ts:247`
  - body: Activity events attach tool args/results and subagent result previews, and the controller saves the serialized activity tree to localStorage after handling events. That can retain sensitive commands, paths, output snippets, or child-agent results without opt-in.
  - confidence: 0.93
- **[medium] Raw tool args are emitted on the activity stream:** `src/agents/pi-embedded-subscribe.handlers.tools.ts:411`
  - body: The new activity event serializes unsanitized tool args into metadata. This can leak values that established user-facing summaries avoid exposing, especially once the activity tree is also saved locally.
  - confidence: 0.91
