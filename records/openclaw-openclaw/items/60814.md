## Summary

Keep open. Current main does not implement the Activity dashboard, and this draft PR still has concrete privacy and correctness issues that should block merge until the author reworks the design.

## What This Changes

The PR adds gateway activity events for agent lifecycle, thinking, tool calls, and subagents, plus a new Control UI Activity tab with execution tree, timeline, metrics, filters, persistence, i18n strings, and tests.

## Best Possible Solution

Rework this into a reviewed, additive activity contract plus a Control UI panel that is ephemeral and redacted by default, preserves tree structure under filters, emits terminal events for failed subagent starts, updates generated i18n through the supported pipeline, and aligns with the broader activity-panel/timeline direction before landing.

## Reproduction Assessment

Not applicable as a user bug reproduction; this is a feature PR review. There is a high-confidence source-review path: current main lacks the Activity tab/route, while the PR diff exposes the localStorage persistence, filtered-tree, and failed-spawn issues directly.

## Solution Assessment

No, the current patch is not the best way to solve the issue as-is. The direction is plausible, but the maintainable path needs an explicit privacy/redaction model, ephemeral default storage semantics, robust tree filtering, and complete lifecycle closure before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Keep activity events ephemeral by default:** `ui/src/ui/activity/activity-controller.ts:247`
  - body: The controller schedules a localStorage save for every activity event. Since the gateway events include summarized tool args/results and subagent results, opening the Activity tab would persist sensitive execution data despite the PR's stated ephemeral default. Gate persistence behind an opt-in or store only redacted/minimal state.
  - confidence: 0.93
- **[P2] Preserve ancestors in filtered trees:** `ui/src/ui/activity/activity-tree.ts:348-349`
  - body: filterTree drops run nodes when the kind filter excludes run, but the view renders the execution tree only from rootNodes. Selecting a tool-only or subagent-only filter can leave the tree panel blank even though matching nodes exist. Keep ancestors needed for display or build filtered roots from matching descendants.
  - confidence: 0.9
- **[P2] Close failed subagent starts:** `src/agents/subagent-spawn.ts:721`
  - body: subagent.start is emitted before the child gateway run request, but the catch path returns an error without emitting a terminal activity event. A failed spawn would leave a running subagent node in the Activity panel until reload or prune. Emit an error/end event on failure or move start emission after successful registration.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The PR introduces a concrete privacy concern by persisting sensitive activity metadata in browser localStorage by default.

Concerns:

- **[medium] Activity metadata is persisted by default:** `ui/src/ui/activity/activity-controller.ts:247`
  - body: The proposed activity events attach summarized tool args/results and subagent result previews, and the controller saves the serialized activity tree to localStorage after handling events. That changes ephemeral execution telemetry into durable browser data and can retain sensitive commands, file paths, output snippets, or child-agent results without an opt-in.
  - confidence: 0.92
