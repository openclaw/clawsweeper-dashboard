## Summary

Keep open. Current main does not implement a real-time Activity dashboard, and the draft PR is useful source work, but the proposed patch is not merge-ready because it persists sensitive activity metadata by default, has tree/filter correctness bugs, leaves failed subagent starts unterminated, and is stale against the current generated-locale contract.

## What This Changes

The PR adds gateway activity events for agent lifecycle, thinking, tools, and subagents, plus a new Control UI Activity tab with execution tree, timeline, metrics, filters, and persisted client-side state.

## Best Possible Solution

Land a narrower reviewed activity telemetry contract and Control UI surface that is ephemeral and redacted by default, preserves tree structure under filters, emits terminal events for all started activity nodes, and regenerates all locale outputs through the supported i18n pipeline.

## Reproduction Assessment

Not applicable as a user bug reproduction; this is a feature PR review. There is high-confidence verification by comparing current main source/docs against the PR diff and by tracing the static failure paths in the proposed activity controller, tree filter, subagent spawn, and i18n changes.

## Solution Assessment

No, the current patch is not the best way to solve the issue as-is. The direction is plausible, but the maintainable path is an additive protocol/UI contract with privacy-safe defaults, generated i18n, and focused behavior tests before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Keep activity events ephemeral by default:** `ui/src/ui/activity/activity-controller.ts:247`
  - body: `handleEvent` schedules a localStorage save for every activity event. Because the new events include summarized tool inputs, tool outputs, and subagent result previews, opening the Activity tab would persist sensitive execution data despite the PR's stated ephemeral default. Gate persistence behind an explicit opt-in or store only redacted/minimal state.
  - confidence: 0.93
- **[P2] Preserve ancestors in filtered trees:** `ui/src/ui/activity/activity-tree.ts:348-349`
  - body: `filterTree` drops run nodes when the kind filter excludes `run`, then rebuilds `rootNodes` only from surviving original roots. Selecting a tool-only or subagent-only filter can leave the tree panel blank even though matching descendants exist. Keep ancestors needed for display or derive filtered roots from matching descendants.
  - confidence: 0.9
- **[P2] Close failed subagent starts:** `src/agents/subagent-spawn.ts:725`
  - body: The PR emits `subagent.start` before the child gateway run request, but the existing catch path returns an error without emitting any terminal activity event. A failed spawn would leave a running subagent node in the Activity panel until reload or pruning. Emit an error/end event on failure or move the start event after successful registration.
  - confidence: 0.84
- **[P2] Regenerate every supported locale:** `ui/src/i18n/locales/en.ts:32`
  - body: The PR adds new English `tabs.activity` and `subtitles.activity` keys but updates only a subset of non-English locale bundles. Current main imports all shipped locales and asserts their flattened key shape equals English, so locales such as `ar`, `fa`, `fr`, `ja-JP`, and others will be missing these keys. Run the Control UI i18n sync pipeline and commit all generated locale outputs.
  - confidence: 0.92

## Security Review

Status: needs_attention

Summary: The diff introduces a concrete privacy concern by persisting activity metadata with tool inputs, tool outputs, and subagent result previews in browser localStorage by default.

Concerns:

- **[medium] Activity metadata is persisted by default:** `ui/src/ui/activity/activity-controller.ts:247`
  - body: The proposed activity events attach summarized tool args/results and subagent result previews, and the controller saves the serialized activity tree to localStorage after handling events. That changes ephemeral execution telemetry into durable browser data and can retain sensitive commands, paths, output snippets, or child-agent results without an opt-in.
  - confidence: 0.92
