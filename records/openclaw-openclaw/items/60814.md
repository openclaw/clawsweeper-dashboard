## Summary

Keep this PR open. Current main still lacks the dedicated activity dashboard, but the draft branch has concrete blockers around privacy, tree filtering, subagent terminal events, and generated locale coverage, so this needs maintainer design review and author rework rather than cleanup closure.

## What This Changes

The PR adds gateway-side activity events and a Control UI Activity tab with an execution tree, timeline, metrics, filters, styles, local persistence, tests, and partial locale updates.

## Best Possible Solution

Land a narrower reviewed activity telemetry contract and Control UI surface that is ephemeral and redacted by default, preserves tree structure under filters, emits terminal events for every started activity node, regenerates all locales, and updates docs/changelog.

## Reproduction Assessment

Not applicable as a user bug reproduction; this is a feature PR review. The static failure paths are high-confidence from the PR diff and current-main contracts.

## Solution Assessment

No, this PR is not the best way to solve the issue as-is. The safer path is an additive protocol/UI design with privacy-safe defaults, complete locale generation, and focused behavior coverage before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Keep activity events ephemeral by default:** `ui/src/ui/activity/activity-controller.ts:247`
  - body: `handleEvent` schedules a localStorage save for every activity event. Because these events include summarized tool inputs, tool outputs, and subagent result previews, opening the Activity tab would persist sensitive execution data despite the PR's stated ephemeral default. Gate persistence behind an explicit opt-in or store only redacted/minimal state.
  - confidence: 0.93
- **[P2] Preserve ancestors in filtered trees:** `ui/src/ui/activity/activity-tree.ts:348-349`
  - body: `filterTree` drops run nodes when the kind filter excludes `run`, then rebuilds `rootNodes` only from surviving original roots. Selecting a tool-only or subagent-only filter can leave the tree panel blank even though matching descendants exist. Keep ancestors needed for display or derive filtered roots from matching descendants.
  - confidence: 0.9
- **[P2] Close failed subagent starts:** `src/agents/subagent-spawn.ts:725`
  - body: The PR emits `subagent.start` before the child gateway run request, but the existing catch path returns an error without emitting any terminal activity event. A failed spawn would leave a running subagent node in the Activity panel until reload or pruning. Emit an error/end event on failure or move the start event after successful registration.
  - confidence: 0.84
- **[P2] Regenerate every supported locale:** `ui/src/i18n/locales/en.ts:32`
  - body: The PR adds new English `tabs.activity` and `subtitles.activity` keys but updates only a subset of non-English locale bundles. Current main imports all shipped locales and asserts their flattened key shape equals English, so locales such as `ar`, `fa`, `fr`, `ja-JP`, and others will be missing these keys. Run the Control UI i18n sync pipeline and commit all generated locale outputs.
  - confidence: 0.92

## Security Review

That changes ephemeral execution telemetry into durable browser data and can retain sensitive commands, paths, output snippets, or child-agent results without an opt-in.
