## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Redact activity tool arguments before emitting them:** `src/agents/pi-embedded-subscribe.handlers.tools.ts:411`
  - body: Current main sanitizes tool args before sending live tool events, but the new activity event serializes the raw args into metadata. That can re-expose values the existing event path redacts, and the controller later persists the same metadata, so feed sanitized/minimal args into activity metadata instead.
  - confidence: 0.9
- **[P2] Keep activity events ephemeral by default:** `ui/src/ui/activity/activity-controller.ts:247`
  - body: handleEvent schedules a localStorage save for every activity event. Because the emitted metadata can include tool inputs, tool outputs, and subagent result previews, opening the Activity tab would persist sensitive execution data despite the PR's stated ephemeral default; gate persistence behind explicit opt-in or persist only redacted state.
  - confidence: 0.93
- **[P2] Preserve ancestors when filtering the activity tree:** `ui/src/ui/activity/activity-tree.ts:348-349`
  - body: filterTree drops nodes whose own kind is not selected and then rebuilds roots only from surviving original roots. A tool-only or subagent-only filter can render an empty tree even when matching descendants exist under a run; keep display ancestors or derive filtered roots from matching descendants.
  - confidence: 0.9
- **[P2] Emit a terminal event for failed subagent starts:** `src/agents/subagent-spawn.ts:725`
  - body: The PR emits subagent.start before the child gateway run request, but the catch path returns an error without any matching activity end or error event. Failed spawns leave a running subagent node in the Activity panel until reload or pruning; emit a terminal event on failure or move the start event after successful registration.
  - confidence: 0.84
- **[P2] Regenerate all shipped locale bundles:** `ui/src/i18n/locales/en.ts:32`
  - body: The PR adds English activity tab keys but updates only a subset of non-English locale bundles. Current main imports all shipped locales and asserts each flattened key shape equals English, so locales such as ar, fa, fr, and ja-JP will be missing these keys after rebase; run the Control UI i18n sync pipeline and commit all generated locale outputs.
  - confidence: 0.92

## Security Review

Status: needs_attention

Summary: The diff introduces security-sensitive activity telemetry handling that can expose raw tool args and persist execution metadata in browser storage by default.

Concerns:

- **[medium] Activity metadata is persisted by default:** `ui/src/ui/activity/activity-controller.ts:247`
  - body: Activity events attach tool args/results and subagent result previews, and the controller saves the serialized activity tree to localStorage after handling events. That changes ephemeral execution telemetry into durable browser data and can retain sensitive commands, paths, output snippets, or child-agent results without opt-in.
  - confidence: 0.92
- **[medium] Raw tool args are emitted on the activity stream:** `src/agents/pi-embedded-subscribe.handlers.tools.ts:411`
  - body: The new activity event serializes the unsanitized tool args into metadata even though current main sanitizes live tool-event args before emission. This can leak values that the established tool stream redacts, especially once the activity tree is also saved locally.
  - confidence: 0.9
