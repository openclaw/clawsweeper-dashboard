## Summary

Keep open. This draft feature PR still adds a meaningful capability that is not on current main, but the proposed patch has blocking correctness and privacy issues plus missing real behavior proof before it can be considered for merge.

## What This Changes

The branch adds gateway-side activity events and a Control UI Activity tab with an execution tree, timeline, filters, metrics, local persistence, i18n updates, and tests.

## Best Possible Solution

Land a narrower maintainer-reviewed activity telemetry contract and Control UI surface that is ephemeral and redacted by default, preserves tree structure under filters, emits terminal events for every started node, regenerates all locales, and coordinates with the open observer-mode tracker.

## Reproduction Assessment

Not applicable as a user bug reproduction; this is a feature PR. The review findings are source-reproducible from the PR head and current-main contracts: default persistence, filtered tree loss, failed-spawn lifecycle gaps, and locale drift are visible in the changed code.

## Solution Assessment

No. The proposed branch is not the best mergeable shape as-is; the safer path is a smaller additive protocol/UI design with privacy-safe defaults, complete locale generation, terminal event coverage, and real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Keep activity events ephemeral by default:** `ui/src/ui/activity/activity-controller.ts:247`
  - body: `handleEvent` schedules a localStorage save for every activity event. Because the emitted metadata includes tool inputs, tool outputs, and subagent result previews, opening the Activity tab would persist sensitive execution data despite the PR's stated ephemeral default. Gate persistence behind an explicit opt-in or persist only redacted/minimal state.
  - confidence: 0.93
- **[P2] Preserve ancestors when filtering the activity tree:** `ui/src/ui/activity/activity-tree.ts:348-349`
  - body: `filterTree` drops nodes whose own kind is not selected and then rebuilds roots only from surviving original roots. A tool-only or subagent-only filter can therefore render an empty tree even when matching descendants exist under a run. Keep display ancestors or derive filtered roots from matching descendants.
  - confidence: 0.9
- **[P2] Emit a terminal event for failed subagent starts:** `src/agents/subagent-spawn.ts:725`
  - body: The PR emits `subagent.start` before the child gateway run request, but the catch path returns an error without any matching activity end/error event. Failed spawns leave a running subagent node in the Activity panel until reload or pruning. Emit a terminal event on failure or move the start event after successful registration.
  - confidence: 0.84
- **[P2] Regenerate all shipped locale bundles:** `ui/src/i18n/locales/en.ts:32`
  - body: The PR adds English `tabs.activity` and `subtitles.activity` keys but updates only a subset of non-English locale bundles. Current main imports all shipped locales and asserts each flattened key shape equals English, so locales such as `ar`, `fa`, `fr`, and `ja-JP` will be missing these keys after rebase. Run the Control UI i18n sync pipeline and commit all generated locale outputs.
  - confidence: 0.92
