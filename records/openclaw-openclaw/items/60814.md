## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Redact activity tool arguments before emitting them:** `src/agents/pi-embedded-subscribe.handlers.tools.ts:411`
  - body: Current main sanitizes tool args before exposing live tool events, but this new activity event serializes raw args into metadata. That can re-expose API keys, Authorization headers, command-embedded secrets, or private paths, especially because the same metadata is later persisted.
  - confidence: 0.9
- **[P2] Keep activity events ephemeral by default:** `ui/src/ui/activity/activity-controller.ts:247`
  - body: `handleEvent` schedules a localStorage save for every activity event. Since activity metadata can include tool inputs, tool outputs, and subagent result previews, opening the Activity tab would persist sensitive execution data despite the stated ephemeral default.
  - confidence: 0.93
- **[P2] Preserve ancestors when filtering the activity tree:** `ui/src/ui/activity/activity-tree.ts:348-349`
  - body: `filterTree` drops nodes whose own kind is not selected and then rebuilds roots only from surviving original roots. A tool-only or subagent-only filter can render an empty tree even when matching descendants exist under a run; keep display ancestors or derive roots from matching descendants.
  - confidence: 0.9
- **[P2] Emit a terminal event when subagent startup fails:** `src/agents/subagent-spawn.ts:725-738`
  - body: The PR emits `subagent.start` before the child gateway run request, but the catch path returns an error without any matching activity end or error event. Failed spawns leave a running subagent node until reload or pruning.
  - confidence: 0.84
- **[P2] Regenerate every shipped locale bundle:** `ui/src/i18n/locales/en.ts:32`
  - body: The PR adds English Activity tab keys but updates only a subset of locale bundles. Current main imports all shipped locales and asserts each flattened key shape equals English, so locales such as ar, fa, fr, and ja-JP would be missing these keys after rebase.
  - confidence: 0.92

## Security Review

Status: needs_attention

Summary: The PR introduces sensitive activity telemetry paths and persists that telemetry locally by default.

Concerns:

- **[medium] Activity metadata is persisted by default:** `ui/src/ui/activity/activity-controller.ts:247`
  - body: Activity events attach tool args/results and subagent result previews, and the controller saves the serialized activity tree to localStorage after handling events. That changes ephemeral execution telemetry into durable browser data and can retain sensitive commands, paths, output snippets, or child-agent results without opt-in.
  - confidence: 0.92
- **[medium] Raw tool args are emitted on the activity stream:** `src/agents/pi-embedded-subscribe.handlers.tools.ts:411`
  - body: The new activity event serializes unsanitized tool args into metadata even though current main sanitizes live tool-event args before emission. This can leak values that the established tool stream redacts, especially once the activity tree is also saved locally.
  - confidence: 0.9
