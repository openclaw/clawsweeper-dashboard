## Summary

This is a valid ClawHub owner-management feature request, but the implementation belongs to the external ClawHub service and separate `clawhub` CLI, while this repository only documents and consumes ClawHub search/install/download APIs.

## What This Changes

Request version-level owner/admin support to hide or delete individual ClawHub skill versions without removing the whole skill listing.

## Best Possible Solution

Implement owner/admin version hide/delete policy in the external ClawHub service and CLI, then update OpenClaw docs or consumers here only if the public contract changes.

## Reproduction Assessment

Not applicable; this is a feature request for registry owner/admin version management, not a broken OpenClaw behavior. Source and docs inspection show OpenClaw consumes ClawHub search/install/download APIs while the requested mutation belongs to ClawHub.

## Solution Assessment

Yes; the narrow maintainable solution is to design authenticated version-level mutation, latest-version policy, auditability, and guardrails in ClawHub itself. OpenClaw should only pick up docs or client changes after that external API exists.
