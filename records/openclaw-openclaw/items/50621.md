## Summary

Keep open: current main and the latest release still apply the generic cron watchdog to main-session `systemEvent` jobs while `wakeMode: "now"` waits on heartbeat work, so the false `cron: job execution timed out` state remains source-reproducible. The duplicate report and closed unmerged fix attempts add useful confirmation, but there is no merged fix or open canonical PR resolving this issue.

## What This Changes

The issue asks cron main-session `systemEvent` jobs to avoid a short scheduler timeout when `wakeMode: "now"` blocks on the main agent turn.

## Best Possible Solution

Apply an agent-turn-scale safety timeout to the blocking main-session `systemEvent` plus `wakeMode: "now"` path while preserving the 10-minute default for non-blocking `systemEvent` jobs and existing isolated `agentTurn` timeout behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a main-session `systemEvent` job with `wakeMode: "now"` awaits `runHeartbeatOnce` while the cron wrapper races that work against the generic 10-minute timeout.

## Solution Assessment

Yes. The narrow maintainable fix is to adjust the timeout policy for the blocking main-session `systemEvent` wake-now path; a broad new config option or global `systemEvent` timeout change is not required for this bug.

## Repair Work Prompt

Fix the cron main-session `systemEvent` timeout bug reported at https://github.com/openclaw/openclaw/issues/50621. Current main gives all `systemEvent` jobs `DEFAULT_JOB_TIMEOUT_MS`, but `sessionTarget: "main"` plus `wakeMode: "now"` awaits `runHeartbeatOnce`, so long main-session agent work can be recorded as `cron: job execution timed out` even when work completes. Update the cron timeout policy so only the blocking main-session `systemEvent` wake-now path uses the agent-turn-scale safety timeout, while preserving the 10-minute default for `next-heartbeat` and preserving existing `agentTurn` explicit `timeoutSeconds` and no-timeout semantics. Add focused regression coverage in timeout-policy and the main wake-now cron path if needed. Do not add a broad new config option, do not make all `systemEvent` jobs unbounded, and do not change isolated agent cleanup behavior. Add a user-facing changelog fix entry unless maintainers decide this is internal-only.
