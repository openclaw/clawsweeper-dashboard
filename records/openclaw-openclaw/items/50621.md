## Summary

Keep open: current main still applies the generic cron watchdog to `systemEvent` jobs, and https://github.com/openclaw/openclaw/pull/86733 is the unmerged closing PR that now owns the fix path.

## What This Changes

Main-session cron `systemEvent` jobs with `wakeMode: "now"` can be marked timed out by the cron wrapper while the agent heartbeat turn is still running.

## Best Possible Solution

Land a clean cron-only fix through https://github.com/openclaw/openclaw/pull/86733 or an equivalent branch, then close this issue after the merge reaches main.

## Reproduction Assessment

Yes at source level: current main maps non-agent `systemEvent` payloads to the 10-minute default timeout while the main-session wake-now path awaits `runHeartbeatOnce`. I did not run a live provider task, but the code path and duplicate report make the failure mode high-confidence.

## Solution Assessment

Yes, conditionally: reusing the existing longer safety ceiling only for the blocking wake-now path is narrower than removing the watchdog or adding new config. The issue should remain open until the linked PR is cleaned up, accepted for its availability tradeoff, and merged.
