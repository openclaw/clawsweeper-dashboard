## Summary

Close as implemented on current main. The legacy root/compat plugin SDK path now re-exports the reply-prefix and reply-pipeline helpers, the root SDK alias still delegates legacy imports through that compat surface, and a root-alias contract test now asserts these helpers resolve as functions. The fix is on main but is not contained in the latest tagged release v2026.4.29.

## What This Changes

The issue reports a Feishu dispatch crash after update because external plugins using the legacy `openclaw/plugin-sdk` root import could not call `createReplyPrefixContext`.

## Best Possible Solution

Keep the additive compat restoration on main, ship it in the next OpenClaw release, and keep directing plugin authors toward `openclaw/plugin-sdk/channel-reply-pipeline` before the legacy root SDK removal window ends.

## Reproduction Assessment

Yes. The high-confidence path is an external plugin importing `createReplyPrefixContext` from `openclaw/plugin-sdk` and exercising inbound dispatch; current source inspection confirms that path now resolves through root alias into compat with the helper present.

## Solution Assessment

Yes. Restoring the missing compat exports is the narrowest maintainable fix while `openclaw/plugin-sdk` remains inside its documented compatibility window; fixing only Feishu would leave other legacy external plugins broken.
