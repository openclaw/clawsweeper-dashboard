## Best Possible Solution

Land #75461 or an equivalent narrow SDK compatibility repair that re-exports the existing reply-prefix and reply-pipeline helpers from `src/plugin-sdk/compat.ts`, adds root/compat contract coverage, and keeps the focused `channel-reply-pipeline` subpath as the recommended migration target.
