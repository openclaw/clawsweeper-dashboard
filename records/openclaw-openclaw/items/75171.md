## Summary

Close as implemented on current main. The legacy root/compat Plugin SDK surface now restores the reply-prefix and reply-pipeline helpers that the Feishu plugin path was missing, and current contract coverage asserts those helpers through the merged root SDK wrapper. The fix is on main after the latest published release, so users on v2026.4.29 or earlier still need a post-fix build/release.

## What This Changes

The issue reports a Feishu dispatch crash after update because the legacy `openclaw/plugin-sdk` root import did not provide `createReplyPrefixContext`.

## Best Possible Solution

Keep the additive compat restoration on main, keep focused `openclaw/plugin-sdk/channel-reply-pipeline` imports as the recommended migration path, and let the next release carry the fix to npm/global-install users.

## Reproduction Assessment

Yes. The issue provides a deterministic Feishu dispatch path, and the underlying failure is reproduced at the contract level by checking that legacy root SDK imports resolve through `root-alias.cjs` into compat where the helpers now exist.

## Solution Assessment

Yes. Restoring the missing helpers on the deprecated root/compat SDK surface is the narrowest maintainable fix while the migration window remains active; fixing only Feishu would leave other external root-import plugins broken.
