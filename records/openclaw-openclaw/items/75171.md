## Summary

Keep open. Current main still has the SDK compatibility gap described by the report: the focused channel reply pipeline SDK exports the reply-prefix helpers, but the deprecated compat surface that the root plugin SDK alias loads for legacy external plugins still does not expose them during the active migration window.

## What This Changes

The issue requests restoring Feishu message dispatch after an update where a legacy `openclaw/plugin-sdk` import resolves `createReplyPrefixContext` to a missing function.

## Best Possible Solution

Restore the missing reply-prefix and reply-pipeline helpers on the deprecated compat-backed root SDK path for the active migration window, keep the focused `channel-reply-pipeline` subpath as the canonical import, and lock both paths with contract tests.

## Reproduction Assessment

Yes. The issue gives a deterministic Feishu dispatch path, and current-main source inspection reproduces the underlying failure condition: legacy root imports resolve through compat, while compat lacks the helper exported by the focused subpath.

## Solution Assessment

Yes. The narrow maintainable fix is to restore the missing public compatibility exports and tests; changing only Feishu would leave other external plugins using the still-supported root/compat surface broken.
