## Summary

Close as implemented on current main. The legacy root/compat SDK surface now re-exports the reply-prefix and reply-pipeline helpers, the root alias still delegates legacy `openclaw/plugin-sdk` imports through that compat module, and the root-alias contract test now asserts these helpers are functions. The fix is on main only; no current release tag contains it yet.

## What This Changes

Restore legacy `openclaw/plugin-sdk` root/compat helper exports so Feishu and other external plugins do not crash on `createReplyPrefixContext` after update.

## Best Possible Solution

Keep the additive compat restoration on main, ship it in the next release, and leave the focused `openclaw/plugin-sdk/channel-reply-pipeline` subpath as the documented migration target for plugin authors.

## Reproduction Assessment

Yes. The issue provides a deterministic Feishu dispatch failure, and the underlying SDK path is source-verifiable: root imports load compat, and compat now exposes the helper that was previously missing.

## Solution Assessment

Yes. Restoring the helper on the deprecated compat/root surface is the narrowest maintainable fix during the active migration window; fixing only Feishu would leave other external root-SDK consumers broken.
