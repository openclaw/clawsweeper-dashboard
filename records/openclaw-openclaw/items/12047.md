## Summary

Current main now handles Telegram document download plus shared text and PDF extraction, but it still intentionally skips DOCX, XLSX, PPTX, and legacy Office files; the prior DOCX PR was closed unmerged because its untrusted-archive parsing was not merge-ready, leaving this issue as the canonical remaining work.

## What This Changes

Add text extraction for document attachments received through Telegram so their contents become available in agent context.

## Best Possible Solution

Add format-specific, resource-bounded Office extractors through the existing shared document-extractor plugin contract, beginning with an explicitly approved format such as DOCX, while preserving MIME sniffing, binary safety, untrusted-content wrapping, and channel-independent behavior.

## Reproduction Assessment

Yes, from current source with high confidence: Telegram forwards saved documents into shared media processing, while Office MIME types are intentionally rejected before extraction and the bundled extractor supports only PDF.

## Solution Assessment

No, a Telegram-local extractor or new channel flag is not the best approach; the narrow maintainable solution is bounded format support in the shared document-extractor contract so all inbound file paths receive consistent safety and behavior.
