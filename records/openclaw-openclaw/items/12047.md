## Summary

Keep open. Current main partially addresses the request by downloading Telegram documents and passing them into shared media context, but Office-style documents are still intentionally skipped before extraction and the related DOCX implementation remains open.

## What This Changes

Request text extraction from Telegram document attachments such as DOCX, PDF, TXT, and Markdown into the agent message context.

## Best Possible Solution

Finish this through the shared document-extract/media pipeline with explicit supported Office MIME types, bounded extraction, preserved binary-safety checks, untrusted-content wrapping, Telegram/shared-media coverage, and public docs.

## Reproduction Assessment

Yes, from source inspection. Telegram forwards saved document media into shared context, but Office MIME types are not in the default supported set and are skipped as binary before extractor lookup, with a regression test asserting that behavior.

## Solution Assessment

No for a Telegram-specific extractor. The maintainable solution is shared document extraction so Telegram, input_file/read flows, and future plugins reuse the same MIME support, limits, and untrusted-content boundaries.
