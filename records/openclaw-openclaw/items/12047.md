Keep open: current main and v2026.6.10 cover Telegram document download plus text/PDF extraction, but Office-style Telegram attachments are still skipped as binary and the related DOCX PR is closed unmerged.

## What This Changes

Request to extract readable text from Telegram document attachments such as DOCX, PDF, TXT, and Markdown and include it in agent-visible message context.

## Best Possible Solution

Finish through the shared document-extractor and media-understanding path with maintainer-approved Office MIME scope, bounded parsers, untrusted-content wrapping, focused Telegram/input_file coverage, and docs.

## Reproduction Assessment

Yes, at source level: current main passes saved Telegram document paths and MIME types into shared media context, but Office MIME types are classified as binary and skipped before extraction. I did not run a live Telegram bot reproduction in this read-only review.

## Solution Assessment

No to a Telegram-only parser or default config toggle. The maintainable solution is shared document extraction so Telegram, input_file, and adjacent attachment surfaces use one bounded document capability.
