## Summary

Keep open. Current main downloads Telegram documents and can extract text/PDF through the shared media path, but Office formats are still intentionally filtered before extraction and the related DOCX candidate remains open.

## What This Changes

Request to extract text from Telegram document attachments such as DOCX, PDF, TXT, Markdown, and similar files so agents can read their contents.

## Best Possible Solution

Finish this through the shared document-extraction pipeline with bounded Office MIME/extractor support, preserved binary-safety checks, untrusted-content wrapping, focused tests, and docs for supported formats.

## Reproduction Assessment

Yes, from source inspection rather than a live Telegram upload. A Telegram DOCX/XLSX/PPTX attachment is downloaded and forwarded as media context, then Office MIME types hit the binary skip path before extraction, with tests asserting that skip behavior.

## Solution Assessment

No for a Telegram-specific extractor. The maintainable solution is shared document extraction so Telegram, OpenResponses input_file, read/media paths use the same MIME support, limits, and untrusted-content boundaries.
