## Summary

Keep open. Current main still only partially satisfies the request: Telegram document attachments are downloaded and passed into shared media context, but Office-style documents remain outside the supported extraction path and are deliberately skipped before extractor lookup; the related DOCX PR is still open and has unresolved security/product-boundary review needs.

## What This Changes

Request to extract text from Telegram document attachments and include that text in the agent context for document formats such as DOCX, PDF, TXT, and Markdown.

## Best Possible Solution

Finish this through the shared document-extract/media pipeline with explicit supported Office MIME types, bounded extraction, preserved binary-safety checks, untrusted-content wrapping, Telegram/shared-media coverage, and public docs for supported formats.

## Reproduction Assessment

Yes, from source inspection. Telegram forwards saved document media into shared context, but Office MIME types are not in the default supported set and are skipped as binary before extractor lookup, with a regression test asserting that behavior.

## Solution Assessment

No for a Telegram-specific extractor. The maintainable solution is shared document extraction so Telegram, input_file/read flows, and future plugins reuse the same MIME support, limits, and untrusted-content boundaries.
