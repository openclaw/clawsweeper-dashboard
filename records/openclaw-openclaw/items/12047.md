## Summary

Keep open. Current main handles Telegram document download and shared text/PDF extraction, but Office-style document attachments are still excluded or skipped before extraction, and the related DOCX PR remains open with product/security questions.

## What This Changes

Request to extract readable text from Telegram document attachments and include it in agent context instead of only exposing a media placeholder.

## Best Possible Solution

Finish this through the shared document-extract/media pipeline with explicit Office MIME support, bounded extraction, preserved binary-safety checks, untrusted-content wrapping, Telegram/shared-media coverage, and public docs.

## Reproduction Assessment

Yes, from source inspection. Telegram forwards saved document media into shared context, but Office MIME types are outside the default file allowlist and are skipped as binary before extractor lookup, with regression coverage asserting that behavior.

## Solution Assessment

No for a Telegram-specific extractor or config-only toggle. The maintainable solution is shared document extraction so Telegram, input_file/read flows, and future plugins reuse the same MIME support, limits, and untrusted-content boundaries.
