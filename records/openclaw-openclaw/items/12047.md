## Summary

Keep open: current main and v2026.6.8 still download Telegram documents and can extract text/PDF through shared paths, but Office-style attachments are still filtered as binary and the bundled document extractor is still PDF-only. The related DOCX implementation PRs were closed unmerged, so the central requested capability remains unresolved.

## What This Changes

Request to extract readable text from Telegram document attachments and include it in agent-visible message context.

## Best Possible Solution

Finish this through the shared document-extractor/media path with maintainer-approved Office MIME support, bounded parser/resource limits, untrusted-content wrapping, and focused Telegram plus shared media/input_file coverage.

## Reproduction Assessment

Yes, source-reproducible. Current main shows Telegram documents reach MediaPaths/MediaTypes, while Office MIME types are excluded or skipped as binary before shared extraction; no live Telegram run was performed in this read-only review.

## Solution Assessment

No to a Telegram-only parser or toggle. The maintainable solution is to extend the shared document extraction path so Telegram, input_file, and other attachment surfaces share the same bounded document capability.
