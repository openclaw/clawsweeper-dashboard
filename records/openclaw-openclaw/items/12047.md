## Summary

Keep open. Current main and v2026.6.6 already download Telegram documents and extract text/PDF attachments through the shared media path, but Office-style documents are still skipped as binary before extraction and the document-extract plugin still advertises only PDF extraction.

## What This Changes

The issue requests extracting text from Telegram document attachments and adding that text to agent-visible message context instead of only passing a `<media:document>` placeholder.

## Best Possible Solution

Finish this through the shared document-extractor/media path, adding maintainer-approved Office MIME/extractor support with bounded archive/XML handling and regression coverage for Telegram plus shared input_file/media behavior.

## Reproduction Assessment

Yes. Source inspection shows Telegram document attachments reach the shared media context, while Office MIME types are treated as binary and skipped before document extraction; no live Telegram run was performed in this read-only review.

## Solution Assessment

No to a Telegram-specific parser or config toggle. The maintainable path is shared document-extractor support so Telegram, input_file, and other attachment surfaces share one bounded extraction contract.
