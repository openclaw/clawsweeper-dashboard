## Summary

Keep open: current main and v2026.6.8 already download Telegram documents and extract text/PDF through the shared path, but Office-style document attachments are still intentionally skipped as binary and the related DOCX implementation pull requests are closed unmerged.

## What This Changes

The issue requests extracting readable text from Telegram document attachments so agents can process files such as PDFs, text files, and Office documents instead of seeing only a media placeholder.

## Best Possible Solution

Finish this through the shared document-extractor and media-understanding path with maintainer-approved Office MIME scope, bounded parsers/resource limits, untrusted-content wrapping, and focused Telegram plus shared input_file coverage.

## Reproduction Assessment

Yes, at source level. Current main forwards Telegram document media to shared context, but Office MIME types are filtered as binary before extraction; I did not run a live Telegram bot reproduction in this read-only review.

## Solution Assessment

No to a Telegram-only parser or config toggle. The maintainable solution is shared document extraction so Telegram, input_file, and adjacent attachment surfaces use one bounded document capability.
