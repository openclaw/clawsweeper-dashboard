## Summary

Keep open. Current main downloads Telegram document attachments and extracts text/PDF content through the shared media path, but Office-style documents are still intentionally skipped before extraction and the closed DOCX PR did not land a safe replacement.

## What This Changes

Request text extraction for Telegram document attachments so agents can read files instead of receiving only a media placeholder.

## Best Possible Solution

Implement this through the shared document-extract/media-understanding path with maintainer-approved Office formats, parser ownership, resource limits, and focused Telegram plus shared file extraction coverage.

## Reproduction Assessment

Yes, at source level. Current main downloads Telegram documents into MediaPaths/MediaTypes, but Office MIME types are skipped as binary before extraction and no Office extractor exists.

## Solution Assessment

No for a Telegram-only parser or ad hoc config toggle. The maintainable solution is a shared document-extractor implementation with explicit format, dependency, safety, and resource-limit decisions.
