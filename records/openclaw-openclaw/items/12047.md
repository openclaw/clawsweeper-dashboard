## Summary

Keep open: current main downloads Telegram documents and can extract text/PDF content through shared paths, but Office-style attachments still get filtered as binary and the bundled document extractor is still PDF-only. The related DOCX implementation PRs were closed unmerged, so the central requested capability has not landed.

## What This Changes

The issue requests extracting readable text from Telegram document attachments so agents can process documents instead of seeing only media placeholders.

## Best Possible Solution

Finish this through the shared document-extractor/media path with maintainer-approved Office MIME support, bounded parser limits, untrusted-content wrapping, and focused Telegram plus shared media/input-file coverage.

## Reproduction Assessment

Yes. Current source shows Telegram document attachments reach `MediaPaths`/`MediaTypes`, while Office MIME types are treated as binary and skipped before shared extraction; no live Telegram run was performed in this read-only review.

## Solution Assessment

No to a Telegram-only parser or toggle. The maintainable solution is to extend the shared document extraction path so Telegram, input_file, and other attachment surfaces share the same bounded document capability.
