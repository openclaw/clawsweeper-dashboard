## Summary

Keep open. Current main handles Telegram document download plus shared text/PDF extraction, but Office formats are still filtered as binary before extraction, and the related DOCX PR remains open with untrusted ZIP/XML parsing and product-boundary questions.

## What This Changes

Request to extract text from Telegram document attachments and include it in agent context instead of only showing a media placeholder.

## Best Possible Solution

Finish this through the shared document extraction pipeline with bounded Office MIME/extractor support, preserved binary-safety checks, untrusted-content wrapping, Telegram/shared-media coverage, and docs for supported formats.

## Reproduction Assessment

Yes, from source inspection rather than a live Telegram upload. Telegram forwards saved document media into shared context, but Office MIME types are treated as binary and skipped before extractor lookup, with regression coverage asserting that skip.

## Solution Assessment

No for a Telegram-specific extractor. The maintainable path is a shared document extraction capability so Telegram, input_file, read/media paths, and future plugins use the same MIME support, limits, and untrusted-content boundaries.
