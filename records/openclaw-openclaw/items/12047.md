## Summary

Keep open. Current main downloads and forwards Telegram document media and can extract text/PDF-like files through the shared path, but Office-style document attachments are still excluded or skipped before extraction, and the related DOCX implementation remains open for maintainer/security review.

## What This Changes

The issue requests text extraction for Telegram document attachments so agents receive document contents instead of only media placeholders.

## Best Possible Solution

Finish the capability through the shared document-extract/media pipeline with explicit Office MIME support, bounded ZIP/XML extraction, preserved binary-safety checks, untrusted-content wrapping, Telegram/shared-media coverage, and public docs.

## Reproduction Assessment

Yes, from source inspection. Telegram forwards saved document media into shared context, but Office MIME types are excluded from default file support and skipped as binary before extractor lookup, with regression coverage asserting that skip.

## Solution Assessment

No for a Telegram-specific extractor or config-only toggle. The maintainable solution is shared document extraction so Telegram, input_file/read flows, and future plugins reuse the same MIME support, limits, and untrusted-content boundaries.
