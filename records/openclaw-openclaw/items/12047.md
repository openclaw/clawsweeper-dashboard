## Summary

Keep open: current main and v2026.6.10 still download and route Telegram document attachments into the shared media path, but Office-style documents are intentionally skipped as binary and the bundled extractor only advertises PDF support.

## What This Changes

Request to extract readable text from Telegram document attachments so agents see document contents instead of only placeholders.

## Best Possible Solution

Finish this through the shared document-extractor and media-understanding path with maintainer-approved Office MIME scope, bounded parsers/resource limits, untrusted-content wrapping, and focused Telegram plus shared input_file coverage.

## Reproduction Assessment

Yes, at source level: Telegram passes saved document paths and MIME types into shared media context, but Office MIME types are classified as binary and skipped before extraction. I did not run a live Telegram bot in this read-only review.

## Solution Assessment

No to a Telegram-only parser or config toggle. The maintainable solution is shared document extraction so Telegram, input_file, and adjacent attachment surfaces use one bounded document capability.
