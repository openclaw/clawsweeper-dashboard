## Summary

Keep open: current main and v2026.6.8 download Telegram documents and can extract text/PDF content through shared paths, but Office-style attachments are still skipped as binary and the bundled document-extract plugin remains PDF-only; the related DOCX implementation PRs are closed unmerged.

## What This Changes

Request text extraction for Telegram document attachments so document contents are included in agent-visible context instead of only a media placeholder.

## Best Possible Solution

Finish this through the shared document-extractor/media path with maintainer-approved bounded Office MIME support, parser/resource limits, untrusted-content wrapping, and focused Telegram plus shared media/input_file coverage.

## Reproduction Assessment

Yes, at source level. Current main forwards Telegram document media to shared context, but Office MIME types are filtered as binary before extractFileContentFromSource; I did not run a live Telegram bot reproduction in this read-only review.

## Solution Assessment

No to a Telegram-only parser or new config toggle as the durable fix. The maintainable solution is shared document extraction so Telegram, input_file, and adjacent attachment surfaces use one bounded document capability.
