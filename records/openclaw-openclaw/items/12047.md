## Summary

Keep open: current main and v2026.6.8 download Telegram documents and can extract text/PDF content through shared paths, but Office-style document attachments are still filtered as binary and the bundled document-extract plugin remains PDF-only; the related DOCX implementation PRs were closed unmerged.

## What This Changes

The issue requests Telegram document attachment text extraction so text-like files become agent-visible context instead of only a `<media:document>` placeholder.

## Best Possible Solution

Finish this through the shared document-extractor/media path with maintainer-approved Office MIME support, bounded parser limits, untrusted-content wrapping, and focused Telegram plus shared media/input-file coverage.

## Reproduction Assessment

Yes, at source level. Current main forwards Telegram document media to shared context, but Office MIME types are filtered as binary before `extractFileContentFromSource`; no live Telegram run was performed in this read-only review.

## Solution Assessment

No to a Telegram-only parser or config toggle. The maintainable solution is to extend the shared document extraction path so Telegram, input_file, and other attachment surfaces share the same bounded document capability.
