## Summary

Keep open: current main downloads Telegram documents and can extract text/PDF content through shared media paths, but Office-style attachments still get skipped as binary and the bundled document extractor only advertises PDF. The related DOCX implementation attempts were closed unmerged, so the central requested capability has not landed.

## What This Changes

Request text extraction from Telegram document attachments so document contents become agent-visible context instead of only `<media:document>` placeholders.

## Best Possible Solution

Finish this through the shared document extraction path with maintainer-approved Office MIME/extractor support, bounded parser resource limits, untrusted-content wrapping, and focused Telegram plus media/input-file coverage.

## Reproduction Assessment

Yes. Current source shows Telegram document attachments reach `MediaPaths`/`MediaTypes`, while Office MIME types are treated as binary and skipped before extraction; no live Telegram run was performed in this read-only review.

## Solution Assessment

No to a Telegram-only toggle or parser. The maintainable solution should extend the shared document-extractor/media path so Telegram, input_file, and other attachment surfaces use the same bounded document capability.
