## Summary

Keep open. Current main covers Telegram document download plus shared text/PDF extraction, but Office-style attachments are still blocked before extractor lookup and the related DOCX implementation remains open with security/product review concerns.

## What This Changes

Requests extracting text from Telegram document attachments and including the extracted content in agent context.

## Best Possible Solution

Finish this through the shared document-extract/media pipeline with explicit Office MIME support, bounded extraction, preserved binary-safety checks, untrusted-content wrapping, Telegram/shared-media coverage, and public docs.

## Reproduction Assessment

Yes, from source inspection. Telegram forwards saved document media into shared context, but Office MIME types are outside the default file allowlist and are skipped as binary before extractor lookup, with regression coverage asserting that behavior.

## Solution Assessment

No for a Telegram-specific extractor or config-only toggle. The maintainable path is shared document extraction so Telegram, input_file/read flows, and future plugins reuse the same MIME support, limits, and untrusted-content boundaries.
