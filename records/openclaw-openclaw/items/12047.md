## Summary

Keep open. Current main downloads Telegram documents and can extract text/PDF through shared media paths, but Office MIME types are still intentionally skipped as binary and the DOCX candidate PR remains open.

## What This Changes

Request to extract text from Telegram document attachments, including text/PDF/Office documents, so agents receive readable content instead of only `<media:document>` placeholders.

## Best Possible Solution

Finish this through the shared document-extraction pipeline with bounded Office MIME/extractor support, preserved binary-safety checks, untrusted-content wrapping, focused tests, and docs for supported formats.

## Reproduction Assessment

Yes, from source inspection rather than a live Telegram upload. A Telegram Office document is downloaded and forwarded as media context, then Office MIME types hit the binary skip path before extraction, with tests asserting that skip behavior.

## Solution Assessment

No for a Telegram-specific extractor. The maintainable path is shared document extraction so Telegram, input_file, and read/media flows use the same MIME support, limits, and untrusted-content boundaries.
