## Summary

Keep open. The completion-wake cache-stability bug is plausible from current source, but this PR head is stale against current prompt semantics, still misses direct and channel prompt context, and has no after-fix real behavior proof.

## What This Changes

The branch adds gateway `agent` synthesis and seam tests for `extraSystemPrompt` on inter-session `task_completion` wakes when callers omit it.

## Best Possible Solution

Refresh this PR or replace it with a narrow current-main fix that uses the same prompt builders normal chat uses for direct, group, and channel sessions, then prove the cache behavior with focused tests plus redacted runtime cache-trace or log evidence.

## Reproduction Assessment

Yes. Current source shows inter-session `task_completion` announce calls reach the gateway without `extraSystemPrompt`, while normal chat assembles richer direct/group/channel prompt context; a focused gateway seam test can reproduce the mismatch.

## Solution Assessment

No. The gateway seam is the right place to fix the completion-wake subset, but this branch does not yet match current direct/channel prompt semantics and its assertions preserve stale prompt bytes.
