## Summary

Keep open. The PR is useful and targets a real gateway prompt-cache divergence, but the current diff is not merge-ready against current main: wake prompt synthesis still omits the direct/dm chat context that normal chat turns include, and the added assertions expect stale inbound metadata and group-context text.

## What This Changes

The PR adds gateway `agent` prompt synthesis for inter-session `task_completion` wakes from persisted session delivery metadata, preserves explicit `extraSystemPrompt`, and adds gateway seam tests for synthesized, channel, metadata-free, and explicit prompt cases.

## Best Possible Solution

Refresh this PR onto current main and make completion-wake prompt synthesis reuse the same current session-derived prompt sections as ordinary chat turns: inbound metadata v2, direct-chat context for direct/dm sessions, group-chat context where normal chat includes it, explicit `extraSystemPrompt` pass-through, and no synthesized metadata for metadata-free sessions.
