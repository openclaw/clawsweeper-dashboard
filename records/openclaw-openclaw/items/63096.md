## Summary

Keep open. Current main still needs a completion-wake prompt-context fix, but this PR is not merge-ready because the synthesized prompt no longer matches current normal-chat prompt semantics and after-fix real behavior proof is missing.

## What This Changes

The branch adds gateway `agent` prompt synthesis for inter-session `task_completion` wakes from persisted session metadata, plus gateway seam tests for synthesized, explicit, channel, and metadata-free cases.

## Best Possible Solution

Refresh this PR or replace it with a narrow gateway fix that reuses current normal-chat prompt semantics for inbound metadata, direct/dm context, and group/channel context, then prove cache stability with focused tests plus redacted runtime/cache evidence.

## Reproduction Assessment

Yes. Source inspection shows current main forwards only `request.extraSystemPrompt` for inter-session `task_completion` wakes while normal chat assembles inbound, direct, and group/channel prompt context; a focused gateway seam test can exercise that path and inspect the forwarded prompt.

## Solution Assessment

No. The gateway seam is the right fix boundary, but this patch omits direct/dm and channel-as-group prompt context and locks in stale prompt bytes instead of matching the current builders.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

  - confidence: 0.9

  - confidence: 0.88

Current main emits `openclaw.inbound_meta.v2` without conversation ids, and group context now says `You are in a Discord group chat.`, so the tests will fail after a refresh or preserve stale prompt bytes.
  - confidence: 0.92
