## Summary

Keep open. Current main still lacks persisted completion-wake prompt synthesis, and this PR remains relevant, but the latest head is not mergeable as-is because it omits current direct/channel prompt context, asserts stale prompt bytes, and lacks after-fix real behavior proof.

## What This Changes

The PR adds gateway `agent` prompt synthesis for inter-session `task_completion` wakes from persisted session metadata, plus gateway seam tests for synthesized, explicit, channel, and metadata-free cases.

## Best Possible Solution

Refresh this PR or a narrow replacement onto current main, synthesize completion-wake prompt context from the same current builders normal chat uses, and prove prompt/cache stability with focused tests plus real cache-trace or terminal evidence.

## Reproduction Assessment

Yes. A focused gateway seam test can invoke `agent` with `inputProvenance.kind: "inter_session"`, a `task_completion` internal event, and persisted direct/dm plus channel session entries, then compare the forwarded prompt sections with current normal chat assembly.

## Solution Assessment

No. The narrow gateway seam is the right direction, but this PR does not yet match current prompt semantics for direct/dm and channel sessions and its assertions preserve stale trusted-metadata bytes.
