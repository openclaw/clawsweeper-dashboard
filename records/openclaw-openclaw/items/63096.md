## Summary

Keep open. Current main still has the missing inter-session completion-wake prompt synthesis, and open issue #63030 is still the canonical user problem, so this PR is useful source work. The latest PR head is not ready to merge because it is dirty against main and its implementation/tests no longer match current prompt assembly for direct, channel, and trusted metadata prompt bytes.

## What This Changes

The PR adds gateway `agent` prompt synthesis for inter-session `task_completion` wakes from persisted session metadata, preserves explicit `extraSystemPrompt`, and adds gateway seam tests for synthesized, channel, metadata-free, and explicit prompt cases.

## Best Possible Solution

Refresh this PR or a narrow replacement onto current main and make completion-wake prompt synthesis reuse the same current session-derived prompt sections as ordinary chat turns: inbound metadata v2, direct-chat context for direct/dm sessions, group-chat context for group/channel sessions, explicit `extraSystemPrompt` pass-through, and no synthesized metadata for metadata-free sessions. The tests should assert current builders or current prompt bytes and the user-facing fix should get a changelog entry.

## Reproduction Assessment

Yes. A focused gateway seam test can invoke `agent` with `inputProvenance.kind: "inter_session"`, a `task_completion` internal event, and persisted direct/dm plus channel session entries, then compare the forwarded `extraSystemPrompt` sections with current normal chat prompt assembly in `get-reply-run.ts`.

## Solution Assessment

No. The proposed direction is the right narrow gateway seam, but this PR is not the best current implementation because it omits direct/dm and channel prompt sections and its tests preserve stale trusted metadata and group-context expectations.
