## Summary

Keep open: the reported bug is still source-reproducible on current main, but this PR is not mergeable as-is because it misses current direct/channel prompt semantics, asserts stale prompt bytes, and lacks after-fix real behavior proof.

## What This Changes

The PR adds gateway agent prompt synthesis and seam tests for inter-session task-completion wakes when `extraSystemPrompt` is omitted.

## Best Possible Solution

Refresh this PR or replace it with a current-main fix that reuses normal prompt-context assembly for completion wakes and proves stable cache traces.

## Reproduction Assessment

Yes. Source inspection shows subagent announce sends `inter_session` `task_completion` runs without rebuilt `extraSystemPrompt`, while normal chat builds richer inbound/direct/group prompt context below the cache boundary.

## Solution Assessment

No. The current PR manually recreates only part of normal chat prompt assembly; the maintainable fix should align or share that assembly so direct, group, channel, and current inbound metadata bytes all match current main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Synthesize direct chat context for wake runs:** `src/gateway/server-methods/agent.ts:256`
  - body: Normal direct and DM chat turns append `buildDirectChatContext` to `extraSystemPrompt`, but the new wake builder only joins inbound metadata and group context. Direct completion wakes would still produce different system-prompt bytes from normal direct sessions, so the cache-stability fix misses that path.
  - confidence: 0.9
- **[P2] Treat channel sessions like normal chat turns:** `src/gateway/server-methods/agent.ts:253-254`
  - body: Current main treats `chatType: "channel"` as group-like for prompt assembly and includes `buildGroupChatContext` on normal turns. This helper only builds group context for exact `"group"`, so channel completion wakes still diverge from ordinary channel turns.
  - confidence: 0.88
- **[P2] Update assertions for current prompt bytes:** `src/gateway/server-methods/agent.test.ts:926-932`
  - body: These assertions still expect `chat_id` in trusted inbound metadata and a subject-specific group line. Current main emits `openclaw.inbound_meta.v2` without conversation ids, and group context now says `You are in a Discord group chat.`, so the tests will fail after a refresh or push the implementation back toward stale prompt bytes.
  - confidence: 0.92
