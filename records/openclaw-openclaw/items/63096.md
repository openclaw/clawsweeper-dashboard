## Summary

Keep open. The branch targets a real gateway prompt-cache seam, but the current head is conflicted, still diverges from current normal-chat prompt shaping for direct and channel sessions, contains stale prompt-byte assertions, and lacks after-fix real behavior proof.

## What This Changes

The PR adds gateway agent logic and seam tests to synthesize persisted inbound/group extraSystemPrompt context for inter-session task_completion wakes when no explicit prompt is supplied.

## Best Possible Solution

Refresh this PR or replace it with a current-main fix that derives completion-wake prompt context from the same normal-chat builders for direct, group, and channel sessions, then prove cache stability with focused tests plus redacted runtime cache-trace or log evidence.

## Reproduction Assessment

Yes. Current source shows inter-session task_completion announce calls reach the gateway without extraSystemPrompt while normal chat assembles inbound/direct/group context; a focused gateway seam test can reproduce the mismatch without a live Anthropic run.

## Solution Assessment

No. The gateway seam is a plausible fix point, but this branch does not yet match current direct/channel prompt semantics and its assertions preserve stale prompt bytes.
