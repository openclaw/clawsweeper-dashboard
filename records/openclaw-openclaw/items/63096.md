## Summary

Keep open. The completion-wake cache bug is source-reproducible and current main still lacks the requested synthesis, but this PR is not merge-ready because it misses current direct/channel prompt semantics, asserts stale prompt bytes, and lacks after-fix real behavior proof.

## What This Changes

The branch adds gateway agent logic and seam tests to rebuild persisted inbound/group prompt context for inter-session task-completion wakes when `extraSystemPrompt` is omitted.

## Best Possible Solution

Refresh or replace this PR with a current-main implementation that shares normal chat prompt-context assembly for completion wakes, covers direct/group/channel sessions in tests, and includes redacted cache trace or log proof of stable system prompt behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows subagent completion announces reach the gateway with inter-session provenance and internal events but no rebuilt `extraSystemPrompt`, while normal chat assembles inbound/direct/group context.

## Solution Assessment

No. The gateway seam is plausible, but this branch does not yet match current direct/channel prompt semantics or current inbound/group prompt bytes, so it needs a refresh before it is the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Include direct-chat context in wake synthesis:** `src/gateway/server-methods/agent.ts:256`
  - body: Normal direct/dm chat turns append `buildDirectChatContext` to `extraSystemPrompt`, but this helper only joins inbound metadata and group context. Direct completion wakes would still produce different system-prompt bytes from normal direct sessions, so the cache-stability fix misses that path.
  - confidence: 0.9

  - body: Current main treats `chatType: "channel"` as group-like for prompt assembly and includes `buildGroupChatContext` on normal turns. This helper only builds group context for exact `"group"`, so channel completion wakes still diverge from ordinary channel turns.
  - confidence: 0.88
- **[P2] Refresh assertions for current prompt bytes:** `src/gateway/server-methods/agent.test.ts:926-932`
  - body: These assertions still expect `chat_id` in trusted inbound metadata and a subject-specific group line. Current main emits `openclaw.inbound_meta.v2` without conversation ids, and group context now says `You are in a Discord group chat.`, so the tests will fail after a refresh or push the implementation back toward stale prompt bytes.
  - confidence: 0.92
