## Best Possible Solution

## Reproduction Assessment

Yes, at source level. Current main sends inter-session completion wakes through the gateway with only caller-provided `extraSystemPrompt`, while normal chat assembles richer inbound/direct/group prompt context below the cache boundary; I did not run a live Anthropic cache trace.

## Solution Assessment

No. The PR manually reconstructs a subset of prompt assembly, but the maintainable fix should share or faithfully mirror current normal assembly so direct, channel, group, and delivery-mode prompt bytes stay aligned.
