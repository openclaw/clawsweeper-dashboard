## Summary

Keep open. This PR targets a real gateway prompt-cache bug, but the branch still diverges from current main prompt semantics and lacks after-fix real behavior proof.

## What This Changes

The PR adds gateway agent logic and seam tests to synthesize persisted session prompt context for inter-session task-completion wakes when no explicit extra system prompt is supplied.

## Best Possible Solution

Refresh this PR or replace it with a current-main fix that derives completion-wake prompt context from the same normal-chat builders for direct, group, and channel sessions, then prove cache stability with focused tests plus redacted runtime cache-trace or log evidence.

## Reproduction Assessment

Yes, from source inspection: inter-session task-completion announce calls still reach the gateway without rebuilt extraSystemPrompt, while normal chat assembles richer inbound/direct/group context. A focused gateway seam test can reproduce the mismatch without a live Anthropic run.

## Solution Assessment

No. The gateway seam is plausible, but this branch does not yet match current direct/channel prompt semantics or current inbound/group prompt bytes.
