## What This Changes

The PR adds a gateway helper and seam tests to synthesize session-derived `extraSystemPrompt` for inter-session `task_completion` wakes when no explicit prompt is supplied.

## Best Possible Solution

Refresh or replace this with a current-main fix that derives completion-wake prompt context from the same normal prompt-context semantics, covers direct/channel/group and source-reply delivery modes, and includes redacted cache-trace proof.

## Reproduction Assessment

Yes, at source level. Current main sends inter-session completion wakes through the gateway without rebuilding `extraSystemPrompt`, while normal chat assembles richer inbound/direct/group prompt context below the cache boundary.

## Solution Assessment

No. The PR manually reconstructs only part of the normal prompt assembly; the maintainable fix should share or faithfully mirror the current normal assembly so direct, channel, group, and delivery-mode prompt bytes stay aligned.
