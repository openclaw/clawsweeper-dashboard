## Summary

Keep open. Current main and the latest release still render raw parsed config in `/config show` replies, while this PR is a narrow security fix that still needs real behavior proof and a small test-hint cleanup before maintainer handling.

## What This Changes

The PR redacts full and path-specific `/config show` chat replies with the existing config redaction helper and adds regression tests.

## Best Possible Solution

Land this PR or an equivalent narrow patch after adding real behavior proof and tightening the regression tests, then close the linked issue through the normal security release process.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure a sensitive value such as `gateway.auth.token`, then invoke `/config show` or `/config show gateway.auth.token`; current main stringifies the raw parsed config into the reply.

## Solution Assessment

Yes for the production approach. Reusing `readBestEffortRuntimeConfigSchema()` plus `redactConfigObject()` before both path lookup and full JSON rendering is the narrowest maintainable fix, but the tests should mark mocked hints `sensitive: true` or cover a non-heuristic sensitive field.

Overall correctness: patch is correct
