## Summary

Keep open. Current main and the latest release still render raw `/config show` data, while this PR contains a plausible narrow production fix but still needs real behavior proof, a small test-quality fix, and maintainer security handling before merge.

## What This Changes

The PR redacts full and path-specific `/config show` replies with the existing schema-aware config redaction helper and adds regression tests for secret redaction.

## Best Possible Solution

Land this PR or an equivalent narrow patch after adding real behavior proof and tightening the regression tests, then close https://github.com/openclaw/openclaw/issues/65623 through the normal security release process.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure a sensitive value such as `gateway.auth.token`, then invoke `/config show` or `/config show gateway.auth.token`; current main stringifies the raw parsed config into the reply.

## Solution Assessment

Yes for the production approach. Reusing `readBestEffortRuntimeConfigSchema()` plus `redactConfigObject()` before both path lookup and full JSON rendering is the narrowest maintainable fix, but the tests should mark mocked hints `sensitive: true` or cover a non-heuristic sensitive field.

## Review Findings

Overall correctness: patch is correct

Overall confidence: 0.87

Full review comments:

- **[P3] Mark mocked redaction hints as sensitive:** `src/auto-reply/reply/commands-gating.test.ts:328-330`
