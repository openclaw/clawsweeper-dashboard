## Summary

Keep this PR open. Current main and the latest release still serialize raw `snapshot.parsed` for full and path-specific `/config show` replies, while this PR is the focused security-positive implementation for linked bug #65623. The production approach matches the existing gateway config redaction path, but the new tests should be tightened so they actually exercise schema-hint redaction rather than heuristic fallback.

## What This Changes

This PR redacts the cloned parsed config before rendering full or path-specific `/config show` chat replies and adds command-gating tests for secret redaction.

## Best Possible Solution

Land this PR or an equivalent narrow patch after tightening the regression tests to use `sensitive: true` hints or a non-heuristic sensitive field, then close linked bug #65623 as fixed with normal security release follow-up.

## Reproduction Assessment

Yes. Current main has a source-level reproduction: configure a sensitive value such as `gateway.auth.token`, then invoke owner `/config show` or `/config show gateway.auth.token`; the handler stringifies `snapshot.parsed` without redaction.

## Solution Assessment

Yes for the production direction. Reusing `readBestEffortRuntimeConfigSchema()` plus `redactConfigObject()` before rendering is the narrowest maintainable fix because it matches the existing gateway config API redaction contract; only the test setup needs correction.
