## Summary

Keep open. The PR is a narrow and useful security fix for a current-main secret disclosure path, but the external PR still lacks after-fix real behavior proof and the new regression tests should exercise schema-hint redaction instead of only heuristic redaction.

## What This Changes

The PR applies schema-aware config redaction before rendering full and path-scoped `/config show` replies and adds regression tests for redacted output.

## Best Possible Solution

Merge this PR or an equivalent narrow fix after the contributor adds redacted real behavior proof and the tests mark mocked schema hints sensitive, then close the linked security issue through the normal release process.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure a sensitive value such as `gateway.auth.token`, then invoke `/config show` or `/config show gateway.auth.token`; current main stringifies the raw parsed config into the chat reply.

## Solution Assessment

Yes for the production change. Reusing `readBestEffortRuntimeConfigSchema()` plus `redactConfigObject()` before both lookup and full rendering is the narrowest maintainable fix, but the tests should mark mocked hints `sensitive: true` or use a non-heuristic sensitive field.
