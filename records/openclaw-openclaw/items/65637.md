## What This Changes

The PR applies existing schema-aware config redaction before rendering full or path-scoped `/config show` replies and adds regression tests for redacted output.

## Best Possible Solution

Merge this PR or an equivalent narrow fix after real redacted behavior proof, a conflict refresh, and the schema-hint test correction, then close the linked security issue through the normal security release process.

## Reproduction Assessment

Yes. Source inspection on current main shows `/config show` clones `snapshot.parsed`, then full and path-scoped show responses stringify raw values from that clone without redaction.

## Solution Assessment

Yes for the production change. Reusing `readBestEffortRuntimeConfigSchema()` and `redactConfigObject()` before both lookup and rendering is the narrowest maintainable fix, but the tests should mark mocked hints as sensitive and proof is still needed.
