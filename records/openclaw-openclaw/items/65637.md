## Summary

Keep open: current main still renders raw `/config show` output, and this PR is a narrow, useful security fix. It still needs contributor real behavior proof before merge, and the added tests should exercise schema-hint redaction rather than only heuristic redaction.

## What This Changes

The PR applies config redaction before full and path-scoped `/config show` rendering and adds command-gating regression tests.

## Best Possible Solution

Land this PR or an equivalent narrow patch that redacts before both full and path-scoped `/config show` rendering, adds schema-hint regression coverage, and follows the security release process.

## Reproduction Assessment

Yes. Source inspection on current main shows `handleConfigCommand` clones `snapshot.parsed` and stringifies either the full object or a selected path without calling the config redaction helper.

## Solution Assessment

Yes for the production change. Reusing `readBestEffortRuntimeConfigSchema()` and `redactConfigObject()` before lookup and rendering matches the existing gateway `config.get` redaction contract, but the tests should mark mocked hints as sensitive or use a non-heuristic sensitive field.
