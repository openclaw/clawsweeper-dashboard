## What This Changes

The PR redacts full and path-specific `/config show` replies with `redactConfigObject` and runtime schema hints, plus command-gating regression tests.

## Best Possible Solution

Land this PR or an equivalent narrow maintainer-approved patch after rebasing, adding redacted real behavior proof, tightening the schema-hint test, and following the security release process.

## Reproduction Assessment

Yes. Source inspection on current main shows `handleConfigCommand` clones `snapshot.parsed` and stringifies either the full object or a selected path without applying config redaction.

## Solution Assessment

Yes for the production change. Reusing `readBestEffortRuntimeConfigSchema()` and `redactConfigObject()` before lookup and rendering matches the existing gateway `config.get` redaction contract, but the tests should mark mocked hints as sensitive or use a non-heuristic sensitive field.
