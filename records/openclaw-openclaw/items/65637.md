## Summary

Keep open. Current main and the latest tagged release still render `/config show` from the raw parsed config, while this PR applies the existing redaction path and is directionally correct. The only concrete issue found is test coverage: the new mocked schema hints omit `sensitive: true`, so they do not exercise the schema-hint redaction path described by the PR.

## What This Changes

The PR redacts full and path-specific `/config show` replies with the existing schema-aware config redaction helper and adds regression tests for secret redaction.

## Best Possible Solution

Land this PR or an equivalent narrow patch after tightening the regression tests to cover `sensitive: true` schema hints or a non-heuristic sensitive field, then close linked bug #65623 through the normal security release process.

## Reproduction Assessment

Yes. Current main has a high-confidence source-level reproduction: configure a sensitive value such as `gateway.auth.token`, then invoke owner or internal `/config show` or `/config show gateway.auth.token`; the handler serializes `snapshot.parsed` through `parsedBase` without redaction.

## Solution Assessment

Yes for the production approach. Reusing `readBestEffortRuntimeConfigSchema()` with `redactConfigObject()` before rendering is the narrowest maintainable fix because it matches the existing gateway `config.get` redaction contract; the safer test path is to mark mocked hints `sensitive: true` or use a non-heuristic sensitive field.
