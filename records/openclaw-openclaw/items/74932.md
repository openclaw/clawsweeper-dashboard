## Summary

Keep open. The PR targets the right heartbeat model-selection path, but the proposed stale detector still misses persisted auto-fallback overrides after fallback notice state has been cleared, so the linked bug remains reproducible for long-lived sessions.

## What This Changes

The PR adds heartbeat-only stale auto-fallback override detection and clearing, threads heartbeat context through reply model selection, and updates focused tests, model-failover docs, and the changelog.

## Best Possible Solution

Persist durable auto-fallback origin metadata, or an equivalent changed-default marker, when the auto override is written so heartbeat can clear only stale direct auto overrides after the configured default changes.

## Reproduction Assessment

Yes. Seed a direct session override with providerOverride/modelOverride/modelOverrideSource="auto" but no fallbackNoticeSelectedModel, then run heartbeat model selection after changing agents.defaults.model.primary; the PR keeps the old fallback selected.

## Solution Assessment

No. The PR changes the right surface, but fallback notice fields are transient display state rather than durable origin metadata, so the safer fix needs durable origin tracking or a conservative equivalent.
