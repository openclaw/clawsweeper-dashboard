## Summary

This PR should stay open because the stale-override detector still depends on transient fallback notice fields that current runtime can clear while leaving the auto override persisted, so the linked heartbeat pinning bug remains reachable for long-lived sessions.

## What This Changes

The branch adds heartbeat-only stale auto-fallback override detection and clearing, focused regression tests, model-failover docs, a changelog entry, and small test helper adjustments.

## Best Possible Solution

Persist durable metadata for the configured model that produced an auto fallback override, then clear only direct heartbeat auto overrides whose recorded origin differs from the current configured default.

## Reproduction Assessment

Yes, source-reproducible. Seed a heartbeat session with providerOverride/modelOverride/modelOverrideSource="auto" but no fallbackNoticeSelectedModel after normal fallback notice clearing, then change agents.defaults.model.primary; the PR detector preserves the stale override.

## Solution Assessment

No. The PR changes the right model-selection surface, but fallbackNoticeSelectedModel is not durable enough to decide whether an auto override was created for the current default.
