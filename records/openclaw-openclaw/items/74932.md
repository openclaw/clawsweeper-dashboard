## Summary

Keep this PR open: it targets the right heartbeat model-selection path, but the patch still misses persisted auto fallback overrides after the fallback notice metadata has been cleared, so the linked heartbeat pinning bug remains reproducible for long-lived sessions.

## What This Changes

The PR adds heartbeat-only stale auto-fallback override detection in reply model selection, with focused tests, model-failover docs, and a changelog entry.

## Best Possible Solution

Persist durable auto-fallback origin/default metadata, or an equivalent changed-default marker, and use it to clear stale direct heartbeat auto overrides while preserving user, legacy, parent, and unchanged-default behavior.

## Reproduction Assessment

Yes. Seed a direct session override with providerOverride/modelOverride/modelOverrideSource="auto" but no fallbackNoticeSelectedModel, then run heartbeat model selection after changing agents.defaults.model.primary; the PR keeps the old fallback selected.

## Solution Assessment

No. The PR changes the right surface, but fallback notice fields are transient display state, so the safer fix needs durable origin tracking or an explicit compatibility policy for missing metadata.
