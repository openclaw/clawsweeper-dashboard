## Summary

This PR still needs changes before merge: it targets the right heartbeat model-selection path, but the stale override detector depends on transient fallback notice fields that current runtime can clear while leaving the auto model override in place.

## What This Changes

The PR adds heartbeat-only stale auto-fallback override clearing, focused model-selection tests, model-failover docs, and a changelog entry for heartbeat runs following updated default models.

## Best Possible Solution

Persist durable auto-fallback origin metadata and clear only direct heartbeat auto overrides whose recorded origin differs from the current configured default, while preserving user, legacy, parent, and unchanged-default behavior.
