## Summary

Keep open: the PR targets the right heartbeat model-selection bug, but the stale detector still depends on transient fallback notice fields that current main can clear and persist before the default model changes.

## What This Changes

The PR adds heartbeat-aware stale direct auto-fallback override clearing, focused regression coverage, model-failover docs, and a changelog entry.

## Best Possible Solution

Persist durable auto-fallback origin metadata and clear only direct heartbeat auto overrides whose recorded origin differs from the current configured default, while preserving user, legacy, parent, explicit heartbeat, and unchanged-default behavior.

## Reproduction Assessment

Yes. A high-confidence source reproduction is a heartbeat session with a direct `modelOverrideSource: "auto"` override after fallback notice fields have been cleared, followed by an `agents.defaults.model.primary` change; the PR detector returns false and the old override remains eligible.

## Solution Assessment

No. The PR changes the right surface, but transient fallback notice state is not durable enough to decide whether an auto-fallback override came from an older configured default.
