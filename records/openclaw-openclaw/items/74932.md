## Summary

Keep open: this PR targets the right heartbeat model-selection bug, but the latest head still preserves stale auto overrides when fallback notice metadata has been cleared, and the external real-behavior proof gate is not satisfied.

## What This Changes

The branch adds heartbeat-only stale auto-fallback override detection/clearing in reply model selection, plus focused tests, model-failover docs, and a changelog entry.

## Best Possible Solution

Persist durable auto-fallback origin metadata, or an equivalent changed-default marker, when the auto override is written so heartbeat can clear only stale direct auto overrides after the configured default changes.

## Reproduction Assessment

Yes, source-reproducible. Seed a heartbeat session with providerOverride/modelOverride/modelOverrideSource="auto" but no fallbackNoticeSelectedModel, then change agents.defaults.model.primary and run heartbeat model selection; the PR keeps the old fallback selected.

## Solution Assessment

No. The PR changes the right surface, but it relies on fallback notice fields that current runtime clears and persists, so durable origin metadata or a conservative compatibility marker is safer.
