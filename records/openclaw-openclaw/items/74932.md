## Summary

Keep open. The PR targets a valid heartbeat model-selection bug, but the patch still misses the notice-cleared path because stale detection depends on transient fallback notice metadata that current runtime can clear while leaving the auto override persisted.

## What This Changes

The PR adds heartbeat-aware stale auto-fallback override detection, model-selection regression tests, model-failover docs, and a changelog entry.

## Best Possible Solution

Persist durable auto-fallback origin metadata when writing fallback-owned selection, then clear only direct heartbeat auto overrides whose recorded origin differs from the current configured default.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a heartbeat session with modelOverrideSource "auto" but no fallbackNoticeSelectedModel after notice clearing, followed by a default-model change, makes the PR helper return false and keep the stale override.

## Solution Assessment

No. The PR changes the right seam, but using fallback notice fields as the origin record is not durable enough; the fix should store fallback-origin metadata separately from display notice lifecycle.
