## Summary

Keep open: this PR targets the right heartbeat model-selection path and has proof plus green checks, but the patch still misses auto fallback overrides after the fallback notice metadata has already been cleared.

## What This Changes

The PR adds heartbeat-only stale auto-fallback override detection and clearing based on fallback notice metadata, with focused tests, model-failover docs, and a changelog entry.

## Best Possible Solution

Persist durable origin metadata for auto fallback overrides and clear only direct heartbeat auto overrides whose recorded origin differs from the current configured default.

## Reproduction Assessment

Yes, source-reproducible. Seed a heartbeat session with a direct `modelOverrideSource: "auto"` override after fallback notice fields have been cleared, then change `agents.defaults.model.primary`; the PR's helper returns false and preserves the stale override.

## Solution Assessment

No. The PR changes the right surface, but `fallbackNoticeSelectedModel` is notice state that current runtime clears, so durable auto-fallback origin metadata is the safer fix.
