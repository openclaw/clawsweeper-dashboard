## Summary

Keep open: the PR targets the right model-selection surface and includes real terminal proof, but the stale check still depends on transient fallback notice metadata that current runtime can clear while leaving the auto override in place.

## What This Changes

The PR adds heartbeat-aware stale auto-fallback override detection, model-selection regression tests, model-failover docs, and a changelog entry.

## Best Possible Solution

Persist durable auto-fallback origin metadata when writing fallback-owned selection, then clear only direct heartbeat auto overrides whose recorded origin differs from the current configured default.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a heartbeat session with `modelOverrideSource: "auto"` but no `fallbackNoticeSelectedModel`, followed by a default-model change, makes the PR helper return false and keep the stale override.

## Solution Assessment

No. The PR changes the right area, but using transient fallback notice fields as the origin record is not durable enough; the fix should store fallback-origin metadata separately from display notice lifecycle.
