## Summary

Keep open; this is a plausible bug-fix PR with sufficient terminal proof, but the latest diff still depends on transient fallback notice metadata and misses the persisted state after those notices are cleared.

## What This Changes

The PR adds heartbeat-aware stale auto-fallback override detection, regression tests, model-failover docs, and a changelog entry.

## Best Possible Solution

Persist durable auto-fallback origin metadata when writing an auto fallback selection, then clear only direct heartbeat auto overrides whose recorded origin differs from the current configured default while preserving explicit heartbeat, user, legacy, parent, and unchanged-default behavior.

## Reproduction Assessment

Yes, from source inspection. A heartbeat session with a direct `modelOverrideSource: "auto"` override whose fallback notice fields were cleared by a later non-fallback run, followed by an `agents.defaults.model.primary` change, makes the added detector return false and keeps the old override eligible.

## Solution Assessment

No. The PR changes the right selection surface, but transient fallback notice state is not durable enough to decide whether an auto-fallback override came from an older configured default.
