## Summary

Keep open: current main does not already implement the approval audit-log RPC, and the PR branch still has concrete access-control, secret-redaction, availability, protocol-artifact, and changelog blockers before it should merge.

## What This Changes

The branch adds a JSONL exec-approval decision audit helper, appends approval resolution records from the gateway handler, registers `exec.approval.auditLog`, and exports a validator for audit-log query params.

## Best Possible Solution

Keep the feature direction, but revise the source branch so approval history is guarded by the existing approval/admin boundary, persisted argv uses the shared tool/log redaction coverage, audit reads are async and bounded, protocol registry/generated clients/docs are synchronized, and the user-facing feature has tests plus a changelog entry.

## Reproduction Assessment

Yes. The blockers are reproducible from the PR head by inspecting `exec.approval.auditLog` scope registration, the new argv redactor, and the synchronous audit read path; focused tests can assert read-scope access denial, argv redaction for `--token secret`/Bearer/lowercase forms, and bounded async reads.

## Solution Assessment

No. Persisting approval decisions is a reasonable additive gateway feature, but this implementation is not the narrowest maintainable solution until it reuses the established approval scope and shared redaction contracts and avoids synchronous full-file reads on the gateway request path.

- **[P1] Gate audit-log reads with approval scope:** `src/gateway/method-scopes.ts:109`
  - body: `exec.approval.auditLog` is registered under `operator.read`, while live exec approval methods require `operator.approvals`. The returned history includes command text, args, agent/session IDs, and resolver identity, so read-only operator tokens would gain access to approval data they cannot see live.
  - confidence: 0.91

  - body: The array redactor only masks uppercase env-var style `KEY=value` entries. Common argv forms such as `--token secret`, `--password=secret`, lowercase `token=secret`, bearer headers, and raw token-shaped values can still be written to `approvals.jsonl`; reuse the existing tool/log redaction helper or equivalent coverage before persisting argv.
  - confidence: 0.88

  - body: `readFileSync` loads and parses the complete append-only audit file before applying `limit`. A long-lived gateway can accumulate a large file, and one `exec.approval.auditLog` request would block the Node event loop; use an async bounded or streaming read path instead.
  - confidence: 0.83
- **[P2] Register audit params in protocol artifacts:** `src/gateway/protocol/schema/exec-approvals.ts:156-163`
  - body: Adding `ExecApprovalAuditLogParamsSchema` here is not enough for a public gateway RPC. The new params/result surface is missing from the protocol schema registry, generated schema types, generated Swift models, and protocol docs, so generated clients will not learn the new contract.
  - confidence: 0.79
