## Summary

Keep open. The PR is active and useful in direction, but the current branch still introduces concrete access-control, secret-redaction, availability, and protocol-contract blockers on a security-sensitive approval audit surface.

## What This Changes

The branch adds persisted exec approval decision JSONL auditing, wires a new `exec.approval.auditLog` RPC with filters, and exports a matching gateway protocol params schema.

## Best Possible Solution

Keep the audit-log feature direction, but revise this branch so historical approval reads use the approval/admin access boundary, persisted command and argv fields reuse the shared tool redaction coverage, audit reads are async and bounded or streaming, and protocol docs plus generated artifacts are updated in one change.

## Reproduction Assessment

Yes. A focused unit test can show the PR would allow `exec.approval.auditLog` with `operator.read`, and direct redaction cases such as `--token secret`, `token=secret`, and bearer-style argv entries demonstrate the custom array redactor gap. I did not run tests because this review is read-only.

## Solution Assessment

No. Persisting approval decisions is a reasonable additive gateway feature, but this implementation is not the best merge path until the access boundary, persisted redaction, read strategy, and protocol contract sync are corrected.

## Review Findings

Full review comments:

- **[P1] Gate audit log reads with approval scope:** `src/gateway/method-scopes.ts:110`
  - body: This registers `exec.approval.auditLog` under `operator.read`, while existing approval RPCs and approval broadcasts require `operator.approvals`. The audit rows include command text, args, agent/session ids, and resolver identity, so read-only operator tokens would gain historical approval data they cannot see live.
  - confidence: 0.91
- **[P1] Reuse existing tool redaction for persisted argv:** `src/infra/exec-approval-audit.ts:35-40`
  - body: The new array redactor only masks uppercase env-var-style `KEY=value` entries. Common argv forms such as `--token secret`, `--password=secret`, lower-case `token=secret`, and bearer-header arguments can still be written to `approvals.jsonl`; use the existing tool redaction coverage or equivalent before persistence.
  - confidence: 0.88
- **[P2] Avoid blocking the gateway on audit reads:** `src/infra/exec-approval-audit.ts:85`
  - body: `readFileSync` loads and parses the entire append-only audit file before applying `limit`. A long-running gateway can accumulate a large file, making one audit-log RPC block the event loop; use an async bounded or streaming read path.
  - confidence: 0.84
- **[P2] Sync the audit RPC with protocol artifacts:** `src/gateway/protocol/schema/exec-approvals.ts:153-161`
  - body: Adding `ExecApprovalAuditLogParamsSchema` here does not update the public protocol definitions, generated schema types, Swift models, or protocol docs. Generated clients will not learn the new RPC contract unless those artifacts are updated with the schema change.
  - confidence: 0.79
