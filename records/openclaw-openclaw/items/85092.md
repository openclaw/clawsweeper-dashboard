## What This Changes

The issue requests that `openclaw infer model auth logout --provider <p>` persist profile removal to the agent selected by `OPENCLAW_AGENT_DIR` or fail instead of reporting a no-op success.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/85129 or an equivalent fix that resolves the scoped agent directory before logout, computes `removedProfiles` from the locked store being mutated, and covers scoped/default store isolation.

## Reproduction Assessment

Yes. The issue gives concrete env-scoped logout steps, and current main source shows the infer logout path resolving the configured default agent dir while runtime profile listing can include inherited profiles.

## Solution Assessment

Yes. The linked PR direction is the narrow maintainable fix: select the env-scoped agent dir first, mutate the locked local store, and report only profiles actually removed from that store.
