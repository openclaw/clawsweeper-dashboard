## Summary

Keep open: current `main` and the latest release still have a credible scoped-auth-store bug in the infer CLI logout path. The command still resolves the default agent directory for logout, while the documented models status path honors `OPENCLAW_AGENT_DIR` when no `--agent` is supplied.

## What This Changes

The issue reports that `openclaw infer model auth logout --provider` returns a removed profile for a non-default `OPENCLAW_AGENT_DIR` but does not persist the change to that agent's auth store.

## Best Possible Solution

Make the infer CLI auth logout resolve the same scoped agent directory that the operator targeted, remove profiles from the correct owner store, and report only profiles actually removed or fail clearly.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: set `OPENCLAW_AGENT_DIR` for a secondary agent and run `openclaw infer model auth logout --provider <id>`; this read-only review did not run the disk-writing command.

## Solution Assessment

No. The current implementation is not the best solution because it derives the write target from the configured default agent and can treat a no-save update as success; the narrow maintainable fix is an owner-aware scoped logout path with regression coverage.
