## Summary

Keep open: current main still resolves `openclaw infer model auth logout` through the configured default agent directory, so the scoped-agent logout bug remains source-reproducible, and the linked fixing PR is still open.

## What This Changes

The issue reports that env-scoped infer model auth logout claims to remove a provider profile from a secondary agent but leaves that agent's `auth-profiles.json` unchanged.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/85129 or an equivalent narrow fix that resolves the env-scoped agent directory before logout and reports only profiles actually removed from the locked store.

## Reproduction Assessment

Yes. I did not run a live logout in this read-only review, but the issue provides concrete env-scoped steps and current main source shows the mismatched default-agent logout path.

## Solution Assessment

Yes. The linked PR direction is the narrow maintainable fix: select the env-scoped agent dir first, mutate the locked local store, and compute `removedProfiles` from that same store.
