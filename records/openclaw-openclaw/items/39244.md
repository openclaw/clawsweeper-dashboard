Overall confidence: 0.91

Full review comments:

- **[P1] Add nsSandbox to the strict config schema:** `src/config/types.tools.ts:253-258`
  - body: This adds `tools.exec.nsSandbox` to the public exec config type, but the strict runtime schema still omits the key. Users setting the documented config would get an unrecognized-key validation error instead of enabling bwrap.
  - confidence: 0.94
- **[P1] Mount the planned safe-bin executable paths:** `src/infra/exec-bwrap-sandbox.ts:232-235`
  - body: `buildBwrapArgs()` re-resolves configured safe-bin names from trusted dirs, while enforced execution renders each segment's resolved executable path. On symlink or `/bin` versus `/usr/bin` layouts, bwrap can mount one path while the command executes another, causing sandboxed safeBins to fail.
  - confidence: 0.91
- **[P1] Mount the exact selected shell path:** `src/agents/bash-tools.exec-host-gateway.ts:207-218`
  - body: `makeBwrapParams()` strips a full shell path down to a basename, but runtime execution appends the full selected shell path after `--`. Shells such as `/usr/local/bin/bash` or Nix shells can be absent inside the namespace and fail before the command starts.
  - confidence: 0.9
- **[P2] Reject broad binds that contain sensitive paths:** `src/infra/exec-bwrap-sandbox.ts:378-380`
  - body: `isDeniedBindPath()` only rejects paths inside the denied path, so an extra bind for the user's home directory can still bring `.ssh` or `.gnupg` into the sandbox. Reject parent binds that contain denied paths too.
  - confidence: 0.82
- **[P2] Preserve network semantics for custom safe bins:** `src/infra/exec-bwrap-sandbox.ts:304-313`
  - body: The gateway always supplies `commandBins`, and `shouldShareNetwork()` disables networking whenever none of those names are hardcoded. Any custom profiled safeBin that legitimately needs network but is not listed will silently lose network under bwrap.
  - confidence: 0.78
- **[P2] Split out lifecycle-script allowlist changes:** `package.json:435-439`
  - body: The package diff changes install-time lifecycle-script policy, which is unrelated to safeBins bwrap isolation. Drop this from the branch or split it into a separately reviewed package-policy change.
  - confidence: 0.88
