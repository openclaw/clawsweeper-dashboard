## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Mount the exact planned executable paths:** `src/infra/exec-bwrap-sandbox.ts:232-235`
  - body: `buildBwrapArgs()` re-resolves each configured safe-bin name from trusted dirs, but enforced execution runs the resolved path from command analysis. On common `/bin` -> `/usr/bin` layouts this can mount one path while the command executes another, making sandboxed safeBins fail with command-not-found; pass the exact planned executable paths into the mount list.
  - confidence: 0.9
- **[P1] Mount the actual runtime shell path:** `src/agents/bash-tools.exec-host-gateway.ts:207-219`
  - body: `makeBwrapParams()` passes only the selected shell basename, while `runExecProcess()` executes the full shell path after `--`. Users with `/usr/local/bin/bash`, Nix shells, or other configured shells outside the trusted dirs can have every sandboxed command fail before it starts.
  - confidence: 0.91
- **[P1] Accept nsSandbox in the strict config schema:** `src/config/types.tools.ts:253-258`
  - body: This adds `tools.exec.nsSandbox` to the public exec config type and runtime defaults, but the strict zod schema still omits the key. A user setting the documented config will get an unrecognized-key validation error instead of enabling bwrap.
  - confidence: 0.93
- **[P2] Preserve network for unclassified safe-bin profiles:** `src/infra/exec-bwrap-sandbox.ts:304-313`
  - body: The gateway always supplies `commandBins`, and `shouldShareNetwork()` disables networking whenever none of those names are hardcoded. Any custom profiled safeBin that legitimately needs network but is not listed will silently lose network under bwrap.
  - confidence: 0.78
- **[P2] Split out install lifecycle policy changes:** `package.json:435-439`
  - body: The package diff changes `onlyBuiltDependencies`, which controls install-time build-script execution and is unrelated to safeBins bwrap isolation. Drop this from the branch or split it into a separately reviewed package-policy change after rebasing.
  - confidence: 0.86
