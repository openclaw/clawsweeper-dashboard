## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Accept nsSandbox in the strict config schema:** `src/config/types.tools.ts:253-258`
  - body: This adds `tools.exec.nsSandbox` to the public exec config type, but PR-head `ToolExecBaseShape` remains strict and omits the key. Users setting the documented config would get an unrecognized-key validation error instead of enabling bwrap.
  - confidence: 0.94
- **[P1] Mount the executable paths that will actually run:** `src/infra/exec-bwrap-sandbox.ts:232-235`
  - body: `buildBwrapArgs()` re-resolves configured safe-bin names from trusted dirs, while enforced execution renders each segment's resolved executable path. On `/bin` to `/usr/bin` or symlink layouts this can mount one path while the command executes another, causing sandboxed safeBins to fail.
  - confidence: 0.91
- **[P1] Mount the selected shell path, not only its basename:** `src/agents/bash-tools.exec-host-gateway.ts:207-218`
  - body: `makeBwrapParams()` strips a full shell path down to a basename, but runtime execution appends the full selected shell path after `--`. Shells such as `/usr/local/bin/bash` or Nix shells can be absent inside the namespace and fail before the command starts.
  - confidence: 0.91
- **[P2] Deny parent binds of sensitive paths:** `src/infra/exec-bwrap-sandbox.ts:378-384`
  - body: `isDeniedBindPath()` only rejects paths inside the denied path, so an extra bind for the user's home directory can still bring `.ssh` or `.gnupg` into the sandbox. The sensitive-path check needs to reject parent binds that contain denied paths too.
  - confidence: 0.82
- **[P2] Mount env for shebang-based safeBins:** `src/infra/exec-bwrap-sandbox.ts:52-53`
  - body: The default shell mount list omits `env`, so safe-bin scripts with `#!/usr/bin/env ...` shebangs can fail inside bwrap even when the script itself is mounted. This affects custom profiled safeBins and common wrapper scripts unless users also add `env` as a safe bin.
  - confidence: 0.81
- **[P2] Preserve network for unclassified safe bins:** `src/infra/exec-bwrap-sandbox.ts:304-313`
  - body: The gateway always supplies `commandBins`, and `shouldShareNetwork()` disables networking whenever none of those names are hardcoded. Any custom profiled safeBin that legitimately needs network but is not listed will silently lose network under bwrap.
  - confidence: 0.78
- **[P2] Split out install lifecycle policy changes:** `package.json:434-439`
  - body: The package diff changes install-time build-script approval via `pnpm.onlyBuiltDependencies`, which is unrelated to safeBins bwrap isolation. Drop this from the branch or split it into a separately reviewed package-policy change.
  - confidence: 0.88
