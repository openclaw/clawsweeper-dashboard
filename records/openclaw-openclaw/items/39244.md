Rebase or replace this with a focused, security-reviewed exec sandbox change that mounts exact runtime paths, completes the config/docs contract, avoids unrelated install-policy churn, and includes redacted Linux bwrap runtime proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Mount the exact planned executable paths:** `src/infra/exec-bwrap-sandbox.ts:232-235`
  - body: `buildBwrapArgs()` re-resolves each configured safe-bin name from trusted dirs, but the enforced command executes the resolved path from command analysis. On common `/bin` and `/usr/bin` layouts this can mount one path while executing another, so sandboxed safeBins fail with command-not-found; pass the exact planned executable paths into the mount list.
  - confidence: 0.9
- **[P1] Mount the actual runtime shell path:** `src/agents/bash-tools.exec-host-gateway.ts:207-219`
  - body: `makeBwrapParams()` passes only the selected shell basename, while `runExecProcess()` executes the full shell path after `--`. Users with `/usr/local/bin/bash`, Nix shells, or any configured shell outside the trusted dirs can have every sandboxed command fail before it starts.
  - confidence: 0.91
- **[P1] Accept nsSandbox in the strict config schema:** `src/config/types.tools.ts:253-257`
  - body: This adds `tools.exec.nsSandbox` to the public exec config type and runtime defaults, but the strict zod schema still omits the key. A user setting the documented config will get an unrecognized-key validation error instead of enabling bwrap.
  - confidence: 0.93
- **[P2] Preserve network for unclassified safe-bin profiles:** `src/infra/exec-bwrap-sandbox.ts:304-313`
  - body: `shouldShareNetwork()` disables networking whenever `commandBins` is present and none of the names are hardcoded in `NETWORK_REQUIRED_BINS`. Since the gateway always supplies `commandBins`, custom profiled safeBins that legitimately need network will silently lose network under bwrap.
  - confidence: 0.78
- **[P2] Keep env available for shebang-based safe bins:** `src/infra/exec-bwrap-sandbox.ts:52-53`
  - body: The default shell mount list omits `env`, so any safe-bin script with a `#!/usr/bin/env ...` shebang can fail inside bwrap even when the script itself is mounted. Mount `/usr/bin/env` or otherwise account for shebang interpreters for custom safe-bin scripts.
  - confidence: 0.76
- **[P2] Split out install lifecycle policy changes:** `package.json:435-439`
  - body: The package diff changes `onlyBuiltDependencies`, which controls install-time build-script execution and is unrelated to safeBins bwrap isolation. Drop this from the branch or split it into a separately reviewed package-policy change after rebasing.
  - confidence: 0.86
