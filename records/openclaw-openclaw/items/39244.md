  - body: The PR advertises `tools.exec.nsSandbox` as the user-facing way to enable bwrap, but only the TypeScript config type is extended. - **[P1] Mount the exact shell path used at runtime:** `src/agents/bash-tools.exec-host-gateway.ts:208-218`
  - body: The gateway reduces `getShellConfig().shell` to a basename before bwrap planning, but runtime executes the full shell path inside the namespace. If the configured shell is outside the trusted safe-bin dirs or resolves differently, bwrap can start and then fail to exec the shell before any safeBins command runs.
  - confidence: 0.82

  - body: The new `extraBinds` escape hatch only blocks paths inside a narrow denylist. For a feature presented as filesystem isolation, broad ancestor binds such as system or home ancestors need stricter policy or explicit security signoff because they can re-expose sensitive host paths inside the namespace.
  - confidence: 0.78

  - body: This security-boundary PR also changes `package.json` lifecycle metadata for packages unrelated to bwrap. That affects install/supply-chain policy and should be removed from this branch or reviewed separately from the exec sandbox feature.
