  - body: The PR documents `tools.exec.nsSandbox` and adds it to `ExecToolConfig`, but `ToolExecBaseShape` remains strict and does not include this key.   - confidence: 0.95

  - body: This only passes the shell basename into bwrap planning, but `runExecProcess` later execs the full `getShellConfig().shell` path. If `SHELL` resolves to something like `/usr/local/bin/zsh`, bwrap mounts a different path or nothing at all, and every sandboxed safeBins command fails before running.
  - confidence: 0.88

  - body: Only `sh` and `bash` are guaranteed in the namespace, while custom safe-bin scripts can commonly use shebangs such as `/usr/bin/env node` or another interpreter. Mount declared interpreters safely or reject script safe bins that cannot run inside the namespace.
  - confidence: 0.8
- **[P2] Drop unrelated lifecycle allowlist changes:** `package.json:435-439`
  - body: This sandbox branch changes package install lifecycle policy by allowing unrelated native/package build scripts, while current main has moved that policy to `pnpm-workspace.yaml` and explicitly keeps `@discordjs/opus` disabled for default lanes. Keep the exec sandbox change separate from package build policy.
