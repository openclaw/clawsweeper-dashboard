## What This Changes

This PR adds an optional Linux bubblewrap namespace sandbox for `safeBins` exec commands, threads `tools.exec.nsSandbox` through exec defaults/runtime, adds bwrap unit tests, and includes unrelated package/UI/test cleanup.

## Best Possible Solution

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Mount the exact approved executable paths:** `src/infra/exec-bwrap-sandbox.ts:232-235`
  - body: `buildBwrapArgs()` re-resolves every safe-bin name from trusted dirs, but the enforced command runs the resolved execution paths from allowlist analysis. If the plan executes `/usr/bin/head` while this loop mounted `/bin/head`, sandboxed safeBins fail with command-not-found; pass and mount the exact paths used for this run.
  - confidence: 0.9
- **[P1] Mount the actual runtime shell path:** `src/agents/bash-tools.exec-host-gateway.ts:207-219`
  - body: `makeBwrapParams()` passes only the shell basename into the mount list, while `runExecProcess()` executes the configured shell path after `--`. Users with `/usr/local/bin/bash`, Nix shells, or another shell outside the trusted dirs can have every sandboxed command fail to start.
  - confidence: 0.91
- **[P1] Accept nsSandbox in the strict config schema:** `src/config/types.tools.ts:253-257`
  - body: This exposes `tools.exec.nsSandbox` in the public config type and threads it into runtime defaults, but the strict zod config schema still omits the key. A user setting the documented config would get an unrecognized-key validation error instead of enabling bwrap.
  - confidence: 0.93
- **[P2] Preserve network unless policy can prove it is safe to drop:** `src/infra/exec-bwrap-sandbox.ts:304-313`
  - body: The gateway always supplies `commandBins`, and `shouldShareNetwork()` disables networking whenever none of those names are in the hardcoded list. A configured safe-bin profile that legitimately needs network but is not listed here will silently lose network under bwrap, contrary to the PR's stated shared-network default.
  - confidence: 0.78
- **[P2] Split out install lifecycle policy changes:** `package.json:435-439`
  - body: The package diff changes `onlyBuiltDependencies`, which changes install-time code execution policy and is unrelated to the bwrap exec sandbox. Drop it from this security PR or split it into a separately reviewed package-policy change after rebasing.
  - confidence: 0.86
- **[P3] Add the required docs and changelog:** `src/config/types.tools.ts:253-257`
  - body: This adds a user-facing `tools.exec.nsSandbox` feature, but the branch does not update `CHANGELOG.md` or the public exec/safeBins docs. Add release notes and operator guidance so users can discover and safely configure the new mode.
  - confidence: 0.82
