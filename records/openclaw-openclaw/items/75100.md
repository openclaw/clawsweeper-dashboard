## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Compute voice fallback after queued segment flushes:** `extensions/feishu/src/reply-dispatcher.ts:717-724`
  - body: In segment mode, `mediaFallbackText` is resolved before `sendMediaRepliesInOrder` waits behind `segmentDeliveryQueue`. If a final voice payload arrives while a tool-start segment is still sending, the fallback can include text that the earlier queued flush is about to deliver, duplicating it on degraded or failed media sends. Resolve the fallback inside the queued media operation after prior flushes update `segmentDeliveredText`, and cover that in-flight case.
  - confidence: 0.87
- **[P2] Stop dropping finals that are only substrings of delivered text:** `extensions/feishu/src/reply-dispatcher.ts:534-535`
  - body: `resolveUndeliveredSegmentText` returns empty whenever the delivered segment merely contains the final text. A valid short follow-up such as `ok` or `done` can be lost if that word appeared in the pre-tool segment. Limit suppression to exact duplicates, prefix continuations, or real overlap cases so distinct finals still send.
  - confidence: 0.78
