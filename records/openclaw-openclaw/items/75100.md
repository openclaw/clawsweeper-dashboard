## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Resolve media fallback text after queued flushes:** `extensions/feishu/src/reply-dispatcher.ts:717-724`
  - body: In segment mode, `mediaFallbackText` is computed before `sendMediaRepliesInOrder` waits behind an in-flight partial flush. If a final voice/media payload arrives while the tool-start segment is still sending, `segmentDeliveredText` is still empty, so a degraded or failed media upload falls back with the already-delivered partial text duplicated. Compute the fallback after earlier queued segment flushes have updated delivered state, and cover that ordering in a test.
  - confidence: 0.87
- **[P2] Avoid substring-only segment suppression:** `extensions/feishu/src/reply-dispatcher.ts:534-535`
  - body: `resolveUndeliveredSegmentText` drops a final whenever the prior delivered text merely contains it. A valid short follow-up such as `ok` or `done` can be lost if that word appeared in the pre-tool segment. Limit suppression to exact, prefix, or real overlap cases so segment mode does not discard distinct final answers.
  - confidence: 0.78
