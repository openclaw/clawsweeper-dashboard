## Summary

This PR should stay open because the direct-delivery shortcut is present, but literal direct-send text still passes through reply-status suppression and can be dropped before outbound delivery.

## What This Changes

The branch adds a `directDelivery` CLI/Gateway path for `openclaw agent --deliver`, updates outbound payload handling, docs, Swift protocol models, tests, and changelog.

## Best Possible Solution

Preserve every user-authored direct-delivery message before outbound planning, keep relay-status and silent suppression only on agent-composed replies, and land only after maintainers accept the CLI semantics change.

## Solution Assessment
