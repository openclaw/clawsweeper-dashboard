## Summary

Keep open: the PR addresses a real agent delivery problem, but it is not merge-ready because the direct-delivery path still drops literal user text that matches reply housekeeping suppression.

## What This Changes

The PR changes `openclaw agent --deliver` so CLI/Gateway direct notifications send the provided message through outbound delivery without running an agent turn, while internal event delivery still uses the agent-composed path.

## Best Possible Solution

Land a version that preserves every literal direct-delivery message while keeping reply-only suppression and normalization on agent-composed replies, after maintainer approval of the CLI semantics.

## Reproduction Assessment

Yes. Source inspection shows current main sends `--deliver` as a normal agent request, and PR head still drops direct text that matches `isSuppressedRelayStatusText` before outbound delivery.

## Solution Assessment

No, not yet. The direct-delivery shortcut is a plausible boundary, but direct user-authored text needs to bypass all reply-only suppression, not only silent-token parsing.
