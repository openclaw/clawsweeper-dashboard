## Summary

Keep open. The PR now uses an explicit direct-delivery signal and has sufficient terminal proof, but latest head still drops direct user text that matches relay-housekeeping phrases, and the backward-incompatible CLI semantics need maintainer approval before merge.

## What This Changes

The PR adds CLI/Gateway/agent-command plumbing, docs, protocol types, and tests so `openclaw agent --deliver` can send the provided message without running a model turn.

## Best Possible Solution

Land a version that preserves every literal direct-delivery message while keeping reply-only suppression on agent-composed replies, after maintainers explicitly accept the CLI contract change.

## Reproduction Assessment

Yes. Source inspection shows current main sends `--deliver` as an agent-reply request, and PR head still routes direct text through relay-status suppression after only preserving silent tokens.

## Solution Assessment

No, not yet. The explicit direct-delivery gate is the right boundary for the earlier broad-bypass problem, but payload planning must bypass all reply-only suppression when `directDeliveryText` is present.
