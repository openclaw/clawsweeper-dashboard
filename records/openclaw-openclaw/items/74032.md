## Summary

Keep open. Current main still lacks Telegram group-scope native command registration/cleanup for reliable `/status` availability in group/forum-topic contexts, while the focused implementation path is already represented by open PR #6457 rather than a merged fix.

## What This Changes

The issue reports unreliable Telegram `/status` responses in Telegram group forum topics even though normal mention-based replies work in the same topic with `requireMention: false`.

## Best Possible Solution

Land PR #6457 or a current-layout replacement that registers Telegram native commands in both default and group-chat scopes, clears both scopes when native commands are disabled, preserves topic-aware replies, and adds focused Telegram extension tests plus a user-facing changelog entry.

## Reproduction Assessment

Yes for a source-level reproduction: current main only calls default-scope `setMyCommands`/`deleteMyCommands`, while the normal mention/text topic path is separate and already topic-aware. No live end-to-end Telegram forum-topic reproduction was run during this read-only review.

## Solution Assessment

Yes, the best fix belongs in the Telegram plugin native command registration/auth/delivery path, not core routing. PR #6457 is the right narrow direction, but it is still open and should cover disable cleanup before the issue is considered fixed.
