## Summary

Keep open. Current main still has plausible Telegram `/status` divergence from normal group/topic messages: Telegram command-menu sync is still default-scope only, and the group control-command authorization path can block `/status` before the normal mention/text reply path. Related PRs remain open and unmerged, so this is not implemented on main yet.

## What This Changes

The issue asks for Telegram `/status` to respond consistently in the same group forum topic where ordinary mention/text routing already works.

## Best Possible Solution

Land a current-layout Telegram plugin fix that makes group/topic `/status` follow the same intended access policy as ordinary allowed group messages, registers and clears native commands in both default and `all_group_chats` scopes, and keeps topic thread parameters on command replies. Core command handling should stay generic.

## Reproduction Assessment

Unclear for a live end-to-end reproduction because no Telegram bot was run. There is a high-confidence code-level path: a group/topic where normal messages pass can still have `/status` blocked by the current command gate or native command auth when command-specific allowlist authorizers are absent.

## Solution Assessment

Yes, keeping this open is the right current outcome. The narrow fix belongs in the Telegram plugin and should be coordinated with open PR #6457 and adjacent group-command PRs rather than closing this before a fix lands.
