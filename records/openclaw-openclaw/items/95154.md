## Summary

Keep open. The PR addresses a real WebChat restart-recovery notice gap, but the latest branch still writes the fallback notice to canonical history and can rebind a valid custom/forked transcript target, so it needs a focused repair before merge.

## What This Changes

The PR writes an unresumable restart-recovery notice into session transcripts for non-deliverable WebChat/internal sessions and adds an explicit transcript helper target override plus focused recovery tests.

## Best Possible Solution

Use the transcript fallback for non-deliverable WebChat recovery, but target the same visible transcript `chat.history` would read for non-stale `sessionFile` entries and reserve canonical history for missing or stale generated paths.

## Reproduction Assessment

Yes, source-reproducible. Current main rejects WebChat as a deliverable channel and has no transcript fallback, while the PR-introduced `sessionFile` regression follows from the canonical write plus `touchSessionEntry` behavior and existing reader-ordering tests.

## Solution Assessment

No. The transcript fallback is the right layer for the missing WebChat notice, but always writing canonical history is not the safest fix because it can diverge from `chat.history`'s visible transcript target.
