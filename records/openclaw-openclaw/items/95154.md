## Summary

Keep open. Current main still has the WebChat/internal no-notice path, and this PR is the active candidate fix, but the current patch can rebind valid custom or forked transcript targets to canonical history during recovery.

## What This Changes

## Best Possible Solution

Keep the WebChat transcript fallback, but choose the same visible transcript target that chat.history would read for non-stale sessionFile entries and use canonical history only for missing or stale generated targets.

## Reproduction Assessment

Yes, source-reproducible. Current main rejects WebChat as a deliverable channel and has no transcript fallback, while the PR-introduced sessionFile rebind follows from the canonical write plus touchSessionEntry behavior and reader ordering.

## Solution Assessment

No. The transcript fallback is the right layer for the missing WebChat notice, but always writing canonical history is not the safest fix because it can diverge from chat.history's visible transcript target.
