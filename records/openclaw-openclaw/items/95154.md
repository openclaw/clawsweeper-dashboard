## Label Justifications

- `P1`: The PR targets a user-facing WebChat restart path where an active dashboard session can be left failed and blank without a recovery notice.
- `merge-risk: 🚨 compatibility`: The patch can change an existing session's persisted transcript target during recovery, affecting what custom or forked sessions display after upgrade.
- `merge-risk: 🚨 session-state`: The patch writes restart-recovery notice state into persisted session transcripts and can update session metadata.
- `merge-risk: 🚨 message-delivery`: The patch changes fallback behavior for recovery notices when outbound delivery is unavailable and also adjusts model-visible inbound prompt context.

## Summary

Keep open: the WebChat restart-recovery fallback is still useful, but this head is not merge-ready because the new canonical transcript write can rebind valid custom or forked session files, and the branch also carries a separate runtime-context fix now tracked by https://github.com/openclaw/openclaw/pull/95349.

## What This Changes

The PR adds a transcript fallback for non-deliverable WebChat/internal restart-recovery notices, extends the assistant transcript append helper with an explicit sessionFile override, adds regression tests, and also includes a runtime-context prompt fix for duplicated system-event headers.

## Best Possible Solution

Keep the WebChat transcript fallback, but write to the same visible transcript target chat.history would use for valid non-stale sessionFile entries and leave the #95323 runtime-context fix to its focused PR.

## Reproduction Assessment

Yes, source-reproducible: current main rejects WebChat as deliverable and has no transcript fallback after sendUnresumableSessionNotice returns false. I did not run a live WebChat plus managed restart scenario in this read-only pass.

## Solution Assessment

No. The transcript fallback is the right layer for the WebChat notice gap, but always writing canonical history is not the safest fix because current chat.history intentionally preserves valid non-stale sessionFile targets ahead of canonical history.
