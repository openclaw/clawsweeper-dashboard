## Summary

Keep open: the WebChat notice fix is useful, but the current patch can rebind valid non-stale sessionFile-backed sessions to canonical history, so it needs a narrow target-selection repair before merge.

## What This Changes

The PR adds a non-deliverable WebChat/internal restart-recovery transcript notice fallback, an explicit transcript append target parameter, and focused restart-recovery tests.

## Best Possible Solution

Keep the WebChat transcript fallback, but choose the same visible transcript target that chat.history would read for valid non-stale sessionFile entries and use canonical history only for missing or stale generated paths.

## Reproduction Assessment

Yes, source-reproducible. Current main rejects WebChat as a deliverable channel and has no transcript fallback, while the PR-introduced sessionFile rebind follows from the canonical write plus touchSessionEntry behavior and reader ordering.

## Solution Assessment

No. The transcript fallback is the right layer for the missing WebChat notice, but always writing canonical history is not the safest fix because it can diverge from chat.history's visible transcript target.
