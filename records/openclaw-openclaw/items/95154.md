## Label Justifications

- `P1`: The PR targets a user-facing WebChat restart path where an active dashboard turn can be left failed and blank without a recovery notice.

- `merge-risk: 🚨 session-state`: The patch writes assistant recovery notices and touches sessionFile/updatedAt metadata during restart recovery.

## Summary

Keep open. The WebChat restart-recovery fallback is still useful, but this head is not merge-ready because it preserves raw `sessionFile` metadata before an explicit transcript append, which can bypass the existing session-file resolver for relative or malformed paths.

## What This Changes

The PR writes an unresumable-session notice into the transcript for WebChat/internal restart-recovery failures, adds an explicit transcript append target parameter, and adds canonical/stale/custom target tests.

## Best Possible Solution

Keep the WebChat transcript fallback, but normalize the selected transcript target through the existing session-file resolver and add relative/malformed sessionFile regression coverage before merge.

## Reproduction Assessment

Yes, source-reproducible. Current main drops the notice for non-deliverable WebChat recovery, and the remaining PR bug follows from the raw explicit sessionFile target bypassing the resolver used by readers and existing writers.

## Solution Assessment

No. The transcript fallback is the right layer, but preserved sessionFile targets need resolver-backed normalization before append; otherwise the fix can write to a path the visible history does not read.
