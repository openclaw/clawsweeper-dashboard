Keep open. The PR targets a real WebChat restart-recovery notice gap, but the new explicit transcript-file append path can bypass the existing session-rebound guard and mis-associate a new session with an old transcript file.

## What This Changes

## Best Possible Solution

Keep the canonical-file write for stale sessionFile cases, but make explicit transcript writes enforce the expected old sessionId and then prove the WebChat reconnect notice in a real or visibly diagnostic setup.

## Solution Assessment

No. Writing the notice to the transcript is the right narrow direction, but this implementation must preserve session-rebound protection when overriding the write target.
