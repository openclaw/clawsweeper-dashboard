## Summary

Keep open. The PR addresses a real WebChat restart-recovery notice gap, but it is not merge-ready because the current branch can rebind valid custom or forked transcript files to canonical history during failed recovery.

## What This Changes

The PR writes restart-recovery unresumable notices into session transcripts for WebChat/internal sessions that cannot use outbound delivery.

## Best Possible Solution

Keep the WebChat transcript fallback, but write to the same visible transcript target that chat.history would read for non-stale sessionFile entries and use canonical history only for missing or stale generated targets.

## Reproduction Assessment

Yes, source-reproducible. Current main rejects WebChat as a deliverable channel and has no transcript fallback, while the PR-introduced sessionFile rebind follows from the canonical write plus touchSessionEntry behavior and reader ordering.

## Solution Assessment

No. The transcript fallback is the right layer for the missing WebChat notice, but always writing canonical history is not the safest fix because it can diverge from chat.history's visible transcript target.
