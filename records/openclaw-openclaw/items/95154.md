## Summary

Keep open. Current main still lacks the WebChat/internal-channel transcript fallback, but this PR head has a blocking session-state compatibility defect: the fallback always writes canonical history even when the current reader would show a valid non-stale sessionFile target.

## What This Changes

The PR writes unresumable restart-recovery notices into non-deliverable session transcripts, adds an explicit transcript write target parameter, and also strips duplicated system-event prefixes from runtime-context model prompts.

## Best Possible Solution

Keep the WebChat transcript fallback, choose the same visible transcript target that chat.history would read for valid non-stale sessionFile entries, keep stale generated paths canonical, and split or explicitly own the separate runtime-context fix before merge.

## Reproduction Assessment

Yes, source-reproducible. Current main rejects WebChat as deliverable and lacks a transcript fallback, while the PR-introduced rebind follows from canonical target selection plus touchSessionEntry behavior.

## Solution Assessment

No. The transcript fallback is the right layer for the missing WebChat notice, but always writing canonical history is not the safest fix because current readers prefer a valid non-stale sessionFile target.
