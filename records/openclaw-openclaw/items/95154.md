## Summary

Keep open. The PR targets a real WebChat restart-recovery notice gap, but it is not merge-ready because the fallback always writes canonical history and can rebind valid custom or forked `sessionFile` entries away from the transcript `chat.history` would otherwise show.

## What This Changes

The PR adds a restart-recovery fallback that appends an unresumable assistant notice to the transcript for non-deliverable WebChat/internal sessions, plus focused tests and an internal transcript helper target override.

## Best Possible Solution

Keep the WebChat transcript fallback, but target the same visible transcript that `chat.history` would read for non-stale custom or forked session files and use canonical history only for missing or stale generated paths.

## Reproduction Assessment

Yes, source-reproducible. Current main rejects WebChat as a deliverable channel and has no transcript fallback, while the PR-introduced session-file rebind follows from the canonical write plus `touchSessionEntry` behavior and reader ordering.

## Solution Assessment

No. The transcript fallback is the right layer for the missing WebChat notice, but always writing canonical history is not the safest fix because it can diverge from `chat.history`'s visible transcript target.
