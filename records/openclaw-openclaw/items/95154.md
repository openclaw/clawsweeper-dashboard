## Summary

Keep open. The PR addresses a real WebChat restart-recovery no-notice path, but the current head can rebind a valid custom or forked `sessionFile` to canonical history during failed recovery, which can change what `chat.history` shows after upgrade.

## What This Changes

The branch adds a restart-recovery fallback that writes the unresumable notice into a WebChat/non-deliverable session transcript, plus tests and an explicit transcript append target parameter.

## Best Possible Solution

Keep the WebChat transcript fallback, but write to the same visible transcript target that `chat.history` would read for valid non-stale `sessionFile` entries and use canonical history only for missing or stale generated paths.

## Reproduction Assessment

Yes, source-reproducible. Current main rejects WebChat as a deliverable channel and has no transcript fallback, while the PR-introduced `sessionFile` rebind follows from the canonical write plus `touchSessionEntry` behavior and reader ordering.

## Solution Assessment

No. The transcript fallback is the right layer for the missing WebChat notice, but always writing canonical history is not the safest fix because it can diverge from `chat.history`'s visible transcript target.
