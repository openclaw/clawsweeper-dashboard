- **[P1] Do not add a second usage footer config path:** `src/config/types.messages.ts:156-159`
  - body: Current main already supports runtime-authored `/usage full` footers through `usageState` and `messages.usageTemplate`.   - confidence: 0.9
- **[P1] Use the processed text for gateway sends:** `src/infra/outbound/message.ts:249`
  - body: The hook result is only used for the direct outbound payload plan. The gateway branch still sends `params.content`, so gateway-delivered replies can still show fabricated model footers and miss server-authored warnings.
  - confidence: 0.92
- **[P2] Skip the stateful hook during dry runs:** `src/infra/outbound/message.ts:244-249`
  - body: The new hook runs before the `dryRun` return and can persist `contextWarningThresholdsTriggered`. A dry-run send can therefore consume a one-shot warning without delivering it, causing the next real send to omit the warning.
  - confidence: 0.9
- **[P2] Record all crossed thresholds when warning:** `src/infra/outbound/footer-hook.ts:196`
  - body: When usage jumps from below 70% to 88%, the code records only 85%. The skipped 70% threshold remains unwarned and can fire later after higher warnings, violating the one-shot ordered warning contract.
  - confidence: 0.88
- **[P2] Make warning emission atomic with the session write:** `src/infra/outbound/footer-hook.ts:152-169`
  - body: The warning decision reads `alreadyWarned` before the serialized `updateSessionStore` write. Two concurrent sends for the same session can both emit the same threshold before either write records it, so the user can see duplicate one-shot warnings.
  - confidence: 0.83
