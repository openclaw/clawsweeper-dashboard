Full review comments:

- **[P1] Do not add a second usage footer config path:** `src/config/types.messages.ts:157-159`
  - body: Current main already supports runtime-authored `/usage full` footers through `usageState` and `messages.usageTemplate`. Adding `messages.outboundFooter` creates a competing template/config contract; rebase this onto the existing usage footer path or get explicit maintainer approval for a second surface.
  - confidence: 0.9
- **[P1] Use the processed text for gateway sends:** `src/infra/outbound/message.ts:337`
  - body: The hook result is only used for the direct outbound payload plan. The gateway branch still sends `params.content`, so gateway-delivered replies can show fabricated model footers and miss server-authored warnings even when the feature is enabled.
  - confidence: 0.94
- **[P2] Skip the stateful hook during dry runs:** `src/infra/outbound/message.ts:244-249`
  - body: The new hook runs before the `dryRun` return and can persist `contextWarningThresholdsTriggered`. A dry-run send can consume a one-shot warning without delivering a message, so the next real send may incorrectly omit it.
  - confidence: 0.9
- **[P2] Record crossed thresholds when persisting a warning:** `src/infra/outbound/footer-hook.ts:195`
  - body: When usage jumps over multiple thresholds, the PR records only the highest selected threshold. Skipped lower thresholds remain unwarned and can fire later after higher warnings, breaking the once-per-threshold contract.
  - confidence: 0.88
- **[P2] Make warning emission atomic with the session write:** `src/infra/outbound/footer-hook.ts:156-177`
  - body: Warning text is decided from an `alreadyWarned` snapshot before `updateSessionStore` runs. Concurrent sends can both prepend the same one-shot warning; re-evaluate or reserve the threshold inside the serialized writer before emitting user-visible text.
  - confidence: 0.84
