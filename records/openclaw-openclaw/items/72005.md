- **[P1] Use the processed text for gateway sends:** `src/infra/outbound/message.ts:337`
  - body: `processedContent` is only used to build the direct outbound payload plan. The gateway branch still sends `params.content`, so gateway-delivered replies can show fabricated model footers and miss server-authored warnings even when the feature is enabled.
  - confidence: 0.94

  - body: Current main already has runtime-authored `/usage full` rendering through `messages.usageTemplate` and `usageState`. Adding `messages.outboundFooter` as a parallel footer renderer creates two config contracts for the same user-visible footer behavior unless maintainers explicitly choose that split.
  - confidence: 0.9
- **[P2] Skip the stateful hook during dry runs:** `src/infra/outbound/message.ts:244-249`
  - body: The hook executes before the `dryRun` return and can persist `contextWarningThresholdsTriggered`; a dry-run send can consume a one-shot warning without delivering any message, causing the next real send to omit it.
  - confidence: 0.9
- **[P2] Record crossed thresholds when persisting warnings:** `src/infra/outbound/footer-hook.ts:195`
  - body: When usage jumps over multiple thresholds, the PR records only the highest chosen threshold. Lower crossed thresholds remain unwarned and can later emit stale warning text after higher thresholds have already fired.
  - confidence: 0.88
- **[P2] Make warning emission atomic with the session write:** `src/infra/outbound/footer-hook.ts:155-177`
  - body: The hook reads `alreadyWarned`, decides to prepend a warning, and only later writes state. Concurrent sends for the same session can both read the old state and emit the same one-shot warning.
  - confidence: 0.83
