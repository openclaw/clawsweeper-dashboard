## Summary

Keep open. Current main still lacks this feature, but the PR head is not mergeable as-is: gateway mode sends raw content, dry runs can mutate warning state, warning recording is not atomic or complete, and the branch predates current voice-note payload handling.

## What This Changes

The PR adds outbound footer stripping/rendering, context-warning config and session state, sendMessage wiring, tests, generated schema updates, and a new docs page.

## Best Possible Solution

Land this implementation only after the outbound hook reaches every real delivery mode, dry runs stay read-only, warning state is atomically recorded for all crossed thresholds, current media behavior is preserved, and tests/changelog cover those contracts.

## Reproduction Assessment

Yes. Static inspection of PR head 5bfcdd039a7a222e41f137a0c9b7c302e005ce5a reproduces the blockers: gateway mode sends params.content, the hook executes before dryRun, and warning state is decided before the locked write while recording only one threshold.

## Solution Assessment

No. The product direction is sound, but the current patch is not the best mergeable form until the outbound pipeline and session-state semantics are repaired with focused tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Send processed content through gateway mode:** `src/infra/outbound/message.ts:252`
  - body: The hook result is only fed into the outbound payload plan, but the gateway branch still sends `message: params.content`. Gateway-delivered replies can therefore bypass footer stripping/rendering and context-warning prepends, leaving the main feature ineffective for gateway channels.
  - confidence: 0.95
- **[P2] Skip the stateful hook for dry runs:** `src/infra/outbound/message.ts:244-249`
  - body: `applyOutboundFooterHook` runs before the `dryRun` early return, and the hook can persist `contextWarningThresholdsTriggered`. A dry run can consume a one-shot warning without delivering a message, so the next real send may omit it.
  - confidence: 0.92
- **[P2] Record every crossed threshold:** `src/infra/outbound/footer-hook.ts:195`
  - body: When usage jumps across multiple configured thresholds, the hook persists only the selected highest threshold. Lower crossed thresholds remain unwarned and can fire later out of order, breaking the once-per-threshold contract.
  - confidence: 0.91
- **[P2] Make warning check-and-record atomic:** `src/infra/outbound/footer-hook.ts:155-157`
  - body: Warning eligibility is decided from `alreadyWarned` before the locked session-store update. Two concurrent sends for the same session can both read the old state and both emit the same one-shot warning; re-check inside the update before recording.
  - confidence: 0.86
- **[P2] Preserve voice-note payload flags:** `src/infra/outbound/message.ts:252`
  - body: Current main passes `params.asVoice` into both the outbound payload plan and gateway params, but this stale PR head lacks that behavior in the touched `sendMessage` path. A repair/rebase must keep `audioAsVoice`/`asVoice` or voice-note audio sends can regress when this branch lands.
  - confidence: 0.83
- **[P3] Add the user-facing changelog entry:** `CHANGELOG.md:5`
  - body: This is a user-facing feature with new config, outbound behavior, and docs, but the PR does not update `CHANGELOG.md`. Add an Unreleased Changes entry before merge.
  - confidence: 0.88
