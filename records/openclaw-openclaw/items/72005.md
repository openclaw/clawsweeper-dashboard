  - confidence: 0.92

  - body: When usage jumps across multiple configured thresholds, the hook persists only the selected highest threshold. Lower crossed thresholds remain unwarned and can later fire out of order, breaking the once-per-threshold contract.
  - confidence: 0.91

  - body: Warning eligibility is computed from `ctx.entry` before the serialized store update. Two concurrent sends can both emit the same one-shot warning; recheck and claim the threshold inside the `updateSessionStore` mutator.
  - confidence: 0.86
