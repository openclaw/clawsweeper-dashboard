## Summary

Keep this PR open, but the proposed patch is not ready to merge as-is. The link target exists and the PR fixes the GitHub-rendered-source complaint, but changing a docs-internal link to `index.md#...` conflicts with the scoped docs rule that internal docs links stay root-relative and omit `.md` suffixes.

## What This Changes

This PR changes the `installer script` link in `docs/install/node.md` from `/install#alternative-install-methods` to `index.md#alternative-install-methods`.

## Best Possible Solution

The desired end state is a maintainer-approved docs link convention for docs pages that are also read on GitHub: either keep the root-relative Mintlify route, use an explicit public docs URL where appropriate, or update `docs/AGENTS.md` with a narrow exception before merging file-relative `.md` links.

## Reproduction Assessment

Yes. Current main contains `/install#alternative-install-methods` in `docs/install/node.md`, and the destination heading exists in `docs/install/index.md`; the GitHub-rendered-source complaint follows from the leading slash being interpreted outside the docs host.

## Solution Assessment

No. The proposed link likely fixes the GitHub source browsing symptom, but it is not the best repository solution until the docs authoring policy allows this form or a maintainer chooses a policy-consistent alternative.
