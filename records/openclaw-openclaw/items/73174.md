## Best Possible Solution

Decide the docs-link policy for GitHub-rendered source, then use a convention-compliant published docs route or document an exception and land the matching one-line change.

No. The file-relative link solves GitHub source browsing, but it conflicts with the current `docs/AGENTS.md` rule for internal docs links; the safer path is an explicit maintainer policy decision or documented exception.
