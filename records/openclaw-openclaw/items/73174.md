## Summary

Keep this PR open: current main still has the GitHub-source readability issue, but the proposed file-relative link conflicts with the scoped docs rule that internal docs links stay root-relative for Mintlify.

## What This Changes

This PR changes the Node install page's installer-script link from `/install#alternative-install-methods` to `index.md#alternative-install-methods`.

## Best Possible Solution

Codify one docs-link rule and apply it consistently, either keeping the published-docs root-relative route or documenting a GitHub-readable exception before changing this link.

## Reproduction Assessment

Yes. Current main has a leading-slash link in `docs/install/node.md:10`, and the target heading exists in `docs/install/index.md:50`; the GitHub-source 404 follows from that Markdown link resolving against GitHub rather than the docs host.

## Solution Assessment

No. The proposed file-relative link fixes GitHub source browsing, but it conflicts with the scoped docs link convention while the published `/install#alternative-install-methods` route exists.
