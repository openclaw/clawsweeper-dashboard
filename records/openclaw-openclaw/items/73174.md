## Summary

Keep open: the proposed GitHub-source link fix is understandable, but it conflicts with the current scoped docs rule that internal docs links and section references stay root-relative, so this needs maintainer docs-convention judgment before merge.

## What This Changes

The PR changes the Node install page's opening installer-script link from `/install#alternative-install-methods` to `index.md#alternative-install-methods`.

## Best Possible Solution

Document the intended docs link convention, then either keep the published-docs route or explicitly allow this GitHub-source readability exception before landing the link form change.

## Reproduction Assessment

Yes. Current main contains the root-relative link and the destination heading exists; the GitHub-source 404 follows from rendering a leading-slash docs route outside the Mintlify host.

## Solution Assessment

No. The patch is a narrow source-browsing fix, but it is not the best repository solution unless maintainers first approve or document an exception to the scoped docs link policy.
