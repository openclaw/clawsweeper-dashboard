## Summary

Keep this PR open for maintainer docs-policy judgment. The reported GitHub-source link problem is understandable and current main still has the leading-slash link, but the proposed `index.md#...` target conflicts with the scoped docs rule that internal docs links stay root-relative and omit `.md`/`.mdx`.

## What This Changes

This PR changes the Node install page's `installer script` link from `/install#alternative-install-methods` to `index.md#alternative-install-methods`.

## Best Possible Solution

Update or reaffirm the docs-link convention, then land either a policy-approved GitHub-readable link form or keep the published-docs root-relative route.

## Reproduction Assessment

Yes. Current main has a leading-slash internal link in `docs/install/node.md`, and the destination heading exists; the GitHub-source 404 follows from that Markdown link resolving against GitHub rather than the docs host.

## Solution Assessment

No. The proposed file-relative link fixes GitHub source browsing, but it conflicts with the scoped docs link rule; maintainers should approve and document an exception or keep root-relative Mintlify routes.
