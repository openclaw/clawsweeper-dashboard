## Summary

This PR should stay open because it fixes a real GitHub-source navigation problem, but the proposed `index.md#...` link conflicts with the scoped docs rule that internal docs links stay root-relative for Mintlify.

## What This Changes

The PR changes the Node install page's `installer script` link from `/install#alternative-install-methods` to `index.md#alternative-install-methods`.

## Best Possible Solution

Keep the published-docs root-relative convention unless maintainers document a GitHub-readable exception and choose one link form for `docs/**/*.md` pages.

## Reproduction Assessment

Yes. Current main has `/install#alternative-install-methods` in `docs/install/node.md`, and a GitHub-rendered root-relative link resolves to `https://github.com/install#alternative-install-methods` rather than the repo's install overview, while the target heading exists in `docs/install/index.md`.

## Solution Assessment

No. The proposed relative Markdown link fixes source browsing, but it conflicts with the scoped docs rule for root-relative Mintlify links; the safer path is maintainer approval plus a policy update, or keeping the root-relative route.
