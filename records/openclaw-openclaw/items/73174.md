## Summary

Keep this PR open. It addresses a real GitHub-source navigation annoyance, but the current docs authoring policy explicitly requires root-relative, suffixless internal links in docs, so this needs maintainer approval or a documented convention change before landing.

## What This Changes

This PR changes the Node install page's installer-script link from the published-docs `/install#alternative-install-methods` route to a file-relative `index.md#alternative-install-methods` anchor.

## Best Possible Solution

Keep the root-relative published-docs link unless maintainers update the docs link convention; if GitHub source navigation is important, document and enforce the approved exception consistently.

## Reproduction Assessment

Yes. Current main contains the root-relative link and the target heading exists; the GitHub-source complaint follows from rendering a leading-slash docs route outside the Mintlify host.

## Solution Assessment

No. The patch is a narrow GitHub-source workaround, but it conflicts with the scoped docs policy; the safer path is a maintainer-approved convention change or retaining the published-docs route.
