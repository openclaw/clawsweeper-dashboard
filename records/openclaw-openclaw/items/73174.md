## Summary

Keep this PR open. It fixes the GitHub source-browsing complaint, but the proposed file-relative `.md` link conflicts with the scoped docs rule that internal `docs/**/*.md` links stay root-relative for Mintlify-published docs.

## What This Changes

This PR changes the opening `installer script` link in `docs/install/node.md` from `/install#alternative-install-methods` to `index.md#alternative-install-methods`.

## Best Possible Solution

Decide and document the docs link convention, then either keep the published-docs route or intentionally allow a GitHub-source readability exception before landing this link form.

## Reproduction Assessment

Yes. Current main contains the root-relative link and the target heading exists; the GitHub-source complaint follows from rendering a leading-slash docs route outside the Mintlify host.

## Solution Assessment

No. The patch is a narrow GitHub-source fix, but it is not the best repository solution unless maintainers first approve an exception to the scoped docs link policy.
