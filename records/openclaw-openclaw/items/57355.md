## Summary

Current main and v2026.6.8 still lack a native `registerMcpServer` Plugin SDK API; the existing MCP paths cover static config and bundle metadata, so this remains a valid but product/security-sensitive SDK feature request.

## What This Changes

The issue requests a native Plugin SDK `registerMcpServer` API so plugins can register managed stdio MCP servers for embedded OpenAI-backed runs.

## Best Possible Solution

A maintainer-sponsored design should decide on a narrow, additive, validated Plugin SDK MCP registration contract, with explicit process-launching security review and tests across registry capture, embedded runtime projection, precedence, and workspace scoping.

## Reproduction Assessment

Not applicable. This is a feature request, and the source check shows current main and v2026.6.8 do not expose `registerMcpServer` while existing MCP support is bundle/config based.

## Solution Assessment

Unclear until maintainer product and security review. A generic additive Plugin SDK seam may be the right solution if bundle/config MCP cannot express dynamic account-state inventories, but process launching and precedence rules need explicit owner approval.
