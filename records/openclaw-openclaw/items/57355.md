## Summary

Current main and latest release still lack the requested plugin SDK method; existing support is config/bundle MCP only. The issue should stay open because it is a plausible plugin API extension, but it needs maintainer product and security review before implementation.

## What This Changes

The issue requests a native plugin SDK `registerMcpServer` capability for managed stdio MCP servers that OpenClaw wires into embedded OpenAI-backed runs.

## Best Possible Solution

A maintainer-sponsored design should decide whether to add a narrow, validated, additive plugin SDK MCP registration contract, with explicit process-launching security review and tests across registry capture, embedded runtime projection, precedence, and workspace scoping.

## Reproduction Assessment

Not applicable. This is a feature request, and the source check shows current main and v2026.6.8 do not expose `registerMcpServer` while existing MCP support is bundle/config based.

## Solution Assessment

Unclear until maintainer product and security review. A generic additive Plugin SDK seam may be the right solution if bundle/config MCP cannot express dynamic account-state inventories, but process launching and precedence rules need explicit owner approval.
