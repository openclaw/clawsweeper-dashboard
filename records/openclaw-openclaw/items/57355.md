## Summary

Keep open. Current main still does not provide a native Plugin SDK `registerMcpServer` API, and the implemented MCP paths remain bundle-provided or user/operator-configured server definitions. The related open PR #62160 is the active implementation candidate, so this issue should stay open until that PR or a successor lands.

## What This Changes

The issue requests a native Plugin SDK `registerMcpServer` API so plugins can register managed stdio MCP servers for embedded OpenAI-backed runs.

## Best Possible Solution

Review and land a safe generic SDK implementation like #62160, or replace it with a narrower maintainer-approved successor that adds native managed MCP registration, deterministic precedence, workspace scoping, embedded runtime materialization, docs, and focused tests.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection: current main lacks `api.registerMcpServer`, while existing MCP support is limited to bundle metadata and user/operator config.

## Solution Assessment

Yes in direction: a generic Plugin SDK seam is the maintainable way to support dynamic plugin-owned MCP inventories without one-off core behavior. The exact patch should be decided in review because it adds a command/process execution surface.
