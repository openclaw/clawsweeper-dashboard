## Summary

Keep this open: current main still lacks a native Plugin SDK `registerMcpServer` surface, and the only in-repo implementation attempt was closed unmerged. The request is clear and vision-aligned, but it needs maintainer API, product, and security review because it would let native plugins register stdio MCP processes that OpenClaw launches for embedded runs.

## What This Changes

The issue requests a new Plugin SDK `registerMcpServer` API so native plugins can register managed stdio MCP servers with dynamic tools for embedded OpenAI-backed runs.

## Best Possible Solution

Land a maintainer-approved, additive Plugin SDK design that validates managed stdio MCP configs, preserves user config precedence and workspace scoping, wires lifecycle cleanup through the registry/runtime, documents the security boundary, and adds focused SDK, loader, status, and embedded runtime tests.
