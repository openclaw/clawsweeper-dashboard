## Summary

Keep open: current main and v2026.6.10 still lack a native `registerMcpServer` Plugin SDK API, and the only implementation PR is closed unmerged. The request remains a valid plugin-API design item, with positive external motivation from the active `kevinslin/openai-apps` plugin, but it needs maintainer product and security approval because it would add a public plugin process-launching surface.

## What This Changes

The issue requests a native Plugin SDK `registerMcpServer` API so plugins can register managed stdio MCP servers for embedded OpenAI/Codex-backed runs.

## Best Possible Solution

A maintainer-sponsored design should decide on a narrow, additive, validated Plugin SDK MCP registration contract, or explicitly keep dynamic MCP exposure outside core until the security model is settled.

## Reproduction Assessment

Not applicable; this is a feature request, not a report of broken existing behavior. Source inspection shows the requested SDK method is still absent from current main and v2026.6.10 while existing MCP support is bundle/config based.

## Solution Assessment
