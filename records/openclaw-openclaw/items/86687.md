## Summary

Keep open: the sticky-image half is fixed on current main and v2026.6.9, but current main and the latest release still inject the selective group guidance under the default group silent-reply policy; changing that shipped behavior needs maintainer product judgment.

## What This Changes

The issue asks to fix two v5.22 regressions in always-on Discord group chats: over-suppressive group silent-reply prompt guidance and repeated reattachment of prior CLI image cache paths.

## Best Possible Solution

Keep the shipped image-cache fix, then have maintainers choose the intended always-on group silent-reply policy and land focused prompt/config tests for that decision.

## Reproduction Assessment

Yes, source-level: default group silent policy resolves to allow, get-reply-run passes it into buildGroupChatContext, and buildGroupChatContext appends the selective guidance for any policy except disallow. The image replay half no longer reproduces from source because the cache-path skip and tests are present.

## Solution Assessment

Partially: the shipped image-cache detector fix is the right narrow solution, but the remaining group-prompt change is a maintainer policy decision between restoring v5.7 allow-only behavior and preserving current group silence semantics.
