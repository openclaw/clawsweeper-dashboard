## Summary

## What This Changes

The issue requests fixes for two v5.22 regressions: over-suppressed Discord always-on group replies and repeated reattachment of previously cached prompt images.

## Best Possible Solution

Keep the merged image-cache fix, then either explicitly accept and test a group prompt policy change for always-on rooms or document the current default as intended and close only after that maintainer decision.

## Reproduction Assessment

Yes, at source level: current main still builds group context with silent-token guidance for any policy except disallow, and current default group silent policy is allow. The sticky-image half is no longer reproducible from current source because the cache-path filter and regression tests are present.

## Solution Assessment

Partially. The image-cache detector fix is the right narrow solution for sticky reattachment, but the group-prompt fix is not safe to automate until maintainers decide whether default/unset group chats should keep current silent-token guidance or restore the older explicit-allow behavior.
