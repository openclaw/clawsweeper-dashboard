## Summary

Keep open: current main and v2026.6.8 have the sticky-image cache-path fix, but the group-chat prompt still uses the reported non-disallow silent-reply gating, and no open canonical PR owns that remaining policy-sensitive Discord behavior.

## What This Changes

The issue reports two v2026.5.22 regressions: over-broad group-chat selective-reply suppression and repeated reattachment of prior CLI image-cache paths in always-on Discord channels.

## Best Possible Solution

Keep the merged image-cache guard, then make a maintainer decision on the group silent-reply contract and cover the chosen `allow`/always-on behavior with focused prompt tests.

## Reproduction Assessment

Yes, source-level: current main still builds group context with silent-token guidance for any policy except `disallow`, and the default group silent policy resolves to `allow`. The sticky-image half is no longer source-reproducible because the cache-path filter and regression tests are present.

## Solution Assessment

Partially: the image-cache detector fix is the right narrow solution for sticky reattachment, but the remaining group-prompt change is a product-policy decision about default group silence semantics.
