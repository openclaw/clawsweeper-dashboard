## Summary

Keep open: the sticky-image half is fixed on current main and v2026.6.8, but the group-chat selective-reply half still uses the reported permissive `!== "disallow"` gating and no open canonical PR owns the remaining product decision.

## What This Changes

The issue reports two v5.22 always-on Discord regressions: group-chat over-suppression from silent-reply prompt gating and repeated reattachment of previously cached prompt images.

## Best Possible Solution

Keep the shipped image-cache fix, then make an explicit maintainer decision for always-on group silent-reply semantics and cover the chosen policy with focused prompt tests plus live Discord proof when feasible.

## Reproduction Assessment

Yes, at source level: current main still builds group context with silent-token guidance for any policy except `disallow`, and the default group policy resolves to `allow`. The sticky-image half is no longer source-reproducible because the cache-path filter and regression tests are present.

## Solution Assessment

Partially: the image-cache detector fix is the narrow right solution for sticky reattachment, but the remaining group-prompt change is a compatibility-sensitive product-policy decision rather than a safe automatic one-line revert.
