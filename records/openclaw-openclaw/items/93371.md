## Summary

Keep open: this is a focused fix for the open memory/session boundary, but it is not merge-ready because the patch adds a process-lifetime scoped MemoryDB cache with no eviction and the supplied proof is mocked hook-level terminal output rather than real memory-lancedb or gateway behavior.

## What This Changes

The PR changes memory-lancedb automatic recall/capture to use an agent/session-scoped auto-recall store, fail closed without scope, keep dynamic recalled memory in prependContext, and add regression tests.

## Best Possible Solution

Land a scoped, data-only automatic memory boundary with bounded scoped DB lifecycle, explicit maintainer acceptance of the scoped-store reset/no-migration tradeoff, and redacted live gateway or memory-lancedb proof.

## Reproduction Assessment

## Solution Assessment
