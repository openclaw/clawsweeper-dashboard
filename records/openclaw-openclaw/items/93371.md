Keep this PR open but do not merge it as-is: it moves auto-recalled memory out of the user prompt and into plugin system/developer context, which is still model-instruction space and does not resolve the security/session boundary from the linked issue. The branch changes memory-lancedb auto-recall to return recalled memories through `prependSystemContext` instead of `prependContext` and updates two hook regression assertions.

## Best Possible Solution

Use a maintainer-approved data-only, scoped memory recall boundary with runner and Codex coverage proving recalled memories cannot become user or developer instructions.

## Solution Assessment

No. Moving dynamic untrusted recall text into `prependSystemContext` keeps it in model instruction space; the safer fix needs a maintainer-approved scoped, data-only memory boundary.
