## Summary

## What This Changes

The issue reports that session compaction can leave malformed Anthropic thinking blocks in history, causing later requests in the same session to fail until restart.

## Best Possible Solution

Keep the issue open and decide a cache-safe repair that preserves valid provider-owned thinking only when byte-stable, but strips or self-heals invalid historical thinking after compaction with regression coverage for the reported Anthropic error.

## Reproduction Assessment

Yes, at source level: current main can preserve nonblank signed thinking for Claude Sonnet 4.6 while only stripping missing or blank signatures, and the reported invalid-signature text is not covered by the recovery regex. I did not establish a live Anthropic reproduction because this read-only review did not have a real malformed transcript or credentials.

## Solution Assessment

Unclear: stripping all older thinking during compaction would likely stop the failure, but it needs maintainer confirmation because current code intentionally preserves newer Claude thinking blocks for prompt-cache/context behavior. A narrower fix should target compacted or rejected invalid replay history rather than silently changing all valid replay.
