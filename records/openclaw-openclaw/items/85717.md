## Summary

Keep open: current main still has a source-level path where Claude Sonnet 4.6 replay can preserve nonblank thinking blocks after compaction, while the existing recovery path does not match the reported invalid-signature error. The exact live corruption trigger still needs a saved transcript or Anthropic reproduction, and the requested strip-during-compaction behavior conflicts with current Claude prompt-cache preservation policy.

## What This Changes

The issue reports that compacted Anthropic reasoning sessions can replay corrupted thinking blocks and make every later request fail until the session is restarted.

## Best Possible Solution

Decide a Claude 4.6 replay policy that preserves valid provider-owned thinking only when byte-stable, strips or self-heals invalid compacted historical thinking, and adds regression coverage for the reported Anthropic invalid-signature rejection.

## Reproduction Assessment

Yes at source level, but not live: current main can preserve nonblank Claude Sonnet 4.6 thinking blocks after compaction, and the recovery regex does not cover the reported invalid-signature text. I did not establish the exact provider-side corruption trigger without a real malformed transcript or Anthropic credentials.

## Solution Assessment

Unclear: stripping historical thinking during compaction would likely stop this failure, but current code intentionally preserves newer Claude thinking blocks for cache/context behavior. A narrower repair should target compacted or rejected invalid replay history rather than silently changing all valid Claude replay.
