## Summary

Keep open: current main and v2026.6.6 still silently skip durable memory hits from short-term recall, and the linked diagnostic fix is open at https://github.com/openclaw/openclaw/pull/92745 rather than merged.

## What This Changes

The issue asks whether live memory_search should feed deep-sleep short-term recall signals when the recall store only contains session-corpus daily entries and deep promotion stays at zero.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/92745 for diagnostics, then separately decide whether durable memory hits should ever feed short-term promotion.

## Reproduction Assessment

Partly, from source inspection. Current main reproduces why durable MEMORY.md and arbitrary memory/*.md hits do not create short-term recall entries and why low-score one-day session-corpus entries rank zero, but the reporter's full live MCP workspace path was not replayed.

## Solution Assessment

Unclear for the product behavior. The linked diagnostic PR is the narrowest current fix for the silent skip, but deciding whether durable memory hits should feed short-term promotion needs maintainer policy review.
