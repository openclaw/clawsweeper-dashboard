## What This Changes

The issue asks whether live `memory_search` recall hits should feed short-term recall so dreaming deep-sleep can promote durable memories instead of remaining stuck with session-corpus-only candidates.

## Best Possible Solution

Keep this issue open as the canonical user report until the linked memory-core PR either lands or maintainers decide the eligibility/diagnostic policy should take a different shape.

## Reproduction Assessment

Yes from source at medium confidence: `memory_search` calls the recall recorder, but the recorder only admits paths passing `isShortTermMemoryPath`, so durable `MEMORY.md` and non-date `memory/*.md` hits are silently skipped. I did not run a live multi-day memory setup.

## Solution Assessment

Unclear: the linked PR’s diagnostic-only path is a plausible narrow fix, but maintainers still need to decide whether durable `MEMORY.md` and nested `memory/*.md` hits should remain excluded or become eligible short-term signals.
