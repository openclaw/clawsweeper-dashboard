## Summary

Keep open for normal maintainer handling: this maintainer-authored draft has a protected label, and the patch appears like a focused, correct hardening change, but it intentionally changes compatibility by hiding invalid-schema tools from MCP list/call.

## What This Changes

The PR updates the shared MCP tool handler and tests to materialize tool descriptors through runtime schema projection, validate them with the MCP SDK `ToolSchema`, and quarantine invalid schemas from list/call.

## Best Possible Solution

Land only after maintainer acceptance that this shared MCP helper should consistently expose and call only tools that satisfy runtime projection plus the MCP `ToolSchema` contract.

## Reproduction Assessment

Yes at source level: current main evaluates `"type" in params` while listing tools, so a proxy `has` trap for `type` can throw before the MCP response is built. The PR body also includes before/after terminal output, but this read-only review did not rerun it.

## Solution Assessment

Yes, with a maintainer compatibility decision: reusing runtime schema projection and validating against the MCP SDK `ToolSchema` is the narrow contract-boundary fix. The safer alternative, if compatibility must dominate, is an explicit tested compatibility mode rather than silently listing invalid tools.
