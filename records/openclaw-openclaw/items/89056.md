## What This Changes

The PR updates the standalone plugin tools MCP bridge to materialize plugin tool schemas through the runtime projection path, validate descriptors against the MCP SDK `ToolSchema`, quarantine invalid tools from list/call, and add regression coverage.

## Best Possible Solution

Land after CI and explicit maintainer acceptance of hiding invalid-schema tools, keeping list and call backed by the same validated materialized tool set.

## Solution Assessment

Yes: reusing the existing runtime schema projection and then validating against the MCP SDK `ToolSchema` is the narrow owner-boundary fix. The only maintainer choice is whether the intentional invalid-schema quarantine is acceptable for existing standalone MCP clients.
