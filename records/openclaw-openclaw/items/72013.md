## Summary

Keep open. Current main still lets persisted one-shot ACP records with pending stable identities enter startup reconciliation and the warning path; the focused source PR #72023 addressed this but was closed unmerged, and no open duplicate currently tracks the remaining fix.

## What This Changes

Startup ACP identity reconciliation should ignore terminal one-shot ACP records whose persisted identity remains pending so gateway restart warnings do not imply current runtime failure.

## Best Possible Solution

Land a narrow startup guard that skips `mode === "oneshot"` records before `checked` increments, with regression coverage and a changelog entry; handle any legacy closed-state cleanup separately if typed evidence supports it.

## Reproduction Assessment

Yes. A unit fixture can have `listAcpSessionEntries` return ACP metadata with `mode: "oneshot"`, `identity.state: "pending"`, and `acpxSessionId`; current main makes that record eligible for reconcile work and warning output without needing a live ACP backend.

## Solution Assessment

Yes. Skipping one-shot records at startup is the narrowest maintainable fix for historical pending metadata; resolving future one-shot identities after turns does not clean up older persisted records.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/72013 with a narrow ACP startup-only change. In `AcpSessionManager.reconcilePendingSessionIdentities`, skip persisted ACP records whose `session.acp.mode === "oneshot"` before resolving identity and before `checked` increments, so historical one-shot records with pending stable identities do not trigger misleading startup reconcile warnings. Add a focused regression test in `src/acp/control-plane/manager.test.ts` where `listAcpSessionEntries` returns a one-shot ACP meta object with `identity.state: "pending"` and an `acpxSessionId`, and assert the result is `{ checked: 0, resolved: 0, failed: 0 }` with no runtime ensure/status calls. Add a single-line `CHANGELOG.md` fix entry for the user-facing warning cleanup. Treat closed PR #72023 as useful source context, but do not reintroduce a `state === "closed"` guard unless the current `SessionAcpMeta` type/data contract is explicitly updated or a maintainer asks for legacy closed-state handling. Keep the change limited to ACP startup reconcile, its unit test, and changelog.
