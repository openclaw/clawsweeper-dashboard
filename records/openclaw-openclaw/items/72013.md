## Summary

Keep open. Current main and the latest release still count persisted ACP one-shot records with pending stable identities during startup reconciliation, so the reported warning path remains source-reproducible; the focused linked fix was closed unmerged and the adjacent open ACP pressure PR does not suppress the startup warning.

## What This Changes

The issue asks startup ACP identity reconciliation to skip terminal one-shot ACP session records, or otherwise treat unresolved one-shot identities as non-fatal cleanup instead of warning as a current ACP failure.

## Best Possible Solution

Persisted one-shot ACP records should be classified as terminal for the startup scan before checked increments, while active persistent pending identities remain eligible for reconciliation.

## Reproduction Assessment

Yes, source-reproducible. A persisted ACP fixture with mode "oneshot", identity.state "pending", and acpxSessionId satisfies the current predicate and will be counted by startup reconciliation, which then drives the gateway warning when resolution fails.

## Solution Assessment

Yes. Skipping mode === "oneshot" before startup pending-identity reconciliation is the narrowest maintainable fix; changing ensure identity merge semantics or adding a typed closed-state guard would need separate evidence.
