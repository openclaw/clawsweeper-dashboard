## Summary

Keep open. Current main still lets pending one-shot ACP session metadata enter startup identity reconciliation and the gateway warning path; the focused source PR #72023 was closed unmerged, so no landed fix currently covers the reported stale metadata case.

## What This Changes

Startup ACP identity reconciliation should ignore terminal one-shot ACP session records so gateway boot does not warn for stale pending one-shot metadata.

## Best Possible Solution

Land a narrow startup guard that skips `mode === "oneshot"` records before `checked` increments, with regression coverage and a single-line changelog entry.

## Reproduction Assessment

Yes. A unit fixture can have `listAcpSessionEntries` return ACP metadata with `mode: "oneshot"`, `identity.state: "pending"`, and `acpxSessionId`; current main makes that record eligible for reconcile work and warning output without a live ACP backend.

## Solution Assessment

Yes. Skipping one-shot records at startup is the narrowest maintainable fix for historical pending metadata; existing post-turn one-shot identity reconciliation helps future successful runs but does not clean older persisted records.

Fix https://github.com/openclaw/openclaw/issues/72013 with a narrow ACP startup repair. In `AcpSessionManager.reconcilePendingSessionIdentities`, skip persisted ACP records whose `session.acp.mode === "oneshot"` before resolving identity or incrementing `checked`, so stale terminal one-shot records with pending identities do not trigger gateway startup reconcile warnings. Add focused regression coverage in `src/acp/control-plane/manager.test.ts` showing a pending one-shot entry with a stable `acpxSessionId` returns `{ checked: 0, resolved: 0, failed: 0 }` and does not call runtime ensure/status. Use PR #72023 as source context, but do not blindly add a `state === "closed"` guard unless current typed or legacy raw metadata evidence supports it. Add a single-line user-facing `CHANGELOG.md` fix entry. Do not broaden ACP identity contracts, change runtime backend readiness behavior, or touch unrelated ACP startup paths.
