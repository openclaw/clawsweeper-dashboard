## Summary

Keep open. Current main still lets pending ACP records with stable ids reach startup reconciliation without filtering terminal one-shot metadata, so the warning can still recur for historical one-shot records; the focused source PR #72023 was closed unmerged.

## What This Changes

Suppress misleading ACP startup identity-reconcile warnings for terminal one-shot or legacy closed ACP session records with pending identities.

## Best Possible Solution

Land a narrow startup-only guard that skips terminal one-shot and legacy closed ACP metadata before `checked` increments, with regression coverage and a single-line changelog entry.

## Reproduction Assessment

Yes. A unit fixture can return ACP metadata with `mode: "oneshot"`, `identity.state: "pending"`, and an `acpxSessionId`; current predicates make that record eligible for reconcile work and warning output.

## Solution Assessment

Yes. Skipping terminal records at startup is the narrowest maintainable fix for historical pending metadata; resolving identities for future oneshot sessions does not clean up already-persisted terminal records.
