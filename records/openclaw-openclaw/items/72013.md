## Summary

Keep open. Current `main` still lets pending one-shot ACP session metadata enter startup identity reconciliation and the gateway warning path; the focused source PR #72023 was closed unmerged and no open duplicate currently tracks the remaining fix.

## What This Changes

Startup ACP identity reconciliation should ignore terminal one-shot ACP session records so gateway boot does not warn for stale pending one-shot metadata.

## Best Possible Solution

Add a narrow startup guard that skips `mode === "oneshot"` records before `checked` increments, with regression coverage and a changelog entry; treat any legacy closed-state cleanup as separate unless current metadata evidence supports it.

## Reproduction Assessment

Yes. A unit fixture can have `listAcpSessionEntries` return ACP metadata with `mode: "oneshot"`, `identity.state: "pending"`, and `acpxSessionId`; current `main` makes that record eligible for reconcile work and warning output without a live ACP backend.

## Solution Assessment

Yes. Skipping one-shot records at startup is the narrowest maintainable fix for historical pending metadata; the existing post-turn one-shot identity reconciliation helps future successes but does not clean older persisted records.
