## Summary

Keep open. Current main still sends pending ACP session records with stable runtime ids through startup identity reconciliation without checking whether the record is terminal one-shot or closed, so the reported warning can still recur for stale one-shot metadata. The linked fix PR was closed unmerged, and the remaining change is narrow enough for a repair PR.

## What This Changes

Prevent ACP startup identity reconciliation from warning on terminal one-shot or closed ACP session metadata whose identity is still pending.

## Best Possible Solution

Add a startup-only guard in AcpSessionManager.reconcilePendingSessionIdentities before checked increments and runtime work so mode:"oneshot" or state:"closed" ACP records are ignored, while preserving persistent-session reconciliation and the existing post-turn one-shot identity resolution path.

## Reproduction Assessment

Yes. A focused unit fixture can mock listAcpSessionEntries with mode:"oneshot", identity.state:"pending", and acpxSessionId or agentSessionId; current main still treats that record as eligible for startup reconciliation.

## Solution Assessment

Yes. The narrow startup skip is the best fix for stale terminal metadata because it avoids warning on historical one-shot records without changing persistent ACP resume behavior or broadening runtime identity semantics.

## Repair Work Prompt

Fix this issue by adding a narrow startup-only skip in AcpSessionManager.reconcilePendingSessionIdentities for terminal ACP records before checked is incremented or runtime work is attempted. Specifically, skip entries whose ACP metadata has mode === "oneshot" or state === "closed" while preserving persistent-session identity reconciliation and the existing post-turn one-shot identity resolution/close behavior. Treat closed PR #72023 as useful source context, but re-check the current main code and tests rather than copying blindly. Add focused regression tests in src/acp/control-plane/manager.test.ts for pending one-shot records with stable ACPX/agent ids and closed records, asserting result.checked remains 0 and runtime ensure/status is not called. Add a user-facing CHANGELOG.md Fixes entry crediting the reporter and source PR context where appropriate. Do not change core startup logging policy beyond making terminal records ineligible for reconciliation, and do not broaden ACP runtime identity semantics.
