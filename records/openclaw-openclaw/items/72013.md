## Summary

Keep open. Current main still routes persisted pending ACP records with stable ACPX or agent session IDs through startup reconciliation without skipping terminal one-shot or closed metadata, so the reported warning can still recur. The focused related PR #72023 is closed unmerged, and related PR #46537 only addresses future ensured identities, not historical pending one-shot records.

## What This Changes

The issue requests that ACP startup identity reconciliation skip or treat terminal one-shot or closed ACP session records as non-warning cleanup when their stored identity remains pending.

## Best Possible Solution

Add a startup-only skip in `AcpSessionManager.reconcilePendingSessionIdentities` before `checked` and runtime work for terminal one-shot and closed ACP metadata, with focused regression tests and a user-facing changelog entry.

## Reproduction Assessment

Yes. A focused unit fixture can mock `listAcpSessionEntries` with `mode: "oneshot"`, `identity.state: "pending"`, and an `acpxSessionId` or `agentSessionId`; current main still treats that entry as eligible for startup reconciliation.

## Solution Assessment

Yes. A startup guard is the narrowest maintainable fix for historical pending one-shot metadata; resolving future ensured identities alone does not stop already-persisted terminal records from being retried on every restart.
