## Summary

Keep open. Current main still lets historical ACP one-shot records with pending stable identities enter startup reconciliation, so the misleading gateway startup warning can still recur; the focused source PR for this was closed unmerged.

## What This Changes

Suppress ACP startup identity-reconcile warnings for terminal one-shot or legacy closed ACP session records with pending identities.

## Best Possible Solution

Land a narrow startup-only guard that skips terminal one-shot ACP metadata before `checked` increments, with focused regression coverage and a single-line changelog fix entry.

## Reproduction Assessment

Yes. A unit fixture can return ACP metadata with `mode: "oneshot"`, `identity.state: "pending"`, and an `acpxSessionId`; current predicates make that record eligible for reconcile work and warning output.

## Solution Assessment

Yes. Skipping terminal records at startup is the narrowest maintainable fix for historical pending metadata; resolving identities for future one-shot sessions does not clean up already-persisted terminal records.
