## Summary

Keep open. Current main and the latest release still let persisted ACP one-shot records with pending stable identities enter startup reconciliation and the warning path; the focused fix in https://github.com/openclaw/openclaw/pull/72023 was closed unmerged, and adjacent ACP work does not supersede this startup skip.

## What This Changes

Startup ACP identity reconciliation should skip terminal one-shot persisted sessions, or otherwise avoid warning when stale one-shot identity metadata remains unresolved.

## Best Possible Solution

Persisted one-shot ACP records should be classified as terminal for the startup scan, while active persistent pending identities remain eligible for reconciliation.

## Reproduction Assessment

Yes, source-reproducible. A unit fixture with `mode: "oneshot"`, `identity.state: "pending"`, and `acpxSessionId` will be counted by current main, and the gateway warning path runs if reconciliation fails.

## Solution Assessment

Yes. Skipping `mode === "oneshot"` before `checked` increments is the narrowest maintainable fix; broad identity migration or a `closed` state guard would need separate evidence.
