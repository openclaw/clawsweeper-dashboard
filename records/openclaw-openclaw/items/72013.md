## Summary

Keep open: current main and the latest release still send persisted one-shot ACP records with pending stable identities through startup reconciliation, so the repeated warning remains source-reproducible.

## What This Changes

The issue asks startup identity reconciliation to skip terminal one-shot ACP records so stale pending identities do not produce repeated gateway warnings.

## Best Possible Solution

Skip `mode === "oneshot"` before startup pending-identity reconciliation increments `checked`, while keeping persistent pending identities with stable ids eligible and adding a focused manager regression test.

## Reproduction Assessment

Yes. A persisted ACP record with `mode: "oneshot"`, `identity.state: "pending"`, and `acpxSessionId` satisfies the current source predicate, increments `checked`, and can drive the gateway warning if runtime reconciliation fails.

## Solution Assessment

Yes. The narrow maintainable fix is to skip one-shot ACP records before startup pending-identity reconciliation and cover that exact behavior with a focused manager test; broader lifecycle state changes should wait for separate evidence.
