## Summary

Keep open. Current main still routes persisted ACP records with pending stable identities through startup reconciliation without checking terminal `oneshot` or `closed` ACP metadata, so the reported warning can still recur for historical records. The linked focused PR #72023 was closed unmerged, and current main only addresses future successful one-shot identity resolution, not already-persisted pending terminal records.

## What This Changes

Prevent ACP startup identity reconciliation from warning on persisted terminal one-shot or closed ACP session metadata whose identity remains pending.

## Best Possible Solution

Keep this issue open and land a narrow startup-only terminal-record guard with focused regression coverage and a user-facing changelog entry, while leaving the existing successful one-shot post-turn identity reconciliation in place.

## Reproduction Assessment

Yes. A focused unit fixture can return an ACP session entry with `mode: "oneshot"` or `state: "closed"`, `identity.state: "pending"`, and `acpxSessionId` or `agentSessionId`; current main's predicates make that record eligible for startup reconciliation and warning output.

## Solution Assessment

Yes. Skipping terminal one-shot/closed records before `checked` increments and runtime work is the narrowest maintainable fix for historical terminal metadata; resolving future successful one-shot identities does not clean up already-persisted pending records.
