## Summary

Keep open. Current main and v2026.5.7 still let persisted ACP one-shot records with pending stable identities enter startup reconciliation and the warning path; the focused linked fix PR was closed unmerged, and the other related ACP work does not replace this narrow startup skip.

## What This Changes

The issue asks ACP startup identity reconciliation to skip terminal one-shot or closed ACP session records so stale pending metadata does not produce misleading warning logs.

## Best Possible Solution

Land a narrow startup guard that skips `mode === "oneshot"` records before `checked` increments, with focused regression coverage and no broader identity-merge or warm-restore policy change.

## Reproduction Assessment

Yes, from source inspection. A unit fixture can return ACP metadata with `mode: "oneshot"`, `identity.state: "pending"`, and `acpxSessionId`; current main will count it for startup reconciliation and the gateway will warn if reconciliation fails.

## Solution Assessment

Yes. Skipping one-shot records before startup reconciliation is the narrowest maintainable fix for stale terminal metadata; changing ensure identities or broad warm-restore behavior would touch unrelated session semantics.
