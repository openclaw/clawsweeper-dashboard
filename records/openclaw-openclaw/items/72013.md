## Summary

Keep open. Current main and v2026.5.18 still count persisted ACP one-shot records with pending stable identities during startup reconciliation, so the reported warning remains source-reproducible; the focused fix in https://github.com/openclaw/openclaw/pull/72023 was closed unmerged and the open pressure PR is adjacent rather than an exact fix.

## What This Changes

Gateway startup should avoid reconciling persisted terminal ACP one-shot sessions with pending identities so stale records do not produce misleading startup warnings.

## Best Possible Solution

Classify persisted `mode: "oneshot"` ACP records as terminal before startup identity reconciliation increments `checked`, with a focused regression test, while leaving active persistent pending identities eligible for reconciliation.

## Reproduction Assessment

Yes, source-reproducible. A persisted ACP record with `mode: "oneshot"`, `identity.state: "pending"`, and `acpxSessionId` satisfies the current startup predicate and increments `checked`, which drives the gateway warning if reconciliation fails.

## Solution Assessment

Yes. Skipping `mode === "oneshot"` before startup pending-identity reconciliation is the narrowest maintainable fix; adding a `closed` guard should wait for separate type or legacy-data evidence.
