## Summary

Keep open. Current main still sends pending ACP one-shot records with stable ACPX or agent session IDs through startup identity reconciliation, so the reported warning can still recur for historical terminal one-shot metadata; the focused related PR is closed unmerged.

## What This Changes

The issue asks OpenClaw to skip or de-noise ACP startup identity reconciliation for terminal one-shot session metadata that remains pending.

## Best Possible Solution

Add a narrow startup-only guard before checked increments and runtime work so persisted terminal one-shot ACP records are skipped, while preserving persistent-session identity reconciliation and adding focused regression coverage plus a user-facing changelog entry.

## Reproduction Assessment

Yes. A focused unit fixture can mock listAcpSessionEntries with mode:"oneshot", identity.state:"pending", and acpxSessionId or agentSessionId; current main still treats that as eligible for startup reconciliation.

## Solution Assessment

Yes. The startup skip is the narrowest maintainable fix for historical pending one-shot metadata; current post-turn one-shot identity resolution helps future successful runs but does not prevent already-persisted pending one-shot records from being retried on every restart.
