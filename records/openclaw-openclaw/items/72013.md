- `P2`: The issue is a normal-priority ACP startup/session-state bug with misleading warnings and a narrow affected runtime path.
- `impact:session-state`: The report is about persisted ACP session identity metadata being reconciled incorrectly during gateway startup.

## Summary

Keep open: current main and latest release still run startup identity reconciliation on pending stable ACP identities without filtering terminal one-shot sessions, and the issue is already paired with an open closing PR that adds the narrow guard and regression test.

## What This Changes

Startup identity reconciliation should skip terminal one-shot ACP sessions with pending identities so gateway restarts do not log misleading reconcile failures.

## Best Possible Solution

Review and land the linked open PR once validated; it should add the one-shot guard and regression coverage while preserving startup reconciliation for persistent pending identities.

## Reproduction Assessment

Yes, as a source-level reproduction: a one-shot ACP record with `identity.state: "pending"` plus `acpxSessionId` passes current main's pending/stable checks and enters startup reconciliation. I did not run a live gateway restart in this read-only review.

## Solution Assessment

Yes, the linked open PR is the narrow maintainable fix shape: skip one-shot records before identity reconciliation while keeping persistent pending identities eligible. A broader startup fanout fix remains separately tracked by https://github.com/openclaw/openclaw/issues/85366 and https://github.com/openclaw/openclaw/pull/85399.
