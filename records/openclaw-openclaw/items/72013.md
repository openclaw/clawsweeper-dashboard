## Summary

Keep open. Current main still scans persisted ACP records with pending stable identities during startup without excluding terminal one-shot or closed ACP metadata, so the reported warning can still recur for historical session records. The focused source PR that addressed this was closed unmerged, and current main only covers future successful one-shot identity reconciliation after a turn.

## What This Changes

Prevent ACP startup identity reconciliation from warning on persisted terminal one-shot or closed ACP session metadata whose identity remains pending.

## Best Possible Solution

Land a narrow startup-only guard in the ACP session manager that excludes terminal one-shot or closed persisted records before `checked` increments or runtime work begins, with focused regression tests and a user-facing changelog entry. Keep the existing post-turn one-shot identity reconciliation in place for newly successful runs.

## Reproduction Assessment

Yes. A focused unit fixture can return an ACP session entry with `mode: "oneshot"` or `state: "closed"`, `identity.state: "pending"`, and `acpxSessionId` or `agentSessionId`; current main's predicates make that record eligible for startup reconciliation and warning output.

## Solution Assessment

Yes. Skipping terminal one-shot/closed records at startup is the narrowest maintainable fix for historical terminal metadata; resolving future successful one-shot identities does not clean up already-persisted pending records.
