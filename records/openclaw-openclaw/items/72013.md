## Summary

Keep open. Current main and the latest release still let persisted ACP one-shot records with pending stable identities enter startup reconciliation and the gateway warning path; the focused fix in https://github.com/openclaw/openclaw/pull/72023 was closed unmerged, and the adjacent open ACP PRs do not replace this narrow startup skip.

## What This Changes

The issue asks OpenClaw to stop warning during gateway startup when terminal ACP one-shot session metadata still has a pending identity.

## Best Possible Solution

Land a narrow startup guard that skips `mode === "oneshot"` records before `checked` increments, with focused regression coverage and no broader identity-merge or warm-restore policy change.

## Reproduction Assessment

Yes from source inspection. A unit fixture can return ACP metadata with `mode: "oneshot"`, `identity.state: "pending"`, and `acpxSessionId`; current main will count it for startup reconciliation and the gateway will warn if reconciliation fails.

## Solution Assessment

Yes. Skipping one-shot records before startup reconciliation is the narrowest maintainable fix for stale terminal metadata; changing ensure identities or broad warm-restore behavior does not directly clean historical one-shot records and can alter unrelated session semantics.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/72013. Current main counts persisted ACP one-shot metadata with `identity.state: "pending"` and a stable ACPX/agent session id inside `AcpSessionManager.reconcilePendingSessionIdentities`, which can trigger the gateway startup warning even though the one-shot session is terminal. Add a narrow guard in `src/acp/control-plane/manager.core.ts` that skips `session.acp.mode === "oneshot"` before `checked` increments or runtime ensure/reconcile work starts. Add focused coverage in `src/acp/control-plane/manager.test.ts` showing a one-shot pending stable-id record returns `{ checked: 0, resolved: 0, failed: 0 }` and does not call runtime `ensureSession`/`getStatus`. Use https://github.com/openclaw/openclaw/pull/72023 as reference, but do not add a `state === "closed"` branch unless typed persisted-data evidence supports widening `SessionAcpMeta.state`. Do not change `createIdentityFromEnsure` merge semantics, warm-restore resolved sessions, runtime cache eviction, or ACPX PID liveness in this fix. Add a user-facing `CHANGELOG.md` fix entry unless maintainers explicitly handle changelog at landing. Coordinate with adjacent open ACP PRs such as https://github.com/openclaw/openclaw/pull/46949 and https://github.com/openclaw/openclaw/pull/79541 only to avoid conflicts; keep this PR narrow.
