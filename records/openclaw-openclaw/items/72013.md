## Summary

Keep open. Current main and the latest release still count pending one-shot ACP records during startup identity reconciliation, so the reported warning remains source-reproducible; the exact fix PR was closed unmerged and the adjacent open ACP pressure PR does not suppress this startup warning.

## What This Changes

The issue asks startup ACP identity reconciliation to skip terminal one-shot ACP session records so stale pending metadata does not produce misleading gateway warnings.

## Best Possible Solution

Add a narrow startup-scan guard that skips `session.acp.mode === "oneshot"` before pending identity reconciliation, while preserving active persistent pending identity reconciliation and adding regression coverage.

## Reproduction Assessment

Yes, source-reproducible. A persisted ACP record with `mode: "oneshot"`, `identity.state: "pending"`, and `acpxSessionId` satisfies the current startup predicate and will be counted before any warning suppression.

## Solution Assessment

Yes. Skipping one-shot records before startup pending-identity reconciliation is the narrow maintainable fix; changing resolved-identity semantics or adding a typed closed-state path would need separate evidence.
