## Summary

Keep open: current main and the latest release still run startup identity reconciliation on pending one-shot ACP metadata, and a live closing PR already owns the narrow fix.

## What This Changes

Skip terminal one-shot ACP session metadata during startup identity reconciliation so stale pending identities do not produce misleading gateway warnings.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/86711 or an equivalent one-shot guard, while keeping persistent pending identities eligible for startup reconciliation.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: a one-shot ACP record with pending identity and `acpxSessionId` passes the current identity filters and increments `checked`. I did not run a live gateway restart in this read-only review.

## Solution Assessment

Yes, the linked PR's guard is the narrowest maintainable fix: skip `mode: "oneshot"` before runtime re-ensure while preserving persistent-session reconciliation and adding a regression test.
