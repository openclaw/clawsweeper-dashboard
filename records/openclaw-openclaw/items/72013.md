## Summary

Keep open. Current main still routes persisted ACP records with `identity.state: "pending"` plus a stable ACPX or agent session id through startup identity reconciliation without checking whether the record is a terminal one-shot or closed session, so the reported warning can still recur for historical metadata. The related focused PR is closed unmerged, and the newer one-shot identity resolution path helps future successful runs but does not skip already-persisted pending terminal records.

## What This Changes

Startup identity reconciliation should skip terminal one-shot or closed ACP session metadata so stale pending identities do not produce repeated gateway warnings.

## Best Possible Solution

Add a startup-only terminal-record guard before `checked` increments and runtime work, preserving persistent-session identity reconciliation while adding focused regression coverage for pending stable one-shot and closed ACP records plus a user-facing changelog entry.

## Reproduction Assessment

Yes. A focused unit fixture can mock `listAcpSessionEntries` with `mode: "oneshot"`, `identity.state: "pending"`, and `acpxSessionId` or `agentSessionId`; current main's predicates make that record eligible for startup reconciliation.

## Solution Assessment

Yes. The requested startup skip is the narrowest maintainable fix for historical pending terminal metadata; resolving future one-shot identities after successful runs does not stop already-persisted pending one-shot records from being retried on restart.
