## Summary

Keep open. Current main still sends pending ACP records with stable ids through startup reconciliation without filtering terminal one-shot metadata, so the reported warning can still recur for historical records; the focused source PR #72023 was closed unmerged.

## What This Changes

Suppress misleading ACP startup identity-reconcile warnings for terminal one-shot or legacy closed ACP session records with pending identities.

## Best Possible Solution

Land a narrow startup-only guard that skips terminal one-shot and legacy closed ACP metadata before `checked` increments, with focused regression tests and a changelog entry while preserving active persistent-session reconciliation.

## Reproduction Assessment

Yes. A focused unit fixture can have `listAcpSessionEntries` return ACP metadata with `mode: "oneshot"`, `identity.state: "pending"`, and an `acpxSessionId`; current predicates make that record eligible for reconcile work and startup warning output.

## Solution Assessment

Yes. Skipping only terminal records at startup is the narrowest maintainable fix for historical pending metadata; resolving identities for future sessions does not clean up already-persisted terminal records.
