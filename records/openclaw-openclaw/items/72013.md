## Summary

Keep open. Current `main` and the latest release still route persisted ACP one-shot records with `identity.state: "pending"` plus a stable session id through startup identity reconciliation, so the reported warning remains source-reproducible; the focused fix at https://github.com/openclaw/openclaw/pull/72023 was closed unmerged, and the broader open pressure-cleanup PR does not own this exact startup-warning fix.

## What This Changes

The issue requests that ACP startup identity reconciliation skip terminal one-shot ACP session metadata before counting it as a failed pending-identity reconcile.

## Best Possible Solution

Persisted one-shot ACP records should be classified as terminal for startup identity reconciliation before `checked` increments, while persistent pending identities with stable ids remain eligible for reconciliation.

## Reproduction Assessment

Yes, source-reproducible. A persisted ACP record with `mode: "oneshot"`, `identity.state: "pending"`, and `acpxSessionId` satisfies the current predicate, increments `checked`, and can drive the gateway warning when runtime reconciliation fails.

## Solution Assessment

Yes, the narrow fix direction is the best fit: skip `mode === "oneshot"` before startup pending-identity reconciliation and cover it with a focused manager test. A broader identity-state migration or closed-state model should wait for separate evidence.
