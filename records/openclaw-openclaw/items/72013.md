## Summary

Keep open: current main and the latest release still run ACP startup identity reconciliation on pending one-shot session metadata, while https://github.com/openclaw/openclaw/pull/86711 is the open closing PR that owns the focused fix.

## What This Changes

The issue asks ACP startup identity reconciliation to skip terminal one-shot ACP sessions so stale pending identity metadata does not produce misleading startup warnings.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/86711 or an equivalent one-shot guard before checked is incremented, while keeping persistent pending identities eligible for startup reconciliation.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: a one-shot ACP record with pending identity and a stable acpxSessionId passes current main's pending/stable filters and reaches checked because there is no mode guard; I did not run a live gateway restart in this read-only review.

## Solution Assessment

Yes, the linked fix shape is the narrowest maintainable path: skip mode "oneshot" before runtime re-ensure/reconcile, without changing persistent-session reconciliation or adding a new config surface.
