## Summary

Keep open. Current main still lets persisted ACP one-shot records with pending stable identities enter startup reconciliation and the gateway warning path, while the focused reference fix remains closed unmerged and no newer open PR replaces this narrow core behavior.

## What This Changes

The issue asks startup ACP identity reconciliation to skip or quietly ignore terminal one-shot session metadata so stale pending records do not produce misleading gateway warnings.

## Best Possible Solution

Land a narrow startup guard that skips `session.acp.mode === "oneshot"` before `checked` increments, with focused regression coverage and without broad identity migration or warm-restore policy changes.

## Reproduction Assessment

Yes, from source inspection. A unit fixture with ACP metadata `mode: "oneshot"`, `identity.state: "pending"`, and `acpxSessionId` will pass the current reconcile predicate, increment `checked`, and allow the gateway warning if reconciliation fails.

## Solution Assessment

Yes. Skipping one-shot records before startup reconciliation is the narrowest maintainable fix for stale terminal one-shot metadata; resolving all historical identities or adding a `closed` state would be broader and less supported by current types.
