## Summary

Keep open. Current main still preserves config-over-global plugin precedence and records only generic warning-level duplicate diagnostics while the install, doctor, and inspect surfaces do not yet provide the requested inactive-source warning; #76620 is an open unmerged fix linked with closing syntax.

## What This Changes

The issue asks for plugin install, doctor, and inspect diagnostics that warn when an npm/global plugin install is shadowed by an explicit config-selected/local plugin with the same id.

## Best Possible Solution

Review and land #76620 or an equivalent narrow patch that adds install, doctor, and inspect diagnostics while preserving existing duplicate-precedence semantics.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure a config/local source and an installed npm/global source with the same plugin id, then config precedence wins while the install, doctor, and inspect paths omit the requested shadowing repair guidance.

## Solution Assessment

Yes. The narrow maintainable solution is to surface the existing registry/install-record state in CLI diagnostics without changing precedence or auto-editing user config; the linked open PR is the current implementation candidate.
