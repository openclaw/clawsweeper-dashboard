## What This Changes

The PR changes the shared doctor empty-allowlist scan so top-level group allowlist warnings can be skipped when enabled sub-accounts exist, and adds regression coverage for the reporter scenario plus the no-account fallback.

## Best Possible Solution

Keep the narrow doctor scanner fix, but make the enabled-account predicate a concrete boolean or `undefined` before passing it to the warning builder.

## Reproduction Assessment

Yes. Current `main` calls the scanner on the top-level channel record before iterating accounts, so an empty parent allowlist can emit the warning independently of populated sub-accounts.

## Solution Assessment

No as submitted. The scanner-layer approach is appropriate, but this implementation must fix the introduced nullable boolean type error before it is mergeable.
