## Summary

Keep open: the PR fixes the reported parent-scope false positive for the simple named-account shape, but the latest head can also suppress the top-level Telegram warning for existing configs that still have an implicit default account from top-level credentials.

## What This Changes

The branch adds a doctor empty-allowlist scan skip for top-level group warnings and Telegram extra warnings when all non-disabled sub-accounts have their own allowlists, plus regression tests.

## Best Possible Solution

Gate parent-warning suppression on the same account-selection contract that includes implicit default accounts, preserve parent extra warnings when that default is active, and add regression coverage for literal and SecretRef credentials.

## Reproduction Assessment

Yes. Source inspection of PR head shows the skip flag can be set from configured accounts alone, while current Telegram account selection adds an implicit default account from top-level credentials that the scanner never iterates.

## Solution Assessment

No. The scanner is the right layer, but accounts.default-only detection is too narrow; the fix should follow the existing channel account-selection and SecretInput credential contracts.
