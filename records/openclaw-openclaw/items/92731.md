## What This Changes

The PR changes doctor empty-allowlist scanning to suppress top-level group allowlist warnings and Telegram extra warnings when enabled sub-accounts provide effective sender allowlists, with scanner regression tests.

## Best Possible Solution

Land one canonical scanner fix that distinguishes parent-only fallback records from active top-level accounts, then keep the reporter false-positive suppressed without losing real default-account warnings.

## Reproduction Assessment

Yes. Source inspection shows the PR can pass the parent skip when all named accounts have lists, while current Telegram account selection still treats top-level credentials as an enabled `default` account.

## Solution Assessment

No. The scanner layer is a reasonable place to fix the false positive, but the skip predicate must first distinguish parent-only fallback records from active top-level accounts.
