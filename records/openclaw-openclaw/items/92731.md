## Summary

Keep open: the latest head fixes the earlier Telegram extra-warning gap, but the new skip condition is still too broad and can hide doctor group-allowlist diagnostics for active root/default accounts that coexist with named accounts.

## What This Changes

This PR adds a skip flag to doctor empty-allowlist scanning so parent-scope group allowlist warnings and plugin extra warnings are skipped when enabled sub-accounts exist, plus two regression tests.

## Best Possible Solution

Preserve the parent-fallback false-positive fix while still scanning active root/default accounts, then land one canonical fix for the linked issue with focused regression coverage.

## Reproduction Assessment

Yes. Current main source shows the scanner checks the top-level channel record before accounts, and the PR regression is source-reproducible because the new skip flag treats any enabled account as proof that the top-level record is parent-only.

## Solution Assessment

No. The scanner layer is the right place to fix phantom parent warnings, but this implementation is too broad because current account helpers can keep a root/default account active alongside named accounts.
