This PR changes the doctor empty-allowlist scanner to skip top-level group-allowlist and plugin extra warnings for parent channel records with active sub-accounts, and adds regression tests.

## Best Possible Solution

Replace the generic credential heuristic with credential detection that follows the channel/account-selection and SecretInput contracts, then cover literal, SecretRef, env-only, and noncredential parent settings in focused tests.

## Reproduction Assessment

Yes. Source inspection of PR head shows the parent skip is driven by a heuristic that misses canonical SecretRef credentials and env-only implicit default accounts while the current Telegram account-selection contract treats them as default-account credentials.

## Solution Assessment

No. The scanner is the right layer for the reported false warning, but the current heuristic should be replaced with explicit credential/SecretInput semantics or a channel-owned active-parent decision instead of inferring from arbitrary string/object fields.
