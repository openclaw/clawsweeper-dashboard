## Summary

Keep this PR open, but do not merge it as-is: the provider file-backed auth idea remains unimplemented on current `main`, while the daemon-token portion has been superseded and the branch still has blocking credential-handling regressions.

## What This Changes

The PR adds a shared `*_FILE` secret-env resolver for provider/model auth and changes daemon gateway installs/audits to avoid embedding gateway tokens in service environments.

## Best Possible Solution

Rewrite this as a focused current-main provider-secret change that preserves direct-env precedence, keeps key-list separators intact, excludes `_FILE` names from prefix scans, reports unreadable configured files, and leaves the newer daemon token flow in place.

## Reproduction Assessment

Yes. Source inspection shows current `main` still never consults `ANTHROPIC_API_KEY_FILE` or `OPENAI_API_KEYS_FILE`, and the PR defects are visible in the proposed resolver and daemon persistence diff.

## Solution Assessment

No. The `*_FILE` direction may be useful, but this PR is not the best current solution because it mixes in obsolete daemon-token edits and introduces credential rotation, parsing, and diagnostics problems.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Honor explicit install tokens during rotation:** `src/cli/daemon-cli/install.ts:47-48`
  - body: When `openclaw gateway install --token <new-token>` runs against a config that already has `gateway.auth.token`, this early return keeps the old config token and reports persistence success. The later install path then omits the service env token, so the operator's explicit rotation is ignored.
  - confidence: 0.88
- **[P2] Preserve separators for file-backed key lists:** `src/agents/live-auth-keys.ts:111-114`
  - body: `<PROVIDER>_API_KEYS` now flows through `resolveSecretEnvValue()`, which normalizes file contents before `parseKeyList()` can split them. A file containing `key1\nkey2` becomes one concatenated invalid key, breaking multi-key rotation.
  - confidence: 0.84
- **[P3] Report unreadable configured secret files:** `src/utils/secret-env.ts:25-32`
  - body: If a `_FILE` env var points at a typo or unreadable path, all read errors are swallowed and the resolver returns `null`, which is indistinguishable from no configuration. Operators then get a later auth failure or fallback credential without a useful diagnostic.
  - confidence: 0.74
