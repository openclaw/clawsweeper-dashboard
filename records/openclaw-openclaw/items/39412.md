## Summary

Keep this PR open, but it is not mergeable as-is: the provider secret-file part is unique work, while the daemon-token part is stale against current main and the diff introduces concrete credential-handling regressions without real behavior proof.

## What This Changes

The PR adds `*_FILE` provider auth env-var loading, changes daemon gateway installs/audits to prefer config-backed tokens, and adds auth/daemon tests.

## Best Possible Solution

Rework this against current main as a narrower credential-loading change that uses the existing SecretRef/auth resolver contracts, preserves list parsing and token rotation semantics, and includes redacted real setup proof.

## Reproduction Assessment

Yes for the review findings: source inspection of the PR diff shows the token-rotation early return, newline separator loss, and silent file-read failure paths. The missing piece is after-fix proof from a real setup.

## Solution Assessment

No. The daemon portion should integrate with current `resolveGatewayInstallToken()` behavior, and provider file-backed secrets should be reconciled with SecretRef policy instead of adding a silent scalar env-file resolver.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Honor explicit install tokens during rotation:** `src/cli/daemon-cli/install.ts:47-48`
  - body: When `openclaw gateway install --token <new-token>` runs with an existing `gateway.auth.token`, this early return keeps the old config token and reports persistence success. The later install plan can omit the service env token, so the user-supplied rotation is ignored.
  - confidence: 0.88
- **[P2] Preserve separators for file-backed key lists:** `src/agents/live-auth-keys.ts:111-114`
  - body: `<PROVIDER>_API_KEYS` now goes through `resolveSecretEnvValue()` before `parseKeyList()`. A file containing newline-separated keys is normalized into one concatenated invalid key, breaking multi-key rotation.
  - confidence: 0.84
- **[P3] Report unreadable configured secret files:** `src/utils/secret-env.ts:31-32`
  - body: If a `_FILE` env var points at a typo or unreadable path, the resolver swallows the read error and returns `null`, leaving operators with a later auth failure or fallback credential and no clue that the configured secret file failed to load.
  - confidence: 0.74
