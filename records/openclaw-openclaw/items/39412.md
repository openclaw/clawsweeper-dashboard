## Summary

Keep open for maintainer/security review: the provider file-backed auth work is still unique, but the daemon-token half is superseded by current main and the branch has concrete credential-handling regressions plus no real behavior proof.

## What This Changes

The PR adds provider `*_FILE` env-var secret loading and rewrites daemon gateway token install/audit behavior toward config-backed persistence.

## Best Possible Solution

Replace this with a focused provider-secret loading design aligned with SecretRef policy, preserving direct-env precedence, list separators, `_FILE` prefix-scan exclusion, operator-visible diagnostics, and current main's daemon-token flow.

## Reproduction Assessment

Yes. Source inspection is enough to reproduce the patch defects: PR head drops explicit gateway token rotation, normalizes file-backed key lists before splitting, and silently swallows configured secret-file read errors.

## Solution Assessment

No. The provider `_FILE` direction may be useful, but this PR mixes it with stale daemon-token internals and introduces credential-handling regressions; a narrower provider-secret change is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Honor explicit install tokens during rotation:** `src/cli/daemon-cli/install.ts:47-48`
  - body: When `openclaw gateway install --token <new-token>` runs with an existing `gateway.auth.token`, this early return keeps the old config token while reporting persistence success. The later install plan can omit the service env token, so the operator's explicit rotation is ignored.
  - confidence: 0.88
- **[P2] Preserve separators for file-backed key lists:** `src/agents/live-auth-keys.ts:111-114`
  - body: `<PROVIDER>_API_KEYS` now flows through `resolveSecretEnvValue()` before `parseKeyList()`. A file containing `key1\nkey2` can become one concatenated invalid key, breaking multi-key rotation.
  - confidence: 0.84
- **[P3] Report unreadable configured secret files:** `src/utils/secret-env.ts:31-32`
  - body: If a `_FILE` env var points at a typo or unreadable path, all read errors are swallowed and the resolver returns `null`, which is indistinguishable from no configuration and leaves operators with a later auth failure or fallback credential.
  - confidence: 0.74

## Security Review

Status: needs_attention

Concerns:

  - body: The proposed daemon install path can keep an existing config token while treating persistence as successful, causing an explicit `--token` rotation to be dropped and leaving the old gateway credential active.
  - confidence: 0.88
