## Summary

Keep this PR open for maintainer follow-up, but do not merge it as-is. Current main already covers the daemon gateway-token persistence and tokenless service-audit behavior, while the provider `*_FILE` auth convention remains absent; the PR branch also has concrete correctness/security findings around token rotation and file-backed key-list parsing.

## What This Changes

The PR adds ambient provider `*_FILE` secret loading, wires it into model/provider auth paths, and changes daemon gateway installs and audits to avoid embedding gateway tokens in service environments.

## Best Possible Solution

Rebase or rewrite the useful provider-secret portion against current main, drop the now-obsolete daemon gateway-token persistence edits, and implement ambient `*_FILE` handling through a focused shared resolver that preserves direct-env precedence, keeps list separators intact, excludes `*_FILE` names from prefixed key collection, reports unreadable configured files, and covers model auth, live-key rotation, and provider usage auth tests.

## Reproduction Assessment

Yes. The current-main gap is reproducible by code inspection with a focused unit case where only `ANTHROPIC_API_KEY_FILE` or `OPENAI_API_KEYS_FILE` is set; the proposed patch defects are also traceable through the added `persistGatewayTokenToConfig()` and list-var resolver paths.

## Solution Assessment

No. The provider `*_FILE` direction is useful, but this PR is not the best current fix because the daemon half is superseded by main and the branch introduces blocking token-rotation and key-list parsing bugs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Honor explicit install tokens during rotation:** `src/cli/daemon-cli/install.ts:47-48`
  - body: When `openclaw gateway install --token <new-token>` runs against a config that already has `gateway.auth.token`, this early return keeps the old config token and reports persistence success. The later install path then omits the service env token, so the operator's explicit rotation is ignored.
  - confidence: 0.88
- **[P2] Preserve separators for file-backed key lists:** `src/agents/live-auth-keys.ts:111-114`
  - body: `<PROVIDER>_API_KEYS` now flows through `resolveSecretEnvValue()`, which normalizes file contents before `parseKeyList()` can split them. A file containing `key1\nkey2` becomes one concatenated invalid key, breaking multi-key rotation.
  - confidence: 0.84
- **[P3] Report unreadable configured secret files:** `src/utils/secret-env.ts:25-32`
  - body: If a `_FILE` env var points at a typo or unreadable path, all read errors are swallowed and the resolver returns `null`, which is indistinguishable from no configuration. Operators then get a later auth failure or fallback credential without a useful diagnostic.
  - confidence: 0.74
