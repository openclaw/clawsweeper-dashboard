## Summary

Keep this PR open but not merge-ready. The provider `*_FILE` credential idea is not on current main, while the daemon-token half has been superseded by the landed gateway-token flow from https://github.com/openclaw/openclaw/pull/28428, and this branch still has credential-handling regressions plus no real behavior proof.

## What This Changes

The PR adds `*_FILE` fallback support for provider API-key env vars, changes daemon gateway install/audit behavior toward config-backed tokens, and adds related tests.

## Best Possible Solution

Land a narrower provider-secret change that aligns with the existing SecretRef/secret-input contract, preserves list separators and diagnostics, and leaves current main's daemon-token flow intact.

## Reproduction Assessment

Yes. Source inspection of this PR's diff reproduces the introduced defects: existing config tokens can suppress explicit token rotation, file-backed lists normalize away separators, and configured file-read failures are swallowed.

## Solution Assessment

No. The provider file-backed env idea may be useful, but this branch mixes it with stale daemon-token internals and should be narrowed to the credential-loading surface with SecretRef-aligned behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Honor explicit install tokens during rotation:** `src/cli/daemon-cli/install.ts:47-48`
  - body: When `openclaw gateway install --token <new-token>` runs with an existing `gateway.auth.token`, this early return treats the old config token as already persisted. The later install plan omits the service env token, so the user-supplied rotation is ignored and the old credential remains active.
  - confidence: 0.88
- **[P2] Preserve separators for file-backed key lists:** `src/agents/live-auth-keys.ts:111-114`
  - body: `<PROVIDER>_API_KEYS` now goes through `resolveSecretEnvValue()` before `parseKeyList()`. A file containing newline-separated keys can be normalized into one concatenated invalid key, breaking multi-key rotation.
  - confidence: 0.84
- **[P3] Report unreadable configured secret files:** `src/utils/secret-env.ts:31-32`
  - body: If a `_FILE` env var points at a typo or unreadable path, the resolver swallows all read errors and returns `null`. Operators then see a later auth failure or fallback credential with no indication that the configured secret file failed to load.
  - confidence: 0.74
