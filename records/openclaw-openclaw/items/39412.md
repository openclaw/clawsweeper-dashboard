## Summary

Keep this PR open, but do not merge it as-is. Current main still lacks ambient provider `*_FILE` env resolution, so the provider-secret half remains useful, while the daemon-token half has been superseded by newer main code and the PR branch still introduces blocking auth and parsing regressions.

## What This Changes

The PR adds a shared `resolveSecretEnvValue` helper, routes provider/model auth through `*_FILE` env fallbacks, and changes daemon gateway installs/audits to avoid embedding gateway tokens in service environments.

## Best Possible Solution

Keep the useful provider-secret idea, but rewrite it as a focused current-main change: add a shared resolver that preserves direct-env precedence, keeps list separators intact for key rotation, excludes `_FILE` variable names from prefixed-key collection, surfaces unreadable configured files, and leaves the newer daemon gateway token flow in place.

## Reproduction Assessment

Yes. The remaining current-main gap is reproducible by code inspection: `ANTHROPIC_API_KEY_FILE` and `OPENAI_API_KEYS_FILE` are never consulted by `resolveEnvApiKey()` or `collectProviderApiKeys()`, while the PR defects are visible in the added resolver and token persistence diff.

## Solution Assessment

No. The provider `*_FILE` direction may be useful, but this PR is not the best current solution because its daemon persistence code is superseded by main and its provider resolver breaks multi-key list files and hides configured file-read failures.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Honor explicit install tokens during rotation:** `src/cli/daemon-cli/install.ts:47-48`
  - body: When `openclaw gateway install --token <new-token>` runs against a config that already has `gateway.auth.token`, this early return keeps the old config token and reports persistence success. The later install path then omits the service env token, so the operator's explicit rotation is ignored.
  - confidence: 0.88
- **[P2] Preserve separators for file-backed key lists:** `src/agents/live-auth-keys.ts:111-114`
  - body: `<PROVIDER>_API_KEYS` now flows through `resolveSecretEnvValue()`, which normalizes file contents before `parseKeyList()` can split them. A file containing `key1\nkey2` becomes one concatenated invalid key, breaking multi-key rotation.
  - confidence: 0.84
- **[P3] Report unreadable configured secret files:** `src/utils/secret-env.ts:25-32`
  - body: If a `_FILE` env var points at a typo or unreadable path, all read errors are swallowed and the resolver returns `null`, which is indistinguishable from no configuration. Operators then get a later auth failure or fallback credential without a useful diagnostic.
  - confidence: 0.74
