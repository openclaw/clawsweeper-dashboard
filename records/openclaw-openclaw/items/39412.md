## Summary

Keep this PR open because the provider `*_FILE` idea is not implemented on current main, but the branch is not mergeable as-is: it has blocking credential-handling defects, missing real behavior proof, and daemon-token changes that are stale against current main's shared resolver.

## What This Changes

The PR adds provider `*_FILE` secret env-var lookup, changes daemon gateway install/audit behavior to avoid embedding tokens in service env, and adds related auth/audit tests.

## Best Possible Solution

Land a narrower credential-loading change only if maintainers want provider `*_FILE` env compatibility, preserving list separators, surfacing configured file-read failures, and keeping current main's shared daemon-token resolver intact.

## Reproduction Assessment

Yes. Source inspection of the PR diff shows the token-rotation early return, newline separator loss, and silent file-read failure paths, while current-main source inspection confirms provider `_FILE` env lookup is not implemented.

## Solution Assessment

No. The provider `_FILE` direction may be useful, but this implementation conflates scalar and list secret normalization, hides configured file errors, and reworks daemon-token behavior that current main already centralizes differently.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Honor explicit install tokens during rotation:** `src/cli/daemon-cli/install.ts:47-48`
  - body: When `openclaw gateway install --token <new-token>` runs with an existing `gateway.auth.token`, this early return keeps the old config token and reports persistence success. The later install plan can omit the service env token, so the user-supplied rotation is ignored.
  - confidence: 0.88
- **[P2] Preserve separators for file-backed key lists:** `src/agents/live-auth-keys.ts:111-114`
  - body: `<PROVIDER>_API_KEYS` now goes through `resolveSecretEnvValue()` before `parseKeyList()`. A file containing newline-separated keys is normalized into one concatenated invalid key, breaking multi-key rotation.
  - confidence: 0.84
- **[P3] Report unreadable configured secret files:** `src/utils/secret-env.ts:31-32`
  - body: If a `_FILE` env var points at a typo or unreadable path, the resolver swallows the read error and returns `null`. Operators then see a later auth failure or fallback credential with no indication that the configured secret file failed to load.
  - confidence: 0.74

## Security Review
