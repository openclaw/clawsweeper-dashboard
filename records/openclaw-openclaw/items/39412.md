## Summary

Keep this PR open, but it is not merge-ready. Current main still lacks provider `*_FILE` auth support, while the daemon token-persistence half has already moved to a different mainline implementation and the PR introduces credential-rotation, key-list parsing, and diagnostics defects.

## What This Changes

The PR adds `*_FILE` provider auth env resolution, skips `_FILE` names in live key scans, changes daemon gateway token persistence, and updates daemon/auth tests.

## Best Possible Solution

If maintainers want `*_FILE` provider auth, land it as a focused current-main change that preserves list separators, reports unreadable configured files, documents the new env contract, and leaves the newer daemon token flow intact.

## Reproduction Assessment

Yes. Source inspection of current main shows direct env-only provider key lookup, and the PR diff shows the token rotation, list parsing, and unreadable-file behavior without needing a full runtime repro.

## Solution Assessment

No. The `*_FILE` direction may be useful, but this PR is not the best current solution because it mixes already-superseded daemon-token edits with provider auth and has blocking credential-handling bugs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Honor explicit install tokens during rotation:** `src/cli/daemon-cli/install.ts:47-48`
  - body: When `openclaw gateway install --token <new-token>` runs against a config that already has `gateway.auth.token`, this early return keeps the old config token and reports persistence success. The later install path then omits the service env token, so the operator's explicit rotation is ignored.
  - confidence: 0.88
- **[P2] Preserve separators for file-backed key lists:** `src/agents/live-auth-keys.ts:111-114`
  - body: `<PROVIDER>_API_KEYS` now flows through `resolveSecretEnvValue()`, which normalizes file contents before `parseKeyList()` can split them. A file containing `key1\nkey2` becomes one concatenated invalid key, breaking multi-key rotation.
  - confidence: 0.84
- **[P3] Report unreadable configured secret files:** `src/utils/secret-env.ts:25-32`
  - body: If a `_FILE` env var points at a typo or unreadable path, all read errors are swallowed and the resolver returns `null`, which is indistinguishable from no configuration. Operators then get a later auth failure or fallback credential without a useful diagnostic.
  - confidence: 0.74

## Security Review
