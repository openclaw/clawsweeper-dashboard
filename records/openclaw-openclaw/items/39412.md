## Summary

Keep open: the provider `*_FILE` capability is still absent from current `main`, but this PR is not mergeable as-is because its daemon-token changes are superseded by newer main code and the proposed secret resolver introduces concrete auth/rotation regressions.

## What This Changes

The PR adds file-backed provider auth env var support and rewrites daemon gateway installs/audits to persist or omit gateway tokens via config-backed flows.

## Best Possible Solution

Keep the useful provider-secret idea, but rewrite it against current main with direct-env precedence, preserved list separators, visible configured-file diagnostics, `_FILE` exclusion from prefix scans, and no replacement of the newer daemon token flow.

## Reproduction Assessment

Yes. Source inspection shows current main does not read provider `*_FILE` env vars, and the PR findings are visible in the proposed resolver and daemon-token persistence diff; I did not run tests because this was a read-only review.

## Solution Assessment

No. The provider `*_FILE` direction may be useful, but this PR is not the best current solution because it carries obsolete daemon-token edits and introduces credential rotation, list parsing, and diagnostics problems.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Honor explicit install tokens during rotation:** `src/cli/daemon-cli/install.ts:47-48`
  - body: When `openclaw gateway install --token <new-token>` runs with an existing `gateway.auth.token`, this early return keeps the old config token while reporting persistence success. The later install path then omits the service env token, so the operator's explicit rotation is dropped.
  - confidence: 0.88
- **[P2] Preserve separators for file-backed key lists:** `src/agents/live-auth-keys.ts:111-114`
  - body: `<PROVIDER>_API_KEYS` now flows through `resolveSecretEnvValue()`, which normalizes file contents before `parseKeyList()` can split them. A file containing `key1\nkey2` becomes one concatenated invalid key, breaking multi-key rotation.
  - confidence: 0.84
- **[P3] Report unreadable configured secret files:** `src/utils/secret-env.ts:25-32`
  - body: If a `_FILE` env var points at a typo or unreadable path, all read errors are swallowed and the resolver returns `null`, which is indistinguishable from no configuration. Operators then get a later auth failure or fallback credential without a useful diagnostic.
  - confidence: 0.74

## Security Review
