## Summary

This PR still has a useful unmerged provider secret-loading idea, but it is not mergeable as-is: the daemon-token half is superseded by newer current-main gateway token handling, and the `_FILE` implementation has concrete credential-handling regressions.

## What This Changes

The PR adds `*_FILE` provider auth env-var resolution and changes gateway daemon installs/audits to persist auth tokens in config instead of embedding them in service environments.

## Best Possible Solution

Rewrite this as a focused provider-secret loading change aligned with the current SecretRef model, preserving direct-env precedence, list separators, `_FILE` prefix-scan exclusion, and operator-visible diagnostics for configured secret-file failures.

## Reproduction Assessment

Yes, by source inspection for the review blockers: the PR path for explicit gateway-token rotation, file-backed key-list parsing, and unreadable secret files is visible in the diff and review comments. Current main also shows provider auth remains direct-env only, so the useful feature direction is source-confirmed as not implemented.

## Solution Assessment

No. The provider `_FILE` direction may be worth pursuing, but this PR mixes it with daemon-token code that current main has replaced and introduces credential-handling regressions; a narrower rewrite is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Honor explicit install tokens during rotation:** `src/cli/daemon-cli/install.ts:47-48`
  - body: When `openclaw gateway install --token <new-token>` runs with an existing `gateway.auth.token`, this early return keeps the old config token while reporting persistence success. The later install path can then omit the service env token, so the operator's explicit rotation is ignored.
  - confidence: 0.88
- **[P2] Preserve separators for file-backed key lists:** `src/agents/live-auth-keys.ts:111-114`
  - body: `<PROVIDER>_API_KEYS` now flows through `resolveSecretEnvValue()`, which normalizes file contents before `parseKeyList()` can split them. A file containing `key1\nkey2` becomes one concatenated invalid key, breaking multi-key rotation.
  - confidence: 0.84
- **[P3] Report unreadable configured secret files:** `src/utils/secret-env.ts:25-32`
  - body: If a `_FILE` env var points at a typo or unreadable path, all read errors are swallowed and the resolver returns `null`, which is indistinguishable from no configuration and leaves operators with a later auth failure or fallback credential.
  - confidence: 0.74
