## Summary

Keep open: provider `*_FILE` auth support is not on main and may be useful, but this PR is not mergeable as-is because it lacks real behavior proof and has source-reproducible credential/token regressions.

## What This Changes

The PR adds file-backed provider auth env vars, moves daemon gateway token persistence into config, and updates related auth/audit tests.

## Best Possible Solution

Land a narrower, maintainer-approved credential-loading change only after rebasing onto main's SecretRef/token resolver and preserving rotation semantics, list parsing, and diagnostics.

## Reproduction Assessment

Yes. Source inspection of the PR diff shows the token-rotation early return, newline separator loss, and silent file-read failure paths; no runtime reproduction was established in this read-only review.

## Solution Assessment

No. The provider `_FILE` direction may be useful, but this patch conflates scalar and list secret normalization, hides configured file errors, and reworks daemon-token behavior that current main already centralizes differently.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Honor explicit install tokens during rotation:** `src/cli/daemon-cli/install.ts:47-48`
  - body: When `openclaw gateway install --token <new-token>` runs with an existing `gateway.auth.token`, this early return keeps the old config token and reports persistence success. The later install plan can omit the service env token, so the user-supplied rotation is ignored.
  - confidence: 0.88
- **[P2] Preserve separators for file-backed key lists:** `src/agents/live-auth-keys.ts:111-114`
  - body: `<PROVIDER>_API_KEYS` now goes through `resolveSecretEnvValue()` before `parseKeyList()`. A file containing newline-separated keys is normalized into one concatenated invalid key, breaking multi-key rotation.
  - confidence: 0.84
- **[P3] Report unreadable configured secret files:** `src/utils/secret-env.ts:31-32`
  - body: If a `_FILE` env var points at a typo or unreadable path, the resolver swallows the read error and returns `null`, leaving operators with a later auth failure or fallback credential and no clue that the configured secret file failed to load.
  - confidence: 0.74

## Security Review

Status: needs_attention

Summary: The diff changes credential loading and daemon token persistence and has concrete token-rotation and secret-file diagnostic risks.

Concerns:

- **[medium] Explicit gateway token rotation can be ignored:** `src/cli/daemon-cli/install.ts:47`
  - body: The proposed daemon install path can keep an existing config token while treating persistence as successful, leaving the old gateway credential active after an explicit `--token` rotation attempt.
  - confidence: 0.88

  - confidence: 0.72
