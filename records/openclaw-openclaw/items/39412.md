## Summary

Keep open for maintainer review: current main already covers the daemon config-backed token direction, but provider `*_FILE` auth remains absent and this PR head has credential-handling blockers that make it unsafe to merge as-is.

## What This Changes

The PR adds a shared provider-secret `*_FILE` resolver, routes provider auth lookups through it, and changes daemon gateway install/audit tests toward config-backed token persistence.

## Best Possible Solution

Rewrite only the provider secret-file portion against current main if maintainers want `*_FILE`, preserving list separators, warning on unreadable configured files, excluding `*_FILE` from prefix scans, and leaving the current daemon token flow intact.

## Reproduction Assessment

Yes. Source inspection shows current main does not read provider `*_FILE` env vars, and the PR diff shows the credential rotation, list parsing, and diagnostics problems without needing a runtime repro.

## Solution Assessment

No. The provider file-backed secret direction may be useful, but this PR is not the best current solution because the daemon-token edits overlap newer main behavior and the resolver introduces credential-handling regressions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Honor explicit install tokens during rotation:** `src/cli/daemon-cli/install.ts:47-48`
  - body: When `openclaw gateway install --token <new-token>` runs with an existing `gateway.auth.token`, this early return keeps the old config token while reporting persistence success. The later install path then omits the service env token, so the operator's explicit rotation is dropped.
  - confidence: 0.88
- **[P2] Preserve separators for file-backed key lists:** `src/agents/live-auth-keys.ts:111-114`
  - body: `<PROVIDER>_API_KEYS` now flows through `resolveSecretEnvValue()`, which normalizes file contents before `parseKeyList()` can split them. A file containing `key1\nkey2` becomes one concatenated invalid key, breaking multi-key rotation.
  - confidence: 0.84
- **[P3] Report unreadable configured secret files:** `src/utils/secret-env.ts:25-32`
  - body: If a `_FILE` env var points at a typo or unreadable path, all read errors are swallowed and the resolver returns `null`, which is indistinguishable from no configuration. Operators then get a later auth failure or fallback credential without a useful diagnostic.
  - confidence: 0.74

## Security Review

Status: needs_attention

Concerns:

  - body: The proposed daemon install path can keep an existing config token while treating persistence as successful, causing an explicit `--token` rotation to be dropped and leaving the old gateway credential active.
  - confidence: 0.88

  - confidence: 0.72
