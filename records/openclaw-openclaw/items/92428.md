## Best Possible Solution

Keep the QQ Bot plugin-local chunker, but preserve ambiguous header state across block deliveries until a separator, non-table line, or terminal flush confirms or rejects the table.

## Reproduction Assessment

Yes. From source, call the PR-head stateful chunker with only `| A | B |`, then call it again with `|---|---|` plus rows; chunkText() flushes the header before the separator can activate table handling.

## Solution Assessment

No as written. The plugin-local stateful chunker is the right layer, but it must preserve possible table-header state across normal delivery boundaries and only finalize that state when confirmed, rejected, or terminally flushed.
