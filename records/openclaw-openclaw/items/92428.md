## Summary

This PR should stay open: the table-boundary fix is still needed, but the new parser introduces a blocking compatibility regression by rewriting standalone pipe-prefixed text that is not part of a confirmed table. The maintainer-authored and protected-label state also rules out cleanup closure.

## What This Changes

The PR adds QQ Bot-specific stateful markdown chunking that repeats table headers across chunks, carries table state between streamed deliveries, and converts unfinished table rows to plain text fields.

## Best Possible Solution

Only buffer or convert row fragments after a valid header and separator establish active table context; otherwise preserve input exactly, add regression coverage for standalone pipe-prefixed text, and provide redacted live QQ Bot output.

## Reproduction Assessment

Yes. Current main deterministically chunks streamed delivery blocks independently, while the proposed branch deterministically rewrites a standalone final line such as `| foo | bar`.

## Solution Assessment

No. Per-dispatch table state is an appropriate boundary, but fragment buffering and rewriting must only activate after a valid table has been established.
