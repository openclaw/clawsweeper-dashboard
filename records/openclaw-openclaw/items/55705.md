## Summary

Keep open: current main and latest v2026.5.22 still skip updating `sessionsResult` when `sessions.list` resolves to `undefined`, so the Control UI can keep showing a stale Subagent/session count. The open fork PR at https://github.com/BingqingLyu/openclaw/pull/1514 remains the linked implementation candidate rather than a merged fix.

## What This Changes

The issue reports the Control UI showing a stale active Subagent/session count even though API/CLI session listing is empty.

## Best Possible Solution

Review, rebase or replace, and land the linked narrow controller fix with regression coverage while preserving last-known-good data on real request errors.

## Reproduction Assessment

Yes. Seed `state.sessionsResult` with a non-empty result, mock `client.request("sessions.list")` to resolve `undefined`, and call `loadSessions`; current main leaves the stale result because assignment is guarded by `if (res)`.

## Solution Assessment

Yes, the maintainable fix is a narrow success-path clear in `loadSessionsOnce` plus regression coverage. The linked PR’s core direction is right, but final landing should avoid broad generic-error cache clearing and handle current-main controller structure.
