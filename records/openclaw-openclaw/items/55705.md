## Summary

Keep open: current main and the latest release still skip clearing `sessionsResult` when a successful `sessions.list` response resolves to `undefined`, and https://github.com/BingqingLyu/openclaw/pull/1514 is still an open linked implementation candidate for this issue.

## What This Changes

The issue reports that the Control UI shows a stale Subagent/session count even when API or CLI session listing reports no active rows.

## Best Possible Solution

Land a narrow Control UI controller fix that clears `sessionsResult` for empty successful `sessions.list` responses, preserves last-known-good state on generic errors, and adds focused regression coverage.

## Reproduction Assessment

Yes, source-level: seed `state.sessionsResult` with a nonzero count, mock `client.request("sessions.list")` to resolve `undefined`, and call `loadSessions`; current code skips assignment and leaves the stale count.

## Solution Assessment

Yes, the best layer is the Control UI sessions controller because the backend normal empty-list contract already returns a count-bearing result; the linked PR’s idea is right but needs a clean current-main repair with tests.
