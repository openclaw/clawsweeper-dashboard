## Summary

Keep open: current main still preserves a stale Control UI sessions result when `sessions.list` succeeds with no payload, and an open fork PR still references this issue as its fix path, so the issue should wait for a merged OpenClaw fix.

## What This Changes

The issue reports that the Control UI dashboard can show a phantom Subagent/session count even when API or CLI session listing is empty.

## Best Possible Solution

Land a narrow OpenClaw fix that clears stale `sessionsResult` after a successful empty or undefined `sessions.list` response, preserves cached data on real request errors, and adds focused controller regression coverage.

## Reproduction Assessment

Yes, from source inspection: seed `state.sessionsResult` with a nonzero result, mock `client.request("sessions.list")` to resolve `undefined`, and call `loadSessions()`; current main leaves stale state because assignment is guarded by `if (res)`. I did not execute the test locally because the review was required to keep the checkout read-only.

## Solution Assessment

Yes. The narrowest maintainable fix is to replace or clear `sessionsResult` after successful empty or undefined list responses while leaving the generic request-error path as last-known-good cache behavior.
