## Summary

Keep open: current `main` and the latest release still preserve stale `sessionsResult` when a successful `sessions.list` response resolves to `undefined`, and an open linked fork PR still targets the issue but needs maintainer review or replacement before this can close.

## What This Changes

The issue reports that the Control UI shows a stale Subagent/session count even when API/CLI session queries report no active subagents.

## Best Possible Solution

Review, adopt, or replace the linked open PR with a narrow controller fix that clears `sessionsResult` only for empty successful `sessions.list` responses and adds regression coverage.

## Reproduction Assessment

Yes, source-reproducible with high confidence: seed `state.sessionsResult` with a nonzero count, mock `client.request("sessions.list")` to resolve `undefined`, and call `loadSessions`; the current guard leaves the stale result cached. I did not run a live macOS Control UI browser repro in this read-only review.

## Solution Assessment

Yes for the narrow fix shape, but the linked PR needs review because its patch also clears cached sessions on generic errors. The best fix is success-path clearing plus focused tests while preserving last-known-good state on transient request errors.
