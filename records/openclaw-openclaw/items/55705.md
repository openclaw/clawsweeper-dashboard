## Summary

Keep open: current main and v2026.5.20 still skip updating Control UI session state when a successful `sessions.list` response is `undefined`, so the stale count can remain visible. The linked open fork PR at https://github.com/BingqingLyu/openclaw/pull/1514 is the active fix candidate, but it is not merged and appears stale against current main.

## What This Changes

The issue reports that Control UI shows a nonzero active Subagent/session count while API or CLI session listing is empty.

## Best Possible Solution

Review, rebase, or replace the linked fork PR with a narrow current-main fix that clears `sessionsResult` only on successful empty or undefined `sessions.list` responses, preserves last-known-good data for generic request errors, and adds focused regression coverage.

## Reproduction Assessment

Yes, at source level: seed `state.sessionsResult` with a non-empty result, mock `client.request("sessions.list")` to resolve `undefined`, and call `loadSessions()`. Current main skips assignment under the `if (res)` guard, so the old count remains available to the overview card.

## Solution Assessment

Yes, the best fix is a narrow controller repair on the successful response path. The linked PR has the right core idea, but the final current-main change should avoid clearing cached sessions on generic request errors and should include regression coverage.
