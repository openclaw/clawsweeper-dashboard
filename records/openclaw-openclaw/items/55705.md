Keep open: current main and v2026.6.6 still leave `sessionsResult` unchanged when `sessions.list` resolves to `undefined`, while the Overview card renders the cached count. The related fix remains unmerged, so this is still an actionable Control UI session-state bug rather than a cleanup close.

## What This Changes

The issue reports Control UI showing a stale active Subagent/session count after API/CLI session listing is empty.

## Best Possible Solution

Adopt a narrow controller fix on current main that clears `sessionsResult` for empty successful `sessions.list` responses, preserves last-known-good state on generic errors, and adds regression coverage.

## Reproduction Assessment

Yes for source-level reproduction: seed `sessionsResult` with a nonzero count, mock `client.request("sessions.list")` to resolve `undefined`, and call `loadSessions`; current code skips assignment and leaves the stale count. I did not run a live browser repro in this read-only review.

## Solution Assessment

No, current main is not the best solution because it preserves stale cached state on an empty successful response. The best fix is the narrow controller clear for `undefined`/empty success responses plus regression coverage, while preserving generic-error last-known-good behavior.
