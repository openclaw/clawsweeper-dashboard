## Summary

Keep this issue open. Current main still has the stale Control UI sessions-cache path, and the open linked implementation PR #55831 uses closing syntax but has not merged, so this should stay open until that PR is revised and landed or replaced.

## What This Changes

The issue reports the Control UI showing a phantom active Subagent/session count while API/CLI session listing is empty on macOS OpenClaw 2026.3.24.

## Best Possible Solution

Revise and land the linked Control UI controller fix so a successful empty or undefined `sessions.list` response clears `sessionsResult` to `null`, while preserving current last-known-good behavior for generic request errors and adding focused controller coverage for both paths.

## Reproduction Assessment

Yes. A high-confidence controller-level reproduction is to seed `state.sessionsResult` with a nonzero count, mock `client.request("sessions.list")` to resolve `undefined`, and call `loadSessions()`; current main leaves the stale result intact because of the `if (res)` guard.

## Solution Assessment

Yes. The narrow maintainable fix belongs in `ui/src/ui/controllers/sessions.ts`: clear or replace cached sessions state on successful empty/undefined responses, without rewriting the backend `sessions.list` handler that already returns a structured zero-count result on normal success.
