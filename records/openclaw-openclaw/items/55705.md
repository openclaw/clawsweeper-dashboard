## Summary

Keep open: current main still leaves a stale Control UI sessions result in place when `sessions.list` resolves with no payload, and the target-repo fix PR was closed unmerged.

## What This Changes

The issue reports a stale nonzero Control UI Subagent/session count while API/CLI session listings are empty.

## Best Possible Solution

Update the Control UI sessions controller so successful empty or undefined list responses clear cached session state, preserve last-known-good data on generic request errors, and add focused regression coverage plus changelog.

## Reproduction Assessment

Yes. Seed `state.sessionsResult` with a nonzero result, mock `client.request("sessions.list")` to resolve `undefined`, and call `loadSessions()`; current main leaves the stale result because assignment is guarded by `if (res)`.

## Solution Assessment

Yes. The narrowest maintainable fix is to clear or replace `sessionsResult` only after a successful empty/undefined response, keep checkpoint work guarded for real results, and preserve generic-error cache behavior.
