## Summary

Keep open: current main and v2026.6.6 still preserve a stale `sessionsResult` when `sessions.list` resolves to `undefined`, and the linked fix PR is open but conflicting rather than merged.

## What This Changes

The issue reports that the Control Panel shows a stale Subagent/session count while API/CLI session listing is empty.

## Best Possible Solution

Land or adapt the narrow controller fix so successful undefined `sessions.list` responses clear `sessionsResult`, while preserving last-known-good data on generic errors and adding focused regression coverage.

## Reproduction Assessment

Yes, at source level: seed `sessionsResult` with a non-empty result, have `client.request("sessions.list")` resolve to `undefined`, and `loadSessionsOnce` leaves the old result because assignment is guarded by `if (res)`. I did not run a live browser/macOS reproduction in this read-only review.

## Solution Assessment

No, current main is not the best solution because it preserves stale cached state on an undefined successful response. The narrow maintainable fix is to clear only that success-path result and cover it with a controller test.
