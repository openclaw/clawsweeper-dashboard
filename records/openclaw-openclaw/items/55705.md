## What This Changes

The issue asks the Control UI to stop showing a stale Subagent/session count when the backend or API reports no active subagents.

## Best Possible Solution

Land a narrow target-repo controller fix that clears `sessionsResult` on successful empty or undefined `sessions.list` responses, preserves last-known-good data on generic request errors, and adds focused regression coverage.

## Reproduction Assessment

Yes, at source level. Seed `state.sessionsResult` with a non-empty result, mock `client.request("sessions.list")` to resolve `undefined`, and call `loadSessions()`; current main leaves the stale result because assignment is guarded by `if (res)`.

## Solution Assessment
