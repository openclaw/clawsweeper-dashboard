## What This Changes

The issue asks for the Control UI Subagent/Sessions count to match the actual API/CLI session list instead of showing phantom active subagents when the backend reports none.

## Best Possible Solution

Revise and land #55831 or an equivalent narrow controller fix so successful empty or undefined `sessions.list` responses clear cached session state, while preserving last-known-good state for generic request failures and the existing missing-scope clear path. Add a focused controller regression test for that behavior.

## Reproduction Assessment

Yes for the controller failure mode: seed `sessionsResult` with a nonzero count, have `sessions.list` resolve successfully to `undefined`, then call `loadSessions`; current main skips assignment under `if (res)` and leaves the stale count. The full macOS UI screenshot path is not live-reproduced here.

## Solution Assessment

Yes, the best fix direction is a narrow UI controller state update with a regression test, not a backend sessions-list rewrite.
