## What This Changes

The issue reports that the Control UI Subagent/session count can show active sessions even when API/CLI session listing is empty.

## Best Possible Solution

Review, adopt, or replace the open linked PR with a narrow controller fix that clears `sessionsResult` for empty successful `sessions.list` responses while preserving last-known-good state on generic request errors and adding focused regression coverage.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: seed `state.sessionsResult` with a nonzero count, mock `client.request("sessions.list")` to resolve `undefined`, then call `loadSessions`; current code skips assignment and leaves the stale count.

## Solution Assessment

Yes, the best fix is a narrow Control UI controller change. The backend normal empty-list contract already returns a count-bearing object, so this should be handled where the UI cache receives an empty successful response, with tests for that boundary.
