## Summary

Keep open: current main and v2026.6.6 still skip updating `sessionsResult` when `sessions.list` resolves to `undefined`, so the cached session count can remain visible. An open linked fork PR references this issue, but it is unmerged and currently conflicting, so the issue should remain open for maintainer adoption or replacement.

## What This Changes

The issue reports the Control UI Subagent/session count showing stale active sessions when API/CLI session listing is empty.

## Best Possible Solution

Land a narrow Control UI controller fix that clears `sessionsResult` only for empty successful `sessions.list` responses, preserves last-known-good state on generic errors, and adds focused regression coverage.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to seed `sessionsResult` with a nonzero count, mock `client.request("sessions.list")` to resolve `undefined`, and call `loadSessions`; current main skips assignment and leaves the stale count.

## Solution Assessment

No, current main is not the best solution because it preserves stale cached state on an empty successful response. The best fix is a narrow successful-response clear with regression coverage, while preserving generic-error last-known-good behavior.
