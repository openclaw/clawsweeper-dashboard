## What This Changes

The issue reports a Control UI regression where the Subagent/Sessions count shows stale active sessions even though API/CLI session listing is empty.

## Best Possible Solution

Update the Control UI sessions controller so successful empty or undefined list responses clear stale cached session state, preserve last-known-good data on generic request errors, and add focused regression coverage plus changelog.

## Reproduction Assessment

Yes, there is a high-confidence source-level reproduction: seed `state.sessionsResult` with a nonzero result, mock `client.request("sessions.list")` to resolve `undefined`, and call `loadSessions()`. Current main leaves the stale result because assignment is guarded by `if (res)`.

## Solution Assessment

Yes, the narrow fix is to clear or replace `sessionsResult` only after a successful empty or undefined response, keep checkpoint work guarded for real results, and preserve generic-error behavior.
