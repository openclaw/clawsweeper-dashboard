## Best Possible Solution

Put the non-positive-as-absent rule at the gateway tool input boundary, cover at least one real tool path such as `message` or `gateway`, and then provide redacted after-fix runtime proof.

## Reproduction Assessment

Yes, from source inspection. Current main clamps direct `resolveGatewayOptions({ timeoutMs: 0 })` to 1ms, while the current user-facing tool path rejects `timeoutMs: 0` in `readGatewayCallOptions` before dispatch.

## Solution Assessment

No. The PR changes a downstream helper only; the best fix is to normalize zero before the shared gateway option parser rejects it, with regression coverage through a real model-facing tool path.
