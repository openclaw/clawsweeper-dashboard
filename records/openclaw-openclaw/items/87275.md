## Summary

## What This Changes

The PR changes `resolveGatewayOptions` so finite non-positive `timeoutMs` values fall back to the default timeout and adds a direct helper regression test.

## Best Possible Solution

Normalize non-positive gateway `timeoutMs` at the shared gateway option parser/schema boundary, keep explicit positive timeout validation for real overrides, cover a model-facing execution path, and require redacted real behavior proof for this head.

## Reproduction Assessment

Yes. Source inspection shows direct resolver zero-timeout behavior on current main, and also shows the model-facing paths reject `timeoutMs: 0` through `readGatewayCallOptions` before the PR fallback can run.

## Solution Assessment

No. The resolver change is a plausible helper fix, but the best fix belongs at the shared gateway option parser/schema boundary with a regression through a user-facing gateway-backed tool.
