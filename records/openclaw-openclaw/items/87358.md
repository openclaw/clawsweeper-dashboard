## Summary

Close as implemented: current main and shipped releases contain the WebChat stale active-row suppression that prevents a completed turn from being overwritten back to Stop/In progress by a late sessions refresh.

## What This Changes

Report requests WebChat Send/Stop and In progress state to clear after completed assistant replies even when a later sessions refresh reports the session as running.

## Best Possible Solution

Keep the shipped bounded WebChat reconcile fix; use https://github.com/openclaw/openclaw/issues/88104 only for broader turn-complete or context-token symptoms that still reproduce on fixed releases.

## Reproduction Assessment

Yes at source level: the issue and related fix describe a completed terminal chat event followed by a stale active `sessions.list` row, and current tests recreate that sequence. I did not run a fresh live WebChat repro during this read-only review.

## Solution Assessment

Yes: the shipped fix is the maintainable shape because it is gated on a recent local terminal reconcile for the selected active run, rather than broadly clearing every active row with no local run.
