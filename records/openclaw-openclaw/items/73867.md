## Summary

Keep open. Current main still does not wire session-registry pruning into `openclaw tasks maintenance`; the only cron run-session pruning is the separate cron timer reaper, and PR #74023 remains the open implementation candidate with closing syntax for this issue.

## What This Changes

`openclaw tasks maintenance --apply` is requested to also preview/apply a conservative stale cron session-registry sweep while preserving entries for currently running jobs from `jobs-state.json`.

## Best Possible Solution

Land PR #74023 or an equivalent focused change that adds command-level preview/apply session-registry maintenance, protects currently running cron jobs from durable job state, updates text/JSON/docs/changelog output, and adds targeted command coverage.

## Reproduction Assessment

Yes. Source inspection against current main reproduces the gap: `tasksMaintenanceCommand` has no session-registry maintenance branch, and the existing cron reaper runs from cron timer code rather than the CLI maintenance command.

## Solution Assessment

Yes. A narrow command-level sweep with explicit jobs-state protection is the maintainable fix boundary; the already-linked PR should be reviewed and landed or replaced rather than creating a duplicate repair path.
