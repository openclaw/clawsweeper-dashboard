## Summary

Keep open. Current main still lacks command-level session-registry pruning in `openclaw tasks maintenance`, and the issue is already paired with open PR #74023 using closing syntax, so it should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that `openclaw tasks maintenance --apply` also prune stale cron run session registry entries older than seven days while preserving entries for jobs currently marked running in durable cron state.

## Best Possible Solution

Land PR #74023 or an equivalent focused change that adds command-level preview/apply session-registry maintenance, protects currently running cron jobs from durable cron state, keeps non-cron sessions out of scope, updates text/JSON/docs/changelog output, and adds targeted command coverage.

## Reproduction Assessment

Yes. Source inspection against current main is a high-confidence reproduction path: `tasksMaintenanceCommand` only runs task and Task Flow maintenance, while the existing cron run session reaper is a separate cron timer path.

## Solution Assessment

Yes. The narrow maintainable fix is to add a tasks-maintenance-owned session registry sweep with explicit jobs-state protection; because PR #74023 is already open with closing syntax, the issue should track that review rather than spawn a duplicate repair.
