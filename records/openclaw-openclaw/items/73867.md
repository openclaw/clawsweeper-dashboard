## Summary

Keep open. Current main still limits `openclaw tasks maintenance` to task and Task Flow maintenance, while the linked implementation PR remains open and unmerged, so this issue should stay open for normal review and landing of that PR or an equivalent focused fix.

## What This Changes

Request that `openclaw tasks maintenance --apply` also prune stale cron run session-registry entries using conservative age criteria while protecting jobs currently marked running in durable cron state.

## Best Possible Solution

Land PR #74023 or an equivalent focused change that adds preview/apply session-registry maintenance to `tasksMaintenanceCommand`, preserves currently running cron jobs from durable job state, updates text and JSON output plus docs, and locks the behavior with targeted command tests.

## Reproduction Assessment

Yes. Source inspection against current main reproduces the gap: `tasksMaintenanceCommand` has no session-registry maintenance branch, and the existing cron session reaper runs from the cron timer path rather than the CLI maintenance command.

## Solution Assessment

Yes. A command-level sweep with explicit jobs-state protection is the narrow maintainable path; the already-linked PR is the right implementation candidate to review rather than creating duplicate repair work.
