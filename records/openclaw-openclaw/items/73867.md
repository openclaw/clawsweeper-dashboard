## Summary

Keep open because current main still lacks command-level session-registry pruning, and open PR #74023 is the linked implementation candidate with closing syntax for this issue.

## What This Changes

Request `openclaw tasks maintenance --apply` to prune stale cron session registry entries older than the retention window while preserving entries for currently running cron jobs.

## Best Possible Solution

Land #74023 or an equivalent focused command-level maintenance sweep with preview/apply output, active-job protection, docs, changelog, and regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows `tasksMaintenanceCommand` only invokes task and TaskFlow maintenance, while the existing cron session reaper is separate timer-driven cleanup.

## Solution Assessment

Yes. A focused `tasks maintenance` preview/apply sweep with running-job protection is the narrow maintainable direction; the background reaper alone does not provide explicit operator cleanup behavior.
