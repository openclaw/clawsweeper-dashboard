## Summary

Keep open because current main still lacks a `tasks maintenance` session-registry sweep, and the provided discussion shows open PR #74023 with closing syntax for this issue rather than a merged fix.

## What This Changes

The issue requests that `openclaw tasks maintenance --apply` also prune stale cron session registry entries while protecting currently running jobs.

## Best Possible Solution

Review and land #74023, or an equivalent focused fix that adds preview/apply session-registry cleanup with running-job protection, docs, changelog, and regression coverage.

## Reproduction Assessment

Yes. Source inspection against current main reproduces the gap: `tasksMaintenanceCommand` has no session-registry maintenance path, while cron run session pruning exists only in the separate runtime reaper.

## Solution Assessment

Yes. A command-level preview/apply sweep is the narrow maintainable boundary because operators need explicit `tasks maintenance` cleanup behavior, not only timer-driven retention.
