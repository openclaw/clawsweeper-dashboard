## Summary

Keep open because current main still lacks a command-level session-registry sweep, and open PR #74023 already targets this issue with closing syntax but has not merged.

## What This Changes

Request that `openclaw tasks maintenance --apply` also prune stale cron session registry entries older than 7 days while protecting entries for jobs still marked running in `jobs-state.json`.

## Best Possible Solution

Review and land #74023, or an equivalent focused command-level sweep with preview/apply JSON and text output, `jobs-state.json` running protection, docs, changelog, and regression coverage.

## Reproduction Assessment

Yes. Read-only source inspection reproduces the gap: the command path emits only task and TaskFlow maintenance, while cron session pruning is a separate runtime timer path.

## Solution Assessment

Yes. A `tasks maintenance` preview/apply sweep with running-job protection is the narrow maintainable fix; the existing background reaper and `sessions cleanup` docs do not provide the requested operator behavior.
