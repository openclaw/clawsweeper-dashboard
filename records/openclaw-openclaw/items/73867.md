## Summary

Keep open because current main still limits `openclaw tasks maintenance` to task and TaskFlow maintenance, while the requested session-registry sweep is tracked by an open, unmerged implementation PR.

## What This Changes

Request `openclaw tasks maintenance --apply` to prune stale cron run session registry entries while preserving jobs still marked running in cron state.

## Best Possible Solution

Land #74023 or an equivalent focused fix that adds preview/apply session-registry maintenance, protects running cron jobs, reports text/JSON counts, updates docs/changelog, and covers stale/running/non-cron cases in tests.

## Reproduction Assessment

Yes. Source inspection against current main reproduces the gap: the command has no session-registry maintenance path, and the only cron run-session pruning is the separate timer reaper.

## Solution Assessment

Yes. A focused command-level sweep is the right boundary because the existing runtime reaper does not give operators an explicit `tasks maintenance` preview/apply cleanup path.
