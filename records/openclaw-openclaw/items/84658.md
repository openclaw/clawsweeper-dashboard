The PR changes `loadModelsConfigWithSource()` so resolved model command config is always paired with the pre-resolution source snapshot, and adds focused regression coverage for custom-provider SecretRef `apiKey` persistence.

## Solution Assessment

Yes. Re-anchoring the runtime snapshot in `loadModelsConfigWithSource()` is the narrowest maintainable fix because downstream `models.json` generation already has source-authoritative marker handling.
