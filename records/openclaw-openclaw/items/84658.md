## What This Changes

The PR changes model command config loading so resolved runtime config is always paired with the pre-resolution source snapshot, plus tests for SecretRef-backed custom-provider `apiKey` marker persistence.

## Best Possible Solution

Land this boundary fix after focused validation passes, and keep marker-classification follow-up such as the Codex sentinel case in https://github.com/openclaw/openclaw/issues/84376 rather than expanding this PR beyond snapshot preservation.

## Reproduction Assessment

Yes. Source inspection shows current main drops the source snapshot after model command SecretRef resolution, and the PR body includes live local output proving the fixed path persists the env marker instead of the resolved secret.

## Solution Assessment

Yes. Re-anchoring the runtime snapshot in `loadModelsConfigWithSource()` is the narrowest maintainable fix because downstream `models.json` generation already has source-authoritative marker handling.
