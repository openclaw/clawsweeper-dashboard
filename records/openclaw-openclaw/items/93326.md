## Summary

Keep open: current main and v2026.6.9 still only strip malformed XML suffixes from tool path parameters, with no `.docodex` or document-extension guard, and three open closing PRs are candidate fixes that have not merged.

## What This Changes

The issue asks OpenClaw to prevent DOCX-related file paths from being corrupted into `.docodex` before file-tool or command execution.

## Best Possible Solution

Land one canonical fix for structured agent file-tool path parameters that handles known hallucinated DOCX/PPTX/XLSX extensions, keeps free-form shell command text unchanged unless it can be parsed safely, and then close the duplicate candidate PRs.

## Reproduction Assessment

Yes for the execution-side gap: current main leaves a supplied `report.docodex` path unchanged through XML-only path normalization before read/write/edit execution. The intermittent model-side rewrite still lacks a deterministic live turn transcript.

## Solution Assessment

Unclear until maintainers choose the policy: the shared agent file-tool boundary is the right layer, but the open PRs differ on whether to silently correct or reject mutation paths.
