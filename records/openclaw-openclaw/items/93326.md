## Summary

Keep this open: current main has adjacent XML arg-value cleanup, but no source or test coverage for `.docx` to `.docodex` corruption, and all linked fix attempts are closed unmerged.

## What This Changes

The issue reports that DOCX filenames can be corrupted from `.docx` to `.docodex` before file-tool or shell execution.

## Best Possible Solution

Add a narrow central guard for known hallucinated Office-extension path parameters, with retryable guidance and coverage for read/write/edit plus a conscious decision about what can safely be done for free-form exec commands.

## Reproduction Assessment

Partially: no attached log or deterministic live prompt forces GPT-5.4 to emit `.docodex`, but source inspection shows that once a model supplies `report.docodex`, current read/write/edit and exec paths do not reject or correct that extension.

## Solution Assessment

Yes, the maintainable direction is a narrow tool-parameter guard in the existing agent tool normalization layer; broad unknown-extension blocking or unrelated channel/provider changes would be too risky.
