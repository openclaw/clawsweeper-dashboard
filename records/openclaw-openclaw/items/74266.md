## Summary

Keep this PR open. Current main does not already contain an equivalent external auth-proxy recipe, and the PR is a focused docs-only workaround for the still-open credential isolation discussions, but the current head still has a blocking container networking error in the example that should be fixed before merge.

## What This Changes

The PR adds `docs/gateway/external-auth-proxy.md` and registers it in `docs/docs.json` as a Gateway authentication/secrets recipe for routing provider requests through an external auth-injection proxy.

## Best Possible Solution

Merge the recipe only after it clearly separates same-host/systemd, shared-network-namespace, and multi-container deployments, with proxy URLs that are reachable from the OpenClaw runtime. Keep native credential isolation tracked in #11829 and #10659 instead of expanding this PR into a built-in proxy.

## Reproduction Assessment

Yes. The failure is reproducible by inspection: the PR describes tenant/container setups but points the provider `baseUrl` at `localhost`, while current docs state loopback inside a container refers to that container rather than the host-side proxy.

## Solution Assessment

No, not as written. The docs-only workaround is a reasonable lightweight direction, but the container example should use a host-gateway/bridge address or explicitly limit `localhost` to same-network-namespace deployments.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Use a reachable proxy URL in container examples:** `docs/gateway/external-auth-proxy.md:151`
  - body: The recipe's container-oriented example points `baseUrl` at `http://localhost:18080/anthropic`, but the proxy skeleton listens on the Docker bridge and identifies tenants by bridge source IP. In a tenant container, `localhost` is the tenant container itself, so readers following this config will not reach the documented auth proxy. Use a bridge/host-gateway address for multi-container deployments, or scope the localhost example to same-host/shared-network setups.
  - confidence: 0.9
