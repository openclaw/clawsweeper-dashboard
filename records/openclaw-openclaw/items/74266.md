## Summary

Keep open: the docs-only recipe is useful, but the current head still tells a separate tenant container to reach the external auth proxy through `localhost`, which points back at the tenant container rather than the proxy and undermines the security guidance.

## What This Changes

The PR adds a Gateway documentation recipe for routing model-provider requests through an external auth-injection proxy and adds the page to the Authentication and secrets docs navigation.

## Best Possible Solution

Keep the docs recipe, but split separate-container and same-host loopback examples so the main path uses a container-reachable proxy URL and `localhost` is explicitly scoped to valid same-host or same-network-namespace deployments.

## Reproduction Assessment

Yes. Source inspection is enough: the PR describes an external or separate-container proxy but configures the tenant-side provider URL with `localhost`, which resolves inside the tenant container.

## Solution Assessment

No. A docs-only recipe is the right layer for an existing-primitives workaround, but the examples need container-reachable addressing before this is safe official security guidance.
