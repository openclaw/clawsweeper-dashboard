Keep open: the recipe is useful, but it is not merge-ready because the separate-container credential-isolation path still uses a localhost provider URL that would not reach an external proxy from inside the tenant container.

## Best Possible Solution

Keep the recipe, but split same-host and separate-container deployments and use a container-reachable bridge, host-gateway, or service-DNS proxy URL before merge.
