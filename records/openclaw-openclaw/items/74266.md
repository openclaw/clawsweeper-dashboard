## What This Changes

## Best Possible Solution

Publish the recipe after the proxy topology examples clearly split same-host loopback from container-reachable proxy addressing and docs validation passes on a refreshed branch.

## Reproduction Assessment

Yes. Source and docs inspection are enough: the PR describes an external or separate-container proxy but uses `localhost`, while existing Docker docs state that loopback inside a container points at that container itself.

## Solution Assessment

No. A docs-only recipe is the right layer for this existing-primitives workaround, but the examples need container-reachable addressing before they are safe official security guidance.
