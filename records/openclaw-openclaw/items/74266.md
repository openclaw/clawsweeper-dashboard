## Summary

Keep open: the docs workaround is useful and fits current SecretRef guidance, but the PR still uses a `localhost` provider endpoint in a separate-container recipe, which would make operators copy a credential-isolation setup that cannot reach the proxy from the tenant container.

## What This Changes

The PR adds a Gateway documentation recipe and navigation entry for using an external auth-injection proxy with existing model provider `baseUrl` and SecretRef behavior.

## Best Possible Solution

Publish the recipe after it clearly separates same-host or shared-network localhost setups from container-reachable proxy addresses such as Docker bridge IPs, service DNS, `host.docker.internal`, or Kubernetes service DNS.

## Reproduction Assessment

Yes. Source/docs inspection is enough: the PR's architecture and configuration examples describe a tenant container but route provider traffic to `localhost`, while the inline proxy sample binds a Docker bridge address.

## Solution Assessment

No. A docs-only workaround is a reasonable path, but this PR is not the best form until the networking example matches the documented deployment boundary.
