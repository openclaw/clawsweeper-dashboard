## Summary

Keep open, but this PR is not merge-ready as-is. Current main still has the reported `/think` menu bug, but this branch fixes it by treating missing Ollama reasoning metadata as reasoning-capable instead of feeding the already-discovered runtime catalog metadata into the menu path; the open member-authored runtime-catalog PR is the better canonical direction.

## What This Changes

The PR changes the Ollama provider policy so `reasoning: undefined` returns the full Ollama thinking profile instead of the off-only profile.

## Best Possible Solution

Land one runtime-catalog-aware native `/think` menu fix such as https://github.com/openclaw/openclaw/pull/94067, then close or supersede narrower provider-policy duplicate PRs after proof.

## Reproduction Assessment

Yes for source-level reproduction: current main’s Telegram menu path omits the runtime catalog while Ollama discovery already has `reasoning` metadata, and the linked issue gives shipped 2026.6.8 Telegram/Ollama steps. I did not run a live Telegram/Ollama endpoint in this read-only review.

## Solution Assessment

No. The patch is a plausible mitigation, but the best fix is to pass discovered runtime catalog metadata into native `/think` menu choice resolution rather than treating all missing Ollama reasoning metadata as capable.
