## Summary

Keep open for maintainer review: #93835 is still real on current main, but this PR changes the wrong layer by treating missing Ollama reasoning metadata as reasoning-capable instead of getting the runtime catalog into native `/think` menu resolution.

## What This Changes

The PR changes Ollama's provider thinking profile resolver so `reasoning: undefined` returns the full Ollama thinking profile instead of the off-only profile.

## Best Possible Solution

Pass the runtime model catalog into native `/think` choice resolution so Ollama's discovered `reasoning: true/false` reaches the existing provider policy, then close or retarget this fallback change if #94067 lands.

## Reproduction Assessment

Yes for source reproduction: #93835 gives concrete OpenClaw 2026.6.8 Telegram steps, and current main shows native menus omit the runtime catalog while the reply/setter path can hydrate it.

## Solution Assessment

No. This is a plausible symptom workaround, but the best fix is to pass the runtime catalog into native `/think` menu resolution rather than changing provider policy for unknown capability metadata.

- **[P1] Preserve unknown Ollama reasoning as unknown:** `extensions/ollama/provider-policy-api.ts:65`
  - body: This maps every missing Ollama `reasoning` hint to the full thinking profile. Pass the runtime catalog into native menu resolution instead.
