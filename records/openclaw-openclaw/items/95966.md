## Summary

Keep open. Current main has synchronous persistence hooks, outbound message hooks, and general secret redaction, but it does not implement a protected-file/content-provenance filter or a UI blocked-event workflow; the adjacent security-profile issue only partially overlaps and does not supersede this narrower request.

## What This Changes

The issue requests a protected-content filtering workflow that lets agents read configured sensitive files but prevents their contents from being persisted or broadcast to channels.

## Best Possible Solution

Define a maintainer-approved protected-output policy that covers transcript persistence, outbound delivery/recovery, and live UI events together, preferably through existing hook/plugin seams unless a core API gap is confirmed.
