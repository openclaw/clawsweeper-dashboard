## Summary

Keep open. Current main and the latest release have generic hook/redaction/display seams, but they do not implement protected-file/content classification, similarity matching, safe replacement/block events, or a complete live UI/outbound-delivery workflow; the related hook and redaction PRs only partially overlap.

## What This Changes

Requests a protected-content output filtering mechanism that lets agents read protected files while preventing that content from being persisted, delivered to channels, or briefly shown in the UI.

## Best Possible Solution

Define a maintainer-approved protected-output security policy and the minimal core/plugin API seams for transcript persistence, outbound delivery, and live UI projection, with content detection kept plugin-owned where feasible.
