## Summary

Keep open. Current main has generic transcript persistence hooks, outbound delivery hooks, live display normalization, and redaction, but it still lacks a protected-file/content-provenance policy plus a live UI blocked-event workflow; the related security-profile issues and guard-hook PRs only partially overlap.

## What This Changes

The issue requests protected-content filtering across tool-result persistence, channel output, UI blocked handling, audit logging, and new protected-content configuration.

## Best Possible Solution

Define a maintainer-approved protected-output security policy and the minimal core/plugin API seams that cover transcript persistence, outbound delivery/recovery, and live UI events, with detection logic plugin-owned where possible.
