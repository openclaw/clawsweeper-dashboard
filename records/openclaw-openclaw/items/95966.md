## Summary

Keep open. Current main and v2026.6.10 have generic transcript hooks, outbound delivery hooks, live display normalization, and redaction, but they still do not provide a protected-file/content-provenance policy or live blocked-content UI/protocol workflow.

## What This Changes

Request a protected-content output filter that classifies protected files, rewrites or blocks matching tool/assistant output, and updates channel/UI handling with new config/API.

## Best Possible Solution

Define a maintainer-approved protected-output security policy and the minimal core/plugin API seams that cover transcript persistence, outbound delivery/recovery, and live UI events, with detection logic plugin-owned where possible.
