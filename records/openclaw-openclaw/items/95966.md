## Summary

Keep open. Current main has generic transcript hooks, persisted tool-result redaction, and blocked user-turn display handling, but it still lacks the requested protected-file/content provenance policy, config/API shape, similarity classifier, audit behavior, and live channel/UI blocked-content workflow.

## What This Changes

The issue requests a protected-content filtering workflow that lets agents read sensitive files while preventing protected content from being persisted, broadcast, or briefly shown in the UI.

## Best Possible Solution

Define a maintainer-approved protected-output security policy and the minimal core/plugin API seams for transcript persistence, outbound delivery, and live UI projection, with detection logic plugin-owned where possible.
