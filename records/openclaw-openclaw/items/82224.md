  - confidence: 0.91

  - body: The top picker fallback advertises `/codex plugins menu`, and the plugins help button sends `/codex plugins help`, but the current plugins handler only implements list/enable/disable and returns an unknown-subcommand response for those strings. Make those paths real commands or keep them out of typeable/fallback/button surfaces.
