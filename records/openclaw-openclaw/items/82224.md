## Summary

Keep open: the feature direction and supplied live proof are useful, but current head still has blocking security, correctness, and docs issues around chat-triggered persistent Codex plugin config writes.

## What This Changes

Adds `/codex` chat pickers and `/codex plugins` management, widens first-party Codex plugin marketplace handling, and adds focused Codex/Telegram tests.

## Best Possible Solution

Land this only after mutating plugin verbs are explicitly gated, first-time adds activate the global Codex plugin policy correctly, docs match the widened marketplace contract, and the related data-layer PR is resolved.

## Reproduction Assessment

Yes for the review findings: source inspection shows the config mutation path has no write-authority gate and the add path does not set the global enablement flag. The original request is a feature addition, not a current-main bug reproduction.

## Solution Assessment

No. A plugin-owned chat surface is a reasonable direction, but this patch is not merge-ready until the mutation authority boundary, global enablement behavior, and docs contract are fixed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Gate plugin config mutations before dispatch:** `extensions/codex/src/command-handlers.ts:490`
  - body: This new branch sends every `/codex plugins` call to handlers that can add, enable, disable, toggle, or remove persisted `openclaw.json` entries, but `/codex` only requires basic sender authorization. Mutating verbs need an owner or explicit operator write/admin check before they reach the IO layer.
  - confidence: 0.86
- **[P2] Enable the global Codex plugins switch when adding entries:** `extensions/codex/src/command-plugins-management.ts:140-145`
  - body: `add` writes only the per-plugin entry with `enabled: true`. The resolver still requires `codexPlugins.enabled === true`, so adding the first plugin to a config without that global switch reports success while the plugin remains disabled after refresh.
  - confidence: 0.91
- **[P2] Update the documented marketplace boundary:** `extensions/codex/openclaw.plugin.json:117`
  - body: The manifest now accepts `openai-bundled` and `openai-primary-runtime`, but the public configuration reference still says V1 only supports `openai-curated`. Users would see a schema that accepts values the docs call unsupported.
  - confidence: 0.88
