## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Gate Codex picker callbacks with Telegram auth:** `extensions/telegram/src/bot-handlers.runtime.ts:2620-2621`
  - body: This new `cdx_*` branch runs before the native command callback path and renders or edits Codex picker content without the comparable command/model callback authorization preflight. In group or allowlist-sensitive setups, a sender who can see the inline keyboard can open the dynamic plugin inventory even when the equivalent `/codex` command flow would be rejected, so run the same authorization check before rendering these pickers.
  - confidence: 0.84
- **[P2] Make advertised plugin menu commands work:** `extensions/codex/src/command-handlers.ts:248-249`
  - body: The new fallback surface tells users and non-Telegram renderers to run `/codex plugins menu`, and the Telegram plugin submenu sends `tgcmd:/codex plugins help`, but the plugin handler still only implements `list`, `enable`, and `disable`. Those advertised commands fall into the unknown-subcommand path, so either add real `menu`/`help` handling or stop advertising them as executable commands.
  - confidence: 0.9
- **[P2] Match the effective Codex plugin state:** `extensions/telegram/src/codex-picker-buttons.ts:205-209`
  - body: The picker treats every entry without `enabled: false` as ON, but `/codex plugins list` and the config docs make the effective state depend on the global `codexPlugins.enabled` flag too. When the global switch is off, this shows inactive plugins in the disable picker and hides them from the enable picker, contradicting the command surface the buttons mirror.
  - confidence: 0.9
