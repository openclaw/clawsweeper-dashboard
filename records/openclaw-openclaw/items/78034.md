## What This Changes

The PR skips the Control UI local-media client allowlist while bootstrap preview roots are empty, routes availability to the existing assistant-media metadata check, adds grouped-render regression coverage, and edits CHANGELOG.md.

## Best Possible Solution

Land the focused empty-roots fallback after redacted real Control UI proof is supplied and the release-owned changelog edit is removed, while preserving populated-root blocking and server-side media authorization.

## Reproduction Assessment

Yes, at source level: render a local `MEDIA:` attachment while `localMediaPreviewRoots` is `[]`, and current main returns `Outside allowed folders` before the `assistant-media?meta=1` server check can run. I did not execute a live Control UI scenario in this read-only review.

## Solution Assessment

Yes, with cleanup: falling through to the existing authenticated server metadata check only while roots are empty is the narrowest maintainable fix, and the PR mirrors the invariant for local image blocks. The branch still needs real behavior proof and the release-owned changelog edit removed.
