## Summary

Keep open. Current main still hard-denies local Control UI attachments when `localMediaPreviewRoots` is empty, and this PR is a plausible narrow fix, but it is not merge-ready because lint is failing on the new test and the real behavior proof is mock-only.

## What This Changes

The PR skips the Control UI local-media preview-root gate while roots are empty, mirrors that behavior for local images, adds grouped-render regression coverage, and adds a changelog entry.

## Best Possible Solution

Fix the lint line, add accepted after-fix Control UI/runtime proof or a maintainer proof override, then land the narrow empty-roots fallback while preserving populated-root blocking.

## Reproduction Assessment

## Solution Assessment

Yes for the code direction, but not for merge readiness. Falling through only during the empty-bootstrap window is narrower than weakening the root predicate globally, but the lint blocker and real-proof gate still need resolution.
