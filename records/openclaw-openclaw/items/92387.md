## Label Justifications

- `P1`: The shipped Anthropic Vertex path can reject valid memory-hit requests and force model fallback whenever the cache-control budget is filled.

## Summary

The patch is a focused, correct fix for a shipped Anthropic Vertex regression: current main applies cache-control policy after the shared transport has already split and budgeted the payload, so removing the second policy owner prevents valid four-marker requests from becoming invalid five-marker requests. The PR is member-authored, cleanly mergeable, fully green on relevant checks, and has no concrete correctness or security blocker.

## What This Changes

The PR removes Anthropic Vertex’s redundant post-transport cache-control policy pass, forwards caller payload hooks unchanged, and adds regression coverage for preserving the shared transport’s four-marker budget.

## Best Possible Solution

Keep shared Anthropic request construction as the sole cache-control budget owner, with the Vertex adapter limited to SDK/client adaptation and transparent forwarding of finalized-payload hooks.

## Reproduction Assessment

Yes at source level. The shared transport finalizes a four-marker payload before the current Vertex hook reapplies cache policy to the uncached dynamic suffix, matching the linked production five-marker rejection; this review did not execute a live Vertex request.

## Solution Assessment

Yes. Removing the redundant post-build policy owner is narrower and less drift-prone than adding finalized-payload detection or a second marker cap inside the Vertex plugin.
