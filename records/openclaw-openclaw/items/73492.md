The PR adds optional `sessionId` and `sessionKey` to `ProviderCreateStreamFnContext` and threads them through provider stream registration in embedded attempt, compaction, and `/btw` paths.

## Best Possible Solution

Land the additive provider `createStreamFn` context seam after correcting the JSDoc, adding a changelog entry, and covering the passthrough paths while leaving the related `prepareExtraParams` work to #54725.

## Solution Assessment

Yes, with cleanup. Optional fields on the existing provider `createStreamFn` context are the narrow generic seam for custom provider transports, while #54725 is an adjacent `prepareExtraParams` hook change and does not replace this fix.
