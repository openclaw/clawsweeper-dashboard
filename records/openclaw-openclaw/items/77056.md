## Summary

Keep open: current main still matches the reported hot paths, and an open same-author PR (#77057) is already linked with closing syntax to address them. The issue should stay open until that PR is reviewed and merged or explicitly superseded.

## What This Changes

The issue requests reducing routine Control UI `sessions.list` pressure for large session stores by caching CLI-provider classification and bounding chat-turn session refreshes.

## Best Possible Solution

Review and either land or supersede PR #77057 so chat-turn refreshes are bounded and CLI-provider classification is cached without changing explicit Sessions page/search behavior.

## Reproduction Assessment

No, not with high confidence from an executed benchmark in this review. Source inspection does show the current-main chat completion refresh and row-enrichment paths that match the reported large-store slowdown.

## Solution Assessment

Yes, the proposed direction is the narrowest maintainable fix if PR #77057 keeps the cap to routine chat-turn refreshes and caches CLI-provider lookup by stable config/backend inputs. The explicit Sessions page/search flows should remain unrestricted.
