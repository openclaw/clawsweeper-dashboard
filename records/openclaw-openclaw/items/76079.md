## Summary

Keep open. This autofix-enabled PR is a plausible replacement for #76067, but the exact head has blocking integration issues against current main: it drops a current exported API used by MCP plugin-tool serving and replaces the bounded factory cache with an unbounded descriptor cache.

## What This Changes

The PR replaces plugin tool factory-result caching with a descriptor cache captured from validated `api.registerTool(...)` results, updates plugin/tool docs and tests, and adds a changelog entry.

## Best Possible Solution

Rebase onto current main, preserve the standalone plugin-tool preload API, keep descriptor caching bounded and context-safe, then rerun focused plugin-tool tests and the changed gate.

## Reproduction Assessment

Yes for the review findings. The export break is source-provable by comparing current main's `src/mcp/plugin-tools-serve.ts` import with the PR head export block, and the cache growth issue is source-provable from the unbounded `Map` plus high-cardinality context key fields.

## Solution Assessment

No. Descriptor caching is a reasonable direction for the latency work, but this exact implementation needs current-main API integration and an eviction policy before it is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve the standalone preload export:** `src/plugins/tools.ts:34-36`
  - body: Current main has `src/mcp/plugin-tools-serve.ts` importing `ensureStandalonePluginToolRegistryLoaded` from this module, but this branch's export block only re-exports the descriptor-cache reset helpers and the raw head has no `ensureStandalonePluginToolRegistryLoaded` symbol. Rebase the descriptor-cache implementation on top of the current standalone preload path; otherwise MCP plugin-tool serving will fail to build.
  - confidence: 0.93
- **[P2] Bound the descriptor cache:** `src/plugins/tool-descriptor-cache.ts:17`
  - body: The new descriptor cache is a process-global `Map` with no eviction, while its key includes request/session fields such as `sessionId`, `sessionKey`, and `deliveryContext`. The removed factory cache capped entries at 64 per factory, so long-running gateways with many sessions or config objects will retain descriptor arrays indefinitely unless this cache is bounded or its key space is reduced.
  - confidence: 0.88
