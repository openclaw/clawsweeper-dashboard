## Summary

Keep open. Current main still lacks the requested `sessions_send` media/local-file surface: lower Gateway session/chat paths can carry attachments, but the agent-facing tool schema, nested send payload, and A2A announce delivery remain text-only. ## What This Changes

The issue requests image/media file support for inter-agent `sessions_send`, including local media parameters and native platform delivery rather than text-only routing.

## Best Possible Solution

Define a first-class `sessions_send` media contract that reuses existing attachment plumbing while specifying cross-session file access, size/type limits, provenance, target-agent input behavior, native delivery semantics, docs, and regression coverage.

## Reproduction Assessment

Yes for the current limitation: source inspection on current main shows `sessions_send` accepts and forwards only text while adjacent Gateway session/message paths support attachments. No runtime repro is needed because this is a missing feature surface, not an intermittent bug.

## Solution Assessment

Unclear. Reusing existing `sessions.send`/`chat.send` attachment handling is the likely technical direction, but local-file policy and native A2A media delivery need maintainer design first.
