## Summary

Keep open. Current main still has media support on adjacent Gateway/message surfaces, but the agent-visible `sessions_send` tool and its A2A announce path remain text-only, so this specific cross-session media contract is not implemented or superseded.

## What This Changes

Request first-class image/media file support on `sessions_send`, including local media parameters and native channel delivery where possible.

## Best Possible Solution

Define a first-class `sessions_send` media contract that reuses the existing chat/send attachment pipeline and explicitly specifies local-path policy, target-agent input behavior, and native channel announce delivery.

## Reproduction Assessment

Yes. A static reproduction is enough: current main exposes no media/attachment parameter on `sessions_send`, and the execution path constructs nested agent/send payloads from `message` only.

## Solution Assessment

Unclear. Reusing the existing `chat.send`/`sessions.send` attachment plumbing looks like the narrowest direction, but the local file access and A2A/native delivery semantics need maintainer design before an automated fix is safe.
