## Summary

Keep open. Current main still has media/attachment support on adjacent Gateway and message surfaces, but the agent-facing `sessions_send` contract and A2A announce delivery remain text-only, and no canonical duplicate or merged fix covers the requested cross-session media behavior.

## What This Changes

The issue requests image/media support for `sessions_send`, including local file-path input and native channel media delivery between agents.

## Best Possible Solution

Define a first-class `sessions_send` media contract that reuses existing attachment plumbing while specifying cross-session file access, size/type limits, provenance, target-agent input behavior, and native channel delivery semantics.

## Reproduction Assessment

Yes for the current limitation: source inspection on current main shows `sessions_send` accepts and forwards only `message`, while adjacent `sessions.send` and `message` paths already have attachment/media plumbing. No runtime test was needed because this is a missing feature surface, not an intermittent bug.

## Solution Assessment

Unclear. Reusing existing `chat.send` and `sessions.send` attachment handling looks like the narrow technical direction, but the local-file policy and A2A native-delivery behavior need maintainer design before implementation.
