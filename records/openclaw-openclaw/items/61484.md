## Summary

## What This Changes

The issue requests adding image/media support to agent-to-agent `sessions_send`, including local media paths and native channel media delivery where available.

## Best Possible Solution

Define a first-class `sessions_send` media contract that reuses existing attachment plumbing while specifying cross-session file access, size/type limits, provenance, target-agent input behavior, and native delivery semantics.

## Solution Assessment

Unclear. Reusing existing `sessions.send`/`chat.send` attachment handling is the likely technical direction, but local-file policy and native A2A media delivery need maintainer design first.
