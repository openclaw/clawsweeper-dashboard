## Summary

Keep open. Current main still has attachment/media plumbing on adjacent Gateway and message surfaces, but the agent-facing `sessions_send` tool and its A2A announce delivery remain text-only, so the requested cross-session media contract is not implemented or superseded.

## What This Changes

Add image/media support to `sessions_send` so agents can pass local media files between sessions and prefer native channel media delivery.

## Best Possible Solution

Define a first-class `sessions_send` media contract that reuses existing attachment plumbing while specifying cross-session file access, size/type limits, provenance, target-agent input behavior, and native channel delivery.

## Reproduction Assessment

Yes. A static current-main reproduction is enough: `sessions_send` exposes and forwards only `message`, while lower `sessions.send` and `agent` paths already have attachment plumbing that the tool does not use.

## Solution Assessment

Unclear. Reusing existing `chat.send`/`sessions.send` attachment handling looks like the narrowest technical direction, but the local-file policy and A2A native-delivery behavior need maintainer design before implementation.
