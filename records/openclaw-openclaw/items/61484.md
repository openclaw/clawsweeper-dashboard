## Summary

Keep open. Current main has strong media support on adjacent surfaces, including `openclaw message send --media`, RPC/chat attachments, and related implemented media fixes, but the actual `sessions_send` agent tool still exposes and forwards text only. Related reports such as #57254, #58543, #61029, and #63497 are useful context for the existing media pipeline, but none supersedes this specific cross-session `sessions_send` media contract.

## What This Changes

Request support for sending image/media files through `sessions_send`, including local media inputs and native channel media delivery where supported.

## Best Possible Solution

Define a first-class `sessions_send` media contract that reuses the existing gateway attachment/media pipeline, preserves inter-session provenance, and clearly separates target-agent input attachments from native channel delivery of the final A2A announcement.

## Reproduction Assessment

Yes. A static reproduction is enough: the current `sessions_send` tool schema has no media field, and its execution path reads only `message` before starting the target agent run.

## Solution Assessment

Unclear. Reusing the existing `chat.send`/RPC attachment path looks like the narrowest implementation direction, but native platform delivery and local-path access policy need maintainer design before this is safe to automate.
