## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Keep the Claude rescue runner opt-in:** `scripts/systemd/openclaw-rescue-watchdog.service:9`
  - body: The service template sets `OPENCLAW_RESCUE_RUNNER` to the Claude runner, so following the systemd install docs enables AI workspace edits and incident-bundle handoff whenever the timer detects a failure. Keep this unset or commented by default and require an explicit operator opt-in.
  - confidence: 0.92
- **[P2] Clear suppression state after healthy probes:** `scripts/rescue-watchdog.sh:175-177`
  - body: When the probe is healthy, the script exits without clearing the previous fingerprint and timestamp. A fail, recover, then fail-again sequence inside the cooldown window will skip the second real outage as a duplicate.
  - confidence: 0.93
- **[P2] Use collision-free incident directories:** `scripts/rescue-watchdog.sh:196-198`
  - body: Incident IDs are only second-granularity and `mkdir -p` reuses an existing directory, so overlapping timer/manual runs can write into the same incident bundle. Use a unique directory or fail-and-retry creation to keep captures independent.
  - confidence: 0.91
- **[P2] Persist duplicate state before running the rescue command:** `scripts/rescue-watchdog.sh:240`
  - body: The watchdog writes `state.env` only after the runner returns, so an overlapping invocation during a long rescue run sees no latest fingerprint and starts another runner cycle for the same outage. Persist suppression before invoking the runner or guard the run with a lock.
  - confidence: 0.9
