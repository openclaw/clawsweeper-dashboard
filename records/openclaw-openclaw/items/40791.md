## What This Changes

The PR adds an optional external rescue watchdog script, Claude rescue runner, systemd user service/timer templates, setup docs, and several test-only type annotation updates.

## Best Possible Solution

Choose one supported gateway-watchdog boundary, then land a hardened opt-in implementation with tests, sensitive-bundle guidance, and redacted real behavior proof.

## Solution Assessment

No, not as-is. The external watchdog idea may be useful, but this branch needs script/security hardening, contributor proof, and a maintainer-chosen boundary against overlapping core watchdog proposals.

  - confidence: 0.88

  - body: The fingerprint is saved only after the runner and post-health probe finish, so another invocation during a long rescue can start a duplicate capture, notification, and remediation attempt for the same outage.
  - confidence: 0.89

  - body: load_state sources state.env from an environment-configurable state directory without regular-file, owner, permission, or symlink checks, so replacing that file could execute shell code under the operator account on the next watchdog run.
  - confidence: 0.84
