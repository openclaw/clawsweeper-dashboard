## Summary

Keep open: current main still lacks an external rescue loop, and this branch has unresolved source-reproducible correctness and security issues plus missing real behavior proof before it can be considered mergeable.

## What This Changes

This PR adds optional external rescue-watchdog and Claude runner shell scripts, systemd user timer templates, rescue-watchdog docs, and branch-sync test typing cleanups.

## Best Possible Solution

Choose one canonical watchdog direction, then either harden this external script path or supersede it with the core/CLI watchdog work while preserving useful operator guidance and real runtime proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: the stale suppression, same-second incident collision, late duplicate-state persistence, and sourced state-file risk follow from the proposed shell script logic. The broader watchdog behavior still lacks attached after-fix real behavior proof from a real or clearly logged synthetic run.

## Solution Assessment

No, not as-is. The external watchdog may be useful, but it changes operator recovery policy and command-execution surface, overlaps other open watchdog work, and needs script hardening plus maintainer direction before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Clear suppression state after healthy probes:** `scripts/rescue-watchdog.sh:175-177`
  - body: When no incident is detected, the script exits before loading or clearing prior state, so a fail, recover, fail sequence inside the cooldown can be skipped as a duplicate even though it is a fresh outage after recovery.
  - confidence: 0.86
- **[P2] Use collision-free incident directories:** `scripts/rescue-watchdog.sh:196-198`
  - body: The incident id only has second-level precision and mkdir -p reuses an existing path, so overlapping timer/manual runs in the same second can mix artifacts or leave partial bundles instead of independent captures.
  - confidence: 0.88
- **[P2] Persist suppression state before running rescues:** `scripts/rescue-watchdog.sh:250`
  - body: The fingerprint is saved only after the runner and post-health probe finish, so another invocation during a long rescue can start a duplicate capture, notification, and remediation attempt for the same outage.
  - confidence: 0.89

## Security Review

Status: needs_attention

Summary: The diff adds opt-in command execution and diagnostic capture, and the state-file and incident-bundle handling need hardening before merge.

Concerns:

- **[medium] Harden sourced watchdog state:** `scripts/rescue-watchdog.sh:49`
  - body: load_state sources state.env from an environment-configurable state directory without visible regular-file, owner, permission, or symlink checks; replacing that file could cause shell code execution under the operator account on the next watchdog run.
  - confidence: 0.84

  - confidence: 0.82
