Keep open. Current main still lacks the proposed external rescue loop, while this branch has source-verifiable watchdog defects, security hardening gaps, missing real behavior proof, and overlapping open watchdog implementations that need maintainer product/security direction before merge.

## What This Changes

The branch adds optional external rescue watchdog and Claude runner scripts, systemd user timer templates, setup docs, and several test-only TypeScript annotation changes.

## Best Possible Solution

Pick one canonical watchdog model through maintainer review of the overlapping implementation work, then land a hardened and tested implementation with clear incident-data handling, no unrelated test churn, and redacted real behavior proof.

## Solution Assessment

No. The external watchdog idea may be useful, but this branch is not the best mergeable form until maintainers choose the rescue architecture and security posture and the concrete script defects are fixed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Clear suppression state after healthy probes:** `scripts/rescue-watchdog.sh:175-177`
  - body: When incident_reason is empty, the script exits before loading or clearing state.env, so a fail -> recover -> fail sequence inside OPENCLAW_RESCUE_COOLDOWN_SEC can be treated as a duplicate and skip the fresh incident capture, runner, and notification.
  - confidence: 0.86
- **[P2] Use collision-free incident directories:** `scripts/rescue-watchdog.sh:196-198`
  - body: incident_id uses only second-level timestamp precision and mkdir -p reuses existing paths, so overlapping timer/manual runs in the same second can write mixed or partial artifacts into one incident bundle.
  - confidence: 0.88
- **[P2] Persist suppression state before running rescues:** `scripts/rescue-watchdog.sh:250`
  - body: The latest fingerprint is saved only after the runner and post-health probe complete, so another watchdog invocation during a long rescue run can start a duplicate incident capture, notification, and remediation attempt for the same outage.
  - confidence: 0.89

## Security Review

Status: needs_attention

Summary: The diff adds opt-in command execution and diagnostic capture, and this pass found concrete state-file and incident-data hardening gaps.

Concerns:

- **[medium] Harden sourced watchdog state:** `scripts/rescue-watchdog.sh:49`
  - body: load_state sources state.env from an environment-configurable state directory without regular-file, owner, permission, or symlink checks. Replacing that file would cause shell code execution under the operator account on the next watchdog run.
  - confidence: 0.84
- **[medium] Document sensitive incident bundle handling:** `docs/automation/rescue-watchdog.md:38`
  - body: The docs describe bundles containing health/status snapshots, gateway log tails, git state, diff stats, and runner output, and the runner adds the incident directory to Claude context. Operators need explicit permissions, retention, and redaction guidance before enabling this flow.
  - confidence: 0.82
