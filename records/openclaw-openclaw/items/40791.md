  - body: When `incident_reason` is empty, the script exits before loading or clearing `state.env`, so a fail -> recover -> fail sequence inside `OPENCLAW_RESCUE_COOLDOWN_SEC` can be treated as a duplicate and skip fresh incident capture, runner execution, and notification.

  - confidence: 0.88

  - body: The latest fingerprint is saved only after the runner and post-health probe complete, so another watchdog invocation during a long rescue run can start a duplicate incident capture, notification, and remediation attempt for the same outage.
  - confidence: 0.89

## Security Review

Status: needs_attention

Summary: The diff adds opt-in command execution and diagnostic capture, and the state-file plus incident-bundle handling need hardening before merge.
