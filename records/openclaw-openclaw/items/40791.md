Full review comments:

- **[P2] Keep the rescue runner explicitly opt-in:** `scripts/systemd/openclaw-rescue-watchdog.service:9`
  - body: The docs frame the runner as optional, but the shipped systemd service sets `OPENCLAW_RESCUE_RUNNER` by default. An operator following the copy-and-enable instructions would enable Claude Code execution on every incident, expanding the command/edit surface beyond plain monitoring.
  - confidence: 0.9
- **[P2] Clear duplicate suppression after a healthy probe:** `scripts/rescue-watchdog.sh:175-177`
  - body: When the watchdog sees a healthy cycle it exits without resetting `LAST_FINGERPRINT` or `LAST_TRIGGERED_AT`. A fail, recover, fail sequence inside the cooldown can then skip the second outage as a duplicate and miss capture, notification, and runner execution.
  - confidence: 0.88
- **[P2] Generate collision-free incident directories:** `scripts/rescue-watchdog.sh:196-198`
  - body: The incident id only has second resolution and `mkdir -p` reuses an existing path. Two invocations in the same second can write into the same bundle, leaving mixed artifacts instead of independent captures.
  - confidence: 0.86
- **[P2] Persist suppression before launching the runner:** `scripts/rescue-watchdog.sh:240`
  - body: The script writes suppression state only after the runner completes. During a long rescue run, another timer/manual invocation sees no current fingerprint and can launch a second runner against the same outage.
  - confidence: 0.86
