Adds optional external rescue-watchdog and Claude runner shell scripts, systemd user timer templates, setup docs, and narrow test typing updates.

## Solution Assessment

No, not as-is. The external watchdog idea may be useful, but this branch needs script/security hardening and maintainer direction against overlapping core and CLI watchdog proposals before it is the maintainable path.

- **[P2] Keep the rescue runner opt-in:** `scripts/systemd/openclaw-rescue-watchdog.service:9`
  - body: The sample systemd service sets `OPENCLAW_RESCUE_RUNNER` to the Claude runner by default, so following the documented copy/enable path hands incidents to Claude Code in `acceptEdits` mode without a separate explicit runner opt-in.
  - confidence: 0.9
- **[P2] Clear suppression state after healthy probes:** `scripts/rescue-watchdog.sh:175-177`
  - body: When no incident is detected, this branch exits before loading or clearing prior state, so a fail, recover, fail sequence inside the cooldown can be skipped as a duplicate even though it is a fresh outage.
  - confidence: 0.86
- **[P2] Use collision-free incident directories:** `scripts/rescue-watchdog.sh:196-198`
  - body: The incident id only has second-level precision and `mkdir -p` reuses an existing path, so overlapping timer/manual runs in the same second can mix artifacts or leave partial bundles instead of independent captures.
  - confidence: 0.88
- **[P2] Persist suppression state before running rescues:** `scripts/rescue-watchdog.sh:250`
  - body: The fingerprint is saved only after the runner and post-health probe finish, so another invocation during a long rescue can start a duplicate capture, notification, and remediation attempt for the same outage.
  - confidence: 0.89
- **[P2] Parse state without sourcing shell code:** `scripts/rescue-watchdog.sh:51`
  - body: `load_state` sources `state.env` from an environment-configurable state directory without regular-file, owner, permission, or symlink checks, so replacing that file could execute shell code under the operator account on the next watchdog run.
  - confidence: 0.84

## Security Review
