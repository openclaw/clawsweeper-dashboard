## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Keep the rescue runner explicitly opt-in:** `scripts/systemd/openclaw-rescue-watchdog.service:9`
  - body: The service template sets `OPENCLAW_RESCUE_RUNNER` to the Claude runner by default. An operator who enables the timer would also enable AI-assisted workspace edits on incidents, which contradicts the optional-runner boundary and needs an explicit opt-in instead.
  - confidence: 0.9
- **[P2] Reset duplicate suppression after healthy probes:** `scripts/rescue-watchdog.sh:175-177`
  - body: Healthy runs exit without clearing `state.env`, so a fail, recover, fail sequence within the cooldown can treat the second outage as a duplicate and skip capture, notification, and runner execution. Clear the suppression state after a healthy probe or include the healthy cycle in the state machine.
  - confidence: 0.88
- **[P2] Generate collision-free incident directories:** `scripts/rescue-watchdog.sh:196-198`
  - body: Incident IDs only use second precision and `mkdir -p` reuses an existing directory, so overlapping timer/manual runs in the same second can write mixed or partial bundles. Add a unique suffix or use atomic `mktemp -d` under the incident root.
  - confidence: 0.86
- **[P2] Persist suppression before running the rescue command:** `scripts/rescue-watchdog.sh:250`
  - body: The duplicate-suppression state is written only after the runner finishes, leaving long rescue runs invisible to overlapping invocations. Save state before launching the runner or guard the whole run with a lock so the same outage cannot trigger duplicate remediation.
  - confidence: 0.84
