Full review comments:

- **[P2] Require the model before memory-lancedb defaults it:** `extensions/openai-compatible-embeddings/embedding-provider.ts:84-88`
  - body: This missing-model guard does not catch the documented memory-lancedb case: current main defaults an omitted `embedding.model` to `text-embedding-3-small` before `adapter.create` receives `options.model`. With `provider: "openai-compatible"` plus a `baseUrl` but no model, the adapter will send the OpenAI default model to the local server instead of failing fast as promised; add provider-specific validation before defaulting or pass enough raw config to detect omission.
  - confidence: 0.88
- **[P2] Wire advertised headers through memory-lancedb:** `extensions/openai-compatible-embeddings/embedding-provider.ts:30`
  - body: The provider advertises extra request headers, but `memory-lancedb` still rejects unknown `embedding.headers` keys and only forwards `apiKey`/`baseUrl` into `remote`. A user following the PR body or changelog header claim gets `embedding config has unknown keys: headers` before this adapter runs, so either add schema/pass-through support or remove the public/header support claim from this bundled provider path.
  - confidence: 0.86
- **[P3] Add labeler coverage for the new plugin:** `extensions/openai-compatible-embeddings/openclaw.plugin.json:2`
  - body: This introduces a new bundled plugin directory, but the PR does not add a matching `.github/labeler.yml` entry. Repo policy requires new plugin surfaces to update labeler/GitHub labels, and without it future changes under this plugin will not get the expected extension label for routing.
  - confidence: 0.82
