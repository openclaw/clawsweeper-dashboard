- **[P2] Require an explicit model for openai-compatible:** `extensions/openai-compatible-embeddings/embedding-provider.ts:84-88`
  - body: The missing-model guard cannot fire through memory-lancedb because that config parser defaults omitted `embedding.model` to `text-embedding-3-small` before this adapter runs. A config with `provider: "openai-compatible"`, `baseUrl`, and `dimensions` but no model will send the OpenAI default model name to the local server instead of failing fast, so please make the model explicit for this provider or preserve whether it was omitted.
  - confidence: 0.87
- **[P2] Wire advertised headers through memory-lancedb:** `extensions/openai-compatible-embeddings/embedding-provider.ts:30`
  - body: This provider advertises `embedding.headers`, but memory-lancedb still rejects unknown embedding keys and only forwards `apiKey` and `baseUrl` into `remote`. Users following the advertised header path will fail config validation before this adapter runs, so either add schema/pass-through support with tests or remove the public header claim.
  - confidence: 0.88

  - body: This adds a new bundled plugin surface, but `.github/labeler.yml` has no `extensions/openai-compatible-embeddings/**` entry. Future changes to this plugin will miss a scoped plugin label, so please add the labeler rule alongside the new plugin.
