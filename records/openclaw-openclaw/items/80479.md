  - body: Adding this manifest/package makes `openai-compatible-embeddings` part of the generated plugin inventory, but the PR does not add `docs/plugins/reference/openai-compatible-embeddings.md` or update the generated inventory/reference indexes. `scripts/generate-plugin-inventory-doc.mjs --check` renders from every `extensions/*/package.json` plus `openclaw.plugin.json`, so release/preflight inventory checks will go stale and the new provider will be missing from the canonical plugin reference.

  - body: The new adapter advertises per-plugin `embedding.headers` and reads `options.remote?.headers`, but `memory-lancedb` currently accepts and forwards only `apiKey` and `baseUrl` into `remote`. Any operator who needs a reverse-proxy or custom auth header gets a config validation failure or a dropped header, so either add `headers` to the memory-lancedb config path and pass it through, or remove the advertised option.
  - confidence: 0.82
