## Summary

Keep open: the dedicated provider direction matches the reported local-server embedding gap and the PR supplies real terminal proof, but the patch should fix the advertised `headers` support mismatch before merge and still needs maintainer product review for bundling a new code plugin.

## What This Changes

The PR adds a bundled `openai-compatible-embeddings` memory embedding provider plugin with docs, tests, package metadata, a lockfile entry, and a Vitest config.

## Best Possible Solution

Accept the provider only after the memory-lancedb config, docs, and tests agree on supported fields and maintainers confirm that this generic provider belongs in the bundled plugin set; otherwise keep the workaround docs and route the provider externally.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature PR. The PR body supplies live after-change terminal proof against llama.cpp, and source inspection confirms current main only offers the existing `openai` and `lmstudio` alternatives for this use case.

## Solution Assessment

No, not as written. A dedicated no-warmup provider is a maintainable direction, but the patch advertises per-plugin `headers` support that memory-lancedb does not currently accept or forward.
