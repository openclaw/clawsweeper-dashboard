## Summary

The PR targets the right Discord message-loss bug, but the patch is incomplete: removing the handler guard only reaches `deliverDiscordReply`, and the shared durable outbound planner still drops `isReasoning` payloads before Discord sends them. The branch also has mock/unit proof only, not real Discord after-fix proof.

## What This Changes

The branch removes two Discord `isReasoning` suppression guards in `processDiscordMessageInner` and updates handler tests to expect reasoning payload delivery.

## Best Possible Solution

Carry opted-in Discord reasoning payloads through durable delivery as visible `Thinking` messages while preserving hidden-reasoning suppression, then add durable-send regression coverage and real Discord proof.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: `/reasoning on` emits `isReasoning` block replies, current Discord handler suppresses them, and this PR still leaves the lower durable planner suppression in place.

## Solution Assessment

No. Removing the local handler guards is only a partial fix; the best fix must transform or exempt explicit Discord reasoning before durable planning, or at the Discord adapter boundary, with durable-path regression coverage.
