Keep open: this is a plausible focused bug-fix PR with sufficient terminal proof and no blocking review findings, but it intentionally changes plugin discovery policy for untracked global source directories and still needs maintainer acceptance plus normal CI completion.

## What This Changes

The PR changes plugin discovery and package-entry resolution so untracked global source checkouts may load TypeScript package entries while install-record-managed paths remain strict, with tests, docs, changelog, and one Google test assertion update.

## Best Possible Solution

Land this PR or a maintainer-adjusted variant that preserves managed-install strictness, allows intentional untracked source checkouts, and documents the stale untracked-directory surprise case.

## Reproduction Assessment

Yes. Current main is source-reproducible: a global package entry pointing at TypeScript without built JavaScript follows the package-entry path and is skipped with the compiled-runtime warning; I did not run a live gateway repro because this review is read-only.

## Solution Assessment

Yes, with maintainer policy acceptance. Classifying managed paths from install records is narrower than flipping manifest precedence and preserves strict behavior for managed installs and explicit runtime entries; a startup diagnostic for untracked source loads could remain a follow-up.
