## Summary

Keep open. Current main still has the source-reproducible plugin discovery bug, and this active PR is the linked implementation candidate; I found no discrete blocking correctness issue, but maintainer review is still needed for the intentional global untracked-source loading policy.

## What This Changes

The PR changes plugin discovery and package-entry resolution so untracked global source checkouts may load TypeScript package entries while install-record-managed paths remain strict, with tests, docs, changelog, and one Google test type assertion update.

## Best Possible Solution

Land this PR or a maintainer-adjusted variant that keeps install-record-managed paths strict, allows intentional untracked source checkouts, and documents or diagnoses the upgrade-surprise case.

## Reproduction Assessment

Yes. Current main is source-reproducible: global directory discovery resolves package entries before manifest/default fallback, and global TypeScript package entries without built JavaScript hit the compiled-runtime warning path; I did not run a live gateway repro because this review is read-only.

## Solution Assessment

Yes, with maintainer policy acceptance. Classifying managed paths from install records is narrower than flipping manifest precedence and preserves strict behavior for managed installs and explicit runtime entries; a startup diagnostic for untracked source loads could be a follow-up.
