## Summary

Keep open: this PR is a plausible narrow bug fix and I found no discrete code-review bug, but it still needs maintainer merge/policy handling because it changes the plugin discovery contract and the current head has a failing `check-test-types` gate.

## What This Changes

The PR distinguishes managed install-record plugin directories from untracked global source checkouts so local TypeScript package entries can load while managed installs remain compiled-output strict, with regression tests, docs, and a changelog entry.

## Best Possible Solution

Land the install-record based discovery policy after maintainers accept the contract and the failing type gate is resolved, preserving the managed-vs-untracked regression tests and docs.

## Reproduction Assessment

Yes. Current main is source-reproducible: global directory discovery resolves package entries before fallback, and the package-entry resolver rejects global TypeScript package entries without built JavaScript output; I did not run a live repro because this review was read-only.

## Solution Assessment

Yes, with maintainer policy approval. Scoping the relaxed TypeScript fallback to untracked global directories via install records is narrower than flipping package-vs-manifest precedence and preserves explicit runtime entries plus managed install strictness.
