## Summary

Keep open: this focused draft PR appears to address the linked plugin discovery problem and I found no blocking correctness issue, but it still needs maintainer policy approval and normal merge readiness before landing.

## What This Changes

The PR separates managed install-record plugin directories from untracked global source checkouts so TypeScript package entries can load locally while managed installs remain compiled-output strict, with tests, docs, and changelog coverage.

## Best Possible Solution

Land the narrow managed-vs-untracked discovery contract if maintainers accept the policy, preserving strict install-record handling and the added regression coverage.

## Reproduction Assessment

Yes. Current main is source-reproducible: global package entries go through strict built-runtime resolution, and the existing discovery test expects source-only TypeScript package entries in the global extensions directory to be skipped.

## Solution Assessment

Yes, with maintainer policy approval. Scoping strict compiled-runtime validation to install-record paths is narrower than flipping all package-vs-manifest precedence and preserves explicit runtimeExtensions plus bundled fallback behavior.
