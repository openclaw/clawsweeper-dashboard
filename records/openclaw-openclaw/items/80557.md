## Summary

Keep open: this is an active contributor PR for a source-reproducible plugin discovery bug that current main still has. The patch looks directionally correct with no blocking findings found, but maintainer acceptance is still needed for the policy that untracked global plugin directories count as local source checkouts.

## What This Changes

The PR lets untracked global source-checkout plugin directories load TypeScript package entries while keeping install-record paths strict, with regression tests, docs, changelog, and a small scheduled-turn test assertion update.

## Best Possible Solution

Land the install-record based discovery policy if maintainers accept the contract, with an explicit operator-surprise note or diagnostic if they want that upgrade behavior called out before merge.

## Reproduction Assessment

Yes. Current main is source-reproducible: global package discovery resolves TypeScript package entries before manifest/index fallback, and the resolver rejects global-origin TypeScript package entries without built JavaScript output.

## Solution Assessment

Yes, with maintainer policy acceptance. Scoping the relaxed TypeScript fallback to directories not named by install records is narrower than flipping package-vs-manifest precedence and preserves strict managed installs.
