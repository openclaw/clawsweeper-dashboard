## Summary

Keep open: current main and v2026.6.10 still have a source-reproducible Slack outbound mirror route that can create a new thread-suffixed delivery-mirror session instead of reusing the conversation's canonical thread session.

## What This Changes

The issue asks OpenClaw to mirror Slack outbound replies into the existing conversation thread session instead of accumulating per-reply delivery-mirror sessions and incorrect transcript headers.

## Best Possible Solution

Fix Slack outbound mirror routing so inherited same-thread replies mirror into the canonical Slack thread session while preserving explicit reply targets and ordinary top-level channel sessions.

## Reproduction Assessment

Yes, at source level: the outbound mirror path can resolve and create a thread-suffixed route before persisting a missing session entry, while reporter evidence shows the same shape in a live Slack deployment. I did not run a live Slack workspace replay.

## Solution Assessment

Yes. The narrow maintainable direction is to canonicalize Slack inherited same-thread reply routing for delivery mirrors, not to add a config flag or change generic precedence for every channel.
