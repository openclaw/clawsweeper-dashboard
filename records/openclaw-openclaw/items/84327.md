## Label Justifications

- `P2`: The reported symptom was user-visible Telegram message loss with a limited group-chat delivery surface.
- `impact:message-loss`: The issue concerns generated assistant replies not being delivered to a messaging channel.

## Summary

Current main and v2026.5.19 no longer match the reported default-settings behavior: normal group/channel requests default to automatic final delivery, while explicit `message_tool` and ambient room-event modes remain intentionally private with diagnostics.

## What This Changes

The issue asks for Telegram group-chat assistant text to remain visible when the model produces final text but skips the `message` tool.

## Best Possible Solution

Keep the shipped behavior: normal group requests should use automatic final delivery by default, and operators should opt into `message_tool` or ambient room events only when private final text is desired unless the agent explicitly sends.

## Reproduction Assessment

No for the reported default-settings path on current main. Source, tests, and shipped docs show normal group/channel requests now default to automatic final delivery; only explicit `message_tool` or `room_event` paths keep final text private by design.

## Solution Assessment

Yes. Restoring automatic delivery for normal group requests fixes the default silent-drop path without weakening the stricter ambient/shared-room delivery contract.
