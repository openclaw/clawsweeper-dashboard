- `impact:message-loss`: The report is about generated assistant text not appearing in Telegram when no message-tool delivery occurs.

## Summary

Close as superseded by the prior maintainer policy pass: current main intentionally keeps normal final text private in `message_tool_only` group/channel mode, logs suppressed finals for diagnostics, and documents the supported alternatives instead of auto-posting missed message-tool replies.

## What This Changes

The issue requests automatic fallback delivery for Telegram group/chat turns when the model emits final assistant text but does not call the `message` tool.

## Best Possible Solution

Keep the current privacy-preserving behavior and diagnostics; users who want normal assistant finals posted visibly should use `messages.groupChat.visibleReplies: "automatic"`, while tool-only rooms should rely on a tool-reliable model/runtime.

## Reproduction Assessment

Yes, at source level: current code and tests show `message_tool_only` group/channel finals are suppressed unless explicitly marked for delivery. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

No, not as requested: auto-posting missed finals conflicts with the prior maintainer policy decision and current docs/tests. The supported solution is diagnostics plus the existing `automatic` visible-replies mode when visible normal finals are desired.
