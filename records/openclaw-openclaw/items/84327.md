## What This Changes

The issue reports that Telegram group/topic assistant text can be generated but silently suppressed when `message_tool_only` mode is active and the model skips the `message` tool.

## Best Possible Solution

Review and land the linked scoped fallback PR, ideally with regression coverage for undelivered assistant text and a negative case where side-effect delivery already succeeded.

## Reproduction Assessment

Yes from source inspection and reporter trajectory data, though I did not run a live Telegram repro. Current main uses `message_tool_only` for Telegram room events and suppresses unmarked assistant payloads, matching the reported silent drop path.

## Solution Assessment

Yes, likely. The linked PR is the narrow implementation path because it marks fallback assistant payloads only when the message tool, block streaming, or another side-effect delivery path did not already send content.
