## What This Changes

The issue asks OpenClaw to deliver Telegram group-chat assistant text as a fallback when `message_tool_only` mode is active and the model skips the `message` tool.

## Best Possible Solution

Use https://github.com/openclaw/openclaw/pull/84328 as the decision point: land a tested fallback if maintainers want this policy change, or close the PR and point users to `messages.groupChat.visibleReplies: "automatic"` or a stronger tool-calling model.
