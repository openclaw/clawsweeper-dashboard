## Summary

Keep open. Current main still accepts route binding comments but does not carry the matched binding comment into the route result, session metadata, Gateway session row display name, or Control UI session label path; the related Feishu display-name PR at https://github.com/openclaw/openclaw/pull/73399 is still open and does not cover the requested binding-comment priority.

## What This Changes

The issue asks the Control UI sessions list to show a friendly label for Feishu/OpenID session rows by preferring `binding.comment`, then agent name, then the technical identifier.

## Best Possible Solution

Add a generic session display-label contract sourced from the matched route binding comment, preserve explicit manual session labels/display names as overrides, and fall back to agent identity/name before technical keys for all `sessions.list` consumers.

## Reproduction Assessment

Yes, as a high-confidence source-level reproduction. Configure a Feishu direct route binding with `comment: "税务师"`, receive a DM from an `ou_...` user, and current main has no path from the matched binding comment into route results, session metadata, Gateway rows, or the Control UI label.

## Solution Assessment

Yes. Solving this at the route/session metadata boundary is the narrow maintainable approach; a UI-only formatter cannot know which binding matched and would leave non-UI `sessions.list` consumers inconsistent.
