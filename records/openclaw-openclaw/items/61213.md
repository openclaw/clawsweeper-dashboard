## Summary

Keep open. Current main and v2026.6.8 still support route binding comments in config but do not propagate them through route resolution, session metadata, Gateway session rows, or the Sessions view, so the Control UI readability request remains valid.

## What This Changes

Request to show human-friendly Web Control UI session labels by preferring `binding.comment`, then agent name, before falling back to opaque open_id or session keys.

## Best Possible Solution

Carry a matched route display label through the generic route/session metadata and Gateway row paths, preferring manual labels where applicable, then `binding.comment`, then configured agent identity/name, then the existing technical fallback.

## Reproduction Assessment

Yes, at source level: a route binding can contain `comment`, but current route resolution, session metadata, Gateway row construction, and the Sessions view never carry that value into `sessions.list` or the Web Control UI. I did not run a live Feishu tenant reproduction in this read-only pass.

## Solution Assessment

Yes, the best direction is the shared route/session metadata boundary rather than a Feishu-only formatter or a UI-only string guess. That keeps Web Control UI and `sessions.list` consistent while preserving existing manual session labels.
