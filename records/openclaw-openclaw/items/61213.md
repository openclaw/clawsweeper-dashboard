## Summary

Keep open. Current main still accepts `binding.comment`, but route resolution, session metadata, Gateway session rows, and the Control UI do not carry the matched binding comment into session display labels; the open account-name fallback PR covers a related but narrower path and does not satisfy this requested `binding.comment -> agent.name -> technical id` priority.

## What This Changes

Request to make Web Control UI session labels prefer matched route binding comments, then agent names, before Feishu open IDs or raw session keys.

## Best Possible Solution

Carry the matched route's friendly display label through the generic route/session metadata boundary, preserve explicit session labels/display names as overrides, and keep raw IDs as the final fallback for all `sessions.list` consumers.

## Reproduction Assessment

Yes, at source level. Configure a Feishu direct route binding with `comment: "税务师"`, receive a DM from the matching `ou_...` peer, and current main has no code path carrying that matched comment into the resolved route, session metadata, Gateway row, or Control UI label.

## Solution Assessment

Yes, the maintainable direction is to solve this at the route/session metadata boundary rather than only formatting the UI. That keeps Control UI and other `sessions.list` consumers consistent while preserving existing manual labels and display names.
