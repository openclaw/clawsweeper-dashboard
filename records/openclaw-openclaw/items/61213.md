## Summary

Keep open. Current main still supports `binding.comment` in config but does not carry it through route resolution, inbound session metadata, Gateway session rows, or the Sessions view, so the central readability request remains valid.

## What This Changes

The issue requests friendlier Control UI identifiers for Feishu/OpenAPI-style sessions, preferring binding comments or agent names over raw open IDs and long session keys.

## Best Possible Solution

Carry a matched route display label through the generic route/session metadata boundary, preferring `binding.comment`, then configured agent name, then existing technical identifiers while preserving manual session labels.

## Reproduction Assessment

Yes at source level: a Feishu route binding can contain `comment`, but current route resolution and session metadata do not carry that value into `sessions.list` or the Sessions view. I did not run a live Feishu tenant reproduction in this read-only review.

## Solution Assessment

Yes, the best fix is at the generic route/session metadata boundary, not a Feishu-only or UI-only formatter. That lets Web Control UI and other `sessions.list` consumers share the same friendly fallback while keeping existing manual labels authoritative.
