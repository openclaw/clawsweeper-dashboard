## Summary

Keep open. Current main and v2026.6.8 still define `bindings[].comment`, but route resolution, session metadata, Gateway session rows, and the Control UI Sessions view do not carry that comment into a human-readable display label.

## What This Changes

The issue asks for the Web Control UI to prefer `binding.comment`, then `agent.name`, then the technical session/open-id identifier when displaying channel-bound sessions.

## Best Possible Solution

Carry a matched route display label through the shared route/session metadata and Gateway row path, preserving manual session labels first, then using `binding.comment`, then configured agent identity/name, then the existing technical fallback.

## Reproduction Assessment

Yes, at source level: a route binding can contain `comment`, but current route resolution, session metadata, Gateway row construction, and the Sessions view never carry that value into `sessions.list` or the Web Control UI. I did not run a live Feishu tenant reproduction in this read-only pass.

## Solution Assessment

Yes. The best direction is the shared route/session metadata boundary rather than a Feishu-only formatter or UI-only string guess, while preserving existing manual session labels as the top priority.
