## Summary

Keep open. Current main still accepts `binding.comment`, but route resolution, session metadata, Gateway rows, and the Control UI do not propagate it into session display labels; https://github.com/openclaw/openclaw/pull/79251 was a closed narrower account-name fallback, and https://github.com/openclaw/openclaw/issues/84753 is broader cross-channel display-name policy work.

## What This Changes

The issue requests friendlier Web Control UI session labels, preferring `binding.comment` then `agent.name` before opaque Feishu `ou_...` IDs or long technical session keys.

## Best Possible Solution

Carry the matched route's friendly display label through the generic route/session metadata boundary, preserve explicit session labels and display names as overrides, and keep raw IDs as the final fallback for all `sessions.list` consumers.

## Reproduction Assessment

Yes, at source level. Configure a Feishu direct route binding with `comment: "税务师"`, receive a DM from the matching `ou_...` peer, and current main has no path carrying that matched comment into the resolved route, session metadata, Gateway row, or Control UI label.

## Solution Assessment

Yes. The maintainable path is to solve this at the route/session metadata boundary rather than only formatting the UI, so Control UI and other `sessions.list` consumers stay consistent while explicit labels still win.
