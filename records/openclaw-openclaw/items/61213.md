## Summary

Keep open. Current main and the latest release still expose `bindings[].comment`, but the route result, persisted session metadata, Gateway session row/search projection, and Control UI Sessions view do not carry that value into the human-readable display path; the related open PRs only cover truncation or account-name fallback.

## What This Changes

Issue requests that the Web Control UI prefer a human-readable `binding.comment`, then `agent.name`, before falling back to opaque Feishu `open_id` or raw session-key identifiers.

## Best Possible Solution

Carry a matched route display label through the shared route/session metadata and Gateway row/search path, preserving manual labels first, then `binding.comment`, then configured agent identity/name, then the existing technical fallback.

## Reproduction Assessment

Yes, at source level: a route binding can contain `comment`, but current route resolution, session metadata, Gateway row/search construction, and the Sessions view never carry that value into `sessions.list` or the Web Control UI. I did not run a live Feishu tenant reproduction in this read-only review.

## Solution Assessment

Yes, the best fix is the shared route/session metadata boundary rather than a Feishu-only formatter or UI string guess. It should preserve existing manual labels and coordinate with the open truncation/account-name PRs as partial adjacent work.
