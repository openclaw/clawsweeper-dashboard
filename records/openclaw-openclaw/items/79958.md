## Summary

Keep open: the branch removes the usage request for the custom/local OpenAI-compatible paths that need it to emit usage metadata, so it does not fix the linked zero-usage bug and conflicts with current main's tests and usage path.

## What This Changes

This PR changes OpenAI Chat Completions streaming request construction to send `stream_options.include_usage` only when `compat.supportsUsageInStreaming` is true.

## Best Possible Solution

Preserve the usage request for affected OpenAI-compatible streams and fix stream consumption or fallback handling so trailing llama.cpp timing/usage reaches normalized transcript and session usage.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Source-level yes, live no: current main only records SDK-yielded usage chunks, the OpenAI SDK skips SSE data after `[DONE]`, and the linked issue includes llama.cpp log/curl evidence for a trailing usage chunk.

## Solution Assessment

Is this the best way to solve the issue? No: suppressing `stream_options.include_usage` removes the request that makes affected custom/local backends emit usage metadata, so saved usage can still remain zero.
