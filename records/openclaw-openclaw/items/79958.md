## Summary

Keep open: the PR changes the right area, but this patch would stop custom/local OpenAI-compatible backends from requesting the usage/timings data that the linked llama.cpp report needs, and the supplied proof is only a synthetic payload check rather than an after-fix OpenClaw run.

## What This Changes

This PR gates OpenAI Chat Completions streaming `stream_options.include_usage` on `compat.supportsUsageInStreaming` in `src/agents/openai-transport-stream.ts`.

## Best Possible Solution

Keep requesting streaming usage for local/custom OpenAI-compatible backends, then preserve the final usage chunk or recover llama.cpp timing metadata with focused regression coverage and redacted real-run proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Source-level yes, live no: current main records only SDK-yielded usage chunks while OpenAI Node v6.37.0 skips post-`[DONE]` SSE data, and the linked issue supplies llama.cpp logs plus direct curl evidence.

## Solution Assessment

Is this the best way to solve the issue? No: gating off `stream_options.include_usage` removes the request that makes the affected custom/local backend emit usage or timings, so the saved usage can remain `0/0/0`.
