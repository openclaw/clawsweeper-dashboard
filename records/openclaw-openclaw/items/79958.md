## Summary

Keep open: the PR targets a real OpenAI-compatible streaming usage bug, but this patch removes the usage request for the affected custom backends instead of preserving the usage or timings data that `/status` and compaction need, and the supplied proof is synthetic rather than an after-fix OpenClaw run.

## What This Changes

The PR changes `buildOpenAICompletionsParams` so Chat Completions streaming requests include `stream_options.include_usage` only when `compat.supportsUsageInStreaming` is true.

## Best Possible Solution

Keep requesting streaming usage for local/custom OpenAI-compatible backends, then capture the final usage or llama.cpp timings data reliably and add focused regression coverage plus redacted real-run proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Source-level yes, live no: current main only records SDK-yielded usage chunks, OpenAI Node 6.37.0 skips post-`[DONE]` SSE messages, and the linked issue supplies llama.cpp logs plus direct curl evidence.

## Solution Assessment

Is this the best way to solve the issue? No: gating off `stream_options` removes the request that makes the affected backend emit usage data, so the zero-usage behavior can remain instead of being fixed.
