## Summary

Keep this PR open, but it should not merge as written: it suppresses the usage request for the affected false-compat custom/local OpenAI-compatible backends instead of preserving the llama.cpp usage data reported in https://github.com/openclaw/openclaw/issues/79897.

## What This Changes

The PR changes `buildOpenAICompletionsParams` so `stream_options: { include_usage: true }` is only sent when `compat.supportsUsageInStreaming` is true.

## Best Possible Solution

Preserve llama.cpp usage visibility by fixing the OpenAI-completions stream handling to capture the final usage chunk or normalize llama.cpp timing metadata, with focused regression coverage and redacted live-run proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Source-level yes, live no: current main requests streaming usage, the linked issue provides llama.cpp logs plus saved `0/0/0` usage, and the parser only records yielded usage chunks, but I did not run a live llama.cpp server.

## Solution Assessment

Is this the best way to solve the issue? No: this patch removes the usage request for the affected false-compat backends rather than making OpenClaw consume or recover the usage data that `/status` and compaction need.
