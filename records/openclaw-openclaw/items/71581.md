## Summary

Keep open. Current main still preserves same-agent per-channel forced heartbeat keys for this path, and https://github.com/openclaw/openclaw/pull/72648 is an open unmerged implementation PR that targets this issue.

## What This Changes

Extend `session.dmScope: "main"` so exec-event and cron-event heartbeat wakes for single-owner agents resolve to the agent main session instead of a per-channel DM session.

## Best Possible Solution

Review, repair, land, or replace https://github.com/openclaw/openclaw/pull/72648 with a narrow dmScope-aware routing seam that normalizes system-event enqueue keys and forced heartbeat targets together while preserving current exec trust and delivery contracts.

## Reproduction Assessment

Yes, at source level. Enqueue an exec or cron system event under a same-agent per-channel direct-message key, request an `exec-event` or `cron` wake, and current main resolves and peeks that per-channel key rather than `agent:<id>:main`.

## Solution Assessment

Yes, directionally. The maintainable solution is to normalize enqueue and wake keys together under the existing single-owner `dmScope=main` policy; changing only heartbeat resolution would strand queued events under the original key.
