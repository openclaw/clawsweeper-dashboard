## Summary

Keep open. Current `main` still routes same-agent forced exec-event and cron-event heartbeat wakes by the literal per-channel session key, and the issue is paired with open unmerged PR #72648 using closing syntax.

## What This Changes

Honor `session.dmScope=main` for same-agent exec-event and cron-event heartbeat wakes so single-owner DM setups process those internal events in the agent main session.

## Best Possible Solution

Review, land, or replace PR #72648 with a narrow change that normalizes same-agent direct-message system-event enqueue keys and forced heartbeat wake targets to the agent main session only when the existing `dmScope=main` single-owner condition is satisfied.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to configure `session.dmScope=main`, enqueue an exec or cron system event under `agent:main:telegram:default:direct:<userId>`, and request an `exec-event` or `cron-event` wake with that forced key; current `main` resolves and peeks the per-channel queue instead of `agent:main:main`.

## Solution Assessment

Yes. The requested direction is the narrow maintainable fix if enqueue and wake routing are normalized together; changing only heartbeat-session resolution would be incomplete because queued system events are keyed before preflight peeks and drains them.
