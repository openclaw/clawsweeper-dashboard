## Summary

Keep open. Current main still routes same-agent exec-event and cron-event heartbeat wakes by the literal per-channel session key, and open PR #72648 references this issue with closing syntax, so closure should wait for a merged or replacement fix.

## What This Changes

Honor `session.dmScope=main` when resolving target sessions for `exec-event` and `cron-event` heartbeat wakes.

## Best Possible Solution

Merge a narrow fix that applies the existing single-owner `dmScope=main` condition to both system-event enqueue keys and heartbeat wake targets while preserving non-main and multi-owner behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to configure `session.dmScope=main`, enqueue an exec or cron system event under a same-agent per-channel direct key, and request an `exec-event` or `cron-event` wake with that forced key; current main resolves and peeks that per-channel queue.

## Solution Assessment

Yes, if enqueue and wake routing are normalized together. Changing only heartbeat-session resolution would be incomplete because queued system events are keyed before preflight peeks and consumes them.
