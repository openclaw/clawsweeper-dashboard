## Summary

Keep open. Current main still preserves same-agent exec-event and cron-event forced session keys literally instead of applying the `session.dmScope=main` direct-message consolidation rule, and the issue is paired with open unmerged PR #72648 using closing syntax.

## What This Changes

The issue requests that `session.dmScope=main` also collapse exec-event and cron-event heartbeat wake routing into the agent main session for single-owner direct-message agents.

## Best Possible Solution

Review, land, or replace PR #72648 with a narrow core change that normalizes same-agent direct-message system-event enqueue keys and forced heartbeat wake targets to the agent main session only when the existing `dmScope=main` single-owner condition is satisfied, while preserving explicit group, channel, cross-agent, subagent, and custom cron routing.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to configure `session.dmScope=main` for a single-owner direct-message setup, enqueue an exec or cron system event under `agent:main:telegram:default:direct:<userId>`, and request an exec-event or cron wake with that forced key; current main resolves and peeks that per-channel queue instead of `agent:main:main`.

## Solution Assessment

Yes, the requested direction is the narrow maintainable fix if enqueue and wake routing are normalized together. Using only heartbeat-session resolution would be incomplete because queued system events are keyed before preflight peeks and drains them.
