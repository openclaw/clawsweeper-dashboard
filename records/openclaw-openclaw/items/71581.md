## Summary

Keep open. Current main still preserves same-agent forced session keys for heartbeat preflight and exact system-event queues, and open PR #72648 references this issue with closing syntax but is not merged.

## What This Changes

Route exec-event and cron-event heartbeat wakes and queued system events to the agent main session when `session.dmScope=main` is effective for a single-owner agent.

## Best Possible Solution

Review, land, or replace PR #72648 with a narrow fix that normalizes same-agent DM system-event enqueue keys and forced heartbeat wake targets to the main session only under the existing `dmScope=main` single-owner condition, with exec and cron coverage for queue key agreement.

## Reproduction Assessment

Yes. Configure `session.dmScope=main`, enqueue an exec or cron system event under a same-agent per-channel direct-message key, and request the corresponding forced heartbeat wake; current main resolves and peeks that per-channel queue instead of `agent:<id>:main`.

## Solution Assessment

Yes. The requested direction is the narrow maintainable fix if enqueue and wake routing are normalized together; closing the issue before the linked PR merges would be premature.
