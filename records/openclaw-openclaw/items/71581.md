## Summary

Keep open. Current main still preserves same-agent per-channel forced exec-event and cron-event heartbeat session keys instead of applying `session.dmScope=main`, and open PR #72648 references this issue with closing syntax but is not merged.

## What This Changes

Honor `session.dmScope=main` for same-agent exec-event and cron-event heartbeat wakes and queued system events so internal follow-ups land in the agent main session.

## Best Possible Solution

Review, land, or replace PR #72648 with a narrow fix that normalizes both enqueue keys and forced heartbeat wake targets under the existing single-owner `dmScope=main` precondition.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to configure `session.dmScope=main`, enqueue an exec or cron event under `agent:main:telegram:default:direct:<userId>`, and request an `exec-event` or `cron-event` wake; current main resolves and peeks that per-channel queue instead of `agent:main:main`.

## Solution Assessment

Yes. Normalizing both system-event enqueue keys and forced heartbeat targets is the narrow maintainable direction; changing only heartbeat-session resolution would be incomplete because queued events are keyed before preflight peeks and drains them.
