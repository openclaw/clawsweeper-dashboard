## Summary

Keep open. Current main still preserves same-agent forced heartbeat and cron session keys literally, while the single-owner `session.dmScope=main` rule is wired into inbound DM routing; the issue is also paired with open, unmerged PR #72648 using closing syntax.

## What This Changes

The issue requests that `session.dmScope=main` also route exec-event and cron-event heartbeat wakes for same-agent direct-message sessions into the agent main session.

## Best Possible Solution

Review, land, or replace #72648 with a narrow core change that normalizes both system-event enqueue keys and forced heartbeat wake targets for same-agent direct-message sessions when `dmScope=main` and the existing single-owner allowlist rule is satisfied, while preserving explicit group, channel, cross-agent, and custom cron routing.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to configure `dmScope=main` with a single-owner Telegram allowlist, enqueue an exec or cron system event under `agent:main:telegram:default:direct:<userId>`, and request an exec-event or cron wake with that forced key; current main resolves and peeks the per-channel queue instead of `agent:main:main`.

## Solution Assessment

Yes. The requested direction is the narrow maintainable fix if it normalizes enqueue and wake routing together; a heartbeat-only rewrite would still miss events left in the original per-channel system-event queue.
