## Summary

Keep open. Current main still preserves same-agent per-channel forced heartbeat keys for the exec-event and cron/system-event paths, and an open unmerged PR explicitly references this issue as its fix path.

## What This Changes

Request dmScope-aware routing so exec-event and cron-event heartbeat wakes for single-owner agents with `session.dmScope: "main"` process in the canonical main session.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/72648 with a narrow dmScope-aware routing change that normalizes system-event enqueue keys and forced heartbeat targets together under the existing single-owner policy.

## Reproduction Assessment

Yes, at source level. Enqueue an exec or cron system event under a same-agent per-channel direct-message key with `session.dmScope: "main"`; current main preserves that forced key through enqueue, wake resolution, and queue lookup rather than resolving it to `agent:<id>:main`.

## Solution Assessment

Yes, directionally. The narrow maintainable fix is to apply the existing single-owner `dmScope=main` policy to both event enqueue and heartbeat wake resolution; changing only one side would strand queued events under the other key.
