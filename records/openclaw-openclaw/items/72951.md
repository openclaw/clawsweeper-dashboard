The PR adds gateway session-history logic and tests to recover `senderLabel` from hidden `openclaw.runtime-context` rows while keeping those rows out of visible history.

## Best Possible Solution

Port the recovery into the current shared transcript projection/index path so sync, async, recent, and active v3 history all hide runtime-context rows while attaching a fresh parsed label to the preceding visible user message, then add focused regression tests and changelog.

Yes. A transcript containing a user message followed by a hidden `openclaw.runtime-context` custom row with `Sender (untrusted metadata)` will currently be projected without `senderLabel` because both sync and async readers only emit message/compaction records.

## Solution Assessment

No. The idea is right, but the current PR hooks an older reader shape; the maintainable fix is to share the runtime-context filtering and label-attachment behavior across the current transcript projection and indexed async paths.
