## What This Changes

The PR adds gateway session-history helpers and regression tests to recover `senderLabel` from hidden `openclaw.runtime-context` rows while keeping those rows out of `readSessionMessages()` output.

## Best Possible Solution

Handle hidden runtime-context `custom_message` entries consistently in both the active-branch and raw fallback readers, keep those rows hidden from visible history, preserve the clean model-facing prompt boundary, add v3 active-branch regression coverage, and add a single-line Unreleased/Fixes changelog entry.

## Solution Assessment

No. Recovering from raw JSONL rows is a useful partial fix, but the narrow maintainable solution is to share the runtime-context detection and sender-label attachment with the active-branch loop that current transcripts use.
