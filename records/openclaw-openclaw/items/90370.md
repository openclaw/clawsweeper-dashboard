## Summary

Keep open as the canonical storage-backend product/design tracker. Current main and v2026.6.9 are still SQLite-only for shared state, per-agent state, plugin state, synchronous transactions, and built-in memory storage; the related dual-backend issue was closed back to this issue, while session-storage and snapshot-durability work only cover narrower adjacent slices.

## What This Changes

Request optional local PostgreSQL support as a configurable backend or integration target so deployments can consolidate OpenClaw and team data instead of running SQLite alongside PostgreSQL.

## Best Possible Solution

Keep this as the canonical storage-backend design item until maintainers choose SQLite-only core storage, snapshot-based durability, or a staged backend-neutral storage architecture with explicit migration, config, credential, and transaction contracts.
