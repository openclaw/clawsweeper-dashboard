## What This Changes

The issue requests a configurable PostgreSQL backend in place of SQLite for OpenClaw internal state such as task registry, TaskFlow, plugin index, and channel monitor state.

## Best Possible Solution

Maintainers should first decide whether internal storage remains SQLite-only or becomes an explicit backend/plugin API; any PostgreSQL path should follow that design with migration and upgrade proof.

## Reproduction Assessment

## Solution Assessment

No; the suggested direct config switch is not the best safe fix today. The maintainable path would be a maintainer-approved storage/backend contract with migrations before implementing PostgreSQL.
