## Summary

Keep open: current main still has a source-provable direct empty-output gap, but this PR is not merge-ready because it can bypass delivery suppression, misses a reasoning-only empty-output case, is currently conflicting, and lacks real behavior proof.

## What This Changes

The PR adds a static empty-output fallback reply in auto-reply dispatch plus regression tests for direct fallback and channel no-op behavior.

## Best Possible Solution

Land a narrow dispatch-level empty-output fallback after final queued counts are known, gated by delivery policy, canonical direct-chat policy, routed-origin state, queued output counts, block/TTS output, and intentional handled/no-text branches.

## Reproduction Assessment

Yes. By source inspection, a direct dispatch with replyResolver returning undefined, no block/tool callbacks, and zero queued counts still completes silently on current main.

## Solution Assessment

No. A dispatch-level fallback is the right shape, but this PR must honor suppressDelivery, count sendable output after filtering reasoning replies, and use normalizeChatType or intentionally extend the chat-type contract with tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Honor delivery suppression before sending fallback:** `src/auto-reply/reply/dispatch-from-config.ts:1048-1063`
  - body: The fallback guard never checks `suppressDelivery`, so a direct turn with `sendPolicy: "deny"` or `sourceReplyDeliveryMode: "message_tool_only"` and an empty resolver would now call `sendFinalReply` even though the current policy requires final delivery to remain suppressed.
  - confidence: 0.92
- **[P2] Count sendable output after filtering replies:** `src/auto-reply/reply/dispatch-from-config.ts:1053`
  - body: The guard checks raw `replies.length === 0`, but dispatch skips `isReasoning` replies before final delivery. If the resolver returns only reasoning payloads, final/tool/block counts remain zero and the new fallback will not send, leaving the same silent completion this PR is meant to cover.
  - confidence: 0.78
