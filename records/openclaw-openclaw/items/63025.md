## Summary

Keep open. The current-main gap is source-visible, but this PR is not merge-ready because the fallback bypasses explicit delivery-suppression policy, misses reasoning-only no-output cases, diverges from the shared chat-type contract, lacks real behavior proof, and is currently dirty against main.

## What This Changes

The branch adds a static direct/private empty-output fallback in `dispatchReplyFromConfig` and regression tests for fallback delivery and channel no-op behavior.

## Best Possible Solution

Land or supersede this through the canonical no-visible-final tracker with a scoped dispatch fallback that respects delivery suppression, canonical chat type normalization, sendable-reply filtering, routed-origin delivery, block/TTS output, and intentional silent branches.

## Reproduction Assessment

Yes, at source level. Current main can receive an undefined resolver result, produce no final/tool/block counts, and return without a fallback; I did not run a live Telegram or Feishu reproduction in this read-only review.

## Solution Assessment

No. A dispatch-level fallback is a plausible fix shape, but this patch must first honor suppression policy, count sendable output after filtering, align with `normalizeChatType`, resolve direct-silence policy, and provide real behavior proof.

  - body: The fallback guard does not check `suppressDelivery`, so a direct turn with `sendPolicy: "deny"` or `sourceReplyDeliveryMode: "message_tool_only"` can now call `sendFinalReply` even though current policy requires user delivery to remain suppressed.
  - confidence: 0.92

  - body: The guard checks raw `replies.length === 0`, but dispatch skips `isReasoning` replies before final delivery. A resolver that returns only reasoning payloads still produces no sendable final/tool/block output, yet this fallback will not run.
  - confidence: 0.78

  - body: The local `direct|dm|private` helper makes the fallback fire for `private`, but `normalizeChatType` does not currently support that value. Use the shared normalizer or deliberately extend the shared contract and tests.
  - confidence: 0.76
