## Summary

Keep open: current main still has a source-proven direct-message empty-output gap, but this PR is not merge-ready because the fallback can bypass delivery suppression, misses reasoning-only empty output, lacks real behavior proof, and is currently merge-conflicting.

## What This Changes

The PR adds a static empty-output fallback reply in auto-reply dispatch for direct/private contexts, plus regression tests for fallback and channel no-op behavior.

## Best Possible Solution

Land a narrow dispatch-level empty-output fallback after final queued counts are known, guarded by delivery policy, canonical direct-chat semantics, routed-origin state, queued output counts, block/TTS output, and intentional handled/no-text branches.

## Reproduction Assessment

Yes. By source inspection on current main, a direct dispatch with replyResolver returning undefined, no block/tool callbacks, and zero queued counts still completes without a visible fallback.

## Solution Assessment

No. A dispatch-level fallback is the right shape, but this PR must honor suppressDelivery, count sendable output after filtering reasoning replies, and align chat-type handling with the canonical contract or add an intentional contract extension.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Honor delivery suppression before sending fallback:** `src/auto-reply/reply/dispatch-from-config.ts:1048-1063`
  - body: The fallback guard does not check `suppressDelivery`, so a direct turn with `sendPolicy: "deny"` or `sourceReplyDeliveryMode: "message_tool_only"` and an empty resolver can now call `sendFinalReply` even though current policy requires user delivery to remain suppressed.
  - confidence: 0.92
- **[P2] Count sendable replies after filtering reasoning:** `src/auto-reply/reply/dispatch-from-config.ts:1053`
  - body: The guard checks raw `replies.length === 0`, but dispatch skips `isReasoning` replies before final delivery. A resolver that returns only reasoning payloads still produces no sendable final/tool/block output, yet this fallback will not run.
  - confidence: 0.78
