## Summary

Keep open: current main still has the silent zero-output dispatch gap, but this PR is not mergeable as-is because the fallback bypasses delivery suppression, misses reasoning-only no-output cases, and lacks real behavior proof.

## What This Changes

The PR adds a direct/private empty-output fallback final reply in `dispatchReplyFromConfig` plus tests for the fallback and channel no-op path.

## Best Possible Solution

Land a guarded dispatch-level empty-output fallback that respects delivery suppression, uses canonical chat-type policy, counts actual sendable output after filtering, and includes focused tests plus real Telegram/direct-chat proof.

## Reproduction Assessment

Yes, by source inspection. Current main turns an undefined resolver result into `replies = []` and returns with zero queued final/tool/block output, so a direct dispatch with no callbacks can complete without a user-visible reply.

## Solution Assessment

No. A dispatch-level fallback is the right shape, but this implementation should honor `suppressDelivery` and decide based on sendable output after filtering rather than raw `replies.length`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Honor delivery suppression before sending fallback:** `src/auto-reply/reply/dispatch-from-config.ts:1050-1063`
  - body: The new fallback guard never checks `suppressDelivery`, so a direct turn with `sendPolicy: "deny"` or explicit `message_tool_only` and an empty resolver would now call `sendFinalReply`, violating the current no-delivery policy and tests.
  - confidence: 0.92
- **[P2] Count sendable output after filtering replies:** `src/auto-reply/reply/dispatch-from-config.ts:1053-1057`
  - body: The guard requires `replies.length === 0`, but dispatch later skips `isReasoning` replies before final delivery. A reasoning-only resolver result still leaves final/tool/block counts at zero, yet the fallback will not fire, preserving the silent completion path this PR is meant to fix.
  - confidence: 0.78
