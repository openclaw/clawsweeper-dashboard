## Summary

Keep open. This PR targets a real current-main silent completion path, but the proposed fallback still bypasses delivery-suppression policy and lacks required after-fix real behavior proof from the external contributor.

## What This Changes

The PR adds a static direct/private chat fallback final reply when `dispatchReplyFromConfig` finishes with no final, tool, or block output, plus two regression tests.

## Best Possible Solution

Land a narrow dispatch-level empty-output fallback only after queued counts are final and the guard respects delivery suppression, routed-origin state, block/TTS output, sendable reply filtering, intentional silent branches, and canonical chat-type semantics.

## Reproduction Assessment

Yes, at source level. Current main can complete a direct dispatch with an undefined resolver result, no block/tool callbacks, and zero queued output without sending a visible fallback.

## Solution Assessment

No. A dispatch-level fallback is the right shape, but this patch must honor `suppressDelivery`, count sendable replies after reasoning filtering, and use the canonical chat-type contract unless maintainers intentionally extend it.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Honor delivery suppression before sending fallback:** `src/auto-reply/reply/dispatch-from-config.ts:1048-1063`
  - body: The fallback guard does not check `suppressDelivery`, so a direct turn with `sendPolicy: "deny"` or `sourceReplyDeliveryMode: "message_tool_only"` can now call `sendFinalReply` even though current policy requires user delivery to remain suppressed.
  - confidence: 0.92
- **[P2] Count sendable replies after filtering reasoning:** `src/auto-reply/reply/dispatch-from-config.ts:1053`
  - body: The guard checks raw `replies.length === 0`, but dispatch skips `isReasoning` replies before final delivery. A resolver that returns only reasoning payloads still produces no sendable final/tool/block output, yet this fallback will not run.
  - confidence: 0.78
- **[P3] Use the canonical chat-type contract:** `src/auto-reply/reply/dispatch-from-config.ts:104-107`
  - body: The local `direct|dm|private` helper makes the fallback fire for `private`, but `normalizeChatType` does not currently support that value. Use the shared normalizer or deliberately extend the shared contract and tests.
  - confidence: 0.76
