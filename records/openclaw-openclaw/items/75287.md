## Summary

Keep open. Current main still lets an unowned CLI root such as `foo` enter lazy plugin CLI registration and the default proxy startup path before Commander can fail it, so the central invalid-command lifecycle bug is not fixed even though related gateway catalog-pressure paths have partial mitigations.

## What This Changes

The issue asks OpenClaw to fail unknown CLI commands before plugin/provider loading, avoid leaving hot child processes, and reduce gateway hot-path plugin catalog reload pressure.

## Best Possible Solution

Keep this open until OpenClaw classifies primary CLI roots from core command metadata plus lightweight plugin manifest/alias ownership before any managed proxy startup or runtime plugin loading. An unowned root such as `foo` should skip broad plugin CLI registration, avoid unnecessary proxy/runtime startup, print a normal unknown or unavailable-command error, exit promptly, and leave no child process behind.

## Reproduction Assessment

Yes. The reporter supplied a high-confidence process-group reproduction for `openclaw foo`, and current-main source inspection still shows deterministic unscoped plugin CLI loading and default proxy routing for arbitrary non-builtin primary roots. I did not execute the live repro because the review must remain read-only.

## Solution Assessment

Not applicable as a patch. The narrow maintainable direction is a pre-runtime CLI ownership gate backed by manifest/descriptor metadata plus proxy/runtime cleanup coverage, rather than another gateway catalog timeout or broad provider-loading mitigation.

The observable bug is that an invalid command such as `openclaw foo` can start managed proxy routing, load provider/runtime plugin CLI registrars, print an unavailable-command error, and leave a hot child process. Implement a narrow pre-runtime ownership gate for primary CLI roots: built-in roots should keep existing behavior, manifest/descriptor-owned plugin roots should still lazily register only the owning plugin(s), and unowned roots should avoid managed proxy startup plus broad runtime/plugin CLI loading and fail promptly with a normal unknown/unavailable-command error.
