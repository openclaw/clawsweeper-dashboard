## Summary

Keep open. Current main still routes unowned CLI roots through managed proxy startup and lazy plugin CLI registration, while the related gateway catalog work only mitigates part of the broader saturation report.

## What This Changes

The issue asks for invalid CLI commands such as `openclaw foo` to fail fast without provider/runtime plugin loading or surviving hot child processes, and for related gateway catalog hot paths to avoid repeated event-loop saturation.

## Best Possible Solution

Add a manifest/descriptor-backed CLI ownership gate before proxy startup and plugin runtime loading so unowned roots fail promptly, while owned plugin CLI roots still activate only their owning plugin.

## Reproduction Assessment

Yes. The reporter supplied direct `openclaw foo` process-group repros, and current-main source/tests still show the same proxy plus plugin-registration route for non-builtin, non-reserved, unowned command roots.

## Solution Assessment

Not applicable as a patch. The narrow maintainable fix is to validate CLI root ownership before managed proxy startup and broad plugin CLI registration, rather than adding another timeout around gateway catalog discovery.

## Repair Work Prompt

Fix the current-main invalid CLI root lifecycle from https://github.com/openclaw/openclaw/issues/75287. Add a pre-runtime CLI ownership gate so unowned command roots like `openclaw foo` fail quickly before managed proxy startup, provider catalog loading, runtime plugin CLI registration, or any surviving child process. Preserve valid builtin roots and manifest/descriptor-owned plugin CLI roots; if a plugin command is owned but disabled or excluded by config, keep the existing helpful unavailable-command messaging without broad-loading all plugins. Keep the fix in CLI/plugin command ownership policy, not in gateway catalog timeout code. Add/adjust focused tests around `shouldStartProxyForCli`, command-path policy, plugin command registration skip/ownership, and `runCli` invalid-root behavior. Add a single-line user-facing changelog fix entry. Do not broaden plugin runtime loading, do not add core knowledge of bundled plugin IDs where manifest ownership can answer it, and do not change unrelated gateway polling behavior.
