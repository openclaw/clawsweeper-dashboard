## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve transcript adjacency when collapsing duplicates:** `ui/src/ui/chat/build-chat-items.ts:363`
  - body: Because this collapse runs after `searchOpen` has filtered out non-matching messages, two identical messages separated by a nonmatching transcript entry become adjacent in the filtered array and render as one `×2` bubble. That hides the intervening context in search results even though the documented behavior is only for consecutive duplicates, so the collapse needs to preserve original transcript adjacency or reset across filtered-out messages.
  - confidence: 0.88
