## Summary

Keep open. Current main still sanitizes several structured tool-call leak formats, but the reported bare `exec{"command": ...}` shape is not covered by the shared visible-text pipeline and can still be treated as ordinary assistant text.

## What This Changes

The issue asks OpenClaw to stop exposing bare pseudo tool calls such as `exec{"command":"openclaw status","description":"Check OpenClaw version and status"}` in shell and Telegram replies.

## Best Possible Solution

Add a conservative shared visible-text sanitizer stage, or a small helper called by that pipeline, for standalone bare tool-name-plus-balanced-JSON pseudo calls while preserving literal code/prose examples and normal JSON text.

## Reproduction Assessment

Yes. The issue has concrete steps, and current source inspection gives a deterministic reproduction path: `exec{"command": ...}` does not match the XML, bracketed block, downgraded marker, or diagnostic sanitizer shapes, so it remains visible text.

## Solution Assessment

Yes for the recommended direction. The narrow maintainable fix belongs in the shared assistant visible-text path because the same extracted text feeds final delivery and history surfaces; Telegram-only filtering would be incomplete.
