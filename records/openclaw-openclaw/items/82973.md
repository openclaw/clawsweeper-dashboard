## Summary

Keep open: the follow-up commit addresses the earlier completions payload gap, but the resolver change is now too broad and would bypass the documented/tested exclusion for non-Anthropic Bedrock models. Real Docker/oMLX behavior proof is also still missing.

## What This Changes

This PR passes explicit cacheRetention through the embedded-runner resolver for openai-completions providers, emits prompt_cache_retention="24h" for long-retention completions payloads, and adds tests plus changelog text.

## Best Possible Solution

Narrow the resolver change so existing provider-family exclusions, especially non-Anthropic Bedrock, stay intact while explicit cacheRetention reaches opted-in OpenAI-compatible completions providers, then add focused coverage and a redacted live oMLX/Docker trace.

## Reproduction Assessment

Source-level yes: current main clearly drops explicit cacheRetention before the openai-completions transport sees it. A high-confidence end-to-end Docker/oMLX reproduction is still missing.

## Solution Assessment

No, not as written. The transport follow-up fixes the earlier payload gap, but the resolver widening must preserve established non-Anthropic Bedrock behavior or add a narrower compatibility signal.
