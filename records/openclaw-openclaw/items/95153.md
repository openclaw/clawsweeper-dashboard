## Summary

Keep open: the latest PR head protects explicit New Chat and initial-task paths, but it still changes the shared no-key `sessions.create` behavior from fresh dashboard creation to implicit active-session adoption and still lacks real Control UI reconnect proof.

## Best Possible Solution

Keep no-key create fresh by default, or add an explicit maintainer-approved adopt/reconnect contract with docs, SDK compatibility coverage, and real Control UI refresh/New Chat proof.

## Reproduction Assessment

## Solution Assessment

No. Gateway-side reconnect reuse may be useful, but implicit adoption inside a public create method is not the best shape without an explicit adopt/reconnect contract or maintainer-approved API direction.
