## Summary

Keep open: the follow-up commit narrows reuse for label/task/model/message requests, but explicit Control UI New Chat still uses parentSessionKey plus emitCommandHooks without those fields, so this PR can still return the old active dashboard session instead of creating the requested fresh session. The PR also still has only targeted test output, not real Control UI/Gateway reconnect proof.

## What This Changes

The PR adds Gateway logic to return the latest active dashboard session for no-key sessions.create calls and adds focused tests for reuse, empty/ended stores, label, and task cases.

## Best Possible Solution

Keep the Gateway-side passive reconnect fix, but gate it to truly passive adoption requests or an explicit reconnect/adopt mode, with regression coverage for parent/hook New Chat and redacted real Control UI proof.

## Solution Assessment

No. Gateway-side passive reconnect reuse is a plausible fix layer, but this implementation is still too broad because it does not preserve parent/hook create intent.
