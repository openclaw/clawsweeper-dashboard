## Summary

Keep open: the PR targets a real session-continuity bug, but the current guard is too broad and can bypass existing `sessions.create` behavior for explicit New Chat, parent linkage, labels/models, and initial task/message sends; it also still lacks real Control UI reconnect proof.

## What This Changes

The branch adds a Gateway helper that reuses the latest active `agent:<id>:dashboard:<uuid>` session for no-key `sessions.create` calls and adds three focused gateway tests.

## Best Possible Solution

Narrow dashboard-session reuse to passive reconnect/adoption requests only, while preserving existing `sessions.create` behavior for parent/hook, label/model, task/message, and explicit New Chat intents.

## Reproduction Assessment

Yes, source-level. Current code and tests show no-key create requests can carry explicit user intent, parent linkage, labels/models, or an initial task, and the PR's early return would bypass those behaviors whenever an active dashboard entry exists.

## Solution Assessment

No. Gateway-side reuse is a plausible layer for passive reconnects, but this implementation is too broad and should preserve create-intent paths before merge.
