## Summary

Keep open: the latest head still changes the shared Gateway/SDK no-key create contract by implicitly adopting an active dashboard session, and the contributor proof remains Gateway test output rather than real Control UI reconnect proof.

## What This Changes

The branch changes Gateway `sessions.create` so empty no-key requests return the latest active dashboard session for the agent instead of always creating a new dashboard key, with regression tests for reuse and create-intent guard cases.

## Best Possible Solution

Preserve empty `sessions.create` as fresh-create by default, or add a maintainer-approved explicit reconnect/adopt signal with docs, SDK compatibility coverage, and live Control UI proof.

## Reproduction Assessment

Yes for the source-level RPC behavior: current main falls back to a random dashboard key when no key is requested, and PR head returns an active dashboard key. No high-confidence live Control UI browser reproduction was supplied.

## Solution Assessment

No. The gateway boundary is plausible, but implicit adoption inside a public create method is not the safest fix without an explicit contract or maintainer approval; the latest guard only protects some create-intent cases.
