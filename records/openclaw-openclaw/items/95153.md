## Label Justifications

- `P1`: The PR is a fix candidate for active session-state regressions and currently introduces a high-impact New Chat/session continuity regression.
- `merge-risk: 🚨 compatibility`: The patch changes existing no-key `sessions.create` API semantics for callers that use label, model, parent, task, or message without an explicit key.

## Summary

Keep open: the PR targets a real session-continuity problem, but the patch reuses an existing dashboard session for every no-key `sessions.create` call, which breaks explicit New Chat and initial-task creation paths. The PR also has only test-harness proof, not real Gateway/browser behavior proof.

## What This Changes

The PR adds gateway logic to reuse the latest active dashboard session for no-key `sessions.create` calls and adds three session-creation regression tests.

## Best Possible Solution

Narrow dashboard-session reuse to a passive reconnect/adoption case and preserve existing create semantics whenever the request carries parent, hook, label, model, task, or message intent.

## Reproduction Assessment

Yes, source-level. Current tests and UI code show no-key create requests with explicit user intent, parent linkage, or an initial task, and the PR's new early return would bypass those behaviors whenever an active dashboard entry exists.

## Solution Assessment

No. Gateway-side reuse is a reasonable layer for passive reconnects, but this implementation is too broad and needs to preserve existing create-intent paths.
