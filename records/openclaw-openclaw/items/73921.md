## Summary

Keep open. Current main still routes local `infer model run --model` through the agent-aware simple-completion selection path, while the docs promise fully qualified local provider/model smokes; the issue is also paired with open PR #74028 through closing syntax, so it should stay open until that PR lands or is superseded.

## What This Changes

`openclaw infer model run --model <provider/model>` should honor fully qualified Copilot model overrides in the local CLI path instead of selecting the agent/default model.

## Best Possible Solution

Review and either land PR #74028 or replace it with the same narrow behavior: fully qualified local model-run overrides should prepare the requested provider/model directly, while unqualified refs keep the existing agent-aware default-selection path and gateway forwarding stays explicit.

## Reproduction Assessment

Yes. The issue includes concrete CLI repro commands and response-metadata observations, and current main still exposes the source seam where local `model.run` passes the override through `prepareSimpleCompletionModelForAgent`.

## Solution Assessment

Yes. Direct preparation for fully qualified local overrides is the narrowest maintainable fix because it preserves existing agent-aware selection for unqualified/default runs and leaves the already-explicit gateway path unchanged.
