## What This Changes

The issue asks for `openclaw infer model run --model <provider/model>` to honor fully qualified Copilot model overrides instead of routing through the configured default model.

## Best Possible Solution

Review and land PR #74028, or replace it with the same narrow behavior: fully qualified local `infer model run` overrides should prepare the requested provider/model directly, while unqualified refs keep the agent-aware default-selection path and gateway forwarding remains explicit.

## Reproduction Assessment

Yes. The issue includes concrete CLI commands and response-metadata observations, and a focused unit reproduction can assert that local `--model github-copilot/<id>` dispatches the requested provider/model instead of the agent default path.

## Solution Assessment

Yes. Direct preparation for fully qualified local overrides is the narrowest maintainable fix because it preserves existing default/alias behavior for unqualified refs and leaves the gateway override path explicit.
