## Summary

Keep open. Current main still routes the local `infer model run` override through the agent-aware simple-completion path, and open PR #74028 explicitly closes this issue as the active focused repair.

## What This Changes

The issue requests that `openclaw infer model run --model <provider/model>` honor fully qualified local model overrides instead of falling back to the configured default model.

## Best Possible Solution

Review and land PR #74028, or replace it with the same narrow behavior: provider-qualified local model-run overrides should prepare the requested provider/model directly, while unqualified refs continue through existing agent-aware default selection and gateway forwarding remains explicit.
