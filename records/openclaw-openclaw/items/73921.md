## Summary

Keep open. Current main still sends local `infer model run --model` through the agent-aware simple-completion preparation path, the docs promise fully qualified local provider/model smokes, and open PR #74028 is already paired with this issue through closing syntax. The issue should stay open until that PR lands or is superseded.

## What This Changes

The issue requests that `openclaw infer model run --model <provider/model>` honor fully qualified Copilot model overrides in the local CLI path instead of routing to the default Anthropic model.

## Best Possible Solution

Review and either land PR #74028 or replace it with the same narrow behavior: fully qualified local model-run overrides should prepare the requested provider/model directly, while unqualified refs continue through the existing agent-aware default-selection path and gateway forwarding remains explicit.

## Reproduction Assessment

Yes. The issue includes concrete CLI commands and response-metadata observations, and current main still exposes the source seam where local `model.run` delegates the override through `prepareSimpleCompletionModelForAgent`.

## Solution Assessment

Yes. Direct preparation for fully qualified local overrides is the narrowest maintainable fix because it preserves existing agent-aware selection for unqualified/default runs and leaves the already-explicit gateway path unchanged.
