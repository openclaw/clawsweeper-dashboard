## Summary

Keep open. Current main still routes local `infer model run --model` through the agent-aware simple-completion preparation path, the docs promise provider-qualified local smoke tests, and open PR #74028 is already the active closing implementation path.

## What This Changes

The issue requests that `openclaw infer model run --model <provider/model>` honor fully qualified local model overrides instead of silently routing to the configured/default model.

## Best Possible Solution

Land or replace PR #74028 with the same narrow behavior: fully qualified local model-run overrides should prepare the requested provider/model directly, unqualified refs should continue through existing agent-aware default selection, and gateway forwarding should remain explicit.

## Reproduction Assessment

Yes. The issue includes concrete CLI commands and metadata observations, and current main still exposes the local CLI seam where `params.model` is delegated through `prepareSimpleCompletionModelForAgent`.

## Solution Assessment

Yes. Direct preparation for fully qualified local overrides is the narrowest maintainable direction because it preserves agent-aware selection for defaults/unqualified refs while making explicit provider probes deterministic.
