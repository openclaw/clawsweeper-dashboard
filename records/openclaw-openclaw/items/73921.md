## Summary

Keep open. Current main still routes local `infer model run --model` through the agent-aware simple-completion selection path, and open PR #74028 is paired to this issue with closing syntax, so the issue should remain open until that PR lands or is superseded.

## What This Changes

The issue reports that `openclaw infer model run --model <provider/model>` ignores fully qualified GitHub Copilot model overrides and routes to the configured/default Anthropic model instead.

## Best Possible Solution

Land or replace the narrow CLI fix so fully qualified local overrides prepare the requested provider/model directly, while unqualified refs keep agent-aware default selection and gateway forwarding stays explicit.

## Reproduction Assessment

Yes. The issue includes concrete CLI commands and response-metadata observations, and current main exposes the matching local dispatch seam in `runModelRun`.

## Solution Assessment

Yes. The #74028 direction is the narrowest maintainable fix because it changes only fully qualified local overrides and preserves the existing unqualified/default and gateway paths.
