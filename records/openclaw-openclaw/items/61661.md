## Summary

Keep open. Current main still restores scoped Control UI session state without filtering historical routed direct/group keys, and the linked fix candidate at https://github.com/openclaw/openclaw/pull/78741 is open rather than merged.

## What This Changes

Control UI should avoid restoring persisted historical routed direct/group session keys on startup and fall back to the main session unless a session is explicitly requested.

## Best Possible Solution

Review and land a storage-layer fallback that resets persisted routed direct/group session keys to the canonical main session while preserving explicit session links; https://github.com/openclaw/openclaw/pull/78741 is the current implementation candidate.

## Reproduction Assessment

Yes. From source inspection, seeding scoped Control UI local settings with a routed `sessionsByGateway` entry such as `agent:main:feishu:direct:ou_xxx` will flow through `loadSettings()` and then be preserved by `applySessionDefaults()`.

## Solution Assessment

Yes, the storage restore layer is the right fix boundary. The current linked PR still needs to cover all routed key shapes and satisfy its review/proof requirements before this issue can close.
