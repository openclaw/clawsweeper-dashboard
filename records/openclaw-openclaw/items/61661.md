## Summary

Keep open. Current main still hydrates the Control UI session from persisted scoped local settings without filtering routed direct/group session keys, and the later main-session normalization preserves those keys as deliberate non-main selections. The backend exposes a real main session separately, so this remains a narrow Control UI restore bug rather than a heartbeat/routing bug.

## What This Changes

The issue asks Control UI startup restore to fall back to the real main session instead of reopening a locally persisted historical routed direct/group session when no session was explicitly requested.

## Best Possible Solution

Add a Control UI startup-restore guard that maps persisted routed direct/group session keys back to the main/default session, while leaving explicit URL sessions, in-app session picker navigation, and gateway routing/delivery semantics unchanged.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to seed `openclaw.control.settings.v1*` or `sessionsByGateway` with `agent:main:feishu:direct:ou_xxx`, load the Control UI without a `session` URL parameter, and observe `loadSettings()` plus app boot state select that routed key before snapshot defaults preserve it.

## Solution Assessment

Yes. The issue's proposed Control UI restore fallback is the narrowest maintainable fix; changing heartbeat, backend route resolution, or channel delivery would address the wrong layer.

## Repair Work Prompt

Fix openclaw/openclaw#61661. In Control UI startup restore, persisted local settings (`openclaw.control.settings.v1*` / `sessionsByGateway`) should not reopen historical routed channel sessions such as `agent:<agentId>:<channel>:direct:<id>`, `agent:<agentId>:<channel>:group:<id>`, or account-scoped variants where `direct`/`group` appears after the agent segment, when no URL `session` parameter was explicitly provided. Fall back to the main/default session and let existing snapshot normalization resolve `main`/`agent:main:main` to the real `sessionDefaults.mainSessionKey`. Preserve explicit `?session=` / hash `session` deep links, manual session picker navigation, and all gateway routing/heartbeat/delivery behavior. Likely implement in `ui/src/ui/storage.ts` or a small helper used before persisted settings hydrate app state, with tests in `ui/src/ui/storage.node.test.ts` and `ui/src/ui/app-settings.test.ts` if URL behavior is touched. Do not change gateway route resolution or channel delivery semantics. Add a `CHANGELOG.md` Fixes entry credited with `Thanks @junoclaws` if the final patch is user-facing.
