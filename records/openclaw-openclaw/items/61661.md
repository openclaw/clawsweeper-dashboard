## Summary

Keep open because current main still restores persisted routed Control UI session keys unchanged, and an open linked PR is the active closing path rather than a merged fix.

## What This Changes

The issue asks Control UI to fall back to the real main session on reload when persisted settings contain historical routed direct/group session keys.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/78741 with a storage-restore sanitizer that resets historical routed direct/group/dm/thread session keys to main while preserving explicit session navigation and non-peer agent sessions.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: seed `openclaw.control.settings.v1` / `sessionsByGateway` with a routed key such as `agent:main:feishu:direct:ou_xxx`, call `loadSettings()`, and the restored key flows into startup settings before gateway defaults preserve it as user-selected.

## Solution Assessment

Yes, with maintainer policy acceptance. The storage restore boundary is the narrowest maintainable place to sanitize historical routed keys, and the linked PR is the current implementation candidate for that approach.
