## Summary

Keep open. Current main still restores the Control UI session from scoped local settings without filtering routed direct/group session keys, and the later main-session defaulting path preserves those keys as deliberate non-main selections.

## What This Changes

The issue requests that Control UI startup restore prefer the real main session instead of persisted historical routed direct/group session keys unless that routed session was explicitly requested.

## Best Possible Solution

Add a Control UI startup-restore guard that maps persisted routed direct/group session keys back to the canonical main/default session while leaving explicit session links and gateway routing semantics unchanged.

## Reproduction Assessment

Yes. Seed scoped Control UI local settings with a `sessionsByGateway` entry such as `agent:main:feishu:direct:ou_xxx`, load the Control UI without an explicit `session` URL parameter, and current source paths select that key before preserving it during snapshot defaulting.

## Solution Assessment

Yes. The narrow UI restore fallback is the best direction; changing heartbeat, backend route resolution, or channel delivery would address the wrong layer.
