## Summary

Keep open. Current main still trusts persisted Control UI session keys during startup restore, and the linked closing PR is still open rather than merged.

## What This Changes

The issue asks Control UI startup to fall back to the main session instead of restoring historical routed direct/group session keys from persisted local settings.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/78741 with a storage-layer sanitizer that resets persisted routed direct/group session keys to main while preserving explicit session selection.

## Reproduction Assessment

Yes, at source level. Seed `openclaw.control.settings.v1` / `sessionsByGateway` with a routed key such as `agent:main:feishu:direct:ou_xxx`, then `loadSettings()` returns it and gateway defaults preserve it as a specific selected session.

## Solution Assessment

Yes. The storage restore boundary is the narrow maintainable place to sanitize historical routed keys; the issue should remain open until the linked PR lands or a replacement covers the routed-key contract.
