## Summary

Keep open. Current main still restores scoped Control UI session state without filtering persisted historical direct/group routed session keys, and the linked fix candidate at https://github.com/openclaw/openclaw/pull/78741 is still open rather than merged.

## What This Changes

Control UI should fall back to the real main session on reload when persisted local settings point at a historical routed direct/group session key.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/78741 with a storage-layer restore sanitizer that resets persisted routed direct/group session keys to main by default and covers the shared routed-session key contract.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: seed `openclaw.control.settings.v1` / `sessionsByGateway` with a routed key such as `agent:main:feishu:direct:ou_xxx`, call `loadSettings()`, and the value flows into startup settings before `applySessionDefaults()` preserves it as user-selected.

## Solution Assessment

Yes, the restore/storage layer is the right boundary for this bug. The issue should stay open until the linked PR lands or a replacement covers all routed direct/group key shapes and retains explicit session-selection behavior.
