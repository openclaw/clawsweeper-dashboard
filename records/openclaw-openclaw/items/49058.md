## Summary

Keep open. Current main already implements the single-session/top-level Mattermost DM reply behavior and separate websocket liveness handling, but it does not yet populate structured `ReplyToBody`/`ReplyToSender` for Mattermost DM replies or add the proposed typing fixes; the PR as-is also conflicts with current main by preserving direct-message `replyToId` roots that current tests intentionally suppress.
