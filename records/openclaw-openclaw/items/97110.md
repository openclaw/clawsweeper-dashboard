## Summary

Keep open: this maintainer draft adds a useful native app i18n inventory, but the new CI check is wired into a job that native-only app changes intentionally skip, so the central drift-detection behavior is not yet reliable.

## What This Changes

Adds a generated native app string inventory, a TypeScript check/sync generator, package scripts, test-project mapping, and a CI step intended to catch Android and Apple inventory drift.

## Best Possible Solution

Keep the inventory generator, but wire `native:i18n:check` to a dedicated or native-triggered CI path and add routing coverage for native-only app changes.

## Reproduction Assessment

## Solution Assessment

No. The generator is a plausible foundation, but the CI integration is not the best fix until the check runs from a native-triggered job or a dedicated inventory job.
