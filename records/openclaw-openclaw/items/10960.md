## Summary

Keep open: current main and the latest release still implement boundary-oriented steering with streaming-only acceptance gates, while the requested pause-or-abort, inject, and resume lifecycle remains unresolved and is already paired with open maintainer-reviewed implementation work.

## What This Changes

The issue requests a soft-steer mode that can pause or abort an active generation or long-running operation, inject a new user message into context, and resume with that input immediately.

## Best Possible Solution

Keep this issue open as the product/runtime tracker and land https://github.com/openclaw/openclaw/pull/73365 or a narrower successor after maintainers decide the lifecycle semantics and update regression coverage/docs.

## Reproduction Assessment

Yes, as a source-level feature gap: configure steer mode, keep a session active during a non-streaming tool phase or long generation, and send another inbound message; current main either skips or rejects same-turn injection instead of pause/abort/resume. I did not run a live channel scenario in this read-only review.

## Solution Assessment

Not applicable as a patch. The requested direction is plausible, but the maintainable solution needs reviewed lifecycle semantics and a maintainer decision about changing steer behavior versus adding an opt-in soft-steer mode.
