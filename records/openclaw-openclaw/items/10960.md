- `P2`: This is a normal-priority core runtime feature/product tracker with real active-run steering impact, but open PRs already cover the urgent implementation slices.

## Summary

Keep open: current main and latest v2026.6.6 still implement boundary/streaming-gated steering, while true pause-or-abort, inject, and resume behavior remains unmerged and is already represented by open implementation PRs needing maintainer review.

## What This Changes

Request mid-stream soft steering so user input can pause or abort an active generation or tool wait, enter context immediately, and resume with the new guidance.

## Best Possible Solution

Keep this as the broad soft-steer tracker, then land or replace the coordinated current-main successors for lifecycle-aware injection and same-session admission while preserving documented fallback semantics.

## Reproduction Assessment

Yes. Source inspection shows current main only attempts steering when `isStreaming` is true and embedded/reply queues reject non-streaming active runs; related live comments report the same delayed-injection behavior across Discord, Telegram, and WebChat.

## Solution Assessment

No complete solution is on main yet. The maintainable path is to coordinate lifecycle-aware injection with same-session admission and backend fallback semantics, rather than starting a separate narrow repair branch.
