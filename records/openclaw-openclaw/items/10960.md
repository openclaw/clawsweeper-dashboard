## Summary

Keep open: current main still implements boundary-oriented steering with streaming-only acceptance gates, while the requested soft-steer pause-or-abort, inject, and resume lifecycle remains unresolved and is already paired with open implementation work that needs maintainer review.

## What This Changes

Request to let users steer an active agent turn immediately by pausing or aborting current generation, injecting the new message into context, and resuming with that input.

## Best Possible Solution

Keep this as the product/runtime tracker and land https://github.com/openclaw/openclaw/pull/73365 or a narrower successor after maintainers decide default versus opt-in semantics and document the lifecycle contract.

## Reproduction Assessment

Yes, as a source-level feature gap: configure steer mode, keep a session active during a non-streaming tool phase or long generation, and send another inbound message; current main either skips or rejects same-turn injection instead of pause/abort/resume.

Not applicable as a patch. The requested direction is plausible, but the maintainable solution needs reviewed lifecycle semantics and a maintainer decision about changing `steer` behavior versus adding an opt-in soft-steer mode.
