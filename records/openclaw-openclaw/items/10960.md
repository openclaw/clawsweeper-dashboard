## Summary

Keep open: current main still implements boundary-oriented, streaming-gated steering rather than the requested pause-or-abort, inject, and resume lifecycle, and the issue remains paired with open implementation work that needs maintainer product/runtime review.

## What This Changes

The issue requests a soft-steer capability that can interrupt an active generation or long-running turn, inject the new user message into context, and resume with that updated context.

## Best Possible Solution

Keep this as the product/runtime tracker and review or land https://github.com/openclaw/openclaw/pull/73365, https://github.com/BingqingLyu/openclaw/pull/2307, or a narrower successor after the default-versus-opt-in soft-steer semantics are decided and documented.

## Reproduction Assessment

Yes, as a source-level feature gap: current main gates same-turn steering on streaming state or documented runtime boundaries, and docs explicitly say steering does not interrupt in-flight tools. I did not run a live channel scenario, so this proves the requested behavior is absent rather than reproducing a broken current contract.

## Solution Assessment

Not applicable as a patch. The requested direction is plausible, but the maintainable solution needs reviewed lifecycle semantics and a maintainer decision about changing `steer` behavior versus adding an opt-in soft-steer mode.
