## What This Changes

The issue requests a soft-steer mode that can interrupt active generation or long-running work, inject a new user message, and resume with that context.

## Best Possible Solution

Keep this as the canonical soft-steer runtime tracker, then land https://github.com/openclaw/openclaw/pull/73365 or a maintainer-approved successor with documented lifecycle semantics and regression coverage, coordinated with https://github.com/openclaw/openclaw/issues/70319.

## Solution Assessment

Unclear until maintainer review completes. The maintainable solution is lifecycle-aware active-run steering and interruption semantics with stopped-state, fallback, backend compatibility, docs, and regression coverage rather than a queue-policy-only tweak.
