## Summary

Keep open: current main still implements and documents `steer` as runtime-boundary steering with followup fallback, not the requested pause-or-abort, inject, and resume lifecycle, and there are still open implementation PRs for this surface.

## What This Changes

The request asks for soft-steer behavior that can pause or abort an active generation, inject a new user message immediately, and resume with that context instead of waiting for a model or tool boundary.

## Best Possible Solution

Keep this as the runtime/product tracker until https://github.com/openclaw/openclaw/pull/73365, https://github.com/BingqingLyu/openclaw/pull/2307, or a narrower successor lands with documented lifecycle semantics and regression coverage in the embedded runner and reply-run registry paths.

## Reproduction Assessment

Yes, from source. On current main, an active `steer` run with `isActive=true` and `isStreaming=false` skips immediate injection, the queue helpers reject non-streaming handles, and the queue policy resolves to followup behavior.

## Solution Assessment

Unclear until maintainer review completes. The maintainable path should be lifecycle-aware injection or abort/resume handling with explicit stopped-state, fallback, and abuse-control semantics, not a queue-policy-only tweak.
