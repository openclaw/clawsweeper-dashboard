## Summary

Keep open: current main still has boundary-oriented steering and followup fallback, while the requested pause-or-abort, inject, and resume lifecycle remains unlanded and is paired with open implementation work that needs maintainer/product review.

## What This Changes

Request mid-stream soft steering so a user message sent during active generation or long tool execution can interrupt, enter context, and resume without killing the whole run.

## Best Possible Solution

Define a lifecycle-aware steering contract and land https://github.com/openclaw/openclaw/pull/73365 or a narrower successor with docs, regression coverage, and real runtime proof.

## Reproduction Assessment

Yes, as a source-level feature gap: a steer-mode active session whose run is active but not streaming skips the current injection gate or returns not_streaming/false. I did not run a live channel scenario, so this proves the requested behavior is absent rather than reproducing a broken documented contract.

## Solution Assessment

Not applicable as a patch. The requested direction is plausible, but a queue-policy-only change would be too narrow; the maintainable solution needs reviewed lifecycle semantics, backend bounds, fallback behavior, docs, and regression coverage.
