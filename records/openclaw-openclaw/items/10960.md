## Summary

Keep open. Current main and the latest release still use streaming/boundary-gated steering and fallback behavior, while the requested pause-or-abort, inject, and resume lifecycle remains unmerged and is already represented by open implementation PRs.

## What This Changes

Request mid-stream soft steering so a user message can interrupt an active generation or long-running turn, enter context immediately, and resume with the new input.

## Best Possible Solution

Keep this as the product/runtime tracker; review and land https://github.com/openclaw/openclaw/pull/73365 or a focused successor, then document and test the final streaming, non-streaming, abort/resume, fallback, and backend-failure semantics.

## Reproduction Assessment

Yes. Source inspection shows active steering is gated on streaming in the runner, embedded queue, and reply registry; a run that is active but not streaming falls back instead of receiving the requested immediate soft-steer lifecycle.

## Solution Assessment

No. Current main is not the requested solution; the narrow active/non-streaming steering PR direction is plausible, but true pause-or-abort, inject, and resume behavior needs maintainer product semantics before it can be called the best fix.
