## Summary

Keep open: current main and the latest release still implement boundary-based steering, not the requested pause-or-abort, inject, and resume soft-steer lifecycle, and the narrower active-run fix remains in open review.

## What This Changes

The issue requests true mid-stream soft steering so new user input can interrupt an active generation, enter context immediately, and let the run resume with that input.

## Best Possible Solution

Keep this as the broad soft-steer tracker, land or replace the focused active non-streaming fix, and make a maintainer product decision on true stream abort/resume semantics with docs and regression coverage.

## Reproduction Assessment

Yes, at source level. With queue mode `steer`, a message sent while the active run is in a non-streaming tool phase cannot follow the requested soft-injection path because current main checks `isStreaming` before queueing and rejects non-streaming active handles.

## Solution Assessment

No, current main is not the requested solution. The maintainable path is lifecycle-aware steering for active non-streaming boundaries plus an explicit maintainer decision on true stream abort/resume behavior.
