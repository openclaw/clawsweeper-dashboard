## Summary

Keep open: current main still implements boundary-oriented steering with followup fallback, and live GitHub context shows this issue is paired with open implementation work that needs maintainer review before the tracker can close.

## What This Changes

The issue requests a soft-steer mode that can interrupt active generation or long tool work, inject a new user message, and resume with that input in context.

## Best Possible Solution

Keep this as the runtime/product tracker until a maintainer lands a lifecycle-aware steering fix with documented semantics and focused regression coverage.

## Reproduction Assessment

Yes, as source-level current behavior rather than a strict bug repro. An active steer run with isActive=true and isStreaming=false skips immediate injection, both queue helpers reject non-streaming handles, and the queue policy falls back to followup.

## Solution Assessment

Unclear until maintainer review completes. The maintainable path is a lifecycle-aware injection or abort/resume design with explicit stopped-state, fallback, backend failure, and abuse-control semantics, not a queue-policy-only tweak.
