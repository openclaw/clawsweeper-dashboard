## Summary

Keep open: current main still uses streaming/runtime-boundary steering with followup fallback, and active implementation PRs are still open rather than merged.

## What This Changes

Request mid-stream steering so a user message can pause or abort active generation, enter context immediately, and resume with that new input instead of waiting for a tool/model boundary.

## Best Possible Solution

Keep this open as the runtime/product tracker until #73365 or a narrow successor lands with docs and regression coverage for embedded runner, reply-run registry, fallback, backend failure, stopped-state, and abort/resume behavior.

## Reproduction Assessment

Yes, from source. On current main, an active session using `messages.queue.mode: "steer"` with `isActive=true` and `isStreaming=false` skips immediate injection and falls back to `enqueue-followup`.

## Solution Assessment

Unclear until maintainer review completes. Current main's boundary/followup behavior is not the requested soft-steer lifecycle; a lifecycle-aware injection change with stopped-state and abuse guards appears to be the narrow maintainable direction.
