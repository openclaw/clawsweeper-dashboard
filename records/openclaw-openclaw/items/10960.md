- `impact:message-loss`: User guidance can be delayed into a later turn instead of being delivered to the intended active run.

## Summary

Keep open: current main and the latest release still implement boundary-based steering with streaming-only admission gates, while the requested pause-or-abort, inject, and resume soft-steer lifecycle remains unresolved and paired with open, unmerged implementation work.

## What This Changes

Request true mid-stream soft steering so a user message can interrupt active generation or long tool work, enter context immediately, and let the agent resume with that input.

## Best Possible Solution

Keep this as the broad soft-steer tracker, land or replace the focused active non-streaming/admission fixes, and make a maintainer product decision on true pause-or-abort/resume semantics with docs and regression coverage.

## Reproduction Assessment

Yes, at source level. With queue mode `steer`, a message sent while the active run is in a non-streaming tool phase cannot follow the requested soft-injection path because current main checks `isStreaming` before queueing and rejects non-streaming active handles.

## Solution Assessment

No, current main is not the requested solution. The maintainable path is lifecycle-aware steering for active non-streaming boundaries plus an explicit maintainer decision on true stream abort/resume behavior.
