## Summary

Keep open: current main and the latest release still define boundary-oriented steering with followup fallback, while the requested pause-or-abort, inject, and resume lifecycle remains an active runtime/product decision with open implementation PRs.

## What This Changes

Request true mid-stream steering so an active generation or long tool phase can pause or abort, inject the user's message into context, and resume with that input.

## Best Possible Solution

Define a lifecycle-aware active-run steering contract for streaming, non-streaming tool execution, stopped-state fallback, abort/resume, backend failure, limits, docs, and regression coverage, then land the approved implementation.

## Reproduction Assessment

Yes, as source-level current behavior: steer injection is attempted only while streaming, active non-streaming handles return not_streaming, and docs state Pi steering is delivered after tool calls before the next LLM call. This is a feature-gap reproduction, not a broken existing contract.

## Solution Assessment

Unclear until maintainer review finishes. A queue-policy-only change would be too narrow; the implementation should integrate run lifecycle, abort/resume, backend compatibility, partial output, and tests.
