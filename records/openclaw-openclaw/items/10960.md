## Summary

Keep open: current main still has boundary-oriented steering and streaming-gated injection, while the requested pause-or-abort, inject, and resume lifecycle remains unlanded and is already paired with open implementation work that needs maintainer/product review.

## What This Changes

The issue requests a soft-steer mode that can interrupt an in-flight agent generation or long-running turn, inject a user message into context, and resume with that input instead of waiting for the next model or tool boundary.

## Best Possible Solution

Define the lifecycle-aware steering contract and land https://github.com/openclaw/openclaw/pull/73365 or a narrower successor with docs, regression coverage, and real runtime proof for streaming, active non-streaming, abort/resume, fallback, backend failure, and reply-run registry behavior.

Yes, as a source-level feature gap: an active steer-mode run that is not streaming either skips the current injection gate or returns `not_streaming`. I did not run a live channel scenario, so this proves the requested behavior is absent rather than reproducing a broken documented contract.

Not applicable as a patch. The requested direction is plausible, but the maintainable solution needs reviewed lifecycle semantics and backend bounds rather than a queue-policy-only tweak.
