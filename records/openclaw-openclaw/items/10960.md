## Summary

Keep open. Current main and the latest release still use stream-only steering gates and boundary-style steering, while the requested pause-or-abort, inject, and resume lifecycle remains a product/runtime decision; the linked implementation paths are open or conflicting, not merged.

## What This Changes

Request true soft-steer message injection that can pause or abort active generation, add the new user input to context, and resume with that input instead of waiting for the current boundary.

## Best Possible Solution

Keep this as the product/runtime tracker; review and land https://github.com/openclaw/openclaw/pull/73365 or a focused successor, then decide separately whether full pause/abort/capture/resume semantics should extend beyond boundary steering.

## Reproduction Assessment

Yes. Source inspection shows active steering is gated on streaming in the runner, embedded queue, and reply registry; a run that is active but not streaming falls back instead of receiving the requested immediate soft-steer lifecycle.

## Solution Assessment

Unclear. The narrow active/non-streaming steering PR direction is plausible for the boundary-steer gap, but the requested true pause-or-abort, inject, and resume behavior needs maintainer product semantics before it can be called the best solution.
