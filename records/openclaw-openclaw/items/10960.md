## Summary

Current main still provides model-boundary steering and a separate abort-and-replace interrupt mode, not the requested context-aware pause/abort, inject, and resume lifecycle. The issue should remain the product/runtime tracker because two linked implementation PRs are still open, including protected maintainer PR https://github.com/openclaw/openclaw/pull/73365, and both branches currently conflict with the evolved main runtime.

## What This Changes

Requests interruption of an active agent generation so a new user message can be injected into context and the run can resume with awareness of that input.

## Best Possible Solution

Extend the canonical reply-run lifecycle with one soft-steer operation that either delivers at the next safe model boundary or aborts and restarts from committed transcript state, with explicit partial-output, tool, failure, and fallback semantics and no parallel queue owner.

## Reproduction Assessment

Yes at source level: current main explicitly rejects embedded steering when the handle is not streaming, documents steering only at model boundaries, and defines interrupt as abort-and-replace rather than context-aware resume. No live run was needed to establish that product gap.

## Solution Assessment

No, neither current boundary steering nor the current interrupt mode fully solves the request. The narrowest maintainable direction is to extend the existing reply-run lifecycle rather than add a second queue, polling loop, or independent paused-state machine.
