## Summary

Current main still implements boundary-based steering rather than the requested pause-or-abort, inject, and resume lifecycle, and this issue remains paired with open implementation PRs; it should stay open for maintainer product and lifecycle review.

## What This Changes

Add a soft-steer lifecycle that interrupts active generation, injects new user input, and resumes with preserved context.

## Best Possible Solution

Keep this issue as the product/runtime tracker and land a lifecycle-safe implementation that either preserves existing `steer` semantics or introduces a distinct soft-steer contract, with explicit transcript, tool-execution, stopped-state, backend-failure, and fallback behavior.

## Reproduction Assessment

Yes, from source: current main only accepts OpenClaw steering on streaming handles and otherwise waits for a later turn, while `interrupt` aborts and starts a replacement turn rather than resuming the same run.

## Solution Assessment

No, not as an unconditional replacement for current `steer`; maintainers should first define whether abort-and-resume is a distinct mode or a compatible lifecycle extension, then require state-safe implementation and real-run proof.
