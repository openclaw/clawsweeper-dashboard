## Summary

Current main still provides model-boundary steering rather than the requested interrupt, inject, and context-preserving resume lifecycle. Two implementation candidates remain open and conflicting, including the protected maintainer-labeled replacement, so this issue is still necessary as the canonical product/runtime tracker.

## What This Changes

Add an active-run lifecycle that can interrupt generation, inject newly arrived user input, and resume with that input in context.

## Best Possible Solution

Refresh or replace the canonical maintainer PR with one lifecycle-aware design that explicitly covers native steering, active tool execution, stopped state, partial-output persistence, abort/resume fallback, backend rejection, and follow-up ordering across embedded and Codex runtimes.

## Reproduction Assessment

Yes, at source-contract level. Current main gates ordinary steering on streaming state and documents model-boundary delivery rather than a pause-or-abort, inject, and resume lifecycle.

## Solution Assessment

No current proposal is yet proven to be the best solution. Polling would duplicate runtime ownership, while the direct-injection and abort-based branches require a maintainer decision about partial output, tool cancellation, and permanent cross-runtime semantics.
