## Summary

Keep open: current main and the latest release still gate active-run steering on streaming/admission behavior, and the remaining work is already represented by open implementation PRs that need maintainer review before this tracker can close.

## What This Changes

Request true mid-stream soft-steer behavior so a user message can enter an active generation or long tool phase and the run can continue with that new context.

## Best Possible Solution

Keep this as the broad soft-steer tracker, then land reviewed current-main successors that coordinate https://github.com/openclaw/openclaw/pull/73365 with the admission-path work in https://github.com/openclaw/openclaw/pull/89238 and preserve documented fallback semantics.

## Solution Assessment

No complete solution is on main yet. The maintainable path is to coordinate lifecycle-aware injection with same-session admission and backend fallback semantics rather than starting a separate narrow repair branch.
