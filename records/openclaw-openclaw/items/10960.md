## Summary

Keep open: current main still implements boundary-oriented, streaming-gated steering rather than the requested pause-or-abort, inject, and resume lifecycle, and there is active implementation work at https://github.com/openclaw/openclaw/pull/73365 plus an open closing fork PR. ## What This Changes

Add true mid-stream soft steering so new user input can pause or abort an active generation, enter context, and resume with that input instead of waiting for a tool or model boundary.

## Best Possible Solution

Keep this issue as the product/runtime tracker and review or land https://github.com/openclaw/openclaw/pull/73365 or a narrower successor with explicit lifecycle semantics, docs, regression coverage, and real runtime proof.

## Reproduction Assessment

Yes, as a source-level feature gap: current main still gates same-turn steering on streaming state or runtime boundaries, and docs explicitly say steering does not interrupt in-flight tools. I did not run a live channel scenario, so this proves the requested behavior is absent rather than reproducing a broken documented contract.

## Solution Assessment

Not applicable as a patch. The requested direction is plausible, but the maintainable solution needs reviewed lifecycle semantics and an explicit product choice about changing `steer` behavior versus adding an opt-in soft-steer mode.
