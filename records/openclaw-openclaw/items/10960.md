## Summary

Keep this issue open. Current main and the latest release still implement boundary/fallback steering rather than the requested pause-or-abort, inject, and resume behavior, and this issue is paired with open implementation PRs including https://github.com/BingqingLyu/openclaw/pull/2307 and https://github.com/openclaw/openclaw/pull/73365.

## What This Changes

Request true soft steer so user messages sent during an active generation or tool phase can pause or abort the run, enter context immediately, and resume instead of waiting for the next boundary.

## Best Possible Solution

Keep this issue as the product/runtime tracker until an implementation such as https://github.com/openclaw/openclaw/pull/73365 or a narrower successor lands with documented lifecycle semantics and regression coverage.
