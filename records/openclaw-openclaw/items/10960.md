## What This Changes

The issue requests true mid-stream steer behavior that can pause or abort active generation, inject a new user message into context, and resume with that input.

## Best Possible Solution

Keep this as the broad runtime/product tracker, land or replace https://github.com/openclaw/openclaw/pull/73365 for the active non-streaming boundary fix, and decide separately whether true stream abort/resume belongs in the same lifecycle contract or a narrower follow-up.

## Solution Assessment

No, current main is not the requested solution. The best path is a lifecycle-aware steering fix for active non-streaming boundaries plus a maintainer product decision on true stream abort/resume semantics.
