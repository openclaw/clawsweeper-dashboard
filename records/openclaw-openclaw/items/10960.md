## Summary

Keep open: current main still implements boundary-oriented steering with followup fallback, while the requested soft-steer lifecycle remains active product/runtime work tied to open implementation PR https://github.com/openclaw/openclaw/pull/73365.

## What This Changes

The issue requests a soft-steer mode that can pause or abort active generation, inject a new user message into context, and resume with that input instead of waiting for the next tool/model boundary.

## Best Possible Solution

Keep this as the runtime/product tracker until a maintainer lands or replaces the lifecycle-aware steering fix, with documented semantics and focused regression coverage for embedded runs, reply-run registry behavior, channel queue entry points, fallback, and diagnostics.

## Reproduction Assessment

Yes, as source-level current behavior rather than a strict bug repro: active non-streaming steer attempts are skipped or rejected, then active steer falls back to followup. A live channel reproduction was not run in this read-only review.

## Solution Assessment

Unclear until maintainer review completes. The maintainable path is a lifecycle-aware injection or abort/resume design with explicit stopped-state, fallback, backend failure, diagnostics, and abuse-control semantics, not a queue-policy-only tweak.
