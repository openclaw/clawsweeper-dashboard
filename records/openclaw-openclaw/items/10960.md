## Summary

Keep open: current main and the latest release still have boundary/streaming-gated steering behavior, the requested pause-or-abort/inject/resume lifecycle remains missing, and open implementation PRs already own pieces of the work.

## What This Changes

The issue requests a soft-steer mode that can interrupt an active generation or long tool phase, inject the user's new message into context, and resume with awareness of that input.

## Best Possible Solution

Keep this as the broad soft-steer tracker, then land a reviewed successor that coordinates lifecycle-aware active-run injection with active-turn admission, fallback semantics, docs, and regression coverage.

## Reproduction Assessment

Yes for the current limitation: current main has a clear source path where active non-streaming runs skip immediate steering and fall back to later delivery. Full true pause/abort/resume behavior remains a product/runtime design request rather than a fully reproduced existing contract.

## Solution Assessment

No standalone autonomous fix is best here. The maintainable solution is coordinated maintainer review of the lifecycle-aware injection and active-turn admission work, with docs and tests for the final steering contract.
