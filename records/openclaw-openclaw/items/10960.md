## Summary

Keep open: current main still implements steer as streaming/runtime-boundary queueing with followup fallback, and the remaining lifecycle-aware soft-steer work is already paired with open maintainer PR #73365.

## What This Changes

Request a soft-steer lifecycle that can pause or abort active generation, inject new user input immediately, and resume with that context instead of waiting for a tool or model boundary.

## Best Possible Solution

Keep this open as the runtime/product tracker until #73365 or a narrow successor lands with docs and regression coverage for embedded runner, reply-run registry, fallback, backend failure, and abort/resume behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path is an active session using `messages.queue.mode: "steer"` where `isActive` is true but `isStreaming` is false; current main skips immediate injection and enqueues a followup.

## Solution Assessment

Unclear until maintainer review completes. The requested direction is valid, but the maintainable path should be the focused lifecycle-aware implementation in #73365 or a narrow successor rather than another autonomous repair branch.
