## Summary

Keep open: current main still implements boundary-oriented, streaming-gated steering, not the requested pause-or-abort, inject, and resume lifecycle, and the remaining work is already tied to open implementation review.

## What This Changes

The issue requests soft steering that interrupts an in-flight agent turn, injects a new user message into context, and resumes generation without waiting for the current model/tool boundary.

## Best Possible Solution

Define the lifecycle-aware steering contract and land https://github.com/openclaw/openclaw/pull/73365 or a narrower successor with docs, regression coverage, and real runtime proof for streaming, active non-streaming, abort/resume, fallback, backend failure, and reply-run registry behavior.

Not applicable as a patch. The requested direction is plausible, but the maintainable solution needs reviewed lifecycle semantics and an explicit product choice about whether this changes `steer` or becomes an opt-in `soft-steer` mode.
