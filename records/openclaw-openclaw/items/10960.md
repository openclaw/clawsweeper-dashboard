## Summary

Keep open: current main still implements boundary-oriented steering with followup fallback rather than the requested pause-or-abort, inject, and resume lifecycle, and the issue is already paired with open implementation work that needs maintainer review.

## What This Changes

Request mid-stream soft-steer behavior that can pause or abort generation, inject new user input, and resume with the new context.

## Best Possible Solution

Keep this as the runtime/product tracker until https://github.com/openclaw/openclaw/pull/73365, https://github.com/openclaw/openclaw/pull/75226, or a narrower successor lands with documented lifecycle semantics and focused regression coverage.

## Reproduction Assessment

Yes, as source-level current behavior rather than a strict bug repro. An active steer run with isActive=true and isStreaming=false skips immediate injection, both queue helpers reject non-streaming handles, and the queue policy falls back to followup.

## Solution Assessment

Unclear until maintainer review completes. The maintainable path is lifecycle-aware injection or abort/resume handling with explicit stopped-state, fallback, backend failure, and abuse-control semantics, not a queue-policy-only tweak.
