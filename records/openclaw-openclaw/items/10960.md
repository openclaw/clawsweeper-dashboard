## Summary

Keep open: current main still implements boundary-oriented steering with followup fallback, and the requested soft-steer lifecycle remains active runtime/product work tied to open implementation PR https://github.com/openclaw/openclaw/pull/73365.

## What This Changes

Request immediate mid-stream soft steering that pauses or aborts an active generation, injects the new user message into context, and resumes with awareness of that input.

## Best Possible Solution

Keep this as the runtime/product tracker until a maintainer lands https://github.com/openclaw/openclaw/pull/73365 or a narrower successor with documented lifecycle semantics and focused regression coverage for embedded runs, reply-run registry behavior, channel queue entry points, fallback, and diagnostics.

## Reproduction Assessment

Yes, as source-level current behavior rather than a strict bug repro: active non-streaming steer attempts are skipped or rejected, then active steer falls back to followup. A live channel reproduction was not run in this read-only review.

## Solution Assessment

Unclear until maintainer review completes. The maintainable path is a lifecycle-aware injection or abort/resume design with explicit stopped-state, fallback, backend failure, diagnostics, and abuse-control semantics, not a queue-policy-only tweak.
