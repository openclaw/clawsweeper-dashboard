## Summary

Keep open. Current main and the latest release still use streaming/model-boundary steering with followup fallback, while the requested pause-or-abort, inject, and resume behavior is still unmerged and paired with open implementation PRs.

## What This Changes

The issue requests true mid-stream soft steering so a user message can interrupt an active generation or long tool phase, enter the active context immediately, and let the agent resume with that input.

## Best Possible Solution

Keep this as the runtime/product tracker until lifecycle-aware steering is merged with docs and regression coverage for streaming, non-streaming/tool phases, abort/resume, fallback, and backend failure semantics.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with `messages.queue.mode` set to `steer`, an active run that is not currently streaming skips or rejects same-turn steering and falls back to later delivery.

## Solution Assessment

No. Current main is not the requested solution; the active/non-streaming PR direction is plausible, but true pause-or-abort, inject, and resume semantics still need maintainer product review before they can be called the best fix.
