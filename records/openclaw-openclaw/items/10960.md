## Summary

Keep open. Current main and the latest release still do not implement true soft-steer pause/abort/resume behavior, and the remaining work is already split across open implementation PRs that need maintainer review rather than cleanup closure.

## What This Changes

The issue requests mid-stream soft steering so a user message can pause or abort active generation or a long tool phase, enter the current context immediately, and let the agent resume with the new input.

## Best Possible Solution

Keep this as the broad soft-steer runtime tracker, then land or replace the reviewed current-main PR path for same-session admission plus lifecycle-aware injection with docs and tests covering streaming, active tool execution, fallback, and abort/resume semantics.

## Reproduction Assessment

Yes. A high-confidence source path exists: with queue mode steer, current main can wait at dispatch admission, and the active-run steering gates still require isStreaming before user text reaches the active context.

## Solution Assessment

Unclear as stated. The direction is plausible, but the best implementation needs maintainer-approved lifecycle semantics and coordination between the admission fix, lifecycle-aware injection, fallback behavior, and public docs.
