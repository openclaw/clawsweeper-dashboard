## Summary

Keep open: current main still implements steer as streaming/runtime-boundary queueing with followup fallback, and this issue is paired with open implementation PRs, including a maintainer-labeled canonical path, so cleanup closure would be premature.

## What This Changes

Request a soft-steer lifecycle that can pause or abort an active generation, inject new user input immediately, and resume with that context instead of waiting for the next tool or model boundary.

## Best Possible Solution

Keep this open as the runtime/product tracker until a maintainer-approved lifecycle-aware steer path lands, likely #73365 or a narrow successor, with docs and regression coverage for embedded runner, reply-run registry, backend failure, fallback behavior, and abort/resume handling.

Yes. A high-confidence source-level reproduction path is an active session using `messages.queue.mode: "steer"` where `isActive` is true but `isStreaming` is false; current main skips immediate injection and enqueues a followup.

Unclear until maintainer review completes. The requested direction is valid, but the maintainable path should be the focused lifecycle-aware implementation in #73365 or a narrow successor rather than another autonomous repair branch.
