## Summary

Keep this issue open: current main still documents and implements boundary-oriented steering rather than the requested pause-or-abort, inject, and resume lifecycle, and there are still open implementation candidates that should be reviewed instead of closing the tracker.

## What This Changes

The issue requests a soft-steer mode that can interrupt or pause an active generation, inject the new user message into context, and resume with that input immediately.

## Best Possible Solution

Keep this issue open as the product/runtime tracker until maintainers land https://github.com/openclaw/openclaw/pull/73365 or a focused successor, or explicitly choose a narrower soft-steer lifecycle design.

## Reproduction Assessment

Yes, as a source-level feature gap: configure steer mode, keep a session active during a non-streaming/tool phase or long generation, and send another inbound message; current main either skips or rejects immediate injection instead of pause/abort/resume. ## Solution Assessment

Unclear as a final design: the requested direction is plausible, but the best implementation needs maintainer agreement on default behavior versus an opt-in soft-steer queue mode and on abort/resume semantics across runtimes.
