## Summary

Keep open: current main and the latest release still implement boundary-oriented steering with streaming-only acceptance gates, while the requested pause-or-abort, inject, and resume lifecycle remains unresolved and paired with open implementation work.

## What This Changes

The issue requests mid-stream soft steering so a user message can interrupt or pause an active agent generation, enter context immediately, and resume with the new input.

## Best Possible Solution

Keep this issue open as the core runtime tracker until the maintainer-reviewed soft-steer implementation lands or maintainers choose a narrower lifecycle design.

## Reproduction Assessment

Yes, as a source-level feature gap: configure steer mode, keep a session active during a non-streaming/tool phase or long generation, and send another inbound message; current main either skips or rejects same-turn injection instead of pause/abort/resume.

## Solution Assessment

Not applicable as a patch. The requested direction is plausible, but the maintainable solution needs reviewed lifecycle semantics and a maintainer decision about changing default steer behavior versus adding an opt-in soft-steer mode.
