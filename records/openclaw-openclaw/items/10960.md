## Summary

Keep this issue open: current main still rejects steering for active embedded runs that are not streaming and documents embedded steering at model/tool boundaries rather than the requested pause-or-abort, inject, and context-aware resume lifecycle. The issue is also paired with open implementation PRs, including protected maintainer PR https://github.com/openclaw/openclaw/pull/73365, so it should remain the product/runtime tracker until maintainers choose and land a lifecycle-safe implementation.

## What This Changes

Add true mid-generation user-message injection that interrupts an active generation, incorporates the new input, and resumes with preserved context instead of waiting for a runtime boundary.

## Best Possible Solution

Rebase a single canonical implementation onto the current embedded-runner architecture and define one shared lifecycle contract covering active acceptance, stopped and mismatched runs, model streaming, tool execution, partial-output persistence, backend rejection, and followup fallback.

## Reproduction Assessment

Yes at source level. Current main explicitly rejects an active embedded handle when it is not streaming, and current docs describe model-boundary delivery and post-run fallback; no live runtime execution was needed to establish that the requested lifecycle remains absent.

## Solution Assessment

Unclear. The desired user outcome is valid, but the two open PRs choose different lifecycle semantics, and maintainers must decide whether embedded runtimes should accept safe same-run injection or use an abort-and-new-turn path before implementation can be considered the best fix.
