## Summary

Keep open: current main still implements `steer` as runtime-boundary steering with followup fallback, not the requested pause-or-abort, inject, and resume lifecycle, and related implementation PRs are still unresolved.

## What This Changes

Request true mid-stream soft steering that pauses or aborts an active generation, injects the user's follow-up, and resumes with the new context.

## Best Possible Solution

Keep this as the runtime/product tracker until https://github.com/openclaw/openclaw/pull/73365, https://github.com/BingqingLyu/openclaw/pull/2307, or a narrower successor lands with documented lifecycle semantics and regression coverage.

## Reproduction Assessment

Yes, from source. On current main, an active `steer` run with `isActive=true` and `isStreaming=false` skips immediate injection and resolves to followup behavior, which demonstrates the missing soft-steer lifecycle without a live channel run.

## Solution Assessment

Unclear until maintainer review completes. The narrow maintainable solution should be lifecycle-aware injection or abort/resume handling with explicit stopped-state and fallback semantics, not a queue-policy-only tweak.
