## Summary

Keep open: current main still implements boundary-oriented steering and followup fallback, while the requested pause-or-abort, inject, and resume lifecycle remains active runtime work tied to open implementation and interruptibility threads.

## What This Changes

Request support for soft steering that can interrupt active generation or long tool phases, inject a new user message, and resume with that context.

## Best Possible Solution

Keep this issue as the canonical runtime tracker, then land https://github.com/openclaw/openclaw/pull/73365 or a maintainer-approved successor with documented lifecycle semantics and regression coverage, while coordinating broader interruption fencing through https://github.com/openclaw/openclaw/issues/70319.

## Reproduction Assessment

Yes, as a source-level current-behavior path: current main attempts automatic steering only while streaming, rejects non-streaming active handles, and documents runtime-boundary delivery. This is not a strict bug reproduction because the current docs still define boundary-oriented steering while the issue requests a broader soft-steer lifecycle.

## Solution Assessment

Unclear until maintainer review completes. The maintainable solution is a lifecycle-aware active-run steering and interruption contract with explicit stopped-state, fallback, backend compatibility, rate or size bounds, docs, and regression coverage rather than a queue-policy-only tweak.
