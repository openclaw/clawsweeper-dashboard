## Summary

Keep open: current main still implements and documents boundary-oriented steering rather than the requested pause-or-abort, inject, and resume lifecycle, and this issue is paired with open implementation work including https://github.com/BingqingLyu/openclaw/pull/2307 and https://github.com/openclaw/openclaw/pull/73365.

## What This Changes

The issue requests a soft-steer mode that can interrupt active generation or long-running work, inject a new user message into context, and resume with that input.

## Best Possible Solution

Keep this as the product/runtime tracker and review or land https://github.com/openclaw/openclaw/pull/73365, https://github.com/BingqingLyu/openclaw/pull/2307, or a narrower successor after lifecycle semantics and default-versus-opt-in behavior are decided and documented.

Yes, as a source-level feature gap: current main gates same-turn steering on streaming/runtime boundaries and the docs explicitly say steering does not interrupt in-flight tools. Not applicable as a patch. The requested direction is plausible, but the maintainable solution needs reviewed lifecycle semantics and a maintainer decision about changing `steer` behavior versus adding an opt-in soft-steer mode.
