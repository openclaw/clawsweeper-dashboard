## Summary

Keep open: current `main` still implements `steer` as a streaming/runtime-boundary path with followup fallback, not the requested pause-or-abort, inject, and resume lifecycle, and the issue is still paired with open implementation PRs that should be reviewed or superseded before closing.

## What This Changes

Request mid-stream soft steering so a user message can pause or abort an active generation, enter context immediately, and resume with the new input.

## Best Possible Solution

Keep this as the runtime/product tracker until #73365 or a narrower successor lands with documented lifecycle semantics and regression coverage for embedded runner, reply-run registry, fallback, backend failure, stopped-state, and abort/resume behavior.

## Reproduction Assessment

Yes, from source. On current `main`, an active session using `messages.queue.mode: "steer"` with `isActive=true` and `isStreaming=false` skips immediate injection and falls back to `enqueue-followup`; that reproduces the missing soft-steer lifecycle without running a live channel.

## Solution Assessment

Unclear until maintainer review completes. Current `main` is not the requested solution, and the narrow maintainable direction is a lifecycle-aware injection path with stopped-state, fallback, and abuse guards rather than a queue-policy-only tweak.
