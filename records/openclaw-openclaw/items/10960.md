## Summary

Current main still implements boundary-oriented steering: active steer messages are accepted only through streaming-gated paths, with documented fallback to a later prompt rather than the requested pause or abort, inject, and resume lifecycle. The issue also has open implementation candidates, so cleanup should keep it open for maintainer review.

## What This Changes

The request is to add mid-stream soft steering so a user message can interrupt an active generation or long tool phase, enter context immediately, and let the run resume with the new input.

## Best Possible Solution

Keep this issue open as the product/runtime tracker until the maintainer-reviewed soft-steer implementation lands or maintainers choose a narrower lifecycle design.

## Reproduction Assessment

Yes, as a source-level feature gap: configure steer mode, keep a session active during a non-streaming/tool phase or long generation, and send another inbound message; current main either skips or rejects same-turn injection instead of pause/abort/resume. I did not run a live channel reproduction in this read-only cleanup pass.

## Solution Assessment

Not applicable as a patch. The requested direction is plausible, but the maintainable solution needs reviewed lifecycle semantics and a maintainer decision about changing default steer behavior versus adding an opt-in soft-steer mode.
