## Summary

Keep open: current main still has streaming-only steer injection gates and documented runtime-boundary/fallback steering, while #73365 remains the active maintainer implementation path.

## What This Changes

Request true mid-stream soft steering so a new user message can interrupt or pause an active generation/tool phase, enter context, and resume without waiting for the next boundary.

## Best Possible Solution

Keep this open as the runtime/product tracker until #73365 or a narrow successor lands with docs and regression coverage for embedded runner, reply-run registry, fallback, backend failure, and abort/resume behavior.

## Reproduction Assessment

Yes, from source. On current main, an active session using `messages.queue.mode: "steer"` with `isActive=true` and `isStreaming=false` skips immediate injection and falls back to `enqueue-followup`.

## Solution Assessment

Unclear until maintainer review completes. The narrow maintainable path appears to be #73365 or a successor lifecycle-aware injection change, not a separate repair branch while open PRs already cover the area.
