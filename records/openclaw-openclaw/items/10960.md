## Summary

Keep open: current main still implements `steer` as runtime-boundary steering with followup fallback, not the requested pause-or-abort, inject, and resume lifecycle, and an open PR still references this issue with closing syntax.

## What This Changes

The issue requests mid-stream soft steering so a new user message can interrupt active generation, enter context immediately, and let the agent resume with that input.

## Best Possible Solution

Keep this as the runtime/product tracker until #73365, #2307, or a narrower successor lands with documented lifecycle semantics and regression coverage for streaming, tool-phase, fallback, stopped-state, backend failure, and abort/resume behavior.

## Reproduction Assessment

Yes, from source. On current main, an active `steer` run with `isActive=true` and `isStreaming=false` skips immediate injection and resolves to followup behavior, which reproduces the missing soft-steer lifecycle without a live channel run.

## Solution Assessment

Unclear until maintainer review completes. The narrow maintainable solution should be lifecycle-aware injection or abort/resume handling with explicit stopped-state and fallback semantics, not a queue-policy-only tweak.
