## Summary

Keep open: current main and the latest release still implement boundary-based, streaming-gated steering rather than the requested pause-or-abort, inject, and resume soft-steer lifecycle, and related implementation PRs remain open and unmerged.

## What This Changes

Request true mid-stream soft steering so user messages can interrupt or pause an active generation, enter context immediately, and resume with the new input.

## Best Possible Solution

Keep this as the broad soft-steer tracker and resolve it through a maintainer-approved lifecycle design that lands the active non-streaming and admission fixes, documents fallback and abort/resume semantics, and covers embedded and reply-run paths.

## Reproduction Assessment

Yes, at source level for the current limitation: with queue mode steer, active non-streaming phases are rejected or deferred by streaming gates. The broader pause-or-abort/resume lifecycle is a requested feature, not an already-defined behavior contract.

## Solution Assessment

No, current main is not the requested solution. The maintainable path is lifecycle-aware steering for active non-streaming boundaries plus an explicit maintainer decision on true stream abort/resume behavior.
