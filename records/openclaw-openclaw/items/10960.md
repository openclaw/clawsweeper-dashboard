## Summary

Keep open: current main and the latest release still implement boundary-oriented steering, while the requested pause-or-abort, inject, and resume lifecycle is covered by open implementation PRs that still need maintainer review.

## What This Changes

The issue requests mid-stream soft steering so a user message can interrupt active generation, enter context immediately, and let the agent resume with the new input.

## Best Possible Solution

Define a lifecycle-aware steering contract and land https://github.com/openclaw/openclaw/pull/73365 or a narrower successor with docs, regression coverage, and real runtime proof.

## Reproduction Assessment

Yes, as a source-level feature gap: current main can be checked with a steer-mode active session whose run is active but not streaming, where the queue path skips injection or returns `not_streaming`. This proves the requested behavior is not present, not that current documented behavior is broken.

## Solution Assessment

Not applicable as a patch. The requested direction is plausible, but a queue-policy-only change would be too narrow; the maintainable solution needs a reviewed lifecycle contract with backend bounds, fallback behavior, docs, and regression coverage.
