## Summary

Keep open: current main still documents and implements boundary/streaming-gated steering, while the requested pause-or-abort, inject, and resume lifecycle remains unmerged and product-sensitive.

## What This Changes

Requested support for mid-stream soft steering so a user message can interrupt or pause an active generation, enter context immediately, and let the agent resume with the new input.

## Best Possible Solution

Keep this issue open as the broad product/runtime tracker, then land a maintainer-reviewed lifecycle-aware steering PR or narrower successor that defines streaming, tool execution, abort/resume, fallback, backend rejection, and regression coverage.

## Reproduction Assessment

Yes, at source level. Current main still documents boundary-based steering and the active-run queue helpers reject non-streaming active runs, so the requested pause/abort-inject-resume lifecycle is absent.

## Solution Assessment

Unclear as a final implementation. A core/runtime lifecycle-aware steering path is the right layer, but maintainers still need to decide fallback, abort, stopped-state, partial-output, and backend-rejection semantics.
