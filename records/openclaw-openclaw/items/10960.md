## Summary

Keep open: current main and latest release still implement boundary-oriented steering with followup fallback, while true pause-or-abort, inject, and resume behavior remains an unmerged runtime/product change with open implementation paths.

## What This Changes

Request true mid-stream soft steering so a new user message can interrupt an active generation or long tool phase, enter context, and let the agent continue with the new input.

## Best Possible Solution

Define and land a lifecycle-aware steering contract covering streaming, active non-streaming/tool execution, stopped-state fallback, abort/resume, backend failure, limits, docs, and regression coverage.

## Reproduction Assessment

Yes, as source-level behavior: use steer mode with an active session where isActive is true but isStreaming is false, and current main skips immediate injection or returns not_streaming. This proves a feature gap against the requested behavior, not a broken current contract.

## Solution Assessment

Unclear until maintainer review finishes. A queue-policy-only change would be too narrow; the maintainable solution needs coordinated run lifecycle, backend compatibility, direct-injection bounds, and tests.
