## Summary

Keep this issue open because current main still limits normal steer injection to streaming runs, the requested abort-inject-resume lifecycle remains a product decision, and open implementation PRs already own the work.

## What This Changes

Requests mid-generation steering that interrupts an active run, injects new user input into its context, and resumes with awareness of that input.

## Best Possible Solution

Land or replace the lifecycle-aware implementation after defining one tested contract for streaming generation, active tool execution, stopped state, interruption, partial transcript handling, backend rejection, and follow-up fallback.

## Reproduction Assessment

Yes, at source level: current main explicitly rejects steer delivery for non-streaming embedded and reply runs, and the documentation defines separate steer and interrupt semantics. No live runtime reproduction was executed in this read-only review.

## Solution Assessment

Unclear. The open lifecycle-aware PR is the best current implementation candidate, but maintainers must decide whether the desired contract is broader non-streaming steering, explicit interruption, or a new combined abort-and-resume behavior consistent with Pi and Codex runtimes.
