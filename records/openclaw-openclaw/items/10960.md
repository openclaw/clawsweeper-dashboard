## Summary

Keep open: current main still implements boundary-oriented, streaming-gated steering rather than the requested pause-or-abort, inject, and resume lifecycle, and an open PR still references this issue with closing syntax. ## What This Changes

The issue requests lifecycle-aware mid-stream steering so a user message can interrupt active generation, enter context immediately, and resume without waiting for a tool or model boundary.

## Best Possible Solution

Define the lifecycle-aware steering contract and land https://github.com/openclaw/openclaw/pull/73365 or a narrower successor with docs, regression coverage, and real runtime proof for streaming, active non-streaming, abort/resume, fallback, backend failure, and reply-run registry behavior.

Not applicable as a patch. The requested direction is plausible, but the maintainable solution needs reviewed lifecycle semantics and an explicit product choice about changing `steer` behavior versus adding an opt-in soft-steer mode.
