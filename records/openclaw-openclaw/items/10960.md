## Summary

Current main still does not provide the requested generic pause-or-abort, inject, and resume lifecycle: normal reply steering is attempted only for runs reported as streaming, then falls back to later queue handling. Keep the tracker open because the central work is still represented by the open maintainer-labeled implementation at https://github.com/openclaw/openclaw/pull/73365, while the alternate closing PR is also open but conflicting.

## What This Changes

Requests true mid-generation user-message interruption that aborts or pauses the active generation, injects the new input, and resumes with preserved context.

## Best Possible Solution

Keep one canonical lifecycle-aware implementation that defines backend capability handling and safe semantics for streaming output, tools in flight, stopped runs, abort/resume, partial transcript persistence, fallback, and regression coverage across embedded and reply-run paths.

## Reproduction Assessment

Yes, at source level: current main only attempts normal same-turn steering while the run reports streaming, and the documented non-streaming behavior is a later followup rather than pause/inject/resume. No live runtime execution was needed to establish this feature gap.

## Solution Assessment

Unclear: lifecycle-aware active/stopped injection is the narrowest current implementation direction, but maintainers must decide whether aborting a non-streaming run and scheduling a new turn fulfills the requested context-preserving soft-resume semantics.
