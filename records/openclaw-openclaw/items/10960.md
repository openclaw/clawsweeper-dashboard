## Summary

Keep open: current main still implements boundary-oriented steer handling with followup fallback for non-streaming active runs, while the requested pause-or-abort, inject, and resume lifecycle remains unlanded and paired with open implementation work.

## What This Changes

Request mid-stream soft steering so a user message can interrupt active generation, enter the run context immediately, and resume the agent with the new input.

## Best Possible Solution

Keep this as the runtime/product tracker until https://github.com/openclaw/openclaw/pull/73365, https://github.com/BingqingLyu/openclaw/pull/2307, or a narrower successor lands with documented lifecycle semantics and regression coverage in the embedded runner and reply-run registry paths.

## Reproduction Assessment

Yes, from source on current main. An active steer run with isActive=true and isStreaming=false skips immediate injection, both queue helpers reject non-streaming handles, and the queue policy falls back to followup behavior.

## Solution Assessment

Unclear until maintainer review completes. The maintainable path is lifecycle-aware injection or abort/resume handling with explicit stopped-state, fallback, backend failure, and abuse-control semantics, not a queue-policy-only tweak.
