## Summary

Keep open: current main still implements `steer` as boundary-based steering with followup fallback, not the requested pause-or-abort, inject, and resume lifecycle, and open implementation PRs are still active for this surface.

## What This Changes

Request true mid-stream soft steering so an active generation or long tool phase can be paused or aborted, receive new user input, and resume with that context.

## Best Possible Solution

Keep this as the runtime/product tracker until https://github.com/openclaw/openclaw/pull/73365, https://github.com/BingqingLyu/openclaw/pull/2307, or a narrower successor lands with documented lifecycle semantics and regression coverage for embedded and reply-run steering.

## Reproduction Assessment

Yes, from source inspection. With `messages.queue.mode: "steer"`, `isActive=true`, and `isStreaming=false`, current main skips immediate injection, rejects non-streaming queue helpers, and falls back to followup behavior.

## Solution Assessment

Unclear until maintainer review completes. The maintainable path should be lifecycle-aware injection or abort/resume handling with explicit stopped-state, fallback, backend-failure, and abuse-control semantics, not a queue-policy-only tweak.
