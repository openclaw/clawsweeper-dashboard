## Label Justifications

- `P2`: This is a normal-priority WhatsApp auth-storage improvement with durability and security impact, but it is not a confirmed urgent regression blocking all users today.
- `impact:data-loss`: The request targets corrupted or lost WhatsApp credential and Signal-key state currently persisted across Baileys JSON files.
- `impact:security`: The affected store contains WhatsApp credentials and Signal keys, and changing it alters sensitive auth persistence.
- `impact:session-state`: The issue concerns linked WhatsApp session state, key material, migration, and reconnect stability.

## Summary

Keep open: current main and v2026.5.19 still build the WhatsApp socket on Baileys `useMultiFileAuthState`, while the shipped hardening only wraps `creds.json` writes and leaves multi-file Signal-key persistence in place. The prior consolidated auth-state PR was closed unmerged after review concerns, and the latest discussion adds concrete durability and security failure modes rather than making the request obsolete.

## What This Changes

Replace WhatsApp's Baileys multi-file JSON credential and Signal-key store with a production database-backed auth store, including caching and migration for existing sessions.

## Best Possible Solution

Build a migration-aware WhatsApp plugin auth store, likely with a local transactional SQLite default, that replaces Baileys multi-file credential and Signal-key persistence while preserving existing account directories, login/status/logout behavior, and documented setup paths.

## Reproduction Assessment

No, not as a strict runtime bug reproduction; this is a feature/design request. Source inspection is high-confidence that current main and v2026.5.19 still use the Baileys multi-file auth path the issue asks to replace.

## Solution Assessment

Yes on direction, but unclear on exact design. The maintainable path is a WhatsApp plugin-owned transactional auth store with safe migration and live durability proof, not treating the current `creds.json` hardening or the closed prior PR as complete.
