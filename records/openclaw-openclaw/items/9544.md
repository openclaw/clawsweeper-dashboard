## Summary

Keep open. Current main and v2026.5.2 still use Baileys useMultiFileAuthState for the WhatsApp plugin, and the related implementation PR was closed unmerged.

## What This Changes

Replace the bundled WhatsApp plugin's Baileys multi-file JSON auth state with a production-grade SQL or consolidated auth store for credentials and signal keys.

## Best Possible Solution

Implement a WhatsApp-plugin-owned production auth store with safe migration from existing Baileys files, recovery behavior, logout cleanup, docs/config alignment, and regression coverage for fresh and migrated sessions.

## Reproduction Assessment

Not applicable as a feature and security-hardening request. Current-state verification is high confidence because source, docs, tests, upstream guidance, and v2026.5.2 all show Baileys multi-file auth remains in use.

## Solution Assessment

Unclear for the exact SQL/Postgres shape. The WhatsApp plugin is the right boundary, but maintainers should choose the storage backend, migration policy, caching semantics, and recovery model before implementation.
