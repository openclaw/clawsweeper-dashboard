## Summary

Keep open. Current main and the latest release still use Baileys multi-file auth for the bundled WhatsApp plugin, and the related implementation PR was closed unmerged.

## What This Changes

Replace the bundled WhatsApp plugin's Baileys multi-file auth state with a production auth store for credential and signal-key persistence.

## Best Possible Solution

Implement a WhatsApp-plugin-owned production auth store with safe migration, recovery, docs/config updates, and focused regression coverage for fresh install and existing multi-file credentials.

## Reproduction Assessment

Not applicable as a feature/security-hardening request. Current-state verification is high confidence because source, docs, config comments, tests, upstream dependency source, and v2026.5.2 all show Baileys multi-file auth remains in use.

## Solution Assessment

Unclear for the proposed SQL shape. The WhatsApp plugin is the right boundary, but storage backend, migration semantics, caching, permissions, and recovery behavior need maintainer design first.
