## Summary

Keep open. Current main and v2026.5.2 still rely on Baileys `useMultiFileAuthState` for the bundled WhatsApp plugin, and the related implementation PR #66863 was closed unmerged.

## What This Changes

Request replacing the WhatsApp plugin's Baileys multi-file auth persistence with a production SQL or consolidated auth-state store for credentials and signal keys.

## Best Possible Solution

Implement a WhatsApp-plugin-owned production auth store with safe migration from Baileys files, preserved status/logout behavior, aligned docs/config, regression coverage, and security review.

## Reproduction Assessment

Not applicable as a feature and security-hardening request. Current-state verification is high confidence because source, docs, tests, v2026.5.2, and upstream guidance all show Baileys multi-file auth remains in use.

## Solution Assessment

Unclear for the exact SQL/Postgres shape. The WhatsApp plugin is the right boundary, but maintainers should choose the backend, migration policy, caching semantics, and recovery model before implementation.
