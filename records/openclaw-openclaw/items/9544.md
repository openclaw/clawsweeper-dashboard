## Summary

Keep open. Current main and the latest release still use Baileys multi-file auth for the bundled WhatsApp plugin, and the related implementation PR was closed unmerged, so the requested production auth-store replacement has not landed.

## What This Changes

Replace the bundled WhatsApp plugin's Baileys `useMultiFileAuthState` JSON credential and signal-key storage with a production SQL/database-backed auth store.

## Best Possible Solution

A WhatsApp-plugin-owned auth store should replace Baileys multi-file key persistence, migrate existing credentials safely, preserve login/logout/status behavior, update docs/config help, and add regression coverage.

## Reproduction Assessment

Not applicable as a feature/security-hardening request. Current-state verification is high confidence: source, docs, config comments, tests, upstream dependency source, and the latest release all show Baileys multi-file auth is still in use.

## Solution Assessment

Unclear for implementation details. Moving this into the bundled WhatsApp plugin is the right boundary, but the storage format, migration semantics, caching, permissions, and recovery behavior need maintainer design first.
