## Summary

Keep open. Current main and v2026.6.6 still use Baileys multi-file auth for WhatsApp, the prior replacement attempt closed unmerged, and this issue is maintainer-authored so conservative cleanup should not close it.

## What This Changes

The issue requests replacing Baileys `useMultiFileAuthState` in the bundled WhatsApp plugin with a production database-backed auth store for credentials and signal keys.

## Best Possible Solution

Keep this issue open until maintainers land a WhatsApp plugin-owned production auth store with safe migration from Baileys files, security review, docs updates, and fresh-install plus upgrade proof.

## Solution Assessment

Unclear without maintainer design review. Replacing the auth store is the right direction, but the durable fix should follow OpenClaw's SQLite/state-migration rules and preserve status, logout, backup, `authDir`, and existing session behavior.
