## Summary

Keep open. ## What This Changes

The issue requests replacing the bundled WhatsApp plugin's Baileys multi-file auth-state storage with a database-backed production auth store.

## Best Possible Solution

Keep this issue open until the WhatsApp plugin has a maintainer-approved database-backed auth store, safe migration/doctor coverage, public docs, and fresh-install plus upgrade/corruption tests.

## Solution Assessment

Unclear as an implementation plan. A database-backed auth store aligns with OpenClaw's SQLite/storage direction and upstream Baileys guidance, but maintainers still need to choose the schema, migration, backup, and failure semantics.
