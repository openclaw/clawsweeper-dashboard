## Summary

Keep open. Current main still uses Baileys `useMultiFileAuthState` for the WhatsApp plugin, and the related implementation PR #66863 was closed unmerged, so the requested production auth-store replacement has not landed.

## What This Changes

Replace the bundled WhatsApp plugin's Baileys multi-file auth state with a production database-backed credential and signal-key store with caching.

## Best Possible Solution

Implement a WhatsApp-plugin-owned production auth store with safe migration from existing Baileys files, preserved login/logout/status behavior, aligned docs/config, regression coverage, and security review.

## Reproduction Assessment

Not applicable as a feature and security-hardening request. Current-state verification is high confidence because source, docs, tests, and upstream guidance all show Baileys multi-file auth remains in use.

## Solution Assessment

Unclear for the exact SQL/Postgres shape. The WhatsApp plugin is the right boundary, but maintainers should choose the backend, migration policy, caching semantics, and recovery model before implementation.
