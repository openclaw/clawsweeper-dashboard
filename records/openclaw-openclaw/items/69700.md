## Summary

Keep open: current main still has a Memory Wiki docs/contract mismatch. The code treats direct `sources/*.md` files as structured source pages that need frontmatter, while the plugin docs still describe `sources/` as imported raw material without explaining that raw content is preserved inside wrapper pages.

## What This Changes

The issue reports that `openclaw wiki lint` requires frontmatter on raw `sources/` imported Markdown files despite Memory Wiki docs describing `sources/` as raw material.

## Best Possible Solution

Clarify the Memory Wiki docs that `sources/` contains structured source-page wrappers: imported raw content is preserved inside generated pages with required frontmatter, while arbitrary bare Markdown dropped directly into `sources/` is not lint-clean today.

## Reproduction Assessment

Yes at source level. A frontmatter-free direct `sources/paths.md` is collected as a source page, parsed with empty frontmatter, and then lint emits missing `id`, missing `pageType`, and unknown-freshness diagnostics.

## Solution Assessment

Yes for the focused repair. Clarifying the docs is narrower and safer than excluding `sources/` from lint because ingest, bridge import, provenance, compile, and search all rely on structured source-page wrappers.
