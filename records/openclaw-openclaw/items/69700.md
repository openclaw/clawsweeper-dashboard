## Summary

Keep open: current main still has a Memory Wiki docs/contract mismatch. The implementation treats direct Markdown under `sources/` as structured wiki source pages that need metadata, while the plugin docs still describe the directory as imported raw material without explaining the wrapper contract.

## What This Changes

The issue asks OpenClaw to resolve the conflict between Memory Wiki lint requiring frontmatter in `sources/` files and docs that describe `sources/` as raw imported material.

## Best Possible Solution

Clarify the Memory Wiki docs that `sources/` stores structured source-page wrappers: imported raw content is preserved inside generated pages with required frontmatter and provenance, while bare frontmatter-free Markdown under `sources/` is not lint-clean today.

## Reproduction Assessment

Yes at source level. A frontmatter-free direct `sources/paths.md` is collected as a source page, parsed with empty frontmatter, and then lint emits missing `id`, missing `pageType`, and unknown freshness diagnostics.

## Solution Assessment

Yes for the focused repair. Clarifying the docs is narrower and safer than exempting `sources/` from lint because ingest, bridge import, provenance, compile, and search rely on structured source-page wrappers.
