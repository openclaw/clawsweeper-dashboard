## Summary

Keep open: current main and v2026.5.12 still have the Memory Wiki docs/contract mismatch. The implementation treats `sources/*.md` as structured source pages with required frontmatter, while the plugin docs still describe `sources/` as raw material without clarifying that raw content is preserved inside structured wrapper pages.

## What This Changes

The issue asks Memory Wiki to resolve the contradiction between lint requiring source-page frontmatter and docs describing `sources/` as raw imported material.

## Best Possible Solution

Clarify the Memory Wiki docs that `sources/` contains structured source-page wrappers with required frontmatter; raw imported content is preserved inside those pages, while arbitrary bare Markdown dropped directly into `sources/` is not lint-clean today.

## Reproduction Assessment

Yes, at source level. A frontmatter-free direct `sources/paths.md` is collected as a source page, parsed with empty frontmatter, and then lint emits missing `id`, missing `pageType`, and unknown-freshness diagnostics.

## Solution Assessment

Yes for the focused repair. Clarifying the docs is narrower and safer than excluding `sources/` from lint because ingest, bridge import, provenance, compile, and search all rely on structured source-page wrappers.
