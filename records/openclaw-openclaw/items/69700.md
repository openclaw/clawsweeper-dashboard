## Summary

Current main now implements the central lint behavior: intentionally marked unmanaged raw Markdown under `sources/` no longer needs OpenClaw page frontmatter or freshness metadata, while generated and sync-tracked source pages still surface malformed metadata.

## What This Changes

The issue asks for memory-wiki lint to stop requiring `id`, `pageType`, and `updatedAt` frontmatter on raw imported Markdown under `sources/`, or to clarify the source-page contract.

## Best Possible Solution

Keep the current main behavior: marked unmanaged raw source pages bypass wiki metadata and freshness lint, while generated or sync-tracked source pages remain validated.

## Reproduction Assessment

No current-main failing reproduction remains for the fixed path: marked unmanaged raw source pages no longer emit the reported frontmatter or freshness lint issues. Unmarked source pages still intentionally lint as structured pages, which is a separate product contract from the reported frontmatter requirement.

## Solution Assessment

Yes. The landed fix is the best bounded solution because it allows intentional raw source pages without OpenClaw frontmatter while preserving malformed generated-import detection from the related `https://github.com/openclaw/openclaw/issues/64696` guardrail.
