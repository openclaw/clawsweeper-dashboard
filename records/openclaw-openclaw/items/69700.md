## Summary

Keep open: current main still has an actionable Memory Wiki docs/contract mismatch. Source pages are compiled and linted as structured wiki pages, and import paths wrap raw content with source-page frontmatter, while the plugin docs still describe `sources/` as raw material without explaining that raw imports live inside structured wrappers.

## What This Changes

The issue requests resolving a Memory Wiki mismatch where frontmatter-free files in `sources/` are linted as structured pages despite docs describing that directory as raw imported material.

## Best Possible Solution

Clarify the Memory Wiki docs that `sources/` stores structured source pages: raw imported content is preserved inside generated wrappers with required frontmatter, while arbitrary bare Markdown drops are not lint-clean source pages.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a frontmatter-free top-level `sources/paths.md` is collected as a source page, parsed with empty frontmatter, and lint emits the reported missing `id`, missing `pageType`, and missing freshness diagnostics; I did not run the CLI because `wiki lint` writes a report.

## Solution Assessment

Yes for the focused repair. Clarifying the docs is narrower and safer than excluding `sources/` from lint, because compile, search, import, provenance checks, and tests all depend on structured source pages.
