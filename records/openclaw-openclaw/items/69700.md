## Summary

Keep open: current main and the latest release still have a Memory Wiki docs/contract mismatch. The implementation treats `sources/*.md` as structured source pages that need `id`, `pageType`, and freshness metadata, while the plugin docs still describe `sources/` as raw imported material without explaining the wrapper contract.

## What This Changes

Clarify or change Memory Wiki lint behavior for frontmatter-free raw Markdown files placed under `sources/`.

## Best Possible Solution

Clarify the Memory Wiki docs that `sources/` stores structured source-page wrappers: raw imported content is preserved inside generated pages with required frontmatter, while arbitrary bare Markdown in `sources/` is not lint-clean.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a frontmatter-free top-level `sources/paths.md` is collected as a source page, parsed with empty frontmatter, and lint emits missing `id`, missing `pageType`, and missing-freshness diagnostics.

## Solution Assessment

Yes for the focused repair. Clarifying the docs is narrower and safer than excluding `sources/` from lint because compile, search, import provenance, and tests all depend on structured source pages.
