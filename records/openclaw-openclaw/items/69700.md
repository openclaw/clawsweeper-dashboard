## Summary

Keep open: current main and the latest release still have a Memory Wiki docs/contract mismatch. The implementation treats top-level `sources/*.md` as structured source pages that need `id`, `pageType`, and freshness metadata, while the plugin docs still describe `sources/` as raw imported material without explaining the structured wrapper contract.

## What This Changes

The issue asks Memory Wiki to resolve whether raw files in `sources/` should be exempt from frontmatter lint or documented as structured source pages.

## Best Possible Solution

Clarify the Memory Wiki docs that `sources/` contains structured source pages: raw imported content is preserved inside generated wrappers with required frontmatter, while arbitrary bare Markdown drops are not lint-clean source pages.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a frontmatter-free top-level `sources/paths.md` is collected as a source page, parsed with empty frontmatter, and lint emits missing `id`, missing `pageType`, and missing freshness diagnostics.

## Solution Assessment

Yes for the focused repair. Clarifying the docs is narrower and safer than excluding `sources/` from lint because compile, search, import provenance, and tests all depend on structured source pages.
