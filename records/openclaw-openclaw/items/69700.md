## Summary

Current main and the v2026.6.8 release now support marked unmanaged raw Memory Wiki source pages without `id`, `pageType`, or freshness lint, while still linting malformed managed imports, so the reported runtime problem is fixed.

## What This Changes

The issue asks Memory Wiki lint to stop requiring structured frontmatter on raw Markdown kept under `sources/`, or otherwise clarify the source-page contract.

## Best Possible Solution

Keep the shipped raw-source marker behavior and handle any remaining public docs cleanup by revising or closing the outdated wrapper-only docs PR separately.

## Reproduction Assessment

No failing current-main repro remains. Source and regression tests show marked raw source pages no longer produce the reported `missing-id`, `missing-page-type`, or `stale-page` lint, while unmarked or malformed managed source pages remain intentionally lint-visible.

## Solution Assessment

Yes. The merged marker-based fix is the maintainable path because it allows raw source evidence without frontmatter while preserving lint coverage for malformed generated or source-sync tracked imports.
