## Summary

Keep open: current main still has a Memory Wiki docs/contract mismatch. The implementation treats top-level `sources/*.md` files as structured source pages that require `id`, `pageType`, and freshness metadata, while the plugin docs still describe `sources/` as imported raw material without explaining the wrapper contract.

## What This Changes

The issue asks for the Memory Wiki `sources/` contract to stop conflicting between raw imported-material wording and lint-required frontmatter.

## Best Possible Solution

Clarify the Memory Wiki docs that `sources/` contains structured source-page wrappers: raw imported content is preserved inside generated pages with required frontmatter, while arbitrary bare Markdown in `sources/` is not lint-clean.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: place a frontmatter-free top-level `sources/paths.md`, and compile/lint will collect it as a source page with empty metadata and emit the reported missing-frontmatter and freshness diagnostics.

## Solution Assessment

Yes for the focused repair. Clarifying the docs is narrower and safer than excluding `sources/` from lint because ingest, bridge import, compile, search, and provenance checks all treat source entries as structured wrappers.
