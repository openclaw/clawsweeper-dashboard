## Summary

Keep open: current main still has the Memory Wiki docs/contract mismatch. The implementation treats `sources/*.md` as structured source pages that lint for `id`, `pageType`, and freshness metadata, while the plugin docs still describe `sources/` as imported raw material without explaining the wrapper contract.

## What This Changes

The issue asks OpenClaw to resolve a Memory Wiki mismatch where frontmatter-free files in `sources/` are described as raw imported material but lint treats them as structured pages requiring metadata.

## Best Possible Solution

Update the Memory Wiki docs to say `sources/` contains structured source-page wrappers with required frontmatter, while imported raw content is preserved inside those wrappers.

## Reproduction Assessment

Yes. Source inspection shows a top-level `sources/*.md` file without frontmatter is collected as a source page, parsed with empty metadata, and lint emits the reported missing `id`, missing `pageType`, and missing freshness diagnostics.

## Solution Assessment

Yes for the focused repair. Clarifying the docs is narrower and safer than excluding `sources/` from lint because ingest, compile, provenance, and current tests all treat source files as structured page wrappers.
