## Summary

Keep open: current main still has a Memory Wiki docs/contract mismatch. The implementation treats `sources/*.md` as structured source-page wrappers that require frontmatter, while the plugin docs still describe `sources/` as imported raw material without explaining that raw imported content is preserved inside those wrappers.

## What This Changes

The issue asks Memory Wiki to stop requiring frontmatter for raw files under `sources/`, or to clarify the documented `sources/` contract.

## Best Possible Solution

Clarify the Memory Wiki docs that `sources/` contains structured source-page wrappers with required frontmatter, while imported raw content is preserved inside generated source pages with provenance.

## Reproduction Assessment

Yes, at source level. A frontmatter-free direct file such as `sources/paths.md` is collected as a source page, parsed with empty frontmatter, and then lint emits the missing `id`, missing `pageType`, and missing-freshness diagnostics.

## Solution Assessment

Yes for the focused repair. Clarifying the docs is narrower and safer than excluding `sources/` from lint because ingest, bridge import, provenance, compile, search, and get all rely on structured source-page wrappers.
