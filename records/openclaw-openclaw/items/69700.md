## Summary

Keep open: current main and v2026.5.18 still have the Memory Wiki docs/contract mismatch. The implementation treats `sources/*.md` as structured source pages with required frontmatter, while the plugin docs still describe `sources/` as raw material without clarifying that raw content is preserved inside structured wrapper pages.

## What This Changes

The issue reports that `openclaw wiki lint` requires `id`, `pageType`, and freshness frontmatter for bare files under `sources/`, conflicting with docs that describe `sources/` as imported raw material.

## Best Possible Solution

Clarify the Memory Wiki docs that `sources/` contains structured source-page wrappers with required frontmatter, while raw imported content is preserved inside those pages rather than stored as arbitrary bare Markdown.

## Reproduction Assessment

Yes, at source level. A frontmatter-free `sources/*.md` file is collected as a source page, parsed with empty frontmatter, then lint emits missing `id`, missing `pageType`, and missing-freshness diagnostics.

## Solution Assessment

Yes for the narrow repair. Clarifying the docs is safer than excluding `sources/` from lint because current ingest, bridge import, compile, provenance, and search behavior all rely on structured source-page wrappers.
