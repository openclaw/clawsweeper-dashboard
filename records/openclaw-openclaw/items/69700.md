## Summary

Keep open: current main still has an actionable Memory Wiki docs/contract mismatch. Source pages are compiled and linted as structured wiki pages, while the public plugin docs still describe `sources/` as raw material without explaining that raw imports are wrapped with required source-page frontmatter.

## What This Changes

The issue asks Memory Wiki either to stop requiring frontmatter for raw imported files under `sources/` or to clarify the documented `sources/` contract.

## Best Possible Solution

Document `sources/` as structured source-page storage, with raw imported content preserved inside generated source-page wrappers; leave lint behavior unchanged unless maintainers explicitly approve a separate raw-drop feature.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a frontmatter-free top-level `sources/paths.md` is collected as a source page, parsed with empty frontmatter, and lint emits missing `id`, missing `pageType`, and missing-freshness diagnostics; I did not run the CLI because this review is read-only and `wiki lint` writes a report.

## Solution Assessment

Yes for a focused repair. Clarifying the docs is narrower and safer than excluding all `sources/` pages from lint, because compile, search, import, provenance checks, and tests all depend on structured source pages.
