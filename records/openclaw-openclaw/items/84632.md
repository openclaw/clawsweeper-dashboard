## Label Justifications

- `P1`: The issue describes a shipped regression that can persist real provider credentials in generated agent state during a normal operator command.
- `impact:security`: The affected behavior can write resolved secrets to disk as plaintext.
- `impact:auth-provider`: The bug is in model provider API-key SecretRef resolution and marker persistence.

## Summary

Keep open: current main and the latest v2026.5.19 release still have the source-snapshot drop in the unpinned models command path, and an open closing PR already targets the fix.

## What This Changes

Prevent `models status --probe` and generated `models.json` refreshes from persisting resolved plaintext API keys for SecretRef-backed custom model providers.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84658 or an equivalent boundary fix that preserves the pre-resolution source snapshot through models command regeneration, with the Codex marker-only subcase left to https://github.com/openclaw/openclaw/issues/84376.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: model command SecretRefs are resolved, the unpinned path drops the pre-resolution source snapshot, and the generator then treats resolved config as the secret marker source.

## Solution Assessment

Yes. Preserving the source snapshot at `loadModelsConfigWithSource()` is the narrowest maintainable fix because the downstream `models.json` writer already has source-authoritative marker enforcement.
