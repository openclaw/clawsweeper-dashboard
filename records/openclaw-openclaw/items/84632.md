## Summary

Keep open: current `main` is still source-reproducible for the SecretRef source-snapshot gap, and the issue is already paired with an open closing fix PR that has not landed.

## What This Changes

The issue reports that `models status --probe` can rewrite generated per-agent `models.json` files with resolved plaintext `apiKey` values for custom SecretRef-backed providers.

## Best Possible Solution

Land or revise https://github.com/openclaw/openclaw/pull/84658 so the models command path preserves the pre-resolution source snapshot through `models.json` generation, with regression coverage for custom SecretRef-backed provider apiKeys.

## Reproduction Assessment

Yes. Current source shows `models status` resolves `models.providers.*.apiKey`, then the unpinned branch stores only the resolved config, so downstream `models.json` generation can use resolved values as `sourceConfigForSecrets`.

## Solution Assessment

Yes. Preserving the source snapshot is the maintainable fix shape because it keeps marker persistence source-authoritative instead of adding provider-specific marker exceptions after the source proof is already gone.
