## Summary

Keep open: current `main` and the latest `v2026.5.20` release still have the source-snapshot gap, and the linked fix PR is still open rather than merged or shipped.

## What This Changes

The issue reports that `models status --probe` can regenerate per-agent `models.json` with resolved plaintext custom-provider API keys instead of non-secret SecretRef markers.

## Best Possible Solution

Land a reviewed security fix that preserves the pre-resolution source config through models status/probe regeneration and keeps marker-format false positives tracked separately in `https://github.com/openclaw/openclaw/issues/84376`.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: resolve a custom provider `apiKey` SecretRef through `models status`, lose the source snapshot on the unpinned branch, then let `ensureOpenClawModelsJson()` treat the resolved config as `sourceConfigForSecrets`. I did not run the live command because this review must not write agent state.

## Solution Assessment

Yes, preserving the source snapshot through `loadModelsConfigWithSource()` is the narrowest maintainable fix for this issue. A marker-only change would not help once the source SecretRef object has already been replaced by the resolved plaintext value.
