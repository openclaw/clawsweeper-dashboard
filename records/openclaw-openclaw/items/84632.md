## What This Changes

The issue reports that `openclaw models status --probe` can regenerate agent `models.json` with resolved plaintext API keys for SecretRef-backed custom providers instead of non-secret markers.

## Best Possible Solution

Land or revise https://github.com/openclaw/openclaw/pull/84658 so models commands preserve the pre-resolution source snapshot through regeneration, with custom-provider regression coverage and separate handling for the codex marker audit issue.

## Reproduction Assessment

Yes. The current source shows `models status` resolves `models.providers.*.apiKey`, drops the source snapshot on the unpinned path, and lets the generator use the resolved config as `sourceConfigForSecrets`; I did not run the live command because this read-only review must not write agent state.

## Solution Assessment

Yes, the linked PR’s source-snapshot preservation is the narrow maintainable direction. A marker-only audit change would not fully solve this issue if the command path has already lost the pre-resolution SecretRef shape.
