## What This Changes

The issue asks OpenClaw to make embedded-run upstream timeouts configurable and preserve provider 429/Retry-After details instead of collapsing them into generic timeout errors.

## Best Possible Solution

Use the existing `models.providers.<id>.timeoutSeconds` knob for long provider calls, keep the current rate-limit classification, and define a maintainer-approved structured Retry-After field on the embedded-run error contract when upstream headers are available.
