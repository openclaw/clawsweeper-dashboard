## Summary

Keep open. Current main still lacks Feishu `ackReaction` support, so the PR targets a real parity gap, but the proposed patch regresses the shared empty-string disable contract and misses the required changelog entry.

## What This Changes

The PR adds Feishu account/channel `ackReaction` config and docs, extends shared ack-reaction resolution with a Feishu channel default, and passes the resolved emoji into Feishu typing-reaction creation with tests.

## Best Possible Solution

Land a corrected version of the Feishu `ackReaction` parity change that preserves empty-string disable semantics, documents valid Feishu emoji type names, adds release-note coverage, and leaves the native typing API question to #69572.

## Reproduction Assessment

Yes. A focused reproduction is a config with `messages.ackReaction: ""` or `channels.feishu.accounts.<id>.ackReaction: ""`; the current resolver returns `""`, while the PR diff would fall back to `Typing` and still call the Feishu reaction API.

## Solution Assessment

No. The current patch is close, but the maintainable fix must distinguish explicit disable from invalid Feishu emoji fallback and add the required changelog entry before merge.
