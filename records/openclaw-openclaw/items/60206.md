## Summary

Keep open. Current main still lacks Feishu ackReaction support, but this PR remains conflicting and has patch-level correctness blockers plus missing real Feishu/Lark behavior proof before it can merge.

## What This Changes

The PR adds Feishu channel/account ackReaction config and docs, extends resolveAckReaction with a channel default, and passes the resolved value into Feishu typing reactions.

## Best Possible Solution

Land a rebased Feishu ackReaction feature that preserves the shared empty-string disable contract, validates against known Feishu emoji_type names, publishes the setting in plugin metadata/docs, and leaves native typing API replacement to https://github.com/openclaw/openclaw/issues/69572.

## Reproduction Assessment

Yes. Source inspection is enough for the patch bugs: the shared resolver preserves empty ackReaction, while the PR converts it back to Typing, and its regex accepts aliases that the checked Feishu emoji constants do not list.

## Solution Assessment

No. The feature direction is reasonable, but this patch should preserve disable semantics, validate known Feishu emoji names before the API call, update plugin metadata, rebase, and include real Feishu/Lark proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve empty ackReaction as a disable signal:** `extensions/feishu/src/typing.ts:125-131`
  - body: `resolveAckReaction` treats `""` as an explicit disable value, but this path makes an empty resolved emoji incompatible and falls back to `Typing`. Configs such as `messages.ackReaction: ""` or a Feishu account override would still call `messageReaction.create`, so skip dispatch when the resolved value is exactly empty.
  - confidence: 0.93
- **[P2] Validate Feishu emoji types before sending them:** `extensions/feishu/src/typing.ts:125-137`
  - body: The `/^\w+$/` guard only filters Unicode; shared aliases like `eyes` or `party_parrot` still pass even though they are not Feishu emoji type names in the checked constants. If Feishu rejects them, the non-backoff catch path silently drops the typing indicator, so validate known names or fall back before the API call.
  - confidence: 0.86
- **[P3] Publish the new setting in plugin metadata:** `extensions/feishu/src/config-schema.ts:189`
  - body: The runtime schema now accepts `ackReaction`, but plugin metadata still only advertises `typingIndicator` in top-level and per-account config properties. Discovery and setup surfaces that read plugin metadata will miss the new setting until the manifest is updated on rebase.
  - confidence: 0.78
