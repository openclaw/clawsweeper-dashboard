## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve empty ackReaction as disabled:** `extensions/feishu/src/typing.ts:125-131`
  - body: `resolveAckReaction` deliberately returns `""` when users disable reactions, but this path treats the empty string as incompatible and falls back to `Typing`. Configs such as `messages.ackReaction: ""` or a Feishu account override would still call `messageReaction.create`, so skip dispatch when the resolved value is exactly empty.
  - confidence: 0.93
- **[P2] Validate real Feishu emoji types before sending:** `extensions/feishu/src/typing.ts:125-137`
  - body: The `/^\w+$/` guard only filters Unicode; shared aliases like `eyes` or `party_parrot` still pass even though they are not Feishu emoji type names in the checked constants. If Feishu rejects them, the non-backoff catch path drops the typing indicator, so validate known names or fall back before the API call.
  - confidence: 0.86
- **[P3] Publish ackReaction in plugin metadata:** `extensions/feishu/src/config-schema.ts:189`
  - body: The runtime schema now accepts `ackReaction`, but Feishu plugin metadata still advertises only `typingIndicator` in top-level and per-account config properties. Discovery and setup surfaces that read plugin metadata will miss the new setting until the manifest is updated.
  - confidence: 0.78
