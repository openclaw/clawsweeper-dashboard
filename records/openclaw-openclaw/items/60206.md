## Summary

Keep open. Current main does not already provide Feishu ackReaction customization, and this PR is still blocked by an empty-string disable regression, insufficient Feishu emoji validation, stale plugin metadata, and missing real Feishu/Lark proof.

## What This Changes

Adds Feishu channel/account ackReaction config, shared resolver channelDefault support, Feishu typing reaction selection, tests, and docs.

## Best Possible Solution

Land a corrected Feishu reaction-customization feature that preserves empty-string disable semantics, validates or safely falls back using Feishu emoji type names, updates plugin metadata, and leaves native typing replacement to https://github.com/openclaw/openclaw/issues/69572.

## Reproduction Assessment

Yes. Source inspection is enough: the shared resolver preserves an empty ackReaction, while the PR's Feishu typing path converts that value back to Typing and still calls messageReaction.create; live Feishu proof was not run in this read-only review.

## Solution Assessment

No. The feature direction is reasonable, but this patch should preserve the shared disable contract, validate against known Feishu emoji names or fall back before the API call, and keep manifest metadata aligned.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve empty ackReaction as a disable signal:** `extensions/feishu/src/typing.ts:125-131`
  - body: `resolveAckReaction` preserves `""` as an explicit disable value, but this path treats it as incompatible and falls back to `Typing`. Configs such as `messages.ackReaction: ""` or a Feishu account override would still call `messageReaction.create`, so skip dispatch when the resolved value is exactly empty.
  - confidence: 0.93
- **[P2] Validate Feishu emoji names before sending:** `extensions/feishu/src/typing.ts:125-131`
  - body: The `/^\w+$/` guard only filters Unicode; shared aliases like `eyes` or `party_parrot` still pass even though they are not Feishu emoji type names in the checked constants. If Feishu rejects them, the non-backoff catch path silently drops the typing indicator, so validate known names or fall back before the API call.
  - confidence: 0.86
- **[P3] Sync Feishu manifest metadata:** `extensions/feishu/src/config-schema.ts:187`
  - body: The runtime schema accepts `ackReaction`, but `openclaw.plugin.json` still only advertises `typingIndicator` in top-level and per-account config metadata. Discovery and setup surfaces that read plugin metadata will miss the new setting until the manifest is updated.
  - confidence: 0.78
