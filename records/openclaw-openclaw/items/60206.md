## Summary

Keep open. The feature is not implemented on current main, and this PR is a useful direction, but the current head still has blocking correctness issues around disable semantics, Feishu emoji validation, plugin metadata, and missing real Feishu proof.

## What This Changes

The PR adds Feishu channel/account `ackReaction` config and docs, extends the shared resolver with a Feishu channel default, and passes the resolved value into Feishu typing reaction creation.

## Best Possible Solution

Land a corrected Feishu reaction-customization feature that preserves empty-string disable semantics, validates or falls back using Feishu emoji type names, updates plugin metadata, and leaves native typing replacement to https://github.com/openclaw/openclaw/issues/69572.

## Reproduction Assessment

Yes. The PR regressions are source-reproducible from the diff: empty `ackReaction` becomes `Typing`, and word aliases pass the regex despite not being checked Feishu emoji type names; I did not run live Feishu validation in this read-only review.

## Solution Assessment

No. The feature direction is maintainable, but the patch should preserve the shared disable contract, validate against known Feishu emoji names or fall back before the API call, and keep manifest metadata aligned.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Skip typing when ackReaction resolves to empty:** `extensions/feishu/src/typing.ts:125-131`
  - body: `resolveAckReaction` preserves `""` as an explicit disable value, but this branch treats it as incompatible and falls back to `Typing`. Configs that disabled reactions globally or for the Feishu account would still call `messageReaction.create`, so the start path should skip dispatch when the resolved value is exactly empty.
  - confidence: 0.93
- **[P2] Validate against Feishu emoji type names:** `extensions/feishu/src/typing.ts:125-131`
  - body: The `/^\w+$/` guard only filters Unicode; shared aliases such as `eyes` or `party_parrot` still pass even though they are absent from the checked Feishu emoji constants. If Feishu rejects them, the non-backoff catch path silently drops the typing indicator; validate known Feishu names or fall back before the API call.
  - confidence: 0.86
- **[P3] Keep the Feishu plugin manifest schema in sync:** `extensions/feishu/src/config-schema.ts:187`
  - body: The runtime schema accepts `ackReaction`, but `openclaw.plugin.json` still only advertises `typingIndicator` in top-level and per-account config metadata. Discovery/setup surfaces that read plugin metadata will miss the new setting until the manifest is updated.
  - confidence: 0.78
