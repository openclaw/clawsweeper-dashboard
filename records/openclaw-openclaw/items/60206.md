## Summary

Keep open: current main still lacks Feishu ackReaction support, and this PR remains directionally useful, but the patch has merge-blocking correctness gaps and no real Feishu/Lark proof yet.

## What This Changes

The PR adds Feishu channel/account ackReaction config and docs, resolves a typing emoji through the shared ackReaction helper, and passes it into the Feishu typing reaction call.

## Best Possible Solution

Land a rebased Feishu ackReaction feature that preserves empty-string disable semantics, validates known Feishu emoji type names or falls back safely, updates plugin metadata, and leaves native typing replacement to the separate Feishu API discussion.

## Reproduction Assessment

Yes. Source inspection is enough for the introduced patch bugs: the shared resolver preserves an empty ackReaction, while the PR converts it back to `Typing`, and the regex accepts non-Feishu aliases that the checked emoji constants do not list.

## Solution Assessment

No. The feature direction is reasonable, but the patch should preserve the shared disable contract, validate against known Feishu emoji names or fall back before the API call, keep plugin metadata aligned, and include real Feishu/Lark proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve empty ackReaction as a disable signal:** `extensions/feishu/src/typing.ts:125-131`
  - body: `resolveAckReaction` treats `""` as an explicit disable value, but this path makes an empty resolved emoji incompatible and falls back to `Typing`. Configs such as `messages.ackReaction: ""` or a Feishu account override would still call `messageReaction.create`, so skip dispatch when the resolved value is exactly empty.
  - confidence: 0.93
- **[P2] Validate Feishu emoji types before sending them:** `extensions/feishu/src/typing.ts:125-131`
  - body: The `/^\w+$/` guard only filters Unicode; shared aliases like `eyes` or `party_parrot` still pass even though they are not Feishu emoji type names in the checked constants. If Feishu rejects them, the non-backoff catch path silently drops the typing indicator, so validate known names or fall back before the API call.
  - confidence: 0.86
- **[P3] Publish the new setting in plugin metadata:** `extensions/feishu/src/config-schema.ts:189`
  - body: The runtime schema now accepts `ackReaction`, but `openclaw.plugin.json` still only advertises `typingIndicator` in top-level and per-account config metadata. Discovery and setup surfaces that read plugin metadata will miss the new setting until the manifest is updated.
  - confidence: 0.78
