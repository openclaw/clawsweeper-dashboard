- **[P1] Keep the full command inspectable before approval:** `src/infra/approval-view-model.ts:75`
  - body: This changes native approval views to expose only the summarized command text for long commands. Approval prompts are the user's security review point, and current main only accepts exec approvals after the sanitized command display remains bounded; after this change a user can approve a hidden tail without any full sanitized command inspection path. Please keep the summary for readability but add an owner-approved way to inspect/copy the full sanitized command before approval.
  - confidence: 0.86

Current main already resolves omitted decisions from `ask` and shows the unavailable warning, so the rebase should keep that behavior rather than using the stale default-only fallback.
  - confidence: 0.78
