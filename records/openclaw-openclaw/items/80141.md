## Summary

Keep open: this PR addresses a real, source-reproducible approval UX bug, but the current patch still double-summarizes native Matrix/Telegram approval text and the supplied proof does not show the visible modal or native prompt behavior.

## What This Changes

The branch adds display-only exec approval command summarization, carries allowedDecisions through Control UI approval parsing/rendering, updates docs/changelog, and adds focused approval/UI tests.

## Best Possible Solution

Apply summarization exactly once per human-facing approval surface, preserve the full sanitized command for matching/storage, honor gateway-provided allowed decisions in UI actions, and add visible after-fix proof for the modal or native prompt path.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main broadcasts allowedDecisions but the Control UI parser drops them and the modal renders all actions, while long sanitized command text has no prompt-level summary.

## Solution Assessment

No. The direction is right, but the patch should avoid re-summarizing native approval text and needs real visible approval-surface proof before merge.
