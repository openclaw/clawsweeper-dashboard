Fix https://github.com/openclaw/openclaw/issues/79435. Current main can report `channel: (unknown)` for `openclaw sandbox explain --session agent:main:telegram:direct:<id>` because `inferProviderFromSessionKey` in `src/commands/sandbox-explain.ts` falls through to `normalizeAnyChannelId`, which only checks the active plugin channel registry. Do not add new config flags or widen elevated defaults.
