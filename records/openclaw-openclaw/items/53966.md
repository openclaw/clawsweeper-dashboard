## Summary

Keep open. Current main now covers the delivery-queue missing-scope subset, but this PR still targets other Slack diagnostics and its helper bypasses the existing redaction contract, so it should not merge as written.

## What This Changes

The PR adds a Slack-local `formatSlackError()` helper with unit tests and routes selected Slack probe, scope, startup resolution, stream preview, reaction, slash reply, and thread-resolution error paths through it.

## Best Possible Solution

Use a Slack-plugin-local formatter that allowlists Slack SDK detail fields, redacts the final assembled text through the plugin-safe logging SDK export, adds token-redaction regression coverage, and keeps the delivery-queue fix already on main.

## Reproduction Assessment

Yes. A focused unit check can pass a token-shaped Slack/Web API error into the PR's `formatSlackError()` and observe the raw token in the returned string, while current main's `formatErrorMessage()` redacts the same token patterns.

## Solution Assessment

No as written. A Slack-local formatter is the right boundary for API details, but replacing `formatErrorMessage` call sites must preserve final-string redaction and include the required changelog entry.
