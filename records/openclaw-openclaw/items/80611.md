## Summary

Keep open: current main still leaves the default text status summary on paths that can enter manifest-backed model normalization and per-row CLI-provider discovery, and the open draft fix PR is the implementation path to review rather than a pre-merge close.

## What This Changes

The issue asks to keep default/non-deep `openclaw status` on a shallow read-only path instead of spending about 20 seconds in repeated plugin metadata/model normalization work.

## Best Possible Solution

Repair and land https://github.com/openclaw/openclaw/pull/80614 or a narrow replacement that keeps status summary model/runtime labeling off expensive metadata paths while leaving broader daemon metadata-cache work with https://github.com/openclaw/openclaw/issues/78461.

## Reproduction Assessment

Yes, source-backed rather than locally timed: the reported `/usr/bin/time -p openclaw status` path has concrete v2026.5.7 timings, and current main still contains the status parsing/runtime-label paths that can trigger the metadata work.

## Solution Assessment

Yes for the narrow direction: disable manifest normalization for status-only model parsing, cache CLI-provider detection within the status render, and avoid duplicate row enrichment. The issue should not close until the linked PR or an equivalent replacement lands.
