## Summary

Current main and the latest release still settle visible ACP text before final-mode TTS is attempted, so Telegram can receive a text reply before the voice supplement. This should stay open as a focused Telegram/TTS delivery improvement; the related open PR deletes and resends after the fact, which does not fully address the no-churn goal.

## What This Changes

Request to change ACP final-mode TTS delivery so caption-capable Telegram voice replies can carry the final text as a caption instead of sending visible text before audio.

## Best Possible Solution

Add an explicit captioned-voice delivery capability, defer ACP final visible-text settling only when final-mode TTS can use that capability, and flush the original text if synthesis or media delivery fails.

## Reproduction Assessment

Yes from source inspection. On current main and v2026.5.12, ACP finalization settles visible text before final TTS synthesis, then sends the generated voice media as a supplement without caption text.

## Solution Assessment

Yes, the proposed defer-then-fallback shape is the narrowest maintainable direction if it is guarded by an explicit captioned-voice channel capability. A Telegram-only core special case or unconditional defer would be less safe.
