## Summary

Keep open. Current main and the latest release still settle visible ACP text before final TTS, and the linked closing PR is open rather than merged, so this issue should track the fix until that PR is repaired, proven, and landed.

## What This Changes

Request deferring final-mode TTS visible-text delivery on Telegram so a successful voice note can carry the answer as a caption, with text fallback on TTS failure.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/83988 after it gates deferral to caption-capable voice delivery, preserves non-Telegram ordering, adds focused regression coverage, and includes real Telegram proof; then close this issue after merge.

## Solution Assessment

Yes for the issue direction: deferring text only for caption-capable voice delivery with a text fallback is the narrow maintainable shape. The open PR needs that capability guard before it is the best implementation.
