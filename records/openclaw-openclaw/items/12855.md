## Summary

Keep open. Current main and v2026.5.19 ship a default-off package auto-updater, but they still lack the requested schedule/timezone contract, confirm and notify-only modes, channel-targeted notifications, and post-restart version/changelog/health reporting.

## What This Changes

The issue requests a first-class `update.auto` workflow with configurable schedule, confirmation or notify-only modes, silent mode, and post-update notifications.

## Best Possible Solution

Extend the existing core updater with approval modes, timezone-aware scheduling, safe notification targeting, persisted restart state, changelog/health reporting, docs, and tests while leaving the Control UI manager to https://github.com/openclaw/openclaw/issues/65011.
