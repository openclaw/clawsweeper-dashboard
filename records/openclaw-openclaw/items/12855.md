## Summary

Keep open. Current main and v2026.5.20 still ship the narrower default-off package auto-updater, while the requested schedule/timezone contract, confirm and notify-only modes, channel-targeted notifications, and post-restart version/changelog/health reporting remain absent; https://github.com/openclaw/openclaw/issues/65011 is a UI/discoverability follow-up, not a replacement for this engine workflow.

## What This Changes

Request adds a built-in `update.auto` workflow with schedule/timezone, confirm/notify/silent modes, channel notifications, and post-restart reporting.

## Best Possible Solution

Extend the existing core package updater with approval modes, timezone-aware scheduling, safe notification targeting, persisted restart state, changelog/health reporting, docs, and tests while leaving Control UI discoverability to https://github.com/openclaw/openclaw/issues/65011.

## Solution Assessment
