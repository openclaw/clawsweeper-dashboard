## Summary

Keep open: current main has a useful default-off package auto-updater, but it still lacks the requested wall-clock or cron scheduling, timezone selection, confirm and notify-only modes, configured channel notifications, and post-restart old/new version plus changelog and health reporting. The latest discussion already treats this as the canonical updater engine/workflow tracker, with the Control UI issue kept separate for discoverability.

## What This Changes

Add first-class `update.auto` scheduling, approval modes, notification targeting, and post-update reporting.

## Best Possible Solution

Keep this as the canonical updater engine/workflow request and extend `update.auto` through a designed schema, runtime, docs, and tests contract for scheduling, approval, notifications, restart-state, changelog, health, and audit behavior.
