## Summary

Keep open: current main and v2026.5.12 still ship only the narrower package auto-updater. The requested scheduling, approval modes, notification targeting, and post-restart reporting remain unimplemented, and the discussion already treats this as the canonical updater engine/workflow tracker.

## What This Changes

Request adds a first-class `update.auto` workflow with configurable schedule/timezone, confirm/silent/notify-only modes, channel-targeted notifications, and post-update old/new version reporting.

## Best Possible Solution

Keep this as the canonical updater engine/workflow request and extend `update.auto` with a designed schema, runtime, docs, and tests contract for scheduling, approval, notifications, restart-state, changelog, health, and audit behavior, while leaving https://github.com/openclaw/openclaw/issues/65011 as the separate UI/discoverability follow-up.
