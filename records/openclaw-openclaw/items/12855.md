## Summary

Keep this open: current main and the latest release have a useful default-off package auto-updater, but they still do not implement the requested first-class workflow for wall-clock scheduling, confirm/notify-only modes, channel-targeted post-update notices, or old/new version plus health reporting.

The issue requests a built-in `update.auto` workflow with configurable schedule, approval mode, and post-update channel notification behavior.

## Best Possible Solution

Keep this as the canonical updater engine/workflow request and design the approval, scheduling, delivery, and post-restart reporting layer on top of the current package auto-updater and `update.run` primitives.
