## Summary

Keep open: current main still implements a narrower default-off package auto-updater, while this issue asks for a first-class scheduled, confirmed or notify-only workflow with channel notifications and post-restart reporting. Recent discussion confirms the shipped `update.auto.*` primitives are useful but incomplete, and the separate UI/discoverability issue does not replace this engine/workflow request.

## What This Changes

Request to add a configurable built-in auto-update workflow with schedule/timezone, confirm/silent/notify-only modes, channel notifications, and post-update version/changelog/health reporting.

## Best Possible Solution

Keep this as the canonical updater engine/workflow request, and extend the existing updater only after maintainers define the scheduling, approval, notification target, restart-state, changelog, health, and audit contract.
