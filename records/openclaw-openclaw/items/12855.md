## Summary

Keep open: current main has a default-off package auto-updater, but it still lacks the requested wall-clock or cron schedule, timezone, confirm and notify-only modes, channel-targeted notifications, and post-restart old/new version plus changelog/health reporting. The related Control UI request at https://github.com/openclaw/openclaw/issues/65011 is a separate discoverability/product surface, while this issue remains the canonical updater engine/workflow tracker.

## What This Changes

The issue requests a first-class `update.auto` workflow with configurable schedule, timezone, confirmation or notify-only modes, channel notifications, and post-update reporting.

## Best Possible Solution

Keep this as the canonical updater engine/workflow request, then extend `update.auto` only after maintainers define the scheduling, approval, notification-target, restart-state, changelog, health, and audit contracts.
