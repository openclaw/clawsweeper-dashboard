## Summary

Keep open. Current main and v2026.5.20 still ship the narrower default-off package auto-updater, while the requested schedule/timezone contract, confirm and notify-only modes, channel-targeted notifications, and post-restart version/changelog/health reporting remain absent; the related UI/discoverability issue is separate follow-up work.

## What This Changes

The issue requests a first-class auto-update workflow with schedule/timezone configuration, confirmation or notify-only modes, silent mode, channel notifications, and post-update reporting.

## Best Possible Solution

Extend the existing core package updater with approval modes, timezone-aware scheduling, safe notification targeting, persisted restart state, changelog/health reporting, docs, and tests while leaving Control UI discoverability to https://github.com/openclaw/openclaw/issues/65011.
