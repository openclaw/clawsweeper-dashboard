## Summary

Keep open. Current main has a default-off package auto-updater, but the requested first-class scheduled, confirmable, channel-notifying update workflow is still missing and the live discussion keeps this as the canonical engine/workflow tracker.

## What This Changes

Request a built-in `update.auto` workflow with schedule/timezone settings, confirm/notify-only/silent modes, channel notifications, and post-restart update reporting.

## Best Possible Solution

Keep this as the canonical updater engine/workflow request and extend the existing package auto-updater with additive config, scheduler, approval, notification, restart-state, docs, and tests while leaving UI discoverability to the separate manager issue.
