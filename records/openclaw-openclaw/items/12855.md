## Summary

Keep this open: current main and v2026.6.6 have a default-off package auto-updater, but not the requested first-class workflow for wall-clock or cron scheduling, confirmation or notify-only modes, channel-targeted notification, or post-restart reporting.

## What This Changes

Request to add a first-class updater workflow with configured schedule/timezone, confirm/notify/silent modes, and post-update channel notifications.

## Best Possible Solution

Keep this issue as the canonical updater engine/workflow request until maintainers choose the update policy, approval semantics, channel-targeting model, and restart-state storage.
