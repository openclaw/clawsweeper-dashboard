## Summary

Keep open: current main and v2026.6.8 ship a narrower default-off package auto-updater, but this issue still asks for scheduled approval modes, channel-targeted notification, post-restart reporting, and an on-demand chat update surface that need maintainer product/security design.

Add a built-in auto-update workflow with configurable schedule, confirmation or notification modes, target channel selection, and post-update version/health reporting.

## Best Possible Solution

Keep this as the canonical updater workflow tracker until maintainers choose the approval policy, scheduling API, channel-targeting model, and restart-state storage, then implement it on top of the supervised update.run/update handoff path with schema, docs, and tests.

Unclear as an exact implementation: the capability is useful and belongs near the core updater, but the proposed update.auto expansion needs maintainer product/security design before code should land. A safer path is one supervised updater workflow that reuses update.run and separates scheduling, approval, delivery, and restart-state ownership cleanly.
