## Summary

Keep open: current main ships a narrower default-off package auto-updater, but it still lacks the requested wall-clock/cron scheduling, confirm or notify-only modes, configured channel notifications, and rich post-restart reporting. The discussion supports that boundary: #12896 is closed unmerged, Bortlesboat confirmed the existing primitives are usable, and #65011 tracks a separate UI/discoverability follow-up rather than replacing this engine/workflow request.

## What This Changes

Request adding built-in `update.auto` scheduling, confirm/notify/silent modes, channel notifications, and post-update version/changelog/health reporting.

## Best Possible Solution

Keep this as the canonical updater engine/workflow request, extend the existing package updater and restart-sentinel/update delivery surfaces after maintainers settle the approval, scheduling, and notification contract, and leave #65011 as the separate UI/discoverability follow-up.
