## Summary

Keep open: current main has a useful default-off package auto-updater, but it still lacks the requested scheduled confirm/notify workflow, channel-targeted notifications, and rich post-restart reporting.

## Best Possible Solution

Keep this as the canonical updater engine/workflow request and extend the existing package auto-updater plus restart-sentinel/update delivery surfaces after the desired approval, scheduling, and notification contract is settled.
