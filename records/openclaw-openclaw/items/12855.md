## Summary

Keep open. Current main and the latest release have a default-off package auto-updater, but they still lack the requested scheduled confirmation/notify-only workflow, channel-targeted post-update notifications, and restart summary behavior; related discussion treats this as the canonical engine tracker rather than the separate UI follow-up.

## What This Changes

The issue requests a first-class `update.auto` workflow with scheduled checks, confirm/notify-only/silent modes, notification targets, and post-update reporting.

## Best Possible Solution

Extend the existing core updater with approval modes, timezone-aware scheduling, safe notification targeting, persisted restart state, changelog/health reporting, docs, and tests while leaving the Control UI manager to the separate follow-up.
