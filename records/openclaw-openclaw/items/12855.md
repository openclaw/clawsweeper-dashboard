## Summary

Keep open: current main and the latest release ship a useful default-off package auto-updater, but they still do not implement the requested first-class workflow for scheduled confirm/notify modes, timezone or cron scheduling, channel-targeted notifications, post-restart reporting, or the newer on-demand chat update surface.

## What This Changes

Requests a built-in auto-update workflow with `update.auto` scheduling, confirmation/silent/notify-only modes, timezone handling, channel notifications, post-update reporting, and related supervised chat update behavior.

## Best Possible Solution

Keep this as the canonical updater engine/workflow tracker and design one supervised update workflow on top of `update.run` or the external-updater direction, with explicit approval modes, safe channel delivery, durable restart reporting, docs, schema/help, tests, and coordination with https://github.com/openclaw/openclaw/issues/64892 and https://github.com/openclaw/openclaw/issues/77511.

## Reproduction Assessment

Not applicable: this is a feature request rather than a broken existing behavior report. Source inspection gives high confidence that current main lacks the requested config fields, approval modes, scheduler, channel notification, and chat update command surfaces.

## Solution Assessment

No: the current package auto-updater is a useful foundation but does not solve the requested workflow. The best path is a maintainer-approved, supervised updater engine that avoids parallel update doors and keeps approval, delivery, restart state, and docs/tests in one design.
