## Summary

Keep open: current main and the latest release ship a narrower default-off package auto-updater, but they still lack the requested wall-clock/cron scheduling, timezone selection, confirm and notify-only modes, notification targeting, and post-restart old/new version plus changelog/health reporting. The live discussion and related duplicate closure already treat this as the canonical updater engine/workflow tracker, while the Control UI item remains separate for discoverability.

## What This Changes

The issue requests a built-in `update.auto` workflow with schedule/timezone configuration, confirm/silent/notify-only modes, notification targeting, and post-update status reporting.

## Best Possible Solution

Keep this as the canonical updater engine/workflow request and extend `update.auto` with a designed schema, runtime, docs, and tests contract for scheduling, approval, notifications, restart-state, changelog, health, and audit behavior.
