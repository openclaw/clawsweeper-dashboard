## Summary

Keep this issue open: current main and the latest release ship a narrower default-off package auto-updater, while the central scheduled approval and post-update notification workflow remains missing and has no viable replacement tracker.

## What This Changes

Add configurable scheduled update checks and applies with confirm, silent, and notify-only modes plus targeted post-restart reporting.

## Best Possible Solution

Keep this as the canonical updater-workflow decision and design one core path that composes existing update execution, scheduling, outbound delivery, restart continuation, and health checks while keeping silent apply explicitly opt-in.
