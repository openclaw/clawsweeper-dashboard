## Summary

Keep open. Current main still implements the narrower default-off package auto-updater, while the requested wall-clock/cron scheduling, confirmation/notify-only modes, channel targeting, and post-restart old/new reporting remain absent; the provided maintainer-context review also keeps this as the updater engine/workflow tracker rather than treating the UI follow-up as a replacement.

## What This Changes

Request adds a first-class `update.auto` workflow with configurable scheduling, approval modes, post-update notifications, and channel targeting.

## Best Possible Solution

Keep this as the canonical updater engine/workflow request and extend the existing package updater with additive config, scheduler, approval, notification, restart-state, docs, and tests while leaving UI discoverability to https://github.com/openclaw/openclaw/issues/65011.

## Reproduction Assessment

## Solution Assessment

Yes for the product direction, but not as a single automated repair. The maintainable path is to extend the existing updater engine and config contract after maintainer product/security slicing, not close this behind the UI follow-up or revive the closed branch wholesale.
