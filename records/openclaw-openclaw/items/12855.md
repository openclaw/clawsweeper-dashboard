## Summary

Keep open: current main has a default-off package auto-updater, but it still lacks the requested scheduled/timezone workflow, confirm and notify-only modes, channel-targeted notifications, and post-restart reporting. Recent related discussion treats this as the canonical updater engine request, while the UI/discoverability and safe-update issues cover separate follow-up surfaces.

## What This Changes

Request a first-class `update.auto` workflow with scheduling, approval modes, notification targeting, and post-update old/new version reporting.

## Best Possible Solution

Keep this as the canonical updater engine/workflow request, then extend `update.auto` only after maintainers define the scheduling, approval, notification, restart-state, changelog, health, and audit contracts.
