## Summary

Keep open. Current main and the latest release still ship the narrower default-off package auto-updater, while the requested schedule/timezone contract, confirm and notify-only modes, channel-targeted notifications, and post-restart version/changelog/health reporting remain absent; the separate Control UI follow-up does not replace this engine workflow request.

## What This Changes

Request a first-class built-in auto-update workflow with scheduled checks, confirmation or notify-only modes, and post-update channel notifications.

## Best Possible Solution

Extend the existing core package updater with approval modes, timezone-aware scheduling, safe notification targeting, persisted restart state, changelog/health reporting, docs, and tests while leaving Control UI discoverability to https://github.com/openclaw/openclaw/issues/65011.
