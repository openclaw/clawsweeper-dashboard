## Summary

Keep open. Current main still ships only the narrower default-off package auto-updater, while the requested wall-clock/cron schedule, confirm/notify-only modes, target-channel notifications, and post-restart old/new reporting remain absent; the latest maintainer-context comments also keep this as the canonical updater engine/workflow tracker.

## What This Changes

Request first-class `update.auto` workflow settings for scheduled auto-update checks, confirmation/silent/notify-only modes, channel notifications, and post-update reporting.

## Best Possible Solution

Keep this as the canonical updater engine/workflow request and extend the existing package updater with additive config, scheduler, approval, notification, restart-state, docs, and tests while leaving Control UI discoverability to https://github.com/openclaw/openclaw/issues/65011.
