## Summary

Keep open. Current main still ships the narrower default-off package auto-updater, while wall-clock or cron scheduling, timezone selection, confirm/notify-only modes, target-channel notifications, and post-restart old/new reporting remain absent; the related UI work at https://github.com/openclaw/openclaw/issues/65011 is a separate follow-up, and https://github.com/openclaw/openclaw/issues/79194 was closed back to this canonical workflow request.

## What This Changes

The issue requests a first-class auto-update workflow with configurable schedule/timezone, confirm/notify-only/silent modes, post-update notification, and channel targeting.

## Best Possible Solution

Keep this as the canonical updater engine/workflow request and extend the existing package updater with additive config, scheduler, approval, notification, restart-state, docs, and tests while leaving Control UI discoverability to https://github.com/openclaw/openclaw/issues/65011.

## Reproduction Assessment

Not applicable as a runtime bug. Source inspection gives high confidence that current main lacks the requested feature because schema, runtime, docs, and tests only cover the narrower package auto-updater.

## Solution Assessment

Yes for keeping this open as the canonical product direction, but not as a single automated repair. The maintainable path is to extend the existing updater engine after maintainer product and security slicing, not close this behind the UI follow-up or revive the closed branch wholesale.
