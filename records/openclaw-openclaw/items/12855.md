## Summary

Keep open: current main has a useful default-off package auto-updater, but it still lacks the requested schedule/timezone, confirm/notify-only modes, channel-targeted notification, and post-restart old/new changelog/health workflow. The issue requests extending `update.auto` into a first-class scheduled auto-update workflow with approval modes, channel-targeted notifications, and post-restart reporting.
