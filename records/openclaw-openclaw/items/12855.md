## Summary

Keep open: current main has a useful default-off package auto-updater, but it still lacks the requested scheduled, confirm/notify-only, channel-targeted, post-restart workflow; https://github.com/openclaw/openclaw/issues/65011 is a separate UI/discoverability follow-up, and https://github.com/openclaw/openclaw/issues/79194 was closed back to this canonical engine tracker.

## What This Changes

Add a first-class `update.auto` workflow with schedule/timezone controls, confirm/notify-only/silent modes, channel notifications, and post-restart version/changelog/health reporting.

## Best Possible Solution

Keep this as the canonical updater engine/workflow tracker and extend the existing updater only after maintainers define the scheduling, approval, notification, restart-state, changelog, and health-reporting contract.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives a high-confidence verification path: the schema, types, runtime, docs, and tests still lack the requested workflow fields and behaviors.

## Solution Assessment
