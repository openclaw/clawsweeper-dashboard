## Summary

Keep open. Current main and the latest v2026.5.20 release still ship the narrower default-off package auto-updater, while the requested schedule/timezone contract, confirm and notify-only modes, channel-targeted notifications, and post-restart version/changelog/health reporting remain absent.

The issue requests a first-class `update.auto` workflow with configurable scheduling, approval modes, channel notifications, and post-update reporting.

## Best Possible Solution

Extend the existing core package updater with approval modes, timezone-aware scheduling, safe notification targeting, persisted restart state, changelog/health reporting, docs, and tests while leaving Control UI discoverability to https://github.com/openclaw/openclaw/issues/65011.
