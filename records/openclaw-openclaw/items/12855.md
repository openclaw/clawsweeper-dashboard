## Summary

Keep open. Current main and v2026.5.20 still provide only the narrower default-off package auto-updater, while schedule/timezone configuration, confirm and notify-only modes, channel-targeted notifications, and post-restart version/changelog/health reporting remain unimplemented; the separate UI follow-up at https://github.com/openclaw/openclaw/issues/65011 does not replace this engine/workflow request.

## What This Changes

The issue requests a built-in `update.auto` workflow with schedule/timezone settings, confirm/silent/notify-only modes, and post-update notifications.

## Best Possible Solution

Extend the existing core package updater with approval modes, timezone-aware scheduling, safe notification targeting, persisted restart state, changelog/health reporting, docs, and tests while leaving Control UI discoverability to https://github.com/openclaw/openclaw/issues/65011.
