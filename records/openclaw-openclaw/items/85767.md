## Summary

Keep open: current main still awaits the memory flush before starting the visible reply path, and this PR is a focused attempt to move that work off the critical path. It is not merge-ready without real runtime proof and maintainer acceptance of the changed visible-error and timeout-abandonment semantics.

## What This Changes

This PR defers auto-reply memory flush to post-reply maintenance, adds a per-session pending-flush barrier plus maintenance ReplyOperation, and expands focused tests.

## Best Possible Solution
