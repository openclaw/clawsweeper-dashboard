## Summary

## What This Changes

The PR defers auto-reply memory flushes until after reply completion, adds a per-session pending-flush barrier before later preflight compaction, introduces a maintenance ReplyOperation, and expands focused regression coverage.

## Best Possible Solution

Land the deferred-flush design only if maintainers accept the bounded session-state and availability tradeoff, with focused auto-reply tests plus broader pre-merge proof covering reply delivery and next-turn compaction ordering.
