## Summary

Keep this PR open. The post-reply flush direction is useful, but the latest timeout path can still block a later preflight indefinitely if the aborted flush promise never settles, and the PR still needs real runtime proof plus maintainer acceptance for changing visible memory-flush errors to log-only output.

## What This Changes

The PR defers auto-reply near-threshold memory flush work until after reply completion, adds a pending-flush barrier and maintenance reply operation, and expands focused regression tests.

## Best Possible Solution

Keep the post-reply flush design, but make the timeout contract either genuinely bounded and terminal or explicitly fail closed, decide the visible-error delivery policy, and require redacted real runtime proof before merge.
