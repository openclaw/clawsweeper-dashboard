## Summary

Current main still does not bundle Sysbox, but it now exposes a public, arbitrary-id sandbox backend plugin contract and an npm/ClawHub OpenShell backend proves that non-Docker runtimes can own lifecycle, execution, filesystem bridging, and runtime-specific configuration outside core. The requested Sysbox integration is therefore useful optional infrastructure that can be built without adding Sysbox-specific host policy or another Docker runtime knob to OpenClaw core.

## What This Changes

Add Sysbox-backed, socketless Docker sandbox isolation and optionally expose OCI runtime selection for sandbox containers.

## Best Possible Solution

Publish and maintain a Sysbox sandbox backend as an npm/ClawHub plugin with plugin-owned configuration, host prerequisite checks, lifecycle handling, security documentation, and real Linux proof; open a narrow core SDK issue only for a demonstrated generic blocker.
