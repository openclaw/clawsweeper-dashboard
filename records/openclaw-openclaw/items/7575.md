## Summary

Keep open. Current main still lacks Sysbox/sysbox-runc support, a Docker runtime selector, and Docker `--runtime` wiring, while maintainer discussion preserves this as the canonical security-sensitive runtime architecture tracker.

## What This Changes

Request to add Sysbox/sysbox-runc-based sandbox isolation and optional Docker runtime selection for socketless Docker-in-Docker deployments.

## Best Possible Solution

Keep this as the canonical sandbox runtime design tracker until maintainers approve either a strict core Docker runtime selector or a Sysbox sandbox backend/plugin with host prerequisite docs, security guidance, and focused tests.

## Reproduction Assessment

Not applicable: this is a feature/security architecture request, not a broken existing behavior. Source inspection on current main confirms Sysbox/sysbox-runc, `sandbox.docker.runtime`, and Docker `--runtime` wiring remain absent.

## Solution Assessment

Unclear: a core `agents.defaults.sandbox.docker.runtime` selector would solve part of the request, but Sysbox also changes host prerequisites and isolation semantics. Maintainers should choose between a core selector and a plugin/backend design before implementation.
