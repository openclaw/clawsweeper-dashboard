## Summary

Keep open. Current main still lacks Sysbox/sysbox-runc support, a Docker sandbox runtime selector, and Docker `--runtime` create wiring; live discussion also keeps this scope separate from the shipped Docker-socket sandbox work.

## What This Changes

Request support for Sysbox-based Docker sandbox isolation, including host runtime setup and optional Docker runtime selection for sandbox containers.

## Best Possible Solution

Keep this as the canonical sandbox runtime design tracker until maintainers approve either a strict core Docker runtime selector or a Sysbox backend plugin with host prerequisite and security guidance.

## Reproduction Assessment

Not applicable: this is a feature/security architecture request, not broken existing behavior. Current source and docs confirm the requested Sysbox/runtime selector and Docker `--runtime` wiring are absent.

## Solution Assessment

Unclear: a core `agents.defaults.sandbox.docker.runtime` selector would solve part of the request, but Sysbox also changes host prerequisites and isolation semantics. Maintainers should choose between a core selector and a plugin/backend design before implementation.
