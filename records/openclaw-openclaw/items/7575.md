## Summary

Keep open: current main has Docker, SSH, OpenShell, and plugin sandbox-backend support, but it still does not implement Sysbox/sysbox-runc or a Docker runtime selector; this remains a security-sensitive product decision rather than cleanup.

## What This Changes

Request host-level Sysbox/sysbox-runc sandbox support that removes the Docker socket mount and optionally exposes Docker runtime selection in sandbox config.

## Best Possible Solution

Keep this open until maintainers choose either a conservative core Docker runtime selector with schema/types/docs/tests or a plugin-first Sysbox sandbox backend using the existing `registerSandboxBackend` seam and documented host prerequisites.

## Reproduction Assessment

Not applicable: this is a feature/security architecture request, not a broken existing behavior. Source and docs checks confirm current main lacks Sysbox/sysbox-runc and `sandbox.docker.runtime`.

## Solution Assessment

Unclear: the requested direction is plausible, but the safest maintainable shape requires maintainer choice between a core Docker runtime selector and a plugin-backed Sysbox backend with a reviewed threat model.
