## Summary

Keep open. Current main still does not implement Sysbox/sysbox-runc, does not accept `agents.defaults.sandbox.docker.runtime`, and does not pass Docker `--runtime`; the issue discussion also preserves this as the security-sensitive sandbox runtime architecture tracker rather than completed Docker-socket sandboxing work.

## What This Changes

Request adding Sysbox-based Docker sandbox isolation, including host runtime registration, socketless gateway Docker-in-Docker, and optional Docker runtime selection.

## Best Possible Solution

Keep this as the canonical sandbox runtime design tracker until maintainers approve either a strict core Docker runtime selector with docs/tests or a Sysbox backend plugin with host prerequisite and security guidance.

## Reproduction Assessment

Not applicable: this is a feature/security architecture request, not a broken existing behavior. Source and docs on current main confirm the requested Sysbox/runtime selector and Docker `--runtime` wiring are absent.

## Solution Assessment

Unclear: a core `agents.defaults.sandbox.docker.runtime` selector would solve part of the request, but Sysbox also changes host prerequisites and isolation semantics. Maintainers should choose between a core selector and a plugin/backend design before implementation.
