## Summary

Keep open: current main still lacks Sysbox/sysbox-runc support and `agents.defaults.sandbox.docker.runtime`; related runtime-selector PRs are closed unmerged, and the discussion already keeps this as a security-sensitive design tracker rather than completed sandbox cleanup.

## What This Changes

Request to add Sysbox/sysbox-runc sandbox isolation or Docker runtime selection so sandboxed execution can avoid the host Docker socket path.

## Best Possible Solution

Keep this open until maintainers choose either a constrained core Docker runtime selector with schema/types/docs/tests, or a plugin-first Sysbox sandbox backend using `registerSandboxBackend` with documented host prerequisites and security tradeoffs.
