## Summary

Keep open. Current main still lacks Sysbox/sysbox-runc support and a Docker runtime selector, while the issue discussion explicitly keeps this scope separate from the shipped Docker-socket sandbox and sandbox-common work.

## What This Changes

The issue requests Sysbox/sysbox-runc support for Docker sandbox isolation without mounting the host Docker socket, plus possible runtime selection through sandbox Docker config.

## Best Possible Solution

Keep this as the canonical design tracker until maintainers choose either a narrow core `agents.defaults.sandbox.docker.runtime` selector with schema/create-args/docs/tests, or a Sysbox backend plugin using `registerSandboxBackend` with host prerequisites and security tradeoffs documented.

## Reproduction Assessment

Not applicable as a feature request. Current-main verification is high-confidence: source inspection shows no Sysbox references, no `sandbox.docker.runtime` type/schema key, and no Docker `--runtime` create wiring.

## Solution Assessment

Unclear. The direct host-level Sysbox plan should not be applied as a routine fix; the maintainable path needs maintainer choice between a core runtime selector and a sandbox backend plugin.
