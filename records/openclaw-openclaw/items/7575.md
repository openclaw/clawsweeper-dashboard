## Summary

Keep open: current main still lacks Sysbox/sysbox-runc integration or a Docker runtime selector, and recent discussion explicitly treats this as the canonical sandbox runtime architecture/security tracker rather than completed Docker-socket sandbox support.

## What This Changes

Request host-level Sysbox Docker runtime support and/or configurable sandbox Docker runtime selection to reduce reliance on mounting the host Docker socket for sandbox containers.

## Best Possible Solution

Keep this as the canonical sandbox runtime architecture tracker until owners choose either a constrained Docker runtime selector or a plugin-backed Sysbox backend with documented host prerequisites, threat model, config semantics, and tests.
