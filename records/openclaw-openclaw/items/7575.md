## Summary

Keep open. Current main still does not implement Sysbox/sysbox-runc, a Docker sandbox runtime selector, or Docker `--runtime` create wiring, and the live discussion treats this as a distinct sandbox runtime architecture item rather than completed Docker-socket sandbox support.

## What This Changes

The issue requests Sysbox/sysbox-runc support for Docker sandbox isolation, including host runtime setup and optional Docker runtime selection in sandbox config.

## Best Possible Solution

Keep this as the canonical Sysbox sandbox runtime design tracker until maintainers approve either a strict core Docker runtime selector or a plugin-backed Sysbox backend with host prerequisite, rollback, docs, and tests.

## Solution Assessment

Unclear: the requested direction is plausible, but adding host-level Sysbox directly to Docker deployment changes the trust boundary. Maintainers should choose between a narrow core Docker runtime selector and a Sysbox backend plugin before implementation.
