## Summary

## What This Changes

The branch feeds runtime model catalog data into native `/think` menu and autocomplete resolution for Telegram, Slack, and Discord, plus adds a cache-only model-catalog snapshot with Discord startup and gateway reload warmups.

## Best Possible Solution

Land one focused runtime-catalog fix for native `/think` choices if maintainers accept the cache-only catalog option, then close or supersede the broader provider-policy and core-fallback sibling PRs.

## Reproduction Assessment

Yes. I did not execute the live Telegram/Ollama flow in this read-only review.

## Solution Assessment

Yes, this appears to be the best fix shape: feeding native `/think` menu choices the same runtime catalog used by current-level/default thinking paths is narrower than changing Ollama's unknown-reasoning policy globally. The cache-only SDK-visible option remains the maintainer contract choice before merge.
