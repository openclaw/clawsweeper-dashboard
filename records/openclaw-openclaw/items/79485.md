## Summary

Keep open. Current main supports exact `agents.defaults.models` allowlists and broad configured/auth-backed visibility, and Discord already paginates picker rows, but there is no operator config for dynamic provider-only visibility; adding one is a feature/config contract that needs maintainer design.

## What This Changes

Request a provider-scoped dynamic model visibility policy so `/model` and `/models` can show all discovered models only for selected providers without exact per-model allowlists.

## Best Possible Solution

Define one model visibility contract that preserves exact `agents.defaults.models` precedence, optionally filters dynamic discovery by provider, and applies consistently across shared catalog resolution, Gateway `models.list`, text `/models`, Discord picker, schema/help, and docs.

## Reproduction Assessment

Not applicable as a feature/config request. Source inspection is high-confidence that current main lacks provider-scoped dynamic visibility and only supports exact model allowlists or broad configured/auth-backed catalog visibility.

## Solution Assessment

Unclear because there is no proposed patch and the config contract needs design. The suggested `agents.defaults.modelProviders` shape is plausible, but it should be evaluated against existing exact allowlists, Gateway catalog views, Discord picker behavior, and the broader account-scoped allowlist work.
