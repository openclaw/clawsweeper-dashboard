## Summary

Keep open. The audit cleanup is directionally right, but the proposed local Vertex client corrupts Anthropic SDK nullable-header containers in `prepareOptions`, which can drop helper and beta headers before the final request is built.

## What This Changes

The PR removes `@anthropic-ai/vertex-sdk`, adds direct `google-auth-library@10.6.2`, updates the lockfile, and replaces the dependency-backed Anthropic Vertex client with a local `BaseAnthropic` subclass.

## Best Possible Solution

Keep the dependency cleanup, but make the local client preserve Anthropic SDK nullable-header semantics and add focused regression coverage before merge.

## Reproduction Assessment

Yes. Passing an SDK `buildHeaders(...)` result through the PR's `prepareOptions` reproduces the issue because `mergeHeaders` enumerates `values` and `nulls` instead of preserving the contained headers.

## Solution Assessment

No. Removing the vulnerable dependency chain is the right direction, but the implementation should use SDK-compatible header merging and include coverage for the local client's default auth/header path.
