## Summary

Current main still returns `skipped: disabled` after enqueueing a main-session one-shot system event, and two open PRs explicitly claim this issue; keep the issue open until one canonical fix is reviewed and merged.

## What This Changes

Fix main-session one-shot `at` jobs so an immediate heartbeat skipped as disabled does not leave the job terminally skipped, disabled, and undeleted after its system event is queued.

## Best Possible Solution

Choose one canonical PR after proving the enqueue/consumption contract, then ensure a disabled immediate heartbeat cannot create a zombie one-shot or duplicate its system event, with focused scheduled-run and restart coverage.

## Reproduction Assessment

Yes. Current main has a high-confidence source reproduction: enqueue a main-session `systemEvent`, receive `runHeartbeatOnce` as `skipped: disabled`, then apply the skipped one-shot transition that disables instead of deleting the job; linked released-gateway proof reports the same persisted state.

## Solution Assessment

Unclear. A focused fix is appropriate, but maintainers must choose whether event enqueue is the success boundary or whether the one-shot remains retryable, and the retry option must rule out duplicate enqueue before it is the best solution.
