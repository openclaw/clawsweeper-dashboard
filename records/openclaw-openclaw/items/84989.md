## Summary

Keep open. Current main still scopes `sessions_send` through session-tree visibility and the `message` path does not define a persistent-DM fallback for isolated lanes, but the requested default behavior would change background-session send authority and needs maintainer product/security review.

## What This Changes

Request that isolated, cron, or delegated sessions can notify an agent's configured DM channel through `message` or `sessions_send`, or receive a clearer binding error.

## Best Possible Solution

Define an explicit out-of-session notification contract, likely an opt-in agent/channel route or policy, keep default session isolation safe, and improve errors to name the missing binding or policy.

## Reproduction Assessment

No full live high-confidence reproduction was run for the Telegram DM path. Source inspection does reproduce the likely guard shape: `sessions_send` is tree-scoped by default and the `message` path lacks an agent persistent-DM fallback.

## Solution Assessment

No, not as a default persistent-DM fallback. The safer solution is an explicit opt-in or route-binding rule plus clearer preflight errors, because this changes a session/security boundary.
