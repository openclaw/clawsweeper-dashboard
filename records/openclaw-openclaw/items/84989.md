## Summary

Keep open. Current main and v2026.6.8 still enforce session-scoped send visibility and do not treat persistent DM pairings as implicit recipients, so the remaining work is a security/product decision about background-session outbound authority.

## What This Changes

The issue requests that isolated cron, delegated, and subagent sessions use the agent's configured DM/channel binding for `message` and `sessions_send`, or at least emit a clearer preflight error.

## Best Possible Solution

Define a least-privilege out-of-session notification contract, likely explicit agent/channel opt-in plus clearer blocked-send errors, while preserving default session isolation.

## Solution Assessment

No for implicit default fallback. The safer direction is an explicit operator-controlled out-of-session notification contract plus clearer preflight errors, not silent persistent DM inheritance.
