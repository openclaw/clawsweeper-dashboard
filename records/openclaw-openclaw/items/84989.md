## Summary

Keep open: current main and v2026.6.11 still enforce session-scoped visibility for `sessions_send`, and `message` has no explicit out-of-session DM authority model, so the requested behavior remains a security-sensitive product decision rather than implemented behavior.

## What This Changes

The issue asks for isolated, cron, and delegated sessions to send `message` or `sessions_send` output to the agent’s configured DM/channel binding instead of being limited to the calling session’s visible channels.

## Best Possible Solution

Add a maintainer-approved explicit out-of-session outbound messaging authority with clear diagnostics and focused cron/subagent/live-channel proof, while keeping the default session-scoped behavior safe.

## Solution Assessment

No final fix shape is confirmed; explicit opt-in plus clearer errors is safer than silently making isolated sessions inherit all persistent DM bindings.
