## Summary

Keep open. Current main still scopes `sessions_send` to session-tree visibility and documents isolated cron jobs as not inheriting ambient routing; the newer comments refine the fix toward explicit opt-in and clearer errors, so this needs product/security review rather than cleanup closure.

## What This Changes

The issue asks for isolated, cron, or delegated sessions to send completion notifications through an agent's configured DM channel, or at least receive a clearer visibility error.

## Best Possible Solution

Define a least-privilege out-of-session notification contract, likely an explicit agent/channel opt-in or route binding, preserve safe default session isolation, and improve preflight errors so agents know which binding or policy blocked the send.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Partially: source inspection shows the `sessions_send` tree-visibility reject and no persistent-DM fallback, but no live Telegram or Docker repro was run in this read-only review.

## Solution Assessment

Is this the best way to solve the issue? No as an unconditional resolution-order change; the safer maintainable path is an explicit opt-in or route-binding contract plus clearer preflight errors.
