## Summary

Keep open. Current main and v2026.6.11 still enforce requester/session visibility for `sessions_send` and have no explicit out-of-session DM authority model for `message`; the related sandbox allowlist PR only overlaps the message-tool availability half and does not settle this authorization behavior.

## What This Changes

Request isolated, cron, and delegated agent sessions to notify configured DM channels through `message` or `sessions_send`, preferably with explicit out-of-session authority and clearer preflight errors.

## Best Possible Solution

Add a maintainer-approved explicit out-of-session outbound messaging authority with clear diagnostics and focused cron/subagent coverage, rather than silently inheriting every persistent DM binding.

## Reproduction Assessment

Source-level yes: current code shows the `sessions_send` visibility guard and message cross-context policy path, but I did not run the reporter's live Telegram/Docker repro.

## Solution Assessment

No final fix shape is confirmed; explicit opt-in plus clearer errors is safer than making isolated sessions silently inherit all agent DM bindings by default.
