## Summary

Keep open: current main still contains source paths that match the reported native subagent delivery failure, including orphan pruning for ended runs and no durable `resultText` propagation through `agent.wait`.

## What This Changes

The issue reports that native `sessions_spawn runtime=subagent` runs can finish successfully while the parent never receives the child output and the task run delivery remains pending.

## Best Possible Solution

Apply a source-level fix that durably carries the child final text through completion before cleanup and treats missing child session rows as normal for already-ended runs, with focused regression coverage for native subagent delivery.

## Reproduction Assessment

No live current-main smoke was run, but the issue is source-reproducible: the current orphan resolver still prunes missing child session rows without an `endedAt` guard, and the wait/announce path does not preserve final text as a wait result.

## Solution Assessment

Yes, the right direction is a source-level fix rather than dist patches: add the completed-run guard in the registry helper and make final reply text a durable, typed completion payload before cleanup can remove the child session.
