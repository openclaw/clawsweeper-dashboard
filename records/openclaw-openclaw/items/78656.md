## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/78656 on current main. Completed native subagent runs can lose final output and leave `delivery_status` pending because completion capture depends on child session history after cleanup and `resolveSubagentRunOrphanReason` treats a missing child session row as an orphan even when `entry.endedAt` is already set. Implement source-level changes only: do not edit bundled dist files; add an ended-run guard for missing session rows; propagate durable final output through the agent lifecycle/wait snapshots and subagent registry lifecycle before cleanup; preserve existing delivery semantics and idempotency. Add regression coverage for completed missing-session rows not being pruned, `resultText`/`replyText` preservation through wait normalization, and native `sessions_spawn runtime=subagent` delivery status. Add a `CHANGELOG.md` fix entry thanking @moeedahmed. Stop and ask for maintainer review if the fix would require a new config option, public API redesign, or broad delivery-policy change.
