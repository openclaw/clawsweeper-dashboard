Keep open: current main still contains the reported source-level failure paths. The merged in-process announce handoff is useful partial work, but it does not add the completed-run orphan guard or a durable child final-text payload through `agent.wait` before cleanup.

## What This Changes

The issue asks for native subagent completion output to reach the parent agent, with `delivery_status` moving to `delivered` and `agent.wait` returning the child output.

## Best Possible Solution

Apply a source-level registry/wait fix that freezes child final text before cleanup, treats missing child session rows as expected for already-ended completed runs, preserves the in-process announce handoff, and adds focused regression plus live proof.
