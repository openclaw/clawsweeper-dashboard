## What This Changes

The issue asks for native subagent completions to preserve final output and deliver it back to the parent instead of leaving task delivery pending.

## Best Possible Solution

Land one source-level fix path that preserves child final text through wait/delivery, lets already-ended missing-session runs proceed to announce cleanup instead of orphan pruning, and updates the persistence/wait regressions while coordinating with the durable-delivery thread.

## Reproduction Assessment

Yes. Source inspection on current main shows an ended restored subagent with a missing child session entry still returns missing-session-entry and is pruned before announce cleanup, while agent.wait normalization still has no final text field.

## Solution Assessment

Yes. The maintainable fix is source-level, not dist patching: preserve a frozen child result before cleanup, carry it through the wait/delivery contract, and only prune truly active orphaned runs.
