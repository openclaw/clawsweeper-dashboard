## Summary

Keep open: current main still matches the reported native subagent delivery bug. The ended-run orphan path still prunes a completed child when its session store row is gone, and the wait/result capture path still has no durable final-text propagation before cleanup.

## What This Changes

The issue reports that native `sessions_spawn runtime="subagent"` completions leave task delivery pending and return empty or timed-out output to the parent instead of delivering the child result.

## Best Possible Solution

Implement the source-level fix that freezes child final output before cleanup and treats a missing child session row as non-orphan for already-ended runs, with focused regression coverage; leave broader durable-delivery policy to the existing tracker.

## Reproduction Assessment

Yes at source level, though not with a live smoke in this review. Current main still has the ended-run false-orphan path and no durable final-text propagation through `agent.wait` or the freeze path.

## Solution Assessment

Yes. The best fix is the reporter's source-level direction, not dist patches: carry the terminal child output into the frozen completion payload before cleanup and preserve stale-unended orphan pruning without pruning completed runs.
