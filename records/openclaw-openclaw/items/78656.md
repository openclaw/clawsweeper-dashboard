## Summary

Keep open: current main still matches the reported native subagent delivery failure in source and tests, and the prior ClawSweeper source concerns remain unresolved by a merged PR or current implementation.

## What This Changes

The issue asks for native subagent completions to deliver the child output back to the parent and stop leaving completed task runs in pending delivery.

## Best Possible Solution

Apply a source-level fix that freezes the child final output before cleanup can remove the session state and treats missing child session rows as non-orphan for already-ended runs, with regression coverage for native subagent delivery.

## Reproduction Assessment

Yes, source-level: current main still sends ended runs with missing child session rows through orphan pruning, and the wait/announce contract still lacks a durable final-text payload. I did not run a live current-main subagent smoke in this read-only review.

## Solution Assessment

Yes, the right fix is source-level rather than dist patching: preserve completed child output as a typed/durable result before cleanup and add the completed-run guard to the orphan resolver path.
