## Summary

Keep open: the merged in-process announce handoff improves the active delivery path, but current main still has source paths matching this report: ended runs can be reconciled as missing-session orphans, and the registry completion flow still has no durable final-text payload from `agent.wait` before cleanup.

## What This Changes

The issue reports native subagent completion output not reaching the parent agent, leaving `task_runs.delivery_status` pending and `agent.wait` timing out with no output.

## Best Possible Solution

Apply a source-level fix that freezes child final text before cleanup and treats missing child session rows as expected for already-ended completed runs, without reverting the in-process announce handoff from the merged partial fix.

## Reproduction Assessment

Yes, by source and existing test contract rather than a live run: current main still prunes ended restored runs with missing child session rows and still lacks a durable final-text field in the registry wait path. A live E2E should be added in the fix PR to prove the full parent-delivery behavior.

## Solution Assessment

Yes, the right direction is still a source-level registry/wait fix, not dist patching. The merged in-process handoff is a useful partial improvement, but it does not remove the false-orphan path or make child final text durable before cleanup.
