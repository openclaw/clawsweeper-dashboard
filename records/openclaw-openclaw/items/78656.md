## Summary

Keep open: current main still has a source-proven completed-run orphan path, and the public completion contract says sub-agents should announce results back to the requester. Recent merged work partially improved the handoff path, but it did not add the completed-run missing-session guard requested here.

## What This Changes

The issue asks for native subagent completions to deliver the child output to the parent and mark the subagent task delivery as delivered instead of leaving `delivery_status` pending.

## Best Possible Solution

Apply a source-level fix that preserves already-ended runs with missing child session rows long enough to complete the announce cleanup path, carries durable child final text into the pending delivery payload, and updates the focused persistence and wait-result regressions.

## Reproduction Assessment

Yes, at source level: an already-ended restored run with a missing child session entry is still classified as `missing-session-entry` and pruned before announce cleanup, and the current test suite asserts that behavior. I did not run a live subagent E2E in this read-only review.

## Solution Assessment

Yes, the best fix is a narrow source-level change rather than dist patching: guard completed missing-session rows from orphan pruning, preserve frozen child output through delivery, and add regression coverage around restored completed runs and parent-visible output.
