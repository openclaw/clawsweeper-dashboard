## Summary

Keep open: current `main` still contains a source-level restore/orphan path that can prune an already-ended subagent before completion delivery, and the public `agent.wait` result path still does not carry final reply text. Recent main work added useful frozen-result and in-process announce pieces, but it does not fully satisfy the issue’s expected behavior.

## What This Changes

The issue requests a source-level fix so native subagent completion results are delivered to the parent agent and completed runs are not left pending or pruned as orphans.

## Best Possible Solution

Apply a source-level registry/wait fix that treats missing child session rows as expected for already-ended runs, preserves frozen child final text through cleanup and wait/delivery contracts, and adds focused regression plus live subagent proof.

## Reproduction Assessment

Yes, by source-level reproduction rather than a live run: an ended restored subagent run with a missing child session row still enters the orphan-prune path before announce cleanup. ## Solution Assessment

Yes, the best path is a source-level fix, not dist patching: add the completed-run guard, preserve the existing frozen-result handoff, and carry child final text through the durable wait/delivery path before cleanup can remove the child session.
