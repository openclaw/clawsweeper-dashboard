## Summary

Keep open: the merged Codex-specific fix covers the related `completed:null` notification path, and current main already freezes completion text, but the core `sessions_spawn runtime=subagent` orphan path still treats a completed run with a missing child session entry as `missing-session-entry`, which leaves the original cleanup/delivery concern unresolved.

## What This Changes

The issue asks for native `sessions_spawn` subagent completion output to be delivered to the parent instead of leaving `task_runs.delivery_status` pending when a child completes or is cleaned up.

## Best Possible Solution

Patch core subagent registry orphan reconciliation so completed runs that still need parent delivery are not pruned solely because the child session entry was cleaned up, while preserving stale active-run cleanup and adding focused regression coverage.

Do we have a high-confidence way to reproduce the issue? No live current-main repro was run, but source inspection shows the remaining orphan path still treats a completed run with a missing child session entry as `missing-session-entry`, and current tests preserve that behavior.

## Solution Assessment
