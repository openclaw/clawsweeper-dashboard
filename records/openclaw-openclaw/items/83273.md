## Summary

Keep open for human review: the branch moves Telegram debounce bypass toward the right authorization-gated path, but source inspection shows unauthorized slash stop variants can still force an immediate keyed debounce flush. Automation is also paused by the maintainer stop command on this PR.

## What This Changes

The branch gates Telegram stop debounce cancellation/null-key bypass on abort-control authorization and strengthens the unauthorized group bare-stop debounce test.

## Best Possible Solution

Make unauthorized abort controls, including `/stop` and `/stop@bot`, stay on the normal debounced path while authorized abort controls still cancel pending Telegram buffers and run immediately; add slash-variant regression coverage.

## Reproduction Assessment

Yes, at source level: on the candidate branch, unauthorized `/stop` has `shouldBypassInboundDebounce` false but remains non-debounceable through the control-command policy, causing the debouncer to flush an existing same-key buffer immediately. I did not execute the focused test in this read-only review.

## Solution Assessment

No. Auth-gating the null debounce key is the right direction for bare stop, but the fix also needs to prevent unauthorized slash stop variants from taking the debouncer's immediate keyed flush path.
