## Summary

Keep open: the PR is a plausible plugin-local feature, but the current head still has blocking correctness gaps in the capture filter and config validation, and the supplied proof does not show the changed real auto-capture behavior. Lowering the default capture threshold also needs maintainer product judgment.

## What This Changes

The PR adds a `captureMinChars` setting to memory-lancedb config, manifest metadata, runtime auto-capture filtering, and tests, with a new default of 4.

## Best Possible Solution

Keep the setting plugin-local if accepted, but resolve one min/max contract across parser, manifest, runtime, tests, and docs while preserving prompt-injection and CJK safeguards.

## Reproduction Assessment

Yes for the PR defects: source inspection of the PR head shows the stale 10-character guard and default-max validation gap. The requested `captureMinChars` option itself is a feature/config addition, not a current-main bug reproduction.

## Solution Assessment

No as submitted; the plugin-local config boundary is appropriate, but the patch must honor the resolved minimum everywhere, validate against the resolved maximum, align the range test, and provide stronger real proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P2] Honor captureMinChars below ten characters:** `extensions/memory-lancedb/index.ts:576`
  - body: The new `minChars` check is followed by the existing non-CJK `< 10` guard, so trigger-bearing English text such as `prefer X` still passes the new lower bound and then gets skipped. Replace the duplicate hard floor with the resolved minimum or remove it, and cover the sub-10 trigger case in `shouldCapture` tests.
  - confidence: 0.96
- **[P2] Validate min against the resolved max:** `extensions/memory-lancedb/config.ts:155-160`
  - body: This comparison only runs when `captureMaxChars` is explicitly set, but parsing later defaults max to 500. A config like `{ captureMinChars: 1000 }` can parse into a range where no message can satisfy both limits, so compare against the resolved max value.
  - confidence: 0.94
- **[P3] Align the expected range text:** `extensions/memory-lancedb/index.test.ts:255`
  - body: The implementation and manifest use a 1-10000 range, but this test expects `captureMinChars must be between 1 and 1000`. Make the expected message match the public contract so the test documents the intended bound.
  - confidence: 0.9
