## Summary

Keep open: current main does not implement `captureMinChars`, and the PR is still a useful feature proposal, but the submitted patch has blocking correctness issues and only mock/test proof.

## What This Changes

The PR adds a `captureMinChars` setting to the memory-lancedb config, runtime filter, manifest schema, UI hints, and schema tests while lowering the default minimum to 4.

## Best Possible Solution

Keep the setting plugin-local, honor resolved min/max limits end to end, validate against defaults, document the accepted behavior, and require real runtime proof before merge.

## Reproduction Assessment

Yes for the PR defects: source inspection of PR head shows the stale sub-10 non-CJK guard, the default-max validation gap, and the mismatched expected error string. The requested capability itself is a feature, not a broken current-main behavior with an existing contract.

## Solution Assessment

No as submitted; the plugin-local config boundary is appropriate, but the implementation must honor the resolved minimum, validate against the resolved maximum, align tests/docs, and prove a real capture path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P2] Honor captureMinChars below ten characters:** `extensions/memory-lancedb/index.ts:576`
  - body: The new `minChars` check still falls through to the unchanged later guard that rejects non-CJK text shorter than 10 characters. With the default set to 4, short English triggers can still be skipped, so replace the hard floor with the resolved minimum and cover it in `shouldCapture` tests.
  - confidence: 0.96
- **[P2] Validate against the resolved capture max:** `extensions/memory-lancedb/config.ts:155-160`
  - body: This comparison only runs when `captureMaxChars` is explicitly set, but parsing later defaults the max to 500. A config like `{ captureMinChars: 1000 }` can therefore parse into an impossible range where no text can satisfy both bounds; compare against the resolved max.
  - confidence: 0.94
- **[P2] Align the expected range error:** `extensions/memory-lancedb/index.test.ts:255`
  - body: The implementation and manifest use a 1-10000 range, but this test expects `captureMinChars must be between 1 and 1000`. The extension test suite will fail unless the intended range and expected message are consistent.
  - confidence: 0.99
