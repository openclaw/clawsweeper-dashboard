## Summary

Keep open: current main still lacks the user-facing `openclaw agent --session-key` flag and the discussion shows real downstream need, but this PR is not merge-ready because it still needs real CLI behavior proof and shared validation for malformed session keys.

## What This Changes

The PR adds `--session-key <key>` to `openclaw agent` and threads it through `AgentCliOpts` into session-key resolution.

## Best Possible Solution

Expose the flag through the CLI, but add shared pre-dispatch session-key validation, focused gateway/local fallback coverage, and redacted real CLI proof before merge.

## Reproduction Assessment

Yes for the review finding by source inspection: the PR forwards user-provided `sessionKey` into local-capable CLI execution, while current embedded validation does not reject malformed `agent:` keys that Gateway rejects. No real CLI command was run in this read-only review.

## Solution Assessment

No, not as-is. The CLI surface is narrow and matches an existing lower-level contract, but the patch should centralize validation before dispatch and include real CLI proof before merge.

## Review Findings
