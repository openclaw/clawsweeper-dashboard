## What This Changes

The PR adds a `--session-key <key>` option to `openclaw agent` and threads it into gateway session-key resolution so scripts can route turns to explicit session buckets.

## Best Possible Solution

Land a rebased branch that exposes the flag through the existing sessionKey contract, validates explicit keys once before gateway/local dispatch, documents the selector, and includes focused tests plus redacted real CLI proof.

## Reproduction Assessment

Yes for the blocking review finding. Source inspection shows gateway mode rejects malformed agent-prefixed session keys while embedded local/fallback execution would accept the newly exposed `opts.sessionKey` without the same shape check.

## Solution Assessment

No, not as-is. The CLI flag fits an existing lower-level contract, but the maintainable solution validates explicit keys once before gateway/local dispatch and documents the new public selector.
