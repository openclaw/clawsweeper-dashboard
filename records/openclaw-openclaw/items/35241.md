## Summary

Keep this PR open: current main still lacks the `openclaw agent --session-key` CLI surface and downstream discussion confirms a real integration need, but the branch is not merge-ready because it lacks real behavior proof and exposes a gateway/local validation mismatch for malformed session keys.

## What This Changes

The PR adds a `--session-key <key>` option to `openclaw agent` and threads it through `AgentCliOpts` into gateway session-key resolution.

## Best Possible Solution

Expose the flag through the CLI, but add shared pre-dispatch session-key validation, focused gateway/local fallback coverage, and redacted real CLI proof before merge.

## Reproduction Assessment

Yes for the review finding by source inspection: the diff forwards user-provided `sessionKey` into local-capable CLI execution, while current embedded validation does not reject malformed `agent:` keys that Gateway rejects. No real CLI run was performed in this read-only review.

## Solution Assessment

No, not as-is. The requested CLI surface is narrow and consistent with the existing lower-level contract, but the patch should centralize validation before dispatch and include real CLI proof before merge.
