## Summary

Keep open: current main still lacks the `openclaw agent --session-key` CLI selector, and downstream comments describe a real integration need, but the PR needs real behavior proof plus validation/docs fixes before merge.

## What This Changes

The branch adds `--session-key <key>` to `openclaw agent` and threads it through `AgentCliOpts` into session-key resolution.

## Best Possible Solution

Expose the selector through CLI, Gateway, and embedded paths with shared session-key shape and agent-mismatch validation, focused parsing/gateway/local tests, docs/help coverage, and maintainer changelog at landing.

## Reproduction Assessment

Not applicable as a feature PR. Source inspection shows current main lacks the CLI flag while the resolver and Gateway protocol already accept `sessionKey`.

## Solution Assessment

No, not as currently proposed. The direction is narrow, but the best fix validates malformed canonical keys before both Gateway and embedded execution and documents/tests the new selector.
