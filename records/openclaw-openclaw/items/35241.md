## Summary

Keep this PR open, but do not merge it as-is: the requested CLI flag is useful and matches existing protocol plumbing, yet the patch exposes raw session keys without preserving the gateway/local validation contract and omits the required docs/changelog coverage.

## What This Changes

This PR adds a `--session-key <key>` option to `openclaw agent` and threads it through CLI option typing and gateway session resolution.

## Best Possible Solution

Land the CLI flag with one shared pre-dispatch validation path, focused regression tests for gateway and embedded fallback behavior, and aligned CLI docs plus changelog.

## Reproduction Assessment

Not applicable as a feature PR. Source inspection gives a high-confidence behavior check: current main lacks the CLI flag, while the gateway protocol and embedded command path already carry `sessionKey`.

## Solution Assessment

No, not as currently proposed. Surfacing the existing `sessionKey` field is the right direction, but the PR should validate malformed `agent:` keys before both gateway and embedded execution and include docs, changelog, and regression coverage.
