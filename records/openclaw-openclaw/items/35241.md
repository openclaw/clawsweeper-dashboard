## Summary

Keep open. Current main still lacks the requested CLI selector and downstream discussion supports the use case, but this PR is not merge-ready because it exposes raw session-key routing without shared pre-dispatch validation, docs, and real CLI proof.

## What This Changes

The PR adds a `--session-key <key>` option to `openclaw agent` and forwards it through `AgentCliOpts` into session resolution for Gateway and embedded fallback paths.

## Best Possible Solution

Land a rebased version that adds the flag with shared session-key shape and agent-mismatch validation before Gateway dispatch, `--local`, and embedded fallback, plus CLI/docs coverage and real CLI proof; keep `--no-fallback` as a separate follow-up.

## Reproduction Assessment

Not applicable as a feature PR. Source inspection gives a high-confidence baseline: current main does not expose `--session-key` on `openclaw agent`, while the resolver and Gateway protocol already accept explicit session keys.

## Solution Assessment

No, not as-is. The direction is narrow and consistent with existing lower-level contracts, but the mergeable solution should validate explicit keys consistently before Gateway dispatch, `--local`, and embedded fallback, then document and prove the new selector.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Validate session keys before local fallback:** `src/commands/agent-via-gateway.ts:118`
  - body: The new option lets raw `opts.sessionKey` reach both Gateway dispatch and embedded execution, but the embedded path reached by `--local` or transport fallback only checks agent-id mismatch and does not reject malformed `agent:` keys. Gateway rejects those keys, so the same CLI input can be rejected in normal Gateway mode but create or use an invalid local session when fallback runs. Move shared shape and agent-mismatch validation before dispatch/fallback and cover both paths.
  - confidence: 0.88
- **[P3] Document the session-key selector:** `src/cli/program/register.agent.ts:30`
  - body: This adds a public session selector, but the CLI examples and docs still enumerate only `--to`, `--session-id`, and `--agent`. Document accepted key shapes, precedence, and that the value is a session identifier rather than a secret so integrations do not have to infer behavior from source.
  - confidence: 0.82
