## Summary

Keep open: current main still lacks the `openclaw agent --session-key` CLI selector, and the PR remains useful, but the patch is not merge-ready because it exposes user-provided session keys without shared local/fallback validation and has no after-fix real CLI proof.

## What This Changes

The PR adds a `--session-key <key>` selector to `openclaw agent` and threads it through CLI session-key resolution for gateway and local execution paths.

## Best Possible Solution

Refresh the PR to expose the CLI selector with shared malformed-key and agent-mismatch validation before gateway/local dispatch, docs/help coverage, focused tests, and redacted real CLI proof.

## Reproduction Assessment

Not applicable as a feature PR. Source inspection gives a high-confidence current-main baseline: `openclaw agent` does not expose `--session-key`, while the resolver and Gateway protocol already accept `sessionKey`.

## Solution Assessment

No, not as currently proposed. The direction is narrow, but the mergeable solution should validate explicit keys consistently for Gateway, `--local`, and embedded fallback paths, then document and prove the new selector.
