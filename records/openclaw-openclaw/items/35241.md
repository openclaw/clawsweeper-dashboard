## Summary

Keep open. Current main still does not expose `openclaw agent --session-key`, and the PR is aimed at a real CLI gap because lower-level agent and Gateway paths already understand `sessionKey`; however, the patch should not merge as-is because it exposes malformed agent-prefixed keys to the embedded fallback path without the Gateway validation.

## What This Changes

The PR adds `openclaw agent --session-key <key>`, extends `AgentCliOpts`, treats it as a valid session selector, and passes it into session-key resolution for Gateway requests and local fallback.

## Best Possible Solution

Refresh this as a small CLI surface patch: register `--session-key <key>`, add it to `AgentCliOpts`, pass it through Gateway and local embedded paths, add shared pre-dispatch validation for malformed `agent:` keys and agent mismatches, cover Gateway/local/fallback behavior with focused tests, and update CLI docs/help. Keep the requested `--no-fallback` follow-up separate.

## Reproduction Assessment

Yes. Static inspection of current main is enough to reproduce the missing surface: `openclaw agent` registration and docs do not list `--session-key`, while the embedded command type, resolver, Gateway schema, and Gateway handler already support `sessionKey`.

## Solution Assessment

No, not as proposed. The direction is right, but the implementation should validate user-provided session keys before Gateway dispatch and embedded fallback, then add focused tests and docs so the new public flag behaves consistently.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Validate session keys before gateway fallback:** `src/commands/agent-via-gateway.ts:155`
  - body: This line makes user-provided `--session-key` feed session resolution and the Gateway request, but malformed `agent:` keys are only rejected by the Gateway. `agentCliCommand` catches Gateway failures and falls back to `agentCommand`, whose local path accepts `sessionKey` without the same shape validation, so `openclaw agent --session-key agent:main ...` can create or use an invalid embedded session instead of returning the validation error. Add shared pre-dispatch validation and a fallback test before exposing the flag.
  - confidence: 0.87
