## What This Changes

The PR adds a `--session-key <key>` option to `openclaw agent` and passes it into Gateway CLI session resolution.

## Best Possible Solution

Land the selector after adding shared session-key validation, focused CLI/Gateway/local fallback tests, docs/help coverage, and redacted real CLI proof; keep `--no-fallback` as a separate follow-up.

## Reproduction Assessment

Not applicable as a feature PR. Source inspection confirms current main lacks the CLI flag while lower-level Gateway and resolver surfaces already accept `sessionKey`.

## Solution Assessment

No, not as currently proposed. The direction is narrow, but the best fix validates malformed canonical keys before both Gateway and embedded execution and documents/tests the new selector.
