## Summary

Keep open: the request is still useful and not implemented on current main, but this branch should not merge until malformed-key validation, docs, and real CLI proof are addressed.

## What This Changes

The PR adds a `--session-key <key>` selector to `openclaw agent` and threads it into gateway and embedded session-key resolution.

## Best Possible Solution

Expose the CLI selector through the existing resolver/protocol, with shared malformed-key validation, focused tests, CLI docs/help coverage, real CLI proof, and a maintainer-owned changelog at landing.

## Reproduction Assessment

Not applicable as a feature PR. Source inspection confirms current main lacks the CLI flag while the lower-level resolver and Gateway protocol already accept `sessionKey`.

## Solution Assessment

No, not as currently proposed. The direction is narrow, but the maintainable fix validates malformed canonical keys before both Gateway and embedded execution and documents/tests the new selector.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Validate session keys before local fallback:** `src/commands/agent-via-gateway.ts:118`
  - body: Adding `--session-key` makes arbitrary keys user-reachable, but malformed canonical keys like `agent:main` are only rejected by the Gateway. If `--local` is used or the Gateway transport fails, the embedded path receives the raw key and can create or reuse a malformed session, so add shared pre-dispatch validation while preserving custom namespaces such as `agentgram:test-room`.
  - confidence: 0.88
