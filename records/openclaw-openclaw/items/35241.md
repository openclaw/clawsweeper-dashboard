## Summary

Keep open: current main still lacks the user-facing flag, and downstream discussion supports the session-isolation use case, but the proposed patch is not merge-ready because it exposes raw session keys without shared validation and has no real CLI behavior proof.

## What This Changes

The PR adds an `openclaw agent --session-key <key>` CLI option and threads that value into gateway/local agent session routing.

## Best Possible Solution

Add the flag with shared pre-dispatch session-key shape and agent-mismatch validation, cover both gateway and embedded paths with focused tests, and require redacted real CLI proof before merge.

## Reproduction Assessment

Yes for the review finding by source inspection: current main shows Gateway validation for malformed and mismatched explicit keys while the embedded path lacks the malformed-key check. I did not run a live CLI command in this read-only review.

## Solution Assessment

No, not as-is. The flag fits the existing lower-level contract, but the maintainable solution validates explicit keys once before gateway/local dispatch and proves the real CLI path after the patch.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Validate explicit key shape and agent mismatch before dispatching to either path.
