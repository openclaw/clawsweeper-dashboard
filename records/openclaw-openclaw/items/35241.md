Keep open. Current main still lacks the requested CLI flag, and downstream comments describe valid automation use cases, but the PR needs validation, docs/tests, and real behavior proof before it should merge.

## What This Changes

The PR adds a `--session-key <key>` option to `openclaw agent` and forwards it through the CLI gateway session resolver.

## Best Possible Solution

Land the CLI flag as an additive feature with shared pre-dispatch session-key validation, focused gateway/local tests, docs, and maintainer-owned changelog before release.

## Reproduction Assessment

Not applicable as a feature PR. Source inspection gives a high-confidence behavior check: current main lacks the CLI flag while the Gateway protocol and resolver already support `sessionKey`.

## Solution Assessment

No, not as currently proposed. The direction is right, but the patch should validate malformed `agent:` keys before gateway/local dispatch and document/test both gateway and embedded fallback behavior.

- **[P2] Validate session keys before embedded fallback:** `src/commands/agent-via-gateway.ts:118-120`
  - body: Adding this flag makes raw session keys user-reachable. Validate `--session-key` once before dispatch or fallback, while preserving custom namespaces like `agentgram:test-room`.
  - confidence: 0.88
