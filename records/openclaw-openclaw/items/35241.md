## What This Changes

The PR adds a `--session-key <key>` option to `openclaw agent` and threads it through `AgentCliOpts` into gateway session-key resolution.

## Best Possible Solution

Land the CLI flag as an additive feature with shared pre-dispatch session-key validation, focused gateway/local tests, CLI docs, and maintainer-owned changelog before release.

## Reproduction Assessment

Not applicable as a feature PR. Source inspection gives a high-confidence behavior check: current main lacks the CLI flag while the Gateway protocol and session resolver already support `sessionKey`.

## Solution Assessment

No, not as currently proposed. The direction is right, but the patch should validate malformed `agent:` keys before gateway/local fallback, preserve custom namespaces, add focused tests, and document the new CLI option.

- **[P2] Validate session keys before embedded fallback:** `src/commands/agent-via-gateway.ts:118-120`
  - body: Adding this flag makes raw session keys user-reachable.
