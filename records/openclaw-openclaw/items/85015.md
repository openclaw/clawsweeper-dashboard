## What This Changes

The issue reports that the Skills configuration wizard counts eligible skills but does not list them or show an empty-state hint when no skill dependencies are missing.

## Best Possible Solution

Land one narrow empty-state fix in `setupSkills` with focused regression coverage and CLI wording that points users to the existing skills inspection commands.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `openclaw config --section skills` reaches `setupSkills`, and an all-eligible report makes `installable` and `missing` empty so no post-confirmation prompt or empty-state note appears.

## Solution Assessment

Yes for the minimal fix. A note using the existing `openclaw skills list --verbose` and `openclaw skills check` surfaces is the narrowest maintainable solution; a full enable/disable wizard would be separate product work.
