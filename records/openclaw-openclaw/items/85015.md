## What This Changes

The issue requests that the Skills config wizard show an eligible-skill list, an enable/disable path, or at least a clear empty-state note when there are no missing skill requirements to install.

## Best Possible Solution

Land the focused empty-state note in setupSkills and keep the larger eligible-skill enable/disable UI as separate product work if maintainers want it later.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `openclaw config --section skills` reaches setupSkills, and an all-ready report leaves baseInstallable empty, skips the multiselect, skips missing-env prompts, and returns without an explanatory note.

## Solution Assessment

Yes for the minimal fix. A small empty-state note that points to `openclaw skills list --verbose` is the narrowest maintainable repair; a full enable/disable skills UI should remain a separate feature decision.
