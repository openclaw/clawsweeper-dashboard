## What This Changes

Request to enable tool-loop detection by default and add follow-up guards/diagnostics after a Telegram/vLLM agent loop continued across auto-compaction.

## Best Possible Solution

Land a maintainer-approved narrow reliability change that preserves progress-aware semantics, updates schema/docs/tests/changelog, and treats post-compaction guarding or disabled-detector diagnostics as separately reviewable follow-ups.
