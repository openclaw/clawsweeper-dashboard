Full review comments:

- **[P1] Gate profile loading to live test runs:** `test/test-env.ts:338`
  - body: `installTestEnv` calls `loadProfileEnv(realHome)` unconditionally, so ordinary non-live test runs can import provider keys, config paths, and host-specific variables from the developer's real profile. Keep profile loading behind live/real-home mode or an explicit option so isolated tests stay deterministic.
  - confidence: 0.93
- **[P2] Snapshot live env after loading the profile:** `test/test-env.ts:336-348`
  - body: `liveEnvSnapshot` is captured before the profile is loaded, but live staging later uses that snapshot to resolve `OPENCLAW_CONFIG_PATH` and `OPENCLAW_STATE_DIR`. If those values only come from the profile, staging can copy the wrong config or auth state.
  - confidence: 0.88
