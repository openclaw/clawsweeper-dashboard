- **[P1] Gate profile loading to live test runs:** `test/test-env.ts:338`
  - body: `installTestEnv` now calls `loadProfileEnv(realHome)` before checking whether this is a live or real-HOME run. Gate this behind live/real-home mode or an explicit test option.
  - confidence: 0.92
- **[P2] Capture live env after loading the profile:** `test/test-env.ts:336-348`
  - body: `liveEnvSnapshot` is captured before `loadProfileEnv(realHome)`, but `stageLiveTestState` later uses that snapshot for `OPENCLAW_CONFIG_PATH` and `OPENCLAW_STATE_DIR`. If those are exported only from `.profile`, live staging copies the default state instead of the operator-selected config/auth state.
  - confidence: 0.9
