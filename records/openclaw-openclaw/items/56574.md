## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Gate profile loading to live test runs:** `test/test-env.ts:338`
  - body: installTestEnv now calls loadProfileEnv(realHome) unconditionally, so normal non-live runs can import provider keys, config paths, and host-specific variables from the developer's real .profile. Keep this behind live/real-home mode or an explicit option so isolated tests stay deterministic.
  - confidence: 0.93
- **[P2] Snapshot live env after loading the profile:** `test/test-env.ts:336-348`
  - body: liveEnvSnapshot is captured before .profile is loaded, but stageLiveTestState later uses that snapshot to resolve OPENCLAW_CONFIG_PATH and OPENCLAW_STATE_DIR. If those values only come from .profile, live staging copies the wrong config/auth state.
  - confidence: 0.88

## Security Review
