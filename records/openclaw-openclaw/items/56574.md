## Summary

Keep open: current main still lacks the registry-level tool dedupe and pluginConfig tool-factory threading, and this PR is useful, but PR head introduces live-test profile/config staging regressions that should block merge until the branch is rebased or split.

## What This Changes

The PR changes plugin registry/loading/tool resolution to dedupe same-plugin tool registrations, pass per-plugin config to tool factories, reuse gateway-bindable registries for default requests, and update live-test HOME/profile staging docs/tests.

## Best Possible Solution

Land a refreshed or split change that keeps exact same-plugin registration dedupe and pluginConfig threading with focused regression tests, preserves the newer current-main gateway compatibility behavior, fixes live-test profile staging, and adds the required changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: PR head calls loadProfileEnv before live gating and snapshots env before profile loading, while current main still unconditionally appends plugin tool registrations and calls factories without pluginConfig.

## Solution Assessment

No as currently proposed. The plugin registry/tool-context boundary is the right direction, but mixing it with credential-sensitive live-test staging and omitting the changelog makes this PR head unsuitable to merge as-is.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Restrict profile env loading to live runs:** `test/test-env.ts:338`
  - body: `installTestEnv` now calls `loadProfileEnv(realHome)` before checking whether this is a live or real-HOME run. That imports missing variables from the developer's real `.profile`, including provider keys or config paths, into normal isolated tests and makes the default test lane depend on host state. Gate this behind live/real-home mode or an explicit test option.
  - confidence: 0.92
- **[P2] Snapshot env after loading the profile:** `test/test-env.ts:336-348`
  - body: `liveEnvSnapshot` is captured before `loadProfileEnv(realHome)`, but `stageLiveTestState` later uses that snapshot for `OPENCLAW_CONFIG_PATH` and `OPENCLAW_STATE_DIR`. If those are exported only from `.profile`, live staging copies the default state instead of the operator-selected config/auth state. Capture the snapshot after profile loading or pass the post-load env values to staging.
  - confidence: 0.9
- **[P3] Add the required changelog entry:** `src/plugins/registry.ts:313-340`
  - body: This PR changes user-visible plugin tool behavior by deduping registrations and exposing per-plugin config to tool factories, but it does not add a `CHANGELOG.md` entry. Repo policy requires user-facing fix changes to be recorded, so add one concise entry for the plugin tool fix.
  - confidence: 0.86
