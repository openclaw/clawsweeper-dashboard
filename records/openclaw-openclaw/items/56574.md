## Summary

Keep open. Current main still lacks the pluginConfig handoff and exact duplicate tool-registration guard, but this PR is conflicting/stale, lacks after-fix real behavior proof, and still carries live-test profile staging defects that should block merge until the branch is refreshed or split.

## What This Changes

The PR changes plugin registry/tool resolution to dedupe exact per-plugin tool registrations, pass pluginConfig into tool factory context, relax gateway-bindable registry reuse, and update live-test home/profile staging docs/tests.

## Best Possible Solution

Refresh or split the branch so the focused plugin registry/tool-context fixes land against current plugin contracts, while live-test profile staging is repaired separately or dropped and real behavior proof is added.

## Reproduction Assessment

Yes, source-level only. A plugin that registers the same tool twice will still append duplicate registry entries on current main, and a tool factory that reads ctx.pluginConfig will receive a context without that field.

## Solution Assessment

No. The plugin fix direction is useful, but this PR is not the narrowest maintainable merge candidate while it is stale/conflicting and mixes registry fixes with live-test profile staging changes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Gate profile loading to live test runs:** `test/test-env.ts:338`
  - body: `installTestEnv` calls `loadProfileEnv(realHome)` unconditionally in the PR head, so ordinary non-live test runs can import provider keys, config paths, and host-specific variables from the developer's real profile. Keep profile loading behind live/real-home mode or an explicit option so isolated tests stay deterministic.
  - confidence: 0.93
- **[P2] Snapshot live env after loading the profile:** `test/test-env.ts:336-348`
  - body: `liveEnvSnapshot` is captured before the profile is loaded, but live staging later uses that snapshot to resolve `OPENCLAW_CONFIG_PATH` and `OPENCLAW_STATE_DIR`. If those values only come from the profile, staging can copy the wrong config or auth state.
  - confidence: 0.88
