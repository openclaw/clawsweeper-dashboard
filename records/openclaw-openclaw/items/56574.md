## Summary

Keep open, but this PR should not merge as-is: current main still lacks the registry-side pluginConfig/tool-dedupe pieces, while the gateway-bindable reuse part is already present and the PR head has a live-test profile-loading regression plus no real behavior proof.

## What This Changes

The PR adds same-plugin tool registration dedupe, passes plugin config into plugin tool factories, reuses gateway-bindable registries for default requests, and updates live-test home/profile staging docs and tests.

## Best Possible Solution

Refresh or split the patch so current main only receives the still-missing registry/tool-context fixes with focused tests, preserves the live-test profile gate, and includes redacted real-plugin or live-run proof.

Yes for source-level reproduction: current main lacks pluginConfig in the tool factory path, and PR head visibly loads .profile before any live-mode gate. I did not run tests because this review is read-only.

## Solution Assessment

No. The registry/tool-context direction is plausible, but this branch is not the narrowest maintainable solution as-is because it mixes useful plugin fixes with stale loader changes and live-test profile regressions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Gate profile loading to live test runs:** `test/test-env.ts:338`
  - body: installTestEnv now calls loadProfileEnv(realHome) unconditionally, so normal non-live test runs can import provider keys, config paths, and host-specific variables from the developer's real .profile. Keep this behind live/real-home mode or an explicit option so isolated tests stay deterministic.
  - confidence: 0.93
- **[P2] Snapshot live env after loading the profile:** `test/test-env.ts:336-348`
  - body: liveEnvSnapshot is captured before .profile is loaded, but stageLiveTestState later uses that snapshot to resolve OPENCLAW_CONFIG_PATH and OPENCLAW_STATE_DIR. If those values only come from .profile, live staging copies the wrong config/auth state.
  - confidence: 0.88
