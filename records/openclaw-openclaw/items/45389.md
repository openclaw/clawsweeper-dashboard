## Summary

Keep open. Current main still has adjacent timeout mitigations, cron backoff, and an Active Memory-only timeout breaker, but it does not implement the requested durable session-level LLM timeout circuit breaker with default/per-agent configuration, persisted session failure state, degraded or paused session handling, and operator notification. The open PR #42933 is the closest implementation candidate and remains unmerged.

## What This Changes

Add a configurable session-level circuit breaker that stops automatic retries after consecutive LLM timeouts, marks the affected session degraded, and emits an operator notification.

## Best Possible Solution

Keep the feature request open and consolidate it with the active breaker work. The desired end state is a documented core agent/session breaker with default and per-agent configuration, persisted per-session timeout/error streak state across heartbeat, cron, auto-resume, and interactive runs, success/manual reset clearing the streak, a clear degraded or paused state, and sanitized operator events or webhooks.

## Reproduction Assessment

Not applicable as a normal bug reproduction because this is a feature request. The high-confidence check is source inspection: current main lacks the requested config fields, durable session state, and heartbeat/cron/session enforcement paths, while existing timeout tests cover only in-run compaction fallthrough.

## Solution Assessment

Unclear for the exact proposed config shape. A persisted configurable session-level breaker is the right direction, but maintainers should decide pause/reset/alert semantics and review or rework PR #42933 rather than treating current timeout compaction, cron cleanup, or Active Memory recall guards as sufficient.
