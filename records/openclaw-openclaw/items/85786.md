## Summary

Keep open. Current main and the latest release still render failed tool payloads through success-oriented tool-output UI, while this PR adds focused UI handling, regression tests, and sufficient live-output proof. I found no blocking correctness or security finding, but the fallback JSON error heuristic is a compatibility choice for maintainers before merge.

## What This Changes

The branch adds tool-result error detection and error styling across chat tool cards, grouped standalone summaries, sidebar copy, and focused UI tests.

## Best Possible Solution

Land the explicit error-state UI propagation with focused regression coverage after maintainers either accept the bounded error-field fallback or narrow it to known failure shapes.

## Reproduction Assessment

Yes. Source inspection of current main and v2026.5.22 shows tool-result error flags are not propagated into ToolCard and the grouped/sidebar renderers use Tool output success copy for error-shaped payloads; I did not run a UI repro in this read-only pass.

## Solution Assessment

Yes, mostly. Propagating explicit isError/is_error through the UI renderer is the narrow maintainable fix; the fallback JSON heuristic is bounded but should be explicitly accepted or narrowed by maintainers.
