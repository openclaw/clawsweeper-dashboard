## Summary

Close as implemented. The failing surface described in the report matches Slack App Home/assistant DM `message_changed` events, and current main now rehydrates those events into normal inbound Slack messages while preserving the account-scoped Slack routing path. The fix is included in shipped release v2026.4.29.

## What This Changes

Request to make inbound DM replies from a secondary Slack Socket Mode workspace reach OpenClaw and route to the correct account-specific conversation.

## Best Possible Solution

Keep the Slack-specific fix in the Slack plugin: assistant/App Home DM event normalization and account-aware routing belong in the channel implementation, with core staying on the generic route/session contracts.

## Solution Assessment
