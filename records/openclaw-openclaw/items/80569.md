## Summary

Keep open: current main and the latest shipped release still leave the heartbeat fallback `notify=false` marker as ordinary text, while https://github.com/openclaw/openclaw/pull/80737 is the open closing PR that owns the fix path.

## What This Changes

Telegram heartbeat/completion fallback replies should consume a trailing standalone `notify=false` control marker as silent delivery metadata instead of sending it visibly.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/80737 or an equivalent maintainer branch that strips only standalone trailing `notify=false`, forwards `silent: true`, suppresses marker-only fallback replies, preserves inline text, and covers heartbeat plus exec-completion fallback tests.

Yes, source-level: current main's heartbeat fallback normalization never consumes a trailing `notify=false`, and the durable send call does not pass `silent`. I did not run a live Telegram current-main reproduction in this read-only review.
