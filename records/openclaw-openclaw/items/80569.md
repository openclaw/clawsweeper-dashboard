## Summary

Keep open: current main still has the reported source-level gap, and an open linked fix PR is already the implementation path. The issue should stay open until that PR or another canonical fix lands and closes it.

## What This Changes

The issue asks OpenClaw to stop leaking trailing Telegram heartbeat/completion `notify=false` control markers into message text and to apply silent Telegram delivery metadata instead.

## Best Possible Solution

Land one canonical heartbeat fix that strips only a standalone trailing `notify=false`, suppresses marker-only replies, forwards `silent: true` through durable Telegram delivery, and preserves inline user text.

## Reproduction Assessment

No high-confidence live Telegram reproduction was run in this review. The source path is reproducible from current main: heartbeat prompts can produce `notify=false`, fallback normalization does not consume it, and the durable send omits `silent`.

## Solution Assessment

Yes, the narrow heartbeat-fallback conversion is the best fit. It preserves structured `heartbeat_respond` as canonical and avoids a broad Telegram/body parser that could strip legitimate user text.
