## Summary

Keep open: current main still has a source-reproducible Telegram/heartbeat delivery gap, and the linked open fix PR is draft with review evidence that it only partially covers the reported durable delivery path.

## What This Changes

The issue asks OpenClaw to convert trailing standalone Telegram `notify=false` reply markers into silent delivery metadata and remove the marker from visible message text.

## Best Possible Solution

Land one narrow fix at the durable/Telegram outbound boundary that strips only standalone trailing `notify=false`, preserves inline text, carries silent metadata through text and media sends, and proves the behavior with redacted real Telegram output.

## Reproduction Assessment

Yes at source level: current main preserves fallback reply text, the heartbeat durable send does not pass `silent`, and Telegram only applies `disable_notification` when structured `silent` metadata is present. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

Yes for the requested issue direction, but not for the current linked draft PR. The maintainable fix should normalize the marker at the heartbeat/shared durable outbound boundary and reuse the existing Telegram `silent` send contract, not only patch one native reply path.
