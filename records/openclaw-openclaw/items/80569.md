## Summary

Keep open: current main still has the heartbeat fallback leak, and the open closing PR is the active implementation path for the fix.

## What This Changes

The issue requests that Telegram heartbeat/completion replies consume trailing `notify=false` as silent-delivery metadata instead of showing it in the message body.

## Best Possible Solution

Review, repair, and land https://github.com/openclaw/openclaw/pull/80737 or an equivalent maintainer branch that strips only standalone trailing markers, forwards `silent: true`, suppresses marker-only replies, preserves inline text, and covers heartbeat plus exec fallback tests.

## Reproduction Assessment

Yes. Source inspection shows a heartbeat fallback reply ending in `notify=false` would pass through `normalizeHeartbeatReply` as visible text and then be sent without `silent` metadata on current main.

## Solution Assessment

Yes, the narrow maintainable direction is to use the existing heartbeat fallback and outbound `silent` metadata boundary rather than adding Telegram-specific policy. The linked PR should be reviewed and tightened before this issue closes.
