## Summary

Keep open: current main still leaves the trailing fallback marker as normal text and sends the heartbeat payload without silent delivery metadata, while https://github.com/openclaw/openclaw/pull/80737 is the open closing PR for the canonical fix path.

## What This Changes

The issue reports that Telegram heartbeat/completion replies can leak a trailing `notify=false` marker into the visible message instead of converting it into silent delivery metadata.

## Best Possible Solution

Land one canonical fix through https://github.com/openclaw/openclaw/pull/80737 or an equivalent branch that strips only standalone trailing markers, forwards `silent: true`, suppresses marker-only replies, preserves inline text, and covers heartbeat plus exec-completion fallback tests.

## Reproduction Assessment

Yes. Current main’s heartbeat fallback normalizer leaves a trailing standalone `notify=false` in `normalized.text`, and the durable send call does not pass a silent flag, so the marker would be delivered as normal text.

## Solution Assessment

Yes. The narrow maintainable fix is to keep structured `heartbeat_respond` canonical while defensively converting only the legacy standalone trailing marker in the affected fallback path.
