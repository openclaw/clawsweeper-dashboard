## Summary

Keep open. Current main and the latest shipped tag still pin Pi 0.74.0 with the optional native clipboard package in the dependency graph, and gateway/embedded-agent startup still statically imports the Pi root package that evaluates the upstream clipboard loader path.

## What This Changes

The issue asks OpenClaw to prevent gateway startup from crashing through the Pi runtime's Linux native clipboard addon.

## Best Possible Solution

Keep the issue open until gateway/background startup no longer evaluates Pi's native clipboard loader unless clipboard functionality is actually needed, preferably via an import boundary, lazy load, or upstream Pi update.

## Reproduction Assessment

Yes for source reproduction: current main still has gateway and embedded-runner static imports from the Pi root package, and Pi 0.74.0 still reaches clipboard-native.js at root-module evaluation. I did not run a live Linux crash repro, but two reporters supplied coredump/workaround evidence.

## Solution Assessment

Yes: the narrow maintainable fix is to keep native clipboard loading out of gateway/background startup through a Pi import boundary, lazy load, or upstream Pi update. The TERMUX_VERSION workaround is useful evidence but should not become the OpenClaw contract.
