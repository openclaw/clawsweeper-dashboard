## Summary

Keep open. Current main still has the crash-prone dependency path: OpenClaw pins Pi 0.74.0, Pi's root export still reaches clipboard code, the published clipboard-native module still requires `@mariozechner/clipboard` on Linux display sessions, and gateway/agent startup paths still statically import the Pi root package.

## What This Changes

The issue requests preventing OpenClaw Gateway startup from loading and crashing in the native `@mariozechner` clipboard addon on Linux gateway/background sessions.

## Best Possible Solution

Add an OpenClaw-owned Pi import/runtime boundary that prevents native clipboard loading during gateway/background startup, with focused regression coverage and without making `TERMUX_VERSION` the supported public workaround.

## Reproduction Assessment

Yes for source reproduction: current main still imports the Pi root package in gateway/agent startup paths, and the exact Pi 0.74.0 package still evaluates the native clipboard require on Linux display sessions. No live crash reproduction was run in this review.

## Solution Assessment

Yes, a narrow import boundary or lazy clipboard guard is the best maintainable direction. Renaming the `.node` file or co-opting `TERMUX_VERSION` is useful workaround evidence but should not become the OpenClaw contract.
