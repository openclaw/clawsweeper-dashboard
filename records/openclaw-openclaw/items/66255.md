Prevent OpenClaw Gateway startup from loading or crashing through the Pi runtime's native @mariozechner clipboard addon on Linux.

## Best Possible Solution

Move Pi root-package evaluation behind a safe lazy/import boundary or consume an upstream Pi fix so gateway/background startup never loads native clipboard unless clipboard functionality is actually needed.
