## Best Possible Solution

Add a Feishu-owned pacing helper or scheduler used by all message create/reply-style delivery paths, keep the default compatible, and expose the config consistently in schema, manifest, docs, and tests.

## Solution Assessment

No. This is a plausible chunk-only mitigation, but the best fix is a shared Feishu outbound pacing layer plus aligned manifest, docs, and tests so the new config means what users expect.
