## Summary

Keep open: current main and v2026.6.8 still only support light/REM/deep text-memory dreaming, while this asks for a new parametric LoRA/fine-tuning phase with checkpointing and rollback that needs product, API, and security design.

## What This Changes

Add an optional post-deep-sleep consolidation phase that distills dreaming/session evidence into LoRA or provider fine-tuning checkpoints with regression guards and rollback.

## Best Possible Solution

Define a maintainer-approved parametric consolidation design first, with a memory/plugin/provider boundary, default-off safety gates, source exclusions, dry-run review, evaluation, checkpoint activation, rollback, docs, and focused tests.
