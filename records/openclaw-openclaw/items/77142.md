## Summary

Keep open: current main has only the documented light/REM/deep text-memory dreaming pipeline and no LoRA/fine-tuning support, while the requested parametric training phase is a broad opt-in model-training feature that needs maintainer product, security, and API direction.

## What This Changes

Request adds an optional post-deep dreaming consolidation phase that distills memory artifacts into LoRA/provider fine-tuning checkpoints with regression guards and rollback config.

## Best Possible Solution

Define parametric consolidation as an opt-in design first, with explicit data-provenance, consent, checkpoint activation, rollback, and plugin/API boundaries before implementation.

## Reproduction Assessment

Not applicable; this is a feature request, not a report of broken existing behavior. Source and docs inspection confirms current main intentionally exposes the light/REM/deep text-memory pipeline only.

## Solution Assessment

Unclear; adding a fourth phase directly to memory-core is not yet demonstrably the best path. A ClawHub/code-plugin implementation or a small SDK seam may be safer if maintainers want this capability outside bundled core.
