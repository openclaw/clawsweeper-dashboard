## Summary

Keep open: current main does not implement parametric consolidation, and the request is a broad new memory/provider capability that needs maintainer design and security review before code work.

## What This Changes

Add an optional post-deep dreaming phase that distills memory/session experience into LoRA or provider fine-tuning checkpoints with safety guards.

## Best Possible Solution

Define a maintainer-reviewed design that either adds a stable memory-consolidation plugin API or an opt-in memory-core phase with privacy, evaluation, provider, and rollback guarantees.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug report. Source and docs inspection show current main supports only light, REM, and deep dreaming phases.

## Solution Assessment

Unclear: the requested LoRA/fine-tuning phase is plausible product direction, but direct implementation is not yet the narrowest maintainable solution. Maintainers should first decide the memory-core versus plugin boundary and the training-data safety contract.
