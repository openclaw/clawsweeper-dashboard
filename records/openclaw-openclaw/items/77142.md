## Summary

Keep open: current main and the latest shipped release still expose only the light/REM/deep text-memory dreaming pipeline, while this asks for a broad parametric-learning phase with new memory/plugin boundaries, provider/local training behavior, config, evaluation, and safety policy. Related dreaming issues cover distillation, indexing, wildcard ranking, and cleanup, but none supersedes the LoRA/provider fine-tuning request.

## What This Changes

Add an optional fourth dreaming `consolidation` phase that distills experience into training data and fine-tunes LoRA/adapters with checkpoints and regression guards.

## Best Possible Solution

Define a maintainer-reviewed design for plugin-owned or memory-core-owned parametric consolidation, including stable artifact APIs, opt-in config, provider/local training contracts, privacy gates, evaluation, provenance, and rollback before implementation.
