## Summary

Keep open: current main and the latest shipped release still expose the existing text-memory dreaming pipeline, while this asks for a broad new parametric-learning surface with training, provider/local execution, checkpoints, rollback, provenance, and security policy. The reporter's follow-up adds useful security requirements, but that makes this a maintainer product/API/security design item rather than cleanup closure.

## What This Changes

Add an optional post-deep dreaming consolidation phase that distills session and memory outputs into LoRA or provider fine-tuning checkpoints with regression guards and rollback config.

## Best Possible Solution

Define a maintainer-reviewed parametric consolidation design covering ownership, training-artifact APIs, provider/local training contracts, opt-in config, privacy gates, evaluation, provenance, checkpoint activation, rollback, docs, and tests.
