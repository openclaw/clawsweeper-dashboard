## Summary

Keep open: current main only implements the light, REM, and deep text-memory dreaming pipeline, while the requested parametric consolidation phase is a broad new feature that needs maintainer product, plugin-boundary, provider, and safety design before code work.

## What This Changes

Request adds an optional post-deep dreaming consolidation phase that distills memory/session output into LoRA or provider fine-tuning checkpoints with opt-in safety controls.

## Best Possible Solution

Define a maintainer-reviewed design for optional parametric consolidation that is opt-in, auditable, rollbackable, and plugin-capable before adding phase, config, provider, or checkpoint code.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug report. Source and docs inspection on current main confirm the existing dreaming pipeline has only light, REM, and deep phases.

## Solution Assessment

Unclear: the proposed LoRA/fine-tuning phase is a useful design sketch, but not yet the narrowest maintainable implementation path. The safer path is a maintainer design review around memory/plugin boundaries, training-data safety, provider behavior, evaluation, and rollback semantics.
