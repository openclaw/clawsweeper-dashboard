## Summary

Keep open: current main and v2026.5.7 still expose only the light/REM/deep text-memory dreaming pipeline, while this asks for a new parametric-learning phase with training, checkpoint, rollback, provider/local execution, and safety policy. The related dreaming issues cover text distillation, review queues, ranking, and ingestion quality, but none supersedes the LoRA/provider fine-tuning request.

## What This Changes

The issue requests an optional post-deep dreaming consolidation phase that distills memory artifacts into LoRA or provider fine-tuning checkpoints.

## Best Possible Solution

Define a maintainer-reviewed parametric consolidation design first, including memory/plugin ownership, stable training-artifact APIs, opt-in config, provider/local training contracts, privacy gates, evaluation, provenance, and rollback.
