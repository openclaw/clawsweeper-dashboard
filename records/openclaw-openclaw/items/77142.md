## Summary

Keep open: current main supports only the existing text-memory dreaming pipeline, while parametric consolidation is a broad new memory/provider feature that needs API, config, privacy, evaluation, and rollback design. The reporter's follow-up usefully narrows the security requirements, but it does not make this a conservative cleanup close.

## What This Changes

Add an optional post-deep dreaming consolidation phase that distills memory/session output into LoRA or provider fine-tuning adapters with checkpoints and rollback.

## Best Possible Solution

Define a maintainer-reviewed design for plugin-owned or memory-core-owned consolidation, including stable SDK seams, opt-in data controls, dry-run review, provenance, evaluation gates, provider behavior, and rollback before implementation.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug report. Source and docs inspection show current main supports light, REM, and deep text-memory dreaming only.

## Solution Assessment

Unclear: the proposed LoRA/fine-tuning phase is directionally coherent but not yet a narrow maintainable implementation path. The safer route is a design review around memory/plugin ownership, training-data safety, provider integration, and activation/rollback semantics.
