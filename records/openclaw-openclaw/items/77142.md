## Summary

Keep open: current main only has the documented light/REM/deep text-memory dreaming pipeline, while the requested parametric consolidation phase is a broad new feature that needs memory/plugin boundary, provider, config, and safety decisions before implementation.

## What This Changes

Request adding an optional post-deep-sleep consolidation phase that distills dreaming output into LoRA/adapter fine-tuning with checkpoints and rollback guards.

## Best Possible Solution

Define a maintainer-reviewed design for plugin-owned or memory-core-owned parametric consolidation, including stable data boundaries, safety gates, provider behavior, and evaluation before adding phase/config support.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug report. Source and docs inspection show current main supports only light, REM, and deep dreaming phases.

## Solution Assessment

Unclear: the proposed LoRA/fine-tuning phase is not yet a safe narrow implementation path. The maintainable route is a design review around memory plugin boundaries, training-data safety, provider integration, and rollback semantics before code work.
