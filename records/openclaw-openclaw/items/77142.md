## Summary

Keep open: current main still implements Dreaming as an opt-in light/REM/deep text-memory pipeline, while this asks for a new parametric training/checkpoint/rollback surface that needs maintainer product, security, memory, and provider/API design before implementation.

The issue requests an optional post-deep-sleep Dreaming consolidation phase that distills memory and session artifacts into LoRA or provider fine-tuning checkpoints with guards and rollback.

## Best Possible Solution
