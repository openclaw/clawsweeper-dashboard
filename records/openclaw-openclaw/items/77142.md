## Summary

Keep open: current main and v2026.5.7 still expose the light/REM/deep text-memory dreaming pipeline, while this asks for a broad new parametric-learning phase with training, provider/local execution, checkpoints, rollback, provenance, and safety policy. The reporter's follow-up adds useful security requirements, but that reinforces that this needs maintainer product/API/security design rather than cleanup closure.

## What This Changes

Request an optional fourth dreaming consolidation phase that distills session and dreaming artifacts into LoRA/fine-tuned adapters with opt-in config, checkpoints, regression guards, and rollback.

## Best Possible Solution

Define a maintainer-reviewed parametric consolidation design first, including ownership, stable training-artifact APIs, provider/local training contracts, opt-in config, privacy gates, evaluation, provenance, checkpoint activation, rollback, docs, and tests.
