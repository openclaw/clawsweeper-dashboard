## Summary

Keep open: current main still sends supervised npm-global Control UI update.run calls through the managed-service handoff, records the handoff as skipped/pending, and schedules a SIGUSR1 restart after handoff start. The later launchd fixes reduce adjacent failure modes, but the issue comments include live no-op/latest and service-not-found evidence after that stack, and the remaining fix shape still needs maintainer product judgment plus macOS proof.

## What This Changes

The issue reports that the Control UI Update button can break or hang a macOS launchd-managed npm-global Gateway when update.run reaches a skipped or managed-handoff restart path.

## Best Possible Solution

Keep this as the canonical tracker, decide the update.run contract for latest/no-op supervised package updates, and close only after live macOS npm-global launchd proof covers the Control UI/update.run no-op and service recovery paths.

## Reproduction Assessment

No high-confidence current-main live repro was run. Source inspection shows the same supervised global update.run handoff and SIGUSR1 restart path still exists, and issue comments provide live macOS evidence on recent releases.

## Solution Assessment

Unclear: short-circuiting no-op updates is plausible, but the comments and current code leave a maintainer choice between no-op return, explicit refusal, and a stronger durable handoff contract.
