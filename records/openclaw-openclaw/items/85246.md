## Summary

Keep open: current main still routes supervised npm-global `update.run` through the managed-service handoff and schedules an `update.run` SIGUSR1 restart after handoff start. ## What This Changes

The issue requests that the Control UI `update.run` path not break an npm-global, launchd-managed Gateway when the update is already at the latest version.

## Best Possible Solution
