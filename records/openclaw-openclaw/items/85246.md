## Best Possible Solution

Run the direct macOS npm-global LaunchAgent proof for the UI Update button or `gateway call update.run` on current main; close as covered if it passes, or make the remaining fix in the control-plane update contract around durable handoff versus explicit no-op/refusal if it still fails.

## Solution Assessment

Unclear. A no-restart-on-latest change may address the reported skipped status, but the current documented contract intentionally refreshes and restarts package-managed installs; the safer direction depends on live proof of whether durable handoff or explicit refusal/no-op is needed.
