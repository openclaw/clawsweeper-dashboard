## Summary

Keep open: current main still uses the supervised global `update.run` managed-service handoff plus restart shape, and a maintainer explicitly kept this issue open until the exact UI Update/npm-global/launchd latest-version flow has direct macOS proof.

## What This Changes

The issue requests that the Control UI Update path avoid breaking a launchd-managed npm-global Gateway when `update.run` is already at the latest version.

## Best Possible Solution

Keep the managed-update contract, then prove the exact macOS LaunchAgent/npm-global latest-version UI path and either make the launchd handoff durable or fail before stopping the Gateway when that handoff cannot be made safe.

## Reproduction Assessment

No high-confidence live reproduction is available for current main. Source inspection confirms the reported `update.run` handoff plus SIGUSR1 restart path, but the macOS launchd collision after the adjacent fixes still needs direct proof.

## Solution Assessment

Unclear. A no-restart-on-latest change may be too narrow and conflicts with documented refresh semantics; the safer direction is a durable launchd-owned handoff or an explicit safe refusal before the Gateway stops.
