## Summary

Keep open: current main and the latest release still route supervised npm-global `update.run` through the managed-service handoff, return a `managed-service-handoff-started` skipped result, and schedule the `update.run` restart path that live macOS comments show can leave the launchd Gateway unhealthy.

## What This Changes

The issue asks for the Control UI/Gateway `update.run` flow to avoid or safely recover the managed-service handoff/restart when an npm-global launchd install is already up to date.

## Best Possible Solution

Keep this as the canonical tracker, decide the `update.run` no-op and launchd handoff contract, then close only after live macOS npm-global LaunchAgent proof covers the Control UI/update.run latest/no-op and service recovery paths.
