## Summary

Keep open for a blocking correctness fix: the PR’s new post-compaction cursor does not advance for the normal production tool path, so the guard can miss the repeated outcomes it is meant to abort.

## What This Changes

The PR adds a default-on post-compaction loop guard for `runEmbeddedPiAgent`, exposes `tools.loopDetection.postCompactionGuard` config/docs/schema, tracks a diagnostic outcome sequence, and adds unit/integration/config tests plus a changelog entry.

## Best Possible Solution

Fix the outcome cursor so both matched result updates and unmatched pushes count as newly observable outcomes, then add a regression that uses the production `recordToolCall` plus `recordToolCallOutcome` flow.

## Reproduction Assessment

Yes, from source inspection. The production hook records a tool call first and later updates that same record with the result, while the PR’s sequence counter advances only when `recordToolCallOutcome` pushes an unmatched record.

## Solution Assessment

No. The guard design is narrow and plausible, but the current implementation observes only one outcome-recording branch; the safer fix is to mark every newly populated `resultHash` as observable and cover the production path in tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Count matched tool outcomes in the cursor:** `src/agents/tool-loop-detection.ts:734`
  - body: Production tools normally call `recordToolCall` before execution and then `recordToolCallOutcome` updates that existing entry with `resultHash`. This increment only runs when no existing entry matched, so `currentSeq - lastObservedToolOutcomeSeq` stays at zero for ordinary successful tool calls and the post-compaction guard never observes the repeated `(tool,args,result)` outcomes it is supposed to abort. Increment the sequence when a matched record first gets its result too, and add a regression that uses `recordToolCall` followed by `recordToolCallOutcome` rather than pushing outcome records directly.
  - confidence: 0.93
