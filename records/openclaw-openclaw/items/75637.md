## Summary

Keep open. Current main still has the misleading vector-degradation warning, and this PR is a narrow, reviewable fix for the reported #75624 behavior rather than obsolete work already implemented on main.

## What This Changes

The PR changes memory-core manager and CLI vector-degradation warning text to distinguish sqlite-vec load failures from missing embedding-provider dimensions, adds a focused helper regression test, and adds an Unreleased changelog entry.

## Best Possible Solution

Land this branch after maintainer review and CI, keeping the loadError-based sqlite-vec distinction, the missing-provider helper regression test, and the changelog credit for the linked report.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: no embedding provider leads memory-core to write FTS-only chunks with vectorReady false and no sqlite-vec loadError, and the current helper then emits the sqlite-vec-unavailable warning.

## Solution Assessment

Yes. Using loadError to decide whether to name sqlite-vec is the narrowest maintainable fix because actual sqlite-vec load failures set that field, while the missing-provider degraded path does not.
