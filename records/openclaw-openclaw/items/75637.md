This PR updates memory-core manager and CLI vector-degradation warning text, adds a regression test for the no-loadError case, and adds a CHANGELOG entry for the linked memory warning bug.

## Best Possible Solution

## Reproduction Assessment

Yes. Configure memory with vector storage enabled but no embedding provider, then index FTS-only chunks; current main writes chunks with vectorReady=false and no sqlite-vec loadError, so the helper emits the misleading sqlite-vec warning.

## Solution Assessment

Yes. Using loadError to decide whether to name sqlite-vec is the narrowest maintainable fix because actual sqlite-vec load failures populate that field, while missing embedding dimensions do not.
