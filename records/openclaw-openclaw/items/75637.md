## Summary

Keep open for normal PR review. Current main still has the misleading memory-core helper warning when vector writes are degraded without a sqlite-vec load error, and this PR is a narrow, apparently correct fix with no blocking review findings or security concerns.

## What This Changes

The PR updates memory-core vector degradation warnings in the manager helper and CLI index path to distinguish sqlite-vec load errors from missing embedding dimensions, adds a helper regression test, and adds an Unreleased changelog fix entry.

## Best Possible Solution

Land this branch after normal maintainer review and CI, preserving the loadError-based sqlite-vec distinction and the missing-dimensions warning text without broadening the memory status contract.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction path: FTS-only indexing can call writeChunks with vectorReady false and no vector loadError, and the current helper then emits the sqlite-vec-unavailable warning.

## Solution Assessment

Yes. Using loadError to select the sqlite-vec-specific message is the narrowest maintainable fix because sqlite-vec load failures set that field, while the missing-provider/no-dimensions path leaves it absent.
