## Summary

## What This Changes

This PR changes memory-core manager and CLI vector-degradation warnings to distinguish sqlite-vec load failures from missing embedding-provider dimensions, adds a focused helper regression test, and adds a changelog entry for #75624.

## Best Possible Solution

Land this branch after maintainer review and targeted CI, preserving the loadError-based sqlite-vec wording, the missing-provider warning text, the helper regression test, and the changelog credit for the linked user report.

## Reproduction Assessment

Yes. The linked report gives a concrete live path, and current source independently reproduces the logic: no provider writes FTS-only chunks with `vectorReady=false` and no `loadError`, causing the helper to blame sqlite-vec.

## Solution Assessment

Yes. The PR's `loadError` discriminator is the narrowest maintainable fix for this bug because sqlite-vec failures already populate that field, while missing embedding-provider dimensions do not.
