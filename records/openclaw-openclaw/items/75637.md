## What This Changes

The PR changes memory-core manager and CLI vector-degradation warnings to distinguish sqlite-vec load failures from missing embedding dimensions, adds a focused regression test, and adds an Unreleased changelog entry.

## Best Possible Solution

Land this branch after maintainer review and targeted CI, preserving the loadError-based sqlite-vec distinction and the regression coverage for the missing-provider warning.

Yes. Source inspection gives a high-confidence reproduction path: configure memory with vector writes enabled but no embedding provider, index chunks in FTS-only mode, and current main calls the warning helper with vectorReady false and no sqlite-vec loadError.

## Solution Assessment

Yes. Using loadError to decide whether to name sqlite-vec is the narrowest maintainable fix because actual sqlite-vec failures set that field, while missing-provider degraded writes do not.
