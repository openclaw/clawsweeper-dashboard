## Summary

Keep open. Current main still misattributes the missing-embedding-provider degraded write as sqlite-vec unavailable, and this focused PR uses the existing loadError discriminator with no blocking review findings.

## What This Changes

The PR changes memory-core manager and CLI vector-degradation warnings to mention sqlite-vec only when loadError is present, adds a missing-provider warning regression test, and adds a changelog entry.

## Best Possible Solution

Land this focused diagnostic fix after maintainer review and exact-head validation, preserving the loadError-based sqlite-vec distinction and missing-provider regression coverage.

## Reproduction Assessment

Yes. Current main has a code-level reproduction: no-provider FTS-only indexing calls writeChunks with vectorReady=false and empty embeddings, and the helper then emits the sqlite-vec unavailable warning even when loadError is undefined.

## Solution Assessment

Yes. Using vector.loadError is the narrowest maintainable discriminator because sqlite-vec load failures set it, while missing-provider FTS-only writes do not.
