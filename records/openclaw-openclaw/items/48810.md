## Summary

Keep this open because current main still lacks the compaction-fork reconciliation, and an open closing PR already owns the fix path with real behavior proof.

## What This Changes

The issue reports that compaction retry can append duplicate same-parent compaction events, leaving one as a dead-end branch that breaks downstream parentId-chain reconstruction.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79635 if the bounded compaction-specific repair remains acceptable, then close this issue after merge.

## Reproduction Assessment

Yes for the corrupted artifact path: current main's session-file repair would leave the same-parent compaction fork unchanged, and the issue provides concrete JSONL showing that shape. I did not live-reproduce the retry race that creates the artifact.

## Solution Assessment

Yes, the best current path is the linked PR: it adds a bounded load-time repair for compaction-vs-compaction fork losers while preserving ordinary transcript tree branches. A separate source-side retry prevention design can follow if maintainers want that additional hardening.
