## Summary

Keep this open. Current main still treats Pi transcripts as a tree/DAG and OpenClaw readers follow the active SessionManager branch, but the compaction retry path still does not supersede, deterministically replace, or reconcile duplicate same-parent compaction entries. That leaves the reported raw JSONL parentId-chain integrity concern unresolved for downstream consumers that walk parentId as a linear list.

## What This Changes

The issue requests compaction retry behavior that prevents duplicate same-parent compaction entries from creating dead-end forks in JSONL parentId-chain reconstruction.

## Best Possible Solution

Maintainers should choose and document the transcript invariant explicitly: either add compaction-retry reconciliation or supersession with regression coverage, or make the active-branch reader contract the public integration path so downstream tools do not treat raw parentId as globally linear.

## Reproduction Assessment

Unclear. The issue includes concrete raw JSONL showing the duplicate same-parent compaction shape, but the reporter says they cannot reproduce it and this read-only review did not run a live compaction retry scenario.

## Solution Assessment

Unclear. Superseding or deterministic retry IDs may be the right fix if raw JSONL must stay linear, but the current SessionManager tree model means maintainers need to decide the contract before a narrow code change is safe.
