## Summary

Current main still has the reported ACP background-task completion behavior, and an open closing PR already targets this issue. The issue should remain open until PR #74103, or an equivalent reviewed fix, lands.

## What This Changes

The issue requests ACP background-task completions to use sanitized progress as the terminal summary for non-blocked successes and to raise the progress-summary cap from 240 to about 1200 characters.

## Best Possible Solution

Land PR #74103 after review, or replace it with an equivalent narrow ACP/task patch that raises the bounded progress cap, derives successful `terminalSummary` only from cleaned user-safe progress, and preserves blocked/internal-progress protections.

## Reproduction Assessment

Yes. The current-main source path is deterministic: non-thought ACP output fills `taskProgressSummary`, ordinary non-blocked success returns `{}`, terminal finalization stores a null summary, and delivery formats only `terminalSummary`.

## Solution Assessment

Yes. The requested fix direction is the narrow maintainable solution if it uses sanitized successful progress as the terminal fallback while preserving the existing blocked and internal-progress safeguards.
