## Summary

Keep open: current main still drops ordinary ACP background-task progress from `terminalSummary` and keeps the 240-character cap; https://github.com/openclaw/openclaw/pull/74103 remains the open closing implementation candidate, while the merged whitespace fix only covered adjacent chunk-boundary behavior.

## What This Changes

The issue asks for ACP background-task completion delivery to expose ordinary progress as `terminalSummary` and increase the progress summary cap beyond 240 characters.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/74103 or an equivalent narrow ACP manager fix with regression coverage for ordinary progress fallback, silent-token suppression, and the chosen cap.

Yes. Source inspection shows ACP progress is accumulated, ordinary successful progress resolves to `{}`, the success path persists `terminalSummary: null`, and terminal delivery formats only `terminalSummary`; I did not run a live ACP session in this read-only review.

## Solution Assessment

Yes, keeping this open for the linked implementation PR is the right path. The narrow fix belongs in `src/acp/control-plane/manager.core.ts` with tests that preserve blocked/internal suppression while making ordinary progress visible.
