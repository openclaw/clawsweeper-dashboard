## Summary

Keep open: current main and the latest release still show the reported ACP background-task summary behavior, and PR #74103 remains open/unmerged with a review finding, so this should stay open until that fix or an equivalent reviewed change lands.

## What This Changes

The issue requests ACP background-task completions to retain a larger progress summary and use sanitized non-blocked progress as terminalSummary instead of delivering bare done messages.

## Best Possible Solution

Land a narrow ACP manager fix that derives successful terminal summaries from cleaned safe progress, keeps blocked/internal suppression intact, accepts a bounded cap increase, and adds focused regression coverage.

## Reproduction Assessment

Yes. On current main, a parent-owned ACP run with non-thought text_delta progress reaches a 240-character progress cap, the resolver returns `{}` for ordinary progress, terminalization stores `terminalSummary: null`, and task delivery emits the bare completion message.

## Solution Assessment

Yes, the fix belongs in `src/acp/control-plane/manager.core.ts` with focused task-delivery tests; the linked PR is directionally right but needs skip-token cleanup before merge.
