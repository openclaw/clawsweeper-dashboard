## Summary

Keep open: current main and the latest shipped tag still have the reported ACP background-task behavior, and open PR #74103 already targets this issue with closing syntax. The issue should remain open until that PR, or an equivalent reviewed fix, lands.

## What This Changes

Request to make successful ACP background-task completions include a useful terminal summary derived from progress output and raise the ACP progress-summary cap beyond 240 characters.

## Best Possible Solution

Review and land PR #74103, or replace it with an equivalent narrow ACP/task patch that raises the bounded progress cap, derives successful terminal summaries only from safe cleaned progress, preserves blocked outcome handling, and adds regression coverage for useful summaries plus suppressed internal/control text.

## Reproduction Assessment

Yes. The source path is deterministic: non-thought ACP text updates `taskProgressSummary`, ordinary non-blocked success returns no terminal summary, terminalization stores null, and delivery formats the bare done message.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the ACP control-plane terminal-result path and its focused task delivery tests; the safer version is not a blind fallback, but a sanitized fallback that keeps existing internal-progress protections.
