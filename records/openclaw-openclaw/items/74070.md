## Summary

Keep open: current main and the latest release still match the reported ACP background-task behavior, and PR #74103 is the open closing implementation candidate.

## What This Changes

OpenClaw should surface useful terminal summaries for successful ACP background-task completions and use a longer progress-summary cap.

## Best Possible Solution

Review and land PR #74103 or an equivalent narrow ACP manager fix with regression coverage for ordinary progress fallback, blocked/internal suppression, and the chosen cap.

## Reproduction Assessment

Yes. Source inspection on current main shows progress is capped at 240, ordinary successful ACP progress does not become terminalSummary, and the delivery formatter only appends terminalSummary.

## Solution Assessment

Yes, with review. The narrow fix belongs in src/acp/control-plane/manager.core.ts, but it should preserve the existing internal-progress-detail suppression behavior while increasing the cap deliberately.
