## What This Changes

The issue requests useful ACP background-task completion summaries by deriving terminalSummary from progressSummary for successful non-blocked completions and raising the progress summary cap from 240 to about 1200 characters.

## Best Possible Solution

Land PR #74103 after review, or replace it with an equivalent narrow ACP/task patch that raises the bounded progress cap, derives terminalSummary only from cleaned user-safe successful progress, preserves blocked outcome handling, and adds regression coverage for useful successful summaries plus suppressed internal/control text.

## Reproduction Assessment

Yes. The static current-main path is high-confidence: non-thought ACP output fills taskProgressSummary, ordinary successful non-blocked completion returns {}, terminal finalization stores a null summary, and delivery formats only terminalSummary.

## Solution Assessment

Yes on the requested fix boundary, with a safety caveat. Falling back from cleaned progress output and increasing the cap is maintainable, but the final patch should include sanitization and tests so internal ACP chatter remains suppressed.
