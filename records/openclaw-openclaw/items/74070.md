## Summary

Keep open: current main and `v2026.4.29` still have the reported ACP background-task summary behavior, and open PR #74103 is the active closing implementation candidate rather than a reason to close the issue early.

## What This Changes

The issue requests ACP background-task completions to use successful progress text as `terminalSummary` and to raise the progress-summary cap from 240 to about 1200 characters.

## Best Possible Solution

Land a narrow ACP manager fix that derives safe successful terminal summaries from cleaned progress text, increases the bounded cap, and adds regression coverage for both useful success summaries and suppressed internal/control-token output.

## Reproduction Assessment

Yes. Static source tracing on current main shows non-thought ACP `text_delta` progress is capped at 240 characters, ordinary successful progress resolves to `{}`, terminalization stores `terminalSummary: null`, and task delivery then emits the bare completion message.

## Solution Assessment

Yes. The maintainable fix belongs in `src/acp/control-plane/manager.core.ts` with focused task-delivery tests; PR #74103 is directionally aligned but still needs review for the suppression/sanitization edge cases before this issue closes.
