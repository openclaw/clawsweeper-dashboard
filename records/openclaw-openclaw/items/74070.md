## Summary

Keep open: current main and the latest release still contain the reported ACP background-task summary behavior, and open PR #74103 is already linked with closing syntax, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Request that ACP background-task completions derive successful terminal summaries from cleaned progress text and raise the progress-summary cap from 240 to about 1200 characters.

## Best Possible Solution

Land a narrow ACP manager change that derives successful terminal summaries from safe cleaned progress, keeps blocked/internal suppression intact, raises the cap with a bounded value, and adds focused regression coverage.

## Reproduction Assessment

Yes. The source path shows non-thought ACP `text_delta` chunks are capped into `taskProgressSummary`, ordinary progress resolves to `{}`, terminalization stores `terminalSummary: null`, and delivery then emits the bare completion message.

## Solution Assessment

Yes. The fix belongs in `src/acp/control-plane/manager.core.ts` with task-delivery regression tests; the linked PR is directionally correct but should be reviewed against the existing internal-detail suppression contract.
