## Summary

Keep open: current main still has the reported ACP background-task behavior, and open PR #74103 is already the canonical implementation candidate with closing syntax. Closing this issue before that PR, or an equivalent reviewed fix, lands would be premature.

## What This Changes

The issue requests ACP background-task completions to preserve useful successful progress as a terminal summary and raise the progress-summary cap from 240 characters to about 1200.

## Best Possible Solution

ACP background-task completion should keep a larger bounded progress summary, derive successful terminal summaries only from sanitized user-safe progress, preserve blocked/internal suppression, and add focused regression coverage for useful success summaries plus suppressed internal/control text.

## Reproduction Assessment

Yes. The deterministic source path on current main still caps progress at 240 characters, returns no terminal summary for ordinary successful progress, stores `terminalSummary: null`, and formats completion as a bare done message.

## Solution Assessment

Yes, the narrow maintainable fix belongs in `src/acp/control-plane/manager.core.ts` with task-delivery regression tests. The linked PR is the right review target, but the accepted version should avoid exposing raw internal progress.
