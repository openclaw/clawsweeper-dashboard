## Summary

Keep open: current main and the latest provided release still retain the reported ACP background-task behavior, and open PR #74103 is the linked implementation candidate that should be reviewed instead of closing this issue early.

## What This Changes

The issue requests that successful ACP background-task completions fall back from progressSummary to terminalSummary and that the progress summary cap increase beyond 240 characters.

## Best Possible Solution

Review and land PR #74103 or an equivalent narrow ACP manager fix with tests for ordinary progress fallback, blocked/internal suppression, and cap behavior.

## Reproduction Assessment

Yes by source inspection. Current main captures non-thought ACP text_delta output into progressSummary, stores terminalSummary as null for ordinary successful completions, and formats successful delivery from terminalSummary only.

## Solution Assessment

Yes for the fallback path, with one caveat: the cap increase is a small product judgment. The narrow maintainable fix belongs in src/acp/control-plane/manager.core.ts, with tests covering sanitization and non-blocked completion summaries.
