## Summary

Keep open: current main and v2026.6.6 still leave ordinary successful ACP background-task completions without a terminalSummary and still cap progress at 240 characters. The linked fix PR was closed unmerged, while the adjacent merged chunk-spacing fix did not cover this fallback/cap issue.

## What This Changes

The issue asks for ACP background-task completions to surface progressSummary as terminalSummary and to raise the progress-summary cap from 240 to about 1200 characters.

## Best Possible Solution

Land a narrow ACP background-task fix that falls back from sanitized progressSummary to terminalSummary for ordinary successful completions, strips trailing ANNOUNCE_SKIP, raises the bounded progress cap, and updates regression coverage.

## Reproduction Assessment

Yes, by source inspection: the resolver returns {} for ordinary non-blocked progress, the turn runner persists that as null terminalSummary, and current tests assert terminalSummary is undefined for a successful progress-only ACP completion. I did not run a live ACP session in this read-only review.

## Solution Assessment

Yes, the best fix belongs in the ACP background-task helper/turn-result tests rather than in downstream delivery formatting. The prior PR direction is right, but the current code location must be used and the ANNOUNCE_SKIP guard should be included.
