## Summary

Keep open: current main still has the reported ACP background-task summary behavior, and open PR #74103 is the implementation path rather than a reason to close this issue before merge.

## What This Changes

The issue asks for ACP background-task completions to use safe `progressSummary` text as `terminalSummary` and to raise the progress cap from 240 to about 1200 characters.

## Best Possible Solution

Land a reviewed ACP manager change, likely via #74103 or an equivalent patch, that derives successful terminal summaries from sanitized progress, raises the cap to a bounded value, and covers both useful-summary and suppression cases.

## Reproduction Assessment

Yes. On current main, non-thought ACP `text_delta` output is capped into `progressSummary`, ordinary successful progress resolves to no `terminalSummary`, and task delivery then emits the bare completion message.

## Solution Assessment

Yes. The narrowest maintainable fix belongs in `src/acp/control-plane/manager.core.ts` with focused task-delivery regression coverage, while preserving blocked outcome handling and internal-detail suppression.
