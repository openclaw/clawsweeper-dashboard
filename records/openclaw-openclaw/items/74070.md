## Summary

Keep open: current main and the latest release still show the reported ACP background-task behavior, and open PR #74103 already targets this issue with closing syntax. The issue should remain open until that PR, or an equivalent reviewed fix, lands.

## What This Changes

The issue requests that ACP background-task success completions surface a useful terminal summary from progress text and that the progress-summary cap increase beyond 240 characters.

## Best Possible Solution

ACP background-task completions should keep a larger bounded progress summary, derive successful terminal summaries only from sanitized safe progress, preserve blocked/internal suppression, and add regression coverage for useful success summaries plus suppressed internal/control text.

## Reproduction Assessment

Yes. The deterministic source path on current main still truncates ACP progress at 240 characters, returns no terminal summary for ordinary successful progress, stores `terminalSummary: null`, and formats the completion as a bare done message.

## Solution Assessment

Yes. The narrow maintainable solution belongs in `src/acp/control-plane/manager.core.ts` with focused task-delivery tests, provided the fallback uses the existing user-facing sanitization rules rather than exposing raw progress blindly.
