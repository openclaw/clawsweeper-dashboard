## Summary

Keep open: current main and v2026.6.6 still return no terminalSummary for ordinary successful ACP progress, persist that as a null terminal summary, and cap progress at 240 characters. The direct fix PR was closed unmerged, and the adjacent merged chunk-boundary fix did not cover this fallback/cap issue.

## What This Changes

Fix ACP background-task completion summaries so ordinary successful progress becomes the terminalSummary and progress summaries are not clipped at 240 characters.

## Best Possible Solution

Land a narrow ACP background-task fix that falls back from sanitized progressSummary to terminalSummary for ordinary successful completions, strips trailing ANNOUNCE_SKIP, raises the bounded progress cap, and updates regression coverage.

## Reproduction Assessment

Yes. Source inspection shows the resolver returns {} for ordinary non-blocked progress, the turn runner persists that as null terminalSummary, and a current test asserts progressSummary without terminalSummary for a successful ACP completion.

## Solution Assessment

Yes, the best fix belongs in the ACP background-task helper and turn-result tests rather than downstream delivery formatting. The prior PR direction is right, but it needs the current helper split and the ANNOUNCE_SKIP guard.
