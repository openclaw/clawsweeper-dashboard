## Summary

Keep open: current main and v2026.6.10 still drop ordinary successful ACP progress from `terminalSummary`, but related maintainer feedback rejected the direct progress-buffer fallback as unsafe without a terminal-only result contract.

## What This Changes

The issue requests ACP background-task success completions to include useful terminal summary text and to raise the progress-summary truncation cap.

## Best Possible Solution

Add an owner-approved ACP terminal-result payload and thread only that terminal signal into detached-task `terminalSummary`, leaving live progress bounded and private unless maintainers explicitly change that exposure model.

## Reproduction Assessment

Yes. Current main has a deterministic source-level reproduction: ordinary non-blocked ACP progress resolves to no `terminalSummary`, and successful task delivery only appends `terminalSummary`.

## Solution Assessment

No. The issue's direct fallback is not the best fix after maintainer feedback and Codex source inspection; the maintainable path is a terminal-only ACP result signal rather than publishing accumulated streamed progress.
