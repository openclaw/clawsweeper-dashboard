## Summary

Keep open. Current main still has a source-reproducible stale Control UI run-state path, and open PR #76894 uses closing syntax for this issue, so the issue should stay open until that PR is reviewed and merged or closed.

## What This Changes

The issue requests that WebUI/Control UI return the input control from Stop to Send after a completed assistant reply and avoid carrying stale running state into later messages.

## Best Possible Solution

Review and land or close PR #76894; if it does not land, fix Control UI reconciliation so matching terminal chat lifecycle events clear cached session active-run state and cover that with a regression test.

## Reproduction Assessment

Yes, by source inspection: start with the current session row cached as `hasActiveRun: true`, then deliver a matching terminal chat event. `chatRunId` clears, but the preserved session row can still make `canAbort` true.

## Solution Assessment

Yes. The narrow maintainable fix is to reconcile the cached session active-run flag on matching terminal chat lifecycle events, or otherwise refresh the session row after terminal cleanup; the linked PR is aimed at that boundary but is not merged.
