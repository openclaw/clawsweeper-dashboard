## Summary

Keep open. Current main still has a source-reproducible stale Control UI run-state path, and the linked open PR #76894 uses closing syntax for this issue, so this should remain open until that PR is reviewed and merged or closed.

## What This Changes

The issue requests that Control UI clear stale active-run state after an assistant reply completes so the Stop button returns to Send without a page refresh.

## Best Possible Solution

Review and land or close PR #76894; after a merged fix, GitHub can close this issue as implemented by the merge.

## Reproduction Assessment

Yes, by source inspection: start with the current session row cached as `hasActiveRun: true`, then deliver a matching terminal chat event. `chatRunId` clears, but the preserved session row can keep `canAbort` true.

## Solution Assessment

Yes. The narrow maintainable fix is to reconcile the cached session active-run flag on matching terminal chat lifecycle events, or otherwise refresh the session row after terminal cleanup; the linked PR appears scoped to that boundary but is not merged.
