Keep open. Current main still has a source-reproducible stale Control UI active-run path, and #76894 is already an open closing PR that should be reviewed, landed, or replaced before this issue closes.

## What This Changes

Make the Control UI switch the Stop button back to Send after a completed assistant reply instead of preserving stale active-run state.

## Best Possible Solution

Review and land #76894 or an equivalent regression-tested Control UI reconciliation patch that clears stale `hasActiveRun` for the current session on terminal chat/session lifecycle events without changing abort semantics.

## Reproduction Assessment

Yes, by source inspection: seed the current session row with `hasActiveRun: true`, then process a matching final chat event. `chatRunId` clears, but the session row can remain true and keep `canAbort` true.

## Solution Assessment

Yes. The narrow maintainable fix is Control UI reconciliation on terminal events or session refresh; a broader protocol change is not required unless maintainers want `hasActiveRun` emitted in lifecycle snapshots.
