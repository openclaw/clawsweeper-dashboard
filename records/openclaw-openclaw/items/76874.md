## What This Changes

The issue requests robust WebUI reconciliation so a completed assistant reply clears stale active-run state and switches the Stop button back to Send.

## Best Possible Solution

Ensure terminal chat completion also reconciles the Control UI's cached session active-run state so Stop switches back to Send and later user messages start separate turns.

## Reproduction Assessment

Yes, by source inspection: start with the current session row cached as `hasActiveRun: true`, then deliver a matching terminal `chat` event. Current main clears `chatRunId` but can preserve the cached row value that keeps `canAbort` true.

## Solution Assessment

Yes. The narrow maintainable solution is to clear or refresh the current session row's `hasActiveRun` on matching terminal chat/session lifecycle events; a broader protocol redesign is not required unless maintainers want to include active-run state in lifecycle snapshots.
