Keep open. Current main still has a source-reproducible stale Control UI run-state path, and open PR #76894 uses closing syntax for this issue, so it should remain open until that PR is reviewed and merged or closed.

## What This Changes

The issue asks for Control UI/WebUI to clear stale active-run state so Stop switches back to Send after a completed assistant reply.

## Best Possible Solution

Review and land or close PR #76894; after merge, this issue can close as implemented by that PR.

## Reproduction Assessment

Yes. By source inspection on current main, a cached current session row with `hasActiveRun: true` can survive a matching terminal chat event after `chatRunId` is cleared, leaving `canAbort` true.

## Solution Assessment

Yes. The narrow maintainable solution is to reconcile the cached current session active-run flag on matching terminal chat lifecycle completion, with regression coverage for the Stop-button predicate path.
