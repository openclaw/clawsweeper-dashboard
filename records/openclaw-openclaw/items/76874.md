## What This Changes

The issue asks for WebUI run-state reconciliation so the Stop button returns to Send after a completed assistant reply without requiring a page refresh.

## Best Possible Solution

Review and land or close PR #76894; the maintainable fix should reconcile the cached session active-run flag on matching terminal chat lifecycle events and keep regression coverage for that path.

## Reproduction Assessment

Yes, by source inspection: start with the current session row cached as `hasActiveRun: true`, then process a matching terminal chat event. `chatRunId` clears, but the preserved session row can keep `canAbort` true.

## Solution Assessment

Yes. The narrow solution is to clear or refresh the cached current-session `hasActiveRun` state when the matching terminal event proves the run has ended; PR #76894 appears aimed at that boundary but has not merged.
