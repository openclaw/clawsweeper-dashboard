## What This Changes

The issue requests that Control UI clear stale active-run state after a completed assistant reply so Stop returns to Send and later messages start separate turns.

## Best Possible Solution

Review and land or close PR #76894; the final state should reconcile cached session active-run state when terminal chat events complete so Stop returns to Send without a page refresh.

## Reproduction Assessment

Yes, by source inspection: start with the current session row cached as `hasActiveRun: true`, then deliver a matching terminal `chat` event. `chatRunId` clears, but the cached session row can keep `canAbort` true.

## Solution Assessment

Yes. The narrow maintainable fix is to clear or refresh the current session's cached `hasActiveRun` when a matching terminal chat lifecycle event completes; PR #76894 is already scoped to that boundary but is not merged.
