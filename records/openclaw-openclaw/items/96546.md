## Summary

Keep open: current main still evaluates the inbound daily-stale session during auto-reply initialization and passes the previous entry into the archive path before the later active-run queue gate, while the closing fix remains an open unmerged PR.

## What This Changes

Report that an implicit daily reset during an in-flight auto-reply turn can archive the live transcript path and split the turn's output.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/96715 or an equivalent fix that defers the entire implicit daily rollover while the matching reply run is active.

## Reproduction Assessment

Yes at source level: current main still evaluates daily staleness before the run gate and archives stale session transcripts, while transcript writers append by path.

## Solution Assessment

Yes: deferring the whole implicit rollover while the active run owns the session is the narrow owner-boundary fix because it preserves both the session store entry and live transcript path.
