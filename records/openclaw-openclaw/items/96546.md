## Label Justifications

- `P1`: A default daily-reset boundary can split an active turn's transcript and leave the resulting session disconnected from the output, breaking a real session workflow.
- `impact:data-loss`: The reported output remains on disk but is split into archived and orphaned files that the new session does not reference, making user/session data effectively lost from normal access paths.
- `impact:message-loss`: The running turn's post-boundary output can be written to an orphan transcript instead of the session that should contain the message history.
- `impact:session-state`: The session row rotates to a new session while the active run continues writing through the old path, creating session/transcript state drift.

## Summary

Keep open: current main still evaluates the default daily boundary inside inbound session initialization and can archive the previous transcript before the later active-run queue guard runs. The related daily-reset scheduler PR is adjacent, but it is open and does not cover this auto-reply inbound archive path.

## What This Changes

The issue reports that an implicit daily session rollover during an active inbound turn archives the live transcript before active-run handling, splitting one turn across archived and orphaned JSONL files.

## Best Possible Solution

Add a narrow guard for implicit stale daily/idle rollover in auto-reply session initialization so active runs are interrupted, waited on, or the archive is deferred before any transcript rename, with regression coverage proving the running turn stays in one referenced transcript.

## Reproduction Assessment

Yes at source level: current main still makes the daily stale decision in initSessionState before runPreparedReply's active-run gate, and the archive path renames the transcript while append helpers reopen by path. I did not run the full repro command in this read-only review.

## Solution Assessment

Yes: the best fix is a focused session lifecycle guard on the auto-reply inbound rollover path, not a broad scheduler-only change. The sibling compaction fix shows the intended invariant that destructive transcript mutation must not happen until active runs are stopped or bypassed.
