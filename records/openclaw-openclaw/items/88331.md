## Label Justifications

- `P1`: The report describes a real Discord agent workflow losing final assistant delivery after Codex app-server progress on the latest release.
- `impact:message-loss`: The failure mode ends the turn before the final assistant response is delivered to the channel.
- `impact:session-state`: The app-server turn is aborted as a completion-idle timeout after current-turn progress, leaving the session outcome inconsistent with observed Codex activity.

## Summary

Keep open: current main still limits the longer post-reasoning reply window to `message_tool_only`, while automatic assistant delivery after no-tool reasoning/raw reasoning falls back to the short completion-idle timeout. The related post-tool PR addresses a sibling path but does not cover this no-tool automatic-reply case.

## What This Changes

The issue requests that native Codex app-server automatic assistant-reply turns with current-turn reasoning or non-final commentary progress use the existing post-reasoning reply window instead of the short completion-idle timeout.

## Best Possible Solution

Implement a focused Codex app-server watchdog change that treats current-turn reasoning/commentary progress in automatic assistant delivery as post-reasoning reply progress, while preserving fast failure for accepted no-progress turns.

## Reproduction Assessment

Yes from source inspection, but not from a live Discord run during this read-only review. Current main has a test asserting non-source reasoning completion still uses the short completion-idle guard, matching the reported failure shape.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing post-reasoning reply guard for automatic assistant delivery after current-turn reasoning/commentary progress, while keeping the short guard for accepted turns with no progress.
