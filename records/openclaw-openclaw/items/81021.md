## Summary

## What This Changes

The PR adds a shared CLI backend output digest helper, logs `outBytes` and short SHA-256 `outHash` fields for Claude live-session turns and synthetic cron `before_agent_reply` replies, and adds focused tests.

## Best Possible Solution

Land a narrow implementation that records stable byte count and content fingerprint for both live Claude turns and synthetic cron replies without logging response text.

## Reproduction Assessment

Yes. Source inspection on current main shows the Claude live-session turn log has no `outBytes` or `outHash`, and the handled cron `before_agent_reply` path returns before any comparable CLI backend turn log.

## Solution Assessment

Yes. A shared digest helper used at the two completion sites is the narrowest maintainable solution for this diagnostic surface, and the tests check that raw assistant text is not logged.
