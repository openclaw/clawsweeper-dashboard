## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Preserve custom homes outside external /Users paths:** `src/daemon/launchd-path.ts:81-82`
  - body: This fallback uses USER/LOGNAME/USERNAME whenever HOME is not already `/Users/<name>` or `.../Users/<name>`. A custom or network home such as `/var/root` or `/Volumes/HomeDisk/alice` would now manage the LaunchAgent under `/Users/<user>/Library/LaunchAgents` instead of the actual configured home, so install/restart/uninstall can target the wrong plist.
  - confidence: 0.9
- **[P2] Keep the Swift mirror from remapping arbitrary homes:** `apps/macos/Sources/OpenClaw/GatewayLaunchAgentManager.swift:36-47`
  - body: The macOS app resolver mirrors the same broad fallback: any non-`/Users/<name>` home with a short username is redirected to `/Users/<shortUserName>`. That can make the companion app inspect or remove a different LaunchAgent than the user's configured home uses.
  - confidence: 0.86
