## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Preserve homes that are not external user homes:** `src/daemon/launchd-path.ts:81-82`
  - body: For a custom home such as `/Volumes/HomeDisk/alice` or `/var/root`, this fallback uses `USER`/`LOGNAME` and returns `/Users/<name>` even though the configured home is not an external `/.../Users/<name>` account home. That changes install/restart/uninstall to operate on an unrelated plist path; only proven external user-home cases should remap, otherwise keep `home`.
  - confidence: 0.89
- **[P2] Align the app snapshot resolver with the daemon policy:** `apps/macos/Sources/OpenClaw/GatewayLaunchAgentManager.swift:36-47`
  - body: The Swift app now mirrors the same broad short-name fallback: any non-`/Users` home with `NSUserName()` resolves to `/Users/<name>`. If the daemon preserves a custom same-volume or network home, the app will read a different plist for token and log discovery; this reader should use the accepted daemon policy or keep `homeDirectoryForCurrentUser`.
  - confidence: 0.8
