## Summary

Keep open: current main still source-reproduces the bundled capability contract propagation bug, and https://github.com/openclaw/openclaw/pull/81312 is an active open PR with closing syntax for this issue.

## What This Changes

The issue reports that bundled capability plugins such as Feishu/Lark lose manifest `contracts.tools` when `createCapabilityPluginRecord` synthesizes runtime records.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81312 or an equivalent narrow patch that carries manifest contracts into bundled capability records with regression coverage.

## Reproduction Assessment

Yes at source level. Current main creates bundled capability records without `contracts`, then validates captured tools against `record.contracts` while Feishu declares its tools in the manifest.

## Solution Assessment

Yes. Propagating `manifest.contracts` into `createCapabilityPluginRecord` matches the regular loader path and fixes the broken existing plugin contract without adding a new plugin API.
