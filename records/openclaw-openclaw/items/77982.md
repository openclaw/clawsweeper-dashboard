## What This Changes

The issue requests preserving plugin manifest tool contracts through bundled capability plugin record creation so Lark/Feishu tools register correctly.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81312 or an equivalent narrow patch that carries manifest contracts into bundled capability records with regression coverage.

## Reproduction Assessment

Yes at source level. Current main creates bundled capability records without `contracts`, then validates captured tools against `record.contracts` while Feishu declares its tools in the manifest.

## Solution Assessment

Yes. Propagating `manifest.contracts` into the synthetic bundled record matches the regular loader path and fixes the broken existing plugin contract without adding a new API.
