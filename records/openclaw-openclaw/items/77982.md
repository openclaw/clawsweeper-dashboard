## Summary

Keep open: current main still drops manifest contracts in the bundled capability runtime, so declared tool contracts are not visible when captured tools are validated.

## What This Changes

The issue asks OpenClaw to propagate `contracts.tools` into bundled capability runtime plugin records so Feishu/Lark tools declared by a plugin can register.

## Best Possible Solution

Propagate `manifestRecord.contracts` into `createCapabilityPluginRecord`, preserve contract enforcement, and add a bundled capability runtime regression test for a plugin that declares and registers tools.

## Reproduction Assessment

Yes, by source inspection: a bundled capability plugin with `contracts.tools` and `registerTool` reaches validation with `record.contracts` undefined. I did not run a live runtime reproduction in this read-only sweep.

## Solution Assessment

Yes, the narrow fix is to keep the existing `contracts.tools` enforcement and pass the manifest contracts through the bundled capability record, matching the normal loader path.

In `src/plugins/bundled-capability-runtime.ts`, pass `manifestRecord.contracts`/`manifest.contracts` through `createCapabilityPluginRecord` and store it on the returned `PluginRecord`, matching `createPluginRecord` behavior without weakening `contracts.tools` enforcement. Add focused regression coverage for `loadBundledCapabilityRuntimeRegistry` proving a bundled capability plugin with manifest `contracts.tools` can register a declared tool and still rejects undeclared tools.
