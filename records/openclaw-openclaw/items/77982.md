## Summary

Keep open: current main still drops manifest contracts when building bundled capability runtime records, so captured tool registrations can be treated as undeclared even though the manifest registry has the contracts available.

## What This Changes

The issue requests that plugin tool contracts survive the bundled capability runtime path so openclaw-lark/Feishu tools can register instead of producing contracts.tools diagnostics.

## Best Possible Solution

Propagate manifestRecord.contracts into the bundled capability PluginRecord and add a regression test for a bundled capability plugin that declares tools in its manifest and registers them at runtime.

## Reproduction Assessment

Yes from source inspection: current main creates the bundled capability PluginRecord without contracts, then validates captured tools against record.contracts. A live openclaw-lark run was not performed in this read-only sweep.

## Solution Assessment

Yes for the title/root cause: passing manifestRecord.contracts through createCapabilityPluginRecord is the narrowest fix and matches the normal loader path. If openclaw-lark only declares contracts in its entry export, that plugin should add manifest contracts or a separate SDK change should be discussed.
