## Summary

Keep open: current main and the latest release still drop bundled capability manifest contracts before captured tool validation, and the open closing PR is the active focused fix path.

## What This Changes

The issue reports that bundled capability plugin records omit `contracts`, causing Feishu/Lark tools declared in `contracts.tools` to be rejected as undeclared.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81312, or deliberately fold the same manifest-contract propagation and regression coverage into the broader hardening PR before closing this issue.

## Reproduction Assessment

Yes, source-level: current main creates bundled capability PluginRecords without `contracts` and then validates captured tools against `record.contracts`, while the Feishu/Lark manifest declares the affected tools.

## Solution Assessment

Yes, with sequencing review: preserving `manifest.contracts` on the synthesized bundled PluginRecord is the narrow manifest-first fix and aligns the bundled runtime with the normal loader record path.
