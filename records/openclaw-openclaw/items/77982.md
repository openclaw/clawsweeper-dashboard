## Summary

Keep open: current main and v2026.6.10 still drop bundled capability manifest contracts before tool-contract validation, and https://github.com/openclaw/openclaw/pull/81312 is the open closing PR that owns the focused fix path.

## What This Changes

The issue reports that bundled capability plugin records drop `contracts.tools`, causing declared Lark/Feishu tools to be rejected during registration.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81312 if its manifest-first contract propagation remains correct after maintainer review and current checks.

## Reproduction Assessment

Yes, source-level. Current main creates bundled capability PluginRecords without `contracts` and then validates captured tools against `record.contracts`, while the Feishu/Lark manifest declares affected tools.

## Solution Assessment

Yes. Preserving `manifest.contracts` on the synthesized bundled PluginRecord is the narrow manifest-first fix and aligns the bundled runtime with the normal loader record path.
