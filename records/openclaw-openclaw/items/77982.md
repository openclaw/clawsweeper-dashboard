## Summary

Keep open: current main still drops manifest contracts in the bundled capability runtime path, and open linked PRs already use closing syntax for this issue, so the issue should close only after one fix lands.

## What This Changes

Ensure bundled capability plugin records preserve manifest `contracts.tools` so Feishu/Lark and similar plugin tools can register.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/81312 or an equivalent narrow patch that propagates manifest contracts into bundled capability records with regression coverage, then let the closing PR resolve this issue.

## Reproduction Assessment

Yes. Current main creates bundled capability `PluginRecord`s without `contracts`, then validates captured tools against `record.contracts`, while Feishu declares its tools in the manifest.

## Solution Assessment

Yes. Propagating `manifest.contracts` into `createCapabilityPluginRecord` is the narrow maintainable fix and matches the regular loader path; the issue should remain open until the linked PR or equivalent fix lands.
