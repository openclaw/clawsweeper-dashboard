## What This Changes

The issue requests fixing channel shutdown so a stop timeout does not leave stale task/abort bookkeeping that blocks health-monitor restarts.

## Best Possible Solution

Land PR #72912 or an equivalent gateway lifecycle fix that force-retires health-monitor-wedged channel tasks, clears stale bookkeeping, marks retired runtime state accurately, gates late callbacks, and updates focused regression coverage plus changelog.

## Reproduction Assessment

Yes. Current main contains a unit-level reproduction in src/gateway/server-channels.test.ts where a never-settling task times out on stop, a second start is attempted, and the test expects no replacement start while runtime remains running with the timeout error.

## Solution Assessment

Yes for the forced-retirement direction in PR #72912, but the minimal two-map-delete proposal is not enough by itself. The safer maintainable fix should also prevent callbacks from an old timed-out task from overwriting the replacement lifecycle.
