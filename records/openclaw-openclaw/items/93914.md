## Best Possible Solution

Keep the watchdog-layer fix direction, but exclude `runner_entered` alone from clearing the pre-execution guard while allowing later setup phases to count as progress, then add redacted real cron proof.

## Solution Assessment

No. The watchdog-layer repair is plausible, but this implementation is too broad because it treats runner entry itself as enough progress to disable the pre-execution stall guard.
