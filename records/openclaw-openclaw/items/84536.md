## Summary

Keep open: current main has partial preemptive-overflow compaction and terminal error handling, but this report describes a plausible remaining user-visible failure where recovery never resumes and no channel error is delivered. The closest canonical work is still open, so closing this now would risk losing a fresh multi-agent field report.

## What This Changes

The issue asks for embedded preemptive context-overflow failures to either compact and resume or visibly notify the user instead of leaving webchat/Telegram sessions stuck in processing.

## Best Possible Solution

Make preemptive and mid-turn overflow paths reliably compact-and-resume or emit a visible terminal channel error while clearing processing/lane state, then close this issue only after the canonical stuck-session work proves that behavior.

## Reproduction Assessment

No, not end-to-end with high confidence. The issue provides concrete logs and current source shows plausible precheck-overflow, compaction, and stale-processing paths, but I did not reproduce the silent delivery failure in a live MiniMax/webchat or Telegram setup.

## Solution Assessment

Unclear. A fix should target terminal delivery and liveness cleanup around the embedded overflow path, not only expose the overflow ratio or tune compaction policy.
