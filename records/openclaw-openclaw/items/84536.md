## What This Changes

The issue reports that embedded webchat/Telegram sessions hit preemptive context overflow during long tool loops, log a terminal error, then remain stuck without a user-visible channel message.

## Best Possible Solution

Make embedded overflow terminal paths either compact and resume or visibly deliver the final error to WebChat/Telegram while clearing active run and session liveness state, with regression coverage for runner result-to-channel delivery.

## Reproduction Assessment

No, not as a high-confidence end-to-end reproduction. The issue now has concrete steps/logs and current source shows a plausible overflow-to-undelivered-WebChat-final path, but I did not run a live MiniMax/WebChat or Telegram scenario.

## Solution Assessment

Unclear from current evidence whether changing the overflow ratio would solve the user-visible failure. The better fix is to prove and repair terminal delivery/liveness cleanup for overflow paths before adding config or policy knobs.
