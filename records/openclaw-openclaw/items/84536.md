## Summary

Keep open: current main has recovery code for preemptive context-overflow paths, but this fresh report describes a remaining user-visible failure where the session stays stuck and no channel error is delivered after the precheck/compaction path.

## What This Changes

The issue reports that preemptive context-overflow handling can terminate embedded agent sessions without notifying the channel or clearing the processing state.

## Best Possible Solution

Make the preemptive and mid-turn overflow terminal path reliably either compact-and-resume or deliver a visible channel error while clearing the processing/lane state, with regression coverage across the embedded runner and at least one channel delivery path.

## Reproduction Assessment

No, not as a high-confidence end-to-end reproduction. The source and logs show a plausible precheck-overflow, compaction, and stale-processing path, but I did not run a live MiniMax/webchat or Telegram scenario to reproduce the silent channel failure.

## Solution Assessment

Unclear whether the existing recovery path is sufficient. The next fix should prove and repair terminal delivery/liveness cleanup instead of only changing the hardcoded overflow ratio or adding a config knob.
