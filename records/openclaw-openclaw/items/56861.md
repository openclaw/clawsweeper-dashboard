## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep attachment encoding behind apiMode:** `extensions/signal/src/send.ts:157-163`
  - body: This adds a baseUrl hostname heuristic in `sendMessageSignal`, but current main already routes Signal sends through `cfg.channels.signal.apiMode` and keeps container base64 conversion inside `client-container.ts`. Rebasing this as-is would create a competing source of truth that misclassifies common proxied or tunneled deployments and can change attachment behavior for existing setups.
  - confidence: 0.9
- **[P2] Read quoted media from quote thumbnails:** `extensions/signal/src/monitor/event-handler.ts:755-761`
  - body: signal-cli quote attachments are `JsonQuotedAttachment` records with an optional `thumbnail` attachment, not regular inbound `SignalAttachment` objects with a top-level `id`. Looking for `quote.attachments[].id` means quoted image/media context will normally never be fetched.
  - confidence: 0.87
- **[P2] Route quoted media through MediaPaths:** `extensions/signal/src/monitor/event-handler.ts:215-217`
  - body: The PR stores fetched quoted media only in `ReplyToMediaPath`/`ReplyToMediaType`, but OpenClaw's media-understanding path normalizes media from `MediaPath`/`MediaPaths`/`MediaUrl`/`MediaUrls`. The agent still will not receive quoted images as media unless a custom template reads these new ad-hoc fields.
  - confidence: 0.88
