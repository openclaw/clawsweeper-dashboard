Full review comments:

- **[P1] Route quoted media through the media pipeline:** `extensions/signal/src/monitor/event-handler.ts:217`
  - body: The new quoted attachment path only stores the downloaded file in `ReplyToMediaPath`/`ReplyToMediaType`, but model media ingestion reads `MediaPath`/`MediaPaths` or the shared supplemental quote-media finalizer. A user replying to an image still will not give the model that image, so the headline reply-target media feature silently misses the media pipeline.
  - confidence: 0.89
- **[P2] Check quote author and UUID independently:** `extensions/signal/src/monitor/event-handler.ts:658`
  - body: `quote.author ?? quote.authorUuid` discards `authorUuid` whenever a phone author is present. In UUID-only bot identity setups, a reply to the bot that includes both fields can fail the phone comparison and never check the matching UUID, so reply-to-bot implicit mentions can still be skipped in requireMention groups.
  - confidence: 0.83
