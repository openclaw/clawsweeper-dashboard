## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Route quoted media through the shared media context:** `extensions/signal/src/monitor/event-handler.ts:215-217`
  - body: These new ReplyToMedia* fields are not consumed by the inbound media pipeline. Current main projects media into MediaPath/MediaPaths from `params.media` or `supplemental.quote.media`, and media understanding only reads those fields, so a replied-to image/audio can be downloaded but still unavailable to the model.
  - confidence: 0.91
- **[P2] Compare quote author and UUID independently:** `extensions/signal/src/monitor/event-handler.ts:657`
  - body: `quote.author ?? quote.authorUuid` ignores the UUID whenever a phone author is present. In UUID-only or phone-hidden bot identity setups, a reply to the bot can fail the implicit mention check and be skipped in mention-gated groups.
  - confidence: 0.84
- **[P2] Use the resolved MIME type for data URIs:** `extensions/signal/src/send.ts:16-35`
  - body: `localFileToDataUri` guesses MIME type from the file extension even though `resolved.contentType` is available at the call site. Temp or cached attachment paths can have no useful extension, producing `application/octet-stream` data URIs for known media.
  - confidence: 0.8
- **[P3] Update the Signal native mention docs:** `extensions/signal/src/monitor/event-handler.ts:638`
  - body: This changes Signal native mention metadata from unsupported to supported behavior, but `docs/channels/signal.md` still tells users Signal does not support native mentions. If this lands, the docs should describe the supported scope accurately.
  - confidence: 0.78
