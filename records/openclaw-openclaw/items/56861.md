- **[P2] Route quoted media through the shared media path:** `extensions/signal/src/monitor/event-handler.ts:215-216`
  - body: The quoted attachment is only exposed as `ReplyToMediaPath`/`ReplyToMediaType`, but the model media pipeline normalizes media from `MediaPath`/`MediaPaths`/`MediaUrl`/`MediaUrls`. Use the current supplemental quote media resolver or append the quote media to the shared media facts; otherwise prompts about the quoted image/audio still will not receive that media.
  - confidence: 0.9
- **[P2] Check quote phone and UUID independently:** `extensions/signal/src/monitor/event-handler.ts:657`
  - body: `quote.author ?? quote.authorUuid` throws away the UUID whenever a phone author is also present. In UUID-only bot deployments, a reply to the bot can therefore fail `isReplyToBot` and be skipped by requireMention gating even though the quote carries the matching `authorUuid`.
  - confidence: 0.84
- **[P2] Avoid guessing remote attachment mode from baseUrl:** `extensions/signal/src/send.ts:160`
  - body: The local-vs-remote decision depends on whether `baseUrl` looks like localhost, but SSH tunnels and proxies can expose a remote signal-cli on loopback, while non-loopback URLs can still have shared file access. This can keep remote attachments broken or change working setups without an explicit transport/config contract.
  - confidence: 0.78
