## Summary

Keep open. Current main still lacks the PR's central Signal behavior, and the PR remains useful, but the latest diff would not fully deliver quoted-media model context and can miss reply-to-bot detection in UUID-only configurations.

## What This Changes

The PR adds Signal quoted-message attachment handling, structured mention and reply-to-bot mention detection, and data-URI outbound attachments for non-local signal-cli daemons.

## Best Possible Solution

Port the useful Signal changes onto current main, route quoted media through the existing media context fields, compare quote phone and UUID independently, preserve local file-path sends, and cover the behavior with focused Signal tests.

## Reproduction Assessment

Yes. The quoted-media bug follows from the PR adding only `ReplyToMediaPath` while `normalizeAttachments` ignores that key, and the UUID bug follows from `quote.author ?? quote.authorUuid` short-circuiting before UUID comparison when both fields are present.

## Solution Assessment

No. The direction is reasonable, but the current diff is not the narrowest maintainable implementation because it adds ad-hoc reply media fields instead of using the established `MediaPaths` contract and collapses quote identity before matching bot UUIDs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Route quoted media through MediaPaths:** `extensions/signal/src/monitor/event-handler.ts:216-217`
  - body: The PR downloads the quoted attachment but exposes it only as `ReplyToMediaPath`/`ReplyToMediaType`. `normalizeAttachments` ignores those keys and only consumes `MediaPath(s)`/`MediaUrl(s)`, so quoted image/audio still will not reach model media processing. Append quoted media to the normal media arrays while preserving reply metadata.
  - confidence: 0.9
- **[P2] Check quote UUID independently:** `extensions/signal/src/monitor/event-handler.ts:657`
  - body: `dataMessage.quote?.author ?? dataMessage.quote?.authorUuid` ignores `authorUuid` whenever `author` is present. In UUID-only bot configurations, a reply to the bot can be misclassified as not mentioning the bot and skipped by mention gating. Compare `quote.author` and `quote.authorUuid` separately.
  - confidence: 0.86
