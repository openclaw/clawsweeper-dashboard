## Summary

Keep open. The Signal direction is useful, but the current PR head is not merge-ready because quoted media is read from the wrong shape, fetched quoted media is not routed through the existing media-ingestion contract, reply-to-bot identity matching can miss UUID-only setups, and no redacted real Signal runtime proof is present.

## What This Changes

The PR adds Signal quoted-message media fetching, structured mention/reply-to-bot detection, and conditional data URI encoding for outbound attachments.

## Best Possible Solution

Port the useful Signal changes onto current main using upstream-shaped quote attachments, existing MediaPaths/MediaTypes ingestion, API-mode-owned attachment encoding, focused tests, and redacted live Signal proof.

## Reproduction Assessment

Yes for the blocking PR findings: the PR-head source can be checked against upstream signal-cli JsonQuote/JsonQuotedAttachment shape and current-main media normalization. I did not run a live Signal setup in this read-only review.

## Solution Assessment

No. The direction is reasonable, but the current patch should not merge until it follows the upstream quote contract, routes quoted media through existing ingestion fields, compares phone and UUID identity separately, and includes real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Read quoted media from the nested thumbnail:** `extensions/signal/src/monitor/event-handler.ts:797`
  - body: signal-cli models quote attachments as JsonQuotedAttachment, where the downloadable id lives on the nested thumbnail JsonAttachment rather than on the quote attachment itself. This top-level id search will skip real quoted media, so reply-target media usually will not be fetched.
  - confidence: 0.9
- **[P2] Route quoted media through MediaPaths:** `extensions/signal/src/monitor/event-handler.ts:219-220`
  - body: The PR stores fetched quoted media only in ReplyToMediaPath/ReplyToMediaType, but the media-understanding pipeline normalizes MediaPath(s) and MediaUrl(s). Even a successful quote fetch will not reach model media processing.
  - confidence: 0.92
- **[P2] Compare quote phone and UUID independently:** `extensions/signal/src/monitor/event-handler.ts:660`
  - body: Using quote.author ?? quote.authorUuid ignores the UUID whenever author is present. UUID-only bot configurations can miss replies to the bot and then skip mention-gated group messages.
  - confidence: 0.86
- **[P3] Preserve the resolved MIME type for data URIs:** `extensions/signal/src/send.ts:16-34`
  - body: The helper re-derives MIME type from the local filename even though resolved.contentType is available at the call site. Extensionless or temporary cached files can be sent as application/octet-stream despite a known content type.
  - confidence: 0.78
