## Summary

Keep open. Current main still lacks the PR's central Signal improvements, but the PR is not ready to merge because quoted media is exposed through ad-hoc fields that the shared media pipeline does not consume, reply-to-bot matching can miss UUID-only bot identity cases, and the branch is currently dirty against main.

## What This Changes

The PR updates the Signal monitor, Signal event types, and outbound send path to add quoted-message attachment handling, structured Signal mention/reply-to-bot detection, and data URI attachment sending for non-local signal-cli daemons.

## Best Possible Solution

Update or replace the PR with a current-main-compatible Signal implementation that routes quoted attachments through the standard media context fields, preserves quote metadata, checks quote phone and UUID identities independently, adds focused Signal tests, and includes the required changelog entry.

## Reproduction Assessment

Yes. The quoted-media bug is reproducible by adding a Signal quote attachment test and observing that normalizeAttachments ignores ReplyToMediaPath, while the UUID bug follows from a quote payload containing both author and authorUuid with only accountUuid configured.

## Solution Assessment

No. The feature direction is reasonable, but this patch should use the existing media ingestion contract and independent identity comparisons rather than adding unconsumed reply-media fields and collapsing quote identity to one value.
