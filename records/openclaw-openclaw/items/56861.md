## Summary

Keep open. Current main does not yet provide the central Signal behavior, and this PR still has blocking correctness issues: quoted media is downloaded into ad-hoc reply fields that the media-understanding pipeline does not consume, and reply-to-bot mention detection can miss UUID-only bot accounts. The earlier review comments were useful context; the latest PR commit addresses the unconditional data-URI and can-detect-mention concerns, but not all remaining correctness gaps.

## What This Changes

The PR adds Signal quoted-message media handling, structured mention and reply-to-bot mention detection, and data-URI outbound attachments for non-local signal-cli daemons.

## Best Possible Solution

Keep the PR open, but require a focused update that adapts it to current main, routes quoted Signal media through the standard media context fields, preserves reply metadata, and covers structured mention, UUID-only reply-to-bot, quoted-media, and remote attachment behavior with targeted Signal tests.

## Reproduction Assessment

Yes. The quoted-media bug is reproducible by adding a Signal quote attachment test and observing that `normalizeAttachments` only reads `MediaPath(s)`/`MediaUrl(s)`, while the UUID reply-to-bot bug follows from a quoted message with both `author` and `authorUuid` under an `accountUuid`-only config.

## Solution Assessment

No. The requested direction is reasonable, but this implementation should compare quote phone and UUID independently and feed quoted media into the existing media ingestion contract instead of adding unconsumed `ReplyToMediaPath` fields.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Route quoted media through MediaPaths:** `extensions/signal/src/monitor/event-handler.ts:216-217`
  - body: The PR downloads the quoted attachment but only exposes it as `ReplyToMediaPath`/`ReplyToMediaType`. The media-understanding pipeline normalizes attachments exclusively from `MediaPath(s)` and `MediaUrl(s)`, so prompts about the quoted image/audio still won't receive that media. Append quoted media to the normal media arrays while keeping reply metadata for templates.
  - confidence: 0.9
- **[P2] Check quote UUID independently:** `extensions/signal/src/monitor/event-handler.ts:657`
  - body: `dataMessage.quote?.author ?? dataMessage.quote?.authorUuid` ignores `authorUuid` whenever `author` is present. In UUID-only bot configurations supported by this extension, a reply to the bot can fail `isReplyToBot` and be skipped by mention gating. Compare `quote.author` and `quote.authorUuid` separately before deciding.
  - confidence: 0.86
