## Summary

Keep open: current main does not implement the quoted-media and remote native signal-cli attachment parts, while the PR still has blocking source issues and no contributor-supplied real Signal proof. ## What This Changes

The PR changes the Signal plugin event handler/types and send path to add quoted media context, structured mention/reply-to-bot detection, and data URI encoding for remote signal-cli attachment sends.

## Best Possible Solution

Port the useful Signal changes onto current main using upstream-shaped quote thumbnails, existing MediaPaths/MediaTypes ingestion, independent phone/UUID matching, API-mode-owned attachment encoding, focused tests, and redacted live Signal proof.

## Reproduction Assessment

Yes for the blocking PR findings: the PR-head source can be checked against upstream signal-cli quoted attachment types and current-main media normalization. I did not run a live Signal setup in this read-only review.

## Solution Assessment

No. The direction is useful, but this patch should not merge until it follows the upstream quote contract, routes quoted media through existing ingestion fields, compares phone and UUID identity separately, and includes real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Read quoted media from the thumbnail:** `extensions/signal/src/monitor/event-handler.ts:797`
  - body: signal-cli models quote attachments as a wrapper with the downloadable JsonAttachment under `thumbnail`. Searching for a top-level `id` means real quoted media is skipped before `fetchAttachment` runs.
  - confidence: 0.9
- **[P2] Route quoted media through MediaPaths:** `extensions/signal/src/monitor/event-handler.ts:219-220`
  - body: `ReplyToMediaPath` and `ReplyToMediaType` are not consumed by `normalizeAttachments`; model media processing reads `MediaPath(s)` and `MediaUrl(s)`, so downloaded quoted media will not reach the agent.
  - confidence: 0.92
- **[P2] Compare quote phone and UUID independently:** `extensions/signal/src/monitor/event-handler.ts:660`
  - body: Using `quote.author ?? quote.authorUuid` skips UUID matching whenever `author` is present. UUID-only bot configs can miss replies to the bot and then skip mention-gated group messages.
  - confidence: 0.86
- **[P3] Preserve the resolved MIME type in data URIs:** `extensions/signal/src/send.ts:16-34`
  - body: The helper re-derives MIME type from the filename even though attachment resolution already returns `contentType`; extensionless cached files can be sent as `application/octet-stream` despite a known type.
  - confidence: 0.78
