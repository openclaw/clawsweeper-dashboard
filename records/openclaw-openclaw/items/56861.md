## Summary

Keep open: the Signal direction is useful, but the current PR should not merge as-is because quoted media handling does not match signal-cli's upstream shape, downloaded reply media bypasses current model media ingestion, quote identity matching is lossy, the branch conflicts with current main, and no real Signal proof is present.

## What This Changes

The PR updates the Signal plugin event handler/types and send path to add quoted-message media context, structured mention/reply-to-bot detection, and conditional data URI encoding for outbound attachments.

## Best Possible Solution

Port the useful Signal changes onto current main using upstream quote.thumbnail data, existing MediaPaths/MediaTypes ingestion, independent phone/UUID matching, API-mode-owned attachment encoding, focused tests/docs, and redacted live Signal proof.

## Reproduction Assessment

Yes for the blocking PR findings: the PR-head source can be checked against upstream signal-cli quoted attachment types and current-main media normalization. I did not run a live Signal setup in this read-only review.

## Solution Assessment

No. The direction is useful, but this patch should not merge until it follows the upstream quote contract, routes quoted media through existing ingestion fields, compares phone and UUID identity separately, and includes real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Read quoted media from thumbnail:** `extensions/signal/src/monitor/event-handler.ts:797`
  - body: signal-cli models quote attachments as wrappers with the downloadable JsonAttachment under `thumbnail`. Looking for a top-level `id` means real quoted media is skipped before `fetchAttachment` runs.
  - confidence: 0.91
- **[P2] Route quoted media through MediaPaths:** `extensions/signal/src/monitor/event-handler.ts:870-871`
  - body: The new reply media fields are not consumed by `normalizeAttachments`, which reads `MediaPath(s)` and `MediaUrl(s)`. Downloaded quoted images/audio will not reach model media processing.
  - confidence: 0.92
- **[P2] Compare quote phone and UUID independently:** `extensions/signal/src/monitor/event-handler.ts:660`
  - body: Using `quote.author ?? quote.authorUuid` skips UUID matching whenever a phone author exists. UUID-only bot configs can miss replies to the bot and then skip mention-gated group messages.
  - confidence: 0.86
- **[P3] Preserve the resolved MIME type in data URIs:** `extensions/signal/src/send.ts:164`
  - body: The call only passes the resolved file path, so the helper re-derives MIME from the extension even though `resolved.contentType` is already known. Extensionless cached files can be sent as `application/octet-stream`.
  - confidence: 0.78
