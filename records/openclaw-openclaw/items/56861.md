## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Route quoted media through normalized media fields:** `extensions/signal/src/monitor/event-handler.ts:218-220`
  - body: The PR downloads quoted attachments but exposes them only as `ReplyToMediaPath` and `ReplyToMediaType`. Current media ingestion reads `MediaPath`/`MediaPaths`/`MediaUrl`/`MediaUrls` or the shared `supplemental.quote.media` path, so the quoted image/audio still will not reach model media processing unless a custom template knows these ad-hoc keys.
  - confidence: 0.91
- **[P2] Compare quote author and UUID independently:** `extensions/signal/src/monitor/event-handler.ts:660-667`
  - body: `quote.author ?? quote.authorUuid` means `authorUuid` is never checked when `author` exists. In UUID-backed bot identity setups, a reply to the bot can be treated as not mentioning the bot and then skipped by Signal group mention gating.
  - confidence: 0.88
- **[P3] Preserve resolved content type in data URIs:** `extensions/signal/src/send.ts:16-35`
  - body: `localFileToDataUri` derives MIME type from the file extension even though `resolved.contentType` is available at the call site. Cached or temporary attachment files can lack a useful extension, causing remote Signal sends to encode the right bytes with a generic or wrong MIME type.
  - confidence: 0.75
