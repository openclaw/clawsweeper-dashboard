## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep attachment encoding at the Signal API boundary:** `extensions/signal/src/send.ts:163-164`
  - body: Current main already routes sends through the Signal client adapter, and container mode converts file paths to base64 data URIs inside `client-container.ts`. Reintroducing hostname-based conversion in `send.ts` creates a competing source of truth; after rebase it can either bypass native/local file semantics or hand a data URI to container code that expects to read a file path.
  - confidence: 0.88
- **[P2] Read quoted media from quote thumbnails:** `extensions/signal/src/monitor/event-handler.ts:797`
  - body: signal-cli wraps quoted attachments as `JsonQuotedAttachment`, where the downloadable `JsonAttachment` with an id is under `thumbnail`. Looking for a top-level id skips real quoted media before `fetchAttachment` can run.
  - confidence: 0.9
- **[P2] Route quoted media through MediaPaths:** `extensions/signal/src/monitor/event-handler.ts:216-217`
  - body: The patch stores quoted attachments only in `ReplyToMediaPath`/`ReplyToMediaType`, but OpenClaw media normalization consumes `MediaPath(s)` and `MediaUrl(s)`. The agent will not receive the quoted image/audio for model media processing.
  - confidence: 0.92
- **[P2] Compare quote phone and UUID independently:** `extensions/signal/src/monitor/event-handler.ts:660`
  - body: Collapsing `quote.author` and `quote.authorUuid` skips UUID matching whenever the deprecated author field is present, and the upstream contract also exposes `authorNumber`. UUID-only or number-specific bot configs can miss reply-to-bot and then drop mention-gated group replies.
  - confidence: 0.84
- **[P3] Preserve resolved MIME type in data URIs:** `extensions/signal/src/send.ts:16-35`
  - body: `localFileToDataUri` derives MIME type from the filename even though attachment resolution already has `contentType`. Extensionless cached files can be sent as `application/octet-stream` despite a known media type.
  - confidence: 0.78
