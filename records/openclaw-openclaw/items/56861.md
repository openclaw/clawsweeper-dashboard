## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Route quoted media through inbound media facts:** `extensions/signal/src/monitor/event-handler.ts:219-220`
  - body: Quoted attachments are downloaded but then exposed only as `ReplyToMediaPath`/`ReplyToMediaType`. The media-understanding and prompt media paths consume `MediaPath`/`MediaPaths` or current main's supplemental quote media seam, so the advertised quoted-image/audio context still will not reach the model unless users add custom templates for these ad-hoc fields.
  - confidence: 0.9
- **[P2] Compare quote author fields independently:** `extensions/signal/src/monitor/event-handler.ts:660-668`
  - body: `quote.author ?? quote.authorUuid` discards `authorUuid` whenever a phone author is present. In UUID-only bot account setups, a reply to the bot can be classified as not addressed to the bot and skipped by mention gating even though the payload includes the matching UUID.
  - confidence: 0.86
- **[P3] Use the resolved MIME type for data URIs:** `extensions/signal/src/send.ts:16-34`
  - body: `localFileToDataUri` re-infers MIME from the temp file extension even though the resolver already has `resolved.contentType`. Extensionless or generic cache files can be sent as `application/octet-stream`, which weakens the remote attachment compatibility this PR is trying to add.
  - confidence: 0.78
