## Summary

Keep open: the Signal direction is useful, but this PR is not mergeable as-is because quoted media is read from the wrong signal-cli shape, downloaded quote media bypasses OpenClaw media ingestion, reply-to-bot matching can miss UUID-only configs, and contributor real Signal proof is still absent.

## What This Changes

The PR modifies Signal inbound handling/types and send logic to add quoted-message media context, structured native mention/reply-to-bot detection, and data URI attachment sending for remote signal-cli.

## Best Possible Solution

Port the useful Signal pieces onto current main with upstream-shaped quote.thumbnail handling, MediaPaths/MediaTypes ingestion, independent phone/UUID matching, API-mode-owned attachment encoding, focused tests, docs where behavior changes, and redacted live Signal proof.

## Reproduction Assessment

Yes for the blocking PR findings: PR-head source, signal-cli JsonQuote/JsonQuotedAttachment source, and current-main normalizeAttachments provide a high-confidence source reproduction. I did not run a live Signal setup in this read-only review.

## Solution Assessment

No. The current patch should not merge until it follows the upstream quote contract, routes quoted media through existing media ingestion, compares phone and UUID identity separately, and includes real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Read quoted media from the thumbnail:** `extensions/signal/src/monitor/event-handler.ts:797`
  - body: signal-cli wraps quoted attachments as JsonQuotedAttachment; the downloadable JsonAttachment with id is under thumbnail. Searching for a top-level id means real quoted media is skipped before fetchAttachment runs.
  - confidence: 0.9
- **[P2] Route quoted media through MediaPaths:** `extensions/signal/src/monitor/event-handler.ts:219-220`
  - body: Downloaded quoted media is stored only in ReplyToMediaPath/ReplyToMediaType, but OpenClaw media normalization consumes MediaPath(s) and MediaUrl(s). The agent will not receive the quoted image/audio for model media processing.
  - confidence: 0.92
- **[P2] Compare quote phone and UUID independently:** `extensions/signal/src/monitor/event-handler.ts:660`
  - body: Collapsing quote.author and quote.authorUuid skips UUID matching whenever the legacy author field is present. UUID-only bot configs can miss reply-to-bot and then drop mention-gated group replies.
  - confidence: 0.86
- **[P3] Preserve resolved MIME type in data URIs:** `extensions/signal/src/send.ts:16-34`
  - body: The helper derives MIME type from the filename even though attachment resolution already has contentType. Extensionless cached files can be sent as application/octet-stream despite a known media type.
  - confidence: 0.78
