## Summary

Keep open: current main supports per-message Telegram silent delivery and silent error replies, but it does not have an account, group, or topic default-silent config; this is a new Telegram configuration feature that needs maintainer product and precedence review.

## What This Changes

Request a Telegram config option, optionally scoped per account, group, or topic, that makes outbound messages silent by default while allowing per-message notification overrides.

## Best Possible Solution

Add a Telegram-owned config design that keeps existing loud delivery as the default, clearly defines account/group/topic precedence, applies consistently across agent replies, auto-responses, tool results, and durable sends, and preserves explicit per-message overrides for urgent paths.
