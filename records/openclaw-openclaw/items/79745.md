## Best Possible Solution

Merge a version that keeps per-agent mcporter isolation but never persists secret-bearing configured env or authenticated remote URLs into generated config, with focused tests for stdio env secrets and URL query auth.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Avoid persisting configured stdio env secrets:** `extensions/memory-core/src/memory/qmd-manager.ts:2314`
  - body: `mcporter config get --json` serializes the configured server env, and this line copies every entry into the generated per-agent `mcporter.json`. If a custom stdio QMD server carries tokens, API keys, private endpoints, or other secret-bearing env, the PR duplicates that material into OpenClaw-managed agent state. Preserve only safe placeholders or use a non-persisted/external path for secret-bearing env before writing this file.
  - confidence: 0.91
- **[P2] Treat URL query credentials as auth material:** `extensions/memory-core/src/memory/qmd-manager.ts:210-211`
  - body: This check only treats URL username/password as credentials. A configured remote server such as `https://host/mcp?token=...` or `?api_key=...` will still be copied into the generated agent-scoped config, even though mcporter preserves the query string as part of `baseUrl`. Detect auth-shaped query parameters, or avoid generating a persisted config for any remote URL with sensitive query material.
  - confidence: 0.88
