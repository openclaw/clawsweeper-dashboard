## Summary

Keep this PR open, but it still needs maintainer review before merge because the core `--config` isolation approach is sound while one configured-stdio path can silently drop mcporter lifecycle/logging semantics, and the branch still carries unrelated UI i18n/script churn.

## What This Changes

The PR rewrites QMD mcporter execution to use per-agent generated mcporter configs and cleaned mcporter process envs, adds QMD/mcporter regression coverage, and includes unrelated UI i18n/generated metadata plus script formatting updates from rebasing.

## Best Possible Solution

Land the per-agent mcporter isolation only after configured stdio server lifecycle/logging is preserved losslessly or left external, and after maintainers explicitly accept or split the unrelated UI/script changes.

## Reproduction Assessment

Yes, from source and dependency inspection: current main passes qmd-scoped XDG env into mcporter, and mcporter 0.10 uses XDG_CONFIG_HOME for config discovery unless an explicit config path is supplied. I did not run a live gateway reproduction in this read-only pass.

## Solution Assessment

No, not yet. The explicit `--config` approach is the right layer for per-agent isolation, but rewriting configured stdio servers through lossy mcporter JSON is not the safest final fix.

Full review comments:

- **[P1] Preserve configured stdio lifecycle before generating:** `extensions/memory-core/src/memory/qmd-manager.ts:2467`
  - body: `mcporter config get --json` is a lossy representation: in mcporter v0.10.2 it serializes command/args/cwd/env/tool filters but not `lifecycle` or `logging`, and ordinary `qmd mcp` stdio servers default to no keep-alive unless raw lifecycle is present. This branch treats any configured `qmd mcp` server without env/auth-like material as generated, so an existing config with `lifecycle: "keep-alive"` or daemon logging is rewritten into OpenClaw-managed state without those settings. Please keep these configured stdio servers external unless the raw definition can be copied losslessly, or explicitly preserve the lifecycle/logging contract in generated config with coverage.
