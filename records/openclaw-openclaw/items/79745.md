## Summary

Keep this PR open but do not merge it yet. The branch is a useful fix for QMD/mcporter agent isolation, but the latest patch still misses two mcporter contract details and can rewrite configured servers that should remain external.

## What This Changes

The PR changes `memory-core` QMD mcporter config and spawn handling to use agent-scoped generated mcporter configs while preserving configured mcporter servers when they carry user-owned behavior.

## Best Possible Solution

Keep the per-agent generated mcporter config design, but make the preservation classifier match mcporter's real config discovery and raw schema before merge.

## Reproduction Assessment

Yes, source-reproducible from the patch and mcporter source: the PR-added raw probe omits the workspace project config layer and checks only top-level `logging.enabled`, while mcporter loads project config and defines daemon logging as `logging.daemon.enabled`.

## Solution Assessment

No, not yet. The per-agent generated config design is the right layer, but the implementation must preserve every configured mcporter server shape that mcporter itself can discover before this is the best fix.
