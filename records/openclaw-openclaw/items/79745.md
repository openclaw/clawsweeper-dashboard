## What This Changes

The PR changes memory-core QMD searches to generate agent-scoped mcporter config when safe, preserve external mcporter configs for env/auth cases, split QMD and mcporter process env, parse log-prefixed mcporter JSON, and stabilize one MCP runtime test wait.

## Best Possible Solution

Land this after maintainer acceptance that the generated/external mcporter config split is the intended upgrade-safe boundary for QMD memory search.

## Reproduction Assessment

Yes, source-reproducible: current main builds QMD-scoped XDG env and passes it to mcporter, while mcporter 0.10.2 uses XDG_CONFIG_HOME for config discovery when no explicit config path is supplied. I did not run the full live gateway failure locally in this read-only review.

## Solution Assessment

Yes, the current PR is a maintainable fix shape: it makes mcporter config selection explicit, keeps QMD env on the generated QMD server entry, and preserves env/auth-bearing user configs externally. The remaining question is maintainer acceptance of the compatibility-sensitive boundary, not a narrow code repair.
