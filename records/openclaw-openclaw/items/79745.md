## Summary

Keep open: this is a focused bug-fix PR, but it still drops keep-alive lifecycle for configured stdio qmd mcporter servers because it rebuilds the agent-scoped config from mcporter's serialized JSON, which omits lifecycle. The real behavior proof is sufficient for the main agent-isolation path, so the blocker is a code-path regression rather than missing proof.

## What This Changes

The PR generates per-agent QMD mcporter configs, passes them with `--config`, cleans QMD XDG variables from mcporter process env, preserves configured mcporter servers, parses noisy JSON output, adds qmd-manager tests, and updates the changelog.

## Best Possible Solution

Keep the per-agent config and clean-env approach, but reapply or preserve keep-alive lifecycle for configured stdio qmd servers and add a regression test where `mcporter config get --json` output lacks lifecycle.

## Reproduction Assessment

Yes. Source inspection plus mcporter v0.10.2 source show current main leaks QMD XDG env into mcporter, and the PR's configured-stdio path drops keep-alive when `config get --json` output omits lifecycle; I did not run a live daemon reproduction in this read-only pass.

## Solution Assessment

No, not as submitted. Passing an explicit per-agent `--config` and cleaning mcporter's process env is the right narrow direction, but configured stdio qmd entries must remain daemon-capable before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep configured qmd servers daemon-capable:** `extensions/memory-core/src/memory/qmd-manager.ts:2133-2136`
  - body: `toMcporterRawServerEntry` returns a configured stdio server after copying only the `mcporter config get --json` fields and injecting QMD env. mcporter v0.10.2 does not serialize `lifecycle`, so a configured qmd server that relies on `lifecycle: keep-alive` is written to the agent-scoped config without keep-alive and `mcporter daemon start --config ...` will not keep it warm. Please reapply keep-alive for stdio qmd entries while preserving the configured command/env fields.
  - confidence: 0.9
