## Summary

Keep open: current main still has the QMD/mcporter XDG leak, and this PR is a plausible focused fix with sufficient real terminal proof and no blocking review findings. It should continue through maintainer review and remaining CI rather than be closed or sent to a duplicate repair lane.

## What This Changes

The PR updates memory-core's QMD manager to generate per-agent mcporter configs, invoke mcporter with `--config` and a clean process env, preserve configured stdio/remote servers, parse log-prefixed JSON, and add focused tests/changelog.

## Best Possible Solution

Land the per-agent `--config` approach after normal maintainer review and completed CI, keeping configured-server preservation and keep-alive reapplication.

## Reproduction Assessment

Yes. ## Solution Assessment

Yes. Explicit per-agent mcporter configs plus a clean mcporter process env solve the XDG discovery regression without adding a new OpenClaw config knob, and the latest head preserves configured stdio/remote servers and keep-alive behavior.
