## Best Possible Solution

## Reproduction Assessment

Yes. The source path is high-confidence: configure memory.qmd.mcporter with a stdio qmd server, and the PR classifies it as external, starts mcporter without OpenClaw's per-agent QMD env, and mcporter's stdio transport inherits only process.env plus definition.env.

## Solution Assessment

No. Splitting mcporter config discovery from QMD process state is the right direction, but treating every configured stdio server as external drops the state isolation the PR is supposed to preserve.
