## Summary

Keep this PR open: the QMD/mcporter isolation fix is useful and the supplied terminal proof is meaningful, but one P1 compatibility defect remains because configured stdio mcporter servers can still be rewritten from lossy `config get --json` output and lose lifecycle/logging semantics.

## What This Changes

The PR splits QMD and mcporter spawn environments, writes per-agent mcporter config for generated QMD MCP servers, preserves some external/auth-bearing mcporter definitions, tolerates log-prefixed JSON, and localizes Skill Workshop switcher copy.

## Best Possible Solution

Keep generated per-agent mcporter config for unconfigured/default QMD, but preserve configured stdio mcporter definitions unless OpenClaw can round-trip the raw definition including lifecycle/logging and cover that path with tests.

## Reproduction Assessment

Yes. The remaining defect is source-reproducible by combining mcporter 0.10.2's lossy `config get --json` serializer with the PR branch that generates config for serialized `qmd mcp` stdio servers without visible env/auth-like material.

## Solution Assessment

No. The explicit `--config` and env split are the right direction, but rewriting configured stdio servers from an incomplete serialized definition is not the safest compatibility boundary.
