## Summary

Keep open: the central agent-scoped mcporter fix is well motivated and has useful real-environment proof, but the latest head still introduces a security-sensitive persisted config path that can duplicate configured secrets into per-agent OpenClaw state.

## What This Changes

The PR changes memory-core QMD mcporter handling to generate agent-scoped mcporter configs, pass them to mcporter calls, split qmd and mcporter environments, preserve configured server definitions, parse log-prefixed JSON, and add focused QMD manager tests.

## Best Possible Solution

Land this after generated mcporter configs avoid persisting arbitrary configured env or URL credentials, with tests that cover secret-shaped stdio env and remote URL auth cases.

## Solution Assessment

No, not yet. The explicit agent-scoped mcporter config direction is dependency-backed, but the current implementation needs a secret-safe preservation path for configured stdio env and remote URL auth before it is the best mergeable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Avoid persisting configured stdio env secrets:** `extensions/memory-core/src/memory/qmd-manager.ts:2314`
  - body: For a configured stdio mcporter server, this copies every env entry from the user's mcporter config into the generated per-agent mcporter.json under OpenClaw state. If that config contains API keys, tokens, or wrapper credentials, the PR duplicates secrets into each agent directory; preserve custom env without writing arbitrary values there, or keep those setups on an explicit safe external path.
  - confidence: 0.9
- **[P2] Treat URL query credentials as remote auth material:** `extensions/memory-core/src/memory/qmd-manager.ts:2334`
  - body: Remote servers only stay external when auth appears in known keys or URL userinfo. A configured endpoint such as `https://host/mcp?token=...` has auth material in `baseUrl` but would still be copied into the generated agent config, so check sensitive URL query parameters before returning generated mode.
  - confidence: 0.82
