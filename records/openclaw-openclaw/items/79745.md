## Summary

Keep open. The PR addresses a real QMD/mcporter breakage and now has useful proof, but it is not merge-ready because the generated mcporter config is rewritten before daemon startup and calls, which can make mcporter treat the daemon as stale on repeated searches.

## What This Changes

The branch updates memory-core QMD manager code and tests to generate agent-scoped mcporter configs, strip QMD XDG env from mcporter spawns, preserve configured mcporter servers, parse log-prefixed JSON, and key mcporter daemon starts per generated config path.

## Best Possible Solution

Land after generated mcporter config writes are content-stable, repeated-search coverage proves the daemon stays warm, and the remote-header persistence tradeoff is either accepted by owners or avoided.
