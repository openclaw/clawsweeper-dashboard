## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.83

Full review comments:

- **[P1] Preserve configured stdio env or gate the break:** `extensions/memory-core/src/memory/qmd-manager.ts:2313`
  - body: For configured stdio mcporter servers, this replaces the user's configured `env` with OpenClaw's generated QMD env. mcporter treats `env` as the stdio command environment and merges it into the spawned process, so existing wrappers or auth setups that rely on env values can stop working after upgrade. Keep these servers on the original config, preserve a reviewed safe subset, or get an explicit maintainer-approved breaking policy with tests/docs.
  - confidence: 0.87
- **[P2] Cache mcporter config resolution off the search path:** `extensions/memory-core/src/memory/qmd-manager.ts:2421`
  - body: `runMcporter()` now calls `ensureMcporterConfig()` before every mcporter call, and multi-collection searches can repeat that config probe per collection. That adds a `mcporter config get` child process plus generated-config filesystem checks to normal memory search, even though repo policy treats this metadata as process-stable. Resolve/cache the config mode per manager or lifecycle boundary instead of rediscovering it per query.
  - confidence: 0.78
