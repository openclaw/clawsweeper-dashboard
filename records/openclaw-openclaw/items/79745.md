## Summary

Keep open for maintainer review and possible merge: the latest head appears to address the prior auth-classification blocker, the fix is still useful, and current main does not yet solve the QMD/mcporter XDG leak. ## What This Changes

The PR isolates QMD mcporter config and daemon state per agent, preserves configured mcporter server definitions where safe, separates qmd and mcporter spawn environments, and makes mcporter JSON parsing tolerate log-prefixed output.

## Best Possible Solution

Land the QMD/mcporter isolation fix after maintainer compatibility signoff and required checks, keeping auth-bearing or user-env mcporter definitions on the original user-owned config path.

## Reproduction Assessment

## Solution Assessment

Mostly yes. The explicit per-agent --config path with a clean mcporter process env is the narrow dependency-aligned fix, with the remaining decision being whether maintainers accept the upgrade behavior for existing QMD mcporter deployments.
