Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve raw configured qmd server semantics:** `extensions/memory-core/src/memory/qmd-manager.ts:2401-2405`
  - body: This branch decides a configured `qmd` + `["mcp"]` stdio server is safe to rewrite into the generated per-agent config. The dependency contract does not support that round trip: `mcporter config get --json` serializes command/args/cwd/env/tool filters but omits fields such as `lifecycle` and `logging`, so an existing server that only customized those settings is silently run with different behavior after upgrade. Keep configured stdio servers on the original mcporter config unless the raw config can be copied without losing supported fields.
  - confidence: 0.9
- **[P1] Keep per-agent QMD env for external stdio qmd:** `extensions/memory-core/src/memory/qmd-manager.ts:2457-2465`
  - body: External-mode mcporter calls delete `QMD_CONFIG_DIR` and `XDG_CACHE_HOME` from mcporter's process env. mcporter's stdio runtime builds the server child env from its own `process.env`, so configured stdio QMD servers that stay external no longer inherit OpenClaw's per-agent QMD config/cache paths and can search a user/global QMD index instead of the active agent's state. Preserve the agent QMD env for the actual QMD child or require an explicit maintainer-approved tradeoff before merging.
  - confidence: 0.86
