## Summary

Keep this PR open: it addresses a real QMD/mcporter memory-search regression and the latest head fixes the earlier content-stability concern, but the current patch can still miss user XDG-based mcporter configs and needs maintainer attention for copied remote auth metadata.

## What This Changes

The PR updates memory-core QMD mcporter integration to generate per-agent mcporter configs, strip QMD XDG env from mcporter spawns, preserve configured mcporter servers, parse log-prefixed JSON, and add regression tests.

## Best Possible Solution

Keep the per-agent explicit mcporter config approach, but preserve the user's original mcporter discovery environment and either avoid duplicating remote auth metadata or document an explicit maintainer-approved storage decision with tests.

## Reproduction Assessment

We have a source-reproducible path: current main passes QMD-scoped XDG env into mcporter, while mcporter's config discovery uses XDG_CONFIG_HOME unless an explicit config path is supplied. I did not run a live gateway reproduction in this read-only review.

## Solution Assessment

No, not yet: the explicit per-agent config is the right direction, but the patch should preserve the original mcporter discovery env and settle the remote auth-metadata storage decision before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P1] Preserve user mcporter XDG config discovery:** `extensions/memory-core/src/memory/qmd-manager.ts:2271-2274`
  - body: `buildMcporterProcessEnv()` clones `this.env` after the constructor has already replaced XDG_CONFIG_HOME/XDG_CACHE_HOME with QMD's per-agent paths, then deletes those keys. That fixes the leaked QMD env, but it also drops a user's original XDG_CONFIG_HOME, so `mcporter config get <custom> --json` will not find servers stored at `$XDG_CONFIG_HOME/mcporter/mcporter.json`. Capture a separate mcporter env from `process.env` before adding QMD XDG overrides, and keep QMD XDG only in the generated stdio server entry.
  - confidence: 0.87
