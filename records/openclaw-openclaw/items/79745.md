## Summary

Keep this PR open: the per-agent `--config` direction matches the reported QMD/mcporter failures, but the current patch appears to rewrite the generated mcporter config on every call, which can make mcporter treat the keep-alive daemon as stale and restart it repeatedly.

## What This Changes

The PR adds agent-scoped mcporter config generation for QMD memory search, strips QMD-scoped XDG vars from mcporter process launches, preserves configured server definitions, parses log-prefixed JSON, updates tests, and adds a changelog entry.

## Best Possible Solution

Keep the per-agent explicit mcporter config approach, but make generated config writes idempotent so daemon mtimes stay stable and QMD keep-alive remains warm across repeated searches.

## Reproduction Assessment

Yes. Current main source passes QMD-scoped XDG variables into mcporter and raw-parses mcporter stdout, and the related reports include concrete failure logs for both the XDG leak and log-prefixed JSON path; I did not run a live repro because this review is read-only.

## Solution Assessment

No, not as written. The per-agent `--config` and clean mcporter env are the right shape, but the generated config should only be written when content changes, otherwise mcporter's mtime-based stale check can repeatedly cold-restart the keep-alive daemon.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Avoid rewriting the mcporter config before every call:** `extensions/memory-core/src/memory/qmd-manager.ts:2076`
  - body: `ensureMcporterConfig()` always writes `mcporter.json`, and `runMcporter()` calls it before every mcporter command. mcporter records config layer mtimes in daemon metadata and treats a changed mtime as stale, so the search call after `daemon start` can immediately restart the per-agent daemon, and later searches can keep cold-starting instead of reusing QMD keep-alive. Please skip the write when the generated content is unchanged, or otherwise keep the config mtime stable, and add a repeated-search regression.
  - confidence: 0.87
