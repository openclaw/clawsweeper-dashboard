## Summary

Keep this PR open for normal maintainer review and CI; the latest head addresses the known QMD/mcporter XDG regression and I did not find a discrete blocking bug in the patch.

## What This Changes

The PR writes per-agent mcporter configs for QMD, invokes mcporter with that config and a clean process env, preserves configured server definitions, parses log-prefixed JSON, and adds focused tests plus a changelog entry.

## Best Possible Solution

Land the per-agent --config approach after normal maintainer review and CI, keeping configured-server preservation and clean mcporter process env behavior.

## Reproduction Assessment

Yes. Current main source passes QMD-scoped XDG variables into mcporter, mcporter 0.10.2 uses XDG_CONFIG_HOME for config discovery when --config is absent, and the linked issue supplies concrete failure steps and logs.

## Solution Assessment

Yes. Explicit per-agent mcporter configs plus a clean mcporter process env solve the XDG discovery regression without adding a new OpenClaw config knob, while preserving custom stdio and remote server definitions.
