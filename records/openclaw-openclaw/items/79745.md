## Summary

Keep open for maintainer review before merge. The patch appears to address the QMD/mcporter isolation bug with sufficient real local proof and focused coverage, but it changes compatibility-sensitive mcporter config and daemon selection for existing QMD memory deployments.

## What This Changes

The branch changes memory-core QMD mcporter searches to generate agent-scoped mcporter config/state, split QMD and mcporter process environments, preserve existing mcporter server definitions where needed, parse log-prefixed mcporter JSON, and makes one MCP runtime test wait for server logs.

## Best Possible Solution

Land the PR after maintainers accept the generated-vs-external mcporter config model, keeping default and env-free QMD servers agent-scoped while preserving user-owned env/auth remote configs externally.

## Reproduction Assessment

Yes. Current main passes QMD-scoped XDG env into mcporter, and mcporter 0.10.2 source resolves config from XDG_CONFIG_HOME unless an explicit config source is provided; the linked reports and PR proof match that path.

## Solution Assessment

Yes. The branch uses mcporter's explicit --config contract for agent-scoped generated configs and falls back to the original mcporter config when copying env or auth material would be unsafe, which is narrower than a blanket env strip.
