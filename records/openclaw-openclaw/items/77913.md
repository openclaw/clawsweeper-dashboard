## Summary

Keep open: the PR is a plausible fix with sufficient real behavior proof, but it is still incomplete because existing-session post-action checks and tab-list redaction can still run direct-mode DNS pinning after the initial navigation path is fixed.

## What This Changes

The PR adds an `external-browser-proxy` browser navigation mode for non-OpenClaw driver profiles, skips Node DNS pinning in that mode, exports the SSRF hostname helper, and adds proxy-mode/navigation-guard tests.

## Best Possible Solution

Thread the profile-derived browser proxy mode through initial navigation, post-action result checks, new-tab checks, and tab-list redaction so every existing-session browser policy check uses the same SSRF trust boundary.

## Reproduction Assessment

Yes. Source inspection shows current main classifies existing-session as direct and DNS-pins hostnames, and the PR body provides a deterministic split-DNS or hosts-file reproduction path for the false positive.

## Solution Assessment

No, not yet. The new external-browser-proxy mode is the right narrow direction, but the patch remains partial until that mode is propagated to existing-session post-action and tab-list result checks.

## Review Findings
