## Summary

This PR is superseded by the merged post-core plugin convergence work, which repairs configured plugin payloads after the package swap and before gateway restart without allowing fresh plugin installs during the active package-manager phase.

## What This Changes

The PR changes doctor update-time plugin repair so runtime harness plugins are excluded from update deferral and adds Codex runtime regression coverage.

## Best Possible Solution

Keep the shipped post-core convergence design and close this branch rather than reintroducing package-manager plugin installs during the active package swap.

## Reproduction Assessment

Yes, source inspection shows the historical narrow path: a configured Codex runtime entering repair with only OPENCLAW_UPDATE_IN_PROGRESS is part of the update-deferral behavior. The full current update flow now has a later post-core convergence pass before gateway restart, so the observable update failure is covered there.

## Solution Assessment

No, this branch is no longer the best way to solve the issue. The merged post-core convergence path is safer because it preserves active package-swap deferral and repairs configured plugin payloads before restart.
