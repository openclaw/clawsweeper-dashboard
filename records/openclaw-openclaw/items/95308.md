## What This Changes

The PR adds an `issue_comment` pre-filter in `.github/workflows/clawsweeper-dispatch.yml` so only ClawSweeper command comments mint the target repository token and run acknowledgement/comment-router dispatch.

## Best Possible Solution

Land the narrow pre-filter once maintainers accept or verify the live workflow behavior, keeping generic activity forwarding separate from command-only acknowledgement and router dispatch.

Yes. This is the narrowest maintainable fix in the target workflow: move the existing command predicate ahead of the target-token and command-dispatch steps while leaving generic activity forwarding intact.
