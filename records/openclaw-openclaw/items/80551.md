## Summary

Keep open: current main and v2026.6.10 still do not provide the requested typed Plugin SDK `agent_start` hook or `llm_input` prompt split fields, although adjacent runId correlation for legacy `before_agent_start` and `agent_end` has landed. The remaining work is a public Plugin SDK contract decision, not conservative cleanup.

## What This Changes

The issue requests a typed Plugin SDK `agent_start` hook before `llm_input`, parent/requester run correlation, and optional `userPrompt` / `prependedContext` fields on `llm_input` for tracing plugins.

## Best Possible Solution

Approve a minimized tracing lifecycle contract, then implement it across Plugin SDK types, embedded runner dispatch, Gateway/subagent run identity paths, docs, API baselines, and focused hook-order tests.

## Reproduction Assessment

Yes, at source level: current main exposes adjacent hooks and runId backfill, but the Plugin SDK hook union and `llm_input` event type still lack the requested `agent_start`, `userPrompt`, and `prependedContext` contract.

## Solution Assessment

Unclear pending maintainer API design. A typed additive hook is plausible and aligned with the Plugin SDK direction, but maintainers need to choose timing, payload minimization, and prompt-split guarantees first.
