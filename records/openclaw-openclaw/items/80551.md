## Summary

Keep open: current main and v2026.6.6 still lack the requested typed plugin `agent_start` hook and `llm_input` prompt-split fields, the linked implementation PR is closed unmerged, and the related open observability/lifecycle issues do not exactly own this trace-correlation contract.

## What This Changes

The issue requests a typed `agent_start` plugin lifecycle hook before the first `llm_input` plus `userPrompt` and `prependedContext` fields for tracing correlation.

## Best Possible Solution

Approve one narrow additive Plugin SDK lifecycle contract that emits run-start correlation before first model input, defines prompt-split semantics and access gating, threads parent run identity through Gateway/subagent paths, and updates focused tests plus plugin docs together.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the missing capability: the plugin hook catalog lacks `agent_start`, `llm_input` lacks `userPrompt` and `prependedContext`, and the embedded runner dispatches only the existing `llm_input` event at model-input time.

## Solution Assessment

Unclear. A typed additive hook is plausible and fits the plugin API direction, but the exact public contract should be approved before reviving the closed branch or starting an automated implementation.
