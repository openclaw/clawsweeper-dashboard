## What This Changes

The issue requests an additive typed `agent_start` plugin lifecycle hook before first `llm_input`, parent/child run correlation fields, and `llm_input` `userPrompt` / `prependedContext` fields for tracing plugins.

## Best Possible Solution

Approve one narrow tracing lifecycle contract, then implement it across the embedded runner, Gateway/subagent run identity paths, Plugin SDK types, docs, and focused hook-order tests.

## Reproduction Assessment

Yes, at source level: current `main` exposes adjacent plugin hooks but not the requested typed plugin `agent_start` hook or `llm_input` split fields. This is a missing capability, not a broken documented behavior.

## Solution Assessment

Unclear pending maintainer API design. A typed additive hook is plausible and aligned with the Plugin SDK direction, but maintainers need to choose the public timing, payload minimization, and prompt-split guarantees first.
