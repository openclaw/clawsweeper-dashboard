## Summary

Keep open: current main and latest release still lack the typed plugin `agent_start` hook and `llm_input` prompt-split fields; the linked implementation PR is closed unmerged, and the broader observability issue at https://github.com/openclaw/openclaw/issues/50291 does not fully own this exact hook-ordering and prompt-split contract.

## What This Changes

Request adds a typed `agent_start` plugin hook before the first `llm_input` event and optional `userPrompt` / `prependedContext` fields on `llm_input` for tracing correlation.

## Best Possible Solution

Approve one narrow Plugin SDK lifecycle contract that emits run-start correlation before first model input, defines prompt-split semantics, and covers embedded runner, Gateway, and subagent paths with focused tests and docs.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source inspection confirms the requested plugin `agent_start` hook and `llm_input` prompt-split fields are absent from current main and v2026.6.6.

## Solution Assessment

Unclear pending maintainer API approval. A typed additive hook is plausible and aligned with the plugin API boundary, but maintainers should approve hook timing, field names, and prompt-split guarantees before implementation.
