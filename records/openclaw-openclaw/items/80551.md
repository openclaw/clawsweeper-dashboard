## Summary

Keep open: current `main` and latest release still lack the requested typed plugin `agent_start` hook and the `llm_input` prompt split fields, while the linked implementation PR is closed unmerged and the broader lifecycle-hooks issue does not fully own this trace-correlation contract.

## What This Changes

The issue requests a typed `agent_start` Plugin SDK lifecycle hook plus `llm_input` prompt split fields so tracing plugins can correlate parent and child agent runs before model input.

## Best Possible Solution

Approve one narrow Plugin SDK lifecycle contract that emits run-start correlation before first model input, documents prompt-split semantics, and covers embedded runner, CLI/Gateway, and subagent paths with focused tests.

## Reproduction Assessment

Yes, at source level: current `main` and v2026.6.6 expose adjacent hook surfaces but do not expose the requested typed plugin `agent_start` hook or `llm_input` prompt split fields. This is a missing capability rather than a broken documented behavior.

## Solution Assessment

Unclear pending maintainer API design. A typed additive hook is plausible and fits the Plugin SDK direction, but maintainers need to choose the public hook name, timing, payload minimization, and prompt-split guarantees before implementation.
