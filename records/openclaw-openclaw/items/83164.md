## Summary

Keep open: current main and the latest shipped release still enqueue accepted Discord reactions as passive system events without requesting a heartbeat wake, and an open linked PR is already the concrete fix path.

## What This Changes

Request accepted Discord reaction notifications to wake the routed agent session after the reaction system event is successfully queued.

## Best Possible Solution

Review and land either the scoped Discord PR or the shared notification wake policy, preserving existing Discord reaction gates and waking only after successful enqueue.

## Reproduction Assessment

Yes. Source inspection on current main shows accepted Discord reactions can reach enqueueSystemEvent without any requestHeartbeat call on that path.

## Solution Assessment

Unclear until maintainers choose the policy shape. The narrow Discord implementation is feasible, but the open shared notification wake PR may be the more maintainable cross-channel direction.
