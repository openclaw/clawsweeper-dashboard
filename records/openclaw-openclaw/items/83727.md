## Summary

Keep open: the reporter's Twitch/Twurple error matches current main and the latest release, where the refreshing auth provider adds the token without registering the chat intent that Twurple ChatClient later queries.

## What This Changes

The issue reports that the @openclaw/twitch plugin connects briefly, then fails because Twurple cannot fetch a chat-intent token for the configured bot user.

## Best Possible Solution

Register the Twitch chat intent when adding the refreshing-token user, preserve the existing static-token path, and add focused regression coverage for the addUserForToken intent argument.

## Reproduction Assessment

Yes, with high confidence at the source level: current main and v2026.5.18 register the refreshing Twitch token without the chat intent that Twurple ChatClient queries. I did not run a live Twitch credential reproduction in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is to pass the existing Twurple chat intent during refreshing-token user registration and cover that dependency contract in the Twitch client test.
