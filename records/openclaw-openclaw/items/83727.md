## Summary

Keep open: the reporter's Twitch/Twurple auth error matches current main, where the Twitch plugin adds a refreshing token without registering the chat intent that Twurple ChatClient later queries.

## What This Changes

The issue asks OpenClaw to fix the Twitch plugin so valid bot credentials can connect instead of failing with Twurple's unknown chat intent error.

## Best Possible Solution

Register the Twitch chat intent with the Twurple refreshing auth provider when adding the user, preserve existing static-token behavior, and add focused regression coverage for the addUserForToken intent argument.

## Reproduction Assessment

Yes, with high confidence at the source level: current main omits the chat intent when adding a refreshing Twitch token, and Twurple's ChatClient always queries the chat intent. I did not perform a live Twitch credential reproduction in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is to pass the existing Twurple chat intent when registering the user token and to pin that dependency contract in Twitch client tests. No new config or product decision is needed.
