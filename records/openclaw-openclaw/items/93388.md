## Summary

Keep open: the Google default direction is plausible, but the PR would advertise and send an OpenRouter image model slug that OpenRouter's live catalog does not expose, and doctor would silently downgrade Pro/Nano Banana Pro configurations to Flash.

## What This Changes

The PR switches Google/OpenRouter image-generation defaults and docs/tests to `gemini-3.1-flash-image` and adds doctor migrations for retired Gemini/Imagen image model refs.

## Best Possible Solution

Keep the Google Flash default only where Google's API supports it, use a currently supported OpenRouter slug or leave OpenRouter unchanged until its catalog exposes a stable replacement, and migrate Pro preview configs to stable Pro rather than Flash.
