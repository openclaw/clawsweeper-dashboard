## Summary

Keep open. Current main still runs bundled runtime-deps installs through pnpm without `CI=true` or a `confirmModulesPurge=false` equivalent, while both sync and async install paths use non-TTY stdio. The report is specific, current, source-backed, and narrow enough for an automated repair PR.

## What This Changes

The issue requests that bundled plugin runtime dependency staging avoid pnpm's non-TTY modules-purge prompt when Gateway startup or repair runs under launchd/systemd.

## Best Possible Solution

Land a narrow runtime-deps package-manager fix that makes internal bundled dependency installs explicitly non-interactive for pnpm, covers both sync and async staging through the shared helper, and records the user-facing service-startup fix in the changelog.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to invoke the bundled runtime-deps pnpm install path under non-TTY stdin with an existing modules directory and no `CI` or purge-confirmation override; current code supplies that environment and pnpm source shows the exact abort.

## Solution Assessment

Yes. The narrowest maintainable solution is to set `CI=true` or pnpm's `confirmModulesPurge=false` equivalent in the internal runtime-deps install environment/args and add focused tests, without changing broader plugin loading or dependency resolution policy.
