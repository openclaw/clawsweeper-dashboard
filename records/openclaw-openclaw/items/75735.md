## Summary

Keep open. Current main still sends bundled runtime-deps installs through pnpm without `CI=true` or a `confirmModulesPurge=false` equivalent, while both install execution paths run with non-interactive stdin. The existing ClawSweeper comment is still accurate, and the report is current, source-backed, narrow, and suitable for a focused repair PR.

## What This Changes

Make bundled plugin runtime dependency staging work under launchd/systemd by preventing pnpm from aborting when it wants to purge `node_modules` without a TTY.

## Best Possible Solution

Land a narrow bundled runtime-deps package-manager fix that makes internal pnpm staging explicitly non-interactive through the shared helper, covers both sync and async repair paths, updates the documented pnpm command if args change, and adds a changelog entry for the service-startup fix.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to run the current bundled runtime-deps pnpm install path under non-TTY stdin with an existing `node_modules` tree that pnpm wants to purge; current OpenClaw supplies non-interactive stdio and no pnpm confirmation override.

## Solution Assessment

Yes. The narrowest maintainable fix is to set `CI=true` or pass `--config.confirm-modules-purge=false` from the shared bundled runtime-deps package-manager code so all startup, doctor, and per-plugin repair callers inherit it without changing plugin loading policy.

## Repair Work Prompt

Fix the bundled runtime-deps pnpm non-TTY staging failure reported in openclaw/openclaw#75735. Current main runs internal bundled runtime-deps installs with non-interactive stdio but without `CI=true` or `confirmModulesPurge=false`, so pnpm can abort with `ERR_PNPM_ABORTED_REMOVE_MODULES_DIR_NO_TTY` when it needs to purge `node_modules` under launchd/systemd. Keep the fix narrow: update the shared bundled runtime-deps package-manager environment or pnpm args so both sync and async staging paths inherit the non-interactive confirmation behavior; do not change broader plugin activation, dependency resolution, or service management policy. Add focused tests in `src/plugins/bundled-runtime-deps.test.ts` for the env/args and the pnpm runner path. Update `docs/plugins/dependency-resolution.md` if the documented pnpm command changes, and add a user-facing `CHANGELOG.md` entry with reporter credit. Avoid touching dependency versions, lockfiles, bundled plugin manifests, or unrelated runtime-deps repair behavior.
