## Summary

Keep open. The linked bug is source-reproducible on current main, but this PR only broadens the registry phase check and does not fix the production preflight path where no queue backend is attached and steering is skipped while the operation is not streaming.

## Best Possible Solution

Make WebChat follow-ups during preflight compaction either drain after compaction/session rotation or return a clear busy/error response, with production-path coverage and a changelog entry.
