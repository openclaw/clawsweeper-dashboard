## Summary

Keep open. The linked WebChat compaction bug is source-reproducible, but this PR only admits compacting phases after a reply backend is attached; the production preflight path runs before backend attachment and skips steering because streaming is false.

## Best Possible Solution

Make follow-ups during preflight compaction either drain through the production queue after compaction/session rotation or return a clear busy response, with regression coverage on the real runner path and a changelog entry.
