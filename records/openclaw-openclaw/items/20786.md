## Summary

Keep open: current main still does not implement Telegram Business message support. Business update names are present through grammY defaults, but the Telegram plugin has no `business_message`/`business_connection` runtime handlers, no Business-aware dedupe or sequencing, no outbound `business_connection_id` path, and an open fork PR still references this issue with closing syntax.

## What This Changes

Request support for Telegram Business connected chats by handling `business_connection` and `business_message` updates in the Telegram plugin.

## Best Possible Solution

Keep this as the canonical tracker and ship a refreshed Telegram plugin implementation with Business connection state, Business message routing, dedupe/sequencing support, safe outbound `business_connection_id` handling, tests, docs, and live proof.

## Reproduction Assessment

Yes. Source inspection on current main shows Business update names can be delivered, but there are no `business_message` or `business_connection` handlers and no outbound `business_connection_id` mapping, so the reported Business chat flow cannot work through the existing pipeline.

## Solution Assessment

Yes on the product boundary: the Telegram plugin is the right place to solve this. The existing open fork PR should be reviewed as a candidate, but prior review found account-scoping, disconnect, dedupe, and outbound mapping concerns that should be addressed in a refreshed implementation.
