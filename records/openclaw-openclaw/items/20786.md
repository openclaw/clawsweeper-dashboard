## Summary

Keep open because current main still lacks Telegram Business runtime handling, outbound `business_connection_id` support, Business update dedupe/sequencing, and docs, while a closing-syntax fork PR remains an active implementation candidate.

## What This Changes

Add Telegram Business support so connected bots can receive `business_connection` and `business_message` updates, route Business messages through Telegram inbound handling, and reply with the correct Business connection context.

## Best Possible Solution

Ship a Telegram plugin implementation that handles Business connect/disconnect state per account, routes `business_message` through the existing DM pipeline, updates dedupe/sequential keys, applies `business_connection_id` only where appropriate, and documents setup plus access-control behavior.

## Reproduction Assessment

Yes, from source inspection. Current main can request the Business update names through grammY defaults, but it has no Business handler routing `business_message` into `handleInboundMessageLike()` and no outbound `business_connection_id` mapping.

## Solution Assessment

Yes for the requested direction: the implementation belongs in the Telegram plugin rather than generic core. The open fork PR or a replacement still needs maintainer review for account-scoped state, disconnect behavior, outbound method support, tests, and docs.
