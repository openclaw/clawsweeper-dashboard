## Summary

Keep open because current main still lacks Telegram Business runtime support: Business update names may be allowed, but there are no handlers for `business_message` or `business_connection`, no Business-aware dedupe/sequencing, no outbound `business_connection_id` mapping, and no focused docs/tests. The closed target-repo PR and the open fork PR show this remains the canonical tracker rather than an implemented or duplicate item.

## What This Changes

Request Telegram plugin support for Telegram Business connection/message updates so Business-connected personal chats can be received and answered.

## Best Possible Solution

Ship a refreshed Telegram plugin implementation that handles Business connect/disconnect state per Telegram account, routes `business_message` through the existing DM authorization and inbound pipeline, updates dedupe/sequential keys, applies `business_connection_id` only where supported, and documents setup plus access-control behavior.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes, from source inspection: current main has no Business handlers or outbound `business_connection_id` mapping, so Business updates cannot reach the inbound pipeline or reply path.

## Solution Assessment

Is this the best way to solve the issue? Yes for the requested direction: the work belongs in the Telegram plugin boundary, but the existing fork PR or a replacement still needs maintainer review for persistence, account scoping, access control, docs, and live proof.
