## Summary

Keep open: current main and v2026.5.12 still only admit Telegram Business update names through grammY defaults; the Telegram plugin does not process Business updates or attach `business_connection_id` for replies. The open fork PR at https://github.com/BingqingLyu/openclaw/pull/1324 still uses closing syntax for this issue, so this should remain the canonical tracker until a corrected implementation lands or is replaced.

## What This Changes

Request Telegram Business bot support by handling `business_message` and `business_connection` updates and replying with the correct `business_connection_id`.

## Best Possible Solution

Ship a refreshed Telegram plugin implementation with account-scoped durable Business connection state, Business message routing through the existing DM pipeline, Business-aware dedupe/sequencing, safe outbound `business_connection_id` injection, docs, tests, and live Telegram Business proof.

## Reproduction Assessment

Yes, at source level. Current main allows Telegram Business update names through grammY defaults but has no Business handlers or outbound `business_connection_id` mapping, so the reported Business chat flow cannot pass through the existing Telegram pipeline.

## Solution Assessment

Yes on product boundary: the Telegram plugin is the right place to solve this, but adding `allowed_updates` alone is not enough. The maintainable fix needs Business connection state, inbound routing, dedupe/sequencing, outbound mapping, docs, tests, and live proof.
