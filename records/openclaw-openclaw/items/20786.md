## Summary

Keep open because current main still lacks Telegram Business runtime support, and the open fork implementation that uses closing syntax still needs Telegram-owner review or replacement.

## What This Changes

Request Telegram Business support so bots can receive `business_connection` and `business_message` updates, route Business messages through the Telegram inbound path, and reply with `business_connection_id`.

## Best Possible Solution

Ship Telegram plugin support that handles Business connection state per account, routes `business_message` through the existing DM pipeline, applies `business_connection_id` on supported outbound methods, and documents setup plus access control.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes from source inspection: current main has no Business handlers or outbound `business_connection_id` mapping, so Business updates cannot reach the existing inbound or reply paths.

## Solution Assessment

Is this the best way to solve the issue? Yes for the requested boundary: the fix belongs in the Telegram plugin, but the exact state persistence, account scoping, and access-control behavior need maintainer review before merge.
