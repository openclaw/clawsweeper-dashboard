Keep open: current main and the latest stable release still lack Telegram Business runtime support. The issue remains the canonical tracker because the target-repo implementation PR is closed unmerged and the fork PR with closing syntax is still open for maintainer review.

## What This Changes

Request to support Telegram Business Bot updates by handling `business_message` and `business_connection` updates in the Telegram plugin.

## Best Possible Solution

Ship a refreshed Telegram plugin implementation that handles Business connect/disconnect state per account, routes `business_message` through the existing inbound and DM authorization pipeline, updates dedupe/sequential keys, applies `business_connection_id` only where supported, and documents setup plus access-control behavior.

Yes. Source inspection on current main shows no `business_message` or `business_connection` handlers and no outbound `business_connection_id` mapping, so Business updates cannot reach the inbound pipeline or reply path.

## Solution Assessment

Yes for the requested direction: the work belongs in the Telegram plugin boundary. The current path should be maintainer review of the open fork PR or a replacement implementation because persistence, account scoping, access control, docs, and live proof still need decisions.
