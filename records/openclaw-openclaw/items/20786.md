## Summary

Keep open because current main still only allows Telegram Business update names; it does not process Business messages, persist/apply business_connection_id for replies, or document the flow, and a closing implementation candidate is still pending review.

## What This Changes

The issue requests Telegram Business support by handling business_connection and business_message updates in the Telegram plugin.

## Best Possible Solution

Ship a Telegram plugin change that registers Business handlers before normal message handling, routes business_message through the existing inbound path, persists account-scoped business_connection_id state for supported replies, updates dedupe/sequencing, and adds focused tests plus Telegram docs.

## Reproduction Assessment

Yes. Source inspection is enough: current main can allow Business update names, but no Business handler passes business_message into handleInboundMessageLike and no outbound business_connection_id mapping exists.

## Solution Assessment

Yes for the requested direction: this belongs in the Telegram plugin rather than core. The pending implementation or a replacement should be reviewed for durable state, account isolation, outbound method support, tests, and docs before this issue closes.
