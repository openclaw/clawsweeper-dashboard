## Summary

Keep open because current main still allows Telegram Business update names but does not process Business messages, model them for dedupe/sequencing, persist/apply business_connection_id for replies, or document the flow.

## What This Changes

Enable the Telegram plugin to receive and respond to Telegram Business business_connection and business_message updates.

## Best Possible Solution

Ship an owner-local Telegram plugin implementation with Business handlers, durable per-account connection mapping for supported replies, dedupe/sequential coverage, focused tests, and Telegram docs.

## Reproduction Assessment

Yes. Source inspection against current main shows Telegram can allow the update names, but no Business handler routes business_message into handleInboundMessageLike and no outbound business_connection_id mapping exists.

## Solution Assessment

Yes for the requested direction. This belongs in the Telegram plugin rather than core, with maintainer review focused on state durability, account isolation, outbound method support, tests, and docs.
