## Summary

Keep open: current main and the latest release still do not implement Telegram Business message support. Business update names are available through grammY defaults, but OpenClaw has no `business_message` or `business_connection` runtime handlers, no Business-aware dedupe/sequencing path, no outbound `business_connection_id` handling, and an open fork PR still references this issue with closing syntax.

## What This Changes

Add Telegram Business Bot support by handling `business_connection` and `business_message` updates, preserving Business connection state, and routing Business DMs through the Telegram channel pipeline.

## Best Possible Solution

Ship a refreshed Telegram plugin implementation with account-scoped durable Business connection state, Business message routing through the existing DM pipeline, Business-aware dedupe/sequencing, safe outbound `business_connection_id` injection, docs, tests, and live Telegram Business proof.

## Reproduction Assessment

Yes. Source inspection on current main shows Business update names can be delivered, but there are no `business_message` or `business_connection` handlers and no outbound `business_connection_id` mapping, so the reported Business chat flow cannot work through the existing pipeline.

## Solution Assessment

Yes on the product boundary: the Telegram plugin is the right place to solve this. The implementation should be broader than only adding `allowed_updates`, because it also needs Business connection state, dedupe/sequencing, outbound mapping, access-control review, tests, docs, and live proof.
