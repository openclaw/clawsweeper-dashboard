## Summary

Keep open. Current main still only partially covers Telegram Business: allowed update names are present, but Business updates are not handled, deduped/sequenced, mapped for outbound replies, documented, or tested; the remaining fork implementation should be reviewed or replaced before this tracker closes.

## What This Changes

Request Telegram Business bot support in the Telegram plugin, including business_connection/business_message handling, inbound routing, and outbound replies with business_connection_id.

## Best Possible Solution

Ship a refreshed Telegram plugin implementation that handles Business connect/disconnect state per account, routes business_message through DM authorization and the inbound pipeline, applies business_connection_id only for supported outbound methods, and documents setup plus access-control behavior.

## Reproduction Assessment

Yes. The source-level path is high confidence: current main can allow Business update names, but there is no Business handler to pass business_message into handleInboundMessageLike.

## Solution Assessment

Yes for the requested direction: this belongs in the Telegram plugin rather than core. The known source branch is not the best implementation as-is because prior review found dedupe, account-isolation, disconnect, and outbound mapping issues.
