## Summary

Keep this issue open. Current main still allows Telegram Business update names to be requested, but it does not process them: there are no `business_message` or `business_connection` runtime handlers, no Business fields in Telegram dedupe/sequential contexts, no outbound `business_connection_id` mapping, and no focused Telegram Business docs/tests. Open PR #53629 and fork mirror #1324 are already implementation candidates using closing syntax, so this tracker should stay open until one lands or is replaced.

## What This Changes

Request Telegram Business bot support in the Telegram plugin, including `business_connection`/`business_message` handling, inbound routing for Business messages, and outbound replies with `business_connection_id`.

## Best Possible Solution

Keep this issue as the canonical Telegram Business support tracker, then review and land PR #53629 or an equivalent Telegram plugin change that registers Business handlers before the normal message handler, routes Business messages through the existing inbound path, persists/applies `business_connection_id` for supported replies, extends dedupe/sequential keys, and adds focused tests plus docs.
