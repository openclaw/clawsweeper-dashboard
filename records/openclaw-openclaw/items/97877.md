## Summary

Keep open: current main and the latest release still block the empty-error retry when replay metadata reports prior side effects, and the two linked closing PRs are open candidate fixes rather than merged provenance.

## What This Changes

The issue asks OpenClaw to retry empty transient provider-error turns after prior session tool side effects when the failed turn itself produced no tools, delivery, or visible content.

## Best Possible Solution

Keep this issue open until one linked fix PR lands with regression coverage that separates current-attempt side effects from accumulated replay history while preserving same-attempt replay blockers.

## Reproduction Assessment

Yes at source level: construct an empty `stopReason: "error"` assistant turn with `replayMetadata.hadPotentialSideEffects: true`, and current main rejects retry before the `[empty-error-retry]` path. I did not reproduce a live OpenRouter 5xx in this read-only review.

## Solution Assessment

Yes, the best fix direction is to distinguish side effects from the failed current attempt from accumulated session replay history. The issue should stay open while maintainers review or consolidate the two linked fix PRs.
