## Summary

Keep open: current main and the latest release still reject the silent-error retry when replay metadata says the attempt has prior potential side effects, so the reported transient-provider failure path remains source-reproducible.

## What This Changes

The issue requests a fix for empty provider-error retries being blocked after tool activity by accumulated `hadPotentialSideEffects` replay metadata.

## Best Possible Solution

Separate current failed-turn side-effect evidence from accumulated replay history for empty provider-error retries, while keeping the retry blocked for side effects produced by the failed turn itself.

## Reproduction Assessment

Yes at source level: current main still passes accumulated replay state into attempts and the silent-error retry rejects attempts whose replay metadata reports side effects. I did not run a live OpenRouter 5xx reproduction in this read-only review.

## Solution Assessment

Yes, the best fix is to split current-attempt side-effect evidence from accumulated replay safety for this retry path, preserving the guard for side effects in the failed turn itself.
