## Summary

Keep open: current main and the latest release still block the silent-error retry whenever accumulated replay metadata reports prior side effects, and two open PRs already target this issue with closing syntax.

## What This Changes

The issue asks OpenClaw to retry clean transient provider-error turns after earlier session tool use instead of silently terminating Discord agent sessions.

## Best Possible Solution

Land one focused fix that separates current-attempt side-effect evidence from accumulated session history for empty provider-error retries, then close this issue from the merged PR.

## Reproduction Assessment

Yes at source level: current main passes accumulated replay state into attempts and shouldRetrySilentErrorAssistantTurn rejects hadPotentialSideEffects before retrying. I did not live-force an OpenRouter 5xx during this read-only review.

## Solution Assessment

Yes: the narrow best fix is to track current-attempt side effects separately from accumulated replay history while preserving the retry block for side effects produced by the failed turn itself.
