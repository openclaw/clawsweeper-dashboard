## What This Changes

## Best Possible Solution

Land one narrow fix that bases retry safety on current failed-attempt side effects while preserving blockers for tools, delivery, spawns, async work, and terminal output, then close this issue.

## Solution Assessment

Yes; the best fix is to distinguish current failed-attempt side effects from accumulated session replay history, with regression coverage for an empty provider-error turn after earlier tool use.
