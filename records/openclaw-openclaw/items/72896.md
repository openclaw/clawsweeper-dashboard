## Summary

Keep open: the Slack failed-delivery accounting fix is useful, but the PR should not merge as-is because the new `chat.postMessage` retry classifier retries ambiguous transport failures that current main deliberately avoids for write calls, which can duplicate user-visible Slack replies.

## What This Changes

The PR adds Slack-local transient `chat.postMessage` retries, failed-delivery-aware reply visibility accounting, focused tests, and a changelog entry.

## Best Possible Solution

Land the Slack failed-count accounting fix, but keep outbound send retries limited to non-duplicating failure modes or add an explicit idempotency/deduplication mechanism before retrying ambiguous `chat.postMessage` transport errors.

## Reproduction Assessment

Yes. The source-level reproduction is a mocked Slack send that fails once with a wrapped transient request error plus a Slack turn where queued final delivery is paired with dispatcher failed counts; current main lacks the new socket retry and failed-count correction paths.

## Solution Assessment

No, not as written. The failed-count accounting direction is narrow, but retrying `ECONNRESET`/`ETIMEDOUT` after `chat.postMessage` without idempotency can duplicate visible Slack replies and conflicts with current main's write retry posture.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Avoid retrying ambiguous postMessage writes:** `extensions/slack/src/send.ts:260-263`
  - body: Current main disables generic Slack write retries and only reintroduced DNS-only retries because user-visible posts can be duplicated when a request reaches Slack but the client sees a transport error. This classifier retries `ECONNRESET`, `ETIMEDOUT`, and similar `chat.postMessage` failures without an idempotency or dedupe key, so a response-side reset/timeout can post the same reply more than once. Please limit retries to failure modes known to occur before Slack can accept the message, or add a concrete dedupe/idempotency mechanism.
  - confidence: 0.86
