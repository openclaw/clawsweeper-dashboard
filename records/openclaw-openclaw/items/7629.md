## Best Possible Solution

Make a maintainer scope decision; if accepted, implement a plugin-owned, compatibility-preserving hybrid retrieval path with typed config, FTS setup on `text`, query-text propagation, vector-only fallback, docs, and targeted tests.

## Reproduction Assessment

## Solution Assessment

Unclear until maintainers decide scope. If official `memory-lancedb` should reach parity with built-in memory search, the narrowest maintainable path is plugin-owned hybrid retrieval with explicit config/default behavior and vector-only fallback.
