## Summary

Keep open. Current main still leaves the bundled `memory-lancedb` plugin on vector-only retrieval with no FTS/BM25 index, no hybrid config surface, and no query text passed into database search; the related implementation PR was closed unmerged.

## What This Changes

Add optional hybrid BM25 plus vector retrieval to `memory-lancedb`, including FTS index setup, typed config, and query-text-aware search.

## Best Possible Solution

Implement a narrow `extensions/memory-lancedb` parity feature that preserves vector-only fallback while adding documented, tested hybrid retrieval.

## Reproduction Assessment

Not applicable as a feature request. The current-main gap is high-confidence from source inspection: LanceDB plugin search is vector-only and has no FTS index or hybrid config surface.

## Solution Assessment

Yes, if maintainers want this parity feature. The maintainable boundary is the bundled `memory-lancedb` plugin, with query text propagation and fallback behavior kept local rather than changing core memory contracts broadly.
