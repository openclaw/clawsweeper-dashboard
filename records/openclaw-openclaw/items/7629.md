## Summary

Keep open: current main still leaves the bundled `memory-lancedb` plugin on vector-only retrieval, with no FTS/BM25 index setup, no hybrid config surface, and no query-text search path. The closed implementation PR attempted this exact feature but was not merged, and the external `memory-lancedb-pro` alternative noted in prior discussion does not satisfy bundled-plugin parity.

## What This Changes

Add hybrid BM25 plus vector retrieval to the bundled `memory-lancedb` plugin, including FTS setup, query-text propagation, and typed config controls.

## Best Possible Solution

If maintainers want bundled-plugin parity, implement hybrid retrieval plugin-locally in `extensions/memory-lancedb` with typed config, manifest/docs updates, FTS index maintenance, query-text propagation, vector-only fallback, and focused regression coverage.

## Reproduction Assessment

Not applicable as a feature request. Source inspection of current main gives high confidence that `memory-lancedb` still lacks hybrid config, FTS setup, and a query-text database search path.

## Solution Assessment

Unclear until maintainers choose the bundled-plugin policy and defaults. The likely implementation boundary is `extensions/memory-lancedb`, but native LanceDB hybrid versus local fusion, default enablement, and scoring semantics need owner direction first.
