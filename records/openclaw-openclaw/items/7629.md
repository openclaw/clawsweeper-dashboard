## Summary

Keep open: current main and v2026.6.5 still leave the official memory-lancedb plugin on vector-only retrieval, and the only direct implementation PR was closed unmerged. The community memory-lancedb-pro/ClawHub discussion is useful context, but it does not implement this requested capability in the official plugin.

## What This Changes

The issue requests hybrid BM25 plus vector retrieval support for the bundled memory-lancedb plugin.

## Best Possible Solution

Implement plugin-owned hybrid retrieval in memory-lancedb with explicit config, FTS index maintenance on text, query-text propagation, verified LanceDB JS API usage or a tested local fusion fallback, docs, and focused tests.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a bug repro, but source inspection gives high confidence the capability is absent: current memory-lancedb creates no FTS index, exposes no hybrid config, and passes only vectors into db.search.

## Solution Assessment

Is this the best way to solve the issue? The direction is plausible and the right boundary is memory-lancedb, but maintainers still need to choose defaults, fallback behavior, ranking weights, and whether native LanceDB hybrid or local fusion is the permanent implementation.
