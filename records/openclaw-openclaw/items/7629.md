## Summary

Keep open: origin/main still leaves the bundled `memory-lancedb` plugin on vector-only retrieval with no hybrid config, FTS setup, or query-text database search path. The matching implementation at https://github.com/openclaw/openclaw/pull/7636 was closed unmerged, and the external `memory-lancedb-pro` plugin discussion is a community workaround rather than bundled-plugin parity.

## What This Changes

Add optional BM25 plus vector hybrid retrieval to the bundled `memory-lancedb` plugin with FTS indexing, config weights, and query-text search.

## Best Possible Solution

Implement this as a plugin-local bundled `memory-lancedb` parity feature with typed config and manifest docs, FTS index maintenance on `text`, original query propagation, vector-only fallback, and targeted regression tests.
