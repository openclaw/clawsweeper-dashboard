## What This Changes

The issue requests BM25 full-text plus vector hybrid retrieval for `memory-lancedb`, including FTS indexing and hybrid weighting config.

## Best Possible Solution

Implement this in `extensions/memory-lancedb` after maintainer direction on defaults and fallback policy: add typed config/docs/manifest support, maintain an FTS index on `text`, thread query text through recall/search/forget/duplicate paths, and cover vector-only upgrade behavior.

Yes. Source inspection gives a high-confidence path: configure `memory-lancedb`, run recall or `ltm search` for an exact keyword, and current main still embeds the query and calls LanceDB `vectorSearch` without an FTS/BM25 path.
