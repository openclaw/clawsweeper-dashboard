## Best Possible Solution

Maintainers should decide whether memory-lancedb remains a supported parity target; if approved, implement FTS index lifecycle, original-query propagation, and a LanceDB 0.30.0-compatible fusion strategy entirely inside the plugin with backward-safe defaults and integration proof, otherwise correct the README and document vector-only scope.

## Solution Assessment

No, the issue’s exact API sketch should not be implemented verbatim. The maintainable path is plugin-local, but it must use the pinned LanceDB 0.30.0 FTS/index contracts and explicitly settle fusion scoring, defaults, fallback, and upgrade behavior.
