## Summary

Keep open: current main and v2026.6.5 still leave the official memory-lancedb plugin on vector-only LanceDB retrieval, with no FTS index creation, no hybrid config, and no query-text path into database search. The community memory-lancedb-pro discussions are useful external context, but they do not land this requested capability in the official plugin, and the direct implementation PR was closed unmerged.

## What This Changes

Request hybrid BM25 plus vector retrieval for the official memory-lancedb plugin, including FTS indexing, hybrid configuration, and LanceDB hybrid search behavior.

## Best Possible Solution

Implement plugin-owned hybrid retrieval in memory-lancedb with maintainer-approved config/defaults, FTS index maintenance on text, query-text propagation through recall/forget/CLI/hooks and duplicate checks, verified LanceDB JS API usage or a tested local fusion fallback, docs, and focused tests for both hybrid and vector-only behavior.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes: source inspection shows memory-lancedb creates no FTS index, exposes no hybrid config, and passes only vectors into db.search, so exact keyword/BM25 recall cannot participate in the current retrieval path.

## Solution Assessment

Is this the best way to solve the issue? The owner boundary is right in memory-lancedb, but the exact config shape, default behavior, ranking weights, dependency API use, and fallback semantics need maintainer product/API agreement before implementation.
