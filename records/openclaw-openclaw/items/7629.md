## Summary

Keep open: current main still leaves the bundled `memory-lancedb` plugin on vector-only retrieval, with no FTS/BM25 index setup, no hybrid config, and no query-text search path; the exact implementation PR was closed unmerged, and the external `memory-lancedb-pro` thread is a ClawHub/community alternative rather than bundled-plugin parity.

## What This Changes

Add hybrid BM25 plus vector retrieval to the bundled `memory-lancedb` plugin, including LanceDB FTS setup and configurable vector/text weighting.

## Best Possible Solution

Make an explicit maintainer decision on whether bundled `memory-lancedb` should gain hybrid retrieval; if yes, implement it plugin-locally with typed config, manifest/docs updates, FTS index maintenance, query-text propagation, vector-only fallback, and targeted regression tests.
