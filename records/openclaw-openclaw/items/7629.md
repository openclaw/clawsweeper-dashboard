## Summary

Current main still provides vector-only retrieval in the bundled memory-lancedb plugin, and the only linked implementation PR closed unmerged after mixing this feature with broad unrelated changes. The parity request remains useful, but its default, scoring, and fallback behavior require maintainer product judgment before a focused replacement implementation.

## What This Changes

Add BM25 full-text retrieval alongside vector similarity in memory-lancedb, including FTS indexing, query-text propagation, result fusion, and configuration.

## Best Possible Solution

Implement one focused memory-lancedb-owned hybrid query path against the pinned JavaScript API, carrying query text beside embeddings, defining stable score semantics, proving FTS lifecycle behavior, and using an explicit maintainer-approved default that preserves upgrades.

## Solution Assessment

Partly: plugin-local FTS plus vector fusion is the correct ownership boundary, but the proposed enabled-by-default weighted configuration is not established as the safest upgrade-compatible design.

- The score normalization and fusion contract must be defined because current minScore handling assumes L2-derived vector similarity rather than BM25 or hybrid relevance scores.
