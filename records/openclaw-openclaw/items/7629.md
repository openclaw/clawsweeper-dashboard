## Summary

Keep open: current main still leaves the bundled `memory-lancedb` plugin on vector-only retrieval, with no FTS/BM25 index setup, no hybrid config surface, and no query-text database search path. The earlier implementation attempt at https://github.com/openclaw/openclaw/pull/7636 was closed unmerged, and the remaining LanceDB parity work is a feature/API decision rather than a stale or already-implemented item.

## What This Changes

Requested adding BM25/full-text hybrid retrieval to the bundled `memory-lancedb` plugin so exact keyword matches can participate alongside vector similarity.

## Best Possible Solution

Implement a plugin-local `memory-lancedb` parity feature with typed config and manifest/docs entries, FTS index maintenance on `text`, original query propagation, verified LanceDB FTS/reranker or local fusion behavior, vector-only fallback, and focused regression coverage.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high confidence that the requested hybrid config, FTS setup, and query-text database search path are absent.

## Solution Assessment

Yes for the requested direction: `extensions/memory-lancedb` is the right owner boundary. The exact defaults, weighting, min-score behavior, and LanceDB-native versus local-fusion strategy need maintainer review before implementation.
