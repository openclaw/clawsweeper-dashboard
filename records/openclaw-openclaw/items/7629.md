## Summary

Keep open: current main still leaves the bundled `memory-lancedb` plugin on vector-only retrieval, with no FTS/BM25 index creation, no hybrid config surface, and no query-text database search path. The matching implementation at https://github.com/openclaw/openclaw/pull/7636 was closed unmerged, and no canonical open follow-up appears to own this LanceDB-specific parity work.

## What This Changes

Request hybrid BM25 plus vector search for `memory-lancedb`, including FTS index setup, hybrid weighting config, and query-text propagation into recall/search paths.

## Best Possible Solution

Implement a plugin-local `memory-lancedb` parity feature with typed config and manifest/docs entries, FTS index maintenance on `text`, original query propagation, verified hybrid search or tested local fusion fallback, vector-only fallback, and focused regression coverage.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high confidence that the requested hybrid config, FTS setup, and query-text database search path are absent.

## Solution Assessment

Yes for the requested direction: the narrow maintainable implementation belongs in `extensions/memory-lancedb`. The exact defaults, weighting, min-score semantics, and LanceDB-native versus local-fusion strategy need maintainer review first.
