## Summary

The requested hybrid retrieval is useful, but it no longer needs to be added to OpenClaw’s bundled plugin: the existing plugin API and LanceDB JavaScript contract are sufficient, and the published ClawHub `memory-lancedb-pro` plugin already provides vector-plus-BM25 retrieval. Keeping a second bundled implementation would add config, ranking, migration, and fallback policy to core for capability users can install externally.

## What This Changes

Add hybrid BM25 and vector retrieval to the bundled LanceDB memory plugin.

## Best Possible Solution

Keep the bundled plugin lean and direct users who need hybrid LanceDB retrieval to the published ClawHub plugin; revisit core only if that external implementation exposes a concrete missing plugin API.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request, and source inspection confirms the bundled plugin is vector-only while the requested behavior is available through an external plugin.

## Solution Assessment

No for adding it to the bundled plugin: the current plugin surface already supports an external implementation, and the published ClawHub solution avoids adding another configuration and compatibility contract to OpenClaw core.
