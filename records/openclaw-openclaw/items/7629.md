## Summary

Keep open: current main still leaves the bundled `memory-lancedb` plugin on vector-only LanceDB retrieval with no FTS/BM25 index, no hybrid config surface, and no query-text database search path; the matching implementation at https://github.com/openclaw/openclaw/pull/7636 was closed unmerged.

## What This Changes

Add BM25 plus vector hybrid search to the bundled `memory-lancedb` plugin, including FTS indexing, query-text propagation, and tunable retrieval weights.

## Best Possible Solution

Implement this as a plugin-local bundled `memory-lancedb` parity feature with typed config and manifest docs, FTS index maintenance on `text`, original query propagation, vector-only fallback, and focused regression coverage.

## Reproduction Assessment

Not applicable as a feature request. Source inspection of current main gives high confidence that the requested hybrid config, FTS setup, and query-text database search path are absent.

## Solution Assessment

Yes, the narrow maintainable direction is a plugin-local parity enhancement under `extensions/memory-lancedb`. The exact default enablement, weighting, and native LanceDB versus local fusion fallback strategy should get maintainer review first.
