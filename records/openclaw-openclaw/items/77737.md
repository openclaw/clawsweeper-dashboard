## Summary

Current main already fixes the reported 2026.5.3 upgrade failure by repairing missing plugin-local `openclaw` peer links during npm plugin update, which is the path that can leave the Brave plugin unable to register its `webSearchProviders` provider.

## What This Changes

OpenClaw should keep Brave available as managed `web_search` after upgrading to 2026.5.3 when the Brave plugin and API key are configured.

## Best Possible Solution

Keep the current-main peer-link repair and let the next release carry it to users affected by 2026.5.3 upgrades.

## Reproduction Assessment

Yes by source inspection: an unchanged npm plugin that declares `peerDependencies.openclaw` but lacks `node_modules/openclaw` would fail to resolve SDK imports and therefore not contribute Brave's provider registration. I did not run a live Docker/Brave API reproduction in this read-only review.

## Solution Assessment

Yes. Repairing the plugin-local `openclaw` peer link before treating an npm plugin as unchanged is narrower and safer than changing Brave credential logic or `web_search` provider fallback behavior.
