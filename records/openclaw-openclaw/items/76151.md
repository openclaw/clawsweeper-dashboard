## Summary

Keep open: current main still has the ownership-blocked plugin diagnostics defects, and this PR is a narrow fix with no blocking review findings. The linked #76144 should remain open for the remaining docs and container ownership-durability work.

## What This Changes

The branch carries ownership-blocked plugin id metadata through discovery diagnostics, deduplicates duplicate discovery diagnostics, rewrites configured-missing-plugin messages for ownership-blocked plugins, and adds discovery tests plus a changelog entry.

## Best Possible Solution

Land this diagnostics fix after maintainer approval, then keep #76144 open for the documentation and container ownership-durability follow-up.

## Reproduction Assessment

Yes. On current main, a UID mismatch creates a suspicious-ownership discovery diagnostic without `pluginId`, merged discovery results can append duplicates, and configured missing plugin ids still flow into stale-config warning text.

## Solution Assessment

Yes. Propagating `idHint` into diagnostics and deduplicating by plugin id, source, and message is the narrow maintainable fix; a direct config-validation regression test would be useful but is not a blocking alternative.
