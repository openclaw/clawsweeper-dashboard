## Summary

Keep open. Current main still scopes isolated cron progress to a cron run session and the Web UI only renders live agent/tool events for the selected or subscribed session; I did not find a current bridge that streams cron run progress into the watched chat before completion. The remaining blocker is product shape, not a small mechanical fix.

## What This Changes

The issue asks for cron job execution logs to stream into the Web UI chat session in real time instead of appearing only after the cron task completes.

## Best Possible Solution

Choose the intended cron progress surface, then add a focused live visibility path with coverage for isolated cron run sessions, selected chat filtering, and final delivery behavior.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection shows the reported visibility gap is source-reproducible: isolated cron progress is scoped to a per-run cron session while the Web UI live chat stream filters to the selected session.

## Solution Assessment

Unclear until maintainers choose the product surface. Streaming directly into the watched chat is plausible, but current docs and UI already treat isolated cron as a separate run session, so the safer fix starts with that product decision.
