## Summary

Keep open. Current main and v2026.6.8 still attach isolated cron live run output to a per-run cron session, while the Web UI selected-chat and tool-stream paths accept only the selected/equivalent session or selected global alias; no related issue or PR fully owns the watched-chat live-log behavior.

## What This Changes

The issue requests real-time cron job execution logs to appear in the watched Web UI chat session instead of only after the task completes.

## Best Possible Solution

Maintainers should choose the intended live-progress destination for isolated cron runs, then implement one consistent Web UI path with focused source tests and real Control UI cron-run proof.

## Reproduction Assessment

Yes, at source level: isolated cron runs use a per-run cron session key, while selected Web UI chat/tool handlers filter by the selected session key. A live Web UI cron recording or run-log/session snapshot is still missing.

## Solution Assessment

Unclear until maintainers choose the product surface. Streaming directly into the watched chat is plausible, but current code treats isolated cron runs as separately scoped automation, so the safer path starts with a destination decision.
