## Summary

Keep open. Current main routes some cron-run system events back toward the main/global session, but isolated cron agent execution still registers live run output on a per-run cron session while the Web UI selected-chat path remains exact-session scoped, so the watched-chat realtime log behavior is not clearly implemented.

## What This Changes

Issue requests real-time OpenClaw cron job execution logs to appear in the Web UI chat session as operations complete instead of only after the cron task finishes.

## Best Possible Solution

Maintainers should choose whether cron progress belongs in the selected chat, the cron run session, or a separate activity/cron surface, then implement that live visibility path without weakening cron session isolation.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection shows the reported visibility gap remains plausible because isolated cron progress is still attached to a per-run cron session while the selected Web UI chat path is exact-session scoped.

## Solution Assessment

Unclear until maintainers choose the product surface. Streaming directly into the watched chat is plausible, but current code treats cron runs as separately scoped automation, so the safer path starts with a destination decision.
