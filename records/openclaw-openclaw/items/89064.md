## Summary

Keep open. Current main and the latest release still make isolated cron execution run-scoped while the Web UI selected-chat path is one-session scoped, and the open cron chat-lane PR is only a partial overlap rather than a fix for this Web UI live-log destination.

## What This Changes

The issue reports that OpenClaw cron-job execution logs reach the Web UI chat only after the whole task completes instead of streaming after each assistant operation.

## Best Possible Solution

Define the intended live cron progress destination, then update cron run/session event routing and Web UI subscription/history handling so progress appears there without breaking isolated cron run sessions.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection shows a plausible current-main path where isolated cron progress is attached to a run-scoped session while the Web UI selected-chat path ignores nonmatching `session.message` events.

## Solution Assessment

Unclear until maintainers choose the product surface. Streaming into the selected chat is plausible, but current code also preserves run-scoped cron sessions and run-history chat links, so the destination decision should come before implementation.
