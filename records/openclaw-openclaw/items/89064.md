## Summary

Keep open. Current main and v2026.6.8 still do not provide live cron-operation streaming into the selected Web UI chat, but current docs describe cron as isolated/final-result delivery, so this needs maintainer UX direction before a fix PR.

## What This Changes

The issue asks for cron job live execution logs to appear in the Web UI chat session as operations complete instead of only after the whole task finishes.

## Best Possible Solution

Maintainers should choose and document the intended live-progress destination for cron runs, then align Control UI streaming behavior with that contract and add focused source plus real Web UI proof.

## Reproduction Assessment

Yes at source level: isolated cron events use a per-run session key and Web UI chat/tool handlers filter against the selected session key. A live Ubuntu/npm Web UI cron reproduction has not been run in this read-only review.

## Solution Assessment

Unclear; streaming directly into the watched chat may be useful, but current code and docs treat cron execution as separately scoped automation with final-result delivery, so the best solution needs maintainer UX direction first.
