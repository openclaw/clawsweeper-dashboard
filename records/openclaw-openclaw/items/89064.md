## Summary

Keep open. Current main and v2026.6.9 still do not define or implement live cron-operation streaming into the watched Web UI chat; source shows run-scoped cron events filtered away from other selected sessions, and maintainer UX direction is needed before repair.

## What This Changes

The issue requests real-time cron-job assistant operation logs in the Web UI chat session while the cron run is still in progress.

## Best Possible Solution

Define the intended live-progress destination for cron runs, then align cron event routing, Web UI stream filtering, docs, and focused tests with that contract.

## Reproduction Assessment

Yes at source level: current main emits isolated cron activity under a run-scoped session key while the Web UI chat/tool handlers filter against the selected session key. A live Ubuntu/npm Web UI cron reproduction has not been run in this read-only review.

## Solution Assessment

Unclear; streaming into the watched chat may be useful, but current code and docs treat cron runs as isolated automation with final-result delivery, so maintainers should choose the intended live-progress destination first.
