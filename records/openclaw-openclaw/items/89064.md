## Summary

Keep open. Current main still keeps isolated cron execution progress on a per-run cron session while the Web UI chat path only renders live agent/tool events for the selected or subscribed session, so the requested watched-chat streaming behavior remains missing and needs a product decision before implementation.

## What This Changes

The issue asks for cron job execution logs to appear live in the Web UI chat session instead of only after the cron task completes.

## Best Possible Solution

Choose the intended cron progress destination, then add a focused live visibility path that preserves session isolation and final delivery behavior.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection shows the reported visibility gap is source-reproducible because cron progress is attached to an isolated run session while the Web UI chat stream accepts only the selected/subscribed session.

## Solution Assessment

Unclear until maintainers choose the product surface. Streaming directly into the watched chat is plausible, but current code and docs treat cron runs as separately scoped automation, so the safer path starts with that destination decision.
