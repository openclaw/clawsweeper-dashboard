## Summary

Keep open. Current main and the latest release still route isolated cron live output through run-scoped cron sessions while the selected Web UI chat path is exact-session scoped, and the related open cron chat-lane PR addresses a different session-key stamping bug rather than this real-time Web UI log destination.

## What This Changes

The issue asks for real-time cron job execution logs to appear in the watched Web UI chat session instead of only after completion.

## Best Possible Solution

Define the intended live cron progress destination, then update cron run/session event routing and Web UI subscription/history handling so progress appears there without breaking isolated cron run sessions.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection shows the reported gap remains plausible because isolated cron progress is attached to a run-scoped cron session while the selected Web UI chat path ignores nonmatching session.message events.

## Solution Assessment

Unclear until maintainers choose the product surface. Streaming into the selected chat is plausible, but current code deliberately preserves run-scoped cron sessions and a run-history chat link, so the destination decision should come before implementation.
