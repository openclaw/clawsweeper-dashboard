  - body: The schema now accepts temporalDecay.model and temporalDecay.weibullShape, but ResolvedMemorySearchConfig and mergeConfig still preserve only enabled and halfLifeDays. Users can configure `model: "weibull"` successfully and still get the exponential default at runtime.
  - confidence: 0.94
- **[P2] Use one temporal-decay config surface:** `extensions/memory-core/openclaw.plugin.json:25-44`
  - body: This adds `plugins.entries.memory-core.config.temporalDecay`, but memory scoring is resolved from `agents.*.memorySearch.query.hybrid.temporalDecay`. Unless runtime code reads this plugin field, users configuring it get an accepted no-op; otherwise remove it and keep the documented memorySearch path.
  - confidence: 0.88
- **[P2] Correct the beta less-than-one guidance:** `docs/pr-weibull-decay.md:40`
  - body: For `0 < beta < 1`, `(t / halfLife)^beta` is larger before half-life and smaller after half-life, so decay is faster early and slower late. The added table says the inverse, which would make users tune the wrong curve.
  - confidence: 0.95
- **[P2] Align shape validation with normalization:** `src/config/zod-schema.agent-runtime.ts:760`
  - body: The runtime helper clamps out-of-range shapes, but the config schema rejects the same values with min/max validation while the PR text and docs describe clamping and self-healing. Either accept and normalize those values or document strict config validation instead.
  - confidence: 0.9
