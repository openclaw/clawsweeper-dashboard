## Summary

Keep open. Current main does not implement Weibull temporal decay, and the PR head still has blocking correctness and documentation issues: the new config keys are accepted by schema but not carried through the resolved memory-search config path, the β < 1 docs describe the inverse curve, and config validation does not match the documented clamp behavior.

## What This Changes

The PR adds an opt-in Weibull temporal decay model to memory-core scoring, updates config/schema/type surfaces and generated schema output, adds a PR design doc, and adds targeted Weibull temporal-decay tests.

## Best Possible Solution

Land a revised memory-core change that keeps exponential as the default, threads model and weibullShape through resolveMemorySearchConfig into MemoryIndexManager, aligns schema validation with the documented runtime behavior, fixes the β-direction docs, updates user-facing memory config docs/changelog if maintainers accept the feature, and leaves broader importance/access-boost work to the canonical memory-quality issue.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: configure memorySearch.query.hybrid.temporalDecay.model as "weibull" on the PR head, and the schema accepts it while resolveMemorySearchConfig still returns only enabled and halfLifeDays to memory-core.

## Solution Assessment

No as-is. The math and opt-in default are plausible, but the PR is not the best maintainable solution until the resolved config path, docs, and validation semantics match the advertised behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Thread the model through resolved memory config:** `src/config/zod-schema.agent-runtime.ts:766-767`
  - body: The schema accepts temporalDecay.model and temporalDecay.weibullShape, but the resolver that feeds memory search still preserves only enabled and halfLifeDays. A user setting model: "weibull" in agent memorySearch config will still run the exponential default because those fields never reach MemoryIndexManager.
  - confidence: 0.93
- **[P2] Correct the beta less-than-one decay text:** `docs/pr-weibull-decay.md:39`
  - body: For 0 < β < 1, (t / halfLife)^β is larger than the exponential ratio before half-life and smaller after half-life, so the multiplier decays more early and less later. The added table says the opposite, which would teach users to tune the inverse curve.
  - confidence: 0.95
- **[P2] Align shape validation with clamp semantics:** `src/config/zod-schema.agent-runtime.ts:767`
  - body: The runtime helper clamps out-of-range shapes, but z.number().min(0.1).max(5) rejects those same user config values before runtime. Either accept and clamp through the runtime path or document strict config validation instead of promising self-healing bounds.
  - confidence: 0.9
