## Summary

Keep open. Current main is still exponential-only, and this PR is a useful but incomplete contributor feature: the branch adds accepted Weibull config fields that are not fully wired through the resolved memory-search path, also leaving misleading plugin config/docs/validation surfaces before merge.

## What This Changes

The PR adds an opt-in Weibull temporal decay model for memory-core hybrid scoring, extends config schemas/types/plugin metadata, adds a design doc, and adds targeted temporal-decay tests.

## Best Possible Solution

Land a revised memory-core change only if maintainers accept the feature: keep exponential as the default, expose one documented config path, thread model and weibullShape through resolveMemorySearchConfig into MemoryIndexManager, align schema behavior with runtime normalization, update the public memory configuration docs and changelog, and leave broader importance/access-boost work to #57307.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path on the PR head: configure memorySearch.query.hybrid.temporalDecay.model as "weibull" and the schema accepts it, but resolveMemorySearchConfig still returns only enabled and halfLifeDays.

## Solution Assessment

No as-is. The opt-in math is plausible and backward-compatible, but the current patch is not the best maintainable solution until config routing, plugin metadata, docs, validation semantics, and release notes describe the same behavior.
