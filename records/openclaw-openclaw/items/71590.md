## Summary

Keep open. Current main still has only the exponential temporal decay model, and the PR head still has concrete merge blockers: the new accepted config keys are not carried through resolved memory settings, a plugin-level config path appears to be an accepted no-op, docs still invert beta less-than-one behavior, validation semantics conflict with runtime clamping, and the user-facing feature lacks a changelog entry.

## What This Changes

This PR adds an opt-in Weibull temporal decay model to memory-core hybrid scoring, extends memory search schemas/types and plugin metadata, adds a design doc, and adds Weibull temporal-decay tests.

## Best Possible Solution

Land a revised memory-core change only if maintainers accept the feature: keep exponential as the default, expose one documented config path under memorySearch, thread model and weibullShape through resolveMemorySearchConfig into MemoryIndexManager, align schema validation with runtime normalization, update public memory docs and changelog, and leave broader importance/access-boost work to #57307.

## Reproduction Assessment

Yes. On the PR head, configure agents.defaults.memorySearch.query.hybrid.temporalDecay.model as "weibull"; the schema accepts it, but resolveMemorySearchConfig still returns only enabled and halfLifeDays, so the manager receives the default exponential model.

## Solution Assessment

No as-is. The opt-in math and backward-compatible default are plausible, but the implementation is not the narrowest maintainable solution until config routing, plugin metadata, docs, validation behavior, and release notes all describe the same behavior.
