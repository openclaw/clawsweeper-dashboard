## Summary

Keep open. Current main still follows the configured file-log fallback for implicit local `logs.tail` failures and does not implement the requested Linux/systemd journal preference or provenance output; the reporter’s validated prototype is useful context but is not on main and no linked PR closes this issue.

## What This Changes

The issue asks for Linux `openclaw logs --follow` to prefer the active systemd gateway journal and print CLI/service/log provenance before tailing.

## Best Possible Solution

Add a Linux-only implicit-local fallback that detects the active managed gateway service, tails its user journal by PID when RPC cannot provide logs, prints concise redacted provenance and mismatch warnings, and leaves explicit remote RPC plus Control UI file-log behavior unchanged.

## Solution Assessment
