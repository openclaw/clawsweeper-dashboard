## Summary

Keep open. Current main still has the reported failure class: bundled MCP/LSP tools are appended after provider schema normalization, LiteLLM does not register schema-normalization hooks, and the OpenAI/Gemini schema cleaners still miss the specific proxy/tool-schema cases described in the issue discussion.

## What This Changes

Request to make OpenClaw normalize or clearly diagnose MCP tool schemas before LiteLLM-routed OpenAI/Gemini providers reject MS365/workspace-style tools.

## Best Possible Solution

Add provider-aware normalization and diagnostics for final effective tools, including bundled MCP/LSP tools, with regression coverage for OpenAI-compatible LiteLLM and Gemini proxy schemas rather than an MS365-specific workaround.
