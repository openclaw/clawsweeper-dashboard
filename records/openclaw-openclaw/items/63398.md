## Summary

Keep open. Current main still has the reported failure class: bundled MCP/LSP tools are appended after provider schema normalization, LiteLLM has no schema-normalization hook, and the OpenAI/Gemini compatibility helpers do not cover LiteLLM proxy routes or all malformed nested array schemas described in the latest comment.

## What This Changes

OpenClaw should sanitize provider-incompatible MCP tool schemas before LiteLLM-routed OpenAI/Gemini fallback requests instead of sending raw MCP input schemas that upstream validators reject.

## Best Possible Solution

Add final effective-tool schema normalization and diagnostics for bundled MCP/LSP tools on provider-compatible routes, with targeted regression coverage for LiteLLM-routed OpenAI/Gemini schemas containing nested nullable type arrays and array items gaps.

## Reproduction Assessment
