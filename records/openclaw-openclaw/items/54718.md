## Summary

Close as superseded: current main and the active RFC have moved trusted-proxy loopback handling away from this branch’s implicit token fallback and toward explicit loopback opt-in plus separate local service identity design.

## What This Changes

The PR adds trusted-proxy loopback shared-secret fallback behavior and related auth tests in `src/gateway/auth.ts` and `src/gateway/auth.test.ts`.

## Best Possible Solution

Keep the current explicit trusted-proxy contract and handle remaining internal/local caller auth through the canonical service-identity RFC rather than rebasing this fallback branch.

## Reproduction Assessment

Yes. Source inspection is enough: the PR adds trusted-proxy token fallback, while current startup validation rejects trusted-proxy plus token before that runtime path can be used.

## Solution Assessment

No. The safer current direction is the open RFC’s separate local service identity path, not adding another token fallback and implicit loopback trusted-proxy acceptance branch.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep token fallback aligned with startup validation:** `src/gateway/auth.ts:390-396`
  - body: This fallback accepts `auth.token` in trusted-proxy mode, but `assertGatewayAuthConfigured` still throws when trusted-proxy and a shared token are configured. A real gateway using the advertised token setup fails during startup before this branch can run, while the new tests bypass startup validation by calling `authorizeGatewayConnect` directly.
  - confidence: 0.91
- **[P1] Require the explicit loopback proxy opt-in:** `src/gateway/auth.ts:301-302`
  - body: This changes loopback trusted-proxy requests to pass when they have a non-local Host and forwarding headers, without requiring `gateway.auth.trustedProxy.allowLoopback`. Current main and docs make loopback proxy trust an explicit operator opt-in because local processes can try to send identity headers; this would widen that auth boundary silently.
  - confidence: 0.88

## Security Review

Status: needs_attention

  - confidence: 0.9

  - confidence: 0.86
