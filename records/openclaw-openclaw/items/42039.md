## Summary

Keep open: current main still has the requested gap, and related reports have already been consolidated here rather than superseding it. Source inspection shows the missed-job controls exist only as internal service dependencies, the public config schema/docs omit them, and the exact disable semantics need maintainer product/API judgment.

## What This Changes

Expose cron missed-job restart controls in public configuration so operators can limit, stagger, or disable startup catch-up runs.

## Best Possible Solution

Add a backward-compatible public cron missed-run policy owned by the cron config/job contract, with schema/types/help/docs/tests updated together and explicit semantics for skip, limit, stagger, and one-shot jobs.

## Reproduction Assessment

Yes, at source level. Current main omits the requested keys from CronConfig and the strict zod cron schema, while startup still invokes runMissedJobs and the internal deps remain unreachable from openclaw config.

## Solution Assessment

Unclear. The requested internal knobs are relevant, but exposing them directly is not enough because maxMissedJobsPerRestart: 0 currently defers instead of disables missed jobs; maintainers should choose an explicit public missed-run policy before implementation.
