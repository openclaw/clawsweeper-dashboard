## Summary

Keep open: current main still keeps the missed-job restart controls internal, strict public config rejects the requested cron keys, and related duplicate reports were already closed back to this canonical API/product decision.

## What This Changes

Expose cron missed-job restart controls so operators can limit, stagger, or disable startup catch-up behavior from public cron configuration.

## Best Possible Solution

Define a backward-compatible cron missed-run policy, then update config types, strict schema/help, docs, gateway wiring, optional per-job protocol, and restart catch-up tests together.

## Reproduction Assessment

Yes. Source inspection shows current main rejects the requested public cron keys, does not wire them into CronService, and still replays or defers startup missed jobs without a public skip policy.

## Solution Assessment

No for the literal expose-only proposal. The safer solution is an explicit missed-run policy because maxMissedJobsPerRestart: 0 currently defers eligible jobs instead of skipping catch-up.
