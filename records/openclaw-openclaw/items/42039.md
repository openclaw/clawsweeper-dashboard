## Summary

Keep open. Current main still keeps the missed-job restart controls internal, while duplicate reports have already been closed into this issue as the canonical tracker.

## What This Changes

Expose cron missed-job restart controls in public OpenClaw config so users can limit, stagger, or disable startup catch-up behavior.

## Best Possible Solution

Define a backward-compatible public cron missed-run policy, wire it through config schema/help/docs into CronService deps, and cover restart catch-up behavior with focused tests.

## Reproduction Assessment

Yes, from source inspection. Persist an enabled recurring cron job with a past nextRunAtMs and start the cron service: startup catch-up collects runnable missed jobs, while the strict public cron config schema still rejects the requested keys.

## Solution Assessment

Unclear. Exposing the two internal names is narrow, but the issue also asks for disable/skip semantics that the current maxMissedJobsPerRestart implementation does not provide by itself.
