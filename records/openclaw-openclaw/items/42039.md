## Summary

Keep open: current main still keeps the missed-job restart controls internal, rejects the requested public cron keys, and has no global or per-job skip policy. The related duplicate reports point back here, so this remains the canonical product/API decision rather than a cleanup close.

## What This Changes

Expose public cron missed-job restart controls so operators can limit, stagger, or skip catch-up runs after gateway restart.

## Best Possible Solution

Define a backward-compatible cron missed-run policy, then update config types, strict schema/help, docs, gateway wiring, optional per-job protocol, and restart catch-up tests together.

## Reproduction Assessment

Yes. Source inspection shows current main rejects the requested public cron keys, does not wire them into CronService, and still runs or defers startup missed jobs without a public skip policy.

## Solution Assessment

No for the literal expose-only proposal. The safer solution is an explicit missed-run policy because maxMissedJobsPerRestart: 0 currently defers eligible jobs instead of skipping catch-up.
