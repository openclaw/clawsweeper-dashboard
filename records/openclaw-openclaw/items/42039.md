## Summary

Keep open: current main still keeps the missed-job controls internal, while the public cron config, docs, and job protocol lack the requested restart catch-up policy. Related duplicate reports already point back here, so this remains the canonical product/API decision.

## What This Changes

Requests public cron missed-job catch-up controls, including `maxMissedJobsPerRestart`, `missedJobStaggerMs`, and optionally per-job skip behavior.

## Best Possible Solution

Define a backward-compatible cron missed-run policy, then update config types, strict schema/help, docs, gateway wiring, per-job protocol if chosen, and restart catch-up tests together.

## Reproduction Assessment

Yes. This is source-reproducible: current main rejects the requested public cron keys, does not wire them through the gateway, and still runs or defers startup missed jobs without a public skip policy.

## Solution Assessment

No for the literal expose-only proposal. The safer solution is an explicit missed-run policy because `maxMissedJobsPerRestart: 0` currently defers eligible missed jobs rather than skipping catch-up.
