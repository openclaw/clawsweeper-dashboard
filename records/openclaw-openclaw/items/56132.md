## Summary

Keep open: current main and v2026.6.10 still support Feishu/Lark current-conversation binding only for DMs or topic-scoped groups, so plain group `thread: true` subagent routing remains unresolved and needs maintainer product direction.

## What This Changes

Request to restore Feishu/Lark plain group `sessions_spawn({ thread: true, mode: "session" })` binding so different group chats can route follow-ups to dedicated subagent personas.

## Best Possible Solution

Define the Feishu plain-group current-conversation binding contract first, then align plugin-owned binding resolution, inbound routing, docs, and regression tests to that policy while preserving the existing DM/topic behavior.

## Reproduction Assessment

Yes, at source level. Current main still gates Feishu current-conversation binding to DMs or topic-scoped groups, and existing Feishu hook tests expect non-topic group spawns to return the reported error.

## Solution Assessment

Unclear as an implementation choice. Enabling plain groups mechanically is plausible, but the maintainable fix needs a maintainer-approved policy for group-level versus sender/topic/static binding semantics.
