## Summary

Keep open: current main and v2026.5.22 still reject Feishu/Lark plain group-chat `thread=true` subagent binding, and the related broader Feishu group-subagents request does not cleanly supersede this narrower binding-policy decision.

## What This Changes

Request to restore Feishu/Lark group-chat `thread=true` subagent binding so plain groups can bind to dedicated subagent personas, not only DMs or topic conversations.

## Best Possible Solution

Make an explicit Feishu plugin policy, then either support plain-group current-conversation subagent binding with delivery and ambiguity tests or document static group bindings and topic-only thread binding as the supported path.

## Reproduction Assessment

Yes. Source inspection shows a Feishu `to: "chat:..."` request with `threadRequested` and no thread id returns null and emits the reported error, and the current test suite asserts that unsupported path.

## Solution Assessment

Unclear. Restoring plain-group binding is plausible, but maintainers first need to choose whether the binding key is the group id, sender-scoped id, topic-scoped id, static group binding, or intentionally unsupported.
