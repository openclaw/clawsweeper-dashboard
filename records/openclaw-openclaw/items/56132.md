## Summary

Keep open: current main still rejects Feishu plain-group `thread=true` subagent current-conversation binding, while the documented support is for DMs, topic threads, and static group routing. The remaining work is a Feishu routing/product policy decision, not conservative cleanup closure.

## What This Changes

Restore Feishu plugin subagent `thread=true` current-conversation binding for plain group chats so each group can bind to its own subagent persona.

## Best Possible Solution

Make an explicit Feishu product decision, then either implement plain-group current-conversation subagent binding in the Feishu plugin with delivery and ambiguity tests or document static group `bindings` as the supported replacement.

## Reproduction Assessment

Yes. Source inspection on current main shows `to: "chat:oc_group_chat"` with no `threadId` resolves to null and returns the reported error, and the Feishu hook test asserts that unsupported path.

## Solution Assessment

Unclear. Restoring support may match the reporter's former workflow, but maintainers first need to choose the safe Feishu routing policy for plain groups versus static group bindings and topic/sender-scoped sessions.
