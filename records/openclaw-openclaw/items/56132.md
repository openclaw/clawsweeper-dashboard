## Summary

Keep open: current main and v2026.6.10 still limit Feishu/Lark current-conversation subagent thread binding to DMs or topic-scoped groups, so plain non-topic group `thread=true` binding remains unresolved and needs a maintainer policy decision.

## What This Changes

Restore Feishu/Lark plain group `thread=true` subagent session binding so a group chat can stay bound to a dedicated subagent instead of routing everything through the main agent.

## Best Possible Solution

Define the Feishu plain-group current-conversation binding contract, then align the plugin-owned resolver/binding path, inbound routing, docs, and regression tests while preserving existing DM and topic behavior.

## Reproduction Assessment

Yes, at source level. Current main still rejects non-topic Feishu group current-conversation binding and existing tests assert the reported error; I did not run a live Feishu/Lark tenant.

## Solution Assessment

Unclear as an implementation choice. Enabling plain groups mechanically is plausible, but the maintainable fix needs a maintainer-approved policy for whole-group versus sender/topic/static binding semantics.
