## Summary

Keep open: current main still rejects the Feishu plain-group current-conversation binding path, and changing that behavior needs maintainer product/routing judgment rather than cleanup closure.

## What This Changes

Restore Feishu plain group `sessions_spawn({ thread: true, mode: "session" })` current-conversation binding so groups can keep dedicated subagent sessions.

## Best Possible Solution

Make an explicit Feishu product decision, then either implement plain-group current-conversation subagent binding in the Feishu plugin with delivery and ambiguity tests or document static group `bindings` as the supported replacement.

## Reproduction Assessment

Yes. Source inspection shows a Feishu requester with `to: "chat:oc_group_chat"`, no `threadId`, and `threadRequested: true` resolves to `null` and returns the reported error; the existing Feishu hook test asserts that behavior.

## Solution Assessment

Unclear. Restoring support may be the right user-facing fix, but maintainers first need to choose whether runtime plain-group binding is supported or whether static group routing is the intended model.
