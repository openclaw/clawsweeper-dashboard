## What This Changes

The issue requests restoring Feishu/Lark plain group-chat `thread: true` subagent current-conversation binding so group-specific child sessions/personas can receive follow-up messages.

## Best Possible Solution

## Solution Assessment

Unclear: enabling plain group binding mechanically may work, but it would choose product semantics before maintainers decide between group-level, sender-scoped, topic-scoped, static binding, or unsupported behavior.
