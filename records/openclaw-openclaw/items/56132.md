## Summary

Keep open: current main and v2026.6.6 still reject Feishu/Lark plain group-chat `thread=true` subagent binding, while static group routing and DM/topic current-conversation binding are adjacent but do not restore the requested plain-group behavior. The related Feishu group-subagents and channel-threading-policy issues overlap, but neither cleanly supersedes this narrower product decision.

## What This Changes

The issue asks to restore Feishu/Lark `thread=true` subagent current-conversation binding for plain group chats so each group can keep a dedicated subagent persona.

## Best Possible Solution

Make an explicit Feishu plugin policy for plain group current-conversation binding, then either support it with delivery and ambiguity tests or document static group bindings and topic-only thread binding as the supported path.

## Reproduction Assessment

Yes, at source level. A Feishu requester with `to: "chat:..."`, `threadRequested: true`, and no topic/thread id resolves to null and emits the reported direct-message-or-topic error.

## Solution Assessment

Unclear. Restoring plain group binding is plausible, but maintainers first need to choose whether the binding key is the group id, sender-scoped id, topic-scoped id, static group binding, or intentionally unsupported.
