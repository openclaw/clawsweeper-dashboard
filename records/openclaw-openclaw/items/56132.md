## Summary

Keep open: current main still fails closed for Feishu plain-group `thread=true` subagent binding, and the related open Feishu group-subagents issue is broader rather than an exact owner for this plain-group binding policy decision.

## What This Changes

The issue requests restoring Feishu group-chat `sessions_spawn(..., thread=true)` subagent session binding so each group can keep a dedicated subagent/persona.

## Best Possible Solution

Make an explicit Feishu plugin policy, then either support plain-group current-conversation subagent binding with delivery and ambiguity tests or document static group bindings/the broader group-subagent design as the supported path.

## Reproduction Assessment

Yes. Source inspection on current main shows `to: "chat:oc_group_chat"` without a Feishu topic/thread id resolves to null and returns the reported error, and the current test suite asserts that unsupported path.

## Solution Assessment

Unclear. A patch could add plain-group binding, but maintainers first need to choose whether a plain group should bind to the group id, sender-scoped id, topic-scoped id, static group bindings, or remain unsupported.
