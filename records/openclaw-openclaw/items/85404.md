## Summary

Keep this PR open for normal maintainer merge review: current main has a source-visible race where same-key requests can mint different session IDs before the later session-key persistence and execution lane, while the PR is focused, has live after-fix proof, and has no blocking review findings.

## What This Changes

The branch adds a per-session-key reservation helper for brand-new agent session resolution, wires agent-command preparation to use it, adds regression tests, and updates the changelog.

## Best Possible Solution

Land the reservation helper after maintainers accept pre-execution session-key reservation as the intended invariant for stable OpenAI-compatible session keys.

## Reproduction Assessment

Yes, at source level: current main resolves a missing session-key entry by minting a UUID before later session-key persistence and execution queueing. I did not run the live repro in this read-only review, but the PR body supplies after-fix live Gateway/curl proof.

## Solution Assessment

Yes, this is a narrow maintainable fix: it serializes only the brand-new session-key reservation path while leaving explicit session ids and already-fresh sessions off the queue. The only maintainer decision is accepting early minimal reservation as the intended session-state behavior.
