## What This Changes

The PR sets Content-Disposition on Control UI assistant-media responses so image media is inline and non-image media downloads with the original basename.

## Best Possible Solution

Encode Content-Disposition with a Node-valid ASCII filename fallback plus UTF-8 filename*, keep previewable media inline and documents as attachments, and require redacted real HTTP or browser proof before merge.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.10 stream assistant-media files without Content-Disposition, and the linked issue provides document-download steps. I did not run a live browser save-dialog reproduction in this read-only review.

## Solution Assessment

No. Server-side attachment disposition is the right direction, but this branch needs Node-safe filename encoding and real HTTP or browser proof before it is the best mergeable fix.
