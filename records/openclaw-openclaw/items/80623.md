## Summary

Keep open. The reported failure is source-reproducible, but this PR changes the public manifest contract by treating an invalid boolean value as wildcard tool ownership, and it still lacks after-fix proof from a real plugin/runtime flow.

## What This Changes

The PR converts `contracts.tools: true` to `["*"]`, lets `"*"` satisfy plugin tool contract checks, and adds helper tests for wildcard declarations.

## Best Possible Solution

Keep `contracts.tools` as explicit tool names unless maintainers approve and document a wildcard manifest mode; otherwise fix the linked issue with clearer invalid-manifest diagnostics and plugin-author docs.

## Reproduction Assessment

Yes. Source inspection against current `origin/main` shows `contracts.tools: true` is dropped by `normalizeTrimmedStringList`, then `registerTool` emits the reported missing-contract diagnostic.

## Solution Assessment

No. The PR solves the symptom by adding a new wildcard manifest mode, but the narrower maintainable path is diagnostics/docs unless maintainers explicitly approve the API change.
