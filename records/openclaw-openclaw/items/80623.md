## Summary

Keep open for maintainer review: the patch is focused, but it changes the public manifest contract from explicit `string[]` tool ownership to a boolean wildcard, which current docs and the linked issue triage say should remain an explicit array unless maintainers choose a new API mode. The PR also lacks required after-fix real behavior proof.

## What This Changes

The PR maps `contracts.tools: true` to `['*']`, treats `'*'` as allowing any plugin tool name, and adds wildcard helper tests.

## Best Possible Solution

Keep `contracts.tools` explicit unless maintainers intentionally approve and document a wildcard manifest mode; otherwise fix the linked report with clearer invalid-manifest diagnostics and plugin-author docs.

## Reproduction Assessment

Yes. Source inspection shows current main drops boolean `contracts.tools` through `normalizeTrimmedStringList`, then `registerTool` emits the reported missing-contract diagnostic.

## Solution Assessment

No. The proposed boolean-to-wildcard behavior is a new manifest mode, while the current contract and prior linked triage point to diagnostics/docs unless maintainers explicitly choose that API change.
