## Summary

Keep open for maintainer decision: current main and public docs require explicit string-array tool contracts, while this PR changes the public manifest contract to wildcard ownership and lacks real behavior proof.

## What This Changes

This PR treats `contracts.tools: true` as `['*']`, adds wildcard handling to plugin tool contract checks, and adds helper tests.

## Best Possible Solution

Keep explicit tool-name arrays unless maintainers intentionally approve and document a wildcard manifest mode; otherwise resolve the linked report with clearer invalid-manifest diagnostics and plugin-author docs.

## Reproduction Assessment

Yes by source inspection, not by a live run: current main normalizes non-array `contracts.tools` to `[]`, and `registerTool` then emits the reported missing-contract diagnostic when no declared names remain.

## Solution Assessment

No. Accepting boolean `true` as wildcard ownership is a public manifest contract change; the narrower maintainable fix is diagnostics/docs unless maintainers approve the new mode.
