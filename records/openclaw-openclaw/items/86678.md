## Summary

## What This Changes

This PR moves session snapshot cloning/freezing from session-store writes to lazy snapshot reads and resolves external OAuth profiles once per auth-profile save, with focused regression tests.

## Best Possible Solution

Land the narrow performance fix only after required CI plus maintainer acceptance of the exact-commit proof gap or added live/profile proof, while keeping auth/session contracts unchanged.

## Reproduction Assessment

Yes, source-level: current main eagerly writes frozen session snapshots during session-store writes and resolves external auth profiles inside each OAuth persistence check. I did not run a live gateway CPU profile locally.

## Solution Assessment

Yes, with a proof caveat: deferring immutable snapshot publication to reads and reusing one external-auth resolution per save is the narrow maintainable performance shape. The remaining question is whether maintainers want exact-head live/profile proof before merge.
