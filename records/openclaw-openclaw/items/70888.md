## Summary

Keep open: current main still has the no-config early return that skips persisted session-store metadata, while this PR applies a narrow helper fix with focused regression coverage and sufficient terminal proof.

## What This Changes

The PR lets no-config subagent depth lookups read the default persisted session store and adds depth/announce regression tests plus changelog text.

## Best Possible Solution

Land the narrow helper change with its regression tests after normal maintainer merge and check handling, keeping routing and delivery policy unchanged.

## Reproduction Assessment

Yes, source-reproducible. Create a default session-store entry for an `agent:main:explicit:...:acct:default` requester and call `getSubagentDepthFromSessionStore(key)` without `cfg`; current main exits before the disk lookup and falls back to depth 0.

## Solution Assessment

Yes. Removing the no-config guard is the narrow maintainable fix because `resolveStorePath(undefined, ...)` already defines the default-store behavior and `readSessionStore()` handles absent or invalid files safely.
