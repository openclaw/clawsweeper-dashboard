The PR restores default on-disk session-store lookup for no-config subagent depth checks, adds regression tests for explicit requester announce classification, updates the changelog, and refreshes one Swift protocol model field.

## Reproduction Assessment

## Solution Assessment

Yes. Removing the early `cfg` guard is the narrow maintainable fix because it lets the existing `resolveStorePath()` default-store contract run while preserving missing-store fallback behavior; the remaining concern is CI validation, not the design of the patch.
