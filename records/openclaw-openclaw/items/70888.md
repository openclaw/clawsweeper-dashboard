## Summary

Keep open for normal review and merge handling: current main still has the no-`cfg` early return that makes the reported depth lookup source-reproducible, and this PR is a narrow fix with regression coverage and no blocking review findings.

## What This Changes

The PR lets no-config subagent depth lookups read the default persisted session store for agent-scoped explicit requester keys, with depth/announce regression tests and a changelog entry.

## Best Possible Solution

Land the narrow helper change with the regression tests after normal maintainer check and merge handling, keeping routing and delivery policy unchanged.

## Reproduction Assessment

Yes, source-reproducible. On current main, a default session-store entry for an `agent:main:explicit:...:acct:default` key is skipped because `resolveEntryForSessionKey()` exits when `cfg` is omitted, so fallback segment counting returns depth 0.

## Solution Assessment

Yes. Removing the no-config guard is the narrow maintainable fix because `resolveStorePath(undefined, ...)` already defines the default-store behavior and `readSessionStore()` safely handles missing or invalid files.
