Keep this PR open: current main still has the no-config early return that skips the default session-store lookup, and the PR is a narrow, well-scoped fix with targeted regression coverage and sufficient terminal proof.

## What This Changes

The branch removes the no-config early return in subagent depth lookup, lets `resolveStorePath` use the default session store, and adds depth/announce regression tests plus a changelog entry.

## Best Possible Solution

Land the narrow helper change with its regression tests after normal maintainer merge/check handling; keep routing and delivery policy unchanged.
