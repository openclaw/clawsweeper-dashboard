## Summary

## What This Changes

The PR removes the no-config early return from subagent depth lookup, adds explicit requester/default-store regression tests for depth and announce classification, and adds a changelog entry.

## Best Possible Solution

Merge the focused helper fix, regression tests, and changelog entry after exact-head CI is green or the unrelated failures are explicitly triaged.

## Solution Assessment

Yes. Removing the no-cfg early return is the narrowest maintainable fix because `resolveStorePath()` already handles omitted store config, and the added tests cover stored depth, stored ancestry, missing-store fallback, and announce classification.
