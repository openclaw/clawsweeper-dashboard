## What This Changes

The PR lets subagent depth resolution read the default on-disk session store when cfg is omitted and adds regression tests for explicit acct-suffixed requester sessions plus announce classification.

## Best Possible Solution

Land the narrow helper fix after normal maintainer review and validation, keeping the behavior scoped to generic session-depth resolution rather than changing routing or delivery policy.

## Reproduction Assessment

Yes. Persist an agent:main:explicit:...:acct:default entry with spawnDepth in the default sessions.json and call getSubagentDepthFromSessionStore(key) without cfg; current main returns depth 0 because it skips the store and falls back to segment counting.

## Solution Assessment

Yes. Removing the cfg early return and using params.cfg?.session?.store is the narrowest maintainable fix because resolveStorePath() already owns the default sessions.json fallback.
