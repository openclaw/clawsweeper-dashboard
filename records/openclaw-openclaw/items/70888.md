## Summary

Keep open for normal maintainer review and merge handling: current main and the latest release still skip the default session store in no-config depth lookups, while this PR is narrow, covered by focused regressions, and I found no blocking correctness or security issue.

## What This Changes

The PR removes the no-config early return in subagent depth resolution, lets agent-scoped keys read the default session store, and adds depth/announce classification regression tests plus a changelog entry.

## Best Possible Solution

Land the narrow helper change with the regression tests after normal maintainer checks, keeping routing and delivery policy unchanged.

## Reproduction Assessment

Yes. On current main, a persisted explicit requester key reaches no-cfg depth lookup from announce code, resolveEntryForSessionKey returns before reading the default store, and the explicit key falls back to depth 0; the PR body also supplies terminal proof of the before/after behavior.

## Solution Assessment

Yes. Removing the no-cfg guard is the narrow maintainable fix because resolveStorePath(undefined, { agentId }) already defines the default-store path and readSessionStore safely falls back for missing or invalid stores.
