## What This Changes

## Best Possible Solution

Yes. Seed the default sessions.json with an agent:main:explicit:...:acct:default entry and call getSubagentDepthFromSessionStore(key) without cfg; source inspection shows current main returns before reading that store and falls back to depth 0.

## Solution Assessment

Yes. Removing the no-cfg guard is the narrow maintainable fix because resolveStorePath(undefined, { agentId }) already defines the default-store path and readSessionStore already tolerates missing or invalid stores.
