## What This Changes

The PR lets subagent depth resolution consult the default on-disk session store when cfg is omitted and adds regression tests for explicit requester depth plus announce classification.

## Best Possible Solution

Land the helper fix after normal maintainer review, keeping the change scoped to session-depth lookup rather than changing delivery policy.

## Reproduction Assessment

Yes. Persist an agent:main:explicit:...:acct:default entry with spawnDepth in the default sessions.json and call getSubagentDepthFromSessionStore(key) without cfg; current main falls back to segment counting and returns 0.

## Solution Assessment

Yes. Removing the cfg early return is the narrowest maintainable fix because resolveStorePath() already owns the default sessions.json fallback and the tests cover the missing-store fallback.
