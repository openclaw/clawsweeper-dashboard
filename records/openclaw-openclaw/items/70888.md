## Summary

Keep open for normal PR review and merge handling: current main and the latest release still skip the default session-store lookup on no-config depth resolution, while this PR applies a narrow fix with focused regression coverage and sufficient terminal proof.

## What This Changes

This PR removes the no-config early return from subagent depth resolution, lets agent-scoped keys use the default session store, adds depth and announce classification regression tests, and updates the changelog.

## Best Possible Solution

Land the narrow helper change with the regression tests after normal maintainer checks, keeping routing and delivery policy unchanged.

## Reproduction Assessment

Yes. On current main, a persisted agent-scoped explicit requester key reaches a no-cfg depth lookup, resolveEntryForSessionKey returns before reading the default store, and the fallback segment count for an explicit key is 0.

## Solution Assessment

Yes. Removing the no-cfg guard is the narrow maintainable fix because resolveStorePath(undefined, { agentId }) already defines the default-store path and readSessionStore safely falls back for missing or invalid files.
