## Summary

Keep open: the branch is not mergeable as-is because it validates a security-sensitive ACPX policy before the owning acpx runtime supports it, misses dangerous-flag metadata, and lacks real no-TTY behavior proof.

## What This Changes

This PR adds `allow` to ACPX `nonInteractivePermissions` validation, manifest schema, parser tests, and ACP agent docs.

## Best Possible Solution

If maintainers want this policy, implement and release it in `openclaw/acpx` first, then bump OpenClaw's pin, add dangerous-flag metadata, update the setup docs, and prove a no-TTY write/exec run.

## Reproduction Assessment

Yes for the source-level config gap: current main rejects `nonInteractivePermissions=allow`. No for the promised runtime behavior: the pinned and current upstream `acpx` runtime still contracts `deny|fail`, and no after-fix no-TTY write/exec run has been shown.

## Solution Assessment

No. Validating and documenting `allow` only in the OpenClaw wrapper is not the best fix because the permission semantics live in `acpx`, and this auto-approval mode also needs dangerous-config handling.

  - confidence: 0.84
