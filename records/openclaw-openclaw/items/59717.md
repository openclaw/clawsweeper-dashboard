## Summary

Keep open, but this PR should not merge as-is. It makes OpenClaw advertise a trusted auto-approval policy value without ACPX runtime support, dangerous-config metadata, setup-doc coverage, or after-fix no-TTY behavior proof; the linked issue was also closed because `permissionMode=approve-all` already covers the documented full-approval path.

## What This Changes

The PR adds `allow` to ACPX `nonInteractivePermissions` validation and manifest schema, adds parser tests, and documents the option in ACP agent docs.

## Best Possible Solution

If maintainers still want a separate ACPX non-interactive auto-approval policy, land it in `openclaw/acpx` first, consume a released pin, declare the dangerous flag, update setup docs, and require redacted real behavior proof.

## Reproduction Assessment

Yes for the source-level config gap: current main rejects `nonInteractivePermissions=allow` through schema validation. No for the promised runtime behavior: the pinned and current upstream `acpx` runtime still contracts `deny|fail`, and no after-fix no-TTY write/exec run is shown.

## Solution Assessment

No. Relaxing OpenClaw wrapper validation and docs is not the right boundary until ACPX itself supports the semantics, the value is treated as dangerous config, and the current setup docs plus real runtime proof are updated.
