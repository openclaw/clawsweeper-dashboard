## Summary

This PR should close rather than merge because current main already supports headless ACPX auto-approval through `permissionMode=approve-all`, while this branch adds an unsupported `allow` wrapper value and misses required security metadata.

## What This Changes

The PR adds `allow` to ACPX `nonInteractivePermissions` validation, manifest schema, docs, and parser tests.

## Best Possible Solution

Keep the shipped `permissionMode=approve-all` path; any separate ACPX-only auto-approval mode should start in `openclaw/acpx`, be consumed from a released pin, and be declared as dangerous config.

## Reproduction Assessment

Yes for the source-level gap: current main rejects `nonInteractivePermissions=allow` through schema validation. For the promised runtime behavior, no high-confidence real run is shown, and current supported auto-approval is already `permissionMode=approve-all`.

## Solution Assessment

No. Relaxing OpenClaw wrapper validation is not the best fix unless ACPX itself supports the semantics and the new bypass is treated as dangerous config.
