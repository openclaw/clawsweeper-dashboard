## Summary

Keep open: current main still has the targeted readiness and Discord reply-safety gaps, but this PR is not merge-ready because the new sanitizer would drop Discord payloads that contain only presentation, interactive, or channelData content, and the user-facing fix is missing normal changelog coverage.

## What This Changes

The PR gates `sessions.list` during Gateway startup, retries approval/Discord READY readiness failures, and adds Discord send-boundary text scrubbing with tests and status notes.

## Best Possible Solution

Preserve the startup resiliency work, make the final Discord scrub use the shared outbound content predicate so non-text payloads survive, and land a normal changelog entry.

## Reproduction Assessment

Yes by source inspection: current main has the one-restart Discord READY failure path and unfiltered Discord reply delivery, and the PR's new sanitizer drop condition is visible in the patch. I did not run live Discord or Gateway soak tests in this read-only review.

## Solution Assessment

No: the direction is appropriate, but this patch drops Discord component/channelData-only payloads that existing outbound delivery supports. The safer fix is to scrub text while preserving any payload with media, presentation, interactive, channelData, or audio voice content.

## Review Findings

Overall correctness: patch is incorrect
