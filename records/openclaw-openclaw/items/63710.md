## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Restrict URL preload to explicit image refs:** `src/agents/pi-embedded-runner/run/images.ts:42`
  - body: `HTTP_URL_REGEX_SOURCE` matches any HTTP(S) URL in the prompt, so ordinary links such as documentation pages will trigger a gateway-side fetch before the model call. This broadens the outbound request surface and adds latency even when the response is later discarded as non-image; limit native preloading to clear image refs or another explicit image contract.
  - confidence: 0.93
- **[P1] Forward the configured SSRF policy:** `src/agents/pi-embedded-runner/run/images.ts:474`
  - body: The new `remote-url` load path calls `loadWebMedia` with only `maxBytes`, while existing remote media callers pass the configured SSRF policy. Deployments that rely on `tools.web.fetch.ssrfPolicy` will get a different security boundary for native prompt images than for the existing image tool path.
  - confidence: 0.95
- **[P2] Keep HTTP URL dedupe case-sensitive:** `src/agents/pi-embedded-runner/run/images.ts:292`
  - body: Remote URL dedupe reuses the filesystem normalization helper, which lowercases keys on Windows. HTTP path casing can be significant, so prompts containing both `Image.png` and `image.png` can collapse to one native image and silently lose a valid remote ref.
  - confidence: 0.86
- **[P2] Do not consume skipped bracket URLs:** `src/agents/pi-embedded-runner/run/images.ts:347`
  - body: When a `[media attached: path | url]` entry has a local path, this code marks the URL as seen even though it does not emit a `remote-url` ref. If the same URL appears explicitly later in the prompt, the full-prompt scan suppresses it and can still drop the image.
  - confidence: 0.84
- **[P2] Trim closing parentheses from URL refs:** `src/agents/pi-embedded-runner/run/images.ts:96`
  - body: The URL regex can include a closing `)` from parenthesized or Markdown-style text, and the sanitizer does not remove it. That makes the fetch target `image.png)` instead of the intended image URL, so a common prompt shape can fail to load the remote image.
  - confidence: 0.83
