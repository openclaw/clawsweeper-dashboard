- `P1`: The PR addresses a user-facing mixed-image workflow but currently introduces security-sensitive remote-fetch regressions before merge.

## Summary

This PR targets a valid mixed-source native-vision bug that current main still has, but the patch is not merge-ready because it broadens prompt-triggered remote fetching, skips existing SSRF/sandbox protections, and lacks convincing after-fix real behavior proof.

## What This Changes

The PR changes embedded-runner prompt image detection/loading to treat HTTP(S) URLs as `remote-url` image refs and adds focused tests for mixed local plus remote image inputs.

## Best Possible Solution

Rework this into a narrow native prompt-image fix that preloads only intended image refs, preserves SSRF and sandbox media contracts, covers URL edge cases, and includes redacted WebUI or runtime proof that both images reach the first native vision request.

## Reproduction Assessment

Yes, source-level. Current main filters HTTP(S) prompt image refs out of native image detection, and focused tests still expect mixed local-plus-HTTPS prompts to detect only the local image ref.

## Solution Assessment

No. The PR changes the right ingestion surface, but it is not the best mergeable form because it fetches arbitrary prompt links and skips established SSRF and sandbox protections.

  - body: This branch handles `remote-url` before the sandbox validation path, so sandboxed runs can fetch prompt URLs directly through the gateway even though the current image tool rejects remote URLs in sandboxed mode. Keep the sandbox boundary consistent before merging.

  - body: `addRemoteUrlRef` reuses `normalizeRefForDedupe`, which lowercases keys on Windows for local paths. HTTP URL paths can be case-sensitive, so Windows deployments can collapse two distinct remote image URLs and silently drop one from native vision input.
  - confidence: 0.84

  - body: The trailing punctuation sanitizer does not remove `)`, but the URL regex allows it. Parenthesized or Markdown-style image URLs can be fetched with the closing parenthesis included, causing avoidable load failures.
  - confidence: 0.86
