## Summary

Keep this PR open. It targets a real mixed-image ingestion bug, but it is not merge-ready because after-fix proof is insufficient, the branch is currently conflicting, and the diff introduces prompt-triggered remote fetch paths that skip existing SSRF and sandbox contracts.

## What This Changes

The PR adds HTTP(S) prompt-image URL detection and preloading in the pi embedded runner, plus regression tests, so remote URL images can be merged with inline images for native vision requests.

## Best Possible Solution

Rework native remote-image preloading to constrain intended image refs, preserve the image-tool SSRF and sandbox contracts, fix URL dedupe/normalization, rebase, and land with redacted after-fix WebUI proof.

## Reproduction Assessment

Yes, source-level. Current main filters HTTP(S) prompt image refs out of native image detection, and the focused mixed local-plus-HTTPS test still expects only the local image ref to be detected.

## Solution Assessment

No. The patch changes the right ingestion surface, but it is not the best mergeable form because it fetches arbitrary prompt links and skips established SSRF and sandbox protections.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Restrict prompt URL preloads to intended images:** `src/agents/pi-embedded-runner/run/images.ts:42`
  - body: `HTTP_URL_REGEX_SOURCE` matches every HTTP(S) URL, and the later prompt scan preloads each match before discarding non-images. That makes ordinary links trigger gateway outbound requests, adding latency and privacy exposure outside the stated image-URL scope.
  - confidence: 0.92
- **[P1] Forward the configured SSRF policy:** `src/agents/pi-embedded-runner/run/images.ts:460-462`
  - body: The new `remote-url` branch calls `loadWebMedia` with only `maxBytes`. Current media tooling forwards `tools.web.fetch.ssrfPolicy`, so configured allow/block policy is silently dropped for native remote images.
  - confidence: 0.94
- **[P1] Keep sandboxed turns from remote prompt fetches:** `src/agents/pi-embedded-runner/run/images.ts:458-462`
  - body: This branch runs before sandbox path validation and ignores `options.sandbox`, so sandboxed sessions can fetch prompt URLs through the host gateway while the existing image-tool sandbox path only reads sandbox-validated local media.
  - confidence: 0.9
- **[P2] Use URL-safe dedupe keys for remote refs:** `src/agents/pi-embedded-runner/run/images.ts:290`
  - body: Remote URL dedupe reuses `normalizeRefForDedupe`, which lowercases on Windows for filesystem paths. Case-sensitive URL paths like `/Image.png` and `/image.png` can be collapsed, dropping an intended image.
  - confidence: 0.82
- **[P2] Do not mark skipped bracket URLs as seen:** `src/agents/pi-embedded-runner/run/images.ts:347`
  - body: When an attachment bracket has both a local path and URL, the code records the URL as seen even though it did not emit a remote ref. A later explicit copy of the same URL in the prompt is then suppressed and can be lost if the local path is unavailable.
  - confidence: 0.8
- **[P2] Trim closing delimiters from prompt URLs:** `src/agents/pi-embedded-runner/run/images.ts:96`
  - body: The URL regex can include a closing parenthesis, but the sanitizer only strips punctuation like commas and periods. Markdown or parenthesized image URLs can fetch the wrong path with a trailing `)`.
  - confidence: 0.78
