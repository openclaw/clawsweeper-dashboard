## What This Changes

## Best Possible Solution

Rebase or replace the PR with a narrow native prompt-image fix that only preloads intended image refs, preserves SSRF and sandbox policy, covers URL edge cases, and includes redacted after-fix WebUI or runtime proof.

## Reproduction Assessment

Yes, source-level. Current main still filters HTTP(S) prompt image refs out of native image detection, and the focused mixed local-plus-HTTPS test expects only the local image ref to be detected.

## Solution Assessment

No. The PR changes the right ingestion surface, but it is not the best mergeable form because it fetches arbitrary links and skips established SSRF and sandbox protections.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Restrict prompt URL preloads to image refs:** `src/agents/pi-embedded-runner/run/images.ts:42`
  - body: `HTTP_URL_REGEX_SOURCE` matches every HTTP(S) URL, and the prompt scan preloads each match before discarding non-images. A normal prompt that mentions a docs page or tracking link would now trigger an outbound gateway request outside the stated image-URL scope.
  - confidence: 0.95
- **[P1] Forward the configured SSRF policy:** `src/agents/pi-embedded-runner/run/images.ts:460-462`
  - body: The new `remote-url` branch calls `loadWebMedia` with only `maxBytes`, unlike existing media tools that pass the configured remote-media SSRF policy. Operators using custom allowlists or private-network policy would get different network behavior for prompt-preloaded images.
  - confidence: 0.94
- **[P1] Keep sandboxed runs from fetching remote prompt URLs:** `src/agents/pi-embedded-runner/run/images.ts:458-462`
  - body: This branch handles `remote-url` before the sandbox path validation path, so sandboxed agent runs can fetch HTTP(S) prompt URLs even though the current image tool rejects remote URLs under `sandboxConfig`. That weakens the sandbox media boundary for native prompt images.
  - confidence: 0.9
- **[P2] Keep remote URL dedupe case-sensitive:** `src/agents/pi-embedded-runner/run/images.ts:290`
  - body: `addRemoteUrlRef` reuses `normalizeRefForDedupe`, which lowercases keys on Windows for filesystem paths. URL path casing can be server-significant, so two distinct remote images such as `/Image.png` and `/image.png` can collapse into one native image input on Windows.
  - confidence: 0.82
- **[P2] Do not consume skipped bracket URLs:** `src/agents/pi-embedded-runner/run/images.ts:340-348`
  - body: When a bracketed attachment has both a local path and URL, this branch records the URL in `seen` even though no remote ref is emitted. If the same URL is explicitly repeated later in the prompt, the full-prompt scan suppresses it and can still drop the intended remote image.
  - confidence: 0.8
- **[P2] Trim closing delimiters from detected URLs:** `src/agents/pi-embedded-runner/run/images.ts:96-118`
  - body: The HTTP regex can include a closing parenthesis, but the trimming helper only removes comma/period-style punctuation. Parenthesized or Markdown-style image URLs can therefore fetch `image.png)` instead of `image.png` and fail unnecessarily.
  - confidence: 0.78
