## Summary

Keep open. The PR targets a source-reproducible image-input bug, but the proposed remote-URL preload introduces security and behavior regressions and still lacks after-fix real WebUI proof.

## What This Changes

The PR adds HTTP(S) image URL detection and preloading to native prompt-image assembly, plus regression tests for mixed inline and remote image prompts.

## Best Possible Solution

Rework native prompt-image URL preload to only accept intentional image refs, preserve existing SSRF and sandbox contracts, and prove the mixed-input WebUI flow with focused tests plus redacted after-fix runtime evidence.

## Reproduction Assessment

Yes, source-level. Current main ignores HTTP(S) refs in detectImageReferences, so a mixed inline-image plus HTTPS URL prompt cannot include the URL image in the first native vision payload.

## Solution Assessment

No. The PR changes the right surface, but it needs to preserve existing SSRF and sandbox contracts and avoid broad prompt-triggered fetches before it is mergeable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Avoid fetching every HTTP URL as a prompt image:** `src/agents/pi-embedded-runner/run/images.ts:42`
  - body: HTTP_URL_REGEX_SOURCE matches any HTTP(S) URL, and the new full-prompt scan preloads each match before discarding non-images. A normal prompt mentioning a docs page or tracking link would now trigger an outbound gateway request, adding latency and privacy exposure outside the stated image-URL scope.
  - confidence: 0.95
- **[P1] Forward configured SSRF policy for remote images:** `src/agents/pi-embedded-runner/run/images.ts:460-462`
  - body: The remote-url branch calls loadWebMedia() with only maxBytes. Existing media tool paths resolve and pass tools.web.fetch.ssrfPolicy, so configured allowlists or private-network controls would not be honored for native remote-image preloads.
  - confidence: 0.94
- **[P1] Preserve sandbox remote-image restrictions:** `src/agents/pi-embedded-runner/run/images.ts:458-462`
  - body: This branch runs before sandbox path validation, so a sandboxed run with an HTTP(S) prompt image would fetch the remote URL. The existing image tool rejects remote URLs under sandbox mode, so this changes the sandbox contract for the new native preload path.
  - confidence: 0.9
- **[P2] Do not mark skipped bracket URLs as seen:** `src/agents/pi-embedded-runner/run/images.ts:340-348`
  - body: When a [media attached: path | url] entry has a local path, this branch adds the URL dedupe key even though it emits no remote-url ref. If the same URL is repeated explicitly later, the prompt-wide scan suppresses it and the remote image is never loaded.
  - confidence: 0.86
- **[P2] Use URL-specific dedupe keys for remote refs:** `src/agents/pi-embedded-runner/run/images.ts:290-295`
  - body: addRemoteUrlRef() reuses normalizeRefForDedupe(), which lowercases keys on Windows for filesystem paths. HTTP URL paths can be case-sensitive, so Windows gateways can collapse two distinct remote image URLs into one native image input.
  - confidence: 0.84
- **[P2] Trim closing parentheses from detected HTTP image URLs:** `src/agents/pi-embedded-runner/run/images.ts:96-118`
  - body: The URL regex allows closing parentheses, but the punctuation trimmer does not remove them. Parenthesized or Markdown-style links such as (https://example.com/image.png) are fetched as image.png), causing avoidable native image-load failures.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The patch introduces a prompt-triggered remote fetch path without preserving existing SSRF and sandbox protections.

Concerns:

- **[medium] Prompt text can trigger unintended outbound fetches:** `src/agents/pi-embedded-runner/run/images.ts:42`
  - body: Because the new regex treats every HTTP(S) URL as an image candidate, ordinary links in vision-model prompts cause gateway network requests even when they are not image URLs.
  - confidence: 0.95
- **[medium] Remote image preload omits SSRF policy:** `src/agents/pi-embedded-runner/run/images.ts:460`
  - body: The new remote-url loader calls loadWebMedia() without passing the configured tools.web.fetch.ssrfPolicy, so native image preloads do not honor the same policy used by existing media tool paths.
  - confidence: 0.94
- **[medium] Sandboxed runs can fetch remote prompt URLs:** `src/agents/pi-embedded-runner/run/images.ts:458`
  - body: The remote-url branch executes before sandbox validation and does not reject HTTP(S) refs when options.sandbox is present, unlike the current image tool sandbox behavior.
  - confidence: 0.9
