## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Do not preload every HTTP link in prompt text:** `src/agents/pi-embedded-runner/run/images.ts:42`
  - body: The new HTTP URL regex matches any `http://` or `https://` link, and the later prompt scan loads each match before the first model call. That means ordinary docs or website links in a prompt become outbound gateway fetches even when they are not explicit image inputs, adding latency and privacy exposure; narrow this to clear image intent or another maintainer-approved boundary.
  - confidence: 0.88
- **[P1] Forward the configured SSRF policy for remote preloads:** `src/agents/pi-embedded-runner/run/images.ts:460-462`
  - body: This new `remote-url` branch calls `loadWebMedia` with only `maxBytes`, so it does not preserve the configured remote-media SSRF policy used by existing media tools. Thread the policy from the runner/config into `detectAndLoadPromptImages` and `loadImageFromRef`, or keep remote preloading disabled until that boundary is explicit.
  - confidence: 0.91
