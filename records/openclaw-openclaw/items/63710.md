## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Restrict remote preload to explicit image URLs:** `src/agents/pi-embedded-runner/run/images.ts:42`
  - body: `HTTP_URL_REGEX_SOURCE` matches every HTTP(S) URL, and the new detector then preloads those URLs before checking whether they are images. A normal prompt that mentions a docs link would now trigger an outbound gateway fetch, which is outside the PR's explicit-image-URL scope and adds privacy and latency risk.
  - confidence: 0.91
- **[P1] Pass the configured SSRF policy into URL image loads:** `src/agents/pi-embedded-runner/run/images.ts:458-462`
  - body: The new `remote-url` branch calls `loadWebMedia` with only `maxBytes`. Current media tools pass the shared `tools.web.fetch.ssrfPolicy` for remote image references, so prompt URL images would bypass operator allowlists/private-network policy and behave differently from the existing image tool path.
  - confidence: 0.9
- **[P2] Do not mark skipped bracket URLs as loaded refs:** `src/agents/pi-embedded-runner/run/images.ts:343-348`
  - body: When a `[media attached: path | url]` entry has a local path, this branch adds the URL to `seen` even though it does not emit a `remote-url` ref. If the same URL is repeated explicitly later in the prompt, the full-prompt scan suppresses it and the remote image can still be lost.
  - confidence: 0.86
- **[P2] Use URL-specific dedupe keys for remote refs:** `src/agents/pi-embedded-runner/run/images.ts:290-295`
  - body: `addRemoteUrlRef` reuses the path dedupe normalizer, which lowercases keys on Windows. HTTP URL paths can be case-sensitive, so `https://host/Image.png` and `https://host/image.png` can collapse to one native image and silently drop the other.
  - confidence: 0.82
- **[P2] Trim closing delimiters from HTTP image refs:** `src/agents/pi-embedded-runner/run/images.ts:96`
  - body: The URL regex allows `)` and the trailing punctuation sanitizer does not remove it, so a parenthesized or Markdown-style URL can be fetched as `image.png)` instead of `image.png`. That makes common prompt formatting fail even though the intended image URL is valid.
  - confidence: 0.79
