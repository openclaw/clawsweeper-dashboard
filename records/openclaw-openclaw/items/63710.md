## Summary

Keep open. Current main still ignores HTTP(S) prompt image URLs in native prompt image injection, so this PR targets a real bug paired with open #62514, but the current diff introduces concrete outbound-fetch and SSRF-policy regressions that should block merge until reworked.

## What This Changes

The PR adds HTTP(S) URL detection and preloading to embedded-runner prompt image handling, plus tests for mixed inline-image and remote-image prompts.

## Best Possible Solution

Keep the PR and #62514 open, then rework the embedded-runner image path so only clearly intended remote image refs are preloaded, configured media SSRF policy is threaded through the native path, and tests cover mixed inline plus remote images, arbitrary non-image links, bracket dedupe, URL casing, punctuation, and policy propagation.

## Reproduction Assessment

Yes. Current main has a source-level reproduction: `detectImageReferences()` ignores HTTP(S) URLs, current tests assert that local plus remote URL prompts detect only the local ref, and the runner only sends loaded `imageResult.images` to the first native vision request.

## Solution Assessment

No. The PR changes the right surface, but the current patch is not the best solution because it fetches arbitrary HTTP(S) prompt links and does not honor the existing SSRF-policy contract; a safer fix should narrow URL detection and pass the configured policy through remote image preloading.

  - body: The `remote-url` branch calls `loadWebMedia` with only `maxBytes`, while the existing image and PDF tool paths pass the configured `tools.web.fetch.ssrfPolicy`. Native remote image preloads would ignore configured hostname or private-network controls.
  - confidence: 0.94
- **[P2] Avoid marking skipped bracket URLs as seen:** `src/agents/pi-embedded-runner/run/images.ts:343-347`
  - body: When a `[media attached: path | url]` entry has a local path, this branch adds the URL dedupe key even though it emits no `remote-url` ref. If the same URL appears explicitly later in the prompt, the later scan suppresses it and the remote image is never loaded.
  - confidence: 0.86
- **[P2] Use URL-specific dedupe keys for remote refs:** `src/agents/pi-embedded-runner/run/images.ts:292`
  - body: `addRemoteUrlRef` reuses `normalizeRefForDedupe`, which lowercases keys on Windows for filesystem paths. HTTP URL paths can be case-sensitive, so Windows gateways can collapse two distinct remote image URLs into one native image input.
  - confidence: 0.84

  - body: The URL regex allows `)`, but the punctuation trimmer does not remove it. Parenthesized or Markdown-style links such as `(https://example.com/image.png)` are fetched as `image.png)`, causing avoidable native image-load failures.
  - confidence: 0.82
