## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Forward SSRF policy into remote prompt image loads:** `src/agents/pi-embedded-runner/run/images.ts:460-462`
  - body: This new remote-url branch calls `loadWebMedia` with only `maxBytes`, unlike the sibling image tool path that forwards the configured remote-media SSRF policy. Any operator policy under `tools.web.fetch.ssrfPolicy` is skipped for prompt URL preloads, so this new automatic network path can behave differently from the existing guarded media tools.
  - confidence: 0.91
- **[P1] Limit native preload detection to explicit image URLs:** `src/agents/pi-embedded-runner/run/images.ts:42`
  - body: `HTTP_URL_REGEX_SOURCE` matches every HTTP(S) URL in the prompt, and the new scan preloads those refs before the model request. A normal prompt containing a docs or website link will now trigger an outbound gateway fetch even though non-images are discarded later, which expands latency and privacy/security exposure beyond explicit image inputs.
  - confidence: 0.89
- **[P2] Trim closing delimiters from remote image URLs:** `src/agents/pi-embedded-runner/run/images.ts:96-118`
  - body: The URL regex can include a trailing `)` from parenthesized or Markdown-style text, but the sanitizer only strips commas and sentence punctuation. Common prompts like `(https://example.com/image.png)` will fetch `image.png)` and may fail to load the intended image.
  - confidence: 0.83
- **[P2] Keep remote URL dedupe case-sensitive on Windows:** `src/agents/pi-embedded-runner/run/images.ts:290-295`
  - body: Remote URL refs reuse `normalizeRefForDedupe`, which lowercases on Windows for filesystem paths. HTTP path casing can be significant, so `https://host/Image.png` and `https://host/image.png` may collapse to one native image on Windows.
  - confidence: 0.78
- **[P2] Only dedupe bracket URLs when emitting a remote ref:** `src/agents/pi-embedded-runner/run/images.ts:340-348`
  - body: When a media-attached bracket has both a local path and a URL, this branch marks the URL as seen even though it does not emit a remote-url ref. If the user repeats that URL elsewhere in the prompt, the later full-prompt scan suppresses it and can lose the only loadable remote image.
  - confidence: 0.74
