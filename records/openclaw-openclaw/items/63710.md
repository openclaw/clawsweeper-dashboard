## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Forward the remote media SSRF policy:** `src/agents/pi-embedded-runner/run/images.ts:460-462`
  - body: The new `remote-url` branch calls `loadWebMedia` with only `maxBytes`, while the existing image tool forwards the configured remote-media SSRF policy and read-idle timeout for HTTP(S) image refs. This makes native vision URL preloads follow a different security and availability contract than the supported remote image path.
  - confidence: 0.91
- **[P1] Restrict URL preload to explicit image refs:** `src/agents/pi-embedded-runner/run/images.ts:42`
  - body: `HTTP_URL_REGEX_SOURCE` matches every HTTP(S) URL, and the later loader fetches the URL before discarding non-images. A normal prompt that mentions documentation or a website will now trigger an outbound gateway request and possible delay even though the PR only intends explicit image URLs.
  - confidence: 0.88
- **[P2] Trim closing delimiters before fetching URLs:** `src/agents/pi-embedded-runner/run/images.ts:96-118`
  - body: The sanitizer trims commas and sentence punctuation but not a closing `)`, so Markdown or parenthesized image URLs such as `(https://example.com/a.png)` are accepted as `https://example.com/a.png)` and fetched from the wrong path.
  - confidence: 0.78
- **[P2] Do not mark skipped bracket URLs as seen:** `src/agents/pi-embedded-runner/run/images.ts:340-348`
  - body: When a media-attached bracket has both a local path and a URL, this branch adds the URL to `seen` even though it did not emit a `remote-url` ref. If the same URL is repeated later as an explicit prompt image, the full-prompt URL scan suppresses it and the image can still be dropped.
  - confidence: 0.75
