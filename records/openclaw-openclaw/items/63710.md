## Summary

Keep open. Current main still has the mixed inline-plus-HTTP image gap, but this PR is not ready to merge because the patch introduces prompt-triggered remote fetch risks, misses the existing media SSRF-policy contract, and lacks after-fix real behavior proof.

## What This Changes

The PR adds HTTP(S) prompt image URL detection and preloading to native vision image assembly, with regression tests for mixed inline plus remote image inputs.

## Best Possible Solution

Rework the native prompt-image path to preload only clearly intended remote image refs, preserve media SSRF and sandbox contracts, and add focused regression coverage plus redacted after-fix WebUI proof.

## Reproduction Assessment

Yes, source-level. Current main still drops HTTP(S) refs in `detectImageReferences()`, so an existing inline image plus an HTTPS image URL cannot merge the URL image into the first native vision payload.

## Solution Assessment

No. The PR changes the right surface, but this implementation is not the best fix because it preloads too broadly, bypasses the established media SSRF policy, and changes sandboxed remote-image behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Avoid fetching every HTTP URL as a prompt image:** `src/agents/pi-embedded-runner/run/images.ts:42`
  - body: `HTTP_URL_REGEX_SOURCE` matches any HTTP(S) URL, and the new full-prompt scan preloads each match before discarding non-images. A normal prompt that mentions a docs page or tracking link would now trigger an outbound gateway request, adding latency and privacy exposure outside the stated image-URL scope.
  - confidence: 0.95
- **[P1] Forward the configured SSRF policy for remote images:** `src/agents/pi-embedded-runner/run/images.ts:460-462`
  - body: The `remote-url` branch calls `loadWebMedia()` with only `maxBytes`. Existing media tool paths resolve and pass `tools.web.fetch.ssrfPolicy`, so configured allowlists or private-network controls would not be honored for native remote-image preloads.
  - confidence: 0.94
- **[P1] Keep sandboxed runs from preloading remote URLs:** `src/agents/pi-embedded-runner/run/images.ts:458-462`
  - body: This branch runs before the sandbox path validation block, so a sandboxed run with an HTTP(S) prompt image would fetch the remote URL. The existing image tool rejects remote URLs under sandbox mode, so this changes the sandbox contract for the new native preload path.
  - confidence: 0.9
- **[P2] Do not mark skipped bracket URLs as seen:** `src/agents/pi-embedded-runner/run/images.ts:340-348`
  - body: When a `[media attached: path | url]` entry has a local path, this branch adds the URL dedupe key even though it emits no `remote-url` ref. If the same URL is repeated explicitly later, the prompt-wide scan suppresses it and the remote image is never loaded.
  - confidence: 0.86
- **[P2] Use URL-specific dedupe keys for remote refs:** `src/agents/pi-embedded-runner/run/images.ts:290-295`
  - body: `addRemoteUrlRef()` reuses `normalizeRefForDedupe()`, which lowercases keys on Windows for filesystem paths. HTTP URL paths can be case-sensitive, so Windows gateways can collapse two distinct remote image URLs into one native image input.
  - confidence: 0.84
- **[P2] Trim closing parentheses from detected HTTP image URLs:** `src/agents/pi-embedded-runner/run/images.ts:96-118`
  - body: The URL regex allows `)`, but the punctuation trimmer does not remove it. Parenthesized or Markdown-style links such as `(https://example.com/image.png)` are fetched as `image.png)`, causing avoidable native image-load failures.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The patch introduces a prompt-triggered remote fetch path without preserving existing SSRF and sandbox protections.

Concerns:

- **[medium] Prompt text can trigger unintended outbound fetches:** `src/agents/pi-embedded-runner/run/images.ts:42`
  - body: Because the new regex treats every HTTP(S) URL as an image candidate, ordinary links in vision-model prompts cause gateway network requests even when they are not image URLs.
  - confidence: 0.95

  - body: The remote-url branch executes before sandbox validation and does not reject HTTP(S) refs when `options.sandbox` is present, unlike the current image tool sandbox behavior.
