Full review comments:

- **[P1] Restrict URL preload to explicit image refs:** `src/agents/pi-embedded-runner/run/images.ts:42`
  - body: `HTTP_URL_REGEX_SOURCE` matches every HTTP(S) URL, and the new full-prompt scan then calls `loadWebMedia` before MIME rejection. A vision prompt that mentions an ordinary docs link now causes the gateway to make an outbound request and wait for it, expanding privacy and latency risk outside the stated image-URL fix.
  - confidence: 0.93
- **[P1] Forward the configured SSRF policy:** `src/agents/pi-embedded-runner/run/images.ts:458-462`
  - body: The new `remote-url` branch calls `loadWebMedia` with only `maxBytes`, while the existing remote media tool path forwards the shared `tools.web.fetch.ssrfPolicy`. Thread that policy from the runner config into `detectAndLoadPromptImages` and `loadImageFromRef` so this new prompt-driven fetch path honors configured SSRF restrictions.
  - confidence: 0.92
- **[P2] Do not consume skipped bracket URL refs:** `src/agents/pi-embedded-runner/run/images.ts:340-347`
  - body: When a `[media attached: path | url]` bracket has a local path, this code marks the URL as seen even though it does not emit a `remote-url` ref. If the same URL is explicitly repeated elsewhere in the prompt, the later scan suppresses it and the native request can still miss that image.
  - confidence: 0.86
- **[P2] Keep remote URL dedupe case-sensitive:** `src/agents/pi-embedded-runner/run/images.ts:285-295`
  - body: `addRemoteUrlRef` reuses `normalizeRefForDedupe`, which lowercases keys on Windows for local filesystem paths. HTTP URL paths can be case-sensitive, so `https://host/Image.png` and `https://host/image.png` can collapse to one native image on Windows.
  - confidence: 0.82
- **[P2] Trim closing URL delimiters before fetching:** `src/agents/pi-embedded-runner/run/images.ts:96-110`
  - body: The URL regex can include a closing parenthesis, but the punctuation trimmer does not remove `)`. Parenthesized or Markdown-style image URLs can therefore be fetched as `image.png)` and fail even though the visible URL is valid.
  - confidence: 0.8
