## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Limit prompt URL preloads to clear image URLs:** `src/agents/pi-embedded-runner/run/images.ts:42`
  - body: `HTTP_URL_REGEX_SOURCE` matches every HTTP(S) link, so a normal prompt that mentions a docs page or login URL now triggers a gateway fetch before the model call. Non-image responses are discarded only after the request, which expands outbound traffic and can delay turns; restrict this to explicit image URL shapes or another maintainer-approved image-ref signal.
  - confidence: 0.92
- **[P1] Forward SSRF policy for remote prompt images:** `src/agents/pi-embedded-runner/run/images.ts:460-462`
  - body: The new `remote-url` branch calls `loadWebMedia` with only `maxBytes`, while current remote-media callers can pass the configured SSRF policy. This silently skips operator SSRF exceptions and restrictions on the new prompt-image fetch path; thread the same policy through `detectAndLoadPromptImages` and `loadImageFromRef`.
  - confidence: 0.9
- **[P2] Keep remote URL dedupe case-sensitive:** `src/agents/pi-embedded-runner/run/images.ts:290-295`
  - body: `addRemoteUrlRef` uses the path dedupe normalizer, which lowercases on Windows. URL paths can be case-sensitive, so prompts containing `https://host/Image.png` and `https://host/image.png` can collapse to one image on Windows and silently drop valid image input.
  - confidence: 0.82
- **[P2] Do not suppress repeated URL refs from attachment metadata:** `src/agents/pi-embedded-runner/run/images.ts:340-348`
  - body: When a `[media attached: path | url]` bracket also has a local path, this branch marks the URL as seen without emitting a `remote-url` ref. If the user repeats that same URL later as the actual prompt image, the full-prompt URL scan suppresses it and the remote image is never loaded.
  - confidence: 0.8
- **[P2] Trim closing URL delimiters before fetching:** `src/agents/pi-embedded-runner/run/images.ts:96`
  - body: The URL regex can capture a closing `)` from parenthesized or Markdown-style links, but the trailing punctuation trimmer does not remove it. That makes the runtime fetch `image.png)` instead of `image.png`, causing avoidable load failures for common prompt formatting.
  - confidence: 0.78
