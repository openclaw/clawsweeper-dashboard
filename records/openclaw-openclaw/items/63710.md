## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Restrict URL preload to explicit image refs:** `src/agents/pi-embedded-runner/run/images.ts:42`
  - body: `HTTP_URL_REGEX_SOURCE` is scanned across the whole prompt and every match is sent to `loadImageFromRef`, so a normal link like `https://docs.example/guide` now triggers a gateway fetch before the model turn. Please restrict native preloading to explicit image URLs, or add a cheap image-extension/media contract gate before fetching.
  - confidence: 0.93
- **[P1] Thread SSRF policy into remote image fetches:** `src/agents/pi-embedded-runner/run/images.ts:460-462`
  - body: The new `remote-url` branch calls `loadWebMedia` with only `maxBytes`, while sibling remote media paths pass the configured `tools.web.fetch.ssrfPolicy` and read-idle timeout. This bypasses configured host/private-network policy and can stall on slow non-image responses.
  - confidence: 0.94
- **[P2] Preserve URL casing when deduping remote refs:** `src/agents/pi-embedded-runner/run/images.ts:290`
  - body: `addRemoteUrlRef` reuses the filesystem dedupe helper, which lowercases keys on Windows. HTTP URL paths can be case-sensitive, so `https://host/Image.png` and `https://host/image.png` can collapse and silently drop one image from the native payload.
  - confidence: 0.86
- **[P2] Do not mark skipped bracket URLs as loaded:** `src/agents/pi-embedded-runner/run/images.ts:340-348`
  - body: When a media-attached bracket has both a local path and URL, this branch adds the URL to `seen` even though it does not emit a `remote-url` ref. A user repeating that URL later in the prompt can then have the explicit remote image suppressed.
  - confidence: 0.84
- **[P2] Trim closing delimiters from URL refs:** `src/agents/pi-embedded-runner/run/images.ts:96`
  - body: The URL regex can capture a closing parenthesis, but the trailing-punctuation trim does not remove it. Parenthesized or Markdown-style image URLs can be fetched with `)` appended and fail even though the prompt contains a valid image URL.
  - confidence: 0.82
