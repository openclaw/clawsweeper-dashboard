- **[P1] Restrict URL preload to explicit image refs:** `src/agents/pi-embedded-runner/run/images.ts:42`
  - body: `HTTP_URL_REGEX_SOURCE` matches any HTTP(S) URL, so ordinary prompts that mention a docs page or website now cause the gateway to fetch it before the model call. The stated scope is explicit image URLs, so this should filter to image-extension URLs or another explicit image-ref contract before calling `loadWebMedia`.
  - confidence: 0.91
- **[P1] Forward the configured SSRF policy:** `src/agents/pi-embedded-runner/run/images.ts:460-462`
  - body: The new `remote-url` branch calls `loadWebMedia` with only `maxBytes`, while the existing media fetch contract accepts `ssrfPolicy` and media tools pass `tools.web.fetch.ssrfPolicy`. This prompt-triggered remote fetch path needs the same policy threaded from the caller before merge.
  - confidence: 0.9
- **[P2] Keep remote URL dedupe case-sensitive:** `src/agents/pi-embedded-runner/run/images.ts:290-295`
  - body: `addRemoteUrlRef` reuses the filesystem dedupe helper, which lowercases keys on Windows. HTTP URL paths can be case-sensitive, so two distinct image URLs that differ only by case would collapse into one native image ref on Windows deployments.
  - confidence: 0.84
- **[P2] Do not mark skipped bracket URLs as seen:** `src/agents/pi-embedded-runner/run/images.ts:340-348`
  - body: When a bracketed media entry has a local path and a URL, this branch adds the URL to `seen` even though it does not emit a `remote-url` ref. A later explicit occurrence of the same URL in the prompt is then suppressed, so the URL image can still be dropped if the local path cannot load.
  - confidence: 0.82
- **[P2] Trim closing parens from matched URLs:** `src/agents/pi-embedded-runner/run/images.ts:96-118`
  - body: The URL regex allows a trailing `)`, and the punctuation trimmer does not remove it. Parenthesized or Markdown-style image URLs can therefore fetch `image.png)` instead of `image.png`, causing avoidable remote image load failures.
  - confidence: 0.8
