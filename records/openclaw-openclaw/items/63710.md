## Review Findings

Overall confidence: 0.9

Full review comments:

- **[P1] Restrict prompt URL preloads to image refs:** `src/agents/pi-embedded-runner/run/images.ts:42`
  - body: `HTTP_URL_REGEX_SOURCE` matches every HTTP(S) URL in the prompt, so normal links such as docs or issue URLs will trigger a gateway fetch before being discarded as non-images. Limit native preloads to explicit image refs or another maintainer-approved image-only signal so this bug fix does not widen outbound fetching for ordinary prompt text.
  - confidence: 0.9
- **[P1] Forward the configured SSRF policy:** `src/agents/pi-embedded-runner/run/images.ts:458-462`
  - body: The new `remote-url` branch calls `loadWebMedia` with only `maxBytes`, so configured `tools.web.fetch.ssrfPolicy` is ignored on native prompt-image preloads even though existing media tools pass that policy for remote media. Thread the policy from the runner into `detectAndLoadPromptImages` and `loadImageFromRef`.
  - confidence: 0.92
- **[P1] Keep sandboxed runs from fetching remote prompt URLs:** `src/agents/pi-embedded-runner/run/images.ts:458-462`
  - body: This branch runs before the sandbox path validation and does not check `options.sandbox`, which means a sandboxed session can fetch a remote URL from prompt text. Existing image-tool behavior rejects remote URLs when sandboxed, so preserve that boundary for native preloads too.
  - confidence: 0.89
