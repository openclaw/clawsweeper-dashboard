## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Constrain automatic URL image preloads:** `src/agents/pi-embedded-runner/run/images.ts:42`
  - body: `HTTP_URL_REGEX_SOURCE` matches any HTTP(S) URL, and the new detector feeds those matches into a pre-model `loadWebMedia` call. That means a normal prompt mentioning a docs link or website will trigger an outbound gateway fetch before the model call, adding latency and exposing links that were not explicitly image inputs; gate this path to image-intent refs or leave generic URLs on the existing tool path.
  - confidence: 0.88
- **[P1] Forward the configured SSRF policy:** `src/agents/pi-embedded-runner/run/images.ts:460-462`
  - body: The `remote-url` branch calls `loadWebMedia` with only `maxBytes`, so this new prompt-fetch surface does not receive the configured `cfg.tools.web.fetch.ssrfPolicy` that the existing image tool forwards for URL media. Thread the remote-media SSRF policy through `detectAndLoadPromptImages` and `loadImageFromRef` before adding native prompt URL fetches.
  - confidence: 0.9
- **[P2] Preserve remote URL case when deduping:** `src/agents/pi-embedded-runner/run/images.ts:290-295`
  - body: `addRemoteUrlRef` reuses the path dedupe normalizer, which lowercases keys on Windows. HTTP URL paths can be case-sensitive, so prompts containing both `https://host/Image.png` and `https://host/image.png` can collapse to one native image on Windows; use a URL-specific dedupe key that does not lowercase the path.
  - confidence: 0.8
- **[P2] Trim closing delimiters from HTTP image refs:** `src/agents/pi-embedded-runner/run/images.ts:96-118`
  - body: The URL regex allows `)` and the trailing punctuation sanitizer does not remove it, so a common parenthesized URL like `(https://example.com/image.png)` is normalized as a fetch for `/image.png)`. Trim closing delimiters that are not part of the URL before loading remote prompt images.
  - confidence: 0.78
