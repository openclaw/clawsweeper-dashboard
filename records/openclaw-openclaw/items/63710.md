## Summary

Keep open. The PR targets a real current-main image-input bug that is still paired with open #62514, but the proposed patch is not merge-ready because it expands prompt-triggered outbound fetching and does not thread the existing media SSRF-policy contract through the new native remote-image path.

## What This Changes

The PR adds HTTP(S) image URL detection and preloading to native prompt image handling, with regression tests for mixed inline/local plus remote image prompts.

## Best Possible Solution

Rework the native prompt image path to accept clearly intended remote image refs, avoid fetching arbitrary links, thread `tools.web.fetch.ssrfPolicy` from the embedded runner into `loadWebMedia()`, and add focused coverage for mixed inline plus remote images, non-image links that must not fetch, punctuation/dedupe edge cases, and SSRF-policy propagation.

## Reproduction Assessment

Yes. Current main has a source-level reproduction: a prompt containing a local/inline image plus `https://mysite.com/photo2.jpg` produces only the local detected ref, so the remote URL cannot be merged into the first native vision image payload.

## Solution Assessment

No. The PR changes the right surface, but the current implementation is not the best fix because it treats every HTTP(S) URL as a preload candidate and omits the established SSRF-policy parameter; a safer fix should narrow detection and preserve the existing media-fetch policy contract.

Full review comments:

- **[P1] Avoid fetching every HTTP URL as a prompt image:** `src/agents/pi-embedded-runner/run/images.ts:40`
  - body: `HTTP_URL_REGEX_SOURCE` matches any HTTP(S) URL, and the new preload path fetches each detected ref before discarding non-images. A normal prompt that mentions a docs page or tracking link would now trigger an outbound gateway request, adding latency and privacy exposure outside the stated image-URL scope.
  - confidence: 0.95
- **[P1] Forward the configured SSRF policy for remote images:** `src/agents/pi-embedded-runner/run/images.ts:458-461`
  - body: The `remote-url` branch calls `loadWebMedia` with only `maxBytes`, unlike existing media tool paths that pass `remoteMediaSsrfPolicy`. This means configured `tools.web.fetch.ssrfPolicy` allowlists or private-network controls are not honored for native remote image preloads.
  - confidence: 0.94
- **[P2] Do not mark skipped bracket URLs as seen:** `src/agents/pi-embedded-runner/run/images.ts:344-350`
  - body: When a `[media attached: path | url]` entry has a local path, this branch adds the URL dedupe key even though it does not emit a `remote-url` ref. If the same URL is repeated explicitly elsewhere in the prompt, the later scan suppresses it and the remote image is never loaded.
  - confidence: 0.86
- **[P2] Use URL-specific dedupe keys for remote refs:** `src/agents/pi-embedded-runner/run/images.ts:292-296`
  - body: `addRemoteUrlRef` reuses `normalizeRefForDedupe`, which lowercases keys on Windows for filesystem paths. HTTP URL paths can be case-sensitive, so Windows gateways can collapse two distinct remote image URLs into one native image input.
  - confidence: 0.84
- **[P2] Trim closing parentheses from detected image URLs:** `src/agents/pi-embedded-runner/run/images.ts:93-106`
  - body: The URL regex allows `)` and the punctuation trimmer does not remove it, so parenthesized or Markdown-style links such as `(https://example.com/image.png)` are fetched as `image.png)`. That causes avoidable native image-load failures for common prompt formatting.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The PR introduces a new prompt-triggered remote fetch path that broadens outbound requests and bypasses the existing configurable media SSRF policy.

Concerns:

- **[medium] Prompt text can trigger unintended outbound fetches:** `src/agents/pi-embedded-runner/run/images.ts:40`
  - body: Because the new regex treats every HTTP(S) URL as an image candidate, ordinary links in vision-model prompts cause gateway network requests even when they are not image URLs.
  - confidence: 0.95
- **[medium] Remote image preload omits SSRF policy:** `src/agents/pi-embedded-runner/run/images.ts:458`
  - body: The new `remote-url` loader calls `loadWebMedia` without passing the configured `tools.web.fetch.ssrfPolicy`, so native image preloads do not honor the same policy used by existing media tool paths.
  - confidence: 0.94
