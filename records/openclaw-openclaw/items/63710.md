  - body: `HTTP_URL_REGEX_SOURCE` matches every HTTP(S) URL, and the prompt scan preloads each match before discarding non-images. A normal vision prompt that mentions a docs page or tracking link would now trigger an outbound gateway request outside the stated image-URL scope.

  - confidence: 0.96
- **[P1] Keep sandboxed runs from fetching remote prompt URLs:** `src/agents/pi-embedded-runner/run/images.ts:458-462`
  - body: The `remote-url` branch returns before the existing sandbox path validation branch and does not check `options.sandbox`. In sandboxed sessions, prompt text can now cause gateway-side remote fetches instead of following the current sandbox media contract.
  - confidence: 0.9
- **[P2] Trim closing URL delimiters before fetching:** `src/agents/pi-embedded-runner/run/images.ts:96-118`
  - body: The HTTP URL regex can capture a closing `)` from parenthesized or Markdown-style text, and the trailing punctuation sanitizer does not remove it. That fetches `image.png)` rather than `image.png`, causing avoidable remote image load failures.
  - confidence: 0.86
