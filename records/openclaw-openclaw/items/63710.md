## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Forward the SSRF policy for remote prompt images:** `src/agents/pi-embedded-runner/run/images.ts:460-462`
  - body: This new `remote-url` branch loads user-supplied HTTP(S) refs with `loadWebMedia` but only passes `maxBytes`. The sibling image tool forwards `tools.web.fetch.ssrfPolicy` into the same media loader, so native prompt-image URL loading would bypass configured media-fetch policy and behave differently from the existing fallback path.
  - confidence: 0.88
- **[P1] Avoid fetching every hyperlink in vision prompts:** `src/agents/pi-embedded-runner/run/images.ts:42`
  - body: `HTTP_URL_REGEX_SOURCE` matches any HTTP(S) URL, and the detector scans the entire prompt before `loadImageFromRef` fetches each `remote-url`. A normal prompt that mentions a docs page or unrelated link would now cause a gateway network request before the model call, even though non-images are discarded only after the fetch.
  - confidence: 0.86
- **[P2] Trim closing delimiters from remote image URLs:** `src/agents/pi-embedded-runner/run/images.ts:96-102`
  - body: The URL regex allows `)`, and the trailing-punctuation sanitizer does not remove it. Parenthesized or Markdown-style image URLs such as `(https://example.com/photo.png)` will be normalized as a URL ending in `photo.png)`, causing the native preload to fetch the wrong path.
  - confidence: 0.82
- **[P2] Keep remote URL dedupe case-sensitive on Windows:** `src/agents/pi-embedded-runner/run/images.ts:290-294`
  - body: `addRemoteUrlRef` uses `normalizeRefForDedupe`, which lowercases keys on Windows for local filesystem paths. HTTP path casing can be significant, so `https://host/Image.png` and `https://host/image.png` can be collapsed and one image silently dropped from native vision input on Windows deployments.
  - confidence: 0.78
