## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Restrict URL preload to explicit image refs:** `src/agents/pi-embedded-runner/run/images.ts:42`
  - body: `HTTP_URL_REGEX_SOURCE` matches every HTTP(S) URL, and the new prompt scan preloads every match before the model call. A prompt that merely mentions a normal link will now trigger a gateway fetch and only discard the response after download, so please limit detection to explicit image URLs or another maintainer-approved scope.
  - confidence: 0.93
- **[P1] Forward the configured SSRF policy:** `src/agents/pi-embedded-runner/run/images.ts:460-462`
  - body: The new `remote-url` branch calls `loadWebMedia` with only `maxBytes`, while existing remote media callers pass the configured SSRF policy through this API. This means prompt URL images would ignore operator policy/allowlist settings, so the policy needs to be threaded from the caller and covered by a regression test.
  - confidence: 0.91
- **[P2] Do not suppress repeated bracket URLs when skipped:** `src/agents/pi-embedded-runner/run/images.ts:340-348`
  - body: When a media attachment bracket has both a local path and a URL, this branch records the URL as seen even though it does not emit a `remote-url` ref. If the same URL is explicitly repeated elsewhere in the prompt, the later full-prompt scan skips it, so the remote image can still be lost when the local path is unavailable.
  - confidence: 0.82
- **[P3] Trim closing parens from matched image URLs:** `src/agents/pi-embedded-runner/run/images.ts:96`
  - body: The URL regex can include a trailing `)`, but the punctuation trimmer does not remove it, so parenthesized or Markdown-style image URLs are fetched with the wrong path. Please trim balanced closing delimiters before accepting the normalized URL.
  - confidence: 0.78
