Full review comments:

- **[P1] Forward the remote-media SSRF policy:** `src/agents/pi-embedded-runner/run/images.ts:460-462`
  - body: The new `remote-url` branch calls `loadWebMedia` with only `maxBytes`. Existing remote media tool paths pass the configured SSRF policy into `loadWebMedia`, so this prompt-preload path would skip operator allow/block rules on outbound media fetches.
  - confidence: 0.93
- **[P1] Restrict prompt URL preloads to image refs:** `src/agents/pi-embedded-runner/run/images.ts:42`
  - body: `HTTP_URL_REGEX_SOURCE` matches every HTTP(S) URL, and the full-prompt scan fetches each match before discarding non-images. Ordinary links in prompts would now trigger gateway network requests and latency even when they are not intended as image inputs.
  - confidence: 0.91
- **[P2] Do not suppress repeated URLs from attachment brackets:** `src/agents/pi-embedded-runner/run/images.ts:343-348`
  - body: When a bracket contains both a local path and URL, this `else` marks the URL as seen even though no `remote-url` ref was emitted. If the user repeats that URL later and the local path fails to load, the explicit URL is suppressed and the image can still be lost.
  - confidence: 0.86
- **[P2] Keep remote URL dedupe case-sensitive:** `src/agents/pi-embedded-runner/run/images.ts:291-293`
  - body: Remote URL refs reuse `normalizeRefForDedupe`, which lowercases keys on Windows for filesystem paths. HTTP path casing can be significant, so two distinct image URLs that differ only by case collapse into one native image on Windows.
  - confidence: 0.82
