Full review comments:

- **[P1] Restrict URL preload to explicit image refs:** `src/agents/pi-embedded-runner/run/images.ts:42`
  - body: `HTTP_URL_REGEX_SOURCE` matches any HTTP(S) URL, so ordinary prompt links now trigger gateway network fetches before the model call. Please limit eager preload to explicit image URLs or another maintainer-approved image-ref contract.
  - confidence: 0.93
- **[P1] Forward SSRF policy into remote image loads:** `src/agents/pi-embedded-runner/run/images.ts:458-474`
  - body: The new `remote-url` branch calls `loadWebMedia` with only `maxBytes`, so it skips the configured remote-media SSRF policy that existing media tools pass through. Thread the policy into this path before allowing prompt-driven remote fetches.
  - confidence: 0.9
- **[P2] Redact remote URLs before debug logging:** `src/agents/pi-embedded-runner/run/images.ts:463-471`
  - body: The new debug logs include `ref.resolved`, which can contain signed image URLs or query tokens from user prompts. Use the existing redaction pattern before logging remote fetch failures or non-image responses.
  - confidence: 0.84
- **[P2] Keep HTTP URL dedupe case-sensitive:** `src/agents/pi-embedded-runner/run/images.ts:292`
  - body: `addRemoteUrlRef` reuses the filesystem dedupe helper, which lowercases on Windows. HTTP paths can be case-sensitive, so two distinct image URLs can collapse into one native image input on Windows deployments.
  - confidence: 0.8
- **[P2] Do not suppress repeated bracket URLs:** `src/agents/pi-embedded-runner/run/images.ts:347`
  - body: When a bracketed attachment has both a local path and URL, this branch marks the URL as seen even though no remote ref is emitted. A user-authored repeat of the same URL later in the prompt is then skipped, which can still lose the image if the local path cannot load.
  - confidence: 0.78
- **[P3] Trim closing parens from detected HTTP URLs:** `src/agents/pi-embedded-runner/run/images.ts:96`
  - body: The HTTP regex can capture a closing `)` from Markdown or parenthesized text, but the sanitizer only trims commas and sentence punctuation. That makes the preload fetch the wrong URL for common `(https://.../image.png)` prompts.
  - confidence: 0.76
