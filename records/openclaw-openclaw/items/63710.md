- **[P1] Restrict native URL preloads to image refs:** `src/agents/pi-embedded-runner/run/images.ts:42`
  - body: `HTTP_URL_REGEX_SOURCE` matches every HTTP(S) URL, and the later scan feeds each match into `loadWebMedia` before the model call. That means an ordinary prompt link can trigger an outbound gateway fetch even when it is not an image, so this needs an explicit image-ref guard or maintainer-approved syntax before merge.
  - confidence: 0.88
- **[P1] Forward SSRF policy for remote image loads:** `src/agents/pi-embedded-runner/run/images.ts:460-462`
  - body: The new `remote-url` branch calls `loadWebMedia(ref.resolved, { maxBytes })` without any `ssrfPolicy`. Since `loadWebMedia` forwards that option to the guarded remote fetch and sibling media tools pass the configured web-fetch policy, this path would bypass operator SSRF policy for prompt-triggered remote image loads.
  - confidence: 0.86
