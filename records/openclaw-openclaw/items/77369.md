## Summary

Keep open: the reported Telegram progress-draft restart leak is source-reproducible on current main because draft message IDs are process-local and cleanup only knows the current dispatch's active or archived previews.

## What This Changes

Request to clear Telegram progress-draft messages, such as the default "Sifting..." label, after an OpenClaw process restart.

## Best Possible Solution

Telegram should track unfinished progress draft message IDs across process lifetime and clear only stale unfinished drafts on startup or next dispatch, while preserving finalized replies.

## Reproduction Assessment

Yes, there is a high-confidence source-level reproduction path: send a Telegram progress draft, restart before final cleanup, and the new process has no streamMessageId to delete. I did not run a live Telegram bot restart in this read-only review.

## Solution Assessment

A persisted pending-ID cleanup path is the best narrow direction; trying to fetch arbitrary stale Telegram messages without stored IDs is not supported by the current code paths and would be less reliable.
