## Summary

Keep open: the keepalive part is supported, but the current PR is not merge-ready because the polling-start status change can preserve an old healthy state across a fresh reconnect that never completes getUpdates.

## What This Changes

The PR enables TCP keepalive for Telegram undici connections, stops polling-start patches from writing disconnected state, widens shared channel connect grace to 300s, and updates Telegram polling tests.

## Best Possible Solution

Keep the undici keepalive change, but repair Telegram polling lifecycle state so a fresh start without getUpdates becomes unhealthy after the grace window and status diagnostics share the same grace semantics.

## Reproduction Assessment

Yes for the PR-introduced regression: source inspection shows runtime patch merging can preserve connected:true while the PR clears transport timestamps. The original NAT-expiry stall was not independently reproduced in this read-only review.

## Solution Assessment

No as written. The keepalive use matches undici’s supported connector options, but the polling status lifecycle needs an explicit connecting or unhealthy state for starts that never reach getUpdates.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve startup liveness after reconnects:** `extensions/telegram/src/polling-status.ts:18-23`
  - body: `setRuntime` patch-merges status, and channel startup refreshes `lastStartAt` without clearing an old `connected:true`. With this patch, `notePollingStart` clears transport timestamps but omits `connected`, so after the 300s grace a reconnect that hangs before first `getUpdates` can skip both disconnected and stale-socket checks and remain healthy. Publish an explicit connecting/disconnected signal or add a stale-startup health branch with coverage.
  - confidence: 0.91
- **[P2] Align Telegram status grace with health grace:** `src/gateway/channel-health-policy.ts:55`
  - body: The shared health grace moves to 300s, but Telegram status issue collection still suppresses disconnected polling issues for only 120s. During the newly allowed startup window, `openclaw channels status` can still tell users the polling account is unhealthy while the health monitor intentionally waits. Share the grace constant or update the Telegram diagnostic coverage.
  - confidence: 0.84
