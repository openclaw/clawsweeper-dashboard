## What This Changes

The issue requests that `nodes invoke screen.snapshot` preserve large base64 screenshot payloads so screenshots can be saved or delivered downstream.

## Best Possible Solution

Resolve https://github.com/openclaw/openclaw/pull/94281 first: land a compatible, proven raw-invoke fix or close it as superseded by the shipped `screen_snapshot` file-output path, then close this issue with that rationale.

## Reproduction Assessment

## Solution Assessment

No final solution decision yet. The dedicated `screen_snapshot` action is the safer shipped default path, but changing the generic raw invoke output contract needs an explicit maintainer compatibility decision through the linked PR.
