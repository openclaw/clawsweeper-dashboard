## What This Changes

The issue reports that `nodes invoke screen.snapshot` can return inline base64 screenshot data that is truncated before it can be saved or sent downstream.

## Best Possible Solution

Resolve https://github.com/openclaw/openclaw/pull/94281 first: either land a compatible, proven raw-invoke fix or close it as superseded by the shipped `screen_snapshot` file-output path, then close this issue with that rationale.

## Reproduction Assessment

## Solution Assessment

No final solution decision yet. The dedicated `screen_snapshot` action is the safer shipped default path, but changing generic raw invoke output needs an explicit compatibility decision through the linked PR.
