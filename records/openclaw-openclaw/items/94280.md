## Summary

Keep open: current main and v2026.6.10 provide the safer file-backed `screen_snapshot` action and block raw media invoke by default, but the same-author closing PR for explicitly enabled raw invoke behavior is still open and needs a maintainer compatibility decision.

## What This Changes

The issue reports that raw `nodes invoke screen.snapshot` returns inline base64 that is truncated before the screenshot can be saved, written, or sent.

## Best Possible Solution

Resolve https://github.com/openclaw/openclaw/pull/94281 first: either land a compatible, proven raw-invoke fix or close it as superseded by the shipped `screen_snapshot` file-output path, then close this issue with that rationale.

## Reproduction Assessment

Yes at source level: explicitly enabled raw media invoke still returns raw JSON under the 16,000-character default cap, while the issue reports a larger inline screenshot payload. I did not run a live Windows node in this read-only review.

## Solution Assessment

No final solution decision yet. The dedicated `screen_snapshot` action is the safer shipped default path, but changing the generic raw invoke output contract needs explicit maintainer compatibility judgment through the linked PR.
