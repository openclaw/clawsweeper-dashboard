## Summary

Keep open. Current main and the latest release still expose a whole-install backup create/verify model, while the open generic restore work does not cover agent-scoped archives, per-agent config extraction, or `--as` clone rewrites.

## What This Changes

Add `--agent` and `--as` support for agent-scoped backup, restore, and clone operations.

## Best Possible Solution

Build this on top of a hardened generic restore foundation with scoped manifests, `--agent` and `--as`, config-entry rewrites, dry-run/conflict handling, credential/session policy, docs, and focused safety tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The current-main absence is high-confidence from CLI registration, backup planner/source, docs, release inspection, and the open generic restore PR scope.

## Solution Assessment

Unclear as an immediate implementation path. The proposed CLI shape is coherent, but the maintainable route should first settle generic restore safety and credential/session clone policy before adding per-agent rewrite behavior.
