## Summary

Keep open. Current main still only supports whole-install backup create/verify, and the per-agent restore/clone request remains distinct from the open generic restore work because it needs agent-scoped archives plus config/path rewrite behavior.

## What This Changes

Add `--agent` support to backup/restore and `--as` support to restore so a single agent can be backed up, restored, or cloned without restoring the full instance.

## Best Possible Solution

Land a safe generic restore foundation first, then add per-agent backup/restore/clone with scoped manifests, `--agent`/`--as`, config-entry rewrites, conflict handling, docs, and targeted safety tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The current-main absence is high-confidence from CLI registration, backup planner/manifest source, docs, and registration tests showing create/verify only.

## Solution Assessment

Unclear as an immediate implementation path. The proposed CLI shape is coherent, but the maintainable path should build on the generic restore safety model and explicitly define credential/session clone policy first.
