## Summary

Keep open: current main and latest release still provide whole-install backup create/verify only, while per-agent `--agent` backup, restore, and restore-as clone behavior remains missing. The broader backup umbrella and open encrypted snapshot PR overlap, but neither is a safe canonical replacement for the per-agent config/state/session/security contract.

## What This Changes

The issue requests per-agent backup, restore, and clone support through `openclaw backup create --agent`, restore with `--agent`, and restore-as clone behavior.

## Best Possible Solution

Keep this open as a maintainer-reviewed per-agent backup/restore design slice under the broader native backup lifecycle, with archive contents, credential/session portability, restore safety, and clone identity rules settled before implementation.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Current source, docs, latest release, and targeted command searches give a high-confidence feature-gap check that the requested `--agent`/`--as` restore/clone surface does not exist.

## Solution Assessment

Unclear as an implementation shape until maintainers decide restore and clone policy. The safest direction is to extend the backup manifest/state model deliberately rather than scripting raw directory copies around config, credentials, and session state.
