## Summary

Keep open: current main and v2026.6.8 still expose whole-install backup creation and verification only, while the requested per-agent `--agent`/`--as` restore and clone workflow is missing. The broader backup umbrella and open encrypted-snapshot PR overlap, and linked community tools are useful workarounds, but none currently settles the first-party per-agent config/state/session/security contract.

## What This Changes

The issue requests `openclaw backup`/restore support for backing up, restoring, and cloning one configured agent with its workspace, agent state, sessions, and config entry.

## Best Possible Solution

Design a maintainer-sponsored core backup/restore extension that reuses the manifest model, defines per-agent archive contents and secret/session portability, and adds guarded restore/clone tests plus docs before implementation.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source, docs, tests, and release inspection give a high-confidence feature-gap check that the requested `--agent`/`--as` restore/clone surface does not exist.

## Solution Assessment

Unclear until maintainers decide the restore/clone policy. The best shape is likely an extension of the current backup manifest and per-agent state model, not a quick file-copy wrapper around internal directories.
