## Summary

Keep open: current main and the latest release still expose whole-install backup creation and archive verification only, while the requested per-agent restore/clone workflow remains missing and needs maintainer product/security design.

## What This Changes

Request to add `--agent` and `--as` support for backing up, restoring, and cloning one agent's workspace, state, sessions, and config entry.

## Best Possible Solution

Design this as a maintainer-sponsored core backup/restore extension that reuses the existing manifest model, explicitly defines secret/session portability, and adds focused create/restore/clone tests plus docs before implementation.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source and docs do give a high-confidence check that the requested `--agent`/`--as` restore/clone surface does not exist on current main.

## Solution Assessment

Unclear until maintainers decide the restore/clone policy. The best shape is likely an extension of the current backup manifest and agent state model, not a quick file-copy wrapper around internal directories.
