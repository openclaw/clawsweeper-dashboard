## Summary

Keep open. Current main and the latest release still expose whole-install backup creation/verification only, while the open generic restore work does not cover agent-scoped archives, per-agent config extraction, or `--as` clone rewrites.

## What This Changes

Request adds agent-scoped backup, restore, and clone operations to the backup/restore CLI via `--agent` and `--as`.

## Best Possible Solution

Land a safe generic restore foundation first, then add per-agent backup/restore/clone with scoped manifests, `--agent`/`--as`, config-entry rewrites, conflict handling, docs, and focused safety tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The current-main absence is high-confidence from CLI registration, backup planner/manifest source, docs, release inspection, and registration tests showing create/verify only.

## Solution Assessment

Unclear as an immediate implementation path. The proposed CLI shape is coherent, but the maintainable path should build on the generic restore safety model and explicitly define credential/session clone policy first.
