## Summary

Keep open: current main and v2026.6.6 still provide whole-install backup creation and verification only, while per-agent restore/clone behavior remains missing and needs product/security design.

## What This Changes

Add per-agent backup, restore, and clone CLI support for one configured agent, including archive scoping and optional restore under a new agent id.

## Best Possible Solution

Define a narrow core per-agent backup/restore/clone contract, including archive contents, secret handling, conflict behavior, path rewrites, dry-run output, verification, and upgrade safety before accepting code.
