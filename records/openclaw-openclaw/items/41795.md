## Summary

Keep open: current main already supports standard Telegram proxy env routing, but it does not implement the PR's new `OPENCLAW_TELEGRAM_PROXY` surface, and the submitted branch targets the removed `src/telegram` layout rather than the current Telegram plugin.

## What This Changes

The PR adds a Telegram-specific proxy env resolver and wires it into polling, send, probe, audit, and tests under the old `src/telegram` tree.

## Best Possible Solution

If maintainers want a Telegram-specific proxy override, add it in `extensions/telegram/src` with explicit precedence, malformed-value behavior, docs, changelog, and coverage for every Bot API path; otherwise keep the existing standard env and managed proxy contract.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a feature PR review. Source inspection shows current main lacks `OPENCLAW_TELEGRAM_PROXY` and the PR's changed paths no longer exist on current main.

## Solution Assessment

No. The related standard proxy bug is already handled on current main, but this PR is not the best current implementation because it targets removed files and adds a new public env surface without maintainer agreement, docs, or current-plugin coverage.
