## What This Changes

The PR moves Gateway update-check scheduling out of the post-attach await path into a deferred starter with cleanup/failure handling and adds regression tests for ordering, shutdown races, and startup failure logging.

## Best Possible Solution

Keep the update checker off the Gateway ready path while preserving the existing scheduler, broadcast payload, and cleanup contract, then land after normal maintainer review accepts the deferred warning behavior.
