## Summary

Keep open: current main and v2026.6.8 still close the gateway WebSocket without unrefing the underlying socket, and the linked closing PR is open rather than merged.

## What This Changes

The issue reports that `openclaw message send --json` delivers a Discord message but the CLI process never exits because gateway sockets remain active.

## Best Possible Solution

Land a gateway-client teardown fix that releases completed CLI send socket handles without truncating JSON output, preferably through https://github.com/openclaw/openclaw/pull/89792 or a narrow replacement with live CLI proof.

## Reproduction Assessment

No high-confidence live reproduction was executed here. The reporter provided concrete Discord/process evidence, and current-main plus `ws` source inspection shows the reported send path reaches gateway-client shutdown while the WebSocket socket remains ref'd unless a fix like the linked PR lands.

## Solution Assessment

Yes for the owner boundary: this should be solved in gateway-client teardown rather than by forcing `process.exit()`, which could truncate `--json` output. The exact implementation still needs the linked PR review and proof to settle the safest shutdown-mode behavior.
