## Summary

Keep open: the force-pushed PR is a focused Telegram progress-draft fix, but the patch can still leave superseded overflow progress previews visible because `clear()` only deletes the active draft while retained superseded previews are skipped. It also still lacks required real Telegram after-fix proof.

## What This Changes

The PR changes Telegram progress-mode final delivery to clear the answer draft instead of rotating to a new draft, and updates dispatch tests to expect `clear()` in progress mode.

## Best Possible Solution

Delete every progress-only preview owned by the turn before normal final delivery, including retained overflow previews, and add live Telegram proof plus focused coverage for the overflow cleanup path.

## Reproduction Assessment

Yes at source level: a Telegram progress-mode draft with enough compact/raw progress lines to exceed the preview limit can roll into a retained superseded preview, while the PR clears only the active message. I did not run live Telegram in this read-only review.

## Solution Assessment

No. Calling `clear()` is the right direction for the active progress draft, but the fix also needs to cover progress-owned superseded previews and include real Telegram before/after proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Delete superseded progress drafts before final reply:** `extensions/telegram/src/bot-message-dispatch.ts:768`
  - body: With `streamMode === "progress"`, this clears only the current preview message. `createTelegramDraftStream` can roll an over-limit draft into a retained superseded message, and dispatch skips retained previews, so a long progress-only draft can still leave stale progress bubbles before the normal final reply. Track and delete those progress-owned superseded previews, or avoid retaining them when the draft is being cleared.
  - confidence: 0.88
