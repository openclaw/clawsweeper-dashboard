## Summary

Keep open. The direction is plausible, but the patch should not merge yet because `retain: false` only clears the active progress draft while overflow/superseded progress drafts can still be retained, and the external PR has tests but no required real Telegram behavior proof.

## What This Changes

The PR adds a `streaming.progress.retain` Telegram/channel config flag, updates schema/UI metadata/docs, and changes progress-mode final delivery to clear the draft when the flag is false.

## Best Possible Solution

Keep the default released behavior, but make `retain: false` delete all progress-only preview messages it owns, including superseded overflow drafts, then attach real Telegram before/after proof and let the config/schema/docs checks gate the public option.

## Reproduction Assessment

Yes for the review finding at source level: overflow previews are emitted with `retain: true`, the dispatch handler skips deleting those retained superseded previews, and the PR's new `clear()` path only clears the active draft. No live Telegram reproduction or after-fix proof was provided.

## Solution Assessment

No. The config direction is reasonable, but the implementation should also track/delete retained superseded progress-only preview messages when retention is disabled, and it needs live Telegram proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Delete superseded progress drafts when retain is false:** `extensions/telegram/src/bot-message-dispatch.ts:774`
  - body: When `streaming.progress.retain` is `false`, this only clears the active draft message. Long progress drafts can roll over through `onSupersededPreview(... retain: true)`, and the existing handler intentionally skips deleting those older preview messages, so users can still be left with stale progress messages despite opting out of retention. Track and delete superseded progress-only previews, or avoid marking them retained when this mode is disabled.
  - confidence: 0.89
