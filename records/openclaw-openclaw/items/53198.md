## Summary

Keep open. Current main still has the reported gap: docs promise the Discord elevated allowFrom fallback, runtime elevated authorization only consults a channel-plugin fallback hook that Discord does not provide, and sandbox explain still reads only explicit tools.elevated.allowFrom values. The open #53259 PR is the active implementation candidate with closing syntax, so this issue should stay open until that PR lands or is superseded.

## What This Changes

Discord elevated exec should authorize senders from channels.discord.allowFrom when tools.elevated.allowFrom.discord is omitted, and sandbox explain should report that same effective authorization path.

## Best Possible Solution

Land or supersede #53259 with a Discord-owned elevated fallback adapter or shared resolver used by both resolveElevatedPermissions and sandbox explain, with regression coverage for omitted fallback, explicit empty override, per-agent restrictions, and diagnostics consistency.
