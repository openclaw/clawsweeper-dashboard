## Summary

Keep open: current main and v2026.6.10 still document but do not implement the Discord elevated allowFrom fallback, and https://github.com/BingqingLyu/openclaw/pull/1288 is open but conflicting and unmerged. The issue remains source-reproducible and security-sensitive because the fix changes elevated exec authorization semantics.

## What This Changes

Discord elevated exec should honor the documented fallback from `channels.discord.allowFrom` when `tools.elevated.allowFrom.discord` is omitted, and `sandbox explain` should report the same authorization path.

## Best Possible Solution

Adopt or replace the linked repair with a narrow Discord plugin-owned fallback shared by runtime elevated auth and `sandbox explain`, with tests for omitted fallback and explicit empty override behavior.

## Reproduction Assessment

Yes. Source inspection shows the docs promise the fallback, current runtime only calls a channel `allowFromFallback` hook, no Discord implementation is registered, and diagnostics read only explicit elevated config.

## Solution Assessment

Yes. The narrow maintainable fix is to add a Discord-owned fallback hook and reuse the same resolution logic in `sandbox explain`, while preserving explicit-empty override behavior.
