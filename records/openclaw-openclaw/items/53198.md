## What This Changes

The issue reports that Discord elevated authorization and sandbox diagnostics ignore the documented fallback from `channels.discord.allowFrom` when `tools.elevated.allowFrom.discord` is omitted.

## Best Possible Solution

Land a narrow security-reviewed fix that wires Discord's documented channel allowFrom fallback through the channel/plugin elevated adapter and makes sandbox explain use the same resolver, with regression tests for omitted versus explicit empty allowlists.

## Reproduction Assessment

Yes, source-reproducible. Current main's runtime resolver asks for a plugin fallback that Discord does not provide, and sandbox explain reads only `tools.elevated.allowFrom.<channel>` while the issue includes concrete config and logs.

## Solution Assessment

Yes, the narrow resolver/adapter fix is the maintainable direction; the open linked fix PR targets that shape, but it needs conflict resolution and security review before the issue can close.
