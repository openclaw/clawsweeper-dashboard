## Summary

Keep open: current main and v2026.6.10 still do not implement the documented Discord elevated allowFrom fallback, while a linked open fork PR remains conflicting and unmerged. The report is source-reproducible and security-sensitive because the fix changes elevated exec authorization semantics.

## What This Changes

Discord elevated authorization and `openclaw sandbox explain` should honor the documented fallback from `channels.discord.allowFrom` when `tools.elevated.allowFrom.discord` is omitted.

## Best Possible Solution

Land a narrow Discord plugin-owned fallback shared by runtime elevated auth and `sandbox explain`, with regression coverage for omitted fallback and explicit empty override behavior.

## Reproduction Assessment

Yes. At source level, current main still documents the Discord fallback but has no Discord `elevated.allowFromFallback` implementation and `sandbox explain` reads only explicit elevated allowFrom config.

## Solution Assessment

Yes. The narrow maintainable fix is to implement the documented Discord fallback through the channel/plugin-owned allowlist path and reuse the same resolution logic in diagnostics.
