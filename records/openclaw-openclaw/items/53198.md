## Summary

Keep open: current main and the latest release still do not implement the documented Discord elevated allowFrom fallback, and the linked candidate PR is open but conflicting and not merged.

## What This Changes

The issue reports that Discord elevated exec does not honor the documented fallback from `channels.discord.allowFrom` when `tools.elevated.allowFrom.discord` is omitted.

## Best Possible Solution

Land a focused Discord-owned fallback implementation shared by runtime elevated authorization and `sandbox explain`, with regression coverage for fallback, explicit empty override, per-agent restriction, and Discord account scope.

## Reproduction Assessment

Yes. Source inspection shows the docs promise a Discord fallback, while current runtime auth only uses an unimplemented Discord plugin fallback hook and `sandbox explain` reads only explicit elevated allowlists.

## Solution Assessment

No landed fix exists yet. The best repair is not another core duplicate resolver; it should keep Discord-specific allowlist resolution in the channel/plugin-owned seam and make diagnostics reuse the same effective allowlist path.
