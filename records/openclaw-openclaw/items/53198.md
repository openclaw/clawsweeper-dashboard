## Summary

Keep open: current main and the latest shipped release still document the Discord elevated fallback, but runtime authorization only consults a channel-plugin fallback hook that Discord does not register, and `sandbox explain` still reads only explicit elevated allowlists. The open closing PR at https://github.com/openclaw/openclaw/pull/53259 remains the right fix path to review, update, or supersede.

## What This Changes

Discord elevated exec should honor the documented `channels.discord.allowFrom` fallback when `tools.elevated.allowFrom.discord` is omitted, and `sandbox explain` should report that path consistently.

## Best Possible Solution

Land or supersede https://github.com/openclaw/openclaw/pull/53259 with a Discord-owned `elevated.allowFromFallback` adapter backed by `resolveDiscordAccountAllowFrom`, shared by runtime auth and `sandbox explain`, with explicit-empty and per-agent deny semantics covered.

## Solution Assessment
