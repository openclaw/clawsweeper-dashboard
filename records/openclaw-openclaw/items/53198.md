## Summary

Keep open: current main still documents the Discord elevated allowFrom fallback, but runtime elevated authorization depends on a channel-plugin fallback hook that Discord does not expose, and sandbox explain still reads only explicit elevated allowlists. The open #53259 PR remains the implementation candidate with closing syntax.

## What This Changes

Make Discord elevated authorization and sandbox explain honor the documented fallback from `channels.discord.allowFrom` when `tools.elevated.allowFrom.discord` is omitted.

## Best Possible Solution

Land or supersede #53259 with a narrow Discord-owned fallback resolver reused by both runtime elevated authorization and sandbox explain diagnostics.

## Reproduction Assessment

Yes. A source-level reproduction is clear on current main: configure only `channels.discord.allowFrom`, omit `tools.elevated.allowFrom.discord`, and the runtime fallback remains undefined because Discord has no `elevated.allowFromFallback`; sandbox explain likewise reads only explicit elevated allowlists.

## Solution Assessment

Yes. A narrow Discord fallback resolver shared by runtime and diagnostics is the most maintainable fix, without changing provider-wide policy or weakening explicit elevated allowlist overrides.
