Keep open: current main still documents the Discord elevated fallback, but runtime authorization only consults a channel-plugin fallback hook that Discord does not register, and `sandbox explain` still reads only explicit elevated allowlists. ## What This Changes

The issue reports that Discord elevated exec no longer honors the documented fallback from `channels.discord.allowFrom` when `tools.elevated.allowFrom.discord` is omitted, and that `sandbox explain` reports the path incorrectly.

## Best Possible Solution

Land or supersede https://github.com/openclaw/openclaw/pull/53259 with a Discord-owned `elevated.allowFromFallback` adapter backed by `resolveDiscordAccountAllowFrom`, shared by runtime auth and `sandbox explain`, with regression coverage for fallback, explicit-empty override, per-agent restrictions, and account-scoped Discord config.
