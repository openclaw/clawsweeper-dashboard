## Summary

Keep open. Current main still lacks configurable Gemini and xAI search base URLs, so the contribution remains relevant, but the proposed patch is not merge-ready because it adds the new x_search override to a legacy config surface, misses a legacy Grok baseUrl fallback, and overlaps the Gemini portion with the open Gemini-only implementation in #57496.

## What This Changes

The PR adds baseUrl resolution, plugin metadata/schema updates, generated config baseline updates, and tests for Gemini web_search, xAI Grok web_search, and xAI x_search.

## Best Possible Solution

Rebase or split the work so Gemini coordinates with #57496, while the xAI changes use plugin-owned config contracts for both webSearch.baseUrl and xSearch.baseUrl, preserve backward-compatible legacy fallbacks, include baseUrl in relevant cache keys, update public docs and generated schema artifacts, and keep the web-tool guarded request path intact.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the missing behavior: Gemini and both xAI search helpers still build requests from hardcoded endpoint constants rather than configurable baseUrl values.

## Solution Assessment

No. The direction is useful, but this patch is not the best merge shape because it exposes x_search.baseUrl on the legacy tools config instead of the canonical xAI plugin config and misses one legacy Grok baseUrl fallback; the Gemini side should also be coordinated with #57496.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Move x_search baseUrl into plugin config:** `src/config/types.tools.ts:520-522`
  - body: `x_search` is now configured under `plugins.entries.xai.config.xSearch.*`, while `tools.web.x_search.*` is legacy and doctor-migrated. Adding the new public override only to `tools.web.x_search.baseUrl` leaves the current manifest/UI/docs path unable to save `xSearch.baseUrl`, so users following current docs still cannot configure a per-tool x_search endpoint.
  - confidence: 0.87
- **[P2] Preserve the legacy Grok baseUrl fallback:** `extensions/xai/x-search.ts:57-68`
  - body: The new x_search resolver falls back from explicit x_search config to `plugins.entries.xai.config.webSearch.baseUrl`, but skips `tools.web.search.grok.baseUrl`. Existing installs that make Grok `web_search` use a proxy through the legacy scoped Grok setting would still send `x_search` to the default xAI endpoint, recreating the inconsistency this patch is meant to fix.
  - confidence: 0.82
