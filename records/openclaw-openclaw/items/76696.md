## Summary

Keep open. The PR adds a new protected Mantis QA feature, and the latest redaction repair covers normal report/summary artifacts but still leaves a Discord guild id exposed in one uploaded failure-artifact path.

## What This Changes

This PR adds a private `qa mantis discord-smoke` CLI/runtime/test, a manual secret-backed GitHub workflow, Mantis docs/glossary, and related changelog/docs updates.

## Best Possible Solution

Keep the Mantis smoke direction, but redact every Discord response identifier used in failure messages before any workflow artifact can be public.

## Reproduction Assessment

Yes for the PR finding by source inspection: a channel/guild mismatch can throw an error containing `params.channel.guild_id`, and the catch path writes that message to an artifact uploaded by the workflow. This is not a current-main user bug reproduction because the PR adds a new QA feature.

## Solution Assessment

No. The overall Mantis implementation is a maintainable direction, but the redaction fix should also cover response metadata before assertions and add a redacted failure-path regression test.
