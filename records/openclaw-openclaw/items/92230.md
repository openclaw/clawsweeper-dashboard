## Summary

Keep open: current main and the latest release do not have the `/model` choice menu yet, but this PR still lets the new shared `/model` menu reach Telegram's generic callback path, where long provider/model refs can exceed Telegram callback limits.

## What This Changes

The PR adds dynamic configured-model choices to the shared `/model` command registry, filters them through allowed models, and adjusts Discord dispatch/tests so bare `/model` still opens the Discord model picker.

## Best Possible Solution

## Solution Assessment

No, not as-is. The shared registry is a good Slack fix location and the Discord preservation is narrow, but Telegram needs a compact or gated callback path before this is the best cross-channel solution.
