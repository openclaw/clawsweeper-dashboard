## Summary

Keep open: current main still lacks the requested Slack-friendly `/model` choices, but this PR is not merge-ready because the shared `/model` args menu also reaches Telegram's generic menu path and can emit overlong raw callbacks instead of the compact model-picker callbacks.

## What This Changes

This PR adds configured provider/model choices and an argument menu to the shared `/model` command registry, filters choices by allowed models, and updates Discord tests/dispatch so bare Discord `/model` still opens the dedicated picker.

## Best Possible Solution

Keep registry-driven Slack `/model` choices and Discord picker preservation, but make Telegram use compact `mdl_*` callbacks or skip the shared `/model` menu before merge.

## Reproduction Assessment

Yes. Source inspection shows PR head adds a shared `/model` args menu, while Telegram resolves shared menus before dispatch and builds raw `tgcmd:/model provider/model` callback data without the existing compact callback guard.

## Solution Assessment

No, not as-is. The shared registry is a good Slack fix location and the Discord preservation is narrow, but Telegram needs a compact or gated callback path before this is the best cross-channel solution.
