## Summary

This PR is still useful because current main and v2026.6.8 do not expose configured `/model` choices, but it should not merge as-is: the shared registry change also activates Telegram's generic arg-menu path, which bypasses the existing compact `mdl_*` callback guard for model selections.

## What This Changes

The PR adds configured provider/model choices to the shared `/model` command metadata and adjusts Discord tests/dispatch so bare Discord `/model` still opens the dedicated picker.

## Best Possible Solution

Keep the Slack registry-driven `/model` choices, but gate Telegram out of the shared `/model` menu or route Telegram selections through existing compact model callback helpers before merge.

## Reproduction Assessment

Yes. Source inspection shows the PR adds a shared `/model` argsMenu, while Telegram resolves shared menus before native command dispatch and serializes choices into raw `tgcmd:/model ...` callback data.

## Solution Assessment

No, not as-is. The shared registry is a good Slack fix location and the Discord preservation is narrow, but Telegram needs a compact or gated callback path before this is the best cross-channel solution.
