## Summary

Current main and the latest release still attach Telegram `chat_window` prompt context into persistent DM turns, and the remaining fix is already owned by an open closing PR, so this issue should stay open until that PR lands or is replaced.

## What This Changes

Suppress Telegram recent chat-window context for existing persistent private DMs so the persistent session transcript remains the source of conversation history.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/89855, or replace it with an equivalent narrow session-aware Telegram filter that keeps fresh DM, group, topic, and reply context behavior intact.

## Reproduction Assessment

Yes. Source inspection shows current main still builds Telegram `chat_window` context from recent cached messages and forwards it into `UntrustedStructuredContext`; I did not run a live Telegram transport repro.

## Solution Assessment

Yes. The best fix is a narrow filter after Telegram route/session resolution so existing persistent DMs can be distinguished from fresh DMs, replies, topics, and groups.
