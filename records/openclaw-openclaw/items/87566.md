## Summary

Keep this open: current main and the latest release still build Telegram `chat_window` context and forward it into persistent DM turns, while https://github.com/openclaw/openclaw/pull/89855 is the open closing PR that owns the narrow fix path.

## What This Changes

Telegram direct messages are reported to inject recent chat-window context into persistent sessions, duplicating transcript history and increasing token usage.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/89855, or replace it with an equivalent narrow session-aware Telegram filter that preserves fresh DM, reply, topic, and group context behavior.

## Reproduction Assessment

Yes, at source level. Current main still creates a Telegram `chat_window` from recent cached messages and forwards it into `UntrustedStructuredContext` for the session context payload; I did not run a live Telegram transport repro.

## Solution Assessment

Yes. The best fix is a narrow filter after Telegram route/session resolution, because that layer can distinguish existing persistent DMs from fresh DMs, replies, topics, and groups.
