## Summary

Keep this open: current main and v2026.6.9 still build and forward Telegram `chat_window` context for persistent DM turns, while the linked open PR is only a fix candidate, not a merged implementation.

## What This Changes

Avoid injecting Telegram recent chat-window context into existing persistent private DM sessions because the session transcript already contains that history.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/89855, or replace it with an equivalent narrow session-aware Telegram filter that preserves fresh DM, reply, topic, and group context behavior.

## Reproduction Assessment

Yes, at source level. Current main builds recent Telegram chat-window context and forwards it into the inbound session payload without persistent-DM filtering; I did not run a live Telegram transport repro.

## Solution Assessment

Yes. The best fix is a narrow filter after Telegram route and session resolution, because that layer can distinguish existing persistent DMs from fresh DMs, replies, DM topics, forum topics, and groups.
