Keep open: current main still builds and forwards Telegram `chat_window` context into persistent DM turns, and https://github.com/openclaw/openclaw/pull/89855 is the open, mergeable candidate that directly closes the remaining work.

## What This Changes

Telegram DMs should stop adding a recent chat-window `Conversation context` block when an existing persistent session transcript already carries that history.

## Best Possible Solution

Review and land the narrow session-aware filter in https://github.com/openclaw/openclaw/pull/89855 while preserving fresh DM, reply, group, and topic context.

## Reproduction Assessment

Yes. Current main source shows Telegram private DMs can resolve to a persistent session while recent `chat_window` prompt context is still built and forwarded into the inbound turn.

## Solution Assessment

Yes. The open PR's route/session-layer filter is the narrowest maintainable fix because it can distinguish existing plain DMs from fresh DMs, replies, groups, and topics.
