## Summary

Keep this PR open. Current main still does not implement chat `/sessions` or `/session <target>` switching, but the proposed branch is not ready to merge because it switches the live session entry with only a partial metadata snapshot and leaves transcript-retention edge cases unresolved.

## What This Changes

The PR adds a `/sessions` chat command, per-session-key LRU history, session switching by index/id/back, `session.historyLimit`, generated config schema updates, and targeted tests.

## Best Possible Solution

Keep the implementation path, but make the switch boundary explicit and complete: either persist a full per-session state snapshot for switchable history or store each switchable session as its own entry and switch a pointer. The final version should restore or deliberately clear every session-scoped runtime/security field, respect disabled history before listing or switching, integrate history transcripts with maintenance cleanup, fire the appropriate lifecycle boundaries, update public slash-command docs, and cover those behaviors with focused tests.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to create two direct-chat sessions under one session key with different session-scoped fields such as `authProfileOverride`, `execHost`/`execSecurity`, `cliSessionIds`, `acp`, `estimatedCostUsd`, or fallback state, then switch with `/sessions 2`; the PR only reapplies the small metadata subset and leaves omitted fields from the outgoing session on the resumed one.

## Solution Assessment

No. The feature direction is plausible, but this patch is not the safest solution because it mutates one live `SessionEntry` and relies on a partial allowlist of restored fields; a complete snapshot/switch contract with maintenance and lifecycle integration would be more maintainable.

Full review comments:

- **[P1] Restore security-sensitive session state on switch:** `src/auto-reply/reply/commands-sessions.ts:277-280`
  - body: This branch rewrites the live `SessionEntry` to the target session and then reapplies only the limited `SessionHistoryItem.metadata` subset. Fields such as `authProfileOverride`, `elevatedLevel`/`exec*`, CLI handles, ACP runtime state, runtime model fields, fallback notices, abort state, and cost totals are neither saved nor cleared, so a resumed session can inherit the outgoing session's credentials, execution policy, or provider-side thread. Persist and restore a complete session snapshot, or explicitly clear every omitted session-scoped field before saving the switch.
  - confidence: 0.91
- **[P2] Gate switching when session history is disabled:** `src/auto-reply/reply/commands-sessions.ts:258-264`
  - body: `historyLimit` is read only after the target has already been resolved from existing `sessionHistory`. If an operator changes `session.historyLimit` to `0` while stale history exists, `/sessions` can still list and switch to prior sessions, then archive the current session as the queue is trimmed to zero. Check the disabled-history case before listing or resolving switch targets.
  - confidence: 0.86
- **[P2] Archive retained history when disabling history:** `src/auto-reply/reply/session.ts:650-651`
  - body: When `historyLimit === 0`, the new session path clears `sessionEntry.sessionHistory` but only archives `previousSessionEntry`; older retained history entries are dropped from state without being archived. Archive each cleared history item before emptying the queue so transcript files do not become orphaned outside the reset/archive lifecycle.
  - confidence: 0.84
- **[P2] Keep history transcripts referenced during disk sweeps:** `src/config/sessions/types.ts:257`
  - body: The new `sessionHistory` entries intentionally keep old transcripts switchable, but the session disk-budget sweep still marks only each top-level entry's active transcript and compaction checkpoints as referenced. With `session.maintenance.maxDiskBytes` enabled, a history transcript can be deleted as unreferenced while `/sessions` still lists it. Include `sessionHistory` artifacts in the referenced-path set and add coverage for enforced cleanup.
  - confidence: 0.82
