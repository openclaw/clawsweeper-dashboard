## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Restore the full session boundary before switching:** `src/auto-reply/reply/commands-sessions.ts:115-142`
  - body: `applyHistoryMetadata` restores only a small subset of `SessionEntry`. After `/sessions <target>`, fields such as auth profile overrides, CLI bindings, ACP runtime state, elevated/exec policy, abort cutoffs, costs, fallback notices, runtime model fields, and usage mode remain from the outgoing session, so the resumed transcript can run under the wrong credentials or execution policy.
  - confidence: 0.91
- **[P1] Protect retained history transcripts from cleanup:** `src/auto-reply/reply/session.ts:643-649`
  - body: This PR keeps old session transcripts switchable by storing them in `sessionHistory`, but the disk-budget sweep still only treats active store entries and compaction checkpoints as referenced. With `session.maintenance.maxDiskBytes`, a retained history transcript can be deleted while `/sessions` still lists it, so switching back can land in an empty or missing transcript.
  - confidence: 0.88
- **[P2] Honor disabled history before resolving targets:** `src/auto-reply/reply/commands-sessions.ts:258-264`
  - body: `historyLimit` is read only after targets are built from persisted `sessionHistory`, so an operator who sets `session.historyLimit` to `0` can still list and switch to stale history entries until a reset happens. The switch then pushes the current session and immediately evicts it, archiving the conversation the user was just in despite history being disabled.
  - confidence: 0.86
- **[P3] Match session ID prefixes before numeric indexes:** `src/auto-reply/reply/commands-sessions.ts:219-221`
  - body: Numeric input is always treated as a list index before exact or prefix session-id matching, so a valid numeric session-id prefix like `/sessions 1234` can never select the matching session and instead jumps to row 1234 or reports not found. That contradicts the advertised `sessionId` prefix behavior.
  - confidence: 0.74
