## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Restore the full session boundary on switch:** `src/auto-reply/reply/commands-sessions.ts:277-280`
  - body: The switch mutates `sessionId`/`sessionFile` and then reapplies only the small `SessionHistoryItemMetadata` subset. Fields such as auth profile source, runtime model/fallback, CLI/ACP handles, exec/elevated policy, pending delivery, and plugin state can remain from the outgoing session, so the resumed transcript may run under the wrong credential or tool boundary. Snapshot/restore or rebuild the full session boundary before persisting.
  - confidence: 0.9
- **[P1] Protect retained session transcripts from cleanup:** `src/auto-reply/reply/commands-sessions.ts:263`
  - body: This pushes the current transcript into `sessionHistory` so it remains switchable, but current disk-budget cleanup only protects each active `SessionEntry` and compaction checkpoints. A cleanup pass can treat retained history transcripts as unreferenced before `/sessions` switches back to them; include `sessionHistory` artifacts in cleanup/ref-counting or use a storage model cleanup already owns.
  - confidence: 0.86
- **[P2] Honor disabled history before listing or switching:** `src/auto-reply/reply/commands-sessions.ts:213`
  - body: The command builds its list and resolves targets from persisted `sessionHistory` before consulting `session.historyLimit`. If an operator changes the limit to `0` while old history exists, stale sessions remain listable and switchable and the current session is immediately evicted during the switch; return the disabled state before listing or switching stale history.
  - confidence: 0.82
- **[P3] Resolve session IDs before numeric shortcuts:** `src/auto-reply/reply/commands-sessions.ts:219-222`
  - body: Numeric target handling runs before exact or prefix id matching, so a valid numeric-only session id or advertised prefix is interpreted as a list index instead of an id. Prefer exact/prefix matching before numeric fallback or make the shortcut syntax unambiguous.
  - confidence: 0.72
