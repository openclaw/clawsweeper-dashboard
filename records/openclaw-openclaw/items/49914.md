## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Restore the full session entry on switches:** `src/config/sessions/types.ts:81-112`
  - body: `SessionHistoryItemMetadata` is the only state reapplied after rewriting the live entry, but it omits auth profile overrides, exec/elevated policy, fast/usage flags, runtime model/provider, fallback/cost/status, CLI handles, system prompt report, and ACP state. Switching can therefore resume one transcript while continuing under another conversation's credentials, provider thread, or execution policy.
  - confidence: 0.92
- **[P2] Honor the advertised /session switch command:** `src/auto-reply/reply/commands-sessions.ts:18`
  - body: The PR body advertises `/session <number>`, `/session <sessionId>`, and `/session back`, but this handler only matches `/sessions`. Users following the documented singular command will still hit the existing lifecycle handler and get usage text instead of switching.
  - confidence: 0.9
- **[P2] Check disabled history before resolving targets:** `src/auto-reply/reply/commands-sessions.ts:213-219`
  - body: Targets are built from persisted `sessionHistory` before `session.historyLimit` is checked. If an operator changes the limit to `0`, stale sessions can still be listed or selected, and selecting one archives the current session under the disabled-history path.
  - confidence: 0.88
- **[P2] Match ID prefixes before numeric indexes:** `src/auto-reply/reply/commands-sessions.ts:219-221`
  - body: Numeric input is interpreted as a rendered list index before exact or prefix session-id matching, so a session id with a numeric-only unique prefix cannot be selected by the documented prefix and may switch to the wrong numbered row.
  - confidence: 0.84
- **[P2] Archive prior history when disabling retention:** `src/auto-reply/reply/session.ts:650-651`
  - body: When `session.historyLimit` is `0`, this branch clears `sessionHistory` but only archives the immediately previous session later. Older retained history entries are dropped from state without being archived, leaving transcript files outside the normal reset-archive retention flow.
  - confidence: 0.82
- **[P2] Protect retained history from disk cleanup:** `src/auto-reply/reply/session.ts:649`
  - body: The PR intentionally keeps historical transcripts switchable while they remain in `sessionHistory`, but disk-budget cleanup still only treats active `sessionId` values as referenced. With `session.maintenance.maxDiskBytes`, retained transcripts can be removed while `/sessions` still lists them.
  - confidence: 0.78
- **[P2] Block switching while the current run is active:** `src/auto-reply/reply/commands-sessions.ts:277-281`
  - body: This switch path rewrites `params.sessionEntry.sessionId` immediately. If the old session still has active or queued work, `/stop` and late usage/state updates can target the newly selected session instead of the in-flight run.
  - confidence: 0.76
