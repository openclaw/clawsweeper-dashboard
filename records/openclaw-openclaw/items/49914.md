## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Restore the complete session boundary on switch:** `src/auto-reply/reply/commands-sessions.ts:280`
  - body: This switch mutates the active `SessionEntry` into the target session and then restores only `SessionHistoryItem.metadata`. Current session entries also carry auth profile overrides, exec/elevated policy, CLI/ACP handles, abort/fallback/cost/model state, delivery context, and plugin state; those remain from the outgoing session and can make the resumed conversation run under the wrong credentials, tool policy, or provider thread.
  - confidence: 0.91
- **[P1] Enforce disabled history before listing or resolving targets:** `src/auto-reply/reply/commands-sessions.ts:258`
  - body: `historyLimit` is resolved only after `/sessions` has already built target candidates from persisted `sessionHistory`, and the list path never checks it. If an operator changes `session.historyLimit` to `0` while old history exists, stale sessions remain visible and switchable despite the documented disabled mode.
  - confidence: 0.88
- **[P2] Protect history transcripts from disk-budget cleanup:** `src/config/sessions/types.ts:257`
  - body: Adding `sessionHistory` makes old transcripts live while they are no longer the active `entry.sessionId`. Current disk-budget cleanup only counts active entry artifacts and compaction checkpoint files, so `session.maintenance.maxDiskBytes` can remove a transcript that `/sessions` still lists as switchable.
  - confidence: 0.86
- **[P2] Document the final chat command contract:** `src/auto-reply/commands-registry.shared.ts:297-300`
  - body: The PR registers a new user-facing `/sessions` command, but the source docs still list only `/session idle`, `/session max-age`, and `/focus` for session management. Add the final command form and retention semantics to the user docs after maintainers settle `/sessions` versus `/resume`.
  - confidence: 0.82
- **[P2] Prefer exact or prefix IDs before numeric indexes:** `src/auto-reply/reply/commands-sessions.ts:219-222`
  - body: Numeric input is treated as a rendered list index before exact or prefix session-id lookup. A session whose unique id prefix is numeric-only, such as `1234`, can never be selected by the advertised prefix form because `/sessions 1234` jumps to index 1234 or reports not found.
  - confidence: 0.78
