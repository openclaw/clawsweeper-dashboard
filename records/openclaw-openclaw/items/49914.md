## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Restore the full session boundary before switching:** `src/auto-reply/reply/commands-sessions.ts:124-125`
  - body: `applyHistoryMetadata` restores only a subset of `SessionEntry` after the command rewrites `sessionId`. Fields such as `authProfileOverride`, CLI/ACP runtime handles, `/elevated` and `/exec` policy, `fastMode`, runtime model state, and fallback notice state remain from the outgoing session, so the resumed conversation can continue under the wrong credentials, provider thread, or sandbox policy.
  - confidence: 0.91
- **[P1] Protect retained history transcripts from cleanup:** `src/auto-reply/reply/session.ts:649`
  - body: This stores old sessions in `sessionHistory` so their transcripts remain switchable, but the disk-budget reference scanner still only protects active entries and compaction checkpoint files. With `session.maintenance.maxDiskBytes` in enforce mode, retained history transcripts can be pruned while `/sessions` still lists them, making a switch land in an empty or missing transcript.
  - confidence: 0.86
- **[P2] Honor disabled history before listing or switching:** `src/auto-reply/reply/commands-sessions.ts:170`
  - body: `/sessions` builds its target list from persisted `sessionHistory` before checking `session.historyLimit`. If an operator changes the limit to `0`, stale retained sessions remain listable and switchable until rollover, contradicting the disable contract and causing the outgoing current session to be archived during that switch path.
  - confidence: 0.84
- **[P3] Prefer ID matches before numeric indexes:** `src/auto-reply/reply/commands-sessions.ts:219-222`
  - body: Numeric input is treated as a rendered list index before exact or prefix session-id matching. That makes a numeric-only session-id prefix such as `/sessions 1234` impossible to select even though the PR advertises prefix matching; check exact/prefix matches before falling back to list indexes or require an explicit index marker.
  - confidence: 0.77
