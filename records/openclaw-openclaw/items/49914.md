- **[P1] Restore or clear the full session boundary:** `src/auto-reply/reply/commands-sessions.ts:277-280`
  - body: This switch rewrites the live `SessionEntry` to the target session but only reapplies a small metadata subset. Auth profile overrides, runtime model identity, CLI/ACP handles, exec/elevated policy, fallback notices, costs, abort state, pending delivery, and plugin-owned state can remain from the outgoing session, so a resumed transcript can run under the wrong credentials, runtime, or security policy.
  - confidence: 0.9
- **[P1] Protect retained history transcripts from cleanup:** `src/auto-reply/reply/session.ts:643-649`
  - body: The PR keeps previous transcripts switchable by storing them in `sessionHistory`, but the current disk-budget cleanup only counts active entry transcript paths and compaction checkpoint files as referenced. With `session.maintenance.maxDiskBytes`, a retained history transcript can be deleted while `/sessions` still lists it, causing switch-back to land on missing or empty history.
  - confidence: 0.86
- **[P2] Honor disabled history before exposing switch targets:** `src/auto-reply/reply/commands-sessions.ts:258-264`
  - body: `buildSwitchableSessionList` runs before `session.historyLimit` is checked, so stale persisted history remains listable and switchable after an operator sets the limit to `0`. Gate list/switch behavior on the resolved limit, or explicitly archive/clear stale history before exposing targets.
  - confidence: 0.88
