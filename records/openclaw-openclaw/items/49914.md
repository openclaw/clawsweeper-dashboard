  - body: This rewrites the active entry to the target session and then restores only `SessionHistoryItem.metadata`. Current main stores other session-scoped fields on the same entry, including auth profile, runtime model, CLI/ACP bindings, fallback notices, exec/elevated policy, response usage, and abort state; leaving those from the outgoing session means a resumed transcript can continue with the wrong credentials, runtime, or execution policy.
  - confidence: 0.93
- **[P1] Honor historyLimit=0 before building switch targets:** `src/auto-reply/reply/commands-sessions.ts:213-214`
  - body: `session.historyLimit=0` is documented as disabling switchable history, but this builds targets from persisted `sessionHistory` before checking the limit. After an operator turns the knob off, `/sessions <n>` can still switch to retained sessions and the zero-limit eviction loop can archive the session the user was just in.
  - confidence: 0.9
- **[P2] Protect retained history transcripts from cleanup:** `src/auto-reply/reply/session.ts:648-649`
  - body: This keeps previous transcripts live by assigning them to `sessionHistory`, but the current disk-budget reference scan protects only active store entries and checkpoints. Under `session.maintenance.maxDiskBytes`, retained `<oldSessionId>.jsonl` files can still look unreferenced and be deleted while `/sessions` continues to list them as switchable.
  - confidence: 0.86
