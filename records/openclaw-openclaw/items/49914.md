Full review comments:

- **[P1] Restore all session-owned state before switching:** `src/auto-reply/reply/commands-sessions.ts:280`
  - body: `applyHistoryMetadata` restores only a small hand-picked subset after `sessionId` is rewritten. On current main, `SessionEntry` also carries auth profile overrides, agent runtime overrides, CLI bindings, exec/elevated settings, ACP/runtime model fields, and other session-owned state, so `/sessions back` can resume an old transcript under the outgoing session's credentials/runtime or execution policy.
  - confidence: 0.88
- **[P1] Keep retained history transcripts referenced during cleanup:** `src/auto-reply/reply/session.ts:649`
  - body: This PR intentionally leaves prior transcripts switchable in `sessionHistory`, but disk-budget cleanup only marks each active store entry transcript and compaction checkpoints as referenced. With `session.maintenance.maxDiskBytes` enabled, a retained history transcript can be deleted while `/sessions` still lists it, so switching back can open an empty/default transcript.
  - confidence: 0.82
- **[P2] Resolve session IDs before numeric indexes:** `src/auto-reply/reply/commands-sessions.ts:219-221`
  - body: The parser treats any all-digit argument as a list index before trying exact or prefix session-id lookup. Since session ids can begin with hex digits, a unique numeric prefix such as `/sessions 1234` cannot select that session and instead jumps to item 1234 or reports not found, breaking the advertised prefix form.
  - confidence: 0.78
