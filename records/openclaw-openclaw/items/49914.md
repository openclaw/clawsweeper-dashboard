Full review comments:

- **[P1] Restore the complete session boundary before switching:** `src/auto-reply/reply/commands-sessions.ts:280`
  - body: This mutates the live `SessionEntry` to the target id but restores only a metadata subset. Later turns still read session-scoped auth overrides, CLI/ACP handles, execution/elevated policy, usage/cost, runtime model, fallback, and abort state, so a resumed session can run under another conversation's credentials, provider thread, or sandbox state.
  - confidence: 0.9
- **[P2] Honor disabled history before listing or switching:** `src/auto-reply/reply/commands-sessions.ts:258`
  - body: If `session.historyLimit` is set to `0` after history already exists, the handler still lists and resolves persisted history before applying the limit. That contradicts the disable contract and can archive the current session as a side effect of a switch that should be rejected.
  - confidence: 0.86
- **[P2] Reference retained history transcripts during cleanup:** `src/auto-reply/reply/session.ts:649`
  - body: The PR keeps old transcripts switchable by storing them in `sessionHistory`, but disk-budget cleanup only marks active session files and compaction checkpoint files as referenced. Retained old transcripts can be removed while `/sessions` still lists them, so switching back reopens a missing or empty transcript.
  - confidence: 0.84
