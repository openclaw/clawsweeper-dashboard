## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Restore the full session boundary before switching:** `src/auto-reply/reply/commands-sessions.ts:280`
  - body: `/sessions <target>` rewrites the live `SessionEntry` and then only reapplies the small `SessionHistoryItem.metadata` subset. Current entries also carry auth profiles, CLI/ACP runtime handles, elevated/exec policy, fast/usage settings, runtime model, fallback notice, cost, and abort state; leaving those from the outgoing session can resume the old conversation under the wrong credentials, provider-side thread, or execution boundary.
  - confidence: 0.9
- **[P1] Protect retained history transcripts from cleanup:** `src/auto-reply/reply/session.ts:649`
  - body: This PR intentionally keeps prior transcripts unarchived while they remain in `sessionHistory`, but current session cleanup only treats each entry's active `sessionId` artifacts as referenced. Under maintenance or `maxDiskBytes`, those retained history transcripts can be pruned as unreferenced primary transcripts while `/sessions` still lists them, so switching back can land in an empty session.
  - confidence: 0.86
- **[P2] Honor disabled history before listing or switching:** `src/auto-reply/reply/commands-sessions.ts:170`
  - body: If an operator changes `session.historyLimit` to `0` after history already exists, this command still builds targets from the stored history and can switch into them. Because the limit is only applied after target resolution, the current session is then pushed and immediately evicted/archived even though history is supposed to be disabled.
  - confidence: 0.86
- **[P3] Prefer session-id matches before numeric indexes:** `src/auto-reply/reply/commands-sessions.ts:219`
  - body: Numeric input is treated as a list index before exact or prefix session-id lookup. A session id with a unique all-digit prefix, which is common for UUID-style ids, cannot be selected by that advertised prefix and will instead jump to that index or report not found.
  - confidence: 0.72
