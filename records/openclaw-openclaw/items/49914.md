## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Restore the full session state before switching:** `src/auto-reply/reply/commands-sessions.ts:277-280`
  - body: This rewrites the live `SessionEntry` to the target `sessionId` and then reapplies only a small metadata subset. On current main, session entries also carry auth profile overrides, runtime model/harness fields, CLI and ACP handles, execution policy, plugin state, pending delivery, fallback notices, and delivery routes; leaving those from the previously active session can resume a transcript with the wrong credentials, provider thread, sandbox policy, or outbound route. Rebuild the active entry from a complete per-session snapshot, or fail closed for state you cannot safely restore.
  - confidence: 0.91
- **[P1] Honor disabled history before resolving switch targets:** `src/auto-reply/reply/commands-sessions.ts:258-264`
  - body: The handler builds `allItems` from persisted `sessionHistory` before checking `historyLimit`, so after an operator sets `session.historyLimit=0`, `/sessions` can still expose and switch to old retained sessions. A switch in that state also pushes the current session into `nextHistory` and immediately archives it through the eviction loop. Gate both listing and switching when the resolved limit is zero, and archive or clear retained history consistently.
  - confidence: 0.89
- **[P2] Keep retained history transcripts referenced by cleanup:** `src/auto-reply/reply/session.ts:649`
  - body: This stores previous transcripts as switchable history instead of archiving them, but current maintenance only treats the active `entry.sessionId` and compaction checkpoints as referenced. With `session.maintenance.maxDiskBytes` or unreferenced-artifact cleanup enabled, a retained history transcript can be deleted while `/sessions` still lists it, so switching back opens an empty or missing conversation. Include `sessionHistory` transcript files in the referenced-artifact set or archive them instead of advertising them as switchable.
  - confidence: 0.82
