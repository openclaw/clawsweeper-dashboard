## Summary

Keep open. This PR is still the canonical implementation path for https://github.com/openclaw/openclaw/issues/9959, but the current branch is conflicting, lacks inspectable real behavior proof, and still has correctness/security blockers in session-state restoration and command semantics.

## What This Changes

The PR adds a chat `/sessions` command, per-SessionKey LRU `sessionHistory`, retained transcript switching, `session.historyLimit` config/schema entries, and focused tests for listing and switching chat sessions.

## Best Possible Solution

Refresh the branch, define a complete per-session switch restore/clear/lifecycle contract, align the documented command syntax with implementation, protect retained transcripts through cleanup, and provide redacted real chat proof before maintainer review.

## Reproduction Assessment

Yes, for review by source inspection: current main lacks the built-in chat session switching, and the PR's remaining failure paths are visible in the proposed handler and metadata snapshot. I did not run tests or live chat proof because this review is read-only.

## Solution Assessment

No, not as-is. The feature direction matches the canonical request, but the proposed implementation needs a complete session-state contract, command behavior that matches the advertised syntax, disabled-history handling, transcript cleanup updates, and real chat proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Restore the full session state on switches:** `src/config/sessions/types.ts:81-112`
  - body: `SessionHistoryItemMetadata` is the snapshot reapplied after the live entry is rewritten, but it omits session-scoped auth profile overrides, execution policy, fast/usage flags, runtime model/provider, CLI/ACP handles, fallback/cost/status, and system prompt/plugin state. Switching can therefore resume one transcript while continuing under another conversation's credentials, provider thread, or execution policy.
  - confidence: 0.9
- **[P2] Honor the advertised singular switch command:** `src/auto-reply/reply/commands-sessions.ts:148-150`
  - body: The PR body and canonical feature request advertise `/session <number>`, `/session <sessionId>`, and `/session back`, but this new handler only matches `/sessions`. Users following the documented singular command still hit the existing lifecycle/settings handler instead of switching sessions.
  - confidence: 0.88
- **[P2] Check disabled history before listing switch targets:** `src/auto-reply/reply/commands-sessions.ts:168-213`
  - body: The handler builds list/switch targets from persisted `sessionHistory` before checking `session.historyLimit`. If history is changed to `0` after entries already exist, stale sessions can still be listed or selected, and selecting one evicts the current session under the disabled-history path.
  - confidence: 0.86
- **[P2] Match session ID prefixes before numeric indexes:** `src/auto-reply/reply/commands-sessions.ts:219-221`
  - body: Numeric input is interpreted as a rendered list index before exact or prefix session-id matching. A session id with a numeric-only unique prefix such as `1234...` cannot be selected by its documented prefix and may even switch to entry 1234 if enough sessions exist.
  - confidence: 0.82
- **[P2] Archive cleared history when disabling retention:** `src/auto-reply/reply/session.ts:650-651`
  - body: When `session.historyLimit` is `0`, this branch clears `sessionHistory` but only archives the immediately previous session later. Older retained history entries are dropped from state without being archived, leaving their transcript files outside the normal reset-archive retention flow.
  - confidence: 0.8
- **[P2] Preserve retained history transcripts during cleanup:** `src/auto-reply/reply/session.ts:648-650`
  - body: The PR intentionally keeps historical transcripts switchable while they remain in `sessionHistory`, but disk-budget cleanup still treats only each active `sessionId` as referenced. With `session.maintenance.maxDiskBytes`, retained transcripts can be removed while `/sessions` still lists them, so switching back can reopen an empty/default transcript.
  - confidence: 0.78
