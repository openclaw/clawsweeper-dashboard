## Summary

Keep this PR open, but it is not merge-ready: the central chat-session switching feature is still useful and current main does not implement it, while the latest PR head still has source-proven command-contract, state-restoration, cleanup, and proof blockers.

## What This Changes

The PR adds a `/sessions` chat command, per-SessionKey LRU session history retention, `session.historyLimit` config/schema support, command-list visibility filtering, and focused tests.

## Best Possible Solution

Define and implement a complete chat-session switch contract that restores or clears all session-scoped state, protects retained transcripts through cleanup and disable-history paths, aligns the slash-command UX with the requested surface, and is proven in a real chat channel.

## Reproduction Assessment

Yes, for the review findings by source inspection: the latest PR head only matches `/sessions`, resolves history targets before applying `historyLimit`, and restores a partial metadata snapshot. I did not run tests because this review is read-only.

## Solution Assessment

No. The direction is useful, but this branch should not merge until the state contract, disabled-history behavior, retained-transcript cleanup, documented command forms, and real chat proof are complete.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Restore complete session state when switching history:** `src/config/sessions/types.ts:81-112`
  - body: `SessionHistoryItemMetadata` is the only state reapplied after rewriting the live `SessionEntry`, but it omits auth profile overrides, CLI/ACP handles, exec/elevated policy, fast/usage flags, runtime model/fallback state, abort cutoffs, system reports, and cost. A resumed session can inherit credentials, execution policy, provider-side thread, or status/cost from the session the user just left.
  - confidence: 0.9
- **[P2] Wire the advertised `/session` switch forms:** `src/auto-reply/reply/commands-sessions.ts:148-150`
  - body: The linked request and PR body advertise `/session <number>`, `/session <sessionId>`, and `/session back`, but this handler only matches `/sessions`. The existing `/session` handler remains the lifecycle settings handler, so users following the requested command form get usage text instead of switching sessions.
  - confidence: 0.92
- **[P2] Reject disabled history before resolving stale targets:** `src/auto-reply/reply/commands-sessions.ts:213-214`
  - body: When `session.historyLimit` is changed to `0` while old history is still stored, this path builds switch targets from `sessionHistory` before reading the limit. That keeps old sessions listable and switchable after history is disabled and can archive the current conversation when it is pushed and immediately evicted.
  - confidence: 0.88
- **[P2] Keep retained history transcripts referenced during cleanup:** `src/config/sessions/types.ts:252-257`
  - body: Adding `sessionHistory` makes those transcripts intentionally switchable, but disk-budget cleanup still only treats active store entries as referenced. A retained history transcript can be pruned while `/sessions` still lists it, so switching back can reopen a missing or empty conversation.
  - confidence: 0.82
- **[P2] Archive cleared history transcripts when disabling history:** `src/auto-reply/reply/session.ts:650-651`
  - body: When `session.historyLimit` is `0`, the rollover path drops the whole existing `sessionHistory` array but only archives the immediate previous session. Older retained history transcripts are removed from state without being archived, leaving them orphaned outside the normal reset archive flow.
  - confidence: 0.84
- **[P3] Match ID prefixes before numeric row indexes:** `src/auto-reply/reply/commands-sessions.ts:219-222`
  - body: Numeric-only input is interpreted as a rendered row index before exact or prefix session-id matching. A valid session-id prefix such as `1234` can never select the matching session and instead jumps to row 1234 or reports not found, contradicting the advertised prefix behavior.
  - confidence: 0.82
