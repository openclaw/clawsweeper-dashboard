## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Restore the full session boundary on switch:** `src/auto-reply/reply/commands-sessions.ts:280`
  - body: This mutates the live entry into the target session and then restores only `SessionHistoryItem.metadata`. Current `SessionEntry` also carries auth profile overrides, exec/elevated policy, fast/usage settings, CLI session ids, ACP runtime, abort/fallback/cost/model state, and other session-scoped fields; those remain from the outgoing session after the switch, so resumed turns can run under the wrong credentials, tool policy, or provider thread. Store/restore the full boundary or switch by rebinding to a complete existing entry.
  - confidence: 0.9
- **[P1] Honor disabled history before listing or switching:** `src/auto-reply/reply/commands-sessions.ts:258`
  - body: `historyLimit` is resolved only after `allItems` and the target are built from persisted `sessionHistory`. If an operator sets `session.historyLimit` to `0` while stale history exists, `/sessions` still exposes old sessions and `/sessions 2` can switch to one, then the eviction loop archives the current session. That contradicts the documented disabled-history behavior.
  - confidence: 0.88
- **[P2] Protect retained history transcripts from cleanup:** `src/config/sessions/types.ts:257`
  - body: Adding `sessionHistory` makes prior transcripts live while they are no longer the active `entry.sessionId`. The disk-budget cleanup path on current main only marks active entry artifacts and compaction checkpoints as referenced, so `session.maintenance.maxDiskBytes` can remove a transcript that `/sessions` still lists as switchable. Include history item paths in referenced artifacts or archive/drop them consistently.
  - confidence: 0.86
- **[P2] Document the new command contract:** `src/auto-reply/commands-registry.shared.ts:296-300`
  - body: The branch registers a user-facing `/sessions` command but does not update `docs/tools/slash-commands.md`; current docs still list only `/session idle`, `/session max-age`, and `/focus`. The PR body also mentions `/session <number|id|back>` forms that this code does not implement, so users and native-command reviewers get conflicting command guidance.
  - confidence: 0.82
