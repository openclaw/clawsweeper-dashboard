## Summary

Keep open: current main still lacks chat-session switching, but this PR has blocking source-level state, command-contract, cleanup, and proof-gate issues, so it needs contributor follow-up plus maintainer review rather than cleanup closure.

## What This Changes

The PR adds a `/sessions` chat command, per-SessionKey LRU session history, `session.historyLimit` config/schema support, transcript retention/eviction behavior, command-list visibility filtering, and tests.

## Best Possible Solution

Define a complete chat-session switch contract that restores or clears all session-scoped runtime and security state, protects retained transcripts through maintenance and disable-history paths, aligns command UX/docs, and is proven in a real chat channel.

## Reproduction Assessment

Yes, for the review findings by source inspection: the PR head only matches `/sessions`, resolves switch targets before applying `historyLimit`, restores a partial metadata snapshot, and leaves retained-history cleanup paths incomplete. I did not run tests because this review is read-only.

## Solution Assessment

No. The direction is useful, but the branch should not merge until the session-state contract, disabled-history behavior, retained-transcript cleanup, documented command forms, and real chat proof are complete.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Restore complete state when switching sessions:** `src/config/sessions/types.ts:81-112`
  - body: `SessionHistoryItemMetadata` is the only state reapplied after rewriting the live `SessionEntry`, but it omits auth profile overrides, exec/elevated policy, fast/usage flags, runtime model/fallback state, abort cutoffs, system reports, cost, CLI handles, and ACP metadata. A resumed session can inherit credentials, execution policy, provider-side thread, or status/cost from the session the user just left.
  - confidence: 0.9
- **[P2] Honor the advertised /session switch commands:** `src/auto-reply/reply/commands-sessions.ts:148-150`
  - body: The linked request and PR body advertise `/session <number>`, `/session <sessionId>`, and `/session back`, but this handler only matches `/sessions`. Users following the requested command form still hit the existing lifecycle-settings handler instead of switching sessions.
  - confidence: 0.9
- **[P2] Check historyLimit before resolving retained targets:** `src/auto-reply/reply/commands-sessions.ts:212-214`
  - body: When `session.historyLimit` is changed to `0` while old history remains stored, this path still builds switch targets from `sessionHistory` before reading the limit. That keeps old sessions listable and switchable after history is disabled and can archive the current conversation when it is pushed and immediately evicted.
  - confidence: 0.88
- **[P2] Keep retained history transcripts referenced during cleanup:** `src/config/sessions/types.ts:253-257`
  - body: Adding `sessionHistory` intentionally leaves prior transcripts switchable, but disk-budget cleanup still only marks active store entries as referenced. A retained history transcript can be pruned while `/sessions` still lists it, so switching back can reopen a missing or empty conversation.
  - confidence: 0.82
- **[P2] Archive dropped history when disabling retention:** `src/auto-reply/reply/session.ts:650-651`
  - body: When `session.historyLimit` is `0`, the reset path clears the whole existing history array but only archives the immediate previous session later. Older retained history transcripts are removed from state without being archived, leaving them orphaned outside normal reset-archive handling.
  - confidence: 0.84
- **[P3] Resolve numeric prefixes before row numbers:** `src/auto-reply/reply/commands-sessions.ts:219-221`
  - body: Numeric-only input is interpreted as a rendered row index before exact or prefix session-id matching. A valid numeric session-id prefix can never select the matching session and instead jumps to that row or reports not found, contradicting the advertised prefix behavior.
  - confidence: 0.8
