Full review comments:

- **[P1] Restore or clear the full session boundary:** `src/auto-reply/reply/commands-sessions.ts:277-280`
  - body: This switch rewrites the active `SessionEntry` to the target session and only reapplies the small `SessionHistoryItem.metadata` subset. Current sessions also carry auth profiles, CLI bindings, ACP metadata, exec/elevated defaults, abort cutoffs, runtime model fields, fallback/cost state, and plugin/delivery state; those can be inherited from the outgoing session, so a resumed chat can continue under the wrong credentials, runtime thread, or execution policy.
  - confidence: 0.9
- **[P1] Protect retained history transcripts from cleanup:** `src/auto-reply/reply/session.ts:648-649`
  - body: The PR intentionally keeps previous transcripts live by storing them in `sessionHistory`, but disk-budget cleanup only treats each active entry transcript as referenced. With a session disk budget enabled, retained history files can be deleted while `/sessions` still lists them, so switching back can land in a missing transcript.
  - confidence: 0.84
- **[P2] Honor disabled history before listing or switching:** `src/auto-reply/reply/commands-sessions.ts:170`
  - body: When `session.historyLimit` is changed to `0`, this handler still builds `/sessions` output and switch targets from persisted `sessionHistory` before checking the limit. During that window old sessions remain visible and selectable even though history is disabled; return the disabled-history response before list or target resolution.
  - confidence: 0.8
