## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Honor disabled history before building switch targets:** `src/auto-reply/reply/commands-sessions.ts:170`
  - body: `buildSwitchableSessionList` is called before this handler checks `session.historyLimit`, so a store that already has `sessionHistory` entries still lists and switches them after the operator sets the new limit to `0`. The switch path then pushes the current session and evicts it immediately, archiving the conversation the user is in while disabled history is supposed to make prior sessions unreachable.
  - confidence: 0.91
- **[P1] Restore the full session boundary before switching:** `src/auto-reply/reply/commands-sessions.ts:280`
  - body: After rewriting `sessionId`, this only reapplies `SessionHistoryItem.metadata`; fields such as auth profile overrides, CLI/ACP runtime handles, fast/usage settings, elevated/exec policy, fallback notice, runtime model, cost, and abort cutoffs remain from the outgoing session. A resumed conversation can therefore run under the wrong credentials, provider-side thread, or execution policy.
  - confidence: 0.9
- **[P1] Protect retained history transcripts from cleanup:** `src/auto-reply/reply/session.ts:649`
  - body: The PR keeps old transcripts switchable by saving `sessionHistory`, but disk-budget cleanup still treats only active store-entry transcript paths as referenced. A retained session can remain visible in `/sessions` while its primary transcript has been pruned, so switching back can land in an empty or wrong transcript.
  - confidence: 0.85
- **[P2] Emit lifecycle hooks for switch transitions:** `src/auto-reply/reply/commands-sessions.ts:277-281`
  - body: `/sessions <target>` changes the active `sessionId` and persists it without going through the existing `session_end`/`session_start` paths. Plugins that allocate or release per-session resources from lifecycle hooks will not observe this transition and can stay attached to the wrong session.
  - confidence: 0.78
