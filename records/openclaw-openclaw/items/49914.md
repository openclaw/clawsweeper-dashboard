- **[P1] Restore the full session state on switches:** `src/config/sessions/types.ts:74-112`

Switching sessions can therefore resume one transcript while continuing under another conversation's credentials, provider thread, or policy.
  - confidence: 0.88
- **[P2] Honor the advertised /session switch commands:** `src/auto-reply/commands-registry.shared.ts:293-309`
  - body: The PR body advertises `/session <number>`, `/session <sessionId>`, and `/session back`, but the added registry entry is for `/sessions` with a captured target. Users following the documented singular command form can still hit the existing lifecycle handler instead of the new switch path.
  - confidence: 0.86
- **[P2] Check disabled history before resolving old targets:** `src/auto-reply/reply/commands-sessions.ts:148-213`
  - body: The switch command builds targets from persisted history before enforcing `session.historyLimit=0`. If an operator disables history after entries already exist, old sessions can remain listable or switchable until another reset, contrary to the advertised disable behavior.
  - confidence: 0.82
