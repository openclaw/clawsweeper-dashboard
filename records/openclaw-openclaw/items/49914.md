- **[P1] Restore the full session boundary on switch:** `src/auto-reply/reply/commands-sessions.ts:277-280`
  - body: This mutates the active entry to the target session and then restores only `SessionHistoryItem.metadata`. Current session entries also carry auth profile, runtime model, CLI/ACP bindings, exec/elevated policy, usage, fallback, abort, and report state; leaving those from the outgoing session can resume a transcript under the wrong credentials, runtime, or execution policy.
  - confidence: 0.93
- **[P1] Honor disabled history before resolving targets:** `src/auto-reply/reply/commands-sessions.ts:213-214`
  - body: `session.historyLimit=0` is documented as disabling switchable history, but target resolution still reads retained `sessionHistory` before checking the limit. After an operator turns the knob off, `/sessions <n>` can switch to stale retained sessions and the zero-limit eviction loop can archive the session the user was just in.
  - confidence: 0.9
- **[P2] Protect retained history transcripts from cleanup:** `src/auto-reply/reply/session.ts:648-649`
  - body: This keeps previous transcripts live by assigning them to `sessionHistory`, but current disk-budget cleanup protects only active store entries and compaction checkpoints. With `session.maintenance.maxDiskBytes`, retained transcripts can be deleted while `/sessions` still lists them as switchable.
  - confidence: 0.86
