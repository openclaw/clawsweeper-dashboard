Full review comments:

- **[P1] Restore the full session contract before switching:** `src/auto-reply/reply/commands-sessions.ts:277-280`
  - body: This rewrites the live `SessionEntry` to the target session and only reapplies `SessionHistoryItem.metadata`. Current main stores auth/profile sources, runtime harnesses, exec/elevated policy, pending delivery, CLI/ACP handles, fallback notices, plugin state, costs, and routing fields on the same entry, so the resumed session can inherit credentials, runtime, policy, or delivery state from the previous session.
  - confidence: 0.93
- **[P2] Gate history commands when history is disabled:** `src/auto-reply/reply/commands-sessions.ts:170`
  - body: `/sessions` builds its list from persisted `sessionHistory` before checking `session.historyLimit`, so changing the config to `0` can still expose and switch to older sessions until another reset. Return the disabled-history response before listing or resolving targets, and make the cleanup path deliberate.
  - confidence: 0.88
- **[P2] Archive retained history before clearing it:** `src/auto-reply/reply/session.ts:650-652`
  - body: When `historyLimit === 0`, this clears the retained history array but later archives only `previousSessionEntry`; older retained entries are removed from state without normal archive handling. Archive each retained item before dropping it, or preserve state until a maintenance path owns the transition.
  - confidence: 0.82
