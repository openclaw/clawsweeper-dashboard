  - body: The PR now keeps previous transcripts switchable by copying `sessionHistory`, but the existing disk-budget and unreferenced-artifact sweep only mark active entry transcript paths and compaction checkpoints as referenced. Under session maintenance, a retained history transcript can be deleted while `/sessions` still lists it, so switching back can land in an empty or missing conversation.
  - confidence: 0.88

  - body: `buildSwitchableSessionList` and target selection run before the configured history limit is considered. If an operator changes `session.historyLimit` to `0` after history exists, `/sessions` can still expose stale targets and switching will immediately evict/archive the current session instead of refusing because history is disabled.
  - confidence: 0.86
- **[P2] Prefer ID prefix matches before numeric indexes:** `src/auto-reply/reply/commands-sessions.ts:219-221`
  - body: The command treats any all-digit argument as a list index before trying exact or prefix session-id matching. Numeric UUID prefixes are valid advertised `sessionId prefix` inputs, so `/sessions 1234` can never select a session whose unique id prefix is `1234`; it either jumps to row 1234 or reports not found.
  - confidence: 0.78
- **[P2] Document the new session retention config:** `src/config/types.base.ts:120-121`
