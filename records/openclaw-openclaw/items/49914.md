## Review Findings

Overall confidence: 0.88

Full review comments:

- **[P1] Restore the full session boundary before switching:** `src/auto-reply/reply/commands-sessions.ts:115-141`
  - body: This restores only a small metadata subset after rewriting `sessionId`, leaving durable fields such as auth profile, runtime model, CLI/ACP handles, exec/elevated policy, fallback state, cost, plugin state, and current-main session cleanup fields from the outgoing session. Switching back can therefore resume one transcript under another session's credentials, runtime, or permissions; rebase and preserve/clear the complete per-session boundary before persisting the switch.
  - confidence: 0.92
- **[P1] Block switching when history is disabled:** `src/auto-reply/reply/commands-sessions.ts:258-264`
  - body: `session.historyLimit=0` is documented as disabling switchable history, but stale `sessionHistory` entries are still targetable and this path pushes the outgoing session before trimming to zero. An operator who disables history can still expose old sessions and unexpectedly archive the current transcript during the switch.
  - confidence: 0.88
- **[P2] Reference retained history transcripts during disk cleanup:** `src/auto-reply/reply/session.ts:643-649`
  - body: This keeps previous transcripts switchable while they remain in `sessionHistory`, but the disk-budget sweep only protects the active transcript for each store entry. With `session.maintenance.maxDiskBytes` enabled, a retained history transcript can be deleted as unreferenced while `/sessions` still lists it, so switching back lands on an empty or missing transcript.
  - confidence: 0.82
- **[P3] Document the new chat command in slash-command docs:** `src/auto-reply/commands-registry.shared.ts:300-310`
  - body: The PR registers a new user-visible `/sessions` command, but the branch's slash-command docs still list only `/session idle` and `/session max-age`. Add the new command and switching syntax to the public command docs so the documented chat surface matches the registry.
  - confidence: 0.78
