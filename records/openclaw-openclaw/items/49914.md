## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Restore the full session state before switching:** `src/auto-reply/reply/commands-sessions.ts:277-280`
  - body: This rewrites the live `SessionEntry` to the target session id and then reapplies only a small metadata subset. Current main also stores auth profile overrides, runtime model/harness fields, CLI and ACP handles, execution policy, plugin state, pending delivery, fallback notices, delivery routes, and lifecycle state; leaving those from the previous session can resume a transcript with the wrong credentials, provider thread, sandbox policy, or outbound route. Rebuild from a complete per-session snapshot, or fail closed for state that cannot be safely restored.
  - confidence: 0.91
- **[P1] Honor disabled history before resolving switch targets:** `src/auto-reply/reply/commands-sessions.ts:258-264`
  - body: The handler builds switch targets from persisted `sessionHistory` before checking `historyLimit`, so after an operator sets `session.historyLimit=0`, `/sessions` can still expose and switch to old retained sessions. A switch in that state also pushes the current session into `nextHistory` and immediately archives it through the eviction loop. Gate both listing and switching when the resolved limit is zero, and archive or clear retained history deliberately.
  - confidence: 0.88
- **[P2] Keep retained history transcripts referenced by cleanup:** `src/auto-reply/reply/session.ts:649`
  - body: The PR intentionally keeps older transcripts switchable by saving them in `sessionHistory`, but current disk-budget cleanup only treats active store entries and compaction checkpoints as referenced. With `session.maintenance.maxDiskBytes`, retained history files can be removed as unreferenced while `/sessions` still lists them, so switching back can land in an empty or missing transcript. Include history items in the referenced-artifact set or archive/remove them from history before cleanup can delete them.
  - confidence: 0.84
