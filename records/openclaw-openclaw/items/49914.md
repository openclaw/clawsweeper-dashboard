## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Restore the full session boundary before switching:** `src/auto-reply/reply/commands-sessions.ts:277-280`
  - body: The switch path rewrites `sessionEntry.sessionId` and then only reapplies the small `SessionHistoryItem.metadata` subset. Fields such as auth profile overrides, CLI/ACP runtime handles, exec/elevated policy, runtime model fields, fallback state, and cost/usage state can therefore leak from the outgoing session into the resumed one; the history item needs to restore a complete safe session boundary or switch by loading the target entry instead of mutating a partial snapshot.
  - confidence: 0.9
- **[P1] Protect retained history transcripts from cleanup:** `src/auto-reply/reply/session.ts:648-649`
  - body: This PR intentionally leaves history transcripts switchable, but the cleanup and disk-budget code still only mark active store entries as referenced. A retained history transcript can be pruned as an unreferenced primary transcript while `/sessions` continues to list it, so switching back can land in an empty or missing conversation; history entries need to participate in referenced-path accounting or be archived consistently.
  - confidence: 0.86
- **[P2] Honor disabled history before listing or switching:** `src/auto-reply/reply/commands-sessions.ts:170`
  - body: When `session.historyLimit` is set to `0`, `/sessions` still builds its list from any persisted `sessionHistory` entries. Switching to one of those entries then pushes the current session and immediately evicts it under the zero limit, which archives the conversation the user was just in despite history being disabled; gate listing and switching on the resolved limit before exposing stale history.
  - confidence: 0.88
- **[P3] Prefer unique ID prefixes before numeric indexes:** `src/auto-reply/reply/commands-sessions.ts:219-222`
  - body: The command treats any all-digit argument as a rendered list index before trying exact or prefix session-id matching. Numeric session-id prefixes are therefore impossible to select even though the PR advertises sessionId prefix switching; resolve exact and unique-prefix IDs before falling back to numeric index selection, or document index-only numeric input.
  - confidence: 0.76
