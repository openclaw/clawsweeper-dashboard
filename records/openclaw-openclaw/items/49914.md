  - body: `/sessions` rewrites the live `SessionEntry` to the target session id/file and only reapplies the small `SessionHistoryItemMetadata` subset. Fields such as auth profile, CLI/ACP bindings, exec/elevated policy, fast/usage settings, runtime model, abort/fallback/cost/report state can remain from the outgoing session, so the user can resume one transcript while future turns use another session's credentials or runtime policy.
  - confidence: 0.9
- **[P1] Protect retained history transcripts from cleanup:** `src/auto-reply/reply/session.ts:649`
  - body: This keeps previous session transcripts switchable by storing `sessionHistory`, but the existing cleanup paths only mark each active entry transcript and compaction checkpoints as referenced. With session maintenance or disk-budget cleanup enabled, a retained history transcript can be pruned while `/sessions` still lists it, so switching back opens missing or empty context.
  - confidence: 0.88
- **[P2] Honor disabled history before selecting targets:** `src/auto-reply/reply/commands-sessions.ts:213`
  - body: The switch target list is built from persisted `sessionHistory` before checking `session.historyLimit`. If an operator changes the limit to `0` after history exists, stale sessions remain listable and switchable until another reset clears them, contradicting the disable contract and potentially exposing old conversations.
  - confidence: 0.86
