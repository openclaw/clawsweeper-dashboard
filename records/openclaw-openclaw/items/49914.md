## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Restore the full session boundary before switching:** `src/auto-reply/reply/commands-sessions.ts:277-283`
  - body: This mutates the live `SessionEntry` to the target session and then restores only the small history metadata subset. Fields such as `authProfileOverride`, CLI/ACP handles, exec/elevated policy, fallback/model runtime state, abort state, cost, and plugin runtime state remain from the outgoing session, so a resumed chat can continue under the wrong credentials, provider-side thread, or tool policy.
  - confidence: 0.91
- **[P2] Honor disabled history before listing retained sessions:** `src/auto-reply/reply/commands-sessions.ts:213-215`
  - body: `buildSwitchableSessionList` includes retained `sessionHistory` before the command checks `session.historyLimit`. If an operator changes the limit to `0`, `/sessions` can still show and switch to stale entries, and the switch path then evicts and archives the current active session even though switchable history is supposed to be disabled.
  - confidence: 0.88
- **[P2] Reference retained transcripts during cleanup:** `src/auto-reply/reply/session.ts:649-653`
  - body: The PR keeps previous transcript files switchable by storing them in `sessionHistory`, but the existing disk-budget and unreferenced-artifact cleanup only sees live session entries/checkpoints. A retained transcript can therefore be pruned while `/sessions` still advertises it as resumable.
  - confidence: 0.82
