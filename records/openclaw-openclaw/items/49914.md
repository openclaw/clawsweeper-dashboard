## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Restore the full session boundary before switching:** `src/config/sessions/types.ts:81-112`
  - body: The new history snapshot only restores a subset of SessionEntry, while current sessions also carry auth profiles, CLI/ACP runtime handles, execution/elevated policy, fallback notices, costs, plugin state, and other per-session fields. Because the switch path mutates the live entry and applies only this subset, a resumed session can inherit another session's credentials or runtime policy.
  - confidence: 0.92
- **[P2] Honor disabled history before listing or switching:** `src/auto-reply/reply/commands-sessions.ts:170`
  - body: `/sessions` builds switch targets from persisted `sessionHistory` before checking `session.historyLimit`. If an operator changes the limit to `0`, stale history remains listable and switchable until another rollover, contradicting the disable contract and potentially archiving the current session during the attempted switch.
  - confidence: 0.86
- **[P2] Reference retained history transcripts during cleanup:** `src/config/sessions/types.ts:253-257`
  - body: This field makes old transcripts intentionally switchable, but current disk-budget cleanup only references each entry's active transcript and compaction checkpoints. Under session maintenance, a retained history transcript can be treated as unreferenced and deleted while `/sessions` still advertises it.
  - confidence: 0.82
