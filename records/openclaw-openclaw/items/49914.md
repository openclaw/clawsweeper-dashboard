## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Restore the full session boundary on switch:** `src/auto-reply/reply/commands-sessions.ts:280`
  - body: This switch mutates the existing `SessionEntry` to a target `sessionId` and reapplies only `SessionHistoryItem.metadata`. Current main stores auth profiles, runtime model/provider fields, CLI/ACP handles, abort state, fallback notices, execution policy, costs, and plugin/session reports on the same entry, so `/sessions back` can resume one transcript while continuing under another session's credentials, sandbox, provider thread, or status state.
  - confidence: 0.88
- **[P1] Honor disabled history before resolving switch targets:** `src/auto-reply/reply/commands-sessions.ts:213`
  - body: `session.historyLimit=0` is documented as disabling switchable history, but the command builds targets from `sessionHistory` before checking the limit. If an operator disables history after entries exist, `/sessions` can still list retained sessions and `/sessions <n>` can switch to them, then evict/archive the current session as a side effect.
  - confidence: 0.86
- **[P2] Reference retained history transcripts during cleanup:** `src/config/sessions/types.ts:257`
  - body: The PR adds `sessionHistory` entries that intentionally keep old transcripts switchable, but the disk-budget cleanup owner was not updated to treat those retained transcript paths as referenced. Deployments with `session.maintenance.maxDiskBytes` can delete a history transcript while `/sessions` still lists it, so switching back lands in an empty or missing conversation.
  - confidence: 0.82
- **[P2] Resolve ID prefixes before numeric indexes:** `src/auto-reply/reply/commands-sessions.ts:219`
  - body: Numeric-only input is interpreted as a rendered row index before exact or prefix session-id matching. UUID prefixes often start with digits, so `/sessions 1234` can never select a session whose unique id prefix is `1234`; it jumps to row 1234 or reports not found instead of honoring the advertised prefix contract.
  - confidence: 0.78
- **[P2] Emit lifecycle hooks for switched sessions:** `src/auto-reply/reply/commands-sessions.ts:277`
  - body: The switch path changes the active session id and persists it without emitting `session_end` for the outgoing session or `session_start` for the resumed one. Those hooks are the documented boundary for plugins that allocate or release per-session resources, so plugins can stay attached to the wrong session after `/sessions back`.
  - confidence: 0.73
