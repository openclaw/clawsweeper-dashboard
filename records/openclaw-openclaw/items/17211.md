## Summary

Keep this issue open: current main provides compaction transcript snapshots, branch/restore operations, context-engine compaction hooks, and restart recovery, but it still lacks the requested durable task-state contract, deterministic replay packet, checkpoint diffing, and validation lifecycle; the linked passive-recovery PR is narrower, conflicting, and not a safe canonical replacement.

## What This Changes

Add a native checkpoint engine with create/list/resume/diff APIs, structured task-state contracts, deterministic replay packets, and missing-field validation hooks.

## Best Possible Solution

Design one versioned session checkpoint contract that composes with existing transcript snapshots and context engines, stores canonical durable state in SQLite, and resumes only validated informational state without inheriting approvals or replaying side effects.

## Reproduction Assessment

Not applicable; this requests new deterministic checkpoint and replay capabilities rather than reporting a failure of an existing documented contract. Source inspection provides a high-confidence capability-gap check instead.

## Solution Assessment

Unclear as proposed: native APIs are plausible, but the best solution must first establish the canonical boundary among Gateway session protocol, SQLite persistence, context-engine plugins, and restart recovery, with explicit replay-safety rules.
