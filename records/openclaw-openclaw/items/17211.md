## Summary

Keep this feature umbrella open: current main has compaction checkpoint list/get/branch/restore for transcript snapshot recovery, but it does not provide the requested native checkpoint create/list/resume/diff API, deterministic replay packet, task-boundary contract, diffing, or validation hooks.

## What This Changes

Request a first-class native checkpoint engine with create/list/resume/diff APIs, deterministic replay, structured task-boundary fields, and validation hooks for safer long-running task continuation.

## Best Possible Solution

Define a versioned checkpoint/replay contract that composes existing compaction checkpoints, session persistence, Codex compaction boundaries, and security rules with explicit create/list/resume/diff and validation semantics.
