## Summary

Current main and the latest release provide compaction snapshot listing, branching, and restore, but not the requested task-boundary checkpoint contract, deterministic replay packets, validation, or diff semantics. The linked passive-recovery PR is materially different and is not a safe canonical replacement, so this feature still needs maintainer product and architecture review.

## What This Changes

Add a native checkpoint engine with structured task-boundary state, deterministic resume packets, validation, listing, and diff APIs.

## Best Possible Solution

Define a maintainer-approved, versioned checkpoint contract that composes with existing compaction snapshots, stores canonical durable state in the appropriate SQLite database, makes recovered data non-authoritative and approval-safe, and exposes the minimum core or plugin API needed for deterministic resume, validation, and audit diffing.
