## Summary

Keep this issue open: current main still has compaction checkpoint snapshot listing/branch/restore and restart recovery, but not the requested native checkpoint.create/list/resume/diff APIs, deterministic replay packet, task-boundary contract, diffing, or missing-field validation lifecycle.

## What This Changes

Request a native checkpoint engine with deterministic resume, checkpoint diffing, structured task-boundary fields, and validation hooks to reduce context drift after compaction.

## Best Possible Solution

Design one versioned session checkpoint contract that composes with existing transcript snapshots and context engines, persists canonical task state safely, and resumes only validated state without replaying side effects or inheriting approvals.

## Solution Assessment
