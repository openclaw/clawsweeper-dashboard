## Summary

Keep open. Current main still only provides compaction checkpoint snapshot list/get/branch/restore behavior, while the requested native checkpoint engine needs new checkpoint APIs, deterministic replay packets, task-boundary contracts, diffing, and validation hooks; maintainer discussion also explicitly kept this umbrella open.

## What This Changes

Request adds first-class checkpoint.create/list/resume/diff APIs with structured task-boundary contracts, deterministic replay packets, diffs, and validation hooks.

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

No current implementation solves the issue. The existing compaction checkpoint branch/restore path is useful, but the maintainable path is a versioned checkpoint/replay/diff API design before implementation or plugin delegation.
