## Summary

Keep open. Current main has compaction checkpoint snapshot recovery and UI branch/restore controls, but it still lacks the requested native checkpoint engine, checkpoint.create/list/resume/diff APIs, deterministic replay packets, task-boundary contract fields, diffing, and validation hooks; maintainer discussion also explicitly kept this umbrella open.

## What This Changes

Request to add first-class checkpoint.create/list/resume/diff APIs with structured task-boundary state, deterministic replay packets, diffing, and missing-field validation hooks.

## Best Possible Solution

Keep this as the native checkpoint/replay umbrella and design a versioned Gateway/session persistence plus runtime/context-engine contract before implementation, with compatibility for existing compaction checkpoint branch/restore controls.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The current-main gap is high-confidence from the registered Gateway methods, protocol schemas, server behavior, tests, docs, and absence of the requested checkpoint/replay method surface.

## Solution Assessment

No current implementation solves the issue. The existing compaction snapshot recovery path is useful but not the complete or narrowest maintainable solution for deterministic checkpoint/replay/diff semantics; the next step is a versioned protocol and runtime contract design.
