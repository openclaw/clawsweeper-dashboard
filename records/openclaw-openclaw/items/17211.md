## Summary

Keep open. Current main still only provides compaction-scoped checkpoint inspection plus snapshot branch/restore, while the requested native checkpoint engine with create/list/resume/diff, deterministic replay packets, task-boundary contracts, diffing, and validation hooks remains unimplemented.

## What This Changes

Add first-class task-boundary checkpoint APIs and deterministic replay/diff semantics for stable long-running task continuation.

## Best Possible Solution

Design a versioned native checkpoint/replay contract across Gateway/session protocol, persistence, context-engine/runtime, UI, docs, validation policy, and tests while preserving compaction branch/restore as an adjacent recovery primitive.

## Reproduction Assessment

Not applicable: this is a feature request rather than a broken current-main behavior report. The remaining gap is source-verifiable from the Gateway registry, protocol schema, server handlers, context-engine interface, tests, docs, and live discussion.

## Solution Assessment

No current implementation solves the issue. Treating existing compaction branch/restore as deterministic resume would conflate snapshot recovery with the requested native replay/diff engine; the maintainable path is a maintainer-led API and replay-safety design.
