## Summary

Keep open. Current main has compaction-scoped checkpoint inspection/branch/restore and restart recovery, but it still lacks the requested native checkpoint.create/list/resume/diff APIs, deterministic replay packets, checkpoint diffing, task-boundary contracts, and missing-field validation hooks; prior live discussion also explicitly kept this umbrella open.

## What This Changes

Request adding native checkpoint APIs, structured task-boundary state, deterministic replay packet generation, diffing, and validation hooks for reliable long-session resume.

## Best Possible Solution

Design a versioned native checkpoint/replay contract across Gateway/session protocol, persistence, context-engine/runtime, UI, docs, validation policy, and tests while preserving compaction branch/restore as an adjacent recovery primitive.

## Reproduction Assessment

Not applicable: this is a feature/API request rather than a broken current-main behavior report. The gap is source-verifiable from the Gateway method registry, protocol schema, server handlers, context-engine interface, tests, docs, and live discussion.

## Solution Assessment

No current implementation solves the issue. The maintainable path is maintainer-led API and replay-safety design for a native checkpoint/replay/diff contract, not treating compaction snapshot recovery as deterministic resume.
