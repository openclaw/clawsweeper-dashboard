## Summary

Keep open: current main and v2026.5.12 still expose compaction checkpoint snapshot recovery only, while this asks for a new native checkpoint/replay/diff contract. The live discussion already distinguishes the existing compaction feature from this broader session-state API work.

## Best Possible Solution

Design a versioned checkpoint/replay contract across Gateway/session protocol, persistence, runtime/context-engine hooks, UI, docs, and tests while preserving existing compaction branch/restore as an adjacent recovery primitive.
