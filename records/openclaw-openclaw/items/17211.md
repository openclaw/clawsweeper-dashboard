## Summary

Keep open. Current main has compaction checkpoint snapshot recovery and UI branch/restore controls, but it still does not provide the requested native checkpoint engine, deterministic replay packets, task-boundary contract fields, diffing, or validation hooks; the existing maintainer discussion also explicitly kept this umbrella open.

## What This Changes

The issue requests first-class checkpoint create/list/resume/diff APIs with structured task-boundary state, deterministic replay, diff/audit output, and missing-field validation.

## Best Possible Solution

Keep this as the native checkpoint/replay umbrella. A complete solution should define versioned Gateway/session protocol and persistence for checkpoint.create/list/resume/diff, structured done/saved/dropped/next state, deterministic replay packets, diff/audit data, validation hooks, UI/docs/tests, and a plugin/runtime API only after the core contract is clear.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The current-main gap is high-confidence from inspecting registered Gateway methods, protocol schemas, server behavior, tests, docs, and plugin/context-engine contracts.

## Solution Assessment

No current implementation solves the issue. The existing compaction snapshot branch/restore path is useful but not the best or complete answer for deterministic checkpoint/replay/diff semantics; the maintainable path is a versioned protocol and runtime contract design.
