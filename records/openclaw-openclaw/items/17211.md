## Summary

Keep open. Current main has compaction checkpoint snapshot branch/restore and context-engine compaction hooks, but it still does not implement the requested native checkpoint engine with checkpoint.create/list/resume/diff APIs, deterministic replay packets, task-boundary contracts, diffing, or native validation hooks; recent issue comments also kept this umbrella open.

## What This Changes

Request adding first-class checkpoint.create/list/resume/diff APIs with structured task-boundary state, deterministic replay packets, diffing, and validation hooks.

## Best Possible Solution

Keep this as the native checkpoint/replay umbrella and design a versioned Gateway/session/context-engine contract that builds on compaction checkpoint branch/restore without treating it as deterministic replay.

Not applicable as a bug reproduction; this is a feature request. The current-main gap is source-verifiable from method registration, protocol schema, server behavior, tests, docs, context-engine types, and exact absence searches.

## Solution Assessment
