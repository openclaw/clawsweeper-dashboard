## Summary

Keep open. Current main still only provides compaction-scoped checkpoint inspection and snapshot branch/restore; it does not implement the broader native checkpoint engine with create/list/resume/diff APIs, deterministic replay packets, task-boundary contracts, diffing, or validation hooks. Existing live discussion also explicitly kept this umbrella open after distinguishing it from compaction checkpoint recovery.

## What This Changes

The issue requests a first-class native checkpoint engine for task-boundary checkpoint creation, deterministic resume/replay, checkpoint diffing, and structured missing-field validation.

## Best Possible Solution

Design a versioned native checkpoint/replay contract across Gateway/session protocol, persistence, context-engine/runtime, UI, docs, validation policy, and tests while preserving current compaction branch/restore as an adjacent recovery primitive.

## Reproduction Assessment

Not applicable: this is a feature request rather than a broken current-main behavior report. The remaining gap is source-verifiable from the Gateway method registry, protocol schema, server handlers, context-engine interface, tests, docs, and existing discussion.

## Solution Assessment

No current implementation solves the issue. The maintainable path is a maintainer-led API and replay-safety design for a native checkpoint/replay/diff contract, not treating compaction snapshot recovery as deterministic resume.
