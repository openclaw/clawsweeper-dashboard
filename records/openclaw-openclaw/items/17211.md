## Summary

Keep open. Current main still provides compaction checkpoint snapshot recovery, not the requested native checkpoint engine with create/list/resume/diff APIs, deterministic replay packets, task-boundary contracts, diffing, or missing-field validation hooks; prior maintainer-facing discussion also kept this umbrella open.

## What This Changes

Request first-class checkpoint APIs, deterministic replay packet generation, diffing, and task-boundary validation for reliable long-running task resume.

## Best Possible Solution

Keep this as the native checkpoint/replay umbrella and design a versioned Gateway/session contract that builds on current compaction checkpoint branch/restore without treating it as deterministic replay.
