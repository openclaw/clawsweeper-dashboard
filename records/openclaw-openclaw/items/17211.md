## Summary

Keep this feature umbrella open. ## What This Changes

Requests a native checkpoint engine with deterministic resume, replay packets, structured task-boundary state, validation, and checkpoint diffs.

## Best Possible Solution

Define one versioned checkpoint contract that extends the existing compaction snapshot model and uses canonical session persistence, with explicit task-state validation, deterministic replay, resume, diff/audit semantics, and no inherited approvals or automatic execution.
