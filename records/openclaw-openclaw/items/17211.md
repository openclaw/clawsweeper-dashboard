## Summary

Keep open: current main and the latest release still only expose compaction checkpoint list/get/branch/restore for transcript snapshot recovery, not the requested native checkpoint create/list/resume/diff engine with deterministic replay packets, task-boundary contracts, diffing, and validation hooks.

## What This Changes

Request to add first-class checkpoint.create/list/resume/diff APIs with structured task-boundary state, deterministic replay, diff/audit output, and missing-field validation.

## Best Possible Solution

Define a versioned native checkpoint/replay contract that composes existing compaction checkpoints, canonical session persistence, security rules, Gateway/UI/API surfaces, and validation semantics before implementation.
