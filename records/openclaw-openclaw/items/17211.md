## Summary

Keep open. Current main still has compaction-scoped checkpoint inspection and snapshot branch/restore, but not the requested native checkpoint engine with create/list/resume/diff APIs, deterministic replay packets, task-boundary contracts, diffing, or validation hooks.

## What This Changes

Request a first-class native checkpoint engine for deterministic task resume, replay, diffing, and structured task-boundary validation.

## Best Possible Solution

Design a versioned native checkpoint/replay contract across Gateway/session protocol, persistence, context-engine/runtime, UI, docs, validation policy, and tests while preserving current compaction branch/restore as an adjacent primitive.

Not applicable: this is a feature request rather than a broken current-main behavior report. The current-main gap is source-verifiable from the method registry, protocol schema, server handlers, context-engine interface, tests, and live related discussion.

## Solution Assessment

No current implementation solves the issue. The maintainable path is maintainer-led API, product, and replay-safety design for a native checkpoint/replay/diff contract rather than treating compaction snapshot recovery as deterministic resume.
