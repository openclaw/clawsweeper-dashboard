## Summary

Keep open: current main does not already implement session-scoped exhausted-model cooldown persistence, but this PR still has a real session-store routing bug, lacks required real behavior proof, and overlaps another open implementation branch.

## What This Changes

The PR adds session-scoped persistence for exhausted model cooldowns, a configurable retry window, session-store fields, cleanup/reset handling, reserved session keys, and focused tests.

## Best Possible Solution

Keep one canonical implementation that threads explicit session identity through all fallback cooldown read, write, and clear paths, reconciles the overlapping PR, and includes focused regression coverage plus redacted real runtime proof.

## Reproduction Assessment

Yes for the PR blocker: source inspection of PR head shows sessionAgentId is accepted but not forwarded into cooldown write/clear calls, while current main's default agentDir basename is not the active agent id. I did not run tests because this review is constrained to read-only inspection.

## Solution Assessment

No. The requested feature is plausible, but this implementation should pass the active session agent id through every persisted cooldown access and resolve the overlapping implementation before merge.
