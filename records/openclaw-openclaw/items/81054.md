## Summary

Keep open: current main does not implement session-scoped model-exhaustion persistence, but this PR still has a blocking session-store routing bug and lacks real behavior proof. It also overlaps an open sibling PR and a related open model-cooldown issue that should be coordinated before merge.

## What This Changes

The branch adds session-scoped model exhaustion state, a configurable retry window, session-store cleanup/reset plumbing, reserved session keys, and fallback tests.

## Best Possible Solution

Keep one canonical implementation that threads the active session agent id through all model-fallback read/write/clear paths, resolves the overlapping PR/issue, and includes focused regression coverage plus redacted real runtime proof.

## Reproduction Assessment

Yes, source-reproducible for the review blocker: PR-head cooldown writes omit sessionAgentId while current main's default agentDir ends in `/agent`, so real session-store lookups can target the wrong store. I did not run tests because this review is constrained to read-only inspection.

## Solution Assessment

No. The feature direction is plausible, but this implementation should pass the active session agent id through every fallback call path and health-store helper, coordinate the overlapping implementation, and provide real behavior proof before merge.
