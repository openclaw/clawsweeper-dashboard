## What This Changes

The branch adds session-store model health fields, model-fallback skip/persist/clear logic, maintenance pruning, and tests for same-session exhausted-model cooldowns.

## Best Possible Solution

Keep the PR open for a corrected implementation that uses an intentional session-key resolver boundary, aligns schema/docs/tests with the new config, clears every persisted cooldown shape on reset/model changes/doctor repair, and includes redacted real-run proof.

  - body: The new import pulls `resolveStoredSessionKeyForSessionId` from `../config/sessions.js`, but current main exports that symbol from `src/agents/command/session.ts`, not the config sessions barrel. This will fail build/typecheck before the persistence path can run.
  - confidence: 0.95

  - body: `modelExhaustionRetryMinutes` is added to the defaults type and read from config, but `AgentDefaultsSchema` is strict and does not include the key. Users who configure the advertised TTL would have their config rejected.
  - confidence: 0.91
- **[P2] Clear exhaustedModels on explicit model changes:** `src/sessions/model-overrides.ts:139-142`
  - body: The model-switch cleanup deletes only the scalar `modelHealth*` fields, leaving the new `exhaustedModels` map active. A user-selected model can still be skipped until the old cooldown expires after an explicit model change.
  - confidence: 0.88
- **[P2] Drop exhaustedModels during reply session reset:** `src/auto-reply/reply/agent-runner-session-reset.ts:86-89`
  - body: `nextEntry` spreads the previous session entry and clears only the scalar health fields, so `/reset` carries `exhaustedModels` into the new session id. That contradicts the session reset semantics and can keep skipping models after reset.
  - confidence: 0.87
- **[P3] Clear exhaustedModels in doctor state repair:** `src/commands/doctor-session-state-providers.ts:397-400`
  - body: The doctor repair path now clears scalar model-health fields with other runtime model state, but it leaves the new `exhaustedModels` map behind. `openclaw doctor --fix` can therefore report repaired model state while the fallback loop still skips the stale models.
  - confidence: 0.78
