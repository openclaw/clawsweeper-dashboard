## Summary

Keep this open: current main and the latest release still build Feishu post-mode outbound text as one `md` element after literal `<at user_id="...">name</at>` mention text is prepended. The duplicate issue is already closed in favor of this canonical thread, and an open overlapping PR exists, but that PR is broad and currently conflicting, so the remaining work is still not implemented on main.

## What This Changes

The issue reports that Feishu post-mode outbound at-mentions render as plain text and may also be affected by stale bot-name cache identity collisions.

## Best Possible Solution

Keep this canonical issue open, then either land/extract the focused outbound mention portion from the overlapping Feishu PR or make a narrow Feishu send-path fix that emits native post `at` elements while leaving cache behavior unchanged unless the current collision is proven.

## Reproduction Assessment

Yes from source inspection for the rendering root cause: current main prepends `<at user_id=...>` text and then wraps the full string in a single post `md` element. No live Feishu client run was performed in this read-only review, so this is source-reproducible rather than fully reproduced.

## Solution Assessment

Yes for the rendering layer: native Feishu post `at` elements are the narrow plugin-owner fix for outbound mentions. Unclear for the cache half, because the named cache/spec symbols are absent on current main and should not be changed without finding a current reproducible collision.
