## Summary

This should stay open: current main and the latest release still construct Feishu outbound post messages as one `md` element after inserting `<at user_id="...">name</at>` text, so the reported mention-rendering problem is not implemented away. The data-layer root-cause names in the report do not match current main symbols, so that half needs re-validation before changing cache behavior.

## What This Changes

The issue asks the Feishu plugin to make outbound post-mode @ mentions render as real Feishu mentions and avoid bot-name cache ambiguity.

## Best Possible Solution

Repair Feishu outbound post construction so mention targets serialize as Feishu post `at` elements while preserving markdown/table handling, and validate any bot-name cache claim against current code before changing data paths.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only pass. Source inspection does show the same payload-level path the issue describes: mentions become `<at user_id>` text and are then wrapped in one post `md` element.

## Solution Assessment

Unclear until the Feishu payload contract is rechecked or live-tested. The narrow likely fix is outbound post element splitting, but the reported cache half should not be changed until its current-main equivalent is identified.
