## Summary

Keep open: current main and the latest release cover flat API-key rotation plus per-profile/model cooldowns, but still lack a shared quota-bucket identity for Gemini project or billing-account cooldown. https://github.com/openclaw/openclaw/issues/41135 and https://github.com/openclaw/openclaw/pull/64127 overlap, but neither supersedes this narrower bucket-aware key-pool tracker.

## What This Changes

Request native provider key pools with bucket/project-aware cooldown and backoff on quota-shaped 429s, especially for Gemini multi-project key setups.

## Best Possible Solution

Define an additive provider-owned quota-scope identity on auth candidates, integrate it with existing auth-profile/key rotation and cooldown state, and keep log-watching auth-file rewrites as external workarounds.

## Reproduction Assessment

Yes, at source level: current main exposes flat key rotation and per-profile/model cooldowns but has no shared quota-bucket identity. I did not run a live exhausted Gemini project in this read-only review.

## Solution Assessment

No, the sidecar log-watcher and auth-file rewrite are not the best permanent OpenClaw solution. The maintainable path is a typed provider/auth quota identity integrated with existing cooldown and embedded failover paths, but the public surface needs maintainer direction.
