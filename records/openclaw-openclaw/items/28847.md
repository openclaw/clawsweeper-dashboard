## Summary

Keep open: current main covers generic API-key list collection, caller-scoped key rotation, auth-profile cooldown, and model fallback, but it still lacks a first-class provider key-pool/quota-bucket contract for Gemini project-scoped quotas. The reporter's Keymanager sidecar, the aashari fork comment, Jackten's embedded-runner note, and related routing/quota issues are useful prior art, but none supersede this narrower Gemini bucket-aware request.

## What This Changes

Request provider key pools with bucket/quota-group-aware cooldown and rotation for Gemini project-scoped 429 quota failures.

## Best Possible Solution

Define an additive provider/auth-pool quota-bucket contract, then wire embedded runtime key/profile selection and cooldown state through it with docs and focused regression coverage while preserving current profile fallback defaults.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config design request. Source inspection gives high confidence that current main lacks quota-bucket config and embedded key-list rotation, but no live Gemini 429 run was attempted.

## Solution Assessment

Unclear as submitted. The sidecar demonstrates useful operational behavior, but the maintainable upstream solution should be a typed quota-bucket/auth-pool contract rather than log-tailing auth-file rewrites.
