## Summary

Keep open: current main has per-auth-profile cooldowns, model-scoped cooldowns, profile rotation caps, and Google project-aware OAuth token payloads, but it still lacks shared quota-bucket/project grouping for Gemini key pools.

## What This Changes

The issue requests provider key pools with bucket-aware cooldown/backoff for Gemini quota projects to avoid 429 retry storms.

## Best Possible Solution

Define a generic auth-profile quota-bucket/provider hook contract, then implement Gemini project-aware shared cooldown and rotation with focused docs and regression coverage while preserving current profile defaults.

## Reproduction Assessment

Not applicable for a strict bug reproduction. The report includes plausible Gemini 429 evidence, and current-main source/docs show the remaining gap is a missing bucket-aware capability rather than broken existing documented behavior.

## Solution Assessment

No. The sidecar and current per-profile cooldown behavior are useful workarounds, but the maintainable upstream path needs a generic quota-bucket auth/provider contract before Gemini project grouping is added.
