## Summary

Keep open. Current main has adjacent support for Gemini key lists, rate-limit classification, profile cooldowns, model-scoped cooldowns, and capped same-provider profile rotation, but the central requested behavior is still missing: explicit Gemini quota-bucket/project grouping that cools all sibling keys/profiles in the same Google project after one 429/RESOURCE_EXHAUSTED signal.

## What This Changes

Request provider key pools with 429/RESOURCE_EXHAUSTED cooldown/backoff, Gemini project quota-bucket grouping, minimum rotation spacing, and hot key switching without a gateway restart.

## Best Possible Solution

Add a provider-owned quotaBucket/quotaGroup contract for Gemini API-key profiles or credential metadata, persist bucket-level health alongside per-profile usage state, and update Gemini rate-limit handling so one 429/RESOURCE_EXHAUSTED cools every key/profile in that project bucket before choosing a different bucket.
