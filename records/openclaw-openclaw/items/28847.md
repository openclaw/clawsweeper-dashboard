## Summary

Keep open: current main supports plain provider key-list rotation and auth-profile cooldown/fallback, but it still lacks first-class Gemini project/quota-bucket metadata plus bucket-wide cooldown and selection. Related work does not supersede this: https://github.com/openclaw/openclaw/issues/41135 is broader provider-profile routing, while https://github.com/openclaw/openclaw/issues/80451 was closed into this issue as the canonical quota-bucket tracker.

## What This Changes

Request first-class provider key pools with Gemini project/quota-bucket-aware cooldown and rotation on 429/RESOURCE_EXHAUSTED to avoid retry storms.

## Best Possible Solution

Define an additive provider/auth-pool quota-bucket contract, feed bucket health into key/profile selection and embedded runtime auth preparation, document Gemini project grouping, and preserve existing plain key rotation and auth-profile fallback defaults.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API design request. Source, docs, tests, and related issue state give high confidence that current main lacks quota-bucket metadata and bucket-wide cooldown, but no live Gemini 429 scenario was run.

## Solution Assessment

Unclear as submitted. The sidecar and fork are useful prior art, but the maintainable OpenClaw path should be a typed provider/auth-pool quota-bucket contract rather than log tailing and rewriting auth files.
