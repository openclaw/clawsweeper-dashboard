## Summary

Keep open: current main supports plain provider key lists, one-operation key rotation, and auth-profile cooldowns, but it still lacks quota-bucket or Gemini-project metadata and bucket-wide cooldown/selection. A related Gemini billing-account report was already closed into this issue as the canonical tracker, so this remains the right place for maintainer product/API scoping.

## What This Changes

The issue requests first-class provider key pools with bucket-aware cooldown/backoff for Gemini project-scoped 429 and RESOURCE_EXHAUSTED failures.

## Best Possible Solution

Define an additive provider/auth quota-bucket contract, feed bucket health into key/profile selection and embedded runtime auth preparation, document Gemini project grouping, and preserve existing plain key rotation and auth-profile fallback defaults.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API design request. Source, docs, tests, and related issue state give high confidence that current main lacks quota-bucket metadata and bucket-wide cooldown, but no live Gemini 429 scenario was run.

## Solution Assessment

Unclear as submitted. The sidecar and public fork are useful prior art, but the maintainable OpenClaw path should be a typed provider/auth-pool quota-bucket contract rather than log tailing and rewriting auth files.
