## Summary

Keep open: current main has multi-key collection, explicit key-rotation helpers, auth-profile cooldowns, and documented model/profile failover, but it still lacks a quota-bucket contract for shared Gemini project limits and the embedded runner still appears to rotate auth profiles rather than provider key-list entries.

## What This Changes

Request first-class provider key pools with rate-limit cooldown/backoff, Gemini project bucket grouping, and no-restart key rotation to avoid retry storms.

## Best Possible Solution

Define a generic quota-bucket auth/profile contract, then wire Gemini project-aware shared cooldown and any approved embedded LLM key-list retry through prepared runtime auth with focused docs and regression coverage.

## Reproduction Assessment

No high-confidence live reproduction was established for a strict bug. Source inspection does show the current-main gap: no quota-bucket schema exists, and embedded LLM auth does not use the provider key-list rotation helper.

## Solution Assessment

Unclear as submitted. The sidecar and comments prove the operational need, but the maintainable upstream path needs a generic quota-bucket/provider-auth contract plus a separately scoped embedded key-list retry decision.
