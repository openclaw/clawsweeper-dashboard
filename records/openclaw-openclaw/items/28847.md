## Summary

Keep open: current main supports plain multi-key retry and auth-profile/model cooldowns, but it still lacks a first-class quota-bucket or Gemini project grouping contract for bucket-wide cooldown and selection.

## What This Changes

Request adds provider key pools with bucket/quota-group metadata and bucket-wide cooldown/backoff for Gemini project-level 429s without gateway restarts.

## Best Possible Solution

Define an additive provider/auth quota-bucket contract, feed bucket health into key/profile selection and embedded runtime auth preparation, document Gemini project grouping, and preserve existing plain key rotation and auth-profile fallback defaults.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API design request. Source, docs, tests, and related issue state give high confidence that current main lacks quota-bucket metadata and bucket-wide cooldown, but no live Gemini 429 scenario was run.

## Solution Assessment

Unclear as submitted. The sidecar and fork are useful prior art, but the maintainable OpenClaw path should be a typed provider/auth-pool quota-bucket contract rather than log tailing and rewriting auth files.
