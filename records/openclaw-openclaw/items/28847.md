## Summary

Keep open. Current main has generic API-key collection, rate-limit key rotation helpers, auth-profile cooldowns, and docs, but it still does not provide Gemini project/bucket metadata or a bucket-aware routing contract, and the embedded runner still advances auth profiles rather than provider key-list entries for the same profile.

## What This Changes

Request first-class provider key pools with rate-limit cooldown/backoff, Gemini quota-bucket grouping, and hot key rotation to avoid 429 retry storms.

## Best Possible Solution

Keep this open, split the documented embedded-runner key-list rotation gap into a narrow bug fix if desired, and route Gemini project bucket semantics through the broader provider-pool design in https://github.com/openclaw/openclaw/issues/41135.

## Reproduction Assessment

No high-confidence live reproduction for the full Gemini project-bucket 429 scenario. Source inspection does show current main lacks bucket/quota-group config and that embedded runs advance auth-profile candidates rather than provider key-list candidates.

## Solution Assessment

Unclear as requested. The maintainable path should separate a narrow documented API-key rotation bug from a product-reviewed provider-pool and bucket-routing contract.
