## Summary

Keep open: current main supports generic key-list rotation and auth-profile cooldown/failover, but it still lacks the requested quota-bucket metadata and bucket-wide cooldown/rotation contract for Gemini project-scoped 429s. The related open routing and rate-limit trackers overlap, but none fully owns this narrower Gemini key-pool bucket behavior yet.

## What This Changes

The issue requests provider key pools with quota-bucket metadata and bucket-wide cooldown/backoff for Gemini project-scoped 429 handling.

## Best Possible Solution

Define an additive provider/auth-pool quota-bucket contract, route embedded/runtime key selection through bucket-aware health state, document Gemini project grouping, and preserve current profile fallback defaults.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API design request. Source and docs inspection gives high confidence that current main lacks bucket/quota-group metadata, but I did not run a live Gemini 429 scenario.

## Solution Assessment

Unclear as submitted. The sidecar demonstrates useful behavior, but the maintainable OpenClaw path should be a typed provider/auth-pool bucket contract rather than log-tailing and rewriting auth files.
