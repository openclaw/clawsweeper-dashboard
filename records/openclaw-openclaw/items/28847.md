## Summary

Keep open: current main covers generic multi-key collection, rate-limit-shaped key rotation helpers, and auth-profile cooldown/failover, but it still lacks Gemini project/quota-bucket grouping and a first-class embedded-runner key-pool contract. The reporter's Keymanager sidecar and the aashari fork are useful prior art, while the related open items cover broader routing, Google 429 classification, or Z.AI-specific rotation rather than a merged replacement for this request.

## What This Changes

Add provider key pools with bucket-aware cooldown/backoff for Gemini project-scoped quotas and hot key/profile rotation on 429s.

## Best Possible Solution

Define an additive provider-pool/quota-bucket contract, then wire embedded runtime key/profile selection and cooldown state through that contract with docs and focused regression coverage while preserving current profile-fallback defaults.
