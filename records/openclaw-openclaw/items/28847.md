## Summary

Keep open: current `main` has auth-profile rotation, per-profile cooldowns, model-scoped cooldowns, and fallback throttling, but it still lacks the requested shared quota-bucket/project grouping for Gemini key pools.

## What This Changes

Request first-class provider key pools with Gemini project/quota-bucket-aware cooldown and rotation on 429 quota signals.

## Best Possible Solution

Define a generic auth-profile quota-bucket/provider hook contract, then implement Gemini project-aware shared cooldown and rotation with focused docs and regression coverage while preserving current defaults.

## Reproduction Assessment

Not applicable for a strict bug reproduction. The report includes plausible Gemini 429 evidence, and current-main source/docs show the remaining request is a missing bucket-aware capability rather than broken existing behavior.

## Solution Assessment

No. The sidecar and current per-profile cooldown behavior are useful workarounds, but the maintainable upstream path is a generic quota-bucket auth/provider contract before Gemini project grouping is implemented.
