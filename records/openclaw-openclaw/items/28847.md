## Summary

Keep open: current main supports generic multi-key collection, local key rotation, auth-profile cooldowns, and model/profile fallback, but it still has no quota-bucket identity, bucket-wide cooldown, or Gemini project-aware key selection. Related work is useful prior art, but no merged/open viable replacement currently owns the exact bucket-aware provider key-pool behavior.

## What This Changes

The issue requests provider key pools with bucket-aware cooldown/backoff for Gemini project-level 429s so OpenClaw avoids retrying keys or sibling models that share an exhausted quota bucket.

## Best Possible Solution

Define one additive provider/auth quota-bucket contract, feed bucket health into key/profile selection and embedded runtime auth preparation, document Gemini project grouping, and preserve existing plain key rotation plus auth-profile fallback defaults.

## Reproduction Assessment

Yes at source level: current main has plain key/profile cooldown and rotation paths but no quota-bucket identity or bucket-wide cooldown decision. No fresh live Gemini project-quota 429 run was executed in this read-only review.

## Solution Assessment

Unclear as submitted: the sidecar and fork are useful prior art, but the maintainable OpenClaw path should be a typed provider/auth quota-bucket contract rather than log tailing or rewriting auth files.
