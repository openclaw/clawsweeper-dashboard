## Summary

Keep open: current main covers parts of the request with multi-key env collection, rate-limit key-rotation helpers, auth-profile rotation, and profile cooldowns, but source inspection still shows no Gemini/project quota-bucket contract and no embedded-runner use of the provider key-list rotation helper. The recent comment identifying the narrower embedded-runner gap is consistent with the current code, while the broader pool-routing work overlaps but is not fully replaced by https://github.com/openclaw/openclaw/issues/41135.

## What This Changes

Request first-class provider API key pools with rate-limit cooldown/backoff and Gemini project bucket-aware rotation without requiring a gateway restart.

## Best Possible Solution

Keep this as an open core auth/provider design item until maintainers define a generic quota-bucket contract, then wire Gemini project-aware shared cooldowns and any accepted embedded key-list rotation through the runtime auth path with docs and regression coverage.

## Reproduction Assessment

Not applicable for the primary request because this is a feature/config design item, not a regression with a pre-existing bucket contract. Source inspection does give high confidence that current main lacks quota-bucket state and embedded key-list rotation in the agent runner.

## Solution Assessment

Unclear as submitted. The sidecar demonstrates a useful operational direction, but the maintainable upstream solution should be a generic auth/profile quota-bucket contract plus a separately accepted embedded-runner key-list path, not a log-tailing rewrite of auth files.
