## Summary

Keep open: current main has plain multi-key retry and profile/model cooldowns, but no first-class quota-bucket or Gemini project grouping contract; the related open PRs are partial and this issue remains the canonical bucket-aware design tracker.

## What This Changes

The issue requests provider key pools with quota-bucket-aware cooldown/backoff for Gemini project-scoped 429s, plus hot key rotation without gateway restart.

## Best Possible Solution

Define an additive provider/auth quota-bucket contract, feed bucket health into key/profile selection and embedded runtime auth preparation, document Gemini project grouping, and preserve existing plain key rotation and auth-profile fallback defaults.

## Reproduction Assessment

Not applicable as a strict bug reproduction; this is a feature/API design request. Source, docs, tests, and related issue state give high confidence that current main lacks quota-bucket metadata and bucket-wide cooldown, but no live Gemini 429 scenario was run.

## Solution Assessment

Unclear as submitted. The sidecar and fork are useful prior art, but the maintainable OpenClaw path should be a typed provider/auth-pool quota-bucket contract rather than log tailing and rewriting auth files.
