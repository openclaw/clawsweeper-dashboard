## Summary

Keep open: current main and v2026.6.10 support flat API-key rotation, auth-profile rotation, and model/profile cooldowns, but still have no quota-bucket or Gemini project identity for cooling down a whole project and rotating away from that bucket. The sidecar and fork discussed in the issue are useful workaround evidence, and the related provider-pool/circuit-breaker items overlap, but none fully supersedes this request.

## What This Changes

The issue requests provider key pools with bucket-aware cooldown/backoff for Gemini-style project-scoped 429 and RESOURCE_EXHAUSTED failures.

## Best Possible Solution

Define an additive provider-owned quota-scope identity on auth/key candidates, integrate it with existing cooldown/fallback state, and reconcile the API shape with the broader provider-profile routing tracker before implementation.

## Reproduction Assessment

Yes, at source level: current main exposes flat key rotation and per-profile/model cooldowns but has no shared quota-bucket identity. I did not run a live exhausted Gemini project in this read-only review.

## Solution Assessment

Partly: the requested native bucket-aware behavior targets the right product gap, but the sidecar log-watcher/auth-file rewrite should remain a workaround rather than the core design. The best fix needs a typed provider/auth quota identity and maintainer-approved config or plugin API shape.
