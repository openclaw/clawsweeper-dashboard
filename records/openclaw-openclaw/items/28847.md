## Summary

Keep open: current main covers generic key-list collection, rate-limit-shaped key rotation helpers, and auth-profile cooldown/failover, but it still lacks Gemini project/quota-bucket grouping and a first-class embedded-runner key-pool contract. The reporter's Keymanager sidecar, the aashari fork comment, and Jackten's embedded-runner analysis are useful prior art; the related broader routing issue and Z.AI PR do not supersede this request.

## What This Changes

Request first-class provider key pools with Gemini project/quota-bucket-aware cooldown and backoff to avoid 429 retry storms.

## Best Possible Solution

Define an additive provider-pool/quota-bucket contract, then wire embedded runtime key/profile selection and cooldown state through that contract with docs and focused regression coverage while preserving current profile-fallback defaults.

Not applicable as a bug reproduction; this is a feature/config design request. Source inspection gives high confidence current main lacks quota-bucket config and embedded key-list rotation, but no live Gemini 429 repro was run.
