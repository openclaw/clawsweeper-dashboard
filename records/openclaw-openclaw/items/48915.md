## Summary

Current `main` and the latest release still leave the reported snake_case Anthropic tool-call repair gap unresolved, and open PR #50802 already tracks the focused fix with closing syntax.

## What This Changes

The issue asks OpenClaw to recognize snake_case tool-call block types such as `tool_use`, `tool_call`, and `function_call` when repairing corrupted session history.

## Best Possible Solution

Land or replace PR #50802 with a focused fix that keeps session repair, replay normalization, and tool-call extraction on a consistent tool-call type set, with regression coverage for malformed snake_case blocks.

## Reproduction Assessment

Yes. A source-level reproduction is an assistant content block like `{ type: "tool_use", id: "call_1", name: "read" }` without `input`; current `isRawToolCallBlock` does not recognize it, so the malformed block bypasses the guarded repair branch.

## Solution Assessment

Yes. Extending the shared tool-call recognition path to include snake_case variants is the narrowest maintainable fix; the safer version keeps all duplicate recognizers in agreement.
