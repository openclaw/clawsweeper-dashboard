## Solution Assessment

Yes, the snapshot belongs at SDK/session ingress because it prevents repeated unsafe descriptor reads while leaving provider-side normalization and extension policy untouched. The only unresolved question is whether maintainers accept snapshot semantics for SDK callers that mutate descriptors after session creation.
