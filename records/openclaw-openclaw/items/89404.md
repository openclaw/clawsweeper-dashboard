## Summary

## What This Changes

The PR snapshots SDK custom tool definitions before session startup, drops unreadable descriptors, and preserves callable receivers for wrapped custom tool hooks.

## Best Possible Solution

Land the ingress snapshot approach only after maintainers explicitly accept process-stable SDK custom tool descriptor semantics and normal draft/check gates are ready.

## Reproduction Assessment

Yes. Source inspection shows current main maps options.customTools through tool.name and later reads descriptor metadata during registry refresh, so hostile getters can throw on session setup.

## Solution Assessment

Yes, if maintainers accept the SDK compatibility choice. Snapshotting at SDK/session ingress prevents repeated unsafe descriptor reads and preserves callback receivers; the safer alternative for live-descriptor compatibility is guarded reads on each refresh.
