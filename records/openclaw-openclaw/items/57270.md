## Summary

Keep open. Current main still has the reported docs/schema mismatch, and open PR #57293 already targets the exact fix with closing syntax, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks for the documented global `tools.exec.approvalRunningNoticeMs` config key to be accepted by the schema or removed from the docs.

## Best Possible Solution

Land or supersede #57293 with a narrow schema/base-schema regeneration and regression coverage while preserving strict rejection of unrelated unknown `tools.exec` keys.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: global `ToolsSchema` routes `tools.exec` through strict `ToolExecSchema`, whose base shape lacks the documented key, so the reported config is rejected before runtime use.

## Solution Assessment

Yes. Moving the field into `ToolExecBaseShape` is the narrowest maintainable fix because runtime already reads the global value and the per-agent schema already accepts the same nonnegative integer.
