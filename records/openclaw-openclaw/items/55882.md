## Summary

Keep open. Current main and v2026.5.20 now cover adjacent heartbeat active-run preemption and pending-final replay cases, but source still only captures final payloads after they exist and explicit interrupt-mode aborts can end active work without a durable promised-deliverable record.

## What This Changes

The issue asks OpenClaw to preserve promised-but-unfinished agent outputs across task-switch interruptions with durable pending-deliverable or auto-resume semantics.

## Best Possible Solution

Define a typed session-continuity contract that records promised deliverables before preemptive switches and resolves them only by send, explicit deferral, cancellation, or supersession, while reusing the shipped heartbeat deferral and final-delivery recovery safeguards.

## Reproduction Assessment

## Solution Assessment

Unclear until maintainers define the product contract. The shipped heartbeat active-run guard is the right narrow fix for one related race, but it is not a complete durable promised-deliverable or auto-resume solution.
