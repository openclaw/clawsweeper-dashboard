## Summary

Keep open. Current main has added adjacent heartbeat and pending-final protections, but it still does not persist a promised unfinished deliverable before interruption or define how system/heartbeat/task switches resume, defer, cancel, or clear that work.

## What This Changes

The issue asks OpenClaw to preserve and resume promised user-visible deliverables across task switching or interruption without requiring a gateway restart.

## Best Possible Solution

Define a typed session-continuity contract that records promised deliverables before preemptive switches and resolves them only through send, explicit deferral, cancellation, or supersession while reusing existing heartbeat, commitment, and pending-final safeguards.

## Reproduction Assessment

No high-confidence live reproduction was established on current main. Source inspection still shows the central missing contract: pending-final capture starts after final payloads exist, while some interruption paths can abort or supersede active work without a durable promised-deliverable record.

## Solution Assessment

Unclear until maintainers define the product contract. A durable pending-deliverables stack may be the right direction, but the safer implementation needs explicit cancellation, supersession, replay, duplicate-prevention, and configured-private-final semantics first.
