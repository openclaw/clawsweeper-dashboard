## Summary

Keep open. Current main has narrower durable-final-delivery, heartbeat replay, and restart-recovery pieces, but it still does not record or resume a promised-but-unfinished deliverable before a non-restart interrupt aborts the active run.

## What This Changes

The issue requests durable session continuity so promised agent deliverables are not silently lost when higher-priority tasks interrupt an in-flight run.

## Best Possible Solution

Define a typed session-continuity contract that records pending deliverables before preemptive switches and resolves them only by send, explicit deferral, cancellation, or supersession, with duplicate-delivery safeguards and live channel proof.

## Reproduction Assessment

No, we do not have a high-confidence live reproduction against current main. Source inspection still shows a credible path: interrupt mode can abort active work, while durable pending-final capture only happens after final payloads exist and replay is heartbeat/restart scoped.

## Solution Assessment

Unclear until maintainers define the product contract. A durable pending-deliverables direction is plausible, but the safe implementation needs explicit promise detection, resume/defer/cancel semantics, and duplicate-delivery safeguards.
