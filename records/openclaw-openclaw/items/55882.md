## Summary

Keep open. Current main has adjacent heartbeat deferral, pending-final replay, and inferred-commitment safeguards, but it still lacks a durable pre-interrupt pending-deliverables contract for promised unfinished work, and the latest discussion adds release-level external-channel evidence.

## What This Changes

The issue asks OpenClaw to preserve promised user-facing deliverables across task switches and interruptions with a durable pending-deliverables or task-stack mechanism.

## Best Possible Solution

Define a typed session-continuity contract that records promised deliverables before preemptive switches and resolves them only by visible send, explicit deferral, cancellation, or supersession.

## Reproduction Assessment

No high-confidence live end-to-end reproduction was established against current main. Source inspection still shows a credible path where interrupt-mode aborts active work before any promised deliverable is persisted, while recent 2026.5.20 discussion reports a related external-chat delivery miss.

## Solution Assessment

Unclear until maintainers define the product contract. A durable pending-deliverables layer is directionally plausible, but the permanent fix needs explicit cancellation, supersession, replay, and duplicate-prevention semantics.
