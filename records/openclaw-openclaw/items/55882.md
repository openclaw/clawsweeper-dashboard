## Summary

Keep open. Current main has adjacent heartbeat deferral, pending-final replay, and inferred-commitment safeguards, but it still does not persist an unfinished promised deliverable before interrupt-mode aborts or define the user-turn replay and auto-resume contract needed for this report.

## What This Changes

The issue asks OpenClaw to preserve and resume promised in-flight deliverables when heartbeat, system, or operational turns interrupt a long-running user task.

## Best Possible Solution

Define a typed session-continuity contract that records promised deliverables before preemptive switches and resolves them only by send, explicit deferral, cancellation, or supersession while reusing the shipped heartbeat and pending-final safeguards.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection shows a credible current-main path where interrupt mode can abort active work while durable pending-final capture only starts after final payloads already exist.

## Solution Assessment

Unclear until maintainers define the product contract. A durable pending-deliverables stack may be the right direction, but it needs explicit cancellation, supersession, replay, and duplicate-prevention semantics before implementation.
