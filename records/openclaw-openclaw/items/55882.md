## Summary

Keep this open. Current main has narrower pending-final-delivery and heartbeat fixes, but it still lacks a durable per-session record for arbitrary promised-but-unfinished outputs before a non-restart interrupt or task switch.

## What This Changes

The issue asks OpenClaw to durably remember and resume promised agent deliverables when a session is interrupted or task-switched, including Telegram/autonomy-guard cases without a gateway restart.

## Best Possible Solution

Define a typed session-continuity contract that records pending deliverables before preemptive switches and clears them only when sent, explicitly deferred, cancelled, or superseded, with regression coverage for non-restart interruption.

## Reproduction Assessment

No live high-confidence Telegram/autonomy-guard reproduction was run. Source inspection gives a credible current-main path: interrupt mode aborts active work, while durable pending delivery is only written after final payloads exist and only heartbeat replay consumes it automatically.

## Solution Assessment

Unclear until maintainers define the product contract. The durable pending-deliverables direction is maintainable, but stretching heartbeat replay, opt-in commitments, or in-process follow-up queues would not solve the general promised-output case cleanly.
