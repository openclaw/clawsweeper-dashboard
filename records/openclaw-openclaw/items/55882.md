## Summary

Keep open. Current main has adjacent safeguards for inbound message queues, restart recovery, outbound delivery retries, task tracking, and optional inferred commitments, but none of them provide a durable per-session queue for arbitrary unfinished promised outputs after non-restart task switching.

## What This Changes

The issue requests durable per-session tracking and automatic resumption or deferral of promised chat deliverables when an agent is interrupted by higher-priority work without a gateway restart.

## Best Possible Solution

Add a core session-continuity mechanism that persists typed pending deliverables before preemptive task switches and clears them only when the promised output is sent, explicitly deferred, cancelled, or superseded. The implementation should include regression coverage for a long-running direct chat task interrupted by unrelated high-priority work without a gateway restart.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Unclear for the exact live Telegram/autonomy-guard sequence because this was read-only, but the code supports a focused regression shape: start a long-running direct chat turn, inject unrelated active-session work, and verify no durable pending-deliverable record is created.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose the product contract, but the narrowest maintainable direction is a typed core pending-deliverable record integrated with session/turn continuity rather than relying on optional inferred commitments or the inbound follow-up queue alone.
