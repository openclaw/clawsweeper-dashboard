## Summary

Keep open. Current main still records durable pending-final text only after final payloads exist and replays it only through heartbeat, so the broader non-restart promised-output/task-switching case is not solved; related issues are adjacent rather than canonical replacements.

## What This Changes

The issue requests durable tracking and automatic resolution of promised in-flight deliverables when an agent is interrupted by another task before the final output is sent.

## Best Possible Solution

Define a typed session-continuity contract that records pending deliverables before preemptive switches and resolves them only by send, explicit deferral, cancellation, or supersession, with focused regression and live channel proof.

## Reproduction Assessment

No, we do not have a high-confidence live reproduction against current main. Source inspection shows a credible path: interrupt mode can abort active work, while durable pending delivery is only recorded after final payloads exist and replay is heartbeat-scoped.

## Solution Assessment

Unclear until maintainers define the product contract. A durable pending-deliverables direction is plausible, but the safe implementation needs explicit promise detection, resume/defer/cancel semantics, and duplicate-delivery safeguards.
