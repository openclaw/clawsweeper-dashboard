## What This Changes

The issue requests a durable session-continuity or pending-deliverables mechanism so interrupted long-running chat tasks resume or explicitly clear promised outputs instead of silently dropping them.

## Best Possible Solution

Define a typed session-continuity contract that records promised deliverables before preemptive switches and resolves them only by send, explicit deferral, cancellation, or supersession while reusing the shipped heartbeat, commitment, and pending-final safeguards.
