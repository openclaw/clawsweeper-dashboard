## Summary

Keep open. Current main has important adjacent safeguards, but the remaining report is broader: unfinished promised work can still be aborted or superseded before a final payload exists, and external-chat visible-delivery semantics still need a maintainer product decision.

## What This Changes

The issue asks OpenClaw to preserve and resume promised user-facing deliverables across task switching, heartbeat/system interruptions, and external-chat delivery paths.

## Best Possible Solution

Define a typed session-continuity contract that records promised deliverables before preemptive switches and resolves them only by send, explicit deferral, cancellation, or supersession, while coordinating with existing pending-final and message-tool visibility policy.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only review. Source inspection shows a credible current-main path where interrupt mode can abort active work before pending-final capture exists, and the latest issue comment adds production evidence from v2026.5.20.

## Solution Assessment

Unclear until maintainers define the product contract. A durable pending-deliverables stack may be the right direction, but it needs explicit semantics for cancellation, supersession, replay, duplicate prevention, and message-tool-only visible delivery.
