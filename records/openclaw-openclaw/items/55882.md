Keep open. Current main has several adjacent fixes, but it still does not provide a durable pending-deliverable or task-continuation contract for promised user-visible outputs after non-restart interruptions.

## What This Changes

The issue requests durable handling for promised agent deliverables so task switches, heartbeat/system events, or operational interruptions do not silently drop final user-visible output.

## Best Possible Solution

Define a maintainer-approved session-continuity and visible-delivery contract that composes with active reply-run guards, pending final delivery, commitments, Task Flow, and channel delivery receipts, then split the implementation into narrow PRs.
