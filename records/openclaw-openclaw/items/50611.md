## Summary

Keep open. Current main and v2026.5.18 still collapse the memory-flush/preflight token gate to `null` when the reserve plus soft threshold consumes the context window, and an open closing-linked PR remains the implementation candidate.

## What This Changes

The issue reports that `reserveTokensFloor` equal to the model context window prevents memory flush or compaction from triggering.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/50694 with one shared positive-threshold calculation used by both preflight compaction and runtime memory flush, plus regression coverage for reserve values equal to or above the context window.

## Reproduction Assessment

Yes. Source inspection on current main gives a direct path: `contextWindowTokens=200000`, `reserveTokensFloor=200000`, and any nonnegative soft threshold make the shared gate return `null` instead of flushing or preflight compacting.

## Solution Assessment

Yes for keeping this issue open while the closing-linked PR remains unmerged. The narrow maintainable fix is a shared context-window-aware positive-threshold helper used by both preflight compaction and runtime memory flush.
