## Summary

Keep open. Current main and the latest release still disable token-based memory flush and preflight compaction when the reserve plus soft threshold consumes the whole context window, and https://github.com/openclaw/openclaw/pull/50694 remains the open implementation candidate with closing linkage.

## What This Changes

Report that memory flush and preflight compaction never trigger when `reserveTokensFloor` is configured equal to the context window.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/50694 or a replacement that centralizes a positive context-window-aware threshold for both preflight compaction and memory flush, with regression coverage for equal and greater-than-context reserve values.

## Reproduction Assessment

Yes, by source inspection on current main. With `contextWindowTokens` and `reserveTokensFloor` both `200000`, the shared gate computes a non-positive threshold and returns `null`, so token-based memory flush and preflight compaction return false.

## Solution Assessment

Yes for keeping this issue open while the linked PR is reviewed. The maintainable fix is shared threshold normalization for both runtime paths, not a separate duplicate repair job.
