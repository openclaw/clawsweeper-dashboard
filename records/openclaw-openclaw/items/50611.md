## Summary

Keep open. Current main and the latest release still disable the token-based memory-flush and preflight-compaction gates when the reserve and soft threshold consume the context window, and the closing-linked implementation PR remains open.

## What This Changes

The issue asks OpenClaw to prevent memory flush and preflight compaction from being disabled when `reserveTokensFloor` is equal to the model context window.

## Best Possible Solution

Review, refresh, and land https://github.com/openclaw/openclaw/pull/50694 or an equivalent shared positive-threshold fix, with regression coverage for reserve values equal to or above the context window.

## Reproduction Assessment

Yes. Source inspection on current main gives a direct path: `contextWindowTokens=200000`, `reserveTokensFloor=200000`, and any nonnegative soft threshold make the shared gate return `null` instead of flushing or preflight compacting.

## Solution Assessment

Yes for keeping this issue open while the closing-linked PR remains unmerged. The narrow maintainable fix is a shared context-window-aware positive-threshold helper used by both preflight compaction and runtime memory flush.
