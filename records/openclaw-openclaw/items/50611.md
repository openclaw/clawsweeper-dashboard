## Summary

Keep open. Current main and the latest release still let a non-positive token threshold disable memory flush/preflight compaction when `reserveTokensFloor` is equal to or above the context window, and the target-repo fix attempts are closed unmerged.

## What This Changes

The issue reports that memory flush does not trigger when `reserveTokensFloor` equals the model context window.

## Best Possible Solution

Keep this issue open until maintainers choose the reserve-floor policy and land a focused fix that guarantees a positive effective memory-flush and preflight-compaction threshold, with regression coverage for equal and greater-than-context reserve values.

## Reproduction Assessment

Yes, source-level: with `contextWindowTokens=200000`, `reserveTokensFloor=200000`, and the default memory flush soft threshold, current main computes a non-positive threshold and `shouldRunMemoryFlush` returns false before token pressure can trigger a flush.

## Solution Assessment

Unclear. The report's 25% default is one plausible policy, but the maintainable fix should normalize the effective reserve below the context window or enforce a positive threshold after maintainers choose the compatibility behavior.
