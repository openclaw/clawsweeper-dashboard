## What This Changes

## Best Possible Solution

Keep the focused hardening, but route all dreaming markdown replacements through the existing safe atomic helper with mode preservation, only prune refs after confirmed missing regular files, and require redacted real-run proof before merge.

No. Atomic writes and dangling-ref cleanup are plausible pieces of the fix, but the best implementation should reuse the existing mode-preserving atomic helper and the existing regular-file/ENOENT source-check semantics instead of adding a broader deletion path.
