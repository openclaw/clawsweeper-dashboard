## Summary

Keep open. Current main and the latest release still reject `#` in exec SecretRef ids before provider execution, and the linked open fix PR references this issue with closing syntax, so the issue should remain open until that PR is merged or closed.

## What This Changes

Allow exec SecretRef ids such as `secret#json_key` so AWS Secrets Manager JSON-key selectors validate and gateway startup can reach the exec resolver.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/80731; after it merges, close this issue with the merged commit and release provenance.

## Solution Assessment
