## Summary

Keep open: the PR addresses a real current-main doctor-warning gap, but the latest head still needs a small repair to avoid over-warning on empty/defaults-only channel config and to keep fallback wording precise.

## What This Changes

The PR adds a doctor preview warning, regression tests, and a changelog entry for configured group/channel chats that still default to private `message_tool` visible replies when the message tool is available.

## Best Possible Solution

Land the doctor advisory with focused tests after tightening it to real configured channels and keeping the runtime delivery policy unchanged.

## Reproduction Assessment

Yes, by source inspection. Current main resolves configured group/channel turns with the message tool available as `message_tool_only`, while the doctor warning helper returns early with no warning for that same available-message-tool case.

## Solution Assessment

Yes, with small adjustments. Moving the default-private advisory before the unavailable-target early return is the narrow maintainable fix, but the predicate and fallback-sensitive wording should be tightened before merge.
