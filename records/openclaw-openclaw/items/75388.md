## Summary

Keep open: the latest head fixes the simple current-main doctor warning gap and the prior fallback wording issue, but the new advisory can still be suppressed by an unrelated or mixed unavailable-agent policy.

## What This Changes

The PR adds a doctor warning, focused tests, and a changelog entry for configured group/channel chats that still default to private message-tool visible replies when the message tool is available.

## Best Possible Solution

Keep the doctor-only advisory, but compute it against routed channel targets and their effective message-tool availability so message-capable routes get the private-default hint while fallback routes keep the unavailable-message warning.

## Reproduction Assessment

Yes, by source inspection. Current main returns no doctor warning when the message tool is available, while the runtime resolver still defaults group/channel turns to message_tool_only unless visibleReplies is automatic.

## Solution Assessment

Yes for the simple configured-channel case, with one narrow correction still needed. The new advisory should be route-scoped so unrelated or mixed unavailable agents do not suppress the private-default warning for routed message-capable channels.
