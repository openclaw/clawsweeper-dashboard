## Summary

Keep open: current main still has the missing doctor advisory, and this PR is the right narrow direction, but the latest head still has two small warning-logic issues that should be repaired before merge.

## What This Changes

The PR updates doctor visible-reply warnings, focused tests, and the changelog so configured group/channel chats can warn when they still default to private `message_tool` visible replies.

## Best Possible Solution

Land the doctor advisory after matching the real configured-channel predicate and avoiding fallback-contradictory wording, while keeping the runtime delivery policy unchanged.

## Reproduction Assessment

Yes, by source inspection. On current main, a configured channel with the messaging tool available resolves channel turns to `message_tool_only`, while `collectVisibleReplyToolPolicyWarnings` returns no warning because no message-tool target is unavailable.

## Solution Assessment

Yes, with small changes. Moving the advisory before the unavailable-target early return is the narrow maintainable fix, but it should use real-channel detection and avoid claiming private replies when runtime will fall back to automatic delivery.
