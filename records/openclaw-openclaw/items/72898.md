## Summary

Keep open. Current main still treats the reported `imsg` `chat.db` permission denial as stderr noise or a non-fatal probe error, while this PR plugs the error into the existing fatal-probe path and matches the upstream `imsg` error shape. I found no blocking correctness or security issue in the diff.

## What This Changes

The PR adds iMessage RPC `chat.db` permission-denied detection in the client/probe path, fails pending RPC calls, terminates the `imsg rpc` child on fatal startup permission errors, and adds a probe regression test.

## Best Possible Solution

Review and land this narrow iMessage plugin fix, while keeping the broader process-context / Full Disk Access architecture request tracked separately in https://github.com/openclaw/openclaw/issues/43440.

## Solution Assessment

Yes. This is the narrowest maintainable fix for the observable retry/diagnostic bug because it stays inside the iMessage plugin client/probe boundary and uses the existing monitor fatal-probe behavior rather than adding iMessage-specific policy to core.
