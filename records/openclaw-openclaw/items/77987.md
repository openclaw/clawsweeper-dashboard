  - body: When the cap is full of active lockouts, this insert path creates the fresh entry, runs eviction, and can delete that fresh entry because locked entries are skipped. Preserve a fail-closed overflow state or use the replacement branch’s shape.

  - body: `maxEntries` is now an exported limiter option, but this line accepts the raw value. Passing `0`, a negative number, or `NaN` can evict the just-created entry and prevent the limiter from remembering failed auth attempts for that configuration.
