## Risks / Open Questions

- `maxEntries` is consumed raw, so `0`, negative, fractional, or non-finite SDK inputs can undermine or disable retention instead of clamping to a positive integer.
- The new 10,000-entry default changes exported limiter behavior under high-cardinality failed-auth traffic by evicting oldest non-locked entries instead of retaining every failed-attempt key until prune.
- Active lockouts are intentionally skipped during eviction, so the nominal cap is not an absolute memory ceiling when locked entries alone exceed it.
- If operators should tune the cap, the public `gateway.auth.rateLimit` type, strict schema, and docs need to expose and validate the field before merge.
