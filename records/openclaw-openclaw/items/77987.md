## What This Changes

## Best Possible Solution

Land the insert-time cap after maintainers explicitly choose the API boundary: keep it SDK/internal-only with the default tradeoff, or wire `maxEntries` through public gateway config schema/docs and validation.

## Reproduction Assessment

## Solution Assessment

Mostly yes. The gateway-owned insert-time cap is the narrow fix for the reported map growth, but the PR should merge only after the SDK-only versus public gateway config boundary is explicit.
