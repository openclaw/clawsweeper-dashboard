## Label Justifications

- `P1`: The report describes a real Discord group workflow becoming uncontrollable because old queued messages are processed as fresh agent turns after long delays.
- `impact:message-loss`: The failure mode is channel delivery/order drift where stale Discord messages reach the agent pipeline as current turns while current steering is delayed.

- `maturity:stable`: The affected Discord channel surface is listed as M4 Stable, including Conversation Routing and Delivery, in the checked-in maturity taxonomy.

## Summary

Keep open: current main still accepts and serializes Discord inbound jobs without a source-age guard or stale marker before normal agent dispatch, and the remaining fix needs maintainer policy for thresholds, direct/control exceptions, coalescing, and /stop queue semantics.

## What This Changes

The issue reports Discord group-channel backlog events being processed as fresh agent turns after a long queue delay.

## Best Possible Solution

Define a Discord/group-channel freshness policy that carries source, receive, enqueue, append, and dispatch age; stale-marks or coalesces low-value room events before normal turns; and gives /stop explicit semantics for pending Discord jobs.

## Reproduction Assessment

Yes at source level, but not with a live Discord replay in this pass. Discord source timestamps are preserved, accepted jobs serialize by session/channel key, and current code lacks a source-age guard before normal dispatch.

## Solution Assessment

No final fix shape is confirmed yet. A blind Discord-only threshold is only a plausible mitigation; the best fix needs maintainer-chosen freshness policy, direct/control exceptions, coalescing behavior, and /stop queued-work semantics.
