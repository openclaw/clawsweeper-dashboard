Keep this PR open. Current main has already absorbed the stale-plugin cleanup and bundled-plugin load-path cleanup through newer doctor helpers, but it still lacks the PR's runtime-drift reporting for state-dir mismatch, test-state service usage, and service-vs-CLI version mismatch; the PR remains useful but needs rebase/focus and one concrete symlink-related fix before merge.

## What This Changes

This PR adds doctor repairs for stale plugin config and bundled plugin overrides, plus gateway status/doctor reporting for service state-dir drift, test-state runtime usage, and service version mismatch.

## Best Possible Solution

Rebase or replace this PR with a narrow patch that keeps current main's shared doctor cleanup helpers, adds tested runtime-drift reporting to gateway status and doctor, and canonicalizes state-dir classification consistently before deciding whether the service is using ~/.openclaw-tests.

## Solution Assessment

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Canonicalize test-state detection in status:** `src/cli/daemon-cli/status.gather.ts:216-219`
  - body: The mismatch comparison uses realpaths, but test-state detection still checks `daemonConfigSummary.stateDir`. If the service state dir is a symlink outside `~/.openclaw-tests` that resolves into a test-state root, `gateway status` will miss `serviceUsesTestStateDir` even though this patch is meant to flag that drift. Pass the normalized daemon state dir into the classifier or let the classifier resolve realpaths.
  - confidence: 0.87
- **[P2] Canonicalize test-state detection in doctor:** `src/commands/doctor-gateway-services.ts:317-319`
  - body: Doctor computes `normalizedServiceStateDir` for drift comparison, but then calls `detectOpenClawTestStateDir` with the raw `serviceStateDir`. A symlinked service path outside `~/.openclaw-tests` that resolves into a test-state root will skip `gateway-state-dir-test-profile`, leaving the repair prompt silent for the drift this PR tries to detect.
  - confidence: 0.87
