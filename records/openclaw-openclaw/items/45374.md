## What This Changes

This PR adds doctor stale-plugin config pruning, bundled plugin path cleanup, test-state state-dir warnings, and gateway status/doctor service-version and state-dir drift reporting with regression tests.

## Best Possible Solution

Base a narrow replacement or rework on current main: keep the shipped stale-plugin repair path, add canonicalized runtime-drift diagnostics in status and doctor, and cover symlinked test-state dirs.

## Reproduction Assessment

Yes. Source inspection of PR head shows both status and doctor compare realpaths for mismatch but pass the raw service state dir to detectOpenClawTestStateDir, so a symlink outside ~/.openclaw-tests that resolves inside it misses the intended warning.

## Solution Assessment
