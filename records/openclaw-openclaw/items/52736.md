## Summary

Keep open. Current main still has attachment upload, paste/drop, preview, removal, and `chat.send` serialization, but it does not implement the requested screenshot/scissors capture button or region capture flow in the Control UI composer.

## What This Changes

Request to add a Control UI chat-composer screenshot/scissors button that captures a screen region, previews the image, and sends it through existing chat attachments.

## Best Possible Solution

Implement a focused Control UI composer enhancement that reuses ChatAttachment storage, preview/removal, and `chat.send.attachments`, with maintainer-approved capture and privacy UX plus targeted tests/docs.

## Reproduction Assessment

Not applicable as a feature request. The verification path is source inspection: current main only has file picker, paste, and drop attachment intake, and no Control UI screen-capture API or screenshot button.

## Solution Assessment

Unclear for the exact capture mechanism until maintainers choose the browser/native permission model. The maintainable direction is clear: reuse the existing attachment pipeline instead of adding a parallel chat-send contract.
