## Summary

Keep open. Current main and v2026.6.8 already support Control UI image/file attachments through paste, file selection, drag/drop, preview/removal, and `chat.send` serialization, but neither has the requested screenshot/scissors button or region-capture flow.

## What This Changes

Request to add a Control UI chat composer screenshot/scissors button that captures a screen region, previews it as an attachment, and sends it via `chat.send`.

## Best Possible Solution

If maintainers accept the feature, add a first-class composer capture control that reuses `ChatAttachment` registration, preview/removal, and `chat.send` serialization, with explicit permission, cancel, and error behavior.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Source, docs, release, and test inspection show attachment support exists while the requested screenshot capture control is absent.

## Solution Assessment

Unclear: reusing the existing attachment pipeline is the right implementation boundary, but the exact capture UX and browser/native permission model need maintainer product review first.
