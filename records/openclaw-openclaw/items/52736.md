## Summary

Keep open. Current main still has the Control UI attachment groundwork for file picker, paste/drop image intake, previews, removal, and `chat.send` serialization, but it still does not implement the requested screenshot/scissors capture button or region/screen capture flow in the chat composer. The supplied related items are adjacent Control UI button/file requests, not a canonical duplicate for this screenshot workflow.

## What This Changes

Add a screenshot/scissors button to the Control UI chat input that captures a screen region, shows a removable preview thumbnail, and sends the image through the existing chat attachment API.

## Best Possible Solution

Implement this as a focused Control UI composer enhancement: add a screenshot/scissors control, use an approved capture path, convert the captured image into the existing ChatAttachment payload store, reuse preview/removal and `chat.send.attachments`, add targeted view/controller tests, and update the Control UI docs when the workflow ships.

## Reproduction Assessment

Not applicable as a feature request. We do have a high-confidence verification path by inspecting the composer and searching current main for screen-capture APIs; current main shows only upload, paste, and drop attachment paths.

## Solution Assessment

Unclear for the exact capture mechanism until maintainers choose the browser/native permission model. The maintainable direction is clear: reuse the existing attachment pipeline and avoid adding a parallel `chat.send` contract.
