## Summary

Keep open. Current main supports attachment upload, image paste/drop, previews, removal, and `chat.send` serialization, but it still has no screenshot or screen-region capture button in the Control UI composer; the remaining work is a new UX with permission and privacy choices.

## What This Changes

The issue requests a Control UI chat-input screenshot capture button that can capture a screen region, preview the image, and send it through existing chat attachments.

## Best Possible Solution

If maintainers want this UX, add a dedicated capture affordance that feeds the existing attachment pipeline and explicitly handles permission prompts, region selection, preview/removal, browser support, and fallback to file or paste upload.
