## Summary

Keep open. Current main and v2026.6.6 have the Control UI attachment pipeline for paste, drop, file picker, preview/removal, and `chat.send` serialization, but the requested screenshot/scissors capture button and screen-region capture flow are still not implemented.

## What This Changes

The issue requests a Control UI chat composer screenshot-capture button that captures a screen region, previews it, and sends it as an existing chat attachment.

## Best Possible Solution

If maintainers want this UX, add a dedicated Control UI capture affordance that feeds the existing attachment pipeline and explicitly defines permissions, privacy, region selection, browser/native fallback, cancel/failure behavior, preview/removal, and send serialization coverage.
