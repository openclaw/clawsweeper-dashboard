## Summary

This draft external PR should stay open, but it is not merge-ready: the new LLM/tool-call middleware lookup uses preserved global hook-runner state instead of the active runtime registry, and the supplied proof is focused tests only for a new execution-interception capability.

## What This Changes

The PR adds bundled-plugin PI middleware contracts for streaming LLM calls and tool calls, wires them into PI runtime paths, exposes SDK registration types, and adds focused middleware tests.

## Best Possible Solution

Keep the PR as draft until middleware lookup uses active/request-scoped registry state, the SDK/docs contract is deliberately aligned, and live agent proof with a real bundled middleware is supplied.

## Reproduction Assessment

No high-confidence runtime reproduction was run in this read-only review. Source inspection gives a high-confidence failure path: current main can preserve a global gateway hook registry while replacing the active runtime registry, and the PR reads middleware from the preserved global registry.

## Solution Assessment

No. The generic seam direction may be reasonable, but the implementation should resolve middleware from the active/request-scoped runtime registry or pass the intended registry explicitly before exposing LLM and tool-call interception.
