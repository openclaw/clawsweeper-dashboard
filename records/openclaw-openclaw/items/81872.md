## Summary

Keep open: the feature direction is plausible, but the patch still looks up runtime middleware from the preserved global hook registry, which can skip the active plugin registry or run stale middleware, and the external proof is unit-test-only.

## What This Changes

The branch adds bundled-plugin PI middleware contracts for streaming LLM and tool-call execution, SDK and manifest registration support, runner/tool-path wiring, and focused middleware tests.

## Best Possible Solution

Keep this as a draft feature until middleware lookup is active/request-scoped, the SDK and docs contract is deliberately approved, and live agent proof with a real bundled middleware is supplied.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection shows a clear failure path because current main can preserve the global hook registry while changing the active registry, and the PR helpers read the global registry.

## Solution Assessment

No. The generic seam fits the plugin boundary, but the implementation should use the active/request-scoped registry, or pass the intended registry explicitly, before exposing execution interception.

## Review Findings

Overall correctness: patch is incorrect
