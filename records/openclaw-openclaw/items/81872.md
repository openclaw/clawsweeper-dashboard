## What This Changes

The PR adds bundled-plugin PI streaming LLM and tool-call middleware registrations, manifest contract validation, SDK type/API exports, runtime wiring, and focused middleware tests.

## Best Possible Solution

Keep the PR draft until middleware lookup follows active or request-scoped registry state, streaming failure semantics are fail-closed or explicitly approved, SDK/docs/baseline surfaces are aligned, and live plugin middleware proof is supplied.

## Reproduction Assessment

No high-confidence live reproduction was run in this read-only review. Source inspection shows the registry mismatch path because current main can preserve a global hook registry while replacing the active plugin registry.

## Solution Assessment

No. The extension seam is plausible, but the current implementation should use the active/request-scoped runtime registry and should not silently bypass streaming middleware failures before exposing this execution policy surface.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Use the active registry for middleware lookup:** `src/plugins/agent-tool-call-middleware.ts:61-62`
  - body: These helpers read middleware from `getGlobalPluginRegistry()`, but current loader behavior can preserve the gateway hook registry while installing a different active runtime registry. In that state, current middleware can be skipped and stale gateway middleware can still wrap PI LLM/tool calls; follow the existing tool-result middleware precedent by using the active registry or by passing the intended registry explicitly.
  - confidence: 0.84
- **[P2] Fail closed when streaming middleware setup fails:** `src/plugins/agent-streaming-llm-middleware.ts:96-100`
  - body: This catch logs a middleware setup failure and continues with the unwrapped `streamFn`, so a bundled policy middleware that throws while constructing its wrapper is bypassed and the model call proceeds. For an execution-interception surface, propagate the failure or return a failed stream path unless maintainers explicitly choose best-effort semantics.
  - confidence: 0.76
