## Summary

Keep this draft PR open: the feature direction is plausible, but the proposed middleware lookup can read the preserved global gateway registry instead of the active runtime registry, and the external PR only supplies focused test output rather than real behavior proof.

## What This Changes

The PR adds bundled-plugin PI middleware contracts for streaming LLM calls and tool calls, wires them into the PI runner/tool path, exposes SDK/API types, and adds focused registry tests.

## Best Possible Solution

Keep the PR as a draft until middleware lookup follows the active or explicitly passed runtime registry, the SDK/docs/API contract surface is deliberately approved, and live plugin middleware proof is supplied.

## Reproduction Assessment

No high-confidence runtime reproduction was run in this read-only review. Source inspection shows the likely failure path: current main can preserve a global gateway registry while setting a different active registry, and the new helpers read the global registry.

## Solution Assessment

No. The seam direction is reasonable, but the implementation should use the active/request-scoped plugin runtime registry or pass the intended registry explicitly before exposing tool-call and LLM-stream interception.
