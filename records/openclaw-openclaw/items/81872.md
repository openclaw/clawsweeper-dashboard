## Summary

Keep this draft PR open, but do not merge it as-is: the new execution middleware is looked up from preserved global hook-runner state rather than the active runtime registry, and the PR still lacks real after-fix agent proof for the external contributor gate.

## What This Changes

The PR adds bundled-plugin PI middleware contracts for streaming LLM calls and tool calls, wires them into the PI runner/tool path, exposes SDK types/API methods, and adds focused tests.

## Best Possible Solution

Keep the seam draft until middleware lookup is active/request-scoped, the SDK and manifest contract docs/baseline are aligned, and a live agent run with a real bundled middleware demonstrates the behavior.

## Reproduction Assessment

No high-confidence runtime reproduction was run in this read-only review. Source inspection provides a high-confidence failure path because current main preserves a global gateway registry while setting a different active runtime registry, and the PR reads middleware from the global registry.

## Solution Assessment

No. The middleware seam direction is plausible, but the implementation should use the active/request-scoped runtime registry or pass the intended registry explicitly before exposing LLM and tool-call interception.
