## What This Changes

The PR adds bundled-plugin PI middleware contracts for streaming LLM and tool-call execution, wires them into the PI runner/tool path, exposes SDK and manifest registration, and adds focused tests.

## Best Possible Solution

Keep the PR as a draft until middleware lookup is active/request-scoped, SDK/API follow-through is complete, maintainer product/security approval is explicit, and live bundled-middleware proof is supplied.

## Reproduction Assessment

No high-confidence live behavior reproduction was run. Source inspection gives a high-confidence failure path because current main can preserve the global gateway hook registry while changing the active registry, and the PR helpers read the global registry.

## Solution Assessment

No. The generic middleware seam is plausible, but the implementation should use the active/request-scoped registry or receive the intended registry explicitly, and the SDK baseline needs to be updated before merge.
