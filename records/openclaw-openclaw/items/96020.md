## Summary

Keep open: current main and the latest release still return stale `pending` delivery metadata on normal async `sessions_send` A2A paths, and two open closing PRs already own competing fix attempts that need review rather than a new cleanup close.

## What This Changes

The issue asks OpenClaw to stop returning `delivery.status: "pending"` from `sessions_send` after an inter-session A2A message has actually been delivered or accepted.

## Best Possible Solution

Keep this issue open until one linked PR lands a narrow `sessions_send` metadata contract that removes the misleading pending state while preserving the documented skipped case.

## Reproduction Assessment

Yes, by source inspection: current main creates non-skipped `sessions_send` delivery metadata as `pending` before launching async A2A and returns that same object from normal response paths. I did not run a live A2A scenario in this read-only review.

## Solution Assessment

Unclear: the issue is valid, but the best final fix is a maintainer contract choice between omitting optional async metadata and adding a true observed completion mechanism.
