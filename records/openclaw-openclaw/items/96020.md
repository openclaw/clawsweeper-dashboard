## Summary

Keep open: current main and the latest release still return the same `pending` delivery metadata on normal async `sessions_send` A2A paths, and two open closing PRs are already competing candidate fixes.

## What This Changes

The issue asks OpenClaw to stop reporting `sessions_send` async inter-session delivery as permanently `pending` after the message has actually been delivered.

## Best Possible Solution

Keep this issue open until one linked PR lands a single accepted `sessions_send` response contract with proof for async A2A and skipped-A2A paths.

## Reproduction Assessment

Yes, by source inspection: current main constructs non-skipped `sessions_send` delivery metadata as `pending` before launching the async A2A flow and returns it from normal response paths. I did not run a live A2A scenario in this read-only review.

## Solution Assessment

Unclear which proposed fix is the best final contract; omitting async delivery metadata appears safer than terminal-looking statuses, but maintainers should choose the public response semantics before merge.
