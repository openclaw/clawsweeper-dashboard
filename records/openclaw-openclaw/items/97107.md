## Summary

Keep open: this is a plausible fix for the linked exec false positive, but the patch widens a security-sensitive script-preflight bypass beyond the current documented YOLO contract and has no after-fix real behavior proof.

## What This Changes

The PR threads exec security into script preflight and skips the fail-closed ambiguous-interpreter heuristic when `security` is `full`, with regression tests for the affected command cases.

## Best Possible Solution

Keep the fail-closed heuristic for approval-gated exec modes unless maintainers explicitly approve a broader policy change; the narrow repair should reuse the effective YOLO predicate or update docs and tests for the new policy.

## Reproduction Assessment

Yes, from source inspection: current main reaches script preflight for `security=full` unless `ask` is also `off`, and the existing fail-closed cases reject ambiguous interpreter commands. I did not run the test suite because this review is read-only.

## Solution Assessment

No, not as written: checking only `security` is broader than the current ask-off YOLO contract. The safer fix is to carry the effective ask mode into the decision or have maintainers explicitly approve and document the broader policy.
