## Summary

Current main already solves the proxy-only web_fetch problem through a shipped explicit trusted env-proxy opt-in, and the PR branch's automatic strict-mode proxying is superseded by that safer boundary.

## What This Changes

This PR makes strict web_fetch auto-select env-proxy dispatch from HTTP(S)_PROXY, updates proxy tests, and bundles cron NO_REPLY/topic-delivery docs and regressions.

## Best Possible Solution

Keep the shipped explicit tools.web.fetch.useTrustedEnvProxy path and current cron silent-reply handling; do not merge automatic ambient strict proxying.

## Reproduction Assessment

Yes for the PR review concern: the diff sets dispatcherPolicy once from the initial URL before redirects, and current guarded fetch reuses the supplied policy through the redirect loop. The original proxy-only deployment problem is solved on current main with the explicit trusted env-proxy config.

## Solution Assessment

No. The PR's automatic ambient proxying in strict web_fetch is not the best solution; current main's explicit trusted-proxy opt-in is narrower, documented, and evaluates proxy/NO_PROXY per redirected URL.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve strict DNS pinning by default:** `src/agents/tools/web-guarded-fetch.ts:46-50`
  - body: The patch makes strict web_fetch switch to env-proxy dispatch whenever HTTP(S)_PROXY is present. That moves target DNS resolution to the proxy path without the shipped trusted-proxy opt-in, so default SSRF DNS pinning is no longer preserved.
  - confidence: 0.86
- **[P2] Re-evaluate proxy mode after redirects:** `src/agents/tools/web-guarded-fetch.ts:37-40`
  - body: Choosing dispatcherPolicy from only the initial URL means an http request that redirects to https can miss HTTPS_PROXY when only that variable is set. fetchWithSsrFGuard follows redirects with the fixed policy, so the redirected hop can bypass the configured proxy.
  - confidence: 0.79

## Security Review

Status: needs_attention

Summary: The PR changes strict web_fetch proxying in a way that bypasses the shipped explicit trusted-proxy boundary.

Concerns:

- **[medium] Preserve strict web_fetch DNS pinning:** `src/agents/tools/web-guarded-fetch.ts:46`
  - body: Auto-selecting EnvHttpProxyAgent from ambient HTTP(S)_PROXY lets strict web_fetch skip local target DNS pinning by default. That behavior belongs behind tools.web.fetch.useTrustedEnvProxy, where operators explicitly accept proxy-side DNS enforcement responsibilities.
  - confidence: 0.84
