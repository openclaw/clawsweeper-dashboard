## Summary

Keep open: current `main` still has a source-visible path where Gateway emits a final chat event without a message when no accepted assistant text reaches the chat buffer, and the reporter’s follow-up Gateway logs show exactly that no-assistant-event shape. ## What This Changes

The issue reports MiniMax/OpenClaw Gateway WebSocket runs that emit tool and lifecycle events but skip assistant/chat delta text before `state: "final"`.

## Best Possible Solution

Trace one current MiniMax tool-result run from raw provider stream through embedded assistant events and Gateway WebSocket chat projection, then fix the narrow layer that drops visible assistant text and add regression coverage.
