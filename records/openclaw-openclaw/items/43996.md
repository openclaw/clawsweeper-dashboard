## Summary

Keep open: current main and the latest release still unconditionally add Docker `--security-opt no-new-privileges` when creating sandbox containers, and the only linked fix PR is closed unmerged and conflicting. ## What This Changes

The issue asks OpenClaw to recover or diagnose Docker sandbox startup failures on hosts where `--security-opt no-new-privileges` makes `/usr/bin/sleep` fail with EPERM.

## Best Possible Solution

Keep the secure default, then add a maintainer-approved break-glass or diagnostic path with explicit security wording, audit coverage, regression tests, and real Docker proof on an affected host.
