## Summary

Keep open: current main still unconditionally applies Docker `--security-opt no-new-privileges` when creating sandbox containers, and the issue has a concrete host-level EPERM reproduction plus an open linked fix PR that has not merged.

## What This Changes

The issue requests a safe compatibility path for Docker sandbox hosts where `--security-opt no-new-privileges` makes even `/usr/bin/sleep` fail with EPERM.

## Best Possible Solution

Land a refreshed secure-default compatibility fix that omits `no-new-privileges` only through an explicit break-glass setting or equally visible diagnostic path, then close this issue after merge verification.

## Reproduction Assessment

Yes. The issue provides a focused Docker command that fails only with `--security-opt no-new-privileges`, and current main unconditionally supplies that flag before running `sleep infinity`; I did not rerun Docker in this read-only pass.

## Solution Assessment

Yes in direction, but not landed. A secure-default opt-out with dangerous-config reporting is the narrowest maintainable fix; a silent fallback without surfacing the security downgrade would need stronger product/security review.
