## Summary

Keep open: current main and the latest tagged release still unconditionally pass Docker `--security-opt no-new-privileges` before starting the sandbox `sleep infinity` process, and the open linked fix PR has not merged. ## What This Changes

The issue requests a safe compatibility path for Docker sandbox hosts where `--security-opt no-new-privileges` makes the sandbox sleep command fail with EPERM.

## Best Possible Solution

Land the linked PR or a refreshed replacement that keeps `no-new-privileges` enabled by default, adds an explicit dangerous opt-out with docs/tests/diagnostics, and includes redacted proof from an affected Docker host.

## Reproduction Assessment

Yes for source-level review: the reporter supplied a focused Docker matrix where only `--security-opt no-new-privileges` fails, and current main unconditionally applies that flag before `sleep infinity`. I did not rerun the host-specific Docker/runc failure locally.

## Solution Assessment

Yes in direction, but it is not landed. A secure-default explicit opt-out with dangerous-config reporting is the narrowest maintainable path; a silent fallback that weakens containment would need stronger security/product approval.
