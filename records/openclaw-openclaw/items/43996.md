## Best Possible Solution

Keep the hardened default, then add a maintainer-approved operator-controlled opt-out or diagnostic/recovery path with dangerous-config reporting, docs, focused tests, and real Docker proof.

## Reproduction Assessment

## Solution Assessment

Unclear until maintainers choose the security tradeoff. The narrowest maintainable path appears to be an explicit operator-controlled opt-out or clear diagnostic/recovery flow that preserves the secure default rather than a silent fallback.
