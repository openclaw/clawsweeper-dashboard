## Summary

Current `main` still mandates the Docker hardening option that the reporter isolated as the startup failure, and the open linked PR is the only focused implementation candidate. Keep this issue open until maintainers decide whether an explicit security-weaker opt-out is an acceptable product contract and then review, land, or close that PR.

## What This Changes

Provide a safe compatibility path for Docker hosts where mandatory `no-new-privileges` causes sandbox containers to exit before commands can run.

## Best Possible Solution

Keep `no-new-privileges` enabled by default and, if maintainers accept the tradeoff, land an explicitly dangerous opt-out with security-audit/status visibility, clear documentation, scoped precedence tests, and after-fix proof on both an affected Docker host and a normal hardened host.
