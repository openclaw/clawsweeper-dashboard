## Summary

Keep open. Current main and the latest release still apply Docker `no-new-privileges` unconditionally, there is no config/schema opt-out or startup retry path, and open PR #70229 remains the linked implementation candidate rather than a merged fix.

## What This Changes

Request a Docker sandbox compatibility path for hosts where `--security-opt no-new-privileges` makes sandbox containers fail to execute even `/usr/bin/sleep`.

## Best Possible Solution

Review and land or replace PR #70229 with a secure default-on break-glass setting, dangerous-config reporting, docs, and regression tests.

## Reproduction Assessment

Yes. The reporter supplied a minimal `docker run --security-opt no-new-privileges openclaw-sandbox:bookworm-slim /usr/bin/sleep 1` reproduction, and current source inspection confirms OpenClaw still sends that flag.

## Solution Assessment

Yes. A default-on break-glass config option with audit/docs coverage is safer than a silent fallback retry because it avoids weakening sandbox hardening without operator intent.
