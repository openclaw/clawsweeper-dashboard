## Summary

Keep open: current main still always applies Docker `--security-opt no-new-privileges`, and the issue has a concrete host-level EPERM reproduction plus an open linked fix PR. The issue should not be closed before https://github.com/openclaw/openclaw/pull/70229 or an equivalent fix lands.

## What This Changes

The issue asks OpenClaw to recover from or clearly surface sandbox startup failures on hosts where Docker rejects `--security-opt no-new-privileges`.

## Best Possible Solution

Land a refreshed secure-default compatibility fix that omits `no-new-privileges` only through an explicit break-glass setting or equally explicit fallback/diagnostic path, then close this issue as fixed.

## Reproduction Assessment

Yes. The issue provides a focused Docker command that fails only with `--security-opt no-new-privileges`, and current main unconditionally supplies that flag before running `sleep infinity`; I did not run Docker in this read-only pass.

## Solution Assessment

Yes in direction, but not done yet. A secure-default opt-out with dangerous-config reporting is a narrow maintainable fix, while silently falling back without surfacing the security downgrade would need extra policy review.
