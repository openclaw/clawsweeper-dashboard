## Summary

Keep open: current main and the latest release still unconditionally add Docker `no-new-privileges`, and an open linked fix PR already owns the proposed compatibility knob, so this should stay open until that PR lands or is closed.

## What This Changes

The issue requests sandbox compatibility for Docker hosts where `--security-opt no-new-privileges` makes even `/usr/bin/sleep` fail with EPERM.

## Best Possible Solution

Land a refreshed equivalent of https://github.com/openclaw/openclaw/pull/70229: keep the secure default, omit the flag only when explicitly configured false, document it as break-glass compatibility, and report the weaker setting in dangerous-config diagnostics.

## Reproduction Assessment

Yes. The reporter provided a minimal Docker command that isolates the host failure, and current main unconditionally uses the same Docker security option when creating sandbox containers.

## Solution Assessment

Yes in direction, but not complete yet. A default-on explicit opt-out with dangerous-config reporting is the narrowest maintainable fix; a silent fallback would hide a sandbox-hardening downgrade.
