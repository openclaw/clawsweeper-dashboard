## Summary

Keep open: current main and the latest release still unconditionally pass Docker `--security-opt no-new-privileges` when creating sandbox containers, and the closing implementation PR is still open rather than merged. The report has strong source-level and reporter-provided reproduction evidence, but the fix path needs security/product review because the likely opt-out weakens sandbox hardening.

## What This Changes

The issue asks for a config opt-out, fallback, or clearer diagnostics when Docker `no-new-privileges` makes OpenClaw sandbox containers exit with `exec /usr/bin/sleep: operation not permitted`.

## Best Possible Solution

Land the linked PR or a refreshed replacement that keeps `no-new-privileges` enabled by default, adds a deliberate dangerous opt-out with docs/tests/diagnostics, and includes redacted proof from an affected Docker host.

## Reproduction Assessment

Yes for source-level review. The issue provides a focused Docker command that fails only with `--security-opt no-new-privileges`, and current main still unconditionally applies that flag before running `sleep infinity`; I did not rerun the host-specific Docker/runc failure locally.

## Solution Assessment

Yes in direction, but it is not landed. A secure-default explicit opt-out with dangerous-config reporting is the narrowest maintainable fix; a silent fallback that weakens containment would need stronger security/product approval.
