## Summary

Keep this open: current main and the latest release still unconditionally pass Docker `--security-opt no-new-privileges` for sandbox containers, and the linked fix PR is open, unmerged, and currently conflicting.

## What This Changes

The issue reports Docker sandbox startup failures on hosts where `--security-opt no-new-privileges` makes `/usr/bin/sleep` fail with EPERM, and asks for an opt-out, fallback, or clearer diagnostic.

## Best Possible Solution

Land or replace the linked PR with a refreshed default-on `noNewPrivileges` compatibility knob, clear break-glass docs, and dangerous-config diagnostics on the current collector path.

## Reproduction Assessment

Yes. The reporter provided a focused Docker command that fails only with `--security-opt no-new-privileges`, and current main unconditionally adds that flag when creating sandbox containers; I did not run a live Docker repro in this read-only sweep.

## Solution Assessment

Yes in direction, but not closeable yet. A default-on opt-out with dangerous-config reporting is the narrowest maintainable fix, but this issue should stay open until https://github.com/openclaw/openclaw/pull/70229 lands or is replaced.
