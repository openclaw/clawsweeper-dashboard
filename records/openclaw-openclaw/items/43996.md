## Summary

Keep open: current main still unconditionally adds Docker `--security-opt no-new-privileges` for sandbox containers, and the linked fix PR is open rather than merged.

## What This Changes

Request a sandbox Docker compatibility path for hosts where `no-new-privileges` makes the container entry command fail with `operation not permitted`.

## Best Possible Solution

Land or replace PR #70229 with a refreshed default-on compatibility knob plus docs, schema, tests, and dangerous-config reporting on the current collector path.

## Reproduction Assessment

Yes for the current-main trigger: the reporter isolated the host failure to Docker `--security-opt no-new-privileges`, and source inspection shows OpenClaw always passes that flag when creating sandbox containers. I did not run Docker on the affected host.

## Solution Assessment

Yes in direction, but not complete yet. A default-on `noNewPrivileges` opt-out with dangerous-config reporting is the narrow fix, and the open linked PR should be reviewed or refreshed rather than closing this issue now.
