## Summary

Keep open: current main and the latest release still always pass Docker `no-new-privileges` for sandbox containers, while https://github.com/openclaw/openclaw/pull/70229 is an open, unmerged fix path that explicitly targets this issue.

## What This Changes

The issue requests a sandbox compatibility fix for Docker hosts where `--security-opt no-new-privileges` causes even `/usr/bin/sleep` to fail with `operation not permitted`.

## Best Possible Solution

Land or deliberately replace the linked compatibility PR with an equivalent default-secure opt-out or fallback plus diagnostics and regression coverage.

## Reproduction Assessment

Yes. The issue gives a minimal Docker command that fails only when `--security-opt no-new-privileges` is present on the affected host, and current main still unconditionally emits that flag for sandbox creation; I did not run the host-specific Docker matrix locally.

## Solution Assessment

Yes in direction, but not complete on current main. A default-on opt-out with dangerous-config reporting, or an automatic fallback with the same security warning, is the narrow maintainable fix and the open linked PR is the current review path.
