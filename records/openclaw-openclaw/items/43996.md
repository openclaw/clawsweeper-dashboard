## Summary

Keep open. Current main still applies Docker `no-new-privileges` unconditionally, the public config/schema surface still has no opt-out or fallback path, and open PR #70229 remains the linked implementation candidate rather than a merged fix.

## What This Changes

The issue requests a safe compatibility path for hosts where Docker `--security-opt no-new-privileges` makes sandbox containers exit immediately with `exec /usr/bin/sleep: operation not permitted`.

## Best Possible Solution

Refresh, review, and land or replace PR #70229 with a secure default-on Docker sandbox setting wired through config types, strict schema, config resolution, Docker create args, dangerous-config reporting, docs, and regression tests; close this issue only after that fix merges and ships.

## Reproduction Assessment

Yes. The reporter provided a minimal `docker run --security-opt no-new-privileges openclaw-sandbox:bookworm-slim /usr/bin/sleep 1` reproduction, and current source inspection confirms OpenClaw still applies that flag to Docker sandbox containers.

## Solution Assessment

Yes. An explicit default-on break-glass configuration knob with audit/docs coverage is the narrowest maintainable fix; a silent fallback retry would relax sandbox hardening without clear operator intent.
