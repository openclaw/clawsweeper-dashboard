## Best Possible Solution

Keep `no-new-privileges` as the secure default and add a maintainer-approved, explicit, audited compatibility path plus diagnostics for hosts where Docker rejects that option.

## Reproduction Assessment

Yes at source/report level, but not by live Docker repro in this checkout. The issue provides a host-level matrix where only `--security-opt no-new-privileges` fails, and current main plus v2026.6.8 still always pass that option.

No final implementation is present yet. The previous PR’s explicit opt-out is a plausible narrow direction, but the best fix needs maintainer approval for the sandbox hardening tradeoff and should avoid a silent hardening downgrade.
