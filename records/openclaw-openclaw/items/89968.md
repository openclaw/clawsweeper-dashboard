## Best Possible Solution

Keep the collector-boundary fix, make channel-row collection atomic before merging into the shared map, then refresh the branch onto current main and land after the normal changed gate.

## Reproduction Assessment

Yes. Current main reads plugin record fields directly inside both metadata collector loops, so a throwing field can abort metadata collection before later healthy rows are processed; this read-only review did not run the focused test.

## Solution Assessment

No as submitted. Per-record collector isolation is the right boundary, but the channel path should stage row-local mutations and merge them only after the row is fully read so poisoned rows cannot leave partial state behind.
