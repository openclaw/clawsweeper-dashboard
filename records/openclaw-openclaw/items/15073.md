## Summary

Keep open: current main and v2026.6.10 still only retry provider/model candidates inside the original agent run context, while the requested fallback agent/workspace/context-profile contract still needs maintainer product design.

## What This Changes

Request an agent-level or model-specific fallback context option so a fallback model can run with a smaller workspace/context instead of the primary agent prompt.

## Best Possible Solution

Define one maintainer-approved fallback context contract, then update config types, strict schemas, docs, fallback candidate metadata, reply-runner/session plumbing, and regression tests together.

## Reproduction Assessment

Yes, at source level. Current-main types, strict schemas, docs, fallback candidate resolution, and attempt execution show retries vary provider/model only while reusing the caller's run context.

## Solution Assessment

No implementation is under review yet. The issue is valid, but maintainers should choose fallbackAgent/fallbackWorkspace semantics or a narrower model-context-profile contract before any repair PR picks public config and session semantics.
