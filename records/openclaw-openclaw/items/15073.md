## Summary

Keep open: current main and v2026.6.10 still implement fallback as provider/model candidate retry inside the original run context, while this request needs a maintainer-approved fallback agent/workspace/context-profile contract.

## What This Changes

The issue requests an option for model fallback to use a different agent workspace/context, or an equivalent model-specific context profile, when falling back to smaller local models.

## Best Possible Solution

Keep this issue open for a first-class fallback context design that chooses fallbackAgent/fallbackWorkspace semantics or a narrower model-specific context profile/budget mechanism, then updates config, schemas, docs, candidate metadata, session plumbing, and regression tests together.

## Reproduction Assessment

Yes, at source level. Current-main types, strict schemas, docs, fallback candidate resolution, and attempt execution show retries vary provider/model only while reusing the caller's run context.

## Solution Assessment

No implementation is under review yet. The issue is valid, but maintainers should choose a fallback context-profile or cross-agent handoff contract before any repair PR picks public config and session semantics.
