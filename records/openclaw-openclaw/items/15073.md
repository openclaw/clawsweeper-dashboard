## Summary

Keep open: current main still lacks a first-class fallback-context contract, and related context-budget/profile reports were already closed as duplicates of this canonical design thread. The remaining work is core API and session-state semantics, so it should not be auto-closed or queued as a narrow repair.

## What This Changes

The issue requests an agent-level fallback or equivalent model-specific context/profile mechanism so smaller fallback models do not receive the full primary agent workspace context.

## Best Possible Solution

Define a first-class fallback-context contract, then update config types, strict schemas, schema help/docs, fallback candidate metadata, reply-runner/session plumbing, and regression tests for alternate fallback context without corrupting original agent state.

## Reproduction Assessment

Yes for the current missing capability: source inspection on current main shows strict config surfaces lack the requested fields, fallback candidates carry only provider/model, and retry callers reuse the original workspace/session/skills state. No live outage test was run because this is a feature/design request, not a narrow broken contract.

## Solution Assessment

Unclear. The requested fallbackAgent shape may be viable, but maintainers should first choose between agent switching, workspace override, and model-specific context-profile or budget semantics.
