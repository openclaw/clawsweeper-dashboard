## Summary

Keep open: current main and latest release still model fallback as provider/model retry selection inside the original agent/session context, while this request needs a first-class fallback context design and maintainer API decision.

## What This Changes

The issue requests agent-level or model-specific fallback context so a fallback model can use a different agent, workspace, or smaller bootstrap profile instead of the primary agent's full context.

## Best Possible Solution

Keep this open for a first-class fallback context design that chooses either typed context-profile/budget semantics or cross-agent handoff ownership for state, auth, transcript, workspace, skills, and delivery.

## Reproduction Assessment

Yes, at source level. Current-main types, strict schemas, docs, and fallback execution show retries vary provider/model only while reusing the caller's agent, workspace, agentDir, session file, and skills snapshot.

## Solution Assessment

No implementation is under review yet. The issue is valid, but maintainers should choose a fallback context-profile or cross-agent handoff contract before any repair PR picks public config and session semantics.
