## Summary

Keep open: current main and the latest release still implement fallback as provider/model retry selection inside the original run context, while this request needs a maintainer-approved fallback context/session design.

## What This Changes

Request an agent-level fallback or model-specific context/workspace mechanism so smaller fallback models do not inherit the primary agent's full workspace prompt, skills, and session context.

## Best Possible Solution

Keep this issue open for a first-class fallback context design that chooses either typed context-profile/budget semantics or cross-agent handoff ownership for state, auth, transcript, workspace, skills, and delivery.

## Reproduction Assessment

Yes, at source level. Current-main types, strict schemas, docs, and fallback execution show retries vary provider/model only while reusing the caller's agent, workspace, agentDir, session file, and skills snapshot.

## Solution Assessment

No implementation is under review yet. The issue is valid, but maintainers should choose a fallback context-profile or cross-agent handoff contract before any repair PR picks public config and session semantics.
