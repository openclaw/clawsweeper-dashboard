- `impact:session-state`: The requested fallback semantics affect workspace context, prompt state, skills, transcripts, and potentially agent identity.

## Summary

Keep this issue open: current main still changes only the provider/model candidate during fallback, while the agent, workspace, bootstrap context, and session ownership remain fixed; overlapping requests continue to treat this issue as the canonical unresolved design.

## What This Changes

Add a fallback mode that can use a separate agent workspace/context or a model-specific reduced context profile when the primary model fails.

## Best Possible Solution

Define one candidate-specific context contract—prefer a bounded model context profile when only prompt reduction is needed, reserving full fallback-agent switching for cases that require a distinct identity—and keep session, auth, transcript, workspace, and skills ownership explicit with focused upgrade and regression coverage.

## Reproduction Assessment

Yes at the source level: fallback selection changes only provider/model while the config, candidate, and bootstrap contracts have no alternate agent, workspace, or context selector. A live provider failure was not invoked during this read-only review.

## Solution Assessment

Unclear pending a product decision: full agent switching satisfies the broad request but introduces substantial identity and session-state semantics, while a model-specific context profile may solve the reported overload with a narrower and safer contract.
