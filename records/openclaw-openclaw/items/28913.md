## Summary

Keep open: current main and v2026.5.19 still lack a wrapper/provenance guard or accepted captureFilter for Memory LanceDB autoCapture, and the related open provenance/trust issues do not supersede this narrower source-filtering request.

## What This Changes

Request a Memory LanceDB autoCapture filter so wrapped external untrusted content is not stored as long-term memory.

## Best Possible Solution

Define a maintainer-approved memory-capture trust policy, then update Memory LanceDB config, schema, docs, and regression tests so wrapped or provenance-marked untrusted content cannot be auto-stored unintentionally.

## Reproduction Assessment

Yes, source-level. With autoCapture enabled, short user-role text containing EXTERNAL_UNTRUSTED_CONTENT markers plus a trigger such as always reaches shouldCapture and the store path because no wrapper/provenance guard or accepted captureFilter setting exists.

## Solution Assessment

Unclear. captureFilter.excludeWrapped is plausible, but default skipping for wrapped external content or a broader provenance-aware persistence rule may be safer and needs maintainer security/product judgment first.
