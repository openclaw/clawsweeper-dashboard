## Summary

Keep open: current main and the latest release still lack any autoCapture wrapper/provenance filter, and related broader memory-trust issues remain open rather than superseding this narrower Memory LanceDB capture-policy request.

## What This Changes

The issue requests a `memory.captureFilter.excludeWrapped` option so Memory LanceDB autoCapture skips content wrapped in external-untrusted boundary tags.

## Best Possible Solution

Define a maintainer-approved Memory LanceDB capture trust policy, then align runtime filtering, config/manifest/docs, and regression tests so wrapped or provenance-marked untrusted content is not auto-stored unintentionally.

## Reproduction Assessment

Yes, source-level. With `autoCapture` enabled, user-role text containing `EXTERNAL_UNTRUSTED_CONTENT` markers plus a trigger such as `always` reaches `shouldCapture` and the store path because no wrapper/provenance guard or accepted `captureFilter` setting exists.

## Solution Assessment

Unclear. `captureFilter.excludeWrapped` is a plausible narrow option, but a default skip for wrapped external content or a broader provenance-aware persistence rule may be safer and needs maintainer security/product judgment first.
