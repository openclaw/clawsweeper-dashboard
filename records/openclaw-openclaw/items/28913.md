## Summary

Keep open. Current main still has no `captureFilter` or source/provenance-aware auto-capture guard in `memory-lancedb`; the related provenance, channel-wrapping, and trust-tagging issues remain broader design tracks rather than canonical replacements for this narrow auto-capture gap.

## What This Changes

Request adding a `memory.autoCapture` filter that skips turns containing configured untrusted external-content wrappers before regex-based memory capture.

## Best Possible Solution

Add a maintainer-approved source/provenance-aware auto-capture policy in `memory-lancedb`, with config/schema/docs and regression coverage that preserves normal direct-user preference capture while excluding marked untrusted external content.

## Reproduction Assessment

Yes, at source level. A short role=user text block containing `EXTERNAL_UNTRUSTED_CONTENT` markers plus a broad trigger such as `always` reaches `extractUserTextContent` and `shouldCapture` with no wrapper/provenance filter, while the requested `captureFilter` key is absent from strict config parsing.

## Solution Assessment

Unclear as a final design. The requested wrapper exclusion targets the right memory-poisoning surface, but the maintainable fix should use maintainer-approved shared boundary or provenance semantics rather than ad hoc marker-name parsing.
