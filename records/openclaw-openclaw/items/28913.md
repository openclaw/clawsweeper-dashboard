## Summary

Keep open. Current main still lacks `captureFilter` or a source/provenance-aware auto-capture guard in `memory-lancedb`, and the open related provenance/security architecture issues are broader tracks rather than a canonical replacement for this narrow persistent-memory gap.

## What This Changes

Requests a `memory-lancedb` autoCapture filter that skips `EXTERNAL_UNTRUSTED_CONTENT`-wrapped text before memory trigger matching and storage.

## Best Possible Solution

Add a maintainer-approved source/provenance-aware auto-capture policy in `memory-lancedb`, with config/schema/docs and regression coverage that preserves direct-user preference capture while excluding marked untrusted external content.

## Reproduction Assessment

Yes, at source level. A role=user message containing `EXTERNAL_UNTRUSTED_CONTENT` markers plus a broad trigger such as `always` is extracted and passed to `shouldCapture`, while the requested `captureFilter` key is absent from strict config parsing.

## Solution Assessment

Unclear as a final design. The wrapper exclusion targets the right memory-poisoning surface, but the maintainable fix should reuse approved external-content or provenance semantics rather than ad hoc marker-name parsing.
