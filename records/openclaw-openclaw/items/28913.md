## Label Justifications

- `P2`: The report identifies a meaningful but bounded persistent-memory security path that has now been addressed.
- `impact:session-state`: The issue concerns external content being persisted into memory and resurfacing in later sessions.
- `impact:security`: The reported failure mode is indirect prompt injection crossing into durable memory.

## Summary

Current main and the shipped `v2026.6.5` release already block the reported wrapper-based memory-poisoning path by sanitizing and rejecting untrusted envelope content before auto-capture trigger matching and persistence; broader persistent provenance remains separately tracked.

## What This Changes

Add source-aware filtering so automatic memory capture does not persist content marked as externally untrusted.

## Best Possible Solution

Keep the shipped structural sanitizer as the single auto-capture policy and continue any generalized, cross-store source-provenance design in https://github.com/openclaw/openclaw/issues/48519.

## Reproduction Assessment

No. On current main and in `v2026.6.5`, auto-capture sanitizes and rejects canonical untrusted envelope content before trigger evaluation and storage, so the described wrapper-based persistence path no longer reproduces.

## Solution Assessment

No. A configurable list of security wrapper strings would duplicate the shipped structural guard and create a brittle second policy surface; typed provenance beyond envelopes belongs in the broader canonical architecture issue.
