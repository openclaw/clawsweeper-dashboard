## Summary

Keep open: current main and v2026.6.5 still lack a Memory LanceDB autoCapture wrapper/provenance filter, and the related open memory-trust issues are broader design threads rather than a merged fix for this narrower capture-policy request.

## What This Changes

The issue requests a source-based autoCapture filter that prevents wrapped untrusted external content from being stored as durable Memory LanceDB memories.

## Best Possible Solution

Define a maintainer-approved Memory LanceDB capture trust policy, then align runtime filtering, config/manifest/docs, and regression tests so wrapped or provenance-marked untrusted content is not auto-stored unintentionally.

## Reproduction Assessment

Yes, source-level. With autoCapture enabled, wrapped user-role text that survives sanitizeForMemoryCapture can still reach shouldCapture and db.store because there is no wrapper/provenance guard or accepted captureFilter setting.

## Solution Assessment

Unclear. captureFilter.excludeWrapped is a plausible narrow option, but a default skip for wrapped external content or broader provenance-aware persistence rule may be safer and needs maintainer security/product judgment first.
