## Summary

Keep open: current main still lacks wrapper- or provenance-aware filtering in the Memory LanceDB autoCapture path, and the related memory-trust/provenance issues are broader design context rather than a shipped fix for this focused policy/config request.

## What This Changes

Add a Memory LanceDB autoCapture filter that skips wrapped untrusted external content before memory capture triggers run.

## Best Possible Solution

Define a maintainer-approved memory-capture trust policy, then align Memory LanceDB config/schema/docs and regression tests so wrapped untrusted external content is not auto-stored in a way that can poison future memory recall.

## Reproduction Assessment

Yes, source-level. With autoCapture enabled, user-role text under captureMaxChars that contains EXTERNAL_UNTRUSTED_CONTENT markers plus a trigger like `always` reaches shouldCapture and the store path because no wrapper/provenance guard or config option exists.

## Solution Assessment

Unclear. `captureFilter.excludeWrapped` is a plausible narrow opt-in, but a default skip for wrapped external content or broader provenance-aware persistence policy may be safer and needs maintainer judgment first.
