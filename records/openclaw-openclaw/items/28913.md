## Summary

Keep open: current main still lets Memory LanceDB autoCapture evaluate and store user-role text that contains external-content wrapper markers when a normal memory trigger matches, and no accepted captureFilter or provenance-aware capture policy exists yet.

## What This Changes

Add source-based filtering to Memory LanceDB autoCapture so wrapped external content such as EXTERNAL_UNTRUSTED_CONTENT is not silently persisted as long-term memory.

## Best Possible Solution

Define a maintainer-approved memory-capture trust policy, then update Memory LanceDB config/schema/docs and regression tests so wrapped or provenance-marked untrusted external content cannot be auto-stored unintentionally.

## Reproduction Assessment

Yes, source-level. With autoCapture enabled, short user-role text containing EXTERNAL_UNTRUSTED_CONTENT markers plus a trigger such as always reaches shouldCapture and the store path because no wrapper/provenance guard or accepted captureFilter setting exists.

## Solution Assessment

Unclear. captureFilter.excludeWrapped is a plausible narrow option, but default skipping for wrapped external content or a broader provenance-aware persistence rule may be safer and needs maintainer security/product judgment first.
