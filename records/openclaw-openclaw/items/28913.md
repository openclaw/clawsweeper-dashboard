## Summary

Keep open: current main and v2026.5.19 still let Memory LanceDB autoCapture evaluate user-role text containing external-content wrapper markers, and there is no captureFilter or provenance-aware memory-capture policy. The related provenance/trust issues are useful context, but they are broader open design items rather than a shipped fix for this narrower autoCapture gap.

## What This Changes

Add source- or wrapper-based filtering to Memory LanceDB autoCapture so wrapped untrusted external content is not stored as durable memory.

## Best Possible Solution

Define a maintainer-approved memory-capture trust policy, then update Memory LanceDB config/schema/docs and regression tests so wrapped or provenance-marked untrusted external content cannot be auto-stored unintentionally.

## Reproduction Assessment

Yes, source-level. With autoCapture enabled, short user-role text containing EXTERNAL_UNTRUSTED_CONTENT markers plus a trigger such as always reaches shouldCapture and the store path because no wrapper/provenance guard or accepted captureFilter setting exists.

## Solution Assessment

Unclear. captureFilter.excludeWrapped is a plausible narrow option, but a default skip for wrapped external content or a broader provenance-aware persistence rule may be safer and needs maintainer security/product judgment first.
