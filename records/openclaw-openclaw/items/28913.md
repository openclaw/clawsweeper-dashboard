## Summary

Keep open. Current main still lacks wrapper-aware auto-capture filtering in `memory-lancedb`, and the related open issues track broader prompt-injection/provenance architecture rather than replacing this narrow memory policy request.

## What This Changes

Add a `captureFilter` option for `memory-lancedb` auto-capture so wrapper-marked untrusted content is excluded from automatic memory storage.

## Best Possible Solution

Add a maintainer-approved `memory-lancedb` capture policy that uses a shared external-content/provenance boundary where possible, updates runtime parsing, manifest/UI hints, docs, and regression tests, and preserves normal direct-user preference capture.

## Reproduction Assessment

Yes, at source level. A `captureFilter.excludeWrapped` config is not accepted today, and wrapper-marked user text can still flow into `shouldCapture` before any wrapper/provenance filter is applied.

## Solution Assessment

Yes, with a design caveat. An opt-in wrapper exclusion is a narrow maintainable direction for this memory-poisoning path, but maintainers should approve the public config shape and prefer shared boundary helpers over ad hoc marker parsing.
