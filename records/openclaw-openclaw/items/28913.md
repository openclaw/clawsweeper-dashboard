## Summary

Keep open. Current main still lacks wrapper-aware `autoCapture` filtering in `memory-lancedb`, and the related open issues track broader prompt-injection/provenance architecture rather than replacing this narrow memory policy request.

## What This Changes

Add a `memory.captureFilter.excludeWrapped` option so `autoCapture` skips content inside external-content wrapper tags before memory trigger matching.

## Best Possible Solution

Add a maintainer-approved `memory-lancedb` capture policy that reuses shared external-content boundary/provenance semantics, updates config parsing, manifest/UI hints, docs, and regression tests, and preserves normal direct-user preference capture.

## Reproduction Assessment

Yes, at source level. Current main rejects the requested config key and still lets wrapper-marked user text reach `shouldCapture` before any external-content or provenance filter is applied.

## Solution Assessment

Yes, with a design caveat. An opt-in wrapper exclusion is a narrow maintainable direction for this memory-poisoning path, but maintainers should approve the config shape and prefer shared boundary/provenance helpers over ad hoc marker parsing.
