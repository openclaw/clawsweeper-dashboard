## Summary

Keep open: current main and v2026.5.20 still let Memory LanceDB autoCapture evaluate wrapped external-content text without a source/provenance guard, and the related trust/provenance issues remain broader open design work rather than a shipped fix.

## What This Changes

The issue requests a Memory LanceDB autoCapture filter that skips messages containing configured external-content wrapper tags before trigger-based memory capture.

## Best Possible Solution

Define a maintainer-approved, framework-owned memory capture trust policy, then update Memory LanceDB config/schema/docs and regression tests so wrapped or provenance-marked untrusted content cannot be auto-stored unintentionally.

## Reproduction Assessment

Yes, source-level. With autoCapture enabled, user-role text containing EXTERNAL_UNTRUSTED_CONTENT markers plus a trigger such as always reaches shouldCapture and the store path because no wrapper/provenance guard or accepted captureFilter setting exists.

## Solution Assessment

Unclear. captureFilter.excludeWrapped is a plausible narrow option, but a default skip for wrapped external content or a broader provenance-aware persistence rule may be safer and needs maintainer security/product judgment first.
