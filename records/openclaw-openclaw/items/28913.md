## Summary

Keep open: current main and v2026.5.20 still let Memory LanceDB autoCapture evaluate wrapped external-content text without a provenance or wrapper guard, and the proposed filter remains a security/product design decision rather than shipped behavior.

## What This Changes

Add a Memory LanceDB `captureFilter` option so autoCapture can skip text wrapped in `EXTERNAL_UNTRUSTED_CONTENT` markers.

## Best Possible Solution

Define a maintainer-approved memory capture trust policy, then update Memory LanceDB config/schema/docs and regression tests so wrapped or provenance-marked untrusted content is not auto-stored unintentionally.

## Reproduction Assessment

Yes, source-level. With autoCapture enabled, user-role text containing `EXTERNAL_UNTRUSTED_CONTENT` markers plus a trigger such as `always` reaches `shouldCapture` and the store path because no wrapper/provenance guard or accepted `captureFilter` setting exists.

## Solution Assessment

Unclear. `captureFilter.excludeWrapped` is a plausible narrow option, but a default skip for wrapped external content or a broader provenance-aware persistence rule may be safer and needs maintainer security/product judgment first.
