## Summary

Keep open: current main still allows Memory LanceDB autoCapture to evaluate and store matching user text without checking external-content wrappers or structured provenance, and the related provenance/trust issues are broader design context rather than a shipped replacement for this focused hardening request.

## What This Changes

Add a Memory LanceDB autoCapture filter that skips content wrapped with external-untrusted markers before heuristic memory capture.

## Best Possible Solution

Define a maintainer-approved memory-capture trust policy, then update Memory LanceDB config/schema/docs and regression tests so wrapped or provenance-marked untrusted external content cannot be auto-stored into durable memory unintentionally.

## Reproduction Assessment

Yes, source-level. With autoCapture enabled, user-role text under captureMaxChars that contains EXTERNAL_UNTRUSTED_CONTENT markers plus a trigger like always reaches shouldCapture and the store path because no wrapper/provenance guard or accepted captureFilter setting exists.

## Solution Assessment

Unclear. captureFilter.excludeWrapped is a plausible narrow opt-in, but a default skip for wrapped external content or a broader provenance-aware persistence policy may be safer and needs maintainer judgment first.
