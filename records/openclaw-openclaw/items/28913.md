## Summary

Keep open: current main still lacks wrapper- or provenance-aware filtering in the Memory LanceDB autoCapture path, and the open broader provenance issue does not fully supersede this narrower memory capture policy/config request.

## What This Changes

Add a Memory LanceDB autoCapture filter, such as `captureFilter.excludeWrapped`, so wrapped untrusted external content is not stored as long-term memory.

## Best Possible Solution

Define a maintainer-approved memory-capture policy that prevents wrapped untrusted external content from being auto-stored, then align Memory LanceDB config/schema/docs and regression tests with that decision.

## Reproduction Assessment

Yes, source-level. With `autoCapture: true`, a user-role text block under `captureMaxChars` containing external-content markers plus a trigger word reaches `shouldCapture` and the store path because neither the heuristic nor config exposes wrapper filtering.

## Solution Assessment

Unclear. `captureFilter.excludeWrapped` is a plausible narrow opt-in, but a default skip for wrapped external content or a broader provenance-aware persistence policy may be safer and needs maintainer judgment first.
