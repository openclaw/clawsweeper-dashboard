## Summary

Keep open. Current main still lacks the requested wrapper-aware auto-capture filter: the memory-lancedb config and manifest do not expose `captureFilter`, and the auto-capture path still sends extracted user text into broad trigger matching without checking external-content wrappers. Related #29442 and #48519 are broader security/provenance proposals, not a replacement for this narrow plugin setting.

## What This Changes

Add a `captureFilter.excludeWrapped` option so memory-lancedb auto-capture can skip content wrapped in configured untrusted-source markers before trigger matching.

## Best Possible Solution

Implement a backward-compatible memory-lancedb `captureFilter.excludeWrapped` setting in the plugin config parser, manifest schema/UI hints, docs, and auto-capture hook, with regression tests for wrapped untrusted trigger text and ordinary direct preference capture.

## Reproduction Assessment

Yes, at source level. A config containing `captureFilter.excludeWrapped` would be rejected by the strict memory-lancedb parser/manifest, and marker-only wrapped trigger text can still reach `shouldCapture` because no wrapper filter is applied before broad trigger matching.

## Solution Assessment

Yes, with a design caveat. The opt-in wrapper filter is a narrow maintainable fix for this specific memory-poisoning path, but maintainers should approve the config shape and prefer a public external-content boundary helper over ad hoc marker parsing.
