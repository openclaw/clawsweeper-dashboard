## Summary

Keep open. Current main still has no `captureFilter` config or wrapper-aware auto-capture policy in `memory-lancedb`, while the related open issues cover broader provenance/trust architecture rather than replacing this narrow memory policy request.

## What This Changes

The issue requests an opt-in `memory-lancedb` auto-capture filter that skips configured external-content wrapper tags before memory trigger matching.

## Best Possible Solution

Add a maintainer-approved `memory-lancedb` capture policy that reuses shared external-content/provenance semantics, updates config parsing, manifest/UI hints, docs, and regression tests, and preserves normal direct-user preference capture.

## Reproduction Assessment

Yes, at source level. A user/cron turn containing compact `EXTERNAL_UNTRUSTED_CONTENT` text with a broad trigger such as `always` reaches `shouldCapture` with no wrapper or provenance filter, while the requested `captureFilter` config is rejected by strict parsing.

## Solution Assessment

Yes, with a design caveat. An opt-in wrapper exclusion is a narrow maintainable direction for this memory-poisoning path, but maintainers should approve the config shape and prefer shared boundary/provenance helpers over ad hoc marker parsing.
