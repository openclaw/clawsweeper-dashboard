## Summary

Keep open: current main still lacks source- or wrapper-aware filtering for `memory-lancedb` auto-capture, and the related provenance/security issues are broader open design tracks rather than a merged or terminal replacement. The request is security-relevant and plausible from source inspection, but the fix adds a new memory policy/config surface and should be coordinated with the persistent-provenance work before implementation.

## What This Changes

The issue requests an opt-in `captureFilter` for memory `autoCapture` so wrapped untrusted external content is not stored as long-term memory.

## Best Possible Solution

Add a framework-owned way for memory auto-capture to skip untrusted or provenance-tainted text, with schema, manifest, docs, and regression coverage, either as a narrow memory-lancedb option or as the first slice of the versioned durable-provenance contract.

## Reproduction Assessment

Yes, from source inspection: a user-role message containing wrapped external content and a natural trigger such as `always` reaches `shouldCapture`, and the current auto-capture loop has no wrapper/provenance skip before embedding and storage. I did not run a live LanceDB flow because this review is read-only.

## Solution Assessment

Unclear as a final design. `captureFilter.excludeWrapped` is a narrow backwards-compatible mitigation, but maintainers should decide whether this belongs as a memory plugin config knob or under the broader persistent provenance policy surface.

Concerns:

- **[medium] Filter untrusted content before memory persistence:** `extensions/memory-lancedb/index.ts:1086`
  - body: The memory plugin currently stores captured user text after `shouldCapture` without checking external-content wrappers or source provenance, so a wrapped external payload with a natural memory trigger can become durable memory if it reaches the user-message capture path.
  - confidence: 0.86
- **[medium] Avoid treating wrapper filtering as full provenance:** `src/security/external-content.ts:63`
  - body: External-content markers are text boundaries, not durable provenance metadata; a narrow wrapper filter can reduce one capture path but will not protect derived content after summarization or marker removal.
  - confidence: 0.78
