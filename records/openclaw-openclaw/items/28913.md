## Summary

Keep open: current main still lacks wrapper- or provenance-aware filtering in the Memory LanceDB autoCapture path, and the broader provenance/taint issues are useful related context but do not replace this focused memory capture policy/config request.

## What This Changes

The issue requests a configurable Memory LanceDB autoCapture filter that skips wrapped untrusted external content before heuristic memory storage.

## Best Possible Solution

Define a maintainer-approved memory-capture policy that prevents wrapped untrusted external content from being auto-stored, then align Memory LanceDB config/schema/docs and regression tests with that decision.

## Reproduction Assessment

Yes, source-level. With autoCapture enabled, user-role text under captureMaxChars that contains EXTERNAL_UNTRUSTED_CONTENT markers plus a trigger like “always” reaches shouldCapture and the store path because no wrapper/provenance guard or config option exists.

## Solution Assessment

Unclear. captureFilter.excludeWrapped is a plausible narrow opt-in, but a default skip for wrapped external content or broader provenance-aware persistence policy may be safer and needs maintainer judgment first.
