## Summary

Keep open: current main has Memory LanceDB auto-capture and core external-content wrappers, but no source-aware capture filter or wrapper-aware skip in the auto-capture path. The related broader provenance issue remains open, but it does not fully supersede this narrower Memory LanceDB policy/config request.

## What This Changes

Requested source-based filtering for memory auto-capture so wrapped untrusted external content is not stored as memory.

## Best Possible Solution

Define a maintainer-approved memory-capture policy that prevents wrapped untrusted external content from being auto-stored, then align the Memory LanceDB schema, manifest, docs, and regression tests with the chosen default/config behavior.

## Reproduction Assessment

Yes, source-level: enable Memory LanceDB autoCapture and pass a user-role text block containing EXTERNAL_UNTRUSTED_CONTENT markers plus a trigger word like always under captureMaxChars; current shouldCapture has no wrapper guard before storage. I did not run a live memory write because this review is read-only.

## Solution Assessment

Unclear. The proposed captureFilter option is a plausible narrow fix, but a default skip for wrapped external content or a broader provenance-aware policy may be safer and needs maintainer judgment.
