## Summary

Keep open. Current main still lacks `captureFilter` or an equivalent source/provenance-aware auto-capture guard in `memory-lancedb`, and the related open trust/provenance issues are broader design tracks rather than a clear canonical replacement for this narrow persistent-memory gap.

## What This Changes

The issue requests a memory auto-capture filter that excludes wrapped untrusted external content, such as `EXTERNAL_UNTRUSTED_CONTENT`, from persistent LanceDB memories.

## Best Possible Solution

Add a maintainer-approved source/provenance-aware auto-capture policy in `memory-lancedb`, with config/schema/docs and regression coverage that preserves direct-user preference capture while excluding marked untrusted external content.

## Reproduction Assessment

Yes, at source level. A role=user message containing `EXTERNAL_UNTRUSTED_CONTENT` markers plus a broad trigger such as `always` is extracted and passed to `shouldCapture`, while the requested `captureFilter` key is absent from strict config parsing.

## Solution Assessment

Unclear as the final design. Wrapper exclusion targets the right memory-poisoning surface, but the maintainable fix should reuse approved external-content or provenance semantics instead of ad hoc marker-name parsing.
