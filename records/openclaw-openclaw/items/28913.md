## Summary

Keep open. Current main still lacks wrapper-aware auto-capture filtering for memory-lancedb, and the two related open issues are broader security/provenance proposals rather than replacements for this narrow memory setting.

## What This Changes

Add an opt-in memory-lancedb autoCapture filter that skips turns containing configured external-content wrapper tags before regex capture triggers run.

## Best Possible Solution

Implement a maintainer-approved memory-lancedb capture policy in the config parser, manifest/UI hints, docs, and auto-capture hook, with regression coverage for wrapped untrusted trigger text and normal direct preference capture.

## Reproduction Assessment

Yes, at source level. A config containing captureFilter.excludeWrapped is rejected today, and marker-only wrapped user text containing a broad trigger such as always can reach shouldCapture because no wrapper filter is applied before matching.

## Solution Assessment

Yes, with a design caveat. An opt-in wrapper exclusion is a narrow fix for this memory-poisoning path, but maintainers should settle the public config shape and prefer a shared external-content boundary helper over ad hoc marker parsing.
