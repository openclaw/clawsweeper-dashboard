## Summary

Keep open: current main still lacks source- or wrapper-aware filtering for memory-lancedb auto-capture. The requested config is not accepted, the manifest has no schema/UI surface for it, and the auto-capture hook still runs extracted user text through trigger matching without checking external-content wrappers; related #29442 and #48519 are broader open security/provenance discussions, not replacements for this narrow request.

## What This Changes

Add a `captureFilter.excludeWrapped` option for autoCapture so turns containing configured wrapper tags such as `EXTERNAL_UNTRUSTED_CONTENT` are skipped before memory trigger matching.

## Best Possible Solution

Implement a backward-compatible memory-lancedb `captureFilter.excludeWrapped` config path in the plugin parser, manifest, docs, and tests, then apply it before `shouldCapture` in the `agent_end` auto-capture path. Keep ordinary direct user preference capture working, and leave broader privilege-separation/provenance architecture to #29442 and #48519.
