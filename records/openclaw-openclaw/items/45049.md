## Label Justifications

- `P1`: The issue describes a broken agent/tool workflow that can deliver fabricated tool-use claims or skip required actions in real tool-enabled sessions.
- `impact:security`: The report concerns protocol-boundary enforcement for tool invocation, hooks, policy checks, and prompt-injection-resistant execution provenance.
- `impact:message-loss`: When the model emits prose or fake tool text instead of a real call, the intended tool-mediated action or reply can be silently skipped.
- `impact:session-state`: The failure leaves transcripts with assistant claims or fake result text that are not backed by matching tool-call/tool-result ledger entries.

## Summary

Keep open: current main and v2026.6.10 still do not provide the requested general agent-loop enforcement for fake tool-call text. The merged Gateway HTTP tool-choice work is useful but scoped to caller-supplied HTTP tools, while core agent turns still need a maintainer security/product decision on warning, suppression, promotion, retry, or fail-closed behavior.

## What This Changes

The issue requests structural enforcement so assistant text that looks like a tool call cannot pass as a completed agent response without a real protocol-visible tool invocation and ledger entry.

## Best Possible Solution

Define a maintainer-approved agent-turn contract that keeps real tool execution protocol-visible, reconciles the existing promotion path with strict fake-tool handling, and adds focused tests/docs for local models, Gateway HTTP tools, provider streams, and final response delivery.

## Reproduction Assessment

Yes, from source inspection. Current main still has warning-only fake-tool-text diagnostics, a structured-only dispatch gate, and no general agent-loop retry or fail-closed enforcement when a turn produces no structured `toolCall`.

## Solution Assessment

Unclear. A protocol boundary is the right layer for security, but maintainers need to choose the default behavior for fake text, bounded promotion, forced tool choice, retries, suppression, and local-model compatibility before implementation.
