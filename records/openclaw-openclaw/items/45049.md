## Label Justifications

- `P1`: The report covers a real agent/tool workflow failure with security and delivery implications for local-model users, but it is not an emergency outage of all core runtime paths.
- `impact:security`: The issue is about enforcing the structural boundary between assistant text and real tool calls that pass through policy and audit hooks.
- `impact:message-loss`: When the model emits a fake tool call or fake result as final text, the intended tool action is effectively not delivered to the tool pipeline.
- `impact:session-state`: The failure creates transcript and tool-ledger drift: user-visible text can imply a tool call or result that the session state never recorded.

## Summary

Keep open. Current main has useful partial mitigations for standalone serialized tool-call text and the merged gateway `tool_choice` work, but the reported mixed prose/JSON fake-call path is still diagnosed as text rather than enforced, retried, or routed through a maintainer-approved hard policy.

## What This Changes

The issue asks OpenClaw to prevent agents from simulating tool calls in assistant text and to enforce real structured tool invocation when tools are clearly required.

## Best Possible Solution

Keep this as the canonical narrow bug for fake tool-call text in the agent loop, with any hard retry or response-gate policy designed alongside the broader enforcement work rather than hidden in a provider-specific shim.

## Reproduction Assessment

Yes, at source level. Current main only promotes recognized standalone serialized tool-call formats and otherwise logs tool-shaped assistant text as ordinary text, while the issue discussion includes concrete LiteLLM/vLLM reproductions for mixed prose or fabricated tool-result output.

## Solution Assessment

Unclear. The merged gateway `tool_choice` fix and current text-call promotion are useful partial mitigations, but the best full fix needs maintainer product/security direction on forced retries, rejection, deterministic routing, and compatibility with provider-specific tool-call formats.
