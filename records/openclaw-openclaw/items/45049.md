## Summary

Keep open: current main and v2026.6.8 contain useful diagnostics, sanitization, and Gateway HTTP `tool_choice` enforcement, but the internal agent-loop policy for simulated or plain-text tool-call behavior remains unresolved.

## What This Changes

The issue asks OpenClaw to enforce real structured tool invocation instead of accepting or delivering assistant text that simulates tool use.

## Best Possible Solution

Define one core policy boundary for internal agent turns: structured tool calls enter the tool pipeline, while simulated text is rejected, retried, or allowed only through an explicit maintainer-approved compatibility path.

## Reproduction Assessment

Yes. Current main and v2026.6.8 show internal plain-text tool-call compatibility plus a diagnostic path for unsupported shaped text, and the issue comments provide concrete shipped-version reproductions.

## Solution Assessment

No. The merged Gateway `tool_choice` implementation is a useful partial fix, but it does not settle the internal agent-loop protocol boundary requested here.
