Full review comments:

- **[P1] Preserve the existing OAuth redirect on upgrade:** `src/agents/mcp-oauth.ts:47`
  - body: This changes the fallback redirect URI for every OAuth MCP server that has no explicit `oauth.redirectUrl`. Stored dynamic client registrations are keyed only by server name and URL, so an existing client registered with `http://127.0.0.1:8989/oauth/callback` can later be reused with `http://localhost:8989/oauth/callback`; OAuth servers commonly require registered redirect URI exact match apart from loopback port. Keep the old default for existing client info or add a tested fallback/migration instead of replacing it unconditionally.
  - confidence: 0.86
- **[P2] Prove the OAuth error path with a failing regression:** `src/agents/mcp-http-fetch.test.ts:201-224`
  - body: The new test uses a foreign response that still has a body. Current main's `buildManagedMcpResponse` already wraps bodyful responses in a global `Response`, so this case does not reproduce the reported `[object Response]` error from a JSON DCR failure. Add a regression that fails against current main for the real MCP SDK OAuth path, or move the normalization to the path that actually passes a foreign `Response` to `parseErrorResponse`.
  - confidence: 0.78
