The linked merged change repairs outbound replies after controller restart, while this report contains conflicting signals—claimed zero runs alongside `model_call:started`—and substantial plugin/socket-exhaustion confounders, so conservative maintainer follow-up is warranted.

## What This Changes

Investigate a 2026.6.5 upgrade-associated WhatsApp outage where inbound messages do not complete agent processing, plus recovery friction when newer config metadata blocks downgrade.

## Best Possible Solution

Ship and document the existing intentional-downgrade override, then reproduce the WhatsApp message-to-run flow on a clean current-main setup with optional plugins disabled and healthy network resources; fix the shared queue, model-call, or lifecycle owner identified by that proof rather than extending the outbound successor-socket workaround.
