## Summary

Keep open. Current main still has no route/ACP binding-level `tools.allow` or `tools.deny` surface: the config schema rejects it, typed binding records cannot carry it, and tool filtering has no binding-scoped policy step.

## What This Changes

The issue requests optional binding-level `tools.allow` and `tools.deny` policy for route and ACP bindings so operators can restrict tools per channel or group.

## Best Possible Solution

Add an end-to-end additive binding policy surface: schema/types, route and ACP propagation, runtime enforcement in the shared policy pipeline, docs, and focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: strict route/ACP binding schemas omit `tools`, and downstream route/ACP records plus the policy pipeline have no place to carry or enforce binding-scoped policy.

## Solution Assessment

Unclear pending maintainer precedence policy. The declarative binding-policy direction is reasonable, but a schema-only change like the closed related PR is not enough because it would accept config without runtime enforcement.
