## Summary

Keep open: current main and latest release still do not accept or enforce `bindings[].tools`, and the linked schema-only PR was closed because it would validate a security-sensitive policy without making denied tools unavailable.

## What This Changes

Request to add per-route/ACP binding `tools.allow`/`tools.deny` so specific channels or groups can restrict tool access independently of global or per-agent policy.

## Best Possible Solution

Define or decline one canonical binding-scoped tool-policy contract; if accepted, implement schema/types, matched-binding propagation, embedded and gateway enforcement, ACP behavior, docs, and regression coverage together.

## Reproduction Assessment

Yes. Source inspection shows strict route/ACP binding schemas omit `tools`, and the embedded/gateway policy pipelines never receive a binding-scoped policy payload.

## Solution Assessment

No. A schema-only change is not the best solution; the maintainable path needs a product/security decision plus end-to-end enforcement across route, gateway, embedded, and ACP paths.
