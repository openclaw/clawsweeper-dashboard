## Summary

Keep open. Current main and the latest release still rethrow Bun’s direct miss for the optional warning-filter import, and the older related report/closed fix attempt describes the same unresolved bootstrap failure rather than a merged implementation.

## What This Changes

Running OpenClaw under Bun fails at startup because optional `./dist/warning-filter.*` misses are not recognized as direct module-not-found errors and are rethrown.

## Best Possible Solution

Land a narrow launcher fix that recognizes Bun direct module misses for the optional warning-filter import, preserves transitive dependency failure reporting, and adds targeted launcher/Bun regression coverage plus a changelog entry.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: current main and v2026.5.22 still import missing optional `./dist/warning-filter.*` files and only recognize Node-style direct misses. I did not run a live Bun command in this read-only pass.

## Solution Assessment
