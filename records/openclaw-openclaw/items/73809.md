## Summary

Keep this PR open: current main still lacks the same-key isolated heartbeat transcript archive path, while the PR implements and tests the narrow fix. The only identified blocker is the repo-required user-facing changelog entry before merge.

## What This Changes

The PR captures the previous isolated heartbeat session before overwriting the store, archives that old transcript with `reason: "reset"`, and adds a regression test for same-key isolated heartbeat rotation.

## Best Possible Solution

Land the heartbeat-owned fix after adding the required changelog entry and keeping the targeted regression plus changed-gate validation green; after merge, #65564 can close as implemented.

Yes. ## Solution Assessment

Yes, with one policy cleanup. Capturing `previousSessionId` before the overwrite and reusing `archiveRemovedSessionTranscripts` with `reason: "reset"` is the narrow maintainable fix because it preserves existing live-reference protection and leaves unrelated heartbeat behavior alone.
