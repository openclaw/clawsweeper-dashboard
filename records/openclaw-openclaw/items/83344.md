## Summary

Current main and the latest release still match the reported WebChat refresh failure class: `chat.history` reads a bounded raw tail before visible projection, preview metadata is tail-bounded, and composer draft/queue state remains in memory. Keep open because the directly linked implementation PR is still open and intentionally leaves preview/title metadata out of scope.

## What This Changes

WebChat refresh should restore recent visible chat history, avoid stale internal preview metadata, and preserve composer draft/queued messages when transcript tails are dominated by internal tool traffic.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/83992 or a replacement for history backfill plus composer persistence, then keep or split the remaining preview/title metadata work explicitly.

## Reproduction Assessment

Yes, at source level: current main still reads a bounded raw transcript tail before visible-message projection and keeps queue/draft state in component memory. I did not run a live browser F5 repro, so this is source-reproducible rather than fully reproduced.

## Solution Assessment

Unclear for the whole issue: the linked PR is a reasonable scoped fix for history backfill and same-tab composer persistence, but preview/title metadata remains out of scope and storage/queue replay semantics need maintainer acceptance.
