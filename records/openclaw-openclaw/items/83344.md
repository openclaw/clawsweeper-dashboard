## Summary

Keep open: current main still has a source-supported WebChat refresh failure class where bounded raw transcript tails are projected after internal/silent records are selected, and related queue/draft refresh work remains open for maintainer scoping.

## What This Changes

Ensure WebChat refresh restores recent visible conversation history and accurate session preview metadata when the transcript tail is dominated by internal tool or silent records.

## Best Possible Solution

Fix chat.history and session preview to gather enough display-visible rows before applying the WebChat window, then handle queue/draft persistence through the related refresh-state track.

## Reproduction Assessment

Yes, source inspection gives a concrete current-main path: the UI asks for a 100-row chat.history window, the gateway selects a raw 100-row tail before projection, and preview readers scan bounded tail windows. A live browser F5 repro was not run here.

## Solution Assessment

Unclear for the whole issue: the history/preview part is a narrow maintainable bug fix, but queue/draft retention and long-history UX need maintainer scoping with https://github.com/openclaw/openclaw/issues/51549, https://github.com/openclaw/openclaw/pull/73154, and https://github.com/openclaw/openclaw/issues/77136.
