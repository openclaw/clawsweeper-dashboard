## Summary

Keep open: current main still omits `apache-arrow` from the `memory-lancedb` package while the locked LanceDB dependency declares it as a peer, and open PR #76919 already references this issue as its implementation path.

## What This Changes

The issue reports that `openclaw ltm` hangs because the bundled `memory-lancedb` plugin packages `@lancedb/lancedb` without its required `apache-arrow` peer dependency.

## Best Possible Solution

Keep the fix in `memory-lancedb` packaging by declaring LanceDB's Arrow peer directly and adding contract coverage so packaged `ltm` installs resolve the required dependency without moving LanceDB ownership into core.

## Reproduction Assessment

No live high-confidence CLI reproduction was established. Source inspection gives a clear path: documented `openclaw ltm` commands load the `memory-lancedb` runtime, that runtime imports `@lancedb/lancedb`, and the package manifest omits the peer dependency the lockfile says LanceDB requires.

## Solution Assessment

Yes. The narrow maintainable solution is a plugin-owned dependency declaration plus package/runtime dependency contract coverage; current core memory and Gateway ownership should stay unchanged.
