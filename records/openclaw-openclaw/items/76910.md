## Summary

Keep open: current main still omits `apache-arrow` from the bundled `memory-lancedb` package while the `ltm` CLI path reaches `@lancedb/lancedb`, and an open linked PR already uses closing syntax for this issue.

## What This Changes

The issue reports that `openclaw ltm` hangs because the bundled `memory-lancedb` plugin does not declare LanceDB's required `apache-arrow` peer dependency.

## Best Possible Solution

Review and land or reject PR #76919; the end state should keep the dependency fix inside `memory-lancedb` packaging and contract coverage rather than moving LanceDB ownership into core.

## Reproduction Assessment

No live high-confidence CLI reproduction was established. Source inspection gives a clear path: the documented `openclaw ltm` command loads `memory-lancedb`, which imports `@lancedb/lancedb`, while the plugin package omits that package's required `apache-arrow` peer.

## Solution Assessment

Yes, the narrow maintainable solution is to declare and test the missing plugin-owned runtime dependency. Core Gateway memory behavior should stay unchanged unless package validation exposes a shared dependency-contract gap.
