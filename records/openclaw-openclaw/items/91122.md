## Summary

Keep open: current main and the latest release still expose `channel.inbound` without the legacy `channel.turn` runtime alias, and the linked open PR owns the compatibility decision.

## What This Changes

The issue requests restoring a backward-compatible `core.channel.turn.*` alias for older channel plugins after the runtime API was renamed to `core.channel.inbound.*`.

## Best Possible Solution

Review and either land or explicitly reject https://github.com/openclaw/openclaw/pull/91245, keeping `channel.inbound` as the canonical API while preserving older shipped plugin behavior if maintainers accept that compatibility contract.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: old plugins that dereference `core.channel.turn.run` will hit an absent `turn` key because current main only exposes `channel.inbound.*`. I did not run a credentialed Feishu or BlueBubbles live channel in this read-only review.

## Solution Assessment

Yes, the linked PR's same-object alias is the narrowest maintainable repair if maintainers accept the compatibility contract. Updating individual plugins would help specific channels but would not protect other older external plugins compiled against the old runtime surface.
