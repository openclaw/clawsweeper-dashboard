## Summary

Keep open: current main and the latest release still expose `channel.inbound` without the legacy `channel.turn` alias, and https://github.com/openclaw/openclaw/pull/91245 is the open closing PR for this compatibility path. ## What This Changes

Request restoring deprecated `core.channel.turn.*` as a compatibility alias for legacy channel plugins after the runtime moved to `core.channel.inbound.*`.

## Best Possible Solution

Review and either land a focused object-identical alias with deprecation docs/tests, or make a no-shim decision and close the linked compatibility work together.

## Reproduction Assessment

## Solution Assessment

Yes if maintainers choose compatibility: an object-identical alias in `createRuntimeChannel()` plus SDK type/mock/docs coverage is the narrow owner-boundary fix. If maintainers reject the alias contract, the better solution is plugin migration and closing the linked alias PR intentionally.
