## Summary

Keep open: current main and the latest release still omit the deprecated `channel.turn` runtime alias, and the linked open PR owns the compatibility repair before this issue can close.

## What This Changes

The issue requests restoring the deprecated `core.channel.turn.*` runtime surface as a compatibility alias for plugins compiled before the `channel.inbound.*` rename.

## Best Possible Solution

Review and resolve https://github.com/openclaw/openclaw/pull/91245; if maintainers accept the compatibility contract, land the object-identical alias while keeping `channel.inbound.*` canonical for new plugin code.

## Reproduction Assessment

Yes. Current main exposes `channel.inbound.*` without a `channel.turn` member, and the runtime mock test asserts `turn` is absent, so an old compiled plugin calling `core.channel.turn.run()` has a source-level undefined-property path.

## Solution Assessment

Yes technically, if maintainers accept the compatibility contract. Restoring the alias at the plugin runtime owner boundary is narrower than patching affected plugins one by one, with the remaining decision being how broad and long-lived the legacy `turn` surface should be.
