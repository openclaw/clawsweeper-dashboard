## Summary

Keep open: current main and v2026.6.8 still expose `channel.inbound` without the legacy `channel.turn` runtime alias, and the linked open PR owns the compatibility decision before this issue can close.

## What This Changes

The issue requests restoring the legacy `core.channel.turn.*` runtime surface as a compatibility alias for `core.channel.inbound.*` after old channel plugins began failing on inbound messages.

## Best Possible Solution

Refresh and review https://github.com/openclaw/openclaw/pull/91245; if maintainers accept the compatibility contract, land the object-identical alias while keeping `channel.inbound.*` canonical for new plugin code.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: old plugins that dereference `core.channel.turn.run` hit an absent `turn` key because current main only exposes `channel.inbound.*`; I did not run a credentialed Feishu or BlueBubbles live channel.

## Solution Assessment

Yes, if maintainers accept the compatibility contract. Restoring the alias at the plugin runtime owner boundary is narrower than patching one affected plugin at a time, but the compatibility policy decision remains open.
