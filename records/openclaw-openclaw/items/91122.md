## Summary

Keep open: current main and the latest stable release still expose `channel.inbound` without the legacy `channel.turn` runtime alias, and a focused open PR already owns the compatibility decision.

## What This Changes

The issue requests restoring `core.channel.turn.*` as a backward-compatible alias for `core.channel.inbound.*` so older channel plugins do not fail inbound dispatch.

## Best Possible Solution

Keep this issue open while maintainers review and either land or reject the linked compatibility-alias PR, without starting a duplicate fix lane.

## Reproduction Assessment

Yes, by source inspection: current main and `v2026.6.8` lack `channel.turn`, while published older channel plugins call `core.channel.turn.run()`. I did not run a live Feishu or BlueBubbles transport setup.

## Solution Assessment

Yes if maintainers accept the compatibility contract: restoring `channel.turn` as the same runtime object as `channel.inbound` is the narrow core-boundary fix. Updating only one plugin would leave other old compiled channel plugins exposed.
