## Summary

## What This Changes

The issue requests restoring the legacy `core.channel.turn.*` plugin-runtime alias after the runtime channel API was renamed to `core.channel.inbound.*`.

## Best Possible Solution

Review and resolve https://github.com/openclaw/openclaw/pull/91245; if maintainers accept the compatibility contract, land the identity alias while keeping `channel.inbound.*` canonical for new plugin code.

## Reproduction Assessment

Yes. Source inspection shows current main exposes `channel.inbound.*` but no `channel.turn`, so an older plugin calling `core.channel.turn.run()` will hit an undefined property; no live Feishu or BlueBubbles credentials were exercised in this read-only review.

## Solution Assessment

Yes technically, if maintainers accept the compatibility contract. Restoring an object-identical alias at the plugin runtime boundary is narrower than patching affected plugins one by one, but the API compatibility policy decision remains open.
