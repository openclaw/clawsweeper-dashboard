## Summary

Keep this open: current `main` and `v2026.6.6` still expose `channel.inbound` without the legacy `channel.turn` alias, and an open PR already closes this issue by restoring that compatibility surface. The exact Feishu `2026.5.28` claim is mixed, but older Feishu and BlueBubbles packages confirm the central legacy-call failure mode.

## What This Changes

The issue reports that old channel plugins compiled against `core.channel.turn.*` break after the runtime API was renamed to `core.channel.inbound.*`.

## Best Possible Solution

Keep this issue open while maintainers review and either land or reject the linked compatibility-alias PR.

## Reproduction Assessment

Yes from source: current main and `v2026.6.6` expose `channel.inbound` without `channel.turn`, and published older channel packages still call `core.channel.turn.run`. I did not run a live Feishu or BlueBubbles environment.

## Solution Assessment

Yes, if maintainers accept the compatibility contract: restoring `channel.turn` as the same object as `channel.inbound` is the narrow core-boundary fix. Updating only one affected plugin would leave other old compiled channel plugins exposed.
