## What This Changes

The issue reports that QQ Bot group messages are processed inbound but replies and sessions_send announcements are routed to webchat instead of back to the QQ group.

## Best Possible Solution

Land a scoped QQ Bot route-persistence fix with focused sessions_send regression coverage and redacted live QQ Bot group proof, while preserving generic webchat fallback behavior for other channels.

## Reproduction Assessment

No high-confidence live reproduction was run in this review. The source path is reproducible: QQ Bot builds the correct group target, but current main does not pass it through `updateLastRoute` and does not opt QQ Bot into stored announce-target lookup.

## Solution Assessment

Yes, the best fix direction is narrow plugin-owned route persistence plus metadata-gated stored announce lookup and regression coverage. A broad core fallback change would be more likely to hide channel-owned metadata bugs.
