## Summary

Keep this PR open, but the current patch should not merge as-is: current main still has the reported audit-noise behavior, while PR head still classifies `auto` from the audit process environment and can suppress exposed-gateway findings. The related duplicate attempts are closed/unmerged, so this remains the active candidate if the security-audit classifier and proof are corrected.

## What This Changes

The PR changes gateway security-audit bind classification, downgrades `gateway.trusted_proxies_missing` to `info` for loopback-equivalent binds, emits it for non-loopback binds, and adds tests plus redacted audit fixtures.

## Best Possible Solution

Classify audit exposure from the same effective bind host the gateway runtime will use, or fail closed when that host cannot be determined, then provide redacted real `openclaw security audit --json` output for loopback, auto-loopback, and auto-exposed cases.

## Reproduction Assessment

Yes. Source inspection shows current main still hard-codes the loopback trusted-proxy finding as `warn`, and PR-head source shows the remaining false-negative path for `auto` by reading process environment instead of the effective runtime bind host.

## Solution Assessment

No. The severity split is a plausible fix direction, but this implementation still uses a heuristic that can diverge from runtime bind resolution and the submitted proof does not demonstrate the claimed effective-host plumbing.
