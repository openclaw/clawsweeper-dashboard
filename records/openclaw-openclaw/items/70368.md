## Summary

Keep open: this is the canonical live PR for the closed loopback trusted-proxy audit issue, but the current branch is not merge-ready because it can suppress critical gateway exposure findings for ambiguous `auto` binds and it lacks after-fix real behavior proof from the contributor.

## What This Changes

The PR adds a gateway bind classifier, uses it to tier several gateway security-audit findings for loopback-equivalent versus non-loopback binds, and extends gateway audit tests.

## Best Possible Solution

Revise the branch toward a conservative effective-bind audit model: downgrade only definitely local-only exposure, fail closed for ambiguous `auto`, keep same-host reverse-proxy guidance visible, and leave changelog/docs alignment to the final landing path.

## Reproduction Assessment

Yes. Current main source and tests still hard-code the loopback trusted-proxy finding as `warn`, and the PR regression is source-reproducible by comparing the new `auto` classifier with runtime `resolveGatewayBindHost` fallback behavior; I did not run tests because this was a read-only review.

## Solution Assessment

No. The branch is directionally useful, but classifying security-audit exposure from configured bind strings and the auditor process environment is not the safest fix; the audit should use an effective/probed bind result or fail closed when locality cannot be proven.
