## What This Changes

This PR changes gateway security-audit bind classification and trusted-proxy severity handling, adds tests, and commits redacted audit-evidence fixtures.

## Best Possible Solution

Classify gateway audit exposure from the effective runtime bind host, or fail closed for ambiguous `auto` and fallback cases, while keeping non-loopback exposure findings loud and providing redacted real audit output for the covered scenarios.

Yes. Source inspection of live PR head shows the classifier still depends on the audit process environment and config-only custom loopback checks, while runtime bind resolution can fall back to `0.0.0.0`; I did not run tests because this is a read-only review.

## Solution Assessment

No. The severity split is a plausible direction, but the implementation should use the effective runtime bind host or fail closed before security-audit gates suppress or downgrade exposure findings.
