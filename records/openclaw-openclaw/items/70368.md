Keep this PR open: current main still has the old loopback `warn` behavior and the duplicate PRs are closed, so this remains the canonical live fix path. The PR changes gateway security-audit bind classification for `gateway.trusted_proxies_missing` and sibling gateway exposure findings, adds a loopback-equivalence helper and tests, and commits redacted audit JSON fixtures.

No. The intended severity split may be reasonable, but this implementation still uses a process-environment heuristic for `auto` instead of the effective runtime bind host, and the submitted proof does not demonstrate the claimed corrected path.
