## Summary

Keep open. The PR addresses a real gateway security-audit noise/security taxonomy problem, but its current classifier can suppress critical exposure findings for ambiguous `bind="auto"`, and the contributor still has not provided after-fix real behavior proof.

## What This Changes

The PR adds a gateway bind classifier, uses it to tier several gateway security-audit findings for loopback-equivalent versus non-loopback binds, and expands gateway audit tests.

## Best Possible Solution

Revise toward a conservative effective-bind audit model: downgrade only definitely local-only exposure, fail closed for ambiguous `auto`, preserve same-host reverse-proxy guidance, and keep focused regression coverage.

## Reproduction Assessment

Yes, source-level. Current main still hard-codes the loopback trusted-proxy finding as `warn`, and the PR regression is visible by comparing the new `auto` classifier with runtime bind resolution; I did not run tests because this read-only review must not create artifacts.

## Solution Assessment

No. The branch is directionally useful, but classifying exposure from the configured bind string and auditor process environment is not the safest fix; use an effective/probed bind result or fail closed when locality cannot be proven.
