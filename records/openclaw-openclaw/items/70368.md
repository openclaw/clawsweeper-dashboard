## Summary

Keep open: current main has not implemented the requested audit-severity change, and the latest branch still has a security-audit false-negative risk around `gateway.bind="auto"`; the added proof snapshots do not resolve that disputed path.

## What This Changes

The PR adds a loopback-equivalent gateway bind classifier, applies it to gateway security-audit findings, expands audit tests, and commits redacted audit-evidence fixtures.

## Best Possible Solution

Drive the audit classification from the same effective bind host used by gateway startup, or fail closed for ambiguous `auto`, then regenerate redacted real audit output for loopback, bare-metal auto, and exposed-auto cases.

## Reproduction Assessment

Yes. Source inspection is enough: PR head classifies `auto` using `isContainerEnvironment()`, while runtime `auto` can resolve to `0.0.0.0`; I did not run tests because this is a read-only review.

## Solution Assessment

No. The severity split is useful, but the implementation should classify from the effective runtime bind host or fail closed before security-audit gates depend on it.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Fail closed for auto bind classification:** `src/config/gateway-bind.ts:41-42`
  - body: `bind="auto"` is still treated as loopback whenever the audit process is not in a container, but runtime can resolve the same mode to `0.0.0.0` when `127.0.0.1` cannot be bound. Because this predicate now suppresses `gateway.bind_no_auth` and `gateway.control_ui.allowed_origins_required` and downgrades `gateway.trusted_proxies_missing`, exposed deployments can lose critical/warn audit findings; pass the effective bind host into the audit or keep `auto` non-loopback unless locality is proven.
  - confidence: 0.93
- **[P3] Force the bare-metal auto proof path:** `tools/audit-evidence/capture.sh:105-106`
  - body: The `auto-bind-non-container` fixture is generated without forcing container detection false, so when this script runs in a container it records the same exposed result as the simulated-container case. That does not prove the bare-metal `auto` behavior that the classifier uses to downgrade security findings.
  - confidence: 0.87
