## Summary

Keep open: current main has not implemented the requested audit severity split, and the PR remains useful, but the latest diff has a blocking security-audit false-negative risk plus missing real behavior proof.

## What This Changes

The PR adds a gateway bind classifier, applies it to gateway security-audit severity and suppression decisions for loopback-equivalent versus non-loopback binds, and expands gateway audit tests.

## Best Possible Solution

Revise toward an effective-bind audit model: downgrade only definitely local-only cases, fail closed for ambiguous `auto`, keep same-host reverse-proxy guidance visible, and require redacted real CLI/audit output before merge.

## Reproduction Assessment

Yes, source-reproducible. Current main still hard-codes the loopback finding as `warn`, and the PR regression is visible by comparing the new `auto` classifier with runtime `resolveGatewayBindHost` fallback behavior; I did not run tests because this was a read-only review.

## Solution Assessment

No. The branch is directionally useful, but classifying security-audit exposure from configured bind strings and the auditor process environment is not the safest fix; use an effective/probed bind result or keep `auto` non-loopback unless locality is proven.
