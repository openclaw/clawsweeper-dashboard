## Summary

Keep open: the branch is still directionally useful, but the latest head still derives `auto` exposure from the audit process environment and uses that predicate to suppress or downgrade gateway security-audit findings. The newly added audit JSON is not enough proof because the nominal non-container case does not actually force a non-container environment.

## What This Changes

The PR changes gateway security-audit findings for missing trusted proxies by classifying loopback-equivalent binds as lower severity, non-loopback binds as warnings, and adds audit evidence fixtures.

## Best Possible Solution

Rework the audit to classify exposure from the same effective bind host used by gateway startup, or fail closed for ambiguous `auto`, then regenerate redacted real `openclaw security audit --json` proof for loopback and exposed-auto paths.

## Reproduction Assessment

Yes. Source inspection is enough: current main still warns on literal loopback, and PR head still classifies `auto` with `isContainerEnvironment()` while runtime resolution can produce `0.0.0.0`; I did not run tests because this was a read-only review.

## Solution Assessment

No. The useful part is the loopback/non-loopback severity split, but the implementation should use an effective/probed bind result or fail closed for `auto` before those security-audit gates depend on the classifier.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Fail closed for auto bind classification:** `src/config/gateway-bind.ts:41-42`
  - body: `bind="auto"` is still treated as loopback whenever the audit process is not in a container, but runtime can resolve the same mode to `0.0.0.0` when `127.0.0.1` cannot be bound. Because this predicate now suppresses `gateway.bind_no_auth` and `gateway.control_ui.allowed_origins_required` and downgrades `gateway.trusted_proxies_missing`, exposed deployments can lose critical/warn audit findings; pass the effective bind host into the audit or keep `auto` non-loopback unless locality is proven.
  - confidence: 0.93

  - body: The `auto-bind-non-container` fixture is generated without forcing container detection false, so when this script runs in a container it records the same exposed result as the simulated-container case. That does not prove the bare-metal `auto` behavior that the classifier uses to downgrade security findings.
  - confidence: 0.88
