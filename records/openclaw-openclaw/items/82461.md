## Summary

Keep open. Current main already defines and honors `agents.defaults.verboseDefault`, but the agent-facing `gateway config.patch` path still blocks that setting by design, so changing it needs maintainer product/security judgment rather than conservative cleanup.

## What This Changes

Request to persist verbose mode across sessions by allowing agent-facing config writes to `agents.defaults.verboseDefault` or by adding an equivalent persistent verbose setting.

## Best Possible Solution

Make an explicit Gateway-tool policy decision; if approved, allowlist the exact global and per-agent verbose default paths with guard tests and docs, while coordinating broader policy with https://github.com/openclaw/openclaw/issues/77439 and https://github.com/openclaw/openclaw/issues/77886.

## Reproduction Assessment

Yes. Source inspection shows a patch changing `agents.defaults.verboseDefault` becomes a changed leaf outside `ALLOWED_GATEWAY_CONFIG_PATHS`, so `assertGatewayConfigMutationAllowed` throws before the Gateway RPC runs.

## Solution Assessment

Unclear until maintainers decide the trust boundary. If this path should be agent-tunable, reusing the existing `verboseDefault` config is narrower than adding a second persistent verbose key.
