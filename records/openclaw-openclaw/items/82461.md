## Summary

Close as duplicate/superseded: the remaining work is the protected agent-facing config-write policy, not a missing verbose default schema field, and that policy is already tracked by open canonical issues for configurable allowlists and owner-approved protected changes.

## What This Changes

Request to persist verbose mode across sessions by allowing agent-facing `gateway config.patch` to set `agents.defaults.verboseDefault`, or by adding an equivalent persistent verbose option.

## Best Possible Solution

Keep the design discussion on the canonical config-write policy issues and include `agents.defaults.verboseDefault` and per-agent verbose defaults as explicit candidate paths there instead of adding a second verbose config key.

## Reproduction Assessment

Yes. Source inspection on current main shows `agents.defaults.verboseDefault` is outside `ALLOWED_GATEWAY_CONFIG_PATHS`, and the guard throws the reported protected-path error before the Gateway RPC runs.

## Solution Assessment

Yes for cleanup. Closing this path-specific satellite in favor of the open policy trackers is the narrowest maintainable outcome; implementation should happen through an approved config-write policy rather than a new duplicate verbose setting.
