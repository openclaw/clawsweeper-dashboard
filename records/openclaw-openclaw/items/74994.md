## Summary

Keep open. The helper approach is useful, but the patch still leaves the documented deleted-cwd `openclaw tui` path exposed through PATH bootstrap and later TUI startup, and it is missing the required user-facing changelog entry.

## What This Changes

The branch exports a `safeCwd()` helper, routes home-dir, main-module, runtime dotenv, and CLI dotenv cwd reads through it, and adds deleted-cwd home-dir tests.

## Best Possible Solution

Land one consolidated deleted-cwd startup fix across dotenv, home-dir, main-module detection, PATH bootstrap, and TUI startup, with focused regressions and a single Unreleased Fixes changelog bullet.

## Reproduction Assessment

Yes. Related #73676 gives a concrete create/cd/delete/run `openclaw tui` repro, and source tracing shows this PR still leaves raw `process.cwd()` calls on that command path.

## Solution Assessment

No. `safeCwd()` is the right narrow direction, but the proposed patch is incomplete until the remaining startup callers share the same safe cwd handling and the original TUI path has regression coverage.
