## What This Changes

Add Mattermost server-side thread-history backfill when a supported thread turn has no in-memory history after restart or session clear.

## Best Possible Solution

Land one canonical Mattermost plugin fix that backfills supported thread context from Mattermost's thread API only on recovery, bounds processing by historyLimit, and includes live Mattermost proof.

## Solution Assessment

Yes, with scope guards. The best fix is Mattermost-owned recovery backfill through the existing client/history path, but maintainers should finish one open candidate PR with recovery-aware gating and live proof.
