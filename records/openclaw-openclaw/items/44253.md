## Summary

Keep this open: current main and v2026.6.6 still lack a `selfDeny` or delegate-only tool policy, and the current implementation still treats parent tool policy as inherited child-session policy. The request is source-reproducible, but it needs maintainer product and security design because it adds a config surface and an exception to normal non-escalating tool inheritance.

## What This Changes

Request to add a per-agent delegate-only tool policy so an orchestrator can withhold tools from its own model session while still allowing subagents to inherit them.

## Best Possible Solution

Design a maintainer-approved delegate-only policy that removes selected tools only from the parent model surface, keeps normal `deny` absolute, and proves native subagent plus ACP child-session behavior.

## Reproduction Assessment

Yes, source-level. Current schema, docs, runtime policy assembly, and tests show monotonic allow/deny inheritance with no parent-self-only removal stage; I did not run a live gateway scenario.

## Solution Assessment

Unclear. `selfDeny` is a plausible shape, but the best solution depends on maintainer choice of syntax, security semantics, and interaction with inherited allow/deny behavior.
