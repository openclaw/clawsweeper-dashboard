## Summary

Keep open. Current main still has no per-agent `selfDeny`, `delegateOnly`, `allowEscalate`, or equivalent subagents-only tool scope; the strict agent tools schema and the ordered policy pipeline still make the requested config-level guarantee unavailable.

## What This Changes

The issue requests a per-agent tool policy key that hides tools from an orchestrator agent while preserving those tools for spawned subagents.

## Best Possible Solution

Define a maintainer-approved subagents-only tool scope that preserves hard `deny` precedence, hides selected tools from the parent model surface, and documents/tests the full config contract.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to deny a tool through profile or per-agent policy and then try to allow it through `tools.subagents.tools`; current source and docs show the subagent layer only narrows the already-resolved tool set.

## Solution Assessment

Unclear. `selfDeny` or `delegateOnly` is a plausible narrow shape, but the best solution needs maintainer approval for the public config name and security precedence before implementation.
