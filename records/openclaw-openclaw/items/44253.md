## Summary

Keep open. Current main still has no per-agent `selfDeny`, `delegateOnly`, `allowEscalate`, or equivalent subagents-only tool scope; the strict config schema and monotonic tool-policy pipeline still make the requested config-level guarantee unavailable, and the fix needs maintainer approval because it changes security-sensitive tool precedence.

## What This Changes

Requests a per-agent delegate-only tool policy so orchestrator agents can hide tools such as `exec`, `write`, and `edit` from their own model surface while still allowing subagents to inherit them.

## Best Possible Solution

Define a documented and tested subagents-only tool scope that preserves hard deny precedence, removes selected tools from the parent model-visible list, and keeps child eligibility explicit across core, bundled tools, and plugin tools.

## Reproduction Assessment

Yes. Source inspection reproduces the limitation: per-agent/profile deny filtering happens before the final subagent policy, and the strict config schema has no accepted delegate-only key.

## Solution Assessment

Unclear. `selfDeny` or `delegateOnly` is a plausible shape, but the best solution needs maintainer approval for the public config name and security precedence before implementation.
