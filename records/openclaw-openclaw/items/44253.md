## Summary

Keep open: current main and the latest shipped release still expose only one-way tool filtering, and the related issues are adjacent rather than superseding; this remains an authorization-sensitive config/product decision.

## What This Changes

Request a per-agent tool scope such as `tools.selfDeny` or `delegateOnly` so an orchestrator can hide tools from its own model surface while leaving them eligible for subagents.

## Best Possible Solution

Define a maintainer-approved delegate-only tool scope that hides selected tools from the parent model, keeps them eligible only for subagent filtering, preserves hard-deny precedence, and updates schema, docs, metadata, and policy tests together.

## Reproduction Assessment

Yes, at source level. Current main has no accepted `selfDeny` or delegate-only config key, and the policy pipeline only narrows the already-resolved tool set.

## Solution Assessment

Unclear. The requested direction is plausible, but the best fix needs maintainer agreement on the public config name, child-eligible pool semantics, hard-deny precedence, and security review before implementation.

## Security Review

Status: needs_attention

Summary: No patch was reviewed, but the requested feature is authorization-sensitive because it would split parent model-visible tools from child session capabilities.

- **[medium] Preserve hard-deny precedence:** `src/agents/tool-policy-pipeline.ts:188`
  - body: A delegate-only implementation must not let subagent policy restore tools removed by global, provider, agent, sender, sandbox, inherited, or explicit deny policy, or it could widen access to exec, write, or edit.
  - confidence: 0.9

  - body: The per-agent tools schema is strict and currently has no parent-hidden child-eligible field, so adding one needs aligned schema, types, docs, metadata, and compatibility tests.
  - confidence: 0.86
- **[medium] Do not use exec approvals as the split boundary:** `src/infra/exec-approvals.ts:921`
  - body: Exec approvals resolve policy by agent key, not by parent-versus-subagent model surface, so the tool-policy layer needs to carry the boundary explicitly if the feature is accepted.
  - confidence: 0.78
