## Summary

Keep open. Current main still has no parent-hidden but child-eligible tool scope, and the request would add a new security-sensitive public authorization/config contract rather than fix a broken existing contract.

## What This Changes

Request to add a per-agent delegate-only tool policy so an orchestrator can hide tools from its own model surface while keeping them available to worker subagents.

## Best Possible Solution

Define a maintainer-approved delegate-only tool scope that hides selected tools from the parent model, keeps them eligible only for subagent filtering, preserves hard-deny precedence, and updates schema, docs, metadata, and policy tests together.

## Reproduction Assessment

Yes. Source inspection on current main shows the strict per-agent tools schema lacks selfDeny/delegateOnly, the subagent schema is only a global nested tool policy, and the policy pipeline only narrows the already-resolved tool set.

## Solution Assessment

Unclear. A selfDeny/delegateOnly field is plausible, but the best fix needs maintainer agreement on the public config name, child-eligible pool semantics, hard-deny precedence, and security review before implementation.
