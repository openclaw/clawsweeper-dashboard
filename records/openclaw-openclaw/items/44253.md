## Summary

Keep open. Current main still lacks a per-agent parent-hidden, child-eligible tool scope; the existing policy pipeline filters whole tools through inherited deny/allow state, and the related open issues cover adjacent but not superseding behavior.

## What This Changes

Request adds a per-agent `tools.selfDeny` or equivalent config key so an orchestrator can hide powerful tools from its own model surface while still allowing spawned subagents to inherit and use them.

## Best Possible Solution

Define a maintainer-approved delegate-only tool scope that hides selected tools from the parent model, keeps them eligible only for subagent filtering, preserves hard-deny precedence, and updates schema, docs, metadata, and focused policy tests together.

## Reproduction Assessment

Yes, at source level. Current main has no accepted `selfDeny` or delegate-only config key, and the policy pipeline only filters the already-resolved tool set plus inherited child policy.

## Solution Assessment

Unclear. A delegate-only field is plausible, but the best fix needs maintainer agreement on naming, child-eligible pool semantics, hard-deny precedence, and security review before implementation.
