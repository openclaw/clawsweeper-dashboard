## Summary

Keep open. Current main still lacks a parent-hidden but child-eligible tool scope, and the request is a new security-sensitive public config contract rather than a broken existing behavior.

## What This Changes

Request to add per-agent tool policy such as `selfDeny` or `delegateOnly` so an orchestrator cannot directly use selected tools while its subagents can inherit them.

## Best Possible Solution

Define a maintainer-approved delegate-only tool scope that hides selected tools from the parent model, keeps them eligible only for subagent filtering, preserves hard-deny precedence, and updates schema, docs, metadata, and policy tests together.

## Reproduction Assessment

Yes. As a source-level current-main check, the strict per-agent tools schema lacks `selfDeny`/`delegateOnly`, the policy pipeline only narrows the surviving tool set, and the docs state subagent allowlists cannot add back tools removed earlier.

## Solution Assessment

Unclear. `selfDeny` or `delegateOnly` is plausible, but the best fix needs maintainer agreement on the public config name, child-eligible pool semantics, hard-deny precedence, and security review before implementation.
