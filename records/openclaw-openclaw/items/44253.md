## Summary

Keep open: current main and release v2026.6.5 still enforce monotonic tool filtering, so the requested direct-use versus delegated-use boundary remains missing and requires an explicit product/security decision rather than a narrow bug fix.

## What This Changes

Add a per-agent tool policy that blocks selected tools from the orchestrator’s own session while preserving them for its worker subagents.

## Best Possible Solution

Define one lane-scoped direct/delegated authorization contract with explicit deny precedence, execution-time enforcement, persisted capability provenance, effective-policy diagnostics, focused tests, and security documentation.

## Reproduction Assessment

Yes. Current main applies parent policy before subagent and inherited policy, and its diagnostics explicitly state that subagent allow cannot restore a tool removed by earlier filtering.

## Solution Assessment

No, not as only a parent-side `selfDeny` schema filter. The maintainable solution must enforce and audit direct-versus-delegated authority across policy resolution, invocation guards, capability persistence, diagnostics, and documentation.
