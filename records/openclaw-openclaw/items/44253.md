## Summary

Keep open. Current main still lacks a parent-hidden, child-eligible tool scope; source, docs, and tests show subagent policy only narrows the resolved tool set, and related issues do not supersede this authorization/config decision.

## What This Changes

Add a per-agent delegate-only tool policy so an orchestrator can hide tools from its own model surface while leaving them eligible for spawned subagents.

## Best Possible Solution

Define a maintainer-approved delegate-only tool scope that hides selected tools from the parent model, keeps them eligible only for subagent filtering, preserves hard-deny precedence, and updates schema, docs, metadata, diagnostics, and policy tests together.

## Reproduction Assessment

Yes, at source level. Current main has no accepted selfDeny/delegateOnly config key, and the policy pipeline only narrows the already-resolved tool set instead of keeping a separate child-eligible pool.

## Solution Assessment

Unclear. The requested selfDeny/delegateOnly shape is plausible, but the best fix needs maintainer agreement on public config naming, pool semantics, diagnostics, compatibility, and security review before implementation.
