## Summary

Keep open. Current main still lacks a `selfDeny`, `delegateOnly`, or equivalent parent-hidden but subagent-eligible tool scope, and the remaining work is a new security-sensitive config contract rather than a cleanup close candidate.

## What This Changes

Add a per-agent tool policy option that hides selected tools from an orchestrator agent while keeping them available for its spawned subagents.

## Best Possible Solution

Define a maintainer-approved delegate-only tool scope that hides selected tools from the parent model surface, keeps them eligible for subagent filtering, preserves hard deny precedence, and updates schema, docs, metadata, and focused policy tests together.

## Reproduction Assessment

Yes, at source level. Denying or profile-filtering a tool at the parent level and then trying to re-allow it through `tools.subagents.tools` still only narrows the already-resolved tool list on current main.

## Solution Assessment

Unclear. `selfDeny` or `delegateOnly` is plausible, but the best fix needs maintainer agreement on the public config name, child-eligible pool semantics, and security precedence before implementation.
