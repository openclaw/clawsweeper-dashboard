## Summary

Keep open. Current main still lacks a parent-hidden but child-eligible per-agent tool scope, and the related per-spawn `toolsAllow` work does not solve this config-level orchestrator boundary.

## What This Changes

Add a per-agent `tools.selfDeny` or delegate-only policy so an orchestrator can hide tools from its own model while keeping them eligible for spawned subagents.

## Best Possible Solution

Define a maintainer-approved delegate-only tool scope that hides selected tools from the parent model, keeps them eligible only for subagent filtering, preserves hard-deny precedence, and updates schema, docs, metadata, and policy tests together.

## Reproduction Assessment

Yes, at source level. Current main has no accepted `selfDeny` or delegate-only config key, and the policy pipeline only narrows the already-resolved tool set.

## Solution Assessment

Unclear. A selfDeny/delegateOnly field is plausible, but the best fix needs maintainer agreement on the public config name, child-eligible pool semantics, hard-deny precedence, and security review before implementation.
