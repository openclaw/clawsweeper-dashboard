## Summary

Keep open. Current main still lacks a per-agent delegate-only/self-deny tool policy, and the existing schema plus ordered tool pipeline still make the requested config-level guarantee unavailable for orchestrator agents.

## What This Changes

Requests a per-agent tool policy that hides selected tools from the parent agent while preserving those tools for spawned subagents.

## Best Possible Solution

Define a maintainer-approved subagents-only tool scope that preserves hard deny precedence, hides selected tools from the parent model surface, and updates schema, docs, and focused policy tests.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to remove a tool through profile or per-agent policy and then allow it through `tools.subagents.tools`; current code and docs show the subagent layer only narrows the already-resolved tool set.

## Solution Assessment

Unclear. `selfDeny` or `delegateOnly` is a plausible shape, but the best solution needs maintainer approval for the public config name, inheritance model, and security precedence before implementation.
