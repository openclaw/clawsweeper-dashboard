## Summary

Keep open: current main and the latest shipped release still only support one-way tool filtering, while this request needs a new authorization-sensitive config contract rather than a narrow bug fix.

## What This Changes

The issue requests a per-agent delegate-only tool policy so an agent can hide tools from its own model surface while still allowing subagents to inherit them.

## Best Possible Solution

Define a maintainer-approved delegate-only tool scope that hides selected tools from the parent model, keeps them eligible only for subagent filtering, preserves hard-deny precedence, and updates schema, docs, metadata, and policy tests together.

## Reproduction Assessment

Yes, at source level. Current main has no accepted delegate-only config key, and the policy pipeline only narrows the already-resolved parent-visible tool set before child inheritance.

## Solution Assessment

Unclear. The requested direction is plausible, but the durable fix needs maintainer agreement on the public config name, child-eligible pool semantics, hard-deny precedence, and security review before implementation.
