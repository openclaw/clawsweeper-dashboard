## Summary

Keep open. Current main still lacks a parent-hidden but child-eligible tool scope, and this is a new security-sensitive public config contract rather than a broken existing contract.

## What This Changes

The issue requests a per-agent delegate-only tool policy so an orchestrator cannot directly see or call selected tools while its subagents can still inherit and use them.

## Best Possible Solution

Define a maintainer-approved delegate-only tool scope that hides selected tools from the parent model, keeps them eligible only for subagent filtering, preserves hard-deny precedence, and updates schema, docs, metadata, and policy tests together.

## Reproduction Assessment

Yes. Source inspection on current main shows the strict per-agent tools schema lacks selfDeny/delegateOnly, the policy pipeline only narrows the surviving tool set, and the docs state subagent allowlists cannot add back tools removed earlier.

## Solution Assessment

Unclear. A selfDeny/delegateOnly field is plausible, but the best fix needs maintainer agreement on the public config name, child-eligible pool semantics, hard-deny precedence, and security review before implementation.
