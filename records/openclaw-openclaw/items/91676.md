The issue requests an opt-in `skills.workshop.allowSandboxedAgents` setting so sandboxed agents can receive the governed `skill_workshop` tool.

## Best Possible Solution

Keep the secure default, then implement one maintainer-approved opt-in or warning path with schema, docs, focused tests, and explicit sandbox security rationale.

## Solution Assessment

Unclear until maintainers choose the security model: a default-off flag is plausible, but honoring existing sandbox tool policy may be the cleaner owner surface.
