## What This Changes

Add an opt-in path, proposed as `skills.workshop.allowSandboxedAgents`, so sandboxed agents can receive the built-in `skill_workshop` tool.

## Best Possible Solution

Keep default-deny behavior unless maintainers approve one canonical opt-in or warning path, then implement that path with schema/docs/tests and real sandbox proof.

## Solution Assessment

Unclear. The default-off flag is plausible, but the issue itself asks whether existing sandbox tool policy should own construction instead, so maintainers should choose the security/config boundary first.
