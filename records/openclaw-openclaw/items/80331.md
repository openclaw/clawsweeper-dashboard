## Summary

Keep open: the hardening direction is useful, but the current PR head still regresses ENOTDIR-style missing extra-path reads and the external proof is only test-wrapper output, so it is not merge-ready.

## What This Changes

The PR adds symlink-parent and realpath checks around memory extraPaths directory reads, exports fs-safe helpers, and adds readMemoryFile regression tests.

## Best Possible Solution

Keep the symlink-parent and realpath hardening, align the missing-path branch with fs-safe's ENOENT/ENOTDIR contract, add a non-directory-parent regression test, and require redacted runtime proof before merge.

## Reproduction Assessment

Yes, source-reproducible: current main lexically authorizes extra-path directory reads and only rejects a symlink at the final path, so a symlinked intermediate directory can redirect the read. I did not run tests because this review is read-only and test commands create artifacts.

## Solution Assessment

No, not as written: the hardening direction is narrow and maintainable, but the new authorization helper needs to treat ENOTDIR as a missing path and cover that case before it is the best fix.
