## Summary

Keep open: this PR addresses a real current-main extraPaths symlink traversal in the memory read path and the code direction looks targeted, but it is not merge-ready because the external proof is only unit-test output and CI currently has failing lint aggregate checks.

## What This Changes

The branch adds fs-safe parent-symlink and realpath containment checks for memory extra-path reads plus focused readMemoryFile regression tests.

## Best Possible Solution

Land a targeted readMemoryFile fix that rejects symlinked intermediate components for configured extraPaths, with regression tests and contributor-supplied real behavior proof; keep broader race-proof file-open work separate.

## Reproduction Assessment

Yes, there is a high-confidence source reproduction path: configure an extraPaths directory, place a symlinked subdirectory inside it, and request a .md file through that symlink. Current main authorizes by lexical containment and only checks the leaf path, so the redirected read can pass.

## Solution Assessment

Yes, the proposed direction is the narrow maintainable fix for static symlink traversal: reject symlinked parents and require realpath containment before allowing extra-root reads. Full active-race pinning is a separate lower-level primitive rather than a reason to broaden this PR.
