## Summary

Keep open. Current main still has the two source-level gaps this PR addresses: `format` failover reasons still record auth-profile failures, and configured fallbacks are appended without checking tool compatibility; the remaining issues I found are docs/changelog drift, not blocking runtime correctness.

## What This Changes

The PR exempts `format` failover reasons from auth-profile cooldown recording, adds a `requiresTools` fallback filter for explicitly tool-incompatible configured fallback models, passes that flag from embedded agent runs, and adds focused tests.

## Best Possible Solution

Land the focused fallback/cooldown behavior after updating operator-facing failover docs and adding the required changelog entry; keep broader primary-model non-tool execution or rejection behavior under #64175.

## Reproduction Assessment

Yes. Source inspection on current main shows `format` still records as an auth-profile failure reason, and configured fallback candidates are added without a `supportsModelTools` check.

## Solution Assessment

Yes, with small docs and changelog follow-up. The PR uses the existing `supportsModelTools` contract, keeps unknown tool support permissive, and scopes the new skip to embedded runs that require tools.
