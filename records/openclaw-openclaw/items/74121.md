## Summary

Keep open. Current main still records `format` failover reasons as auth-profile failures and still adds configured fallback models without checking tool capability, so the PR is not obsolete. The latest head is now a narrow fallback/cooldown change rather than the blank-message repair described in the body, and it needs docs and changelog cleanup before maintainer merge handling.

## What This Changes

The PR adds a `requiresTools` fallback-resolution path, skips configured fallback models with `compat.supportsTools === false` for embedded agent runs, and exempts `format` failover reasons from auth-profile cooldown recording with focused tests.

## Best Possible Solution

Land the focused fallback/cooldown behavior after updating operator-facing failover docs and adding the required release-note entry, while leaving current main's existing blank-prompt defenses in place and keeping the broader primary-model capability issue separate.
