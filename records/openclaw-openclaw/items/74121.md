## Summary

Keep this PR open, but it needs changes before merge. Current main already covers the format-cooldown and OpenRouter tool-metadata parts of the original problem, while the remaining fallback-filter slice has a correctness issue for raw/no-tool agent runs and the PR body still describes blank-message changes that are no longer in the diff.

## What This Changes

The branch adds a `requiresTools` fallback-candidate filter, passes it from `agent-command`, exempts `format` auth-profile failures from cooldown, and adds focused tests.

## Best Possible Solution

Rebase and narrow the PR to the remaining fallback policy only: if OpenClaw should skip explicitly tool-incompatible fallbacks, derive that from the actual run mode/tool-construction state, preserve raw/no-tool fallback behavior, and leave the already-fixed blank-message, format-cooldown, and OpenRouter metadata work on current main.

## Reproduction Assessment

Yes. Source inspection shows the PR marks every embedded agent fallback chain as tool-required, while current main has raw `modelRun`/`promptMode: "none"` paths that deliberately disable tool construction.

## Solution Assessment

No. Reusing `supportsModelTools()` is a reasonable contract, but the proposed `requiresTools: true` is broader than the runtime condition; the safer fix is to gate fallback filtering on whether the run will actually construct/use tools.
