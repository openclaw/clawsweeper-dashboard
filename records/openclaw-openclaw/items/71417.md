## Summary

Keep open. Current main still resumes a fresh stored `agent:<id>:main` session when `openclaw agent --agent <id>` is invoked without a nonblank `--session-id`, and it still lacks a fresh/no-resume flag or resumed-session disclosure; the open `--new-session` request covers only part of the UX.

## What This Changes

Request that `openclaw agent --agent <id>` avoid or clearly disclose implicit reuse of the persisted agent main session when no explicit session selector is provided.

## Best Possible Solution

Add a backward-compatible fresh/no-resume intent through CLI, local execution, Gateway protocol, and Gateway handler paths, surface resumed-session state, and coordinate the flag-only portion with https://github.com/openclaw/openclaw/issues/54864.

## Reproduction Assessment

Yes. A source-level path is clear on current main: omit a nonblank `--session-id`, target an agent, and the CLI/local/Gateway paths resolve `agent:<id>:main` and reuse a fresh stored `sessionId`; I did not run a live CLI repro in this read-only review.

## Solution Assessment

Unclear pending maintainer UX direction. A backward-compatible fresh/no-resume control plus resumed-session disclosure looks narrower than flipping the default, but the exact flag, metadata, banner, and suppression policy need a maintainer call.
