## Summary

Keep open. Current main and the latest release still allow `openclaw agent --agent <id>` to resolve the agent main session and reuse a fresh stored session without a fresh/no-resume flag or resume disclosure; the related flag request at https://github.com/openclaw/openclaw/issues/54864 overlaps but does not fully own the banner/default-policy decision.

## What This Changes

Request that `openclaw agent --agent <id>` stop silently resuming the most recent agent session by changing the default, adding a fresh-session flag, or disclosing resume behavior.

## Best Possible Solution

Choose a backward-compatible fresh/no-resume contract, then implement it consistently through CLI parsing, local execution, Gateway protocol/handler, JSON metadata, docs, and focused regression tests while coordinating with https://github.com/openclaw/openclaw/issues/54864.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path: run `openclaw agent --agent <id>` twice without a nonblank `--session-id`, and the CLI/Gateway/local paths resolve the agent main session key and can reuse the stored session id; I did not run a live CLI repro in this read-only review.

## Solution Assessment

Unclear pending maintainer UX direction. A backward-compatible fresh/no-resume flag plus resume disclosure looks safer than flipping the default immediately, but the exact flag, banner, JSON metadata, and suppression policy need a maintainer call.
