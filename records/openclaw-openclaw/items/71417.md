## Summary

Keep open. Current main has fixed the adjacent explicit `--session-id` remapping path, but `openclaw agent --agent <id>` with no real session id still resolves to the agent main session, reuses a fresh stored session id, and has no fresh-run flag, Gateway protocol field, resume banner, or docs disclosure.

## What This Changes

Request explicit fresh-run behavior or visible resume disclosure for `openclaw agent --agent <id>` invoked without `--session-id`.

## Best Possible Solution

Preserve resume-by-default compatibility, add an explicit fresh/no-resume control through CLI, local, Gateway, and protocol paths, surface resumed-session metadata or a stderr banner, and document both modes.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: seed a fresh `agent:<id>:main` session entry, invoke `openclaw agent --agent <id> -m ...` without a real `--session-id`, and both local and Gateway paths select the stored session id.

## Solution Assessment

Yes. A backward-compatible flag plus disclosure is the narrowest maintainable direction; changing the default to fresh would risk breaking scripts that intentionally depend on main-session continuity.
