## Summary

Keep open. Current main and v2026.5.18 still reuse the agent main session for `openclaw agent --agent <id>` without a fresh/no-resume flag or resume disclosure, and the related fresh-flag issue does not fully cover the banner/default-policy decision.

## What This Changes

The issue requests a fresh-session/no-resume or resume-disclosure contract for `openclaw agent --agent <id>` when no explicit session selector is supplied.

## Best Possible Solution

Choose a backward-compatible `openclaw agent` resume contract, then implement a fresh-session flag or no-resume mode plus observable resume metadata/banner across CLI, local execution, Gateway protocol/handler, docs, and focused tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: run `openclaw agent --agent <id>` twice without a nonblank `--session-id`, and the CLI/Gateway/local paths resolve the agent main session key and can reuse the stored session id; I did not run a live CLI repro in this read-only review.

## Solution Assessment

Unclear pending maintainer UX direction. A backward-compatible fresh/no-resume flag plus resumed-session disclosure looks safer than flipping the default immediately, but the flag name, banner, JSON metadata, and suppression policy need a maintainer call.
