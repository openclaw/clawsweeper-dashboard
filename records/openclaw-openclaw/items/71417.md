## Summary

Keep open. Current main still reuses a fresh stored agent main session for `openclaw agent --agent <id>` when no nonblank `--session-id` is passed, with no fresh/no-resume control or resume disclosure; the related flag-only request covers part of the UX but this still needs maintainer direction.

## What This Changes

Request that `openclaw agent --agent <id>` stop silently resuming the agent main session by default or expose a fresh-session flag and resume banner.

## Best Possible Solution

Choose one backward-compatible fresh/no-resume contract, then implement it consistently across CLI parsing, local execution, Gateway protocol/handler, JSON metadata, docs, and focused regression tests while coordinating the flag-only request.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: omit a nonblank `--session-id`, pass `--agent`, and the CLI/local/Gateway paths resolve the agent main session key and reuse a fresh stored `sessionId`; I did not run a live CLI repro in this read-only review.

## Solution Assessment

Unclear pending maintainer UX direction. A backward-compatible fresh/no-resume control plus resumed-session disclosure looks safer than flipping the default immediately, but the exact flag, banner, JSON metadata, and suppression policy need a maintainer call.
