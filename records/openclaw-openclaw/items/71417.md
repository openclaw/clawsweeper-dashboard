## Summary

Keep open. Current main still preserves implicit resume-by-default for a fresh stored agent main session, and the remaining UX choice is not fully superseded by the related fresh-session flag request.

## What This Changes

The issue requests an explicit fresh-session control or visible resume disclosure for `openclaw agent` when no session id is provided.

## Best Possible Solution

Preserve backward-compatible resume-by-default behavior, add an explicit fresh/no-resume intent through CLI, local, Gateway protocol, and Gateway handler paths, surface resumed-session state, and align docs plus focused tests while coordinating with https://github.com/openclaw/openclaw/issues/54864.

## Reproduction Assessment

Yes. A source-level path is clear: seed a fresh `agent:<id>:main` entry, then run `openclaw agent --agent <id> -m ...` without a nonblank `--session-id`; current local and Gateway paths resolve that key and reuse its stored session id without a fresh flag or banner.

## Solution Assessment

Unclear pending maintainer UX direction. A backward-compatible fresh/no-resume control plus resume disclosure appears narrower than flipping the default, but the exact flag, alias, metadata, and suppression policy need a maintainer call.
