## Summary

Keep open. Current main still resolves omitted or blank session ids for `openclaw agent --agent <id>` to the agent main session key and reuses a fresh stored session without a CLI fresh-session flag, Gateway protocol intent, resume banner, or docs path.

## What This Changes

The issue requests a way to prevent or clearly disclose implicit resume when `openclaw agent --agent <id>` is run without `--session-id`.

## Best Possible Solution

Preserve resume-by-default compatibility, add an explicit fresh-run control, plumb it through local and Gateway paths, expose resume metadata or a banner, and update docs/tests/changelog.

## Reproduction Assessment

Yes. Seed a fresh `agent:<id>:main` session entry, then invoke `openclaw agent --agent <id> -m ...` without a real `--session-id`; the current local and Gateway resolvers select the stored session id.

## Solution Assessment

Yes. A backward-compatible opt-in fresh flag plus disclosure is the narrowest maintainable direction; flipping the default to fresh would risk breaking scripts that rely on multi-turn agent-main sessions.
