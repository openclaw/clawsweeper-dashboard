## Summary

Keep open: the Feishu multi-bot duplicate-reply problem is still real on current main, but this PR is not merge-ready because it is conflicting, lacks real Feishu proof, and can drop or misroute messages for non-literal-main coordinators.

## What This Changes

The branch adds Feishu mention extraction, dispatch-time main/specialist arbitration, regression tests, and a design note for multi-bot group routing.

## Best Possible Solution

Adopt one Feishu multi-bot mention contract that derives coordination from configured account metadata or the approved inbound metadata path, covers `group` and `topic_group`, avoids substring matching, and includes focused tests plus redacted real Feishu proof.

## Reproduction Assessment

Yes, at source level: current main starts one Feishu monitor per enabled account and dispatches with only that account's bot identity, so sibling bot mentions are not arbitrated. I did not establish a live Feishu tenant reproduction in this read-only review.

## Solution Assessment

No: dispatch-time arbitration is a plausible direction, but this implementation hard-codes the coordinator, skips `topic_group`, and adds unsafe substring matching. A safer solution should derive the coordinator or use the structured metadata approach and prove the real Feishu path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Derive the coordinator account from Feishu config:** `extensions/feishu/src/monitor.ts:38`
  - body: This policy assumes the coordinator account ID is always `main`, but Feishu supports arbitrary account IDs and `channels.feishu.defaultAccount`. If a deployment uses `default` or `router` and mentions that coordinator plus a specialist, both accounts can satisfy the specialist skip branch and the message can be dropped instead of routed once.
  - confidence: 0.91
- **[P2] Apply arbitration to topic groups:** `extensions/feishu/src/monitor.ts:97`
  - body: Feishu `topic_group` is a group chat type in current main, but this guard returns before arbitration for anything other than `group`. Multi-bot topic conversations can therefore keep the duplicate-reply behavior this PR is meant to prevent.
  - confidence: 0.88
- **[P2] Require token boundaries for implicit mentions:** `extensions/feishu/src/bot.ts:259`
  - body: The new fallback matches configured mention patterns with `includes`, so the default `@main` pattern also matches plain text such as `@mainbot` or `@maintenance`. That can turn an unmentioned group message into a dispatch when mention gating should have held it.
  - confidence: 0.86
