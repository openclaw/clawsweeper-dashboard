## Summary

Keep this PR open, but do not merge it as submitted: the Feishu duplicate-reply problem is still present on current main, while the branch has concrete correctness blockers around coordinator selection, topic groups, and mention parsing.

## What This Changes

The PR adds Feishu mentioned-open-id extraction, dispatch-time group mention arbitration between the main account and specialist bot accounts, regression tests, an implicit text-mention fallback, and a design note.

## Best Possible Solution

Implement a Feishu-owned arbitration layer on current main that derives the coordinator from Feishu account configuration, covers group and topic_group, ignores broadcast @all, preserves debounce/dedupe/reaction/broadcast behavior, and includes regression tests plus a changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: run multiple enabled Feishu bot accounts in one group, send a structured mention to a sibling bot while another account can dispatch without being mentioned, and current main has only account-local mention gating and per-account dedupe.

## Solution Assessment

No. Dispatch-time arbitration is the right direction, but the submitted patch hard-codes the coordinator, misses topic_group, uses substring implicit mentions, and needs to preserve current @all filtering before it is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Derive the coordinator account from Feishu config:** `extensions/feishu/src/monitor.ts:35`
  - body: This hard-codes the coordinator as `main`, but Feishu account IDs and default-account selection are configurable. In deployments without a literal `main`, multi-specialist mentions can make every mentioned specialist skip while no configured coordinator handles the message.
  - confidence: 0.91
- **[P2] Apply arbitration to topic groups:** `extensions/feishu/src/monitor.ts:94-96`
  - body: Current Feishu handling treats both `group` and `topic_group` as group chats, but this policy returns early unless the raw chat type is exactly `group`. Multi-bot topic groups would keep the duplicate-reply behavior this PR is meant to prevent.
  - confidence: 0.86
- **[P2] Require token boundaries for implicit mentions:** `extensions/feishu/src/bot.ts:256-260`
  - body: The implicit fallback uses substring matching, so a fallback pattern such as `@main` can match plain text like `@maintenance` or `@mainbot` and mark the bot as mentioned without a real structured Feishu mention. Match exact mention tokens or drop this fallback path.
  - confidence: 0.88
- **[P2] Filter broadcast mentions in the shared extractor:** `extensions/feishu/src/bot.ts:197-205`
  - body: Current main deliberately excludes Feishu broadcast `@all`/`@_all` from bot-mention handling. The new shared extractor should filter those broadcast mentions before feeding `checkBotMentioned` and arbitration, otherwise rebasing this patch can reintroduce broadcast messages as addressed bot mentions.
  - confidence: 0.82
