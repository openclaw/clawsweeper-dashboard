## Summary

Keep this PR open. Current main still lacks cross-account Feishu group mention arbitration for ordinary inbound dispatch, but the submitted branch is not merge-ready because it can drop coordinated messages in non-`main` deployments and can false-trigger implicit text mentions.

## What This Changes

The PR adds Feishu group mention extraction and arbitration across bot accounts, modifies inbound receive dispatch, extends Feishu mention tests, and adds a design note.

## Best Possible Solution

Implement a narrow Feishu-owned arbitration layer before ordinary dispatch that uses structured mentioned bot open_ids, derives the coordinator from Feishu account configuration, preserves ordinary and broadcast dedupe semantics, and covers the three group mention policies with regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: configure multiple enabled Feishu bot accounts in one group, set the coordinator/group path so it can dispatch without being mentioned, and send a group event mentioning a sibling bot; current main starts all accounts, dedupes per account, and only checks whether the current bot was mentioned.

## Solution Assessment

No, not as submitted. The arbitration direction is right, but the PR should derive the coordinator from account resolution and avoid substring-only implicit matching before it is a maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Derive the coordinator account instead of hard-coding main:** `extensions/feishu/src/monitor.ts:35`
  - body: This policy assumes the coordinator account id is always `main`, but Feishu account ids are configurable and the documented default is `default`. In deployments without a literal `main`, multi-specialist mentions can cause every specialist to skip and no account to coordinate the message.
  - confidence: 0.9
- **[P2] Require token-boundary matching for implicit mentions:** `extensions/feishu/src/bot.ts:256-260`
  - body: The new implicit fallback uses substring matching, so `@main` can match text like `@maintenance` or `@mainbot` and dispatch unexpectedly when there was no structured Feishu mention. Match exact mention tokens or remove the fallback path.
  - confidence: 0.86
