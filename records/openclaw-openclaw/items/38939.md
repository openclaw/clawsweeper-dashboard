## Summary

Keep open: current main still has the Feishu multi-account routing gap, but this PR is not merge-ready because it lacks real Feishu proof and introduces blocking routing correctness problems.

## What This Changes

The PR adds Feishu dispatch-time mention arbitration, shared mention ID extraction, regression tests, and a design note for multi-bot group routing.

## Best Possible Solution

Adopt one Feishu multi-bot mention contract that derives the coordinator from configured account metadata, covers group and topic_group chats, avoids substring mention matching, includes focused regression tests, and has redacted real Feishu proof.

## Reproduction Assessment

Yes, at source level: current main starts one Feishu monitor per enabled account, and each account-scoped handler only checks whether its own bot identity was mentioned. I did not establish a live Feishu tenant reproduction in this read-only review.

## Solution Assessment

No, not as submitted. Dispatch-time arbitration may be viable, but this patch hard-codes the coordinator, excludes topic_group chats, adds substring text-mention matching, and overlaps the open structured-metadata alternative.
