## Summary

Keep open: current main still has account-local Feishu mention handling and per-account dedupe, so the multi-bot duplicate-reply problem is not solved; the submitted patch needs real Feishu proof and correctness fixes before merge.

## What This Changes

The PR adds Feishu mention extraction, dispatch-time arbitration between a coordinator/main account and specialist bot accounts, regression tests, and a design note.

## Best Possible Solution

Implement Feishu-owned mention arbitration using the configured coordinator/default account, covering group and topic_group while preserving broadcast filtering, dedupe, debounce, and @all behavior, with focused tests and redacted live Feishu proof.

## Reproduction Assessment

Yes, at source level: current main registers Feishu inbound handling per account, gates mention handling against only the current bot identity, and dedupes by account namespace. I did not run a live Feishu multi-bot group in this read-only review.

## Solution Assessment

No. Dispatch-time arbitration is a plausible owner boundary, but this patch is not the best implementation because it hard-codes the coordinator, skips topic_group, substring-matches implicit mentions, and regresses the existing broadcast-mention filtering contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Derive the coordinator account from Feishu config:** `extensions/feishu/src/monitor.ts:38`
  - body: This hard-codes the coordinator as `main`, but Feishu account IDs and default-account selection are configurable. If the coordinator is named `default`, `ops`, or another account id, a message mentioning that coordinator plus a specialist can make both mentioned accounts skip instead of routing once.
  - confidence: 0.91
- **[P2] Apply arbitration to topic groups:** `extensions/feishu/src/monitor.ts:97-99`
  - body: Current Feishu code treats both `group` and `topic_group` as group chats, but this policy returns early unless the raw chat type is exactly `group`. Multi-bot topic groups would keep the duplicate-reply behavior this PR is intended to prevent.
  - confidence: 0.86
- **[P2] Require token boundaries for implicit mentions:** `extensions/feishu/src/bot.ts:256-260`
  - body: The implicit fallback uses substring matching, so a fallback pattern such as `@main` can match plain text like `@maintenance` or `@mainbot` and mark the bot as mentioned without a real structured Feishu mention. Match exact mention tokens or remove this fallback path.
  - confidence: 0.88
- **[P2] Filter broadcast mentions in the shared extractor:** `extensions/feishu/src/bot.ts:197-211`
  - body: Current main deliberately excludes Feishu broadcast `@all` and `@_all` from bot-mention handling. The new shared extractor should filter those broadcast mentions before feeding `checkBotMentioned` and arbitration, otherwise broadcast-only messages can be treated as addressed bot mentions.
  - confidence: 0.82
