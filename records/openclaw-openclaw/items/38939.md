## Summary

Keep open: current main still has a source-level Feishu multi-account duplicate or misrouted reply path, but this branch is not merge-ready because it lacks real Feishu proof and introduces coordinator, topic-group, and substring-matching defects.

## What This Changes

The branch adds Feishu inbound mention extraction and main/specialist dispatch arbitration, with regression tests and a design note.

## Best Possible Solution

Implement one Feishu multi-bot mention contract using the configured coordinator/default account, cover `group` and `topic_group`, use boundary-safe text matching, add focused regression tests, and require redacted live Feishu proof.

## Reproduction Assessment

Yes, at source level: current main starts every enabled Feishu account and each account-scoped handler only checks whether its own bot identity was mentioned. I did not establish a live Feishu tenant reproduction.

## Solution Assessment

No, not as submitted. Dispatch-time arbitration may be viable, but this patch hard-codes the coordinator, excludes `topic_group`, and adds substring text-mention matching; the safer path needs an agreed Feishu multi-bot contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Derive the coordinator account from config:** `extensions/feishu/src/monitor.ts:38`
  - body: This hard-codes the coordinator as `main`, but Feishu account IDs and default-account selection are configurable. Deployments using `default`, `ops`, or another `channels.feishu.defaultAccount` can misroute multi-bot mentions, and multi-specialist mentions can be skipped by every addressed account instead of being handled once.
  - confidence: 0.9
- **[P2] Apply arbitration to topic_group chats:** `extensions/feishu/src/monitor.ts:97-99`
  - body: Current Feishu handling treats both `group` and `topic_group` as group-chat surfaces, but this helper returns early unless the raw chat type is exactly `group`. Multi-bot topic groups would keep the duplicate-reply behavior this PR is meant to prevent.
  - confidence: 0.86
- **[P2] Require token boundaries for implicit mentions:** `extensions/feishu/src/bot.ts:259`
  - body: The fallback uses substring matching, so a pattern such as `@main` can match plain text like `@maintenance` or `@mainbot` and mark the bot as mentioned without a real structured Feishu mention.
  - confidence: 0.88
