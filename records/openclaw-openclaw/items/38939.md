## Summary

Keep this PR open for maintainer review: the Feishu duplicate-reply bug is source-reproducible on current main, but the submitted patch still has correctness blockers, lacks real Feishu after-fix proof, and overlaps an unresolved metadata-vs-arbitration product decision.

## What This Changes

The PR adds Feishu mention extraction and dispatch-time main/specialist arbitration for overlapping group mentions, plus regression tests and a design note.

## Best Possible Solution

Adopt one Feishu multi-bot mention contract that uses the configured coordinator/default account, covers group and topic_group chats, avoids substring mention matching, and includes focused regression tests plus redacted real Feishu proof.

## Reproduction Assessment

Yes, at source level: current main starts one Feishu monitor per enabled account, and each account-scoped handler only checks whether its own bot identity was mentioned. I did not establish a live Feishu tenant reproduction in this read-only review.

## Solution Assessment

No, not as submitted. Dispatch-time arbitration may be viable, but this patch hard-codes the coordinator, excludes topic_group chats, adds substring text-mention matching, and overlaps the open structured-metadata alternative.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Derive the coordinator account from Feishu config:** `extensions/feishu/src/monitor.ts:38`
  - body: This hard-codes the coordinator as `main`, but Feishu account IDs and default-account selection are configurable. Deployments using `default`, `ops`, or another `channels.feishu.defaultAccount` can misroute multi-bot mentions, and multi-specialist mentions can be skipped by every addressed account instead of being handled once.
  - confidence: 0.91
- **[P2] Apply arbitration to topic groups:** `extensions/feishu/src/monitor.ts:97-99`
  - body: Current Feishu handling treats both `group` and `topic_group` as group-chat surfaces, but this helper returns early unless the raw chat type is exactly `group`. Multi-bot topic groups would keep the duplicate-reply behavior this PR is meant to prevent.
  - confidence: 0.86
- **[P2] Require token boundaries for implicit mentions:** `extensions/feishu/src/bot.ts:259`
  - body: The fallback uses substring matching, so a pattern such as `@main` can match plain text like `@maintenance` or `@mainbot` and mark the bot as mentioned without a real structured Feishu mention.
  - confidence: 0.88
