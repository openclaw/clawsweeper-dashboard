## Summary

Keep open: current main still does not implement Feishu sibling-account mention arbitration, and the related metadata approach remains open rather than superseding this PR. The branch is still potentially useful, but it is blocked by missing real Feishu proof, unresolved product direction, and concrete correctness findings.

## What This Changes

The branch adds Feishu group mention extraction, dispatch arbitration across main/specialist accounts, regression tests, and a design note.

## Best Possible Solution

Choose one Feishu multi-bot mention strategy, then implement it on current main with configured default-account selection, group and topic_group coverage, preserved broadcast/self filtering, focused regression tests, and redacted live multi-bot proof.

## Reproduction Assessment

Yes, at source level: current main can start multiple enabled Feishu accounts, and each receive path claims processing with the accountId namespace before dispatch. I did not run a live Feishu tenant in this read-only review.

## Solution Assessment

No. Dispatch-time arbitration may be viable, but this patch hard-codes main, skips topic_group, uses substring fallback matching, and does not preserve broadcast mention filtering; maintainers should choose arbitration versus metadata before porting.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Derive the coordinator account from Feishu config:** `extensions/feishu/src/monitor.ts:38`
  - body: This hard-codes the coordinator as `main`, but Feishu account IDs and default-account selection are configurable. Deployments without a literal `main` account can make every mentioned specialist skip while no configured coordinator handles the message.
  - confidence: 0.91
- **[P2] Apply arbitration to topic_group chats:** `extensions/feishu/src/monitor.ts:97-99`
  - body: Current Feishu handling treats both `group` and `topic_group` as group chats, but this policy returns early unless the raw chat type is exactly `group`. Multi-bot topic groups would keep the duplicate-reply behavior this PR is meant to prevent.
  - confidence: 0.86
- **[P2] Require token boundaries for implicit mentions:** `extensions/feishu/src/bot.ts:256-260`
  - body: The implicit fallback uses substring matching, so a fallback pattern such as `@main` can match plain text like `@maintenance` or `@mainbot` and mark the bot as mentioned without a real structured Feishu mention.
  - confidence: 0.88
- **[P2] Filter broadcast mentions in the shared extractor:** `extensions/feishu/src/bot.ts:197-211`
  - body: Current main excludes Feishu broadcast `@all` and `@_all` from bot-mention handling. The new shared extractor should filter those broadcast mentions before feeding `checkBotMentioned` and arbitration, otherwise broadcast-only messages can be treated as addressed bot mentions.
  - confidence: 0.82
