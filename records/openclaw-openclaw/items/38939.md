## Summary

Keep this PR open for maintainer review: the Feishu duplicate-reply problem still exists on current main, but this branch has blocking correctness issues and lacks real Feishu multi-bot proof.

## What This Changes

The PR adds Feishu mention extraction, dispatch-time main/specialist arbitration, regression tests, and a design note for multi-bot group mentions.

## Best Possible Solution

Choose one Feishu multi-bot mention contract, then implement it with configured coordinator selection, group and topic_group coverage, safe mention matching, focused tests, and redacted live Feishu proof.

## Reproduction Assessment

Yes, at source level: current main starts every enabled Feishu account and processes each message under the current account id, while mention detection only identifies the current bot. I did not establish a live Feishu tenant reproduction.

## Solution Assessment

No. Dispatch-time arbitration may be viable, but this branch hard-codes the coordinator, skips topic_group chats, and adds substring fallback matching that can create false-positive mentions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Derive the coordinator account from config:** `extensions/feishu/src/monitor.ts:35`
  - body: This hard-codes the coordinator as `main`, but Feishu account IDs and default-account selection are configurable. Deployments using `default`, `ops`, or `channels.feishu.defaultAccount` can misroute or drop multi-bot mentions because the arbitration never identifies the configured coordinator.
  - confidence: 0.9
- **[P2] Apply arbitration to topic_group chats:** `extensions/feishu/src/monitor.ts:93-95`
  - body: Current Feishu handling treats `group` and `topic_group` as group chats, but this policy returns early unless the raw chat type is exactly `group`. Multi-bot topic groups would keep the duplicate-reply behavior this PR is meant to prevent.
  - confidence: 0.86
- **[P2] Require token boundaries for implicit mentions:** `extensions/feishu/src/bot.ts:259`
  - body: The implicit fallback uses substring matching, so a fallback pattern such as `@main` can match plain text like `@maintenance` or `@mainbot` and mark the bot as mentioned without a real structured Feishu mention.
  - confidence: 0.88
