## Summary

Keep this PR open: the reported Feishu multi-bot duplicate-reply problem is still plausible on current main, but the submitted branch is not mergeable as-is because it hard-codes the coordinator account, misses topic groups, can false-trigger text fallback mentions, and lacks real Feishu after-fix proof.

## What This Changes

The PR adds Feishu mention extraction, dispatch-time main/specialist arbitration, regression tests, and a design note for group mention routing.

## Best Possible Solution

Choose one Feishu multi-bot routing contract, then implement it with configured coordinator selection, group and topic_group coverage, safe mention matching, focused tests, and redacted live proof.

## Reproduction Assessment

Yes, at source level: current main starts one Feishu monitor per enabled account and only deduplicates under the current accountId, so sibling accounts can still process the same group event. I did not establish a live Feishu tenant reproduction.

## Solution Assessment

No. Dispatch-time arbitration may be viable, but this branch hard-codes the coordinator, skips topic_group chats, and uses substring text fallback matching that can create false positive mentions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Derive the coordinator account from config:** `extensions/feishu/src/monitor.ts:38`
  - body: This hard-codes the coordinator as `main`, but Feishu account IDs and default-account selection are configurable. Deployments without a literal `main` account can make multiple mentioned specialist accounts all skip while no configured coordinator handles the message.
  - confidence: 0.91
- **[P2] Apply arbitration to topic_group chats:** `extensions/feishu/src/monitor.ts:97-99`
  - body: Current Feishu handling treats both `group` and `topic_group` as group chats, but this policy returns early unless the raw chat type is exactly `group`. Multi-bot topic groups would keep the duplicate-reply behavior this PR is meant to prevent.
  - confidence: 0.86
- **[P2] Require token boundaries for implicit mentions:** `extensions/feishu/src/bot.ts:256-260`
  - body: The implicit fallback uses substring matching, so a fallback pattern such as `@main` can match plain text like `@maintenance` or `@mainbot` and mark the bot as mentioned without a real structured Feishu mention.
  - confidence: 0.88
