## Summary

Keep this PR open for maintainer review: the token-reduction goal is useful, but the patch is incomplete against current main and overlaps the broader open hook-based tool/skill narrowing proposal in #68734.

## What This Changes

The PR adds an optional `AssembleResult.toolsOverride` field and a PI embedded-runner stream wrapper that filters model-visible tool schemas by override tool names.

## Best Possible Solution

Choose one additive per-turn tool and skill visibility contract, then implement it consistently across PI and Codex harnesses with documented undefined-versus-empty semantics and focused regression tests.

## Reproduction Assessment

Yes. Source and PR-head diff review are enough to reproduce the review issues: empty overrides are ignored, engines do not receive selectable schemas, and Codex context-engine turns would not consume the new result field.

## Solution Assessment

No. The direction is useful, but this PR is not the best complete solution until maintainers choose the shared visibility API relative to #68734 and apply it consistently across runtimes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Honor explicit empty toolsOverride arrays:** `src/agents/pi-embedded-runner/run/attempt.ts:2131`
  - body: The guard only wraps the stream when `toolsOverride.length > 0`, so an engine returning `toolsOverride: []` to hide all schemas still sends the full default tool set. Use field presence for the override signal, or explicitly document and warn that empty arrays mean passthrough.
  - confidence: 0.96
- **[P2] Apply tool overrides to Codex harness requests:** `src/context-engine/types.ts:18`
  - body: Adding `toolsOverride` to the shared `AssembleResult` lets any context engine return it, but this patch only consumes it in the PI runner. Codex app-server turns also call `assemble()` and would continue to send the full `toolBridge.specs`, so the same engine would behave differently by runtime.
  - confidence: 0.9
- **[P2] Expose selectable schemas or make the override name-based:** `src/context-engine/types.ts:18`
  - body: The new field requires engines to return `AnyAgentTool[]`, but the current assemble contract only exposes available tool names. Plugin authors cannot implement the PR body's `params.tools.filter(...)` example without private schema access, so the API should either pass selectable tools or accept names.
  - confidence: 0.9
