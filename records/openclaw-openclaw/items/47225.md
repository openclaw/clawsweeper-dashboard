## Summary

Keep open: the feature direction is plausible, but this PR still has blocking API and semantics gaps, overlaps the broader hook-based narrowing proposal, is currently conflicting, and lacks contributor real behavior proof.

## What This Changes

The PR adds a context-engine `toolsOverride` result field plus embedded-runner stream-wrapper wiring to filter model-visible tool schemas by name.

## Best Possible Solution

Choose one documented dynamic narrowing seam that exposes a derivable host-provided tool catalog, defines undefined versus empty-list behavior, and is covered by focused tests plus redacted real-run proof.

## Reproduction Assessment

Yes for the PR review findings: source inspection of the PR diff shows the present-empty guard, and current main shows context-engine assembly receives only a tool-name set rather than live tool objects. This is a source-level review reproduction, not a live product bug reproduction.

## Solution Assessment

No as written. The feature direction is reasonable, but the patch should expose a returnable name allowlist or pass host-provided tool objects into `assemble()`, and it must define present-empty semantics before adding the public contract.
