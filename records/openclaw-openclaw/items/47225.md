## Summary

Keep open: the feature direction is useful, but the PR still has blocking contract issues, lacks contributor real behavior proof, and the related hook-based narrowing work at https://github.com/openclaw/openclaw/pull/68734 is open and not a merged replacement.

## What This Changes

The branch adds `toolsOverride?: AnyAgentTool[]` to `AssembleResult` and wraps embedded-runner stream calls to filter model-visible tool schemas by override names.

## Best Possible Solution

Settle one documented dynamic tool-narrowing seam, make the return value derivable from host-provided inputs, define undefined versus empty-list behavior, add focused regression coverage, and require redacted real-run proof.

## Reproduction Assessment

Yes for the review findings: source inspection and the PR diff show current `assemble()` exposes only `availableTools?: Set<string>` while the PR returns `AnyAgentTool[]`, and the proposed guard skips `[]`. I did not run tests because this was a read-only review.

## Solution Assessment

No as written. The direction is useful, but the PR should use a host-returnable name allow-list or pass real tool objects into `assemble()`, and it must define present-empty behavior before adding the public contract.
