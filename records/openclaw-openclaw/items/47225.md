## Summary

Keep open: current main does not implement this context-engine tool-filtering seam, and the branch still needs API contract work plus real behavior proof. The related hook-based narrowing PR is relevant, but it is still open and not a shipped replacement.

## What This Changes

The PR adds an optional `toolsOverride` result for context engines and wraps embedded-runner stream calls to filter model-visible tool schemas.

## Best Possible Solution

Choose one documented dynamic narrowing seam, make it derivable from host-provided inputs, define undefined versus empty-list behavior, and add focused regression coverage plus redacted real-run proof.

## Reproduction Assessment

Yes for the review findings: source inspection and the PR diff show current `assemble()` exposes only `availableTools?: Set<string>` while the PR returns `AnyAgentTool[]`, and the proposed guard skips `[]`. This is a source-level PR review reproduction, not a live bug repro.

## Solution Assessment

No as written. The feature direction is useful, but the PR should either expose a returnable name allow-list or pass real tool objects into `assemble()`, and it must define present-empty behavior before adding the public contract.
