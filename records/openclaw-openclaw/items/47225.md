## Best Possible Solution

Choose one documented dynamic narrowing seam, make it derivable from host-provided inputs, define undefined versus empty-list semantics, and add focused tests plus redacted real-run proof.

Yes for the PR review findings: source inspection and the PR diff show the proposed result cannot be built from current host inputs and that present empty overrides are skipped. This is a source-level PR review reproduction, not a live bug repro.

## Solution Assessment

No as written. The direction is useful, but the PR should expose a contract context engines can derive from host inputs and define empty-list behavior before adding the public API.
