## Summary

Keep open: the feature is plausible and not implemented on current main, but this PR should not merge as written because the new public contract is not returnable from the current assemble inputs, empty overrides are silently ignored, and external real behavior proof is still missing.

## What This Changes

The PR adds an optional `toolsOverride` array to context-engine assembly and wraps the embedded runner stream function to filter model-visible tool schemas by name.

## Best Possible Solution

Settle one maintained dynamic narrowing seam, make the contract returnable from host-provided inputs, define undefined versus empty-list behavior, then add focused regression coverage and redacted real-run proof.

## Reproduction Assessment

Yes for the review findings: source inspection shows the PR ignores a present empty override and current main exposes only tool names to `assemble()`, not `AnyAgentTool[]`. I did not run tests because this was a read-only review.

## Solution Assessment

No as written. The direction is useful, but the PR should use a host-returnable name allow-list or pass real tool objects into `assemble()`, and it must define present-empty behavior before adding a public plugin contract.
