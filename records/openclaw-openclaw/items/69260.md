## Summary

Keep open: current main still has only the bridge-client provider-env scrubber, while the ACP runtime/acpx path lacks generic auth-contract metadata, per-agent provider-env scrubbing, and fail-closed preflight for Gemini-style mixed-credential hosts.

## What This Changes

Requests defense-in-depth ACP/acpx auth hooks so ACP-backed agents can declare expected auth mode, scrub incompatible ambient provider keys, and fail closed before launch.

## Best Possible Solution

Design a generic ACP/acpx auth contract that lets agents declare expected auth mode, fallback policy, provider-env scrubbing, and preflight behavior, then document mixed-credential host guidance.

## Reproduction Assessment

No. The issue gives a credible staged A/B and source review confirms the requested hardening seams are absent, but this review did not run a disposable Gemini CLI fixture against current main.

## Solution Assessment

Yes as a direction. A generic ACP/acpx contract is the maintainable route because it avoids Gemini-specific core policy while giving ACP agent owners a fail-closed boundary.
