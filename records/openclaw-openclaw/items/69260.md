## Summary

Keep open: current main still lets ACPX child harnesses inherit ambient provider auth env and lacks the requested auth-contract/preflight surface. The linked implementation PR is still open with closing syntax, and the later Claude ACP evidence supports treating this as a generic ACP auth-boundary issue rather than a Gemini-only report.

## What This Changes

Request ACP-backed harnesses to declare auth expectations, scrub incompatible provider auth env before child spawn, fail closed or preflight fragile routes, and document mixed-credential hosts.

## Best Possible Solution

Review and either land or reject https://github.com/openclaw/openclaw/pull/81200 for the focused child-env boundary, then track broader auth-contract, doctor, and opt-in UX only if maintainers accept that larger surface.

## Solution Assessment

Yes for keeping this issue open: the linked PR is the concrete implementation candidate and should be reviewed before closure. The focused wrapper strip is a maintainable first slice, while generic auth-contract, preflight, doctor, and opt-in behavior are product/security decisions.
