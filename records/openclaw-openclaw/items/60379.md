## Summary

Keep open. Current main still implements `/model` as a persistent session model switch, and the existing `/btw`, cron `--model`, and CLI one-shot surfaces do not provide a normal one-turn chat reply on a different model while preserving the session default and queued/background work behavior.

## What This Changes

Request a one-turn chat model override, such as `/ask` or `/model --once`, that answers with a selected model without changing the session default.

## Best Possible Solution

Add a first-class one-turn model override for normal chat runs that reuses existing model allowlist/auth-profile validation, carries the selected provider/model only through the current reply, and leaves session model state, queued followups, heartbeat, and cron resolution untouched.

## Reproduction Assessment

Yes. A source-level reproduction is clear: an authorized mixed-content `/model openai/gpt-4o hello` resolves through the inline directive path, writes session model override fields, and later turns read those stored fields.

## Solution Assessment

Yes. The requested direction is the narrow maintainable fix; a parser-only alias would be incomplete because the run path must avoid session persistence and queue retargeting while still using existing model validation contracts.
