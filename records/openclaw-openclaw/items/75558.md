## Summary

Keep open. Current main still skips runtime conversation-binding lookup for non-topic Telegram groups, and this PR targets that gap directly with a narrow Telegram plugin change plus a focused regression test. I found no blocking correctness issue in the proposed patch.

## What This Changes

The PR updates Telegram conversation route resolution so native slash commands consult runtime conversation bindings for non-topic group chats and adds a session-metadata regression test for a bound supergroup `/status` command.

## Best Possible Solution

Land the narrow Telegram plugin fix after normal CI so runtime binding lookup uses the same top-level group conversation id that the Telegram binding adapter stores, while preserving the existing DM and forum-topic routing behavior.

## Reproduction Assessment

Yes. The source-level reproduction is clear: create a runtime binding for `telegram/default/-1001234567890`, invoke native `/status` in a non-topic supergroup, and current main never consults that binding because `threadBindingConversationId` is `undefined`.

## Solution Assessment

Yes. The proposed change is the narrowest maintainable fix because it only changes the runtime binding conversation id selection in the Telegram route helper and adds a regression test at the native command/session-metadata boundary.
