## Summary

Keep open. Current main still carries the reported native Codex session-scoped request identity, `prompt_cache_key`, and prior Responses item replay shape for tool-call history, while #76832 is an open implementation candidate rather than a merged fix.

## What This Changes

The issue reports that Telegram sessions using `openai-codex/*` can replay a prior tool-call turn's assistant text on unrelated later messages due to native Codex transport/cache/replay state plus Lossless Claw context pollution.

## Best Possible Solution

Land a focused OpenAI provider transport/cache/replay fix for native ChatGPT Codex Responses, then track any remaining Lossless Claw ledger pollution as a separate plugin follow-up.

## Reproduction Assessment
