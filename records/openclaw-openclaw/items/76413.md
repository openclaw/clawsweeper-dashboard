Keep open. Current main still has the reported native OpenAI Codex request identity, cache, and replay-state shape, while #76832 is an open implementation candidate rather than a merged fix.

## What This Changes

The issue asks OpenClaw to stop Telegram `openai-codex/*` sessions from replaying a prior tool-call turn's assistant reply on later unrelated messages.

## Best Possible Solution

Land a focused OpenAI provider transport/cache/replay fix for native ChatGPT Codex Responses, then track any remaining Lossless Claw ledger pollution as a separate plugin follow-up.

## Reproduction Assessment

Yes at source level: current main still constructs the session-scoped request identity, `prompt_cache_key`, and prior Responses item replay shape described for tool-call turns. No live Telegram/OpenAI-Codex/LCM end-to-end run was performed in this read-only pass.

Yes for the core portion: a #76832-style provider-owned fix is the narrowest maintainable path because it scopes the changed cache/replay behavior to native ChatGPT Codex while preserving custom Codex-compatible endpoints.
