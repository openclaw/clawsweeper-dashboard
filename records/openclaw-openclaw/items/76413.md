## Summary

Keep open. Current main and the latest shipped release still preserve the native OpenAI Codex request identity, prompt-cache key, and Responses replay state implicated by the report, while #76832 is an open implementation candidate rather than a merged fix.

## What This Changes

Telegram sessions using native `openai-codex` Responses should not replay a prior tool-call turn's assistant text into later unrelated messages.

## Best Possible Solution

Land the focused OpenAI provider transport/cache/replay fix for native ChatGPT Codex Responses, then track any remaining Lossless Claw context pollution as a separate plugin follow-up.

## Reproduction Assessment

Yes, at source level: current main still builds the session-scoped request identity, session-based `prompt_cache_key`, and prior Responses item replay shape described for tool-call turns. I did not run a live Telegram/OpenAI-Codex/LCM end-to-end reproduction in this read-only pass.

## Solution Assessment

Yes for the core portion: a provider-owned fix like #76832 is the narrowest maintainable path because it scopes the cache/replay change to native ChatGPT Codex while preserving custom Codex-compatible endpoints. The Lossless Claw contributor should remain a separate plugin follow-up unless maintainers identify a missing core API seam.
