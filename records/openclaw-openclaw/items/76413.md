## Summary

Keep open. Current main and the latest release still carry the reported native Codex request identity, cache key, and Responses replay-state shape, while #76832 is an open implementation candidate rather than a merged fix.

## What This Changes

OpenAI Codex Telegram tool-call sessions should not replay a prior assistant reply into unrelated later turns because transport/cache/replay state leaks across turns.

## Best Possible Solution

Review and land a focused native ChatGPT Codex transport/cache/replay fix, then track any remaining Lossless Claw reconciliation problem separately if it still reproduces.

## Reproduction Assessment

Yes at source level: current main constructs the three implicated state carriers for native `openai-codex` Responses turns after tool-call history. I did not run a live Telegram/ChatGPT Codex/LCM end-to-end scenario in this read-only pass.

## Solution Assessment

Yes for the core portion: a provider-owned OpenAI Codex transport/cache/replay fix like #76832 is the narrowest maintainable path. Lossless Claw ledger pollution should stay a separate plugin follow-up unless maintainers find a missing core plugin API seam.
