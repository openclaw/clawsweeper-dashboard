## Summary

Keep open. Current main still has the reported OpenAI Codex PI transport preconditions, and the remaining question is not whether the report is stale but which provider-specific policy should change without regressing shared OpenAI Responses replay behavior.

## What This Changes

The issue reports replayed OpenAI Codex Telegram replies after tool-call turns, attributed to transport request correlation, Responses item replay, prompt cache parameters, and lossless-claw session glob handling.

## Best Possible Solution

Add a narrowly scoped Codex Responses transport policy only after confirming the live backend contract, preserving direct OpenAI Responses replay behavior unless there is Codex-specific evidence to branch it.

## Reproduction Assessment

Partially. Source inspection gives a high-confidence path for the reported preconditions on current main, but this review did not reproduce the end-to-end Telegram/ChatGPT replay symptom with live credentials.

## Solution Assessment

Unclear. The proposed patches are plausible, but the reasoning/item-id part conflicts with intentional OpenAI Responses replay coverage and the lossless-claw part belongs to an external plugin/config contract.
