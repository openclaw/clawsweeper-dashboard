## Summary

Keep open. Current main still lacks the requested cron-owned outbound delivery telemetry: the cron paths have job/run/agent/account context and durable sends return message IDs, but only generic delivery diagnostics/hooks exist and no duplicate or fixing PR was found.

## What This Changes

Emit a structured `cron.delivery.sent` log/event for each outbound cron delivery attempt with cron job/run attribution, agent/account IDs, platform message IDs, and no message content.

## Best Possible Solution

Add cron-owned structured telemetry at the durable send boundary for direct cron announcements and failure-notification sends after maintainers approve the event contract and privacy-safe attempt semantics.

## Reproduction Assessment

Yes. Source inspection on current main is enough to reproduce the gap: cron dispatch has the attribution inputs and durable sends expose message IDs, but no `cron.delivery.sent` or equivalent cron-owned per-attempt log exists.

## Solution Assessment

Yes, with maintainer-owned schema details. A focused cron telemetry wrapper or explicit telemetry parameter at the durable send boundary is narrower than relying on generic delivery diagnostics, which lack cron attribution.
