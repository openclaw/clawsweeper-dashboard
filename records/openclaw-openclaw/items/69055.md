## Summary

Keep open. Current main still lacks the requested privacy-safe, per-attempt `cron.delivery.sent` record; the newer cron diagnostic helpers add cron fields for stuck-session logs only, not outbound send success/error logs with returned platform message IDs.

## What This Changes

Request adds a structured `cron.delivery.sent` log line for every outbound cron delivery attempt with cron job/run, agent/account, target, status, and platform message IDs.

## Best Possible Solution

Add a cron-owned telemetry wrapper or explicit delivery telemetry parameter at the direct cron outbound send boundary, emitting one success/error record from durable send outcomes without payload or prompt content.

## Reproduction Assessment

Yes. Source inspection on current main shows cron dispatch has the required job/agent/session context and returned send results, but no cron-owned per-attempt log receives both attribution and platform message IDs.

## Solution Assessment

Yes. A focused cron telemetry path at the outbound send boundary is the narrow maintainable solution; generic diagnostics lack attribution and `message_sent` hooks can include content.
