## Summary

Keep open. Current main still lacks the requested privacy-safe `cron.delivery.sent` record; cron delivery code has the needed cron/job/session context and durable send results, but no cron-owned per-attempt success/error log with returned platform message IDs.

## What This Changes

Request structured `cron.delivery.sent` logging for each outbound cron delivery attempt with cron, agent, account, target, platform message id, and status fields.

## Best Possible Solution

Add a cron-owned telemetry emission at the cron send boundaries, using durable send outcomes to log one ID-only success/error record per platform send attempt while preserving privacy constraints.

## Reproduction Assessment

Yes. Source inspection on current main shows the cron dispatch and notification send paths have the needed inputs and returned `messageId` results, but no `cron.delivery.sent` or equivalent per-attempt cron-owned log exists.

## Solution Assessment

Yes. A focused cron telemetry wrapper or explicit delivery telemetry parameter at the durable send boundary is the narrow maintainable path; generic delivery diagnostics lack attribution, and `message_sent` hooks can carry content.
