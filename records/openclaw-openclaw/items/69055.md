## Summary

Keep open. Current main and the latest release still do not emit the requested cron-owned per-attempt delivery log with cron/job/run/account/platform-message attribution; the remaining work is a privacy-sensitive diagnostic feature that needs maintainer schema and semantics decisions.

## What This Changes

Request to add a structured `cron.delivery.sent` log line for every outbound cron delivery attempt with cron job, run, agent, account, channel, target, platform message ID, and status fields.

## Best Possible Solution

Define a bounded cron delivery telemetry contract and emit exactly one record per cron-owned outbound send attempt from the direct announce and failure-notification send boundaries using durable send results.

## Reproduction Assessment

Yes for the missing behavior: source inspection on current main and v2026.5.18 shows no `cron.delivery.sent` event or delivery telemetry plumbing, while the cron dispatch and durable send result paths already carry the necessary IDs. This is a feature gap rather than a failing current contract.

## Solution Assessment

Yes directionally, but not ready for automated implementation. Emitting at the cron-owned durable send boundaries is the narrow path, while maintainers should first settle schema, retry/cardinality behavior, and privacy limits.
