## Summary

Keep open. Current main still lacks the requested cron-owned per-attempt `cron.delivery.sent` telemetry with job/run/agent/account attribution and platform message IDs, and the broader open dispatch-ACK issue does not replace this platform-send attribution request.

## What This Changes

Request exactly one structured ID-only log record for each outbound cron delivery success or error, including cron job/run attribution, agent/account identity, channel target, and platform message ID.

## Best Possible Solution

Add a cron-owned ID-only telemetry emission at the durable outbound send boundary, covering direct cron announcements and failure notifications while reusing existing send results and privacy-safe diagnostics.

## Reproduction Assessment

Yes. Source inspection on current main shows the cron dispatch inputs and durable send results exist, but there is no `cron.delivery.sent` or equivalent cron-attributed per-attempt log for success or error outcomes.

## Solution Assessment

Yes, with maintainer-owned schema details. Emitting near the durable platform send boundary is the narrow maintainable direction; relying on generic delivery diagnostics would still miss cron attribution and message IDs.
