## Summary

Keep open. Current main still lacks the requested cron-owned `cron.delivery.sent` telemetry; cron dispatch has the job/run/agent/account inputs and outbound delivery returns platform message IDs, but no structured per-attempt cron delivery log exists for success or error outcomes.

## What This Changes

Request structured, privacy-safe cron delivery telemetry that logs one `cron.delivery.sent` record per outbound delivery attempt with cron/job/run/agent/account attribution and platform message IDs.

## Best Possible Solution

Add a cron-owned telemetry emission at the outbound send boundary that records ID-only success/error outcomes per attempted platform send while reusing durable send results and preserving privacy constraints.

## Reproduction Assessment

Yes. Source inspection on current main is enough to reproduce the gap: cron dispatch has attribution inputs and outbound sends expose message IDs, but no `cron.delivery.sent` or equivalent per-attempt cron-owned log exists.

## Solution Assessment

Yes, with maintainer-owned schema details. A focused cron telemetry wrapper or explicit telemetry parameter at the durable send boundary is narrower than relying on generic delivery diagnostics, which lack cron attribution.
