## Summary

Keep open. Current main still lacks the requested cron-owned `cron.delivery.sent` structured log record; existing diagnostic events and `message_sent` hooks do not carry cron job/run attribution in a privacy-safe per-attempt log.

## What This Changes

Request to emit exactly one structured `cron.delivery.sent` log record per outbound cron delivery attempt with cron job/run, agent/account, channel/recipient, platform message ID, and status attribution.

## Best Possible Solution

Add a cron-owned, privacy-safe telemetry path at the direct outbound delivery boundary that emits exactly one success/error record per send attempt and maps returned platform message IDs without logging payload text.

## Reproduction Assessment

Yes. Code inspection on current main reproduces the missing behavior: `dispatchCronDelivery` sends through `deliverOutboundPayloads`, but no logger receives cron job/run ids plus returned platform message ids.

## Solution Assessment

Yes. A focused cron telemetry wrapper or explicit delivery telemetry parameter is the narrow maintainable fix because generic outbound hooks lack cron attribution and can include content.
