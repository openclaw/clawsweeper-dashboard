## Repair Work Prompt

Fix the `claude-cli` live-session EPIPE bug reported in https://github.com/openclaw/openclaw/issues/77178. Current main starts the bundled Claude CLI backend as `liveSession: "claude-stdio"` with `stdinMode: "pipe-open"` and writes stream-json user messages to the child; if Claude exits or closes stdin early, `write EPIPE` can surface before OpenClaw gives a useful diagnostic. Keep the fix scoped to the Claude live-session/supervisor stdin path unless evidence proves the Anthropic plugin defaults are wrong. Do not add a config flag or change non-Claude CLI backend behavior. Add regression tests for closed stdin/EPIPE in the Claude live-session path, preserve stderr/exit details for the user, and add a user-facing CHANGELOG entry. If the fix requires pinning or changing an upstream Claude CLI dependency/flag contract, stop for maintainer review.
