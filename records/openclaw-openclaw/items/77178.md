## Summary

OpenClaw’s current Claude live-session path is still capable of surfacing a raw stdin `write EPIPE` when the Claude process exits before or during the first streamed prompt, and I could not find a merged issue-specific fix or current-main/release proof that the Docker/OAuth failure is gone.

## What This Changes

The issue reports an EPIPE failure when OpenClaw spawns a Claude CLI live session in Docker/Unraid using OAuth.

## Best Possible Solution

Keep the issue open for a focused Claude CLI live-session repro/fix that converts fast child-exit or stdin EPIPE into either a successful streamed turn or a clear actionable Claude CLI auth/container diagnostic, with regression coverage for the async stdin error path.

## Reproduction Assessment

Yes from source, but not from a live container: if the Claude CLI process exits before or during the first streamed stdin write, the current code can still reject the turn with the raw write error. The reported Docker/Unraid/OAuth setup was not reproduced live.

## Solution Assessment

No close is warranted. The narrow maintainable fix is in the Claude CLI live-session write/startup error path, plus a live Docker or equivalent proof; adding config or moving ownership would be premature.
