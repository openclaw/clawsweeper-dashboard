## Summary

Keep this open: current main still has a supported Claude CLI live-session stdin path that can surface raw `write EPIPE` when the child exits before the first turn is accepted, and there is no merged fix that covers this supervisor-backed path.

## What This Changes

The issue reports an immediate `write EPIPE` when OpenClaw starts a Claude CLI OAuth live session in Docker on Unraid.

## Best Possible Solution

Make the shared process supervisor and Claude live-session startup path treat early child-stdin closure as a handled child-process failure, surfacing Claude stderr/exit details instead of an uncaught or user-facing raw EPIPE.

## Reproduction Assessment

No live Docker reproduction was established, but there is a high-confidence source path: current main starts Claude live sessions with pipe-open stdin and writes the first turn without the async stdin EPIPE guard used in the older exec helper.

## Solution Assessment

Yes, the narrow maintainable fix is shared stdin error propagation plus Claude live-session startup diagnostics; a Docker/Unraid-specific workaround or new config knob would be premature.
