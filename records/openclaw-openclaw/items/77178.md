## Summary

Keep open: current main still has a plausible Claude CLI live-session EPIPE path. The bundled backend defaults to warm stdio, and the live-session runner currently fails the turn with the raw stdin write callback error if the just-spawned Claude process closes before the first input is accepted.

## What This Changes

The issue requests that Claude CLI OAuth sessions started from Docker/Unraid avoid immediate `write EPIPE` and begin the conversation normally.

## Best Possible Solution

Make the Claude CLI live-session runner handle early stdin closure without surfacing raw EPIPE, preserving live stdio by default and adding focused regression coverage for first-turn child exit.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection shows current main will enter the Claude stdio live-session path by default and fail the turn with the stdin write callback error if the spawned Claude process closes before the first JSON input is accepted.

## Solution Assessment

Unclear until the live early-exit cause is confirmed, but the narrow maintainable fix belongs in the Claude live-session path rather than a broad process-supervisor change. The repair should either retry/degrade that first turn safely or report the Claude CLI exit/stderr reason instead of raw EPIPE.
