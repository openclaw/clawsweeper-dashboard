## Best Possible Solution

Preserve generic-channel reasoning suppression, allow Telegram’s opted-in durable reasoning through shared dispatch, and avoid outbound-plan suppression until after Telegram has split the payload into the reasoning lane.

## Reproduction Assessment

Yes. Current source shows the shared dispatcher suppresses `isReasoning` block/final payloads, and Telegram also normalizes through a helper that returns `null` for reasoning payloads before `splitTextIntoLaneSegments` can use the marker.

## Solution Assessment

No. The shared-dispatch opt-in is the right shape, but the patch is incomplete while the real Telegram `dispatcherOptions.deliver` path still normalizes and can drop `isReasoning` before lane splitting.
