## Best Possible Solution

Keep the typed shared-dispatch opt-in, but make Telegram consume or strip `isReasoning` only after the reasoning-lane decision so durable Thinking reaches the reasoning lane while generic channels and TTS remain suppressed.

## Reproduction Assessment

Yes, source-reproducible: current main emits `isReasoning` reply items for `/reasoning on`, shared dispatch suppresses them, and this PR still leaves a Telegram outbound normalization gate that can suppress the same marker before lane splitting. ## Solution Assessment

No, not yet: the shared opt-in is a good owner-boundary shape, but the fix is incomplete unless Telegram handles reasoning before normalization or strips the marker only after deciding the reasoning-lane delivery.
