## Label Justifications

- `impact:other`: The concrete impact is webchat input responsiveness and international text-entry usability, which does not fit the more specific owned impact labels.

## Summary

Keep open: current main and v2026.6.6 still resize the Control UI chat textarea synchronously on every input event, and the open CJK composition replacement does not coalesce that resize path, so the reported typing-lag problem remains distinct and source-reproducible.

## What This Changes

The issue asks for the Control UI WebChat textarea auto-resize path to avoid per-keystroke forced reflow during CJK IME composition.

## Best Possible Solution

Land a focused Control UI composer change that coalesces textarea resizing to one animation-frame task, coordinates cleanly with the composition-event fix, and proves smooth CJK typing in macOS Chrome and Safari.

## Reproduction Assessment

Yes at source level, but not as a live current-main run. The reported macOS Chrome/Safari IME steps map directly to the current input handler calling synchronous textarea resize on every input event.

## Solution Assessment

Unclear whether the exact proposed snippet is the final best patch, but the solution direction is right: resize should be coalesced at the composer textarea boundary and reviewed with the active composition-event changes.
