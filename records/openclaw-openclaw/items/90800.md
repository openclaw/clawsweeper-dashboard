## Summary

Keep open: current main and v2026.6.6 still resize the Control UI chat textarea synchronously on every input event, and the open related CJK IME PR covers draft-sync composition rather than a merged resize-coalescing fix.

## What This Changes

The issue requests coalescing Control UI webchat textarea auto-resize work so CJK IME composition does not lag on each pinyin keystroke.

## Best Possible Solution

Fold a coalesced textarea resize path into the Control UI composer IME repair, or land a focused follow-up that proves responsive CJK composition in Chrome and Safari.

## Reproduction Assessment

Yes, source-level: current main handles every composer `input` event by calling `adjustTextareaHeight`, which writes height, reads `scrollHeight`, and writes height again. I did not run a live macOS IME session in this read-only review.

## Solution Assessment

Unclear whether a resize-only patch is complete. The maintainable end state is likely resize coalescing coordinated with the existing composition-state fix so draft sync and IME performance do not diverge.
