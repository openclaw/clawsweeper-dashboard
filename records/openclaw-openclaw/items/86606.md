## Summary

Keep open: the PR is aimed at the right message-delivery bug, but the new helper still gives up after a short fixed compaction retry window and the proof is simulated rather than a real compacting run.

## What This Changes

This PR adds a shared active-wake retry helper for compacting outcomes in subagent announcement delivery and adds regression tests for steering and generated-completion handoff paths.

## Best Possible Solution

Keep the shared helper, but make compacting retries honor wakeOptions.deliveryTimeoutMs and cancellation, add a regression for compaction longer than the fixed schedule but within timeout, and provide real compacting-run proof.

## Reproduction Assessment

Yes. Current main source shows compacting is returned as a distinct queue failure and the affected active-wake callers only retry transcript-commit unsupported; the PR body also includes before/after focused Vitest output, though I did not run tests in this read-only review.

## Solution Assessment

No. A shared helper is the right implementation shape, but the helper should wait until delivery timeout or cancellation rather than stop after a fixed 15-second schedule.
