## Summary

Keep open. The PR addresses a plausible gateway-status SSRF gap, but the patch is incomplete: mapped IPv6 link-local metadata addresses bypass it, the broad hostname block conflicts with documented `.local` and wide-area DNS-SD gateway routes, and this external PR still lacks real CLI behavior proof.

## What This Changes

The PR adds gateway-status WebSocket URL hostname filtering and a new security test file for explicit probe targets.

## Best Possible Solution

Implement gateway-status-specific target validation that reuses shared IP/link-local/cloud-metadata classifiers, preserves documented private gateway routes, and adds mapped-IPv6 regression coverage plus real CLI proof.

## Reproduction Assessment

Yes, for source-level review. Current main accepts explicit/configured ws(s) target URLs and passes `target.url` to `probeGateway()`, and Node URL canonicalization shows the PR's mapped-IPv6 metadata forms bypass the new string-prefix guard.

## Solution Assessment

No. The PR touches the right helper, but the maintainable fix should use shared IP/link-local/cloud-metadata classifiers and a gateway-specific hostname policy instead of broad `isBlockedHostname()` suffix blocking.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Reject mapped IPv6 metadata addresses:** `src/commands/gateway-status/helpers.ts:103-106`
  - body: The new guard only rejects `0.0.0.0` and hosts starting with `169.254.`, but Node canonicalizes `ws://[::ffff:169.254.169.254]:80` to `[::ffff:a9fe:a9fe]`. That hostname will not match the prefix and can still reach the gateway probe path, leaving the SSRF mitigation bypassable.
  - confidence: 0.94
- **[P2] Preserve documented gateway hostnames:** `src/commands/gateway-status/helpers.ts:110-112`
  - body: Calling `isBlockedHostname()` rejects `.local` and `.internal` targets, but OpenClaw documents `.local` LAN routes and wide-area DNS-SD/private gateway domains such as `openclaw.internal.`. This would silently drop legitimate gateway-status probes.
  - confidence: 0.86
