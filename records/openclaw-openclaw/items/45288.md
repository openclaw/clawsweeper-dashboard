## Summary

Keep open: this is a plausible security hardening PR for gateway-status, but the patch is incomplete and would also drop documented gateway hostnames. The external PR still lacks after-fix real behavior proof, so it should stay in human security review rather than be cleaned up as stale or superseded.

## What This Changes

The PR adds WebSocket hostname filtering to gateway-status target normalization and a new security test file for metadata, wildcard, blocked-hostname, localhost, LAN, and public targets.

## Best Possible Solution

Use a gateway-specific URL policy built on the shared IP-literal/special-use classifier for metadata, link-local, wildcard, and malformed IPs while preserving documented Bonjour, LAN, and tailnet hostnames.

## Reproduction Assessment

Yes. Source inspection shows current main accepts explicit/configured ws(s) gateway-status URLs and passes them to probeGateway(), and a focused Node check reproduces the PR's mapped-IPv6 bypass in its string policy.

## Solution Assessment

No. The PR targets the right gateway-status surface, but the ad hoc hostname suffix policy should be replaced with shared IP classification plus compatibility tests for documented gateway hostnames.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Reject mapped IPv6 metadata addresses:** `src/commands/gateway-status/helpers.ts:103-106`
  - body: The new guard only blocks 0.0.0.0 and hostnames starting with 169.254., so an explicit URL like `ws://[::ffff:169.254.169.254]:80` canonicalizes to `::ffff:a9fe:a9fe` and bypasses the mitigation. That leaves the cloud metadata endpoint reachable despite the SSRF fix.
  - confidence: 0.94
- **[P2] Preserve documented gateway hostnames:** `src/commands/gateway-status/helpers.ts:110-112`
  - body: Calling `isBlockedHostname()` here rejects `.local` and `.internal`, but OpenClaw documents Bonjour `<hostname>.local` gateway hints and wide-area DNS-SD domains such as `openclaw.internal.`. Legitimate explicit or configured gateway-status probes would be silently dropped.
  - confidence: 0.86

## Security Review

Status: needs_attention
