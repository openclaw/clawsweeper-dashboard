## Summary

Keep open. The PR targets a real gateway-status hardening gap, but the current diff leaves the advertised metadata SSRF protection bypassable with IPv4-mapped IPv6 and rejects documented gateway hostnames, and it still lacks real behavior proof.

## What This Changes

The PR adds hostname/IP filtering to gateway-status target normalization and a new security test file for metadata, wildcard, blocked-hostname, localhost, LAN, and public WebSocket URLs.

## Best Possible Solution

Implement a gateway-status-specific target policy using shared IP literal and special-use classification to block metadata, link-local, wildcard, and malformed IPs while preserving documented localhost, LAN, Bonjour .local, and configured wide-area gateway names.

## Reproduction Assessment

Yes for source-level review: current main accepts ws(s) gateway-status targets into the probe path, and a focused check shows the PR's added string checks still allow mapped IPv6 metadata literals. I did not run a live gateway-status probe.

## Solution Assessment

No. The PR touches the right helper, but the fix should reuse shared IP/special-use classification and apply a gateway-specific hostname policy instead of broad .local/.internal blocking.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Reject mapped IPv6 metadata addresses:** `src/commands/gateway-status/helpers.ts:103-106`
  - body: The new guard only checks literal `0.0.0.0` and hostnames starting with `169.254.`. `URL` canonicalizes `ws://[::ffff:169.254.169.254]:80` to `::ffff:a9fe:a9fe`, so the PR still allows a metadata endpoint through the advertised SSRF mitigation.
  - confidence: 0.94
- **[P2] Preserve documented gateway hostnames:** `src/commands/gateway-status/helpers.ts:110-112`
  - body: The `isBlockedHostname()` check rejects `.local` and `.internal` names, but OpenClaw documents Bonjour `<hostname>.local` and wide-area DNS-SD domains such as `openclaw.internal.` as valid gateway discovery/direct routes. That would silently omit legitimate gateway-status probes.
  - confidence: 0.86
