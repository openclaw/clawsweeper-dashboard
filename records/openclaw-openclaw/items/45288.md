## Summary

Keep this PR open: the hardening direction is useful, but the proposed patch is incomplete for the stated SSRF threat model and conflicts with documented gateway discovery targets.

## What This Changes

The PR adds hostname/IP filtering to gateway-status WebSocket URL normalization and adds a security test file for blocked metadata, internal, wildcard, and allowed gateway targets.

## Best Possible Solution

Implement gateway-specific URL validation that blocks metadata/link-local/wildcard IP literals, including mapped IPv6, while preserving documented local, private-LAN, and discovery-domain gateway targets, with regression tests and a changelog entry.

## Reproduction Assessment

Yes. Node URL canonicalization shows the mapped IPv6 metadata URL becomes `[::ffff:a9fe:a9fe]`, which bypasses the PR's dotted-prefix check, and source/docs inspection shows the `.local` and wide-area-domain regression path.

## Solution Assessment

No. The proposed string-prefix and shared-hostname-block approach is not the narrowest maintainable fix; it should use shared IP literal classification plus gateway-specific hostname policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Reject IPv4-mapped metadata addresses:** `src/commands/gateway-status/helpers.ts:103-106`
  - body: The new guard only blocks `0.0.0.0` and hostnames starting with `169.254.`. A target like `ws://[::ffff:169.254.169.254]:80` canonicalizes to `[::ffff:a9fe:a9fe]`, so the metadata endpoint still bypasses this mitigation.
  - confidence: 0.94
- **[P2] Preserve documented gateway discovery hosts:** `src/commands/gateway-status/helpers.ts:110-112`
  - body: Calling `isBlockedHostname()` rejects `.local` and `.internal` targets, but OpenClaw documents `.local` LAN gateways and wide-area DNS-SD domains such as `openclaw.internal.`. This would drop legitimate configured or explicit gateway probes.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `src/commands/gateway-status/helpers.ts:96-112`
  - body: This is a user-facing `fix(gateway-status)` security hardening, but the PR only changes the helper and tests. Repo policy requires a `CHANGELOG.md` entry for user-facing fixes before merge.
  - confidence: 0.79
