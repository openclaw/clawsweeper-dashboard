## Summary

Keep this PR open: the gateway-status hardening direction is valid, but the patch is incomplete for IPv4-mapped IPv6 metadata targets and rejects documented `.local` gateway routes.

## What This Changes

The PR adds hostname/IP filtering to gateway-status WebSocket URL normalization and adds security tests for metadata, internal, wildcard, localhost, LAN, public-hostname, and protocol cases.

## Best Possible Solution

Keep the hardening direction, but implement a gateway-specific target policy that parses IP literals through shared IP/SSRF helpers, rejects metadata/link-local/wildcard targets including mapped IPv6 forms, and preserves documented `.local` and private-LAN gateway semantics.

## Reproduction Assessment

Yes. The bypass can be reproduced with Node URL canonicalization for IPv4-mapped metadata literals, and current main shows gateway-status forwards normalized explicit/configured targets into the probe path.

## Solution Assessment

No. String-prefix hostname checks plus the generic HTTP blocked-hostname helper are not the narrowest maintainable gateway policy; shared IP classification with gateway-specific route rules is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Reject IPv4-mapped metadata addresses:** `src/commands/gateway-status/helpers.ts:103-106`
  - body: The new block only checks dotted `169.254.` strings and `0.0.0.0`. `URL` canonicalizes `ws://[::ffff:169.254.169.254]:80` to an IPv6 hostname like `::ffff:a9fe:a9fe`, so the metadata endpoint bypasses this mitigation. Use the shared IP parser/SSRF helper and add mapped IPv6 cases before merging.
  - confidence: 0.94
- **[P2] Preserve documented .local gateway targets:** `src/commands/gateway-status/helpers.ts:110-112`
  - body: `isBlockedHostname()` rejects `.local` for both `ws://` and `wss://`, but OpenClaw documents `.local` gateway routes for private LAN/Bonjour flows with TLS or explicit private-network opt-in. This would drop legitimate targets such as `wss://gateway.local:18789` instead of probing them.
  - confidence: 0.84
