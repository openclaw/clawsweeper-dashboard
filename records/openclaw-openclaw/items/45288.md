## Summary

Keep open: this is a plausible security fix surface, but the patch is incomplete because mapped IPv6 metadata addresses bypass the guard, it blocks documented gateway hostnames, and the external PR still lacks after-fix real behavior proof.

## What This Changes

The PR adds string-based gateway-status WebSocket target validation plus a new helper security test suite for metadata, wildcard, hostname, localhost, LAN, and protocol cases.

## Best Possible Solution

Implement a gateway-status-specific target policy that uses shared IP/special-use classification for literals while preserving documented localhost, LAN, Bonjour .local, and wide-area DNS-SD hostnames.

## Reproduction Assessment

Yes for source-level review: current main accepts explicit/configured ws(s) targets into the probe path, and Node URL canonicalization shows mapped IPv6 metadata literals evade the PR's string prefix check. I did not run a live gateway-status probe.

## Solution Assessment

No. The PR touches the right helper, but the maintainable fix should reuse shared IP/special-use classification and use gateway-specific hostname policy instead of broad isBlockedHostname() suffix blocking.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Reject mapped IPv6 metadata addresses:** `src/commands/gateway-status/helpers.ts:103-106`
  - body: The new guard only blocks hostnames equal to 0.0.0.0 or beginning with 169.254. Node canonicalizes `ws://[::ffff:169.254.169.254]:80` to an IPv6 hostname such as `::ffff:a9fe:a9fe`, so this still reaches the probe path despite representing the metadata address the patch is meant to block.
  - confidence: 0.94
- **[P2] Preserve documented gateway hostnames:** `src/commands/gateway-status/helpers.ts:110-112`
  - body: Calling `isBlockedHostname()` rejects `.local` and `.internal` names, but OpenClaw documents Bonjour `<hostname>.local` and wide-area DNS-SD domains such as `openclaw.internal.` as valid gateway routes. With this change, gateway-status would omit legitimate probes for those documented targets.
  - confidence: 0.86
