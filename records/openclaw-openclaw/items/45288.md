## Summary

Keep open: the security concern is plausible and current main still accepts explicit/configured WebSocket targets, but this PR cannot merge as-is because the mitigation leaves a mapped-IPv6 metadata bypass, conflicts with documented gateway discovery hostnames, and lacks real behavior proof.

## What This Changes

The PR adds hostname checks and security tests for gateway-status WebSocket target normalization to reject metadata, wildcard, and blocked hostname inputs.

## Best Possible Solution

Implement gateway-specific URL validation that blocks metadata, link-local, wildcard, and malformed IP literals including mapped IPv6, while preserving documented LAN and wide-area gateway discovery hosts.

## Reproduction Assessment

Yes. Source inspection shows explicit/configured WebSocket targets pass through `normalizeWsUrl()` into `probeGateway()`, and Node URL canonicalization demonstrates the PR branch's mapped-IPv6 metadata bypass; repo tests were not run because this was a read-only review.

## Solution Assessment

No. The string-prefix and shared-hostname-denylist approach misses mapped IPv6 metadata addresses and rejects documented `.local`/`.internal` gateway discovery domains; the safer fix should reuse shared IP-literal classification with a gateway-specific hostname policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Reject mapped IPv6 metadata addresses:** `src/commands/gateway-status/helpers.ts:103-106`
  - body: The new guard only blocks `0.0.0.0` and dotted hosts starting with `169.254.`. Node canonicalizes `ws://[::ffff:169.254.169.254]:80` to `[::ffff:a9fe:a9fe]`, so the metadata endpoint still bypasses this mitigation.
  - confidence: 0.94
- **[P2] Preserve documented gateway discovery hosts:** `src/commands/gateway-status/helpers.ts:110-112`
  - body: Calling `isBlockedHostname()` here rejects `.local` and `.internal` targets, but OpenClaw documents LAN `local.` browsing and wide-area DNS-SD domains such as `openclaw.internal.`, so this would drop legitimate explicit or configured gateway probes.
  - confidence: 0.87
