## Summary

Keep open. Current main does not implement gateway-status target filtering, and the PR’s SSRF hardening is still incomplete because IPv4-mapped metadata addresses bypass the new string checks while the generic hostname block would drop documented `.local` gateway targets.

## What This Changes

The PR adds hostname/IP filtering to gateway-status WebSocket URL normalization and a new security test file for blocked metadata/internal/wildcard targets and allowed gateway endpoints.

## Best Possible Solution

Keep the hardening direction, but replace string-prefix checks with a gateway-specific policy that uses the shared IP classifier for literals, blocks metadata/link-local/wildcard targets including mapped IPv6 forms, and preserves documented `.local` and private-LAN gateway routes.

## Reproduction Assessment

Yes. The bypass is reproducible with Node URL canonicalization, and source inspection shows the PR’s proposed checks would miss the canonical mapped IPv6 hostname while rejecting `.local` through the shared hostname block.

## Solution Assessment

No. The proposed generic hostname block plus dotted-prefix IP checks are not the narrowest maintainable fix; shared literal-IP classification with gateway-specific host rules is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Reject IPv4-mapped metadata addresses:** `src/commands/gateway-status/helpers.ts:103-106`
  - body: The new guard only checks `0.0.0.0` and dotted hostnames starting with `169.254.`. `ws://[::ffff:169.254.169.254]:80` canonicalizes to an IPv6 hostname like `[::ffff:a9fe:a9fe]`, so the metadata endpoint still bypasses this mitigation; use the shared IP classifier and add mapped-form coverage.
  - confidence: 0.94
- **[P2] Preserve .local gateway targets:** `src/commands/gateway-status/helpers.ts:110-112`
  - body: `isBlockedHostname()` rejects `.local` for both `ws://` and `wss://`, but OpenClaw documents `.local`/Bonjour gateway routes for private LAN flows. This would drop legitimate targets such as `wss://gateway.local:18789` instead of probing them.
  - confidence: 0.84
