## Summary

Keep open: the PR addresses a real gateway-status SSRF gap, but the proposed mitigation is incomplete and not merge-ready because mapped IPv6 metadata addresses bypass the new guard, documented gateway hostnames would be dropped, real behavior proof is missing, and the branch is currently dirty against main.

## What This Changes

The PR adds hostname filtering to gateway-status WebSocket target normalization and adds focused security tests for explicit probe URLs.

## Best Possible Solution

Implement gateway-status-specific target validation that reuses shared link-local/cloud-metadata IP classifiers, preserves supported private gateway routes, adds mapped-IPv6 regression coverage, and is backed by redacted real CLI proof.

## Reproduction Assessment

Yes, for source-level review: current main accepts explicit/configured `ws(s)` targets and passes them to `probeGateway()`, and Node URL canonicalization shows mapped-IPv6 metadata forms bypass the PR's string-prefix guard.

## Solution Assessment

No. The PR touches the right helper, but the maintainable fix should use shared IP/link-local/cloud-metadata classifiers with a gateway-specific hostname policy instead of broad `isBlockedHostname()` suffix blocking.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Reject mapped IPv6 metadata addresses:** `src/commands/gateway-status/helpers.ts:103-106`
  - body: The new guard only rejects `0.0.0.0` and hosts starting with `169.254.`, but `ws://[::ffff:169.254.169.254]:80` canonicalizes to `[::ffff:a9fe:a9fe]` and bypasses this branch. That leaves the SSRF mitigation incomplete for the metadata endpoint class this PR is trying to block.
  - confidence: 0.94
- **[P2] Preserve documented gateway hostnames:** `src/commands/gateway-status/helpers.ts:108-109`
  - body: Calling `isBlockedHostname()` rejects `.local` and `.internal` targets, but OpenClaw documents `.local` LAN routes and wide-area DNS-SD/private gateway domains such as `openclaw.internal.`. This would silently drop legitimate gateway-status probes.
  - confidence: 0.86
