## Summary

Keep open. This is a plausible security hardening PR, but the patch is not mergeable as-is because it leaves a mapped-IPv6 metadata bypass, blocks documented gateway hostnames, lacks after-fix real behavior proof, and currently conflicts with main.

## What This Changes

The PR adds `normalizeWsUrl()` hostname/IP filtering and a new gateway-status security test file for WebSocket target SSRF cases.

## Best Possible Solution

Implement gateway-status-specific target validation that reuses shared IP/link-local/cloud-metadata classifiers for literals, preserves documented localhost/LAN/.local/wide-area DNS-SD routes, and adds mapped-IPv6 regression coverage plus real CLI proof.

## Reproduction Assessment

Yes, for source-level review. Current main accepts explicit/configured ws(s) targets and passes `target.url` to `probeGateway()`, and Node URL canonicalization shows the PR's mapped-IPv6 metadata case becomes `[::ffff:a9fe:a9fe]` and bypasses the string-prefix guard.

## Solution Assessment

No. The PR touches the right helper, but the maintainable fix should use shared IP/link-local/cloud-metadata classifiers and a gateway-specific hostname policy instead of broad `isBlockedHostname()` suffix blocking.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Reject mapped IPv6 metadata addresses:** `src/commands/gateway-status/helpers.ts:103-106`
  - body: The new guard only rejects `0.0.0.0` and hosts starting with `169.254.`, but `URL` canonicalizes `ws://[::ffff:169.254.169.254]:80` to `[::ffff:a9fe:a9fe]`. That still reaches `resolveTargets()` as an explicit gateway probe target, leaving the metadata SSRF mitigation bypassable.
  - confidence: 0.94
- **[P2] Preserve documented gateway hostnames:** `src/commands/gateway-status/helpers.ts:110-112`
  - body: Calling `isBlockedHostname()` rejects `.local` and `.internal` names, but OpenClaw documents Bonjour `<hostname>.local` and wide-area DNS-SD domains such as `openclaw.internal.` as valid gateway routes. This would silently drop legitimate gateway-status probes.
  - confidence: 0.86
