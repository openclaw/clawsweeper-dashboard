## Summary

Keep open: the reported gateway-status SSRF class is plausible, but this patch is not merge-ready because the mitigation has a mapped-IPv6 metadata bypass, conflicts with documented private gateway hostnames, lacks real behavior proof, and is currently conflicting.

## What This Changes

The PR adds hostname validation and focused security tests for gateway-status WebSocket target normalization to reject metadata, wildcard, `.internal`, and `.local` probe targets.

## Best Possible Solution

Implement gateway-status-specific validation that blocks link-local/cloud-metadata IP literals via shared classifiers, preserves documented private gateway routes, and adds mapped-IPv6 regression coverage.

## Reproduction Assessment

Yes, for source-level review: current main accepts explicit/configured `ws(s)` targets and passes them into `probeGateway()`, and Node URL canonicalization shows mapped-IPv6 metadata forms bypass the PR's string-prefix guard.

## Solution Assessment

No. The PR touches the right helper, but the maintainable fix should use shared link-local/cloud-metadata IP classifiers with a gateway-specific hostname policy instead of broad `isBlockedHostname()` suffix blocking.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Reject mapped IPv6 metadata addresses:** `src/commands/gateway-status/helpers.ts:103-106`
  - body: The new guard only rejects `0.0.0.0` and hosts starting with `169.254.`, but `ws://[::ffff:169.254.169.254]:80` canonicalizes to `[::ffff:a9fe:a9fe]` and bypasses this branch. That leaves the SSRF mitigation incomplete for the metadata endpoint class this PR is trying to block.
  - confidence: 0.94
- **[P2] Preserve documented gateway hostnames:** `src/commands/gateway-status/helpers.ts:108-109`
  - body: Calling `isBlockedHostname()` rejects `.local` and `.internal` targets, but OpenClaw documents `.local` LAN routes and wide-area DNS-SD/private gateway domains such as `openclaw.internal.`. This would silently drop legitimate gateway-status probes.
  - confidence: 0.86
