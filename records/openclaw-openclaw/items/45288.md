## Summary

Keep open: this addresses a real source-reproducible gateway-status SSRF class, but the proposed patch is incomplete and over-broad, and current main has not implemented a replacement fix.

## What This Changes

The PR adds hostname validation to `normalizeWsUrl()` and a new gateway-status security test file for WebSocket target SSRF cases.

## Best Possible Solution

Implement gateway-specific URL validation that blocks metadata, link-local, wildcard, and malformed IP literals including mapped IPv6, while preserving documented LAN and wide-area gateway discovery hosts.

## Reproduction Assessment

Yes. Current main source shows explicit/configured WebSocket targets pass through `normalizeWsUrl()` with only scheme filtering into `probeGateway()`, and Node URL canonicalization demonstrates the PR branch's IPv4-mapped metadata bypass.

## Solution Assessment

No. The PR's string-prefix and shared-hostname-denylist approach misses mapped IPv6 metadata addresses and blocks documented `.local`/`.internal` gateway discovery; the safer fix should reuse shared IP-literal classification with a gateway-specific hostname policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Reject IPv4-mapped metadata addresses:** `src/commands/gateway-status/helpers.ts:103-106`
  - body: The new guard only blocks `0.0.0.0` and dotted hosts starting with `169.254.`. Node canonicalizes `ws://[::ffff:169.254.169.254]:80` to `[::ffff:a9fe:a9fe]`, so this metadata endpoint still bypasses the mitigation.
  - confidence: 0.94
- **[P2] Preserve documented gateway discovery hosts:** `src/commands/gateway-status/helpers.ts:110-112`
  - body: Calling `isBlockedHostname()` here rejects `.local` and `.internal` targets, but OpenClaw documents `local.` LAN gateways and wide-area DNS-SD domains such as `openclaw.internal.`, so this would drop legitimate explicit or configured gateway probes.
  - confidence: 0.87
