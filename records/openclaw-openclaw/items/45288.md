## Summary

Keep open: the PR is a plausible security hardening, but the proposed mitigation is incomplete for mapped-IPv6 metadata targets, over-blocks documented gateway hostname patterns, and still lacks after-fix real behavior proof.

## What This Changes

The PR adds WebSocket target hostname checks to gateway-status URL normalization and a new security test file for metadata, wildcard, and blocked-hostname rejection.

## Best Possible Solution

Implement gateway-specific URL validation that blocks metadata, link-local, wildcard, and malformed IP literals using shared IP classification while preserving documented gateway hostnames.

## Reproduction Assessment

Yes, by source inspection and a focused Node URL canonicalization check. resolveTargets() feeds explicit/configured ws(s) targets into probeGateway(), and mapped-IPv6 metadata URLs avoid the PR's prefix/set checks; I did not run the PR branch because this review is read-only.

## Solution Assessment

No. The string-prefix plus shared-hostname-denylist approach misses mapped IPv6 metadata addresses and rejects documented gateway host suffixes; a gateway-specific policy on top of the shared IP-literal classifier is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Reject mapped IPv6 metadata addresses:** `src/commands/gateway-status/helpers.ts:103-106`
  - body: The new guard only rejects `0.0.0.0` and dotted hosts starting with `169.254.`. Node normalizes `ws://[::ffff:169.254.169.254]:80` to `[::ffff:a9fe:a9fe]`, so the same metadata endpoint still bypasses this SSRF fix.
  - confidence: 0.94
- **[P2] Preserve documented gateway hostnames:** `src/commands/gateway-status/helpers.ts:110-112`
  - body: Calling `isBlockedHostname()` here rejects `.local` and `.internal`, but OpenClaw documents LAN `local.` and wide-area domains such as `openclaw.internal.` for gateway discovery. That would drop legitimate explicit or configured gateway-status probes instead of only blocking metadata hosts.
  - confidence: 0.86
