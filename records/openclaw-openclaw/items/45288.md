- **[P1] Reject mapped IPv6 metadata targets:** `src/commands/gateway-status/helpers.ts:103-106`
  - body: The new guard only rejects dotted hosts that equal `0.0.0.0` or start with `169.254.`. `URL` canonicalizes `ws://[::ffff:169.254.169.254]:80` to an IPv6 hostname such as `::ffff:a9fe:a9fe`, so the explicit gateway-status target still gets accepted and the SSRF fix is incomplete.
  - confidence: 0.95
- **[P1] Keep private gateway DNS names probeable:** `src/commands/gateway-status/helpers.ts:108-109`
  - body: Using the shared hostname deny-list here rejects `.local` and `.internal` gateway targets, but current gateway docs support private `.local` and trusted private-DNS gateway URLs. That can silently remove an existing configured `gateway.remote.url` or `--url` target from status probes during upgrade.
  - confidence: 0.88
