## Summary

Keep open: current main still accepts arbitrary explicit/configured gateway-status ws(s) targets into the probe path, but this PR is not mergeable as-is because its hostname policy has a metadata-address bypass, rejects documented gateway hostnames, and lacks real behavior proof.

## What This Changes

The PR adds normalizeWsUrl hostname/IP filtering for gateway-status WebSocket targets plus a new helper security test file for metadata, wildcard, hostname, localhost, LAN, and protocol cases.

## Best Possible Solution

Implement a gateway-status-specific target policy that reuses shared IP/link-local/cloud-metadata classification for literals while preserving documented localhost, LAN, Bonjour .local, and configured wide-area DNS-SD gateway hostnames.

## Reproduction Assessment

Yes for source-level review: current main accepts explicit/configured ws(s) targets into resolveTargets() and runGatewayStatusProbePass() sends target.url to probeGateway(). I did not run a live gateway-status command, but Node URL canonicalization confirms the PR's mapped IPv6 metadata bypass.

## Solution Assessment

No. The PR touches the right helper, but the maintainable fix should reuse the shared IP/special-use classifiers and apply a gateway-specific hostname policy instead of broad isBlockedHostname() suffix blocking.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Reject mapped IPv6 metadata addresses:** `src/commands/gateway-status/helpers.ts:103-106`
  - body: The new guard only blocks hosts equal to 0.0.0.0 or starting with 169.254.; Node canonicalizes `ws://[::ffff:169.254.169.254]:80` to an IPv6 hostname such as `[::ffff:a9fe:a9fe]`, so the metadata endpoint still reaches the probe path despite this SSRF mitigation.
  - confidence: 0.94
- **[P2] Preserve documented gateway hostnames:** `src/commands/gateway-status/helpers.ts:110-112`
  - body: Calling `isBlockedHostname()` rejects `.local` and `.internal` names, but OpenClaw documents Bonjour `<hostname>.local` and wide-area DNS-SD domains such as `openclaw.internal.` as valid gateway routes, so this would omit legitimate gateway-status probes.
  - confidence: 0.86
