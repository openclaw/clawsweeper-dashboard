## Summary

Keep this PR open. Current main still has the gateway-status target validation gap, but this PR should not merge as-is because its string checks miss IPv4-mapped IPv6 metadata addresses and reject documented `.local` gateway targets.

## What This Changes

The PR adds gateway-status WebSocket hostname/IP filtering in `normalizeWsUrl()` plus a new security test file for blocked and allowed probe URLs.

## Best Possible Solution

Keep the hardening direction, but implement a gateway-specific target policy that uses the shared IP parser/classifier for literal addresses, rejects metadata/link-local/wildcard targets including mapped IPv6 forms, and preserves documented private-LAN and `.local` gateway semantics.

## Reproduction Assessment

Yes. The issue is reproducible by resolving explicit gateway-status targets through the current main path and by checking Node URL canonicalization for `ws://[::ffff:169.254.169.254]:80`, which bypasses the PR's dotted-prefix guard.

## Solution Assessment

No. String-prefix host checks plus the generic HTTP SSRF hostname block are not the narrowest maintainable gateway-status fix; shared IP classification with gateway-specific hostname rules is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Reject IPv4-mapped metadata addresses:** `src/commands/gateway-status/helpers.ts:103-106`
  - body: The new guard only checks dotted `169.254.` strings and `0.0.0.0`. `ws://[::ffff:169.254.169.254]:80` canonicalizes to an IPv6 hostname like `::ffff:a9fe:a9fe`, so the metadata endpoint bypasses the SSRF mitigation this PR is meant to add.
  - confidence: 0.94
- **[P2] Preserve documented `.local` gateway targets:** `src/commands/gateway-status/helpers.ts:110-112`
  - body: `isBlockedHostname()` rejects `.local` hosts, but OpenClaw documents `.local` gateway routes for private LAN and Bonjour flows. This would drop legitimate status targets such as `wss://gateway.local:18789` instead of probing them.
  - confidence: 0.84
