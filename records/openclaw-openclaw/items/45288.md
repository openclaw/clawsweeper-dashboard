## Summary

Keep this PR open, but do not merge it as-is. Current main still lacks gateway-status SSRF filtering, while the PR’s string-prefix approach leaves an IPv4-mapped IPv6 metadata bypass and rejects documented `.local` gateway targets.

## What This Changes

The PR adds hostname/IP filtering to `normalizeWsUrl()` and a new gateway-status security test file for blocked metadata, internal, wildcard, and allowed gateway targets.

## Best Possible Solution

Keep the hardening direction, but implement a gateway-specific validator that uses the shared IP classifier for literals, blocks metadata/link-local/wildcard targets including mapped IPv6 forms, and preserves documented `.local` and private-LAN gateway routes.

## Reproduction Assessment

Yes. The bypass is reproducible with Node URL canonicalization, and source inspection shows the PR’s proposed checks would miss the canonical mapped IPv6 hostname while rejecting `.local` through the shared hostname block.

## Solution Assessment

No. The proposed generic hostname block plus dotted-prefix IP checks are not the narrowest maintainable fix; shared literal-IP classification with gateway-specific host rules is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Reject IPv4-mapped metadata addresses:** `src/commands/gateway-status/helpers.ts:103-106`
  - body: The new guard only checks `0.0.0.0` and dotted hostnames starting with `169.254.`. `ws://[::ffff:169.254.169.254]:80` canonicalizes to `[::ffff:a9fe:a9fe]`, so the metadata endpoint still bypasses this mitigation; use the shared IP classifier and add mapped-form coverage.
  - confidence: 0.94
- **[P2] Preserve .local gateway targets:** `src/commands/gateway-status/helpers.ts:110-112`
  - body: `isBlockedHostname()` rejects `.local` for both `ws://` and `wss://`, but OpenClaw documents `.local`/Bonjour gateway routes for private LAN flows. This would drop legitimate targets such as `wss://gateway.local:18789` instead of probing them.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `src/commands/gateway-status/helpers.ts:96-112`
  - body: This is a user-facing `fix(gateway-status)` security hardening, but the PR only changes the helper and tests. Repo policy requires a `CHANGELOG.md` entry for user-facing fixes before merge.
  - confidence: 0.79
