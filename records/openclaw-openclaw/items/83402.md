## Summary

Keep open. ## What This Changes

The issue asks OpenClaw to make cron embedded agent tool schemas compatible with MiMo `mimo-v2.5-pro` when the model is configured through a custom Xiaomi Token Plan OpenAI-compatible provider.

## Best Possible Solution

Maintainers should choose either a migration/doctor path from custom Xiaomi aliases to `xiaomi-token-plan` or a narrowly scoped endpoint/model-based compat inheritance rule with cron regression coverage.
