## Summary

Keep open. Current main and v2026.5.22 include the bundled Xiaomi empty-array-items fix and the MiMo reasoning replay fix, but the reported `xiaomi-coding` Token Plan route is a custom OpenAI-completions provider and source inspection still shows a plausible path where Xiaomi-owned tool-schema compat does not reach that custom provider before cron embedded runs submit tools.

## What This Changes

The issue reports custom Xiaomi Token Plan MiMo cron embedded agent runs failing with `400 Param Incorrect` schema rejection while normal non-cron sessions still work.

## Best Possible Solution

Carry Xiaomi-native Token Plan endpoint/model compatibility into custom OpenAI-completions provider ids before tool-schema normalization, with focused regressions for cron-style embedded tool payloads and no new config knob.

## Reproduction Assessment

No high-confidence live reproduction is available because this review lacks a Xiaomi Token Plan credential and raw rejected payload. The reporter supplied concrete cron logs, and current source still shows a custom-provider route that can miss the Xiaomi schema compat before tool serialization.

## Solution Assessment

Yes: the maintainable fix is to make provider/model compatibility resolution cover known Xiaomi Token Plan endpoints for custom provider ids, instead of adding a cron-only workaround or asking users to rename providers.
