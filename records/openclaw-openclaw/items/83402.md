Keep open. The report is credible: current main has a Xiaomi-only empty-array-items schema compatibility path, but a custom `xiaomi-coding` OpenAI-completions provider can still miss that provider-owned compat before cron embedded runs submit tools.

## What This Changes

The issue reports MiMo mimo-v2.5-pro rejecting cron embedded agent tool schemas with `400 Param Incorrect` through a custom Xiaomi-compatible OpenAI completions provider after the 2026.5.12 Xiaomi schema fix.

## Best Possible Solution

Extend provider/model compatibility so Xiaomi-native custom OpenAI-completions routes get the same tool-schema cleanup as the bundled Xiaomi provider, with regression coverage for the cron embedded-run path.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not live: the reporter supplied concrete cron logs and current source shows a plausible custom-provider compat gap, but this review did not establish a failing MiMo Token Plan call on current main.

## Solution Assessment

Is this the best way to solve the issue? Yes: fix provider/model compat resolution for Xiaomi-native custom routes before tool submission, instead of adding a cron-only workaround or a new public config knob.
