## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/83402. Current main's Xiaomi schema fix from https://github.com/openclaw/openclaw/pull/82575 sets `omitEmptyArrayItems` only for models resolved through the bundled `xiaomi` provider. Custom OpenAI-completions providers pointing at known Xiaomi/MiMo Token Plan endpoints, such as `https://token-plan-cn.xiaomimimo.com/v1` with model `mimo-v2.5-pro`, can miss that compat and cron embedded tools may still send synthetic `items: {}`. Implement a narrow provider/model compat path so Xiaomi-native custom routes receive the same empty-array-items cleanup before `normalizeAgentRuntimeTools` and `buildOpenAICompletionsParams`. Do not add a public config knob, do not add a cron-only special case, and preserve typed/false/null `items` behavior. Add focused regression coverage for the custom provider route and the final tool schema payload. Add a changelog entry if the fix is user-facing.
