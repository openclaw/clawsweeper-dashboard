## Summary

Keep open. Current main still suppresses default-verbosity exec/bash failures before the mutating-tool warning path, and tests still assert that non-timeout `Command not found` exec errors produce no failure payload.

## What This Changes

The issue asks OpenClaw to report failed exec commands instead of allowing success-looking assistant output after failed file creation or execution.

## Best Possible Solution

Update the embedded-run payload policy so failed mutating exec/bash actions, including command-not-found, emit a concise default-visible failure payload while preserving explicit warning suppression, verbose raw details, `sessions_send` suppression, and timed-out cron behavior.

## Reproduction Assessment

Yes. Current source and tests provide a high-confidence unit-level reproduction through `buildEmbeddedRunPayloads` with `lastToolError.toolName = "exec"`, `error = "Command not found"`, default verbosity, and assistant text; the live NemoClaw/NVIDIA path was not rerun.

## Solution Assessment

Yes. A narrow payload-policy fix is the best path because the exec runtime already classifies command-not-found and the subscribe handler already retains failed tool state.
