## Summary

Keep open. Current main still has a prompt-visible Slack skill that names a `slack` tool, but Claude Code only gets OpenClaw MCP-prefixed bundle tools plus the generic `message` tool; the requested Slack workspace/app admin actions are also still absent and need a maintainer policy decision.

## What This Changes

The issue asks to expose Slack actions to Claude Code/ToolSearch and add Slack workspace/app administration actions with token/config gating.

## Best Possible Solution

Align the Slack skill with the actually callable Claude surface, then add a disabled-by-default Slack admin action group with SecretRef-compatible credentials, scope docs, and focused dispatcher/config tests after maintainers choose the authority model.

## Reproduction Assessment

Yes. A high-confidence source-level path exists: the Slack skill advertises `slack`, Claude Code allows only `mcp__openclaw__*`, bundle MCP discovery skips channel plugins, and the Slack dispatcher has no requested admin actions.

## Solution Assessment

Unclear. The proposed MCP bridge plus admin group is plausible, but the best fix depends on maintainer choices about the Claude tool contract and high-privilege Slack token policy; a narrower interim fix could also stop loading or rewrite the misleading Slack skill for harnesses that cannot call it.
