## Summary

Keep open. Current main still lacks a Slack-specific Claude/MCP tool bridge and the requested Slack admin action group; `userToken` read/write plumbing now exists, but `appConfigToken` and the workspace/app administration endpoints are still absent and need maintainer security direction.

## What This Changes

The issue requests that Slack channel actions become callable from Claude Code tool search and that Slack workspace/app administration actions be added behind explicit token/config gates.

## Best Possible Solution

Choose the Claude/MCP exposure contract first, then split the work into a Slack action bridge or skill-loading fix plus a separate disabled-by-default Slack admin action group with documented scopes and SecretRef-compatible tokens.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: the Slack skill advertises `slack`, Claude Code is limited to `mcp__openclaw__*`, the channel MCP server exposes only generic tools, and the requested admin APIs are not present in the Slack plugin.

## Solution Assessment

Unclear. The reporter’s proposed bridge directions are plausible, but the maintainable path should be chosen by maintainers before adding high-privilege Slack admin actions.
