## Summary

Keep open. Current `main` still has the reported Slack/Claude surface mismatch, and the Slack workspace/app admin actions are a new high-privilege capability that needs maintainer direction before implementation.

## What This Changes

The issue asks for Slack channel actions to be callable from Claude Code and for new Slack workspace/app administration actions with token plumbing and action-group gating.

## Best Possible Solution

Align the Slack skill with the callable Claude surface, then add a disabled-by-default Slack admin action group with SecretRef-compatible credentials, scope docs, and focused dispatcher/config tests after maintainers choose the authority model.

## Reproduction Assessment

Yes. A high-confidence source-level path exists: the Slack skill advertises `slack`, Claude Code allows only `mcp__openclaw__*`, bundle MCP loading skips channel plugins, and the Slack dispatcher/config has no requested admin group.

## Solution Assessment

Unclear. The proposed MCP bridge plus admin group is plausible, but the best fix depends on maintainer choices about the Claude tool contract and Slack admin-token authority; a narrower interim fix could also stop loading or rewrite the misleading Slack skill for harnesses that cannot call it.
