## Summary

Keep open. Current main still has a prompt-visible Slack skill that describes a `slack` tool while Claude Code is constrained to OpenClaw MCP-prefixed tools, and the requested Slack workspace/app administration actions are not implemented.

## What This Changes

The issue requests exposing Slack channel actions to Claude Code/MCP discovery and adding gated Slack workspace/app administration actions.

## Best Possible Solution

Align Slack skill loading with the callable Claude surface first, then add a disabled-by-default Slack admin action group with SecretRef-compatible tokens, scope documentation, and focused dispatcher/config tests once maintainers choose the exposure shape.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: the Slack skill advertises `slack`, Claude Code only allows `mcp__openclaw__*`, bundle MCP discovery skips channel plugins, and the Slack dispatcher/docs lack the requested admin actions.

## Solution Assessment

Unclear. The proposed MCP bridge and admin action group are plausible, but maintainers should first choose the Claude/MCP contract and high-privilege Slack token policy.
