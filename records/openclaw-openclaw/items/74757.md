## Summary

Keep open. Current main still leaves the Slack skill/tool exposure contract inconsistent for Claude sessions and still lacks the requested Slack workspace/app administration actions, so this needs maintainer product and security direction before a repair PR.

## What This Changes

Expose Slack actions to Claude-discoverable tooling and add Slack workspace/app admin actions with token plumbing and an action-group gate.

## Best Possible Solution

Yes. A high-confidence source-level reproduction exists: the Slack skill advertises `slack`, current Claude/MCP exposure uses the shared `message`/bundle surfaces, and the Slack dispatcher has no admin action cases for the requested operations.

Unclear. The requested direction is plausible, but the issue intentionally bundles a tool-exposure contract decision with high-privilege Slack admin capabilities, so maintainers should pick the policy before implementation.
