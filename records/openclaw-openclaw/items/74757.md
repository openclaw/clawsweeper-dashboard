## Summary

Keep open. Current main only partially overlaps the request: Slack now has `userToken` plumbing, but the prompt-visible `slack` skill still does not correspond to a Slack-named Claude/MCP tool, and the requested workspace/app admin actions plus `appConfigToken` policy are still missing.

## What This Changes

Request to make Slack actions callable/discoverable from Claude Code and add Slack workspace/app administration actions with token plumbing and action gating.

## Best Possible Solution

Align Slack skill guidance with the actual Claude/MCP callable surface first, then add a disabled-by-default Slack admin action group with SecretRef-compatible tokens, documented Slack scopes, and tests for token selection and action gating.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: the Slack skill advertises `slack`, Claude only allows `mcp__openclaw__*`, bundle discovery skips channel plugins, and the Slack dispatcher has no cases for the requested admin operations.

## Solution Assessment

Unclear. The proposed direction is plausible, but maintainers should first choose the bridge versus skill-loading behavior and the policy for high-privilege Slack admin tokens.
