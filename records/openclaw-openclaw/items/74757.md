## Summary

Keep open. Current main still has a mismatch between the prompt-visible Slack skill and the callable Claude/MCP surface, and the requested Slack workspace/app administration actions are not implemented; the remaining work needs maintainer product and security direction.

## What This Changes

The issue requests making Slack actions callable from Claude sessions and adding Slack workspace/app administration actions such as channel creation, invites, user lookup, member listing, app manifest changes, and archival.

## Best Possible Solution

Align Slack skill loading with the actual callable Claude surface first, then add a disabled-by-default Slack admin action group with SecretRef-compatible token plumbing, documented Slack scopes, and focused dispatcher/config tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: the Slack skill advertises `slack`, Claude is constrained to `mcp__openclaw__*`, bundle MCP discovery skips channel plugins, and the Slack dispatcher lacks the requested admin actions.

## Solution Assessment

Unclear. The reporter's proposed MCP bridge and admin action group are plausible, but maintainers should first choose the Claude/MCP exposure shape and the admin-token policy because those choices affect security and plugin architecture.
