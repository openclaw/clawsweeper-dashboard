## Summary

Keep open. Current main has partial Slack user-token plumbing, but the reported Claude Code visibility mismatch remains source-reproducible and the requested Slack workspace/app admin actions are still absent; this needs maintainer direction before implementation.

## What This Changes

Add or adjust Slack agent tooling so Claude Code can call Slack actions and optionally perform Slack admin operations with user/app-configuration token support.

## Best Possible Solution

Align the Slack skill with the actually callable Claude surface, then add a disabled-by-default Slack admin action group with SecretRef-compatible credentials, scope docs, and focused dispatcher/config tests after maintainers choose the authority model.

## Reproduction Assessment

Yes. A high-confidence source-level path exists: Slack is channel-plugin registered, Claude allows only OpenClaw MCP-prefixed tools, the skill still tells agents to call `slack`, and the dispatcher lacks the requested admin actions.

## Solution Assessment

Unclear. The proposed MCP bridge plus admin group is plausible, but the durable fix depends on maintainer choices about the Claude tool contract and Slack admin-token authority.
