Keep open. Current main still has the central Slack tool-surface mismatch and lacks the requested Slack workspace/app admin operations; the remaining work is valid but needs maintainer product/security direction before implementation.

## What This Changes

## Best Possible Solution

Choose the Slack exposure model first, then implement a harness-safe skill change and/or MCP bridge plus a disabled-by-default admin action group with SecretRef credentials, scope docs, and focused tests.

## Solution Assessment

Unclear until maintainers choose the authority model. A thin MCP bridge plus default-off admin group is plausible, but hiding or rewriting the misleading skill may be the safer interim fix for harnesses that cannot call Slack actions.
