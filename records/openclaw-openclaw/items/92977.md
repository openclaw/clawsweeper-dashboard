## What This Changes

The PR adds optional chaining around `model.baseUrl.includes(...)` in `src/agents/sessions/sdk.ts` so models without `baseUrl` do not crash in attribution header checks.

## Best Possible Solution

Land the null guard with focused regression coverage and redacted after-fix Bedrock output, then let the linked issue close after merge.

## Reproduction Assessment

Yes from source inspection: on current main, telemetry is enabled by default and `createAgentSession()` calls `getAttributionHeaders()` before `streamSimple()`, where a Bedrock-shaped model without `baseUrl` throws on `.includes()`. I did not run a live AWS Bedrock call in this read-only review.

## Solution Assessment

Yes for immediate mitigation: optional chaining is the narrow fix at the crash site and preserves the existing provider checks. The cleaner long-term shape is to replace the bespoke helper with the existing optional-`baseUrl` provider attribution resolver, but that is not required to fix this shipped crash.
