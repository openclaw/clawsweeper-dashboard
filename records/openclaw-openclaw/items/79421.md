## Summary

Keep open: current main still default-denies non-bundled conversation hooks without a warning-level operational signal, and `openclaw plugins doctor` still omits warn-level diagnostics.

## What This Changes

The issue requests visible warning output when conversation-hook registration is blocked by missing `plugins.entries.<id>.hooks.allowConversationAccess=true`.

## Best Possible Solution

Keep the default-deny policy, but promote blocked conversation-hook registration diagnostics to warning-level startup logging while preserving inspect diagnostics; doctor warning surfacing can be included if kept narrow.

## Reproduction Assessment

Yes, source-reproducible: load a non-bundled plugin that registers `llm_input`, `llm_output`, `before_agent_finalize`, or `agent_end` without `allowConversationAccess=true`. Current tests and source show the hooks are dropped and only warn diagnostics are retained, while gateway/doctor do not expose them as warning-level operational output.

## Solution Assessment

Yes. The narrow maintainable fix is to keep the access gate unchanged and add a warning-level loader or gateway log path, with focused regression tests; changing the policy default or adding config would be broader than needed.
