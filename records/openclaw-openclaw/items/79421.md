## Summary

Keep open: current main still blocks non-bundled conversation hooks without `allowConversationAccess` using only warn diagnostics, while gateway startup logs non-error diagnostics at info level and plugin doctor ignores warn diagnostics.

## What This Changes

The issue requests visible operational warnings when conversation-hook registration is denied by the plugin loader policy.

## Best Possible Solution

Keep the default-deny policy, but surface denied conversation-hook registrations through warning-level startup logging and add focused regression coverage; including warn diagnostics in plugin doctor would make the operator path clearer.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: a non-bundled plugin registering conversation hooks without `allowConversationAccess` gets zero typed hooks and warn diagnostics only. I did not run a live gateway reproduction because this review is read-only.

## Solution Assessment

Yes. The narrow maintainable fix is to preserve the access gate and existing diagnostics while promoting the denial to a warning-level operational surface, with tests around loader diagnostics and gateway or doctor output.
