## Summary

Keep open: current main still blocks non-bundled conversation hooks without `allowConversationAccess` by recording only warn diagnostics, while gateway startup downgrades non-error diagnostics to info and `openclaw plugins doctor` filters warn diagnostics out.

## What This Changes

The issue asks OpenClaw to make denied conversation-hook registration visible in operational warnings instead of only plugin-scoped diagnostics.

## Best Possible Solution

Keep the default-deny policy, but promote the existing warn diagnostic to warning-level startup logging and optionally include warn diagnostics in plugin doctor output while preserving inspect diagnostics.

## Reproduction Assessment

Yes. Current source and the existing loader test show that a non-bundled plugin registering conversation hooks without `allowConversationAccess` gets zero typed hooks and only warn diagnostics in the registry report.

## Solution Assessment

Yes. The narrow maintainable fix is to route the existing denial diagnostic to `logger.warn` during plugin load, with optional doctor surfacing, instead of changing the access policy or adding a new permission model.
