## Summary

Keep open: current main and `v2026.6.8` still block non-bundled conversation hooks with a warn diagnostic, route non-error gateway plugin diagnostics through the info logger, and omit warn diagnostics from `openclaw plugins doctor`; the docs/type discoverability pieces are now improved but the operator-warning path remains unresolved.

## What This Changes

The issue requests better operator and plugin-author visibility when the plugin loader blocks non-bundled conversation-hook registration for missing `plugins.entries.<id>.hooks.allowConversationAccess=true`.

## Best Possible Solution

Keep the secure default-deny gate, route blocked conversation-hook diagnostics as warning-level startup output, and add a maintainer-approved `plugins doctor` path for warning-level hook-policy denials.

## Reproduction Assessment

Yes, from source: current main still drops non-bundled conversation hooks without `allowConversationAccess=true`, logs the resulting warn diagnostic through the gateway info path, and excludes warn diagnostics from `plugins doctor`. I did not run a live gateway because this review is read-only and the source/tests cover the path directly.

## Solution Assessment

No final code solution is present on current main. The best fix is narrower than the original report now that docs are improved: promote the runtime warning surface and add a maintainer-approved doctor warning path without changing the default-deny policy.
