## Summary

Keep open: current main and v2026.6.10 still default-deny non-bundled conversation hooks with only a warn diagnostic, still route non-error plugin diagnostics through the gateway info path, and still omit warn diagnostics from `openclaw plugins doctor`; docs and type discoverability are improved, but the operator-warning path is not fully resolved.

## What This Changes

The issue requests more visible diagnostics when non-bundled plugins register conversation hooks without `plugins.entries.<id>.hooks.allowConversationAccess=true`.

## Best Possible Solution

Keep the secure default-deny gate, promote blocked conversation-hook diagnostics to warning-level startup output, and add a maintainer-approved `plugins doctor` path for hook-policy warning diagnostics.

## Reproduction Assessment

Yes, source-level: a non-bundled plugin registering conversation hooks without `allowConversationAccess=true` still gets warn diagnostics and no typed-hook registrations, while `plugins doctor` filters those warnings out. I did not run a live gateway because this is a read-only review, but current source and tests cover the path directly.

## Solution Assessment

No final fix is present on current main. The best fix is narrower than changing hook policy: make the existing denial visible through warning-level startup output and a targeted doctor warning path without weakening the default-deny boundary.
