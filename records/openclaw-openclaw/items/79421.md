## Summary

Keep open: current main still blocks non-bundled conversation-hook registration without `allowConversationAccess=true`, still routes warn diagnostics through the gateway info path, and `openclaw plugins doctor` still filters out warn diagnostics. Docs and type discoverability have improved, but the operator-visible warning/doctor behavior remains unresolved.

## What This Changes

The issue asks OpenClaw to make default-denied non-bundled conversation-hook registrations visible outside the buried plugin diagnostic sink, with better operator and author discoverability.

## Best Possible Solution

Keep the secure default-deny gate, promote blocked conversation-hook diagnostics to warning-level startup output, and add a maintainer-approved doctor path for hook-policy warnings.

## Reproduction Assessment

Yes, source-level: current loader code and tests show a non-bundled plugin registering conversation hooks without `allowConversationAccess=true` gets warn diagnostics and no typed-hook registrations. I did not run a live gateway because this review is read-only.

## Solution Assessment

No final fix is present on current main. The best path is narrower than changing hook policy: make the existing denial visible through warning-level startup output and a targeted doctor warning path without weakening the default-deny boundary.
