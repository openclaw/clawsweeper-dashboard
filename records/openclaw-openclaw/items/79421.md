## Summary

Keep open: current main still blocks non-bundled conversation hooks without a warning-level operational signal, and `plugins doctor` still filters out the warn diagnostic that explains the block.

## What This Changes

The issue requests that blocked conversation-hook registrations for non-bundled plugins surface as startup warnings, and optionally through `openclaw plugins doctor`, instead of only runtime inspect diagnostics.

## Best Possible Solution

Keep the default-deny hook policy, but promote denied conversation-hook registrations to warning-level startup logging and surface warn-level plugin diagnostics in doctor while preserving runtime inspect diagnostics.

## Reproduction Assessment

Yes. Source inspection and the existing loader test give a high-confidence current-main path: non-bundled conversation hooks without `allowConversationAccess` produce zero typed hooks and only warn diagnostics.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing warn diagnostic as an operational warning in the gateway/plugin load path and include warn diagnostics in `plugins doctor`, without changing the access policy.
