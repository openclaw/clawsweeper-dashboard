## Review Findings

Overall confidence: 0.94

Full review comments:

- **[P1] Remove the duplicate history result declaration:** `src/agents/tools/sessions-history-tool.ts:132-133`
  - body: This declares `const result` twice in the same scope, so `sessions-history-tool.ts` cannot parse and any path that imports the tool will fail before runtime.
  - confidence: 0.99
- **[P1] Preserve forced tool-payload redaction:** `src/agents/tools/sessions-history-sanitize.ts:17`
  - body: The new sanitizer calls `redactSensitiveText`, which becomes a no-op when log redaction is configured off. Current `sessions_history` deliberately forces tool-payload redaction so recalled secrets stay masked on this tool surface.
  - confidence: 0.94
- **[P2] Keep reasoning replay out of the public schema:** `src/agents/tools/sessions-history-tool.ts:23`
  - body: Adding `preserveLatestAssistantReasoning` as a normal `sessions_history` argument lets a model-callable tool return raw `thinking` or `redacted_thinking` blocks through the JSON tool result, reopening the boundary this patch is trying to close.
  - confidence: 0.91
- **[P2] Fetch enough history before filtering previews:** `src/agents/tools/sessions-list-tool.ts:271-275`
  - body: `sessions_list` still requests only `messageLimit` rows from `chat.history` and filters invisible sanitized turns afterward, so a newest reasoning-only turn can leave the preview empty instead of backfilling to the previous visible message.
  - confidence: 0.86
- **[P2] Treat tool-call-only previews as invisible:** `src/agents/tools/sessions-history-sanitize.ts:158-164`
  - body: `hasVisibleHistoryPreviewContent` treats any non-text block as visible, so after reasoning is stripped an assistant turn containing only an internal tool-call payload can consume the preview slot instead of backfilling to readable text.
  - confidence: 0.79

## Security Review

Status: needs_attention

Summary: The diff introduces concrete security-sensitive regressions by weakening forced session-history redaction and adding a model-callable raw-reasoning bypass.

Concerns:

- **[high] Forced redaction regression:** `src/agents/tools/sessions-history-sanitize.ts:17`
  - body: Switching from forced tool-payload redaction to `redactSensitiveText` can expose credentials in `sessions_history` when `logging.redactSensitive` is off, contrary to current main’s safety-boundary behavior.
  - confidence: 0.94
- **[medium] Model-callable reasoning bypass:** `src/agents/tools/sessions-history-tool.ts:23`
  - body: The public `preserveLatestAssistantReasoning` parameter allows normal tool callers to serialize provider-owned reasoning blocks back into the tool result payload.
  - confidence: 0.91
