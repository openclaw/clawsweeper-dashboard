## Summary

Keep open. Current main still does not implement the PR’s central behavior for `sessions_history`, but the PR branch is not mergeable as-is and would also regress tool-output redaction and expose a model-callable reasoning-preservation bypass.

## What This Changes

The PR extracts session-history sanitization, strips `thinking` and `redacted_thinking` blocks from `sessions_history` and `sessions_list` outputs, adds a latest-assistant reasoning preservation option, and adds focused tests.

## Best Possible Solution

Implement the sanitizer on current main so public session recall strips `thinking` and `redacted_thinking` blocks, preserves non-empty assistant turn shape only where needed, keeps replay-only reasoning preservation internal to provider replay paths, retains forced tool-payload redaction, and covers direct `sessions_history` output plus preview backfill with regression tests.

## Reproduction Assessment

Yes. A source-level reproduction is clear: current main’s `sessions_history` sanitizer returns `type: "thinking"` content blocks, and the current test suite still asserts that a returned thinking block exists after signature removal.

## Solution Assessment

No. The requested direction is correct, but this PR is not the best mergeable fix until the syntax error, redaction regression, public reasoning-preservation bypass, and preview backfill behavior are addressed.
