## Summary

Keep open. The reported `sessions_history` reasoning-block bug is still present on current `main`, but this PR head is dirty against `main` and is not mergeable as-is because it has a syntax error plus concrete redaction and reasoning-boundary regressions.

## What This Changes

The PR extracts session-history sanitization, strips `thinking` and `redacted_thinking` blocks from `sessions_history` and `sessions_list` outputs, adds a public `preserveLatestAssistantReasoning` option, and adds focused regression tests.

## Best Possible Solution

Land a current-main implementation that strips `thinking` and `redacted_thinking` from public `sessions_history` and `sessions_list` tool payloads, keeps any provider replay preservation internal to replay-only code paths, preserves forced tool-payload redaction, fetches enough preview history before filtering invisible turns, and adds focused regression tests plus the required changelog entry.

## Reproduction Assessment

Yes. A source-level reproduction is clear on current `main`: `sessions_history` still returns `type: "thinking"` content blocks after only truncating their text and deleting `thinkingSignature`, and the current tests still assert that such a block remains.

## Solution Assessment

No. The requested direction is right, but the current PR head is not the best merge path because it cannot parse and it mixes the sanitizer with a public replay-preservation bypass and a forced-redaction regression; a narrow rebased fix or replacement branch should keep replay preservation internal.
