## Summary

Keep open. The underlying `sessions_history` reasoning-block bug is still source-reproducible on current main, but this PR head is not merge-ready because it has a syntax error plus security-sensitive redaction and reasoning-boundary regressions.

## What This Changes

The PR extracts session-history sanitization, strips `thinking` and `redacted_thinking` blocks from `sessions_history` and `sessions_list` outputs, adds a public `preserveLatestAssistantReasoning` option, and adds focused tests.

## Best Possible Solution

Land a narrow current-main fix that strips `thinking` and `redacted_thinking` from public `sessions_history` and `sessions_list` payloads, preserves forced tool-payload redaction, keeps provider replay preservation internal, backfills visible previews correctly, and adds regression coverage plus the required changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main shows `sessions_history` still returns `type: "thinking"` content blocks after removing only the signature, and the existing test suite still asserts that block remains.

## Solution Assessment

No. The requested direction is right, but the current PR head is not the best merge path because it cannot parse and it mixes public tool-output sanitization with a model-callable reasoning bypass and a forced-redaction regression.
