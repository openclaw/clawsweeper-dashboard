## Summary

Keep this PR open: the current head still has blocking security-policy issues in Gateway loopback proxy routing, and the PR is protected by `maintainer` plus `clawsweeper:human-review`.

## What This Changes

The PR adds `proxy.loopbackMode`, replaces Gateway loopback proxy bypass with global-agent `NO_PROXY` registration, and updates proxy docs, schema, changelog, and tests.

## Best Possible Solution

Keep this in maintainer review until proxy-mode child Gateway clients use managed proxy routing, default bypass authorization is checked against a separate configured Gateway target, the unrelated changelog bullet is removed, and exact-head checks pass.

## Reproduction Assessment

Yes. Source inspection of PR head shows the `proxy` mode path only skips `NO_PROXY` registration while bypassed child CLI commands do not start proxy routing, so their plain `ws` Gateway connection can still go direct.

## Solution Assessment

No. `proxy.loopbackMode` is a plausible direction, but this patch is not yet the narrow maintainable fix because child proxy bootstrap and configured-target enforcement remain incomplete.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Bootstrap proxy routing for inherited proxy mode:** `src/gateway/client.ts:290`
  - body: When a bypassed child CLI inherits `OPENCLAW_PROXY_LOOPBACK_MODE=proxy`, this branch only skips `NO_PROXY` registration and still constructs a plain `ws` client without starting global-agent or passing a proxy agent. Those Gateway control-plane connections can therefore go direct instead of through the managed proxy.
  - confidence: 0.91
- **[P1] Authorize bypass against the configured Gateway target:** `src/gateway/client.ts:290`
  - body: `GatewayClientOptions` still has only the actual `url`, and `registerManagedProxyGatewayLoopbackNoProxy(url)` authorizes that value directly. A loopback override can therefore authorize its own direct `NO_PROXY` entry instead of being checked against the configured Gateway URL promised by the docs.
  - confidence: 0.9
- **[P3] Remove the unrelated changelog entry:** `CHANGELOG.md:89`
  - body: This PR adds a Sandbox/Windows Docker bind-source changelog bullet, but the diff does not change that surface. Landing it would publish an unrelated release note from this PR.
  - confidence: 0.82
