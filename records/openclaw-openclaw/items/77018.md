## Summary

Keep open: the current PR head still has security-relevant proxy bypass gaps in the Gateway loopback path, and the protected maintainer/human-review labels require human handling before merge.

## What This Changes

The PR adds `proxy.loopbackMode`, updates managed-proxy state/env handling, changes GatewayClient loopback bypass behavior, and adds docs, changelog, and proxy/Gateway tests.

## Best Possible Solution

Keep the configured-Gateway URL guard, then make inherited `proxy` mode either bootstrap/use a concrete managed proxy path from inherited env or fail closed, with regression coverage for forked control-plane CLIs and custom configured ports.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: run a proxy-started parent that forks a `networkProxy: "bypass"` local control-plane CLI with `OPENCLAW_PROXY_ACTIVE=1` and `OPENCLAW_PROXY_LOOPBACK_MODE=proxy`; the child GatewayClient creates a WebSocket without a direct or proxy agent.

## Solution Assessment

No. `proxy.loopbackMode` is a reasonable policy knob, but the current implementation does not actually enforce the promised proxy route in inherited children and no longer restricts the default bypass to the configured Gateway URL.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Route inherited proxy-mode loopback through the proxy:** `src/gateway/client.ts:308`
  - body: When a bypassed child CLI inherits `OPENCLAW_PROXY_LOOPBACK_MODE=proxy`, `startProxy()` has not bootstrapped `global-agent` in that process. This line drops the direct agent and then opens `new WebSocket` with no proxy agent, so `ws` falls back to Node `http.request`, which does not honor proxy env by itself.
  - confidence: 0.86
- **[P1] Require the configured Gateway URL before bypassing:** `src/gateway/client.ts:308`
  - body: In `gateway-only` mode, any loopback `GatewayClient` URL still gets a direct agent and the bypass helper clears proxy env. That contradicts the new docs/changelog promise that only the configured Gateway URL bypasses the managed proxy, so caller-provided loopback URLs can still skip filtering.
  - confidence: 0.82
