## Summary

Keep open: current main and the latest release still lack `<session-recap>` stripping in the shared inbound metadata path, and an open linked fork PR already owns the proposed fix but is conflicting and not maintainer-modifiable.

## What This Changes

The issue reports that leading `<session-recap>` internal context blocks render visibly in Control UI and TUI user messages instead of being stripped from user-visible text.

## Best Possible Solution

Review, rebase, or replace https://github.com/BingqingLyu/openclaw/pull/252 with a narrow shared-stripper fix that removes only leading standalone `<session-recap>` and `<session_recap>` blocks while preserving mid-message literal text.

## Reproduction Assessment

Yes from source inspection: a leading `<session-recap>` block does not match `SENTINEL_FAST_RE`, so both shared UI and TUI stripping paths return it unchanged. I did not run a live UI/TUI reproduction in this read-only review.

## Solution Assessment

Yes, the best fix belongs in the shared inbound metadata stripping helper because both Control UI and TUI already depend on it for user-visible text. The fix should be leading-only and preserve mid-message user-authored recap text.
