## What This Changes

Request to stop internal `<session-recap>...</session-recap>` context blocks from rendering as visible user text in Control UI and TUI.

## Best Possible Solution

Land a narrow shared-stripper fix that removes only leading standalone `<session-recap>` or `<session_recap>` blocks while preserving mid-message and malformed user-authored text.

## Solution Assessment

Yes, the right fix boundary is the shared inbound metadata stripper because both Control UI and TUI already route user-visible text through it. The current open fork PR follows the right direction but needs maintainer adoption, rebase, or replacement before merge.
