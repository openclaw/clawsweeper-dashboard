## Label Justifications

- `P2`: The bug exposes internal context in two user-visible interfaces but is localized to shared message normalization.
- `impact:session-state`: Internal session recap state is rendered as visible transcript content.

## Summary

Current main and the latest release retain the reported source path: both affected interfaces use a shared inbound metadata stripper whose fast path recognizes existing sentinels but not session-recap blocks. The issue remains necessary; the only linked open implementation is a conflicting fork PR, so maintainers should resolve that branch before commissioning a replacement.

## What This Changes

Prevent leading internal session-recap scaffolding from appearing in user-visible Control UI and TUI messages.

## Best Possible Solution

Implement one narrow leading session-recap removal in the shared inbound metadata parser, preserve literal mid-message and incomplete blocks, and add parser plus Control UI and TUI regression coverage.

## Reproduction Assessment

Yes at high confidence from source inspection. A message beginning only with the reported recap block misses the sentinel fast path and is returned unchanged by the exact helpers used by Control UI and TUI.

## Solution Assessment

Yes, with a narrow boundary. Updating the shared inbound metadata stripper is preferable to separate UI/TUI filters because both affected consumers already rely on that canonical helper.
