The issue reports that leading `<session-recap>` context scaffolding is rendered as visible user text in the Control UI and TUI instead of being stripped by the shared inbound metadata sanitizer.

Land or import a narrow shared sanitizer fix that strips complete leading recap blocks while preserving timestamp stripping, active-memory stripping, existing metadata stripping, malformed-block preservation, and literal mid-message recap text.

## Solution Assessment

Yes. The shared inbound metadata sanitizer is the right boundary; the fix should add leading recap-block stripping and regression coverage without stripping mid-message user-authored recap text.
