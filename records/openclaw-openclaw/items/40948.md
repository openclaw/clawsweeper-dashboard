## Summary

Keep this issue open: current main still has no `session-recap` handling in the shared inbound metadata sanitizer, and open PR #41013 already targets this exact issue with closing syntax.

## What This Changes

The issue requests stripping internal leading `<session-recap>` / `<session_recap>` blocks from user-visible Control UI and TUI message text.

## Best Possible Solution

Land PR #41013 or an equivalent narrow shared-sanitizer fix that strips only complete leading session-recap blocks while preserving mid-message and malformed literals.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: leading `<session-recap>...</session-recap>` text misses `SENTINEL_FAST_RE`, so both exported strip functions return it unchanged before Control UI/TUI rendering.

## Solution Assessment

Yes. The best fix is in `src/auto-reply/reply/strip-inbound-meta.ts`, keeping both UI surfaces on the shared sanitizer instead of adding separate display-layer stripping.
