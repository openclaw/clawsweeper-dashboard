## Summary

Keep open: current main still lacks `session-recap` handling in the shared inbound metadata stripper, and open PR #41013 already targets this issue with closing syntax.

## What This Changes

The issue asks OpenClaw to strip leading `<session-recap>...</session-recap>` scaffolding from user-visible Control UI and TUI message rendering.

## Best Possible Solution

Land PR #41013 or an equivalent narrow shared-sanitizer fix that strips only complete leading session-recap blocks while preserving mid-message and malformed literals.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a leading `<session-recap>...</session-recap>` block misses `SENTINEL_FAST_RE`, so both exported strip functions return it unchanged before Control UI and TUI rendering.

## Solution Assessment

Yes. The narrowest maintainable fix belongs in `src/auto-reply/reply/strip-inbound-meta.ts`, keeping Control UI and TUI on the shared sanitizer instead of adding separate display-layer stripping.
