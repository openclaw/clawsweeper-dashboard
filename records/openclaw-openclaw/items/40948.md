## Summary

Keep open: current main and v2026.6.6 still do not strip leading `<session-recap>` blocks, and the linked fixing branch is open but conflicting rather than merged.

## What This Changes

The issue reports that internal `<session-recap>` context scaffolding is rendered as visible user text in the Control UI and TUI.

## Best Possible Solution

Land a narrow shared-stripper fix that removes only leading standalone `<session-recap>` or `<session_recap>` blocks while preserving mid-message user-authored text, then close the issue after merge.

## Reproduction Assessment

Yes from source inspection: a leading `<session-recap>` block without any current sentinel text does not match `SENTINEL_FAST_RE`, so the shared stripper returns it unchanged. I did not run a live UI/TUI repro in this read-only review.

## Solution Assessment

Yes: the best fix belongs in `src/auto-reply/reply/strip-inbound-meta.ts`, because both TUI and Control UI already route visible user text through the shared stripper. The linked branch follows the right shape but must be conflict-resolved, adopted, or replaced before merge.
