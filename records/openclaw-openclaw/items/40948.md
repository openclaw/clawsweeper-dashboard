## Best Possible Solution

Land a narrow shared-stripper fix that removes only leading standalone `<session-recap>` or `<session_recap>` blocks while preserving mid-message user-authored text, then close this issue after the fix merges.

## Reproduction Assessment

## Solution Assessment

Yes, the best fix belongs in the shared inbound metadata stripping path because both the TUI and Control UI already call that helper for user-visible text. The fix should stay leading-only and preserve mid-message literal recap text.
