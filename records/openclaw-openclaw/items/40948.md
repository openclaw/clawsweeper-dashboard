## Summary

Keep open: current `main` and latest release still have the sentinel-only fast path that lets a leading `<session-recap>` block pass through unchanged, and the linked fix candidate remains open but conflicting.

## What This Changes

The issue asks OpenClaw to strip leading `<session-recap>` scaffolding from user-visible Control UI and TUI message rendering.

## Best Possible Solution

Review and land the linked fix PR after rebasing, or replace it with the same narrow shared-stripper fix plus regression tests for Control UI/TUI paths.

## Reproduction Assessment

Yes, source-reproducible: pass a string beginning with `<session-recap>...</session-recap>` through `stripInboundMetadata()` or `stripLeadingInboundMetadata()` and current main returns it unchanged because no fast-path sentinel matches.

## Solution Assessment

Yes, the best fix is in the shared inbound metadata stripper because both Control UI and TUI already route user-visible cleanup through that module; add focused tests that strip only leading recap blocks and preserve mid-message literals.
