## What This Changes

Request to strip leading `<session-recap>` agent-context blocks from Control UI/TUI user-visible rendering while preserving mid-message literal text.

## Best Possible Solution

Land or replace the linked fork PR with a narrow shared-sanitizer patch and regression tests that strip complete leading recap blocks while preserving malformed and mid-message literal text.

## Reproduction Assessment

Yes, at source level: a leading `<session-recap>...</session-recap>` block does not match `SENTINEL_FAST_RE`, so both stripper entry points can return it unchanged before UI/TUI rendering.

## Solution Assessment

Yes, the shared inbound metadata stripper is the right boundary; the fix should add narrow leading recap-block stripping plus regression coverage for leading, empty, malformed, chained, and mid-message cases.
