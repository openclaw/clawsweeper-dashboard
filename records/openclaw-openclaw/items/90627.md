## Best Possible Solution

Expose the expanded `wiki_apply.op` set as a flat string enum, keep the alias normalization in the memory-wiki plugin, and add a small redacted real-runtime proof before merge.

## Reproduction Assessment

Yes from source inspection: current `origin/main` documents `openclaw wiki apply synthesis` and `metadata`, while `WikiApplySchema` only accepts `create_synthesis` and `update_metadata`. I did not run a live Gateway reproduction because this review is read-only.

## Solution Assessment

No, not quite yet. Accepting aliases at the memory-wiki schema and normalizer is the right boundary, but the provider-facing op schema should use a flat enum rather than preserving an `anyOf` literal union.
