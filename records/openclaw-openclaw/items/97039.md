## What This Changes

This PR adds an exact-type-first pass to llm-core JSON-schema union coercion and tests nullable `anyOf` MCP-style arguments, plus bundle-MCP materialization test assertions around null handling.

## Best Possible Solution

Keep the nullable-union preservation, narrow the exact-match behavior so unrelated primitive unions keep existing coercion semantics, repair or remove the direct materialization stripping assertions, and add redacted configured MCP proof before choosing this as the canonical fix.

## Reproduction Assessment

Yes, at source level: current main can coerce `null` to `""` when an `anyOf` tries a string branch before a null branch. I did not run the reported awslabs EKS MCP server or capture live wire payloads.

## Solution Assessment

No, not as submitted. The validation layer is the right place for nullable-union preservation, but the patch should avoid changing unrelated primitive union precedence and should not assert direct MCP execute stripping that is not implemented.
