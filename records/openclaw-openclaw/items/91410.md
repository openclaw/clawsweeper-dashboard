## Summary

Keep open; this PR targets a real message-delivery leak, but the proposed guard is too narrow because the main agent delivery plan can still carry the original reserved string forward, and the PR does not include real behavior proof.

## What This Changes

This PR adds a reserved meta-string guard for `current`, `self`, `this`, and `me` inside the session delivery target resolver.

## Best Possible Solution

Reject or normalize reserved literal targets at the delivery-plan boundary so every outbound path consumes the same decision, then add regression coverage for `resolveAgentDeliveryPlan` and redacted real behavior proof for the reported Telegram-style misdelivery.

## Reproduction Assessment

Yes for the source-level path: current `main` accepts `explicitTo` as a raw string, and `resolveAgentDeliveryPlan` can continue carrying the original value after resolver processing. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

No. A resolver-local guard is only a plausible partial fix; the better fix must make the delivery plan and downstream target resolution consume the rejected or sanitized target consistently.
