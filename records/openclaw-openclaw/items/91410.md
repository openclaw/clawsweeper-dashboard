## Summary

Keep this PR open, but the current patch should not merge: it fixes the raw literal-target leak for the tested session fallback path while still turning reserved explicit targets into ordinary implicit routing, which can fall through to session history or configured channel defaults.

## Best Possible Solution

Carry a fail-closed reserved-target outcome through the resolver and delivery plan so it cannot inherit session history or `defaultTo` unless a typed current-session binding exists, then add explicit-channel/default-fallback regression tests and fresh Telegram proof.
