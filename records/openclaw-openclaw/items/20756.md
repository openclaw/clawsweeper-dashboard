## Summary

Keep open. Current main still leaves omitted message-tool `accountId` unresolved unless it comes from an explicit parameter, runner default, or agent binding; WhatsApp then falls back to the default-account helper, and the direct implementation PR #20801 is closed unmerged.

## What This Changes

Auto-select the sole enabled account for a selected message-tool channel when no `accountId` is supplied, while preserving explicit/default account behavior and clear ambiguity errors.

## Best Possible Solution

Add a shared outbound account resolver after channel selection that chooses an enabled configured default or exactly one enabled configured account, with clear zero-or-multiple-account errors.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: omitted `accountId` stays unset in the runner and WhatsApp resolves it through the default helper, so a disabled `default` plus one enabled named account still targets `default`.

## Solution Assessment

Yes. A generic resolver through `ChannelConfigAdapter` after channel selection is the narrowest maintainable solution; a WhatsApp-only fallback would duplicate shared routing policy and miss other plugin channels.
