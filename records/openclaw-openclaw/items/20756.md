## Summary

Keep open. Current main and the latest release still do not infer an omitted message-tool account from the only enabled account, and the open linked implementation at https://github.com/openclaw/openclaw/pull/85671 is not merged yet.

## What This Changes

Request auto-selecting the only enabled/configured account when the message tool omits `accountId`, instead of falling back to a disabled `default` account.

## Best Possible Solution

Review, refresh, and land the linked shared resolver PR or a narrow replacement that preserves explicit/default/bound-account precedence and documents the accepted omitted-account policy.

## Reproduction Assessment

Yes, at source level. Current main leaves omitted `accountId` unresolved after explicit/default/bound checks, and WhatsApp then resolves the omitted account to `default`, matching the disabled-default failure path.

## Solution Assessment

Yes for the owner boundary: a shared resolver after channel selection is better than a WhatsApp-only fallback. The current open PR is the right candidate shape, but it needs merge/proof/product-policy resolution before this issue closes.
