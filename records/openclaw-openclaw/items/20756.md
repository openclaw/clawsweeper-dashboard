## Summary

Keep open. Current main and the latest release still leave omitted message-tool `accountId` unresolved before WhatsApp falls back to the default account, and the linked fixing PR is open but conflicting and not merged.

## What This Changes

The issue requests auto-selecting the only enabled configured account for the selected message channel when `accountId` is omitted.

## Best Possible Solution

Land a shared selected-channel account resolver that preserves explicit/default/bound account precedence, uses plugin adapter enabled/configured hooks, auto-selects exactly one enabled configured account, and documents/tests clear zero-enabled and multiple-enabled errors.

## Reproduction Assessment

Yes, at source level. Current main leaves omitted `accountId` unset unless explicit/default/binding sources provide one, and WhatsApp then resolves through the default-account helper.

## Solution Assessment

Yes, the best fix shape is a generic selected-channel account resolver using plugin adapter hooks. A WhatsApp-only fallback would duplicate outbound routing policy and miss the same class in other multi-account channels.
