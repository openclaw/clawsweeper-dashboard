## Summary

Keep open. Current main still leaves omitted message-tool account selection unresolved: the shared runner only preserves explicit/default/bound account IDs, WhatsApp still falls back to the default-account helper, and the linked implementation pull request is closed unmerged.

## What This Changes

The issue requests omitted message-tool sends to choose the only enabled account for the selected channel instead of falling back to a disabled default account.

## Best Possible Solution

Add a shared outbound account resolver after channel selection that uses the selected channel plugin adapter to prefer an enabled configured default, select exactly one enabled/configured account, and error clearly otherwise.

## Reproduction Assessment

Yes, at source level. With accountId omitted, current main leaves the shared runner without an account and WhatsApp resolves through the default-account helper, so a disabled default plus one enabled named account still targets default.

## Solution Assessment

Yes. A generic resolver through the selected channel plugin config adapter is the narrow maintainable path; a WhatsApp-only fallback would duplicate shared routing policy and miss other plugin channels.
