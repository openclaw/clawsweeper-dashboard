## Summary

Keep open. Current main and the latest release still leave omitted message-tool `accountId` unresolved in the shared runner, then WhatsApp falls back to the default-account helper, so a disabled `default` account plus one enabled named account can still route to `default`; the related implementation PR was closed unmerged.

## What This Changes

The issue requests that the message tool auto-select the only enabled account for a selected channel when `accountId` is omitted instead of falling back to a disabled `default` account.

## Best Possible Solution

Centralize omitted-account resolution after channel selection using the selected channel plugin adapter, then document and test enabled default, single enabled, zero enabled, and multiple enabled account cases.

## Reproduction Assessment

Yes, at source level. With `accountId` omitted, current main leaves the runner account unset and WhatsApp resolves through the default-account helper, so a disabled `default` plus one enabled named account still targets `default`.

## Solution Assessment

Yes. A generic selected-channel account resolver is the narrow maintainable fix; a WhatsApp-only fallback would duplicate account-routing policy and miss the same class in other plugin channels.
