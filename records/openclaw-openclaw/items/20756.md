## Summary

Keep open. Current main still preserves only explicit, runner-default, or bound account IDs; omitted WhatsApp sends fall through to the default-account helper, and the direct implementation PR is closed unmerged.

## What This Changes

Request auto-selection of the sole enabled channel account when the message tool omits `accountId`.

## Best Possible Solution

Add a shared outbound account resolver after channel selection that chooses an enabled configured default account or exactly one enabled configured account, with clear zero-account and multiple-account errors.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: omitted `accountId` stays unset in the shared runner and WhatsApp resolves it through the default helper, so a disabled `default` plus one enabled named account still targets `default`.

## Solution Assessment

Yes. A generic resolver through the selected channel plugin config adapter is the narrowest maintainable direction; a WhatsApp-only fallback would duplicate shared routing policy and miss other plugin channels.
