## What This Changes

The issue requests a Feishu monitor cleanup fix so HTTP servers are fully closed before `httpServers` entries and related state are cleared.

## Best Possible Solution

Review, refresh, and land or replace https://github.com/openclaw/openclaw/pull/48588 with a narrow Feishu-owned awaitable HTTP shutdown helper and regression coverage, while keeping ResourceManager architecture work in the separate follow-up issues.

## Reproduction Assessment

Yes, at source level: current main still calls `server.close()` and clears Feishu HTTP server state before the close callback in both stop cleanup and webhook abort cleanup. A live restart loop would only be needed to measure the external `EADDRINUSE` or memory-growth symptom.

## Solution Assessment

Yes, the narrow immediate solution is to make Feishu HTTP shutdown awaitable and bounded before clearing state. The broader ResourceManager proposal is useful follow-up work, not a prerequisite for fixing this user-facing bug.
