## Summary

Keep open because current main and the latest release still have the reported Feishu HTTP server shutdown ordering, and open PR #48588 is already the paired implementation candidate with closing syntax.

## What This Changes

The issue asks Feishu monitor shutdown to await HTTP server close before clearing `httpServers` state so restarts do not leak resources or hit port reuse failures.

## Best Possible Solution

Land or replace #48588 with a narrow Feishu plugin fix that makes HTTP server teardown awaitable, bounds stuck closes, preserves existing WebSocket behavior, and adds focused regression coverage; keep broader ResourceManager work in #48793/#49804 separate.

## Reproduction Assessment

Yes, at source level: current main still calls `server.close()` before immediately deleting or clearing the Feishu HTTP server state. A live Feishu restart loop would only be needed to measure the external memory or port symptom.

## Solution Assessment

Yes, the best immediate fix is a narrow Feishu HTTP cleanup change rather than waiting for the broader cross-plugin ResourceManager proposal. The existing open PR should be reviewed, refreshed, landed, or replaced rather than spawning a competing repair job.
