Full review comments:

- **[P2] Preserve Teams caller identity in session keys:** `extensions/voice-call/src/providers/msteams.ts:226`
  - body: `from` is the raw Teams AAD id here, but the streaming path then lets `CallManager` derive `sessionKey` with `resolveVoiceCallSessionKey`, which strips every non-digit because it was designed for phone numbers. That means two different GUIDs can collapse to the same `voice:<digits>` key, or collide with a phone-derived key, so caller memory/transcript context can be mis-associated. Please key Teams calls from the exact AAD id or force an explicit per-call/provider-scoped session key, and add coverage for distinct AAD callers.
