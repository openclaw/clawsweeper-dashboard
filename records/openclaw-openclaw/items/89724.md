## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Gate Teams transcripts on recording status:** `extensions/voice-call/src/providers/msteams.ts:279-287`
  - body: `call.speech` is persisted and used to generate an agent response with no OpenClaw-side state proving the external worker first called `updateRecordingStatus` successfully. Microsoft documents that Media Access API media content, or derived data, must not be recorded or persisted before that success response, so the bridge protocol needs a recording-status signal and this path should hold or drop transcripts until it is active. ([learn.microsoft.com](https://learn.microsoft.com/en-us/graph/api/call-updaterecordingstatus?view=graph-rest-1.0&utm_source=openai))
  - confidence: 0.91
- **[P2] Fail fast before binding Teams without STT/TTS:** `extensions/voice-call/src/runtime.ts:564-565`
  - body: When `streaming.enabled` is false this branch only logs a warning, and `wireMsteamsRuntime` also only warns when transcription or TTS is unavailable, but the code still binds the Teams WebSocket listener. That lets operators receive authenticated calls that close as `not-ready` or fail during playback instead of getting a config validation error before exposure.
  - confidence: 0.88
- **[P2] Buffer early audio until STT is ready:** `extensions/voice-call/src/providers/msteams.ts:263-266`
  - body: `session.start` kicks off `sttSession.connect()` asynchronously, and this guard silently drops every audio frame until `isConnected()` flips true. A real Teams worker can send caller speech immediately after the start frame, so the first utterance can be lost; use a bounded pre-ready buffer or close on connect failure instead of dropping media.
  - confidence: 0.86
- **[P2] Document Teams setup before exposing the provider:** `extensions/voice-call/openclaw.plugin.json:262`
  - body: The manifest now exposes `provider: "msteams"` and its config, but the voice-call docs still only describe Twilio, Telnyx, Plivo, and Mock. Before shipping this config surface, add public setup/security docs covering the external worker, HMAC headers, bindAddress posture, required STT/TTS streaming, recording-status obligation, and inbound-only behavior.
  - confidence: 0.84
