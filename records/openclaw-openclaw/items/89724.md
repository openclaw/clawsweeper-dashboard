## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] End sessions when the Teams socket closes:** `extensions/voice-call/src/msteams-media-stream.ts:358-360`
  - body: After `session.start`, a plain WebSocket close only calls `cleanupConnection` and logs the close. If the worker drops the socket without first sending `session.end`, the provider never receives `onSessionEnd`, so streaming calls can stay active until timeout and realtime calls can remain in `realtimeCalls`. Track started sessions and emit a terminal `onSessionEnd` from the close handler.
  - confidence: 0.88
- **[P2] Preserve Teams caller identity in session keys:** `extensions/voice-call/src/providers/msteams.ts:262-263`
  - body: The streaming path registers Teams calls with `from = session.caller.aadId`, but CallManager builds the default voice session key by phone-normalizing `from`. AAD GUIDs are not phone numbers, so this can collapse different users to the same digit-only key or fall back to per-call state for all-letter GUIDs. Use an explicit Teams caller session key or extend the session-key helper for non-phone caller IDs.
  - confidence: 0.84
- **[P3] Add setup metadata for the msteams config:** `extensions/voice-call/openclaw.plugin.json:303-324`
  - body: The manifest schema adds required `msteams.port` and `msteams.sharedSecret`, but the setup/UI hint surfaces still do not label the new `msteams.*` fields or mark the shared secret as sensitive, and the provider help text still omits `msteams`. Add matching metadata in the manifest and runtime config hints so operators can discover and configure the required bridge settings.
  - confidence: 0.72
