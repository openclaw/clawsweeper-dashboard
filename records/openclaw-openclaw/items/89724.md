## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve Teams caller identity in streaming session keys:** `extensions/voice-call/src/providers/msteams.ts:262-263`
  - body: The streaming path passes `session.caller.aadId` as `from`, but `CallManager` builds the default `per-phone` session key by stripping non-digits from that value. Teams AAD IDs are not phone numbers, so different callers can collapse into the same `voice:*` key or lose stable caller memory. Pass an explicit Teams-scoped session key or add a provider-aware caller identity path before using the default session scope.
  - confidence: 0.91
- **[P2] Remove or wire the documented realtime echo option:** `extensions/voice-call/README.md:167`
  - body: The README advertises `realtime.suppressInputDuringPlayback`, but the strict realtime config schema and plugin manifest do not expose that key, and `runtime.ts` never passes it into `setRealtimeRuntime`. A user following the docs would get an invalid or no-op setting; either delete the README mention or add the schema, manifest, runtime wiring, and tests.
  - confidence: 0.88
