- **[P1] Skip unresolved OAuth sidecar profiles:** `src/agents/auth-profiles/persisted.ts:822`
  - body: This copies every profile from legacy `auth-profiles.json` into SQLite when no store row exists. Legacy OAuth sidecar entries can have only `oauthRef` and no inline access/refresh; current auth selection treats those as `unresolved_ref`, and doctor intentionally keeps them in JSON until sidecar repair materializes the secret. If runtime loads once before doctor, SQLite owns the bad profile id, later doctor import preserves that existing row, and the repaired JSON can be removed while OAuth stays unusable. Filter these profiles out or run the sidecar materialization path first, with a regression test.
