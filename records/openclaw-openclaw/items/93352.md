## Summary

## What This Changes

The branch adds a runtime lazy import that reads legacy per-agent `auth-profiles.json` into the SQLite auth profile store and adds an API-key regression test.

## Best Possible Solution

Limit the runtime safety net to safe static credentials or fully materialized OAuth profiles, keep sidecar and comprehensive cleanup in doctor, and require redacted upgraded-state proof before merge.

## Reproduction Assessment

Yes by source inspection: current main skips per-agent legacy `auth-profiles.json` when no SQLite store exists, and auth selection can then return no usable profile order. I did not run a release-to-release upgrade in this read-only review.

## Solution Assessment

No. The static-profile lazy import is plausible, but importing every coerced legacy profile crosses doctor-owned OAuth sidecar repair; the safer fix is to filter unresolved sidecar profiles or materialize them before SQLite import.
