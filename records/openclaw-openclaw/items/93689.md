## Summary

Keep open for maintainer review: the branch addresses the linked Zalo target-resolution bug and the prior maintainer feedback with no blocking code findings, but it intentionally broadens Zalo direct-target classification and still needs human acceptance before merge.

## What This Changes

This PR changes the bundled Zalo plugin so nonblank normalized Zalo targets are treated as opaque Bot API chat_id values and adds resolver coverage for numeric, hex-like, dotted, prefixed, and blank cases.

## Best Possible Solution

Land the plugin-local opaque chat_id predicate after maintainer re-review confirms the widened Zalo direct-target behavior, keeping shared outbound heuristics unchanged.

## Reproduction Assessment

Yes by source inspection. Current main still uses the numeric-only Zalo target predicate, and the shared resolver only returns a normalized direct target when looksLikeTargetId succeeds, so dotted or hex-like chat_id values miss the direct path.

## Solution Assessment

Yes. Given Zalo documents chat_id as a String with an abc.xyz example, a Zalo-local opaque nonblank predicate is cleaner than changing shared heuristics or encoding an undocumented regex grammar.
