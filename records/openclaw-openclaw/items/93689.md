## Summary

Keep open: this is a focused, viable bug-fix PR with no blocking patch defect found, but it still needs maintainer re-review because the fix intentionally broadens Zalo direct-target delivery behavior.

## What This Changes

The branch changes the Zalo plugin target resolver from the shared numeric-only predicate to a Zalo-local nonblank normalized chat_id predicate and adds regression coverage for numeric, hex-like, dotted, prefixed, and blank targets.

## Best Possible Solution

Land the plugin-local opaque chat_id predicate after maintainer re-review accepts the widened direct-target behavior, while keeping shared outbound heuristics unchanged.

## Reproduction Assessment

Yes. Current main wires Zalo targetResolver.looksLikeId to a numeric-only helper, and the shared outbound resolver only accepts direct targets when that predicate returns true; I did not run a live Zalo credentialed send.

## Solution Assessment

Yes. A Zalo-local opaque string predicate is the best boundary because the upstream sendMessage contract treats chat_id as a String; a shared heuristic change would be too broad and the earlier hex regex encoded an undocumented grammar.
