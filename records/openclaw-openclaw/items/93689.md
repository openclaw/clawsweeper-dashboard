## Summary

Keep open: the PR is a focused, viable fix for the open Zalo nonnumeric `chat_id` bug, with no blocking code findings; maintainer re-review should decide whether to accept the intentionally broadened direct-target heuristic.

## What This Changes

The PR changes the Zalo channel target resolver to treat normalized nonblank Zalo targets as opaque chat IDs and adds regression coverage for numeric, hex-like, dotted, prefixed, and blank targets.

## Best Possible Solution

Land the plugin-local opaque `chat_id` predicate after maintainer re-review accepts the widened direct-target behavior, while keeping shared outbound heuristics unchanged.

## Reproduction Assessment

Yes. Current main wires Zalo `targetResolver.looksLikeId` to a numeric-only helper, and the shared outbound resolver only accepts direct targets when that predicate returns true.

## Solution Assessment

Yes. A Zalo-local opaque string predicate matches the upstream `chat_id` contract better than a shared heuristic change, an undocumented hex-only grammar, or requiring directory lookup for direct upstream IDs.
