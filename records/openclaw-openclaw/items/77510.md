## Summary

Current main does not require WhatsApp senders present in `allowFrom` to complete a separate pairing step; the docs, source, and shared tests show `pairing` mode merges configured allowlists with pairing-store approvals.

## What This Changes

The issue asks for WhatsApp docs or defaults to clarify how `dmPolicy: "pairing"` interacts with `allowFrom` for direct messages.

## Best Possible Solution

Keep the current union behavior and docs; ask for exact account config and normalized sender logs only if a live WhatsApp DM still drops despite matching the effective `allowFrom`.

## Reproduction Assessment

No. Current main shows WhatsApp direct senders in the effective `allowFrom` are admitted before pairing is requested, and the shared tests cover merged configured and pairing-store allowlists.

## Solution Assessment

Not applicable; there is no patch to accept. Changing the default to `allowlist` based on non-empty `allowFrom` is not needed for the current behavior and would be a separate product decision.
