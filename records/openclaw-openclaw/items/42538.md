## Summary

Current main already fixes the reported health/runtime mismatch: health snapshots now merge channel runtime state, refresh stale cached values when runtime lifecycle differs, and include regression coverage for `running` and `connected` staying true.

## What This Changes

The issue asks for `health` to report the same live WhatsApp `running` and `connected` state as `channels.status`.

## Best Possible Solution

Keep the shipped runtime-aware health implementation; no additional core change is needed for this report.

## Reproduction Assessment

Yes, source-level. The current regression test builds a runtime snapshot with `running: true` and `connected: true` and verifies those values appear in health, which covers the reported mismatch without requiring live WhatsApp in this read-only review.

## Solution Assessment

Yes. The implemented fix is generic at the gateway health/runtime boundary rather than WhatsApp-specific, and cache freshness now also checks runtime lifecycle drift before serving cached health.
