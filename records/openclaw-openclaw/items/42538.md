## Summary

Current main and shipped v2026.5.12 already merge gateway runtime snapshots into health generation, and WhatsApp summaries read `running`/`connected` from that merged snapshot. The recent channel/config repair PR is distinct, but this specific health/status mismatch is covered.

## What This Changes

The issue requests that the gateway health endpoint merge live channel runtime state so WhatsApp `running` and `connected` match `channels.status`.

## Best Possible Solution

Keep the shipped runtime-backed health snapshot implementation and close this report as fixed in v2026.5.12.

## Reproduction Assessment

No live WhatsApp repro was run in this read-only sweep. Source inspection shows the reported old mismatch is addressed because current health generation now receives the same gateway runtime state used by channel status paths.

## Solution Assessment

Yes. The current implementation fixes the narrow source of drift by reusing the existing channel runtime snapshot and plugin status contract instead of adding a WhatsApp-specific workaround or new config.
