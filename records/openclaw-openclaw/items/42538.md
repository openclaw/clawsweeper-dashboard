## What This Changes

Issue reports that the gateway health endpoint reports WhatsApp as not running or connected while channels.status reports the live channel state correctly.

## Best Possible Solution

Keep the shipped runtime-aware health snapshot implementation and ask for a fresh report only if a live channel still disagrees between health and channels.status on v2026.5.20 or newer.

Yes. The issue gives concrete CLI comparison steps, and source/tests show the old missing-runtime-state path and the current runtime-aware health refresh path; I did not run a live WhatsApp session in this read-only review.

## Solution Assessment

Yes. Passing channelManager.getRuntimeSnapshot into health snapshot construction and refreshing stale cached health when lifecycle fields differ is the narrow owner-boundary fix for this mismatch.
