## Summary

Current main implements the reported fix: gateway health snapshots now receive the channel runtime snapshot, merge per-account runtime fields before channel summaries are built, and WhatsApp therefore reads live `running`/`connected` values instead of defaulting to false. The shipped target-repo fix is PR #72417, which explicitly carried forward the earlier focused work from #42586 and #42543, and it is included in `v2026.4.26`.

## What This Changes

The issue requests that the gateway `health` endpoint report the same live WhatsApp `running` and `connected` values as `channels.status`.

## Best Possible Solution

Keep the shipped runtime-backed health implementation in place; any remaining health/status mismatches for other channels or credential metadata should stay in narrower follow-up reports rather than keeping this WhatsApp `running`/`connected` bug open.
