## Summary

## What This Changes

The PR routes Tavily search/extract JSON response parsing through the shared byte-limited response reader and adds regression coverage for small and oversized streamed JSON bodies.

## Best Possible Solution

Land the shared-reader implementation after maintainers accept the 16 MiB fail-closed compatibility tradeoff; if that ceiling is too low for Tavily extract, tune the cap in this PR while keeping bounded reads.

## Reproduction Assessment

Yes. Current main's Tavily parser uses response.json() on untrusted success bodies, and the PR body gives a concrete local node:http no-Content-Length streaming proof; I did not rerun that harness in this read-only review.

## Solution Assessment

Yes. Reusing the existing plugin-sdk response-limit-runtime helper is the narrow plugin-boundary fix and matches readProviderJsonResponse; a Tavily-only custom reader would duplicate the shared contract.
