## Summary

Keep open: the bug fix is useful, but the current patch is not merge-ready because it can add a projection-time mirror on top of the current durable transcript mirror and it builds visible history from raw tool-call arguments instead of the sanitized delivered reply.

## What This Changes

The PR mirrors successful targetless `message.send` tool replies into chat and session history when the assistant finishes with `NO_REPLY`.

## Best Possible Solution

Keep the legacy-history backfill only if it is constrained to transcripts without an existing durable mirror and it reuses sanitized delivered source-reply data instead of raw tool-call arguments.

## Reproduction Assessment

Yes. The PR body gives a real transcript projection proof for the reported disappearing-reply case, and source inspection shows a current-main transcript with `message.send`, `NO_REPLY`, and a later `delivery-mirror` would project both the synthetic and durable mirrors.

## Solution Assessment

No. The durable mirror contract on current main is the safer source of truth; this PR should only backfill legacy transcripts and should not reconstruct delivered text from raw tool-call arguments.
