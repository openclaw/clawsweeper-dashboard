## Summary

Keep this PR open, but it is not merge-ready: the bug and proof are credible, while the current patch can re-display raw message-tool arguments that the real message tool intentionally sanitizes before delivery.

## What This Changes

The PR mirrors successful targetless `message.send` tool-call text into gateway chat history so a `NO_REPLY` final message does not leave reload history without a visible assistant reply.

## Best Possible Solution

Keep the history mirror, but derive mirrored text from the sanitized delivered payload/result path or the same exported sanitizer the message tool uses, with a regression covering reasoning-tag arguments.

## Reproduction Assessment

Yes, from source inspection: current main drops assistant `NO_REPLY` rows and has no generic message-tool mirror in `projectChatDisplayMessages`, so a successful targetless `message.send` followed by `NO_REPLY` can leave no visible assistant history entry.

## Solution Assessment

No, not as written. Mirroring successful message-tool replies is the right area, but the implementation must use sanitized delivered text or a shared sanitizer rather than raw model tool-call arguments.
