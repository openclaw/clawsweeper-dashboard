## Summary

Keep this PR open for maintainer review. Current main still accepts blank or whitespace-only `name` and `description` values in the skill-creator quick validator, while this PR is a narrow owner-surface fix with focused regression coverage and no blocking review findings found.

## What This Changes

This PR updates `skills/skill-creator/scripts/quick_validate.py` to reject trimmed-empty required skill metadata fields and adds four unittest cases for empty and whitespace-only `name` and `description` values.

## Best Possible Solution

Land this PR, or apply an equivalent maintainer patch scoped to the quick validator and its unittest file, so required skill metadata is rejected when blank before packaging while valid skills keep existing behavior.

## Reproduction Assessment

Yes. A skill directory whose `SKILL.md` contains `name: ""`, `name: '   '`, `description: ""`, or `description: '   '` can be traced through current-main `validate_skill()` to the strip-plus-truthy guards and success return.

## Solution Assessment

Yes. The proposed patch is the narrowest maintainable fix: reject trimmed-empty required fields immediately after type checks and add the four edge-case tests without changing packaging, loader precedence, or unrelated skill validation rules.
