## Summary

Close as a duplicate of the older open credential-encryption tracker #43794. Current main still lacks the requested encrypted `.env`/`ENC:` implementation, but the remaining security-design work is already covered by #43794’s broader encrypted-at-rest credential storage scope, including migration, keychain/master-password handling, recovery, and rotation.

## What This Changes

The issue requests built-in encrypted-at-rest handling for sensitive `.env` configuration values, including master-key setup, `ENC:` storage/decryption, and key rotation.

## Best Possible Solution

Keep #43794 as the canonical security-owned tracker for encrypted-at-rest credential storage, and fold this issue’s `.env`/`ENC:` details into that design rather than maintaining a second parallel tracker.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a feature request rather than a runtime bug; source and docs provide a high-confidence negative check that `ENC:` `.env` encryption is absent and SecretRefs remain env/file/exec only.
