## Summary

Keep open: this draft maintainer PR is not implemented on main, and the proposed repair can rewrite a Codex OAuth PI route into `openai/*` with `agentRuntime.id: "auto"` even though the current Codex harness does not auto-claim `openai` providers.

## What This Changes

The PR adds doctor warnings and `doctor --fix` repair for `openai-codex/*` primary model refs, rewriting them to `openai/*` with `agentRuntime.id: "auto"`, plus tests, docs, and changelog.

## Best Possible Solution

Keep the PR open and either repair legacy refs to a runtime that selects Codex today, or first extend the Codex harness auto-claim path for canonical OpenAI refs, then update the OpenAI provider and Codex docs together.

## Reproduction Assessment

Yes for the source-level mismatch: the PR writes `openai/*` plus `auto`, while current main's Codex harness only auto-supports provider `codex` and auto selection falls back to PI. I did not run a live Codex app-server turn.

## Solution Assessment

No: the current proposed repair is not the safest solution until `auto` can actually claim canonical OpenAI refs or the repair uses a forced Codex runtime. The user-facing OpenAI provider docs also need to match whichever contract is chosen.
