## What This Changes

The PR changes the Windows Scheduled Task install fallback in `src/daemon/schtasks.ts` to use schtasks exit code 1 instead of matching English/Spanish access-denied text.

## Best Possible Solution

Add exit code 1 as an additional fallback trigger while preserving the existing access-denied coverage, then add focused regression coverage and redacted Windows install proof.

## Reproduction Assessment

Yes. Source inspection shows current main only handles English/Spanish access-denied text plus timeout cases, and the PR would make the existing code-5 access-denied fallback test fail; I did not run tests because this review is read-only.

## Solution Assessment

No. Replacing the text match outright is not the safest fix; the maintainable path is to add code 1 while retaining the existing denied-path coverage and proving the Windows behavior.
