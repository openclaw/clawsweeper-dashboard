## Summary

Keep open: current main still has a source-level sessions counter mismatch, and the requested archive-exclusion behavior conflicts with current comments and tests that intentionally keep reset/deleted archives searchable.

## What This Changes

The issue requests aligning session memory status/indexing so reset/deleted archive artifacts do not inflate live session counters, plus follow-up on dirty-state consistency.

## Best Possible Solution

Make status and indexing share one documented session-file enumeration contract, then separately decide and document whether reset/deleted archives belong in active session search or an archive-only corpus.

## Reproduction Assessment

Yes for the counter mismatch by source inspection: a sessions directory containing live `.jsonl` files plus `.jsonl.reset.*` or `.jsonl.deleted.*` files can produce indexed session rows that exceed the status scanner total. The dirty-state race is not yet reproduced with high confidence.

## Solution Assessment

No for applying the preferred archive-skipping fix automatically: current tests and comments intentionally keep reset/deleted archives searchable. The safer first repair is to align the status denominator with the indexing contract, with archive visibility decided by maintainers.
