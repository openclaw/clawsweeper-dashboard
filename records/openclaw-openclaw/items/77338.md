## Summary

Keep open: current main still has a source-level mismatch between the sessions status denominator and the indexed sessions rows, and the requested archive-exclusion behavior conflicts with current tests/comments that intentionally keep reset/deleted archives searchable.

## What This Changes

The issue asks OpenClaw to stop `openclaw memory status` from reporting `sessions` counts where indexed rows exceed scanned files, and to decide whether reset/deleted session archives should remain in active memory search.

## Best Possible Solution

Unify status and indexing around one documented session-file enumeration contract, then separately decide whether reset/deleted archives remain searchable or move to an archive-only corpus.

## Reproduction Assessment

Yes for the counter mismatch by source inspection: a directory with primary `.jsonl` files plus `.jsonl.reset.*` or `.jsonl.deleted.*` archives will be indexed through the usage-counted helper while the status denominator uses a strict `.jsonl` suffix scan. The secondary dirty-state race is not yet reproduced with high confidence.

## Solution Assessment

No for the issue's preferred archive-skipping change as an automatic repair: current code comments and tests intentionally keep reset/deleted archives searchable. The safest narrow fix is to make status and indexing share the same enumeration semantics, with archive search policy handled by maintainers.
