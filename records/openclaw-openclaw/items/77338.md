## Summary

Keep open: current main still has the sessions source-counter mismatch by source inspection, while the requested archive-skipping behavior conflicts with existing comments and tests that deliberately keep reset/deleted session archives searchable.

## What This Changes

The issue requests fixing `openclaw memory status` sessions counts and active search indexing so reset/deleted session archives are not treated inconsistently with live transcript files.

## Best Possible Solution

Define one shared session-file enumeration contract for status and indexing, then make an explicit maintainer policy decision for whether reset/deleted archives remain searchable or move out of active recall.

## Reproduction Assessment

Yes for the counter mismatch by source inspection: a sessions directory with live `.jsonl` files plus `.jsonl.reset.*` or `.jsonl.deleted.*` archives will use different denominator and numerator filters. The dirty-state race is unclear because I did not establish a deterministic current-main reproduction path.

## Solution Assessment

No for applying the preferred archive-skipping fix automatically: current comments and tests intentionally keep those archives searchable. The narrower maintainable fix is to align or document the status/index enumeration, with archive recall policy handled deliberately.
