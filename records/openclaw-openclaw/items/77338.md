## Summary

Keep open: current main still has a source-level sessions counter mismatch, and the requested archive-exclusion behavior conflicts with current tests/comments that intentionally keep reset/deleted archives searchable.

## What This Changes

The issue asks OpenClaw memory status and session indexing to use consistent session-file filters and to decide whether reset/deleted session archives belong in active memory search.

## Best Possible Solution

Unify memory status and indexing around one documented session-file enumeration contract, then make an explicit maintainer policy call on whether reset/deleted archives remain searchable or move out of the active sessions corpus.

## Reproduction Assessment

Yes for the counter mismatch by source inspection: a sessions directory with live .jsonl files plus .jsonl.reset.* or .jsonl.deleted.* archives will index the broader helper set while status counts only names ending in .jsonl. The dirty-state race is still unclear.

## Solution Assessment

No for the issue's preferred archive-skipping fix as an automatic repair, because current tests and comments intentionally keep reset/deleted archives searchable. The narrow maintainable counter repair is to share or document the same enumeration for status and indexing, with archive recall semantics decided separately.
