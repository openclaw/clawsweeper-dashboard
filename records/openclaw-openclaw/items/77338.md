## Summary

Keep open: current main still has a source-level sessions counter mismatch, while the requested archive-skipping behavior conflicts with intentional reset/deleted archive recall tests and comments.

## What This Changes

The issue asks OpenClaw to stop `memory status` sessions counters from overflowing and to revisit whether reset/deleted session archives should be indexed as live session-search content.

## Best Possible Solution

Align `memory status` and session indexing on one documented file-enumeration contract, or split live and archive counts explicitly, while making a deliberate maintainer decision about reset/deleted archive recall semantics.

## Reproduction Assessment

Yes for the counter mismatch by source inspection: a sessions directory with live `.jsonl` files plus `.jsonl.reset.*` or `.jsonl.deleted.*` archives will use different denominator and numerator filters. The dirty-state race is unclear because no high-confidence current-main reproduction path was established.

## Solution Assessment

No for the issue’s preferred archive-skipping fix as an automatic repair: current comments and tests intentionally keep these archives searchable. The narrow maintainable fix is to share or document the session-file enumeration used by status and indexing, with archive retention handled as explicit policy.
