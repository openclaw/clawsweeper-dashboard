## Summary

Keep open: current main still has a source-level sessions count mismatch, and the archive-skipping part of the requested fix conflicts with existing code comments and tests that intentionally keep reset/deleted archives searchable.

## What This Changes

The issue asks for memory sessions status/indexing to stop treating reset/deleted archive snapshots as live indexed session files, or at least to make status and indexing use the same session-file enumeration.

## Best Possible Solution

Make status and indexing share one documented session-file enumeration contract, then decide separately whether reset/deleted archives stay searchable or move to an archive-only corpus.

## Reproduction Assessment

Yes for the counter mismatch by source inspection: a sessions directory with live `.jsonl` plus `.jsonl.reset.*` or `.jsonl.deleted.*` files will be indexed through the usage-counted helper while the status denominator uses only names ending in `.jsonl`. The dirty-state race is not yet reproduced with high confidence.

## Solution Assessment

No for the preferred archive-skipping fix as an automatic change: current tests and comments intentionally keep reset/deleted archives searchable. The narrow maintainable repair is to align the status denominator with the indexing helper, with archive search semantics decided by maintainers.
