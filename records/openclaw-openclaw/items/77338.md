## Summary

Keep open: current main still has the source-level sessions status counter mismatch, while reset/deleted archive searchability is now intentional in code, tests, and the related closed archive-recall issue.

## What This Changes

The issue asks OpenClaw to fix `memory status` sessions counters that can show more indexed session files than scanned files, and to decide whether reset/deleted archives should remain in active session memory search.

## Best Possible Solution

Align `memory status` with the same usage-counted session transcript contract used by indexing, while leaving reset/deleted archive recall and retention policy to the broader maintainer decision tracked at https://github.com/openclaw/openclaw/issues/56609.

## Reproduction Assessment

Yes for the counter mismatch by source inspection: current main counts only filenames ending in `.jsonl` for status totals, while the session indexer enumerates usage-counted reset/deleted archives. The dirty-state race is unclear because this read-only review did not establish a current-main runtime reproduction path.

## Solution Assessment

No for the requested archive-skipping fix as an automatic repair: current code, tests, and the related closed archive-drop issue intentionally keep reset/deleted archives searchable. The maintainable narrow fix is to align the status denominator with the indexer contract, with retention/search semantics decided explicitly.
