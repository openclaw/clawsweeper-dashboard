## Summary

Keep open: current main still has the source-level session count mismatch, and the requested archive-skipping behavior conflicts with current tests and changelog language that intentionally keep reset/deleted session archives searchable.

## What This Changes

The issue asks OpenClaw to fix memory session status overflow caused by divergent session-file filters and to revisit indexing of reset/deleted session archives plus a possible post-index dirty-state race.

## Best Possible Solution

Align memory status source totals with the same session-file enumeration contract used by indexing, then handle any change to reset/deleted archive searchability as an explicit maintainer policy decision.

## Reproduction Assessment

Yes for the counter mismatch by source inspection: a sessions directory with `active.jsonl` plus `active.jsonl.reset.*` or `.deleted.*` will be indexed by the usage-counted helper but counted by status only if the name ends with `.jsonl`. No high-confidence current-main reproduction was established for the nondeterministic dirty-state report.

## Solution Assessment

No for the proposed archive-skipping fix as-is, because current code, tests, and changelog intentionally keep reset/deleted archives searchable. The narrow maintainable repair is to share enumeration for status/counting unless maintainers decide to change archive search policy.
