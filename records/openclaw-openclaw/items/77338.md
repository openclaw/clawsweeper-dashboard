## Summary

Keep open: current main still has a source-level sessions counter mismatch, and the suggested archive-skipping fix conflicts with existing code comments, tests, and changelog entries that intentionally keep reset/deleted archives searchable.

## What This Changes

The issue requests fixing `openclaw memory status` sessions counters that can report indexed files above scanned totals and deciding whether reset/deleted session archives should be indexed as active search data.

## Best Possible Solution

Align `memory status` and indexing on one documented session-file enumeration contract, then make an explicit maintainer decision for reset/deleted archive searchability.

## Reproduction Assessment

Yes for the sessions counter mismatch: source inspection on current main shows `scanSessionFiles` counts only names ending in `.jsonl`, while full session sync indexes a helper that includes `.jsonl.reset.*` and `.jsonl.deleted.*`. The dirty-state race remains unclear.

## Solution Assessment

No for the preferred archive-skipping fix as an automated repair: existing comments, tests, and changelog entries intentionally keep these archives searchable. The safer path is to first align the status denominator with the indexing contract or deliberately change the retention/search policy.
