## Summary

Keep open: current main still has a source-level sessions status count mismatch, while the requested archive exclusion conflicts with intentionally tested reset/deleted archive recall behavior.

## What This Changes

Request consistent session-file filtering for memory status and session indexing, with reset/deleted archives excluded from active session counts/search or otherwise reconciled.

## Best Possible Solution

Define one documented session-file enumeration contract shared by status and sync, then make an explicit maintainer decision on whether reset/deleted archives belong in active search, archive-only search, or no search.

## Reproduction Assessment

Yes for the counter mismatch by source inspection: status totals count strict `.jsonl` files while indexing uses the usage-counted helper that includes reset/deleted archives. The dirty-state race remains unclear.

## Solution Assessment

No for the preferred archive-skipping fix as an automatic change: current tests and changelog intentionally preserve archive recall. The narrow maintainable fix is shared enumeration for status/indexing plus a separate maintainer policy call on deleted/reset archive search semantics.
