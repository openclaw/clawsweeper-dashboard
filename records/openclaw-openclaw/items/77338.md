## Summary

Keep open: current main still has a source-level mismatch between the sessions status denominator and the sessions indexer, while the requested archive-skipping behavior conflicts with intentional archive searchability tests and comments.

## What This Changes

The issue asks for `memory status` and session indexing to stop treating reset/deleted archive snapshots as live session files, and reports a possible dirty-state race after forced indexing.

## Best Possible Solution

Align `memory status` and indexing on one documented session-file enumeration contract, then make an explicit maintainer decision about whether reset/deleted archives belong in active session search or a separate archival corpus.

## Reproduction Assessment

Yes for the counter mismatch by source inspection: a sessions directory containing live `.jsonl` files plus `.jsonl.reset.*` or `.jsonl.deleted.*` archives will feed different file sets to status and indexing. The dirty-state race is unclear because I did not establish a current-main failing path for it.

## Solution Assessment

No for the preferred archive-skipping change as an automatic fix: current comments and tests intentionally keep reset/deleted archives searchable. The narrow maintainable fix is to align the counter with the indexing contract, with archive retention/search semantics decided deliberately.
