## Summary

Keep this open: current main and latest release still use a strict `.jsonl` denominator for `openclaw memory status`, while the session indexer uses the usage-counted transcript predicate. The open linked fix PR is the current landing path, so this issue should remain open until that PR or an equivalent fix reaches main.

## What This Changes

`openclaw memory status` should stop reporting a sessions numerator larger than its denominator when reset/deleted archive transcripts are present.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/95452 or an equivalent narrow fix that reuses the canonical usage-counted session transcript predicate for the status denominator.

## Reproduction Assessment

Yes, at source level. Current main still renders status totals from a strict `.jsonl` scan while session indexing uses the usage-counted helper; the issue also includes fleet SQLite evidence, but I did not run a live CLI reproduction in this read-only review.

## Solution Assessment

Yes for the active fix shape, but not for excluding reset/deleted archives from the index. The narrow maintainable solution is to share `isUsageCountedSessionTranscriptFileName()` between the status denominator and the session indexer.
