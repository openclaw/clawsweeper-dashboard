## Summary

Keep this open: current main and the latest release still have the session-status denominator using a strict `.jsonl` suffix while the session indexer uses the usage-counted session helper. The active linked PR is the right fix path, so this issue should stay open until that PR lands or is replaced.

## What This Changes

The issue reports a memory status/session-index count mismatch where reset/deleted session archive files are indexed differently than they are counted in `openclaw memory status`.

## Best Possible Solution

Land a narrow fix that makes the status denominator reuse `isUsageCountedSessionTranscriptFileName()`, preserving reset/deleted archive recall while excluding backup, checkpoint, and trajectory artifacts consistently.

## Reproduction Assessment

Yes. Source inspection shows current main counts status denominator files with a strict `.jsonl` suffix while session indexing uses the usage-counted helper that includes `.jsonl.reset.*` and `.jsonl.deleted.*`; I did not run a live CLI reproduction in this read-only review.

## Solution Assessment

Yes for the active fix shape, but not for the issue body's original archive-exclusion suggestion. The narrow maintainable solution is to share the existing indexer filename predicate in status output.
