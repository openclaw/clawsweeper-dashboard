## Summary

Keep open: current main still has the source-level sessions counter mismatch, while reset/deleted archive searchability is now intentional in code and tests rather than something this cleanup pass should close as stale.

## What This Changes

The issue asks to fix `openclaw memory status` sessions counters that can exceed their scanned total and to reconsider indexing reset/deleted session archives as active memory.

## Best Possible Solution

Align `memory status` with the same usage-counted session transcript contract used by indexing, while leaving reset/deleted archive retention and search policy to the broader maintainer decision tracked at https://github.com/openclaw/openclaw/issues/56609.

## Reproduction Assessment

Yes for the counter mismatch by source inspection: current main counts only names ending in `.jsonl` for status totals, while the indexer enumerates usage-counted reset/deleted archives. The dirty-state race is unclear because this read-only review did not establish a current-main reproduction path.

## Solution Assessment

No for the requested archive-skipping fix as an automatic repair: current comments, tests, and the related closed archive-drop issue intentionally keep reset/deleted archives searchable. The maintainable code fix is to align the status denominator with the indexing contract, with retention/search semantics decided explicitly.
