## Summary

Keep open: current main still has a source-level sessions counter mismatch, and the requested archive-exclusion behavior conflicts with existing comments/tests that intentionally keep reset/deleted archives searchable.

## What This Changes

The issue requests aligned session-file enumeration for memory status/indexing, exclusion or cleanup of reset/deleted archives from active search, and investigation of a post-force-index dirty-state race.

## Best Possible Solution

Make status and indexing share one documented session-file enumeration contract, then explicitly decide and document whether reset/deleted archives belong in active session search or an archive-only corpus.

## Reproduction Assessment

Yes for the counter overflow by source inspection: with session sources enabled, reset/deleted archive files are indexed by the usage-counted helper but excluded from the status scan because they do not end in .jsonl. No high-confidence reproduction was established for the dirty-state race.

## Solution Assessment

No for the preferred archive-skipping repair as an automatic fix: current comments and tests intentionally preserve reset/deleted archives in search. The maintainable path is to align the counter contract first and get a maintainer decision on archive recall semantics.
