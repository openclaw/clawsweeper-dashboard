## Summary

Keep open: current main still has a source-level sessions counter mismatch, and the requested archive-skipping behavior conflicts with existing comments and tests that intentionally keep reset/deleted archives searchable.

## What This Changes

The issue asks to fix `openclaw memory status` sessions counters overflowing when reset/deleted session archives are indexed, and to investigate dirty state after forced indexing.

## Best Possible Solution

Align `memory status` and indexing on one documented session-file enumeration contract, then make an explicit maintainer decision for reset/deleted archive searchability.

## Reproduction Assessment

Yes for the counter mismatch by source inspection: a sessions directory with live `.jsonl` files plus `.jsonl.reset.*` or `.jsonl.deleted.*` archives will use different denominator and numerator filters. The dirty-state race is unclear because no high-confidence current-main reproduction path was established.

## Solution Assessment

No for the preferred archive-skipping fix as an automatic repair: current code comments and tests intentionally keep these archives searchable. The narrow maintainable fix is to share or document the session-file enumeration used by status and indexing, with archive retention/search policy handled deliberately.
