## Summary

Keep open: current main still has a source-level sessions counter mismatch, while the requested archive-skipping behavior conflicts with existing comments and tests that deliberately keep reset/deleted session archives searchable.

## What This Changes

The issue asks `memory status` and session indexing to stop treating reset/deleted session archive artifacts inconsistently, and also reports post-force dirty-state nondeterminism.

## Best Possible Solution

Align `memory status` and indexing around one documented session-file enumeration contract, then explicitly document and test the chosen reset/deleted archive recall semantics.

## Reproduction Assessment

Yes for the counter mismatch by source inspection: a sessions directory with live `.jsonl` files plus `.jsonl.reset.*` or `.jsonl.deleted.*` archives will be indexed by the broader helper but counted by the strict status scanner. The dirty-state race remains unclear because I did not reproduce it against current main.

## Solution Assessment

No for the requested archive-skipping change as an automatic fix: current comments and tests intentionally keep these archives searchable. The narrow maintainable fix is a shared/documented enumeration contract, with archive retention handled by maintainer policy.
