## What This Changes

The branch adds a Feishu `feishu_chat` `list` action, optional `accountId`/per-agent routing, optional `chat_id`, pagination/sort parameters, and expanded chat-tool unit tests.

## Best Possible Solution

Land a rebased Feishu plugin change that adds joined-chat listing through the existing tool-account seam while preserving `member_info`, current runtime/import contracts, diagnostic error formatting, and account-disable enforcement.

## Reproduction Assessment

Yes. A static reproduction is clear from the PR head: pass a disabled Feishu `accountId` with valid credentials and default `tools.chat`, and execution reaches `createFeishuClient` because the branch checks only the per-tool flag.

## Solution Assessment

No, not as currently proposed. The feature direction matches the existing Feishu tool-account pattern, but the branch needs a rebase plus disabled-account, import, `member_info`, diagnostics, and changelog fixes before it is the narrow maintainable solution.
