## What This Changes

The branch extends the Feishu `feishu_chat` tool with joined-chat listing, pagination/sort parameters, optional `accountId`/per-agent routing, and expanded unit coverage.

## Best Possible Solution

Land a rebased Feishu plugin change that adds joined-chat listing through the existing tool-account seam while preserving `member_info`, diagnostics, import/dependency contracts, account-disable enforcement, tests, and changelog.

## Reproduction Assessment

Yes. From PR-head source, a disabled but configured Feishu `accountId` can be passed to `feishu_chat` while another enabled account registers the tool, and execution reaches `createFeishuClient` after only checking `tools.chat`.

## Solution Assessment

No. The feature direction fits the current Feishu tool-account pattern, but the branch needs a focused rebase that preserves current chat behavior and rejects disabled accounts before it is the narrow maintainable solution.
