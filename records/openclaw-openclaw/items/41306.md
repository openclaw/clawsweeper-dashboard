## Summary

Keep open: current main still lacks the requested Feishu chat-listing/per-agent chat routing behavior, but this PR is not mergeable as-is because it regresses current Feishu contracts, has a disabled-account execution path, and only provides mocked/unit-test proof.

## What This Changes

The branch adds a `list` action to `feishu_chat`, exposes pagination/sort/account parameters, switches chat execution to account-aware routing, and expands Feishu chat unit tests.

## Best Possible Solution

Land a rebased Feishu chat-listing change through the existing tool-account seam while preserving current chat actions, imports, diagnostics, account-disable enforcement, focused tests, and maintainer-owned release notes.

## Reproduction Assessment

Yes. Source inspection shows a configured disabled account can be passed through `accountId` after another enabled account registers `feishu_chat`; PR-head execution checks `tools.chat` but not `resolvedAccount.enabled` before creating the client.

## Solution Assessment

No. The feature direction fits the Feishu tool-account pattern, but this branch needs a focused rebase and disabled-account guard before it is the narrow maintainable solution.
