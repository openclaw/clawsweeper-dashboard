## Summary

Keep this PR open. Current main still lacks the proposed `feishu_chat` list action and per-agent routing, so the contribution is not obsolete, but the branch is stale against current main and still has a concrete disabled-account routing/security defect before it is mergeable.

## What This Changes

The PR adds a `list` action to `feishu_chat`, optional chat/account routing parameters, account-aware execution, and focused Feishu chat tests.

## Best Possible Solution

Rebase the PR onto current main, keep the existing `member_info` action, add the chat-list action and pagination/sort forwarding, route through the existing Feishu account seam, and reject disabled or chat-disabled accounts at execution time with focused tests for each path.

## Reproduction Assessment

Yes. A static reproduction is clear: configure one enabled Feishu account and one disabled account with `tools.chat: true`, then call the PR head `feishu_chat` with the disabled `accountId`; the code resolves the disabled account and creates a client without checking `enabled`.

## Solution Assessment

No, not as currently proposed. The feature direction matches the existing Feishu routing seam, but the patch needs a rebase and an account-level enabled guard before it is the narrow maintainable fix.

Status: needs_attention
