## Summary

Keep open: the PR targets a valid gateway SecretRef log-noise bug, but the new skip predicate is too broad and can bypass gateway snapshot resolution because of inactive exec-backed targets.

## What This Changes

The PR adds target exec SecretRef detection to `resolveCommandSecretRefsViaGateway` and a regression test that skips gateway resolution for an active exec-backed model-provider target.

## Best Possible Solution

Narrow the pre-gateway skip to exec refs that are active for this command or otherwise known to make the gateway request fail, while preserving gateway snapshot resolution for inactive exec refs and unrelated active credentials.

## Reproduction Assessment

Yes at source level: current main sends reply target SecretRefs through gateway `secrets.resolve`, and the linked issue shows the missing active target-exec skip. The blocking PR edge is also source-reproducible because the new helper collects exec targets without checking active/inactive state.

## Solution Assessment

No as written: skipping known active exec target refs is the right fix shape, but the implementation should not let inactive exec refs bypass gateway resolution for unrelated active credentials.
