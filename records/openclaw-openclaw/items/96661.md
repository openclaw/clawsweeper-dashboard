## Summary

Keep open: the PR targets a real SecretRef log-noise bug, but the patch skips gateway snapshot resolution for target exec refs even when a caller intentionally disables local exec, which can turn an already-resolved gateway assignment into an unresolved command config. The PR also has only source/unit-test proof, not real gateway/reply behavior proof.

## What This Changes

The PR broadens `resolveCommandSecretRefsViaGateway` so target exec SecretRefs bypass gateway `secrets.resolve` and resolve through the local fallback path.

## Best Possible Solution

Preserve gateway snapshot resolution when local exec is disabled, limit the new pre-skip to policy-safe cases, and add regression coverage plus redacted real gateway/reply proof.

## Reproduction Assessment

Yes at source level: current main only pre-skips gateway for gateway credential exec refs, and the PR-introduced no-local-exec regression follows directly from the new unconditional target-exec skip plus the local fallback policy. I did not run a live gateway turn in this read-only review.

## Solution Assessment

No. The shared resolver is the right area, but the current implementation is too broad because it bypasses gateway snapshot resolution even when local exec is deliberately disabled.
