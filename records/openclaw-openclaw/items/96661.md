## Summary

Keep open: the revised head fixes the earlier no-local-exec blocker and adds real terminal proof, but the new target-exec shortcut can still skip gateway snapshot resolution because of an inactive exec SecretRef while other active targets need the gateway assignment.

## What This Changes

The PR broadens command SecretRef gateway pre-skip logic to include target exec SecretRefs and updates command-secret gateway tests.

## Best Possible Solution

Limit the target-exec gateway shortcut to command-active exec refs, preserve gateway snapshot resolution for inactive exec refs mixed with other active targets, and keep the existing live proof path once the repair lands.

## Reproduction Assessment

Yes at source level: current main reaches gateway secrets.resolve for agent-runtime targets, and the PR head can bypass that gateway call based on any discovered target exec ref. I did not run a live mixed-target repro in this read-only review.

## Solution Assessment

No. The shared resolver is the right layer, but the target-exec skip should honor active/inactive target classification so inactive configured exec refs do not suppress gateway snapshot resolution for unrelated active targets.
