## Summary

Keep open. Current main still builds the plugin metadata snapshot and manifest registry before deriving the narrowed startup plugin plan, and the 2026.5.12 user datapoint says discovery still scales with the bundled plugin tree despite `plugins.allow`.

## What This Changes

The issue asks OpenClaw to scope bundled plugin discovery before manifest reads and validation so allowlisted installs do not pay heap and latency cost for unused plugins.

## Best Possible Solution

Carry a conservative startup/control-plane plugin-id scope into metadata snapshot and installed-index registry reconstruction before manifest reads, while preserving existing activation semantics and config validation behavior.

## Reproduction Assessment

Yes at source level: current main still builds metadata and an unscoped manifest registry before the narrowed startup plugin plan is known. I did not run a live heap or strace reproduction in this read-only pass.

## Solution Assessment

Yes, discovery-time scoping is the right fix direction, but it should use existing plugin activation/config rules rather than treating `tools.allow` as the only source of truth.
