## Summary

Keep open. Current main still builds broad plugin metadata and manifest registries before deriving the runtime/startup plugin scope, so the reported discovery-time manifest cost remains source-confirmed despite adjacent runtime reuse work.

## What This Changes

The issue requests filtering bundled plugin discovery before manifest reads/validation when a narrow tool/plugin allowlist makes most bundled plugins irrelevant.

## Best Possible Solution

Add a conservative pre-manifest plugin scope for startup and tool-control-plane metadata, pass it through metadata snapshot and installed-index registry reconstruction, and fall back to the full scan when the scope cannot be proven without changing plugin trust or activation semantics.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: with a non-empty installed index, current main reaches `loadPluginManifestRegistryForInstalledIndex` without `pluginIds`, then reads every candidate manifest before tool/startup filtering can narrow the set.

## Solution Assessment

Yes, discovery-time scoping is the right direction, but it should not blindly treat every `tools.allow` value as a plugin id; it should derive a safe scope from explicit plugin ids, config-owned startup needs, and manifest/tool contract metadata, with full-scan fallback.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/70533 with a narrow plugin metadata/discovery scoping PR. The observable bug is that Gateway/tool startup builds a full installed-index manifest registry before runtime startup/tool scopes are known, so narrow allowlist configs still read and validate every bundled plugin manifest. Implement a conservative pre-manifest plugin-id scope and thread it through `loadPluginMetadataSnapshot`, `loadPluginLookUpTable`, `loadManifestContractSnapshot`, and `loadPluginManifestRegistryForInstalledIndex`; preserve current behavior by falling back to full metadata when a safe scope cannot be derived. Do not make `tools.allow` override `plugins.allow`, do not weaken plugin trust/path safety checks, do not eagerly import runtime modules, and preserve manifest-first contracts for third-party plugins. Add focused regression tests proving scoped metadata calls pass `pluginIds` before manifest reads while broad/ambiguous configs still scan fully. Add a single-line CHANGELOG.md fix entry because this is user-facing performance behavior.
