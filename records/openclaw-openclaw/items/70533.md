## Summary

Keep open. Current main still builds the plugin metadata snapshot and installed-index manifest registry without a narrowed plugin-id scope, and recent 2026.5.12 user evidence says pruning bundled plugin manifests reduces discovery work even when `plugins.allow` is configured.

## What This Changes

The issue asks OpenClaw to apply plugin/tool allowlist scoping before reading and validating bundled plugin manifests during discovery.

## Best Possible Solution

Carry a conservative startup/control-plane plugin-id scope into metadata snapshot and installed-index registry reconstruction before manifest reads, while preserving existing activation semantics and compatibility modes.

## Reproduction Assessment

Yes, at source level: current main still builds the metadata snapshot and manifest registry before deriving a narrowed startup plugin scope. I did not run a live heap profile in this read-only pass.

## Solution Assessment

Yes. Discovery-time scoping is the right direction, but it should use the existing plugin activation/config rules rather than applying `tools.allow` literally as the only source of truth.
