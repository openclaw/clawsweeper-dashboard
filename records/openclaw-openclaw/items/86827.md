- `impact:session-state`: The failure centers on persisted session status and reply-operation state after a timed-out or failed run.

## Summary

Keep open: the live v2026.5.22 Octo evidence describes real message loss, and current main still has a plausible core path where failed/timeout session state or active reply-operation state can leave a group turn with no new agent run or visible error.

## What This Changes

The issue asks OpenClaw to recover or visibly fail when a group-channel session remains in failed/timeout state after a prior AI turn timeout.

## Best Possible Solution

Fix the shared core session/reply lifecycle so a failed or timed-out group turn cannot silently suppress the next plugin-channel message, and add regression coverage for group timeout recovery through the plugin dispatch path.

## Reproduction Assessment

No, not with high confidence in this checkout: the reporter supplied live Octo evidence and current source supports a plausible stale session/reply-operation path, but the external Octo plugin needed for the exact group repro is absent.

## Solution Assessment

Unclear: auto-resetting failed/timeout sessions to idle may be part of the fix, but the safer core solution should also clear or surface stale reply-operation admission so plugin channels do not silently drop the next turn.
