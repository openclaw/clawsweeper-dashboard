## What This Changes

The issue reports that `openclaw proxy sessions --limit abc` reaches SQLite and crashes with `datatype mismatch` instead of rejecting the invalid CLI option.

## Best Possible Solution

Review and land or decline https://github.com/openclaw/openclaw/pull/84260; the desired end state is strict debug proxy numeric validation before runtime dispatch.

## Reproduction Assessment

Yes. Source inspection on current main shows invalid `--limit` can become a non-nullish empty string through Commander and then be bound into SQLite `LIMIT ?`; I did not execute the CLI in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to reject invalid values in `src/cli/proxy-cli.ts` before runtime dispatch, using strict positive-integer parsing for limits/timeouts and the shared port parser for bind ports.
