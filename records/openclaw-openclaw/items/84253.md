## Summary

Keep this open: current main still has the invalid debug proxy numeric option path, and the linked fix PR is open and unmerged, so the issue should remain the tracking item until that PR lands or is replaced.

## What This Changes

The issue asks OpenClaw to reject invalid debug proxy numeric CLI options before they reach runtime SQLite code.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/84260, then resolve this issue based on that outcome.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: invalid `--limit` can leave the CLI parser with a non-number value, the sessions runtime forwards it, and the SQLite store binds it into `LIMIT ?`; I did not run the CLI in this read-only review.

## Solution Assessment

Yes. Rejecting invalid values at the Commander option parser boundary is the narrow maintainable fix, and the open linked PR is already the implementation candidate for that path.
