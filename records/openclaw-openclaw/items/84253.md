## What This Changes

Reject invalid debug proxy numeric CLI options before runtime dispatch so `openclaw proxy sessions --limit abc` fails with a CLI argument error instead of leaking a SQLite datatype mismatch.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84260 or an equivalent CLI-boundary validation patch, then close this issue as implemented.

## Reproduction Assessment

Yes. Source inspection on current main matches the reported path: invalid `--limit` can become a non-number Commander value, bypass the runtime default, and reach SQLite `LIMIT ?`; I did not run the CLI because this review is read-only.

## Solution Assessment

Yes. The maintainable fix is to reject malformed debug-proxy numeric options in `src/cli/proxy-cli.ts` before runtime dispatch, with focused Commander parser coverage and a real CLI smoke proof.
