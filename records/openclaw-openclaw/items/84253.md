## What This Changes

The issue asks for debug proxy numeric CLI options, especially `proxy sessions --limit`, to reject invalid values before reaching SQLite.

## Best Possible Solution

Review and land the linked CLI parser fix with regression coverage, keeping malformed numeric input rejection at the `src/cli/proxy-cli.ts` boundary.

## Reproduction Assessment

Yes. Source inspection on current main shows invalid `--limit` can become a non-number passed into `listSessions()` and bound into SQLite `LIMIT ?`; the issue and linked PR also provide terminal proof.

## Solution Assessment

Yes. Rejecting invalid debug proxy numeric arguments in `src/cli/proxy-cli.ts` before runtime dispatch is the narrowest maintainable fix.
