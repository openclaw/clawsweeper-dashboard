## Summary

This should stay open: current main and the latest release still route invalid debug proxy numeric values through the permissive parser, and the same author has an open closing PR that should be reviewed or landed before the issue closes.

## What This Changes

The report asks for debug proxy numeric CLI options, especially `openclaw proxy sessions --limit abc`, to fail at argument parsing instead of leaking a SQLite datatype error.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84260, or an equivalent parser-boundary fix, so invalid debug proxy numeric values are rejected before runtime dispatch.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: invalid `--limit` returns `undefined`, Commander turns the required option value into an empty string, and the runtime passes that non-nullish value into SQLite `LIMIT ?`.

## Solution Assessment

Yes. The best fix is to validate numeric values in `src/cli/proxy-cli.ts` before runtime dispatch; the open linked PR implements that general shape and should be reviewed for merge.
