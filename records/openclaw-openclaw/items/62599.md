## Summary

Keep open. Plain text status is partially mitigated on current main, but the detailed JSON path still loads/probes the configured memory plugin in the CLI process, and an open closing PR is already paired with this issue rather than merged or superseded.

## What This Changes

The issue reports that status initializes the configured memory plugin in the CLI process and can show memory/vector state that disagrees with the live gateway.

## Best Possible Solution

Resolve https://github.com/openclaw/openclaw/pull/62643 or replace it with a runtime/passive status fix that prevents CLI-local plugin startup/probe side effects and avoids misleading vector state.

## Reproduction Assessment

Yes for the source-level path: current main’s status --json --all flow reaches the memory resolver, obtains a CLI-local manager with purpose "status", and probes vector availability. The exact third-party macOS false-vector state is supported by the issue logs but was not live-run here.

## Solution Assessment

Unclear. The passive purpose="status" contract may be acceptable, but maintainers still need to decide whether the better fix is gateway-authoritative status or stricter suppression of CLI-local plugin side effects.
