## Summary

Keep open. Current main mitigates fast status output, but memory-reporting status paths still construct and probe a CLI-local memory manager for custom memory slots, so the reported side-effect/live-gateway mismatch remains unresolved.

## What This Changes

The issue asks for `openclaw status` to stop locally initializing memory plugins and avoid reporting CLI-local vector state that can disagree with the live gateway.

## Best Possible Solution

Make memory-reporting status either gateway-authoritative for live plugin health or explicitly passive/indeterminate without probing plugin startup paths.

## Reproduction Assessment

Yes for the central path from source: a custom memory slot still causes memory-reporting status to construct and probe a local status-purpose memory manager. I did not live-reproduce the exact macOS/sqlite-vec false-vector output against current main.

## Solution Assessment

No final solution is on main yet. The existing fast-status and custom-slot fixes are useful mitigations, but the best remaining solution needs a maintainer-approved status boundary.
