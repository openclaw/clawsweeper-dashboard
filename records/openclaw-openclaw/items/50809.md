## Risks / Open Questions

- Inbound Android SMS would expose private local messages to model turns, so permission prompts, allowlist defaults, phone-number normalization, logging redaction, and retention need explicit security design.
- Outbound SMS replies through `sms.send` create real phone-network side effects and must remain aligned with dangerous node-command policy and capability gates.
- The ownership boundary is unresolved: Android receiver support must live in the node app, while gateway routing could be a bundled channel, an external plugin with a generic node-event seam, or a narrower Android-node command surface.
