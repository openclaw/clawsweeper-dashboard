## Summary

Keep open. Current main and the latest release still allow raw host exec to run the reported OpenClaw config/provider/model mutations and gateway restart without a deterministic confirmation boundary in full/off mode.

## What This Changes

Require explicit confirmation before agent-initiated exec runs example OpenClaw config/model/provider changes or gateway restarts.

## Best Possible Solution

Define a deterministic OpenClaw self-management exec policy covering config writes, provider/model/default changes, gateway lifecycle, updates, and self-removal, with regression tests and docs explaining how it composes with full/off host automation.

## Reproduction Assessment

Yes, at source level: current main resolves host exec to full/off and the approval predicate does not prompt for the reported config or restart commands. I did not run the destructive live commands because they would intentionally mutate config and restart the gateway.

## Solution Assessment

Unclear until maintainers define the self-management command taxonomy and the intended exception to full/off host exec semantics. Prompt-only or example-comment-only mitigation is not durable enough; the maintainable fix should be deterministic exec approval or preflight policy with regression coverage.
