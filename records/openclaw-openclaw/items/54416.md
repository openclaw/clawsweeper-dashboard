## Summary

Keep open: current main and v2026.6.6 still allow raw gateway/node exec to run under full/off policy without a self-management confirmation class, while the existing prompt guidance and gateway config tool guard do not cover `openclaw config set` or `openclaw gateway restart` executed through shell.

## What This Changes

Issue reports that an agent executed example OpenClaw configuration commands and restarted the gateway without explicit user confirmation.

## Best Possible Solution

Define a core exec-boundary policy for OpenClaw self-management commands that requires explicit user-approved intent before model-initiated config writes, model/default changes, gateway lifecycle, update, uninstall, or similar commands run through raw shell execution.

## Reproduction Assessment

Yes, at source level. Current main still has a raw exec path where gateway/node full/off policy does not require confirmation for shell commands, while only prompt guidance discourages OpenClaw CLI lifecycle use without explicit request.

## Solution Assessment

No, the current mitigations are not enough. Prompt guidance and first-class gateway config guards are useful, but the best fix needs a hard raw-exec self-management boundary coordinated with the broader tool confirmation design at https://github.com/openclaw/openclaw/issues/23451.
