## Summary

Keep open. Current main and the latest release still allow raw gateway/node host exec under `security="full"` plus `ask="off"` without a deterministic self-management confirmation class for the reported config/model/default writes or gateway restart; prompt guidance and the narrow existing shell guards do not close this path.

## What This Changes

The issue asks OpenClaw to prevent agents from executing example model-switching config commands and gateway restarts without explicit user approval.

## Best Possible Solution

Define a deterministic exec approval or preflight boundary for OpenClaw self-management commands, covering config/model-default writes, gateway lifecycle, update, install/uninstall, and similar operations, with docs and regression coverage for how it composes with full/off host exec.

## Reproduction Assessment

Yes at source level: current main still has full/off raw host exec paths that do not prompt for `openclaw config set` or `openclaw gateway restart`. I did not run the destructive live commands because they would mutate configuration and restart the gateway.

## Solution Assessment

Unclear until maintainers define the self-management command taxonomy and the intended exception to full/off host exec semantics. Prompt-only or example-comment wording would be too soft; the durable fix belongs in deterministic exec approval/preflight policy plus tests.
