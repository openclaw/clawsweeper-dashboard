## Summary

Keep open: current main and the latest v2026.6.8 release still leave the reported raw shell path without a hard confirmation boundary for OpenClaw config/model/default writes or gateway restart commands under full/off exec policy.

## What This Changes

The issue asks OpenClaw to prevent agents from automatically executing example config, model-switch, and gateway restart commands without explicit user confirmation.

## Best Possible Solution

Define a hard exec-boundary policy for model-initiated OpenClaw self-management commands, coordinated with https://github.com/openclaw/openclaw/issues/23451 and preserving the first-class gateway config guard as the preferred safe path.

## Reproduction Assessment

Yes, at source level. Current main still allows raw gateway/node exec to skip approval under full/off policy, while the hard shell guard and first-class Gateway config guard do not cover shell `openclaw config set` and restart commands.

## Solution Assessment

Unclear until maintainers define the security policy. Prompt-only mitigation is too soft; the maintainable fix is a hard exec-boundary classification that composes with existing exec approvals and the broader tool confirmation design.
