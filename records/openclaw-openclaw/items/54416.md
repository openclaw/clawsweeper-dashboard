## Summary

Keep open. Current main has prompt guidance and a guarded first-class gateway config tool, but the raw exec path still does not enforce a hard confirmation boundary for model-issued OpenClaw self-management commands such as config writes, model changes, gateway restarts, or uninstall/update flows when host exec is in full/off mode.

## What This Changes

The issue requests that example OpenClaw config/model/gateway commands shown in an agent response not be executed automatically through exec without explicit user confirmation.

## Best Possible Solution

Add an exec-boundary policy for OpenClaw self-management commands so model-initiated config writes, model switching, gateway lifecycle, update, uninstall, reset, and similar operations require explicit user-approved intent even when raw exec otherwise runs in full/off mode. Keep the structured gateway tool as the preferred config/restart path and align the policy with #65307.
