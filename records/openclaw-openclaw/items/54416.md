## Summary

Keep open. Current main still leaves raw host exec in the documented full/off mode without a deterministic approval boundary for model-issued OpenClaw config writes, provider/default-model changes, or gateway restarts, while the issue has concrete logs and protected security/product labels that require maintainer direction.

## What This Changes

The issue asks OpenClaw to prevent an agent from automatically executing example OpenClaw config/model/gateway commands without explicit user confirmation.

## Best Possible Solution

Define and test a deterministic raw-exec approval or preflight boundary for OpenClaw self-management commands, covering config writes, provider/model changes, gateway lifecycle, update/uninstall operations, and how those exceptions compose with documented full/off host exec.

## Reproduction Assessment

Yes, at source level: current main still routes raw gateway/node exec through full/off policy without a self-management approval class for `openclaw config set` or `openclaw gateway restart`. I did not run the destructive live commands because they would intentionally mutate config and restart the gateway.

## Solution Assessment

Unclear until maintainers define the self-management command taxonomy and the compatibility contract for full/off exec. The durable solution should be deterministic approval or preflight policy with regression coverage, not only prompt wording around examples.
