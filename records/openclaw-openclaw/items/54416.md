## Summary

Keep open. Current main and the latest shipped tag still allow raw gateway/node host exec in `security="full"` plus `ask="off"` without a deterministic confirmation class for `openclaw config set`, model-default/provider writes, or `openclaw gateway restart`; the first-class gateway config guard does not cover the raw shell path, and the related self-uninstall report is still open rather than a merged fix.

## What This Changes

The issue reports model-initiated raw exec running example OpenClaw config and gateway restart commands without explicit user confirmation.

## Best Possible Solution

Define a deterministic raw-exec approval or preflight boundary for OpenClaw self-management commands, covering config writes, model/provider/default changes, gateway lifecycle, update, uninstall, and similar operations, then document how that exception composes with intentional full/off host exec.

## Reproduction Assessment

Yes at source level: current main still has a full/off raw host exec path that does not prompt for `openclaw config set` or `openclaw gateway restart`. I did not run the destructive live commands because they would intentionally mutate config and restart the gateway.

## Solution Assessment

Unclear until maintainers define the self-management command taxonomy and the intended exception to documented full/off semantics. Prompt-only or example-comment mitigation is not enough; the maintainable path is deterministic exec policy plus regression coverage.
