## Summary

Keep open. Current main and the latest release still allow the reported OpenClaw config/provider changes and gateway restart to flow through raw host exec in full/off mode without a deterministic self-management confirmation boundary.

## What This Changes

The issue reports that an agent can execute example OpenClaw config and gateway restart commands without explicit user confirmation.

## Best Possible Solution

Add a deterministic exec-boundary policy for OpenClaw self-management commands, covering config/provider/model writes, gateway lifecycle, updates, and self-removal while documenting how that exception composes with intentional full/off host automation.

## Reproduction Assessment

Yes, at source level: current main resolves non-sandbox host exec to full/off and the approval predicate does not prompt for the reported OpenClaw config or restart commands. I did not run the destructive live commands because they would intentionally mutate config and restart the gateway.

## Solution Assessment

No for the prompt-only or example-comment-only suggestions; they are not durable enforcement. The maintainable fix should be deterministic exec approval or preflight policy with regression coverage, after maintainers define the self-management command taxonomy.
