## Summary

Keep open. Current main and latest release still allow raw gateway/node host exec in full/off mode without a hard confirmation class for OpenClaw config writes, model-default changes, or gateway lifecycle commands; the newer prompt guidance and first-class gateway config guard do not close the raw `exec` path reported here.

## What This Changes

The issue reports that an agent executed example `openclaw config set` and `openclaw gateway restart` commands from a Feishu conversation without explicit user confirmation.

## Best Possible Solution

Define a deterministic raw-exec approval/preflight boundary for OpenClaw self-management commands, covering config writes, model-default changes, gateway lifecycle, updates, uninstall/self-removal, and similar operations, with docs and regression coverage that state how it composes with full/off host exec.

## Reproduction Assessment

Yes at source level: current main still has a full/off host exec path that does not prompt for raw `openclaw config set` or `openclaw gateway restart`. I did not run the destructive/mutating live commands in the review checkout.

## Solution Assessment

Unclear until maintainers define the self-management command taxonomy and YOLO-mode exception. Prompt-only or comment-only mitigation is not enough; the durable fix belongs in deterministic exec approval/preflight policy plus tests.
