## Summary

Keep open: current main and v2026.6.10 still leave raw host exec without a hard confirmation boundary for model-issued OpenClaw config/model-default writes or gateway restart commands under full/off policy.

## What This Changes

The issue asks OpenClaw to prevent agents from automatically executing example OpenClaw config, model-switch, and gateway restart commands without explicit user confirmation.

## Best Possible Solution

Define and implement a hard exec-boundary policy for model-initiated OpenClaw self-management commands, coordinated with the broader tool confirmation design and existing exec/plugin approval paths.

## Reproduction Assessment

Yes at source level: current main still allows raw gateway/node exec to run under documented `security=full` plus `ask=off`, and the current guard does not classify `openclaw config set` or `openclaw gateway restart` as confirmation-required. I did not run a live destructive Feishu repro because the code and tests prove the relevant path without mutating user config.

## Solution Assessment

Unclear until maintainers settle the policy. A hard command/tool approval boundary is the maintainable direction; a prompt-only instruction would not reliably stop same-response text plus tool-call execution.
