## Summary

Keep open. Current main still allows the reported OpenClaw config/default-model/gateway restart commands to run through raw host exec under the documented `security=full` plus `ask=off` policy, and the existing hard blocks cover narrower cases only.

## What This Changes

The issue asks OpenClaw to stop agents from automatically running example configuration and gateway restart commands without explicit user confirmation.

## Best Possible Solution

Define a deterministic exec approval or preflight boundary for OpenClaw self-management commands, covering config writes, provider/model/default changes, gateway lifecycle, update, and self-removal while documenting how that exception composes with intentional full/off host automation.

## Reproduction Assessment

Yes, at source level: current main resolves non-sandbox host exec to full/off and the approval predicate does not prompt for the reported OpenClaw config or restart commands. I did not run the destructive live commands because they would intentionally mutate config and restart the gateway.

## Solution Assessment

Unclear until maintainers define the self-management command taxonomy and the intended exception to full/off host exec semantics. Prompt-only or example-comment-only mitigation is not durable enough; the maintainable fix should be deterministic exec approval or preflight policy with regression coverage.
