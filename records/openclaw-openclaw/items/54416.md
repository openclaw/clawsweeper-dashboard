## Summary

Keep open. Current main and v2026.5.19 still implement and document raw gateway/node host exec with `security=full` plus `ask=off` and no deterministic confirmation class for model-issued OpenClaw config writes, provider/model changes, or gateway restarts; the first-class gateway config guard does not cover raw shell execution.

## What This Changes

The issue asks OpenClaw to require explicit confirmation before model-issued example commands mutate provider/model configuration or restart the gateway through raw exec.

## Best Possible Solution

Define a deterministic raw-exec approval or preflight boundary for OpenClaw self-management commands, covering config writes, model/default changes, provider credentials, gateway lifecycle, updates, and self-removal, then document how that exception composes with full/off host exec.

## Reproduction Assessment

Yes at source level: current main's raw gateway and node exec paths keep `security=full` plus `ask=off` without prompting for `openclaw config set` or `openclaw gateway restart`. I did not run the destructive live commands because they would intentionally mutate config and restart the gateway.

## Solution Assessment

Unclear as an unattended repair until maintainers define the self-management command taxonomy and full/off exception. Prompt-only or example-comment-only mitigation is not durable enough; the fix should be deterministic exec approval/preflight policy with regression coverage.
