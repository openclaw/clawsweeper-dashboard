## Summary

Keep open. Current main still documents and implements raw gateway/node exec with non-sandbox host defaults of `security=full` plus `ask=off`, and the approval path still lacks a deterministic confirmation class for model-issued OpenClaw config writes, provider/model changes, or gateway restarts.

## What This Changes

The issue requests a hard confirmation boundary before agent-issued raw exec runs OpenClaw config/model/provider changes or gateway restarts that were only shown as example commands.

## Best Possible Solution

Define a deterministic raw-exec approval or preflight boundary for OpenClaw self-management commands, covering config writes, model/default changes, provider credentials, gateway lifecycle, updates, and self-removal while documenting how that exception composes with full/off host exec.

## Reproduction Assessment

Yes, at source level: current main's raw gateway and node exec paths keep `security=full` plus `ask=off` without prompting for `openclaw config set` or `openclaw gateway restart`. I did not run the destructive live commands because they would intentionally mutate config and restart the gateway.

## Solution Assessment

Unclear until maintainers define the self-management command taxonomy and full/off exception. Prompt-only or example-comment-only mitigation is not durable enough; the fix should be deterministic exec approval or preflight policy with regression coverage.
