## Summary

Keep open. Current main and v2026.5.18 still document and implement full/off host exec with no self-management confirmation, and raw gateway/node exec still does not hard-prompt for OpenClaw config writes, model-default changes, or gateway restart.

## What This Changes

The issue requests preventing model-issued example commands from mutating OpenClaw configuration or restarting the gateway without explicit user confirmation.

## Best Possible Solution

Define a deterministic raw-exec approval or preflight boundary for OpenClaw self-management commands, covering config writes, model-default/provider changes, gateway lifecycle, update, uninstall, and similar operations, with docs and regression coverage for full/off host exec.

## Reproduction Assessment

Yes at source level: current main still has a full/off host exec path that does not prompt for raw `openclaw config set` or `openclaw gateway restart`. I did not run the destructive live commands.

## Solution Assessment

Unclear as an unattended implementation because the protected command taxonomy and full/off exception need maintainer/security policy. Prompt-only or comment-only mitigation would not be enough; the durable fix belongs in deterministic exec approval/preflight policy plus tests.
