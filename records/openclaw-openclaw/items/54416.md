## Summary

Keep open: current `main` and v2026.6.8 still have raw exec paths where `openclaw config set`, provider/model-default edits, and `openclaw gateway restart` are not covered by a hard self-management confirmation boundary under full/off policy. The broader tool confirmation issue is related, but this report preserves a concrete config/auth-provider reproduction that should stay visible until maintainers settle the security policy.

## What This Changes

The issue requests a guard so agent-produced example configuration commands are not automatically executed without explicit user confirmation.

## Best Possible Solution

Define a hard raw-exec self-management confirmation boundary for OpenClaw config writes, provider/model-default changes, gateway lifecycle commands, update, uninstall, and similar commands, coordinated with the broader confirmation-gate design.

## Reproduction Assessment

Yes, at source level. Current `main` still allows raw gateway/node exec to skip approval under full/off policy, while the hard shell guard and first-class Gateway config guard do not cover the reported shell `config set` and restart path.

## Solution Assessment

Unclear until maintainers define the security policy. Prompt-only mitigation is too soft; the maintainable fix is a hard exec-boundary classification that composes with existing exec approvals and the broader tool confirmation design.
