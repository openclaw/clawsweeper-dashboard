## Summary

Keep open. Current main has prompt guidance and a guarded first-class gateway config tool, but the raw `exec` path still lacks a hard confirmation boundary for model-issued OpenClaw config/model/gateway self-management commands in the documented `security=full`, `ask=off` host-exec mode.

## What This Changes

The issue asks OpenClaw to stop agents from executing example model/configuration commands, such as `openclaw config set` and `openclaw gateway restart`, without explicit user confirmation.

## Best Possible Solution

Add a hard raw-exec boundary for OpenClaw self-management commands while keeping the first-class gateway tool as the preferred config/restart surface and coordinating the command taxonomy with #65307 and #39528.

## Reproduction Assessment

Yes. The issue logs show the raw `exec` calls, and current source gives a high-confidence static reproduction path: gateway/node host exec defaults to `security=full`, `ask=off`, and the approval predicate does not prompt in that mode.

## Solution Assessment

No for prompt-only mitigation; it is useful guidance but not a sufficient security boundary. The maintainable fix belongs in raw exec approval/preflight policy, with a maintainer-approved command taxonomy and focused regression coverage.
