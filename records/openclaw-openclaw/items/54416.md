## Summary

Keep open. Current `main` and v2026.5.18 still allow raw gateway/node host exec in full/off mode without a deterministic confirmation class for OpenClaw config writes, model-default changes, provider credentials, or gateway lifecycle commands; the related denylist work and self-uninstall issue do not close this concrete config/restart gap.

## What This Changes

The issue asks OpenClaw to stop model-issued example shell commands from changing provider/model config or restarting the gateway without explicit user approval.

## Best Possible Solution

Define a deterministic raw-exec approval or preflight boundary for OpenClaw self-management commands, covering config writes, model/default changes, provider credentials, gateway lifecycle, updates, and uninstall/self-removal, then document how that exception composes with full/off host exec.

## Reproduction Assessment

Yes at source level: current main still has gateway and node full/off raw host exec paths that do not prompt for `openclaw config set` or `openclaw gateway restart`. I did not run the destructive live commands because they would intentionally mutate config and restart the gateway.

## Solution Assessment

Unclear until maintainers define the self-management command taxonomy and full/off exception. Prompt-only or example-comment-only mitigation is not enough; the durable fix belongs in deterministic exec approval/preflight policy plus tests.
