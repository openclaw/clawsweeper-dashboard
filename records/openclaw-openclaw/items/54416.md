## Summary

Keep open: current main and v2026.6.10 still leave the reported raw `exec` path without a hard confirmation boundary for model-issued OpenClaw config/model-default writes or gateway restart commands under full/off host exec policy.

## What This Changes

The issue reports that an agent can execute example `openclaw config set` and `openclaw gateway restart` commands without explicit user confirmation.

## Best Possible Solution

Define and implement a hard exec-boundary policy for model-initiated OpenClaw self-management commands, coordinated with the broader confirmation-gate design while preserving explicit trusted automation semantics.

## Reproduction Assessment

Yes, at source level. Current main still allows raw gateway exec to skip approval under full/off policy, while the hard shell guard and first-class approval predicate do not cover `openclaw config set` or `openclaw gateway restart`.

## Solution Assessment

Unclear until maintainers define the security policy. Prompt-only mitigation is too soft; the maintainable direction is a hard exec-boundary classifier that composes with existing exec approvals and the broader confirmation-gate design.
