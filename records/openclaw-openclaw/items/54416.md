## Summary

Keep open: current main and the latest release still have a raw exec path where gateway/node full/off policy does not require confirmation for `openclaw config set` or `openclaw gateway restart`; the existing hard guards cover narrower cases and the first-class gateway config tool only protects `gateway config.patch/apply`.

## What This Changes

The issue reports unintended execution of example `openclaw config set` and `openclaw gateway restart` commands through the agent exec path without user confirmation.

## Best Possible Solution

Define a hard exec-boundary policy for OpenClaw self-management commands that requires explicit user-approved intent before model-initiated config writes, provider/model-default changes, gateway lifecycle commands, update, uninstall, or similar commands can run through raw shell execution.

## Reproduction Assessment

Yes, at source level. Current main still has a raw exec path where gateway/node full/off policy does not require confirmation for shell commands, while only narrower guards block `/approve`, channel login, or first-class gateway config writes.

## Solution Assessment

No for the prompt-only alternatives; prompt guidance is soft. The maintainable fix needs a hard raw-exec self-management boundary coordinated with the broader confirmation-gate design at https://github.com/openclaw/openclaw/issues/23451.
