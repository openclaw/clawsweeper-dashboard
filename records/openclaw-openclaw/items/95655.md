## Summary

Keep open: this PR addresses a real Mattermost private-channel session split, but the patch is not merge-ready because the new `chatType` can become `group` while the route peer and resulting session key still stay in the `channel` namespace. It also lacks after-fix real behavior proof and does not add a regression test for the changed Mattermost route behavior.

## What This Changes

The PR adds a Mattermost `inferTargetChatType` hook, reorders shared outbound chat-type inference to consult plugin hooks before `channel:` defaults, and uses a monitor-populated cache to classify Mattermost private-channel deliveries as `group`.

## Best Possible Solution

Land a narrow Mattermost plugin fix that derives `peer.kind`, `chatType`, `from`, and session key from authoritative channel type or resolved target metadata, preserves public channels as `channel`, adds regression tests, and includes redacted real Mattermost proof.

## Reproduction Assessment

Yes for source-level reproduction: current main maps Mattermost type `P` to `group`, while current outbound session routing builds non-user Mattermost routes as `channel`. I did not run a live Mattermost workspace reproduction in this read-only review.

## Solution Assessment

No as submitted. The Mattermost plugin boundary is the right layer, but this patch changes `chatType` without aligning the route peer/session key and lacks focused regression plus real behavior proof.
