## Summary

Keep open: the patch still misses the documented raw `NODE_COMPILE_CACHE` root case, and the supplied proof exercises a copied helper rather than the patched launcher. Current main, docs, and the Node compile-cache contract make the root/scoped mismatch real, so this needs a code/test revision or maintainer cache-contract direction before merge.

## What This Changes

The PR adds an `isCompileCacheAlreadyAnchored` helper and uses it in `openclaw.mjs` to skip respawn when the active cache path is equal to or nested under the desired packaged cache directory.

## Best Possible Solution

Pick the cache contract in the launcher and tests: either support the documented stable root without repeated fresh-invocation respawns while preserving safe invalidation, or update docs/doctor to require a supported scoped cache path.

## Reproduction Assessment

Yes. The linked issue includes a current-main packaged-fixture two-launch repro, and current main still compares Node's raw-root versioned child with OpenClaw's scoped desired root while docs recommend a raw root.

## Solution Assessment

No. This patch is maintainable for an already-scoped cache path, but it does not solve the documented raw-root setup and does not include launcher regression coverage.
