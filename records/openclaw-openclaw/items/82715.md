## Summary

Keep open: the PR targets a real launcher bug, but this patch does not handle the fresh `NODE_COMPILE_CACHE` base path and the supplied proof only exercises a copied helper, not the changed launcher behavior.

## What This Changes

The PR adds a private containment helper in `openclaw.mjs` and uses it to skip packaged compile-cache respawn when the active cache path is equal to or nested under the desired root.

## Best Possible Solution

Revise the packaged launcher guard around the actual fresh env-enabled cache path, preserve package-scoped cache roots, and add focused launcher coverage showing repeated packaged runs do not set the packaged respawn marker.

## Reproduction Assessment

Yes for the review finding. Source inspection plus a local Node command show `NODE_COMPILE_CACHE=<base>` produces `<base>/v...`, while OpenClaw computes `<base>/openclaw/<version>/<marker>`, so the proposed helper still returns false for the documented fresh-invocation path.

## Solution Assessment

No. The containment helper only helps when the active cache is already below the computed scoped root; the reported fresh process remains outside that root and still respawns.

## Review Findings
