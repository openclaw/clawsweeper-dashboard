The branch changes legacy `src/agents` Doubao and BytePlus default cost constants to nonzero cache pricing and adds two agent tests asserting those constants stay nonzero.

## Best Possible Solution

Yes, as a source-level reproduction. Current main has zero cache prices in the active manifest rows, those rows flow through the manifest provider loader, and `estimateUsageCost` charges cached tokens from those exact fields.
