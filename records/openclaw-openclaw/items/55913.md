## Summary

Keep open: the underlying cache-pricing bug is still present on current main, but this PR as written targets deleted legacy `src/agents` provider files and would not update the active manifest-backed BytePlus or Volcengine catalogs.

## What This Changes

This PR changes legacy Doubao and BytePlus cost constants under `src/agents` and adds tests asserting nonzero `cacheRead` and `cacheWrite` defaults.

## Best Possible Solution

Retarget or replace the PR so the active extension manifests carry vendor-backed per-million or tiered cache pricing with extension-level regression coverage and real behavior proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes, as a source-level reproduction: current main has zero cache pricing in the active manifest rows, and `estimateUsageCost` charges cached-token usage from those fields.

## Solution Assessment

Is this the best way to solve the issue? No. The intent is correct, but the patch updates legacy files that are gone from current main and flattens pricing that should be modeled from the active provider manifests with vendor-backed tier/cache semantics.
