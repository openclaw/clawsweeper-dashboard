## Summary

Keep open: the Gemma sentinel-leak failure is plausible and worth fixing, but this branch is not merge-ready because it mixes an unrelated sensitive Telegram export, rewrites shared guarded-fetch URLs, mutates legitimate command literals, and is stale against current main's file-tool wrapper path.

## What This Changes

The PR adds sentinel-token sanitizers for exec, web_fetch/guarded fetch, and file/search tool arguments, plus a bundled portable-context Telegram command that exports USER.md, MEMORY.md, and recent-session context.

## Best Possible Solution

Land a narrow current-main sanitizer at explicit tool boundaries with prefix-only behavior, focused regression tests, and real proof; split portable context into a separate plugin/security review or ClawHub-style path.

## Reproduction Assessment

Yes for the source-level failure shape: current main forwards exec commands raw and validates web_fetch URLs with new URL, so sentinel-prefixed tool args fail before useful output. I did not run a live Gemma or Telegram reproduction in this read-only review.

## Solution Assessment

No. The safer solution is a narrow, prefix-only sanitizer at the active bash, web_fetch, and current file/search tool boundaries, with the portable context export handled separately after product/security review.
