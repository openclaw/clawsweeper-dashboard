## Summary

Keep open: the reported Gemma/sentinel-token failure is plausible, but this patch rewrites command and URL inputs too broadly, including inside the generic SSRF guard, and the external PR still lacks after-fix real behavior proof.

## What This Changes

The PR adds sanitizers for special-token leakage in `web_fetch` URLs and `exec` command arguments, wiring them into web fetch, guarded fetch, and bash exec with focused unit tests.

## Best Possible Solution

Keep the repair at the narrow tool/model-argument boundary, strip only well-scoped leading sentinel leakage with regression coverage for literal command/URL data, and preserve generic fetch guard validation semantics.

## Reproduction Assessment

Yes for the review findings from source inspection: the PR diff rewrites command and generic guarded-fetch inputs before existing validation/execution paths. I did not reproduce the reporter's Gemma deployment issue live.

## Solution Assessment

No: sanitizing leading model leakage can be a reasonable fix, but this implementation applies the repair too broadly and in the wrong network layer. A narrower sanitizer at the `web_fetch` and `exec` tool boundary is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Keep URL repair out of the generic SSRF guard:** `src/infra/net/fetch-guard.ts:171`
  - body: `fetchWithSsrFGuard` is a generic network security boundary exported beyond `web_fetch`. Sanitizing here means callers that currently pass malformed input like `prefix https://host` will fetch the embedded URL instead of failing URL validation, so keep this repair in the `web_fetch` tool boundary or make it an explicit caller opt-in.
  - confidence: 0.9
- **[P2] Only strip sentinel leakage where it cannot be literal command text:** `src/agents/tools/sanitize-command.ts:29-33`
  - body: This removes `<|...|>`, `<|`, and `|>` anywhere in the shell command. Commands such as `printf '%s\n' '<|im_start|>'` or `grep '<|foo|>' file` would no longer execute the requested literal text; limit repair to the observed leading leakage pattern before the command starts.
  - confidence: 0.86
- **[P3] Emit the promised web-fetch sanitize event:** `src/agents/tools/web-fetch.ts:507`
  - body: The PR body promises `webfetch.sanitize_special_tokens`, but both URL sanitizer call sites run without logging. Operators would have no monitoring signal when URL repair changes a tool input.
  - confidence: 0.78
