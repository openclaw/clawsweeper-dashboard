## Summary

Keep this PR open, but it should not merge as-is: the reported tool-argument failure is plausible from current source, while the patch still rewrites a shared SSRF primitive, mutates literal shell-token content, and lacks after-fix web_fetch proof.

## What This Changes

The PR adds sentinel-token sanitizers for leaked model output and wires them into bash exec, web_fetch URL handling, the shared SSRF fetch guard, and file-tool path arguments with unit coverage.

## Best Possible Solution

Keep sanitization at current LLM tool-argument boundaries, preserve strict shared SSRF validation, make command/path repair prefix-only, log URL repairs, and add focused regressions plus real web_fetch proof.

## Reproduction Assessment

Yes at source level: current main passes exec command text to runExecProcess and web_fetch rejects malformed URLs with new URL, so a leaked leading sentinel can produce the reported shell or URL failure. I did not establish a live Gemma reproduction in this review.

## Solution Assessment

No. Sanitizing malformed LLM tool arguments is a reasonable fix direction, but this patch applies URL repair inside a shared SSRF primitive and strips token-looking shell content beyond the malformed leading prefix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep URL sanitizing out of the shared fetch guard:** `src/infra/net/fetch-guard.ts:171`
  - body: `fetchWithSsrFGuard` is an SDK-facing shared primitive that accepts caller-supplied `init`; sanitizing `params.url` here can turn malformed input with an embedded `http(s)` scheme into a real outbound request carrying headers or body. Keep URL repair at the `web_fetch` tool boundary and leave the guard strict.
  - confidence: 0.88
- **[P2] Preserve literal sentinel tokens in commands:** `src/agents/tools/sanitize-command.ts:27-47`
  - body: These replacements remove token-looking text anywhere in the command, so a legitimate command such as `rg '<|assistant|>' docs` is rewritten to search for different text. Restrict repair to malformed leading leakage and preserve quoted or literal command content; the file path sanitizer should not inherit global shell rewrites either.
  - confidence: 0.84
- **[P3] Log URL sanitizer changes as promised:** `src/agents/tools/web-fetch.ts:507`
  - body: The PR body promises `webfetch.sanitize_special_tokens`, but `runWebFetch` silently mutates `params.url`. If the URL sanitizer remains, emit the monitoring event only when the URL actually changes.
  - confidence: 0.78
