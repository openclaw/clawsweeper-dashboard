## Best Possible Solution

Keep the sanitizer fix narrow at model-facing tool boundaries, preserve shared SSRF validation semantics, rebase path handling onto current wrappers, and split portable-context into a separate reviewed plugin or ClawHub path.

## Solution Assessment

No. The maintainable path is a narrow sanitizer at model-facing tool boundaries with prefix-scoped semantics; the shared guarded-fetch rewrite and bundled context export should not ride along with this bug fix.

Full review comments:

- **[P1] Keep URL cleanup out of guarded fetch:** `src/infra/net/fetch-guard.ts:171`
  - body: `fetchWithSsrFGuard` is shared SSRF-protected fetch infrastructure, not just the model-supplied `web_fetch` tool. Sanitizing here can convert previously invalid input into a real request with the caller's existing init and headers, so URL cleanup should stay at the web_fetch tool boundary.
  - confidence: 0.9
- **[P1] Do not rewrite shell command contents globally:** `src/agents/tools/sanitize-command.ts:27-47`
  - body: The sanitizer removes `<|...|>`, `<<|...|`, and `|>` across the whole command, including quoted grep patterns, prompt-snapshot inspections, or literal output text. Exec should only strip a bounded leaked prefix before shell parsing; otherwise valid commands can run with different contents.
  - confidence: 0.88
- **[P1] Remove the bundled context export from this fix:** `extensions/portable-context/index.ts:54-64`
  - body: This adds a new bundled `/export-context` command that reads hardcoded Gemma/Luna memory and session files, calls a local vLLM endpoint, and returns a Telegram document. That is an unrelated, security-sensitive feature and needs a separate product/security review or plugin path before it belongs in core.
  - confidence: 0.92

  - body: The file-tool patch is wired through `normalizeToolParams`, but current main no longer has that chokepoint; read/write/edit now route through the current validation and read wrappers. As-is, the claimed file path coverage cannot apply cleanly to current main.
