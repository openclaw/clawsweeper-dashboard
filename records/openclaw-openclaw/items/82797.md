## Label Justifications

## Summary

Keep open: the reported sentinel-token tool failure is plausible and current main does not sanitize these tool arguments, but this PR is not merge-ready because it rewrites command/URL inputs too broadly, targets a stale file-tool chokepoint, and bundles an unrelated security-sensitive Telegram export command.

## What This Changes

The PR adds sanitizer helpers for bash exec, web_fetch URLs, and file-tool path args, wires URL cleanup into web_fetch and the shared SSRF guard, and adds a bundled portable-context `/export-context` Telegram command plugin.

## Best Possible Solution

Keep the sanitizer fix narrow and tool-boundary scoped, make command/path stripping prefix-only and quote-aware, keep shared SSRF behavior unchanged, and split portable-context into a separate reviewed plugin or ClawHub path.

## Reproduction Assessment

Yes at source level, but not as a live Gemma reproduction in this review. Current main passes raw exec commands to bash and validates raw web_fetch URLs, while the PR body supplies redacted live logs showing observed sentinel leaks.

## Solution Assessment

No. The maintainable fix is a narrow sanitizer at model-facing tool boundaries with quote-aware/prefix-only semantics; the bundled context export and shared guarded-fetch rewrite should not ride along with this bug fix.

- **[P1] Keep URL cleanup out of guarded fetch:** `src/infra/net/fetch-guard.ts:171`
  - body: `fetchWithSsrFGuard` is the shared SSRF-protected fetch path for plugins/providers, not just model-supplied `web_fetch` args.   - body: The sanitizer removes `<|...|>`, `<<|...|`, and `|>` across the whole command, including quoted grep patterns, prompt-snapshot inspections, or literal output text. Exec should only strip a bounded leaked prefix before shell parsing; otherwise valid commands run with different contents.
  - confidence: 0.88
- **[P1] Remove the bundled context export from this fix:** `extensions/portable-context/index.ts:54-64`
  - body: This adds a new bundled `/export-context` command that reads hardcoded Gemma/Luna memory and session files, calls a local vLLM endpoint, and returns a Telegram document. That is an unrelated, security-sensitive feature and needs a separate product/security review or plugin path before it belongs in core.
  - confidence: 0.92
