## Summary

Keep open. The reported sentinel-token failure is plausible, but this PR is not merge-ready because it broadens the fix into shared network security code, globally rewrites shell text, adds an unrelated sensitive Telegram export command, and still lacks complete real behavior proof.

## What This Changes

The branch adds sentinel-token sanitizers for exec, web_fetch, guarded fetch, and file-tool path args, plus a bundled Telegram `/export_context` portable-context command.

## Best Possible Solution

Keep the sanitizer narrowly scoped to model-facing tool boundaries, make command/path cleanup prefix-only and quote-aware, leave shared SSRF behavior unchanged, and split `/export_context` into a separate reviewed plugin or ClawHub path.

## Reproduction Assessment

Yes at source level, but not as a live Gemma reproduction in this review. Current main passes raw exec commands to bash and validates raw web_fetch URLs, while the PR body supplies redacted live logs showing observed sentinel leaks.

## Solution Assessment

No. The maintainable fix is a narrow sanitizer at model-facing tool boundaries with prefix-only semantics; the bundled context export and shared guarded-fetch rewrite should not ride along with this bug fix.
