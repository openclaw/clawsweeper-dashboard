## Summary

Keep open: the reported smaller-model sentinel leakage is plausible, but this patch has blocking correctness and security issues in the shared network guard and shell-command sanitizer, and the external PR still lacks inspectable after-fix real behavior proof.

## What This Changes

The PR adds URL and bash command sanitizers for leaked `<|...|>`-style model tokens and wires them into web_fetch, fetchWithSsrFGuard, and bash exec with unit tests.

## Best Possible Solution

Keep the repair at the LLM tool-argument boundary, make it leading-prefix-only, leave fetchWithSsrFGuard strict, add focused regression coverage, and require redacted real-run output or logs before merge.

## Reproduction Assessment

Yes at source level for the malformed-argument paths: current main forwards exec commands to the shell path and validates web_fetch URLs with new URL, so leading sentinel text can cause the reported bash or URL failure. I did not reproduce a live Gemma session.

## Solution Assessment

No. Sanitizing LLM arguments is a plausible direction, but this implementation rewrites the shared guarded-fetch primitive and strips token-like substrings throughout shell commands instead of only repairing malformed leading leakage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Keep URL sanitizing out of the shared fetch guard:** `src/infra/net/fetch-guard.ts:171`
  - body: `fetchWithSsrFGuard` is exported through SDK-facing paths and is used by providers/plugins with arbitrary `init` headers and bodies. With this change, an invalid caller value like `junkhttps://attacker.example` becomes a real request instead of failing validation, so the caller's request init can be sent to a URL embedded in bad input. Keep this repair at the `web_fetch` LLM-argument boundary and leave the generic guard strict.
  - confidence: 0.88
- **[P2] Do not strip sentinel tokens from the whole shell command:** `src/agents/tools/sanitize-command.ts:27-32`
  - body: These regexes remove token-looking text anywhere in the command string. A legitimate command such as `rg '<|assistant|>' docs` becomes `rg '' docs`, which changes the requested operation and can produce much broader output. Limit repair to the malformed leading prefix, or otherwise avoid modifying quoted/literal command content.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The diff introduces a network-boundary regression by rewriting invalid guarded-fetch URLs into real outbound requests with caller-provided request init.

Concerns:

- **[medium] Shared fetch guard can redirect malformed input to a real URL:** `src/infra/net/fetch-guard.ts:171`
  - body: Because sanitizeUrlInput takes the first embedded http(s) scheme and fetchWithSsrFGuard carries caller-provided headers/body into the request, malformed external input can become an outbound request to a different destination instead of being rejected.
