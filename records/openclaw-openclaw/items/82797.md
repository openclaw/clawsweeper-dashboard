## Summary

Keep open: the reported smaller-model tool-argument failure is plausible, but the current head has blocking correctness and security issues, is not aligned with current main's file-tool parameter path, and does not prove all changed real behavior.

## What This Changes

The branch adds LLM sentinel-token sanitizers for bash exec, web-fetch, and file-tool inputs, and also adds a bundled portable-context `/export` command plugin.

## Best Possible Solution

Keep repair at LLM tool-argument boundaries, preserve the shared SSRF guard's strict validation, rebase file-tool handling onto current main, and split or drop the portable-context command unless maintainers explicitly approve it as a separate plugin feature.

## Reproduction Assessment

Yes at source level: current main passes exec command text to shell execution and rejects malformed web_fetch URLs with `new URL`, so leaked leading sentinels can cause the reported syntax or URL failures. I did not establish a live Gemma reproduction in this read-only review.

## Solution Assessment

No. Sanitizing leaked LLM tool arguments is a reasonable direction, but this implementation crosses into the shared SSRF guard, rewrites literal command/path content, targets a stale file-param path, and mixes in an unrelated Telegram-visible plugin feature.
