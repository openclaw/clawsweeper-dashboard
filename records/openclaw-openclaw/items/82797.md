## Summary

Keep open: the source-level sanitizer bug is plausible and useful, but this PR is not merge-ready because it mutates literal exec input, broadens shared guarded-fetch URL handling, and includes unrelated Telegram data-export, loop-guard, payload, and compaction behavior changes that need splitting and proof.

## What This Changes

The branch adds sentinel-token sanitizers for exec, web_fetch, file/search tool args, changes loop detection, payload fallback, and compaction behavior, and adds a bundled Telegram `/export_context` portable-context plugin.

## Best Possible Solution

Split this into a narrow boundary-local sanitizer fix that preserves literal command/path content and keeps shared guarded fetch strict; review portable-context, loop-guard, payload, and compaction changes separately with owner approval and real proof.

## Reproduction Assessment

Yes for the source-level failure shape: current main passes exec commands raw and strictly rejects malformed guarded-fetch URLs. I did not establish a live Gemma/Telegram reproduction in this read-only review.

## Solution Assessment

No. The best fix is a narrow sanitizer at tool-input boundaries with false-positive guards; the current branch mixes that fix with unrelated data export, loop/default, and stale-path changes.

Overall confidence: 0.9

Full review comments:

- **[P1] Constrain command sanitizing to leaked prefixes:** `src/agents/tools/sanitize-command.ts:29`
  - body: `sanitizeCommandInput` removes every `<|...|>` sequence anywhere in the command. That turns an intentional command such as `grep '<|im_start|>' tokenizer.txt` into a different command, so exec can no longer faithfully run user/model-supplied shell text.
  - confidence: 0.9
- **[P1] Keep URL recovery out of shared guarded fetch:** `src/infra/net/fetch-guard.ts:171`
  - body: Sanitizing inside `fetchWithSsrFGuard` changes every SSRF-guarded caller, not only `web_fetch`. A malformed input containing an embedded `http(s)://` substring can become a real outbound request with the caller's headers, proxy, and policy, where current main would fail closed.
  - confidence: 0.88
- **[P1] Remove the context export command from this sanitizer PR:** `extensions/portable-context/index.ts:221-224`
  - body: This new bundled command reads USER.md, MEMORY.md, recent session JSONL, calls a local vLLM endpoint, and sends a CONTEXT.md document through Telegram. That is a new private-data export surface and should be split for explicit product/security review and Telegram proof.
  - confidence: 0.93
- **[P2] Require no-progress evidence before blocking repeats:** `src/agents/tool-loop-detection.ts:593-597`
  - body: The new generic repeat block fires from consecutive identical tool+args history only, but the error text claims prior attempts returned the same result. In loop-detection-enabled setups this can block legitimate repeated calls even when results are changing or progress was not checked.
  - confidence: 0.78
