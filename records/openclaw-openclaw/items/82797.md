## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep sentinel URL cleanup out of the SSRF guard:** `src/infra/net/fetch-guard.ts:174`
  - body: `fetchWithSsrFGuard` is the shared fail-closed network boundary for provider setup, plugin downloads, media/provider calls, and web tools. Sanitizing `params.url` here can turn malformed caller input such as a prefixed string containing `https://...` into a real outbound request with the caller's existing init, method, and headers, instead of rejecting it as invalid. Apply this only at the `web_fetch` tool boundary or behind an explicit caller option.
  - confidence: 0.91
- **[P1] Preserve literal token text inside shell commands:** `src/agents/tools/sanitize-command.ts:27-47`
  - body: The sanitizer removes `<|...|>` and `|>` globally before bash runs, so valid commands that inspect prompt snapshots, grep tokenizer markers, print sentinel literals, or refer to matching filenames are silently changed. Limit stripping to the observed leading sentinel prefix or return a clear retryable tool error when the input is ambiguous.
  - confidence: 0.88
- **[P1] Rebase file-path sanitizing onto the current wrappers:** `src/agents/pi-tools.params.ts:119`
  - body: This branch sanitizes file-tool paths through `normalizeToolParams`, but current main removed that chokepoint and now validates/executes read/write/edit through `pi-tools.read.ts` wrappers. After rebase, the claimed read/write/edit path coverage either conflicts or misses the live execution path, so the sanitizer needs to be moved to the current wrapper/record boundary and retested there.
  - confidence: 0.87
- **[P1] Split portable-context from this sanitizer fix:** `extensions/portable-context/index.ts:157-160`
  - body: A sanitizer PR should not also add a bundled Telegram command that reads a fixed `gemma` workspace/session, calls a hardcoded local model endpoint, and returns a context document over `mediaUrl`. That is a separate sensitive product/plugin change and should be removed from this PR or reviewed independently through the plugin/ClawHub path.
  - confidence: 0.93
