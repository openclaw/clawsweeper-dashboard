Keep open. The sentinel-leak symptom is plausible and partly proven, but this branch has grown into broad agent/runtime work with concrete security, compatibility, and availability blockers, and GitHub reports it is currently dirty against main.

## What This Changes

Adds sanitizer logic for tool commands, URLs, file/search/tool-call arguments, plus loop/compaction/fallback changes, a bundled Telegram context export command plugin, and an operator PM2 launcher.

## Best Possible Solution

Split this into a narrow direct tool-boundary sanitizer for exec, web_fetch, and path-like file inputs, then review private context export, hallucination correction, loop blocking, compaction, and deployment config as separate owner-approved work.

## Solution Assessment

No for the full branch: the narrow sanitizer may be maintainable, but the PR mixes it with unrelated private-data export, shared fetch behavior, loop policy, compaction, and operator deployment changes that need separate review.
