## Summary

Keep open: the reported sentinel-token failure is plausible and worth fixing, but this PR is not merge-ready because it broadens the fix into shared fetch behavior, rewrites shell contents globally, adds an unrelated sensitive Telegram export command, lacks complete after-fix proof, and is currently conflicting with main.

## What This Changes

The PR adds sanitizer helpers for bash exec commands, web_fetch URLs, and file-tool path arguments, wires URL cleanup into web_fetch and the shared SSRF guard, and adds a bundled portable-context Telegram export command.

## Best Possible Solution

Keep the sanitizer fix narrow at model-facing tool boundaries, make command/path cleanup prefix-scoped and quote-aware, leave shared SSRF behavior unchanged, and split portable-context into a separate reviewed plugin or ClawHub path.

## Reproduction Assessment

Yes, at source level: current main passes raw exec commands to bash and validates raw web_fetch URLs, while the PR body supplies redacted live logs for observed Gemma sentinel leaks. I did not run a live Gemma/Telegram reproduction in this read-only review.

## Solution Assessment

No. The best fix is a narrower sanitizer at model-facing tool boundaries; the shared SSRF guard mutation, global shell-content rewrite, stale file-tool chokepoint, and bundled context export should not ride along with this bug fix.

Full review comments:

- **[P1] Keep URL cleanup out of guarded fetch:** `src/infra/net/fetch-guard.ts:174`
  - body: fetchWithSsrFGuard is shared infra for plugins/providers, not only model-supplied web_fetch arguments. Sanitizing here can convert previously invalid input into a real request with the caller's existing init and headers, so URL cleanup should stay at the web_fetch tool boundary.
  - confidence: 0.9
- **[P1] Do not rewrite shell command contents globally:** `src/agents/tools/sanitize-command.ts:27-47`
  - body: The sanitizer removes <|...|>, <<|...|, and |> across the whole command, including quoted grep patterns, prompt-snapshot inspections, or literal output text. Exec should only strip a bounded leaked prefix before shell parsing; otherwise valid commands can run with different contents.
  - confidence: 0.89
- **[P1] Remove the bundled context export from this fix:** `extensions/portable-context/index.ts:223-224`
  - body: This adds a new bundled export_context command that reads hardcoded Gemma/Luna memory and session files, calls a local vLLM endpoint, and returns a Telegram document. That is an unrelated, security-sensitive feature needing separate product/security review or a plugin path.
  - confidence: 0.93
- **[P1] Rebase path sanitizing onto current wrappers:** `src/agents/pi-tools.params.ts:115-119`
  - body: The file-tool patch is wired through normalizeToolParams, but current main no longer has that chokepoint; read/write/edit now route through the current validation wrappers. As written, the claimed file path coverage will not apply cleanly to current main.
  - confidence: 0.86
