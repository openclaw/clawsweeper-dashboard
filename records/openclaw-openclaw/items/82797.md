## Summary

Keep this PR open: the reported tool-argument failure is plausible and source-reproducible, but the current patch is not mergeable because it widens the shared SSRF guard, rewrites legitimate shell literals, and still lacks after-fix proof for the web_fetch URL path.

## What This Changes

The PR adds sentinel-token sanitizers for bash exec commands and web_fetch URLs, wires them into bash exec, web_fetch, and the shared SSRF fetch guard, and adds sanitizer unit tests.

## Best Possible Solution

Keep sanitization at the LLM tool-argument boundaries, restrict both sanitizers to leading sentinel leakage, leave fetchWithSsrFGuard strict, log changed URL inputs, and add regressions for invalid guard URLs plus literal shell-token content.

## Reproduction Assessment

Yes at source level: current main passes bash exec commands into runExecProcess and validates web_fetch URLs with new URL before fetching, so leading sentinel text can cause the reported shell or URL failure. I did not establish a live Gemma reproduction in this review.

## Solution Assessment

No. Sanitizing malformed LLM tool arguments is a reasonable direction, but this patch applies URL rewriting inside a shared SSRF primitive and strips token-looking shell content beyond the malformed leading prefix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep URL sanitizing out of the shared fetch guard:** `src/infra/net/fetch-guard.ts:171`
  - body: fetchWithSsrFGuard is an SDK-facing network primitive with caller-supplied init. Sanitizing params.url here can turn malformed input with an embedded http(s) scheme into a real outbound request instead of failing validation, so keep this repair at the web_fetch tool boundary and leave the shared guard strict.
  - confidence: 0.88
- **[P2] Limit command sanitizing to leading leaked prefixes:** `src/agents/tools/sanitize-command.ts:27-32`
  - body: These replacements remove token-looking text anywhere in the shell command. A legitimate command such as `rg '<|assistant|>' docs` would be changed to search for an empty string, producing different and potentially much broader output.
  - confidence: 0.84
- **[P3] Log URL sanitizer changes as promised:** `src/agents/tools/web-fetch.ts:507`
  - body: The PR body says URL changes emit `webfetch.sanitize_special_tokens`, but the web_fetch path mutates params.url silently. If URL sanitizing remains, log only when the value actually changes so operators can monitor the leak signal.
  - confidence: 0.78
