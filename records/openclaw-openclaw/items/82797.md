## What This Changes

The PR adds sentinel-token sanitization for exec, web-fetch, file/search/tool-call arguments and also adds portable-context Telegram export, loop/hallucination guards, compaction/fallback changes, and an operator PM2 wrapper.

## Best Possible Solution

Split this into a narrow exec/web-fetch/path-like sanitizer with preservation tests, and move Telegram export, hallucination/loop/compaction policy, and operator deployment files into separately reviewed work with privacy and upgrade proof.

## Reproduction Assessment

Yes for the source-level failure shape: current main sends exec commands, web-fetch URLs, and file paths through the existing validators without this sentinel-strip step. The live Gemma leakage itself is still contributor-supplied runtime proof rather than independently reproduced here.

## Solution Assessment

No. A narrow sanitizer for command/url/path-like fields is plausible, but the current branch is not the best solution because it bundles unrelated Telegram export, operator deployment, compaction, loop, and hallucination policy changes.
