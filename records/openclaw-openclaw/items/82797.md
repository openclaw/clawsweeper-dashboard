## Best Possible Solution

Land a narrow rebased sanitizer that only normalizes LLM-owned tool inputs, and split the portable-context export into a separate plugin/ClawHub or product-security discussion.

## Solution Assessment

Is this the best way to solve the issue? No. The sanitizer direction is plausible, but the branch should be narrowed and rebased instead of adding a sensitive Telegram export feature and changing shared SSRF guard semantics.
