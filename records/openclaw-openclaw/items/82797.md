## Summary

Keep open for maintainer review: the sentinel-token failure is plausible and source-reproducible, but this PR is not merge-ready because it broadens shared network behavior, rewrites exec input semantics, and bundles a sensitive Telegram context-export feature.

## What This Changes

The PR adds sentinel-token sanitizers for exec, web_fetch/guarded fetch, file/search tool arguments, and an unrelated bundled portable-context Telegram command that exports USER.md, MEMORY.md, and recent context.

## Best Possible Solution

Land a narrow sanitizer-only change at the model-facing tool ingress points with prefix-only semantics, current-main tests, and live proof; split the context export into a separate ClawHub/plugin security review.

## Reproduction Assessment

Yes for the source-level failure shape: current main forwards exec commands raw and validates web_fetch URLs with new URL, so sentinel-prefixed tool args fail before useful output. I did not independently run a live Gemma or Telegram reproduction.

## Solution Assessment

No. The useful sanitizer should be a narrow tool-ingress fix, while shared guarded-fetch cleanup and bundled private context export need separate maintainer decisions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Constrain command cleanup to leaked prefixes:** `src/agents/tools/sanitize-command.ts:29`
  - body: This replacement runs over the whole shell string, so a valid command such as `printf '<|im_start|>' > token.txt` is changed before bash sees it. The fix is meant to recover malformed tool-call prefixes; keep cleanup to the leading leaked prefix or another unambiguous argument boundary.
  - confidence: 0.86
- **[P1] Keep sanitizer out of shared guarded fetch:** `src/infra/net/fetch-guard.ts:174`
  - body: `fetchWithSsrFGuard` is used by provider transports, media downloads, auth/API clients, downloads, plugins, and web tools. Sanitizing here can turn an invalid sentinel-prefixed URL into a real outbound request while preserving caller headers, proxy mode, and SSRF policy; apply URL recovery at the `web_fetch` argument boundary instead.
  - confidence: 0.9
- **[P1] Split the context export out of this PR:** `extensions/portable-context/index.ts:221-223`
  - body: This registers `/export_context` as a bundled command that reads USER.md, MEMORY.md, and recent session JSONL from hard-coded local agent paths and returns a Telegram document. That is a separate sensitive data-export feature with auth/product review needs, so it should not ship inside a sanitizer fix.
  - confidence: 0.92
