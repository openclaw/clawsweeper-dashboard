Full review comments:

- **[P1] Keep fresh-image pruning state out of module scope:** `src/agents/pi-embedded-runner/run/history-image-prune.ts:176-180`
  - body: `activeTurnHasFreshImages` is shared by every attempt in this Node process, but provider replay reads it after another attempt may have overwritten it. That can make one session prune all recent images because another session had an image, or leave stale images in an image session because a later no-image turn reset the flag. Scope this decision to the installed transform or attempt instead.
  - confidence: 0.88
