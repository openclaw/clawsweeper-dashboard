## Summary

Keep this PR open. The follow-up commit moves the fix in the right direction and addresses some discussion points, but the provider replay path still depends on process-global fresh-image state, and the new export is not reflected in the shared attempt test mock.

## What This Changes

The PR adds a history-image pruning override, uses aggressive pruning for fresh-image turns in prompt-build and provider replay paths, adds regression coverage, and updates the changelog.

## Best Possible Solution

Keep the aggressive pruning behavior, but bind the fresh-image decision to the current attempt or installed transform instance, update affected tests/mocks, and place the changelog note in the standard Unreleased Fixes section.

## Reproduction Assessment

Yes. Source inspection shows current main preserves the last three completed image turns, and the PR head still has source-reproducible concurrency and test-mock failures.

## Solution Assessment

No. The pruning direction is narrow and maintainable, but the replay path should receive attempt-local state rather than reading shared module state, and the test mock/changelog need repair before merge.
