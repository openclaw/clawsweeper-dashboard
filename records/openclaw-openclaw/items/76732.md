## Summary

Keep open: the PR targets a valid stale-image contamination bug, but the diff only applies aggressive pruning to before_prompt_build hook messages while the model replay transform still uses the default three-turn image preserve window, so recent history images can still reach the model.

## What This Changes

The PR adds a configurable history-image prune window, passes zero preserved turns when the current attempt has fresh images, and adds a helper-level pruning test.

## Best Possible Solution

Propagate the fresh-attachment signal into the model replay pruning path, preserve the conservative three-turn default for non-image turns, and cover that production path with regression tests plus the required changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: submit a fresh image in a session whose recent completed turns still contain image blocks, and the current transform used for model replay continues preserving those recent history images.

## Solution Assessment

No. The proposed helper option is useful, but applying it only to hook prompt-build messages is not the narrow complete fix because activeSession.prompt still relies on the default provider replay transform.
