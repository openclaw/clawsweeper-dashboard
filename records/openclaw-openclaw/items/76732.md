## Summary

Keep open. The PR targets a valid current-main stale-image bug, but the latest head still uses process-wide fresh-image state for provider replay, making the privacy-sensitive cleanup unreliable when embedded attempts overlap.

## What This Changes

The PR adds an aggressive history-image pruning option for fresh-image turns, threads it through prompt-build and provider replay paths, adds focused coverage, and adds a changelog entry.

## Best Possible Solution

Keep the aggressive pruning behavior, but pass the fresh-image decision through attempt-local replay transform state while preserving the raw transcript and the current three-turn default for no-image turns.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main preserves the last three completed image turns, and the PR head’s shared replay flag can be changed by another overlapping attempt before `activeSession.prompt(...)` triggers provider replay.

## Solution Assessment

No. The requested direction is narrow and maintainable, but provider replay needs attempt-local state instead of a process-wide flag, and the changelog entry should move under the existing `Fixes` section.
