## Summary

Keep open: the PR targets a real multimodal-history contamination bug from #66702, but the diff is incomplete and the added test is currently written against the wrong API behavior.

## What This Changes

This PR adds an optional recent-turn preservation override for history-image pruning, passes it when the current turn has images, and adds a focused pruning test.

## Best Possible Solution

Thread fresh-image state into the provider replay pruning path used by activeSession.prompt, keep the default three-turn preservation when no fresh images are present, fix the regression test to inspect the returned replay view, and add the required changelog entry.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: a recent completed image turn remains preserved by the installed replay transform even when the current turn has fresh images. The added test defect is also source-reproducible because the helper returns a cloned array or null rather than mutating input or returning a boolean.

## Solution Assessment

No. The direction is right, but applying the override only to before_prompt_build hook messages does not update the actual provider replay view, and the new test needs to assert the returned pruned messages.
