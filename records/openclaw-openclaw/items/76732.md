## Summary

Keep this PR open because it targets a valid multimodal history-contamination bug, but the current patch does not fix the provider replay path used for the actual model prompt and its new test asserts the helper contract incorrectly.

## What This Changes

The PR adds an optional recent-turn preservation override to history-image pruning, applies it to before_prompt_build hook messages when the current turn has images, and adds a pruning unit test.

## Best Possible Solution

Thread fresh-image state into the provider replay transform used by activeSession.prompt, keep the default three-turn preservation when no fresh images are present, assert the returned pruned clone in regression coverage, and add the active changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main shows recent completed image turns remain preserved by default, and the transform used by activeSession.prompt still uses that default when a fresh-image turn is submitted.

## Solution Assessment

No. The direction is right, but the patch only applies aggressive pruning to hook-visible messages and does not update the actual provider replay view; the test also checks the wrong return/mutation contract.
