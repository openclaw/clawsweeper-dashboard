## Summary

Keep open. Current main still fast-exits recurring `wakeMode: "now"` main-session cron jobs on retryable busy heartbeat skips and records `ok`, while the linked payload fix and broader ghost-run diagnostics do not cover this specific skipped-heartbeat path.

## What This Changes

Ensure recurring `wakeMode: "now"` main-session cron jobs do not log successful ghost runs when the first heartbeat attempt is skipped for retryable busy reasons.

## Best Possible Solution

Retry recurring wake-now main cron jobs for temporary `requests-in-flight` and `lanes-busy` skips within the existing bounded wait, keep `cron-in-progress` deferral, and update regression coverage plus changelog.
