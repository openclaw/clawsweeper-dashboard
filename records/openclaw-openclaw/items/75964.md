## Summary

Keep open. Current main still fast-exits recurring `wakeMode: "now"` main-session cron jobs on retryable busy heartbeat skips and records `ok`, while #73189 only covers payload delivery after a heartbeat actually runs and #63106 tracks broader ghost-run diagnostics.

## What This Changes

Recurring main-session `wakeMode: "now"` cron jobs should retry temporary busy heartbeat skips instead of reporting success before the queued system event is processed.

## Best Possible Solution

Retry recurring wake-now main cron jobs for temporary `requests-in-flight` and `lanes-busy` skips within the existing bounded wait, keep the `cron-in-progress` deferral, and update regression coverage plus changelog.

Preserve the `cron-in-progress` immediate deferral because the cron job's own active marker blocks direct wake-now. Do not broaden this into generic ghost-run warning work from #63106 or change unrelated heartbeat scheduler semantics.
