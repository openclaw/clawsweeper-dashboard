## Summary

Keep open: this is a useful missing CLI inspection feature, but the branch is not mergeable as-is because it still fails on removed core channel imports, duplicates runtime prompt assembly, and lacks contributor real behavior proof.

## What This Changes

The PR adds an `openclaw agents view-system-prompt [agentId]` subcommand with `--model` and `--channel` options and stderr prompt metadata.

## Best Possible Solution

Implement prompt inspection through a shared prompt-preview helper that reuses runtime prompt assembly and plugin prompt APIs, with documented live-only omissions and focused CLI coverage.

## Reproduction Assessment

Yes for the PR findings: CI reproduces the unresolved imports, and source inspection shows the preview path diverges from current runtime prompt assembly. This is not a current-main bug reproduction because main lacks the requested feature.

## Solution Assessment

No. The current branch duplicates prompt assembly in the CLI command; the maintainable direction is to share a runtime prompt-preview path and route channel-specific behavior through plugin APIs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Route channel prompt helpers through plugin APIs:** `src/commands/agents.commands.view-system-prompt.ts:31-33`
  - body: These imports target core `src/signal` and `src/telegram` modules that are absent on current main, and CI fails with TS2307 for all three imports. Use the channel plugin prompt APIs instead of hard-coding removed bundled channel internals from the core CLI command.
  - confidence: 0.98
- **[P2] Use runtime capability collection for channel previews:** `src/commands/agents.commands.view-system-prompt.ts:78-99`
  - body: This path combines raw channel capabilities with a Telegram special case, so it misses plugin prompt capabilities and thread-bound spawn capabilities that real runs include via `collectRuntimeChannelCapabilities`. `--channel` can therefore print guidance that does not match an actual agent turn.
  - confidence: 0.92
- **[P2] Build prompt params from the selected workspace:** `src/commands/agents.commands.view-system-prompt.ts:101-105`
  - body: Passing `process.cwd()` makes repo/source resolution depend on the shell launch directory rather than the selected agent workspace. Embedded runs build prompt params with the effective workspace as `cwd`, so this can render a different Runtime section for the inspected agent.
  - confidence: 0.88
- **[P2] Share attempt prompt assembly before printing:** `src/commands/agents.commands.view-system-prompt.ts:164-187`
  - body: Calling `buildEmbeddedSystemPrompt` directly skips attempt-level behavior such as system prompt overrides, provider contributions/transforms, source path, native command guidance, sandbox info, and effective tool filtering. Configurations using those hooks will see a preview that is not the submitted prompt.
  - confidence: 0.86
