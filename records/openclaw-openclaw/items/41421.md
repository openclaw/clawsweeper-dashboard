## What This Changes

The PR adds an `openclaw agents view-system-prompt [agentId]` CLI subcommand that assembles and prints a prompt preview with `--model`, `--channel`, and stderr metadata.

## Best Possible Solution

Implement prompt inspection through a shared prompt-preview path that reuses runtime prompt assembly and plugin APIs, with documented live-only omissions and focused CLI coverage.

## Reproduction Assessment

Yes for the PR findings: CI reproduces TS2307 on the added imports, and source inspection shows preview assembly diverges from current runtime assembly. This is not a current-main bug reproduction because the requested command is absent on main.

## Solution Assessment

No. The branch duplicates runtime prompt assembly and deep-imports removed channel modules; the maintainable path is a shared prompt-preview helper that carries prepared runtime facts and plugin prompt adapters forward.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Route channel prompt helpers through plugin APIs:** `src/commands/agents.commands.view-system-prompt.ts:31-33`
  - body: These imports target `src/signal` and `src/telegram` modules that are absent at the PR head, and the `check` job fails with TS2307 for all three lines. Use the channel plugin prompt APIs instead of hard-coding removed channel internals from the core CLI command.
  - confidence: 0.98
- **[P2] Use runtime capability collection for channel previews:** `src/commands/agents.commands.view-system-prompt.ts:78-99`
  - body: This manually combines raw channel capabilities with a Telegram special case, so it misses plugin prompt capabilities and thread-bound spawn capabilities that real runs include through `collectRuntimeChannelCapabilities`. `--channel` can therefore print Runtime/Messaging guidance that does not match an actual agent turn.
  - confidence: 0.92
- **[P2] Build prompt params from the selected workspace:** `src/commands/agents.commands.view-system-prompt.ts:101-105`
  - body: Passing `process.cwd()` makes repo/source resolution depend on the shell launch directory rather than the selected agent workspace. Embedded runs build prompt params with the effective workspace as `cwd`, so this can render a different Runtime section for the agent being inspected.
  - confidence: 0.88
- **[P2] Share attempt prompt assembly before printing:** `src/commands/agents.commands.view-system-prompt.ts:164-187`
  - body: Calling `buildEmbeddedSystemPrompt` directly skips attempt-level behavior such as system prompt overrides, provider transforms, source path, native command guidance, sandbox info, and effective tool filtering. Configurations using those hooks will see a preview that is not the submitted prompt.
  - confidence: 0.86
