  - body: These imports no longer resolve on current main because Telegram and Signal are extension-owned rather than core `src/telegram` or `src/signal` modules. Since the command promises the assembled prompt, this duplicate path can produce text that differs from production runs.
  - confidence: 0.9

  - body: The embedded runtime passes the effective workspace as both `workspaceDir` and `cwd`, but this command uses `process.cwd()`. Running it from a different checkout or non-workspace directory can render a Runtime section that does not match the selected agent workspace.
  - confidence: 0.86
- **[P2] Keep workspace notes aligned with runtime behavior:** `src/commands/agents.commands.view-system-prompt.ts:131-134`
  - body: This adds `Loaded context files: ...` to `workspaceNotes`, but production uses that field for runtime reminders while loaded files are represented through `contextFiles`. The preview can therefore include instructions that a real run never sends.
  - confidence: 0.82
