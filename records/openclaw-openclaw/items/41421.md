## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use current channel helper imports:** `src/commands/agents.commands.view-system-prompt.ts:31-33`
  - body: The new command imports `../signal/reaction-level.js` and `../telegram/*`, but those modules do not exist under current core `src`, so the branch cannot build. Route this through the generic channel/plugin prompt helpers instead of importing channel internals from core.
  - confidence: 0.96
- **[P2] Collect runtime channel capabilities:** `src/commands/agents.commands.view-system-prompt.ts:78-80`
  - body: This only calls `resolveChannelCapabilities`, while real embedded runs use `collectRuntimeChannelCapabilities`, which adds plugin prompt capabilities, native approval UI capability, and thread-bound spawn capabilities. `--channel` previews can therefore omit capabilities the real prompt includes.
  - confidence: 0.89
- **[P2] Gate heartbeat text through the runtime helper:** `src/commands/agents.commands.view-system-prompt.ts:153-156`
  - body: Resolving heartbeat text for every default-agent preview bypasses the runtime checks for policy, cadence, trigger, and `includeSystemPromptSection`. Disabled heartbeat configs would show prompt text the agent does not actually receive.
  - confidence: 0.91
- **[P2] Use the agent workspace as prompt cwd:** `src/commands/agents.commands.view-system-prompt.ts:101-105`
  - body: Passing `process.cwd()` makes repo/source resolution depend on where the operator launches the CLI. Embedded runs build prompt params with the effective agent workspace, so this can render a different Runtime section for the selected agent.
  - confidence: 0.88
- **[P2] Apply runtime prompt wrappers before output:** `src/commands/agents.commands.view-system-prompt.ts:164-187`
  - body: Calling `buildEmbeddedSystemPrompt` directly skips the current attempt-level wrapper for system prompt overrides, provider prompt contributions/transforms, source path, native command guidance, and sandbox info. Configs/providers using those hooks will see a preview that is not the submitted prompt.
  - confidence: 0.86
- **[P2] Normalize the simulated channel once:** `src/commands/agents.commands.view-system-prompt.ts:81-82`
  - body: The Telegram/Signal branches compare `opts.channel` directly, while the embedded runtime normalizes channel IDs before channel-specific prompt checks. Mixed-case or alias inputs can resolve generic hints but skip capability or reaction guidance.
  - confidence: 0.83
