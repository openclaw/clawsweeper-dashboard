## Summary

Keep open: current main still lacks the command, and this stale branch is not merge-ready because it imports removed channel internals and duplicates prompt assembly in ways that diverge from production runtime behavior.

## What This Changes

Adds an `openclaw agents view-system-prompt [agentId]` subcommand that prints a locally assembled system prompt preview with `--model`, `--channel`, and stderr metadata.

## Best Possible Solution

Land a rebased implementation that reuses a production prompt-composition helper where feasible, documents unavoidable static-preview limits, and adds parity coverage for channel capabilities, heartbeat gating, workspace cwd, provider transforms, tool metadata, and stderr output.

## Reproduction Assessment

Yes. Static inspection is enough: current main lacks the subcommand, and the PR diff imports modules absent from current main while assembling prompt inputs differently from `runEmbeddedAttempt`.

## Solution Assessment

No. The feature direction is useful, but duplicating prompt assembly in a CLI handler is not the narrowest maintainable solution now that production prompt composition includes plugin channel capabilities, heartbeat policy, workspace context, sandbox metadata, effective tools, and provider transforms.
