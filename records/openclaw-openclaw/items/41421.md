## Summary

Keep open. The feature is useful, but this PR is stale against current prompt/channel/runtime composition and would print prompts that diverge from real embedded runs; current main also does not already implement the command.

## What This Changes

The PR adds an `openclaw agents view-system-prompt [agentId]` CLI subcommand, a new prompt-preview handler, and a command-barrel export.

## Best Possible Solution

Land a rebased implementation that shares or extracts the production prompt-composition path, with docs, changelog, and parity tests for channel capabilities, workspace resolution, heartbeat policy, provider overlays/transforms, sandbox/tool policy, and metadata output.

## Reproduction Assessment

Yes. Static inspection is enough: current main has no command, and the PR head both imports paths absent from current main and assembles prompt inputs differently from the production embedded run path.

## Solution Assessment

No. The requested feature is reasonable, but duplicating prompt assembly inside a CLI handler is not the best maintainable shape; a shared production prompt-composition seam would avoid the drift shown in this review.

  - confidence: 0.9
- **[P2] Resolve prompt params from the agent workspace:** `src/commands/agents.commands.view-system-prompt.ts:101-105`
  - body: Passing `process.cwd()` lets an unrelated launch directory influence repo-root fallback when the selected workspace has no git root. Embedded runtime passes the effective workspace as `cwd`, so this command can print a Runtime section that production would not send.
  - confidence: 0.86
- **[P2] Gate heartbeat guidance through runtime policy:** `src/commands/agents.commands.view-system-prompt.ts:153-156`
  - body: This injects heartbeat guidance for the default agent whenever it is selected, but runtime also checks trigger, cadence, per-agent policy, and `includeSystemPromptSection`. Normal-run previews can include heartbeat instructions that the agent would not receive.
  - confidence: 0.91
- **[P2] Apply runtime tool policy before rendering:** `src/commands/agents.commands.view-system-prompt.ts:52-57`
  - body: The command creates tools without the selected model, message provider, session, sandbox, or owner context, so model/channel/provider filtering is skipped. The prompt can list tool guidance that the actual runtime removes.
  - confidence: 0.88
- **[P2] Apply provider prompt contributions before printing:** `src/commands/agents.commands.view-system-prompt.ts:164-187`
  - body: The command prints the direct `buildEmbeddedSystemPrompt` result, while runtime resolves provider prompt contributions and applies provider/plugin system-prompt transforms. Provider overlays and text transforms can be missing from the displayed prompt.
  - confidence: 0.92
- **[P2] Keep workspace notes aligned with runtime:** `src/commands/agents.commands.view-system-prompt.ts:130-134`
  - body: Injecting `Loaded context files: ...` through `workspaceNotes` adds prompt text production does not send and misses the runtime workspace reminders production may send. The file contents already flow through `contextFiles`.
  - confidence: 0.84

  - body: This is a user-facing `feat(cli)` command, but the PR does not update `CHANGELOG.md`. OpenClaw policy requires user-facing feature changes to be recorded before merge.
