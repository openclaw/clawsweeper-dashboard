## Summary

Keep open. The requested CLI surface is not implemented on current main, but this PR is not mergeable as-is because CI fails on unresolved imports, the preview path bypasses important runtime prompt assembly, and the external contributor has not provided real command-run proof.

## What This Changes

The PR adds an `openclaw agents view-system-prompt [agentId]` CLI subcommand with `--model` and `--channel` preview options plus stderr prompt metadata.

## Best Possible Solution

Expose prompt inspection through a shared maintained preview path that reuses runtime prompt assembly where feasible, stays behind channel/plugin APIs, documents static-preview limits, and includes focused CLI coverage plus real command proof.

## Reproduction Assessment

Yes for the PR findings: the GitHub CI log shows TS2307 for the added imports, and source inspection shows the preview path diverges from current runtime prompt assembly. I did not run the branch locally because this is a read-only review.

## Solution Assessment

No. Duplicating prompt assembly inside a new command is brittle; a shared prompt-preview helper or an explicit static-preview contract that reuses runtime and plugin seams is the narrower maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Route channel prompt helpers through plugin APIs:** `src/commands/agents.commands.view-system-prompt.ts:31-33`
  - body: These imports point at core `src/signal` and `src/telegram` modules that do not exist on current main, so the PR fails CI with TS2307. The command needs to use the channel plugin prompt/helper surface instead of hard-coding bundled channel internals from core CLI code.
  - confidence: 0.98
- **[P2] Collect runtime channel capabilities for previews:** `src/commands/agents.commands.view-system-prompt.ts:78-99`
  - body: This builds capabilities from config-only helpers plus hardcoded Telegram handling. Real runs normalize the channel and call `collectRuntimeChannelCapabilities`, which also includes plugin prompt capabilities and thread-bound capabilities, so `--channel` previews can omit instructions the agent actually receives.
  - confidence: 0.92
- **[P2] Build runtime metadata from the agent workspace:** `src/commands/agents.commands.view-system-prompt.ts:101-105`
  - body: Passing `process.cwd()` makes repo/source resolution depend on where the operator launches the CLI. Embedded runs pass the effective agent workspace as `cwd`, so this can render a different Runtime section for the selected agent.
  - confidence: 0.87
- **[P2] Share attempt prompt assembly for the rendered prompt:** `src/commands/agents.commands.view-system-prompt.ts:164-187`
  - body: Calling `buildEmbeddedSystemPrompt` directly skips attempt-level behavior such as system prompt overrides, provider transforms/contributions, source path, native command guidance, sandbox info, bootstrap notices, and effective tool filtering. Configurations using those hooks will see a preview that is not the submitted prompt.
  - confidence: 0.86
