## Summary

Keep open: current main still lacks the requested CLI command, but this PR head does not build and duplicates prompt assembly in ways that can diverge from real agent runs; it also lacks contributor real behavior proof.

## What This Changes

The PR adds an `openclaw agents view-system-prompt [agentId]` CLI subcommand with `--model` and `--channel` options plus stderr prompt metadata.

## Best Possible Solution

Expose prompt inspection through a shared maintained preview path that reuses embedded attempt assembly where possible, keeps channel behavior behind plugin helpers, adds focused CLI coverage, and documents any unavoidable static-preview limits.

## Reproduction Assessment

Yes for the PR findings: GitHub CI logs show unresolved imports, and source inspection confirms prompt assembly diverges from the current runtime path. I did not run the command locally because this is a read-only review.

## Solution Assessment

No. Duplicating prompt assembly inside the command is brittle; a shared prompt-preview helper or explicit static-preview contract would be the narrower maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Route channel prompt helpers through plugins:** `src/commands/agents.commands.view-system-prompt.ts:31-33`
  - body: These imports point at `src/signal` and `src/telegram`, but those core paths do not exist on current main, and CI is already failing with unresolved imports. Use the channel plugin prompt helper surface instead of importing plugin internals from the core command.
  - confidence: 0.98
- **[P2] Collect runtime channel capabilities for previews:** `src/commands/agents.commands.view-system-prompt.ts:78-99`
  - body: This builds capabilities from config-only helpers plus hardcoded Telegram handling. Real runs normalize the channel and call `collectRuntimeChannelCapabilities`, which also includes plugin prompt and thread-bound capabilities, so `--channel` previews can omit instructions the agent actually receives.
  - confidence: 0.91
- **[P2] Build runtime info from the agent workspace:** `src/commands/agents.commands.view-system-prompt.ts:105`
  - body: Passing `process.cwd()` lets repo/source resolution depend on where the operator launches the CLI. Embedded runs pass the effective agent workspace as `cwd`, so this can render a different Runtime section for the selected agent.
  - confidence: 0.86
- **[P2] Share attempt prompt assembly instead of rendering directly:** `src/commands/agents.commands.view-system-prompt.ts:164-187`
  - body: Calling `buildEmbeddedSystemPrompt` directly skips attempt-level handling for prompt overrides, provider transforms/contributions, source path, native command guidance, sandbox info, heartbeat policy, and effective tool filtering. Configs using those hooks will see a preview that is not the submitted prompt.
  - confidence: 0.87
