## What This Changes

The PR adds `openclaw agents view-system-prompt [agentId]` plus `--model` and `--channel` options to print a locally assembled agent system prompt and stderr metadata.

## Best Possible Solution

Implement this as a shared runtime prompt-preview helper that reuses embedded attempt prompt assembly and plugin prompt APIs, with explicit output text for any live-only state that cannot be represented offline.

## Reproduction Assessment

Yes for the PR findings: source inspection on the PR head shows imports to removed `src/telegram` and `src/signal` paths, and current runtime code shows the preview path is not the assembled prompt path. This is not a current-main bug reproduction because current main does not contain the requested command.

## Solution Assessment

No. The maintainable fix is to share runtime prompt assembly and plugin prompt APIs rather than manually re-creating a partial prompt builder inside a CLI command.
