## Summary

Keep open. The feature is useful and the author addressed earlier review comments, but the branch is stale against current main: it imports removed channel paths, misses current generic prompt/channel helpers, and no longer matches the embedded runtime prompt assembly contract.

## What This Changes

The PR adds an `openclaw agents view-system-prompt [agentId]` CLI command that renders an assembled agent system prompt with optional model/channel overrides and stderr metadata.

## Best Possible Solution

Share or extract a current prompt-preview assembly path from the embedded runtime, explicitly document any live-only omissions, then add focused tests and a changelog entry for the new CLI command.

## Reproduction Assessment

Yes for the PR findings. Source inspection of the PR head shows unresolved channel imports and prompt assembly paths that diverge from current main; I did not run a build because this review is read-only and `pnpm` is unavailable in the shell.

## Solution Assessment

No. The current PR duplicates runtime assembly and has fallen behind the generic prompt/channel helpers; the safer direction is a shared helper or a deliberately scoped preview path with explicit limitations.

Full review comments:

- **[P1] Use current channel helper imports:** `src/commands/agents.commands.view-system-prompt.ts:31-33`
  - body: The new command imports `../signal/reaction-level.js` and `../telegram/*`, but those modules do not exist in the PR head/current main, so the branch cannot build. Use the generic channel prompt helpers such as `collectRuntimeChannelCapabilities` and `resolveChannelReactionGuidance` instead of core importing channel-specific paths.
  - confidence: 0.95
- **[P2] Honor heartbeat prompt gating:** `src/commands/agents.commands.view-system-prompt.ts:153-156`
  - body: This injects heartbeat text for every default-agent preview. Current runtime only includes it after heartbeat policy/cadence and `includeSystemPromptSection` checks, so disabled or per-agent heartbeat configs would show a Heartbeats section the real agent does not receive.
  - confidence: 0.9
- **[P2] Use the agent workspace for prompt params:** `src/commands/agents.commands.view-system-prompt.ts:101-105`
  - body: Passing `process.cwd()` makes repo-root and docs/source resolution depend on where the operator launches the CLI. Embedded runs pass the effective workspace as `cwd`, so this can render a different Runtime section for the selected agent.
  - confidence: 0.86
- **[P2] Apply runtime prompt overrides and contributions:** `src/commands/agents.commands.view-system-prompt.ts:164-187`
  - body: Calling `buildEmbeddedSystemPrompt` directly skips current runtime wrapping for `resolveSystemPromptOverride`, provider prompt contributions, provider transforms, source path, native command guidance, and sandbox info. Configs/providers using those hooks will see a preview that is not the prompt production sends.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `src/cli/program/register.agent.ts:270`
  - body: This is a user-facing CLI feature, but the PR only changes command files. OpenClaw policy requires user-facing `feat` changes to update `CHANGELOG.md`, so the command would ship without release notes.
  - confidence: 0.92
