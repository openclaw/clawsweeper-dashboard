## Summary

Keep open. Current main still lacks the command, but the PR branch is stale against the current prompt/channel architecture and would produce prompt previews that diverge from real embedded runs.

## What This Changes

The PR adds an `openclaw agents view-system-prompt [agentId]` CLI subcommand with `--model` and `--channel` options plus stderr prompt metadata.

## Best Possible Solution

Land a rebased implementation that shares or extracts the production prompt-composition path, then add focused CLI docs and parity tests for channel/plugin prompt capabilities, workspace cwd, heartbeat policy, provider prompt contributions/transforms, and metadata output.

## Reproduction Assessment

Yes. Static inspection is enough: current main has no command, and the PR diff imports paths absent from current main while assembling prompt inputs differently from `runEmbeddedAttempt`.

## Solution Assessment

No. The feature direction is useful, but duplicating prompt assembly inside the CLI handler is not the narrowest maintainable fix; a shared production prompt-composition seam would reduce drift.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Replace removed channel imports:** `src/commands/agents.commands.view-system-prompt.ts:31-33`
  - body: These imports target `src/signal` and `src/telegram` modules that no longer exist on current main, so the branch will fail after rebase. Use the current extension API exports or generic channel prompt helpers instead of direct channel internals.
  - confidence: 0.97
- **[P2] Collect channel capabilities through the runtime helper:** `src/commands/agents.commands.view-system-prompt.ts:76-78`
  - body: This calls `resolveChannelCapabilities` directly, but real runs use `collectRuntimeChannelCapabilities`, which merges plugin-owned prompt capabilities and native approval hints. Channel previews can therefore omit capabilities the real prompt includes.
  - confidence: 0.9
- **[P2] Resolve prompt params from the agent workspace:** `src/commands/agents.commands.view-system-prompt.ts:102`
  - body: Passing `process.cwd()` makes repo/runtime fields depend on the shell launch directory. Embedded runtime passes the effective agent workspace as `cwd`, so this command can print a different prompt than a real run.
  - confidence: 0.92
- **[P2] Gate heartbeat guidance through runtime policy:** `src/commands/agents.commands.view-system-prompt.ts:151-154`
  - body: This injects heartbeat guidance for the default agent without the runtime trigger and heartbeat-policy checks. Normal-run previews can include heartbeat instructions that the agent would not receive.
  - confidence: 0.88
- **[P2] Apply provider prompt contributions before printing:** `src/commands/agents.commands.view-system-prompt.ts:163-187`
  - body: The command prints the direct `buildEmbeddedSystemPrompt` result, while runtime resolves provider prompt contributions and applies provider/plugin system-prompt transforms. Provider overlays can be missing from the displayed prompt.
  - confidence: 0.91
- **[P2] Keep workspace notes aligned with runtime:** `src/commands/agents.commands.view-system-prompt.ts:127-130`
  - body: Injecting `Loaded context files: ...` through `workspaceNotes` adds prompt text production does not send and misses the runtime workspace reminders production may send. The file contents already flow through `contextFiles`.
  - confidence: 0.86
