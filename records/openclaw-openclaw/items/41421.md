## Summary

## What This Changes

The PR adds an `openclaw agents view-system-prompt [agentId]` CLI command that assembles and prints an agent system prompt with optional `--model` and `--channel` overrides plus stderr metadata.

## Best Possible Solution

Land a current-main-compatible diagnostic command that reuses shared prompt context resolution, documents any intentional preview limits, and includes docs, changelog, and focused regression coverage.

The prompt-fidelity issues are also source-reproducible by comparing the PR implementation with current runtime prompt assembly.

## Solution Assessment

No. The feature direction is useful, but the implementation should use current generic channel/prompt helpers or extract a shared preview helper instead of duplicating stale owner-specific runtime logic.
