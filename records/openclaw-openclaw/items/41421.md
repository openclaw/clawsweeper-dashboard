## Summary

Keep open: current main still lacks the requested CLI command, but this PR is not correct as-is because it is stale against current prompt assembly and imports channel internals that no longer exist in core.

## What This Changes

The PR adds an `openclaw agents view-system-prompt [agentId]` subcommand, registers it under `agents`, and adds a handler that prints a locally assembled prompt preview with `--model` and `--channel` options.

## Best Possible Solution

Land a rebased implementation that shares the production prompt-composition path where possible, documents any intentional preview limits, and adds parity coverage for channel capabilities, workspace cwd, heartbeat policy, provider overlays/transforms, and metadata output.

## Reproduction Assessment

Yes. Static inspection is enough: current main lacks the command, and the PR diff imports removed channel internals while assembling prompt inputs differently from `runEmbeddedAttempt`.

## Solution Assessment

No. The feature direction is useful, but duplicating prompt assembly inside the CLI handler is not the narrowest maintainable solution now that production prompt composition includes provider/plugin transforms, channel capability helpers, heartbeat policy, sandbox context, and effective tool filtering.

## Review Findings

Full review comments:

- **[P1] Replace removed channel imports:** `src/commands/agents.commands.view-system-prompt.ts:31-33`
  - body: These imports target `src/signal` and `src/telegram`, but current main has no such core directories; Telegram/Signal helpers now live behind extension APIs, so the branch will fail to compile after rebase.
  - confidence: 0.97
- **[P2] Use the runtime capability helper:** `src/commands/agents.commands.view-system-prompt.ts:76-78`
  - body: Calling `resolveChannelCapabilities` directly misses plugin prompt capabilities and native approval/thread-spawn hints merged by `collectRuntimeChannelCapabilities`, so `--channel` previews can omit guidance real runs include.
  - confidence: 0.9
- **[P2] Use the agent workspace for prompt params:** `src/commands/agents.commands.view-system-prompt.ts:102`
  - body: Passing `process.cwd()` makes repo-root resolution depend on the shell launch directory. Embedded runtime passes the effective agent workspace as `cwd`, so the Runtime section can differ from a real run.
  - confidence: 0.92
- **[P2] Gate heartbeat guidance through runtime policy:** `src/commands/agents.commands.view-system-prompt.ts:151-154`
  - body: The command injects heartbeat guidance for the default agent without the runtime trigger, cadence, agent-policy, and `includeSystemPromptSection` checks, so normal-run previews can include a Heartbeats section production omits.
  - confidence: 0.88
- **[P2] Apply provider prompt overlays before printing:** `src/commands/agents.commands.view-system-prompt.ts:163-187`
  - body: The command prints the raw `buildEmbeddedSystemPrompt` result without resolving provider prompt contributions or applying provider/plugin system-prompt transforms, so provider overlays can be missing from the displayed prompt.
  - confidence: 0.91
- **[P2] Match runtime workspace notes:** `src/commands/agents.commands.view-system-prompt.ts:127-130`
  - body: Injecting `Loaded context files: ...` through `workspaceNotes` adds prompt text production does not send and misses the runtime BOOTSTRAP/local embedded reminders production may send.
