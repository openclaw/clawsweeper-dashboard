## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Match heartbeat injection policy:** `src/commands/agents.commands.view-system-prompt.ts:154-155`
  - body: This previews a heartbeat section for every default-agent run, but current embedded runtime only passes `heartbeatPrompt` when `shouldInjectHeartbeatPrompt` is true, which is limited to heartbeat-triggered runs and config that permits system-prompt guidance. A normal `view-system-prompt` call would therefore show instructions the agent does not receive.
  - confidence: 0.9
- **[P2] Use runtime channel prompt helpers:** `src/commands/agents.commands.view-system-prompt.ts:78-99`
  - body: The channel branch is hand-rolled from raw `opts.channel`, while embedded runtime normalizes the channel and uses `collectRuntimeChannelCapabilities`, message-action discovery input, and `resolveChannelReactionGuidance`. This misses plugin prompt capabilities, thread-bound spawn capabilities, account-aware hints, and aliases, so `--channel` can render a different prompt than production.
  - confidence: 0.87
- **[P2] Use the agent workspace as prompt cwd:** `src/commands/agents.commands.view-system-prompt.ts:105`
  - body: Passing `process.cwd()` lets the Runtime `repoRoot` depend on where the CLI was launched; embedded runs pass the effective agent workspace as both `workspaceDir` and `cwd`. Invoking this command from another checkout can therefore show a repo root that the selected agent would not see.
  - confidence: 0.84
- **[P2] Do not invent workspace notes for context files:** `src/commands/agents.commands.view-system-prompt.ts:131-134`
  - body: This writes `Loaded context files: ...` into `workspaceNotes`, but embedded runtime uses that field for its BOOTSTRAP commit reminder and embedded-mode notice. The preview can add a system instruction production never sends while omitting the real reminder.
  - confidence: 0.86
- **[P2] Gate ACP guidance with the availability helper:** `src/commands/agents.commands.view-system-prompt.ts:174`
  - body: Setting `acpEnabled` from config alone enables ACP spawn guidance by default even when no ACP backend is registered or healthy; runtime uses `isAcpRuntimeSpawnAvailable(...)` and disables it in sandboxed contexts. The preview can advertise unavailable ACP behavior.
  - confidence: 0.82
- **[P3] Add the required changelog entry:** `src/cli/program/register.agent.ts:273-277`
  - body: This adds a user-facing `openclaw agents` command but the PR only changes source files. OpenClaw policy requires a `CHANGELOG.md` entry for user-facing `feat` changes before merge.
  - confidence: 0.93
