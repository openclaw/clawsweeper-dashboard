## Summary

Keep open: current main still does not have this CLI command, and the PR head is stale against current prompt/channel assembly enough that it would fail to build and render prompts that diverge from runtime behavior.

## What This Changes

The PR adds an `openclaw agents view-system-prompt [agentId]` subcommand that renders an agent system prompt with optional model/channel overrides and stderr metadata.

## Best Possible Solution

Implement a shared prompt-preview assembly path aligned with the embedded runtime, document any live-only omissions, then add focused tests and a changelog entry.

## Reproduction Assessment

Yes for the PR defects. Source inspection of the PR head shows missing imports and prompt assembly paths that diverge from current main, and the PR head check-runs already show build failures.

## Solution Assessment

No. The current PR duplicates runtime assembly and has fallen behind the generic prompt/channel helpers; the safer path is a shared helper or a deliberately scoped static preview with explicit limitations.

- **[P1] Replace removed channel imports:** `src/commands/agents.commands.view-system-prompt.ts:31-33`
  - body: The new command imports `../signal/reaction-level.js` and `../telegram/*`, but those modules do not exist at the PR head or current main, so the branch cannot build.   - confidence: 0.96
- **[P2] Honor heartbeat prompt gating:** `src/commands/agents.commands.view-system-prompt.ts:153-156`
  - body: This injects heartbeat text for every default-agent preview. Current runtime only includes it after heartbeat policy, cadence, and `includeSystemPromptSection` checks, so disabled or per-agent heartbeat configs would show a Heartbeats section the real agent does not receive.
  - confidence: 0.91
- **[P2] Use the agent workspace for prompt params:** `src/commands/agents.commands.view-system-prompt.ts:101-105`
  - body: Passing `process.cwd()` makes repo-root and docs/source resolution depend on where the operator launches the CLI. Embedded runs pass the effective workspace as `cwd`, so this can render a different Runtime section for the selected agent.
  - confidence: 0.88

  - body: Calling `buildEmbeddedSystemPrompt` directly skips current runtime handling for prompt overrides, provider/runtime prompt contributions, source path, native command guidance, sandbox info, and effective tools. Configs/providers using those hooks will see a preview that is not the prompt production sends.
  - confidence: 0.84

  - body: This is a user-facing CLI feature, but the PR only changes command files. OpenClaw policy requires user-facing `feat` changes to update `CHANGELOG.md`, so the command would ship without release notes.
  - confidence: 0.92
