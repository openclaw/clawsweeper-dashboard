  - confidence: 0.91
- **[P2] Use the agent workspace for prompt params:** `src/commands/agents.commands.view-system-prompt.ts:101-105`
  - body: Passing `process.cwd()` makes repo/source resolution depend on where the operator launches the CLI. Embedded runs pass the effective agent workspace, so previews for a selected agent can show a different Runtime section than production.
  - confidence: 0.88
- **[P2] Gate heartbeat text through runtime policy:** `src/commands/agents.commands.view-system-prompt.ts:153-156`
  - body: This always resolves default-agent heartbeat text from `agents.defaults.heartbeat.prompt`. Current runtime first checks heartbeat agent policy, cadence, trigger, and `includeSystemPromptSection`, so disabled or per-agent heartbeat configs can show prompt text the agent would not receive.
  - confidence: 0.9
- **[P2] Build through the attempt prompt path:** `src/commands/agents.commands.view-system-prompt.ts:164-187`
  - body: Calling `buildEmbeddedSystemPrompt` directly bypasses `buildAttemptSystemPrompt`, provider prompt contributions/transforms, source path, native command guidance, sandbox info, and the effective tool set.
