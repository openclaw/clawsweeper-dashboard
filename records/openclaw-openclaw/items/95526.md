## Summary

Keep open: current main still routes long QQBot markdown through a plugin-local chunker and sends each chunk as its own QQ message, so the v2026.6.9 single-reply grouping problem remains unresolved rather than implemented or superseded.

## What This Changes

The issue requests that a long QQBot markdown reply remain one logical QQ message/bubble instead of being emitted as independent chunk messages.

## Best Possible Solution

Define a QQBot-owned long-markdown delivery policy that preserves table/fence validity while keeping one logical user-visible reply when the QQ API supports it, with gateway/direct tests and live QQ C2C proof.

## Reproduction Assessment

Yes by source inspection and the reporter's shipped-version logs: current main still chunks QQBot markdown and sends each chunk through a separate QQ message call, though no live QQ reproduction was run during this read-only review.

## Solution Assessment

No current fix is in place; #92428 improved markdown table chunk validity but left or intensified the user-visible multi-bubble behavior, so the next fix should be in QQBot outbound/chunker policy rather than shared core chunking.
