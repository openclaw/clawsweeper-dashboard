## What This Changes

The issue reports that QQBot long markdown replies in v2026.6.9 are split into multiple independent QQ message bubbles instead of one logical visible reply.

## Best Possible Solution

Keep this open for a plugin-local QQBot delivery fix that preserves one logical visible reply, backed by live QQ proof and focused chunker/outbound regression coverage.

## Reproduction Assessment

No high-confidence live reproduction was run. The reporter's logs plus current source are source-reproducible: long markdown enters `sendMarkdownReply`, is split by the QQBot chunker, and each chunk is sent through its own send loop.

## Solution Assessment

Unclear. The fix belongs in the QQBot plugin boundary, but maintainers should choose between a larger single markdown send, a supported QQ reply-chain strategy, or stream-message semantics after live QQ API/client proof.
