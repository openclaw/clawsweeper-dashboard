The issue asks for long QQBot markdown replies to be delivered as one logical visible QQ message instead of one independent bubble per chunk.

## Best Possible Solution

Implement a plugin-local QQBot delivery path that preserves one logical visible long-markdown reply, with live QQ C2C proof and focused regression coverage for the chosen strategy.
