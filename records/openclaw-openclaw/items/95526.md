## Summary

Keep open. Current main and v2026.6.9 still route QQBot static markdown through chunking and one send call per chunk, while the related merged QQBot markdown work only made chunks valid rather than preserving a single visible bubble.

## What This Changes

The issue requests that QQBot deliver long markdown replies as one logical QQ message or one collapsed reply chain instead of multiple independent message bubbles.

## Best Possible Solution

Keep this open for a plugin-local QQBot fix that preserves one logical visible reply, backed by live QQ C2C proof and focused regression coverage.
