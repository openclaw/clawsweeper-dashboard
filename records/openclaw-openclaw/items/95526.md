The issue requests that QQBot long markdown replies render as one logical QQ message instead of multiple independent chunk bubbles.

## Best Possible Solution

Implement a plugin-local QQBot delivery strategy that preserves one logical visible reply for long markdown, with live QQ C2C proof and focused chunker/dispatch regression coverage.
