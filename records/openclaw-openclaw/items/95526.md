Keep open. Current main and v2026.6.10 still route QQBot long markdown through chunking and one static send per chunk, while the merged QQBot markdown fixes only preserve valid table chunks rather than one visible QQ reply.

Issue reports that long QQBot markdown replies in v2026.6.9 are delivered as multiple independent QQ message bubbles instead of one logical QQ reply.

## Best Possible Solution

Keep this open for a QQBot-owned delivery fix that preserves one logical visible reply for long markdown, with live QQ C2C proof and focused regression coverage for tables, fences, and long replies.
