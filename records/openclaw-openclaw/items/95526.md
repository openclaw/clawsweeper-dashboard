## What This Changes

Reports a QQBot C2C regression where long markdown replies in v2026.6.9 are split into multiple visible QQ message bubbles instead of one logical reply.

## Best Possible Solution

Keep this open for a plugin-local QQBot fix that preserves one logical visible reply, backed by live QQ C2C proof and focused regression coverage.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection plus the reporter's logs show the static markdown path still chunks a long reply and sends each chunk as its own QQ API message.

## Solution Assessment

Unclear. The fix belongs in the QQBot plugin boundary, but maintainers should choose between one larger markdown send, official C2C stream-message reuse, or a proven reply-chain strategy after live QQ API/client proof.
