## Summary

Keep open. Current main and the v2026.6.9 tag still route QQBot markdown text through the static chunk planner and send each chunk with a separate QQ message API call, so the reported user-visible split remains unresolved while the right QQ API/client strategy still needs maintainer and live-platform confirmation.

## What This Changes

The issue asks QQBot to preserve one logical visible markdown reply for long C2C responses instead of sending each chunk as an independent QQ message bubble.

## Best Possible Solution

Keep this as a QQBot plugin bug and land a plugin-local fix that preserves one logical visible C2C markdown reply with live QQ proof and focused regression coverage.
