## Summary

Keep open. Current main and v2026.6.9 still send QQBot markdown reply chunks as separate static QQ messages, and the merged QQBot markdown work only made chunks valid rather than preserving one visible reply bubble.

## What This Changes

The issue requests a QQBot C2C markdown delivery fix so long assistant replies remain one logical visible reply instead of separate bubbles per chunk.

## Best Possible Solution

Keep this open for a QQBot plugin-local fix that preserves one logical visible reply, with live QQ proof deciding between a larger static markdown send, official C2C stream reuse, or a proven reply-chain strategy.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection plus the reporter's logs show the static markdown path still chunks a long reply and sends each chunk as its own QQ API message, but QQ client rendering needs live C2C proof.

## Solution Assessment

Unclear. The fix belongs in the QQBot plugin boundary, but maintainers should choose the delivery strategy after live QQ API/client proof rather than assuming static chunk merging is always safe.
