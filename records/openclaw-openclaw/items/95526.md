## Summary

Keep open: current main and v2026.6.10 still keep the QQBot static markdown path that chunks one logical reply and sends each chunk independently; the newer official C2C stream path is an opt-in mode, not proof that the reported upgrade path is fixed.

## What This Changes

The issue reports QQBot long markdown replies in v2026.6.9 being delivered as separate visible message bubbles instead of one logical QQ reply.

## Best Possible Solution

Keep this issue open for a QQBot plugin-local delivery decision that preserves one logical visible reply for long markdown, with live QQ C2C proof and focused regression coverage across the static and official-stream paths.

## Reproduction Assessment

No live current-main QQ run was executed, but source inspection reproduces the mechanics: static markdown replies are chunked and each chunk is sent as its own QQ send. The QQ client bubble/collapse behavior still needs live proof.

## Solution Assessment

Unclear. The fix belongs in the QQBot plugin boundary, but maintainers should choose between a safe static-path aggregation, official C2C stream behavior, or another proven reply-chain strategy after live QQ API/client validation.
