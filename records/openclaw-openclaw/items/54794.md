## Summary

Keep open. Current main still lacks Telegram inline-query runtime handling, answer-result plumbing, config/schema/docs, and tests; the related implementation PR is closed and unmerged, and the reporter added fresh context today rather than letting this go stale.

## What This Changes

Add Telegram inline query support so users can invoke an OpenClaw Telegram bot from any chat with `@botname query`, likely behind explicit configuration.

## Best Possible Solution

Keep this as the canonical Telegram inline-query tracker and implement it in the Telegram plugin with explicit config/schema/docs, conservative authorization and session semantics, bounded inline results, tests, and live Telegram proof.
