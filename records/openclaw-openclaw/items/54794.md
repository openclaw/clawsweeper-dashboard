## Summary

Keep open: current main still lacks Telegram inline-query handling, answer delivery, public configuration, docs, and focused tests. The closed implementation attempt remains useful reference material, but it was not merged and had unresolved correctness/security concerns; the newer Guest Mode work is adjacent and does not replace inline-result support.

## What This Changes

Requested support for Telegram inline queries so users can invoke the Telegram bot from other chats with @botname query and receive inline results.

## Best Possible Solution

Keep this as the canonical Telegram inline-query tracker and implement a reviewed Telegram plugin design with opt-in config, conservative authorization/session semantics, bounded answerInlineQuery handling, docs/tests, and real Telegram proof.

## Reproduction Assessment

Yes, as a source-level feature gap: current main has allowed-update metadata from grammY but no inline_query/chosen_inline_result handler, answerInlineQuery path, or public inline-query config/docs. I did not run a live Telegram bot because the missing runtime path is directly visible in source.

## Solution Assessment

Unclear as a concrete implementation plan. The Telegram plugin is the right owner boundary, but maintainers still need to decide authorization, privacy, session scope, deadlines, result semantics, and relation to Guest Mode before code lands.
