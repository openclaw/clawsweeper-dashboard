## What This Changes

The issue requests Telegram inline query support so users can invoke the Telegram bot from any chat with `@botname query`, route the query through OpenClaw, and return inline results.

## Best Possible Solution

Keep this as the canonical tracker until maintainers approve a Telegram plugin design with explicit auth, privacy/session, timeout, result, config/docs, and test coverage boundaries.

## Solution Assessment

## Security Review

Status: needs_attention

Summary: The issue itself is security-sensitive because inline queries add a new arbitrary-chat Telegram ingress path into OpenClaw.

Concerns:

- **[medium] Define authorization and privacy boundaries first:**
  - body: Inline queries can be triggered from chats where the bot is not a normal participant, so maintainers need a clear rule for allowed senders, session binding, where answers appear, and what context is exposed before implementation.
  - confidence: 0.85
- **[medium] Avoid query text becoming executable instructions:**
  - body: The closed reference PR review found prompt-injection risk from embedding user query text into shell-style agent instructions; a future design should avoid command construction from inline query text or strictly isolate it.
  - confidence: 0.8
