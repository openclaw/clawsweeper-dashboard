## Summary

Keep open. Current main still lacks Telegram inline-query UX: inline-query strings appear only in update metadata and BotFather capability/status reporting, while the Telegram runtime has no `inline_query` or `chosen_inline_result` handler, no `answerInlineQuery`/`InlineQueryResult` path, and no config/schema/docs/tests for enabling or constraining inline mode. The related implementation PR #51963 is closed and unmerged, so it remains reference material rather than shipped behavior.

## What This Changes

Add Telegram inline query support so users can invoke the bundled Telegram bot as `@botname query` from other chats and return inline results.

## Best Possible Solution

Keep this as the canonical Telegram inline-query tracker. The desired end state is a reviewed implementation inside the bundled Telegram plugin with explicit config/schema/docs, conservative authorization and session semantics, bounded request handling, `InlineQueryResultArticle` responses, chosen-result handling if needed, and focused Telegram plugin tests; #51963 should be mined only as reference material.
