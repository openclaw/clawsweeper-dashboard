## Summary

Keep open. Current main and the latest release still lack a Telegram inline-query runtime path; the only related code reports BotFather account capability and supports existing inline buttons. The related implementation PR is closed and unmerged with prior correctness/security concerns, so this remains a real feature request needing maintainer design rather than cleanup closure.

## What This Changes

Add Telegram inline query support so users can invoke the bot from any Telegram chat with `@botname query` and receive inline results without switching to the bot DM.

## Best Possible Solution

Keep this as the canonical tracker and implement a reviewed Telegram plugin feature with config/schema/docs, explicit authorization/session/privacy semantics, bounded `answerInlineQuery` results, optional chosen-result behavior, focused regression tests, and live Telegram proof.

## Reproduction Assessment
