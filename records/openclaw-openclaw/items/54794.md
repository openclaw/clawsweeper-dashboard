## Summary

Keep open. Current main still lacks Telegram inline-query support: the Telegram plugin registers reaction, callback, message, and channel-post handlers, but there is no `inline_query` or `chosen_inline_result` handler and no `answerInlineQuery` result path; the related implementation PR is closed and unmerged.

## What This Changes

Request support for Telegram inline queries so `@botname query` can be invoked from any chat and return inline results through OpenClaw.

## Best Possible Solution

Keep this as the canonical Telegram inline-query tracker and implement the feature in the bundled Telegram plugin with reviewed config/schema/docs, conservative auth and session semantics, bounded response handling, and focused tests.

## Reproduction Assessment

Yes. Source inspection on current main shows no `inline_query` or `chosen_inline_result` handler and no `answerInlineQuery` path, so the missing runtime route is reproducible without a live Telegram bot.

## Solution Assessment

Unclear for automation. The feature belongs in the Telegram plugin, but the safest design needs maintainer decisions about auth, privacy, session keys, timeouts, and inline-result UX before a narrow implementation PR is appropriate.
