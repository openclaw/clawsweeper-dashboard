## Risks / Open Questions

- Inline queries originate from arbitrary Telegram contexts, so implementation needs explicit sender authorization, privacy/session boundaries, result limits, rate limiting or deadlines, and safe handling of selected results before routing into agent turns.
- The existing `inlineQueries=true/false` status flag can be mistaken for OpenClaw runtime support even though it only reports the Telegram bot account capability.
- The Guest Bots and bot-to-bot tracker overlaps the arbitrary-chat ingress design, so maintainers should coordinate both efforts without conflating inline-result UX with peer delegation semantics.
