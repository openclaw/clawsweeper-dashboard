## Summary

Keep open. Current main still does not implement Telegram inline-query UX: the Telegram plugin has message, reaction, callback, and channel-post handlers, but no inline_query/chosen_inline_result handler, no answerInlineQuery or InlineQueryResult response path, and no config/docs/test surface for enabling or constraining inline mode. The related implementation PR #51963 is closed and unmerged, so it is useful reference material rather than shipped behavior.

## What This Changes

Add Telegram inline-query support so users can invoke an OpenClaw Telegram bot from any chat with `@botname query` and receive shareable inline results routed through OpenClaw.

## Best Possible Solution

Keep this as the canonical Telegram inline-query tracker. The end state should be a reviewed implementation inside the bundled Telegram plugin with explicit config/schema/docs, conservative sender authorization and session semantics, bounded request handling, `InlineQueryResultArticle` responses, optional chosen-result handling if needed, and focused Telegram plugin tests; #51963 can be mined for reference but should not be treated as shipped.

## Reproduction Assessment

Yes. On current main, a code-level reproduction is high confidence: inspect registered Telegram handlers and search for inline-query answer paths, and there is no handler that would process a Telegram `inline_query` update. A live Telegram bot is not needed to prove the missing runtime path.

## Solution Assessment

Unclear for an automated fix. The requested feature belongs in the Telegram plugin, but the best implementation needs maintainer decisions around authorization, privacy, session keys, deadlines, and result UX before a narrow repair PR is safe.
