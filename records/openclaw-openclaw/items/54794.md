## What This Changes

Add opt-in Telegram inline query support so users can invoke an OpenClaw Telegram bot as `@botname query` from other chats and receive inline result answers.

## Best Possible Solution

Keep this as the canonical Telegram inline-query tracker and implement a Telegram-owned, opt-in design with explicit authorization, isolated session/result semantics, bounded answerInlineQuery handling, config/schema/docs, focused tests, and live Telegram proof.

## Solution Assessment

Unclear as a concrete implementation plan. The Telegram plugin is the right owner boundary, but maintainers still need to decide auth, privacy, session scope, result semantics, deadlines, and coordination with Guest Mode before code lands.
