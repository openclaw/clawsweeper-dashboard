## Summary

Keep open. Current main still lacks Telegram inline-query support: the Telegram plugin exposes allowed-update metadata and a BotFather capability/status flag, but no `inline_query` or `chosen_inline_result` handler, no `answerInlineQuery` result path, and no public config/docs surface; the related implementation PR #51963 is closed and unmerged.

## What This Changes

Request adding Telegram inline query support so users can invoke the Telegram bot with `@botname query` from any chat.

## Best Possible Solution

Keep this as the canonical Telegram inline-query tracker and implement a reviewed Telegram plugin feature with explicit config/schema/docs, conservative sender/session semantics, bounded `answerInlineQuery` handling, and focused tests; use #51963 only as reference material.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection on current main does high-confidence confirm the missing handler, result path, config, and docs surface.

## Solution Assessment

Unclear for automation. The feature belongs in the Telegram plugin, but the best implementation requires maintainer decisions about authorization, privacy, session keys, result UX, and deadlines before a narrow fix PR is safe.
