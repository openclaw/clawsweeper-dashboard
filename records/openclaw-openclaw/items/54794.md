## Summary

Keep open. Current main still lacks a Telegram inline-query runtime, response path, config contract, docs, and tests; the earlier implementation PR is closed and unmerged, and the newer Guest Bots discussion is related but not a replacement for this inline-query tracker.

## What This Changes

Request to add Telegram inline query support so users can invoke an OpenClaw bot from other Telegram chats with `@botname query`.

## Best Possible Solution

Keep this as the canonical Telegram inline-query tracker and implement a reviewed Telegram plugin feature with explicit config/schema/docs, conservative authorization and session semantics, bounded inline results, focused tests, live Telegram proof, and coordination with the separate Guest Bots tracker.

## Solution Assessment
