## Summary

Keep open. Current main still lacks a Telegram inline-query handler, answer path, public config/docs contract, and tests; the earlier inline-query PR is closed unmerged, and the newer Guest Bots issue is related but not a replacement.

## What This Changes

Request support for invoking the Telegram bot from any chat with `@botname query` and returning inline-query results.

## Best Possible Solution

Keep this as the canonical inline-query tracker and implement it in the Telegram plugin with explicit config/schema/docs, conservative authorization and session semantics, bounded InlineQueryResult responses, focused tests, and live Telegram proof.

## Reproduction Assessment

Not applicable as a broken-behavior bug repro; this is a feature request. Source inspection gives a high-confidence feature gap because current main has no inline-query/chosen-result handler or answerInlineQuery path.

## Solution Assessment

No for an autonomous repair lane. The Telegram plugin is the right owner boundary, but the feature needs maintainer decisions on auth, privacy, session scope, result semantics, rate limits, deadlines, and coordination with Guest Bots.
