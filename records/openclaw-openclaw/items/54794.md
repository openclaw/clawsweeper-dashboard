## What This Changes

Request to add Telegram inline-query support so users can invoke an OpenClaw Telegram bot from any chat with `@botname query`, with agent routing and opt-in configuration.

## Best Possible Solution

Keep this as the canonical Telegram inline-query tracker, coordinate it with the separate Guest/Bot-to-Bot work, and implement a Telegram-owned opt-in design with conservative auth, isolated session/result semantics, bounded `answerInlineQuery` handling, docs, tests, and live Telegram proof.

## Solution Assessment

Unclear as a concrete implementation plan. The Telegram plugin is the right owner boundary, but maintainers still need to decide auth, privacy, session scope, result semantics, deadlines, and coordination with Guest Mode before code lands.

- Guest Mode may satisfy part of the same user workflow, but it is a separate Telegram update and delivery model and does not close the inline-result UX requested here.
