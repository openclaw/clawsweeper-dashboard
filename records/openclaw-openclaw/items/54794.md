## Summary

Keep open. Current main still lacks Telegram inline-query UX: the Telegram plugin handles reactions, callbacks, migrated messages, regular messages, and channel posts, but has no inline_query/chosen_inline_result handler or answerInlineQuery result path; the related implementation PR is closed and unmerged.

## What This Changes

Request Telegram inline-query support so users can invoke the Telegram bot from any chat with `@botname query` and receive shareable inline results.

## Best Possible Solution

Keep this as the canonical Telegram inline-query tracker and implement it in the bundled Telegram plugin with explicit config/schema/docs, conservative authorization and session semantics, bounded `answerInlineQuery` handling, optional chosen-result handling, and focused regression coverage; PR #51963 can be mined for reference only.

## Reproduction Assessment

Yes. Source inspection on current main shows no `inline_query` or `chosen_inline_result` handler and no `answerInlineQuery` response path, so a live Telegram bot is not needed to verify the missing runtime path.

## Solution Assessment

Unclear for automation. The feature belongs in the Telegram plugin, but the best implementation needs maintainer decisions about authorization, privacy, session keys, deadlines, and inline-result UX before a narrow repair PR is safe.
