## Summary

Keep open: current main and the latest shipped release still have the asymmetric CONNECT tunnel socket error handling, and the issue is paired with an open contributor PR that targets the same fix path.

## What This Changes

The issue reports that the debug proxy CONNECT tunnel lacks a client socket error handler and can crash or leak resources when the client disconnects abruptly.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/82444 or an equivalent narrow fix that handles errors from both CONNECT tunnel sockets, then close this issue after merge with focused regression or real-proxy proof.

## Reproduction Assessment

Yes, at source level: current main and v2026.5.12 pipe both CONNECT sockets but only attach an error listener to upstreamSocket, while Node throws an unhandled 'error' event when no listener is registered. I did not run the full kill-client proxy repro in this read-only pass.

## Solution Assessment

Yes, the proposed symmetric socket cleanup is the narrow maintainable fix for the implicated code path. The issue should stay open while the paired PR is reviewed, proved, and either landed or replaced.
