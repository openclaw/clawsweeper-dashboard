## Summary

Keep open: current main still has the reported CONNECT tunnel socket-handling gap, and the same-author open PR remains the active fix path rather than a reason to close this issue before merge.

## What This Changes

The issue reports that the debug proxy CONNECT tunnel lacks a clientSocket error handler and should destroy the paired socket on abrupt client disconnect.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82444 or an equivalent narrow fix that handles both CONNECT tunnel socket error paths, then close this issue after merge with regression or live proof.

## Reproduction Assessment

Yes, at source level: current main pipes both CONNECT sockets but only handles upstreamSocket errors, and Node treats an unhandled error event as thrown. I did not run the full live proxy kill-client reproduction in this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix is symmetric CONNECT socket cleanup: record client socket errors, destroy the paired upstream socket, and destroy the client socket on upstream errors. The active PR is the right consolidation point, pending patched real-behavior proof.
