## What This Changes

The issue reports that the debug proxy can crash on abrupt stream disconnects because one of the sockets or response streams lacks an error handler.

## Best Possible Solution

Keep this issue open as the canonical bug report while maintainers refresh and review https://github.com/openclaw/openclaw/pull/88052, or explicitly split the remaining upstreamRes crash into a new canonical issue before closing this one.

## Reproduction Assessment

Yes. Current main source gives a high-confidence reproduction path for the remaining same-class upstreamRes crash because that stream is consumed with data/end listeners but no error listener; I did not run a live crash probe in this read-only review.

## Solution Assessment

Yes. The best fix is the narrow stream-consumption error handler in the HTTP forwarding path, while keeping the already-shipped CONNECT socket fix intact.
