## Summary

Keep open: current main still has the CONNECT tunnel socket-handling gap, and the issue is paired with an open contributor PR for the same fix path.

## What This Changes

Handle abrupt debug proxy CONNECT client disconnects by logging client socket errors, destroying the upstream socket, and using destroy-based cleanup for upstream errors.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82444 or an equivalent narrow fix with focused proof, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes, at source level: current main pipes the CONNECT sockets but only registers an upstream socket error listener, and Node throws unhandled error events. A live repro would start the debug proxy, establish a CONNECT tunnel, then abruptly reset the client.

## Solution Assessment

Yes, the symmetric clientSocket error handler plus destroy-based paired-socket cleanup is the narrow maintainable fix. Because an open paired PR already carries that direction, the best path is review and land that PR rather than create a duplicate repair.
