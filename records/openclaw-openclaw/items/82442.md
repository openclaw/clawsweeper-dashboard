## Summary

Keep open: current main still has the reported CONNECT tunnel socket-handling gap, and the same author has an active paired fix PR, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that the debug proxy CONNECT tunnel lacks a client socket error handler and can crash or leak cleanup on abrupt client disconnects.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/82444 or an equivalent narrow fix that handles errors from both CONNECT tunnel sockets, adds focused regression coverage when feasible, and closes this issue after merge.

## Reproduction Assessment

Yes, at source level: current main pipes the CONNECT sockets but only the upstream socket has an error listener, and a Node runtime probe confirms an unhandled error event throws. I did not run the full proxy kill-client scenario in this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix is symmetric socket error cleanup in the existing CONNECT handler: record the client socket error, destroy the paired upstream socket, and use destroy() for upstream-error cleanup. The active paired PR already carries that direction, so the issue should wait for that review and merge path.
