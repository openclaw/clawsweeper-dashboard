## Summary

Keep open: current main still has the debug proxy CONNECT socket-handling gap, and an open contributor PR already declares the closing fix path for this issue.

## What This Changes

The issue asks for the debug proxy CONNECT tunnel to handle abrupt client-side socket errors without crashing the OpenClaw process.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/82444 or an equivalent narrow fix with real disconnect proof, then close this issue through the merge.

## Reproduction Assessment

Yes, at source level: current main pipes the CONNECT sockets but lacks a clientSocket error listener, and Node throws unhandled 'error' events when no listener is registered. A live proof would start the debug proxy, open a CONNECT tunnel, and abruptly reset the client.

## Solution Assessment

Yes, the proposed direction is the narrow maintainable fix: handle client-side socket errors and destroy the paired upstream socket, with symmetric cleanup on upstream errors. The best current path is to review and land the already-linked PR rather than open a duplicate repair branch.
