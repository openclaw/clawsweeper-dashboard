## Summary

Keep open: current main still has the CONNECT tunnel socket-handling gap, and the same reporter has an open paired fix PR at https://github.com/openclaw/openclaw/pull/82444. The prior ClawSweeper review and a later community confirmation still match the current source.

## What This Changes

The issue reports that the debug proxy CONNECT tunnel lacks a client socket error handler and can crash or leak cleanup on abrupt client disconnects.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82444, or an equivalent narrow fix, with patched proxy proof before closing this issue.

## Reproduction Assessment

Yes, at source level: current main pipes both CONNECT sockets but only handles upstreamSocket errors, and Node throws unhandled "error" events without a listener. A live proof would start the debug proxy, establish a CONNECT tunnel, then abruptly reset the client socket.

## Solution Assessment

Yes. The narrow maintainable solution is symmetric error handling for clientSocket and upstreamSocket, with destroy() cleanup on the paired socket; the open paired PR implements that direction but still needs patched real-behavior proof.
