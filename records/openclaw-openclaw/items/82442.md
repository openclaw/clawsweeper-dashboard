## Summary

Keep open: current main still lacks a clientSocket error handler in the debug proxy CONNECT tunnel, and the issue is paired with an open closing PR. The right cleanup is to review and land that PR, or an equivalent fix, before closing this issue.

## What This Changes

The issue reports that abrupt client disconnects can emit an unhandled clientSocket error in the debug proxy CONNECT tunnel and crash the process.

## Best Possible Solution

Keep the issue open until https://github.com/openclaw/openclaw/pull/82444 is reviewed and either lands with proof or is replaced by an equivalent narrow fix that handles client socket errors in the CONNECT tunnel.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main pipes an established CONNECT tunnel without a clientSocket error listener, while Node treats unhandled EventEmitter error events as process-fatal. I did not execute the abrupt-disconnect scenario in this read-only review.

## Solution Assessment

Yes, the proposed direction is the narrow maintainable fix: add a clientSocket error handler that records the error and tears down the paired upstream socket, with regression proof for the tunnel failure mode. The issue should close only after the linked PR or an equivalent patch lands.
