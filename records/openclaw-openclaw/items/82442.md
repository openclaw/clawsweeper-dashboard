Keep open: current main still has the reported debug proxy CONNECT socket-handling gap, and https://github.com/openclaw/openclaw/pull/82444 is an open same-author fix path for the same behavior.

## What This Changes

Handle abrupt client-side disconnect errors in debug proxy CONNECT tunnels by recording the error and destroying the paired upstream socket.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82444, or an equivalent narrow fix with regression or real-behavior proof, then close this issue through the merged change.

## Reproduction Assessment

Yes, at source level: current main pipes the CONNECT sockets but only upstreamSocket has an error listener, and Node throws unhandled 'error' events without a listener. A live proof would start the debug proxy, open a CONNECT tunnel, then abruptly reset the client.

## Solution Assessment

Yes, the requested symmetric clientSocket error handler plus destroy-based paired-socket cleanup is the narrow maintainable fix. The best path is to review and land the existing paired PR rather than create a duplicate repair job.
