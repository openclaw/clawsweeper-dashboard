## Best Possible Solution

Merge the narrow payload normalization after the contributor adds redacted real OpenRouter DeepSeek V4 proof or a maintainer explicitly overrides that proof gate, then let the linked issue close through the merged PR.

## Reproduction Assessment

Yes, at source level: current main can build nested `reasoning.effort` for OpenRouter compat and then add top-level `reasoning_effort` in the DeepSeek V4 wrapper on the same payload. I did not run a live OpenRouter request in this read-only review.

## Solution Assessment

Yes, with the proof caveat: current docs/tests intentionally keep top-level `reasoning_effort` for OpenRouter DeepSeek V4, so deleting the nested equivalent in the final DeepSeek V4 wrapper is the narrowest maintainable fix. A merged-compat gate would risk dropping the existing DeepSeek V4 replay/backfill behavior that OpenRouter currently owns.
