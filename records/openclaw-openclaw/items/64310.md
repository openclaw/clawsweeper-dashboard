## What This Changes

The PR makes audit-seams collect workspace package paths per run and pass them into missing-package analysis instead of mutating a module-global array.

## Best Possible Solution

Land the scoped per-run state fix with focused regression coverage once after-fix proof demonstrates stable repeated audit output.

## Reproduction Assessment

Yes, by source inspection. Current main appends extension package paths into a module-global array on each `main()` run, then reads that accumulated array when building `missingPackages`.

## Solution Assessment

Yes. Passing a fresh sorted/de-duplicated list from `collectWorkspacePackagePaths()` into `buildMissingPackages()` removes the mutable global without changing the script contract; the redundant sort noted in bot review is harmless.
