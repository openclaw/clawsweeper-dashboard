## What This Changes

The PR makes `scripts/audit-seams.mjs` build a fresh workspace package path list per `main()` run and pass it into missing-package auditing.

## Best Possible Solution

Land the focused script-state fix after maintainer review, ideally with a small regression test in `test/scripts/audit-seams.test.ts` for repeated in-process `main()` calls.

## Reproduction Assessment

Yes. Current main has a direct static reproduction path: call exported `main()` twice in the same Node process and the collector appends extension package paths to the same module-global array before `buildMissingPackages()` reads it.

## Solution Assessment

Yes. The proposed approach is the narrowest maintainable fix: keep the base list immutable, build a fresh sorted/de-duped list per run, and thread it explicitly into the only missing-package consumer.
