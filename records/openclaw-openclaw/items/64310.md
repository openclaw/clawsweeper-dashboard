## Summary

Keep open: current main and the latest tagged release still retain the mutable module-global `workspacePackagePaths`, while the PR is a focused, plausible fix for the repeated in-process `main()` idempotency bug. This needs maintainer review and merge judgment, not cleanup closure or a replacement branch.

## What This Changes

The PR changes `scripts/audit-seams.mjs` so workspace package paths are collected from an immutable base list per run and passed into missing-package auditing instead of accumulating in module-global state.

## Best Possible Solution

Land the focused script-state fix after maintainer review, preferably with a small regression test in `test/scripts/audit-seams.test.ts` that calls `main()` twice in-process and verifies stable `missingPackages` package paths.

## Reproduction Assessment

Yes. Static inspection gives a high-confidence reproduction path: import `main()` from current main, call it twice in the same Node process, and the second call will append the extension package paths to the same module-global array again.

## Solution Assessment

Yes. The PR is the narrowest maintainable fix because it removes the mutable shared list from the repeated-run path and threads the collected list explicitly into the one consumer; the remaining redundant sort is harmless defensive ordering.
