## What This Changes

This PR localizes workspace package path collection in `scripts/audit-seams.mjs` and passes the resulting list into missing-package auditing so repeated in-process `main()` calls do not reuse mutated module state.

## Best Possible Solution

Land this narrow state-localization fix after the contributor adds real proof of repeated same-process `main()` calls and refreshes the branch against current `main`.

## Reproduction Assessment

Yes. By source inspection, importing `main()` and calling it twice in the same Node process will run `collectWorkspacePackagePaths()` twice, and current main appends extension paths into a module-global array before `buildMissingPackages()` reads it; I did not execute the script because this review is read-only.

## Solution Assessment

Yes, this is the best fix shape. Returning a fresh collected package-path list and passing it to the consumer removes the hidden module state at the producer boundary; clearing or deduping the old global later would leave the fragile stateful design in place.
