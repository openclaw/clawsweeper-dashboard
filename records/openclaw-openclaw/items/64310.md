## What This Changes

The PR makes `scripts/audit-seams.mjs` collect workspace package paths as a fresh sorted list per `main()` call and pass it into missing-package auditing instead of mutating module-global state.

## Best Possible Solution

Land the state-localizing script fix after the contributor adds real behavior proof and the branch/checks are refreshed against current main.

## Reproduction Assessment

Yes by source inspection: current main appends extension package paths into a module-global array before reading it for missing-package output, so two in-process `main()` calls can accumulate duplicate package paths. I did not execute the script in this read-only review.

## Solution Assessment

Yes: returning a freshly collected package-path list and passing it into `buildMissingPackages()` is the narrow maintainable fix; clearing or de-duping the old global at the consumer would keep hidden state in the module.
