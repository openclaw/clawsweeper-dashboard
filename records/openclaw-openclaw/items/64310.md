## What This Changes

The PR changes `scripts/audit-seams.mjs` so workspace package paths are collected from an immutable base list per `main()` invocation instead of appended to a mutable module-global array.

## Best Possible Solution

Land this narrow hidden-state fix after redacted terminal/log proof shows repeated in-process `main()` calls no longer duplicate package entries and required checks are refreshed on current `main`.

## Reproduction Assessment

Yes by source inspection: import `main()` and call it twice in the same Node process; current `main` appends extension package paths into a module-global array before reading it for `missingPackages`. I did not execute the script because this review is read-only.

## Solution Assessment

Yes: returning a freshly collected package-path list and passing it into `buildMissingPackages()` removes hidden module state at the producer/consumer boundary. Clearing or de-duping the old global at the consumer would leave the fragile module state in place.
