## What This Changes

The PR changes `scripts/audit-seams.mjs` to collect workspace package paths from a fresh per-run base list and pass that list into missing-package auditing instead of reusing a mutable module-global array.

## Best Possible Solution

Land the focused script-state fix after maintainer review, with a small regression test in `test/scripts/audit-seams.test.ts` that exercises repeated in-process `main()` calls and verifies stable `missingPackages` package paths.

## Reproduction Assessment

Yes. Current main has a direct static reproduction path: call exported `main()` twice in the same Node process and the collector appends extension package paths to the same module-global array before `buildMissingPackages()` reads it.

## Solution Assessment

Yes. The proposed fix is the narrowest maintainable direction: keep the base list immutable, build a fresh sorted/de-duped list per run, and thread it explicitly into the only missing-package consumer.
