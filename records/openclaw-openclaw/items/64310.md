## What This Changes

The branch makes audit-seams workspace package path collection local to each run and passes the collected list into missing-package analysis.

## Best Possible Solution

Land the fresh-list producer-boundary fix with a small idempotency proof or regression test, then refresh the branch so current required checks can run.

## Reproduction Assessment

Yes, from source inspection: current main keeps package paths in module-global state, `collectWorkspacePackagePaths()` appends to it, and exported `main()` can be called more than once in the same process. A local runtime probe did not execute because this checkout lacks installed `typescript`, so the reproduction is source-backed rather than run-backed.

## Solution Assessment

Yes for the code shape: returning a fresh collected list and threading it into `buildMissingPackages()` removes the hidden module state without adding API, config, or a second behavior path. The remaining blockers are proof and branch refresh, not a different implementation layer.
