## What This Changes

The PR changes `scripts/audit-seams.mjs` to collect workspace package paths per `main()` run and pass that list into missing-package analysis instead of appending into a module-global array.

## Best Possible Solution

Land the per-run state fix after redacted real-output proof shows repeated audit runs stay stable; add a focused regression test if maintainers want a durable guardrail.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: import current-main `main()` in one Node process, call it twice, and the module-global package path list can accumulate extension package entries before `missingPackages` is built.

## Solution Assessment

Yes. Passing a freshly collected package-path list into `buildMissingPackages()` removes the mutable state without changing the script contract; the redundant second sort is harmless.
