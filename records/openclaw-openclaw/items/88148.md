## Summary

This issue should stay open because current `main` and the latest release still keep bootstrap snapshots in a session-keyed `Map` without a size bound or eviction. The linked implementation PR at https://github.com/openclaw/openclaw/pull/88149 is open and unmerged, so it is the active fix path rather than close evidence.

## What This Changes

The issue requests bounding the gateway bootstrap snapshot cache so distinct session keys do not accumulate cached bootstrap bundles indefinitely.

## Best Possible Solution

Land or replace the linked bounded-cache fix with a small capacity and eviction contract in `src/agents/bootstrap-cache.ts`, plus regression coverage for churn and active-key reuse.

## Reproduction Assessment

Yes, by source inspection: current `main` stores bootstrap snapshots in a session-keyed `Map` and never evicts entries when distinct session keys are loaded. I did not run a runtime churn script during this read-only review.

## Solution Assessment

Yes, a bounded cache with deterministic eviction and focused regression coverage is the narrow maintainable fix for this cache layer. The linked open PR is the current implementation candidate to review and land or replace.
