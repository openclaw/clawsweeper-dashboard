## Summary

This issue should stay open: current `main` and the latest release still keep bootstrap snapshots in a session-keyed `Map` without a size bound, and the linked implementation PR is open and unmerged. The report is actionable, but the existing PR is the active fix path rather than close evidence.

## What This Changes

Cap the gateway bootstrap snapshot cache so session-keyed bootstrap bundles are evicted instead of retained without bound.

## Best Possible Solution

Merge or replace the linked bounded-cache implementation with a deterministic cap and eviction contract in `src/agents/bootstrap-cache.ts`, plus regression coverage for churn and active-key reuse.

## Reproduction Assessment

Yes, by source inspection: current `main` stores bootstrap snapshots in a session-keyed `Map` and never evicts entries when distinct session keys are loaded. I did not run a churn script because this is a read-only review and the source path is clear.

## Solution Assessment

Yes: a small bounded cache with deterministic eviction and focused regression coverage is the narrow maintainable fix for this cache layer. The linked open PR is the current implementation candidate to review and land or replace.
