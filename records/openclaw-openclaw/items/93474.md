## Summary

Keep open: the PR improves the env-var mirror path, but it is not merge-ready because it still misses the npmrc/userconfig registry configuration described by the linked issue and supported by npm itself.

## What This Changes

The PR changes update metadata fetches to build the OpenClaw package-target URL from `npm_config_registry` or `NPM_CONFIG_REGISTRY` and adds a focused test for that env-backed path.

## Best Possible Solution

Complete a central update metadata registry resolver that honors npm registry env plus npmrc/userconfig/home config sources, keeps the public registry as the final fallback, and covers the fetch helper with focused regression tests.

## Reproduction Assessment

Yes, source-level: current main and `v2026.6.6` build the public npm registry URL directly, while npm resolves registry values from both env and userconfig/npmrc sources. I did not run a full Gateway startup update path.

## Solution Assessment

No. The helper is the right boundary, but the env-only implementation is too narrow for the linked `npm config get registry` and `.npmrc` cases; the maintainable fix is a central resolver that covers npm's normal registry sources without adding an OpenClaw-specific option.
