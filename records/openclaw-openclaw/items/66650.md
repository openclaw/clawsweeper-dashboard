The PR adds a `maximum: 1000` schema cap for `sessions_history.limit`, clamps outgoing `chat.history` requests to 1000, and adds a schema assertion.

## Best Possible Solution

## Solution Assessment

Yes. Clamping at the `sessions_history` tool boundary is narrower than raising the Gateway protocol cap or parsing validation errors for retry, though the branch still needs real behavior proof and direct runtime coverage before merge.
