## Summary

Keep open. This MEMBER-authored PR has the protected `maintainer` label, and review found a blocking doctor-warning regression: default command fallback capability would emit future-removal warnings even when a channel has not declared transition metadata.

## What This Changes

The PR adds plugin doctor capability metadata, SDK/runtime allowFrom fallback controls, doctor warning and repair migration logic, and docs/tests for later disabling legacy allowFrom fallback families.

## Best Possible Solution

Keep the generic transition machinery, but make doctor warnings and repair fire only for fallback families explicitly declared by channel metadata so existing configs stay quiet until plugin-owned opt-outs land.

## Reproduction Assessment

Yes for the review finding: source inspection shows `collectDoctorPreviewWarnings` passes defaulted doctor capabilities into empty-allowlist scanning, and the command warning condition accepts the default true value without explicit transition metadata. I did not execute the PR branch.

## Solution Assessment

No. The generic doctor/runtime seam is the right direction, but the command fallback warning should be gated on explicit channel transition metadata or a declared migration target before merge.
