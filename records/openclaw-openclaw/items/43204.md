## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Defer TTL handler invocation before catching failures:** `src/auto-reply/reply/get-reply.ts:127`
  - body: `Promise.resolve(ttlHandler())` invokes the handler before `.catch(...)` is attached, so a synchronous throw from a custom TTL handler escapes the timer callback instead of being logged and contained. Start the promise chain with `Promise.resolve().then(() => ttlHandler())` or wrap the call in `try/catch`.
  - confidence: 0.91
- **[P2] Drop the stale Tlon Git dependency bump:** `extensions/tlon/package.json:7`
  - body: This PR is about auto-reply TTL feedback, but this line changes a GitHub-sourced Tlon dependency that current main no longer uses in `extensions/tlon/package.json`. Keep package-resolution changes out of this branch to avoid reintroducing unrelated supply-chain churn.
  - confidence: 0.86
- **[P2] Avoid flipping global connection mode during remote probe:** `apps/macos/Sources/OpenClaw/OnboardingView+Pages.swift:473`
  - body: Setting `state.connectionMode = .remote` during the onboarding probe can trigger the app-wide connection-mode observer and real lifecycle work, such as stopping the local gateway, even though the user only clicked a connection check. The probe should use an isolated endpoint/config path instead of mutating global mode.
  - confidence: 0.78
