## Summary

Keep this PR open, but it is not merge-ready: the branch flips MiniMax usage output from the documented `% left` contract to `% used`, while current fetch normalization already converts MiniMax raw remaining fields into consumed `usedPercent` before the formatter derives remaining quota. The PR also has only synthetic formatter proof and a current lint failure.

## What This Changes

The PR special-cases MiniMax provider usage formatters to display consumed percentages as `% used` and adds a standalone reproduction script.

## Best Possible Solution

Keep MiniMax on the documented `% left` quota display unless maintainers explicitly decide to change that product contract, then update code, docs, and regression tests together.

## Reproduction Assessment

Yes for the current output path: source inspection shows main formats MiniMax `usedPercent` as remaining quota via `% left`. That does not establish the reported behavior as a bug, because the MiniMax docs explicitly describe `% left` as the intended display.

## Solution Assessment

No. The PR is not the best fix because it changes the generic formatter against current docs and tests; the safer path is to preserve `% left` or make a maintainer-approved contract change with docs, tests, and real proof.
