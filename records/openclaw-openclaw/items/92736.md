## Summary

Keep open: the PR changes MiniMax from the documented normalized `% left` status display to a provider-specific `% used` formatter, but current parser code and tests already treat MiniMax count fields as remaining quota before producing `usedPercent`.

## What This Changes

This PR adds a MiniMax-specific formatter branch that renders `window.usedPercent` as `% used` and adds a synthetic repro script for the formatter output.

## Best Possible Solution

Preserve the normalized `% left` display unless redacted live MiniMax evidence proves the parser contract is wrong, then fix the parser and tests at the MiniMax fetch boundary.

## Reproduction Assessment

No for the live user issue: the formatter path is source-clear, but the real MiniMax API/dashboard semantics are still unresolved. The PR regression is source-reproducible because current parser tests normalize 98 remaining out of 100 to `usedPercent: 2`, which this PR would print as `2% used`.

## Solution Assessment

No. The formatter branch is the wrong layer for a disputed provider semantics bug; the maintainable fix is live contract verification followed by a MiniMax parser/test change if needed.
