## Summary

Keep open. The PR addresses a real missing archive-read feature, but it is not merge-safe yet: the branch bypasses current main's bounded `chat.history` reader, emits duplicate archive `seq` metadata, misses required docs/changelog coverage, and changes reset-archive visibility through the agent-facing `sessions_history` tool.

## What This Changes

The PR adds an optional `includeArchived` parameter to `chat.history` and `sessions_history`, reset-archive discovery/aggregation helpers with synthetic reset markers, tests, and regenerated Swift protocol models.

## Best Possible Solution

Land a coordinated additive archive-read path only after it preserves bounded history reads, stable metadata, archive filename coverage, public docs/release notes, and approved reset-archive visibility semantics.

## Reproduction Assessment

Yes. Static source review is enough here: current main rejects `includeArchived`, while the PR diff shows the new archive path and the duplicate per-archive sequence counter without needing a live gateway run.

## Solution Assessment

No, not as proposed. A read-only opt-in archive view is plausible, but it should be rebased onto the bounded reader path and approved for reset-archive visibility semantics before merge.
