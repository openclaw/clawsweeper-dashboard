Full review comments:

- **[P1] Gate model-facing archive recovery:** `src/agents/tools/sessions-history-tool.ts:32-36`
  - body: Adding `includeArchived` directly to the model-facing `sessions_history` schema lets any agent with that tool pull `.reset.*` transcript content back into context. Because `/reset` currently removes those turns from active history, this needs a user-intent or permission boundary before the tool can expose it.
  - confidence: 0.88
- **[P1] Bound archive reads before chaining:** `src/gateway/session-utils.fs.ts:214`
  - body: `readArchivedTranscriptMessages` reads each archive with `fs.readFileSync(...).split(...)`, and the caller loops over every reset archive before the normal `chat.history` byte/window caps run. A single `includeArchived` request can therefore block the Gateway or load very large retained histories into memory.
  - confidence: 0.86
- **[P2] Carry sequence numbers across archive segments:** `src/gateway/session-utils.fs.ts:225-233`
  - body: `messageSeq` restarts inside each archive file, and the primary segment can also start at `seq: 1`. Current history pagination and SSE state use `__openclaw.seq` as the cursor/watermark, so duplicate sequence values across the combined response can break paging and inline update state.
  - confidence: 0.82
