## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Keep archived history out of model tools by default:** `src/agents/tools/sessions-history-tool.ts:34`
  - body: Adding `includeArchived` to the model-facing `sessions_history` schema lets the agent opt itself back into `.reset.*` transcripts. Those archives are content the user moved out of the active session with `/new` or `/reset`, so archive recovery needs an explicit human/API policy boundary before it is exposed as a normal agent tool parameter.
  - confidence: 0.86
- **[P1] Bound archive reads before concatenating transcripts:** `src/gateway/session-utils.fs.ts:291-302`
  - body: `includeArchived=true` reads every reset archive with `readFileSync` and concatenates all messages before chat.history applies the requested limit or byte caps. A session with many or large reset archives can block the gateway and allocate far beyond the request budget; use the current bounded async transcript/index path or otherwise cap per segment before aggregation.
  - confidence: 0.88
- **[P2] Make archive sequence numbers monotonic:** `src/gateway/session-utils.fs.ts:221-233`
  - body: Each archive starts `messageSeq` at 0, and the primary transcript then starts its own sequence at 1, so the combined response can contain duplicate `__openclaw.seq` values. Current history and SSE code use seq values for windowing/reconciliation, so the recovery view needs monotonic combined seq values or explicit segment-scoped metadata.
  - confidence: 0.78
