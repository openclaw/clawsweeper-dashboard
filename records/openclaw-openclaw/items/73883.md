## What This Changes

The PR adds an optional includeArchived flag to chat.history and sessions_history, reset-archive discovery/aggregation with synthetic reset markers, tests, and regenerated Swift protocol models.

## Best Possible Solution

Land a coordinated additive archive-read path that preserves bounded history reads, stable transcript metadata, public docs/release notes, and approved reset-archive visibility semantics.

## Reproduction Assessment

Not applicable for the feature request itself, but yes for the merge blockers: static inspection of current main and the provided PR diff shows the missing selector on main and the PR's bounded-reader and seq issues.

## Solution Assessment

No, not as proposed. A read-only opt-in may be a maintainable direction, but the implementation should stay on an equivalent bounded reader path and document the reset-archive visibility contract before merge.
