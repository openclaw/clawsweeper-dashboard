## Summary

Keep open. The feature is not implemented on current `main` and addresses real reset-archive history gaps, but this PR is not mergeable as-is because it bypasses the current bounded `chat.history` reader, duplicates archive `seq` metadata, lacks public docs/changelog coverage, and needs maintainer approval for exposing reset archives through an agent tool.

## What This Changes

The PR adds optional `includeArchived` support to `chat.history` and `sessions_history`, plus reset-archive discovery/aggregation helpers, tests, and regenerated Swift protocol models.

## Best Possible Solution

Land a coordinated additive archive-read feature that preserves bounded async history reads, continuous metadata, generated clients, docs/changelog coverage, and maintainer-approved reset-archive privacy semantics.

## Reproduction Assessment

Yes. Static inspection is enough: current `main` rejects `includeArchived`, while the PR diff routes archive reads through a full synchronous aggregator; two reset archives would each start their local `seq` counter at 1.

## Solution Assessment

No, not as proposed. A read-only opt-in is plausible, but the mergeable version should stay on the bounded reader path and settle agent-tool reset-archive visibility before exposing it publicly.
