## Summary

Keep open. The feature is useful source work for archive recovery, but the PR is not merge-ready because it is stale against current bounded chat.history reads, duplicates transcript seq metadata across archive segments, lacks required public docs/changelog coverage, and exposes reset archives through a model-facing tool without maintainer privacy approval.

## What This Changes

The PR adds optional includeArchived support to chat.history and sessions_history, reset-archive discovery/aggregation helpers with tests, and regenerated Swift protocol models.

## Best Possible Solution

Land a coordinated additive archive-read feature only after rebasing over current bounded chat.history behavior, preserving continuous metadata, documenting the public contract, adding a changelog entry, and getting maintainer approval for agent-tool archive visibility.

## Reproduction Assessment

Yes. Static inspection is enough: current main rejects includeArchived, while the PR diff shows duplicate seq with two reset archives and a read path that bypasses current bounded chat.history reads.

## Solution Assessment

No, not as proposed. A read-only opt-in is plausible, but the mergeable solution should preserve the current bounded async reader path and make reset-archive privacy semantics explicit before exposing it through sessions_history.
