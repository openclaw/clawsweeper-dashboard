- **[P1] Gate archived history before exposing it to agents:** `src/agents/tools/sessions-history-tool.ts:31`
  - body: This adds `includeArchived` to the model-facing `sessions_history` schema and forwards it unconditionally. The PR rationale depends on a user-explicit recovery view, but an agent can now choose `includeArchived: true` itself and pull `.reset` content that `/reset` moved out of the live session. Keep this on a user-controlled RPC/UI path or add an explicit permission/user gate before exposing it as a tool parameter.
  - confidence: 0.89
- **[P1] Keep archive history reads bounded:** `src/gateway/server-methods/chat.ts:1513-1515`
  - body: The `includeArchived` branch switches `chat.history` to `readSessionMessagesIncludingArchives`, which reads every matching archive and the full primary transcript before projection and byte caps. Current main bounds history reads by messages, lines, and bytes before display projection; preserve equivalent bounds for archive segments so one RPC cannot force the Gateway to process unbounded reset history.
  - confidence: 0.86
- **[P2] Rebase the schema change onto the current protocol package:** `src/gateway/protocol/schema/logs-chat.ts:31`
  - body: This edits the old `src/gateway/protocol` schema path, but current main validates `chat.history` from `packages/gateway-protocol/src/schema/logs-chat.ts` and already added `agentId` to `ChatHistoryParams`. After rebase, add `includeArchived` to the current schema/generated Swift surface and cover it with the current protocol tests, or clients will reject the parameter or lose selected-agent history support.
