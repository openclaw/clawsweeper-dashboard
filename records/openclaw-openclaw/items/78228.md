The PR now addresses the earlier changelog and group-policy test concerns, but the new DM quiet path still counts as no final response and can finalize Telegram status reactions as an error, and the supplied real-behavior proof exercises group delivery rather than the changed DM no-response behavior.

## What This Changes

The PR gates Telegram's synthesized no-response silent-reply fallback to group chats, updates direct/group dispatcher tests, and adds a changelog fix entry.

## Best Possible Solution

Land one canonical Telegram adapter fix that suppresses synthetic DM fallback delivery, treats successful quiet DM no-response turns as handled completion, preserves real error and group fallback behavior, and closes the linked user reports after merge.

## Reproduction Assessment

Yes. Current main has a high-confidence source reproduction: a direct Telegram session with `queuedFinal: false`, no dispatch error, and no prior delivery reaches the synthetic `NO_REPLY` branch, and the current test expects a rewritten visible reply.

## Solution Assessment

No as submitted. The `isGroup` guard is the right narrow boundary, but the quiet DM path must also be counted as handled/done and the PR needs real DM proof or a maintainer-accepted substitute before merge.
