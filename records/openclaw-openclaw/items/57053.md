## Summary

Keep open. Current main and the latest shipped release still expose `session_status` as rendered status text plus minimal metadata, while the requested structured remaining time/token budget contract remains absent and is only partially adjacent to the open token-usage API request.

## What This Changes

Request to expose remaining session budget, such as remaining seconds and tokens, through `session_status` or a dedicated lightweight budget tool for multi-agent orchestration.

## Best Possible Solution

Define an additive documented agent-facing budget contract on `session_status.details` or a dedicated budget tool, backed by authoritative time/token accounting plus visibility-aware tests and docs.

## Reproduction Assessment

Not applicable as a bug reproduction; yes as a source-level feature-gap check. Current main and `v2026.5.20` accept only `sessionKey`/`model` and return rendered `statusText` without structured remaining-budget fields.

## Solution Assessment

Unclear until maintainers choose the public API shape and semantics. Structured budget data is a maintainable direction, but the exact fields, accounting sources, and visibility rules need product/API judgment first.
