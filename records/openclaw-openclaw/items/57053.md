## Summary

Keep open. Current main and the latest release still expose `session_status` as rendered status text plus minimal metadata, while the requested structured remaining time/token budget contract is absent; the related open usage and capacity issues are adjacent but do not own the combined remaining-budget preflight request.

## What This Changes

The issue requests structured remaining session budget data from `session_status` or a new lightweight `session_budget` tool.

## Best Possible Solution

Define an additive documented budget contract on `session_status.details` or a new supported budget tool, backed by authoritative time and token accounting with visibility-aware tests and docs.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and docs inspection on current main and v2026.5.18 give a high-confidence behavior check: `session_status` returns rendered `statusText` and does not expose structured budget fields.

## Solution Assessment

Unclear until maintainers choose the public API shape and semantics. Structured budget data is a maintainable direction, but the exact fields, accounting sources, and visibility rules should be settled first.
