## Summary

Keep open. Current `main` still exposes `session_status` as a formatted status card plus minimal structured metadata, and it does not provide stable `remaining_seconds`, `remaining_tokens`, elapsed, timeout, or budget fields for skill orchestration. The related #57705 is adjacent sub-agent capacity work, but it does not supersede this distinct time/token budget request.

## What This Changes

The issue requests structured remaining session budget data from `session_status`, or a new lightweight `session_budget` tool, so skills can preflight expensive multi-agent workflows.

## Best Possible Solution

Define a stable structured budget contract, likely `session_status.details.budget` or a dedicated `session_budget` tool, backed by existing context/token accounting plus a documented current-run deadline source. Add focused tests for the returned fields and update the session tools/token-use docs so skill authors can rely on the contract.

## Reproduction Assessment

Not applicable as a feature request, but source inspection gives a high-confidence current-behavior check: `session_status` currently returns formatted text and minimal metadata, not the requested structured budget fields.

## Solution Assessment

Unclear. Structured budget data is a maintainable direction, but the best surface and exact time/token semantics should be settled before a repair worker implements a public tool contract.
