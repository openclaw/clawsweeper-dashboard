Request to expose remaining session time and token budget as structured data through `session_status` or a new lightweight `session_budget` tool.

## Best Possible Solution

Define an additive documented budget contract on `session_status.details` or a dedicated budget tool, backed by authoritative time/token accounting plus visibility-aware tests and docs.
