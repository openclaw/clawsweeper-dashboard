## Summary

Keep open. Current main and the latest release still do not expose structured remaining time/token budget data through `session_status`; the adjacent context-budget work is reporting-only and the open token-usage request does not cover the `remaining_seconds` part of this request.

## What This Changes

Request an agent-facing structured session budget contract, either by extending `session_status` with remaining time/token fields or adding a dedicated lightweight budget tool.

## Best Possible Solution

Define an additive documented agent-facing budget contract on `session_status.details` or a dedicated budget tool, reusing transcript, gateway usage, and context-budget sources with explicit source, availability, and privacy semantics.

## Reproduction Assessment

Yes as a source-level feature-gap check, not as a bug reproduction. Current main and `v2026.5.22` accept only `sessionKey` and `model` and return rendered `statusText` without structured remaining-budget fields.

## Solution Assessment

Unclear until maintainers choose the public API shape and semantics. Structured budget data is a maintainable direction, but the exact fields, accounting sources, and visibility rules need product/API judgment first.
