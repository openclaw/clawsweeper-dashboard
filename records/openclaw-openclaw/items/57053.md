## Summary

Keep open. Current main still only returns a status-card response plus minimal metadata from `session_status`; source, docs, and exact-field search do not show the requested structured remaining time/token budget. The open #57705 is adjacent sub-agent capacity work, not a replacement for this time/token budget contract.

## What This Changes

Expose remaining session time and token budget as structured data through `session_status` or a dedicated `session_budget` tool so skills can preflight expensive orchestration.

## Best Possible Solution

Define a stable structured budget contract, likely `session_status.details.budget` or a dedicated `session_budget` tool, backed by authoritative current-run deadline data plus existing context/token accounting. Document the semantics and add focused tests for the returned fields before skills depend on them.
