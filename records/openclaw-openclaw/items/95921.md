## What This Changes

The issue asks to fix `openclaw commitments` text-table truncation so long IDs or scopes do not overflow fixed-width columns.

## Best Possible Solution

Land one narrow fix that keeps truncated cells within their column width, preferably by using the existing single-character ellipsis convention with regression coverage for long ID and scope alignment.

## Reproduction Assessment

Yes, at source level. The current helper returns an 18-character ID for a 16-character column and then passes it through `padEnd(16)`, though I did not run a seeded full CLI store in this read-only review.

## Solution Assessment

Yes. Switching to the existing one-character ellipsis convention is the narrowest maintainable fix; reserving three characters for ASCII `...` would also fit the width but would diverge from sibling CLI helpers and show less content.
