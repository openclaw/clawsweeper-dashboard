## Summary

Keep open: current main and the latest release still contain the three-character ellipsis overflow in `openclaw commitments`; two open PRs already target this exact issue, so the next step is to review and land one fix rather than close the issue early.

## What This Changes

The issue reports that `openclaw commitments` misaligns fixed-width table columns when long IDs or scopes are truncated because the helper appends a three-character ellipsis.

## Best Possible Solution

Land one narrow commitments-command fix that uses a single-character ellipsis and adds regression coverage for long ID and scope table alignment.

## Reproduction Assessment

Yes. The current helper and a focused Node check show that a long ID produces an 18-character value for a 16-character cell, shifting the following column by two positions.

## Solution Assessment

Yes. Matching sibling CLI helpers by using `…` is the narrow maintainable fix; it changes only truncated display text and should be paired with regression coverage for ID and scope alignment.
