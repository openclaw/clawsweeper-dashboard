## What This Changes

Request to pass through upstream-returned image-generation usage/cost metadata from OpenAI-compatible/LiteLLM, direct OpenAI/Azure, Google, fal/Flux, and MAI routes without synthesizing fake usage.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/85466 with provider-local passthrough into the existing `ImageGenerationResult.metadata` bag, preserving only upstream-returned metadata and no synthesized usage.

## Reproduction Assessment

Yes, at source level: current main forwards `ImageGenerationResult.metadata`, but the checked provider paths do not populate response-level OpenAI-compatible usage, direct OpenAI usage, Google `usageMetadata`, or the named fal response fields. I did not run live generations in this read-only review.

## Solution Assessment

Yes, with maintainer acceptance of the additive metadata contract: provider-local passthrough into the existing metadata bag is the narrowest maintainable solution and avoids synthesized token counts. The broader closed analytics/storage PR is not the right replacement path.
