## Summary

Keep open: the PR is still not merge-ready because the new plugin-resolution diagnostic loop evaluates the cached descriptor wrapper instead of the actual tool descriptor, and the registration path still drops plugin-authored availability before diagnostics can see empty groups.

## What This Changes

Adds an optional `onHiddenDiagnostic` planner callback, calls it for hidden planner diagnostics, adds planner unit coverage, and attempts to warn from plugin tool resolution.

## Best Possible Solution

Preserve or validate plugin-authored availability at the descriptor-registration or plugin-resolution boundary, emit only malformed authoring diagnostics there, and keep the planner callback only if maintainers want that exported API.

## Reproduction Assessment

Yes. Source inspection shows captured plugin descriptors still omit `availability`, and missing availability evaluates as available, so the new plugin-resolution warning path will not reproduce the requested empty-group warning.

## Solution Assessment

No. The callback is a plausible support hook, but the current integration evaluates the wrong object after the relevant metadata is dropped; the best fix is a deliberate registration or plugin-resolution diagnostic boundary.
