## What This Changes

The branch adds an optional `BuildToolPlanOptions.onHiddenDiagnostic` callback, invokes it when availability diagnostics hide a descriptor, and adds a planner unit test.

## Best Possible Solution

Surface `unsupported-signal` authoring diagnostics once at descriptor registration or plugin/tool resolution, and keep a planner callback only if a real owner wires and wants that API.

## Reproduction Assessment

Yes. Current-main source shows empty availability groups produce `unsupported-signal` diagnostics and `buildToolPlan` only stores them in `hidden`; search found no production caller that would emit the new callback.

## Solution Assessment

No. The callback is a plausible helper, but the best fix is to surface malformed descriptor diagnostics at a lifecycle-owned registration or plugin resolver boundary before claiming the linked bug is fixed.
