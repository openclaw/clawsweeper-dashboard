## What This Changes

The PR adds an optional `BuildToolPlanOptions.onHiddenDiagnostic` callback, invokes it from `buildToolPlan`, adds planner test coverage, and adds a plugin-resolution warning loop for hidden availability diagnostics.

## Best Possible Solution

Preserve or validate plugin-authored availability at descriptor capture or registration, warn once for malformed authoring diagnostics, and keep the planner callback only if maintainers want that exported API.

## Reproduction Assessment

Yes. Source inspection shows the PR loop iterates `CachedPluginToolDescriptor` wrappers, while current capture omits plugin-authored `availability` and missing availability evaluates as available, so the claimed plugin warning path cannot work as written.

## Solution Assessment

No. The callback is a plausible support hook, but the best fix for the linked behavior is at descriptor registration or plugin resolution before availability is dropped, with warning filtering for malformed authoring diagnostics.
