## Summary

Keep open: this PR adds a useful planner hook, but it does not yet fix the linked observable bug because no current production caller wires the callback and the plugin registration path still drops tool-authored availability before diagnostics can be surfaced.

## What This Changes

Adds an optional `onHiddenDiagnostic` callback to `BuildToolPlanOptions`, invokes it for hidden planner entries, and covers the callback with a planner unit test.

## Best Possible Solution

Surface `unsupported-signal` authoring diagnostics once at the descriptor-registration or plugin/tool resolution boundary while keeping the evaluator pure; keep this callback only if a real owner wires and documents it.

## Reproduction Assessment

Yes. Current-main source shows empty availability groups return `unsupported-signal` diagnostics and `buildToolPlan` only records them in `hidden`; search found no production planner caller that would emit them.

## Solution Assessment

No. The callback is a plausible helper, but the best fix is to surface malformed descriptor diagnostics at a lifecycle-owned registration or plugin resolver boundary before claiming the linked bug is fixed.
