## Summary

Keep open: current main still lacks the Feishu card-update feature, but this PR is not mergeable as-is because the update helper skips supported `open_message_id` callback shapes, the contributor has not provided real Feishu proof, and the branch includes unrelated Feishu reply-dispatcher/test churn.

## What This Changes

The PR adds a Feishu `uc` update-card payload and handler-side card patching for card actions, parses `message_id`, and also changes reply-dispatcher session-token footer plumbing while deleting monitor comment-handler tests.

## Best Possible Solution

Rebase this into a focused Feishu change that keeps `uc`/`updateCard` support, normalizes the target card id across `message_id`, `open_message_id`, and `context.open_message_id`, adds targeted tests for cancel/confirm/quick/malformed/missing-id cases, removes unrelated churn, and includes redacted live Feishu proof.

## Reproduction Assessment

Yes. Source inspection shows current main lacks `uc` and never calls `updateCardFeishu`; the PR-specific regression is also source-reproducible because the helper only accepts `message_id` while current parser/tests support `open_message_id` callbacks.

## Solution Assessment

No. The implementation direction is reasonable, but the narrow maintainable solution must reuse the existing callback target fallback and keep unrelated Feishu footer/session-store and monitor-comment test changes out of this feature PR.
