## Summary

Keep open. The Feishu card-update direction is useful, but this PR is not mergeable as-is because it misses existing callback card-id targets, adds a banned broad SDK runtime import, lacks real Feishu behavior proof, and overlaps unsettled Feishu card-update work.

## What This Changes

The branch adds a Feishu `uc` update-card envelope field, parses card-action `message_id`, patches cards from `handleFeishuCardAction`, threads session footer data through reply dispatch, and removes some comment monitor test coverage.

## Best Possible Solution

Replace or rework this as a focused Feishu plugin change that uses callback-provided card ids, supported runtime seams, targeted regression coverage, and redacted live Feishu proof while coordinating with the open card-update threads.

## Reproduction Assessment

Yes, source-level. The PR patch checks only `event.message_id` for updates while current main accepts `open_message_id` callback targets, and the added broad SDK import matches the repository boundary guard.

## Solution Assessment

No. The feature direction is plausible, but this branch is not the best mergeable solution until callback id normalization, SDK import cleanup, scope cleanup, live proof, and overlap with related Feishu card-update work are resolved.
