The branch adds a Feishu `uc` update-card envelope field, parses top-level `message_id`, calls `updateCardFeishu` around card actions, threads session footer data through reply dispatch, and removes some comment monitor test coverage.

## Best Possible Solution

## Reproduction Assessment

Yes, source-level. PR head checks only `event.message_id` for updates while current main accepts callback `open_message_id` targets, and the added broad SDK import matches the repository boundary guard.

## Solution Assessment

No. The feature direction is useful, but this branch is not the best mergeable solution until callback id normalization, SDK import cleanup, scope cleanup, live proof, and overlap with related Feishu card-update work are resolved.
