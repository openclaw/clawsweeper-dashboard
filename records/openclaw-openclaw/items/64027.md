## Summary

Keep open: current main does not implement Feishu `uc` card updates, but this PR is not mergeable as-is because it only patches `event.message_id`, adds a banned broad plugin SDK import, lacks live Feishu proof, and overlaps the unmerged implementation at https://github.com/openclaw/openclaw/pull/56435.

## What This Changes

The PR adds a Feishu card-action `uc` update-card payload, parses a top-level `message_id`, calls `updateCardFeishu` around approval/quick actions, and changes reply-dispatcher footer/session plumbing while deleting comment monitor tests.

## Best Possible Solution

Land a focused Feishu plugin implementation that uses the clicked callback card id consistently, keeps plugin SDK imports on supported narrow/runtime seams, restores unrelated test churn, adds targeted regression coverage, and includes redacted live Feishu proof.

## Reproduction Assessment

Yes, source-level. The PR helper only checks `event.message_id` while current main already supports card callbacks carrying only top-level or context `open_message_id`, and the PR also introduces a broad SDK import that the repository guard flags.

## Solution Assessment

No. The feature direction is reasonable, but this branch needs callback-id normalization, SDK import cleanup, scope cleanup, and real Feishu proof before it is the best mergeable solution.
