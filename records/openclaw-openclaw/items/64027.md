## Summary

Keep open: current main still lacks the requested Feishu in-place card-update feature, but this PR is not merge-ready because it skips currently supported `open_message_id` callback targets and lacks contributor-supplied live Feishu proof.

## What This Changes

The branch adds a Feishu card-action `uc` update-card payload and handler patch call, parses top-level `message_id`, and also changes reply-dispatcher footer/session plumbing while deleting monitor comment-handler test coverage.

## Best Possible Solution

Rebase into a focused Feishu plugin change that normalizes the card patch target across `message_id`, `open_message_id`, and `context.open_message_id`, restores unrelated test coverage/churn, adds targeted card-update tests, and includes redacted live Feishu proof.

## Reproduction Assessment

Yes, source-level. Current main lacks `uc`/`updateCard`, and PR head source shows the new patch helper only accepts `event.message_id` while current main supports `open_message_id` callback shapes.

## Solution Assessment

No. The implementation direction is plausible, but the narrow maintainable solution must preserve current callback id normalization and keep unrelated Feishu reply-dispatcher/test churn out of this feature PR.
