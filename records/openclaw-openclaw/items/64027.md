Keep open. The branch adds a Feishu `uc` card-action update payload, parses `message_id`, updates cards on cancel/confirm/quick actions, and also changes Feishu reply footer session-store behavior while deleting comment-monitor handler tests.

## Best Possible Solution

Land a focused Feishu plugin implementation that normalizes the original card target across callback shapes, uses existing runtime/session seams, keeps unrelated reply-footer and test churn out, coordinates overlapping card-update work, and includes redacted live Feishu proof.
