## Summary

Keep open: current main does not implement in-place Feishu card updates, and this is a plausible feature path, but the branch still has a source-level callback target blocker and lacks real Feishu behavior proof.

## What This Changes

The branch adds a Feishu card-action `uc` update-card payload, parses a top-level `message_id`, calls `updateCardFeishu` around button actions, and also changes Feishu reply-dispatcher footer/session plumbing while deleting monitor comment-handler test coverage.

## Best Possible Solution

Rebase into a focused Feishu plugin change that normalizes the card patch target across `message_id`, `open_message_id`, and `context.open_message_id`, restores unrelated test churn, adds targeted card-update tests, and includes redacted live Feishu proof.

## Reproduction Assessment

Yes, source-level. Current main supports `open_message_id` callback shapes, while the PR helper only checks `event.message_id`, so callbacks with only `open_message_id` would skip the in-place card patch.

## Solution Assessment

No. The feature direction is reasonable, but the maintainable fix needs to preserve current callback id normalization and keep unrelated Feishu reply-dispatcher and comment-test churn out of this branch.
