## Summary

Keep open. Current main still source-reproducibly drops `event.messages` pushed by internal `message:received` and `message:sent` hooks, and open maintainer-labeled PR #41375 is the active closing path for this bug.

## What This Changes

Internal hook messages pushed from `message:received` and `message:sent` handlers should be delivered to users instead of silently disappearing.

## Best Possible Solution

Land PR #41375 or an equivalent maintainer-approved patch that drains `message:received` replies only when a safe route exists and updates hook docs to define non-replyable lifecycle hooks.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to register a `message:received` internal hook that pushes to `event.messages` and send an inbound routed message; current main fire-and-forgets the hook and never drains the mutated event.

## Solution Assessment

Yes. The active PR #41375 direction is the narrowest maintainable fix because it delivers replies only on replyable surfaces and clarifies docs instead of making every lifecycle hook auto-reply.
