Keep open: current main and the latest release still trigger message:received internal hooks without draining event.messages, and the active fix is the open maintainer PR at https://github.com/openclaw/openclaw/pull/41375. ## What This Changes

The issue reports that internal hook replies pushed into event.messages by message:received/message:sent hooks are not sent back to the user.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/41375, or an equivalent maintainer-approved current-main repair, that drains message:received hook messages through the existing reply target and send-policy path while leaving message:sent lifecycle-only unless maintainers explicitly expand that contract.

## Reproduction Assessment

Yes. Current main creates and fires the message:received internal hook event without retaining it, so a handler-pushed event.messages value has no source path to delivery.

## Solution Assessment

No, not complete on main. The narrow replyable-surface fix direction is appropriate, but the open PR still needs maintainer repair for targeting, send-policy, and hardening blockers before it can close this issue.
