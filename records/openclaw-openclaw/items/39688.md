Keep open: current main and the latest release still drop `message:received` hook replies at source level, and the same-repo pull request that explicitly closes this issue is still open and needs maintainer follow-up before the tracker can close.

## What This Changes

The issue asks for internal hook replies pushed into `event.messages` from `message:received` and `message:sent` hooks to be sent back to the user.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/41375 with a current-main fix that drains `message:received` hook replies through existing reply routing and send-policy checks, while keeping `message:sent` lifecycle-only unless maintainers explicitly expand that contract.

## Reproduction Assessment

Yes, at source level for `message:received`: current main fires the hook without retaining the event, so handler-pushed `event.messages` are dropped. The `message:sent` part is currently documented as lifecycle-only, so it is not a strict bug without a maintainer contract change.

## Solution Assessment

Yes for keeping the issue open until the linked PR lands or is replaced. The linked PR's replyable-surface boundary appears safer than automatically sending replies from lifecycle-only `message:sent` hooks.
