## Summary

Keep open. Current main and v2026.6.10 still leave bundled session-memory on the command-only reset path for automatic daily/idle rollovers, and https://github.com/openclaw/openclaw/pull/71792 explicitly closes this issue but remains open and conflicting, so this should stay open until maintainers land, replace, or reject the reset-persistence contract.

## What This Changes

Request a session reset or before-reset hook event so bundled session-memory and custom hooks can persist context before automatic daily or idle session rollovers, with a secondary hook-config schema mismatch also reported.

## Best Possible Solution

Land, replace, or explicitly reject one canonical implementation that persists session-memory on automatic daily/idle rollover without surprising existing command hooks or losing lifecycle-hook behavior.

## Reproduction Assessment

Yes. At source level, current main can replace stale daily/idle sessions while bundled session-memory still handles only internal command:new and command:reset events; I did not run a live rollover because this review is read-only.

## Solution Assessment

Unclear pending maintainer product/API choice. The gap is real, but the best permanent fix depends on whether maintainers want command-compatible reset dispatch, before_reset extension, a new typed reset event, or session_end-based memory behavior.
