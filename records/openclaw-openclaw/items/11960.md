## Summary

Keep open. Current main still leaves automatic daily/idle stale rollovers on the lifecycle-only path, while open PR #71792 explicitly closes this issue and PR #61675 overlaps, so maintainers should review or supersede those implementations before closing this issue.

## What This Changes

The issue requests a reset-time hook surface so session-memory and custom hooks can persist context before automatic daily or idle session rollovers clear the previous session.

## Best Possible Solution

A maintainer-approved patch should make automatic daily and idle stale rollovers invoke the same effective pre-reset persistence surface as manual resets, pass previous transcript/reason context to session-memory and custom hooks, and add daily plus idle regression coverage.

## Reproduction Assessment

Yes. Configure a stale daily or idle session and send the next non-system message: current main creates a new session, archives the previous transcript, emits lifecycle hooks, and leaves session-memory gated to command:new/command:reset.

## Solution Assessment

Unclear. The requested direction is valid, but the exact implementation should be selected from PR #71792, PR #61675, or a maintainer-approved replacement so OpenClaw does not gain competing reset hook semantics.
