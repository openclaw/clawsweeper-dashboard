## Summary

Keep open. Current main and v2026.5.12 still leave automatic daily/idle rollovers on the lifecycle-only path while bundled session-memory only listens to command:new and command:reset; open implementation PRs already cover this issue, so another repair lane would duplicate maintainer selection.

## What This Changes

Request a session reset hook surface so automatic daily/idle resets can persist memory before clearing the previous session.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/71792, https://github.com/openclaw/openclaw/pull/61675, or a maintainer-approved replacement that persists session-memory exactly once for automatic daily and idle rollovers while preserving manual reset and lifecycle semantics.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure an existing session to become stale by daily or idle policy and send the next inbound message; current main creates a new session and lifecycle hooks while session-memory remains limited to command:new and command:reset.

## Solution Assessment

Unclear. The persistence gap is real, but the exact hook/API contract needs maintainer selection because open branches propose different command, before_reset, and session-event shapes.
