## Summary

Keep open. Current main still leaves automatic daily and idle session rollovers on the lifecycle-only path for reset persistence, and open implementation PRs already cover this issue for maintainer review.

## What This Changes

The issue requests a session reset hook or equivalent persistence surface so bundled session-memory and custom hooks can save context when daily or idle auto-resets clear a session.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/71792, https://github.com/openclaw/openclaw/pull/61675, or a maintainer-approved replacement that gives automatic daily and idle rollovers the same effective pre-reset memory persistence surface as manual resets with regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure a stale daily or idle session, send the next inbound message, and current main rotates through archive plus session_end/session_start while session-memory remains limited to command:new and command:reset.

## Solution Assessment

Unclear. The persistence gap is real, but maintainers should choose a coherent reset-hook contract from the open implementations or a replacement before changing this hook surface.
