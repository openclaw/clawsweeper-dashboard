## Summary

Keep open: current main and the latest release still do not make automatic daily/idle rollovers invoke the same memory-persistence surface as manual resets, and the open candidate implementations still need maintainer selection rather than another cleanup close.

## What This Changes

Request a session reset or before-reset hook surface so session-memory and custom hooks can save context before automatic daily/idle resets, with the secondary per-hook config schema mismatch noted in the report.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/71792, https://github.com/openclaw/openclaw/pull/61675, or a maintainer-approved replacement that provides one canonical pre-reset persistence contract for daily and idle rollovers with regression coverage.

## Reproduction Assessment

Yes, source-level with high confidence: current main creates new sessions for stale daily/idle rollover and emits only session_end/session_start, while session-memory only handles command:new and command:reset. I did not run a live rollover because this review is read-only.

## Solution Assessment

Unclear as an implementation choice: the persistence gap is real, but the best permanent hook shape still needs maintainer selection between the open branches or a replacement lifecycle contract.
