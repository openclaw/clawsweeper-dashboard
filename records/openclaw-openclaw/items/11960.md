## Summary

Keep open: current main and the latest release still leave automatic daily/idle rollover on the session lifecycle-only path, while bundled session-memory still only accepts manual command events. Two open PRs target this gap, so the right maintainer action is to choose, land, or replace that implementation rather than close the issue or queue another fix.

## What This Changes

The issue requests a session reset hook/event so session-memory and custom hooks can persist context before automatic daily or idle session resets, with hook-specific config schema support as a secondary concern.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/71792, https://github.com/openclaw/openclaw/pull/61675, or a maintainer-approved replacement that gives automatic daily/idle rollovers the same effective pre-reset persistence opportunity as manual resets with no-double-fire coverage.

## Reproduction Assessment

Yes, at source level with high confidence: current main captures the previous session during stale daily/idle rollover but only emits typed lifecycle hooks, while bundled session-memory ignores non-command events. I did not run a live rollover because this is a read-only review.

## Solution Assessment

Unclear as a concrete implementation choice. A reset persistence hook is directionally appropriate, but maintainers should choose the canonical public hook or memory-finalization shape before landing one of the open PRs or a replacement.
