## Summary

Keep open. Current main still lacks a reset-persistence hook path for automatic daily/idle rollovers, and https://github.com/openclaw/openclaw/pull/71792 explicitly closes this issue while overlapping implementation PRs remain open.

## What This Changes

Request automatic daily/idle session resets to fire a reset hook so session-memory and custom hooks can persist previous context before rollover.

## Best Possible Solution

Land one maintainer-approved implementation that gives automatic daily and idle rollovers the same effective pre-reset memory persistence surface as manual resets, with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure a stale daily or idle session, send the next inbound message, and current main rotates through archive plus session_end/session_start while session-memory remains limited to command:new and command:reset.

## Solution Assessment

Unclear. The persistence gap is real, but maintainers should choose one coherent reset-hook contract from the open implementations or a replacement before changing this surface.
