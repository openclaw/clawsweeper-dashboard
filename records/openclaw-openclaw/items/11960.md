## Summary

Keep open: the primary automatic daily/idle reset persistence gap still exists on current main and in v2026.6.5. The secondary hook-config schema mismatch is fixed, but automatic rollovers still emit only session lifecycle hooks while bundled session-memory still listens only to command:new and command:reset; open linked PRs need maintainer contract selection before this can close.

## What This Changes

Request a session reset or before-reset hook surface so bundled and custom memory hooks can persist previous session context during automatic daily or idle resets, plus hook-specific config schema support.

## Best Possible Solution

Ship one canonical reset-persistence contract that persists bundled session-memory exactly once for automatic daily and idle rollovers, preserves manual reset and transcript archive behavior, documents the chosen hook shape, and adds focused daily plus idle regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure daily or idle reset, let an existing session become stale, then send a normal message; initSessionState creates a new session and emits lifecycle hooks only, while session-memory ignores those events.

## Solution Assessment

Unclear. The persistence gap is real, but the issue asks for a new hook/lifecycle contract and overlapping open PRs propose different surfaces, so maintainers should select the canonical approach before merge.
