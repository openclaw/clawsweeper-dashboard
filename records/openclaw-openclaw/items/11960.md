## Summary

Keep open. Current main still captures the old session on daily/idle rollover but only emits lifecycle hooks there, while the session-memory hook remains gated to manual command events and open PR #71792 explicitly closes this issue with overlapping PR #61675 still open.

## What This Changes

Request a session reset hook surface so daily and idle auto-resets can persist memory before the old session context is cleared.

## Best Possible Solution

Land PR #71792, PR #61675, or a maintainer-approved replacement that gives automatic daily/idle rollovers the same effective pre-reset memory persistence surface as manual resets with daily and idle regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure a stale daily or idle session, send the next user message, and current main archives/rotates through lifecycle hooks while session-memory remains restricted to command:new and command:reset.

## Solution Assessment

Unclear. The requested behavior is valid, but maintainers should choose between the open implementations or a replacement so OpenClaw does not gain competing reset hook semantics.
