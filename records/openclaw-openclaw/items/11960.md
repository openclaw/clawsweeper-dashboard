## Summary

Keep open. Current main still leaves automatic daily/idle rollovers on the lifecycle-only path while bundled session-memory only handles command:new and command:reset; an open PR explicitly closes this issue, with other overlapping implementation PRs still awaiting maintainer selection.

## What This Changes

Add a session reset hook surface so automatic daily and idle session rollovers can persist memory before the previous session is cleared.

## Best Possible Solution

Land one maintainer-approved implementation, likely https://github.com/openclaw/openclaw/pull/71792 or a selected replacement over the overlapping PRs, that gives daily and idle rollovers the same effective pre-reset memory persistence surface as manual resets with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure a stale daily or idle session, send the next inbound message, and current main rotates/archive plus session_end/session_start while session-memory remains limited to command:new and command:reset.

## Solution Assessment

Unclear. The persistence gap is real, but the exact hook contract is a maintainer product/API decision because open PRs propose different command, before_reset, and session-event shapes.
