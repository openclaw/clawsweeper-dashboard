## Summary

Keep open. Current main still leaves automatic daily and idle rollovers on the lifecycle-only path for reset persistence, and https://github.com/openclaw/openclaw/pull/71792 explicitly closes this issue while https://github.com/openclaw/openclaw/pull/61675 overlaps the same hook gap.

## What This Changes

The issue requests a reset lifecycle hook or equivalent session-memory persistence surface for automatic daily and idle resets, plus permissive per-hook config schema support.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/71792, https://github.com/openclaw/openclaw/pull/61675, or a maintainer-approved replacement that gives automatic daily/idle rollovers the same effective pre-reset memory persistence surface as manual resets with daily and idle regression coverage.
