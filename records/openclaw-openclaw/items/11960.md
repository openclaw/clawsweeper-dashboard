## Summary

Keep open. Current main still leaves automatic daily/idle rollovers on the lifecycle-only path for memory persistence, while open PR #71792 explicitly closes this issue and PR #61675 overlaps the same hook gap.

## What This Changes

The issue requests a session reset hook surface so session-memory and custom hooks can persist context when daily or idle automatic session resets occur.

## Best Possible Solution

Review and land PR #71792, PR #61675, or a maintainer-approved replacement that gives automatic daily/idle rollovers the same effective pre-reset memory persistence surface as manual resets, with daily and idle regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure a stale daily or idle session, send the next inbound message, and current main rotates through archive plus session_end/session_start while session-memory remains limited to command:new and command:reset.

## Solution Assessment

Unclear. The requested persistence gap is real, but maintainers should choose one coherent reset-hook contract from the open implementations or a replacement before changing the public surface.
