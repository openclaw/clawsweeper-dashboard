## Summary

Keep open. Current main still routes automatic daily and idle rollovers through transcript archiving plus session lifecycle hooks while the bundled session-memory path remains limited to command new/reset events, and https://github.com/openclaw/openclaw/pull/71792 already explicitly closes this issue.

## What This Changes

Request a reset-time hook/event so session-memory and custom hooks can persist context before automatic daily or idle session rollover, with hook-specific config schema support.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/71792, https://github.com/openclaw/openclaw/pull/61675, https://github.com/openclaw/openclaw/pull/50933, or a maintainer-approved replacement that gives automatic daily and idle rollovers the same effective pre-reset memory persistence surface as manual resets with regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure a stale daily or idle session, send the next inbound message, and current main rotates through archive plus session_end/session_start while session-memory remains limited to command:new and command:reset.

## Solution Assessment

Unclear. The persistence gap is real, but maintainers should choose one coherent reset-hook contract from the open implementations or a replacement before changing this surface.
