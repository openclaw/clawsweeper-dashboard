## Summary

Keep open. Current main still leaves automatic daily/idle rollover on the lifecycle-only path while bundled session-memory only accepts command:new and command:reset; the secondary hook-config schema mismatch is fixed, but the primary reset persistence surface is still unresolved and already has open implementation PRs awaiting maintainer selection.

## What This Changes

Requests a session reset or before-reset hook surface so session-memory and custom hooks can persist context before automatic daily or idle resets, plus hook-specific config schema support.

## Best Possible Solution

Land one maintainer-approved implementation, likely https://github.com/openclaw/openclaw/pull/71792 or a selected replacement, that makes automatic daily/idle rollovers invoke the same effective pre-reset persistence surface as manual resets with focused daily and idle regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure a stale daily or idle session, send the next inbound message, and current main creates a new session plus session_end/session_start while session-memory remains limited to command:new and command:reset.

## Solution Assessment

Unclear. The persistence gap is real, but the exact hook contract is a maintainer product/API decision because open branches propose different command, before_reset, and session-event shapes.
