## Summary

Keep open. Current main still does not route automatic daily or idle stale rollovers through the reset persistence path consumed by session-memory; it only fixes the secondary hook-config schema mismatch. Because open PR #71792 explicitly closes this issue and PR #61675 overlaps, maintainers should review or supersede those implementations before this issue is closed.

## What This Changes

Request to add a session reset or before-reset hook surface for automatic daily/idle resets, and to allow hook-specific session-memory configuration keys.

## Best Possible Solution

A maintainer-approved patch should make automatic daily and idle stale rollovers invoke the same effective pre-reset persistence surface as manual resets, pass previous session entry/transcript data and reset reason to session-memory and custom hooks, preserve typed lifecycle hooks, and add daily plus idle regression coverage.

## Reproduction Assessment

Yes. Configure a stale daily or idle session and send the next non-system message: current main creates a new session with resetTriggered false, archives the previous transcript, emits lifecycle hooks, and leaves session-memory gated to command:new/command:reset.

## Solution Assessment

Unclear. The requested direction is valid, but the best implementation should be selected from PR #71792, PR #61675, or a maintainer-approved replacement so the repo does not gain competing reset hook semantics.
