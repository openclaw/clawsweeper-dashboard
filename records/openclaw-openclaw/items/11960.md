## Summary

Keep open. Current main still captures stale daily/idle rollover state but only emits session lifecycle hooks, while bundled session-memory still listens only to command:new and command:reset; https://github.com/openclaw/openclaw/pull/71792 explicitly closes this issue and https://github.com/openclaw/openclaw/pull/61675 overlaps, so this should wait for maintainer selection and merge/rejection of the open implementation path.

## What This Changes

The issue requests a reset lifecycle hook for automatic daily/idle session resets so session-memory and custom hooks can persist context before rollover, plus schema support for hook-specific hook config keys.

## Best Possible Solution

Land one maintainer-approved implementation, likely https://github.com/openclaw/openclaw/pull/71792 or a replacement chosen over https://github.com/openclaw/openclaw/pull/61675, that gives automatic daily and idle rollovers the same effective pre-reset memory persistence surface as manual resets with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure a stale daily or idle session, send the next inbound message, and current main rotates through archive plus session_end/session_start while session-memory remains limited to command:new and command:reset.

## Solution Assessment

Unclear. The persistence gap is real, but maintainers should choose one coherent reset-hook contract from the open implementations or a replacement before changing this surface.
