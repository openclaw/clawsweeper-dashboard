## Summary

Keep open: current main still does not make automatic daily/idle rollovers invoke the same pre-reset persistence surface that manual resets use, and the linked implementation work is still open rather than merged.

## What This Changes

Request to add a session reset or before-reset hook so bundled session-memory and custom hooks can persist context before automatic daily/idle rollovers, with a secondary hook-config schema cleanup.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/71792, https://github.com/openclaw/openclaw/pull/61675, or a maintainer-approved replacement that gives automatic daily/idle rollovers the same effective pre-reset persistence surface as manual resets and covers no-double-fire tests.

## Reproduction Assessment

Yes, at source level with high confidence: current main captures the previous session during stale daily/idle rollover but only emits typed lifecycle hooks, while bundled session-memory ignores non-command events. I did not run a live rollover because this is a read-only review.

## Solution Assessment

Unclear as a concrete implementation choice. A reset persistence hook is directionally appropriate, but maintainers should choose the canonical public hook/memory-finalization shape before landing one of the open PRs or a replacement.
