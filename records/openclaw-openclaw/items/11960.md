## Summary

Keep open: current main and v2026.6.6 still do not make automatic daily/idle rollovers invoke the command reset or before_reset persistence surface that bundled session-memory uses. The secondary hook config schema mismatch is fixed, and open related PRs target the gap, so this needs maintainer selection rather than another cleanup close.

## What This Changes

The issue requests a session reset hook so automatic daily and idle rollovers can persist session memory before context is cleared.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/71792, https://github.com/openclaw/openclaw/pull/61675, or a maintainer-approved replacement that gives automatic daily and idle rollovers the same effective pre-reset persistence surface as manual resets.

## Reproduction Assessment

Yes, at source level: automatic daily/idle stale rollovers can replace a session while session-memory still handles only internal command:new and command:reset events. I did not run a live rollover because this review is read-only.

## Solution Assessment

Unclear pending maintainer product/API choice. The gap is real, but the best permanent fix depends on whether maintainers want command-compatible reset dispatch, before_reset on automatic rollovers, a typed session reset event, or a combined surface.
