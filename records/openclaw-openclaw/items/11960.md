## Summary

Keep open. Current main and the latest v2026.6.6 release still make automatic daily/idle rollovers emit typed session lifecycle hooks only, while bundled session-memory remains limited to internal `command:new` and `command:reset`; the secondary hook-config schema mismatch is already fixed. The open closing implementation at https://github.com/openclaw/openclaw/pull/71792 and overlapping https://github.com/openclaw/openclaw/pull/61675 should be reviewed, replaced, or rejected before this issue is closed.

## What This Changes

The issue requests a reset-time hook surface so automatic daily and idle session rollovers can persist session-memory before old context is cleared.

## Best Possible Solution

Review and land one maintainer-approved implementation that persists session-memory on automatic daily/idle rollovers, preserves typed lifecycle hooks, avoids surprising broad command handlers, and adds focused daily plus idle regression coverage.

## Reproduction Assessment

Yes at source level: current main can roll over stale daily/idle sessions and preserve the previous entry, while bundled session-memory still only runs for internal `command:new` and `command:reset`. I did not run a live rollover in this read-only review.

## Solution Assessment

Unclear as a final API shape. The persistence gap is real, but maintainers should choose whether the best contract is `before_reset`, a typed session reset event, lifecycle-based session-memory handling, or synthetic command events before landing a branch.
