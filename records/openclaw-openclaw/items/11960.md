## Summary

Keep open because current main still does not route automatic daily/idle rollovers through the reset persistence surface, and a live open PR explicitly closes this issue while another open PR overlaps the same hook contract work.

## What This Changes

Request a session reset or before-reset hook so automatic daily/idle session rollovers can persist session-memory before prior context is cleared.

## Best Possible Solution

Ship one canonical implementation that persists session-memory exactly once for automatic daily and idle rollovers, preserves transcript archival/manual reset behavior, documents the selected hook contract, and adds focused daily plus idle regression coverage.

## Reproduction Assessment

Yes. Source inspection against current main shows stale daily/idle rollover captures previousSessionEntry and then only fires session_end/session_start, while bundled session-memory returns unless it receives command:new or command:reset.

## Solution Assessment

Unclear. The persistence gap is real, but https://github.com/openclaw/openclaw/pull/71792 and https://github.com/openclaw/openclaw/pull/61675 propose overlapping contracts, so maintainers should choose the canonical surface before merge.
