## Summary

Keep open. Current main still does not make automatic daily/idle rollovers invoke the same reset persistence surface that manual `/new`, `/reset`, and programmatic session resets use; the secondary hook-config schema mismatch is fixed, but the primary memory-persistence hook gap remains. https://github.com/openclaw/openclaw/pull/71792 explicitly closes this issue and https://github.com/openclaw/openclaw/pull/61675 overlaps, so this should wait for maintainer review of those implementations or a replacement.

## What This Changes

Request to add a session reset or before-reset hook so automatic daily and idle session rollovers can persist session memory before context is cleared, plus a per-hook config schema allowance.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/71792, https://github.com/openclaw/openclaw/pull/61675, or a maintainer-approved replacement that makes automatic daily and idle rollovers invoke a pre-reset persistence surface with previous session context and focused regression coverage.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: configure daily or idle rollover with session-memory enabled, let initSessionState mint a new stale session, and the automatic path emits only session lifecycle hooks while session-memory ignores non-command events. I did not run tests because this review is read-only.

## Solution Assessment

Unclear. The requested persistence surface is valid, but the best permanent shape needs maintainer choice between internal command reuse, typed before_reset extension, a new session reset event, or a combined contract.
