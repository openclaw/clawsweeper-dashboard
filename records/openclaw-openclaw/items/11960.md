## Summary

Keep open. Current main still has the automatic daily/idle rollover path capture and archive the previous session, but it only emits typed session lifecycle hooks there; the bundled session-memory hook remains limited to command:new and command:reset, and open PR #71792 explicitly closes this issue with #61675 overlapping.

## What This Changes

The issue requests a reset lifecycle hook or equivalent persistence event so bundled session-memory and custom hooks can save context during automatic daily or idle session rollovers.

## Best Possible Solution

Land PR #71792, PR #61675, or a maintainer-approved replacement that gives automatic daily/idle rollovers the same effective pre-reset memory persistence surface as manual resets with daily and idle regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure a stale daily or idle session, send the next user message, and current main archives/rotates through lifecycle hooks while session-memory remains restricted to command:new and command:reset.

## Solution Assessment

Unclear. The requested behavior is valid, but maintainers should choose one reset-hook contract from the open implementations or a replacement so OpenClaw does not gain competing semantics.
