## Summary

Keep open. Current main and the latest shipped release still leave automatic daily/idle rollover on the lifecycle-only path while bundled session-memory only accepts command:new and command:reset; the secondary hook-config schema mismatch is fixed, but the primary reset persistence surface remains unresolved and has open implementation PRs awaiting maintainer selection.

## What This Changes

Add a session reset or before-reset hook that fires for automatic daily/idle resets so session-memory and custom hooks can persist the previous session before rollover.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/71792 or a maintainer-approved replacement that persists memory exactly once for automatic daily and idle rollover while preserving manual reset and lifecycle semantics.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure a stale daily or idle session, send the next inbound message, and current main creates a new session plus session_end/session_start while session-memory remains limited to command:new and command:reset.

## Solution Assessment

Unclear. The persistence gap is real, but the exact hook/API contract is a product decision because open branches propose different command, before_reset, and session-event shapes.
