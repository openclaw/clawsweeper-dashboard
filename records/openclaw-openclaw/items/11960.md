## Summary

Keep open because current main still has the reset-persistence gap: automatic daily and idle rollovers capture prior session state and emit typed lifecycle hooks, but they do not reach the manual reset hook path that bundled session-memory consumes. The issue also has active overlapping implementation candidates at https://github.com/openclaw/openclaw/pull/71792 and https://github.com/openclaw/openclaw/pull/61675, so the next step is maintainer contract selection rather than cleanup closure or a new autonomous fix lane.

## What This Changes

The issue requests a session reset hook or equivalent persistence surface for automatic daily and idle resets so session-memory can save context before rollover.

## Best Possible Solution

Ship one canonical reset-persistence contract that persists bundled session-memory exactly once for automatic daily and idle rollovers, preserves manual reset and transcript archive behavior, documents the chosen hook shape, and adds focused daily plus idle regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure daily or idle reset, let an existing session become stale, then send a normal message; initSessionState creates a new session and emits lifecycle hooks only, while session-memory ignores those events.

## Solution Assessment

Unclear. The persistence gap is real, but solving it changes the hook/session lifecycle contract and two open PRs propose overlapping approaches, so maintainers should select the canonical surface before merge.
