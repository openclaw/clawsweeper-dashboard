## Summary

Keep open. Current main still does not route automatic daily/idle rollovers through the reset persistence surface used by bundled session-memory; the secondary hook config schema mismatch is already fixed, but the primary hook behavior remains unresolved and has an open candidate PR.

## What This Changes

The issue requests a session reset hook so automatic daily and idle resets can persist session memory before context is cleared.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/61675 or a maintainer-approved replacement that emits one canonical pre-reset persistence surface for automatic daily/idle rollover, updates session-memory, and adds daily plus idle regression coverage.

## Reproduction Assessment

Yes from source inspection: stale daily/idle rollover captures and archives the previous session, then emits only typed session lifecycle hooks, while bundled session-memory still listens only to command `new`/`reset` events. I did not run a live/runtime repro in this read-only review.

## Solution Assessment

No final solution is selected yet. The narrow maintainable path is a maintainer-approved reset lifecycle contract that reuses existing reset surfaces where possible instead of adding a parallel one-off hook path.
