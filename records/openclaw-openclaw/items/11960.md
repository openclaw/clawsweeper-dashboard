## Summary

Keep open. Current main and v2026.5.12 still leave automatic daily/idle rollovers on the lifecycle-only path, and https://github.com/openclaw/openclaw/pull/71792 explicitly closes this issue while https://github.com/openclaw/openclaw/pull/61675 overlaps, so cleanup closure would race the maintainer hook-contract decision.

## What This Changes

Add a reset lifecycle hook or persistence surface so session-memory and custom hooks can save context before automatic daily or idle session resets, plus allow per-hook config keys.

## Best Possible Solution

Ship one canonical implementation that persists session-memory exactly once for automatic daily and idle rollovers, preserves transcript archival and manual reset behavior, documents the selected hook contract, and adds regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure an existing session to become stale by daily or idle policy and send the next inbound message; current main starts a new session and lifecycle hooks while bundled session-memory remains limited to command:new and command:reset.

## Solution Assessment

Unclear. The persistence gap is real, but maintainers need to choose between the overlapping synthetic command reset, before_reset, and new session-event shapes before landing a durable API.
