## Summary

Keep open. Current main and the latest release still do not make automatic daily/idle rollovers invoke the reset persistence surface used by bundled session-memory; the issue is now paired with open implementation candidates that need maintainer contract selection rather than another cleanup close.

## What This Changes

The issue requests a session reset or before-reset hook so bundled session-memory and custom hooks can persist previous session context during automatic daily and idle rollovers.

## Best Possible Solution

Land, replace, or explicitly reject one canonical implementation that persists session-memory on automatic daily/idle rollover without surprising existing command hooks or losing lifecycle-hook behavior.

## Reproduction Assessment

Yes, at source level with high confidence: current main can replace stale daily/idle sessions while bundled session-memory still handles only internal command:new and command:reset events. I did not run a live rollover because this review is read-only.

## Solution Assessment

Unclear pending maintainer product/API choice. The gap is real, but the best permanent fix depends on whether maintainers want a command-compatible reset dispatch, before_reset extension, new typed reset event, or session_end-based memory behavior.
