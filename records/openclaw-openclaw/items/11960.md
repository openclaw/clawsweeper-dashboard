## Summary

Keep open. Current main and the latest shipped tag still have the automatic daily/idle rollover path limited to session lifecycle hooks while bundled session-memory only accepts command:new and command:reset; open implementation PRs already target this issue, so closing or queueing another repair would be premature.

## What This Changes

The issue requests a session reset hook or equivalent persistence surface so automatic daily and idle resets can save session memory before clearing context.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/71792, https://github.com/openclaw/openclaw/pull/61675, or a maintainer-approved replacement that persists session-memory exactly once for automatic daily and idle rollovers and documents the chosen hook contract with regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure an existing session to become stale by daily or idle policy and send the next inbound message; current main creates a new session and lifecycle hooks while session-memory remains limited to command:new and command:reset.

## Solution Assessment
