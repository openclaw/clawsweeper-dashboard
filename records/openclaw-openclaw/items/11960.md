## Summary

Keep open. Current main still leaves automatic daily/idle rollovers on the session lifecycle path while bundled session-memory only persists command:new and command:reset, and an open PR explicitly closes this issue with another open PR overlapping the same hook gap.

## What This Changes

This issue requests a session reset/before-reset hook or equivalent memory persistence surface for daily/idle auto-resets, plus a schema fix for hook-specific session-memory config keys.

## Best Possible Solution

Ship one canonical implementation that persists session-memory exactly once for automatic daily and idle rollovers, preserves transcript archival and manual reset behavior, documents the selected hook contract, and adds focused daily plus idle regression coverage.

## Reproduction Assessment

## Solution Assessment

Unclear. The persistence gap is real, but maintainers need to choose the hook contract before landing one of the overlapping implementations.
