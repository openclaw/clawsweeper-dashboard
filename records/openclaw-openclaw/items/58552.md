## Summary

Keep open for maintainer review. Current main does not implement the requested Apple Silicon optimization suite, and the PR branch is not safe to merge because the diff introduces blocking regressions in command execution, process cleanup, ESM runtime behavior, and new thread-pool helpers.

## What This Changes

The PR adds Apple Silicon detection utilities, process/shell/process-tree behavior changes, hybrid runtime/thread-pool/tensor helpers, docs, tests, and a verification script.

## Best Possible Solution

Narrow this to benchmark-backed macOS ARM64 improvements that preserve current process, shell, and ESM contracts, with separate design review for any ML runtime, thread-pool, GPU, or ANE work.

## Reproduction Assessment

Yes for the blocking patch defects: static inspection of the PR diff reproduces the maxBuffer reduction, detach/process-group cleanup regressions, non-async await syntax errors, ESM require/__filename failures, missing symbols, and unresolved worker execution path. No high-confidence reproduction exists for the optimization claims because the PR does not include benchmark or live Apple Silicon proof.

## Solution Assessment

No. The current PR is much broader than the stated optimization goal and changes sensitive command-execution and process-cleanup contracts; the safer path is a smaller measured patch with Linux, Intel macOS, and Apple Silicon regression coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.97

Full review comments:

- **[P1] Preserve runExec default maxBuffer capacity:** `src/process/exec.ts:115`
  - body: Object-options callers that omit `maxBuffer` now get `OPTIMAL_BUFFER_SIZE` instead of leaving the value undefined for Node's default. That drops the cap to 64 KiB or 128 KiB and can make existing timeout-only `runExec` calls fail with `ERR_CHILD_PROCESS_STDIO_MAXBUFFER`.
  - confidence: 0.97
- **[P1] Keep Unix process-group SIGKILL off Apple checks:** `src/process/kill-tree.ts:81-83`
  - body: On non-Apple-Silicon Unix hosts, `useProcessGroup` becomes false, so the grace-period force kill skips `process.kill(-pid, "SIGKILL")` and only targets the parent PID. Children can survive even though this helper is meant to terminate the tree.
  - confidence: 0.96
- **[P1] Clean up detached timeout process groups:** `src/process/exec.ts:253`
  - body: The Apple Silicon path sets `detached = true`, creating a new process group, but the timeout handler still calls `child.kill("SIGKILL")` on only the direct child. Grandchildren can survive timeout cleanup on the platform this PR is trying to improve.
  - confidence: 0.94
- **[P2] Make dynamic-import tests async:** `src/utils/platform.test.ts:54`
  - body: This test body and several later `it` callbacks use `await import()` inside non-async functions, which is a TypeScript syntax error. The new test file will not compile until those callbacks are marked `async` or rewritten.
  - confidence: 0.99
- **[P2] Replace CommonJS require in the ESM helper:** `src/utils/platform.ts:25`
  - body: This package is ESM, but the new platform helper calls `require()` inside function bodies. Those paths can throw `ReferenceError` at runtime, and the shell helper lookup also points at a non-existent local module.
  - confidence: 0.93
- **[P2] Respect configured SHELL before preferring zsh:** `src/agents/shell-utils.ts:61-66`
  - body: The new Apple Silicon branch returns zsh before the existing `process.env.SHELL` handling. Users who explicitly configured bash or another supported shell on macOS ARM64 would silently get zsh instead.
  - confidence: 0.92
- **[P1] Define thread-pool sizing helpers:** `src/utils/thread-pool.ts:25-29`
  - body: `getDefaultThreadPoolConfig()` calls `getPhysicalCpuCount()` and `getPerformanceCoreCount()`, but this file does not import or define either symbol. The new thread-pool module cannot typecheck as written.
  - confidence: 0.97
- **[P2] Use an ESM-safe worker entrypoint:** `src/utils/thread-pool.ts:83-86`
  - body: The new thread pool constructs workers with `new Worker(__filename)`, but `__filename` is not defined in this ESM package. Creating a `ThreadPool` can fail before any task runs.
  - confidence: 0.9
- **[P1] Preserve thread-pool task resolvers:** `src/utils/thread-pool.ts:181-184`
  - body: `run()` stores the caller's Promise handlers, then `submitTaskToWorker()` immediately overwrites the same active task entry with no-op callbacks. Completed tasks would not settle the Promise returned to callers.
  - confidence: 0.93
- **[P1] Add worker-side execute-task handling:** `src/utils/thread-pool.ts:175-179`
  - body: The main thread posts `execute-task` messages to workers, but the module never defines a worker-side `parentPort` message loop to run the task and emit completion or error messages. Dispatched work can remain pending forever.
  - confidence: 0.91
- **[P2] Stop calling missing thread-pool exports:** `src/utils/tensor-ops.ts:396-405`
  - body: The tensor helpers destructure `getOptimalParallelFactor` and `getOptimalBatchSize` from `./thread-pool.js`, but the new thread-pool module exports neither name. Calling these helpers will fail at runtime.
  - confidence: 0.9
- **[P2] Avoid shell interpolation of SHELL:** `src/utils/platform.ts:157-159`
  - body: `getShellCommand()` builds an `execSync` shell command from `process.env.SHELL`. A value containing quotes or shell metacharacters could run unintended commands; use `fs.access` or `execFile` with argv instead.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: The diff touches command execution and introduces an environment-controlled shell command sink.

Concerns:

- **[medium] Environment-controlled shell command sink:** `src/utils/platform.ts:157`
  - body: `getShellCommand()` interpolates `process.env.SHELL` into an `execSync` shell string. If that environment value contains quotes or shell metacharacters, OpenClaw can execute unintended commands under its own process context.
  - confidence: 0.9
- **[medium] Detached timeout cleanup can leak descendants:** `src/process/exec.ts:253`
  - body: The PR sets `detached: true` for Apple Silicon command execution but keeps timeout cleanup as direct `child.kill("SIGKILL")`. Low-level command execution should not leave descendant processes running after timeout.
  - confidence: 0.88
