## Summary

This PR targets a real current-main gap, but it should stay open because the branch does not compile, renders `errorMessage` too broadly, and still lacks external after-fix real behavior proof.

## What This Changes

The PR adds Swift `errorMessage` decoding and propagation for chat messages, renders provider error text when assistant history content is empty, and adds a ChatViewModel regression test.

## Best Possible Solution

Preserve peer `errorMessage` through the Swift model and copy paths, then render it only for assistant turns with normalized `stopReason == "error"` and no visible text, with regression coverage for stale-error and non-text successful turns.

## Reproduction Assessment

Yes, at source level. Current main drops peer `errorMessage` in the Swift chat message model and `primaryText(in:)` only renders text content, while the backend keeps detailed provider failures in peer `errorMessage`.

## Solution Assessment

No. Preserving and rendering `errorMessage` is the right direction, but this implementation must compile and must limit fallback rendering to true assistant error turns.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.97

Full review comments:

- **[P1] Initialize decoder locals before building fallback content:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatModels.swift:198-199`
  - body: The new fallback closure reads `self.errorMessage` before `self.content` has been initialized in `init(from:)`, and the `macos-swift` job fails with `'self' used before all stored properties are initialized`. Decode `errorMessage` into a local before assigning stored properties.
  - confidence: 0.98
- **[P2] Gate decoder fallback to assistant error turns:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatModels.swift:198-204`
  - body: The decoder creates visible text from `errorMessage` whenever decoded content is empty. Successful or silent rows can carry stale/background `errorMessage`, so synthesize fallback text only for assistant turns whose normalized `stopReason` is `error`.
  - confidence: 0.93
- **[P2] Gate primaryText fallback to true error turns:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatView.swift:442`
  - body: `primaryText` falls back to `message.errorMessage` whenever no text blocks are found. Tool-call-only or otherwise successful assistant messages can then display stale provider failures as the reply; require an assistant error turn before using this fallback.
  - confidence: 0.91
