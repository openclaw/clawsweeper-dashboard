## Summary

Keep open: current main still drops peer `errorMessage` in Swift chat history, but this PR cannot merge because the decoder does not compile, the fallback renders provider errors without an error-turn guard, and the external real-behavior proof gate is still missing.

## What This Changes

The PR adds Swift ChatUI `errorMessage` decoding and propagation, visible fallback text for empty assistant turns, and a ChatViewModel regression test.

## Best Possible Solution

Preserve peer `errorMessage` through the Swift model and copy paths, then render it only for assistant turns with normalized `stopReason == "error"` and no visible text, with stale-error/tool-only regression coverage.

## Reproduction Assessment

Yes, at source level. A current-main Swift chat-history payload with `role: "assistant"`, `content: []`, `stopReason: "error"`, and non-empty `errorMessage` decodes to no visible text; the PR head also has a live macos-swift compile failure at the changed decoder line.

## Solution Assessment

No. The intended fix boundary is appropriate, but this implementation must avoid reading `self` before all stored properties are initialized and must render peer error text only for true assistant error turns.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.97

Full review comments:

- **[P1] Initialize decoder locals before building fallback content:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatModels.swift:198-199`
  - body: The new fallback closure reads `self.errorMessage` before `self.content` has been initialized in `init(from:)`, and the live macos-swift job fails with `'self' used before all stored properties are initialized`. Decode into locals or a helper before assigning stored properties.
  - confidence: 0.98
- **[P2] Gate decoder fallback to assistant error turns:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatModels.swift:198-204`
  - body: The decoder creates visible text from `errorMessage` whenever decoded content is empty. Successful silent rows can carry stale/background `errorMessage`, so synthesize fallback text only for assistant turns whose normalized `stopReason` is `error`.
  - confidence: 0.93
- **[P2] Gate primaryText fallback to true error turns:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatView.swift:442-445`
  - body: `primaryText` falls back to `message.errorMessage` whenever no text blocks are found. Tool-call-only or otherwise successful assistant messages can then display stale provider failures as the reply; require an assistant error turn before using this fallback.
  - confidence: 0.91
