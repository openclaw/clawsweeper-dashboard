## Summary

Keep open: current main still has generic Claude live-session timeout handling and no pending MCP/tool-use tracker, while the related cascade and trajectory items cover adjacent observability gaps rather than this specific pending-tool diagnostic.

## What This Changes

The issue requests tracking pending Claude/MCP tool-use events in Claude live sessions and logging long-running pending tools for post-mortem visibility.

## Best Possible Solution

Add a bounded pending-tool tracker in `src/agents/cli-runner/claude-live-session.ts` using the actual Claude stream-json event shape, emit redacted pending-tool names/ids/ages in warning or timeout diagnostics, and cover it with focused live-session tests.

## Reproduction Assessment

Yes, source-reproducible: current main has no pending-tool state in the Claude live-session turn/session objects and no tool-event branch in the JSONL line handler. I did not run a live hung MCP tool call in this read-only review.

## Solution Assessment

Yes, the requested diagnostic is a narrow maintainable direction, but the implementation should not copy the exact pseudo-code blindly because current Claude stream-json exposes tool activity through `stream_event` content blocks rather than top-level `tool_use_start`/`tool_use_result` events. The safer fix is to track tool ids/names/ages and avoid raw argument logging.
