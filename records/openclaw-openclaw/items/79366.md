## Summary

Keep open: current main still has no pending-tool tracker in the Claude live-session parser, and the related open work covers the resume cascade and trajectory sidecars rather than the missing MCP tool identity in watchdog diagnostics.

## What This Changes

Add redacted tracking and logging for long-pending Claude CLI MCP tool calls so no-output watchdog incidents identify the likely hung tool.

## Best Possible Solution

Add a redacted pending-tool tracker in the Claude live-session stream parser and include pending tool summaries in long-pending warnings and no-output timeout diagnostics, without changing retry, fallback, or session semantics.

Yes, source-reproducible: current main routes `claude-cli` through `runClaudeLiveSessionTurn`, parses stream-json lines, and aborts on no-output timeout without retaining pending tool state. I did not run a live hung MCP tool session.

## Solution Assessment

Yes, the diagnostic direction is narrow and maintainable if implemented against Claude's actual assistant/user and stream-event shapes. The fix should log only redacted tool name/id/age summaries and leave execution behavior unchanged.

## Repair Work Prompt

Implement the pending MCP/Claude tool diagnostic requested by https://github.com/openclaw/openclaw/issues/79366. In `src/agents/cli-runner/claude-live-session.ts`, add per-turn tracking for Claude stream-json tool calls using the documented Claude Agent shapes: record assistant `message.content` blocks with `type: "tool_use"` and clear matching user `message.content` blocks with `type: "tool_result"` / `tool_use_id`; optionally use `stream_event.content_block_start` for earlier tool name visibility if tests prove the shape. Warn once when a pending tool exceeds a conservative threshold derived from the existing no-output window or a local constant, and include redacted pending tool summaries in the no-output timeout diagnostic. Do not log raw MCP arguments or secrets, do not change retry/fallback/session-resume semantics, and do not broaden core/plugin boundaries. Add focused regression coverage for tool-use tracking, clearing on tool_result, multiple tools, and timeout diagnostics in the Claude live-session tests. Add a changelog entry only if maintainers treat this operator diagnostic as user-visible.
