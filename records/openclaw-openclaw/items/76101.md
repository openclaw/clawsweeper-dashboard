## Summary

Keep open: current main still shows the bare Lobster `openclaw.invoke --tool llm-task` example while the embedded runner only forwards existing environment variables and Gateway `/tools/invoke` still requires authenticated operator access. The related allowlist report is adjacent, not a replacement for the missing URL/auth context.

## What This Changes

The issue asks OpenClaw to make documented Lobster `openclaw.invoke --tool llm-task` steps runnable by injecting, documenting, or replacing the required Gateway URL/auth context.

## Best Possible Solution

Document the complete supported `openclaw.invoke` setup now, including URL, auth, tool policy, and the security boundary; leave automatic credential/context injection for a separately reviewed runtime design.

## Reproduction Assessment

Yes. The report gives a concrete current-release path, and current main statically confirms the docs omit URL/auth while the embedded runner only forwards existing environment variables and Gateway requires auth.

## Solution Assessment

A docs-first fix is the safest immediate solution. Automatic credential injection is not clearly the best fix without a security-reviewed boundary because `/tools/invoke` shared-secret auth is full operator access.
