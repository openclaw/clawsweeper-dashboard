## Summary

Keep open: current main still documents bare Lobster `openclaw.invoke --tool llm-task` steps, while the embedded runner only forwards ambient environment and the Gateway invoke endpoint requires explicit URL/auth context.

## What This Changes

The issue requests that Lobster/LLM Task docs or the embedded runner provide the Gateway URL and auth context required for documented `openclaw.invoke --tool llm-task` workflow steps.

## Best Possible Solution

Document the complete supported `openclaw.invoke` environment/auth setup now, and defer automatic embedded-runner credential injection until there is an explicit scoped auth design.

## Reproduction Assessment

Yes. The issue gives a concrete current-release repro, and current main statically confirms that the docs omit required env/auth while the runner only forwards existing `process.env` and Gateway invoke requires auth.

## Solution Assessment

Yes for the immediate repair: a docs-first correction is the narrowest maintainable fix. Automatically injecting Gateway credentials into Lobster shell steps is not the best default without a reviewed scoped-auth boundary.

## Security Review

Status: needs_attention

Summary: The issue is security-sensitive because a runtime fix could leak full-operator Gateway credentials into workflow shell environments.

Concerns:

- **[medium] Avoid implicit operator-token injection:** `docs/gateway/tools-invoke-http-api.md:43`
  - body: `/tools/invoke` treats shared-secret bearer auth as full operator access, so silently exporting the Gateway token/password into Lobster command environments would materially expand credential exposure unless a scoped boundary is designed first.
  - confidence: 0.9
- **[medium] Shared-secret invokes restore full operator scope:** `src/gateway/tools-invoke-http.ts:45`
  - body: The runtime handler deliberately maps shared-secret auth on `/tools/invoke` to full operator scopes and owner semantics, making automatic credential forwarding a security design change rather than a small convenience fix.
  - confidence: 0.86
