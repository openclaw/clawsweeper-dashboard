## Summary

Keep open: current main still documents the Lobster `openclaw.invoke --tool llm-task` pattern without the URL/auth setup required by the embedded runner and Gateway tool-invoke auth path.

## What This Changes

The issue asks OpenClaw to make the documented Lobster-to-`llm-task` invocation path runnable or to document the required Gateway URL and auth setup.

## Best Possible Solution

Document the complete supported `openclaw.invoke` Gateway URL/auth setup now, and treat automatic credential injection as a separate reviewed auth-boundary change.

## Reproduction Assessment

Yes. The report gives a concrete current-release path, and current main confirms the docs omit URL/auth while the embedded runner only forwards existing `process.env`.

## Solution Assessment

A docs-first repair is the safest narrow fix; automatic runner credential injection is not clearly the best solution without a security design for the `/tools/invoke` operator-auth boundary.
