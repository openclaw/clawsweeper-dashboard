## Summary

Keep open: current main still documents the bare Lobster `openclaw.invoke --tool llm-task` step, while the embedded runner only forwards ambient environment and the Gateway invoke endpoint requires explicit URL/auth context.

## What This Changes

The issue asks OpenClaw to either make documented Lobster `openclaw.invoke --tool llm-task` steps runnable from the embedded runner or document the required Gateway URL/auth setup and supported bridge clearly.

## Best Possible Solution

Update the Lobster and LLM Task docs to show the supported `OPENCLAW_URL`/`OPENCLAW_TOKEN` or `--url`/`--token` setup and link the Gateway tools-invoke auth boundary; leave automatic credential injection for a separate scoped-auth design.

## Reproduction Assessment

Yes. The issue gives a concrete current-release repro, and current main statically confirms the docs omit the required env/auth while the embedded runner only forwards ambient `process.env` and upstream Lobster requires `OPENCLAW_URL` or `--url`.

## Solution Assessment

Yes for the immediate fix: a docs-first correction is the narrowest maintainable path. Injecting Gateway credentials into the embedded runner is not the best default repair without a reviewed scoped-auth boundary.
