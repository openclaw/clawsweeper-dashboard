## Summary

Keep open: current main still has a documented Lobster `openclaw.invoke --tool llm-task` workflow step that omits the URL/auth context required by the shipped shim and Gateway invoke endpoint.

## What This Changes

The issue asks OpenClaw to make the documented Lobster-to-`llm-task` invocation path runnable by supplying or documenting the required Gateway URL and auth context.

## Best Possible Solution

The end state should be a docs-first correction that shows the complete supported `openclaw.invoke` prerequisites and explicitly avoids implicit credential injection until a scoped auth boundary exists.

## Reproduction Assessment

Yes. The issue gives exact current-release repro steps, and current main source confirms the docs omit required env/auth while the embedded runner only forwards ambient `process.env` and Gateway invoke requires auth.

## Solution Assessment

Yes for the immediate repair. Updating the Lobster and LLM Task docs is the narrowest maintainable fix; automatic credential injection would be a separate security/product design change.
