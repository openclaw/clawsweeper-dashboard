## Summary

Keep this open: current main and the latest shipped tag still omit `accountId` on the `/hooks/agent` ingress path, and the linked closing PR remains open and unmerged.

## What This Changes

The issue reports that `/hooks/agent` drops `accountId` before cron delivery, breaking multi-account channel routing for WhatsApp and similar channels.

## Best Possible Solution

Land the linked PR or a narrow replacement that threads trimmed `accountId` through hook normalization, replay scope, `CronDelivery`, tests, and public hook docs while preserving sanitized hook names and current cron message-tool policy.

## Reproduction Assessment

Yes. Source inspection on current main shows `/hooks/agent` never normalizes `payload.accountId`, hook-created delivery only carries `mode`, `channel`, and `to`, and cron can only route by `accountId` after that field reaches `CronDelivery`.

## Solution Assessment

Yes in direction, but not complete. The maintainable fix is a narrow hook-to-cron contract update with replay-key, tests, docs, and real behavior proof, so this issue should stay open until the linked PR or a replacement merges.
