## Summary

Keep this issue open: current main and latest release still drop `accountId` from `/hooks/agent`, and the open closing PR has not merged yet.

## What This Changes

The issue requests preserving `accountId` from `POST /hooks/agent` through cron delivery so multi-account channel hooks target the intended account.

## Best Possible Solution

Land or replace the linked fix so hook dispatch builds canonical cron delivery carrying `accountId`, with regression coverage and reviewed delivery fallback behavior.

## Reproduction Assessment

Yes, at source level. Current main drops `accountId` during hook payload normalization and builds cron delivery without it; I did not run a live WhatsApp multi-account setup.

## Solution Assessment

Yes, the best direction is to pass `accountId` through the hook boundary into the existing cron delivery contract rather than create a separate hook-only routing path. The open linked PR should be reviewed, repaired, landed, or replaced.
