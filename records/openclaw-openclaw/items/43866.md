## Summary

Keep open. Current main and the latest release still omit `accountId` from `/hooks/agent`, and the open linked fix at https://github.com/openclaw/openclaw/pull/43951 has not merged.

## What This Changes

Preserve `accountId` from `POST /hooks/agent` through hook normalization, dispatch, and `CronDelivery` for multi-account channel delivery.

## Best Possible Solution

Land a narrow rebased fix through https://github.com/openclaw/openclaw/pull/43951 or a replacement PR that parses/trims hook `accountId`, includes it in replay scope and `CronDelivery`, preserves sanitized metadata and current cron tool policy, and updates focused tests/docs.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path without live WhatsApp setup. On current main, `/hooks/agent` normalization omits `accountId` and dispatch builds delivery without it, so the field cannot reach multi-account delivery resolution.

## Solution Assessment

Partially. Forwarding `accountId` through the hook delivery path is the narrow maintainable fix, but the linked branch should be rebased and kept to that scope rather than changing unrelated cron message-tool policy.
