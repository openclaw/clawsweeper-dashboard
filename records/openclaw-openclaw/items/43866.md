## What This Changes

The issue asks for `/hooks/agent` to preserve a request `accountId` through hook normalization and cron delivery routing for multi-account channels.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/43951, or a narrow replacement, that threads trimmed `accountId` through hook normalization, replay scope, cron delivery, tests, and public hook docs while preserving current hook-name sanitization and cron message-tool policy.

## Reproduction Assessment

Yes. Source inspection on current main shows `/hooks/agent` never normalizes `payload.accountId`, hook-created delivery only carries `mode`, `channel`, and `to`, and cron can route by `accountId` only after that field reaches `CronDelivery`.

## Solution Assessment

Yes in direction, but not finished. The maintainable solution is a narrow hook-to-cron contract update with replay-key, tests, docs, and real behavior proof; this issue should stay open until the linked PR or a replacement merges.
