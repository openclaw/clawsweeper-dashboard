## Summary

Keep open. Current main still lacks a documented Discord category allowlist contract, and preflight only supplies thread-parent metadata to the channel resolver, so ordinary child channels under a category do not inherit category settings; the related thread-grandparent report also treats this issue as the canonical tracker.

## What This Changes

The issue requests Discord guild allowlists to accept category entries so child channels inherit settings without per-channel config edits or gateway restarts.

## Best Possible Solution

Define a Discord-owned category allowlist syntax and precedence model, wire ordinary channel category parent metadata into the existing resolver, and update schema, docs, and focused preflight/allowlist tests.

## Reproduction Assessment

Yes, at source level. A fetched guild channel can carry parentId, but current preflight only passes thread parent metadata into allowlist matching, so a child channel under a configured category will not inherit category settings today.

## Solution Assessment

Unclear until maintainers choose the public config contract. Reusing the existing parent fallback looks like the narrowest implementation once category syntax and precedence are defined.
