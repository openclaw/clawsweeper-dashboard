## Summary

Keep open: current main and v2026.6.10 still lack Discord category-level guild allowlist inheritance, and the open item remains the canonical tracker for a config and authorization-boundary decision.

## What This Changes

Add Discord guild allowlist support for category-level channel config so child channels can inherit settings such as requireMention without per-channel config edits.

## Best Possible Solution

Approve one Discord-owned category inheritance contract, then update resolver metadata, config types/schema, docs, audit/status behavior, and focused tests with explicit channel-over-category precedence.

## Reproduction Assessment

Yes, at source level. Current main has no category config field and the Discord resolver only checks direct channel, one parent, and literal wildcard candidates, so category-level entries cannot authorize child channels.

## Solution Assessment

Unclear until maintainers choose the public config contract and security precedence. The narrow implementation belongs in the Discord plugin/config surface, but automation should not choose the authorization syntax or widen ingress policy on its own.
