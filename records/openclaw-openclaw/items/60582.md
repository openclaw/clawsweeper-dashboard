## Summary

Keep open: current main and v2026.6.6 still do not provide Discord category-level guild channel inheritance, and the remaining work is security-sensitive config behavior that needs maintainer-approved syntax and precedence.

## What This Changes

The issue requests Discord guild allowlist support for category-level channel settings so new child channels inherit options such as `requireMention` without config edits or gateway restarts.

## Best Possible Solution

Approve one Discord-owned category inheritance contract, wire direct child and thread grandparent category metadata into the existing allowlist resolver, preserve exact-channel override precedence, and update types/schema/docs plus focused resolver and preflight tests.

## Reproduction Assessment

Yes, at source level: a nonempty Discord guild `channels` map containing only a category key is not matched for ordinary child channels or thread grandparent categories, so the resolver returns denied. I did not run a live Discord repro in this read-only review.

## Solution Assessment

Unclear: category inheritance is the right problem area, but the proposed `category:<id>` form is only one possible config shape and needs maintainer approval for syntax, precedence, and canonical `enabled` semantics.
