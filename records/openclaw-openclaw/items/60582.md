## Label Justifications

- `P2`: This is a normal-priority Discord access-control enhancement with clear user impact but no urgent outage or regression.
- `impact:security`: The request changes which Discord guild channels are authorized to trigger OpenClaw from an allowlist.
- `impact:message-loss`: Messages in intended child channels under a category are ignored unless every channel is listed explicitly or a broader fallback is configured.

## Summary

Keep open. Current main and the latest release still expose Discord guild `channels` as exact entries plus one parent fallback and literal `*`, while inbound message preflight does not pass ordinary channel category metadata into that resolver.

## What This Changes

Add Discord category-level guild channel allowlist config so new channels under a category inherit settings such as `requireMention: false` without per-channel edits or gateway restarts.

## Best Possible Solution

Define a Discord-owned category allowlist syntax and precedence model, wire category and thread-grandparent metadata through the existing resolver, then update types, schema, docs, audit/status output, and focused resolver/preflight tests.

## Reproduction Assessment

Yes, at source level: current main has parent metadata available from Discord channel objects, but inbound preflight only supplies thread-parent metadata to the allowlist resolver and the public config contract has no category syntax. I did not run a live Discord repro in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public config contract and precedence. Once that decision is made, extending the existing Discord resolver path is likely the narrowest maintainable solution.
