## Summary

Keep open: current main and the latest release still do not provide Discord category-level guild channel inheritance, and the remaining work needs maintainer-approved access-control syntax and precedence.

## What This Changes

The issue requests Discord guild allowlist support for category-level channel config so child channels inherit settings without per-channel config edits or gateway restarts.

## Best Possible Solution

Approve one Discord-owned category inheritance contract, wire direct child and thread grandparent category metadata into the allowlist resolver, preserve exact-channel override precedence, and update types, schema, docs, audit/status behavior, and focused tests together.

## Reproduction Assessment

Yes, at source level: a nonempty Discord guild channels map containing only a category key is not matched for ordinary child channels or thread grandparent categories, so the resolver returns denied. I did not run a live Discord repro in this read-only review.

## Solution Assessment

Unclear until maintainers choose the permanent syntax and authorization precedence. Category inheritance belongs in the Discord allowlist/preflight path, but the proposed category:<id> form is only one possible config contract.
