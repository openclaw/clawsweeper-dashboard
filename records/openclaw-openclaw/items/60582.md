## Summary

Keep open: current main and the latest release still lack Discord category-level guild channel inheritance, and this remains the canonical tracker for a security-sensitive config/authorization decision.

## What This Changes

The issue requests Discord category-level allowlist config so child channels inherit settings like `requireMention` without listing every channel ID.

## Best Possible Solution

Approve one Discord-owned category inheritance contract, then update resolver/preflight metadata, config types, schema, docs, audit/status behavior, and focused tests with exact-channel override precedence preserved.

## Reproduction Assessment

Yes, at source level: a nonempty Discord guild `channels` map containing only a category key has no category lookup path on current main, so an ordinary child channel resolves denied. I did not run a live Discord repro in this read-only review.

## Solution Assessment

Unclear until maintainers choose the permanent syntax and precedence. Category inheritance belongs in the Discord allowlist/preflight path if approved, but the proposed `category:<id>` form is only one possible config contract.
