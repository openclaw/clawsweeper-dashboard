## Label Justifications

- `P2`: This is a normal-priority Discord access-control enhancement with clear scope but no urgent regression or outage.
- `impact:security`: The request changes which Discord guild channels are authorized to trigger OpenClaw from a configured allowlist.
- `impact:message-loss`: Messages in intended child channels under a category are ignored unless each channel is listed explicitly or a broader fallback is configured.

## Summary

Keep open. Current main still lacks a documented Discord category allowlist contract, message preflight only supplies thread-parent metadata to the channel resolver, and the related thread-grandparent report was closed in favor of this issue as the canonical tracker.

## What This Changes

Request support for Discord category IDs in guild channel allowlists so child channels inherit settings without per-channel config edits or gateway restarts.

## Best Possible Solution

Define a Discord-owned category allowlist syntax and precedence model, wire ordinary channel category and thread-grandparent metadata through the existing resolver path, then update schema, docs, status/audit behavior, and focused allowlist/preflight tests.

## Reproduction Assessment

Yes, at source level. A fetched Discord guild channel can carry parentId, but current message preflight only passes thread parent metadata into allowlist matching, so category child channels do not inherit a category entry today.

## Solution Assessment

Unclear until maintainers choose the public config contract. Reusing the existing direct-parent-wildcard resolver shape is likely the narrowest implementation once category syntax, precedence, and thread-grandparent behavior are explicitly defined.
