## Summary

Keep open. Current main still does not apply Discord guild allowlist settings from a category parent to ordinary text channels: fetched channel metadata preserves `parentId`, and the resolver can fall back to a parent entry, but normal message preflight only supplies thread-parent metadata. Related #8972 overlaps the dynamic-channel use case, but it is a broader pattern/glob request and is not a canonical replacement for category inheritance.

## What This Changes

The issue requests Discord guild allowlist entries that apply to every child channel in a Discord category, so newly-created channels inherit settings such as `requireMention: false` without config edits or gateway restarts.

## Best Possible Solution

Implement the behavior inside the Discord plugin by feeding non-thread guild channel parent metadata into the existing fallback resolver, preserving direct channel overrides over category defaults and wildcard fallback, then document the chosen category config syntax and add focused preflight/schema/docs coverage.

## Reproduction Assessment

Yes. A code-level reproduction is a guild text channel whose fetched `channelInfo.parentId` matches a configured category entry while the direct channel is absent; current preflight never passes that non-thread parent id into allowlist resolution, so the category config is not applied.

## Solution Assessment

Unclear. Reusing the existing parent-fallback resolver is likely the narrowest implementation, but maintainers should first choose the public category key syntax and confirm how it relates to #8972.
