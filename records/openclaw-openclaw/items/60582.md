## Summary

Keep open. Current main has the raw Discord parent category metadata and an existing parent-fallback resolver, but ordinary guild text-channel preflight still only passes thread-parent metadata into that resolver, and the public config/docs do not define category allowlist syntax.

## What This Changes

The issue requests Discord category-level guild allowlist entries so child channels inherit settings such as `requireMention: false` without per-channel config edits or gateway restarts.

## Best Possible Solution

Define the Discord-owned category allowlist syntax, feed non-thread `channelInfo.parentId` into the existing fallback resolver, preserve direct channel overrides over category defaults and wildcard fallback, then update schema, docs, and focused preflight tests.

## Reproduction Assessment

Yes at source level, but this is a feature request rather than a broken documented behavior. A guild text channel whose fetched `parentId` matches a configured category entry will not inherit today because preflight only passes thread parent metadata into allowlist resolution.

## Solution Assessment

Unclear until maintainers choose the public category syntax and precedence model. Reusing the existing parent fallback looks like the narrowest implementation, but the access-control contract should be decided before automation changes it.
