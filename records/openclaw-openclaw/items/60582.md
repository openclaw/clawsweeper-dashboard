## Summary

Keep open. Current main still has only exact channel and supplied-parent fallback matching; Discord preflight does not feed ordinary guild text-channel category parent metadata into that resolver, and the public config/docs do not define category allowlist syntax.

## What This Changes

The issue requests Discord guild channel config entries that apply to every child channel under a category without per-channel config edits or gateway restarts.

## Best Possible Solution

Define a Discord-owned category allowlist syntax and precedence model, feed non-thread channelInfo.parentId into the existing resolver, then update schema, docs, and focused preflight/allowlist tests.

## Reproduction Assessment

Yes at source level. A fetched guild text channel with parentId matching a configured category entry will not inherit today because preflight only forwards thread parent metadata into allowlist resolution.

## Solution Assessment

Unclear until maintainers choose the public category syntax and access-control precedence. Reusing the existing parent fallback appears to be the narrowest implementation once that contract is decided.
