## Summary

Keep open: current main and v2026.6.10 still lack Discord category-level guild channel inheritance, and this remains the canonical tracker for a security-sensitive config and authorization decision.

## What This Changes

Request category-level Discord guild allowlist config so child channels inherit settings without per-channel config edits or gateway restarts.

## Best Possible Solution

Approve one Discord-owned category inheritance contract, then update resolver/preflight metadata, config types, schema, docs, audit/status behavior, and tests while preserving exact-channel override precedence.

## Reproduction Assessment

Yes. Source inspection shows the current Discord allowlist path has no category config field, no category lookup, and no preserved `parent_id` metadata for child-channel resolution; I did not run a live Discord repro.

## Solution Assessment

Unclear. Category inheritance belongs in the Discord plugin surface if maintainers approve it, but the best public contract and precedence model still need a product and security decision.
