- `impact:security`: Category inheritance would decide which Discord guild messages cross the configured channel authorization boundary.
- `impact:message-loss`: Messages in intended child channels are currently denied unless each channel is listed exactly or a broad wildcard is used.

## Summary

Keep open: current main and v2026.6.8 still do not provide Discord category-level guild channel inheritance, and the remaining work needs a maintainer-approved authorization syntax and precedence model.

## What This Changes

The issue requests Discord guild allowlist entries that can target a category so child channels inherit settings such as `requireMention: false` without per-channel config edits.

## Best Possible Solution

Approve one Discord-owned category inheritance contract, then update the allowlist resolver, preflight category metadata, exported types, schema, docs, status/audit behavior, and focused tests together while preserving exact-channel override precedence.

## Reproduction Assessment

Yes, at source level: a nonempty Discord guild `channels` map containing only a category key has no current-channel or immediate-parent match for ordinary child channels, and thread messages only get the immediate parent channel candidate. I did not run a live Discord repro in this read-only review.

## Solution Assessment

Unclear until maintainers choose the permanent syntax and precedence. Category inheritance belongs in the Discord allowlist/preflight path, but the proposed `category:<id>` form is only one possible current-schema contract.
