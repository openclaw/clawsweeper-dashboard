## Summary

Keep open: current main and v2026.6.10 still lack Discord category-level guild channel inheritance, and the closed related reports point back to this issue as the canonical tracker. The remaining work is real but needs maintainer-approved config syntax, precedence, and security review before implementation.

## What This Changes

Add Discord category-level guild allowlist inheritance so child channels can share allowlist and mention settings without per-channel config edits.

## Best Possible Solution

Approve one Discord-owned category inheritance contract and implement it across preflight metadata, resolver precedence, types/schema, docs, audit/status behavior, and focused tests.

## Reproduction Assessment

Yes, at source level: normal child-channel messages do not pass category parent metadata into allowlist matching, and the resolver has no category tier. I did not run a live Discord repro in this read-only review.

## Solution Assessment

Unclear until maintainers choose the permanent syntax and precedence. The maintainable implementation belongs in the Discord allowlist/preflight path, but `category:<id>` versus a `categories` map is still a product/security decision.
