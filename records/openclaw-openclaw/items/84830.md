## Summary

Keep open: the feature is useful, but the patch currently lets an old persisted budget snapshot survive session resets and then be rendered as the new session's estimated context usage.

## What This Changes

The PR adds a typed pre-prompt context-budget snapshot to embedded-agent metadata, persists it on session entries, exposes it on gateway session rows, and renders it as a `/status` fallback when fresh usage is absent.

## Best Possible Solution

Keep the reporting-only shape if maintainers accept the contract, but clear or session-id-guard `contextBudgetStatus` anywhere runtime token/session state is reset before merge.

## Reproduction Assessment

Yes for the review finding: source inspection shows reset helpers retain the new field while clearing token usage, and `/status` uses that field when totals are missing or zero. I did not run tests because this was a read-only review.

## Solution Assessment

No, not until the reset boundary is handled; the reporting approach is otherwise narrow, but persisted session state must be cleared or guarded when a new session is created.

- **[P2] Clear budget snapshots when resetting sessions:** `src/status/status-message.ts:855-858`
