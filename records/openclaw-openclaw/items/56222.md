## Summary

Keep open. Current main still omits a Gateway runtime version from the daemon status shape and read-probe path, while open PR #73225 is the already-paired implementation candidate and remains unmerged.

## What This Changes

Expose the running Gateway version in `openclaw gateway status --json` and the MCP-facing gateway status output for post-update verification.

## Best Possible Solution

Land a reviewed change that carries existing Gateway runtime version metadata through daemon probe, gather, JSON, text, and MCP-facing status surfaces, with tests covering the read-probe fallback when handshake metadata is unavailable.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: current main omits version from the daemon status `gateway` shape, and the `--require-rpc` path discards the `status.runtimeVersion` payload that could populate it.

## Solution Assessment

Yes. Reusing existing handshake `server.version` and `status.runtimeVersion` metadata is the narrow maintainable fix; because PR #73225 is already linked with closing syntax, this issue should stay open until that PR is reviewed and either landed or closed.
