## Summary

Keep open. Current main only partially surfaces a handshake-derived Gateway version and still lacks the requested daemon JSON `gateway.version` plus read-probe `status.runtimeVersion` fallback; the open implementation PR at https://github.com/openclaw/openclaw/pull/73225 directly closes this issue and should be handled first.

## What This Changes

Expose the running Gateway version in `openclaw gateway status` and MCP `gateway_status` structured output so update smoke tests can confirm the serving version.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73225, or an equivalent narrow patch, so the status response exposes `gateway.version` from handshake metadata with `status.runtimeVersion` as the read-probe fallback.

## Reproduction Assessment

Yes. Source inspection on current main shows the daemon status JSON shape still omits `gateway.version`, and the read-probe path still discards the existing `status.runtimeVersion` payload.

## Solution Assessment

Yes. Reusing existing handshake `server.version` plus the existing status RPC `runtimeVersion` is the narrow maintainable solution; no new RPC method, auth model, or config option is needed.
