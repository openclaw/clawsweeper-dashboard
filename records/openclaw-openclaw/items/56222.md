## Summary

Keep open. Current main still only has partial Gateway version display and does not expose the requested `gateway.version` JSON field or `status.runtimeVersion` read-probe fallback; the open implementation PR at https://github.com/openclaw/openclaw/pull/73225 carries closing syntax for this issue and should be reviewed first.

## What This Changes

Request to expose the running Gateway version in `openclaw gateway status --json` and MCP `gateway_status` output for post-update and operator smoke checks.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73225, or an equivalent narrow patch, so daemon status exposes `gateway.version` from handshake metadata with `status.runtimeVersion` as the read-probe fallback.

## Reproduction Assessment

Yes. Source inspection on current main shows `GatewayStatusSummary` and returned daemon JSON omit `gateway.version`, while the `--require-rpc` probe discards the status RPC payload that already contains `runtimeVersion`.

## Solution Assessment

Yes. Reusing existing handshake `server.version` and status RPC `runtimeVersion` is the narrow maintainable path; no new RPC method, auth model, or config option is needed.
