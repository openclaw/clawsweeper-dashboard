## Summary

Keep open. Current main still only partially covers the bundled report: `agents.list[].name` reaches some agent-list surfaces, but assistant identity/session-label paths still use different precedence, the topbar still has no token/cost/cache readout, and the QClaw Windows Electron upgrade/deletion path is not present in this repository.

## What This Changes

The issue asks for QClaw upgrades to stop deleting the Windows desktop app and for Control UI to honor configured agent names and restore visible token usage.

## Best Possible Solution

Split the report into owner-specific tracks: route the QClaw Windows installer deletion to the QClaw package owner, then define and implement one Control UI identity precedence contract and one deliberate usage-status placement.

No high-confidence in-repo reproduction exists for the QClaw upgrade deletion because that installer is not in this checkout.

## Solution Assessment

No for one automated OpenClaw core fix. The maintainable path is to separate the external installer issue from focused Control UI name-precedence and usage-display decisions, then land each with targeted tests after ownership is clear.
