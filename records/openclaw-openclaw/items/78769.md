## Summary

Keep open: current main preserves active session model/runtime state and has no opt-in propagation policy, so this is a new session/config feature that needs maintainer product judgment.

## What This Changes

The issue requests a config option to propagate agent model config changes into existing active sessions while preserving user-selected per-session model overrides.

## Best Possible Solution

If maintainers want this behavior, add an explicit opt-in session policy with schema/help/docs and focused tests that clear only non-user runtime model cache for affected agent defaults on reload/restart.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs give high confidence that current behavior preserves session model state unless reset, explicit `/model`, session patching, allowlist validation, or doctor cleanup changes it.

## Solution Assessment

Unclear until maintainer review: the proposed opt-in config is plausible and safer than always propagating, but it changes active-session continuity semantics and should be approved as product policy first.
