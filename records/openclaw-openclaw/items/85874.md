## Reproduction Assessment

No, not as written: failing before session setup is the right direction, but putting the validator inside the existing public SDK helper is a compatibility-sensitive API behavior change. A safer fix preserves the old helper and introduces an explicitly validated boundary or gets maintainer approval plus compatibility coverage.
