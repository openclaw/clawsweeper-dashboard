The issue requests avoiding a first-dispatch standalone plugin registry reload when a compatible gateway-startup registry is already active.

## Best Possible Solution

Land a compatibility-aware registry reuse fix with regression coverage for gateway-startup rich load options versus dispatch-time lean options, without weakening workspace, plugin-scope, or config-change guards.

## Solution Assessment
