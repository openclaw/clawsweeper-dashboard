## What This Changes

The issue reports a redundant first-dispatch plugin registry reload when dispatch-time lean load options do not match the richer gateway-startup cache key.

## Best Possible Solution

Yes for keeping this open. The best code direction is a compatibility-aware warm-registry reuse path with regression coverage, not closing from an open PR or using a broad fallback that weakens cache guards.
