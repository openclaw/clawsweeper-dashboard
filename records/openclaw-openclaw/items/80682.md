## What This Changes

The issue asks OpenClaw to avoid reloading the standalone runtime plugin registry on first inbound dispatch when gateway startup has already loaded a compatible active registry.

## Best Possible Solution

Land a compatibility-aware warm-registry reuse fix with regression coverage for rich gateway-startup options versus lean dispatch options, plus mismatch cases for workspace, config, plugin scope, and runtime mode.

Yes. The issue gives concrete real-host steps and logs, and current main source still traces dispatch through ensureRuntimePluginsLoaded into a strict compatibility miss followed by loadOpenClawPlugins; I did not run the live heap profile locally.
