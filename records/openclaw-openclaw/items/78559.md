## What This Changes

The PR marks Discord tool deliveries with a channelData progress flag, preserves marked text in the Discord reply sanitizer, and adds Discord monitor/delivery regression tests.

## Best Possible Solution

Keep the final-send boundary fix, but only preserve known-safe progress payloads or continue applying the trace-line scrub to marked payloads, then add a regression for unsafe marked tool text and attach real Discord proof.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: current main sends Discord replies through sanitizeDiscordFrontChannelText, whose trace regex strips Read and Exec labeled lines. I did not rerun a live Discord bot in this read-only review.

## Solution Assessment

No. The PR fixes the missing progress text at the right boundary, but marking every tool payload bypasses a sanitizer that intentionally strips internal trace labels; the narrower solution is to gate the marker or keep the trace scrub active for marked payloads.
