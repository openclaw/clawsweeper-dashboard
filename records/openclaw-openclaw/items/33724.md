## Risks / Open Questions

- A configurable backup directory or retention count can expose credential-bearing config copies unless permissions, path validation, and cross-filesystem behavior are designed together.
- Changing backup names or locations can break suspicious-config recovery because write, audit, observe, and restore paths currently assume `${configPath}.bak`.
- The trigger controls and UI placement are product policy, and overlapping work in https://github.com/openclaw/openclaw/issues/20460, https://github.com/openclaw/openclaw/issues/39923, https://github.com/openclaw/openclaw/pull/39982, and https://github.com/openclaw/openclaw/pull/40392 should be reconciled before implementation.
