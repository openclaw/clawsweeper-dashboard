## Summary

Keep open. This PR is still an active implementation candidate for the linked Docker skill-install problem, but the proposed apt fallback introduces a privileged package-manager trust change, still has a sudo interactivity bug, and lacks real Linux apt install proof.

## What This Changes

The PR adds a Linux apt-get fallback for missing-Homebrew `kind: "brew"` skill installers, refactors apt installs into a shared helper, and expands mocked fallback tests.

## Best Possible Solution

Choose one documented Linux/Docker dependency strategy that preserves the untrusted-skill boundary, then add focused regression coverage plus real Docker or Linux install proof.

## Reproduction Assessment

Yes. Source inspection shows current main returns the brew-missing failure for brew installers without Homebrew, and the PR diff shows the new apt-get execution path that creates the review findings; I did not run a live Docker or apt install.

## Solution Assessment

No. The proposed fallback addresses the visible failure, but it changes the privilege and trust model; a safer fix needs maintainer approval and either trusted/opt-in gating or one of the open Docker/onboarding alternatives.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Gate apt fallback to trusted skill sources:** `src/agents/skills-install.ts:553-555`
  - body: The new missing-brew branch invokes apt-get for any brew installer after only warning on non-bundled sources. Workspace and personal skill metadata can be attacker-controlled, so root or passwordless-sudo Linux hosts could run privileged package installs that current main rejects with a brew-missing error; gate this to trusted or allowlisted installers, or require explicit opt-in, before running apt.
  - confidence: 0.92
- **[P2] Keep sudo apt commands non-interactive:** `src/agents/skills-install.ts:327-328`
  - body: The shared apt helper probes sudo with `sudo -n true` but then runs `sudo apt-get update/install` without `-n`. If sudo allows the probe but prompts or denies apt-get, the installer can block at a password prompt until timeout; pass `-n` to the follow-up sudo commands or close stdin.
  - confidence: 0.78

## Security Review
