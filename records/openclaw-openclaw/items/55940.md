## Summary

Keep open: the PR targets a real Docker/Linux skill-install failure, but the current patch is not mergeable because it adds privileged apt installs from skill metadata, leaves follow-up sudo apt commands interactive, lacks after-fix real Linux proof, and now conflicts with main.

## What This Changes

The PR adds a Linux apt-get fallback for missing-Homebrew brew skill installers and expands mocked fallback tests.

## Best Possible Solution

Choose one documented Docker/Linux skill-dependency strategy; if apt fallback is selected, gate it to trusted or explicit opt-in installs, keep sudo non-interactive, add focused tests, and provide real Linux proof.

## Reproduction Assessment

Yes. Source inspection shows current main returns the brew-missing failure for a brew installer when no brew executable resolves, and the openai-whisper skill is still brew-only; I did not run a live Docker or apt install in this read-only review.

## Solution Assessment

No. The apt fallback addresses the visible symptom, but it is not the best mergeable solution until maintainers choose the package-manager trust posture and the patch gates privileged installs and keeps sudo non-interactive.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Gate apt fallback to trusted skill sources:** `src/agents/skills-install.ts:553-555`
  - body: This missing-brew branch runs apt-get for any brew installer metadata after only warning on non-bundled skill sources. Workspace and personal skill metadata can be attacker-controlled, so root or passwordless-sudo Linux hosts could now run privileged package installs that current main rejects with a brew-missing error; gate this to trusted or explicitly opted-in installers before invoking apt.
  - confidence: 0.92
- **[P2] Keep sudo apt commands non-interactive:** `src/agents/skills-install.ts:327-328`
  - body: The shared apt helper probes sudo with `sudo -n true` but then runs `sudo apt-get update/install` without `-n`. Since the command runner inherits stdin when no input is provided, sudo can still prompt or block until timeout when apt-get needs a password or is denied; pass `-n` to the follow-up sudo commands or close stdin.
  - confidence: 0.78
