## What This Changes

Adds a Linux apt-get fallback for missing-Homebrew brew skill installers, refactors apt install handling, and expands mocked fallback tests.

## Best Possible Solution

Choose one documented Docker/Linux skill-dependency strategy that preserves the untrusted-skill boundary; if apt fallback is selected, gate it to trusted or explicit opt-in installs, keep sudo non-interactive, add focused tests, and require real Docker/Linux proof.

No. The apt fallback addresses the visible error, but it is not the best mergeable solution until maintainers choose the package-manager trust posture; safer directions include trusted or opt-in apt gating, Linuxbrew Docker support, or container-aware filtering.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Gate apt fallback to trusted skill sources:** `src/agents/skills-install.ts:553-555`
  - body: This branch invokes apt-get for any brew installer after only warning on non-bundled skill sources. Workspace and personal skill metadata can be attacker-controlled, so root or passwordless-sudo Linux hosts could now run privileged package installs that current main rejects with a brew-missing error; gate the fallback to trusted or explicitly opted-in installers before running apt.
  - confidence: 0.92
- **[P2] Keep sudo apt commands non-interactive:** `src/agents/skills-install.ts:327-328`
  - body: The helper probes sudo with `sudo -n true` but then runs `sudo apt-get update/install` without `-n`. Since the command runner inherits stdin when no input is provided, sudo can still prompt or block until timeout in sudoers configurations where apt-get needs a password or is denied; pass `-n` to the follow-up sudo commands or close stdin.
  - confidence: 0.78
