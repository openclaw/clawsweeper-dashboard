## Summary

Keep open: the PR targets a real Docker/Linux skill-install failure, but the current diff turns arbitrary brew installer metadata into privileged apt installs and needs maintainer security/product judgment before merge.

## What This Changes

The PR refactors the skill apt installer helper and adds a Linux no-Homebrew fallback that installs brew formulas through apt-get, with regression tests for success and failure paths.

## Best Possible Solution

Resolve the Docker/Linux no-Homebrew skill path through one explicit product decision that preserves the untrusted-skill boundary, then document the chosen behavior.

## Reproduction Assessment

Yes. Source inspection shows current main returns the brew-missing failure for brew installers without Homebrew, and the PR diff shows the new apt-get execution path that creates the review findings.

## Solution Assessment

No. The proposed fallback fixes the visible install failure, but it is broader than the current trust model; a trusted-source gate, allowlist, explicit opt-in, Docker/Linuxbrew support, or container-specific hiding would be safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Gate apt fallback to trusted skill sources:** `src/agents/skills-install.ts:553-555`
  - body: The new no-brew branch invokes apt-get for any brew installer after only warning on non-bundled skill sources. Workspace and personal skill metadata can be attacker-controlled, so on root or passwordless-sudo Linux hosts this lets a third-party skill trigger privileged OS package installation where main currently returns a brew-missing error; gate this to trusted or allowlisted installers, or require explicit opt-in, before running apt.
  - confidence: 0.92
- **[P2] Keep sudo apt commands non-interactive:** `src/agents/skills-install.ts:327-329`
  - body: The helper probes with sudo -n true, but then runs sudo apt-get update/install without -n. If sudo allows the probe but prompts or denies apt-get, the installer can block at a password prompt until timeout; use sudo -n for the follow-up apt commands or otherwise close stdin.
  - confidence: 0.78
