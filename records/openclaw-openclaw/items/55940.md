## Summary

Keep open. Current main still has the missing-brew failure path, but this PR is not merge-ready because the apt fallback changes the skill-install trust boundary, leaves sudo apt calls interactive, and lacks real Linux behavior proof.

## What This Changes

The PR refactors apt installation into a shared helper, adds a Linux apt-get fallback for missing-Homebrew brew skill installers, and expands mocked fallback tests.

## Best Possible Solution

Choose one documented Docker/Linux skill-dependency strategy that preserves the untrusted-skill boundary, then add focused regression coverage and real Docker or Linux install proof.

## Reproduction Assessment

Yes, by source inspection. Current main returns the brew-missing failure for a brew installer when no brew executable resolves, and the reporter's openai-whisper example remains brew-only; I did not run a live Docker or apt install.

## Solution Assessment

No. The apt fallback addresses the visible error, but it is not the best solution until maintainers choose the package-manager trust posture; safer directions include trusted/opt-in apt gating, Linuxbrew Docker support, or container-aware filtering.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Gate apt fallback to trusted skill sources:** `src/agents/skills-install.ts:553-555`
  - body: The new missing-brew branch invokes apt-get for any brew installer after only warning on non-bundled skill sources. Workspace and personal skill metadata can be attacker-controlled, so root or passwordless-sudo Linux hosts could now run privileged package installs that current main rejects with a brew-missing error; gate this to trusted/allowlisted installers or explicit opt-in before running apt.
  - confidence: 0.92
- **[P2] Keep sudo apt commands non-interactive:** `src/agents/skills-install.ts:327-328`
  - body: The shared apt helper probes sudo with `sudo -n true` but then runs `sudo apt-get update/install` without `-n`. Since the command runner inherits stdin when no input is provided, sudo can still prompt or block until timeout in sudoers configurations where apt-get needs a password or is denied; pass `-n` to the follow-up sudo commands or close stdin.
  - confidence: 0.78
