## Summary

Keep this PR open, but do not merge it as-is. Current main still has the reported Linux no-Homebrew gap for brew-backed skill installers, while the PR’s apt fallback remains security-sensitive because it can turn installer metadata into root or passwordless-sudo package installation from non-bundled skills; related alternative fixes are still open and have not clearly superseded this PR.

## What This Changes

This PR refactors the skill apt installer helper and adds a Linux no-Homebrew fallback so brew installer specs can run apt-get or sudo apt-get install for the formula, with regression tests for root, sudo, failure, no apt-get, tap-qualified formulas, and macOS behavior.

## Best Possible Solution

Resolve the Docker/Linux no-Homebrew skill install failure through one explicit product path: either gate any apt fallback to trusted or allowlisted installers with clear opt-in semantics, adopt a Docker/Linuxbrew image solution, or hide unsupported brew-only install choices in container onboarding. The end state should preserve the current untrusted-skill boundary and document whichever Linux behavior is chosen.

## Reproduction Assessment

Yes. Current main can be traced directly: a Linux run with a brew installer and no brew executable reaches resolveBrewMissingFailure, while the PR’s own tests show the new root and sudo apt-get calls that create the review finding.

## Solution Assessment

No, not as written. The PR fixes the observable missing-dependency path, but the generic apt fallback is broader than the current trust model; a trusted-source gate, allowlist, explicit opt-in, or the Docker/onboarding alternatives would be safer.

Full review comments:

- **[P1] Gate apt fallback to trusted skill sources:** `src/agents/skills-install.ts:553-555`
  - body: The new no-brew branch invokes apt-get for any brew installer after only warning on non-bundled skill sources. Workspace and personal skill metadata can be attacker-controlled, so on root or passwordless-sudo Linux hosts this lets a third-party skill trigger privileged OS package installation where main previously returned a brew-missing error; gate this to trusted/allowlisted installers or explicit opt-in before running apt.
  - confidence: 0.92

  - body: The helper probes with sudo -n true, but then runs sudo apt-get update/install without -n. If sudo allows the probe but prompts or denies apt-get, the installer can sit at a password prompt until the timeout; use sudo -n for the follow-up apt commands or otherwise close stdin.
  - confidence: 0.78
