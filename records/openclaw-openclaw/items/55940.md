## Summary

Keep this PR open for maintainer security and product review. Current main still returns the Homebrew-missing failure for brew-kind skill installs without Homebrew, so the contribution is not implemented or obsolete, but the PR's apt fallback would expand skill installer metadata into root/passwordless-sudo package installation without source gating.

## What This Changes

The PR adds a Linux apt-get fallback for brew-kind skill installer specs when Homebrew is unavailable and adds tests for root, sudo, failure, tap-qualified formula, and macOS/no-apt paths.

## Best Possible Solution

Keep the PR open until maintainers choose the Docker/Linux skill-install direction. If apt fallback is accepted, it should be explicitly gated to trusted bundled/managed installers, a config or CLI opt-in, or a formula allowlist, with regression coverage for non-bundled sources that must continue to fail or prompt instead of running privileged package installs.

## Reproduction Assessment

Yes. A brew-kind installer on Linux with no Homebrew follows the current-main branch at src/agents/skills-install.ts:540-542 and returns the brew-missing failure; the PR tests model the proposed apt fallback paths, though I did not run tests because this review is read-only.

## Solution Assessment

No. The PR addresses the observable Docker failure for some plain formula names, but automatic root/passwordless-sudo apt install for arbitrary brew metadata is not the narrowest safe default; a trusted-source gate, explicit opt-in, allowlist, or the linked Docker Linuxbrew/onboarding alternatives would be safer.
