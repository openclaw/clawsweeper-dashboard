## Summary

Keep open: current main still has the Docker/Linux brew-skill failure, but this PR's apt fallback changes the installer trust boundary by running privileged package installs from skill metadata and overlaps other open solution paths for #14593.

## What This Changes

The PR refactors apt installer handling and adds a Linux no-Homebrew brew-installer fallback that runs apt-get or sudo apt-get for brew formulas, with fallback tests.

## Best Possible Solution

Choose one #14593 policy that preserves the untrusted-skill boundary, implement it narrowly, document the behavior, and close the other competing fix PRs once the canonical path lands.

## Reproduction Assessment

Yes. Source inspection shows current main returns the brew-missing failure for brew installers without Homebrew, and the PR diff shows the new apt-get execution path that creates the review findings.

## Solution Assessment

No. The proposed fallback fixes part of the visible install failure, but it is broader than the current trust model; a trusted-source gate, allowlist, explicit opt-in, Docker/Linuxbrew support, or container-specific hiding would be safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Restrict apt fallback to trusted brew installers:** `src/agents/skills-install.ts:553-555`
  - body: The new no-brew branch invokes apt-get for any brew installer after only warning on non-bundled skill sources. Workspace and personal skill metadata can be attacker-controlled, so on root or passwordless-sudo Linux hosts this lets a third-party skill trigger privileged OS package installation where main currently returns a brew-missing error; gate this to trusted or allowlisted installers, or require explicit opt-in, before running apt.
  - confidence: 0.92
- **[P2] Keep sudo apt commands non-interactive:** `src/agents/skills-install.ts:327-329`
  - body: The helper probes with sudo -n true, but then runs sudo apt-get update/install without -n. If sudo allows the probe but prompts or denies apt-get, the installer can block at a password prompt until timeout; use sudo -n for the follow-up apt commands or otherwise close stdin.
  - confidence: 0.78
