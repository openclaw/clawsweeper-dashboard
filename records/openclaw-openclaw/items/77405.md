## Summary

Keep this open: current `main` still has a source-level path that fragments assistant text around live tool events into separate Control UI assistant groups, matching the report.

## What This Changes

The issue asks Control UI Webchat to render assistant text from tool-calling turns as one coherent response, or at least coalesce adjacent text fragments instead of many bubbles.

## Best Possible Solution

Fix Control UI’s active-run item building or rendering so one assistant run displays coherent text around inline/collapsible tool cards, without adding a new `gateway.webchat` config option.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: an assistant delta sets `chatStream`, a tool start moves it to `chatStreamSegments`, and `buildChatItems` plus `renderStreamingGroup` render each segment as a separate assistant group.

## Solution Assessment

Yes. The narrow maintainable fix is in the Control UI frontend projection/grouping layer; changing Gateway event emission or adding a batching config would be broader than needed.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/77405. Control UI Webchat currently fragments assistant responses during tool-calling turns because `handleAgentEvent` flushes active `chatStream` into `chatStreamSegments`, `buildChatItems` renders each segment as a separate stream item, and `renderStreamingGroup` wraps each stream item in a separate assistant group/bubble. Make a narrow frontend fix so assistant text fragments from one active run display as a coherent assistant response, or at least adjacent assistant text fragments are coalesced, while preserving tool-card ordering, the show/hide tool calls toggle, reading indicator behavior, final `chat.history` reload semantics, and transcript durability. Do not add a `gateway.webchat` config option and do not change Gateway event emission unless source inspection proves the UI cannot solve it. Add focused regression coverage for a turn with text, tool card, more text, and another tool card. Add a one-line user-facing changelog fix entry.
