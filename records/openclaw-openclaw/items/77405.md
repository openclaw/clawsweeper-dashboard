## Summary

Keep open: current main still builds one stream item per assistant text segment around tool-call events, and each stream item renders as its own assistant group, matching the reporter's and prior ClawSweeper root-cause analysis.

## What This Changes

The issue asks Control UI Webchat to render a multi-tool assistant turn as a coherent assistant response instead of separate bubbles for each text fragment.

## Best Possible Solution

Coalesce text-only assistant stream fragments for a single active Control UI run while preserving interleaved tool cards and durable transcript semantics.

## Solution Assessment

Yes: the narrow maintainable fix is UI-side coalescing of active-run assistant text fragments while keeping tool-card interleaving/order. Adding a `gateway.webchat` streaming/batching config option would be unnecessary surface for a default rendering bug.
