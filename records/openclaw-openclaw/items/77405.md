## Summary

Keep open: current main still has a clear Control UI render path that fragments an active assistant turn around tool calls. Tool events freeze the current text into `chatStreamSegments`, `buildChatItems` emits each segment/current stream as separate stream items, and each stream item renders as its own assistant group.

## What This Changes

The issue requests Control UI WebChat to render assistant text from tool-calling turns as one coherent bubble, or at least merge adjacent assistant text fragments instead of stacking many separate bubbles.

## Best Possible Solution

Coalesce text-only assistant stream fragments for a single active run in the Control UI while preserving tool cards and transcript/history semantics, with focused regression coverage.

## Reproduction Assessment

Yes, source-reproducible: drive a text delta, then a tool event, then more text and another tool event; current main accumulates multiple `chatStreamSegments` and renders each one via `renderStreamingGroup`. I did not need a live browser run to confirm the code path.

## Solution Assessment

Yes: a narrow UI-side coalescing fix is the best path. Adding a `gateway.webchat` streaming/batching config option would be extra surface area for a default rendering bug.
