## What This Changes

Report that Control UI Webchat should keep an assistant tool-calling turn visually coherent instead of rendering each text fragment as a separate bubble.

## Best Possible Solution

Keep #63956 as the canonical tracker and implement a Control UI-only grouped streaming render path there, preserving tool-card ordering and transcript semantics.

## Reproduction Assessment

Yes. Source inspection on current main shows text-tool-text turns become multiple stream items and each stream item renders as its own assistant group; I did not need a live browser run to verify that path.

## Solution Assessment

Yes. The narrow fix belongs in the Control UI renderer/buildChatItems path under the canonical #63956 thread; adding gateway.webchat batching config would be extra surface area for a default rendering bug.
