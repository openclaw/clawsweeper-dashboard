## Repair Work Prompt

Fix issue https://github.com/openclaw/openclaw/issues/77405. In Control UI WebChat, active assistant text produced during a tool-calling turn should not render as many fragmented assistant bubbles. Keep tool cards visible and ordered, but coalesce text-only assistant stream fragments for a single active run into one coherent streaming group or equivalent single assistant group. Do not add a `gateway.webchat` config option. Preserve final `chat.history` reload semantics, tool-card expansion behavior, persisted transcript rendering, and duplicate-message collapse behavior. Add focused regression coverage for `buildChatItems` with multiple `streamSegments`, `toolMessages`, and live `stream`, plus any render/controller coverage needed. Add a user-facing changelog fix entry unless maintainers decide this is purely internal.
