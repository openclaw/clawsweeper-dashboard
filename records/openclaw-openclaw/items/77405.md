## Summary

Keep open: current main still has a source-reproducible Control UI rendering path that splits one assistant turn into multiple assistant bubbles around tool-call events, and no merged fix or duplicate was evident from the provided discussion plus local source inspection.

## What This Changes

The issue asks Control UI Webchat to coalesce assistant text fragments during tool-calling turns instead of rendering each fragment as a separate bubble.

## Best Possible Solution

Coalesce text-only assistant stream fragments for one active run in the Control UI render/build layer while preserving standalone tool cards and transcript semantics, with focused regression coverage for active streaming and history-backed turns.

## Reproduction Assessment

Yes, source-reproducible: an active assistant text delta followed by a tool event and then more assistant text creates multiple `chatStreamSegments`, and current main renders each segment through its own assistant streaming group. I did not run a live browser session, but the inspected source path is direct and deterministic.

## Solution Assessment

Yes, the narrow maintainable fix is UI-side coalescing of assistant text fragments for the same run. Adding a `gateway.webchat` batching/streaming config option would add product surface for a default rendering bug.
