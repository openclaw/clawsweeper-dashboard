Keep open: current main still fragments active Control UI assistant text around tool calls because text is split into stream segments, each segment is emitted as a separate stream item, and each stream item renders as its own assistant group.

## What This Changes

The issue requests that Control UI Webchat render assistant text around tool calls as one coherent bubble or coalesced adjacent text fragments.

## Best Possible Solution

Coalesce text-only assistant stream fragments for one active Control UI run while preserving interleaved tool cards and final history semantics, with focused regression coverage.

## Repair Work Prompt

Fix the Control UI Webchat fragmentation reported in openclaw/openclaw#77405. Current main moves active `chatStream` text into `chatStreamSegments` on tool events, `buildChatItems` emits every segment and the live stream as separate stream items, and `renderStreamingGroup` wraps each stream item in its own assistant group. Implement a narrow UI-side fix so text-only assistant fragments for one active run render as one coherent assistant bubble/group or otherwise coalesced adjacent text, while preserving tool card ordering/visibility and avoiding duplicate persisted assistant messages after the final `chat.history` reload. Do not add a `gateway.webchat` streaming/batching config option. Add focused regression coverage and a changelog entry if the UI behavior changes user-visible output.
