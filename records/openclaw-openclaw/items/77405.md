## Summary

Keep open: current main still has a clear Control UI render path that splits one active assistant turn into multiple assistant stream groups when tool events interrupt streamed text.

## What This Changes

Request that Control UI WebChat coalesce assistant text fragments during tool-calling turns so one assistant reply does not render as many separate bubbles.

## Best Possible Solution

Coalesce text-only assistant stream fragments for one active run in Control UI rendering while preserving tool cards and transcript/history semantics.

## Reproduction Assessment

Yes, source-reproducible: drive an active chat stream with text, then a tool event, then more text and another tool event; current main stores multiple chatStreamSegments and renders each segment as its own assistant stream group.

## Solution Assessment

Yes: a narrow UI-side coalescing fix is the best path. Adding a gateway.webchat streaming or batching config option would add product surface for a default rendering bug.
