## Summary

Keep this PR open, but it still needs changes before merge. The aggregation direction is valid and the PR includes useful real UI proof, but the current head drops existing `usage.cost` range presets and bypasses current main's durable cost-cache/cacheStatus contract.

## What This Changes

The PR aggregates gateway `usage.cost` across configured agents, scopes the cost cache by resolved agents, adds gateway/UI coverage, fixes date-only Usage UI labels, and adds a changelog entry.

## Best Possible Solution

Keep the aggregation and UI date fix, but rebase it onto current usage-cost cache plumbing: resolve the agent set, call the durable cached summary per agent with sync refresh, merge totals/daily/cacheStatus, and preserve all date-range params.

## Reproduction Assessment

Yes. Source inspection shows current main fans out `sessions.usage` across configured agents while `usage.cost` remains unscoped, and the PR body includes before/after screenshots for the visible UI symptom.

## Solution Assessment

No, not as-is. Aggregating across configured agents is the right narrow fix, but this head must preserve `params?.range` and the current durable `loadCostUsageSummaryFromCache`/`cacheStatus` contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Forward range presets into usage.cost:** `src/gateway/server-methods/usage.ts:587-593`
  - body: Current main accepts `params?.range` for `usage.cost`, but this handler rebuild omits it. Callers using presets such as `7d`, `90d`, or `all` will silently get the default window, so cost totals can disagree with the selected range.
  - confidence: 0.92
- **[P2] Use the durable cost cache per agent:** `src/gateway/server-methods/usage.ts:454-486`
  - body: The new fan-out calls raw `loadCostUsageSummary`, bypassing current main's durable cache refresh path and its `cacheStatus` contract. Route each resolved agent through `loadCostUsageSummaryFromCache` and merge statuses so the gateway avoids direct rescans and the UI keeps stale/refresh visibility.
  - confidence: 0.9
