## Summary

Keep open: this PR targets a real current-main gateway/UI usage mismatch and has credible real behavior proof, but the branch is stale against current main and would drop range presets plus the durable usage-cost cache path before merge.

## What This Changes

The PR updates gateway `usage.cost` to aggregate transcript-backed cost summaries across configured agents, adds agent-scope cache/test coverage, fixes Control UI date-only labels, and adds a changelog entry.

## Best Possible Solution

Keep the focused cross-agent aggregation, but rebase/adapt it so per-agent fan-out uses `loadCostUsageSummaryFromCache`, preserves merged cache status, forwards `range`, and keeps the current gateway/UI regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: configure multiple gateway agents with usage-bearing transcripts outside the default scope, call `usage.cost` and `sessions.usage` for the same range, and the former stays on an unscoped cost-summary path while the latter fans out across configured agents.

## Solution Assessment

No, not in the current branch state. The aggregation direction is the right narrow fix, but the implementation should be updated to preserve current-main `range` and durable-cache behavior before merge.
