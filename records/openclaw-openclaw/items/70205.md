## Summary

Keep open: the PR addresses a valid source-reproducible gateway/UI usage bug and includes sufficient real behavior proof, but the branch is stale against current main and would regress newer `usage.cost` range and durable-cache behavior unless updated before merge.

## What This Changes

The PR changes gateway `usage.cost` to aggregate transcript-backed cost summaries across configured agents, scopes the cache by agent set, and fixes Control UI date-only usage labels.

## Best Possible Solution

Update this PR or an equivalent replacement so agent aggregation is layered on current `loadCostUsageSummaryFromCache` behavior, forwards `range`, merges cache status safely, and keeps the existing gateway/UI regression coverage before landing.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: configure multiple gateway agents with usage-bearing transcripts outside the default scope, call `usage.cost` without `agentId`, and it stays on the unscoped cost-summary path while `sessions.usage` scans configured agents.

## Solution Assessment

No, not in the current branch state. The aggregation direction is the right narrow fix, but the branch should be rebased/adapted to preserve current-main `range` and durable-cache behavior before it is mergeable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Forward the range preset into usage.cost:** `src/gateway/server-methods/usage.ts:587-593`
  - body: Current main forwards `range: params?.range` into `parseDateRange` for `usage.cost`, but this branch rewrites the handler without that field. Calls from the Usage UI that use presets like `7d`, `90d`, or `all` would fall back to the default cost window while `sessions.usage` still uses the selected range, reintroducing inconsistent dashboard data. Please keep the `range` parameter when adding the agent scope.
  - confidence: 0.9
- **[P2] Preserve the durable cost cache path:** `src/gateway/server-methods/usage.ts:481-486`
  - body: Current main no longer has the gateway call raw `loadCostUsageSummary`; it uses `loadCostUsageSummaryFromCache` with `requestRefresh`/`sync-when-empty` and avoids memoizing `refreshing` summaries as fresh. The new per-agent fan-out calls the raw loader and drops merged `cacheStatus`, so merging this stale branch would undo the durable cache refresh behavior and can force full transcript scans again. Please build the per-agent aggregation on the cached loader and merge cache status.
  - confidence: 0.86
