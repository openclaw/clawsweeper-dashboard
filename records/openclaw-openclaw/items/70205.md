## Summary

Keep open: the aggregation and UI date-label fix are the right direction and the PR has real after-fix proof, but the current branch still regresses `usage.cost` range presets and bypasses the durable cost-summary cache path on current main.

## What This Changes

The PR changes gateway `usage.cost` to resolve configured agent scope, merge per-agent cost summaries, add cache-scope and aggregation tests, update Control UI usage date-label handling, and add a changelog entry.

## Best Possible Solution

Keep the aggregation and UI label fix, but implement it through per-agent cached summaries, merge cache status deliberately, and preserve current range-preset parsing.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure multiple gateway agents with usage-bearing transcripts, call `usage.cost` and `sessions.usage` over the same selected range, and compare current main's single-scope cost path against the PR's aggregation path.

## Solution Assessment

No, not as-is. The focused aggregation approach is maintainable, but it must preserve current-main range preset parsing and durable cached-loader behavior before merge.
