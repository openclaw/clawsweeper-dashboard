## Summary

Keep this PR open: it is a focused and useful gateway/UI bug fix with sufficient real behavior proof, but the current head still regresses two current-main contracts before it can merge.

## What This Changes

The PR changes gateway `usage.cost` to aggregate configured-agent cost summaries, adds gateway/UI coverage, updates the changelog, and fixes date-only Usage UI labels.

## Best Possible Solution

Keep the aggregation and UI label fix, but repair the gateway path to preserve `params?.range`, aggregate through the per-agent durable cached loader, and merge cache status for the combined response.

## Reproduction Assessment

Yes, from source inspection: configure multiple gateway agents with usage-bearing transcripts, call `usage.cost` and `sessions.usage` for the same range, and current main’s single-scope cost path can diverge from session usage while the PR’s intended path aggregates agents. I did not run a live gateway repro in this read-only pass.

## Solution Assessment

No, not as-is. The aggregation direction is the right narrow fix, but the current branch must preserve current-main range preset parsing and durable cached cost loading before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Forward range presets into usage.cost:** `src/gateway/server-methods/usage.ts:587-593`
  - body: Current main passes `params?.range` into `parseDateRange`, but this branch drops it when rebuilding the `usage.cost` handler. Preset callers using ranges such as `7d`, `90d`, or `all` will fall back to the default cost window, so the cost endpoint can disagree with the selected usage range.
  - confidence: 0.92
- **[P2] Use the durable cost cache per agent:** `src/gateway/server-methods/usage.ts:454-486`
  - body: The new aggregation helper calls raw `loadCostUsageSummary` for every resolved agent. Current main serves gateway cost summaries through `loadCostUsageSummaryFromCache` with sync refresh and `cacheStatus`, so this should fan out through the cached loader and merge cache status instead of reintroducing raw transcript scans.
  - confidence: 0.9
