## Summary

Keep open: the underlying multi-agent `usage.cost` bug is still valid and this PR has useful proof, but the current head should not merge because it regresses current-main durable usage cache behavior, range presets, and session lineage/cacheStatus handling.

## What This Changes

The PR updates gateway usage cost aggregation, related tests, Control UI usage date formatting, and the changelog so unscoped `usage.cost` can aggregate across configured agents.

## Best Possible Solution

Land a rebased or replacement patch that aggregates `usage.cost` across the same configured-agent scope as `sessions.usage` while preserving durable cache/cacheStatus, range presets, session lineage rollups, and the UI date-label fix.

## Reproduction Assessment

Yes, by source inspection: configure multiple gateway agents with usage-bearing transcripts outside the default transcript directory, then compare unscoped `usage.cost` with `sessions.usage` for the same range. I did not run a live gateway in this read-only review.

## Solution Assessment

No, not as-is. Aggregating by configured agent is the right narrow fix, but the current PR head must be adapted to current main so it does not drop durable cache status, range presets, or session lineage rollups.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Route agent scans through the durable usage cache:** `src/gateway/server-methods/usage.ts:454-486`
  - body: The aggregate path calls raw `loadCostUsageSummary` for each resolved agent, bypassing current main's `loadCostUsageSummaryFromCache` refresh/cacheStatus contract. That loses stale/refresh visibility and forces direct rescans instead of preserving the durable transcript aggregate cache.
  - confidence: 0.91
- **[P2] Forward range presets to parseDateRange:** `src/gateway/server-methods/usage.ts:587-593`
  - body: This handler rebuild omits `params.range` when calling `parseDateRange`, while current main supports presets such as `7d`, `90d`, and `all`. Callers using those presets would silently get the default date window.
  - confidence: 0.93
- **[P2] Preserve session lineage and cache status:** `src/gateway/server-methods/usage.ts:837-845`
  - body: Current main rolls `sessions.usage` across `includedSessionIds` and reads each through `loadSessionCostSummaryFromCache` so cache status reaches the UI. This stale head calls raw `loadSessionCostSummary` once for the listed session, dropping historical lineage totals and cacheStatus reporting.
  - confidence: 0.9
