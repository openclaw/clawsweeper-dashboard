## Summary

Keep open: the PR addresses a real multi-agent usage.cost bug and has sufficient visual proof, but the current head would regress range presets and the durable usage cache/cacheStatus contract on current main.

## What This Changes

The PR makes gateway usage.cost aggregate transcript-backed cost summaries across resolved gateway agents and updates Control UI date-only usage labels, tests, and changelog text.

## Best Possible Solution

Keep the aggregation and date-label behavior, but adapt aggregation to current durable usage cache by merging per-agent cached summaries and cacheStatus while preserving range presets.

## Reproduction Assessment

Yes. Source inspection shows usage.cost on current main still uses one cached summary path while sessions.usage discovers configured gateway agents, and the PR screenshots show the before/after dashboard symptom.

## Solution Assessment

No, not as-is. Aggregating across configured agents is the right narrow fix, but the branch must preserve params?.range and route each per-agent summary through loadCostUsageSummaryFromCache with merged cacheStatus.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Forward range presets into usage.cost:** `src/gateway/server-methods/usage.ts:587-593`
  - body: Current main accepts range presets for usage.cost, but this handler rebuild omits params?.range when calling parseDateRange. Callers using presets such as 7d, 90d, or all would fall back to the default window and get cost totals for the wrong selected range.
  - confidence: 0.92
- **[P2] Use the durable cost cache per agent:** `src/gateway/server-methods/usage.ts:454-486`
  - body: The new fan-out calls raw loadCostUsageSummary for each scope, bypassing current main's durable loadCostUsageSummaryFromCache path and its cacheStatus contract. Route each resolved agent through the durable cached loader and merge statuses so the gateway avoids direct rescans and the UI keeps stale/refresh visibility.
  - confidence: 0.9
