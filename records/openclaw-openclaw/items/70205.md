## Summary

Keep open: the PR targets a real source-visible multi-agent usage mismatch and has sufficient visual proof, but the current head still regresses current-main range presets and durable usage-cost cache behavior, so it should not merge as-is.

## What This Changes

The branch adds gateway helpers and tests to aggregate `usage.cost` across configured agents, updates agent-scoped cache keys, fixes Usage UI date-only labels, and adds a changelog entry.

## Best Possible Solution

Keep the aggregation and UI label fix, but implement it through the current durable per-agent cost cache, preserve all date-range inputs, and merge cache status for the combined response.

## Reproduction Assessment

Yes. From source inspection, configure multiple gateway agents with usage-bearing transcripts outside the default agent, then call `sessions.usage` and `usage.cost` for the same range; current main fans out session usage while cost usage remains scoped to one transcript scope.

## Solution Assessment

No, not as-is. Aggregating across configured agents is the right narrow fix, but the PR must preserve `params?.range` and use `loadCostUsageSummaryFromCache` per agent instead of raw transcript scans.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Forward range presets into usage.cost:** `src/gateway/server-methods/usage.ts:587-593`
  - body: Current main passes `params?.range` into `parseDateRange`, but this branch omits it when rebuilding the `usage.cost` handler. Preset callers using ranges such as `7d`, `90d`, or `all` will fall back to the default cost window, so cost totals can disagree with the selected usage range.
  - confidence: 0.92
- **[P2] Use the durable cost cache per agent:** `src/gateway/server-methods/usage.ts:454-486`
  - body: The aggregation helper fans out through raw `loadCostUsageSummary`, bypassing `loadCostUsageSummaryFromCache` and its sync refresh plus `cacheStatus` contract. That reintroduces direct transcript scans and drops cache status from gateway cost responses that current main already preserves.
  - confidence: 0.9
