## Summary

Keep open, but do not merge as-is. The aggregation idea and contributor proof are useful, and the Greptile review/comment thread supports the original direction, but current `main` has since changed the usage scope and cache contract in the same area; this branch is now conflicting and would need a rebase plus maintainer scope decision before it is safe.

## What This Changes

The PR changes gateway `usage.cost` so unscoped requests aggregate configured agent cost summaries, updates cache scoping and gateway/UI tests, and fixes Control UI date-only usage labels.

## Best Possible Solution

Rebase the PR onto current main and make `usage.cost` share the same explicit/default/all-agents scope contract as `sessions.usage`, while preserving `loadCostUsageSummaryFromCache` background refresh and merged cache-status behavior; if all-agents dashboard scope is desired, add it as a shared product/UI scope instead of only changing the cost endpoint.

## Reproduction Assessment

Yes, from source inspection rather than an executed local repro: with a configured default agent that is not `main`, current `sessions.usage` resolves the default agent while current `usage.cost` calls the cost cache without an agent id and therefore reads the `main` transcript cache. The PR's screenshots also show the reported UI symptom improving on the branch.

## Solution Assessment

No, not as currently proposed against main. The aggregation direction is plausible, but the branch must first reconcile current main's default-agent scope and durable background cache contract; otherwise it can recreate dashboard scope drift and availability regressions.

Full review comments:

- **[P1] Match the current sessions.usage scope:** `src/gateway/server-methods/usage.ts:353`
  - body: Current main now locks list-style `sessions.usage` without `agentId` to the default agent, and the Control UI requests both `sessions.usage` and `usage.cost` without an agent id. Returning every configured agent here makes the daily cost chart cover a broader scope than the header/session rows, recreating the dashboard mismatch this PR is trying to fix. Either keep no-agent `usage.cost` on the same default-agent scope, or make an explicit all-agents scope a shared `sessions.usage`/UI contract.
  - confidence: 0.92
- **[P1] Preserve background cost-cache refreshes:** `src/gateway/server-methods/usage.ts:454-486`
  - body: Current main routes `usage.cost` through `loadCostUsageSummaryFromCache({ requestRefresh: true, refreshMode: "background" })` so large profiles do not block the request path and the UI receives cache-status metadata. This helper calls `loadCostUsageSummary` directly for each agent, so resolving the conflict by taking this branch would drop the durable cache path, cache status, and the merged availability fix. Rebase the aggregation on top of `loadCostUsageSummaryFromCache` and merge per-agent `cacheStatus` values.
  - confidence: 0.9
