## Summary

Keep open: the fix direction is useful and the PR includes convincing UI proof, but the live branch is merge-conflicting and stale against current main's background usage-cache path. As written, the aggregation uses the raw cost loader and can reintroduce blocking `usage.cost` scans unless it is rebased onto the current durable cache behavior.

## What This Changes

The PR aggregates gateway `usage.cost` totals and daily buckets across configured agents, adds explicit agent scoping/validation tests, and changes Control UI date-only usage labels to avoid timezone day shifts.

## Best Possible Solution

Rebase the aggregation onto current main's durable background cost cache, preserve range presets/cacheStatus/session-lineage behavior, and keep explicit agent scoping plus the UI date-label fix.

## Reproduction Assessment

Yes, from source and supplied proof: current main's `usage.cost` path is unscoped while `sessions.usage` discovers configured agents, and the PR screenshots show the daily chart changing from `No data` to populated data. I did not run a live gateway repro in this read-only review.

## Solution Assessment

No as currently written against current main. The aggregation direction is narrow and plausible, but the implementation must be rebased onto `loadCostUsageSummaryFromCache` instead of bypassing the background cache path.
