## Summary

Keep this PR open. Current main still has the `usage.cost` scope mismatch: the handler calls the transcript cost loader without an agent scope, while `sessions.usage` already discovers configured agents. The PR is a focused fix for that endpoint/UI behavior, and I did not find a blocking correctness or security issue in the reviewed diff.

## What This Changes

The PR changes gateway `usage.cost` to resolve and validate configured agent scopes, merge transcript-backed cost summaries across agents with scoped caching, and adds Control UI date-only label handling plus gateway/UI regression tests.

## Best Possible Solution

Land a focused aggregation fix for `usage.cost` using the existing gateway agent scope, while keeping broader dedicated usage logging, cost-unit normalization, and OpenRouter billed-cost accuracy work in their narrower follow-ups such as #13219 and #68066.

## Reproduction Assessment

Yes. A multi-agent config with cost-bearing transcripts outside the default transcript directory reproduces the mismatch on current main because `usage.cost` is unscoped while `sessions.usage` scans configured agents; the PR adds mocked gateway tests for that exact path.

## Solution Assessment

Yes. The PR uses the existing `listAgentsForGateway` scope, validates explicit agent requests, scopes cache keys by resolved agents, and isolates partial scan failures, which is the narrowest maintainable fix for this endpoint mismatch without taking on the broader usage-log proposal.
