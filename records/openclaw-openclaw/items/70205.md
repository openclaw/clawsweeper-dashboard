## Summary

Keep open: the bug is valid and the aggregation direction is useful, but the current PR head would regress range presets and durable cost-cache behavior that current main now relies on.

## What This Changes

The PR changes gateway `usage.cost` to resolve configured agents, aggregate per-agent cost summaries with scoped cache keys and partial-failure handling, update tests, add a changelog entry, and render date-only UI labels locally.

## Best Possible Solution

Keep the cross-agent aggregation, but adapt it to call the durable cached cost loader per resolved agent, merge `cacheStatus`, preserve selected `range`, and retain the UI date-label fix with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure multiple agents with usage-bearing transcripts outside the default scope, call `sessions.usage` and `usage.cost` for the same range, and current main fans out only the former.

## Solution Assessment

No, not in the current branch state. The aggregation direction is the right narrow fix, but the branch must preserve current-main range handling and durable-cache semantics before merge.
