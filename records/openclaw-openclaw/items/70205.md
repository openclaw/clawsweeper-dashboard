## Summary

Keep open: the PR targets a real gateway Usage dashboard scope bug and includes credible before/after proof, but the latest head still regresses current `usage.cost` range handling and bypasses the durable transcript aggregate cache.

## What This Changes

The branch changes gateway `usage.cost` to resolve configured agent scopes, merge per-agent cost summaries, key the cache by agent scope, add gateway/UI tests, update the changelog, and preserve date-only Control UI usage labels.

## Best Possible Solution

Keep the focused aggregation and UI date-label fix, but repair the gateway path so each resolved agent is read through the durable cached loader, merged with cache status, and parsed with the same date-range contract as current main.

## Reproduction Assessment

Yes at source level. Configure multiple gateway agents with usage-bearing transcripts, call `sessions.usage` and unscoped `usage.cost` for the same selected range, and current main fans out sessions while cost stays on one transcript scope.

## Solution Assessment

No, not as-is. The aggregation direction is the narrow maintainable fix, but this branch must preserve current-main `range` parsing and durable cache/cacheStatus behavior before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Forward range presets into usage.cost:** `src/gateway/server-methods/usage.ts:587-593`
  - body: Current main passes `params?.range` into `parseDateRange`, but this rewrite omits it. Calls using presets such as `7d`, `90d`, or `all` would fall back to the default cost window while `sessions.usage` still honors the selected range.
  - confidence: 0.92
- **[P2] Fan out through the durable cost cache:** `src/gateway/server-methods/usage.ts:454-486`
  - body: Current main serves `usage.cost` via `loadCostUsageSummaryFromCache({ requestRefresh: true, refreshMode: "sync-when-empty" })` and preserves cache status. This helper calls raw `loadCostUsageSummary` per agent, which reintroduces full transcript scans and drops refresh/cache status from the response.
  - confidence: 0.88
