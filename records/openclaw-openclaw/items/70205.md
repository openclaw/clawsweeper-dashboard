## Summary

Keep open: this is still a useful focused fix for the gateway usage scope mismatch, but the current PR head would regress durable usage-cache status, range presets, and session lineage rollups from current main.

## What This Changes

This PR aggregates unscoped gateway `usage.cost` across configured agents, adds gateway/UI coverage, fixes date-only usage labels, and updates the changelog.

## Best Possible Solution

Land a rebased or replacement patch that aggregates `usage.cost` over the same configured-agent scope as `sessions.usage` while preserving durable cache/cacheStatus, range presets, session lineage rollups, and the UI date-label fix.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure multiple gateway agents with usage-bearing transcripts outside the default agent, then compare unscoped `usage.cost` with `sessions.usage` for the same range; current main fans out `sessions.usage` but leaves `usage.cost` on the default cost scope.

## Solution Assessment

No, not as-is. Aggregating by configured agent is the right narrow fix direction, but this PR head must be adapted to current main so it preserves durable cache status, range presets, and session lineage rollups.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Route agent scans through the durable cache:** `src/gateway/server-methods/usage.ts:454-486`
  - body: The aggregate path calls raw `loadCostUsageSummary` for each resolved agent, bypassing current main's durable `loadCostUsageSummaryFromCache` refresh/cacheStatus contract. That loses stale/refresh visibility and can cache a refreshing summary as fresh in the gateway layer.
  - confidence: 0.91
- **[P2] Forward range presets to date parsing:** `src/gateway/server-methods/usage.ts:587-593`
  - body: The rebuilt `usage.cost` handler no longer passes `params.range` into `parseDateRange`, while current main supports presets such as `7d`, `90d`, and `all`. Callers using those presets would silently get the default date window.
  - confidence: 0.93
- **[P2] Preserve session lineage usage rollups:** `src/gateway/server-methods/usage.ts:837-845`
  - body: Current main rolls `sessions.usage` across `includedSessionIds` and reads each through `loadSessionCostSummaryFromCache`. This stale head loads one raw session summary, dropping historical lineage totals and cache status reporting.
  - confidence: 0.9
