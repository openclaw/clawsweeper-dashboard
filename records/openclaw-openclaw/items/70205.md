## Summary

Keep open: the aggregation bug is real and this PR has useful after-fix proof, but the current head would regress newer `usage.cost` range-preset and durable aggregate-cache behavior now on `main`.

## What This Changes

This PR changes gateway `usage.cost` to resolve configured agent scopes, merge per-agent cost summaries, update cache coverage, and preserve Control UI date-only daily labels.

## Best Possible Solution

Keep the focused cross-agent aggregation, but implement per-agent fan-out through `loadCostUsageSummaryFromCache`, merge `cacheStatus`, forward `range`, and retain the UI date-label fix.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: configure multiple gateway agents with usage-bearing transcripts, call `usage.cost` and `sessions.usage` for the same range, and current main keeps `usage.cost` on an unscoped cost path while `sessions.usage` fans out across configured agents.

## Solution Assessment

No, not as-is. The aggregation direction is the narrow maintainable fix, but the branch should be repaired to preserve current-main `range` parsing and durable cache behavior before merge.

## Review Findings

Overall correctness: patch is incorrect

  - confidence: 0.88
