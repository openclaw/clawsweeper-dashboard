## Summary

Keep open: the PR targets a real gateway usage bug and includes useful proof, but the current head still regresses current-main range handling and durable cost-cache behavior before merge.

## What This Changes

The PR changes gateway `usage.cost` to aggregate cost summaries across configured agents, scopes cache keys by agent set, adds gateway/UI tests, fixes Control UI date-only labels, and updates the changelog.

## Best Possible Solution

Keep the focused cross-agent aggregation and UI date-label fix, but fan out through `loadCostUsageSummaryFromCache`, merge cache status, and preserve current-main range parsing.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure multiple gateway agents with usage-bearing transcripts, call `usage.cost` and `sessions.usage` for the same selected range, and current main keeps those date/range and cache contracts that the PR head does not fully preserve.

## Solution Assessment

No, not as-is. The aggregation direction is the right narrow fix, but the branch should preserve range presets and durable cached-loader behavior before merge.
