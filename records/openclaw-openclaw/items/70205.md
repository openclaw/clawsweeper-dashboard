## Summary

Keep open: the PR addresses a real multi-agent usage-cost scope bug, but the current head still drops existing `usage.cost` range preset forwarding and bypasses the durable transcript aggregate cache path that current main uses.

## What This Changes

The PR changes gateway `usage.cost` to resolve configured agent scope, merge per-agent daily/totals, cache by agent scope, add gateway/UI coverage, and fix Control UI date-only usage labels.

## Best Possible Solution

Keep the focused aggregation and UI label fix, but update the gateway path to pass `params?.range`, fan out through `loadCostUsageSummaryFromCache`, merge cache status, and preserve agent-scoped cache keys.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure multiple agents with usage-bearing transcripts, call `usage.cost` and `sessions.usage` over the same selected range, and compare PR-head behavior against current main's range and cache contracts.

## Solution Assessment

No, not as-is. The aggregation direction is the right narrow fix, but the branch should preserve current-main range preset parsing and durable cached-loader behavior before merge.
