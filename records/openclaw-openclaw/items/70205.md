## Summary

Keep open: the bug and fix direction are valid, and the PR has useful real UI proof, but the current head is stale/conflicting against main and would regress current durable usage-cache, range preset, and session-lineage behavior.

## What This Changes

The PR aggregates unscoped gateway `usage.cost` across configured agents, adds gateway/UI coverage, fixes date-only Control UI day labels, and adds a changelog entry.

## Best Possible Solution

Land a rebased or replacement patch that aggregates `usage.cost` over the same configured-agent scope as `sessions.usage` while preserving durable cache/cacheStatus, range presets, session lineage rollups, and the UI date-label fix.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure multiple gateway agents with usage-bearing transcripts outside the default agent, then compare unscoped `usage.cost` with `sessions.usage` for the same range; current main fans out `sessions.usage` but keeps `usage.cost` on the default cost scope.

## Solution Assessment

No, not as-is. Aggregating by configured agent is the right narrow fix direction, but this PR head must be adapted to current main so it preserves durable cache status, range presets, and session lineage rollups.
