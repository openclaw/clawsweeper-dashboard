## Best Possible Solution

Land the aggregation after rebasing onto current main, preserving background cache refresh/cacheStatus semantics and covering both multi-agent aggregation and the current cache-refresh contract in focused tests.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: call `usage.cost` without `agentId` in a multi-agent setup and it currently resolves the default agent transcript cache, while `sessions.usage` enumerates configured gateway agents. I did not run the gateway because this review is read-only.

## Solution Assessment

No, not as currently rebased: the aggregation direction is the right fix, but the PR head must be updated to preserve current main's background cache-refresh contract before merge. The safer implementation is the same agent aggregation layered on top of `loadCostUsageSummaryFromCache` per resolved agent.
