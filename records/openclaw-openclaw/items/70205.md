## Summary

Keep this PR open: the cross-agent aggregation and UI date-label fix are useful, but the current branch still needs the previously identified gateway repair before merge because it bypasses the existing durable cost cache and drops `range` preset forwarding from `usage.cost`.

## What This Changes

The PR changes gateway `usage.cost` to resolve configured agent scopes, merge per-agent cost summaries, update cost-cache coverage, add gateway/UI regression tests, and preserve date-only usage labels in the Control UI.

## Best Possible Solution

Keep the focused aggregation and UI date-label fix, but repair the gateway implementation so per-agent fan-out preserves current-main `range` parsing and durable cache semantics before merge.

## Reproduction Assessment

Yes. Source inspection and the PR proof give a high-confidence path: configure multiple gateway agents with usage-bearing transcripts, call `usage.cost` without `agentId`, and compare it with `sessions.usage` for the same range.

## Solution Assessment

No, not as currently proposed. The aggregation direction is the narrow maintainable fix, but the branch should forward `range` and aggregate through the durable cached loader before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Forward range presets into usage.cost:** `src/gateway/server-methods/usage.ts:587-593`
  - body: Current main forwards `range: params?.range` into `parseDateRange`, but this rewrite drops it. Calls using presets such as `7d`, `90d`, or `all` can fall back to the default cost window while `sessions.usage` still uses the selected range, leaving dashboard data inconsistent.
  - confidence: 0.92
- **[P2] Use the durable cost cache for each agent:** `src/gateway/server-methods/usage.ts:454-486`
  - body: Current main serves gateway cost summaries through `loadCostUsageSummaryFromCache` with refresh and `cacheStatus` handling, but the new fan-out calls raw `loadCostUsageSummary`. That can undo durable aggregate cache behavior and force full transcript scans again, so the per-agent path should use the cached loader and merge cache status.
  - confidence: 0.88
