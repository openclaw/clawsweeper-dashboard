## What This Changes

The issue reports intermittent builtin `memory_search` failures where the live tool returns `index metadata is missing` even though the durable SQLite memory index is valid.

## Best Possible Solution

Land or replace a focused memory-core fix that isolates or serializes safe reindex from live search/status, preserves true provider/model/settings mismatch blocking, and includes deterministic race coverage.

## Reproduction Assessment

## Solution Assessment

No fixed solution is on main yet. The best fix is manager-level reindex/search/status coordination with regression coverage, not only preserving hits in the tool wrapper.
