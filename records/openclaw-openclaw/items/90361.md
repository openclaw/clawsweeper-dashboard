## Summary

Keep open: current main has partial memory metadata/reindex cleanup, but it still lacks the active reader/reindex guard for this race class, and an open closing PR owns the direct fix path.

## What This Changes

The issue asks OpenClaw to stop builtin memory_search from returning disabled/unavailable with missing index metadata during overlapping search/reindex work when the durable memory index is valid.

## Best Possible Solution

Land or replace one manager-level memory-core fix that prevents search/status readers from observing in-progress reindex state, preserves real provider/model mismatch failures, and coordinates adjacent stale-handle and durability fixes.

## Reproduction Assessment

Yes, at source level: current main still has the shared-manager temp-DB reindex window plus post-search status fail-closed behavior. I did not run a live concurrent gateway reproduction in this read-only review.

## Solution Assessment

Yes for keeping this issue open behind the manager-level fix path; a tool-wrapper-only mitigation would be too narrow. The final implementation still needs maintainer consolidation across overlapping memory-core lifecycle PRs.
