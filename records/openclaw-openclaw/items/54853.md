## Summary

Keep open. Current main still does not provide the requested automatic per-agent target-channel account binding for Feishu tool calls from another channel; the existing behavior only covers explicit Feishu tool parameters, same-id contextual account routing, and Feishu default-account fallback.

## What This Changes

Request support for binding an agent to a target channel accountId so, for example, a DingTalk-originated agent can automatically use the correct Feishu account when calling Feishu tools.

## Best Possible Solution

Add a generic, documented target-channel account resolution contract keyed by active agentId plus target channel. Feishu should preserve explicit tool accountId first, then use the agent's bound Feishu account, then valid same-id contextual routing when appropriate, then channels.feishu.defaultAccount/runtime fallback; update config types/schema/help/docs and add DingTalk-to-Feishu regression coverage.
