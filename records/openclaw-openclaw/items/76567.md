## Summary

Keep open because this valid bug report is paired with an open same-author fix PR. Current main already has slot-scoped memory runtime loading and regression coverage, but cleanup should resolve the linked PR and this issue together rather than closing only one side.

## What This Changes

Runtime memory checks should consult or load the plugin configured in `plugins.slots.memory` before reporting that no active memory runtime exists.

## Best Possible Solution

Resolve the linked fix PR and this issue together, keeping the narrow slot-scoped runtime load and regression coverage already present on main.

## Reproduction Assessment

Yes. The source path is clear: status, doctor, or memory-search checks can ask the memory runtime facade before a full runtime registry is initialized, and the regression coverage now exercises loading the configured slot.

## Solution Assessment

Yes. Loading only the configured memory slot before returning unavailable is the narrow maintainable fix and avoids broad plugin activation; the remaining action is PR/issue disposition, not a new design.
