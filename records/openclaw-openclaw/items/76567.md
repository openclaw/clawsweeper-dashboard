## What This Changes

Runtime memory checks should load the plugin selected by `plugins.slots.memory` before reporting no active memory runtime.

## Best Possible Solution

Keep the shipped current-main implementation and handle #76495 separately as superseded by #76393 if it remains open.

## Reproduction Assessment

## Solution Assessment

Yes. The current implementation is the narrow maintainable fix: consult normalized `plugins.slots.memory`, respect disable/deny semantics, reuse any compatible active registry, and standalone-load only the configured plugin when needed.
