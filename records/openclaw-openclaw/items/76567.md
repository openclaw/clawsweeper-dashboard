## Summary

Keep open because this issue is paired with the same-author open PR #76495, even though current main already contains the slot-scoped memory runtime loading behavior and regression coverage.

## What This Changes

Runtime memory checks should consult and load the plugin named by `plugins.slots.memory` instead of returning no active memory runtime when the registry is not preloaded.

## Best Possible Solution

Keep the current slot-scoped runtime loading on main, then resolve the paired PR as already implemented or retarget it only if it still carries unique release-branch work.

## Reproduction Assessment

No. I did not establish a failing current-main reproduction; source inspection shows current main now loads the configured memory slot and the regression test covers that path.

## Solution Assessment

Yes. The current implementation is the narrow maintainable fix because it loads only the configured memory slot, respects disabled/denied plugin config, and avoids broad plugin loading.
