## Summary

Keep open. Current main and the latest release still allow memory search to index both workspace memory summaries and raw session transcripts, and the public docs still lack the requested overlap warning.

## What This Changes

Document that `session-memory` hook output and `memorySearch.sources["sessions"]` overlap, and recommend using one source path for the same conversations unless duplicate indexing is intentional.

## Best Possible Solution

Add concise guidance in the memory config and hooks docs recommending either raw session transcript indexing or session-memory summaries for the same conversation, leaving runtime dedupe and output-path configurability to separate design work.

## Reproduction Assessment

Yes, at source level. Current main still allows `sources: ["memory", "sessions"]` with `experimental.sessionMemory`, while the session-memory hook writes summaries under `<workspace>/memory/`, so two independent index paths can ingest the same conversation content.

## Solution Assessment

Yes for the immediate scope. A docs warning is the narrowest maintainable fix for the reported expectation; provenance-aware dedupe or configurable hook output should stay separate follow-up work.
