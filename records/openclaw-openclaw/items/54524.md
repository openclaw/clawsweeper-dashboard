## Summary

Keep open. Current main still permits the reported overlap: session transcript indexing and session-memory hook Markdown output can both feed memory search, and the public docs still do not warn users that combining them can duplicate session-derived content and embedding work.

## What This Changes

The issue asks for documentation or design guidance to avoid duplicate indexing when `memorySearch.experimental.sessionMemory` with `sources: ["memory", "sessions"]` is used alongside the bundled `session-memory` hook.

## Best Possible Solution

Add concise public guidance to the memory config and session-memory hook docs recommending one memory path unless users intentionally want both; leave runtime dedupe or configurable output directories to separate design work.

## Reproduction Assessment

Yes, at source level. Current main accepts the combined `memory` and `sessions` sources with session memory enabled, writes session-memory output into workspace memory, and scans that directory as the memory source; I did not run live indexing because this was a read-only review.

## Solution Assessment

Yes for the immediate docs/design request. A narrow docs warning is the lowest-risk maintainable fix; provenance-aware query dedupe or configurable hook output should remain separate follow-up work.

## Repair Work Prompt

Fix the docs-only overlap reported in https://github.com/openclaw/openclaw/issues/54524. Add concise public guidance explaining that `memorySearch.experimental.sessionMemory` with `sources: ["memory", "sessions"]` overlaps with the bundled `session-memory` hook because the hook writes session-derived Markdown into `<workspace>/memory/`, which the normal memory source indexes. Recommended boundary: update `docs/reference/memory-config.md` and `docs/automation/hooks.md`, and mirror any user-facing CLI hook wording in `docs/cli/hooks.md` if needed. Tell users to use `sources: ["memory"]` when relying on the hook, or disable/redirect the hook if they intentionally index raw session transcripts; do not add runtime dedupe, new config, or output-directory behavior in this PR. Coordinate wording with https://github.com/openclaw/openclaw/issues/60034 and mention that runtime/outputDir changes are separate design work. Keep OpenClaw docs terminology and Mintlify link rules; no changelog is needed for docs-only guidance unless maintainers request one.
