## What This Changes

This issue asks for documentation or design guidance so users do not combine the bundled session-memory hook with raw session transcript indexing and duplicate session-derived memory results.

## Best Possible Solution

Land or intentionally close https://github.com/openclaw/openclaw/pull/93093; if maintainers want stronger behavior, track provenance-aware runtime dedupe as separate memory design work.

## Reproduction Assessment

Yes, at source level. Current main can enable both memory and sessions sources, the hook writes summaries into the indexed memory tree, and session rows are indexed separately; I did not run a live memory search because the docs fix path is already clear.

## Solution Assessment

Yes for the current docs-scoped issue. The narrow maintainable fix is to warn in both the memory config reference and hook docs; runtime dedupe or a new output-directory option would be broader follow-up work.
