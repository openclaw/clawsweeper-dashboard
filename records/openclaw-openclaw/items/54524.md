## Summary

Keep open. Current main still allows the reported overlap, the public docs still describe session transcript indexing and the session-memory hook separately without warning about duplicate session-derived content, and the related output-directory PR remains open and only partially mitigates the issue.

## What This Changes

Document or design a mitigation for duplicate memory-search indexing when session transcript indexing and the session-memory hook both ingest the same conversation.

## Best Possible Solution

Add concise public docs guidance that these mechanisms overlap, recommending `sources: ["memory"]` when relying on the session-memory hook or disabling/redirecting the hook when indexing raw session transcripts.

## Reproduction Assessment

Yes. I did not run a live embedding search, but current-main source inspection gives a high-confidence path: enable `experimental.sessionMemory` with both sources and enable the session-memory hook, which lets two independent index paths ingest session-derived content.

## Solution Assessment

Yes for the immediate report. A narrow docs repair is the lowest-risk maintainable solution; provenance-aware runtime dedupe or a hook output-directory feature can be handled separately.

## Repair Work Prompt

Fix the docs for https://github.com/openclaw/openclaw/issues/54524. Current main allows `memorySearch.experimental.sessionMemory: true` with `sources: ["memory", "sessions"]`; the bundled `session-memory` hook writes session-derived Markdown to `<workspace>/memory/`; `listMemoryFiles` scans that directory; session transcripts are indexed separately as source `sessions`; and search dedupe is keyed to chunk/source/path rather than shared session provenance. Add concise public guidance in `docs/reference/memory-config.md`, `docs/automation/hooks.md`, and the bundled hook metadata/docs if appropriate, warning that the two mechanisms overlap and recommending either `sources: ["memory"]` when relying on the hook, or transcript indexing without hook-generated memory summaries. Coordinate wording with the related open output-directory PR, but do not add a new runtime dedupe feature, config option, localized docs, or broad memory architecture changes in this repair. No changelog is expected for docs-only guidance.
