## Summary

Keep open. Current main still matches the reported bug path: cron isolated-agent fallback does not pass the embedded-runner result classifier, and fatal error payload delivery can still flow into persisted delivered state.

## What This Changes

The issue requests cron isolated-agent handling that treats incomplete-turn fallback output as a failed fallback or error notification rather than successful delivered cron content.

## Best Possible Solution

Cron should reuse the embedded-runner fallback classifier when invoking model fallback, continue to the next eligible candidate or fail with failure-notification semantics when all candidates return incomplete terminal output, and persist failed/error notification delivery separately from successful cron output delivery.

## Reproduction Assessment

Yes. A deterministic test can mock an isolated embedded run that returns only the incomplete-turn warning as an error payload, let direct announce delivery succeed, and assert that current main can produce or persist status "error" with delivered state.

## Solution Assessment

Yes. The narrow maintainable fix is to wire the existing classifier into the cron fallback call and normalize fatal error-output delivery persistence; a regex-only lastDelivered patch would leave fallback acceptance and monitoring semantics split across layers.

## Repair Work Prompt

Fix the cron isolated-agent incomplete-turn fallback/delivery-state bug tracked by https://github.com/openclaw/openclaw/issues/72985. In `src/cron/isolated-agent/run-executor.ts`, pass a `classifyResult` callback to `runWithModelFallback` for embedded Pi runs using `classifyEmbeddedPiRunResultForModelFallback`; add the smallest runtime export/import seam needed and preserve CLI-provider behavior. Then normalize fatal cron error payload delivery so a run with terminal status `error` does not persist successful cron-output delivery (`lastDelivered: true`) merely because the warning/error text was sent. Prefer existing failure-notification and delivery-state concepts over a warning-text-only regex patch; only add a new persisted field or docs if the implementation truly needs it. Add focused regression tests for fallback continuing/rejecting incomplete terminal embedded results and for error runs not persisting delivered=true as useful cron output. Add a user-facing CHANGELOG.md fix entry with reporter credit. Do not change unrelated fallback policy, provider defaults, dependency pins, or broad cron scheduling behavior.
