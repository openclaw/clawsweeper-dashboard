## Summary

Keep open. Current main still has the reported cron isolated-agent gap: cron does not pass the embedded-runner fallback classifier into model fallback, and fatal error payload delivery can still persist as delivered content when the outbound send succeeds.

## What This Changes

The issue asks cron isolated-agent fallback to avoid silently delivering incomplete-turn warning text and to stop recording failed cron output as successfully delivered content.

## Best Possible Solution

Cron isolated runs should reuse the existing embedded-runner result classifier and persist fatal-output notification separately from successful cron content delivery.

## Reproduction Assessment

Yes. A deterministic source-level repro can mock a cron isolated embedded run that returns only the incomplete-turn warning as an error payload, let outbound delivery succeed, and assert that current main can persist status error with delivered state.

## Solution Assessment

Yes. The narrow maintainable fix is to wire the existing classifier into the cron fallback call and normalize fatal-output delivery semantics; a warning-text regex only in cron delivery would leave fallback behavior split across layers.
