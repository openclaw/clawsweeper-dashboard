## Summary

Keep open. Current main still has the reported isolated-cron gap: the cron fallback call does not use the existing embedded-runner incomplete-result classifier, and the cron result path can still combine an error terminal status with a delivered flag that later persists into job state.

## What This Changes

The issue asks cron isolated-agent runs to reject incomplete-turn fallback output as failed fallback attempts and to avoid recording failed error-output delivery as successful cron content delivery.

## Best Possible Solution

Cron should reuse the embedded-runner fallback classifier when invoking `runWithModelFallback`, continue to the next eligible fallback for incomplete terminal output, and persist failed output separately from successful cron content delivery.

## Reproduction Assessment

Yes. A deterministic test can mock an isolated embedded run that returns only the incomplete-turn warning as an error payload, let announce delivery succeed, and assert that current main can still return or persist `status: "error"` with delivered state.

## Solution Assessment

Yes. The narrow maintainable fix is to wire the existing classifier into cron fallback and normalize failed-output delivery persistence; a regex-only `lastDelivered` patch would leave fallback acceptance and monitoring semantics split across layers.
