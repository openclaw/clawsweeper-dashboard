## Summary

Keep open. Current main still has the reported cron isolated-agent path: model fallback is invoked without the embedded-runner result classifier, fatal incomplete-turn text can be converted into deliverable cron payloads, and finalization can carry `status: "error"` together with a successful channel-send `delivered` value into persisted job state.

## What This Changes

The issue requests cron isolated-agent fallback runs to treat incomplete-turn fallback text as failed cron content instead of delivered output, and to make run status and delivery status unambiguous.

## Best Possible Solution

Wire the embedded-runner fallback classifier into cron isolated model fallback, then record fatal fallback text as failed cron content while preserving separate channel-send diagnostics.

## Reproduction Assessment

Yes, by source inspection. A focused regression test can make `runWithModelFallback` return an embedded `isError` incomplete-turn warning, let direct delivery succeed, and observe `status: "error"` with delivered state on current main.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing embedded-runner result classifier at the cron fallback boundary and normalize fatal-output delivery semantics; a late warning-text regex in persistence would be brittle.
