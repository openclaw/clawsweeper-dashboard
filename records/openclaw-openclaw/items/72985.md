## Summary

Keep open. Current main still routes cron isolated runs through model fallback without the embedded-runner result classifier, and delivery persistence still records successful channel delivery independently from a terminal cron error, so the reported incomplete-turn warning and `status=error` plus `lastDelivered=true` state remains reachable.

## What This Changes

The issue asks cron isolated-agent runs to treat incomplete-turn fallback text as failed output, improve fallback/alert behavior, and avoid persisting error runs as successfully delivered cron content.

## Best Possible Solution

Wire the existing embedded-runner fallback classifier into cron isolated model fallback, then persist fatal-output notification separately from successful cron content delivery.

## Reproduction Assessment

Yes. A focused test can mock a cron isolated embedded result containing only the incomplete-turn warning as an error payload, let direct announce delivery succeed, and assert current main returns/persists `status: "error"` with delivered state.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing classifier at the cron fallback boundary and normalize fatal delivery semantics; a service-layer warning-text regex alone would leave fallback behavior split across layers.
