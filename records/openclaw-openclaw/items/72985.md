## Summary

Keep open. Current main still has the reported isolated-cron gap: the cron fallback path does not pass the existing embedded-runner result classifier into model fallback, and fatal error payloads can still be delivered and persisted as successful delivery while the run status is error.

## What This Changes

The issue asks cron isolated-agent fallback failures to stop delivering incomplete-turn warning text as normal cron output and to stop persisting error rows as delivered content.

## Best Possible Solution

Cron isolated runs should reuse the existing embedded-runner fallback classifier and persist failure-alert/error delivery separately from successful cron content delivery.

## Reproduction Assessment

Yes. A deterministic test can mock an isolated embedded run that returns only the incomplete-turn warning as an error payload, allow outbound announce delivery to succeed, and assert that current main can return status error with delivered true and persist that state.

## Solution Assessment

Yes. The narrow maintainable solution is to wire the existing classifier into the cron model-fallback call and normalize fatal-output delivery semantics; a final regex only at jobs-state persistence would leave fallback behavior split across layers.
