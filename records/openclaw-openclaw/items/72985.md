## Summary

Keep open. Current main still has the reported source path: cron isolated fallback accepts embedded incomplete-turn results without the existing embedded-runner classifier, then fatal-output delivery can be persisted independently as delivered while the run status is error.

## What This Changes

This issue asks cron isolated-agent fallback runs to treat incomplete-turn fallback output as a failed model result and avoid recording failed cron content as delivered output.

## Best Possible Solution

Reuse the embedded-runner fallback classifier at the cron isolated model-fallback boundary and make terminal fatal incomplete-turn output record failed cron content delivery, with focused regression coverage.

## Reproduction Assessment

Yes, from source inspection. A focused mock can make runWithModelFallback return an embedded isError payload containing the incomplete-turn warning, let direct delivery succeed, and observe an error run with delivered state on current main.

## Solution Assessment

Yes. The narrow maintainable fix is to wire the existing embedded-runner classifier into cron fallback and normalize fatal-output delivery semantics; a delivery-layer text regex alone would miss empty and reasoning-only fallback failures.
