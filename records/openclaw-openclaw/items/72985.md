## Summary

Keep open. Current main at 6a54aac still has the cron isolated fallback call without result classification, and cron state persistence derives delivered status from channel-send success independently of run error status, so the reported production state remains reachable.

## What This Changes

The issue asks cron isolated-agent runs to treat incomplete-turn fallback text as failed model output and separate failed-run notifications from successful cron content delivery.

## Best Possible Solution

Cron isolated runs should reuse the embedded-runner fallback classifier and persist error-notification delivery separately from successful cron content delivery.

## Reproduction Assessment

Yes. A deterministic unit path can mock a cron isolated embedded result containing only the incomplete-turn warning as an error payload, let announce delivery succeed, and assert current main can still produce status error with delivered state.

## Solution Assessment

Yes. The narrow maintainable fix is to wire the existing embedded-result classifier into cron model fallback and normalize fatal-output delivery semantics; a last-mile warning-text regex alone would leave fallback behavior split across layers.
