## Summary

Keep open. Current main still matches the reported cron isolated-agent failure mode: the cron fallback caller does not pass the embedded-runner result classifier, and the isolated delivery/state path can still represent an error run with delivered output state. The earlier ClawSweeper comment remains accurate on the central code paths.

## What This Changes

The issue asks cron isolated-agent fallback runs to treat incomplete-turn warning output as a fallback/failure condition and avoid persisting `lastRunStatus: "error"` together with `lastDelivered: true`.

## Best Possible Solution

Cron isolated runs should pass the embedded-runner fallback classifier into `runWithModelFallback`, continue to eligible fallback candidates or fail through failure-notification semantics when all candidates return incomplete terminal output, and persist useful cron output delivery separately from any delivered error notification.

## Reproduction Assessment

Yes. Existing test seams can mock an isolated embedded run that returns only the incomplete-turn warning as an error payload, request announce delivery, and assert current main can propagate an error status with delivered state; I inspected the source paths rather than running tests because this review is read-only.

## Solution Assessment

Yes. The narrow maintainable fix is to wire the existing classifier into the cron fallback call and normalize failed-output delivery persistence; a regex-only `lastDelivered` patch would leave fallback acceptance and monitoring semantics split across layers.
