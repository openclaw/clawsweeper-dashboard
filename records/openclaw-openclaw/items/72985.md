## Summary

Keep open. Current main still has the cron isolated model fallback path accepting embedded incomplete-turn error payloads without the existing result classifier, then delivery and persistence can still record a fatal cron run independently from a successful channel send.

## What This Changes

The issue reports cron isolated-agent provider fallback failures being delivered as normal cron output and persisted as `lastRunStatus: "error"` with `lastDelivered: true`.

## Best Possible Solution

Wire the embedded-runner result classifier into cron isolated model fallback, then treat fatal fallback text as failed cron content delivery while preserving separate channel-send diagnostics for operators.

## Reproduction Assessment

Yes. A focused source-level test can make `runWithModelFallback` return an embedded `isError` payload with the incomplete-turn warning, let direct delivery succeed, and observe `status: "error"` with delivered state on current main.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing embedded-runner fallback classifier at the cron fallback boundary and normalize fatal-output delivery semantics; a timer-layer warning regex would be too late and too brittle.
