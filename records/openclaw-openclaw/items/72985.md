## Summary

Keep open. Current main still has the reported cron isolated-agent gap: cron invokes model fallback without the embedded-runner result classifier, and fatal error payload delivery can still flow into persisted delivered state.

## What This Changes

The issue requests cron isolated-agent fallback handling that retries or escalates incomplete-turn fallback output and avoids recording failed runs as delivered content.

## Best Possible Solution

Cron isolated runs should reuse the existing embedded-runner fallback classifier and persist fatal-output notification separately from successful cron content delivery.

## Reproduction Assessment

Yes. A deterministic test can mock an isolated embedded run that returns only the incomplete-turn warning as an error payload, let outbound announce delivery succeed, and assert that current main can persist status error with delivered state.

## Solution Assessment

Yes. The narrow maintainable fix is to wire the existing classifier into the cron fallback call and normalize fatal-output delivery semantics; adding only a last-minute warning-text regex would leave fallback behavior split across layers.
