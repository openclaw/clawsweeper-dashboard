## Summary

Keep open. Current main still accepts embedded incomplete-turn error payloads as a terminal cron fallback result, can send that fatal text through direct cron delivery, and persists `lastDelivered=true` independently from `lastRunStatus=error`.

## What This Changes

The issue requests that isolated cron model fallback not deliver incomplete-turn error text as normal output and not persist failed cron runs as delivered.

## Best Possible Solution

Reuse the existing embedded-runner fallback classifier at the cron fallback boundary and make fatal incomplete-turn output fail cron content delivery semantics, with focused regression coverage and a changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence mock path: make cron `runWithModelFallback` return an embedded `isError` incomplete-turn payload, let direct delivery succeed, and current main can return `status: "error"` with delivered state preserved.

## Solution Assessment

Yes. The narrow maintainable fix is to wire the existing embedded-runner classifier into isolated cron fallback and normalize fatal-output delivery semantics; a warning-text-only regex in persistence would miss empty and reasoning-only classified failures.
