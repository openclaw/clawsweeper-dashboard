## Summary

Keep open. Current main still accepts cron isolated embedded-runner incomplete-turn results without the existing model-fallback result classifier, and the cron service still persists channel delivery independently from terminal run failure, so the reported `status: "error"` plus `lastDelivered: true` state remains reachable.

## What This Changes

Fix cron isolated-agent fallback and delivery semantics so incomplete-turn warning text is not treated as successful cron content delivery and fallback failures remain observable.

## Best Possible Solution

Wire the embedded-runner result classifier into cron isolated model fallback, then record fatal-output notification separately from successful cron content delivery in run results, persisted state, run logs, and monitoring-facing docs if the field contract changes.

## Reproduction Assessment

Yes. A focused test can mock `runWithModelFallback` returning only the incomplete-turn warning as an error payload, let direct announce delivery succeed, and assert current main returns and persists `status: "error"` with delivered state.

## Solution Assessment

Yes for the proposed repair direction. Reusing the existing embedded-runner fallback classifier at the cron fallback boundary plus normalizing fatal delivery semantics is narrower and more maintainable than matching warning text only in the persistence layer.
