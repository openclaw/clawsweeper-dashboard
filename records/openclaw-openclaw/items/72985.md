## Summary

Keep open. Current main still leaves cron isolated model fallback without the embedded-runner result classifier, and fatal embedded error payloads can still be delivered and persisted as channel delivery independently from the terminal cron error.

## What This Changes

Cron isolated-agent runs should not silently deliver the embedded incomplete-turn warning as normal fallback output or persist that failed run as successful content delivery.

## Best Possible Solution

Wire the existing embedded-runner result classifier into cron isolated model fallback, then ensure fatal incomplete-turn output is recorded as failed cron content delivery rather than a normal delivered cron result.

## Reproduction Assessment

Yes. Source inspection shows a focused mock can make runWithModelFallback return an embedded isError payload containing the incomplete-turn warning, allow direct delivery to succeed, and observe status error with delivered state on current main.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing embedded-runner classifier at the cron fallback boundary and normalize fatal-output delivery semantics; a delivery-layer warning-text regex alone would miss empty and reasoning-only fallback failures.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/72985. Cron isolated-agent model fallback must treat embedded incomplete-turn warning/empty terminal results as failed fallback candidates by wiring `classifyEmbeddedPiRunResultForModelFallback` into the `runWithModelFallback` call in `src/cron/isolated-agent/run-executor.ts`. Then ensure the final all-fallback fatal incomplete-turn path does not persist successful cron content delivery: avoid delivering the fatal fallback text as normal cron output, or mark that terminal result as not delivered while preserving useful delivery diagnostics. Add regression coverage for a primary provider failing, a fallback provider returning an embedded `isError` payload with “Agent couldn't generate a response”, fallback classification continuing to the next candidate, and the all-candidates-failed case persisting `lastRunStatus: "error"` without `lastDeliveryStatus: "delivered"` for that fatal output. Keep changes scoped to cron isolated fallback/delivery persistence; do not change generic channel adapters or broad cron delivery semantics. Add a single-line CHANGELOG.md fix entry.
