## Summary

Keep open: current main still has the cron isolated-agent source path where an embedded incomplete-turn fallback can bypass model-fallback result classification, be converted into sendable fatal output, and persist as `status: "error"` with delivered content state.

## What This Changes

The issue asks cron isolated-agent fallback failures to stop delivering incomplete-turn warnings as normal cron output and to avoid recording failed cron content as delivered.

## Best Possible Solution

Reuse the embedded-runner fallback classifier at the cron fallback boundary, then record fatal fallback text as failed cron content while keeping channel-send and failure-alert diagnostics separate.

## Reproduction Assessment

Yes, by source inspection. A focused regression can mock `runWithModelFallback` returning an embedded `isError` incomplete-turn warning, let direct delivery succeed, and observe the current path producing an error status with delivered state.

## Solution Assessment

Yes. The narrow maintainable fix is to wire the existing embedded-runner result classifier into cron model fallback and normalize fatal-output delivery semantics before persistence; a late warning-text regex in `timer.ts` would be brittle.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/72985 on current main. The narrow bug: isolated cron uses `runWithModelFallback` without the embedded-runner result classifier, so an incomplete-turn fallback result such as `Agent couldn't generate a response` can be treated as terminal output, sent through cron delivery, and persisted as `status: "error"` plus delivered content state. Wire `classifyEmbeddedPiRunResultForModelFallback` into the cron isolated-agent fallback path, and ensure fatal incomplete-turn/error payloads are not recorded as successful cron content delivery while preserving separate channel-send diagnostics and existing failure-alert behavior. Do not add a new config option, do not implement a broad retry policy unless required by the narrow fix, and do not mask real tool/provider failures with late `timer.ts` string matching. Add focused regression coverage for classified incomplete-turn fallback, persisted delivered state for fatal cron content, and existing nonfatal presentation-warning behavior. Add a single-line changelog entry because this is user-visible cron behavior.
