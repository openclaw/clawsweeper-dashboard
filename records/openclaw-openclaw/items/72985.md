## Summary

Keep open. Current main still matches the reported cron isolated-agent gap: cron fallback does not pass the embedded-runner result classifier into model fallback, and fatal cron output can still carry successful outbound delivery state into persisted job state.

## What This Changes

The issue reports that cron isolated-agent fallback can deliver an incomplete-turn warning as normal cron output and persist `lastRunStatus: "error"` together with `lastDelivered: true`.

## Best Possible Solution

Cron isolated runs should reuse the existing embedded-runner fallback classifier when invoking model fallback, continue past incomplete terminal outputs to the next eligible fallback candidate, and persist fatal-output delivery separately from successful cron content delivery.

## Reproduction Assessment

Yes. A deterministic test can mock an isolated embedded run returning only the incomplete-turn error payload, let outbound delivery succeed, and assert that current main can still produce `status: "error"` with delivered state.

## Solution Assessment

Yes. The narrow maintainable fix is to wire the existing classifier into cron fallback and normalize fatal-output delivery persistence; a late regex-only `lastDelivered` patch would leave fallback acceptance and monitoring semantics split across layers.
