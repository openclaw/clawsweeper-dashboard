## Summary

Keep open. Current main and the latest shipped tag still accept the reported Tailscale Serve plus explicit no-auth configuration, and the linked fixing PR at https://github.com/openclaw/openclaw/pull/50631 remains open, conflicting, and unmerged.

## What This Changes

The issue requests hardening Gateway auth so `gateway.auth.mode="none"` cannot expose the Gateway through Tailscale Serve without credentials.

## Best Possible Solution

Land or replace the linked fix with shared validation that rejects explicit no-auth plus Tailscale Serve before runtime and install/onboarding plans, while preserving trusted loopback-only no-auth and updating focused tests/docs.

## Reproduction Assessment

Yes, at source level. Current main accepts `gateway.auth.mode="none"` with `gateway.tailscale.mode="serve"`, has a test asserting that shape succeeds, and then authorizes mode none before Tailscale identity or shared-secret checks run.

Yes, the fail-closed direction is the narrow maintainable fix. The current linked PR moves in that direction, but maintainers should ensure shared preflight coverage and real Tailscale proof before landing.
