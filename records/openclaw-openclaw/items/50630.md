The issue reports that explicit no-auth Gateway mode combined with Tailscale Serve can expose Gateway access to Tailnet peers without credentials.

## Best Possible Solution

Review, rebase, prove, and land or replace https://github.com/openclaw/openclaw/pull/50631 with shared validation that rejects explicit no-auth plus Tailscale Serve while preserving trusted loopback-only no-auth use.

Yes. Fail-closed validation for explicit no-auth plus Tailscale Serve is the narrow maintainable direction, with matching tests and upgrade proof so operators do not discover the rejection only at runtime.
