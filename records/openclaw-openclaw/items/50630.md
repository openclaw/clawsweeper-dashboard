## Best Possible Solution

Review, rebase, prove, and land or replace https://github.com/openclaw/openclaw/pull/50631 with shared startup validation that rejects explicit no-auth plus Tailscale Serve while preserving trusted loopback-only no-auth and updating focused tests/docs.

## Solution Assessment

Yes, the fail-closed startup-validation direction is the narrow maintainable fix. The linked PR moves in that direction, but it needs rebase, upgrade messaging, and real Tailscale behavior proof before landing.
