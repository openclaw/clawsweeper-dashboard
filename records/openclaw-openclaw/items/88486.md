## Summary

Keep open for normal maintainer review: current main still omits the interactive diagnostic session UUID, and the PR is a narrow source-backed fix with no blocking review findings. The remaining merge concern is accepting the additive plugin/exporter-visible session metadata change.

## What This Changes

The PR hoists the interactive session-store lookup, passes a matching session UUID into diagnostic lifecycle events, and adds regression tests for normal and native command-target dispatch.

## Best Possible Solution

## Solution Assessment

Yes. Carrying the UUID only when the store entry session key matches the lifecycle session key is the narrow maintainable fix and avoids native command-target mis-association.
