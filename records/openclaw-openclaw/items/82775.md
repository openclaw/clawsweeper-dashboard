## Summary

Keep open: current main still has the reported Codex bridge liveness gap for silent internal shutdown after post-tool raw assistant progress. The earlier raw tool-output watchdog fix is present, but this issue is distinct and needs a Codex bridge contract decision rather than cleanup closure.

## What This Changes

The issue reports that a Codex internal shutdown can abort an in-flight turn without sending `turn/completed`, `error`, or another terminal notification to the OpenClaw Codex bridge.

## Best Possible Solution

Have Codex emit a terminal shutdown/abort notification for this internal abort, or add a protocol-grounded OpenClaw liveness fallback with focused regression coverage for silent post-tool shutdowns.

## Reproduction Assessment

Yes, at source level: simulate a dynamic tool response followed by post-tool raw assistant `rawResponseItem/completed` and no terminal event, and current main falls back to the long terminal watchdog rather than the short completion watchdog. No live Codex internal shutdown was reproduced in this pass.

## Solution Assessment

Unclear: the cleanest fix is likely a Codex-side terminal shutdown event, while an OpenClaw liveness probe is a plausible fallback that needs owner judgment on timeout semantics.
