## Summary

Keep open: current main still leaves the reported silent Codex internal-shutdown path to the long terminal or supervisor fallback after post-tool raw assistant progress. The issue is source-reproducible, but the fix shape needs owner judgment between an upstream Codex terminal event and an OpenClaw liveness fallback.

## What This Changes

The issue asks for Codex internal shutdowns during a turn to propagate a terminal event or otherwise let the OpenClaw Codex bridge recover promptly instead of waiting for stuck-session aborts.

## Best Possible Solution

Have Codex emit a terminal shutdown/abort notification for this internal abort, or add a protocol-grounded OpenClaw liveness fallback with focused regression coverage for silent post-tool shutdowns.

## Reproduction Assessment

Yes, at source level rather than with a live Codex process. The current bridge can move from a post-tool raw assistant notification into the long terminal fallback while a silent Codex shutdown emits no terminal event, matching the reported stuck-session shape.

## Solution Assessment

Unclear for an automated fix. The cleanest solution appears to be a Codex-side terminal shutdown event, while an OpenClaw fallback needs maintainer judgment on timeout semantics and false-positive risk.
