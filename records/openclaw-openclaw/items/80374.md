## What This Changes

Resumed Claude CLI sessions should reload the same SOUL.md, IDENTITY.md, USER.md, and harness bootstrap context that fresh sessions receive.

## Best Possible Solution

Update or replace the linked PR so both one-shot and `claude-stdio` resumed Claude CLI turns write and append the generated harness prompt file, add focused regression coverage, and then merge it to close this issue.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a resumed Claude CLI run sets `useResume`, the default prompt policy resolves to no prompt for an existing session, and the runner suppresses prompt-file creation and prompt flags.

## Solution Assessment

Yes, the requested direction is the right fix boundary, but the implementation must cover the actual runner resume paths. A backend default-only change is not sufficient while `executePreparedCliRun`, `buildCliArgs`, and the live-session argv builder still drop the prompt file on resume.
