## Repair Work Prompt

Fix the Claude CLI resume regression reported in https://github.com/openclaw/openclaw/issues/80374. Current main builds fresh Claude CLI sessions with `--append-system-prompt-file <generated system-prompt.md>`, but resumed sessions set `useResume` and drop the generated prompt file, so workspace bootstrap files such as `SOUL.md`, `IDENTITY.md`, and `USER.md` are not reloaded after a gateway restart or idle Claude process exit. Keep the change scoped to the Claude CLI backend/runner path: make resumed Claude CLI invocations preserve the generated OpenClaw harness prompt payload when the installed Claude CLI supports the flag combination, and update tests that currently assert prompt-file suppression on resume. Do not add a new config option, do not move provider-owned behavior into unrelated core policy, and do not change other CLI backends except where the existing `systemPromptWhen` contract is already meant to apply. Add a user-facing changelog fix entry if repo policy requires it.
