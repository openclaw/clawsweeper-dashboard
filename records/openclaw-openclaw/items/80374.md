## What This Changes

The issue reports that resumed Claude CLI sessions omit harness initialization files such as SOUL.md, IDENTITY.md, and USER.md, causing identity drift after resume.

## Best Possible Solution

Update and land https://github.com/openclaw/openclaw/pull/81199, or replace it, so resumed one-shot and claude-stdio live paths write and pass the generated harness prompt file with regression coverage and real resume proof.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction path: Claude CLI resume selects resumeArgs, prompt resolution treats the session as not new, prompt-file creation is skipped, and both buildCliArgs and the live-session path suppress the prompt flags.

## Solution Assessment

No current-main solution is present. The best path is to review and land an updated https://github.com/openclaw/openclaw/pull/81199, or a replacement, that covers both generic resume argv construction and the Claude live-session resume path.
