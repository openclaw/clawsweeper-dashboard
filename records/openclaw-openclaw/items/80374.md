## Summary

Keep this open: current main and the latest shipped release still have the Claude CLI resume path that skips harness system-prompt file injection, and the open closing PR has not merged.

## What This Changes

The issue reports that Claude CLI resumed sessions omit the harness system prompt file, so SOUL.md, IDENTITY.md, USER.md, and related initialization files are not reloaded after resume.

## Best Possible Solution

Update the linked fixing PR so both one-shot and `claude-stdio` resume paths create and append the generated harness prompt, add regression coverage for resumed argv, then merge it so this issue closes normally.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction path: a resumed Claude CLI run sets `useResume`, skips prompt-file creation, and suppresses system-prompt flags, matching the reporter's fresh-versus-resumed argv evidence.

## Solution Assessment

Yes for the issue direction, but the open PR as reviewed is not sufficient yet. The maintainable fix needs to cover the Claude CLI runner and live-session resume paths, not only the Anthropic backend default.
