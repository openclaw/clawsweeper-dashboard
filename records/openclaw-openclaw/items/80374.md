## Summary

Keep this open: current main still has the Claude CLI resume path that drops the generated harness system prompt, and the open closing PR has not merged yet.

## What This Changes

Resumed Claude CLI sessions should keep loading harness initialization files such as SOUL.md, IDENTITY.md, and USER.md instead of resuming as generic Claude.

## Best Possible Solution

Patch the Claude CLI runner and live-session resume path so resumed turns create and pass the generated system-prompt file, then merge the linked fix PR or a narrow replacement with regression and live argv proof.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: resumed Claude CLI runs set `useResume`, skip prompt-file creation, and suppress system-prompt flags while docs say bootstrap files are injected every turn.

## Solution Assessment

Yes for the issue direction, but the open PR is not sufficient as reviewed. The narrow maintainable fix belongs in the Claude CLI resume runner and live-session argv path, not only in a backend default change.
