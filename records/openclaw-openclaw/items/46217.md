## Summary

Keep open. Current main has broad Control UI zh-CN coverage, but the schema-driven Settings/Config surface still renders English section metadata, field labels/help, and several hard-coded config-view strings; the closed broader zh-CN issues do not cover this remaining settings-schema path.

## What This Changes

The issue asks for the Chinese Settings section UI and configuration items to be translated into Chinese.

## Best Possible Solution

Add a maintained i18n path for Settings/Config chrome plus schema/uiHint labels and help text, regenerate locale bundles through the existing pipeline, and cover the zh-CN Settings path with focused UI/i18n tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: set the Control UI locale to zh-CN and open Settings/Config/Advanced Settings; the config form renders English SECTION_META, hard-coded config-view copy, and English config.schema/uiHints labels/help.

## Solution Assessment

No, a one-off zh-CN locale edit is not the best fix. The maintainable solution is to localize the Settings/schema metadata source of truth and regenerate locales, while preserving plugin/schema contracts.
