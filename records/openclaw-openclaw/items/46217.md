## Summary

Keep open. Current main and the latest release still leave major Control UI Settings text and schema-derived field labels in English for Chinese locales, while the related open PRs are partial, conflicting, or not merge-ready.

## What This Changes

Request to translate the Control UI Settings/configuration section into Chinese for Chinese-language users.

## Best Possible Solution

Keep this as the canonical Settings localization tracker, land deliberate i18n slices through the existing UI pipeline, and define a maintainer-approved schema/uiHint localization contract for field labels and help text.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: choose zh-CN or zh-TW in Control UI and open Settings; section labels, toolbar/search strings, and schema-derived field labels still come from English literals or locale-unaware metadata.

## Solution Assessment

Yes for keeping this open as the canonical tracker, but not as an automatic one-shot fix. The maintainable implementation should use the existing Control UI i18n pipeline plus a maintainer-approved schema/uiHint localization contract.
