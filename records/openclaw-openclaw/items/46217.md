## Summary

Keep open. Current main and the latest release still leave major Control UI Settings surfaces in English for zh-CN/zh-TW users, and the related open PRs are partial or still awaiting maintainer review rather than merged fixes.

## What This Changes

Request to translate the Control UI Settings/configuration section into Chinese so Chinese users do not see English Settings labels.

## Best Possible Solution

Keep this as the canonical Settings localization tracker, land or close the open PR slices deliberately, and define a maintainable locale-aware path for Settings chrome plus schema/uiHint labels and help text.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: select zh-CN or zh-TW in the Control UI and open Settings; category labels, toolbar/search strings, and schema-derived field labels still come from English literals or locale-unaware metadata.

## Solution Assessment

Yes for keeping a canonical tracker, but not as a broad automatic one-shot fix. The maintainable solution should use the existing Control UI i18n pipeline and a maintainer-approved schema/uiHint localization contract.
