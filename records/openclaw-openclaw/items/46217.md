## Summary

Keep open: current main and the latest release still contain English Settings chrome, category labels, and schema-derived field labels, while the related open PR only covers a toolbar slice and explicitly leaves the schema/category localization decision out of scope.

## What This Changes

The issue requests fuller Chinese localization for the Control UI Settings section, especially Settings/configuration labels that remain in English.

## Best Possible Solution

Keep this as the canonical Settings localization tracker, land focused i18n slices through the generated UI pipeline, and decide the locale-aware schema/uiHint field-label contract before implementing the remaining field-label work.

## Reproduction Assessment

Yes. Source inspection on current main shows Settings chrome, category labels, and schema-derived field labels still render from English literals, schema titles, uiHints, or `humanize` without a locale lookup; no live browser run was performed in this read-only review.

## Solution Assessment

No single broad translation patch is currently the best fix. The maintainable path is focused i18n key extraction through the generated locale pipeline, plus an explicit maintainer decision for schema/uiHint field-label localization before field labels are translated.
