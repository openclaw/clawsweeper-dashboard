## Summary

Keep open. Current main has zh-CN Control UI support and translated top-level Settings labels, but the Settings and Quick Settings surfaces still contain hardcoded English strings and English schema metadata, so the requested Chinese Settings localization remains incomplete.

## What This Changes

The issue requests fuller Chinese localization for the Control UI Settings section.

## Best Possible Solution

Define a scoped Settings i18n pass that moves visible Settings copy into the English locale source, regenerates locale bundles, and decides how schema/plugin field metadata should be localized.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: set the Control UI locale to zh-CN and open Settings or Advanced Settings; current main still renders many English literals from the Settings view files plus English form metadata.

## Solution Assessment

Unclear. The existing Control UI i18n pipeline is the right mechanism for visible strings, but full Settings localization needs maintainer scope on schema-derived and plugin-provided metadata before this should become an automated fix.
