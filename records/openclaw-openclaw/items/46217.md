## Summary

Keep open: current main has Chinese Control UI support, but the Settings/config form still renders core section metadata and schema-derived configuration labels/help from English constants and runtime hints, so this narrower localization request remains valid.

## What This Changes

The issue requests fuller Simplified Chinese localization for configuration items in the Control UI Settings section.

## Best Possible Solution

Define a maintainable localization strategy for Settings section metadata and config schema/UI-hint text, then route the chosen strings through the Control UI i18n pipeline and regenerate zh-CN artifacts.

## Reproduction Assessment

Yes from source inspection. With zh-CN supported, current main still renders Settings section metadata and config field labels/help from English `SECTION_META`, schema labels, and `ConfigUiHints` rather than locale keys; I did not run a live browser session.

## Solution Assessment

No for a narrow automatic fix. Static section labels can be localized directly, but complete Settings item localization should first decide how runtime schema and plugin/channel UI hints become localizable without hand-maintaining generated zh-CN output.
