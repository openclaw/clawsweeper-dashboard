## Summary

Keep open: current main and the latest release still render core Settings labels and some Settings chrome in English for Chinese locales, and the only related PR is an open, conflicting partial toolbar slice that explicitly leaves schema labels out of scope.

## What This Changes

Request to translate the Chinese Control UI Settings/configuration section into Chinese.

## Best Possible Solution

Adopt one maintainer-approved Settings i18n contract: route toolbar and section chrome through UI translation keys, add an explicit locale-aware schema/uiHint label fallback, and regenerate zh-CN/zh-TW bundles through the existing pipeline.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: select zh-CN or zh-TW in Control UI, open Settings, and the form labels still come from English schema/uiHint metadata or English literals rather than localized keys.

## Solution Assessment

Unclear as a single implementation sweep. Translating obvious toolbar chrome is straightforward, but schema/uiHint labels need a maintainer-approved i18n contract and fallback behavior before a durable full fix.
