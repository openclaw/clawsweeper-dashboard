## Summary

Keep open: current main and the latest release still leave visible Settings text in English, including schema-derived field labels, section/category metadata, toolbar chrome, and quick-settings chrome. The related open PRs are useful but partial, so they do not make this issue obsolete.

## What This Changes

The issue requests fuller Chinese localization for the Control UI Settings section, especially configuration labels and Settings UI text that still appear in English.

## Best Possible Solution

Adopt one maintainer-approved Settings i18n contract: route toolbar, quick-settings, section chrome, and schema/uiHint labels through UI translation keys with English fallback, then regenerate zh-CN/zh-TW through the existing pipeline.

## Reproduction Assessment

Yes. Source inspection on current main shows visible Settings field labels, section/category labels, toolbar strings, and quick-settings chrome still come from English literals or schema metadata without locale lookup.

## Solution Assessment

Unclear for implementation shape. Existing `t()` keys fit toolbar and chrome, but schema/uiHint field-label localization needs a maintainer-approved key shape and fallback contract before a durable full fix.
