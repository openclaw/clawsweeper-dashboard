## Summary

Keep open. Current main supports zh-CN in the Control UI and translates top-level Settings navigation, but the Settings, Quick Settings, and schema-driven config form still contain substantial hardcoded English copy.

## What This Changes

Request Chinese localization of the Control UI Settings configuration surfaces that still appear in English under zh-CN.

## Best Possible Solution

Define a scoped Settings localization plan that moves fixed Settings copy into the English locale source, regenerates zh-CN, and decides how schema and plugin metadata should be localized.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: set the Control UI locale to zh-CN and open Quick Settings or Advanced Settings; the current render files still emit many English literals and schema-derived labels.

## Solution Assessment

Unclear. The existing i18n pipeline is the right mechanism for fixed UI copy, but full Settings localization needs maintainer scope for schema and plugin metadata before an automated repair is the best path.
