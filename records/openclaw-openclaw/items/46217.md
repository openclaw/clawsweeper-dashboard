## Summary

Keep open: current main and latest v2026.6.6 still render Settings form labels, section/category labels, and some toolbar chrome from English schema metadata or literals, while the only related open PR is a partial toolbar slice that explicitly leaves schema labels and section metadata out of scope.

## What This Changes

Requested fuller Chinese localization of the Control UI Settings section, especially English configuration labels shown under Chinese locales.

## Best Possible Solution

Adopt one maintainer-approved Settings i18n contract: route toolbar and section chrome through UI translation keys, add locale-aware schema/uiHint label lookup with English fallback, and regenerate zh-CN/zh-TW through the existing pipeline.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: choose zh-CN or zh-TW, open Settings, and form labels/section labels still come from English schema/uiHint metadata or hardcoded English literals instead of locale keys.

## Solution Assessment

Unclear as a single implementation sweep. Toolbar strings fit the existing t() pipeline, but schema/uiHint field labels need a maintainer-approved i18n lookup and fallback contract before a durable full fix.
