## Summary

Keep open. Current main still has zh-CN Control UI support, but the Settings and Advanced Settings surfaces continue to render many English literals and schema-derived English labels/descriptions, so the requested Chinese localization is not implemented or superseded.

## What This Changes

Translate the Chinese Control UI Settings section, including configuration item labels and supporting text, into Chinese.

## Best Possible Solution

Keep this open and scope a Settings i18n pass that moves visible Settings copy into the English locale source, regenerates locale outputs through the existing pipeline, and defines how config schema title/description metadata should be localized for core and plugin-provided fields.

## Reproduction Assessment

Yes. Set the Control UI locale to zh-CN and open Settings or Advanced Settings; source inspection on current main shows English literals and English schema-backed metadata still rendered in that surface.

## Solution Assessment

Unclear. The existing Control UI i18n pipeline is the right path for visible strings, but full configuration-item localization needs a maintainer decision on locale-aware schema metadata or a staged fallback policy.
