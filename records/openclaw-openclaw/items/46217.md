## Summary

Keep open: current main and v2026.6.8 still render Settings field labels, section/category metadata, and some Settings chrome from English schema metadata or literals, while the only open related PR is explicitly a partial toolbar slice.

## What This Changes

The issue requests fuller Chinese localization for the Control UI Settings section, especially configuration labels that remain English under zh-CN/zh-TW.

## Best Possible Solution

Adopt one maintainer-approved Settings i18n contract: route toolbar and section chrome through UI translation keys, add locale-aware schema/uiHint label lookup with English fallback, and regenerate zh-CN/zh-TW through the existing pipeline.

## Reproduction Assessment

Yes. Source inspection on current main shows Settings labels and sections still come from English uiHints/schema titles or hardcoded literals, so selecting zh-CN or zh-TW and opening Settings can still expose English copy.

## Solution Assessment

Unclear for implementation shape. Toolbar strings fit the existing t() pipeline, but schema/uiHint field labels need a maintainer-approved lookup and fallback contract before a durable full fix.
