## Summary

Keep open. Current main still has the Settings/Quick Settings and schema-driven config labels/help in English under zh-CN; the closed broader zh-CN issues covered generated locale keys, not this remaining Settings/schema path.

## What This Changes

The issue asks for Chinese translations for the Control UI Settings/configuration section.

## Best Possible Solution

Route Settings/Quick Settings/Advanced Settings copy through the existing i18n source, add a maintained localization strategy for schema/uiHints labels and help, regenerate locale bundles, and cover the zh-CN Settings path with focused UI/i18n tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: set the Control UI locale to zh-CN and open Settings/Advanced Settings; hard-coded Settings copy plus schema/uiHint labels and help still render English.

## Solution Assessment

No, a one-off zh-CN locale edit is not the best solution because foreign-language locale files are generated and the field labels come from schema/uiHints. The maintainable fix needs source-of-truth i18n coverage for Settings chrome and schema metadata.
