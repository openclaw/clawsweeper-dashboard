- **[P1] Preserve DM topic session keys for transcript context:** `extensions/telegram/src/bot-handlers.runtime.ts:1236-1242`
  - body: This new transcript-context lookup resolves direct sessions without the bot's topic capability, so `shouldUseTelegramDmThreadSession` falls back false.   - confidence: 0.9

  - body: The new recent reader returns every assistant row, but model replay drops OpenClaw transcript-only assistant messages such as delivery mirrors and gateway-injected rows. Feeding those bookkeeping rows into Telegram prompt context can duplicate or stale the assistant context instead of matching model-visible history.
  - confidence: 0.86

  - body: `session-store-runtime` is a public plugin SDK subpath, and this patch adds a new function and type without updating the SDK docs or making the contract explicit. That leaves third-party plugin authors with an undocumented transcript API surface.
  - confidence: 0.8
