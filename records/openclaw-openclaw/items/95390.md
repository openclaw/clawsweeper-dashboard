- **[P1] Preserve DM topic session keys for transcript context:** `extensions/telegram/src/bot-handlers.runtime.ts:1236-1242`
  - body: This new transcript lookup resolves the direct session without passing the bot's topic capability, so `shouldUseTelegramDmThreadSession` falls back false. Private chats with `message_thread_id` on topics-enabled bots can therefore read the base DM transcript instead of the topic-scoped transcript.
  - confidence: 0.9
- **[P2] Filter transcript-only assistant rows from recent context:** `src/config/sessions/transcript.ts:180-183`
  - body: The new reader returns every assistant row, but model replay and the existing latest-assistant reader skip OpenClaw `delivery-mirror` and `gateway-injected` rows. Feeding those bookkeeping rows into Telegram prompt context can stale or duplicate assistant context that the model would otherwise never replay.
  - confidence: 0.86
- **[P1] Document or remove the new public SDK export:** `src/plugin-sdk/session-store-runtime.ts:160-164`
  - body: `session-store-runtime` is a public plugin SDK subpath, and this patch adds a new function and type without matching plugin SDK docs or an explicit contract decision. Either keep the implementation behind an existing transcript SDK seam or update the public contract materials before merge.
  - confidence: 0.8
- **[P3] Use toReversed for the recent transcript order:** `src/config/sessions/transcript.ts:257`
  - body: CI lint fails on this new `recent.reverse()` call because the repo enforces `Array#toReversed()` for reversed copies. Returning `recent.toReversed()` should clear the PR-introduced lint failure without changing behavior.
  - confidence: 0.96
