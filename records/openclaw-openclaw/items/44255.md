## Summary

Keep this PR open, but it is not ready to merge: the intended security hardening still leaves plugin-handled channel sends and polls unredacted, and the diff has additional dynamic-redaction risks. The related broader guardrail proposal at https://github.com/openclaw/openclaw/pull/72733 is still open, so maintainers should choose the security boundary before repair or replacement.

## What This Changes

The PR adds dynamic secret discovery/redaction for outbound channel messages, CLI startup initialization, tests, and a new security-redaction document.

## Best Possible Solution

Define one reviewed sanitization boundary before channel plugin dispatch and fallback delivery, or replace this with a maintainer-approved guardrail design that covers model and channel egress consistently.

## Reproduction Assessment

Yes. A high-confidence source reproduction is a plugin-handled Telegram, Discord, Slack, or Mattermost send or poll containing a secret: the PR computes redacted fallback values, but plugin dispatch still receives unchanged action params.

## Solution Assessment

No. The current patch redacts only part of the delivery path; the maintainable fix is to sanitize cloned outbound action params once before plugin dispatch and reuse that sanitized payload for fallback delivery.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Pass redacted params to plugin send actions:** `src/infra/outbound/outbound-send-service.ts:108-110`
  - body: `redactedMessage` is computed, but `tryHandleWithPluginAction` still dispatches the original `ctx.params`. Plugin send handlers read message text from those params, so plugin-based channels can still send raw secrets.
  - confidence: 0.95
- **[P1] Route redacted poll params through plugins:** `src/infra/outbound/outbound-send-service.ts:184-186`
  - body: The redacted poll question/options are only used by the fallback path. Plugin poll handlers run first and read `pollQuestion`/`pollOption` from unchanged params, so secrets still leak when a plugin handles the poll.
  - confidence: 0.94
- **[P1] Emit dynamic masks with a function replacer:** `src/logging/redact.ts:112`
  - body: Passing `masked` as a string replacement lets `$&`, `$'`, and related sequences in preserved secret prefixes/suffixes be expanded by JavaScript. Use a function replacer so the mask is emitted literally.
  - confidence: 0.91
- **[P2] Tighten auth key discovery:** `src/logging/redact-init.ts:20`
  - body: The `/auth/i` key pattern also matches benign keys such as `author` and `authorization`, causing long non-secret values to be registered globally and redacted from outbound messages.
  - confidence: 0.88
