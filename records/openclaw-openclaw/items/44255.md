## Summary

Keep open: current main does not implement this outbound redaction, and the PR still has merge-blocking security defects in the plugin delivery path and dynamic redaction logic. The broader guardrail work in #72733 is open, not merged, and should inform whether this outbound-only approach is repaired or superseded.

## What This Changes

The PR branch adds dynamic sensitive-value registration, config/env redaction initialization at CLI preaction, outbound send/poll redaction, docs, and redaction tests.

## Best Possible Solution

Rework the design around one central sanitized action-param boundary before plugin dispatch and fallback delivery, preserving existing logging redaction semantics and coordinating with #72733 before merging.

## Reproduction Assessment

Yes. A high-confidence source reproduction is a plugin-handled Slack, Discord, Telegram, or Mattermost send/poll containing a known secret: the PR computes redacted fallback values, but plugin dispatch receives unchanged `ctx.params`, and bundled handlers read message and poll text from those raw params.

## Solution Assessment

No. The PR redacts fallback delivery but misses the dominant plugin delivery path; the narrower maintainable fix would sanitize cloned action params once before plugin dispatch and feed the same sanitized values into fallback delivery with channel regression coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Pass redacted params to plugin send actions:** `src/infra/outbound/outbound-send-service.ts:108-110`
  - body: `redactedMessage` is computed, but `tryHandleWithPluginAction` still dispatches the original `ctx.params`. Slack, Discord, Telegram, Mattermost, and other plugin handlers read `message` from those params, so plugin-based sends still deliver the raw secret.
  - confidence: 0.95
- **[P1] Route redacted poll params through plugins:** `src/infra/outbound/outbound-send-service.ts:184-186`
  - body: The redacted poll question/options are only used by the fallback path. Plugin poll handlers run first and read `pollQuestion`/`pollOption` or equivalent fields from unchanged action params, so secrets in poll text still leak whenever a plugin handles the poll.
  - confidence: 0.94
- **[P1] Use a function replacer for dynamic masks:** `src/logging/redact.ts:112`
  - body: Passing `masked` as a string replacement lets `$&`, `$'`, and similar sequences in the preserved prefix/suffix be interpreted by JavaScript. A discovered secret containing those characters can reinsert the full match; use a function replacer so the mask is emitted literally.
  - confidence: 0.91
- **[P2] Tighten auth key matching:** `src/logging/redact-init.ts:20`
  - body: The `/auth/i` key pattern also matches benign keys like `author`, `authenticate`, and `authorization`. Long non-secret values under those keys become globally redacted from outbound messages, which can silently corrupt legitimate content.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The patch is intended as hardening but still leaks secrets through plugin delivery and adds dynamic-redaction edge cases.

Concerns:

- **[high] Plugin delivery bypasses outbound redaction:** `src/infra/outbound/outbound-send-service.ts:108`
  - body: The PR computes redacted send/poll text but does not sanitize `ctx.params` before plugin dispatch; bundled plugin handlers read raw params and can send secrets to external services.
  - confidence: 0.95
- **[high] Dynamic replacement can reinsert matched secrets:** `src/logging/redact.ts:112`
  - body: Using a string replacement for a mask that may contain `$` replacement metacharacters can expand back to the full matched secret instead of the intended mask.
  - confidence: 0.91
- **[medium] Broad secret discovery can redact benign data:** `src/logging/redact-init.ts:20`
  - body: The `/auth/i` scan treats author-like metadata and other non-secret auth substrings as sensitive, adding their values to a global redaction set and altering user-visible outbound content.
  - confidence: 0.86
