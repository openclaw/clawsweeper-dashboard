## Summary

Keep open: current main has not implemented outbound message redaction, and this PR still misses the primary plugin delivery paths while adding security-sensitive redaction edge cases.

## What This Changes

The PR adds dynamic secret registration, config/env secret scanning, CLI startup initialization, outbound send/poll redaction, documentation, and redaction tests.

## Best Possible Solution

Define one sanitized outbound action-param boundary before plugin dispatch and fallback delivery, then cover bundled plugin send and poll paths with regression tests.

## Reproduction Assessment

Yes. A high-confidence source reproduction is a plugin-handled Slack, Discord, Telegram, or Mattermost send or poll containing a secret: the PR computes redacted fallback values, but plugin dispatch still receives unchanged action params.

## Solution Assessment

No. The PR redacts only part of the delivery path; the maintainable fix is to sanitize cloned action params once before plugin dispatch and use the same sanitized payload for fallback delivery.

## Review Findings

Full review comments:

- **[P1] Pass redacted params to plugin send actions:** `src/infra/outbound/outbound-send-service.ts:108-110`
  - body: `redactedMessage` is computed, but `tryHandleWithPluginAction` still dispatches the original `ctx.params`. Plugin send handlers read message text from those params, so plugin-based channels still send raw secrets.
  - confidence: 0.95
- **[P1] Route redacted poll params through plugins:** `src/infra/outbound/outbound-send-service.ts:184-186`
  - body: The redacted poll question/options are only used by the fallback path. Plugin poll handlers run first and read `pollQuestion`/`pollOption` from unchanged params, so secrets still leak when a plugin handles the poll.
  - confidence: 0.94
- **[P1] Emit dynamic masks with a function replacer:** `src/logging/redact.ts:112`
  - body: Passing `masked` as a string replacement lets `$&`, `$'`, and related sequences in preserved secret prefixes/suffixes be expanded by JavaScript. Use a function replacer so the mask is emitted literally.
  - confidence: 0.91
- **[P2] Tighten auth key discovery:** `src/logging/redact-init.ts:20`
  - body: The `/auth/i` key pattern also matches benign keys such as `author` and `authorization`, causing long non-secret values to be registered globally and redacted from user-visible outbound messages.
  - confidence: 0.88
