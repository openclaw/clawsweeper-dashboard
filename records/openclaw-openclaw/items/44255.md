## Summary

Keep open: the security goal is worthwhile, but the PR is not mergeable because plugin-handled channel sends and polls still receive raw action params, dynamic replacement can reinsert secrets, broad key discovery can redact benign content, and the broader guardrail alternative is still only an open PR.

## What This Changes

The PR adds dynamic sensitive-value registration, config/env scanning, CLI startup initialization, outbound send/poll redaction, documentation, and redaction tests.

## Best Possible Solution

Define a maintainer-approved sanitized outbound action-param boundary before plugin dispatch and fallback delivery, or supersede it with a reviewed broader guardrail design that covers the same external-disclosure risk.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: send or poll through a plugin-handled channel with a secret in message/poll params, and the PR dispatches unchanged action params before the redacted fallback values are used.

## Solution Assessment

No. The current PR redacts only part of the delivery path; the safer maintainable fix is to sanitize cloned action params once before plugin dispatch and align that with the broader guardrail decision.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Pass redacted params to plugin sends:** `src/infra/outbound/outbound-send-service.ts:108-110`
  - body: `redactedMessage` is computed, but plugin handling still receives the original `ctx.params`. Plugin send handlers read message text from those params, so plugin-based channels can still send raw secrets.
  - confidence: 0.95
- **[P1] Route redacted poll params through plugin actions:** `src/infra/outbound/outbound-send-service.ts:184-187`
  - body: The redacted poll question/options are only used by the fallback `sendPoll` path. Plugin poll handlers run first and read `pollQuestion`/`pollOption` from unchanged params, so secrets still leak when a plugin handles the poll.
  - confidence: 0.94
- **[P1] Use a function replacer for dynamic masks:** `src/logging/redact.ts:113`
  - body: Passing `masked` as a string replacement lets `$&`, `$'`, and related sequences in preserved secret prefixes/suffixes be expanded by JavaScript. Use a function replacer so the mask is emitted literally.
  - confidence: 0.91
- **[P2] Tighten auth key discovery:** `src/logging/redact-init.ts:20`
  - body: The `/auth/i` pattern also matches benign keys such as `author` and `authorization`, causing long non-secret values to be registered globally and redacted from user-visible outbound messages.
  - confidence: 0.88
