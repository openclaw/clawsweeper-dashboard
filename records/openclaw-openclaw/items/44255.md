## Summary

Keep this PR open, but it is not mergeable: the hardening goal is useful, yet the diff still leaves plugin-handled channel delivery unsanitized, adds dynamic-redaction edge cases, and lacks required real behavior proof for an external security change.

## What This Changes

The PR adds runtime secret discovery and dynamic redaction, wires redaction into outbound send/poll fallback paths and CLI startup, and adds redaction docs/tests.

## Best Possible Solution

Define one reviewed sanitization boundary before plugin dispatch and fallback delivery, with focused regression tests and redacted live channel proof, or supersede this branch with a maintainer-approved guardrail design.

## Reproduction Assessment

Yes at source level: a plugin-handled Telegram, Discord, Slack, or Mattermost send or poll containing a secret still receives raw `ctx.params` even though the PR computes redacted fallback values. I did not run a live channel repro.

## Solution Assessment

No. The current patch redacts only part of the delivery path; the narrower maintainable fix is to sanitize cloned outbound action params once before plugin dispatch, or to replace this with the broader reviewed guardrail direction.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Pass redacted params into plugin sends:** `src/infra/outbound/outbound-send-service.ts:110-112`
  - body: `redactedMessage` is computed, but plugin send handling still receives the original `ctx.params`. Plugin-based channels read message text from those params, so raw secrets can still be sent externally.
  - confidence: 0.95
- **[P1] Route redacted poll params through plugins:** `src/infra/outbound/outbound-send-service.ts:187-189`
  - body: The redacted poll question/options are only used by the fallback path. Plugin poll handlers run first with unchanged params, so secrets in poll text can still leak through Telegram or Discord poll delivery.
  - confidence: 0.94
- **[P1] Emit dynamic masks with a function replacer:** `src/logging/redact.ts:113`
  - body: Passing `masked` as a string replacement lets `$&`, `$'`, and related sequences in preserved secret prefixes or suffixes expand. Use a function replacer so the mask is emitted literally.
  - confidence: 0.91
- **[P2] Tighten the auth key pattern:** `src/logging/redact-init.ts:20`
  - body: The `/auth/i` discovery rule also matches benign keys such as `author` and `authorization`, causing unrelated long strings to be registered and redacted from outbound messages.
  - confidence: 0.88
- **[P2] Use one redaction length threshold for env discovery:** `src/logging/redact-init.ts:67-70`
  - body: Environment scanning still requires length 18 while the dynamic redactor accepts length 12, so env-backed custom secrets from 12 to 17 characters can be skipped and sent unredacted.
  - confidence: 0.84
