## Summary

Keep open. The branch is directionally useful, but PR head still risks suppressing retryable provider timeouts and leaves one cron timeout shape uncovered; the real behavior proof is also partial for the central run-budget path.

## What This Changes

The PR tags terminal abort reasons, threads abort signals into model fallback callers, records embedded run-budget timeouts, updates failover metadata, and adds regression tests and a changelog entry.

## Best Possible Solution

Constrain terminal abort detection to explicit abort signals, AbortError causes, or the structured run-budget marker, recognize cron timeout variants, and add redacted live proof of a run-budget timeout skipping fallback.

## Reproduction Assessment

Yes. Source inspection shows current main normalizes `TimeoutError` into retryable timeout failover, while PR head would rethrow a direct `TimeoutError` before normalization and exact-match only the base cron timeout string.

## Solution Assessment

No, not as-is. The direction is narrow, but the implementation should avoid treating top-level provider timeouts as terminal and should match phase-suffixed cron timeout reasons before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Limit terminal timeout checks to abort causes:** `src/agents/model-fallback.ts:215-233`
  - body: `isTerminalAbortFromError` includes the thrown error itself, so a provider or SDK error with `name === "TimeoutError"` is rethrown before `coerceToFailoverError` can classify it as a retryable timeout. Restrict this helper to AbortError causes or an explicit run-budget marker so ordinary provider timeouts still fall back.
  - confidence: 0.9
- **[P2] Match phase-suffixed cron timeout reasons:** `src/agents/model-fallback.ts:163-164`
  - body: The string branch exact-matches only `cron: job execution timed out`, but current cron timeout messages can include `(last phase: ...)`. Those terminal cron timeouts will still fall through into fallback retries; match the stable prefix or use a structured abort reason.
  - confidence: 0.88
