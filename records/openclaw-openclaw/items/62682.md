## Summary

Keep this PR open: it is a useful, well-evidenced bug fix, but it still misses one abortable fallback caller. The main agent/gateway paths now carry terminal-abort state, but compaction fallback still has an existing abort signal and does not pass it into `runWithModelFallback`, so a terminal abort during compaction can still cascade to another compaction model.

## What This Changes

The PR changes agent and gateway fallback handling so terminal aborts from run-budget timeouts, HTTP disconnects, and cron timeout reasons short-circuit model fallback instead of retrying.

## Best Possible Solution

Land this after every abortable `runWithModelFallback` caller with an existing abort signal, including compaction, passes that signal through while preserving retryable provider-timeout fallback behavior.

## Reproduction Assessment

Yes. Current main is source-reproducible: `scheduleAbortTimer` aborts with a TimeoutError-shaped reason, while `model-fallback.ts` treats timeout aborts as retryable instead of terminal.

## Solution Assessment

No, not yet. The core approach is maintainable and well-scoped, but it should also pass the existing compaction abort signal into the fallback helper before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Propagate abort signals through compaction fallback:** `src/agents/pi-embedded-runner/compact.ts:429-435`
  - body: `compactEmbeddedPiSessionDirect` already accepts `params.abortSignal` and passes it to `compactWithSafetyTimeout`, but the surrounding `runWithModelFallback` call never receives that signal. If a client disconnect or run-budget abort happens during compaction, the first compaction candidate can be returned as a failed fallback attempt and the next compaction model can still start, which preserves the retry churn this PR is trying to stop for this abortable path. Pass `abortSignal: params.abortSignal` here and add a focused regression for terminal aborts during compaction fallback.
  - confidence: 0.82
