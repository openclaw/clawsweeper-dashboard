## Summary

Keep this PR open. The bug is valid and the direction is useful, but the current diff still has two source-backed correctness blockers: it can make ordinary provider timeouts non-retryable, and it misses phase-suffixed cron timeout reasons.

## What This Changes

This PR threads abort context into agent model fallback and marks terminal run aborts, client disconnects, and run-budget timeouts so they stop fallback or assistant rotation instead of retrying.

## Best Possible Solution

Keep the terminal-abort approach, but constrain error-chain detection to explicit abort wrappers or structured run-budget markers and match cron timeout variants before merge.

## Reproduction Assessment

Yes. Source inspection gives high-confidence reproduction paths: a top-level provider `TimeoutError` now bypasses fallback, and a cron abort reason with `(last phase: ...)` misses the PR’s exact terminal string match.

## Solution Assessment

No, not as-is. The intended boundary is narrow and maintainable, but the implementation must preserve provider-timeout fallback and cover current cron timeout reason shapes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep top-level provider timeouts retryable:** `src/agents/model-fallback.ts:215-233`
  - body: `isTerminalAbortFromError` includes the thrown error itself in `candidates`, so any provider or SDK error with `name === "TimeoutError"` is rethrown before `coerceToFailoverError` can classify it as a retryable timeout. Current main treats that shape as `reason: "timeout"`; limit terminal detection to AbortError causes or an explicit run-budget marker so ordinary provider timeouts still try configured fallbacks.
  - confidence: 0.9
- **[P2] Match phase-suffixed cron timeout reasons:** `src/agents/model-fallback.ts:123-126`
  - body: The string branch exact-matches only `cron: job execution timed out`, but current cron timeout messages can include `(last phase: ...)`. Those terminal cron aborts will still miss detection and continue fallback retries; match the stable prefix or pass a structured abort reason instead.
  - confidence: 0.92
