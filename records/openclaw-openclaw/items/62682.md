## What This Changes

The branch adds terminal-abort detection and propagation so run-budget, client-disconnect, cron-timeout, compaction, and OpenClaw abort-wrapper paths stop model fallback or auth rotation once the run is already over.

## Best Possible Solution

Land this branch if maintainers accept the terminal-abort compatibility boundary; otherwise narrow the approved abort markers while preserving ordinary provider, SDK, idle-stream, and auth-profile fallback behavior.

## Reproduction Assessment

Yes, at source level: current main only has caller-signal terminal checks, while the embedded run-budget path aborts a private controller and failover policy still treats ordinary `timedOut` as rotation/fallback eligible. I did not run a live timeout repro in this read-only review.

## Solution Assessment

Yes, this appears to be the maintainable fix shape: it distinguishes OpenClaw-owned abort wrappers from provider timeouts and threads a specific run-budget flag through failover. The remaining question is maintainer acceptance of the compatibility boundary, not a concrete code defect.
