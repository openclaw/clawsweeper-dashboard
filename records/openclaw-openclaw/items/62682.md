## Summary

Keep open for maintainer review. The patch appears to address a real source-reproducible fallback bug on current main, and I found no blocking code or security finding, but the behavior change is compatibility-sensitive because it intentionally stops configured fallback chains earlier for terminal aborts.

## What This Changes

The PR threads abort signals into agent fallback paths and adds terminal-abort classification/tests so run-budget, cron, compaction, and HTTP client-disconnect aborts short-circuit model fallback.

## Best Possible Solution

Land this PR if maintainers accept terminal-abort fail-fast semantics, while leaving broader cron remaining-budget tuning to the separate cron PR.

## Reproduction Assessment

Yes. Source inspection shows current main has TimeoutError-producing run-budget aborts and string-producing cron aborts while model fallback only distinguishes AbortError from non-timeout errors and does not inspect terminal abort reasons.

## Solution Assessment

Yes, with a maintainer policy caveat. Centralizing terminal-abort classification in model fallback and threading existing abort signals through callers is the narrow maintainable fix, while broader cron budget allocation belongs to the separate cron PR.
