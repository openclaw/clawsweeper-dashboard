- `P1`: The PR targets an active agent fallback regression and currently introduces a blocking regression in timeout fallback behavior.

- `merge-risk: 🚨 message-delivery`: Misclassifying retryable model timeouts as terminal can stop an agent run before another configured candidate has a chance to produce a user-visible response.

## Summary

Keep open: the branch still addresses a real fallback-abort gap, but the current implementation would also make ordinary LLM idle-timeout aborts terminal and bypass the existing profile/model fallback path.

## What This Changes

The PR adds terminal-abort detection and timeout metadata so model fallback and assistant failover stop for run-budget, HTTP disconnect, cron timeout, compaction, and OpenClaw abort-wrapper timeout paths.

## Best Possible Solution

Mark only the whole-run budget abort source as terminal, keep LLM idle timeout fallback/profile rotation intact, and rebase the PR so duplicated current-main plumbing drops out cleanly.

## Reproduction Assessment

Yes. Source inspection shows the idle-timeout hook calls `abortRun(true, error)`, the prompt path wraps that signal through `abortable()`, and the PR's new `isTerminalAbortFromError` would treat the resulting marker-wrapped TimeoutError as terminal before fallback can run.

## Solution Assessment

No. The right fix is to identify the run-budget timer specifically, not every OpenClaw abortable wrapper with a TimeoutError cause, because current failover-policy tests deliberately keep LLM idle timeouts retryable.
