## Summary

Keep this PR open for maintainer review. Current main has the base caller-abort handling from merged related work, but it does not fully cover this PR's unique terminal-abort paths: tagged HTTP disconnects, cron string reasons, wrapper-cause detection for the embedded runner's private run-budget controller, and the `timedOutByRunBudget` failover state.

## What This Changes

The PR extends terminal abort handling across model fallback, HTTP disconnect, embedded run-budget timeout, compaction fallback, failover policy, and trajectory metadata.

## Best Possible Solution

Merge or narrow this branch after an owner accepts the terminal-abort boundary as the supported fallback contract, preserving retryable provider and LLM idle timeout behavior.

## Reproduction Assessment

Yes. Source inspection shows current main can produce a private embedded-runner TimeoutError from the run-budget controller while the fallback layer only has base caller-signal terminality; the PR's tests and proof target that missing thrown-error/metadata boundary.

## Solution Assessment

Yes, with maintainer signoff. The marker-gated thrown-error check plus explicit `timedOutByRunBudget` flag is a bounded fix that preserves ordinary provider and LLM idle timeout fallback, but it changes compatibility-sensitive fallback/auth semantics.
