## Summary

Keep this PR open: the terminal-abort direction is useful and the updated body now supplies real runtime proof, but latest head still classifies a top-level provider `TimeoutError` as terminal before failover normalization, which would skip configured fallback models for ordinary provider timeouts.

## What This Changes

This PR adds terminal abort detection and abort-signal propagation through model fallback, embedded runner, gateway, cron, auto-reply, trajectory metadata, tests, and changelog text for run-budget timeout and client-disconnect cases.

## Best Possible Solution

Keep the terminal-abort handling, but restrict thrown-error terminal detection to explicit `AbortError` wrappers or known terminal signal/string reasons while preserving direct provider `TimeoutError` fallback behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: make `runWithModelFallback` receive a direct provider `Error` whose `name` is `TimeoutError`; PR head rethrows before normalization, while current main would classify it as a retryable timeout failover.

## Solution Assessment

No, not as-is. The overall direction is maintainable, but the thrown-error helper is too broad because it treats the top-level provider error itself as terminal instead of only inspecting explicit terminal abort signals or abort wrapper cause chains.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Restrict terminal detection to abort wrappers:** `src/agents/model-fallback.ts:230`
  - body: `isTerminalAbortFromError` starts with `err` itself, so a provider/SDK error whose top-level `name` is `TimeoutError` is treated as terminal and rethrown before `coerceToFailoverError` can convert it into a retryable timeout. Current main intentionally lets ordinary provider timeouts continue through configured fallback models, so this would turn provider timeouts into complete failures. Start from the cause chain of an outer `AbortError` or a known terminal wrapper, and add a regression for direct provider `TimeoutError` fallback.
  - confidence: 0.92
