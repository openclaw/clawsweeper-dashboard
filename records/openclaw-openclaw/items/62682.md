## Summary

Keep open: current main has caller-signal terminal abort handling, but this PR still adds unique private embedded-runner run-budget and string/cause abort classification. ## What This Changes

The PR adds terminal-abort handling for run-budget, HTTP disconnect, cron timeout, compaction, and embedded abort-wrapper paths so model fallback/auth rotation stops when the run is already over.

## Best Possible Solution

Merge after maintainers accept the terminal-abort compatibility boundary and current proof/CI remain tied to the exact head; otherwise narrow the approved terminal sources before landing.

## Reproduction Assessment

Yes. Current main lacks private embedded-runner abort wrapper detection and still treats ordinary run-budget `timedOut` as fallback/rotation eligible, which matches the reported cascade path.

## Solution Assessment

Yes, with a maintainer-policy caveat. Marker-gated wrapper detection plus an explicit run-budget flag is narrower than blanket timeout-abort terminality, but maintainers must accept which abort sources are terminal.
