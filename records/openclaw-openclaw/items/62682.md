## Summary

Keep open: the core approach and supplied proof are strong, but one abortable compaction fallback caller still drops its existing abort signal, so terminal compaction aborts can still cascade to fallback compaction models.

## What This Changes

The PR changes agent and gateway fallback handling so terminal aborts from run-budget timeouts, HTTP disconnects, and cron timeout reasons stop the model fallback chain instead of retrying.

## Best Possible Solution

Land the terminal-abort approach after the compaction fallback caller also passes its abort signal and focused tests prove terminal compaction aborts short-circuit while retryable provider failures still fall back.

## Reproduction Assessment

Yes, by source inspection. Current main shows scheduleAbortTimer aborting with a TimeoutError-shaped reason while model-fallback.ts treats timeout aborts as retryable, and the PR body supplies after-fix Docker/log proof for the main HTTP and cron-shaped paths.

## Solution Assessment

No, not yet. The main design is the right owner-boundary fix, but the compaction fallback caller should pass params.abortSignal into runWithModelFallback before this is merge-ready.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Propagate compaction abort signals into fallback:** `src/agents/pi-embedded-runner/compact.ts:429-435`
  - body: `compactEmbeddedPiSessionDirect` passes `params.abortSignal` into `compactWithSafetyTimeout`, but the surrounding `runWithModelFallback` call never receives that signal. If a run-budget timeout or HTTP disconnect aborts during compaction, `compactWithSafetyTimeout` can throw the terminal `TimeoutError`/`ClientDisconnectError` directly; without the signal, the new terminal-abort guard can still treat the first compaction attempt as retryable and start another compaction model after the run is already over. Pass `abortSignal: params.abortSignal` here and add a focused regression for terminal aborts during compaction fallback.
  - confidence: 0.84
