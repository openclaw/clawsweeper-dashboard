## Summary

Keep open. The terminal-abort approach is well scoped and the supplied real-environment proof is sufficient, but one abortable compaction fallback caller still does not pass its existing signal into `runWithModelFallback`, so terminal disconnect/run-budget aborts can still cascade there.

## What This Changes

This PR threads terminal abort handling through model fallback, tags HTTP disconnect aborts, adds embedded run-budget timeout metadata, and adds tests plus a changelog entry.

## Best Possible Solution

Pass the existing compaction abort signal into the fallback helper, add focused regression coverage for terminal aborts during compaction fallback, then land this terminal-abort fix while leaving the proactive cron remaining-budget gate to the related cron PR.

## Reproduction Assessment

Yes. Current main is source-reproducible: the embedded run-budget timer creates a TimeoutError-shaped abort, while the current fallback helper treats timeout aborts as retryable; the PR body also supplies after-fix Docker/log proof for the intended behavior.

## Solution Assessment

No, not yet. The core direction is the narrow maintainable fix, but every abortable fallback caller with an existing signal, including compaction, should pass that signal through before merge.
