## Summary

Keep this active PR open: it is a focused fix with strong proof, but the current terminal-from-error guard is still too broad and can turn retryable provider timeout wrappers into terminal aborts.

## What This Changes

The PR changes agent model fallback handling so terminal aborts from run-budget expiry, HTTP client disconnects, cron timeout strings, and compaction aborts stop fallback retries instead of cascading through candidate models.

## Best Possible Solution

Tag run-budget and client-disconnect aborts with a terminal-specific marker, keep generic provider timeout wrappers on the retryable failover path, and then land the centralized terminal-abort plumbing.

## Reproduction Assessment

Yes. Current main source shows timeout-shaped AbortError values flow through retryable failover normalization, and the PR body supplies Docker/bundle proof for the terminal cases it wants to change.

## Solution Assessment

No, not quite. The centralized terminal-abort approach is right, but the thrown-error guard needs a terminal-specific marker so provider timeout wrappers remain retryable.
