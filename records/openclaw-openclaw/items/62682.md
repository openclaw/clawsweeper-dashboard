## Summary

Keep open. Current main still treats run-budget and cron timeout aborts as retryable fallback failures, while this PR is an active, useful fix with sufficient runtime proof and no discrete blocking code finding from this review; the remaining question is maintainer acceptance of the fail-fast compatibility change.

## What This Changes

The PR adds terminal-abort classification and abort-signal plumbing so run-budget, cron, compaction, and HTTP client-disconnect aborts stop model fallback instead of trying more candidates.

## Best Possible Solution

Land this PR if maintainers accept terminal-abort fail-fast semantics, while leaving broader cron remaining-budget tuning to the separate cron PR.

## Reproduction Assessment

Yes. Source inspection shows current main has TimeoutError-producing run-budget aborts and string-producing cron aborts while model fallback only distinguishes AbortError from non-timeout errors and does not inspect terminal abort reasons.

## Solution Assessment

Yes, with a maintainer policy caveat. Centralizing terminal-abort classification in model fallback and threading existing abort signals through callers is the narrow maintainable fix; broader cron budget allocation belongs to the separate cron PR.
