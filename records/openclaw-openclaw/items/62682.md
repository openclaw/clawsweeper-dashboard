## Summary

Keep open: the fix direction and proof are strong, but one abortable compaction fallback caller still does not pass its existing abort signal into the new terminal-abort fallback boundary, so the PR is not merge-ready yet.

## What This Changes

The PR changes agent and gateway fallback handling so terminal aborts from run-budget timeouts, HTTP disconnects, and cron timeout reasons stop model fallback instead of retrying.

## Best Possible Solution

Pass the existing compaction abort signal into compactEmbeddedPiSessionDirect's runWithModelFallback call and add focused regression coverage for terminal aborts during compaction fallback while preserving retryable provider-timeout fallback.

## Reproduction Assessment

Yes. Current main is source-reproducible: scheduleAbortTimer aborts with a TimeoutError-shaped reason, while model-fallback treats timeout aborts as retryable instead of terminal.

## Solution Assessment

No, not yet. The core approach is maintainable, but it needs to include the existing compaction abort signal at the fallback boundary before merge.
