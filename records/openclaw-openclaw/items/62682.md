## Summary

Keep open: the PR targets a valid source-reproducible timeout/fallback bug, but the latest head still over-classifies direct `TimeoutError` values as terminal, misses phase-suffixed cron timeout reasons, and needs real run-budget proof before merge.

## What This Changes

The branch adds terminal abort detection to model fallback, tags HTTP disconnect aborts, threads abort signals through callers, adds `timedOutByRunBudget` metadata, updates failover policy/tests and trajectory metadata, and adds a changelog entry.

## Best Possible Solution

Use a precise terminal run-budget contract that preserves provider-timeout fallback, covers current cron timeout reason variants with a stable marker or prefix, coordinates with https://github.com/openclaw/openclaw/pull/52365, and includes redacted runtime proof for the timeout path.

## Reproduction Assessment

Yes, source-reproducible. Current main's embedded run-budget timer marks a timeout, the assistant failover policy treats non-compaction/non-tool timeouts as fallback-eligible, and model fallback continues timeout-shaped candidate failures.

## Solution Assessment

No, not as-is. The direction is narrow, but terminal detection should avoid top-level direct `TimeoutError` matches and should handle current cron timeout reason variants through a structured marker or stable prefix.
