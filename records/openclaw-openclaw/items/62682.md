## Summary

Keep open. ## What This Changes

The branch adds terminal abort classification for run-budget, cron timeout, HTTP disconnect, and OpenClaw abortable-wrapper paths, plus failover-policy metadata and tests to stop model fallback when the run is already over.

## Best Possible Solution

## Reproduction Assessment

Yes, this appears to be the maintainable fix shape: the PR distinguishes OpenClaw-owned abort wrappers from provider timeout wrappers and threads a specific run-budget flag through failover. The remaining question is maintainer acceptance of the compatibility boundary, not a concrete code defect.
