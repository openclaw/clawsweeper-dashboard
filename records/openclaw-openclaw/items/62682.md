## What This Changes

The PR adds terminal abort detection to model fallback, propagates abort signals, tags HTTP client disconnects, threads a run-budget timeout flag through embedded failover/trajectory metadata, and adds tests plus a changelog entry.

## Best Possible Solution

Land the scoped PR after green CI or targeted maintainer validation, preserving provider/LLM idle timeout fallback while stopping fallback only for whole-run terminal aborts.

## Reproduction Assessment

Yes. ## Solution Assessment

Yes. The latest approach is the narrow maintainable shape: a dedicated `timedOutByRunBudget` flag handles the private embedded timer path, while signal/error reason checks cover caller-driven HTTP and cron terminal aborts without changing normal provider timeout fallback.
