## Summary

Keep open. The PR improves external abort plumbing, but it still does not give the model fallback layer the embedded runner's own timeout signal, so the central run-budget timeout path can still rotate profiles and escalate to model fallback.

## What This Changes

The PR adds terminal abort detection to model fallback, threads optional abort signals through agent, auto-reply, and cron callers, tags HTTP disconnect aborts, and adds unit coverage.

## Best Possible Solution

Expose an explicit embedded-run terminal timeout marker to the fallback layer without conflating provider/request timeouts, keep the HTTP disconnect tagging if maintainers want that behavior, and add the required changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a focused reproduction path: run an embedded agent with a short `timeoutMs` and multiple fallbacks, then assert the embedded runner's own `scheduleAbortTimer` stops after the first candidate rather than relying on a synthetic external `AbortSignal`.

## Solution Assessment

No. Passing the caller abort signal is appropriate for HTTP disconnects and cron wrapper aborts, but it is not the narrowest complete fix for the embedded runner's private run-budget timeout.
