## Summary

Keep open. Current main preserves established external route state, but it still does not add the requested opt-in WebChat-to-last-external-channel mirroring; the config key is absent, WebChat route inheritance is explicitly blocked, and the related implementation PR is closed unmerged.

## What This Changes

Add a default-off session config option that mirrors Control UI/WebChat replies to the session's last established external channel.

## Best Possible Solution

Add a documented default-false session-level option that keeps WebChat delivery intact and mirrors only complete replies through the existing external routeReply path when a valid persisted external route exists.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: seed lastChannel/lastTo for an external route, send via a WebChat chat.send client, and current main resolves the turn as internal WebChat with no external route for reply delivery.

## Solution Assessment

Yes. The requested default-off flag is the narrow maintainable direction because it avoids restoring the old implicit global mirroring while reusing existing persisted route state and outbound delivery code.

## Repair Work Prompt

Implement the default-off WebChat-to-last-external-channel mirroring feature for openclaw/openclaw#44302. Add a session-level config option, preferably `session.mirrorWebchatToLastExternalChannel` unless maintainers have a newer naming convention, defaulting to false. When enabled and a turn originates from WebChat/Control UI, keep normal WebChat delivery and additionally mirror complete final replies to the session's persisted external route (`deliveryContext`/`lastChannel`/`lastTo`) only when that route is valid and deliverable. Do not restore implicit global mirroring, do not mirror when no external route exists, do not mirror external-origin turns, do not mirror partial streaming/tool-progress payloads, and do not make mirror failure block WebChat delivery. Update config types/schema/help/generated config docs as required, add focused regression tests for enabled/default/no-route/non-webchat/failure cases, and update user-facing docs/changelog for the new option. Related refs: #44302, #44261, #43353, #52068.
