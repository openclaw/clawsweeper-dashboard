## Summary

Keep open: current main and the latest shipped release still treat WebChat as an internal reply sink, expose no opt-in mirroring config, and the related implementation attempts are closed rather than merged. This remains a product/config decision for cross-channel delivery semantics.

## What This Changes

The issue requests a default-off session config option that mirrors Control UI/WebChat replies to the last established external channel route.

## Best Possible Solution

Add a narrow, documented, default-off session policy for completed WebChat replies to mirror only to a valid persisted external route after maintainers approve the delivery semantics.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source and docs inspection gives high confidence that current main and v2026.5.12 intentionally do not mirror ordinary WebChat replies to external channels.

## Solution Assessment

Unclear as a final implementation path. A default-off session policy is plausible, but maintainers still need to approve cross-channel mirroring semantics before a focused implementation should proceed.
