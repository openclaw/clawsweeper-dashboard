## Summary

Keep open: current main still intentionally keeps WebChat sends internal, exposes no opt-in mirroring config, and this issue remains the canonical product decision while the related implementation attempts are closed or still broad/conflicting.

## What This Changes

Request adds an opt-in session config that mirrors Control UI/WebChat-originated replies to the session's last active external channel when a persisted external route exists.

## Best Possible Solution

Add a narrow, documented, default-off session policy for completed WebChat replies to mirror only to a valid persisted external route, with route-safety safeguards and live external-channel proof.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source inspection gives high confidence that current main has no opt-in config and intentionally keeps ordinary WebChat sends internal.

## Solution Assessment

Unclear as a final implementation path. A default-off session policy is plausible, but maintainers still need to approve mirroring semantics and the broad open PR should likely be split or replaced.
