## Summary

Keep open: current main still has no opt-in WebChat-to-external mirroring config, and the remaining work is a product/privacy decision rather than a conservative cleanup close.

## What This Changes

Request an opt-in session config that mirrors Control UI/WebChat replies to the session's last established external channel.

## Best Possible Solution

Define a narrow, documented, default-off session policy for completed WebChat replies to mirror only to a valid persisted external route after maintainers approve privacy, stale-route, and failure semantics.

Not applicable as a bug reproduction because this is a feature/config request. Source, tests, current main, and linked issue context give high confidence that ordinary WebChat replies intentionally stay in WebChat today.

## Solution Assessment

Unclear as a final implementation path. A default-off session policy is plausible, but maintainers still need to approve cross-channel mirroring, stale-route, duplicate-delivery, and privacy semantics.
