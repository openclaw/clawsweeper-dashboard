## Label Justifications

- `impact:message-loss`: Replies can remain visible only in WebChat when the operator expected the external Discord, Telegram, or similar channel to receive them.

## Summary

Keep open: current main and v2026.5.12 still make WebChat an internal reply sink with no opt-in external mirroring config, and the linked implementation attempts were closed rather than merged. This remains a product and privacy decision for cross-channel delivery semantics.

## What This Changes

The issue requests a default-off session config option that mirrors WebChat-originated replies to the session's last active external channel.

## Best Possible Solution

Add a narrow, documented, default-off session policy for completed WebChat replies to mirror only to a valid persisted external route after maintainers approve the privacy and stale-route semantics.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source, tests, docs, and v2026.5.12 release inspection give high confidence that ordinary WebChat replies intentionally stay in WebChat today.

## Solution Assessment

Unclear as a final implementation path. A default-off session policy is plausible, but maintainers still need to approve cross-channel mirroring semantics before a focused implementation should proceed.
