## Summary

Keep open. Current main still lacks the requested default-off WebChat-to-external-channel mirroring config and deliberately keeps normal Control UI/WebChat turns internal; the overlapping open implementation PR is broad, blocked by review findings, and not a clean canonical fix yet.

## What This Changes

Add an opt-in session config that mirrors Control UI/WebChat replies to the session's last active external channel.

## Best Possible Solution

Add a narrow, documented, default-false session policy that mirrors only completed WebChat replies through the existing delivery path when a valid persisted external route exists, with stale-route and privacy safeguards.

## Reproduction Assessment

Not applicable as a bug reproduction because this is an additive feature request. Source inspection gives high confidence that current main intentionally keeps WebChat turns internal and lacks the requested opt-in config.

## Solution Assessment

Unclear until maintainers settle the product contract. A default-off session option is plausible, but it should be split from the broad open PR and validated with real Discord/Telegram or equivalent channel proof.
