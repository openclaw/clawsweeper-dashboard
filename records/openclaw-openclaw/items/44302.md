## Summary

Keep open. Current main and the latest release still keep ordinary Control UI/WebChat sends on WebChat unless an explicit external delivery route is requested, the requested default-off mirroring config is absent, and the related implementation PRs are closed unmerged rather than safe canonical fixes.

## What This Changes

Add a default-off session config option that mirrors Control UI/WebChat replies to the session's last established external channel.

## Best Possible Solution

Implement this only after maintainer approval of a default-off cross-channel mirroring policy with stale-route and authorization safeguards, schema/docs updates, and focused routing coverage.

## Reproduction Assessment

Yes, at source level: Control UI sends `chat.send` with `deliver: false`, Gateway resolves that as WebChat without `ExplicitDeliverRoute`, and the reply policy blocks external routing for internal WebChat turns. I did not run a live Discord or Telegram WebChat flow in this read-only review.

## Solution Assessment

Unclear. A default-off mirroring flag is safer than restoring implicit fan-out, but maintainers still need to choose the product and security model versus explicit per-send delivery or the broader force-origin policy.
