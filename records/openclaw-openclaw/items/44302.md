## Summary

Keep open. Current main still lacks any `mirrorWebchatToLastExternalChannel`-style config and intentionally keeps normal Control UI/WebChat sends on the internal route, while preserving external route metadata for other follow-up paths; the remaining request is a real default-off feature/config decision, not an already-shipped bug fix.

## What This Changes

The issue requests an opt-in session config option that mirrors Control UI/WebChat replies to the session's last active external channel.

## Best Possible Solution

Add a narrow, documented, default-false session policy that mirrors only completed WebChat replies through the existing delivery path when a valid persisted external route exists, with privacy and stale-route safeguards.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection confirms current main intentionally keeps WebChat/UI turns internal and lacks the requested opt-in config.

## Solution Assessment

Unclear as a final implementation path. A default-off config is plausible, but maintainers still need to settle semantics before a narrow routing/delivery change should land.
