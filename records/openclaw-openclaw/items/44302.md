## Summary

Keep open: current main and v2026.5.20 still lack the requested opt-in mirroring policy, and the related mirroring PRs were closed unmerged. The remaining work is a real cross-channel feature with privacy and duplicate-delivery semantics that needs maintainer product/security judgment before implementation.

## What This Changes

Add a default-off session config option that mirrors Control UI/WebChat replies to the session's last established external channel.

## Best Possible Solution

Define a narrow, documented, default-off session policy that mirrors only completed WebChat replies to a valid persisted external route after privacy, stale-route, duplicate-delivery, failure, and sendPolicy semantics are approved.
