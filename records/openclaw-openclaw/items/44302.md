## Summary

Keep open. Current main preserves external route metadata but still intentionally keeps normal Control UI/WebChat replies internal and lacks the requested opt-in mirroring config; the overlapping open PR is broad and not yet a clean canonical fix.

## What This Changes

The issue requests a default-off session config option that mirrors Control UI/WebChat replies to the session's last established external channel.

## Best Possible Solution

Add a narrow, documented default-false session policy that mirrors only completed WebChat replies through the existing delivery path when a valid persisted external route exists, with stale-route and privacy safeguards.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection confirms current main intentionally keeps WebChat/UI turns internal and lacks the requested opt-in config.

## Solution Assessment

Unclear as an implementation path. A default-off option is plausible, but maintainers still need to settle the public config shape and likely split the broad open implementation candidate before landing.
