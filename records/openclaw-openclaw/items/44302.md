## Summary

Keep open. Current main and v2026.6.10 still lack the requested opt-in mirroring config and intentionally keep WebChat-originated replies local unless explicit external delivery is set; related attempts are closed unmerged or broader open work that still needs maintainer product/security direction.

## What This Changes

Request an optional config setting that mirrors Control UI/WebChat replies to the session's last active external channel for multi-device workflows.

## Best Possible Solution

Define one explicit cross-surface delivery policy with stale-route, no-route, duplicate-delivery, authorization, and privacy rules; then either implement this narrow config or fold the workflow into an accepted pinning/echo model with live channel proof.

## Reproduction Assessment

Yes, at source level. Current routing and gateway code keep WebChat-originated replies local unless explicit external delivery is set, tests assert that behavior, and the requested config key is absent.

## Solution Assessment

Unclear until maintainers choose the policy. A default-off mirror flag is plausible, but the best fix must define route freshness, authorization, duplicate-delivery, and privacy behavior before code changes.
