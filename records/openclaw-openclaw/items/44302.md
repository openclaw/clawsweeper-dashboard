## Label Justifications

## Summary

Keep open. Current main and v2026.6.10 still keep WebChat/Control UI replies local unless explicit external delivery is requested, the requested default-off mirroring config key is absent, and the exact prior implementation attempt at https://github.com/openclaw/openclaw/pull/52068 is closed unmerged; this remains a product and security policy decision.

## What This Changes

Add a default-off config option to mirror WebChat/Control UI replies to a session's last established external channel.

## Best Possible Solution

Define one operator-controlled cross-surface delivery policy with explicit stale-route, no-route, authorization, privacy, and live-channel proof requirements; either implement this config narrowly or fold it into an accepted pinning/echo model.

## Reproduction Assessment

Yes, at source level. Current routing code and tests keep WebChat-originated replies local unless explicit delivery is set, and the requested mirroring config key is absent on current main and v2026.6.10.

## Solution Assessment

Unclear until maintainers choose the policy. A default-off flag is plausible, but the safer solution must define stale-route, authorization, no-route, duplicate-delivery, and privacy behavior before implementation.
