## Summary

Keep open. Current main still preserves established external route state but does not implement the requested opt-in WebChat-to-last-external-channel mirroring; the public config key is absent, WebChat route inheritance remains blocked, and the related implementation PR is closed unmerged.

## What This Changes

Add an optional session config flag that mirrors WebChat or Control UI replies to the session's last active external channel when an external route exists.

## Best Possible Solution

Add a documented default-false session-level option that keeps WebChat delivery intact and mirrors only complete replies through the existing routeReply path when a valid persisted external route exists.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: seed lastChannel/lastTo for an external route, send through a WebChat chat.send client, and current main resolves the turn as internal WebChat with no external route for reply delivery.

## Solution Assessment

Yes. The requested default-off flag is the narrow maintainable direction because it avoids restoring implicit global mirroring while reusing existing persisted route state and outbound delivery code.

## Repair Work Prompt

Implement the opt-in WebChat-to-last-external-channel mirroring feature for https://github.com/openclaw/openclaw/issues/44302. Add a default-false session-level config option in the existing session config namespace, update schema/help/labels/generated config docs, and document the behavior. Preserve current default behavior: WebChat replies still go only to WebChat unless the flag is enabled. When enabled, mirror only complete sendable WebChat-origin replies to a valid persisted external route using the existing routeReply path and persisted lastChannel/lastTo/deliveryContext state; do not mirror when no external route exists, when the origin is already external, when the target is webchat/internal, or when sendPolicy blocks delivery. Check routeReply's { ok: false, error } result and log mirror failures without blocking WebChat delivery. Add focused regression tests for enabled mirroring, default-off behavior, no-route behavior, external-origin behavior, sendPolicy interaction, and mirror failure logging. Add a user-facing changelog entry under the active Changes section, crediting @rbutera if appropriate. Do not restore unconditional implicit mirroring or broaden plugin/channel routing policy beyond this config-gated feature.
