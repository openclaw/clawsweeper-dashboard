## Summary

Keep open: current main and v2026.5.20 still intentionally keep ordinary Control UI/WebChat replies internal, and the requested default-off mirroring config does not exist. The closed implementation attempts are useful prior art, but this remains a product and privacy-sensitive routing decision rather than conservative cleanup.

## What This Changes

Request to add an opt-in session config that mirrors Control UI/WebChat replies to the session's last active external channel.

## Best Possible Solution

Define a narrow, documented, default-off session policy that mirrors only completed WebChat replies to a valid persisted external route after maintainers approve privacy, stale-route, duplicate-delivery, failure logging, and sendPolicy semantics.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source, tests, current main, latest release, and linked issue context give high confidence that ordinary WebChat replies intentionally stay in WebChat today.

## Solution Assessment

Unclear as a final implementation path. A default-off session policy is plausible, but maintainers still need to approve cross-channel mirroring, stale-route, duplicate-delivery, sendPolicy, and privacy semantics.
