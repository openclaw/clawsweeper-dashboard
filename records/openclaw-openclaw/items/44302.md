## Summary

Keep open: current main and the latest release still intentionally keep ordinary WebChat replies internal, and no opt-in mirroring key exists. The related implementation attempts are closed/unmerged, so this remains a product and privacy decision rather than conservative cleanup.

## What This Changes

Request an optional default-off session config to copy WebChat-originated agent replies to the session's last established external channel.

## Best Possible Solution

Define a narrow, documented, default-off session policy that mirrors only completed WebChat replies to a valid persisted external route after privacy, stale-route, duplicate-delivery, failure logging, and sendPolicy semantics are approved.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source, tests, current main, latest release, and linked issue context give high confidence that ordinary WebChat replies intentionally stay in WebChat today.

## Solution Assessment

Unclear as a final implementation path. A default-off session policy is plausible, but maintainers still need to approve cross-channel mirroring, stale-route, duplicate-delivery, sendPolicy, and privacy semantics.
