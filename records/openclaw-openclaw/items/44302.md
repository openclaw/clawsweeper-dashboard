## Summary

Keep open. Current main preserves existing external route state for sessions, but it still does not add the requested default-off WebChat-to-last-external-channel mirroring behavior, and the related implementation PR is closed unmerged.

## What This Changes

Request an opt-in session config flag that mirrors Control UI/WebChat replies to the session’s last established external channel while keeping default WebChat-only delivery.

## Best Possible Solution

Add a default-false session-level mirroring option that keeps WebChat as the primary delivery path and sends complete replies through the existing external `routeReply` path only when a valid persisted external route exists.

## Reproduction Assessment

Yes. Source inspection provides a current-main static reproduction: seed an external last route, send from a WebChat client, and `chat.send` emits WebChat context with no external destination.

## Solution Assessment

Yes, the requested opt-in flag is the narrow maintainable direction for this workflow. It avoids restoring the old implicit mirroring behavior globally while reusing the preserved last-route state already present on current main.
