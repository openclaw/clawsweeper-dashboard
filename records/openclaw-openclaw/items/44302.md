## Summary

Keep open. Current main still intentionally keeps WebChat replies on WebChat unless an explicit delivery route is set; it preserves existing external route state, but does not add the requested opt-in mirroring feature or config key.

## What This Changes

Request an opt-in session config that mirrors WebChat or Control UI replies to the session's last established external channel.

## Best Possible Solution

Add a documented default-false session-level option that mirrors only complete WebChat replies through the existing external `routeReply` path when a valid persisted external route exists.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: seed a session with external `lastChannel`/`lastTo`, send via a WebChat `chat.send` client, and current main resolves the reply as internal WebChat with no external delivery route.

## Solution Assessment

Yes, with maintainer sign-off on the public config shape. A default-off option is the narrow maintainable direction because it avoids restoring implicit global mirroring while reusing persisted route state.

## Repair Work Prompt

Implement the optional WebChat-to-last-external-channel mirroring feature for https://github.com/openclaw/openclaw/issues/44302. Add a default-false session config key, preferably `session.mirrorWebchatToLastExternalChannel` unless maintainers choose a different name, and wire schema/types/help/generated config docs. Keep normal WebChat delivery unchanged, then mirror only complete assistant replies through the existing external routeReply path when the originating surface is WebChat/internal, the option is enabled, and the session has a valid persisted external route (`deliveryContext`/`lastChannel`/`lastTo`). Do not restore implicit mirroring, do not mirror without an established external route, do not mirror external-origin turns, do not mirror partial streaming/status/typing events, and log routeReply `{ ok: false }` failures without blocking WebChat delivery. Update docs and focused tests; stop for maintainer review if the config key or routing policy needs broader product direction.
