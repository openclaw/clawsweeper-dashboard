## Summary

## What This Changes

The issue asks for top-level `mcp.servers.*.env.*` and `mcp.servers.*.headers.*` to accept SecretRef objects instead of only scalar values.

## Best Possible Solution

Land a reviewed version of https://github.com/openclaw/openclaw/pull/69417 or a narrow replacement that supports structured SecretRefs for top-level MCP env/headers while preserving shipped literal behavior and updating policy, audit, docs, and tests.

## Reproduction Assessment

Yes, at source level. Current main’s schema/types accept only scalar MCP env/header values, and runtime conversion drops object values such as SecretRefs before launch.

## Solution Assessment

Yes directionally, but the best path is maintainer review of the existing implementation PR or a narrow replacement because compatibility and security-policy semantics remain unsettled.
