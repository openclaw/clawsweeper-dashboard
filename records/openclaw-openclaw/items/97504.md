## Label Justifications

- `P1`: Misclassified provider-side AbortErrors during compaction can silently degrade or lose useful conversation context in real long-running agent sessions.
- `merge-risk: 🚨 session-state`: Merging changes whether compaction retries, falls back, or propagates errors, which can alter stored summaries and transcript continuity.

## Summary

Keep open: this is a focused inner-compaction bug fix that is not already on current main, and the source path supports the reported provider-side AbortError misclassification. I found no blocking patch defect, but the retry/fallback behavior changes compaction session-state semantics and should get normal maintainer review before merge.

## What This Changes

The PR changes compaction summarization and compaction-safeguard provider fallback to retry or fall back on provider-side AbortError when the caller signal is not aborted, with regression tests.

## Best Possible Solution

Land this after maintainer review accepts the bounded retry/fallback semantics and exact-head checks stay green; keep https://github.com/openclaw/openclaw/issues/92043 and https://github.com/openclaw/openclaw/pull/92899 separate for the aggregate timeout partial-progress problem.

## Reproduction Assessment

Yes at source level: current main's inner compaction paths treat AbortError as terminal regardless of caller signal state, and the PR adds focused tests for provider-side AbortError recovery versus caller-aborted propagation. I did not execute tests in this read-only review.

## Solution Assessment

Yes, likely the best narrow fix: it reuses the AbortSignal already threaded through compaction, matches the merged outer model-fallback pattern, and avoids new config or API surface. The main remaining proof gap is live provider-disconnect evidence, not a different code shape.
