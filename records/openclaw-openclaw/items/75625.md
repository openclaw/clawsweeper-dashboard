## Summary

Keep open for maintainer review. Current main still has the media provider-inventory visibility bypass, and this PR is a narrow fix with focused tests and no discrete correctness finding; the remaining decision is whether maintainers want this stricter behavior that suppresses inventory even in full verbose mode, or the competing #75550 behavior that preserves full verbose output.

## What This Changes

This PR changes the shared agent media tool-result handler to suppress non-error image_generate/video_generate provider inventory results with details.providers from visible output even when full tool output is enabled, adds hidden/full-mode regression tests, and adds a changelog entry.

## Best Possible Solution

Keep one canonical fix for #75166: land either this stricter suppression or #75550 after deciding the intended full verbose/debug behavior, while preserving structured tool results and generated media delivery and keeping the handler-level image/video regression tests plus changelog coverage.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: pass an image_generate or video_generate action=list result with details.providers through handleToolExecutionEnd while shouldEmitToolOutput is false, and the current handler routes it to emitToolOutput through shouldEmitCompactToolOutput.

## Solution Assessment

Unclear pending maintainer policy. The patch is narrow and security-harder for shared chat surfaces, but it intentionally hides provider inventory even in full verbose mode while #75550 preserves that debug output, so maintainers should choose the desired operator behavior.
