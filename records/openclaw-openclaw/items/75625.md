## Summary

Keep open for maintainer review. Current main still has the provider-inventory compact-output bypass, and this PR plausibly fixes that exposure, but it intentionally changes `/verbose full` semantics and competes with the narrower open #75550 approach that preserves full verbose inventory visibility.

## What This Changes

This PR suppresses image/video provider inventory list output in the agent media handler even when full tool output is enabled, updates tests/changelog, and includes Matrix, Slack dependency, CLI, pairing, and gateway test/typing CI repairs.

## Best Possible Solution

Choose one canonical fix for #75166, keep structured provider details available to the agent context, keep generated media delivery unchanged, and make the `/verbose full` inventory policy explicit in code/tests/docs or by selecting the narrower #75550 behavior.

## Solution Assessment

Unclear. Suppressing the hidden-output bypass is the right class of fix, but suppressing explicit full verbose output and bundling unrelated CI repairs require maintainer policy and branch-selection judgment.
