## Summary

Keep this PR open. Current main still contains the media provider-inventory visibility bypass, and this PR is a focused hardening patch with regression coverage; the remaining maintainer decision is whether provider inventories should also be hidden from full verbose tool-output mode, especially because #75550 proposes the narrower hidden-output-only behavior.

## What This Changes

The PR changes agent media tool-result emission so `image_generate` and `video_generate` list results with `details.providers` are not emitted as visible tool output, adds hidden/full-verbose regression tests for both tools, and records the fix in the changelog.

## Best Possible Solution

Land one canonical fix for #75166 that keeps provider inventories internal on shared chat surfaces by default, preserves structured tool results for the agent and generated media delivery, and either documents full suppression or provides an explicit direct/admin path for operator-visible inventory.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: feed `handleToolExecutionEnd` a successful `image_generate` or `video_generate` list result with text content and `details.providers` while `shouldEmitToolOutput()` is false; current main routes it through `emitToolOutput`.

## Solution Assessment

Unclear. Removing the default compact provider-inventory bypass is the narrow maintainable fix, but suppressing full verbose output is a product/security policy choice because `/verbose full` is documented as forwarding tool outputs and #75550 keeps that explicit path visible.
