## Summary

Keep open: current main still does not provide gateway-wide macOS idle-sleep prevention, but this PR is not merge-ready because it changes default power behavior, lacks real macOS runtime proof, and still has an async spawn-error defect acknowledged in review.

## What This Changes

The PR adds a macOS-only gateway run-loop helper that starts `caffeinate -i -w <gateway pid>` and kills it when the gateway exits.

## Best Possible Solution

If maintainers want default macOS keep-awake behavior, rebase it onto the current gateway lifecycle code with explicit process-launch policy, async spawn-error handling, focused coverage, and redacted real macOS gateway proof.

## Reproduction Assessment

Yes for the blocking PR defect: the diff spawns `caffeinate` without an async `error` listener, and a Node probe showed missing executables report through the child `error` event after `spawn` returns. No high-confidence real macOS proof was supplied for the idle-sleep benefit itself.

## Solution Assessment

No, not as submitted. A keep-awake helper may be viable, but this version needs product approval for default power behavior, hardened spawn handling, focused tests, and real macOS proof before merge.
