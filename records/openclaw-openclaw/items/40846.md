## Summary

Keep open: current main does not implement gateway-wide macOS keep-awake behavior, and the merged launchd `ProcessType=Interactive` mitigation only covers LaunchAgent/App Nap behavior. This PR still needs maintainer approval for default power behavior, async spawn-error handling, and real macOS proof.

## What This Changes

The PR adds a macOS-only gateway run-loop helper that spawns `caffeinate -i -w <gateway pid>` and stops it when the gateway exits.

## Best Possible Solution

If maintainers want gateway-wide macOS keep-awake behavior, rebase it onto the current gateway lifecycle with explicit launch policy, async spawn-error handling, focused coverage, and redacted real macOS proof.

## Reproduction Assessment

Yes for the blocking PR defect: the diff spawns `caffeinate` without an async `error` listener, and a focused Node probe shows missing executables report through the child `error` event after `spawn()` returns. No high-confidence real macOS proof was supplied for the idle-sleep benefit itself.

## Solution Assessment

No, not as submitted. A keep-awake helper may be viable, but this version needs product approval for default power behavior, hardened spawn handling, focused tests, and real macOS proof before merge.
