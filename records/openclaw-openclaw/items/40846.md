## Summary

Keep open: current main does not implement gateway-owned macOS keep-awake behavior, but this branch is not merge-ready because the helper has an unhandled child-process error path, the branch conflicts with the refactored gateway run loop, and the contributor still needs real macOS proof.

## What This Changes

The branch adds a macOS-only `caffeinate -i -w <gateway pid>` child process in `src/cli/gateway-cli/run-loop.ts` and kills it when the gateway loop exits.

## Best Possible Solution

If maintainers want gateway-owned macOS keep-awake behavior, land a rebased implementation that hardens the child-process lifecycle, covers the helper with focused tests, makes the power-policy choice explicit, and includes redacted real macOS proof; otherwise continue the operator-helper direction in https://github.com/openclaw/openclaw/pull/61464.

## Reproduction Assessment

Yes for the blocking PR defect: source inspection shows no child `error` listener, and a Node v24.15.0 probe reproduced the async `ENOENT` unhandled-error crash behavior. The claimed macOS idle-sleep benefit itself still lacks high-confidence real-device proof.

## Solution Assessment

No, not as submitted. A `caffeinate` helper may be maintainable if maintainers approve the policy, but this branch needs rebase, hardened spawn handling, focused tests, and real macOS proof first.
