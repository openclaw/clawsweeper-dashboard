## Summary

Keep this PR open: the requested gateway keep-awake behavior is not on current main, but the submitted patch is not ready because it changes default macOS power behavior, lacks real macOS runtime proof, and has a concrete child-process error-handling defect.

## What This Changes

The PR adds a macOS-only gateway run-loop helper that starts `caffeinate -i -w <gateway pid>` and kills it when the gateway exits.

## Best Possible Solution

If maintainers want default keep-awake behavior, land a rebased macOS implementation with explicit process-launch policy, async error handling, focused coverage, and redacted real macOS gateway proof.

## Reproduction Assessment

Yes for the blocking PR defect: the diff plus a local Node probe show failed `spawn()` calls emit an async `error` event outside the PR's `try/catch`. The long-duration macOS idle-sleep benefit itself is not proven by inspectable PR evidence.

## Solution Assessment

No, not as submitted. A keep-awake helper may be viable, but this version needs maintainer approval for default macOS power behavior, hardened spawn handling, focused tests, and real macOS proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Handle caffeinate spawn errors:** `src/cli/gateway-cli/run-loop.ts:36-40`
  - body: When `caffeinate` cannot be executed, `child_process.spawn()` emits an asynchronous `error` event instead of throwing, so this `try/catch` will not handle the failure. Attach an `error` listener or otherwise handle ENOENT/EACCES so an optional keep-awake helper cannot crash gateway startup.
  - confidence: 0.93

## Security Review

Status: needs_attention

Summary: The patch adds a gateway-managed child process launch through PATH without async spawn-error handling.

Concerns:

- **[low] Constrain caffeinate execution:** `src/cli/gateway-cli/run-loop.ts:36`
  - body: `spawn("caffeinate", ...)` resolves through PATH and currently lacks an `error` listener, so a bad or missing executable can affect gateway startup. Prefer a clear macOS system-path or resolution policy, keep fixed arguments, and handle ENOENT/EACCES without crashing.
  - confidence: 0.78
