## Summary

Keep open: current main still does not implement default gateway `caffeinate`, but this PR is not mergeable as submitted because it needs maintainer approval for the default macOS power-policy change, real macOS runtime proof, a rebase, and a spawn-error fix.

## What This Changes

The PR adds a macOS-only gateway run-loop helper that starts `caffeinate -i -w <gateway pid>` and cleans it up when the gateway exits.

## Best Possible Solution

If maintainers want default macOS keep-awake behavior, land a rebased implementation with explicit process-launch policy, async spawn-error handling, focused coverage, and redacted real macOS gateway proof.

## Reproduction Assessment

Yes for the blocking PR defect: the diff and author acknowledgment show the spawned helper has no async `error` listener. No high-confidence proof was supplied for the long-duration macOS idle-sleep benefit itself.

## Solution Assessment

No, not as submitted. A keep-awake helper may be viable, but this version needs product approval for default power behavior, hardened spawn handling, focused tests, and real macOS proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Handle caffeinate spawn errors:** `src/cli/gateway-cli/run-loop.ts:34-40`
  - body: When `caffeinate` cannot be executed, `child_process.spawn()` reports the failure through the child process `error` event rather than this surrounding `try/catch`. Add an `error` listener so an optional keep-awake helper cannot crash gateway startup.
  - confidence: 0.93

## Security Review

Status: needs_attention
