## Summary

Keep open: current main still lacks macOS gateway keep-awake behavior, and the related items do not supersede this core gateway change. The PR is useful but needs changes before merge because the new child-process path can crash on spawn failure and the user-visible behavior lacks changelog coverage.

## What This Changes

The PR adds a macOS-only `caffeinate -i -w <pid>` helper to `runGatewayLoop` and cleans it up when the gateway exits.

## Best Possible Solution

Land a macOS-only keep-awake implementation with explicit gateway lifecycle ownership, resilient child-process handling, focused coverage, changelog documentation, and a maintainer-approved default policy.

## Reproduction Assessment

Yes for the blocking PR defect: a local Node check reproduced that failed child-process spawn emits an unhandled `error` event after `spawn()` returns. The requested macOS idle-sleep improvement itself was not live-verified here.

## Solution Assessment

No, not as submitted. A macOS keep-awake helper may be a reasonable direction, but this diff needs error handling, tests, changelog coverage, safer executable resolution or documented PATH reliance, and a rebase before it is mergeable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Handle caffeinate spawn errors:** `src/cli/gateway-cli/run-loop.ts:33-37`
  - body: Node emits a ChildProcess `error` event when the executable cannot be spawned, and an EventEmitter `error` without a listener can crash the process. This code only wraps `spawn()` in `try/catch`, so a missing or restricted `caffeinate` can take down gateway startup instead of degrading gracefully; attach an `error` listener before `unref()` and cleanup return.
  - confidence: 0.91
- **[P2] Add a changelog entry for the gateway power change:** `src/cli/gateway-cli/run-loop.ts:61`
  - body: This is a user-visible feature: macOS gateway runs now prevent idle sleep. The PR only changes `run-loop.ts`, but OpenClaw policy requires user-facing feat/fix/perf changes to update `CHANGELOG.md`, so the release notes would omit this behavior.
  - confidence: 0.84
