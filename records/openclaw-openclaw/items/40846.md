## Summary

Keep open: current main still does not implement gateway runtime idle-sleep prevention, and the merged launchd/App Nap work only partially overlaps. The submitted branch still needs maintainer approval for the default macOS power policy, hardened child-process handling, and contributor-supplied real behavior proof before merge.

## What This Changes

The PR adds a macOS-only `caffeinate -i -w <pid>` child process to the gateway run loop and kills it when the loop exits.

## Best Possible Solution

If maintainers want default keep-awake behavior, land a hardened macOS implementation with trusted `caffeinate` resolution, async spawn-error handling, focused run-loop coverage, and real macOS proof; otherwise keep this as an explicit operator helper/workaround instead of core default behavior.

## Reproduction Assessment

Yes for the blocking PR defect: the diff plus a Node 24 probe show failed spawns emit an async `error` event outside the PR's `try/catch`. The long-duration macOS idle-sleep benefit itself is not proven by the PR evidence.

## Solution Assessment

No, not as submitted. A keep-awake helper may be viable, but this version needs maintainer approval for the default power policy, child-process hardening, focused tests, and real macOS proof before merge.
