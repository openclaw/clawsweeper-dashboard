## Summary

Keep open: current main still does not implement gateway-wide macOS idle-sleep prevention, but this PR is not mergeable as submitted because it lacks real macOS behavior proof, has an unhandled child-process spawn failure, and needs maintainer approval for default power-behavior policy.

## What This Changes

The PR adds a macOS-only gateway run-loop helper that spawns `caffeinate -i -w <gateway pid>` and stops it when the gateway exits.

## Best Possible Solution

If maintainers want gateway-owned macOS keep-awake behavior, rebase onto the current run loop with explicit launch policy, async spawn-error handling, focused coverage, and redacted real macOS proof.

## Reproduction Assessment

Yes for the blocking PR defect: the PR diff lacks an async `error` listener, and Node's child-process contract plus a focused probe show missing executables report through the child `error` event after `spawn()` returns. No high-confidence real macOS proof was supplied for the idle-sleep benefit itself.

## Solution Assessment

No, not as submitted. A `caffeinate` helper may be a viable implementation, but this version needs product approval for default power behavior, hardened spawn handling, a rebase, and real macOS proof before merge.
