## Summary

Keep open: current main still does not implement gateway-wide macOS idle-sleep prevention, and this PR is conflicted with the refactored gateway run loop, lacks required real macOS proof, and has a confirmed async spawn-error defect.

## What This Changes

The PR adds a macOS-only gateway run-loop helper that starts `caffeinate -i -w <pid>` while the gateway is running and kills it on exit.

## Best Possible Solution

If maintainers want gateway-owned macOS keep-awake behavior, rebase it onto the current run loop, harden the child-process lifecycle, add focused coverage, and require redacted real macOS proof before merge.

## Reproduction Assessment

Yes for the blocking PR defect: the diff lacks a child `error` listener, and a focused Node probe shows missing executables emit `error` after `spawn()` returns. No high-confidence real macOS proof was supplied for the idle-sleep benefit itself.

## Solution Assessment

No, not as submitted. A `caffeinate` helper may be viable if maintainers approve the policy, but this version needs rebase, async spawn-error handling, tests, and real macOS proof before it is the maintainable fix.

## Review Findings
