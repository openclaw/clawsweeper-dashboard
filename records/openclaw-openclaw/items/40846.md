## Summary

Current main only has a narrower launchd `ProcessType=Interactive` App Nap mitigation, so this gateway-wide macOS `caffeinate` change is not obsolete. This PR still needs maintainer product approval for default power behavior, contributor-supplied real macOS proof, and async spawn-error handling before merge.

## What This Changes

The PR adds a macOS-only gateway run-loop helper that spawns `caffeinate -i -w <gateway pid>` and kills it when the gateway exits.

## Best Possible Solution

If maintainers want gateway-wide macOS keep-awake behavior, rebase it onto the current gateway lifecycle with explicit launch policy, async spawn-error handling, focused coverage, and redacted real macOS proof.

## Reproduction Assessment

Yes for the blocking PR defect: the diff spawns `caffeinate` without an async `error` listener, and a focused Node probe confirms missing executables report through the child `error` event after `spawn()` returns. No high-confidence real macOS proof was supplied for the idle-sleep benefit itself.

## Solution Assessment

No, not as submitted. A keep-awake helper may be viable, but this version needs product approval for default power behavior, hardened spawn handling, focused tests, and real macOS proof before merge.
