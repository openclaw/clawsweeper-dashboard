## Summary

Keep open: current main still lacks the macOS gateway keep-awake behavior, and the PR remains useful but not mergeable because its child-process handling, changelog coverage, and command-resolution hardening are incomplete.

## What This Changes

This PR adds a macOS-only `caffeinate -i -w <pid>` helper in `runGatewayLoop` to prevent idle sleep while the gateway process is running.

## Best Possible Solution

Land a maintainer-approved macOS keep-awake path with owned gateway lifecycle cleanup, robust spawn error handling, constrained executable resolution, focused tests, and a changelog entry.

## Reproduction Assessment

Yes for the blocking PR defect: a Node 24.14.1 check reproduced that a failed child-process spawn emits an unhandled `error` event after `spawn()` returns when no listener is attached. The macOS idle-sleep improvement itself was not live-verified here.

## Solution Assessment

No, not as submitted. A macOS keep-awake helper may be a reasonable direction, but this diff needs child-process error handling, executable-resolution hardening or documentation, focused tests, changelog coverage, and maintainer approval for the default power behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Handle caffeinate spawn errors:** `src/cli/gateway-cli/run-loop.ts:33-37`
  - body: Node emits a ChildProcess `error` event when the executable cannot be spawned, and an EventEmitter `error` without a listener can crash the gateway. This code only wraps `spawn()` in `try/catch`, so a missing or restricted `caffeinate` can take down startup instead of degrading; attach an `error` listener before `unref()`.
  - confidence: 0.91
- **[P2] Add a changelog entry for the gateway power change:** `src/cli/gateway-cli/run-loop.ts:61`
  - body: This is a user-visible feature: macOS gateway runs now prevent idle sleep. The PR only changes `run-loop.ts`, but OpenClaw policy requires user-facing feat/fix/perf changes to update `CHANGELOG.md`, so release notes would omit the behavior.
  - confidence: 0.84
