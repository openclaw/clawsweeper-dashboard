## Summary

Keep open: the requested gateway keep-awake behavior is not on current main, and this PR still needs maintainer product approval, a rebase onto the refactored run loop, async spawn-error handling, and real macOS proof before merge.

## What This Changes

Adds a macOS-only gateway run-loop helper that starts `caffeinate -i -w <pid>` and kills it when the gateway loop exits.

## Best Possible Solution

If maintainers want gateway-owned macOS keep-awake behavior, implement it with an explicit approved launch policy, hardened child-process lifecycle handling, focused tests, and redacted real macOS proof; otherwise keep host keep-awake helpers outside the default gateway run loop.

## Reproduction Assessment

Yes for the blocking PR defect: the diff lacks a child `error` listener, and a focused Node probe shows missing executables emit `error` after `spawn()` returns. No high-confidence real macOS proof was supplied for the idle-sleep benefit itself.

## Solution Assessment

No, not as submitted. A `caffeinate` helper may be viable if approved, but this version needs rebase, async spawn-error handling, and real macOS proof before it is the maintainable fix.
