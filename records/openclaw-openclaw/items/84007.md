## Summary

Keep open: this is a focused bug-fix PR with strong proof for the stored/configured thinking path, but the branch still uses only persisted session `thinkingLevel` for caller inheritance, so default-derived caller thinking can remain lost.

## What This Changes

The PR changes native `sessions_spawn(runtime="subagent")` thinking resolution to use explicit spawn thinking, requester/target/global subagent defaults, then the requester session's persisted thinking level, with focused tests.

## Best Possible Solution

Derive the caller fallback from the same effective runtime thinking value used by normal agent runs, while preserving explicit spawn, requester subagent, target subagent, and global subagent precedence with regression coverage for stored and default-derived callers.

## Reproduction Assessment

Yes. Source inspection shows current main's native subagent resolver lacks caller inheritance, and the PR still reads only persisted `thinkingLevel` while normal agent runs can derive effective thinking from config or provider/model defaults.

## Solution Assessment

No. The PR is on the right path for stored and subagent-configured thinking, but the maintainable fix should pass or resolve the caller's effective runtime thinking rather than re-reading only one persisted field.
