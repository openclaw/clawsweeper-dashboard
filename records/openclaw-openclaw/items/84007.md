## Summary

Keep open: this PR fixes much of the linked subagent thinking inheritance bug and has useful live proof, but the latest effective-default fallback still ignores the caller session's selected/runtime model when no `thinkingLevel` is stored, so some omitted-thinking children can inherit the wrong provider/model default.

## What This Changes

The branch threads explicit, requester, target, global, and caller thinking fallback through native subagent spawns and adds resolver/spawn regression tests.

## Best Possible Solution

Derive the caller fallback from the same persisted selected/runtime model and thinking chain used by normal agent runs, while preserving the PR's explicit, requester, target, and global subagent precedence.

## Reproduction Assessment

Yes. Source inspection shows current `main` lacks caller thinking fallback, and the PR head still has a clear source-level repro for callers with a selected/runtime model but no stored `thinkingLevel`.

## Solution Assessment

No, not yet. The direction is right, but the caller fallback should use the parent session's persisted selected/runtime model before using the configured default model.
