## Summary

Keep open: the latest head has good proof and fixes stored/configured thinking inheritance, but it still falls back to the requester agent's configured model instead of the requester session's persisted selected/runtime model when no `thinkingLevel` is stored.

## What This Changes

The PR threads requester, target, global, and caller thinking fallback through native subagent spawn planning and adds resolver/spawn regression tests.

## Best Possible Solution

Keep the PR approach, but derive the caller fallback from the requester session's persisted selected/runtime model before configured model defaults while preserving the documented explicit and subagent override precedence.

## Reproduction Assessment

Yes. Current main source lacks caller thinking fallback, and PR-head source still has a clear edge-case repro: a requester session entry with selected/runtime model metadata but no stored `thinkingLevel` falls through to configured model defaults.

## Solution Assessment

No. The solution is close, but it should use the requester session's persisted selected/runtime model before configured model fallback so child thinking matches the caller's effective model behavior.
