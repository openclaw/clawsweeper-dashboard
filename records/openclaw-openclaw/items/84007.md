## Summary

Keep open: the PR fixes a real source-reproducible subagent thinking inheritance bug and includes sufficient live proof, but one P2 correctness issue remains because the new caller fallback still skips persisted session model metadata.

## What This Changes

The PR changes native subagent spawn planning and tests so omitted child thinking can fall back through explicit/configured subagent defaults and caller thinking state.

## Best Possible Solution

Land after caller thinking fallback derives from stored thinkingLevel, then persisted selected/runtime model metadata, then requester/global/model defaults, with a focused regression test for that order.

## Reproduction Assessment

Yes. Source inspection on current main shows caller thinking is not passed into subagent planning, and the linked issue provides transcript evidence where omitted thinking falls to low while explicit thinking works; I did not run tests because this review is read-only.

## Solution Assessment

No. The fix direction is right, but the implementation should use the caller session's persisted selected/runtime model metadata before falling back to configured default-model policy.
