## Summary

Keep open: the reported bug and fix direction are valid and the supplied proof is strong, but the current patch still ignores persisted selected/runtime model metadata when deriving the caller fallback, so some sessions can inherit the wrong model-specific thinking default.

## What This Changes

The PR updates native subagent spawn planning so omitted thinking falls back through requester/target/global subagent config and requester session/default thinking, with focused resolver and spawn tests.

## Best Possible Solution

Land after caller fallback derives thinking from stored thinkingLevel, then persisted selected/runtime model metadata, then requester/global/model defaults, with regression coverage for that order.

## Reproduction Assessment

Yes. Current main source does not pass caller thinking into subagent planning, and the linked issue includes transcript evidence where children start at low while explicit thinking is honored.

## Solution Assessment

No. The fix direction is right, but the caller fallback should use stored thinkingLevel and then persisted selected/runtime model metadata before configured defaults; otherwise selected-model sessions remain wrong.
