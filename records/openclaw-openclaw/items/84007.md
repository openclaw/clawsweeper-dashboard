## Summary

Keep open: this is a focused fix for a source-reproducible subagent thinking inheritance bug, but the current patch still turns caller-session inheritance into an explicit child `thinking` override, which can break mixed-provider targets instead of letting normal target-model support handling remap it.

## What This Changes

The PR adds requester-agent and caller-session thinking fallback inputs to native subagent spawn planning, persists inherited child `thinkingLevel`, and expands subagent thinking tests.

## Best Possible Solution

Carry a source-aware thinking plan: explicit spawn and configured subagent defaults can remain explicit, while caller-session inheritance should be persisted onto the child session without being forwarded as an explicit override, with mixed-provider unsupported-level coverage.

## Reproduction Assessment

Yes, at source level. Current main's resolver cannot receive requester or caller thinking even though the docs promise caller inheritance, and the linked issue includes concrete child transcripts showing omitted-thinking subagents starting at `low`.

## Solution Assessment

No, not quite. The inheritance direction is right, but caller-session inheritance should not be collapsed into an explicit `thinking` override because downstream provider/model validation treats explicit overrides more strictly than persisted/default thinking.
