## What This Changes

The PR changes native subagent spawn planning so child sessions inherit requester-agent subagent defaults and persisted requester session thinking when no higher-precedence thinking is supplied.

## Best Possible Solution

Land the inheritance fix once maintainers accept the documented default behavior change, keeping explicit spawn thinking and configured subagent thinking as the operator override path.

## Solution Assessment

Yes, if maintainers accept the default behavior change. Carrying requester config and persisted caller thinking into the existing resolver is the narrow maintainable fix, and the PR adds coverage for explicit, requester, target, global, caller, and `off` precedence.
