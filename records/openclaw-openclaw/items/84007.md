## Summary

## What This Changes

The PR changes native subagent spawn planning so child sessions can inherit requester thinking defaults when explicit spawn thinking and configured subagent defaults are absent.

## Best Possible Solution

Keep the inheritance fix, but align the final requester fallback with current agent execution by applying requester `thinkingDefault` before selected-model/model default fallback and add the focused regression for that case.
