## Summary

Keep open: the PR fixes the central stored-thinking/default inheritance path and includes sufficient live/log proof, but the latest head still misses caller selected/runtime model metadata when no `thinkingLevel` is stored, so a child can inherit thinking from config/default model policy instead of the caller's effective model.

## What This Changes

The PR adds requester/default thinking fallback to native subagent spawn planning, persists inherited child `thinkingLevel`, and extends subagent spawn resolver/seam tests.

## Best Possible Solution

Resolve caller effective thinking from stored `thinkingLevel`, then persisted selected/runtime model fields with the existing model resolver, then agent/global/default policy, while keeping the explicit/configured/`off` precedence tests.

Yes, source-reproducible: current main does not pass caller thinking into subagent planning, while the linked issue provides child transcript evidence starting at low despite configured/high caller state. I did not run local tests because this review is read-only.

## Solution Assessment

No, not yet: the inheritance approach is right, but the latest fallback should use persisted selected/runtime model metadata before default model policy. That is the narrowest maintainable repair because the repo already has a resolver for those session fields.
