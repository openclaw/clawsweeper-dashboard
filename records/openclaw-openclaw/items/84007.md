## Summary

This PR should stay open for normal maintainer review: it is a focused, source-supported fix for the linked session-state bug, and I found no blocking correctness or security findings. The remaining decision is whether maintainers accept the documented default-inheritance behavior change for existing unconfigured native subagent spawns.

## What This Changes

The PR changes native subagent spawn planning so requester-agent subagent defaults and the caller session thinking level can flow into child sessions when no explicit or higher-precedence subagent thinking is set.

## Best Possible Solution

Land the docs-aligned inheritance fix if maintainers accept the default behavior change, while preserving explicit sessions_spawn.thinking and configured subagent thinking as the higher-precedence override paths.

## Reproduction Assessment

Yes. Current docs promise caller thinking inheritance, while current main's resolver and spawn path cannot receive requester-agent config or persisted caller thinking; the linked report at https://github.com/openclaw/openclaw/issues/55790 includes concrete failing transcripts.

## Solution Assessment

Yes, assuming maintainers accept the default behavior change. Threading requester config and caller session thinking into the existing spawn planner is narrower than adding a parallel resolver or new config surface, and explicit/configured precedence is preserved.
