## Summary

Current main and released source already normalize MiniMax usage as remaining quota and render the display as actual percent left, so the reported inverted label is already fixed.

## What This Changes

The issue asks for MiniMax usage/status output to stop showing a consumed percentage as `% left` and instead show either used percentage or actual remaining percentage.

## Best Possible Solution

Keep the current normalized `% left` behavior; if a running install still shows the old line, treat it as an update/runtime-state report with fresh `openclaw status --usage --json` output.

## Reproduction Assessment

No; current main and v2026.5.5 source invert MiniMax remaining fields to internal `usedPercent` and format `% left` as `100 - usedPercent`, so the provided 98-remaining/2-used example would render as 98% left.

## Solution Assessment

Yes; normalizing provider snapshots to `usedPercent` internally and deriving `% left` at display boundaries is the narrowest maintainable fix and preserves the existing provider usage contract.
