## Summary

Keep open: current `main` still has the source-level false-warning path, and the linked open PR with closing syntax should be reviewed or replaced before this issue is closed.

## What This Changes

The issue asks Doctor to stop reporting a resolvable `gateway.auth.token` SecretRef as unavailable.

## Best Possible Solution

Review and consolidate the open PRs into one Doctor fix that resolves safe Gateway token SecretRefs, treats exec refs as explicitly unverified or opt-in, and adds focused regression coverage.

## Reproduction Assessment

Yes, at source level. Configure local token auth with `gateway.auth.token` as a resolvable exec SecretRef and run `openclaw doctor`; current source detects the SecretRef but `resolveGatewayAuth()` ignores SecretRef object values and reaches the unavailable warning branch.

## Solution Assessment

Unclear. Reusing the existing SecretRef-aware Gateway token resolver is the right direction, but Doctor needs a policy-aware exec path or an explicit unverified diagnostic rather than calling skipped exec refs unavailable.
