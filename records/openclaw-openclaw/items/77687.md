## What This Changes

The issue requests that `openclaw doctor` stop reporting a configured Gateway auth token SecretRef as unavailable when that SecretRef resolves.

## Best Possible Solution

Review and consolidate the linked PRs into one doctor fix that resolves safe Gateway token SecretRefs, treats exec refs as explicitly unverified or opt-in, and adds focused regression coverage.

## Reproduction Assessment

Yes. Configure local token auth with `gateway.auth.token` as a resolvable exec SecretRef and run `openclaw doctor`; current source detects the SecretRef but `resolveGatewayAuth()` ignores SecretRef object values and reaches the unavailable warning branch.

## Solution Assessment

Unclear. Reusing the existing SecretRef-aware Gateway token resolver is the right direction, but doctor needs a policy-aware exec path or an explicit unverified diagnostic rather than calling skipped exec refs unavailable.
