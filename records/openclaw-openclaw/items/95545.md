## Summary

This PR is a focused fix for a real Skill Workshop update-overwrite footgun, but it is not merge-ready yet: the new apply guard can reject valid complete skills, still misses empty or small destructive replacements, and lacks convincing real tool-path proof.

## What This Changes

Adds Skill Workshop apply-time validation for update proposals and regression tests for delta-note and section-dropping drafts before writing live SKILL.md.

## Best Possible Solution

Keep the apply-time guard at the service write choke point, but make detection structural enough to reject empty and delta drafts without blocking complete skills, then add focused regression coverage and real tool-path proof.

## Solution Assessment
