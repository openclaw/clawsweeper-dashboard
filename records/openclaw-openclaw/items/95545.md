## Summary

Keep open: the apply-time guard is the right layer and the PR now includes redacted real apply-path proof, but the guard still fail-closes a valid complete update when an existing H1-bearing skill is updated to an H1-less SKILL.md body.

## What This Changes

The PR adds Skill Workshop apply-time validation and regression tests to reject delta-style update drafts, empty replacement bodies, and heading identity swaps before overwriting a live SKILL.md.

## Best Possible Solution

Keep the delta-note protection, but allow any non-empty complete SKILL.md body and compare H1 identity only when both previous and proposed bodies contain H1s.

## Reproduction Assessment

Yes. Source inspection shows current main writes stripped update proposal content directly, and PR-head source shows the remaining compatibility failure for an H1-bearing previous skill updated to a complete H1-less body.

## Solution Assessment

No. The apply-time guard is the right layer, but the best fix should preserve the SKILL.md format contract by only comparing H1 identity when both bodies provide H1 headings.
