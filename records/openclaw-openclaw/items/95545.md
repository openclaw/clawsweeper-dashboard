## Summary

Keep this PR open, but it should not merge as-is: the central data-loss footgun is real, while the new section-shrink heuristic blocks valid complete Skill Workshop rewrites and the external-PR real behavior proof is not sufficient yet.

## What This Changes

Adds an apply-time validator for Skill Workshop update proposals and service tests for delta-note and section-dropping update drafts.

## Best Possible Solution

Keep the delta/proposal-note protection, but make the validator target that invariant narrowly and allow complete replacement SKILL.md rewrites unless maintainers add an explicit destructive-rewrite override contract.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: on current main, an update proposal body is stripped with stripProposalFrontmatterForSkill() and written to the live skill without checking whether it is a delta note. I did not run tests because this review is read-only.

## Solution Assessment

No, not as written: the delta-note guard is the right layer, but the section-shrink heuristic is broader than the bug and blocks valid complete replacements. A narrower content-shape check, or an explicit destructive-rewrite override, is safer.
