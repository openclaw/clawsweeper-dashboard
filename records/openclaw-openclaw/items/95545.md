## Summary

The PR should stay open because it fixes a real Skill Workshop overwrite footgun, but the current implementation still adds a broader fail-closed H1 requirement that breaks documented valid skills; it also needs external real behavior proof before merge.

## What This Changes

The PR adds apply-time validation and tests for Skill Workshop update proposals so delta-style notes cannot be applied as full replacement `SKILL.md` content.

## Best Possible Solution

Keep the delta-note protection, but preserve the existing SKILL.md format contract by requiring a non-empty complete body and comparing H1 identity only when the relevant complete skill bodies actually have H1 headings.

## Reproduction Assessment

Yes. Source inspection shows current main strips proposal frontmatter and writes the result as the live SKILL.md without checking for delta-note content; the PR regression is also source-reproducible against the documented H1-less skill format.

## Solution Assessment

No, not yet. The apply-time guard is the right layer and narrower than changing proposal creation, but the current implementation overreaches by requiring every update body to have an H1 instead of preserving the existing SKILL.md contract.
