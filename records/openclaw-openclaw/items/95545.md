## Summary

Keep this PR open: the apply-time guard is the right layer for the Skill Workshop footgun, but the latest strong H1 marker still creates a compatibility false positive for complete skills whose preserved identity contains `update proposal`.

## What This Changes

The PR adds Skill Workshop update-apply validation so delta/proposal-note drafts are rejected before replacing an existing `SKILL.md`, with regression coverage for invalid drafts and valid complete update bodies.

## Best Possible Solution

Keep the apply-time guard, but make wrapper detection compare against the previous skill identity before rejecting strong H1 markers, and add a regression for a valid preserved H1 containing `update proposal`.

## Reproduction Assessment

Yes from source inspection: current main strips proposal frontmatter and writes the remaining update body directly to the live `SKILL.md`, so a delta-note-shaped update body can replace the actual skill procedure. The PR body also includes a redacted real apply-path reproduction of that failure class.

## Solution Assessment

No as currently written: the apply-time guard is the right layer, but the strong H1 marker should not reject a complete update that preserves the existing skill identity. The narrow best fix is to make wrapper detection identity-aware and add that regression test.
