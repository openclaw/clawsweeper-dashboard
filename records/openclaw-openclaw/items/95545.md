## Summary

Keep open: the PR is on the right shared apply-time boundary for Skill Workshop updates, but the latest head still adds a fail-closed H1 requirement that rejects valid complete SKILL.md replacements when an existing H1-bearing skill is intentionally rewritten without an H1.

## What This Changes

The PR adds Skill Workshop update-apply validation and regression tests to reject delta/proposal-note drafts before replacing an existing SKILL.md.

## Best Possible Solution

Keep the apply-time delta-note protection, but allow non-empty complete H1-less replacements and compare H1 identity only when both old and new bodies include H1s.

## Reproduction Assessment

Yes. Source inspection of PR head shows an H1-bearing existing skill updated to a complete H1-less body hits the new previousH1 && !proposedH1 throw, while current docs and loader code allow H1-less skill bodies.

## Solution Assessment

No. The apply-time guard is the right layer, but the H1 check should not make a decorative heading mandatory when frontmatter already carries the skill identity.
