## Summary

Keep open: the guard is useful and in the right apply-time layer, but the current patch still rejects a valid complete replacement when an H1-bearing skill is updated to an H1-less SKILL.md body.

## What This Changes

The PR adds Skill Workshop update-apply validation to reject delta-style drafts before overwriting a live SKILL.md.

## Best Possible Solution

Keep the delta-note protection, but allow non-empty complete SKILL.md bodies without a proposed H1 and compare H1 identity only when both bodies provide H1s.

## Reproduction Assessment

Yes. Source inspection of PR head shows an H1-bearing existing skill updated to a non-empty complete H1-less body throws at apply time, while docs and loader prove that H1-less SKILL.md bodies are valid.

## Solution Assessment

No. The apply-time guard is the right layer, but the current identity check should compare H1s only when both bodies have H1s and should not make a decorative heading mandatory.
