## Summary

Keep open: current main and latest release still retain the restart catch-up replay predicate without a schedule-activation/current-window boundary, and `cron.remove` still removes the job row without aborting an active run. The open schedule-activation PR is a relevant partial fix path, but it is still open/conflicting and does not cover cron delete, gateway stop, or emergency outbound-stop semantics.

## What This Changes

The issue reports WhatsApp cron jobs replaying outside configured hours after gateway reconnection and not stopping through available cron/gateway controls.

## Best Possible Solution

Review, repair, and land https://github.com/openclaw/openclaw/pull/92398 or an equivalent activation-boundary fix for restart replay, then define and document the supported operator stop contract across cron remove, task cancellation, gateway stop, and emergency outbound halt behavior.
