## Summary

Keep open. The PR targets a real false-positive launch-evidence path, but the current patch can still spawn the fallback while the foreground gateway owns the port, so the managed gateway can hit `EADDRINUSE` and exit before it ever becomes the persistent listener.

## What This Changes

This PR changes Windows Scheduled Task startup polling to snapshot listener PIDs before `/Run`, ignore pre-existing gateway listeners as launch evidence, and add a mocked startup-fallback regression test.

## Best Possible Solution

Make the Windows scheduled-task launch path produce or verify a real managed listener after the foreground gateway exits, then prove the end-to-end native Windows flow stays reachable after the foreground window closes.

## Reproduction Assessment

Yes for the source-level failure mode: the PR filters the existing foreground PID, then can spawn fallback while the same port remains occupied. No live Windows Task Scheduler reproduction was run in this read-only review.

## Solution Assessment

No. The patch fixes one evidence check, but the best fix must also handle the still-bound foreground port and prove the replacement gateway actually persists.
