## Summary

Keep open. Neither current main nor v2026.6.5 supports operator-defined domain/path deny rules or navigation warnings; the existing shared browser guard only enforces SSRF/private-network policy and hostname allow exceptions. The request is valid and vision-aligned, but matching scope, redirects, warning behavior, and the new public config contract require explicit product and security decisions.

## What This Changes

Add operator-configurable URL/domain/path blocking, with an optional warning mode, to browser navigation.

## Best Possible Solution

Add an operator-only deny policy to the existing shared browser navigation guard after maintainers define normalized host/path and redirect semantics; keep warnings structured, cover every existing navigation entry point, and do not ship a centrally curated default domain list.

## Reproduction Assessment

Not applicable: this issue requests a new operator-controlled browser policy rather than reporting behavior that violates an existing documented contract.

## Solution Assessment

No, not as an independent `browser.denylist` matcher or bundled site list. The narrowest maintainable solution is an approved extension of the existing shared navigation-policy evaluator so initial URLs, redirects, final URLs, and action-triggered navigation cannot drift.

## Risks / Open Questions

- URL-pattern semantics remain undefined for normalization, percent encoding, ports, query strings, fragments, redirects, and case sensitivity; ambiguous rules could be bypassed or block unintended destinations.
- The product scope must distinguish top-level navigation from popups, downloads, subresources, service workers, and other browser traffic so operators do not receive a misleading security guarantee.
- A bundled default list of bot-protected domains would require continuous policy ownership, could quickly become stale, and may confuse bot-detection avoidance with a security boundary.
- A warnlist would alter model-facing tool output and needs a structured, deterministic contract that does not encourage repeated retries or inject arbitrary operator text.
