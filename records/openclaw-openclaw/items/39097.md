Keep open. Current `main` and latest shipped `v2026.5.7` still expose only browser SSRF/private-network hostname policy, not an operator-configurable URL/domain/path denylist or warnlist, and the requested public browser policy needs maintainer product/security design before implementation.

The issue requests an operator-configurable browser URL/domain/path denylist, with optional warnlist behavior, so browser navigation can fail fast on known bot-blocked sites.
