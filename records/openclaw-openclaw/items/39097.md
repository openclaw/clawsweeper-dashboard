## Summary

Keep open. Current main still has browser SSRF/private-network policy only, with no operator URL/domain/path denylist or warnlist, and the requested behavior needs maintainer policy choices before implementation.

## What This Changes

Add operator-configurable browser URL/domain/path blocking, optionally warning, so navigation to known bot-blocking sites fails early with a clear message.

## Best Possible Solution

Approve a browser URL policy contract in the browser plugin/config layer, then thread it through config types, strict schema/help, resolved browser config, navigation preflight/result/redirect guards, docs, tests, and changelog.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a feature/config request. Static inspection of current main shows no browser.denylist, browser.blockedUrls, or browser.warnlist config or navigation guard behavior.

## Solution Assessment

Unclear. The browser config plus centralized navigation guard is the right layer, but maintainers still need to choose public key names, matching semantics, redirect behavior, warning behavior, and default-list ownership.
