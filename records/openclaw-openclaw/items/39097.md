## Summary

Keep open. Current main still has only SSRF/private-network browser navigation policy, and there is no operator-configurable URL/domain/path denylist or warnlist despite the prior ClawSweeper review reaching the same conclusion.

## What This Changes

Add a browser config option for operator-maintained blocked URL patterns, with an optional warnlist, so navigation fails early with a clear message on known-blocked sites.

## Best Possible Solution

Define a maintainer-approved browser URL policy primitive that flows through config types, strict schema/help, config resolution, all navigation guard call sites, docs, and focused tests.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a feature request. Static inspection of current main confirms the requested config fields and navigation policy behavior are absent.

## Solution Assessment

Unclear. A centralized browser config plus navigation-guard implementation is the right layer, but maintainers still need to choose public key names, matching semantics, redirect handling, warnlist behavior, and default-list policy.
