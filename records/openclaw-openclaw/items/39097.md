## Summary

Keep open. Current main and v2026.6.10 still provide browser SSRF/private-network controls and hostname allowlist exceptions, but not the requested browser URL/path denylist, warnlist, or curated bot-blocked-domain policy; the remaining work is a security-sensitive config/product decision, with https://github.com/openclaw/openclaw/issues/39685 only partially overlapping as a broader web-tool egress firewall request.

## What This Changes

The issue requests a browser navigation denylist or warnlist config for blocked domains and URL paths so known bot-blocked sites fail fast with clearer agent feedback.

## Best Possible Solution

Define a browser-owned navigation policy contract in the browser plugin, including host/path matching, redirect and frame scope, advisory-versus-hard-block behavior, schema/help/docs/tests, and its relationship to the broader egress policy tracked in https://github.com/openclaw/openclaw/issues/39685.
