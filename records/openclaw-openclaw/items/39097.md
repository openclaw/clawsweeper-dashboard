## Summary

Keep open. Current main and v2026.6.6 have browser SSRF/private-network checks, hostname allowlist exceptions, and a browser URL-pattern helper for response/wait matching, but they still do not expose an operator browser denylist, blocked URL-path policy, or warnlist for navigation.

## What This Changes

Requests a browser navigation config surface for blocked URL/domain/path patterns, with clear blocked-navigation errors and an optional advisory warnlist.

## Best Possible Solution

Define an operator-owned browser navigation policy in the browser plugin guard, including host/path matching, redirect and frame scope, advisory-versus-hard-block semantics, schema/help/docs updates, and no bundled third-party default list unless maintainers approve that policy.
