## Summary

Keep open. Current main and v2026.5.12 only expose SSRF/private-network browser navigation controls; no operator URL/domain/path denylist or warnlist exists, and adding one is a public config plus security-sensitive navigation-policy design.

## What This Changes

Request an operator-configurable browser URL/domain/path denylist, plus optional warnlist, that blocks or warns before browser navigation to known bot-detection sites.

## Best Possible Solution

Define a central browser URL policy primitive with typed config, strict schema/help/docs, pre-navigation plus final-URL/redirect enforcement, warnlist response semantics, default-list ownership rules, and focused regression coverage.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a feature/config request. Source inspection against current main and v2026.5.12 shows only SSRF/private-network browser policy, not denylist/warnlist fields.

## Solution Assessment

Unclear. A centralized browser navigation policy is the likely layer, but maintainers still need to decide config names, matching semantics, redirect/subframe handling, warnlist response shape, and default-list ownership.
