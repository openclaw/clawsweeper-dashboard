## What This Changes

The issue requests a browser navigation denylist or warnlist for blocked domains and URL patterns.

## Best Possible Solution

Define one maintainer-approved browser navigation policy contract in the browser plugin, with schema, docs, and tests covering host/path matching, redirects, subframes/subresources, warn versus hard-block semantics, and its relationship to broader web-tool egress policy.
