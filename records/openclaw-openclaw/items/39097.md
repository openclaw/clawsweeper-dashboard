## Summary

Keep open. Current main still does not provide an operator-configurable browser URL/domain/path denylist or warnlist; the implemented browser policy remains SSRF/private-network blocking with hostname allowlist exceptions. The prior ClawSweeper review in the issue comments reached the same conclusion, and a fresh pass over current main did not find a newer implementation.

## What This Changes

Request an operator-configurable browser URL/domain/path denylist, with optional warnlist behavior, so known bot-protected sites fail fast with a clear browser navigation error.

## Best Possible Solution

Add a maintained browser URL policy surface that is explicit about matching semantics and redirect handling, then wire it through config types, strict schema/generated config docs, the resolved browser policy, all browser navigation paths, route error responses, and focused tests. Treat warnlist and bundled default domain lists as separate decisions unless maintainers intentionally include them in the same design.
