## Summary

Close as expected current behavior rather than a Feishu transport bug: current main intentionally keeps group/channel normal final replies private by default unless the agent uses the message tool or the operator restores legacy automatic replies.

## What This Changes

The issue requests restoring Feishu group chat visible replies after v2026.5.x processes group messages but reports `queuedFinal=false` and sends no normal final reply back to the room.

## Best Possible Solution

Keep the documented message-tool-only group default, and use `messages.groupChat.visibleReplies: "automatic"` or a tool-call-reliable model when legacy visible final replies are desired.

## Reproduction Assessment

Yes. Source inspection and existing tests show current main produces the reported no-final-send shape for group/channel turns when the model returns only a normal final answer under the default message-tool-only policy.

## Solution Assessment

Yes. The narrow maintainable path is operator configuration or model/tool-call behavior, not reverting the repository-wide group/channel default in Feishu-specific code.
