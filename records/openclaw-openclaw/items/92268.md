## Summary

The issue requests a supported OpenClaw opt-in for Codex app-server `workspace-write` outbound network access without switching to `danger-full-access`.

## Best Possible Solution

Keep the shipped OpenClaw sandbox egress bridge as the supported path, leaving plain native Codex `workspace-write` network access restricted by default.

## Reproduction Assessment

## Solution Assessment

Yes. Deriving Codex app-server network access from OpenClaw sandbox egress is the safer maintainable solution because it preserves filesystem containment and keeps raw Codex `config.toml` from bypassing OpenClaw policy.
