## Summary

Keep this PR open. Current main still lacks both edge-case guards, Slack SDK source supports the request-error wrapper premise, and the patch is a narrow functional fit, but it needs the required user-facing changelog entry before merge.

## What This Changes

The PR broadens infra unhandled-rejection handling to treat Slack Web API empty request-error wrappers and nullish rejection reasons as non-fatal, with regression coverage in the unhandled-rejections tests.

## Best Possible Solution

Land a rebased version of this narrow infra hardening with the required changelog entry, preserving fatal behavior for non-null non-transient rejections and for Slack API/platform/auth errors.

## Reproduction Assessment

Yes. A source-level reproduction is clear: on current main, a `slack_webapi_request_error` with `Error("A request error occurred: ")` has no matching transient signal, and an `undefined` rejection reason reaches the generic fatal exit path.

## Solution Assessment

Yes, with one process fix. The request-wrapper classification matches the Slack Web API RequestError contract, and the nullish-reason guard addresses the Socket Mode failure shape; the safer merge path is to add the required changelog and rerun focused infra tests after rebasing.
