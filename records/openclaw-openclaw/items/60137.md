## What This Changes

The PR adds Slack Web API request-wrapper transient classification, suppresses nullish unhandled rejection reasons as non-fatal, and adds regression tests for those cases.

## Best Possible Solution

Land a rebased version of this narrow infra hardening with a single-line `CHANGELOG.md` entry, preserving fatal behavior for non-null non-transient rejections and for Slack API/platform/auth failures.

Yes. A source-level reproduction is clear on current main: an empty `slack_webapi_request_error` wrapper has no transient signal, and `undefined` or `null` reaches the generic fatal unhandled-rejection path.
