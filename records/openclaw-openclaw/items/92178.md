## Summary

Keep open: this is a focused start, but it is not merge-ready because it guards only the audit formatter while malformed paired-device scopes can still reach another `.trim()` during authorization, and the after-fix proof does not exercise the real gateway handshake path.

## What This Changes

The PR extracts `formatAuditList`, skips non-string items before trimming, exports it for tests, and adds helper-level regression coverage plus a repro script.

## Best Possible Solution

Normalize paired-device roles, approved scopes, and token scopes at the gateway/device-pairing boundary before audit logging and authorization, then prove the reported CLI write flow against malformed stored pairing data.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Partially: source inspection confirms the current formatter crash and a remaining sibling crash path, but the PR proof does not run the full CLI-to-gateway handshake with malformed stored pairing data.

## Solution Assessment

Is this the best way to solve the issue? No; guarding the audit formatter is necessary but too narrow because paired-device scope arrays still reach authorization helpers that trim entries without validating them.
