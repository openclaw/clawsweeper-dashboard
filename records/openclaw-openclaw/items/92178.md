## What This Changes

The PR guards gateway upgrade-audit list formatting against non-string entries, adds focused unit coverage, and adds a direct helper reproduction script.

## Best Possible Solution

Normalize persisted pairing role and scope lists once at the pairing-state boundary using unknown-safe string-list normalization, then prove the actual WebSocket upgrade path returns a structured pairing outcome without any uncaught trim error.

## Reproduction Assessment

Yes at source level: pairing records accept unvalidated array members, and multiple handshake consumers invoke trim on those members. The submitted script establishes only the formatter failure, not a complete current-main gateway reproduction.

## Solution Assessment

No. Guarding the audit formatter is one-sided because authorization consumes the same unnormalized data; the maintainable fix is canonical normalization before all handshake consumers.
