## What This Changes

The issue asks for the Control UI gateway log display to strip or render ANSI escape sequences instead of showing raw codes.

## Best Possible Solution

Land a focused Control UI logging fix that shares one ANSI display strategy across every `logs.tail` UI consumer, normalizes displayed message/subsystem/fallback text, keeps raw export behavior deliberate, and adds ANSI-bearing regression coverage.

## Solution Assessment

Yes. UI-side display normalization is the narrowest maintainable fix because it improves browser readability without changing the server `logs.tail` contract or raw export semantics.
