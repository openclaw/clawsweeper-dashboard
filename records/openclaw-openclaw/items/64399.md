Keep open. Current main and the latest tagged release still preserve ANSI-bearing log-tail text in the dedicated Control UI Logs tab and render it directly; the duplicate issue was closed in favor of this one, and the related fix PR was closed unmerged.

## What This Changes

The issue asks for the Control UI gateway log panel to stop showing raw ANSI escape sequences and render or normalize styled log text.

## Best Possible Solution

Land a focused Control UI logging fix that uses one ANSI display strategy for all `logs.tail` UI consumers, normalizes displayed message/subsystem/fallback text, keeps raw export behavior intentional, and adds ANSI-bearing regression coverage.

## Solution Assessment

Yes. UI-side display normalization is the narrowest maintainable fix because it improves browser readability without changing the `logs.tail` server contract or raw export behavior.
