## Summary

Keep open. Current main still lets ANSI-bearing `logs.tail` text reach the dedicated Control UI Logs tab unchanged; the Overview tail has only a local strip helper, and the prior focused fix PR was closed unmerged.

## What This Changes

The issue asks for Control UI gateway logs to strip or render ANSI escape sequences instead of displaying raw codes.

## Best Possible Solution

Add a shared Control UI ANSI display helper used by the Logs tab and Overview tail, normalize displayed message/subsystem/fallback text, keep raw log/export data unchanged, and cover structured plus plain ANSI-bearing lines with regression tests.

## Reproduction Assessment

Yes. A source-level reproduction is clear: feed an ANSI-bearing JSONL or plain line into `parseLogLine()` and render it through `renderLogs()`; current main preserves the escape-coded text and displays it directly.

## Solution Assessment

Yes. UI-side display normalization is the narrowest maintainable fix because it addresses the browser readability bug without changing server log files, CLI fallback behavior, or raw exports.
