## Summary

Keep open. Current main still lets ANSI-bearing `logs.tail` text reach the dedicated Control UI Logs tab unchanged; the Overview tail has only a local strip helper, and the related fix PR was closed unmerged.

## What This Changes

The issue requests that the Control UI log panel render or strip ANSI escape sequences instead of displaying raw escape-code text.

## Best Possible Solution

Add one shared Control UI ANSI display strategy for every `logs.tail` UI consumer, keep raw log/export data unchanged, and cover structured plus plain ANSI-bearing lines with regression tests.

## Reproduction Assessment

Yes. A source-level reproduction is clear: feed an ANSI-bearing JSONL or plain line into `parseLogLine()` and render it through `renderLogs()`; the current parser preserves the escape-coded text and the Logs view displays it directly.

## Solution Assessment

Yes. A narrow UI display normalization is the safest fix boundary; changing server-side log tailing would risk altering raw exports, CLI behavior, and log-file semantics.
