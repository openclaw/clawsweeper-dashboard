## Summary

Keep open. Current main still lets ANSI-bearing `logs.tail` text reach the dedicated Control UI Logs tab without display normalization; only the Overview tail strips ANSI locally, and the linked fix PR #64431 was closed unmerged while duplicate #45001 points back here.

## What This Changes

The issue requests that the Control UI log panel stop showing raw ANSI escape sequences in gateway log output.

## Best Possible Solution

Adopt one shared Control UI ANSI display-normalization helper across the Logs tab and Overview tail, covering message, subsystem, and fallback display text while preserving raw log/export behavior.

## Reproduction Assessment

Yes, at source level. Feed an ANSI-bearing JSONL or plain line into `parseLogLine()` and render it through `renderLogs()`; current main preserves the escape-coded text and interpolates it directly.

## Solution Assessment

Yes. UI-side display normalization is the narrowest maintainable fix because it addresses the browser readability bug without changing server log files, CLI formatting, or `logs.tail` raw payload semantics.
