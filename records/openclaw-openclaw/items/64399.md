## Summary

Keep open. Current main and the latest release still let ANSI-bearing gateway log-tail text reach the dedicated Control UI Logs tab unchanged; the duplicate report was closed in favor of this issue, and the related fix branch was closed unmerged.

## What This Changes

The issue requests that the Control UI log panel stop displaying raw ANSI escape sequences in gateway logs and instead normalize or render them for readability.

## Best Possible Solution

Land a focused Control UI logging fix that shares one ANSI display strategy across every `logs.tail` UI consumer, normalizes displayed message/subsystem/fallback text, keeps raw export behavior deliberate, and adds ANSI-bearing regression coverage.

## Reproduction Assessment

Yes, at source level: ANSI-styled gateway log text can flow from `readConfiguredLogTail()` through `parseLogLine()` into `renderLogs()` without display normalization. I did not run a live browser UI in this read-only review.

## Solution Assessment

Yes. UI-side display normalization is the narrowest maintainable fix because it improves browser readability without changing the server `logs.tail` contract or raw export behavior.
