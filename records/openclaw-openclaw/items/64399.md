## Summary

Keep open. Current main and the latest release still let ANSI-bearing gateway log text reach the dedicated Control UI Logs tab without stripping or rendering normalization; the only ANSI cleanup I found is local to the Overview log tail.

## What This Changes

The issue requests that Control UI gateway logs render or normalize ANSI-colored log text instead of showing raw escape sequences.

## Best Possible Solution

Add a shared Control UI ANSI display helper for every `logs.tail` UI consumer, normalize displayed message/subsystem/fallback text, keep raw tail/export data deliberate, and add ANSI-bearing regression coverage.

## Reproduction Assessment

Yes, at source level: ANSI-styled gateway log text can flow from `readConfiguredLogTail()` through `parseLogLine()` into `renderLogs()` without stripping or rendering normalization. I did not run a live browser UI in this read-only review.

## Solution Assessment

Yes. UI-side display normalization is the narrowest maintainable fix because it improves browser readability without changing the server `logs.tail` contract or raw export semantics.
