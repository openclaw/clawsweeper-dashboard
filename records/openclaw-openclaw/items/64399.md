## Summary

Keep open. Current main and v2026.5.20 still let ANSI-bearing `logs.tail` text reach the dedicated Control UI Logs tab unchanged, while the duplicate report at https://github.com/openclaw/openclaw/issues/45001 was closed in favor of this issue and the related fix PR at https://github.com/openclaw/openclaw/pull/64431 was closed unmerged.

## What This Changes

The issue asks the Control UI log panel to stop displaying raw ANSI escape sequences from gateway log lines.

## Best Possible Solution

Land a focused Control UI logging fix that centralizes ANSI display normalization for all `logs.tail` UI consumers, preserves raw export semantics, and adds regression coverage for structured and plain ANSI-bearing log lines.

## Reproduction Assessment

Yes, at source level: ANSI-styled gateway log text can be returned by `readConfiguredLogTail()`, parsed by `parseLogLine()` without normalization, and interpolated directly by `renderLogs()`. I did not run a live browser session in this read-only review.

## Solution Assessment

Yes. UI-side ANSI display normalization, with stripping matching the existing Overview tail behavior, is the narrow maintainable fix because it improves browser readability without changing the server `logs.tail` contract or raw exports.
