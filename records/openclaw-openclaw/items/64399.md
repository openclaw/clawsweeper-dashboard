## Summary

Keep open. Current main and the latest release still preserve ANSI-bearing `logs.tail` text in the dedicated Control UI Logs tab and render it directly; the only current ANSI normalization I found is local to the Overview log tail.

## What This Changes

The issue requests that the Control UI/Gateway Logs panel stop showing raw ANSI escape sequences and render or normalize them for readable browser display.

## Best Possible Solution

Add a shared Control UI ANSI display helper used by every `logs.tail` UI consumer, normalize displayed message/subsystem/fallback text, keep raw tail/export data deliberate, and add ANSI-bearing structured and plain-line regression coverage.

## Reproduction Assessment

Yes, at source level: ANSI-styled gateway log text can flow through `readConfiguredLogTail()`, `parseLogLine()`, and `renderLogs()` without stripping or rendering normalization. I did not run a live browser UI in this read-only review.

## Solution Assessment

Yes. UI-side display normalization is the narrowest maintainable fix because it improves browser readability without changing the server `logs.tail` contract, CLI fallback behavior, or raw export semantics.
