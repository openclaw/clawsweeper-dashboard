## Summary

Keep open. Current `main` still preserves ANSI-bearing `logs.tail` text in the dedicated Control UI Logs tab and renders it directly; the only ANSI handling I found is an Overview-local strip helper, and the focused fix PR was closed unmerged.

## What This Changes

The issue requests Control UI gateway logs to stop displaying ANSI escape sequences as raw text and render or normalize them for readability.

## Best Possible Solution

Land a focused Control UI logging fix that shares one ANSI display-normalization helper across every `logs.tail` UI consumer while leaving server tail data and raw exports intentional.

## Reproduction Assessment

Yes, at source level: an ANSI-bearing `logs.tail` line flows through `parseLogLine()` as `message` or `raw`, and `renderLogs()` interpolates that text directly. I did not run a live browser UI in this read-only review.

## Solution Assessment

Yes. UI-side display normalization shared between the Logs tab and Overview tail is the narrowest maintainable fix; changing gateway log emission or the `logs.tail` raw contract would have wider blast radius.
