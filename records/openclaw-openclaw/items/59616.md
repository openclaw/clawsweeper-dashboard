## Summary

Keep open. Current main still starts the gateway config watcher from only the root config file, so `$include`-only edits have no watched event path; the linked closing PR is the right implementation candidate but is still open and unmerged.

## What This Changes

The issue asks the gateway config hot-reloader to watch files referenced by `$include` so include-only edits trigger reload without a gateway restart.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82698 or an equivalent narrow fix that syncs the reloader's watched files with accepted `$include` paths and covers startup, include-only edits, removed includes, and in-process write ordering.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a root config with `$include` can only reload when `opts.watchPath` emits, so editing only the included file has no watched event path on current main.

## Solution Assessment

Yes. The suggested direction is the narrow maintainable fix: use the existing include scanner and Chokidar add/unwatch support to keep the watched set aligned with accepted config snapshots.
