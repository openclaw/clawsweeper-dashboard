## Summary

Keep open. Current main still watches only the active root config path, while include resolution and the recursive include-path collector exist but are not connected to the gateway reloader; the earlier source PR was closed unmerged rather than superseding the request.

## What This Changes

Request that gateway config hot reload watch files referenced by `$include` so edits to modular config files trigger reloads.

## Best Possible Solution

Land a narrow gateway reloader change that syncs Chokidar’s watched set with the accepted `$include` graph after startup and accepted snapshots, while preserving include security checks and avoiding extra reload loops.

## Reproduction Assessment

Yes, by source inspection. Start the reloader with a root config containing `$include`, edit only the included file, and current main has no watcher on that path to schedule a reload.

## Solution Assessment

Yes. The proposed direction is the narrow maintainable path: reuse the existing include-path collector and diff paths into Chokidar `add`/`unwatch` without changing include resolution semantics.
