## Summary

Keep open. Current main and the latest release still watch only the root config path, while `$include` files are resolved only after a watched reload reads a snapshot; the open draft PR that references this issue is an implementation candidate, not a reason to close before merge.

## What This Changes

The issue asks for gateway config hot-reload to watch files referenced by `$include`, not only the root config file.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82698 or an equivalent narrow fix that keeps the reloader's watched path set aligned with accepted `$include` files, including regression coverage for startup, include-only changes, in-process writes, and stale sync ordering.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a root config containing `$include` can only reload when `opts.watchPath` emits, so editing only an included file has no watched event path on current main.

## Solution Assessment

Yes. The issue's suggested direction is the narrow maintainable fix: use the existing include scanner and Chokidar's dynamic add/unwatch API to sync the watched set after accepted snapshots and writes.
