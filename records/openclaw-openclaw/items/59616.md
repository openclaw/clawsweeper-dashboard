## Summary

Keep open. Current main still constructs the gateway config watcher around only the active root config path, while `$include` resolution and recursive include-path scanning exist but are not connected to the reloader.

## What This Changes

The issue requests that gateway config hot-reload also watch files referenced through `$include` directives.

## Best Possible Solution

Land a narrow gateway reloader change that synchronizes Chokidar’s dynamic watched set with the accepted `$include` graph after startup and each valid snapshot, with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: start the gateway with a root config containing a `$include`, edit only the included file, and current main has no watcher on that included path to schedule a reload.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse or harden the existing include-path collector and diff paths into Chokidar `add`/`unwatch` after valid snapshot reads, without adding a new config option or changing include semantics.
