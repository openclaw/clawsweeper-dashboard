## Summary

Keep open. Current `main` and the latest release still watch only the active root config file, while `$include` resolution and recursive include scanning exist but are not wired into the gateway config reloader.

## What This Changes

The issue requests that gateway config hot-reload watch files referenced through `$include` so edits to included config files trigger reloads without a manual gateway restart.

## Best Possible Solution

Gateway reloader should synchronize Chokidar’s watched set to the validated include graph after startup and accepted config snapshots, while preserving include path confinement guardrails.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a root config containing `$include` can reload only when `opts.watchPath` emits, so editing only an included file has no watched event path to schedule a reload.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse `collectIncludePathsRecursive()` in the gateway reloader and diff paths into `watcher.add()` / `watcher.unwatch()` after accepted snapshots, with focused tests for startup, removal, feedback-loop, and hash-dedupe cases.

Fix https://github.com/openclaw/openclaw/issues/59616. Current `src/gateway/config-reload.ts` watches only `opts.watchPath`, so edits to `$include`-referenced config files do not trigger gateway config reload. Implement dynamic include-file watching inside `startGatewayConfigReloader` using `collectIncludePathsRecursive()` from `src/config/includes-scan.ts` and Chokidar `add` / `unwatch`. Derive watched include paths only from valid/accepted config snapshots, preferably `snapshot.parsed`, so include path guardrails from `src/config/includes.ts` are preserved. Sync on startup/initial accepted state, after successful external reload snapshots, and after in-process config writes where a current snapshot is available. Account for the closed source PR discussion at https://github.com/openclaw/openclaw/pull/59632: avoid startup races, suppress or ignore dynamic `add` feedback loops, and cover include removal. Add focused regression tests in `src/gateway/config-reload.test.ts`; add a single-line `CHANGELOG.md` Fixes entry if this is a ClawSweeper-owned repair PR.
