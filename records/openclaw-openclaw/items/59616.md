## Label Justifications

- `P2`: This is a normal-priority gateway config hot-reload bug with limited blast radius to users who split config through `$include`.
- `impact:other`: The issue has maintainer-visible operational impact because included config edits can be silently ignored until a manual gateway restart, but it does not match the more specific owned impact classes.

## Summary

Keep open: current main and the latest release still watch only the root config path, while the docs describe `$include` source-file edits as hot-reload inputs; the linked fixing PR owns the work but has not merged.

## What This Changes

The issue requests gateway config hot-reload to watch files referenced through `$include` so include-only edits trigger reload without a manual restart.

## Best Possible Solution

Refresh, review, and land a clean dynamic include-watch fix through https://github.com/openclaw/openclaw/pull/90653 or a narrow replacement; the runtime should watch the root config plus the resolved include graph without adding a new user config knob.

## Reproduction Assessment

Yes, from source inspection. Current main passes only the root config path into the reloader and creates Chokidar for only `opts.watchPath`, so an include-only file edit has no watcher event path before a fix lands.

## Solution Assessment

Yes for the issue direction. Reusing the existing include graph and Chokidar add/unwatch APIs is the narrow maintainable fix; adding a new watched-folder config option would be broader and less aligned with the existing `$include` contract.
