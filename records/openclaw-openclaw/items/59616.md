## Summary

Keep open. Current main still only watches the active config file path and has no include-file watch synchronization, while `$include` resolution and recursive include-path scanning already exist. The older broad request was closed as superseded by this issue, the linked implementation PR is closed unmerged, and the broader debounce issue does not cover the missing include discovery behavior.

## What This Changes

The issue requests that gateway config hot-reload watch files referenced by `$include` so edits to included config files trigger reloads without a manual gateway restart.

## Best Possible Solution

Land a narrow gateway reloader change that keeps the active include-watch set synchronized with the current valid `$include` graph while preserving existing debounce, recovery, in-process write dedupe, promotion, and reload planning behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is available: use a root config with a `$include`, start the gateway reloader, edit only the included file, and no reload is scheduled because current main only watches `opts.watchPath`.

## Solution Assessment

Yes. The issue’s proposed direction is the narrow maintainable fix: update `startGatewayConfigReloader` to collect include paths from accepted snapshots and diff them into Chokidar with add/unwatch without changing include resolution semantics.

## Repair Work Prompt

Fix openclaw/openclaw#59616. Current main's `startGatewayConfigReloader` only watches `opts.watchPath`, so edits to files referenced by `$include` do not trigger reload. Implement a narrow reloader-side include-watch sync: after startup/initial successful snapshot and after each accepted external snapshot, compute include paths from the valid authored config snapshot, diff them against a maintained include-watch set, call Chokidar `watcher.add()` for new paths and `watcher.unwatch()` for removed paths, and keep sync failures best-effort/nonfatal. Preserve existing debounce, missing-config retries, in-process write hash dedupe, last-known-good promotion, restart planning, and source-config reload planning. Do not solve #74122 debounce/scoped reload in this PR. Do not broaden include path authority; derive watched paths from valid snapshots or mirror the existing include resolver guardrails. Add focused tests in `src/gateway/config-reload.test.ts` for initial include watch registration, included-file change triggering reload, include removal unwatching, nested include discovery, and graceful sync/read failures. Add a one-line `CHANGELOG.md` Fixes entry with `Thanks @alkor2000`.
