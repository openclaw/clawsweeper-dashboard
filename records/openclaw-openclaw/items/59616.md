## Summary

Keep open. Current main and latest release still watch only the active config path, while `$include` resolution and recursive include scanning exist but are not wired into the gateway reloader; the closed unmerged source PR at https://github.com/openclaw/openclaw/pull/59632 remains useful context.

## What This Changes

Watch `$include`-referenced config files during gateway hot reload so modular config edits trigger reload without a manual restart.

## Best Possible Solution

Gateway reloader should derive the watched include set from accepted root config snapshots and keep Chokidar’s dynamic watch list in sync without weakening include path guardrails.

## Reproduction Assessment

Yes. Source shows Chokidar is created only for `opts.watchPath`, so a root config containing `$include` followed by an edit to only the included file has no watched event path to schedule a reload.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse `collectIncludePathsRecursive()` and diff paths into `watcher.add()` / `watcher.unwatch()` after startup and accepted snapshots, with tests for startup, removal, feedback-loop, and hash-dedupe edge cases.
