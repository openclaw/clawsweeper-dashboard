## Summary

Keep open: the PR addresses a real linked gateway crash pattern, but the new classifier over-matches unrelated module-load failures and needs a narrow repair before merge.

## What This Changes

The PR adds an unhandled-rejection classifier for bundled dist module rotation and tests export-mismatch/module-not-found cases so the gateway exits with a clean restart reason after in-place package upgrades.

## Best Possible Solution

Land a narrowed safety net that only recognizes OpenClaw-owned bundled dist module rotation and preserves normal fatal handling for unrelated dependency module-load failures.

## Reproduction Assessment

Yes for the introduced review finding: source inspection shows any `ERR_MODULE_NOT_FOUND` message containing `runtime.js` satisfies the new classifier. The original in-place upgrade race was not live-reproduced here; the PR relies on deterministic classifier cases.

## Solution Assessment

No. Restarting on real OpenClaw dist rotation is a reasonable mitigation, but the current implementation is too broad; the narrow maintainable fix is to scope the classifier to OpenClaw-owned bundled dist runtime boundaries.
