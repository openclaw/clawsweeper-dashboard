## Summary

Keep open for maintainer review: the patch is a narrow, well-tested mitigation for a real gateway crash path, but it intentionally skips fatal stability-bundle hooks for matched module-load failures and has not been exercised in a live supervised global upgrade.

## What This Changes

The PR adds a dist-module-rotation classifier to the global unhandled-rejection handler, routes OpenClaw-owned bundled `*runtime.js` import failures to a clean restart, and adds classifier/handler tests.

## Best Possible Solution

Keep the narrow runtime restart classifier if maintainers accept log-only diagnostics for this signature; otherwise add a lightweight non-crash diagnostic record before merge.

## Reproduction Assessment

No high-confidence live repro was run in this review. The issue log, Node ESM error-shape probe, and current lazy-runtime/dist-update paths make the failure source-reproducible, but the PR itself says a live in-place global upgrade under a supervised Gateway was not driven.

## Solution Assessment

Yes, as an acceptable mitigation: once a running old chunk sees a replaced dist tree, restarting onto a consistent build is the maintainable runtime response. The unresolved decision is whether the matched branch should skip crash-bundle hooks or emit a lighter diagnostic breadcrumb.
