## Summary

Keep open. Current main still lacks a dist-module-rotation path, and this PR is a targeted fix for a real gateway crash/restart class; the remaining maintainer decision is whether the intentional no-crash-bundle restart path is the right availability tradeoff before merge.

## What This Changes

Adds a dist module-rotation classifier to unhandled-rejection handling so OpenClaw exits for a clean restart and skips fatal hooks, with classifier and handler tests.

## Best Possible Solution

Land a narrow unhandled-rejection classifier that restarts only for OpenClaw-owned dist module rotation and keeps plugin/dependency failures on the fatal path; maintainers should explicitly accept or adjust the no-crash-bundle diagnostic tradeoff before merge.

## Reproduction Assessment

Yes. Current main has no dist-rotation classifier and the linked report shows the exact Node export-mismatch error falling through as an unhandled rejection; this read-only review did not run a live in-place package upgrade.

## Solution Assessment

Yes, as an acceptable mitigation. Once a running process has a mixed old/new dist module graph, exiting for supervisor restart at the global unhandled-rejection boundary is narrower than changing plugin or provider runtime paths, while build-side hardening would not repair an already-loaded old graph.
