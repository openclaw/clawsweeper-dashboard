## Summary

Keep this open. Current main still has coarse task status, delivery status, terminal outcome, audit codes, and prose summary/error fields, but no shared machine-readable blocker/failure taxonomy; related #64413 is still open and narrower delivery-failure work rather than a superseding tracker.

## What This Changes

Request to add shared blocker and failure classification types/helpers and adopt them across task records, audits, summaries, status output, and future retry decisions.

## Best Possible Solution

Define an additive shared task blocker/failure classification contract with explicit unknown handling, map TaskRecord, Task Flow, delivery, audit, and agent-run outcomes into it, expose it compatibly through persisted records, runtime DTOs, CLI JSON/status output, and docs, and coordinate the delivery-specific subset with #64413.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection, which shows current main has status/prose fields and operational audit codes but no shared blocker/failure taxonomy.

## Solution Assessment

Yes at the design-direction level: a centralized additive taxonomy is the maintainable direction. The exact field names, persistence model, and compatibility guarantees should be chosen by maintainers before implementation so consumers do not branch on another ad hoc surface.
