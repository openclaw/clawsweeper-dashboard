## Summary

Keep this open: current main still has task status, delivery status, terminal outcome, Task Flow waiting/blocked state, audit codes, and prose summaries, but no shared machine-readable blocker/failure taxonomy. The related #64413 remains open and covers only delivery failure reasons, so it does not supersede this broader task taxonomy request.

## What This Changes

Request to add a shared machine-readable blocker/failure classification taxonomy across task records, Task Flow, audits, summaries, status output, and future retry logic.

## Best Possible Solution

Define an additive shared task blocker/failure classification contract with explicit unknown handling, map TaskRecord, Task Flow, delivery, audit, and agent-run outcomes into it, and expose it compatibly through persisted records, runtime DTOs, CLI JSON/status output, and docs.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection at bc42952c31d2cabfae7af7899d89e1c196b752d7, which shows status/prose/audit-code surfaces but no shared blocker/failure taxonomy.

## Solution Assessment

Yes at the design-direction level: a centralized additive taxonomy is the maintainable path. The exact names, persistence model, and public compatibility rules should be chosen by maintainers before implementation.
