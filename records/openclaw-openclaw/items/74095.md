## Summary

Keep this open because current main still exposes task state, delivery state, blocked summaries, audit codes, and prose error fields without a shared machine-readable blocker/failure taxonomy. The request is valid, but broad enough to need maintainer contract decisions before implementation.

## What This Changes

The issue requests a shared blocker/failure classification model for task waiting, blocked, failed, transient, and unknown outcomes across task records, audits, summaries, and user-facing status output.

## Best Possible Solution

Define an additive shared task blocker/failure classification contract with explicit unknown handling, map TaskRecord, TaskFlow, delivery, audit, and agent-run outcomes into it, expose it compatibly through persisted records, runtime DTOs, CLI JSON/status output, and docs, and coordinate delivery-specific reason codes with #64413.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection provide a high-confidence verification path: current main has lifecycle/status/prose fields and operational audit codes, but no shared blocker/failure taxonomy.

## Solution Assessment

Yes at the design-direction level: a centralized additive taxonomy is the maintainable direction. The exact classes, persistence fields, external DTO exposure, and compatibility guarantees should be settled by maintainers before implementation.
