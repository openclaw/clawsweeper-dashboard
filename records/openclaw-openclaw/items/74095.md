## Summary

Keep open: current main still has coarse task statuses, delivery statuses, blocked summaries, free-form error text, and operational audit codes, but no shared machine-readable blocker/failure taxonomy. The related delivery-focused issue remains open and narrower, so it does not supersede this broader task-system request.

## What This Changes

Add centralized blocker/failure classification types and helpers, then reuse them across task records, TaskFlow, audits, summaries, and status surfaces.

## Best Possible Solution

Define an additive shared taxonomy contract with explicit unknown handling, then map TaskRecord, TaskFlow, delivery, audit, and runtime DTO outcomes into it consistently.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing behavior report. Source and docs inspection on current main gives high-confidence evidence that the requested taxonomy is not present.

## Solution Assessment

Yes at the design-direction level: a centralized additive taxonomy is the maintainable path. The exact names, persistence model, DTO exposure, and relationship to #64413 need maintainer agreement before implementation.
