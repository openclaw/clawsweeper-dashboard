## Summary

Keep this open. Current main still has coarse task status, delivery status, terminal outcome, audit codes, and prose summary/error fields, but it does not provide the requested shared machine-readable blocker/failure taxonomy; related #64413 is open and narrower delivery-failure work rather than a superseding tracker.

## What This Changes

The issue requests a shared machine-readable taxonomy for task blocker and failure classes across task status, audits, summaries, and user-facing output.

## Best Possible Solution

Define an additive shared task blocker/failure classification contract with explicit unknown handling, map task, Task Flow, delivery, audit, and agent-run outcomes into it, expose it through compatible persisted records, runtime DTOs, CLI JSON/status output, and docs, and coordinate the delivery-specific subset with #64413.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection, which shows current main has status/prose fields and operational audit codes but no shared blocker/failure taxonomy.

## Solution Assessment

Yes at the design-direction level: a centralized additive taxonomy is the maintainable direction. The exact public contract and rollout boundaries should be chosen by maintainers before implementation so consumers do not branch on another ad hoc field.
