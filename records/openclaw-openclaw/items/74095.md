## Summary

Keep this open: current main still exposes task status, delivery status, terminal outcome, TaskFlow blocked/waiting state, and operational audit codes, but not a shared machine-readable blocker/failure taxonomy. The related delivery-focused issue #64413 is open and narrower, so it does not supersede this broader task taxonomy request.

## What This Changes

Request to add a shared machine-readable blocker/failure taxonomy for tasks and reuse it across task records, audits, summaries, and user-facing status output.

## Best Possible Solution

Define an additive shared taxonomy contract with explicit unknown handling, then map TaskRecord, TaskFlow, delivery, audit, plugin DTO, CLI JSON/status, and docs surfaces onto it without breaking existing status fields.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection, which shows current main has status/prose/error/audit fields but no shared blocker/failure taxonomy.

## Solution Assessment

Yes at the design-direction level: a centralized additive taxonomy is the maintainable solution. The exact names, persistence model, DTO compatibility, and relationship to #64413 should be settled before implementation.
