## Summary

Keep this open: current main still has task status, delivery status, blocked summaries, wait metadata, audit codes, and prose errors, but no shared task-level blocker/failure taxonomy that audits, summaries, CLI JSON, and plugin runtime DTOs can reuse. Related #64413 remains open and is narrower, covering delivery failure reasons rather than superseding this broader task taxonomy request.

## What This Changes

Request to add a shared machine-readable blocker/failure taxonomy for background tasks, Task Flow, audits, summaries, and status output.

## Best Possible Solution

Define an additive shared classification contract with explicit unknown handling, then map TaskRecord, Task Flow, delivery, audit, and agent-run outcomes into it across persisted records, runtime DTOs, CLI JSON/status output, and docs.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection, which shows current main has status/prose/audit fields but no shared task blocker/failure taxonomy.

## Solution Assessment

Yes at the design-direction level: a centralized additive taxonomy is the maintainable path. The exact public codes, persistence model, and compatibility rules should be chosen by maintainers before implementation.
