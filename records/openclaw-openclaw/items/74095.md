## Summary

Keep this open: current main still exposes coarse task statuses, delivery statuses, TaskFlow waiting/blocked state, prose errors, and operational audit codes, but not a shared machine-readable blocker/failure taxonomy. The delivery-focused #64413 is related and still open, but it does not cover the broader task taxonomy requested here.

## What This Changes

Request a shared task blocker/failure classification contract and adoption across task records, audits, summaries, and status output.

## Best Possible Solution

Define an additive shared taxonomy with explicit unknown handling, then map TaskRecord, TaskFlow, delivery, audit, and agent-run outcomes into stable public/runtime fields.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection, which shows current main has status/prose/error fields and audit codes but no normalized blocker/failure taxonomy.

## Solution Assessment

Yes at the design-direction level: a centralized additive taxonomy is the maintainable direction. The exact names, persistence shape, DTO exposure, and compatibility rules need maintainer agreement before implementation.
