## Summary

Keep this open: current main still has task and TaskFlow status fields, delivery status, prose errors/summaries, and operational audit codes, but no shared machine-readable blocker/failure taxonomy. The narrower delivery-focused #64413 remains open and related, but it does not supersede this broader task taxonomy request.

## What This Changes

Introduce a shared task blocker/failure taxonomy for machine-readable waiting, blocked, failed, and transient failure states.

## Best Possible Solution

Define an additive shared taxonomy contract with explicit unknown handling, then map TaskRecord, TaskFlow, delivery, audit, and agent-run outcomes into it consistently across public/runtime surfaces.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection, which shows current main has status/prose/error fields and audit codes but no normalized blocker/failure taxonomy.

## Solution Assessment

Yes at the design-direction level: a centralized additive taxonomy is the maintainable solution. The exact names, persistence model, DTO exposure, and public compatibility rules should be settled before implementation.
