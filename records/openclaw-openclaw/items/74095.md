## Summary

Keep open: current main still has task statuses, delivery statuses, free-form errors/summaries, and operational audit codes, but no shared machine-readable blocker/failure taxonomy. The related delivery-focused issue remains open and narrower, so it does not supersede this task-wide taxonomy request.

## What This Changes

Introduce a shared, machine-readable blocker/failure taxonomy for task states and task failure conditions.

## Best Possible Solution

Define an additive shared taxonomy contract with explicit unknown handling, then map TaskRecord, TaskFlow, delivery, audit, and agent-run outcomes into it consistently across public/runtime surfaces.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection provides high-confidence verification that current main has coarse status/prose fields and operational audit codes, but no shared blocker/failure taxonomy.

## Solution Assessment

Yes at the design-direction level: a centralized additive taxonomy is the maintainable path. The exact names, persistence model, DTO exposure, and compatibility rules need maintainer agreement before implementation.
