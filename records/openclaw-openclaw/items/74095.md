## Summary

Keep this open. Current main still exposes task status, Task Flow waiting/blocked state, delivery status, terminal outcome, audit codes, and prose summary/error fields, but it does not provide the requested shared machine-readable blocker/failure taxonomy; related #64413 is open and narrower delivery-failure work, not a superseding tracker for this broader task-system contract.

## What This Changes

Request to add shared task blocker/failure classification types and helpers, then reuse them across task records, Task Flow, audits, summaries, and user-facing status output.

## Best Possible Solution

Define an additive shared task blocker/failure classification contract with explicit unknown handling, map task, Task Flow, delivery, audit, and agent-run outcomes into it, and expose it through compatible persisted records, runtime DTOs, CLI JSON/status output, and docs while coordinating the delivery-specific subset with #64413.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection, which shows current main has status/prose fields and operational audit codes but no shared blocker/failure taxonomy.

## Solution Assessment

Yes at the design-direction level: a centralized additive taxonomy is the maintainable approach. The exact public contract and rollout boundaries should be chosen by maintainers before implementation so consumers do not branch on another ad hoc field.
