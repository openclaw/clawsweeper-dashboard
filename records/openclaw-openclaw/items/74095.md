## Summary

Keep open. Current main still exposes task status, delivery status, Task Flow waiting/blocked state, operational audit codes, and prose error/summary fields, but I found no shared machine-readable blocker/failure taxonomy; related #64413 remains a narrower delivery-failure request rather than a superseding task-system contract.

## What This Changes

Add a shared blocker/failure classification contract for tasks so blocked, waiting, failed, and transient conditions can be handled consistently across audits, summaries, status output, and future retry logic.

## Best Possible Solution

Define an additive shared classification contract with explicit unknown handling, map task, Task Flow, cron delivery, audit, and agent-run outcomes into it, and expose it consistently through persisted records, runtime DTOs, CLI JSON/status output, and docs while keeping #64413 as the delivery-specific input.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection, which shows current main has status/prose/audit fields but no shared blocker/failure taxonomy.

## Solution Assessment

Yes, a centralized additive taxonomy is the maintainable direction, but the exact contract should be approved first so consumers do not branch on another ad hoc field.
