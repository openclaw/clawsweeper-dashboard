## Summary

Keep open: current main still has coarse task/flow statuses, delivery status, audit codes, and prose error fields, but no shared machine-readable blocker/failure taxonomy. The related #64413 is narrower delivery work, so it does not supersede this broader task-system request.

## What This Changes

Request to add shared blocker/failure classification types and helpers for tasks, then reuse them across audits, summaries, and user-facing status output.

## Best Possible Solution

Add an additive shared taxonomy contract with explicit unknown handling, then map TaskRecord, TaskFlow, delivery, audit, and plugin DTO outcomes to it consistently.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main verifies the existing task, flow, audit, persistence, and plugin runtime surfaces use status/prose fields rather than a shared taxonomy.

## Solution Assessment

Yes at the design-direction level: a centralized additive taxonomy is the maintainable path. The exact codes, persistence model, public DTO shape, and relationship to #64413 need maintainer decisions first.
