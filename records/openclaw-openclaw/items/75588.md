## Summary

Keep open. Current main still lacks the scheduled-session-turn SDK seam, and this draft is the active fresh-main slice replacing the closed consolidated stack, but the latest diff has two merge-blocking cron integration bugs: announce delivery omits the `last` channel required on multi-channel configs, and tag cleanup only scans the first `cron.list` page.

## What This Changes

This draft adds Plugin SDK methods for bundled plugins to schedule future session turns through cron and unschedule plugin-owned scheduled turns by tag, with SDK exports, changelog/API baseline updates, and contract tests.

## Best Possible Solution

Keep the additive SDK seam, but align the host helper with the existing cron contract before merge: use a cron-valid announce target such as `channel: "last"`, paginate tag cleanup across all cron list pages, and lock both behaviors with focused contract/server-path tests.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: `scheduleSessionTurn({ deliveryMode: "announce" })` builds cron params that fail current `cron.add` validation on multi-channel configs, and `unscheduleSessionTurnsByTag` can miss matching jobs whenever they are outside the first 200-job `cron.list` page.

## Solution Assessment

No. The host-mediated scheduled-turn seam is the right boundary for bundled plugins, but this implementation needs to match cron's delivery and pagination contracts before it is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Set the announce channel for scheduled turns:** `src/plugins/host-hook-workflow.ts:207`
  - body: When a plugin passes `deliveryMode: "announce"`, this builds `delivery: { mode: "announce" }`. The cron server requires `delivery.channel` when more than one channel is configured, while the CLI defaults that field to `"last"`; on multi-channel installs the SDK call is rejected before the job is created. Include `channel: "last"` or expose an explicit channel for announce delivery.
  - confidence: 0.88
- **[P2] Page through cron.list during tag cleanup:** `src/plugins/host-hook-workflow.ts:278-283`
  - body: `cron.list` is capped at 200 jobs per page, but `unscheduleSessionTurnsByTag` reads only the first page and removes matches from that slice. Installs with more than 200 cron jobs can leave later jobs with the requested plugin tag scheduled, so tag cleanup is not deterministic. Loop over `hasMore`/`nextOffset` or otherwise scan all pages.
  - confidence: 0.86
