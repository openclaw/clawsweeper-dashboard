## Review Findings

Overall correctness: patch is correct

Overall confidence: 0.82

Full review comments:

- **[P3] Prune records when one-shot jobs auto-delete:** `src/plugins/host-hook-workflow.ts:330-345`
  - body: `scheduleSessionTurn` registers every scheduled turn in host runtime state, and the same call defaults `at`/`delayMs` jobs to `deleteAfterRun`. Cron auto-removes successful one-shot jobs, but this helper only prunes the runtime record on explicit tag unschedule or later lifecycle cleanup, so a long-running plugin that schedules many one-shot turns can accumulate stale in-memory scheduler records until reload. Hook the cron removed/finished path or otherwise delete the plugin runtime record when the cron job auto-deletes.
  - confidence: 0.78
