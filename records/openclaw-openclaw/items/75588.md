## Summary

Keep open. Current main still lacks the scheduled-turn Plugin SDK surface, so this PR is the active implementation slice, but the latest head appears to miss a mixed scheduler cleanup case where dynamic cron-backed turns can be skipped during plugin cleanup.

## What This Changes

The PR adds bundled-plugin `scheduleSessionTurn` and `unscheduleSessionTurnsByTag` APIs, cron-backed host workflow helpers, SDK type exports, API baseline updates, a changelog entry, and scheduled-turn contract tests.

## Best Possible Solution

Keep the additive cron-backed SDK seam, but make scheduled turns participate in the same plugin-host cleanup accounting as existing scheduler jobs before merge, with focused mixed static/dynamic cleanup coverage and green exact-head validation.

## Reproduction Assessment

Yes. A source-level reproduction is to load a bundled plugin that calls both `registerSessionSchedulerJob` and `scheduleSessionTurn`, then run plugin cleanup; the current cleanup records branch handles only `registry.sessionSchedulerJobs`, so the dynamic scheduled turn is skipped.

## Solution Assessment

No. Reusing cron with bundled-origin gating is the right direction, but the implementation should include dynamically scheduled turns in cleanup even when registry snapshot scheduler jobs are present.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep dynamic scheduled turns in cleanup records:** `src/plugins/host-hook-workflow.ts:293`
  - body: This registers scheduled turns only in live host runtime state. During real plugin cleanup, `runPluginHostCleanup` passes `registry.sessionSchedulerJobs` when the plugin has any normal scheduler registration; `cleanupPluginSessionSchedulerJobs` then iterates only that snapshot and returns, so a plugin that uses both `registerSessionSchedulerJob` and `scheduleSessionTurn` can leave the cron-backed turn behind across disable/restart. Please add mixed-case coverage and make cleanup include the dynamic scheduled turns.

## Repair Work Prompt

On PR branch `split/plugin-sdk-scheduled-turns`, fix dynamic scheduled-turn cleanup. Observable bug: if a bundled plugin both registers a normal `registerSessionSchedulerJob` and calls `scheduleSessionTurn`, `runPluginHostCleanup` passes `registry.sessionSchedulerJobs` into `cleanupPluginSessionSchedulerJobs`; because the new scheduled turn is only registered in live runtime state and not in that records array, cleanup skips it on disable/restart. Update the implementation so plugin-host cleanup removes both static registry scheduler jobs and dynamic scheduled-turn jobs without deleting preserved restart jobs. Add a focused contract test that creates both kinds, runs cleanup through `runPluginHostCleanup` or `cleanupReplacedPluginHostRegistry`, and asserts the dynamic cron job is removed. Keep the API additive, keep bundled-origin gating, do not change dependencies/workflows/lockfiles, and do not broaden the Plugin SDK beyond this scheduled-turn seam. The existing changelog entry is probably sufficient unless public behavior text changes.
