## Repair Work Prompt

On PR https://github.com/openclaw/openclaw/pull/75588, fix the stale in-flight tag-unschedule race without broadening the public SDK or changing the cron protocol. `unscheduleSessionTurnsByTag` currently checks active registry state before the async helper, but after `cron.list` awaits it removes all matching jobs without confirming the API closure still owns the active registry generation. Pass a `shouldCommit`/owner guard into `unschedulePluginSessionTurnsByTag` and re-check after listing or before each `cron.remove`, so stale plugin API closures abort instead of deleting replacement-generation jobs with the same plugin/tag/session prefix. Add a focused contract test that starts unschedule from an old registry, flips to a replacement registry that schedules a same-tag/session job while the list is pending, and asserts the replacement job is preserved. Keep the changelog entry single-line and do not touch unrelated workflow/API surfaces.
