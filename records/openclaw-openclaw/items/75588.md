## Summary

Keep this PR open: the scheduled-turn SDK seam is useful and source-backed, but the proposed tag cleanup can over-match tags because tag values are embedded in cron names without escaping and later matched with a plain prefix check.

## What This Changes

This PR adds bundled-plugin `scheduleSessionTurn` and `unscheduleSessionTurnsByTag` SDK APIs backed by cron jobs, runtime cleanup, contract tests, API baseline, changelog, and a Korean locale text fix.

## Best Possible Solution

Keep the cron-backed bundled scheduled-turn seam, but encode or validate tag/name segments so tag cleanup is exact before maintainer SDK/API review and CI decide merge readiness.

## Reproduction Assessment

Yes for the review finding: source inspection shows raw tag values are concatenated into cron names and later cleaned with `startsWith`, which can over-match delimiter-bearing tags. Runtime reproduction was not run because this review was read-only.

## Solution Assessment

No, not yet: reusing the existing cron scheduler is the right narrow direction, but tag cleanup needs exact matching through segment encoding or input validation before this is merge-ready.

## Repair Work Prompt

On PR #75588 (`split/plugin-sdk-scheduled-turns`), fix scheduled-turn tag cleanup so `unscheduleSessionTurnsByTag` only removes jobs with the exact requested plugin-owned tag. The current implementation builds cron names from raw `pluginId`, `tag`, `sessionKey`, and `uniqueId` segments and then filters with `name.startsWith(namePrefix)`, which over-matches delimiter-bearing tags such as `nudge:agent:main:main:extra` when unscheduling `nudge`. Prefer encoding or validating tag/name segments in `src/plugins/host-hook-workflow.ts`; keep the public SDK surface narrow and do not add payload extras or broader workflow seams. Add regression coverage in `src/plugins/contracts/scheduled-turns.contract.test.ts` for a delimiter-bearing tag that must not be removed by a shorter tag. Preserve the existing changelog entry unless the public behavior wording changes.
