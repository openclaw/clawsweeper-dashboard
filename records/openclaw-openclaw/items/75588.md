## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Retain one-shot records until cron removes the job:** `src/plugins/host-hook-workflow.ts:389-394`
  - body: `pruneOneShotSchedulerRecordAfterRun` drops the plugin runtime cleanup record at the scheduled wall-clock time plus a grace period for every delete-after-run `at`/`delayMs` job. Cron only auto-deletes one-shot jobs after an `ok` result and keeps failed jobs for retry or error inspection, so a transient failure can leave the cron job alive after this record is gone; disabling or restarting the plugin later will no longer call `cron.remove`, and the scheduled turn can still retry/fire after the plugin should have cleaned it up.
  - confidence: 0.89
- **[P2] Preserve exact tag boundaries during unschedule cleanup:** `src/plugins/host-hook-workflow.ts:466-470`
  - body: `unscheduleSessionTurnsByTag` still matches raw colon-delimited names with `startsWith(namePrefix)`. A job tagged `foo:agent:main:main:x` in the same session starts with the cleanup prefix for tag `foo`, so unscheduling `foo` can remove the longer-tag job too; encode or length-prefix components, or reject delimiter-bearing tags, before relying on prefix matching.
  - confidence: 0.87
