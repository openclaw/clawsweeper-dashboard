## What This Changes

The PR adds `memory-lancedb-pro` to Active Memory's LanceDB default-tool branch and adds a regression test expecting `memory_recall` for that slot.

## Best Possible Solution

Merge a focused fix that makes `memory-lancedb-pro` use `memory_recall`, keeps explicit `toolsAllow` authoritative, updates the public config docs/help, and includes after-patch deployment proof.

## Reproduction Assessment

Yes, at source level: current main returns `memory_search`/`memory_get` for a `plugins.slots.memory` value of `memory-lancedb-pro`. With a slot that only registers `memory_recall`, the embedded recall run has no callable memory tool; I did not run that external plugin locally.

## Solution Assessment

Mostly yes: a small allowlist covers the reported deployment without disturbing explicit `toolsAllow`. The patch should also update docs/manifest help, while capability-based default discovery can remain a follow-up.
