## Summary

Keep this open. Current main still drops or ignores early session-scoped agent activity for externally started runs, and the related implementation PR #66400 is still the open follow-up rather than a merged fix.

## What This Changes

The issue asks for the TUI to show running/activity state for bridge, webhook, cron, and other system-injected events, not only direct TUI-initiated messages.

## Best Possible Solution

Land #66400 or an equivalent narrow TUI fix that adopts session-matching early lifecycle/tool events before the known-run guard while preserving cross-session and `/btw` isolation.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to send `handleAgentEvent` a new lifecycle `start` event for the current session before any matching chat event; current main returns at the unknown-run guard and never sets the TUI status to running.

## Solution Assessment

Yes. The narrowest maintainable direction is a targeted TUI adoption fix using the session key already supplied by gateway events, not a gateway rewrite or new configuration option.
