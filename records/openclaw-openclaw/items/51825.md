## Summary

Keep this open. Current main and the latest shipped release still drop same-session TUI agent lifecycle/tool events when they arrive before a chat event has registered the run, and the focused fix candidate remains open at https://github.com/openclaw/openclaw/pull/66400.

## What This Changes

Show TUI running or elapsed activity for agent work triggered by bridge, webhook, or cron events instead of only direct TUI messages.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/66400 or an equivalent TUI handler change that pre-registers same-session agent events before unknown-run filtering while preserving cross-session, BTW, and optimistic-message protections.

## Reproduction Assessment

Yes, at source level: deliver a current-session agent lifecycle start or tool start event with a new runId before any chat event registers that run, and current main returns at the unknown-run guard before updating activity or rendering a tool card.

## Solution Assessment

Yes. Using the gateway-provided sessionKey to adopt same-session early agent events in the TUI is the narrowest maintainable fix and avoids a gateway rewrite or new configuration surface.
