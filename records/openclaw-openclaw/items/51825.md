## Summary

Keep this open. Current main still drops current-session agent lifecycle/tool events when they arrive before any chat event has registered the run, so the reported invisible activity state remains source-reproducible.

## What This Changes

The issue requests that the TUI show activity for system-injected agent runs such as bridge notifications, webhooks, and cron triggers.

## Best Possible Solution

The end state should be a narrow TUI fix that adopts current-session agent events by gateway-provided `sessionKey` before the known-run guard while preserving cross-session and `/btw` isolation.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deliver a current-session lifecycle `start` or tool `start` agent event before any chat event registers that run ID, and current main returns at the unknown-run guard.

## Solution Assessment

Yes. The narrowest maintainable solution is to accept current-session `agent` events before known-run filtering using `sessionKey`; the related #66400 PR appears aimed at that shape.
