## Summary

Keep this open. Current main and latest release still reject early same-session TUI agent lifecycle/tool events before a chat event registers the run, and the focused open fix path remains https://github.com/openclaw/openclaw/pull/66400.

## What This Changes

The issue requests a TUI activity indicator for agent work started by system-injected events such as bridge notifications, webhooks, and cron triggers.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/66400 or an equivalent TUI handler change that adopts same-session early agent events while preserving cross-session, BTW, and optimistic-message protections.

## Reproduction Assessment

Yes, at source level. Deliver a current-session `agent` lifecycle start or tool start event with a new runId before any `chat` event registers that run, and current main returns at the unknown-run guard before updating activity.

## Solution Assessment

Yes. Using gateway-provided session identity to adopt same-session early agent events in the TUI is the narrowest maintainable fix and avoids a new config surface or gateway rewrite.
