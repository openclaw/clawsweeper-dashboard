## Summary

Keep open: current main still has the uncached provider-usage path, and this PR is the active implementation track, but the branch is not mergeable as-is because it predates current provider-runtime context handling and lacks real OAuth proof.

## What This Changes

The PR adds a root `usage.cacheTtlMs` setting and an in-memory TTL cache around provider usage summaries with focused cache tests.

## Best Possible Solution

Keep a loader-level TTL cache only if it preserves current runtime inputs, isolates entries by the relevant auth/plugin/config boundaries, avoids caching transient failures, and includes focused regression tests plus real OAuth status-call proof.

## Reproduction Assessment

Yes, at source level: status paths call `loadProviderUsageSummary`, and Anthropic usage fetch calls the live OAuth usage endpoint each time. I did not establish a live 429 in this review, so the rate-limit symptom remains source-backed rather than live reproduced.

## Solution Assessment

No, not as currently proposed. A loader-level TTL cache is the right narrow direction, but this branch's cache resolution and keying are not compatible with current provider-runtime/context boundaries.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve the current loader context before caching:** `src/infra/provider-usage.load.ts:76-80`
  - body: Current main callers can pass `config`, `env`, `workspaceDir`, and credential-source policy into `loadProviderUsageSummary`, and the loader forwards that context into provider usage hooks. This patch reads ambient config and is based on the older loader shape, so a rebased version can resolve auth and provider snapshots under the wrong runtime context.
  - confidence: 0.9
- **[P2] Key cached usage by every runtime boundary:** `src/infra/provider-usage.load.ts:30-33`
  - body: The cache key only includes `agentDir` and provider ids. On current main the same provider summary can differ by config, env, workspace plugin resolution, credential-source policy, and provider usage hooks, so one status call can receive another context's cached usage or plan data until the TTL expires.
  - confidence: 0.86
