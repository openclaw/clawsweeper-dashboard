## Best Possible Solution

Land a fresh or rebased provider-usage cache only after it is scoped or bypassed for every auth/snapshot runtime input, avoids caching transient failures, documents the public config, and includes real Anthropic OAuth proof.

No, not as currently proposed. A loader-level cache is plausible, but it must be rebased and scoped to all runtime inputs that affect provider usage output before it is the maintainable fix.
