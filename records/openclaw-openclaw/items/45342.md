## Summary

Keep open: this PR tracks the still-unimplemented provider usage cache work, but the current branch is not mergeable as-is because it lacks real OAuth proof and its cache does not preserve current provider runtime boundaries.

## What This Changes

The PR adds a configurable in-memory TTL cache around provider usage summaries and updates usage config schema/help plus focused provider-usage tests.

## Best Possible Solution

Rework the cache on current main so it preserves loader inputs, isolates entries by every context that can change provider usage output, avoids caching transient failures, and includes focused regression tests plus real OAuth status-call proof.

## Reproduction Assessment

Yes, at source level: status and gateway usage paths call loadProviderUsageSummary, and Anthropic usage fetching performs a live OAuth usage request. I did not establish a live 429 in a real OAuth setup, so the rate-limit symptom remains source-backed rather than live reproduced.

## Solution Assessment

No, not as currently proposed. A loader-level TTL cache is the right narrow direction, but this branch is based on an older loader shape and does not preserve current config/env/workspace/plugin boundaries.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve loader context before caching:** `src/infra/provider-usage.load.ts:76-80`
  - body: Current main callers can pass config, env, workspaceDir, and credential-source policy into loadProviderUsageSummary, and the loader forwards that context into provider auth and usage hooks. This patch reads ambient config and resolves auth with the older providers/auth/agentDir shape, so a rebased cache can fetch or return summaries under the wrong runtime context.
  - confidence: 0.9

On current main, provider usage output can vary by config, env, workspace plugin resolution, credential-source policy, and provider usage hooks, so one status call can receive another context's cached usage or plan data until the TTL expires.
  - confidence: 0.86
