## Summary

Keep open: current main still lacks provider usage caching, but this PR is not merge-ready because the branch predates the context-aware usage loader, the cache boundary is too narrow, and real Anthropic OAuth proof is missing.

## What This Changes

The PR adds a configurable in-memory TTL cache for provider usage summaries plus config schema/help/label/type updates and cache-focused provider usage tests.

## Best Possible Solution

Land a rebased, context-isolated provider usage cache that preserves caller-supplied config/env/workspace/auth policy and includes redacted real Anthropic OAuth proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes at source level: current main status and gateway usage paths still call the uncached loader, and the Anthropic fetcher performs a live usage request per snapshot; I did not reproduce a live 429.

## Solution Assessment

Is this the best way to solve the issue? No, not as currently proposed; a loader-level cache is plausible, but this branch must be rebased around current runtime context inputs and validated in a real OAuth setup.
