Full review comments:

- **[P1] Key cached usage by runtime context:** `src/infra/provider-usage.load.ts:83-85`
  - body: The cache is entered whenever `opts.auth` is absent, but the key only includes `agentDir` and provider ids. On current main, usage resolution also depends on `opts.config`, `opts.env`, `workspaceDir`, credential-source policy, and provider runtime hooks, so one context can receive another context's usage or plan data within the TTL. Include those runtime inputs in the cache key or bypass the cache when they are injected.
  - confidence: 0.92
- **[P1] Preserve the current usage-loader context:** `src/infra/provider-usage.load.ts:76-80`
  - body: This patch introduces an ambient `loadConfig()` path and does not carry the current loader's `config`, `env`, `workspaceDir`, and `skipPluginAuthWithoutCredentialSource` options through auth and provider usage resolution. A rebase must keep those current-main inputs flowing into `resolveProviderAuths` and `resolveProviderUsageSnapshotWithPlugin`; otherwise status usage can resolve the wrong credentials or skip provider-owned behavior.
  - confidence: 0.86
