## Summary

## What This Changes

The issue reports that macOS LaunchAgent generation writes gateway stderr to `/dev/null`, hiding launchd-level diagnostics that should go to a real gateway stderr log.

## Best Possible Solution

Review and either land or revise https://github.com/openclaw/openclaw/pull/90828 so the plist writer, status/status-all/doctor/runtime hints, docs, tests, retention decision, and real macOS launchd proof agree on the intended stderr contract.
