## Summary

Keep open. Current main and the latest release still generate macOS LaunchAgent plists with `StandardErrorPath` set to `/dev/null`, and https://github.com/openclaw/openclaw/pull/90828 is the open candidate fix that should be reviewed or revised before this issue closes.

## What This Changes

The issue reports that macOS LaunchAgent generation writes `StandardErrorPath` as `/dev/null`, discarding gateway stderr diagnostics instead of writing them to `gateway.err.log`.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/90828; this issue should close only after current main generates a real launchd stderr log path and diagnostics agree with that contract.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: current main and `v2026.6.9` still pass `/dev/null` as the launchd plist stderr path for install and restart rewrites; I did not run a live macOS launchd install in this read-only review.

## Solution Assessment

Partly. The linked PR is the likely right owner-boundary fix because it changes the daemon launchd writer and adjacent diagnostics together, but it still needs PR review, real macOS proof, and maintainer logging/retention judgment before merge.
