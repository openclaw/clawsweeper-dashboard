## Summary

Keep open. Current main and the latest release still generate the macOS Gateway LaunchAgent with `StandardErrorPath` set to `/dev/null`; the linked closing PR is open and owns the implementation path, so this issue should stay open until that PR lands or is replaced.

## What This Changes

The issue asks OpenClaw to preserve macOS launchd gateway stderr in a real log file instead of discarding it through the generated LaunchAgent plist.

## Best Possible Solution

Land or revise https://github.com/openclaw/openclaw/pull/90828 so the plist writer, diagnostics, status/status-all output, docs, and tests all agree on the restored stderr-file contract with real macOS proof, then close this issue through the merged PR.
