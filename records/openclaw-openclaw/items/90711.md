## Summary

Keep open. Current main and the latest release still generate macOS LaunchAgent plists with `StandardErrorPath` set to `/dev/null`, and https://github.com/openclaw/openclaw/pull/90828 is the open closing PR that should be reviewed, revised, or landed before this issue closes.

## What This Changes

The issue asks OpenClaw to stop discarding macOS launchd gateway stderr and write it to a real gateway error log instead.

## Best Possible Solution

Review and either land or revise https://github.com/openclaw/openclaw/pull/90828 so plist generation, status/status-all/doctor/runtime hints, docs, retention expectations, and real macOS proof agree on the intended stderr contract.

## Reproduction Assessment

Yes, source-level: current main still passes `/dev/null` as the LaunchAgent `StandardErrorPath` in both install and restart rewrite paths, and the reporter added live macOS reproduction comments. I did not run a live LaunchAgent install in this read-only review.

## Solution Assessment

Yes, routing stderr to the resolved launchd error log is the narrow daemon-layer fix, but the mergeable solution also needs aligned diagnostics/docs and real macOS proof. A new separate fix job is not the best path while the linked PR already owns the implementation.
