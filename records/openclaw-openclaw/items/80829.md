## Summary

Keep open: this PR targets a real current-main silent-drop path, but it still misses the default group/channel delivery path, duplicates narrower error classification, and lacks after-fix real behavior proof.

## What This Changes

Adds a followup billing/quota/rate-limit failure classifier, sends a single followup error notice before returning from the runner catch block, and adds regression tests for that path.

## Best Possible Solution

Route billing/quota/rate-limit followup failures through origin delivery even when normal source replies are tool-only, using shared failover classification and safe user-facing copy.

## Reproduction Assessment

Yes, from source inspection and the linked gateway logs: current main logs the followup provider failure and returns before any payload is sent. I did not run a live Slack reproduction in this read-only review.

## Solution Assessment

No. The PR is directionally useful, but it suppresses the notice on the default group/channel path and should reuse the existing failover classifiers instead of adding a narrower local list.
