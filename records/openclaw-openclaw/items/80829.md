## Summary

Keep this PR open: it targets a real followup silent-drop bug, but the current patch still skips the default group/channel path described by the linked Slack report and duplicates narrower error matching instead of reusing the existing billing/rate-limit classifiers. It also still lacks after-fix real channel or runtime proof, and a human triage comment already asked for those same revisions.

## What This Changes

The PR adds a followup billing/quota/rate-limit error classifier, sends a short error payload on matched followup failures, and adds regression tests for that path.

## Best Possible Solution

Surface followup billing/quota/rate-limit failures through the origin delivery route even when normal source replies are message-tool-only, using the shared failover classifiers and existing safe error-notice patterns.

## Reproduction Assessment

Yes, from source inspection and the linked issue logs: current main catches followup fallback failure, logs it, and returns without delivery. I did not establish a live Slack reproduction in this read-only review.

## Solution Assessment

No. The PR moves in the right direction, but it suppresses the notice on the default group/channel path and maintains a narrower local classifier instead of reusing the shared billing and rate-limit classification contract.
