## Summary

Keep this PR open: it addresses a real current-main silent-drop bug, but the proposed patch still misses the default group/channel path and duplicates a narrower error classifier than the one already used by model failover.

## What This Changes

The PR adds a followup-runner billing/quota/rate-limit classifier, sends a generic error payload for matched followup failures, and adds regression tests for that path.

## Best Possible Solution

Surface billing/quota/rate-limit followup failures through the originating delivery route even when automatic source replies are suppressed, using the shared failover classifiers and source-suppression-safe error payload metadata.

## Reproduction Assessment

Yes, from source inspection and the linked logs: current main catches the followup model-fallback error, logs it, and returns before payload delivery. I did not run a live Slack reproduction in this read-only review.

## Solution Assessment

No. The PR is aimed at the right catch block, but it suppresses the notice on the default group/channel `message_tool_only` path and should reuse the shared billing/rate-limit classifiers instead of a narrower local whitelist.
