## Summary

Keep this PR open: it targets a real followup silent-drop bug, but the current patch still misses the default group/channel path and duplicates narrower billing/rate-limit matching instead of reusing the shared classifiers. It also still lacks after-fix real channel/runtime proof.

## What This Changes

The PR adds a local followup billing/quota/rate-limit classifier, sends a generic error payload for matched followup failures, and adds followup-runner tests.

## Best Possible Solution

Surface billing/quota/rate-limit followup failures through the origin delivery route even when normal source delivery is message-tool-only, using shared billing/rate-limit or fallback-summary classifiers and real channel/runtime proof.

## Reproduction Assessment

Yes, from source inspection and the linked logs: current main's followup catch logs the model-fallback error and returns without sending payloads. I did not run a live Slack reproduction in this read-only review.

## Solution Assessment

No. The proposed patch suppresses the notice on the default group/channel `message_tool_only` path and uses a local classifier that is narrower than the shared failover classifiers.
