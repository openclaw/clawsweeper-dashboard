## Summary

The PR is a plausible fix for the silent followup failure path, but it should stay open because the current patch still suppresses the new notice for `message_tool_only` runs and duplicates narrower billing/rate-limit classification instead of using the shared failover helpers.

## What This Changes

The PR adds a local followup quota/billing/rate-limit classifier, sends a short error payload from the followup-runner catch block, and adds regression tests for that behavior.

## Best Possible Solution

Surface billing/quota/rate-limit followup failures as internal runtime notices through the origin delivery route while preserving intentional room-event/send-policy suppression, and reuse the shared failover classifiers with focused regression coverage.

## Reproduction Assessment

Yes, from source inspection: current `main` catches followup model-fallback failure, logs it, and returns without sending any payload. I did not run a live Slack reproduction in this read-only review, but the linked issue includes matching gateway logs for the Anthropic extra-usage billing path.

## Solution Assessment

No, not yet. The intended local catch-block fix is reasonable, but the current patch should not suppress runtime failure notices for all `message_tool_only` runs and should reuse the shared failover classifiers.
