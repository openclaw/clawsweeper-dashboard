## Summary

Keep open: the PR targets the right followup failure path, but the current diff still suppresses the notice for the group/channel delivery mode implicated by the linked Slack workflow and lacks required after-fix real behavior proof.

## What This Changes

Adds a followup billing/quota/rate-limit classifier, sends a generic error payload on matched followup failures, and adds regression tests for the new path.

## Best Possible Solution

Surface billing/quota/rate-limit followup failures through the origin delivery route even when automatic source delivery is suppressed, preferably by reusing the existing failover classifiers and safe billing/rate-limit copy.

## Reproduction Assessment

Yes, from source inspection: current main's followup catch logs the model-fallback error and returns without sending payloads. I did not run a live Slack reproduction in this read-only review, but the linked issue includes matching gateway logs for the Anthropic extra-usage billing path.

## Solution Assessment

No. The current PR sends a notice for automatic/direct delivery but suppresses it for `message_tool_only`, which is the default group/channel path and matches the reported Slack workflow; it should bypass that suppression for error notices and use the shared classifiers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Don’t suppress failure notices for message-tool-only turns:** `src/auto-reply/reply/followup-runner.ts:384-386`
  - body: The new guard skips the billing/quota notice whenever `sourceReplyDeliveryMode` is `message_tool_only`. Group/channel turns default to that mode, and the linked Slack multi-agent repro is a channel/thread flow, so this would still silently drop the failure on the path this PR is trying to fix. Failure notices should bypass automatic source-delivery suppression while normal successful followup payloads remain private.
  - confidence: 0.88
- **[P2] Reuse the shared billing and rate-limit classifiers:** `src/auto-reply/reply/followup-runner.ts:50-58`
  - body: The new substring list misses billing errors already recognized elsewhere, such as `payment required`, `insufficient credits`, and `credit balance`, so those followup failures would still be silent. Reuse the existing failover classifiers or `FallbackSummaryError` attempt reasons instead of maintaining a narrower local whitelist.
  - confidence: 0.82
