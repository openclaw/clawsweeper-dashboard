## Summary

Keep open: the linked silent-drop bug is valid, but this PR would still suppress the notice on the default group/channel path and duplicates a narrower error classifier. It also lacks after-fix real behavior proof for the external PR gate.

## What This Changes

The PR adds a local followup billing/quota/rate-limit classifier, sends a generic error payload for matched followup failures outside message_tool_only mode, and adds regression tests.

## Best Possible Solution

Surface safe billing/quota/rate-limit followup failure notices through origin delivery even when automatic source replies are suppressed, reusing shared failover classifiers and copy, then prove the behavior in a real channel or runtime path.

## Reproduction Assessment

Yes, from source inspection and the linked logs: current main's followup catch logs the model-fallback failure and returns before sending payloads. I did not run a live Slack reproduction in this read-only review.

## Solution Assessment

No. The PR follows the right local failure path, but it suppresses the notice for default group/channel message_tool_only turns and uses a narrower classifier than existing failover helpers.
