## Summary

Keep open. Current main still has Claude CLI stream parsing and generic display-only tool cards, but no Control UI picker state or structured `AskUserQuestion` result path back into the live Claude CLI turn.

## What This Changes

The issue requests Control UI support for Claude CLI `AskUserQuestion` calls so users can select options and return structured answers instead of stalling the turn.

## Best Possible Solution

Define a generic HITL tool-result contract for Claude CLI sessions, parse `AskUserQuestion`, render a Control UI picker, submit selected answers back to the live turn, and cover it with focused parser/UI tests plus live Claude CLI proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: start a Control UI session backed by `claude-cli` and cause Claude Code to emit `AskUserQuestion`; current main has no parser, picker, or result-submission branch for that tool.

## Solution Assessment

No as a one-line UI fix. The right direction is the requested picker plus a structured result path back into Claude CLI, implemented through a small generic HITL contract rather than only restyling the existing tool card.
