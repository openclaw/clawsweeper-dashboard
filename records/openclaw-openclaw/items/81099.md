## Summary

Keep open. Current main still treats Claude CLI `AskUserQuestion` as a generic tool call with no parser, picker, or result-submission path, so the reported Control UI turn stall remains valid.

## What This Changes

The issue asks OpenClaw's Control UI and `claude-cli` bridge to render Claude Code `AskUserQuestion` prompts and return structured selections to the active turn.

## Best Possible Solution

Define a small Claude CLI HITL/result contract, parse `AskUserQuestion`, render a Control UI picker, submit structured answers back into the active Claude turn, and cover parser/UI plus live Claude CLI behavior.

## Reproduction Assessment

Yes by source inspection. A Control UI session backed by `claude-cli` that emits `AskUserQuestion` reaches generic tool rendering and has no current-main branch that returns a structured tool result to Claude.

## Solution Assessment

No as a narrow UI-only rendering patch. The maintainable fix needs both a picker UI and a cross-boundary result-submission path that matches Claude CLI's HITL protocol.
