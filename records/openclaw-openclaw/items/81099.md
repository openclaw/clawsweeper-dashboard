## Summary

Keep this issue open: current main and the latest release still handle Claude live-session `can_use_tool` requests through a generic exec allow/deny response, with no `AskUserQuestion` prompt/answer bridge. The active implementation path is the open, unmerged PR at https://github.com/openclaw/openclaw/pull/91544, which should be reviewed and landed before this issue closes.

## What This Changes

Claude CLI `AskUserQuestion` calls should render an interactive prompt in the Control UI or reply surface and return structured answer data instead of stalling.

## Best Possible Solution

Review, repair, and land https://github.com/openclaw/openclaw/pull/91544 or a replacement that makes the Claude prompt visible and routes the user's answer back to the paused Claude turn; a richer inline picker can follow if maintainers require first-party UI parity.
