## Summary

Keep open. Current main has Claude CLI stream parsing and generic tool-card display, but no `AskUserQuestion` picker, selection state, or tool-result submission path; the closed skill-prompt report is related context but does not cover this Claude CLI Control UI gap.

## What This Changes

The issue requests Control UI support for claude-cli `AskUserQuestion` by rendering an inline picker and returning the selected answer as a tool result.

## Best Possible Solution

Define a generic interactive tool-result contract for Claude CLI sessions, parse `AskUserQuestion` events, render the Control UI picker, submit selected values back to the live CLI turn, and cover it with focused parser/UI tests plus live Claude CLI proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: start a Control UI session on `claude-cli` and force an `AskUserQuestion` call. Current main has parser/display paths for text/tool cards but no interactive answer submission path.

## Solution Assessment

Unclear as a one-line fix; rendering a picker alone is not enough because Claude CLI also needs a structured result submitted back into the live turn. The safest direction is a small, documented HITL tool-result contract rather than special-casing only card styling.
