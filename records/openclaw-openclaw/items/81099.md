## Summary

Keep open. Current main still has no `claude-cli` `AskUserQuestion` picker or result-submission path: the Claude stream parser only resolves result/text records, the live-session stdin path only sends normal user messages, and the Control UI renders generic tool-call/tool-result cards.

## What This Changes

The issue asks OpenClaw Control UI to render Claude CLI `AskUserQuestion` prompts as interactive pickers and return structured selections to the active Claude CLI turn.

## Best Possible Solution

Define a small Claude CLI HITL/result contract, parse `AskUserQuestion`, render a Control UI picker, submit structured answers back to the live Claude turn, and cover parser/UI plus live Claude CLI behavior.

## Reproduction Assessment

Yes by source inspection. A Control UI session backed by `claude-cli` that emits `AskUserQuestion` reaches only generic tool rendering and no tool-result response branch on current main.

## Solution Assessment

No as a narrow UI-only rendering patch. The maintainable solution needs both the picker UI and the cross-boundary result-submission path back into Claude CLI.
