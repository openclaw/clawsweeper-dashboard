## Summary

Keep open. Current main still accepts `claude-cli` stream-json output only as generic tool-call display and has no `AskUserQuestion` picker or structured result-submission path back into the active Claude turn.

## What This Changes

The issue asks OpenClaw Control UI to render Claude CLI `AskUserQuestion` tool calls as interactive pickers and return structured selections so the stalled turn can continue.

## Best Possible Solution

Define a small Claude CLI HITL/result contract, surface `AskUserQuestion` as a pending Control UI picker, submit structured selections back to the active Claude turn, and cover parser/UI plus live Claude CLI behavior.

## Reproduction Assessment

Yes by source inspection. A `claude-cli` Control UI session that emits `AskUserQuestion` reaches generic tool-call rendering, while the live Claude stdin/result path has no structured answer branch.

## Solution Assessment

No as a narrow UI-only rendering fix. The maintainable repair needs both the picker UI and a defined result-submission path back across the Claude CLI live-session boundary.
