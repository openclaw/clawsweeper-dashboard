## Summary

Keep open. Current main still treats `AskUserQuestion` from `claude-cli` as a generic tool call: the Claude live-session code only resolves final `result` records, Control UI only renders input/output tool cards, and there is no picker submission path back into the active Claude CLI turn.

## What This Changes

Support Claude CLI `AskUserQuestion` tool calls in Control UI by rendering option pickers and returning structured selections to the active `claude-cli` turn.

## Best Possible Solution

Define a small Claude CLI HITL/result contract, parse pending `AskUserQuestion` requests, render a Control UI picker, submit structured answers back into the live Claude turn, and cover parser/UI behavior plus one live Claude CLI proof.

## Reproduction Assessment

Yes, by source inspection. A Control UI session backed by `claude-cli` that emits `AskUserQuestion` reaches generic tool rendering, while the live Claude stdin path only sends normal user messages and has no tool-result response branch.

## Solution Assessment

No as a UI-only card change. The maintainable fix needs both the picker UI and a verified Claude CLI result-submission path across the parser, live-session, gateway, and Control UI boundary.
