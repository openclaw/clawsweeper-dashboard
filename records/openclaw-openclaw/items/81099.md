## Summary

Keep open. Current main still lacks a Claude CLI `AskUserQuestion` picker and result-submission path: the Claude live parser ignores non-result JSONL after recording it, the Control UI builds generic display-only tool cards, and `chat.send` accepts only normal user messages.

## What This Changes

Add Control UI and Claude CLI harness support for rendering `AskUserQuestion` choices and returning structured selections to the active Claude turn.

## Best Possible Solution

Define a small Claude CLI interactive-tool contract at the `claude-cli` boundary, parse `AskUserQuestion`, render a Control UI picker, submit structured answers back to the live turn, and cover parser/UI plus live Claude CLI behavior.

## Reproduction Assessment

Yes by source inspection. A `claude-cli` session that emits `AskUserQuestion` reaches only generic tool rendering and the live Claude loop has no tool-result response branch on current main.

## Solution Assessment

No as a narrow UI-only patch. The maintainable fix needs both the picker UI and a Claude CLI result-submission path owned at the backend/gateway boundary.
