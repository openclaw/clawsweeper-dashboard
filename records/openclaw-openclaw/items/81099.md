## Summary

Keep open. Current main still lacks an end-to-end `claude-cli` `AskUserQuestion` flow: the CLI stream parser ignores non-text tool events, the live stdin path only sends normal user messages, and Control UI projects generic tool cards without a picker or submit path.

## What This Changes

Render Claude CLI `AskUserQuestion` calls as inline Control UI picker prompts and return structured selections to the active Claude turn.

## Best Possible Solution

Define an end-to-end Claude CLI HITL contract, parse the native question request, render a Control UI picker, submit structured answers back to the active Claude turn, and cover parser/UI plus live Claude CLI behavior.

## Reproduction Assessment

Yes, by source inspection. A `claude-cli` Control UI turn that emits `AskUserQuestion` reaches generic tool rendering and there is no current-main branch that returns a structured tool result to the active Claude turn.

## Solution Assessment

No, not as a UI-only picker patch. The maintainable solution needs both the picker affordance and the cross-boundary result-submission path back into Claude CLI using its actual dependency contract.
