## Summary

Keep open. Current main still has no Claude CLI AskUserQuestion HITL path: parsing only produces text/result output, the live Claude stdin path only writes ordinary user messages, and Control UI tool cards are display-only input/output records.

## What This Changes

The issue asks OpenClaw Control UI to render Claude CLI AskUserQuestion prompts as interactive pickers and return the selected answers to the active Claude turn.

## Best Possible Solution

Define a small Claude CLI HITL/result contract, surface AskUserQuestion as a Control UI picker, submit selected labels back through the active Claude turn, and cover parser/UI behavior plus one live Claude CLI proof.

## Reproduction Assessment

Yes, by source inspection rather than a live run. Start a Control UI session backed by claude-cli and trigger AskUserQuestion; current main has no parser branch, picker UI, or answer-submission path for that tool.

## Solution Assessment

No as a narrow UI-only patch. The maintainable fix needs a cross-boundary Claude CLI HITL/result path so the picker answer becomes the expected tool result, not only a different rendering of the existing tool card.
