## Summary

Keep open. Current main still has a Claude CLI text/result parser and display-only Control UI tool cards, but no picker state or structured `AskUserQuestion` result path back into the running Claude CLI turn.

## What This Changes

Render Claude CLI `AskUserQuestion` calls as inline Control UI pickers and return structured selections to the live Claude CLI session.

## Best Possible Solution

Define a generic HITL tool-result contract for Claude CLI sessions, render `AskUserQuestion` as a Control UI picker, submit selected answers back to the live turn, and cover it with focused parser/UI tests plus live Claude CLI proof.

## Reproduction Assessment

Yes, from source inspection rather than a live run. A Control UI session backed by `claude-cli` that emits `AskUserQuestion` will flow through code that ignores non-text/non-result stream records and has no picker or structured result submission branch.

## Solution Assessment

No, not as a narrow UI-only fix. The requested picker is the right user outcome, but the maintainable solution needs a small cross-boundary HITL result contract instead of only restyling the existing generic tool card.
