## Summary

Keep open. Current main still supports warm claude-cli sessions and generic Control UI tool cards, but it does not expose a claude-cli AskUserQuestion request as an interactive picker or route a structured answer back into the active Claude turn.

## What This Changes

The issue requests Control UI support for claude-cli AskUserQuestion tool calls, including inline option picking and structured result submission.

## Best Possible Solution

Define a small Claude CLI HITL/result contract, render an inline Control UI picker for AskUserQuestion, submit structured selections back into the active Claude CLI turn, and cover parser/UI plus live claude-cli behavior.

## Reproduction Assessment

Yes, by source inspection. A claude-cli stream-json AskUserQuestion record would currently pass through generic tool rendering without any branch that renders a picker or writes a structured tool result back to the live Claude process.

## Solution Assessment

No, not as a narrow UI-only patch. The maintainable fix needs both Control UI picker state and a verified Claude CLI result-submission path across the live-session boundary.
