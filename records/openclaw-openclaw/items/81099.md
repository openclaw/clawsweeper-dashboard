## Summary

Keep open. Current main still treats Claude CLI stream-json tool-use output and Control UI tool cards as display-only, so the reported `AskUserQuestion` flow still lacks both an inline picker and a structured result path back into the live Claude CLI turn; the closed skill request at https://github.com/openclaw/openclaw/issues/65681 is related context but does not supersede this backend/UI bug.

## What This Changes

The issue requests Control UI support for Claude CLI's built-in `AskUserQuestion` tool, including inline single/multi-select UI and structured answer submission.

## Best Possible Solution

Add a small generic HITL request/result contract for Claude CLI sessions, render `AskUserQuestion` pickers in Control UI, and return selected answers to the live CLI turn with focused parser/UI tests plus live Claude CLI proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: in a Control UI session backed by `claude-cli`, an `AskUserQuestion` tool-use event is not parsed into an interactive request, and follow-up input is written only as a normal user message with no tool-use parent/result.

## Solution Assessment

Yes with a scope adjustment. The requested picker and structured answer are the right user outcome, but the maintainable fix should be a generic HITL bridge across Claude live-session handling and Control UI rather than only a special-case tool-card renderer.
