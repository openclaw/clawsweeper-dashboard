## Summary

Keep open. Current main still lacks both halves of the reported path: Control UI only renders generic tool input/output cards, and the Claude live-session bridge only writes ordinary user messages back to Claude CLI, so an `AskUserQuestion` call can still stall without a structured result.

## What This Changes

The issue requests structured Control UI handling for `claude-cli` `AskUserQuestion` calls so a picker is shown and selected answers are returned to the active Claude turn.

## Best Possible Solution

Define a small Claude CLI HITL/result contract, parse `AskUserQuestion` requests, render a Control UI picker, submit structured answers back to Claude CLI, and cover the parser, UI, gateway contract, and a live Claude CLI scenario.

## Reproduction Assessment

Yes, by source inspection. A `claude-cli` Control UI session that emits `AskUserQuestion` reaches generic tool rendering, and the current live-session bridge has no structured tool-result write path back to Claude CLI.

## Solution Assessment

No, not as a narrow UI-only picker patch. The maintainable fix needs both the picker UI and the cross-boundary result-submission contract back into the active Claude CLI turn.
