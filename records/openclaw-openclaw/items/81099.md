## Summary

Keep open. Current main still treats Claude CLI `AskUserQuestion` as generic tool/event JSON: the parser, live-stdin path, and Control UI rendering path have no picker state or structured tool-result return, so the reported stall remains source-reproducible.

## What This Changes

The issue asks for Claude CLI `AskUserQuestion` calls in Control UI to render an inline picker and return structured selections instead of stalling as generic tool input.

## Best Possible Solution

Define a small Claude CLI HITL/result contract, parse `AskUserQuestion`, render a Control UI picker, submit structured answers back to the live Claude turn, and cover parser/UI plus live Claude CLI behavior.

## Reproduction Assessment

Yes by source inspection. A Control UI session backed by `claude-cli` that emits `AskUserQuestion` reaches generic tool rendering and has no current-main branch that sends a structured tool result back to Claude CLI.

## Solution Assessment

No as a narrow UI-only rendering patch. The maintainable fix needs both the picker UI and the cross-boundary result-submission path that matches Claude CLI's dependency contract.
