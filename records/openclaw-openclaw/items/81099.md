## Summary

Keep open. Current main still only displays `claude-cli` `AskUserQuestion` as a generic tool call and has no Control UI picker or structured result-submission path back into the active Claude CLI turn.

## What This Changes

The issue asks OpenClaw to handle Claude CLI `AskUserQuestion` tool calls in Control UI by rendering a picker and returning the selected answer as the tool result.

## Best Possible Solution

Define a small Claude CLI HITL/result contract, parse `AskUserQuestion` as pending interaction state, render a Control UI picker, submit structured selections back into the active Claude turn, and cover parser, UI, gateway contract, and live Claude CLI behavior.

## Reproduction Assessment

Yes, by source inspection. A `claude-cli` stream-json `AskUserQuestion` reaches generic tool rendering while the live-session code has no pending-question or tool-result response path.

## Solution Assessment

No as a narrow UI-only rendering patch. The maintainable fix needs both the picker UI and a verified cross-boundary result-submission path back into Claude CLI.
