## Summary

Keep open. Current main still lacks a Claude CLI AskUserQuestion picker and result-submission path: the CLI parser ignores non-result interactive records, live stdin only writes normal user messages, and Control UI tool cards render generic input/output instead of an answerable prompt.

## What This Changes

The issue requests Control UI support for the Claude CLI built-in AskUserQuestion tool so selections are rendered and returned to the active Claude turn.

## Best Possible Solution

Define a small Claude CLI HITL/result contract, parse AskUserQuestion into a pending Control UI picker, submit structured answers back to the live Claude turn, and cover parser/UI plus a live Claude CLI proof.

## Reproduction Assessment

Yes by source inspection. A claude-cli-backed Control UI session that emits AskUserQuestion reaches generic tool rendering, while current main has no parser, UI submit, or stdin tool-result branch for returning the answer.

## Solution Assessment

No for a narrow UI-only patch. The maintainable fix needs both the visible picker and the cross-boundary result-submission path back into Claude CLI.
