## Summary

Keep open. Current main still lacks a Claude CLI `AskUserQuestion` picker and tool-result submission path: the supported `claude-cli` live backend parses only final results/text deltas, stdin writes only normal user messages, and the Control UI renders generic display-only tool cards.

## What This Changes

Render Claude CLI `AskUserQuestion` prompts in Control UI as option pickers and submit structured selections back as tool results.

## Best Possible Solution

Add a small Claude CLI HITL contract across the live-session parser, gateway/UI event model, and Control UI picker, with focused parser/UI tests and live Claude CLI proof.

## Reproduction Assessment

Yes, by source inspection. A `claude-cli` stream-json `AskUserQuestion` tool-use reaches only generic parsing/display paths on current main, and stdin has no tool-result response branch.

## Solution Assessment

No, not as a narrow UI-only picker. The maintainable fix needs both visible picker UI and a backend/gateway path that submits structured answers to the active Claude CLI tool call.
