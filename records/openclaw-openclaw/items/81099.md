## Summary

Keep open. Current main still treats Claude CLI tool-use stream events as generic/display-only tool cards and has no structured picker/result path back to the live Claude CLI session; the related skill prompt request at https://github.com/openclaw/openclaw/issues/65681 is a different ClawHub-scoped request.

## What This Changes

The issue requests that claude-cli-backed Control UI sessions render Claude Code's `AskUserQuestion` tool as an inline picker and return structured selections to the live turn.

## Best Possible Solution

Define a generic Claude CLI HITL tool-result contract, parse `AskUserQuestion`, render a Control UI picker, submit selected answers back to the live turn, and cover the parser/UI/runtime path with focused tests plus live Claude CLI proof.

## Reproduction Assessment

Yes. Source inspection plus the reporter's live capture gives a high-confidence path: a `claude-cli` stream-json tool call is displayed by generic tool-card code, while live-session code ignores non-result records and only sends freeform user messages with `parent_tool_use_id: null`.

## Solution Assessment

No, not as a UI-only card change. The picker is part of the fix, but the maintainable solution must include a structured result path back into the active Claude CLI turn rather than only changing the renderer.
