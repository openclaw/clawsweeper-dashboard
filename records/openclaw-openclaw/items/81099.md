## Summary

Keep open. Current main still has Claude CLI stream/result parsing and generic display-only Control UI tool cards, but no `AskUserQuestion` picker state or structured result-submission path back into the live Claude CLI turn.

## What This Changes

Add Control UI support for Claude CLI `AskUserQuestion` tool calls by rendering a picker and returning structured selections to the stalled tool call.

## Best Possible Solution

Define a small generic human-in-the-loop tool-result contract for Claude CLI sessions, parse `AskUserQuestion`, render a Control UI picker, submit selected answers back to the pending live turn, and cover the parser/UI path plus a live Claude CLI proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: start a Control UI session backed by `claude-cli`, cause Claude Code to emit `AskUserQuestion`, and the current parser/UI/live-session paths have no branch that can render a picker or send a structured result back.

## Solution Assessment

No as a UI-only card special case. The maintainable fix needs the requested picker plus a runtime/protocol result path back into Claude CLI, preferably through a generic HITL contract rather than only changing tool-card rendering.
