- `P1`: A default fresh install can silently send recurring model calls and incur provider spend until the user edits the generated heartbeat file.

## Summary

Keep open: current main and v2026.5.20 still write a default `HEARTBEAT.md` whose docs-only Related link is treated as actionable content, so the empty-heartbeat skip does not apply for a fresh workspace. The issue is narrow, source-reproducible, and aligned with setup reliability and safe defaults.

## What This Changes

Make the default generated `HEARTBEAT.md` count as empty so fresh installs do not make recurring heartbeat model calls until users add tasks.

## Best Possible Solution

Make the generated heartbeat scaffold docs-only and empty by default, with regression coverage that checks the actual generated template through the empty detector and preserves wake/cron/actionable-task behavior.

## Reproduction Assessment

Yes for source-level reproduction: current main writes the reported template and the detector returns actionable for the docs-only link shape, so the heartbeat preflight will not skip. A full live gateway/provider repro was not run because this review was read-only.

## Solution Assessment

Yes, the narrowest maintainable fix is to make the generated template body match the documented empty-file contract and add coverage around the actual workspace template. Detector hardening is acceptable only if it does not mask real link-based tasks.
