## Summary

Keep open. Current main and the latest release still have auth-mode-blind billing copy, and https://github.com/openclaw/openclaw/pull/80895 is an open closing PR for this exact issue, so the issue should wait for that PR to land or be explicitly rejected.

## What This Changes

Requests auth-mode-aware billing error copy so Anthropic Claude CLI/Max OAuth users are not told to top up an API key.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80895 or an equivalent narrow fix that threads auth profile mode into central error formatting, preserves API-key billing copy, and uses neutral OAuth/token recovery copy.

## Reproduction Assessment

Yes for the source-level path: a billing-shaped usage-limit string can enter the billing classifier, and current main formats it through provider/model-only API-key copy. I did not run a live Max OAuth and Telegram reproduction.

## Solution Assessment

Yes. Passing the already-known auth profile mode into the billing formatter is the narrow maintainable fix; globally weakening billing classification would risk regressing real API-key billing errors.
