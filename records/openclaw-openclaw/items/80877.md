The issue asks OpenClaw to stop showing API-key top-up billing copy for Anthropic Max OAuth/token-backed usage-cap failures and use neutral recovery wording instead.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80895 or an equivalent auth-mode-aware formatter/context propagation fix that preserves API-key billing copy for API-key failures and uses neutral recovery wording for OAuth/token-backed usage-cap errors.
