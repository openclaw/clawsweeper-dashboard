## Summary

Keep open. Current main still has an auth-mode-blind billing formatter, and the open fix PR https://github.com/openclaw/openclaw/pull/80895 closes this issue, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Avoid showing API-key/top-up billing copy for Anthropic Max OAuth or token-backed billing-shaped errors while preserving the existing API-key billing message.

## Best Possible Solution

Land the open auth-mode-aware formatter/context propagation fix, keeping hard API-key billing copy for API-key failures and using neutral recovery wording for OAuth/token-backed Anthropic usage-cap errors.

## Reproduction Assessment

Yes. A billing-shaped Anthropic OAuth usage-cap string such as `out of extra usage` reaches `formatAssistantErrorText`, and current main then calls an API-key-only billing formatter; no live Max OAuth/Telegram run was performed here.

## Solution Assessment

Yes. Passing the already-known auth profile mode into the central formatter and downstream failover/payload paths is the narrow maintainable fix, and https://github.com/openclaw/openclaw/pull/80895 is the active implementation candidate.
