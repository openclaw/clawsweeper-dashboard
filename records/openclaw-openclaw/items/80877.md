The issue reports that Anthropic Max OAuth billing-shaped errors are shown with API-key top-up wording in user-facing channels even though no API key is configured.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/80895 or an equivalent auth-mode-aware formatter/context propagation fix that preserves API-key billing copy while using neutral recovery wording for OAuth/token-backed Anthropic usage-cap errors.
