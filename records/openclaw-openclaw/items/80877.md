## Summary

Keep open. Current main and the latest release still format Anthropic OAuth extra-usage/billing-shaped failures with API-key top-up copy, and the linked open PR is the active closing fix path.

## What This Changes

Issue reports that Anthropic Max OAuth billing-shaped errors are formatted as API-key top-up failures and delivered to user channels.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/80895 or an equivalent auth-mode-aware formatter/context propagation fix, preserving API-key billing copy while using neutral recovery wording for OAuth/token-backed Anthropic usage-cap errors.

## Reproduction Assessment

Yes, at source level. A billing-shaped Anthropic OAuth usage-cap string such as `out of extra usage` is classified as billing, then `formatAssistantErrorText` calls the API-key-only billing formatter without any auth-mode context.

## Solution Assessment

Yes. Passing the already-known auth profile mode into the central formatter and downstream failover/payload paths is the narrow maintainable fix, and the open closing PR is the active implementation candidate.
