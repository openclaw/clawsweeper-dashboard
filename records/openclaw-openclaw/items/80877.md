## Summary

Keep open. Current main still routes Anthropic extra-usage billing-shaped errors through API-key/top-up copy with no auth-mode context, and the open closing PR https://github.com/openclaw/openclaw/pull/80895 remains the active fix path.

## What This Changes

The issue asks OpenClaw to avoid API-key/top-up billing copy when Anthropic is configured through OAuth or token-backed Claude/Max auth.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/80895 or an equivalent auth-mode-aware formatter/context propagation fix, preserving API-key billing copy for real API-key failures and using neutral recovery wording for OAuth/token-backed Anthropic usage-cap errors.

Yes. Source inspection shows billing-shaped Anthropic extra-usage text can reach an API-key-only formatter on current main; I did not run a live Max OAuth/Telegram path in this read-only review.

## Solution Assessment

Yes. Passing the already-known auth profile mode into the central formatter and downstream failover/payload paths is the narrow maintainable fix, and the open closing PR is the active implementation candidate.
