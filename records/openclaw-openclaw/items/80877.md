## Summary

Keep open. Current main still formats billing-classified Anthropic OAuth/Claude CLI errors with API-key top-up copy, and the open linked fix PR uses closing syntax, so this issue should stay open until that PR lands or is superseded by an equivalent fix.

## What This Changes

The issue requests auth-mode-aware billing error copy so Anthropic Max OAuth users are not told to top up an API key.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80895, or replace it with an equivalent narrow fix that carries credential mode into centralized user-facing billing error formatting while preserving API-key billing copy.

## Reproduction Assessment

Yes for the source-level formatter path: current main classifies usage-cap-shaped text as billing and then formats it through an API-key-only billing message. I did not run a live Max OAuth and Telegram reproduction in this read-only review.

## Solution Assessment

Yes, the linked PR's direction is the narrowest maintainable fix: pass the already-known auth profile mode into the central formatter and preserve hard billing copy for API-key credentials. Weakening billing classification globally would be riskier because API-key 402/balance failures should still surface as billing errors.
