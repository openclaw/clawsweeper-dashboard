## What This Changes

The issue reports misleading API-key billing copy for Anthropic Max OAuth usage-cap errors that can be delivered to user-facing channels.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/80895 or an equivalent auth-mode-aware formatter/context propagation fix, keeping hard API-key billing copy for API-key failures and using neutral recovery wording for OAuth/token-backed Anthropic usage-cap errors.
