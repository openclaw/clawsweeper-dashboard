## Summary

Keep open. Current main still has an auth-mode-blind billing formatter, and https://github.com/openclaw/openclaw/pull/80895 is already open as the implementation candidate for this issue.

## What This Changes

Make billing error messaging auth-mode aware so Anthropic Max OAuth or Claude CLI usage-limit errors do not tell users to top up an API key.

## Best Possible Solution

Review and land or reject the open auth-mode-aware billing-copy PR; this issue should close only after that fix or an equivalent reaches main.

## Reproduction Assessment

Yes, source-level: current main classifies Claude usage-limit text as billing and formats all billing errors with API-key top-up copy while the run path already knows `authProfileMode`. No live Max OAuth or Telegram run was performed.

## Solution Assessment

Yes. Threading auth profile mode or credential source into the existing billing formatter is the narrow maintainable fix, preserving API-key billing copy while using neutral OAuth/token wording.
