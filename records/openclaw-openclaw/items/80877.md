## Summary

Keep open. Current main still has an auth-mode-blind billing formatter that turns billing-classified Anthropic OAuth usage-cap errors into API-key top-up copy, and the open linked fix PR is the right implementation candidate to review before this issue closes.

## What This Changes

The issue requests Anthropic Max OAuth-aware billing error copy so users without API keys do not receive API-key top-up instructions.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80895, or a maintainer replacement with the same boundary: pass active auth profile mode through all user-facing billing-error formatting paths while preserving hard API-key billing copy.

## Reproduction Assessment

Yes for the source-level path: billing-shaped `out of extra usage` text can classify as billing, and current main formats that branch with API-key/top-up copy because no auth mode reaches the formatter. I did not run a live Max OAuth or Telegram channel reproduction.

## Solution Assessment

Yes, the narrow maintainable fix is to carry the already-known auth profile mode into the central formatter/failover/payload surfaces. Weakening billing classification globally would risk hiding real API-key credit failures.
