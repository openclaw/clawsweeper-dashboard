## Summary

Keep open. Current main still formats billing-classified Anthropic OAuth/Max usage errors with API-key top-up copy, and an open PR already closes this issue, so this should wait for review and landing rather than be closed early or queued for a duplicate repair.

## What This Changes

The issue requests auth-mode-aware billing error copy so Anthropic Max OAuth users are not told to top up an API key.

## Best Possible Solution

Review and land the linked auth-mode-aware formatter PR, preserving API-key billing copy for real API-key balance failures and using neutral recovery copy for OAuth/token-backed Anthropic usage-cap failures.

## Reproduction Assessment

Yes for the source-level path: once an Anthropic OAuth/Max usage-cap string is classified as billing, current main has no credential mode at the formatter boundary and emits API-key/top-up copy. I did not perform a live Max OAuth plus Telegram run during this read-only review.

## Solution Assessment

Yes. Passing the already-known auth profile mode into the central billing formatter is the narrow maintainable fix, and the open PR at https://github.com/openclaw/openclaw/pull/80895 follows that direction without weakening API-key billing classification.
