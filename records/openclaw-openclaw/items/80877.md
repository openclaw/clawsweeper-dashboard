## Summary

Keep open. Current main and the latest release still format Anthropic OAuth/token billing-shaped errors with API-key/top-up copy, and the active closing PR https://github.com/openclaw/openclaw/pull/80895 remains the right fix path.

## What This Changes

The issue asks OpenClaw to avoid API-key billing/top-up wording for Anthropic Max OAuth usage-cap or transient errors.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/80895 or an equivalent auth-mode propagation fix that preserves API-key billing copy while using neutral recovery copy for OAuth/token-backed Anthropic errors.

## Reproduction Assessment

Yes, at source level. A billing-shaped Anthropic OAuth usage string such as out of extra usage reaches formatAssistantErrorText, which currently calls an API-key-only formatter without credential mode.

## Solution Assessment

Yes. Passing the already-known auth profile mode through the central formatter and downstream failover/payload paths is the narrow maintainable fix, and the open closing PR is already following that shape.
