## Summary

Keep open. Current main and the latest release still route billing-shaped Anthropic OAuth usage-cap text to API-key/top-up wording, and the active closing fix path is https://github.com/openclaw/openclaw/pull/80895.

## What This Changes

The issue asks OpenClaw to avoid API-key top-up billing copy for Anthropic Max OAuth or token-backed usage-cap errors.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80895 or an equivalent auth-mode-aware formatter/context propagation fix, then close this issue as implemented.

## Reproduction Assessment

Yes, at source level. A billing-shaped Anthropic usage-cap string such as out of extra usage reaches formatAssistantErrorText, and current main then calls an API-key-only billing formatter with no auth-mode context.

## Solution Assessment

Yes. Passing the already-known auth profile mode into the central formatter and downstream failover/payload paths is the narrow maintainable fix, and https://github.com/openclaw/openclaw/pull/80895 is the active implementation candidate.
