## Summary

## What This Changes

The issue requests OAuth-aware Anthropic billing error text so Max OAuth users are not told to top up an API key.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80895 or an equivalent auth-mode-aware formatter fix, then close this issue as implemented after merge.

## Reproduction Assessment

Yes at source level. A billing-shaped Anthropic extra-usage string matches the billing classifier, and current main formats it through API-key/top-up copy without auth-mode context.

## Solution Assessment

Yes. The narrow maintainable solution is to pass the already-known auth profile mode into the central formatter and downstream user-visible error paths while preserving API-key billing copy for API-key failures.
