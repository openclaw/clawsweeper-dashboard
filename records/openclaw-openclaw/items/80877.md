## What This Changes

The issue asks for Anthropic Max OAuth billing-shaped errors to avoid API-key top-up wording and use neutral recovery copy while preserving API-key billing messages.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80895 or an equivalent narrow fix that carries credential mode into central billing-error formatting without weakening API-key billing handling.

Yes for a source-level reproduction path: once a Claude CLI/OAuth usage-cap response is classified as billing, current main emits API-key/top-up copy because no auth mode reaches the formatter. I did not perform a live Max OAuth/Telegram run.

## Solution Assessment

Yes. The maintainable fix is auth-mode-aware error shaping at the central formatter/failover boundary, preserving hard API-key billing copy while using neutral OAuth/token recovery copy.
