## Summary

Keep open: current main and the latest release already tear down local stdio MCP process trees through the OpenClaw stdio transport, so the suggested `disposeSession()` patch is not the right fix. The SSH/remote-process survivor part is still not covered by current source/docs and no exact canonical issue owns this bundle-MCP-specific gap.

## What This Changes

The issue asks OpenClaw to stop `mcp.servers` bundle-MCP child processes, especially SSH-backed servers, from surviving Gateway restarts.

## Best Possible Solution

Keep the existing local stdio process-tree cleanup, then design a narrow bundle-MCP remote-command cleanup or diagnostic path for SSH-hosted servers that does not duplicate transport teardown or break normal stdio servers.

## Reproduction Assessment

No high-confidence live reproduction was established in this review. Source inspection shows local stdio children are already killed, while SSH-backed remote child ownership is not represented in the current bundle-MCP cleanup path.

## Solution Assessment

No, the suggested `killProcessTree` placement is not the best fix because local stdio transports already do that work. The remaining issue needs a maintainer decision on supported SSH/remote lifecycle behavior, diagnostics, or documentation.
