## Summary

Keep open: the patch is aimed at the right gateway restart path, but it is not merge-ready because one skip-deferral emission path still drops the new intent and the supplied proof does not show the live pending-reply restart scenario.

## What This Changes

The PR adds a `skipDeferral` restart-intent flag, threads it through the scheduled SIGUSR1 restart path, and makes the gateway close-stage reply drain timeout zero for skip-deferral restarts.

## Best Possible Solution

Thread the skip-deferral intent through every SIGUSR1 emission path that bypasses deferral, add regression coverage for the active-deferral upgrade case, and provide redacted live terminal/log proof that the safe skip path no longer waits on pending replies while normal restarts still drain.

## Reproduction Assessment

Yes at source level: current main forwards `skipDeferral` into the scheduler, but restart close drain depends on an intent that is missing on at least one scheduler emit path. I did not run a live channel-originated pending-reply restart.

## Solution Assessment

No; the PR is a plausible narrow fix, but it is not the best complete fix until the active-deferral bypass path carries the same intent and is covered by a regression test.
