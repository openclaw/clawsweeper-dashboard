## Summary

## What This Changes

The PR adds a `skipDeferral` restart-intent flag, threads it through the scheduled SIGUSR1 restart path, and makes the gateway close-stage reply drain use a zero timeout for skip-deferral restarts.

## Best Possible Solution

Thread the same skip-deferral intent through every skip-deferral emit path, add a regression test for bypassing an already-active deferral, and require real terminal/log proof of the restart behavior or an explicit maintainer proof override before merge.

## Reproduction Assessment

## Solution Assessment
