## Best Possible Solution

## Solution Assessment

Yes, if maintainers accept the escape-hatch semantics. Threading the existing restart intent boundary is narrower than a side channel; self-reply filtering or a bounded drain is the safer alternative only if skipped reply draining is not acceptable.
