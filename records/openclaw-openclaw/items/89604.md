## What This Changes

## Best Possible Solution

## Solution Assessment

Yes, if maintainers accept the escape-hatch tradeoff. Threading the existing restart intent boundary is narrower than adding a second side channel; self-reply filtering would be a larger design only needed if skipped reply draining is not acceptable.
