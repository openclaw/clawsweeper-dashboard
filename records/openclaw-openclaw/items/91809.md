## Summary

Keep open: current main partially mitigates model-list latency, but source and tests still show provider-wide `provider/*` configurations using a full-discovery path, which matches the reporter's setup and is not proven fixed or shipped.

## What This Changes

The issue reports that `/models` and `openclaw models list` regressed to multi-second catalog loading with many provider-wide configured models.

## Best Possible Solution

Design a bounded, nonblocking provider-wildcard catalog path that preserves accurate provider-scoped discovery without making `/models` or default model listing wait on synchronous full discovery.

## Reproduction Assessment

Yes from source for the likely remaining path: provider-wide `agents.defaults.models` entries still force full catalog discovery in the gateway browse path. I did not run the reporter's WSL2 timing benchmark in this read-only review.

## Solution Assessment

No complete fix is proven yet. Current main is a partial mitigation for default/non-wildcard paths, but the provider-wildcard behavior needs a maintainer decision on exact discovery versus cached or stale nonblocking results.
