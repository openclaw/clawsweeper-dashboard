- `P2`: The source-backed regression adds several seconds to common model-selection commands across multiple surfaces but does not block agent execution.
- `impact:auth-provider`: Provider catalog discovery and authentication visibility checks are central to both the slowdown and proposed fix.

## Summary

Current main still sends provider-wildcard configurations through full catalog discovery, matching the reported regression, and the linked implementation PR remains open rather than merged.

## What This Changes

Restore fast model listing for wildcard-heavy multi-provider configurations across CLI, gateway, and chat model pickers.

## Best Possible Solution

Keep explicit `--all` exact, adopt one bounded default/configured browse policy across CLI and chat/Gateway, and require representative before/after timings plus completeness checks for provider wildcards and authenticated providers.

## Reproduction Assessment

Yes, from source with high confidence: any `agents.defaults.models` provider wildcard deterministically bypasses the 750 ms bounded path and runs full discovery on current main. A fresh live timing run is still needed to quantify the environment-specific delay.

## Solution Assessment

Not fully settled. Bounding wildcard discovery is the narrow root-path fix, but maintainers must explicitly choose the allowed temporary incompleteness and ensure the same policy covers CLI, Gateway, and chat callers.
