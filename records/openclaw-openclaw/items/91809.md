## What This Changes

The issue reports a regression where `/models` and `openclaw models list` take several seconds with multi-provider model configurations because provider/catalog/auth data is rebuilt on request paths.

## Best Possible Solution

Land or replace the linked fix PR with a maintainer-reviewed implementation that bounds shared `/models` catalog work across CLI, Telegram callbacks, and Discord picker interactions, with regression tests and real channel timing proof.

## Reproduction Assessment

Yes at source level: current Telegram callbacks and Discord picker loading still call the shared builder that performs catalog and auth visibility work before rendering pages. I did not establish a fresh live timing repro in this read-only pass.

## Solution Assessment
