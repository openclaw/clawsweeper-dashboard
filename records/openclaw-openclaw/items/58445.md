## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve trigger context through coalescer flushes:** `src/auto-reply/reply/block-reply-pipeline.ts:181`
  - body: `before_block_reply` depends on `context?.trigger`, but coalesced replies are flushed with `undefined` context. With block streaming coalescing enabled, plugins cannot distinguish `tool_start`, `text_end`, `message_end`, and `run_end` for most text blocks; store or merge context with coalesced payloads before flushing.
  - confidence: 0.91
- **[P2] Treat direct-send cancellation as non-delivery:** `src/auto-reply/reply/reply-delivery.ts:129-130`
  - body: The direct-send path records the block content key before awaiting `onBlockReply`. If a plugin cancels via `BlockReplyCancelledError`, final-reply suppression can still treat the canceled content as delivered and drop it; move tracking after successful delivery or catch cancellation.
  - confidence: 0.86
- **[P3] Document the public block-reply hook:** `src/plugins/types.ts:2125-2136`
  - body: This exposes a public plugin hook contract, but the plugin docs do not describe the event fields, trigger values, text mutation, or cancellation behavior. Add docs/API contract coverage before exposing the new hook to plugin authors.
  - confidence: 0.82
