## Summary

Keep open: current main still lacks the proposed block-reply interception hook, but this PR adds a new public plugin API and its current branch has source-level correctness blockers plus missing real Slack/runtime proof.

## What This Changes

The PR adds a public `before_block_reply` plugin hook, threads block-reply trigger context through streaming delivery, and includes gateway hook session-key routing changes and tests.

## Best Possible Solution

Land a rebased, hook-focused implementation only if maintainers want this public API, with trigger/cancel semantics preserved through coalescing and direct sends plus public docs/API coverage.

## Reproduction Assessment

Yes for the PR defects by source inspection: coalesced replies lose trigger context before the hook runs, and direct-send cancellation can be recorded as delivered. No high-confidence live reproduction was provided for the claimed Slack fragmentation fix after the patch.

## Solution Assessment

No. The hook direction is plausible, but this branch is not the best merge path until it is narrowed away from already-main gateway work, fixes trigger/cancel semantics, documents the public contract, and includes real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve trigger context through coalescer flushes:** `src/auto-reply/reply/block-reply-pipeline.ts:181`
  - body: `before_block_reply` depends on `context?.trigger`, but coalesced replies are flushed with `undefined` context. Store or merge the context with coalesced payloads so plugins can distinguish `tool_start`, `text_end`, `message_end`, and `run_end` after coalescing.
  - confidence: 0.91
- **[P2] Treat direct-send cancellation as non-delivery:** `src/auto-reply/reply/reply-delivery.ts:129-130`
  - body: The direct-send path records the block content key before awaiting `onBlockReply`, and it does not handle `BlockReplyCancelledError`. If a plugin cancels this send, final-reply suppression can think the content was delivered; move tracking after successful delivery or catch cancellation.
  - confidence: 0.86
- **[P3] Document the public block-reply hook:** `src/plugins/types.ts:2125-2136`
  - body: This adds a public plugin hook contract, but the plugin docs do not describe the event fields, trigger values, text mutation, or cancellation behavior. Add docs/API contract coverage before exposing the new hook to plugin authors.
  - confidence: 0.82
