## What This Changes

The PR adds a `before_block_reply` plugin hook with block-reply trigger metadata through the streaming delivery path, plus gateway hook session-key routing changes and related tests/changelog text.

## Best Possible Solution

Update this work on current main as a narrow plugin-hook change: add the hook to the current hook contract/runner surfaces, preserve trigger and assistant-message context through coalescing, document the public API, add a user-facing changelog entry, and drop the already-landed gateway session-routing changes.
