## Summary

Keep this PR open: Telegram Guest Mode is a useful, well-scoped feature with credible live proof, but the current branch still has a blocking routing/security defect for account-bound Telegram setups.

## What This Changes

The PR adds opt-in Telegram Guest Mode support, including guest update ingestion, isolated guest session handling, `answerGuestQuery` delivery, config/schema/docs updates, and Telegram tests.

## Best Possible Solution

Preserve opt-in Guest Mode, but resolve the configured Telegram account/channel route before applying the isolated guest session key, with a regression test for an account-bound guest query.

## Reproduction Assessment

Yes for the blocking PR defect by source inspection: configure a Telegram account/channel binding to a non-default agent, send an allowed guest update, and the branch builds the guest session with the default agent while configured bindings are skipped. The overall request is a new feature, not a current-main bug reproduction.

## Solution Assessment

No: the Telegram plugin and `answerGuestQuery` direction are the right owner boundary, but the current implementation bypasses existing route bindings. The safer fix is to resolve account/channel bindings first, then isolate the guest session for the resolved route.
