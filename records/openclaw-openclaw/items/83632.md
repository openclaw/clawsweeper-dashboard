Full review comments:

- **[P1] Preserve configured guest routing:** `extensions/telegram/src/bot-guest.ts:341`
  - body: Guest turns currently choose the global default agent before building the guest session key, and the context builder skips configured bindings when that override is present. That can route an account or channel bound to a non-default agent under the wrong workspace/model/tool policy; derive the configured route first, then override only the guest session identity.
  - confidence: 0.91
- **[P2] Align the guest enablement contract:** `src/config/types.telegram.ts:97`
  - body: The PR body says Guest Mode is disabled when omitted/false and enabled with `true`, but the schema, types, docs, and tests still publish an `"auto"` mode. For this security-sensitive ingress path, either remove `"auto"` before merge or make the mode an explicit maintainer-accepted contract everywhere.
  - confidence: 0.86
- **[P3] Use grammY's guest API contract:** `extensions/telegram/src/bot-guest.ts:147-151`
  - body: grammY 1.43 exposes `bot.api.answerGuestQuery(guest_query_id, result)` returning `SentGuestMessage`, but this patch wraps `raw.answerGuestQuery` as returning `boolean` and tests that shape. Use the typed helper or update the raw typing/tests so they match the dependency contract.
  - confidence: 0.78
