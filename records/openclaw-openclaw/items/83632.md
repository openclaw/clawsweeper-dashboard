## What This Changes

The PR adds Telegram Guest Mode support for `guest_message` updates, isolated direct-style guest sessions, `answerGuestQuery` delivery, config/docs, and focused Telegram tests.

## Best Possible Solution

Land guest-mode support only after maintainers choose the upgrade default and are comfortable with the account-level allowlist boundary; keep the broader bot-to-bot work tracked separately.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new Telegram feature. Source inspection confirms current main has no guest-message handling, and the PR body supplies real Telegram screenshots for the new behavior.

## Solution Assessment

Mostly yes, subject to maintainer policy choice: the implementation stays in the Telegram plugin and reuses existing routing, allowlist, session, and dispatch seams. The safer alternative is to make Guest Mode explicitly opt-in instead of auto-enabled on upgrade.
