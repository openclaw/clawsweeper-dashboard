## Summary

Keep open: the PR is useful and proof-backed, but the current head still auto-enables Telegram Guest Mode when config is omitted, after the author agreed omitted config should stay disabled. ## What This Changes

## Best Possible Solution

Require explicit `channels.telegram.guest.enabled: true` for the default-preserving path, reserve `"auto"` for explicit metadata-based opt-in, and keep tests/docs aligned with account-level allowFrom isolation.

## Solution Assessment
