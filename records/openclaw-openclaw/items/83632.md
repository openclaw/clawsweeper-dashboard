## Summary

Keep open: this is a useful, proof-backed Telegram feature, but the current patch is not merge-ready because allowed guest turns can bypass existing Telegram account/channel route bindings and run under the default agent/session.

## What This Changes

The PR adds opt-in Telegram Guest Mode handling with guest update admission, isolated guest sessions, answerGuestQuery delivery, Telegram config/schema/docs updates, media context handling, and focused tests.

## Best Possible Solution

Keep the opt-in Guest Mode design, but resolve the existing Telegram account/channel route first, then apply the guest-specific isolated session key and answerGuestQuery delivery with a regression test for bound accounts.

## Reproduction Assessment

Yes for the merge blocker at source level: PR head selects the default agent for guest turns and the session override then skips binding resolution. I did not run tests because this review was read-only.

## Solution Assessment

No for the current patch: the Telegram plugin and answerGuestQuery ownership are the right layer, but route selection is too broad. The safer fix is to resolve the account/channel route first and only then isolate the guest session from source-group bindings.
