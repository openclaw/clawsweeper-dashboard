## Summary

Keep open. The intended token-comparison hardening is valid, but the proposed patch imports from an SDK subpath that current main does not export, so it needs a narrow import-path repair before merge.

## What This Changes

The PR imports `safeEqualSecret` into the diffs artifact store and replaces the direct artifact token inequality check in `DiffArtifactStore.getArtifact`.

## Best Possible Solution

Land the narrow hardening change with `safeEqualSecret` imported from `openclaw/plugin-sdk/security-runtime`, preserving the existing diffs behavior for valid tokens and avoiding new SDK aliases or package export changes.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: applying the PR as provided imports `openclaw/plugin-sdk/browser-security-runtime`, while current main exports and documents only `openclaw/plugin-sdk/security-runtime`.

## Solution Assessment

No, not as currently written. The comparison helper is the right maintainable primitive, but the PR must use the existing supported SDK subpath rather than adding or referencing a stale browser-specific subpath.
