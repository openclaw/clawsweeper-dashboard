## Summary

Keep open. The hardening intent is still useful because current main still compares the diffs artifact token with `!==`, but this PR is not mergeable as written: it imports an unsupported SDK subpath, and the external PR does not include after-fix real behavior proof.

This PR replaces the diffs artifact-store token comparison with `safeEqualSecret` and adds an SDK import in `extensions/diffs/src/store.ts`.

## Best Possible Solution

Land the narrow hardening by adding `safeEqualSecret` to current main's supported `openclaw/plugin-sdk/security-runtime` import in `extensions/diffs/src/store.ts`, then provide redacted real-behavior proof and targeted test output.
