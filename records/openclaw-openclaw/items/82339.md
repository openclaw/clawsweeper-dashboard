## Summary

Keep open: current main still treats USER.md as a workspace-scoped bootstrap file, and a shared human profile would be a new multi-agent product/config contract rather than a broken existing behavior.

## What This Changes

The issue requests a global or shared USER.md human-profile mechanism so multiple agent workspaces can reuse one profile instead of maintaining duplicated per-workspace copies.

## Best Possible Solution

Define whether shared human profile context belongs in core or a plugin, then document and test the chosen contract across workspace bootstrap, config schema/help, and multi-agent docs.

## Reproduction Assessment

Not applicable as a bug reproduction; source, tests, and docs confirm the reported current behavior that USER.md is workspace-scoped and loaded from each resolved agent workspace.

## Solution Assessment

Unclear until maintainers choose the product boundary; a first-class shared profile path or explicit shared bootstrap contract would be more maintainable than relying on copy-based drift, while plugin/hook or symlink paths may be enough if core should stay lean.
