## Summary

Close as ClawHub scope: this is a useful optional cross-agent context feature, but current core intentionally treats USER.md as part of each agent workspace/persona scope and existing hook/plugin surfaces can inject shared human-profile context without adding a new core ownership model.

## What This Changes

The issue asks for a shared/global human profile source for USER.md across multiple agents instead of copying USER.md into each agent workspace.

## Best Possible Solution

Publish a ClawHub code plugin or skill that injects a configured shared human profile through agent:bootstrap or before_prompt_build, and use skipOptionalBootstrapFiles when operators do not want core to seed per-workspace USER.md copies.

## Reproduction Assessment

Not applicable: this is a feature/config request, not a broken documented behavior. Source and docs confirm current main uses per-agent workspace roots and a fixed USER.md bootstrap slot.

## Solution Assessment

Yes. The narrow maintainable direction is the ClawHub/plugin path because current hooks can inject shared profile context; symlink-based sharing is already covered by https://github.com/openclaw/openclaw/issues/38622.
