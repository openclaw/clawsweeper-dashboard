## Summary

Keep open: current main still has a source-level path where multiple accumulated `assistantTexts` become multiple final reply payloads, and an open linked fix PR already owns the repair path.

## What This Changes

Telegram direct replies are reported to fan out multiple accumulated assistant text blocks as separate outbound messages from a single assistant turn.

## Best Possible Solution

Review and land the linked runtime-delivery fix, or close it in favor of an equivalent narrow fix that collapses non-segmented final assistant text while preserving explicit streaming, chunking, media, and long-message behavior.

## Reproduction Assessment

Yes, at source level: current main still maps every non-empty `assistantTexts` entry to final reply payloads, and the dispatcher sends each payload separately. I did not run a live Telegram turn in this read-only review.

## Solution Assessment

Yes for the linked repair direction: the maintainable fix is to collapse non-segmented final assistant text while leaving explicit multi-part block streaming, media directives, and Telegram long-message splitting intact.
