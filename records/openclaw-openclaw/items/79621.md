## Summary

Keeping this open: current main still has a source-level path where multiple accumulated `assistantTexts` become multiple final reply payloads, and Telegram delivery sends each final payload separately.

## What This Changes

Make Telegram direct replies send only the final user-facing assistant answer for a turn instead of fanning out accumulated `assistantTexts` as separate messages.

## Best Possible Solution

Constrain non-block final delivery to the final user-facing assistant answer while preserving explicit streaming, chunking, media, tool, and error reply paths.

## Reproduction Assessment

Yes for a source-level reproduction, but not from a live Telegram run in this read-only review. Passing multiple non-empty `assistantTexts` into the current final-delivery path can produce multiple reply payloads, and Telegram delivery sends each payload separately.

## Solution Assessment

No, the current source path is not the best solution because it treats every accumulated assistant text as an independent final reply. The narrow maintainable fix is to select or derive the final user-facing answer for non-block final delivery and leave explicit block streaming/chunking behavior alone.
