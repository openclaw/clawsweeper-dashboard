## Summary

Keep open: current main still treats missing `plugins.deny` entries as fatal, and an open closing PR at https://github.com/openclaw/openclaw/pull/80516 now owns the narrow stale-deny fix path. The issue should remain open until that PR or the chosen overlapping replacement lands.

## What This Changes

The issue asks for `doctor --fix` to avoid losing repairable config cleanup when multiple validation problems exist, especially stale plugin refs plus model compatibility errors.

## Best Possible Solution

Land the chosen narrow stale-deny fix so missing `plugins.deny` refs follow the existing stale-entry warning semantics, and track any broader incremental `doctor --fix` persistence policy separately.

## Reproduction Assessment

Yes. Source inspection on current main shows a missing `plugins.deny` ref is a fatal validation issue, doctor repair preflight can restore invalid configs from last-known-good, and final config writes throw before persisting if any validation issue remains.

## Solution Assessment

Yes for the scoped root cause: changing stale `plugins.deny` refs to the existing warning path is narrower than weakening general config validation. The issue should not close until the open closing PR or selected replacement lands.
