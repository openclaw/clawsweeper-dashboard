## Summary

Keep open: current main still lacks Control UI typed `/new` hook delivery, and this PR is useful but incomplete because it restores only the internal `command:new` event while leaving the plugin `before_reset` and parent `session_end` paths from the linked regression skipped.

## What This Changes

The PR adds a changelog entry and emits an internal `command:new` hook from `sessions.create` when a parent session key is present.

## Best Possible Solution

Land a gateway-side fix that treats parent-session `sessions.create` as the Control UI `/new` transition and emits the same parent-session hook contract with focused regression coverage.

## Reproduction Assessment

Yes, source-level. `dispatchSlashCommand('/new')` reaches `sessions.create` with `parentSessionKey`, and current `sessions.create` has no command/reset/lifecycle hook emission.

## Solution Assessment

No. Emitting `command:new` helps bundled `session-memory` and custom internal hooks, but the patch does not restore the reported plugin `before_reset` and parent lifecycle behavior; a shared parent-transition hook helper would be narrower and more complete.
