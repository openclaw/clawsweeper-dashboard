## Summary

Current main and shipped releases after the reported 2026.4.27 build have replaced the affected retained-manifest runtime-deps path, and the memory dependencies now match the requested behavior for sqlite-vec and node-llama-cpp. Review target: https://github.com/openclaw/openclaw/issues/78209

## What This Changes

The issue requests that memory-core bundled runtime dependencies stop being pruned from the managed runtime-deps tree and that Ollama memory search not require node-llama-cpp.

## Best Possible Solution

Keep the shipped package-root/dist-runtime dependency model and have users on 2026.4.27 update to v2026.4.29 or newer rather than reintroducing retained-manifest pruning.

## Reproduction Assessment

No high-confidence current-main reproduction: the reported `plugin-runtime-deps` retained-prune path is absent from current source and the release notes mark that tree obsolete. The old-version report is concrete, but I did not run a packaged Gateway live in this read-only review.

## Solution Assessment

Yes. The current package-root `dist-runtime` model plus local-only lazy loading for `node-llama-cpp` is narrower and more maintainable than reviving per-plugin retained-prune state.
