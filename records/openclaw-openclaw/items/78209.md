## Summary

Current main and the latest release no longer use the reported shared bundled `plugin-runtime-deps` install/prune lifecycle; the runtime-deps tree is now legacy cleanup only, while `chokidar`/`sqlite-vec` are declared in package manifests and `node-llama-cpp` is lazy optional local-embedding support.

## What This Changes

The issue asks OpenClaw to stop memory-core and other plugin runtime dependencies from being installed and then pruned out of the auto-managed runtime-deps tree during gateway or CLI lifecycle passes.

## Best Possible Solution

Keep the shipped install/update/doctor-only dependency model and avoid reintroducing startup-time bundled runtime-deps staging or shared install-root pruning.

## Reproduction Assessment

No current-main failing reproduction remains. The historical 2026.4.27-2026.5.5 reports are concrete, but current source and v2026.5.7 release content show the startup/shared `plugin-runtime-deps` installer is gone and those paths are cleanup-only.

## Solution Assessment

Yes. Eliminating runtime package-manager mutation and using package declarations/lazy optional local embeddings is narrower and safer than trying to make competing per-plugin retained manifests converge.
