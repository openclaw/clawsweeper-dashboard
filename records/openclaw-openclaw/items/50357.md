## Summary

Keep this PR open: the docs-only direction is useful for current `main`, but the patch still misstates the requester permission contract and shows an unusable result access pattern, and the open #48377 alternative would change the same create contract instead of documenting it as title-only.

## What This Changes

Updates the Feishu document skill docs to remove `owner_open_id`, state that `create` is title-only, and add a create-then-write workflow example.

## Best Possible Solution

Choose the Feishu create contract first, then either land #48377's create-with-content implementation or correct this docs-only patch to match the current title-only behavior exactly.

## Reproduction Assessment

Yes. Source inspection on current main shows `create` lacks `content`, returns payload data under `details`, and grants only trusted Feishu requesters `edit` access; copying this PR's example would pass an undefined `doc_token`.

## Solution Assessment

No. A docs-only clarification can be the narrow solution if current behavior is kept, but this patch needs factual corrections and maintainers need to choose it over #48377's create-with-content direction.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Describe requester grants as edit access:** `extensions/feishu/skills/feishu-doc/SKILL.md:61`
  - body: The new bullet says the requester gets `full_access`, but `createDoc()` grants `perm: "edit"` and returns `requester_perm_type: "edit"`. This would mislead users about the sharing controls available after document creation.
  - confidence: 0.97
- **[P2] Limit requester grants to trusted Feishu context:** `extensions/feishu/skills/feishu-doc/SKILL.md:61`
  - body: This wording reads as if create always grants requester access, but the runtime only derives a requester open id from Feishu-channel context. Calls from other contexts skip the grant, so the docs need to name that limitation.
  - confidence: 0.91
- **[P2] Read the created token from details:** `extensions/feishu/skills/feishu-doc/SKILL.md:70`
  - body: The example uses `result.document_id`, but tool results wrap payloads under `details` and the tests read `result.details.document_id`. Copying this workflow would pass `undefined` as the follow-up `doc_token`.
  - confidence: 0.96
