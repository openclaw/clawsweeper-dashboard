Full review comments:

- **[P1] Write config backups with owner-only permissions:** `scripts/dev/bootstrap-local-coding-agents.mjs:71`
  - body: openclaw.json can contain sensitive local settings, and current config writes harden both the main file and backups to owner-only mode. This raw backup write has no mode/chmod, so a permissive umask can leave a secret-bearing backup readable by other local users.
  - confidence: 0.9
- **[P2] Use a supported default coding model:** `scripts/dev/bootstrap-local-coding-agents.mjs:13`
  - body: The bootstrap creates agents with openai-codex/gpt-5.3-codex-spark, but current main rejects stale gpt-5.3-codex-spark catalog rows and directs users to openai/gpt-5.5. Freshly bootstrapped agents can fail until users manually edit the generated model.
  - confidence: 0.9
- **[P2] Parse OpenClaw config with JSON5-compatible IO:** `scripts/dev/bootstrap-local-coding-agents.mjs:60-62`
  - body: OpenClaw config accepts JSON5 comments and trailing commas, but this script uses JSON.parse. Existing valid configs will hard-fail even though the runtime loader supports them.
  - confidence: 0.94
- **[P2] Initialize missing config before reading:** `scripts/dev/bootstrap-local-coding-agents.mjs:19-20`
  - body: The documented bootstrap command first requires the config directory and then unconditionally reads openclaw.json. A clean state directory or first-run machine fails before bootstrapping anything, while current OpenClaw config loading treats a missing config as safe defaults.
  - confidence: 0.92
- **[P2] Preserve existing agent overrides:** `scripts/dev/bootstrap-local-coding-agents.mjs:132-135`
  - body: On rerun, the upsert spread replaces existing fields such as workspace, model, and skills with bootstrap defaults. That silently discards local customizations for the same agent IDs instead of only filling missing fields or warning.
  - confidence: 0.88
- **[P2] Move selftest scratch files out of the repository:** `scripts/dev/local-coding-agents-selftest.sh:11`
  - body: The selftest creates .local-agent-selftest.* under the repo root. If the process is killed before the cleanup trap runs, untracked scratch files remain in the working tree and can be accidentally staged.
  - confidence: 0.88
- **[P2] Drop the stale Discord seam-reset hunk:** `extensions/discord/src/monitor/native-command.model-picker.test.ts:9`
  - body: Current main already fixed the Discord native-command seam leak in the merged focused PR at https://github.com/openclaw/openclaw/pull/63267, and this branch is now carrying older overlapping test code without a distinct remaining failure.
  - confidence: 0.84
