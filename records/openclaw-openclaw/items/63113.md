## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Use a supported default coding model:** `scripts/dev/bootstrap-local-coding-agents.mjs:13`
  - body: The bootstrap creates agents with `openai-codex/gpt-5.3-codex-spark`, but current OpenClaw docs direct users to `openai/gpt-5.5` and warn that `openai/gpt-5.3-codex-spark` is not exposed. A fresh bootstrap would create agents that fail until users manually replace the model.
  - confidence: 0.94
- **[P2] Initialize missing config before reading:** `scripts/dev/bootstrap-local-coding-agents.mjs:19-20`
  - body: The documented bootstrap command first requires the config directory and then unconditionally reads `openclaw.json`. In a clean state directory it fails before bootstrapping anything, while current OpenClaw config loading treats a missing config as `{}`.
  - confidence: 0.91
- **[P2] Parse OpenClaw config with JSON5:** `scripts/dev/bootstrap-local-coding-agents.mjs:60-62`
  - body: OpenClaw config loading accepts JSON5 syntax, but this script uses `JSON.parse`. Existing valid configs with comments or trailing commas will make the bootstrap fail.
  - confidence: 0.94
- **[P2] Preserve existing agent overrides:** `scripts/dev/bootstrap-local-coding-agents.mjs:132-135`
  - body: On rerun, the upsert spread overwrites existing fields such as `workspace`, `model`, and `skills` with bootstrap defaults. That silently discards local customizations for the same agent IDs.
  - confidence: 0.86
- **[P2] Move selftest scratch files out of the repo:** `scripts/dev/local-coding-agents-selftest.sh:11`
  - body: The selftest creates `.local-agent-selftest.*` under the repository root. If the process is killed before the cleanup trap runs, the untracked directory remains in the working tree and can be accidentally staged.
  - confidence: 0.88
- **[P3] Guard ripgrep before scanning session logs:** `scripts/dev/local-coding-agents-selftest.sh:76-79`
  - body: `assert_tool_call` calls `rg` directly. On a machine without ripgrep, the selftest fails through a generic command-not-found path instead of a clear prerequisite error or fallback.
  - confidence: 0.78
- **[P3] Use English prompts in repo scripts and docs:** `scripts/dev/local-coding-agents-selftest.sh:90-104`
  - body: The selftest sends German agent prompts from repo-owned scripts, and the new QA doc repeats the same pattern. OpenClaw repo policy expects English user-visible strings, so these examples should be translated before landing.
  - confidence: 0.85
- **[P3] Remove the nonexistent QA asset reference:** `qa/README.md:10`
  - body: The QA README adds `seed-scenarios.json`, but this PR does not add that file and current main does not contain it under `qa/`. The index would point readers at a missing asset.
  - confidence: 0.83
