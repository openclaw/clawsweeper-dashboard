Full review comments:

- **[P1] Write config backups with owner-only permissions:** `scripts/dev/bootstrap-local-coding-agents.mjs:70`
  - body: `openclaw.json` can contain gateway/provider tokens and current OpenClaw backup helpers explicitly harden config backups to `0o600`. This backup writes the raw config to a new file with no explicit mode or chmod, so on permissive umasks it can leave secret-bearing backups readable by other local users.
  - confidence: 0.9
- **[P2] Use a supported default coding model:** `scripts/dev/bootstrap-local-coding-agents.mjs:13`
  - body: The bootstrap creates agents with `openai-codex/gpt-5.3-codex-spark`, but current OpenClaw docs say `openai/gpt-5.3-codex-spark` is not exposed and recommend `openai/gpt-5.5` as the stable default. Freshly bootstrapped agents can fail until users manually change the model.
  - confidence: 0.94
- **[P2] Initialize missing config before reading:** `scripts/dev/bootstrap-local-coding-agents.mjs:19-20`
  - body: The documented bootstrap path first requires the config directory and then unconditionally reads `openclaw.json`. A clean state directory or first-run setup fails before bootstrapping anything, while current OpenClaw config loading treats a missing config as safe defaults.
  - confidence: 0.92
- **[P2] Parse OpenClaw config with JSON5-compatible IO:** `scripts/dev/bootstrap-local-coding-agents.mjs:60-62`
  - body: OpenClaw config accepts JSON5 comments and trailing commas, but this script uses `JSON.parse`. Existing valid configs will hard-fail even though the runtime and docs support them.
  - confidence: 0.94
- **[P2] Preserve existing agent overrides:** `scripts/dev/bootstrap-local-coding-agents.mjs:132-135`
  - body: On rerun, the upsert spread replaces existing fields such as `workspace`, `model`, and `skills` with bootstrap defaults. That silently discards local customizations for the same agent IDs instead of only filling missing fields or warning.
  - confidence: 0.88

  - body: The selftest creates `.local-agent-selftest.*` under the repo root. If the process is killed before the cleanup trap runs, untracked scratch files remain in the working tree and can be accidentally staged.
