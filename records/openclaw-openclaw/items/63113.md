## Summary

Keep open. Current main still lacks the local coding-agent bootstrap/selftest workflow, but the PR has concrete blocking script defects: it defaults new agents to a model current main explicitly suppresses, cannot bootstrap a missing or JSON5 config, can leave selftest scratch files in the repo, and overwrites existing agent customizations on rerun.

## What This Changes

The PR adds local coding-agent bootstrap and selftest scripts, wires `qa:local-agents:*` package scripts, documents the QA workflow, and adds a Discord model-picker test isolation change.

## Best Possible Solution

Revise the PR so the bootstrap uses current supported model defaults or inherits the user's configured default, follows OpenClaw config IO semantics for missing and JSON5 config files, preserves local agent overrides, keeps selftest scratch data outside the repository, and rebases the Discord test change against current main.

## Reproduction Assessment

Yes. Static patch inspection gives high-confidence reproduction paths for the PR defects: a missing config path fails before bootstrapping, a JSON5 config fails `JSON.parse`, the default model is suppressed on current main, and interrupted selftests can leave repo-local scratch files.

## Solution Assessment

No. The idea may still be useful, but this patch is not the safest implementation until it aligns with current config/model contracts and avoids local-state clobbering.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Use a supported default coding model:** `scripts/dev/bootstrap-local-coding-agents.mjs:13`
  - body: The bootstrap creates both agents with `openai-codex/gpt-5.3-codex-spark` by default. Current main suppresses `gpt-5.3-codex-spark` for the OpenAI/Codex catalogs and documents that the current Codex catalog does not expose it, so a fresh bootstrap creates agents that fail unless the user knows to override the model. Use a current supported model such as `openai-codex/gpt-5.5`, or omit the per-agent model so the user's configured default wins.
  - confidence: 0.94
- **[P2] Create missing config before reading:** `scripts/dev/bootstrap-local-coding-agents.mjs:19-20`
  - body: The documented bootstrap command fails on a clean machine or temporary `OPENCLAW_STATE_DIR` because it requires the config directory and then unconditionally reads `openclaw.json` before creating the local agent profiles. Start from an empty config when the file is missing and create the state directory safely.
  - confidence: 0.91
- **[P2] Parse OpenClaw config as JSON5:** `scripts/dev/bootstrap-local-coding-agents.mjs:60-62`
  - body: OpenClaw's config loader accepts JSON5, but this script uses `JSON.parse`, so a valid `openclaw.json` with comments or trailing commas makes the bootstrap fail. Use the same JSON5-compatible config path as the runtime or import the shared parser.
  - confidence: 0.94
- **[P2] Move selftest temp files out of repo:** `scripts/dev/local-coding-agents-selftest.sh:11`
  - body: The selftest creates its scratch directory under the repository root. If the process is killed or crashes, the JSON/proof files remain as untracked working-tree artifacts and can be accidentally staged. Use `${TMPDIR:-/tmp}` or another out-of-repo temp location.
  - confidence: 0.86
- **[P2] Preserve existing agent overrides on upsert:** `scripts/dev/bootstrap-local-coding-agents.mjs:132-140`
  - body: When an `oc-builder` or `oc-github` entry already exists, the spread order overwrites fields such as `workspace`, `model`, and `skills` with defaults on every rerun. That silently discards local customizations; merge only missing fields or warn before replacing user-set values.
  - confidence: 0.82
