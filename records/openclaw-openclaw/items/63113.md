## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Use a supported default coding model:** `scripts/dev/bootstrap-local-coding-agents.mjs:13`
  - body: The bootstrap creates both agents with `openai-codex/gpt-5.3-codex-spark`, but current main suppresses that model for the OpenAI/Codex catalogs. A fresh bootstrap therefore creates agents that fail unless the user knows to override the model; use a supported default such as `openai-codex/gpt-5.5` or omit the per-agent model so the user's default wins.
  - confidence: 0.94
- **[P2] Initialize missing config before reading:** `scripts/dev/bootstrap-local-coding-agents.mjs:19-20`
  - body: The documented bootstrap command hard-fails on a clean machine or temporary state dir because it requires the config directory and then unconditionally reads `openclaw.json`. Start from an empty config when the file is missing and create the state directory safely.
  - confidence: 0.91
- **[P2] Parse OpenClaw config with JSON5:** `scripts/dev/bootstrap-local-coding-agents.mjs:60-62`
  - body: OpenClaw config loading accepts JSON5 syntax, but this script uses `JSON.parse`. Any valid config with comments or trailing commas will make the bootstrap fail, so the script should use the same JSON5-compatible config path as runtime config IO.
  - confidence: 0.94
- **[P2] Move selftest scratch files out of the repo:** `scripts/dev/local-coding-agents-selftest.sh:11`
  - body: The selftest creates its temp directory under the repository root. If the process is killed or crashes before the trap runs, `.local-agent-selftest.*` remains as untracked working-tree state and can be accidentally staged; use `${TMPDIR:-/tmp}` or another out-of-repo path.
  - confidence: 0.88
- **[P2] Preserve existing agent overrides:** `scripts/dev/bootstrap-local-coding-agents.mjs:132-140`
  - body: On rerun, the upsert spread overwrites existing fields such as `workspace`, `model`, and `skills` with bootstrap defaults. That silently discards local customizations for `oc-builder` or `oc-github`; merge only missing fields or warn before replacing user-set values.
  - confidence: 0.84
- **[P3] Remove the nonexistent QA asset reference:** `qa/README.md:10`
  - body: The README adds `seed-scenarios.json`, but this PR does not add that file and current main does not contain it under `qa/`. This leaves the QA index pointing at a missing asset; remove the bullet or add the intended file.
  - confidence: 0.83
