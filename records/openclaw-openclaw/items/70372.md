## Summary

Keep open. The PR targets a real Anthropic/Bedrock unphased-thinking delivery gap, but the current branch still has a blocking provider-agnostic fallback: after any thinking event it can classify later unphased text as `final_answer` before the existing OpenAI Responses pending-phase guard can classify or suppress it. Current `main` does not already contain this PR's state flag, so the work is not obsolete, but it should not merge as-is.

## What This Changes

The PR adds a subscription-state flag for native thinking events and uses it to synthesize a `final_answer` delivery phase for later unphased assistant text, preserving that flag across stream-item resets.

## Best Possible Solution

Keep the Anthropic/Bedrock fix direction, but scope the synthetic phase to the known unphased transport shape or attach an explicit provider/transport signal before the subscription handler. Preserve the OpenAI Responses pending-phase guard and add focused tests for both the intended Anthropic/Bedrock suppression and the OpenAI late-phase protection.

## Reproduction Assessment

Yes for the code-level behavior. The current Anthropic path emits unphased thinking/text events, and the PR diff shows a synthetic `final_answer` phase that would make OpenAI Responses pending-phase checks skip their early return after any thinking event.

## Solution Assessment

No. The proposed fallback is broader than the reported provider gap; the safer solution is a narrowly scoped unphased Anthropic/Bedrock fallback or provider-boundary metadata, with regression tests around the OpenAI pending-phase path.
