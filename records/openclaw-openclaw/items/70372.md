## What This Changes

The PR adds a subscribe-state flag that treats unphased non-OpenAI text after thinking events as `final_answer` delivery.

## Best Possible Solution

Attach reliable phase metadata at the Anthropic/Bedrock transport boundary or structurally suppress unphased thinking-adjacent prelude before channel delivery, with focused regression coverage and redacted real channel proof.

## Reproduction Assessment

Yes for a source-level path: Anthropic emits thinking events plus unphased text blocks, current delivery filtering is phase-dependent, and visible-text extraction falls back to unphased text. I did not run a live Bedrock, Slack, or Telegram replay in this read-only review.

## Solution Assessment

No. The PR labels delivery as `final_answer` but leaves the assistant text unphased, so the extractor can still return the same narration; the safer fix is to classify or suppress the text before channel delivery.
