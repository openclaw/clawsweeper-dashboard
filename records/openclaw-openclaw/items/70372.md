## Summary

Keep this PR open, but it is not merge-ready: the patch is aimed at a real channel-leak bug, yet the current diff can still deliver unphased narration text and the external PR still lacks after-fix Anthropic/Bedrock channel proof.

## What This Changes

The PR adds a `hadThinkingInMessage` subscribe state flag and derives a synthetic `final_answer` delivery phase for non-OpenAI Responses text after thinking events.

## Best Possible Solution

Attach reliable phase metadata at the Anthropic/Bedrock transport boundary or structurally suppress thinking-adjacent/tool-use prelude before channel delivery, with regression coverage and real provider/channel proof.

## Reproduction Assessment

Yes for a source-level path: Anthropic emits unphased thinking and text blocks, current delivery filters only explicit phase metadata, and the visible-text extractor falls back to unphased text. I did not run a live Bedrock/channel replay in this read-only review.

## Solution Assessment

No. The PR labels delivery as `final_answer` but leaves the assistant text unphased, so the extractor can still return the same narration; a safer fix would classify or suppress the text before delivery.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Classify unphased text before delivering it:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:497-501`
  - body: The fallback only labels the delivery path as `final_answer`; the Anthropic text block itself remains unphased. On `text_end`, `extractAssistantVisibleText()` falls back to unphased text when no explicit final-answer block exists, so a thinking event followed by `Let me check...` style prose can still be sent to channel delivery.
  - confidence: 0.88

  - body: `resetAssistantMessageState()` is also used when switching phase-aware stream items inside one assistant message. Clearing `hadThinkingInMessage` there means a later unphased text block in the same thinking message can bypass the synthetic phase path again.
  - confidence: 0.76
