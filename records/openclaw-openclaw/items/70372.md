## Summary

This PR targets a real external-channel leak, but the current flag-based patch is not merge-ready because it still treats unphased thinking-adjacent text as deliverable final-answer text and lacks real Anthropic/Bedrock channel proof.

## What This Changes

The PR adds a subscribe-state flag that derives `final_answer` delivery for unphased non-OpenAI Responses text after thinking events.

## Best Possible Solution

Attach reliable visibility metadata in the Anthropic/Bedrock transport path, or suppress/quarantine unphased thinking/tool-use-adjacent text before any partial, block, or final channel delivery, with focused regression coverage and redacted channel proof.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: the Anthropic transport emits thinking events plus unphased text events, and current visible-text extraction falls back to unphased assistant text; I did not run a live Bedrock, Slack, or Telegram replay in this read-only review.

## Solution Assessment

No. The patch labels the delivery path as `final_answer` but does not classify or suppress the unphased assistant text itself, so the extractor can still return the narration.
