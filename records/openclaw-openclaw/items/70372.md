## Summary

Keep open. Current main still lacks a scoped fix for Anthropic/Bedrock unphased thinking-plus-text delivery, and this PR needs revision because its synthetic `final_answer` fallback both fails to distinguish unphased narration from visible answer text and is broad enough to weaken late-phase delivery protections.

## What This Changes

The PR adds a subscription-state flag for native thinking events and uses it to synthesize a `final_answer` delivery phase for later unphased assistant text, with state initialization and reset plumbing.

## Best Possible Solution

Keep the Anthropic/Bedrock fix direction, but attach an explicit transport-level signal or phase metadata for the unphased stream shape and add focused tests that prove plain narration is suppressed without weakening OpenAI Responses late-phase buffering. Avoid broad message-state inference unless it is scoped to providers that cannot supply phase metadata.

## Reproduction Assessment

Yes for code-level reproduction. A subscription-harness test can emit `thinking_start` followed by an Anthropic/Bedrock-style unphased `text_delta` containing `Let me check...`; the current extractor and sanitizer evidence show that text remains visible unless a stronger structural signal is added.

## Solution Assessment

No. The current PR is not the best fix because it labels all post-thinking unphased text as `final_answer`, which neither distinguishes narration from answer text nor preserves provider-specific late-phase safeguards; a scoped transport contract or explicit metadata is safer.
