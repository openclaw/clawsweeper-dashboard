## Summary

Keep open: the latest PR addresses the earlier OpenAI Responses scoping concern, but the core Anthropic/Bedrock leak is still not convincingly fixed and the external contributor has not provided after-fix real behavior proof.

## What This Changes

The PR adds a `hadThinkingInMessage` subscription-state flag and uses it to synthesize a `final_answer` delivery phase for non-OpenAI-Responses unphased text after native thinking events.

## Best Possible Solution

Attach reliable delivery-phase metadata at the provider/transport boundary, or otherwise suppress the specific unphased thinking-adjacent prelude before channel delivery, with regression coverage for Anthropic/Bedrock and OpenAI Responses plus real channel proof.

## Reproduction Assessment

Yes for a source-level path: Anthropic emits unphased thinking and text events, and the proposed synthetic phase still leaves ordinary unphased text visible through `extractAssistantVisibleText`'s fallback. No live Bedrock/channel replay proof is present.

## Solution Assessment

No. The PR changes the local delivery phase but does not mark or remove the unphased text content it is trying to suppress, so the reported `Let me check...` style leak can still be emitted.
