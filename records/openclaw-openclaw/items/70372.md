## Summary

Keep open: the PR targets a real channel-message leak, but the current patch still allows unphased Anthropic/Bedrock narration to be treated as visible assistant text and it lacks real after-fix channel proof.

## What This Changes

The PR adds a subscribe-state flag that derives a synthetic `final_answer` delivery phase after thinking events for unphased non-OpenAI Responses assistant text.

## Best Possible Solution

Attach reliable visibility metadata at the Anthropic/Bedrock transport boundary, or quarantine and suppress unphased thinking/tool-use-adjacent text before any partial, block, or final channel delivery, with focused regressions and redacted real channel proof.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: Anthropic emits thinking events plus unphased text blocks, current delivery only suppresses explicit commentary phase, and visible-text extraction falls back to unphased text. I did not run a live Bedrock, Slack, or Telegram replay in this read-only review.

## Solution Assessment

No. The PR labels the delivery path as `final_answer` but does not classify or suppress the unphased assistant text itself, so the extractor can still return the narration; the safer fix is earlier transport metadata or pre-delivery quarantine/suppression.
