## Summary

Keep open. The PR targets a valid channel-leak path, but the current diff only synthesizes delivery phase and still allows unphased Anthropic/Bedrock text to be extracted and delivered; it also lacks required real behavior proof.

## What This Changes

The branch adds a per-message thinking flag to subscription state and uses it to synthesize a `final_answer` delivery phase for non-OpenAI Responses streams after thinking events.

## Best Possible Solution

Attach reliable phase metadata at the Anthropic/Bedrock transport boundary or structurally suppress thinking-adjacent/tool-use prelude before channel delivery, with regression coverage and real provider/channel proof.

## Reproduction Assessment

Yes for a source-level path: Anthropic emits unphased thinking and text blocks, current delivery suppresses only explicit commentary phase, and visible-text extraction falls back to unphased text. I did not establish a live Bedrock/channel replay in this read-only review.

## Solution Assessment

No. The PR labels delivery as `final_answer` but leaves the assistant text unphased, so the extractor can still return the same narration; a safer fix would classify or suppress the text before delivery.
