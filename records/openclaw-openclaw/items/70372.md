## Summary

Keep open: the PR targets a real channel leak, but this patch still allows unphased Anthropic/Bedrock text to be treated as deliverable final-answer text, and the external PR still lacks after-fix real behavior proof.

## What This Changes

The PR adds a subscribe-state flag that infers a `final_answer` delivery phase for non-OpenAI Responses text after thinking events.

## Best Possible Solution

Attach reliable phase/visibility metadata at the Anthropic/Bedrock transport boundary or structurally suppress thinking-adjacent tool-use prelude before channel delivery, with regression coverage and redacted live channel proof.

## Reproduction Assessment

Yes for a source-level path: Anthropic text is unphased, thinking is emitted separately, current delivery relies on explicit phase metadata, and visible-text extraction falls back to unphased text. I did not run a live Bedrock or Anthropic channel replay in this read-only review.

## Solution Assessment

No. The PR labels the delivery path as `final_answer` without classifying or suppressing the unphased assistant text that the extractor later returns; the safer fix is to attach real phase metadata or suppress the thinking-adjacent text before delivery.
