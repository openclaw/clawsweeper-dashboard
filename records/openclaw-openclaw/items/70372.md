## Summary

Keep open: the PR targets a real user-visible leak, but the current patch still appears to let unphased narration through and it has no after-fix real behavior proof from the affected provider/channel setup.

## What This Changes

The PR adds a `hadThinkingInMessage` state flag in embedded PI streaming and uses it to synthesize `final_answer` delivery for unphased non-OpenAI Responses text after thinking events.

## Best Possible Solution

Attach reliable phase metadata at the Anthropic/Bedrock transport boundary, or suppress the specific thinking-adjacent/tool-use prelude before channel delivery, with regression coverage and real provider/channel proof.

## Reproduction Assessment

Yes for a source-level path: Anthropic emits unphased thinking and text events, and the PR still lets `extractAssistantVisibleText` fall back to unphased text. I did not run a live Bedrock or channel replay.

## Solution Assessment

No. The patch changes the delivery phase but not the assistant content phase or suppression policy, so the reported `Let me check...` style narration can still be delivered.
