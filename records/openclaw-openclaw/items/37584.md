## Summary

Keep open. Current main still supports root/plugin-scoped web-search configuration only; `agents.list[].tools` remains a strict policy/sandbox shape, so the requested per-agent credential/provider override is not implemented and needs maintainer config-design judgment.

## What This Changes

Request to let per-agent tool settings such as `agents.list[].tools.web.search.apiKey` override global `tools.web.search` for multi-agent setups.

## Best Possible Solution

Define an explicit agent-scoped web-search override contract, including precedence and SecretRef behavior, then align schema/types/help/docs/runtime wiring and focused multi-agent credential-isolation tests.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to configure `agents.list[].tools.web.search`: the strict `AgentToolsSchema` has no `web` field, and web search resolves from root `cfg.tools.web.search`.

## Solution Assessment

Unclear. The requested shape is plausible, but the best solution needs maintainer-owned precedence and SecretRef-scope decisions before implementation.
