## Label Justifications

- `P2`: This is a concrete multi-agent configuration improvement with a clear source-level gap but no current core outage.
- `impact:security`: The request concerns isolating API keys so credentials intended for one agent are not selected for another.
- `impact:auth-provider`: The feature would change web-search provider selection, API-key routing, env fallback, and SecretRef activation semantics.

## Summary

Keep open. Current main still supports root/plugin-scoped web-search settings but not per-agent `tools.web` overrides, and the related env/MCP issues are useful design inputs rather than a replacement for this tool-specific credential-scoping request.

## What This Changes

Add per-agent web/tool configuration so each agent can override root `tools.web.search` settings such as provider, timeout, and API key.

## Best Possible Solution

Define a documented agent-scoped web/tool override contract, then align schema, exported config types, SecretRef target discovery, runtime provider selection, docs, and credential-isolation tests around that contract.

## Reproduction Assessment

Not applicable as a broken-behavior reproduction because this is a feature request. The current gap is source-reproducible: `agents.list[].tools` is strict without `web`, the exported agent tools type lacks `web`, and web-search runtime has no agent-scoped selector.

## Solution Assessment

Unclear. `agents.list[].tools.web.search` is plausible, but maintainers should choose the public namespace and precedence model before implementation because this creates a credential-sensitive contract across agents, plugins, env fallbacks, and SecretRefs.
