## Summary

Keep open: current main and v2026.6.10 still expose web-search provider/key/timeout config only through global `tools.web.search` or plugin config, while `agents.list[].tools` remains policy/execution-only. Because the request adds credential-bearing config surface, it needs maintainer product and security direction before implementation.

## What This Changes

Request to allow per-agent overrides for web-search and tool-specific settings such as provider, API key, timeouts, and nested `tools.web.search` config.

## Best Possible Solution

Define a maintainer-approved agent-scoped tool/web credential contract with precedence, SecretRef handling, redaction, docs, doctor/validation behavior, and focused two-agent regression coverage.

## Reproduction Assessment

Yes, at source level. Current main's strict per-agent tool schema omits `web`, and the web_search runtime reads `config.tools.web.search` globally.

## Solution Assessment

Unclear. The proposed `agents.list[].tools.web` deep merge is plausible, but maintainers should first choose how it composes with plugin-scoped web-search credentials and broader per-agent plugin config work.
