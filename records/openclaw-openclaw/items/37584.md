## Summary

Keep open. Current main still has global/plugin-owned web-search settings only; it does not accept or merge `agents.list[].tools.web.search.*`, and the issue discussion includes a recent reporter confirmation that the request remains valid. This is not stale cleanup or implemented behavior.

## What This Changes

The issue requests agent-scoped tool configuration so each configured agent can override global web-search settings such as API key, provider, and timeouts.

## Best Possible Solution

Define an explicit agent-scoped web-search override contract and precedence model, align schema/types/help/labels/docs with that contract, thread the effective agent search config through web-search tool creation and execution, and cover multi-agent credential isolation with focused tests.
