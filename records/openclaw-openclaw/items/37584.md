## Summary

Keep open: current main and v2026.6.10 still do not support per-agent `tools.web.search` overrides, and the remaining work is a security-sensitive config/product design decision rather than conservative cleanup.

## What This Changes

Request per-agent tool settings so agents can use different web-search provider keys, providers, and timeouts instead of sharing global `tools.web.search` configuration.

## Best Possible Solution

Define a narrow maintainer-approved agent-aware web-search credential/config contract with explicit precedence, SecretRef handling, redaction, docs, and two-agent regression coverage.

## Reproduction Assessment

Yes, at source level. Current main's strict agent tools schema has no `web` field, and web-search provider resolution still reads `cfg.tools.web.search` / plugin config rather than agent-scoped tool settings.

## Solution Assessment

No, not as a broad recursive `agents.list[].tools.web` merge. The safer path is a narrower agent-aware web-search credential/config contract with explicit security, precedence, and plugin/core ownership rules.
