## Summary

Keep open. Current main and the latest release still expose web-search settings at root/plugin scope, while `agents.list[].tools` has no `web` schema/type/runtime merge contract; the related credential-scoping issues are useful design inputs but do not replace this specific request.

## What This Changes

The issue requests agent-scoped overrides for web-search and similar tool settings such as per-agent API keys, provider selection, and timeouts.

## Best Possible Solution

Define a documented agent-scoped web/tool override contract, then align schema, exported config types, SecretRef target discovery, runtime provider selection, docs, and credential-isolation tests around that contract.

## Reproduction Assessment

Yes as a source-level feature gap, not as a broken-behavior bug. Current schema and exported types reject `agents.list[].tools.web`, while runtime web-search config resolves from root/plugin scope with only active-agent auth-profile lookup.

## Solution Assessment

Unclear pending maintainer design. `agents.list[].tools.web.search` is plausible, but maintainers should choose the public namespace and credential precedence model before implementation because this creates a security-sensitive contract across agents, plugins, env fallbacks, auth profiles, and SecretRefs.
