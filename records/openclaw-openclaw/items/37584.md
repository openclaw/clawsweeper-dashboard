## Summary

Keep open: current main and v2026.6.10 still do not support per-agent web-search/tool settings, and the remaining work is a security-sensitive product/config design rather than cleanup.

## What This Changes

The issue requests per-agent overrides for web-search/tool settings such as API keys, provider choice, timeouts, and related `tools.web.search` fields.

## Best Possible Solution

Define a narrow maintainer-approved agent-aware web-search/tool configuration contract with explicit precedence, SecretRef handling, provider/plugin ownership, redaction, docs, and focused two-agent regression coverage.

## Reproduction Assessment

Yes, at source level. Current main's strict agent tools schema has no `web` field, while web-search provider and credential resolution still read global/plugin config and gateway environment sources.

## Solution Assessment

No, not as a broad recursive `agents.list[].tools.web` merge. The maintainable path is a narrower agent-aware web-search credential/config contract with explicit security, precedence, and plugin/core ownership rules.
