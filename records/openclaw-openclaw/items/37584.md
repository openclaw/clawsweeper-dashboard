## Summary

Keep open. Current main still has no agent-scoped web/tool settings contract: web-search settings are root/plugin-scoped, `agents.list[].tools` remains a strict policy/runtime-control block, and the related per-agent env issue is adjacent rather than a replacement.

## What This Changes

Add per-agent tool configuration overrides, especially agent-specific web search provider/API-key settings such as `agents.list[].tools.web.search.apiKey`.

## Best Possible Solution

Define a documented agent-scoped web/tool override contract, then align schema, exported config types, SecretRef target discovery, runtime provider selection, docs, and credential-isolation tests.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence that current main does not accept `agents.list[].tools.web` and resolves web-search settings from root/plugin-scoped configuration.

## Solution Assessment

Unclear. `agents.list[].tools.web.search` is plausible, but maintainers should choose the public namespace and precedence model first because this creates a credential contract across agents, plugins, env fallbacks, and SecretRefs.
