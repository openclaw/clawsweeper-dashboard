## Summary

Keep open. Current main still has root/plugin-scoped web-search configuration and strict per-agent tool policy only; the related per-agent env issue is broader and open, but it does not replace this tool-specific credential/config precedence request.

## What This Changes

The issue requests agent-scoped overrides for nested web-search tool settings such as API key, provider, and timeouts under each agent's `tools` config.

## Best Possible Solution

Define an explicit agent-scoped web-search override contract, then align schema, exported config types, SecretRef target discovery, runtime provider/credential selection, docs/help metadata, and multi-agent credential-isolation tests.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence that current main lacks the requested agent-scoped web-search config shape and resolves web-search settings from root/plugin-scoped configuration.

## Solution Assessment

Unclear. `agents.list[].tools.web.search` is plausible, but maintainers should choose the public namespace and precedence model first because this creates a credential/config contract across agents, plugins, environment fallbacks, and SecretRefs.
