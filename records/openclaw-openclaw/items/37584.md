## Summary

Keep this open: current main still has only selected per-agent tool policy fields, while web-search provider settings and credentials are resolved from global/plugin-owned configuration. The request is valid but needs a maintainer product/security decision before implementation.

## What This Changes

Request per-agent tool settings so entries such as `agents.list[].tools.web.search.apiKey`, provider, and timeouts can override global `tools.web` settings.

## Best Possible Solution

Add a typed agent-aware web-search/tool-settings resolver with explicit global-default, agent-override, plugin-owned config, auth-profile, and SecretRef precedence before adding schema fields.

## Reproduction Assessment

Yes, at source level. Current main's strict agent tool schema rejects the requested `web` field, and web_search execution resolves shared global/plugin configuration rather than an agent-specific overlay.

## Solution Assessment

No, not as an unrestricted deep merge of `tools.web`. A narrower typed resolver is safer because it preserves plugin ownership and makes credential precedence auditable.
