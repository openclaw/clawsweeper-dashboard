## Summary

Keep open. Current main still has no agent-scoped web/tool settings contract; web-search provider selection and credentials remain rooted in global or plugin-scoped config, and related per-agent plugin/env issues are adjacent design inputs rather than replacements.

## What This Changes

The issue requests agent-scoped tool settings so each agent can override global web-search configuration such as provider settings and API keys.

## Best Possible Solution

Define a documented agent-scoped web/tool override contract, then align schema, exported config types, SecretRef target discovery, runtime provider selection, docs, and credential-isolation tests.

## Reproduction Assessment

Not applicable as a bug reproduction, but yes for current behavior by source inspection: current main does not accept `agents.list[].tools.web` and resolves web-search settings from root/plugin-scoped configuration.

## Solution Assessment

Unclear. `agents.list[].tools.web.search` is plausible, but maintainers should choose the public namespace and precedence model first because this creates a credential contract across agents, plugins, env fallbacks, and SecretRefs.
