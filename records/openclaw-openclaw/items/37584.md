## Summary

Keep open. Current main still has only root `tools.web.search` and plugin-owned `webSearch` config, while per-agent `tools` remains a strict policy/sandbox shape, so the requested agent-scoped web-search override is not implemented.

## What This Changes

Request to allow per-agent web-search and other tool-specific settings, such as API keys, provider, and timeouts, under `agents.list[].tools` to override global tool config.

## Best Possible Solution

Define an explicit agent-scoped web-search override contract, including precedence and SecretRef behavior, then align schema/types/help/docs/runtime wiring and focused multi-agent credential-isolation tests.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to configure `agents.list[].tools.web.search`: current main has a strict `AgentToolsSchema` with no `web` field, and the web-search runtime reads only `cfg.tools.web.search`.

## Solution Assessment

Unclear. The feature direction is reasonable, but the best implementation needs maintainer-owned schema, precedence, and SecretRef-scope decisions before a narrow code change is safe.
