## Summary

Keep open. Current main still has root/plugin-scoped web-search configuration and strict per-agent tool policy only, while the related per-agent env issue is adjacent rather than a replacement for this tool-specific credential/config request.

## What This Changes

Request per-agent overrides for nested tool settings such as `agents.list[].tools.web.search.apiKey`, provider, and timeouts.

## Best Possible Solution

Define an explicit agent-scoped web/tool override contract, then align schema, exported config types, SecretRef target discovery, runtime provider selection, docs/help metadata, and multi-agent credential-isolation tests.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence that current main lacks the requested agent-scoped `tools.web.search` config shape and resolves web-search settings from root/plugin-scoped configuration.

## Solution Assessment

Unclear. `agents.list[].tools.web.search` is plausible, but maintainers should choose the public namespace and precedence model first because this creates a credential/config contract across agents, plugins, environment fallbacks, and SecretRefs.
