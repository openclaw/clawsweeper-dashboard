## Summary

Keep open. Current main still has global and plugin-owned web-search configuration only; it does not accept, type, document, or merge `agents.list[].tools.web.search.*`, and the issue discussion includes a recent reporter confirmation that the request remains valid. This is active config-contract design work, not stale cleanup or an implemented feature.

## What This Changes

The issue requests agent-scoped tool configuration so each configured agent can override global web-search settings such as API key, provider, and timeouts.

## Best Possible Solution

Define an explicit agent-scoped web-search override contract and precedence model, align schema/types/help/labels/docs with that contract, thread the effective agent search config through web-search tool creation and execution, and cover multi-agent credential isolation with focused tests.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to configure `agents.list[].tools.web.search` and validate it against current main: the strict `AgentToolsSchema` has no `web` field, and the runtime path reads only `cfg.tools.web.search`.

## Solution Assessment

Unclear. The feature direction is valid, but the exact public config shape and precedence rules should be chosen by maintainers because they affect plugin-owned credential paths and SecretRef isolation.
