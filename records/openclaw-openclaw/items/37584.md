## Summary

Keep open. Current main still lacks an agent-scoped web/tool settings contract, and the adjacent per-agent env issue is related but does not replace this tool credential/provider precedence request.

## What This Changes

The issue requests per-agent tool settings such as `agents.list[].tools.web.search.apiKey` so different agents can override global web-search credentials and provider settings.

## Best Possible Solution

Define one documented agent-scoped credential/config precedence contract, then align schema, exported types, SecretRef target discovery/redaction, runtime web provider resolution, docs, and multi-agent isolation tests around that contract.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence that current main still rejects `agents.list[].tools.web` and resolves web-search settings from root/plugin-scoped configuration.

## Solution Assessment

Unclear. `agents.list[].tools.web.search` is plausible, but maintainers should choose the public namespace and precedence model first because this creates a credential contract across agents, plugins, env fallbacks, and SecretRefs.
