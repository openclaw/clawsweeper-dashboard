## Summary

Current main still lacks per-agent web-search configuration for the reported Brave-key use case: agent tool schemas reject nested `web` settings, Brave reads credentials from global plugin config or gateway environment, and the newer active-agent auth path only applies to providers such as xAI that declare an auth-provider mapping. The issue remains useful, but the proposed generic deep override needs maintainer product and security design because it would add a broad secret-bearing config surface.

## What This Changes

Allow each configured agent to override web-search credentials, provider choice, timeouts, and potentially other tool-specific settings that are currently gateway-global.

## Best Possible Solution

Define a narrow agent-aware web-provider contract with explicit precedence: use each agent’s auth store or SecretRefs for credentials, allow only justified provider-specific nonsecret overrides, and avoid recursively duplicating the full global `tools` configuration under every agent.

## Reproduction Assessment

Yes, at source level. `AgentToolsSchema` is strict and has no `web` field, while Brave credentials resolve from gateway-global plugin config or environment rather than the active agent’s credential store.

## Solution Assessment

No, not as the proposed unrestricted nested `tools.web` merge. A narrower agent-aware provider and credential contract would preserve ownership, secret handling, and deterministic precedence.
