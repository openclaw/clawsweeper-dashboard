## Summary

Keep open. Current main still has only root/plugin-scoped web-search configuration, while per-agent `tools` remains a strict policy/sandbox shape; the reporter also reaffirmed the request after stale marking, so this is valid feature/design work rather than cleanup-close material.

## What This Changes

Allow `agents.list[].tools.web.search` and similar tool-specific settings to override global web-search provider and credential settings per agent.

## Best Possible Solution

Define an explicit agent-scoped web-search override contract, then align schema, exported types, SecretRef resolution, runtime provider selection, docs/help metadata, and focused multi-agent credential-isolation tests.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence that the requested config shape is absent today: `AgentToolsSchema` is strict and the web runtime resolves from root/plugin-scoped web-search config.

## Solution Assessment

Unclear. The requested `agents.list[].tools.web.search` shape is plausible, but the best implementation needs maintainer-owned decisions on precedence, SecretRef activation, and compatibility before a narrow code change is safe.
