## Summary

Keep open: current main and v2026.6.10 still keep `tools.web` and web-search credentials at global or plugin-entry scope, while the requested per-agent credential/tool override remains a product and security-sensitive config design.

## What This Changes

Request per-agent overrides for web-search/tool settings such as API keys, provider choice, timeouts, and other nested `tools.web` config.

## Best Possible Solution

Define a maintainer-approved per-agent credential/tool config contract that coordinates global `tools.web`, plugin-owned `webSearch` config, SecretRefs, and narrowly scoped non-secret tool defaults with regression coverage.

## Reproduction Assessment

Yes, at source level. Current main's strict per-agent tools schema has no `web` field, while the global tools schema and runtime web-search resolver still own web-search config outside the agent entry.

## Solution Assessment

Unclear. The requested shape is plausible, but the best solution must first settle whether per-agent web-search credentials belong under `tools.web`, plugin config overrides, env scoping, or a smaller approved combination.
