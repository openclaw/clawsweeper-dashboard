## Summary

This remains a valid missing capability on current main: agent entries can restrict selected tool behavior, but web-search provider configuration and credentials remain shared. Keep it open for a maintainer product/security decision because generic deep merging would cross global configuration, plugin ownership, auth-profile, and SecretRef boundaries; the related plugin-configuration request does not fully subsume this narrower tool-settings request.

## What This Changes

Request per-agent overrides for web-search credentials, provider choice, timeouts, and potentially other nested tool settings.

## Best Possible Solution

Introduce a typed agent-aware web-search resolution context with explicit global-default and agent-override precedence, preserve provider-owned plugin configuration, and apply the same resolved view to tool creation, late binding, SecretRefs, redaction, diagnostics, and tests.

## Reproduction Assessment

Yes, at source level. Current main's strict agent tool schema rejects the proposed `web` field, and general web-search execution reads shared global/plugin configuration; this demonstrates an absent feature rather than a regression in an existing contract.

## Solution Assessment

No, not as an unrestricted deep merge of the global tools tree. Extending the existing agent-aware provider/auth context through a typed resolver is safer because it can preserve plugin ownership and define credential precedence explicitly.
