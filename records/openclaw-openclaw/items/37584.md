## Summary

Keep open: current main and v2026.6.10 still do not support per-agent `tools.web.search` overrides, and the remaining work is a security-sensitive config/product design decision rather than a cleanup-close candidate.

## What This Changes

The issue requests per-agent overrides for web-search tool settings such as provider, timeout, and API key under `agents.list[].tools.web`.

## Best Possible Solution

Define a narrow agent-aware web-search credential/config contract with explicit precedence, SecretRef/redaction behavior, plugin/core ownership, docs, and focused two-agent regression coverage.

## Reproduction Assessment

Yes, at source level. Current main's strict agent tools schema has no `web` field, while web-search config and credentials still resolve from top-level/plugin config and gateway environment sources.

## Solution Assessment

No, not as a broad recursive `agents.list[].tools.web` merge. The maintainable path is a narrower maintainer-approved web-search credential/config contract with explicit security, precedence, and plugin/core ownership rules.
