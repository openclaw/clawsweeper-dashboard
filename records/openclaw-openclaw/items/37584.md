## Summary

Keep open: current main and latest release still do not support `agents.list[].tools.web` or per-agent web-search provider credentials, and the request remains a security/product design question rather than a cleanup close candidate.

## What This Changes

Requests per-agent overrides for web/tool settings such as `tools.web.search.apiKey`, provider, and timeouts in multi-agent gateway setups.

## Best Possible Solution

Define a narrow maintainer-approved agent-aware web-search/tool configuration contract with explicit precedence, SecretRef handling, redaction, docs, and focused two-agent regression coverage.

## Reproduction Assessment

Yes, at source level. Current main's strict agent tools schema has no `web` field, while web-search provider and credential resolution still read global/plugin config and gateway environment sources.

## Solution Assessment

No, not as a broad recursive `agents.list[].tools.web` merge. The maintainable path is a narrower agent-aware web-search credential/config contract with explicit security, precedence, and plugin/core ownership rules.
