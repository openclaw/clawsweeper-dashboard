## Summary

Keep open. Current main still has no agent-scoped web/tool settings contract: root `tools.web.search` is global, strict `agents.list[].tools` does not accept `web`, and web-search provider/SecretRef resolution has no agent dimension. The related per-agent env issue is useful design input, but it does not replace this tool-specific credential/provider override request.

## What This Changes

Request agent-scoped overrides for web-search and other tool settings, such as `agents.list[].tools.web.search.apiKey`, so different agents can use different search credentials or providers.

## Best Possible Solution

Define a documented agent-scoped web/tool override contract, then align schema, exported config types, SecretRef target discovery, runtime provider selection, docs, and credential-isolation tests.

## Reproduction Assessment

Not applicable as a bug reproduction, but yes for the current behavior by source inspection: current main does not accept `agents.list[].tools.web` and resolves web-search settings from root/plugin-scoped configuration.

## Solution Assessment

Unclear. `agents.list[].tools.web.search` is plausible, but maintainers should choose the public namespace and precedence model first because this creates a credential contract across agents, plugins, env fallbacks, and SecretRefs.
