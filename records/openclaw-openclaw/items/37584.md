## Summary

Keep open: current main and v2026.6.6 still lack per-agent web-search/tool configuration for the reported credential-routing use case. The broader per-agent plugin-config issue at https://github.com/openclaw/openclaw/issues/55401 overlaps, but this issue still has a distinct core `tools.web.search` provider/timeout/credential design question.

## What This Changes

Request to allow agent-scoped web-search/tool settings, such as separate `tools.web.search.apiKey` values per configured agent.

## Best Possible Solution

Define a narrow maintainer-approved agent-aware web-search credential/config contract with explicit precedence, SecretRef handling, docs, tests, and a clear boundary between core tool settings and plugin-owned provider settings.

## Reproduction Assessment

Yes, at source level. Current main's strict `AgentToolsSchema` has no `web` field, while web-search credentials resolve from global tool/plugin config or gateway environment rather than an agent-scoped slot.

## Solution Assessment

No, not as a broad recursive `agents.list[].tools.web` merge. The maintainable path is a narrower agent-aware web-provider credential/config contract with explicit security and precedence rules.
