## Label Justifications

- `P2`: This is a deterministic Gateway race with bounded but real client-visible impact on aborted `chat.send` runs.
- `impact:session-state`: The `chat:<runId>` dedupe cache can store the wrong terminal state for same-run retries and waiters.
- `impact:message-loss`: A late error broadcast can replace or contradict the correct terminal abort message clients should observe.

## Summary

Keep open: current main and the latest release still have the post-dispatch `chat.send` rejection path that can overwrite an abort terminal result, and the linked fixing PR is open rather than merged.

## What This Changes

Prevent a late post-dispatch `chat.send` rejection from overwriting an already-terminal `chat.abort` result and emitting a stale error for the same run.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/91000, or replace it with an equivalent focused fix that preserves abort-first and error-first terminal precedence in the `chat.send` post-dispatch path.

## Reproduction Assessment

Yes. Source inspection shows abort records terminal state, then the post-dispatch catch can still write an error terminal; the reporter also provided a deterministic targeted Vitest failure, but I did not run it in this read-only review.

## Solution Assessment

Yes. A focused fix in the `chat.send` post-dispatch catch path plus regressions for abort-first and error-first orderings is the narrow maintainable solution; the linked PR is the current implementation path.
