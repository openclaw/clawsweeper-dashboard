## Summary

Keep open: current main still has the post-dispatch `chat.send` catch path that can overwrite an already-terminal abort with an error, and the open linked pull request owns the focused fix.

## What This Changes

The issue asks for `chat.send` post-dispatch rejection handling to preserve the terminal abort result when `chat.abort` has already won, while keeping dispatch-first errors terminal.

## Best Possible Solution

Land `https://github.com/openclaw/openclaw/pull/91000` or an equivalent focused Gateway fix with regressions for both abort-first and error-first post-dispatch orderings.

## Solution Assessment

Yes. The narrow maintainable fix is in the `chat.send` post-dispatch catch path, with paired regressions for abort-first and error-first ordering; the open linked PR follows that shape.
