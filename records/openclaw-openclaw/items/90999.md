## Summary

Keep open: current `main` and latest release still have the post-dispatch `chat.send` error catch path, and the linked fixing PR is still open rather than merged.

## What This Changes

The issue reports that a late `chat.send` dispatch rejection can overwrite an already-terminal `chat.abort` result and broadcast a stale error.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/91000, or replace it with an equivalent Gateway fix that preserves abort-first and error-first terminal precedence.

## Reproduction Assessment

Yes. Source inspection shows `chat.abort` can record and broadcast an abort tombstone, while the later post-dispatch catch still writes an error dedupe and broadcasts an error; the reporter and linked PR also provide deterministic focused proof.

## Solution Assessment

Yes. A focused fix in the Gateway `chat.send` post-dispatch rejection path, with regression coverage for abort-first and error-first orderings, is the narrow maintainable solution.
