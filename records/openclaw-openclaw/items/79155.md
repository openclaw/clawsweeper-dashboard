## Summary

Keep open: current main still has a command-normalization path that drops real newline tails before slash skill arguments can reach downstream handling, and the issue describes a narrow regression with source-backed cause.

## What This Changes

Preserve multiline payloads passed after text slash skill commands instead of truncating content after the first newline.

## Best Possible Solution

Patch the shared command-body path so command-head normalization remains intact while multiline payload text is preserved for arg-accepting slash skills and covered by focused regression tests.

## Reproduction Assessment

Yes, source-reproducible: current main slices slash command text at the first newline in `normalizeCommandBody()`, and Telegram passes raw slash text through that helper before setting `CommandBody`. A live gateway run was not needed to identify the implicated path, but should be part of the fix proof if feasible.

## Solution Assessment

Yes, a narrow core fix is the best direction, but it should preserve clean command payloads through both normalization and command-source cleanup rather than only patching the built dist artifact. The safer repair is to add regression tests around `/skill` and direct skill commands before changing behavior.
