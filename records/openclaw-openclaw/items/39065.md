Full review comments:

- **[P1] Keep suppression out of the exported formatter:** `src/pairing/pairing-messages.ts:11`
  - body: `buildPairingReply` is exported through plugin SDK/runtime surfaces, so changing its return type to `string | null` can break callers that pass the result directly to send APIs. Keep the formatter string-returning and put silent suppression in the pairing challenge/transport decision layer, or explicitly version the plugin contract.
  - confidence: 0.9
- **[P1] Wire advertised config through strict schemas:** `src/config/types.discord.ts:220-221`
  - body: This adds `unpairedResponse` to channel config types, but strict schemas for some advertised/runtime-read channels remain without the field on current main. Operators who configure the new key can hit validation failures or an ignored setting instead of the promised response mode.
  - confidence: 0.88
- **[P1] Pass the response mode through IRC pairing replies:** `extensions/irc/src/inbound.ts:219-224`
  - body: The IRC hunk guards a possibly-null reply but still calls `buildPairingReply` without passing any mode, so `silent` and `code-only` cannot take effect on that channel even if the config is accepted. The channel should pass the resolved account response mode or not advertise the field there.
  - confidence: 0.86
- **[P2] Honor silent mode for pending Discord pairings:** `src/discord/monitor/agent-components.ts:530-539`
  - body: The Discord component path only applies the response mode when a pairing row is newly created; repeat attempts still get the hardcoded "Pairing already requested" reply. Silent mode should suppress that pending-pairing response too, or this entry point will still reveal the bot.
  - confidence: 0.87
- **[P3] Record outbound activity only after sending:** `extensions/nextcloud-talk/src/inbound.ts:187-193`
  - body: In the Nextcloud Talk path, `statusSink` is still updated after the guarded send block, so silent mode records outbound activity even when no message was sent. Move the status update inside the `if (replyText)` block.
  - confidence: 0.78
