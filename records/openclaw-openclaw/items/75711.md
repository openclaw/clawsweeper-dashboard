## Summary

Keep open. This is an active feature PR that is not implemented on current main, and the current patch still has source-backed merge blockers around systemd unit resolution, test isolation, and required public CLI documentation/changelog coverage.

## What This Changes

This PR adds a top-level `openclaw feature-lab` CLI diagnostic command that reports feature-lab root/install metadata, deployed SHA, systemd service file/command, and unit coverage for SHA/service wiring.

## Best Possible Solution

Land a revised command that uses the existing gateway service resolver/reader contract, keeps tests fully isolated, and documents the new public CLI surface with the required changelog entry.

## Reproduction Assessment

Yes. Source inspection of the PR diff and current main is enough: the patch hardcodes the default systemd unit path while current main supports `OPENCLAW_SYSTEMD_UNIT` and profile-derived unit names, and the added test writes under the real home service directory.

## Solution Assessment

No. The feature direction is narrow, but the implementation should reuse `resolveSystemdUserUnitPath` or `readSystemdServiceExecStart`, isolate HOME/service paths in tests, and add docs/changelog before merge.
