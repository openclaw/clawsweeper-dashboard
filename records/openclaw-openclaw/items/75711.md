## What This Changes

Land a revised diagnostic command only after it reuses the daemon service resolution/reading contract, keeps filesystem paths injectable for isolated tests, and adds the CLI reference page, docs navigation/index entry, and active changelog line.

Yes. A source-level reproduction is enough: with `OPENCLAW_PROFILE=work` or `OPENCLAW_SYSTEMD_UNIT=custom-unit`, current main resolves a non-default systemd unit path while this PR reads only the default unit file; the test side effect is visible from its real-home service-file write.

## Solution Assessment
