## Summary

Keep open. Current main still lacks the `feature-lab` command, and this PR is a relevant implementation candidate, but the patch has narrow blockers around systemd unit resolution, test isolation, and required public docs/changelog coverage.

## What This Changes

This PR adds a top-level `openclaw feature-lab` status command, lazy CLI registration, command catalog metadata, status implementation, and unit coverage for deployed SHA/service wiring.

## Best Possible Solution

Land a revised command that reuses the existing gateway service resolver/reader contract, keeps tests fully isolated from real user service files, and adds the required CLI reference and changelog entry.

## Reproduction Assessment

Yes. Source inspection is enough: the PR hardcodes the default systemd service path while current main supports `OPENCLAW_SYSTEMD_UNIT` and profile-specific unit names, and the new test writes under the real home service directory.

## Solution Assessment

No. The feature direction is narrow, but the implementation should reuse `resolveSystemdUserUnitPath` or `readSystemdServiceExecStart`, inject temp paths or HOME for tests, and add docs/changelog before merge.
