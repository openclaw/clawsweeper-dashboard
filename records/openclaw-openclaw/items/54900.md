Full review comments:

- **[P1] Keep startup expiry opt-in for existing queues:** `src/infra/outbound/delivery-queue-recovery.ts:215-216`
  - body: `recoverPendingDeliveries` defaults `maxEntryAgeMs` to 10 minutes, and gateway startup calls it without an override. Upgrading can move old first-attempt queue files to `failed/` for users who expected durable crash recovery, so keep the existing no-expiry behavior by default or wire this through explicit documented config/policy.
  - confidence: 0.88
- **[P2] Preserve stale filtering after budget deferrals:** `src/infra/outbound/delivery-queue-recovery.ts:219`
  - body: The stale gate only matches `retryCount === 0` entries with no `lastAttemptAt`, but recovery-budget deferral calls `failDelivery(...)`, which increments retry state without a send. An old never-attempted entry can then bypass this guard on the next startup and still replay out of context.
  - confidence: 0.92
- **[P2] Reconcile uncertain sends before expiring entries:** `src/infra/outbound/delivery-queue-recovery.ts:219`
  - body: This predicate also matches old entries with `recoveryState` such as `send_attempt_started` or `unknown_after_send`. Current main reconciles those with the adapter before replaying or failing; moving them to `failed/` solely by age can skip already-sent reconciliation and commit hooks.
  - confidence: 0.84
