## Summary

Keep open: current main still has the stale outbound replay path, and this PR is the focused implementation candidate, but it cannot merge as-is because the stale predicate misses budget-deferred never-sent entries, the branch is conflicting with current recovery code, real behavior proof is missing, and the TTL/default policy still needs maintainer alignment.

## What This Changes

This PR adds a default 10-minute stale cutoff for first-attempt pending deliveries during startup recovery, reports `skippedStale`, exposes `maxEntryAgeMs`, and adds recovery tests.

## Best Possible Solution

Land or replace this with a rebased shared outbound recovery expiry policy that handles startup recovery, reconnect drain, and budget deferrals while preserving legitimate recent crash retries and aligning operator-facing TTL semantics with https://github.com/openclaw/openclaw/issues/16555.

## Reproduction Assessment

Yes, at source level. Seed an old pending delivery entry with `retryCount: 0` and no `lastAttemptAt`, then run `recoverPendingDeliveries`; current main treats it as eligible and gateway startup calls that recovery path.

## Solution Assessment

No, not as-is. The shared recovery boundary is the right place, but the first-attempt-only predicate misses budget-deferred never-sent entries and the default/config policy still needs maintainer alignment.

- **[P2] Preserve stale filtering after budget deferrals:** `src/infra/outbound/delivery-queue-recovery.ts:219`
  - body: The stale gate only matches `retryCount === 0` with no `lastAttemptAt`, but current recovery budget deferral uses `failDelivery(...)` for unprocessed entries. That increments `retryCount` and sets `lastAttemptAt` without a send, so an old never-sent entry can bypass this guard on the next startup and still replay out of context.
  - confidence: 0.92

After rebase, those entries would be moved to `failed/` before the reconnect ack/skip path under test unless the fixtures are refreshed or the cutoff is disabled for that scenario.
  - confidence: 0.86
