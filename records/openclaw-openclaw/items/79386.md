## Summary

Keep this PR open: the linked watchdog-cascade bug is plausible from current source, but the proposed outer retry path can add an extra cold watchdog attempt after the runner already retried, and the external PR still lacks real behavior proof.

## What This Changes

The PR adds Claude CLI timeout recovery that retries resumed poisoned sessions cold, clears persisted CLI session bindings for resumed timeout failures, and adds regression tests plus a changelog entry.

## Best Possible Solution

Keep the runner-level no-resume retry, but change the outer user-attempt cleanup to avoid a second cold retry on timeout rethrows, then require redacted live proof of poisoned Claude CLI resume recovery before merge.

## Reproduction Assessment

Yes, at source level. Current main can rethrow a `timeout` FailoverError from a resumed CLI session without clearing the persisted binding, but I did not run a live poisoned-transcript gateway reproduction in this read-only review.

## Solution Assessment

No. The runner-level retry is the right boundary for hook correctness, but the outer timeout branch should clear persisted state without launching an extra cold attempt after the runner already retried.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Avoid a second cold retry after runner timeout recovery:** `src/agents/command/attempt-execution.ts:542-548`
