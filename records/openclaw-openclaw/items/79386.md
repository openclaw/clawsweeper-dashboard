## Summary

Keep this PR open: the fix direction is useful and addresses the prior hook-ordering concern from the related stale failover PR, but the new in-runner timeout retry can still leave the poisoned resume binding persisted when the cold retry fails, and the PR still lacks after-fix real behavior proof.

## What This Changes

The PR adds Claude CLI poisoned-resume timeout recovery in the CLI runner and attempt-execution path, with regression tests and a changelog entry.

## Best Possible Solution

Land a hook-aware recovery that guarantees the old Claude CLI resume binding is discarded once a resumed watchdog timeout is observed, including the path where the cold retry itself fails, then add focused regression coverage and redacted real runtime proof.

## Reproduction Assessment

Yes from source inspection and the linked production report, though I did not execute a live poisoned-transcript run in this read-only review. Current main can throw FailoverError reason=timeout for a resumed Claude CLI watchdog kill while only session_expired and missing-transcript paths clear the stored binding.

## Solution Assessment

No. The PR takes the right hook-aware direction, but it should also persist or propagate the resume discard when the in-runner cold retry fails; otherwise the next turn can still start from the same poisoned binding.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Persist the resume discard when retry recovery fails:** `src/agents/cli-runner.ts:497-498`
  - body: Adding `timeout` to the in-runner recovery path means the outer attempt-execution catch no longer sees the first poisoned-resume FailoverError. If the cold retry then fails, `runPreparedCliAgent` returns a failure result instead of throwing, so the new store cleanup in `attempt-execution.ts` never runs and the old `cliSessionBindings["claude-cli"]` can remain persisted for the next turn. Clear the stored binding before taking this recovery path or propagate an invalidation signal even when the cold retry fails.
  - confidence: 0.86
