## Summary

Keep this PR open: the bug-fix direction is useful and source-backed, but the patch can add another full cold watchdog retry after the runner has already retried cold, and the external PR still lacks real behavior proof.

## What This Changes

The PR retries resumed Claude CLI watchdog timeouts without `--resume`, clears the persisted Claude CLI session binding on the user-driven path, and adds regression tests plus a changelog entry.

## Best Possible Solution

Keep the runner-level resumed-timeout cold retry, but make the outer attempt path clear the persisted binding without issuing another cold retry unless it can prove the failed attempt actually used `--resume`; add regression coverage for the failed cold-retry path and real setup proof.

## Reproduction Assessment

Yes, source-reproducible: current main only clears/retries stale bindings for `session_expired` and missing transcripts, while no-output timeouts are thrown as `FailoverError reason="timeout"`. I did not establish a live Claude CLI poisoned-transcript reproduction in this read-only review.

## Solution Assessment

No, not yet. The proposed runner-level retry is the right narrow direction, but the outer timeout branch should not re-run cold after a cold retry already failed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Avoid retrying again after the cold timeout:** `src/agents/command/attempt-execution.ts:548`
