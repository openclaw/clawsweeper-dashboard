The PR reorders shared stale-lock recovery so expired or invalid `createdAt` is honored before PID liveness, adds regression coverage for the reused/live PID case, and removes two redundant image-tool registry casts.

## Best Possible Solution

## Solution Assessment

Yes. Moving timestamp expiry ahead of PID liveness in the shared helper is the narrowest maintainable fix, and the patch keeps non-expired live-PID locks protected by falling back to the existing PID check.
