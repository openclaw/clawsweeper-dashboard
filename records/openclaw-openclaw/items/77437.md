## Summary

Keep open: the current-main regression is source-reproducible and the PR is useful, but the proposed MCP yield callback marks child runs for the requester instead of the yielded subagent run, which can break nested MCP yields and child lifecycle state before merge.

## What This Changes

The PR threads `sessionId` and `onYield` through gateway MCP loopback tool resolution so `sessions_yield` can run on the claude-cli MCP path.

## Best Possible Solution

Land a covered MCP-loopback fix that forwards the existing tool-factory parameters, marks only the yielded session's subagent run when one exists, preserves active child lifecycle state, and adds the required changelog entry.

## Reproduction Assessment

Yes, from source inspection: current main builds MCP loopback tools without `sessionId`/`onYield`, and `sessions_yield` returns `No session context` when `sessionId` is missing. I did not run a live claude-cli loopback reproduction in this read-only review.

## Solution Assessment

No, not as written. The parameter plumbing is the right direction, but the proposed callback matches requester children instead of the yielded run entry and needs regression coverage plus a changelog entry before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Match the yielded run, not its children:** `src/gateway/mcp-http.ts:120`
  - body: This callback runs when the MCP session in `requestContext.sessionKey` calls `sessions_yield`, but the registry pause flag is attached to the run whose `childSessionKey` yielded. Matching `requesterSessionKey` marks that session's children instead, so a nested claude-cli subagent that yields will not have its own run paused, while unrelated child runs can be stamped with `endedAt` before they finish.
  - confidence: 0.87

  - body: This is a user-facing `fix:` for the claude-cli/MCP `sessions_yield` path, and repo policy requires user-facing fixes to update `CHANGELOG.md`. Without that entry the release notes will omit the regression fix.
  - confidence: 0.92
