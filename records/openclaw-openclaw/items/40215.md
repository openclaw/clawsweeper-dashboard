## Summary

Keep open. Current main still formats `/usage` from per-response usage/cost data plus an optional session key, lacks cumulative context/window/model footer data, and has no inherited `agents.defaults.responseUsage`; #69618 is open and explicitly remains the closing implementation path.

## What This Changes

Request to enrich `/usage full` with model and cumulative context-window usage, plus a global `agents.defaults.responseUsage` default.

## Best Possible Solution

## Reproduction Assessment

Yes. Current-main source inspection shows `/usage full` consumes only per-response usage/cost plus session key, while the stored context fields are surfaced through `/status` and session APIs rather than the footer.

## Solution Assessment

Yes as a product direction, but it is not implemented on main. The maintainable path is to review, clean up, and land #69618 or a narrower replacement without unrelated drift.
