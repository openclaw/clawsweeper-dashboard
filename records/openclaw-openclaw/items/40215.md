## Summary

Keep open. Current main still renders the `/usage` footer from per-response token/cache/cost data plus an optional session key, with no cumulative context-used/window/percentage or inherited `agents.defaults.responseUsage` default. The provided GitHub context shows open PR #69618 explicitly closes this issue and remains the implementation candidate, so this should not be closed before that PR or a replacement lands.

## What This Changes

Request to enrich channel `/usage full` footers with model plus cumulative context used/window/percentage and add an inherited `agents.defaults.responseUsage` default.

## Best Possible Solution

Keep this issue open until maintainers review and land #69618 or a narrower replacement. The end state should enrich the late-bound reply footer with model/context/window data, add documented inherited `agents.defaults.responseUsage` semantics while preserving explicit session overrides, and cover formatter, runner, command, config schema, docs, TUI/Web UI, and session tests.

## Reproduction Assessment

Yes. The current-main source path is deterministic: `/usage full` resolves session `responseUsage`, calls `formatResponseUsageLine({ usage, showCost, costConfig })`, and appends only the session key, so a mocked reply with usage cannot produce context/window data.

## Solution Assessment

Yes as a product direction. The maintainable fix belongs in the reply footer/config path with late-bound model/context/session values; however the open PR still needs normal maintainer review before this issue can be closed.
