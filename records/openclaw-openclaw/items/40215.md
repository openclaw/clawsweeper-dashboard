## Summary

Keep open: current main still formats `/usage` as per-response tokens/cache/cost plus optional session key, while cumulative context/model/window data and an inherited default are still only covered by open implementation PRs. The discussion has real user demand, including a recent Feishu fleet report of context overflow without footer visibility.

## What This Changes

Request to show cumulative model and context-window usage in the `/usage` footer and add a global default for response usage.

## Best Possible Solution

Adopt one maintained implementation that enriches `/usage full` with model/context/window/cost data and adds a typed inherited `agents.defaults.responseUsage` contract with explicit override and regression coverage.

## Reproduction Assessment

Yes for the capability gap. Current source inspection shows `/usage full` has no context/model/window inputs and `responseUsage` remains session-scoped rather than inherited from agent defaults.

## Solution Assessment

No for starting a duplicate repair lane. The maintainable path is to review, narrow, or replace the existing open implementation PRs after maintainers choose the public config contract and override semantics.
