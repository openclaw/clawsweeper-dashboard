## Summary

Keep open: the source-level bug report is plausible, but the PR currently makes the bundled Copilot policy diverge from the runtime provider policy for one advertised model, and external real behavior proof is still missing.

## What This Changes

Adds a GitHub Copilot `provider-policy-api` artifact and focused tests so bundled policy lookups can expose `xhigh` thinking levels for selected Copilot models.

## Best Possible Solution

Keep the lightweight policy artifact approach, but make the bundled and runtime Copilot thinking profiles agree for every advertised `xhigh` model and add inspectable redacted dashboard or terminal proof before merge.

## Reproduction Assessment

Yes from source inspection, though not live-reproduced locally. Current main has no Copilot bundled policy artifact, and the session thinking path falls back through provider policy to base thinking levels when no active profile or bundled policy supplies `xhigh`.

## Solution Assessment

No. Adding a provider-owned bundled artifact is the right boundary, but the current PR is not the best complete fix until the bundled list and runtime `COPILOT_XHIGH_MODEL_IDS` list are kept in parity, preferably with coverage that fails on drift.
