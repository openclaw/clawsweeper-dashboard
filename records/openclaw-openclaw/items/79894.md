## Summary

Keep open: this PR targets a real GitHub Copilot cold-path thinking-policy gap, but the branch still makes the bundled policy diverge from the active runtime policy and lacks inspectable real behavior proof.

## What This Changes

The PR adds a GitHub Copilot `provider-policy-api.ts` plus focused tests so bundled provider-policy thinking resolution can return `xhigh` for selected Copilot models.

## Best Possible Solution

Keep the lightweight provider-policy artifact, but derive or mirror the exact same Copilot thinking profile used by the active runtime and cover both cold bundled-policy and active-provider paths.

## Reproduction Assessment

Yes, by source inspection: current main lacks a GitHub Copilot provider-policy artifact, and gateway session defaults/rows read thinking levels through the shared resolver. I did not live-run the dashboard in this read-only review.

## Solution Assessment

No, not yet. Adding the bundled policy artifact is the right boundary, but the duplicated xhigh list must be synchronized with the active runtime profile before this is the narrow maintainable fix.
