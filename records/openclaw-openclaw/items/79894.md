## Summary

Keep open: the bundled policy artifact is the right surface for the cold dashboard path, but this PR is not merge-ready because it diverges from the active Copilot runtime profile, imports a deprecated SDK barrel that fails plugin contract CI, and still lacks inspectable real behavior proof.

## What This Changes

The branch adds a GitHub Copilot bundled `provider-policy-api.ts` plus tests so cold provider-policy resolution can expose `xhigh` for selected Copilot models.

## Best Possible Solution

Keep the bundled policy artifact, but share or mirror one lightweight Copilot thinking-profile source across `index.ts` and `provider-policy-api.ts`, use `string-coerce-runtime`, and attach redacted dashboard or terminal proof.

## Reproduction Assessment

Yes. Source inspection shows current main lacks a GitHub Copilot bundled provider-policy artifact, while gateway session defaults and rows resolve thinking options through the shared provider profile path; I did not live-run the dashboard in this read-only review.

## Solution Assessment

No, not as patched. Adding the bundled artifact is the right boundary, but the runtime and bundled Copilot profiles must agree and the new file must avoid the deprecated SDK barrel before this is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Keep Copilot thinking profiles in sync:** `extensions/github-copilot/provider-policy-api.ts:10-15`
  - body: This bundled policy now advertises `xhigh` for `gpt-5.5` and `claude-opus-4.7-1m-internal`, but the active GitHub Copilot runtime profile still omits those ids. Because the thinking resolver used by session validation prefers an active provider profile before the bundled fallback, the dashboard can accept/show `xhigh` on the cold path and later reject or remap it after the plugin is active.   - confidence: 0.95

Import `normalizeOptionalLowercaseString` from `openclaw/plugin-sdk/string-coerce-runtime` so the synchronous policy surface stays on the supported lightweight subpath.
  - confidence: 0.96
