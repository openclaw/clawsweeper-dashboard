## Summary

Keep open. The PR targets a real startup/dashboard policy gap, but the patch still lets the bundled and active GitHub Copilot thinking profiles disagree for one model and it lacks inspectable after-fix proof.

## What This Changes

Adds a GitHub Copilot `provider-policy-api.ts` and unit tests so bundled provider-policy thinking resolution can return `xhigh` for selected Copilot models.

## Best Possible Solution

Keep the provider-policy artifact, but derive or mirror the same lightweight Copilot thinking profile used by the active runtime and cover both cold bundled and active runtime paths.

## Reproduction Assessment

Yes, by source inspection: current main lacks a GitHub Copilot bundled provider-policy artifact, and gateway defaults/session rows read thinking options through the shared provider profile resolver. I did not live-run the dashboard in this read-only review.

## Solution Assessment

No, not yet. Adding a bundled provider-policy artifact is the right boundary, but this implementation must keep the bundled policy and active runtime Copilot profile synchronized before it is the narrow maintainable fix.

Full review comments:

- **[P2] Sync bundled policy with the runtime profile:** `extensions/github-copilot/provider-policy-api.ts:14`
  - body: For `github-copilot/claude-opus-4.7-1m-internal`, this bundled policy advertises `xhigh`, but the active runtime `COPILOT_XHIGH_MODEL_IDS` still omits that model.   - confidence: 0.82
