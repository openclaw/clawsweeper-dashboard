## Summary

Keep open. Current main still loads the combined gateway session store before checking the transcript session-key cache, and this focused PR addresses that hot path with a fingerprinted cache hit while preserving the existing full-validation fallback; I found no blocking review finding.

## What This Changes

The PR changes gateway transcript session-key resolution to cache keys with a session-store fingerprint, adds focused gateway tests, and records the fix in the changelog.

## Best Possible Solution

Land this focused gateway cache fix after normal maintainer merge handling if maintainers agree it covers the transcript-update store-scan root cause.

## Reproduction Assessment

Yes. The source path is clear: transcript broadcasts call `resolveSessionKeyForTranscriptFile`, current main loads the combined store before its cache lookup, and the PR adds a unit assertion that repeated lookup no longer reloads the store.

## Solution Assessment

Yes. The fingerprinted early return is a narrow maintainable fix because it avoids repeated full store loads while preserving full validation whenever the store fingerprint changes or cannot be computed.
