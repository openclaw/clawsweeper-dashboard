## Summary

Current main now fixes the Google Vertex custom-model visibility path by merging explicit provider config with the implicit provider catalog and resolving ADC-backed Google Vertex auth to a non-secret marker before writable provider filtering. The fix is main-only; it is not in v2026.6.10.

## What This Changes

The issue reports that Google Vertex custom model overrides in openclaw.json do not appear in the model list.

## Best Possible Solution

Keep the current-main provider normalization; users on v2026.6.10 need a later release or a main build for this fix.

## Reproduction Assessment

Yes from source. The released path could drop custom Google Vertex models without writable provider auth details, while current main fills the ADC-backed marker before the writable-provider filter and appends configured provider rows to model-list output.

## Solution Assessment

Yes. The current fix preserves the existing openclaw.json override surface and repairs provider normalization/auth marker handling instead of adding a second override mechanism.
