## What This Changes

The issue reports that `openclaw.json` model overrides under `models.providers.google-vertex.models` are not reflected by `openclaw models list --provider google-vertex --all`.

## Best Possible Solution

Make arbitrary Google Vertex configured model rows appear consistently in model-list and runtime catalog surfaces through the existing `models.providers.google-vertex.models` contract, while leaving credentialed live Vertex discovery to https://github.com/openclaw/openclaw/issues/96597.
