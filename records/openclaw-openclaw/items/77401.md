## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/77401 on current main. The observable bug is that Telegram `streaming.mode: "partial"` can show the final assembled reply before the last tool/progress messages on tool-heavy turns. Keep the fix narrow: add or reuse a shared drain/ack path for channel-visible progress callbacks from embedded agent events/partials so Telegram can flush all pending progress work before final delivery starts. Do not add sleeps, Telegram network timing heuristics, new config, or streaming-mode default changes. Add deterministic regression coverage for a slow progress callback that is invoked fire-and-forget by the embedded agent and would otherwise enqueue after final delivery. Update `CHANGELOG.md` because this is user-facing behavior. Preserve existing chunked-send serialization and preview cleanup behavior.
