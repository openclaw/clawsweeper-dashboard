## Summary

Keep this PR open. It addresses a real current-main `/status` target-session context-window gap, but the branch still drops inherited `agents.defaults.contextTokens` in the new resolved config field and is missing the required user-facing changelog entry.

## What This Changes

## Best Possible Solution

Keep the targeted target-session `/status` fix, make the new resolved config field return `entry.contextTokens ?? agentDefaults?.contextTokens`, add inherited-default regression coverage, and record the user-visible fix in `CHANGELOG.md` without folding in the separate #39859 model-window cap work.

Yes. Current main can be checked by giving `handleStatusCommand` a target session with `contextTokens: 500000` while `params.contextTokens` is `272000`; the code still forwards `272000`. ## Solution Assessment

No, not yet. The `/status` routing change is the right narrow shape, but the resolver must preserve inherited defaults before the new field is a reliable effective context-window value.
