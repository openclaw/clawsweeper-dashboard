## Summary

Keep open: the PR addresses a real source-reproducible /status context-window bug, but the branch is still incorrect because the new resolved config field drops inherited `agents.defaults.contextTokens`, and the external contributor has not posted real behavior proof.

## What This Changes

The PR adds `contextTokens` to resolved agent config, routes `/status` through the target session entry's persisted context window, and adds focused tests.

## Best Possible Solution

Rebase the branch, keep the target-session `/status` routing fix, resolve `contextTokens` as `entry.contextTokens ?? agentDefaults?.contextTokens`, add inherited-default coverage, and require redacted after-fix proof before merge.

## Reproduction Assessment

Yes. Source inspection shows current main can select a target session entry for `/status` while still passing the caller/runtime `params.contextTokens`, so a stored target-session context window can be displayed incorrectly.

## Solution Assessment

No, not yet. The `/status` change is the right narrow fix, but the new resolved config field needs the inherited default fallback and the external contributor still needs to provide real behavior proof.
