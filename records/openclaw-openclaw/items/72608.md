## Summary

Keep open: this PR targets a valid source-reproducible `/status` bug, but the patch still drops inherited `agents.defaults.contextTokens` in the new resolved config field and the external contributor has not posted real after-fix behavior proof.

## What This Changes

The PR adds `contextTokens` to resolved agent config, passes the target session entry's context window into `/status`, and adds regression coverage for both paths.

## Best Possible Solution

Keep the narrow target-session `/status` fix, resolve `contextTokens` with inherited defaults, add default-only coverage, and require redacted after-fix proof before merge.

## Reproduction Assessment

Yes. Source inspection on current main shows `/status` resolves the target session entry but passes `params.contextTokens`; the PR's added mock regression would fail on that path, though I did not run tests in this read-only review.

## Solution Assessment

No, not yet. Passing `targetSessionEntry?.contextTokens ?? params.contextTokens` is the right narrow status change, but the new resolved config field must preserve inherited defaults and the external contributor still needs real behavior proof.
