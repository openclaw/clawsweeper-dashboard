## Summary

Keep open: the target-none heartbeat stale-replay bug is source-reproducible, but this PR clears durable pending-final-delivery state solely because the heartbeat target resolves to `none`, which can discard retry state in shared sessions. The branch also lacks after-fix real behavior proof.

## What This Changes

The PR adds a heartbeat reply option that skips pending-final-delivery replay for `target:none` heartbeats, clears pending fields, and adds fast-path regression coverage.

## Best Possible Solution
