## Summary

Keep open: the PR targets real BlueBubbles helper/self-chat problems, but the patch broadens self-chat suppression past current tested behavior and moves BlueBubbles-specific status parsing into core infrastructure.

## What This Changes

The PR broadens BlueBubbles self-chat reflection suppression, projects helper status from probes, surfaces helper-disconnect status issues, and adds a local imsg fallback for helper-disconnected sends.

## Best Possible Solution

Keep the BlueBubbles helper diagnostics and fallback inside plugin-owned code, while preserving the existing distinction between assistant outbound self-chat reflections and user-authored self-chat prompts.

## Reproduction Assessment

Yes, for the PR regression at source level: current main has a self-chat test that expects user-authored reflected prompts to dispatch, and the PR flips that behavior by caching every self-chat fromMe copy. I did not execute the tests in this read-only review.

## Solution Assessment

No; the helper probe/status work is directionally useful, but the patch should keep self-chat suppression guarded by assistant-outbound evidence and should avoid BlueBubbles-specific fallback parsing in src/infra.
