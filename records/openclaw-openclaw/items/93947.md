## What This Changes

The issue reports that `openclaw setup --non-interactive` can show health-failure hints for `--install-daemon` and `--skip-health` even though those flags are only supported by `openclaw onboard`.

## Best Possible Solution

Land one focused linked PR, preferably the proof-sufficient clean candidate or an equivalent reviewed fix, then close this issue and duplicate PR attempts after the corrected hint reaches main.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `setup --non-interactive` routes into the non-interactive local setup health path, while `setup` lacks the flags named by that path's current recovery hint.

## Solution Assessment

Yes. The narrow maintainable fix is to update the existing health-failure hint and adjacent regression assertion to name `openclaw onboard`, without adding duplicate setup aliases or changing flag ownership.
