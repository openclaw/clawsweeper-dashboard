Keep this issue open: current main still emits the bare `--install-daemon` and `--skip-health` hints from the setup-routed non-interactive path, and the latest release still has the same text. Several open PRs are candidate fixes, so the issue should wait for one focused fix to merge.

## What This Changes

The issue asks for non-interactive setup health-failure hints to point users to `openclaw onboard` for daemon and health-skip flags that setup does not support.

## Best Possible Solution

Land one focused PR that changes the non-interactive local health hint to name `openclaw onboard` and updates the regression test, then close this issue through the merged fix.

## Reproduction Assessment

Yes from source inspection, though I did not run the CLI path because this review is read-only. `setup` routes non-interactive onboarding but lacks the daemon/health flags, while the local health failure still emits those bare flags.

## Solution Assessment

Yes, the narrow text-and-test fix proposed by the linked PRs is the right shape. Adding the onboard-only flags to `setup` would expand the command surface unnecessarily.
