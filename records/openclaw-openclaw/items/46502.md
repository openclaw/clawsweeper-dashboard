## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Re-enable LaunchAgent before direct kickstart:** `src/daemon/launchd.ts:647`
  - body: Current main clears persisted launchctl disabled state before `kickstart -k`, so `openclaw gateway restart` can recover a LaunchAgent that was previously disabled. This branch goes straight to kickstart on the direct path, which can leave the service disabled and unmanaged; keep the `launchctl enable` step before kickstart.
  - confidence: 0.91
- **[P2] Restore launchd repair after kickstart failure:** `src/daemon/launchd.ts:653-654`
  - body: Current main checks whether a failed `kickstart -k` left the LaunchAgent unloaded and tries to bootstrap it before surfacing the original failure. This branch throws immediately on non-not-loaded kickstart failures, so a restart can leave the service unloaded in a state current main repairs.
  - confidence: 0.87
- **[P2] Skip kickstart after bootstrap repair:** `src/daemon/launchd.ts:666`
  - body: Current main treats a successful bootstrap as enough because RunAtLoad already starts the LaunchAgent. This branch bootstraps and then kickstarts again in the not-loaded fallback, which can terminate the freshly booted gateway and reintroduce the startup race current main avoids.
  - confidence: 0.83
