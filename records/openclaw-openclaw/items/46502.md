## Review Findings

Full review comments:

- **[P1] Preserve current probe auth fields:** `src/gateway/probe.ts:19-29`
  - body: Current main callers read `probe.auth.capability` and `probe.server.version` during restart health and gateway status checks. This branch's `GatewayProbeResult` shape omits those fields, so rebasing or merging this file as-is breaks the current probe contract and callers that depend on it.
  - confidence: 0.91
- **[P2] Re-enable LaunchAgent before direct kickstart:** `src/daemon/launchd.ts:647`
  - body: Current main clears persisted `launchctl disable` state before `kickstart -k`, so `openclaw gateway restart` can recover a service that was previously disabled. This branch goes straight to kickstart on the direct path, which can leave the LaunchAgent disabled and unmanaged.
  - confidence: 0.91
- **[P2] Restore launchd repair after kickstart failure:** `src/daemon/launchd.ts:653-654`
  - body: Current main checks whether a failed kickstart left the LaunchAgent unloaded and tries to bootstrap it before surfacing the original failure. This branch throws immediately on non-not-loaded kickstart failures, so restart can leave behind a state current main repairs.
  - confidence: 0.87
- **[P2] Skip kickstart after bootstrap repair:** `src/daemon/launchd.ts:666`
  - body: Current main treats a successful bootstrap as enough because RunAtLoad starts the LaunchAgent. This branch bootstraps and then kickstarts again in the not-loaded fallback, which can terminate the freshly booted gateway and reintroduce the startup race current main avoids.
  - confidence: 0.83
