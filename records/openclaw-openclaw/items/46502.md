Full review comments:

- **[P1] Rebase without dropping probe diagnostics:** `src/gateway/probe.ts:19-29`
  - body: The PR head replaces current main's richer `GatewayProbeResult` contract with the older shape that has no `auth`, `server`, or `connectErrorDetails`, and the implementation also drops the current safe timeout/readiness and non-mutating device identity path. Replay the watchdog changes on top of the current probe contract so status and health diagnostics do not regress.
  - confidence: 0.9
- **[P1] Keep the trusted Windows command wrapper:** `src/process/exec.ts:240-243`
  - body: This branch routes Windows batch execution through `process.env.ComSpec ?? "cmd.exe"`, dropping current main's trusted System32 `cmd.exe` wrapper. A poisoned environment can steer command execution through an untrusted wrapper, so preserve `resolveTrustedWindowsCmdExe()` when adding `baseEnv` and abort support.
  - confidence: 0.92
- **[P2] Preserve Windows process-tree aborts:** `src/process/exec.ts:178-184`
  - body: The new helper only calls `child.kill("SIGKILL")`; current main uses Windows `taskkill /T /F` and close-state settlement so aborts/timeouts kill wrapper child trees and report stable exit state. Without that, watchdog aborts can leave the underlying service command running after the wrapper is killed.
  - confidence: 0.84
