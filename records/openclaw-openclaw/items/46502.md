- **[P1] Restore current probe metadata:** `src/gateway/probe.ts:19-29`
  - body: Current main callers use `probe.server?.version` and `probe.auth.capability` to classify restart health and status. This stale branch keeps the older `GatewayProbeResult` shape without those fields, so merging it as-is would drop the current probe contract and break those callers.

  - body: Current main refuses a LaunchAgent restart when the gateway port is still busy after stale-process cleanup, which avoids kickstarting into a non-owned listener. This branch only runs cleanup and then continues to `kickstart`, losing that safety check.
  - confidence: 0.86
- **[P2] Re-enable LaunchAgent before kickstart:** `src/daemon/launchd.ts:647`
  - body: Current main clears persisted `launchctl disable` state before direct restart so `openclaw gateway restart` can recover a service stopped with `--disable`. This branch goes straight to `kickstart -k`, so the service can remain disabled.
  - confidence: 0.91
- **[P2] Restore kickstart failure bootstrap repair:** `src/daemon/launchd.ts:653-654`
  - body: Current main probes whether a failed kickstart left the LaunchAgent unloaded and attempts a bootstrap repair before surfacing the original failure. This branch throws immediately for non-not-loaded kickstart failures, dropping that recovery path.
  - confidence: 0.88
- **[P2] Avoid kickstarting after bootstrap repair:** `src/daemon/launchd.ts:666`
  - body: Current main treats a successful bootstrap as enough because `RunAtLoad` starts the LaunchAgent. This branch bootstraps and then kickstarts again, which can terminate the freshly booted gateway and reintroduce the startup race current main avoids.
  - confidence: 0.84
