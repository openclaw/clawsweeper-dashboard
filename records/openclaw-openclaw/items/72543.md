## Summary

Keep open. Current main still has the Control UI gap: the shared chat session option builder used by both desktop and mobile chat selectors only includes discovered session rows plus the active selection, so the canonical `agent:main:main` option can still be absent when the user is on another session and `sessions.list` omits the main row.

## What This Changes

The issue requests that Control UI always include a stable canonical main chat session option and normalize main-session aliases to `agent:main:main`.

## Best Possible Solution

Add a focused Control UI fix that derives the canonical main key from Gateway session defaults, injects one stable main-session option into the shared option builder when missing, and preserves existing grouped label/disambiguation behavior across desktop and mobile.

## Reproduction Assessment

Yes. A focused unit reproduction can call `resolveSessionOptionGroups` with active session `agent:main:subagent:...`, a `sessionsResult` that lacks `agent:main:main`, and `state.hello.snapshot.sessionDefaults.mainSessionKey`; current main returns only discovered rows plus the active key.

## Solution Assessment

Yes. The narrow source-level repair belongs in the shared Control UI option builder while reusing the existing Gateway defaults and alias normalization; a generic or agent-derived label is safer than hard-coding the reporter's example name.
