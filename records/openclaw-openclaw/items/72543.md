## Summary

Keep open: current `main` still does not guarantee that the canonical main session appears in the Control UI chat dropdown when `sessions.list` omits it and another session is selected.

## What This Changes

Request that the Control UI chat session dropdown always include and normalize the canonical main session option, mapped to `agent:main:main`.

## Best Possible Solution

Add a focused Control UI fix that derives the canonical main key from Gateway session defaults, injects one stable main-session option when missing, and preserves existing grouping, filtering, and duplicate-label disambiguation.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to call `resolveSessionOptionGroups` with active session `agent:main:subagent:...`, a `sessionsResult` that lacks `agent:main:main`, and Gateway defaults containing `mainSessionKey`; current `main` returns only discovered rows plus the active key.

## Solution Assessment

Yes. The narrowest maintainable repair is in the shared source option builder, using existing Gateway defaults; alias normalization in `app-gateway.ts` should stay focused on selected and persisted settings.
