## Summary

Keep open. Current main still has the reported gap: the shared Control UI session option builder only emits discovered sessions plus the active session, so `agent:main:main` is omitted when the Gateway knows the canonical main session but `sessions.list` does not include it.

## What This Changes

The issue asks the Control UI chat session selector to always expose the canonical main session and normalize `main` aliases to `agent:main:main`.

## Best Possible Solution

Add a shared Control UI helper path that derives the canonical main key from `hello.snapshot.sessionDefaults.mainSessionKey`, guarantees a stable main-session option before or alongside discovered sessions, keeps desktop and mobile selectors aligned, and uses generic agent/session metadata for the label instead of hard-coding a persona name.

## Reproduction Assessment

Yes. A focused unit reproduction can call `resolveSessionOptionGroups` with active session `agent:main:subagent:...`, a `sessionsResult` lacking `agent:main:main`, and Gateway defaults containing `mainSessionKey`; current code returns supplied rows plus the active key, so the canonical main option is absent.

## Solution Assessment

Yes. The narrow maintainable fix is in the shared option-group builder plus existing default-session normalization path; adding a new Gateway API or hard-coding `Chelmsworth` would be broader than needed.
