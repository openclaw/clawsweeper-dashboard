## Summary

Keep open. Current main still has the reported Control UI gap: the shared chat session option builder only includes discovered session rows plus the currently selected session, while the existing alias normalization path does not guarantee a visible canonical main-session option in the dropdown.

## What This Changes

The issue requests that the Control UI chat session dropdown always include and normalize the canonical main session entry `agent:main:main`.

## Best Possible Solution

Add a narrow shared Control UI fix that derives the canonical main key from `hello.snapshot.sessionDefaults.mainSessionKey`, guarantees a generic main-session option before or alongside discovered sessions, keeps desktop and mobile selectors aligned, and avoids hard-coding a persona name.

## Reproduction Assessment

Yes. A focused unit reproduction can call `resolveSessionOptionGroups` with active session `agent:main:subagent:...`, a `sessionsResult` that omits `agent:main:main`, and Gateway defaults containing `mainSessionKey`; current code returns discovered rows plus the active key, so the canonical main option is absent.

## Solution Assessment

Yes. The maintainable fix belongs in the shared Control UI option builder and existing default-session normalization path; a new Gateway API or a hard-coded `Chelmsworth` label would be broader and less generic than needed.
