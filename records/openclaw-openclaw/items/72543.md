## Summary

Keep open. Current main still normalizes some main-session aliases, but the shared Control UI session dropdown builder only includes discovered session rows plus the active key, so it still does not guarantee a visible canonical main-session option when `agent:main:main` is absent from `sessions.list`.

## What This Changes

Request that the Control UI chat session dropdown always include a stable canonical main-session option and normalize `main` aliases to the canonical session key.

## Best Possible Solution

Add a shared Control UI path that derives the canonical main key from `hello.snapshot.sessionDefaults.mainSessionKey`, guarantees that option in the shared session option builder for desktop and mobile, keeps existing alias normalization aligned for persisted settings, and covers the missing case with focused UI unit tests.

## Reproduction Assessment

Yes. A focused unit reproduction can call `resolveSessionOptionGroups` with active session `agent:main:subagent:x`, a `sessionsResult` that omits `agent:main:main`, and Gateway defaults containing `mainSessionKey`; current code returns supplied rows plus the active key, so the canonical main option is absent.

## Solution Assessment

Yes. The narrow maintainable fix is to update the shared option-group builder and existing defaults normalization using the Gateway snapshot defaults; adding a new Gateway API or hard-coding a persona label would be broader than needed.
