## Summary

Keep open: current main still has the observable gap because `resolveSessionOptionGroups` never injects `sessionDefaults.mainSessionKey` when it is absent from `sessions.list`; Gateway already exposes the canonical key, so this is a narrow Control UI source fix.

## What This Changes

The issue requests that the Control UI chat dropdown always include a stable canonical main-session option and normalize main-session aliases to `agent:main:main`.

## Best Possible Solution

Derive the canonical main key from Gateway session defaults, inject one stable main-session option when missing, and preserve the existing grouping, filtering, and duplicate-label behavior.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to call `resolveSessionOptionGroups` with active session `agent:main:subagent:...`, a `sessionsResult` that lacks `agent:main:main`, and Gateway defaults containing `mainSessionKey`; current main returns only discovered rows plus the active key.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the shared Control UI option builder, with existing alias normalization reused or lightly extended only if selected/persisted settings still drift.
