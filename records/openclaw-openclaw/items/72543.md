## Summary

Keep open: current main still does not guarantee that the canonical main session appears in the Control UI chat dropdown when `sessions.list` omits it and another session is active.

## What This Changes

The issue requests a stable canonical main-session option in the Control UI chat session dropdown, plus normalization of `main` aliases to `agent:main:main`.

## Best Possible Solution

Add a focused Control UI fix that derives the canonical main key from Gateway session defaults, injects one stable main-session option when missing, and keeps existing grouping/disambiguation behavior intact.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to call `resolveSessionOptionGroups` with active session `agent:main:subagent:...`, a `sessionsResult` that lacks `agent:main:main`, and Gateway defaults containing `mainSessionKey`; current main returns only discovered rows plus the active key.

## Solution Assessment

Yes. The best repair is the shared source-level option builder using existing Gateway defaults; avoid hard-coding `Chelmsworth` unless maintainers explicitly want persona-specific copy.
