## Summary

Keep open: current main still does not guarantee the canonical main session appears in the Control UI chat dropdown when `sessions.list` omits it and another session is active. The shared dropdown builder only adds discovered rows plus the current session, while Gateway already exposes the canonical main-session defaults needed for a narrow UI fix.

## What This Changes

The issue asks the Control UI chat session dropdown to always include and normalize the canonical `agent:main:main` main-session option.

## Best Possible Solution

Add a focused Control UI source fix that derives the canonical main key from Gateway session defaults, injects a single stable main-session option when missing, and keeps existing grouping and duplicate-label disambiguation intact.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to call `resolveSessionOptionGroups` with active session `agent:main:subagent:child`, Gateway defaults containing `mainSessionKey: "agent:main:main"`, and a `sessionsResult` list that lacks `agent:main:main`; current main returns only the listed rows plus the active key.

## Solution Assessment

Yes. The narrowest maintainable fix belongs in the shared option builder using existing Gateway defaults; the compiled-bundle hotfix and a hard-coded persona label are not the right source-level direction.
