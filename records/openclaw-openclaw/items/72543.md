## Summary

Keep open: current main still has the dropdown gap. Alias normalization is present, but the shared Control UI option builder still only renders discovered session rows plus the active session, so it can omit the canonical main session when another session is active.

## What This Changes

The issue requests a stable canonical main-session option in the Control UI chat dropdown and normalization of main aliases to the canonical session key.

## Best Possible Solution

Add a focused Control UI fix that derives the canonical main key from Gateway session defaults, inserts a stable main-session option when absent, and preserves existing grouping, filtering, and label disambiguation behavior.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to call `resolveSessionOptionGroups` with active session `agent:main:subagent:...`, a `sessionsResult` that lacks `agent:main:main`, and Gateway defaults containing `mainSessionKey`; current main returns discovered rows plus the active key only.

## Solution Assessment

Yes. The narrowest maintainable solution is in the shared session option builder, using existing Gateway defaults; hard-coding `Chelmsworth` in source is not the best default unless product owners choose that label.
