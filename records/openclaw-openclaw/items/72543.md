## Summary

Keep open. Current main still has the reported Control UI gap: the shared chat session option builder lists discovered sessions plus the active key, but does not inject the Gateway-provided canonical main session when `sessions.list` omits it.

## What This Changes

Request to make the Control UI chat dropdown always show a stable canonical main-session option and normalize main-session aliases to `agent:main:main`.

## Best Possible Solution

Add shared Control UI option-building logic that derives the canonical main key from Gateway session defaults, guarantees a stable main-session option alongside discovered sessions, and keeps desktop/mobile selectors plus persisted alias normalization aligned without a new Gateway API.

## Reproduction Assessment

Yes. A focused unit reproduction can call `resolveSessionOptionGroups` with active session `agent:main:subagent:...`, a `sessionsResult` that lacks `agent:main:main`, and Gateway defaults containing `mainSessionKey`; current code returns discovered rows plus the active key, so the canonical main option is absent.

## Solution Assessment

Yes, the narrow maintainable fix is in the shared Control UI option builder plus the existing default-session normalization path. The safer alternative to the proposed example is to use generic agent/session metadata for the label instead of hard-coding `Chelmsworth`.

## Repair Work Prompt

Fix the Control UI chat session dropdown for https://github.com/openclaw/openclaw/issues/72543. The observable bug is that the dropdown can omit the canonical main session when `sessions.list` does not include `agent:main:main` and the active session is another key. Implement the fix in source, not compiled dist: derive the canonical key from `hello.snapshot.sessionDefaults.mainSessionKey` with existing main-key/default-agent fallbacks, inject a stable main-session option in the shared `resolveSessionOptionGroups` path used by desktop and mobile, and keep alias normalization for `main`/default-agent main keys aligned for `sessionKey`, `settings.sessionKey`, and `settings.lastActiveSessionKey`. Do not hard-code `Chelmsworth`; use generic agent/session metadata such as the agent display name plus `/main` when available. Avoid adding a new Gateway API unless the existing snapshot defaults are insufficient. Add focused tests for the missing-main-row case and any settings normalization gap, and add a user-facing `CHANGELOG.md` fix entry with proper non-bot credit.
