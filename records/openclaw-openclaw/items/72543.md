## Summary

Keep open. Current main still has the Control UI gap: the shared chat session option builder used by desktop and mobile selectors only includes discovered rows plus the active selection, so `agent:main:main` can still be absent when the user is on another session and the session list omits the canonical main row.

## What This Changes

Ensure the Control UI chat session dropdown always includes the canonical main session `agent:main:main` and keeps plain `main` aliases normalized to that key.

## Best Possible Solution

Add a focused Control UI fix that derives the canonical main key from Gateway session defaults, injects one stable main-session option into the shared option builder when missing, and preserves existing grouped label/disambiguation behavior across desktop and mobile.

## Reproduction Assessment

Yes. A focused unit reproduction can call `resolveSessionOptionGroups` with active session `agent:main:subagent:...`, a `sessionsResult` that lacks `agent:main:main`, and `state.hello.snapshot.sessionDefaults.mainSessionKey`; current main returns only discovered rows plus the active key.

## Solution Assessment

Yes, the narrow source-level repair belongs in the shared Control UI option builder while reusing the existing Gateway defaults and alias normalization. The safer implementation is to derive a generic or agent-based label instead of hard-coding `Chelmsworth`.
