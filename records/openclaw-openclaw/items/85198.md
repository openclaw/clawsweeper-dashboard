## Summary

Keep open for rework: this is a focused bug-fix attempt, but the patch conflicts with the existing Copilot payload sanitizer that deliberately drops unsafe reasoning replay IDs instead of rewriting them. Merging this as-is could forward an invented reasoning replay reference and still fail before an assistant reply.

## What This Changes

The PR adds a shared Responses replay ID normalizer in `src/agents/openai-transport-stream.ts`, applies it to overlong reasoning and assistant message IDs, and adds a request-builder regression test.

## Best Possible Solution

Keep reasoning replay safety at the Copilot provider boundary: drop or omit unsafe reasoning replay items, prove the wrapped GitHub Copilot Responses path no longer forwards overlong IDs, and only change core if a concrete wrapper-bypass path is confirmed.

## Reproduction Assessment

Unclear for the full user path. Source inspection shows the raw request-builder seam can preserve overlong reasoning IDs, but current main’s registered Copilot wrapper sanitizes payloads before send and the PR proof does not exercise that wrapper or a live Telegram/Copilot route.

## Solution Assessment

No. Hashing reasoning item IDs is not the safest fix because the existing Copilot sanitizer treats unsafe reasoning IDs as server-bound state that should be dropped, so the better repair is to prove and fix any wrapper-bypass/provider-boundary gap.
