## Summary

Keep this open. Current main still has the reported prompt behavior, and the linked closing PR is open and unmerged, so the issue should continue tracking the bug until PR #73566 or an equivalent fix lands.

## What This Changes

`plugins uninstall` should fail cleanly when confirmation stdin closes before an answer, while preserving piped `y`/`n` answers and `--force`.

## Best Possible Solution

Review and land PR #73566 or an equivalent narrow fix that makes the shared prompt helper settle on readline close/EOF, maps that specific condition in `plugins uninstall` to exit 1 with interactive or `--force` guidance, and adds focused prompt plus uninstall regression tests.

## Reproduction Assessment

Yes. Current main still awaits `readline.question()` in the non-forced uninstall path, and a minimal Node readline smoke reproduces exit 13 for closed stdin while newline-delimited `n` input resolves normally.

## Solution Assessment

Yes. Handling EOF in the shared prompt primitive and catching that specific condition in `plugins uninstall` is the narrow maintainable fix; a blanket non-TTY gate would break valid piped answers.
