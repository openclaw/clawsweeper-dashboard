## What This Changes

Request to make `openclaw plugins uninstall <id>` fail cleanly when confirmation stdin closes, while preserving piped `y`/`n` answers and `--force`.

## Best Possible Solution

Review and land PR #73566, or an equivalent narrow fix, so the shared prompt primitive settles on readline close/EOF and `plugins uninstall` maps that specific condition to exit 1 with interactive or `--force` guidance. ## Solution Assessment
