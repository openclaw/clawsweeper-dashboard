## Summary

Keep this open. Current main still has the direct `readline.question()` await in the shared confirmation helper, and the non-forced `plugins uninstall` path still reaches that helper before config or install-record mutation. The linked closing PR #73566 is still the active implementation candidate in the provided context, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests EOF-aware handling for the non-forced `plugins uninstall` confirmation path so closed stdin exits 1 with actionable guidance while piped answers and `--force` continue to work.

## Best Possible Solution

Land PR #73566, or an equivalent narrow fix, so the shared prompt primitive settles on readline close/EOF and `plugins uninstall` maps that condition to exit 1 with interactive or `--force` guidance. Keep the stale-entry cleanup in #73551 separate unless maintainers intentionally combine the work.

## Reproduction Assessment

Yes. Current source still reaches the direct readline await from the non-forced uninstall path, and a focused Node `readline/promises` check against an already-ended input stream exits 13 with the same unsettled top-level-await warning.

## Solution Assessment

Yes. The maintainable fix is EOF-aware handling in the shared prompt helper plus command-level messaging for uninstall; a blanket non-TTY gate would be too broad because valid piped `y` and `n` answers should keep working.
