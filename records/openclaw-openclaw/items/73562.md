## Summary

Keep open: current main still directly awaits the readline confirmation, and the non-forced plugin uninstall path still calls that helper. PR #73566 is the active open implementation candidate, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks for `openclaw plugins uninstall <id>` to handle closed confirmation input with a clear exit-1 error while preserving piped answers and `--force`.

## Best Possible Solution

Ship the narrow EOF-aware prompt helper and uninstall-specific guidance, with focused coverage for closed input plus the existing answer and `--force` paths.

## Reproduction Assessment

Yes. The issue gives deterministic closed-input steps, current main still has the same direct readline await, and a pure Node closed-readable probe showed the question promise remained unsettled.

## Solution Assessment

Yes. An EOF-aware prompt primitive plus command-specific exit-1 guidance is the narrow maintainable fix; a blanket non-TTY gate would be broader and would break valid piped confirmations.
