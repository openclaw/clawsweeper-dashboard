## Summary

Keep this open. Current main still directly awaits the shared readline confirmation in the non-forced plugin uninstall path, closed stdin reproduces Node's exit-13 unsettled-await behavior, and PR #73566 is still an open implementation candidate rather than merged code.

## What This Changes

Make `openclaw plugins uninstall <id>` fail cleanly on closed confirmation stdin while preserving `--force` and piped `y`/`n` answers.

## Best Possible Solution

Review and land PR #73566, or an equivalent narrow fix, so the shared prompt helper settles on readline close/EOF and uninstall converts that case into an actionable exit-1 error.

## Reproduction Assessment

Yes. The issue gives deterministic closed-stdin steps, current main still reaches the same direct readline await, and the minimal local readline reproduction exits 13 with `/dev/null` stdin.

## Solution Assessment

Yes. The best direction is an EOF-aware prompt primitive plus uninstall-specific error guidance; a blanket non-TTY rejection would regress valid piped confirmations.
