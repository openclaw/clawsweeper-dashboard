## Summary

Keep this open. Current main still has the reported EOF/closed-stdin failure shape in the shared prompt helper and `plugins uninstall` confirmation path, and the open closing PR has not merged, so this issue should remain the tracker until PR #73566 or an equivalent fix lands.

## What This Changes

Make `openclaw plugins uninstall <id>` handle closed confirmation stdin cleanly with an actionable non-zero CLI error while preserving piped `y`/`n` answers and `--force`.

## Best Possible Solution

Review and land PR #73566 or an equivalent narrow fix that makes the shared prompt helper settle on readline close/EOF, maps that specific condition in `plugins uninstall` to exit 1 with interactive or `--force` guidance, and adds focused prompt plus uninstall regression tests. Keep the related stale-entry cleanup in #73551 separate unless maintainers intentionally broaden the scope.

## Reproduction Assessment

Yes. Current main still awaits `readline.question()` directly in the non-forced uninstall path, and a local Node readline smoke reproduces exit 13 for EOF while newline-delimited `n` input resolves normally.

## Solution Assessment

Yes. Handling EOF in the shared prompt primitive and catching that specific condition in `plugins uninstall` is the narrow maintainable direction; a blanket non-TTY gate would break valid piped answers.
