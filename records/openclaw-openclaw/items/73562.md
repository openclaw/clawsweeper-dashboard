## Summary

Keep this open. Current main still awaits the shared readline prompt directly in the non-forced uninstall path, and the linked closing PR is still open rather than merged, so this issue remains the right tracker.

## What This Changes

Make `openclaw plugins uninstall <id>` fail cleanly when confirmation stdin closes, while preserving `--force` and piped `y`/`n` answers.

## Best Possible Solution

Review and land PR #73566, or an equivalent narrow fix, so the shared prompt helper settles on readline close/EOF and `plugins uninstall` reports an actionable exit-1 error without blocking valid piped answers.

## Reproduction Assessment

Yes. The issue provides deterministic closed-stdin steps, current main still reaches the same direct readline await, and a minimal local readline reproduction exited 13 with `/dev/null` stdin.

## Solution Assessment

Yes. The narrow maintainable direction is an EOF-aware prompt primitive plus uninstall-specific error guidance; a blanket non-TTY gate would regress working piped `y` and `n` confirmations.
