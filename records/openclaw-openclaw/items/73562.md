## Summary

Keep open: current main still awaits the shared readline confirmation directly, and open PR #73566 is the active closing implementation candidate rather than a reason to close this issue before merge.

## What This Changes

The issue requests that `openclaw plugins uninstall <id>` handle closed confirmation stdin cleanly instead of crashing with Node exit 13.

## Best Possible Solution

Land PR #73566, or an equivalent narrow fix, so the shared prompt helper settles on EOF/close and plugin uninstall reports actionable interactive-TTY or `--force` guidance.

## Reproduction Assessment

Yes. Current main still reaches the direct readline await, and a focused closed-stdin Node readline probe reproduces exit 13 with the unsettled top-level-await warning.

## Solution Assessment

Yes. An EOF-aware prompt primitive plus uninstall-specific handling is the narrow maintainable fix; a blanket non-TTY gate would be broader and would break valid piped confirmations.
