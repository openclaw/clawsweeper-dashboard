## Summary

Keep this open. Current main still has the direct `readline.question()` await in the shared yes/no prompt and `plugins uninstall` still awaits that prompt without a closed-input guard; the linked closing PR is open and unmerged, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks for `openclaw plugins uninstall <id>` to handle closed confirmation stdin by exiting 1 with actionable guidance while preserving piped `y`/`n` answers and `--force`.

## Best Possible Solution

Review and land PR #73566 or an equivalent narrow fix that makes the shared prompt helper settle on readline close/EOF, converts that condition in `plugins uninstall` into exit 1 with interactive or `--force` guidance, and adds focused prompt plus uninstall regression tests. Keep the stale-entry cleanup in #73551 separate unless a maintainer intentionally broadens the same PR.

## Reproduction Assessment

Yes. Current main still routes the non-forced uninstall confirmation through `promptYesNo()`, and a no-write Node readline smoke reproduces exit 13 with closed stdin while preserving successful piped `n` input.

## Solution Assessment

Yes at the design level. EOF-aware handling in the shared prompt primitive plus a command-level user message is the narrow maintainable fix; a blanket non-TTY gate would regress valid piped answers.
