## What This Changes

`plugins uninstall` should handle closed confirmation stdin cleanly, exit 1 with actionable guidance, and preserve `--force` plus piped `y`/`n` answers.

## Best Possible Solution

Review and land PR #73566, or an equivalent narrow fix, so the shared prompt settles on EOF and uninstall reports clear `--force` or interactive-input guidance before any mutation.

## Reproduction Assessment

Yes. Source inspection shows current main still reaches the direct readline await on the non-`--force` uninstall path, and a focused Node probe reproduces the exit-13 unsettled-await behavior with closed stdin.

## Solution Assessment

Yes. An EOF-aware prompt primitive plus uninstall-specific error handling is the narrow maintainable fix; a blanket non-TTY ban would be broader and would break valid piped confirmations.
