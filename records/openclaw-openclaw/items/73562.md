## Summary

Keep open. Current main still has the closed-stdin prompt bug, and open PR #73566 is the active closing implementation candidate rather than a reason to close this issue before it merges.

## What This Changes

The issue requests that `plugins uninstall` handle closed confirmation stdin cleanly instead of letting Node exit with code 13.

## Best Possible Solution

Review and land PR #73566, or an equivalent narrow fix, so the shared prompt helper settles on readline close/EOF and plugin uninstall reports an actionable exit-1 error.

## Reproduction Assessment

Yes. The issue gives deterministic closed-stdin steps, current main still reaches the same direct readline await, and a minimal local readline reproduction exited 13 with Node's unsettled top-level-await warning.

## Solution Assessment

Yes. An EOF-aware prompt primitive plus uninstall-specific guidance is the narrow maintainable fix; a blanket non-TTY gate would incorrectly reject valid piped `y`/`n` answers.
