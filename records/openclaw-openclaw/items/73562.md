## Summary

Keep this open. Current main still awaits the shared readline prompt directly in the non-forced plugin uninstall path, a local closed-stdin readline reproduction exits 13, and open PR #73566 remains the active implementation candidate rather than a reason to close before merge.

## What This Changes

Make `openclaw plugins uninstall <id>` handle closed confirmation stdin with a clean exit-1 error while preserving piped answers and `--force`.

## Best Possible Solution

Review and land PR #73566, or an equivalent narrow fix, so the shared prompt helper settles on readline close/EOF and plugin uninstall reports an actionable exit-1 error without blocking valid piped confirmations.

## Reproduction Assessment

Yes. The issue gives deterministic closed-stdin steps, current main still reaches the same direct readline await, and a local minimal readline harness exited 13 with `/dev/null` stdin.

## Solution Assessment

Yes. The maintainable direction is an EOF-aware prompt primitive plus uninstall-specific error guidance; a blanket non-TTY gate would regress working piped `y` and `n` confirmations.
