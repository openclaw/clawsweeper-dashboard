## Summary

Keep this open. Current main still directly awaits the shared readline prompt in the non-forced plugin uninstall path, the focused Node readline reproduction still exits 13 on closed stdin, and the linked closing PR is open rather than merged.

## What This Changes

Handle closed stdin during `openclaw plugins uninstall <id>` by failing with an actionable exit-1 error while preserving piped confirmations and `--force`.

## Best Possible Solution

Review and land PR #73566, or an equivalent narrow fix, so the shared prompt helper settles on readline close/EOF and plugin uninstall converts that case into clear `--force` guidance without banning valid piped answers.

## Reproduction Assessment

Yes. The issue gives deterministic closed-stdin steps, current main still uses the same direct readline await, and a minimal local readline reproduction exits 13 with `/dev/null` stdin.

## Solution Assessment

Yes. An EOF-aware prompt primitive plus uninstall-specific error handling is the narrow maintainable path; a blanket non-TTY gate would regress working piped `y` and `n` confirmations.
