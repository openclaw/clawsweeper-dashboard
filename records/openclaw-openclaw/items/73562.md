## Summary

Keep open: current main still directly awaits the shared readline confirmation, and open PR #73566 is the active implementation candidate rather than a reason to close this issue before merge.

## What This Changes

Request that `openclaw plugins uninstall <id>` handle closed stdin during confirmation with a clean exit-1 error while preserving piped answers and `--force`.

## Best Possible Solution

Land PR #73566, or an equivalent narrow fix, so the shared prompt helper settles on EOF/close and plugin uninstall turns that case into actionable TTY or `--force` guidance.

## Reproduction Assessment

Yes. Current main still reaches the direct readline await, and a focused Node 22 probe using an already-ended input stream reproduced exit 13 with an unsettled top-level await.

## Solution Assessment

Yes. An EOF-aware prompt primitive plus uninstall-specific handling is the narrow maintainable fix; a blanket non-TTY gate would be broader and would break valid piped confirmations.
