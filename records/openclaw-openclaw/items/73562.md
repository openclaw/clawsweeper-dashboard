## Summary

Keep this open. Current `main` still has the reported failure shape in the shared prompt helper and the non-forced plugin uninstall path, and the linked implementation PR is still open rather than merged, so the issue should remain the tracking item until that PR or an equivalent fix lands.

## What This Changes

The issue requests that `plugins uninstall` fail cleanly when confirmation stdin closes before an answer, while preserving piped `y`/`n` answers and `--force` automation.

## Best Possible Solution

Review and land PR #73566 or an equivalent narrow fix that makes the shared prompt primitive settle on readline close/EOF, maps that specific condition in `plugins uninstall` to exit 1 with interactive or `--force` guidance, and adds focused prompt plus uninstall regression tests. Keep the stale-entry cleanup in #73551 separate unless maintainers intentionally broaden the work.

## Reproduction Assessment

Yes. Current main still routes non-forced uninstall through `promptYesNo()`, and that helper awaits `readline.question()` without close/EOF handling; the issue body also provides a minimal Node 22 readline reproduction for the dependency behavior.

## Solution Assessment

Yes. Handling EOF in the shared prompt helper and catching that specific condition in `plugins uninstall` is the narrow maintainable solution; a blanket non-TTY gate would break supported piped `y`/`n` answers.
