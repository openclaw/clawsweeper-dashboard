## Summary

Keep open. Current `main` still awaits the shared readline prompt directly in the non-forced plugin uninstall path, and open PR #73566 is the active closing implementation candidate rather than a reason to close this issue before merge.

## What This Changes

Make `openclaw plugins uninstall <id>` fail cleanly when confirmation input closes before an answer, while preserving piped confirmations and `--force`.

## Best Possible Solution

Review and land PR #73566, or an equivalent narrow fix, so the shared prompt helper settles on readline close/EOF and plugin uninstall converts that case into an actionable exit-1 error.

## Reproduction Assessment

Yes. The issue provides deterministic closed-input steps, current `main` still reaches the same direct readline await, and a minimal local readline reproduction exited 13 when the question never settled.

## Solution Assessment

Yes. The maintainable fix is an EOF-aware prompt primitive plus uninstall-specific guidance; a blanket non-TTY gate would be broader and would break valid piped `y`/`n` confirmations.
