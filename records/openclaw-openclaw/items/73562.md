## Summary

Keep this open. Current main still has the closed-stdin failure shape in the shared confirmation helper and the `plugins uninstall` non-forced path, and the contributor PR that closes this issue remains open and unmerged.

## What This Changes

The issue requests that `openclaw plugins uninstall <id>` handle closed confirmation stdin with a clear exit-1 CLI error while preserving piped `y`/`n` answers and `--force`.

## Best Possible Solution

Review and land PR #73566 or an equivalent narrow fix that makes the shared prompt helper settle on readline close/EOF, maps that specific condition in `plugins uninstall` to exit 1 with interactive or `--force` guidance, and adds focused prompt plus uninstall regression tests.

## Reproduction Assessment

Yes. Current main still has the direct `readline.question()` await in `promptYesNo()`, and a read-only Node readline smoke reproduces exit 13 for EOF while newline-delimited `n` input resolves normally.

## Solution Assessment

Yes. Handling EOF in the shared prompt primitive and translating that typed condition in `plugins uninstall` is the narrow maintainable fix; a blanket non-TTY gate would break valid piped answers.
