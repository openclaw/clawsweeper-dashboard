## Summary

Keep open: current main still has the direct readline await in the shared prompt helper, and open PR #73566 is the active closing implementation candidate rather than a reason to close this issue before merge.

## What This Changes

`openclaw plugins uninstall <id>` should handle closed confirmation input cleanly instead of letting Node exit with an unsettled top-level-await warning.

## Best Possible Solution

Land PR #73566, or an equivalent narrow fix, so the shared prompt helper settles on readline close/EOF and plugin uninstall converts that case into an actionable exit-1 error.

## Reproduction Assessment

Yes. Current main still reaches the direct `rl.question(...)` await on non-forced uninstall, and a closed-stdin Node probe in this checkout reproduced exit 13 with the unsettled top-level-await warning.

## Solution Assessment

Yes. An EOF-aware prompt primitive plus uninstall-specific guidance is the narrow maintainable fix; a blanket non-TTY gate would be broader and would break valid piped confirmations.
