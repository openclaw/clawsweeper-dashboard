## Summary

Keep this open. Current main still has the reported closed-stdin failure shape in the shared prompt helper and plugin uninstall command path, and the open closing PR has not landed, so GitHub should keep tracking the issue until PR #73566 or an equivalent fix merges.

## What This Changes

The issue asks for `openclaw plugins uninstall <id>` to fail cleanly with exit 1 when confirmation input is closed, while preserving piped `y`/`n` answers and `--force`.

## Best Possible Solution

Review and land PR #73566 or an equivalent narrow fix that makes the shared prompt helper settle on readline close/EOF, maps that specific condition in `plugins uninstall` to exit 1 with interactive or `--force` guidance, and adds focused prompt plus uninstall regression tests. Keep the stale-entry cleanup in #73551 separate unless maintainers intentionally broaden the fix.

## Reproduction Assessment

Yes. Current main still awaits `readline.question()` in the non-forced uninstall path, and an in-memory Node readline smoke reproduces exit 13 for EOF while newline-delimited `n` input resolves normally.

## Solution Assessment

Yes. Handling EOF in the shared prompt primitive and catching that specific condition in `plugins uninstall` is the narrow maintainable direction; a blanket non-TTY gate would break valid piped answers.
