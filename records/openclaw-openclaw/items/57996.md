## Summary

Keep open. Current main still uses per-agent QMD homes and lacks an OpenClaw-owned shared-cache, compaction, WAL checkpoint, read-only, or cache-size policy; the merged related PR only addressed embed stampeding, not the reported cache-growth/write-amplification problem.

## What This Changes

The issue requests reducing extreme QMD per-agent SQLite disk I/O in multi-agent deployments through shared caching, maintenance/tuning, reduced-write modes, or cache limits.

## Best Possible Solution

## Solution Assessment

Unclear. The proposed shared cache, WAL tuning, reduced-write mode, and eviction controls are plausible, but an owner-reviewed measured maintenance/limits design is safer than a drive-by SQLite pragma or cache-layout change.
