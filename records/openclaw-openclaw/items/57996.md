## Summary

Keep open. Current main and v2026.6.6 still isolate QMD under per-agent `qmd/xdg-cache/qmd/index.sqlite` paths and do not expose or run a QMD shared-cache, compaction, eviction, or reduced-write policy; the merged per-agent SQLite cache PR is scaffold/API work with no production caller switched.

## What This Changes

The issue asks OpenClaw to reduce multi-agent QMD SQLite cache write amplification, compaction gaps, and unbounded per-agent cache growth.

## Best Possible Solution

Define an owner-reviewed QMD cache storage and maintenance policy, then implement the smallest measured change with schema/docs alignment and before/after multi-agent disk-I/O proof.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection confirms the per-agent QMD SQLite topology and missing OpenClaw QMD shared-cache, compaction, eviction, or reduced-write controls on current main and v2026.6.6.

## Solution Assessment

Unclear. Shared cache, scheduled cleanup, WAL/checkpoint tuning, reduced-write mode, and eviction caps are plausible, but the best path depends on QMD ownership boundaries, compatibility, and before/after I/O measurements.
