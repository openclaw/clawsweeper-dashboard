## Summary

Keep open. Current main and latest release still use per-agent QMD homes with per-agent `index.sqlite` files, and OpenClaw does not expose or run QMD shared-cache, WAL checkpoint, vacuum, reduced-write, or cache-size maintenance for this multi-agent disk-I/O problem.

## What This Changes

Reduce QMD multi-agent disk I/O by changing or maintaining per-agent SQLite caches through shared storage, checkpoint/vacuum policy, reduced-write mode, or cache limits.

## Best Possible Solution

Define an owner-reviewed QMD cache storage and maintenance policy, then implement the smallest measured change with docs/config alignment and before/after multi-agent disk-I/O proof.

## Reproduction Assessment

## Solution Assessment

Unclear. Shared cache, scheduled QMD cleanup, WAL/checkpoint tuning, reduced-write mode, and eviction caps are plausible, but the best path depends on QMD ownership boundaries, compatibility, and before/after I/O measurements.
