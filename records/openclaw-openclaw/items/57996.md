## Summary

Keep open. Current main and the latest release still use per-agent QMD homes and `index.sqlite` files, and the QMD config/schema/docs do not expose shared-cache, WAL checkpoint, vacuum, reduced-write, or cache-size controls; related shared-memory and SQLite-lock reports are adjacent but do not resolve this QMD disk-I/O report.

## What This Changes

Reduce QMD per-agent SQLite cache write amplification in multi-agent deployments by changing cache layout, maintenance policy, or cache bounds.

## Best Possible Solution

Define an owner-reviewed QMD cache maintenance and storage policy, then implement the smallest measured change with docs, schema/config alignment if needed, and multi-agent disk-I/O proof.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection confirms the per-agent QMD SQLite topology and missing cache-maintenance controls, but the reported macOS multi-agent write volume still needs measured reproduction.

## Solution Assessment

Unclear. Shared cache, scheduled cleanup, WAL/checkpoint tuning, reduced-write mode, and eviction caps are plausible, but the best path depends on QMD ownership boundaries, compatibility, and before/after I/O measurements.
