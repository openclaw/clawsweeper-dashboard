## Label Justifications

## Summary

Keep open: the Feishu p2p delivery bug is real, but this patch changes the dispatcher target shape for group paths that still assume raw Feishu IDs, and the supplied proof is unit/source-only rather than real Feishu WebSocket delivery.

## What This Changes

This PR changes `extensions/feishu/src/bot.ts` so both Feishu reply-dispatcher call sites pass `feishuTo` instead of raw `ctx.chatId`.

## Best Possible Solution

Split or normalize Feishu delivery targets so non-thread p2p visible sends use the sender open_id while group, thread, streaming-card, typing, and silent-reply paths preserve correct raw recipient semantics.

## Reproduction Assessment

Yes, source-reproducible: current main computes a direct `user:<open_id>` target but constructs the reply dispatcher with the raw p2p `ctx.chatId`. Live Feishu WebSocket reproduction was not run in this read-only review.

## Solution Assessment

No, not as written. The p2p target direction is right, but changing the dispatcher `chatId` itself also changes group and streaming recipient semantics; a delivery-target split or dispatcher-wide normalization is safer.
