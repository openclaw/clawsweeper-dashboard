## Summary

Keep open: current main still has a source-root package-state probe path that can resolve extensionless `auth-presence` specifiers to source TypeScript before built JavaScript, and the only linked fix PR was closed unmerged after review found a loader-boundary blocker.

## What This Changes

The issue asks for bundled channel package-state probes in dev checkouts to avoid loading source `auth-presence` modules when built artifacts are available.

## Best Possible Solution

Teach package-state probes to use the matching built bundled artifact with the correct loader boundary/root when available, while preserving a source-loader fallback for fresh or partial checkouts.

## Reproduction Assessment

Yes, source-backed: current main still resolves extensionless package-state probes from the catalog root before JavaScript, and the issue includes Matrix and WhatsApp startup logs from that path. I did not run a live gateway startup repro in this read-only checkout.

## Solution Assessment

No, current main is not the best end state because the source-root probe path remains. The prior PR's path-rewrite approach was close but needs to carry the matching dist root/boundary so the real loader accepts the built artifact.
