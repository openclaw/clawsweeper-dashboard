## Summary

Keep open: current main still has the source-before-built package-state probe path, and the only related fix PR was closed unmerged after review found it would still fail the loader boundary. No merged replacement or canonical duplicate appears to own the remaining work.

## What This Changes

The issue asks OpenClaw to stop bundled Matrix and WhatsApp `persistedAuthState` probes from loading source `auth-presence.ts` files ahead of built artifacts in source checkouts.

## Best Possible Solution

Teach package-state probes to use the matching built bundled artifact with the correct loader boundary when it exists, while preserving a source-loader fallback for stale or missing dist output.

## Reproduction Assessment

Yes, source-backed: current main still resolves extensionless package-state probes from the bundled source root before `.js`, and the issue includes Matrix and WhatsApp startup logs from that path. I did not run a live gateway repro in this read-only review.

## Solution Assessment

No, current main is not the best end state because it still enters the source-first path for bundled auth probes. The prior PR's direction was close, but the fix needs to carry the matching dist root or boundary so `loadChannelPluginModule` can accept the built artifact.
