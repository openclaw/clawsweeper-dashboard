## Summary

Keep open: current main still resolves bundled package-state checkers from the source plugin root with `.ts` before `.js`, the Matrix and WhatsApp source checkers still import SDK subpaths through package self-resolution, and direct current-checkout imports fail with `ERR_MODULE_NOT_FOUND`. The linked attempted fix was closed unmerged after review, and I found no merged or open canonical replacement.

## What This Changes

The issue asks OpenClaw to stop bundled channel package-state probes from loading source `auth-presence.ts` files ahead of built dist artifacts in dev checkouts.

## Best Possible Solution

Add a narrow package-state probe/module-loader fix that uses the matching built bundled artifact and matching loader boundary/root when it exists, keeps a safe source fallback, and covers the real boundary behavior in tests plus a built dev-checkout startup proof.

## Reproduction Assessment

Yes. Current-main source inspection plus read-only Node probes show the resolver selects `extensions/<channel>/auth-presence.ts` for Matrix and WhatsApp, and importing those source checkers fails with `ERR_MODULE_NOT_FOUND`.

## Solution Assessment

Yes, a narrow loader/probe fix is the maintainable path. The closed PR's path-only rewrite is not sufficient; the fix should carry the corresponding dist root/boundary or provide equivalent source self-resolution without broad runtime loading.
