## Summary

Close as superseded by the canonical #46748/#47365 logs-timezone path rather than merge: the branch is dirty, lacks a working UTC opt-out, and includes an unrelated security-observability downgrade.

## What This Changes

The PR changes `openclaw logs` to default `--local-time` to true and also downgrades escaped skill-path logging from warn to debug.

## Best Possible Solution

Keep #46748 as the canonical bug and land a narrow logs-only fix that defaults display output to local time, preserves `--local-time`, adds a clear UTC opt-out, updates docs/tests/changelog, and leaves skill-path warnings unchanged.

## Reproduction Assessment

Yes. Current main provides a source-level reproduction: `formatLogTimestamp` returns UTC ISO output when `localTime` is false, and `registerLogsCli` only enables local display from `--local-time` or a valid `TZ`; the PR head also demonstrates the unsupported `--no-local-time` promise.

## Solution Assessment

No. Defaulting log display to local time is the right direction, but this branch is not the best implementation because it promises an unregistered negated flag, leaves the `TZ` override behavior unsafe for an explicit false value, and mixes in an unrelated warning downgrade.
