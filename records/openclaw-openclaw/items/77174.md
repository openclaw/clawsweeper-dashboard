## Summary

Keep this PR open: it targets a real BlueBubbles reaction cleanup gap, but the added awaited stop-typing call can make a successful reaction fail if the existing unreliable DELETE `/typing` cleanup rejects; it is also draft with a protected maintainer label.

## What This Changes

The branch exports BlueBubbles typing through the action runtime and updates the reaction action/test to clear typing for the resolved chat after sending a reaction.

## Best Possible Solution

Keep the reaction typing cleanup, but make the stop-typing call best-effort and logged, matching the existing reply finalizer pattern.

## Reproduction Assessment

Yes, at source level: current main's react handler sends the reaction and returns without a stop-typing call, and the PR's focused unit-test target can lock in that missing cleanup. I did not live-test against a BlueBubbles server.

## Solution Assessment

No. Clearing typing after reaction is the right boundary, but awaiting the unreliable DELETE cleanup is not the safest implementation; catch/log that cleanup failure so reaction success is preserved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

- **[P2] Keep typing cleanup from failing successful reactions:** `extensions/bluebubbles/src/actions.ts:253`

Catch/log or otherwise swallow this cleanup failure so a successful tapback is not reported as a failed action.
