## Summary

Keep open. The patch is a focused security hardening for the linked Doctor state-dir issue and I did not find a blocking code defect, but the external real-behavior proof is still helper/mock-level and the Doctor docs still describe the old cross-home warning behavior.

## What This Changes

The PR narrows Doctor's extra state-dir scan from `/home` or `/Users` sibling enumeration to the current account's default `.openclaw` path and adds focused helper coverage.

## Best Possible Solution

Land a privacy-preserving Doctor state-dir check with matching Doctor docs and real after-fix `openclaw doctor` output proof, then close the linked issue after merge.

## Reproduction Assessment

Yes. Source inspection of current main shows a high-confidence reproduction path: on Linux or macOS, readable sibling homes with `.openclaw` are collected and printed by the multiple-state-dir warning; I did not run a live multi-user host in this read-only review.

## Solution Assessment

Mostly yes. Scoping the scan to the current account's default state dir is a small maintainable fix for the disclosure and backup-dir false positives, but it needs docs alignment and maintainer acceptance of the changed diagnostic policy.
