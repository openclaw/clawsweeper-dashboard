## Summary

Keep open. Current main still scopes the message action enum and plugin-owned schema fragments, but the shared core schema groups are still built unconditionally, so the focused token-reduction cleanup remains valid and is not superseded by broader #14785.

## What This Changes

Request action-aware trimming of the shared message tool schema so fields are included only when resolved channel actions can use them.

## Best Possible Solution

Implement action-aware shared schema trimming in the core message tool while preserving unscoped all-action behavior, cross-channel action support, and plugin-owned schema contribution visibility.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: create a scoped message tool for a channel whose resolved actions omit thread/event/moderation/presence/sticker management, then inspect that buildMessageToolSchemaProps() still adds those shared groups because it has no action context.

## Solution Assessment

Yes. Passing the resolved action set into the shared property builder and gating shared field groups by action family is the narrow maintainable fix, with regression coverage for scoped and unscoped schemas.
