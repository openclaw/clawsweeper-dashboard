## Summary

Keep open. Current main still filters the `message` action enum and send-only schemas, but any scoped schema with non-send-only actions still receives the broad shared poll/thread/event/moderation/gateway/channel-management/presence groups, so the focused token-reduction request remains valid and is not superseded by the broader tracker at https://github.com/openclaw/openclaw/issues/14785.

## What This Changes

Request to trim the shared `message` tool schema so only fields needed by enabled/resolved channel actions are sent to the model.

## Best Possible Solution

Implement action-aware shared schema trimming in `src/agents/tools/message-tool.ts`, reusing the resolved action set while preserving cross-channel action support and plugin-owned schema contribution behavior.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction path: create a scoped `message` tool for a channel with `send` plus any other action, then inspect its schema and observe the unrelated shared groups still present.

## Solution Assessment

Yes. Passing the resolved action set into the shared property builder, or otherwise gating shared property groups by action family at the same boundary, is the narrow maintainable fix as long as cross-channel and gateway behavior remain covered.
