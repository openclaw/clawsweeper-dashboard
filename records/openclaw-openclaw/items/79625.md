## Summary

Keep open: current main partially improves the channel path by starting channels before plugin services, but it still loads startup plugins before sidecars and keeps heartbeat and cron behind sidecars-ready; the linked open implementation PR is the active fix path.

## What This Changes

The issue asks for ACPX-independent gateway sidecars, especially Discord, heartbeat, and cron, to start in parallel instead of behind ACPX/plugin runtime readiness.

## Best Possible Solution

Review and correct the linked PR, or replace it with a dependency-aware startup change that decouples channel, heartbeat, and cron activation from ACPX while preserving plugin registry readiness.

## Reproduction Assessment

Yes for the remaining current-main lifecycle coupling: source and tests show startup plugins run before sidecars, heartbeat waits for sidecars-ready, and cron starts in post-ready maintenance. I did not live-reproduce the original Windows timestamp sequence.

## Solution Assessment

Yes as an end state, but the implementation should be dependency-aware rather than starting channels before their runtime registry exists. The active fix path is https://github.com/openclaw/openclaw/pull/79756, after its ordering issue is resolved.
