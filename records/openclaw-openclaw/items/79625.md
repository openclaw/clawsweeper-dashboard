## Summary

Keep open: current main still serializes startup plugins before the channel/sidecar phase, heartbeat and cron still wait for sidecars-ready, and the linked closing PR remains open with an acknowledged registry-order blocker.

## What This Changes

The issue requests dependency-driven startup so Discord, heartbeat, and cron can start independently of slow ACPX readiness.

## Best Possible Solution

Land a dependency-aware startup path that makes configured channel availability, heartbeat, and cron independent from slow ACPX readiness while preserving the ACPX readiness contract for ACPX consumers.

## Reproduction Assessment

Yes for source-level reproduction: current main still awaits `plugins.runtime-post-bind` before entering the sidecar phase, and scheduled services still require `startupSidecarsReady`. I did not run the reported Windows cold-boot trace.

## Solution Assessment

No: closing now is not the best path because https://github.com/openclaw/openclaw/pull/79756 is still open and its current approach has an acknowledged channel-registry ordering blocker. The maintainable fix should split or declare startup dependencies instead of starting channels before their runtime registry exists.
