## Best Possible Solution

Land the narrow effectivePatch fix with the regression once required maintainer and CI validation pass, keeping stale lifecycle status suppression centralized in WorkboardStore.

## Reproduction Assessment

## Solution Assessment

Yes. Updating updateCard to avoid mutating caller patches is better than cloning only in bulkUpdate because it preserves the store API contract for both single-card and bulk callers while keeping stale lifecycle suppression in one place.
