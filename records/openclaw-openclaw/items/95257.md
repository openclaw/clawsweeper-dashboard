  - confidence: 0.9

  - body: The linked bug also reports `registerService` callbacks never starting, but this line only makes extra HTTP routes visible. Without a deferred `startPluginServices`/stop lifecycle for the loaded registry, merging this can close the issue while leaving the service half broken.
  - confidence: 0.88

  - body: On normal startup the active registry can be the same object as `pluginRegistry`; concatenating both arrays duplicates every route. Fallthrough or prefix handlers can then be considered twice, and matching no longer reflects the registry's conflict checks.
  - confidence: 0.82
