## Summary

Keep this PR open: the cache-prefix fix is valid and has good real-environment proof, but the current branch should not merge as-is because it is behind current main and drops the newer Responses `tool_choice` forwarding path.

## What This Changes

The branch hoists OpenAI Responses tool conversion before `input`, adds key-order regression coverage, and updates the changelog.

## Best Possible Solution

Land a rebased version of the key-order fix that keeps current-main Responses `tool_choice` forwarding and the direct key-order regression test.

## Reproduction Assessment

Yes. Source inspection shows current main inserts `input` before a later `tools` assignment, and the OpenAI SDK serializes the body with `JSON.stringify`; the PR body adds real wire evidence showing the prefix improves from 15% to 99% after reorder.

## Solution Assessment

Mostly yes. Hoisting the existing converted tools value before `input` is the narrow fix, but this branch also needs to carry forward current-main `tool_choice` forwarding before it is mergeable.
