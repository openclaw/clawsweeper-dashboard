## Summary

Keep open: current main still serializes `input` before a later `tools` assignment, and this PR is a focused fix with a direct regression test and sufficient real Codex Responses proof. I found no blocking functional or security findings in the patch.

## What This Changes

The PR reorders OpenAI Responses request params so converted `tools` are inserted before `input`, adds a key-order regression test, and records the change in the changelog.

## Best Possible Solution

Land the shared Responses builder key-order fix with its regression test after normal CI and maintainer review; keep broader prompt-cache optimization work separate.

## Reproduction Assessment

Yes. Current main places `input` in the params literal before a later `params.tools` assignment, and the OpenAI SDK serializes the request body with `JSON.stringify`, matching the live cache-prefix evidence in the PR discussion.

## Solution Assessment

Yes. Moving the existing converted `tools` value into the params literal ahead of `input` is the narrowest maintainable fix because it preserves field values and adds a direct key-order regression test.
