## What This Changes

The PR hoists OpenAI Responses tool conversion so `tools` serializes before per-turn `input`, adds a key-order regression test, and records the cache-prefix performance result in the changelog.

## Best Possible Solution

Land the shared Responses builder key-order fix with the regression test after maintainer review; keep broader prompt-cache optimization work separate.

## Reproduction Assessment

Yes from source inspection: current main places `input` before a later `params.tools` assignment, and the OpenAI SDK serializes object request bodies with `JSON.stringify`. I did not run a local test in this read-only review, but the PR body and comment thread add live traffic proof.

## Solution Assessment

Yes. Moving the existing converted tools value ahead of `input` preserves field values, targets the cache-critical byte order directly, and adds a focused regression test rather than introducing a broader transport abstraction.
