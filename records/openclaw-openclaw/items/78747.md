## Summary

Keep this PR open: the cache-order fix is well supported and worth landing, but the current branch would regress forced Responses tool selection by dropping `params.tool_choice` when tools are present.

## What This Changes

The PR hoists OpenAI Responses tool conversion so `tools` is serialized before `input`, adds a key-order regression test, and updates the changelog.

## Best Possible Solution

Land a rebased version of the key-order fix that keeps `tools` before `input`, preserves current-main `tool_choice` forwarding, and retains direct regression coverage for both behaviors.

## Reproduction Assessment

Yes. Source inspection shows current main enumerates `input` before a later `tools` assignment, and the OpenAI SDK serializes object bodies with `JSON.stringify`; the PR body adds real wire evidence showing the prefix improves from 15% to 99% after reorder.

## Solution Assessment

Mostly yes. Hoisting the existing converted tools value before `input` is the narrow maintainable fix, but the branch must carry forward current-main Responses `tool_choice` forwarding before it is mergeable.
