Full review comments:

- **[P1] Seed denylist defaults before stamping them managed:** `src/infra/exec-approvals.ts:762-764`
  - body: `normalizeExecApprovals` now writes `managedDefaults.denylistVersion` even when an existing approvals file has no default denylist entries. Because `exec.approvals.get` calls `ensureExecApprovals()` and the UI save path normalizes and persists that file, simply loading/saving approvals can stamp the file as managed; later `exec-policy set --security denylist` will treat the marker as operator intent and skip seeding the documented `default-shell-network-fetch` rule. Existing users can then enable denylist and silently miss the expected curl/wget block, so the marker should only be written after the default rule is present or the raw file already had the marker.
  - confidence: 0.86
