- **[P1] Preserve approval floors for elevated full exec:** `src/agents/bash-tools.exec.ts:1451-1458`
  - body: This branch makes any `elevated` full request set `security = "full"` and skip `processGatewayAllowlist`.   - confidence: 0.9
- **[P1] Keep gateway auto-review in auto mode:** `src/agents/bash-tools.exec.ts:1667-1679`
  - body: `tools.exec.mode=auto` is documented to route approval misses through the native auto reviewer before asking a human, and current main passes `autoReview` into the gateway path. The PR still computes `autoReview`, but the gateway call no longer passes it and `processGatewayAllowlist` goes straight from `requiresAsk` to registering a human approval, so gateway users of auto mode lose the native auto-review step.
  - confidence: 0.86
- **[P1] Restore approval delivery fields in the protocol schema:** `packages/gateway-protocol/src/schema/exec-approvals.ts:182-184`
  - body: The PR removes `requireDeliveryRoute` and `suppressDelivery` from the exec approval protocol schema and generated Swift model, but node approval code still sends both fields for inline auto-review approvals. With `additionalProperties: false`, this can break approval request validation or drop the delivery/suppression contract that keeps inline approvals from being misrouted or visibly posted.
  - confidence: 0.9
