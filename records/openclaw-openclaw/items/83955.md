## Summary

Keep this PR open: the source refactor moves toward the right security boundary, but the served diffs browser runtime is still the checked-in `assets/viewer-runtime.js`, and this PR does not update it. The patch also changes the wrap toolbar SVG while claiming to only move icon generation behind typed names.

## What This Changes

The PR changes the diffs viewer toolbar button API from raw SVG markup strings to typed icon names backed by a fixed SVG map.

## Best Possible Solution

Keep the typed icon-name API, preserve the existing SVG shapes exactly, regenerate and commit the served viewer runtime asset, and prove the gateway-served viewer uses the updated runtime.

## Reproduction Assessment

Yes, by source inspection: current main serves `extensions/diffs/assets/viewer-runtime.js`, and that asset still contains `innerHTML=e.iconMarkup`. I did not run a browser/gateway repro because this is a read-only review.

## Solution Assessment

No, not yet: typed icon names are a maintainable fix shape, but the implementation must update the actual served runtime asset and preserve the existing icon SVGs.
