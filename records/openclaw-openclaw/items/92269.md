## Label Justifications

- `P1`: The reported bug silently loses user replies, while the proposed fix can introduce stale out-of-order delivery in common conversation paths.
- `merge-risk: 🚨 message-delivery`: The fence-key change directly determines which concurrent replies are delivered or suppressed.

## Summary

The reported silent reply loss is credible, but this patch replaces the existing same-target freshness contract with per-message independence across direct chats and group chats. That can allow an older slow answer to arrive after a newer visible answer, and the PR provides no real WhatsApp proof to resolve the delivery risk.

## What This Changes

The PR adds the inbound message SID to foreground reply-fence keys and updates freshness tests so distinct inbound messages on the same target can deliver independently.

## Best Possible Solution

Partition group-chat freshness by a stable sender or explicit supersession lineage while retaining conversation-level freshness for direct chats and same-sender prompts, with regression coverage and real WhatsApp delivery proof for both behaviors.

## Reproduction Assessment

Yes at source level: target-scoped visible-delivery generations can cancel an earlier completed reply after a later visible send, matching the linked production WhatsApp report. A live current-main reproduction was not independently run.

## Solution Assessment

No. Per-message scoping removes cross-message freshness globally instead of narrowly separating independent group participants or explicit reply lineages.
