## Summary

Keep this PR open for revision: it addresses a credible silent message-loss bug, but the message-ID key effectively disables conversation-level stale-reply fencing for ordinary consecutive messages, and the supplied proof is test-only.

## What This Changes

The PR appends the inbound message SID to foreground reply-fence keys and updates freshness tests so separate inbound messages on one target can both deliver.

## Best Possible Solution

Use a supersession identity that separates unrelated group participants while preserving stale-final cancellation for newer prompts that genuinely supersede older work, then prove both invariants with focused tests and a redacted live channel run.

## Reproduction Assessment

Yes at source level: distinct inbound messages currently share a target-scoped generation, and a later visible delivery can cancel an earlier final; the linked issue supplies a matching production WhatsApp observation. A live current-main reproduction was not independently run during this read-only review.

## Solution Assessment

No. Per-message keys address cross-user cancellation by making nearly every inbound message independent, which also removes the original stale-reply protection for same-sender and direct-chat rapid-fire prompts; a sender-aware or otherwise explicit supersession identity is safer.
