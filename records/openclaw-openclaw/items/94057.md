  - confidence: 0.94
- **[P1] Validate rewritten bind sources before Docker sees them:** `src/agents/sandbox/docker.ts:564`
  - body: `buildSandboxCreateArgs` validates the original bind sources before this new rewrite runs, so a prefix map can translate a source that passed allowed-root and blocked-path checks into a different host path before Docker receives `-v`. The mapped source needs to be normalized and validated or explicitly gated before it can cross the sandbox boundary.
  - confidence: 0.88
- **[P2] Thread the prefix map through browser sandbox mounts:** `src/agents/sandbox/workspace-mounts.ts:172`
  - body: The new `hostMountPrefixMap` parameter is only supplied by the normal sandbox container path. `src/agents/sandbox/browser.ts` still calls these mount helpers without the map and appends browser binds raw, so DooD deployments that create the browser sandbox container can still hand container-internal mount sources to the host Docker daemon.
  - confidence: 0.8
