## Summary

Keep this draft maintainer PR open, but it is not merge-ready: the branch adds a useful canonical-provider ClawRouter design while currently allowing an explicitly selected ClawRouter route to disappear and fall back toward normal provider auth.

## What This Changes

The PR adds generic provider transport-routing hooks, a bundled ClawRouter provider plugin, dispatch metadata on models, and runtime/auth changes so canonical provider models can be sent through ClawRouter.

## Best Possible Solution

Land the canonical-provider ClawRouter design only after locked ClawRouter profiles fail closed instead of falling back, and after the new SDK/config/ownership surfaces are documented, labeled, rebased, and revalidated.

## Reproduction Assessment

Yes. Source inspection shows a selected clawrouter:* profile can produce no route when catalog/auth/entitlement fails, after which the embedded runner can treat the model as its canonical provider and continue auth selection outside ClawRouter.

## Solution Assessment

No. The generic hook approach fits the owner boundary better than synthetic model IDs, but the implementation must fail closed for explicit ClawRouter routing and document the new public SDK/config surfaces before it is the best fix.
