## What This Changes

The PR changes the Control UI chat session picker to keep loading through hidden subagent-only pages, suppress raw total counts when hidden rows are filtered, and add unit plus browser E2E coverage.

## Best Possible Solution

Maintain the focused UI fix in this PR, then have a maintainer clear or resolve the human-review gate before automerge or landing.

## Reproduction Assessment

Yes. Current main loads only one raw picker page per Load More click while visible rows are filtered afterward, so a hidden-only page can leave the picker apparently stuck; I did not run tests because this review is read-only.

## Solution Assessment

Yes. Advancing through hidden-only pages and suppressing misleading raw totals is the narrow maintainable UI fix; the alternative server-side include-spawn-children API from the closed earlier PR would add protocol surface that is not needed for this bug.
