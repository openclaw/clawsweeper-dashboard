## What This Changes

The PR updates the Control UI chat session picker to continue loading past hidden subagent-only pages, show visible-row counts when filtering, and cover the behavior with unit plus browser E2E tests.

## Best Possible Solution

Land this narrow UI picker fix through the existing maintainer/automerge gate, while keeping CLI cursor parity and optional subagent-reveal UX as separate work.

## Reproduction Assessment

Yes at source level: current main filters subagent/spawned rows after raw sessions.list pagination and Load more fetches only one raw page per click, so a hidden-only page can leave the visible picker unchanged. I did not run a live browser reproduction in this read-only review.

## Solution Assessment

Yes. Advancing through hidden-only pages and showing the visible navigable count is the narrow UI-layer fix; it avoids changing the Gateway contract, adding config, or changing the default hidden-subagent policy.
