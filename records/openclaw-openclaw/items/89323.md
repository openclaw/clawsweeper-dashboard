## What This Changes

The PR changes the Control UI chat session picker to keep loading through hidden subagent-only pages, hide misleading raw total counts when filtered rows are present, and adds regression coverage for that pagination path.

## Best Possible Solution

Land the focused picker fix after maintainer review and real Control UI behavior proof, leaving any server-side navigable-only filter as a separate product/API decision.

## Reproduction Assessment

Yes, this is source-reproducible: current main filters inactive subagent rows after loading raw paginated results, so a hidden-only page can leave the visible picker unchanged while the footer uses the raw total. I did not run a live Control UI reproduction in this read-only review.

## Solution Assessment

Yes, this is an acceptable narrow UI-side fix for the PR scope: it preserves existing backend/session contracts, follows the picker's current rendered-row projection, bounds hidden-page skipping, and updates count display. A server-side navigable-only filter could be cleaner long term but would require API/product surface beyond this small bug fix.
