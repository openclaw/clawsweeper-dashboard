## What This Changes

The PR changes the message CLI text formatter to branch on `MessageActionRunResult.dryRun` instead of `handledBy === "dry-run"` when rendering dry-run message output.

## Best Possible Solution

Land one narrow formatter fix that checks `dryRun`, preserves real CLI proof, includes or follows with focused send/poll formatter tests, then close the remaining duplicate fix PR and the canonical issue.

## Reproduction Assessment

Yes. I did not execute the CLI in this read-only review, but current-main source plus the linked issue's captured CLI output show send and poll dry-runs carry `dryRun: true` while the formatter checks `handledBy === "dry-run"`.

## Solution Assessment

Yes. Checking `result.dryRun` in the formatter is the narrow maintainable fix; changing outbound `handledBy` semantics would broaden the runner contract unnecessarily.
