## Best Possible Solution

Land one narrow formatter fix that checks `dryRun`, preferably with focused send/poll formatter tests, then close https://github.com/openclaw/openclaw/issues/80507 and the duplicate fix PR.

## Reproduction Assessment

Yes. Current-main source plus the linked issue's captured CLI output show send and poll dry-runs carry `dryRun: true` while the text formatter checks only `handledBy === "dry-run"`.

## Solution Assessment

Yes. Checking `result.dryRun` in the formatter is the narrow maintainable fix; changing outbound `handledBy` semantics would broaden the runner contract unnecessarily.
