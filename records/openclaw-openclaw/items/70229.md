## What This Changes

The PR adds a secure-default `sandbox.docker.noNewPrivileges` Docker sandbox setting, related config/schema/docs/tests, and dangerous-config reporting for `false`.

## Best Possible Solution

Land a refreshed secure-default compatibility knob, or equivalent, that omits `no-new-privileges` only when explicitly false, documents it as break-glass, and reports the downgrade through current dangerous-config diagnostics.

## Reproduction Assessment

Yes for review purposes: the linked issue gives a focused Docker command that fails only with `--security-opt no-new-privileges`, and current main unconditionally applies that flag before `sleep infinity`. I did not rerun the host-specific Docker/runc failure locally.

## Solution Assessment
