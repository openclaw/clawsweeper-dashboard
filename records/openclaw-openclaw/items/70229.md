## Summary

Keep open. The linked bug remains valid on current main, and the PR is useful in direction, but the branch is not merge-ready because it wires the new dangerous-config warning into the older wrapper path while current main collects sandbox Docker dangerous flags in the core contract collector.

## What This Changes

The PR adds a default-on `sandbox.docker.noNewPrivileges` Docker sandbox config option, schema/docs labels, create-args behavior, and tests so operators can explicitly omit Docker `--security-opt no-new-privileges` on incompatible hosts.

## Best Possible Solution

Land a refreshed default-on Docker sandbox compatibility knob that omits `--security-opt no-new-privileges` only when explicitly configured false, documents it as a break-glass host-compatibility setting, and reports the false value through the current core dangerous-config diagnostics.

## Reproduction Assessment

Yes. The linked bug provides a focused Docker command that fails only with `--security-opt no-new-privileges`, and current main unconditionally supplies that flag when creating sandbox containers.

## Solution Assessment

Yes in direction, but not as the current branch. A default-on config knob with dangerous-config reporting is the narrowest maintainable fix, but the PR must be refreshed onto current main's core collector and receive sandbox security review.
