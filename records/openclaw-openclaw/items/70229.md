## Summary

Keep open. Current main still hard-codes Docker `--security-opt no-new-privileges` and has no accepted `sandbox.docker.noNewPrivileges` config key, so the PR is addressing a real remaining bug. The branch still needs maintainer/security review and a refresh against current main because dangerous-config diagnostics now route through the core collector, while the PR diff adds the new warning on the older wrapper path.

## What This Changes

The PR adds a default-true `sandbox.docker.noNewPrivileges` Docker sandbox config field, conditionally omits Docker `--security-opt no-new-privileges` when false, updates schema/docs, and adds resolver, create-args, and dangerous-config tests.

## Best Possible Solution

Land a refreshed, default-on Docker sandbox compatibility knob that only suppresses `--security-opt no-new-privileges` when explicitly configured false, documents it as a break-glass host-compatibility setting, and reports the false value through the current core dangerous-config diagnostics.

## Reproduction Assessment

Yes. The linked bug gives a minimal Docker command that fails only with `--security-opt no-new-privileges`, and current-main source verification shows OpenClaw always includes that flag for sandbox containers.

## Solution Assessment

Yes in direction, but not as the current stale branch. A default-on opt-out is a narrow maintainable fix for incompatible hosts, but the warning must be moved to the current core collector path and reviewed as a sandbox hardening downgrade.
