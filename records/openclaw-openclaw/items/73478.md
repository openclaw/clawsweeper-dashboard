## Summary

Keep open. Current main still projects chat-linked Gateway assistant events into text-only live/final `chat` payloads, while existing generated-image persistence is limited to the non-agent WebChat reply path.

## What This Changes

The issue asks Gateway WebSocket `chat` output/history to include assistant-generated image/media information instead of returning only text after a MiniMax image-generation reply.

## Best Possible Solution

Carry assistant-event media through a provider-agnostic Gateway chat media projection path and reuse the existing managed outgoing image machinery for persisted history and final/live payloads.
