## Summary

Keep open. Current main still projects chat-linked Gateway assistant events into text-only live/final `chat` payloads even though assistant stream data can carry media and the separate WebChat reply path already converts reply media into managed image blocks.

## What This Changes

Report that Gateway WebSocket chat/history output omits generated-image media from assistant messages after MiniMax image generation.

## Best Possible Solution

Carry assistant-event media through the provider-agnostic Gateway chat projection path and reuse managed outgoing image blocks for live/final payloads and persisted history.

## Reproduction Assessment

Yes, at source level. Register a chat-linked run, emit an assistant event with `data.text` plus `data.mediaUrls`, then end the lifecycle; current main only broadcasts text/delta and finalizes from text buffers.

## Solution Assessment

Yes. The narrow maintainable fix belongs in Gateway's generic agent-event projection and managed image attachment handling, not in MiniMax-specific logic or a docs-only workaround.
