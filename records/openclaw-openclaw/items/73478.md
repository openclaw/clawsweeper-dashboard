## Summary

Keep open. Current main still has a Gateway agent-event chat bridge that projects linked assistant runs to WebSocket `chat` payloads from buffered text only, while the assistant event model and the non-agent `chat.send` path already support media-bearing replies.

## What This Changes

The issue reports that Gateway WebSocket `chat.send`/`chat.history` output for a MiniMax image-generation conversation includes assistant text but no generated image information.

## Best Possible Solution

Add a generic Gateway agent-event media projection path that carries assistant event media into WebSocket final chat output and `chat.history` using the existing managed outgoing image attachment machinery, with focused coverage for mixed text+media and media-only assistant events.

## Reproduction Assessment

Yes. A source-level reproduction can register a chat-linked run, emit an assistant event with `data.text` plus `data.mediaUrls`, then emit lifecycle end; current main builds both delta and final chat messages from text buffers only.

## Solution Assessment

Yes. Reusing or factoring the existing managed image projection from the non-agent `chat.send` path is the narrowest maintainable fix; provider-specific MiniMax handling would leave the Gateway contract gap in place.
