## Summary

Keep open. Current main still drops assistant-event media on the Gateway chat projection: agent events can carry `mediaUrls`, but live/final `chat` payloads and run state are text-only, while managed generated-image persistence is only wired through the non-agent WebChat reply path.

## What This Changes

The issue asks Gateway WebSocket `chat` events and `chat.history` to include generated image information for MiniMax-triggered image replies instead of returning only assistant text.

## Best Possible Solution

Add a provider-agnostic Gateway assistant-event media projection that carries `mediaUrl`/`mediaUrls` through live/final chat events and persisted history using the existing managed outgoing image machinery.

## Reproduction Assessment

Yes, by source inspection. Register a chat-linked run, emit an assistant event with `data.text` and `data.mediaUrls`, then end the lifecycle; current main only reads `data.text` and emits text blocks.

## Solution Assessment

Yes. The narrow maintainable fix is in the generic Gateway agent-event projection and shared managed image helpers, not MiniMax-specific handling or a docs-only change.
