## Summary

Keep open. Current main still projects chat-linked Gateway agent assistant events into text-only live/final `chat` payloads, while the managed image attachment path exists for the non-agent WebChat reply path.

## What This Changes

The issue requests Gateway WebSocket `chat` output and `chat.history` to include generated image/media information when `chat.send` triggers image generation through MiniMax.

## Best Possible Solution

Add a generic Gateway agent-event media projection path that carries assistant `mediaUrls` into live/final chat output and persisted WebChat history through the existing managed outgoing image attachment machinery.

## Reproduction Assessment

Yes. A focused source-level reproduction can register a chat-linked run, emit an assistant event with `data.text` plus `data.mediaUrls`, then end the lifecycle; current main only buffers and emits text.

## Solution Assessment

Yes. Reusing or factoring the existing managed-image projection is the narrowest maintainable fix; MiniMax-specific handling or a docs-only change would leave the Gateway contract gap in place.
