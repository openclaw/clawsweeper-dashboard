## Summary

Keep open. Current main still projects chat-linked agent assistant events into live/final Gateway `chat` output as text-only, even though assistant stream data can carry `mediaUrls` and managed image attachment machinery exists elsewhere.

## What This Changes

The issue asks Gateway WebSocket `chat` and `chat.history` output from `chat.send` to include generated image media information instead of only assistant text.

## Best Possible Solution

Add a generic Gateway agent-event media projection path that carries assistant `mediaUrls` into live/final chat output and persisted WebChat history through the existing managed outgoing image attachment machinery.

## Reproduction Assessment

Yes. A focused source-level reproduction can register a chat-linked run, emit an assistant event with `data.text` plus `data.mediaUrls`, then end the lifecycle; current main buffers and emits only text on that path.

## Solution Assessment

Yes. Reusing or factoring the existing managed-image projection is the narrowest maintainable fix; MiniMax-specific handling or docs-only guidance would leave the Gateway contract gap in place.
