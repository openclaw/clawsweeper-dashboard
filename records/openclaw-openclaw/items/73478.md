## Summary

Keep open: current main and the latest release still project Gateway assistant chat events as text-only, while https://github.com/openclaw/openclaw/pull/81136 is an open closing PR that owns the proposed media-url fix.

## What This Changes

Gateway WebSocket chat events for assistant image-generation turns should include generated image/media content instead of only the assistant text.

## Best Possible Solution

Review and land or replace the linked Gateway PR so assistant `mediaUrls` are carried through delta, flush, and final chat payloads without changing the protocol schema.

## Reproduction Assessment

Yes, at source level: register a chat-linked run, emit an assistant event with `data.text` plus `data.mediaUrls`, and current main will build the `chat` payload from text only. I did not run a live WebSocket/model reproduction in this read-only review.

## Solution Assessment

Yes, the best fix boundary is the existing Gateway chat projection: carry assistant media URLs through delta, buffered flush, and final payloads. The issue should remain open while the linked PR is reviewed, rebased, proved, and either landed or replaced.
