## Summary

Keep open. Current main still projects linked agent-run assistant Gateway events onto WebSocket chat as text-only payloads, even though assistant event data can carry media URLs and the non-agent chat.send path already has managed image projection for history/display.

## What This Changes

The issue asks for Gateway WebSocket chat output and chat.history to include generated image/media information instead of returning only assistant text for a MiniMax image-generation reply.

## Best Possible Solution

Add a generic Gateway agent-event media projection path that carries assistant event media into WebSocket final chat output and chat.history through the existing managed outgoing image attachment machinery, with focused coverage for mixed text+media and media-only assistant events.

## Reproduction Assessment

Yes. A focused source-level reproduction can register a chat-linked run, emit an assistant event with data.text plus data.mediaUrls, then emit lifecycle end; current main's chat delta/final payloads are built only from text buffers.

## Solution Assessment

Yes. Reusing or factoring the existing managed image projection is the narrowest maintainable fix; provider-specific MiniMax handling or a docs-only change would leave the Gateway contract gap in place.
