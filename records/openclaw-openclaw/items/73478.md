## Summary

Keep open. Current main still has a Gateway behavior gap: assistant agent events can carry media URLs, but the chat-linked WebSocket projection builds live delta/final `chat` payloads from text buffers only, so generated image attachments can be absent from the WebChat stream/history path.

## What This Changes

The issue reports that Gateway WebSocket `chat.send`/`chat.history` output omits generated image information for a MiniMax image-generation conversation.

## Best Possible Solution

Add a generic Gateway agent-event media projection path that carries assistant event `mediaUrls` into live/final chat output and persisted WebChat history through the existing managed outgoing image attachment machinery, without MiniMax-specific handling.

## Reproduction Assessment

Yes. A focused source-level reproduction can register a chat-linked run, emit an assistant event with `data.text` plus `data.mediaUrls`, then end the lifecycle; current main reads only text/delta for the live and final chat projection.

## Solution Assessment

Yes. Reusing or factoring the existing managed image projection is the narrowest maintainable fix; provider-specific handling or a docs-only change would leave the Gateway contract gap in place.
