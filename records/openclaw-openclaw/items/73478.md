## Summary

Keep open. Current main still drops image media from chat-linked Gateway agent events: assistant stream data can carry `mediaUrls`, but the live/final `chat` projection is text-only while managed image handling is only wired into the non-agent WebChat reply path.

## What This Changes

The issue asks Gateway WebSocket `chat` and `chat.history` output to include image information for generated-image replies instead of returning text-only assistant messages.

## Best Possible Solution

Add a generic Gateway agent-event media projection that carries assistant-event `mediaUrls` into live/final `chat` output and persisted WebChat history through managed outgoing image attachments.

## Reproduction Assessment

Yes. A focused source-level reproduction can register a chat-linked run in the `server-chat.agent-events` harness, emit an assistant event with `data.text` plus `data.mediaUrls`, then end the lifecycle; current code consumes only text/delta for the chat projection.

## Solution Assessment

Yes. Reusing or factoring the existing managed outgoing image projection is the narrowest maintainable fix; MiniMax-specific handling or a docs-only change would leave the Gateway contract gap in place.
