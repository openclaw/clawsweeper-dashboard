## Summary

Keep open. Current `main` still has the Gateway behavior gap: chat-linked assistant events may carry `mediaUrls`, but the live/final WebSocket `chat` projection is still built from text buffers only, while existing managed-image handling is only wired into the non-agent reply path.

## What This Changes

The issue reports that Gateway WebSocket `chat.send`/`chat.history` output omits generated image information for MiniMax image replies.

## Best Possible Solution

Add a generic Gateway agent-event media projection path that carries assistant event `mediaUrls` into live/final chat output and persisted WebChat history through the existing managed outgoing image attachment machinery.

## Reproduction Assessment

Yes. A focused source-level reproduction can register a chat-linked run, emit an assistant event with `data.text` plus `data.mediaUrls`, then end the lifecycle; current `main` only reads text/delta for the live and final chat projection.

## Solution Assessment

Yes. Reusing or factoring the existing managed-image projection is the narrowest maintainable fix; MiniMax-specific handling or a docs-only change would leave the Gateway contract gap in place.
