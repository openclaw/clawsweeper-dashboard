## Summary

Keep open. Current main still has the same Gateway agent-event gap identified in the existing ClawSweeper comment: assistant events can carry media URLs, but the live WebSocket `chat` bridge buffers and emits only text for delta/final chat payloads.

## What This Changes

The issue reports that Gateway WebSocket chat output and `chat.history` omit generated image information for a MiniMax image-generation reply.

## Best Possible Solution

Add a generic Gateway agent-event media projection path that carries assistant event media into final WebSocket chat output and `chat.history`, reusing the existing managed outgoing image attachment machinery and adding focused mixed text+media and media-only tests.

## Reproduction Assessment

Yes. A source-level reproduction can register a chat-linked run, emit an assistant event with `data.text` plus `data.mediaUrls`, then emit lifecycle end; current main builds the live chat delta/final payload from text buffers only.

## Solution Assessment

Yes. The narrow maintainable fix is to factor or reuse the existing managed-image projection for agent-event media; MiniMax-specific handling or a docs-only change would leave the Gateway contract gap in place.
