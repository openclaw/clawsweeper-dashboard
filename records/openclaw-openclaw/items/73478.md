Keep open. Current main still projects chat-linked Gateway assistant events into text-only live/final `chat` payloads, even though assistant stream data can carry media and the separate WebChat reply path already has managed image handling.

## What This Changes

## Best Possible Solution

Carry assistant-event media through the generic Gateway chat projection path and reuse managed outgoing image machinery for live/final payloads and persisted history.
