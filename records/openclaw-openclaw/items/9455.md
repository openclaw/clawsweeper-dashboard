## Summary

Keep open. Current main has the Cron Jobs filter bar, but the two previously preserved acceptance gaps are still present: search cannot match job IDs, and Jobs last-run status cannot filter Unknown or missing statuses.

## What This Changes

The issue requests a compact Cron Jobs list filter bar with search by job name/jobId and filters for enabled state, schedule kind, and last-run status.

## Best Possible Solution

Finish the existing filter bar by adding job.id search, an Unknown last-status option, lastRunStatus/lastStatus normalization with missing status mapped to Unknown, and focused regression tests.

## Reproduction Assessment

Yes. Source inspection on current main shows cron.list search uses name/description/agentId but not job.id, and the UI last-status filter only supports ok/error/skipped without an Unknown/missing bucket or lastRunStatus fallback.

## Solution Assessment

Yes. The narrow maintainable path is to complete the already-merged service/UI filtering behavior rather than redesigning the page or adding a new RPC.
