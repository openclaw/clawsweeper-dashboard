## Summary

Keep open. Current main has the Cron Jobs filter bar from merged #9510, but source inspection shows two explicit acceptance paths are still missing: search by job ID and filtering jobs with missing or Unknown last-run status.

## What This Changes

Request a compact Cron Jobs UI filter bar with search, enabled, schedule-kind, last-run-status, and reset controls.

## Best Possible Solution

Complete the existing filter bar by adding job.id to cron.list search and an Unknown last-status bucket for missing job state, with focused service/controller/view tests.

Yes. Source inspection is enough: the UI passes cronJobsQuery to cron.list, whose haystack omits job.id, and the Jobs last-status type/view/controller only support all/ok/error/skipped, so no Unknown selection can match missing status.

## Solution Assessment

Yes. The narrow maintainable path is to extend #9510’s existing filter plumbing instead of redesigning the Cron Jobs page or changing persisted cron status data.
