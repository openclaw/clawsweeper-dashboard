## Summary

Keep open. Current main includes the Cron Jobs filter bar from merged #9510, but it still misses two explicit acceptance paths: search by job id and filtering jobs whose last-run status is absent or unknown.

## What This Changes

Add a compact Cron Jobs list filter bar with search, enabled, schedule-kind, last-run-status, and reset controls.

## Best Possible Solution

Finish the residual filter behavior on top of #9510: include job ids in search, add an Unknown status option, normalize lastStatus/lastRunStatus with missing values, and cover those paths with focused tests.

## Reproduction Assessment

Yes. Source inspection shows `cron.list` receives the search query but its haystack omits `job.id`, and the Jobs status filter cannot select missing statuses because only all/ok/error/skipped are valid.

## Solution Assessment

No. The merged filter bar is the right base, but current main is not a complete solution until the job-id search and Unknown/missing-status paths are implemented.
