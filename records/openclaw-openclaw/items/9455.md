Keep open. Current main includes the Cron Jobs filter bar from merged #9510, but it still misses explicit acceptance paths for jobId search and an Unknown/missing last-run-status bucket.

## What This Changes

The issue requests a compact Cron Jobs UI filter bar with text search by job name/jobId plus enabled, schedule kind, and last-run status filters.

## Best Possible Solution

Complete the existing filter bar by adding job.id to search and an Unknown last-status bucket for missing status, with focused service/controller/view coverage.

## Solution Assessment

Yes. The narrow maintainable path is to extend #9510’s existing filter plumbing rather than redesign the Cron Jobs page or change persisted cron status data.
