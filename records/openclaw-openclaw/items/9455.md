## Summary

Keep open: the merged filter-bar work covers the main UI shape, but current main still misses explicit acceptance paths for job-ID search and missing/unknown last-run status filtering.

## What This Changes

Add a compact Cron Jobs filter bar with search plus enabled, schedule-kind, last-run-status, and reset controls.

## Best Possible Solution

Finish the existing filter bar by adding `job.id` to `cron.list` search and adding a normalized Unknown last-status bucket based on `lastRunStatus ?? lastStatus`.

## Reproduction Assessment

Yes, at source level. A query that appears only in `job.id` cannot match because `cron.list` excludes `job.id`, and jobs with no last-run status cannot be selected because there is no Unknown filter value or missing-status mapping.

## Solution Assessment

Yes. Extending the existing server-side search haystack and existing Jobs last-status filter is the narrow maintainable path; adding another UI flow or extra RPC would be broader than needed.
