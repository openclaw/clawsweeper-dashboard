## Summary

Keep open. Current main has the Cron Jobs filter bar from merged #9510, but it still misses explicit acceptance paths for job-id search and filtering jobs with absent or unknown last-run status.

## What This Changes

Add a compact Cron Jobs filter bar with text search, enabled/status/kind filters, and a reset path for large job lists.

## Best Possible Solution

Finish the existing filter bar by adding job IDs to cron.list search and adding an Unknown UI filter bucket for missing last-run status without changing the persisted cron run-status protocol.

## Reproduction Assessment

Yes for the remaining acceptance gaps. Source inspection shows Jobs search reaches cron.list but job.id is absent from its haystack, and the last-status filter has no selectable unknown/missing-status path.

## Solution Assessment

Yes. The narrow maintainable path is to build on the existing #9510 filter bar rather than redesigning the Cron Jobs page.
