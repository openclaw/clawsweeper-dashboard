## What This Changes

Add a compact Cron Jobs filter bar above the Jobs list with search by job name/jobId and filters for enabled state, schedule kind, and last-run status.

## Best Possible Solution

Complete the existing filter bar by adding job.id to cron.list search and mapping lastRunStatus ?? lastStatus ?? "unknown" into a Jobs Unknown status option with focused coverage.

## Solution Assessment

Yes. Extending the existing service search haystack and Jobs last-status filter is the narrow maintainable path; a second UI flow or extra RPC would be broader than needed.
