## Summary

Keep open. The merged filter-bar work covers most of the requested UI, but current main still misses the explicit jobId search path and the Unknown/missing last-run-status filter path, and no canonical open PR now owns those leftovers.

## What This Changes

The issue requests a compact Cron Jobs UI filter bar with search by job name/jobId, enabled, schedule kind, last-run status, and reset controls.

## Best Possible Solution

Finish the existing filter bar by adding job.id to cron.list search, adding an Unknown last-status option/bucket, and normalizing lastRunStatus ?? lastStatus with missing status mapped to unknown.

## Reproduction Assessment

Yes. The remaining gaps are source-reproducible: the UI sends search text to cron.list, cron.list omits job.id from its haystack, and the status filter cannot represent or normalize Unknown/missing status.

## Solution Assessment

Yes. The best path is a narrow completion on top of https://github.com/openclaw/openclaw/pull/9510, not a Cron page redesign or cron state schema change.
