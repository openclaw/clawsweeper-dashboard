## Summary

Keep open. Current main includes the Cron Jobs filter bar from merged PR #9510, but it still misses job-id search and an Unknown/missing last-run-status filter, which are explicit parts of the requested behavior.

## What This Changes

The issue requests a Cron Jobs dashboard filter bar with search by job name/jobId, enabled, schedule kind, last-run status including unknown, and reset/clear controls.

## Best Possible Solution

Finish the remaining Cron Jobs filter behavior on top of #9510: search loaded/listed jobs by id as well as name, expose an Unknown last-run-status option, normalize canonical and legacy status fields, and cover those paths with focused service/controller/view tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: search for a known job id substring through the Jobs filter and `cron.list` will not match it because `job.id` is absent from the haystack; a job with no last-run status also cannot be selected because Unknown is not a valid UI/controller filter value.

## Solution Assessment

No. The current partial implementation is the right base, but closing now would leave two requested acceptance paths unimplemented; a narrow follow-up in the cron service search and Control UI filter code is the maintainable fix.
