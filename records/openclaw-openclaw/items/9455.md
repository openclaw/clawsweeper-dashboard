## Summary

Keep open: the merged Cron Jobs filter bar covers most of the requested UI, but current main and the latest release still miss job-ID search and an Unknown/missing last-run-status filter path.

## What This Changes

The issue requests a compact Cron Jobs UI filter bar with text search, enabled, schedule-kind, last-run-status, and reset controls for large job lists.

## Best Possible Solution

Complete the existing filter bar in place by adding `job.id` to the `cron.list` search haystack and adding a normalized Unknown last-status bucket in the controller, view, and tests.

## Reproduction Assessment

Yes, at source level. A query that appears only in `job.id` cannot match because `cron.list` excludes `job.id`, and jobs with absent last-run status cannot be selected because the Jobs status filter has no Unknown value or missing-status mapping.

## Solution Assessment

Yes. Extending the existing `cron.list` haystack and current Jobs last-status filter is the narrow maintainable path; adding a second UI flow or extra RPC would be broader than needed.
