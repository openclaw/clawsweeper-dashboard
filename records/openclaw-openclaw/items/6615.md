## Summary

Keep open. Current main and the latest shipped release still lack a native exec-approvals denylist, while the SafeClaw and CRE links are useful prior art rather than a merged OpenClaw contract; https://github.com/openclaw/openclaw/pull/82596 is the active implementation path and still needs security/product review.

## What This Changes

The issue requests native exec-approvals denylist support so operators can allow most commands while prompting or blocking specific command and argument patterns.

## Best Possible Solution

Land one additive native exec approvals contract after security/product review, with deny-rule precedence, command and argument matching, gateway/node/macOS enforcement, UI/CLI/schema/docs coverage, audit behavior, and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction. The missing feature is source-reproducible with high confidence because current main and v2026.5.12 only accept the existing exec security and ask modes, store allowlists, and have no deny-match decision branch.

## Solution Assessment

Unclear. Native deny rules are a plausible direction and https://github.com/openclaw/openclaw/pull/82596 is exploring them, but the config shape, precedence, prompt versus hard-deny behavior, audit semantics, and host parity need maintainer security/product decisions.
