## Summary

Keep open. Current main and latest release v2026.5.19 still lack native exec-approvals denylist, `ask: "on-match"`, or deny-pattern evaluation; the linked external CRE/SafeClaw ideas are useful prior art, and https://github.com/openclaw/openclaw/pull/82596 is an active implementation attempt, but it has not merged.

## What This Changes

The issue requests native exec-approvals denylist support so broad exec access can still prompt or block matching dangerous command patterns.

## Best Possible Solution

Land one additive native exec approvals contract through https://github.com/openclaw/openclaw/pull/82596 or a successor after security/product acceptance of rule shape, precedence, prompt behavior, audit semantics, host parity, UI, docs, and regression coverage.
