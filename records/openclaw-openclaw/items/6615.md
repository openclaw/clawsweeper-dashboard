## Summary

Keep open. Current main still has allowlist/full/deny exec approval policy only; the linked SafeClaw and CRE implementations are useful prior art, but they do not replace a native OpenClaw policy change for host exec authorization.

## What This Changes

Add native exec-approvals denylist support, including deny-pattern matching on command arguments and an `ask: "on-match"` prompt mode.

## Best Possible Solution

Define an additive native exec approvals policy that evaluates deny rules before broad allow decisions, supports consistent command and argument matching across gateway and node `system.run` paths, exposes the policy in CLI and Control UI, and documents the security model.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high-confidence proof that current main lacks the requested denylist mode, denylist collection, and `ask: "on-match"` handling.

## Solution Assessment

Unclear as proposed. Native core support is the right venue for host exec authorization, but maintainers still need to choose policy shape, precedence, and prompt versus hard-deny behavior before a narrow implementation is safe.
