## Summary

Keep open: current main still has no native exec-approvals denylist or `ask: "on-match"` policy, and the remaining work is a security-sensitive design spanning policy, protocol, CLI, UI, docs, and related #41140.

## What This Changes

Add a native exec approvals denylist policy that allows broad execution except for configured command, path, or argument patterns, optionally prompting on matches.

## Best Possible Solution

Define an additive native exec approvals policy that evaluates deny rules before broad allow decisions, supports consistent command and argument matching across gateway and node `system.run` paths, and updates protocol, CLI, Control UI, docs, and tests in sync with #41140.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main confirms the requested mode, fields, and evaluation path are absent from policy types, normalization, protocol schema, CLI/UI, and docs.

## Solution Assessment

Unclear until maintainers settle the security semantics. The maintainable path is an additive, version-safe native policy change coordinated with #41140, not treating external policy plugins as the core fix.
