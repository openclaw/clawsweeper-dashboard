## Summary

Keep open: current main and v2026.6.10 still lack native exec-approval denylist, denylist collection, or prompt-on-match support. The SafeClaw and CRE links are useful prior art, and https://github.com/openclaw/openclaw/pull/92456 is a direct unmerged implementation candidate, but neither replaces a merged native OpenClaw policy change.

## What This Changes

The issue requests native exec-approval denylist support so broad allow policies can still prompt or block selected command patterns.

## Best Possible Solution

Keep this issue as the canonical native exec-approval denylist tracker until maintainers land a reviewed implementation, explicitly narrow the scope to a separate canonical policy, or reject the feature direction.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.6.10 shows only the existing deny/allowlist/full and off/on-miss/always policy values, with no denylist collection or deny-match branch.

## Solution Assessment

Unclear. A native deny-before-allow policy is plausible and aligned with OpenClaw's security goals, but maintainers still need to settle the exact config shape, prompt versus hard-deny semantics, and relationship to the open implementation candidates.
