## Summary

Keep open. Current main and the latest release still lack native exec-approvals denylist mode, `ask: "on-match"`, or a deny-before-allow evaluation path; the issue remains the canonical tracker while https://github.com/openclaw/openclaw/pull/82596 is still open for implementation review.

## What This Changes

Request native exec-approvals denylist support so broad exec can be allowed while specific command or argument patterns are blocked or escalated for approval.

## Best Possible Solution

Land one additive native exec approvals contract through https://github.com/openclaw/openclaw/pull/82596 or a successor after maintainer/security acceptance of rule shape, precedence, prompt behavior, audit semantics, host parity, UI, docs, and regression coverage.

Not applicable as a bug reproduction. The missing feature is source-reproducible with high confidence because current main and v2026.5.20 only accept existing exec security/ask modes, store allowlists, and have no deny-match decision branch.

## Solution Assessment

Unclear. Native deny rules are plausible, but maintainers still need to settle config shape, deny-before-allow precedence, prompt versus hard-deny behavior, default rules, audit behavior, and host parity.
