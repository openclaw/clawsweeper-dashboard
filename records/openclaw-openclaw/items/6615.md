## Summary

Keep open. Current main and the latest release still lack native exec-approvals denylist mode, denylist storage, `ask: "on-match"`, or deny-before-allow evaluation; https://github.com/openclaw/openclaw/pull/82596 is the active implementation path but still needs maintainer/security acceptance before this issue can close.

## What This Changes

Request to add denylist support for exec approvals so broad execution can be allowed while matching dangerous commands or arguments are blocked or require approval.

## Best Possible Solution

Land one additive native exec approvals denylist contract through https://github.com/openclaw/openclaw/pull/82596 or a successor after maintainer/security acceptance of rule shape, precedence, prompt behavior, audit semantics, host parity, UI, docs, and regression coverage.

## Reproduction Assessment

Yes for source-level absence, not as a failing bug reproduction. Current main and `v2026.5.22` only accept existing exec security/ask modes, store allowlists, and have no deny-match decision branch.

## Solution Assessment

Unclear until maintainer/security acceptance. Native deny rules are plausible, but maintainers still need to settle config shape, deny-before-allow precedence, prompt versus hard-deny behavior, default rules, audit behavior, and gateway/node/macOS parity.
