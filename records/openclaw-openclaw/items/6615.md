## Summary

## What This Changes

Add a native exec-approvals denylist mode with command/argument matching so operators can allow most host exec commands while blocking or prompting on configured dangerous patterns.

## Best Possible Solution

Land one additive native exec approvals contract after security/product review, with deny-rule precedence, consistent command and argument matching, gateway/node/macOS enforcement, UI/CLI/schema/docs coverage, audit behavior, and focused regression tests.

Not applicable as a bug reproduction. The missing feature is source-reproducible with high confidence because current main and v2026.5.12 only accept existing exec security/ask modes, store allowlists, and have no deny-match decision branch.

Unclear. Native deny patterns are plausible and an active PR is exploring them, but the config shape, precedence, prompt versus hard-deny behavior, defaults, audit semantics, and macOS parity need maintainer security/product decisions.
