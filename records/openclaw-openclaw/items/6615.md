## What This Changes

Requests native exec-approval denylist support so operators can allow most commands while blocking or prompting on specific dangerous command and argument patterns.

## Best Possible Solution

Land a maintainer-approved native exec-approval policy that evaluates explicit deny rules before broad allow decisions, supports command and argument matching consistently across gateway and node paths, and updates CLI, Control UI, protocol, docs, and focused tests.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.6.6` can be checked directly: policy types, normalizers, protocol schema, UI options, and docs have no native denylist collection or `on-match` prompt mode.
