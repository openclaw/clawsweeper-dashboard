## Summary

Keep this open. Current main still has no native exec-approvals denylist mode, denylist collection, `ask: "on-match"`, or deny-pattern evaluation path; the linked SafeClaw and CRE projects plus the closed PRs are useful prior art, but they do not replace a merged OpenClaw policy change.

## What This Changes

Add native exec-approvals denylist and argument matching support so broad allow policies can prompt or block specific dangerous command patterns.

## Best Possible Solution

Define an additive native exec approvals policy that evaluates deny rules before broad allow decisions, supports consistent command and argument matching across gateway and node `system.run` paths, exposes the policy in CLI and Control UI, documents the security model, and coordinates the overlap with #41140.

## Reproduction Assessment

Yes. The missing feature is reproducible by read-only source inspection on current main: the requested `denylist`, `security: "denylist"`, `ask: "on-match"`, and `argsMatch` concepts are absent from policy types, normalization, schemas, UI, docs, and execution evaluation.

## Solution Assessment

Unclear that the exact proposed API is the best final design. The maintainable path is a maintainer-reviewed additive security policy with explicit deny-before-allow ordering and compatibility across config, protocol, node/gateway execution, UI, and docs.
