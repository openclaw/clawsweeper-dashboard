## Summary

The issue requests native exec-approvals denylist support so operators can allow most commands while prompting or blocking dangerous command, path, or argument patterns.

## Best Possible Solution

## Solution Assessment

Unclear until maintainer/security acceptance. A native denylist is plausible and aligned with the request, but maintainers still need to settle config shape, denial precedence, prompt versus hard-deny behavior, default rules, audit behavior, and gateway/node/macOS parity.
