## Summary

Keep open: current main and the latest release still do not provide native exec-approval denylist, deny-pattern, or prompt-on-match support, and the open path-deny PR only partially overlaps the broader requested policy.

## What This Changes

The issue requests native exec-approval denylist support so broad allow policies can still prompt or block selected dangerous command patterns.

## Best Possible Solution

Define a maintainer-approved additive native exec approvals policy that evaluates deny rules before broad allow decisions, supports command and argument matching consistently across gateway and node paths, exposes the policy in config/CLI/Control UI/docs, and coordinates the overlap with https://github.com/openclaw/openclaw/issues/71097 and https://github.com/openclaw/openclaw/pull/88075.

Yes. ## Solution Assessment

Unclear. The requested direction is plausible and aligned with the security goal, but maintainers still need to choose the final native policy shape; the narrower path-deny PR does not fully solve this issue.
