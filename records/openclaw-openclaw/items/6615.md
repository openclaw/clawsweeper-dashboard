## Summary

Keep this open. Current main and v2026.6.6 still lack the requested native exec-approvals denylist mode, `denylist` collection, `ask: "on-match"`, and deny-before-allow evaluation; the open related PR only covers a narrower path-argument hard-deny gate and is unmerged.

## What This Changes

The issue requests native exec-approvals denylist support with command/argument matching and prompt-on-match behavior for “allow everything except these patterns” policies.

## Best Possible Solution

Define an additive native exec approvals policy that evaluates deny rules before broad allow decisions, supports consistent command and argument matching across gateway, node, CLI, and UI surfaces, documents the security tradeoffs, and coordinates the narrower path-deny work as a subset rather than the whole fix.

## Reproduction Assessment
