## Summary

Keep open. Current main and v2026.5.12 still expose exec approvals as `deny | allowlist | full` plus `off | on-miss | always` with allowlist-only storage and enforcement, while https://github.com/openclaw/openclaw/pull/82596 is an active but unmerged implementation path that still needs security/product and host-parity review.

## What This Changes

The issue requests native exec-approvals denylist support so broad command execution can be allowed while matching dangerous command patterns still prompt or block.

## Best Possible Solution

Land one additive native exec approvals contract after security/product review, with deny-rule precedence, command and argument matching, gateway/node/macOS enforcement, UI/CLI/schema/docs coverage, audit behavior, and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction. The missing feature is source-reproducible with high confidence because current main and v2026.5.12 only accept the existing exec security/ask modes, store allowlists, and have no deny-match decision branch.

## Solution Assessment

Unclear. Native deny patterns are a plausible direction and https://github.com/openclaw/openclaw/pull/82596 is exploring them, but the config shape, precedence, prompt versus hard-deny behavior, defaults, audit semantics, and host parity still need maintainer security/product decisions.
