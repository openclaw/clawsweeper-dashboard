## Summary

Keep open. Current main still has allowlist-oriented exec approvals only, while this request asks for a new native denylist/on-match policy; related external projects and closed PR attempts are useful prior art but do not replace a merged OpenClaw policy change.

## What This Changes

Add native exec-approvals denylist support with argument/content matching and a prompt-on-match behavior.

## Best Possible Solution

Define an additive native exec approvals deny policy that runs before broad allow decisions, supports consistent command and argument matching across gateway and node system.run paths, and updates schema, CLI/UI, docs, tests, and security guidance together.

## Reproduction Assessment

Yes for the missing feature. Current-main source inspection shows no accepted denylist policy value, denylist collection, on-match ask mode, or deny-pattern evaluation branch in the exec approval path.

## Solution Assessment

Unclear. The requested denylist/on-match shape is plausible, but the safest maintainable fix needs a maintainer security/product decision before implementation because it changes core execution policy semantics.
