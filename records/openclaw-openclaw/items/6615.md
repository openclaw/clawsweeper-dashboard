## Summary

Keep open. Current main and the latest release still lack native exec-approval denylist/on-match support; the active implementation candidate at https://github.com/openclaw/openclaw/pull/82596 still needs maintainer security/product review before this can be resolved.

## What This Changes

Add native exec-approvals denylist/on-match support so operators can allow most exec commands while prompting or blocking matched dangerous command and argument patterns.

## Best Possible Solution

Define and land an additive native exec deny policy after maintainer security/product review, with consistent gateway/node/macOS enforcement, protocol/CLI/UI/docs/tests coverage, and coordination with https://github.com/openclaw/openclaw/pull/82596 plus related exec-policy trackers.

## Reproduction Assessment

Not applicable as a bug reproduction, but source-level proof is high-confidence: current main and v2026.5.12 accept only the existing exec security/ask modes, store allowlists only, and have no deny-match decision branch.

## Solution Assessment

Unclear. A native deny policy is plausible and supported by prior art, but the exact config shape, precedence, prompt/block behavior, cross-host coverage, and audit semantics need maintainer security/product decisions before this is a narrow repair.
