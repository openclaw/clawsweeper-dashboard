Add native exec-approval denylist support so broad host exec can be allowed while matching dangerous command patterns can prompt or block.

## Best Possible Solution

Define and land one additive native exec deny policy after maintainer security/product review, with consistent gateway/node/macOS enforcement, protocol/config/CLI/UI/docs/tests coverage, and clear coordination with the active implementation PR and related exec-policy trackers.
