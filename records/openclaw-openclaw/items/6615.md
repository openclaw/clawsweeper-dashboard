## Summary

Keep open. Current main still lacks a native exec-approvals denylist mode, denylist collection, `ask: "on-match"`, and deny-match evaluation; the external policy engines and closed implementation attempts are useful prior art but do not replace a merged core policy change.

## What This Changes

The issue requests native exec-approvals denylist support with argument matching and prompt-on-match behavior for allow-everything-except-specific-command policies.

## Best Possible Solution

Define an additive native exec approvals policy that evaluates deny rules before broad allow decisions, supports consistent command and argument matching across gateway and node `system.run` paths, exposes configuration in CLI and Control UI, and documents the security model.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection on current main gives high-confidence proof that the requested denylist mode, denylist collection, and `ask: "on-match"` behavior are absent.

## Solution Assessment

Unclear as proposed. Native core support is the right venue for host exec authorization, but maintainers should first settle policy shape, precedence, matching semantics, audit behavior, and prompt versus hard-deny fallback rules.
