## Summary

Keep open. Current main still lacks native exec-approval denylist/on-match support, and related discussion shows this is an unresolved security-sensitive product/config design rather than conservative cleanup.

## What This Changes

The issue requests exec-approval denylist support so operators can allow most commands while prompting or blocking specific command and argument patterns.

## Best Possible Solution

Define an additive native exec approvals policy that evaluates deny rules before broad allow decisions, supports command and argument matching consistently across gateway and node system.run paths, and updates schema, CLI/UI, docs, tests, and security guidance together.

## Reproduction Assessment

Yes for the missing feature. Current-main source inspection shows no accepted denylist policy value, denylist collection, on-match ask mode, or deny-pattern evaluation branch in the exec approval path.

## Solution Assessment

Unclear. The requested denylist/on-match shape is plausible, but the safest maintainable fix needs maintainer security/product design because it changes core execution policy semantics.
