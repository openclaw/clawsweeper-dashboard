## Summary

Current main and v2026.6.5 still lack native deny-pattern exec approvals, and the linked implementation PRs closed unmerged. The request remains useful and aligned with OpenClaw’s security direction, but its allow-by-default semantics, rule precedence, matching language, and core ownership require an explicit maintainer product/security decision before implementation.

## What This Changes

Add an allow-by-default exec-approval policy with command- and argument-aware deny rules that can require approval or block matching commands.

## Best Possible Solution

Define a maintainer-approved additive deny-before-allow policy in the shared exec-approval evaluator, reuse canonical command parsing and argument-pattern matching, and expose identical semantics through gateway and node hosts, protocol, CLI, Control UI, focused tests, and public docs.

## Reproduction Assessment

Yes, at the source-contract level. Current main’s shared enums, normalizer, resolver, protocol schema, host callers, UI, and docs provide a high-confidence way to verify that denylist and prompt-on-match behavior are absent; this is a missing feature rather than a broken existing contract.

## Solution Assessment

Not yet. Reusing the existing argument matcher is appropriate, but maintainers still need to choose an overlay versus a separate mode and define precedence, prompt-versus-hard-deny outcomes, and safe upgrade semantics.

## Risks / Open Questions

- Denylist-by-default allows every dangerous command that an operator failed to anticipate, so it should not silently replace safer deny-by-default behavior.
- Rule precedence across deny, require-approval, allowlist, durable approvals, full mode, and compound shell commands is unresolved.
- A new persisted policy collection and mode would require coordinated schema, normalization, protocol, CLI, UI, docs, and upgrade behavior.
- A plugin-only policy engine could become a competing source of truth unless core exposes an authoritative hook used by every gateway-host and node-host execution path.
