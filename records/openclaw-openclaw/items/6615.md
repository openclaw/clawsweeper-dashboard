## Summary

Keep open: current main still has no native exec-approvals denylist, `security: "denylist"`, `ask: "on-match"`, or deny-pattern evaluation path, and the latest maintainer context treats this as security-policy design rather than cleanup.

## What This Changes

The issue requests native exec-approvals denylist support, including command and argument matching plus an `ask: "on-match"` prompt mode.

## Best Possible Solution

Define an additive native exec approvals policy that evaluates deny rules before broad allow decisions, supports consistent command and argument matching across gateway and node `system.run` paths, and updates protocol, CLI, Control UI, docs, and tests together.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main confirms the requested mode, fields, UI surface, protocol payload, docs, and deny evaluation path are absent.

## Solution Assessment

Unclear until maintainers settle the security design. The maintainable path is a version-safe native policy change coordinated across exec approval surfaces, using the linked plugins and closed PRs as prior art rather than treating them as the core fix.
