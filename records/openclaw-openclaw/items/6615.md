## Summary

Keep open: current main and v2026.6.6 still lack native exec-approvals denylist, `argsMatch`, `ask: "on-match"`, and deny-before-allow evaluation. Related plugins and closed PR attempts are useful prior art, and https://github.com/openclaw/openclaw/pull/88075 covers a narrower path-deny gate, but the requested command/argument denylist policy remains a maintainer security/product decision.

## What This Changes

Request adds native exec-approvals denylist support for allow-everything-except command policies, including argument matching and prompt-on-match behavior.

## Best Possible Solution

Define an additive, version-safe native exec approvals policy that supports deny-before-allow command and argument rules, exposes consistent CLI/UI/docs behavior, and coordinates the narrower path-deny work at https://github.com/openclaw/openclaw/issues/74379 plus the overlapping mode discussion at https://github.com/openclaw/openclaw/issues/71097.

## Reproduction Assessment

Yes at the source level. Current main and v2026.6.6 accept only the existing exec approval modes, store only allowlist entries, and have no deny-match branch, so the requested capability is clearly absent without running a live destructive command.

## Solution Assessment

Unclear as implementation direction. The request is valid and aligned with core exec safety, but the exact native policy shape needs maintainer security/product approval before it should be implemented or closed as covered by narrower path-deny work.
