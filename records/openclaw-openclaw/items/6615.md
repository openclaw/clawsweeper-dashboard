## Summary

Keep open. Current main still has no native exec-approvals denylist mode, denylist collection, `ask: "on-match"`, or deny-pattern evaluation path; the SafeClaw and CRE links plus closed PR attempts are useful prior art, but they do not replace a merged OpenClaw policy change.

## What This Changes

Add native exec-approval denylist support so broad host exec can still prompt or block when configured command, path, or argument patterns match.

## Best Possible Solution

Define an additive native exec approvals policy with explicit deny-before-allow ordering, consistent command and argument matching across gateway and node `system.run`, Control UI and CLI support, docs, tests, and coordination with #41140.

## Reproduction Assessment

Yes. The missing capability is reproducible by source inspection on current main: the requested `denylist`, `security: "denylist"`, `ask: "on-match"`, and `argsMatch` concepts are absent from policy types, schemas, UI, docs, and execution evaluation.

## Solution Assessment

Unclear that the exact proposed API is the best final shape. The maintainable path is a maintainer-reviewed additive security policy that defines evaluation order and compatibility across config, protocol, gateway/node execution, UI, and docs.
