## Summary

Keep open. Current main and the latest release still lack native exec-approval denylist/on-match policy support, while the SafeClaw/CRE links and the open implementation PR are useful prior art rather than a merged core policy change.

## What This Changes

Add native exec-approval denylist support so operators can allow most commands while blocking or prompting for specific dangerous command and argument patterns.

## Best Possible Solution

Define and land an additive native exec deny policy after maintainer security/product review, with consistent gateway/node enforcement, CLI/UI/schema/docs coverage, tests for precedence and command/argument parsing, and coordination with https://github.com/openclaw/openclaw/pull/82596 plus the related exec-policy trackers.

## Reproduction Assessment

Not applicable as a bug reproduction, but source-level proof is high-confidence: current main accepts only the existing exec security/ask modes, stores allowlists only, and has no deny-match decision branch.

## Solution Assessment

Unclear. A native deny policy is plausible and supported by prior art, but the exact config shape, precedence, fallback, audit, and UI behavior need maintainer security/product decisions before this is a narrow repair.
