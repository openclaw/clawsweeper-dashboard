## Summary

Keep open. Current main still lacks native exec-approval denylist/on-match support, and the discussion plus related closed implementation attempts show this remains a security-sensitive product/config design rather than conservative cleanup. The CRE and SafeClaw links are useful prior art, but they do not replace a merged OpenClaw policy change; https://github.com/openclaw/openclaw/issues/41140 was also closed back to this issue as the canonical tracker.

## What This Changes

The issue requests native exec-approval denylist support, including command/argument matching and an on-match approval mode for allow-everything-except policies.

## Best Possible Solution

Define an additive native exec-approval policy that evaluates operator deny rules before broad allow decisions, supports consistent command and argument matching across gateway and node system.run paths, and updates schema, CLI/UI, docs, tests, and security guidance together.

## Reproduction Assessment

Yes for the missing capability. Current-main source, protocol schema, Control UI, docs, and focused tests still accept only the existing allowlist-centered policy values and have no denylist/on-match evaluation path.

## Solution Assessment

Unclear. The requested denylist/on-match direction is plausible, but the maintainable fix needs a security/product decision on the exact policy semantics and overlap with related deny-pattern/path-deny trackers.
