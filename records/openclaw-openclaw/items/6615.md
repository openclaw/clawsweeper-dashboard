## Summary

Keep open. Current main and the latest release still lack native exec-approval denylist/on-match support; the SafeClaw/CRE discussion and prior PRs are useful prior art, but the remaining work is an unmerged security/product decision with an active implementation candidate at https://github.com/openclaw/openclaw/pull/82596.

## What This Changes

Request to add a native exec approvals denylist/on-match policy with command and argument matching so users can allow most exec commands while prompting or blocking dangerous patterns.

## Best Possible Solution

Define and land an additive native exec deny policy after maintainer security/product review, with consistent gateway/node/macOS enforcement, protocol/CLI/UI/docs/tests coverage, and coordination with https://github.com/openclaw/openclaw/pull/82596 plus related exec-policy trackers.

## Reproduction Assessment

Not applicable as a bug reproduction, but source-level proof is high-confidence: current main and v2026.5.12 accept only the existing exec security/ask modes, store allowlists only, and have no deny-match decision branch.

## Solution Assessment

Unclear. A native deny policy is plausible and supported by prior art, but the exact config shape, precedence, prompt/block behavior, macOS coverage, and audit semantics need maintainer security/product decisions before this is a narrow repair.
