## Summary

Keep open. Current main still lacks a native exec-approvals denylist mode, denylist collection, `ask: "on-match"`, and deny-pattern decision path; the linked SafeClaw/CRE implementations and closed PRs are useful prior art but do not replace a merged OpenClaw core policy change.

## What This Changes

Add a native exec-approvals denylist mode with argument-aware match rules and prompt-on-match behavior.

## Best Possible Solution

Define an additive native exec approvals policy that evaluates deny rules before broad allow decisions, supports consistent command and argument matching across gateway and node `system.run` paths, exposes the policy in CLI and Control UI, and documents the security model.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection on current main gives high-confidence proof that the requested denylist mode, denylist collection, and `ask: "on-match"` handling are still absent.

## Solution Assessment

Unclear as proposed. Native core support is the right venue for host exec authorization, but maintainers still need to choose policy shape, precedence, fallback, and prompt versus hard-deny behavior before implementation is safe.
