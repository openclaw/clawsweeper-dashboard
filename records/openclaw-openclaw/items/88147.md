## What This Changes

The issue asks OpenClaw to stop retaining oversized parsed session-store objects and snapshots in gateway memory after session-store hydration.

## Best Possible Solution

Review and either land or close https://github.com/openclaw/openclaw/pull/88150; the desired end state is a size-aware session-store object/snapshot cache that preserves disk as source of truth and has focused regression coverage.

## Solution Assessment
