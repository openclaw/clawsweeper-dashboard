## Summary

Keep this PR open: current main still has the elevated-default follow-up gap, and the PR is a narrow, well-scoped fix with no blocking review findings from this read-only pass.

## What This Changes

The PR preserves trusted elevated exec defaults across approval follow-up agent runs using an internal session-scoped token and forwards those defaults into embedded execution.

## Best Possible Solution

Land the internal token-based propagation after normal maintainer review and CI, keeping elevated-default transfer session-scoped and out of the public gateway schema.

## Reproduction Assessment

Yes. Source inspection on current main shows approval follow-up dispatch carries session/routing context but drops `defaults.elevated`, and the exec tool fails elevated requests when that default object is absent.

## Solution Assessment

Yes. The PR is the narrow maintainable fix because it transfers only trusted internal elevated defaults through a random, one-use, session-bound token without adding a public schema field or changing approval policy.
