## Summary

Keep open for maintainer review: current main still drops trusted elevated defaults across approved exec follow-up runs, and this PR appears to implement the narrow internal propagation fix without blocking review findings.

## What This Changes

The PR carries trusted elevated exec defaults through approval follow-up runs via a short-lived session-bound token, forwards them into embedded agent execution, and adds regression coverage plus a changelog entry.

## Best Possible Solution

Land the internal token-based propagation after normal maintainer review, keeping elevated-default transfer session-scoped and out of the public gateway schema.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes, source-reproducible: current main sends approval follow-up runs without `bashElevated`, while the exec tool requires `defaults.elevated` for elevated `on`/`ask` to remain available.

## Solution Assessment

Is this the best way to solve the issue? Yes, based on the diff: the PR transfers only trusted internal elevated defaults through a random session-bound token and avoids changing approval policy or the public agent schema.
