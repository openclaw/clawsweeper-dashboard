## Summary

Keep open: current main still drops elevated defaults across exec approval follow-up agent runs, and this PR is a narrow source-level fix with no blocking review findings found.

## What This Changes

The PR carries trusted elevated exec defaults through approval follow-up continuations using an internal session-bound token, forwards them into embedded runs, and adds regression tests plus a changelog entry.

## Best Possible Solution

Land the internal token-based propagation after normal maintainer review and CI, keeping elevated-default transfer session-scoped and out of the public gateway schema.

## Reproduction Assessment

Yes, source-reproducible. Current main has a clear path where the approved exec follow-up carries routing context but no `bashElevated`, while the exec tool requires `defaults.elevated` to keep elevated `on`/`ask` available.

## Solution Assessment

Yes. The PR is the narrow maintainable fix because it transfers only trusted internal elevated defaults through a random session-bound token and avoids adding a public `agent` schema field or changing approval policy.
