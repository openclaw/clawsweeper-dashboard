## Summary

Keep this PR open for normal maintainer review: current main still has the elevated-follow-up state drop described by the linked issue, and the PR appears to address that narrow bug without adding a public gateway field or dependency.

## What This Changes

The PR preserves trusted elevated exec defaults across approval follow-up runs with an internal session-bound token, forwards them into embedded agent execution, and adds regression tests plus a changelog entry.

## Best Possible Solution

Land the internal token-based propagation after maintainer review, CI, and mergeability cleanup, keeping elevated-default transfer session-scoped and out of the public gateway schema.

## Reproduction Assessment

Yes, source-reproducible. Current main forwards approval follow-up routing but not `bashElevated`, while the exec tool requires `defaults.elevated` to keep elevated `on`/`ask` available for subsequent commands.

## Solution Assessment

Yes. The PR is the narrow maintainable fix because it transfers only trusted internal elevated defaults through a random session-bound token and avoids changing approval policy or exposing a public `agent` schema field.
