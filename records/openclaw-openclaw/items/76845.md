## Summary

Keep this PR open: current main still drops elevated defaults across exec approval follow-up runs, and the PR is a narrow source-backed fix with focused regression coverage.

## What This Changes

The PR carries trusted elevated exec defaults through approval follow-up runs via an internal session-bound token, forwards them into embedded execution, and adds tests plus a changelog entry.

## Best Possible Solution

Land the internal token-based propagation after normal maintainer review, CI, and any needed rebase, keeping elevated-default transfer session-scoped and out of the public gateway schema.

## Reproduction Assessment

Yes, source-reproducible. Current main sends the approval follow-up without `bashElevated`, and the exec tool requires `defaults.elevated` to keep elevated `on`/`ask` available.

## Solution Assessment

Yes. The PR is the narrow maintainable fix because it transfers only trusted internal elevated defaults through a random one-use session-bound token without changing approval policy or the public `agent` schema.
