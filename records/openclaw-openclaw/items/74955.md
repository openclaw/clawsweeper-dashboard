## Summary

Keep this PR open: current main still has the message-tool outbound TTS gap, and the PR has useful live before/after proof, but the patch is not mergeable as written because it can synthesize during dry-run sends and drops scoped TTS context.

## What This Changes

The PR adds tagged-TTS normalization to outbound message-tool sends, plus a regression test and changelog entry.

## Best Possible Solution

Land a current-main TTS normalization pass for message-tool sends that is dry-run safe, scoped by agent/account/channel, preserves existing media and `audioAsVoice` behavior, and has focused regression coverage.

## Reproduction Assessment

Yes. Source inspection shows `message(action="send")` reaches the outbound send path without a TTS hook, while final replies call the scoped TTS normalizer before delivery.

## Solution Assessment

No, not as written. Reusing the existing TTS hook is the right narrow approach, but this patch must skip dry-run synthesis, pass scoped context into the runtime call, and isolate TTS prefs in the test before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Skip TTS synthesis during dry-run sends:** `src/infra/outbound/message-action-runner.ts:593-597`
  - body: This hook runs before the existing no-delivery dry-run boundary, so `openclaw message send --dry-run` can still send the message body to a configured external TTS provider. Add the dry-run guard before loading or invoking the TTS runtime.
  - confidence: 0.9
- **[P2] Pass scoped TTS context into the hook:** `src/infra/outbound/message-action-runner.ts:599-604`
  - body: The gate resolves TTS with `agentId` and `accountId`, but the runtime call omits both. Because `maybeApplyTtsToPayload` re-resolves auto mode, provider, voice, and prefs from those fields, scoped overrides can pass the gate and then synthesize with the wrong effective config or skip unexpectedly.
  - confidence: 0.91
- **[P3] Isolate TTS prefs in the regression test:** `src/infra/outbound/message-action-runner.tagged-tts.test.ts:20-30`
  - body: `shouldAttemptTtsPayload` reads the local TTS prefs path before `messages.tts.auto`. Because this test does not pin `prefsPath` or isolate `OPENCLAW_TTS_PREFS`, a developer or CI image with local prefs set to `off` can make the tagged case skip the mocked hook.
  - confidence: 0.82
