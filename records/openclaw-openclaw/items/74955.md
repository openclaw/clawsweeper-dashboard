## Summary

Keep open: current main still lacks the message-tool tagged-TTS hook, but this PR head still has blocking correctness issues around scoped TTS context and dry-run side effects. The remaining work is narrow enough for an automated repair on the PR branch.

## What This Changes

The PR adds a `shouldAttemptTtsPayload`-gated TTS hook to `handleSendAction`, a tagged-TTS outbound regression test, and a changelog entry.

## Best Possible Solution

Land the narrow outbound-send hook after it skips dry-run, passes scoped TTS context, preserves current asVoice plumbing, and keeps TTS parsing/runtime policy unchanged.

## Reproduction Assessment

Yes, by source inspection. On current main, tagged text sent through `handleSendAction` reaches outbound send without invoking `maybeApplyTtsToPayload`, while the auto-reply path does invoke that hook with scoped context.

## Solution Assessment

No, not as written. The hook belongs in this path, but it must pass `agentId`/`accountId`, avoid dry-run provider side effects, and make the regression test independent of local TTS prefs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Pass TTS context into the runtime hook:** `src/infra/outbound/message-action-runner.ts:599-604`
  - body: The gate resolves TTS with `agentId` and `accountId`, but the runtime call omits both. Since `maybeApplyTtsToPayload` re-resolves auto mode, provider, voice, and prefs from those same fields, scoped overrides can pass the gate and then synthesize with the wrong effective config. Pass `agentId` and `accountId: accountId ?? undefined` into this call.
  - confidence: 0.91
- **[P2] Skip TTS synthesis during dry-run sends:** `src/infra/outbound/message-action-runner.ts:593-597`
  - body: This hook runs before `sendMessage` reaches its dry-run return, so `openclaw message send --dry-run` can still call a configured TTS provider and create an audio artifact. Add `!dryRun` to the gate so dry-run stays a planning-only path with no provider side effects.
  - confidence: 0.86
- **[P3] Isolate TTS prefs in the regression test:** `src/infra/outbound/message-action-runner.tagged-tts.test.ts:20-30`
  - body: `shouldAttemptTtsPayload` reads the local TTS prefs path before `messages.tts.auto`. Because this test does not pin `prefsPath` or isolate `OPENCLAW_TTS_PREFS`, a developer or CI image with local prefs set to `off` can make the tagged case skip the mocked hook. Use a test-local nonexistent prefs path or isolate the env.
  - confidence: 0.82
