## Summary

Keep open. Current main still provides usage and cost reporting, but it does not define global/per-agent spend budgets or enforce a pre-model-call admission decision; the related open issues are overlapping but not a clear canonical replacement.

## What This Changes

Request optional global and per-agent daily/monthly model-spend budgets enforced by the Gateway before dispatch, with status and doctor visibility.

## Best Possible Solution

Define one core budget policy for model spend, with strict config/schema/docs, durable per-agent accounting or reservations, shared pre-dispatch enforcement across gateway, fallback, cron, and subagent paths, structured over-budget outcomes, and status/doctor reporting.

## Reproduction Assessment

Not applicable as a feature request. The current-main gap is high-confidence by source inspection: strict config schemas reject the proposed budget surface and model dispatch paths do not check spend before provider attempts.

## Solution Assessment

Unclear until maintainers choose the product shape. Gateway-level enforcement appears to be the right boundary for reliable prevention, but config scope, reservations, fallback behavior, and any plugin API alternative need explicit design approval.
