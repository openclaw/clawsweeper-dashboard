## Summary

Keep open. Current main and v2026.5.18 still provide cost visibility plus a narrow idle-timeout runaway breaker, but they do not define a native per-agent budget config contract or enforce spend admission before model dispatch; recent comments add useful production design input rather than a reason to close.

## What This Changes

Request optional per-agent daily/monthly cost budgets enforced at the gateway before model dispatch, with block/warn/pause actions and operator status visibility.

## Best Possible Solution

Define one native gateway spend-admission contract covering per-agent config, estimated-cost reservation and settlement, structured budget receipts, reset windows, fallback attribution, and operator visibility before wiring enforcement into provider dispatch.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main lacks a per-agent budget schema and pre-dispatch spend enforcement path.

## Solution Assessment

Unclear as a final implementation. Gateway admission before provider dispatch appears to be the right boundary, but schema, accounting, reservation, reset, fallback, receipt, and action semantics need maintainer product decisions first.
