## Summary

Keep open. Current main and the latest shipped tag still provide cost/usage visibility and provider transport plumbing, but not a per-agent spend-budget config contract, not a pre-dispatch gateway admission gate, and not settled semantics for budget windows, reservations, receipts, hierarchy, or over-budget actions.

## What This Changes

Request optional daily/monthly per-agent cost budgets enforced before model dispatch at the gateway, with operator-visible status and configurable over-budget behavior.

## Best Possible Solution

Define a native gateway spend-admission contract that covers config, approximate or reserved accounting, serialized budget receipts, reset windows, fallback/model attribution, and operator visibility before wiring it into provider dispatch.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main lacks the requested per-agent budget schema and pre-dispatch spend-enforcement path.

## Solution Assessment

Unclear as a final implementation. Gateway admission before provider dispatch appears to be the right boundary, but schema, accounting, reservation, reset-window, fallback, and receipt semantics need maintainer product decisions first.
