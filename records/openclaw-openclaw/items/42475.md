## Summary

Keep open. Current main has cost reporting, provider quota/cooldown handling, and a plugin `before_agent_run` block gate, but it still lacks native per-agent spend-budget config, pre-dispatch estimate/reservation/admission, durable budget receipts, status/doctor UX, and block/warn/pause semantics; narrower spend-limit reports have already been consolidated here rather than replacing it.

## What This Changes

The issue requests optional gateway-enforced per-agent spend budgets with daily/monthly caps and block, warn, or pause actions before model dispatch.

## Best Possible Solution

Keep this as the canonical per-agent gateway spend-budget design tracker and have maintainers define an opt-in native budget contract covering config, accounting persistence, pre-dispatch reservation/admission, fallback/concurrency behavior, receipts, status/doctor UX, and plugin interaction.

## Reproduction Assessment

Yes at source level: current main exposes cost reporting and plugin blocking primitives, but schema/runtime/docs lack a native per-agent budget config and pre-dispatch spend admission path. This is a feature gap, not a failing existing contract.

## Solution Assessment

Unclear until maintainers scope it. Native core may be justified for generic model-dispatch budget admission, but the implementation must avoid duplicating plugin-only policies and should reuse existing usage/cost, session suspension, and hook primitives where appropriate.
