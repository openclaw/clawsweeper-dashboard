## Summary

Keep open. Current main still provides cost/usage visibility and reactive provider cooldown handling, but it does not define a per-agent spend budget config or a gateway admission check before model dispatch; the recent comments add useful external implementation patterns and confirm this needs maintainer product scoping rather than cleanup closure.

## What This Changes

Request optional daily/monthly per-agent spend budgets enforced by the gateway before model dispatch, with configurable block/warn/pause actions and status/doctor visibility.

## Best Possible Solution

Define one native gateway spend-admission contract covering config, accounting/reservation, fallback attribution, structured over-budget errors, and operator visibility, then implement it against provider dispatch with regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main lacks a per-agent budget schema and pre-dispatch spend enforcement path.

## Solution Assessment

Unclear as a final implementation. Gateway admission before provider dispatch is the right boundary, but schema, accounting, reservation, reset, fallback, and action semantics need maintainer product decisions first.
