## Summary

Keep open. Current main and v2026.6.6 have cost reporting plus pre-run blocking hooks, but they still lack the requested per-agent budget config, atomic spend reservation ledger, pre-dispatch per-call enforcement semantics, and status/doctor UX; related cost-control issues now point back here as the canonical tracker.

## What This Changes

Request optional per-agent daily/monthly spending caps enforced before gateway model dispatch so operators can prevent runaway provider costs.

## Best Possible Solution

Define an opt-in budget admission contract with atomic reserve/commit/release accounting, structured block receipts, status/doctor visibility, and a maintainer-approved core-versus-plugin boundary before implementation.
