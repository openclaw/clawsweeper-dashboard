## Summary

Keep open. Current main and the latest shipped tag still provide usage/cost visibility and plugin hooks, but there is no native per-agent spend-budget config contract or gateway admission check that reserves or blocks before provider dispatch; the recent discussion adds useful requirements around receipts, reservation ordering, and concurrency rather than making the request obsolete.

## What This Changes

Request optional per-agent daily/monthly spend budgets enforced by the gateway before model/provider dispatch.

## Best Possible Solution

Define one backward-compatible gateway spend-admission contract that covers native config, per-agent attribution, estimated or reserved accounting, serialized receipts, reset windows, fallback/model attribution, status/doctor visibility, and plugin-hook coexistence before wiring enforcement into provider dispatch.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main lacks the requested per-agent spend-budget schema and pre-dispatch spend-enforcement path.

## Solution Assessment

Unclear as a final implementation. Gateway admission before provider dispatch is a plausible boundary, but native core scope versus plugin/API extension, accounting source, reservation order, reset windows, and over-budget actions need maintainer product decisions first.
