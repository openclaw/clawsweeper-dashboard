## Summary

Keep open. Current main and the latest shipped tag still have usage and cost visibility, but no per-agent budget config contract and no gateway spend-admission check before provider dispatch; the recent comments add useful design constraints around reservations, receipts, and concurrency rather than making the item obsolete.

## What This Changes

Request optional per-agent daily/monthly cost budgets enforced by the gateway before model calls, with block/warn/pause behavior and status/doctor visibility.

## Best Possible Solution

Define one native gateway spend-admission contract covering config, approximate or reserved accounting, serialized budget receipts, reset windows, fallback/model attribution, and operator visibility before wiring enforcement into provider dispatch.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main lacks the requested per-agent budget schema and pre-dispatch spend-enforcement path.

## Solution Assessment

Unclear as a final implementation. Gateway admission before provider dispatch appears to be the right boundary, but schema, accounting, reservation, reset-window, fallback, receipt, and action semantics need maintainer product decisions first.
