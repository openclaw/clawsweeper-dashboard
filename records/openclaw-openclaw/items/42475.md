## Summary

Keep open. Current main still has cost visibility and provider quota/cooldown handling, but not the requested native per-agent budget schema or pre-dispatch spend-admission gate; the recent comments add reservation, receipt, hierarchy, and concurrency semantics that need maintainer scoping.

## What This Changes

Request optional per-agent daily/monthly spend budgets enforced by the gateway before metered model calls, with status/doctor visibility and configurable over-budget actions.

## Best Possible Solution

Define a native gateway spend-admission contract covering config, accounting or reservation, structured over-budget receipts, fallback attribution, and operator visibility before wiring enforcement into provider dispatch.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main lacks the requested per-agent budget schema and pre-dispatch spend-enforcement path.

## Solution Assessment

Unclear as a final implementation. Gateway admission before provider dispatch appears to be the right boundary, but schema, accounting, reservation, reset-window, fallback, and receipt semantics need maintainer product decisions first.
