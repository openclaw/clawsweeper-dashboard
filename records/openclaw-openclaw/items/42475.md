## Summary

Keep open. Current main still provides cost and usage reporting, provider quota visibility, and storage/control-plane limits, but it has no accepted per-agent spend budget config and no pre-dispatch model-call admission gate; the related open issues are broader limit/rate-control discussions rather than a clear canonical replacement for this more specific request.

## What This Changes

Request adds optional daily/monthly per-agent model spend budgets enforced by the Gateway before dispatching model calls, with status and doctor visibility.

## Best Possible Solution

Define one core model-spend budget policy with strict config/schema/docs, durable per-agent accounting or reservations, shared pre-dispatch enforcement across Gateway/fallback/subagent paths, structured over-budget outcomes, and status/doctor reporting.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-main gap: strict schemas reject the proposed budget surface and model dispatch paths do not inspect spend before starting attempts.

## Solution Assessment

Unclear until maintainers choose the product shape. Gateway-level admission is a plausible boundary, but config scope, durable accounting/reservation, fallback and subagent behavior, and plugin-hook participation need explicit design approval.
