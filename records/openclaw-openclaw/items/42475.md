## Summary

Keep open. Current main has cost and quota visibility plus reactive provider cooldown/fallback behavior, but it still lacks an accepted per-agent cost budget config and a pre-dispatch gateway/model-call enforcement gate; the closed spend-ceiling report at https://github.com/openclaw/openclaw/issues/38248 explicitly points remaining cost-guard work here, while broader rate-limit/token-limit trackers remain related design inputs.

## What This Changes

The issue requests optional per-agent daily/monthly cost budgets enforced before gateway model dispatch, with warn/block/pause behavior and status/doctor visibility.

## Best Possible Solution

Define an additive core budget-policy contract at the gateway or agent-runtime model-dispatch boundary, with strict config/schema/docs/status/doctor support and focused tests for pre-dispatch allow/warn/block/pause behavior.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and docs inspection give high confidence that current main lacks the requested budget config and pre-dispatch spend-enforcement path.

## Solution Assessment

Unclear as an implementation plan until maintainers choose the policy contract. The direction is valid, but a safe fix needs explicit decisions on accounting authority, reset windows, and over-budget actions before code changes.
