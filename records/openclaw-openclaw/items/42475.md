## Summary

Keep open. Current main still has cost and quota visibility plus provider cooldown handling, but no per-agent spend-budget config contract or pre-dispatch gateway admission gate; the discussion adds useful external patterns and sharper design questions around reservations, receipts, hierarchy, and over-budget actions.

## What This Changes

Add optional per-agent daily/monthly cost budgets that the gateway enforces before model dispatch, with configurable over-budget behavior and operator status/doctor visibility.

## Best Possible Solution

Define a native gateway spend-admission contract covering config, approximate or reserved accounting, serialized receipts, reset windows, fallback attribution, and operator visibility before wiring enforcement into model dispatch.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection on current main gives high confidence that the agent schema and provider dispatch path lack the requested budget fields and pre-dispatch spend gate.

## Solution Assessment

Unclear as a final implementation. Gateway admission before provider dispatch appears to be the right boundary, but schema, accounting, reservation, reset-window, fallback, and receipt semantics need maintainer product decisions first.
