## Summary

Keep open. Current `main` and `v2026.6.6` still have cost visibility plus a generic pre-run plugin block hook, but not the requested native per-agent spend budget config, reservation/accounting ledger, gateway admission policy, or status/doctor UX; related narrower cost-budget issues point back here as the canonical design tracker.

## What This Changes

The issue requests optional per-agent daily and monthly model-spend budgets enforced before gateway model dispatch.

## Best Possible Solution

Keep this as the canonical product/design tracker and have maintainers choose a scoped opt-in budget contract covering schema, SQLite-backed accounting/reservation, gateway/provider admission semantics, and status/doctor reporting.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new configurable budget-enforcement capability. Source inspection gives high confidence that current main and the latest release have usage visibility and a plugin block seam, but no native per-agent spend-budget admission policy.

## Solution Assessment

Unclear until maintainers choose the product contract. Gateway-level pre-dispatch admission is the right design center, but the exact fix needs decisions on config scope, reservation/accounting storage, fallback behavior, and what stays core versus plugin-owned.
