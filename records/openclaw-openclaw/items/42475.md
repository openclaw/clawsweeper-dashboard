## Summary

Keep open. Current main and the current release have cost reporting and a generic pre-model plugin block hook, but they still do not provide native per-agent spend-budget config, pre-dispatch cost reservation/admission, budget receipts, failover/probe semantics, or status/doctor UX; the external wallet/proxy/mandate examples in the discussion are useful design input but do not close the core product gap.

## What This Changes

The issue requests optional per-agent daily/monthly model-spend budgets enforced by the gateway before model dispatch, with block/warn/pause behavior and operator visibility.

## Best Possible Solution

Keep #42475 as the canonical design issue and require maintainer-approved opt-in budget admission semantics before any implementation PR.

## Reproduction Assessment

Yes at source level: current main exposes cost aggregation and a generic pre-run block hook, but no native budget config, ledger, reservation, or pre-dispatch spend admission policy.

## Solution Assessment

Unclear until maintainers scope the product/API boundary; gateway admission is the right native enforcement point, while the existing plugin hook is useful but insufficient for first-class budget accounting and receipts.
