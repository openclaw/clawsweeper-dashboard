## Summary

Keep open. Current main and latest release still provide cost accounting plus a generic pre-run block hook, but not the requested native per-agent spend-budget config, reservation/admission policy, or status/doctor UX; related narrower cost-budget issues have been consolidated here rather than replacing it.

## What This Changes

The issue requests optional gateway-level per-agent spending budgets with daily/monthly caps, pre-dispatch enforcement, and status/doctor visibility.

## Best Possible Solution

Keep this issue open as the canonical per-agent gateway spend-budget design item, then approve an opt-in budget policy covering schema, pre-dispatch admission/reservation, persistence, status, doctor, docs, and upgrade tests together.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.6 have strict agent schemas with no native budget settings, while `before_agent_run` only provides a generic pre-run block primitive.

## Solution Assessment

Unclear until maintainers choose scope. The hook path can support external policy, but native gateway budgets need explicit decisions for config shape, spend windows, reservation semantics, persistence, and operator UX.
