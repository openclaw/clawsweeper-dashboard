## Summary

Keep open. Current main still has cost visibility, reactive provider failure handling, and a pre-model plugin gate, but it does not define a native per-agent budget config or gateway spend-admission contract; the recent ATXP, nornr, KinthAI, MartinLoop, and ALEF discussion adds useful design input that needs maintainer scoping rather than cleanup closure.

## What This Changes

The issue requests optional daily/monthly per-agent spend budgets enforced by the gateway before metered model dispatch, with status visibility and configurable over-budget behavior.

## Best Possible Solution

Define a native gateway spend-admission contract covering config, authoritative accounting or reservation, fallback attribution, structured over-budget receipts/errors, and operator visibility before wiring enforcement into provider dispatch.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main lacks a per-agent budget schema and pre-dispatch spend enforcement path.

## Solution Assessment

Unclear as a final implementation. Gateway admission before provider dispatch appears to be the right boundary, but schema, accounting, reservation, reset, fallback, and receipt semantics need maintainer product decisions first.
