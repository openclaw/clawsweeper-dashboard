## Summary

Keep open. Current main still has cost tracking and aggregation, but not the requested per-agent daily/monthly budget contract or a gateway/model-attempt admission gate. The related open items cover narrower failover ceilings or broader rate/token limiting, and the recent comments add useful production context rather than making this obsolete.

## What This Changes

Add optional per-agent daily/monthly cost budget configuration enforced before gateway model dispatch, with warn/block/pause behavior plus status and doctor visibility.

## Best Possible Solution

Define one core budget policy for agent model spend, with strict config/schema/docs, durable per-agent spend accounting or reservations, a pre-model-call decision point shared by gateway, CLI, cron, fallback, and subagent paths, structured over-budget outcomes, and status/doctor reporting. If maintainers prefer external enforcement, first add a documented pre-model budget hook with enough pricing, agent, run, and reservation state to block reliably before provider dispatch.

## Reproduction Assessment

Not applicable as a feature request, but the current-main gap is high-confidence by source inspection. The strict schemas reject the proposed budget surface and the gateway/model-attempt paths do not perform a spend admission check before dispatch.

## Solution Assessment

Unclear until maintainers choose product scope. The requested core/gateway direction appears narrower and more reliable than today’s plugin hooks for this boundary; a plugin-only path would need a new first-class pre-model-call budget API before it could enforce all relevant runs.
