## Summary

Keep open. Current main has cost visibility, provider usage readouts, and reactive quota/rate-limit suspension, but it still lacks native per-agent spend-budget config and a pre-dispatch gateway admission gate that can block, warn, or pause before model cost is incurred.

## What This Changes

Requests optional per-agent daily/monthly spend budgets enforced before gateway model dispatch, with block/warn/pause behavior plus status and doctor visibility.

## Best Possible Solution

Keep this as the canonical core design tracker for an opt-in per-agent spend guard, then define one gateway admission contract covering schema, accounting, reservations, reset windows, fallback interaction, and status/doctor UX.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence that the requested per-agent budget config and pre-dispatch spend enforcement surface are absent on current main.

## Solution Assessment

Unclear as a final implementation. Gateway admission before provider dispatch is the right boundary, but maintainers need to settle accounting, reservation, reset, fallback, and user-facing action semantics before implementation.
