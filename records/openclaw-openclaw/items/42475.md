## Summary

Keep open. Current main and v2026.6.6 still provide cost reporting plus a generic pre-model plugin block hook, but not the requested built-in per-agent budget config, spend reservation/accounting policy, gateway admission behavior, or status/doctor UX; related narrower reports were consolidated into this item, so this remains the canonical product-design tracker.

## What This Changes

The issue requests optional gateway-level per-agent daily/monthly spend budgets that can block, warn, or pause model dispatch before provider calls.

## Best Possible Solution

Keep this as the canonical design tracker and have maintainers approve a scoped, opt-in budget policy covering schema, SQLite accounting/reservation, provider/fallback admission semantics, and operator-facing status/doctor output.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new configurable budget-enforcement capability. Source inspection gives high confidence that current main has cost observability and a plugin block seam, but no built-in per-agent spend budget admission policy.

## Solution Assessment
