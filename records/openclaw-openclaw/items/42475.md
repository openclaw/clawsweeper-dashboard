## Summary

Keep open. Current main has a useful pre-run block hook and cost observability, but it still lacks the requested per-agent budget config, atomic spend ledger/reservation semantics, pre-dispatch per-call enforcement, and status/doctor UX; recent comments add concrete production design input rather than making the issue stale.

## What This Changes

Request optional per-agent daily/monthly cost caps enforced before metered gateway model dispatch, with configurable block/warn/pause behavior and operator visibility.

## Best Possible Solution

Define a maintainer-approved, opt-in budget admission design: a per-call reservation contract with agent/run/model identity and structured block receipts, backed by canonical accounting and surfaced in status/doctor, with policy living in a plugin unless core ownership is explicitly chosen.

## Reproduction Assessment

Not applicable: this is a new enforcement capability, not a broken existing contract. Source inspection against current main and v2026.6.6 confirms cost reporting and a run-level block hook exist, but no per-agent spend-budget enforcement is implemented.

## Solution Assessment

No, the issue's proposed direct session-cost sum plus agent config is not yet the best complete fix because it does not settle concurrent reservations, pricing uncertainty, retries/fallbacks, or ownership. The best path is a scoped admission/accounting design first, then implementation behind an opt-in policy surface.
