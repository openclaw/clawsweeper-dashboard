## Summary

Keep open. Current main still has usage and cost reporting, provider quota visibility, storage budgets, and reactive provider/fallback handling, but it does not have a strict per-agent spend-budget config or a shared pre-dispatch admission gate for model calls; the related live issues are broader or adjacent rather than a clean replacement for this specific gateway-budget request.

## What This Changes

The issue requests optional per-agent daily/monthly model-spend budgets enforced before gateway model dispatch, plus status and doctor visibility.

## Best Possible Solution

Define one additive core model-spend budget policy with strict defaults/per-agent config, durable per-agent accounting or reservation, shared pre-dispatch enforcement across gateway/fallback/subagent paths, structured over-budget outcomes, and status/doctor reporting.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs inspection give a high-confidence current-main gap because the proposed budget config is absent from strict schemas and model dispatch paths do not check spend before starting attempts.

## Solution Assessment

Unclear until maintainers choose the public contract. Gateway-level admission is a plausible boundary, but config scope, accounting authority, reservation behavior, fallback/subagent semantics, and operator UX need product design before implementation.
