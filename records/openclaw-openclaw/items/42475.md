## Summary

Keep open. Current main has usage visibility and reactive provider/failover handling, but no accepted per-agent cost-budget config or pre-dispatch gateway admission gate; related discussion also treats this as the narrower cost-budget tracker rather than a resolved or stale report.

## What This Changes

The issue requests optional per-agent daily and monthly cost-budget enforcement before gateway model dispatch.

## Best Possible Solution

Define an additive core budget-policy contract at the model-dispatch boundary, with per-agent/default schema, estimated-cost reservation, post-response accounting, explicit reset windows, status/doctor visibility, and tests for warn/block/pause or any accepted action set.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and docs inspection give high confidence that current main lacks the requested budget config and pre-dispatch enforcement path.

## Solution Assessment

Unclear as a concrete implementation until maintainers choose the accounting source, reset semantics, and action model. The maintainable direction is a single gateway/agent-runtime policy surface rather than ad hoc checks in cost-reporting code.
