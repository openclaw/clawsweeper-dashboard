## Summary

Keep open. Current main and the latest release still provide cost reporting plus a generic pre-run plugin block hook, but they do not implement native per-agent spend-budget config, pre-dispatch admission/reservation, receipts, status/doctor UX, or failover/probe semantics; narrower budget issues are already consolidated here.

## What This Changes

The issue requests optional daily and monthly per-agent cost budgets enforced by the gateway before model dispatch, with block/warn/pause and status/doctor behavior.

## Best Possible Solution

Keep this as the canonical design item and require maintainer-approved scope for opt-in budget config, pre-dispatch admission/reservation, persistence, receipts, status/doctor/docs, and upgrade tests before implementation.

## Reproduction Assessment

Yes at source level: current main lacks the requested agent budget config and native pre-dispatch spend gate, so this is a feature gap rather than a failing existing-behavior bug.

## Solution Assessment

Unclear until maintainers choose scope; the narrowest maintainable path is an opt-in native budget policy that reuses existing cost visibility and hook primitives without creating competing config, plugin, or provider-routing semantics.
