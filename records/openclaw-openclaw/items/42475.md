## Summary

Keep open. Current main and v2026.6.10 still have cost reporting and a generic pre-run block hook, but not native per-agent spend-budget config, pre-dispatch reservation/admission, budget receipts, or status/doctor UX; related narrower budget issues have been consolidated here rather than replacing it.

## What This Changes

Request optional per-agent daily/monthly cost budgets enforced before model dispatch at the gateway, with status and doctor visibility.

## Best Possible Solution

Keep this as the canonical design item for one opt-in per-agent budget policy covering admission/reservation, accounting, receipts, status/doctor/docs, and upgrade tests.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.10 lack the requested native budget config, spend ledger/reservation gate, and status/doctor UX. This is a feature gap rather than a broken existing-behavior bug with a failing runtime reproduction path.

## Solution Assessment

Unclear until maintainers choose scope. The narrowest maintainable direction is one opt-in native budget policy that reuses existing cost reporting and pre-run hook primitives without creating competing config or provider-routing semantics.
