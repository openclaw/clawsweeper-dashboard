## Summary

Keep open. Current main and v2026.6.6 provide cost/usage observability plus a generic pre-model block hook, but they still lack the requested native budget config, spend reservation/accounting, gateway admission policy, and status/doctor UX; related narrower reports have been consolidated into this issue, so it remains the canonical product tracker.

## What This Changes

Request optional per-agent daily and monthly cost budgets enforced before gateway model dispatch, with block/warn/pause behavior and operator-facing status and doctor visibility.

## Best Possible Solution

Approve a phased, opt-in budget admission contract that covers public schema, SQLite-backed spend and reservation records, provider/fallback semantics, and operator-visible status and doctor output.

## Reproduction Assessment

Not applicable as a bug reproduction: this requests a new configurable budget-enforcement capability. Source inspection gives high confidence that current main has observability and a generic block seam, but no built-in per-agent spend budget admission policy.

## Solution Assessment

Unclear until maintainers choose the product contract. The issue identifies a real gap, but the best fix needs explicit scope for config, accounting, reservation, fallback behavior, and core versus plugin ownership.
