## Summary

Keep open. Current main still has cost/usage reporting and provider cooldown/fallback behavior, but no accepted per-agent spend budget config and no shared pre-dispatch model-call admission gate; the newer related issues remain broader policy discussions rather than a canonical replacement for this specific gateway budget request.

## What This Changes

Request optional per-agent daily/monthly model-spend budgets enforced by the gateway before dispatching model calls.

## Best Possible Solution

Define one opt-in core model-spend budget policy with strict config/schema/docs, durable per-agent accounting or reservations, shared pre-dispatch enforcement across Gateway/fallback/subagent paths, structured over-budget outcomes, and status/doctor visibility.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a feature request; source inspection gives high confidence that current main lacks the requested config surface and pre-dispatch spend gate.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose the product contract; gateway-level admission is plausible, but accounting authority, action semantics, fallback/subagent handling, and status UX need explicit scoping first.
