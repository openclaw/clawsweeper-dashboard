## Summary

Keep open. Current main and v2026.6.6 have cost/usage observability plus a generic pre-model block hook, but not the requested native per-agent cost budget config, spend reservation/accounting policy, gateway admission behavior, or status/doctor UX; narrower related reports were closed into this issue, so it remains the canonical design tracker.

## What This Changes

Request adds optional per-agent daily/monthly cost budgets enforced before gateway model dispatch, with block/warn/pause actions and status/doctor visibility.

## Best Possible Solution

Define one canonical admission and spend-governance design: ownership boundary, budget config/schema, reservation/accounting store, block/warn/pause semantics, provider/fallback behavior, status/doctor output, and how plugins participate without duplicating core observability.

## Reproduction Assessment

Yes, source-level. Current main exposes usage/cost summaries and a generic before_agent_run block hook, but source and v2026.6.6 config surfaces have no daily/monthly per-agent cost budget or gateway admission enforcement.

## Solution Assessment

Unclear. The direction is useful, but the best solution depends on maintainer choices about core versus plugin ownership, reservation accuracy, provider fallback behavior, and operator-visible status/doctor semantics.
