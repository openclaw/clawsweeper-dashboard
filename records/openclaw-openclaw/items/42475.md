## Summary

Keep open. Current main exposes cost reporting, provider quota/cooldown handling, and a `before_agent_run` plugin block gate, but it still lacks native per-agent spend-budget config, pre-dispatch admission/reservation, durable budget receipts, and status/doctor UX.

## What This Changes

The issue requests optional gateway-level daily/monthly per-agent spend caps with block, warn, or pause behavior before model dispatch.

## Best Possible Solution

Keep this as the canonical design tracker for an opt-in native budget contract covering config, accounting persistence, pre-dispatch admission, reservations, receipts, status/doctor UX, and plugin interaction.

## Reproduction Assessment

Yes at source level: current main exposes cost reporting and plugin blocking primitives, but schema/runtime/docs lack native per-agent budget config and pre-dispatch spend admission.

## Solution Assessment

Unclear until maintainers scope it; native core may be justified for generic model-dispatch budget admission, but it must avoid duplicating plugin-only policy and reuse existing usage, suspension, and hook primitives where appropriate.
