## Summary

Keep open. Current main provides cost reporting and a generic pre-run block hook, but it still lacks the requested native per-agent spend-budget config, pre-dispatch admission/reservation policy, failover/probe semantics, and status/doctor UX; narrower budget requests have already been consolidated here rather than replacing it.

## What This Changes

Request optional per-agent daily or monthly spend budgets enforced before gateway model dispatch, with block, warn, pause, or downgrade behavior and status visibility.

## Best Possible Solution

Keep #42475 as the canonical design item and require maintainer-approved scope for opt-in budget config, pre-dispatch reservation/admission, persistence, status/doctor/docs, and upgrade tests before implementation.

## Reproduction Assessment

Yes at source level: strict agent schemas have no budget fields, usage.cost only reports transcript-backed summaries, and before_agent_run is a generic hook rather than a native budget policy.

## Solution Assessment

Unclear until maintainers choose scope; the hook path is a useful external policy primitive, but native budgets need decisions on config shape, accounting accuracy, reservations, failover handling, persistence, and UX.
