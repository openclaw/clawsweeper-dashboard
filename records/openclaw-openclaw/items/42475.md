## Summary

Keep open. Current main and the latest release still provide cost reporting plus a generic pre-model block hook, but not the requested built-in per-agent budget config, reservation/accounting policy, gateway admission behavior, or status/doctor UX; this remains the canonical product-design tracker for pre-dispatch spend guardrails.

## What This Changes

Request optional daily/monthly per-agent cost budgets enforced before gateway model dispatch, with block/warn/pause behavior and operator-visible status.

## Best Possible Solution

Keep this issue as the canonical design tracker and define a phased, opt-in gateway budget policy with canonical accounting, pre-dispatch reservation, clear block/warn/pause semantics, and aligned status/doctor reporting.

## Reproduction Assessment

Yes at source level. Current main has per-agent usage reporting and a generic before_agent_run block hook, but no built-in cost-budget config, pre-dispatch reservation, or gateway enforcement ledger.

## Solution Assessment

Unclear until maintainers choose the product/API shape. The best direction is not a one-off config patch, but a scoped admission and accounting design that respects core/plugin boundaries and upgrade-safe config rules.
