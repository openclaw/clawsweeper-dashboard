## Summary

Keep open. Current main and the latest release still provide cost/usage visibility plus reactive provider cooldown/fallback handling, but they do not define a per-agent budget config contract or enforce a pre-dispatch spend gate; the recent discussion adds useful external implementation context around quota hierarchy, pessimistic reservation, structured receipts, reset windows, and action policy that needs maintainer scoping.

## What This Changes

Request optional per-agent daily/monthly cost budgets enforced by the gateway before model dispatch, with operator-visible budget status and configurable warn/block/pause behavior.

## Best Possible Solution

Define one native gateway spend-admission contract covering config, accounting or reservation, fallback attribution, structured over-budget errors or receipts, and operator visibility before wiring enforcement into provider dispatch.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main lacks a per-agent budget schema and pre-dispatch spend enforcement path.

## Solution Assessment

Unclear as a final implementation. Gateway admission before provider dispatch appears to be the right boundary, but schema, accounting, reservation, reset, fallback, and action semantics need maintainer product decisions first.
