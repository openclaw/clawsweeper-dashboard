## Summary

Keep open. Current main has cost/usage reporting and reactive provider cooldown/failover handling, but no accepted budget config or pre-dispatch gateway spend gate; related reports treat this as the narrower canonical cost-budget request rather than a duplicate.

## What This Changes

Request optional global and per-agent daily/monthly cost budgets enforced before gateway model dispatch, with block/warn/pause actions and status/doctor visibility.

## Best Possible Solution

Define an additive budget-policy contract at the model-dispatch or gateway admission boundary, with schema, accounting source, reset windows, actions, status/doctor UX, and tests coordinated with the broader limit and plugin API trackers.

Not applicable as a bug reproduction. Source and docs inspection gives high confidence that current main lacks the requested budget config and pre-dispatch enforcement path.

## Solution Assessment
