## Summary

Keep open. Current main and v2026.6.8 still provide cost visibility and a generic pre-run plugin block hook, but they do not implement native per-agent spend-budget config, pre-dispatch budget reservation/admission, budget receipts, status/doctor UX, or failover/probe semantics; narrower budget issues have been consolidated here rather than replacing it.

## What This Changes

Request to add optional per-agent daily/monthly cost budgets enforced by the gateway before model dispatch, with block/warn/pause behavior and status/doctor visibility.

## Best Possible Solution

Keep this issue as the canonical design item and require maintainer-approved scope for opt-in budget config, pre-dispatch admission/reservation, persistence, status/doctor/docs, and upgrade tests before implementation.

## Reproduction Assessment

Yes at source level: current main and v2026.6.8 lack the requested agent budget config and native pre-dispatch spend gate. This is a feature gap rather than a broken existing-behavior bug with a failing runtime reproduction path.

## Solution Assessment

Unclear until maintainers choose scope. The narrowest maintainable direction is one opt-in native budget policy that reuses existing cost visibility and hook primitives without creating competing config, plugin, or provider-routing semantics.
