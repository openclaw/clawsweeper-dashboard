## Summary

Keep open. Current main still provides cost visibility, plugin gate hooks, reactive cooldown/fallback handling, and an idle-timeout runaway breaker, but it does not provide native per-agent spend-budget config or gateway admission that blocks/warns/pauses before provider dispatch.

## What This Changes

The issue requests optional per-agent daily/monthly cost budgets enforced by the gateway before model dispatch, with status/doctor visibility and reset semantics.

## Best Possible Solution

Keep this as the scoped core design tracker for an opt-in per-agent spend guard, then design one additive gateway admission contract that covers config, accounting, reset windows, fallback interaction, status/doctor UX, and plugin boundary expectations.

## Reproduction Assessment

Not applicable as a bug reproduction. This is a feature request, and source inspection on current main gives high confidence that the requested per-agent spend-budget config and native pre-dispatch enforcement surface are absent.

## Solution Assessment

Unclear as a final implementation. Gateway admission before provider dispatch is the right boundary, but the maintainable solution needs maintainer scoping for accounting source of truth, reset semantics, actions, fallback coordination, and plugin extension points.
