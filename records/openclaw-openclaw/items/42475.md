## Summary

Keep open. Current main still has cost and usage reporting, but no accepted config surface or pre-dispatch gateway admission check for per-agent spend budgets; the overlapping related issues broaden the design space rather than replacing this request.

## What This Changes

This issue requests optional per-agent daily/monthly model-spend budgets enforced by the gateway before model calls, with status and doctor visibility.

## Best Possible Solution

Define one core model-spend budget policy with strict config/schema/docs, durable per-agent accounting or reservations, shared pre-dispatch enforcement across gateway, fallback, cron, and subagent paths, structured over-budget outcomes, and status/doctor reporting.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-main gap: strict schemas reject the proposed budget surface and gateway/fallback paths do not inspect spend before model attempts.

## Solution Assessment

Unclear until maintainers choose the product shape. Gateway-level admission is a plausible boundary, but config scope, durable reservation/accounting, fallback/subagent behavior, and plugin-hook participation need explicit design approval.
