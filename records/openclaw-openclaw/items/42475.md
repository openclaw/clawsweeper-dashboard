## Summary

Keep open. Current main and the latest release still provide cost visibility, provider quota/status reporting, auth-profile cooldowns, and a narrow idle-timeout runaway breaker, but they do not define or enforce per-agent spend budgets before model dispatch; recent discussion adds product details that need maintainer scoping.

## What This Changes

Requests optional per-agent daily/monthly cost budgets enforced before gateway model dispatch, with block/warn/pause behavior and status/doctor visibility.

## Best Possible Solution

Define one native gateway spend-admission contract that preserves unlimited behavior by default and covers config, estimation/reservation, concurrency, fallback/subagent attribution, structured receipts, and operator status/doctor surfaces.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main lacks a per-agent budget schema and pre-dispatch spend-enforcement path.

## Solution Assessment

Unclear as a final implementation. Gateway admission before provider dispatch appears to be the right boundary, but the schema, accounting, reservation, reset, fallback, receipt, and action semantics need maintainer product decisions first.
