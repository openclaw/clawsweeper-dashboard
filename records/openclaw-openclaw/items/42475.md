## Summary

Keep open. Current main still provides cost visibility and provider usage windows, but it does not have a per-agent spend-budget config contract or a pre-dispatch gateway admission gate; the recent discussion also adds reservation ordering, receipts, hierarchy, and over-budget action choices that need maintainer scoping.

## What This Changes

The issue requests optional per-agent daily/monthly cost budgets enforced before gateway model dispatch, with status/doctor visibility and configurable over-budget actions.

## Best Possible Solution

Define a native gateway spend-admission contract covering config, approximate or reserved accounting, serialized budget receipts, reset windows, fallback attribution, and operator visibility before wiring enforcement into model dispatch.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main lacks the requested per-agent budget schema and pre-dispatch spend-enforcement path.

## Solution Assessment

Unclear as a final implementation. Gateway admission before provider dispatch appears to be the right boundary, but schema, accounting, reservation, reset-window, fallback, and receipt semantics need maintainer product decisions first.
