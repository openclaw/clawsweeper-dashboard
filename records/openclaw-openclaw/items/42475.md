## Summary

Keep open. Current main still observes and aggregates usage cost, and has reactive cooldown/fallback behavior, but it does not accept per-agent spend-budget config or block model dispatch before the provider call; related cleanup also treats this as the scoped budget tracker rather than a duplicate of the broader outbound-limits work.

## What This Changes

Request adds optional per-agent daily/monthly cost budgets enforced at the gateway before model dispatch, with status/doctor visibility and block/warn/pause behavior.

## Best Possible Solution

Keep this as the scoped core tracker for an opt-in per-agent spend guard, then design one additive gateway admission contract that covers config, persistence, cost estimation, fallback interaction, status/doctor UX, and documented plugin boundaries.

## Reproduction Assessment

Not applicable as a bug reproduction. This is a feature request, and source inspection on current main gives high confidence that the requested budget config and pre-dispatch enforcement surface are absent.

## Solution Assessment

Unclear as a final design. Gateway admission before dispatch is the right boundary, but the maintainable solution needs maintainer scoping for budget accounting, reset semantics, action behavior, fallback coordination, and external/plugin extension points.
