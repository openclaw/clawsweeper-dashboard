## Summary

Keep open. Current main has cost/usage visibility and provider quota snapshots, but it still lacks a native per-agent budget schema plus a pre-dispatch spend-admission/reservation gate before model transport; the recent discussion adds receipt, concurrency, hierarchy, and action semantics that need maintainer scoping rather than cleanup closure.

## What This Changes

The issue requests config-driven per-agent daily/monthly cost budgets enforced by the gateway before model calls, with status and doctor visibility.

## Best Possible Solution

Define a native gateway spend-admission contract covering config, accounting or reservation, fallback attribution, structured over-budget receipts/errors, and operator visibility before wiring enforcement into provider dispatch.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main lacks the requested per-agent budget schema and pre-dispatch spend-enforcement path.

## Solution Assessment

Unclear as a final implementation. Gateway admission before provider dispatch appears to be the right boundary, but schema, accounting, reservation, reset-window, fallback, and receipt semantics need maintainer product decisions first.
