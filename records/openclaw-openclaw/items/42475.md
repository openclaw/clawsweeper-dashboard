## Summary

Keep open. Current main has cost/usage reporting and a pre-model plugin blocking primitive, but it still lacks the requested built-in per-agent spend-budget config, pre-dispatch reservation/enforcement, and status/doctor surfaces.

## What This Changes

The issue requests optional per-agent daily/monthly model-spend budgets enforced before gateway dispatch, with block/warn/pause actions and status/doctor visibility.

## Best Possible Solution

Keep #42475 as the canonical per-agent cost-budget tracker and scope an opt-in budget admission contract across agent config, provider dispatch/fallback/probes, durable accounting/reservations, status/doctor, and docs; use before_agent_run/plugins for interim external guards.

## Reproduction Assessment

## Solution Assessment

Unclear until maintainers scope it: before_agent_run is the right interim plugin gate, but built-in gateway budgets need an owner-approved admission, reservation, fallback, status, and doctor contract.
