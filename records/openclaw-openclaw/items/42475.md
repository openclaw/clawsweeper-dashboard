## Summary

Keep open. Current main has usage visibility, provider quota/cooldown handling, and session disk-budget cleanup, but it still lacks a native per-agent spend-budget schema and a pre-dispatch gateway admission/reservation gate; the recent discussion adds receipt and concurrency design constraints that need maintainer scoping.

## What This Changes

Request optional per-agent daily/monthly cost budgets enforced at the gateway before model dispatch, with block/warn/pause behavior and status/doctor visibility.

## Best Possible Solution

Define a native gateway spend-admission contract covering config, accounting or reservation, serialized decision receipts, fallback attribution, structured over-budget errors, and operator visibility before wiring enforcement into provider dispatch.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main lacks the requested per-agent budget schema and pre-dispatch spend-enforcement path.

## Solution Assessment

Unclear as a final implementation. Gateway admission before provider dispatch appears to be the right boundary, but schema, accounting, serialized reservation/receipt, reset-window, fallback, and action semantics need maintainer product decisions first.

## Risks / Open Questions
