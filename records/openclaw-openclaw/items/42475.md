## Summary

Keep open. Current main still has cost visibility, provider usage windows, and reactive auth-profile cooldowns, but it does not have the requested per-agent budget schema or a gateway pre-dispatch spend-admission gate; the recent discussion adds accounting, reservation, receipt, and concurrency choices that need maintainer scoping.

## What This Changes

Request optional daily/monthly per-agent cost budgets enforced before gateway model dispatch, with over-budget actions and operator status visibility.

## Best Possible Solution

Define a native gateway spend-admission contract covering config, approximate or reserved accounting, serialized budget receipts, reset windows, fallback attribution, and operator visibility before wiring enforcement into model dispatch.

## Reproduction Assessment

## Solution Assessment

Unclear as a final implementation. Gateway admission before provider dispatch appears to be the right boundary, but schema, accounting, reservation, reset-window, fallback, and receipt semantics need maintainer product decisions first.
