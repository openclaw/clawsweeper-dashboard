## Summary

Keep open. Current main has usage-cost reporting and a plugin pre-run block hook, but it still lacks the requested built-in per-agent spend budget with pre-dispatch enforcement, action semantics, and operator/status visibility.

## What This Changes

Request optional per-agent daily/monthly model-spend budgets enforced before gateway model dispatch, with block/warn/pause actions and status/doctor visibility.

## Best Possible Solution

Define one opt-in core budget-policy contract for model-provider dispatch first, with a real pre-dispatch admission/reservation path, operator-visible receipts/status, and docs/tests for fresh and upgrade behavior.
