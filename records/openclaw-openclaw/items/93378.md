## Summary

Keep open. The PR targets a real Telegram polling recovery bug, but the age-only replacement path can delete a healthy same-token polling lease after five minutes and start a competing poller, which breaks the current active-poller guard.

## What This Changes

The PR modifies Telegram polling lease acquisition to replace a non-aborted same-token lease after a five-minute staleness window instead of always throwing a duplicate-polling error.

## Best Possible Solution

Preserve duplicate-poller rejection for non-aborted live leases and recover only from explicit stopped or aborted lifecycle state, with regression coverage for a live lease older than five minutes.

## Reproduction Assessment

Yes for the PR-introduced regression at source level: a normal Telegram monitor holds its lease through `runUntilAbort()`, so a healthy lease can be older than five minutes. The original transient-timeout crash loop still needs live or equivalent Telegram proof.

## Solution Assessment

No. The patch is a plausible recovery attempt, but wall-clock age is not a safe stale-owner signal for a long-running poller; the fix should use explicit lifecycle state.
