## Summary

Keep open: the linked Telegram outage is real, but this PR is not merge-ready because age-only replacement of a non-aborted polling lease can start a competing same-token poller, and the supplied proof is test/simulation-only.

## What This Changes

The PR changes Telegram polling lease acquisition to delete and reacquire a same-token non-aborted lease once it is older than five minutes.

## Best Possible Solution

Preserve the active duplicate-poller guard and route recovery through explicit stopped/aborted lifecycle state or a gateway-level restart/escalation path with live or equivalent Telegram proof.

## Reproduction Assessment

Yes for the PR-introduced regression at source level: current main lets a healthy Telegram poller hold a non-aborted lease indefinitely, so the PR would replace it after five minutes. The original transient-timeout crash loop still needs live or equivalent proof for the exact fix path.

## Solution Assessment

No. The patch is a plausible mitigation, but wall-clock lease age is not a safe owner-death signal for a long-running poller; explicit lifecycle state is the safer fix boundary.
