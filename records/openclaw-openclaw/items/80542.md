## Summary

Keep open. The Docker pnpm 11 fix is directionally valid and has real Testbox Docker build proof, but the PR introduces a docs/setup mismatch: it tells users to set `OPENCLAW_INSTALL_BROWSER=1` with `scripts/docker/setup.sh` even though that setup script does not forward that build arg.

## What This Changes

The PR updates Docker install/prune handling for pnpm 11 workspace manifests, refreshes Docker-related docs, adds Dockerfile tests, and adds a changelog entry.

## Best Possible Solution

Land the Docker pnpm 11 workspace/prune fix after aligning the setup script and docs so `OPENCLAW_INSTALL_BROWSER` actually bakes Chromium when used with `scripts/docker/setup.sh`.

## Reproduction Assessment

Source-reproducible. Current main's Docker install layer omits `packages/*` manifests while selected plugins commonly depend on workspace packages, and the PR body provides after-fix Testbox Docker build proof; I did not run the failing current-main Docker build in this read-only review.

## Solution Assessment

No. The core pnpm 11 Docker direction is plausible and backed by real Docker proof, but the docs portion is not complete until `scripts/docker/setup.sh` forwards `OPENCLAW_INSTALL_BROWSER` or the new docs stop recommending that setup path.
