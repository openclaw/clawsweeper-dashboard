Keep open. Current main still lacks a guard that distinguishes Control UI’s own bottom-pinning scrolls from user scroll events, and the linked fix PR is still open, so this issue should remain open until that PR is reviewed and merged or replaced.

The issue reports intermittent Control UI webchat drift away from the newest streamed assistant output when the user is already at or near the bottom.
