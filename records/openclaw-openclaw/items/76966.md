## Summary

Keep open. Current main still has a plausible source-level path where Control UI programmatic scrolling can corrupt the bottom-sticky state, and open PR #76991 specifically references this issue with a proposed fix that has not merged yet.

## What This Changes

Control UI webchat should keep streaming assistant output pinned to the newest content while the user is already at or near the bottom.

## Best Possible Solution

Review and land #76991, or an equivalent Control UI scroll-state guard with regression coverage, then close this as implemented.

## Reproduction Assessment

Yes, from source inspection. Current main can schedule a programmatic scroll during `chatStream` updates, then handle the resulting scroll event as if the user moved away from bottom when content height has grown.

## Solution Assessment

Likely yes. A narrow Control UI guard that distinguishes programmatic scroll events from user scrolls, with `app-scroll.test.ts` coverage, is the maintainable fix boundary.
