## Summary

Keep open. Current main still lets a chat scroll event directly flip `chatUserNearBottom` based only on distance from bottom, while the linked open PR #76991 specifically targets this issue with a programmatic-scroll guard; this issue should stay open until that PR or an equivalent fix merges.

## What This Changes

Control UI webchat should remain pinned to the newest streaming assistant output while the user is already near the bottom, without overriding intentional user scroll-away.

## Best Possible Solution

Merge #76991 or an equivalent focused Control UI scroll-module fix with regression coverage that ignores programmatic scroll events while still honoring intentional user scroll-away.

## Reproduction Assessment

Yes, at source level. Simulating a programmatic bottom scroll followed by `handleChatScroll` with expanded `scrollHeight` and a distance from bottom over 450 px will flip `chatUserNearBottom` to `false` on current main; I did not run a live browser smoke in this read-only review.

## Solution Assessment

Yes in direction. Distinguishing programmatic scroll events from user scroll-away events inside `ui/src/ui/app-scroll.ts`, with a regression test, is the narrow maintainable fix; the linked PR appears to attempt that exact repair path.
