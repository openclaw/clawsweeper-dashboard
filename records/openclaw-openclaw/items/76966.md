## Summary

Keep open. Current main still has the plausible Control UI scroll bug path, and an open fix PR already references this issue with closing syntax, so the issue should wait for that PR to land or be superseded.

## What This Changes

The issue reports that Control UI/WebChat loses bottom stickiness during streaming assistant output when the user was already near the bottom.

## Best Possible Solution

Review and land or supersede #76991 with focused regression coverage for programmatic scroll events, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: while streaming grows `scrollHeight`, an app-triggered `scrollTo` event with distance from bottom at or above 450 px will make `handleChatScroll` set `chatUserNearBottom` to false. I did not run a live browser stream in this read-only review.

## Solution Assessment

Yes for the current workflow: keeping the issue open while #76991 is reviewed is the narrowest maintainable path. The proposed guard belongs in the Control UI scroll state boundary rather than adding product policy or configuration.
