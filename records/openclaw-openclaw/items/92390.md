## Summary

The current-main bug is source-reproducible and the focused patch matches the generic fallback invariant: a channel that declares no group or channel capability cannot safely mint a group session. The implementation PR should remain open for normal review and merge with its linked issue, but contributor proof is still mock-only because no affected live plugin was exercised after the fix.

## What This Changes

The PR makes outbound fallback routing use a direct session for bare targets on direct-only channel plugins and adds focused direct-only plugin route tests.

## Best Possible Solution

Land the capability-aware fallback with its focused regression coverage after a redacted live direct-only plugin run confirms one canonical session is reused for inbound and outbound traffic.

## Reproduction Assessment

Yes at source level: current main lets an ID-like bare target retain the generic group fallback and then builds a group session key even when the plugin declares only direct chat. The PR's focused fixture reproduces that path, although this review did not run a live WeChat setup.

## Solution Assessment

Yes. Extending the existing capability-aware correction in `inferPeerKind` is a narrow generic fix at the session-key decision boundary, avoids plugin-specific policy in core, and preserves the explicit user/channel and group-capable paths.
