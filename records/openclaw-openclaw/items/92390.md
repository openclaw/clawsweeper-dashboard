## Summary

The PR is a focused and internally consistent fix for the linked shipped WeChat session split: current core already uses channel capabilities to correct the generic `group` result for channel-only plugins, and this extends the same fallback rule to direct-only plugins. Source tracing and focused tests support the patch, but the external-PR merge gate remains open because the supplied evidence does not run the affected real plugin.

## What This Changes

The PR makes outbound fallback session routing classify group-defaulted targets as direct when a channel plugin supports only direct chats, and adds a direct-only test plugin plus two regression cases.

## Best Possible Solution

Keep the capability-based correction in the existing outbound peer-kind fallback, retain the focused regression coverage, and verify that a real WeChat DM reply reuses the inbound direct session without creating a group session.

## Reproduction Assessment

Yes, at source level with high confidence: a direct-only plugin lacking chat-type inference takes the normalized `group` default into fallback session-key construction. This review did not run a live WeChat setup.

## Solution Assessment

Yes. Extending the existing capability-aware `group` correction in `inferPeerKind` is the narrowest consistent fix and avoids embedding WeChat-specific ID rules in core.
