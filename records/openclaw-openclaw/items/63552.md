## Summary

Keep open. Current main still leaves ordinary non-live `models.json` provider discovery broad when users only configure explicit providers, and this PR targets that real path; merge should wait for contributor-supplied real behavior proof and maintainer reconciliation with the broader open proposal at https://github.com/openclaw/openclaw/pull/72352.

## What This Changes

This PR threads explicitly configured models providers into models-config provider discovery so implicit discovery can be limited to owning provider plugins, with fallback and live-filter regression tests.

## Best Possible Solution

Land one reconciled models-config discovery change that scopes non-live discovery from configured provider and model hints, preserves live filters plus `providerDiscoveryProviderIds`, and keeps replace-mode discovery skipping tracked through https://github.com/openclaw/openclaw/issues/66957 and its linked fix path.

## Reproduction Assessment

Yes. A high-confidence source path is `ensureOpenClawModelsJson` with `cfg.models.providers` set and no `providerDiscoveryProviderIds`: current main passes `explicitProviders` through but still returns `undefined` for non-live discovery scoping.

## Solution Assessment

Unclear as the final merge path. The explicit-provider filter is a narrow maintainable slice, but it should be reconciled with current main and the broader configured-model/provider approach in https://github.com/openclaw/openclaw/pull/72352 before choosing what to land.
