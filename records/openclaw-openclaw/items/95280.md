## What This Changes

The issue asks OpenAI-compatible native reasoning streams to seal the thinking block before final answer text so DeepSeek answers do not merge into the reasoning block under `/reasoning on`.

## Best Possible Solution

Review and land or revise https://github.com/openclaw/openclaw/pull/95283 so OpenAI-completions emits one `thinking_end` before text/tool lanes while preserving tag-based reasoning behavior.

Yes, source-level reproduction is high confidence: feed OpenAI-completions chunks with `reasoning_content` followed by `content`, and current main opens reasoning on `thinking_delta` but closes it only on the later end-of-stream `thinking_end`.
