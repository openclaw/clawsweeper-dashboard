Keep open: current main and v2026.6.8 still leave the OpenAI-completions native reasoning boundary unfixed, while open linked PRs now own candidate fixes. This issue should close only after one candidate merges or maintainers reject the fix path.

## What This Changes

The issue reports that DeepSeek/OpenAI-completions native reasoning stays open across the first answer text delta, so `/reasoning on` merges the final answer into the thinking block instead of delivering a separate answer.

## Best Possible Solution

Review and land a single provider-boundary PR that emits one `thinking_end` before text/tool lanes for native reasoning, with regression coverage; then close this issue when merged.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: feed OpenAI-completions chunks with `reasoning_content` followed by `content` and no `thinking_end`, and current main opens reasoning on `thinking_delta` but closes it only at end-of-stream.

## Solution Assessment

Yes. The best fix belongs at the OpenAI-completions provider boundary so every channel sees a sealed reasoning block before the first answer text or tool event, while preserving tag-based reasoning parsing.
