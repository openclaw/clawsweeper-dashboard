## What This Changes

The issue reports that DeepSeek-style OpenAI-completions streams can append the final answer into the reasoning block under `/reasoning on` because no `thinking_end` is emitted at the reasoning-to-content transition.

## Best Possible Solution

Review and land one focused provider-boundary fix that emits exactly one `thinking_end` before the answer/tool lane while preserving tag-based reasoning behavior.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: an OpenAI-completions stream with `reasoning_content` followed by `content` opens the reasoning stream and emits text before the current provider emits `thinking_end`.

## Solution Assessment

Yes. The best fix belongs at the provider boundary because `openai-completions` owns the native reasoning-to-text/tool transition before channels and subscribers consume events.
