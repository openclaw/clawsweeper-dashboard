## Summary

Keep open: this is the right focused PR for the linked auth bug, but the current patch still misses current tokenless node-host configs and duplicates narrower loopback semantics, so merging it would leave the reported credential-write path partially unresolved and could block supported local gateway setups.

## What This Changes

The PR adds a `models auth login` preflight that reads node-host config and rejects non-loopback remote gateway auth writes, adds regression coverage and a changelog entry, and also removes an unused daemon inspect helper.

## Best Possible Solution

Repair this PR by detecting non-loopback persisted gateway config without a token requirement, reusing the shared gateway loopback helper, adding focused block/allow coverage, and dropping unrelated daemon cleanup before merge.

## Reproduction Assessment

## Solution Assessment

No, not as written. A pre-auth remote-node guard is the narrow maintainable fix, but it should key off persisted non-loopback gateway state and shared loopback semantics rather than `nodeHostConfig.token` and a copied helper.
