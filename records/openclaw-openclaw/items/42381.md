## Summary

Keep open: this is the right fix direction for the linked credential-placement bug, but the current PR head still leaves a remote-node auth-write path reachable and duplicates narrower loopback logic than current gateway code uses.

## What This Changes

The PR adds a pre-auth `models auth login` guard for remote node hosts, regression coverage, a changelog entry, and removes an unused daemon inspect helper.

## Best Possible Solution

Merge a repaired pre-auth guard that blocks all non-loopback node-host auth-login local writes before provider auth, treats a missing gateway host as local, reuses shared loopback detection, and keeps focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main writes provider auth profiles through the local `agentDir`, and the PR head still permits a tokenless `node.json` with a non-loopback gateway to bypass the new guard.

## Solution Assessment

No. The pre-auth guard is the right narrow direction, but this implementation should not depend on `nodeHostConfig.token` and should reuse the shared gateway loopback helper instead of introducing a narrower duplicate.
