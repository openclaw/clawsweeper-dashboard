Full review comments:

- **[P1] Block tokenless remote node hosts too:** `src/commands/models/auth.ts:728-729`
  - body: `runNodeHost` saves the remote gateway fields before resolving credentials and does not write the resolved token back to `node.json`, so real node-host configs can be tokenless.   - confidence: 0.9
- **[P1] Use the gateway loopback semantics:** `src/commands/models/auth.ts:719-723`
  - body: This helper only allows `localhost`, `127.0.0.1`, and `::1`, while the gateway code accepts supported loopback forms such as `localhost.`, the 127/8 range, bracketed `::1`, and IPv4-mapped loopback. A local gateway using one of those forms would be treated as remote and blocked from auth login.
  - confidence: 0.88
