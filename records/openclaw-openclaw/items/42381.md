## Summary

Keep open: the PR targets a valid OAuth credential-placement bug, but the branch still misses remote node-host configs without `node.json.token` and can block valid loopback gateway hosts, so it needs a small repair before merge.

## What This Changes

The PR imports node-host config into `models auth login`, adds an early remote-node OAuth guard, and adds one regression test for the blocking path.

## Best Possible Solution

Land a focused guard that blocks local OAuth writes whenever saved node-host gateway host is non-loopback, reuses the shared loopback classifier, covers tokenless remote and loopback allow cases, and records the fix in the changelog.

## Reproduction Assessment

Yes. Source inspection shows current main writes provider OAuth profiles to the resolved local agent store, while node-host config can point at a remote gateway and gateway credentials may come from env/config/password rather than `node.json.token`.

## Solution Assessment

No. Blocking local OAuth writes on remote node hosts is the right narrow mitigation, but this PR should remove the token precondition, reuse the shared loopback helper, and add the required changelog entry.
