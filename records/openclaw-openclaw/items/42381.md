## Summary

Keep open: current main still has the reported local OAuth-write path, and this PR is a useful focused fix candidate, but the current head still misses tokenless remote node-host configs and duplicates a narrower loopback check than the shared gateway helper.

## What This Changes

The PR adds an early `models auth login` guard for node-host configs, mocks node-host config in auth tests, adds a remote-node regression test, and records the fix in the changelog.

## Best Possible Solution

Land a focused version that blocks when `nodeHostConfig.gateway.host` is non-loopback regardless of token, reuses `src/gateway/net.ts`'s loopback classifier, and covers tokenless remote plus loopback allow cases.

## Reproduction Assessment

Yes. Source inspection shows current main writes provider auth profiles to the local `agentDir`, and `runNodeHost` can persist a remote gateway host while credentials come from config or env instead of `node.json.token`.

## Solution Assessment

No. Blocking remote node-host auth writes is the right narrow direction, but the current PR should key off the gateway host without requiring `token` and should reuse the shared loopback classifier.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Remove the token precondition from the remote-node guard:** `src/commands/models/auth.ts:620`
  - body: `NodeHostConfig.token` is optional, and `runNodeHost` saves the gateway target before resolving credentials from config/env/password paths. Returning when the token is absent lets tokenless remote node hosts still perform the local OAuth write this guard is meant to block.
  - confidence: 0.93
- **[P2] Reuse the shared loopback classifier:** `src/commands/models/auth.ts:611-615`
  - body: The new helper only accepts `localhost`, `127.0.0.1`, and `::1`, while `src/gateway/net.ts` already treats other loopback forms such as 127/8, bracketed IPv6, IPv4-mapped loopback, and localhost absolute-form names as local. Valid local node-host configs could be rejected.
  - confidence: 0.88
