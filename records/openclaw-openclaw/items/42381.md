## Summary

Keep open: the PR targets a real source-reproducible auth bug, but the current patch still leaves remote node-host auth writes reachable in tokenless configs and can block valid loopback gateway forms. It also lacks after-fix real behavior proof from a remote node-host plus gateway setup.

## What This Changes

This PR adds a pre-login node-host guard, regression test, and changelog entry to block `openclaw models auth login` before local auth-profile writes on remote node hosts.

## Best Possible Solution

Update this PR or a narrow replacement to guard on non-loopback `nodeHostConfig.gateway.host` regardless of token, reuse the shared gateway loopback classifier, cover tokenless and loopback cases, and require redacted real remote-node proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main resolves a local agent directory and persists provider auth profiles there, while node-host config can point at a remote gateway where model calls happen. I did not run a live remote gateway OAuth flow.

## Solution Assessment

No. Blocking remote node-host auth writes is the right narrow direction, but this implementation should key off the non-loopback gateway host without requiring `token` and should reuse the existing gateway loopback classifier.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Remove the token precondition from the guard:** `src/commands/models/auth.ts:620`
  - body: `NodeHostConfig.token` is optional, and node-host gateway auth can come from environment or config while `nodeHostConfig.gateway.host` still points at a remote gateway. Returning when `token` is absent leaves the local auth-write path reachable for those remote node hosts.
  - confidence: 0.93
- **[P2] Reuse the shared loopback classifier:** `src/commands/models/auth.ts:611-615`
  - body: This helper only treats empty, `localhost`, `127.0.0.1`, and `::1` as local, while gateway code already accepts broader loopback forms like `127/8`, `[::1]`, IPv4-mapped loopback, and `localhost.`. Valid local node-host configs could be blocked as remote.
  - confidence: 0.88
