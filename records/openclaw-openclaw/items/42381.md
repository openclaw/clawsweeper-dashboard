## Summary

Keep open: the PR still targets a valid current-main auth bug, but the current branch has a blocking token-gated guard, narrower-than-shared loopback parsing, missing external real behavior proof, and a conflicting branch state.

## What This Changes

The branch adds a preflight guard and regression test for `openclaw models auth login` on remote node hosts, plus a changelog entry and real-behavior-proof workflow/scripts now identical to current main.

## Best Possible Solution

Land a refreshed guard that blocks non-loopback node-host auth-login writes before local profile persistence, reuses shared loopback semantics, and includes tokenless/loopback coverage plus redacted real setup proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main resolves and persists provider auth profiles in the local agent store while node-host config can target a remote gateway; I did not run a live OAuth flow.

## Solution Assessment

No. Blocking remote node-host auth writes is the right narrow direction, but this branch should not require `nodeHostConfig.token`, should reuse shared loopback parsing, and still needs real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Remove the token precondition from the guard:** `src/commands/models/auth.ts:620`
  - body: `NodeHostConfig.token` is optional, and node hosts can authenticate to the gateway through runtime config or environment while `nodeHostConfig.gateway.host` still records the remote target. Returning when `token` is absent leaves the local auth-write path reachable on remote node hosts.
  - confidence: 0.93
- **[P2] Reuse the shared loopback classifier:** `src/commands/models/auth.ts:611-615`
  - body: The new helper only accepts empty, `localhost`, `127.0.0.1`, and `::1`, but gateway code already treats forms like 127/8, bracketed IPv6, and IPv4-mapped loopback as local. Use the shared semantics, with the node-host default host, so valid local configs are not blocked.
  - confidence: 0.88
