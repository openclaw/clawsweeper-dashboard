## Summary

Keep open: current main still lacks a remote node-host guard for `models auth login`, and the live PR addresses the right bug but still has two blocking defects in the guard logic.

## What This Changes

This PR adds a remote node-host guard to `openclaw models auth login`, regression coverage, and a changelog entry to avoid node-local provider auth profile writes for gateway-backed installs.

## Best Possible Solution

Land or replace this PR with a focused guard that blocks all non-loopback node-host auth-login writes using shared loopback semantics, with tests for tokenless remote configs and valid local loopback forms.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main writes provider auth profiles to the local agent store while saved node-host config can point at a remote gateway; I did not run a live OAuth flow.

## Solution Assessment

No. Blocking remote node-host auth writes is the right narrow fix, but the current PR should not depend on `nodeHostConfig.token` and should reuse the shared gateway loopback helper.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Remove the token precondition from the guard:** `src/commands/models/auth.ts:716`
  - body: `NodeHostConfig.token` is optional, and `runNodeHost` saves the remote gateway target before resolving credentials from runtime config or environment. Returning when the saved token is absent leaves tokenless remote node hosts able to run provider auth and write local profiles, so the reported credential-consumption path remains reachable.
  - confidence: 0.92
- **[P2] Use the shared loopback host check:** `src/commands/models/auth.ts:707-711`
  - body: This duplicate helper only accepts empty, `localhost`, `127.0.0.1`, and `::1`, while `src/gateway/net.ts` already handles local gateway forms such as localhost variants, 127/8, bracketed IPv6, and IPv4-mapped loopback. Reusing the shared helper avoids blocking valid local node-host configs.
  - confidence: 0.86
