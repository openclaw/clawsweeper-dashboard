## Summary

Keep open: the PR addresses a real remote node-host auth bug and now includes terminal proof, but the current head still leaves tokenless remote node-host configs able to write local auth profiles and adds a narrower duplicate loopback check that can reject valid local gateway hosts.

## What This Changes

The PR adds a `models auth login` guard for remote node-host configs, regression coverage, and a changelog entry.

## Best Possible Solution

Land or replace this PR with a focused guard that blocks all non-loopback node-host auth-login writes using shared loopback semantics, with regression tests for tokenless remote configs and valid local loopback forms.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main writes provider auth profiles to the local agent store while saved node-host config can point at a remote gateway; I did not run a live OAuth flow.

## Solution Assessment

No. Blocking remote node-host auth writes is the right narrow fix, but the guard should not depend on `nodeHostConfig.token` and should reuse the shared gateway loopback helper.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Remove the token precondition from the guard:** `src/commands/models/auth.ts:716`
  - body: `NodeHostConfig.token` is optional, and `runNodeHost` saves the remote gateway target before resolving credentials from runtime config or environment. Returning when the saved token is absent leaves tokenless remote node hosts able to run provider auth and write local profiles, so the reported credential-consumption path remains reachable.
  - confidence: 0.92
- **[P2] Use the shared loopback host check:** `src/commands/models/auth.ts:707-711`
  - body: This duplicate helper only accepts empty, `localhost`, `127.0.0.1`, and `::1`, while `src/gateway/net.ts` already accepts the repo's local gateway forms such as localhost variants, 127/8, bracketed IPv6, and IPv4-mapped loopback. Reusing the shared helper avoids blocking valid local node-host configs.
  - confidence: 0.86
