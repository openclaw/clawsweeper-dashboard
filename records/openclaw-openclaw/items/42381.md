## Summary

Keep open: current main still has the node-host/local OAuth write bug, and this PR targets the right surface, but the branch leaves tokenless remote node-host configs unblocked, duplicates a narrower loopback check, and lacks the required changelog entry.

## What This Changes

The branch imports node-host config into `models auth login`, adds an early remote-node OAuth guard, and adds regression coverage for the blocking path.

## Best Possible Solution

Land a focused guard that blocks local OAuth writes whenever `nodeConfig.gateway.host` is non-loopback, reuses the shared gateway loopback helper, covers tokenless remote and loopback allow cases, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main has no node-host check before `modelsAuthLoginCommand` persists provider OAuth profiles to the local `agentDir`, and node-host config can point at a remote gateway even when `token` is absent.

## Solution Assessment

No. Blocking local OAuth writes on remote node hosts is the right narrow mitigation, but this branch should key off `nodeConfig.gateway.host` regardless of `token`, reuse the shared loopback classifier, and add the required changelog entry.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Remove the token precondition from the remote-node guard:** `src/commands/models/auth.ts:544-545`
  - body: `NodeHostConfig.token` is optional, and `runNodeHost` stores the gateway target before resolving credentials from env/config/password paths. Returning when `!nodeConfig?.token` lets remote node hosts without a stored token still perform the local OAuth write this PR is meant to block.
  - confidence: 0.93
- **[P2] Reuse the shared loopback host classifier:** `src/commands/models/auth.ts:537-539`
  - body: The new helper rejects local forms that `src/gateway/net.ts` already accepts, including localhost absolute-form names, 127/8, bracketed IPv6, and IPv4-mapped loopback. Valid local node-host configs could be blocked.
  - confidence: 0.88
- **[P3] Add the required changelog entry:** `CHANGELOG.md:33`
  - body: This changes user-visible `openclaw models auth login` behavior, so repo policy requires a single-line entry under the active Unreleased Fixes section before merge.
  - confidence: 0.95
