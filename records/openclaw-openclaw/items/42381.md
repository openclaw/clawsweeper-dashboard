## Summary

Keep open: the PR targets a real current-main bug, but the proposed guard is still incomplete because tokenless remote node-host configs bypass it and the patch reimplements a narrower loopback check than the shared gateway helper.

## What This Changes

The branch imports node-host config into `models auth login`, adds an early remote-node OAuth guard, and adds regression coverage for the blocking path.

## Best Possible Solution

Land a focused CLI guard for remote node-host OAuth logins that fails before provider auth, uses the shared loopback helper, covers tokenless remote configs, and documents the user-facing fix.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main runs provider OAuth and writes profiles to the local `agentDir` with no node-host config guard, while node-host config can point at a remote gateway without a stored token.

## Solution Assessment

No. Blocking local OAuth writes on remote node hosts is the right narrow direction, but this branch should key off non-loopback `nodeConfig.gateway.host` regardless of `token`, reuse the shared loopback classifier, and add the required changelog entry.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Remove the token precondition from the node-host guard:** `src/commands/models/auth.ts:544-545`
  - body: `NodeHostConfig.token` is optional, and `runNodeHost` stores the gateway target before resolving credentials from env/config/password paths. Returning when `!nodeConfig?.token` lets tokenless remote node hosts still perform the local OAuth write this PR is meant to block.
  - confidence: 0.93
- **[P2] Reuse the shared loopback host classifier:** `src/commands/models/auth.ts:537-539`
  - body: The new helper rejects local forms that `src/gateway/net.ts` already accepts, including localhost absolute-form names, 127/8, bracketed IPv6, and IPv4-mapped loopback. Valid local node-host configs could be blocked.
  - confidence: 0.88
- **[P3] Add the required changelog entry:** `CHANGELOG.md:42`
  - body: This changes user-visible `openclaw models auth login` behavior, so repo policy requires a single-line entry under the active Unreleased Fixes section before merge.
  - confidence: 0.95
