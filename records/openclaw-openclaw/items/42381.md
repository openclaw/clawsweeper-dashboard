## Summary

Keep open: the PR is aimed at a real auth-boundary bug, but the current diff still lets tokenless remote node-host configs proceed into local provider auth writes and reimplements loopback detection with narrower semantics than the shared gateway helper.

## What This Changes

The PR adds a pre-auth `models auth login` remote-node guard, regression coverage, a changelog entry, and an unrelated unused-helper removal in daemon inspect code.

## Best Possible Solution

Repair the guard to block any non-loopback node-host config before provider auth, reuse the shared gateway loopback helper, cover tokenless remote and loopback allow cases, then land with the fail-closed behavior documented in the changelog.

## Reproduction Assessment

Yes. Source inspection shows current main writes provider auth profiles through the local `agentDir`, and the PR diff shows a tokenless non-loopback node-host config would bypass the new guard.

## Solution Assessment

No. The fail-closed guard is the right direction, but the current implementation should key off non-loopback node-host gateway state rather than token presence and should reuse the shared loopback contract.
