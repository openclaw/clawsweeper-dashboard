## Summary

Keep open: the current PR head still leaves a tokenless remote-node path able to proceed into local auth writes, and its local loopback parser is narrower than the existing gateway helper. The direction is good and the supplied terminal proof covers the intended blocking path, but the patch still needs narrow repairs before merge.

## What This Changes

The PR adds a pre-auth remote node-host guard, regression test, changelog entry, and an unused daemon inspect helper cleanup.

## Best Possible Solution

Merge a repaired version that blocks every non-loopback node-host auth-login write before provider auth, treats missing gateway host as local, reuses the shared gateway loopback semantics, and covers tokenless remote plus loopback cases in tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main resolves a local `agentDir` and persists provider auth results there while node-host config can point at a remote gateway; the PR’s remaining tokenless bypass is also source-reproducible from the guard condition.

## Solution Assessment

No. A pre-auth guard is the right narrow solution, but this patch should remove the optional-token precondition, use the shared loopback helper semantics, and add focused coverage for the missed cases.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Block tokenless remote node hosts:** `src/commands/models/auth.ts:728`
  - body: `NodeHostConfig.token` is optional, and `runNodeHost` stores the remote gateway target before resolving credentials from config or environment. With this return, a saved remote gateway config with no stored token still proceeds into provider auth and local profile writes.
  - confidence: 0.92
- **[P2] Reuse the shared loopback host helper:** `src/commands/models/auth.ts:719-723`
  - body: This helper only accepts empty host, `localhost`, `127.0.0.1`, and `::1`. The existing gateway helper accepts localhost dot variants, 127/8, bracketed IPv6, and IPv4-mapped loopback, so local node-host configs using those forms would be blocked incorrectly.
  - confidence: 0.86
