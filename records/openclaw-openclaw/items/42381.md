## Summary

Keep open: this is a focused fix for a source-reproducible auth-write bug, but the current patch still token-gates the remote-node guard, reimplements loopback matching too narrowly, and lacks after-fix real behavior proof.

## What This Changes

The PR adds a `models auth login` preflight guard for remote node-host configs, a regression test, and a changelog entry.

## Best Possible Solution

Update this PR to block any non-loopback node-host auth-login write independent of stored token, reuse shared gateway loopback semantics, add tokenless and loopback coverage, and include redacted real remote node-host/gateway proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main resolves a local agent directory and persists provider auth profiles there while node-host config can point at a remote gateway. I did not run a live remote gateway OAuth flow.

## Solution Assessment

No. Blocking remote node-host auth writes is the right narrow direction, but the guard should key off the non-loopback gateway target without requiring `token` and should reuse the shared loopback classifier.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Remove the token precondition from the guard:** `src/commands/models/auth.ts:620`
  - body: `NodeHostConfig.token` is optional, and node hosts can resolve gateway credentials from runtime config or environment while `nodeHostConfig.gateway.host` still records the remote target. Returning when `token` is absent leaves the local auth-write path reachable on remote node hosts.
  - confidence: 0.93
- **[P2] Reuse the shared loopback classifier:** `src/commands/models/auth.ts:611-615`
  - body: The new helper accepts only empty, `localhost`, `127.0.0.1`, and `::1`, but gateway code already treats forms like `localhost.`, 127/8, bracketed IPv6, and IPv4-mapped loopback as local. Reuse the shared helper, with the node-host default host, so valid local configs are not blocked.
  - confidence: 0.88
