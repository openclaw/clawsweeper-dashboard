## Summary

Keep open: this PR targets a real source-reproducible auth bug and is still the active closing implementation for the linked user report, but the current head still leaves part of the credential-write path open and lacks required real behavior proof. Maintainer follow-up comments are useful evidence that the direction is wanted, but the latest branch is conflicting and not merge-ready.

## What This Changes

The PR adds a preflight guard to `models auth login`, regression coverage, a changelog entry, and a real-behavior-proof workflow/script commit.

## Best Possible Solution

Rebase or replace this branch with a focused guard that blocks all non-loopback node-host auth-login writes using shared loopback semantics, covers tokenless and loopback cases, and includes redacted real remote node-host proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main resolves and writes provider auth profiles to the local agent store while node-host config can point at a remote gateway; I did not run a live remote OAuth flow.

## Solution Assessment

No. Blocking remote node-host auth writes is the right narrow direction, but this implementation should drop the token precondition, reuse the shared loopback helper, and add the missing proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Remove the token precondition from the guard:** `src/commands/models/auth.ts:620`
  - body: `NodeHostConfig.token` is optional, and node-host gateway auth can come from config or environment after the remote gateway target is saved. With this return path, a remote node host without a persisted token can still run provider auth and write local profiles, leaving the reported refresh-token consumption path reachable.
  - confidence: 0.92
- **[P2] Use the shared loopback host check:** `src/commands/models/auth.ts:611-615`
  - body: This helper accepts only empty, `localhost`, `127.0.0.1`, and `::1`, while `src/gateway/net.ts` already handles localhost variants, 127/8, bracketed IPv6, and IPv4-mapped loopback forms. Reusing that helper avoids blocking valid local node-host configs.
  - confidence: 0.87
