## Summary

Keep open: this PR targets a real source-reproducible auth bug, but the current head still leaves tokenless remote node-host configs unblocked, uses a narrower duplicate loopback classifier, and lacks after-fix real behavior proof for the remote node/gateway path.

## What This Changes

This PR adds a `models auth login` remote node-host guard, regression coverage, and a changelog entry for blocking local credential writes on gateway-backed node hosts.

## Best Possible Solution

Land a focused guard that blocks non-loopback `nodeHostConfig.gateway.host` regardless of token, reuses the shared gateway loopback helper, covers tokenless remote and loopback allow cases, and includes real remote-node proof.

## Reproduction Assessment

Yes, as a source-level reproduction rather than a live remote setup. Current main persists `models auth login` profiles to the local agent store, while node-host config records a remote gateway host and gateway credentials can come from env/config without `node.json.token`.

## Solution Assessment

No. Blocking remote node-host auth writes is the right narrow fix, but this implementation should key off the non-loopback gateway host without requiring `token` and should reuse the existing shared loopback classifier.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Remove the token precondition from the remote-node guard:** `src/commands/models/auth.ts:620`
  - body: `NodeHostConfig.token` is optional, and node-host auth can come from env/config/password paths while `nodeHostConfig.gateway.host` still points at a remote gateway. Returning when the token is absent leaves the reported local auth-write path reachable for those remote node hosts.
  - confidence: 0.93
- **[P2] Reuse the shared loopback classifier:** `src/commands/models/auth.ts:611-615`
  - body: This local helper only accepts `localhost`, `127.0.0.1`, and `::1`, while gateway code already treats broader loopback forms such as `127/8`, `[::1]`, IPv4-mapped loopback, and `localhost.` as local. Valid local node-host configs could be blocked as remote.
  - confidence: 0.88
