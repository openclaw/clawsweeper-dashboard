## Summary

Keep open. The bug and mitigation direction are valid, and current main still lacks a remote-node guard for `models auth login`, but this PR is not mergeable as written because it gates the guard on optional node-host token state that normal node-host config does not set, duplicates a narrower loopback classifier, and omits the required user-facing changelog entry.

## What This Changes

The PR imports node-host config into `models auth login`, adds a pre-auth remote node-host guard, and adds a regression test for blocking OAuth login when node config points at a remote gateway.

## Best Possible Solution

Land a rebased fix that detects remote node-host state from `nodeConfig.gateway.host`, blocks the unsafe local OAuth write before provider auth runs, uses the shared gateway loopback classifier, and covers remote-without-token plus loopback allow paths in the models auth tests.

## Reproduction Assessment

Yes. The linked bug gives concrete node-host-to-remote-gateway steps, and source inspection confirms current main writes OAuth profiles to the local `agentDir` while normal node-host config can record a remote gateway without a token.

## Solution Assessment

No. Blocking local OAuth writes on remote node hosts is the right narrow mitigation, but this implementation misses the normal node-host config shape and should reuse the shared loopback helper instead of adding a narrower duplicate.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Do not gate the guard on nodeConfig.token:** `src/commands/models/auth.ts:544-545`
  - body: `runNodeHost` saves the gateway target in node.json but does not require or assign `token`, so a normal remote node host can load config with `gateway.host` and no `token`. This early return lets the OAuth flow run and write credentials locally in the exact scenario this PR is meant to block.
  - confidence: 0.93
- **[P2] Use the shared loopback host classifier:** `src/commands/models/auth.ts:537-539`
  - body: This local helper only accepts `localhost`, `127.0.0.1`, and `::1`, while OpenClaw already has `isLoopbackHost` for supported loopback forms such as `127.0.0.2`, bracketed IPv6, and IPv4-mapped loopback. Reuse the shared helper so local node-host setups are not incorrectly blocked.
  - confidence: 0.88
- **[P3] Add the required changelog entry:** `CHANGELOG.md:25`
  - body: This is a user-facing `fix` to the `openclaw models auth login` command, but the PR only changes the command and test. Add a single-line entry under the active `CHANGELOG.md` Fixes section with contributor credit before merge.
  - confidence: 0.95
