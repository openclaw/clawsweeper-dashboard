## Summary

Keep open: this PR is the active fix path for the linked auth bug, and maintainer follow-up explicitly superseded the separate draft with this contributor branch, but the current patch still leaves tokenless remote node hosts able to write local auth profiles and misclassifies some valid loopback hosts.

## What This Changes

The PR adds a node-host guard to `openclaw models auth login`, a regression test, and a changelog entry for blocking local auth-profile writes when a remote gateway is configured.

## Best Possible Solution

Land a focused guard that blocks non-loopback `nodeHostConfig.gateway.host` regardless of token, reuses the shared gateway loopback helper, covers tokenless and loopback cases, and includes real remote-node proof.

## Reproduction Assessment

Yes, as a source-level reproduction. Current main resolves and writes provider auth profiles to the local agent store, while node-host config can point at a remote gateway whose credentials come from env/config rather than `node.json.token`.

## Solution Assessment

No. Blocking remote node-host auth writes is the right narrow direction, but this implementation should key off the non-loopback gateway host without requiring `token` and should reuse the existing gateway loopback classifier.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Remove the token precondition from the guard:** `src/commands/models/auth.ts:620`
  - body: `NodeHostConfig.token` is optional, and node-host gateway auth can come from env/config while `nodeHostConfig.gateway.host` still points at a remote gateway. Returning when `token` is absent leaves the local auth-write path reachable for those remote node hosts.
  - confidence: 0.93
- **[P2] Reuse the shared loopback classifier:** `src/commands/models/auth.ts:611-615`
  - body: This helper only treats empty, `localhost`, `127.0.0.1`, and `::1` as local, while gateway code already accepts broader loopback forms like `127/8`, `[::1]`, IPv4-mapped loopback, and `localhost.`. Valid local node-host configs could be blocked as remote.
  - confidence: 0.88
