## Summary

Keep open: the PR is pointed at a valid P1 auth-boundary bug, but the latest head still has a P1 bypass for tokenless remote node-host configs and uses narrower loopback detection than the gateway code already supports.

## What This Changes

The PR adds a pre-auth `models auth login` remote node-host guard, regression coverage, a changelog entry, and removes an unused daemon inspect helper.

## Best Possible Solution

Merge a repaired pre-auth guard that blocks all non-loopback node-host auth-login local writes before provider auth, aligns with shared loopback semantics, and keeps focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main writes provider auth profiles through the local `agentDir`, and the PR head's guard returns when `nodeHostConfig.token` is missing even with a non-loopback gateway host.

## Solution Assessment

No, not as written. The direction is right, but the guard should key off non-loopback gateway configuration regardless of token presence and reuse the existing loopback semantics before merge.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Block tokenless remote node hosts:** `src/commands/models/auth.ts:728`
  - body: `nodeHostConfig.token` is optional, so a node config with a remote `gateway.host` but no saved token returns here and then continues into the local auth-profile write path. That leaves the reported credential-placement/security bug reachable for tokenless, password, env-auth, or pairing setups; block based on the non-loopback gateway config instead of token presence.
  - confidence: 0.92
- **[P2] Reuse the shared loopback host semantics:** `src/commands/models/auth.ts:719-723`
  - body: This helper only accepts exact `localhost`, `127.0.0.1`, and `::1`, while the gateway helper already accepts valid loopback forms such as localhost absolute names, bracketed/mapped IPv6, and 127/8 addresses. Using a narrower duplicate can incorrectly block local node-host auth login after upgrade; reuse the shared semantics and cover allow cases.
  - confidence: 0.86
