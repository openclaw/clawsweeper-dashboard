## Summary

Keep open: current main still has the reported node-host auth write bug, and this PR is the right narrow direction, but the latest head still leaves tokenless remote node-host configs unblocked and duplicates a narrower loopback classifier.

## What This Changes

The PR adds a pre-login guard and regression test to stop `models auth login` from writing provider profiles locally on remote node hosts, plus a changelog entry.

## Best Possible Solution

Land a focused patch that blocks non-loopback node-host auth login based on `gateway.host` regardless of token, reuses `src/gateway/net.ts`'s loopback helper, and covers tokenless remote plus local loopback allow cases.

## Reproduction Assessment

Yes. Source inspection shows current main writes provider auth profiles to the local `agentDir`, while `runNodeHost` can persist a remote gateway host before credentials are resolved from config or env, so tokenless remote configs can still hit the reported local-write path.

## Solution Assessment

No. Blocking remote node-host auth writes is the right narrow direction, but the proposed guard should key off the non-loopback gateway host without requiring `token` and should reuse the shared loopback classifier.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Remove the token precondition from the remote-node guard:** `src/commands/models/auth.ts:620`
  - body: `NodeHostConfig.token` is optional, and `runNodeHost` saves the gateway target before resolving credentials from config/env/password paths. Returning when the token is absent lets tokenless remote node hosts still perform the local OAuth write this guard is meant to block.
  - confidence: 0.93
- **[P2] Reuse the shared loopback classifier:** `src/commands/models/auth.ts:611-615`
  - body: The new helper only accepts `localhost`, `127.0.0.1`, and `::1`, while `src/gateway/net.ts` already treats other loopback forms such as 127/8, bracketed IPv6, IPv4-mapped loopback, and localhost absolute-form names as local. Valid local node-host configs could be rejected.
  - confidence: 0.88
