## Summary

Keep open: current main still lacks the remote node-host auth-login guard, but this PR is not merge-ready because the guard token-gates remote configs, duplicates narrower loopback semantics, and still lacks after-fix real behavior proof.

## What This Changes

The branch adds a remote-node-host preflight guard and regression test for `models auth login`, a changelog entry, and proof-gate workflow/scripts that are already identical on current main.

## Best Possible Solution

Land a refreshed guard that blocks any non-loopback node-host auth login before local profile writes, reuses shared loopback host logic, and includes tokenless/loopback tests plus redacted real setup proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main resolves a local agent directory and persists provider auth profiles there while node-host config can point at a remote gateway; I did not run a live OAuth flow.

## Solution Assessment

No. Blocking remote node-host auth writes is the right narrow direction, but the guard should key off the non-loopback gateway target without requiring `token`, reuse shared loopback semantics, and be refreshed against current main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Remove the token precondition from the guard:** `src/commands/models/auth.ts:620`
  - body: `NodeHostConfig.token` is optional, and node hosts can authenticate to the gateway through runtime config or environment while `nodeHostConfig.gateway.host` still records the remote target. Returning when `token` is absent leaves the local auth-write path reachable on remote node hosts.
  - confidence: 0.93
- **[P2] Reuse the shared loopback classifier:** `src/commands/models/auth.ts:611-615`
  - body: The new helper only accepts empty, `localhost`, `127.0.0.1`, and `::1`, but gateway code already treats forms like 127/8, bracketed IPv6, and IPv4-mapped loopback as local. Use the shared semantics, with the node-host default host, so valid local configs are not blocked.
  - confidence: 0.88
