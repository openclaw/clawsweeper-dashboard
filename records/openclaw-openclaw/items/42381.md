## Summary

Keep open: current main still has the reported local auth-profile write path, and this PR is the active closing implementation path for the linked bug, but the current head remains blocked by an incomplete guard, missing real behavior proof, and a dirty/conflicting state.

## What This Changes

The PR adds a remote node-host guard for `openclaw models auth login` with regression coverage and changelog text, and its current head also carries real-behavior-proof workflow scripts.

## Best Possible Solution

Land or replace this PR with a focused guard that blocks all non-loopback node-host auth-login writes using shared loopback semantics, keeps the linked issue open until merge, and includes redacted real remote node-host proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main resolves and writes provider auth profiles to the local agent store while saved node-host config can point at a remote gateway; I did not run a live OAuth flow.

## Solution Assessment

No. Blocking remote node-host auth writes is the right narrow fix, but this implementation should remove the token precondition, use the shared loopback helper, rebase, and add real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Remove the token precondition from the guard:** `src/commands/models/auth.ts:620`
  - body: `NodeHostConfig.token` is optional, and node-host gateway auth can be supplied from runtime config or environment after the remote gateway target is saved. Returning here leaves tokenless remote node hosts able to run provider auth and write local profiles, so the reported credential-consumption path remains reachable.
  - confidence: 0.92
- **[P2] Use the shared loopback host check:** `src/commands/models/auth.ts:611-615`
  - body: This helper only allows empty, `localhost`, `127.0.0.1`, and `::1`, while `src/gateway/net.ts` already accepts localhost variants, 127/8, bracketed IPv6, and IPv4-mapped loopback forms. Reusing the shared helper avoids blocking valid local node-host configs.
  - confidence: 0.86
