## Summary

Keep open: the PR is the right narrow fix direction, but the latest head still leaves tokenless remote node-host configs able to proceed into local auth writes and duplicates weaker loopback parsing than the shared gateway helper.

## What This Changes

The PR adds a pre-auth `models auth login` guard for remote node hosts, regression coverage, a changelog entry, and removes an obsolete daemon inspect helper.

## Best Possible Solution

Merge a repaired pre-auth guard that blocks all non-loopback node-host auth-login local writes before provider auth, treats missing gateway host as local, reuses shared loopback detection, and keeps focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main writes provider auth profiles through the local `agentDir`, and the PR head still permits a tokenless `node.json` with a non-loopback gateway to bypass the new guard.

## Solution Assessment

No. The pre-auth guard is the right narrow direction, but this implementation should not depend on `nodeHostConfig.token` and should reuse the shared gateway loopback helper instead of introducing a narrower duplicate.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Block tokenless remote node hosts:** `src/commands/models/auth.ts:728`
  - body: `NodeHostConfig.token` is optional, and `runNodeHost` persists the remote gateway before credentials are resolved from config or env.   - confidence: 0.86
