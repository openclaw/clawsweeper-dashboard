## Summary

Keep open: current main still lacks a remote-node-host guard for `models auth login`, and this PR targets the right bug but remains incorrect because the guard depends on an optional node token and duplicates a narrower loopback classifier.

## What This Changes

The branch imports node-host config into `models auth login`, adds a pre-provider remote-node guard, and adds regression coverage for the blocking path.

## Best Possible Solution

Land a focused update that blocks remote node-host OAuth writes from `nodeConfig.gateway.host`, reuses the shared gateway loopback helper, covers remote-without-token and loopback allow cases, and adds the required changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a node host with `node.json.gateway.host` set to a non-loopback remote gateway can run `models auth login`, and current main persists the provider result to the local `agentDir`.

## Solution Assessment

No. Blocking local OAuth writes on remote node hosts is the right narrow mitigation, but this PR should not depend on `nodeConfig.token` and should reuse the existing loopback classifier.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Remove the token precondition from the remote-node guard:** `src/commands/models/auth.ts:544-545`
  - body: `runNodeHost` persists the gateway target without requiring `node.json.token`, while credentials can come from env, config, password, or pairing. Returning when `!nodeConfig?.token` lets remote node hosts continue the local OAuth write this PR is meant to block.
  - confidence: 0.93
- **[P2] Reuse the shared loopback host classifier:** `src/commands/models/auth.ts:537-539`
  - body: The new helper only accepts empty host, `localhost`, `127.0.0.1`, and `::1`. `src/gateway/net.ts` already handles supported local forms such as `localhost.`, 127/8, bracketed IPv6, and IPv4-mapped loopback, so the duplicate helper can block valid local node-host setups.
  - confidence: 0.88
- **[P3] Add the required changelog entry:** `CHANGELOG.md:32`
  - body: This changes user-visible `openclaw models auth login` behavior. Add a single-line entry under the active Unreleased Fixes section before merge.
  - confidence: 0.95
