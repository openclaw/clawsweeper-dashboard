## Summary

Keep open. Current main still has the linked remote-node OAuth write bug, and this PR is a useful active implementation candidate, but the patch still misses ordinary remote node-host configs and needs a shared loopback helper plus changelog coverage before merge.

## What This Changes

The branch imports node-host config into `models auth login`, adds a pre-provider remote-node guard, and adds a regression test for the blocking path.

## Best Possible Solution

Land a focused update that blocks remote node-host local OAuth writes based on `nodeConfig.gateway.host`, reuses the shared gateway loopback classifier, covers remote-without-token and loopback allow cases, and adds the required changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main shows `models auth login` resolves and writes a local `agentDir`, while node-host config can record a remote gateway without a token; the linked bug's OAuth failure follows without needing to consume a live refresh token.

## Solution Assessment

No. Blocking local OAuth writes on remote node hosts is the right narrow mitigation, but this implementation should not depend on `nodeConfig.token` and should reuse the existing loopback classifier.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Remove the token precondition from the remote-node guard:** `src/commands/models/auth.ts:544-545`
  - body: `runNodeHost` saves the gateway target in `node.json` without requiring `config.token`, and credentials can come from gateway remote config, env, password, or device pairing. Returning when `!nodeConfig.token` lets a normal remote node host run the OAuth flow and write the profile locally, which is the failure this PR is meant to prevent.
  - confidence: 0.93
- **[P2] Reuse the shared loopback host classifier:** `src/commands/models/auth.ts:537-539`
  - body: The local helper only allows an empty host, `localhost`, `127.0.0.1`, and `::1`. OpenClaw already has `isLoopbackHost` for supported loopback forms such as `localhost.`, 127/8, bracketed IPv6, and IPv4-mapped loopback, so the duplicate helper can incorrectly block valid local node-host setups.
  - confidence: 0.88
- **[P3] Add the required changelog entry:** `CHANGELOG.md:20`
  - body: This changes user-visible `openclaw models auth login` behavior. Add a single-line entry under the active Unreleased Fixes section before merge.
  - confidence: 0.95
