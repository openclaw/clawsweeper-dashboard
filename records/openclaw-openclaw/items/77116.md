## Summary

Keep open because the reporter added concrete logs/config and current main still generates Feishu channel metadata that treats defaulted fields as required; PR #77151 is the active open fix path and should land before this issue closes.

## What This Changes

Existing Feishu/Lark channel configs from 2026.4.x should keep working after upgrading to 2026.5.x without gateway config-validation crash loops.

## Best Possible Solution

Review and land PR #77151 or an equivalent narrow fix that generates channel config JSON schemas in input mode, regenerates bundled metadata, and adds regression coverage for optional-with-default channel fields.

## Reproduction Assessment

Yes, source-reproducible. The reporter's config omits several fields that the Feishu Zod schema defaults, while current generated channel metadata still marks those fields required and the validation path consumes that metadata during startup.

## Solution Assessment

Yes. Passing `io: "input"` in the channel schema generator matches the existing plugin config helper and fixes the root input-vs-output schema mismatch without changing Feishu runtime semantics.
