Keep open because current main and the latest checked Feishu package still expose a strict generated channel schema that can reject upgrade-era Feishu configs, and the issue-linked fix commit is not present on current main.

## What This Changes

The issue reports that upgrading to OpenClaw 2026.5.2 made Feishu/Lark config validation fail and crash-loop the gateway for an existing multi-account setup.

## Best Possible Solution

Land a narrow channel-schema conversion fix that uses Zod input-mode JSON Schema for bundled channel manifests, regenerates metadata, and covers the reported Feishu upgrade config without relaxing strict unknown-key validation globally.

## Reproduction Assessment

Yes. The current generated Feishu schema and the published Feishu package manifests are strict and require defaulted fields that upgrade-era configs omit, giving a high-confidence source reproduction path for config validation failure.

## Solution Assessment

Yes. The best fix is the narrow `toJSONSchema` input-mode conversion plus regenerated metadata and regression tests, not a global `additionalProperties` relaxation or startup-time migration.
