## Summary

Keep open because the updated report is concrete and points to an unresolved packaged-upgrade/plugin-ownership failure: current OpenClaw’s official Feishu schema accepts the reported fields, but validation can use a configured external channel plugin’s `channelConfigs.feishu.schema`, and the reporter is using `@larksuite/openclaw-lark`, not the official `@openclaw/feishu` package.

## What This Changes

The issue reports a Feishu/Lark gateway crash loop after upgrading to OpenClaw 2026.5.2 with an external `@larksuite/openclaw-lark` plugin and a previously working `channels.feishu` multi-account config.

## Best Possible Solution

Decide whether `openclaw-lark` compatibility is owned in core or in that external package, then make the active `channels.feishu` schema accept the documented multi-account Feishu config or provide an explicit migration path.

## Reproduction Assessment

Unclear for a high-confidence current-main reproduction. The reporter supplied a concrete packaged 2026.5.2 log/config, and source shows how an external `channelConfigs.feishu.schema` can reject `channels.feishu`, but this checkout does not include the `@larksuite/openclaw-lark` package schema to replay the exact failure.

## Solution Assessment

Unclear. The narrow fix may be an external `@larksuite/openclaw-lark` schema update, a core compatibility fallback, or a doctor migration, and maintainers should choose that boundary before implementation.
