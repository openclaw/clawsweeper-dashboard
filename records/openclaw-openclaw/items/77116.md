## Summary

Keep open: the reporter supplied concrete Feishu crash-loop logs and config, and current main still omits `io: "input"` when generating channel JSON Schema from Zod, so generated Feishu metadata can still reject/default-require fields that should be valid user input. A branch-side commit directly referenced this issue with the likely narrow fix, but that fix is not present in the current main checkout.

## What This Changes

The issue asks OpenClaw to stop Feishu/Lark gateway crash loops after upgrade when existing `appId`/`appSecret` and policy config fields are present.

## Best Possible Solution

Land a focused schema-generation fix that uses Zod input semantics for channel config JSON Schema, regenerates bundled channel metadata, and adds a regression test proving existing Feishu configs remain valid after upgrade.

## Solution Assessment
