## Summary

Keep open. Current main and the latest release still provide Web Push subscription/test plumbing and inbound message hooks, but no built-in opt-in trigger connects incoming channel messages to browser push notifications; the remaining work is a product/security/API decision, not cleanup closure.

## What This Changes

The issue requests an opt-in Control UI browser push notification trigger for inbound non-webchat channel messages.

## Best Possible Solution

Define one privacy-safe notification contract: either a core-owned opt-in inbound-message Web Push policy with clear payload and channel rules, or a public plugin/Gateway notification API coordinated with the broader Control UI notification trackers.
