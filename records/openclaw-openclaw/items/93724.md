## Summary

Keep open. Current main and v2026.6.9 still have Web Push subscription/test plumbing and inbound message hooks, but no first-class opt-in trigger that sends browser push notifications for incoming channel messages; this needs maintainer product and privacy scoping rather than cleanup closure.

## What This Changes

Request an opt-in Control UI browser push notification policy that automatically sends Web Push notifications when non-webchat channel messages are received.

## Best Possible Solution

Define one privacy-safe notification contract first, then implement an explicit opt-in path that reuses the existing Web Push sender and canonical inbound hook context without changing default channel privacy behavior.
