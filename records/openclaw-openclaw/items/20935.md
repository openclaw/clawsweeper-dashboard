## Summary

Keep open. Current main has a memory host event journal, but it is limited to recall, promotion, and dreaming lifecycle events and does not provide the requested append-only audit log for every `MEMORY.md` / `memory/*.md` mutation with why/session/user context or tamper-evidence. The Pinchy comment is useful prior art, and the maintainer comment already points this toward first-class memory/security design rather than cleanup closure.

## What This Changes

The issue requests an append-only audit log for all agent memory file changes, recording what changed, when, why/trigger, session/user context, and tamper-detection support.

## Best Possible Solution

Design a first-class memory mutation audit layer around the shared write paths for `MEMORY.md` and `memory/*.md`, with a documented schema for timestamp, operation, relative path, content summary/hash, trigger/why, session/run/user context where available, and an explicit tamper-evidence/privacy strategy.
