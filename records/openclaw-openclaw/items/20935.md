## Summary

Keep open: current main and v2026.6.10 still provide lifecycle-oriented memory host events and guarded append-only memory-flush writes, not the requested all-write memory mutation audit with actor, trigger, session/user context, and tamper-evidence semantics. The open Memory Audit PR and prior closed audit PR are useful context, but neither is a merged core fix for this issue.

## What This Changes

The issue requests an append-only audit log for all agent memory-file changes, including what changed, when, why, session/user context, and tamper-detection support.

## Best Possible Solution

Design a maintainer-approved memory mutation audit seam around every OpenClaw-owned durable-memory writer, persist structured records in the appropriate SQLite/plugin-owned state, and document privacy, retention, access, deletion, and tamper-evidence semantics.

## Reproduction Assessment

Not applicable as a user-level bug reproduction: this is a new audit capability request rather than a failure of an existing documented contract. Source inspection gives high-confidence gap evidence because the current event types, memory write paths, docs, and release code are explicit.

## Solution Assessment

No, not as a simple workspace JSONL sidecar or narrow lifecycle-event extension. The maintainable path needs a first-class mutation-audit contract with privacy and tamper-evidence semantics across all durable memory writers.
