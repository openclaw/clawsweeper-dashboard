## Summary

Keep open: current main has a memory lifecycle event journal, but it still does not provide the requested append-only mutation audit for every memory-file change with why/session/user context or tamper-evidence, and the maintainer discussion explicitly kept this as first-class memory/security design work.

## What This Changes

Request an append-only audit log for agent memory-file changes, including what changed, when, why, session/user context, and tamper-detection support.

## Best Possible Solution

Define a versioned memory mutation audit contract around shared MEMORY.md and memory/*.md write surfaces, then document the privacy and integrity semantics publicly.

## Reproduction Assessment

Not applicable. This is a feature request; source inspection shows current main lacks the requested mutation-audit capability rather than failing an existing documented contract.

## Solution Assessment

No. The current lifecycle event journal is useful groundwork, but the maintainable solution needs a first-class mutation audit layer with explicit coverage, privacy, and tamper-evidence decisions.
