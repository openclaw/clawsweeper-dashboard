## Summary

Keep open: current main still has a lifecycle memory event journal and memory-wiki bridge, but not the requested append-only mutation audit for every memory file write with why/session/user context or tamper-evidence. The Pinchy comment is useful prior art, and the closed unmerged memory-audit PR at https://github.com/openclaw/openclaw/pull/30119 reinforces that this needs first-class memory/security design rather than cleanup closure.

## What This Changes

Request an append-only audit log for agent memory file changes, including what changed, when, why, session/user context, and tamper-detection support.

## Best Possible Solution

Define a versioned memory mutation audit contract around every authoritative MEMORY.md and memory/*.md write surface, including privacy/redaction and integrity semantics, then implement it with focused tests and public memory docs.

## Reproduction Assessment

Not applicable. This is a feature request; source inspection shows current main lacks the requested mutation-audit capability rather than failing an existing documented contract.

## Solution Assessment

Unclear. An append-only audit trail is a plausible direction, but schema, redaction, retention, integrity, actor/session mapping, and authoritative write-surface ownership need maintainer security/product design before code work starts.
