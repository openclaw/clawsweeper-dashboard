## Summary

Keep open: current main and v2026.6.5 still expose only a lifecycle memory event journal, not the requested append-only mutation audit log for every memory-file change with why/session/user context and tamper-evidence. ## What This Changes

Request to add an append-only audit log for agent memory-file changes, including what changed, when, why, session/user context, and tamper-detection support.

## Best Possible Solution

Define a versioned memory mutation audit contract around every authoritative MEMORY.md and memory/*.md write surface, with privacy/redaction, retention, actor/session mapping, integrity semantics, focused tests, public docs, and a clear relationship to Memory Audit and durable-state provenance hooks.
