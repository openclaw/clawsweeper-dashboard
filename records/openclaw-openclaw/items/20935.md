## Summary

Keep open: current main and the latest release still provide a lifecycle memory event journal, memory indexing, short-term promotion, and append-only flush safeguards, but not the requested all-mutation audit trail with what/when/why/session/user context or tamper-evidence. Prior discussion also points to Pinchy as useful prior art and explicitly keeps this as memory/security product design work.

## What This Changes

The issue requests an append-only audit log for all agent memory-file mutations with timestamp, reason, session/user context, and tamper-detection support.

## Best Possible Solution

Define a versioned memory mutation audit contract around every authoritative memory-file write surface, with privacy/redaction, retention, actor/session mapping, integrity semantics, tests, and public memory docs.
