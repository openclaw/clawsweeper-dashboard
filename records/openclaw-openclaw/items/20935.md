## Summary

Keep open: current main has a memory host lifecycle event journal and an append-only memory-flush guard, but it still does not provide the requested all-memory-file mutation audit with why/session/user context or tamper-evidence. Prior discussion also treats Pinchy as useful prior art and keeps this as first-class memory/security design work.

## What This Changes

Add an append-only audit log for agent memory-file changes with timestamp, trigger/why, session/user context, and tamper-detection support.

## Best Possible Solution

Define a versioned memory mutation audit contract around all authoritative MEMORY.md and memory/*.md write surfaces, including privacy/redaction and integrity semantics, then implement it with focused tests and public memory docs.
