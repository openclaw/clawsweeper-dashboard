## Summary

Keep open: current main still has a lifecycle memory event journal, not the requested all-memory-file mutation audit with why/session/user context or tamper-evidence. The Pinchy comment and the closed unmerged audit PR are useful prior art, while the broader provenance issue remains related but does not supersede this memory-specific request.

## What This Changes

Request an append-only audit log for changes to MEMORY.md and memory files, including what changed, when, why/trigger, and session/user context.

## Best Possible Solution

Define a versioned memory mutation audit contract around every authoritative MEMORY.md and memory/*.md write surface, including privacy/redaction, retention, and integrity semantics, then implement it with focused tests and public memory docs.
