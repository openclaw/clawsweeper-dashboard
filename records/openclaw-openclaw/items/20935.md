## Summary

Keep open: this remains a first-class memory/security feature gap, not a cleanup close. Current main has lifecycle memory events and public memory artifacts, but it still lacks an append-only mutation audit for every MEMORY.md and memory/*.md write with trigger/session/user context and tamper-evidence; the discussion also records Pinchy as useful prior art and prior maintainer follow-up rather than a shipped core implementation.

## What This Changes

Request to add an append-only audit log for all agent memory-file mutations, including changed content, timestamp, trigger/why, session/user context, and tamper-evidence.

## Best Possible Solution

Define a versioned memory mutation audit contract around every authoritative MEMORY.md and memory/*.md write surface, including privacy/redaction and integrity semantics, then implement it with focused tests and public memory docs.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection shows current main lacks the requested memory mutation-audit capability rather than failing an existing documented contract.

## Solution Assessment

Unclear. An append-only audit trail is a plausible direction, but schema, retention, redaction, integrity, actor/session mapping, and authoritative write-surface ownership need maintainer security/product design before code work starts.
