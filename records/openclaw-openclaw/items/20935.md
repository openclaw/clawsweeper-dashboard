## Summary

Keep open: current main and v2026.5.12 still provide lifecycle memory events, memory public artifacts, and memory index metadata, but not the requested append-only mutation audit for every MEMORY.md and memory/*.md write with why/session/user context and tamper-evidence. ## What This Changes

Request an append-only audit log for agent memory-file changes that records what changed, when, why, session/user context, and tamper-detection support.

## Best Possible Solution

Define a versioned memory mutation audit contract around every authoritative memory-file write surface, with privacy/redaction, retention, actor/session mapping, integrity semantics, tests, and public memory docs.
