## Summary

Keep open: current main still has a memory lifecycle event journal and memory search/index metadata, but not the requested append-only mutation audit for every MEMORY.md and memory/*.md change with why/session/user context and tamper-evidence. Prior discussion cites Pinchy as useful prior art and a closed memory-audit PR, but neither is a shipped OpenClaw implementation.

## What This Changes

Add an append-only audit trail for agent memory-file changes, recording what changed, when, why, session/user context, and tamper-detection support.

## Best Possible Solution

Define a versioned memory mutation audit contract around every authoritative MEMORY.md and memory/*.md write surface, including redaction, retention, actor/session mapping, integrity semantics, focused tests, and public memory docs.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection shows current main lacks the requested mutation-audit capability rather than violating an existing documented contract.

## Solution Assessment

Unclear. An append-only audit trail is a plausible direction, but schema, privacy, retention, tamper-evidence, and authoritative write-surface ownership need maintainer memory/security design first.
