## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Use shared shell-wrapper parsing for deny payloads:** `src/agents/bash-tools.exec.ts:1472-1475`
  - body: The new deny gate extracts an inner payload only through `extractShellWrappedCommandPayload`, whose helper returns null unless the executable is a POSIX shell. A command such as `pwsh -Command "cat ~/.openclaw/secrets/foo.env"` remains a single argv token instead of being tokenized as the inner command, so the exec tool path can still reach a denied path even though the system.run path already has shared cmd/PowerShell wrapper parsing available.
  - confidence: 0.91
- **[P1] Preserve Windows backslashes before deny matching:** `src/infra/exec-deny-path.ts:130-132`
  - body: `tokenizeShellPayload` treats every backslash outside single quotes as an escape, so a Windows path like `C:\Users\alice\.ssh\id_rsa` is turned into `C:Usersalice.sshid_rsa` before candidate selection and glob matching. That breaks the advertised cross-platform deny behavior for common Windows cmd/PowerShell payloads.
  - confidence: 0.88
- **[P2] Require path-like tokens before cwd resolution:** `src/infra/exec-deny-path.ts:236-238`
  - body: When `cwd` is present, the matcher resolves every non-flag token under that cwd, including executable names and plain words. This can deny unrelated commands with confusing `arg="echo"`-style messages whenever the cwd itself falls under a broad denied pattern; keep the intended bare filename cases, but do not resolve arbitrary non-path tokens.
  - confidence: 0.78
