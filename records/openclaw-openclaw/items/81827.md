## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve Windows paths before deny matching:** `src/agents/bash-tools.exec.ts:1461`
  - body: The gateway deny gate parses raw command text with `splitShellArgs()`, which applies POSIX-style backslash escaping before `denyPathPatterns` sees the argv. On a Windows gateway, an unquoted path such as `C:\Users\alice\.ssh\id_rsa` can be transformed before matching, letting configured Windows secret globs miss the path.
  - confidence: 0.9
- **[P1] Scan cmd and PowerShell payloads in the gateway gate:** `src/agents/bash-tools.exec.ts:1472-1475`
  - body: The gateway gate extracts inner payloads through a helper that only recognizes POSIX shells. Denied paths hidden behind `cmd.exe /c` or `pwsh -Command` can remain inside one command-string argument instead of being tokenized and matched, so reuse the existing shared wrapper resolver here.
  - confidence: 0.87
- **[P1] Match shell backslash semantics before denying paths:** `src/infra/exec-deny-path.ts:130-132`
  - body: `tokenizeShellPayload()` treats every backslash as a generic escape and preserves escaped newlines, while POSIX shells join backslash-newline continuations before execution. A denied path split across a continuation can miss the glob even though the shell executes the joined path.
  - confidence: 0.86
- **[P2] Ignore unquoted shell comments during tokenization:** `src/infra/exec-deny-path.ts:149-155`
  - body: The tokenizer never stops at an unquoted `#` that begins a shell comment, so `bash -c 'echo ok # ~/.openclaw/secrets/foo.env'` is denied even though the shell ignores that path. Follow the existing shell argv comment behavior to avoid false denials.
  - confidence: 0.8
- **[P3] Skip executable tokens when resolving bare cwd args:** `src/infra/exec-deny-path.ts:236-242`
  - body: When `cwd` is present, every non-flag token is resolved against it, including the executable name. That can produce confusing false denials such as `arg="echo"` when the process cwd is inside a denied directory; only operand-like bare filenames should get cwd resolution.
  - confidence: 0.74
