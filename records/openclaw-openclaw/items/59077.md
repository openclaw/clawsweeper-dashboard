## Summary

Keep open. Current main and the latest release still ship the reported shared lifecycle reaction defaults, the stall timers still emit the unclear symbols by default, and the discussion has independent user confirmation that the proposed clearer mapping would be easier to understand.

## What This Changes

The issue requests clearer default lifecycle status reaction emoji, especially replacing the soft and hard stall defaults with more readable in-progress and warning indicators.

## Best Possible Solution

Make a small shared channel UX change: align `DEFAULT_EMOJIS` with clearer status symbols, update Telegram-safe fallback variants for any unsupported shared defaults, keep the existing override API and timing behavior unchanged, and refresh focused tests plus any user-facing docs/changelog that describe the defaults.

## Reproduction Assessment

Yes. A high-confidence reproduction path is to create a status reaction controller with no `messages.statusReactions.emojis` override, call a progress state such as `setThinking()`, and advance `DEFAULT_TIMING.stallSoftMs` and `stallHardMs`; current source and tests show it emits `🥱` and `😨`.

## Solution Assessment

Yes. Changing the shared defaults is the narrowest maintainable fix because per-user config overrides only help after someone notices and diagnoses the confusing defaults; the safer implementation is to preserve overrides/timing while adding Telegram-specific fallbacks for unsupported clearer symbols.
