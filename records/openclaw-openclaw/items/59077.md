## Summary

Keep open. Current main and the latest release still ship the reported shared lifecycle reaction defaults, the stall timers still emit the unclear symbols by default, a second user confirmed the confusion, and the older related yawn-emoji issue/PR did not land a fix.

## What This Changes

The issue requests clearer default lifecycle status reaction emoji, especially replacing the soft and hard stall defaults with more readable in-progress and warning indicators while preserving the config override path.

## Best Possible Solution

Make a small shared channel UX change: align `DEFAULT_EMOJIS` with clearer status symbols, update Telegram-safe fallback variants for unsupported shared defaults, keep the existing override API and timing behavior unchanged, and refresh focused tests plus the user-facing changelog.

## Reproduction Assessment

Yes. Create a status reaction controller with no `messages.statusReactions.emojis` override, call a progress state such as `setThinking()`, and advance `DEFAULT_TIMING.stallSoftMs` and `DEFAULT_TIMING.stallHardMs`; current source and tests show it emits `🥱` and `😨`.

## Solution Assessment

Yes. Changing the shared defaults is the narrowest maintainable fix because per-user config overrides only help after someone notices and diagnoses the confusing defaults; the safer implementation is to preserve overrides/timing while adding Telegram-specific fallbacks for unsupported clearer symbols.
