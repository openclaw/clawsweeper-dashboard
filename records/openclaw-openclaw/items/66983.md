## Summary

Keep open. Current main and the latest release still lack a first-class web Canvas/A2UI node platform, and the node-role device-auth bypass requested here remains an unresolved Gateway security/product decision rather than conservative cleanup.

## What This Changes

The issue requests first-class browser tab support as A2UI canvas node surfaces, including explicit web platform command policy and a token-authenticated node pairing path through proxies.

## Best Possible Solution

Define a reviewed Gateway web Canvas-node contract with explicit platform policy, constrained browser-node identity/auth, origin and revocation handling, canvas-only command scope, tests, docs, and coordination with the browser action bridge.

## Reproduction Assessment

Yes, at source level. Current main still has no web platform bucket for Canvas/A2UI and still rejects device-less node-role sessions through the missing-device identity policy; no live browser repro was run for this read-only review.

## Solution Assessment

No, not as a blanket shared-token or `dangerouslyDisableDeviceAuth`-style node bypass. The safer path is a purpose-built browser Canvas-node enrollment design with explicit surface ownership, origin, revocation, and canvas-only semantics.
