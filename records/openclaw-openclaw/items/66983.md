## Summary

Keep open. Current main still has no first-class web Canvas/A2UI node platform and still rejects device-less node-role enrollment, so this remains valid feature and Gateway trust-boundary design work rather than maintainer cleanup.

## What This Changes

Request to support browser tabs as web Canvas/A2UI node surfaces with explicit web platform defaults, proxy-friendly node-role auth/pairing, and documentation.

## Best Possible Solution

Define a reviewed Gateway web Canvas-node contract with explicit platform policy, constrained browser-node identity/auth, origin and revocation handling, canvas-only command scope, regression tests, docs, and coordination with the browser action bridge.

## Reproduction Assessment

Yes, at source level. Current main still has no web platform bucket for Canvas/A2UI and still rejects device-less node-role sessions through reject-device-required.

## Solution Assessment

No, not as a blanket shared-token or dangerouslyDisableDeviceAuth-style node bypass. The safer path is a purpose-built browser Canvas-node enrollment design with explicit surface ownership, origin, revocation, and canvas-only semantics.
