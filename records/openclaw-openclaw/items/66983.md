## Summary

Keep open. Current main still lacks a first-class web Canvas/A2UI node platform and still rejects device-less node-role enrollment, so this remains valid Gateway trust-boundary feature work rather than conservative cleanup.

## What This Changes

Add first-class browser/web Canvas node support so browser tabs can register as A2UI surfaces and receive Canvas A2UI commands through the Gateway.

## Best Possible Solution

Define a reviewed Gateway web Canvas-node contract with explicit platform policy, constrained browser-node identity/auth, origin and revocation handling, canvas-only command scope, tests, docs, and coordination with the browser action bridge.

## Reproduction Assessment

Yes, at source level. Current main still has no web platform bucket for Canvas/A2UI and still rejects device-less node-role sessions through `reject-device-required`.

## Solution Assessment

No, not as a blanket shared-token or `dangerouslyDisableDeviceAuth`-style node bypass. The safer path is a purpose-built browser Canvas-node enrollment design with explicit surface ownership, origin, revocation, and canvas-only semantics.
