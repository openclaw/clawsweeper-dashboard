## Summary

Keep open. Current main and v2026.6.10 still lack a first-class browser Canvas-node enrollment contract; the remaining work changes Gateway node identity, browser-origin trust, and Canvas command authorization rather than being conservative cleanup.

## What This Changes

Request to make browser tabs first-class A2UI Canvas node surfaces, including explicit web platform defaults and a proxy/token-auth enrollment path.

## Best Possible Solution

Define a security-reviewed Gateway web Canvas-node contract with explicit platform policy, browser-node identity/auth, origin and revocation handling, canvas-only command scope, tests, docs, and coordination with browser action ingress.

## Solution Assessment

No, not as the proposed shared-token or dangerouslyDisableDeviceAuth-equivalent bypass. The maintainable path is a purpose-built browser Canvas-node enrollment design with explicit surface ownership, origin, revocation, and canvas-only semantics.
