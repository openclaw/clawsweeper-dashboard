## Summary

Keep open. Current main still lacks a first-class web/browser canvas-node platform and still intentionally rejects device-less node-role browser enrollment, while the recent FSB comment reinforces that this needs scoped surface ownership rather than a blanket auth bypass.

## What This Changes

Request to let browser tabs register as node-role A2UI canvas surfaces by adding explicit web platform policy, browser-node auth/pairing support, and docs.

## Best Possible Solution

Define a reviewed Gateway web canvas-node contract with explicit platform policy, constrained browser-node identity/auth, canvas-only command scope, revocation/origin handling, regression tests, and docs coordinated with the related action bridge.

## Reproduction Assessment

Yes, at source level. A browser-style node with platform web is not a first-class platform and a device-less node-role connect path reaches reject-device-required on current main.

## Solution Assessment

No, not as a blanket dangerouslyDisableDeviceAuth equivalent or shared-token node bypass. The maintainable path is a purpose-built browser canvas-node enrollment design with explicit identity, origin, revocation, trusted-proxy, and canvas-only semantics.
