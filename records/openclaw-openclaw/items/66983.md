## Summary

Keep open. Current main still lacks an explicit web/browser canvas-node contract, and the remaining blocker is node-role browser enrollment/auth rather than a cleanup-close candidate.

## What This Changes

The issue requests first-class browser-tab canvas nodes for A2UI, including explicit web platform command policy, node-role auth/pairing support behind reverse proxies, and docs for the supported pattern.

## Best Possible Solution

Define a reviewed Gateway web canvas-node contract with explicit web platform policy, constrained browser-node enrollment/auth, canvas-only command scope, regression tests for Origin/device identity/pairing behavior, and public docs.

## Reproduction Assessment

Yes, at source level. A role "node" browser client with platform "web", shared auth, and no device identity reaches reject-device-required, and browser-origin node auto-approval is explicitly blocked.

## Solution Assessment

No, not as a blanket dangerouslyDisableDeviceAuth-equivalent or shared-token bypass for node role. The safer path is a purpose-built web node enrollment flow with explicit identity, origin, revocation, and canvas-only semantics.
