## Summary

Keep open. Current main still lacks a first-class web/browser canvas-node platform and still rejects device-less node-role browser enrollment; the new FSB comment is useful trust-model context, but no core implementation has landed.

## What This Changes

The issue requests explicit browser tab support as A2UI canvas node surfaces, including web platform defaults, node-role browser enrollment/auth, and optional docs.

## Best Possible Solution

Define a reviewed Gateway web canvas-node contract with explicit platform policy, constrained browser-node identity/auth, canvas-only command scope, revocation/origin handling, regression tests, and docs coordinated with the browser action bridge.

## Reproduction Assessment

Yes, at source level. A browser-style node with platform web is not a first-class platform and a device-less node-role connection still reaches reject-device-required on current main.

## Solution Assessment

No, not as a blanket dangerouslyDisableDeviceAuth equivalent or shared-token node bypass. The maintainable path is a purpose-built browser canvas-node enrollment design with explicit identity, origin, revocation, trusted-proxy, and canvas-only semantics.
