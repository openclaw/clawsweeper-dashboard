## Summary

Keep open. Current main and v2026.6.8 still do not define a first-class web Canvas/A2UI node platform, and browser-origin node enrollment remains a Gateway security/product decision rather than conservative cleanup.

## What This Changes

The issue requests browser tabs to register as A2UI Canvas node surfaces, including an explicit web platform policy and a node-role auth path for trusted browser deployments.

## Best Possible Solution

Define a security-reviewed Gateway web Canvas-node contract with explicit platform policy, browser-node identity/auth, origin and revocation handling, canvas-only scope, tests, docs, and coordination with browser action ingress.

## Reproduction Assessment

Yes, at source level. Current main lacks a web platform bucket and explicit Canvas plugin default, the plugin platform type excludes web, and node-role browser-style bypasses still fail device identity or pairing policy.

## Solution Assessment

No, not as the proposed shared-token or dangerouslyDisableDeviceAuth-equivalent bypass. The maintainable path is a purpose-built browser Canvas-node enrollment design with explicit surface ownership, origin, revocation, and canvas-only semantics.
