## Summary

Keep open. Current main and the latest release still lack a first-class browser Canvas node contract, and the requested node-role browser enrollment path is a security/product decision rather than conservative cleanup.

## What This Changes

The issue requests first-class web browser tabs as Canvas/A2UI node surfaces, including explicit web platform command policy and browser-origin node pairing/auth support.

## Best Possible Solution

Define a security-reviewed Gateway web Canvas-node contract with explicit platform policy, browser-node identity/auth, origin and revocation handling, canvas-only scope, tests, docs, and coordination with browser action ingress.

## Reproduction Assessment

Yes, at source level. Current main lacks a web platform bucket and explicit Canvas plugin default, and node-role browser-style bypasses still fail device identity or pairing policy.

## Solution Assessment

No, not as the proposed shared-token or dangerouslyDisableDeviceAuth-equivalent bypass. The maintainable path is a purpose-built browser Canvas-node enrollment design with explicit surface ownership, origin, revocation, and canvas-only semantics.
