## Summary

Keep open. Current main still lacks an explicit web/browser canvas-node contract, and source/tests intentionally reject the device-less node-role browser enrollment path this request needs.

## What This Changes

Request to let browser tabs register as web canvas nodes that can receive A2UI canvas commands, with explicit web platform allowlisting and node-role auth/pairing support.

## Best Possible Solution

Define a reviewed Gateway web canvas-node contract with explicit web platform policy, constrained browser-node enrollment/auth, canvas-only command scope, regression tests for Origin/device identity/pairing behavior, and public docs.

## Reproduction Assessment

Yes, at source level. A role "node" browser client with platform "web", shared auth, and no device identity reaches reject-device-required, and browser-origin trusted-CIDR node auto-approval is explicitly blocked.

## Solution Assessment

No, not as a blanket dangerouslyDisableDeviceAuth-equivalent or shared-token bypass for node role. The safer path is a purpose-built web node enrollment flow with explicit identity, origin, revocation, and canvas-only semantics.
