## Summary

Keep open. Current main still has no first-class `web` node platform for Canvas/A2UI and still rejects device-less node-role enrollment, so this remains valid feature and trust-boundary design work rather than cleanup.

## What This Changes

Request support for browser tabs to register as A2UI canvas node surfaces, including explicit `web` platform defaults and a safe node-role auth/pairing path for browser clients behind proxies.

## Best Possible Solution

Define a reviewed Gateway web canvas-node contract with explicit platform policy, constrained browser-node identity/auth, canvas-only command scope, origin and revocation handling, regression tests, and docs coordinated with the browser action bridge.

## Reproduction Assessment

Yes, at source level. A browser-style node with `platform: "web"` is not recognized as a first-class platform, and a device-less node-role session still reaches `reject-device-required`.

## Solution Assessment

No, not as a blanket `dangerouslyDisableDeviceAuth` equivalent or shared-token node bypass. The safer maintainable path is a purpose-built browser canvas-node enrollment design with explicit identity, origin, revocation, trusted-proxy, and canvas-only semantics.
