## Summary

Keep open. Current main still lacks a first-class web canvas-node platform and still rejects device-less node-role enrollment, so this remains active Gateway feature and security design work rather than conservative cleanup.

## What This Changes

Request explicit web/browser canvas-node support so browser tabs can register as A2UI canvas surfaces and receive canvas A2UI commands through the Gateway.

## Best Possible Solution

Define a reviewed Gateway web canvas-node contract with explicit platform policy, constrained browser-node identity/auth, canvas-only command scope, revocation/origin handling, regression tests, and docs coordinated with the browser action bridge.

## Reproduction Assessment

Yes, at source level. A browser-style node with platform web is not recognized as a first-class canvas platform, and a device-less node-role connection still reaches reject-device-required.

## Solution Assessment

No, not as a blanket dangerouslyDisableDeviceAuth equivalent or shared-token node bypass. The safer path is a purpose-built browser canvas-node enrollment design with explicit identity, origin, revocation, trusted-proxy, and canvas-only semantics.
