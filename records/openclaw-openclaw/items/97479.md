## What This Changes

The PR adds runtime-loaded plugin provenance to plugin health snapshots so `/status plugins` can split actually loaded runtime plugins from installed-but-not-active inventory.

## Best Possible Solution

Land this status-only fix after real behavior proof confirms the detailed `/status plugins` output in an actual setup, while preserving compact status and the installed inventory fallback semantics.

## Reproduction Assessment

Yes from source inspection, though I did not run a live repro: current main marks enabled installed inventory as `loaded`, merges it with runtime state, and the detailed formatter counts every merged `status === "loaded"` record under `Loaded:`.

## Solution Assessment

Yes, this is the best fix shape: it adds runtime provenance at the status snapshot boundary and uses the existing live-registry surface set, avoiding both eager loading of all configured plugins and changing installed inventory semantics.
