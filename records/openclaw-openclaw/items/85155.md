## Summary

Keep open: the prompt wording targets the linked provider-substitution bug, but this PR also changes `sessions_spawn` slash-model precedence in a way that can silently reroute existing provider/model overrides. Current `main` and v2026.6.9 still have the reported prompt wording, so this is an active fix candidate rather than a closeable item.

## What This Changes

The PR changes Model Aliases prompt wording and tests, and updates subagent model override resolution so slash-form aliases are looked up before slash-form passthrough.

## Best Possible Solution

Land the prompt wording with a resolver/test contract that keeps explicit provider/model refs from being rewritten; if maintainers intentionally prefer alias-first slash routing, document and prove the provider/auth upgrade impact.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.9 still emit the old Model Aliases line and `sessions_spawn` currently preserves slash-form overrides before alias lookup. I did not establish a deterministic live provider-swap run because final tool selection depends on model behavior after reading the prompt.

## Solution Assessment

No, not as-is: the prompt wording fix is the right narrow layer, but alias-before-passthrough for slash-form values can contradict exact provider/model routing. The best merge shape needs an explicit collision rule and proof that provider/auth routing remains upgrade-safe.
