## Summary

Keep open: the linked provider-substitution bug is still present on current main and the latest release, but this PR is not merge-ready because its slash-form alias resolution can change existing provider/auth routing for explicit-looking model refs.

## What This Changes

The PR changes the Model Aliases system-prompt wording, updates prompt tests, and changes subagent model override alias resolution so exact slash-form aliases resolve before provider/model passthrough.

## Best Possible Solution

Land the prompt wording with runtime resolution that preserves exact configured provider/model refs ahead of slash-form aliases, or explicitly approve and document alias-first routing as a compatibility break.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.8 still emit the old Model Aliases line and adjacent tests expect it. I did not establish a deterministic live provider-swap run because final tool selection depends on model behavior after reading the prompt.

## Solution Assessment

No, not as-is: the prompt wording fix is the right narrow layer, but alias-before-passthrough for slash-form values can contradict exact provider/model routing. The best merge shape needs an explicit collision rule and tests that preserve upgrade-safe exact refs unless maintainers intentionally choose alias-first routing.
