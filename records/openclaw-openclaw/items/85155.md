The PR changes the agent system prompt’s Model Aliases wording and updates the focused prompt test so exact provider/model refs are described as verbatim inputs and aliases as shortcuts for unqualified requests.

## Best Possible Solution

## Reproduction Assessment

Yes for the prompt defect from source: current main and v2026.6.1 emit the ambiguous alias guidance when model aliases are present. The exact downstream model-provider substitution is supported by the linked report and PR proof, but I did not run a live LLM dispatcher scenario.

## Solution Assessment

Yes, with a merge-risk caveat: runtime helpers already preserve qualified provider/model refs, so changing the model-facing guidance is the narrowest layer for this reported substitution. A stronger alternative would be a live proof or a separate runtime guard if maintainers want deterministic enforcement beyond prompt wording.
