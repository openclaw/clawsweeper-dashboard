## What This Changes

The PR changes the agent system prompt's model-alias guidance so explicit provider/model refs are described as verbatim and aliases are described as shortcuts for unqualified model requests.

## Best Possible Solution

Land this focused prompt wording and regression-test update after maintainer review and normal merge gating, then let the linked issue close through the PR.

## Reproduction Assessment

Yes, source-reproducible: current main and v2026.6.1 render the older alias guidance, and the linked report explains how that model-facing text can invite provider substitution before sessions_spawn receives the model field. I did not run a live LLM provider-swap loop because the failure depends on model interpretation.

## Solution Assessment

Yes, this is the best narrow fix: runtime code already preserves qualified provider/model refs, so correcting the prompt guidance matches the existing invariant without adding config, removing aliases, or changing provider routing. Broader alternatives such as suppressing aliases for multi-provider overlaps would need product policy and more compatibility review.
