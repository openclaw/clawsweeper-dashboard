## Summary

Keep this PR open, but it needs a narrow wording repair before merge: the exact-provider/model guidance is the right direction, while the new “unqualified model requests” clause is broader than the current alias contract supports.

## What This Changes

The PR changes the Model Aliases system-prompt sentence and its focused Vitest expectation so exact provider/model overrides are described as verbatim while aliases are described as shortcuts.

## Best Possible Solution

Revise the prompt sentence and test so exact provider/model refs are passed through verbatim while configured aliases remain valid shortcuts when the requester did not already specify a different exact ref.

## Reproduction Assessment

Yes, source-level: current main still emits the old Model Aliases sentence, and the PR body supplies before/after live prompt output showing the changed generated prompt. I did not run the prompt-generation command locally because this review is read-only.

## Solution Assessment

No; the direction is right, but this wording is not the best final shape because it narrows aliases to “unqualified” requests despite current slash-form alias support. The safer fix is a one-line prompt/test adjustment that keeps exact refs verbatim without excluding configured aliases.
