## What This Changes

The PR changes the Model Aliases system-prompt wording and adjacent prompt test so exact provider/model refs are treated as verbatim while aliases are framed as shortcuts for unqualified model requests.

## Best Possible Solution

Land the focused prompt wording change with the existing regression test, then close the linked issue after merge if maintainers agree prompt guidance is the intended boundary for model-chosen substitutions.

## Reproduction Assessment

Yes, at source level: current main renders the old Model Aliases sentence whenever configured aliases are present, and the PR body supplies before/after generated prompt output. I did not run a live provider-swap scenario because the downstream substitution depends on model behavior after reading the prompt.

## Solution Assessment

Yes for this PR scope: changing the model-facing instruction is the narrowest maintainable fix because runtime paths already preserve full provider/model strings once the tool argument is supplied. A runtime rejection policy would be a separate product decision because the tool cannot reliably know whether a model-chosen alternate provider was intentional from context alone.
