Yes. From source, the merge result advertises slash-form aliases in the prompt, while current sessions_spawn model resolution returns any value containing `/` before checking aliases.

## Solution Assessment

No. A prompt-only change is a plausible mitigation for exact provider/model passthrough, but it is not the best fix while it advertises alias behavior that the spawn resolver does not implement.
