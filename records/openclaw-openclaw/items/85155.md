## Summary

Keep open for maintainer review: the patch is a narrow prompt/test fix that aligns the model-facing alias guidance with the existing exact `provider/model` pass-through resolver, and I found no blocking code findings. The remaining concern is auth/provider routing safety because prompt wording can influence model choices but cannot deterministically guarantee every downstream LLM will preserve explicit provider overrides.

## What This Changes

This PR changes one agent system-prompt sentence and its focused test so explicit `provider/model` refs are described as pass-through while aliases are shortcuts for unqualified model requests.

## Best Possible Solution

Merge the narrow prompt/test update if maintainers accept prompt-level mitigation for https://github.com/openclaw/openclaw/issues/61949; track a separate deterministic routing guard only if alias-heavy sessions still swap providers after this lands.

## Reproduction Assessment

Yes for the prompt-level defect: current main and `v2026.6.1` render the ambiguous alias-preference sentence, and the PR body supplies before/after generated prompt output. The downstream provider swap remains model-behavior-dependent rather than a deterministic local repro.

## Solution Assessment

Yes, this is the best narrow fix for the prompt defect because exact `provider/model` preservation already exists in the resolver and the prompt should not invite aliases to override it. A stronger deterministic guard would be a separate follow-up if real sessions still swap providers.
