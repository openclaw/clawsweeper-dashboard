## What This Changes

The branch updates the agent system prompt's Model Aliases wording and its prompt-renderer test so aliases are described as shortcuts rather than preferred replacements for qualified model refs.

## Best Possible Solution

Land the focused prompt wording and test, while leaving alias construction and runtime model selection unchanged unless future evidence shows alias injection itself needs redesign.

## Reproduction Assessment

Yes at the prompt level: current main renders the ambiguous Model Aliases line whenever modelAliasLines are present. The downstream provider swap is model-behavior-dependent, but the prompt defect that drives it is directly source-reproducible and the PR body supplies before/after live prompt output.

## Solution Assessment

Yes; changing the OpenClaw-owned prompt renderer is the narrowest maintainable fix because runtime selection already preserves qualified provider/model refs. Broader options such as suppressing aliases for overlapping providers would be a product change beyond the reported prompt defect.
