## Summary

This PR should stay open for normal maintainer review: the current main prompt still invites aliases for model overrides, while the linked provider-routing report and current docs support preserving explicit `provider/model` refs verbatim. The patch is narrow, in the right prompt-rendering layer, and I found no blocking review finding.

## What This Changes

The PR updates the model-alias section in the agent system prompt and its focused prompt test so aliases are described as shortcuts for unqualified model requests, not replacements for explicit provider/model overrides.

## Best Possible Solution

Land the narrow prompt-layer wording change with the focused system-prompt regression test, keeping exact provider/model routing semantics and alias shortcut support aligned with the model docs.

## Reproduction Assessment

Yes for the prompt-level bug: current main renders the ambiguous alias-preference line whenever model aliases are present in a non-minimal prompt. I did not establish a deterministic live provider-swap reproduction because the downstream LLM choice is stochastic.

## Solution Assessment

Yes. The PR fixes the model-facing guidance at the prompt-rendering owner boundary, preserves existing alias rendering and config behavior, and avoids broader routing or config changes that are not needed for this bug.
