  - body: `generateShrinkwrap()` runs `npm install --package-lock-only` in a fresh temp dir, so `--check` can change when transitive ranges publish new versions even if the commit did not change.   - confidence: 0.87

  - body: Current main is already `2026.5.21`, but this generated root shrinkwrap still declares `2026.5.20`. The new tarball validator requires shrinkwrap and package versions to match, so the branch needs to rebase and regenerate root plus plugin shrinkwraps before it can merge.
  - confidence: 0.9
- **[P2] Include package-local overrides in normalization:** `scripts/generate-npm-shrinkwrap.mjs:190-198`
  - body: `packageJsonForShrinkwrap()` merges package-local overrides into the temporary manifest, but normalization only builds rules from `pnpm-workspace.yaml`. A package-local override hidden by a dependency-owned shrinkwrap can therefore survive in the generated lockfile instead of being enforced like the manifest says.
  - confidence: 0.84
- **[P2] Hydrate bundled deps when manifests are already normalized:** `scripts/lib/plugin-npm-package-manifest.mjs:578-581`
  - body: The early no-change return runs the pack callback before `installPackageLocalBundledDependencies()`. If a publishable package is already normalized and `OPENCLAW_PLUGIN_NPM_BUNDLE_DEPENDENCIES=1`, it can declare bundled dependencies without installing `node_modules`, producing a tarball that omits the runtime dependency files.
  - confidence: 0.86
