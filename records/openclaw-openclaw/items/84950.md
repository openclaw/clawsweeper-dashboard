Full review comments:

- **[P1] Preserve compatible auth overrides on model patches:** `src/gateway/sessions-patch.ts:464-504`
  - body: This branch drops the current-main `preserveAuthProfileOverride` path when `sessions.patch` changes or resets a model. `applyModelOverrideToSessionEntry` deletes `authProfileOverride*` unless that flag is passed, so same-provider switches such as Anthropic to Anthropic now lose the user's selected auth profile and revert the current-main fix from https://github.com/openclaw/openclaw/pull/85014.
  - confidence: 0.96
- **[P1] Generate shrinkwrap from reviewed lock state:** `scripts/generate-npm-shrinkwrap.mjs:235-239`
  - body: `generateShrinkwrap()` runs `npm install --package-lock-only` in a fresh temp directory before writing the checked-in shrinkwrap. That means an unchanged commit can start producing different shrinkwraps as npm registry transitives or npm resolution behavior change, which is risky for a release security gate; derive the output from reviewed stable dependency evidence or get explicit release/secops acceptance for live resolution.
  - confidence: 0.84
- **[P2] Normalize package-local overrides too:** `scripts/generate-npm-shrinkwrap.mjs:190-198`
  - body: `packageJsonForShrinkwrap()` merges package-level overrides with workspace overrides, but `normalizeShrinkwrapOverrides()` only builds exact override rules from `pnpm-workspace.yaml`. A publishable package-local override can therefore be ignored when a dependency-owned shrinkwrap pins a different subtree version; pass the merged overrides for the package being generated and cover that case.
  - confidence: 0.9
