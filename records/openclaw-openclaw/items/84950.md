Full review comments:

- **[P1] Derive shrinkwrap checks from reviewed state:** `scripts/generate-npm-shrinkwrap.mjs:235-239`
  - body: `generateShrinkwrap()` creates a fresh temp package and runs `npm install --package-lock-only`, so `--check` can change or fail when the live registry changes even if the reviewed PR did not. For a publishable lockfile hardening path, the check should be deterministic from reviewed release state or an explicit documented live-resolution policy.
  - confidence: 0.88
- **[P2] Include package overrides during normalization:** `scripts/generate-npm-shrinkwrap.mjs:192`
  - body: `packageJsonForShrinkwrap()` merges package-local overrides into the temp manifest, but `normalizeShrinkwrapOverrides()` only builds override rules from workspace overrides. A package-local override hidden by a dependency-owned shrinkwrap can therefore survive in the generated lockfile despite the package asking for a pin.
  - confidence: 0.83
- **[P2] Install bundled deps before the unchanged return:** `scripts/lib/plugin-npm-package-manifest.mjs:591-601`
  - body: When `OPENCLAW_PLUGIN_NPM_BUNDLE_DEPENDENCIES=1` is set, this early return can skip `installPackageLocalBundledDependencies()` for a package that already has normalized metadata. That can let `npm pack` publish a manifest declaring bundled dependencies while `node_modules` was never hydrated.
  - confidence: 0.86
