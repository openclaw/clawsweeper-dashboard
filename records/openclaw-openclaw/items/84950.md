## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Derive shrinkwrap checks from reviewed state:** `scripts/generate-npm-shrinkwrap.mjs:235-239`
  - body: `--check` regenerates shrinkwrap by running `npm install --package-lock-only` in a fresh temp directory, so the result can change when the registry publishes a newer transitive version even if this repo did not change. That makes release checks drift and weakens the reviewed-lockfile contract this PR is adding.
  - confidence: 0.88
- **[P2] Include package overrides during normalization:** `scripts/generate-npm-shrinkwrap.mjs:192`
  - body: The temporary manifest merges package-local overrides, but the normalization pass only derives override rules from `pnpm-workspace.yaml`. A publishable package with its own override can still emit a shrinkwrap that violates that package's intended pinning when embedded shrinkwraps are involved.
  - confidence: 0.82
- **[P2] Install bundled deps before the unchanged return:** `scripts/lib/plugin-npm-package-manifest.mjs:591-601`
  - body: When bundle mode is enabled and the package/manifest overlay is already unchanged, this early return skips `installPackageLocalBundledDependencies()` entirely. A normalized package can then declare bundled dependencies but pack without the corresponding `node_modules` payload.
  - confidence: 0.86
- **[P2] Record missing root optional bundled dependencies:** `scripts/lib/plugin-npm-package-manifest.mjs:227-229`
  - body: If a bundled dependency is a root optional dependency that npm skipped for the current platform, `readInstalledPackageJson()` returns undefined and the helper never records an install spec. That leaves portable packs missing a package their metadata asked npm to bundle.
  - confidence: 0.78
