## Summary

Keep open: latest head still leaves the same-message retry guard unreachable, so the Telegram duplicate replay reported in #76424 would continue despite the new test passing.

## What This Changes

The PR adds embedded run-loop state intended to turn same-currentMessageId overflow compaction retries into continuation retries, plus a regression test.

## Best Possible Solution

Land a narrow run-loop fix that records the submitted currentMessageId, uses a continuation prompt plus one-shot persistence suppression on same-id compaction retries, and proves the second attempt inputs in the regression test.

## Reproduction Assessment

Yes. #76424 provides concrete Telegram logs with duplicate message_id replay, and current main’s run loop shows non-mid-turn overflow compaction retrying with the original prompt/currentMessageId unless a continuation override is set.

## Solution Assessment

No. The intended direction is narrow and maintainable, but this patch never assigns the state it compares and the regression test would pass without the intended retry behavior.
