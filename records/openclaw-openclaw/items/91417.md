## Summary

## What This Changes

The PR adds optional identity passthrough to the CLI channel outbound runtime, adds Slack outbound text/media identity forwarding, and adds unit coverage for CLI text/media identity handoff.

## Best Possible Solution

Thread normalized per-agent identity through both cron and heartbeat durable sends, reuse one Slack identity-normalization path for all Slack outbound variants, and require live or maintainer-run Slack proof before merge.

Yes, from source: current main drops identity in the CLI send context and Slack direct text/media callbacks, and heartbeat delivery still calls durable send without identity. I did not run a live Slack round-trip.

## Solution Assessment

No, this is only a partial fix. The maintainable fix should cover heartbeat as well as cron and should avoid duplicating Slack identity mapping logic that already validates emoji shortcode shape.
