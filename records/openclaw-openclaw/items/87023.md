## Summary

Keep open: current main and the latest release still lack first-class MCP binary-to-channel relay, and PR #94663 is an open partial foundation rather than a merged solution for the broader hook, channel, transcript, and security design.

## What This Changes

Request first-class relay of MCP image, audio, and resource tool-result content to channel attachments with a documented hook/channel seam and transcript rewrite behavior.

## Best Possible Solution

Define a maintainer-approved MCP media relay contract that stages safe MCP binary content into OpenClaw-owned media, routes through existing channel outbound paths, and explicitly rewrites transcript/model context.

## Reproduction Assessment

Yes. Source inspection on current main shows MCP audio/resources are coerced to text, embedded base64 images are not extracted as sendable media, and MCP-provenance local media paths are filtered unless they are HTTP(S).
