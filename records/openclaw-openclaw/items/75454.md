## Summary

Keep open. Current main still intentionally excludes `github-copilot` from automatic Anthropic 1M promotion, and a narrower explicit-config precedence bug remains in status rendering because a persisted session cap can be passed as `contextTokensOverride` before provider-model config is consulted.

## What This Changes

The issue asks OpenClaw to report/apply the 1M context window for GitHub Copilot Claude Opus 1M model refs, or at least honor explicit `contextWindow`/`contextTokens` config instead of showing a 272k cap.

## Best Possible Solution

Make explicit provider-model `contextWindow` or `contextTokens` configuration authoritative through resolver, `status`, and `session_status` for the selected model, while leaving broad Copilot 1M auto-promotion behind a separate provider-contract decision.

## Reproduction Assessment

Yes. Configure a `github-copilot/claude-opus-4.x-1m` provider model with a 1M context value and render status for a same-model session entry that already persisted `contextTokens: 272000`; current main can pass the persisted value as the resolver override before config lookup.

## Solution Assessment

Yes for the explicit configuration precedence part; unclear for automatic Copilot 1M promotion. The narrow maintainable fix is to honor explicit config over persisted/discovered caps without removing the Anthropic/Claude CLI provider gate.
