## Summary

Keep open. Current main still lets a persisted 272k session context cap override explicit provider-model context configuration in status surfaces, while Copilot-wide 1M auto-promotion remains a separate provider policy question.

## What This Changes

Request that GitHub Copilot Claude Opus `*-1m` models honor a configured 1M context window instead of showing or persisting a 272k cap.

## Best Possible Solution

Make explicit provider-model `contextWindow` or `contextTokens` authoritative through resolver, `status`, and `session_status`; leave Copilot implicit 1M promotion to a separate provider-policy decision.

## Reproduction Assessment

Yes. A deterministic source-level repro is a GitHub Copilot Claude Opus 1M session with persisted `contextTokens: 272000` plus explicit 1M model config; current status paths pass the persisted value as the resolver override before config lookup.

## Solution Assessment

Yes for the explicit configuration precedence repair; unclear for broad Copilot 1M auto-promotion. The narrow maintainable fix is to honor operator config over stale persisted/discovered caps without changing the Anthropic/Claude CLI provider gate.
