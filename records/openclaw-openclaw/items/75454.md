## Summary

Keep this issue open. Current main has improved configured context lookup, but it still intentionally keeps GitHub Copilot outside the Anthropic 1M promotion paths, and the status path can still let a persisted 272k session cap win before configured provider-model context is consulted. The reporter briefly said the filing was mistaken, but then reopened it, so there is no maintainer or reporter terminal close signal.

## What This Changes

The issue asks OpenClaw to report and use a 1M context window for GitHub Copilot Claude Opus 4.x 1M models, either by recognizing 1M model names, honoring explicit configured context values, or allowing explicit `context1m` opt-in for Copilot.

## Best Possible Solution

Make explicit provider-model `contextWindow` and `contextTokens` authoritative for the selected model across model resolution, session persistence, status, and `session_status`. Keep automatic GitHub Copilot 1M name promotion or provider-gate removal as a separate maintainer policy decision after Copilot's long-context contract is verified.

## Reproduction Assessment

Yes for source-level reproduction: configure a Copilot Claude model with a larger provider-model context value, keep a session entry with `contextTokens: 272000`, and the current status path can return the persisted override first. The existing resolver test also reproduces the intentional non-promotion for `github-copilot/claude-opus-4.7`.

## Solution Assessment

Unclear for the broad request to auto-promote GitHub Copilot 1M models, because provider support has not been verified in docs or live tests. Yes for the narrow repair: explicit configured provider-model context should win over discovered or persisted lower caps for that same selected model.
