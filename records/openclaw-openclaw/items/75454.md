## Summary

Keep open. Current main still lets persisted/session context caps override explicit provider-model context configuration in status surfaces, while adjacent open PRs cover Copilot 1M policy but are not merged and do not prove the explicit config-precedence fix on main.

## What This Changes

Honor 1M context windows for GitHub Copilot Claude Opus 4.x 1M models, including explicit `contextWindow`/`contextTokens` configuration and optional 1M promotion behavior.

## Best Possible Solution

Keep this open until either the related Copilot 1M PRs land with exact provider caps and status coverage, or a narrow follow-up makes explicit provider-model context configuration authoritative over stale persisted/session caps.

## Reproduction Assessment

Yes. A source-level repro is a GitHub Copilot Claude Opus 1M session with `entry.contextTokens: 272000` plus config declaring 1M for the same provider/model; current status paths pass the persisted value as `contextTokensOverride` before config lookup. ## Solution Assessment

Yes for the narrow explicit-config precedence repair; unclear for broad Copilot auto-promotion. The maintainable path is to honor operator-configured context caps through resolver/status while letting the open Copilot provider PRs handle known `*-1m` capability policy.
