## Summary

Keep open. Current main still lets a persisted or caller-supplied context cap win before explicit provider-model context configuration, while Copilot-wide 1M auto-promotion remains a provider-policy question rather than a cleanup close.

## What This Changes

The issue asks OpenClaw to report and use the configured 1M context window for GitHub Copilot Claude Opus 1M aliases instead of preserving a 272k cap.

## Best Possible Solution

Make explicit provider-model `contextWindow` or `contextTokens` authoritative in resolver and status surfaces, while leaving implicit GitHub Copilot 1M promotion to a separate maintainer policy decision.

## Reproduction Assessment

Yes. A deterministic source-level reproduction is a GitHub Copilot Claude Opus session with persisted `contextTokens: 272000` plus explicit 1M provider-model config; current status paths pass the persisted value as the resolver override before config lookup.

## Solution Assessment

Yes for the explicit configuration precedence repair; unclear for broad Copilot 1M auto-promotion. The narrow maintainable solution is to honor configured caps over stale persisted/discovered caps without changing the Anthropic/Claude CLI provider gate.
