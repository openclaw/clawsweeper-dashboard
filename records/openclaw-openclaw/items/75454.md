## Summary

Keep open. Current main still allows a persisted 272k session context cap to win over explicit provider-model context configuration in status surfaces, while automatic GitHub Copilot 1M promotion remains a separate provider-policy question.

## What This Changes

The issue asks OpenClaw to report and use 1M context for GitHub Copilot Claude Opus 1M models, either by honoring explicit configured context limits or by extending 1M promotion rules.

## Best Possible Solution

Make explicit provider-model `contextWindow` or `contextTokens` authoritative in resolver, `session_status`, and status summary output, and leave broad Copilot implicit 1M promotion for a separate provider-policy decision.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a GitHub Copilot Claude Opus session with persisted `contextTokens: 272000` plus explicit `models.providers.github-copilot.models[]` context set to 1M; current status paths pass the persisted value as the resolver override before config lookup.

## Solution Assessment

Yes for the explicit config-precedence part; it is the narrowest maintainable fix and avoids changing Copilot-wide 1M promotion policy. The implicit `*-1m` auto-promotion branch should wait for maintainer/provider confirmation.
