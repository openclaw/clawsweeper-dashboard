## Summary

Keep open. Current main still has a source-reproducible precedence bug where persisted or caller-supplied context tokens can override explicit provider-model context configuration in status/session_status paths; the broader Copilot 1M auto-promotion request remains outside the documented Anthropic/Claude CLI policy.

## What This Changes

The issue requests that GitHub Copilot Claude Opus 1M models report/use a 1M context window, or at least that explicit contextWindow/contextTokens config override stale or discovered lower caps.

## Best Possible Solution

Honor explicit provider-model contextWindow/contextTokens across resolver and status surfaces, while keeping Copilot implicit 1M promotion as a separate provider-policy decision.

## Reproduction Assessment

Yes. A high-confidence source reproduction is a GitHub Copilot Claude session with persisted `contextTokens: 272000` plus explicit 1M provider-model config; current status paths pass the persisted value as `contextTokensOverride`, and the resolver returns it before config lookup.

## Solution Assessment

Yes for the explicit configuration precedence repair. The narrow maintainable fix is to let explicit provider-model config beat stale persisted/discovered caps without changing the documented Anthropic/Claude CLI 1M provider gate.
