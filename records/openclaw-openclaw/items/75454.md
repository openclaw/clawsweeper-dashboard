## Summary

Keep this issue open. Current main still limits Anthropic 1M promotion to `anthropic` and `claude-cli`, while the status path can also let a persisted runtime context snapshot such as 272k override an explicit provider-model configuration for the same selected Copilot model.

## What This Changes

The issue requests that GitHub Copilot Claude Opus 1M models report/use a 1M context window, or that explicit provider-model `contextWindow` / `contextTokens` configuration take precedence over discovered Copilot metadata.

## Best Possible Solution

Make explicit provider-model `contextWindow` / `contextTokens` authoritative through model resolution and status/session rendering, including when a session already persisted a smaller discovered runtime cap. Keep automatic GitHub Copilot `*-1m` promotion or provider-gate removal separate until maintainers verify Copilot's long-context contract.

## Reproduction Assessment

Yes for a focused source-level/unit reproduction: configure `github-copilot/claude-opus-4.x-1m` with a larger provider-model context value and use a session entry persisted with 272k `contextTokens`; current status rendering can prefer the persisted value. A live Copilot end-to-end run was not performed in this read-only review.

## Solution Assessment

Unclear for the broad auto-promotion request, but clear for the config precedence path. The narrowest maintainable fix is to make explicit provider-model context config win over discovered or persisted caps for the same selected model; expanding Copilot 1M promotion should wait for maintainer/provider confirmation.
