## Summary

Keep open. Current main partially protects configured provider-model context values from discovery, but status paths can still report a stale persisted 272k session cap before reading explicit config, and broader GitHub Copilot 1M promotion remains a provider policy question.

## What This Changes

Request that GitHub Copilot Claude Opus 1M model refs honor a configured 1M context window or explicit 1M opt-in instead of being reported as 272k.

## Best Possible Solution

Make explicit provider-model `contextWindow` or `contextTokens` authoritative through resolver, detailed status, summary status, and session_status, while leaving Copilot auto-promotion as a separate maintainer decision.

## Reproduction Assessment

Yes. Configure a `github-copilot/claude-opus-4.x-1m` provider model with a 1M context value, keep an existing same-model session with `contextTokens: 272000`, and inspect status/summary status; current source passes that persisted value as the resolver override before config lookup.

## Solution Assessment

Yes for the explicit-configuration precedence part; unclear for broad Copilot 1M auto-promotion. The narrow maintainable fix is to honor explicit config over stale persisted/discovered caps and add focused regression tests without removing the Anthropic/Claude CLI provider gate.
