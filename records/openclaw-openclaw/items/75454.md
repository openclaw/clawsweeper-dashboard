## Summary

Keep open. Current main still leaves `github-copilot` outside automatic Anthropic 1M promotion, and the narrower status/session-status bug remains: a stale persisted cap can be passed as an override before explicit provider-model context config is consulted.

## What This Changes

The issue asks OpenClaw to honor or report 1M context windows for GitHub Copilot Claude Opus `*-1m` models instead of showing a discovered or persisted 272k cap.

## Best Possible Solution

Make explicit provider-model `contextWindow` or `contextTokens` authoritative through resolver, status, and `session_status` for the selected model, while leaving broad Copilot automatic 1M promotion for a separate maintainer decision.

## Reproduction Assessment

Yes. A source-level reproduction is clear: configure `github-copilot/claude-opus-4.x-1m` with a 1M provider-model context value, keep a same-model session entry with `contextTokens: 272000`, and status rendering can still return the persisted override before the config lookup.

## Solution Assessment

Yes for the explicit configuration precedence fix; unclear for automatic Copilot 1M promotion. The narrow maintainable repair is to honor user/provider-model config over stale persisted caps without removing the Anthropic/Claude CLI provider gate.

## Repair Work Prompt

Fix the explicit context-window precedence bug for this issue. Current behavior: status/session_status can display a stale persisted 272k cap for `github-copilot/claude-opus-4.x-1m` even when `models.providers.github-copilot.models[].contextWindow` or `contextTokens` is configured as 1M, because `buildStatusMessage` passes persisted context as `contextTokensOverride` and `resolveContextTokensForModel` returns that before provider-model config. Make explicit provider-model context config authoritative for the selected same-model status/session_status path. Do not remove the existing `github-copilot` exclusion from Anthropic/Claude CLI 1M auto-promotion, and do not add suffix-based `*-1m` auto-promotion unless a maintainer explicitly approves that policy. Add focused regression coverage for resolver/status/session_status with configured `github-copilot` 1M context and a stale persisted 272000 cap. Add a user-facing `CHANGELOG.md` fix entry. Preserve existing fallback-session tests where persisted runtime snapshots intentionally win when the active model differs or is unknown.
