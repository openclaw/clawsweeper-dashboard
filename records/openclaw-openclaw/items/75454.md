## Summary

Keep open. Current main still intentionally limits 1M Claude promotion to `anthropic` and `claude-cli`, and the tests preserve non-promotion for `github-copilot`; the remaining actionable part is the generic explicit-config precedence/status path, not broad Copilot auto-promotion.

## What This Changes

The issue requests that `github-copilot/claude-opus-4.x-1m` models report and use a 1M context window, or at least honor explicit `contextWindow` / `contextTokens` configuration instead of a discovered 272k cap.

## Best Possible Solution

Make explicit provider-model context configuration authoritative across runtime model resolution, context-window guards, session persistence, and status/session_status display. Leave Copilot `*-1m` auto-promotion or ungated `context1m` support as a separate maintainer decision after provider behavior is verified.

## Reproduction Assessment

Partially. There is a high-confidence source-level reproduction for Copilot non-promotion and for the config/discovered-metadata precedence seam; reproducing the exact reported `33k/272k` status line needs the full config and stored session state.

## Solution Assessment

Unclear for the broad provider-policy request. The narrow maintainable first fix is explicit config precedence with focused tests; dropping provider gates or trusting every `*-1m` suffix is broader than current docs and tests support.
