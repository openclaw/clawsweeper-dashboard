## Summary

Keep open. Current main still returns persisted or caller-supplied context token caps before explicit provider-model context configuration in status/session paths, so a stored 272k cap can mask a configured 1M window; Copilot-wide implicit 1M promotion remains a separate provider-policy question.

## What This Changes

Honor 1M context for GitHub Copilot Claude Opus `*-1m` models, especially when explicit provider-model `contextWindow` or `contextTokens` config sets 1M.

## Best Possible Solution

Honor explicit provider-model `contextWindow/contextTokens` across resolver and status surfaces while leaving Copilot implicit 1M auto-promotion to a separate provider-policy follow-up.

## Reproduction Assessment

Yes. A high-confidence source reproduction is a GitHub Copilot Claude session with persisted `contextTokens: 272000` plus explicit 1M provider-model config; current status paths pass the persisted cap as `contextTokensOverride`, and the resolver returns it before config lookup.

## Solution Assessment

Yes for the explicit configuration precedence repair. The narrow maintainable fix is to let explicit provider-model config beat stale persisted/discovered caps without changing the documented Anthropic/Claude CLI 1M provider gate.

## Repair Work Prompt

Fix the explicit context configuration precedence bug for issue https://github.com/openclaw/openclaw/issues/75454. Current main lets persisted/session `contextTokens` such as 272000 override `models.providers.github-copilot.models[].contextWindow/contextTokens` such as 1000000 in `resolveContextTokensForModel`, `session_status`, and status summary paths. Make explicit provider-model config win over stale persisted/discovered caps for the same active provider/model, add focused regression tests for GitHub Copilot Claude Opus with a persisted 272k cap and configured 1M cap, and add a user-facing CHANGELOG.md fix entry. Do not broaden implicit `context1m` or Opus 4.7 auto-promotion to all GitHub Copilot models unless maintainers separately approve that policy change.
