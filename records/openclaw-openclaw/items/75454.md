## Summary

Keep open. Current main still allows a persisted or caller-supplied 272k context cap to win before explicit provider-model context configuration, while broad Copilot 1M auto-promotion remains a separate provider-policy decision.

## What This Changes

The issue asks OpenClaw to honor 1M context for GitHub Copilot Claude Opus 1M model refs, or at least let explicit `contextWindow`/`contextTokens` config override Copilot-discovered 272k metadata.

## Best Possible Solution

Make explicit provider-model `contextWindow` or `contextTokens` authoritative in resolver and status surfaces, and leave Copilot-wide implicit 1M promotion to a separate policy decision.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a GitHub Copilot Claude Opus 1M session with explicit 1M model config plus a persisted or discovered 272k cap; current status paths pass that cap as the resolver override before config lookup.

## Solution Assessment

Yes for the explicit configuration precedence repair; unclear for broad Copilot 1M auto-promotion. The narrow maintainable fix is to honor operator config over stale persisted/discovered caps without expanding the Anthropic/Claude CLI provider gate.
