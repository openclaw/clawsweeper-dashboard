## Summary

Keep open. Current main still lets a persisted or caller-supplied 272k context cap win before explicit provider-model context configuration, while Copilot-wide 1M auto-promotion remains an intentional provider-policy question.

## What This Changes

The issue requests that GitHub Copilot Claude Opus `*-1m` models resolve to a 1M context window instead of a 272k cap, either by name, by explicit `context1m`, or by honoring configured context values over discovered metadata.

## Best Possible Solution

Make explicit provider-model `contextWindow` or `contextTokens` authoritative in resolver and status surfaces; leave Copilot implicit 1M promotion to a separate maintainer policy decision.

## Reproduction Assessment

Yes. A deterministic source-level reproduction is a GitHub Copilot Claude Opus 1M session with persisted `contextTokens: 272000` plus explicit 1M model config; current status paths pass the persisted value as the resolver override before config lookup.

## Solution Assessment

Yes for the explicit configuration precedence repair; unclear for broad Copilot 1M auto-promotion. The narrow maintainable fix is to honor operator config over stale persisted/discovered caps without changing the Anthropic/Claude CLI provider gate.
