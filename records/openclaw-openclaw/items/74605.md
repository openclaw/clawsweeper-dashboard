## Summary

Keep open. Current main still normalizes Codex inbound image `file://` paths with raw `new URL(...).pathname`, while this PR moves that path through the public local-file helper and adds focused coverage for encoded, remote-host, and malformed file URL cases. The title-search related items are adjacent media-normalization work on other surfaces, not a superseding fix for this Codex conversation adapter; the only merge-readiness issue I found is the missing required changelog entry.

## What This Changes

The PR changes the Codex conversation-turn adapter to normalize local image `file://` media paths through `safeFileURLToPath`, fall back to remote media URLs when the local file URL is unusable, and add unit coverage for decoded, remote-host, and malformed file URLs.

## Best Possible Solution

Merge a narrow version of this PR after adding a single `CHANGELOG.md` `Unreleased` / `Fixes` bullet with contributor credit; keep the implementation in the Codex plugin adapter and reuse the existing plugin SDK file-access helper rather than adding a Codex-specific parser.
