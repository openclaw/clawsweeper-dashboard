## Summary

This PR still addresses a real doctor/security mismatch, but it is not ready to merge as-is because the new trusted-proxy proof can still downgrade a runtime-rejected host-interface proxy setup to WARNING. ## What This Changes

The branch updates `src/commands/doctor-security.ts` and its tests so complete, host-scoped `trusted-proxy` Gateway auth counts as authentication proof for exposed-bind doctor diagnostics while several invalid trusted-proxy configurations stay critical.

## Best Possible Solution

Keep the doctor change, but first make its trusted-proxy proof match the runtime fail-closed local-interface checks and then land only with maintainer acceptance of the guarded severity downgrade.

No, not yet. The PR is the right layer and mostly the right shape, but it must also reject host-interface trusted proxy entries so doctor does not claim auth proof for a runtime-rejected trusted-proxy path.
