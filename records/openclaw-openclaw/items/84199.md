## Summary

Keep open for maintainer review and landing: current main and the latest release still emit raw `voice:*` generated Voice Call session keys, while this focused PR moves generated keys to the canonical `agent:<agent>:voice:*` namespace and preserves explicit keys.

## What This Changes

The PR updates Voice Call generated session keys, docs, and focused tests so per-phone and per-call generated keys are agent-scoped while explicit route or outbound request keys pass through unchanged.

## Solution Assessment
