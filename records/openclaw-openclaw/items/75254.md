## Summary

Keep this PR open. The terminal backstop is useful, but the current diff changes the existing WebChat `chat.send` timeout contract by capping omitted and explicit-zero timeouts at 30s, and it is missing the required changelog entry for a user-visible gateway fix.

## What This Changes

The PR adds a WebChat `chat.send` terminal timeout/error backstop, creates missing transcript files before eager user transcript updates, and adds focused gateway regression coverage.

## Best Possible Solution

Keep the terminal-event backstop, but derive it from the existing `chat.send` timeout semantics, preserve explicit zero as no-timeout, cover omitted/zero/positive timeout cases, and add a changelog entry.

## Reproduction Assessment

Yes. A focused regression can run WebChat `chat.send` while dispatch waits for abort with omitted `timeoutMs` and with `timeoutMs: 0`; the PR helper would schedule a 30s terminal timer even though current main treats those as configured/default timeout and no-timeout respectively.

## Solution Assessment

No. The transcript-file guard and terminal backstop direction are reasonable, but the fixed WebChat-only 30s fallback creates a second timeout policy instead of reusing the existing `chat.send` contract.
