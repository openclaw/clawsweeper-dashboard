## Summary

Keep this PR open, but do not merge it as-is. The terminal backstop and missing-transcript guard target a real WebChat/gateway failure mode, but the patch introduces a 30-second WebChat fallback that bypasses the existing `chat.send` timeout contract for normal Control UI sends.

## What This Changes

The PR adds a WebChat-only `chat.send` terminal timeout timer, creates missing transcript files before eager user transcript updates, and adds focused gateway chat regression coverage.

## Best Possible Solution

Keep the terminal backstop, but derive its deadline from the existing `chat.send` timeout calculation or registered abort expiry, preserve omitted and explicit-zero timeout semantics, add omitted/zero regression cases, and include the required changelog entry before landing.

## Reproduction Assessment

Yes. A focused regression can run WebChat `chat.send` with the dispatch mock waiting for abort while omitting `timeoutMs` or passing `timeoutMs: 0`; the PR schedules a 30-second terminal timer for those cases even though current `main` resolves them through the shared gateway timeout path.

## Solution Assessment

No. The transcript-file guard and terminal-event backstop are reasonable, but the fixed 30-second fallback is not the narrowest maintainable solution because it creates a second timeout policy for WebChat instead of reusing the existing `chat.send` contract.
