## Summary

Keep this PR open. The proposed WebChat terminal backstop and transcript-file guard address a plausible gateway finalization gap, but the current patch introduces a WebChat-only 30s fallback for omitted or zero `timeoutMs`, which conflicts with the existing `chat.send` timeout contract and ordinary Control UI sends. It also needs the required user-facing changelog entry before landing.

## What This Changes

The branch adds a WebChat-only `chat.send` terminal timeout backstop, creates missing transcript files before eager user transcript updates, and adds a focused gateway regression test.

## Best Possible Solution

Keep the terminal backstop and transcript-file initialization, but make the terminal deadline come from the existing `chat.send` timeout semantics or registered abort expiry, preserve explicit `timeoutMs: 0`, add omitted/zero timeout regressions, and include a user-facing changelog entry.

## Reproduction Assessment

Yes. A focused regression can run WebChat `chat.send` with the dispatch mock waiting for abort while omitting `timeoutMs` or passing `timeoutMs: 0`; the PR would schedule a 30s terminal timer for those cases even though current main resolves them through the shared timeout contract.

## Solution Assessment

No. The direction is reasonable, but the fixed 30s fallback creates a second WebChat-specific timeout policy instead of reusing the existing `chat.send` contract, so it is not the narrowest maintainable fix.
