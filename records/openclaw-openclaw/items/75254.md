## Summary

Keep this draft PR open but block merge: current main still lacks the WebChat terminal backstop, while the PR’s fixed 30s WebChat fallback would abort ordinary Control UI sends that omit `timeoutMs` and explicit no-timeout sends.

## What This Changes

The PR adds a WebChat-only `chat.send` terminal timeout backstop, creates missing transcript files before eager user transcript updates, and adds focused gateway regression coverage.

## Best Possible Solution

Keep the terminal backstop, but derive its deadline from the existing `chat.send` timeout or abort registration semantics, add omitted and zero timeout regressions, and include the required changelog entry before landing.

## Reproduction Assessment

Yes. A focused regression can run WebChat `chat.send` with dispatch waiting for abort while omitting `timeoutMs` and while passing `timeoutMs: 0`; the PR’s helper would schedule a 30s terminal timer even though current main treats those cases as the long default and no-timeout sentinel.

## Solution Assessment

No. The transcript-file guard and terminal-event backstop are reasonable, but a hard WebChat-only 30s fallback creates a second timeout policy instead of reusing the existing `chat.send` contract.
