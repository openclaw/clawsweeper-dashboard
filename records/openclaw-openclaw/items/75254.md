## Summary

Keep this PR open, but it needs changes before merge. The transcript-file initialization and WebChat terminal-backstop direction are useful, but the PR's 30-second fallback changes current `chat.send` timeout semantics for ordinary Control UI/WebChat sends where `timeoutMs` is omitted, and for explicit `timeoutMs: 0` no-timeout sends. The branch also lacks the required user-facing changelog entry.

## What This Changes

The PR adds a WebChat-only `chat.send` terminal timeout, creates missing transcript files before eager user transcript updates, and adds focused gateway chat regression coverage.

## Best Possible Solution

Keep the terminal backstop, but derive its deadline from the same resolved `chat.send` timeout contract already used by the gateway, preserve explicit `timeoutMs: 0` as no-timeout, add omitted/zero timeout regressions, and add a `CHANGELOG.md` Fixes entry before landing.

## Reproduction Assessment

Yes. A focused regression can run a WebChat `chat.send` whose dispatch never emits a final event while omitting `timeoutMs` or setting `timeoutMs: 0`; the PR schedules a 30-second fallback even though current main resolves those cases through the existing agent timeout/no-timeout contract.

## Solution Assessment

No. The missing-transcript guard and terminal-backstop direction are reasonable, but the fixed 30-second fallback is not the narrowest maintainable fix because it bypasses the gateway's existing `chat.send` timeout semantics.
