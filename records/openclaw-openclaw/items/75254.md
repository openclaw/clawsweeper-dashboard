## Summary

Keep open for human review: the WebChat terminal-backstop idea addresses a real gateway delivery problem, but this draft is not merge-ready because it shortens omitted-timeout WebChat sends to 30 seconds and aborts with a reason that does not match the existing timeout/fallback contract.

## What This Changes

The PR adds a WebChat-only `chat.send` terminal timeout path in gateway chat handling and a focused regression test for runs that do not emit a terminal event.

## Best Possible Solution

Keep a terminal-error backstop only if it preserves the configured run timeout for omitted-timeout WebChat sends, uses the established timeout abort path, and includes focused gateway coverage plus redacted real WebChat proof.

## Reproduction Assessment

Yes for the review findings: current WebChat requests omit `timeoutMs`, current gateway resolves omitted values through `resolveAgentTimeoutMs`, and the PR caps that omitted case to 30 seconds. I did not run a live WebChat scenario in this read-only review.

## Solution Assessment

No. The PR is a plausible mitigation, but the best fix should preserve configured timeout semantics and use the existing timeout abort contract instead of adding a WebChat-local 30s cap with a generic abort reason.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve omitted-timeout WebChat runs:** `src/gateway/server-methods/chat.ts:1660-1663`
  - body: The Control UI WebChat caller does not send `timeoutMs`, so this branch routes normal sends through the fallback case here. Current `chat.send` resolves omitted timeouts with `resolveAgentTimeoutMs`, whose default is 48 hours and can be configured; capping the terminal backstop to 30s means any valid WebChat turn that takes longer than 30 seconds will be aborted and broadcast as an error even though the existing run timeout has not expired. Please use the resolved run timeout for the omitted case, and keep any shorter cap behind an explicit caller/config choice.
  - confidence: 0.93
- **[P2] Use the timeout abort contract:** `src/gateway/server-methods/chat.ts:2155`
  - body: This timeout path aborts the active controller with a generic `Error`, but current model fallback treats caller aborts as terminal only when the abort signal reason is `TimeoutError`, restart, or client disconnect. A generic abort reason can let the fallback chain keep trying candidates after the terminal WebChat timeout instead of stopping at the timed-out run. Route the timeout through the same `stopReason: "timeout"` abort path or create a `TimeoutError` reason before aborting.
  - confidence: 0.86
