## Summary

Keep open: this is a plausible WebChat/Gateway bug-fix PR, but the current patch would change the existing `chat.send` timeout contract and it lacks required real behavior proof.

## What This Changes

The PR adds a WebChat `chat.send` terminal-event timeout backstop, creates missing transcript files before eager user transcript updates, and adds gateway regression coverage.

## Best Possible Solution

Keep the terminal-event backstop idea, but derive it from the existing resolved `chat.send` timeout semantics, preserve explicit zero as no-timeout, and prove the behavior in a real WebChat or Gateway run.

## Reproduction Assessment

Yes. A source-level reproduction is clear: WebChat `chat.send` with omitted `timeoutMs` or explicit `timeoutMs: 0` would schedule the PR's 30s terminal timer even though current main resolves omitted values through config/defaults and treats zero as no-timeout.

## Solution Assessment

No. The transcript-file guard and terminal backstop are reasonable, but the timeout backstop should reuse the existing resolved timeout contract instead of adding a WebChat-only 30s policy.
