## Summary

Keep open: this PR targets a real WebChat terminalization area also tracked by https://github.com/openclaw/openclaw/issues/79803, but the current diff would cap omitted/zero WebChat timeouts at 30s and the external draft still lacks after-fix runtime proof.

## What This Changes

The PR adds a WebChat `chat.send` terminal-event timeout backstop, initializes missing transcript files before eager transcript updates, and adds gateway regression coverage.

## Best Possible Solution

Keep the terminal backstop and transcript-file guard only if the timeout derives from the existing resolved `chat.send` contract, explicit zero remains no-timeout, and a real WebChat or Gateway run proves the terminal error behavior.

## Reproduction Assessment

Yes. A source-level reproduction is clear: on the PR head, WebChat `chat.send` with omitted `timeoutMs` or `timeoutMs: 0` reaches the new 30s fallback, while current main resolves omitted values through config/defaults and treats zero as no-timeout.

## Solution Assessment

No. The transcript-file guard and terminal backstop direction are reasonable, but the timeout backstop should reuse the existing resolved timeout semantics instead of adding a WebChat-only 30s policy.
