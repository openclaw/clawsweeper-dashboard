## Summary

Keep this PR open: the current diff still changes the existing WebChat `chat.send` timeout contract by capping omitted and zero timeouts at 30s, and it is missing the required changelog entry for a user-visible fix.

## What This Changes

The PR adds a WebChat `chat.send` terminal timeout backstop, creates missing transcript files before eager user transcript updates, and adds focused gateway regression coverage.

## Best Possible Solution

Keep the terminal-event backstop, but derive it from the existing `chat.send` timeout contract, preserve explicit zero as no-timeout, cover omitted/zero/positive timeout cases, and add a changelog entry.

## Reproduction Assessment

Yes. A focused gateway regression can mock WebChat dispatch waiting for abort, then call `chat.send` with omitted `timeoutMs` and with `timeoutMs: 0` to show the PR schedules a 30s terminal timer where current main uses default/no-timeout semantics.

## Solution Assessment

No. The transcript-file guard and terminal backstop are reasonable, but the fixed WebChat-only 30s fallback is not the narrowest maintainable solution because it bypasses the existing timeout resolver contract.
