## Summary

Keep this draft PR open, but it is not ready to merge because the proposed WebChat backstop changes established timeout semantics for omitted and zero `timeoutMs`, and the user-facing fix lacks the required changelog entry.

## What This Changes

The PR adds a WebChat `chat.send` terminal timeout backstop, creates missing transcript files before eager user transcript updates, and adds gateway regression coverage.

## Best Possible Solution

Keep the terminal-event backstop, but derive it from the existing `chat.send` timeout contract, preserve explicit zero as no-timeout, cover omitted/zero/positive timeout cases, and add a changelog entry.

## Reproduction Assessment

Yes. A focused regression can run WebChat `chat.send` with dispatch waiting for abort while omitting `timeoutMs` and while passing `timeoutMs: 0`; the PR helper would schedule a 30s terminal timer even though current main treats those as the configured/default timeout and no-timeout sentinel respectively.

## Solution Assessment

No. The transcript-file guard and terminal backstop are reasonable, but the fixed WebChat-only 30s fallback creates a second timeout policy instead of reusing the existing `chat.send` semantics.
