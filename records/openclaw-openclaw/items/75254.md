## Summary

Keep this PR open. Current main still lacks the proposed WebChat terminal backstop, but the PR is draft and the already-posted ClawSweeper review identifies a real timeout-contract regression: the new fallback would cap ordinary omitted-timeout and explicit no-timeout WebChat sends at 30 seconds.

## What This Changes

The PR adds a WebChat-only `chat.send` terminal timeout timer and missing transcript-file initialization in the gateway handler, plus focused gateway chat regression coverage.

## Best Possible Solution

Keep the terminal backstop, but derive its deadline from the same resolved `chat.send` timeout semantics already used by the gateway, preserve explicit `timeoutMs: 0` as no-timeout, add focused omitted/zero timeout regressions, and include the required changelog entry before landing.

## Reproduction Assessment

Yes. A focused regression can mock a WebChat `chat.send` dispatch that never emits a final event while omitting `timeoutMs` or setting `timeoutMs: 0`; the PR helper would schedule the 30-second timeout despite current main resolving those cases to the existing agent/no-timeout contract.

## Solution Assessment

No. The transcript-file guard and terminal backstop direction are reasonable, but the fixed 30-second fallback is not the narrowest maintainable solution because it bypasses the existing gateway timeout contract used by `chat.send`.
