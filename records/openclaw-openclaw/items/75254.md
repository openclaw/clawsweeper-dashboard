## Summary

Keep open: the PR targets a real WebChat finalization gap, but the current patch regresses the existing `chat.send` timeout contract by imposing a 30s WebChat fallback for omitted and zero `timeoutMs`, and it lacks the required changelog entry.

## What This Changes

The PR adds a WebChat-specific `chat.send` terminal timeout backstop, creates missing transcript files before eager user transcript updates, and adds focused gateway regression coverage.

## Best Possible Solution

Keep the terminal backstop, but derive its deadline from the shared `chat.send` timeout/abort semantics, preserve omitted and `timeoutMs: 0` behavior, add regressions for both cases, and include the changelog entry.

## Reproduction Assessment

Yes. A focused regression can make WebChat `chat.send` dispatch wait for abort, then verify omitted `timeoutMs` and `timeoutMs: 0` do not hit a 30s terminal backstop while a small positive timeout still emits the expected terminal error.

## Solution Assessment

No. The transcript-file guard is reasonable, but the fixed WebChat-only fallback duplicates and weakens the existing timeout contract; the safer fix is to reuse the resolved timeout semantics and avoid scheduling a terminal cap for no-timeout cases.
