## Summary

Keep this draft PR open. The WebChat terminal-backstop direction is useful and current main does not already contain it, but the proposed patch should not merge as-is because it caps omitted and explicit zero `chat.send` timeouts at 30 seconds and omits the required changelog entry.

## What This Changes

The PR adds a WebChat-only `chat.send` terminal timeout backstop, initializes missing transcript files before eager user transcript updates, and adds focused gateway chat regression coverage.

## Best Possible Solution

Keep the terminal backstop and transcript-file initialization, but derive the terminal deadline from the same resolved `chat.send` timeout semantics already used by the gateway, preserve explicit `timeoutMs: 0` as no-timeout, cover explicit positive, omitted, and zero timeout cases, and add the required Unreleased changelog entry.

## Reproduction Assessment

Yes. A focused regression can mock a WebChat `chat.send` dispatch that never emits a final event while omitting `timeoutMs` or setting `timeoutMs: 0`; the PR resolver would schedule a 30-second timeout despite current main resolving those cases through the existing default/no-timeout contract.

## Solution Assessment

No. The transcript-file guard and terminal-backstop direction are reasonable, but the fixed 30-second fallback is not the narrowest maintainable fix because it bypasses the established gateway timeout contract used by `chat.send`.
