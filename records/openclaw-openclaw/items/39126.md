## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Resolve gateway auth before trust RPCs:** `src/discord/monitor/trust-command.ts:63-68`
  - body: The Discord command builds a raw `GatewayClient` with only URL and scopes. In token/password or device-auth deployments this sends no usable auth because device identity is not supplied, so `/trust` and `/untrust` fail for authorized owners. Reuse the normal gateway call path or pass resolved credentials/device identity.
  - confidence: 0.92
- **[P1] Gate trust commands on owner IDs only:** `src/discord/monitor/trust-command.ts:35-51`
  - body: This authorizes against the Discord account `allowFrom` list and still permits dangerous name/tag matching. Normal chat allowlists are broader than owner-only admin identity, and name matching can admit impersonation for a command that grants unrestricted exec. Use the owner allowlist/resolver and require explicit stable IDs.
  - confidence: 0.9
- **[P2] Hide expired windows from trust status:** `src/gateway/server-methods/exec-approvals.ts:212-217`
  - body: The status handler returns any cached trust window without checking `expiresAt`. Enforcement ignores expired windows, but clients calling this RPC can still display an active trust state after expiry until another path notices it. Filter expired windows or revoke them before responding.
  - confidence: 0.86
- **[P2] Ignore stale trust responses after agent switches:** `src/tui/tui.ts:766-767`
  - body: `refreshTrustWindow` requests the current agent's status, but writes the result unconditionally when the promise resolves. If the user changes agents while the request is in flight, the old response can populate the footer for the new agent and the 30s throttle delays correction.
  - confidence: 0.83
- **[P2] Count commands that finish after window end:** `src/infra/trust-audit.ts:148-149`
  - body: Audit entries are appended after command completion, but the summary drops entries with `ts > endedAt`. A command started during trust and finishing after expiry or revocation is then omitted from the revocation summary, undercounting commands and failures.
  - confidence: 0.78
