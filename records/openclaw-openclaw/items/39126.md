Overall confidence: 0.9

Full review comments:

- **[P1] Move Discord trust commands into the plugin:** `src/discord/monitor/provider.ts:94`
  - body: Current main removed the src/discord shim tree and builds the active Discord provider from extensions/discord/src. Registering /trust from src/discord resurrects a stale path, so the command will not ship through the current Discord plugin after rebase.
  - confidence: 0.91
- **[P1] Resolve Gateway auth before trust RPCs:** `src/discord/monitor/trust-command.ts:63-68`
  - body: The Discord trust command constructs a GatewayClient directly with URL/scopes instead of using the established bootstrap path that resolves token/password auth, TLS settings, device identity, readiness, and method support. In authenticated deployments, /trust and /untrust can fail or use the wrong operator context.
  - confidence: 0.92
- **[P1] Require ID-only owner checks for trust commands:** `src/discord/monitor/trust-command.ts:48-51`
  - body: The owner check claims this unrestricted exec elevation requires explicit owners, but it still allows dangerous name/tag matching. Mutable names are not safe authorization material for granting temporary full exec access.
  - confidence: 0.9
- **[P2] Hide expired windows from trust status:** `src/gateway/server-methods/exec-approvals.ts:212-218`
  - body: The status RPC returns any cached trust window without checking expiresAt. Enforcement ignores expired windows, but clients can still show an active trust state after expiry until another path clears it.
  - confidence: 0.86
- **[P2] Ignore stale trust responses after agent switches:** `src/tui/tui.ts:766-767`
  - body: refreshTrustWindow writes the result for whichever agent was current when the request started, even if the user switches agents before it resolves. The footer can then show the wrong agent's trust state until the throttle permits another fetch.
  - confidence: 0.83
- **[P2] Count commands that finish after window end:** `src/infra/trust-audit.ts:148-149`
  - body: Audit entries are appended after command completion, but the summary filter drops entries with timestamps after endedAt. Commands started during the trust window but finishing after expiry or revocation are omitted from the summary.
  - confidence: 0.78

## Security Review
