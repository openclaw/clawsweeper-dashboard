The issue reports that the cron run-history Open Chat button in v2026.5.6 opens the wrong historical session for a completed cron job.

Yes, source inspection gives a high-confidence reproduction path even though no live Gateway run was performed. The UI opens entry.sessionKey from cron run history, while current main derives a per-run runSessionKey but persists the session store entry only under agentSessionKey.
