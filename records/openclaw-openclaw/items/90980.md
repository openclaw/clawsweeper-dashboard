## Summary

Keep open: current main and the latest release still leave Docker sandbox control-plane calls unbounded when a Docker child never closes or errors, and the linked fix PR is open rather than merged.

## What This Changes

The issue asks OpenClaw to bound Docker sandbox exec/init calls so a wedged Docker engine cannot leave sandbox initialization and agent runs hanging indefinitely.

## Best Possible Solution

Review and land a refreshed current-main fix, or a narrow replacement for https://github.com/openclaw/openclaw/pull/91015, that bounds Docker control-plane waits, preserves sandbox workspace behavior, documents any config/default surface, and proves wedged-Docker behavior with redacted real evidence.

## Reproduction Assessment

Yes at source level, but not with a fresh live OrbStack repro in this review. A Docker child that never emits `error` or `close` leaves `execDockerRaw` pending, and sandbox init reaches that helper without a timeout or signal.

## Solution Assessment

No final solution is on main yet. The linked PR is a plausible fix direction, but it still needs current-main review/refresh or replacement plus maintainer judgment on the timeout and fail-soft boundary.
