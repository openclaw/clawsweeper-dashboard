## Summary

Keep open: current main and the latest release still have unbounded Docker sandbox initialization calls, and the overlapping timeout PR has not landed or proven the fail-soft behavior this report asks for.

## What This Changes

Bound Docker sandbox initialization when Docker hangs and degrade so non-sandboxed agents remain available instead of the gateway startup wedging.

## Best Possible Solution

Land a bounded sandbox-init timeout and clear fail-soft behavior, preferably by refreshing https://github.com/openclaw/openclaw/pull/66836 or opening a narrower replacement with regression coverage and live wedged-Docker proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence repro path: a Docker child that never exits keeps `execDockerRaw` pending, and sandbox init calls that path without a deadline. I did not run a live OrbStack wedged-engine repro.

## Solution Assessment

Unclear as a complete fix: a timeout/deadline is the right first layer, but the fail-soft behavior and whether to add a user-visible timeout config need maintainer direction.
