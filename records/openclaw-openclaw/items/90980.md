## Summary

Keep open: current main and the latest release still have unbounded Docker sandbox control-plane calls, and the linked closing PR is open, conflicting, and not yet a safe canonical fix. The issue is actionable, but cleanup closure would be premature.

## What This Changes

The issue asks OpenClaw to bound hung Docker sandbox exec/init calls and fail soft so a wedged Docker engine does not take all agents offline.

## Best Possible Solution

Land one canonical current-main fix that bounds Docker control-plane calls, keeps sandboxed failures explicit, leaves non-sandboxed work available, documents operator recovery, and closes this issue from the merged PR.

## Reproduction Assessment

Yes, source-reproducible: current main awaits Docker sandbox init through helpers with no default deadline, so a child process that never emits error or close can keep the promise pending. I did not run the live macOS/OrbStack wedged-engine scenario.

## Solution Assessment

Yes at the problem level: bounding Docker control-plane calls and failing soft is the right direction. The exact timeout/default/config and recovery policy should be settled through the linked PR or a current-main replacement before closing.
