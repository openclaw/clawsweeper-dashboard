## Summary

This PR should stay open because the central runtime fix is valid and the PR body now has sufficient terminal proof, but the current patch still has a blocking test-isolation bug and leaves the model/docs-facing `security` argument inconsistent with the new behavior.

## What This Changes

The PR makes normal exec calls use configured `tools.exec.security` instead of model-supplied `security`, adds regression tests for the policy boundary, and adds a changelog entry.

## Best Possible Solution

Keep configured exec security authoritative, isolate approval state in the new regression tests, and align or remove the model-facing `security` schema/docs wording while coordinating with the broader schema-boundary PR if it lands first.

## Reproduction Assessment

Yes. Source inspection of current main shows `params.security` flows through `normalizeExecSecurity()` and `minSecurity()`, so configured `full` plus model `allowlist` becomes allowlist and can reach the allowlist-miss path; I did not run live commands in this read-only review.

## Solution Assessment

No for the PR as submitted. Using configured security is the narrow runtime fix, but the patch should also isolate test state and update the advertised schema/docs semantics before merge.
