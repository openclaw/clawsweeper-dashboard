## Summary

Keep open: current main still lets model-supplied exec `security` affect the configured policy, and the latest PR diff appears to fix that narrow bug while addressing the prior test-isolation and wording blockers.

## What This Changes

This PR makes normal exec calls use configured exec security, updates exec schema/docs wording and the changelog, and adds isolated regression tests for model-supplied security arguments.

## Best Possible Solution

Land the narrow configured-security fix after normal merge gates, then handle broader model-facing exec policy-field hiding in the separate canonical PR.

## Reproduction Assessment

Yes. Source inspection of current main shows `params.security` flows through `normalizeExecSecurity()` and `minSecurity()`, which can turn configured `full` into allowlist or configured `allowlist` into deny before the gateway/node enforcement path.

## Solution Assessment

Yes. Treating configured security as authoritative is the narrow runtime fix for this bug, while the broader schema split belongs in https://github.com/openclaw/openclaw/pull/76370.
