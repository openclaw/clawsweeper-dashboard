## Summary

Keep open: the runtime fix direction is correct, but the latest PR head still has an unsafe test setup, stale agent-visible security-argument wording, and no after-fix real behavior proof for this external security-sensitive exec change.

## What This Changes

This PR makes configured exec security authoritative for normal exec calls, adds regression tests for model-supplied security arguments, and adds a changelog entry.

## Best Possible Solution

Keep the runtime invariant, isolate approval state in the new tests, align schema/docs for the ignored normal security argument, then add redacted real exec proof and rerun the targeted checks.

## Reproduction Assessment

Yes. Source inspection of current main shows params.security flows through normalizeExecSecurity and minSecurity, so configured full plus model allowlist becomes allowlist and can reach the allowlist-miss path; I did not run live commands because this was a read-only review.

## Solution Assessment

No for the PR as submitted. Ignoring model-supplied security is the narrow runtime fix, but the test harness must not touch real exec-approval state and the advertised argument semantics need to match the new behavior.
