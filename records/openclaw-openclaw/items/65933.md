## Summary

Keep open: the runtime fix direction is sound and the PR body now includes copied terminal proof, but the new tests still run through exec approval state without isolating HOME/state and the model-visible `security` argument remains advertised despite becoming a no-op for normal calls.

## What This Changes

This PR makes normal exec calls use configured `tools.exec.security` regardless of model-supplied `security` args, adds security-floor regression tests, and adds a changelog entry.

## Best Possible Solution

Keep configured exec security authoritative, isolate approval state in the new regression tests, and align the model-facing schema/docs with the ignored normal `security` argument or reconcile the broader schema cleanup with https://github.com/openclaw/openclaw/pull/76370.
