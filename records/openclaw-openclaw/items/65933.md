## What This Changes

The PR changes exec tool policy resolution to ignore model-supplied security arguments in favor of the configured operator policy, preserves explicitly granted elevated-full mode, and adds regression coverage plus a new maxSecurity policy helper test.

## Best Possible Solution

Merge a narrow fix that makes configured exec security authoritative for normal tool calls, preserves the explicitly operator-granted elevated-full override, removes the unused maxSecurity helper and tests, keeps focused full/allowlist/deny regression coverage, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: configured security full plus model-supplied security allowlist is currently reduced through minSecurity to allowlist, and the gateway allowlist-miss branch then denies commands without a matching allowlist or durable approval.

## Solution Assessment

No, not as currently packaged. Ignoring model-supplied security is the safest runtime fix, but the branch should remove the exported maxSecurity helper and add the missing changelog entry before merge.
