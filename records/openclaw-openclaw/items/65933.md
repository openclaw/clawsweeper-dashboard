## Summary

Keep open: the runtime change addresses a real current-main bug, but the new regression test executes the approval path without isolating HOME, which can mutate real exec-approval state and make tests environment-dependent.

## What This Changes

The PR makes configured exec security authoritative for normal exec calls, preserves the elevated-full override, adds security-floor tests, and records the fix in CHANGELOG.

## Best Possible Solution

Keep the runtime fix, isolate exec-approval state in the new tests, align the advertised security argument semantics, then rerun the targeted test and changed gate.

## Reproduction Assessment

Yes. Source inspection of current main shows params.security is normalized and merged through minSecurity, so configured full plus model allowlist becomes allowlist and can reach the allowlist-miss path.

## Solution Assessment

No for the PR as submitted. Ignoring model-supplied security is the right narrow runtime direction, but the added test harness must not read or write real exec-approval state.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Isolate HOME before executing the new exec tests:** `src/agents/bash-tools.exec.security-floor.test.ts:10-11`
  - body: These tests call `tool.execute`, which reaches gateway approval handling and `ensureExecApprovals()`. Because the setup only captures `SHELL`, a local or CI run can create or rewrite `$HOME/.openclaw/exec-approvals.json` and can also fail based on whatever approval policy already exists; set a temp `HOME`/`USERPROFILE` and restore it like the existing exec-approval tests.
  - confidence: 0.88
- **[P3] Align the advertised security argument with the new behavior:** `src/agents/bash-tools.exec.ts:1558`
  - body: After this change, normal exec calls ignore the `security` argument, but the tool schema and docs still advertise it as a per-call enforcement mode. Please either remove/deprecate it from the agent-visible contract where compatible or update the docs/schema text so users and models do not rely on a no-op security override.
  - confidence: 0.72
