The branch makes `createExecTool` use configured exec security regardless of the tool-call `security` argument, keeps explicit elevated-full override behavior, and adds regression coverage for full and allowlist exec-security cases.

## Best Possible Solution

Land a cleaned version that makes configured exec security final for normal exec calls, preserves only operator-granted elevated-full override, removes stale helper/import residue, isolates regression tests from local approval files, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes. The current-main source gives a high-confidence reproduction path: configure exec security as `full`, send a tool call with `security: "allowlist"`, and the `minSecurity` merge makes the effective policy allowlist before host approval enforcement.

## Solution Assessment

Yes, with cleanup. Ignoring model-supplied exec security is the narrowest safe policy because exec security is operator configuration; the safer alternative to the current branch is the same production change without stale imports/helper code, with isolated tests and changelog coverage.
