## Summary

Keep open: the runtime invariant is valid and the PR now has sufficient terminal proof, but the branch still has a merge-blocking test isolation problem plus a schema/docs mismatch for the ignored `security` argument.

## What This Changes

The PR makes normal exec calls use configured exec security instead of model-supplied `security`, adds regression tests, and adds a changelog entry.

## Best Possible Solution

Keep configured exec security authoritative, make the new tests hermetic, and align the model-facing schema/docs so ignored normal `security` overrides are no longer advertised.

## Reproduction Assessment

Yes. Source inspection of current main shows `params.security` flows through `normalizeExecSecurity()` and `minSecurity()`, so configured `full` plus model `allowlist` becomes allowlist and can reach the allowlist-miss path.

## Solution Assessment

No for the PR as submitted. The runtime fix is the narrow maintainable direction, but the tests need isolated home state and the model-visible/doc `security` contract should be updated or reconciled with https://github.com/openclaw/openclaw/pull/76370 before merge.
