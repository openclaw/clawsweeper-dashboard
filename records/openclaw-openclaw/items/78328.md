## Summary

Keep open: the reported onboard dead-end is valid and the fix direction is narrow, but the PR currently bypasses existing disabled channel/plugin guards in one new fallback path and is missing the required user-facing changelog entry.

## What This Changes

This PR updates channel onboarding to reinstall trusted npm-backed external channel plugins from the catalog when stale channel config hides the plugin from normal discovery, and adds focused unit coverage.

## Best Possible Solution

Keep the catalog reinstall fallback, but preserve the existing disabled-config guard before any install/enable action and add the required changelog entry.
