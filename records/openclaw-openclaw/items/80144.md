## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Avoid caching degraded provider-failure results:** `extensions/memory-core/src/memory/manager.ts:494-496`
  - body: `searchWithProvider` can return keyword fallback or `[]` after a retryable provider failure, but the caller still stores that result when `cacheTtlMs` is enabled. A temporary embedding outage can make later identical queries skip the provider and reuse degraded results for the full TTL, so only cache results when the provider path actually succeeded or carry a cacheable/degraded flag back from provider search.
  - confidence: 0.9
- **[P2] Expire old query cache entries before storing more:** `extensions/memory-core/src/memory/manager.ts:375-378`
  - body: Expired entries are removed only when the same key is read or when the whole cache is cleared. A long-running gateway with many unique queries can retain stale result arrays indefinitely, so sweep expired entries or enforce a size bound before adding new cache entries.
  - confidence: 0.88
