## What This Changes

The PR adds a Mattermost native slash command HTTPS callback guard before registration and updates Mattermost docs/tests for derived and explicit HTTP callbacks.

## Best Possible Solution

## Solution Assessment

Yes for the remaining transport gap: the guard belongs in the Mattermost registration path before token-bearing callbacks are registered over cleartext. It still needs maintainer/security acceptance of the compatibility break, a refreshed PR body, and real setup proof before merge.
