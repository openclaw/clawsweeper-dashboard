## Summary

The branch adds a Mattermost monitor-side HTTPS check that skips native slash-command registration for derived or explicit non-HTTPS callback URLs, with docs and monitor tests.

## Best Possible Solution

## Solution Assessment

Yes if maintainers choose HTTPS-only as the Mattermost slash callback policy. If same-host HTTP remains supported, the safer solution is a narrower gate with matching docs and tests.
