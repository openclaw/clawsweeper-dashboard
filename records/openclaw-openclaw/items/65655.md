## Summary

Keep open. The security bug remains real on current main, but this PR as submitted still leaves explicit non-HTTPS callback URLs and an optional trigger-binding path that can bypass the intended hardening; it also overlaps a newer maintainer-labelled Mattermost slash-validation PR, so it should not be closed or auto-repaired yet.

## What This Changes

The PR updates Mattermost slash command docs, tests, and runtime code to reject insecure derived callback URLs and associate registered command tokens with slash triggers before accepting callbacks.

## Best Possible Solution

Keep this PR open while maintainers choose the canonical Mattermost slash-auth fix. The desired end state is one reviewed implementation that enforces the chosen HTTPS callback policy for both derived and explicit URLs, binds callbacks to the registered command identity, and keeps Mattermost docs plus focused regression tests aligned.

## Reproduction Assessment

Yes. The current-main source path is clear: omit commands.callbackUrl and resolveCallbackUrl derives http://, then the callback handler accepts any registered account token for the caller-supplied trigger; the PR findings reproduce by using an explicit http:// callbackUrl or constructing the handler without commandTokenTriggers.

## Solution Assessment

No, not as submitted. The patch is in the right area, but the narrow maintainable fix should make trigger binding required or fail closed when missing and should apply the non-HTTPS callback policy consistently to explicit and derived callback URLs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Reject explicit non-HTTPS callback URLs:** `extensions/mattermost/src/mattermost/monitor-slash.ts:168`
  - body: The new guard only runs when callbackUrl is omitted. If an operator explicitly configures an http:// callback, registration still proceeds and sends reusable Mattermost command tokens over cleartext despite the new HTTPS docs. Apply the same fail-closed or warning policy to the resolved URL regardless of whether it was derived.
  - confidence: 0.9
- **[P2] Make trigger binding required:** `extensions/mattermost/src/mattermost/slash-http.ts:55`
  - body: commandTokenTriggers is optional, so a handler constructed without it silently falls back to token-only validation and accepts a valid token for any trigger. Since this PR relies on token-to-trigger binding for the auth fix, make the map required or fail closed when it is absent.
  - confidence: 0.84
