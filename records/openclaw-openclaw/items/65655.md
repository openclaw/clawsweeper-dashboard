## Summary

Keep open: this PR still contains the unique Mattermost HTTPS callback transport gate that current main and the latest release do not have, but it needs real Mattermost/runtime proof and maintainer acceptance of the HTTPS-only upgrade impact before merge.

## What This Changes

The PR adds an HTTPS-only guard before Mattermost native slash-command registration and updates Mattermost docs plus monitor-slash tests for derived and explicit HTTP callback URLs.

## Best Possible Solution

Merge a narrow Mattermost plugin transport hardening only after maintainer security/compatibility acceptance and redacted real setup proof, while preserving current main's command-specific token validation.

## Solution Assessment

Unclear until maintainers accept the HTTPS-only policy. If that policy is accepted, the PR is a narrow plugin-local fix; otherwise the safer solution is to narrow the gate and docs to the chosen unsafe cases.
