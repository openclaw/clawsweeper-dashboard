## Summary

Keep open. The token replay half is superseded by the merged Mattermost validation refresh, but current main still derives and registers `http://` Mattermost slash callback URLs when `commands.callbackUrl` is omitted, and this security-sensitive PR is stale, incomplete for explicit HTTP callbacks, and lacks real behavior proof.

## What This Changes

The PR changes Mattermost slash-command docs, registration, state, HTTP handler, and tests to reject non-HTTPS derived callbacks and add token-to-trigger binding.

## Best Possible Solution

Rework this into a narrow current-main follow-up that preserves the merged https://github.com/openclaw/openclaw/pull/72923 validation architecture and enforces the chosen Mattermost callback URL transport policy with focused docs and tests.

## Reproduction Assessment

Yes. A source-level current-main path remains: enable Mattermost native slash commands without `channels.mattermost.commands.callbackUrl`, and `resolveCallbackUrl` derives an `http://` URL that registration still uses after only a warning.

## Solution Assessment

No, not as submitted. The branch addresses the right surface, but it is stale against the merged command-validation architecture and its HTTPS guard does not cover explicit non-HTTPS callback URLs.

  - confidence: 0.9
