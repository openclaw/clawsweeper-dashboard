## Summary

Keep open. The PR still targets a real Mattermost security issue, but current main has already superseded the token-replay half, the proposed callback transport guard is incomplete, and the external PR lacks real behavior proof.

## What This Changes

The PR updates Mattermost slash-command docs, registration, state, HTTP handler, and tests to reject insecure derived callbacks and bind registered tokens to slash triggers.

## Best Possible Solution

Rework or replace this branch with a narrow Mattermost-plugin-local fix on current main that enforces the chosen non-HTTPS callback policy before registration, preserves PR 72923 validation, and updates focused docs/tests.

## Reproduction Assessment

Yes. Source inspection on current main shows omitted Mattermost slash `callbackUrl` still derives `http://...`, and the monitor passes it to registration after only a warning; I did not run a live Mattermost roundtrip in this read-only review.

## Solution Assessment

No, not as submitted. The branch addresses the right surface, but its token-binding approach is superseded by stronger current-main validation and its transport guard misses explicit `http://` callback URLs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Reject explicit non-HTTPS callback URLs:** `extensions/mattermost/src/mattermost/monitor-slash.ts:168-174`
  - body: The new guard only runs when `callbackUrl` is omitted. If an operator explicitly configures an `http://` callback, registration still proceeds over cleartext despite the PR docs and rationale requiring HTTPS for Mattermost slash callbacks.
  - confidence: 0.9
- **[P2] Make trigger binding fail closed:** `extensions/mattermost/src/mattermost/slash-http.ts:55`
  - body: `commandTokenTriggers` is optional, so a handler constructed without it silently falls back to token-only validation. Since this patch relies on token-to-trigger binding for the security fix, make the binding required or reject callbacks when the map is missing.
  - confidence: 0.84
