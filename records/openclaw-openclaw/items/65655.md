## Summary

Keep open: the PR still targets a real Mattermost slash-callback security gap, but it is conflicting, partially superseded by current-main command validation, missing real behavior proof, and incomplete on callback transport policy.

## What This Changes

The PR changes Mattermost slash-command docs, registration, callback state, HTTP auth, and tests to fail closed on omitted non-HTTPS derived callback URLs and bind callback tokens to slash triggers.

## Best Possible Solution

Rework or replace this with a narrow Mattermost plugin fix that preserves current-main command validation, rejects the agreed non-HTTPS callback cases before registration, updates docs/tests, and includes redacted real Mattermost/gateway proof.

## Reproduction Assessment

Yes, at source level. Enable Mattermost native slash commands without `channels.mattermost.commands.callbackUrl`; current main derives an `http://` callback URL and proceeds into registration after only the warning path.

## Solution Assessment

No, not as submitted. The token-binding half is superseded by stronger current-main validation, and the remaining transport fix is too narrow because explicit `http://` callback URLs still bypass it.
