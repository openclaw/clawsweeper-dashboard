## Summary

Keep open. The PR has useful security intent, but current main already supersedes the token-replay half, the remaining callback-transport fix is incomplete and merge-conflicting, and the PR still lacks real Mattermost/gateway proof.

## What This Changes

The branch changes Mattermost slash-command docs, registration, callback state, HTTP auth, and tests to block omitted non-HTTPS derived callback URLs and bind callback tokens to slash triggers.

## Best Possible Solution

Rebase or replace this with a narrow Mattermost plugin fix that preserves current-command validation, implements the agreed non-HTTPS callback policy, updates docs/tests, and includes redacted live Mattermost/gateway proof.

## Reproduction Assessment

Yes, at source level. Enable Mattermost native slash commands without channels.mattermost.commands.callbackUrl; current main derives an http:// callback URL and proceeds into registration after only the warning path.

## Solution Assessment

No, not as submitted. The token-binding half is superseded by stronger current-main validation, and the remaining transport fix is stale and too narrow because explicit http:// callback URLs still bypass it.
