## Summary

Keep open: the PR still targets a real Mattermost slash-command security issue, but it is merge-conflicting, partly superseded by current main, incomplete for explicit HTTP callback URLs, and missing real Mattermost/gateway proof.

## What This Changes

This PR updates Mattermost native slash-command docs, registration, callback state, HTTP auth, and tests to reject omitted non-HTTPS derived callback URLs and bind callback tokens to slash triggers.

## Best Possible Solution

Rework or replace this as a narrow Mattermost plugin transport fix that preserves current-main command validation, rejects the agreed non-HTTPS callback cases before registration, updates docs/tests, and includes redacted real Mattermost/gateway proof.

## Reproduction Assessment

Yes, at source level. Enable Mattermost native slash commands without channels.mattermost.commands.callbackUrl; current main derives an http:// callback URL and proceeds into registration after only the warning path.

## Solution Assessment

No, not as submitted. The token-binding half is superseded by stronger current-main validation, the remaining transport fix is too narrow for explicit http:// callback URLs, and the branch needs conflict rework plus real proof.
