## Best Possible Solution

Land this or a replacement Mattermost plugin transport gate after maintainers accept the fail-closed upgrade behavior, the contributor adds redacted real setup proof, and the PR body matches the narrowed scope.

## Reproduction Assessment

Yes, source-level. Current main derives an http:// Mattermost slash callback when commands.callbackUrl is omitted and proceeds toward registration without an HTTPS gate; I did not run a live Mattermost network capture in this read-only review.

## Solution Assessment

Yes if maintainers accept the breaking hardening: the guard belongs in the Mattermost registration path before token-bearing callbacks are registered over cleartext. If preserving HTTP deployments wins, the safer alternative is a maintainer-approved staged or opt-in strict mode.
