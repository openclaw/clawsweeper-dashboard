## Summary

Keep open: the linked Mattermost security issue still has an unresolved cleartext callback transport gap on current main and this PR is the paired implementation path, but the branch is stale and incorrect as submitted after the merged command-validation work, and it lacks real behavior proof.

## What This Changes

The PR changes Mattermost slash-command docs, registration, state, HTTP callback auth, and tests to reject omitted non-HTTPS derived callback URLs and bind command tokens to slash triggers.

## Best Possible Solution

Rebase or replace this with a narrow Mattermost plugin-local HTTPS fail-closed callback policy fix on current main, preserving current command validation and adding focused docs, tests, and real Mattermost proof.

## Reproduction Assessment

Yes. Source inspection of current main and v2026.5.7 shows omitted Mattermost slash `callbackUrl` derives `http://...` and the monitor warns then registers; I did not run a live Mattermost roundtrip in this read-only review.

## Solution Assessment

No, not as submitted. The token-replay half is already solved more strongly on current main, and the remaining transport fix should be narrowed/rebased while rejecting the chosen class of non-HTTPS callback URLs before registration.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Reject explicit non-HTTPS callback URLs:** `extensions/mattermost/src/mattermost/monitor-slash.ts:168-174`
  - body: The new guard only runs when `callbackUrl` is omitted. If an operator explicitly configures an `http://` callback, registration still proceeds over cleartext despite the PR docs and rationale requiring HTTPS for Mattermost slash callbacks.
  - confidence: 0.91
- **[P2] Make trigger binding fail closed:** `extensions/mattermost/src/mattermost/slash-http.ts:55`
  - body: `commandTokenTriggers` is optional, so a handler constructed without it silently falls back to token-only validation. Since this patch relies on token-to-trigger binding for the security fix, make the binding required or reject callbacks when the map is missing.
  - confidence: 0.84
