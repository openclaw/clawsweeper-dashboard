## Summary

Keep open: current main and v2026.5.12 still allow the derived cleartext Mattermost slash callback path, but this branch is not ready as submitted because the token-replay half is superseded by stronger current-main validation and the remaining HTTPS transport fix needs a narrow rebase plus real behavior proof.

## What This Changes

The PR updates Mattermost slash-command docs, registration, state, callback auth, and tests to reject omitted non-HTTPS derived callback URLs and bind callback tokens to slash triggers.

## Best Possible Solution

Ship a narrow Mattermost plugin-local transport fix on current main that rejects the chosen class of non-HTTPS resolved slash callback URLs before registration, preserves current command-specific validation, and updates focused docs/tests with redacted real Mattermost or gateway proof.

## Reproduction Assessment

Yes. A source-level reproduction remains: enable Mattermost native slash commands without `channels.mattermost.commands.callbackUrl`, and current main derives an `http://` callback, warns, then registers it. The cross-trigger token replay half no longer reproduces on current main because callbacks are resolved to a specific registered command before token validation.

## Solution Assessment

No, not as submitted. The token-binding approach is superseded by stronger current-main validation, while the remaining transport fix is too narrow for explicit `http://` callbacks and needs maintainer policy/rebase work plus real behavior proof.
