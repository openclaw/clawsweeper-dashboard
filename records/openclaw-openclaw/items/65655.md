## Summary

Keep open: the current diff targets the remaining Mattermost HTTPS fail-closed gap, and current main plus v2026.6.6 still allow derived HTTP callback registration. ## What This Changes

This PR adds an HTTPS-only gate before Mattermost native slash-command registration and updates the Mattermost docs and monitor-slash tests for derived and explicit HTTP callbacks.

## Best Possible Solution

Land a narrow Mattermost plugin transport hardening only after redacted real-environment proof and maintainer acceptance of the HTTPS-only startup behavior, while preserving the current per-command token validation already on main.

## Reproduction Assessment

Yes, at source level: enable Mattermost native slash commands, omit channels.mattermost.commands.callbackUrl, and current main derives an http:// callback while the monitor proceeds to registration after warning. I did not run a live Mattermost roundtrip in this read-only review.

## Solution Assessment

Yes if maintainers choose HTTPS-only as the Mattermost slash callback policy; otherwise the safer path is to narrow the gate to the unsafe derived/public HTTP cases and document any explicit local exception.
