## Summary

Keep open. Current main still has the cmd-specific Windows shell-wrapper allowlist gate, while current wrapper resolution recognizes PowerShell/pwsh as shell-wrapper transports, so the hardening direction remains valid. The submitted patch is not ready as-is because it changes denial text without updating existing assertions, and it leaves node docs and the required changelog entry behind.

## What This Changes

The PR broadens the node-host Windows allowlist shell-wrapper block from cmd.exe-only to all shell-wrapper invocations, updates the denial message to mention PowerShell, and adds an exec-policy regression test for a PowerShell wrapper.

## Best Possible Solution

Keep the security posture fail-closed for Windows shell-wrapper transports in allowlist mode, with denial copy, node docs, changelog, and regression tests aligned so cmd.exe, powershell, and pwsh cases are covered consistently.
