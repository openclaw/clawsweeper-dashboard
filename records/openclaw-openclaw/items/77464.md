## Summary

Keep open: this PR targets a real source-reproducible plugin registry bug, but the submitted derived-path change regresses the existing explicit `installRecords` override semantics.

## What This Changes

The PR adds managed-npm recovery checks to plugin registry snapshots, adds registry/status regressions, and updates the changelog for external plugin visibility after package-manager upgrades.

## Best Possible Solution

Land this after preserving explicit `installRecords` overrides while letting absent overrides use the default managed-npm recovery reader, with focused regression coverage and exact-head validation.

## Reproduction Assessment

Yes. Current main source shows the persisted snapshot can be trusted before managed-npm recovery records are compared, and the stale derived path can bypass the recovery reader by forcing records from the stale persisted index.

## Solution Assessment

No as submitted. The recovery direction is narrow and maintainable, but the derived path should preserve caller-provided `params.installRecords` and only fall back to the default reader when no explicit override is supplied.
