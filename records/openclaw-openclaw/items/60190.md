## Summary

Keep open. Current main has a shared `messages.inbound` debounce surface and some channel-specific coalescers, but the documented shared policy still excludes media and several mainstream handlers either bypass media batching or would lose media metadata, so the requested image-plus-text aggregation remains incomplete.

## What This Changes

Request configurable delayed aggregation so multi-part channel messages such as image plus text can reach the agent as one conversation turn.

## Best Possible Solution

Extend the existing `messages.inbound` debouncer into a documented media-aware batching contract that preserves text, media metadata, dedupe IDs, and latest-message reply identity, then add focused coverage for at least one media-heavy channel plus shared contract behavior.

## Reproduction Assessment

Yes. A high-confidence source reproduction is a media-bearing message followed quickly by text on Discord, Slack, Signal, or QQ Bot: the shared policy bypasses `hasMedia`, and QQ Bot drains immediately rather than waiting for a quiet window.

## Solution Assessment

Unclear. The per-channel delay direction fits the existing architecture, but the best fix should extend `messages.inbound` and media-preserving channel merge semantics instead of adding a separate `messageAggregationDelayMs` surface.
