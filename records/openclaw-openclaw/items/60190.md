## Summary

Keep open. Current main has `messages.inbound` debouncing and a few channel-specific coalescers, but the shared contract is still documented and implemented as text-only, while Discord, Slack, Signal, Microsoft Teams, and QQ Bot still do not provide the requested media-plus-text quiet-window aggregation.

## What This Changes

The issue requests configurable per-channel delayed aggregation so multi-part user inputs such as image or sticker plus text reach the agent as one conversation turn.

## Best Possible Solution

Extend the existing `messages.inbound` debounce contract into documented media-aware batching that preserves text, media metadata, dedupe IDs, and latest-message reply identity, then cover at least one media-heavy channel plus shared contract behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: enable `messages.inbound`, send a media-bearing message followed quickly by text on Discord, Slack, Signal, Microsoft Teams, or QQ Bot, and current main either bypasses debounce through media gates or processes the queue immediately without the requested quiet window.

## Solution Assessment

Unclear. The requested per-channel delay is directionally right, but current main already has `messages.inbound.byChannel`; the maintainable fix should extend that existing contract for media instead of adding a separate `messageAggregationDelayMs` surface.
