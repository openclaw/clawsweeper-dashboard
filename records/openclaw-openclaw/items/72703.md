## Summary

Keep open. Current main still prepends queued runtime system events into user prompt fields, and the linked closing PR remains open rather than merged. The heartbeat-prompt wording is a separate scope question for maintainers, not a reason to close the validated system-event bug.

## What This Changes

The issue asks for background task updates, deferred maintenance notices, and heartbeat/system prompts to avoid being delivered as user-role/user prompt messages.

## Best Possible Solution
