## Summary

Keep open. Current main still matches the reported behavior, and an open unmerged closing PR already tracks the narrow implementation path, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Background task, maintenance, and heartbeat-generated updates are requested to avoid entering the model as user-role prompt text.

## Best Possible Solution

Route drained runtime system-event blocks into runtime system context, keep them out of user prompt and transcript fields, and treat heartbeat prompt behavior as a separate product decision if maintainers want to change it.

## Reproduction Assessment

Yes. A high-confidence source/test reproduction exists by mocking `drainFormattedSystemEvents` in `runPreparedReply` and inspecting the `runReplyAgent` payload; current tests assert the system line appears in `commandBody` and `followupRun.prompt`.

## Solution Assessment

Yes. Fixing the prepared-reply prompt assembly seam is the narrow maintainable path because that is where queued system events become model input; changing heartbeat semantics would be broader than this bug fix.
