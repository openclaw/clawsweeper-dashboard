## Summary

Keep open. Current main still drains queued runtime system events as `System:` text and prepends them to user prompt bodies, and the open linked fix PR has not merged, so this should wait for maintainer review/landing or an equivalent fix.

## What This Changes

The issue requests moving background task updates, deferred maintenance notices, and heartbeat/system-event prompts out of the user message stream into system-role or dedicated runtime context.

## Best Possible Solution

Land #72761 or an equivalent narrow change that routes drained queued runtime system events into `extraSystemPrompt` or another runtime system-context channel, keeps them out of `commandBody`, `followupRun.prompt`, and transcript prompt text, and preserves the existing untrusted-event owner downgrade.

## Reproduction Assessment

Yes. Current main has a read-only reproduction path in source and tests: mocked `drainFormattedSystemEvents` output is asserted to appear in `commandBody` while staying out of `extraSystemPrompt`.

## Solution Assessment

Yes for queued runtime system events. Moving drained events into runtime system context is the narrow maintainable fix; changing heartbeat prompt semantics should remain a separate maintainer scope decision if desired.
