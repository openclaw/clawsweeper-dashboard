## Summary

Keep open. Current main still prepends drained runtime system events into user prompt fields, and the linked unmerged PR is the right path to review before this issue is closed.

## What This Changes

Background task updates, deferred maintenance notices, and related runtime system events should be delivered through system/runtime context instead of user-role prompt text.

## Best Possible Solution

Review and land PR #72761, or an equivalent narrow change, so queued runtime system events move into runtime system context while preserving trusted/untrusted ownership semantics.

## Reproduction Assessment

Yes. Current main has a source-backed reproduction by mocking `drainFormattedSystemEvents` in `runPreparedReply`; the existing focused test asserts the system line appears in `commandBody` and not in runtime system prompt context. I did not run the test because this review was required to keep the checkout read-only.

## Solution Assessment

Yes. Moving drained runtime system events at the prepared-reply prompt assembly seam is the narrowest maintainable fix; broader heartbeat delivery policy should remain a separate maintainer scope decision.
