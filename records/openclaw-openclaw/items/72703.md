## Summary

Keep open because current main still routes drained runtime system events through user-prompt bodies, while linked PR #72761 remains open and unmerged; closing before merge would drop the active implementation path.

## What This Changes

Background task updates, deferred maintenance notices, and heartbeat prompts should use system-role or another non-user prompt mechanism instead of polluting user-role messages.

## Best Possible Solution

Land PR #72761 or an equivalent narrow prompt-assembly fix for queued runtime system events, then split heartbeat role-policy work only if maintainers still want to change the documented heartbeat contract.

## Reproduction Assessment

Yes. Source inspection on current main shows queued system events flow into `commandBody` and `followupRun.prompt`, with current tests asserting that placement; I did not execute tests in this read-only review.

## Solution Assessment

Yes for queued system events: the prepared-reply prompt assembly seam is the narrow maintainable fix point. The heartbeat prompt portion is documented as a user message today and should remain a separate maintainer policy decision.
