## Summary

Keep open. Current main still routes drained runtime system events into user prompt fields, and the open closing PR is the right implementation path to review before this issue is closed.

## What This Changes

Request background task updates, deferred maintenance notices, and heartbeat-related runtime notifications be kept out of user-role model messages and delivered as system context or a dedicated runtime channel.

## Best Possible Solution

Land #72761 or an equivalent narrow change that routes drained runtime system events into runtime system context, keeps them out of user-authored prompt and transcript fields, and preserves trusted/untrusted ownership semantics.

## Reproduction Assessment

Yes. Current main has a high-confidence source/test reproduction by mocking `drainFormattedSystemEvents` in `runPreparedReply`; the focused test asserts the system line appears in `commandBody` and `followupRun.prompt`.

## Solution Assessment

Yes. Moving queued runtime system events at the reply prompt assembly seam is the narrow maintainable fix; task delivery policy and intentional heartbeat user-message behavior should stay separate unless maintainers choose a broader product change.
