## Summary

Keep open. Current main still drains queued runtime system events into the prepared-reply user prompt path, and open PR #72761 is already the linked closing implementation path rather than a merged fix.

## What This Changes

Background task updates, deferred maintenance notifications, and heartbeat prompts are requested to use system context or another non-user delivery path instead of user-role prompt text.

## Best Possible Solution

Review and land PR #72761 or an equivalent narrow change that routes queued runtime system events into runtime system context while keeping user-authored prompt and transcript fields clean.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: mock or enqueue a formatted system event, run prepared reply assembly, and inspect `commandBody`/`followupRun.prompt` versus `extraSystemPrompt`; I did not execute tests because `pnpm` is unavailable in this read-only environment.

## Solution Assessment

Yes for the queued system-event portion. The prepared-reply prompt assembly seam is the narrow maintainable fix point; heartbeat prompt role policy should be split into a separate maintainer decision if it remains desired.
