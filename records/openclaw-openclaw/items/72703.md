## Summary

Keep open. Current main still drains queued runtime system events into prepared-reply user prompt bodies, and open PR #72761 is the linked closing implementation path rather than a merged fix.

## What This Changes

The issue requests routing background task, deferred maintenance, and heartbeat/system notifications outside the user-role prompt stream.

## Best Possible Solution

Review and land PR #72761 or an equivalent narrow fix that routes queued runtime system events into runtime system context while leaving heartbeat prompt policy as an explicit follow-up if maintainers want that changed.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: mock or enqueue a formatted system event, run prepared reply assembly, and inspect `commandBody`/`followupRun.prompt` versus `extraSystemPrompt`; I did not execute tests because `pnpm` is unavailable in this environment.

## Solution Assessment

Yes for queued runtime system events: the prepared-reply prompt assembly seam is the narrow maintainable fix point. The heartbeat prompt portion should stay separate unless maintainers decide its role policy should change too.
