## Summary

Keep open. Current main still prepends drained runtime system events into prepared user prompt fields, and the linked open PR #72761 is the active unmerged closing path rather than a reason to close this issue before merge.

## What This Changes

Background task updates, deferred maintenance notifications, and heartbeat prompts should be delivered through system/runtime context or another non-user mechanism instead of user-role prompt text.

## Best Possible Solution

Land PR #72761 or an equivalent narrow change that routes drained runtime system events through runtime system context while preserving heartbeat behavior until maintainers decide that policy separately.

## Reproduction Assessment

Yes. Current main has a high-confidence source reproduction by mocking drainFormattedSystemEvents in runPreparedReply; the focused test already asserts the event text appears in commandBody and not runtime system context.

## Solution Assessment

Yes for queued runtime system events: fixing the prepared-reply prompt assembly seam is the narrow maintainable path. The heartbeat prompt role question is separate and needs maintainer judgment.
