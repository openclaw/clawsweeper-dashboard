## Summary

Keep open. Current main still routes drained runtime system events into user prompt fields, and the open closing PR is the right path to review before this issue is closed.

## What This Changes

The issue asks for background task updates, deferred maintenance notices, and related system-generated prompts to avoid user-role prompt/transcript fields and instead use system context or a dedicated non-user mechanism.

## Best Possible Solution

Land PR #72761 or an equivalent narrow change that moves drained runtime system events into runtime system context while keeping trusted/untrusted ownership semantics intact.
