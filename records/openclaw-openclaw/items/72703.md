## Summary

Keep open. Current main still routes drained runtime system events into user prompt fields, and the open closing PR is the active implementation path to review before this issue should close.

## What This Changes

Background task, deferred maintenance, and heartbeat-related runtime notices should stop appearing as user-role prompt text and instead use system/runtime context where appropriate.

## Best Possible Solution

Land #72761 or an equivalent narrow change that routes drained runtime system events into runtime system context while keeping task delivery policy and heartbeat prompt semantics explicit.

## Reproduction Assessment

Yes. Current main has a high-confidence source/test reproduction: mock `drainFormattedSystemEvents` in `runPreparedReply` and observe the event text in `commandBody` rather than runtime system context.

## Solution Assessment

Yes for the queued runtime event path. Moving the drained event block at the prepared-reply prompt assembly seam is the narrow maintainable fix; heartbeat prompt behavior should be a separate maintainer scope decision if it also needs to change.
