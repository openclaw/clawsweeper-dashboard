## Summary

Keep open. Current main still routes drained runtime system events into user prompt fields, and PR #72761 remains open/unmerged with closing syntax, so this should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that background task updates, deferred maintenance notifications, and heartbeat prompts be delivered through system/runtime context instead of user-role prompt text.

## Best Possible Solution

Land PR #72761 or an equivalent narrow change that moves drained runtime system events into runtime system context while keeping user-authored prompt and transcript fields clean.

Yes. Current main has a high-confidence source reproduction by mocking `drainFormattedSystemEvents` in `runPreparedReply`; the existing focused test already asserts the mocked event appears in `commandBody` rather than `extraSystemPrompt`.

## Solution Assessment

Yes for queued runtime system events: fixing the prepared-reply prompt assembly seam is the narrow maintainable path. Heartbeat prompt role policy is separate and should remain an explicit maintainer decision.
