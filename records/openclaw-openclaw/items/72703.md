## Summary

Keep open. Current main still routes drained runtime system events into user prompt fields, and the open closing PR is the tracked implementation path rather than a reason to close this issue before merge.

## What This Changes

The issue asks for OpenClaw-generated background task, maintenance, and heartbeat/system notices to avoid entering the model as user-role prompt text.

## Best Possible Solution

Land PR #72761 or an equivalent narrow change that routes drained runtime system events into runtime system context while keeping user-authored prompt and transcript fields clean.

## Reproduction Assessment

Yes. Current main has a source-backed reproduction by mocking `drainFormattedSystemEvents` in `runPreparedReply`; the existing focused tests assert the system line appears in `commandBody` and `followupRun.prompt`.

## Solution Assessment

Yes. Moving queued runtime system events at the reply prompt assembly seam is the narrow maintainable fix; heartbeat prompt policy should remain a separate explicit product decision.
