## Summary

Keep open. Current main still routes drained runtime system events into user prompt-body fields, and the linked closing PR is open and unmerged, so this issue should wait for that PR or an equivalent fix rather than be closed early.

## What This Changes

The issue asks OpenClaw to deliver background task, deferred maintenance, and heartbeat/system notifications as system/runtime context instead of polluting user-role prompt text.

## Best Possible Solution

Land #72761 or an equivalent narrow patch that routes drained runtime system events into `extraSystemPrompt` or another runtime system-context channel, keeps them out of user prompt and transcript prompt text, preserves trusted/untrusted ownership behavior, and updates the focused tests/docs to match the new contract.

## Reproduction Assessment

Yes. The current source and focused test provide a high-confidence reproduction path: mock `drainFormattedSystemEvents`, call `runPreparedReply`, and observe the `System:` line in `commandBody` rather than runtime system context.

## Solution Assessment

Yes for queued runtime system events. Moving drained events to runtime system context is the narrow maintainable fix; changing heartbeat prompt semantics should be split out unless maintainers explicitly want that scope included.
