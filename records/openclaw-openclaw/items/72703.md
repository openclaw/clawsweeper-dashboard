## Summary

Keep open. Current main still routes drained runtime system events into user prompt fields, and the open closing PR #72761 is the right implementation path to review before closing this issue.

## What This Changes

Background task, deferred maintenance, and heartbeat-generated updates should not be delivered to the model as user-authored prompt text.

## Best Possible Solution

Land #72761 or an equivalent narrow change that routes drained runtime system events into runtime system context while preserving trusted/untrusted ownership semantics.

## Reproduction Assessment

Yes. Current main has a source-level reproduction by mocking `drainFormattedSystemEvents` in `runPreparedReply`; existing focused tests assert the system line appears in `commandBody` and `followupRun.prompt` instead of runtime system context.

## Solution Assessment

Yes for the linked PR direction, but not for closing now. Moving queued runtime system events at the reply prompt assembly seam is the narrow maintainable fix; heartbeat prompt policy should stay explicit unless maintainers choose broader scope.
