## Summary

Keep open: current main still has no diagnostic liveness startup-grace gate, and https://github.com/openclaw/openclaw/pull/79922 is an open closing PR for this issue, so the issue should stay open until that PR is merged or closed.

## What This Changes

The issue asks OpenClaw to suppress diagnostic liveness WARN emission during gateway cold start with a startup grace window while keeping liveness sampling active.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/79922; the fix should gate diagnostic liveness event/WARN emission during startup while preserving sampler resets, with any env/config knob handled separately.

## Reproduction Assessment

Yes. Current main starts diagnostic heartbeat from gateway startup without a grace option, samples liveness every heartbeat, and warns when a liveness sample coincides with open diagnostic work; a focused fake-timer diagnostic heartbeat test can exercise this without the Windows setup.

## Solution Assessment

Yes. The narrow maintainable fix is to add an internal startupGraceMs boundary and gate only liveness event/WARN emission while continuing to call sampleLiveness; exposing an env/config knob should be a separate maintainer choice.
