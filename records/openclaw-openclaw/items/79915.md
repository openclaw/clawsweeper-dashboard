## Summary

Keep open: current main still lacks a diagnostic startup-grace guard, and the linked open fix PR should land before this issue is closed.

## What This Changes

The issue requests suppressing false diagnostic liveness warnings during cold-start blocking by adding a startup grace window while keeping steady-state liveness monitoring intact.

## Best Possible Solution

Review and land a corrected https://github.com/openclaw/openclaw/pull/79922 that keeps liveness sampling active during startup grace while suppressing warning/event emission.

## Reproduction Assessment

Yes. Current main starts diagnostic liveness sampling at gateway startup and has no grace check before liveness event or warning emission, so a focused fake-timer test with queued work during the first heartbeat window should reproduce the warning path.

## Solution Assessment

Yes for the issue direction: an internal startup-grace boundary is the narrowest fix. The implementation should call the sampler every tick and gate only event/WARN emission; the optional env/config knob should stay separate.
