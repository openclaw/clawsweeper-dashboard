## Summary

Keep open: current main still lacks the diagnostic heartbeat startup-grace guard, and https://github.com/openclaw/openclaw/pull/79922 is an open closing PR for the fix, so this issue should stay open until that PR is reviewed and merged or replaced.

## What This Changes

Add a diagnostic heartbeat startup-grace window so normal gateway cold start does not emit false liveness warnings.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/79922 with a narrow internal startup-grace fix that suppresses startup liveness emissions without adding the optional user config knob.

## Reproduction Assessment

Yes. Current main starts the diagnostic heartbeat during gateway boot and has no grace check before liveness event or warning emission, so a focused fake-timer test with queued diagnostic work in the first heartbeat window should reproduce the warning path.

## Solution Assessment

Yes, the best direction is a narrow internal startup-grace boundary. The linked PR is the right implementation lane, but reviewers should ensure it preserves liveness sampler reset and baseline behavior from https://github.com/openclaw/openclaw/pull/76401 before merge.
