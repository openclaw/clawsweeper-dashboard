## Summary

Keep open: current main still starts diagnostic liveness sampling without a startup-grace guard, and the active linked PR is the right close path once it lands.

## What This Changes

Add a diagnostic heartbeat startup-grace window so cold-start event-loop delay does not emit false liveness warnings while the sampler still runs for metric resets.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79922 or an equivalent narrow patch, keeping any configurable grace-window knob as a separate follow-up.

## Reproduction Assessment

Yes. Source inspection shows current main samples liveness immediately after gateway startup with no grace check; a focused fake-timer diagnostic test with open work and a mocked liveness sample should reproduce the warning path.

## Solution Assessment

Yes. The narrowest fix is to add an internal startup-grace boundary that still runs the sampler for resets but suppresses liveness event and WARN emission until the grace expires.
