## Summary

Keep open: the main WhatsApp direct routing change is pointed at the right bug, but the PR still misses the documented WhatsApp broadcast-DM path, so the direct-chat isolation guarantee is incomplete before merge.

## What This Changes

The PR adds a route-level DM-scope override, applies it to WhatsApp inbound direct routing and isolated last-route writes, and updates tests, docs, and changelog for account-aware WhatsApp direct session keys.

## Best Possible Solution

Update the PR so every WhatsApp inbound direct path, including broadcast fan-out, uses account-aware per-contact session keys and has regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows WhatsApp direct messages inherit the default main DM scope, and the proposed patch leaves direct broadcast messages source-reproducible because buildBroadcastRouteKeys still uses cfg.session?.dmScope.

## Solution Assessment

No. The proposed route override is the right narrow mechanism for ordinary inbound DMs, but it is incomplete until WhatsApp broadcast DMs reuse the same account-aware direct scope.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Carry WhatsApp DM isolation into broadcast routes:** `extensions/whatsapp/src/auto-reply/monitor/on-message.ts:97-99`
  - body: The new `dmScopeOverride` only affects the initial route, but configured WhatsApp broadcast DMs rebuild each agent route in `buildBroadcastRouteKeys` using `cfg.session?.dmScope`. With the default `main`, a broadcast DM still runs each broadcast agent on `agent:<agentId>:main`, so different direct contacts can keep sharing context even though the PR documents all direct chats as account-aware. Thread the same `per-account-channel-peer` scope into direct broadcast route construction and cover it in the broadcast test.
  - confidence: 0.88
