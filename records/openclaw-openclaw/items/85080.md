## What This Changes

The issue reports that `openclaw update` can overwrite user-customized `agents.defaults.model`, per-agent model overrides, and `plugins.entries.*` blocks with distribution defaults.

## Best Possible Solution

Preserve authored agent model/auth-routing intent and plugin entry config across update-time doctor and post-core plugin convergence, with focused regression coverage and packaged update proof.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection gives a credible current-main path: update-time doctor/config writes are followed by channel-only pre-update restoration, with no equivalent restoration for the reported model defaults or plugin entries.

## Solution Assessment

Yes, the best fix is to extend the update merge/restore boundary and add regression coverage rather than add a new config flag first. The implementation should preserve effective OAuth routing while still allowing intentional doctor migrations such as Codex route canonicalization.
