## Summary

Keep this PR open: it addresses a real config hardening bug, and the author addressed several earlier automated review notes, but the current head still blocks valid exact deny entries, omits talk-capable node commands from the validation vocabulary, and lacks real after-fix CLI behavior proof.

## What This Changes

The PR adds write-time validation for `gateway.nodes.denyCommands` in `openclaw config set`/`unset` and dry-run paths with regression tests.

## Best Possible Solution

Land a focused validator that shares the runtime/audit command vocabulary, accepts exact known IDs before pattern guidance, covers talk-capable commands, and includes redacted real CLI proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: current main accepts any string array for `gateway.nodes.denyCommands`, while runtime removes only exact known allowlist entries, so a typo can be written and then ignored.

## Solution Assessment

No. The validation direction is narrow and useful, but this implementation should first match runtime/audit semantics, fix the two validator gaps, rebase, and show real `openclaw config` behavior proof.
