## Summary

Keep open: current main and the latest release still implement the reported Gemini CLI OAuth client credential extraction fallback, and the `security` protected label makes this a maintainer security/product decision rather than cleanup-close material.

## What This Changes

The issue asks OpenClaw to remove or explicitly gate automatic extraction of Gemini CLI OAuth client credentials from another application installation.

## Best Possible Solution

Make Gemini CLI OAuth client configuration an explicit, documented operator choice in the Google plugin, with compatibility-conscious upgrade proof for existing users instead of silently reading Gemini CLI files.

## Reproduction Assessment

Yes. Source inspection shows the fallback path, and existing tests intentionally exercise extraction from multiple Gemini CLI install layouts and the session cache; I did not run a live Gemini install because this review is read-only.

## Solution Assessment

Unclear. Removing the fallback is the clean security direction, but maintainers need to choose the compatibility policy for existing `google-gemini-cli` OAuth users before implementation.
