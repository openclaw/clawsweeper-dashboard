## Summary

Keep open. Current main still implements the reported Gemini CLI OAuth client-config extraction, caches extracted values for the process, and uses them as the fallback when explicit OAuth client environment variables are absent. The latest release tag also contains the same behavior. The existing provider/docs warnings cover unofficial Gemini CLI OAuth risk, but they do not explicitly disclose or gate reading Gemini CLI source/bundle files for OAuth client ID and secret. The related #41991 PR is diagnostics-only, and the protect-mcp comment is useful credential-vault context rather than a fix for this bundled behavior.

## What This Changes

The issue requests removing or explicitly gating OpenClaw's automatic extraction of Gemini CLI OAuth client ID and secret from a local Gemini CLI installation.

## Best Possible Solution

Decide the security/product posture for the bundled Google plugin, then either remove automatic Gemini CLI OAuth client-config extraction by default or gate it behind extraction-specific opt-in/confirmation. Update `extensions/google/oauth.credentials.ts`, the OAuth flow/tests, the provider prompt, and the public Google provider docs; handle #41991 as a separate diagnostics PR or fold diagnostics into the chosen hardening path.
