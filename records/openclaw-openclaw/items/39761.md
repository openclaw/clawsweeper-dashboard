## Summary

Keep open. The feature direction is useful, but the branch is not mergeable as-is because current main moved the public hook contract and active durable delivery path away from the files this PR changes, and the external PR still lacks real after-fix behavior proof.

## What This Changes

This PR adds a best-effort `agentContext` payload to `message_sending` hooks, carrying tool names, token usage, context-fill percentage, agent/session identity, channel, and response length from embedded-agent runs.

## Best Possible Solution

Define one backward-compatible typed outbound provenance or agent-completion context on the active hook contract and durable send paths, with focused tests, docs, and redacted real-send proof.

## Reproduction Assessment

Yes, source-level: current main and `v2026.5.12` both omit agent reasoning context from the active `message_sending` event and runtime payload. I did not run a live send in this read-only review.

## Solution Assessment

No. The direction is useful, but this branch is not the best mergeable solution until it is rebased onto the active hook contract and durable delivery surfaces and maintainers approve the public context shape.
