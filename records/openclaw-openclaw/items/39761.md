## Summary

Keep open: the requested hook context is a useful public plugin API direction, but this PR is not mergeable as-is because it targets stale contract and delivery surfaces while current main’s active paths still omit `agentContext`.

## What This Changes

The PR adds an optional `agentContext` payload to `message_sending` hooks and threads embedded-agent tool, token, session, channel, and response-length metadata through outbound delivery.

## Best Possible Solution

Define one backward-compatible typed agent/provenance context on the active `message_sending` contract and durable delivery paths, backed by docs, focused tests, and redacted runtime proof.

## Reproduction Assessment

Yes, source-level: current main’s active hook event and runtime payload omit `agentContext`, while this PR changes stale or no-longer-active surfaces. I did not run a live send in this read-only review.

## Solution Assessment

No. The direction is useful, but this branch is not the best mergeable solution until it is rebased onto the active hook contract and durable delivery surfaces with maintainer approval for the public context shape.
