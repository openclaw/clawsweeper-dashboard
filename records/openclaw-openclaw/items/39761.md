## What This Changes

The PR adds optional agent reasoning metadata to `message_sending` hook events by threading tool-call, token-usage, session, channel, and response-length data through outbound delivery.

## Best Possible Solution

Rebase onto current main and define one backward-compatible typed completion/provenance context on the active hook event and durable send surfaces, with docs and tests for every agent-originated path that populates it.

Yes, source-level: current main's active hook event and runtime payload omit `agentContext`, while this PR changes stale or no-longer-active surfaces. I did not run a live send.

## Solution Assessment

No. The direction is useful, but the patch is stale and partial; the safer solution is a current-main rebase plus maintainer approval for the public hook contract and durable delivery coverage.
