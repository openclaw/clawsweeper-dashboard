## Summary

Keep this PR open, but the current patch should not merge as-is because it is stale against current main's hook/delivery split and still leaves common agent-originated sends without the proposed metadata.

## What This Changes

The PR adds optional `agentContext` metadata to `message_sending` hook events and threads embedded-agent tool, usage, session, and response details through agent command delivery, outbound delivery, tests, and the changelog.

## Best Possible Solution

Define the public agent-context contract on `hook-message.types.ts`, then thread a consistent best-effort run context through every agent-originated `message_sending` path with tests for command delivery, inbound replies, non-agent sends, retries/replay, tool metadata, model identity, and token/context values.

## Reproduction Assessment

Yes. Static inspection on current main shows `PluginHookMessageSendingEvent` has no `agentContext`, and both outbound and inbound `runMessageSending()` calls build events without agent-run metadata.

## Solution Assessment

No, not as-is. The optional field is a plausible direction, but this patch targets stale type/delivery surfaces and only covers one delivery path and latest-attempt metadata.
