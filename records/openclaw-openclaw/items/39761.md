The PR adds optional `agentContext` metadata to `message_sending` hook events by threading embedded-runner tool, token, session, and response-length data through outbound delivery, plus tests and a changelog entry.

## Best Possible Solution

Rebase onto current main and define one backward-compatible typed agent/provenance context on the active hook-message and durable send surfaces, with tests and docs for which agent-originated paths populate it.

Yes, source-level for the review findings: current main's active hook event/payload and durable send paths omit `agentContext`, and the PR diff changes legacy or deleted surfaces. I did not run a live send.
