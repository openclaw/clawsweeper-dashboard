## Summary

Keep open: current main still lacks this capability, but the branch is stale against the current hook/delivery surfaces and leaves common agent reply paths plus retry/failover metadata incomplete.

## What This Changes

The PR adds optional `agentContext` metadata to `message_sending` hook events, threads embedded-agent tool/usage/session data through agent delivery and outbound delivery, and adds tests plus a changelog entry.

## Best Possible Solution

Define the public agent-context contract on the current hook-message type surface, then thread it through every agent-originated `message_sending` path with tests documenting retry, non-agent, stop-reason, and model-identity behavior.

## Reproduction Assessment

Yes. Static inspection on current main shows the hook event has no `agentContext`, and the normal inbound reply path invokes `runMessageSending()` without any agent-run metadata.

## Solution Assessment

No, not as-is. The optional field is a plausible direction, but this patch targets stale type/delivery surfaces and does not cover common reply paths or run-level tool metadata across attempts.
