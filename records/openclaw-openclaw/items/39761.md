## Summary

Keep this PR open: the feature is useful, but the branch is stale against current main and the patch still defines an inconsistent public hook contract for current delivery paths.

## What This Changes

The PR adds optional agent reasoning metadata to `message_sending` hook events, threads embedded-runner tool/token/session data into outbound delivery, and adds tests plus a changelog entry.

## Best Possible Solution

Rebase onto current main and define the hook contract on the active hook-message and durable send surfaces, with explicit tests for either all agent-originated deliveries or a clearly documented narrower scope.

## Reproduction Assessment

Yes, source-level. Current main lacks `agentContext` on the active hook event type and the PR only populates it in the old CLI agent delivery path, while common durable reply paths still send no agent-run metadata.

## Solution Assessment

No. The direction is useful, but the current patch is stale and partial; the maintainable fix is a current-main rebase plus a maintainer decision on delivery-surface coverage before exposing a public plugin hook field.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Update the active hook event type:** `src/plugins/types.ts:611-616`
  - body: On current main, `PluginHookMessageSendingEvent` lives in `src/plugins/hook-message.types.ts` and is re-exported through `hook-types`. Adding `agentContext` only in `src/plugins/types.ts` leaves the actual hook runner/event contract unchanged after rebase, so the new delivery event either fails typecheck or exposes a different SDK type than the runner uses.
  - confidence: 0.86
- **[P2] Populate agent context consistently for agent sends:** `src/commands/agent/delivery.ts:223-226`
  - body: The proposed public contract says the context is present when a message originates from an agent run, but this patch only builds it in the CLI agent command path. Current main's durable reply and turn delivery paths send through `sendDurableMessageBatch` without this metadata, so hooks can silently skip policy checks for common agent replies.
  - confidence: 0.88
