## Summary

Keep open: current main still lacks the requested agent reasoning context on `message_sending`, but this PR is stale, conflicting, and targets legacy surfaces, so it needs a rebase, API-scope review, and real behavior proof before it can merge.

## What This Changes

The PR adds optional agent reasoning metadata to `message_sending` hook events and threads embedded-runner tool, token, session, and response-length data toward outbound delivery.

## Best Possible Solution

Define one maintainer-approved, backward-compatible provenance context on the active hook-message and durable send surfaces, then document exactly which agent-originated paths populate it and cover those paths with focused tests and real send proof.

## Reproduction Assessment

Yes, source-level for the review findings: current main’s active hook event and delivery hook construction omit `agentContext`, while the PR adds it only to stale/direct-delivery surfaces. I did not run a live send in this read-only review.

## Solution Assessment

No. The direction is useful, but the current patch is not the best implementation because it targets removed or inactive APIs and only partially covers agent-originated sends; the maintainable path is a current-main rebase around the durable delivery contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Move agentContext onto the active hook contract:** `src/plugins/types.ts:611-616`
  - body: This adds the public field to `src/plugins/types.ts`, but current main defines the active `PluginHookMessageSendingEvent` in `src/plugins/hook-message.types.ts` and re-exports it through `hook-types`. After rebase, plugin authors would still not get the advertised field unless the active contract is updated.
  - confidence: 0.88
- **[P2] Populate agentContext through durable sends:** `src/commands/agent/delivery.ts:220-226`
  - body: This builds `agentContext` only in the old command delivery path and passes it to direct outbound delivery. Current command replies and common reply routing use `sendDurableMessageBatch()`, so many agent-originated sends would still reach `message_sending` without the new context.
  - confidence: 0.87
- **[P2] Accumulate tool metadata across run attempts:** `src/agents/pi-embedded-runner/run.ts:1439`
  - body: Successful returns still expose only the current `attempt.toolMetas`. If an earlier retry/failover attempt used tools and a later attempt returns without tool calls, hooks can see `toolCallCount: 0` for a turn that did use tools, which breaks the proposed verification use case.
  - confidence: 0.76
