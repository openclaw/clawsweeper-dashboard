## Summary

Keep open. Current main still does not expose agent reasoning context on the active `message_sending` hook contract, and this PR remains useful but needs a current-main rebase, API-scope review, and real behavior proof before merge.

## What This Changes

The PR adds optional agent reasoning metadata to `message_sending` hook events, threads embedded-runner tool/token/session data into outbound delivery, and adds unit coverage plus a changelog entry.

## Best Possible Solution

Define the public hook contract on the active `hook-message` and durable delivery surfaces, decide whether `agentContext` covers all agent-originated sends or is explicitly best-effort, then add focused docs and contract tests for that chosen scope.

## Reproduction Assessment

Yes, source-level. Current main lacks `agentContext` on the active hook event type, and the PR only threads it through an older/narrow agent command delivery path while common durable reply paths still send no agent-run metadata.

## Solution Assessment

No. The direction is useful, but the proposed patch is stale and partial; the maintainable fix is a current-main rebase plus maintainer agreement on the public hook coverage contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Update the active hook event type:** `src/plugins/types.ts:611-616`
  - body: On current main, `PluginHookMessageSendingEvent` lives in `src/plugins/hook-message.types.ts` and is re-exported through `hook-types`. Adding `agentContext` only in `src/plugins/types.ts` leaves the actual runner/SDK hook contract unchanged after rebase, so hook authors would not get the advertised typed field.
  - confidence: 0.86
- **[P2] Populate agent context consistently for agent sends:** `src/commands/agent/delivery.ts:223-226`
  - body: The proposed public contract says the context is present when a message originates from an agent run, but this patch only builds it in the CLI agent command path. Current main's durable reply paths call `sendDurableMessageBatch` without this metadata, so hooks can silently skip policy checks for common agent replies.
  - confidence: 0.88
