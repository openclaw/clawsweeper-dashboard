## Summary

Keep open: the feature direction is useful, but the current branch is stale and partial against current main. The active hook contract has moved, common durable agent reply paths would still omit the proposed metadata, and the external PR still lacks after-fix real behavior proof.

## What This Changes

This PR adds optional `agentContext` metadata to the `message_sending` plugin hook, threads embedded-runner tool/token/session data into outbound delivery, and adds tests plus a changelog entry.

## Best Possible Solution

Rebase onto current main and define one documented agent-run metadata contract on the active `hook-message.types.ts` plus durable delivery pipeline, with explicit tests for the intended coverage.

## Reproduction Assessment

Yes, source-level. Current main lacks `agentContext` on the active hook event and hook construction path, while the PR head only threads it through an old CLI delivery surface.

## Solution Assessment

No. The direction is useful, but the current patch is not the narrow maintainable fix until it is rebased and maintainers decide whether the field covers all agent sends or a clearly documented subset.
