## Summary

Keep open: this is a useful public hook API proposal, but the current branch still targets stale hook/delivery surfaces and lacks required real behavior proof. Current main still does not expose agent provenance on the active `message_sending` contract, so the feature is not implemented elsewhere, but this PR is not ready to merge as-is.

## What This Changes

The PR adds optional `agentContext` metadata to `message_sending` hooks by threading tool names, token usage, context fill, session identity, channel, and response length from embedded agent runs through outbound delivery.

## Best Possible Solution

Define one maintainer-approved typed outbound provenance context on the active `message_sending` event and durable send paths, coordinated with stopReason and model-provenance work, with docs, focused tests, and redacted real-send proof.

## Reproduction Assessment

Yes, source-level: current main's active hook event and runtime payload omit `agentContext`, while agent run metadata already carries related signals. I did not run a live channel send in this read-only review.

## Solution Assessment

No. The branch is a useful prototype, but the maintainable fix needs an approved public API shape on `src/plugins/hook-message.types.ts` and the active durable delivery path, not the stale type and command shim edited here.
