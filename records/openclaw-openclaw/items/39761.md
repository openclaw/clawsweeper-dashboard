## Summary

Keep open: current main still does not expose agent reasoning metadata on the active `message_sending` hook, but this PR is not merge-ready because it targets stale hook/delivery surfaces, lacks real behavior proof, and needs a maintainer API-scope decision for coverage across agent delivery paths.

## What This Changes

This PR adds optional `agentContext` metadata to `message_sending` hook events, threads embedded-runner tool/token/session data into outbound delivery, and adds focused tests plus a changelog entry.

## Best Possible Solution

Rebase onto current main and define one backward-compatible agent completion/provenance context on the active hook-message and durable send surfaces, with tests and docs for which agent-originated paths populate it.

## Reproduction Assessment

Yes, source-level for the review findings. Current main lacks `agentContext` on the active hook event and call sites, while the PR only threads the proposed field through stale CLI-era surfaces.

## Solution Assessment

No. The direction is useful, but this patch is stale and partial; the safer solution needs a current-main rebase plus maintainer approval for the durable delivery API coverage.
