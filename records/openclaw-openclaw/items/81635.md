## Summary

Keep open: the PR has valid terminal proof for the raw `.oxlintrc.json` path, but it still leaves OpenClaw's supported lint wrapper suppressing the entire `eslint/no-underscore-dangle` rule, so merging this as-is would leave two different lint policies.

## What This Changes

The PR adds an `.oxlintrc.json` `eslint/no-underscore-dangle` allow-list for `__testing` and `require_`.

## Best Possible Solution

Make the config, wrapper, and wrapper tests express one policy: either keep `eslint/no-underscore-dangle` disabled for supported lint runs, or remove the wrapper suppression and enforce the rule with the narrow allow-list.

## Reproduction Assessment

Yes for the review finding. Current main routes supported lint commands through `scripts/run-oxlint.mjs`, which calls `applyLocalOxlintPolicy` and still injects `--allow eslint/no-underscore-dangle`.

## Solution Assessment

No. A config-only allow-list helps raw/editor oxlint invocations, but the maintainable fix is to align `.oxlintrc.json`, the wrapper policy, and wrapper tests after the intended rule policy is chosen.
