## Summary

## What This Changes

The PR adds an `.oxlintrc.json` rule entry allowing `__testing` and `require_` for `eslint/no-underscore-dangle`.

## Best Possible Solution

Keep one lint policy in one supported path: either retain wrapper-level suppression, or enforce the rule through `.oxlintrc.json` while removing the wrapper suppression and updating wrapper tests.

Yes for the review finding. Source inspection shows root lint scripts route through `scripts/run-oxlint.mjs`, which calls `applyLocalOxlintPolicy`; that helper and its tests still add `--allow eslint/no-underscore-dangle`.

## Solution Assessment

No. A config-only allow-list is incomplete while the supported wrapper path still suppresses the whole rule; the maintainable fix is to align config, wrapper behavior, and wrapper tests under one chosen policy.
