## Best Possible Solution

Choose one lint policy and keep it in one place: either disable `eslint/no-underscore-dangle` in shared config and remove the wrapper flag/tests, or enforce it with a complete allow-list plus wrapper and test updates.

## Reproduction Assessment

Yes for the review finding: source inspection shows `scripts/run-oxlint.mjs` calls `applyLocalOxlintPolicy`, and that helper still injects `--allow eslint/no-underscore-dangle` while the PR only changes `.oxlintrc.json`. I did not run oxlint because this review is read-only.

No. Adding a config allow-list while the wrapper suppresses the entire rule leaves supported CI/local wrapper behavior unchanged; the safer fix is to align config, wrapper, and tests under one policy.
