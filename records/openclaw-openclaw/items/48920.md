## Summary

Keep open. The original heartbeat `isolatedSession` mismatch is fixed in the latest stable release, but the broader live-docs-versus-stable-release drift remains real and needs a maintainer docs/release policy decision rather than cleanup closure.

## What This Changes

Request that live docs indicate or version-gate unreleased OpenClaw configuration and features so users on stable releases do not copy unsupported keys.

## Best Possible Solution

Define an explicit docs compatibility policy, then implement the matching docs publish/version metadata and release-compatibility checks, with the bundled local-docs work treated as complementary rather than a full replacement.

## Reproduction Assessment

Yes. The exact `isolatedSession` key now works in `v2026.5.7`, but the drift class still reproduces because live docs publish `openclaw path`/`oc-path` pages while the latest stable tag lacks that docs page and plugin package path.

## Solution Assessment

No. Closing as implemented would lose the unresolved docs/release policy gap; the narrow maintainable path is for maintainers to choose and document how live docs relate to stable releases, then add the corresponding workflow or validation guard.
