## Summary

Keep open: the original heartbeat `isolatedSession` mismatch is now shipped, but the broader live-docs-versus-stable-release bug is still source-reproducible because public docs advertise `toolResultMaxChars` up to `1000000` while the latest stable release schema still caps it at `250000`.

## What This Changes

The issue asks OpenClaw to keep live docs in step with stable releases or clearly version-label docs so users do not copy unsupported config into released builds.

## Best Possible Solution

Define the public docs compatibility policy, then implement release-pinned or version-labeled docs plus generated config-doc checks or diagnostics so stable-release users are not guided into unsupported config.

## Reproduction Assessment

Yes, at source level. Public docs currently advertise `toolResultMaxChars` up to `1000000`, while the latest stable release schema still rejects that range with a `250000` cap.

## Solution Assessment

Unclear as an implementation path. A one-off rollback for the current config example is not the best durable fix; maintainers need to choose and implement a public docs versioning, release-gating, or version-labeling policy.
