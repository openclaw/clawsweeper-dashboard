## Summary

Keep open. The original heartbeat `isolatedSession` mismatch is now shipped in the latest release, but the broader live-docs-versus-latest-release failure class remains source-reproducible because current main documents and accepts `channels.discord.agentComponents.ttlMs` while `v2026.5.20` does not, and the docs publish workflow still mirrors main.

## What This Changes

The issue asks OpenClaw to keep live docs aligned with released config support, or clearly version-gate docs, so users do not configure unreleased strict-schema keys.

## Best Possible Solution

Define an explicit compatibility policy for `docs.openclaw.ai`, then implement matching release metadata, version badges or publish gating, and generated config-doc compatibility checks; coordinate with https://github.com/openclaw/openclaw/issues/71301 without treating it as the sole replacement.

## Reproduction Assessment

Yes, at source level. Current main documents and accepts `channels.discord.agentComponents.ttlMs`, `v2026.5.20` lacks that schema/docs support, and the docs workflow still publishes main-sourced docs.

## Solution Assessment

Unclear until maintainers choose the docs compatibility model. The durable fix is not another one-off key update; it is a release/docs policy and guardrail that prevents the next strict-schema mismatch.
