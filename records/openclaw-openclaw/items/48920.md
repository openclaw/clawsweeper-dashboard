## Summary

Keep open. The original heartbeat key mismatch is shipped in the latest release, but the broader docs-versus-release drift remains unresolved because the docs publish path still mirrors current main and there is no clear public versioning or compatibility policy for live docs.

## What This Changes

The issue asks for live docs to stay aligned with released OpenClaw versions or show version/availability metadata so users do not copy unsupported config keys.

## Best Possible Solution

Define an explicit docs compatibility policy, then implement matching publish/version metadata and release-compatibility checks while coordinating with https://github.com/openclaw/openclaw/issues/71301 for bundled local docs.

## Reproduction Assessment

Yes. Source inspection shows the publish workflow still mirrors current main, and current main already has post-release docs/changelog content while the latest stable tag remains `v2026.5.12`; the exact heartbeat `isolatedSession` key is now shipped.

## Solution Assessment

No. Closing as implemented would only account for the original heartbeat key, while the maintainable solution needs an explicit docs/release compatibility policy and matching workflow or validation guard.
