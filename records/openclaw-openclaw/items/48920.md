## Summary

Keep open. The original `isolatedSession` example is now shipped, but the same live-docs-ahead-of-release failure class still exists: live docs now describe Discord realtime wake-name config that current `main` accepts and latest release `v2026.5.22` rejects under the strict schema.

## What This Changes

The issue asks OpenClaw to keep live docs in step with released config support, or clearly show version availability so users do not copy unreleased config keys.

## Best Possible Solution

Define an explicit compatibility policy for `docs.openclaw.ai`, then implement the matching release metadata, version badges or publish gating, and generated config-doc compatibility checks against the latest release tag.

## Reproduction Assessment

Yes, at source level. Live docs currently show `channels.discord.voice.realtime.requireWakeName`, current main accepts it, and `v2026.5.22` does not include that strict-schema key.

## Solution Assessment

Unclear until maintainers choose the docs compatibility model. The durable fix is not another one-off docs edit; it is a release/docs policy and guardrail that prevents the next strict-schema mismatch.
