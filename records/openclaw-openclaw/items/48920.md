## Summary

Keep open: the original `isolatedSession` mismatch is now shipped, but the same live-docs-versus-stable-release failure class remains source-reproducible with `tools.toolSearch.mode: "directory"` after v2026.6.6.

## What This Changes

The issue reports that live docs documented heartbeat `isolatedSession` before the reporter's installed release accepted that config key, causing strict config validation failure and Telegram disruption.

## Best Possible Solution

Define an explicit compatibility policy for `docs.openclaw.ai`, then implement the matching workflow/site metadata and generated config-docs checks so latest-stable users are not guided into unsupported config.

## Reproduction Assessment

Yes, at source level. Current main documents and accepts `tools.toolSearch.mode: "directory"`, while the latest stable tag v2026.6.6 accepts only `code` and `tools`; I did not run a packaged install because this review is read-only.

## Solution Assessment

Unclear as an implementation path. A one-off docs edit is not the best durable fix; maintainers need to choose release-pinned docs, version badges/gating, latest-only docs with warnings, version-aware diagnostics, or a combined policy.
