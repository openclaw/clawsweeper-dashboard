## Summary

Keep open. The original `isolatedSession` release mismatch is now shipped, but the live-docs-ahead-of-release failure class is still reproducible: live docs now describe `tools.toolSearch.mode: "directory"` while the latest stable tag rejects that mode.

## What This Changes

The issue asks OpenClaw to prevent live documentation from describing config keys or features that are not supported by the latest released package.

## Best Possible Solution

Choose and implement a release/docs compatibility policy that keeps latest-release users from following public docs into unsupported config or CLI surfaces, while linking the broader bundled-docs work as complementary follow-up.

## Reproduction Assessment

Yes, at source level. Live docs currently advertise `tools.toolSearch.mode: "directory"`, while the latest stable tag's schema accepts only `code` and `tools`; I did not run a packaged install because this review is read-only.

## Solution Assessment

Unclear as an implementation path. A one-off docs edit is not the best durable fix; maintainers need to choose release-pinned docs, version badges/gating, latest-only docs with clear warnings, version-aware diagnostics, or a combined policy.
