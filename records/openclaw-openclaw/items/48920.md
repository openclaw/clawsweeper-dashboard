## Summary

Keep open. The original `isolatedSession` failure is shipped now, but the broader live-docs-versus-release drift remains reproducible because the public docs still mirror current main and can advertise post-release config keys that latest stable releases reject.

## What This Changes

The issue asks for live docs to stay aligned with released OpenClaw versions or visibly mark version availability so documented config keys do not fail on latest stable.

## Best Possible Solution

Define the public docs compatibility policy, then implement matching publish/version metadata and release-compatibility checks while coordinating bundled local docs work with https://github.com/openclaw/openclaw/issues/71301.

## Reproduction Assessment

Yes. Source and live-site inspection show `docs.openclaw.ai` exposes current-main per-agent bootstrap keys, while the latest stable tag `v2026.5.12` has a strict agent schema without those keys.

## Solution Assessment

No. Closing as implemented would only account for the original heartbeat key; the maintainable fix needs an explicit docs/release compatibility policy and matching workflow or validation guard.
