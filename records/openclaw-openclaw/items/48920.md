## Summary

Keep open. The original `isolatedSession` heartbeat mismatch is shipped in the latest release now, but the live-docs release policy is still unresolved: the publish workflow mirrors `main` docs to the public docs repo and records source SHA metadata, not release compatibility metadata.

## What This Changes

The issue asks OpenClaw to keep live documentation aligned with released versions and make docs versioning clear so users do not copy unsupported config keys.

## Best Possible Solution

Define the public docs compatibility policy, then implement matching release/version metadata, publish gating or version badges, and config-doc compatibility checks while coordinating but not collapsing this into https://github.com/openclaw/openclaw/issues/71301.

## Reproduction Assessment

Yes at source level. The workflow still publishes docs from `main` and records source SHA metadata without release compatibility data; the exact `isolatedSession` rejection is no longer current in `v2026.5.18`.

## Solution Assessment

Unclear until maintainers choose the docs compatibility model. Closing as implemented would cover only the original heartbeat key, while the maintainable fix needs an explicit live-docs release policy and guardrails.
