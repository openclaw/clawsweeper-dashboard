## What This Changes

The issue reports live docs documenting config that the installed release did not accept, causing strict config parsing to fail after users followed the docs.

## Best Possible Solution

Adopt a maintainer-approved docs/release policy that makes public docs clearly version-coupled to stable releases or visibly marks unreleased current-main features, with https://github.com/openclaw/openclaw/issues/71301 as the broader bundled-docs design track.

## Reproduction Assessment

Yes for the docs/release drift class: current main docs and CLI source now contain unreleased cron command-job behavior while the latest release tag does not. The original `isolatedSession` example itself is no longer failing in `v2026.6.1` because that key has shipped.

## Solution Assessment

No automatic code fix is clearly best yet; the durable solution is a maintainer-selected docs versioning or deploy policy rather than another one-off schema shim. The related bundled-docs proposal may be part of that solution, but it is not merged or shipped.
