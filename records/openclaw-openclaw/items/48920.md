## Summary

Keep open. The original heartbeat `isolatedSession` mismatch has shipped, but the broader live-docs-versus-latest-release failure is still source-reproducible with current main docs/config changes made after `v2026.5.18`.

## What This Changes

The issue reports that live public docs can describe config keys that are not accepted by the latest released OpenClaw version, causing strict config parsing failures for users following the docs.

## Best Possible Solution

Define an explicit compatibility policy for `docs.openclaw.ai`, then implement matching release metadata, version badges or publish gating, and generated config-doc compatibility checks; treat https://github.com/openclaw/openclaw/issues/71301 as complementary bundled-agent-docs work rather than the sole replacement.

## Reproduction Assessment

Yes, at source level. Current main documents and accepts Telegram `nativeToolProgress` keys that are not present in the latest release tag, while the docs publish workflow still mirrors main docs rather than release-matched docs.

## Solution Assessment

Unclear until maintainers choose the docs compatibility model. The durable fix is not the individual heartbeat or Telegram key; it is the release/docs policy and guardrails that prevent the next strict-schema mismatch.
