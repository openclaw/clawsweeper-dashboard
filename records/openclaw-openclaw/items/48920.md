## Summary

Keep open. The original `isolatedSession` mismatch is fixed in the latest release, but the live-docs-versus-release class still source-reproduces with `tools.toolSearch`: current main documents and accepts it, while `v2026.5.7` does not ship that docs page or schema support, and the docs publish workflow still mirrors main pushes.

## What This Changes

The issue requests that live OpenClaw docs stay in step with released versions or clearly show version availability so users do not apply unsupported config keys.

## Best Possible Solution

Define the compatibility contract for `docs.openclaw.ai`, then implement the matching publish workflow, version metadata, generated docs/schema checks, and/or installed-version validation messages while coordinating with the bundled-docs work in https://github.com/openclaw/openclaw/issues/71301.

## Reproduction Assessment

Yes, from source inspection. Current main documents `tools.toolSearch`, while `v2026.5.7` does not ship that doc page and its strict config schema lacks `tools.toolSearch`, so following live docs on the latest release can still produce the reported unrecognized-key class.

## Solution Assessment

Unclear because there is no patch and the right fix depends on maintainer policy. Removing one docs page would address today’s example, but a maintainable solution needs a release/docs compatibility contract or version-aware diagnostics.
