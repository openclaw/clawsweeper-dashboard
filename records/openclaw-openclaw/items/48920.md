## Summary

Keep open. The original `isolatedSession` mismatch is resolved in the latest release, but the broader live-docs-versus-release drift remains reproducible: live/current heartbeat docs advertise `heartbeat.skipWhenBusy`, current main accepts it, and the latest release `v2026.4.27` does not.

## What This Changes

The issue asks OpenClaw to keep live docs aligned with released behavior, or clearly mark version availability, so users do not copy unsupported heartbeat config keys into released installs.

## Best Possible Solution

Define the compatibility contract for `docs.openclaw.ai`, then implement the matching publish workflow, version metadata, generated config-doc compatibility checks, and/or version-aware validation messages. Treat #71301 as complementary bundled-docs work, not a complete replacement for this public live-docs policy gap.

## Reproduction Assessment

Yes. The original `isolatedSession` reproduction is fixed in `v2026.4.27`, but the same class reproduces by comparing live/current heartbeat docs for `skipWhenBusy` with the strict `v2026.4.27` heartbeat schema that lacks that key.

## Solution Assessment

Unclear because there is no patch and the best fix depends on maintainer policy. A validation-only error would reduce confusion, but it would not fully solve live docs advertising unreleased config without a versioning or publishing contract.
