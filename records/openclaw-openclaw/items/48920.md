## Summary

Keep open. The original `isolatedSession` mismatch is shipped in the latest release, but current main has already added new docs/schema keys after `v2026.5.19` and the docs publish workflow still mirrors `main`, so the same live-docs-versus-latest-release strict-config failure class remains source-reproducible.

## What This Changes

The issue asks OpenClaw to keep live docs aligned with released builds or show version availability so users do not copy unreleased strict config keys into released configs.

## Best Possible Solution

Define an explicit compatibility policy for `docs.openclaw.ai`, then implement matching release metadata, version badges or publish gating, and generated config-doc compatibility checks; coordinate with https://github.com/openclaw/openclaw/issues/71301 without treating it as the sole replacement.

## Reproduction Assessment

Yes, at source level. Current main documents and accepts Discord voice config keys that are not present in `v2026.5.19`, while the docs sync workflow still publishes main-sourced docs rather than release-matched docs.

## Solution Assessment

Unclear until maintainers choose the docs compatibility model. The durable fix is not another one-off key update; it is a release/docs policy and guardrail that prevents the next strict-schema mismatch.
