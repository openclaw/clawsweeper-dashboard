## Summary

Keep open: the original `isolatedSession` mismatch is now shipped, but the live-docs-versus-stable-release drift remains source-reproducible. Current live docs advertise `toolResultMaxChars` values up to `1000000`, while the latest stable release schema still caps that config at `250000`, so a stable-release user can still follow public docs into unsupported config.

## What This Changes

The issue asks for docs.openclaw.ai to stop advertising unreleased configuration or feature behavior, or to show release/version context, after live heartbeat docs led released users into unsupported config.

## Best Possible Solution

Define the public docs compatibility policy, then implement the matching release-pinned or version-labeled docs workflow plus generated config-doc checks or diagnostics so stable-release users are not guided into unsupported setup/config.

## Reproduction Assessment

Yes, at source level. Live docs currently publish a main-only `toolResultMaxChars` range while the latest stable release schema still rejects that range.

## Solution Assessment

Unclear as an implementation path. A one-off rollback for the current config example is not the best durable fix; maintainers need to choose and implement a public docs versioning, release-gating, or version-labeling policy.
