Keep open. The original heartbeat `isolatedSession` mismatch is shipped in `v2026.5.3`, but the same release-drift class still reproduces: live docs are synced from current `main` and now document `openclaw models auth list`, while the latest release does not register that subcommand.

Request to keep live docs aligned with the latest release, or expose versioning, so released users do not follow unsupported config or CLI documentation.

## Best Possible Solution

Define the compatibility contract for `docs.openclaw.ai`, then implement the matching release-tag publish/version metadata/gating and any generated docs/schema or validation checks that prevent latest docs from misleading released users.
