## Summary

Keep open. The original `isolatedSession` mismatch is fixed in the latest release, but the live-docs-versus-release class still reproduces: current main documents `heartbeat_respond`, while `v2026.4.29` does not ship that tool or its docs, and the docs publish workflow still mirrors main rather than a release tag.

## What This Changes

The report asks for live docs to stay in sync with released OpenClaw versions or clearly show version availability for documented config and features.

## Best Possible Solution

Define the compatibility contract for `docs.openclaw.ai`, then implement the matching publish workflow, version metadata, generated docs/schema checks, and/or installed-version validation messages while coordinating with #71301 for bundled agent-facing docs.
