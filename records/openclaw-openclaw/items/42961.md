## Summary

Keep open: the PR addresses a valid current-main OpenRouter Anthropic caching bug, but the branch predates later endpoint-gating and prompt-cache refactors and should not merge as-is.

## What This Changes

The PR threads `cacheRetention` into OpenRouter Anthropic cache marker injection so `none` disables markers, `short` keeps the default marker, and `long` emits a 1-hour TTL, with focused unit tests.

## Best Possible Solution

Keep endpoint-based OpenRouter detection, make Anthropic marker injection retention-aware, add regression coverage for default/none/long, and include a user-facing changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure `cacheRetention: "none"` or `"long"` for an `openrouter/anthropic/*` model on a verified OpenRouter route, and current main still injects only `{ type: "ephemeral" }` markers.
