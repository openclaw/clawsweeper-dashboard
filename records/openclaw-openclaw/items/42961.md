## Summary

Keep open. Current main still ignores `cacheRetention` for OpenRouter Anthropic payload markers, and this PR remains useful source work, but the branch is stale against the current endpoint-gated wrapper and needs a changelog entry before merge.

## What This Changes

The PR threads `cacheRetention` into OpenRouter Anthropic cache marker injection so `none` suppresses markers, `short` keeps the default ephemeral marker, and `long` emits `ttl: "1h"`, with unit coverage for those cases.

## Best Possible Solution

Land a refreshed OpenRouter Anthropic retention fix on current main: resolve the effective `cacheRetention` for verified OpenRouter Anthropic routes, pass it into the endpoint-gated payload wrapper/helper, keep default short behavior, suppress markers for `none`, emit `{ type: "ephemeral", ttl: "1h" }` for `long`, preserve thinking-block cleanup, add focused tests, and include a user-facing changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to configure `cacheRetention: "none"` or `"long"` for an `openrouter/anthropic/*` model and inspect the current payload wrapper: it always calls the short-only marker helper when the OpenRouter guard passes.

## Solution Assessment

No for the current diff as written, because it predates current endpoint-class gating and shared helper layout. The direction is right, but the maintainable fix is a refresh onto current `prompt-cache-retention.ts`, `proxy-stream-wrappers.ts`, and `anthropic-payload-policy.ts` without widening proxy behavior.
