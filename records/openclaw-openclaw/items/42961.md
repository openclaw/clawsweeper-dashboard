## Summary

Keep this PR open. Current main still ignores `cacheRetention` on the OpenRouter Anthropic marker path, so the PR addresses a real remaining cost/caching bug, but the branch predates current provider-wrapper refactors and needs a careful refresh before merge.

## What This Changes

The PR makes OpenRouter Anthropic models resolve `cacheRetention`, suppresses cache markers for `none`, emits a one-hour `ttl` for `long`, keeps default short behavior, and adds regression tests for those retention values.

## Best Possible Solution

Land a refreshed retention-aware OpenRouter Anthropic implementation on current main: resolve OpenRouter Anthropic `cacheRetention`, pass it into the current endpoint-gated payload wrapper/helper, keep default short behavior, suppress markers for `none`, emit `{ type: "ephemeral", ttl: "1h" }` for `long`, preserve thinking-block cleanup, and document the behavior with focused regression tests and a changelog entry.

Yes. No for the current diff as-is, because it was written against older files and wrapper composition. The direction is correct, but the maintainable solution is a refresh onto current `prompt-cache-retention.ts`, `proxy-stream-wrappers.ts`, and the shared Anthropic marker helper while preserving endpoint gating.
