## Best Possible Solution

Land a focused SDK hardening patch after guarding unreadable DeepSeek union variants and keeping the provider-family helper/tests as the canonical shared path.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a readable DeepSeek anyOf/oneOf array containing an unreadable object variant is preserved by recursive normalization and then spread during merge, which can still throw.

## Solution Assessment

No. The shared guarded-reader approach is the right layer, but the PR needs one more guard/test around DeepSeek selected union variants before it is the best fix.
