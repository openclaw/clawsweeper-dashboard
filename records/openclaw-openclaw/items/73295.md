## Summary

Keep open. Current main still lacks the Discord-specific stuck-session circuit breaker, and the PR branch is useful but not merge-ready because the latest diff can leave later same-key Discord sessions unprotected and can drain recovery work before the aborted run has actually released the reply-run registry.

## What This Changes

The PR adds a Discord-only queued follow-up stuck-session circuit breaker, wires active-run liveness/abort callbacks into reply execution, exports the helper, and adds focused Vitest coverage.

## Best Possible Solution

Land a corrected, narrowly scoped Discord follow-up recovery path that refreshes or clears breaker timers with queue lifecycle, waits for abort settlement or force-clear before draining, preserves queued session state, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes. The stale-timer bug is reproducible with fake timers by arming a timer for one same-key Discord session, draining that queue before the threshold, then enqueueing a later stuck session; the abort race is reproducible by keeping the reply-run registry active after abortActiveRun returns true.

## Solution Assessment

No, not as written. The feature direction is reasonable, but the PR needs timer lifecycle cleanup/replacement and a settled abort-and-drain boundary before it is the narrow maintainable fix.
