## Summary

Keep open because current main and the latest release still lack per-job retry count/delay/backoff support for recurring cron failures, and the open implementation candidate at https://github.com/openclaw/openclaw/pull/51001 still needs maintainer review before this issue can close.

## What This Changes

Request recurring cron jobs to support configurable per-job retry count, retry delay, and optional backoff so failed daily jobs can re-run before the next natural schedule.

## Best Possible Solution

Review, repair, and land https://github.com/openclaw/openclaw/pull/51001 or an equivalent additive implementation with durable recurring retry policy, CLI/protocol support, persisted retry state, docs, generated clients, and focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows recurring failures still schedule against the natural next run/backoff floor while the requested CLI/protocol per-job retry controls are absent; no live cron job was run in this read-only review.

## Solution Assessment
