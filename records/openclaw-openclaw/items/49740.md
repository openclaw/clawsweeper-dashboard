## Summary

## What This Changes

Request configurable cron retry controls so recurring jobs retry after transient failures instead of waiting for the next scheduled run.

## Best Possible Solution

Keep the shipped global recurring transient retry behavior, and open a separate product-design issue only if maintainers want per-job retry override flags or a visible `retrying` list state.
