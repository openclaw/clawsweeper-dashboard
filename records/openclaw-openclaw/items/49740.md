## Summary

Keep open because current main and the latest release still lack per-job recurring cron retry controls, while the open closing PR at https://github.com/openclaw/openclaw/pull/51001 remains unmerged and conflicting.

## What This Changes

Add per-job retry options such as `--retry-count` and `--retry-delay` so failed recurring cron jobs can retry before the next natural schedule.

## Best Possible Solution

Review, repair, and land https://github.com/openclaw/openclaw/pull/51001 or an equivalent additive implementation with durable recurring retry policy, CLI/protocol support, persisted scheduler state, docs, and focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows recurring failures still schedule against the natural next run/backoff floor while the requested CLI/protocol retry controls are absent; no live cron run was executed in this read-only review.

## Solution Assessment

Yes for triage: keeping this issue open while the linked closing PR is repaired/reviewed is the right path. The eventual implementation should validate complete retry policy, preserve cadence after retry success/exhaustion, and make edit/one-shot semantics explicit.
