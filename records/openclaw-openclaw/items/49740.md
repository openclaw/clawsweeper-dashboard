## Summary

Keep open because current main still lacks configurable recurring cron retry controls, and the linked implementation candidate at https://github.com/openclaw/openclaw/pull/51001 is open and unmerged.

## What This Changes

Request per-job cron retry options such as `--retry-count`, `--retry-delay`, and optional backoff so failed recurring jobs retry before the next natural schedule.

## Best Possible Solution

Review, repair, and land https://github.com/openclaw/openclaw/pull/51001 or an equivalent additive implementation that defines durable recurring retry policy, CLI/protocol support, scheduler state, docs, and focused regression coverage.
