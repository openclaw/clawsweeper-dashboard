Keep open because current main still lacks configurable recurring cron retry controls, and PR #51001 is an open, unmerged implementation candidate that links this issue for closure.

## What This Changes

Add configurable retry controls for failed recurring cron jobs, including retry count/delay/backoff CLI options and scheduler retries before the next natural run.

## Best Possible Solution

Review and land PR #51001 or an equivalent implementation that adds durable per-job recurring retry configuration, CLI add/edit flags, protocol/schema support, scheduler state that retries before the natural schedule without cadence drift, docs, and focused regression tests.
