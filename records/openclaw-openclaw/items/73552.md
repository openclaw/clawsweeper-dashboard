## Summary

Keep open. Current main still performs a single SearXNG request with the requested category and returns a successful empty result set; the generic web-search fallback only advances on unavailable providers, thrown errors, or structured missing-key errors. The existing ClawSweeper keep-open context still matches the current code.

## What This Changes

The issue asks OpenClaw to retry category-specific SearXNG web searches with general search when the categorized response returns no results.

## Best Possible Solution

Add a SearXNG-provider-local retry: after an empty successful response for a non-general categorized request, retry once with categories set to general, while preserving endpoint validation, timeout, cache TTL behavior, language, count limits, wrapping, and existing error handling.

## Reproduction Assessment

Yes. A high-confidence reproduction is a mocked SearXNG response where categories is weather and the JSON body is {"results":[]}; current main builds one weather URL and returns count 0 without issuing a second general request.

## Solution Assessment

Yes. The provider-local retry is the narrowest maintainable solution because generic runWebSearch should continue treating successful zero-result payloads as valid for other providers.
