## Summary

Keep open. Current main still accepts and forwards SearXNG categories, performs one categorized request, and returns a successful zero-result payload without retrying the same query as `general`; the existing generic provider fallback only advances to another provider on unavailable tools, thrown errors, or structured missing-key errors.

## What This Changes

The issue asks OpenClaw to retry category-specific web searches as general searches when the category-specific request returns no results.

## Best Possible Solution

Add the retry in the SearXNG plugin client: after an empty categorized response, retry once with `categories` set to `general` when the original effective category set is non-empty and does not already include `general`, while preserving timeout, cache, endpoint validation, count, language, wrapping, and error behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a mocked `runSearxngSearch` flow with `categories: "weather"` where SearXNG returns `{"results":[]}`; current code builds one categorized URL and returns `count: 0` without a second general request.

## Solution Assessment

Yes. A provider-local retry is the narrowest maintainable fix because zero results can be valid in the generic provider loop, while this report is about SearXNG category-engine compatibility.
