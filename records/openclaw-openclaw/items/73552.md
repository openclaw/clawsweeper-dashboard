## Summary

Keep open. Current main still sends a single SearXNG request with the requested category and returns a successful empty result set without retrying `general`; the generic web-search fallback only advances on unavailable providers, thrown failures, or structured missing-key errors.

## What This Changes

Request automatic fallback from a category-specific web search to general search when a category search returns no results.

## Best Possible Solution

Add a SearXNG-provider-local retry after an empty categorized response: retry once with `categories` set to `general` when an effective category was requested and does not already include `general`, while preserving endpoint validation, timeout, cache TTL, language, count limits, result wrapping, and existing error behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a mocked SearXNG endpoint where the first request with `categories=weather` returns `{ "results": [] }` and a second `categories=general` request would return results; current code performs only the first request and returns `count: 0`.

## Solution Assessment

Yes. Provider-local fallback is the narrowest maintainable fix because a zero-result payload can be a valid successful search for other providers, while this report is specifically about SearXNG category-engine compatibility.
