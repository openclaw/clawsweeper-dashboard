## Summary

Keep open. Current main still performs a single SearXNG request with the requested category and treats a successful empty result set as terminal; the requested category-to-general retry is not implemented, and the issue is recent, unprotected, and narrow enough for a focused repair PR.

## What This Changes

The issue requests that category-specific web search retry as general search when the provider returns no category results.

## Best Possible Solution

Add a SearXNG-provider-local retry: after a successful empty response for a non-general categorized request, retry once with categories set to general, while preserving endpoint validation, timeout, cache TTL, language, count limits, content wrapping, and existing error handling.

## Reproduction Assessment

Yes. A deterministic mocked SearXNG response where categories is weather and the JSON body is {"results":[]} should show current main issuing one weather request and returning count 0 without a second general request.

## Solution Assessment

Yes, if implemented inside the SearXNG plugin. A generic core fallback on empty results would be broader and less correct because zero results can be a valid successful response for other providers.
