## Summary

Keep open. Current main still sends one SearXNG category request and returns a successful empty result set without retrying as general search, while generic web-search fallback only advances on provider unavailability, thrown errors, or structured availability errors.

## What This Changes

Request that category-specific web searches retry the same query as general search when the category request returns zero results.

## Best Possible Solution

Implement a one-time retry inside the SearXNG client for non-general categorized empty results, preserving existing cache, count, language, timeout, endpoint guard, and error handling behavior.

## Reproduction Assessment

Yes. A mocked SearXNG JSON response of {"results":[]} for a weather-category request should show current main making one categorized request and returning count 0 without a second general request.

## Solution Assessment

Yes. The narrowest maintainable solution is a SearXNG-local retry because core runWebSearch should not globally reinterpret every successful zero-result provider response as a failure.
