## Summary

Keep open: this is useful work toward the memory-core dreaming bug, but the latest head is not merge-ready while the revised gate is not shared with diagnostics/status, the required changelog entry is absent, and the widened containment relocation still has an adjacent-content over-capture path.

## What This Changes

The PR changes memory-core dreaming sorting, promotion gate calculations, relocation matching, docs, and regression tests so daily-only signals can score candidates but no longer satisfy `minRecallCount`.

## Best Possible Solution

Land a repaired version that keeps the recall-first ordering, aligns diagnostics/status/docs/changelog with the final promotion contract, and tightens quality-2 relocation so promotion never persists unrelated adjacent daily-note text.

## Reproduction Assessment

Yes. Source inspection of current main shows the light dreaming sort prioritizes recency before recall count and promotion gates compare a daily-inclusive aggregate to `minRecallCount`; this review used source proof rather than executing tests.

## Solution Assessment

No. The implementation direction is narrow, but the PR is not the best complete solution until explain/status output, changelog, and the remaining quality-2 relocation over-capture case are fixed and maintainers accept the gate contract reversal.
