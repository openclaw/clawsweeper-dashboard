## Summary

Keep this PR open: it addresses a real memory-core ranking/gating bug, but the latest patch is not merge-ready because the new rank/apply gate is not reflected in promote-explain or status diagnostics, and the user-facing fix still lacks a changelog entry.

## What This Changes

The PR changes memory-core light dreaming ordering, promotion gate counting, live-note relocation matching, related regression tests, and memory/dreaming docs so daily-only signals can score candidates but no longer satisfy `minRecallCount`.

## Best Possible Solution

Land one aligned memory-core fix after maintainer acceptance of the gate semantics: recall-first light ordering, daily-only exclusion from the gate, grounded evidence handled consistently in rank/apply/explain/status/docs, a changelog entry, and changed-lane validation.
