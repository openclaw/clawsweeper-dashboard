## Summary

Keep open because the protected `maintainer` label prevents automatic cleanup, but the branch should not merge as-is: current main and merged https://github.com/openclaw/openclaw/pull/86578 define cron `session:<id>` values as opaque runtime session keys, while this PR reintroduces path-safe validation that would break valid separator-bearing targets.

## What This Changes

The PR adds a cron custom-session validator that rejects `/`, `\`, NUL, and `SAFE_SESSION_ID_RE`-incompatible custom targets during cron normalization and gateway cron execution, with regression tests for fallback behavior.

## Best Possible Solution

Close this PR as superseded by merged https://github.com/openclaw/openclaw/pull/86578, keeping opaque cron session targets and strict run-log job-id validation.

## Reproduction Assessment

Yes, source-level. The PR head routes custom `session:<id>` values through a path-safe/`SAFE_SESSION_ID_RE` validator, while current main and release tests preserve separator-bearing opaque session targets; I did not run tests because this review is read-only.

## Solution Assessment

No. The best fix is the merged https://github.com/openclaw/openclaw/pull/86578 shape: treat `sessionTarget` as an opaque runtime routing key and enforce path safety only where cron job IDs are used for run-log paths.
