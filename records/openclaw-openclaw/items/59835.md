## Summary

Current `main` already ships the cron session-target hardening this PR is trying to add, via a slightly different implementation: create/update paths reject unsafe custom session IDs, persisted unsafe jobs are allowed to load only so run-time can fail closed, and colon-delimited custom session IDs remain intentionally supported.

## Evidence
