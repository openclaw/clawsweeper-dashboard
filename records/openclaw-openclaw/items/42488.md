## Summary

Keep open: current main does not implement `old_text`/`new_text` aliases; the edit tool now requires the newer `edits[]` payload and has an explicit test rejecting legacy top-level alias parameters. Because the requested fix targets an older contract, maintainer judgment is needed on whether to add a compatibility adapter for the current schema.

## What This Changes

Accept `old_text` and `new_text` as edit-tool parameter aliases for models that emit that snake_case shape.

## Best Possible Solution

Decide whether OpenClaw should tolerate legacy top-level edit aliases by converting them into the current `edits[]` payload, or close the request intentionally if the strict current schema is the desired contract.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that top-level `old_text`/`new_text` remain unsupported on current main because validation requires `path` plus `edits[]`.

## Solution Assessment

Unclear. The proposed March implementation targets removed `CLAUDE_PARAM_GROUPS`/schema-alias code; if maintainers want compatibility now, the narrower path is an adapter around the current `edits[]` contract.
