## Summary

Keep open. Current main still has a fixed built-in gateway role set plus scoped operator tokens, but the requested custom named roles, `gateway.roles` config, pairing role assignment, and `openclaw roles create` surface remain unimplemented and security-sensitive product work.

## What This Changes

The issue requests custom gateway roles with user-defined scope combinations, config support, pairing assignment, and CLI role creation.

## Best Possible Solution

Decide whether OpenClaw should support custom gateway RBAC; if yes, add a reviewed versioned role schema, role-to-scope resolver, pairing/rotation semantics, CLI/docs/protocol updates, and regression tests that prevent privilege widening.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection is high confidence because custom role names are rejected today and the requested config/CLI surfaces are absent.

## Solution Assessment

Unclear. Scoped operator tokens cover part of the least-privilege use case, but custom named roles would be a broader authorization product decision rather than a narrow mechanical fix.
