## Summary

Keep open. Current main still has fixed gateway roles plus scoped operator tokens, but custom named roles, `gateway.roles` config, pairing-time role assignment, and an `openclaw roles create` surface remain unimplemented security-sensitive feature work.

## What This Changes

Add configurable custom gateway roles with user-defined scope sets, pairing role assignment, and role-management CLI commands.

## Best Possible Solution

Decide the gateway RBAC product and security model; if accepted, add a versioned `gateway.roles` schema, fail-closed role-to-scope resolver, pairing/rotation semantics, CLI/docs/protocol updates, and regression tests for privilege containment.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection confirms the requested role, config, pairing, and CLI surfaces are missing on current main.

## Solution Assessment

Unclear. Existing scoped operator tokens cover part of the least-privilege use case, but custom named RBAC is broader authorization design that needs maintainer security/product direction before implementation.
