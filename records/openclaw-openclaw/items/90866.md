## Summary

Keep open: current main and the latest release cover Discord typing keepalive and event-driven progress drafts, but they still do not implement the central early speculative Discord placeholder posted in the inbound ACK window.

## What This Changes

Request Discord to show immediate in-chat thinking feedback, including a typing heartbeat and an editable placeholder that can show tool progress before the final reply.

## Best Possible Solution

Reuse the existing Discord draft/progress lifecycle, but add a maintainer-approved rule for when progress mode may create an accepted-message placeholder before work events and how it is finalized or removed.

## Reproduction Assessment

Not applicable as a strict bug reproduction, but yes at source level for the missing feature: current Discord accepted-message handling prestarts typing and event-driven drafts, not an unconditional ACK-window placeholder.

## Solution Assessment

No, not as written: the issue bundles already-implemented heartbeat/tool-progress behavior with the still-missing speculative placeholder. The maintainable path is to extend the existing Discord progress-draft lifecycle rather than add a parallel placeholder system.
