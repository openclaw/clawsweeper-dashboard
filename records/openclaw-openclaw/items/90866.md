## What This Changes

The issue requests Discord visible-feedback parity: keep typing alive during long thinking, post an early Thinking placeholder, edit it into the final output, and optionally show tool progress.

## Best Possible Solution

Have maintainers decide whether Discord progress mode should start the existing draft preview at inbound acceptance or reply start, then implement that through the current draft/finalization path with live Discord proof.

## Solution Assessment
