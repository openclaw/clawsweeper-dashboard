## Summary

Keep open: current main and latest release cover the Discord typing heartbeat and event-driven editable progress drafts, but they still do not implement the central ACK-window editable “Thinking…” placeholder for text-only accepted turns. The remaining work is a visible Discord UX/product decision about channel noise, cleanup, and when an early temporary message should appear.

## What This Changes

Request Discord parity for long-running turns by keeping typing alive, posting an early editable “Thinking…” placeholder, and optionally updating that placeholder with tool progress.

## Best Possible Solution

Extend the existing Discord progress-draft path, if maintainers approve the UX, so the same temporary message lifecycle owns the early placeholder, progress edits, final replacement, and cleanup instead of adding a parallel placeholder system.

## Reproduction Assessment

Yes, at source level: current main prestarts typing and supports progress drafts, but the draft stream starts from progress callbacks rather than an unconditional inbound ACK-window placeholder. A live Discord run is still needed to prove the exact timing target for a future implementation.

## Solution Assessment

No, not as a separate placeholder system. The maintainable direction is to extend or explicitly start the existing Discord progress-draft stream because it already owns temporary message editing, cleanup, and finalization.
