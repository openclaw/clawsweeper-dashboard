## Summary

Keep open: current main covers the typing heartbeat and Discord progress-draft/tool-progress pieces, but it still intentionally does not create the issue's central early speculative placeholder for text-only accepted turns. The remaining behavior is a Discord product/UX decision, not a stale or already-implemented bug report.

## What This Changes

The issue requests Discord parity with Slack-style immediate Thinking feedback: typing heartbeat, an early editable placeholder, and optional tool progress in that placeholder.

## Best Possible Solution

Make a maintainer product decision on whether Discord progress mode should start the existing draft stream at accepted-message time or reply-start time, then implement that through the current draft-preview controller rather than a parallel placeholder path.

## Reproduction Assessment

Yes, source-level: current main explicitly keeps text-only accepted Discord progress-mode turns from starting a draft, while the issue asks for an ACK-window placeholder. A live Discord run would still be needed to measure exact timing and cleanup behavior.

## Solution Assessment

No, not as a direct implementation request: the issue bundles already-covered typing/progress pieces with the missing early placeholder. The best path is to decide whether the existing Discord progress draft should start earlier and reuse that controller if maintainers want the behavior.
