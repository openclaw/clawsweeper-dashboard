## Summary

Keep open: current main and v2026.6.9 cover Discord typing prestart/keepalive and event-driven editable progress drafts, but they still preserve no draft for text-only accepted turns, which is the remaining early placeholder request.

## What This Changes

The issue asks Discord to add a persistent typing heartbeat, an early editable "Thinking..." placeholder, and optional tool-progress edits for parity with Slack-style placeholder UX.

## Best Possible Solution

If maintainers want this UX, extend the existing Discord draft-preview/finalization controller to materialize an approved early placeholder for accepted text-only turns, with explicit default/config and cleanup semantics.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.9 explicitly do not start Discord progress drafts for text-only accepted turns, while typing keepalive and event-driven progress drafts already exist. I did not run live Discord proof in this read-only review.

## Solution Assessment
