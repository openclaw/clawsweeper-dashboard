## Summary

Keep open: current main now covers the typing heartbeat and Discord progress-draft/tool-progress pieces, but the central always-early ACK-window speculative placeholder is still missing and current tests intentionally keep text-only accepted turns from starting a progress draft.

## What This Changes

Request Discord to keep typing feedback alive and add an immediate editable “Thinking…” style placeholder, with optional tool-call progress, for parity with Slack’s proposed placeholder UX.

## Best Possible Solution

Reuse the existing Discord progress-draft lifecycle and decide whether progress mode should start a temporary draft at accepted inbound or reply start, with cleanup/final-edit coverage and live Discord proof.

## Reproduction Assessment

Yes, at source level: current main starts Discord progress drafts from work/progress callbacks and explicitly tests that text-only accepted turns do not start a progress draft. I did not run a live Discord timing repro, so the exact ACK-window latency still needs real transport proof.

## Solution Assessment

No, not exactly as written: the issue bundles already-shipped typing heartbeat and tool-progress behavior with the still-missing early placeholder. The best solution is a maintainer-approved extension of the existing Discord progress-draft path, not a parallel placeholder system.
