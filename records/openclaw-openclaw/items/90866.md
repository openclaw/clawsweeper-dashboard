## Summary

Keep open: current main and v2026.6.8 cover Discord typing keepalive and event-driven progress drafts, but they still deliberately do not start an editable draft for text-only accepted turns, so the central ACK-window placeholder request remains a product decision.

## What This Changes

The issue requests Discord typing keepalive plus an early editable Thinking/progress placeholder for parity with Slack-style perceived-latency feedback.

## Best Possible Solution

If maintainers want this UX, extend the existing Discord progress-draft path with an approved early lifecycle trigger and reuse its current edit, finalization, and cleanup behavior rather than adding a parallel placeholder system.

## Reproduction Assessment

Yes, source-level: current main and v2026.6.8 explicitly do not start Discord progress drafts for text-only accepted turns, while the requested behavior is an early editable placeholder for those silent gaps. A live Discord run was not performed in this read-only review.

## Solution Assessment

No as a separate placeholder stack. The maintainable direction is to extend the existing Discord progress-draft stream at a maintainer-approved early lifecycle point, because that path already owns temporary message editing, finalization, and cleanup.
