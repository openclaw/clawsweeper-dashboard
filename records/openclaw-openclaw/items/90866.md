## Summary

Keep open: current main and latest release cover the Discord typing heartbeat plus event-driven progress draft/tool-progress pieces, but they still do not implement the central ACK-window editable “Thinking…” placeholder. The remaining work is a Discord UX/product decision because it changes visible channel noise and cleanup expectations.

## What This Changes

Request Discord to add an early speculative “Thinking…” placeholder, keep typing alive during long thinking, and optionally edit the placeholder with tool-call progress for parity with Slack.

## Best Possible Solution

Extend the existing Discord progress-draft path, if maintainers want this UX, so the temporary message starts at an approved early lifecycle point and reuses the current edit/finalize/cleanup behavior.

## Reproduction Assessment

Yes, source-level: current main prestarts typing and supports progress drafts, but the draft stream starts from progress callbacks rather than an unconditional inbound ACK-window placeholder. A live Discord run is still needed to prove the exact timing target for a future implementation.

## Solution Assessment

No, not as a parallel placeholder system. The maintainable direction is to extend or explicitly start the existing Discord progress-draft stream, because it already owns temporary message editing, cleanup, and finalization.
