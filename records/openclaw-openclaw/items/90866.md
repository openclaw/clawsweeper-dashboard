## Best Possible Solution

Reuse the existing Discord progress-draft controller and make an explicit maintainer decision on whether `progress` mode should start a temporary draft at inbound acceptance or reply start, with cleanup/final-edit behavior and live Discord proof preserved.

## Solution Assessment
