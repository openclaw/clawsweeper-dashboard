## Summary

Keep open: current main still has Discord typing keepalive and event-driven progress drafts, but it does not implement the requested ACK-window editable “Thinking…” placeholder for text-only accepted Discord turns.

## What This Changes

Request to add Discord immediate visible thinking feedback: a typing heartbeat plus an editable placeholder/progress message that finalizes into the reply.

## Best Possible Solution

If maintainers approve the UX, extend the existing Discord progress-draft stream with an early accepted-turn placeholder trigger and reuse its current edit, finalization, and cleanup behavior.

## Reproduction Assessment

Yes, source-level: current main explicitly keeps Discord progress drafts event-driven and tests that text-only accepted turns do not start a draft. I did not run a live Discord session in this read-only review.

## Solution Assessment

No as a separate placeholder stack. The maintainable path is to extend the existing Discord progress-draft lifecycle at a maintainer-approved early trigger, because that code already owns temporary message creation, edits, finalization, and cleanup.
