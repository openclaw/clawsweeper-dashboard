The issue requests Discord parity with Slack-style early speculative thinking feedback, including a typing heartbeat and an optional placeholder/progress message while an agent is reasoning.

## Best Possible Solution

Reuse the existing Discord progress-draft controller if maintainers want this UX, but start it from a chosen early lifecycle point with live proof for timing, cleanup, and final edit behavior instead of adding a parallel placeholder path.
