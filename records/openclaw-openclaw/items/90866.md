## Summary

Keep open: current main and v2026.6.6 cover Discord typing heartbeat and event-driven progress drafts, but they still do not implement the central ACK-window speculative placeholder for accepted text-only turns.

## What This Changes

The issue requests Discord typing keepalive plus an immediate editable "Thinking..." placeholder with optional tool-progress updates, matching the Slack placeholder proposal.

## Best Possible Solution

Have maintainers decide whether Discord progress mode should start an immediate temporary draft at inbound acceptance or reply start, then reuse the existing draft stream and finalization path with live Discord proof.

## Reproduction Assessment

Yes, source-level: current main explicitly keeps text-only accepted Discord turns from starting a progress draft, while progress drafts start from work/reasoning/tool callbacks. Live Discord timing is still needed to validate any future <=3s target.

## Solution Assessment

No, not as written: the issue bundles already-solved heartbeat/tool-progress behavior with the still-missing early placeholder. The narrow maintainable path is to extend the existing Discord progress draft lifecycle rather than add a parallel placeholder system.
