## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Preserve wakeups that arrive during an active drain:** `extensions/telegram/src/polling-session.ts:1157`
  - body: `requestImmediateDrain()` calls `drainOnce()` directly, but `drainOnce()` returns immediately while `drainActive` is true. If the worker writes an update after the active drain has already listed the spool but before it exits, both the post-write and `spooled` wakeups can be dropped and the new update still waits for the periodic drain, which is the latency/stall class this PR is meant to remove. Please record a pending drain request and run another drain as soon as the current one finishes.
  - confidence: 0.82
