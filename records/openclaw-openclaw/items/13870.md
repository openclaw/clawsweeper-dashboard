## Summary

Keep open with high confidence. Current main and latest release still only expose client-supplied `displayName` or the raw device id for paired devices, while the requested durable operator-owned label, rename RPC, CLI command, and Control UI action are still absent; related work at https://github.com/openclaw/openclaw/issues/23916 and https://github.com/openclaw/openclaw/pull/57449 covers broader display metadata, not the rename workflow.

## What This Changes

Request to let operators assign and display human-friendly names for paired devices in the CLI and Control UI.

## Best Possible Solution

Add an additive paired-device label workflow with a durable operator-owned label separate from reconnect metadata, expose it through Gateway protocol, CLI, Control UI, docs, and tests, and keep broader metadata display as separately tracked work.

Not applicable as a bug reproduction; this is a feature request. Source, docs, current-main, and release inspection give high confidence that the requested paired-device rename workflow is still absent.

## Solution Assessment

Yes. The maintainable path is an additive operator-owned label field and rename method; reusing client-supplied `displayName` is not sufficient because reconnect metadata can overwrite it.
