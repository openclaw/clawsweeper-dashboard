## What This Changes

The issue requests human-friendly names for paired devices in the web UI and `openclaw devices list`, including a rename command and durable display in device lists.

## Best Possible Solution

Land or replace the linked narrow implementation that adds a durable operator-owned paired-device label across Gateway protocol, CLI, Control UI, docs, and authorization coverage while keeping broader metadata display separate.

## Reproduction Assessment

## Solution Assessment

Yes. A separate operator-owned label with explicit precedence over client displayName is the narrow maintainable solution; reusing displayName would let reconnect metadata overwrite the user label.
