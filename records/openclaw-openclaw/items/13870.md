The issue requests user-assignable, human-friendly names for paired devices in the web UI and `openclaw devices list`.

Add an additive paired-device label workflow with a durable operator-owned label separate from reconnect metadata, expose it through Gateway protocol, CLI, and Control UI, and document display precedence.
