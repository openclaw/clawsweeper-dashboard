The issue requests human-friendly, user-assigned labels for paired devices in the web UI and `openclaw devices list`.

## Best Possible Solution

Add an additive paired-device label workflow with a durable operator-owned label separate from reconnect `displayName`, expose it through Gateway protocol, CLI, and Control UI, and document display precedence while keeping broader metadata display separate.

## Solution Assessment

Yes. A separate operator-owned label is the maintainable path because reconnect metadata currently owns `displayName` and can update it independently.
