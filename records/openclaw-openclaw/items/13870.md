## What This Changes

Request to let operators assign and display human-friendly labels for paired devices in the CLI and Control UI instead of relying on cryptographic IDs or reconnect metadata.

## Best Possible Solution

Add an additive paired-device label workflow with a durable operator-owned label separate from reconnect displayName, expose it through Gateway protocol, CLI, and Control UI, and document display precedence.

## Reproduction Assessment

Yes for the absence of the requested capability. Source inspection on current main shows no durable operator label, paired-device rename method, CLI command, or Control UI action.

## Solution Assessment

Yes. A separate operator-owned label is the maintainable path because reconnect metadata currently owns displayName and can update it independently.
