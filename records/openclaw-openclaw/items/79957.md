## Summary

Keep open: current main still has the duplicate OpenRouter DeepSeek V4 reasoning-field path, and an open linked fix PR already references this issue with closing syntax, so the issue should stay open until that PR lands or is closed.

## What This Changes

The issue reports that OpenRouter DeepSeek V4 Flash requests with thinking enabled send both `reasoning_effort` and `reasoning.effort`, causing OpenRouter to return HTTP 400.

## Best Possible Solution

Land or close the linked fix PR after maintainers accept its runtime proof and regression coverage, leaving current main with exactly one reasoning-effort representation for OpenRouter DeepSeek V4.

## Reproduction Assessment

Yes. Source inspection shows a non-off OpenRouter DeepSeek V4 request can receive nested `reasoning.effort` from the OpenRouter path and then top-level `reasoning_effort` from the DeepSeek V4 wrapper on current main.

## Solution Assessment

Yes. The narrow maintainable fix is to remove one representation in the OpenRouter DeepSeek V4 payload path and add coverage for the composed wrapper; https://github.com/openclaw/openclaw/pull/79961 already attempts that.
