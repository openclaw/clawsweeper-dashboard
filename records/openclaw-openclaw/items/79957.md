## Summary

Keep open: current main still has a source-reproducible path that can emit both OpenRouter nested reasoning and DeepSeek V4 top-level `reasoning_effort`; an open linked fix PR already declares it closes this issue, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks OpenClaw to send only one reasoning-effort field for `openrouter/deepseek/deepseek-v4-flash` thinking-enabled requests.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79961 or an equivalent narrow fix that guarantees OpenRouter DeepSeek V4 payloads contain exactly one reasoning effort field while preserving DeepSeek V4 replay shaping.

## Reproduction Assessment

Yes at source level: the OpenRouter wrapper can inject nested `reasoning.effort`, then the DeepSeek V4 wrapper can re-add top-level `reasoning_effort` before the payload reaches the transport. I did not run a live OpenRouter request.

## Solution Assessment

Yes in direction: the narrow maintainable fix is to normalize the DeepSeek V4 OpenRouter payload after the wrapper stack so only one reasoning-effort shape survives. The linked PR takes that direction and should be reviewed with focused regression coverage.
