## Summary

Keep open: current main still has a source-reproducible OpenRouter DeepSeek V4 duplicate-reasoning payload path, and an open unmerged closing PR tracks the narrow fix, so the issue should stay open until that PR lands or is replaced.

## What This Changes

OpenRouter DeepSeek V4 thinking requests should stop sending both top-level `reasoning_effort` and nested `reasoning.effort` in one payload.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79961 or an equivalent narrow fix that emits exactly one reasoning-effort representation for OpenRouter DeepSeek V4 and adds composed-wrapper regression coverage.

## Reproduction Assessment

Yes, at source level: current main composes the OpenRouter reasoning wrapper with the DeepSeek V4 wrapper, so non-off thinking can leave nested `reasoning.effort` and then add top-level `reasoning_effort`. I did not run a live OpenRouter request.

## Solution Assessment

Yes, the narrow maintainable fix is to make the final OpenRouter DeepSeek V4 payload choose exactly one reasoning-effort field. The open linked PR takes that direction, so the issue should remain open until that fix is merged or replaced.
