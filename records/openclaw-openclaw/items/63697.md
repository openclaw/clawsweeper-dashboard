## Summary

Keep this PR open. Current main still does not implement the central slash-command routing change, and the PR branch remains a live implementation candidate, but the branch has blocking correctness and security-sensitive review findings that make it unsafe to merge as-is.

## What This Changes

The PR changes the shared native Chat UI and iOS/macOS chat surfaces to forward reset/compact slash commands through `chat.send`, while also adding broader composer, model picker, tool-result rendering, splash, and iOS exec-approval UI changes.

## Best Possible Solution

Keep this PR open for maintainer review, but split or re-scope it into a focused shared native Chat UI change that forwards reset/compact slash commands through `chat.send` with Swift regression coverage. Broader UI polish and offline APNs expectations should be reviewed separately and coordinated with #67493.

## Reproduction Assessment

Yes. On current main, the high-confidence reproduction is the existing Swift view-model path: enter `/new` or `/compact` in `OpenClawChatViewModel`, call `send()`, and the current tests/source show `resetSession` or `compactSession` is called with no `chat.send` run id.

## Solution Assessment

No. The PR's core direction is sound, but the current branch is not the best mergeable solution because it bundles unrelated UI/auth changes and introduces blocking regressions; the safer fix is a smaller slash-routing patch that preserves reset/compact transcript semantics and existing operator-scope compatibility.
