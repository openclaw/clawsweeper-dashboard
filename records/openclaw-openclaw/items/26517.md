## Summary

Keep open. Current main still only partially satisfies the request: labeled session backend primitives and existing-session switching exist, but the visible WebChat New Session control still resets the current session with `/new`, and the requested in-WebChat create/manage flow plus agent-controlled create/relabel API remain unresolved. The provided GitHub context also shows open maintainer PR #73725 as the active create/switch candidate, so this issue should not be closed before that work lands or is explicitly superseded.

## What This Changes

Enable multiple isolated labeled WebChat sessions with create/switch UI, custom labels, and an agent-facing session label/create API while preserving the default main session behavior.

## Best Possible Solution

Keep this as the canonical WebChat multi-session tracker. Review and land or close PR #73725 for the focused New Chat to `sessions.create` create/switch flow, then explicitly split the remaining agent-controlled create/relabel API and broader session-management UI into scoped follow-ups.

The narrow maintainable first step is the open PR #73725 direction: create a dashboard session with `sessions.create`, switch WebChat to it safely, and leave broader agent API and session-management UI scope for explicit maintainer design.
