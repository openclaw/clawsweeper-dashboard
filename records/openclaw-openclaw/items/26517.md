## Summary

Current main implements the central WebChat multi-session request and it shipped in v2026.5.2: New Chat now creates a separate dashboard session through `sessions.create`, the selector renders labeled session rows, and session labels can be created or edited through the existing sessions APIs/UI.

## What This Changes

Request to add multiple isolated labeled WebChat sessions with create/switch UI and an agent-facing label/create concept.

## Best Possible Solution

Keep the shipped WebChat create/switch implementation and handle any future tab/sidebar or agent relabeling refinements as narrower follow-up requests.

## Reproduction Assessment

## Solution Assessment

Yes. The merged path uses the existing session control plane (`sessions.create`, `sessions.patch`, session list rows, and WebChat switching) instead of adding a parallel storage or bespoke `session_label_set` API, which is the narrower maintainable implementation for the central request.
