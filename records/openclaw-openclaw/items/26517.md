## Summary

Keep open. Current main still only partially implements the request: labeled session backend primitives and existing-session switching exist, but the visible WebChat New Session button still sends `/new` as a reset in the current session, the WebChat selector has no create/rename/delete flow, and the agent tool surface lacks the proposed create/relabel API. Open maintainer PR #73725 remains the active create/switch candidate and is unmerged in the provided GitHub context.

## What This Changes

Add multiple isolated labeled WebChat sessions with create/switch UI, custom labels, and an agent-facing session label/create API while preserving the default main session behavior.

## Best Possible Solution

Keep this issue as the canonical WebChat multi-session tracker. Review and land or close PR #73725 for the focused New Chat to `sessions.create` create/switch flow, then explicitly split or scope the remaining agent-controlled create/relabel API and broader rename/delete/sidebar/tab session-management work.
