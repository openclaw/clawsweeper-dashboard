## Summary

Keep open. Current main has direct Gateway HTTP/RPC tool invocation and SDK support, but the requested direct path is broader: coding tools such as exec/read/write are still not wired into the shared invoke resolver, cron only accepts agentTurn/systemEvent payloads, and there is no generic `/tool` or CLI direct-invoke surface; related #63919 and #18160 cover only parts of the remaining work.

## What This Changes

Add a first-class way to invoke OpenClaw tools directly from chat, CLI, or cron without sending the request through an LLM turn.

## Best Possible Solution

Keep this as the umbrella direct-invocation feature, review #63919 for HTTP coding-tool availability, and split cron/direct-exec plus generic slash or CLI surfaces into narrowly gated follow-ups that preserve the existing tool policy and deny-list model.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection gives a high-confidence current-state map: HTTP/RPC direct invoke exists, while cron direct payloads and generic slash/CLI invocation do not.

## Solution Assessment

No. Current main is a useful partial solution for Gateway/SDK clients, but it does not cover the requested coding-tool, cron, slash, and CLI surfaces; the maintainable path is to decompose and gate the remaining surfaces deliberately.
