## Summary

Keep open. Current main now has direct HTTP `/tools/invoke`, SDK/RPC `tools.invoke`, and deterministic skill slash-command dispatch, but the requested umbrella still has meaningful unresolved pieces: generic slash/CLI invocation, direct cron `toolCall` or `exec` payloads, and Gateway exposure of coding tools such as `exec` through the direct-invoke resolver.

## What This Changes

The issue requests a way to invoke OpenClaw tools directly from chat/slash commands, CLI, and cron without an LLM inference turn.

## Best Possible Solution

Keep the shipped HTTP/RPC/SDK and skill-command direct-invoke paths, then handle the remaining work as scoped, security-reviewed follow-ups for cron execution, CLI/slash UX, and Gateway coding-tool exposure.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection shows the current direct-invoke paths that exist and the requested cron, CLI, slash, and coding-tool pieces that remain absent.

## Solution Assessment

Unclear for the whole umbrella because it spans multiple public surfaces with different security models. The maintainable path is to land or reject scoped follow-ups such as https://github.com/openclaw/openclaw/issues/18160 and https://github.com/openclaw/openclaw/pull/63919 rather than treating one broad issue as a single fix PR.
