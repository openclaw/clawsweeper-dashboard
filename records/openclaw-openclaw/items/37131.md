## Summary

Keep open. Current main has direct HTTP `/tools/invoke`, SDK/RPC `tools.invoke`, and skill slash-command support, but the broad request still has meaningful remaining product and security work: generic slash/CLI tool invocation, direct cron `toolCall` or `exec` payloads, and coding-tool exposure through the Gateway direct-invoke resolver are not fully landed.

## What This Changes

Request direct invocation of OpenClaw tools from slash commands, CLI, and cron payloads without an LLM turn.

## Best Possible Solution

Keep the shipped HTTP/RPC/SDK direct-invoke surfaces, then split the remaining CLI, slash, cron, and coding-tool work into scoped security-reviewed follow-ups.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection confirms the existing direct-invoke surfaces and the missing cron, generic slash/CLI, and Gateway coding-tool pieces on current main.

## Solution Assessment

Unclear for the whole umbrella because it crosses HTTP, RPC, chat commands, CLI, cron, and high-risk tool policy. The maintainable path is to review scoped follow-ups such as https://github.com/openclaw/openclaw/issues/18160, https://github.com/openclaw/openclaw/pull/63919, and https://github.com/openclaw/openclaw/issues/67832 rather than queueing one broad fix.
