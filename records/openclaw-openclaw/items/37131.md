## Summary

Keep open. Current main has HTTP, RPC, and SDK direct tool-invoke foundations, but the broader request is not complete: generic slash/CLI direct invocation, direct cron tool payloads, and coding-tool exposure still need scoped product and security review.

## What This Changes

The issue requests invoking OpenClaw tools directly from chat, slash commands, CLI, and cron without routing through an LLM turn.

## Best Possible Solution

Keep the shipped HTTP/RPC/SDK direct-invoke surfaces, then decide the remaining slash, CLI, cron, coding-tool, and session-spawn pieces through scoped security-reviewed follow-ups.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives high confidence that HTTP/RPC/SDK direct invoke exists while generic slash/CLI direct invocation and direct cron tool payloads remain absent on current main.

## Solution Assessment

No for the umbrella as written: one broad direct-invoke path would cross too many trust boundaries at once. The maintainable path is scoped review of the existing follow-ups for coding tools, CLI plan execution, cron direct exec, and direct session spawn.

Status: needs_attention
