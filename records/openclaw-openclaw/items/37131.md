## Summary

Keep open. Current main has direct Gateway HTTP/RPC tool invocation and SDK support, but this umbrella request is only partially implemented: coding tools are still separate from the shared direct resolver, cron has no direct tool payload kind, and there is no generic `/tool` or core CLI direct-invoke surface.

## What This Changes

The issue requests a direct tool invocation path for deterministic OpenClaw tools through slash commands, CLI, cron payloads, or other explicit inputs without an LLM turn.

## Best Possible Solution

Keep this as the umbrella feature, review #63919 for HTTP coding-tool availability, keep #18160 as the cron direct-exec slice, and split any generic slash or CLI direct-invoke design into separately gated follow-ups.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection gives a high-confidence current-state map: direct HTTP/RPC invoke exists, while cron direct payloads, generic slash/CLI invocation, and coding-tool wiring are not fully present.

## Solution Assessment

No. Current main is a useful partial solution for Gateway and SDK clients, but the maintainable path is to decompose the remaining cron, coding-tool, slash, and CLI surfaces with the existing tool policy guardrails intact.
