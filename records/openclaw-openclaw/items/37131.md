## Summary

Keep open. Current main has HTTP/RPC/SDK direct invocation plus skill-specific slash command tool dispatch, but the broader request is still not complete: generic CLI/slash direct invocation, coding tools on the Gateway invoke surface, and cron toolCall/exec payloads remain unresolved and security-sensitive.

## What This Changes

Request a direct tool invocation path from chat/slash commands, CLI, and cron payloads so deterministic tool calls can bypass model inference.

## Best Possible Solution

Keep the shipped HTTP/RPC/SDK and skill-command direct dispatch, then settle the remaining CLI, coding-tool, cron, and session-spawn surfaces through scoped security-reviewed follow-ups.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives high confidence that some direct-invoke paths exist, while generic CLI/slash invocation, coding-tool exposure, and cron toolCall/exec payloads are still absent or pending on current main.

## Solution Assessment

Unclear for the umbrella request because it crosses Gateway HTTP/RPC, chat commands, CLI, cron, coding tools, and high-risk tool policy. The maintainable direction is to review scoped follow-ups rather than let automation choose the whole product/security boundary.
