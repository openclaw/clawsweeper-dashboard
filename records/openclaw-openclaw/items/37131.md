## Summary

Keep open. Current main has useful direct-invoke pieces, but the umbrella request is still only partially satisfied: generic slash/CLI direct tool invocation, direct cron tool/exec payloads, and coding-tool exposure through the Gateway direct-invoke resolver remain open and security-sensitive.

## What This Changes

Request direct OpenClaw tool invocation from chat/slash commands, CLI, and cron without a model turn.

## Best Possible Solution

Keep the shipped HTTP/RPC/SDK direct-invoke surfaces, then decide the remaining slash, CLI, cron, and coding-tool pieces through scoped security-reviewed follow-ups.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives high confidence on the current state: HTTP/RPC/SDK direct invoke exists, while generic slash/CLI invocation and direct cron tool payloads do not.

## Solution Assessment

Unclear for the whole umbrella because it crosses Gateway HTTP/RPC, chat commands, CLI, cron, coding tools, and high-risk tool policy. The maintainable path is to review scoped follow-ups such as https://github.com/openclaw/openclaw/pull/63919, https://github.com/openclaw/openclaw/pull/82232, https://github.com/openclaw/openclaw/issues/18160, and https://github.com/openclaw/openclaw/issues/67832.
