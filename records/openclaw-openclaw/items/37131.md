## Summary

Keep open. Current main has important direct-invoke foundations, but the request is still only partially satisfied and the remaining slash/CLI/cron/coding-tool surfaces require maintainer product and security scoping.

## What This Changes

The issue requests a generic way to invoke OpenClaw tools directly from chat commands, CLI, or cron payloads without an LLM turn.

## Best Possible Solution

Keep the shipped HTTP/RPC/SDK direct-invoke surfaces, then decide the remaining slash, CLI, cron, and coding-tool pieces through scoped security-reviewed follow-ups.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives high confidence that HTTP/RPC/SDK direct invoke exists, while generic slash/CLI direct invocation and direct cron tool payloads are still absent on current main.

## Solution Assessment

Unclear for the whole umbrella because it crosses Gateway HTTP/RPC, chat commands, CLI, cron, coding tools, and high-risk tool policy. The maintainable path is to review scoped follow-ups such as https://github.com/openclaw/openclaw/pull/63919, https://github.com/openclaw/openclaw/pull/82232, https://github.com/openclaw/openclaw/issues/18160, and https://github.com/openclaw/openclaw/issues/67832.
