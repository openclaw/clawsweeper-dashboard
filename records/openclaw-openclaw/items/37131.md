## Summary

Keep open: current main has useful partial no-LLM direct invocation through Gateway HTTP, SDK RPC, and skill-owned command dispatch, but the requested chat, CLI, cron, and coding-tool surfaces are still incomplete and need a security/product decision.

## What This Changes

Request to invoke OpenClaw tools directly from chat or slash commands, CLI flags, and cron payloads without an LLM turn.

## Best Possible Solution

Keep this as the umbrella direct-invocation design tracker until maintainers define the common trust model and either land the remaining chat/CLI/cron/Gateway slices or split them into canonical narrower issues.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high-confidence gap evidence: cron has no tool payload branch, openclaw agent has no --tool/--args mode, generic /tool chat dispatch is absent, and Gateway direct invocation still does not generally expose coding tools.

## Solution Assessment

No; the current implementation is only a partial solution to the umbrella request. The safer path is a maintainer-approved direct-invocation contract, then focused changes for Gateway coding tools, cron payloads, CLI flags, and any generic chat syntax.
