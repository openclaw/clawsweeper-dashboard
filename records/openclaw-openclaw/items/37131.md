## Summary

Keep open. Current main has direct Gateway HTTP/RPC and SDK tool invocation, but the requested umbrella still has meaningful missing pieces: coding tools are not wired into the current resolver, cron has no direct tool/exec payload kind, and generic slash/CLI invocation remains a product/security design choice.

## What This Changes

The issue requests deterministic direct invocation of OpenClaw tools from chat, slash command, CLI, or cron payloads without an LLM inference turn.

## Best Possible Solution

Keep the existing HTTP/RPC/SDK path, then land scoped, security-reviewed follow-ups for coding-tool exposure, cron direct execution, and any generic slash/CLI user surface.

## Reproduction Assessment

Not applicable. This is a feature request, and current-main source inspection gives a high-confidence state map: HTTP/RPC/SDK direct invoke exists, while cron, generic slash/CLI, and coding-tool availability are incomplete.

## Solution Assessment

No. The existing Gateway/RPC path is a useful partial implementation, but the maintainable solution is decomposed, security-reviewed slices rather than one broad direct-execution surface.
