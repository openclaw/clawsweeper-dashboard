## Security Review

Status: needs_attention

Summary: The issue is security-sensitive because it expands direct access to host, filesystem, session, message, and scheduled automation tools without model mediation.

Concerns:

- **[high] Preserve default-deny for dangerous direct tools:** `src/security/dangerous-tools.ts:9`
  - body: Current main default-denies high-risk HTTP tools including `exec`, `spawn`, `apply_patch`, `sessions_spawn`, `sessions_send`, `cron`, `gateway`, and `nodes`; any expansion needs explicit opt-in, owner semantics, hook execution, policy filtering, and auditability.
  - confidence: 0.93
- **[high] Define unattended cron execution policy first:** `src/gateway/protocol/schema/cron.ts:171`
  - body: Cron schemas currently allow only `systemEvent` and `agentTurn`; adding direct `toolCall` or `exec` payloads would schedule unattended host actions and needs explicit sandbox, approval, timeout, retry, output retention, cancellation, and delivery rules.
  - confidence: 0.89
