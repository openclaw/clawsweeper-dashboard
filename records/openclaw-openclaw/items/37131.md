## Summary

Keep open. Current main has direct Gateway HTTP/RPC and SDK tool invocation, plus direct skill-command dispatch for declared skills, but the requested umbrella still has meaningful missing pieces: cron has no direct tool/exec payload kind, generic slash/CLI invocation is not implemented, and coding-tool exposure remains an active product/security slice.

## What This Changes

The issue requests direct invocation of OpenClaw tools from chat or slash commands, CLI, and cron payloads without an intervening LLM turn.

## Best Possible Solution

Keep the existing HTTP/RPC/SDK and skill-command direct-invoke paths, then land scoped, security-reviewed follow-ups for cron direct execution, coding-tool exposure, and any generic slash/CLI user surface.

## Reproduction Assessment

Not applicable. This is a feature request, and current-main source inspection gives a high-confidence state map: HTTP/RPC/SDK direct invoke exists, while cron payloads, generic slash/CLI invocation, and parts of high-risk coding-tool exposure remain incomplete.

## Solution Assessment

No. The requested umbrella should not land as one broad direct-execution path; the maintainable direction is to keep the current Gateway/RPC implementation and split the remaining surfaces into security-reviewed product slices.
