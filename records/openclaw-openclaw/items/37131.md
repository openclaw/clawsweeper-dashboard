## Summary

Keep open. Current main has partial no-LLM direct invocation through Gateway HTTP and skill-owned command dispatch, but the umbrella request still remains across generic chat `/tool` syntax, CLI `--tool/--args`, cron direct tool payloads, and HTTP access to coding tools such as exec/read/write/apply_patch.

## What This Changes

Request support for invoking OpenClaw tools directly from explicit chat/slash, CLI, cron, and automation inputs without routing through an LLM turn.

## Best Possible Solution

Define a maintainer-approved direct-tool invocation contract that is explicit about allowed surfaces, auth/ownership semantics, deny/allow policy, result delivery, transcript/audit behavior, and payload versioning, then land the narrower HTTP and cron slices against that contract or split them into canonical follow-up issues.
