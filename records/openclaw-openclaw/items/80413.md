Keep open: the latest branch now distinguishes raw `claude-cli/*` from `anthropic-cli/*`, but it still changes a documented doctor migration contract and bundles an unrelated root `defaultModel` schema allowance that conflicts with existing config behavior.

## What This Changes

The PR changes doctor legacy runtime-model normalization to preserve raw `claude-cli/*` refs, adds doctor regression coverage, and carries schema/test/generated changes for config preservation and rebasing.

## Best Possible Solution

Merge only a narrow doctor fix that preserves the agreed `claude-cli` behavior, updates docs/tests for that contract, and moves or removes the `defaultModel` schema change unless its full config contract is approved separately.
