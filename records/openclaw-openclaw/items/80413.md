## What This Changes

The branch changes doctor config normalization to preserve `claude-cli/*`, adds schema allowances for root `defaultModel` and agent `description`, updates related tests, and regenerates Swift protocol for `internalRuntimeHandoffId`.

## Best Possible Solution

Preserve `claude-cli/*` doctor configs without disabling `anthropic-cli/*` migration, add coverage for both raw prefixes, and keep docs/tests aligned with the provider/runtime contract.

## Reproduction Assessment

Yes. From source, current main rewrites `claude-cli/*` through the doctor normalizer, and the PR-introduced `anthropic-cli/*` regression is also source-reproducible through `normalizeProviderId("anthropic-cli")`.

## Solution Assessment

No. The fix should distinguish the raw `claude-cli` prefix from `anthropic-cli` before skipping migration, rather than checking the normalized `legacyProvider` value shared by both paths.
