## Summary

Keep open: the PR addresses a plausible doctor/session-continuity bug and the supplied real behavior proof is sufficient, but the current guard is too broad and would stop `anthropic-cli/*` legacy refs from migrating. It also changes the documented CLI runtime migration contract, so maintainer product/owner review is needed before the branch should land.

## What This Changes

This PR changes doctor legacy model-ref normalization and tests to preserve `claude-cli/*` refs, and also carries config-schema allowances, generated Swift protocol output, and Discord test edits.

## Best Possible Solution

Decide the CLI provider/runtime contract, then implement the doctor repair by distinguishing the raw input prefix so actual `claude-cli/*` session continuity is preserved without regressing `anthropic-cli/*` migration, with tests and docs aligned.

## Reproduction Assessment

Yes. Source inspection shows current main rewrites `claude-cli/*` through `migrateLegacyRuntimeModelRef`, and the PR regression is also source-reproducible because `anthropic-cli` normalizes to `claude-cli` before the new skip condition runs.

## Solution Assessment

No. The proposed guard is not the narrowest maintainable fix because it keys off the normalized migrated provider rather than the raw input prefix, and it changes a documented migration contract without updating that contract.
