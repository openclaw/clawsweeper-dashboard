## Summary

Keep open: this PR targets a real doctor rewrite bug, but the proposed guard is too broad and would stop `anthropic-cli/*` refs from migrating through the existing legacy path. The branch also carries unrelated config-schema and generated Swift protocol changes, so the focused model-ref fix should be repaired and reviewed before merge.

## What This Changes

This PR changes doctor legacy model-ref normalization and tests to preserve `claude-cli/*`, while also adding schema allowances for root `defaultModel` and agent `description` plus a generated Swift gateway model field.

## Best Possible Solution

Land a narrow doctor fix that preserves real `claude-cli/*` refs without breaking `anthropic-cli/*` legacy migration, with regression coverage and docs/contract alignment.

## Reproduction Assessment

Yes. Source inspection shows current main rewrites `claude-cli/*` through `migrateLegacyRuntimeModelRef`, and the PR regression follows because `anthropic-cli` normalizes to `claude-cli` before the new skip condition checks `legacyProvider`.

## Solution Assessment

No. The proposed guard should distinguish the raw input provider prefix or revise the migration helper contract, then cover both `claude-cli/*` preservation and `anthropic-cli/*` migration.
