## Summary

Keep open: the doctor/session-continuity problem is plausible, but this PR still introduces a blocking alias regression because the new guard preserves `anthropic-cli/*` refs as well as real `claude-cli/*` refs.

## What This Changes

This PR changes doctor legacy model-ref normalization and tests to preserve `claude-cli/*`, adds schema allowances for root `defaultModel` and agent `description`, and regenerates the Swift gateway model for `internalRuntimeHandoffId`.

## Best Possible Solution

Fix Claude CLI session continuity while preserving `anthropic-cli/*` migration, and align tests/docs with the chosen provider/runtime contract before merge.

## Reproduction Assessment

Yes, from source inspection. Current main rewrites `claude-cli/*` through `migrateLegacyRuntimeModelRef`, and the PR regression follows because `anthropic-cli` normalizes to `claude-cli` before the new skip condition checks `legacyProvider`.

## Solution Assessment

No. The current PR guard is not the best fix because it keys off the normalized migrated alias; a safer fix should distinguish the raw input prefix or revise the migration helper contract, then cover both `claude-cli/*` and `anthropic-cli/*` cases.
