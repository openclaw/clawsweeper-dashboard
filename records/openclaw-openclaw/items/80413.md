## Summary

Keep open: the reported doctor rewrite bug is plausible, but this patch is too broad because it skips on normalized `legacyProvider`, which also covers the separate `anthropic-cli/*` legacy alias, and the PR only provides unit-test proof.

## What This Changes

The PR changes doctor legacy config normalization to leave `claude-cli/*` model refs unchanged, updates the migration test, and also carries schema/protocol edits for `defaultModel`, agent descriptions, and `internalRuntimeHandoffId`.

## Best Possible Solution

Preserve working `claude-cli/*` session continuity in doctor without disabling `anthropic-cli/*` legacy migration, and align regression tests/docs with the chosen provider/runtime contract.

## Reproduction Assessment

Yes, from source: current main's normalizer rewrites `claude-cli/*` via `migrateLegacyRuntimeModelRef`, and the existing doctor migration test asserts that behavior. The PR-introduced regression is also source-reproducible because `anthropic-cli/*` normalizes to the same `legacyProvider` value used by the new skip guard.

## Solution Assessment

No. The narrow fix should distinguish the raw `claude-cli` prefix from normalized `anthropic-cli`, or otherwise update the alias contract and tests, rather than skipping every migrated ref whose normalized legacy provider is `claude-cli`.
