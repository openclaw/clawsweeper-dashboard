## Summary

Keep open. Current main still gives gateway-executed plugin message sends an explicit 10000 ms timeout, and open PR #74169 has not merged yet.

## What This Changes

The issue asks for large WeCom media sends through `openclaw message send` to complete instead of timing out through the gateway after 10000 ms.

## Best Possible Solution

Review, land, or replace PR #74169 with a narrow core fix that stops forcing a 10000 ms timeout for gateway message actions and adds regression coverage for the no-config CLI media path.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: the CLI passes no timeout override, the message-action runner fills `10_000`, and gateway-mode plugin sends use `message.action` with that explicit timeout.

## Solution Assessment

No for current main. The narrow maintainable solution should cover the message-action gateway timeout resolver or CLI gateway options, not only the lower fallback used when callers omit `timeoutMs`.
