## Summary

Keep open. Current main still routes CLI gateway-executed message actions through an explicit 10000 ms timeout, and the linked closing PR is still open, so the issue should remain open until that implementation lands or is replaced.

## What This Changes

The issue reports that sending an approximately 10 MB media file through the WeCom plugin times out after 10000 ms via the OpenClaw gateway.

## Best Possible Solution

Review PR #74169, but ensure the final fix covers the explicit message-action timeout used by CLI gateway media sends and adds focused regression coverage for the no-config path.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: CLI message sends pass no timeout override, the message-action runner fills `10_000`, and gateway-mode plugin sends call `message.action` with that explicit timeout.

## Solution Assessment

Unclear for the linked PR as described. Changing only `resolveGatewayCallTimeout` is likely insufficient because message actions already pass `timeoutMs: 10_000`; the narrower durable fix should address the message-action timeout resolver or CLI gateway options as well.
