## Summary

Keep open. Current main still routes gateway-executed message sends through an explicit 10000 ms timeout, and open PR #74169 is the active closing implementation, so this issue should stay open until that fix lands or is replaced.

## What This Changes

A CLI `openclaw message send` with a roughly 10 MB media file through WeCom times out at 10000 ms instead of completing.

## Best Possible Solution

Land or replace PR #74169 with a narrow core fix that removes the forced 10000 ms budget for gateway message actions and adds regression coverage for the CLI media-send path.

## Reproduction Assessment

Yes, by source inspection. CLI message sends pass no timeout override, the message-action runner fills `10_000`, and gateway-mode plugin sends use `message.action` with that explicit timeout.

## Solution Assessment

Unclear for the open PR as described. Changing only `resolveGatewayCallTimeout` would not help callers that already pass `timeoutMs: 10_000`; the safer fix should cover `resolveGatewayActionOptions` or the CLI gateway options as well.
