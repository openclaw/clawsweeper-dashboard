## Summary

Keep open. Current main still routes CLI message sends for gateway-executed plugin actions through an explicit 10000 ms `message.action` timeout, and PR #74169 remains open rather than merged.

## What This Changes

Report asks OpenClaw to send a roughly 10 MB file through the WeCom plugin without the CLI gateway call timing out after 10000 ms.

## Best Possible Solution

Review and land or replace PR #74169 with a narrow fix that covers the no-config/no-env CLI `message send` path for gateway-executed plugin actions.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: for a gateway-executed plugin action with no timeout override, `openclaw message send --media ...` reaches `message.action` with an explicit 10000 ms timeout.

## Solution Assessment

Unclear for the linked PR as written, because it changes `resolveGatewayCallTimeout` while the message-action runner already passes an explicit 10000 ms timeout. A complete fix should either raise/remove that explicit default or route it through the intended timeout resolver.
