## Summary

Keep open. Current main still applies a 10000 ms timeout to gateway-executed message actions, and open PR #74169 is the active unmerged implementation lane for the same report.

## What This Changes

The issue requests fixing large-file sends through the WeCom channel that time out after 10000 ms via the local gateway.

## Best Possible Solution

Review, land, or replace PR #74169 with a narrow fix that covers CLI gateway message-action media sends and adds regression coverage for the no-timeout-override path.

## Reproduction Assessment

Yes. Source inspection shows CLI message sends pass no timeout override, the message-action runner fills `10_000`, and gateway-executed plugin sends use `message.action` with that explicit timeout.

## Solution Assessment

Unclear for the open PR as described. Changing only `resolveGatewayCallTimeout` may not fix callers that already pass `timeoutMs: 10_000`; the safer repair should cover the message-action timeout resolver or CLI gateway options too.
