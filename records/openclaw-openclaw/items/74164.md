## Summary

Keep open. Current main still forces gateway-executed message actions through a 10000 ms timeout, and the open closing PR is unmerged, so the issue should remain open until that implementation lands or is replaced.

## What This Changes

Sending a roughly 10 MB file through the external WeCom channel via `openclaw message send --media` times out at 10000 ms instead of completing.

## Best Possible Solution

Review, land, or replace PR #74169 with a narrow core fix that stops forcing 10000 ms for gateway message actions and adds regression coverage for the no-config CLI media path.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: CLI message sends pass no timeout override, the action runner fills `10_000`, and gateway-mode plugin sends use `message.action` with that explicit timeout.

## Solution Assessment

Unclear for PR #74169 as described. Changing only `resolveGatewayCallTimeout` would not help callers that already pass `timeoutMs: 10_000`; the safer fix should also cover the message-action timeout resolver or CLI gateway options.
