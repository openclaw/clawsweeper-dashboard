## Summary

Keep open. Current main still has 10000 ms timeout defaults on the gateway-executed message action path, and the issue is already paired with an open closing PR, so it should stay open until that PR or an equivalent fix lands.

## What This Changes

Sending an approximately 10 MB file through `openclaw message send --channel wecom --media ...` times out through the local gateway after 10000 ms.

## Best Possible Solution

Land or replace the linked PR with a narrow fix that makes gateway-executed message actions use the intended longer default timeout, with coverage for the no-config/no-env `message send` path.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: the CLI gateway plugin action path explicitly supplies a 10000 ms timeout to `message.action`. A live WeCom account would be needed only to prove the full 10 MB upload end to end.

## Solution Assessment

Unclear for the linked PR as described, because fixing only `resolveGatewayCallTimeout` may not affect callers that already pass `timeoutMs: 10000`. The maintainable fix should cover the message action timeout resolver or stop forcing the shorter default for gateway-executed sends.
