## Summary

Keep open. Current main still routes CLI gateway-executed message actions through an explicit 10000 ms timeout, and the linked closing PR is still unmerged, so the issue should remain open for that implementation lane.

## What This Changes

The issue reports that `openclaw message send --channel wecom` times out after 10000 ms when sending a roughly 10 MB media file through the gateway.

## Best Possible Solution

Review, land, or replace PR #74169 with a narrow fix that covers the message-action timeout resolver or CLI gateway options and adds regression coverage for the no-config CLI media path.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: CLI media sends pass no timeout override, the message-action runner fills `10_000`, and gateway-mode plugin sends use `message.action` with that explicit timeout.

## Solution Assessment

No, current main is not the solution yet. PR #74169 is the active path, but a complete fix should also cover callers that already pass `timeoutMs: 10_000` from the message-action runner.
