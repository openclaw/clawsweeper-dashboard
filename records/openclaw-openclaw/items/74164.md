## Summary

Keep open. Current main still routes gateway-executed message sends through an explicit 10000 ms action timeout, and the linked closing PR is open rather than merged.

## What This Changes

The issue reports that `openclaw message send --channel wecom --media` times out after 10000 ms when sending an approximately 10 MB file.

## Best Possible Solution

Review, land, or replace PR #74169 with a narrow core fix covering gateway-executed message actions from CLI media sends, with regression coverage for the no-explicit-timeout path.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `openclaw message send --media` reaches `runMessageAction`, gateway-mode channel plugins call `message.action`, and the wrapper passes an explicit `10_000` ms timeout before a large file upload can finish.

## Solution Assessment

Unclear for the open PR as written. The maintainable fix should cover the message-action timeout resolver or CLI gateway options, because changing only generic `resolveGatewayCallTimeout` does not help callers that already pass `timeoutMs: 10_000`.
