## Summary

Keep open. Current main still forces a 10000 ms timeout on CLI gateway-executed message actions, and open closing PR #74169 remains unmerged, so the issue should stay open until that fix lands or is replaced.

## What This Changes

Sending a roughly 10 MB file through the WeCom channel with `openclaw message send` should complete instead of failing with `gateway timeout after 10000ms`.

## Best Possible Solution

Align gateway-executed message sends with the intended handshake/config/env timeout instead of forcing 10000 ms, and add regression coverage for the no-config CLI gateway plugin send path.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: CLI message send omits a timeout override, the message-action runner fills in `10_000`, and gateway-executed plugin sends use that RPC path.

## Solution Assessment

Unclear for the linked PR as described. Changing only `src/gateway/call.ts` may not affect callers that already pass `timeoutMs: 10_000`, so the safer fix should cover the message-action timeout resolver or stop forcing the shorter default there.
