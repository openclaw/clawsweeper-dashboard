## Summary

Keep open. Current main still supplies a 10000 ms timeout on the CLI gateway-executed message action path, and the issue is paired with open closing PR #74169, so it should remain open until that PR or an equivalent fix lands.

## What This Changes

The report asks OpenClaw to stop timing out after 10000 ms when sending a roughly 10 MB file through the WeCom channel via `openclaw message send --media`.

## Best Possible Solution

The durable fix should make gateway-executed message sends use an upload-appropriate timeout from config or a dedicated message-action default, with focused coverage for the no-config CLI send path.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: `openclaw message send` passes no timeout override, gateway-mode plugin sends go through `message.action`, and current main explicitly supplies `10000` ms to that gateway call.

## Solution Assessment

Unclear for the linked PR as described. A generic `resolveGatewayCallTimeout` change is useful only if it also affects the message action path that already passes an explicit `10000` ms timeout.
