## Summary

Keep open. Current `main` still has a 10000 ms timeout on the CLI gateway message-action path, and open PR #74169 is an unmerged closing implementation candidate that should be reviewed or replaced before this issue closes.

## What This Changes

The issue reports that sending an about-10 MB file with `openclaw message send --channel wecom --media ...` times out through the gateway after 10000 ms.

## Best Possible Solution

Land or replace #74169 with a narrow fix that makes CLI gateway-executed message sends use the intended longer or configurable RPC timeout, with regression coverage for the no-config/no-env path.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: `messageCommand` supplies no timeout, `resolveGatewayActionOptions` defaults to 10000 ms, and gateway-executed sends pass that value to `message.action`; a live WeCom account is only needed for end-to-end upload proof.

## Solution Assessment

Unclear for #74169 as described. Changing only `resolveGatewayCallTimeout` may not affect callers that already pass `timeoutMs: 10000`, so the safer fix should cover the message-action gateway default or stop forcing the shorter explicit timeout.
