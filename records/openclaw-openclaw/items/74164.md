## Summary

Keep open. Current main still applies a 10000 ms timeout to gateway-executed message actions, and the linked closing PR is open and unmerged, so this issue should remain active until that fix lands or is replaced.

## What This Changes

The issue asks for large media sends through the WeCom plugin to complete instead of failing with a 10000 ms gateway timeout.

## Best Possible Solution

Review, land, or replace PR #74169 with a narrow core fix that stops forcing 10000 ms for gateway message actions and adds regression coverage for CLI media sends.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: CLI media sends pass no timeout override, the message-action runner fills `10_000`, and gateway-mode plugin sends use `message.action` with that explicit timeout.

## Solution Assessment

No for the currently linked PR as described. Changing only the generic gateway timeout resolver is unlikely to cover callers that already pass `timeoutMs: 10_000`; the fix should cover the message-action timeout resolver or the CLI gateway options too.
