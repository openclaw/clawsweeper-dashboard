## Summary

Keep open. Current main still forces the CLI gateway-executed `message.action` path through a 10000 ms timeout, and open PR #74169 already references this bug as the active implementation lane.

## What This Changes

The issue reports that sending an approximately 10 MB file through the WeCom channel with `openclaw message send --media` times out after 10000 ms.

## Best Possible Solution

Land or replace PR #74169 with a narrow fix that covers the CLI `message.action` path, including the no-config/no-env case and a regression test for gateway-executed plugin sends.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: `openclaw message send` reaches `runMessageAction` without a gateway timeout override, then gateway-executed plugin sends call `message.action` with an explicit default of 10000 ms.

## Solution Assessment

Unclear for the current linked PR. The maintainable fix should either stop forcing the shorter message action default or pass the intended longer timeout into that path, because a generic fallback change alone may not be reached.
