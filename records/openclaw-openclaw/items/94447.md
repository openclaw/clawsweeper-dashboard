## Summary

Keep open: this is a focused exec bug-fix PR, but the current patch expands `~` through OpenClaw's effective home, so installs with `OPENCLAW_HOME` set can still resolve host exec workdirs away from the OS account home.

## What This Changes

The PR imports `expandHomePrefix()` and applies it in `resolveWorkdir()` before statting and returning the local host exec workdir.

## Best Possible Solution

Use one canonical host exec fix that expands leading `~` against the OS account home, preserves the current fallback policy unless maintainers separately choose fail-fast behavior, and covers divergent `HOME`/`OPENCLAW_HOME`.

## Reproduction Assessment

Yes. Source inspection shows local gateway exec reaches `resolveWorkdir()` and current main plus `v2026.6.8` stat the literal `~/...` path before falling back; I did not run a live exec command in this read-only review.

## Solution Assessment

No. Expanding leading `~` is a good narrow direction, but this implementation uses the effective OpenClaw home instead of the OS account home used by sibling host file tools.
