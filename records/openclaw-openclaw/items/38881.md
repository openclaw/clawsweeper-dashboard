## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Move the switch before media understanding:** `src/auto-reply/reply/get-reply.ts:303-314`
  - body: The helper runs only after `applyMediaUnderstanding` has already executed with the pre-switch model, so image turns can incur separate image understanding and then a full vision-model run. Resolve the image-capable model before media understanding, or skip/reconcile that pass after switching.
  - confidence: 0.92
- **[P2] Use typed image attachment detection:** `src/auto-reply/reply/get-reply.ts:87-89`
  - body: `MediaPath` and `MediaPaths` are generic media fields that can carry PDFs, audio, or video, while URL-only images can arrive through URL fields. Use normalized attachment kind or MIME metadata so non-image messages do not switch and supported image inputs are not missed.
  - confidence: 0.9
- **[P2] Require a known non-vision current model:** `src/auto-reply/reply/get-reply.ts:95-99`
  - body: When the active provider/model is absent from the catalog, this path treats it as non-vision and can silently override custom or newly configured vision-capable models. Only switch after positively identifying that the current model cannot accept images.
  - confidence: 0.88
- **[P2] Honor imageModel fallbacks:** `src/auto-reply/reply/get-reply.ts:101-103`
  - body: The existing `imageModel` contract supports `primary` plus `fallbacks`, including fallback-only configs, but this helper reads only `imageModel.primary`. Reuse the existing candidate/fallback resolution path so valid fallback entries can be selected.
  - confidence: 0.86
