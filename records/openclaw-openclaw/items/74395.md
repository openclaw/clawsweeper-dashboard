## Summary

Keep open: current main has narrowed some provider-auth clobber paths, but Google API-key onboarding and setup still expose preview-model default promotion, and the broader cost-tier consent policy is not settled.

## What This Changes

Request to prevent upgrades, setup, or migrations from silently promoting existing defaults to paid preview models without explicit consent and cost/billing warnings.

## Best Possible Solution

Define default-model changes as explicit user intent: preserve existing primary/fallbacks across upgrade and setup unless the user confirms, add non-interactive consent semantics for cost-tier or preview promotions, and keep the narrower heartbeat/quota fixes on their own linked issues.

## Reproduction Assessment

Partial yes. Source inspection shows current setup can still apply the Google preview default without the existing-default preservation flag, but the exact historical 2026.3.2 to 2026.3.22 migration and billed usage require the reporter's private artifacts.

## Solution Assessment

Unclear for a direct code fix because this issue asks for a product policy across upgrades, setup, changelog warnings, and billing heuristics. The narrow setup/heartbeat and Google preview-default parts are already tracked by https://github.com/openclaw/openclaw/issues/64129 and https://github.com/openclaw/openclaw/issues/79670, but the broader consent model still needs maintainer direction.
