## Summary

Keep open: the merged setup preservation work addresses one narrow overwrite path, but current main and the latest release still use `google/gemini-3.1-pro-preview` as the Google onboarding default and still normalize Gemini Pro aliases to that preview id. The reporter’s historical backup bundle is not attached, but the source, docs, related issue, and another user’s similar comment support this as a real maintainer policy problem rather than a completed fix.

## What This Changes

The issue asks OpenClaw to prevent silent paid-preview default model changes during upgrades/provider setup and add consent, warning, and cost-control policy around model defaults.

## Best Possible Solution

Maintainers should decide a durable paid-preview/default migration policy, then align Google defaults, normalization, docs, doctor/setup warnings, and tests while keeping the existing setup-preservation fix.

## Reproduction Assessment

No for the exact historical migration artifact because the reporter’s backup bundle is not attached. Yes at source level for the remaining policy gap: current main and the latest release still write and normalize Google Pro defaults to a preview model without an evident pricing-consent path.
