## Summary

Keep open: current main has partial default-preservation fixes, but it still does not implement a general consent or warning policy for paid/preview default-model promotions. Google API-key setup still advertises the preview Pro default, setup still calls provider auth without the existing-default preservation flag, and the narrower related reports remain open rather than superseding this broader policy issue.

## What This Changes

The issue asks OpenClaw to stop silent paid or preview default-model promotions during upgrade/setup and add explicit consent or warning surfaces for cost-impacting model changes.

## Best Possible Solution

Define default-model changes as explicit user intent: preserve existing primaries/fallbacks across upgrade and setup unless confirmed, add non-interactive consent semantics for costly or preview promotions, and document the pricing/quota impact clearly.

## Reproduction Assessment

Partial yes. Source inspection shows current setup plus Google API-key onboarding can still promote the preview Pro default without the preservation flag, but the exact historical upgrade write and billing timeline require the reporter's private artifacts.

## Solution Assessment

Unclear for a single automated fix. The requested direction is reasonable, but the maintainable solution needs a product/provider policy decision and should keep the narrower related implementation tracks separate.
