## Summary

Keep open. Current main has a configure-specific preservation fix, but the broader cost-control issue remains: Google defaults and aliases still point at preview models, setup/onboarding still applies provider defaults without a generic consent policy, and the second user report points to a cross-provider default-model surprise rather than a one-off Google artifact.

## What This Changes

The issue asks OpenClaw to prevent silent default-model changes that promote users to higher-cost preview models and to add related billing/cost-control safeguards.

## Best Possible Solution

Define model-default changes as explicit user intent: preserve existing primary/fallbacks by default across upgrade, setup, and migration-style writes, require confirmation for preview or cost-tier promotions, and split cron/tool-grounding follow-ups if maintainers want them.

## Reproduction Assessment

Partial yes. Current main gives a high-confidence source-level reproduction for a remaining setup/onboarding promotion path through `applyAuthChoice` without `preserveExistingDefaultModel`; the exact historical billing event cannot be replayed without the reporter's artifacts.

## Solution Assessment

Unclear until maintainers decide the policy. The configure preservation fix is the right direction for #50268, but a narrow setup-only patch would not fully solve silent price-tier promotion or preview-model consent.
