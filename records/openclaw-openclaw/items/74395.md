## Summary

Keep open. Current main still has cost-sensitive preview defaults and setup/auth paths that can promote or preserve a provider-owned preview primary without a broader consent or pricing-warning policy, while related cost/default-clobber reports remain open. The cron-control subrequest appears partly improved by the current cron tool, but the central model-default safety issue is not resolved.

## What This Changes

Request to prevent upgrades, setup, or migration flows from silently changing configured default models and fallbacks to higher-cost preview models without explicit user consent, pricing notice, and recovery controls.

## Best Possible Solution

Keep this as the model-default safety umbrella: preserve existing global and per-agent primary/fallback settings unless the user explicitly requests a new default, add consent or warning UX for preview/higher-cost promotion, keep explicit model-setting commands authoritative, and split cron self-control plus tool-grounding visibility into narrower follow-ups if current surfaces are still insufficient.

## Reproduction Assessment

Partial yes. Current main gives a high-confidence source-level reproduction of remaining overwrite risk through the Google default helper and setup path; the exact historical billing event still needs the reporter's backup config files and session logs.

## Solution Assessment

Unclear until maintainers choose the cost-safety policy. The configure preservation fix is useful but not broad enough; the maintainable direction is to extend preservation/consent to setup and migration-style default promotion while keeping explicit `--set-default` and model-selection flows intentional.
