## Summary

Keep open: the setup overwrite path has a merged mitigation, but current main and v2026.6.10 still use and document `google/gemini-3.1-pro-preview` as the Google API-key default and still normalize Gemini Pro aliases to that preview ID, so the broader billing-consent/default-policy issue remains unresolved.

## What This Changes

The issue reports that an upgrade/migration silently changed the configured default model from a lower-cost Gemini model to a paid Gemini preview model, causing unexpected billing.

## Best Possible Solution

Maintainers should decide the paid-preview default and migration-consent policy, then align Google defaults, Pro alias normalization, docs, setup or doctor warnings, and focused tests while keeping the existing setup-preservation fix.
