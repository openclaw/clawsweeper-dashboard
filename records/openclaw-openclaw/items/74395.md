- `P1`: The issue reports real unintended paid model routing after persisted model/default behavior changed, including long-running background traffic and significant billing exposure.

## Summary

Keep open: the setup-overwrite slice is fixed by https://github.com/openclaw/openclaw/pull/93658, but current main and the latest release still hard-code the Google API-key onboarding default to `google/gemini-3.1-pro-preview`, normalize Pro aliases to that preview id, and document that model as the Google default. The open related PR at https://github.com/openclaw/openclaw/pull/94069 is only a partial direction because it switches to another preview model rather than settling migration consent, stable-default, quota-warning, or explicit-choice preservation policy.

## What This Changes

The issue asks OpenClaw to stop silently migrating or recommending paid Gemini preview defaults without explicit user consent, pricing context, or cost-control warnings.

## Best Possible Solution

Maintainers should choose a billing-safe Google default and migration-consent policy, then align the Google plugin default, alias/retired-model handling, setup or doctor warnings, public docs, and regression coverage while preserving explicit user model choices.

## Reproduction Assessment

No for the exact historical upgrade artifact because the reporter's backup bundle is not attached. Yes at source level for the remaining policy gap: current main and `v2026.6.10` still choose and normalize Google Pro defaults to a preview model without an evident pricing-consent path.

## Solution Assessment

No, the merged setup-preservation fix is only a partial repair. The best fix needs a maintainer-approved billing/default policy rather than another isolated provider-auth or one-line model swap.
