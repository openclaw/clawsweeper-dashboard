## Label Justifications

- `P1`: The issue reports real unintended paid model usage after persisted default/model-routing changes, with billing exposure in background and long-running agent workflows.
- `impact:data-loss`: The report says OpenClaw overwrote the user's configured primary and fallback chain in persisted config during upgrade.
- `impact:session-state`: The reported session continued recording one user model choice while most calls routed through a different preview model.
- `impact:auth-provider`: The affected surface is provider auth/default model routing and persisted model choice changing to a paid Google preview model.

## Summary

Keep open: current main and v2026.6.10 still encode the disputed Google preview default and preview alias normalization, while the merged setup-preservation fix only covers one overwrite path and the open related PR would switch to another preview default rather than settle the broader paid-default consent policy.

## What This Changes

The issue asks OpenClaw to prevent upgrade/setup migration paths from silently replacing a user's configured default model with more expensive Google Gemini preview defaults without explicit consent or warning.

## Best Possible Solution

Maintainers should choose a durable paid-preview/default migration policy, then align Google defaults, normalization, setup/doctor warnings, docs, and tests while preserving explicit user model choices.

## Reproduction Assessment

No for the exact historical migration artifact because the reporter's backup bundle is not attached. Yes at source level for the remaining gap: current main and v2026.6.10 still default and normalize Google Pro choices to preview models without an evident pricing-consent policy.

## Solution Assessment

No, the already merged setup-preservation fix is only a partial solution. The best path is a maintainer-approved default and billing-safety policy rather than another isolated provider-auth patch.
