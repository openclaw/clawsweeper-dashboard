## Summary

Keep open. Current main still has only a configure-specific preservation fix, while Google defaults and aliases still point at preview models and setup/onboarding can still apply provider defaults without a generic cost-tier consent policy; the newer related #64129 report covers one narrow setup/heartbeat bug but does not supersede this broader migration/default-policy issue.

## What This Changes

The issue asks OpenClaw to prevent silent default-model changes that promote users to higher-cost preview models and to add related billing/cost-control safeguards.

## Best Possible Solution

Define model-default changes as explicit user intent: preserve existing primary/fallbacks by default across upgrade, setup, and migration-style writes, require confirmation for preview or cost-tier promotions, and track the narrower setup/heartbeat fix separately through #64129.

## Reproduction Assessment

Partial yes. Source inspection gives a high-confidence current-main reproduction for a remaining setup/onboarding promotion path through applyAuthChoice without preserveExistingDefaultModel, but the exact historical billing migration cannot be replayed without the reporter's artifacts.

## Solution Assessment

Unclear until maintainers decide the policy. The configure preservation fix is a good narrow precedent, but a setup-only patch would not fully solve silent preview-model or price-tier promotion across upgrades and migrations.
