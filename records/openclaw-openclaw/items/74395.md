## Summary

Keep open. Current main still has the paid Google preview model as the bundled default and has provider auth/setup paths where preservation is caller-scoped or can be bypassed by provider config patches before the generic preservation helper runs. The same discussion also has a second user reporting a similar paid-model surprise, so this is not a cleanup-close candidate.

## What This Changes

The issue asks OpenClaw to prevent silent upgrade/setup migrations from changing a user's default model to higher-cost preview models without explicit consent, pricing notice, and recovery controls.

## Best Possible Solution

Keep this as the model-default cost-control umbrella until OpenClaw has a consistent consent contract: preserve existing global and per-agent primaries/fallbacks across upgrade/setup/configure paths unless the user explicitly chooses a new paid or preview default, warn and offer revert when a migration-set preview default is detected, and split cron stop controls plus tool-grounding indicators into narrower tracked work.

## Reproduction Assessment

Partial yes. Current main gives a high-confidence source-level reproduction of remaining default overwrite risk through the Google default helper, provider auth patch flow, and setup caller; the exact historical billing event needs the reporter's backup config/session bundle.

## Solution Assessment

Unclear until maintainers choose the product policy. The already-landed configure preservation is a useful partial fix, but the maintainable direction is to extend preservation/consent to setup and migration-style default promotion while keeping explicit model-setting flows intentional.
