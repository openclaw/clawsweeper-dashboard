## Summary

Keep open. Current main still contains cost-sensitive default-model overwrite paths around Google API-key setup and first-run setup, while the merged configure preservation work only covers one caller. The second user report in the discussion makes this look like an umbrella policy issue rather than a one-off historical migration artifact.

## What This Changes

The issue requests explicit consent, warning, and recovery behavior before OpenClaw upgrades or reconfigures default models to higher-cost preview variants, with related cost-control requests for cron management and tool-grounding visibility.

## Best Possible Solution

Keep this as the model-default safety umbrella: preserve existing global and per-agent primaries and fallbacks across upgrade, setup, and provider-auth paths unless the user explicitly chooses a new default; add clear warning/revert UX for migration-set preview or higher-cost defaults; keep explicit `--set-default` and `models set` flows authoritative; and split cron control plus tool-grounding visibility into focused follow-ups if maintainers want them tracked separately.

## Reproduction Assessment

Partial yes. Current main gives a high-confidence source-level reproduction of the remaining overwrite risk through the Google API-key auth config patch and setup path; the exact historical billing event cannot be reproduced without the reporter's backup files and session logs.

## Solution Assessment

Unclear until maintainers choose the policy. The configure preservation shipped in #70793 is a useful partial fix, but the best maintainable fix is broader consent/preservation behavior for setup and migration-style default promotion, plus explicit warning copy for preview or higher-cost defaults.
