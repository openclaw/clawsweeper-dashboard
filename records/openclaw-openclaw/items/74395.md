## Summary

Keep open. Current main has a partial configure-path fix, but onboarding/setup can still apply provider auth with `setDefaultModel: true` without the preservation flag; Google auth still advertises and patches in `google/gemini-3.1-pro-preview`, and the discussion includes a second paid-model surprise report.

## What This Changes

The issue asks OpenClaw to prevent silent upgrade/setup migrations from switching existing default models to more expensive preview models without explicit consent.

## Best Possible Solution

Keep this as the model-default safety umbrella: preserve existing primaries and fallbacks across setup/configure/migration unless the user explicitly chooses a new default, add preview/price-tier warning and revert UX, and split cron/tool-grounding into narrower issues if maintainers want them tracked separately.

## Reproduction Assessment

Partial yes. Current main gives a high-confidence source-level reproduction of the remaining overwrite risk through Google provider auth plus first-run setup; the exact historical billing window still needs the reporter's backups/session logs to replay.

## Solution Assessment

Unclear until maintainers choose the cost policy. A narrow setup preservation change would reduce the remaining overwrite path, but the durable fix should be generic consent/preservation plus warning/revert behavior rather than a Google-only special case.
