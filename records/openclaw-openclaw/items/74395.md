## Summary

Keep open. Current main still has a source-level path where provider setup can apply Google’s preview default over an existing primary, while the shipped related fix only preserves the existing default for `openclaw configure`; the issue also has a second paid-model surprise report, so this is not a safe cleanup close.

## What This Changes

The issue asks OpenClaw to stop silent upgrade or setup paths from changing configured default models to more expensive preview variants without explicit user consent.

## Best Possible Solution

Preserve existing global and per-agent primaries and fallback chains across setup, configure, and migration-style provider patches unless the user explicitly chooses a new default, then add preview/price-tier warning and revert UX for migration-set defaults.

## Reproduction Assessment

Partial yes. Current main gives a high-confidence source-level reproduction of the remaining overwrite risk through Google provider auth plus setup; the exact historical billing window cannot be replayed without the reporter’s artifacts.

## Solution Assessment

Unclear until maintainers choose the cost policy. A narrow setup-preservation fix would reduce the remaining overwrite path, but the maintainable solution should be generic consent and preservation behavior rather than a Google-only special case.
