## Summary

Keep open. Current main still has a cost-sensitive default-model overwrite path: Google API-key auth registers a preview default and provider config patch that can set `agents.defaults.model.primary`, while the setup wizard still applies provider auth as a default-model change without the configure preservation flag. The merged configure fix is only partial, and the issue discussion includes a second paid-model surprise report, so this is not a safe cleanup close.

## What This Changes

The issue requests explicit consent, warning, and recovery behavior before OpenClaw upgrades or reconfigures default models to higher-cost preview variants, with related cost-control requests for cron management and tool-grounding visibility.

## Best Possible Solution

Keep this as the model-default safety policy umbrella: preserve existing global and per-agent primaries and fallbacks across upgrade, setup, and provider-auth paths unless the user explicitly chooses a new default; add clear warning/revert UX for migration-set preview defaults; keep explicit `--set-default` and model-setting commands authoritative; and split cron control plus tool-grounding visibility into focused follow-ups if maintainers want them tracked separately.

## Reproduction Assessment

Partial yes. Current main gives a high-confidence source-level reproduction of the remaining overwrite risk through the Google API-key auth config patch and setup path; the exact historical billing event cannot be reproduced without the reporter's backup files and session logs.

## Solution Assessment

Unclear until maintainers choose the policy. The configure preservation shipped in #70793 is a useful partial fix, but the best maintainable fix is broader consent/preservation behavior for setup and migration-style default promotion, plus explicit warning copy for preview or higher-cost defaults.
