## Summary

Keep open. Current main still has a cost-sensitive default-model overwrite path: Google API-key auth advertises and patches in a preview default, and first-run setup applies provider auth with `setDefaultModel: true` without the preservation flag that `openclaw configure` now uses. The discussion also includes a second paid-model surprise report, so this is not a safe cleanup close.

## What This Changes

Prevent upgrade/setup/provider-auth flows from silently replacing existing model defaults or fallbacks with higher-cost preview defaults without explicit consent.

## Best Possible Solution

Keep this as the model-default safety umbrella: preserve existing global and per-agent primaries and fallback chains across setup, configure, and migration-style provider patches unless the user explicitly chooses a new default; add clear preview/price-tier warning and revert UX for migration-set defaults; split the cron control and tool-grounding observations if maintainers want separate tracking.

## Reproduction Assessment

Partial yes. Current main gives a high-confidence source-level reproduction of the remaining overwrite risk through provider auth plus first-run setup; the exact historical billing event cannot be replayed without the reporter's backup files and session logs.

## Solution Assessment

Unclear until maintainers choose the cost policy. A narrow setup preservation fix would reduce the remaining overwrite path, but the maintainable end state should be generic consent/preservation behavior plus warning/revert UX rather than a Google-only special case.
