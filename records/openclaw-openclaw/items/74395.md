## Summary

Keep open. Current main has fixed the narrower configure reauth clobber path from #50268, but the broader cost-control request remains: Google defaults and aliases still point at preview models, setup/onboarding can still apply provider defaults without a generic price-tier consent policy, and a second user report suggests the problem is not limited to one historical Google migration.

## What This Changes

The issue asks OpenClaw to prevent silent/default model changes that promote users to higher-cost preview models, plus related cost-control improvements for cron jobs and tool-grounded completion claims.

## Best Possible Solution

Define model-default changes as explicit user intent: preserve existing primary/fallbacks by default across upgrade, setup, and migration-style writes, require confirmation for preview or cost-tier promotions, and split the cron/tool-grounding follow-ups if maintainers want to pursue them.

## Reproduction Assessment

Partial yes. Current main gives a source-level reproduction for a remaining setup/onboarding promotion path through `applyAuthChoice` without `preserveExistingDefaultModel`; the exact historical billing event cannot be replayed without the reporter's artifacts.

## Solution Assessment

Unclear until maintainers decide the policy. The current configure preservation fix is the right direction for #50268, but a narrow setup-only patch would not fully solve silent price-tier promotion or preview-model consent.
