## Summary

Keep open. Current main still has a setup/provider-auth path that can apply Google’s preview default over an existing primary, while the merged preservation work only clearly covers `openclaw configure`; the second user report and billing impact make this a maintainer policy item rather than safe cleanup.

## What This Changes

The issue requests safeguards against upgrades or provider setup silently replacing user model defaults and fallback chains with higher-cost preview models.

## Best Possible Solution

Make default model changes explicit and cost-aware across setup, configure, and doctor/migration paths: preserve existing primaries and fallbacks by default, warn and offer revert for preview or price-tier upgrades, and split unrelated cron/tool-grounding follow-ups.

## Reproduction Assessment

Partial yes. Current main gives a high-confidence source-level reproduction of the remaining setup overwrite path; the exact historical billing window cannot be replayed without the reporter’s artifacts.

## Solution Assessment

Unclear until maintainer policy is set. A narrow setup-preservation fix would reduce the remaining bug, but the best durable fix is a generic consent/preservation rule rather than a Google-only special case.
