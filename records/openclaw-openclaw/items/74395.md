## Summary

Keep open. Current main still has cost-sensitive preview defaults and setup/provider-auth paths that can overwrite an existing primary outside the already-preserved configure flow, while the issue discussion also includes a second paid-model surprise report. This is not a cleanup close; it needs maintainer policy on preview/higher-cost default consent and narrower follow-ups for the cron and tool-grounding requests.

## What This Changes

The issue asks OpenClaw to prevent upgrades, setup, or migrations from silently replacing existing default and fallback models with preview or higher-cost variants, and to add related billing-warning and cost-control safeguards.

## Best Possible Solution

Keep this issue as the model-default safety policy umbrella: preserve existing global and per-agent primaries and fallbacks across upgrade/setup paths unless the user explicitly accepts a preview or higher-cost replacement, keep explicit model-setting commands authoritative, add warning/revert UX for migration-set preview defaults, and split cron control plus tool-grounding visibility into narrower tracked items.

## Reproduction Assessment

Partial yes. Current main gives a high-confidence source-level reproduction of the remaining overwrite risk through the Google default helper and setup path, but the exact historical billing event needs the reporter's backup config files and session distribution logs.

## Solution Assessment

Unclear until maintainers choose the product policy. The already-landed configure preservation is a useful partial fix, but the maintainable direction is to extend preservation/consent to setup and migration-style default promotion while keeping explicit models set and --set-default flows intentional.
