## Summary

Keep open. Current main still uses a cost-sensitive Google preview default and has default-setting paths that overwrite an existing primary unless the caller opts into preservation; the issue discussion also includes a second paid-model surprise report, so this is not conservative cleanup material.

## What This Changes

The issue asks OpenClaw to prevent upgrades, onboarding, and migrations from silently replacing existing default models and fallbacks with preview or higher-cost models without consent, warnings, and documentation.

## Best Possible Solution

Define and document a model-default safety policy that preserves existing global and per-agent primaries/fallbacks across upgrades and setup reruns unless the user explicitly accepts a preview or higher-cost replacement, keeps explicit `models set` and `--set-default` flows intentional, separates provider ID normalization from default promotion, and adds a visible warning/revert path for migration-set preview defaults.

## Reproduction Assessment

Partial yes. Current main source and tests give a high-confidence reproduction of the remaining overwrite-by-default policy risk, but the exact historical billing artifact cannot be replayed without the reporter's backup config files and session logs.

## Solution Assessment

Unclear until maintainers choose the policy. The narrowest maintainable direction is to make preservation and consent the default for migration/setup flows while keeping explicit default-setting commands intentionally mutating.
