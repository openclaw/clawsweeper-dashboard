## Summary

Keep open. Current main has partial protection for `openclaw configure`, but the bundled Google default is still the preview model and setup/provider-auth paths can still promote provider defaults without a price-tier consent policy; the second user report also points to a broader default-model cost-control problem.

## What This Changes

Request to prevent upgrades or setup flows from silently replacing configured primary/fallback model choices with higher-cost preview defaults, with warnings and related cost-control improvements.

## Best Possible Solution

Adopt a generic model-default preservation and consent policy across setup, configure, doctor/migration-style writes, and per-agent defaults, with explicit user intent required for cost-tier or preview-model promotion.

## Reproduction Assessment

Partial yes. Current main gives a high-confidence source-level reproduction of the remaining overwrite path through the Google default helper and setup call; the exact historical billing event cannot be replayed without the reporter's artifacts.

## Solution Assessment

Unclear until maintainers decide the policy. A narrow setup-preservation patch would reduce one remaining overwrite path, but the maintainable fix should be generic consent/preservation behavior rather than a Google-only special case.
