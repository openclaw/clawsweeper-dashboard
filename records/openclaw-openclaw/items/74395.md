## Summary

Keep open. Current main still has cost-sensitive preview defaults and default-setting paths that can replace an existing primary outside the already-documented `openclaw configure` preservation path, and the thread includes a second similar paid-model surprise report. This needs maintainer policy on model-default consent, preview/higher-cost warnings, and likely split follow-ups for the cron and tool-grounding requests.

## What This Changes

The issue asks OpenClaw to stop silently promoting existing default model selections to higher-cost preview variants during upgrade/setup, preserve user-selected primaries/fallbacks unless explicitly changed, and add clearer cost-control warnings and related controls.

## Best Possible Solution

Define and document a model-default safety policy that preserves existing global and per-agent primaries and fallbacks across upgrade/setup/provider-auth paths unless the user explicitly accepts a preview or higher-cost replacement, separates provider ID normalization from default promotion, adds a visible warning/revert path for migration-set preview defaults, and splits cron control plus tool-grounding visibility into narrower follow-ups.

## Reproduction Assessment

Partial yes. Current main source and tests give a high-confidence reproduction of the remaining overwrite-by-default policy risk, but the exact historical billing event requires the reporter's backup config files and session distribution logs.

## Solution Assessment

Unclear until maintainers choose the policy. The narrowest maintainable direction is likely to make preservation/consent the default for migration and setup flows while keeping explicit `models set` and `--set-default` commands intentional.
