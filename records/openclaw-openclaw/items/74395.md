## Summary

Keep open. Current main still lacks a general consent policy for default-model promotions to preview or higher-cost models; the configure preservation fix is only a partial mitigation, and the open #64129 report tracks a narrower setup/heartbeat bug rather than superseding this broader migration/default-policy issue.

## What This Changes

The issue asks OpenClaw to prevent silent upgrade/setup changes from moving existing model defaults to paid preview variants without explicit user consent.

## Best Possible Solution

Define model-default changes as explicit user intent: preserve existing primary/fallbacks by default across upgrade, setup, and migration-style writes, require confirmation for preview or cost-tier promotions, and keep the narrower setup/heartbeat repair tracked through #64129.

## Reproduction Assessment

Partial yes. Source inspection gives a current-main reproduction for a remaining setup/onboarding promotion path through applyAuthChoice without preserveExistingDefaultModel, but the exact historical billing migration cannot be replayed without the reporter's artifacts.

## Solution Assessment

Unclear until maintainers decide the policy. The configure preservation fix is a good narrow precedent, but a setup-only patch would not fully solve silent preview-model or price-tier promotion across upgrades and migrations.
