## Summary

Keep open: current main still has source-visible paths that can promote Google API-key setup to the preview Pro default, and related narrower reports do not cover the full default-model consent and cost-control policy.

## What This Changes

The issue requests explicit consent, warnings, and safer defaults so upgrades or setup do not silently move existing model traffic onto paid preview models.

## Best Possible Solution

Define default-model changes as explicit user intent: preserve existing primary/fallbacks across upgrade and setup unless the user confirms, add non-interactive consent semantics for costly or preview promotions, and split cron/tool-grounding cost controls into narrower follow-ups.

## Reproduction Assessment

Partial yes. Source inspection shows current setup and Google API-key onboarding can still apply the Google preview default without the existing-default preservation flag, but the exact historical upgrade and billing artifact require the reporter's private backups/session logs.

## Solution Assessment

Unclear for a single code fix. The requested direction is reasonable, but the maintainable path needs a product decision on default-model consent semantics and should keep https://github.com/openclaw/openclaw/issues/64129 and https://github.com/openclaw/openclaw/issues/79670 as narrower implementation tracks.
