## Summary

Keep open. Current main has fixed the narrower configure/provider-auth clobber path from #50268, but the broader cost-control issue remains: Google defaults and aliases still point at preview models, setup/onboarding can still promote provider defaults without a generic price-tier consent policy, and another user reported a similar paid-model surprise with Claude becoming primary.

## What This Changes

The issue asks OpenClaw to prevent silent model-default upgrades to higher-cost preview models and add related billing/cost-control safeguards.

## Best Possible Solution

Define model-default changes as explicit user intent: preserve existing primary/fallbacks by default across upgrade, setup, and migration-style writes, require confirmation for preview or cost-tier promotions, and split cron/tool-grounding follow-ups if maintainers want to pursue them.

## Reproduction Assessment

Partial yes. Current main gives a source-level reproduction for a remaining setup/onboarding promotion path through `applyAuthChoice` without `preserveExistingDefaultModel`; the exact historical billing event cannot be replayed without the reporter's artifacts.

## Solution Assessment

Unclear until maintainers decide the policy. The configure preservation fix is the right direction for #50268, but a setup-only patch would not fully solve silent price-tier promotion or preview-model consent.
