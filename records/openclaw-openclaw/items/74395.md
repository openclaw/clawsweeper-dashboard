## Summary

Keep open: current main has partial default-preservation plumbing, but the broader paid/preview default-model consent and warning policy is still unresolved. The related narrow setup/heartbeat bug remains open with an open fix PR, and another user reported a similar unexpected paid-provider default switch.

## What This Changes

The issue asks OpenClaw to prevent silent default-model migrations or setup flows from promoting users to more expensive preview/paid models without explicit consent or warning.

## Best Possible Solution

Define default-model changes as explicit user intent: preserve existing primaries across upgrade/setup unless confirmed, add non-interactive consent semantics for costly or preview promotions, and document pricing/quota impact where defaults change.

## Reproduction Assessment

Partial yes. Current source shows setup plus Google API-key onboarding can still promote a preview Pro default without the preservation flag, but the exact historical upgrade write and billing timeline require the reporter's private artifacts.

## Solution Assessment

Unclear for an automated fix. The requested direction is reasonable, but the maintainable solution needs maintainer policy for price-tier classification, first-run versus upgrade behavior, non-interactive consent, and how much to warn in changelogs/startup.
