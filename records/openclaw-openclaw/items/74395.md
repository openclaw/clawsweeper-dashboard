## Summary

Keep open. Current main still has cost-sensitive preview defaults and default-setting paths that can overwrite an existing primary outside the already-fixed `openclaw configure` path, and the issue discussion includes a second paid-model surprise report. This is not a cleanup close; it needs maintainer policy on preview/higher-cost default consent and likely narrower follow-up issues for the cron and tool-grounding requests.

## What This Changes

The issue asks OpenClaw to prevent upgrade and migration paths from silently replacing existing user model defaults with higher-cost preview defaults, and to add pricing-aware consent or warnings around those changes.

## Best Possible Solution

Define and document a model-default safety policy that preserves existing global and per-agent primaries and fallbacks across upgrade/setup paths unless the user explicitly accepts a preview or higher-cost replacement, separates provider ID normalization from default promotion, adds a visible warning or revert path for migration-set preview defaults, and splits cron control plus tool-grounding visibility into separate scoped follow-ups.

## Reproduction Assessment

Partial yes. Current main source and tests give a high-confidence reproduction of the remaining overwrite-by-default policy risk, but the exact historical billing event requires the reporter’s backup config files and session distribution logs.

## Solution Assessment

Unclear until maintainers choose the product policy. The narrowest maintainable direction is likely to make preservation/consent the default for migration and setup flows while keeping explicit `models set` and `--set-default` commands intentional.
