## Label Justifications

- `P1`: The issue reports real unintended paid model usage after persisted model routing changed, including background/long-running agent traffic and significant billing exposure.
- `impact:session-state`: The reported problem routes existing session, heartbeat, and cron-style work through a model choice that no longer matches recorded user intent.

## Summary

Keep open: current main has the narrow setup-preservation fix from the related merged PR, but it still uses `google/gemini-3.1-pro-preview` as the Google onboarding default, still normalizes Gemini Pro aliases to that preview id, and the public Google docs still recommend that default. The reporter's exact historical backup bundle is not attached, but the source path plus another user's similar comment support this as an unresolved billing-sensitive default/migration policy issue.

## What This Changes

The issue asks OpenClaw to stop silently changing persisted default model and fallback choices to paid Gemini preview variants during upgrade/setup without explicit consent or warning.

## Best Possible Solution

Maintainers should choose a billing-safe default and migration-consent policy, then align the Google plugin default/normalizer, setup or doctor warnings, public docs, and focused tests while preserving explicit user model choices.

## Reproduction Assessment

No for the exact historical upgrade artifact because the reporter's backup bundle is not attached. Yes at source level for the remaining policy gap: current main and `v2026.6.10` still choose and normalize Google Pro defaults to a preview model without an evident pricing-consent path.

## Solution Assessment

No, the merged setup-preservation fix is only a partial solution. The best fix needs a maintainer-approved default and billing-safety policy rather than another isolated provider-auth patch.
