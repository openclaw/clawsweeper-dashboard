## What This Changes

The PR changes CLI and gateway session-key resolution for `agentId` plus `sessionId`, adds regression tests, defers selected gateway startup work under fast-startup mode, and adjusts Bonjour unhandled-rejection cleanup behavior.

## Best Possible Solution

Rebase or replace the PR with a smaller current-main patch: keep the current session resolver and ingress-agent preservation, add fast-startup deferral only with explicit rejection logging and tests, touch the Bonjour plugin path only if still needed, and add the required changelog entry.

Yes. On the PR head, a gateway `agent` request with both `agentId` and a fresh `sessionId` clears `requestedSessionKey` without preserving `agentId` for ingress dispatch; deferred maintenance rejection under fast startup is also reproducible by making the background task reject and observing that `Promise.allSettled` consumes it without logging.

## Solution Assessment

No. The PR is not the best current solution because current main already solved the session-routing half differently, while the remaining startup work needs a narrower rebased implementation with logging, current Bonjour paths, robust env cleanup in tests, and changelog coverage.

- **[P1] Preserve agent routing for fresh session-id requests:** `src/gateway/server-methods/agent.ts:499-510`
  - body: When `sessionId` is present, this branch clears `requestedSessionKey`, but the PR head does not forward the validated `agentId` into the later ingress dispatch.   - confidence: 0.91
- **[P2] Log deferred startup maintenance failures:** `src/gateway/server-startup-plugins.ts:72-73`
  - body: `Promise.allSettled` consumes failures from channel maintenance and session migration, but the callback ignores rejected results. Operators get no warning when deferred startup work fails, so inspect the settled results and log rejected reasons.
  - confidence: 0.88
- **[P2] Clean up Bonjour rejection handlers on startup errors:** `src/infra/bonjour.ts:329-333`
  - body: The unhandled-rejection handler is registered before cycle creation, but this error path only restores console logging. If cycle creation or listener setup throws before an advertiser is returned, the process-level handler can leak for the rest of the process.
  - confidence: 0.84

  - body: This test sets `OPENCLAW_GATEWAY_FAST_CONFIG` and deletes it only on the happy path. If an import or assertion throws first, later tests inherit fast gateway mode; save the previous value and restore it in a `finally`.
  - confidence: 0.83

  - body: Several trace calls pass the bootstrap start time into a helper whose `stage_ms` is computed from the supplied start time, so those labels report elapsed-since-bootstrap rather than per-stage duration. Capture a local timestamp for each measured stage.
  - confidence: 0.78
