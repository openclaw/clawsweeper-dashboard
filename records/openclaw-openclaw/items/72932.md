## Summary

Keep this PR open, but do not merge it as-is. Current main already carries the core session-id routing repair in a safer form, while the PR head still has a gateway agent-routing regression, unlogged deferred-startup failures, stale Bonjour path work, and a missing changelog entry.

## What This Changes

The PR changes CLI and gateway session-key resolution for `agentId` plus `sessionId`, adds regression tests, defers selected gateway startup work under a fast-startup flag, and adjusts Bonjour unhandled-rejection cleanup behavior.

## Best Possible Solution

Rebase or replace the PR with a smaller patch against current main: keep the existing session resolver and gateway ingress agent preservation, add only the still-needed fast-startup deferral with explicit background-failure logging and robust tests, apply any Bonjour cleanup to `extensions/bonjour/src/advertiser.ts` only if still needed, and include the required changelog entry.

## Reproduction Assessment

Yes. The PR-head gateway regression is reproducible by static call-path inspection: send gateway `agent` params with both `agentId` and a fresh `sessionId`, and the PR clears `requestedSessionKey` without preserving `agentId` for ingress dispatch. The startup observability bug is reproducible by making deferred maintenance reject under fast mode and observing that `Promise.allSettled` consumes the rejection without logging it.

## Solution Assessment

No. The current PR is not the best way to solve the issue because current main already solved the session-routing half differently and the remaining fast-startup work needs a narrower, rebased implementation with logging, current Bonjour paths, and changelog coverage.

Full review comments:

- **[P1] Preserve agent routing for fresh session-id requests:** `src/gateway/server-methods/agent.ts:499-510`
  - body: When `sessionId` is present, this branch clears `requestedSessionKey`, but the PR head does not forward the validated `agentId` into the later ingress dispatch. A fresh request for a named agent can therefore run or store under the default agent; pass the validated agent id through or derive an explicit agent session key.
  - confidence: 0.91
- **[P2] Log deferred startup maintenance failures:** `src/gateway/server-startup-plugins.ts:72-73`
  - body: `Promise.allSettled` consumes failures from channel maintenance and session migration, but the callback ignores rejected results. Operators get no warning when deferred startup work fails, so inspect the settled results and log rejected reasons.
  - confidence: 0.88
- **[P2] Clean up Bonjour rejection handlers on startup errors:** `src/infra/bonjour.ts:332-333`
  - body: The unhandled-rejection handler is registered before cycle creation, but the PR error path only restores console logging. If cycle creation or listener setup throws before an advertiser is returned, the process-level handler can leak for the rest of the process.
  - confidence: 0.84
- **[P3] Restore the fast-startup env var in the test:** `src/gateway/server-startup-post-attach.test.ts:307`
  - body: This test sets `OPENCLAW_GATEWAY_FAST_CONFIG` and deletes it only on the happy path. If an import or assertion throws first, later tests inherit fast gateway mode; save the previous value and restore it in a `finally`.
  - confidence: 0.83
- **[P3] Measure startup trace stages from local starts:** `src/gateway/server-startup-plugins.ts:82`
  - body: Several trace calls pass the bootstrap start time into a helper whose `stage_ms` is computed from the supplied start time, so those stage labels report elapsed-since-bootstrap rather than per-stage duration. Capture a local timestamp for each measured stage.
  - confidence: 0.78
- **[P3] Add the required changelog entry:** `src/gateway/server-methods/agent.ts:499`
  - body: This PR changes user-facing gateway session routing and startup behavior but does not update `CHANGELOG.md`. Add a single-line entry under the active `### Fixes` section with contributor credit before merge.
  - confidence: 0.86
