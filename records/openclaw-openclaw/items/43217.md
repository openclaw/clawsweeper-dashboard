## Summary

Keep open. Current main still does not implement the PR's intended resolved-model `model.reasoning` gate in the active shared Google thinking sanitizer; the PR remains useful but needs rebasing/reworking onto the Plugin SDK shared wrapper and should align Gemini 2.5 fixed-level behavior with the native Google transport's budget mapping.

## What This Changes

This PR changes the older embedded-runner Google thinking sanitizer to gate fallback `thinkingLevel` injection on `model.reasoning` instead of Gemini 3.1 model-id matching, and adds Gemini thinking regression coverage plus live-test wording updates.

## Best Possible Solution

Rework the useful branch onto the current shared Google thinking wrapper: pass resolved model capability into `sanitizeGoogleThinkingPayload`, preserve the existing Gemini 3, Gemma 4, and adaptive Gemini 2.5 paths, map Gemini 2.5 fixed levels consistently with the native Google transport, and add focused wrapper tests for reasoning-enabled and non-reasoning models.

## Reproduction Assessment

Yes for the sanitizer state transition: on current main, a `gemini-2.5-flash` payload with `thinkingConfig: { includeThoughts: true, thinkingBudget: -1 }` and a fixed thinking level reaches the generic negative-budget fallback, which deletes the budget without adding a replacement capability-aware control. The live user-visible reasoning leak was not reproduced in this read-only review.

## Solution Assessment

No as submitted against current main. The intended capability gate is the right direction, but the best current fix belongs in `src/plugin-sdk/provider-stream-shared.ts` and should use Gemini 2.5 budget mapping rather than copying the old `thinkingLevel` injection literally.
