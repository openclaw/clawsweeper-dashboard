## What This Changes

The PR changes the embedded-runner Google thinking sanitizer and tests to use `model.reasoning` instead of Gemini 3.1 model-id matching when replacing negative `thinkingBudget` with `thinkingLevel`.

## Best Possible Solution

Rework the fix in the shared Google wrapper with model-family-aware mapping: Gemini 3 and Gemma 4 use supported `thinkingLevel` paths, Gemini 2.5 uses `thinkingBudget`, and the PR includes focused wrapper tests plus redacted real behavior proof.
