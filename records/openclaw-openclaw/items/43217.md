## Best Possible Solution

Rework the fix in `src/plugin-sdk/provider-stream-shared.ts` with model-family-aware Google thinking mapping, focused shared-wrapper regression coverage, and attached redacted real behavior proof.

## Reproduction Assessment

Yes for a source-level path: current main can be traced through the shared Google wrapper to a Gemini 2.5 negative-budget state where the sanitizer deletes `thinkingBudget` without a capability-aware replacement. The live rawThinking/rawText leak was not reproduced here.

## Solution Assessment

No as submitted. The model-capability direction is useful, but the patch must move to the shared SDK wrapper and preserve Gemini 2.5 `thinkingBudget` versus Gemini 3 `thinkingLevel` behavior. ([ai.google.dev](https://ai.google.dev/gemini-api/docs/thinking))
