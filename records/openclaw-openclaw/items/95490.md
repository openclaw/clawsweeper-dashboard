## Summary

Keep open. The protected `maintainer` label blocks conservative cleanup closure, and current `main` plus v2026.6.9 still contain the source-level path where a Slack/Claude CLI final can be persisted as a normal transcript reply before foreground freshness decides whether visible channel delivery is suppressed.

## What This Changes

The issue asks OpenClaw to prevent or explicitly account for Slack-visible loss of a completed Claude CLI assistant final that was already persisted to the session transcript.

## Best Possible Solution

Choose and land one maintainer-approved invariant for normal CLI assistant finals: a transcript row that looks visible must either reach Slack or carry explicit suppressed-delivery metadata, while preserving stale-reply suppression for truly obsolete deliveries.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No live deterministic Slack/Claude CLI reproduction was established; source inspection confirms the implicated persistence, steer fallback, backend, and freshness-fence paths on current main and v2026.6.9.

## Solution Assessment

Is this the best way to solve the issue? Unclear; the expected invariant is sound, but maintainers need to choose whether the fix belongs in lazy/active foreground fencing, CLI live steering, transcript persistence timing, or explicit suppressed-delivery metadata.
