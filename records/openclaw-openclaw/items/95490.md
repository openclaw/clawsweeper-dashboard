## What This Changes

The issue reports a Slack/Claude CLI race where a completed assistant final can be persisted to the session transcript but suppressed before visible Slack delivery when a newer same-thread followup advances the foreground delivery fence.

## Best Possible Solution

Keep this as the Slack/CLI transcript-delivery invariant bug and make transcript state agree with visible delivery, likely by recording explicit suppressed-delivery metadata or moving suppression ahead of persistence while preserving the foreground freshness fence.

## Reproduction Assessment

Not yet for a live Slack run. Source inspection gives a credible reproduction path through CLI transcript persistence, failed CLI steering injection, followup admission, and foreground freshness cancellation, but the exact race still needs live channel proof.

## Solution Assessment

Unclear. The requested invariant is correct, but the maintainable fix needs an owner decision between CLI steering semantics, transcript-persistence timing, and explicit suppressed-delivery metadata.
