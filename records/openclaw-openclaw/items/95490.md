## What This Changes

The issue asks OpenClaw to prevent Slack-visible delivery and persisted CLI assistant transcript state from drifting when a followup arrives during an active CLI reply run.

## Best Possible Solution

Maintain one invariant for normal CLI assistant finals: a transcript row that looks visible must either be delivered to Slack or carry explicit suppressed-delivery metadata, while preserving the foreground freshness fence for truly stale replies.
