## What This Changes

The issue reports a Slack/Claude CLI delivery race where a normal assistant final can be persisted to the session transcript but suppressed before visible Slack delivery when a follow-up starts a newer foreground generation.

## Best Possible Solution

Land or sponsor one maintainer-approved foreground-delivery policy that prevents queued same-session follow-ups from silently dropping active Slack output, then preserve a transcript invariant: normal visible CLI assistant rows must either be delivered or carry explicit suppressed-delivery metadata.
