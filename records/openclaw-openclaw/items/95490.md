## Summary

Keep open. The issue has the protected `maintainer` label, and current main plus v2026.6.10 still lack the requested invariant: a CLI-owned final can be persisted as a normal assistant transcript row while stale foreground cancellation returns before visible delivery without a suppressed-delivery transcript marker.

## What This Changes

The issue reports a Slack/Claude CLI race where an assistant final can be persisted to the transcript but never visibly delivered after a follow-up creates a newer foreground generation.

## Best Possible Solution

Add maintainer-approved bookkeeping so a CLI-owned final and Slack-visible delivery cannot disagree: either suppress before normal transcript persistence or record an explicit stale-foreground suppressed-delivery marker while keeping the current foreground freshness fence.

## Reproduction Assessment

No live Slack repro was rerun here. Yes at source level: current main still persists CLI finals before delivery settles, still allows foreground freshness to cancel older finals at `beforeDeliver`, and still skips transcript mirroring when final cancellation increases.

## Solution Assessment

Unclear as an implementation choice. The desired invariant is right, but maintainers need to choose whether the fix belongs in CLI persistence timing, foreground suppression metadata, or explicit suppressed-delivery transcript mirroring.
