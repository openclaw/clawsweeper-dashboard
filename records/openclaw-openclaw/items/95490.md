## What This Changes

The issue asks OpenClaw to prevent or account for a Slack race where a CLI assistant final reply is persisted to the transcript but suppressed before visible Slack delivery when a followup arrives in the same thread.

## Best Possible Solution

A maintainer-owned fix should make normal assistant transcript rows and visible Slack delivery agree, either by moving suppression before persistence or by recording explicit suppressed-delivery metadata while preserving the foreground freshness fence.

## Reproduction Assessment

No live deterministic reproduction was established in this read-only pass. Source inspection does show the reported components on current main and the latest release: CLI transcript persistence, optional `queueMessage` steering, queued followups, and foreground final cancellation.
