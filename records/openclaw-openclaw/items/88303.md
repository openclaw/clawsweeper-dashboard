Centralize or share the disabled-browser cleanup predicate so both reset paths skip cleanup for `browser.enabled=false` and `plugins.entries.browser.enabled=false`, while preserving default/enabled cleanup behavior.

Yes by source inspection: current main and the latest release call the browser maintenance facade during rollover/reset whenever session keys exist, even when browser support is disabled. I did not rerun the real PVC repro in this read-only review.

## Solution Assessment
