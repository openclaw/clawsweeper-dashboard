Full review comments:

- **[P1] Keep user-profile shims out of PATH prepend:** `src/infra/path-env.ts:110-113`
  - body: This puts `%LOCALAPPDATA%` pnpm/WindowsApps and `%APPDATA%` npm ahead of the existing PATH. Current main deliberately keeps user-writable/package-manager dirs after trusted OS dirs so they cannot shadow system or already-configured binaries, so this can break command resolution and weaken PATH hijack protection.
  - confidence: 0.91
- **[P1] Validate Windows install roots before trusting them:** `src/infra/path-env.ts:105-106`
  - body: The new branch reads `SystemRoot`/`WINDIR` and Program Files values directly from `process.env` and then prepends child paths. Current main already has `getWindowsInstallRoots`/`getWindowsProgramFilesRoots` to reject UNC, drive-relative, PATH-like, and malformed roots before they influence trusted command lookup; this PR should use that contract.
  - confidence: 0.88
- **[P2] Parse the Windows test PATH with the Windows delimiter:** `src/infra/path-env.test.ts:263`
  - body: The helper now joins a simulated `win32` PATH with `;`, but the assertion splits with the host `path.delimiter`. On Linux CI, drive-letter paths like `C:/...` are split on `:`, so the expected Windows directories are not found and the regression test can fail off Windows.
  - confidence: 0.93
- **[P2] Remove the stale gateway push label hunk:** `src/config/schema.labels.ts:80-84`
  - body: This PATH bootstrap PR also adds gateway push schema labels, which are unrelated to the fix and already exist on current main with slightly different wording. Keeping this hunk creates conflict/duplicate label churn unrelated to Windows PATH behavior.
  - confidence: 0.86
