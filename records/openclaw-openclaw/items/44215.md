## Summary

Keep open: the underlying Windows PATH bootstrap gap still appears valid on current main, but this branch is not merge-ready because it conflicts with current PATH precedence hardening, has a POSIX CI delimiter bug, carries stale schema-label churn, and lacks real Windows behavior proof.

## What This Changes

The PR adds Windows PATH candidates and platform delimiter handling in `src/infra/path-env.ts`, a simulated Windows regression test, and gateway push field labels.

## Best Possible Solution

Rework Windows PATH bootstrap within the current hardening model: trusted system/runtime roots may precede PATH, profile/package-manager dirs should remain append-only, the test should parse win32 PATH with semicolons, unrelated label churn should be removed, and real Windows proof should be attached.

## Reproduction Assessment

Yes. Source inspection shows current main still has no `win32` branch in `candidateBinDirs`, and the linked report describes the minimal-Windows PATH symptom; I did not run a live Windows shell in this read-only review.

## Solution Assessment

No. Windows bootstrap support is the right direction, but this patch should preserve main's PATH-precedence hardening, fix the win32 delimiter assertion, drop the stale label hunk, and include real Windows proof before merge.

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Keep profile-owned Windows shims append-only:** `src/infra/path-env.ts:109-116`
  - body: This adds `%LOCALAPPDATA%\pnpm`, WindowsApps, and `%APPDATA%\npm` to `prepend`, so OpenClaw subprocesses can resolve profile-owned shims before the existing PATH and system roots. Current PATH hardening keeps comparable user-writable/package-manager locations append-only to reduce binary-shadowing risk.
  - confidence: 0.92

  - body: The helper now joins PATH with `;` for `platform: "win32"`, but this test still splits using the host delimiter. On Linux/macOS CI, `C:/...` entries are split at the drive colon, so the test fails independent of the intended behavior.
  - confidence: 0.94

  - confidence: 0.86
