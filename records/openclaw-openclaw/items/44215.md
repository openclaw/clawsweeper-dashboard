Contributor-supplied real Windows behavior proof is also still missing.

## What This Changes

The PR adds Windows-specific PATH bootstrap candidates and platform delimiter handling in `src/infra/path-env.ts`, adds a simulated Windows PATH regression test, and adds gateway push field labels.

## Best Possible Solution

Add Windows PATH bootstrap within the current hardening model: trusted system/runtime roots can precede PATH, profile/package-manager dirs remain append-only, tests parse win32 PATH with semicolons, unrelated label churn is removed, and a real Windows proof artifact is provided.

## Solution Assessment
