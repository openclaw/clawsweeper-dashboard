## Summary

Keep open. The Windows PATH gap appears real, but this patch should not merge as-is because it conflicts with current PATH hardening, has a delimiter-broken regression test, carries an unrelated stale schema-label hunk, and lacks external real Windows behavior proof.

## What This Changes

The branch adds Windows-specific PATH bootstrap candidates and win32 delimiter merging, a focused path-env test, and gateway push schema labels.

## Best Possible Solution

Rebase and implement Windows support within the current PATH hardening model: prepend only trusted runtime/system roots, append user-profile package-manager dirs, fix semicolon-aware tests, drop the schema-label hunk, and add redacted real Windows proof.

## Reproduction Assessment

Yes. Source inspection shows current main lacks Windows-specific bootstrap candidates, and the PR test failure is source-reproducible because it simulates win32 PATH joining but splits with the host delimiter.

## Solution Assessment

No. Windows bootstrap support is the right direction, but this implementation violates the current PATH hardening policy and includes unrelated stale config-label churn.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep user-profile Windows shims append-only:** `src/infra/path-env.ts:109-116`
  - body: This adds `%LOCALAPPDATA%` pnpm/WindowsApps and `%APPDATA%\npm` to `prepend`, so OpenClaw subprocesses would resolve user-profile shims before existing PATH and trusted system dirs. Current PATH hardening keeps user-writable/package-manager locations append-only to avoid shadowing system binaries; these entries should be appended or explicitly opted in instead.
  - confidence: 0.92
- **[P1] Split the simulated Windows PATH with semicolons:** `src/infra/path-env.test.ts:265`
  - body: This test forces `platform: "win32"`, and production now joins PATH entries with `;`, but the assertion still splits with host `path.delimiter`. On Linux/macOS CI that splits `C:/...` drive paths at colons, so the regression test fails independent of the intended Windows behavior.
  - confidence: 0.9
- **[P2] Drop the stale gateway push labels:** `src/config/schema.labels.ts:83-87`
  - body: Current main already defines these `gateway.push.*` labels later in `FIELD_LABELS`, so this unrelated hunk would duplicate keys with older wording when rebased. Remove the schema-label changes from the PATH bootstrap PR.
  - confidence: 0.86
