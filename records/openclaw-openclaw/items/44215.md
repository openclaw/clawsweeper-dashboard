## Summary

The PR should stay open because current `main` still lacks Windows-specific PATH bootstrap behavior, but this branch is not mergeable as-is: it conflicts with current PATH hardening, has a failing cross-platform regression test, includes unrelated stale schema-label churn, and still lacks after-fix real Windows proof.

## What This Changes

The branch adds Windows PATH bootstrap candidates for common user, Program Files, and SystemRoot directories, makes `mergePath` accept a platform delimiter, adds a Windows path-env test, and adds unrelated gateway push labels.

## Best Possible Solution

Add Windows bootstrap through the current hardening model: trust only validated system/runtime roots for prepend, append user-profile package-manager locations, keep tests delimiter-aware, and leave unrelated config labels out.

## Reproduction Assessment

Yes, from source inspection: current `candidateBinDirs` has no `win32` branch, while the PR's own CI also reproduces a test failure in the new Windows case on Ubuntu. I did not run a real Windows shell repro in this read-only review.

## Solution Assessment

No. Windows PATH bootstrap support is useful, but this patch should be reworked to match current command-precedence hardening, fix the semicolon split in the test, and remove unrelated schema-label changes before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Keep user-profile Windows shims append-only:** `src/infra/path-env.ts:109-116`
  - body: This adds `%LOCALAPPDATA%\pnpm`, WindowsApps, and `%APPDATA%\npm` to `prepend`, so OpenClaw subprocesses will resolve user-profile shims ahead of the existing PATH and system roots. Current PATH hardening keeps user-writable/package-manager locations append-only to reduce binary-shadowing risk; the Windows user-profile entries need the same treatment.
  - confidence: 0.92
- **[P1] Split simulated Windows PATH with semicolons:** `src/infra/path-env.test.ts:265`
  - body: The helper now joins PATH with `;` when `platform: "win32"`, but this test still splits with host `path.delimiter`. On Linux/macOS CI that delimiter is `:`, so `C:/...` paths are split at the drive colon and this regression test fails independent of the intended behavior.
  - confidence: 0.94
- **[P2] Drop the duplicate gateway push labels:** `src/config/schema.labels.ts:83-87`
  - body: This PATH bootstrap PR also adds `gateway.push.*` labels near the auth labels, but current main already defines those keys later in `FIELD_LABELS`. When rebased, this hunk is unrelated stale churn and would duplicate or overwrite the existing gateway push wording.
  - confidence: 0.86
