## Summary

Keep open: current main still lacks Windows PATH bootstrap support, but this PR is not mergeable as-is because it changes command-resolution precedence, has a Windows delimiter test bug, includes stale schema-label churn, and still lacks real Windows behavior proof.

## What This Changes

The PR adds Windows-specific PATH bootstrap candidates and platform-specific PATH delimiters, plus a Windows-focused path-env test and gateway push field-label entries.

## Best Possible Solution

Add Windows bootstrap through the current PATH-hardening model: trusted Windows system roots first, user-profile shims append-only, delimiter-aware tests, no unrelated schema-label churn, and real Windows proof.

## Reproduction Assessment

Yes, source-reproducible: current main has no win32 bootstrap branch in candidateBinDirs, and the linked Windows report describes missing standard tool locations. I did not establish a live Windows reproduction in this read-only review.

## Solution Assessment

No. Windows bootstrap support is useful, but this patch should preserve PATH-precedence hardening, parse simulated Windows PATHs with ';', remove stale duplicate labels, and include real Windows proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Keep profile-owned Windows shims append-only:** `src/infra/path-env.ts:109-116`
  - body: This puts %LOCALAPPDATA%\pnpm, WindowsApps, and %APPDATA%\npm in prepend, so OpenClaw subprocesses can resolve profile-owned shims before the existing PATH and trusted roots. Current main keeps comparable user-writable/package-manager locations append-only to reduce binary-shadowing risk.
  - confidence: 0.92
- **[P1] Split the simulated Windows PATH with semicolons:** `src/infra/path-env.test.ts:265`
  - body: The helper joins PATH with ';' for platform: "win32", but this assertion splits with the host delimiter. On Linux/macOS CI, fake C:/... entries are split at the drive colon, so the test can fail for reasons unrelated to the intended behavior.
  - confidence: 0.94

  - body: This PATH bootstrap PR also adds gateway push labels near the auth labels, but current main already defines those keys later in FIELD_LABELS. The unrelated stale hunk would create duplicate object keys when rebased or merged.
  - confidence: 0.86
