## Summary

Keep open: current main still lacks Windows-specific PATH bootstrap, but this PR is not merge-ready because it conflicts with the existing PATH hardening policy, has a host-delimiter test bug, and includes a stale unrelated schema-label hunk.

## What This Changes

This PR adds a Windows-specific PATH bootstrap branch, platform delimiter handling for PATH merging, a Windows regression test, and gateway push schema labels.

## Best Possible Solution

Add Windows bootstrap support in a replacement or revised patch that preserves PATH hardening: trusted runtime/system directories can be searched ahead of PATH, user-profile shims should be appended or explicitly opted in, tests should use Windows delimiters deterministically, and unrelated schema labels should be dropped.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main has no Windows branch in `candidateBinDirs`, and the PR test issue is source-backed because the simulated win32 PATH is joined with semicolons while the assertion still uses the host delimiter.

## Solution Assessment

No. Windows PATH bootstrap support is a valid fix direction, but this patch is not the best solution because it elevates user-writable Windows shim directories ahead of trusted paths and mixes an unrelated stale schema-label hunk into the branch.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Avoid prepending user-writable Windows shims:** `src/infra/path-env.ts:108-117`
  - body: The win32 branch puts `%LOCALAPPDATA%\pnpm`, WindowsApps, and `%APPDATA%\npm` in `prepend`, so OpenClaw subprocesses would resolve user-profile shims before the existing PATH and trusted system dirs. Current main explicitly appends user/package-manager dirs to avoid PATH hijacking, so this weakens the command-resolution model.
  - confidence: 0.92
- **[P1] Split simulated Windows PATH with semicolons:** `src/infra/path-env.test.ts:265`
  - body: The test forces `platform: "win32"`, and the helper joins PATH entries with `;`, but the assertion still splits with host `path.delimiter`. On Linux/macOS CI, `C:/...` entries split at drive-letter colons, so the regression test fails independent of the intended Windows behavior.
  - confidence: 0.88
- **[P2] Drop the unrelated gateway push label hunk:** `src/config/schema.labels.ts:80-84`
  - body: The `gateway.push.*` label additions are unrelated to this PATH change, and current main already contains those labels. Keeping the stale hunk increases rebase conflict risk and makes the PATH fix harder to review.
  - confidence: 0.81
