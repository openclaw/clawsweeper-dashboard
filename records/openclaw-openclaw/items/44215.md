## Summary

Keep open: current main still lacks Windows PATH bootstrap support, but this PR has blocking PATH precedence, test delimiter, and unrelated stale-label issues, plus no contributor-supplied real Windows proof.

## What This Changes

The PR adds Windows-specific PATH bootstrap candidates and platform delimiter handling in `src/infra/path-env.ts`, adds a simulated Windows PATH regression test, and adds gateway push field labels.

## Best Possible Solution

Add Windows bootstrap under the current hardening policy: system/runtime roots may precede PATH, profile/package-manager dirs stay append-only, tests parse win32 PATH correctly, unrelated label churn is removed, and real Windows proof is attached.

## Reproduction Assessment

Yes. Source inspection shows current main has no `win32` branch in `candidateBinDirs`, and the PR's simulated test captures the minimal-Windows PATH case; I did not run a live Windows repro in this read-only review.

## Solution Assessment

No. Windows bootstrap support is the right direction, but this patch should preserve PATH-precedence hardening, fix the win32 delimiter assertion, remove duplicate labels, and include real Windows proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Keep profile-owned Windows shims append-only:** `src/infra/path-env.ts:109-116`
  - body: This adds `%LOCALAPPDATA%\pnpm`, WindowsApps, and `%APPDATA%\npm` to `prepend`, so OpenClaw subprocesses can resolve profile-owned shims before existing PATH and system roots. Current PATH hardening keeps comparable user-writable/package-manager locations append-only to reduce binary-shadowing risk.
  - confidence: 0.92
- **[P1] Split the simulated Windows PATH with semicolons:** `src/infra/path-env.test.ts:265`
  - body: The helper now joins PATH with `;` for `platform: "win32"`, but this test still splits using the host delimiter. On Linux/macOS CI, `C:/...` entries are split at the drive colon, so the regression test fails independent of the intended behavior.
  - confidence: 0.94

  - body: This PATH bootstrap PR also adds `gateway.push` labels near the auth labels, but current main already defines those keys later in `FIELD_LABELS`. The hunk is unrelated stale churn and would duplicate the current keys when rebased.
  - confidence: 0.86
