## Summary

Keep open: current main still lacks Windows-specific PATH bootstrapping, but this branch should not merge as-is because it prepends user-writable Windows shims, has a delimiter-broken regression test, and carries schema-label churn that current main already has elsewhere.

## What This Changes

The branch adds win32 PATH delimiter handling and Windows bootstrap candidate directories in src/infra/path-env.ts, adds a focused regression test, and adds gateway push schema labels.

## Best Possible Solution

Revise Windows bootstrap to prepend only trusted runtime/system roots, append or explicitly opt in user-profile package-manager dirs, fix delimiter-aware tests, and drop the duplicate schema-label hunk.

## Reproduction Assessment

Yes. Source inspection shows current main still has no win32-specific bootstrap branch, and the PR's test bug is source-reproducible because the win32 PATH is joined with ';' but split with the host delimiter.

## Solution Assessment

No. Windows bootstrap support is the right direction, but this implementation conflicts with the current PATH hardening model and includes unrelated duplicate schema labels.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Append user-writable Windows shims:** `src/infra/path-env.ts:109-116`
  - body: The win32 branch adds LOCALAPPDATA pnpm, WindowsApps, and APPDATA npm to `prepend`, so OpenClaw subprocesses would resolve user-profile shims before the existing PATH and trusted system dirs. Current main keeps user-writable/package-manager locations append-only to avoid PATH hijacking, so these entries should be appended or explicitly opted in instead.
  - confidence: 0.92
- **[P1] Split the simulated Windows PATH with semicolons:** `src/infra/path-env.test.ts:265`
  - body: This test forces `platform: "win32"`, and production now joins PATH entries with `;`, but the assertion still splits with host `path.delimiter`. On Linux/macOS CI that delimiter is `:`, so `C:/...` entries are split at drive-letter colons and the regression test fails independent of the intended behavior.
  - confidence: 0.9
- **[P2] Drop the duplicate gateway push labels:** `src/config/schema.labels.ts:83-87`
  - body: Current main already defines these `gateway.push.*` labels later in `FIELD_LABELS`, so this stale hunk would introduce duplicate keys with older wording when rebased. Remove the unrelated label changes from the PATH bootstrap PR.
  - confidence: 0.86
