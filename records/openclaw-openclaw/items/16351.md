## What This Changes

The issue asks for `openclaw config` to avoid surprising wizard entry, validate config files explicitly, and stop `config set` from persisting resolved environment-backed secrets.

## Best Possible Solution

Keep the shipped validation, SecretRef builder, and env-placeholder preservation, then make a maintainer-approved CLI semantics and literal-secret policy change with focused docs and regression coverage.

## Reproduction Assessment

Yes. Source inspection shows bare `openclaw config` still routes into the configure wizard and value mode can still persist raw strings, while `config validate` and existing placeholder restoration are already present.

## Solution Assessment

Unclear until maintainers decide the product and security policy. Showing help for the root command and tightening literal secret writes are plausible, but both change documented CLI or credential-writing semantics.
