## Summary

Keep this open. Current main has implemented the validation and existing env-placeholder preservation pieces, but bare `openclaw config` still intentionally dispatches to the configure wizard and value-mode secret-writing policy still needs maintainer direction.

## What This Changes

The issue requests safer `openclaw config` behavior: show help instead of entering the wizard, add config validation, and avoid persisting resolved environment-backed secrets.

## Best Possible Solution

Keep the implemented validation and env-preservation work, then make a maintainer-approved CLI semantics and secret-writing policy change with matching docs and focused regression tests.
