Keep this issue open. Current main has implemented the validation and existing env-placeholder preservation pieces, but bare `openclaw config` still intentionally opens the guided wizard and value-mode secret-writing policy still needs maintainer direction.

## What This Changes

Request to make `openclaw config` safer by showing help instead of the wizard on configured systems, adding config validation, and avoiding plaintext secret write-back.

## Best Possible Solution

Keep the implemented validation, env-placeholder restoration, and SecretRef builder paths, then make a maintainer-approved CLI behavior and secret-writing policy change with matching docs and focused regression tests.
