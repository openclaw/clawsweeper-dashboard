## Summary

Keep open. Current main has implemented the validation and env-placeholder preservation parts, but bare `openclaw config` is still intentionally implemented and documented as the guided configure wizard entrypoint, so the remaining change needs maintainer product/compatibility direction.

## What This Changes

Request to make bare `openclaw config` show help on configured systems, add explicit config validation, and keep `config set` from persisting resolved env secrets.

## Best Possible Solution

Keep the implemented validation and env-restoration behavior, then make an explicit maintainer decision on root `config` semantics and update CLI code, focused tests, and config/configure docs together.

## Reproduction Assessment

Yes for the remaining root-command behavior: current source shows bare `openclaw config` dispatching to `configureCommandFromSectionsArg`, and empty sections call the configure wizard. Source and docs agree, so a live CLI run was not necessary for this read-only review.

## Solution Assessment

Unclear until maintainer direction is given. The implemented validate and placeholder-restoration pieces are appropriate, but changing the documented root command from wizard entrypoint to help output is a compatibility/product decision rather than a narrow bug fix.

Status: needs_attention
