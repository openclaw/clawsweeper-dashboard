## Summary

Keep this issue open. Current main covers the validation and env-reference write-back parts, but bare `openclaw config` is still implemented and documented as the guided configure wizard entrypoint, and the remaining secret-safe value-mode behavior needs maintainer direction.

## What This Changes

This issue requests safer `openclaw config` behavior: show help instead of starting the wizard for exploratory root invocations, add explicit validation, and avoid persisting resolved env placeholders as plaintext secrets.

## Best Possible Solution

Keep the implemented validation and env-preservation pieces, then make a maintainer decision on the root command semantics and update CLI behavior, docs, tests, and changelog together.
