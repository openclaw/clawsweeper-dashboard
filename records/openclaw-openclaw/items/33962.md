## Summary

Keep open: current main and the latest release still route opt-in LLM slug generation through the default agent model and embedded session lane. A related open PR targets this path, but it has not landed and the lightweight-model policy remains unresolved.

## What This Changes

Request to route session-memory LLM filename slug generation to a lightweight or dedicated model, or a fast fallback, instead of the agent primary model to avoid lane congestion.

## Best Possible Solution

Land a focused slug-generation policy that avoids embedded-runner/default-primary routing for opt-in descriptive slugs, preserves timestamp defaults and timeout/fallback behavior, and updates docs/tests.

## Reproduction Assessment

Yes for source-level behavior: enable hooks.internal.entries.session-memory.llmSlug and current main calls generateSlugViaLLM, resolves the default agent model, and runs the embedded agent on temp:slug-generator. I did not live-reproduce the multi-agent queue buildup.

## Solution Assessment

Unclear. The open completeSimple PR is a plausible direction, but the model policy and simple-completion runtime/auth contracts need maintainer resolution before this is the best fix.
