## Summary

Keep open: current main mitigates the default reset hot path by making LLM slugs opt-in and non-blocking, but the opt-in slug generator still uses the default agent model through the embedded runner, and the direct-completion implementation candidate is still open at https://github.com/openclaw/openclaw/pull/65134.

## What This Changes

Route session-memory LLM slug generation away from the agent primary model, preferably through a lightweight dedicated slug model or fast fallback, to reduce reset-time lane congestion.

## Best Possible Solution

Land or replace the open direct-completion work with a focused slug-generator change that avoids the embedded runner, preserves timeout/fallback semantics, documents the chosen model-routing policy, and keeps the default timestamp path fast.

## Reproduction Assessment

Yes for the source-level behavior: when `llmSlug` is enabled, current main still routes through `resolveDefaultModelForAgent` and `runEmbeddedPiAgent` on `temp:slug-generator`. I did not establish a live multi-agent queue reproduction.

## Solution Assessment

No. The current main mitigation is useful for the default reset path, but it does not implement the requested lightweight model routing for opt-in descriptive slug generation; the open direct-completion PR is a plausible direction but still needs maintainer review.
