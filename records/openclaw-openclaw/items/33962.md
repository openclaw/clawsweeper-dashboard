## Summary

Keep open: current main and the latest release still route enabled LLM slugs through the configured default agent model and `runEmbeddedPiAgent`; timestamp-default/background mitigations reduce the default blast radius but do not solve lightweight slug routing, and the focused replacement PR remains open.

## What This Changes

Request lightweight, decoupled slug generation so session-memory filename slugs do not use the primary agent model or congest lanes during resets.

## Best Possible Solution

Choose and land one focused slug-generation policy that avoids embedded-runner/default-primary routing for optional descriptive slugs while preserving timestamp defaults, background hook behavior, timeout fallback, auth/provider contracts, and focused coverage.

## Reproduction Assessment

Yes at source level: enable `hooks.internal.entries.session-memory.llmSlug`, and session memory reaches `generateSlugViaLLM`, which resolves the default model and runs `runEmbeddedPiAgent` on `temp:slug-generator`. I did not live-reproduce the reported multi-agent lane buildup.

## Solution Assessment

Unclear. Direct simple completion and heartbeat-model reuse are plausible, but maintainers still need to choose whether the right policy is a dedicated slug model key, an existing lightweight model knob, a shorter heuristic fallback, or some combination.
