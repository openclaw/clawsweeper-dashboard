## Summary

Keep open: current main mitigates the default reset path by keeping timestamp slugs as the default, but opt-in LLM slugs still use the default agent model through the embedded runner and the related open PR is the active fix path.

## What This Changes

The issue requests lightweight or dedicated-model routing for LLM slug generation so session-memory filename generation does not compete with the primary agent model lane.

## Best Possible Solution

Land a focused slug-generation policy that avoids embedded-runner/default-primary-model routing for descriptive slugs while preserving timestamp defaults, timeout/fallback behavior, and docs.

## Reproduction Assessment

Yes for source-level behavior: with `llmSlug` enabled, current main resolves the default agent model and runs `runEmbeddedPiAgent` on `temp:slug-generator`. I did not live-reproduce the multi-agent queue buildup.

## Solution Assessment

Unclear. The open PR direction is plausible, but the final fix should follow current simple-completion auth/error/timeout contracts and settle the lightweight model policy deliberately.
