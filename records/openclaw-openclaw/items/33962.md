## Summary

Keep open: current main avoids the default reset hot path by making descriptive LLM slugs opt-in and backgrounded, but opt-in slugs still resolve the default agent model and run through the embedded runner on the `temp:slug-generator` lane. The linked open PR is the active implementation path for removing that runner/model-routing bottleneck.

## What This Changes

The issue requests lightweight or dedicated slug-generation model routing so filename slug generation does not use the agent primary model or congest shared lanes.

## Best Possible Solution

Choose and land a focused slug-generation policy that avoids the embedded-runner/default-primary-model path for descriptive slugs while preserving timestamp defaults, timeout/fallback behavior, and docs.

## Reproduction Assessment

Yes for source-level behavior: with `llmSlug` enabled, current main resolves the default agent model and runs `runEmbeddedPiAgent` on `temp:slug-generator`. I did not live-reproduce the multi-agent queue buildup.

## Solution Assessment

No. Current main is only a mitigation for the default path; the requested lightweight or dedicated model routing remains unresolved, and the right end state needs a maintainer policy choice.
