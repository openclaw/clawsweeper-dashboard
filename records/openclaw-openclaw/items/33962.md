## Summary

Keep open: current main mitigates the default reset hot path by making LLM slugs opt-in/background, but the requested lightweight or dedicated model routing remains unresolved because opt-in `llmSlug` still uses the default configured model through the embedded runner, and https://github.com/openclaw/openclaw/pull/65134 is still the open implementation candidate.

## What This Changes

The issue requests a lightweight or dedicated slug-generation model path so session-memory filename generation does not use the agent primary model and congest shared lanes.

## Best Possible Solution

Land or replace the open implementation with a focused slug-generator path that avoids the embedded runner/default primary model for descriptive slugs, preserves the fast timestamp default and timeout/fallback semantics, and documents the chosen model-routing policy.

## Reproduction Assessment

Yes for source-level behavior: with `llmSlug` enabled, current main resolves the default model and runs `runEmbeddedPiAgent` on `temp:slug-generator`. I did not live-reproduce a multi-agent queue buildup.

## Solution Assessment

No, current main is only a partial mitigation: default timestamp slugs avoid the hot-path model call, but opt-in descriptive slugs still lack lightweight model routing. The direct-completion direction in https://github.com/openclaw/openclaw/pull/65134 is plausible but still needs repair and maintainer policy review.
