## Summary

Keep open: current main mitigates the default reset hot path by making LLM slugs opt-in and backgrounded, but opt-in descriptive slugs still use the default configured model through the embedded runner, so the requested lightweight/dedicated model routing remains unresolved.

## What This Changes

The issue requests routing LLM slug generation to a lightweight or dedicated model, or otherwise avoiding the default primary model, so reset-time slug work does not congest normal agent lanes.

## Best Possible Solution

Land or replace the focused slug-generator implementation so descriptive slugs avoid the embedded-runner/default-primary-model path, preserve timestamp defaults and timeout/fallback semantics, and document the chosen model-routing policy.

## Reproduction Assessment

Yes for source-level behavior: with `llmSlug` enabled, current main resolves the default model and runs `runEmbeddedPiAgent` on `temp:slug-generator`. I did not live-reproduce a multi-agent queue buildup.

## Solution Assessment

No. Current main is only a partial mitigation: default timestamp slugs avoid the hot path, but opt-in descriptive slugs still lack lightweight or dedicated model routing.
