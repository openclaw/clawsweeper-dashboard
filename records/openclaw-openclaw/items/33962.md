## Summary

Current main still routes session-memory slug generation through the embedded-agent/default-model path, and the related implementation candidate remains open rather than merged.

## What This Changes

Request to use a lightweight slug-generation model or fast fallback so `/new` and `/reset` session-memory slugs do not congest agent lanes.

## Best Possible Solution

Replace slug naming with a lightweight direct completion or bounded heuristic path, document the selected model-routing policy, and add regression coverage for model selection, timeout, and fallback behavior.

## Reproduction Assessment

Yes for the source-level behavior: `/new` and `/reset` session-memory still call `generateSlugViaLLM`, which uses `runEmbeddedPiAgent` with the default agent model and `temp:slug-generator` session key. I did not establish a live multi-agent congestion reproduction.

## Solution Assessment

Unclear. #65134's direct-completion approach is a plausible narrow fix, but the issue's preferred dedicated slug model config still requires maintainer product judgment.
