## Summary

Keep open: current main and v2026.5.12 still route opt-in LLM session-memory slug generation through the resolved default agent model and `runEmbeddedPiAgent` lane. Timestamp defaults and timeout configurability reduce blast radius, but the lightweight-model/direct-completion policy is still unresolved and the related open PR is not merge-ready.

## What This Changes

The issue requests routing LLM session slug generation to a lightweight or dedicated model, or a fast fallback, instead of using the agent primary model and congesting the embedded session lane.

## Best Possible Solution

Land a focused slug-generation policy that avoids embedded-runner/default-primary routing for opt-in descriptive slugs, preserves timestamp defaults and timeout/fallback behavior, and updates focused tests plus hook docs.

## Reproduction Assessment

Yes, at source level: enable `hooks.internal.entries.session-memory.llmSlug` and current main calls `generateSlugViaLLM`, resolves the default agent model, and runs the embedded agent on `temp:slug-generator`. I did not live-reproduce the multi-agent queue buildup.

## Solution Assessment

Unclear. Direct simple completion and heartbeat-model reuse are plausible, but maintainers still need to choose the model-routing policy before adding dedicated slug config or reusing an existing lightweight-model knob.
