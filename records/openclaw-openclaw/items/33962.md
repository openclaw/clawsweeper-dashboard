## Summary

Keep open: current main mitigates the original blast radius by making LLM slugs opt-in and backgrounded, but the enabled path still resolves the default agent model and uses the embedded runner. The lightweight slug-routing policy remains unresolved, and the linked open PR is the active implementation candidate rather than proof that this issue is fixed.

## What This Changes

The issue asks for LLM session-memory slug generation to use a lightweight/dedicated model instead of the agent primary model so reset-time naming does not congest model lanes.

## Best Possible Solution

Settle and land one focused slug-generation policy that avoids embedded-runner/default-primary routing for optional descriptive slugs while preserving timestamp defaults, background hook behavior, timeouts, fallback behavior, and auth/provider contracts.

## Reproduction Assessment

Yes, at source level: enable `hooks.internal.entries.session-memory.llmSlug`, and session memory reaches `generateSlugViaLLM`, which resolves the default model and runs `runEmbeddedPiAgent` on `temp:slug-generator`. I did not live-reproduce the reported multi-agent queue buildup.

## Solution Assessment

Unclear. Direct simple completion, heartbeat-model reuse, a dedicated slug model key, and shorter fallback timing are all plausible, but maintainers still need to choose the permanent model-routing contract.
