## Summary

Keep open: current main and v2026.5.12 still route opt-in session-memory LLM slug generation through the resolved default agent model and the embedded-runner lane. The related open PR targets this surface, but it is not merged and the lightweight model-routing policy remains unresolved.

## What This Changes

Request lightweight or dedicated model routing, plus faster fallback behavior, for opt-in session-memory LLM slug generation so resets do not congest primary agent lanes.

## Best Possible Solution

Land a focused slug-generation policy that avoids embedded-runner/default-primary routing for opt-in descriptive slugs while preserving timestamp defaults, timeout/fallback behavior, runtime auth contracts, and focused hook tests/docs.

## Reproduction Assessment

Yes, at source level: enable `hooks.internal.entries.session-memory.llmSlug`, and session memory calls `generateSlugViaLLM`, which resolves the default model and runs `runEmbeddedPiAgent` on `temp:slug-generator`. I did not live-reproduce the multi-agent queue buildup.

## Solution Assessment

Unclear. Direct simple completion and lightweight-model reuse are plausible, but maintainers still need to choose the model-routing policy before adding a dedicated slug config or reusing an existing model knob.
