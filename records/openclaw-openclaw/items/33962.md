## Summary

Current main still routes session-memory slug generation through the embedded-agent/default-model path, and the related implementation candidate is still open rather than merged.

## What This Changes

Requests moving LLM-based session-memory slug generation away from the agent's primary model, preferably via a lightweight slug model or fast fallback.

## Best Possible Solution

Replace slug naming with a lightweight direct completion or bounded heuristic path, document the selected model-routing policy, and add regression coverage for model selection, timeout, and fallback behavior.

## Reproduction Assessment

Yes for the source-level behavior: the `/new` and `/reset` session-memory path still calls `generateSlugViaLLM`, which uses `runEmbeddedPiAgent` with the default/effective agent model and `temp:slug-generator` session key. I did not establish a live multi-agent congestion reproduction.

## Solution Assessment

Unclear: #65134's `completeSimple` plus heartbeat-model preference is a plausible narrow fix, but maintainers still need to decide whether a dedicated slug model config is required for the requested product shape.
