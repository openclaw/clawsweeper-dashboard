## Summary

Current main still uses the full embedded-agent/default-model path for slug naming, and the direct fix path is an open unmerged PR, so this should stay open for maintainer review.

## What This Changes

The request is to route LLM slug generation away from the agent primary model, preferably to a lightweight dedicated model or fast fallback, so session naming does not congest model lanes.

## Best Possible Solution

Land a narrow core change that avoids full embedded-agent slug runs and primary-model routing while preserving the existing timestamp fallback and adding regression coverage for model selection, timeout, and failure behavior.

## Reproduction Assessment

Yes for the source-level behavior: the `/new` or `/reset` session-memory path still calls `generateSlugViaLLM`, which uses `runEmbeddedPiAgent` with the default agent model and `temp:slug-generator` session key. I did not run a live multi-agent congestion reproduction.

## Solution Assessment

Unclear between a dedicated slug model config and reuse of the heartbeat model, but current main is not the best end state because it still pays the full embedded-agent/default-model path for a small slug task.
