## Label Justifications

- `P2`: This is a normal-priority TUI local-mode command routing bug with limited blast radius and a clear source-level fix boundary.
- `impact:message-loss`: Advertised slash commands can be misdelivered as ordinary model prompts instead of command/control responses.
- `impact:session-state`: The fallthrough creates local embedded user/model turns from control commands, affecting transcript and run state.

## Summary

Keep open: current `main` and `v2026.6.10` still advertise shared `/status` and `/compact` in the TUI while unhandled commands fall through to `sendMessage(raw)`. The open linked PR is the right work path, but it is unmerged and currently has review blockers, so this issue should remain open until that PR lands or is replaced.

## What This Changes

The issue requests that advertised TUI local-mode slash commands such as `/status` and `/compact` stop being delivered to the local embedded model as ordinary user prompts.

## Best Possible Solution

Land a local-mode-only command handling path, or a shared command execution adapter, that keeps `/status` and `/compact` out of local model prompts while preserving gateway-connected command forwarding.

## Reproduction Assessment

Yes. This is source-reproducible: slash input routes to `handleCommand`, TUI autocomplete includes shared `/status` and `/compact`, and current `handleCommand` lacks those cases before the default `sendMessage(raw)` branch.

## Solution Assessment

Yes for the requested end state, but the implementation must preserve gateway-connected TUI forwarding. The current open candidate should be reviewed or corrected rather than queueing a separate fix PR.
