## Summary

Keep open. Current main still has the reported TUI local-mode command-routing bug: `/status` and `/compact` are advertised through shared slash-command data, but the TUI local command switch has no local handler for either command and falls through to `sendMessage(raw)`, which sends the slash text into the embedded agent path.

## What This Changes

The issue asks for TUI local embedded mode to stop sending advertised `/status` and `/compact` slash commands to the model when no local handler exists.

## Best Possible Solution

Add an explicit local embedded TUI command boundary for advertised shared commands. `/status` and `/compact` should either execute through reusable shared command logic or render clear local-mode responses, with regression coverage proving they do not enter `sendChat()` or `agentCommandFromIngress()` as ordinary user prompts.

## Reproduction Assessment

Yes. The current-main static path is high confidence: slash input enters `handleCommand()`, autocomplete advertises `/status` and `/compact`, the TUI switch lacks local cases for both commands, and the default branch forwards raw slash text into the embedded chat/model path.

## Solution Assessment

Yes. The narrow maintainable fix is to handle or explicitly reject known advertised slash commands at the local TUI boundary while preserving gateway-connected behavior and unknown-command forwarding.
