## Summary

Keep open: current main still advertises `/status` and `/compact` through TUI slash completion, but the TUI local command switch has no cases for them and the default branch still sends the raw slash text into the embedded agent path.

## What This Changes

The issue asks TUI local mode to stop sending advertised slash commands like `/status` and `/compact` to the embedded model as ordinary user text.

## Best Possible Solution

Add a local embedded TUI command boundary so advertised shared commands either execute reusable shared behavior or render an explicit local-mode response before any embedded send occurs.

## Reproduction Assessment

Yes, at source level. Slash input reaches `handleCommand()`, completion advertises shared `/status` and `/compact`, the local switch lacks those cases, and local `sendChat()` enters `agentCommandFromIngress()` with the raw slash text.

## Solution Assessment

Yes. The narrow maintainable fix is to handle or explicitly reject known advertised shared commands at the local TUI boundary while preserving gateway-mode recovery for unknown slash commands.
