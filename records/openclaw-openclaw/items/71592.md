## Summary

Keep open. Current main still advertises shared `/status` and `/compact` commands in the TUI slash command list, but the local TUI command switch has no `status` or `compact` cases and falls through to `sendChat()`, which starts the embedded agent path in local mode.

## What This Changes

The issue requests that advertised TUI local-mode slash commands such as `/status` and `/compact` be handled locally, routed through shared command execution, or explicitly rejected instead of being sent to the model as user text.

## Best Possible Solution

Add an explicit local embedded TUI command boundary for advertised shared commands. `/status` and `/compact` should either execute through reusable shared command logic or render a clear local-mode response, with regression coverage proving they do not call `sendChat()` / `agentCommandFromIngress()` as ordinary prompts.

## Reproduction Assessment

Yes. The current-main static path is high confidence: slash input reaches `handleCommand()`, `/status` and `/compact` are advertised from the shared registry, the TUI switch lacks both cases, and the default path forwards the raw slash text into the local embedded backend.

## Solution Assessment

Yes. The requested direction is the narrow maintainable fix: handle or explicitly reject known advertised slash commands at the local TUI boundary without changing unrelated Control UI/WebChat behavior or broad command registry semantics.
