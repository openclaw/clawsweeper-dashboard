## Summary

Keep open: current main still advertises shared `/status` and `/compact` commands in TUI slash completion, while local embedded TUI command handling still falls through to `sendMessage(raw)` and then into the embedded agent path. The related open PRs cover `/model` parsing and autocomplete-only behavior, not this local embedded command execution bug.

## What This Changes

The issue asks for advertised `/status` and `/compact` commands in `openclaw tui --local` to be handled or explicitly rejected instead of being sent to the model as user text.

## Best Possible Solution

Add a local embedded TUI command boundary so advertised shared commands either execute reusable shared behavior or render an explicit local-mode response before any embedded send occurs.

## Reproduction Assessment

Yes, at source level. Slash input reaches `handleCommand()`, autocomplete advertises the shared commands, the local switch lacks `status` and `compact`, and local `sendChat()` enters `agentCommandFromIngress()` with the raw slash text.

## Solution Assessment

Yes. The narrow maintainable fix is to handle or explicitly reject known advertised shared commands at the local TUI boundary while preserving gateway-mode recovery for unknown slash commands.
