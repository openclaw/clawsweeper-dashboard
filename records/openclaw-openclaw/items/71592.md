## Summary

Keep open: current main still advertises `/status` and `/compact` in TUI slash completion/help, but the local TUI command switch has no handlers for either command and falls through to `sendMessage(raw)`, which submits the text into the local embedded agent path.

## What This Changes

TUI local mode should stop advertised `/status` and `/compact` slash commands from being submitted as ordinary model prompts.

## Best Possible Solution

Add a local embedded TUI slash-command boundary that executes or explicitly rejects advertised shared commands without forwarding them to the model path, while preserving gateway-mode recovery for unknown slash commands.

## Reproduction Assessment

Yes. Static review of current main gives a high-confidence reproduction path: slash input reaches `handleCommand()`, autocomplete advertises `/status` and `/compact`, the switch lacks both cases, and local embedded `sendChat()` enters `agentCommandFromIngress()` with the raw slash text.

## Solution Assessment

Yes. The narrow maintainable fix is to handle `/status` and `/compact` at the local TUI boundary or route them through reusable shared command logic, rather than removing useful autocomplete or changing the gateway command pipeline.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/71592. In current main, TUI slash input routes through `src/tui/tui-submit.ts` to `createTuiCommandHandlers().handleCommand()`. `src/tui/commands.ts` advertises shared registry aliases including `/status` and `/compact`, but `src/tui/tui-command-handlers.ts` lacks `status` and `compact` cases and defaults to `sendMessage(raw)`. In local embedded mode, `src/tui/embedded-backend.ts` then calls `agentCommandFromIngress()` with the raw slash text, starting an embedded agent/model run. Add a local embedded TUI boundary so `/status` and `/compact` do not enter the model path: reuse shared command/status/compaction logic where feasible, or render explicit local-mode responses if a command is not supportable. Preserve gateway-connected fallback behavior for genuinely unknown slash commands and avoid removing autocomplete unless a command is intentionally unsupported and tested. Add regression tests proving local `/status` and `/compact` do not call `sendChat` / `agentCommandFromIngress`, and keep existing gateway forwarding expectations for unknown commands. Update `CHANGELOG.md` because this is user-facing behavior.
