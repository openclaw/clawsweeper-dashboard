## Summary

Keep open: current main still advertises `/status` and `/compact` in TUI slash completion, but the TUI command switch has no local handlers for those names and the local embedded backend receives the raw slash text as a model run.

## What This Changes

TUI local embedded mode should not send advertised slash commands such as `/status` and `/compact` to the local model as ordinary prompts.

## Best Possible Solution

Add a local embedded TUI command boundary so advertised shared commands either execute reusable shared behavior or render an explicit local-mode response before any embedded send occurs.

## Reproduction Assessment

Yes. Source review gives a high-confidence path: slash input reaches `handleCommand()`, autocomplete advertises `/status` and `/compact`, the local switch lacks both cases, and local `sendChat()` enters `agentCommandFromIngress()` with the raw text.

## Solution Assessment

Yes. The narrow maintainable fix is to handle or explicitly reject known advertised shared commands at the local TUI boundary while preserving gateway-mode recovery for truly unknown slash commands.
