## Summary

Keep open. Current main still matches the reported local TUI bug: `/status` and `/compact` are advertised through the TUI slash-command list, but the local TUI command switch does not handle them and falls through to the embedded chat/model path.

## What This Changes

The issue asks for advertised TUI local-mode slash commands such as `/status` and `/compact` to be handled locally, routed through shared command execution, or explicitly rejected instead of being sent as model text.

## Best Possible Solution

Add a local embedded TUI command boundary for advertised shared commands. `/status` and `/compact` should either execute through reusable shared command logic or render clear local-mode responses, with regression tests proving they do not enter `sendChat()` or `agentCommandFromIngress()` as ordinary prompts.

## Reproduction Assessment

Yes. The current-main static path is high confidence: slash input enters `handleCommand()`, autocomplete advertises `/status` and `/compact`, the local switch lacks those cases, and the default branch forwards the raw slash text into the embedded chat/model path.

## Solution Assessment

Yes. Handling or explicitly rejecting known advertised commands at the TUI local boundary is the narrowest maintainable fix; it preserves gateway-connected behavior while preventing known commands from becoming model prompts.

Fix the TUI local embedded slash-command fallthrough reported in https://github.com/openclaw/openclaw/issues/71592. In current main, TUI slash input goes through `src/tui/tui-command-handlers.ts`, autocomplete advertises shared commands from `src/tui/commands.ts` and `src/auto-reply/commands-registry.shared.ts`, but `/status` and `/compact` are not handled locally and the default branch sends them to `sendMessage(raw)`, which reaches `EmbeddedTuiBackend.sendChat()` and `agentCommandFromIngress()`. Add the narrow local TUI command boundary so advertised known commands do not become model prompts in `openclaw tui --local`: either implement `/status` and `/compact` through reusable shared command logic or render explicit local-mode responses where shared execution is not safe. Add regression tests in `src/tui/tui-command-handlers.test.ts` and/or `src/tui/commands.test.ts` proving `/status` and `/compact` in local TUI do not call `sendChat()` and that advertised commands have a local handler, shared adapter route, or explicit unsupported response. Add a user-facing `CHANGELOG.md` fix entry crediting `Thanks @goslingmanagment`. Do not broaden plugin command semantics or change unrelated TUI commands.
