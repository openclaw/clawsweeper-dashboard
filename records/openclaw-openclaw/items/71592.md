## Label Justifications

- `P2`: This is a normal-priority TUI command routing bug with a clear source path and limited blast radius.
- `impact:message-loss`: Command-looking input is misrouted into the local model message path instead of being handled as a command.
- `impact:session-state`: The fallthrough can create unintended user turns and embedded local runs in the session transcript.

## Summary

Keep open: current main still advertises `/status` and `/compact` in the TUI command surface, but the local command switch has no cases for them and its default branch sends the raw slash text into the embedded agent path.

## What This Changes

TUI local embedded mode should not send advertised `/status` and `/compact` slash commands to the local model path as ordinary user messages.

## Best Possible Solution

Add a local embedded TUI command boundary so advertised shared commands either execute reusable shared behavior or render an explicit local-mode response before any embedded send occurs.

## Reproduction Assessment

Yes, at source level. Slash input reaches `handleCommand()`, completion advertises shared `/status` and `/compact`, the local switch lacks those cases, and local `sendChat()` enters `agentCommandFromIngress()` with the raw slash text.

## Solution Assessment

Yes. The narrow maintainable fix is to handle or explicitly reject known advertised shared commands at the local TUI boundary while preserving gateway-mode recovery for truly unknown slash commands.
