## Summary

Keep open. Current main still advertises `/status` and `/compact` to the TUI, but the local command switch has no `status` or `compact` cases and falls through to `sendMessage(raw)`. In `--local`, that backend is `EmbeddedTuiBackend`, so the slash text is routed into the embedded agent path instead of shared command handling; related PR #51270 is adjacent `/model` work and does not cover this bug.

## What This Changes

Ensure TUI local embedded mode does not send advertised `/status` and `/compact` slash commands to the model as ordinary user prompts.

## Best Possible Solution

Add a local embedded TUI command boundary for advertised shared commands. `/status` and `/compact` should either execute through reusable shared command logic or render a clear local-mode response, with tests proving they do not enter `sendChat()` / `agentCommandFromIngress()` as ordinary user prompts while Gateway-connected forwarding still works.

## Reproduction Assessment

Yes. Static current-main inspection gives a high-confidence reproduction path: slash input reaches `handleCommand()`, autocomplete/help advertise `/status` and `/compact`, the switch lacks both cases, and the local backend sends the raw slash text into the embedded agent path. I did not run a live TUI session because this review is read-only.

## Solution Assessment

Yes. The narrow maintainable solution is to handle or explicitly reject advertised shared commands at the local TUI boundary, preferably through reusable command logic where it does not pull Gateway-only coupling into local mode.
