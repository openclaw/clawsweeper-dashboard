## Summary

Keep this issue open: current main still has the reported local TUI slash-command fallthrough, and the live linked PR that closes it is open and mergeable but not merged.

## What This Changes

The issue asks OpenClaw to stop advertised local TUI slash commands such as `/status` and `/compact` from being sent to the embedded model path as ordinary user text.

## Best Possible Solution

Review and land or reject the linked TUI-local command-routing PR, then close this issue after the fix is merged or a maintainer chooses a different canonical implementation.

## Reproduction Assessment

Yes. Current main source shows slash input goes to `handleCommand()`, local command discovery includes shared `/status` and `/compact`, and unmatched commands fall through to `sendMessage(raw)` and then `agentCommandFromIngress()`.

## Solution Assessment

Yes, the linked PR targets the right boundary by deciding local, unsupported, and forwarded slash inputs before embedded runs start. The issue should stay open while that PR is reviewed because it uses closing syntax and is not merged.
