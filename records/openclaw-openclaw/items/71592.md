## Summary

Keep open: current main and the latest release still have the local TUI slash-command fallthrough, and the linked closing PR is open but unmerged, so this issue should remain the canonical tracker until that fix path lands or is closed.

## What This Changes

The issue reports that TUI local mode advertises `/status` and `/compact` but sends them as ordinary model prompts instead of handling or rejecting them locally.

## Best Possible Solution

Land or revise the linked fix PR so local TUI help/autocomplete and command dispatch agree: every advertised local slash command should be handled, routed through a safe shared adapter, or explicitly rejected before it can reach the embedded model path.

## Reproduction Assessment

Yes from source: slash input enters `handleCommand`, local help/autocomplete still expose shared `/status` and `/compact`, and unhandled command names reach `sendMessage(raw)` and then `agentCommandFromIngress`. I did not run the PTY repro in this read-only pass.

## Solution Assessment
