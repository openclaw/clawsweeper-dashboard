## Summary

Keep this issue open. Current `main` still advertises `/status` and `/compact` in the TUI command surface, but local embedded TUI has no local `status` or `compact` cases, so those slash commands still fall through to the embedded chat path as ordinary model input.

## What This Changes

The issue asks for local embedded TUI to stop forwarding advertised `/status` and `/compact` slash commands as ordinary model prompts and instead handle them or show an explicit local-mode response.

## Best Possible Solution

Local embedded TUI should keep autocomplete and docs honest by ensuring every advertised shared slash command either executes through a reusable local/shared command path or returns an explicit local-mode unsupported response. `/status` and `/compact` specifically should not be added to the transcript or sent into `agentCommandFromIngress()` as ordinary prompts.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current `main`: slash input enters `handleCommand()`, `/status` and `/compact` are advertised by shared-command autocomplete, the local switch lacks cases for them, and the default path forwards raw slash text to `sendChat()` and then `agentCommandFromIngress()` in local mode.

## Solution Assessment

Yes. The narrow maintainable direction is to intercept advertised commands at the local TUI command boundary, either by implementing local handlers or returning explicit unsupported output, rather than globally removing shared commands or continuing to send known slash commands to the model.
