## Summary

Keep open. Current main still has the reported local TUI mismatch: `/status` and `/compact` are advertised by shared slash-command autocomplete/docs, but the TUI command switch has no local cases for them and the default path sends the raw slash text into the embedded model run path.

## What This Changes

TUI local mode should handle or explicitly reject advertised slash commands such as `/status` and `/compact` instead of sending them as ordinary model input.

## Best Possible Solution

Add a local embedded TUI command boundary for advertised shared commands. `/status` and `/compact` should execute through reusable shared command logic when practical, or render clear local-mode responses, with regression coverage proving they do not enter `sendChat()` or `agentCommandFromIngress()` as ordinary prompts.

## Reproduction Assessment

Yes. The current-main static path is high confidence: slash input enters `handleCommand()`, autocomplete advertises `/status` and `/compact`, the switch lacks local cases, and the default branch forwards the raw slash text into the embedded chat/model path.

## Solution Assessment

Yes. The requested direction is the narrow maintainable fix: handle or explicitly reject known advertised slash commands at the local TUI boundary, while preserving gateway-connected behavior and unknown-command forwarding.
