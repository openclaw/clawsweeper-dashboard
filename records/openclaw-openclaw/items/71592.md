## Repair Work Prompt

Fix the local TUI slash-command fallthrough reported in https://github.com/openclaw/openclaw/issues/71592. In `openclaw tui --local`, advertised known shared commands such as `/status` and `/compact` must not reach `sendMessage(raw)` / `EmbeddedTuiBackend.sendChat` as ordinary model prompts. Keep the fix within the TUI command boundary: either handle known unsupported shared commands with an explicit local system response or filter them from local autocomplete/help while preserving gateway-mode forwarding and existing local commands like `/goal`, `/stop`, `/context`, `/help`, `/auth`, `/new`, and any intentional unknown-slash prompt escape hatch. Add focused tests proving local `/status` and `/compact` do not call `sendChat`, and update command-list/help tests if the advertised local command set changes. Do not add config flags, provider changes, broad registry rewrites, or changelog edits.
