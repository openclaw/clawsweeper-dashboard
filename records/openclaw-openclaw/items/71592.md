## Summary

Keep open: current main still advertises `/status` and `/compact` through the TUI command list, but the local TUI command switch has no handlers for either name and forwards them into the embedded model run path.

## What This Changes

TUI local embedded mode should stop sending advertised `/status` and `/compact` slash commands to the model as ordinary user prompts.

## Best Possible Solution

Add a local embedded TUI command boundary for advertised shared commands, with `/status` and `/compact` either executing reusable shared behavior or rendering an explicit local-mode response before any embedded agent send occurs.

## Reproduction Assessment

Yes. Static current-main review gives a high-confidence path: slash input reaches `handleCommand()`, autocomplete advertises `/status` and `/compact`, the switch lacks both handlers, and local embedded `sendChat()` enters `agentCommandFromIngress()` with the raw text.

## Solution Assessment

Yes. The narrow maintainable fix is to handle or explicitly reject the known advertised commands at the local TUI boundary while leaving gateway-mode recovery for truly unknown slash commands intact.
