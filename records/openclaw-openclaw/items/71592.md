## Summary

Keep open: current main still advertises `/status` and `/compact` in TUI completion/help, but the TUI command switch has no local cases for them and the fallback still sends the raw slash text into the embedded local agent path.

## What This Changes

The issue asks for advertised TUI local slash commands such as `/status` and `/compact` to be handled or explicitly rejected instead of being sent as ordinary model text.

## Best Possible Solution

Add a local embedded TUI command boundary so advertised shared commands either execute shared behavior or render an explicit local-mode response before any embedded send occurs.

## Reproduction Assessment

Yes, at source level. Slash input reaches `handleCommand()`, completion advertises shared `/status` and `/compact`, the local switch lacks those cases, and local `sendChat()` enters `agentCommandFromIngress()` with the raw slash text.

## Solution Assessment

Yes. The narrow maintainable fix is to handle or explicitly reject known advertised shared commands at the local TUI boundary while preserving gateway-mode recovery for unknown slash commands.
