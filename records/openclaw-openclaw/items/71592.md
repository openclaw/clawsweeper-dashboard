## Summary

Keep open: current main still advertises `/status` and `/compact` in TUI completion/help, but `handleCommand()` has no local cases for either command and its default path sends the raw slash text as a user message. In `--local`, that send path enters `EmbeddedTuiBackend.sendChat()` and then `agentCommandFromIngress()`, so the reported local embedded fallthrough remains source-provable.

## What This Changes

The issue asks for advertised TUI local slash commands such as `/status` and `/compact` to stop being sent to the embedded model path as ordinary user text.

## Best Possible Solution

Add a local embedded TUI command boundary so advertised shared commands either execute reusable shared/local behavior or render an explicit local-mode response before any embedded send occurs.

## Reproduction Assessment

Yes, at source level. Slash input reaches `handleCommand()`, autocomplete advertises the shared commands, the local switch lacks `status` and `compact`, and local `sendChat()` enters `agentCommandFromIngress()` with the raw slash text.

## Solution Assessment

Yes. The narrow maintainable fix is to handle or explicitly reject known advertised shared commands at the local TUI boundary while preserving gateway-mode recovery for unknown slash commands.
