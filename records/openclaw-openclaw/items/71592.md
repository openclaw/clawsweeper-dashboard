## Summary

Keep open. Current main still advertises `/status` and `/compact` in the TUI, but the local embedded command switch only handles `gateway-status` and falls through to `sendMessage(raw)`, which calls the embedded backend and starts an agent run instead of executing shared slash-command handling.

## What This Changes

The issue reports that local embedded TUI mode advertises `/status` and `/compact` but sends those slash commands as ordinary model input instead of handling or rejecting them locally.

## Best Possible Solution

Add a local embedded TUI command boundary for advertised shared commands. `/status` and `/compact` should either execute through reusable shared command logic or render clear local-mode responses, with tests proving they do not enter `sendChat()` or `agentCommandFromIngress()` as ordinary user prompts.

## Reproduction Assessment

Yes. Static review of current main gives a high-confidence reproduction path: slash input reaches `handleCommand()`, autocomplete advertises `/status` and `/compact`, the TUI switch lacks both cases, and the default branch forwards raw slash text into the embedded backend. I did not run a live TUI session because this review was required to keep the checkout read-only.

## Solution Assessment

Yes. The narrow maintainable fix is to handle or explicitly reject known advertised slash commands at the local TUI boundary while preserving Gateway-connected forwarding for commands that the Gateway can recover.

## Repair Work Prompt

Fix the local embedded TUI slash-command fallthrough reported in https://github.com/openclaw/openclaw/issues/71592. In `openclaw tui --local`, `/status` and `/compact` are advertised by TUI autocomplete but fall through to `sendMessage(raw)`, causing `EmbeddedTuiBackend.sendChat()` and `agentCommandFromIngress()` to treat them as ordinary model prompts. Add the smallest maintainable local command boundary so advertised `/status` and `/compact` either execute through reusable shared command logic or render explicit local-mode responses, and prove they do not call `sendChat()` / `agentCommandFromIngress()` as user text. Preserve Gateway-connected TUI forwarding for shared commands like `/context` and unknown/plugin commands; do not remove shared command autocomplete broadly. Add focused tests in the TUI command/backend tests, and add a user-facing `CHANGELOG.md` fix entry credited to `@goslingmanagment` unless the final change is test-only/internal.
