## Summary

Keep open: current main still source-reproduces the local embedded TUI slash-command fallthrough, and the related open PRs cover `/model` parsing or autocomplete inventory rather than this `/status` and `/compact` execution path.

## What This Changes

The issue requests that advertised TUI local-mode `/status` and `/compact` commands stop being sent to the embedded model path as ordinary user text.

## Best Possible Solution

Add a local embedded TUI command boundary so advertised shared commands either execute reusable shared behavior or render an explicit local-mode response before `sendChat()` is reached.

## Reproduction Assessment

Yes, at source level: slash input reaches `handleCommand()`, autocomplete advertises shared `/status` and `/compact`, the local switch lacks those cases, and local `sendChat()` enters `agentCommandFromIngress()` with the raw slash text. I did not run the interactive TUI because this review was read-only.

## Solution Assessment

Yes. The narrow maintainable fix is to handle or explicitly reject known advertised shared commands at the local TUI boundary while preserving gateway-mode recovery for commands that should still flow through shared command handling.

## Repair Work Prompt

Fix the TUI local embedded slash-command fallthrough reported at https://github.com/openclaw/openclaw/issues/71592. Current `main` advertises shared `/status` and `/compact` through `getSlashCommands()`, but `createCommandHandlers().handleCommand()` has no `status` or `compact` branch, so in `openclaw tui --local` both commands reach `sendMessage(raw)` and then `EmbeddedTuiBackend.sendChat()` / `agentCommandFromIngress()`. Add a narrow local-mode boundary so advertised shared commands either execute reusable shared behavior or render an explicit local-mode unsupported response before any embedded send occurs. Preserve gateway-connected recovery for commands that intentionally flow through shared command handling, and do not broaden this into a full command-system refactor. Add focused regression tests proving local `/status` and `/compact` do not call `sendChat` / `agentCommandFromIngress`, and update autocomplete/handler coverage if needed. Add a user-facing changelog entry if behavior changes. Likely related PRs that are not sufficient fixes: https://github.com/openclaw/openclaw/pull/51270 and https://github.com/openclaw/openclaw/pull/61966.
