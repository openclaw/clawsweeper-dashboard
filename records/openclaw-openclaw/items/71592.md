## Summary

Keep open: current main still advertises `/status` and `/compact` in TUI slash completion/help, but local embedded TUI command handling has no cases for them and falls through to the local model-run path. No closing PR or canonical superseding fix covers this exact local embedded behavior.

## What This Changes

TUI local embedded mode should stop treating advertised `/status` and `/compact` slash commands as ordinary model prompts.

## Best Possible Solution

Add a local embedded TUI command boundary so advertised shared commands either execute reusable shared behavior or render an explicit local-mode response before any embedded send occurs.

## Reproduction Assessment

Yes, at source level. Slash input reaches `handleCommand()`, autocomplete advertises `/status` and `/compact`, the local switch lacks both cases, and local `sendChat()` enters `agentCommandFromIngress()` with the raw slash text.

## Solution Assessment

Yes. The narrow maintainable fix is to handle or explicitly reject known advertised shared commands at the local TUI boundary while preserving gateway-mode recovery for truly unknown slash commands.
