## Label Justifications

- `P2`: This is a normal-priority TUI command routing bug with a clear source-level reproduction and limited blast radius.
- `impact:message-loss`: Command-looking input is misrouted into the local model message path instead of being handled as a command.
- `impact:session-state`: The fallthrough can create unintended user turns and local embedded runs in the session transcript.

## Summary

Keep open: current main still advertises shared `/status` and `/compact` in TUI slash completion, but the local TUI command switch has no cases for them and falls through into the embedded agent send path.

## What This Changes

The issue asks for TUI local embedded mode to stop sending advertised `/status` and `/compact` commands to the model as ordinary user messages.

## Best Possible Solution

Add a local embedded TUI command boundary so advertised shared commands either execute shared behavior or render an explicit local-mode response before any embedded send occurs.

## Reproduction Assessment

Yes, at source level. Slash input reaches handleCommand(), completion advertises shared `/status` and `/compact`, the local switch lacks those cases, and local sendChat() enters agentCommandFromIngress() with the raw slash text.

## Solution Assessment

Yes. The narrow maintainable fix is to handle or explicitly reject known advertised shared commands at the local TUI boundary while preserving gateway-mode recovery for unknown slash commands.
