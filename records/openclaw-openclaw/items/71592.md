## Summary

Keep open: current main still advertises `/status` and `/compact` in the TUI, but the local embedded command switch lacks handlers for both and forwards raw slash text into the embedded agent path.

## What This Changes

The issue asks for advertised TUI local-mode slash commands such as `/status` and `/compact` to be handled or explicitly rejected instead of sent as model prompts.

## Best Possible Solution

Add a local embedded TUI slash-command boundary for advertised shared commands, with `/status` and `/compact` executing through reusable shared logic where feasible or rendering clear local-mode responses, plus tests proving they do not enter the embedded agent run path as user prompts.

## Reproduction Assessment

Yes. Static review of current main gives a high-confidence reproduction path: slash input reaches `handleCommand()`, autocomplete advertises `/status` and `/compact`, the switch lacks both cases, and local embedded `sendChat()` calls `agentCommandFromIngress()` with the raw slash text.

## Solution Assessment

Yes. The narrow maintainable direction is to handle or explicitly reject known advertised slash commands at the local TUI boundary without removing the Gateway recovery path for genuinely unknown commands.
