## Summary

Keep open: this is a fresh onboarding regression with active linked fix work, and current main still launches the recommended hatch TUI without a bounded timeout while the streaming watchdog only resets visible state.

## What This Changes

The issue reports first-run Terminal hatching freezing after the automatic wake message on Ubuntu 26 with OpenAI Cloud in OpenClaw 2026.4.29.

## Best Possible Solution

Review and land the narrow hatch-timeout propagation fix, likely PR #76241 or an equivalent replacement, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes for the actionable timeout contract: the issue gives concrete install and TUI output, and PR #76241 reports a deterministic loopback provider-stall reproduction of wake message, watchdog, idle state, and an open provider request. Natural live OpenAI stalling was not reproduced in the provided PR evidence.

## Solution Assessment

Yes for the direction, not yet for current main: a bounded setup hatch timeout with propagation tests is the narrow maintainable fix. PR #76192's broader watchdog/OAuth changes should be reviewed against PR #76241 to avoid overlapping behavior.
