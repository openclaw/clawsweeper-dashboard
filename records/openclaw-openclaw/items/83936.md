## Summary

Close as duplicate: the same public `after_compaction` summary-payload request is already tracked by the older open canonical issue at https://github.com/openclaw/openclaw/issues/61447. Current main still lacks the field, so this is not implemented; the newer opt-in/privacy proposal here is useful design input for that canonical thread.

## What This Changes

Expose compacted summary content through the public `after_compaction` plugin hook payload so plugins do not scrape internal compaction files.

## Best Possible Solution

Track the implementation and product/API decision on https://github.com/openclaw/openclaw/issues/61447, carrying over this thread's opt-in/default-off and `undefined`-when-unavailable proposal as design input.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high-confidence current-state proof that `after_compaction` lacks summary content while compaction results carry a summary internally.

## Solution Assessment

Yes, closing this thread as a duplicate is the best cleanup path because the same remaining SDK work is already tracked by https://github.com/openclaw/openclaw/issues/61447. The implementation should be designed there as an additive, opt-in public hook contract rather than split across two issues.
