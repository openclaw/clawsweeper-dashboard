## Summary

Keep open: current main and the latest release still expose only compaction counts/session metadata to the public `after_compaction` plugin hook, while the compacted summary string exists internally. A prior exact PR attempted this but was closed as dirty, so the remaining work needs a maintainer API/privacy decision rather than cleanup closure.

## What This Changes

Expose compacted summary content, or a safer structured equivalent, in the public `after_compaction` plugin hook payload.

## Best Possible Solution

Define an additive, documented compaction hook contract that exposes safe summary content or structured distilled items after successful compaction, with focused tests for direct, engine-owned, overflow, and embedded forwarding paths.

## Reproduction Assessment

Yes, source-level: current main builds and persists a compaction summary internally but the public `after_compaction` hook payload exposes only counts and `sessionFile`. No live plugin reproduction was run in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public payload shape. An optional additive field is plausible, but raw full-summary exposure should be an explicit API/privacy decision rather than an automatic fix.
