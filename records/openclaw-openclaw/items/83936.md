## Summary

Keep open: current main still does not expose compacted summary content through the public `after_compaction` plugin hook payload, and the remaining work is an additive plugin SDK/API design decision rather than an already-implemented fix or ClawHub-only extension.

## What This Changes

Request to add compacted summary content, structured distilled items, or an opt-in equivalent to the `after_compaction` plugin hook payload.

## Best Possible Solution

Design an additive, documented plugin hook contract that exposes bounded summary text or structured distilled items when available, defines unavailable behavior for engine-owned/Codex compaction, and covers the public SDK, docs, and regression tests together.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high-confidence current-state proof that `after_compaction` lacks summary content while compaction results do carry a summary internally.

## Solution Assessment

Unclear until maintainers choose the SDK shape. Adding an additive payload field is plausible, but the safest design needs privacy bounds and explicit behavior for runtimes where summary metadata is unavailable.
