## Summary

Keep open: current main still exposes only compaction metadata to the public `after_compaction` plugin hook even though the normal compaction path has summary text internally. The remaining work is an additive plugin SDK/API and privacy contract decision, not an implemented fix, duplicate, or ClawHub-only plugin request.

## What This Changes

Request to expose compacted summary text or structured distilled items in the public `after_compaction` plugin hook payload instead of requiring plugins to parse internal files.

## Best Possible Solution

Design an additive, opt-in plugin hook contract that exposes bounded summary text or structured distilled items when available, defines truncation and `undefined` semantics, and updates the SDK types, docs, and compaction-path tests together.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high-confidence current-state proof that public `after_compaction` lacks summary content while normal compaction results carry a summary internally.

## Solution Assessment

Unclear until maintainers choose the SDK shape. An optional `summary?: string` gated by explicit hook opt-in is plausible and safer than default-on exposure, but it needs privacy, truncation, and unavailable-runtime semantics.
