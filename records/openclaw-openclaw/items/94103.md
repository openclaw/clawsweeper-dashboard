## Summary

Keep open: this PR targets a real unresolved Tailscale Serve duplicate-entry bug, but it is not merge-ready because the no-service startup cleanup would reset all Serve config by default, live Tailscale proof is missing, and several sibling PRs are competing for the same canonical issue.

## What This Changes

The PR adds best-effort `disableTailscaleServe` cleanup before gateway Serve startup enablement and extends gateway Tailscale tests for ordering, serviceName, funnel, preserveFunnel, and cleanup-failure paths.

## Best Possible Solution

Land one canonical fix for the linked bug that removes only OpenClaw-owned stale Serve state or makes broad cleanup an explicit operator choice, with focused regression coverage and redacted live Tailscale before/after proof.

## Reproduction Assessment

Yes for source-level reproduction: current main reapplies `tailscale serve --bg --yes <port>` without startup cleanup, and upstream Tailscale source shows the PR's no-service cleanup maps to global Serve reset. No live Tailscale daemon before/after proof is supplied in this PR.

## Solution Assessment

No. Pre-cleaning before enablement is a plausible repair layer, but this implementation is too broad because the no-service branch turns an opt-in global reset into default startup behavior.
