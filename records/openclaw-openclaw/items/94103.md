## What This Changes

The PR adds best-effort Tailscale Serve cleanup before startup enablement and adds gateway tests for ordering, failure tolerance, serviceName, funnel, and preserveFunnel cases.

## Best Possible Solution

Land one canonical fix for the linked issue that removes only OpenClaw-owned stale Serve state or makes broad cleanup an explicit operator choice, with redacted live Tailscale before/after proof.

## Reproduction Assessment

Yes, source-reproducible but not live-reproduced in this review. Current main reapplies `tailscale serve --bg --yes <port>` without startup cleanup when resetOnExit is false, and the linked issue gives concrete restart plus `tailscale serve status` steps.

## Solution Assessment

No. Pre-cleaning before enablement is a plausible direction, but the current patch is too broad because it turns a previously opt-in cleanup into a default reset of Serve state that may be operator-owned.
