## Summary

Keep open; the target behavior is useful, but this PR is not merge-ready because it makes abort provenance client-controlled, leaves several exact abort-payload tests stale, and only provides helper-script proof rather than a live Control UI/Gateway Stop flow.

## What This Changes

The PR adds abort provenance for UI Stop requests so user-initiated aborts can suppress `EmbeddedAttemptSessionTakeoverError` from the next agent context while non-user aborts remain fatal.

## Best Possible Solution

Make user-stop provenance server-owned or explicitly maintainer-approved as a public contract, keep non-user aborts fatal, refresh all affected UI/realtime/workboard tests, and add live Control UI/Gateway Stop proof.

## Reproduction Assessment

Yes, from source inspection rather than live execution: current main routes `chat.abort` as `stopReason: "rpc"` and only timeout gets an AbortSignal reason, while cleanup takeover errors are rejected into the run. I did not reproduce the full browser Stop flow live.

## Solution Assessment

No. Suppressing takeover noise for true user stops is the right product shape, but the current implementation makes the provenance client-controlled and leaves tests and live proof incomplete.
