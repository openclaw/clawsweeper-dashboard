Keep open; the PR is not merge-ready because its non-timeout `AbortSignal.aborted` heuristic still conflates user Stop with restart and auth-revocation aborts, which can hide session takeover cleanup errors for non-user aborts.

## What This Changes

The PR changes embedded agent cleanup handling so session takeover cleanup errors are logged instead of rejected when the attempt abort signal is aborted and timeout flags are not set.

## Best Possible Solution

Thread explicit user-stop provenance into the embedded attempt cleanup decision, keep timeout/restart/auth-revoked/internal aborts fatal, and prove both the live Stop flow and the non-user abort cases.

## Reproduction Assessment

## Solution Assessment

No. Suppressing on `AbortSignal.aborted` plus timeout exclusions is only a plausible heuristic; the maintainable fix is an explicit user-stop provenance contract with tests for the sibling abort reasons.
