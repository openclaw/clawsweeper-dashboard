## What This Changes

The PR adds a helper and cleanup branch in `runEmbeddedAttempt` to warn and suppress `EmbeddedAttemptSessionTakeoverError` when the attempt abort signal is already aborted and no timeout flag is set.

## Best Possible Solution

Carry explicit user-stop provenance into the embedded attempt cleanup path and suppress only that code, while preserving takeover errors for timeout, restart, superseded, post-compaction, and internal aborts with focused regression coverage and real Stop-button proof.

## Reproduction Assessment

No high-confidence live reproduction for the user-facing Stop-button report was established here; the PR body gives manual steps and copied snippets, but no inspectable artifact. The patch defect is source-reproducible from the shared abort-signal paths for user, restart, and internal aborts.

## Solution Assessment

No: suppressing the context pollution is a plausible direction, but `params.abortSignal?.aborted` is too broad to be the best fix. The maintainable path is an explicit user-stop code plus tests proving non-user abort takeover errors still surface.
