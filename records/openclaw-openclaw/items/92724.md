## Summary

Keep open: this is a focused, useful PR against watcher hot paths that current main still contains. I found no blocking correctness or security findings, but maintainers should consciously accept the write-stability tradeoff from removing Chokidar's dependency-level guard.

## What This Changes

The PR removes Chokidar `awaitWriteFinish` from the Gateway config reload watcher and skills runtime watcher, then adds focused tests asserting those watcher options stay absent.

## Best Possible Solution

Land the narrow watcher-option removal once maintainers accept the debounce/stability tradeoff, without adding a new public config switch for this path.

## Reproduction Assessment

No: I did not reproduce the reporter's exact macOS LaunchAgent CPU loop in this read-only review. Source inspection confirms current main still has the two watcher options named in the issue, and the PR body supplies after-fix macOS runtime evidence.

## Solution Assessment

Yes, with a compatibility caveat: removing dependency-level file-size polling is the narrowest performance fix for the implicated watcher options, provided maintainers are comfortable relying on OpenClaw's debounce and raw skill-file stability handling.
