- `merge-risk: 🚨 compatibility`: The diff changes existing hot-reload semantics for slow or polling-backed file writes, which can break current config and skills refresh expectations.

## Summary

Keep open: the PR is focused and has useful macOS after-fix proof, but it is not merge-ready because it removes change-time write-stability guards from both watcher paths. Chokidar's awaitWriteFinish path only starts polling after add/change events, while current OpenClaw skips invalid config snapshots without retry, so the patch can trade reload correctness for a performance fix that may not address the reported idle loop.

## What This Changes

The PR removes Chokidar awaitWriteFinish options from the Gateway config and skills runtime watchers and adds focused tests asserting those options are absent.

## Best Possible Solution

Keep the performance direction, but replace dependency-level awaitWriteFinish with narrow change-triggered stability or retry logic for config reload and polling SKILL.md changes, then keep the macOS idle proof plus focused watcher tests.

## Reproduction Assessment

No: the linked issue has concrete macOS sampling evidence and current source confirms the two watcher options, but I did not reproduce the original high-idle-CPU loop locally. The PR supplies after-fix macOS proof, not same-machine before/after reproduction.

## Solution Assessment

No: removing awaitWriteFinish is a plausible mitigation, but not the best final shape unless the patch preserves change-time stability for config and polling SKILL.md writes. The safer solution is to avoid idle overhead without dropping partial-write protection.
