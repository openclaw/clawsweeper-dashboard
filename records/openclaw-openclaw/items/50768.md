## Summary

Keep open. The linked external implementation shows a real recurring-orchestration use case, but current main still lacks reusable sub-agent templates, and adding them would change OpenClaw's public config/tool API plus model-visible workspace file reads, so this needs maintainer product/API review rather than cleanup closure.

## What This Changes

The issue requests reusable sub-agent templates that combine a predefined prompt file, model, timeout, and label with a new `sessions_spawn` template parameter.

## Best Possible Solution

Maintainers should first choose the product boundary; if accepted for core, add a typed additive template contract with safe workspace-relative prompt-file reads, override precedence, discovery output, generated schema/docs updates, and focused coverage.

## Reproduction Assessment

Not applicable. This is an additive feature request, and source inspection shows the requested `template` tool parameter and `subagents.templates` config are absent rather than a broken documented behavior.

## Solution Assessment

Unclear. A ClawHub/plugin wrapper could cover some orchestration use cases with `api.runtime.subagent`, but the requested built-in `sessions_spawn.template` API should not be added without maintainer API and security decisions.
