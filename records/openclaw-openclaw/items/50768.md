## Summary

Keep open. Current main still has explicit `sessions_spawn` overrides and global/per-agent sub-agent defaults, but it does not implement reusable sub-agent templates, a `sessions_spawn.template` selector, prompt-file prepending, or template discovery. The linked AIpexForge prior-art shows a real orchestration pattern, but this changes public config/tool contracts and introduces workspace file reads into model context, so it needs maintainer product/API and security-boundary review rather than stale cleanup.

## What This Changes

Add reusable per-agent sub-agent templates that combine prompt files with default model, timeout, and label values and can be selected from `sessions_spawn`.

## Best Possible Solution

Maintainers should first decide whether this belongs in core or as an extension seam. If accepted for core, the end state should be an additive typed template contract with safe workspace-relative prompt-file resolution, bounded reads, explicit override precedence, template discovery, generated schema/docs updates, and focused tests.

## Reproduction Assessment

Not applicable as a feature request. The current-main verification path is high confidence: schema, config, native/ACP spawn execution, docs, and discovery inspection all show no `template` selector, `promptFile` handling, or reusable template discovery.

## Solution Assessment

Unclear until maintainers decide the product/API and security boundary. The requested direction is plausible, but a maintainable implementation needs typed additive config and file-read guardrails rather than ad hoc prompt prepending.
