## Summary

Keep open. Current main still supports explicit `sessions_spawn` parameters plus global/per-agent defaults, but it does not implement reusable sub-agent templates, a `template` selector, prompt-file prepending, or template discovery; the requested change also crosses public config/tool API and file-read security boundaries.

## What This Changes

Request to add reusable per-agent sub-agent templates that provide prompt-file text plus model, timeout, and label defaults for `sessions_spawn`, with explicit tool parameters overriding template values.

## Best Possible Solution

Maintainers should decide the product/API boundary first. If accepted for core, the end state should be an additive typed template contract with safe workspace-relative prompt-file resolution, bounded reads, explicit override precedence, template discovery, generated schema/docs updates, and focused tests; if not accepted, document the skill/plugin orchestration pattern and leave the built-in tool contract unchanged.

## Reproduction Assessment

Not applicable as a feature request. The current-main verification path is high confidence: schema, execution, prompt construction, config schemas, docs, discovery, and tests all show no `template` selector, `promptFile` handling, or reusable template discovery.

## Solution Assessment

Unclear until maintainers decide the API and security boundary. The requested direction is plausible, but the safest implementation would be a typed additive contract with file-read guardrails rather than ad hoc prompt prepending.
