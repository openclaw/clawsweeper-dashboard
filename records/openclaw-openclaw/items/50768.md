## Summary

Keep open. Current `origin/main` still has explicit `sessions_spawn` parameters plus global/per-agent defaults, but no reusable sub-agent template config, `template` selector, prompt-file prepending, or template discovery; this is a new public config/tool feature with security-sensitive file-read design questions.

## What This Changes

The issue requests reusable sub-agent templates that combine a prompt file, model, timeout, and label defaults for repeated `sessions_spawn` calls.

## Best Possible Solution

Maintainers should decide the core API and file-read security boundary first; if accepted, implement an additive typed template contract with safe workspace-relative prompt resolution, override precedence, discovery, schema/docs updates, and focused tests.

## Solution Assessment
