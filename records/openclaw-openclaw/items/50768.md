## Summary

Keep open. Current main and the latest release still lack reusable sub-agent templates, a `sessions_spawn.template` parameter, prompt-file prepending, or template discovery; the related open label-template PR only covers fallback labels. The external AIpexForge implementation shows the orchestration need, but adding this to OpenClaw core would change public config/tool API and introduce workspace prompt-file reads, so it needs maintainer API and security-policy review before implementation.

## What This Changes

Request adds reusable per-agent sub-agent templates with `promptFile`, model, timeout, and label defaults, plus a `sessions_spawn` template parameter with explicit-argument override precedence.

## Best Possible Solution

Maintainers should decide the core versus plugin/ClawHub boundary first; if accepted for core, add a typed template contract with safe prompt-file resolution, bounded reads, override precedence, discovery, schema/docs generation, and focused tests.

## Reproduction Assessment

Not applicable. This is a feature request; source, docs, tests, and latest-release inspection show the requested config/tool API is absent rather than a broken existing contract.

## Solution Assessment

Unclear. The proposal is plausible and the linked external implementation shows real workflow pressure, but core prompt-file reads and a new `sessions_spawn` parameter need maintainer API and security-policy approval before this can be called the best solution.
