## Summary

Keep open. Current main still exposes explicit `sessions_spawn` parameters plus global/per-agent sub-agent defaults, but it does not implement reusable prompt/model/timeout templates, a `template` selector, prompt-file prepending, or template discovery; the requested behavior is a new public config/tool feature with file-read security design questions.

## What This Changes

The issue requests reusable sub-agent templates that bundle a prompt file, model, timeout, and label defaults for `sessions_spawn`.

## Best Possible Solution

Maintainers should decide the core API and file-read security boundary first; if accepted, implement an additive typed template contract with safe workspace-relative prompt resolution, override precedence, discovery, schema/docs updates, and focused tests.
