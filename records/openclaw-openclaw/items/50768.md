## Summary

Keep open. Current main still supports explicit `sessions_spawn` parameters plus global/per-agent defaults, but not reusable sub-agent templates, a `template` selector, prompt-file prepending, or template discovery; the request changes public config/tool API and needs maintainer product and security review.

## What This Changes

The issue requests reusable `agents.list[].subagents.templates` entries and a `sessions_spawn(template=...)` parameter that prepends a prompt file and supplies default model, timeout, and label values.

## Best Possible Solution

Maintainers should decide the core versus skill/plugin/API boundary first; if accepted for core, add an additive typed template contract with safe workspace-relative prompt-file resolution, override precedence, discovery, schema/docs generation, and focused tests.

## Reproduction Assessment

Not applicable as a feature request. The current-main verification path is high confidence: source, strict schemas, docs, and tests show no reusable template config, `template` selector, prompt-file prepending, or template discovery.

## Solution Assessment

Unclear until maintainers decide the API and security boundary. The requested direction is plausible, but the maintainable version would need a typed additive contract and file-read guardrails rather than ad hoc prompt prepending.
