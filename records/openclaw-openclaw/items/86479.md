  - confidence: 0.92

  - body: The public `channel-inbound` subpath no longer exports the shipped `buildChannelTurnContext`, `filterChannelTurnSupplementalContext`, `buildChannelTurnMediaPayload`, or matching `*Turn*` type aliases. Since this PR presents the old surfaces as deprecated compatibility paths, those names should remain as deprecated aliases so existing plugins do not fail immediately on upgrade.
  - confidence: 0.9
- **[P1] Keep inbound-reply-dispatch legacy names:** `src/plugin-sdk/inbound-reply-dispatch.ts:80-98`
  - body: This compatibility subpath now exposes renamed helpers but drops shipped names such as `runPreparedInboundReplyTurn`, `runInboundReplyTurn`, and `recordDroppedChannelTurnHistory`. Existing plugin imports from `openclaw/plugin-sdk/inbound-reply-dispatch` will fail, so keep forwarding aliases while moving bundled callers to the new names.
  - confidence: 0.9

  - body: `channel-message` used to export `hasFinalChannelTurnDispatch`, `hasVisibleChannelTurnDispatch`, and `resolveChannelTurnDispatchCounts`; this branch only keeps the newer `ChannelMessage` names.   - confidence: 0.86
