## Best Possible Solution

Land one canonical lifecycle fix only after the detector is narrowed to gateway registry-miss request errors, maintainers accept safe restart as the intended recovery, and redacted real external-plugin proof shows login, restart, post-restart running status, and ideally message delivery.

## Reproduction Assessment

No live high-confidence reproduction was run in this read-only review. The source path is clear: current main and v2026.6.8 call `channels.start` after local login, and the gateway can reject unloaded external channel plugins with the reported invalid/unknown-channel errors.

## Solution Assessment

No, not as written. The restart fallback is a plausible mitigation, but the detector should use the gateway request-error contract and the PR needs real external-plugin proof plus maintainer agreement that restart is the right canonical recovery.
