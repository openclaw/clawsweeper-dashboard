## What This Changes

The PR adds a CLI fallback that requests `gateway.restart.request` when post-login `channels.start` fails with invalid or unknown channel errors, plus focused channel-auth tests.

## Best Possible Solution

Land one canonical lifecycle fix only after the restart detector is narrowed to gateway registry-miss errors, regression coverage protects false positives, and redacted real external-plugin proof shows login, restart request, post-restart running status, and ideally message delivery.

## Reproduction Assessment

No live high-confidence reproduction was run in this read-only review. The source path is reproducible: current main calls `channels.start` after local login, and the gateway rejects channels missing from the active registry with the reported invalid/unknown-channel errors.

## Solution Assessment

No, not as written. The restart fallback is a plausible mitigation, but it should first narrow detection to gateway registry-miss errors and prove the real external-plugin login path; broader startup-loader reports may need a separate canonical fix.
