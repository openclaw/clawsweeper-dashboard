## What This Changes

The PR adds a CLI post-login fallback that requests `gateway.restart.request` when `channels.start` fails with an unloaded/unknown channel error, plus mocked coverage for the fallback path.

## Best Possible Solution

Land only after maintainers confirm safe restart is the intended unloaded-plugin recovery and the PR body includes redacted real plugin-channel proof showing login, restart request, post-restart running status, and ideally inbound delivery.

## Reproduction Assessment

## Solution Assessment

Acceptable mitigation, not fully proven best. Restart is an existing seam for plugin registry changes, but automatic restart from login needs maintainer acceptance and real after-fix proof before it is the best merge path.
