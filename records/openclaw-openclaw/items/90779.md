## What This Changes

The PR adds a CLI post-login fallback that calls `gateway.restart.request` after selected `channels.start` failures and adds channel-auth tests for that fallback.

## Best Possible Solution

Land one canonical lifecycle fix after the detector is narrowed to structured `INVALID_REQUEST` registry-miss errors, maintainers accept safe restart as the intended recovery, and redacted external-plugin proof shows login, restart, and post-restart channel recovery.

## Reproduction Assessment

## Solution Assessment

No, not as written. The restart fallback is a plausible mitigation, but it should use the gateway request-error contract and needs real external-plugin proof plus maintainer agreement that restart is the canonical recovery.
