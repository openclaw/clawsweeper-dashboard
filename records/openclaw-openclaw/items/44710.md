## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Fail closed when tool filtering hooks fail:** `src/plugins/hooks.ts:768-772`
  - body: `runBeforeToolsResolve` uses the generic modifying-hook path, but neither the default runner policy nor the global runner marks `before_tools_resolve` fail-closed. If a policy hook throws or times out, the error is logged and the unfiltered tool list continues to the model, which breaks the access-control purpose of this hook.
  - confidence: 0.88
- **[P2] Reserve bundled names after client filtering:** `src/agents/pi-embedded-runner/run/attempt.ts:518-521`
  - body: `reservedCoreAndClientNames` is built from `clientToolDefsPre` before the new hook filters client tools. A client-supplied name that the hook later denies can still collide with bundled MCP/LSP registration and suppress or rename trusted bundled tools, so denied tools still influence the final available tool set.
  - confidence: 0.83
- **[P2] Move the hook contract into hook-types:** `src/plugins/types.ts:1817`
  - body: Current main moved the public hook-name list and handler map to `src/plugins/hook-types.ts`, but this PR adds `before_tools_resolve` back to the old `types.ts` contract location. After rebase, registry validation and plugin typing need to update the current contract source instead.
  - confidence: 0.79
