## Summary

## What This Changes

The PR adds a public `before_tools_resolve` plugin hook with allow/deny merging, tool-list filtering in embedded attempt and compaction paths, and focused tests.

## Best Possible Solution

If maintainers want list-level tool filtering, land a rebased hook API that uses `src/plugins/hook-types.ts`, fails closed for policy errors, prevents denied names from affecting bundled/client reservations, includes docs/tests, and has redacted real-run proof.

## Solution Assessment

No. The feature direction may be useful, but this branch should not merge until it is rebased onto the current hook contract, made fail-closed, fixed for denied-name reservation effects, and backed by real behavior proof.
