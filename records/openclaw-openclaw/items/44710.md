Full review comments:

- **[P1] Fail closed when tool filtering hooks fail:** `src/plugins/hooks.ts:771-775`
  - body: `before_tools_resolve` is an access-control gate, but it uses the generic modifying hook path while hook errors are caught by default.   - confidence: 0.86

  - confidence: 0.82
- **[P2] Register the hook in the current hook catalog:** `src/plugins/types.ts:1820`
  - body: Current main keeps public hook names and handler maps in `src/plugins/hook-types.ts`, but this branch adds `before_tools_resolve` to the older `src/plugins/types.ts` surface. After rebase, registry validation and public typing need to update the current contract source or the hook can be treated as unknown.
  - confidence: 0.78
