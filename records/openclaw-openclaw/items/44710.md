## Summary

Keep open. Current main does not implement the requested list-level `before_tools_resolve` hook, and the PR still introduces security-sensitive tool-access behavior with blocking review findings around fail-open policy handling, bundled/client tool name interactions, and stale hook contract placement.

## What This Changes

The PR adds a public `before_tools_resolve` plugin hook intended to filter the assembled model-visible tool list with allow/deny results across normal agent runs and compaction, plus hook runner support and tests.

## Best Possible Solution

If maintainers want this API, land it as a rebased plugin contract change in `src/plugins/hook-types.ts`, make the list-level policy fail closed, evaluate once over the final post-static tool list, ensure denied client tools cannot reserve or suppress bundled tools, and add docs plus regression coverage for normal runs, compaction, client tools, bundled tools, allow intersection, and hook failure behavior.

## Reproduction Assessment

Yes for the review findings: the provided PR diff plus current-main source trace show the new hook would use the generic fail-open path and interact with pre-filter client tool names in bundled tool reservation. Runtime reproduction was not attempted because this review is read-only.

## Solution Assessment

No. The requested API is plausible, but this patch is not the best merge shape until it is rebased to the current hook contract files and its tool-access policy semantics are fail-closed and applied to the final tool set.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Fail closed when tool filtering hooks fail:** `src/plugins/hooks.ts:765`
  - body: `runBeforeToolsResolve` uses the generic modifying-hook path, but the global runner only marks `before_tool_call` as fail-closed. Since unspecified hooks fail open, a thrown or timed-out `before_tools_resolve` policy would log and continue with the unfiltered tool list, exposing tools the plugin intended to hide.
  - confidence: 0.86

  - body: The PR builds bundled MCP/LSP reserved names from client tool definitions before `before_tools_resolve` filters them. A denied client-supplied name can still suppress or rename a trusted bundled tool through collision handling even though that client tool is not allowed in the final set.
  - confidence: 0.82
- **[P2] Move the hook contract to hook-types:** `src/plugins/types.ts:1817`
  - body: Current main moved `PluginHookName`, `PLUGIN_HOOK_NAMES`, and `PluginHookHandlerMap` into `src/plugins/hook-types.ts`, while this diff adds `before_tools_resolve` to the old `types.ts` contract location. After rebase, registry validation and hook runner typing should be updated at the current contract source.
  - confidence: 0.78
