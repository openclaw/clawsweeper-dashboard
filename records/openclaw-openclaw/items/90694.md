## Summary

Keep this PR open: the targeted fix is useful, but current head can still let an IMDS-disabled probe contaminate a concurrent explicit Bedrock IMDS/default-chain probe through shared `process.env`.

## What This Changes

The branch changes Amazon Bedrock memory credential detection to skip AWS SDK/IMDS probing during no-signal auto-detection, preserve explicit Bedrock default-chain probing, memoize probes, and add focused tests.

## Best Possible Solution

Keep the Bedrock-local no-signal guard, but isolate or serialize temporary IMDS disablement so explicit Bedrock default-chain and IMDS credential resolution cannot inherit another probe mode's environment state.

## Reproduction Assessment

Yes, at source level. PR head writes `AWS_EC2_METADATA_DISABLED` on `process.env` for IMDS-disabled probes while allowing a different-mode probe to run concurrently, and the pinned AWS SDK reads `process.env` before choosing the remote provider.

## Solution Assessment

No, not yet. The provider-local boundary is reasonable, but shared environment mutation needs isolation or serialization before this is the best maintainable fix.
