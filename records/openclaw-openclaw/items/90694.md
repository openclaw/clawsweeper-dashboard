## Summary

Keep open: this is a useful, focused bug-fix PR, but the current head is still not merge-ready because explicit Bedrock configurations that use a custom provider id can be treated as non-explicit and lose IMDS/default-chain credential support.

## What This Changes

The PR changes Amazon Bedrock memory credential probing so no-signal auto-detection avoids AWS IMDS while explicit Bedrock configuration can still use the AWS SDK default credential chain.

## Best Possible Solution
