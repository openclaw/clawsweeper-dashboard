## Summary

Keep this PR open: the provider-local no-signal IMDS suppression is the right layer, but the current diff still treats only the literal `bedrock` provider id as explicit and can fail closed for configured Bedrock aliases that currently resolve through `models.providers.<id>.api`.

## What This Changes

The PR changes Amazon Bedrock memory credential probing so no-signal auto-detection avoids AWS IMDS while explicit Bedrock configuration can still use the AWS SDK default credential chain.

## Best Possible Solution

Keep the provider-local no-signal IMDS suppression, but resolve configured Bedrock API aliases before deciding whether an explicit Bedrock memory provider may use the full AWS default chain.
