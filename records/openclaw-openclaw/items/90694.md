## Summary

Keep this PR open: it is a focused candidate fix for the linked Bedrock IMDS probe regression, but the current patch can still let an IMDS-disabled probe contaminate an overlapping explicit allow-IMDS credential probe.

## What This Changes

The PR changes Amazon Bedrock memory credential detection to skip AWS SDK credential-chain loading without AWS signals, preserve IMDS for explicit Bedrock memory configuration, and cache/coalesce process.env credential probes.

## Best Possible Solution

Keep the Bedrock-local no-signal guard, but isolate or serialize temporary IMDS disabling and get maintainer acceptance for the no-signal default-chain compatibility boundary before merge.

## Reproduction Assessment

Yes, at source level. A concurrent `hasAwsCredentials(process.env, ..., { allowImds: false })` probe can set `AWS_EC2_METADATA_DISABLED=true` while an overlapping `{ allowImds: true }` probe starts, and the pinned AWS SDK reads that shared process environment value.

## Solution Assessment

No, not as currently written. The provider-local boundary is reasonable, but the temporary shared environment mutation must be isolated or serialized before this is the best maintainable fix.
