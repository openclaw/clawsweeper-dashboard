## Summary

This PR is still useful because current main and v2026.6.11 still call the AWS SDK default credential chain from Bedrock memory credential detection, but it needs changes before merge because an IMDS-disabled probe can mutate shared `process.env` while a concurrent explicit allow-IMDS probe is running.

## What This Changes

The PR changes Amazon Bedrock memory credential detection to skip AWS SDK/IMDS probing when no AWS credential signals exist, preserve full default-chain probing for explicit Bedrock memory configuration, and add memoization plus tests.

## Best Possible Solution

Keep the Bedrock-local no-signal guard, but isolate or serialize temporary IMDS disablement so explicit Bedrock default-chain and IMDS credential resolution cannot inherit another probe mode's environment state.

## Reproduction Assessment

Yes, at source level. PR head writes `AWS_EC2_METADATA_DISABLED` on `process.env` for IMDS-disabled probes while allowing a different-mode probe to run concurrently, and the AWS SDK reads `process.env` before choosing the remote provider.

## Solution Assessment

No, not yet. The provider-local boundary is reasonable, but shared environment mutation needs isolation or serialization before this is the best maintainable fix.
