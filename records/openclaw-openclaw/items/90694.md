## Summary

Keep this PR open, but do not merge it as-is: it targets the right Bedrock IMDS-noise bug, yet the patch silently changes documented EC2 instance-role credential behavior and only provides mocked proof.

## What This Changes

The branch changes the Amazon Bedrock memory credential probe to skip the AWS SDK chain unless selected AWS signals exist, temporarily disables IMDS during some chain probes, adds a five-minute process-env probe cache, and expands unit coverage.

## Best Possible Solution

Keep the non-AWS sandbox from producing repeated IMDS probes while preserving or explicitly documenting the supported EC2/ECS/EKS role path, with real sandbox proof and EC2-role upgrade proof before merge.

## Reproduction Assessment

Yes, by source inspection: current main calls AWS SDK `defaultProvider()` from Bedrock memory credential detection, and the AWS SDK chain reaches IMDS when earlier sources miss. I did not run the full NemoClaw sandbox reproduction locally.

## Solution Assessment

No, this is not the best fix as written because it prevents plain EC2 instance-profile credentials from being detected unless users discover a new opt-in. A safer solution needs a compatibility-preserving path or an explicit maintainer decision to make IMDS detection opt-in.
