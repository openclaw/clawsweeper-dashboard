## Summary

Keep this PR open: current main and v2026.6.9 still have the Bedrock credential-chain IMDS probe, and this PR is the active candidate fix, but the latest head still lets IMDS-disabled and allow-IMDS probes overlap through shared process environment state.

## What This Changes

The PR changes Amazon Bedrock memory credential detection so no-signal auto-detection skips the AWS SDK credential chain while explicit Bedrock memory configuration can still use the default AWS credential chain.

## Best Possible Solution

Keep the Bedrock-local no-signal guard, but isolate or serialize temporary IMDS disablement so explicit Bedrock default-chain and IMDS credential resolution cannot inherit another probe mode's environment state.

## Reproduction Assessment

Yes, at source level. PR head writes AWS_EC2_METADATA_DISABLED on process.env for IMDS-disabled probes while allowing a different-mode probe to run concurrently, and the AWS SDK reads process.env before choosing the remote provider.

## Solution Assessment

No, not yet. The provider-local boundary is reasonable, but the shared environment mutation needs isolation or serialization before this is the best maintainable fix.
