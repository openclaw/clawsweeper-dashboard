## Summary

Keep this PR open, but it still needs changes before merge: the Bedrock-local fix is aimed at the right code path, yet the latest head still lets an IMDS-disabled probe mutate shared `process.env` while an explicit allow-IMDS probe can overlap and inherit that disabled state.

## What This Changes

The PR changes Amazon Bedrock memory credential detection to skip AWS SDK default-chain loading when no AWS credential signals exist, temporarily disable IMDS for non-IMDS probes, memoize `process.env` probes, and allow IMDS for explicitly configured Bedrock memory providers.

## Best Possible Solution

Keep the Bedrock-local no-signal short-circuit, but isolate or serialize temporary IMDS disabling so explicit Bedrock IMDS/default-chain credential resolution cannot inherit another probe mode's environment state.

## Reproduction Assessment

Yes, at source level. A concurrent `hasAwsCredentials(process.env, ..., { allowImds: false })` probe can set `AWS_EC2_METADATA_DISABLED=true` while an overlapping `{ allowImds: true }` probe starts, and the AWS SDK source reads that shared `process.env` value when deciding whether to use IMDS.

## Solution Assessment

No, not as currently written. The provider-local short-circuit is the right boundary, but the temporary shared environment mutation must be isolated or serialized before the PR is the best maintainable fix.
