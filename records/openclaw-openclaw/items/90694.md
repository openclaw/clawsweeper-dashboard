## Summary

Keep this PR open: it targets a real Bedrock credential-detection regression that is still present on current main and the latest release, but the PR head still has a P1 auth-provider race where one probe mode can temporarily disable IMDS for another.

## What This Changes

The PR changes Amazon Bedrock memory credential detection to skip AWS SDK/IMDS probing when auto-detecting without credential signals, while preserving default-chain probing for explicit Bedrock memory configuration and adding memoization plus tests.

## Best Possible Solution

Keep the Bedrock-local no-signal guard, but isolate or serialize temporary IMDS disablement so explicit Bedrock default-chain and IMDS credential resolution cannot inherit another probe mode's environment state.

## Reproduction Assessment

Yes, at source level. PR head writes `AWS_EC2_METADATA_DISABLED` on `process.env` for IMDS-disabled probes while allowing a different-mode probe to run concurrently, and the AWS SDK reads `process.env` before choosing the remote provider.

## Solution Assessment

No, not yet. The provider-local boundary is reasonable, but shared environment mutation needs isolation or serialization before this is the best maintainable fix.
