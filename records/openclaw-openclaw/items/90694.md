## Summary

Keep this PR open because current main and the latest release still have the Bedrock memory credential path that can fall through to IMDS, but the patch is not merge-ready: overlapping IMDS-disabled and allow-IMDS probes can share `process.env` and break the explicit EC2/IMDS path the PR is trying to preserve.

## What This Changes

The PR changes Amazon Bedrock memory credential detection to skip AWS SDK/IMDS probing when auto-detecting with no credential signals, while preserving default-chain probing for explicit Bedrock memory configuration and adding memoization plus tests.

## Best Possible Solution

Keep the Bedrock-local no-signal guard, but isolate or serialize temporary IMDS disablement so explicit Bedrock default-chain and IMDS credential resolution cannot inherit another probe mode's environment state.

## Reproduction Assessment

Yes, at source level. PR head writes `AWS_EC2_METADATA_DISABLED` on `process.env` for IMDS-disabled probes while allowing a different-mode probe to run concurrently, and the AWS SDK reads `process.env` before choosing the remote provider.

## Solution Assessment

No, not yet. The provider-local boundary is reasonable, but shared environment mutation needs isolation or serialization before this is the best maintainable fix.
