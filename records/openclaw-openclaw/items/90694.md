## Summary

Keep open: the PR targets a real Bedrock memory startup issue, but it is not merge-ready because the patch blocks documented AWS SDK default-chain credential setups and still lacks after-fix proof from a real non-AWS sandbox or AWS role environment. Current main and v2026.6.1 still have the IMDS-probing behavior, so the linked bug remains valid.

## What This Changes

The branch changes Amazon Bedrock memory credential probing to skip the AWS SDK chain unless selected AWS signals are present, temporarily disables IMDS during some probes, memoizes process-env probe results, and adds unit coverage.

## Best Possible Solution

Preserve Bedrock memory's documented AWS SDK default credential chain for explicit configurations, while preventing no-env auto-detection from probing IMDS without making an undocumented opt-in the upgrade path.

## Reproduction Assessment

Source-reproducible: current main and v2026.6.1 call the AWS SDK defaultProvider after static credential checks, and dependency source shows that chain can reach IMDS. I did not run the reported NemoClaw/OpenShell sandbox path, so real-environment reproduction remains unproven here.

## Solution Assessment

No. The patch is a plausible mitigation for no-env sandbox noise, but it is not the best fix because it narrows documented default-chain credential support; the safer direction is to avoid IMDS only in the non-explicit auto-detection path or via existing AWS SDK controls while preserving explicit Bedrock setups.
