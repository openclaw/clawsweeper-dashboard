## Summary

Keep open: the PR is still the active fix path for the Bedrock IMDS probe, but the current patch has a merge-blocking auth-provider race where an IMDS-disabled probe can contaminate an explicit allow-IMDS probe through shared process.env state.

## What This Changes

The PR changes Amazon Bedrock memory credential detection to skip SDK credential-chain loading when no AWS signals exist, temporarily disable IMDS for non-IMDS probes, memoize probes, and allow IMDS for explicit Bedrock memory configuration.

## Best Possible Solution

Keep the Bedrock-local no-signal guard, but isolate or serialize temporary IMDS disabling so explicit Bedrock IMDS/default-chain credential resolution cannot inherit another probe mode's environment state.

## Reproduction Assessment

Yes, at source level. A concurrent hasAwsCredentials(process.env, ..., { allowImds: false }) probe can set AWS_EC2_METADATA_DISABLED=true while an overlapping { allowImds: true } probe starts, and the AWS SDK source reads that shared process environment value.

## Solution Assessment

No, not as currently written. The provider-local boundary is reasonable, but the temporary shared environment mutation must be isolated or serialized before this is the best maintainable fix.
