## Summary

This PR is still necessary because current main still loads the AWS SDK default credential chain from Bedrock memory credential detection, which can fall through to IMDS. The patch looks like the right plugin-local fix with no blocking line-level findings, but it still needs maintainer sign-off on the auth-provider compatibility tradeoff before merge.

## What This Changes

The branch changes Amazon Bedrock memory credential detection to skip AWS default-chain/IMDS probing when Bedrock is not explicitly configured, preserve the full chain for explicit Bedrock memory use, memoize probes, and add regression tests.

## Best Possible Solution

Merge the plugin-local probe split after maintainers accept the explicit-Bedrock configuration boundary, with live AWS default-chain proof added first if they want stronger upgrade evidence.

## Reproduction Assessment

Yes. The linked issue has concrete sandbox logs, and current main source plus the locked AWS SDK source show that Bedrock memory credential detection can fall through to EC2 IMDS when env/bearer credentials are absent.

## Solution Assessment

Yes. The PR is the narrowest maintainable shape I found: it stays in the Bedrock plugin, avoids a new config/env surface, and preserves the AWS SDK chain for explicit Bedrock memory configuration.
