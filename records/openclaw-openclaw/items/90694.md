## Summary

## What This Changes

The PR changes Amazon Bedrock memory credential probing so no-signal auto-detection skips the AWS SDK/IMDS path, while literal Bedrock configuration can still allow the AWS default credential chain.

## Best Possible Solution

Preserve the provider-local no-signal IMDS suppression, but resolve configured Bedrock API aliases before deciding whether an explicit Bedrock memory provider may use the full AWS default chain.

## Solution Assessment

No, not yet. The provider-local probe split is the right layer, but it needs to preserve configured Bedrock API aliases before it is the best fix.
