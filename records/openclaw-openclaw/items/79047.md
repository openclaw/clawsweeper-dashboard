## Summary

Keep open: current main still has only model-switch bookkeeping, bounded/fresh CLI reseed, Claude CLI fallback preludes, and Codex-side injection for narrow side-thread cases; it does not provide the requested operator-controlled bridge for ordinary cross-runtime model switches. The related memory, transcript-replay, tool-continuity, and stale-binding issues are adjacent rather than a clean canonical replacement.

## What This Changes

The issue requests an opt-in continuity bridge so the same agent can preserve useful conversation context when switching across runtime backends.

## Best Possible Solution

Define an explicit same-agent runtime-handoff policy that reuses bounded transcript/reseed primitives only after maintainers settle privacy, auth/provider boundary, token cap, and operator-control semantics.

## Reproduction Assessment

Yes at source level for the missing capability: current main marks live model switches and has bounded reseed/fallback helpers, but no general opt-in runtime-switch continuity bridge. It is not a broken existing contract because the issue explicitly asks for new opt-in behavior.

## Solution Assessment

No automated fix is the best path yet: the requested behavior needs a maintainer product/security decision before choosing a command, config flag, plugin seam, or runtime-owned handoff implementation. The existing reseed, fallback prelude, and Codex inject APIs are useful building blocks, not a completed solution.
