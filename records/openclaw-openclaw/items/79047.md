## Summary

Keep open. Current main still has provider/runtime-scoped continuity plus a fallback-only claude-cli prelude, while the requested opt-in cross-runtime transcript bridge remains a new product/config feature with privacy and token-budget choices.

## What This Changes

The issue requests an opt-in way to bridge conversation context when the same agent switches between runtime backends such as claude-cli and the embedded Pi runtime.

## Best Possible Solution

Design a maintained opt-in cross-runtime handoff that reuses bounded transcript/prelude machinery with clear caps and privacy semantics, without mutating provider-owned CLI session state directly.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: converse through claude-cli, patch the session model to a Pi-runtime model, then the next run uses OpenClaw's session file while prior claude-cli turns remain in Claude's separate transcript. I did not run the live backends in this read-only review.

## Solution Assessment

Unclear because there is no patch yet. The requested direction is plausible, but the best implementation needs maintainer decisions on opt-in surface, summarization versus verbatim tail, caps, privacy, and how to generalize the existing fallback-only prelude.
