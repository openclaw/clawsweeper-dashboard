## Summary

Keep open: current main has partial transcript reseed and Claude-fallback prelude paths, but no operator-controlled continuity bridge for ordinary cross-runtime model switches. The related memory and CLI-session issues are adjacent rather than a clean canonical replacement for this narrower runtime-switch request.

## What This Changes

The issue requests an opt-in mechanism to preserve or summarize conversation context when an agent switches between runtime backends such as Claude CLI and the embedded OpenClaw runtime.

## Best Possible Solution

Define an explicit same-agent runtime-handoff policy, reuse the existing bounded transcript/reseed primitives where safe, keep the default conservative, and make privacy, auth/provider boundaries, and token caps visible before adding any config or command surface.

## Reproduction Assessment

Not applicable as a bug reproduction: the issue explicitly requests new opt-in behavior rather than reporting a broken existing contract. Source inspection confirms current main has only narrower reseed/fallback helpers, not the requested general runtime-switch bridge.

## Solution Assessment

No automated fix is the best path yet: the existing reseed and fallback-prelude code is a useful building block, but the requested behavior needs a maintainer product/security decision about defaults, scope, and provider-owned session continuity.
