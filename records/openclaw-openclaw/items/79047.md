## Summary

Keep open. Current main still has provider/runtime-owned continuity plus bounded fallback/reseed helpers, but it does not provide the requested opt-in bridge for user-initiated switches between claude-cli and the pi runtime; the linked memory and transcript umbrellas are adjacent rather than canonical replacements.

## What This Changes

The issue requests an opt-in way to preserve conversation context when switching an agent between runtime backends such as claude-cli and the pi runtime.

## Best Possible Solution

Design a maintained opt-in runtime handoff that projects a bounded summary or recent-turn window into the new runtime once, with explicit token, privacy, provider/auth, and audit semantics.

## Reproduction Assessment

Yes, from source plus the reporter's 2026.5.7 matrix: same-runtime switches preserve continuity, while claude-cli and pi use separate transcript/session stores and the current model-switch path does not inject a bridge. I did not run a live claude-cli/OpenRouter setup in this read-only review.

## Solution Assessment

Unclear because there is no patch yet. The requested opt-in direction is plausible, but maintainers should choose the user-facing policy surface before reusing the existing fallback-prelude and transcript-reseed machinery.
