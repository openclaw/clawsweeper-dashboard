## Summary

Keep open. Current main preserves same-runtime continuity and has a narrow claude-cli fallback prelude, but it still lacks the requested opt-in bridge for user-initiated cross-runtime model switches; related memory, fallback-visibility, and runtime-parity threads are adjacent rather than replacements.

## What This Changes

The issue requests an opt-in way to carry conversation context across runtime/backend model switches for the same agent.

## Best Possible Solution

Design a maintained opt-in runtime handoff contract that projects a bounded summary or recent turns into the first turn on the new runtime, with explicit privacy, token-budget, and docs semantics.

## Reproduction Assessment

Yes, as a current product gap rather than a broken existing contract. Source inspection shows provider-keyed CLI bindings, model patching without context staging, and fallback-only prelude injection, while the reporter supplied a targeted cross-axis test matrix.

## Solution Assessment

Unclear until maintainer product/API design is chosen. The requested opt-in bridge is plausible, but the maintainable path should define a runtime handoff contract with caps, consent, and audit behavior before implementation.
