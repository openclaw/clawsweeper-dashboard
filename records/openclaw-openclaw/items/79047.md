## Summary

Keep open. Current main has same-runtime continuity, chat-history import for some Claude CLI history, and a fallback-only Claude CLI prelude, but it still lacks the requested opt-in model-input bridge for ordinary cross-runtime model switches.

## What This Changes

The issue requests opt-in continuity bridging when an agent switches between runtime backends so prior conversation context can be available to the next runtime.

## Best Possible Solution

Design a maintained opt-in runtime handoff that uses bounded transcript import or next-turn injection with explicit token and privacy limits, while leaving provider-owned CLI session state intact.

## Reproduction Assessment

Yes, source-reproducible for the current behavior: sessions.patch changes the selected model, CLI bindings are provider-keyed, PI reads its OpenClaw session file, and the only model-input bridge is fallback-only. This is still a feature request rather than broken documented behavior.

## Solution Assessment

Unclear because there is no patch yet. The requested opt-in direction is plausible, but maintainers should first choose the trigger, config surface, token cap, and privacy boundary, likely reusing the existing Claude CLI import/prelude and session injection seams.
