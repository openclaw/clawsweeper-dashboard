## Summary

Keep open. Current main preserves continuity within a runtime and has a narrow claude-cli fallback prelude, but it still lacks the requested opt-in bridge for user-initiated cross-runtime model switches; the related runtime-parity and fallback-visibility threads are adjacent, not a replacement.

## What This Changes

The issue requests an opt-in way to preserve conversation context when an agent switches between runtime backends for the same conceptual agent.

## Best Possible Solution

Design a maintained opt-in runtime handoff that projects a bounded summary or recent turns into the first turn on the new runtime, with explicit token, privacy, and docs semantics, reusing existing prelude/import primitives without mutating provider-owned session state.

## Reproduction Assessment

Yes, as a current product gap rather than a bug: source inspection shows provider-keyed CLI bindings, session model patching, and fallback-only prelude injection, and the reporter supplied a targeted 2026.5.7 cross-runtime matrix. I did not run a live cross-runtime model switch locally.

## Solution Assessment

Unclear until product/API design is chosen. The requested direction is plausible, but the maintainable fix should define an explicit opt-in handoff contract with caps and privacy behavior before implementation.
