## Summary

Keep open: current main still lacks the requested opt-in cross-runtime context bridge; existing live model-switch, CLI reseed, fallback prelude, and session-memory paths are narrower mitigations rather than the requested product contract.

## What This Changes

Request an opt-in mechanism to preserve conversation continuity when the same agent switches across runtime backends.

## Best Possible Solution

Define one explicit, default-off runtime handoff contract with transcript source rules, length caps, auth/channel/workspace guardrails, and live proof for both claude-cli to embedded-runtime directions.

## Reproduction Assessment

Yes, by source inspection plus the reporter's May 10 matrix: current model switching updates selection state, while CLI-backed and embedded runtime transcript paths remain separate except for narrower fallback/reseed cases. I did not run a live claude-cli to embedded-runtime switch in this review.

## Solution Assessment

Unclear: an opt-in bridge is plausible, but maintainers need to choose the durable contract first, including whether this is a config flag, explicit handoff command, memory-backed recall, or a narrower runtime hook. The current code paths are not sufficient as the final solution because their replay rules are scoped to fallback or invalidation recovery.
