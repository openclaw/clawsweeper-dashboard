## Summary

Keep open: current main has live model-switch bookkeeping, shared transcript persistence, Claude CLI reseed/fallback mitigations, and memory alternatives, but it does not provide the requested operator-controlled bridge for ordinary cross-runtime model switches. The remaining work is a product/security contract, not conservative cleanup.

## What This Changes

Request an opt-in way to carry recent conversation context into the first turn after switching an agent between runtime backends.

## Best Possible Solution

Define one operator-controlled runtime handoff contract with transcript source, length cap, auth/channel/workspace guardrails, and live proof for claude-cli to embedded-runtime switching before implementation.

## Reproduction Assessment

Yes by source inspection rather than a live run: current model switching records a pending selection and current CLI reseed/fallback code is scoped to reset/fallback reasons, not ordinary cross-runtime handoff. I did not run a live claude-cli to embedded-runtime switch.

## Solution Assessment

Unclear: an opt-in bridge is plausible, but maintainers need to choose between a config flag, explicit command, memory/session-tail contract, or narrower docs/current-behavior proof before implementation.
