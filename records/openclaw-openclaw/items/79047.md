## Summary

Keep open. Current main still has isolated runtime transcript/session stores plus a fallback-only Claude CLI prelude, but no opt-in bridge for user-initiated claude-cli/Pi runtime switches.

## What This Changes

Add an opt-in mechanism to carry recent conversation context across runtime backend switches, especially between claude-cli and the Pi runtime.

## Best Possible Solution

Design a maintained opt-in runtime handoff that reuses bounded transcript import/prelude machinery, with one-shot semantics, token caps, privacy documentation, and regression coverage before expanding beyond fallback retries.

## Reproduction Assessment

Yes. Source inspection plus the reporter's 2026.5.7 matrix give a high-confidence path: switch the same agent between claude-cli and Pi-backed models, then ask about the prior runtime's conversation; I did not run the live OAuth/API setup locally.

## Solution Assessment

Unclear because there is no patch yet. The requested direction is plausible, but the best implementation should be a bounded opt-in handoff rather than silently merging provider-owned session stores.
