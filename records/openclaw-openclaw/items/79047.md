## Summary

Keep open: current main has bounded CLI reseed and transcript-sharing mitigations, but it still does not provide the requested opt-in cross-runtime context or identity-continuity handoff. The remaining work crosses session state, runtime, config, and security policy boundaries, so it needs maintainer product/security direction rather than cleanup closure.

## What This Changes

The issue requests an opt-in mechanism to preserve or explicitly bridge conversation context when an agent switches models across runtime backends such as `claude-cli`, OpenClaw, and Codex.

## Best Possible Solution

Define an explicit, opt-in runtime handoff contract that records whether the result is `CONTINUE`, `FORK`, or not comparable, with bounded transcript/summary transfer and hard auth, workspace, policy, and tool-approval boundaries.

## Reproduction Assessment

Partly: the source shows the relevant behavior model, especially provider-owned CLI session reuse versus OpenClaw transcript replay, and the reporter provided a detailed empirical matrix. I did not run a live cross-runtime scenario in this read-only review, and the item is a feature request rather than a broken current contract.

## Solution Assessment

Unclear as proposed: a simple transcript prefix or config flag would preserve information, but it would not settle identity continuity or security invariants. The best path is a maintainer-approved handoff contract before implementation.
