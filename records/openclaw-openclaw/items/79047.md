## Summary

Keep open. Current main has per-backend/session continuity and a narrow claude-cli fallback prelude, but it does not provide the requested opt-in bridge for user-initiated cross-backend model switches; this is a new product/config feature, not a stale cleanup close.

## What This Changes

Request an opt-in mechanism to carry summarized or recent transcript context across model changes that switch between runtime backends such as claude-cli, OpenRouter/PI, and codex-cli.

## Best Possible Solution

Define a maintained opt-in handoff path that reuses the existing session transcript/import and fallback-prelude machinery, with explicit token caps and privacy semantics, instead of mutating provider-owned session state directly.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes, from source and docs: the dashboard model picker persists `sessions.patch` model overrides, CLI bindings are provider-keyed, and the only claude-cli context prelude is limited to fallback retries; I did not run a live cross-backend setup in this read-only review.

## Solution Assessment

Is this the best way to solve the issue? Unclear because there is no patch yet; the requested direction is plausible, but maintainers should first choose the user-facing policy surface and then generalize the existing bounded prelude/import mechanisms.
