## Summary

Keep open. Current main has provider-owned CLI continuity plus partial reseed/fallback bridges, but it still lacks a maintained opt-in handoff for ordinary user-initiated switches between CLI runtimes and the PI runtime, especially Claude CLI to PI.

## What This Changes

Request an opt-in mechanism to preserve transcript context when switching an agent between provider-owned CLI runtimes and the PI runtime.

## Best Possible Solution

Define an explicit cross-runtime handoff contract that uses a bounded, sanitized summary or recent-turn tail for both CLI-to-PI and PI-to-CLI model-picker flows, with clear opt-in policy and caps.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes: the reporter supplied a targeted 2026.5.7 cross-axis matrix, and current source shows model patches do not stage a handoff while CLI bindings and PI session files remain separate runtime continuity stores.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose the product surface; the opt-in direction is plausible, but it should reuse the existing bounded reseed/fallback-prelude machinery rather than silently merging provider-owned session state.
