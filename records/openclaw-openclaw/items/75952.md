## Summary

Current main has removed the bundled plugin runtime-deps staging path that produced `plugin-runtime-deps`, so the reported Slack `ws` lookup failure from `2026.4.27` no longer matches the maintained startup/dependency model.

## What This Changes

The issue asks for Slack startup to stop crash-looping on Windows by ensuring `ws` is available to the bundled runtime dependency path.

## Best Possible Solution

Keep the simplified main-branch dependency model and ship it; users on `2026.4.27` should update to a build containing the runtime-deps staging removal rather than adding `ws` to the retired staging path.

## Reproduction Assessment

Yes. The report includes a concrete `2026.4.27` Windows 11 Slack startup path and exact missing-module log under `plugin-runtime-deps`; current-main verification focused on whether that path still exists.

## Solution Assessment

Yes. Removing bundled runtime-deps staging is the narrower maintainable fix for this class of crash than adding one more dependency to a deprecated generated install root.
