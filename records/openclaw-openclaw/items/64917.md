## Summary

Keep open. Current `main` still starts the nested `sessions_send` target run as non-deliverable while carrying the internal WebChat channel hint, still allows an A2A announce step to call Gateway `send`, and still seeds request delivery context before the delivery-policy check.

## What This Changes

The issue asks for `sessions_send` from visible WebChat-backed sessions to internal sessions to keep target execution and replies internal without rebinding the target to WebChat delivery context.

## Best Possible Solution

Define and enforce an internal/non-deliverable `sessions_send` boundary that neither seeds visible WebChat delivery context nor emits UI-visible announce output for internal target runs, while preserving intended peer A2A announce delivery.

## Reproduction Assessment

Yes, at source level. A visible WebChat-backed caller can start a non-deliverable target run that still carries the WebChat channel hint, persists delivery context, and can enter an A2A announce path that calls Gateway `send`.

## Solution Assessment

Unclear until maintainers choose the strict-internal versus peer-announce policy. The narrow maintainable direction is to suppress WebChat context seeding and visible announce/UI output for internal or non-deliverable target runs without breaking intended external A2A delivery.
