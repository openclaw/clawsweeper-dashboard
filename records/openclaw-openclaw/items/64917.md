## Summary

Keep open. Current main still lets `sessions_send` use WebChat-flavored internal target runs, keeps A2A announce delivery active, and can persist WebChat delivery hints onto fresh target sessions.

## What This Changes

The issue asks for `sessions_send` from visible WebChat sessions to internal sessions to remain internal and not rebind the target to WebChat delivery context.

## Best Possible Solution

Define and enforce a strict internal boundary for internal or non-deliverable `sessions_send` target runs while preserving documented peer-session A2A announce delivery for explicit deliverable targets.

## Reproduction Assessment

Yes. Source inspection on current main gives a focused reproduction path: call `sessions_send` from a WebChat-backed caller to a fresh internal target, then assert no visible chat broadcast/send occurs and no WebChat deliveryContext is persisted.

## Solution Assessment

Unclear until maintainers decide the intended boundary. The narrow maintainable direction is to suppress UI-visible broadcasts and WebChat delivery-context seeding for internal/non-deliverable `sessions_send` target runs while leaving explicit peer A2A announce delivery intact.

## Security Review

Status: needs_attention

Summary: This issue reports an existing visible/internal information-boundary leak in the `sessions_send` WebChat and A2A announce path.

Concerns:

- **[medium] Internal replies can surface through WebChat events:** `src/gateway/server-chat.ts:594`
  - body: `sessions_send` target runs are registered like normal visible runs unless something explicitly marks them hidden, and Gateway chat handling broadcasts assistant/chat events when `isControlUiVisible` defaults to true.
  - confidence: 0.84
- **[medium] Fresh targets can inherit WebChat delivery context:** `src/gateway/server-methods/agent.ts:1016`
  - body: The Gateway agent handler persists request delivery hints before checking `deliver: true`, so the `webchat` internal channel used by `sessions_send` can become the target session's stored delivery context.
  - confidence: 0.82
- **[medium] A2A announce remains active for normal internal sends:** `src/agents/tools/sessions-send-tool.ts:455`
  - body: After reading the target reply, `sessions_send` still launches the A2A helper, which can ask for an announce reply and send it through Gateway `send` when a target resolves.
  - confidence: 0.79
