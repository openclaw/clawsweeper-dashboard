## Summary

Keep open. Current main still has the reported visible/internal boundary risk: `sessions_send` starts the target run with `deliver: false` but a WebChat internal channel, Gateway persists request delivery hints before the delivery check, A2A announce delivery remains active, and Control UI event handling defaults runs to visible unless explicitly hidden.

## What This Changes

Request that `sessions_send` from visible WebChat to an internal session remain internal, avoid WebChat delivery-context rebinding, and not announce raw target replies.

## Best Possible Solution

Define and enforce a strict internal boundary for internal or non-deliverable `sessions_send` target runs while preserving documented A2A announce delivery for explicit peer-session routes.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: call `sessions_send` from a WebChat-backed caller to a fresh internal target, then assert no visible chat/agent broadcast or Gateway `send` occurs and no WebChat delivery context is persisted.

## Solution Assessment

Unclear until maintainers confirm the intended boundary. The narrow maintainable fix would suppress UI-visible broadcasts and WebChat delivery-context seeding for internal/non-deliverable `sessions_send` target runs while leaving explicit peer A2A announce delivery intact.

## Security Review

Status: needs_attention

Summary: This issue reports an existing visible/internal information-boundary risk in the `sessions_send` WebChat and A2A announce path.

Concerns:

- **[medium] Internal replies can surface through visible UI events:** `src/gateway/server-chat.ts:594`
  - body: Gateway chat event handling defaults runs to Control-UI-visible and broadcasts assistant/chat events unless the run context explicitly hides them, which can expose internal target replies.
  - confidence: 0.84
- **[medium] Fresh targets can inherit WebChat delivery context:** `src/gateway/server-methods/agent.ts:1016`
  - body: The Gateway agent handler persists request delivery hints before checking `deliver: true`, so the internal WebChat channel used by `sessions_send` can become stored delivery context on a fresh target session.
  - confidence: 0.84
- **[medium] A2A announce remains active after internal sends:** `src/agents/tools/sessions-send-tool.ts:455`
  - body: After reading the target reply, `sessions_send` still starts the A2A flow, which can generate an announce and deliver it through Gateway `send` when a target resolves.
  - confidence: 0.8
