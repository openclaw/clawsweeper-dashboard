## Summary

Keep open. Current main still routes `sessions_send` through WebChat-flavored internal agent runs, keeps the A2A announce path enabled, and persists WebChat delivery hints onto fresh target sessions, so the reported visible/internal boundary issue is not resolved.

## What This Changes

The issue asks for `sessions_send` from visible WebChat sessions to preserve strict internal-only delivery when targeting internal sessions.

## Best Possible Solution

Make `sessions_send` preserve internal boundaries for internal or non-deliverable targets: avoid seeding fresh target sessions with WebChat delivery context, suppress visible Control UI/live chat output for internal handoff runs, and only run or expose announce delivery when an explicit deliverable target or documented peer handoff exists.

## Reproduction Assessment

Yes. A focused regression can invoke `sessions_send` from a WebChat-backed caller to a fresh internal target and assert no visible chat broadcast/send occurs and no WebChat delivery context is persisted on the target; the current source path supports that reproduction.

## Solution Assessment

Unclear as a product decision, but the current implementation is not the best strict-internal solution. The safer fix is a narrow boundary change around `sessions_send` internal/non-deliverable targets while preserving existing external-route and ACP peer-delivery behavior.

## Security Review

Status: needs_attention

Summary: This issue reports an existing information-boundary leak where internal session output can become visible through WebChat/A2A announce routing.

Concerns:

- **[medium] Internal replies can surface through visible WebChat:** `src/agents/tools/sessions-send-tool.ts:296`
  - body: `sessions_send` combines `deliver: false` with WebChat-channel target runs, visible-by-default Gateway event broadcasting, and a follow-up announce path, so a fresh internal target is not isolated from the visible WebChat surface.
  - confidence: 0.84
- **[medium] Fresh targets can inherit WebChat delivery context:** `src/gateway/server-methods/agent.ts:986`
  - body: The Gateway agent handler persists request channel hints into session delivery fields even for `deliver: false` runs, which can bind a fresh target session to WebChat context after an internal handoff.
  - confidence: 0.8
