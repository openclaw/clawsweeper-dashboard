## Summary

Keep open. Current main still lets `sessions_send` use WebChat-flavored non-deliverable target runs, leaves A2A announce active for normal peer sends, and does not mark these target runs hidden from Control UI live events; current docs also describe visible peer announce delivery, so the remaining boundary needs maintainer security/product judgment.

## What This Changes

The issue reports that `sessions_send` from a visible WebChat-backed session to an internal session can leak target replies through announce delivery and persist WebChat delivery context on the target session.

## Best Possible Solution

Define and enforce a strict internal boundary for internal or non-deliverable `sessions_send` target runs while preserving documented peer-session A2A announce delivery where it is explicitly intended.

## Reproduction Assessment

Yes. Source inspection on current main gives a focused reproduction path: call `sessions_send` from a WebChat-backed caller to a fresh internal target, then assert no visible chat broadcast/send occurs and no WebChat deliveryContext is persisted; I did not run a live test because this review is read-only.

## Solution Assessment

Unclear. Current docs describe normal peer `sessions_send` as A2A announce delivery to a visible channel/thread, so maintainers should decide the intended strict-internal boundary; the narrow safer path is to hide non-deliverable target runs from WebChat/Control UI and avoid seeding WebChat context unless explicit delivery is intended.

## Security Review

Status: needs_attention

Summary: This issue reports an existing visible/internal information-boundary risk in the WebChat and A2A announce path.

Concerns:

- **[medium] Internal target runs can surface through live WebChat events:** `src/gateway/server-chat.ts:594`
  - body: Gateway chat handling defaults runs to Control-UI-visible unless the run context explicitly opts out, while the `sessions_send` target path registers only the session key; assistant events from an internal target can therefore reach live WebChat/Control UI subscribers.
  - confidence: 0.84
- **[medium] Fresh targets can inherit WebChat delivery context:** `src/gateway/server-methods/agent.ts:1016`
  - body: The Gateway agent handler persists request delivery hints before checking `deliver: true`, so the `webchat` channel used by non-deliverable `sessions_send` target runs can become a fresh target session's stored delivery context.
  - confidence: 0.82
- **[medium] A2A announce can deliver target output visibly:** `src/agents/tools/sessions-send-tool.a2a.ts:156`
  - body: After the target reply, the A2A helper asks the target for an announce reply and sends it through Gateway `send` when an announce target resolves; that is documented for peer sends but risky for a strict internal delegation boundary.
  - confidence: 0.79
