## What This Changes

Prevent isolated cron announce delivery from mutating an active Telegram peer session transcript and tripping `EmbeddedAttemptSessionTakeoverError`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84603, or an equivalent narrow cron delivery-mirror change that preserves the direct channel send while suppressing or deferring transcript mutation for routed active peer sessions.
