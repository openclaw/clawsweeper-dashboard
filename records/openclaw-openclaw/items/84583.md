## What This Changes

The issue asks OpenClaw to prevent cron announce delivery from conflicting with an actively held Telegram peer session and raising `EmbeddedAttemptSessionTakeoverError`.

## Best Possible Solution

Keep the shipped lock-level delivery-mirror handling for this bug; review the open cron-specific suppression PR separately only if maintainers want to change transcript mirror behavior beyond preventing takeover errors.

## Reproduction Assessment

No current-main failing repro remains for the central cron `delivery-mirror` takeover path. Current source and regression coverage show a `delivery-mirror` append during prompt-lock release is accepted without setting takeover.

## Solution Assessment

Yes. The shipped lock-level fix preserves useful transcript mirrors while distinguishing known OpenClaw transcript-only appends from unowned external session mutations; the open suppression PR is a separate behavior choice, not required for this bug.
