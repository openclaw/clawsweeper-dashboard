## Label Justifications

- `P2`: The bug can abort in-flight channel turns under concurrent announce delivery, but the trigger is bounded to routed peer delivery races rather than a total runtime outage.
- `impact:session-state`: The failure is caused by a session transcript write racing the embedded prompt-release session fence.

## Summary

Keep open: current `main` and the latest release still contain the cron routed-peer transcript mirror path, and the issue is paired with open closing PRs rather than a merged fix.

## What This Changes

The issue reports that cron announce delivery can write into an active Telegram peer session during an embedded prompt-lock release, causing `EmbeddedAttemptSessionTakeoverError` and a failed user turn.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/94563 or an equivalent narrow cron mirror guard lands, preserving direct delivery plus main/threaded-main awareness while avoiding unsafe routed peer transcript writes.

## Reproduction Assessment

Yes. A high-confidence source path remains on current `main`: isolated cron direct delivery can resolve to a per-channel peer session, append a transcript mirror, and trip the embedded prompt-release session fence; recent comments also report the same signature on `v2026.6.8`.

## Solution Assessment

Yes for the issue's cron path. The best fix is a narrow cron dispatcher guard that skips unsafe routed-peer transcript mirrors while leaving direct delivery and main/threaded-main awareness intact; broader takeover variants should stay in their own trackers.
