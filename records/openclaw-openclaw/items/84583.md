## Summary

Keep open: current main still contains the routed-peer cron transcript mirror path, and https://github.com/openclaw/openclaw/pull/94563 plus https://github.com/openclaw/openclaw/pull/84603 are open candidate fixes rather than merged implementation.

## What This Changes

The issue reports cron or announce-mode delivery writing the active user session transcript while an embedded prompt lock is released, causing `EmbeddedAttemptSessionTakeoverError` and visible message failure.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/94563 or an equivalent replacement lands, with review confirming whether the newer `sessions_send` announce reports are covered or need a linked follow-up.

## Reproduction Assessment

Yes. Source inspection on current main and `v2026.6.8` shows cron direct delivery can resolve to a peer session and mirror into that transcript; recent issue comments also provide live logs with the same error signature.

## Solution Assessment

Yes for the cron-specific path: the narrow fix is to suppress or defer unsafe routed-peer transcript mirrors while preserving the direct channel send and intended main-session awareness. It is unclear whether that alone covers the newer `sessions_send` announce reports, so PR review should check that sibling surface before closing this issue.
