## Summary

Keep open. The PR targets a real source-level OpenResponses feature gap, but it still needs real behavior proof and has blocking security/API contract findings around raw reasoning visibility.

## What This Changes

The PR exposes embedded-agent reasoning on `/v1/responses` as non-stream reasoning output items and streaming `response.reasoning.*` events, with related agent/gateway tests.

## Best Possible Solution

Expose reasoning through a scoped, explicitly gated HTTP capture path, update the public Responses contract/docs/tests, and preserve plugin/channel isolation.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes, at source level: current main documents Responses `reasoning` as ignored, lacks reasoning SSE event types, and serializes assistant text without collecting reasoning. I did not run a live reasoning model.

## Solution Assessment

Is this the best way to solve the issue? No, not yet. Returning HTTP reasoning is plausible, but raw thinking should not be published through shared agent events without an explicit scoped gate and documented visibility contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Gate raw reasoning before shared event publishing:** `src/agents/pi-embedded-subscribe.ts:592-596`
  - body: This emits `thinking-raw` through `emitAgentEvent` before the reasoning visibility check. `server-chat` suppresses websocket/node delivery, but plugin host-hook subscriptions without a `streams` filter still receive shared agent events, so hidden model reasoning can reach plugins; keep HTTP capture scoped internally or require an explicit opt-in before publishing raw thinking.
  - confidence: 0.9
- **[P2] Document the new Responses reasoning contract:** `src/gateway/open-responses.schema.ts:351-360`
  - body: This adds public `response.reasoning.*` events and reasoning output items, but the OpenResponses docs still say `reasoning` is accepted and ignored. Clients cannot tell when reasoning is emitted, whether it is gated, or how to consume the new events.
  - confidence: 0.84
