## Summary

Keep open: current main still lacks `/v1/responses` reasoning output, and this PR still has security/API blockers plus missing real behavior proof before it is mergeable.

## What This Changes

The PR adds raw embedded-agent reasoning capture and exposes it as OpenResponses reasoning output items plus streaming `response.reasoning.*` events from `/v1/responses`.

## Best Possible Solution

Expose reasoning through a scoped, explicitly gated HTTP capture path, update the public Responses docs/schema/tests, and preserve plugin/channel isolation.

## Reproduction Assessment

Yes, at source level for the missing API surface. Current main documents Responses `reasoning` as ignored, lacks reasoning stream event types, and serializes assistant text only; I did not run a live reasoning model.

## Solution Assessment

No, not yet. Returning HTTP reasoning is a plausible direction, but publishing raw thinking through shared agent events and leaving the public contract/gating undocumented is not the narrowest maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Gate raw reasoning before shared event publishing:** `src/agents/pi-embedded-subscribe.ts:592-596`
