Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Gate raw reasoning on explicit visibility:** `src/agents/pi-embedded-subscribe.ts:592-595`
  - body: This emits `thinking-raw` before checking `state.streamReasoning`, and the HTTP handler now returns that raw content through `/v1/responses`. A default-off or `/reasoning off` run would still expose internal reasoning to any HTTP caller, despite the current docs treating reasoning visibility as an explicit setting and warning that it can leak internal reasoning.
  - confidence: 0.89
- **[P2] Introduce the reasoning item before streaming deltas:** `src/gateway/openresponses-http.ts:1058-1062`
  - body: The stream sends `response.reasoning.delta` for `reasoningItemId`, but the only `response.output_item.added` sent before this is the assistant message item; the reasoning item is materialized only in the final `response.completed` payload. Incremental clients that reconcile by output-item lifecycle have no item/index to attach these deltas to, so reasoning can be dropped or mis-associated.
  - confidence: 0.82
