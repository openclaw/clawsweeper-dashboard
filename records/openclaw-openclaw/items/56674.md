## Summary

Keep open: this is a plausible Gateway feature, but the current branch still exposes raw reasoning through shared event plumbing without a clear visibility gate, lacks docs for the new public Responses contract, and has no real after-fix behavior proof.

## What This Changes

This PR captures embedded-agent thinking and serializes it as `/v1/responses` reasoning output items and `response.reasoning.*` SSE events, with agent/gateway tests and server-chat suppression for the new raw stream.

## Best Possible Solution

Expose HTTP reasoning through an explicit, scoped visibility gate, keep raw capture out of plugin-visible shared events, document the Responses contract, and require live redacted `/v1/responses` proof before merge.

## Reproduction Assessment

Yes, at source level. Current main documents Responses `reasoning` as ignored, lacks reasoning SSE events, and serializes assistant text without collecting reasoning; I did not run a live reasoning model in this read-only review.

## Solution Assessment

No, not yet. Returning structured HTTP reasoning is plausible, but this implementation needs an explicit visibility/security gate, a plugin-safe capture path, docs, and real behavior proof before it is the maintainable solution.
