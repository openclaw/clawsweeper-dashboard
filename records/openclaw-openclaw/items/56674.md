## Summary

Keep open. The PR targets a real OpenResponses gap, but the proposed raw reasoning capture still needs maintainer handling because it publishes hidden thinking on the shared agent-event bus before the reasoning visibility contract is settled.

## What This Changes

The PR adds raw embedded-agent thinking capture and maps it to OpenResponses reasoning output items and SSE events, with gateway and agent regression tests.

## Best Possible Solution

Ship OpenResponses reasoning through a scoped internal capture path or a documented explicit visibility opt-in, with docs, changelog, and tests covering reasoning-off, authorized opt-in, streaming, tool-call ordering, and plugin subscription exposure.

## Reproduction Assessment

Yes. Source inspection reproduces the issue: current main omits reasoning output, while the PR diff emits `thinking-raw` before the current `state.streamReasoning` gate and plugin subscriptions can receive that stream.

## Solution Assessment

No. Exposing reasoning on the HTTP API is the right direction, but publishing hidden raw thinking on the global agent-event bus is not the safest implementation; a scoped capture channel or explicit authorization gate should come first.
