- **[P1] Gate raw reasoning before returning it over HTTP:** `src/gateway/openresponses-http.ts:815-820`
  - body: This adds collected thinking to the HTTP response whenever any `thinking-raw` event appears, even though current OpenClaw docs still treat the request `reasoning` field as ignored and the external spec treats raw reasoning as optional for safety/privacy. Please preserve default-off behavior with an explicit maintainer-approved visibility gate before emitting raw reasoning. ([openresponses.org](https://www.openresponses.org/specification))
  - confidence: 0.9
- **[P1] Emit every pending client tool call:** `src/gateway/openresponses-http.ts:754-755`
  - body: This tool-call branch still uses only `pendingToolCalls[0]`; current main emits one `function_call` item per pending call. A model that returns multiple client tool calls would expose only the first call to `/v1/responses` clients, so keep the current-main loop while inserting reasoning items.
  - confidence: 0.88
- **[P2] Use the OpenResponses item lifecycle for reasoning streams:** `src/gateway/openresponses-http.ts:1058-1062`
  - body: The new stream emits `response.reasoning.delta` for a reasoning item that was never announced with `response.output_item.added` or a content part, and it omits output/content indexes. Clients reconciling the stream by item lifecycle cannot attach those deltas to the final reasoning item; model reasoning should follow the normal streamable item/content-part pattern or be a documented OpenClaw extension. ([openresponses.org](https://www.openresponses.org/specification))
  - confidence: 0.82
