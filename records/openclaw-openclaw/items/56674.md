## Summary

Keep open: current main still lacks OpenResponses reasoning output, but this PR has a blocking privacy/security issue and lacks contributor real-behavior proof before merge.

## What This Changes

The branch adds `/v1/responses` reasoning output by emitting embedded-agent `thinking-raw` events and collecting them into non-streaming reasoning items and streaming `response.reasoning.*` SSE events, with gateway and agent regression tests.

## Best Possible Solution

Expose reasoning through a scoped HTTP capture path or explicit visibility opt-in that does not publish hidden raw thinking to shared plugin events, then update the Responses docs and focused gateway/agent/plugin-subscription tests.

## Reproduction Assessment

Yes. Source inspection shows current main documents and implements `/v1/responses` as assistant/tool-output-only for reasoning, and the PR diff clearly changes the path by publishing `thinking-raw` events for HTTP capture.

## Solution Assessment

No. Returning reasoning on the HTTP API is the right direction, but this implementation routes hidden raw thinking through the shared agent-event/plugin subscription bus before the existing visibility gate; a scoped capture path or explicit opt-in is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Gate raw reasoning before shared event publishing:** `src/agents/pi-embedded-subscribe.ts:592-596`
  - body: This emits `thinking-raw` through `emitAgentEvent` before `state.streamReasoning` is checked. `emitAgentEvent` notifies the shared listener set, and the plugin runtime forwards those events to agent-event subscriptions unless their optional stream filter excludes them, so plugins can observe hidden raw reasoning even when reasoning visibility is off. Keep HTTP capture scoped/internal or require an explicit visibility opt-in before publishing raw reasoning on the shared bus.
  - confidence: 0.88
