## Summary

Keep this PR open. Current main still does not expose OpenResponses reasoning output, so the contribution targets a real gap, but the proposed implementation has a concrete privacy/API-boundary blocker: it publishes raw thinking before the existing reasoning visibility gate, and plugin agent-event subscriptions can still observe that stream even after the PR suppresses WebSocket/session broadcasts.

## What This Changes

The PR adds raw thinking capture from embedded agent runs and returns reasoning output items and reasoning SSE events from `/v1/responses`, with tests and server-chat suppression for the new `thinking-raw` stream.

## Best Possible Solution

Ship OpenResponses reasoning output behind the existing reasoning visibility authorization or a documented HTTP-specific opt-in, keep raw thinking off broad plugin/session event streams by default, update docs and the changelog, and cover reasoning-off, authorized opt-in, streaming, tool-call ordering, and plugin-subscription exposure in tests.

## Reproduction Assessment

Yes. Source inspection is enough: current main only emits formatted `thinking` when `state.streamReasoning` is enabled, while the PR diff emits `thinking-raw` before that check and the plugin bridge dispatches agent events to matching or broad plugin subscriptions.

## Solution Assessment

No. Exposing reasoning through `/v1/responses` is a maintainable direction, but this implementation should not publish raw thinking on the global agent-event bus before visibility authorization; a scoped internal capture path or explicit gate is safer.
