## Summary

Keep open: current main still lacks OpenResponses reasoning output, but this PR currently publishes raw thinking through the shared agent-event surface before the reasoning visibility gate, which can expose hidden reasoning to plugins; it also lacks after-fix real behavior proof.

## What This Changes

The PR captures embedded-agent thinking and returns it from `/v1/responses` as non-streaming reasoning output items and streaming `response.reasoning.*` SSE events, with agent/gateway regression tests.

## Best Possible Solution

Expose OpenResponses reasoning through a scoped HTTP capture path or explicit visibility opt-in that does not publish hidden raw thinking to global plugin events, then update docs and focused gateway/agent/plugin-subscription tests.

## Reproduction Assessment

Yes. Source inspection shows current main documents and implements `/v1/responses` as assistant/tool-output-only for reasoning, and the PR diff clearly changes the path by publishing `thinking-raw` events for HTTP capture.

## Solution Assessment

No. Returning reasoning on the HTTP API is the right direction, but this implementation routes hidden raw thinking through the shared agent-event/plugin subscription bus before the existing visibility gate; a scoped capture path or explicit opt-in is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Gate raw reasoning before shared event publishing:** `src/agents/pi-embedded-subscribe.ts:592-596`
  - body: This emits `thinking-raw` through `emitAgentEvent` before the `state.streamReasoning` check. `emitAgentEvent` notifies the global agent-event listeners, and the plugin runtime dispatches events to subscriptions unless a stream filter excludes them, so plugins can observe raw hidden thinking when reasoning visibility is off. Keep HTTP capture scoped/internal or require an explicit visibility opt-in before publishing raw reasoning on the shared bus.
  - confidence: 0.88
