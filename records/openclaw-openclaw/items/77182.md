## Summary

Keep open: the diagnostic direction is useful, but the submitted patch has two merge blockers: it adds an unintended live diagnostic artifact and writes raw abort detail into a quoted log field without escaping.

## What This Changes

The PR propagates abort reasons through CLI live/non-live abort errors and chat aborts, logs a new detail field, and adds focused abort-reason tests.

## Best Possible Solution

Keep the abort-reason propagation, remove the local diagnostic artifact, escape or structure the new log detail with regression coverage, and validate the changed lanes before merge.

## Reproduction Assessment

Yes. Source inspection on current main shows fixed-message CLI AbortErrors, live-session failure logs that only include the error kind, and chat aborts that do not propagate `stopReason`.

## Solution Assessment

No, not as submitted. Propagating abort reasons is the narrow maintainable direction, but the mergeable fix should omit the diagnostic artifact and encode the log detail safely.
