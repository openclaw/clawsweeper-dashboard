## Summary

Keep this PR open: the abort-source diagnostic direction is useful and current main still lacks it, but the submitted patch has two concrete merge blockers plus missing after-fix real behavior proof.

## What This Changes

The PR adds abort reason formatting/logging for CLI live and non-live paths, propagates chat.abort stop reasons into AbortSignal.reason, adds abort-reason unit tests, and adds a root diagnostic write-up.

## Best Possible Solution

Keep the abort-reason propagation, remove DIAGNOSTIC.md, escape or structure the abort detail field with regression coverage, and add redacted real runtime proof before merge.

## Reproduction Assessment

Yes for the source-level diagnostic gap: current main creates fixed-message CLI AbortErrors and chat aborts without a signal reason, so the existing log path cannot attribute the trigger. I did not run a live gateway reproduction in this read-only review.

## Solution Assessment

No, not as submitted. Propagating abort reasons is the narrow maintainable direction, but the mergeable patch should drop the diagnostic artifact, encode the log detail safely, and include real after-fix proof.
