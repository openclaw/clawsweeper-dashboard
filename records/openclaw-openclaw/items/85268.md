## Summary

Keep open: current main and v2026.6.8 contain relevant Windows exec hardening, but neither proves this newer Windows 11 / Node 24.14.1 report where even `echo hello` and `node -v` hang with no output. The older Windows exec hang reports overlap, but they were narrower CLI-subcommand or supervisor-settlement failures and are not safe duplicates without fresh native Windows proof.

## What This Changes

The issue reports that all Windows exec tool commands spawn a child process but produce no output and hang until timeout when the gateway is started from PowerShell.

## Best Possible Solution

Reproduce on native Windows with a PowerShell-started gateway, trace whether the failing command uses the supervisor path or direct helper path, then patch that exact process path with focused regression coverage and real `echo hello` plus `node -v` proof.
