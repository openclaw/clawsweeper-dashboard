## Summary

Keep open: current main still has the source-visible active-requester path where a non-streaming embedded Pi run returns undelivered before the requester-agent handoff can run. The issue is paired with an open closing PR, and the newest issue comments report the same symptom on v2026.5.7, so it should stay open until that fix or an equivalent lands.

## What This Changes

Subagent completion announces should remain visible for push channels when the parent requester session is active but not consuming embedded Pi messages.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79059 or an equivalent narrow fix that lets active but non-consuming requester completions reach the requester-agent handoff without restoring raw `sendMessage` delivery.

## Reproduction Assessment

Yes, at source level. Current main can set `expectsCompletionMessage`, resolve an active requester session, make `queueEmbeddedPiMessage` return false for a non-streaming embedded run, and hit the early return before `callGateway`.

## Solution Assessment

Yes for the narrow fallthrough direction in the linked PR; it preserves the requester-agent delivery contract from https://github.com/openclaw/openclaw/pull/78700. A raw `sendMessage` fallback is not the best fix because it would bypass that contract.
