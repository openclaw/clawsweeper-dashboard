## Summary

Keep open: current main still has a source-visible active-requester path where a non-streaming embedded run returns undelivered before the direct requester-agent handoff below can run. The reported symptom is credible, but the requested `sendMessage` fallback overlaps with a recently removed raw-output bypass, so maintainer review should choose the contract-preserving fix.

## What This Changes

The issue asks OpenClaw to deliver subagent completion announcements when the parent embedded Pi run is active but not consuming messages, instead of leaving push-channel users waiting for another message.

## Best Possible Solution

Preserve push-based subagent completion by making active-but-non-consuming requester sessions reach the requester-agent handoff or an equivalent visible delivery path without restoring raw child-output direct sends.

## Reproduction Assessment

Yes, at source level. A focused seam can set `expectsCompletionMessage` true, return an active requester session id, make `queueEmbeddedPiMessage` return false, and observe current main returning undelivered before the later `callGateway` handoff path runs.

## Solution Assessment

Unclear as written. Letting active-but-idle completions reach the requester-agent handoff is likely the right direction, but adding a raw `sendMessage` fallback needs maintainer judgment because recent merged work intentionally removed that bypass.
