## Summary

Keep open: current main still has the active-requester branch that returns undelivered before the requester-agent handoff can run, and an open fixing PR already references this issue but has not merged.

## What This Changes

The issue asks OpenClaw to stop dropping subagent completion announcements when a parent requester session is active but its embedded Pi run cannot be woken.

## Best Possible Solution

Keep this issue open until the open fixing PR or a successor removes the active-requester dead end while preserving the requester-agent handoff contract and proves the push-channel behavior.

## Reproduction Assessment

Yes, at source level: set `expectsCompletionMessage`, return an active requester session id, make `queueEmbeddedPiMessageWithOutcome` fail with `not_streaming`, and current main returns undelivered before `callGateway` can run. The issue discussion also includes recent v2026.5.7 field evidence, but I did not run a live Telegram gateway here.

## Solution Assessment

Yes as a direction: the revised open PR’s fallthrough approach is narrower than restoring raw `sendMessage` delivery and matches the requester-agent handoff contract. The issue should stay open until that PR is reviewed, proven, and merged or replaced.
