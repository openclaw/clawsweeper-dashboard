## Summary

Keep open: current main still returns from the active-requester wake-failure branch before the requester-agent `callGateway` handoff can run, and the open fixing PR at https://github.com/openclaw/openclaw/pull/79059 references this issue with closing syntax.

## What This Changes

The issue reports that subagent completion announcements can be dropped when the requester session is active but the embedded Pi run is not consuming queued messages.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/79059 for the narrow fallthrough fix, then track the durable retry-exhaustion queue proposal separately if maintainers agree it is still needed.

## Reproduction Assessment

Yes, at source level. Set `expectsCompletionMessage` true, resolve an active requester session, make `queueEmbeddedPiMessage` return false for a non-streaming embedded run, and current main returns before the requester-agent `callGateway` handoff.

## Solution Assessment

Yes for the immediate bug: the narrowest maintainable fix is to let the active-but-non-consuming branch fall through to the existing requester-agent handoff without restoring raw external sends. The durable delivery-queue idea in the latest comment is broader follow-up work, not a prerequisite for this fix.
