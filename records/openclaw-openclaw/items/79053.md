## Summary

Keep open: current main still has the active-requester early return before the requester-agent handoff, and the linked fixing PR remains open and needs real behavior proof before merge. Recent issue comments also report the symptom on v2026.5.7, so this is not implemented, stale, or safely superseded.

## What This Changes

The issue asks OpenClaw to stop dropping subagent completion announcements when the parent requester session is active but not consuming embedded Pi messages.

## Best Possible Solution

Land a narrow requester-agent handoff fix, likely through https://github.com/openclaw/openclaw/pull/79059 after proof clears, so active-but-idle requester sessions reach callGateway or queue retry without raw child-output sends.

## Reproduction Assessment

Yes, at source level. Set expectsCompletionMessage true, make the requester active with a session id, have queueEmbeddedPiMessage return false for a non-streaming run, and current main returns before the callGateway requester-agent handoff.

## Solution Assessment

Yes, the narrow fallthrough direction is the best apparent fix because it preserves the requester-agent delivery contract instead of restoring a raw sendMessage fallback. It still needs the linked PR to provide real behavior proof or a maintainer proof override before merge.
