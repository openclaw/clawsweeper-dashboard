## Summary

Keep open: current main still has the active-requester early return, and an open linked fix PR already targets the narrow fallthrough path, so this issue should stay open until that PR lands or is replaced.

## What This Changes

Ensure subagent completion announcements for active but non-consuming parent sessions reach the requester-agent handoff instead of disappearing until a later user turn.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/79059 with the contract-preserving fallthrough fix, then track any separate durable retry or wrong-session routing behavior independently.

## Reproduction Assessment

Yes, at source level. Set expectsCompletionMessage true, resolve an active requester session id, make queueEmbeddedPiMessage return false for a non-streaming embedded run, and current main returns the active-requester error before callGateway can run.

## Solution Assessment

The original raw sendMessage fallback is not the best fit after the recent delivery-contract cleanup. The safer narrow fix is to let the active-but-non-consuming path reach the existing requester-agent handoff, which the open linked PR now attempts.
