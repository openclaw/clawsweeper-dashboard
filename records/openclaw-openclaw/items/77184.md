## What This Changes

The PR adds three type-only status payload re-exports from `plugin-sdk/command-status`, documents that subpath, and adds a type-import regression test.

## Best Possible Solution

Land the narrow type-only export only after refreshing the generated SDK API baseline and making an explicit maintainer call on whether full `StatusSummary` is the intended public contract; keep `ModelCatalogEntry` on `plugin-sdk/agent-runtime`.

## Reproduction Assessment

Yes, source-level. Current main defines the internal status payload types in `src/commands/status.types.ts`, while `src/plugin-sdk/command-status.ts` does not export them through the public SDK subpath.

## Solution Assessment

Mostly yes, but not merge-ready. Re-exporting type-only symbols from the existing narrow `command-status` subpath is the smallest plausible fix; if full `StatusSummary` is too broad, the safer alternative is a narrower public status DTO contract.
