## Summary

Keep open: the branch still contains potentially useful Microsoft Teams history/debounce, mention, and attachment work, but it is not merge-ready because real Teams after-fix proof is missing and the current patch would regress thread-routing semantics that are already on main.

## What This Changes

The PR changes the Microsoft Teams plugin to add thread-scoped sessions/history/debounce, proactive thread targeting, bot-mention-only stripping, Graph attachment fallback tests, and incidental formatting churn across other plugins and core files.

## Best Possible Solution

Rebase or replace this with a narrow Microsoft Teams-only patch on current main that preserves shipped thread-root and idempotent-key routing while adding effective-policy-aware thread-scoped history/debounce and any retained mention or attachment fixes with focused tests and real Teams proof.

## Reproduction Assessment

Yes, source-level for the remaining concern: current main still keys pending history and inbound debounce by the normalized Teams channel id, so rapid same-sender messages or pending history can mix across threads. I did not run a live Teams tenant in this read-only review.

## Solution Assessment

No. The direction is useful, but the current branch is not the best fix because it uses a global-only thread-binding gate and `activity.replyToId`-derived thread keys instead of current main's effective policy expectations and Teams thread-root behavior.
