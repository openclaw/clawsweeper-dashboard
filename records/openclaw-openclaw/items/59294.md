## Summary

Keep open: the Teams bugfix direction is useful, but this external PR still lacks after-fix real Teams proof, is conflicting/overlapping with current main, and has two P2 correctness problems around thread-binding policy and thread identity.

## What This Changes

The PR changes the Microsoft Teams plugin to add per-thread session/history/debounce keys, outbound reply targeting, mention preservation, Graph attachment fallback tweaks, Teams tests, and broad formatting-only edits across other plugin/core files.

## Best Possible Solution

Rebase or replace this with a narrow Microsoft Teams-only patch that reuses current main's threadId/idempotent routing and effective thread-binding policy, adds thread-aware history/debounce plus any retained mention/media fixes, and includes focused tests with redacted real Teams proof.

## Reproduction Assessment

Yes, source-level: current main still computes pending history and inbound debounce from the normalized Teams channel id, so same-sender messages or pending history can still cross threads. I did not run a live Teams tenant in this read-only review.

## Solution Assessment

No. The direction is useful, but the current PR is not the best fix because it bypasses the effective thread-binding policy, uses replyToId instead of the Teams thread root, and needs a rebase around current main's shipped thread-routing behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

- **[P2] Honor the effective thread-binding policy:** `extensions/msteams/src/monitor-handler/message-handler.ts:394-395`
  - body: The new gate only checks `cfg.session?.threadBindings?.enabled === true`, but current policy supports channel/session settings and defaults thread bindings to enabled. Default Teams installs, or configs using `channels.msteams.threadBindings.enabled`, would still take the unthreaded session/history/debounce path this PR is meant to fix.
  - confidence: 0.87
- **[P2] Use the Teams thread root for new keys:** `extensions/msteams/src/monitor-handler/message-handler.ts:395-396`

For deep replies, `replyToId` can point at a nested parent and split one visible Teams thread across multiple OpenClaw session keys.
  - confidence: 0.78
