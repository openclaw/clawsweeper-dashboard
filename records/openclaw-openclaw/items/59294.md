## Summary

Keep open for maintainer handling: current main has shipped the core Teams thread-session and proactive-routing fixes, but this PR still contains potentially useful, unmerged work around thread-scoped history/debounce, mention preservation, and attachment fallback, while the branch itself is not currently mergeable as-is and lacks real after-fix Teams proof.

## What This Changes

The PR changes the Microsoft Teams plugin to isolate channel-thread sessions/history/debounce, target outbound thread replies, preserve non-bot mentions, broaden attachment fallback, add tests, and includes unrelated formatting churn across other plugins/core files.

## Best Possible Solution

Replace or rebase this into a narrow Microsoft Teams-only patch that preserves current main's thread-root/idempotent routing while adding effective-policy-aware thread-scoped history/debounce and any retained mention or attachment fixes with focused tests and real Teams proof.

## Reproduction Assessment

Yes, source-level: current main still computes pending history and inbound debounce with the normalized Teams channel id, so those paths can mix same-sender messages or history across threads. I did not run a live Teams tenant in this read-only review.

## Solution Assessment

No. The direction is useful, but this PR is not the best current solution because it bypasses effective thread-binding policy, uses `replyToId` instead of the Teams thread root, and needs a rebase around current main's shipped thread-routing behavior.
