## Summary

Keep open: current main still lacks the requested pre-tool-call redirect/model/session routing contract, and the related PRs either remain unmerged or cover only adjacent context fields.

## What This Changes

The issue requests a pre-tool-call hook that can inspect preceding assistant output and redirect a pending tool call to another model or session.

## Best Possible Solution

Define an additive typed plugin/runtime routing contract with explicit inspectable assistant context, redirect authority, audit/approval behavior, failure handling, and session/auth/provider boundary rules.

## Reproduction Assessment

Not applicable as a feature/API request. Source inspection on current main gives high confidence that the requested preceding-output and redirect/model/session hook fields and runtime behavior are absent.

## Solution Assessment

Unclear until maintainers choose the API and security model. Extending before_tool_call or adding a parallel routing hook could both work, but the existing prompt-time model hook and open split-routing PR do not satisfy the requested plugin redirect semantics by themselves.
