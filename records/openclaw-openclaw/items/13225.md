## Summary

Keep open. Current main still lacks the requested pre-tool-call model/session redirect contract: `before_tool_call` can rewrite params, block, or require approval, while `before_model_resolve` runs earlier with only prompt/attachment context. The issue discussion and related open PRs show useful partial work, but none has merged a plugin-facing redirect API.

## What This Changes

Request a plugin/runtime hook before tool execution that can inspect preceding assistant context and route a pending tool call to a different model or session.

## Best Possible Solution

Keep this open as a plugin/runtime API design request. The desired end state is an additive typed pre-tool routing contract that defines what assistant context a plugin may inspect and how a pending tool call can be redirected to another provider/model/session, while preserving the existing params rewrite, block, and approval behavior for narrower cases like per-agent TTS argument injection.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. The current-main verification path is high-confidence source/docs inspection showing the existing hook can only rewrite, block, or approve tool calls, and the existing model hook runs before session messages and tool calls exist.

## Solution Assessment

Unclear for the final API shape. Current main is not sufficient, and the safer direction is maintainer-designed additive routing semantics rather than treating partial context PRs or the existing pre-session model hook as a complete fix.
