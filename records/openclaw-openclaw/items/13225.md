## Summary

Keep open: current main still lacks a typed pre-tool-call model/session redirect contract, and the related open work covers context fields or config-level routing without superseding the requested plugin redirect API.

## What This Changes

The issue requests a pre-tool-call hook that can inspect preceding assistant output and redirect a pending tool call to another model or session.

## Best Possible Solution

Define an additive typed pre-tool routing API with inspectable assistant context and explicit redirect semantics while preserving existing params rewrite, block, and approval behavior.

## Reproduction Assessment

Not applicable as a feature request. Source inspection of current main gives high confidence that the requested preceding-output and redirect/model/session hook fields and runtime routing behavior are absent.

## Solution Assessment

Unclear until maintainers choose the API shape. Either extending before_tool_call or adding a separate pre-tool routing hook could work, but prompt-time model hooks and config-level split routing do not satisfy the requested plugin redirect semantics.
