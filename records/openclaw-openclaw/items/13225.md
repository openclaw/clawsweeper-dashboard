## Summary

Keep open: current main and the latest release still lack a `before_tool_call` contract that can inspect pending assistant context and redirect the pending tool call to another model or session; the adjacent open PRs cover context fields or split model routing, not this plugin redirect API.

## What This Changes

Request an additive pre-tool-call hook capability that exposes preceding assistant context and can redirect a pending tool call to another model or session.

## Best Possible Solution

Define an additive typed plugin/runtime routing contract with inspectable assistant context, redirect authority, audit/approval behavior, failure handling, and session/auth/provider boundary rules while preserving existing params rewrite, block, and approval behavior.

## Solution Assessment

Unclear until maintainers choose the API and security model. The current hook system solves block, approval, params rewrite, and pre-turn model selection, but not pending tool-call redirection.
