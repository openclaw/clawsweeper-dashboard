## Summary

Keep open: current main still lacks a typed pre-tool-call redirect/model/session routing contract; the shipped hook surface handles params, blocking, and approval only, while related PRs have not shipped this plugin API.

## What This Changes

The issue requests a pre-tool-call hook that can inspect preceding model output and redirect a pending tool call to another model or session.

## Best Possible Solution

Define an additive plugin/runtime API for pre-tool routing that preserves existing params rewrite/block/approval behavior while specifying visible assistant context, routing authority, auditing, permissions, and failure behavior.

## Reproduction Assessment

Not applicable as a feature request. Source inspection of current hook types, runtime handling, merger behavior, and docs gives high confidence that current main lacks the requested redirect and preceding-output behavior.

## Solution Assessment

Unclear until maintainers choose the API shape. Extending `before_tool_call` or adding a separate pre-tool routing hook could both work, but either path needs product and security semantics first.
