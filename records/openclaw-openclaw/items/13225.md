## Summary

Keep open: current main still lacks a typed pre-tool-call model/session redirect contract. Existing hooks can rewrite params, block, or require approval, while related open PRs cover adjacent context or config-level split routing without superseding the requested plugin redirect semantics.

## What This Changes

Request an additive pre-tool-call hook contract that exposes preceding assistant output and can redirect a pending tool call to another model or session.

## Best Possible Solution

Define an additive typed pre-tool routing API with inspectable assistant context and explicit redirect semantics, while preserving existing params rewrite, block, and approval behavior.

## Reproduction Assessment

Not applicable as a feature request. Source inspection of current main gives high confidence that the requested preceding-output and redirect/model/session hook fields and runtime routing behavior are absent.

## Solution Assessment

Unclear until maintainers choose the API shape. Either extending `before_tool_call` or adding a separate routing hook could work, but redirect authority across provider/model/session boundaries needs product and security design first.
