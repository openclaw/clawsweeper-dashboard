## Summary

Keep open: current main still lacks a typed pre-tool-call redirect/model/session routing contract, and the related items either cover narrower context fields or remain unmerged implementation candidates rather than a shipped replacement.

## What This Changes

The issue requests a pre-tool-call hook that can inspect preceding model output and redirect the pending tool call to another model or session.

## Best Possible Solution

Define an additive typed pre-tool routing API with inspectable assistant context and explicit redirect semantics while preserving existing params rewrite, block, and approval behavior.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high confidence that the requested preceding-output and redirect/model/session hook fields and runtime routing behavior are absent.

## Solution Assessment

Unclear until maintainers choose the API shape. Existing prompt-time model hooks and the open config-level split-routing PR do not satisfy the requested plugin redirect semantics by themselves.
