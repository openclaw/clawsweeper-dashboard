## Summary

Keep open: current main still lacks a pre-tool-call model/session routing contract. The existing hook can rewrite params, block execution, or require approval, but it has no typed assistant-output inspection or redirect result, and the closest related PR is still open and unmerged.

## What This Changes

Request an additive `before_tool_call` capability that can inspect preceding assistant output and redirect a pending tool call to another model or session.

## Best Possible Solution

Define an additive typed plugin/runtime API for pre-tool routing, preserving the existing params rewrite/block/approval behavior while specifying what assistant context is visible and how model/session redirection is authorized and reported.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main lacks the requested redirect/preceding-output hook fields and routing behavior.

## Solution Assessment

Unclear until maintainers choose the API shape. Extending `before_tool_call` or adding a separate pre-tool routing hook could both work, but either needs a product/security contract before implementation.
