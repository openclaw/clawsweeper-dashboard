## Summary

Keep open: current main and v2026.5.20 still do not provide a pre-tool-call hook that can inspect pending assistant output and redirect the pending tool call to another model or session. The adjacent open PRs cover related context or split model routing, but this issue still needs a plugin/runtime API and security design decision.

## What This Changes

Request an additive pre-tool-call plugin/runtime hook capability that exposes preceding model context and can route a pending tool call to a different model or session.

## Best Possible Solution

Define an additive typed plugin/runtime routing contract with inspectable assistant context, redirect authority, audit/approval behavior, failure handling, and provider/session/auth boundary rules while preserving the existing params rewrite, block, and approval behavior.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/API request. Source inspection of current main and v2026.5.20 gives high confidence that the requested preceding-output and redirect/model/session behavior is absent.

## Solution Assessment

Unclear until maintainers choose the API and security model. The existing hook system covers block, approval, params rewrite, and pre-turn model selection, but not pending tool-call redirection.
