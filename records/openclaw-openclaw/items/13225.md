## Summary

Keep open: current main still lacks a `before_tool_call` action that can redirect a pending tool call to another model or session, and the related work covers adjacent context or split-model routing rather than this plugin redirect contract.

## What This Changes

Requests an additive pre-tool-call hook capability to inspect preceding model output and route the pending tool call to a different model or session.

## Best Possible Solution

Define an additive typed plugin/runtime routing contract with inspectable assistant context, redirect authority, audit/approval behavior, failure handling, and session/auth/provider boundary rules.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/API request. Source inspection on current main gives high confidence that the requested preceding-output and redirect/model/session hook behavior is absent.

## Solution Assessment

Unclear until maintainers choose the API and security model. The current hook system solves block, approval, params rewrite, and pre-turn model selection, but not pending tool-call redirection.
