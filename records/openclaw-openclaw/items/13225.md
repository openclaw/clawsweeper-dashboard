## Summary

Keep open: current main and v2026.6.6 still expose before_tool_call as a params/block/approval hook, not a model or session redirect point, and the remaining work is a security-sensitive plugin/runtime API design decision.

## What This Changes

Request a pre-tool-call plugin/runtime hook that can inspect pending tool-call context and redirect that call to another model or session before execution.

## Best Possible Solution

Define an additive typed pre-tool or continuation routing contract after maintainer product and security review, preserving the existing params/block/approval behavior for policy and argument-injection plugins.

## Reproduction Assessment

Not applicable: this is a feature/API request rather than a broken existing behavior report. Source and release inspection give high confidence that current main and v2026.6.6 lack the requested redirect contract.

## Solution Assessment

No for the literal sketched before_tool_call extension as-is. The maintainable solution is a typed routing phase with explicit audit, permission, provider/auth, fallback, context-budget, and session-state semantics.
