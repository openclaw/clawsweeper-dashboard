## Summary

Codex review should keep this open: current main still lacks a pre-tool-call redirect contract, and the related context-field and split-routing PRs do not supersede the requested plugin hook semantics.

## What This Changes

The issue requests a pre-tool-call hook that can inspect preceding assistant context and redirect an imminent tool call to another model or session.

## Best Possible Solution

Define an additive typed plugin/runtime API, either by extending before_tool_call or adding a separate pre-tool routing hook, that exposes the needed assistant/tool-call context and specifies model/provider/session redirect behavior while preserving existing params/block/approval semantics.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence that the requested capability is absent: current types, docs, and runtime handling only support params rewrite, block, and approval.

## Solution Assessment

Unclear as an implementation choice until maintainers settle the API semantics. A narrow additive hook contract looks like the maintainable direction, but model/session redirect behavior needs explicit product and safety design.
