## Summary

Current main still lacks mid-turn model or session routing, and the closest open PR does not settle the required authentication, context-budget, fallback, session, and security semantics. Keep this issue open for maintainer API design rather than treating it as implemented or sending it to an autonomous fix lane.

## What This Changes

Requests a pre-tool-call plugin hook that can inspect preceding assistant context and redirect a pending tool call to another model or session.

## Best Possible Solution

Design an additive, permission-gated routing hook at the model-turn boundary that returns a typed routing decision while core resolves and validates provider, model, authentication, context budget, session continuity, permissions, fallbacks, and audit events.

## Reproduction Assessment

Not applicable; current main consistently implements the documented parameters/block/approval hook, and this issue requests a new routing capability rather than reporting a regression.

## Solution Assessment

No, the proposed raw `before_tool_call.redirect` shape is not the best final design because routing belongs to a turn-level owner boundary and must reuse canonical provider, authentication, context, permission, fallback, and transcript machinery.
