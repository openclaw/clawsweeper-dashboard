## Summary

Keep open: current main and v2026.5.22 still expose `before_tool_call` as params rewrite, block, and approval only; the requested pending tool-call model/session redirect contract remains absent and security-sensitive. Related open work at https://github.com/openclaw/openclaw/pull/66147 and https://github.com/openclaw/openclaw/pull/62701 is adjacent, but neither has merged nor supersedes the plugin redirect/session semantics.

## What This Changes

Request a Plugin SDK/runtime hook that can inspect pending tool-call context and redirect execution to a different model or session before the tool runs.

## Best Possible Solution

Define an additive typed plugin/runtime routing contract with inspectable assistant context, redirect authority, audit and approval behavior, failure handling, and provider/session/auth boundary rules while preserving the existing params rewrite, block, and approval behavior.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a Plugin SDK/runtime API feature request. Source, docs, tests, release, and related-PR inspection give high confidence that the requested preceding-output plus redirect/model/session behavior is absent today.

## Solution Assessment

Unclear until maintainers choose the API and security model. The existing hooks cover block, approval, params rewrite, and pre-turn model selection, but not pending tool-call redirection.
