## Summary

Keep open: current main still lacks the requested typed pre-tool-call redirect/model/session routing contract, and the existing related PRs only cover context fields or an unmerged split-routing approach that does not supersede the plugin redirect semantics.

## What This Changes

Requests a pre-tool-call hook API that can inspect preceding assistant output and redirect pending tool calls to another model or session.

## Best Possible Solution

Define an additive typed pre-tool routing API with inspectable assistant context and explicit redirect semantics while preserving existing params rewrite, block, and approval behavior.

## Reproduction Assessment

Not applicable as a feature request. Source inspection of current main gives high confidence that the requested preceding-output and redirect/model/session hook fields and runtime routing behavior are absent.

## Solution Assessment

Unclear until maintainers choose the API shape. Prompt-time model hooks and config-level split routing do not satisfy the requested plugin redirect semantics by themselves, so this needs a deliberate plugin/runtime contract decision.
