## Summary

Current main supports pre-tool parameter rewriting, blocking, and approval, including agent-aware cases such as TTS argument injection, but it still cannot expose the pending assistant output and redirect that tool turn to another model or session. The related open pull request adds static tool-continuation model configuration, remains conflicting, and does not implement the requested plugin hook contract, so it is neither an implementation nor a safe canonical replacement.

## What This Changes

Requests a pre-tool-call plugin hook that can inspect preceding model output and redirect a pending tool call to another model or session.

## Best Possible Solution

Keep the existing before_tool_call contract focused on argument rewriting, blocking, and approval, then add a separate typed pre-continuation routing phase only after maintainers define its audit, permission, provider/auth, fallback, context-budget, and session-state semantics.

## Reproduction Assessment

Not applicable: this is a request for a new plugin/runtime capability, and current-main source inspection confirms the redirect contract is absent rather than broken.

## Solution Assessment

No, not as the proposed direct extension alone: tool-call policy and model/session continuation have different owners, so a dedicated routing phase with closed security and state semantics is the safer maintainable design.
