## Summary

Keep open: current main still lacks a typed pre-tool-call model/session redirect contract, and the related open PRs cover adjacent context or config-level routing without superseding the requested plugin redirect API.

## What This Changes

The issue requests a pre-tool-call plugin hook that can inspect preceding assistant output and redirect tool calls to a different model or session.

## Best Possible Solution

Define an additive typed pre-tool routing API with inspectable assistant context and explicit redirect semantics while preserving existing params rewrite, block, and approval behavior.

## Reproduction Assessment

Not applicable as a feature request. Source inspection of current main gives high confidence that the requested preceding-output and redirect/model/session hook fields and runtime routing behavior are absent.

## Solution Assessment

Unclear until maintainers choose the API shape. Either extending before_tool_call or adding a separate routing hook could work, but config-level split routing alone does not satisfy the requested plugin redirect semantics.
