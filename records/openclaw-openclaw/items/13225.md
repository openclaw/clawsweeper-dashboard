## Summary

Keep open: current main still lacks a `before_tool_call` contract that can inspect pending assistant context and redirect the pending tool call to another model or session; the adjacent open PRs cover context fields or split model routing, not this plugin redirect API.

## What This Changes

The issue requests a pre-tool-call hook that can inspect pending assistant output and redirect a tool call to another model or session.

## Best Possible Solution

Define an additive typed plugin/runtime routing contract with inspectable assistant context, redirect authority, audit/approval behavior, failure handling, and session/auth/provider boundary rules while preserving existing params rewrite, block, and approval behavior.

Not applicable as a bug reproduction because this is a feature/API request. Source inspection on current main gives high confidence that the requested preceding-output and redirect/model/session hook behavior is absent.

## Solution Assessment

Unclear until maintainers choose the API and security model. The current hook system solves block, approval, params rewrite, and pre-turn model selection, but not pending tool-call redirection.
