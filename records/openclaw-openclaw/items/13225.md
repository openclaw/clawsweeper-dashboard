## Summary

Keep open: current main still lacks the requested typed pre-tool-call redirect/model/session routing contract. Existing hooks cover parameter rewrite, blocking, approval, and prompt-time model selection, while related work remains either context-only or a separate config-level split-routing proposal.

## What This Changes

Add a pre-tool-call hook capability that can inspect preceding assistant output and redirect a pending tool call to another model or session.

## Best Possible Solution

Define an additive typed pre-tool routing API with inspectable assistant context and explicit redirect semantics while preserving the existing params rewrite, block, and approval behavior.

## Reproduction Assessment

Not applicable as a feature request. Source inspection of current main gives high confidence that the requested preceding-output and redirect/model/session hook fields and runtime routing behavior are absent.

## Solution Assessment

Unclear until maintainers choose the API shape. Prompt-time model hooks and config-level split routing do not satisfy the requested plugin redirect semantics by themselves, so this needs a deliberate plugin/runtime contract decision.
