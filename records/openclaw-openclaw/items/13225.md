## Summary

Current main still does not provide the requested pre-tool model/session routing contract. The existing hook can rewrite arguments, block execution, or request approval, while the separate model-resolution hook runs before session messages and tool calls exist; the closest related routing PR remains open and does not implement this plugin API.

## What This Changes

Requests a pre-tool-call plugin hook that can inspect preceding assistant output and redirect a pending tool call to another model or session.

## Best Possible Solution

Define one additive agent-runtime routing decision with explicit context visibility, authorization, provider-auth, transcript ownership, audit, fallback, cancellation, and failure semantics, while preserving the existing rewrite, block, and approval contract.

## Reproduction Assessment

Not applicable; this requests a new plugin/runtime capability rather than reporting broken existing behavior. Source inspection establishes that the capability is currently absent.

## Solution Assessment

The generic plugin-API direction is appropriate, but the proposed event fields and redirect object are not yet proven to be the best design because routing ownership, security, authentication, and session semantics require a maintainer decision.
