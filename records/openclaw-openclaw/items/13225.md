## Summary

Keep open: current main and v2026.5.20 still expose before_tool_call as params rewrite, block, and approval only; the requested pending tool-call model/session redirect contract remains absent and security-sensitive.

## What This Changes

The issue requests a pre-tool-call hook that can inspect preceding model output and redirect a pending tool call to another model or session.

## Best Possible Solution

Define an additive typed plugin/runtime routing contract with inspectable assistant context, redirect authority, audit and approval behavior, failure handling, and provider/session/auth boundary rules while preserving existing params rewrite, block, and approval behavior.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a Plugin SDK/runtime API feature request. Source, docs, release, and related-PR inspection give high confidence that the requested preceding-output plus redirect/model/session behavior is absent today.

## Solution Assessment

Unclear until maintainers choose the API and security model. The existing hooks cover block, approval, params rewrite, and pre-turn model selection, but not pending tool-call redirection.
