## Summary

Keep open. Current main still lacks plugin-facing pre-tool-call model/session redirect semantics: the existing hook can rewrite params, block, or require approval, and the related PRs are partial or open rather than a merged solution.

## What This Changes

Request an additive `before_tool_call` capability to inspect preceding assistant output and redirect a pending tool call to another model/session.

## Best Possible Solution

Design an additive typed pre-tool routing contract, either by extending `before_tool_call` or adding a separate hook, that defines inspectable assistant context and model/provider/session redirect behavior while preserving current param rewrite, block, and approval semantics.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. The high-confidence verification path is source and docs inspection showing the current hook lacks redirect semantics while the existing model hook runs before pending assistant tool calls exist.

## Solution Assessment

Unclear for the final API shape. Current main is not sufficient, and the maintainable path is a maintainer-designed additive routing contract rather than treating partial context fields or pre-session model selection as the complete fix.
