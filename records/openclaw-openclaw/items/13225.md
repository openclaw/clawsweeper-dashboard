## Summary

Keep open: current main still lacks a typed pre-tool-call model/session redirect contract. The existing hook supports params rewrite, blocking, and approval; related open PRs cover adjacent context fields or config-level routing without superseding the requested plugin redirect semantics.

## What This Changes

The issue requests a pre-tool-call plugin/runtime hook that can inspect preceding model output and redirect a pending tool call to another model or session.

## Best Possible Solution

Define an additive typed pre-tool routing API with inspectable assistant context and explicit redirect semantics, while preserving the existing params rewrite, block, and approval behavior.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main lacks the requested preceding-output and redirect/model/session hook fields and runtime routing behavior.

## Solution Assessment

Unclear until maintainers choose the API shape. Either extending before_tool_call or adding a separate routing hook could work, but model/provider/session boundary behavior needs a product and security contract first.
