## Summary

Keep open: current main and v2026.5.20 still lack the requested pre-tool-call redirect/model/session contract, and adjacent PRs cover context or a separate split-model experiment rather than a merged plugin/runtime API.

## What This Changes

Request a `before_tool_call`-time API that can inspect preceding assistant output and route a pending tool call to another model or session.

## Best Possible Solution

Define an additive typed plugin/runtime routing contract with inspectable assistant context, redirect authority, audit/approval behavior, failure handling, and provider/session/auth boundary rules while preserving existing params rewrite, block, and approval behavior.

## Solution Assessment

Unclear until maintainers choose the API and security model. The existing hooks cover block, approval, params rewrite, and pre-turn model selection, but not pending tool-call redirection.
