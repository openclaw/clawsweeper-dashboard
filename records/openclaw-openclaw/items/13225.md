## Summary

Keep open: current main still supports before_tool_call parameter rewrite, blocking, and approval, but not the requested preceding-output inspection or model/session redirect contract. The related split-routing and context-field PRs are closed unmerged, and the newer per-tool routing issue is only a partial overlap rather than a safer canonical replacement.

## What This Changes

The issue requests a pre-tool-call plugin/runtime hook that can inspect preceding assistant output and redirect a pending tool call to another model or session.

## Best Possible Solution

Define an additive typed pre-continuation routing contract after maintainer product and security review, preserving the existing before_tool_call params/block/approval behavior for narrower policy and TTS argument-injection cases.

## Reproduction Assessment

Not applicable: this is a feature/API request rather than a broken existing behavior report. Source inspection gives high confidence that current main and v2026.6.6 lack the requested redirect contract.

## Solution Assessment

No for the proposed direct before_tool_call extension alone. The safer maintainable path is a dedicated or carefully extended typed routing phase with explicit audit, permission, provider/auth, fallback, context-budget, and session-state semantics.
