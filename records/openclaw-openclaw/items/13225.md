## Summary

Current main still lacks the requested plugin-controlled model/session redirect: `before_tool_call` can rewrite parameters, block, or require approval, while model override occurs before the model turn from prompt and attachment inputs. The closest open PR adds static tool-continuation model configuration rather than this hook contract and is currently conflicting, so it neither implements nor safely supersedes the request. This remains a valid but security- and compatibility-sensitive plugin/runtime design item requiring maintainer product judgment.

## What This Changes

Requests a pre-tool-call plugin hook that can inspect preceding model output and redirect a pending tool call to another model or session.

## Best Possible Solution

Introduce one additive typed routing phase at the agent loop's model/tool-turn boundary, preserving the existing `before_tool_call` rewrite/block/approval contract and explicitly defining exposed context, provider/auth resolution, session-state ownership, auditability, permissions, fallback, and failure behavior.

## Reproduction Assessment

Not applicable as a bug reproduction: this issue requests a new plugin API capability. Source inspection nevertheless gives a high-confidence demonstration that current main has neither the requested preceding-output field nor a model/session redirect result.

## Solution Assessment

Unclear for the exact proposed shape. A typed routing extension is appropriate, but a separate agent-loop routing hook is likely safer and more maintainable than overloading the tool-execution hook with session and provider ownership.
