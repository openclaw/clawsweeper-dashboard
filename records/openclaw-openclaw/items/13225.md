## Summary

Keep open. Current main still lacks the requested pre-tool-call model/session redirect contract: `before_tool_call` can rewrite params, block, or require approval, `before_model_resolve` runs before pending tool calls exist, and the related PRs are open, partial, or closed unmerged.

## What This Changes

Request a plugin-facing hook capability that exposes preceding assistant context and can redirect a pending tool call to another model/session for delegation.

## Best Possible Solution

Design an additive plugin/runtime API for pre-tool routing that defines inspectable assistant context, model/provider/session redirect behavior, permissions, audit, fallback, and interaction with existing params rewrite, block, and approval semantics.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. The high-confidence verification path is source and docs inspection showing no redirect field or model/session routing branch on current main.

## Solution Assessment

Unclear for the final API shape. Current main is not sufficient, and the maintainable path is a maintainer-designed additive routing contract rather than treating existing param rewrite or pre-session model selection as the complete fix.
