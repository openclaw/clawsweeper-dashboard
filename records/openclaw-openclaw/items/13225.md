## Summary

Keep open: current main and the latest release still expose `before_tool_call` as policy, approval, blocking, and parameter rewrite only; the requested pre-tool-call model/session redirect contract remains a product and security design item. The open split-model PR is adjacent, not merged, and does not replace the plugin hook redirect/session semantics.

## What This Changes

Request an additive pre-tool-call hook capability that can inspect preceding assistant output and redirect a pending tool call to another model or session.

## Best Possible Solution

Define a maintainer-approved additive plugin/runtime routing contract with inspectable assistant context, constrained redirect authority, audit/approval behavior, failure handling, and provider/session/auth boundary rules while preserving current params rewrite, block, and approval semantics.

## Reproduction Assessment

Not applicable; this is a Plugin SDK/runtime API feature request, not a broken existing behavior with a reproduction path. Source, docs, release-tag, and related-PR inspection confirm the requested pre-tool-call redirect behavior is absent today.

## Solution Assessment

Unclear until maintainers choose the API and security model. Existing hooks cover params rewrite, block, approval, and prompt-time model selection, but the best maintainable solution needs an explicit redirect contract rather than ad hoc model swapping.
