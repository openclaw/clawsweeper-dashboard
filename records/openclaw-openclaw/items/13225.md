## Summary

Keep open: current main and the latest release still expose `before_tool_call` as params rewrite, block, and approval only; the requested pending tool-call model/session redirect contract remains a product and security design gap.

## What This Changes

Request an additive pre-tool-call hook capability that can inspect preceding assistant context and route a pending tool call to another model or session.

## Best Possible Solution

Define an additive typed plugin/runtime routing contract with inspectable assistant context, constrained redirect authority, audit/approval behavior, failure handling, and provider/session/auth boundary rules while preserving existing params rewrite, block, and approval behavior.

## Reproduction Assessment

Not applicable; this is a Plugin SDK/runtime API feature request rather than broken current behavior. Source, docs, release, and related-PR inspection give high confidence that pending tool-call redirect is absent today.

## Solution Assessment

Unclear until maintainers choose the API and security model. Existing hooks cover block, approval, params rewrite, and pre-turn model selection, but not pending tool-call redirection.
