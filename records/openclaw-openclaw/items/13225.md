## Summary

Keep open: current main and v2026.5.22 still expose `before_tool_call` as params rewrite, block, and approval only; the requested pending tool-call model/session redirect contract remains absent. Related open work is adjacent, but it has not merged and does not supersede the plugin redirect/session semantics.

## What This Changes

Request a pre-tool-call hook that can inspect model context and route a pending tool call to another model or session.

## Best Possible Solution

Define an additive typed plugin/runtime routing contract with inspectable assistant context, constrained redirect authority, audit and approval behavior, failure handling, and provider/session/auth boundary rules while preserving the existing params rewrite, block, and approval behavior.

## Solution Assessment

Unclear until maintainers choose the API and security model. Existing hooks cover block, approval, params rewrite, and pre-turn model selection, but not pending tool-call redirection.
