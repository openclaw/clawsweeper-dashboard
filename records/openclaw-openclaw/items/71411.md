## Review Findings

Overall confidence: 0.86

Full review comments:

- **[P1] Fail closed when the message-end gate cannot install:** `src/agents/pi-embedded-runner/run/attempt.ts:393-394`
  - body: `llm_message_end` is an enforceable output gate, but this path returns a no-op when the private `agent.listeners` field is unavailable. If that dependency detail changes, configured message-end policy hooks are silently skipped and assistant output proceeds ungated; surface the failure and block/fail the run when hooks are present, or use a supported listener API with guaranteed ordering.
  - confidence: 0.9
- **[P2] Preserve private transcript permissions during redaction:** `src/plugins/hook-redaction.ts:128`
  - body: `writeFile(tempFile, newContent, "utf-8")` creates the replacement transcript using the process umask, then renames it over the session file. After a redaction, a transcript that may still contain prompts, tool output, or secrets can become more broadly readable; create the temp file with restrictive permissions before renaming.
  - confidence: 0.86
- **[P2] Match duplicate retry prompts exactly:** `src/plugins/hook-redaction.ts:229-231`
  - body: The fallback duplicate cleanup treats substring matches as duplicate user prompts. A prior short message contained in the current prompt can make the function delete the original current prompt instead of a retry duplicate; restrict this path to exact prompt-text matches before removing the last match.
  - confidence: 0.88

## Security Review
