## Summary

This PR should stay open because the intended route change is sound, but the branch still lets persisted Kimi provider configs from prior onboarding overwrite the new OpenAI-compatible route and remain on the hanging Anthropic-compatible stream.

## What This Changes

The PR routes the bundled Kimi coding provider to Kimi's OpenAI-compatible stream and updates thinking-policy handling, legacy URL migration, provider-policy lookup, CLI thinking remap, and focused tests.

## Best Possible Solution

Add a provider-local Kimi `normalizeConfig` policy that migrates legacy coding configs to `openai-completions` plus `/coding/v1` before models-json merge and runtime use, with regression coverage for persisted configs.

## Reproduction Assessment

Yes. Source inspection shows current onboarding produces the legacy Kimi config and the PR's catalog-only normalization is overwritten when explicit provider config is merged back over implicit discovery; the PR body also supplies live before/after gateway evidence for the original timeout.

## Solution Assessment
