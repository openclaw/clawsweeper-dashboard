## Summary

Current main and v2026.6.6 already remove the reported file-backed temp-file producer by storing memory-core short-term recall and phase-signal state in SQLite-backed plugin state; the remaining cleanup branches target the retired JSON path.

## What This Changes

The issue reports that memory-core short-term recall and phase-signal atomic JSON writes could leave orphaned `.tmp` files under `memory/.dreams` after a crash or failed rename.

## Best Possible Solution

Keep the shipped SQLite-backed memory-core state path; if users still see legacy temp litter after upgrading, track that as a fresh one-time doctor/migration cleanup request against current main.

## Reproduction Assessment

No current-main failure path remains for the reported accumulation. Source inspection shows the named writers now use SQLite-backed plugin state instead of JSON temp-file write/rename paths.

## Solution Assessment

Yes. Removing the file-backed temp-file producer through the SQLite-backed memory-core state path is a more durable solution than adding cleanup around obsolete JSON temp files.
