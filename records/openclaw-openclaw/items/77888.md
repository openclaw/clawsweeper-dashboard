## Summary

Keep open: current main still writes short-term recall and phase-signal stores through orphanable `.tmp` paths, while the repair path only normalizes the recall store and stale lock state.

## What This Changes

The issue asks OpenClaw to clean up orphaned temporary files left by interrupted atomic writes in memory-core short-term artifact stores.

## Best Possible Solution

Add conservative stale-temp cleanup to memory-core artifact repair, with exact filename matching, an age guard, unlink-race tolerance, and regression coverage while preserving the current atomic-write pattern.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: seed an old `short-term-recall.json.<pid>.<time>.<uuid>.tmp` or `phase-signals.json.<pid>.<time>.<uuid>.tmp` in `memory/.dreams`, run the existing repair helper, and it will not be considered because the helper never scans the directory.

## Solution Assessment

Yes, the maintainable fix is narrow: keep the owner-local atomic writes and add stale temp cleanup in `repairShortTermPromotionArtifacts`, optionally surfaced in the existing repair summaries.
