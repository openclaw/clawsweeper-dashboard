## What This Changes

The issue asks OpenClaw to update WSL2/Windows remote Chrome CDP docs and diagnostics so IPv6-only [::1] binding and v4tov6 portproxy remediation are surfaced instead of only the stale v4tov4 self-loop case.

## Best Possible Solution
