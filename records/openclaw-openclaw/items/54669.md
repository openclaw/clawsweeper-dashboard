Request to update WSL2 + Windows remote Chrome CDP troubleshooting and diagnostics for Chrome versions that expose CDP on IPv6 loopback while IPv4 `v4tov4` portproxy fails.

## Best Possible Solution

Keep this open and land a focused docs plus diagnostic-hint patch that identifies IPv4 vs IPv6 CDP binding, documents `v4tov6` when Chrome listens on `[::1]`, and avoids automatic Windows networking changes.
