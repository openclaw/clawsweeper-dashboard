## What This Changes

The PR adds shared log redaction patterns and tests for generic JWTs, Basic auth headers, and named security headers used by the `logs.tail` path.

## Best Possible Solution

Amend this PR or land an equivalent narrow patch that fully covers raw Basic auth, `X-OpenClaw-Token`, Pomerium JWT assertions, and unquoted security headers in shared defaults with focused `redactSensitiveText`, `redactSensitiveLines`, and `logs.tail` regression coverage.

## Reproduction Assessment

Yes, at source level with high confidence. `readConfiguredLogTail` applies the shared defaults to returned lines, and a focused regex probe shows the PR Basic auth matcher misses the linked issue sample and leaves padding outside longer matches.

## Solution Assessment

No, not as written. The shared default redactor is the right layer, but the Basic auth matcher and standalone JWT regression test need tightening before this is the best fix.
