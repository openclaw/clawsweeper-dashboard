## Summary

Keep open. This is a useful bug-fix PR for the open `logs.tail` redaction issue, but it is not merge-ready because the proposed Basic auth matcher still misses concrete credentials and the external PR lacks after-fix real behavior proof.

## What This Changes

The PR adds default redaction patterns and tests for JWT-shaped tokens, Basic auth headers, and named security headers in the shared logging redactor used by `logs.tail`.

## Best Possible Solution

Amend this PR, or land an equivalent narrow patch, so shared defaults fully cover raw Basic auth and named security headers with focused `redactSensitiveText`, `redactSensitiveLines`, and `logs.tail` coverage plus real behavior proof.

## Reproduction Assessment

Yes, at source level with high confidence. `readConfiguredLogTail` applies shared redaction to returned lines, and a focused regex probe shows the PR Basic auth rule misses the linked sample and mishandles padded credentials.

## Solution Assessment

No, not as written. The shared default redactor is the right layer, but the Basic auth matcher needs to cover valid short and padded Base64 credentials, and the JWT test should isolate standalone JWT redaction.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92
