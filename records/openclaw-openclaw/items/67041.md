## Summary

Keep this PR open. Current main still lacks the requested shared redaction coverage for `logs.tail`, and the PR is the right implementation candidate for the linked open bug, but the branch needs amendment before merge because the Basic auth pattern can leave padding unmasked and the JWT test does not isolate the new standalone JWT rule.

## What This Changes

The PR adds default logging redaction patterns and tests for standalone JWTs, Basic Authorization headers, and `X-OpenClaw-Token`/`x-pomerium-jwt-assertion`/`X-Api-Key`/`X-Auth-Token` header values.

## Best Possible Solution

Keep this PR as the implementation path, but amend the shared default redaction rules so they fully mask padded Basic auth, standalone JWTs, and the named security headers, with focused coverage for `redactSensitiveText`, `redactSensitiveLines`, and the `logs.tail` endpoint path before merge.

## Reproduction Assessment

Yes. The linked issue provides a concrete `logs.tail` path, current main source confirms the missing defaults, and a direct regex probe against the PR pattern reproduces the padded Basic auth masking gap.

## Solution Assessment

No, not as written. Adding shared default redaction patterns is the right maintainable location, but the Basic auth delimiter and JWT regression test need to be tightened before this is the best fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Mask Basic auth padding with the credential:** `src/logging/redact.ts:44`
  - body: The Basic auth pattern ends with a word boundary after a base64 character class. For padded credentials such as `Authorization: Basic dXNlcm5hbWU6cGFzc3dvcmQ=`, the match backs off before the `=`, leaving padding outside the replacement span. Drop the boundary or use a delimiter that still includes padding in the captured credential.
  - confidence: 0.91
- **[P3] Exercise the standalone JWT pattern directly:** `src/logging/redact.test.ts:215`
  - body: This test puts the JWT behind `Authorization: Bearer`, which is already covered by the existing Bearer regex. It would still pass if the new standalone JWT pattern stopped working, so use a standalone JWT or non-Bearer header sample to protect the behavior this PR adds.
  - confidence: 0.86

## Security Review
