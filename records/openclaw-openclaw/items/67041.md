## Summary

Keep open. The linked bug remains live for raw Basic auth and `X-OpenClaw-Token` on current main, and this PR is a plausible narrow fix path but still has a blocking Basic-auth redaction defect plus missing real behavior proof.

## What This Changes

The PR adds default log redaction regexes and tests for generic JWTs, Basic auth headers, and `X-OpenClaw-Token`/`x-pomerium-jwt-assertion`/`X-Api-Key`/`X-Auth-Token` header values.

## Best Possible Solution

Amend this PR or land an equivalent narrow patch that fully redacts raw Basic auth, `X-OpenClaw-Token`, documented Pomerium JWT assertions, and current JWT shapes in the shared defaults, with focused `redactSensitiveText`, `redactSensitiveLines`, and `logs.tail` regression coverage.

## Reproduction Assessment

Yes, at source level with high confidence. Current main shows `logs.tail` uses shared redaction defaults, and direct regex probes show the defaults still miss raw Basic auth and `X-OpenClaw-Token` while the PR regex misses short and padded Basic auth cases.

## Solution Assessment

No, not as written. The shared default redaction list is the right layer, but the Basic auth matcher and JWT regression test need tightening before this is the best fix.
