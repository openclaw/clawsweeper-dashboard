## Summary

Keep open: current main still lacks the requested default redaction coverage, and this PR is a narrow implementation path, but the diff leaves Basic auth cases under-redacted on a security-sensitive `logs.tail` surface.

## What This Changes

The PR adds default log redaction regexes and tests for standalone JWTs, Basic Authorization headers, and `X-OpenClaw-Token`/`x-pomerium-jwt-assertion`/`X-Api-Key`/`X-Auth-Token` header values.

## Best Possible Solution

Amend this PR's shared default patterns so Basic auth, standalone JWTs, and the named security headers are fully redacted, with focused coverage for `redactSensitiveText`, `redactSensitiveLines`, and `logs.tail`.

## Reproduction Assessment

Yes. Current main source shows the missing defaults, and a direct regex probe against the PR pattern reproduces both the short Basic auth miss and padded-value masking gap.

## Solution Assessment

No, not as written. The shared default redaction list is the right maintainable layer, but the Basic auth matcher and JWT regression test need tightening before this is the best fix.

## Review Findings

Overall correctness: patch is incorrect

  - body: This regex only matches Basic credentials with at least 18 base64 characters and then requires a word boundary.   - confidence: 0.93

  - confidence: 0.87
