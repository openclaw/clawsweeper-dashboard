## Summary

Keep open; this PR targets a real `logs.tail` redaction gap, but the proposed Basic auth regex is incomplete and after-fix real behavior proof is still missing before merge.

## What This Changes

The PR adds default log redaction regexes and tests for JWT-shaped tokens, Basic auth headers, and named security headers in the shared redactor used by `logs.tail`.

## Best Possible Solution

Amend this PR or land an equivalent narrow patch so shared defaults fully redact raw Basic auth and named security headers, preserve current JWT coverage, and add focused redactor plus `logs.tail` proof.

## Reproduction Assessment

Yes, at source level with high confidence. `readConfiguredLogTail` applies shared redaction, and focused probes show current main misses the reported header formats while the PR Basic auth rule misses short credentials and padded endings.

## Solution Assessment

No, not as written. The shared default redactor is the right layer, but the Basic auth matcher needs to cover valid short and padded Base64 credentials, and the JWT test should isolate standalone JWT redaction.
