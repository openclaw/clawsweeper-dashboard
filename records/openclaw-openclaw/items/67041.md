## Summary

Keep open. This is a focused credential-redaction PR for https://github.com/openclaw/openclaw/issues/66832, but the current branch still has a concrete Basic auth masking defect and lacks required after-fix real behavior proof.

## What This Changes

The PR adds default log redaction patterns and tests for generic JWTs, raw Basic auth, and named security headers in the `logs.tail` redaction path.

## Best Possible Solution

Amend this PR or land an equivalent narrow patch that fully covers raw Basic auth, `X-OpenClaw-Token`, Pomerium JWT assertions, and standalone JWTs in shared defaults, with focused redaction and `logs.tail` regression coverage.

## Reproduction Assessment

Yes, at source level with high confidence. `readConfiguredLogTail` applies shared redaction to returned lines, and a focused probe shows the PR Basic auth regex misses the linked issue sample and leaves padding outside a longer credential.

## Solution Assessment

No, not as written. The shared default redactor is the right layer, but the Basic auth matcher needs to cover valid short and padded Base64 credentials, and the JWT test should isolate standalone JWT redaction.
