## Summary

No blocking findings: the patch fixes the linked Telegram DM context duplication at the Telegram message-context boundary without changing config, routing, or delivery behavior.

## What This Changes

The PR filters Telegram `chat_window` prompt context out of existing unthreaded private DM sessions while keeping it for fresh DMs, private DM replies, topics, and groups.

## Best Possible Solution

Land this narrow Telegram-context filter after normal CI/maintainer gates, keeping the session-aware suppression in the payload builder rather than disabling chat-window construction globally.

## Reproduction Assessment

Yes. Current main builds Telegram `chat_window` prompt context and forwards it unconditionally into `UntrustedStructuredContext`, which matches the linked issue's duplicate-DM-context report.

## Solution Assessment

Yes. Filtering after route/session resolution is the right narrow layer because it can distinguish existing DMs from fresh DMs, DM topics, replies, and groups; disabling context construction earlier would be too broad.
