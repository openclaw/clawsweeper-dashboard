## Best Possible Solution

Land the narrow session-aware Telegram context filter after normal maintainer and CI gates, then let the linked issue close through the PR merge.

## Solution Assessment

Yes. Filtering after route and session lookup is the narrowest maintainable fix because that layer can distinguish existing DMs from fresh DMs, replies, DM topics, and groups; disabling context construction earlier would be too broad.
