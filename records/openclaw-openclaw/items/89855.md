## Summary

Keep this PR open as the active, focused fix for the linked Telegram DM duplicate-context bug; current main still forwards Telegram chat-window context into existing DM turns, and the PR’s session-aware filter is narrow with focused coverage.

## What This Changes

The PR changes Telegram inbound context construction to suppress Telegram chat-window prompt context for existing unthreaded private DM sessions while keeping it for fresh DMs, replies, groups, and topic-style contexts.

## Best Possible Solution

Land a narrow session-aware Telegram prompt-context filter if maintainer review and CI stay green, without adding a new config knob or disabling reply/group/topic context globally.

## Reproduction Assessment

Yes. Current main builds Telegram chat_window prompt context and forwards it into UntrustedStructuredContext without checking whether a private DM session already exists.

## Solution Assessment

Yes. Filtering in the inbound payload builder after session routing is narrower than disabling prompt-context construction earlier, because it preserves fresh DM bootstrap context plus reply, group, and topic context.
