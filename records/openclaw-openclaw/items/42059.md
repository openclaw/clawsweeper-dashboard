## Summary

Keep open. The PR contains useful Telegram fixes, and several ideas have since landed or been superseded in current main, but the branch is still a stale pre-plugin batch against old `src/telegram/*` paths and current main does not implement all intended behavior. It should not be merged as-is because it mixes plugin-owned Telegram fixes with approval authorization changes, and the diff still has an actionable `/approve` authorization regression.

## What This Changes

The PR batches Telegram delivery, media, command-menu, network/proxy, draft-stream, metadata-stripping, retry, ACP thread-binding, and approval-flow fixes across old Telegram core files plus shared media, agent, and approval code.

## Best Possible Solution

Split the batch into focused current-main changes under `extensions/telegram` and the shared approval/ACP seams only where needed. Keep the current main implementations for already-solved retry, ACP placement, metadata, reply-parameter, and streaming behavior, and port only the remaining verified Telegram plugin gaps with targeted tests.

## Reproduction Assessment

Yes for source-level reproduction of the remaining gaps: current main visibly hashes only the command list, returns `null` on exhausted `getFile`, keeps a recoverable-only media-group skip path, and does not guard sticky IPv4 fallback on explicit network config. There is not one single runtime reproduction because this PR batches many independent bugs.

## Solution Assessment

No. A stale all-in-one branch against the old Telegram layout is not the best way to solve these issues; the maintainable path is focused plugin-owned PRs that preserve newer current-main behavior and give the approval authorization change a separate security review.

## Review Findings

- **[P2] Gate Telegram approve replies behind authorization:** `src/auto-reply/reply/commands-approve.ts:135-136`
  - body: The refactor lets Telegram `/approve` parsing and the Telegram-specific rejection path run before preserving the generic `allowFrom` silence.
