## Summary

Keep open. Current main has already absorbed some Telegram fixes, but this PR is still a broad stale legacy-layout patch with unresolved current-main gaps and an approval authorization regression that should not be merged as-is.

## What This Changes

The PR batches Telegram delivery, media, command-menu caching, network/proxy, draft-preview, metadata-stripping, retry, ACP thread-binding, and approval-flow fixes across legacy `src/telegram` code plus shared ACP/media/approval helpers.

## Best Possible Solution

Split the still-useful pieces into focused current-main changes under `extensions/telegram`, keep the already-shipped retry/ACP/reply/chunking fixes, and review approval authorization separately with security-focused tests.

## Reproduction Assessment

Yes for source-level review: current main still shows several remaining gaps, including command-menu cache keys, getFile failure handling, media-group abort behavior, explicit-network fallback, metadata stripping, and mention-gate diagnostics. There is no single runtime reproduction because this PR batches many independent Telegram bugs.

## Solution Assessment

No. A stale all-in-one branch against the old `src/telegram` layout is not the best way to solve this; the maintainable path is focused plugin-owned PRs with targeted tests and a separate approval-security review.
