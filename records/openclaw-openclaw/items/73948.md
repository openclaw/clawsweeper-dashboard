## Summary

This PR should stay open: the source and test changes correctly align the security audit boundary with the workspace skill loader, but the added changelog entry points users to the wrong command for this audit finding. ## What This Changes

The PR changes the workspace skill symlink audit to compare `SKILL.md` realpaths against the resolved workspace `skills` directory, updates audit/docs/changelog wording, and adds regression coverage for inside-workspace but outside-skills symlink targets.

## Best Possible Solution

Land the audit boundary alignment with the regression test, but first correct the release note to name `openclaw security audit` as the command that reports this finding; keep broader orphan-skill discovery separate in #46690.

## Reproduction Assessment

Yes. A high-confidence reproduction is a `workspace/skills/linked/SKILL.md` symlink whose target resolves inside the workspace but outside `workspace/skills`; current main’s audit accepts it while the loader rejects the same path shape.

## Solution Assessment

Mostly yes. Matching the audit containment root to the loader’s resolved workspace `skills` root is the narrow maintainable fix; the safer adjustment is only to repair the changelog command wording before merge.
