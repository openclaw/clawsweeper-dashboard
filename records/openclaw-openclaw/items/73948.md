## Summary

Keep open. Current main still has the audit/loader boundary mismatch, and the proposed patch is a narrow, correct-looking fix with no blocking code findings, but it is not implemented on main and GitHub currently reports the PR branch as dirty/unmergeable.

## What This Changes

The PR narrows `skills.workspace.symlink_escape` from the workspace root to the resolved `workspace/skills` root, updates audit/docs/changelog wording, and adds a regression test for symlinks that resolve inside the workspace but outside `workspace/skills`.

## Best Possible Solution

Land this narrow audit-boundary alignment, or an equivalent maintainer-rebased replacement, so `openclaw doctor` reports workspace skill symlink realpaths that the workspace skill loader will reject under the `workspace/skills` configured root.

## Reproduction Assessment

Yes. A high-confidence reproduction is `workspace/skills/linked/SKILL.md` as a symlink to `workspace/shared-skill/SKILL.md`: current main's audit accepts it because the realpath is inside the workspace root, while the loader rejects it because it is outside `workspace/skills`.

## Solution Assessment

Yes, subject to rebase and maintainer/security confirmation. Comparing against the resolved `workspace/skills` root is the narrowest maintainable fix for this mismatch because it matches the configured root passed into the loader without changing skill loading semantics.
