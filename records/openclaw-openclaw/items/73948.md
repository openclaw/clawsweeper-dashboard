## Summary

Keep open: current main still has the audit/loader boundary mismatch, and this PR is the right narrow fix, but the changelog points users at `openclaw doctor` instead of the command that actually runs this audit.

## What This Changes

The PR changes the workspace skill symlink audit to use the resolved workspace `skills` directory as the containment root, updates audit/docs/changelog copy, and adds inside-workspace/outside-skills regression coverage.

## Best Possible Solution

Land the audit-boundary alignment after changing the release note to tell users to run `openclaw security audit`, while preserving @menhguin credit and the source PR context from #60155 and #60513.

## Reproduction Assessment

Yes. A high-confidence reproduction is a `workspace/skills/linked/SKILL.md` symlink whose target is inside the workspace but outside `workspace/skills`; current main’s audit accepts it while the loader rejects it.

## Solution Assessment

Mostly yes. Matching the audit containment root to the loader’s resolved `workspace/skills` root is the narrow maintainable fix; the safer adjustment is only to correct the changelog command wording.
