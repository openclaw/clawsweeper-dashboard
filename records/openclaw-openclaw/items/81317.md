## Summary

Keep open: the room-event design fits the Telegram visible-reply policy and the PR includes real Telegram proof, but PR head still has a visible error fallback path for room events if dispatch fails before a turn result is returned.

## What This Changes

The PR adds room-event semantics for ambient Telegram group messages, preserving them as context while suppressing visible replies unless the agent uses the message tool.

## Best Possible Solution

Add a room-event guard to Telegram dispatch fallback/error delivery and cover it with a regression while preserving the message-tool-only room-event design.

## Reproduction Assessment

Yes, at source level: make the reply dispatcher reject for a context whose InboundTurnKind is "room_event" and dispatchError reaches the shared deliverReplies fallback. I did not run a live Telegram failure-path repro in this read-only review.

## Solution Assessment

No, not yet. The direction matches the Telegram policy, but the PR needs a guard so room-event error and fallback paths stay quiet unless the agent sends through the message tool.
