## Summary

Keep open; the normal room-event path looks well covered, but the dispatch-error fallback can still send a visible Telegram message for ambient group chatter without the message tool. Real behavior proof for the happy path is sufficient and the security pass is clear.

## What This Changes

The branch adds a `room_event` inbound turn kind, routes ambient Telegram group messages through message-tool-only room events, updates prompt/session context accounting, docs, and tests.

## Best Possible Solution

Keep the room-event design, but make room-event error and fallback paths internal/logged unless the agent explicitly sends via the message tool, with focused regression coverage.

## Reproduction Assessment

Yes, for the review finding at source level: a `room_event` dispatch error sets `dispatchError`, and the shared fallback still calls `deliverReplies`. I did not run a live Telegram failure-path repro in this read-only review.

## Solution Assessment

No, not yet. The PR handles the normal quiet room-event path well, but the failure fallback must also stay quiet/logged unless the message tool produced visible output.
