## Summary

Keep this PR open for maintainer review: the room-event direction is plausible, but the patch still leaves visible Telegram paths for ambient turns and the PR only provides synthetic verification, not live Telegram proof.

## What This Changes

The PR adds a room-event turn kind, routes unmentioned always-on Telegram group messages through invisible message-tool-only processing, preserves them in context/history, and updates group-channel docs and tests.

## Best Possible Solution

Land this only after room events suppress every non-message-tool visible path, preserve context/history semantics, and include redacted live Telegram proof for ambient chatter plus a direct group request.

## Reproduction Assessment

Yes, from source for the review findings: with Telegram reasoning streaming enabled, a room event still creates the reasoning draft lane, and status-reaction callbacks still run for room events. I did not run a live Telegram session in this read-only review.

## Solution Assessment

No, not yet. The room-event design is directionally maintainable, but the implementation must suppress reasoning drafts and status reactions for ambient turns before it matches the documented message-tool-only behavior.

Overall correctness: patch is incorrect

With status reactions enabled, ambient room events can visibly mutate the chat message even when no message-tool send happens; gate these reaction paths for `isRoomEvent`.
  - confidence: 0.86
