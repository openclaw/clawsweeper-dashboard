## Summary

Keep open; the direction matches the Telegram room-reply policy and the real behavior proof is sufficient, but PR head still has a source-proven fallback path that can make room-event failures visible without the message tool. The protected maintainer label also means this is not a cleanup-close candidate.

## What This Changes

The branch adds room-event turn semantics for ambient always-on Telegram group chatter so it is context for the agent while visible replies require the message tool.

## Best Possible Solution

Keep the room-event design, but add a guard so Telegram room-event error and fallback delivery paths stay quiet unless the agent sends through the message tool.

## Reproduction Assessment

Yes, at source level: make the reply dispatcher reject for a context whose InboundTurnKind is "room_event" and dispatchError still reaches the shared deliverReplies fallback. I did not run a live Telegram failure-path repro in this read-only review.

## Solution Assessment

No, not yet. The overall direction matches the maintainer Telegram policy, but the PR needs a narrow room-event guard around fallback/error delivery plus a regression test before merge.
