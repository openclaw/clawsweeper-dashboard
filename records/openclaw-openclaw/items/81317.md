## Summary

Keep open: the PR has sufficient live Telegram proof and matches the room-reply policy direction, but it should not merge yet because superseded room-event dispatches can still clear shared group history and erase ambient context in busy groups.

## What This Changes

The PR adds room-event turn semantics across Telegram, channel turn context, reply prompt construction, agent transcript persistence, context maps, tests, and group docs so ambient always-on group messages stay quiet unless the message tool sends.

## Best Possible Solution

Land the room-event model after preserving room history across superseded quiet turns and keeping the existing message-tool-only visible reply contract intact.

## Reproduction Assessment

Yes, from source inspection. Room events are recorded into `groupHistories` before dispatch, Telegram reply fences can supersede older same-session turns, and the superseded branch still clears the whole group history map.

## Solution Assessment

No, not yet. The room-event design is a maintainable direction, but the superseded-dispatch history clear needs the same room-event preservation rule used for ordinary quiet room-event completion.
