## Summary

The PR should stay open because the central room-event design has useful tests and live Telegram proof, but a source-level fallback path can still send a visible Telegram error for ambient room events without a `message(action=send)` call.

## What This Changes

The branch adds `room_event` turn semantics across Telegram inbound context, reply prompt/session handling, context reporting, docs, and regression tests so ambient always-on group chatter is observed quietly unless the agent uses the message tool.

## Best Possible Solution

Keep the room-event design, but make room-event error and fallback paths internal/logged unless the agent explicitly sends through the message tool, with focused regression coverage for dispatch failures.

## Reproduction Assessment

Yes, for the review finding at source level: a `room_event` dispatch error sets `dispatchError`, and the shared fallback still reaches `deliverReplies`. I did not run a live Telegram failure-path repro in this read-only review.

## Solution Assessment

No, not yet. The normal quiet path is covered, but the proposed implementation still needs to suppress visible error fallbacks for room events before it fully matches the `message_tool_only` contract.
