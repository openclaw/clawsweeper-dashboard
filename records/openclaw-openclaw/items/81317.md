## Summary

Keep open: the room-event design and normal-path proof are useful, but the current PR head still lets room-event dispatch errors fall through to a visible Telegram fallback, which breaks the quiet-unless-message-tool contract.

## What This Changes

Adds a `room_event` inbound turn kind and routes ambient always-on Telegram group chatter through quiet, message-tool-only prompting, history, docs, and tests.

## Best Possible Solution

Keep the room-event design, but make the error and fallback paths quiet for room events too, with regression coverage for a thrown dispatch before maintainer review and landing.

## Reproduction Assessment

Yes, for the review finding at source level: a `room_event` whose dispatch throws sets `dispatchError`, then the shared fallback calls `deliverReplies`. I did not run a live Telegram failure repro.

## Solution Assessment

No, not yet. The PR handles normal quiet room-event paths well, but the failure path should also stay internal/logged unless the message tool has produced visible output.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Suppress room-event error fallbacks:** `extensions/telegram/src/bot-message-dispatch.ts:1433-1434`
