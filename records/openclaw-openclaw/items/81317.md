## Summary

Keep open: this PR aligns with the Telegram maintainer policy and has sufficient real behavior proof for the happy paths, but a source-level blocker remains where room-event dispatcher failures can still send the shared Telegram error fallback without a message-tool call.

## What This Changes

The PR adds room-event semantics for ambient Telegram group messages, including prompt shaping, context/history handling, quiet Telegram delivery behavior, docs, and regression tests.

## Best Possible Solution

Keep the room-event model, but make every ambient Telegram room-event visible output path go through the message tool, including dispatcher failures.

## Reproduction Assessment

Yes, at source level: make dispatchReplyWithBufferedBlockDispatcher reject for a room_event; dispatchError is set and the existing fallback deliverReplies branch is still reachable. I did not run a live Telegram failure-path repro in this read-only review.

## Solution Assessment

No, not yet. The room-event design is the maintainable direction, but the Telegram dispatch error fallback needs a narrow isRoomEvent guard and a rejection-path regression test before merge.
