## Summary

Keep open: current main still skips the one-way migration warning when `openclaw secrets configure` proceeds through the interactive apply prompt, and the linked fixing PR is still open, so the issue should not be closed before that PR lands or is replaced.

## What This Changes

Ensure `openclaw secrets configure` shows the irreversible migration confirmation whenever apply proceeds, including after an interactive yes response.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84225 or an equivalent focused fix that derives the warning from the final `shouldApply` state, with regression coverage for the two-confirmation interactive path.
