## Summary

Keep open: current main still derives the irreversible confirmation from `opts.apply`, so the interactive yes path can apply a one-way secrets migration without the warning. An open linked PR already targets this issue, so the issue should stay open until that PR lands or is closed.

## What This Changes

The issue asks for `openclaw secrets configure` to show the one-way migration warning when an interactive user confirms apply, not only when `--apply` was passed.

## Best Possible Solution

Land or replace the linked fix so the irreversible prompt keys off the final write intent after interactive confirmation, with focused regression coverage; close this issue only after the fix merges.

## Reproduction Assessment

Yes, at source level: current `secrets configure` can set `shouldApply` from the interactive prompt while `needsIrreversiblePrompt` remains tied to `opts.apply`, so no `--apply` plus an affirmative answer reaches `runSecretsApply` without the second warning.

## Solution Assessment

Yes, the narrow maintainable fix is to derive the irreversible prompt from final `shouldApply` while preserving `--yes` and `--json`; the linked PR targets that path.
