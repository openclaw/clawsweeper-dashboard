## Summary

Keep open. Current main still defaults sandboxed browser tool calls to the sandbox bridge, that bridge registers the normal Playwright-gated browser routes, and the issue has explicit maintainer direction not to close until isolated-session browser coverage exists.

## What This Changes

The issue requests consistent Playwright-backed browser navigate, snapshot, and act support in isolated sub-agent gateway sessions.

## Best Possible Solution

Define the supported runtime owner for isolated-session browser operations, then add regression coverage proving sandboxed sub-agent navigate, snapshot, and act calls do not hit the Playwright-unavailable 501 when browser support is available.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection gives a medium-confidence path: isolated agent tools receive the sandbox bridge URL, default calls stay on that bridge, and navigate/act still require Playwright in the bridge route runtime.

## Solution Assessment

Unclear because there is no proposed patch. The narrow maintainable solution needs an owner decision between making the sandbox bridge Playwright-capable and explicitly delegating to the host/node browser owner while preserving sandbox policy.
