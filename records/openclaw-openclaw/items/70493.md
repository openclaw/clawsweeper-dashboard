## Summary

Keep open. Current main still sends sandboxed isolated browser calls to the sandbox browser bridge by default, and that bridge registers the normal Playwright-gated browser routes; navigate and act can still hit the reported Playwright-unavailable path, and the maintainer discussion explicitly asked for isolated-session coverage before cleanup.

## What This Changes

The issue requests full Playwright-backed browser automation for isolated sub-agent sessions so navigate, snapshot, and act work consistently with the main session.

## Best Possible Solution

Define the browser runtime owner for isolated sessions, then either make the sandbox browser bridge Playwright-capable or explicitly delegate to the host/node browser owner while preserving sandbox policy, and add isolated sub-agent regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: run a sandboxed isolated sub-agent with browser enabled, call browser navigate or act through the default sandbox target, and verify it can still reach the requirePwAi 501 path when that runtime lacks Playwright.

## Solution Assessment

Unclear because there is no proposed patch. The narrow maintainable fix needs a maintainer choice between sandbox Playwright packaging and explicit host/node delegation, with security boundaries preserved either way.
