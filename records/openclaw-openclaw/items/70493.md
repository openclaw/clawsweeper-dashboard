## Summary

Keep open: current main still has the isolated/sandbox browser path routed through the sandbox bridge and the full navigate/act path remains Playwright-gated, while the maintainer comment explicitly asked for isolated-session coverage before cleanup.

## What This Changes

The issue asks for browser navigate/snapshot/act support to work in isolated sub-agent sessions the same way it works in the main session.

## Best Possible Solution

Define the browser runtime owner for isolated sessions, then either make the sandbox browser bridge Playwright-capable or explicitly delegate browser work to the host/node owner without weakening sandbox policy, with isolated-session regression coverage for navigate or snapshot.

## Reproduction Assessment

Yes. A high-confidence reproduction/validation path is to run an isolated or sandboxed sub-agent with the browser tool allowed, call browser navigate or snapshot through the sandbox bridge, and verify the route no longer hits the requirePwAi 501 path when the owning browser runtime supports Playwright.

## Solution Assessment

Unclear because there is no proposed patch. The best fix needs maintainer choice between sandbox Playwright packaging and host/node browser delegation, while preserving SSRF checks, bridge auth, and allowHostControl policy.
