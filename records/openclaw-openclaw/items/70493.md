## Summary

Keep open. Current main still routes default sandboxed browser calls to the sandbox bridge, that bridge registers the normal Playwright-gated routes, and the live discussion has explicit reviewer direction not to close until isolated-session browser coverage exists.

## What This Changes

The issue requests full Playwright-backed browser tool support for isolated sub-agent sessions so navigate, snapshot, and act work consistently with the main session.

## Best Possible Solution

Define the supported runtime owner for isolated-session browser operations, then add regression coverage proving sandboxed sub-agent navigate/snapshot/act do not hit Playwright-unavailable when browser support is available.

## Reproduction Assessment

No live high-confidence reproduction was run in this pass. Current source provides a medium-confidence reproduction path: sandboxed agent tool wiring passes a sandbox bridge URL, the browser tool defaults to that bridge, and navigate/act still hit requirePwAi in the bridge runtime.

## Solution Assessment

Unclear because there is no proposed patch. The narrow maintainable fix needs an owner decision between making the sandbox bridge Playwright-capable and explicitly delegating to the host/node browser owner while preserving sandbox policy.
