## Summary

Keep open. Current main still wires sandboxed/isolated browser tool calls through the sandbox browser bridge and the same Playwright-gated navigate/snapshot/act routes, and the issue discussion has explicit maintainer direction not to close until isolated-session coverage exists.

## What This Changes

The issue requests full Playwright-backed browser support for isolated session gateways so browser navigate, snapshot, and act work consistently with the main session.

## Best Possible Solution

Define the browser runtime owner for isolated sessions, implement that path without weakening sandbox boundaries, and add an isolated sub-agent or sandbox-bridge regression test proving navigate or snapshot no longer returns the Playwright-unavailable error when the owning browser runtime supports Playwright.
