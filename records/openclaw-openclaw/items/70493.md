## Summary

Keep open. Current main still defaults sandboxed isolated browser calls to the sandbox bridge, that bridge registers the normal Playwright-gated browser routes, and the existing maintainer discussion explicitly says not to close this until isolated-session browser coverage exists.

## What This Changes

The issue requests consistent Playwright-backed browser automation for isolated sub-agent sessions, including navigate, snapshot, and act.

## Best Possible Solution

Define the browser runtime owner for isolated sessions, then add coverage proving isolated sub-agent navigate/snapshot/act do not hit Playwright-unavailable when the supported browser runtime is available.

## Reproduction Assessment

Yes by source inspection, but not by a live run in this pass. A sandboxed isolated sub-agent with browser enabled defaults to the sandbox bridge, and navigate/act still reach requirePwAi on that bridge runtime.

## Solution Assessment

Unclear because there is no proposed patch. The narrow maintainable fix needs an owner decision between making the sandbox bridge Playwright-capable and explicitly delegating to the host/node browser owner while preserving sandbox policy.
