## Label Justifications

- `P1`: The issue describes browser automation failing across isolated sub-agent sessions, which can block real agent workflows rather than cosmetic behavior.

## Summary

Keep open. Current main still routes ordinary sandboxed browser calls through the sandbox bridge, the bridge registers Playwright-gated browser routes, and live maintainer discussion asks for isolated-session coverage plus an owner decision before cleanup.

## What This Changes

The issue requests Playwright-backed browser navigate, snapshot, and act support in isolated sub-agent sessions so they behave consistently with the main session.

## Best Possible Solution

Define whether isolated browser operations are sandbox-owned or host/node-owned, preserve explicit sandbox security policy, and add regression plus real isolated-session proof that navigate or snapshot no longer hits the Playwright-unavailable route when browser support is available.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection gives a medium-confidence reproduction path: sandboxed sessions receive a bridge URL, default browser calls stay on that bridge, and navigate/act still require Playwright in the bridge route runtime.

## Solution Assessment

Unclear because there is no patch. The maintainable fix needs an owner decision between packaging or loading the Playwright-backed runtime for isolated browser routes and explicitly delegating browser control to the host or browser node that owns the capability.
