## Summary

Keep open. Current main still sends default sandboxed browser tool calls to the sandbox bridge, the bridge registers the normal Playwright-gated browser routes, and the issue discussion has explicit reviewer direction not to close until isolated-session browser coverage exists.

## What This Changes

The issue asks for isolated sub-agent browser sessions to support Playwright-backed navigate, snapshot, and act consistently with the main session.

## Best Possible Solution

Define the supported runtime owner for isolated-session browser operations, then add regression coverage proving sandboxed sub-agent navigate/snapshot/act do not hit the Playwright-unavailable path when browser support is available.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection gives a medium-confidence reproduction path: isolated agent tools receive the sandbox bridge URL, default browser calls stay on that bridge, and navigate/act still require Playwright in that route runtime.

## Solution Assessment

Unclear because there is no proposed patch. The narrow maintainable solution needs an owner decision between making the sandbox bridge Playwright-capable and explicitly delegating to the host/node browser owner while preserving sandbox policy.
