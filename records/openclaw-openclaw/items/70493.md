- `P1`: The report describes a broken browser automation workflow for isolated sub-agent sessions, which can block real agent work rather than a cosmetic or speculative improvement.

## Summary

Keep open. Current main still defaults sandboxed browser tool calls to the sandbox bridge, the bridge uses the Playwright-gated browser routes, and the live discussion includes explicit reviewer direction plus a newer host-owned capability suggestion rather than a merged fix.

## What This Changes

Requests full browser navigate, snapshot, and act support in isolated sub-agent gateways so isolated sessions match the main session browser behavior.

## Best Possible Solution

Define the supported runtime owner for isolated-session browser operations, keep sandbox security policy explicit, and add regression plus real isolated-session proof that navigate or snapshot no longer hits the Playwright-unavailable route when browser support is available.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection gives a medium-confidence path: sandboxed agent tools receive a sandbox bridge URL, default browser calls stay on that bridge, and navigate/act still require Playwright in that bridge route runtime.

## Solution Assessment

Unclear because there is no proposed patch. The maintainable fix needs a product/owner decision between packaging Playwright into the isolated runtime and explicitly delegating browser control to the host or node that owns the browser capability.
