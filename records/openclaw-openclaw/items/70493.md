## Summary

Keep open. Current main still routes default isolated/sandboxed browser tool calls through the sandbox bridge, that bridge registers the normal Playwright-gated routes, and the live issue has explicit reviewer direction not to close until isolated-session browser coverage exists.

## What This Changes

The issue requests full Playwright-backed browser tool support for isolated sub-agent sessions so navigate, snapshot, and act behave like the main session.

## Best Possible Solution

Define the supported runtime owner for isolated-session browser operations, preserve sandbox security controls, and add regression coverage proving sandboxed sub-agent navigate or snapshot does not hit the Playwright-unavailable path when browser support is available.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection gives a medium-confidence path: sandboxed agent tools receive a sandbox bridge URL, default browser calls stay on that bridge, and navigate/act still require Playwright in the bridge route runtime.

## Solution Assessment

Unclear because there is no proposed patch. The maintainable solution needs an owner decision between making the sandbox bridge Playwright-capable and explicitly delegating to the host/node browser owner while preserving sandbox policy.
