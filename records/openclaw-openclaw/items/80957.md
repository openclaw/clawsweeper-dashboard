## Summary

Keep open: this is a valid, focused bug-fix PR with sufficient terminal proof and green exact-head checks, but the current patch drops `contextTokens` too broadly for `session_status(current)` and can regress sessions whose live model already matches the persisted model while the persisted snapshot is the only known context window.

## What This Changes

The PR updates `/status` and `session_status(current)` to ignore stale persisted `contextTokens` after model switches, preserve equivalent runtime aliases, and add focused regression tests.

## Best Possible Solution

Make the `session_status(current)` overlay drop persisted `contextTokens` only when the live identity actually differs from the stored identity, and add coverage for preserving valid same-identity runtime snapshots.

## Reproduction Assessment

Yes at source level: current main can be shown from `buildStatusMessage` to let a stale persisted runtime window drive `/status` after a selected/default model change. The review finding is also source-reproducible with `session_status(current)` on a same-identity custom model whose persisted `contextTokens` is the only known window.

## Solution Assessment

No, not as currently patched. The status-renderer direction is narrow and maintainable, but the `session_status(current)` helper should preserve valid persisted runtime snapshots instead of deleting them unconditionally.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Preserve valid live context snapshots:** `src/agents/tools/session-status-tool.ts:232`
  - body: When `session_status(current)` receives a live active model identity and the stored entry already matches that same model, deleting `contextTokens` discards the persisted runtime window. For custom or otherwise unconfigured live models, `buildStatusText` has no other window to resolve and falls back to the default 200k, regressing existing current-session status; only drop the field when the live identity differs from the stored identity, or pass a fresh live runtime window.
