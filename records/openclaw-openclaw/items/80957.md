## Summary

Keep open: the patch is useful, but one live `/status` wrapper path can still pass an active-model-derived stale runtime context window into the selected-model lookup, so the fix needs a small follow-up before merge.

## What This Changes

The PR updates `/status` and `session_status(current)` handling so stale persisted `contextTokens` are ignored after a selected/default model switch while equivalent runtime aliases and real fallback state are preserved.

## Best Possible Solution

Gate active-derived `runtimeContextTokens` out of the stale-runtime branch, add a full `buildStatusText` or native `/status` regression for the no-`contextTokens` caller path, and keep the existing alias-equivalence and real fallback behavior intact.

## Reproduction Assessment

Yes at source level: combine a selected 1.0m provider/model with stale `sessionEntry.modelProvider/model` for a 200k model and call `buildStatusText` without a `contextTokens` parameter, as the native `/status` path does. `buildStatusText` resolves `runtimeContextTokens` from the stale active model and the PR can still pass that value as the final override.

## Solution Assessment

No, not yet. The direction is narrow and maintainable, but the fix should also suppress `explicitRuntimeContextTokens` when `runtimeModelStale` is true or otherwise ensure wrapper callers cannot reintroduce the stale active model window.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Drop stale runtime tokens on stale model paths:** `src/status/status-message.ts:768-773`
  - body: `buildStatusText` computes `runtimeContextTokens` from `modelRefs.active` before `buildStatusMessage` detects `runtimeModelStale`. For native `/status` callers that do not pass a selected `contextTokens` value, this chain can still pass the stale active model's 200k window as `contextTokensOverride`, so a selected 1.0m model can keep rendering `10k/200k`. Gate `explicitRuntimeContextTokens` out for `runtimeModelStale` and add a `buildStatusText` or native `/status` regression for that path.
  - confidence: 0.86
