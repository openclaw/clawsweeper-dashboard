## Summary

Keep this PR open: it targets a real stale `/status` context-window problem, but the patch currently breaks an existing status regression test and the external contributor has not provided after-fix real behavior proof.

## What This Changes

The PR changes `/status` and `session_status(current)` context-window selection to ignore stale persisted `contextTokens` after model switches, with focused regression tests.

## Best Possible Solution

Keep the fix focused on stale model-switch status windows, but preserve runtime-equivalent alias behavior and require redacted real `/status` or `session_status` proof before merge.

## Reproduction Assessment

Yes for the PR regression: GitHub CI on the PR merge SHA fails the existing CLI runtime-alias status test. The original stale-context bug is also source-supported by related reports and the affected context-token resolution path, though I did not run a live session.

## Solution Assessment

No. The direction is narrow, but discarding persisted context tokens on raw selected/active label drift is too broad; it should distinguish stale model-switch metadata from runtime-equivalent aliases.
