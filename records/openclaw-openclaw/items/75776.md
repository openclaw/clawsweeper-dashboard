## Summary

Keep this PR open. The WebChat reliability direction is useful, but latest head still suppresses already-drained system events for the main WebChat session, so generic runtime events can be consumed and then never delivered to the model.

## What This Changes

The PR adds Control UI/WebChat preflight acknowledgements and pending-turn continuity, delays brief disconnect cleanup, adjusts reset transcript allocation and preflight compaction, threads tool allow-list state, and updates tests plus the changelog.

## Best Possible Solution

Land the reliability work after preserving main-session system-event delivery and adding a focused regression test for drained events on main WebChat sessions.

## Reproduction Assessment

Yes. On PR head, enqueue or mock a generic system event for sessionKey agent:main:main, run runPreparedReply, and assert the event reaches runReplyAgent commandBody or followupRun.prompt; the new suppression path removes it after drain.

## Solution Assessment

No. The overall reliability approach is reasonable, but suppressing every drained main-session event is broader than needed; filter only generated UI noise before draining or preserve the drained events in prompt delivery.
