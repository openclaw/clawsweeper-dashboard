## Summary

Keep open: the PR is useful WebChat reliability work, but the current head still introduces a blocking regression by consuming main-session queued system events and then suppressing them from the model prompt. Related merged PRs cover narrower active-send dedupe and compaction-boundary UX, but they do not supersede this broader branch.

## What This Changes

The branch adds WebChat preflight acknowledgement and pending-turn continuity, delayed Control UI disconnect cleanup, reset transcript path handling, earlier preflight compaction, tool-allow plumbing, prompt acknowledgement copy, tests, and a changelog entry.

## Best Possible Solution

Preserve queued system-event delivery for main sessions, keep the useful WebChat receipt/continuity pieces that still apply after rebasing, and drop or reconcile parts already covered by the narrower merged fixes.

## Reproduction Assessment

Yes. A focused source-level reproduction is to enqueue or mock a generic system event for agent:main:main, run runPreparedReply on PR head, and assert it still reaches commandBody or followupRun.prompt; the added suppression path drops it after drainFormattedSystemEvents consumes it.

## Solution Assessment

No. The WebChat reliability direction is reasonable, but suppressing every main-session system-event block is broader than needed; filter only the generated WebChat noise before draining, or avoid draining events that will not be delivered.
