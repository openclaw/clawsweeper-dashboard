## Summary

Keep this PR open: it addresses real WebChat reliability work, but the latest head still introduces a blocking regression by draining main-session system events and then suppressing them before delivery to the model.

## What This Changes

The PR adds WebChat preflight acknowledgements, pending-turn continuity, delayed disconnect cleanup, reset transcript handling, earlier preflight compaction, prompt/tool plumbing, focused tests, and a changelog entry.

## Best Possible Solution

Keep the WebChat reliability work, but preserve generic main-session system-event delivery and then reconcile the UI/history overlap with #74733 before merge.

## Reproduction Assessment

Yes. On PR head, enqueue or mock a generic system event for `agent:main:main`, run `runPreparedReply`, and assert it reaches `commandBody` or `followupRun.prompt`; source inspection shows it is drained and then suppressed.

## Solution Assessment

No. The reliability direction is plausible, but suppressing every drained main-session event is broader than needed; filter only generated UI noise before draining or display while leaving generic runtime events deliverable.
