## Summary

Keep open: the WebChat reliability goal is valid, but the current PR head still drops newly assigned terminal lifecycle fields before persistence and only supplies test-based proof rather than an observed WebChat/Gateway runtime path.

## What This Changes

The PR adds Gateway/Control UI WebChat preflight acknowledgement and pending-turn history continuity, session reset transcript selection changes, earlier preflight compaction checks, prompt guidance, and focused tests.

## Best Possible Solution

Persist fallback terminal lifecycle metadata without overwriting newer lifecycle events, keep the WebChat changes scoped to documented Gateway/UI contracts, and require a real Control UI/Gateway run proof before merge.

## Reproduction Assessment

Yes for the PR blocker: source inspection at PR head 2d68904f686a shows fallback lifecycle fields are deleted before persistence. I did not establish a live browser E2E reproduction of the original WebChat symptom.

## Solution Assessment

No. The WebChat reliability direction is plausible, but the patch is not ready while fallback lifecycle state cannot persist and the proof does not show the changed WebChat/Gateway runtime path.
