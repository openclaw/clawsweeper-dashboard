## Best Possible Solution

Land the WebChat reliability work only after preserving main-session system-event delivery with focused regression coverage for drained events on main WebChat sessions.

Yes. On PR head, enqueue or mock a generic system event for sessionKey agent:main:main, run runPreparedReply, and assert the event reaches runReplyAgent commandBody; the new suppression path drops it after drainFormattedSystemEvents consumes it.

## Solution Assessment

No. The reliability approach is reasonable, but suppressing every system-event block for main WebChat is broader than needed; filter only the unwanted generated noise before draining/display, or avoid draining events that will not be delivered.
