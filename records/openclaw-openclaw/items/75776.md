## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep repeated pending prompts visible:** `src/gateway/server-methods/chat.ts:1032-1041`
  - body: This matcher treats a persisted user message with the same normalized text and a timestamp within an hour as the pending turn. If a user sends the same prompt again while the first copy is already persisted, a `chat.history` refresh suppresses the pending overlay and the new prompt disappears until persistence catches up. Match by run/message identity, or use a stricter sequence/timestamp rule.
  - confidence: 0.91
- **[P2] Use canonical keys for pending history:** `src/gateway/server-methods/chat.ts:1883-1885`
  - body: `chat.send` stores pending messages under the canonical session key, but `chat.history` passes the raw request key into the pending overlay. Supported aliases like `main` can load the same `agent:main:main` session while missing the pending prompt this PR adds; use the canonical key from `loadSessionEntry` for matching.
  - confidence: 0.88
