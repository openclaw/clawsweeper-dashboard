## Summary

Keep this PR open because the current head still has a blocking regression: main-session system events are consumed and then suppressed before they reach the model. The branch is also currently dirty against `main`, so it needs repair and rebase before merge.

## What This Changes

The PR hardens Control UI/WebChat continuity with preflight acknowledgements, pending user-turn history, delayed disconnect cleanup, reset transcript handling, earlier compaction gates, prompt/tool plumbing, focused tests, and a changelog entry.

## Best Possible Solution

Keep the WebChat continuity work only after preserving the existing system-event contract: real queued events and new-session channel summaries must still reach the model, with any unwanted generated noise filtered narrowly or handled on a non-dropping path.

## Reproduction Assessment

Yes. On PR head, enqueue or mock a drained system event for `agent:main:main` and run `runPreparedReply`; the event is consumed, then omitted from `commandBody` and `followupRun.prompt` because the new suppression flag is true.

## Solution Assessment

No. The reliability direction is plausible, but suppressing all drained main-session events is not the narrow maintainable fix; preserve event delivery and filter only the specific generated noise if needed.
