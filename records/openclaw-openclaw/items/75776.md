## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Persist terminal lifecycle fallback:** `src/agents/command/session-store.ts:173-181`
  - body: The new fallback assigns status, endedAt, and runtimeMs, but the function later builds metadataPatch with removeLifecycleStateFromMetadataPatch(next), which deletes those fields before mergeSessionEntry. Because mergeSessionEntry preserves existing fields when the patch omits them, a stored status: "running" can survive and this fallback never marks the run terminal.
  - confidence: 0.94
- **[P3] Narrow the preflight acknowledgement opt-out:** `src/gateway/server-methods/chat.ts:1776`
  - body: Matching the standalone word "exactly" skips acknowledgements for normal prompts like "explain exactly why ..." even though those prompts are not asking for silence or strict output. That undermines the new reliability cue for ordinary WebChat requests; restrict this to explicit output-only or silence phrasing.
  - confidence: 0.86
