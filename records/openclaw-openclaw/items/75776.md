## Summary

Keep this PR open. The patch is not merge-ready because it changes core WebChat/session behavior without a filled problem statement and introduces concrete regressions in default tool availability, reset transcript continuity, and user-text sanitization.

## What This Changes

The PR changes main WebChat send/history behavior with preflight acknowledgements, pending user-message fallback state, runtime tool allowlist plumbing, preflight compaction changes, session reset transcript selection, inbound metadata stripping, Control UI streaming state, and related tests.

## Best Possible Solution

Split this into narrow, source-backed fixes: keep WebChat on configured tool policy unless a documented product setting is approved, preserve reset transcript continuity unless intentionally redesigned with tests/docs, suppress generated system-event prompt noise at its source, and add focused tests plus an active changelog entry for any user-visible WebChat behavior that remains.
