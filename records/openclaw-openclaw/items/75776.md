## Repair Work Prompt

On PR #75776, fix only the ephemeral preflight acknowledgement interaction with Control UI tool streaming. The bug is that `ui/src/ui/app-tool-stream.ts` commits any non-empty `chatStream` into `chatStreamSegments` when the first tool event starts, even when `chatStreamEphemeral === true`; this can preserve the gateway-generated "Got it..." preflight ack as a normal segment. Add a guard so ephemeral preflight text is cleared rather than appended, while real streamed assistant text still commits above tool cards. Add or update focused regression coverage in `ui/src/ui/app-tool-stream.node.test.ts` and/or `ui/src/ui/controllers/chat.test.ts` for preflight ack followed by tool start before any delta. Do not change gateway preflight wording, default tool capability policy, compaction thresholds, reset transcript handling, or broader #74733 ordering behavior. Changelog is already present unless the fix changes user-facing behavior beyond correcting this PR regression.
