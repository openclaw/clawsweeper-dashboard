## Summary

Keep this PR open: current main still has a source-reproducible gap where secondary-agent model discovery inherits default-agent auth state but not generated plugin model catalogs, and the current head adds a bounded local-first, per-provider catalog read-through with focused regression coverage. I found no line-level patch defect, but the change remains compatibility-, auth-provider-, and boundary-sensitive enough for maintainer acceptance before merge.

## What This Changes

The PR adds optional default-agent generated plugin model catalog inheritance to secondary-agent model discovery, threads that directory through cached discovery, updates cache fingerprinting, and adds ModelRegistry regression tests.

## Best Possible Solution

Land after maintainers accept the per-provider default-agent catalog inheritance boundary and normal CI/merge gates pass; otherwise defer to the broader lazy per-agent model discovery design.

## Reproduction Assessment

Yes, source-reproducible: current main reads generated plugin model catalogs only from the current agent directory while the embedded-agent path already has a default-agent inherited auth directory. I did not run a live failing gateway turn in this read-only review, but the PR body includes copied before/after live gateway output.

## Solution Assessment

Yes, this is an acceptable bounded fix for the normal gateway and CLI path because it reuses cached discovery and adds a local-first provider gate instead of running network discovery for every agent at startup. The larger per-agent discovery alternative is broader and better left as a follow-up unless maintainers reject catalog inheritance.
