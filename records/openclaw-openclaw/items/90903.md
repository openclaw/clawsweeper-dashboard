## Summary

No blocking patch defect found. This PR addresses a source-reproducible secondary-agent model catalog bug and has useful live-output proof, but it should stay open for maintainer acceptance of the compatibility-sensitive default-agent catalog inheritance boundary.

## What This Changes

The PR adds default-agent generated plugin model catalog read-through for secondary-agent model discovery, threads the inherited agent directory through cached discovery, expands cache fingerprinting, and adds ModelRegistry regression tests.

## Best Possible Solution

Land after maintainers explicitly accept the per-provider default-agent catalog inheritance boundary and normal CI/merge gates are satisfied; if that boundary is not acceptable, defer to a larger lazy per-agent model discovery design.

## Reproduction Assessment

Yes, source-reproducible: current main reads generated plugin model catalogs only from the current agent directory while the normal embedded-agent path already has a default-agent inherited auth directory. I did not run a live failing gateway turn in this read-only review.

## Solution Assessment

Yes, this is an acceptable bounded fix for the normal gateway/CLI path because it reuses the existing cached discovery owner and adds a local-first provider gate instead of running discovery for every agent at startup. The larger per-agent discovery alternative is broader and should be a follow-up only if maintainers reject catalog inheritance.
