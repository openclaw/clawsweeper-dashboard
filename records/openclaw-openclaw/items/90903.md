## Summary

Keep this PR open for maintainer review. Current main still lacks default-agent generated catalog read-through for secondary agents, and the patch has no concrete blocking code finding, but it intentionally changes a compatibility- and auth-sensitive cross-agent model resolution boundary.

## What This Changes

The PR adds default-agent generated plugin model catalog read-through for secondary agents, threads the inherited agent directory through discovery/cache fingerprinting, and adds ModelRegistry regression tests.

## Best Possible Solution

Land the gated catalog inheritance only after maintainer acceptance of the inherited-only provider boundary, keeping the local-owned provider isolation tests as part of the change.

## Reproduction Assessment

Yes. Current main's registry loads generated plugin catalogs only from the active agent directory, while runtime can create secondary-agent catalog state without model rows; the PR body also supplies after-fix real gateway output for the reported Google model case.

## Solution Assessment

Yes, as a bounded fix. Running model discovery for every agent would address the deeper root cause but is a larger startup/network design change; this PR's local-first per-provider gate is the narrower maintainable path if maintainers accept the inheritance boundary.
