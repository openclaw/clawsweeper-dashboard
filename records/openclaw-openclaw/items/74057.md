## Summary

Keep this PR open. The requested llama.cpp runtime-context fix is still useful and current main does not implement it, but the PR head still parses the wrong `/props` response shape and lacks the required user-facing changelog entry, so it should not merge as-is.

## What This Changes

The PR adds self-hosted OpenAI-compatible discovery of llama.cpp `/props`, maps `/v1/models` `meta.n_ctx_train` to native `contextWindow`, maps runtime context to `contextTokens`, and adds focused provider setup tests.

## Best Possible Solution

Revise the PR so runtime discovery reads `default_generation_settings.n_ctx` from llama.cpp `/props` responses, optionally keeping a top-level fallback for compatibility, while preserving explicit configured caps and `n_ctx_train` as native model metadata; cover that exact response shape in tests and add the required changelog fix entry.

## Reproduction Assessment

Yes. A high-confidence static reproduction is available: the PR head reads `data.n_ctx`, while current llama.cpp docs and source describe the runtime cap at `default_generation_settings.n_ctx`, so the added code will not populate `contextTokens` for real current responses.

## Solution Assessment

No. The product direction is narrow and maintainable, but the current implementation is not the best fix until it parses the documented nested `/props` shape and records the user-facing fix in the changelog.
