## Summary

Keep this PR open. It targets a real gap from the linked llama.cpp context-window bug, but the current PR head still reads the wrong `/props` response shape and omits the required changelog entry, so it should not merge as-is.

## What This Changes

The PR adds opportunistic llama.cpp `/props` discovery to self-hosted OpenAI-compatible provider model discovery, maps `/v1/models` `meta.n_ctx_train` to native `contextWindow`, and sets discovered runtime caps as `contextTokens`.

## Best Possible Solution

Revise this PR so self-hosted provider discovery parses the current llama.cpp `/props` shape, keeps `contextWindow` and `contextTokens` separate, preserves explicit configured caps, adds focused tests for the nested response, and records the user-facing provider fix in the changelog.

## Reproduction Assessment

Yes. A static reproduction is enough: feed the PR parser a current llama.cpp `/props` response with `default_generation_settings.n_ctx`; line 114 reads only `data.n_ctx`, so `contextTokens` remains unset and the linked session denominator bug is not fixed.

## Solution Assessment

No. The direction is the right narrow fix, but the implementation should read `default_generation_settings.n_ctx` and add a matching regression test; keeping a top-level fallback is acceptable for compatibility.
