- **[P1] Propagate the second ingest read failure:** `extensions/memory-wiki/src/ingest.ts:99`
  - body: If both reads fail here, `existing` becomes `""`, and the following write stores the regenerated page without the previous Notes block. A transient window lasting longer than 100 ms, or a persistent read error, still turns unreadable user data into “no prior page”; retry exhaustion should fail closed instead of overwriting notes.
  - confidence: 0.91
- **[P1] Propagate imported-page read exhaustion:** `extensions/memory-wiki/src/source-page-shared.ts:62`
  - body: This fallback keeps the same data-loss path for bridge and unsafe-local source pages: after two read failures, `nextRendered` is built from an empty existing page and the source-sync state can advance after writing it. The retry path should throw after exhaustion so an unreadable existing page is never treated as safe to replace.
  - confidence: 0.9
