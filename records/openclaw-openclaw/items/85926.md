## Summary

Keep open: current main and the latest release still add Copy-button chrome through the shared Control UI markdown renderer used for chat bubbles, so the Dashboard-visible multiline command corruption remains actionable.

## What This Changes

The issue asks Dashboard to render submitted multiline shell/heredoc prompts byte-faithfully without injected Copy text or indentation damage.

## Best Possible Solution

Make chat message rendering role-aware so user prompts preserve multiline shell/heredoc text without code-block Copy chrome while assistant/tool markdown and code-copy behavior stay intact.

## Reproduction Assessment

Yes for a source-level visible-render reproduction: current markdown rendering injects Copy-button markup and grouped chat uses that renderer for user text. I did not run a live WSL Dashboard session, and source inspection suggests the raw `chat.send` payload is still preserved.

## Solution Assessment

Yes, the narrow maintainable fix is a role-aware no-chrome/plain-preserve rendering path for user messages, not changing the gateway send path. The fix should add regression coverage while keeping assistant code-block copy controls.
