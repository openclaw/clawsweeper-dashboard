## Summary

Keep open: current main and v2026.5.22 still render user chat text through the shared Control UI markdown code-block renderer that injects visible Copy-button chrome into fenced and indented code blocks. Source inspection supports the visible Dashboard rendering bug; the send path still appears to pass the trimmed user message separately, so the stronger model-payload corruption claim is not proven yet.

## What This Changes

The issue reports that Dashboard chat rendering injects visible copy-button text into multiline shell/heredoc code blocks after submission.

## Best Possible Solution

Add role-aware rendering or a no-chrome/plain-preserve markdown mode for user chat messages, with regression coverage for multiline shell/heredoc text and existing assistant code-copy behavior.

## Reproduction Assessment

Yes for the visible Dashboard render path: current source and tests show user text goes through the markdown pipeline that injects Copy-button labels into code blocks. No live proof yet shows the Gateway or model receives mutated text.

## Solution Assessment

Yes, the maintainable fix is role-aware rendering or a no-chrome/plain-preserve mode for user messages. The `chat.send` payload path should stay unchanged unless live proof shows mutation outside rendering.
