## Label Justifications

- `P2`: This is a normal-priority Dashboard rendering regression with clear source evidence and limited UI blast radius, but it can make multiline command prompts unreliable from the Dashboard.

## Summary

Keep open: current main and v2026.5.22 still render user chat markdown through the shared code-block renderer that injects visible Copy/Copied chrome, and two open linked fix PRs now own the implementation path. This issue should close after one reviewed fix PR lands, not before.

## What This Changes

Dashboard should preserve user-submitted multiline shell/heredoc code blocks without injecting visible code-block copy chrome.

## Best Possible Solution

Land one focused fix that disables code-block copy chrome for user-authored chat markdown while preserving assistant copy UI, existing markdown normalization, and regression coverage.

## Reproduction Assessment

Yes for the visible Dashboard render path: current main routes user chat markdown through the code-block renderer that inserts Copy/Copied chrome. No live WSL packaged-Dashboard run proves the stronger claim that the agent/model receives mutated text.

## Solution Assessment

Yes, the narrow maintainable fix is a role-aware or no-chrome renderer mode for user messages while keeping assistant copy controls. The chat.send payload path should stay unchanged unless live proof shows input mutation outside rendering.
