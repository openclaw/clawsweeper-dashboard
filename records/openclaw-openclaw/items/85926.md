## Summary

Keep open: current main and the latest release still use the shared Control UI markdown code-block renderer for user chat text, and that renderer injects visible Copy-button chrome into fenced and indented code blocks. Source inspection points to a Dashboard rendering bug; the raw `chat.send` payload path still appears to send the trimmed user message separately.

## What This Changes

The issue asks for Dashboard chat rendering to preserve submitted multiline shell/heredoc text without injecting `Copy` text or damaging indentation.

## Best Possible Solution

Add role-aware rendering or a no-chrome/plain-preserve markdown mode for user chat messages, with regression coverage for multiline shell/heredoc text and existing assistant code-copy behavior.

## Reproduction Assessment

Yes for the visible Dashboard render path: current source and tests show user text is rendered through the markdown pipeline that injects Copy-button labels into code blocks. No live proof yet shows that the Gateway or model receives mutated text.

## Solution Assessment

Yes, the maintainable fix is to make rendering role-aware or add a no-chrome/plain-preserve mode for user messages. The `chat.send` payload path should stay unchanged unless live proof shows input mutation outside rendering.
