## Summary

Keep open: current main and v2026.6.9 still lack semantic or normalized-content duplicate suppression for Skill Research/Workshop proposals, and the requested duplicate policy needs maintainer product direction rather than cleanup closure.

## What This Changes

The issue requests semantic duplicate suppression for auto-created skill drafts whose current volatile proposal IDs and draft hashes can let repeated content create multiple pending drafts.

## Best Possible Solution

Define a lean Skill Research/Workshop proposal-novelty contract, likely normalized content plus skillKey before embeddings or counters, then add focused tests and docs.

## Reproduction Assessment

Yes, source-level: current main only suppresses auto-capture duplicates by exact skillKey and otherwise stores timestamped, randomly suffixed proposal identities. The exact reported 93-event log pattern was not live-reproduced because logs and draft samples are not attached.

## Solution Assessment

Unclear. Semantic embeddings and counters could reduce noise, but a deterministic normalized proposal identity may be the smaller and safer first contract for maintainers to approve.
