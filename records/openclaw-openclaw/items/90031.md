## Summary

Keep open: current main and latest release still define and document the V8 coverage gate, but the default trusted CI workflow does not run `pnpm test:coverage` or an equivalent coverage-threshold lane. The remaining blocker is maintainer rollout policy for CI cost, artifacts, and requiredness, not missing source-level evidence.

## What This Changes

Requested behavior is to add a default trusted validation gate that runs the existing coverage-threshold command and prevents it from silently disappearing.

## Best Possible Solution

Add a maintainer-approved workflow lane that calls `pnpm test:coverage`, uploads a human-readable coverage artifact, and adds a workflow guard test for that package-script call.

## Reproduction Assessment

Yes. Source inspection shows current main and `v2026.6.9` have the coverage scripts, docs, and thresholds, while the default CI workflow does not invoke `pnpm test:coverage` or an equivalent V8 coverage-threshold lane.

## Solution Assessment
