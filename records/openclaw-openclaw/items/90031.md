## Summary

Keep open: current main and v2026.6.8 still define the coverage command, docs, and V8 thresholds, but default trusted CI still does not run `pnpm test:coverage` or an equivalent coverage-threshold lane; the remaining choice is maintainer rollout policy and CI cost/noise tolerance.

## What This Changes

Add a default trusted validation coverage guard that runs `pnpm test:coverage` or a documented equivalent and prevents that guard from silently disappearing.

## Best Possible Solution

Add a maintainer-approved coverage lane that calls the package script, uploads human-readable/lcov artifacts, and adds a workflow guard test, starting advisory or main-only if needed.

## Reproduction Assessment

Yes. Source inspection shows the coverage script, docs, and thresholds exist, while current main and v2026.6.8 workflow searches do not find a default CI lane invoking that coverage command.

## Solution Assessment

Unclear until maintainers choose rollout policy. The narrow maintainable shape is to call `pnpm test:coverage` from CI rather than duplicating raw Vitest command details.
