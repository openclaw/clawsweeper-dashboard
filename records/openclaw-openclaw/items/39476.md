## Summary

Keep open. Current main still exposes the requester session to A2A target prompts and runs reply/announce nested agent steps as `sessions_send`, so the duplicate reverse-send behavior remains source-reproducible; the closed fix attempts were unmerged, and the open private-delivery PR is broader and not a shipped fix.

## What This Changes

The issue requests preventing an A2A target agent from using `sessions_send` back to the requester and duplicating delivery.

## Best Possible Solution

Add an internal scoped guard that hides or denies `sessions_send` during A2A target, reply, and announce runs while preserving inline primary-reply delivery and the existing peer A2A lifecycle.

## Reproduction Assessment

Yes. Source inspection shows the target prompt receives the requester session key and nested A2A reply/announce runs still execute as `sessions_send` without a scoped deny guard.

## Solution Assessment

Yes. A framework-level scoped guard is the narrowest maintainable fix; prompt guidance and private/dispatch delivery modes are broader or less reliable alternatives.
