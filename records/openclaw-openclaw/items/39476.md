## Summary

Keep open. Current main and the latest release still expose the requester session key in A2A prompts and run nested A2A target/requester steps without a scoped `sessions_send` denial, while the exact prior fix PR is closed unmerged and the open private/dispatch work is broader than this narrow duplicate-reply guard.

## What This Changes

This issue asks OpenClaw to prevent an A2A target agent from calling `sessions_send` back to the requester and duplicating cross-agent replies.

## Best Possible Solution

Add an internal scoped tool-availability guard for A2A target runs that blocks reverse `sessions_send` while preserving return-value delivery, provenance, per-target nested lanes, thread routing, and existing announce behavior.

## Reproduction Assessment

Yes. A high-confidence source reproduction path exists: the requester key is rendered into the target prompt, and nested A2A target/requester runs carry `sourceTool: "sessions_send"` without any scoped denial of the `sessions_send` tool.

## Solution Assessment

Yes. A framework-level scoped tool guard is the narrowest maintainable fix for this issue; prompt-only guidance is unreliable, and dispatch/private-delivery semantics belong to the separate broader work.
