## Summary

Keep open. Current `main` still exposes the requester session key to the A2A target and still runs A2A target/reply/announce steps without any scoped `sessions_send` denial; the exact fix PRs are closed unmerged, and the open private-delivery PR is broader than this narrow guard.

## What This Changes

The issue requests preventing an A2A target agent from sending a reverse `sessions_send` back to the requester and producing duplicate channel messages.

## Best Possible Solution

Add an internal scoped tool-availability guard for A2A target and nested runs that blocks reverse `sessions_send` while preserving return-value delivery and the existing A2A lifecycle.

## Reproduction Assessment

Yes, by source inspection. Current main gives the target prompt the requester session key and has no scoped exclusion that removes `sessions_send` from the target A2A tool surface.

## Solution Assessment

Yes. A framework-level scoped guard is the narrowest maintainable fix; prompt-only guidance is unreliable, and broader dispatch/private-delivery semantics belong to separate API work.
