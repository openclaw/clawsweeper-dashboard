## Summary

Keep open: current main and the latest release still route A2A `sessions_send` turns with `sourceTool: "sessions_send"` while exposing the normal `sessions_send` tool to the target run. ## What This Changes

This issue reports duplicate requester-channel messages when an A2A target can reverse-call `sessions_send` during a `sessions_send` flow.

## Best Possible Solution

Keep this as the canonical tracker until the open linked PR or an equivalent provenance-scoped guard lands with real A2A proof.

## Solution Assessment
