## What This Changes

The issue asks OpenClaw to prevent an A2A `sessions_send` target from reverse-calling `sessions_send` back to the requester and duplicating requester-channel output.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/91161, or an equivalent provenance-scoped guard, then close this issue after merged proof shows one requester-visible delivery and no reverse duplicate.

## Reproduction Assessment

No live high-confidence reproduction was run. Source-level evidence shows current main routes A2A turns with `inter_session` / `sessions_send` provenance while still exposing `sessions_send` to the target, and multiple users report the duplicate-delivery behavior.

## Solution Assessment

Yes, pending proof: a provenance-scoped guard that hides only `sessions_send` during A2A-routed turns is narrower than prompt-only mitigation or a broad dispatch-mode redesign.
