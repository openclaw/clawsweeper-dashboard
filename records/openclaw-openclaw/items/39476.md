## Summary

Keep open. Current `main` still gives A2A target/reply/announce runs requester session context and runs nested turns with `sourceTool: "sessions_send"` without a scoped capability guard, so the reverse-send duplicate-message bug remains actionable.

## What This Changes

The issue reports duplicate requester-channel messages when an A2A target uses `sessions_send` back to the requester during `sessions_send` handling.

## Best Possible Solution

Add an internal scoped capability guard for A2A target/reply/announce runs, preferably a `sessions_send` stub that returns a clear blocked tool error, while preserving inline primary-reply delivery and documented A2A lifecycle behavior.

## Reproduction Assessment

Yes for source-level reproduction: current `main` exposes requester session identity in A2A prompts and runs nested A2A turns as `sessions_send` without a scoped tool guard. I did not run a live multi-agent channel reproduction in this read-only review.

## Solution Assessment

Yes. A framework-level scoped guard is the narrowest maintainable fix; prompt-only guidance and private-delivery modes reduce symptoms but do not remove the target agent's reverse-send capability.
