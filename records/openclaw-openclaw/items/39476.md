## Summary

Keep open. Current main and the latest shipped release still expose the requester session key to the A2A target context and run reply/announce steps as `sourceTool: "sessions_send"` without a scoped guard that prevents reverse `sessions_send` calls or duplicate delivery.

## What This Changes

Prevent `sessions_send` A2A target sessions from calling `sessions_send` back to the requester and duplicating replies.

## Best Possible Solution

Add an internal scoped guard that hides or denies `sessions_send` during A2A target, reply, and announce runs while preserving inline primary-reply delivery and the existing peer A2A lifecycle.

## Reproduction Assessment

Yes, by source inspection. Current main exposes the requester session key to the target prompt and has no scoped tool guard on target/reply/announce A2A agent runs.

## Solution Assessment

Yes. A framework-level scoped guard is the narrowest maintainable fix; prompt-only guidance is unreliable, and private/dispatch delivery modes are broader product work.
