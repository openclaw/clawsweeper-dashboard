## Summary

Keep open. Current main and the latest release still expose the requester session key to the A2A target and run nested A2A steps without a scoped `sessions_send` denial, while the exact fix PR attempts are closed unmerged and the open dispatch-mode issue tracks a broader product mode.

## What This Changes

The issue asks OpenClaw to prevent an A2A target agent from using `sessions_send` back to the requester and causing duplicate channel messages.

## Best Possible Solution

Add an internal scoped tool-availability guard for A2A target runs that blocks reverse `sessions_send` while preserving normal return-value delivery and existing announce behavior.

## Reproduction Assessment

Yes. A high-confidence source reproduction path exists: the requester key is rendered into the target prompt, and nested A2A target/requester steps run without a scoped `sessions_send` denial.

## Solution Assessment

Yes. A framework-level scoped tool guard is the narrowest maintainable fix; prompt-only guidance is unreliable, and dispatch-only semantics belong to the separate broader feature request.
