## Summary

Keep open. Current main still exposes the requester session key to A2A target prompts and still runs nested reply/announce agent steps with `sourceTool: "sessions_send"`, without any scoped tool guard that prevents the target from calling `sessions_send` back to the requester.

## What This Changes

The issue asks OpenClaw to prevent an A2A target agent from using `sessions_send` back to the requester and duplicating the same result in the requester channel.

## Best Possible Solution

Add an internal scoped tool-availability guard that hides or denies `sessions_send` during A2A target and nested reply/announce runs, while preserving inline reply delivery and the existing peer A2A lifecycle.

## Reproduction Assessment

Yes, by source inspection. Current main exposes the requester session key to the A2A target and has no scoped exclusion that removes `sessions_send` from target or nested A2A runs.

## Solution Assessment

Yes. A framework-level scoped guard is the narrowest maintainable fix; prompt-only guidance is unreliable, while private-delivery and dispatch-only modes are broader API work.
