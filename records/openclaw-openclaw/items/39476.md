## Summary

Keep open. Current main still exposes the requester session key to the A2A target and still runs nested A2A reply/announce steps with `sourceTool: "sessions_send"` and no scoped denial of `sessions_send`; the exact fix PRs are closed unmerged, while the open dispatch/private-delivery work is broader than this narrow reverse-send guard.

## What This Changes

The issue asks OpenClaw to prevent an A2A `sessions_send` target agent from replying back to the requester via `sessions_send`, which can duplicate user-visible channel messages.

## Best Possible Solution

Add an internal scoped tool-availability guard for A2A target/nested runs that blocks reverse `sessions_send` while preserving return-value delivery and the existing A2A lifecycle.

## Reproduction Assessment

Yes, by source inspection. Current main gives the target prompt the requester session key and has no scoped exclusion that removes `sessions_send` from the target A2A tool surface.

## Solution Assessment

Yes. A framework-level scoped guard is the narrowest maintainable fix; prompt-only guidance is unreliable, and broader dispatch/private-delivery semantics belong to the separate open API discussions.
