## Summary

Keep open. Current `main` and the latest release still expose requester session identity to A2A target prompts and run reply/announce nested agent steps as `sessions_send` without a scoped deny guard; the direct fix PRs are closed/unmerged, and adjacent private-delivery or dispatch proposals do not close this reverse-send duplicate-message bug.

## What This Changes

The issue asks OpenClaw to prevent A2A `sessions_send` targets from calling `sessions_send` back to the requester and duplicating channel messages.

## Best Possible Solution

Add an internal scoped tool guard that hides or denies `sessions_send` during A2A target, reply, and announce nested runs while preserving inline primary-reply delivery and the documented A2A lifecycle.

## Reproduction Assessment

Yes for source-level reproduction: current `main` exposes the requester session key in A2A prompts and runs nested A2A turns as `sessions_send` without a scoped deny guard. I did not run a live multi-agent channel reproduction in this read-only review.

## Solution Assessment

Yes. A framework-level scoped guard is the narrowest maintainable fix; prompt instructions, private delivery, or a new dispatch mode are broader or less reliable alternatives.
