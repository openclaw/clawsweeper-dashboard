## Best Possible Solution

Define one maintainer-approved ACP token-only contract, preserve or deny scopes through an explicit safe client/config shape, document the final flag and security caveat, and require redacted runtime proof before merge.

## Solution Assessment

No. Passing explicit `null` is the right low-level GatewayClient opt-out, but this PR is not the best mergeable solution until it defines the scope-preserving ACP auth contract, docs, and real behavior proof.
