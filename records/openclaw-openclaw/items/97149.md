  - body: `tokenomics` depends on `ctx.internalDiagnostics?.onEvent`, but current main grants that capability only to trusted diagnostics exporter services. As written, the plugin can start and expose its route while recording no model spend.
  - confidence: 0.98
- **[P1] Reuse the canonical usage-cost boundary:** `extensions/tokenomics/index.ts:13-18`

Without a maintainer-approved boundary, operators can get conflicting accounting semantics from two sources of truth.
  - confidence: 0.91
- **[P1] Store spend data in SQLite-backed plugin state:** `extensions/tokenomics/src/ledger.ts:89-96`
  - body: The durable ledger appends `ledger.jsonl`, and the plugin also reads `pricing.json` under the state directory. OpenClaw-owned runtime/plugin state should use the SQLite-backed state contract unless maintainers approve these as named product artifacts, so this creates a second durability and migration path for spend accounting.
  - confidence: 0.95
- **[P2] Propagate ledger write errors through the adapter:** `extensions/tokenomics/src/host-adapter.ts:96`
  - body: The service sets up write-error logging only in `openLedger()`, but writes go through `HostAdapter`, which constructs a bare `Ledger` without `onError`. Disk-full or permission failures are swallowed, so spend entries can be dropped without the operator seeing the existing warning path.
  - confidence: 0.86
- **[P3] Keep missing optional pricing files quiet:** `extensions/tokenomics/src/pricing.ts:241-245`
  - body: `pricing.json` is documented as optional, but `PricingCatalog.load()` warns when the file cannot be statted, and the service loads that path at startup/report time. A normal install with no override catalog will produce warning noise for an expected state.
  - confidence: 0.8
