Full review comments:

- **[P1] Gate or add the diagnostics capability first:** `extensions/tokenomics/src/service.ts:155-160`
  - body: This service will not record any spend on current main. `startPluginServices` only grants `internalDiagnostics` to trusted `diagnostics-otel` and `diagnostics-prometheus` service ids, so `ctx.internalDiagnostics?.onEvent` is undefined here and startup returns after logging the unavailable-capability error. Add a maintainer-approved diagnostics consumer contract/grant before relying on this subscription.
  - confidence: 0.96
- **[P1] Move the spend ledger out of JSONL state files:** `extensions/tokenomics/src/ledger.ts:70-72`
  - body: The plugin's central accounting record is persisted as `ledger.jsonl` under the state dir, with a sidecar `pricing.json`. For a bundled OpenClaw plugin, root policy requires runtime state to use SQLite-backed storage such as plugin KV, the shared state DB, or a justified dedicated schema; a JSONL ledger creates migration and corruption debt for durable spend data.
  - confidence: 0.9
- **[P2] Include cached prompt tokens in ledger totals:** `extensions/tokenomics/src/service.ts:54-55`
  - body: Normal `model.usage` events already include `promptTokens` as `input + cacheRead + cacheWrite`. Taking `usage.input` first drops cached prompt tokens from `tokens_in`, so cached-heavy calls still carry billed cost but underreport total tokens, free/billed split, avoided spend, and counterfactual spend.
  - confidence: 0.94
- **[P2] Treat date-only until bounds as inclusive days:** `extensions/tokenomics/src/service.ts:99-101`
  - body: The docs advertise `until=2026-06-30`, but `Date.parse` turns that into midnight at the start of June 30. Because `entriesIn` keeps only rows with `t <= until`, all later usage on that date is excluded and the displayed window is short; parse date-only `until` as end-of-day while keeping `since` at start-of-day.
  - confidence: 0.93
- **[P3] Add the new plugin to labeler routing:** `extensions/tokenomics/package.json:2`
  - body: This adds a new `extensions/tokenomics` plugin surface but does not add a matching `.github/labeler.yml` entry, so future changes will not get extension-specific routing labels. Root policy asks new plugin surfaces to update labeler/GitHub labels along with the code surface.
  - confidence: 0.82
