## Summary

Keep open: this PR addresses a real agent runtime-safety problem and has useful screenshot proof, but current head still blocks exec/bash before `before_tool_call` hooks and ignores the `cmd` alias on that early path, so existing hook and alias flows can fail before merge.

## What This Changes

The branch adds empty exec/bash and `sessions_spawn` validation, a consecutive tool-error circuit breaker with runner steering/truncation recovery, and Control UI next-run formatting coverage.

## Best Possible Solution

Remove the pre-hook duplicate guard, validate only merged/effective params after trusted policies and hooks while preserving the `cmd` alias, then land the circuit breaker only with maintainer-approved default behavior and regression coverage for hook, alias, breaker, and truncation paths.

## Solution Assessment

No. The runtime-safety direction is useful, but the raw-params guard duplicates the effective-params helper and fires too early; the safer fix is to delete that early guard and keep validation after policy/hook merges.
