Full review comments:

- **[P2] Bypass the shared deploy cache for deploy modes:** `extensions/discord/src/monitor/provider.startup.ts:143-151`
  - body: This still passes the legacy shared `commandDeployHashStorePath` while also adding account-scoped `commandDeployInitialHashes`. Because `DiscordCommandDeployer.loadPersistedHashes()` runs before comparing hashes, an old bare `global:reconcile` entry can make another account skip REST, and `slashCommandDeploy: "always"` can still skip instead of reconciling each restart.
  - confidence: 0.91

  - confidence: 0.89

  - body: The docs now promise `commands.native=false` clears Discord native commands, but the monitor path never calls `clearDiscordNativeCommands`, and the provider test still asserts no clear log. Either implement the explicit clear path with coverage or keep the existing documented skip behavior.
  - confidence: 0.9
