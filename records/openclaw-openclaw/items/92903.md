## Summary

Keep open: current main and v2026.6.9 still lose the routed bound agent before inbound workspace/session resolution, and https://github.com/openclaw/openclaw/pull/95118 is the open closing fix candidate that should be reviewed instead of closing this issue early.

## What This Changes

The issue reports that bound multi-agent channel turns inject bootstrap files from the default workspace instead of the non-default agent workspace configured for the route.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/95118 if maintainers accept the corrected session boundary, or require a focused migration if preserving previously mis-associated default-agent records is necessary.

## Reproduction Assessment

Yes, source-level: configured bindings can carry a non-default route.agentId while current main drops that value from inbound context, and unscoped session resolution then falls back to the default agent. I did not run a live Feishu deployment in this read-only review.

## Solution Assessment

Yes on fix shape, but the issue should stay open until the linked PR is merged or rejected. The narrow maintainable solution is to use the routed agent only as a fallback below explicit and session-key agent ids, with maintainer policy on old mis-associated sessions.
