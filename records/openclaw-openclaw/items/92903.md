## Summary

Keep open: current main and v2026.6.8 still lose the bound route agent before workspace/bootstrap resolution, and the live closing PR remains open rather than merged.

## What This Changes

The issue reports that inbound turns routed to a non-default bound agent inject Project Context from the default workspace instead of that agent's configured workspace.

## Best Possible Solution

Review and land the active fallback-only fix, with an explicit maintainer decision on fresh-session versus migration behavior for previously mis-associated unscoped conversations.

## Reproduction Assessment

Yes, source-level: current main drops route.agentId from inbound context, and unscoped session keys then resolve to the default agent while workspace/bootstrap loading follows that resolved agent. I did not run a live Feishu deployment in this read-only review.

## Solution Assessment

Yes on fix shape: routed agent should be a fallback below explicit and session-key agent precedence, not an override. The remaining best-solution question is maintainer upgrade policy for previously mis-stored conversations.
