## Summary

Keep open: current main and the latest release still drop the bound route agent before session/workspace resolution, and the linked open PR is the active fix path but still needs fallback-only rework and proof before it can safely close this issue.

## What This Changes

The issue reports that channel-bound non-default agents inject Project Context bootstrap files from the default workspace instead of their configured per-agent workspace.

## Best Possible Solution

Land a fallback-only route-agent fix that preserves session-key precedence, aligns workspace and session-store resolution, and adds focused regression coverage before closing this issue.

## Reproduction Assessment

Yes, source-level: current main drops `route.agentId` from inbound context and then resolves unscoped session keys to the default agent before workspace/bootstrap loading. I did not run a live Feishu gateway reproduction because this was a read-only review.

## Solution Assessment

No for the currently open PR as written; carrying the route agent is the right boundary, but it must be fallback-only below session-key precedence. The best fix is the narrower fallback resolver shape with tests for both unscoped-bound and scoped-target sessions.
