## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Bound the TTS response before decoding audio:** `extensions/senseaudio/tts.ts:99-105`
  - body: `response.json()` buffers the whole upstream body and `Buffer.from(hexAudio, "hex")` then allocates decoded audio without any cap. A large SenseAudio or configured-endpoint response can exhaust memory; use the existing bounded response reader/content-length pattern and add oversized-response tests.
  - confidence: 0.9
- **[P2] Use the required provider flag in the docs:** `docs/providers/senseaudio.md:95`
  - body: The docs show `pnpm openclaw infer tts set-provider senseaudio`, but the CLI defines `set-provider` with a required `--provider <id>` option. As written, the setup command will fail for users following the page.
  - confidence: 0.92
- **[P1] Avoid auto-selecting TTS from the ASR key by default:** `extensions/senseaudio/speech-provider.ts:164-165`
  - body: `isConfigured` treats the existing `SENSEAUDIO_API_KEY` as enough to configure the new TTS provider. Since current TTS selection auto-scans configured providers, ASR-only users can start using SenseAudio for TTS after upgrade without opting into this provider; keep automatic selection opt-in or get explicit maintainer approval for that compatibility change.
  - confidence: 0.82
