Full review comments:

- **[P1] Bound SenseAudio TTS responses before decoding:** `extensions/senseaudio/tts.ts:99-105`
  - body: This new path parses the entire provider JSON body and then decodes data.audio into a Buffer without any byte cap. A bad upstream or configured endpoint can return a very large hex payload and force OpenClaw to allocate large JSON and audio buffers before failing; use the existing bounded provider JSON reader and add oversized-response coverage.
  - confidence: 0.88
- **[P1] Avoid auto-selecting TTS from the ASR key by default:** `extensions/senseaudio/speech-provider.ts:164-165`
  - body: isConfigured returns true from SENSEAUDIO_API_KEY even when no TTS provider config exists. Because current TTS selection falls back to the first configured provider, existing ASR-only environments with TTS auto enabled can start sending paid outbound SenseAudio audio after upgrade; require explicit TTS opt-in or get maintainer approval for this shared-key behavior.
  - confidence: 0.84
- **[P2] Use the required provider option in the docs:** `docs/providers/senseaudio.md:95`
  - body: The CLI defines tts set-provider with required --provider <id>, so this example will fail as written. Change it to pnpm openclaw infer tts set-provider --provider senseaudio while keeping the tts convert --model provider/model example unchanged.
  - confidence: 0.93
