## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Bound SenseAudio TTS responses before decoding:** `extensions/senseaudio/tts.ts:99-105`
  - body: `response.json()` reads the full upstream JSON body before this code validates `data.audio`, and then `Buffer.from(hexAudio, "hex")` allocates the decoded audio buffer. Adjacent TTS providers cap generated-audio responses with `readResponseWithLimit`, so a large SenseAudio or custom-endpoint response can consume unbounded memory before failing. Please read the response with a provider-owned max-byte cap and add an oversized-response regression test.
  - confidence: 0.9
- **[P1] Avoid auto-selecting TTS from the ASR key by default:** `extensions/senseaudio/speech-provider.ts:164-165`
  - body: `isConfigured` returns true from the existing `SENSEAUDIO_API_KEY` even when the user has not configured SenseAudio for TTS. Since TTS falls back to the first configured provider in auto-selection order, ASR-only users with `messages.tts.auto` enabled can start using paid outbound SenseAudio TTS after upgrade. Please require explicit TTS provider config or get maintainer approval for this shared-key auto-selection behavior.
  - confidence: 0.84
- **[P2] Use the required provider option in the docs:** `docs/providers/senseaudio.md:95`
  - body: The `set-provider` command declares `--provider <id>` as a required option, so this example will fail as written. Change it to `pnpm openclaw infer tts set-provider --provider senseaudio`; the separate `tts convert --model senseaudio/...` example is already using the correct selection form.
  - confidence: 0.93
