## Summary

Keep open: this maintainer draft is protected and the latest patch still has a concrete correctness issue. The direction is useful, and the PR body includes real Crabbox proof, but the implementation still estimates a line cap and can return an oversized first page for wide-line files under a small context budget.

## What This Changes

The PR lowers the agent read tool’s automatic page budget when a model context window is known and adds a focused small-context regression test.

## Best Possible Solution

Enforce the context-aware byte budget at the read/truncation boundary, preserve explicit `read(limit=...)` and no-context behavior, add a wide-line regression, and have maintainers explicitly accept the smaller implicit default for known small-context models.

## Reproduction Assessment

Yes for the review finding: source inspection shows a 16k-context run targets an 8KB budget, but the patch forwards an estimated line limit while the underlying read still truncates a page at the old 32KB default and the wrapper accepts an oversized first page before capping.

## Solution Assessment

No. The product direction is reasonable, but the current line-estimate approach should be replaced with direct byte-budget enforcement plus a regression using wider lines.
