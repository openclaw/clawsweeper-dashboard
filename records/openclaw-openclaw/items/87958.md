## Summary

## What This Changes

The PR lowers context-aware default `read` output for small-context agents, preserves explicit read limits, and adds focused regression tests for capped pagination edge cases.

## Best Possible Solution

Land after a maintainer explicitly accepts the smaller default read payload as the desired compatibility tradeoff, keeping explicit limits unchanged as the escape hatch.

## Reproduction Assessment

Yes, from source for the narrowed behavior. Current main computes a smaller context share for 16K-token models but clamps it to the 32KB floor; the PR lowers the context-aware floor to 8KB and adds focused tests, though I did not run tests in this read-only review.

## Solution Assessment

Yes, with a compatibility decision. The read wrapper is the narrowest maintainable layer because it already receives `modelContextWindowTokens` and owns adaptive paging; the live tool-result guard is a later context-protection layer, not a substitute for controlling the emitted read payload.
