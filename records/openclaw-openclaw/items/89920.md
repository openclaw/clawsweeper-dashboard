## Summary

The PR is still necessary and the shared toolCallId correlation is the right owner boundary, but the new Codex tool-result adapter infers native command completion from generic tool names and can replace unrelated plugin-tool progress; narrow that adapter before merge. The Matrix live proof convincingly demonstrates the original stale-failure bug and the desired transport edit.

The PR adds private tool-call correlation for command progress, propagates identity through Matrix and four sibling channels, and converts exec-like tool-result events into command-output completion callbacks.

## Best Possible Solution

Keep the private toolCallId correlation and cross-channel propagation, but remove the name-based generic tool inference or gate it on an explicit normalized command-result discriminator emitted by the Codex bridge, with focused native-command and dynamic-tool regression tests.
