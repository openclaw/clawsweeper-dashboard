## Summary

The Matrix defect and after-fix behavior are convincingly established, but this branch changes an observable stable identifier returned by an exported Plugin SDK builder. The recovery fix should remain open until a channel/Plugin SDK owner either preserves the existing identifier contract or explicitly approves and documents the compatibility change.

## What This Changes

The PR propagates tool-call identity through agent progress events and Matrix, Discord, Teams, Slack, and Telegram renderers so completed command output replaces an earlier command-progress line.

## Best Possible Solution

Keep the recovered-command replacement behavior, but preserve the existing public `ChannelProgressDraftLine.id` contract and carry cross-event replacement through a separate internal correlation key; alternatively, explicitly approve, document, and test the identifier migration as a Plugin SDK compatibility change.

## Reproduction Assessment

Yes. The supplied failed-progress/recovered-output sequence, Matrix regression, and real homeserver `m.replace` readback provide a high-confidence reproduction and verification path for the reported Matrix behavior.

## Solution Assessment

No, not without a compatibility decision. Correlating command events by Codex `call_id` is appropriate, but replacing the exported stable progress-line `id` is broader than necessary and can be avoided with a separate internal correlation field.
