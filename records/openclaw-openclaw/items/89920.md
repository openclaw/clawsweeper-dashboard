## Summary

This PR fixes a real stale-progress bug with convincing live Matrix proof and focused cross-channel coverage, and the current merge result cleanly excludes the earlier Docker/QA changes. The implementation is technically coherent, but it changes the stable `id` emitted by a public Plugin SDK progress helper for command-like events, so explicit owner approval of that compatibility tradeoff is still needed before merge.

## What This Changes

The PR propagates tool-call identity through agent progress events and Matrix, Discord, Teams, Slack, and Telegram renderers so completed command output replaces an earlier command-progress line, with Matrix QA and channel regressions.

## Best Possible Solution

Approve and document normalized command execution identity as the supported Plugin SDK behavior, or keep raw public IDs stable and move replacement correlation into an internal compositor key separate from the exported `id`.
