## Summary

Keep open. Current main still does not implement trust windows, but this PR is not mergeable as-is because the trust authority, localhost auth, hard-deny policy, and audit accounting paths have source-reproducible blockers, and the contributor has not provided real-runtime proof.

## What This Changes

This PR adds CLI `trust`, `untrust`, and `trust-status` commands with Gateway RPC/protocol and Swift model updates, in-memory trust-window state, JSONL audit logging, exec-host enforcement, and tests.

## Best Possible Solution

Define one maintainer-approved timed exec approval or trust model with server-owned authority, explicit scope, expiry, hard-deny semantics, audit guarantees, protocol/docs, tests, and redacted real-runtime proof before landing a single canonical implementation.

## Reproduction Assessment

Yes for review purposes. The merge blockers are source-reproducible from PR head and current main by inspecting the trust caller gate, WebSocket client IP handling, deny-policy resolution, and audit predicates.

## Solution Assessment

No. The timed-trust concept may be useful, but this patch is not the best mergeable form until Gateway-owned authority, shared-auth behavior, hard-deny semantics, audit accounting, and real behavior proof are settled explicitly.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Enforce trust authority on the Gateway:** `src/gateway/server-methods/exec-approvals.ts:127-130`
  - body: The trust RPC accepts CLI device identity, while the non-agent restriction is enforced only by client-side environment variables. A CLI process spawned from an agent can still present as the CLI, so the approval bypass needs a server-owned operator/session authority check before granting trust.
  - confidence: 0.86
- **[P1] Keep shared localhost auth usable for trust:** `src/gateway/server-methods/exec-approvals.ts:136`
  - body: This branch requires `isLoopbackAddress(client?.clientIp)` for token/password auth, but the WebSocket connection records `clientIp` as `undefined` for local clients. The no-device localhost auth path described by the PR is therefore rejected.
  - confidence: 0.88
- **[P1] Preserve deny policies as a hard ceiling:** `src/infra/exec-approvals.ts:624-628`
  - body: When a trust window is active, the resolver promotes the agent policy to `security=full` and `ask=off`. Current host policy treats `security=deny` as an execution block, so this bypasses an explicit deny policy unless maintainers intentionally redefine deny semantics.
  - confidence: 0.87
- **[P2] Record commands that started under trust:** `src/agents/bash-tools.exec.ts:570-574`
  - body: The audit append is gated on trust being active both when the command starts and when it finishes. A command that starts during the window but finishes after expiry or revoke ran under bypassed approvals but is omitted from the trust audit trail.
  - confidence: 0.9
- **[P3] Evict expired trust windows lazily:** `src/infra/exec-approvals.ts:171`
  - body: Expired entries remain in the module-level trust cache unless explicitly revoked or the gateway restarts. A long-running gateway with many transient agent IDs can accumulate stale trust records for the process lifetime.
  - confidence: 0.74
