## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Enforce trust authority on the Gateway:** `src/gateway/server-methods/exec-approvals.ts:127-130`
  - body: The trust RPC accepts a normal CLI client with device identity while the non-agent restriction is enforced only by client-side env/TTY checks. Since exec can spawn CLI-like processes, the approval bypass needs server-owned operator/session authority before opening trust.
  - confidence: 0.86
- **[P1] Keep shared localhost auth usable for trust:** `src/gateway/server-methods/exec-approvals.ts:136`
  - body: The shared-auth fallback requires a loopback clientIp, but local WebSocket clients have clientIp cleared for local connections. That rejects the token/password localhost path the PR advertises for CLI sessions without device identity.
  - confidence: 0.88
- **[P1] Preserve deny policies as a hard ceiling:** `src/infra/exec-approvals.ts:624-628`
  - body: When trust is active, the resolver promotes the agent policy to full/off. Current host resolution treats security=deny as a hard block, so this lets a trust grant bypass a configured exec denial unless maintainers intentionally redefine deny semantics.
  - confidence: 0.88
- **[P2] Record commands that started under trust:** `src/agents/bash-tools.exec.ts:570-574`
  - body: The audit append checks that trust is still active after the command finishes. A command that started under the bypass but completes after expiry or revoke can be omitted from the trust audit trail, weakening the feature's accountability story.
  - confidence: 0.9
- **[P3] Evict expired trust windows lazily:** `src/infra/exec-approvals.ts:171`
  - body: Expired entries remain in the module-level cache until explicit revoke or gateway restart. A long-running gateway that sees many transient agent IDs can accumulate stale trust records for the process lifetime.
  - confidence: 0.74
- **[P3] Use one preview limit in audit summaries:** `src/infra/trust-audit.ts:192-202`
  - body: The summary shows all commands up to 10 but drops to only 5 shown at 11 commands. Matching the threshold and preview limit avoids a confusing audit-summary cliff.
  - confidence: 0.7
