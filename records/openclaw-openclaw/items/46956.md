  - body: The RPC accepts a normal CLI client with device identity while the non-agent restriction is enforced only by client-side env/TTY checks. Since exec can spawn CLI-like processes, the approval bypass needs server-owned operator/session authority before opening trust.

  - confidence: 0.88
- **[P1] Preserve deny policies as a hard ceiling:** `src/infra/exec-approvals.ts:624-628`
  - body: When trust is active, the resolver promotes the agent policy to full/off. Current host resolution and tests treat `security=deny` as a hard block, so this bypasses a configured exec denial unless maintainers intentionally redefine deny semantics.
  - confidence: 0.88
- **[P2] Record commands that started under trust:** `src/agents/bash-tools.exec.ts:570-574`
  - body: The audit append checks that trust is still active after the command finishes. A command that started under the bypass but completes after expiry or revoke can be omitted from the trust audit trail, weakening the feature's accountability story.
  - confidence: 0.9

  - body: Expired entries remain in the module-level cache until explicit revoke or gateway restart. A long-running gateway that sees many transient agent IDs can accumulate stale trust records for the process lifetime.
  - confidence: 0.74
