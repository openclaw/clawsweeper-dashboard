## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Enforce trust authority on the Gateway:** `src/gateway/server-methods/exec-approvals.ts:127-130`
  - body: The trust RPC accepts a CLI device identity, while the non-agent restriction is only enforced by client-side environment variables in the CLI. Since that is not a server-owned security boundary, an agent-spawned CLI with operator credentials can self-grant the approval bypass; the grant needs Gateway-owned operator/session authority before opening trust.
  - confidence: 0.86
- **[P1] Keep shared localhost auth usable for trust:** `src/gateway/server-methods/exec-approvals.ts:136`
  - body: The shared-auth fallback requires isLoopbackAddress(client?.clientIp), but local WebSocket clients store clientIp as undefined. That rejects the token/password localhost path the PR explicitly advertises for CLI sessions without device identity.
  - confidence: 0.88
- **[P1] Preserve deny policies as a hard ceiling:** `src/infra/exec-approvals.ts:624-628`
  - body: When trust is active, the resolver rewrites the agent policy to full/off. Current host resolution treats security=deny as a hard execution block, so this lets a trust grant bypass an explicit deny policy unless maintainers intentionally redefine deny semantics.
  - confidence: 0.87
- **[P2] Record commands that started under trust:** `src/agents/bash-tools.exec.ts:570-574`
  - body: The audit append checks that trust is still active after the command finishes. A command that started under the bypass but completes after expiry or revoke can be omitted from the trust audit trail, weakening the feature's accountability story.
  - confidence: 0.9
- **[P3] Evict expired trust windows lazily:** `src/infra/exec-approvals.ts:200-201`
  - body: Expired entries stay in the module-level cache until explicit revoke or gateway restart. A long-running gateway that sees many transient agent IDs can accumulate stale trust records for the process lifetime; getTrustWindow should delete expired entries before returning.
  - confidence: 0.74
