Overall confidence: 0.89

Full review comments:

- **[P1] Register trust methods in core descriptors:** `src/gateway/server-methods-list.ts:28-30`
  - body: Current main no longer uses the old `BASE_METHODS` list as the source of truth; `listGatewayMethods()` advertises `CORE_GATEWAY_METHOD_SPECS`. After rebase, these methods need descriptors and scope tests there, otherwise the new RPC handlers will not be advertised or authorized consistently.
  - confidence: 0.9
- **[P1] Enforce trust authority on gateway-owned metadata:** `src/gateway/server-methods/exec-approvals.ts:124-128`
  - body: The server accepts any CLI client with device identity, while normal CLI calls load device identity by default and the repo does not populate the env vars used by the CLI guard. An agent that can spawn a TTY CLI can therefore look like an interactive CLI unless the Gateway checks a server-owned operator/session origin.
  - confidence: 0.86
- **[P1] Preserve hard-deny as a policy ceiling:** `src/infra/exec-approvals.ts:624-630`
  - body: This override changes an agent configured with `security: "deny"` into `security: "full"` during a trust window, contradicting the current host-policy invariant that config may tighten but must not silently broaden execution. Either reject trust for hard-deny agents or make that override an explicit maintainer-approved, documented, and tested policy.
  - confidence: 0.88
- **[P2] Use local-direct proof for shared auth:** `src/gateway/server-methods/exec-approvals.ts:136-138`
  - body: The PR allows token/password trust calls only when `client.clientIp` is loopback, but current WebSocket handling deliberately stores `undefined` for local direct clients. That makes the advertised loopback shared-auth path fail unless a device identity is also present.
  - confidence: 0.82
- **[P2] Audit commands that started under trust:** `src/agents/bash-tools.exec.ts:567-573`
  - body: The audit append re-checks that the trust window is still active after command completion, so a command that began while approvals were bypassed can disappear from the audit if it finishes after revoke or expiry. The start-time predicate is the accountability boundary the summary needs.
  - confidence: 0.86
