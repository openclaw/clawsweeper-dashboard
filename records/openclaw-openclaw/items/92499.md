Full review comments:

- **[P1] Keep imported sources out of MCPORTER_CONFIG:** `extensions/memory-core/src/memory/qmd-manager.ts:459-460`
  - body: `mcporter config get --json` can serialize imported servers with `source: { kind: "import", path: ... }`, but this helper returns the path regardless of kind. External mode later puts that value into `MCPORTER_CONFIG`, and mcporter treats that env var as a complete mcporter config file, so an imported auth/env-bearing `qmd` server can fail instead of using the user's real mcporter config/import layer. Only pin local mcporter config sources, or leave discovery env intact for imported sources.
