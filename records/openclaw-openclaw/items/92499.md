## Summary

Keep open: this is a useful fix candidate for the QMD/mcporter env leak, but the current head is not merge-ready because its raw mcporter source probe rejects supported commented `.json` configs that mcporter itself accepts.

## What This Changes

The PR routes bare QMD stdio servers through generated per-agent mcporter configs while preserving user/workspace-owned mcporter configs, with docs, doctor-contract scaffolding, and QMD manager tests updated around that behavior.

## Best Possible Solution

Keep the generated/external design, but make the raw mcporter probe use mcporter-compatible JSONC parsing for `.json` and `.jsonc`, or conservatively keep accepted-but-raw-unreadable sources external before merge.

## Reproduction Assessment

Yes. Current main is source-reproducible for the QMD-scoped env leak into mcporter, and the remaining PR defect is source-reproducible from mcporter's JSONC parser contract versus this PR's `.jsonc`-only fallback.

## Solution Assessment

No, not yet. The generated/external split is the right owner-boundary fix, but raw config probing must match mcporter's JSONC contract before this is the best mergeable solution.
