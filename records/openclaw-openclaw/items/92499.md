- **[P1] Do not mask explicit mcporter config failures:** `extensions/memory-core/src/memory/qmd-manager.ts:2759-2761`
  - body: Because discovery mode preserves `MCPORTER_CONFIG`, this catch also handles missing, unreadable, or invalid explicit config files.   - confidence: 0.87
- **[P1] Keep user XDG config in mcporter discovery:** `extensions/memory-core/src/memory/qmd-manager.ts:2901`
  - body: `this.mcporterEnv` is built from `process.env`, not from the QMD-scoped manager env, so deleting `XDG_CONFIG_HOME` here removes a user-supported mcporter config location. A user whose `qmd` server lives at `$XDG_CONFIG_HOME/mcporter/mcporter.json` will miss the external config, fall through to generated mode for the default server, and can search the wrong local qmd server.
  - confidence: 0.86
