- **[P1] Pass the caller signal into mcporter calls:** `extensions/memory-core/src/memory/qmd-manager.ts:3073-3080`
  - body: Current main relies on passing `opts.signal` into `runCliCommand` so timed-out `memory_search` calls SIGKILL the QMD/mcporter child. The new generated/external wrapper only forwards `timeoutMs`, so resolving the conflict by taking this code would let `mcporter call` keep running after the caller aborts; add `signal` to the mcporter wrapper options and thread `params.signal` through the call and retry paths.
  - confidence: 0.89
- **[P3] Document lifecycle as external-owned material:** `docs/concepts/memory-qmd.md:108-110`
  - body: The docs say external mode is used for env, auth headers, relative paths, or daemon logging, but the code also treats `lifecycle` as user-owned external material. Add lifecycle to the docs list so users with existing keep-alive/ephemeral definitions do not expect OpenClaw to rewrite those entries into generated mode.
  - confidence: 0.81
