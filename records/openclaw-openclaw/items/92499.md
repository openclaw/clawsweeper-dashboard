The PR changes QMD memory mcporter handling to generate per-agent mcporter configs for bare stdio QMD servers, keep user-owned mcporter definitions external, add a memory-core doctor contract stub, and document the integration.

## Best Possible Solution

Land the generated-config plus external-preservation approach after aligning config help/type docs and keeping the upgrade proof attached.
