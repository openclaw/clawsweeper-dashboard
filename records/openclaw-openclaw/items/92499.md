## Summary

Keep open: the PR is the current useful fix candidate for the QMD/mcporter env leak, but final head still has a compatibility bug where raw mcporter config probing does not match mcporter's JSONC parser for `mcporter.json`.

## What This Changes

The PR separates mcporter from QMD's per-agent process env, generates per-agent mcporter configs for bare QMD servers, preserves user-owned mcporter configs, and adds docs/tests for the routing behavior.

## Best Possible Solution

Preserve the generated/external design, but make raw mcporter config probing follow mcporter-compatible JSONC semantics before merge.

## Reproduction Assessment

Yes, source-reproducible: base and v2026.6.8 pass QMD-scoped XDG env into mcporter while mcporter 0.10 reads XDG_CONFIG_HOME for config discovery.

## Solution Assessment

No, not quite: the generated/external approach is the right overall shape, but raw config parsing must match mcporter's JSONC contract before this is the best fix.
