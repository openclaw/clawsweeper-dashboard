## What This Changes

Trusted-proxy Control UI WebSocket sessions behind nginx/Authelia lose operator scopes and fail normal read RPCs with `missing scope: operator.read`.

## Best Possible Solution

Land a narrow gateway auth fix that preserves verified trusted-proxy Control UI operator scopes, keeps untrusted and non-Control UI self-declared scopes clamped, and updates the existing regression coverage.

## Reproduction Assessment

Yes, there is a high-confidence source reproduction on current main: the trusted-proxy Control UI path can clear scopes before RPC authorization, and an existing test asserts the empty-scope result. I did not rerun the full nginx/Authelia setup live.

## Solution Assessment

Unclear until the linked patch is reviewed. The maintainable direction is to honor scopes only after verified trusted-proxy auth and origin checks, while proving non-proxy and non-Control UI clients still cannot self-assign operator scopes.
