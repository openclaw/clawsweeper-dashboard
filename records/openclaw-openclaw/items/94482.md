## Summary

Keep open: current main still drops `cacheRetention: "standard"` before it reaches Bedrock Claude stream options, and the two linked closing PRs are open implementation candidates rather than merged fixes.

## What This Changes

The issue reports that Bedrock Claude model params using `cacheRetention: "standard"` are dropped by the central cache-retention resolver instead of being honored or normalized.

## Best Possible Solution

Land one narrow resolver or config-validation fix that handles `standard` deterministically for Bedrock Claude, with regression coverage for Bedrock and sibling Anthropic cache-retention paths, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes from source: the central resolver only accepts `none`, `short`, and `long`, so `standard` falls through for Bedrock Claude and returns undefined after the family check. Adjacent docs and tests show Bedrock Claude is intended to honor explicit `cacheRetention` values, but no current-main test covers `standard`.

## Solution Assessment

Yes in boundary, but not yet on main: the maintainable fix belongs in the central cache-retention resolver or config validation/default path. One of the open linked PRs should land or be replaced before the issue closes.
