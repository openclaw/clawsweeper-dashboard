## Summary

Keep open: current main still drops `cacheRetention: "standard"` before it reaches Bedrock Claude request options, and the two linked closing pull requests are open implementation candidates rather than merged fixes.

## What This Changes

The issue reports that Bedrock Claude model params using `cacheRetention: "standard"` are silently discarded instead of mapping to a supported retention mode or surfacing validation.

## Best Possible Solution

Land one narrow resolver or config-validation fix that handles `standard` deterministically for Bedrock Claude, with regression coverage for Bedrock and sibling Anthropic cache-retention paths, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes at source level: calling `resolveCacheRetention({ cacheRetention: "standard" }, "amazon-bedrock", "openai-completions", "us.anthropic.claude-opus-4-7")` follows the current-main path to `undefined`. I did not execute a live Bedrock call in this read-only review.

## Solution Assessment

Yes in boundary, but not yet on main: the maintainable fix belongs in the central cache-retention resolver or config validation/default path. One of the open linked PRs should land or be replaced before the issue closes.
