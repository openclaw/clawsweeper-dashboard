## What This Changes

The issue asks OpenClaw to stop silently discarding explicit Bedrock Claude `cacheRetention: "standard"` config, either by normalizing it, changing emitted defaults, or warning on unsupported values.

## Best Possible Solution

Land one narrow resolver or validation fix that handles `standard` deterministically for Bedrock Claude, with regression coverage for Bedrock and sibling Anthropic cache-retention paths.
