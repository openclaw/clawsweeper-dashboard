Keep open: current main and the latest release still drop `cacheRetention: "standard"` before it reaches Bedrock Claude request options, and the two linked closing pull requests are open implementation candidates rather than merged fixes.

## What This Changes

The issue reports that Bedrock Claude model params using `cacheRetention: "standard"` are silently discarded by the runtime cache-retention resolver.

## Best Possible Solution

Land one narrow resolver or config-boundary fix that handles `standard` deterministically for Bedrock Claude, with regression coverage for Bedrock and sibling Anthropic cache-retention paths, then close after the fix reaches main.
