## Summary

Keep open: current main and latest release still drop explicit `cacheRetention: "standard"` before Bedrock Claude receives a resolved retention value, and two open closing pull requests already target the same resolver bug.

## What This Changes

The issue asks OpenClaw to stop silently discarding `cacheRetention: "standard"` for Bedrock Claude model params, either by normalizing it, changing defaults, or warning/erroring.

## Best Possible Solution

Land one narrow resolver or validation fix with regression coverage for Bedrock Claude and sibling cache-retention families, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes, at source level. Calling the current resolver with `cacheRetention: "standard"`, provider `amazon-bedrock`, model API `openai-completions`, and a Claude Bedrock model id follows the current-main path to `undefined`.

## Solution Assessment

Yes, the best fix boundary is the central cache-retention resolver or adjacent config validation/default path. The issue should stay open while maintainers review and land one of the linked pull requests or a narrow replacement.
