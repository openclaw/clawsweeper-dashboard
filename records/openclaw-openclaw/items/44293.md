## Summary

Keep open. Current main removed the literal Unix `xargs` pipeline, but `pnpm check:docs` still reaches a formatter path that sends every tracked docs temp path through one `oxfmt` spawn, so the native PowerShell acceptance criteria are still unresolved and the open formatter PR remains the active fix path.

## What This Changes

The issue asks for `pnpm check:docs` to work in native Windows PowerShell without Unix-only shell assumptions.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/48887 or an equivalent maintained patch that batches Windows-sensitive `oxfmt` invocations, preserves tracked-doc semantics, adds focused regression coverage, and validates `pnpm check:docs` on native PowerShell and a Unix shell.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: run `pnpm check:docs` or `pnpm format:docs:check` on native Windows, where current main sends 622 temp docs paths through one formatter process.

## Solution Assessment

Yes. The narrow maintainable fix is to keep the Node formatter path, batch the Windows-sensitive `oxfmt` launches, and add focused tests; the open related PR is the current candidate for that direction.
