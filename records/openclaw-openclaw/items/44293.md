## Summary

Keep open. Current main removed the literal Unix `xargs` pipeline, but `pnpm check:docs` still routes check mode through one unbatched `oxfmt` spawn with every tracked docs temp path, so the native PowerShell acceptance criteria remain unresolved and the open formatter PR is still relevant.

## What This Changes

Request that `pnpm check:docs` use shell-agnostic docs formatting so native Windows PowerShell works without Git Bash or `xargs`.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/48887 or an equivalent maintained patch that batches Windows-sensitive `oxfmt` invocations, preserves tracked-doc semantics, adds regression coverage, and validates `pnpm check:docs` on native PowerShell and a Unix shell.

## Reproduction Assessment

Yes, there is a high-confidence source-level reproduction path: run `pnpm check:docs` or `pnpm format:docs:check` on native Windows, where current main sends 622 temp docs paths through one `oxfmt` process. I did not perform a live Windows run in this read-only Linux review.

## Solution Assessment

Yes, the related PR direction is the narrowest maintainable fix: keep the Node formatter, batch the Windows-sensitive `oxfmt` launches, and add focused tests. Current main is only a partial solution because it removed `xargs` without addressing the single-spawn argument-size exposure.
