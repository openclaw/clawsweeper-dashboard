## Summary

Keep open. Current main removed the literal Unix `xargs` pipeline, but the replacement docs formatter still invokes `oxfmt` once with every tracked docs temp path in check mode. With 463 tracked docs files, the Windows process argument-length concern remains plausible, the latest release still ships the old `xargs` scripts, and open PR #48887 is still the active related implementation candidate.

## What This Changes

Make `pnpm check:docs` run in native Windows PowerShell without requiring Unix `xargs`, while preserving docs formatting checks on Linux and macOS.

## Best Possible Solution

Land or adapt a cross-platform docs formatter that chunks `oxfmt` invocations under Windows command-line limits and includes native PowerShell validation; after that ships, close this issue as implemented and keep #44291 for the broader contributor-command smoke coverage follow-up.
