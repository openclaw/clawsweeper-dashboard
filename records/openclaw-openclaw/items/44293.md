## Summary

Keep open. Current main has removed the literal Unix `xargs` pipeline, but `pnpm check:docs` still funnels the docs formatter through one unchunked `spawnSync` with every tracked docs temp path, so the native PowerShell/Windows acceptance criteria are not proven satisfied and the open implementation candidate #48887 remains relevant.

## What This Changes

Make `pnpm check:docs` run reliably from native Windows PowerShell without relying on Unix-only tooling or oversized formatter invocations.

## Best Possible Solution

Finish the Node docs formatter with chunked `oxfmt` invocations under a Windows-safe argument budget, verify `pnpm check:docs` in native PowerShell and a Unix shell, then reconcile #48887 and keep broader PowerShell smoke coverage in #44291.

## Reproduction Assessment

No, not as a live high-confidence reproduction in this review. The original `xargs` error no longer matches current main, but source inspection shows the current check path still sends 597 temp paths to one Windows-sensitive process launch.

## Solution Assessment

No, current main is only a partial solution because it removes `xargs` while leaving the formatter unchunked. The narrow maintainable fix is to batch formatter invocations and keep the existing docs file selection semantics.
