## Summary

Keep open. Current main removed the literal xargs pipeline, but `pnpm check:docs` still routes through a docs formatter that sends all tracked docs temp paths to one `oxfmt` process, leaving the native Windows PowerShell acceptance criteria unresolved; the active fix path is https://github.com/openclaw/openclaw/pull/48887.

## What This Changes

Make the docs check command run successfully in native Windows PowerShell without requiring Unix xargs or Git Bash.

## Best Possible Solution

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: run `pnpm check:docs` or `pnpm format:docs:check` on native Windows PowerShell, where current main still sends hundreds of temp docs paths through one formatter command.

## Solution Assessment

Yes. The narrow maintainable fix is to keep the Node formatter but batch Windows-sensitive `oxfmt` invocations, preserve tracked-doc filtering, add focused regression coverage, and prove the command in native PowerShell.
