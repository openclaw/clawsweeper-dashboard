## Summary

Keep open: current main and v2026.6.1 replaced the old `xargs` pipeline with a Node formatter, but the implementation still invokes `oxfmt` once with every tracked docs path, and the computed argument string is about 42.8K characters for 662 docs files, so the native PowerShell acceptance criterion remains unproven and likely still needs chunking.

## What This Changes

Request is to make `pnpm check:docs` run in native Windows PowerShell without relying on Unix `xargs` while preserving docs formatting checks on other platforms.

## Best Possible Solution

Keep the Node-based docs formatter, add Windows-safe argument chunking and focused coverage, and leave the broader contributor-command PowerShell smoke matrix to https://github.com/openclaw/openclaw/issues/44291.

## Reproduction Assessment

No high-confidence current-main reproduction of the original `xargs` error remains because current source no longer invokes `xargs`. Source inspection does show the remaining Windows risk: all 662 docs paths are passed to one child process, producing about 42.8K argument characters before Windows quoting overhead.

## Solution Assessment

No. The current Node formatter is the right direction, but calling `oxfmt` once with all docs paths is not the best finish for Windows; a bounded chunking helper plus native PowerShell proof is the safer solution.
