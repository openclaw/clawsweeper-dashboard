## Summary

Keep open. Current main removed the literal `xargs` pipeline, but `pnpm check:docs` still reaches a single unchunked `spawnSync` call with every copied docs path, and the current tracked docs set makes a Windows command-line-length failure plausible rather than resolved. The related open PR #48887 remains the right reconciliation point before this can be closed as implemented.

## What This Changes

The issue asks for `pnpm check:docs` to work in native Windows PowerShell by replacing the Unix-only docs formatting pipeline while preserving the same docs checks on other platforms.

## Best Possible Solution

Finish the current Node formatter by chunking `oxfmt` invocations under a Windows-safe argument budget, verify `pnpm check:docs` in native PowerShell and a Unix shell, then reconcile or close the related implementation PRs and leave the broader PowerShell smoke coverage to #44291.

## Reproduction Assessment

Unclear for current main. The original `xargs` failure no longer matches the current script, but a high-confidence remaining repro would be `pnpm check:docs` in native Windows PowerShell with the current 471 tracked docs files and the unchunked temp-path invocation.

## Solution Assessment

No, current main is not yet the best complete solution because it removes `xargs` but still launches one formatter process with every docs path. A chunked Node implementation is the narrow maintainable fix boundary.
