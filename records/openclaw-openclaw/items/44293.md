## Summary

Keep open. Current main removed the literal Unix `xargs` package-script pipeline, but `pnpm check:docs` still routes all tracked docs temp paths through one unchunked `oxfmt` process, so the native PowerShell acceptance criteria are not proven and the open implementation PR remains relevant.

## What This Changes

The issue requests making `pnpm check:docs` run in native Windows PowerShell without requiring Unix-only shell tools.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/48887 or an equivalent maintained patch that batches Windows-sensitive `oxfmt` invocations, preserves the current tracked-docs semantics, adds regression coverage, and verifies `pnpm check:docs` in native PowerShell and a Unix shell.

## Reproduction Assessment

No live high-confidence Windows reproduction was established in this read-only review. Source inspection does show the current check-mode path still sends all 622 docs temp paths through one process launch, which is enough to keep the issue open.

## Solution Assessment

No, current main is only a partial solution: it removed `xargs` but did not add Windows-safe batching or focused formatter tests. The narrow maintainable fix is the open related PR's direction or an equivalent patch on the same helper.
