## Summary

Keep open. Current main removed the literal Unix `xargs` pipeline, but `pnpm check:docs` still routes all tracked docs temp paths through one unchunked formatter process, so native PowerShell/Windows compatibility is not proven and the open implementation candidate #48887 remains relevant.

## What This Changes

Make `pnpm check:docs` run from native Windows PowerShell without relying on Unix-only tooling or shell-sensitive argument handling.

## Best Possible Solution

Finish the Node docs formatter with chunked `oxfmt` invocations under a Windows-safe argument budget, verify `pnpm check:docs` in native PowerShell and a Unix shell, then reconcile #48887 and keep broader PowerShell smoke coverage in #44291.

## Reproduction Assessment

No live high-confidence reproduction was established in this review. The original `xargs` error no longer matches current main, but source inspection shows `--check` still sends 597 temp paths to one Windows-sensitive process launch.

## Solution Assessment

No, current main is only a partial solution because it removed `xargs` while leaving formatter invocation unchunked. The narrow maintainable fix is to batch formatter runs while preserving the existing tracked-docs selection semantics.
