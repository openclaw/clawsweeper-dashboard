## Summary

Keep open. Current main removed the literal Unix `xargs` pipeline, but `pnpm check:docs` still runs all tracked docs temp paths through one `oxfmt` spawn, so the native PowerShell acceptance criteria remain unresolved and the open formatter PR is still relevant.

## What This Changes

Make the repository docs check run from native Windows PowerShell without relying on Unix-only `xargs` or oversized formatter invocations.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/48887 or an equivalent maintained patch that batches Windows-sensitive `oxfmt` invocations, preserves tracked-doc semantics, adds regression coverage, and validates `pnpm check:docs` on native PowerShell and a Unix shell.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: run `pnpm check:docs` or `pnpm format:docs:check` on native Windows, where current main sends 622 temp docs paths through one `oxfmt` process; I did not perform a live Windows run in this review.

## Solution Assessment

Yes. The narrow maintainable fix is to keep the Node formatter path, batch the Windows-sensitive `oxfmt` launches, and add focused tests; the open related PR is the current candidate for that direction.
