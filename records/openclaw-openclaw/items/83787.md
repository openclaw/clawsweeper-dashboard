Full review comments:

- **[P1] Avoid completing setup from any subdir profile file:** `src/agents/workspace.ts:523-525`
  - body: This returns true after finding any one `SOUL.md`, `IDENTITY.md`, or `USER.md` under hard-coded `main`, `agents`, or `workspace`, then skips root bootstrap writes and marks setup complete. A workspace repo with an unrelated `main/USER.md` can stop getting required `AGENTS.md`/`TOOLS.md` setup, and this bypasses the existing optional-bootstrap contract; tie the fix to the actual spawned workspace path/state instead of directory-name probing.
  - confidence: 0.86
- **[P2] Preserve identityPathCreated when seeding IDENTITY.md:** `src/agents/workspace.ts:551-552`
  - body: When no subdirectory bootstrap files exist, the patch writes `IDENTITY.md` inside the block and then calls `writeFileIfMissing` again to compute `identityPathCreated`, so the second call returns false even for a freshly created identity file. `agents.ts` uses this flag to preserve previous workspace identity content during workspace moves, so keep the first write result instead of writing twice.
  - confidence: 0.9
- **[P2] Keep Docker daemon failures fail-fast:** `src/agents/sandbox/docker.ts:307-309`
  - body: This PR also returns from `ensureDockerImage` when the Docker daemon is unavailable. That helper is called while creating a Docker sandbox, and current main throws a targeted error telling users to start Docker or disable sandboxing; returning here makes enabled sandbox failures surface later as generic Docker operations and is unrelated to the workspace fix. Drop the stale sandbox commits from this PR.
  - confidence: 0.82
