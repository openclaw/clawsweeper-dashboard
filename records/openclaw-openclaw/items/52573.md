## Summary

Keep this open. Current main still lets the Kimi web_search provider return final chat-completion text as a successful search payload when no grounding evidence or citations were collected; the merged setup/replay work and docs clarification are adjacent but do not implement the requested fail-closed behavior.

## What This Changes

Request that Kimi web_search preserve the search-tool contract by returning grounded search output or an explicit provider failure instead of generic chat-style answers.

## Best Possible Solution

Implement the guard inside the Moonshot plugin runtime: treat Kimi output as successful only when native $web_search produced grounding evidence such as search_results, citation URLs, or replayable tool-result payloads, and otherwise return a structured Kimi provider failure with focused regression tests and docs updates if the visible behavior changes.

## Reproduction Assessment

Yes. A mocked Kimi chat-completions response with final text, no search_results, and no replayable tool calls follows the current line 212 or line 239 success paths and reproduces the reported contract break without a live API key.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Moonshot provider runtime rather than core, because Kimi-specific grounding semantics should stay plugin-owned while core continues to pass provider results through generically.
