## Summary

Keep open. Current main still has a source-level config patch bug for numeric-looking Discord guild IDs, and the Discord token SecretRef startup report has enough related history and current code evidence to need a focused runtime-snapshot check rather than cleanup closure.

## What This Changes

The issue reports that Discord token SecretRefs can skip Discord startup and that config patch recursive merge mishandles Discord guild ID object keys.

## Best Possible Solution

Preserve numeric-looking map keys as object keys in config patch operations with a Discord snowflake regression, then separately verify Discord token SecretRefs through startup/runtime snapshot activation and improve diagnostics if unresolved refs still skip startup.

## Reproduction Assessment

Partly: the config patch failure is source-reproducible from buildConfigPatchOperations plus setAtPath creating arrays for numeric-looking path segments. The Discord token failure still needs a focused current-main gateway/runtime-snapshot reproduction because current secrets runtime registers Discord token SecretRef assignments.

## Solution Assessment

Unclear for the bundled issue as a whole. The config patch fix should be a narrow path/merge repair, while the Discord token side should first prove whether runtime snapshot activation or direct token inspection is the failing layer.
