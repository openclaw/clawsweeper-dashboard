## Summary

Keep open: the Discord token SecretRef half appears addressed on current main, but the companion config mutation bug is still source-reproducible because numeric object keys can create arrays where the Discord schema requires an object map.

## What This Changes

The issue asks to fix Discord SecretRef token resolution and config patch/config set handling for 19-digit Discord guild ID object keys.

## Best Possible Solution

Keep the shipped Discord runtime SecretRef behavior and fix config mutation path construction so numeric string keys under object maps stay object keys, with regression coverage for Discord snowflake guild IDs.

## Reproduction Assessment

Yes, at source level: `config patch` flattens JSON object keys into path segments, then `setAtPath` creates arrays for numeric-looking next segments, while the Discord schema requires `guilds` to be an object record. I did not run a mutating CLI repro in this read-only pass.

## Solution Assessment

No, closing is not the best outcome because only the Discord token half appears fixed; the maintainable fix is to preserve object-key intent in config patch/config set paths without breaking real array-index behavior.
