## Best Possible Solution

Land the narrowed parent resolver if maintainers accept `sessionId` as the persisted-parent signal, while leaving broader naming UX to https://github.com/openclaw/openclaw/issues/93422 and https://github.com/openclaw/openclaw/pull/95700.

## Solution Assessment

Yes, if maintainers accept the lineage semantics. The Control UI create-session boundary is the narrowest maintainable layer because it can distinguish UI placeholders from persisted rows before building the gateway request.
