## Summary

Keep open for maintainer review: the narrowed PR addresses a real Control UI session-lifecycle bug, is not implemented on current main or the latest release, and I found no concrete patch defect. The remaining decision is whether maintainers accept suppressing `parentSessionKey` and command hooks for fallback or explicitly non-persisted visible rows.

## What This Changes

The PR changes Control UI new-chat creation so fallback or non-persisted current session rows are not sent as `parentSessionKey`, with regression coverage for `/new`, `/label`, and parent resolution cases.

## Best Possible Solution

Land the narrowed parent guard if maintainers accept the session-lineage boundary, while keeping broader session naming work on the existing linked naming PRs.

## Reproduction Assessment

Yes. Source inspection shows current main and v2026.6.9 send the visible session key as `parentSessionKey` except for `unknown`, while the gateway rejects supplied parents that lack `sessionId`.

## Solution Assessment

Yes, with maintainer acceptance of the session-state boundary. The Control UI create-session boundary is the narrowest maintainable layer because it can distinguish placeholder rows before building the gateway request while preserving persisted parent behavior.
