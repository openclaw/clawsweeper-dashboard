## Summary

Keep open: current main still has the broad descendant selectors, so the PR is addressing a real Control UI bug. The production CSS approach is narrow, but the added nested-chevron browser test can silently pass if the chevron query returns null, so the patch is not ready to merge as-is.

## What This Changes

The PR narrows Control UI config and pending-diff chevron rotation selectors to direct details headers and adds browser tests for nested config objects and pending-change diff panels.

## Best Possible Solution

Land this PR, or an equivalent replacement, after correcting the nested chevron test guard so missing DOM fails loudly. The desired end state is the direct-header CSS selector fix plus focused browser coverage for both nested config chevrons and pending-change diff chevrons.

## Reproduction Assessment

Yes. Current main has a source-level reproduction path: open parent `.cfg-object` details elements can contain closed child `.cfg-object` details, and the descendant selector rotates the child chevron based on the ancestor’s open state.

## Solution Assessment

No, not as-is. The CSS change is the narrowest maintainable production fix, but the regression test should use a null check for the `querySelector` result before this is merge-ready.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Fail when the nested chevron query returns null:** `ui/src/ui/config-form.browser.test.ts:512`
