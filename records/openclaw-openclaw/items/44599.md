## Summary

Keep open: current `main` still rejects whitespace in Docker setup config/workspace paths, the latest release contains the same validator, and an open closing PR already targets this issue. A new reporter also confirmed on May 1, 2026 that they are still seeing the failure.

## What This Changes

The issue reports that Docker setup rejects `OPENCLAW_CONFIG_DIR` paths containing whitespace on Windows, blocking installation from Git Bash.

## Best Possible Solution

Land #44671 or an equivalent focused Docker setup fix that allows ordinary spaces in config/workspace paths, quotes generated compose volume entries where needed, keeps unsafe mount syntax rejected, and adds e2e coverage for spaced paths.

## Reproduction Assessment

Yes. A high-confidence static reproduction is setting `OPENCLAW_CONFIG_DIR` or `OPENCLAW_WORKSPACE_DIR` to a path containing a space before running `scripts/docker/setup.sh`; current source routes those values to `validate_mount_path_value` and fails on `[[:space:]]`. I did not execute the script because this review is read-only and the setup script writes env/compose files.

## Solution Assessment

Yes, the narrow Docker setup repair proposed by the linked closing PR is the right direction. Support-only workarounds or a broad config-dir migration would not fix the deterministic validator and compose-mount behavior that blocks installation.
