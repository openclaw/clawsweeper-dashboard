## What This Changes

The issue reports Docker setup failing on Windows/Git Bash when `OPENCLAW_CONFIG_DIR` or related bind-mount host paths contain spaces.

## Best Possible Solution

Allow ordinary spaces in Docker config/workspace bind-mount source paths, safely quote generated Compose mounts, preserve control-character/YAML-injection guards, and add focused regression coverage plus real Docker/Git Bash proof.

## Reproduction Assessment

Yes. Current main resolves `OPENCLAW_CONFIG_DIR` and `OPENCLAW_WORKSPACE_DIR` and then `validate_mount_path_value` rejects `[[:space:]]`; I did not execute the setup script because this review is read-only.

## Solution Assessment

Yes. The maintainable fix is narrow in `scripts/docker/setup.sh`: allow ordinary spaces, quote generated Compose mounts, preserve injection guards, and add regression coverage; the open linked PR should be reviewed/refreshed or closed before starting duplicate work.
