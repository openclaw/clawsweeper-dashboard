The issue requests that Docker setup accept `OPENCLAW_CONFIG_DIR` and `OPENCLAW_WORKSPACE_DIR` paths containing ordinary whitespace, especially Windows/Git Bash home paths.

## Best Possible Solution

Land a focused fix in `scripts/docker/setup.sh` that allows ordinary spaces in valid host paths, safely emits generated Compose mounts, preserves injection guards, and adds regression coverage.
