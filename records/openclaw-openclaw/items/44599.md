## Summary

Keep open: current main and v2026.5.7 still reject whitespace in the active Docker setup script, and the linked fix PR has not merged.

## What This Changes

Allow Docker setup to accept OPENCLAW_CONFIG_DIR and OPENCLAW_WORKSPACE_DIR values containing ordinary spaces, especially Windows home paths.

## Best Possible Solution

Update scripts/docker/setup.sh with a shared YAML-safe mount writer for all generated mounts, allow ordinary spaces while preserving control-character and injection guards, add focused Docker setup proof, then land the linked or replacement PR.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: set OPENCLAW_CONFIG_DIR or OPENCLAW_WORKSPACE_DIR to a value with a space and run scripts/docker/setup.sh; current main sends it to validate_mount_path_value, which fails with the exact reported message.

## Solution Assessment

Yes. The narrow maintainable fix is to allow ordinary spaces in the active Docker setup path while writing all generated Compose mounts through a safe scalar encoder; closing before the linked PR or replacement merges would be premature.
