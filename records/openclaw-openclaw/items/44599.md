## What This Changes

Allow Docker setup host config/workspace bind-mount paths such as `OPENCLAW_CONFIG_DIR` and `OPENCLAW_WORKSPACE_DIR` to contain ordinary spaces.

## Best Possible Solution

Allow ordinary spaces in Docker bind-mount source paths, quote or otherwise safely emit Compose volume entries, preserve control-character and YAML-injection guards, and land the fix with focused regression coverage plus real Docker/Git Bash proof.

## Reproduction Assessment

Yes for source-level reproduction. Any ordinary-space value in the reported env vars reaches `validate_mount_path_value` and fails before Docker Compose can run, although this review did not run a live Windows/Git Bash Docker scenario.

## Solution Assessment

Yes. The narrow maintainable fix is in `scripts/docker/setup.sh`: relax ordinary-space rejection, keep rejecting control characters and injection-prone mount syntax, safely emit Compose mounts, and add regression coverage for spaced config/workspace paths.
