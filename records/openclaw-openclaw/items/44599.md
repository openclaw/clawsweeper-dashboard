## What This Changes

The issue reports that Docker setup rejects `OPENCLAW_CONFIG_DIR` values containing whitespace, blocking Windows/Git Bash installs when host paths contain spaces.

## Best Possible Solution

Allow ordinary spaces in Docker bind-mount source paths, safely generate Compose volume entries, preserve control-character/YAML-injection guards, and land a refreshed or replacement fix with focused regression coverage plus real Docker/Git Bash proof.

## Reproduction Assessment

Yes for source-level reproduction. Any ordinary-space value in the reported env vars reaches `validate_mount_path_value` and fails before Docker Compose can run.

## Solution Assessment

Yes. The narrow maintainable fix is in `scripts/docker/setup.sh`: relax ordinary-space rejection, keep rejecting control characters and injection-prone mount syntax, and add regression coverage for spaced config/workspace paths.
