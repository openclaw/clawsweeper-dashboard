## Summary

Keep open: current main still rejects whitespace in Docker config/workspace paths, and an open fix PR with closing syntax should be reviewed, updated, or replaced before this issue is closed.

## What This Changes

The issue reports that Docker setup rejects `OPENCLAW_CONFIG_DIR` paths containing whitespace on Windows/Git Bash.

## Best Possible Solution

Land a current-main fix in `scripts/docker/setup.sh` that permits ordinary spaces in config/workspace paths, safely quotes generated compose mounts, preserves control-character/injection guards, and adds regression coverage.

## Reproduction Assessment

Yes. Source inspection shows any `OPENCLAW_CONFIG_DIR` value containing a space reaches `validate_mount_path_value` and exits before setup proceeds; I did not execute the setup script because it writes checkout/runtime files.

## Solution Assessment

Yes, allowing ordinary path spaces while keeping control-character and mount-format hardening is the narrow fix direction. The current open PR needs to be applied to the current `scripts/docker/setup.sh` surface before it can solve this issue.
