## Summary

Keep open: current `main` and latest release `v2026.5.20` still reject whitespace in Docker setup mount paths, while the only linked open fix PR is a conflicting fork PR that needs maintainer handling before a duplicate repair lane starts.

## What This Changes

The issue asks Docker setup to allow `OPENCLAW_CONFIG_DIR` and related bind-mount paths to contain ordinary whitespace on Windows/Git Bash.

## Best Possible Solution

Allow ordinary spaces in Docker bind-mount source paths, safely quote generated Compose volume scalars, preserve control-character/YAML-injection guards, and land a refreshed or replacement fix with focused regression coverage plus real Docker/Git Bash proof.

## Reproduction Assessment

Yes. Current `main` validates the reported env vars through `validate_mount_path_value`, which rejects `[[:space:]]`, so any normal-space path fails before Docker Compose runs.

## Solution Assessment

Yes. The best fix is narrow in `scripts/docker/setup.sh`: allow ordinary spaces, quote generated YAML mount scalars, preserve the existing injection guards, and refresh or replace the linked conflicting fork PR before starting duplicate work.
