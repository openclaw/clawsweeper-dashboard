## What This Changes

The issue reports that Docker setup rejects whitespace-containing OPENCLAW_CONFIG_DIR and OPENCLAW_WORKSPACE_DIR paths, blocking Windows/Git Bash installs where HOME contains spaces.

## Best Possible Solution

Fix the active scripts/docker/setup.sh path by allowing ordinary spaces while preserving control-character and YAML-injection guards, encode every generated Compose mount safely, add focused e2e or real Docker proof, then land the linked or replacement PR and close this issue.

## Solution Assessment
