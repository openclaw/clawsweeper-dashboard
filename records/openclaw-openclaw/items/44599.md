## Summary

Keep open: current main still rejects whitespace in Docker setup config/workspace paths, and open PR #44671 already claims this issue with closing syntax, so it should stay open until that PR or a replacement lands.

## What This Changes

The issue asks Docker setup to accept whitespace-containing OpenClaw config/workspace paths on Windows instead of aborting installation.

## Best Possible Solution

Land or replace #44671 with a current-main change in `scripts/docker/setup.sh` that allows ordinary spaces in config/workspace bind-mount paths, quotes emitted compose volume entries, preserves injection guards, and adds regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a value such as `OPENCLAW_CONFIG_DIR='C:/Users/John Doe/.openclaw'` reaches `validate_mount_path_value` and exits before setup proceeds; I did not execute the setup script because this review is read-only and the script writes runtime files and starts Docker.

## Solution Assessment

Yes. The narrow maintainable fix is to permit ordinary path spaces while continuing to reject control characters and unsafe mount syntax, then quote generated compose mount strings where needed.
