## Summary

Keep open: current main and the latest release still reject whitespace in the active Docker setup paths. The linked fork PR is useful context, but it is open, conflicting, and edits the deleted root setup script rather than the active `scripts/docker/setup.sh`.

## What This Changes

Allow Docker setup to accept ordinary spaces in `OPENCLAW_CONFIG_DIR` and `OPENCLAW_WORKSPACE_DIR` while preserving safe mount validation.

## Best Possible Solution

Land a focused fix in `scripts/docker/setup.sh` that allows ordinary spaces in valid host paths, writes generated Compose mounts with YAML-safe quoting, and adds regression coverage; then close or supersede the stale fork PR.

## Reproduction Assessment

Yes. Source inspection on current main shows that setting `OPENCLAW_CONFIG_DIR` or `OPENCLAW_WORKSPACE_DIR` to a value with a space will hit `validate_mount_path_value` and fail before Docker setup proceeds.

## Solution Assessment

Yes. The narrow maintainable fix is to update the active `scripts/docker/setup.sh` validator and generated Compose mount output, not to revive the deleted root `docker-setup.sh` path.
