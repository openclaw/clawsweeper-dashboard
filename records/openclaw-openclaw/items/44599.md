## Summary

Keep this issue open: current main and the latest release still reject whitespace in the active Docker setup script, and there is an open closing PR that should be reviewed, repaired, or replaced rather than closing the issue early.

## What This Changes

Allow `OPENCLAW_CONFIG_DIR` and `OPENCLAW_WORKSPACE_DIR` paths with whitespace in Docker setup so Windows/Git Bash installs do not fail.

## Best Possible Solution

Land a current-main fix in `scripts/docker/setup.sh` that allows ordinary spaces, preserves control-character and mount-injection guards, emits Docker Compose mounts through a shared YAML-safe writer, and adds focused e2e coverage.

## Reproduction Assessment

Yes, there is a high-confidence source-level reproduction: set `OPENCLAW_CONFIG_DIR` or `OPENCLAW_WORKSPACE_DIR` to a path containing a space and current main reaches `validate_mount_path_value`, which fails `[[:space:]]` before setup can proceed. I did not run the full Docker setup because that would write repo-local setup artifacts in this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix is to update the active `scripts/docker/setup.sh` path and its e2e coverage. The open PR's direction is right, but it is not mergeable as-is because it targets the removed root wrapper and needs safer compose scalar handling.
