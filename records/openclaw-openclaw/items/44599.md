## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/44599. Current main's active Docker setup script `scripts/docker/setup.sh` rejects whitespace in `OPENCLAW_CONFIG_DIR` and `OPENCLAW_WORKSPACE_DIR` via `validate_mount_path_value`, which breaks Windows/Git Bash users whose home path contains spaces. Update only the active `scripts/docker/setup.sh` path, not the deleted root `docker-setup.sh`, so ordinary spaces are allowed while newline, carriage return, tab, comma/colon structure, and YAML injection protections remain intact. Add a YAML-safe scalar helper for generated compose mounts, including `write_extra_compose` and any sandbox socket overlay affected by relaxing validation; preserve Windows backslashes and escape single quotes correctly. Add or update `src/docker-setup.e2e.test.ts` coverage for config/workspace paths containing spaces and quoted generated compose output, while keeping existing invalid mount/control-character tests. Add a `CHANGELOG.md` fix entry if the repair is prepared for landing. Stop if the implementation would require a new config option, a Docker Compose contract change, or weakening injection guards.
