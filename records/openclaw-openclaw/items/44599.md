## Summary

Keep open: current `main` still rejects whitespace in Docker setup host paths, the latest release still contains that rejection, and the live fork PR with closing syntax remains open as the current fix path.

## What This Changes

Allow Docker setup `OPENCLAW_CONFIG_DIR` and `OPENCLAW_WORKSPACE_DIR` host paths to contain ordinary spaces so Windows/Git Bash installs do not fail on user paths with whitespace.

## Best Possible Solution

Review, refresh, land, or replace the linked fork PR with a focused Docker setup change that safely serializes spaced bind mounts, preserves injection guards, and adds regression coverage plus real setup proof.

## Reproduction Assessment
