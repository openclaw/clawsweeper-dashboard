## Summary

Keep open: current main and the latest release still reject whitespace in the Docker setup config/workspace path validators, and #44671 is an open unmerged PR that links this issue for closure. The issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Docker setup should allow ordinary spaces in `OPENCLAW_CONFIG_DIR` and `OPENCLAW_WORKSPACE_DIR`, especially for Windows home paths, without failing installation.

## Best Possible Solution

Land #44671 or an equivalent focused Docker setup fix that allows ordinary spaces in config/workspace paths, quotes generated compose volume entries, keeps unsafe mount syntax rejected, and adds e2e coverage for spaced paths.

## Reproduction Assessment

Yes. A high-confidence static reproduction is setting `OPENCLAW_CONFIG_DIR` or `OPENCLAW_WORKSPACE_DIR` to a path containing a space before running `scripts/docker/setup.sh`; the current source routes those values to `validate_mount_path_value` and fails on `[[:space:]]`. I did not execute the script because this review is read-only and the setup script writes env/compose files.

## Solution Assessment

Yes, the narrow Docker setup repair is the best path. Broad config-dir migration or support guidance would not address the deterministic validator and compose-mount behavior that blocks installation.
