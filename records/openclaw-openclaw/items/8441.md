## Summary

Keep open. Current main still does not accept or apply first-class per-skill `model` and `thinking` config, the direct implementation attempts are closed unmerged, and the remaining open frontmatter PR is a different product direction that does not settle this requested config contract.

## What This Changes

Request first-class `skills.entries.<skill>.model` and `skills.entries.<skill>.thinking` defaults that apply when a skill is invoked.

## Best Possible Solution

Define the product contract, then add typed and schema-backed per-skill defaults that flow through skill invocation into existing model and thinking selection with docs and regression tests.

## Solution Assessment
