## Summary

Keep open. Current main still lacks first-class per-skill `model` and `thinking` config: the type, strict schema, docs, and skill-command invocation path still only support the existing skill fields and prompt rewrite behavior.

## What This Changes

The issue requests first-class per-skill `thinking` and `model` fields under `skills.entries.<skill>` so skill invocations can override reasoning level and model.

## Best Possible Solution

Define and implement a validated per-skill `model`/`thinking` config contract with explicit precedence and authorization semantics, then update invocation plumbing, tests, generated config docs, and public skills docs.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives a high-confidence verification path: current main rejects top-level `model`/`thinking` under `skills.entries.<skill>` and has no skill invocation carrier for those defaults.

## Solution Assessment

No, current main is not a solution to the requested feature. The requested direction is plausible, but the maintainable fix needs explicit precedence and model-authorization decisions before implementation.
