## Summary

Keep open: current main and v2026.5.12 still lack first-class `skills.entries.<skill>.model` and `skills.entries.<skill>.thinking`, and the open SKILL.md routing work at https://github.com/openclaw/openclaw/pull/58142 covers only part of the product question while still needing maintainer/security review.

## What This Changes

The issue requests first-class per-skill `thinking` and `model` fields under `skills.entries.<skill>` so skill invocations can override turn reasoning/model defaults.

## Best Possible Solution

Define one maintainer-approved per-skill routing contract across `skills.entries` and SKILL.md, then implement schema, invocation/model-selection plumbing, precedence, allowed-model/provider/auth policy, focused tests, and public docs together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config contract request. Source, docs, tests, related items, and v2026.5.12 show the requested first-class skill-entry fields and runtime consumer are absent.

## Solution Assessment

Unclear until maintainers choose the public contract. Adding raw fields alone is not the best solution without explicit precedence, allowed-model, provider/auth, fallback, and thinking-level semantics.
