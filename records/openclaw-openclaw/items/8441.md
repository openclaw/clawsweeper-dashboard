## Summary

Keep open. Current main still does not provide first-class `skills.entries.<skill>.model` or `skills.entries.<skill>.thinking`, and the related open frontmatter proposal covers a different model-routing shape without the requested config field or thinking default.

## What This Changes

The issue requests first-class `thinking` and `model` fields under `skills.entries.<skill>` so skill invocations can default to a specific reasoning level and model.

## Best Possible Solution

Define the public per-skill defaults contract, then add typed schema fields, invocation plumbing, precedence rules, model/thinking validation, regression tests, and public docs.

## Solution Assessment
