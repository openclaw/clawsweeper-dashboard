Request adding optional `model` and `thinking` fields under `skills.entries.<skill>` for per-skill model and thinking defaults.

## Best Possible Solution

Define one maintainer-approved per-skill routing contract, then implement schema, invocation plumbing, precedence, provider/auth policy, focused tests, and public docs together.

## Solution Assessment

Unclear until maintainers choose the public contract. Adding raw fields alone is not enough; the implementation needs explicit precedence, allowed-model, provider/auth, fallback, and thinking-level semantics.
