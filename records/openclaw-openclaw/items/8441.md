## Summary

Keep open: current main still lacks first-class `skills.entries.<skill>.model` and `skills.entries.<skill>.thinking`, and the remaining work needs maintainer product/security decisions around routing, precedence, allowlists, and the related SKILL.md frontmatter path.

## What This Changes

Add per-skill `model` and `thinking` fields under `skills.entries.<skill>` and apply them as per-turn defaults when that skill runs.

## Best Possible Solution

Define one maintainer-approved per-skill routing contract across `skills.entries` and SKILL.md, then implement schema, invocation/model-selection plumbing, precedence, allowed-model/provider/auth policy, focused tests, and public docs together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config-contract request. Source, docs, tests, latest release code, and related PR state show the requested first-class skill-entry fields and runtime consumer are absent.

## Solution Assessment

Unclear until maintainers choose the public contract. Adding raw fields alone is not the best solution without explicit precedence, allowed-model, provider/auth, fallback, and thinking-level semantics.
