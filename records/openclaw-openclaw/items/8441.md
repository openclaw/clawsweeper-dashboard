## Summary

Keep open. Current main still lacks first-class `skills.entries.<skill>.model` and `skills.entries.<skill>.thinking`, schema/tests continue to reject unknown top-level per-skill keys, docs do not document these fields, and the direct implementation PRs are closed unmerged while the open frontmatter PR is a different unmerged design.

## What This Changes

The issue requests per-skill `model` and `thinking` defaults under `skills.entries.<skill>` so skill invocations can override model selection and reasoning effort.

## Best Possible Solution

Define the public per-skill defaults contract first, then add typed schema fields, resolved invocation plumbing, precedence rules against explicit user/session/agent defaults, model/thinking validation, regression tests, and public docs.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection on current main shows the requested fields are absent from type/schema/docs and are not carried through skill invocation.

## Solution Assessment

Unclear until maintainers choose the config-versus-frontmatter contract and precedence/security rules. The requested `skills.entries` shape is plausible, but it is a public API/product decision rather than a narrow repair.
