## Summary

Keep open: current main and v2026.5.22 still do not implement first-class per-skill `model` or `thinking` under `skills.entries`, and the active related frontmatter PR is only a partial, conflicting path that does not settle this config contract.

## What This Changes

Request to add first-class `thinking` and `model` fields to `skills.entries.<skill>` so individual skills can default to specific reasoning levels or models.

## Best Possible Solution

Define one operator-controlled per-skill routing contract, then implement schema/types, invocation plumbing, provider-aware model selection, thinking validation, precedence, tests, release-safe docs, and security semantics together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config-contract request. Source, docs, tests, latest-release inspection, and related GitHub state show the requested fields and runtime consumer are still absent.

## Solution Assessment

Unclear until maintainers choose the routing and security contract. Adding raw fields alone is not the best solution without explicit precedence, allowed-model, provider/auth, fallback, turn-boundary, and thinking-level semantics.
