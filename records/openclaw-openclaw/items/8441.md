## Summary

Keep open. Current main still does not implement first-class `skills.entries.<skill>.model` or `skills.entries.<skill>.thinking`, related config PRs are closed unmerged, and the open frontmatter PR is a different model-only design that still needs maintainer security/API judgment.

## What This Changes

The issue requests per-skill `thinking` and `model` defaults under `skills.entries.<skill>` so skill invocations can override turn reasoning and model selection.

## Best Possible Solution

Define the public per-skill defaults contract, then add typed schema fields, invocation plumbing through existing model selection and thinking policy, clear precedence rules, docs, and focused config/invocation tests.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection on current main shows the requested config fields are absent from types/schema/docs and are not carried through skill invocation.

## Solution Assessment

Unclear until maintainers choose the config-versus-frontmatter contract and security/precedence rules. The requested config shape is plausible, but it is not a narrow bug fix and the open frontmatter PR does not settle the `thinking` or `skills.entries` request.
