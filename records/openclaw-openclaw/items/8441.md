## Summary

Keep open. Current main still lacks first-class per-skill `model` and `thinking` config; the related custom config bag work does not drive turn-level model selection, PR #8455 is closed unmerged, and #40536 was closed in favor of this request.

## What This Changes

Request adding optional `model` and `thinking` fields to `skills.entries.<skill>` so skill invocations can use per-skill turn defaults like cron jobs.

## Best Possible Solution

Add validated optional per-skill model/thinking config, carry the resolved defaults through skill command invocation, apply non-persistent per-turn overrides with documented precedence, and update docs, generated schema/help surfaces, tests, and changelog.

## Reproduction Assessment

Yes, from source inspection. Current `main` rejects top-level `model`/`thinking` under `skills.entries.<skill>` and has no command-spec or invocation field that could carry those defaults into model selection.

## Solution Assessment

Yes. First-class core config plus invocation plumbing is the narrow maintainable solution; the existing custom `config` bag and SKILL.md directives cannot provide validated per-user defaults before model selection.
