## Summary

Keep open: current main and the latest release still do not provide first-class `skills.entries.<skill>.model` or `skills.entries.<skill>.thinking`, and the active related implementation is still an open partial frontmatter path that needs maintainer contract and security review.

## What This Changes

Add per-skill `model` and `thinking` config under `skills.entries.<skill>` so skill-triggered turns can use skill-specific model and reasoning defaults.

## Best Possible Solution

Define one maintainer-approved per-skill routing contract, then implement schema/types, invocation and provider-aware model selection, thinking validation, precedence, allowlist or consent behavior, focused tests, real provider proof, and public docs together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config-contract request. Source, docs, tests, latest release inspection, and live related GitHub state show the requested fields and runtime consumer are still absent.

## Solution Assessment

Unclear until maintainers choose the routing and security contract. Adding raw fields alone is not the best solution without explicit precedence, allowed-model, provider/auth, fallback, consent, turn-boundary, and thinking-level semantics.
