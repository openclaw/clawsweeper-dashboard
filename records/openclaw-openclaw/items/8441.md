## Summary

Keep open: current main and v2026.5.22 still do not support first-class `skills.entries.<skill>.model` or `skills.entries.<skill>.thinking`, and the active related frontmatter PR is partial and does not settle this config contract.

## What This Changes

The issue requests per-skill `model` and `thinking` overrides under `skills.entries.<skill>` so skill invocations can choose a model and thinking level.

## Best Possible Solution

Define one operator-controlled per-skill routing contract, then implement schema/types, invocation plumbing before provider/model/auth planning, thinking validation, precedence, fallback behavior, tests, docs, and security semantics together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config-contract request. Source, docs, tests, latest-release inspection, and related GitHub state show the requested fields and runtime consumer are still absent.

## Solution Assessment

Unclear until maintainers choose the routing and security contract. Adding raw fields alone is not the best solution without explicit precedence, allowed-model, provider/auth, fallback, turn-boundary, and thinking-level semantics.
