## Security Review

Status: needs_attention

Summary: Per-skill model routing is security-sensitive because it can change provider choice, prompt destination, credential path, and cost.

Concerns:

- **[medium] Keep model allowlists authoritative:** `docs/concepts/models.md:120`
  - body: The docs make `agents.defaults.models` the allowlist for `/model` and session overrides, so skill-level routing should honor that same policy rather than creating a separate path.
  - confidence: 0.86
- **[medium] Define consent for skill-influenced routing:** `docs/tools/skills.md:182`
  - body: The skills docs tell users to treat third-party skills as untrusted code, so config or metadata that changes model/provider selection needs explicit operator-controlled semantics.
  - confidence: 0.84
- **[medium] Match existing plugin override trust policy:** `docs/gateway/configuration-reference.md:229`
  - body: Existing plugin LLM and subagent model overrides require explicit `allowModelOverride` and optional `allowedModels`; per-skill routing should not be less controlled.
  - confidence: 0.84
