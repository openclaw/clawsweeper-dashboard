## Summary

Keep open. Current main still does not accept or apply first-class per-skill `model` or `thinking` under `skills.entries.<skill>`, the direct implementation pull requests are closed unmerged, and the open frontmatter-only routing work does not supersede this config-and-thinking request.

## What This Changes

Add optional per-skill `thinking` and `model` fields under `skills.entries.<skill>` and apply them when invoking skills.

## Best Possible Solution

Define the config/frontmatter contract first, then add typed schema-backed per-skill defaults that flow through skill invocation into existing model and thinking selection with docs and regression tests.
