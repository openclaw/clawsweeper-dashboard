## Summary

Keep open: current main still lacks first-class `skills.entries.<skill>.model` and `skills.entries.<skill>.thinking`, and the remaining related work only covers part of the request. This needs maintainer product and security review before a new implementation is queued.

## What This Changes

The issue requests per-skill `thinking` and `model` config under `skills.entries.<skill>` so skill invocations can override reasoning level and model selection.

## Best Possible Solution

Define one maintainer-approved per-skill routing contract across `skills.entries` and SKILL.md, then implement schema, invocation plumbing, precedence, provider/auth allowlist policy, focused tests, and public docs together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config contract request. Source, docs, related items, and the latest release show the requested first-class skill-entry fields and runtime consumer are absent.

## Solution Assessment

Unclear until maintainers choose the public contract. Adding raw fields alone is not enough; the implementation needs explicit precedence, allowed-model, provider/auth, fallback, and thinking-level semantics.
