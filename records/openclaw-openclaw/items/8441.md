## Summary

Keep open. Current main still lacks first-class per-skill `model` and `thinking` config support, the prior implementation PR is closed unmerged, and the remaining work needs maintainer decisions on precedence and model/thinking validation.

## What This Changes

Add optional `thinking` and `model` fields to `skills.entries.<skill>` so skill invocation can set per-turn reasoning and model defaults.

## Best Possible Solution

Define the precedence and validation contract, then add typed/schema-backed per-skill defaults that flow through skill invocation into the existing model and thinking selection paths with tests and docs.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a bug reproduction because this is a feature/config request; source inspection gives high confidence that top-level `model` and `thinking` under `skills.entries.<skill>` are rejected today and no invocation override carrier exists.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers settle precedence, allowlist, fallback, and unsupported-thinking semantics; first-class config fields in the skills config and invocation plumbing appear to be the right implementation area.
