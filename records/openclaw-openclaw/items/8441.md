## Summary

Keep open: current main still does not implement first-class `skills.entries.<skill>.model` or `skills.entries.<skill>.thinking`, and the remaining work needs a maintainer-approved config, precedence, allowlist, and provider/auth policy before another implementation is queued.

## What This Changes

Add per-skill `model` and `thinking` config fields under `skills.entries.<skill>` so skill invocations can use task-specific model and reasoning defaults.

## Best Possible Solution

Define one maintainer-approved per-skill routing contract across config and SKILL.md, then implement schema, invocation plumbing, precedence, provider/auth allowlist policy, focused tests, and public docs together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config contract request. Source and docs inspection on current main show the requested first-class skill-entry fields and runtime consumer are absent.

## Solution Assessment

Unclear until maintainers choose the public contract. Adding raw fields alone is not enough; the implementation needs explicit precedence, allowed-model, provider/auth, fallback, and thinking-level semantics.
