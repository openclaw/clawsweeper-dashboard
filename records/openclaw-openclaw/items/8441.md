## Summary

Keep open. Current main still lacks first-class per-skill `model` and `thinking` config: the type/schema/docs only support `enabled`, `apiKey`, `env`, and `config`, and skill command invocation has no carrier for model or thinking defaults; the related implementation PR is closed and unmerged.

## What This Changes

Add per-skill `model` and `thinking` fields under `skills.entries.<skill>` and apply them as defaults when that skill runs.

## Best Possible Solution

Add validated optional `model` and `thinking` fields to skills config, carry resolved per-skill defaults through skill invocation, define precedence and allowlist behavior, then update schema tests, invocation/model-selection tests, and public docs.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives a high-confidence verification path that current main still rejects top-level `model`/`thinking` under `skills.entries.<skill>` and lacks invocation plumbing for them.

## Solution Assessment

No, current main is not a solution to the requested feature. The requested direction is plausible, but the maintainable fix needs explicit precedence and model-allowlist semantics before implementation.
