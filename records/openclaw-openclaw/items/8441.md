## Summary

Keep open. Current main still lacks first-class `skills.entries.<skill>.model` and `skills.entries.<skill>.thinking`, and the closest open follow-up is SKILL.md-frontmatter model routing rather than this config surface plus thinking defaults.

## What This Changes

The issue requests optional `thinking` and `model` fields under `skills.entries.<skill>` so skills can supply per-turn reasoning and model defaults.

## Best Possible Solution

Define one public per-skill defaults contract, then add typed schema support, invocation plumbing, precedence rules, model/thinking validation, focused tests, and public docs.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection on current main shows the requested fields are absent from the type, strict schema, docs, tests, and skill command contract.

## Solution Assessment

Unclear until maintainers choose the public contract and precedence/security policy. The requested `skills.entries` shape is plausible, but this is a product/API decision rather than a narrow bug fix.
