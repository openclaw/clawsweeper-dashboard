## Summary

Keep open. Current main still does not provide first-class `skills.entries.<skill>.model` or `skills.entries.<skill>.thinking`; the config type/schema, docs, command-spec contract, and skill invocation path all still omit that plumbing, while the related implementation PR is closed and unmerged.

## What This Changes

Add per-skill `thinking` and `model` settings under `skills.entries.<skill>` so skill invocations can use configured turn defaults similar to cron `agentTurn` payloads.

## Best Possible Solution

Add validated optional `model` and `thinking` fields to skills config, pass resolved per-skill defaults through skill command invocation, define precedence explicitly, cover schema/invocation/model-selection behavior with tests, and update the skills docs.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Current-main source inspection gives a high-confidence verification path that the requested keys are still unsupported by the strict schema and absent from invocation plumbing.

## Solution Assessment

No, current main is not a solution to the requested feature. The proposed direction is plausible, but the best implementation needs maintainer agreement on precedence and model-allowlist behavior before adding the new config surface.
