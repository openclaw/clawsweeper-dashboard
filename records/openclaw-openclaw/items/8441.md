## Summary

Keep open: current main and the latest release still do not provide first-class `skills.entries.<skill>.model` or `skills.entries.<skill>.thinking`, and the remaining work is a public config/model-routing contract with security and provider-routing implications.

## What This Changes

The issue requests per-skill `thinking` and `model` fields under `skills.entries.<skill>` that apply when a skill is loaded or invoked.

## Best Possible Solution

Define one maintainer-approved per-skill routing contract, then implement schema/types, invocation and model-selection plumbing, precedence, allowlist/consent policy, focused tests, and public docs together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config-contract request. Source, docs, tests, latest release code, and related PR state show the requested first-class fields and runtime consumer remain absent.

## Solution Assessment

Unclear until maintainers choose the contract. Adding raw fields alone is not the best solution without explicit precedence, allowed-model, provider/auth, fallback, and thinking-level semantics.
