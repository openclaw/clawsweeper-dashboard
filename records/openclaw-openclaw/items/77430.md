## Summary

Keep open: current main still has the source path described in the report. Native subagent spawning sends the concrete task only through the generated extra system prompt, but the embedded-runner prompt builder does not include that extra prompt when an agent-level system prompt override is configured.

## What This Changes

The issue requests reliable `sessions_spawn` task and subagent role-context delivery when the target agent has `systemPromptOverride`, plus better transcript diagnostics.

## Best Possible Solution

Compose configured system prompt overrides with generated runtime/subagent `extraSystemPrompt` for native subagent runs, keep the task single-sourced unless maintainers choose a compact visible fallback, and add regression coverage for the override case.

## Reproduction Assessment

Yes, at source level: configure `agents.defaults.systemPromptOverride` or a per-agent override, spawn a native subagent, and the child task/role block travels only as `extraSystemPrompt` while the override branch in `buildAttemptSystemPrompt` ignores that field. I did not run a live macOS/Discord spawn in this read-only pass.

## Solution Assessment

Yes for the narrow composition fix: preserving `extraSystemPrompt` alongside the configured override is the smallest maintainable repair. Duplicating the full task in the visible first user message should be treated separately because current tests and changelog intentionally avoid that duplication.
