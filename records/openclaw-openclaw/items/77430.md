## Summary

Keep open: current main still has a source-reproducible path where a configured `systemPromptOverride` can discard the `sessions_spawn` child role/task prompt, while the visible first child message intentionally contains only the generic bootstrap text.

## What This Changes

The issue requests reliable delivery of delegated subagent task/context when the target agent has `systemPromptOverride`, plus better transcript diagnosability for prompt-composition failures.

## Best Possible Solution

Compose configured system prompt overrides with runtime `extraSystemPrompt` for embedded and CLI-backed runs so subagent role/task context survives, while preserving the current no-duplicate-task first user message unless maintainers explicitly choose a diagnostic fallback.

## Reproduction Assessment

Yes, source-reproducible on current main: configure a target agent with `systemPromptOverride` and spawn a native subagent, and the child role/task is passed only through `extraSystemPrompt`, which the override branches skip. I did not run a live Gateway test in this read-only review.

## Solution Assessment

Yes for the narrow fix: compose the configured override with runtime `extraSystemPrompt` in both embedded and CLI prompt builders. The visible transcript fallback should be secondary because current code intentionally single-sources the task in the system prompt for token reasons.
