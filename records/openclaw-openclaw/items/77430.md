## Summary

Keep open: current main still source-reproduces the core failure because configured system prompt overrides bypass `extraSystemPrompt`, while the visible child message intentionally relies on that hidden prompt for the task.

## What This Changes

The issue asks OpenClaw to reliably deliver `sessions_spawn` child task and role context when the target agent has `systemPromptOverride`, with better transcript diagnosability if prompt composition regresses.

## Best Possible Solution

Preserve generated subagent `extraSystemPrompt` when applying configured system prompt overrides in both embedded and CLI runner prompt builders, with regression tests; treat visible task duplication as a separate maintainer choice.

## Reproduction Assessment

Yes, source-reproducible: configure a target agent with `systemPromptOverride` and spawn a native subagent; the child role/task block is passed as `extraSystemPrompt`, but the override branch skips that field. I did not execute a live run, so this is not marked as fully reproduced.

## Solution Assessment

Yes for the narrow fix: compose the configured override with generated `extraSystemPrompt` before provider transforms and session application. Re-adding the task to the visible user message should not be bundled unless maintainers intentionally change the existing no-duplication policy.
