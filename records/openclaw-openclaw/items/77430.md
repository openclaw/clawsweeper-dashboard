## Summary

Keep open: current main still routes the native subagent task/context through `extraSystemPrompt`, and the configured `systemPromptOverride` path still bypasses that field, so the reported loss of delegated role context is source-reproducible.

## What This Changes

Ensure `sessions_spawn` child runs receive delegated task and generated subagent role context even when the target agent has `systemPromptOverride` configured.

## Best Possible Solution

Compose configured prompt overrides with generated `extraSystemPrompt` for subagent runs and add regression coverage, while keeping the visible task single-sourced unless maintainers explicitly revisit the #72019 token-cost tradeoff.

## Reproduction Assessment

Yes, source-reproducible: `sessions_spawn` sends the generated subagent task/role block only via `extraSystemPrompt`, and `buildAttemptSystemPrompt` drops that field whenever `systemPromptOverrideText` is set. I did not perform a live gateway run in this read-only review.

## Solution Assessment

Yes for the narrow repair: preserve the configured override as the base prompt and compose the generated subagent context around it, preserving existing bootstrap supplement behavior. Duplicating the full task into the visible user message is a separate diagnostic tradeoff because current code intentionally avoids that for #72019.
