## Summary

## What This Changes

Adds a terminal-session lifecycle helper, uses it to rotate CLI and reply sessions away from terminal rows, clears terminal lifecycle markers on new reply rows, and prevents restart-sentinel continuations from dispatching into terminal entries.

## Best Possible Solution

Land the centralized terminal-lifecycle guard after maintainers accept the restart-continuation policy and the PR includes redacted live gateway or Telegram proof for the sentinel path.

## Reproduction Assessment

Yes from source inspection: current main lets timestamp-fresh terminal rows pass normal freshness checks and queues restart continuations without a terminal guard. I did not establish a live Telegram restart reproduction in this read-only review.

## Solution Assessment

Mostly yes: a shared terminal-lifecycle guard across CLI, reply, and restart-sentinel paths is the narrow maintainable fix shape. The remaining question is whether skipping terminal continuations is the accepted policy or whether they should start a fresh session.
