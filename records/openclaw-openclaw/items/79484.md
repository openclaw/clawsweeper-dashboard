## Summary

Keep open: this PR is a narrow, source-backed fix for a real CLI transcript persistence bug that is still present on current main, with no blocking code-review findings. It needs maintainer follow-up because exact-head CI still has failing checks before merge.

## What This Changes

The PR initializes built-in context engines before CLI compaction resolves the active engine, adds a call-order regression test, and records the fix in the changelog.

## Best Possible Solution

Merge this initializer-order fix after exact-head CI is clean, then let it resolve https://github.com/openclaw/openclaw/issues/79446 and inform dedupe/cleanup of related legacy-init reports.

## Reproduction Assessment

Yes. Current main has a source-level path where CLI compaction resolves the default context engine before built-in registration, and the resolver throws the reported legacy-unregistered error when the registry is empty; the linked issue provides shipped v2026.5.7 live log proof.

## Solution Assessment

Yes. Calling the existing idempotent initializer immediately before CLI compaction resolves the engine matches adjacent runtime paths and avoids changing plugin slot defaults, resolver semantics, or product behavior.
