  - body: `writeSupportBundleZip()` uses `fsp.writeFile()` with the default write flag, so an existing file or symlink target keeps its existing permissions. The new private-archive contract should either be enforced with exclusive/hardened writes or scoped to newly created files only.

  - body: The gateway creates a fresh request id for each `node.invoke` and only forwards `idempotencyKey`; the inspected clients do not show side-effect dedupe for that key. Documenting retries as safe can cause clients to execute node commands twice after a lost response.
