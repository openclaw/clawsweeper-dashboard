## Summary

Keep open: current main still emits command-specific list JSON payloads, and the requested versioned envelope plus migration flag is a cross-command public CLI API decision rather than an already implemented fix or duplicate.

## What This Changes

The issue requests a standardized, versioned `--json` list envelope across agents, sessions, models, cron, nodes, and skills commands.

## Best Possible Solution

Define and document a shared list-envelope contract that keeps legacy shapes available during migration and maps command-specific metadata into a stable `meta` field without changing error semantics.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an RFC for a new CLI output contract. Source and docs inspection on current main confirm the existing command-specific JSON shapes are still present.

## Solution Assessment

Unclear until maintainers choose the public CLI API. The proposed versioned envelope and legacy flag are plausible, but the final design should explicitly cover command metadata, gateway passthroughs, and compatibility timing.
