## Summary

Keep open: current main still uses command-specific list JSON shapes, and the requested versioned envelope plus migration flag is a cross-CLI API decision rather than something already implemented or a duplicate of #73077.

## What This Changes

Request to define a versioned, standard JSON envelope for list-style CLI outputs and migrate agents/sessions/models/cron/nodes/skills with legacy compatibility.

## Best Possible Solution

Define a shared list-envelope schema and migration policy, document it as CLI contract, then add backward-compatible opt-in support with regression coverage for each affected list command.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an RFC for a new CLI contract. Source and docs inspection confirm current main still emits the command-specific JSON shapes described by the report.

## Solution Assessment

Unclear until maintainers choose the public CLI contract. The proposed versioned envelope and legacy flag are plausible, but the final design should explicitly cover command metadata, gateway passthroughs, and compatibility timing.
