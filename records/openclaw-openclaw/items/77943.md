## Summary

Keep open: current main still has command-specific JSON payloads for the listed CLI surfaces and no shared versioned list-envelope contract, so this remains a public CLI API design decision rather than cleanup closure.

## What This Changes

Request standardizing list-style CLI `--json` outputs behind a versioned envelope and migration flag across agents, sessions, models, cron, nodes, and skills.

## Best Possible Solution

Define a documented, versioned list-envelope contract with opt-in rollout, legacy compatibility, metadata mapping, and focused regression coverage for each affected command.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an RFC for a new public CLI JSON contract. Source inspection on current main confirms the command-specific JSON shapes still exist.

## Solution Assessment

Unclear until maintainers choose the public API. The proposed versioned envelope and legacy flag are plausible, but the final design needs explicit compatibility, metadata, gateway-error, docs, and test coverage decisions.
