## Summary

Keep open: current main and latest release still have multiple command-specific list JSON shapes and no shared `--json-envelope` or `list-v1` implementation. This is a valid CLI API ergonomics RFC, but the envelope shape and staged compatibility policy need maintainer product direction before implementation.

## What This Changes

Standardize list-style CLI `--json` outputs behind a versioned envelope and staged `--json-envelope` migration across agents, sessions, models, cron, nodes, and skills.

## Best Possible Solution

Define a shared list-envelope schema/helper and opt-in `--json-envelope` mode across the core list commands, then document and stage any default migration only after maintainer approval.

## Reproduction Assessment

Yes. Source inspection on current main shows the reported mixed list JSON shapes and no shared `--json-envelope` or `list-v1` implementation; I did not execute CLI commands under the read-only checkout constraint.

## Solution Assessment

Unclear. A shared versioned envelope is plausible and maintainable, but the exact public schema and compatibility schedule are maintainer API-contract decisions.
