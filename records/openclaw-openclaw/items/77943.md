## Summary

Current main still has command-specific JSON list payloads and no shared `--json-envelope`/list-v1 contract, so this RFC remains open for a public CLI API decision rather than cleanup closure.

## What This Changes

Request to standardize list-style CLI `--json` outputs behind a versioned envelope with legacy migration support across agents, sessions, models, cron, nodes, and skills.

## Best Possible Solution

Define a documented, versioned list-envelope contract that preserves legacy shapes during migration and maps command-specific metadata into a stable field without changing error semantics unexpectedly.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an RFC for a new public CLI JSON contract. Source inspection on current main confirms the existing command-specific JSON shapes are still present.

## Solution Assessment

Unclear until maintainers choose the public API. The proposed versioned envelope and legacy flag are plausible, but the final design should explicitly cover command metadata, gateway passthroughs, compatibility timing, and docs/tests across each affected command.
