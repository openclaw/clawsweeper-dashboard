## Summary

Keep open: current main and the latest release still expose command-specific list JSON shapes, and standardizing them needs a maintainer-owned public CLI/API contract and migration policy.

## What This Changes

Request to add a shared versioned JSON envelope and migration flag for list-style CLI `--json` outputs across agents, sessions, models, cron, nodes, and skills.

## Best Possible Solution

Define a shared list-envelope helper/schema plus an opt-in flag across the core list commands, then document and stage any default migration after maintainer approval.

## Reproduction Assessment

Yes. Source inspection on current main shows mixed list JSON shapes and no shared `--json-envelope` or `list-v1` surface; I did not execute CLI commands because this review is read-only.

## Solution Assessment

Unclear until maintainers choose the public contract. A versioned envelope is plausible, but the exact schema, flag name, default behavior, and legacy window are compatibility-sensitive API decisions.
