## Summary

Keep open: current main and v2026.6.10 still have command-specific list JSON shapes and no shared `--json-envelope` or `list-v1` implementation. The request is a valid CLI/API ergonomics RFC, but the schema, flag name, default behavior, and staged compatibility policy need maintainer product direction before implementation.

## What This Changes

Request to standardize list-style CLI `--json` output behind a versioned envelope and staged compatibility flag across agents, sessions, models, cron, nodes, and skills commands.

## Best Possible Solution

Define a shared list-envelope schema/helper and opt-in flag across the core list commands, then document and stage any default migration after maintainer approval.

## Reproduction Assessment

Yes. Source inspection on current main shows mixed list JSON shapes and no shared `--json-envelope` or `list-v1` surface; I did not execute CLI commands because this review is read-only.

## Solution Assessment

Unclear until maintainers choose the public contract. A versioned envelope is plausible, but the exact schema, flag name, default flip, and legacy window are compatibility-sensitive API decisions.
