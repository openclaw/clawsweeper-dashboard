## Summary

Keep open for maintainer review. The patch is not merge-ready because it removes the shipped QR bootstrap handoff that current main and a merged prior PR rely on, and the external real behavior proof is only import smokes rather than a real setup-code pairing run.

## What This Changes

This PR makes setup-code bootstrap profiles node-only, stops fresh bootstrap-token node connects from silent pairing approval, removes extra role tokens from `hello-ok`, and updates gateway/bootstrap/pairing tests.

## Best Possible Solution

Keep the hardening goal as a maintainer security/product decision: if operator handoff must be removed, preserve a one-handshake node bootstrap path or add coordinated client wait/retry onboarding with real setup-code proof.

## Reproduction Assessment

Yes for the current-main behavior and the PR regression from source inspection: current main auto-approves the baseline QR bootstrap path, while the PR makes that path return `PAIRING_REQUIRED` until a later retry. No live PR behavior was demonstrated.

## Solution Assessment

No. Deleting gateway silent approval alone is not the narrowest maintainable fix because it reopens the shipped QR onboarding failure unless native clients or the setup-code flow are updated and proven live.
