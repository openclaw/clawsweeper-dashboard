## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Handle failed delivered marks before returning ran:** `src/auto-reply/continuation/work-dispatch.ts:292`
  - body: `markPendingWorkDelivered` can return `false` when the flow/revision update does not commit, but this path ignores that result and returns `ran`. A crash in that gap can re-deliver the same continuation after restart, so this needs to requeue, skip, or error when the durable delivery mark fails.
  - confidence: 0.92
- **[P1] Keep legacy Codex provider ids out of live routing:** `extensions/codex/harness.ts:74-75`
  - body: This tokenized match accepts legacy ids such as `openai-codex` and `openai:codex` as live Codex harness providers. Current OpenClaw docs and doctor repair migrate those ids to canonical `openai`, and upstream Codex's built-in provider id is `openai`, so live routing should not reintroduce the legacy aliases.
  - confidence: 0.9
- **[P1] Do not inject cross-session returns as trusted system context:** `src/auto-reply/continuation/targeting.ts:113-116`
  - body: Cross-session delegate text can be model-produced and target another session when the gate is enabled, but it is enqueued as trusted. That bypasses OpenClaw's untrusted system-event rendering boundary for model-controlled text and needs a maintainer-approved trust model or an untrusted default.
  - confidence: 0.86
