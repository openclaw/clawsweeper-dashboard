- **[P1] Keep omitted hook agent IDs behind the allowlist:** `src/gateway/hooks.ts:304`
  - body: With `hooks.allowedAgentIds` set, an omitted `agentId` still dispatches to the default agent downstream. Returning `true` here lets callers reach that default agent even when it is not allowlisted; authorize the resolved effective target instead.
  - confidence: 0.93

  - body: This now always offers `Always Allow` and removed the `allowedDecisions`/`ask` policy from prompt decoding. Requests that are supposed to allow only `allow-once` or `deny` can create durable approvals, widening the exec approval boundary.
  - confidence: 0.91
- **[P2] Restore the non-manual Codex compaction guard:** `extensions/codex/src/app-server/compact.ts:131`
  - body: Current main skips native Codex app-server compaction for budget/automatic triggers because the app-server owns that lifecycle. Removing the guard makes OpenClaw invoke native compaction on automatic triggers too, which can fight the harness and change session state unexpectedly.
  - confidence: 0.82
- **[P2] Keep config-open failures visible:** `ui/src/ui/controllers/config.ts:505`
  - body: `config.openFile` can return `{ ok: false, error, path }`; this path now ignores the response and catches errors without setting `lastError`. Users in headless or failed-open environments lose the visible error and file-path fallback.
  - confidence: 0.88
