## Summary

Keep open: current main does not implement the requested continuation tools, but this PR is not merge-ready because the patch weakens the system-event trust boundary for existing external plugin callers.

## What This Changes

The PR adds opt-in agent self-continuation via `continue_work`, `continue_delegate`, and `request_compaction`, with continuation config, TaskFlow-backed state, compaction hooks, tracing, docs, tests, and cross-channel system-event metadata changes.

## Best Possible Solution

Preserve the current untrusted-by-default system-event boundary for existing plugin callers, add an explicit internal trusted path for continuation enrichment, and then review the continuation API/config surface as an opt-in core capability.

## Reproduction Assessment

Yes. Source inspection shows current main sanitizes all queued system-event text, while the PR makes that sanitization conditional on a new downgrade flag even though the enqueue API is public to plugins.

## Solution Assessment

No. The continuation design may be useful, but the trust-boundary change is not the safest implementation; existing external system-event producers should remain untrusted by default and continuation-owned enrichment should opt into a separate trusted path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Keep existing system events untrusted by default:** `src/infra/system-events.ts:154`
  - body: Current main sanitizes every `enqueueSystemEvent` payload before it can be rendered as `System:` prompt text. This change flips the default so callers that do not pass `forceSenderIsOwnerFalse` or `trusted:false` keep raw `System:` / `[System]` markers, but `enqueueSystemEvent` is exported through `plugin-sdk/system-event-runtime` and existing installed plugins will not know to pass the new flag. That creates a prompt-injection regression for external plugin/channel system events; keep the public enqueue path sanitized by default and add a separate explicit trusted internal path for continuation-owned enrichment.
  - confidence: 0.9
- **[P2] Update the public SDK contract for tracer exports:** `src/plugin-sdk/diagnostic-runtime.ts:34-47`
  - body: This adds continuation tracer values and types to the public `plugin-sdk/diagnostic-runtime` subpath, but the SDK docs and generated API baseline are not updated in this diff. Plugin authors need the new public contract documented and baseline-locked before release, especially because this PR also wires a bundled diagnostics plugin to those exports.
  - confidence: 0.78
