Full review comments:

- **[P1] Handle failed delivered marks before returning ran:** `src/auto-reply/continuation/work-dispatch.ts:292`
  - body: markPendingWorkDelivered can return false when the TaskFlow revision update is not durably committed, but this call ignores that and still returns ran.   - confidence: 0.9
- **[P1] Do not revive openai-codex provider routing:** `extensions/codex/harness.ts:73-75`
  - body: This alias path makes openai-codex-style provider ids route into the Codex harness even though OpenClaw docs and doctor migration treat openai-codex as legacy input to rewrite to openai, and upstream Codex's built-in provider id is openai. Keeping a live alias creates a competing source of truth for stale config.
  - confidence: 0.88
- **[P1] Do not trust cross-session delegate text by default:** `src/auto-reply/continuation/targeting.ts:113-116`
  - body: Delegate return text is model-produced content, but cross-session deliveries enqueue it as trusted System context for the recipient. When crossSessionTargeting is enabled this lets one session inject unsanitized trusted instructions into another session, so the receive path needs an untrusted downgrade or an explicit maintainer-approved trust contract.
  - confidence: 0.87
