## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Handle failed delivered marks before returning ran:** `src/auto-reply/continuation/work-dispatch.ts:292`
  - body: `markPendingWorkDelivered()` returns `false` when the expected-revision update is not durably applied, but this call ignores that and reports the wake as `ran`. In that case the process can crash or recover without the required `succeeded` marker, so the same continuation turn can be consumed again despite the no-redelivery invariant documented in `work-store.ts`.
  - confidence: 0.88
- **[P1] Do not revive openai-codex provider routing:** `extensions/codex/harness.ts:73-75`
  - body: This accepts `openai-codex`/`OpenAI-Codex`/`openai_codex` as live Codex harness provider ids at priority 100. Current docs and doctor migrations treat `openai-codex` as legacy input that must be rewritten to canonical `openai`, so this creates a competing runtime path for stale config instead of forcing the supported repair path.
  - confidence: 0.86
- **[P1] Do not trust cross-session delegate text by default:** `src/auto-reply/continuation/targeting.ts:113-116`
  - body: Delegate return text can be model-produced, but the delivery path always enqueues it with `trusted: true`, including for cross-session targets. When `crossSessionTargeting` is enabled this lets one session inject unsanitized system-context text into another session; downgrade/sanitize that text or require an explicit maintainer-approved trusted injection contract before merging.
  - confidence: 0.82
