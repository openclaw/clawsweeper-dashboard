## Summary

Keep open, but do not merge as-is: the PR still lacks real Telegram proof and the diff regresses the documented `replyToMode` draft-stream threading path by removing `replyToMessageId` from Telegram draft previews.

## What This Changes

The PR removes implicit Telegram draft-stream reply targets, softens assistant reply-tag prompt guidance, and updates Telegram, system-prompt, and qa-lab mock tests.

## Best Possible Solution

Land a rebased, narrow change that reduces routine explicit reply tags while preserving configured Telegram `replyToMode` draft threading, backed by live Telegram proof.

## Reproduction Assessment

Yes. Source inspection on current main shows non-off `replyToMode` is passed into Telegram draft stream creation and then converted into `reply_to_message_id`; I did not run a live Telegram session in this read-only review.

## Solution Assessment

No. Removing `replyToMessageId` from all draft streams is broader than the routine-quote problem and conflicts with the documented `replyToMode` contract; the safer fix is to narrow prompt or explicit-tag behavior while preserving configured implicit threading.
