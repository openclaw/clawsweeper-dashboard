## Best Possible Solution

## Reproduction Assessment

Yes for the regression risk: current-main source and tests show replyToMode forwards replyToMessageId into Telegram draft streams, and the PR diff removes that forwarding. I did not run a live Telegram session, and the PR does not include real Telegram proof for the claimed routine-quote improvement.

## Solution Assessment

No: narrowing assistant reply-tag guidance may be the right direction, but removing draft-stream reply metadata is not the best fix because it conflicts with the existing replyToMode threading contract. The safer path preserves draft threading and changes only the prompt/explicit-tag behavior unless maintainers intentionally update the contract.
