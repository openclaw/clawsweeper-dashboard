## Best Possible Solution

Refresh on current main and split routine preview behavior from explicit quote finalization: routine streamed previews should avoid implicit native quotes, while explicit reply tags/current-message quote replies must still send a quoted preview or quoted final message.

## Reproduction Assessment

## Solution Assessment

No. Removing the default draft-stream reply target is plausible for routine previews, but the patch also removes the mechanism that lets retained streamed finals preserve explicit native quote replies.
