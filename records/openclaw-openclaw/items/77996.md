## Summary

Keep open: the PR targets a real source-reproducible reset-history gap and now includes useful live gateway terminal proof, but it is not merge-ready because archived reads bypass the existing chat-history projection/budget safeguards and the new English UI strings were not synced into generated locale files.

## What This Changes

The PR adds archived transcript discovery/read RPCs, exposes archive previews on session rows, renders archived transcript messages/images in the Control UI Sessions panel, and adds gateway/UI tests plus a changelog entry.

## Best Possible Solution

Keep the archive-browser direction, but make `sessions.archived.read` return display-safe, byte-bounded history data and sync generated Control UI locale outputs before normal maintainer review proceeds.

## Reproduction Assessment

Yes. From source inspection, reset/delete paths archive transcript files on disk while current main has no Control UI archived-transcript surface, and the linked issue includes a concrete archive filename from OpenClaw 2026.5.3-1.

## Solution Assessment

No, not yet. The chosen UI/API direction is narrow and plausible, but the archived-read RPC should reuse the existing chat-history projection and byte-budget safeguards, and the Control UI i18n generated outputs need to be committed.
