## Summary

Keep open. Current main still lets `openclaw sandbox list` inspect OpenShell registry entries without loading the full plugin registration that provides the OpenShell backend manager, so the persistent stopped-status symptom remains source-reproducible.

## What This Changes

The issue reports that OpenShell sandboxes created through the configured OpenClaw backend are shown as stopped or missing after enabling the OpenShell plugin.

## Best Possible Solution

Add a generic sandbox CLI backend activation path for sandbox lifecycle and status commands before registry inspection, with regression coverage and no OpenShell-specific hardcode in core.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `openclaw sandbox list` falls through to the default no-plugin CLI policy, OpenShell only registers its backend manager during full plugin registration, and managerless entries are displayed as stopped.

## Solution Assessment

Yes for the known stopped-status symptom. The maintainable fix is a generic sandbox CLI plugin/backend activation path, not an OpenShell-specific hardcode or config migration.
