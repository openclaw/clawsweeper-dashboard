Yes, at source level: current main leaves sandbox commands on the default no-plugin-loading policy, OpenShell registers its manager only during full plugin registration, and the listing path reports stopped when that manager is missing.

## Solution Assessment

Yes, the open linked PR is the current best candidate because it repairs the missing plugin registration path with focused tests; the remaining question is maintainer acceptance of all-plugin activation for sandbox CLI commands.
