## Summary

Keep open: current main and the latest release still leave sandbox CLI commands on the default no-plugin-loading path, so OpenShell can create/register runtime state during agent execution while `openclaw sandbox list` still reports plugin-owned entries as stopped; the open linked pull request is the candidate fix and should be reviewed rather than closing this issue early.

## What This Changes

The issue asks OpenClaw to make the bundled OpenShell sandbox backend report live sandbox status correctly through the normal sandbox CLI.

## Best Possible Solution

Review and land or revise https://github.com/openclaw/openclaw/pull/87696, then close this issue once the OpenShell sandbox CLI status path is fixed on main.

## Reproduction Assessment

Yes, at source level: current main defaults sandbox commands to no plugin loading, OpenShell registers its manager only during full plugin registration, and missing managers report registry entries as stopped. I did not run a live WSL+OpenShell environment in this read-only review.

## Solution Assessment

Unclear as a final merge shape: the linked PR is the narrowest fix using the current CLI/plugin registry surface, but maintainers still need to accept full plugin activation for sandbox commands or request a narrower sandbox-backend activation scope.
