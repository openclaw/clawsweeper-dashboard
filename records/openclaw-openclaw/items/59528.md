## Summary

Keep open. Current main still lets `openclaw sandbox list` inspect OpenShell registry entries without loading the full OpenShell plugin registration that provides the backend manager, so the stopped-status symptom remains source-reproducible.

## What This Changes

The issue reports that OpenShell-backed sandboxes appear stopped in `openclaw sandbox list` after enabling the OpenShell sandbox backend.

## Best Possible Solution

Load or activate configured sandbox backend plugins for sandbox lifecycle/status CLI commands before registry inspection, then add focused regression coverage without hardcoding OpenShell in core.

## Reproduction Assessment

Yes, as a high-confidence source-level reproduction rather than a live WSL/OpenShell run. On current main, `openclaw sandbox list` can run with plugin loading disabled, leaving the OpenShell backend manager unregistered and causing registry entries for that backend to be displayed as stopped.

## Solution Assessment

Yes for the known stopped-status symptom. The maintainable fix is a generic sandbox CLI backend activation path, coordinated with the existing split-module registry work, not an OpenShell-specific hardcode.

## Repair Work Prompt

Fix the OpenShell sandbox CLI stopped-status bug reported in https://github.com/openclaw/openclaw/issues/59528. On current main, `openclaw sandbox list` has no sandbox command policy, so CLI bootstrap uses the default `loadPlugins: "never"`; OpenShell registers its backend manager only during full plugin registration, and `src/agents/sandbox/manage.ts` renders registry entries as stopped when `getSandboxBackendManager("openshell")` is missing. Add a generic sandbox lifecycle/status CLI startup path that loads/activates configured sandbox backend plugins before `sandbox list` and `sandbox recreate` inspect/remove registry entries. Do not hardcode OpenShell in core, do not add a new config option, and do not duplicate the process-global backend registry repair tracked by https://github.com/openclaw/openclaw/issues/67610 and https://github.com/openclaw/openclaw/pull/67623; coordinate with that path if current externalized-plugin behavior requires it. Add focused regression coverage for CLI command policy/startup and sandbox manager registration/fallback behavior, and add a changelog fix entry because the behavior is user-visible.
