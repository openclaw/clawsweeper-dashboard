## Summary

Keep open: current main and the latest release still leave `openclaw sandbox` on the default no-plugin-loading path, so OpenShell backend managers are not registered for sandbox status. The open linked PR is the candidate fix and should be reviewed or revised before this issue closes.

## What This Changes

The issue reports that OpenShell-backed sandbox entries stay `stopped` in `openclaw sandbox list` after enabling the bundled OpenShell plugin.

## Best Possible Solution

Review and land or revise https://github.com/openclaw/openclaw/pull/87696 with an accepted sandbox-backend activation policy, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes, at source level: current main leaves sandbox commands on no-plugin startup, OpenShell registers its manager only during full plugin registration, and missing managers report registry entries as stopped. I did not run a live WSL+OpenShell environment in this read-only review.

## Solution Assessment

Not yet as a final outcome: the open linked PR is the concrete candidate fix, but maintainers still need to accept broad plugin activation for sandbox commands or choose a narrower backend-manager activation path.
