## Summary

Keep open: current main and the latest release still lack a sandbox CLI plugin-loading policy, so OpenShell can register its backend for agent runtime but not for sandbox CLI status lookup; the open linked pull request is the active candidate fix and should be reviewed instead of closing this issue now.

## What This Changes

The issue reports that OpenShell-backed sandboxes enabled through the bundled plugin are shown as stopped or missing by OpenClaw sandbox CLI commands even when OpenShell has created the sandbox.

## Best Possible Solution

Review and land or revise the linked fix so sandbox CLI commands load the backend plugin registration path before reading OpenShell-backed sandbox registry entries.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: sandbox CLI policy does not load plugins, OpenShell registers its backend manager only during full plugin registration, and listSandboxContainers falls back to running false when that manager is absent.

## Solution Assessment

Unclear as a final merge shape. Loading plugins for sandbox commands is a plausible narrow fix using the current CLI/plugin registry surface, but maintainers still need to accept full plugin activation for sandbox commands or request a narrower sandbox-backend activation scope.
