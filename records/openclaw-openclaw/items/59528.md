## Summary

Keep open. Current main and the latest stable release still have the source-level path from the prior ClawSweeper review: `openclaw sandbox list` falls through to a no-plugin-load CLI policy, so an OpenShell registry entry can be rendered as stopped without the OpenShell backend manager ever being registered.

## What This Changes

The issue reports that OpenShell-backed sandboxes enabled from the docs appear stopped in `openclaw sandbox list` and may not show up in `openshell sandbox list`.

## Best Possible Solution

Activate configured sandbox-backend plugins for sandbox lifecycle/status CLI commands before reading registry entries, while coordinating with the process-global registry fix in https://github.com/openclaw/openclaw/pull/67623.

## Reproduction Assessment

Yes, as a source-level reproduction rather than a live WSL/OpenShell run. Current main resolves `openclaw sandbox ...` through the default no-plugin-load policy, bootstrap returns before plugin registration, and `listSandboxContainers()` marks entries without a manager as stopped.

## Solution Assessment

Yes for the known stopped-status symptom. The maintainable fix is generic sandbox CLI backend activation, not an OpenShell-specific hardcode and not only the separate registry-globalization PR.
