## Summary

Keep open. Current main and the latest stable release still allow `openclaw sandbox list` to inspect OpenShell registry entries without loading the plugin that registers the OpenShell backend manager, so those entries can still render as stopped.

## What This Changes

The issue reports that OpenShell-backed sandboxes enabled through the documented plugin setup appear stopped in OpenClaw sandbox status commands.

## Best Possible Solution

Add a generic sandbox CLI startup path that loads or activates configured sandbox backend plugins before lifecycle/status registry inspection, without hardcoding OpenShell, and cover both manager registration and the missing-manager fallback.

## Reproduction Assessment

Yes, source-level. Current main resolves the sandbox CLI through default plugin-skipping startup, while OpenShell registers its backend manager only during full plugin registration and `listSandboxContainers` marks missing managers as stopped.

## Solution Assessment

Yes for the known stopped-status symptom. The maintainable fix is generic sandbox backend plugin activation for sandbox CLI commands, not an OpenShell-specific special case in core status rendering.
