## Summary

Keep open; this docs PR should not merge as-is because it turns one local Gateway cron inventory into public docs and includes retry guidance that conflicts with current source/config docs, with no after-fix real behavior proof.

## What This Changes

The PR adds a new public automation docs page with a generated cron resilience audit from a local Gateway `jobs.json` and links it from the automation index.

## Best Possible Solution

Replace this with a generic, redacted cron resilience checklist tied to the existing cron docs, or keep the generated local audit in private/internal operator docs outside the public documentation set.

## Reproduction Assessment

Not applicable; this is a docs-only PR, not a reported runtime bug. The concerns are reproducible by reading the PR diff against current docs policy and cron source.

## Solution Assessment

No. A public page generated from one operator's `jobs.json` is not the right solution; the maintainable path is generic public guidance or a private internal audit.
