## What This Changes

The PR changes the Gateway session-kill path resolver to distinguish unmatched paths from invalid matched session keys and adds regression coverage for `/sessions/%zz/kill` and `/sessions/%20/kill` returning `400 invalid_request_error` before auth or session lookup.

## Best Possible Solution

Land this focused Gateway parser and regression test after required exact-head checks and the proof-gate status are resolved.

## Reproduction Assessment

Yes, as a source-reproducible current-main bug: the route shape is staged, `decodeURIComponent()` fails or trims empty, and the handler currently returns `false`, allowing fallback handling instead of a deterministic API error.

## Solution Assessment

Yes. The discriminated resolver result is the narrow maintainable fix, matches the sibling session-history invalid-key boundary, and avoids auth/session/kill side effects for malformed matched paths.
