## Summary

Keep open: current main still treats malformed matched session-kill paths as unhandled, while this PR is a narrow parser/test fix with sufficient real Gateway proof and no actionable review findings.

## What This Changes

The PR changes Gateway session-kill path parsing to return `400 invalid_request_error` for matched paths with malformed or decoded-empty session keys and adds focused regression coverage.

## Best Possible Solution

Land the focused parser and regression-test change after normal maintainer review, preserving valid session-kill auth, ownership, lookup, and kill semantics.

## Reproduction Assessment

Yes. Current-main source plus a focused URL/decode check show malformed or decoded-empty matched session-kill paths return `false` from the handler; the PR body also supplies before/after live Gateway output for the same paths.

## Solution Assessment

Yes. Distinguishing unmatched paths from invalid matched keys is the narrow maintainable fix, and it aligns session-kill behavior with the sibling session-history handler without changing valid request semantics.
