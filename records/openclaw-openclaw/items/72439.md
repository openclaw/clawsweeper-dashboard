## Summary

Keep open for merge review: current main still has the malformed session-kill path fallthrough, and this PR addresses it narrowly without changing valid session-kill auth, ownership, scope, or kill semantics.

## What This Changes

This PR changes the Gateway session-kill HTTP parser to return `400 invalid_request_error` for malformed matched session keys, adds focused regression coverage, and records the fix in `CHANGELOG.md`.

## Best Possible Solution

Land this focused gateway parser, regression test, and changelog fix after exact-head checks finish green or any unrelated CI status is explicitly accepted.

## Reproduction Assessment

Yes. Source inspection on current main plus a focused Node URL/decode check shows `/sessions/%zz/kill` matches the session-kill route shape, `decodeURIComponent()` throws, and the handler returns `false` instead of a deterministic API error.

## Solution Assessment

Yes. The discriminated path-resolution result is the narrow maintainable fix, matches the sibling session-history invalid-key behavior, and avoids auth/session/kill side effects for malformed matched paths.
