## Summary

Keep open: current main still has the malformed session-kill path fallthrough, while this PR is narrow, tested, and has no concrete functional or security finding.

## What This Changes

The PR changes Gateway session-kill path parsing to return `400 invalid_request_error` for malformed matched session keys and adds a regression test plus changelog entry.

## Best Possible Solution

Land this narrow gateway parser, regression test, and changelog change after normal PR merge gates, preserving valid session kill auth, ownership, scope, kill, and RPC behavior.

## Reproduction Assessment

Yes. Current main can reproduce the bug from source plus a focused Node URL/decode check: `/sessions/%zz/kill` matches the route shape, `decodeURIComponent()` throws, and the handler treats that as unhandled.

## Solution Assessment

Yes. Distinguishing unmatched paths from matched invalid session keys is the narrowest maintainable fix and aligns session-kill with the sibling session-history API boundary.
