## Summary

## What This Changes

This PR adds a `--all` option to `openclaw models status --probe`, threads it through route and command handling, expands probe target planning to every configured model, and adds focused tests.

## Best Possible Solution

Land a refreshed implementation that preserves default one-model-per-provider probes, adds the all-model mode through CLI and route parsing, updates the summary/docs contract, and includes real provider proof.

Yes. Source inspection on current main shows no `--all`/`probeAll` status path and `buildProbeTargets()` still selects one model with `selectProbeModel()`; I did not run live provider probes in this read-only review.

## Solution Assessment
