## Summary

Keep open. Current main still lacks the requested per-configured-model probe mode, and this PR is a narrow implementation path for #63145, but it should not merge as-is because the exported probe summary type and public docs/changelog are not aligned with the new flag.

## What This Changes

The PR adds `--all` plumbing for `openclaw models status --probe`, expands probe target selection to every configured model candidate, warns when `--all` is used without `--probe`, and adds focused route-parser and probe-target tests.

## Best Possible Solution

Land a refreshed version of this PR that preserves the current one-model-per-provider default, exposes `probeAll` consistently in the probe summary contract, and documents the new user-facing flag before release.

## Reproduction Assessment

Yes. The current-main gap is visible statically: no status `--all` flag is registered or parsed, and `buildProbeTargets()` chooses one model per provider. The remaining PR defect is also directly reproducible from the diff because `AuthProbeOptions.probeAll` is returned through `AuthProbeSummary.options` without being declared there.

## Solution Assessment

No, not as-is. The implementation strategy is the right narrow path because it reuses the existing probe target builder and execution flow, but the summary type, docs, and changelog need to be aligned before merge.
