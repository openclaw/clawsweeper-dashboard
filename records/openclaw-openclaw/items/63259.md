## Summary

Keep open: current main still lacks the requested all-configured-model probe mode, and this PR is the active implementation path, but it is not merge-ready because it still needs real behavior proof plus small contract/docs fixes.

## What This Changes

The PR adds a `--all` mode to `openclaw models status --probe` so every configured model candidate is probed instead of one model per provider.

## Best Possible Solution

Land a refreshed implementation that preserves default one-model-per-provider probes, exposes `probeAll` in the summary contract, documents the CLI flag, keeps focused regression tests, and includes redacted real CLI/provider proof.

## Reproduction Assessment

Yes for the requested feature gap: source inspection of current main shows no status-level `--all`/`probeAll` path, and probe planning still selects one model per provider through `selectProbeModel()`. This is a feature request rather than a broken existing contract.

## Solution Assessment

No, not as-is. Extending the existing `models status --probe` machinery is the right direction, but the PR should also expose `probeAll` in `AuthProbeSummary.options`, update the status docs, and include real behavior proof before merge.
