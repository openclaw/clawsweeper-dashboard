The PR adds a `--all` flag to `openclaw models status --probe`, threads `probeAll` through CLI routing and status execution, expands probe targets across configured model candidates, and adds focused route/probe-target tests.

## Best Possible Solution

Land a refreshed implementation that preserves default one-model-per-provider probes, expands configured model candidates behind `--probe --all`, exposes `probeAll` in the summary contract, documents the flag, and includes redacted real CLI/provider proof.

## Reproduction Assessment

Yes for the requested feature gap: source inspection on current main shows no status-level `--all`/`probeAll` path, while `buildProbeTargets()` still selects one model per provider. This is a feature request rather than a broken existing contract.

## Solution Assessment

No, not as-is. Extending the existing `models status --probe` machinery is the right direction, but this branch should add the summary contract field, update docs, and include real behavior proof before merge.
