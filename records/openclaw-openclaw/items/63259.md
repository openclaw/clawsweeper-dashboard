This PR adds a `--all` status-probe flag, routes it through CLI parsing/status handling, expands probe target selection to all configured model candidates, and adds focused target/parser tests.

## Best Possible Solution

Land a refreshed implementation that preserves default one-model-per-provider probes, exposes `probeAll` in the summary contract, documents the CLI flag, and includes redacted real CLI/provider proof.

Yes. Source inspection of current main shows no status-level `--all` flag, no routed `probeAll` field, and one selected probe model per provider; the PR’s target tests cover the intended expansion but no live provider repro was supplied.

## Solution Assessment

No, not as-is. Extending the existing `models status --probe` path is the narrow maintainable direction, but the exported summary type, public docs, and real behavior proof need to be completed before merge.
