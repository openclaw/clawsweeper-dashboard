## Summary

Keep open. The PR addresses a real feature gap and implements most of the plumbing, but the patch still omits the new mode from the exported probe summary contract, lacks docs, and the external contributor has not supplied real behavior proof.

## What This Changes

Adds a `models status --probe --all` CLI mode that threads a `probeAll` option through command parsing/status probing and expands probe targets to every configured model, with route/parser and probe-target tests.

## Best Possible Solution

Land a refreshed implementation that preserves default one-model-per-provider probes, exposes `probeAll` in the summary contract, documents the CLI flag, and includes redacted real CLI/provider proof.

## Reproduction Assessment

Yes for the requested feature gap. Source inspection of current main shows no status-level `--all`/`probeAll` path, and `buildProbeTargets()` still selects one model per provider through `selectProbeModel()`.

## Solution Assessment

No, not as-is. Extending the existing `models status --probe` machinery is the right direction, but the exported summary type, public docs, and real behavior proof need to be completed before merge.
