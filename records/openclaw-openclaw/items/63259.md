## Summary

Keep open: this is the active implementation for the open per-model probe feature, but it still needs contributor real behavior proof plus small code/docs fixes before merge.

## What This Changes

The PR adds a `models status --probe --all` CLI path that threads `probeAll` through route parsing, Commander, status execution, probe target selection, and focused route/probe target tests.

## Best Possible Solution

Land a refreshed implementation that preserves default one-model-per-provider probes, expands configured model candidates behind `--probe --all`, exposes `probeAll` in the summary contract, updates docs/tests, and includes redacted real CLI/provider proof.

## Reproduction Assessment

Yes for the requested feature gap: current main has no status-level `--all`/`probeAll` route, and `buildProbeTargets()` selects one model per provider through `selectProbeModel()`. This is a feature request rather than a broken existing contract.

## Solution Assessment

No, not as-is. Extending the existing `models status --probe` machinery is the right direction, but the branch should add `probeAll` to `AuthProbeSummary.options`, document the flag, and include real behavior proof before merge.
