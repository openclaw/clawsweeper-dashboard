- `impact:message-loss`: The report is about bot replies being delivered to the wrong Feishu conversation surface instead of the originating DM topic.

## Summary

Keep open: current main still treats Feishu DM topic messages as flat direct messages, and the existing thread-aware routing/reply logic is limited to group/topic-group paths. The report is source-reproducible, but the fix needs a maintainer decision because changing DM topic sessions can split existing DM history and would reverse the intentional direct-DM no-reply-reference behavior.

## What This Changes

Request Feishu DM branched topic messages to route and reply inside the originating topic instead of the main DM window.

## Best Possible Solution

Add a Feishu-owned DM topic mode that preserves the existing flat-DM default unless maintainers choose otherwise, carries root_id/thread_id through session and delivery routing, and documents/tests both upgrade-safe defaults and opted-in DM topic behavior.

## Reproduction Assessment

Yes at source level, but not with a live Feishu run in this review. A p2p event with root_id/thread_id still routes by senderOpenId, omits lastRouteThreadId, and strips the reply target before outbound send.

## Solution Assessment

Unclear as proposed because appending thread IDs to all DM sessions would change existing DM history and reply semantics. The safer path is an explicit Feishu DM topic mode, or a maintainer-approved default change with upgrade tests.
