## Summary

## What This Changes

This PR adds modelId-based Gemini detection to tool schema normalization and regression tests for custom OpenAI-compatible proxy providers.

## Best Possible Solution

Port the modelId-based Gemini check into src/agents/pi-tools-parameter-schema.ts, compose Gemini cleanup with modelCompat keyword stripping, and keep focused regression coverage for Gemini models behind custom OpenAI-compatible providers.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call normalizeToolParameterSchema with a custom proxy provider, a Gemini modelId, modelCompat unsupported keywords, and a Gemini-rejected schema; live NewAPI/OneAPI proof is still missing.

## Solution Assessment

No, not as written. The modelId predicate is the right narrow direction, but the patch needs to land in the current split helper and preserve modelCompat unsupported-keyword stripping after Gemini cleanup.
