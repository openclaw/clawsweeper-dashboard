## Summary

Keep open. The PR addresses a real source-level sandbox hash invalidation gap and has no blocking review findings; it still needs normal maintainer merge judgment and coordination with the later create-flag change.

## What This Changes

The PR adds a neutral sandbox Docker create-args epoch to sandbox and browser config hashes, labels created containers with it, updates hash coverage, and adds a changelog entry.

## Best Possible Solution

Merge this mechanism after maintainer approval, then require #74083 or any future Docker create-flag change to bump the same epoch in the release that changes the flags.

## Reproduction Assessment

Yes, source-level. Current main reuses containers by openclaw.configHash, and that hash excludes code-level buildSandboxCreateArgs flag epochs, so a create-arg-only flag change would not invalidate long-lived containers.

## Solution Assessment

Yes. A neutral epoch in the existing hash inputs is a narrow cache-invalidation mechanism and avoids claiming that the separate `--init` rollout has already shipped.
