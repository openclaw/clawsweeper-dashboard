## Summary

Keep open: current main does not yet include this mechanism, and the PR is a narrow, useful fix for a source-level sandbox config-hash invalidation gap with no blocking review findings.

## What This Changes

The PR adds a neutral Docker create-args epoch to sandbox and browser-sandbox config hashes, stamps it as a Docker label, updates regression tests, and adds a changelog entry.

## Best Possible Solution

Merge this mechanism after maintainer review, then require #74083 and future Docker create-flag changes to bump the epoch in the same release as the flag.

## Reproduction Assessment

Yes, source-level. Current main reuses sandbox and browser containers by openclaw.configHash, and that hash excludes a global buildSandboxCreateArgs create-args epoch, so a flag-only create-args change would not invalidate long-lived containers.

## Solution Assessment

Yes. The PR extends the existing config-hash recreate path with a neutral bootstrap epoch and leaves the actual --init rollout to #74083, avoiding a speculative hash bump.
