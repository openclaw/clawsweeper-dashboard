## Summary

Keep this PR open. Current main still has the sandbox create-args hash gap, while the revised PR is a narrow mechanism-only fix with no blocking correctness or security findings and a clean check-run snapshot.

## What This Changes

The PR adds a neutral Docker create-args epoch to sandbox and browser-sandbox config hashes, stamps it as a Docker label, updates focused tests, and adds a changelog entry.

## Best Possible Solution

Merge the mechanism after maintainer review, then require each future Docker create-flag change to bump the epoch alongside the real flag change.

## Reproduction Assessment

Yes, source-level. Current main reuses containers by openclaw.configHash, and that hash excludes any global buildSandboxCreateArgs epoch, so a create-arg-only flag change would not invalidate long-lived containers.

## Solution Assessment

Yes. A neutral epoch included in both sandbox hashes is the narrowest maintainable mechanism, and the actual flag rollout remains responsible for bumping the value when it ships.
