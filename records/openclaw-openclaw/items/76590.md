## Summary

This PR still changes behavior that is not on current main, and the patch looks like a narrow, appropriate mechanism for invalidating long-lived sandbox containers when global Docker create-args flags change.

## What This Changes

The PR adds a neutral sandbox Docker create-args epoch to sandbox and browser config hashes, stamps it as a diagnostic Docker label, updates hash/recreate tests, and adds an Unreleased changelog entry.

## Best Possible Solution

Merge this mechanism after required checks and maintainer review, then require any future Docker create-flag change such as #74083 to bump the epoch in the same release.

## Reproduction Assessment

Yes, source-level. Current main reuses containers by openclaw.configHash, and that hash cannot change for a global buildSandboxCreateArgs-only flag change because no create-args epoch is part of the hash input.

## Solution Assessment

Yes. A neutral epoch in the existing config-hash path is narrower than adding per-flag migration code, and the PR avoids claiming the separate --init behavior before #74083 lands.
