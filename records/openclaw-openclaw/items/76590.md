## Summary

Keep open for normal PR review: current main omits Docker create-args changes from the sandbox reuse hash, and the PR wires a neutral epoch through the existing recreate path with focused tests and changelog coverage.

## What This Changes

Adds a sandbox Docker create-args epoch to normal and browser config hashes, stamps it as a Docker label, updates hash/recreate tests, and adds a changelog entry.

## Best Possible Solution

Merge this epoch mechanism after required checks and maintainer review, then require any Docker create-flag PR such as #74083 to bump the epoch with the actual flag change.

## Reproduction Assessment

Yes, source-level. Current main reuses containers by openclaw.configHash, and a buildSandboxCreateArgs-only flag change cannot change that hash because no create-args epoch is part of the hash input.

## Solution Assessment

Yes. A neutral epoch in the existing config-hash path is narrower than per-flag migration code, and it avoids claiming the separate --init behavior before #74083 lands.
