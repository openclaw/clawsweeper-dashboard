## What This Changes

The issue reports a legacy plugin install index migration deadlock where stale installs.json metadata can conflict with SQLite installed_plugin_index records and leave npm plugins unloaded.

## Best Possible Solution

Land or reject the narrow authoritative-SQLite archival policy in https://github.com/openclaw/openclaw/pull/94086, then verify this report's unloaded-plugin symptom with plugin ids, gateway logs, and registry/loader diagnostics before closing.

## Reproduction Assessment

Yes for the stale conflict loop: current source returns before archiving plugins/installs.json when conflicts remain. The unloaded-plugin part is not yet high-confidence because the loader path reads SQLite install records, so it needs live gateway/plugin diagnostics.

## Solution Assessment

Unclear as a full fix: state migration is the right owner for retiring stale legacy install indexes, but the safe archival boundary is compatibility-sensitive and the loader symptom may need a separate root-cause check.
