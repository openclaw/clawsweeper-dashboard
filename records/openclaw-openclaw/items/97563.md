## Summary

Keep open: current main still preserves conflicting legacy install-index records, and the open candidate PR only covers one migration slice while this report adds a distinct current-release plugin-unloaded symptom that is not safe to close as a duplicate.

## What This Changes

The issue reports that divergent legacy `plugins/installs.json` and SQLite `installed_plugin_index` records can leave installed npm plugins absent from the gateway-loaded plugin set after upgrade or reinstall.

## Best Possible Solution

Review and, if accepted, land the narrow authoritative-SQLite install-index policy in https://github.com/openclaw/openclaw/pull/94086, then add a focused gateway-loader regression or follow-up if any npm plugin remains absent from the loaded set.

## Reproduction Assessment

## Solution Assessment

Unclear. The right layer appears to be core state migration plus gateway plugin loading validation, but maintainers still need to accept the SQLite-authoritative archival boundary and confirm whether the open PR covers the runtime symptom.
