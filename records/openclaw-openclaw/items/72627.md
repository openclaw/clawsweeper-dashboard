## Summary

Keep open: current main still lacks the requested per-agent directive guard and per-agent automatic-send suppression. The directive-only portion is tracked separately at https://github.com/openclaw/openclaw/issues/73272, but this issue also asks for broader agent-scoped outbound policy design.

## What This Changes

The issue requests opt-in per-agent controls to disable `[[reply_to_current]]` and make assistant final text private unless sent through the `message` tool.

## Best Possible Solution

Define one documented agent-scoped outbound policy that can reject or rewrite `reply_to_current`, preserve explicit `reply_to:<id>` and message-tool sends, and compose predictably with existing visible-reply modes.

## Reproduction Assessment

Yes. Source inspection shows `[[reply_to_current]]` is still parsed into reply metadata, while automatic final delivery is governed by global/group/requested/harness visible-reply policy rather than per-agent config.

## Solution Assessment

Unclear. A per-agent directive guard is plausible, but pairing it with per-agent auto-send suppression overlaps existing `messages.visibleReplies`, native-command exceptions, message-tool availability, and fallback behavior.
