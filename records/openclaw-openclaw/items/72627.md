## Summary

Keep open: current main still accepts and recommends `[[reply_to_current]]`, and the available automatic-send controls are global/group visible-reply modes rather than the requested per-agent outbound policy. The narrower #73272 tracks only the directive opt-out; this issue also covers per-agent automatic-send suppression.

## What This Changes

Request adds per-agent controls to disable the `[[reply_to_current]]` output directive and suppress automatic visible replies so only explicit message-tool sends are delivered.

## Best Possible Solution

Define one agent-scoped outbound policy contract that can reject or rewrite `reply_to_current`, preserve explicit `reply_to:<id>` and message-tool sends, and layer predictably over existing visible-reply modes with schema, docs, prompt, and regression tests updated together.

## Reproduction Assessment

Yes. A high-confidence source-level path exists: assistant text containing `[[reply_to_current]]` is parsed into `replyToCurrent`, while automatic final delivery is controlled by global/group/harness visible-reply policy rather than per-agent config.

## Solution Assessment

Unclear. The directive-only guard is plausible and #73272 is the narrower repair path, but this issue's per-agent auto-send policy needs maintainer design because it overlaps existing visible-reply modes and fallback behavior.
