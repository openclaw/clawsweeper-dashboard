Keep open: current main still lacks both requested per-agent controls. The directive-only part is covered by a narrower open issue, but this broader request also asks for agent-scoped automatic-send suppression and needs maintainer design.

## What This Changes

Requests agent-scoped config to disable `[[reply_to_current]]` processing and suppress automatic visible replies so only explicit message-tool sends reach the channel.

## Best Possible Solution

Define a documented agent-scoped outbound policy contract that can reject or rewrite `reply_to_current`, preserve explicit `reply_to:<id>` and message-tool sends, and compose predictably with existing visible-reply modes.

## Reproduction Assessment

Yes. Source inspection shows assistant text containing `[[reply_to_current]]` is still parsed and propagated, while automatic final delivery is controlled by global/group/requested visible-reply policy rather than per-agent config.

## Solution Assessment

Unclear. A directive-only guard is bounded and separately tracked at https://github.com/openclaw/openclaw/issues/73272, but adding per-agent auto-send suppression needs maintainer design because it overlaps existing visible-reply and native-command behavior.
