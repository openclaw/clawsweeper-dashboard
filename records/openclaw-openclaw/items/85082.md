## What This Changes

The issue reports that Telegram topic `/status` can conflate a fallback model used by an internal subagent announce/recovery turn with the user-facing topic session model.

## Best Possible Solution

Make session usage/status provenance-aware so subagent_announce or other inter-session internal runs are either labeled as internal last-run usage or excluded from the user-facing topic model/fallback fields, with a focused Telegram/forum-topic regression test.
