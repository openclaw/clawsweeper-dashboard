Full review comments:

- **[P1] Authorize agent-scoped SSRF relaxations before applying them:** `src/agents/openclaw-tools.ts:139-142`
  - body: This selects the per-agent `web_fetch` SSRF policy from the session-derived agent and applies it directly to tool construction. Gateway tool invocation accepts `sessionKey`/`agentId` input and passes that session into tool resolution, so a caller that can target a relaxed agent can run `web_fetch` against private/internal targets under that agent's policy. Add an explicit trusted/owner authorization boundary for SSRF-relaxing agent policies, or keep agent overrides restrictive-only until that boundary exists.
  - confidence: 0.87
- **[P1] Normalize alias overrides before merging SSRF policies:** `src/infra/net/ssrf.ts:82`
  - body: The spread merge lets the legacy `allowPrivateNetwork` alias and the new `dangerouslyAllowPrivateNetwork` flag conflict across scopes. For example, a global `allowPrivateNetwork: true` survives an agent override of `dangerouslyAllowPrivateNetwork: false`, so a named agent cannot actually tighten the global private-network posture even though the PR says agent fields override global fields.
  - confidence: 0.83
- **[P2] Report only actual per-agent SSRF widening:** `src/security/audit.ts:1146`
  - body: The new audit warning fires whenever the effective policy allows private-network access, even when the global baseline already allows it and the agent only adds an unrelated narrow exception. That produces misleading `Agent X widens` warnings and hides which agent actually changed the security posture; compare the agent policy against the global baseline before emitting the widening warning.
  - confidence: 0.78
