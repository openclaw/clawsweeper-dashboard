  - confidence: 0.9

  - body: This branch reports `Agent <id> widens web_fetch SSRF access` whenever the effective policy allows private-network access, even if that permission was inherited from the global baseline and the agent only set an unrelated exception. That makes `openclaw security audit` noisy and misleading for operators reviewing real per-agent exceptions.
  - confidence: 0.86
- **[P3] Canonicalize SSRF policy arrays in cache keys:** `src/infra/net/ssrf.ts:91-92`
  - body: `serializeSsrFPolicy()` preserves array order even though hostname policy enforcement normalizes hostnames as sets. Semantically identical policies such as `allowedHostnames: ["a", "b"]` and `["b", "a"]` get different cache keys, so equivalent agents can miss cache entries unnecessarily.
  - confidence: 0.76
