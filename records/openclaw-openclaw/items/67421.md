Full review comments:

- **[P1] Replace the shallow SSRF policy merge:** `src/infra/net/ssrf.ts:80-83`
  - body: The object spread preserves conflicting alias state and replaces list fields instead of normalizing the effective policy. For example, a global `allowPrivateNetwork: true` survives an agent `dangerouslyAllowPrivateNetwork: false`, so a per-agent tightening still allows private-network fetches; normalize aliases and merge list/set fields explicitly before enforcement.
  - confidence: 0.87
- **[P2] Only warn when an agent actually widens SSRF access:** `src/security/audit.ts:1154-1160`
  - body: This branch emits the named-agent private-network warning whenever the effective policy allows private-network access, even if that access came entirely from the global baseline. An agent with only an unrelated override would be reported as widening SSRF access while an agent with no override would not, making the security audit misleading.
  - confidence: 0.82
- **[P3] Canonicalize SSRF policy arrays before cache serialization:** `src/infra/net/ssrf.ts:91-92`
  - body: The cache key serializes `allowedHostnames` and `hostnameAllowlist` in input order, but enforcement treats those lists as normalized sets. Equivalent policies with entries in different order will miss each other's cache entries, so the serializer should sort or reuse the same comparison normalization.
  - confidence: 0.77
