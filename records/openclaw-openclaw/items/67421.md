Full review comments:

- **[P1] Preserve global SSRF policy when merging agent overrides:** `src/infra/net/ssrf.ts:80-83`
  - body: The shallow spread lets an agent policy replace or clear global fields such as `allowedHostnames` and `hostnameAllowlist`. That can silently widen the configured baseline or break globally allowed hosts, so this needs explicit security-preserving merge semantics and tests for arrays, aliases, and undefined values.
  - confidence: 0.9
- **[P1] Authorize agent-selected SSRF widening before applying it:** `src/agents/openclaw-tools.ts:142-144`
  - body: The PR selects the agent SSRF policy from the session-derived agent id and passes it into `web_fetch`. A caller that can choose an agent session could select a more permissive private-network policy unless widening is tied to requester authority or made restrictive-only.
  - confidence: 0.84
- **[P2] Report only actual agent-specific SSRF widening:** `src/security/audit.ts:1154-1160`
  - body: The audit warning checks the effective merged policy, so an agent with an unrelated override is reported as widening access when the global baseline already allowed private-network fetches. Compare agent policy against the global baseline so the security audit does not produce misleading per-agent warnings.
  - confidence: 0.78
- **[P3] Normalize policy arrays before cache serialization:** `src/infra/net/ssrf.ts:91-92`
  - body: The cache key preserves array order even though hostname matching normalizes sets, so semantically identical policies with different array ordering create separate cache entries. Sorting and normalizing before serialization keeps cache identity aligned with enforcement.
  - confidence: 0.73
