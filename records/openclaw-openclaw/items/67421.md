  - confidence: 0.9

  - body: The new serializer keys the fetch cache on only three booleans and two hostname lists. Current main's policy also includes `allowIpv6UniqueLocalRange` and `allowedOrigins`; after rebasing, omitting them would let different effective policies share cached `web_fetch` results across stricter and broader SSRF settings. Reuse a current semantic policy normalizer or include all fields with deterministic ordering.
  - confidence: 0.82
- **[P2] Report only real agent-side SSRF widening:** `src/security/audit.ts:1154-1160`
  - body: This audit branch warns that an agent “widens” access whenever the effective merged policy allows private networking. If the global default already allows private networking, an agent with only a narrow hostname override will still get this warning even though that agent did not widen the boundary; compare the agent policy against the global baseline before choosing the warning title/severity.
  - confidence: 0.78
