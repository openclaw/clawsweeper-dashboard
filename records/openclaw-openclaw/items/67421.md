  - body: This derives the web_fetch SSRF policy from the agent id resolved from `agentSessionKey`. Because gateway/tool construction passes session-scoped keys through this path, a caller that can select `agent:<id>:...` can inherit a named agent configured with `dangerouslyAllowPrivateNetwork` and fetch private targets. Thread an authorization-checked agent id or trusted requester scope here before applying widening SSRF policies.
  - confidence: 0.86

  - body: The shallow spread lets an agent policy replace global SSRF fields verbatim. That can clear or replace hostname restrictions and leave legacy/new private-network alias flags in conflicting states, so the effective boundary can be broader or different than the operator's baseline. Merge only defined values with explicit union/intersection semantics and add tests for empty and clearing cases.
  - confidence: 0.88
