Full review comments:

- **[P1] Preserve baseline policy when merging agent overrides:** `src/infra/net/ssrf.ts:80-83`
  - body: The shallow spread lets an agent policy replace or clear global SSRF fields, including `allowedHostnames`, `hostnameAllowlist`, and legacy/private-network aliases. That violates the PR's baseline/exceptions model and can silently widen or break the configured policy, so merge booleans/arrays with explicit security-preserving semantics and cover undefined/empty-list cases.
  - confidence: 0.91
- **[P1] Authorize agent-selected SSRF widening before use:** `src/agents/openclaw-tools.ts:139-142`
  - body: This makes the session-derived agent config select the `web_fetch` SSRF policy. Because callers can supply or inherit `agent:` session keys through gateway/tool paths, a private-network override must be gated by requester authority or made restrictive-only before it can safely affect fetch policy.
  - confidence: 0.84
- **[P2] Report only real agent SSRF widening:** `src/security/audit.ts:1158`
  - body: The audit warning fires when the effective policy allows private-network access, even if the global baseline already allows it and the agent override only changes a narrow field. That mislabels inherited risk as an agent widening; compare global versus agent policy and warn only when the agent actually broadens access.
  - confidence: 0.82
- **[P3] Normalize policy arrays before using them in cache keys:** `src/infra/net/ssrf.ts:86-94`
  - body: `serializeSsrFPolicy` preserves array order even though SSRF hostname matching treats normalized hostnames/patterns as sets. Equivalent policies with different array order will miss each other's cache entries; normalize and sort the arrays before serializing.
  - confidence: 0.78
