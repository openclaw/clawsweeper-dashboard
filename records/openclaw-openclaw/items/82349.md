## What This Changes

## Best Possible Solution

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Reuse the skill env sanitizer for setup env:** `extensions/skills-setup/index.ts:424-428`
  - body: This only filters a small reserved-key set before passing both `skills.entries.<skillKey>.env` and request env into `bash`. Current skill env handling blocks dangerous override keys such as `NODE_OPTIONS`, `BASH_ENV`, proxy/TLS, and Docker env even when declared, and it only allows sensitive secrets through the declared env contract. Please reuse or expose that sanitizer for setup hooks so the new command-execution path cannot receive dangerous host overrides or undeclared secret keys.
  - confidence: 0.91
- **[P2] Account for the startup activation baseline:** `extensions/skills-setup/openclaw.plugin.json:3-4`
  - body: The new manifest declares `activation.onStartup`, but the bundled metadata test currently asserts the complete startup plugin id list and this PR does not add `skills-setup` there. Either remove startup activation if this RPC should be explicitly enabled, or update the startup policy/baseline intentionally so the bundled metadata check does not fail.
  - confidence: 0.82
