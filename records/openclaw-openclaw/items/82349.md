## Summary

Close as ClawHub: this is an optional community code plugin, the same plugin has already been published on ClawHub, and current OpenClaw policy says optional plugins should live outside core unless there is a clear product/security ownership reason. The core setup-hook design remains separately tracked and should not be resolved by bundling this plugin branch.

## What This Changes

Adds a bundled `skills-setup` plugin that registers an admin-only `skills.setup` gateway RPC, resolves installed skill setup metadata, runs setup scripts with env overlays, and adds focused tests plus labeler metadata.

## Best Possible Solution

Keep `@alexminza/skills-setup` on ClawHub as the community plugin path, and handle any official install/update setup-hook contract through the canonical core issue and SDK helper follow-up.

## Reproduction Assessment

Yes for the PR defects: source inspection shows the env sanitizer bypass and startup baseline mismatch on the PR head. For the setup-hook request itself, reproduction is not applicable because it is a new feature surface.

## Solution Assessment

No. Bundling this community plugin into core is not the best path now that the same plugin is published on ClawHub; any official behavior should be decided through the separate core setup-hook and SDK-surface work.

  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The PR adds an admin command-execution RPC and its branch bypasses current env override hardening, so it should not be merged into core as-is.

Concerns:

- **[high] Setup env bypasses dangerous override filtering:** `extensions/skills-setup/index.ts:424`
  - body: `normalizeEnvMap` forwards any string key except a small reserved set into a bash setup process. Current main blocks dangerous host and override keys such as `BASH_ENV`, `NODE_OPTIONS`, proxy/TLS, Docker, and loader/toolchain env names; setup hooks should use the same policy or a reviewed setup-specific sanitizer.
  - confidence: 0.9
- **[medium] New setup execution surface needs trust-contract review:** `extensions/skills-setup/index.ts:553`
  - body: The PR introduces an admin RPC that runs skill-supplied scripts with caller/config env, so consent, logging/redaction, timeout ownership, startup activation, and ClawHub trust behavior need explicit maintainer agreement before any official core surface ships.
  - confidence: 0.82
