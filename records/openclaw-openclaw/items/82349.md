## Summary

Keep this PR open but blocked: it is an active draft feature PR for a new skill setup execution surface, and source review found merge-blocking correctness/security issues plus missing external real behavior proof.

## What This Changes

Adds a bundled `skills-setup` plugin that registers an admin-only `skills.setup` gateway RPC, parses setup metadata, resolves installed skill directories, runs setup scripts with env overlays, and adds focused tests, labeler metadata, and a lockfile importer.

## Best Possible Solution

Pick one canonical setup-hook contract, reuse or expose the existing skills/frontmatter/env safety helpers, update startup/docs policy deliberately, and require redacted real gateway setup proof before merge.

## Reproduction Assessment

Yes for the PR defects: source inspection shows the env sanitizer bypass and startup baseline mismatch on the PR head against current main. For the linked setup-hook request itself, reproduction is not applicable because it is a new feature surface.

## Solution Assessment

No. The plugin-only RPC is not yet the narrowest maintainable solution because it duplicates internal skill parsing, bypasses the established env-sanitizer contract, and needs a maintainer decision against the core install/update implementation path.

## Security Review
