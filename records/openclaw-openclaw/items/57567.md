## Summary

Keep open. Current main has partial config recovery hardening, but the reported Feishu legacy schema path and general `doctor --fix` rollback from numbered backups are still not fully implemented; the narrower Feishu fix is covered by open PR #65384, not merged on main.

## What This Changes

The issue asks OpenClaw to survive v2026.3.24-to-v2026.3.28 upgrades with legacy Feishu config by producing valid JSON, migrating renamed config keys, and giving `doctor --fix` a usable rollback path.

## Best Possible Solution

Review and land #65384 or an equivalent Feishu-owned doctor migration, then split or implement the broader unparseable-config backup restore behavior with explicit recovery-policy review.

## Reproduction Assessment

Yes. The Feishu failure has a high-confidence source reproduction: legacy `channels.feishu.accounts.main.botName` reaches the active strict account schema and is rejected, while arbitrary malformed JSON is only recovered when it matches last-known-good or valid-suffix cases.

## Solution Assessment

No. Closing this based on current startup recovery would be premature; the maintainable fix is a Feishu-owned compatibility migration plus a separately scoped core doctor restore policy rather than relaxing strict schema validation broadly.
