## Summary

Keep open. Current main still rejects legacy Feishu account configs that use `botName`, has no Feishu doctor compatibility normalizer, and the remaining invalid-JSON recovery story now depends on last-known-good promotion rather than selecting the numbered `.bak` backup path described in the report.

## What This Changes

The issue asks OpenClaw to make v2026.3.24-to-v2026.3.28 upgrades with legacy Feishu config migrate safely, keep valid JSON, and let `doctor --fix` recover without manual config editing.

## Best Possible Solution

Land or replace the Feishu-owned doctor migration, then separately scope a conservative recovery policy for restoring a validated recent backup when `openclaw.json` is unparseable; auth-profile and memory-loss claims should be split only with concrete logs.

## Reproduction Assessment

Yes, source-backed. A legacy `channels.feishu.accounts.main.botName` config still fails strict current-main validation, and current doctor recovery does not implement the requested numbered `.bak` selection path for malformed JSON.

## Solution Assessment

No, closing this issue now would be premature. The narrow Feishu fix belongs in the plugin doctor contract, while the numbered-backup rollback and auth/memory claims need maintainer scoping rather than being treated as already solved.
