## What This Changes

The PR adds an `authMethod === "none"` local-backend branch to `shouldSkipLocalBackendSelfPairing`, adds helper/auth-state tests, and inserts a blank line in `CHANGELOG.md`.

## Best Possible Solution

Wire a narrowly scoped no-auth local backend allowance into the missing-device decision path, preserve remote and browser-origin rejection, add full gateway handshake regression coverage, and add a single-line changelog fix entry.

No. The helper-only change is not the best fix because the reported rejection is decided in `evaluateMissingDeviceIdentity`; the safer repair is to carry a scoped local-backend no-auth allowance into that gate and test the actual connect path.
