## Summary

Keep this PR open: the updated patch is closer to the intended scope, but it still does not let a device-less `auth.mode: "none"` backend connection pass the missing-device decision, and the changelog touch is only a blank line.

## What This Changes

The PR adds a scoped `authMethod === "none"` branch to `shouldSkipLocalBackendSelfPairing`, adds auth-state/helper tests, and touches `CHANGELOG.md`.

## Best Possible Solution

Add a scoped missing-device allowance for browser-origin-free local `gateway-client` backend coordination under `gateway.auth.mode: "none"`, keep remote/browser negatives, update the changelog/docs if the policy is approved, and cover the actual handshake path.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: `auth.mode: "none"` resolves as method `none` with `sharedAuthOk: false`, then a device-less backend connection reaches `reject-device-required` and closes with 1008.

## Solution Assessment

No. The helper-only change is not the best fix because the rejection happens in the missing-device decision; the safer fix is to thread a narrowly scoped no-auth backend allow signal into that policy and test the full path.
