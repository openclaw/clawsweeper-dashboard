## Summary

Keep open. Current main still keys Feishu inbound duplicate suppression to `message_id`, so same-logical-message text deliveries with different ids can pass the early, debounce, and final dedupe gates; related #75078 covers the inverse media-id reuse case and does not replace this issue.

## What This Changes

Request a Feishu inbound dedupe fallback for private-chat messages that repeat as the same logical content with different `message_id` values.

## Best Possible Solution

Add a Feishu-owned logical duplicate fingerprint for text retries that combines stable event identity fields while preserving intentional repeated text and media/file uniqueness, with focused Feishu tests and a changelog entry.

## Reproduction Assessment

Yes. A focused unit reproduction can feed two Feishu private-chat text events for the same account, chat/thread, sender, `create_time`, and content with different `message_id` values; source inspection shows both ids pass the current gates independently.

## Solution Assessment

Unclear in exact implementation. The maintainable direction is Feishu-local and should coordinate with #75078, using a structured logical identity rather than a plain content hash alone.
