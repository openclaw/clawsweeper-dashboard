## Summary

Keep open: current main still has only partial cleanup for auto-created stale route state, while the reported Windows provider-switch path needs a maintainer-defined contract for config updates, auth routing, and session pins. The related open PR covers a narrower reset/fallback cleanup and is not a full replacement.

## What This Changes

Windows provider/model switching can leave stale effective model configuration and session cache state, causing visible provider/model status to disagree with the provider route used for requests.

## Best Possible Solution

Define one provider-switch/reset contract that updates effective config, clears auto-owned stale session route state, preserves intentional pins, and gives doctor/status/docs a matching recovery path.

## Reproduction Assessment

Yes at source level: current main still has model/auth update paths that preserve existing config and session pins without a full provider-switch cleanup transaction. No live Windows provider-switch repro was run in this read-only review.

## Solution Assessment

Unclear: a provider-switch transaction is plausible, but maintainers need to define which stale route state is auto-owned and which session/config pins are intentional before a safe fix can land.
