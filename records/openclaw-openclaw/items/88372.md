The issue requests a Windows-safe provider/model switch path that updates stale config references, clears or refreshes session routing cache, and reports cross-provider fallback state clearly.

## Best Possible Solution

Define one maintainer-approved provider-switch/reset contract that preserves explicit user pins, clears auto-owned stale route state, diagnoses remaining stale refs, and aligns status, docs, onboarding, doctor, and session reset behavior.

## Solution Assessment

Unclear: a provider-switch transaction or reset path is plausible, but the best fix must distinguish stale automatic state from intentional session pins and avoid changing compatibility semantics by accident. The safer end state is a defined contract plus focused tests before implementation.
