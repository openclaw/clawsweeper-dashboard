## Summary

Keep open. Current main has already fixed the exec-approvals atomic-write half, but the transcript half remains: several gateway/session paths still read complete transcript files before dedupe, history limiting, sequence calculation, usage extraction, or compaction.

## What This Changes

The issue requests atomic writes for exec approvals and bounded or streaming transcript reads to avoid approvals-file corruption and transcript memory pressure.

## Best Possible Solution

Keep the shipped exec-approvals replacement logic, then introduce shared bounded JSONL transcript scan/tail helpers and migrate the remaining transcript consumers to use them with focused regression coverage for large files, malformed lines, branch-aware histories, dedupe, history windows, usage extraction, and compaction.
