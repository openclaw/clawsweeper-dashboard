## Summary

Keep open. Current main has fixed the exec-approvals atomic-write half and bounded several transcript hot paths, but multiple current gateway/session helpers still materialize complete transcript files before idempotency, latest-assistant, equivalent-message, compaction-fork, and usage scans.

## What This Changes

The issue asks OpenClaw to write exec-approvals atomically and replace unbounded transcript whole-file reads with streaming or bounded scans.

## Best Possible Solution

Keep the atomic exec-approvals implementation, then add shared streaming or indexed transcript scan helpers and migrate the remaining idempotency, latest-assistant, equivalent-message, usage, and compaction-fork paths with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction path: create a sufficiently large transcript and call the listed helpers, which still whole-file read or accumulate the transcript before scanning.

## Solution Assessment

Yes for the remaining work. Streaming or indexed transcript scans are the narrow maintainable fix, while the existing exec-approvals temp-file rename implementation should remain in place.
