## What This Changes

Request implicit Discord accountId-to-agentId routing when a Discord account name matches a configured agent, instead of falling back to the default agent.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/39485 or a replacement only after it requires an existing configured agent, reports implicit matches distinctly, updates docs/tests, and gets product/security approval with redacted Discord proof.

## Solution Assessment

Unclear. Same-name Discord fallback is plausible, but it changes routing and security semantics and should land only with maintainer approval, configured-agent guarding, distinct diagnostics, regression coverage, docs, and real Discord proof.
