## Summary

Keep open: current main and the latest release still resolve unmatched Discord account traffic through explicit binding tiers and then the default agent. The target-repository implementation PR was closed unmerged, and the remaining fix needs maintainer approval because it changes permission-sensitive Discord routing behavior.

## What This Changes

The issue asks for Discord inbound routing to select an agent whose id matches the Discord account name before falling back to the default agent.

## Best Possible Solution

Choose and document one maintainer-approved Discord account routing contract, guard it to configured runtime-eligible accounts and agents, add regression coverage, and require redacted real Discord proof before landing.

## Reproduction Assessment

Yes. At source level, Discord passes accountId into resolveAgentRoute, and current main falls through to the default agent after explicit binding tiers when no binding matches.

## Solution Assessment

Unclear. Same-name fallback may solve the report, but it changes the documented routing contract; explicit account-level binding or a guarded implicit fallback both need maintainer product and security approval.
