## What This Changes

## Best Possible Solution

## Reproduction Assessment

Yes. Source inspection shows Discord passes accountId into resolveAgentRoute, and current main still falls through to the configured default agent when no binding tier matches; I did not run live Discord in this read-only review.

## Solution Assessment

Unclear. A Discord-only accountId-to-agent fallback is a plausible narrow repair, but the best solution depends on whether maintainers want implicit name matching or explicit bindings as the supported routing contract.
