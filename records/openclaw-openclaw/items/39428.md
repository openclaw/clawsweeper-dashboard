## Summary

Keep open: current main and the latest release still use explicit binding tiers and then the default agent for unmatched Discord accounts, while the linked same-name fallback PR remains open, conflicting, and security/product-sensitive.

## What This Changes

Request implicit Discord routing so a non-default Discord account whose name matches an agent ID routes to that agent without explicit `bindings` entries.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/39485 or a replacement only after same-name Discord routing is approved, guarded to configured agents, documented, tested, and proven in a redacted Discord setup.

## Reproduction Assessment

Yes, at source level: Discord passes the account ID into `resolveAgentRoute`, and current main falls through to the default agent when no explicit binding matches. I did not run live Discord proof in this read-only review.

## Solution Assessment

Unclear. Same-name Discord fallback may be useful, but current docs/tests define explicit bindings followed by default routing, so the policy needs maintainer approval plus configured-agent guards and real Discord proof.
