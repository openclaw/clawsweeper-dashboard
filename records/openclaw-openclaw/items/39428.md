## Summary

Keep open: current main and the latest release still route unmatched Discord accounts through explicit binding tiers and then the default agent, while the same-name and account-level routing implementation PRs are closed unmerged. The remaining work is a permission-sensitive Discord routing contract decision, not conservative cleanup.

## What This Changes

The issue requests Discord inbound routing to choose an agent whose ID matches the Discord account name instead of falling through to the default agent when no explicit binding matches.

## Best Possible Solution

Choose one maintainer-approved Discord account routing contract, document it, guard it to configured runtime-eligible agents/accounts, add regression coverage, and prove it in a redacted real Discord setup.

## Reproduction Assessment

Yes, at source level: Discord passes accountId into resolveAgentRoute, and current main falls through to the default agent after explicit binding tiers. I did not run live Discord proof in this read-only review.

## Solution Assessment

Unclear. The requested same-name fallback may solve the report, but it changes the documented routing contract; a safer final path may be an explicit account-level agent binding or a guarded implicit fallback with product and security approval.
