## Summary

Keep open: current main still falls through from Discord account routing to the default agent after explicit binding tiers, and https://github.com/openclaw/openclaw/pull/39485 directly targets this issue with closing syntax but has not merged.

## What This Changes

Request implicit Discord routing from a channel accountId to a same-named agent before using the default agent fallback.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/39485 or an equivalent maintainer-approved Discord routing change with guards, tests, diagnostics, docs, and real behavior proof.

## Reproduction Assessment

Yes, source-level: a Discord route with accountId "agentA", agents.list containing "agentA", and no bindings still reaches the default fallback on current main. The unresolved part is whether same-name fallback is the intended routing contract.

## Solution Assessment

Unclear until maintainers accept the implicit routing policy. If accepted, the narrowest maintainable fix is a Discord-owned fallback after explicit binding tiers and before default fallback, with a default-account guard and regression coverage.
