## Summary

Keep open. Current main still has a module-local memory capability registry that Memory Wiki bridge import/status read directly, and the bundled plugin runtime mirror still creates the boundary described in the report. The issue is also paired with open draft PR #75064 in the provided GitHub context, so it should remain open while that implementation is reviewed and validated.

## What This Changes

Fix Memory Wiki bridge/status so public artifacts registered by memory-core remain visible across canonical and plugin-runtime-deps mirrored runtime module instances.

## Best Possible Solution

Land or replace #75064 with a narrow shared or host-owned memory capability registry that all mirrored runtime module instances observe, while preserving the existing plugin SDK API and adding regression coverage for Memory Wiki bridge import/status artifact counts.
