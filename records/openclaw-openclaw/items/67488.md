## Summary

Keep open. Current main still lacks structural Unicode smart-quote normalization for malformed tool-call argument JSON, and the linked closing PR remains open and unmerged, so this issue should not be closed before that implementation or an equivalent fix lands.

## What This Changes

The issue requests that isolated cron jobs with announce delivery stop being marked as failed when an edit-tool argument payload contains CJK text, Markdown, and Unicode quote characters even though the agent run and delivery completed.

## Best Possible Solution

Keep this issue open and review or replace PR #67551 as the active implementation path. The desired end state is a narrow embedded-runner repair that normalizes only structural Unicode smart quotes before parsing, keeps literal smart quotes inside values intact, updates balanced JSON scanning consistently, and adds a focused regression test for malformed edit-tool arguments used by isolated cron delivery.

## Reproduction Assessment

Yes. A representative smart-quote tool-call argument object fails the same raw JSON.parse plus ASCII-balanced-fragment parse sequence used on current main, and cron result state code explains how delivery can be recorded as delivered while the run remains error.

## Solution Assessment

Yes, the narrowest maintainable fix belongs in the embedded runner's tool-call argument repair path rather than cron status accounting. Cron is already preserving delivery state separately; the malformed argument JSON should be repaired before tool execution/result handling reports an error.
