## Summary

Keep open. Current main still lacks structural Unicode smart-quote normalization in the malformed tool-call argument repair path, and PR #67551 remains the open, unmerged implementation path for the reported cron failure.

## What This Changes

The issue reports isolated announce cron jobs being marked `error` after successful delivery when edit-tool arguments with CJK/Markdown/smart-quote content fail JSON parsing during post-processing.

## Best Possible Solution

Review, land, or replace PR #67551 with a focused embedded-runner parser repair that normalizes only structural Unicode quotes, preserves literal smart quotes inside values, and adds regression coverage for cron/edit-tool arguments.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path for representative structural U+201C/U+201D tool-call argument JSON: current main parses raw JSON, then a balanced fragment, without Unicode quote normalization.

## Solution Assessment

Yes. The narrowest maintainable fix is in the embedded runner's malformed tool-call argument repair path; changing cron status accounting would only mask the failed argument parse while leaving malformed tool calls broken.
