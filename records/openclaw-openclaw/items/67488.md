## Summary

Keep open. Current main and the latest release still route malformed tool-call arguments through plain JSON parsing plus an ASCII-quote balanced-prefix fallback; the direct smart-quote fix PR is closed unmerged, and the related merged PRs only broaden where the existing repair wrapper runs.

## What This Changes

The issue reports cron runs being marked as errors after successful announce delivery when streamed edit-tool arguments contain Unicode smart quotes or mixed CJK/Markdown content.

## Best Possible Solution

Add delimiter-aware smart-quote repair in the shared balanced JSON/tool-call argument parsing path, preserving literal smart quotes inside values and covering the existing repair behavior with focused regressions.
