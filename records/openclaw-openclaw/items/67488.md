## Summary

Keep open. Current main still parses malformed tool-call argument fragments through raw JSON parsing and an ASCII-quote balanced JSON fallback, while open PR #67551 remains the active closing implementation path.

## What This Changes

Cron isolated agent-turn jobs can be marked as errors after successful announce delivery when edit-tool arguments with CJK/smart-quote content fail JSON parsing during result handling.

## Best Possible Solution

Review, land, or replace PR #67551 with a narrow embedded-runner repair that normalizes only structural Unicode smart quotes before tool-call argument parsing, preserves literal smart quotes inside values, and adds focused regression coverage.

## Reproduction Assessment

Yes. A representative U+201C/U+201D structural-quote argument object fails the same raw JSON.parse path used on current main, and the balanced JSON fallback has no smart-quote string-boundary handling.

## Solution Assessment

Yes. The best fix belongs in the embedded runner's malformed tool-call argument repair path rather than cron status accounting, because cron is already preserving delivery state separately.
