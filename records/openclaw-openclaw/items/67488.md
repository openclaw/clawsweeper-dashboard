## Summary

Keep open. Current main still has no structural Unicode smart-quote normalization in the malformed tool-call argument repair path, and the linked closing PR remains open and unmerged, so the issue should not be closed before that implementation or an equivalent fix lands.

## What This Changes

Cron runs should not be marked as errors after successful delivery when edit-tool argument JSON contains smart quotes, CJK text, or Markdown content.

## Best Possible Solution

Keep this issue open and review, land, or replace PR #67551. The desired end state is a narrow embedded-runner repair that normalizes only structural Unicode smart quotes before tool-call argument parsing, keeps literal smart quotes inside values intact, updates balanced JSON scanning consistently, and adds focused regression coverage for edit-tool arguments used by isolated cron delivery.

## Reproduction Assessment

Yes. A representative U+201C/U+201D structural-quote argument object fails the same raw JSON.parse plus balanced-prefix parse sequence used on current main, and the cron result code explains how delivery can be recorded as delivered while the run remains error.

## Solution Assessment

Yes. The narrowest maintainable fix belongs in the embedded runner's tool-call argument repair path, not in cron status accounting; cron is already preserving delivery state separately.
