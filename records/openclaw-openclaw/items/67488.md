Keep open. Current main and v2026.5.12 still parse malformed tool-call arguments with plain JSON.parse plus an unnormalized balanced-prefix fallback, while the direct smart-quote parser PR is closed unmerged and the merged adjacent PRs only broaden repair enablement by transport.

Fix cron/agent tool-call argument parsing so successful cron runs are not recorded as errors when edit-tool JSON arguments contain CJK/smart-quote style characters.

## Best Possible Solution

Add delimiter-aware smart-quote repair in the shared balanced JSON/tool-call argument parsing path, preserving literal smart quotes inside values and locking the existing prefix/trailing-junk repair behavior with focused regressions.
