## Summary

Current main and v2026.6.8 already provide the central pre-execute argument repair path through `AnyAgentTool.prepareArguments`: plugin tools can define it, `registerTool` accepts those tools, the agent loop runs it before validation/execution, and plugin wrappers preserve plugin scope.

## What This Changes

The issue requests a plugin SDK `registerTool` option that lets third-party plugin tools normalize raw model arguments before execution.

## Best Possible Solution

Keep `AnyAgentTool.prepareArguments` as the canonical pre-validation argument repair hook, with any `normalizeArgs` alias or docs polish handled as a separate maintainer API ergonomics decision.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence that the requested pre-execute repair behavior already exists through `prepareArguments`.

## Solution Assessment

Yes for the central behavior: the shipped `prepareArguments` seam is narrower than adding a second `registerTool` option because it already runs on raw arguments before validation and is preserved for plugin tools.
