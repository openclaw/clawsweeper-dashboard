## Summary

Close as implemented: current main and the latest release already give plugin-registered tools a pre-execute argument-preparation hook through `AgentTool.prepareArguments`, including plugin callback scoping and tests for plugin tools.

## What This Changes

The issue requests a plugin-scoped way for registered tools to normalize or repair raw arguments before `execute()` runs.

## Best Possible Solution

Keep the shipped `prepareArguments` contract as the plugin tool argument-repair path and consider a narrower docs or `defineToolPlugin` ergonomics follow-up only if maintainers want better discoverability.

## Reproduction Assessment

Not applicable; this is a feature/API request, not a runtime bug report. Source inspection and release comparison are the high-confidence proof path, and they show the pre-execute hook already exists.

## Solution Assessment

Yes. The existing `prepareArguments` hook is the narrowest maintainable solution for direct or factory plugin tools; adding `normalizeArgs` as a second options-level hook would duplicate a shipped tool contract unless maintainers want a separate ergonomic alias.
