## Summary

Current main and the latest release already provide the central plugin-owned raw-argument repair path through `AgentTool.prepareArguments`, so adding a parallel `registerTool` `normalizeArgs` option is not necessary for this issue.

## What This Changes

The issue requests an optional plugin SDK hook that lets registered tools normalize or repair incoming arguments before `execute()` runs.

## Best Possible Solution

Keep `AgentTool.prepareArguments` as the canonical plugin-owned raw-argument hook and avoid adding a duplicate `normalizeArgs` option; plugin docs can be improved separately if maintainers want more discoverability.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection confirms plugin tools can already provide `prepareArguments`, which runs on raw arguments before validation and execution.

## Solution Assessment

Yes. The existing `AgentTool.prepareArguments` seam is a narrower shipped solution than adding a second `registerTool` option for the same behavior.
