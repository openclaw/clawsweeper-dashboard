## Label Justifications

## Summary

Current main and v2026.6.8 already provide the central plugin-owned raw argument repair path through `AgentTool.prepareArguments`, which runs before schema validation and execution and is preserved for plugin tools; adding `normalizeArgs` would duplicate that SDK surface.

## What This Changes

Request an optional `registerTool` `normalizeArgs` callback so plugin tools can repair malformed model arguments before `execute()`.

## Best Possible Solution

Keep `AgentTool.prepareArguments` as the canonical plugin tool argument normalizer; if maintainers want better discoverability, track that as a narrow docs or API-naming follow-up.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a plugin SDK feature request. Source and release inspection give high confidence that the requested pre-execute argument repair path already exists through `prepareArguments`.

## Solution Assessment

No for the proposed new `normalizeArgs` option; it would add a second SDK surface for behavior already covered by `prepareArguments`. The shipped seam is the narrower maintainable solution because it runs on raw tool-call arguments before validation and is preserved for plugin tools.
