## Summary

Keep open: `before_tool_call` already lets plugins rewrite object-shaped tool params before `execute()`, but current main still hides non-object raw args from hooks and `registerTool` has no `normalizeArgs` option, so the raw argument-normalization API request remains a product/API decision.

## What This Changes

Request adding an optional `registerTool` `normalizeArgs` callback so plugin-owned tools can repair raw arguments before `execute()`.

## Best Possible Solution

Run a maintainer plugin-SDK design pass for a narrow raw-argument normalization seam, or document `before_tool_call` as the supported path and limit any new API to the non-object/raw cases it cannot cover.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source inspection gives high confidence that object-param repair is already possible through `before_tool_call`, while raw non-object args are not exposed to that hook.

## Solution Assessment

Unclear. A `normalizeArgs` option is plausible, but maintainers should first decide the API boundary and ordering against existing hooks, trusted policies, approvals, and provider-level argument repair.
