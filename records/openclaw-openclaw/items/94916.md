## Label Justifications

## Summary

Current main and the latest release already satisfy the central request through the existing `prepareArguments` hook on `AnyAgentTool`, which plugin tools can provide through `registerTool`; it runs before validation and execution and is preserved inside plugin runtime scope.

## What This Changes

The issue requests an optional plugin SDK hook that lets registered plugin tools normalize malformed arguments before execution.

## Best Possible Solution

Keep `AgentTool.prepareArguments` as the canonical shipped raw-argument normalizer and avoid adding a second `registerTool` option for the same behavior.

## Reproduction Assessment

Not applicable; this is an SDK feature request rather than a bug with a failing reproduction. Source and release inspection give high confidence that the requested pre-execute argument repair path already exists through `prepareArguments`.

## Solution Assessment

Yes for the central user problem: the shipped `prepareArguments` seam is the narrower maintainable solution because it already runs on raw tool-call arguments before validation and execution. A separate `normalizeArgs` option would duplicate the canonical tool contract.
