## Summary

Current main and the latest release already provide the requested pre-execute raw argument repair path through `AgentTool.prepareArguments`, so adding a separate `normalizeArgs` option would duplicate public plugin SDK surface rather than unlock missing behavior.

## What This Changes

Request an optional `registerTool` argument-normalization hook so plugin-owned tools can repair malformed model arguments before `execute()`.

## Best Possible Solution

Keep `AgentTool.prepareArguments` as the canonical shipped raw-argument normalizer and treat any remaining need as documentation polish rather than a second SDK option.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a plugin SDK API request. Source and release inspection show the requested pre-execute argument repair path already exists through `prepareArguments`.

## Solution Assessment

Yes for cleanup close. The existing shipped `prepareArguments` seam is narrower than adding a duplicate `normalizeArgs` `registerTool` option because it already runs on raw arguments before validation.
