## Summary

Keep open. Current main still exposes reminders as ordinary cron jobs and lacks a Reminders tab, reminder-safe cron classification/display metadata, and service/API filtering to separate reminder one-shot jobs from system cron jobs.

## What This Changes

Add a dedicated Reminders tab in Control UI for one-shot reminder cron jobs, with cleaned reminder text, fire time, channel/recipient, delete action, and Cron tab filtering.

## Best Possible Solution

Define one additive cron classification/display metadata contract, have reminder-producing paths set it, expose typed filters, hide reminders from the generic Cron view, and build the Reminders tab with focused protocol, service, UI, and docs coverage.

## Reproduction Assessment

Not applicable as a runtime bug because this is a feature request. Source inspection gives high confidence that the requested Reminders tab, reminder classifier/display metadata, and reminder/non-reminder list filter are still absent on current main.

## Solution Assessment

Yes at the direction level: typed cron metadata and filters are the maintainable path instead of name or payload heuristics. The exact core UI/API boundary needs maintainer product and protocol review before implementation.
