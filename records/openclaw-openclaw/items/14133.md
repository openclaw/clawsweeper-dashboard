## Summary

Keep open. Current main and latest release v2026.6.5 still route reminders through the generic Cron surface: there is no Reminders tab/route, no reminder-safe display metadata, and no cron list filter that can separate reminder one-shot jobs from other cron jobs; the related cron tags/categories issue is open but only covers part of the needed contract.

## What This Changes

The issue requests a dedicated Mission Control Reminders tab that separates reminder one-shot cron jobs from generic recurring cron jobs and displays reminder-safe card content.

## Best Possible Solution

Coordinate this with the open cron tags/categories work, then add an additive cron classification/display metadata contract, reminder-producing paths that set it, typed list filtering, Cron-tab hiding/filtering, and a dedicated Reminders tab with focused protocol/service/UI coverage.

## Reproduction Assessment

Yes, at source level. Current main still documents reminders as cron jobs and the UI/protocol/store surfaces lack a Reminders tab, reminder classifier, or reminder-safe display text field.

## Solution Assessment

Unclear as an immediate implementation path. A Reminders tab is plausible, but the best fix should first align reminder-specific metadata with the broader cron tags/categories contract rather than parsing names or payload text.
