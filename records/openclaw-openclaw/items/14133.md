## Summary

Keep open. Current main still has the generic Cron Control UI, but it does not have a Reminders tab, a typed cron job classifier/display metadata contract, or backend/UI filtering that can separate reminder one-shot jobs from ordinary cron jobs. No open PR appears to reference this issue with closing syntax, and the prior maintainer/Codex discussion also kept it open for the same missing core UI/protocol work.

## What This Changes

The issue requests a dedicated Mission Control Reminders tab that separates reminder one-shot cron jobs from regular cron jobs and displays cleaned reminder text, fire time, recipient/channel, and delete actions.

## Best Possible Solution

Add a backward-compatible cron metadata/display contract, update reminder-producing flows to set it, expose typed list filtering for reminders and non-reminders, hide or filter reminders from the Cron tab, and add a Reminders Control UI tab that renders cleaned reminder text, fire time, channel/recipient, and delete actions with protocol/service/UI tests and docs.

## Reproduction Assessment

Not applicable as a runtime bug report; this is a feature request. We do have high-confidence source verification of the missing surface: current main has no Reminders tab/route, no reminder classifier in the cron schema, and no list filter for reminder jobs.

## Solution Assessment

Yes, the requested direction is the right maintainable shape if implemented through additive cron metadata and typed filtering. A brittle fallback based on job names or raw payload parsing would not be the best solution because it could expose internal agent instructions and misclassify jobs.
