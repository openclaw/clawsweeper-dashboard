## Summary

Keep this open. Current main still only has the generic Cron Control UI surface: no Reminders tab/route, no reminder-specific render path, and no cron job classifier or list filter that can separate reminder jobs from other scheduled jobs.

## What This Changes

The issue requests a dedicated Control UI Reminders tab that separates one-shot reminder cron jobs from recurring cron jobs and shows user-safe reminder cards.

## Best Possible Solution

Add an additive cron classification/display metadata contract, have reminder-producing paths set it, expose typed reminder/non-reminder filtering, and build a Reminders tab that uses that contract with focused protocol, service, UI, and docs coverage.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main still has only the generic Cron UI/protocol path, not a separate Reminders surface.

## Solution Assessment

Yes at the direction level: a cron metadata/filter contract plus a dedicated UI is the maintainable shape. The unsafe alternative is a UI-only heuristic over cron names or raw payload text.
