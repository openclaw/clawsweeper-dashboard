## Summary

Keep open: current main still models and renders reminders as generic cron jobs, with no dedicated Reminders tab, no reminder classifier/display metadata, and no cron list filter that can separate reminders from other scheduled work.

## What This Changes

Add a dedicated Mission Control Reminders tab with card UI, clean reminder text, fire time, recipient/channel, delete actions, and cron filtering based on reminder metadata.

## Best Possible Solution

Add an additive cron classification/display metadata contract, have reminder-producing paths set it, expose typed list filtering, then build a Reminders tab that consumes that contract and hides or filters reminder jobs from the generic Cron tab.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main gives high confidence that the requested tab and classifier/filter contract are absent.

## Solution Assessment

Yes on the general direction, but the exact metadata/filter design needs maintainer alignment. An additive typed cron contract is the safer path than inferring reminders from names or raw payload text.
