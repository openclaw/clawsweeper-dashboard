## Summary

Keep open. Current main still has only the generic Cron Jobs Mission Control surface, with no Reminders tab, no reminder-safe display metadata, and no protocol/service filter that can separate one-shot reminders from other cron jobs; prior issue comments also kept this open for the same missing product/API work.

## What This Changes

The issue requests a dedicated Mission Control Reminders tab that separates one-shot reminder cron jobs from ordinary cron jobs and shows cleaned reminder text, fire time, channel/recipient, and delete actions.

## Best Possible Solution

Define an additive cron classification/display metadata contract, have reminder-producing paths set it, expose typed reminder filters, hide or filter reminders from the generic Cron view, and build a Reminders tab with cleaned text, fire time, channel/recipient, delete actions, and focused protocol/service/UI coverage.

## Reproduction Assessment

Not applicable as a runtime bug because this is a feature request. Source inspection on current main gives high confidence that the Reminders tab, reminder classifier/display metadata, and reminder/non-reminder list filter are still absent.

## Solution Assessment

Yes at the direction level: typed cron metadata and filters are the maintainable path instead of name or payload heuristics. The exact UI/API boundary needs maintainer product review before implementation.
