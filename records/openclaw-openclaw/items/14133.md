## Summary

Keep open. Current main still exposes reminders only through the Cron UI and `cron.list` contract: there is no Reminders route/render path, no cron classifier/display metadata, and no typed reminder filter, while prior issue discussion already kept this open for the same missing product/API surface.

## What This Changes

Request a dedicated Reminders tab in Mission Control plus cron metadata/filtering so one-shot reminder jobs are hidden from generic Cron and shown as safe reminder cards.

## Best Possible Solution

Define an additive cron classification/display metadata contract, have reminder-producing paths set it, expose typed reminder filters, and build the Reminders tab on that contract.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main still has only the generic Cron UI/protocol path, not a separate Reminders surface.

## Solution Assessment

Yes at the direction level: a typed cron metadata/filter contract plus a dedicated UI is the maintainable shape. The exact core-vs-plugin UI boundary needs maintainer product/API judgment before implementation.
