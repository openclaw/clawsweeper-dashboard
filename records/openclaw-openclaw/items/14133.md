## Summary

Keep open. Current main still has only the generic Cron UI and cron.list contract: there is no Reminders tab, no reminder-specific render path, and no stored or protocol-level reminder classifier that could safely hide reminders from Cron or render cleaned reminder text.

## What This Changes

Request a dedicated Mission Control Reminders tab that separates one-shot reminder cron jobs from recurring/system cron jobs and displays user-friendly reminder details.

## Best Possible Solution

Add an additive cron classification/display metadata contract, have reminder-producing paths set it, support typed reminder and non-reminder list filters, hide or filter reminders from Cron, and add a Reminders tab with cleaned text, fire time, target/channel, delete action, tests, and docs.

## Reproduction Assessment

Not applicable as a bug, but the feature gap is source-reproducible: current main has cron navigation, rendering, protocol, and list filters, with no Reminders tab or reminder classifier.

## Solution Assessment

Yes at the direction level, but it needs maintainer product/protocol alignment. The maintainable path is an additive metadata/filter contract rather than parsing cron names or payload text in the UI.
