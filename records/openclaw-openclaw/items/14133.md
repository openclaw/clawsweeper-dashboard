## Summary

Keep open. Current main and the latest shipped release still expose reminders only through the generic Cron jobs surface: there is no Reminders tab, no reminder-safe display metadata, and no reminder/non-reminder filter in the cron storage, protocol, or service path.

## What This Changes

Request a dedicated Mission Control Reminders tab that separates one-shot reminder cron jobs from recurring/system cron jobs and shows cleaned reminder text, fire time, target/channel, and delete controls.

## Best Possible Solution

Choose an additive cron classification/display metadata contract, coordinate it with the generic tags/categories direction, have reminder-producing paths set that metadata, and render a dedicated Reminders UI with typed filtering instead of payload parsing.

## Reproduction Assessment

Yes as a source-level feature gap, not as a runtime bug: current main has Cron navigation/rendering plus cron schema/storage/list filtering, but no Reminders tab or reminder classifier/display metadata.

## Solution Assessment

Yes at the direction level, but the exact contract needs maintainer product/protocol alignment. The maintainable path is additive metadata and typed filtering coordinated with cron tags/categories, not parsing cron names or payload text in the UI.
