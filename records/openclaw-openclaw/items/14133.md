## Summary

Keep open: current main and the latest release still lack a dedicated Reminders route, a stable cron reminder-purpose/display contract, and reminder-specific list filtering; the open cron tags issue overlaps classification but does not replace the requested reminder UX.

## What This Changes

Request a dedicated Mission Control Reminders tab for one-shot reminder cron jobs, with safe display text, fire time, recipient/channel details, delete actions, and filtering out of the generic Cron tab.

## Best Possible Solution

Define one additive typed cron purpose and display-metadata contract that supports reminders and broader categorization, then implement reminder creation, typed filtering, Cron exclusion, a Reminders view, tests, and docs against that contract.

## Reproduction Assessment

Yes. Current-main source shows no Reminders route, no reminder-purpose/display fields in the cron job or list contracts, and no list filter that can exclude reminders from the Cron tab.

## Solution Assessment

Unclear as proposed. The user outcome is sound, but freeform tags or payload extraction alone are not the best permanent fix; a typed purpose/display contract coordinated with the cron categorization issue is safer.
