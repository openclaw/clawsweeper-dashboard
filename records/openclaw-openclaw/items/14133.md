## Summary

Keep open: current main still lacks a Reminders tab, reminder-specific cron classification/display metadata, and typed filtering to separate one-shot reminders from ordinary cron jobs. The related https://github.com/openclaw/openclaw/issues/53107 is still open for broader cron tags/categories and helps frame the protocol work, but it does not supersede this reminder-specific UI request.

## What This Changes

Add a dedicated Reminders tab in Mission Control/Control UI for one-shot reminder cron jobs, with metadata-based filtering, clean display text, target/fire-time cards, and delete controls.

## Best Possible Solution

Add an additive cron classification/display metadata contract aligned with https://github.com/openclaw/openclaw/issues/53107, have reminder-producing paths set it, expose typed list filtering, and build a Reminders tab that consumes it while hiding reminders from the generic Cron tab.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main lacks the requested tab, cron classifier/display metadata, and typed reminder filtering.

## Solution Assessment

Yes at the product direction level, but the exact metadata/filter design needs maintainer alignment. A typed additive cron contract is safer than name or payload matching for identifying reminders.
