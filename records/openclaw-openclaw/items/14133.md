## Summary

Keep open. Current main still has the generic Cron Control UI and cron.list contract, but no Reminders tab, no reminder-specific render path, and no stored or protocol-level reminder classifier that can safely separate one-shot reminders from ordinary cron jobs.

## What This Changes

Add a dedicated Mission Control Reminders tab that separates one-shot reminder cron jobs from recurring/system cron jobs and displays user-friendly reminder metadata.

## Best Possible Solution

Add an additive cron classification/display metadata contract, set it from reminder-producing paths, support typed reminder and non-reminder list filters, and add a Reminders tab with cleaned text, fire time, target/channel, delete action, tests, and docs.

## Reproduction Assessment

Not applicable as a bug, but the feature gap is source-reproducible: current main has cron navigation, rendering, protocol, storage, and list filters with no Reminders tab or reminder classifier.

## Solution Assessment

Yes at the direction level, but it needs maintainer product/protocol alignment. The maintainable path is additive metadata and typed filtering rather than parsing cron names or payload text in the UI.
