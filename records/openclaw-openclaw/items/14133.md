## Summary

Keep open: current main still has only generic Cron Control UI and cron list/protocol surfaces, while the related cron tags/categories issue overlaps on classification but does not cover the dedicated Reminders tab and display UX.

## What This Changes

Request a dedicated Control UI Reminders tab for one-shot reminder cron jobs, backed by cron reminder metadata and filtering so reminders are not shown only as ordinary Cron jobs.

## Best Possible Solution

Define one additive cron purpose/display metadata contract, have reminder creation set it, add list filters, choose Cron-tab visibility, and build the Reminders tab with focused protocol/service/UI tests and docs.

## Reproduction Assessment

Yes. Source inspection reproduces the missing capability: current main has no Reminders route, no reminder-purpose/display fields in cron job/list contracts, and no filter that can exclude reminders from the Cron tab.

## Solution Assessment

Unclear as proposed. A dedicated Reminders UX is reasonable, but tags or payload extraction alone are not the best permanent fix until maintainers choose the shared cron purpose/display contract and Cron-tab policy.
