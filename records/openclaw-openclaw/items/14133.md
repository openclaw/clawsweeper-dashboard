## Summary

Current main and v2026.6.5 still support reminders only as generic cron jobs: the Control UI has no Reminders route, the cron contract has no reminder classification or safe display fields, and list filtering cannot select reminders. The open generic tags issue overlaps only with classification and does not supersede the requested reminder-specific presentation and workflow.

## What This Changes

Add a dedicated Mission Control Reminders tab that separates reminder one-shot cron jobs and shows safe reminder text, fire time, recipient/channel, and deletion controls.

## Best Possible Solution

Define one additive typed cron metadata contract shared with generic categorization, including an owner-set reminder kind and explicit safe display fields, then add server-side reminder filtering and dedicated Control UI presentation without parsing raw agent prompts.

## Reproduction Assessment

Yes. Current-main source provides a high-confidence reproduction of the feature gap: reminders are documented cron jobs, while navigation, persisted job types, list parameters, and UI requests have no reminder-specific contract or surface.

## Solution Assessment

Yes for the user outcome, but freeform tags or payload-text extraction alone are not the best implementation; a typed classification and safe display contract should be established first and coordinated with the generic cron-categories work.
