## Summary

Keep open. Current main still exposes reminders as ordinary cron jobs: there is no Reminders tab/route, no reminder-specific render path, and no stored or protocol-level classifier/display metadata to separate one-shot reminders from recurring/system cron jobs.

## What This Changes

Add a Reminders tab in Mission Control that separates one-shot reminder cron jobs from the generic Cron tab and displays cleaned reminder text, fire time, target/channel, and delete controls.

## Best Possible Solution

Add an additive cron classification/display metadata contract, set it from reminder-producing paths, support typed reminder and non-reminder list filters, and add a Reminders tab with cleaned text, fire time, target/channel, delete action, tests, and docs.

## Reproduction Assessment

Not applicable as a runtime bug, but yes as a source-level feature gap: current main has cron navigation, rendering, protocol, storage, and list filters with no Reminders tab or reminder classifier.

## Solution Assessment

Yes at the direction level, but it needs maintainer product/protocol alignment. The maintainable path is additive metadata and typed filtering rather than parsing cron names or payload text in the UI.
