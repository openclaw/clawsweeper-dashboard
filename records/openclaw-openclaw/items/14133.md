## Summary

Keep open: current main still has only the generic Cron Control UI surface, with no Reminders tab, no reminder-specific render path, and no cron classifier/display metadata that can separate reminder one-shot jobs from ordinary cron jobs.

## What This Changes

The issue requests a dedicated Mission Control Reminders tab that separates reminder one-shot cron jobs from ordinary cron jobs and displays cleaned reminder metadata.

## Best Possible Solution

Add an additive cron classification/display metadata contract, have reminder-producing paths set it, expose typed reminder and non-reminder filtering, and build a Reminders tab with clean text, fire time, recipient/channel, and delete actions.

## Reproduction Assessment

Not applicable as a runtime bug reproduction because this is a feature request. Source inspection gives high confidence that the Reminders tab, reminder metadata, and reminder-specific filtering are absent on current main.

## Solution Assessment

Yes at the product direction level, but the exact metadata/filtering contract needs maintainer alignment. A typed additive cron contract shared with https://github.com/openclaw/openclaw/issues/53107 is safer than matching job names or parsing raw payload text.
