## Summary

Keep open: current main and the latest shipped release still expose reminders as ordinary cron jobs, with no dedicated Reminders Control UI tab, no reminder-specific render path, and no cron classification/display metadata contract to filter reminders out of the Cron tab.

## What This Changes

Add a dedicated Reminders tab in Control UI that separates one-shot reminder cron jobs from generic cron jobs and displays clean reminder details.

## Best Possible Solution

Add an additive cron classification/display metadata contract, have reminder-producing paths set it, expose typed reminder and non-reminder filtering, and build a Reminders tab with clean text, fire time, recipient/channel, and delete actions.

## Reproduction Assessment

Not applicable as a runtime bug reproduction because this is a feature request. Source, docs, and release inspection give high confidence that the Reminders tab, reminder metadata, and reminder-specific filtering are absent.

## Solution Assessment

Yes at the product direction level, but the exact metadata/filtering contract needs maintainer alignment. A typed additive cron contract shared with https://github.com/openclaw/openclaw/issues/53107 is safer than matching job names or parsing raw payload text.
