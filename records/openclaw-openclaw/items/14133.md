## Summary

Keep open: current main and the latest stable release still expose only the generic Cron Jobs UI/protocol/list filters, with no Reminders tab or reminder classifier/display metadata. The related cron tags/categories issue is useful but still open and only covers part of the underlying API need.

## What This Changes

Request adds a dedicated Reminders tab in Control UI/Mission Control by classifying one-shot reminder cron jobs, filtering them out of Cron, and rendering sanitized reminder cards.

## Best Possible Solution

Add an additive cron classification/display metadata contract, have reminder producers set it, expose typed list filters, and build Reminders cards while keeping Cron focused on non-reminder jobs.

## Reproduction Assessment

Yes for a source-level feature gap: current navigation, render, protocol, and list-filter paths show only Cron and no Reminders classifier or tab. No runtime bug reproduction is needed because this is a feature request.

## Solution Assessment

Unclear until maintainers choose the cron classification/display metadata shape. The requested direction looks maintainable if it uses an additive typed contract instead of name or payload string matching.
