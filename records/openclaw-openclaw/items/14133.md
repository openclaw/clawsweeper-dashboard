## Summary

Keep open. Current main still has only the generic Cron Jobs Mission Control surface: there is no Reminders tab/render path, no cron job classifier/display metadata, and no typed list filter that can separate one-shot reminders from ordinary cron jobs; prior issue comments also kept this open for the same missing product/API surface.

## What This Changes

The issue requests a dedicated Mission Control Reminders tab with display-safe reminder cards and cron filtering based on reminder metadata.

## Best Possible Solution

Define an additive cron classification/display metadata contract, have reminder-producing paths set it, expose typed reminder filters, and build the Reminders tab on that contract.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main still has only the generic Cron UI/protocol path, not a separate Reminders surface.

## Solution Assessment

Yes at the direction level: typed cron metadata and filters are the maintainable solution. The exact core-vs-plugin UI boundary needs maintainer product/API judgment before implementation.
