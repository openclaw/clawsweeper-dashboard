## Summary

Current main still lacks the dedicated Reminders route, a stable reminder-purpose/display contract, and reminder-specific list filtering. The open generic cron-tags proposal overlaps classification but does not replace the requested reminder UX or safe presentation fields, so this issue remains necessary and needs a maintainer product/API decision.

## What This Changes

Add a dedicated Mission Control Reminders tab that uses structured reminder metadata to show safe reminder text, fire time, recipient/channel, and deletion while separating reminders from generic cron jobs.

## Best Possible Solution

Define one additive typed cron purpose and user-facing display metadata contract that can support reminders and broader categorization, then implement reminder creation, typed filtering, a dedicated Reminders view, and Cron exclusion without parsing names or agent payload instructions.

## Reproduction Assessment

Yes. Current-main source directly shows no Reminders route, no reminder-purpose job field, and no reminder-specific list filter; this is a high-confidence source reproduction of the missing feature.

## Solution Assessment

The requested user outcome is sound, but freeform tags or payload extraction alone are not the best solution. A typed purpose/display contract coordinated with the broader cron categorization proposal is the narrowest maintainable design.
