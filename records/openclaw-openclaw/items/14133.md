## Summary

Keep open. Current main and the latest shipped release still only expose reminders through the generic Cron jobs surface: there is no Reminders tab, no reminder-safe display metadata, and no reminder/non-reminder filter; the related cron tags/categories issue at https://github.com/openclaw/openclaw/issues/53107 overlaps the classifier contract but does not cover the dedicated Reminders UI.

## What This Changes

Request a dedicated Control UI/Mission Control Reminders tab that separates one-shot reminder cron jobs from ordinary cron jobs and displays cleaned reminder details.

## Best Possible Solution

Choose an additive cron classification/display metadata contract coordinated with tags/categories, have reminder-producing paths set it, add typed reminder/non-reminder list filters, and render a dedicated Reminders UI without parsing raw payloads.

## Reproduction Assessment

Yes, as a source-level feature gap rather than a runtime bug: current main has Cron navigation/rendering plus cron schema/storage/list filtering, but no Reminders tab or reminder classifier/display metadata.

## Solution Assessment

Unclear until maintainers choose the product and protocol shape. The maintainable direction is additive metadata plus typed filtering coordinated with cron tags/categories, not UI parsing of names or payload text.
