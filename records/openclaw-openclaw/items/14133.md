## Summary

Keep open: current main still has only the generic Cron jobs surface in the Control UI, without a dedicated Reminders tab, reminder classification/display metadata, or typed filtering that could separate one-shot reminders from other cron jobs. The related #53107 is still the open protocol/product alignment point for cron tags/categories, but it does not supersede the reminder-specific UI request.

## What This Changes

Request a dedicated Mission Control/Control UI Reminders tab with card-style reminder display, cleaned reminder text, channel/recipient details, delete actions, and cron filtering via reminder metadata.

## Best Possible Solution

Add an additive cron classification and display metadata contract aligned with #53107, have reminder-producing paths set it, expose typed list filtering, then build a Reminders tab that consumes that contract and filters reminder jobs out of the generic Cron tab.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main gives high confidence that the requested tab and classifier/filter contract are absent.

## Solution Assessment

Yes at the product direction level, but the exact metadata/filter design needs maintainer alignment. An additive typed cron contract is safer than name or payload matching for identifying reminders.
