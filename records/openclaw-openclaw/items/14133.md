## Summary

Keep open: current main and v2026.6.8 still have only generic Cron Control UI surfaces, with no Reminders tab, no reminder-purpose/display metadata, and no stable filter that can separate reminders from other scheduled jobs.

## What This Changes

The issue requests a dedicated Mission Control Reminders tab for one-shot reminder cron jobs, with clean reminder text, fire time, recipient/channel display, delete actions, and filtering out of the generic Cron tab.

## Best Possible Solution

Define one additive typed cron purpose/display metadata contract, have reminder creation set it, add list filters, decide Cron-tab visibility, then build the Reminders view with focused protocol/service/UI tests and docs.

## Reproduction Assessment

Yes. Source inspection reproduces the missing capability: current main has no Reminders route, no reminder-purpose/display fields in cron job/list contracts, and no filter that can exclude reminders from the Cron tab.

## Solution Assessment

Unclear as proposed. A dedicated Reminders UX is reasonable, but tags or payload extraction alone are not the best permanent fix until maintainers choose the shared cron purpose/display contract and Cron-tab policy.
