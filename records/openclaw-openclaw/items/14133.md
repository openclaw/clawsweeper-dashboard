## Summary

This should stay open because current main still has only the generic Cron Control UI: no Reminders tab, no reminder-specific cron classifier/filter, and no safe reminder display metadata. The related generic tags issue is useful context, but it does not replace the requested Reminders UI and cleaned reminder-card behavior.

## What This Changes

Add a dedicated Mission Control Reminders tab that separates one-shot reminder cron jobs from recurring cron jobs and displays reminder-friendly text, fire time, channel/recipient, and delete controls.

## Best Possible Solution

Define one additive cron classification/display-metadata contract, align it with the open generic tags work, then add reminder-producing support, list filtering, Cron tab hiding, and a Reminders tab with focused protocol, service, UI, and docs coverage.

## Reproduction Assessment

Not applicable as a runtime bug, but yes as a source-level feature gap: current main has no Reminders tab, no cron reminder classifier, and no reminder filter path. The requested behavior is absent rather than a broken documented behavior with a failing command path.

## Solution Assessment

Yes at the direction level, but the exact API shape needs maintainer product/protocol review. The maintainable path is additive metadata and typed filtering aligned with the generic tags issue, not ad hoc payload/name parsing.
