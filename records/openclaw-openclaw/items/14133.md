## Summary

Keep open: current main still exposes reminders only as generic cron jobs, with no dedicated Reminders tab, reminder classifier/display metadata, or typed cron list filter for separating reminder jobs from other scheduled work.

## What This Changes

Request a dedicated Reminders tab in the Control UI that separates one-shot reminder cron jobs, shows cleaned reminder details, and hides or filters reminders from the generic Cron tab.

## Best Possible Solution

Add an additive cron classification and display metadata contract, have reminder-producing paths set it, expose typed list filtering, then build a Reminders tab that consumes that contract and filters reminder jobs out of the generic Cron tab.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main gives high confidence that the requested tab and classifier/filter contract are absent.

## Solution Assessment

Yes on the general direction, but the exact metadata/filter design needs maintainer alignment. An additive typed cron contract is safer than inferring reminders from job names or raw payload text.
