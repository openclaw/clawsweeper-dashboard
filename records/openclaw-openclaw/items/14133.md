## Summary

Keep open: current main still has only the generic Cron Control UI surface, with no Reminders tab/route, no reminder-specific render path, and no cron metadata/filtering contract that can separate reminder jobs from other one-shot or recurring jobs. The related cron tags/categories issue is relevant and still open, but it does not supersede the reminder-specific UI request.

## What This Changes

Request a dedicated Control UI Reminders tab that separates one-shot reminder cron jobs from generic cron jobs and displays cleaned reminder details.

## Best Possible Solution

Add an additive cron classification/display metadata contract, have reminder-producing paths set it, expose typed list filtering, hide reminder jobs from the generic Cron tab, and build a Reminders tab with clean text, fire time, recipient/channel, and delete actions.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection on current main gives high confidence that the Reminders tab, reminder classifier/display metadata, and reminder-specific filtering are absent.

## Solution Assessment

Yes at the product direction level, but the exact metadata/filter contract needs maintainer alignment. A typed additive cron contract shared with https://github.com/openclaw/openclaw/issues/53107 is safer than matching job names or parsing raw payload text.
