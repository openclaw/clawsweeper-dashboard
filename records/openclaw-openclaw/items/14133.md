## Summary

Keep open. Current main still has a generic Cron Jobs Control UI surface, but it lacks the requested Reminders tab, reminder/non-reminder cron classifier, list filtering, and cleaned reminder display contract; this is a product/protocol feature rather than a narrow bug fix.

## What This Changes

Request a dedicated Reminders tab in Control UI that separates one-shot reminder cron jobs from recurring cron jobs and displays cleaned reminder details.

## Best Possible Solution

Add an additive cron classification and display-metadata contract aligned with the open cron-tags work, have reminder-producing paths set it, add typed list filtering, hide or filter reminders from the Cron tab, and ship a Reminders tab with protocol/service/UI tests and docs.

## Reproduction Assessment

Yes for the feature gap by source inspection: current main has only the generic Cron Jobs route and lacks reminder metadata/filtering in the cron schema and service. This is not a runtime bug reproduction.

## Solution Assessment

Yes at the direction level: a dedicated reminder view needs a real metadata/filtering contract rather than payload/name heuristics. The exact tags versus reminder-type shape should be decided with the related cron-tags issue before implementation.
