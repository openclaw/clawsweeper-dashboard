## Summary

Keep open. Current main still has only the generic Cron Control UI and cron protocol/service list path; it lacks a Reminders tab, reminder display/classification metadata, and typed reminder/non-reminder filtering.

## What This Changes

The issue requests a dedicated Mission Control Reminders tab with cleaned reminder cards and filtering separate from generic Cron jobs.

## Best Possible Solution

Add a backward-compatible cron metadata/display contract, have reminder-producing paths set it, expose typed reminder filtering, hide reminders from generic Cron where appropriate, and render a dedicated Reminders tab with cleaned text, fire time, recipient/channel, and delete actions.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high-confidence verification that current main lacks the requested Reminders tab, cron classifier/display metadata, and backend filter.

## Solution Assessment

Yes, the requested direction is the maintainable path if implemented through typed additive metadata and filtering. A workaround based on job names or raw payload parsing would be weaker and could leak internal instructions.
