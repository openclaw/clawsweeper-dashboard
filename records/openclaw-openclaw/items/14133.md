## Summary

Keep open: current main and v2026.6.5 still expose only the generic Cron UI/protocol/listing path, while the related cron tags/categories request remains open and covers only part of the Reminders-tab work.

## What This Changes

Request a dedicated Mission Control/Control UI Reminders tab that separates one-shot reminder cron jobs from ordinary cron jobs and displays user-facing reminder details.

## Best Possible Solution

Add an additive cron classification/display metadata contract, have reminder-producing paths set it, expose typed list filters, and build Reminders cards while keeping the Cron tab focused on non-reminder jobs.

## Reproduction Assessment

Yes, for a source-level feature gap: current navigation, render, protocol, service, docs, and filter tests show only generic Cron surfaces and no Reminders classifier or tab. No runtime bug reproduction is needed because this is a feature request.

## Solution Assessment

Unclear until maintainers choose the cron classification/display metadata shape. The requested direction is likely maintainable if it uses an additive typed contract instead of name or payload string matching.
