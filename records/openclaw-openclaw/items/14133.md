## Summary

Keep open. Current main and the latest shipped release still expose only the generic Cron jobs surface, with no Reminders tab, no reminder-safe display metadata, and no reminder/non-reminder filter in the cron protocol or service.

## What This Changes

Request a dedicated Mission Control Reminders tab that separates one-shot reminder cron jobs from recurring/system cron jobs and displays user-safe reminder text, fire time, target, and delete actions.

## Best Possible Solution

Add an additive cron classification/display metadata contract, set it from reminder-producing paths, support typed reminder and non-reminder list filters, and add a Reminders UI that renders cleaned text, fire time, target/channel, and delete actions with protocol/service/UI coverage.

## Reproduction Assessment

Yes as a source-level feature gap, not as a runtime bug: current main has Cron navigation, rendering, protocol schema, storage/list filtering, and no Reminders tab or reminder classifier.

## Solution Assessment

Yes at the direction level, but it needs maintainer product/protocol alignment. The maintainable path is additive metadata and typed filtering coordinated with cron tags/categories, not parsing cron names or payload text in the UI.
