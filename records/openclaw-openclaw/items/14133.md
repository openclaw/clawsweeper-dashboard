## Summary

Keep open: current main still lacks the dedicated Reminders Control UI tab, reminder-specific render path, and cron metadata/filtering/display contract needed to separate reminders from generic cron jobs. The related cron tags/categories issue is relevant and still open, but it does not fully cover the reminder-specific UI request.

## What This Changes

Request a dedicated Reminders tab in Mission Control that displays one-shot reminder cron jobs separately from generic cron jobs with clean reminder text, fire time, target/channel, and delete action.

## Best Possible Solution

Add an additive cron classification/display metadata contract, have reminder-producing paths set it, expose typed reminder/non-reminder list filtering, hide reminders from the generic Cron tab as appropriate, and build a Reminders tab with clean text, fire time, recipient/channel, and delete actions.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection on current main gives high confidence that the Reminders tab, reminder classifier/display metadata, and reminder-specific filtering are absent.

## Solution Assessment

Yes at the product direction level, but the exact metadata/filter contract needs maintainer alignment. A typed additive cron contract shared with https://github.com/openclaw/openclaw/issues/53107 is safer than matching job names or parsing raw payload text.
