## Summary

Keep this open. Current main still exposes reminders as ordinary cron jobs: there is no Reminders tab/route, no reminder-specific render path, and no public cron job classifier or list filter that can separate reminder jobs from other scheduled work.

## What This Changes

Add a dedicated Mission Control Reminders tab that shows one-shot reminder cron jobs as clean reminder cards and hides or filters them out of the generic Cron tab.

## Best Possible Solution

Add an explicit cron classification/display metadata contract, have reminder-producing paths set it, expose typed reminder/non-reminder filtering, and build a Reminders tab that uses that contract instead of parsing names or raw payloads.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main still has only the generic Cron UI/protocol path, not a separate Reminders surface.

## Solution Assessment

Yes at the direction level: an additive cron metadata/filter contract plus a dedicated UI is the maintainable shape. The unsafe alternative is a UI-only heuristic over cron names or payload text.
