## Summary

Keep open. Current main still only has the generic Cron Control UI and cron list contract; it lacks a Reminders route, reminder classifier/display metadata, reminder/non-reminder filtering, and a dedicated reminder card view.

## What This Changes

Add a dedicated Mission Control Reminders tab for one-shot reminder cron jobs, backed by cron reminder metadata/filtering and cleaned reminder display.

## Best Possible Solution

Add an additive cron classification/display metadata contract, have reminder-producing paths set it, expose typed reminder/non-reminder filtering, hide reminders from the generic Cron tab as appropriate, and add a Reminders UI with cleaned text, fire time, channel/recipient, delete actions, tests, and docs.

## Reproduction Assessment

Not applicable as a bug reproduction. The feature gap is source-verified: current navigation, rendering, protocol schemas, service filters, and UI filters have no reminder-specific route or classifier path.

## Solution Assessment

Yes at the direction level, but the exact metadata shape needs maintainer product/protocol review. The open cron tags/categories issue can cover part of the classifier contract, but it does not replace the Reminders tab and cleaned reminder display work.
