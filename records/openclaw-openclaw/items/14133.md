## Summary

Keep open: current main still has only the generic Cron Control UI and cron list/protocol surfaces; the related cron tags/categories issue overlaps on classification but does not cover the dedicated Reminders tab, cleaned reminder display, or Cron-tab visibility policy.

## What This Changes

The issue requests a dedicated Reminders tab in Mission Control for one-shot reminder cron jobs, with cleaned reminder cards and filtering out of the generic Cron tab.

## Best Possible Solution

Define one additive cron purpose/display metadata contract, have reminder-producing paths set it, add typed reminder/non-reminder list filters, then build the Reminders tab and Cron-tab visibility policy against that shared model.

## Reproduction Assessment

Yes. Source inspection reproduces the missing capability: current main has no Reminders route/render branch, no reminder-purpose/display fields in cron contracts, and no filter that can exclude reminders from the Cron tab.

## Solution Assessment

Unclear as proposed. A dedicated Reminders UX is reasonable, but tags or payload extraction alone are not the best permanent fix until maintainers choose the shared cron purpose/display contract and Cron-tab visibility policy.
