## Summary

Keep open: current main still has only the generic Cron Control UI surface, with no Reminders tab, reminder-specific cron metadata, or typed filtering to separate one-shot reminders from ordinary cron jobs. The related https://github.com/openclaw/openclaw/issues/53107 is still open and should inform the shared cron classification contract, but it does not supersede this reminder-specific UI request.

## What This Changes

Request a dedicated Reminders tab in Control UI/Mission Control, backed by cron metadata/filtering so one-shot reminder jobs are shown separately from ordinary cron jobs.

## Best Possible Solution

Add an additive cron classification/display metadata contract aligned with the open cron tags work, have reminder-producing jobs set it, expose typed list filtering, hide reminders from the generic Cron tab, and build a Reminders tab with cleaned text, fire time, recipient/channel, and delete actions.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main lacks the requested Reminders tab, cron classifier/display metadata, and reminder-specific filtering.

## Solution Assessment

Yes at the product direction level, but the exact metadata/filter design needs maintainer alignment. A typed additive cron contract is safer than name or payload matching for identifying reminders.
