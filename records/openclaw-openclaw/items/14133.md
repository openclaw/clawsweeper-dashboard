## Summary

Keep open: current main and the latest release still have generic cron management only, without a Reminders route, stable reminder-purpose/display metadata, or reminder-specific list filtering. The related cron tags/categories issue overlaps the needed classifier foundation, but it does not replace the requested Reminders UX or safe human-readable reminder presentation.

## What This Changes

The issue requests a dedicated Mission Control Reminders tab that separates one-shot reminder cron jobs from generic cron jobs and displays safe human-readable reminder details.

## Best Possible Solution

Define one additive typed cron purpose and display-metadata contract that supports reminders and broader categorization, then implement reminder creation, typed filtering, Cron exclusion policy, a Reminders view, tests, and docs against that contract.

## Reproduction Assessment

Yes. Current-main source shows no Reminders route, no reminder-purpose/display fields in the cron job or list contracts, and no list filter that can exclude reminders from the Cron tab.

## Solution Assessment

Unclear as proposed. The user outcome is sound, but freeform tags or payload extraction alone are not the best permanent fix; a typed purpose/display contract coordinated with the cron categorization issue is safer.
