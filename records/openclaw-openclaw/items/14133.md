## Summary

Keep open: current main and the latest release still expose only the generic Cron Mission Control surface, with no Reminders route, reminder classifier/display metadata, or list filter; the related cron tags issue is useful but only partially covers this request.

## What This Changes

The issue requests a dedicated Mission Control Reminders tab that separates reminder-created one-shot cron jobs from normal cron jobs and shows user-friendly reminder details.

## Best Possible Solution

Define one additive cron classification and display-metadata contract, then implement storage, RPC filtering, reminder-producing metadata, Cron-tab exclusion, a Reminders tab, focused protocol/service/UI tests, and docs updates against that shared model.

## Reproduction Assessment

Yes. Source inspection of current main shows no Reminders tab/route, no reminder-specific render branch, and no cron classifier/list filter that could separate reminders from other cron jobs.

## Solution Assessment

Unclear until maintainers choose the shared classifier contract. A Reminders tab is a good end state, but the safe implementation should avoid name or payload string matching and should use an additive cron classification/display-metadata model.
