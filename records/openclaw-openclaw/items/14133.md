## Summary

Keep open. Current main still lacks a Reminders tab, reminder-safe cron metadata/display fields, and service/API filtering that can separate reminder one-shot jobs from ordinary cron jobs; prior review comments and the related open cron tags issue point to the same unresolved product/protocol work.

## What This Changes

Add a dedicated Mission Control Reminders tab and distinguish reminder one-shot cron jobs from generic cron jobs.

## Best Possible Solution

Define an additive cron classification/display metadata contract, have reminder-producing paths set it, expose typed reminder filters, hide or filter reminders from the generic Cron view, and build a Reminders tab with cleaned text, fire time, channel/recipient, delete actions, and focused protocol/service/UI coverage.

## Solution Assessment
