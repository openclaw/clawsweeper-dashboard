## Summary

Keep open: current main and v2026.6.8 still expose reminders only as ordinary cron jobs, with no Reminders tab, no reminder-purpose/display metadata, and no list filter that can safely separate reminders from other scheduled jobs.

## What This Changes

Request a dedicated Mission Control Reminders tab backed by cron metadata/filtering so one-shot reminder jobs are hidden or separated from the generic Cron tab and displayed with safe human-readable text.

## Best Possible Solution

Define one additive typed cron purpose/display metadata contract that covers reminders and broader categorization, then implement reminder creation metadata, list filtering, Cron-tab policy, a Reminders view, tests, and docs against that contract.

## Reproduction Assessment

Yes. Source inspection reproduces the missing capability: current main has no Reminders route, no reminder-purpose/display fields in cron job/list contracts, and no filter that can exclude reminders from the Cron tab.

## Solution Assessment

Unclear as proposed. A dedicated Reminders UX is reasonable, but tags or payload extraction alone are not the best permanent fix until maintainers choose the shared cron purpose/display contract and Cron-tab policy.
