## Summary

Keep open: current main still lacks the requested Reminders tab, reminder-specific cron classification/display metadata, and typed filtering needed to hide reminder jobs from the generic Cron tab. The related cron tags/categories issue is relevant but remains open and does not supersede this reminder-specific Control UI request.

## What This Changes

The issue requests a dedicated Mission Control Reminders tab with card-style reminder display, clean reminder text, recipient/channel details, delete actions, and cron filtering/classification so reminder jobs do not appear as ordinary cron jobs.

## Best Possible Solution

Add an additive cron classification/display metadata contract, have reminder-producing paths set it, expose typed list filtering, hide reminders from the generic Cron tab, and build a Reminders tab with clean text, fire time, recipient/channel, and delete actions.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main gives high confidence that the Reminders tab, reminder classifier/display metadata, and reminder-specific filtering are absent.

## Solution Assessment

Yes at the product direction level, but the exact metadata/filter contract needs maintainer alignment. A typed additive cron contract shared with the open tags/categories work is safer than matching names or parsing raw payload text.
