## Summary

Keep open. Current main still exposes reminders only through the generic Cron Jobs surface: there is no Reminders tab or route, no reminder classifier/display metadata in cron storage or protocol, and no backend/UI filter to separate reminder jobs from ordinary cron jobs.

## What This Changes

Request a dedicated Mission Control Reminders tab that separates one-shot reminder cron jobs from recurring/system cron jobs and shows cleaned reminder text, fire time, target/channel, and delete actions.

## Best Possible Solution

Add an additive cron classification/display metadata contract, have reminder-producing paths set it, support typed reminder and non-reminder filters, and add a Reminders tab with cleaned text, fire time, target/channel, delete action, tests, and docs.

## Reproduction Assessment

Yes, as a source-level feature gap: current main has cron navigation, rendering, protocol, storage, and list filters with no Reminders tab or reminder classifier. This is not a runtime bug reproduction; it is a missing feature in the current cron/Mission Control surface.

## Solution Assessment

Yes at the direction level, but it needs maintainer product/protocol alignment. The maintainable path is additive metadata and typed filtering rather than parsing cron names or payload text in the UI.
