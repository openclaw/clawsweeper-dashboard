## Summary

Keep open: current main and the latest release still expose generic cron management only, with no Reminders route, no stable reminder-purpose/display metadata, and no reminder-specific list filter. The related cron tags/categories issue overlaps the classifier foundation but does not replace the requested Reminders UX or safe human-readable reminder presentation.

## What This Changes

The issue requests a dedicated Mission Control Reminders tab for one-shot reminder cron jobs, including reminder metadata, filtering out of Cron, human-readable text, fire time, recipient/channel, and delete controls.

## Best Possible Solution

Define one additive typed cron purpose and display-metadata contract that supports reminders and broader categorization, then implement reminder creation, typed filtering, Cron exclusion policy, a Reminders view, tests, and docs against that contract.

## Reproduction Assessment

Yes, source inspection is enough to reproduce the missing capability: current main has no Reminders route, no reminder-purpose/display fields in cron job/list contracts, and no filter that can exclude reminders from Cron.

## Solution Assessment

Unclear as proposed. The user outcome is sound, but freeform tags or payload extraction alone are not the best permanent fix; a typed purpose/display contract coordinated with https://github.com/openclaw/openclaw/issues/53107 is safer.
