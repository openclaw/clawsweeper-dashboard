## Summary

Keep open: current main and the latest release still provide only generic cron management, with no Reminders route, no reminder-purpose/display metadata, and no reminder-specific list filter. The related cron tags/categories issue overlaps the classifier foundation but does not replace the requested Reminders UX or safe human-readable reminder presentation.

## What This Changes

The issue requests a dedicated Mission Control Reminders tab that separates one-shot reminder cron jobs from generic recurring cron jobs and displays safe human-readable reminder metadata.

## Best Possible Solution

Define one additive typed cron purpose and display-metadata contract that supports reminders and broader categorization, then implement reminder creation, filtering, Cron tab policy, Reminders UI, tests, and docs against that contract.

## Reproduction Assessment

Yes. Source inspection reproduces the missing capability: current main has no Reminders route, no reminder-purpose/display fields in cron contracts, and no filter that can exclude reminders from Cron.

## Solution Assessment

Unclear as proposed. The user outcome is sound, but the safest implementation needs a typed purpose/display contract coordinated with cron categorization rather than a tags-only or payload-parsing fix.
