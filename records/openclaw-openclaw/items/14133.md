## Summary

Keep open. Current main still exposes reminders through the generic Cron Jobs surface only: there is no Reminders tab/route, no reminder classifier/display metadata in the cron protocol or storage shape, and no backend/UI filter; the open generic cron tags issue can inform the metadata contract but does not cover the reminder-specific UI.

## What This Changes

Request a dedicated Control UI Reminders tab that separates one-shot reminder cron jobs from recurring/system cron jobs and displays cleaned reminder metadata.

## Best Possible Solution

Add an additive cron classification/display metadata contract, have reminder-producing paths set it, support typed reminder and non-reminder filters, and add a Reminders tab with cleaned text, fire time, target/channel, delete action, tests, and docs.

## Reproduction Assessment

Not applicable as a runtime bug, but yes as a source-level feature gap: current main has cron navigation, rendering, protocol, storage, and list filters with no Reminders tab or reminder classifier.

## Solution Assessment

Yes at the direction level, but it needs maintainer product/protocol alignment. The maintainable path is additive metadata and typed filtering rather than parsing cron names or payload text in the UI.
