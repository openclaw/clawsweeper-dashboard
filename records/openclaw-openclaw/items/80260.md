## Summary

Keep open. The PR addresses a real, source-reproducible `/status` display bug and supplies sufficient terminal proof, but it overlaps https://github.com/openclaw/openclaw/pull/80417 and the new cost regression test has a fixture typo that weakens coverage of the cost suppression path.

## What This Changes

The branch changes status rendering so equivalent CLI runtime aliases can display the active OAuth auth label and hide API-key cost when the selected model resolves to an env API key, plus a regression test and changelog entry.

## Best Possible Solution

Land one canonical fix that makes `/status` report the actual active CLI OAuth auth path over unused env keys, combining the renderer-level selected/active display fix with resolver-level precedence if maintainers want both stored-profile and cached-credential paths covered.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence renderer reproduction with selected `anthropic/<model>`, active `claude-cli/<model>`, `modelAuth: "api-key"`, and `activeModelAuth: "oauth"`; current main prefers the selected auth for both label and cost visibility.

## Solution Assessment

Unclear. The renderer change is narrow and correct for the reproduced selected API-key plus active OAuth path, but maintainers should coordinate it with https://github.com/openclaw/openclaw/pull/80417 because that PR covers a related resolver-order path where active OAuth may never reach the renderer.
