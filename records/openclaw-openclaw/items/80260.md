## Summary

Keep open. The production change appears to be the right narrow fix and the PR body supplies sufficient after-fix renderer output, but the added regression test still does not force the selected `api-key` auth mode, so it can pass through the older unknown-auth runtime-alias path instead of proving the reported branch.

## What This Changes

The PR changes `/status` rendering so an equivalent CLI runtime alias with active OAuth can override a selected env api-key auth label and suppress the unused API-key cost line.

## Best Possible Solution

Land the narrow renderer fix after tightening the focused regression test to explicitly set selected `api-key` auth with active OAuth on an equivalent CLI runtime alias.

## Reproduction Assessment

Yes for the shared renderer path. The linked issue and current source show a concrete selected `api-key` label from `ANTHROPIC_API_KEY` can shadow an active `claude-cli` OAuth route; I did not run a full live gateway reproduction in this read-only review.

## Solution Assessment

Mostly yes. The production change is in the shared status renderer and preserves the non-alias api-key path, but the test should explicitly force `modelAuth: "api-key"` so it fails on current main for the reported regression.
