## Summary

Keep open. The reported bug is valid, but this PR only fixes the renderer when `activeModelAuth` is already OAuth; the live `/status` path can still resolve the active `claude-cli` auth label as the env API key before the new branch runs.

## What This Changes

The PR updates `src/status/status-message.ts`, adds a focused `buildStatusMessage` regression test, and adds a changelog entry for the runtime-alias OAuth label/cost display case.

## Best Possible Solution

Fix the live `/status` auth-label path so CLI OAuth wins over alias-inherited env keys for active CLI runtimes, then keep or adapt the renderer cost suppression with end-to-end regression coverage.

## Reproduction Assessment

Yes. Source inspection shows the live `/status` path can pass `api-key` for both selected and active auth because `resolveModelAuthLabel` checks env API keys before cached `claude-cli` OAuth credentials.

## Solution Assessment

No. The renderer fallback is useful, but it is not sufficient by itself because the live caller may never pass the OAuth active label needed to trigger it.
