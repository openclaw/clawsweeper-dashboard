## Summary

Keep open. The PR has the right narrow intent, but it only fixes the renderer when `activeModelAuth` is already OAuth; the real `/status` path can still resolve active `claude-cli` auth as the env API-key before this renderer logic runs.

## What This Changes

The branch updates status-message auth/cost rendering for equivalent runtime aliases, adds one status regression, and adds a changelog entry.

## Best Possible Solution

Land one canonical fix that makes `/status` prefer active CLI OAuth over alias-derived env API-key labels, with production-path coverage through `buildStatusText` or the auth-label resolver plus the renderer cost/label checks.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: production `/status` resolves auth labels before rendering, and `resolveModelAuthLabel("claude-cli")` can return the alias-derived env API-key before checking Claude CLI credentials.

## Solution Assessment

No. The renderer override is useful but incomplete because it only works after `activeModelAuth` is already OAuth; the safer fix must cover the production resolver/status-text path that can still supply API-key for active `claude-cli`.
