## Summary

Keep open: the reported update bug is real, but this PR still blocks the self-update relocation path by making the shared npm resolver return null before npm ownership detection can classify the trapped install. The PR also needs real proof of the patched `openclaw update` path, not only a manual reinstall.

## What This Changes

The PR adds per-Node ephemeral npm-prefix detection in update-global, switches global install argv to PATH npm for Homebrew/nvm/asdf/volta/fnm/n layouts, adds unit coverage, and adds a changelog entry.

## Best Possible Solution

Separate npm ownership detection from reinstall command selection so trapped npm installs still classify as npm while the reinstall command uses PATH npm for known ephemeral prefixes, then prove the actual self-update relocation path.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: with `pkgRoot` inside a Homebrew Cellar-style global install and PATH npm rooted elsewhere, the PR makes npm ownership detection return null and `runSelfUpdate` skips as `not-git-install`.

## Solution Assessment

No. Refusing per-version npm is the right direction, but putting the guard inside `resolvePreferredNpmCommand` is not the best fix because that helper is also used to detect npm ownership.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91
