## Summary

Keep open: the PR targets a real source-reproducible update bug, but the latest diff still makes trapped npm installs undetectable, so the self-update path can skip before it ever relocates the install.

## What This Changes

The PR adds Homebrew Cellar and Node version-manager prefix detection so update install argv falls back to PATH npm, with unit coverage and a changelog entry.

## Best Possible Solution

Split npm ownership detection from install-command selection so ephemeral installs still classify as npm, update argv uses PATH `npm`, and regression coverage proves the runner reaches global update from a trapped install.

## Reproduction Assessment

Yes. The linked issue provides concrete Homebrew dangling-symlink evidence, and current main source shows the resolver can select the adjacent Cellar npm; I did not run a destructive brew upgrade locally.

## Solution Assessment

No. Refusing the per-version npm is the right direction, but doing it inside the shared resolver also breaks manager detection, so the fix should separate ownership detection from install-command selection.
