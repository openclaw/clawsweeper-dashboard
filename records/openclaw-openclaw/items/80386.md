## Summary

Keep open: the PR targets a real updater bug, but its resolver change disables npm ownership detection for the same trapped installs and the supplied proof does not show the patched `openclaw update` path.

## What This Changes

The PR adds ephemeral per-Node npm-prefix detection to update command selection, regression coverage for Homebrew/nvm/asdf/volta/fnm/n layouts, and a changelog entry.

## Best Possible Solution

Split npm ownership detection from install-command selection so trapped installs still classify as npm, then use PATH npm only for the reinstall command when the owning prefix is ephemeral, with runner-level regression coverage.

## Reproduction Assessment

Yes. Source inspection reproduces the PR regression: a trapped install whose PATH npm root points elsewhere loses the final npm ownership fallback and `runSelfUpdate` skips before global update; the linked issue supplies concrete Homebrew dangling-symlink evidence for the underlying bug.

## Solution Assessment

No. Refusing per-version npm is the right direction, but doing it inside the shared resolver is not the best fix because that resolver also proves npm ownership; separating ownership detection from reinstall command selection is safer.

## Review Findings
