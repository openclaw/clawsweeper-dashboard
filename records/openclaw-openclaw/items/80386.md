## Summary

Keep open: this is a focused bug fix, but the current patch breaks the affected Homebrew Cellar update path before the install command is ever built, and the supplied real-world proof does not exercise live `openclaw update` after the resolver change.

## What This Changes

The PR changes the global npm self-update resolver to avoid per-version Node manager prefixes, adds regression coverage for those layouts, and records the fix in the changelog.

## Best Possible Solution

Keep the ephemeral-prefix guard, but split manager classification from command selection so affected Cellar installs are still detected as npm and then update through PATH npm into the stable prefix, with a regression test that exercises `detectGlobalInstallManagerForRoot` or the update target path.

## Reproduction Assessment

Yes. The reported current-main bug is source-reproducible from the package-root-derived npm resolver and the reporter's Homebrew Cellar evidence; the PR regression is also source-reproducible because the update runner skips when manager detection returns `null`.

## Solution Assessment

No. The direction is right, but returning `null` from the only npm fallback used by manager detection prevents the reported Cellar install from reaching the self-update flow; the safer fix is to keep npm detection true while resolving the install command to PATH npm for ephemeral prefixes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep ephemeral installs detectable as npm:** `src/infra/update-global.ts:511`
  - body: The update runner first calls `detectGlobalInstallManagerForRoot` and only performs a package update when that returns a manager. In the reported Cellar case, PATH npm reports the stable `/opt/homebrew/lib/node_modules` root, which does not match the current Cellar package root, so the detector falls through to `resolvePreferredNpmCommand(pkgRoot)`. With this new `return null`, detection returns `null` and `openclaw update` skips as `not-git-install` instead of falling back to PATH npm. Please keep these package roots classified as npm while making the command resolver choose plain `npm`.
  - confidence: 0.91
