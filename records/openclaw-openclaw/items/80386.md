## Summary

Keep open: the PR targets a real updater bug, but the current patch changes a shared resolver so trapped npm installs can stop being detected and self-update can skip instead of relocating. The supplied proof also does not exercise the patched `openclaw update` path.

## What This Changes

The PR adds per-Node npm-prefix detection for Homebrew, nvm, asdf, volta, fnm, and n layouts, falls back to PATH npm for those installs, adds unit coverage, and updates the changelog.

## Best Possible Solution

Separate npm ownership detection from reinstall command selection so trapped installs still classify as npm while known per-Node prefixes reinstall through PATH npm, then add regression coverage for manager detection and self-update relocation.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with `pkgRoot` inside a Homebrew Cellar global install and PATH npm rooted elsewhere, the PR makes npm ownership detection return null and `runSelfUpdate` can skip as `not-git-install`.

## Solution Assessment

No. Refusing per-version npm for reinstall is the right direction, but putting the guard inside the shared resolver is not the best fix because that helper also detects npm ownership.

## Review Findings
