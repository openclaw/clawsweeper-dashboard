## Summary

Keep open: the reported bug is valid, but this PR's current resolver change would make trapped installs undetectable as npm global installs before the relocation path runs, and its real proof does not exercise `openclaw update` after the patch.

## What This Changes

The PR adds ephemeral per-Node npm-prefix detection for Homebrew Cellar, nvm, asdf, volta, fnm, and n installs, updates unit coverage, and adds a changelog fix entry.

## Best Possible Solution

Split npm ownership detection from reinstall-command selection so trapped installs still classify as global npm installs, then use PATH npm or a stable prefix only for the reinstall command and cover the self-update path in tests.

## Reproduction Assessment

Yes. Source inspection shows the high-confidence path: a trapped Cellar package root with PATH npm pointing elsewhere loses the final npm ownership fallback and `runSelfUpdate` skips before relocation.

## Solution Assessment

No. Refusing per-version npm is the right direction, but putting that guard in the shared resolver breaks manager detection; the safer fix separates detection from command selection.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep trapped npm installs detectable:** `src/infra/update-global.ts:510-511`

In the reported state PATH npm points at a stable root, so manager detection misses the Cellar package root and `runSelfUpdate` skips as `not-git-install` before any relocation can happen; split ownership detection from reinstall-command selection and cover that path.
  - confidence: 0.91
