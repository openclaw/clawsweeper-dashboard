## Summary

Keep open: current main and the latest v2026.6.6 release still route `/v1/chat/completions` through the normal full agent ingress path, with no public `x-openclaw-light-context`, `http.systemPrompt`, or `x-openclaw-session` contract. The linked stable-session PR remains open and partial; it does not solve the larger low-context/voice-mode prompt assembly request.

## What This Changes

Request a low-context/voice mode and stable HTTP session controls for `/v1/chat/completions` to reduce time-to-first-byte from full agent context assembly.

## Best Possible Solution

Define a trusted low-context Chat Completions contract, document its tool/memory/bootstrap/security semantics, and land stable session reuse through https://github.com/openclaw/openclaw/pull/69060 or a replacement without conflating the two pieces.
