## Summary

Keep open. Current main still lacks both the stable `x-openclaw-session` hint and the larger HTTP `lightContext`/voice-mode or HTTP-specific prompt policy, while Chat Completions continues through the normal agent ingress path.

## What This Changes

The issue requests a low-context HTTP Chat Completions mode, reusable HTTP session keys, and an optional HTTP-specific system prompt override to reduce TTFB for real-time voice callers.

## Best Possible Solution

Define and document a trusted low-context HTTP prompt policy for Chat Completions, wire it through the existing prompt/bootstrap/tool/memory seams with Gateway tests, and treat the stable-session hint as a separate smaller session-warm improvement.

## Reproduction Assessment

Yes for the source-level gap: current main has no `x-openclaw-light-context`, no stable `x-openclaw-session`, and no HTTP prompt override, while the endpoint still runs through normal agent ingress. I did not reproduce the live latency timing numbers.

## Solution Assessment

Unclear. The requested direction is valid, but the maintainable fix needs explicit public semantics for tools, memory, bootstrap context, agent personality, session behavior, and trusted HTTP security before implementation.
