## Summary

Keep open: current main and v2026.5.18 still route Chat Completions through the full agent ingress path, with no public low-context HTTP contract or HTTP-specific prompt override; the linked stable-session PR only covers the session-warm/QMD portion and is still open.

## What This Changes

The issue requests a low-context or voice-mode path for `/v1/chat/completions`, plus stable HTTP session reuse, to reduce first-token latency for realtime callers.

## Best Possible Solution

Define and document a trusted low-context Chat Completions policy, wire it through existing prompt/bootstrap/tool/memory seams with focused Gateway coverage, and land stable-session reuse through https://github.com/openclaw/openclaw/pull/69060 or a replacement.
