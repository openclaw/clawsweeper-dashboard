## Summary

Keep open: current main and v2026.5.18 still run OpenAI-compatible Chat Completions through the normal full-agent path, with no public low-context HTTP mode, HTTP-specific prompt override, or merged stable `x-openclaw-session` contract.

## What This Changes

The issue requests a low-latency HTTP Chat Completions mode that avoids full agent context assembly and supports stable HTTP session reuse or prompt override for voice/realtime callers.

## Best Possible Solution

Define and document a trusted low-context or relay policy for OpenAI-compatible HTTP calls, wire it through existing prompt/bootstrap/tool/memory seams with focused Gateway tests, and land stable-session reuse through https://github.com/openclaw/openclaw/pull/69060 or a replacement.
