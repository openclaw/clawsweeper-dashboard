## Summary

Keep open: current main and v2026.5.18 still run OpenAI-compatible Chat Completions through the normal agent ingress path with regular session and prompt behavior. The open stable-session PR only covers the warm-cache subproblem, and the related passthrough/no-persistence issue does not replace the remaining low-context voice-mode API decision.

## What This Changes

The issue requests adding a low-context or voice-oriented mode and stable session controls to the OpenAI-compatible Chat Completions endpoint to reduce TTFB caused by full agent context assembly.

## Best Possible Solution

Define and document a trusted low-context Chat Completions contract, reuse existing prompt/bootstrap seams where possible, and coordinate stable-session and passthrough/no-persistence behavior with the open related items.

## Reproduction Assessment

Yes for the source-level gap: current main has no `x-openclaw-light-context`, no `x-openclaw-session`, and no HTTP-specific prompt override, while Chat Completions still calls normal agent ingress. I did not reproduce the live latency timing numbers.

## Solution Assessment

Unclear. The requested direction is plausible, but maintainers need to choose the public low-context HTTP contract; the stable-session portion should land through https://github.com/openclaw/openclaw/pull/69060 or a replacement.
