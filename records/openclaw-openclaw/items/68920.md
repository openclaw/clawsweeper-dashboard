## Summary

Keep open. Current main still lacks the requested low-context or voice-mode policy for OpenAI-compatible Chat Completions, and the only linked implementation is an open partial PR for stable session hints rather than the full prompt/context latency problem.

## What This Changes

The issue requests a low-latency HTTP Chat Completions mode with lighter agent context, stable session reuse, and optional per-agent HTTP prompt override.

## Best Possible Solution

Define and document an opt-in low-context Chat Completions contract that reuses the existing prompt/bootstrap seams safely, and handle stable session hints as a separate smaller API change.

## Reproduction Assessment

Yes for the source-level gap: current main has no `x-openclaw-light-context`, no `http.systemPrompt`, and no stable `x-openclaw-session`, while Chat Completions still enters normal agent ingress with full prompt mode for ordinary sessions. I did not live-reproduce the reporter's exact TTFB timing numbers.

Unclear. The requested direction is plausible, but the best fix needs a public low-context HTTP policy and security contract; the stable-session piece should land through https://github.com/openclaw/openclaw/pull/69060 or a replacement while voice-mode context policy remains separate.
