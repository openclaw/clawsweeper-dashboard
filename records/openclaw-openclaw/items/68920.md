## Summary

Keep open. Current main still lacks an HTTP light-context or voice-mode prompt policy for `/v1/chat/completions`, and the related open stable-session PR only addresses one warm-cache subproblem rather than the full context assembly issue.

## What This Changes

The issue requests a low-latency `/v1/chat/completions` mode that can reduce full agent prompt/context assembly and provide stable sessions for realtime voice callers.

## Best Possible Solution

Define an explicit HTTP prompt/context policy for low-latency callers, then wire it through the existing runner seams with config, docs, and tests. Keep the stable-session work separate unless it lands first as the narrow #69060 sub-fix.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Partial yes: current main has no `x-openclaw-light-context` or equivalent HTTP config path, and `x-openclaw-session` is not parsed. The precise latency numbers still require the reporter's model, workspace, and runtime setup.

## Solution Assessment

Is this the best way to solve the issue? Unclear: a bare header alone is not enough. The maintainable path should first define safe HTTP prompt-policy semantics, then wire `promptMode`/`bootstrapContextMode` and document the security boundary.
