## Summary

Keep open. Current main still lacks the requested low-context or voice-mode policy for OpenAI-compatible Chat Completions, and the related stable-session work is only an open partial fix while the endpoint continues through the normal agent ingress path.

## What This Changes

The issue requests a low-latency HTTP Chat Completions mode with lighter prompt/context assembly, stable session reuse, and possibly an HTTP-specific system prompt.

## Best Possible Solution

Define and document a trusted low-context Chat Completions policy, wire it through the existing prompt/bootstrap/tool/memory seams with focused Gateway tests, and handle the stable-session header as a separate smaller API improvement.

## Reproduction Assessment

Yes for the source-level gap: current main has no `x-openclaw-light-context`, no `http.systemPrompt`, no stable `x-openclaw-session`, and Chat Completions still runs through normal agent ingress with full default prompt mode. I did not reproduce the live latency timing numbers.

## Solution Assessment

Unclear. The requested direction is plausible, but the best fix requires a maintainer-owned public contract for low-context HTTP behavior; the stable-session part should land through https://github.com/openclaw/openclaw/pull/69060 or a replacement while the voice-mode policy remains separate.
