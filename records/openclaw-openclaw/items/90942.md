## Summary

Keep open: the symlinked-launcher fix is still useful, but the current PR can regress source-checkout doctor repair by returning null when the argv1 package root exists but lacks sandbox setup scripts instead of continuing to the cwd fallback.

## What This Changes

The PR exports `resolveSandboxScript`, switches doctor sandbox script lookup to `resolveOpenClawPackageRootSync`, and adds unit tests for symlinked and missing-launcher cases.

## Best Possible Solution

Keep the shared resolver for symlinked launchers, but continue checking cwd or remaining roots when the selected package root lacks the requested setup script; leave package-only auto-build to the separate follow-up.

## Solution Assessment

No, not in the current form. Reusing the shared package-root resolver is the right boundary, but the implementation should keep searching cwd or remaining roots when the first package root lacks the setup script.
