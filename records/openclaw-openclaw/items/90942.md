## Summary

Keep open: the branch addresses the original symlinked-launcher bug in principle, but the current implementation can regress source-checkout doctor repair because it stops after the first package root even when that root lacks the sandbox setup script.

## What This Changes

## Best Possible Solution

Keep the shared resolver for symlinked launchers, but continue checking cwd or remaining candidate roots when the selected package root lacks the requested setup script; leave package-only asset shipping to https://github.com/openclaw/openclaw/issues/90951.
