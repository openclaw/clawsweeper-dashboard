## Label Justifications

- `P1`: The report describes untracked long-lived process trees that can accumulate helpers and degrade real gateway/agent availability.
- `impact:session-state`: The core failure is drift between OpenClaw's tracked exec/session state and live OS process ownership.

## Summary

Keep open: current main and the latest release still use in-memory-only background exec/process ownership, while the linked fixing PR is open, conflicting, and explicitly covers only the in-process/session-loss half rather than durable restart orphan reaping.

## What This Changes

The issue asks OpenClaw to prevent background PTY exec runs from becoming untracked orphan process trees after restart or session-state loss.

## Best Possible Solution

Define and land one core-owned process-supervisor lifecycle path that records active PTY background run ownership durably, reconciles it on startup/session-loss, and explicitly chooses kill versus reattach semantics with focused tests.

## Reproduction Assessment

Yes, at source level: current main keeps supervisor/process sessions in memory, documents process-session loss on restart, and leaves reconcileOrphans as a no-op. I did not run a live restart/orphan-process reproduction in this read-only review.

## Solution Assessment

No, not as a final fix yet. The linked PR is a plausible partial in-process mitigation, but the central restart/session-loss behavior needs a maintainer decision on durable ownership and kill-versus-reattach semantics.
