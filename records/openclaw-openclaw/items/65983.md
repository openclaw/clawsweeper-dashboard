## Summary

Keep open. Current main still has process-local supervisor/session ownership, `reconcileOrphans()` remains a documented no-op, and the public docs still say background process sessions are lost on restart; the directly paired implementation PR #66001 is open and unmerged, so this issue should remain open until that PR is reviewed, landed, revised, or closed.

## What This Changes

Issue reports that background PTY exec runs can outlive in-memory OpenClaw ownership after restart or session-state loss, becoming untracked orphan process trees.

## Best Possible Solution

Review #66001 as the active implementation candidate, then land or revise it so PTY-backed background exec runs have durable active-run metadata plus startup reconciliation that matches the chosen kill-or-reattach policy.

## Reproduction Assessment

No, not fully. Source inspection gives a high-confidence path for ownership loss after restart because supervisor and process-session state are in memory, but the full live helper accumulation was not reproduced in this review.

## Solution Assessment

Unclear. #66001 is a plausible narrow fix for in-process session drift, but the issue's restart/session-loss wording needs an explicit maintainer decision on durable metadata and startup behavior before it can be considered solved.
