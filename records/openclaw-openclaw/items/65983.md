## Summary

Keep open. Current main still has process-local supervisor/session ownership, `reconcileOrphans()` remains a documented no-op, and the background-process docs still say sessions are lost on restart. The directly paired fix PR #66001 is open and unmerged with closing syntax for this issue, so this issue should stay open until that PR is reviewed, landed, revised, or closed.

## What This Changes

The issue requests durable ownership and startup reconciliation for background PTY exec process trees so they cannot survive gateway restart or session-state loss as untracked orphans.

## Best Possible Solution

Review #66001 as the active implementation candidate, then either land it with focused regression coverage and docs alignment or revise it to add durable active-run metadata plus startup reaping/re-registration for PTY-backed background exec runs.

## Reproduction Assessment

Partially. Current main gives a high-confidence source-level reproduction for lost ownership after restart because supervisor runs and process sessions are held in module-local memory; the full live helper accumulation was not reproduced here.

## Solution Assessment

Unclear. #66001 is a plausible narrow fix for in-process session drift, but the issue’s restart/session-loss wording likely needs an explicit maintainer decision on durable metadata and startup behavior before this can be called solved.
