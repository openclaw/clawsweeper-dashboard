## Summary

Keep open because current main still uses memory-only supervisor/session ownership for background exec, the docs still state sessions are lost on process restart, and the paired closing PR #66001 is open rather than shipped.

## What This Changes

The issue requests durable tracking and startup reconciliation for PTY-backed background exec runs so they cannot survive restart or session-state loss as untracked process trees.

## Best Possible Solution

Review #66001 as the active implementation candidate, then land or revise a fix that adds durable active-run metadata plus startup reaping or re-registration semantics for PTY-backed background exec runs with focused tests and docs alignment.

## Reproduction Assessment

Partially. Current main gives a high-confidence source-level reproduction for lost ownership after restart because supervisor runs and process sessions are held in memory, but the full live helper-process accumulation was not reproduced here.

## Solution Assessment

Unclear. #66001 is a plausible narrower fix for in-process session drift, but this issue’s restart/session-loss path likely needs explicit durable metadata and startup behavior before it is fully solved.
