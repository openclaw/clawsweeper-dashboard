## Summary

Keep open. Current main and the latest release still use process-local supervisor/session ownership with `reconcileOrphans()` as a no-op, and the directly linked closing PR remains open with review blockers.

## What This Changes

The issue requests durable cleanup or reconciliation for background PTY exec runs that can survive gateway restart/session loss and become untracked orphan process trees.

## Best Possible Solution

Review, revise, and land or replace https://github.com/openclaw/openclaw/pull/66001 with ownership-aware reconciliation, while making the cold-start stale PTY policy explicit as kill, re-register, or a narrower follow-up.

## Reproduction Assessment

Yes, source-reproducible: start a background PTY exec tracked only in module-local supervisor/session maps, then lose gateway process/session state; current `reconcileOrphans()` is a no-op and `process list` reads only in-memory sessions. I did not run a live macOS orphan-process reproduction.

## Solution Assessment

Unclear. In-process reconciliation is a reasonable smaller fix direction, but the broader restart/session-loss behavior still needs maintainer policy on whether stale PTY runs should be killed, re-registered, or split into follow-up work.
