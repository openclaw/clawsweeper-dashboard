The issue asks OpenClaw to prevent background PTY exec process trees from surviving restart or session-state loss as untracked orphans.

## Best Possible Solution

Implement a lifecycle-owned core process-supervisor policy that reconciles lost in-process sessions and covers restart/cold-start orphan cleanup, with real PTY process-tree validation before closing.

Not yet: the linked PR is a plausible in-process mitigation, but it explicitly does not solve durable restart/cold-start reaping. Maintainers should decide the stale-orphan policy and then land either one complete fix or a clearly scoped split with a tracked follow-up.
