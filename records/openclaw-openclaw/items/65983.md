## Summary

Keep open. Current main still uses process-local supervisor/session ownership, `reconcileOrphans()` is still a documented no-op, and the directly linked implementation candidate at https://github.com/openclaw/openclaw/pull/66001 remains open and unmerged.

## What This Changes

Report requests durable ownership and reconciliation for background PTY exec runs so restart or session-state loss cannot leave untracked live process trees.

## Best Possible Solution

Review, revise, and land or close https://github.com/openclaw/openclaw/pull/66001, with an explicit maintainer decision on whether stale PTY runs after restart should be killed, re-registered, or handled by a narrower follow-up.

## Reproduction Assessment

No live end-to-end reproduction was established here. Source inspection gives a high-confidence path for ownership loss because supervisor state, process sessions, and process-list visibility are in memory while orphan reconciliation is still a no-op.

## Solution Assessment

Unclear. The linked PR is the active implementation path, but it covers only part of the issue and still needs maintainer judgment on durable restart behavior before this can be considered solved.
