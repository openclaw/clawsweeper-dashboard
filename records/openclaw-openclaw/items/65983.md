## Summary

Keep open. Current main still has process-local supervisor/session ownership, a no-op orphan reconciler, and docs that explicitly state background sessions are lost on restart; the directly paired implementation PR #66001 is open, unmerged, and uses closing syntax for this issue, so this issue should stay open until that PR is reviewed, landed, or closed.

## What This Changes

The issue asks OpenClaw to prevent PTY-backed background exec process trees from surviving gateway restart or session-state loss as untracked orphaned workers.

## Best Possible Solution

Review #66001 as the active implementation candidate, then land it if it matches the intended lifecycle contract or revise it to include durable startup orphan reaping/re-registration plus focused regression coverage for PTY background exec visibility and cleanup.

## Reproduction Assessment
