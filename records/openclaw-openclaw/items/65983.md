## Summary

Keep open. Current main and v2026.5.12 still use in-memory-only exec/process ownership with `reconcileOrphans()` as a no-op, and the linked closing PR remains open and conflicting with review blockers.

## What This Changes

Request durable ownership or reconciliation for background PTY exec runs so restart or session-state loss cannot leave untracked orphan process trees.

## Best Possible Solution

Review, revise, and land or replace https://github.com/openclaw/openclaw/pull/66001 with ownership-aware reconciliation, while making the stale PTY cold-start policy explicit.

## Reproduction Assessment

Yes, source-reproducible: start a background PTY exec tracked only in module-local supervisor/session maps, then lose gateway process or session state; current `reconcileOrphans()` is a no-op and `process list` reads only in-memory sessions. I did not run a live macOS orphan-process reproduction.

## Solution Assessment

Unclear. In-process reconciliation is a reasonable smaller fix direction, but the broader restart/session-loss behavior still needs maintainer policy on whether stale PTY runs should be killed, re-registered, or split into follow-up work.
