## Summary

Keep open: current main and the latest release still keep background exec/process ownership in memory and leave `reconcileOrphans()` as a no-op, while the linked closing PR is still open and conflicting and only covers the in-process half of the lifecycle problem.

## What This Changes

The issue asks OpenClaw to prevent background PTY exec process trees from becoming untracked orphans after gateway restart or session-state loss.

## Best Possible Solution

Land one lifecycle-owned process-supervisor fix that covers lost in-process ownership and the restart/cold-start orphan policy, or explicitly split those two scopes with a tracked follow-up and real PTY process-tree proof.

## Reproduction Assessment

Yes, at source level: current main uses in-memory Maps for background exec sessions, `process list` reads those Maps, and `reconcileOrphans()` is still a no-op. I did not run a live macOS/Linux PTY restart orphan scenario in this read-only review.

## Solution Assessment

Not yet. The linked PR is a plausible in-process mitigation, but maintainers still need to decide the cold-start orphan policy before this issue is fully solved.
