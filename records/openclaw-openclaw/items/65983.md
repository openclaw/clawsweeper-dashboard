## Summary

Keep open. Current main and v2026.5.12 still use in-memory-only exec/process ownership with a no-op `reconcileOrphans()`, and the linked closing PR is still open, conflicting, and narrower than the restart-orphan report.

## What This Changes

The issue requests durable ownership and startup/session-loss reconciliation for background PTY exec runs so OpenClaw does not lose track of live worker process trees.

## Best Possible Solution

Revise and land or replace https://github.com/openclaw/openclaw/pull/66001 with ownership-aware reconciliation, then make the cold-start stale PTY policy explicit and covered by focused tests.

## Reproduction Assessment

Partially: source inspection gives high confidence that OpenClaw can lose exec/process ownership and report no sessions because tracking is in memory and `reconcileOrphans()` is a no-op. This review did not run a live macOS restart test proving the PTY process tree survives independently.

## Solution Assessment

Unclear. In-process reconciliation is a reasonable smaller fix direction, but the complete solution needs ownership-aware session checks and a maintainer decision for cold-start stale PTY runs.
