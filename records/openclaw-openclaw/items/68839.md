## What This Changes

The PR guards follow-up queue final cleanup with `FOLLOWUP_QUEUES.get(key) === queue` and adds a regression test for the `/stop` plus late-drain replacement-queue race.

## Best Possible Solution

Land this identity guard and regression test, or an equivalent patch, so https://github.com/openclaw/openclaw/issues/68838 closes through the merge.

## Reproduction Assessment

Yes. Source inspection shows D1 can await with a captured queue while `/stop` clears that key, then `enqueueFollowupRun(..., false)` can register Q2 before D1's key-only finalizer runs; the PR regression stages that interleaving directly.

## Solution Assessment

Yes. The identity check is the narrowest maintainable fix because it only prevents stale final cleanup from deleting a replacement queue while preserving explicit clear paths and rescheduling behavior.
