## Summary

Keep open. Current main has already covered most of the intended heartbeat/cron runtime forwarding and cron silent-token behavior, but it has not fully incorporated the PR's command system-prompt path: `resolveCommandsSystemPromptBundle` still resolves bootstrap context without forwarding `opts.bootstrapContextMode` or a heartbeat/default run kind. The branch should not be merged wholesale because main has since refactored and already carries broader runtime/cron pieces, but one narrow source change remains useful.

## What This Changes

The PR threads lightweight bootstrap context mode/run-kind through command system-prompt and embedded compaction paths, adds regression tests, and also changes cron announce delivery/docs to suppress silent `NO_REPLY` replies.

## Best Possible Solution

Port only the missing command-system-prompt forwarding onto current main, add a focused regression test and required changelog entry, and leave the already-present embedded-runner and cron silent-delivery behavior as-is.

## Reproduction Assessment

Yes for the remaining gap: a unit-level mock of `resolveBootstrapContextForRun` in `resolveCommandsSystemPromptBundle` can assert that heartbeat `opts.bootstrapContextMode: "lightweight"` is forwarded with `runKind: "heartbeat"`. Current main source inspection shows that call currently omits both fields.

## Solution Assessment

No, not as a direct merge of this stale branch. The underlying approach is sound, but the best current solution is a narrow current-main patch for `commands-system-prompt.ts` plus its regression and changelog, because the larger runtime/cron pieces are already present on main.
