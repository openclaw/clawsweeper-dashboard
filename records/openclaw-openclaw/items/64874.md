## Summary

Keep open: the diagnostic gap is still real, but this draft PR is not mergeable as-is because it targets stale status code, hard-codes a configurable limit, misses an active warning path, and has no real behavior proof.

## What This Changes

The PR changes bootstrap warning text, edit mismatch diagnostics, and status formatting to surface workspace bootstrap file truncation more visibly.

## Best Possible Solution

Land a rebased or replacement implementation that uses the active status and warning paths, reuses SessionSystemPromptReport plus analyzeBootstrapBudget, derives configured bootstrap limits, and includes focused tests plus after-fix proof.

## Reproduction Assessment

Yes, source-reproducible: current main has /context and doctor bootstrap diagnostics, while active /status does not read injectedWorkspaceFiles and edit mismatch recovery only appends generic current file contents. I did not run a live session because this review is read-only.

## Solution Assessment

No. The direction is useful, but this patch targets a stale status builder, misses the active embedded notice path, hard-codes a configurable limit, and introduces an unused parameter; the safer fix should reuse the existing bootstrap report and analysis contracts.

  - confidence: 0.92

  - confidence: 0.9

  - body: This changes `appendBootstrapPromptWarning`, but embedded runs compose the once-mode system notice through `buildBootstrapPromptWarningNotice`. The warning that still says only to treat Project Context as partial will not gain the edit-failure guidance unless the notice path is updated too.
