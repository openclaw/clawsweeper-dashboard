## Summary

Keep open: the diagnostic direction is useful, but this draft PR is conflicting, lacks contributor real-behavior proof, and still has correctness issues against current main.

## What This Changes

The branch adds a bootstrap truncation warning sentence, workspace-file health lines to `/status`, and a large-file truncation hint for edit mismatch errors.

## Best Possible Solution

Rebase or replace the branch with a narrow implementation in the active status builder that reuses `SessionSystemPromptReport`, `analyzeBootstrapBudget`, and resolved bootstrap limits, then add focused tests and after-fix proof.

## Reproduction Assessment

Yes, source-reproducible: current `/context` reads `systemPromptReport.injectedWorkspaceFiles`, while active `/status` does not, and edit mismatch recovery appends generic current content only. I did not run a live session because this review is read-only.

## Solution Assessment

No. The direction is useful, but the current PR targets a stale status surface, hard-codes an obsolete/default-sensitive bootstrap limit, and adds an unused parameter; the safer fix should reuse current shared bootstrap contracts.
