## Summary

Keep open. The direction is useful, but this draft still has missing real behavior proof and concrete correctness blockers: the workspace-file status is added to the stale status builder, the edit hint adds an unused parameter, and the hint uses a fixed 20,000-character threshold instead of the resolved bootstrap limit.

## What This Changes

The branch adds bootstrap truncation edit guidance, a workspace-files section to `/status`, and a large-file edit mismatch hint across three agent/status files.

## Best Possible Solution

Render workspace-file health in the active status builder using SessionSystemPromptReport, analyzeBootstrapBudget, and resolved config limits; derive edit hints from the same resolved limits; add focused regression tests while leaving enforcement or auto-archive policy to https://github.com/openclaw/openclaw/issues/45415.

## Reproduction Assessment

Yes, by source inspection. Current `/status` does not read `systemPromptReport.injectedWorkspaceFiles`, while `/context` already reports bootstrap file status and edit mismatch errors currently only include a current-content snippet.

## Solution Assessment

No. The diagnostic direction is useful, but the current patch targets the old status builder, hard-codes a stale bootstrap limit, and introduces an unused parameter under `noUnusedParameters`; the safer path is to reuse the current status/report contracts.
