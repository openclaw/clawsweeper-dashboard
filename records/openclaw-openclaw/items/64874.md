## Summary

Keep open. The requested diagnostics are still not fully present on current main, and this PR is a plausible implementation path, but the branch has correctness blockers, is a draft/conflicting external PR, and lacks required real behavior proof.

## What This Changes

The branch adds a bootstrap truncation warning sentence, workspace-file health lines to `/status`, and a large-file truncation hint for edit mismatch errors.

## Best Possible Solution

Rebase and repair the branch or replace it with a narrow implementation that uses the active status builder plus shared bootstrap budget/report contracts, then add focused tests and real behavior proof.

## Reproduction Assessment

Yes, source-reproducible: current `/context` has injected workspace file data, but current `/status` does not read it, and edit mismatch errors only append current file contents. I did not run a live session because this review is read-only.

## Solution Assessment

No. The direction is useful, but the current patch targets a stale status surface, hard-codes an obsolete/default-sensitive bootstrap limit, and adds an unused parameter; the safer fix should reuse `SessionSystemPromptReport`, `analyzeBootstrapBudget`, and resolved bootstrap limits.
