## Summary

Keep open: the diagnostic direction is useful and source-reproducible, but this draft PR lacks after-fix real behavior proof and still has blocking correctness issues against current main.

## What This Changes

The PR adds clearer bootstrap truncation warning text, a workspace-file health section to `/status`, and a large-file truncation hint for edit mismatch errors.

## Best Possible Solution

Land a rebased or replacement implementation that uses the active status and warning paths, reuses `SessionSystemPromptReport` plus `analyzeBootstrapBudget`, derives configured bootstrap limits, and includes focused tests plus after-fix proof.

## Reproduction Assessment

Yes, source-reproducible: current `/context` and doctor have bootstrap diagnostics, while active `/status` does not read `injectedWorkspaceFiles` and edit mismatch recovery only appends generic current file contents. I did not run a live session because this review is read-only.

## Solution Assessment

No. The direction is useful, but this patch targets a stale status builder, misses the active embedded notice path, hard-codes a configurable limit, and introduces an unused parameter; the safer fix should reuse the existing bootstrap report and analysis contracts.
