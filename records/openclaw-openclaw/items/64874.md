## Summary

Keep open: the diagnostic direction is useful and still relevant, but this external draft has no after-fix real behavior proof, is dirty against current main, and the current diff has blocking correctness issues.

## What This Changes

The branch adds bootstrap truncation warning copy, `/status` workspace-file health output, and large-file edit mismatch hints across bootstrap, host-edit, and status code.

## Best Possible Solution

Render workspace-file health in the active status builder using `SessionSystemPromptReport`, `analyzeBootstrapBudget`, and resolved config limits; derive edit hints from the same limits; add focused regression tests while leaving save enforcement or auto-archive policy to https://github.com/openclaw/openclaw/issues/45415.

## Reproduction Assessment

Yes, by source inspection. Current `/status` does not read `systemPromptReport.injectedWorkspaceFiles`, while `/context` already reports bootstrap file status and edit mismatch errors currently only include a current-content snippet.

## Solution Assessment

No. The diagnostic direction is useful, but the current patch targets a stale status builder, hard-codes a stale bootstrap limit, and introduces an unused parameter; the safer repair is to reuse the active status/report contracts and resolved limits.

  - confidence: 0.92
