## Summary

Keep open. The requested diagnostic direction is useful and still not fully present on current main, but this draft PR is not merge-ready because it misses active runtime surfaces, hard-codes an obsolete/configurable limit, has a typecheck blocker, and lacks contributor real behavior proof.

## What This Changes

The PR adds bootstrap truncation guidance to warning text, workspace-file health output to `/status`, and a large-file truncation hint for edit mismatch errors.

## Best Possible Solution

Rebase or replace the branch with a narrow implementation in the active status and embedded notice paths that reuses `SessionSystemPromptReport`, `analyzeBootstrapBudget`, and resolved bootstrap limits, then add focused tests and after-fix proof.

## Reproduction Assessment

Yes, source-reproducible: current `/context` and doctor have partial bootstrap diagnostics, but active `/status` omits `systemPromptReport.injectedWorkspaceFiles`, and edit mismatch recovery only appends generic current file contents. I did not run a live session because this review is read-only.

## Solution Assessment

No. The direction is useful, but this patch misses the active embedded warning and status builder paths, hard-codes a configurable bootstrap limit, and introduces an unused parameter; the safer fix should reuse the existing bootstrap report and analysis contracts.
