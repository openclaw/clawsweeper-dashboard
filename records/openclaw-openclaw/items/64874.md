## Summary

Keep this PR open for now: the diagnostic direction is useful, but the current patch is draft, lacks after-fix real behavior proof, and still has blocking implementation issues against current main.

## What This Changes

The PR adds bootstrap truncation guidance to injected warnings, a workspace-file health section to `/status`, and a large-file hint on edit mismatch errors.

## Best Possible Solution

Rework the PR against current main by adding the workspace-file health to the active status builder, reusing `SessionSystemPromptReport` plus `analyzeBootstrapBudget`, deriving configured bootstrap limits, updating both warning paths, and covering the behavior with focused tests plus real output proof.

## Reproduction Assessment

Yes, source-reproducible: current main exposes bootstrap health through `/context` and docs, while active `/status` does not read `injectedWorkspaceFiles` and edit mismatch recovery only appends generic current file contents. I did not run a live session because this review is read-only.

## Solution Assessment

No. The direction is useful, but this patch targets stale status wiring, hard-codes a configurable limit, misses the embedded warning notice path, and introduces an unused parameter; the safer implementation should reuse the existing bootstrap report and analysis contracts.
