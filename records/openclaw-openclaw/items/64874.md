## Summary

Keep open. The PR addresses a real current-main diagnostics gap, but it is not merge-ready because the status path moved, the edit hint uses a stale fixed threshold, and the user-facing change needs release notes.

## What This Changes

The PR adds bootstrap truncation edit guidance, workspace bootstrap file health in `/status`, and a large-file truncation note on edit mismatch errors.

## Best Possible Solution

Adapt the diagnostics to current main by rendering workspace-file health from `src/status/status-message.ts`, reusing `SessionSystemPromptReport` and resolved bootstrap limits, adding focused tests, and adding a changelog entry if this lands as a user-facing fix.

## Reproduction Assessment

Yes. A session whose `systemPromptReport.injectedWorkspaceFiles` contains a bootstrap file over the 12,000-character default demonstrates that `/context` can report truncation while current `/status` does not, and edit mismatch errors only include the current-content snippet.

## Solution Assessment

No. The direction is useful, but the branch should move the status work to the current builder and derive thresholds from report/config-backed bootstrap limits rather than hard-coding 20,000 chars.
