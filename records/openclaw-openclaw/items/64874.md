## Summary

Keep open: the diagnostic direction is useful, but this PR is still incorrect against current main and needs contributor real-behavior proof before merge.

## What This Changes

The PR adds edit-failure guidance to bootstrap truncation warnings, a workspace-file health section to `/status`, and a large-file edit mismatch hint.

## Best Possible Solution

Render workspace-file health in the active status builder using `SessionSystemPromptReport`, `analyzeBootstrapBudget`, and resolved config limits, with focused regression tests while leaving enforcement or auto-archive policy to the linked feature discussion.

## Reproduction Assessment

Yes, by source inspection. Current `/status` does not read `systemPromptReport.injectedWorkspaceFiles`, while `/context` already reports bootstrap file status and edit mismatch errors currently only include a current-content snippet.

## Solution Assessment

No. The direction is useful, but the patch targets the old status builder, hard-codes a stale bootstrap limit, and introduces an unused parameter under `noUnusedParameters`.
