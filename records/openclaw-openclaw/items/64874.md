## Summary

This PR should stay open as a plausible diagnostic improvement for https://github.com/openclaw/openclaw/issues/45415, but the current draft branch is not mergeable as-is because it targets a stale status builder, hard-codes a configurable bootstrap limit, misses an active warning path, and lacks real behavior proof.

## What This Changes

The PR branch changes bootstrap warning text, edit mismatch diagnostics, and `/status` output to make truncated workspace bootstrap files more visible.

## Best Possible Solution

Land a rebased or replacement implementation that uses the active status and warning paths, reuses SessionSystemPromptReport plus analyzeBootstrapBudget, derives configured bootstrap limits, and includes focused tests plus after-fix proof.

## Reproduction Assessment

Yes, source-reproducible: current `/context` and doctor have bootstrap diagnostics, while active `/status` does not read injectedWorkspaceFiles and edit mismatch recovery only appends generic current file contents. I did not run a live session because this review is read-only.

## Solution Assessment

No. The direction is useful, but this patch targets a stale status builder, misses the active embedded notice path, hard-codes a configurable limit, and introduces an unused parameter; the safer fix should reuse the existing bootstrap report and analysis contracts.

  - confidence: 0.92

  - body: The edit mismatch hint is gated on a fixed 20,000-character threshold, but current docs and config default `bootstrapMaxChars` to 12,000 and operators can override it. A default-truncated bootstrap file between 12K and 20K would still get only the generic mismatch hint; derive this from the report/config-backed bootstrap limit instead.
  - confidence: 0.9

  - body: This changes `appendBootstrapPromptWarning`, but embedded runs compose the once-mode system notice through `buildBootstrapPromptWarningNotice`. The warning that still says only to treat Project Context as partial will not gain the edit-failure guidance unless the notice path is updated too.
