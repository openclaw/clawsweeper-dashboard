## Summary

Keep open. The PR addresses a real diagnostics gap, but the diff is not mergeable as written because the `/status` work targets an obsolete entry point, the edit hint hard-codes the old bootstrap limit, and the external PR lacks real after-fix behavior proof.

## What This Changes

The PR adds bootstrap truncation edit guidance, a workspace-file health section to `/status`, and a large-file edit mismatch hint.

## Best Possible Solution

Rework this PR or a narrow replacement to render workspace-file health from `src/status/status-message.ts`, reuse `SessionSystemPromptReport` plus `analyzeBootstrapBudget` with resolved limits, and add focused regression tests while leaving enforcement or auto-archive policy to the linked feature discussion.

## Reproduction Assessment

Yes, by source inspection. Current `/status` does not read `systemPromptReport.injectedWorkspaceFiles`, while `/context` already reports bootstrap file status, and edit mismatch errors currently only include a current-content snippet.

## Solution Assessment

No. The direction is useful, but the current patch targets the old status builder and hard-codes 20,000 instead of using report/config-backed bootstrap limits.
