## Summary

## What This Changes

The issue reports that compaction and reset paths can drop or mis-normalize `messageProvider` and can ignore zero-token compaction accounting, breaking session identity and safeguard behavior.

## Best Possible Solution

Review, update, and land or replace https://github.com/openclaw/openclaw/pull/69270 with shared provider resolution, nonnegative compaction accounting, and targeted regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows missing `messageProvider` in command reset, gateway reset, and subscribe compaction hook contexts, plus positive-only token guards that skip valid `tokensAfter: 0` results.

## Solution Assessment

Yes. The narrow maintainable fix is a shared normalized provider resolver plus nonnegative compaction accounting; this issue should stay open until the linked PR merges or is superseded by an equivalent focused fix.
