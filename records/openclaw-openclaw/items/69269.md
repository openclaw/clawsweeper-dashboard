## Summary

Keep open: current `main` still shows the reported reset/compaction invariant gaps, and this issue is paired with an open fixing PR using closing syntax, so it should close only after that PR or an equivalent fix lands.

## What This Changes

The issue reports that compaction and reset paths can lose or mis-normalize hook `messageProvider` and leave stale token/cost accounting after zero-token compaction.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/69270 with a narrow fix that centralizes provider resolution for reset/compaction hooks and accepts zero-token compaction accounting with regression coverage.

## Reproduction Assessment

Yes. The high-confidence source reproduction is to inspect current-main hook contexts for missing `messageProvider`, then follow `tokensAfter: 0` through the positive-only accounting helpers that prevent cached totals from resetting.

## Solution Assessment

Yes for the linked fix direction. A shared normalized provider resolver plus nonnegative compaction accounting is the narrow maintainable shape; the issue should remain open until that implementation merges or is superseded.
