## Summary

## What This Changes

The issue requests consistent reset/compaction session invariants: normalized `messageProvider` in hook contexts when known, no fabricated providers for non-channel keys, and refreshed token/cost accounting after zero-token compaction results.

## Best Possible Solution

Review, refresh, and land or replace https://github.com/openclaw/openclaw/pull/69270 with shared provider resolution, nonnegative compaction accounting, and targeted regression coverage for reset, gateway reset, subscribe compaction, and zero-token compaction results.

## Solution Assessment

Yes. The maintainable fix shape is the linked coordinated PR's shared provider resolver plus nonnegative compaction accounting; this issue should stay open until that PR lands or is superseded by an equivalent focused fix.
