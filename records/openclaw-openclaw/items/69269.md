## Summary

Keep open: current main and the latest release still show the reported session invariant gaps by source inspection, and the linked implementation PR https://github.com/openclaw/openclaw/pull/69270 remains open, conflicting, and unmerged.

## What This Changes

The issue asks for compaction and reset paths to preserve normalized messageProvider hook context and refresh token/cost accounting even when compaction yields zero tokens.

## Best Possible Solution

Review, refresh, and land or replace https://github.com/openclaw/openclaw/pull/69270 with shared provider resolution, nonnegative compaction accounting, and regression coverage for reset, gateway reset, subscribe compaction, and zero-token compaction outcomes.

## Reproduction Assessment

Yes. Source inspection on current main shows missing messageProvider on reset and subscribe compaction hook contexts, plus positive-only token guards that skip a valid zero-token compaction result.

## Solution Assessment

Yes. The maintainable fix shape is the linked coordinated PR or an equivalent replacement that centralizes provider resolution and treats finite nonnegative compaction token results as authoritative.
