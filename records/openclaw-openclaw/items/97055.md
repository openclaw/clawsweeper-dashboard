## Summary

Keep open: current main and the latest release still drop the configured custom Ollama memory provider id before endpoint lookup, and https://github.com/openclaw/openclaw/pull/97059 is the open closing fix candidate for this issue.

## What This Changes

The issue asks for `memorySearch.provider` custom Ollama provider ids to preserve their configured endpoint and auth when creating memory embeddings.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/97059 with provider-id propagation plus upgrade-safe cache identity handling.

## Reproduction Assessment

Yes at source level. Current main resolves a custom provider id to the Ollama memory adapter, then the gateway memory-specific path omits the provider id and the Ollama adapter overwrites it before endpoint lookup.

## Solution Assessment

Yes for the issue direction. The narrow maintainable fix is to pass the existing provider id through the gateway memory-specific path and stop the Ollama adapter from hardcoding the base provider, with the linked PR handling proof and cache identity review.
