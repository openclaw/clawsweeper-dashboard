The issue requests a documented or CLI-supported safe migration path for 5.28-era `openai-codex/*` Codex/ChatGPT OAuth deployments moving to the June 6.x train.

## Best Possible Solution

Coordinate the open migration-plan and diagnostic work into one supported upgrade path that tells 5.28-era Codex OAuth operators exactly when doctor will migrate, when legacy routes are retained, how to validate SQLite auth import/order, and when to restart or re-auth.
