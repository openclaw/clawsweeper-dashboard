## Summary

Keep open: current main still relies on writing another boundary for empty safeguard preparations, and the regression coverage explicitly accepts repeated empty boundary writes, so this is not implemented-on-main or stale cleanup.

## What This Changes

The issue reports that safeguard-mode compaction can re-enter the empty-preparation boundary path indefinitely for broken-state sessions, blocking attached agent invocations until the session is quarantined.

## Best Possible Solution

Add idempotent empty-preparation suppression in the safeguard or Pi trigger path, scoped to the session boundary and reset by new real conversation entries, with regression coverage for repeated retries.

## Reproduction Assessment

No high-confidence end-to-end reproduction was established. The source path is reproducible by invoking the safeguard handler with empty `messagesToSummarize` and `turnPrefixMessages`, and current tests model repeated empty preparations, but the broken persisted session state was not recreated live.

## Solution Assessment

Yes, the requested direction is the right class of fix. The narrow maintainable solution is to suppress re-evaluation of the same empty boundary until new real content arrives, rather than changing compaction mode or relying on operator quarantine.
