## Summary

Keep open: current main still has the same safeguard empty-preparation branch that writes another boundary marker and relies on Pi honoring that marker, while the existing regression test explicitly allows repeated empty preparations to write repeated boundaries. I did not establish the broken journal state live, but the source path matches the reporter’s logs closely enough to treat this as a valid narrow bug report.

## What This Changes

The issue reports that safeguard-mode compaction can repeatedly re-trigger on broken sessions whose summarizable and turn-prefix entries contain no real conversation messages.

## Best Possible Solution

Add an idempotent suppression point for empty safeguard preparations scoped to the session boundary or route safeguard auto-compaction through the OpenClaw-owned compaction path, with regression coverage for repeated empty preparations without new real entries.

## Reproduction Assessment

No high-confidence live reproduction was established. The report provides deterministic after-state logs, and current source still exposes the same empty-preparation safeguard branch, so the bug is source-reproducible but not fully exercised end to end here.

## Solution Assessment

Yes, the requested direction is a reasonable bug fix, but the safest implementation should suppress repeated evaluation of the same empty boundary rather than adding an operator workaround or changing the default compaction mode.
