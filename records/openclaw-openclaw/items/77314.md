## What This Changes

The issue reports that safeguard-mode compaction can repeatedly re-trigger on sessions with no real conversation messages because the written compaction boundary is not honored by the upstream trigger.

## Best Possible Solution

Add an idempotent suppression point for repeated empty safeguard preparations scoped to the same session boundary, with regression coverage that proves the same empty prefix does not re-enter a grace-timeout loop.
