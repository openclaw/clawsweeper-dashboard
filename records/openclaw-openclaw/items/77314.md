## Summary

Keep open: current main still returns a boundary-only compaction result for empty safeguard preparations, and the current regression test explicitly permits repeated empty preparations to write another boundary instead of suppressing same-state retries. Prior fixes cover adjacent no-real-message cases but do not prove this broken-session loop is resolved.

## What This Changes

Prevent safeguard-mode compaction from repeatedly re-triggering on broken sessions whose compaction preparation has no real conversation messages.

## Best Possible Solution

Add an idempotent suppression or skip point scoped to the unchanged empty preparation/boundary, with regression coverage for repeated empty auto-compaction without new real conversation entries.

## Solution Assessment

Yes. A narrow fix in the safeguard or embedded-runner compaction boundary is preferable to changing the default compaction mode or adding an operator workaround.
