## Summary

Keep open: current main still has the empty-preparation safeguard branch that writes another boundary marker, and the current regression test explicitly permits repeated empty preparations to write repeated boundaries rather than suppressing the broken-state retry loop.

## What This Changes

Break the safeguard compaction re-trigger loop for broken-state sessions with no real conversation messages while keeping safeguard mode enabled.

## Best Possible Solution

Add a scoped idempotency or boundary-honoring guard for empty safeguard preparations, with regression coverage proving repeated empty preparations without new real entries do not keep the runner in compaction grace retry.

## Reproduction Assessment

No high-confidence live reproduction was established. The issue is source-reproducible from current main because the reported log line and empty-preparation branch are present, and current tests expect repeated empty preparations to write another boundary.

## Solution Assessment

Yes, a narrow OpenClaw-owned compaction fix is the best direction. The safer path is to suppress repeated evaluation of the same empty boundary rather than disabling safeguard mode or relying on operator session quarantine.
