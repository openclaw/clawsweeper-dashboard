## Summary

Keep open: current main still contains the empty-preparation safeguard branch described in the report, and the existing regression test explicitly allows repeated empty preparations to write repeated boundary summaries. The issue is credible and narrow, but I did not establish a live broken-session reproduction.

## What This Changes

The issue reports that safeguard-mode compaction can repeatedly re-trigger on a broken session with no real conversation messages despite writing a boundary marker.

## Best Possible Solution

Add an idempotent suppression point for repeated empty safeguard preparations scoped to the unchanged session boundary, with regression coverage that prevents the same broken prefix from re-entering the compaction grace loop while preserving normal compaction for real new content.

## Reproduction Assessment

No high-confidence live reproduction was established. The issue is source-reproducible because current main still returns the reported boundary from the empty-preparation safeguard branch and has a test expecting repeated empty preparations to write another boundary.

## Solution Assessment

Yes, the best direction is a narrow idempotency guard in the safeguard/embedded-runner compaction path. Operator watchdogs or switching `mode` to `default` are workarounds, not the maintainable fix.

Fix the compaction safeguard re-trigger loop reported in https://github.com/openclaw/openclaw/issues/77314. Current main still returns a boundary result from `src/agents/pi-hooks/compaction-safeguard.ts` when both `messagesToSummarize` and `turnPrefixMessages` have no real conversation messages, and `src/agents/pi-hooks/compaction-safeguard.test.ts` currently expects repeated empty preparations to write repeated boundaries. Add an idempotent suppression or boundary-aware guard scoped to the same unchanged empty preparation/session boundary so a broken session cannot keep re-entering the compaction grace loop. Confirm the Pi `session_before_compact` trigger contract from installed dependency source or types before finalizing the seam.
