## Summary

Keep open: the PR now has sufficient real local-GGUF CLI proof, but the code still applies the llama.cpp `local.modelPath` identity rule to every `transport: "local"` embedding adapter, which is broader than the public plugin contract supports.

## What This Changes

The PR changes memory-core status identity resolution and tests so indexes configured with `memorySearch.provider: "local"` and only `local.modelPath` compare against that model path.

## Best Possible Solution

Scope the `local.modelPath` status identity rule to the resolved llama.cpp adapter contract, preserve model-based identity for other local transports, and land one canonical fix with regression coverage.

## Reproduction Assessment

Yes. Current main is source-reproducible because status-only identity uses `settings.model` or fallback before provider init while llama.cpp indexes store `local.modelPath`; the PR body also supplies real before/after local-GGUF CLI output.

## Solution Assessment

No. The status mirror is the right layer, but keying the modelPath rule off `transport: "local"` is broader than the current plugin contract; it should be scoped to llama.cpp or an explicit adapter capability.
