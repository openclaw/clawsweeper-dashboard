    fix(doctor): drop inert legacy cron notify when cron.webhook is unset
    
    `openclaw doctor --fix` left top-level `notify: true` cron metadata in place when
    `cron.webhook` was unset, so every run re-detected the same job and re-emitted
    the identical legacy-notify preview and warning — an endless repair loop the
    operator could never clear.
    
    The top-level `notify` flag is inert: cron delivery is driven by `delivery`,
    never by `notify`. When there is no migration target and `cron.webhook` is unset,
    `migrateLegacyNotifyFallback` now drops the dead flag (marking the store changed)
    for both no-target branches — `delivery.mode` none/webhook/absent and existing
    announce/legacy-chat delivery — leaving the existing delivery unchanged. A
    configured-but-invalid `cron.webhook` still surfaces the actionable "not a valid
    HTTP(S) URL" warning so the operator can fix it.
    
    Fixes #44460.
    
    Co-Authored-By: Claud
