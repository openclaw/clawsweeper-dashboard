## Summary

Keep open: current main still leaves the reported no-webhook `notify: true` branch unresolved, and https://github.com/openclaw/openclaw/pull/48742 is an open closing PR that already owns the implementation path.

## What This Changes

Doctor should resolve or clearly report legacy cron jobs with top-level `notify: true` when `cron.webhook` is unset instead of claiming normalization while leaving the field.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/48742 with a focused cron-only fix that resolves inert `notify: true` when no `cron.webhook` exists and adds regression coverage for `delivery.mode: "none"` without unrelated provider changes.

## Reproduction Assessment

Yes. By source inspection, `--fix --yes` auto-approves repair, and a job with `notify: true`, `delivery.mode: "none"`, and no `cron.webhook` reaches the warning branch without deleting `notify`.

## Solution Assessment

Yes for the narrow cron-doctor direction. The safest implementation is a focused `src/commands/doctor-cron.ts` change plus regression tests; the currently paired PR should be split, replaced, or reviewed carefully because it includes unrelated provider-secret normalization changes.
