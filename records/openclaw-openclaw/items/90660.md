Yes. Source inspection shows the gateway skips unconfigured accounts before invoking `startAccount`, while this PR's startup-slot list would count enabled unconfigured accounts and delay configured accounts sorted after them.

## Solution Assessment

No. Promoting the configured default account is the right direction, but the proof-backed companion PR is the better solution because it preserves the configured/token predicate alongside duplicate-token runtime filtering and includes real startup proof.
