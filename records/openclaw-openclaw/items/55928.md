## Summary

Keep open. Current main still has no internal-namespace denial on the HTTP session-history path, while the PR's potential merge commit adds the focused 403 guard and regression coverage; the remaining blockers are real gateway proof, documentation of the new exception, and maintainer acceptance of the compatibility break.

## What This Changes

This PR adds a routing helper and HTTP session-history guard that returns 403 for subagent, cron, and ACP session keys, plus HTTP rejection and WebSocket-preservation tests.

## Best Possible Solution

Merge the HTTP-only internal-session denial after real gateway proof, a short docs update for the reserved namespace exception, and maintainer acceptance that WebSocket history remains available for runtime flows.

## Reproduction Assessment

Yes. Source inspection on current main shows the HTTP history handler resolves and loads the target store without an internal-namespace denial, so seeding an internal session row and calling `/sessions/:key/history` is a high-confidence source reproduction path.

## Solution Assessment

Mostly yes. The runtime guard is narrow and placed before transcript reads while preserving WebSocket runtime flows; the patch should also document the new public HTTP exception and include real gateway proof.
