## Summary

Keep open: current main still serves HTTP session history after auth without rejecting internal subagent, cron, or ACP namespaces, and this PR is a narrow hardening fix, but the external-PR real behavior proof gate is still unmet.

## What This Changes

The PR adds an internal session-key classifier, rejects internal keys in the HTTP session-history handler, and adds regression coverage for HTTP rejection while preserving WebSocket history behavior.

## Best Possible Solution

Land the HTTP-only internal-session guard after redacted real HTTP proof is added and final checks stay green, while keeping WebSocket `chat.history` available for internal runtime callers.

## Reproduction Assessment

Yes. Source inspection on current main shows `/sessions/:key/history` authorizes `chat.history`, canonicalizes bare keys to store keys, and reads transcript history without rejecting subagent, cron, or ACP namespaces.

## Solution Assessment

Yes. The narrow maintainable fix is to reject internal raw or canonical keys after auth/scope authorization and before store/SSE reads, while preserving WebSocket history for internal runtime flows.
