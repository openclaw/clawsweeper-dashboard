## Summary

Keep this PR open. Current main still serves HTTP session history for internal namespaces, and the PR’s raw-plus-canonical HTTP guard is the right narrow fix, but the user-facing gateway access-control change still needs the required changelog entry before landing.

## What This Changes

The PR adds an internal gateway session-key predicate, rejects HTTP `/sessions/:key/history` reads for subagent, cron, and ACP namespaces, and adds regression coverage while preserving WebSocket `chat.history`.

## Best Possible Solution

Add one Unreleased `### Fixes` changelog line, then land the HTTP-only guard with raw/canonical checks and explicit WebSocket preservation.

## Reproduction Assessment

Yes. Seed an internal transcript such as `agent:main:cron:daily`, request `/sessions/cron%3Adaily/history` with `operator.read` on current main, and the source path canonicalizes then reads history instead of returning 403.

## Solution Assessment

Yes, aside from the changelog omission. Checking both the raw request key and resolved canonical key after auth but before store/transcript reads is the narrowest maintainable HTTP fix, while leaving WS `chat.history` available avoids breaking internal runtime flows.
