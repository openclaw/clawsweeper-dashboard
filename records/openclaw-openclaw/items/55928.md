## Best Possible Solution

## Solution Assessment

Yes as a code change. Blocking at the HTTP history boundary after auth and before transcript reads is narrow and avoids the earlier WebSocket over-blocking regression; the remaining blockers are proof and merge-risk acceptance, not a different implementation.
