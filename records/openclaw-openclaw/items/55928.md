## Summary

Keep open: current main still reads HTTP session history for internal namespace keys after auth/scope checks, and this PR's HTTP-only guard looks correct; the remaining blocker is contributor-supplied real behavior proof, not a code finding.

## What This Changes

The branch adds a gateway HTTP session-history guard for internal subagent, cron, and ACP session keys, a shared routing helper, and regression tests that keep the WebSocket boundary explicit.

## Best Possible Solution

Merge the HTTP-only guard after redacted after-fix HTTP proof is added and final merge checks stay green.

## Reproduction Assessment

Yes. Source inspection on current main shows `/sessions/:key/history` authorizes `operator.read`, canonicalizes bare keys such as `cron:daily`, and reads transcript history without rejecting internal namespaces.

## Solution Assessment

Yes. The narrow maintainable fix is to reject internal keys on the HTTP history path after auth/scope authorization and before store reads, while preserving WebSocket `chat.history` for internal runtime callers.
