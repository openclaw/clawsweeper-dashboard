## Best Possible Solution

Keep the directory-first resolver design, but make every Telegram delivery mode that can send messages fail closed on reserved-literal directory misses before session-route/default fallback, then get explicit maintainer acceptance for the plugin SDK contract.

## Reproduction Assessment

Yes, source-reproducible: on PR head, heartbeat `target: "telegram"` with `to: "current"` can hit a reserved-target error in `resolveChannelTarget`, but that non-ok result is ignored before raw Telegram route fallback.

## Solution Assessment

No, not yet. Directory-first reserved-target resolution is the right owner-boundary shape, but heartbeat canonicalization must treat reserved misses as terminal instead of best-effort route fallback.
