## Summary

## What This Changes

Adds a channel-plugin `reservedLiterals` target-resolver contract, configures Telegram to reserve `current`, `self`, `this`, and `me`, rejects those literals in shared outbound resolution, propagates agent delivery errors, and updates plugin docs/tests.

## Best Possible Solution

Land the plugin-owned fail-closed resolver fix only after maintainer API signoff and current-head real gateway or Telegram proof, then let the linked canonical issue close through the merged PR.

## Reproduction Assessment

Yes by source inspection: current main and `v2026.6.8` have no reserved-literal guard before directory lookup or direct fallback, and Telegram normalizes username-shaped bare strings such as `current` as handle-like targets.

## Solution Assessment

Mostly yes: prior maintainer feedback rejected a global guard, and this PR moves the rule to a plugin-owned boundary that covers directory lookup, direct fallback, message-action, gateway send, and agent-delivery paths. The remaining blocker is maintainer acceptance of the new SDK field plus real behavior proof.
