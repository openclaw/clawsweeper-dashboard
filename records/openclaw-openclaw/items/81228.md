## Summary

Keep open for maintainer handling: the PR has the protected `maintainer` label and the diff reverses a documented interactive-only safety policy for orphan transcript archiving. The linked report was already closed as superseded by the open audited cleanup request at https://github.com/openclaw/openclaw/issues/77941.

## What This Changes

The branch removes the interactive confirmation gate from `doctor` orphan transcript archiving so `doctor --fix/--yes` can archive orphan primary transcripts non-interactively, and updates the state-integrity regression test.

## Best Possible Solution

Keep doctor orphan transcript archiving interactive-only unless maintainers explicitly change the safety policy; implement the operator-friendly path as an audited `sessions cleanup` archive/prune mode tracked by https://github.com/openclaw/openclaw/issues/77941.

## Reproduction Assessment

Yes, source-reproducible: create an unreferenced primary transcript and current main will report it while the prompter blocks non-interactive archival because the prompt requires interactive confirmation. This behavior is also covered by current tests and docs.

## Solution Assessment

No. Removing the gate is not the narrowest maintainable fix because it reintroduces the silent session-history rename risk documented in the prior safety report; the safer route is a maintainer-approved audited cleanup flow.
