## Summary

This PR is still useful: current main and the latest release still parse Vertex authorized_user ADC token refresh responses with `response.json()`, while the PR narrowly switches that path to byte-based JSON parsing with gzip handling and adds focused regression coverage. I found no actionable patch defects or security concerns.

## What This Changes

The PR updates the Google plugin's Vertex ADC authorized_user token refresh parser to read response bytes, gunzip compressed OAuth token bodies when needed, and cover the gzip path in Google transport tests.

## Best Possible Solution

Merge the focused Google plugin fix after normal exact-head CI and maintainer review, keeping the gzip handling local to the Vertex ADC token refresh boundary.

## Reproduction Assessment

Yes. Current main's token refresh path uses `response.json()`, and a gzip-compressed token `Response` reproduces the JSON parse failure shape that the current code then turns into a missing `access_token` error.

## Solution Assessment

Yes. The patch changes only the implicated token response parser, preserves the custom `fetchImpl` seam, and adds regression coverage for compressed token responses while existing plain-token tests continue to cover the normal path.
