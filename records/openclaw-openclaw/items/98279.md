## Summary

Keep open: current main still suppresses stale exec approval follow-ups with log lines or an ok summary, and there is no dedicated structured suppression event in the diagnostic/gateway event contract for operator tooling.

## What This Changes

The issue requests an additive structured gateway/diagnostic event when stale exec approval-timeout follow-ups are suppressed instead of only logging the suppression.

## Best Possible Solution

Add a bounded diagnostic/gateway event at the existing suppression points and expose it through the existing structured stability/event stream without changing delivery behavior or adding a sidecar log.

## Reproduction Assessment

Yes, at source level: the stale follow-up suppression branches are visible on current main and only log or return an ok summary. I did not run a live Slack/gateway timeout scenario in this read-only review.

## Solution Assessment

Yes, the requested direction is the narrow maintainable fix: emit through the existing diagnostic/gateway event infrastructure at the suppression point. A sidecar approval-timeouts log would duplicate runtime state and should not be added.
