## Label Justifications

- `P0`: The issue reports permanent deletion of user-authored memory-wiki notes from a routine transient storage outage.
- `impact:data-loss`: The affected path can delete imported wiki pages and their human-notes blocks, losing user-authored content.

## Summary

Keep open: current main still treats a transiently missing `unsafe-local` configured path as an empty active set and then runs the destructive imported-source prune, which can delete imported pages containing user-authored human-notes blocks.

## What This Changes

The issue reports that memory-wiki unsafe-local sync prunes imported pages and loses user-authored human notes when a configured source path is temporarily unavailable.

## Best Possible Solution

Teach unsafe-local sync to track which configured source scopes were successfully scanned, prune only entries belonging to available scopes, and add regression coverage proving transient outages preserve pages and notes while real deletions still prune.

## Reproduction Assessment

Yes. The issue includes a production-function current-main repro, and source inspection confirms the same missing-path to empty-active-set to destructive-prune path remains at the reviewed SHA.

## Solution Assessment

Yes, but the best fix should be per configured source scope rather than a blanket zero-artifact skip, so genuine deletions in successfully scanned paths still prune correctly.
