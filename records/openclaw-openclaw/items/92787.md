## Summary

Keep open: the patch is not merge-ready because a home-directory boundary still allows outside-workspace parent files such as `~/.openclaw/AGENTS.md` and `~/AGENTS.md` to enter prompt context for the documented default workspace, and the supplied proof does not show the actual loaded context-file set after the fix.

## What This Changes

This PR adds a `homedir()` guard to `loadProjectContextFiles()` so ancestor `AGENTS.md`/`CLAUDE.md` discovery stops above the user's home directory.

## Best Possible Solution

Bound project-context discovery to the active workspace or project root, preserve ancestor ordering only inside that boundary, and add proof plus regression coverage showing outside-workspace parent files are excluded.

## Reproduction Assessment

Yes, source-level. For a default workspace under the user's home, the PR guard still allows the loop to visit same-home parents such as `~/.openclaw` and the home directory before those files are injected into the system prompt.

## Solution Assessment

No. A home boundary is only a partial mitigation; the maintainable fix is a workspace or project-root boundary aligned with OpenClaw's workspace contract and Codex's project-root discovery behavior.
