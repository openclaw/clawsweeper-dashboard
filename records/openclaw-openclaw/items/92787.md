## Summary

Keep open; this PR is not merge-ready because a home-directory boundary still permits the linked report's same-home outside-workspace examples, such as `~/.openclaw/AGENTS.md` and `~/AGENTS.md`, to enter project context.

## What This Changes

The PR adds a home-directory boundary check to `loadProjectContextFiles()` so ancestor `AGENTS.md`/`CLAUDE.md` discovery stops before directories above the user's home.

## Best Possible Solution

Bound project-context discovery to the configured workspace or project root, preserve ancestor ordering only inside that boundary, and prove outside-workspace parent files are excluded.

## Reproduction Assessment

Yes, source-level. Current main walks from `cwd` to `/`, and this PR would still walk through home-tree parents, so a workspace under `~/.openclaw/workspace` would still load `~/.openclaw/AGENTS.md` and `~/AGENTS.md`.

## Solution Assessment

No. A home boundary is only a partial mitigation; the narrow maintainable fix is a workspace or project-root boundary aligned with the linked report and OpenClaw workspace context docs.
