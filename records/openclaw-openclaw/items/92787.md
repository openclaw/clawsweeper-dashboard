## Summary

This PR should stay open because the proposed `homedir()` boundary is an incomplete security fix: for the documented default workspace, same-home parent files outside the workspace can still be loaded into prompt context.

## What This Changes

Adds a home-directory boundary check to `loadProjectContextFiles()` so ancestor `AGENTS.md` / `CLAUDE.md` discovery stops above the user's home directory.

## Best Possible Solution

Bound project-context discovery to the configured workspace or project root, keep root-to-cwd ordering only inside that boundary, and add regression plus real loaded-context proof.

## Reproduction Assessment

Yes, source-level. Current main and the latest release walk to filesystem root, and the PR still walks through same-home parents for the default `~/.openclaw/workspace` layout.

## Solution Assessment

No. A home-directory boundary is only a partial mitigation; the maintainable fix is a workspace or project-root boundary aligned with the reported security boundary and Codex project-doc behavior.
