## Summary

Keep open: this PR is not mergeable as-is because `/vc chat mode` only replies that the mode changed without persisting anything, and `/vc list` bypasses the access check used by the existing voice commands.

## What This Changes

The PR adds `/vc list` and `/vc chat` subcommands to the Discord voice command implementation, with `chat` showing or claiming to set the voice mode.

## Best Possible Solution

Restore the documented Discord command split first: `/voice` should reach TTS, `/talkvoice` should manage Talk voice selection, and any new `/vc` subcommand should preserve voice-command authorization and either persist mode changes correctly or stay read-only.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: the PR’s `mode` branch only replies and returns, while current manager/session code reads configured voice mode during join. The authorization gap is also source-reproducible because `ListCommand` returns `manager.status()` details without the existing `ensureVoiceCommandAccess` gate.

## Solution Assessment

No. Adding `/vc list` and a non-persistent `/vc chat mode` path is not the narrow fix for the linked Discord `/voice` TTS routing problem, and the added `/vc` behavior needs access and persistence fixes before it can be considered.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Persist the requested voice mode before confirming it:** `extensions/discord/src/voice/command.ts:349-351`
  - body: The new `/vc chat mode` path validates the option and replies that the mode was set, but it never updates config, the manager, or the active session. After the user rejoins, `DiscordVoiceManager.join()` will still use the existing configured mode, so this confirmation is false.
  - confidence: 0.9
- **[P2] Authorize list before exposing voice sessions:** `extensions/discord/src/voice/command.ts:285-287`
  - body: `/vc list` returns active channel mentions directly from `manager.status()` without the `ensureVoiceCommandAccess` check used by `/vc status` and `/vc leave`. In allowlisted guilds, an unauthorized user could learn active voice session channel IDs by running the new command.
  - confidence: 0.88
