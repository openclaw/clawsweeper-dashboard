## Summary

Keep open, but the current patch should not merge because it broadens exact model allowlist authorization to undeclared provider IDs; current main preserves provider hyphens and already has explicit provider alias seams.

## What This Changes

The PR adds hyphen-stripped provider/model variants to the model allowlist builders and unit tests for that behavior.

## Best Possible Solution

Keep provider IDs exact by default; if a real stripped-provider path exists, fix that path to preserve the configured ID or honor only declared provider/plugin aliases with real setup proof.

## Reproduction Assessment

No high-confidence reproduction exists for the reported current-main mismatch: source inspection found exact provider IDs preserved in allowlist parsing and provider normalization. The PR-introduced authorization broadening is source-reproducible from the diff.

## Solution Assessment

No. Unconditional hyphen stripping is broader than the claimed mismatch; preserving the original configured provider ID or resolving only declared provider aliases is the narrower maintainable direction.
