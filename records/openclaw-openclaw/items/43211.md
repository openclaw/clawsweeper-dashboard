## Summary

Keep open. The PR still broadens exact model allowlist authorization by adding undeclared hyphen-stripped provider IDs, while current main keeps provider IDs exact except for known aliases and the claimed runtime hyphen-stripping path is not proven.

## What This Changes

The PR adds hyphen-stripped provider/model key variants to model allowlist builders and adds unit tests for those variants.

## Best Possible Solution

Keep provider IDs exact by default; if a real stripped-provider path exists, fix that path to preserve the configured ID or honor only declared provider/plugin aliases, then prove it in a real setup.

## Reproduction Assessment

No high-confidence reproduction exists for the reported runtime mismatch: source inspection did not find a current path that strips arbitrary provider hyphens. The blocking authorization broadening is source-reproducible from the PR diff adding stripped variants to both allowlist builders.

## Solution Assessment

No. Unconditional hyphen stripping is broader than the stated mismatch; preserving the configured provider ID or resolving only declared provider/plugin aliases is the narrower maintainable direction.
