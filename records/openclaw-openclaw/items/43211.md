## Summary

Keep open: the PR is a plausible bug-fix attempt, but the patch currently broadens an exact model allowlist into undeclared provider aliases and lacks real behavior proof for the reported hyphen-stripping path.

## What This Changes

The PR adds an allowlist-key variant helper and tests so a configured hyphenated provider/model ref also allows the hyphen-stripped provider form.

## Best Possible Solution

Keep provider IDs exact by default; if a real path strips hyphens, preserve the configured provider ID upstream or honor only explicit provider/plugin aliases with focused coverage.

## Reproduction Assessment

For the PR-introduced defect, yes: the diff source clearly adds undeclared hyphen-stripped variants to all allowlist checks. For the originally claimed current-main permission failure, no real setup path or specific runtime path that strips arbitrary hyphens is provided.

## Solution Assessment

No. Unconditional provider hyphen folding is broader than the reported mismatch; explicit aliases or preserving the original provider id is the narrower maintainable solution.
