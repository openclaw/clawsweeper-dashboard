## Summary

Keep this PR open but not merge-ready: the branch tries to fix an unproven provider-id mismatch by broadening exact model allowlist entries into implicit provider aliases, which is risky for provider routing and credentials.

## What This Changes

The PR adds a helper and tests so a hyphenated allowlist provider key also allows the same provider id with hyphens removed.

## Best Possible Solution

Keep exact provider ids as the default authorization boundary; if a real mismatch exists, preserve the configured provider id upstream or honor only declared provider/plugin aliases with denial coverage for neighboring ids.
