## Summary

Keep open: the PR is a focused external proposal, but the current diff is not safe to merge because it turns exact allowlist entries into permission for undeclared provider IDs. Current main keeps allowlist keys exact and already has explicit provider alias hooks; the reported runtime path that strips arbitrary hyphens still lacks real proof.

## What This Changes

The PR adds hyphen-stripped provider/model key variants to model allowlist builders and adds unit tests for those variants.

## Best Possible Solution

Keep provider IDs exact by default; if a real stripped-provider path exists, fix that path to preserve the configured provider ID or resolve only declared provider/plugin aliases, then prove the real setup.

## Reproduction Assessment

Unclear for the reported bug: I did not find a current source path that strips arbitrary hyphens from provider IDs. The PR-introduced authorization broadening is source-reproducible from the diff adding stripped variants to both allowlist builders.

## Solution Assessment

No. Unconditional hyphen stripping is broader than the stated mismatch; declared aliases or preserving the original provider ID is the narrower maintainable direction.
