## Summary

Keep open: the proposed fix changes exact model allowlist authorization by treating distinct provider IDs as equivalent, and the PR does not identify a current runtime path that strips provider hyphens or provide real runtime proof.

## What This Changes

The PR adds hyphen-stripped provider/model key variants to model allowlist construction and unit coverage for those variants.

## Best Possible Solution

Keep provider IDs exact by default; if a hyphen-stripped spelling is valid, resolve it through declared provider/plugin alias metadata or fix the resolver that loses the configured provider ID.

## Reproduction Assessment

Unclear for the original report because no current runtime path that strips provider hyphens is identified. The PR-introduced authorization broadening is source-reproducible from the diff adding stripped variants to `allowedKeys`.

## Solution Assessment

No. Unconditional hyphen folding is broader than the reported mismatch and changes an authorization boundary; declared aliases or preserving the original provider ID would be narrower.

## Review Findings
