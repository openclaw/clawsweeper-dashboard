## Summary

Keep open: the diagnostic propagation is useful, but this patch is not merge-ready because the new auto-retry can replay side-effectful turns, the final failure copy bypasses normal success-path bookkeeping, and the proof is still test-only.

## Best Possible Solution

Keep the metadata propagation and specific copy, remove or gate retry on replay-safe/no-side-effect evidence, and route the failure payload through the normal completed-run success path with redacted real-behavior proof.
