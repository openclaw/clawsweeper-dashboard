## Summary

Keep open: the diagnostic goal is valid, but this PR is not merge-ready because the new auto-retry can repeat side-effectful tool turns, the final-message path bypasses completed-run bookkeeping, and the external PR still lacks real behavior proof.

## What This Changes

The PR propagates embedded-run trajectory terminal errors into auto-reply metadata, retries one non-deliverable terminal turn, and adds a dedicated user-facing failure message.

## Best Possible Solution

Keep the metadata propagation and specific copy, remove or strictly gate automatic retry to provably side-effect-free attempts, and inject the failure payload through the normal completed-run path with focused tests and real proof.
