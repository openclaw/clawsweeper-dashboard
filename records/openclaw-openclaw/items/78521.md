## Summary

Keep open: the shared transport-boundary hardening is a valid security direction, but this patch is not mergeable because its raw-marker short-circuit lets attacker-controlled toolResult text skip wrapping and sanitization, and contributor-owned real payload proof is still missing.

## What This Changes

## Best Possible Solution

Keep the shared-boundary approach, but always treat toolResult text as untrusted: run it through wrapExternalContent, sanitize raw marker strings as content, add injected-marker regression coverage, and require redacted real payload proof before merge.
