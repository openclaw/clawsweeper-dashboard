## Summary

Keep this PR open, but it should not merge as-is: the transport-boundary hardening is a useful direction, yet the substring guard trusts attacker-controlled marker text and bypasses the sanitizer, and the external contributor has not provided real behavior proof.

## What This Changes

The PR wraps text `toolResult` blocks in `transformTransportMessages` with `wrapExternalContent` and updates focused tests for wrapped and already-wrapped tool results.

## Best Possible Solution

Keep the shared transport-boundary hardening, remove trust in raw marker substrings, add spoofed-marker regression coverage, and require redacted real transport/payload proof before merge.

## Reproduction Assessment

Yes, for the PR-introduced bypass: source inspection shows the guard skips wrapping on raw marker substrings, and a reviewer harness against PR head reproduced marker-injected tool output reaching the payload without a security notice.

## Solution Assessment

No. Wrapping at the shared transport boundary is reasonable, but raw tool-result text should always go through `wrapExternalContent` or a strict internally generated wrapper validator before skipping sanitization.
