## Summary

Keep open: the PR is a plausible security hardening, but the new marker-substring skip can let crafted raw tool output bypass wrapping, and the external PR still lacks after-fix real behavior proof.

## What This Changes

The PR adds external-content wrapping for `toolResult` text in `transformTransportMessages` and updates regression tests for wrapped and already-wrapped tool results.

## Best Possible Solution

Use the existing wrapper/sanitizer without trusting raw marker substrings, add coverage for marker-spoofed tool output, and attach real transport-run proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: pass a `toolResult` text containing `EXTERNAL_UNTRUSTED_CONTENT` through the PR version and the new precheck returns the raw text instead of wrapping it.

## Solution Assessment

No. The direction is reasonable, but trusting attacker-controlled marker substrings is not the safest implementation; the patch should always sanitize/wrap raw tool output or validate a full internally generated wrapper shape before skipping.
