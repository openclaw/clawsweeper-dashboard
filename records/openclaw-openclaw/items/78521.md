## Summary

Keep open. The hardening direction is useful, but the PR currently introduces a high-confidence bypass by treating raw boundary-marker substrings inside untrusted tool output as already wrapped, and the PR body still lacks contributor-owned real behavior proof.

## What This Changes

The PR adds external-content wrapping for text `toolResult` blocks in `transformTransportMessages` and adjusts focused transport-message tests.

## Best Possible Solution

Keep the transport-boundary hardening, remove the raw-marker short-circuit or replace it with trustworthy full-wrapper validation, add spoofed-marker regression coverage, and require redacted real payload proof.

## Reproduction Assessment

Yes, for the PR-introduced bypass. Source inspection shows the substring guard returns raw tool-result text when marker names appear, and a reviewer harness against PR head reproduced `changed=false` and `securityNotice=false`.

## Solution Assessment

No. Wrapping at the shared transport boundary is reasonable, but this implementation must not skip wrapping based on attacker-controlled marker substrings; it should sanitize raw tool output or validate a full internally generated wrapper before preserving it.
