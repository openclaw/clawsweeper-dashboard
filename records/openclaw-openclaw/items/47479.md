## Summary

Keep open: the Docker `.env` bug is still relevant, but this PR edits the obsolete root setup script while current main uses `scripts/docker/setup.sh`; the active script still lacks the general `.env` defaults loader and quote-preserving writes.

## What This Changes

The PR adds a manual `.env` loader, quote-preserving environment writes, `OPENCLAW_INSTALL_BROWSER` build-arg passthrough, and a `docker-compose.override.yml` ignore entry to the old root Docker setup script.

## Best Possible Solution

Retarget the dotenv defaults loader and quote-preserving managed writes to `scripts/docker/setup.sh`, update the Docker setup e2e coverage, and leave already-implemented browser build-arg and ignore changes as-is on main.
