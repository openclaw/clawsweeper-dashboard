## Summary

Keep open: the underlying Docker `.env` handling problem is still present in the active setup path, but this PR is not mergeable as-is because it patches the removed root `docker-setup.sh` instead of `scripts/docker/setup.sh` and still lacks real behavior proof.

## What This Changes

The PR adds a manual `.env` loader, quote-preserving `.env` writes, `docker-compose.override.yml` ignores, and an `OPENCLAW_INSTALL_BROWSER` build arg to the former root Docker setup script.

## Best Possible Solution

Retarget the dotenv defaults loader and quote-preserving `upsert_env` writes to `scripts/docker/setup.sh`, preserve caller environment precedence, and update focused Docker setup e2e coverage.
