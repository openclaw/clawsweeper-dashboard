## Summary

Keep this open: the underlying Docker setup dotenv bug still appears valid on current main, but the branch is not mergeable as written because it patches the removed root setup script instead of the supported Docker setup path.

## What This Changes

The PR changes `.gitignore` and root `docker-setup.sh` to load `.env` defaults, preserve quoted whitespace values when rewriting `.env`, and pass `OPENCLAW_INSTALL_BROWSER` as a Docker build argument.

## Best Possible Solution

Retarget the dotenv defaults loader and quote-preserving `upsert_env` writes to `scripts/docker/setup.sh`, preserve caller environment precedence, update focused Docker setup e2e coverage, and coordinate with https://github.com/openclaw/openclaw/pull/62431 if it lands first.

## Reproduction Assessment

Yes, source-reproducible. Current main still writes whitespace package values bare through `upsert_env()` and has no general `.env` defaults loader in the supported setup script; I did not execute Docker setup because this review is read-only.

## Solution Assessment

No. The parser/writer direction is plausible, but this PR applies it to the removed root entrypoint instead of the supported `scripts/docker/setup.sh` flow, so it is not the narrow maintainable fix as written.
