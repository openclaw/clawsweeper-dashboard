## Summary

Keep open. Current main still matches the reported Control UI v2 composer usability regression: the visible bordered composer contains textarea and toolbar chrome with no focus delegation from the wrapper, and the controls remain small icon-only buttons.

## What This Changes

The issue asks to restore usable Control UI v2 chat composer behavior by making the visible chat line fully clickable and improving cramped icon-only controls.

## Best Possible Solution

Fix the Control UI composer so all non-control visible entry chrome focuses the textarea, the apparent input area matches the real typing target, and composer controls have practical hit targets and discoverability while preserving Send, Queue, Stop, attachments, Talk, paste, auto-height, slash-command, and Shift+Enter behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path is to open the Control UI composer and click inside the bordered lower composer area outside the textarea and buttons; current main has no wrapper click handler that focuses the textarea from that chrome.

## Solution Assessment

Yes for a narrow repair. Focus delegation plus targeted hit-target and discoverability improvements are the maintainable fix; reverting wholesale to the v1 composer layout would need separate product judgment.
