## Summary

Keep open. Current main still has the Control UI composer regression: the visible bordered input wrapper includes non-textarea chrome that does not focus the textarea, and desktop composer controls remain small icon-only buttons; related v2 UI fixes do not cover this composer-specific issue.

## What This Changes

The issue asks to restore v1-level chat composer usability in Control UI v2, including a usable click target for the visible input area and more discoverable buttons.

## Best Possible Solution

Repair only the composer interaction: focus the textarea when non-control composer chrome is clicked, and improve desktop control hit targets/discoverability without reverting the whole composer to the v1 layout.

## Reproduction Assessment

Yes, at source level. Open Control UI and click the visible `.agent-chat__input` chrome in the lower toolbar area outside buttons; current main has no wrapper focus delegation and the textarea is only the upper child.

## Solution Assessment

Yes for a narrow repair. Focus delegation plus targeted desktop hit-target/discoverability improvements are the maintainable fix; a full v1 layout rollback is broader product work.
