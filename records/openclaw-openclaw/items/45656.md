## Summary

Keep open. Current main and the latest release still render the Control UI composer as one visible bordered input with a separate textarea and toolbar, but there is no wrapper focus delegation and desktop composer controls remain compact icon-only buttons.

## What This Changes

The issue requests restoring v2 Control UI chat composer usability so the visible chat line reliably accepts text focus and the composer controls are easier to discover and click.

## Best Possible Solution

Repair the composer interaction by focusing the textarea from non-control composer chrome and improving desktop control hit targets/discoverability without reverting the whole v2 composer layout.

## Reproduction Assessment

Yes, at source level. Open the Control UI and click the visible `.agent-chat__input` chrome in the lower toolbar area outside actual buttons; current main has no wrapper focus delegation and the textarea is only the upper child.

## Solution Assessment

Yes for a narrow repair. Focus delegation plus targeted desktop hit-target/discoverability improvements are the maintainable fix; a full v1 layout rollback would be broader than this bug repair.
