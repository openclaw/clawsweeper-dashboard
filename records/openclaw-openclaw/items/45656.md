## Summary

Keep open. Current main still matches the reported Control UI v2 composer usability regression: the visible bordered composer includes non-textarea chrome that does not focus the textarea, and the composer controls remain small icon-only buttons.

## What This Changes

The issue asks for the Control UI v2 chat composer to regain usable click targets, space efficiency, and clearer controls comparable to the v1 chat line.

## Best Possible Solution

Fix the composer so visible non-control input chrome focuses the textarea, the apparent typing target matches the actual target, and the controls have practical hit targets and discoverability without regressing Send, Queue, Stop, attachments, Talk, paste, auto-height, slash-command, or Shift+Enter behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path is to open the Control UI composer and click inside the bordered lower composer area outside the textarea/buttons; current main has no wrapper focus delegation for that chrome.

## Solution Assessment

Yes for a narrow repair. Focus delegation plus targeted hit-target/discoverability improvements are the maintainable path; reverting the whole composer to v1 layout is broader product work.
