## Summary

Keep open. Current main still matches the reported Control UI v2 composer usability regression: the visible bordered composer includes non-textarea chrome that does not focus the textarea, and the controls remain small icon-only buttons.

## What This Changes

The issue asks for the v2 Control UI chat composer to restore a usable click target, layout density, and clear button affordances comparable to the v1 chat line.

## Best Possible Solution

Repair the Control UI composer so every visible non-control part of the input chrome focuses the textarea, and adjust button hit targets/discoverability without regressing Send, Queue, Stop, attachments, Talk, paste, slash commands, auto-height, or Shift+Enter behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path is to open the Control UI composer and click inside the bordered `.agent-chat__input` area outside the textarea and buttons; current main has no wrapper focus delegation for that chrome.

## Solution Assessment

Yes for a narrow repair. Focus delegation plus targeted hit-target and discoverability improvements are the maintainable path; reverting the whole composer to the v1 layout is broader product work.
