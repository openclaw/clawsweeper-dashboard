## Summary

Keep open. Current main still matches the reported Control UI v2 composer usability regression: the bordered composer contains non-textarea chrome without focus delegation, and the composer controls remain compact icon-only buttons.

## What This Changes

The issue requests restoring usable Control UI v2 chat composer behavior, especially click targeting in the visible chat line and more discoverable, appropriately sized composer controls.

## Best Possible Solution

Repair the Control UI composer so visible non-control input chrome focuses the textarea, and improve composer control hit targets/discoverability while preserving send, queue, stop, attachments, Talk, slash commands, paste, auto-height, and Shift+Enter behavior.

## Reproduction Assessment

Yes. A high-confidence source-level path is to open the Control UI chat composer and click inside the bordered `.agent-chat__input` area outside the textarea and buttons; current main has no wrapper focus delegation for that chrome.

## Solution Assessment

Yes for a narrow repair. Focus delegation plus targeted hit-target and discoverability improvements are the maintainable fix; reverting the whole composer to the v1 layout is broader product work.
