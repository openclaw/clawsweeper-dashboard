## Summary

Keep open. Current main still matches the reported Control UI v2 composer usability regression: the visible bordered composer has chrome outside the textarea with no focus delegation, and the composer controls remain small icon-only buttons.

## What This Changes

The issue asks for the Control UI v2 chat composer to regain v1-level usability for click target, layout density, and button discoverability.

## Best Possible Solution

Repair the Control UI composer so the visible non-control input chrome focuses the textarea, and improve button hit targets/discoverability without regressing send, queue, stop, attachments, Talk, slash commands, paste, auto-height, or Shift+Enter behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path is to open the Control UI chat composer and click inside the bordered `.agent-chat__input` area outside the textarea and buttons; current main has no wrapper focus delegation for that chrome.

## Solution Assessment

Yes for a narrow repair. Focus delegation plus targeted hit-target and discoverability improvements are the maintainable fix; reverting the whole composer to the v1 layout is broader product work.
