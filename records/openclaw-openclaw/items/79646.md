## Summary

Keep open: current main still closes the blockquote style after the inner paragraph separator is appended, and Telegram renders that style directly as `<blockquote>` HTML, so the reported extra blank line remains source-reproducible.

## What This Changes

The issue requests that blockquote style spans exclude the trailing paragraph separator so Telegram HTML does not place a blank line inside the `<blockquote>` element.

## Best Possible Solution

Fix the shared Markdown IR span boundary so terminal paragraph separators are outside blockquote styles, then add focused IR and Telegram HTML regressions while preserving existing spacing behavior.

## Reproduction Assessment

Yes: this is source-reproducible on current main with `> `gpt`\n\nbody`; `paragraph_close` appends `\n\n` while the blockquote style is still open, and Telegram renders that style span as `<blockquote>`. I did not run tests because this review was required to keep the checkout read-only.

## Solution Assessment

Yes: the maintainable fix belongs in the shared Markdown IR/block style handling rather than a Telegram-only HTML post-process, with tests proving the blockquote span ends before the separator.
