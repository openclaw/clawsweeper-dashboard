The remaining work is a real Discord session-context feature that needs maintainer policy on defaults, visibility filtering, and config shape before implementation.

## What This Changes

The issue requests automatic injection or metadata exposure of Discord parent-channel pinned messages when a thread-bound session starts.

## Best Possible Solution

Choose the Discord parent-pin inheritance policy first, then implement it in the Discord plugin with existing read-target and context-visibility gates, focused tests, and docs for default versus opt-in behavior.

Unclear until maintainers choose the product policy. Automatic injection is plausible, but the safest maintainable solution depends on default behavior, opt-in config, and supplemental-context filtering decisions.
