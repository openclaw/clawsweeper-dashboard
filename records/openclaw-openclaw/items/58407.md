The remaining work is a real feature request that needs maintainer policy on default behavior, context visibility, and possible config surface before implementation.

## What This Changes

Request to automatically expose or inject parent Discord channel pinned messages into Discord thread-bound session context, optionally through metadata or a config flag.

## Best Possible Solution

Implement this in the Discord plugin only after maintainer policy direction, using existing context-visibility/read-target gates and focused Discord tests/docs rather than adding a broad core channel API.

## Reproduction Assessment

Yes for source-level behavior, but not as a failing bug repro. Current main clearly carries Discord thread parent metadata and supports manual pin reads, while the inbound thread context path does not fetch or inject parent-channel pin content.

## Solution Assessment

No automated implementation path is the best fix yet. Automatic parent-pin injection is plausible, but the maintainable solution depends on maintainer decisions about default behavior, opt-in config, and untrusted supplemental-context filtering.
