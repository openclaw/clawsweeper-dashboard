Keep open: current main still has Discord thread starter context, parent model/transcript inheritance, and manual pin listing, but it does not fetch or inject parent-channel pins into thread sessions. ## What This Changes

Request automatic inheritance or injection of parent-channel pinned Discord messages when a Discord thread-bound session starts.

## Best Possible Solution

If accepted, implement this in the Discord plugin as opt-in, bounded parent-pin context with explicit untrusted-context handling, schema/docs updates, focused tests, and redacted real Discord proof.
