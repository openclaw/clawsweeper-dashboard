## Summary

Keep open: current main still supports Discord thread starter context, parent transcript/model inheritance, and manual pin listing, but it does not fetch or inject parent-channel pins for a thread session. The remaining work is a feature/product decision because parent pins would add untrusted channel content to session context and likely need an opt-in config plus size and visibility rules.

## What This Changes

Request that Discord thread sessions inherit the parent channel's pinned messages as system context, metadata, or an opt-in config-driven context source.

## Best Possible Solution

If accepted, implement this inside the Discord plugin as an opt-in, bounded, untrusted parent-pin context path with focused thread-session tests and Discord docs.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main does not fetch or inject parent-channel pins during Discord thread inbound context construction.

## Solution Assessment

Unclear. The request is plausible, but system context versus metadata, default behavior, opt-in config, size limits, and trust handling should be decided before code changes.
