## Summary

Keep open. Current main resolves Discord thread parent metadata and supports manual pin listing, but it does not fetch or inject parent-channel pinned messages into Discord thread turns. The exact request also has no open duplicate or closing PR found in the current issue search.

## What This Changes

Request automatic, configurable inheritance of a Discord thread's parent-channel pinned messages into the thread session context.

## Best Possible Solution

Add an opt-in Discord plugin feature that fetches a bounded set of parent-channel pins for thread messages with a known parent id, wraps them as untrusted supplemental context or explicit metadata, and documents the behavior without enabling parent transcript inheritance by default.

## Reproduction Assessment

Yes. Static inspection of current main is enough to reproduce the missing behavior: the Discord thread context builder never calls the existing pin-list helper or carries a pin field, while preflight already knows the parent channel id.

## Solution Assessment

Yes, if implemented as a narrow opt-in Discord plugin feature using untrusted context. The reporter's system-context option should be avoided because Discord pins are external user content, not trusted system authority.
