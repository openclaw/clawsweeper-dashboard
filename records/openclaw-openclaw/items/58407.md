## Summary

Keep open: current main and the latest release still expose Discord thread parent metadata and a manual `listPins` action, but neither automatically fetches nor injects parent-channel pinned messages into Discord thread session context. The remaining work is a real Discord context feature, but it needs maintainer policy on defaults, visibility filtering, and config shape before implementation.

## What This Changes

Request automatic Discord thread sessions to inherit parent-channel pinned-message context or metadata, optionally controlled by configuration.

## Best Possible Solution

Choose the Discord parent-pin inheritance policy first, then implement it in the Discord plugin using the existing pin read helper, read-target and context-visibility gates, focused tests, and docs for the selected default or opt-in behavior.

## Reproduction Assessment

Yes, at source level: current main carries Discord thread parent metadata and supports manual pin reads, while the inbound thread context path does not fetch or inject parent-channel pin content. I did not run a live Discord gateway repro because this cleanup review is read-only.

## Solution Assessment

Unclear until maintainers choose the product policy. Automatic injection is plausible, but the safest maintainable solution depends on default behavior, opt-in configuration, and supplemental-context filtering decisions.
