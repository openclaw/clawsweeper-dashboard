The branch adds Telegram turn-local media tracking plus a pure helper and tests to filter final-reply media URLs already delivered through block replies.

## Best Possible Solution

Ship a mediaUrl-aware dedupe fix with regression coverage, preferably at the shared reply-media finalization boundary or with Telegram integration coverage, plus a changelog entry and real non-streaming Telegram proof.

## Reproduction Assessment

No live current-main Telegram reproduction was established in this read-only review. The linked issue comments and current source show a credible source-reproducible path: non-streaming block media can be sent directly before the final MEDIA-derived payload is delivered.

## Solution Assessment

No. The requested direction is narrow, but this implementation needs to clear legacy `mediaUrl` when filtering media and needs real Telegram proof before it is the best maintainable fix.
