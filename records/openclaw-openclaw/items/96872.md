## What This Changes

The PR updates Codex plugin thread config to force-refresh deferred missing app inventory, enable OpenClaw-configured accessible plugin apps per thread, and add docs plus regression tests.

## Best Possible Solution

## Reproduction Assessment

Yes at source level: current main suppresses the initial app inventory refresh when activation is required and refreshes only after install attempts, so an already-active activation transition can leave inventory missing. ## Solution Assessment

Yes for the delayed-inventory bug; the force-refresh and reread happen in the existing owner layer that builds thread config. The per-thread enablement override is technically supported by Codex but needs maintainer acceptance because it changes how OpenClaw treats Codex disabled app state.
