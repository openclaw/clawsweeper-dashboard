- **[P1] Match orphan detection to persisted session files:** `src/infra/session-health-file-discovery.ts:115-117`
  - body: `discoverOrphanTranscripts` assumes active transcripts are named `${sessionId}.jsonl`, but current sessions can persist topic/custom `sessionFile` names. That can classify live transcripts as orphaned and archive them, so build the referenced set from resolved session-file paths instead.
  - confidence: 0.93
- **[P1] Use archive timestamps for stale archive checks:** `src/infra/session-health-file-discovery.ts:141-165`
  - body: Stale `.deleted` and `.reset` checks use file `mtimeMs`, while the executor archives by `fs.rename()` with a timestamp suffix. A newly archived old transcript can be treated as stale and deleted in the same Tier 1 pass; parse the archive timestamp suffix instead.
  - confidence: 0.92
- **[P1] Honor reset archive retention for reset purges:** `src/infra/session-health-remediation-executor.ts:526-527`
  - body: Reset archive deletion uses `snapshot.maintenance.pruneAfterMs`, but current config has a dedicated `resetArchiveRetention` value that can be disabled. This can remove reset recovery artifacts earlier than configured.
  - confidence: 0.9
- **[P2] Keep orphaned temp files out of critical reconciliation:** `src/infra/session-health-collector.ts:337-338`
  - body: The collector sets `reconciliationRecommended` for any orphaned temp file, and the derived index health treats that flag as critical. A single crashed `.tmp` artifact should not force an overall critical Mission Control alert.
  - confidence: 0.87
