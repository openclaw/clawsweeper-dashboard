  - body: `discoverOrphanTranscripts` assumes every referenced transcript is named `${sessionId}.jsonl`, but current session rows can carry `sessionFile` and topic/custom transcript filenames.   - body: The stale deleted/reset scans use substring classification plus filesystem mtime instead of `parseSessionArchiveTimestamp`. Because orphan archival is an `fs.rename`, an old transcript can be archived and then immediately purged in the same Tier 1 run, and primary files with `.deleted.` in their name can be misclassified; use the existing artifact helpers and archive timestamp as the retention clock.
  - confidence: 0.93
- **[P1] Honor reset archive retention for reset purges:** `src/infra/session-health-remediation-executor.ts:526-527`
  - body: Reset cleanup uses `snapshot.maintenance.pruneAfterMs`, not `resetArchiveRetention`, so configs that set a longer reset archive window or `false` to disable reset cleanup can still have `.reset` recovery artifacts removed on the shorter default window.
  - confidence: 0.95
- **[P2] Keep orphan temp drift out of critical reconciliation:** `src/infra/session-health-collector.ts:337-338`
  - body: The collector sets `reconciliationRecommended` when any orphaned temp file exists, and the derived surface promotes that flag to critical. A single crashed atomic-write `.tmp` should follow the warning/Tier 0 cleanup path instead of creating a false critical index-health alert.
