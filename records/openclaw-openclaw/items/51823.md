## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Resolve orphans through sessionFile before archiving:** `src/infra/session-health-file-discovery.ts:117`
  - body: `discoverOrphanTranscripts` strips `.jsonl` and compares only against indexed `sessionId` values, but current session entries can persist a different `sessionFile`. A live transcript with a custom filename is treated as orphaned and can be renamed to `.deleted.*`, making the active session look missing.
  - confidence: 0.92
- **[P1] Use archive timestamps for stale archive checks:** `src/infra/session-health-file-discovery.ts:141`
  - body: The stale archive scanners use file `mtime`, while the executor archives orphans with `fs.rename`, which preserves the old transcript mtime. Running Tier 1 can therefore archive an old orphan and immediately purge it in the same pass, breaking the archive-first safety model.
  - confidence: 0.91
- **[P1] Honor resetArchiveRetention for reset cleanup:** `src/infra/session-health-remediation-executor.ts:527`
  - body: `executeArchiveStaleResetTranscripts` uses `snapshot.maintenance.pruneAfterMs`, but current session maintenance has a separate reset archive retention setting, including `false` to disable reset cleanup. This can delete reset recovery archives earlier than configured.
  - confidence: 0.93
- **[P1] Do not make orphan temp files reconciliation-critical:** `src/infra/session-health-collector.ts:338`
  - body: The collector sets `reconciliationRecommended` when any orphaned temp file exists, and `deriveIndexHealth` turns that into a critical index state. A single crashed `.tmp` file bypasses the warning path and can produce false critical session-health alerts.
  - confidence: 0.86
- **[P2] Re-evaluate stale status when serving cached health:** `src/infra/session-health-runner.ts:61`
  - body: The runner computes and caches `overallLevel` only when collection runs, so if the collector stalls the cached surface can keep reporting healthy instead of transitioning to `stale_data`. Recompute staleness from `measuredAt` when the health RPC reads the cache or preserve a read-time freshness wrapper.
  - confidence: 0.84
