## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Honor explicit sessionFile paths when finding orphans:** `src/infra/session-health-file-discovery.ts:115-117`
  - body: `discoverOrphanTranscripts` treats the active transcript as `${sessionId}.jsonl`, but OpenClaw session entries can persist a different `sessionFile` and current maintenance code resolves that path before deciding what is referenced. A live session with an explicit transcript filename will be classified as an orphan here, so executing the archive action can rename the active transcript and make the session look missing.
  - confidence: 0.93
- **[P1] Avoid purging freshly archived orphan transcripts:** `src/infra/session-health-remediation-executor.ts:414-416`
  - body: When `--execute-tier 1` runs, the plan order archives orphan transcripts before purging stale `.deleted` archives. This `fs.rename` preserves the original file mtime, and stale discovery filters by mtime, so an old orphan can be renamed to `.deleted.<now>` and then deleted in the same execution, breaking the archive-first safety model.
  - confidence: 0.91
- **[P1] Use reset archive retention for reset purges:** `src/infra/session-health-remediation-executor.ts:525-527`
  - body: Reset archive deletion uses `snapshot.maintenance.pruneAfterMs`, but current configuration has a dedicated `resetArchiveRetention` policy that can be longer than pruneAfter or `false` to disable cleanup. This can remove `.reset` recovery archives earlier than configured, so the collector/plan/executor need to carry and honor `resetArchiveRetentionMs` for reset files.
  - confidence: 0.92
- **[P2] Do not mark orphaned temp files as reconciliation-critical:** `src/infra/session-health-derive.ts:137-139`
  - body: The collector sets `reconciliationRecommended` when any orphaned temp file exists, and `deriveIndexHealth` turns that flag into a critical index-health indicator. That means a single crashed-write `.tmp` artifact bypasses the intended orphan warning path and raises overall health to critical without real index drift.
  - confidence: 0.86
- **[P2] Re-evaluate stale health state when serving health:** `src/gateway/server/health-state.ts:85-87`
  - body: The runner derives `overallLevel` once and `health-state` later serves the cached derived surface directly. If the collector stalls after a healthy snapshot, the cached `overallLevel` stays healthy instead of transitioning to `stale_data`, so operators can miss the collector-stopped signal the derivation code is meant to provide.
  - confidence: 0.84
- **[P2] Add the required changelog entry:** `docs/cli/sessions.md:73`
  - body: This PR adds user-facing CLI flags, cleanup output, and Mission Control health visibility, but the PR file list does not include `CHANGELOG.md`. Repo policy requires user-facing `feat` changes to have a changelog entry before merge.
  - confidence: 0.89
