  - body: Stale `.deleted` and `.reset` checks use file `mtimeMs`, while the executor archives by `fs.rename()` with a timestamp suffix. A newly archived old transcript can be treated as stale and deleted in the same Tier 1 pass; parse the archive timestamp suffix instead.
  - confidence: 0.92

  - confidence: 0.9

  - body: The collector marks any orphaned temp file as `reconciliationRecommended`, and the derived index health treats that flag as critical. A single crashed `.tmp` artifact should not force an overall critical Mission Control alert.
  - confidence: 0.87

  - body: The runner caches `SessionHealthSurface` at collection time and the health RPC returns that cached surface unchanged. If the collector stalls after a healthy run, the UI will keep receiving a healthy `overallLevel` instead of aging into `stale_data`.
