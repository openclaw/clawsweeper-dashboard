- **[P1] Match orphan checks to resolved session files:** `src/infra/session-health-file-discovery.ts:115-117`
  - body: `discoverOrphanTranscripts` assumes active transcripts are named `${sessionId}.jsonl`, but current session entries can persist `sessionFile` paths and current cleanup resolves those paths before deciding what is referenced. This can classify a live transcript as orphaned and archive it, so build the referenced set from resolved canonical session-file paths instead.
  - confidence: 0.93
- **[P1] Use archive timestamps for stale archive checks:** `src/infra/session-health-file-discovery.ts:141-165`
  - body: Stale `.deleted` and `.reset` checks use file mtime, while the executor archives by `fs.rename()` with a fresh timestamp suffix. Because rename preserves the original transcript mtime, `--execute-tier 1` can archive an old orphan and then delete it in the same pass; parse the archive timestamp suffix instead.
  - confidence: 0.92
- **[P1] Honor reset archive retention for reset purges:** `src/infra/session-health-remediation-executor.ts:526-527`
  - body: Reset archive deletion uses `snapshot.maintenance.pruneAfterMs`, but current config has a dedicated `resetArchiveRetention` value that can also disable reset cleanup. This can remove reset recovery artifacts earlier than configured, so carry and use the resolved reset retention value.
  - confidence: 0.9
- **[P2] Keep orphan temp files out of critical reconciliation:** `src/infra/session-health-collector.ts:337-338`
  - body: The collector sets `reconciliationRecommended` for any orphaned temp file, and `deriveIndexHealth` treats that flag as critical. A single crashed `.tmp` artifact should follow the warning path already intended for orphan temp artifacts, not force an overall critical Mission Control alert.
