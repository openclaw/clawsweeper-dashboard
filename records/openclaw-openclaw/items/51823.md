  - confidence: 0.9

  - body: The stale archive helpers compare retention against file mtime. Since orphan archival is a rename and can preserve the old transcript mtime, `--execute-tier 1` can archive an old orphan and then delete the newly archived file in the same pass instead of retaining it from the archive timestamp.
  - confidence: 0.9
- **[P2] Honor resetArchiveRetention for reset purge:** `src/infra/session-health-remediation-executor.ts:526-527`
  - body: Reset archive deletion uses `snapshot.maintenance.pruneAfterMs`, but current config has a separate `resetArchiveRetention` setting, including `false` to disable cleanup. Carry the resolved reset retention value into the snapshot/executor before deleting `.reset` archives.
  - confidence: 0.93
- **[P2] Keep orphan temp files out of critical reconciliation:** `src/infra/session-health-collector.ts:337-338`
  - body: The collector sets `reconciliationRecommended` whenever any orphaned temp file exists, and `deriveIndexHealth` makes that critical. A single crashed-write `.tmp` should stay on the warning/Tier 0 cleanup path instead of escalating the whole health surface to critical.
  - confidence: 0.86

  - body: The executor can permanently remove `.reset` recovery archives using the generic prune window even when `resetArchiveRetention` is longer or disabled, undermining the configured retention contract.

  - confidence: 0.88

  - body: The orphan discovery path ignores persisted `sessionFile` values and can rename valid custom transcript files as deleted archives, creating a local data-integrity risk.
