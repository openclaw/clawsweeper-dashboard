  - body: The PR starts from an older cleanup command implementation and bypasses current main's gateway cleanup / `runSessionsCleanup` path. Merging it as-is would drop newer cleanup behavior and summary fields; move the remediation hooks onto the current cleanup-service flow instead.
  - confidence: 0.88

  - body: `discoverOrphanTranscripts` assumes every active filename is `${sessionId}.jsonl`, but current session entries can persist a different `sessionFile`. Those valid transcripts can be classified as orphaned and archived, making sessions look missing.
  - confidence: 0.9
- **[P2] Use archive timestamps for stale archive eligibility:** `src/infra/session-health-file-discovery.ts:141`
  - body: The stale archive helpers compare retention against file mtime. Since orphan archival is a rename that preserves the old transcript mtime, `--execute-tier 1` can archive an old orphan and then delete the newly archived file in the same pass.
  - confidence: 0.9
- **[P2] Honor resetArchiveRetention for reset purge:** `src/infra/session-health-remediation-executor.ts:526-527`
  - body: Reset archive deletion uses `snapshot.maintenance.pruneAfterMs`, but current config has a separate `resetArchiveRetention` setting, including `false` to disable cleanup. Carry the resolved reset retention value into the snapshot/executor before deleting `.reset` archives.
  - confidence: 0.93
- **[P2] Keep orphan temp files out of critical reconciliation:** `src/infra/session-health-collector.ts:337-338`
  - body: The collector sets `reconciliationRecommended` for any orphaned temp file, and `deriveIndexHealth` makes that critical. A single crashed-write `.tmp` should follow the warning/Tier 0 cleanup path, not escalate the whole surface to critical.
  - confidence: 0.86
