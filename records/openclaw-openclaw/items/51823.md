## What This Changes

The PR adds session-health collection and derived UI visibility, remediation dry-run/execution paths for `openclaw sessions cleanup`, docs/tests, and a pre-commit `OPENCLAW_SKIP_CHECK` gate bypass.

## Best Possible Solution

Split or rework the feature so read-only session-health visibility can land independently, while any execution path remains gated until it follows persisted transcript paths, archive timestamp semantics, reset retention policy, and calibrated health severity.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Match orphan discovery to sessionFile metadata:** `src/infra/session-health-file-discovery.ts:117`
  - body: `discoverOrphanTranscripts` only strips `.jsonl` and compares that value to indexed session IDs, but current session entries can point at custom or topic transcript filenames via `sessionFile`. Those live files can be treated as orphans and archived, making active sessions look missing; build the referenced set from resolved `sessionFile` paths instead.
  - confidence: 0.93
- **[P1] Use archive timestamps for stale archive discovery:** `src/infra/session-health-file-discovery.ts:141`
  - body: Stale `.deleted` checks use file `mtime`, while the executor archives orphans with `fs.rename`, which preserves the old transcript mtime. When tier 1 runs archive-orphan before purge-deleted in the same execution, an old live transcript can be renamed and immediately deleted instead of retained as an archive; parse the `.deleted.<timestamp>` suffix for retention.
  - confidence: 0.92
- **[P1] Honor reset archive retention when purging resets:** `src/infra/session-health-remediation-executor.ts:526-527`
  - body: `executeArchiveStaleResetTranscripts` uses `snapshot.maintenance.pruneAfterMs`, but reset archives have their own `resetArchiveRetentionMs` policy and can be disabled. This can delete reset recovery artifacts earlier than configured, so reset cleanup needs the dedicated reset retention value and should no-op when it is disabled.
  - confidence: 0.9
- **[P2] Keep orphan temp files out of reconciliation-critical state:** `src/infra/session-health-collector.ts:337-338`
  - body: The collector sets `reconciliationRecommended` for any orphaned temp file, and `deriveIndexHealth` turns that flag into a critical index-health state before the orphan-temp warning branch can apply. A single crashed `.tmp` artifact should not force an overall critical Mission Control alert.
  - confidence: 0.87
