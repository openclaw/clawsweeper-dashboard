## Risks / Open Questions

- The new trace API intentionally returns full prompts, tool schemas, invocation parameters, response chunks, and raw payloads when the source-checkout, env-flag, admin, and loopback gates pass; maintainers need to accept that as the debugging security boundary.
- The store is count-limited but request payloads are not byte-limited or truncated, so a dev Gateway with tracing enabled can retain up to 200 very large prompt/tool payloads in memory until eviction.
- Exporting `recordDevLlmTrace*` from `openclaw/plugin-sdk/diagnostic-runtime` may create third-party plugin API expectations unless this is intentionally approved and documented as SDK surface.
- Existing Control UI device tokens that were approved without `operator.admin` may not show the Traces tab until scope upgrade or re-pairing, so the upgrade behavior should be intentional.
