The PR changes restart sentinel summary formatting so every successful config.patch sentinel says restart required, and adds one regression test for that formatting.

## Best Possible Solution

Carry an explicit restart-required fact from the config write/reload decision into restart sentinel payloads, then format pending, completed, and hot-reload/no-restart notices distinctly without adding new config surface.

## Solution Assessment

No. This PR is a plausible narrow formatter patch, but the best fix is to carry restart-required state from the config reload decision and distinguish pending restart-required, completed restart, and hot-reloaded writes.
