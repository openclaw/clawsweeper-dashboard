## Summary

Keep open for maintainer review: this is a focused bug-fix PR, but the formatter-only change is too broad and would label hot-reloaded or already-consumed config.patch sentinels as still requiring a restart.

## Best Possible Solution

Carry explicit restart-required state from the config reload decision into the persisted sentinel and format pending, completed, and hot-reloaded config writes distinctly before landing.

## Solution Assessment

No. The formatter-only fix is too broad; the maintainable fix is to carry explicit restart-required state from the reload plan and pass completed-state formatting when startup consumes a sentinel.
