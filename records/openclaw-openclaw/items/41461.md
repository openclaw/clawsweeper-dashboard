## Summary

Keep open: the reported OAuth beta bug is valid and the PR direction is useful, but the patch only updates the older embedded-runner hunk while current main routes Anthropic wrapping through the bundled Anthropic plugin, where the same gate still remains.

## What This Changes

The PR changes embedded-runner Anthropic extra-param wrapping to always apply the beta-header wrapper and updates embedded-runner tests for default and OAuth beta injection.

## Best Possible Solution

Land the behavior in the current bundled Anthropic provider plugin: re-merge required OAuth/default betas for direct Anthropic OAuth requests even without extra-param betas, preserve explicit model-level beta headers, keep implicit betas suppressed for custom-compatible endpoints, and add focused regression coverage plus changelog.

## Reproduction Assessment

Yes, source-backed. Direct Anthropic OAuth with an `sk-ant-oat` token, model-level `anthropic-beta`, and no `context1m` or `anthropicBeta` extra param skips the active wrapper and lets model headers replace the seeded OAuth beta.

## Solution Assessment

No. Always applying OAuth/default beta merging is the right direction, but the PR needs to be ported to `extensions/anthropic/stream-wrappers.ts` instead of only patching the older embedded-runner guard.
