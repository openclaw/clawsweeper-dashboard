## Summary

Keep open. Current main still source-reproduces the Anthropic OAuth beta/header overwrite path, but this PR changes an older embedded-runner wrapper gate instead of the active Anthropic plugin/transport header path and lacks inspectable real behavior proof.

## What This Changes

The branch changes the pi-embedded-runner Anthropic beta wrapper gate to run whenever `provider === "anthropic"` and updates extra-param tests for default and OAuth beta headers without `context1m`.

## Best Possible Solution

Land a corrected replacement in the active Anthropic plugin/transport header path that preserves OAuth/default betas when model-level `anthropic-beta` is present, while keeping implicit betas off custom Anthropic-compatible endpoints and adding redacted real request proof.

## Reproduction Assessment

Yes, source-backed. Use a direct Anthropic model with an `sk-ant-oat` token, no extra-param beta, and a model-level `anthropic-beta`; current main skips the plugin beta wrapper and later model headers can replace the seeded OAuth beta.

## Solution Assessment

No. Preserving OAuth betas is the right direction, but this PR edits stale embedded-runner logic and does not preserve the current direct-endpoint boundary for custom Anthropic-compatible providers.
