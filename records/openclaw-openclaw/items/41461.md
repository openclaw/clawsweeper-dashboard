## Summary

Keep open. The bug is source-backed, but this PR still targets the old embedded-runner surface while current main owns Anthropic wrapping in the Anthropic plugin, and the external PR lacks inspectable real behavior proof.

## What This Changes

The PR changes embedded-runner Anthropic beta-header wrapping to run even when no explicit Anthropic betas are resolved, and updates embedded-runner tests for default and OAuth beta headers.

## Best Possible Solution

Land a narrow replacement in extensions/anthropic/stream-wrappers.ts that applies OAuth/default beta merging for direct Anthropic requests even when extra-param betas are empty, preserves custom-endpoint suppression, and adds plugin-level regression coverage.

## Reproduction Assessment

Yes, source-backed rather than live. Use a direct Anthropic model with an sk-ant-oat token, no context1m or anthropicBeta extra param, and a model-level anthropic-beta header; the active plugin skips the wrapper and model/options headers can replace the seeded OAuth beta.

## Solution Assessment

No. Always applying OAuth/default beta merging is the right direction for direct Anthropic OAuth, but this patch targets stale embedded-runner code instead of the active Anthropic plugin and needs a direct-endpoint guard.
