## Summary

Keep this PR open, but do not merge it as-is. The underlying OAuth beta-header bug still exists on current main, and the PR is useful source work for #41444, but it targets an older embedded-runner path instead of the active Anthropic provider plugin wrapper now used by current main.

## What This Changes

The PR changes Anthropic extra-param wrapping so `createAnthropicBetaHeadersWrapper` runs even when no explicit Anthropic extra-param beta is configured, and adds OAuth-token regression coverage in the embedded-runner extra-params tests.

## Best Possible Solution

Update this work for current main by moving the unconditional beta-wrapper composition into `extensions/anthropic/stream-wrappers.ts`, passing an empty explicit beta list when needed, preserving custom-endpoint suppression for implicit Anthropic betas, and adding focused wrapper coverage for OAuth plus model-level `anthropic-beta` with no extra-param beta.

## Reproduction Assessment

Yes. The high-confidence reproduction path is an Anthropic OAuth token plus a model/provider-level `anthropic-beta` header and no `context1m` or `anthropicBeta` extra param; current main's wrapper gate leaves `options.headers` without the OAuth beta, while transport header merging lets the model-level header overwrite the seeded OAuth beta.

## Solution Assessment

No, not as-is. Always applying the beta wrapper is the right direction, but the patch must land in the active Anthropic plugin wrapper path and account for direct versus custom Anthropic-compatible endpoints.
