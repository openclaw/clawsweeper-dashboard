## Summary

Keep open: the PR is directionally aimed at a real OAuth beta-header bug, but its diff changes the stale embedded-runner guard while current main routes Anthropic wrapping through the active plugin, where the empty-beta gate still remains; the external PR also lacks inspectable real behavior proof.

## What This Changes

The branch makes the old embedded-runner Anthropic beta wrapper run whenever the provider is Anthropic and updates embedded-runner tests for default and OAuth beta headers.

## Best Possible Solution

Land a narrow replacement in extensions/anthropic/stream-wrappers.ts that applies OAuth/default beta merging for direct Anthropic requests even with empty extra params, preserves custom-endpoint suppression, and adds plugin-level regression coverage.

## Reproduction Assessment

Yes, source-backed rather than live. Use a direct Anthropic model with an sk-ant-oat token, no context1m or anthropicBeta extra param, and a model-level anthropic-beta header; the active plugin skips the wrapper and later header merging can replace the seeded OAuth beta.

## Solution Assessment

No. Always applying OAuth/default beta merging is the right direction for direct Anthropic OAuth, but this patch targets stale embedded-runner code instead of the active Anthropic plugin and needs a direct-endpoint guard.
