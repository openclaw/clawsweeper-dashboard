## Summary

Keep open: the reported OAuth beta-header gap is still source-reproducible on current main, but this PR is not mergeable as-is because it targets the stale embedded-runner surface, would over-apply implicit Anthropic betas to custom endpoints, and lacks inspectable real behavior proof.

## What This Changes

The PR changes embedded-runner Anthropic beta-header wrapping to run for every Anthropic provider and updates embedded-runner tests for default and OAuth beta headers.

## Best Possible Solution

Land a narrow replacement in `extensions/anthropic/stream-wrappers.ts` that applies OAuth/default beta merging for direct Anthropic requests with empty extra-param betas, preserves custom-endpoint suppression, and adds plugin-level regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: use a direct Anthropic model with an `sk-ant-oat` token, no `context1m` or `anthropicBeta` extra param, and a model-level `anthropic-beta`; the active wrapper is skipped and later header merging can replace the seeded OAuth beta.

## Solution Assessment

No. Always applying OAuth/default beta merging is the right direction for direct Anthropic OAuth, but this patch targets stale embedded-runner code instead of the active Anthropic plugin and needs a direct-endpoint guard.
