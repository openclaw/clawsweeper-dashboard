## Summary

Keep this PR open, but do not merge it as-is. The bug and proposed direction are valid, yet current main now routes Anthropic stream wrapping through the bundled Anthropic provider plugin, while this PR changes the older embedded-runner path and tests; the active plugin wrapper is still gated on explicit betas, so the reported OAuth plus model-level header case remains unfixed.

## What This Changes

The PR changes Anthropic extra-param stream wrapping to always compose the beta-header wrapper and updates embedded-runner extra-params tests for default and OAuth beta injection.

## Best Possible Solution

Land the useful behavior in the current Anthropic provider plugin: ensure direct Anthropic OAuth requests always re-merge required OAuth/default betas even when no extra-param beta is configured, preserve explicit model-level beta headers, suppress implicit betas for custom endpoints, and add focused active-wrapper regression coverage.

## Reproduction Assessment

Yes. The source-backed reproduction is an Anthropic OAuth token, a model/provider-level `anthropic-beta` header, and no `context1m` or `anthropicBeta` extra param; current main skips the plugin beta wrapper, so model headers can overwrite the transport-seeded OAuth beta.

## Solution Assessment

No, not as-is. Always applying OAuth/default beta merging is the right direction, but the PR must be rebased or replaced so the change lands in `extensions/anthropic/stream-wrappers.ts` with endpoint-aware behavior rather than only the older embedded-runner path.
