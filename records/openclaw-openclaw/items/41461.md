## Summary

Keep open. The reported OAuth/header bug is still source-reproducible on current main, but this PR is not merge-ready because it patches the old embedded-runner guard instead of the active Anthropic plugin wrapper and it lacks after-fix real behavior proof.

## What This Changes

The PR changes embedded-runner Anthropic extra-param wrapping to always install the beta-header wrapper and updates embedded-runner tests for default and OAuth beta injection.

## Best Possible Solution

Land a narrow replacement in `extensions/anthropic/stream-wrappers.ts` that applies OAuth/default beta merging for direct Anthropic OAuth even without explicit betas, preserves custom-endpoint suppression, and adds plugin-level regression coverage.

## Reproduction Assessment

Yes, source-backed. Use a direct Anthropic model with an `sk-ant-oat` token, no `context1m` or `anthropicBeta` extra param, and a model-level `anthropic-beta` header; the active plugin skips the wrapper and later headers can replace the seeded OAuth beta.

## Solution Assessment

No. Always applying beta merging is directionally right for direct Anthropic OAuth, but this PR changes obsolete embedded-runner code instead of the active Anthropic plugin and does not preserve the documented custom-endpoint guard.
