## Summary

Keep this PR open. It addresses a real Anthropic OAuth beta-header regression, but the branch changes the older embedded-runner wrapper while current main routes Anthropic provider wrapping through the bundled Anthropic plugin, where the same gate still exists.

## What This Changes

The PR branch changes `src/agents/pi-embedded-runner/extra-params.ts` to always apply the Anthropic beta-header wrapper and updates embedded-runner extra-param tests for default and OAuth beta injection.

## Best Possible Solution

Land the useful behavior in the current bundled Anthropic provider plugin: ensure direct Anthropic OAuth requests always re-merge required OAuth/default betas even when no extra-param beta is configured, preserve explicit model-level beta headers, suppress implicit betas for custom-compatible endpoints, and add focused active-wrapper regression coverage.

## Reproduction Assessment

Yes. The source-backed reproduction is direct Anthropic OAuth (`sk-ant-oat`) plus model/provider-level `anthropic-beta` and no `context1m` or `anthropicBeta` extra param; current main skips the active wrapper and lets model headers replace the seeded OAuth beta.

## Solution Assessment

No. Always applying OAuth/default beta merging is the right direction, but the PR must be rebased or replaced so the change lands in `extensions/anthropic/stream-wrappers.ts` with endpoint-aware behavior instead of only the older embedded-runner hunk.
