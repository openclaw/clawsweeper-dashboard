## Summary

Keep open. The PR targets the old embedded-runner wrapper path while current main’s active Anthropic plugin still skips OAuth/default beta merging when no extra-param beta is configured, and the external PR still lacks inspectable real-behavior proof.

## What This Changes

The PR always wraps Anthropic requests in the old embedded-runner extra-param path and updates embedded-runner tests for default and OAuth beta injection.

## Best Possible Solution

Land a narrow replacement in `extensions/anthropic/stream-wrappers.ts` that applies OAuth/default beta merging for direct Anthropic OAuth without explicit betas, preserves custom endpoint suppression, and adds plugin-level regression coverage.

## Reproduction Assessment

Yes, source-backed rather than live. Use a direct Anthropic model with an `sk-ant-oat` token, no `context1m` or `anthropicBeta` extra param, and a model-level `anthropic-beta` header; the active plugin skips the wrapper and later header merging can replace the seeded OAuth beta.

## Solution Assessment

No. Always applying OAuth/default beta merging is directionally right for direct Anthropic OAuth, but this patch targets the stale embedded-runner path instead of the active Anthropic plugin and needs a direct-endpoint guard.
