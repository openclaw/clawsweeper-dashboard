## Summary

Keep open. The Anthropic OAuth beta/header bug is still source-reproducible on current main, but this PR changes the older embedded-runner guard instead of the active Anthropic provider plugin path and still lacks inspectable real behavior proof.

## What This Changes

The PR changes embedded-runner Anthropic beta-header wrapping and tests so OAuth/default beta injection runs even when no extra-param Anthropic betas are configured.

## Best Possible Solution

Land a narrow replacement in `extensions/anthropic/stream-wrappers.ts` that applies OAuth/default beta merging for direct Anthropic requests even with empty extra-param betas, preserves custom-endpoint suppression, adds plugin-level regression coverage, and includes redacted live proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: use a direct Anthropic model with an `sk-ant-oat` token, no `context1m` or `anthropicBeta` extra param, and a model-level `anthropic-beta`; the active wrapper is skipped and later model headers can replace the seeded OAuth beta.

## Solution Assessment

No. Always applying OAuth/default beta merging is the right direction for direct Anthropic OAuth, but this patch targets stale embedded-runner logic instead of the active Anthropic plugin and needs a direct-endpoint guard.
