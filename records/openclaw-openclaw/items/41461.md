## Summary

Keep open. This fixes a real Anthropic OAuth beta-header regression, but the branch changes stale embedded-runner code while current main routes Anthropic stream wrapping through the bundled Anthropic plugin, so the active path remains broken and custom-endpoint beta suppression still needs to be preserved.

## What This Changes

The PR always applies the Anthropic beta-header wrapper in the old embedded-runner extra-param path and updates embedded-runner tests for default and OAuth beta injection.

## Best Possible Solution

Land a narrow replacement in `extensions/anthropic/stream-wrappers.ts` that applies OAuth/default beta merging for direct Anthropic OAuth without explicit betas, keeps implicit betas suppressed on custom endpoints, and adds plugin-level regression coverage.

## Reproduction Assessment

Yes, source-backed rather than live. Use a direct Anthropic model with an `sk-ant-oat` token, no `context1m` or `anthropicBeta` extra param, and a model-level `anthropic-beta` header; the active plugin skips the wrapper and later header merging can replace the seeded OAuth beta.

## Solution Assessment

No. Always applying OAuth/default beta merging is directionally right for direct Anthropic OAuth, but this patch targets stale embedded-runner code instead of the active Anthropic plugin and needs a direct-endpoint guard to preserve custom endpoint behavior.
