## Summary

Keep open: the PR addresses a valid Anthropic OAuth regression, but it patches the older embedded-runner surface while current main routes Anthropic stream wrapping through the bundled Anthropic plugin, where the same guard still skips OAuth beta merging.

## What This Changes

The PR changes embedded-runner Anthropic extra-param wrapping to always apply the beta-header wrapper and updates embedded-runner tests for default and OAuth beta injection.

## Best Possible Solution

Land a narrow replacement in the bundled Anthropic provider plugin that preserves required OAuth/default betas for direct Anthropic OAuth requests even without extra-param betas, keeps custom endpoint suppression, and adds focused regression coverage plus changelog.

## Reproduction Assessment

Yes, source-backed. Direct Anthropic OAuth with an `sk-ant-oat` token, model-level `anthropic-beta`, and no `context1m` or `anthropicBeta` extra param skips the active wrapper and allows later headers to replace the seeded OAuth beta.

## Solution Assessment

No. Always applying the beta merge is the right direction for direct Anthropic OAuth, but this PR changes obsolete embedded-runner code instead of the current Anthropic plugin and does not show the custom-endpoint guard the current contract needs.
