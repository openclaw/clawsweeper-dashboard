## Summary

Keep open: the reported Anthropic OAuth/header bug is still source-backed, but this PR should not merge as-is because it patches stale embedded-runner glue while current main routes Anthropic wrapping through the bundled provider plugin.

## What This Changes

The PR changes embedded-runner Anthropic extra-param wrapping to always apply the beta-header wrapper and updates embedded-runner tests for default and OAuth beta injection.

## Best Possible Solution

Land a narrow replacement in the bundled Anthropic provider plugin that preserves OAuth/default betas for direct Anthropic OAuth requests, keeps implicit betas suppressed on custom endpoints, and adds focused regression coverage.

## Reproduction Assessment

Yes, source-backed. Direct Anthropic OAuth with an `sk-ant-oat` token, no `context1m` or `anthropicBeta` extra param, and a model/options `anthropic-beta` header skips the active wrapper and lets later headers replace the seeded OAuth beta.

## Solution Assessment

No. Always applying beta merging is the right direction for direct Anthropic OAuth, but this PR patches obsolete embedded-runner code instead of the active Anthropic plugin and does not preserve the documented custom-endpoint guard.
