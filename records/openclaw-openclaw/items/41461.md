## Summary

Keep open. The reported OAuth beta-header regression is still source-reproducible on current main, but this PR changes the stale embedded-runner path instead of the active Anthropic plugin wrapper and still lacks inspectable real behavior proof.

## What This Changes

The PR makes the embedded-runner Anthropic beta header wrapper run even when no explicit beta is resolved, and updates embedded-runner tests for default and OAuth beta injection.

## Best Possible Solution

Land a narrow replacement in extensions/anthropic/stream-wrappers.ts that applies OAuth/default beta merging for direct Anthropic OAuth even with empty extra params, preserves custom endpoint suppression, and adds plugin-level regression coverage.

## Reproduction Assessment

Yes, source-backed rather than live. Use a direct Anthropic model with an sk-ant-oat token, no context1m or anthropicBeta extra param, and a model-level anthropic-beta header; the active plugin skips the wrapper and later header merging can replace the seeded OAuth beta.

## Solution Assessment

No. Always applying OAuth/default beta merging is directionally right for direct Anthropic OAuth, but this patch targets the stale embedded-runner path instead of the active Anthropic plugin and needs a direct-endpoint guard.
