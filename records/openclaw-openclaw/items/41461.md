## Summary

Keep open: this PR targets a real Anthropic OAuth regression, but the patch is not mergeable because it changes the older embedded-runner wrapper gate instead of the active Anthropic plugin/transport path, and the external PR still lacks inspectable real behavior proof.

## What This Changes

The PR changes pi-embedded-runner Anthropic beta wrapping to run whenever the provider is `anthropic` and updates embedded-runner tests to expect default/OAuth beta headers without `context1m`.

## Best Possible Solution

Land a replacement that preserves OAuth/default betas when model-level `anthropic-beta` is present in the active Anthropic plugin/transport path, while keeping custom Anthropic-compatible endpoints free of implicit Anthropic betas.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: use a direct Anthropic model with an `sk-ant-oat` token, no extra-param beta, and a model-level `anthropic-beta`; the wrapper is skipped and later model headers can replace the seeded OAuth beta.

## Solution Assessment

No. Preserving OAuth betas is the right direction, but this PR edits stale embedded-runner logic instead of the active Anthropic plugin/transport header merge and also misses the direct-endpoint boundary.
