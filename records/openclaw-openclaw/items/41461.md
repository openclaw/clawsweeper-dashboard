## Summary

Keep this PR open as useful source work, but it is not merge-ready because it patches the older embedded-runner surface while current main routes Anthropic beta-header behavior through the bundled Anthropic plugin.

## What This Changes

The PR changes embedded-runner Anthropic beta wrapping to always install the beta-header wrapper and updates embedded-runner tests for default and OAuth beta injection.

## Best Possible Solution

Land the behavior in the current bundled Anthropic provider plugin: re-merge required OAuth/default betas for direct Anthropic OAuth requests even without extra-param betas, preserve explicit model-level beta headers, keep implicit betas suppressed for custom-compatible endpoints, and add focused regression coverage plus changelog.

## Reproduction Assessment

Yes, source-backed. Direct Anthropic OAuth with an sk-ant-oat token, model-level anthropic-beta, and no context1m or anthropicBeta extra param leaves the active wrapper uncomposed and lets model headers overwrite the seeded OAuth beta.

## Solution Assessment

No. Always applying or otherwise re-merging OAuth/default beta handling is the right direction, but the patch needs to be ported to the current Anthropic plugin wrapper instead of the stale embedded-runner guard.
