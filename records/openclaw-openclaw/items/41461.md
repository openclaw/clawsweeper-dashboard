## Summary

Keep open: the linked OAuth bug is source-reproducible on current main, but this PR should not merge as-is because it targets the older embedded-runner wrapper path while current Anthropic behavior is owned by the plugin wrapper, and it would also risk adding implicit Anthropic beta headers to custom endpoints.

## What This Changes

The PR changes embedded-runner Anthropic beta-header wrapping and tests so the wrapper runs for Anthropic even when no explicit beta extra params are resolved.

## Best Possible Solution

Land a narrow replacement in `extensions/anthropic/stream-wrappers.ts` that applies OAuth/default beta merging for direct Anthropic requests even when extra-param betas are empty, preserves custom-endpoint suppression, and adds plugin-level regression coverage.

## Reproduction Assessment

Yes, source-backed rather than live. Use a direct Anthropic model with an `sk-ant-oat` token, no `context1m` or `anthropicBeta` extra param, and a model-level `anthropic-beta` header; the active wrapper is skipped and later header merging can replace the seeded OAuth beta.

## Solution Assessment

No. Always applying OAuth/default beta merging is the right direction for direct Anthropic OAuth, but this patch targets stale embedded-runner code instead of the active Anthropic plugin and needs a direct-endpoint guard.
