Keep open for maintainer review: current main still depends on Commander's runtime-only `rawArgs` during lazy reparse, and this PR replaces that with OpenClaw-owned argv context plus nested argv scoping. I found no blocking review findings or security concerns.

## What This Changes

The branch stores CLI parse argv in OpenClaw program context, uses it for lazy command reparse, scopes nested plugin argv, and adds focused CLI/browser/plugin regression tests.

## Best Possible Solution

Land the focused argv ownership refactor after current-head CI stays green, keeping lazy CLI reparse independent of Commander's private runtime field.

## Reproduction Assessment

Yes. Source inspection shows current main reparses lazy commands through Commander's untyped `rawArgs`, and the PR body gives a concrete before/after `browser --json tabs` CLI path that previously failed at argument parsing.

## Solution Assessment

Yes. Owning argv in OpenClaw program context and resolving it through ancestor commands is narrower and more maintainable than depending on Commander's internal `rawArgs` field.
