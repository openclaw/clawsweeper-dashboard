## Summary

Keep open: the cleanup is still useful because current main and v2026.5.22 still read Commander's private `rawArgs`, but this PR head still breaks the lazy CLI smoke path by storing argv at program construction instead of from the actual parse call, and the supplied proof is only test output.

## What This Changes

The branch stores CLI parse argv in program context, resolves it through command ancestors during lazy reparse, passes rewritten argv into `buildProgram`, and adds focused CLI/program regression tests.

## Best Possible Solution

Keep the OpenClaw-owned argv direction, but update argv storage from the actual parse invocation and require redacted real CLI output for a lazy command before maintainer merge.

## Reproduction Assessment

Yes for the PR defect: the PR head's `checks-node-agentic-cli` annotations show `program.smoke.test.ts` lazy command cases failing after `program.parseAsync(argv, { from: "user" })` reaches `action-reparse.ts`. I did not run tests locally because this was a read-only review.

## Solution Assessment

No, not yet: owning argv outside Commander is a maintainable direction, but the current patch stores construction-time argv and misses supported parse-time argv callers. The safer fix is to capture or update argv from the actual parse invocation while preserving the new non-Commander-owned storage path.
