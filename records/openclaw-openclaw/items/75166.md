## Summary

Keep open. Current main still contains the compact media provider-inventory output bypass, the focused media handler test still asserts that hidden video inventory is emitted, and Discord guild messages are classified as channel turns that are not covered by the existing group-summary suppression. Open PR #75550 explicitly closes this issue and #75645 is a related open implementation candidate, so the issue should stay open until one canonical fix merges.

## What This Changes

Suppress or more tightly gate `image_generate` and `video_generate` provider inventory output so provider/model/configuration/auth-hint details do not appear as visible tool output in Discord group or other shared channel surfaces by default.

## Best Possible Solution

Land one focused fix that treats media `action=list` provider inventory as internal by default on shared chat surfaces while preserving structured tool results for agent context and preserving generated media delivery. The merged fix should include regression coverage for both image and video inventory with hidden output, plus an explicit maintainer decision on direct/verbose/debug visibility.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: return an `image_generate` or `video_generate` list result with `details.providers` while `shouldEmitToolOutput()` is false, and current main still routes that through `emitToolOutput`; Discord guild turns are `ChatType: "channel"`, not the existing `group` suppression path.

## Solution Assessment

Yes, the requested suppression or stricter gating direction is the right class of fix. The narrow maintainable solution is to remove or restrict the compact provider-inventory bypass before external chat delivery without changing provider list generation or generated media attachment delivery.
