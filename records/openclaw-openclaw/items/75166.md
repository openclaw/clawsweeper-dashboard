## Summary

Keep open. Current main still contains the compact media provider-inventory emission path, and an open unmerged PR (#75550) already uses closing syntax for this issue, so the right cleanup is to review and land or reject that PR rather than closing the issue early.

## What This Changes

The issue asks OpenClaw to prevent image_generate/video_generate provider inventory from being emitted as visible tool output in Discord group/channel surfaces.

## Best Possible Solution

Review and land #75550, or an equivalent narrow fix, so media provider inventory from action=list remains internal by default on shared chat surfaces while structured tool results and generated media attachment delivery continue to work.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to process an image_generate or video_generate list result containing details.providers while ctx.shouldEmitToolOutput() is false; current main still emits that text, and Discord guild turns use ChatType channel.

## Solution Assessment

Yes. Removing or tightly gating the compact provider-inventory bypass is the narrow maintainable fix; changing provider list generation or generated media delivery would be broader than needed.
