## Summary

Keep open. Current `main` and the latest release still contain the compact image/video provider-inventory emission path, existing tests assert that inventory emits even when normal tool output is disabled, and Discord guild messages are `channel` surfaces that remain eligible for shared tool-result delivery. The closed Slack trace leak in #70912 is related context but does not cover this separate media inventory path.

## What This Changes

Suppress or more strictly gate `image_generate` and `video_generate` `action=list` provider inventory so it is not posted as visible tool output in Discord group channels by default.

## Best Possible Solution

Make media provider inventory from `action=list` an internal tool result by default on shared chat surfaces, preserving the structured result for the agent and preserving generated media delivery. If maintainers want operator-visible inventory, expose it only through an explicit direct/debug/admin path with regression coverage for image, video, Discord guild delivery, and media attachment delivery.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: return an `image_generate` or `video_generate` list result with `details.providers` while `shouldEmitToolOutput()` is false, and current main still routes that through `emitToolOutput`; Discord guild turns use `ChatType: "channel"`, which remains tool-summary eligible.

## Solution Assessment

Yes, the requested suppression or surface-gating direction is the right class of fix. The narrow maintainable solution is to remove or restrict the compact provider-inventory bypass before external chat delivery, not to change provider list generation or generated media attachment delivery.
