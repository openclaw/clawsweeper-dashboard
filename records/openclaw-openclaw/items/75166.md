## Summary

Keep open. Current main still contains the compact image/video provider-inventory output bypass, current media tests still assert that hidden video inventory emits, and open closing PRs #75550 and #75728 are already tracking candidate fixes.

## What This Changes

The issue reports that `image_generate` and `video_generate` provider inventory can be delivered as visible tool output in Discord shared channels, exposing provider/model/configuration/auth-hint metadata.

## Best Possible Solution

Land one canonical fix that keeps media provider inventory internal by default on shared chat surfaces while preserving structured tool results for the agent and generated media delivery.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: return an `image_generate` or `video_generate` list result with `details.providers` while `shouldEmitToolOutput()` is false, and current main still routes the text through `emitToolOutput`.

## Solution Assessment

Yes, suppression or stricter gating is the right solution class. The safest implementation should remove the compact bypass or tag/suppress provider-inventory text before non-direct chat delivery without changing generated media delivery.
