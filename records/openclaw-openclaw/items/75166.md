## Summary

Keep open. Current main still contains the compact media provider-inventory emission bypass, and the existing test still asserts the leak path when normal tool output is disabled. An open linked fix PR (#75550) is the right implementation candidate, so this issue should remain open until that PR lands or maintainers choose a different visibility policy.

## What This Changes

The issue requests suppressing or gating `image_generate` and `video_generate` provider inventory so Discord group/channel surfaces do not show provider, model, configuration, capability, or auth-hint details by default.

## Best Possible Solution

Review and land #75550 or an equivalent narrow change that keeps `action=list` media provider inventory internal by default on shared chat surfaces, while preserving structured tool results for the agent and leaving generated media attachment delivery unchanged.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: process an `image_generate` or `video_generate` list result with `details.providers` while `shouldEmitToolOutput()` is false; current main still routes that through `emitToolOutput`, and Discord guild turns are classified as `channel`.

## Solution Assessment

Yes. Removing or restricting the compact provider-inventory bypass before external chat delivery is the narrow maintainable fix; changing provider list generation itself would be broader than needed.
