## Summary

Keep open. Current main still has the compact media provider-inventory emission path, the colocated media test still asserts emission while normal tool output is disabled, and an open closing PR (#75550) is already tracking the narrow fix, so this issue should remain open until that PR lands or is closed.

## What This Changes

Suppress `image_generate` and `video_generate` provider inventory from default external Discord group/channel tool-output delivery while preserving internal tool results and generated media delivery.

## Best Possible Solution

Keep media provider inventory from `action=list` internal by default on shared chat surfaces, preserve structured results for the agent, and preserve generated media attachment delivery. If operator-visible inventory is desired, expose it only through an explicit direct/debug/admin path with regression coverage for image, video, Discord guild delivery, and media delivery.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to process an `image_generate` or `video_generate` list result containing `details.providers` while `shouldEmitToolOutput()` returns false; current main still routes that through `emitToolOutput`, and the existing media test asserts this behavior for video.

## Solution Assessment

Yes. The requested fix direction is the narrow maintainable path: remove or restrict the compact provider-inventory bypass before external chat delivery, without changing provider list generation or generated media delivery.
