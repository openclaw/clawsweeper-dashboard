## Summary

Keep open. Current main still contains the compact media provider-inventory output bypass, current media tests still assert that hidden video inventory is emitted, and Discord guild messages are classified as channel turns that can receive tool summaries. Open PRs #75550 and #75728 explicitly target this issue, so the issue should remain open until one canonical fix merges.

## What This Changes

The issue asks to prevent image_generate/video_generate provider inventory list output from becoming visible tool output on Discord group/channel surfaces.

## Best Possible Solution

Land one canonical fix that treats media action=list provider inventory as internal by default on shared chat surfaces while preserving structured tool results for the agent and generated media delivery. The final policy should explicitly decide whether DMs, full verbose output, or debug/admin paths may show inventory.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: return an image_generate or video_generate list result with details.providers while shouldEmitToolOutput() is false, and current main routes the resulting text through emitToolOutput; Discord guild turns are ChatType "channel" and remain tool-summary eligible.

## Solution Assessment

Yes, the requested suppression or stricter gating direction is the right class of fix. The narrowest maintainable solution is to remove or restrict the compact provider-inventory bypass before external chat delivery, with a maintainer choice on whether the defense belongs in the media handler, dispatcher, or both.
