## Summary

Keep open. Current main still has the compact media provider-inventory bypass, a focused test still asserts that hidden video inventory is emitted, and Discord guild traffic is classified as channel traffic that can receive tool summaries. Open PRs #75550 and #75728 both target this issue, so the issue should stay open until one canonical fix merges.

## What This Changes

Suppress `image_generate` and `video_generate` provider inventory output so provider/model/configuration/auth-hint details do not appear as visible Discord group/channel tool output by default.

## Best Possible Solution

Land one canonical fix that treats media `action=list` provider inventory as internal by default on shared chat surfaces while preserving structured tool results for the agent and generated media delivery. The final policy should explicitly decide whether DMs, verbose output, or debug/admin paths may display inventory.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: process an `image_generate` or `video_generate` list result containing `details.providers` with `shouldEmitToolOutput()` false, and current main routes the resulting text through `emitToolOutput`; Discord guild turns are `ChatType: "channel"` and remain tool-summary eligible.

## Solution Assessment

Yes. The requested suppression or stricter gating is the right class of fix; the narrow maintainable solution is to remove or restrict the compact provider-inventory bypass before external chat delivery while preserving generated media handling.
