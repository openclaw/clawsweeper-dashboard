## What This Changes

The issue reports that image_generate/video_generate action=list provider inventory can be posted as visible tool output in Discord group/channel surfaces and asks for it to stay internal by default.

## Best Possible Solution

Land one canonical fix that keeps provider inventory internal by default on shared surfaces while preserving structured tool results for the agent and generated media delivery.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: return an image_generate or video_generate list result with details.providers while shouldEmitToolOutput() is false, and current main still routes the text through emitToolOutput.

## Solution Assessment

Yes. Suppression or stricter gating is the right solution class; the safest fix should remove the compact bypass or tag/suppress provider-inventory text before non-direct chat delivery without changing generated media delivery.
