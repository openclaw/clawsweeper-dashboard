## Summary

Keep open. Current main still contains the media provider-inventory output bypass, the focused media test still asserts that hidden video inventory is emitted, and Discord guild messages are classified as channel turns that are not covered by the existing group-summary suppression. Because open PRs #75550 and #75625 explicitly close this issue, the issue should stay open until one canonical fix merges.

## What This Changes

The issue requests keeping `image_generate` and `video_generate` provider inventory from being emitted as visible tool output on Discord group/channel surfaces by default.

## Best Possible Solution

Land one canonical fix that treats media `action=list` provider inventory as internal by default on shared chat surfaces while preserving structured tool results for the agent and preserving generated media delivery. The maintainer decision should explicitly cover whether DMs, full verbose output, or debug/admin paths may show inventory.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: return an `image_generate` or `video_generate` list result with `details.providers` while `shouldEmitToolOutput()` is false, and current main still routes that through `emitToolOutput`; Discord guild turns use `ChatType: "channel"`, which remains tool-summary eligible.

## Solution Assessment

Yes, the requested suppression or stricter gating direction is the right class of fix. The safest maintainable path is to remove or restrict the compact provider-inventory bypass before external chat delivery, without changing provider list generation or generated media attachment delivery.
