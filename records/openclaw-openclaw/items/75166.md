## Summary

Keep open. Current main still contains the compact media provider-inventory output bypass, the focused media handler test still asserts hidden video inventory is emitted, and open implementation PRs #75550 and #75728 are already linked as candidates to resolve this issue after maintainer review.

## What This Changes

Suppress or gate `image_generate` and `video_generate` provider inventory output so runtime provider/model/configuration/auth-hint details do not appear as visible Discord group-channel tool output by default.

## Best Possible Solution

Land one canonical fix that treats media `action=list` provider inventory as internal by default on shared chat surfaces, keeps structured tool results available to the agent, preserves generated media delivery, and explicitly documents whether DMs, verbose/full output, or debug/admin paths may show inventory.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: return an `image_generate` or `video_generate` list result with text plus `details.providers` while `shouldEmitToolOutput()` is false, and current main routes it through `emitToolOutput`; Discord guild turns are `ChatType: "channel"` and remain tool-summary eligible.

## Solution Assessment

Yes, suppression or stricter gating is the right class of fix. The maintainer decision is whether the narrow handler-level removal in #75550 or the shared-surface classifier/dispatcher policy in #75728 is the best long-term boundary.
