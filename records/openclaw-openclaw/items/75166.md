## Summary

Keep open. Current main still allows image/video provider inventory text to bypass the normal hidden tool-output gate, and open closing PRs #75550 and #75728 already track candidate fixes.

## What This Changes

Suppress image_generate/video_generate provider inventory from Discord group/channel-visible tool output while preserving internal structured results and generated media delivery.

## Best Possible Solution

Land one canonical fix that keeps media provider inventory internal by default on shared surfaces while preserving generated media delivery and direct or explicit diagnostics.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: pass an image_generate or video_generate list result with text and details.providers through handleToolExecutionEnd while shouldEmitToolOutput() is false, and current main still calls emitToolOutput.

## Solution Assessment

Yes. Suppressing or strictly gating provider-inventory text before external delivery is the right solution class; the remaining choice is the narrower handler-only fix versus the broader dispatcher tagging approach.
