## Summary

Keep open. Current main still contains the compact media provider-inventory output bypass, the focused media handler test still asserts that hidden video inventory emits, and the issue is already paired with open closing PRs #75550 and #75728 that should be reviewed before this issue is closed.

## What This Changes

The issue reports that image_generate/video_generate action=list provider inventory can bypass normal tool-output visibility and appear as visible output in Discord shared channels.

## Best Possible Solution

Land one canonical fix that keeps media action=list provider inventory internal by default on shared chat surfaces, preserves structured tool results for the agent, and leaves generated media delivery unchanged. The fix should include regression coverage for image_generate and video_generate inventory on hidden/shared surfaces and document any allowed direct/debug diagnostic path.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by passing an image_generate or video_generate list result with details.providers through handleToolExecutionEnd while shouldEmitToolOutput() is false; current main then calls emitToolOutput via the compact inventory bypass, and Discord guild turns are ChatType channel.

## Solution Assessment

Yes, suppression or stricter gating is the right solution class. The narrowest maintainable approach is to remove or restrict the compact provider-inventory bypass and, if maintainers want defense in depth, add a typed dispatch-level filter for non-direct surfaces.
