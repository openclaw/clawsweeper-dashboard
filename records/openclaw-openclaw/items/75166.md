## Summary

Keep open. Current main still contains the compact media provider-inventory output bypass, current tests still assert hidden video inventory emits, and open closing PRs #75550 and #75728 already track candidate fixes.

## What This Changes

The issue reports that `image_generate` and `video_generate` provider inventory can bypass hidden tool-output policy and appear in Discord group/channel chat.

## Best Possible Solution

Land one canonical fix that keeps media provider inventory internal or direct/explicit only while preserving structured tool results and generated media delivery.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: return an `image_generate` or `video_generate` list result with `details.providers` while `shouldEmitToolOutput()` is false, and current main still routes the text through `emitToolOutput`.

## Solution Assessment

Yes. Suppression or stricter gating is the right solution class; the safest fix should prevent non-direct/shared delivery of provider inventory without changing generated media delivery.
