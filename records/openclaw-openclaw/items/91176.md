## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Gate realtime audio before provider processing:** `extensions/voice-call/src/providers/msteams.ts:563`
  - body: `handleAudioFrame` forwards every realtime Teams audio frame into `realtimeCall.pushAudio()` before checking whether Teams recording is active. `pushAudio()` then sends the PCM into the realtime provider bridge, so the default `requireRecordingStatus` path still processes caller audio before the recording gate opens. Please drop or defer realtime input until recording is active, or explicitly require an opt-out before any provider sees the audio.
  - confidence: 0.92
- **[P2] Preserve AAD IDs in Teams session keys:** `extensions/voice-call/src/providers/msteams.ts:294`
  - body: For inbound Teams calls this line feeds `caller.aadId` into the existing CallManager path, whose `resolveVoiceCallSessionKey()` strips all non-digits because it was built for phone numbers. Two distinct GUID callers can collapse to the same `voice:<digits>` key under the default `per-phone` scope, leaking voice memory across Teams users. Use an explicit opaque Teams session key or teach the resolver about non-phone caller IDs before registering the call.
  - confidence: 0.85
- **[P2] Add answerTimeoutMs to the manifest schema:** `extensions/voice-call/openclaw.plugin.json:324-331`
  - body: `config.ts` and the tests support `msteams.outbound.answerTimeoutMs`, but the plugin manifest keeps `msteams.outbound.additionalProperties` false and only lists `enabled`, `workerBaseUrl`, and `tenantId`. A user config that sets the documented timeout can be rejected or hidden by manifest-driven setup/validation even though runtime accepts it.
  - confidence: 0.9
- **[P3] Update the README for outbound Teams callbacks:** `extensions/voice-call/README.md:101`
  - body: The README still says the `msteams` provider is inbound-only and that `initiateCall` throws, which directly contradicts this PR's outbound call-back implementation. Please update the Teams section and example config so users know how to configure `msteams.outbound` and when outbound Teams calls are supported.
  - confidence: 0.86
