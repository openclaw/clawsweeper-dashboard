## Summary

Keep open. Current main still lacks a public configurable local-media root allowlist for the image/PDF media tools, and the remaining change is a security-sensitive config/product decision that overlaps open implementation paths rather than stale cleanup.

## What This Changes

Add a configurable local-media root allowlist for the image tool so external iMessage attachment paths can be read without copying them into the workspace.

## Best Possible Solution

Define one canonical opt-in root policy, wire it through shared media root resolution and loadWebMedia, document the config, and add image/PDF/schema tests without weakening managed inbound-media or workspaceOnly boundaries.

## Reproduction Assessment

Yes, for the current limitation. Source and tests show an outside-workspace local image path is checked against default/workspace roots only and rejected; I did not run a live iMessage flow in this read-only review.

## Solution Assessment

Unclear. A configurable allowlist is plausible, but maintainers need to decide whether the right contract is per-media roots, iMessage attachment-root inheritance, or the broader tools.fs.roots policy already under review.
