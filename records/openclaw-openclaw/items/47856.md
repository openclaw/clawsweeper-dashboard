## Summary

Keep open. Current main still lacks a public configurable media local-root allowlist for the image/media tools: the resolver returns only workspace/default roots, the image tool passes that set directly into media loading, and the strict config schemas do not expose a root field. The request remains valid product/security work and overlaps active implementation paths in https://github.com/openclaw/openclaw/pull/52951 and https://github.com/openclaw/openclaw/pull/39034.

## What This Changes

The issue requests an opt-in config field that lets the image tool read local files from additional administrator-approved roots, especially iMessage attachment directories outside the default workspace/media roots.

## Best Possible Solution

Define one canonical explicit-root policy, wire it through shared media root resolution, schema/types/docs, and image/PDF tests, and preserve managed inbound-media plus workspaceOnly semantics.

## Reproduction Assessment

Yes, for the current behavior from source and tests: an external iMessage attachment path reaches the image tool, resolveMediaToolLocalRoots does not include any configured external root, and existing image-tool coverage pins outside-root rejection. This proves the gap, but the requested allowlist is a feature/config addition rather than a broken documented contract.

## Solution Assessment

Unclear. A configurable allowlist is plausible, but the best solution needs maintainer/security judgment between the broad tools.fs.roots contract and a narrower media-only or iMessage-root contract.
