## Summary

Keep open. Current main still lacks a public configurable local-media root allowlist for image/PDF media tools, and the open related PRs overlap but are not merged or settled enough to supersede this canonical product/security discussion.

## What This Changes

The issue asks for an opt-in `mediaLocalRoots`-style config field so image/media tools can read trusted local directories such as iMessage attachment roots without copying files into the workspace.

## Best Possible Solution

Define one shared, opt-in local media/filesystem root contract with narrow validation, schema/types/docs, image/PDF/media-loader wiring, and tests that preserve managed-inbound and `workspaceOnly` protections.

## Reproduction Assessment

Yes, for the current limitation. Source and regression tests show outside-workspace local image/PDF paths are rejected and current schemas expose no public extra-root config; I did not run a live iMessage flow in this read-only review.

## Solution Assessment

Unclear. A configurable allowlist is plausible, but maintainers need to decide whether the canonical contract is media-tool roots, iMessage attachment-root inheritance, or the broader `tools.fs.roots` policy.
