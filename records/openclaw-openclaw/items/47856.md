## Summary

Keep open. Current main and v2026.6.6 partially address iMessage attachment handling through managed inbound media and channel-scoped iMessage roots, but they still do not provide the requested public configurable local-media-root contract; that remaining work is a filesystem-access product/security decision.

## What This Changes

The issue requests a configurable media local-root allowlist for the image tool so trusted attachment directories can be read without copying files into the workspace.

## Best Possible Solution

Define one scoped, explicit filesystem/media-root contract that preserves safe defaults, validates non-root paths, documents agent/channel/workspace-only semantics, and routes image/PDF/media loading through the shared policy.

## Reproduction Assessment

Yes, at source level. Current main still lacks a public configured-root field, the image tool uses the shared resolver directly, and `workspaceOnly` disables channel inbound roots; I did not run live iMessage because this was a read-only review.

## Solution Assessment

No for the exact image-only knob until maintainers settle the security contract. A shared scoped filesystem/media-root policy is safer than splitting the same read boundary across per-tool allowlists.
