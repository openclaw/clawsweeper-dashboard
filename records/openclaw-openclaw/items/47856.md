## Summary

Keep open. Current main has partial coverage for iMessage attachment roots when the image tool has matching channel/account context and for managed inbound media, but it still does not provide a supported configurable local-root contract for image/media tools; that remaining work is a security-sensitive product/config decision.

## What This Changes

Request to add a configurable local media root allowlist for the image tool so trusted attachment directories outside the default roots can be read without copying files into the workspace.

## Best Possible Solution

Define one scoped local-media access contract that preserves workspace-only isolation by default and makes trusted-root widening agent/channel-bound, schema-backed, documented, and regression-tested.

## Reproduction Assessment

Yes at source level: current `main` still has no public configured local-root field, returns only workspace roots when workspace-only is active, and disables channel inbound roots in that mode. I did not run tests because this review is read-only.

## Solution Assessment

No, the proposed broad `tools.image.mediaLocalRoots` knob is not clearly the safest final shape. A shared scoped filesystem/media-root policy would avoid splitting the same security boundary across per-tool knobs.
