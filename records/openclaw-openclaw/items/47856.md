## Summary

Keep open. Current main and the latest release still lack a supported configurable local-media-root contract for the image/media tools; the remaining work is a security-sensitive product decision, not stale cleanup.

## What This Changes

The issue asks for an opt-in `tools.image.mediaLocalRoots` or similar config field so image/media tools can read trusted local attachment directories outside OpenClaw-managed roots.

## Best Possible Solution

Define one scoped, schema-backed local filesystem/media-root contract that preserves workspace-only isolation by default, validates non-root host paths, is channel/agent-aware, and covers image/media tools with focused regression tests.

## Reproduction Assessment

Yes, at source level. Current main has no public local-root config field, the image tool uses the shared resolver directly, and `workspaceOnly` disables channel inbound roots; I did not run live iMessage in this read-only review.

## Solution Assessment

No for the exact image-only knob until maintainers settle the security contract. A shared scoped filesystem/media-root policy is safer than splitting the same read boundary across per-tool allowlists.
