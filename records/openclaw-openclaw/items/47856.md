## Summary

Current main still lacks a public configurable local-media root allowlist for image/PDF media tools: the tools still use fixed default/workspace roots, and the schema/types expose no `mediaLocalRoots`-style config. The related iMessage and broad `tools.fs.roots` PRs are still open rather than merged, so this remains a live product/security decision.

## What This Changes

The issue requests a configurable media local-root allowlist so image/media tools can read trusted external attachment directories such as iMessage Messages Attachments without manually copying files.

## Best Possible Solution

Define one shared, opt-in local media/filesystem root contract with explicit narrow validation, schema/types/docs, image/PDF/media-loader wiring, and tests that preserve managed-inbound and workspaceOnly protections.

## Reproduction Assessment

Yes, for the current limitation at source level. Current main routes image/PDF local paths through fixed roots and tests still reject outside-workspace image paths, while schemas/types expose no public extra-root config.

## Solution Assessment

Unclear. A configurable allowlist is plausible, but maintainers need to decide whether the canonical solution is media-tool roots, iMessage attachment-root inheritance, or the broader tools.fs.roots policy.
