## Summary

Keep open. Current main still lacks a public image-tool local media root allowlist; the managed inbound-media exemption covers OpenClaw-owned media paths, but not external iMessage attachment directories, and the remaining work needs maintainer/security judgment.

## What This Changes

Request adding a `tools.image.mediaLocalRoots`-style config field so the image tool can read configured external attachment directories such as iMessage attachments.

## Best Possible Solution

Define and document a narrow opt-in media root contract, or explicitly route trusted channel attachment roots into the image/media tool resolver, while preserving managed inbound-media behavior and existing external-path rejection coverage.

## Reproduction Assessment

Yes. Source inspection plus current tests give a high-confidence path: call the image tool with an absolute local image outside the workspace/default roots and it rejects with the allowed-directory error.

## Solution Assessment

Unclear. A configurable allowlist is plausible, but the best maintainable fix needs a security/product decision on root validation, `workspaceOnly` semantics, and whether #39034's channel-owned iMessage attachment roots should be the narrower boundary.
