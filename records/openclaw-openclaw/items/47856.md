## Summary

Keep open. Current main still lacks a public configurable media local-root allowlist for image/PDF/media tools; supporting comments also show the workspaceOnly and Docker-staging pressure, while current managed-inbound handling only partially covers OpenClaw-owned inbound media and not external iMessage attachment directories.

## What This Changes

Request adding a `tools.image.mediaLocalRoots`-style config field so image/media tools can read explicitly allowed local attachment directories outside the default roots.

## Best Possible Solution

Choose one canonical explicit-root policy, wire it through shared media root resolution, schema/types/docs, and image/PDF tests, while preserving managed inbound-media and workspaceOnly semantics.

## Reproduction Assessment

Yes, for the current limitation: source and tests show an external local image path reaches the image tool and is checked against default/workspace roots only. This proves the gap, but the requested allowlist is a feature/config addition rather than a broken documented contract.

## Solution Assessment

Unclear. A configurable allowlist is plausible, but the best solution needs maintainer/security judgment between broad `tools.fs.roots`, channel attachment-root inheritance, and a narrower media-only root contract.
