## Summary

Keep open. Current main still lacks a public configurable local-media root allowlist for the image/PDF media tools, and the remaining work is a security-sensitive product/config decision with overlapping open implementation paths rather than cleanup.

## What This Changes

Add a configurable local media root allowlist for image/media tools so outside-workspace attachments such as iMessage files can be read without copying them into the workspace.

## Best Possible Solution

Define one canonical opt-in local media root policy, validate only narrow explicit roots, thread it through shared media root resolution and loadWebMedia for image/PDF, document it, and preserve managed inbound-media plus workspaceOnly boundaries.

## Reproduction Assessment

Yes, for the current limitation. Source and tests show an outside-workspace local image path is checked against default/workspace roots only and rejected; I did not run a live iMessage flow in this read-only review.

## Solution Assessment

Unclear. A configurable allowlist is plausible, but maintainers need to decide whether the canonical contract is media-tool roots, iMessage attachment-root inheritance, or the broader tools.fs.roots policy.
