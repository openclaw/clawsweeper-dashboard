## Summary

Keep open, but do not merge as-is. Current main still lacks the embedded Pi, queue, and subagent cleanup in `chat.abort`, so the bug-fix direction is valid; this branch makes that cleanup session-wide in paths that currently have run-id and owner-isolation contracts, and it still lacks required real behavior proof.

## What This Changes

This PR imports embedded Pi abort, queue cleanup, and subagent stop helpers into `chat.abort` and adds mocked gateway tests for session-key and run-id abort paths.

## Best Possible Solution

Implement scoped `chat.abort` cleanup that only affects authorized aborted targets, preserves run-id and owner isolation, and proves the WebChat Stop path with focused tests plus redacted runtime evidence.

## Reproduction Assessment

Yes, at source level: WebChat Stop routes to `chat.abort`, current main lacks embedded Pi, queue, and subagent cleanup in that handler, and `/stop` uses those cleanup primitives. I did not run a live WebChat reproduction in this read-only review.

## Solution Assessment

No. The cleanup goal is right, but copying session-wide `/stop` cleanup into both `chat.abort` branches is not the narrowest maintainable fix because those branches already have run-id and owner-filtered contracts.

## Review Findings

Full review comments:

- **[P1] Keep run-id cleanup scoped to the targeted run:** `src/gateway/server-methods/chat.ts:1227-1232`
  - body: After aborting a single `runId`, this also aborts the embedded session and clears queues for the whole session. If another prompt is active or queued on the same session, canceling one run can drop or abort unrelated work the caller did not target.
  - confidence: 0.88
