## Summary

Keep open. The PR targets a real Stop-vs-/stop gap, but the diff turns narrower chat.abort paths into whole-session cleanup and can interrupt unrelated same-session or mixed-owner work.

## What This Changes

The PR adds embedded Pi abort, queue clearing, subagent stopping, and regression tests to the chat.abort sessionKey and runId paths.

## Best Possible Solution

Define a cleanup contract that preserves runId and owner isolation, then add scoped embedded-run, queue, and subagent cleanup with concurrent and mixed-owner regression coverage.

## Reproduction Assessment

Yes. A source-level reproduction can create two active or queued runs in one session, or mixed-owner active runs, then abort one run or one owner and observe that the PR’s session-keyed cleanup can clear or stop work outside the authorized target.

## Solution Assessment

No. Directly mirroring /stop is unsafe because /stop is session-targeted, while chat.abort(runId) and owner-filtered session aborts are narrower; the safer fix needs run-scoped cleanup or last-authorized-run semantics.
