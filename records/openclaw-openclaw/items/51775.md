## Summary

Keep open: current `main` still has the WebChat Stop cleanup gap, but this PR is not merge-ready because it broadens scoped aborts into session-wide cleanup and still lacks real behavior proof.

## What This Changes

The PR adds embedded Pi abort, session queue clearing, and subagent stopping to both `chat.abort` RPC abort paths, plus focused mocked tests for session-key and run-id aborts.

## Best Possible Solution

Rebase and implement boundary-aware cleanup that preserves run-id and owner scopes, with mixed-owner and concurrent-run regression tests plus redacted real WebChat Stop proof.

## Reproduction Assessment

Yes at source level: WebChat Stop calls `chat.abort`, current `main` lacks the embedded Pi, queue, and subagent cleanup helpers in that RPC handler, while the stop-command path uses them. I did not run a live WebChat reproduction in this read-only review.

## Solution Assessment

No. Mirroring `/stop` is useful for a full session stop, but applying the same session-level cleanup after a run-id abort or partial owner-filtered abort conflicts with current run and owner scopes.
