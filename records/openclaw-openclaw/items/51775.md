## Summary

Keep open: the underlying stop-button gap is valid, but this patch turns targeted and owner-filtered aborts into whole-session cleanup, which conflicts with current authorization and queue cleanup contracts.

## What This Changes

The PR adds embedded Pi run abort, queue clearing, and subagent stop calls to both `chat.abort` paths, plus tests for those cleanup calls.

## Best Possible Solution

Define an authorized cleanup contract first, then add cleanup only when it is run-scoped or when all affected session work is proven safe to stop, with regression tests for concurrent runs and mixed-owner sessions.

## Reproduction Assessment

Yes. A source-level reproduction can create two active or queued runs in one session, abort only one run or one device owner's run, and observe that the PR's session-keyed cleanup would clear or stop work outside the authorized target.

## Solution Assessment

No. Mirroring `/stop` directly is not the best fix because `/stop` is session-targeted, while `chat.abort(runId)` and owner-filtered session aborts have narrower authorization and scoping semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep runId cleanup scoped to the targeted run:** `src/gateway/server-methods/chat.ts:1227-1231`
  - body: The runId branch aborts one authorized controller, then calls session-keyed cleanup helpers for the whole session. If another active or queued prompt shares that session, canceling one run can clear queues or stop embedded work that belongs to another run. Gate this on last-active-run semantics or add a genuinely run-scoped cleanup path.
  - confidence: 0.93
- **[P1] Preserve owner isolation for session abort cleanup:** `src/gateway/server-methods/chat.ts:1176-1184`
  - body: `abortChatRunsForSessionKeyWithPartials` only aborts run IDs authorized for the requester, but the added cleanup runs for the whole session once any owned run was aborted. In a mixed-owner session, that can clear queues or stop subagents for another device's still-active work.
  - confidence: 0.91
