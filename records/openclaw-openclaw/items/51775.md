## Summary

Keep this PR open, but not mergeable as-is: it targets a real Stop/chat.abort gap, yet the patch makes run-id and owner-filtered aborts perform session-wide cleanup and still lacks after-fix real behavior proof.

## What This Changes

The PR adds embedded Pi abort, sub-agent stop, and session queue cleanup calls to both `chat.abort` RPC paths and adds focused unit coverage for those calls.

## Best Possible Solution

Keep the real `chat.abort` cleanup goal, but make cleanup respect abort scope: run-id aborts must not clear unrelated same-session work, and session-key aborts must only perform session-wide cleanup when the requester is authorized to stop the whole affected session.

## Reproduction Assessment

Yes, at source level: current UI and docs route Stop to `chat.abort`, while current `chat.abort` lacks the cleanup calls that `/stop` uses. I did not establish a live webchat reproduction in this read-only pass.

## Solution Assessment

No. Directly mirroring `/stop` in both RPC abort paths is too broad because `chat.abort(runId)` is run-scoped and session-key aborts are owner-filtered.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep run-id cleanup scoped to the targeted run:** `src/gateway/server-methods/chat.ts:1228-1230`
  - body: After a successful `chat.abort` by `runId`, this calls `abortEmbeddedPiRun(active.sessionId)` and clears queues for the whole session. If another prompt is active or queued on the same session, canceling one run can now abort or drop unrelated work that the caller did not target.
  - confidence: 0.88
- **[P1] Preserve owner isolation for session-key cleanup:** `src/gateway/server-methods/chat.ts:1177-1185`
  - body: The session-key path first aborts only runs authorized for the requesting client, but this cleanup still aborts the embedded session, clears queues, and stops sub-agents for the whole session key. That bypasses the existing mixed-owner contract where another device's active run remains alive.
  - confidence: 0.86
