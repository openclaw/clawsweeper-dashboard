## Best Possible Solution

Implement scoped `chat.abort` cleanup that only affects authorized aborted targets, preserves run-id and owner isolation, and proves WebChat Stop with focused tests plus redacted runtime evidence.

## Reproduction Assessment

Yes, at source level: WebChat Stop calls `chat.abort`, current main’s RPC handler lacks the embedded Pi, queue, and subagent cleanup helpers, and `/stop` uses those helpers. I did not run a live WebChat reproduction in this read-only review.

## Solution Assessment

No. The cleanup goal is valid, but copying session-wide `/stop` cleanup into run-id and owner-filtered RPC branches is not the narrowest maintainable fix because those branches already have narrower targeting and authorization contracts.
