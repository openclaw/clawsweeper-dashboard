## Summary

Keep open: the Stop-button bug is real and current main still lacks the requested embedded-run cleanup, but this PR’s cleanup is session-scoped and can violate existing runId and owner isolation semantics.

## What This Changes

The PR adds embedded Pi abort, session queue clearing, and subagent stopping to both `chat.abort` sessionKey and runId paths, plus tests for those cleanup calls.

## Best Possible Solution

Define a cleanup contract that preserves runId and owner isolation, then add embedded-run and queue cleanup only when scoped to the targeted run or proven safe for all remaining session work, with regression coverage for concurrent and mixed-owner sessions.

## Reproduction Assessment

Yes. A source-level reproduction can create two active or queued runs in one session, or mixed-owner active runs, then abort one run or one owner and observe that the PR’s session-keyed cleanup can clear or stop work outside the authorized target.

## Solution Assessment

No. Directly mirroring `/stop` is unsafe because `/stop` is session-targeted, while `chat.abort(runId)` and owner-filtered session aborts are narrower; the safer fix needs run-scoped cleanup or last-authorized-run semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep runId cleanup scoped to the targeted run:** `src/gateway/server-methods/chat.ts:1227-1231`
  - body: The runId branch aborts one authorized controller, then calls session-keyed cleanup helpers for the whole session. If another active or queued prompt shares that session, canceling one run can clear queues or stop embedded work that belongs to another run. Gate this on last-active-run semantics or add a genuinely run-scoped cleanup path.
  - confidence: 0.93
- **[P1] Preserve owner isolation for session abort cleanup:** `src/gateway/server-methods/chat.ts:1176-1184`
  - body: `abortChatRunsForSessionKeyWithPartials` only aborts run IDs authorized for the requester, but the added cleanup runs for the whole session once any owned run was aborted. In a mixed-owner session, that can clear queues or stop subagents for another device’s still-active work.
  - confidence: 0.91
