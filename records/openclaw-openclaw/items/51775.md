## Summary

Keep open. The PR targets a valid WebChat Stop bug, but the current diff broadens cleanup beyond existing run-id and owner-filtered abort contracts, the author acknowledged the scope concern, and no after-fix real behavior proof is present.

## What This Changes

The PR imports embedded Pi abort, queue cleanup, and subagent stop helpers into `chat.abort` and adds mocked gateway tests for the session-key and run-id abort paths.

## Best Possible Solution

Implement scoped `chat.abort` cleanup that only affects authorized aborted targets, preserves run-id and owner isolation, and proves the WebChat Stop path with focused tests plus redacted runtime evidence.

## Reproduction Assessment

Yes, at source level: the UI Stop button routes to `chat.abort`, current main lacks embedded Pi/queue/subagent cleanup in that handler, and `/stop` uses those cleanup primitives. I did not run a live WebChat reproduction in this read-only review.

## Solution Assessment

No. The cleanup goal is right, but copying session-level `/stop` cleanup into both `chat.abort` branches is not the narrowest maintainable fix because `chat.abort` already has run-id and owner-filtered contracts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep run-id cleanup scoped to the targeted run:** `src/gateway/server-methods/chat.ts:1227-1232`
  - body: After aborting a single `runId`, this also aborts the embedded session and clears queues for the whole session. If another prompt is active or queued on the same session, canceling one run can drop or abort unrelated work the caller did not target.
  - confidence: 0.88
- **[P1] Preserve owner isolation for session-key cleanup:** `src/gateway/server-methods/chat.ts:1176-1185`
  - body: The session-key path first aborts only runs authorized for the requesting client, but this cleanup still clears queues, aborts the embedded session, and stops subagents for the whole session key. That bypasses the mixed-owner behavior where another device's run remains active.
  - confidence: 0.86
