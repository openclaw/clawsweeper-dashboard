## Summary

Keep open: this PR addresses a plausible Stop-button bug, but the current diff turns narrower chat.abort paths into session-wide cleanup that can interrupt unrelated or unauthorized work, and the external PR does not include real behavior proof.

## What This Changes

The PR adds embedded Pi abort, queue clearing, sub-agent stopping, and regression tests to the chat.abort sessionKey and runId paths.

## Best Possible Solution

Define and implement scoped cleanup that preserves runId and requester ownership, with regression coverage for concurrent and mixed-owner sessions plus after-fix real behavior proof.

## Reproduction Assessment

Yes for the PR-introduced regressions: source inspection and existing tests show session-scoped aborts can be partial by owner, while the PR cleanup is whole-session. A focused test can create mixed-owner or multi-run state and observe unrelated queues/sub-agents being stopped.

## Solution Assessment

No. Mirroring /stop is unsafe because /stop is session-targeted, while chat.abort(runId) and owner-filtered session aborts are narrower; the safer solution needs run-scoped cleanup or explicit last-authorized-run semantics.
