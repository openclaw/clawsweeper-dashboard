## Summary

Keep open: this is a focused, well-tested fix for the same-file embedded prompt-window race, and I found no blocking diff defect. The remaining blocker is maintainer acceptance of the hot-path availability tradeoff from the new per-session-file queue.

## What This Changes

The PR adds a process-global per-session-file prompt-turn queue for embedded prompt windows, adds a separate idle-wait release path, and expands session-lock regression coverage.

## Best Possible Solution

Land this as the cross-lane same-file prompt-window fix after maintainer sign-off on the queue/timeout tradeoff, and track the vanilla same-lane race through https://github.com/openclaw/openclaw/pull/86584.

## Solution Assessment
