## Summary

Keep open for maintainer review: the PR is a focused fix for the linked session-file prompt race, current main still lacks a file-scoped prompt-window guard, and I did not find a blocking correctness or security issue in this branch.

## What This Changes

The branch adds a process-local per-resolved-session-file prompt-turn queue to the embedded session-lock controller, adds a non-prompt idle release for compaction waits, and covers same-file and failure cleanup paths with tests.

## Best Possible Solution

Land this file-scoped guard after required checks and maintainer acceptance of the bounded queue risk, then use it as the canonical fix for the linked issue and supersede the older competing PR.

## Reproduction Assessment

Yes at source/proof level: current main releases the session write lock during provider streaming and later fences the file, while the linked issue has live logs of same-file takeover failures. I did not run a live gateway repro in this read-only review.

## Solution Assessment

Yes: serializing prompt windows by resolved session file uses the existing fence key, keeps compaction idle outside the prompt queue, and avoids a broader stuck-recovery policy change. The remaining decision is the maintainer-owned availability tradeoff of queueing same-file prompts.
