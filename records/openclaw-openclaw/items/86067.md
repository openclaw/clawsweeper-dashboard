## Repair Work Prompt

On PR https://github.com/openclaw/openclaw/pull/86067, fix the file-scoped prompt-window guard lifecycle. `EmbeddedAttemptSessionLockController.dispose()` must release and clear any `activePromptSessionTurn` in addition to releasing `heldLock`, so a controller disposed after `releaseForPrompt()` cannot leave same-file waiters blocked. Add a focused regression in `src/agents/pi-embedded-runner/run/attempt.session-lock.test.ts` where `firstController.releaseForPrompt()` is followed by `firstController.dispose()`, then `secondController.releaseForPrompt()` on the same session file completes. Keep the change scoped; do not change config, provider routing, session-fence semantics, or unrelated tests.
