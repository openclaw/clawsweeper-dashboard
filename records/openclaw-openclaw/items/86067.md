## Summary

Keep open: this is an active P1 bugfix PR for an embedded-runner same-session race that current main still does not solve. The patch is focused and has sufficient before/after controller proof with no blocking findings, but maintainers should explicitly accept the availability tradeoff of a process-global prompt-turn queue before merge.

## What This Changes

The branch adds a per-session-file prompt-turn queue in the pi embedded runner, separates compaction idle release from prompt release, and adds regression coverage for same-file serialization and cleanup paths.

## Best Possible Solution

Land the focused prompt-window guard after maintainer review accepts the same-file availability tradeoff and exact-head checks are clean or explicitly waived as unrelated, while keeping the separate same-lane fixes in their own PRs.

## Solution Assessment

Yes, with maintainer acceptance of the availability tradeoff: the patch addresses the race at the session-lock boundary and keeps compaction idle waits out of the prompt queue. The safer alternative would be an explicit bounded wait policy if maintainers do not want provider aborts/timeouts to be the only hung-stream backstop.
