## Summary

Keep this PR open: the current main still lacks the file-scoped prompt-window guard, and the proposed merge result looks focused with no blocking correctness findings. The remaining maintainer decision is whether to accept the availability tradeoff of a process-local prompt-turn queue in the embedded runner hot path.

## What This Changes

The PR adds a per-session-file embedded prompt-window queue, adds a separate idle-wait release path for compaction waits, switches the post-prompt compaction release to that path, and expands session-lock regression coverage.

## Best Possible Solution

Land the focused guard if maintainers accept the bounded serialization tradeoff, while keeping the separate same-lane and queue-drain fixes in their own linked PRs.

## Reproduction Assessment

Yes. The linked issue and PR body provide a concrete same-session-file embedded-controller race, and source inspection confirms current main releases prompt locks without a file-scoped queue; I did not rerun the repro in this read-only review.

## Solution Assessment

Yes for the cross-lane same-file race. The fix is scoped to the resolved session JSONL prompt window, preserves the takeover fence as the safety net, and separates non-prompt compaction idle waits from real provider prompt turns.
