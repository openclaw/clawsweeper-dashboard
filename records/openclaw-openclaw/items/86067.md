## What This Changes

Adds a per-session-file prompt-turn queue to the embedded attempt session-lock controller, splits compaction idle waits onto `releaseForSessionIdleWait`, and adds regression tests for serialization and cleanup paths.

## Best Possible Solution

Land this focused cross-lane prompt-window guard after maintainers accept the availability tradeoff and exact-head gates, while leaving the separately scoped same-lane fixes to their own linked PRs.

## Reproduction Assessment

Yes: current main still releases prompt locks without a file-scoped guard, and the PR body provides a before/after controller reproduction showing the race before the patch and delayed second-lane delivery after it. I did not rerun the proof locally in this read-only review.

## Solution Assessment

Yes for the cross-lane same-session file race: the change is narrow, keeps the existing takeover fence as the safety net, and separates compaction waits from real prompt windows. It is not the complete solution for the same-lane races, which are correctly split into separate linked PRs.
