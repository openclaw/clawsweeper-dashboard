## What This Changes

The PR adds a per-session-file prompt-window queue for embedded runner locks, introduces a non-prompt idle-wait release path, and adds session-lock regression coverage.

## Best Possible Solution

Land the guard after dispose also releases any active prompt turn and a regression proves teardown cannot wedge same-file waiters.

## Solution Assessment

No, not yet. The per-file queue is the right fix shape, but the lifecycle invariant is incomplete until disposal participates in the same prompt-turn cleanup contract.
