## Summary

Keep this PR open for maintainer review rather than cleanup-close. I found no blocking correctness finding, and the supplied proof is strong, but the new process-global same-file prompt queue sits in the embedded-runner hot path and still deserves an explicit availability-risk decision before merge.

## What This Changes

Adds a per-session-file prompt-turn queue to serialize embedded-runner prompt releases, separates compaction idle waits from prompt releases, and adds focused session-lock regression tests.

## Best Possible Solution
