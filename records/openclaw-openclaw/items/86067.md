## Summary

Keep open: the PR is still useful and current main does not already implement the file-scoped prompt-window guard. I found no blocking correctness finding, but the new queue is availability-sensitive and still needs maintainer acceptance plus normal merge gates.

## What This Changes

This PR adds a process-global prompt-window queue keyed by resolved embedded-session JSONL file, separates non-prompt idle waits from prompt-window releases, and adds regression coverage for same-file serialization and cleanup paths.

## Best Possible Solution

Land the narrow queue-based fix after maintainer acceptance of the bounded availability tradeoff, while keeping the separate same-lane race work in https://github.com/openclaw/openclaw/pull/86584 and https://github.com/openclaw/openclaw/pull/86585 separate.
