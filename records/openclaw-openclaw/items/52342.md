## Summary

Keep open, but do not merge as-is: the Feishu topic-group bug is source-reproducible, while this PR's prefix-only classifier can regress DM policy and native topic-group handling and still lacks real Feishu proof.

## What This Changes

This PR adds a Feishu group-chat classifier for `p2p` `oc_` chat IDs, applies it to inbound routing and mention-forward detection, and updates Feishu tests/fixtures.

## Best Possible Solution

Land a revised Feishu-owned classifier that uses verified topic/OpenChat metadata, preserves `topic_group` and true DM/card-action behavior, and keeps outbound session consistency tracked by https://github.com/openclaw/openclaw/issues/58537.

## Reproduction Assessment

Yes, by source inspection. A synthetic Feishu event with `chat_type: "p2p"`, an `oc_...` chat id, and `root_id` or `thread_id` reaches current main with `isGroup === false` because routing consults only `isFeishuGroupChatType(ctx.chatType)`.

## Solution Assessment

No. The PR touches the right Feishu call sites, but the bare `oc_` prefix rule is overbroad and the helper drops native `topic_group`; a safer fix should use verified topic/OpenChat metadata in a shared classifier.
