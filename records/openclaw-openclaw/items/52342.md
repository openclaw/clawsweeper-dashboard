## Summary

Keep open: this PR targets a real Feishu topic-group routing bug, but the branch is not safe to merge because it classifies every `p2p` `oc_` chat as group traffic and would drop current `topic_group` handling on an authorization/session-routing boundary.

## What This Changes

The PR adds `isFeishuGroupChat`, wires it into Feishu inbound routing and mention forwarding, adds helper tests, and rewrites Feishu DM fixtures from `oc_` IDs to `p2p_` IDs.

## Best Possible Solution

Revise the Feishu-owned classifier to distinguish actual OpenChat/topic-group webhook events from true DMs using verified Feishu metadata, preserve native `topic_group`, add regression coverage and a changelog entry, and leave outbound consistency tracked in #58537.

## Reproduction Assessment

Yes. Source inspection shows current main routes `chat_type: "p2p"` through the direct path in both `bot.ts` and `mention.ts`, matching the linked #52238 logs; the PR regressions are visible from its proposed helper and fixture rewrites.

## Solution Assessment

No. The PR changes the right Feishu-owned surfaces, but `p2p` plus `oc_` is too broad and the helper omits `topic_group`; a verified topic/OpenChat discriminator with preserved DM behavior is the safer fix.
