## Summary

Keep this PR open: it targets a real Feishu routing bug, but the current patch is not safe to merge because the prefix-only classifier can misroute protected DM paths and drops native `topic_group` handling, and the contributor proof is only unit-test output.

## What This Changes

The PR adds a Feishu helper that treats `p2p` `oc_` chat IDs as group chat, applies it in inbound routing and mention-forward detection, and updates Feishu tests/fixtures.

## Best Possible Solution

Land a revised Feishu-owned classifier that uses verified topic/OpenChat metadata, preserves native `topic_group` and true DM/card-action behavior, and leaves outbound session consistency to the canonical follow-up.

## Reproduction Assessment

Yes, by source inspection. A synthetic Feishu event with `chat_type: "p2p"`, an `oc_...` chat ID, and `root_id` or `thread_id` reaches current main with `isGroup === false` because routing consults only `isFeishuGroupChatType(ctx.chatType)`.

## Solution Assessment

No. The PR touches the right Feishu call sites, but the bare `oc_` prefix rule is overbroad and misses native `topic_group`; a safer fix should share a verified metadata-backed classifier across routing and mention-forward detection.
