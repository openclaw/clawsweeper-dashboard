## Summary

Keep this PR open but do not merge it as-is: it targets a real Feishu topic-group routing bug, but the branch changes an authorization-sensitive direct/group classifier too broadly and lacks real behavior proof.

## What This Changes

The PR adds a Feishu chat-type helper, uses it in inbound message routing and mention-forward detection, adds helper tests, and rewrites several Feishu DM fixtures from `oc_` IDs to `p2p_` IDs.

## Best Possible Solution

Land a Feishu-owned classifier that keeps `topic_group` as group, distinguishes anomalous topic/OpenChat webhook events from true DMs using verified topic metadata or Feishu chat metadata, and leaves the separate outbound session-key mismatch to https://github.com/openclaw/openclaw/issues/58537.

## Reproduction Assessment

Yes, from source inspection. A synthetic Feishu event with `chat_type: "p2p"`, an `oc_...` chat ID, and topic metadata reaches current main with `isGroup === false`, while the PR regressions are visible from the proposed helper branches against current tests.

## Solution Assessment

No. The PR touches the right Feishu-owned surfaces, but classifying all `p2p` + `oc_` events as groups is broader than the current DM safeguards and the helper must preserve native `topic_group` behavior.
