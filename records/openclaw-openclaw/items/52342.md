## Summary

Keep open. Current main still has the Feishu topic-group `p2p` routing bug described by the linked report, but this PR is not merge-ready because the prefix-only classifier can regress existing `p2p` `oc_` direct/card-action paths and drops current `topic_group` handling.

## What This Changes

The PR adds a Feishu group-chat classifier, wires it into inbound routing and mention-forward detection, adds helper tests, and rewrites Feishu DM fixtures from `oc_` IDs to `p2p_` IDs.

## Best Possible Solution

Land a Feishu-owned classifier that preserves `topic_group`, distinguishes anomalous `p2p` OpenChat topic events from true direct traffic with a verified signal, applies it consistently to inbound routing and mention-forward detection, and records the user-facing fix in `CHANGELOG.md`; keep #58537 for outbound/session consistency.

## Reproduction Assessment

Yes, source-reproducible. The linked bug report provides live steps and logs for a Feishu topic group reported as `chat_type: "p2p"`, and current main routes that shape through the direct path; the PR regressions are also visible from the helper predicate and existing tests.

## Solution Assessment

No. The PR touches the right Feishu-owned surfaces, but `p2p` plus `oc_` is too broad and the helper omits current `topic_group` behavior; a verified topic-group discriminator is the safer maintainable fix.
