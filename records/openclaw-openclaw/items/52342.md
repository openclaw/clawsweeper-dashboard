## Summary

Keep open: this PR targets a real Feishu routing bug, but the current patch is not merge-ready because its prefix-only classifier can regress current DM/card-action safeguards, drops native `topic_group` handling, and still lacks real Feishu behavior proof.

## What This Changes

The PR adds a Feishu chat-type helper, applies it to inbound routing and mention-forward detection, adds helper tests, and rewrites Feishu DM fixtures from `oc_` IDs to `p2p_` IDs.

## Best Possible Solution

Land or replace this with a Feishu-owned shared classifier that uses verified topic/OpenChat metadata, preserves native `topic_group` and restrictive DM/card-action paths, and leaves outbound session consistency tracked in https://github.com/openclaw/openclaw/issues/58537.

## Reproduction Assessment

Yes, by source inspection. A Feishu event with `chat_type: "p2p"`, an `oc_...` chat id, and topic metadata reaches current main with `isGroup === false` because routing consults only `isFeishuGroupChatType(ctx.chatType)`.

## Solution Assessment

No. The PR touches the right call sites, but the bare `oc_` prefix rule is overbroad and the helper drops native `topic_group`; a safer fix should use verified topic/OpenChat metadata in a shared classifier.
