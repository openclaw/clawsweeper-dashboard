## Summary

Keep open: current main still has the Feishu `p2p` OpenChat/topic-group inbound routing gap, but this branch is not merge-ready because the prefix-only classifier conflicts with shipped `oc_` p2p DM/card-action safeguards, drops native `topic_group` handling, lacks real Feishu proof, and is currently conflicting.

## What This Changes

The PR adds a Feishu chat classification helper, uses it in inbound routing and mention-forward detection, adds helper tests, and changes Feishu DM fixtures from `oc_` IDs to `p2p_` IDs.

## Best Possible Solution

Land a Feishu-owned classification path that distinguishes actual OpenChat/topic-group webhook events from true DMs using verified event or chat metadata, then apply it consistently to inbound routing and mention-forward detection while keeping outbound session-key consistency aligned with https://github.com/openclaw/openclaw/issues/58537.

## Reproduction Assessment

Yes, by source inspection. A synthetic Feishu event with `chat_type: "p2p"`, an `oc_...` chat id, and `root_id` or `thread_id` still reaches current main with `isGroup === false` because routing consults only `isFeishuGroupChatType(ctx.chatType)`.

## Solution Assessment

No. The PR touches the right call sites, but the bare `oc_` prefix rule is overbroad and the helper drops native `topic_group`; a safer fix should use verified topic/OpenChat metadata in a shared classifier.
