## Summary

Keep open. The reported Feishu topic-group routing bug is still valid on current main, but this PR is not merge-ready because the prefix-only classifier can misclassify existing p2p `oc_` direct-message/card-action traffic and would also drop current `topic_group` support.

## What This Changes

The PR adds a Feishu group-chat helper, uses it in inbound routing and mention-forward detection, adds helper tests, and changes Feishu DM fixtures from `oc_` IDs to `p2p_` IDs.

## Best Possible Solution

Land a Feishu-owned classifier that preserves `topic_group`, identifies anomalous topic/open-chat `p2p` events with a verified signal such as Feishu chat metadata or topic/root fields, and applies that classifier consistently to inbound routing and mention-forward detection with regression tests, docs, and changelog coverage. Coordinate outbound session-route consistency with https://github.com/openclaw/openclaw/issues/58537 rather than relying on a bare prefix rule.

## Reproduction Assessment

Yes. The paired bug report provides live steps and logs for a Feishu topic group reported as `chat_type: "p2p"`, and current main code shows that event shape still routes as direct. The PR regressions are also code-reproducible from the helper predicate plus current p2p `oc_` DM/card-action tests.

## Solution Assessment

No. The PR changes the right Feishu-owned surfaces, but `p2p` plus `oc_` is too broad and the helper omits current `topic_group` behavior; a verified topic-group discriminator is the safer maintainable fix.
