## Summary

Keep this PR open, but it is not ready to merge: current main still has the reported Feishu `p2p` topic-group routing gap, while this branch uses an over-broad `oc_` prefix rule, drops native `topic_group` handling, and has only test/CI proof rather than real Feishu behavior proof.

## What This Changes

The PR adds a Feishu group-chat helper, routes bot and mention-forward classification through it, adds helper tests, and rewrites several Feishu DM fixtures from `oc_` to `p2p_` IDs.

## Best Possible Solution

Land a Feishu-owned classifier that uses verified topic/OpenChat evidence, preserves native `topic_group` and true DM/card-action behavior, covers bot routing plus mention-forward, and leaves outbound session consistency tracked in https://github.com/openclaw/openclaw/issues/58537.

## Reproduction Assessment

Yes, by source inspection. A synthetic Feishu event with `chat_type: "p2p"`, an `oc_...` chat ID, and topic metadata reaches current main with `isGroup === false` because routing consults only `isFeishuGroupChatType(ctx.chatType)`.

## Solution Assessment

No. The PR touches the right Feishu-owned surfaces, but a bare `p2p` plus `oc_` prefix is too broad and the helper omits current `topic_group` support; a verified topic/OpenChat signal is the safer fix.
