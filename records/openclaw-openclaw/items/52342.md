## Summary

Keep open. The Feishu topic-group routing bug is real and this PR touches the right inbound surfaces, but the proposed classifier is not merge-ready: it treats every `chat_type: "p2p"` plus `oc_` chat ID as group traffic, while current main still preserves `p2p` `oc_` direct-message/card-action paths and already treats native `topic_group` as group traffic.

## What This Changes

The PR adds a Feishu chat classification helper, applies it to bot routing and mention-forward detection, adds helper tests, and rewrites several Feishu DM fixtures from `oc_` IDs to `p2p_` IDs.

## Best Possible Solution

Revise the Feishu-owned classification path so actual OpenChat/topic-group webhook events are recognized with a verified signal such as Feishu chat metadata, `thread_id`, `root_id`, or a documented event field, then apply that centralized classifier to inbound routing and mention-forward detection while preserving current `p2p` direct-message behavior and coordinating outbound consistency with #58537.

## Reproduction Assessment

Yes. The paired bug #52238 gives concrete steps and logs for a Feishu topic group reported as `chat_type: "p2p"`, and current main source confirms that such events still follow the direct path. The PR regressions are also reproducible from current tests that preserve `p2p` `oc_` DM/card-action behavior and from the PR helper’s own predicate.

## Solution Assessment

No. The PR targets the right boundary, but `p2p` plus `oc_` is an overbroad rule against current shipped DM/card-action policy and it misses `topic_group`; a narrower verified topic-group discriminator is safer.
