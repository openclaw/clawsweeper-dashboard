## Summary

Keep open: the PR targets a real source-reproducible Feishu routing bug, but the proposed prefix-only classifier can regress DM/security-sensitive paths, misses current `topic_group` semantics, lacks real Feishu behavior proof, and is currently conflicting.

## What This Changes

The PR adds a Feishu chat-type helper, uses it in inbound routing and mention-forward detection, adds helper tests, and rewrites Feishu DM test fixtures away from `oc_` chat IDs.

## Best Possible Solution

Land a Feishu-owned shared classifier that distinguishes actual OpenChat/topic-group webhook events from true DMs using verified event or chat metadata, preserves native `topic_group`, and keeps outbound session consistency aligned with https://github.com/openclaw/openclaw/issues/58537.

## Reproduction Assessment

Yes, by source inspection. A synthetic Feishu event with `chat_type: "p2p"`, an `oc_...` chat id, and `root_id` or `thread_id` still reaches current main with `isGroup === false` because routing consults only `isFeishuGroupChatType(ctx.chatType)`.

## Solution Assessment

No. The PR touches the right call sites, but the bare `oc_` prefix rule is overbroad and the helper drops native `topic_group`; a safer fix should use verified topic/OpenChat metadata in a shared classifier.
