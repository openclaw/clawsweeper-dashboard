## Summary

Keep open: this PR targets a real Feishu routing bug, but the current prefix-only classifier is not merge-ready because it can regress protected `p2p`/`private` `oc_` DM paths, drops current `topic_group` semantics, and still lacks real Feishu behavior proof.

## What This Changes

The PR adds a Feishu chat-classification helper that treats `p2p` `oc_` chat IDs as groups, wires it into inbound routing and mention-forward detection, and updates Feishu tests/fixtures.

## Best Possible Solution

Land a revised Feishu-owned classifier that uses verified topic/OpenChat metadata, preserves `topic_group` and true DM/card-action behavior, and keeps outbound session consistency tracked by https://github.com/openclaw/openclaw/issues/58537.

Yes, by source inspection. A synthetic Feishu event with `chat_type: "p2p"`, an `oc_...` chat id, and `root_id` or `thread_id` still reaches current main with `isGroup === false` because routing consults only `isFeishuGroupChatType(ctx.chatType)`.

## Solution Assessment

No. The PR touches the right call sites, but the bare `oc_` prefix rule is overbroad and the helper drops native `topic_group`; a safer fix should use verified topic/OpenChat metadata in a shared classifier.
