## Summary

Keep open: this is a real Feishu inbound routing bug, but the current PR is not merge-ready because the prefix-only classifier can regress protected DM paths, drops the existing native topic-group type, lacks real Feishu proof, and the live PR is conflicting.

## What This Changes

The PR adds a Feishu group-chat helper, applies it to inbound routing and mention-forward detection, adds helper tests, and rewrites Feishu DM fixtures from `oc_` IDs to `p2p_` IDs.

## Best Possible Solution

Land a Feishu-owned shared classifier that distinguishes actual OpenChat/topic-group webhook events from true DMs using verified event or chat metadata, then apply it consistently to inbound routing and mention-forward detection while keeping outbound session consistency aligned with https://github.com/openclaw/openclaw/issues/58537.

## Reproduction Assessment

Yes, by source inspection. A synthetic Feishu event with `chat_type: "p2p"`, an `oc_...` chat id, and topic metadata still reaches current main with `isGroup === false` because routing consults only `isFeishuGroupChatType(ctx.chatType)`.

## Solution Assessment

No. The PR touches the right call sites, but the bare `oc_` prefix rule is overbroad and the helper drops native `topic_group`; a safer fix should use verified topic/OpenChat metadata in a shared classifier.
