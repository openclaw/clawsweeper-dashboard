## Summary

Keep open for maintainer follow-up: this PR targets a real Feishu topic-group routing bug, but the current branch is not merge-ready because the prefix-only classifier can regress protected DM/card-action paths and lacks real Feishu behavior proof.

## What This Changes

The branch adds a Feishu group-chat classifier for `p2p` `oc_` chat IDs, uses it in inbound routing and mention-forward detection, adds helper tests, and rewrites Feishu DM fixtures from `oc_` to `p2p_`.

## Best Possible Solution

Land a Feishu-owned shared classifier that uses verified topic/OpenChat metadata, preserves native `topic_group` and true DM behavior, covers inbound routing plus mention-forward, and leaves outbound session consistency aligned with https://github.com/openclaw/openclaw/issues/58537.

## Reproduction Assessment

Yes, by source inspection. A synthetic Feishu event with `chat_type: "p2p"`, an `oc_...` chat ID, and topic metadata reaches current main with `isGroup === false` because routing consults only `isFeishuGroupChatType(ctx.chatType)`.

## Solution Assessment

No. The PR touches the right Feishu surfaces, but the prefix-only classifier is too broad and fails to preserve native `topic_group`; a safer fix should use verified topic/chat metadata instead of bare `oc_` prefix promotion.

## Review Findings

Overall correctness: patch is incorrect
