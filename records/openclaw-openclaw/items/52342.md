## Summary

Keep open: the PR targets a real Feishu topic-group routing bug, but its `p2p` + `oc_` predicate is overbroad and it would also drop current `topic_group` group handling, so it is not merge-ready.

## What This Changes

The PR adds a Feishu chat classifier that treats `group` or `p2p` + `oc_` as group traffic, wires it into inbound routing and mention forwarding, adds helper tests, and rewrites Feishu DM fixtures to `p2p_` IDs.

## Best Possible Solution

Revise the Feishu-owned classifier to preserve existing `topic_group` handling and DM safeguards, use a verified topic-group signal for anomalous `p2p` OpenChat events, add regression coverage plus a changelog entry, and leave outbound/session consistency tracked in #58537.

## Reproduction Assessment

Yes. Current main routes `chat_type: "p2p"` through the direct path in both `bot.ts` and `mention.ts`, matching the linked #52238 logs, and the PR regressions are visible from the proposed helper branches against current tests.

## Solution Assessment

No. The PR changes the right Feishu-owned surfaces, but `p2p` + `oc_` is too broad and the helper omits `topic_group`; a verified topic/OpenChat discriminator with preserved DM behavior is the safer fix.
