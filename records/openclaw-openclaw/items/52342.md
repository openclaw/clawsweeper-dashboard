## Summary

Keep this PR open for revision: it targets a real Feishu routing bug, but the proposed `p2p` + `oc_` prefix classifier is too broad for current-main DM/card-action behavior, omits native `topic_group`, and still lacks real Feishu after-fix proof.

## What This Changes

The branch adds an `isFeishuGroupChat` helper for `p2p` `oc_` chat IDs, applies it to Feishu inbound and mention-forward routing, adds helper tests, and rewrites several DM fixtures to `p2p_` IDs.

## Best Possible Solution

Revise or replace the branch with a Feishu-owned classifier that uses verified topic/OpenChat evidence, preserves native `topic_group` and true DM/card-action behavior, covers bot routing plus mention-forward, and keeps outbound session consistency tracked in https://github.com/openclaw/openclaw/issues/58537.

## Reproduction Assessment

Yes, by source inspection. A synthetic Feishu event with `chat_type: "p2p"`, an `oc_...` chat ID, and topic metadata reaches current main with `isGroup === false` because routing consults only `isFeishuGroupChatType(ctx.chatType)`.

## Solution Assessment

No. The PR touches the right Feishu surfaces, but `p2p` plus bare `oc_` prefix is too broad and the helper rejects native `topic_group`; a verified topic/OpenChat signal is the safer fix.
