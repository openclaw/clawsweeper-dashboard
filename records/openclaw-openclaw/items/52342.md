## Summary

Keep open: the Feishu topic-group bug is real, but this PR is not mergeable because its bare `p2p` + `oc_` heuristic can reclassify current DM/card-action paths as groups and it drops native `topic_group` support; external after-fix real behavior proof is also missing.

## What This Changes

The PR adds a Feishu `isFeishuGroupChat` helper, uses it for inbound routing and mention-forward detection, adds helper unit tests, and rewrites several Feishu DM fixtures from `oc_` to `p2p_` IDs.

## Best Possible Solution

Land a Feishu-owned classifier that treats only verified topic/OpenChat events as group traffic, preserves current `topic_group` and p2p DM/card-action behavior, covers bot and mention paths, and leaves outbound/session consistency tracked in https://github.com/openclaw/openclaw/issues/58537.

## Reproduction Assessment

Yes, by source inspection. Current main parses `chat_type: "p2p"`, `oc_...`, `root_id`, and `thread_id` into context, then routes from `isFeishuGroupChatType(ctx.chatType)`, so the reported anomalous topic event follows the direct path; I did not run live Feishu.

## Solution Assessment

No. The PR changes the right Feishu-owned surfaces, but a bare `p2p` + `oc_` prefix is too broad and the helper omits current `topic_group` support; a verified topic/OpenChat signal with preserved DM/card-action coverage is safer.
