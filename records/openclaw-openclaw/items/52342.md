## Summary

Keep this PR open, but it should not merge as-is: it targets a real Feishu routing bug, yet the current patch uses an overbroad `p2p` plus `oc_` prefix classifier, regresses native `topic_group` handling, and still lacks redacted real Feishu topic-group proof.

## What This Changes

The PR adds a Feishu chat classifier for `p2p` `oc_` IDs, applies it to inbound routing and mention-forward detection, adds helper tests, and rewrites several Feishu DM fixtures from `oc_` to `p2p_` IDs.

## Best Possible Solution

Land a Feishu-owned classifier that distinguishes actual OpenChat/topic-group webhook events from true DMs using verified topic/chat metadata, preserves native `topic_group` and p2p/card-action behavior, applies consistently to inbound routing and mention-forward detection, and coordinates outbound session consistency with https://github.com/openclaw/openclaw/issues/58537.

## Reproduction Assessment

Yes, by source inspection rather than live Feishu proof. A synthetic Feishu event with `chat_type: "p2p"`, an `oc_...` chat ID, and topic metadata reaches current main with `isGroup === false` because routing consults only `isFeishuGroupChatType(ctx.chatType)`.

## Solution Assessment

No. The PR touches the right Feishu surfaces, but the prefix-only classifier is too broad for authorization-sensitive routing and also drops native `topic_group` events from the group path.
