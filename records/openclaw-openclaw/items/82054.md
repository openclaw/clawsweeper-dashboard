## Summary

Keep open: this is a valid core/product feature request, and current main still has automatic compaction, cache-TTL pruning, transcript rewrite plumbing, and bundled tool-result reducers, but no caller-visible allocate/free lifecycle for individual tool, skill, or plugin outputs.

## What This Changes

The issue requests a stable ID plus caller-controlled free/release mechanism for stale context outputs from tools, skills, and plugins.

## Best Possible Solution

Design an additive context-lifecycle contract in the context engine and plugin SDK that can mark outputs stale or rewrite prompt-visible content while preserving transcript validity, replay pairing, auditability, and native harness constraints.

## Reproduction Assessment

Not applicable: this is a feature request, not broken behavior under an existing contract. Source and docs confirm current main has automatic pruning/compaction paths but no caller-visible free API.

## Solution Assessment

Unclear: the requested allocate/free model is plausible, but the best solution needs maintainer architecture review around stable IDs, prompt-only versus transcript rewrites, replay constraints, and SDK exposure.
