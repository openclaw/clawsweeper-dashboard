## Summary

Keep open: this is a real core/product feature request for caller-controlled context lifecycle, not a stale bug or ClawHub-only bundle. Current main has compaction, cache-TTL in-memory pruning, tool-result truncation, and context-engine rewrite hooks, but no public stable allocate/free API for tool, skill, or plugin results.

## What This Changes

The issue requests a caller-controlled way to mark prior tool, skill, or plugin context entries as stale and reclaim their prompt tokens.

## Best Possible Solution

Design an additive context-lifecycle contract in the context engine/plugin SDK that can safely mark result entries stale or rewrite prompt-visible content while preserving provider transcript validity and history semantics.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken defined behavior. Source review confirms current main lacks the requested manual free API, but that is an unimplemented capability rather than a bug reproduction.

## Solution Assessment

Unclear: the malloc/free model is plausible, but the safest API needs maintainer design across entry IDs, context-engine lifecycle hints, plugin SDK exposure, and transcript replay constraints.
