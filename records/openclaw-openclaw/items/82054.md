## Summary

Keep open: this is a valid core/product feature request for caller-controlled context lifecycle. Current main has compaction, in-memory session pruning, internal transcript rewrite hooks, and tool-result truncation, but no public stable allocate/free API for tool, skill, or plugin outputs.

## What This Changes

The issue requests stable IDs for tool/skill/plugin context outputs and an explicit caller-driven free operation to reclaim context tokens.

## Best Possible Solution

Design an additive context-lifecycle contract in the context engine and plugin SDK that can mark outputs stale or rewrite prompt-visible content while preserving transcript validity, auditability, and provider replay constraints.

## Reproduction Assessment

Not applicable: this is a feature request, not broken behavior under an existing contract. Source review confirms current main lacks the requested caller-visible allocate/free API.

## Solution Assessment

Unclear: the requested malloc/free model is plausible, but the safest solution needs maintainer design around stable IDs, prompt-only versus transcript rewrites, replay constraints, and SDK exposure.
