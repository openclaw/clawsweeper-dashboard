## Summary

Keep open: this PR is not mergeable as-is because the proposed pre-restart shutdown controller changes existing deferral behavior into an unconditional hard exit while active work is still pending, and the claimed soft-abort/socket-close phases are not wired to real active work.

## What This Changes

The PR adds a gateway restart shutdown-controller/config surface for phased force drain and also adds a Discord REST API timeout config path.

## Best Possible Solution

Keep the existing deferral semantics as the default, make any force-drain escalation explicit and integrated with the current run-loop drain/recovery path, then prove it with focused tests and real restart logs.

## Reproduction Assessment

Yes for the blocking review issue: source inspection shows `deferGatewayRestartUntilIdle` always creates a default ShutdownController and reaches `process.exit(0)` around 40s while pending work remains. Current docs and tests define a 300000ms default deferral and an indefinite `0` mode.

## Solution Assessment

No. The current PR puts a hard-exit cascade before the existing SIGUSR1 run-loop drain/recovery machinery; the safer solution is to preserve current defaults and integrate any escalation with the existing active-run abort/session recovery path.

- **[P1] Preserve restart deferral before hard exit:** `src/infra/restart.ts:521-533`
  - body: `opts.forceDrain` is optional, but this code normalizes `undefined` into default phase timings and then calls `process.exit(0)` on hardKill. Make force drain explicitly opt-in or derive its hard-kill budget from the existing deferral contract.
  - confidence: 0.94
- **[P2] Wire shutdown phases to actual active work:** `src/gateway/shutdown-controller.ts:91-116`
  - body: The controller keeps its own AbortController and socket sets, but this PR does not register any real gateway model calls or WebSocket connections with them. As written, softAbort and forceClose only operate on empty sets, so the new cascade mostly logs until hardKill exits the process. Hook this into the existing active run/chat abort/server close paths or remove the inactive phases.
  - confidence: 0.88
