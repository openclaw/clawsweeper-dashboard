## Summary

Keep open: this PR addresses a real Discord generated auto-thread-title failure with reasoning models and has sufficient log proof, but the latest diff still forwards a fixed 4096 runtime `maxTokens` override without clamping it to the selected model's output limit.

## What This Changes

The PR raises Discord generated auto-thread title maxTokens from 512 to 4096, extends the background title-generation timeout from 10s to 60s, updates the focused assertion, and adds a changelog entry.

## Best Possible Solution

Merge a version that keeps the 60s background timeout and 4096 default headroom, but computes the effective title cap as the positive minimum of 4096 and the selected model's `maxTokens`, with focused regression coverage.

## Reproduction Assessment

Yes, at source and PR-log level. Current main and `v2026.5.7` send a fixed 512-token title request with a 10s abort, and the PR body includes MiniMax M2.7 logs showing thinking-only empty output before the larger cap and non-empty text afterward.

## Solution Assessment

No, not as written. Raising timeout and reasoning headroom is the right narrow direction, but the fixed 4096 override should be capped by the selected model's configured output limit before provider transports receive it.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Clamp title tokens to the selected model limit:** `extensions/discord/src/monitor/thread-title.ts:28`
  - body: The PR raises the Discord title cap to 4096 and still forwards that value unchanged as `options.maxTokens`. The simple-completion helper preserves caller options, and Anthropic/OpenAI transports prefer runtime `maxTokens` over `model.maxTokens`, so a selected model whose catalog output limit is below 4096 can receive an over-limit request and silently skip the background rename. Compute an effective cap from the positive minimum of 4096 and `params.model.maxTokens`, with coverage for both default and clamped cases.
  - confidence: 0.9
