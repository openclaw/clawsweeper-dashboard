## Summary

Keep this PR open for changes before merge. The timeout/headroom increase is useful and backed by real copied runtime output, but the current diff still forwards a fixed 4096-token override without respecting the selected model's configured output cap, so some valid configured models can still fail the background rename path.

## What This Changes

The PR raises Discord generated auto-thread-title maxTokens from 512 to 4096, extends the title-generation timeout from 10s to 60s, updates the focused unit assertion, and adds a changelog entry.

## Best Possible Solution

Preserve the 4096 reasoning headroom and 60s background timeout, but compute an effective title maxTokens that clamps to any positive selected model maxTokens before calling the simple-completion helper, with focused regression coverage.

## Reproduction Assessment

Yes, at source and PR-log level. Current main forwards a fixed 512-token title request with a 10s abort, and the PR body includes copied MiniMax M2.7 logs showing thinking-only empty output before the larger cap/timeout and non-empty text afterward.

## Solution Assessment

No, not as written. Raising the timeout and reasoning headroom is the right narrow direction, but the fixed 4096 request should be clamped to the selected model's configured output limit before provider transports see it.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Clamp title tokens to the selected model limit:** `extensions/discord/src/monitor/thread-title.ts:31`
  - body: The new 4096 cap is forwarded unchanged as `options.maxTokens` for every selected title model. The simple-completion helper preserves caller options, and Anthropic/OpenAI transports prefer runtime `maxTokens` over `model.maxTokens`, so any configured model with a lower output cap can receive an over-limit request and the background rename can still silently skip. Clamp to the positive minimum of this cap and `params.model.maxTokens` before calling the completion helper.
  - confidence: 0.9
