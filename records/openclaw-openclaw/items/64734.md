## Summary

Keep open. The bug is plausible and the PR includes useful real runtime evidence, but the current diff should not merge until the raised fixed token cap is clamped to the selected model's declared output cap.

## What This Changes

The branch raises Discord generated thread-title maxTokens from 512 to 4096, raises the timeout from 10s to 60s, updates the unit expectation, and adds a changelog entry.

## Best Possible Solution

Keep the timeout increase and larger reasoning-model budget, but compute the title maxTokens as the positive minimum of the Discord cap and prepared model.maxTokens, with a low-cap regression test.

## Reproduction Assessment

Yes. Source inspection shows current main uses a fixed 512-token cap and 10s timeout in background generated-title completion, and the PR body includes MiniMax M2.7 before/after runtime logs; I did not rerun live Discord.

## Solution Assessment

No as written. Raising the Discord-owned timeout and budget is the right narrow direction, but the fixed 4096 request should be bounded by the selected model's declared output cap before it is forwarded.
