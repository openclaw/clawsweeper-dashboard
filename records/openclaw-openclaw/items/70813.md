## Summary

Keep open. Current main still lacks WhatsApp edit/delete/unsend support, and this PR is the focused implementation candidate, but it is not merge-ready because the new send adapter misses current main's LID-aware routing and the external PR still lacks real WhatsApp behavior proof plus maintainer safety review.

## What This Changes

The PR adds WhatsApp Web edit, delete, and unsend message actions across discovery, gateway dispatch, Baileys send adapters, docs, changelog, capabilities, and focused tests.

## Best Possible Solution

Keep the implementation in the WhatsApp plugin, rebase it onto current main's Baileys/LID-routing surface, fix the adapter issue, then require maintainer acceptance plus redacted live WhatsApp proof before merge.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature PR. The current feature gap is source-verifiable on main because WhatsApp advertises only react/poll, routes only react, and rejects non-react actions.

## Solution Assessment

Unclear as merge-ready: the PR uses the right WhatsApp plugin-owned seams and Baileys payload family, but it needs LID-aware recipient resolution, live behavior proof, and maintainer safety semantics review.
