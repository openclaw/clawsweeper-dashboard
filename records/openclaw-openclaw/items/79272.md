## Summary

Keep open: this PR fixes a real WhatsApp `forceDocument` gap and supplies adequate after-fix payload proof, but the latest head still introduces a forced-document caption regression for WhatsApp group mentions.

## What This Changes

The PR threads `forceDocument`/`asDocument` through WhatsApp outbound media sending, updates the WhatsApp document payload selector and filename preservation, and refreshes CLI/docs/schema snapshots, tests, and changelog copy.

## Best Possible Solution

Keep the targeted WhatsApp plumbing, but repair the forced-document branch to preserve resolved mention captions and add focused coverage before landing.

## Reproduction Assessment

Yes. Source inspection shows current main carries the generic `forceDocument` option to channel outbound context, but WhatsApp drops it before `sendMessageWhatsApp` and then selects image payloads by MIME without an `asDocument` guard; I did not run a live WhatsApp send in this read-only review.

## Solution Assessment

No, not as submitted. The owner-module approach is the narrow maintainable fix, but the new forced-document payload branch should use `resolvedPayloadText.text` so group caption mention behavior stays aligned with existing media/document sends.
