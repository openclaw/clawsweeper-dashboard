## Summary

This PR is a focused fix for a real WhatsApp force-document gap, and the follow-up commit appears to address the earlier generated snapshot and CLI/docs copy blockers. It still needs one narrow correction before merge because the new forced-document payload branch bypasses the resolved caption text used by the existing media paths.

## What This Changes

The PR threads `forceDocument`/`asDocument` through WhatsApp outbound media sending, updates the WhatsApp document payload selector and filename preservation, adds tests, refreshes prompt snapshots, and updates CLI/docs/changelog copy.

## Best Possible Solution

Keep the targeted WhatsApp plumbing, change the forced-document branch to reuse the resolved caption text, add a group-mention regression for that branch, and then land with the existing docs, snapshots, tests, and changelog updates.

## Reproduction Assessment

Yes, at source level: current main reads `forceDocument`/`asDocument` in the message action path but WhatsApp drops it before `sendMessageWhatsApp`, and the payload selector routes image MIME to `image` instead of `document`. I did not run a live WhatsApp send in this read-only review.

## Solution Assessment

No, not as submitted. The overall approach is the narrow owner-module fix, but the new forced-document payload branch should use `resolvedPayloadText.text` so group caption mention behavior stays aligned with the existing media/document paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve resolved captions for forced documents:** `extensions/whatsapp/src/inbound/send-api.ts:87`
  - body: The new `asDocument` branch builds the document caption from raw `text`, while the existing image/video/document branches use `resolvedPayloadText.text` before adding mention metadata. In WhatsApp groups with LID-backed mention replacement, forced-document captions would keep the unreplaced phone token while adding LID mentions, so native mention rendering can regress on the new path. Use `resolvedPayloadText.text` here and cover forced-document captions in the mention test.
