## Summary

Keep open: the PR addresses a real current-main WhatsApp force-document gap, but as submitted it still introduces a forced-document caption regression for WhatsApp group mentions.

## What This Changes

The branch wires `forceDocument`/`asDocument` through WhatsApp outbound media sending, forces document payload selection, preserves filenames, and updates CLI/docs/schema snapshots, tests, and changelog copy.

## Best Possible Solution

Keep the targeted WhatsApp plumbing, but build the forced-document caption from the resolved mention-aware text and cover that branch with a group mention regression before merge.

## Reproduction Assessment

Yes, at source level: current main parses `forceDocument`/`asDocument` in the generic message action path, but WhatsApp drops the flag before `sendMessageWhatsApp` and the send API selects image payloads by MIME. I did not run a live WhatsApp account send in this read-only review.

## Solution Assessment

No, not as submitted. The approach is the narrow owner-module implementation, but the new forced-document branch should use `resolvedPayloadText.text` so group caption mention behavior stays aligned with the existing media and document paths.
