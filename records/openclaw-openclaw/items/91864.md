## Summary

Keep open: the PR targets a real QMD-backed `memory search --json` hang, but the submitted forced-exit path is not merge-ready because it can truncate JSON output, race recall state, and bypass normal manager cleanup without live QMD proof.

## What This Changes

The PR changes memory-core's `memory search --json` path to write JSON, optionally start recall tracking, and schedule a production-only `process.exit()`.

## Best Possible Solution

Keep the QMD CLI termination mitigation, but move the exit scheduling after normal recall/manager cleanup and stdout/stderr flushing, then attach live QMD terminal proof that JSON output is valid and the command exits.

Yes, there is a credible reproduction path from the linked QMD report and current source still follows the reported JSON-return path. I did not run a live QMD backend in this read-only review, so the exact hang still needs live proof before merge.

## Solution Assessment

No. A forced exit may be an acceptable mitigation for lingering QMD handles, but this implementation exits too close to stdout, recall tracking, and manager cleanup.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Flush CLI output before forcing process exit:** `extensions/memory-core/src/cli.runtime.ts:1287`
  - body: `defaultRuntime.writeJson()` ultimately writes to `process.stdout`, and Node documents that `process.exit()` can terminate before stdout/stderr I/O completes. This new `setImmediate(() => process.exit(...))` can make piped JSON invalid; use a flush-aware scheduler like the Matrix CLI pattern before exiting.
  - confidence: 0.9
- **[P1] Let recall tracking and manager cleanup settle before exit:** `extensions/memory-core/src/cli.runtime.ts:1277-1287`
  - body: The patch starts `recordShortTermRecalls()` and then schedules a production-only forced exit from inside the manager callback. That can run before the recall write and before `withManager()` finishes its `finally` cleanup, despite current tests treating JSON recall tracking and manager close as expected behavior.
  - confidence: 0.84
