## Summary

Keep open: current main still rejects `historyLimit: 0` for `GroupChatSchema`, while docs and sibling history-limit schemas treat zero as a valid disable value. Open PR #65359 references this issue with closing syntax, so the issue should stay open until that PR or an equivalent fix merges.

## What This Changes

Allow `messages.groupChat.historyLimit` and per-agent `groupChat.historyLimit` to accept `0` by aligning `GroupChatSchema.historyLimit` with the other nonnegative history-limit schemas.

## Best Possible Solution

Land PR #65359 or an equivalent narrow patch that changes only the group-chat history-limit schema to nonnegative validation, refreshes the generated schema, and keeps focused regression coverage for zero, positive, and negative values.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: both global and per-agent group-chat config paths validate through `GroupChatSchema`, whose current `historyLimit` validator rejects `0`.

## Solution Assessment

Yes. The narrowest maintainable fix is to change this one schema field from positive to nonnegative and update the generated schema/test coverage; the reload path does not need a broader behavior change for this report.
