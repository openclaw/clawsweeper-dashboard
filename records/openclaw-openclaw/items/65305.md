## What This Changes

Allow `GroupChatSchema.historyLimit` to accept `0` so group chat history can be disabled consistently with sibling history-limit settings.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/65359 or an equivalent narrow `.min(0)` fix with schema and Gateway `config.patch` regression coverage, then close this issue after it reaches main.

Yes, at source level. Current main validates `messages.groupChat.historyLimit` through `GroupChatSchema`, which uses `.positive()` even though docs and sibling schemas define `0` as valid; I did not run live proof because this review is read-only.

## Solution Assessment

Yes. The narrow maintainable fix is to change only the group-chat history lower bound to `.min(0)` and cover both direct schema validation and Gateway `config.patch`.
