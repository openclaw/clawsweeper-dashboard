## Summary

Keep open: current main still validates `messages.groupChat.historyLimit` with a strictly positive Zod number, while the docs and sibling history-limit schemas allow `0`; the linked same-author PR should be reviewed or replaced rather than closing this issue now.

## What This Changes

The issue asks OpenClaw to accept `messages.groupChat.historyLimit: 0` so group chat history can be disabled consistently with other history-limit settings.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/65359 or an equivalent current-main patch that changes the group-chat validator to `.min(0)`, adds focused boundary coverage, and runs the config schema validation path before closing this issue.
