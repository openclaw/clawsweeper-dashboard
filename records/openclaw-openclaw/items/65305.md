## Summary

Keep open: current main still defines `GroupChatSchema.historyLimit` with `.positive()`, so `messages.groupChat.historyLimit: 0` remains invalid even though the docs and sibling history-limit schemas allow zero. The provided GitHub context also shows open PR #65359 references this issue with closing syntax, so the issue should stay open until that PR or an equivalent fix merges.

## What This Changes

Allow `GroupChatSchema.historyLimit` to accept `0` so global and per-agent group-chat history can be disabled consistently with channel and DM history limits.

## Best Possible Solution

Review and land PR #65359, or an equivalent narrow patch, changing only `GroupChatSchema.historyLimit` to `.min(0)` and adding regression coverage for zero, positive, and negative values. The generated config schema/baseline should be updated if the repo's schema checks require it.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: `messages.groupChat` and per-agent `groupChat` both validate through `GroupChatSchema`, whose `historyLimit` currently requires a positive integer, while docs and runtime history helpers define `0` as disabled.

## Solution Assessment

Yes. The narrowest maintainable fix is to align `GroupChatSchema.historyLimit` with the existing `.min(0)` history-limit schemas and add focused schema tests; PR #65359 appears to match that direction.
