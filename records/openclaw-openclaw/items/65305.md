## Summary

Keep open: current main still rejects `messages.groupChat.historyLimit: 0` through `GroupChatSchema`, while docs and adjacent history-limit schemas support zero; the open linked PR remains the canonical fix path.

## What This Changes

Allow `messages.groupChat.historyLimit` and per-agent `groupChat.historyLimit` to accept `0` as a valid no-history value.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/65359 or an equivalent current-main patch that changes `GroupChatSchema.historyLimit` to `.min(0)` and adds focused validation/schema coverage.

## Reproduction Assessment

Yes. A high-confidence source-level path exists: global and per-agent group-chat config validate through `GroupChatSchema`, whose `.positive()` validator rejects `0`, and `config.patch` validates through the same merged config path.

## Solution Assessment

Yes. The narrow maintainable fix is to change the single group-chat validator to `.min(0)` and cover both Zod validation and runtime/base JSON Schema output; broader reload error surfacing should stay separate.
