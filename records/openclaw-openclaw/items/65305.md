## Summary

Keep this issue open: current main and latest release still reject the documented `historyLimit: 0` value, and an open linked PR is the active fix path rather than a reason to close the report early.

## What This Changes

Allow `messages.groupChat.historyLimit` and per-agent `groupChat.historyLimit` to accept `0` so group chat history can be disabled consistently.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/65359 or an equivalent narrow `.min(0)` schema fix with Zod and Gateway `config.patch` regression coverage, then close this issue after it reaches main.

## Reproduction Assessment

Yes, at source level. Current main validates `messages.groupChat.historyLimit` through `GroupChatSchema`, which uses `.positive()` even though docs and sibling schemas define `0` as valid; I did not run live Gateway proof because this review is read-only.

## Solution Assessment

Yes. Changing the single `GroupChatSchema.historyLimit` lower bound to `.min(0)` and covering both direct schema validation and Gateway `config.patch` is the narrowest maintainable fix.
