## Summary

The native rich-table capability is useful and has positive live evidence, but this branch is not merge-ready: supported rich methods can reject individual payloads without reaching the established fallback, and the patch expands the new transport into pairing, preview, and progress-draft workflows that the issue and supplied proof do not establish.

Adds Telegram Bot API 10.1 rich-message send, edit, and draft transport attempts with fallback across final replies, direct sends, pairing replies, stream previews, and native tool-progress drafts.

## Best Possible Solution

Keep rich Markdown limited to the proven single-chunk final send/edit path, place it before—not instead of—the canonical HTML-to-plain fallback chain, and defer pairing, preview, and progress-draft adoption until each path has independent need and live proof.
