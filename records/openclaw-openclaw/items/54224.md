## Summary

Keep open. Current main covers Slack channel history and thread reads through the existing `read`/`readMessages` path with user-token-preferred reads, but Slack message search is still absent; open PR #65691 is the active implementation candidate for the remaining `search.messages` work.

## What This Changes

Expose Slack read-only message retrieval actions for user-token read-only workflows, including channel history, message search, and thread history.

## Best Possible Solution

Review and either land or close #65691; the desired end state is a Slack-owned `search` message action backed by Slack `search.messages`, documented and tested separately from the existing `read` history/thread path.

## Reproduction Assessment

Yes. Source inspection on current main shows the existing Slack `read` path covers channel history and thread replies, while Slack message search is not advertised or implemented.

## Solution Assessment

Yes. Keeping this issue open for #65691 is the narrowest maintainable path: keep history/thread reads on `read`, and add Slack `search.messages` as a separate least-privilege read action.
