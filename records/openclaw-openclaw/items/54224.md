## Summary

Keep open. Current main covers Slack channel history and thread reads through the existing `read`/`readMessages` path with user-token-preferred read behavior, but it still does not implement or advertise Slack message search; open PR #65691 remains the active implementation candidate for that remaining work.

## What This Changes

Request to expose read-only Slack message retrieval through the message tool, covering channel history, message search, and thread history for user-token read-only workflows.

## Best Possible Solution

Review and either land or close #65691. The desired end state is a Slack-owned generic `search` message action backed by Slack `search.messages`, exposed only when appropriate user-token read credentials are available, wired through the existing message action contract and CLI, documented at https://docs.openclaw.ai/channels/slack, and covered by focused discovery, dispatch, runtime, formatting, and least-privilege tests; the existing `read` action can remain the channel history and thread history solution.
