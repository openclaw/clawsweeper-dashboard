## What This Changes

Expose Slack read-only message retrieval actions for channel history, thread history, and message search in user-token read-only workflows.

## Best Possible Solution

Land one coherent Slack-plugin-owned search path that keeps `read` as the history/thread action, adds a gated `search` action backed by Slack `search.messages` or an explicitly chosen RTS contract, and includes focused docs, tests, and real Slack proof.

## Solution Assessment

Yes for the broad direction: the maintainable fix belongs in the Slack plugin through the existing generic message-action contract. It still needs maintainer choice because overlapping PRs implement related search/action surfaces and the security boundary needs review.
