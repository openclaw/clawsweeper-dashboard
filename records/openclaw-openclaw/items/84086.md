## What This Changes

The issue requests a fix for Codex app-server native thread churn in mixed human/other-agent Discord channel sessions when room-event delivery mode changes the `message` tool schema.

## Best Possible Solution

Add a focused Codex app-server regression for alternating Discord human and room-event turns, then make the thread-level `message` tool contract stable across delivery modes while preserving message-tool delivery and authorization checks.

## Solution Assessment

Unclear until a patch exists. The right direction is narrower than the owner-only PRs: stabilize the Codex dynamic tool contract for delivery-mode flips while keeping `message_tool_only` delivery and sender authorization fail-closed.
