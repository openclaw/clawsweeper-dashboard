## Best Possible Solution

Land or replace the linked Codex app-server fix so the `message` dynamic tool contract stays stable across delivery-mode flips while message-tool delivery and sender authorization remain enforced at runtime.

## Solution Assessment

Yes. Stabilizing the Codex thread-level `message` tool contract across delivery-mode flips is the narrowest maintainable direction, as long as per-turn message-tool delivery and authorization continue to fail closed.
