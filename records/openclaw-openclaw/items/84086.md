The issue reports Discord multi-agent Codex app-server native thread churn when room-event delivery flips the `message` dynamic tool between deferred and direct registration shapes.

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

Yes for the implementation direction, but not for closure yet. The focused fix should make `message` stable across delivery modes and this issue should remain open until the linked PR merges or maintainers choose a replacement.
