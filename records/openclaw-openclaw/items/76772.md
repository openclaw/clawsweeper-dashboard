## What This Changes

The issue asks for `session_status` to bind `sessionKey="current"` to the executing embedded session and report the selected/current model instead of stale runtime model stamps.

## Best Possible Solution

Define the status contract so `current` is the executing run's canonical session and the status card clearly separates selected/current model intent from last runtime execution metadata.

No high-confidence live reproduction was established in this read-only review. Source inspection does identify the risky resolver fallback and cached-runtime-model paths, but the exact Telegram requester key needs confirmation.
