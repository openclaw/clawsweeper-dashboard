## Summary

Keep open: the report is concrete and current main still has source-level gaps in the `session_status` current-session and model-display paths, with no focused regression for the stale legacy direct row plus live main-session mismatch.

## What This Changes

Ensure `session_status({"sessionKey":"current"})` binds to the live embedded session and reports current model intent instead of stale legacy or cached runtime model metadata.

## Best Possible Solution

Add a regression seeding active `agent:main:main` plus a stale legacy direct row, then make `session_status` bind to the running session key and display selected model intent while exposing cached runtime or last-run model only as secondary history.

## Reproduction Assessment

No, there is not a high-confidence live reproduction from this review. Source inspection shows a plausible current-main path, and the existing tests do not cover the reported stale direct-session plus `modelOverride` mismatch.

## Solution Assessment

Unclear until maintainers decide the selected-model versus runtime/fallback status contract. The narrow maintainable direction is to bind `current` to the live run session and separate current intent from cached last-run metadata without removing fallback diagnostics.
