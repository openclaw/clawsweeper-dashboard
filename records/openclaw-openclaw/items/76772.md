## Summary

Keep open: current main still has a source-level path where embedded tools receive the direct runtime-policy key, and `session_status({sessionKey:"current"})` can fall back through that requester instead of the live `agent:main:main` session. The selected-versus-active model display is only partially separated, with tests still preserving runtime-only model fields as the selected card model when no explicit override exists.

## What This Changes

The issue requests `session_status` to bind `current` to the executing embedded session and report current model intent rather than stale direct-session or last-run model metadata.

## Best Possible Solution

Bind `session_status` `current` to the live embedded run session key while preserving the direct runtime-policy key for policy and sandbox checks, then expose selected model intent separately from active or last-run runtime metadata.

## Reproduction Assessment

Yes for a source-level reproduction: current main passes the runtime-policy key into embedded tool construction, and `current` can fall back through that requester key. I did not establish a live Telegram replay in this read-only review.

## Solution Assessment

Unclear for the full issue until maintainers settle the selected-versus-active model status contract. The narrow maintainable direction is to bind `current` to the live run key and keep runtime or last-run model data as secondary diagnostics.
