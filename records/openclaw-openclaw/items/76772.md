## Summary

Keep open: current main still has a source-level path where embedded runs can hand the direct runtime-policy key to tools while the live conversation session is `agent:main:main`, and the broader selected-model versus runtime-model status contract remains unsettled.

## What This Changes

The issue requests that `session_status({"sessionKey":"current"})` bind to the running embedded session and report selected model intent instead of stale persisted runtime model identity.

## Best Possible Solution

Bind `session_status` `current` to the live run session key while preserving the direct runtime-policy key for policy/sandbox checks, then separate selected model intent from active or last-run runtime metadata in status output.

## Reproduction Assessment

Yes for a source-level reproduction: current main passes `runtimePolicySessionKey` into the embedded tool context, and `current` fallback can resolve through that requester key. No live Telegram replay was established in this read-only review.

## Solution Assessment

Unclear for the full issue until maintainers confirm the selected-model versus runtime-model status contract. The narrow maintainable fix is to bind `current` to the live run key and retain runtime/last-run model data as secondary diagnostics.
