## Summary

Keep open: current main has useful pieces for restart deferral, stale-process cleanup, startup-unavailable RPC gating, and /readyz, but it still does not define one shared partial-reload/degraded-state contract that Control UI and sessions.resolve can use instead of surfacing a missing-session or generic connection failure.

## What This Changes

The issue asks for macOS launchd Gateway plugin/config maintenance to avoid leaving Control UI in a partially alive state that may require doctor-assisted recovery.

## Best Possible Solution

Define one Gateway degraded/reload readiness contract for partial reload, restart-deferred, and incomplete plugin-set states, have sessions.resolve return that temporary state instead of plain missing-session errors, and have Control UI surface it while the narrower WebChat notice work is reviewed separately.
