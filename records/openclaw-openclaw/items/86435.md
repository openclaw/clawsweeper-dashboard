## Summary

Keep open: current main has restart deferral, stale-process cleanup, and /readyz readiness probes, but source still does not show a shared partial-reload/degraded-state contract for Control UI or sessions.resolve.

## What This Changes

The issue requests more robust macOS launchd/plugin-maintenance recovery so partial Gateway reloads do not surface as generic Control UI failures or misleading missing-session errors.

## Best Possible Solution

Define one Gateway reload/readiness state for partial reload, restart-deferred, or incomplete plugin-set conditions, and have sessions.resolve and Control UI present that state instead of treating it as a missing session or generic connection failure.
