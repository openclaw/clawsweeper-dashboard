## Summary

Keep this PR open, but it is not merge-ready: the latest head still does not return the new cache flag from the transport compat resolver, can drop the dynamic suffix when no user message exists, lacks real DeepSeek or equivalent payload proof, and still needs a maintainer choice between this compat option and the open config-free sibling fix.

## What This Changes

The branch adds a `compat.disableBoundaryAwareCache` option for OpenAI-completions models, auto-detects it for DeepSeek/Xiaomi, and tries to move below-boundary prompt context into the first user message.

## Best Possible Solution

Land one canonical DeepSeek prefix-cache fix that returns the new flag through the transport resolver if this compat API stays, preserves below-boundary context in every supported message shape, proves the transport path with focused regression tests, and includes redacted real-provider or payload-level cache proof.
