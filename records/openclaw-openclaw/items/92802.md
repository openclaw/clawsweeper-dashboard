## What This Changes

The PR adds a `chat-workbench__main` wrapper and moves the queue, side result, new-message control, and composer into the main chat column beside the workspace file rail.

## Best Possible Solution

Constrain the composer and related chat controls to the main workbench column while preserving full-width chat behavior when the workspace rail is hidden.

## Reproduction Assessment

Yes from source inspection: current main renders the workspace rail inside `.chat-workbench` but renders the queue and composer after that grid, so the composer is not constrained to the main grid column when the rail exists. I did not run a browser repro in this read-only review.

## Solution Assessment

No as currently patched; the wrapper approach is the right shape, but it also needs to inherit the old fill behavior under the rail-hidden flex breakpoint.

- **[P2] Make the new main wrapper fill the flex workbench:** `ui/src/styles/chat/sidebar.css:15-20`
