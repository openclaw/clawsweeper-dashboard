Full review comments:

- **[P1] Narrow the filter before using it as a default-on block:** `src/web-fetch/egress-filter.ts:49`
  - body: This turns every change made by `redactSensitiveText` into a thrown provider fallback error. The default redactor masks broad diagnostic patterns like `?key=...`, `?token=...`, and `?code=...`, while `web_fetch` passes the requested URL through this wrapper before Firecrawl or another provider runs; a legitimate URL with a long non-secret query value can now fail after upgrade. Please use an egress-specific credential predicate or add explicit maintainer-approved broad blocking with compatibility coverage.
  - confidence: 0.84
- **[P3] Document the new web_fetch opt-out setting:** `src/config/schema.help.ts:945-946`
  - body: This adds `tools.web.fetch.enableEgressFilter` as a public config key, but the web_fetch docs and configuration reference still describe provider fallback without this default-on block or the opt-out. Users need the docs to discover the new failure mode and the escape hatch.
  - confidence: 0.74
