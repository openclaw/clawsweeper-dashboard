## Best Possible Solution

Repair Tlon approval preview creation so stored and sent previews are UTF-16 safe in the runtime producer path, keep the existing SDK helper within the plugin boundary, and add focused regression coverage.

## Reproduction Assessment

Yes, from source inspection: the runtime approval producers pre-slice message previews with raw `.slice(0, 100)`, and the formatter returns already-100-code-unit previews unchanged. I did not run a live Tlon flow.

## Solution Assessment

No. The proposed formatter-only change is a plausible helper improvement, but the best fix must cover the producer path that creates the persisted preview or remove the pre-slicing so the safe formatter actually handles the boundary.
