## Summary

Keep open: current main still lacks a supported way for native self-delivered channel sends to enter the canonical message:sent path, and the linked fix PR is open rather than merged.

## What This Changes

The issue requests a plugin SDK delivery-report seam so channel plugins can report self-delivered outbound sends into the canonical message:sent path.

## Best Possible Solution

Land a narrow, maintainer-approved delivery-report seam that lets native self-delivered channel sends report into the canonical message_sent/message:sent pipeline without exposing raw hook internals.

Unclear pending maintainer API review. A generic SDK report seam fits the plugin boundary better than exposing core hook emitters, but the exact public subpath and idempotency contract should be settled in the linked PR.
