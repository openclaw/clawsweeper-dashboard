## Summary

Keep this PR open, but do not merge it as-is. Current main still rejects the reported Windows drive-letter bind form, so the contribution is not obsolete; however the patch changes a secops-owned sandbox bind policy path and still relies on POSIX-style, case-sensitive path comparisons after admitting Windows paths.

## What This Changes

The PR updates sandbox Docker bind config parsing and runtime host-path validation to accept Windows drive-letter bind sources such as `D:/data:/src:ro`.

## Best Possible Solution

Define one shared Windows-aware sandbox host-path contract: parse binds with `splitSandboxBindSpec`, accept only drive-absolute forms, preserve the original Docker path, compare security policy keys with Windows case semantics, keep existing-ancestor canonicalization for junction/symlink escapes, and cover config validation, runtime validation, audit reporting, and Docker create-args behavior with tests before adding a user-facing changelog entry.

## Reproduction Assessment

Yes. The PR body gives concrete Windows 11 Docker Desktop steps, and current main source inspection shows the same failure path: config validation splits `D:/...` at the drive-letter colon and runtime validation rejects non-`/` sources.

## Solution Assessment

No. The requested direction is valid, but this patch spreads ad hoc Windows checks across schema and runtime code without making blocked-path and allowed-root comparisons Windows-aware; the safer fix is a centralized host-path policy helper with secops review.

## Review Findings

Full review comments:

- **[P1] Compare Windows policy paths case-insensitively:** `src/agents/sandbox/host-paths.ts:31-34`
  - body: After this patch, Windows drive-letter binds reach the blocked-path and allowed-root checks, but those checks still use case-sensitive string prefixes. Windows paths are normally case-insensitive, so a bind whose directory casing differs from the configured root or blocked home path can be misclassified. Normalize a separate Windows policy key, or use a Windows-aware comparator, while preserving the original path for Docker.
