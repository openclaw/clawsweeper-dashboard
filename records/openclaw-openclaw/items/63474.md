## Summary

Keep this PR open, but it is not merge-ready. Current `main` still lacks the intended stale `installs`/`plugins` channel-config repair, while the PR branch has reachability bugs that leave non-Feishu channels and account-only stale keys unrepaired before strict schema validation.

## What This Changes

The PR branch adds a legacy migration and tests intended to strip legacy `installs` and `plugins` keys from channel configs before validation, with Feishu schema rejection coverage.

## Best Possible Solution

Implement the repair in the current doctor/shared migration path with a generic channel-level legacy rule that detects stale keys across all configured channels and nested accounts, then cover the real validation path so `doctor --fix` repairs the config before strict channel schema validation.

## Reproduction Assessment

Yes. The linked report has concrete upgrade steps, and current source shows stale `installs`/`plugins` keys still pass through to strict Feishu/channel schema validation unless a legacy issue is detected first.

## Solution Assessment

No. The PR is directionally right, but the proposed rule does not make the migration reachable for all claimed cases and it targets a stale file layout; the safer fix is a current-main doctor/shared migration plus validation-path regression tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Match stale-key rules for every channel:** `src/config/legacy.migrations.channels.ts:136`
  - body: The load path only applies legacy migrations after `findLegacyConfigIssues` finds a matching rule, but this rule is anchored to `channels.feishu`. A config like `channels.slack.installs` will not report a legacy issue and will still reach strict channel validation, even though this migration claims to strip stale keys from every `channels.*` entry. Use a generic `channels` rule that scans configured channel entries.
  - confidence: 0.9
- **[P2] Detect account-only stale keys before validation:** `src/config/legacy.migrations.channels.ts:90`
  - body: `hasStaleChannelKeys` only checks the channel object itself. If the only stale key is under `channels.feishu.accounts.main.plugins`, the `channels.feishu` rule does not match, the migration is skipped, and strict account schema validation still rejects the config. Include nested `accounts.*` entries in the rule match.
  - confidence: 0.9
