## Summary

Keep this PR open, but it is not merge-ready. The linked Feishu upgrade bug still exists on current `main`, while the PR branch targets the old config-migration layout and its legacy-rule trigger would miss the generic and account-only cases it claims to repair.

## What This Changes

The PR branch adds a legacy config migration plus Feishu schema tests intended to strip legacy `installs` and `plugins` keys from channel configs before strict channel validation.

## Best Possible Solution

Implement the repair in the current doctor/shared migration path with a generic `channels` legacy rule that scans every configured channel and nested account for stale `installs` or `plugins`, then strip those keys before strict channel schema validation without relaxing Feishu schemas.

## Reproduction Assessment

Yes. The linked issue provides concrete upgrade steps, and current-main source shows stale `channels.feishu.installs/plugins` keys have no matching legacy migration before strict Feishu validation rejects additional properties.

## Solution Assessment

No. The PR is directionally useful, but the current branch is stale and its matcher does not make the migration reachable for all claimed cases; the safer fix is a current-main doctor/shared migration with validation-path regression coverage.
