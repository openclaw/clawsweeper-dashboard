## Summary

Keep open: current main and the latest release still leave the MCP bridge permission and approval maps without TTL cleanup, while this PR adds a focused TTL sweeper, close-time cleanup, and regression coverage. I found no blocking code defect, but the hardcoded expiry behavior is a compatibility tradeoff for long-lived operator workflows and should remain a maintainer acceptance point before merge.

## What This Changes

The PR wraps MCP bridge pending Claude permission and approval records with tracking timestamps, adds a lazy TTL sweeper plus close-time cleanup, and adds regression tests for expiry and cleanup behavior.

## Best Possible Solution

Land a bounded pending-state cleanup after maintainers accept the expiry defaults, with cap/FIFO left to a separate follow-up unless they want that protection in the same patch.
