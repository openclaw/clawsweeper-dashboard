## What This Changes

The PR wraps MCP channel bridge pending permission and approval entries with timestamps, adds a lazy unrefed TTL sweeper plus close-time clearing, and adds focused regression tests.

## Best Possible Solution

Land this scoped lifecycle fix after normal maintainer review, then close https://github.com/openclaw/openclaw/issues/71646 and keep hard caps plus session binding as separate work.

## Reproduction Assessment

Yes. Current main stores unique permission and approval entries until matching replies or resolved frames arrive, and close() does not clear those maps; the PR body also includes a base/head real wall-clock measurement for the Claude permission map.

## Solution Assessment

Yes. A lazy unrefed TTL sweeper plus close-time clearing is a narrow, protocol-compatible fix for the slow leak; hard caps and session binding are valid separate axes rather than blockers for this patch.
