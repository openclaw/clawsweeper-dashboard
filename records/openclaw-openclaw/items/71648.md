## Summary

## What This Changes

The PR wraps MCP channel-bridge pending permission and approval maps in timestamped entries, adds a lazy TTL sweeper and close cleanup, and adds fake-timer regression coverage plus one fixture update.

## Best Possible Solution

Land this scoped TTL and close-clear lifecycle fix after normal maintainer review, while keeping hard caps and session binding as separate follow-up work.

## Reproduction Assessment

Yes, by source inspection. Unique Claude permission requests and approval-requested events remain in current-main Maps until matching replies or resolved frames arrive, and close() does not clear those Maps.

## Solution Assessment

Yes. A lazy TTL sweeper plus close-time clearing is the narrowest maintainable fix for the slow leak without changing MCP tool or gateway protocol contracts.
