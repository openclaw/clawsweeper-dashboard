## Summary

Keep open. The PR is a useful, narrow fix for a real MCP bridge memory-lifecycle bug still present on current main, and the prior self-terminating sweeper concern has been addressed, but the branch still lacks the required user-facing changelog entry before merge.

## What This Changes

The PR wraps MCP channel bridge pending Claude permission and approval maps in timestamped entries, adds a lazy self-terminating TTL sweeper and close-time clearing, guards post-close writes, and adds focused regression tests plus a fixture update.

## Best Possible Solution

Land the MCP bridge memory-bound fix after adding one Unreleased changelog entry, keep the MCP tool/protocol surface unchanged, and leave burst caps or session-bound permission semantics to their separate follow-ups.

## Reproduction Assessment

Yes. On current main, adding a Claude permission request without a matching yes/no reply or an approval requested event without a resolved event leaves entries in the pending maps, and close() does not clear them.

## Solution Assessment

Yes for the scoped slow-leak bug. A lazy TTL sweeper plus close cleanup is the narrowest maintainable fix here; the remaining changelog entry is merge preparation, not a design problem.
