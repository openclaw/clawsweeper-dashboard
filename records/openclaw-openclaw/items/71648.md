## What This Changes

The PR wraps MCP channel bridge pending Claude permission and approval entries with timestamps, adds lazy TTL sweeping plus close cleanup and post-close guards, and adds focused regression tests plus a fixture update.

## Best Possible Solution

Add the missing changelog entry, keep the narrow TTL/close-cleanup implementation, and land through normal CI/rebase review; treat burst caps and session-bound permission replies as separate follow-up work rather than expanding this PR.

## Reproduction Assessment

Yes. On current main, a Claude permission request without a matching `yes/no <id>` reply remains in `pendingClaudePermissions`, and an approval requested event without a resolved event remains in `pendingApprovals` despite any `expiresAtMs` field.

## Solution Assessment

Yes for the scoped slow-leak bug. The lazy unref'd TTL sweeper plus close cleanup preserves the MCP tool/protocol shape and directly addresses the unbounded retention path; the safer alternative is only to add the required changelog entry before merge, not to broaden this PR.
