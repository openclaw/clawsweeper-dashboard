## Summary

Keep this PR open: the goal is plausible, but the current patch does not actually avoid the expensive reload path, can reuse stale per-run loader inputs, and introduces a test syntax error, while the attached logs only show pre-fix behavior.

## What This Changes

The PR adds an in-memory `DefaultResourceLoader` cache keyed by `cwd` and `agentDir`, cache helper tests, and reload metadata calls in embedded attempt and compaction paths.

## Best Possible Solution

Rework the optimization so expensive discovery is skipped only when resolved resources and per-run settings/factories remain coherent, then add focused tests plus redacted after-fix timing proof from real embedded runs.

## Reproduction Assessment

Yes for the PR defects: source inspection shows both changed call sites still call `reload()`, and the added TTL test contains `await` inside a non-async callback. I did not execute tests because this review is read-only, but the failure path is visible in the PR source.

## Solution Assessment

No. The proposed cache does not bypass the expensive reload, can reuse stale loader inputs across runs, and needs a safer cache boundary or service-level design before it can be the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Make the TTL test callback async:** `src/agents/pi-embedded-runner/resource-loader.test.ts:227`
  - body: This new test uses `await` inside a non-async `it` callback, so the file will fail to parse/type-check before the cache expiry behavior can be tested. Mark the callback `async` or replace the wait with fake timers.
  - confidence: 0.99
- **[P2] Skip reload on cached resource loaders:** `src/agents/pi-embedded-runner/resource-loader.ts:131-134`
  - body: Returning a cached loader here does not avoid the slow path because both embedded callers still immediately execute `await resourceLoader.reload()`. In Pi 0.74, `reload()` is where `packageManager.resolve()` runs, so repeated runs within the TTL still pay the advertised 5-9s cost.
  - confidence: 0.96
- **[P2] Keep cached loaders tied to per-run inputs:** `src/agents/pi-embedded-runner/resource-loader.ts:128-134`
  - body: The cache key ignores `settingsManager` and `extensionFactories`, but `DefaultResourceLoader` stores those constructor inputs and the factories close over per-run session/model/config state. A later run in the same workspace can therefore reuse the first run's loader state instead of the current run's settings and factories.
  - confidence: 0.9
