## Summary

Keep open: the bottleneck is credible, but this branch is not mergeable because cache hits still call the expensive reload path, the cached loader can retain stale per-run settings and extension factories, the new TTL test has blocking defects, and after-fix real behavior proof is missing.

## What This Changes

The PR adds a TTL-backed in-memory cache around embedded Pi `DefaultResourceLoader` creation, reload metadata helpers, attempt/compact reload markers, and cache behavior tests.

## Best Possible Solution

Cache only stable resource discovery work while rebuilding run-specific settings and extension state, then prove repeated embedded runs with redacted after-fix timing logs.

## Reproduction Assessment

Unclear for live reproduction: the PR body and related issue provide credible timing logs, and source shows current main calls `reload()` on every embedded attempt and compaction path. I did not establish a live 5-9s reproduction during this read-only review.

## Solution Assessment

No: this PR reuses a mutable loader while still unconditionally reloading it. A safer solution should cache stable discovery results or return explicit reload state while keeping per-run settings and extension factories fresh.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.96

Full review comments:

- **[P1] Make the TTL pruning test async:** `src/agents/pi-embedded-runner/resource-loader.test.ts:227-240`
  - body: This new test uses `await` inside a non-async `it` callback, so the test file will not parse and the branch cannot rely on this coverage. Mark the callback async or switch to fake timers before merge.
  - confidence: 0.99
- **[P2] Use a TTL that can expire in the test:** `src/agents/pi-embedded-runner/resource-loader.test.ts:229-242`
  - body: The test sets `OPENCLAW_RESOURCE_LOADER_CACHE_TTL_MS` to `100`, but `resolveCacheTtlMs()` clamps positive values to at least 10,000 ms. Waiting 150 ms will not expire the entry, so the expectation remains wrong after the callback is made async.
  - confidence: 0.98
- **[P2] Skip reload on valid cache hits:** `src/agents/pi-embedded-runner/resource-loader.ts:132-134`
  - body: Returning a cached loader still sends both callers through `await resourceLoader.reload()`, and upstream `reload()` immediately calls `packageManager.resolve()`. Cache hits therefore keep paying the synchronous discovery cost this PR is meant to remove.
  - confidence: 0.97
- **[P2] Keep run-specific loader inputs fresh:** `src/agents/pi-embedded-runner/resource-loader.ts:132-134`
  - body: The cached loader was constructed with the first run's `settingsManager` and `extensionFactories`, but later calls with the same workspace and agent dir can have different compaction or context-pruning config. Reusing that loader can omit required inline extensions or reload resources from stale settings.
  - confidence: 0.93
