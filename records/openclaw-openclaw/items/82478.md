## Summary

Keep open. This PR targets a real embedded resource-loader hotspot in current main, but the branch is not mergeable: one new test does not parse, cache hits can reuse stale per-run loader inputs, and the call sites still execute the expensive reload path, so the claimed latency win is not achieved. After-fix real behavior proof is also missing.

## What This Changes

This PR adds an in-memory `DefaultResourceLoader` cache for embedded attempt and compaction runs, marks reload completion in both call sites, and adds cache behavior tests.

## Best Possible Solution

Rework embedded resource loading so expensive Pi package discovery is avoided only when the current run's settings, extension factories, and resource inputs are still valid, or move reusable prepared resource facts earlier per the architecture policy, then prove before/after latency in a real embedded run.

## Reproduction Assessment

Yes for the structural hot path: current main constructs an embedded Pi resource loader and awaits `reload()`, and pi-coding-agent 0.74.0 reload always calls `packageManager.resolve()`. The exact 5-9 second magnitude is not reproduced here and currently comes from the PR's pre-fix logs.

## Solution Assessment

No. The proposed patch still reloads cached loaders, ignores per-run `settingsManager` and `extensionFactories`, and adds a non-parsing test; the safer direction is a cache or prepared-resource design that preserves per-run boundaries and has after-fix benchmark proof.

  - body: This new test contains `await` inside a non-async `it` callback, so the test file will not parse and the changed checks cannot run. Mark the callback `async` or use fake timers.
  - confidence: 0.99
- **[P2] Key cached loaders by the per-run inputs they own:** `src/agents/pi-embedded-runner/resource-loader.ts:134`
  - body: The cache returns a loader created with the first run's `settingsManager` and `extensionFactories` for every later run with the same cwd/agentDir. Those inputs are constructor-owned by `DefaultResourceLoader`, so later config or session-specific factories can be ignored until TTL expiry.
  - confidence: 0.91
- **[P2] Skip reload on cache hits before claiming the latency win:** `src/agents/pi-embedded-runner/run/attempt.ts:1846`
  - body: The claimed 5-9s cost is in `DefaultResourceLoader.reload()` and `packageManager.resolve()`, but both attempt and compaction still call `await resourceLoader.reload()` before marking the loader reloaded. A cache hit still pays the reported cost.
  - confidence: 0.94
