## Summary

Keep this PR open: the linked SDK packaging bug is valid, but this patch is not merge-ready because the proposed bundled dependency conflicts with the existing npm package release guard and the PR lacks after-fix pnpm-global proof.

## What This Changes

The PR adds `zod` to the root package `bundledDependencies` and also changes memory-core session-corpus paths from `memory/.dreams` to `.openclaw/dreams`.

## Best Possible Solution

Make `openclaw/plugin-sdk/zod` resolve from installed pnpm-global packages without shipping a root `node_modules` tree, then add package-artifact/global-install coverage and a changelog entry.

## Reproduction Assessment

Yes. Source and related issue evidence provide a high-confidence source reproduction: current `src/plugin-sdk/zod.ts` emits a bare zod re-export and the public package export points plugin consumers at that built artifact; I did not run a fresh pnpm global install in this read-only review.

## Solution Assessment

No. Adding `zod` as a root bundled dependency may make Node lookup work, but it conflicts with the package release guard; the safer fix is to make the SDK artifact self-contained or otherwise remove the unresolved bare runtime import and prove it from an installed tarball.
