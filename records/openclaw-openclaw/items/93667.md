## Summary

Keep open: current main and v2026.6.9 still do not implement Feishu elapsed/status footer rendering, the exact fix PR closed unmerged, and the remaining open Feishu footer PR only adds a custom footer string.

## What This Changes

The issue requests rendering elapsed-time and completion-status footers for built-in Feishu streaming cards when `channels.feishu.footer.elapsed/status` are configured.

## Best Possible Solution

Settle the Feishu-plugin-owned footer/status config, then implement it across schema, manifest metadata, streaming-card payload/update logic, docs, focused tests, and live Feishu/Lark proof while coordinating with the broader presentation tracker.

## Reproduction Assessment

Yes at source level: current main and v2026.6.9 create/update Feishu streaming cards with content plus a fixed note only, and neither schema nor manifest defines `footer.elapsed/status`. I did not run live Feishu/Lark reproduction in this read-only review.

## Solution Assessment

No for immediate automation: the implementation belongs in the Feishu plugin, but the public footer config shape should be settled before another fix PR is queued. The safer path is to coordinate this focused elapsed/status behavior with https://github.com/openclaw/openclaw/issues/60748.
