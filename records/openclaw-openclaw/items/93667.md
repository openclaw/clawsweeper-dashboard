## What This Changes

The issue requests that the built-in Feishu channel honor configured `footer.elapsed` and `footer.status` by rendering elapsed time and completion/error/stopped status in streaming card footers.

## Best Possible Solution

Ship one Feishu-plugin-owned fix that aligns config schema/metadata with streaming-card footer rendering for elapsed/status, backed by focused tests and redacted live Feishu proof.
