## Summary

Keep open: current main still matches the source-level failure shape, and no canonical open PR or merged fix supersedes it. The fix belongs in shared SSRF/provider transport plus uncaught network-error classification.

## What This Changes

The issue reports a gateway crash when SSRF-pinned provider fetches rotate from IPv4 to an unreachable IPv6 address for dual-stack Anthropic DNS, with `EADDRNOTAVAIL` not classified as benign.

## Best Possible Solution

Fix shared SSRF/provider transport to keep DNS pinning while avoiding unreachable AAAA selection in IPv4-working environments, classify `EADDRNOTAVAIL` consistently, and cover the path with focused regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `api.anthropic.com` currently resolves to A and AAAA records, pinned lookup preserves both and rotates through them, and `EADDRNOTAVAIL` is not classified benign; I did not run the full gateway on an IPv6-unroutable host.

## Solution Assessment

Yes. The maintainable fix is narrower than a new config knob: adjust the shared pinned dispatcher/family-selection behavior and error classification while preserving SSRF safeguards, then add regression tests and live proof if feasible.

Current main routes provider requests through `buildGuardedModelFetch` -> `fetchWithSsrFGuard` -> pinned SSRF dispatcher; `createPinnedLookup` preserves mixed A/AAAA results and round-robins, so a later lookup can return unreachable IPv6 on dual-stack DNS with no IPv6 route, and `EADDRNOTAVAIL` is missing from benign/transient classification. Implement a narrow shared fix in the SSRF/provider transport path and uncaught network classifier. Preserve SSRF DNS pinning, do not add an `api.anthropic.com` host exception, do not add a new config option unless maintainers explicitly ask, and avoid broad cache layers. Add regression coverage for mixed A/AAAA pinned lookup/dispatcher behavior and `EADDRNOTAVAIL` classification. Add a user-facing changelog entry if the resulting behavior is shipped as a fix. If feasible, include live or Crabbox proof on a host where DNS returns AAAA but global IPv6 is unreachable.
