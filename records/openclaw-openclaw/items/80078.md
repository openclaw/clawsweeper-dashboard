The issue requests a Gateway crash fix for IPv6-unreachable networks where SSRF DNS pinning can rotate from A to AAAA records and EADDRNOTAVAIL is not treated as benign.

## Best Possible Solution

Fix the shared SSRF/provider transport path so pinned lookup does not rotate into unreachable IPv6 on IPv4-working hosts, classify EADDRNOTAVAIL consistently, and cover both with focused regression tests.
