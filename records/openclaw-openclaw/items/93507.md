## Summary

Keep open: current main and the latest release still lack the requested quick-create model selector and existing-job configured/default model labels, while open PRs are still competing candidate fixes rather than merged implementation.

## What This Changes

The issue requests Control UI support for choosing a cron job model during creation, storing it on the cron payload, and clearly showing configured or default models for existing cron jobs.

## Best Possible Solution

Land the full-scope Control UI reuse of the existing payloadModel contract, preferably through https://github.com/openclaw/openclaw/pull/95341 after proof is accepted, then resolve narrower overlapping work deliberately.

## Reproduction Assessment

Yes. Source inspection of current main shows the quick-create wizard has no model field/input/mapping and existing job rendering omits configured/default model labels, while the advanced cron form already persists payloadModel.

## Solution Assessment

Yes. The best solution is a narrow Control UI change that reuses payloadModel and payload.model for quick-create selection plus row/detail display, without changing cron runtime, config schema, or provider resolution.
