## What This Changes

Add Web UI support for selecting a cron job model during creation and clearly showing each job's configured or default model in cron list/detail views.

## Best Possible Solution

Review, prove, and land or decline https://github.com/openclaw/openclaw/pull/95341, then close this issue once the focused UI behavior is on main; keep the broader main-session systemEvent override question in https://github.com/openclaw/openclaw/issues/42631 separate.

## Reproduction Assessment

Yes. Source inspection of current main shows no quick-create model field/input/mapping and no configured/default model labels in cron job rows/details, while the existing advanced cron form persists payloadModel.

## Solution Assessment

Yes for the likely fix shape. The narrow maintainable path is to reuse the existing payload.model/payloadModel contract in the Control UI instead of changing cron runtime, config schema, or model resolution.
