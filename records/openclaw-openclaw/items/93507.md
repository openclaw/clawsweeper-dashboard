Keep open: current main and the latest release still lack the requested quick-create model selector and configured/default model display, and the issue now has open implementation candidates that should be reviewed or resolved before the issue closes.

## What This Changes

The issue requests Control UI support for selecting a cron job model during creation, persisting it on the cron payload, and showing configured/default model labels in cron job list and detail views.

## Best Possible Solution

Land one focused Control UI change, preferably the broader candidate or equivalent, that reuses payloadModel, covers quick-create selection plus row/detail model labels, and then close this issue after merge.

## Reproduction Assessment

Yes. Source inspection of current main shows the quick-create wizard has no model field, input, or payloadModel mapping, while the advanced editor already supports payloadModel.

## Solution Assessment

Yes for the scoped UI path. The narrow maintainable solution is to reuse the existing payloadModel contract and model suggestions, add quick-create input/mapping, and render configured/default labels without changing cron runtime schema.
