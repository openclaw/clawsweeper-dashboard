## What This Changes

The PR adds an error listener for debug-proxy upstream HTTP responses, guards 502 status writes after headers are sent, and adds a regression test for origin aborts.

## Best Possible Solution

Land the narrow upstream response error handler and regression test after normal maintainer review and required checks, without expanding this into a broader proxy refactor.

## Reproduction Assessment

Yes, source-reproducible with high confidence. Current main consumes `upstreamRes` without an error listener, and Node throws unhandled EventEmitter `error` events when no listener is attached.

## Solution Assessment

Yes. Adding the listener where `upstreamRes` is consumed is the narrow owner-boundary fix; request-level and CONNECT socket errors already have sibling handlers.
