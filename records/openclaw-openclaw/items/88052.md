## What This Changes

The branch adds `upstreamRes.on("error")` handling in the debug proxy HTTP forwarding path and adds a regression test for an origin that aborts mid-response.

## Best Possible Solution

Land the narrow upstream response error handler and focused regression test after required checks pass, without expanding this PR into a broader proxy refactor.

## Reproduction Assessment

Yes from source inspection, though I did not run a live repro in this read-only review. Current main reads `upstreamRes` without an error listener, and Node's EventEmitter contract makes an unhandled `'error'` process-fatal.

## Solution Assessment

Yes. Adding the error listener at the `upstreamRes` owner boundary is the narrowest maintainable fix, and the existing request-level and CONNECT error handlers explain why the sibling paths do not need the same edit.
