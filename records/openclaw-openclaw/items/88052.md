## What This Changes

Adds an upstream response `error` listener in the debug proxy HTTP forwarding path and a Vitest case for an origin that aborts mid-response.

## Best Possible Solution

Rebase or resolve conflicts, preserve the upstream response error handler and focused regression coverage, then run the targeted proxy test and changed checks before merge.

## Reproduction Assessment

Yes. Source inspection on current main shows the upstream `IncomingMessage` has `data` and `end` listeners but no `error` listener, and the Node EventEmitter contract makes an unhandled `error` event process-fatal.

## Solution Assessment

Yes. The response callback owns `upstreamRes`, and adding the missing stream error listener is the narrowest maintainable fix; buffering whole responses just to guarantee a 502 after partial upstream data would broaden debug proxy behavior.
