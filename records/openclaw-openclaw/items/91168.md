## Summary

Keep open: current main and the latest release still show the TUI `/model` picker description from the model name, but this PR is not merge-ready because it adds a parallel exact-key runtime lookup that can disagree with execution policy, changes a public Swift initializer compat surface, and still has only test/typecheck proof. The broader related path in https://github.com/openclaw/openclaw/pull/90328 is useful and proof-positive, but it is still open and does not by itself land the TUI behavior from this PR.

## What This Changes

The branch adds a TUI/gateway `runtimeLabel` model-choice field, populates it as `CLI` for exact configured CLI runtime pins, and uses it as the `/model` picker description.

## Best Possible Solution

Expose runtime labels by reusing the canonical runtime-policy metadata path for TUI and gateway model choices, preserve Swift source compatibility, and align with the broader metadata contract if maintainers choose https://github.com/openclaw/openclaw/pull/90328 as the API direction.
