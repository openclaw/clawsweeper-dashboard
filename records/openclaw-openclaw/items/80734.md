Keep open: current main does not already provide the per-worktree lock scope, and this PR is a plausible narrow implementation with sufficient proof and no blocking code findings, but it still needs maintainer acceptance of the new local policy env var plus exact-head CI completion.

## What This Changes

The PR adds `OPENCLAW_HEAVY_CHECK_LOCK_SCOPE=worktree`, docs, and a focused test so local heavy-check locks can be scoped to the current worktree while Git-common-dir locking remains the default.

## Best Possible Solution

Land the opt-in if maintainers accept the local lock-scope contract, preserving Git-common-dir serialization as the default and coordinating with the active heavy-check hardening work.
