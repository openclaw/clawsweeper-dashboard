## What This Changes

This PR adds `Page.bringToFront` before direct CDP screenshot capture and extends CDP screenshot mocks/tests for activation ordering and activation-failure tolerance.

## Best Possible Solution

Land the activation-based fix if maintainers accept screenshot foregrounding; otherwise require a focused restore-previous-active-tab variant with matching ordering and failure-tolerance coverage.

## Reproduction Assessment

Yes, from source and contributor proof, but I did not run a local browser repro in this read-only review. Current main reaches `Page.captureScreenshot` without activation, and the PR body gives live CDP before/after timings for a backgrounded headless tab.

## Solution Assessment

Unclear until maintainers decide the focus contract. `Page.bringToFront` is the narrowest CDP-level mitigation I found, but restoring the prior active tab is safer if screenshots must not leave the target foregrounded.
