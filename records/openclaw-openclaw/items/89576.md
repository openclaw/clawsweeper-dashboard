## Summary

Keep open: this is a focused, proof-backed browser screenshot bug fix, but merging it intentionally changes multi-tab foreground state and needs maintainer compatibility acceptance rather than conservative cleanup closure.

## What This Changes

Adds `Page.bringToFront` before direct CDP screenshot capture and updates CDP screenshot mocks/tests for activation ordering and activation-failure handling.

## Best Possible Solution

Land the activation-based fix if maintainers accept screenshot foregrounding; otherwise request a focused restore-previous-tab variant with the same ordering and failure-tolerance coverage.

## Reproduction Assessment

Yes, from source and contributor live CDP proof, though I did not rerun a local browser repro in this read-only review. Current main reaches `Page.captureScreenshot` without activation, and the PR body gives before/after timings for a backgrounded headless tab.

## Solution Assessment

Unclear until maintainers decide the focus contract. `Page.bringToFront` is the narrowest CDP-level mitigation I found, but restoring the prior active tab is safer if screenshots must not leave the target foregrounded.
