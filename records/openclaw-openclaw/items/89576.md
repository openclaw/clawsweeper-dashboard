## Summary

Keep open: current main still reaches direct CDP screenshot capture without activation and no merged replacement owns this exact fix. ## What This Changes

The PR adds `Page.bringToFront` before direct CDP screenshot capture and updates CDP screenshot mocks/tests for activation ordering and activation failure.

## Best Possible Solution

Land the activation-based fix only after maintainers choose the browser screenshot focus contract; if foregrounding is not acceptable, add restore-previous-tab behavior with focused coverage.

## Solution Assessment

Unclear until the focus contract is decided. `Page.bringToFront` is a plausible minimal CDP fix, but whether screenshot should leave the target foregrounded or restore the prior active tab is a product/compatibility decision.
