## What This Changes

The issue requests that QMD-backed zero-hit `memory_search` calls return promptly instead of synchronously forcing QMD update work inside the interactive tool call.

## Best Possible Solution

Land one narrow backend-aware fix that preserves non-QMD zero-hit retry behavior while keeping QMD refresh asynchronous, debounced, or explicit outside the interactive `memory_search` deadline.

## Reproduction Assessment

Yes, at source level: current main still awaits a forced sync after zero `memory_search` hits, and QMD sync awaits update work with a longer timeout budget. The reporter also provided live v2026.6.5 confirmation after setup drift was fixed.

## Solution Assessment

Yes, the best fix shape is backend-aware zero-hit repair: keep the existing non-QMD sync-and-retry path but prevent QMD update maintenance from blocking the interactive tool call.
