## What This Changes

QMD-backed zero-hit `memory_search` should return promptly instead of forcing synchronous QMD maintenance inside the interactive tool call.

## Best Possible Solution

Review and land the backend-aware QMD guard from the open fix PRs, preserving non-QMD zero-hit retry behavior and keeping any QMD refresh asynchronous, debounced, or explicit outside the interactive tool deadline.

## Solution Assessment

Yes, the best fix shape is backend-aware zero-hit repair: keep non-QMD sync-and-retry behavior while preventing QMD update maintenance from running synchronously inside `memory_search`. The open PRs implement that narrow direction; this issue should remain open until one is merged or replaced.
