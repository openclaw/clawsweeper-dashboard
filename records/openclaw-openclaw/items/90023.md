## What This Changes

The issue reports that QMD-backed `memory_search` zero-hit queries synchronously force a memory sync, stalling interactive turns.

## Best Possible Solution

Land one canonical memory-core fix, preferably the linked proof-supplied PR or an equivalent backend-aware guard that skips synchronous QMD zero-hit forced sync while preserving non-QMD retry behavior, then close this issue from the merged PR.

## Reproduction Assessment

Yes. Source inspection shows that a QMD-backed zero-hit `memory_search` still awaits `manager.sync({ reason: "search", force: true })`, and the reporter reconfirmed the same behavior on a live v2026.6.5 deployment.

## Solution Assessment

Yes. The best fix direction is a narrow backend-aware guard that keeps QMD update maintenance out of the interactive zero-hit search path while preserving existing non-QMD retry behavior.
