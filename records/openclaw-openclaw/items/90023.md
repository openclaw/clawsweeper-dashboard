## Summary

Keep open. Current main and the latest release still contain the zero-hit forced-sync path for QMD-backed `memory_search`, and open PRs remain candidate fixes rather than merged provenance.

## What This Changes

Prevent QMD-backed `memory_search` zero-hit results from running a synchronous forced sync in the interactive tool path.

## Best Possible Solution

Land one focused memory-core fix that keeps QMD zero-hit refresh work out of the user-visible `memory_search` wait path, while preserving intentional non-QMD retry behavior or moving QMD freshness repair to a bounded async path.

## Reproduction Assessment

Yes from source and reporter live updates: current main still forces `manager.sync({ reason: "search", force: true })` after zero-hit `memory_search`, and QMD sync awaits update work. I did not run a live large-corpus QMD timing repro in this read-only pass.

## Solution Assessment

Yes, the narrow fix direction is best: keep QMD update work out of the interactive zero-hit search path or bound it, while preserving the existing non-QMD retry behavior unless maintainers intentionally change that contract.
