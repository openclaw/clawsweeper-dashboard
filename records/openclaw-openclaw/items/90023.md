## Summary

Keep open. Current main and the latest release still contain the synchronous zero-hit forced-sync path in `memory_search`, and two open closing PRs already own the concrete fix path.

## What This Changes

The issue reports that QMD-backed `memory_search` zero-hit queries synchronously force a memory sync, causing interactive turns to stall or time out.

## Best Possible Solution

Review and land the backend-aware guard from the open memory-core fix PRs, preserving non-QMD zero-hit retry behavior while keeping QMD refresh asynchronous, debounced, or explicit outside the interactive tool deadline.

## Reproduction Assessment

Yes, at source level: current main still awaits a forced sync after zero `memory_search` hits, and QMD sync awaits update work. The reporter also supplied live `v2026.6.5` confirmation, but I did not run an independent live QMD timing repro in this read-only pass.

## Solution Assessment

Yes, the narrow maintainable fix is backend-aware zero-hit repair: keep non-QMD sync-and-retry behavior while preventing QMD update maintenance from running synchronously inside `memory_search`. The open PRs implement that direction, so this issue should remain open until one lands or is replaced.
