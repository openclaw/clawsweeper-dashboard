## Summary

Keep open: the PR fixes the selected-day leak at the right backend boundary, but it also treats semantic all-time ranges as timestamp-required, so legacy untimestamped usage can disappear from the Dashboard All preset and gateway all-range summaries.

## What This Changes

The branch changes session usage cache/direct aggregation to skip untimestamped transcript entries when date bounds are supplied and adds cache/direct regression tests.

## Best Possible Solution

Keep the backend selected-day fix, but treat semantic all-time requests such as no bounds or startMs === 0 as allowed to include untimestamped legacy usage, with regression coverage for cache and direct paths.

## Reproduction Assessment

Yes by source inspection: current main lets untimestamped usage rows pass finite-range filters into totals/modelUsage, and the PR adds tests for that selected-day leak. I did not run tests because this review is read-only.

## Solution Assessment

No as written. The backend boundary is the right place to fix selected-day summaries, but the patch should exempt semantic all-time ranges from the timestamp requirement before merge.
