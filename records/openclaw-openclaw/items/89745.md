## What This Changes

The PR changes session usage cache/direct summary filtering so finite date ranges skip untimestamped transcript records while preserving all-time summaries, with regression tests for cached and direct paths.

## Best Possible Solution

Land the backend range-filter fix with the cache/direct regression coverage, then close the linked dashboard usage issue as implemented.

## Reproduction Assessment

Yes, by source inspection: current main's cache and direct summary builders only reject timestamped rows outside a finite range, so untimestamped usage can still be merged into bounded totals and modelUsage. I did not execute tests because this review is read-only.

## Solution Assessment

Yes. Fixing the shared backend session summary builders is the best layer because gateway and UI aggregation consume their returned modelUsage; UI-only filtering would leave API/cache behavior inconsistent.
