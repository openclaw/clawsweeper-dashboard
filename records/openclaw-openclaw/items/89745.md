## Summary

Keep open: the backend aggregation boundary is the right place for this fix, but the patch also treats all-time/range-all requests as timestamp-required and can undercount legacy untimestamped usage.

## What This Changes

The branch changes `src/infra/session-cost-usage.ts` to skip untimestamped transcript entries for bounded session usage summaries and adds cache/direct regression coverage in `src/infra/session-cost-usage.test.ts`.

## Best Possible Solution

Keep the backend cache/direct fix, but make the timestamp requirement apply only to genuinely bounded selected-date ranges and add all-time/range-all regression coverage.

## Reproduction Assessment

Yes. The original selected-day bug is source-reproducible from current main, and the new all-time regression is source-reproducible because `range: "all"` and the UI All preset pass `startMs: 0` into the PR's timestamp-required path.

## Solution Assessment

No, not yet. Fixing the backend cache/direct summary builders is the right boundary, but the implementation needs to distinguish finite selected ranges from all-time ranges before it is the best fix.
