## What This Changes

The branch skips untimestamped transcript entries for finite session usage ranges in both cache-derived and direct summaries, and adds cache/direct regression fixtures.

## Best Possible Solution

Land the backend range-filter fix with the paired cache/direct regression coverage, while preserving all-time untimestamped summaries.

## Reproduction Assessment

Yes by source inspection: current main only rejects timestamped rows outside the finite range, then still aggregates untimestamped usage into totals/modelUsage. I did not run the regression tests because this review is read-only.

## Solution Assessment

Yes. Fixing the backend cache and direct summary builders is the best boundary because gateway and UI aggregation consume their modelUsage values; UI-only filtering would leave API/cache behavior inconsistent.
