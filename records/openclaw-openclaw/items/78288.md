## Summary

Keep open: this is the right formatter surface for the linked exec-node display request, but the submitted patch can show a node label for non-node exec hosts and needs a narrow repair before merge.

## What This Changes

The PR appends a node label to exec transparency detail strings, adds formatter tests, and adds an Unreleased changelog entry.

## Best Possible Solution

Land a narrow formatter change that shows node context only for node-target exec calls, preserves cwd/raw/explain formatting, and covers explicit non-node hosts.

## Reproduction Assessment

Yes. Source inspection shows current main routes exec transparency through resolveExecDetail, which ignores node, while the schema and runtime carry node only for node-target execution.

## Solution Assessment

No as submitted. Updating resolveExecDetail is the right surface, but the node label needs to be gated so explicit gateway or sandbox calls with a stray node arg do not display false target information.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88
