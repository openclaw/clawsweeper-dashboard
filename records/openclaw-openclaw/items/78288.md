## Summary

Keep this open: the PR targets a valid exec transparency improvement, but the current diff can show a node label for explicit non-node exec calls, which would misrepresent where the command runs.

## What This Changes

This PR appends a trimmed `node` value to exec transparency details, adds formatter coverage, and adds an Unreleased changelog entry.

## Best Possible Solution

Land a narrow formatter change that shows node context only for node-target exec calls, preserves cwd/raw/explain formatting, and covers explicit non-node hosts.

## Reproduction Assessment

Yes. Source inspection shows current main routes exec transparency through `resolveExecDetail`, which ignores `node`, while the schema and runtime carry `node` only for node-target execution.

## Solution Assessment

No as submitted. Updating `resolveExecDetail` is the right surface, but the node label needs to be gated so explicit gateway or sandbox calls with a stray `node` arg do not display false target information.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Gate the node label to node-target exec calls:** `src/agents/tool-display-exec.ts:401-402`

As written, args like `{ host: "gateway", node: "raspberrypi" }` would run on the gateway but display `node: raspberrypi`, so please gate the label and add a regression for explicit non-node hosts.
