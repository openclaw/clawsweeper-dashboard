## Summary

The PR is still useful and focused, but it should remain open because the formatter currently appends `node: <name>` whenever a `node` argument is present, even if the exec host is explicitly not `node`. That can make the transparency message claim a remote target for gateway/sandbox/auto calls that would not execute on that node.

## What This Changes

The branch adds a target-node fragment to exec transparency details, adds formatter tests for node/cwd cases, and adds a changelog entry.

## Best Possible Solution

Land the formatter change after the node fragment is gated to `host === "node"` and explicit non-node host regression coverage is added, while preserving cwd, raw-command, and explain-mode formatting.

## Reproduction Assessment

Yes. The supplied diff and PR body show the intended formatter path, and source-level review of the patch shows the proposed node fragment is keyed only on `record.node`, not the actual exec host.

## Solution Assessment

No as submitted. Updating `resolveExecDetail` is the right surface, but the label must be tied to node-target exec args so transparency text stays truthful.
