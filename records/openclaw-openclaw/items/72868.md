## Summary

Keep this PR open: it addresses a source-reproducible Google thinking-off bug, but the current patch is still over-broad for non-budget Google model IDs and lacks the required changelog entry plus auditable real behavior proof.

## What This Changes

The PR changes Google thinking payload sanitization to seed or force `thinkingConfig.thinkingBudget = 0` for `thinkingLevel: "off"`, adds Google wrapper regression tests, switches Gemma 4 matching to a shared helper, and updates the Plugin SDK API baseline hash.

## Best Possible Solution

Revise the sanitizer so both off-mode injection branches require Gemini 2.5 budget capability, keep the Pro/Gemma/Gemini 3 exclusions, add negative coverage for non-budget Google IDs, add the single-line changelog entry, and attach redacted live output proving the faster thinking-off path.

## Reproduction Assessment

Yes, as a source-level reproduction. On current main, a direct sanitizer call with `{ config: {} }`, `modelId: "gemini-2.5-flash"`, and `thinkingLevel: "off"` leaves the payload without `thinkingConfig`.

## Solution Assessment

No. The shared sanitizer is the right layer, but the patch should positively gate zero-budget injection to budget-capable Gemini 2.5 models and still needs changelog plus auditable real behavior proof.
