## Summary

Keep open. Current main still misses the Gemini 2.5 Flash thinking-off payload path, but this PR is not merge-ready because its exclusion-based guard also injects `thinkingBudget: 0` for non-budget Google model IDs and the user-facing fix lacks changelog coverage.

## What This Changes

The branch updates shared Google thinking payload sanitization to seed or force `thinkingConfig.thinkingBudget = 0` for `thinkingLevel: "off"` requests and adds regression coverage for config, generationConfig, no-container, and excluded-model paths.

## Best Possible Solution

Keep the shared sanitizer fix, but make the forced-zero path a positive Gemini 2.5 budget-capability path while preserving the existing Gemini 3, Gemma 4, and thinking-required exceptions.

## Reproduction Assessment

Yes. Code inspection gives a high-confidence reproduction: on current main, `sanitizeGoogleThinkingPayload({ payload: { config: {} }, modelId: "gemini-2.5-flash", thinkingLevel: "off" })` leaves the payload unchanged because the inner sanitizer returns when `thinkingConfig` is absent.

## Solution Assessment

No, not as currently written. The shared sanitizer is the right layer, but both injection branches should require `isGoogleGemini25ThinkingBudgetModel(params.modelId)` and the branch needs negative coverage for non-budget model IDs plus a changelog entry.
