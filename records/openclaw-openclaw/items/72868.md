## Summary

Keep open: the reported Google thinking-off bug is valid on current main, but the PR head still injects `thinkingBudget=0` too broadly, lacks the required changelog entry, and needs attachable real behavior proof before merge.

## What This Changes

The PR changes shared Google thinking payload sanitization to seed or force `thinkingConfig.thinkingBudget = 0` for `thinkingLevel: "off"`, updates Google wrapper regression tests, switches Gemma 4 matching to a shared helper, and regenerates the Plugin SDK API baseline hash.

## Best Possible Solution

Revise this PR so both injection branches require the positive Gemini 2.5 budget-capability helper, add negative coverage for non-budget Google IDs, keep the Gemma/Gemini 3 exclusions, add a single-line Unreleased Fixes changelog entry, and attach redacted live run output showing the improved thinking-off path.

## Reproduction Assessment

Yes. Source inspection on current main shows the sanitizer returns when `thinkingConfig` is absent, so a direct call with `{ config: {} }`, `modelId: "gemini-2.5-flash"`, and `thinkingLevel: "off"` leaves the payload unchanged.

## Solution Assessment

No. The shared sanitizer is the right layer, but the current predicate should be a positive Gemini 2.5 budget-capability gate rather than broad negative exclusions, and the PR still needs changelog plus real behavior proof.
