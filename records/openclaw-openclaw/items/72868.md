## Summary

Keep this PR open. The original Gemini 2.5 Flash thinking-off bug is real on current main and the PR is pointed at the right shared sanitizer, but the current diff still injects `thinkingBudget: 0` for non-budget Google model IDs and is missing the required user-facing changelog entry.

## What This Changes

The PR updates shared Google thinking payload sanitization to seed or force `thinkingConfig.thinkingBudget = 0` for `thinkingLevel: "off"` requests and adds regression tests for config, generationConfig, no-container, and excluded-model paths.

## Best Possible Solution

Revise this PR in place: keep the shared sanitizer fix for Gemini 2.5 Flash and Flash-Lite thinking-off requests, require the positive Gemini 2.5 budget-capability helper in both injection branches, add non-budget negative coverage, and add a single-line Unreleased Fixes changelog entry.

## Reproduction Assessment

Yes. On current main, unit-calling `sanitizeGoogleThinkingPayload({ payload: { config: {} }, modelId: "gemini-2.5-flash", thinkingLevel: "off" })` leaves the payload unchanged; on PR head, the same shape with `modelId: "gemini-2.0-flash"` would receive `thinkingBudget: 0` because the new guard is only exclusion-based.

## Solution Assessment

No, not as currently written. The shared sanitizer is the right layer, but the maintainable fix is to gate forced budget injection on `isGoogleGemini25ThinkingBudgetModel(...)` in both branches and cover the non-budget negative case.
