## Summary

Keep open. The PR fixes a real current-main gap for Gemini 2.5 Flash thinking-off calls, but the patch is not merge-ready because its forced-zero guard is still exclusion-based and would affect non-budget Google IDs; it also lacks the required user-facing changelog entry.

## What This Changes

The PR changes shared Google thinking payload sanitization to seed or force `thinkingConfig.thinkingBudget = 0` for `thinkingLevel: "off"` and adds embedded-runner regression tests for eligible and excluded model shapes.

## Best Possible Solution

Revise this PR in place so both forced-zero injection branches require the positive Gemini 2.5 budget-capability helper, keep the existing excluded-model behavior, add non-budget negative coverage, and add one Unreleased Fixes changelog line.

## Reproduction Assessment

Yes. On current main, a pure unit call with `{ payload: { config: {} }, modelId: "gemini-2.5-flash", thinkingLevel: "off" }` leaves the payload unchanged; on PR head, the same negative-exclusion predicate would also inject `thinkingBudget: 0` for a non-budget ID such as `gemini-2.0-flash`.

## Solution Assessment

No, not as currently written. The shared sanitizer is the right layer, but the maintainable fix is a positive `isGoogleGemini25ThinkingBudgetModel(...)` gate in both new injection branches plus focused coverage and changelog release-note coverage.
