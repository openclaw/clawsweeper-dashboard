## Summary

Keep open. The branch addresses a real current-main Google thinking-off gap, but its current off-mode predicate is still too broad for non-Gemini-2.5 budget model IDs and the user-facing fix is missing the required changelog entry.

## What This Changes

The PR changes shared Google thinking payload normalization to create or force `thinkingConfig.thinkingBudget = 0` when `thinkingLevel` is off, updates Gemma 4 model-id matching, and adds regression tests for config, generationConfig, and no-container payloads.

## Best Possible Solution

Revise this PR: keep the shared sanitizer fix for Gemini 2.5 Flash and Flash-Lite, but require the existing Gemini 2.5 budget-family helper in both off-mode injection branches, preserve the Gemini 2.5 Pro, Gemini 3, and Gemma 4 exclusions, add non-budget regression coverage, and add the Unreleased Fixes changelog entry. Leave Vertex/Gemini CLI wrapper widening to #73416.

## Reproduction Assessment

Yes. The original bug is reproducible by inspecting or unit-calling `sanitizeGoogleThinkingPayload({ payload: { config: {} }, modelId: "gemini-2.5-flash", thinkingLevel: "off" })`, which current main leaves unchanged because it returns when `thinkingConfig` is absent. The PR's introduced blocker is likewise reproducible with a non-budget ID such as `gemini-2.0-flash` at PR head.

## Solution Assessment

No, not as currently written. The shared sanitizer is the right layer, but the maintainable fix is to gate forced `thinkingBudget=0` on Gemini 2.5 budget-capable models instead of applying it to every non-excluded Google model ID.
