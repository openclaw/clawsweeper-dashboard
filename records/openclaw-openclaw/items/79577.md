## Summary

Keep open: the report matches current source, where an emitted restart token can remain in-flight if the SIGUSR1 handler rejects authorization, causing later valid restarts to coalesce instead of retrying.

## What This Changes

Fix the gateway update-run restart state so an ignored update SIGUSR1 cannot leave future gateway.restart requests permanently coalesced.

## Best Possible Solution

Make update-run restarts use a durable or shared authorization/handoff path, clear or fail the emitted token when SIGUSR1 is rejected, and surface restart failure instead of leaving the old gateway running.

## Solution Assessment
