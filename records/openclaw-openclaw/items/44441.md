## Summary

Keep open. Current main still lacks the proposed macOS watchdog runbook and the related watchdog PRs are still open, but this PR is not merge-ready because the sample does not match the advertised unhealthy-process recovery behavior and omits the managed service environment needed for non-default profiles.

## What This Changes

This PR adds a macOS docs section for an optional operator-managed LaunchAgent watchdog and links it from the updating rollback docs.

## Best Possible Solution

Revise the runbook to match current contracts: either document a reachability-only LaunchAgent with profile/config/state propagation, or defer to the canonical rescue-watchdog design before adding supported recovery guidance.

## Reproduction Assessment

Not applicable as a docs-only PR. The review has a high-confidence static check: current health exits successfully after Gateway reachability, and current service setup carries profile/config/state variables that the sample LaunchAgent omits.

## Solution Assessment

No, not as currently written. The docs-only approach is narrow and the PATH hardening is useful, but the recovery predicate and environment example need to match the existing Gateway health and managed-service contracts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Align the watchdog gate with the documented failure mode:** `docs/platforms/mac/bundled-gateway.md:107`
  - body: The new section says this covers process-alive-but-unhealthy cases, but this check exits before remediation whenever `openclaw health --json` can reach the Gateway. Current health handling treats channel issues as nonfatal, so degraded-but-responsive cases skip `doctor` and restart; either frame this as reachability-only or use a predicate that fails for the intended state.
  - confidence: 0.9
- **[P2] Propagate the managed service environment:** `docs/platforms/mac/bundled-gateway.md:133-137`
  - body: The LaunchAgent example only sets `PATH`, while managed Gateway services preserve `OPENCLAW_PROFILE`, `OPENCLAW_STATE_DIR`, and `OPENCLAW_CONFIG_PATH`. For the documented `ai.openclaw.<profile>` services, the watchdog can probe or restart the default Gateway instead of the intended service unless those variables are carried through.
  - confidence: 0.84
