## Summary

Keep open. The docs-only patch still has two correctness blockers: the example watchdog is described as process-alive unhealthy recovery while it gates only on Gateway reachability, and the LaunchAgent example does not carry the managed service environment for profile/custom-state installs.

## What This Changes

The branch adds an optional macOS LaunchAgent Gateway watchdog runbook and links it from the update rollback docs.

## Best Possible Solution

Revise the runbook to match the current health and service-environment contracts, or fold the guidance into a maintainer-approved Gateway watchdog design before publishing it.

## Reproduction Assessment

Yes, for the PR defects by source inspection: the proposed script gates on `openclaw health --json` while current main treats Gateway reachability as success, and the plist omits service env fields current main preserves. I did not run a live macOS LaunchAgent.

## Solution Assessment

No, not as written. The safer docs path is to describe the sample as reachability-only, parse degraded JSON deliberately, propagate managed service environment, or wait for the broader watchdog direction to settle.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Align the watchdog gate with the promised failure mode:** `docs/platforms/mac/bundled-gateway.md:107-108`
  - body: The new section says this recovers process-alive-but-unhealthy cases, but `openclaw health --json` exits successfully whenever the Gateway RPC is reachable; current main reports channel issues as nonfatal. This means degraded-but-responsive Gateways skip the documented doctor/restart path unless the example parses the JSON state or narrows the claim to reachability failures.
  - confidence: 0.9
- **[P2] Carry the managed service environment into the watchdog:** `docs/platforms/mac/bundled-gateway.md:133-137`
  - body: The LaunchAgent example only forwards `PATH`, but managed Gateway services also carry profile, state-dir, and config-path environment. Non-default profile or custom state installs can make the watchdog probe or restart the default Gateway instead of the intended service.
  - confidence: 0.84
