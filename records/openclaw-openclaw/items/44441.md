## Summary

Keep open: the docs-only PR is useful in shape, but it still has correctness issues in the sample watchdog contract, lacks real macOS LaunchAgent proof, and overlaps unsettled Gateway watchdog work.

## What This Changes

The PR adds a macOS LaunchAgent watchdog recovery runbook to Gateway docs and links it from the update rollback docs.

## Best Possible Solution

Revise the runbook to match supported health and managed-service contracts, or fold it into the canonical Gateway watchdog direction before publishing.

## Reproduction Assessment

Yes, for the PR defects by source inspection: the sample gates on `openclaw health --json` while current main treats Gateway reachability as success, and the LaunchAgent example omits managed profile/config/state environment. I did not establish a live macOS LaunchAgent reproduction.

## Solution Assessment

No, not as written. The docs should either narrow the example to a reachability watchdog, deliberately parse degraded health state, and mirror managed-service environment, or wait for the broader watchdog design to settle.

- **[P2] Align the watchdog gate with the promised failure mode:** `docs/platforms/mac/bundled-gateway.md:107-109`
