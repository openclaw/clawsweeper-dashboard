## Summary

Keep open: this docs-only PR includes useful launchd PATH hardening after review, but the current head still documents an unhealthy-process watchdog that only gates on Gateway reachability, omits managed profile/state/config environment, and lacks real macOS LaunchAgent proof.

## What This Changes

The branch adds a macOS Gateway docs section for an optional operator-managed LaunchAgent watchdog and links it from the update rollback docs.

## Best Possible Solution

Revise the runbook to match current health and managed-service contracts, or defer it to the broader Gateway watchdog design before publishing.

## Reproduction Assessment

Yes, for the PR defects by source inspection: the sample gates on openclaw health --json while current main treats Gateway reachability as success and channel issues as nonfatal, and the LaunchAgent sample lacks the environment fields managed services persist. I did not establish a live macOS LaunchAgent reproduction.

## Solution Assessment

No, not as written. The docs should either narrow the example to a reachability watchdog, deliberately parse degraded health state, propagate managed service environment, or wait for the broader watchdog direction to settle.
