## Summary

Keep open. Current main still lacks the macOS watchdog runbook, and the PR is focused and docs-only, but source-backed review findings remain around the health predicate and profile/config targeting before it should merge.

## What This Changes

The PR adds an optional macOS LaunchAgent watchdog runbook to the macOS gateway docs and links it from the updating guide rollback section.

## Best Possible Solution

The docs should either describe a reachability-only LaunchAgent honestly or use a predicate that fails for the intended degraded-but-responsive state, and the example should target the same profile/config/state environment as the managed Gateway service. If OpenClaw wants productized recovery instead, the docs should land with that design rather than as a standalone runbook.

## Reproduction Assessment

Not applicable as a docs PR, but the review concerns have a high-confidence source reproduction path: current main lacks the proposed docs, and healthCommand explicitly treats successful Gateway RPC reachability as a successful command outcome.

## Solution Assessment

No, not as currently written. The docs-only approach is appropriately narrow, but the sample needs the health predicate and managed-service environment corrected before it is the safest maintainable guidance.
