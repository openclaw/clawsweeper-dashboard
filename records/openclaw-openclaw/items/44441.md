## What This Changes

The branch adds an optional macOS LaunchAgent watchdog runbook to the Gateway docs and links it from the update rollback docs.

## Best Possible Solution

Revise the runbook to match current health and service-environment contracts with real macOS LaunchAgent proof, or fold the guidance into the broader maintainer-approved Gateway watchdog design.

## Reproduction Assessment

Yes, for the PR defects by source inspection: the sample gates on openclaw health --json while current main treats Gateway reachability as success, and the LaunchAgent sample omits managed service environment fields. I did not establish a live macOS LaunchAgent reproduction.

## Solution Assessment

No, not as written. The docs should either narrow the example to a reachability watchdog, parse degraded health deliberately, propagate managed service environment, or wait for the broader watchdog direction to settle.
