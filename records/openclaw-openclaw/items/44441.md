## Summary

Keep open. Current main still lacks the proposed macOS watchdog runbook, but this PR is not merge-ready because the sample watchdog does not actually detect the degraded-but-responsive state it advertises and does not mirror the managed service profile/config environment. The earlier launchd PATH review issue was addressed in the PR head, so the remaining blockers are narrower than the first draft.

## What This Changes

The PR adds an optional macOS LaunchAgent watchdog recovery runbook to the Gateway on macOS docs and links it from the updating guide rollback section.

## Best Possible Solution

The docs should either present this as a reachability-only LaunchAgent and explicitly require operators to mirror the managed Gateway profile/config/state environment, or wait for the canonical rescue-watchdog design to define the supported recovery policy before adding a macOS runbook.

## Reproduction Assessment

Not applicable as a docs-only PR. The review concerns have a high-confidence static reproduction path: current main's health command exits successfully after a reachable Gateway RPC, and the service environment builder shows the profile/config/state variables the sample does not propagate.

## Solution Assessment

No, not as currently written. The docs-only approach is narrow, and the PATH hardening is useful, but the sample still does not match its advertised unhealthy-process behavior or the existing managed service environment contract.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P2] Gate the watchdog on the documented unhealthy state:** `docs/platforms/mac/bundled-gateway.md:107`
  - body: The new section says this watchdog covers process-alive-but-unhealthy cases, but `openclaw health --json` exits successfully when the Gateway RPC is reachable and treats channel issues as nonfatal. Degraded-but-responsive cases will skip the recovery block, so either frame this as reachability-only or use a predicate that fails for the intended state.
  - confidence: 0.9
- **[P2] Propagate the managed service environment:** `docs/platforms/mac/bundled-gateway.md:133-137`
  - body: This LaunchAgent example only sets PATH, so `health`, `doctor`, and `gateway restart` target the default profile/config/state. For profile-specific services documented earlier on this page, the watchdog can probe or restart the wrong Gateway unless the example carries OPENCLAW_PROFILE and relevant config/state env.
  - confidence: 0.84
