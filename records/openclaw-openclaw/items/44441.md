## Summary

Keep open with blocking review findings: current main does not already contain this runbook, and the PR example still conflicts with the current health-command and managed-service environment contracts despite the useful PATH hardening follow-up.

## What This Changes

This PR adds an optional macOS LaunchAgent watchdog recovery runbook and links it from the update rollback docs.

## Best Possible Solution

Revise the runbook to match supported contracts: either document a reachability-only LaunchAgent with profile/config/state propagation, or defer this guidance to the canonical rescue-watchdog work.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes, by static inspection of the PR diff against current main: `healthCommand` keeps exit success after Gateway RPC reachability, and managed Gateway services preserve profile/config/state variables that the sample LaunchAgent omits.

## Solution Assessment

Is this the best way to solve the issue? No, not as currently written; the PATH hardening is useful, but the narrow maintainable path is to align the predicate and environment example with current contracts or wait for the broader rescue-watchdog design.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Align the watchdog gate with the promised failure mode:** `docs/platforms/mac/bundled-gateway.md:107-109`
  - body: The new section says this covers process-alive-but-unhealthy cases, but `openclaw health --json` succeeds whenever the Gateway RPC is reachable and reports channel issues as nonfatal. This makes the watchdog skip remediation for degraded-but-responsive gateways; either frame the example as reachability-only or use a predicate that fails for the intended state.
  - confidence: 0.9
- **[P2] Carry the managed service environment into the watchdog:** `docs/platforms/mac/bundled-gateway.md:133-137`
  - body: The LaunchAgent example only forwards `PATH`, while managed Gateway services preserve `OPENCLAW_PROFILE`, `OPENCLAW_STATE_DIR`, and `OPENCLAW_CONFIG_PATH`. For `ai.openclaw.<profile>` services, this can make the watchdog probe or restart the default Gateway instead of the intended one.
  - confidence: 0.84
