## Summary

Keep open: current main does not include this macOS watchdog runbook, and the PR still has correctness blockers plus missing real behavior proof before merge. The related rescue-watchdog PRs are active but not a shipped or merged replacement for this specific docs change.

## What This Changes

This PR adds optional macOS LaunchAgent watchdog recovery documentation and links it from the update rollback docs.

## Best Possible Solution

Revise the runbook to match supported contracts: either make it a reachability-only LaunchAgent with profile/config/state propagation, or defer the guidance to the canonical rescue-watchdog design.

## Reproduction Assessment

Yes, for the PR defects by source inspection. The sample gates on `openclaw health --json` while current health exits successfully after Gateway RPC reachability, and the LaunchAgent sample omits service environment that current managed services preserve.

## Solution Assessment

No, not as written. The PATH hardening is useful, but the narrow maintainable path is to align the health predicate and LaunchAgent environment with current contracts or wait for the broader rescue-watchdog design.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Align the watchdog gate with the promised failure mode:** `docs/platforms/mac/bundled-gateway.md:107-109`
  - body: The section says this covers process-alive-but-unhealthy cases, but `openclaw health --json` succeeds whenever the Gateway RPC is reachable and reports channel issues as nonfatal. A degraded-but-responsive gateway will skip remediation here, so either frame this as reachability/timeout recovery or use a predicate that fails for the documented unhealthy state.
  - confidence: 0.9
- **[P2] Carry the managed service environment into the watchdog:** `docs/platforms/mac/bundled-gateway.md:133-137`
  - body: The LaunchAgent example only forwards PATH, while managed Gateway services preserve `OPENCLAW_PROFILE`, `OPENCLAW_STATE_DIR`, and `OPENCLAW_CONFIG_PATH`. For `ai.openclaw.<profile>` installs, this can make the watchdog probe or restart the default Gateway instead of the intended one.
  - confidence: 0.84
