## Summary

Keep open: the docs-only PR is not ready to merge because the sample overpromises process-alive unhealthy recovery while gating only on `openclaw health --json` exit status, omits profile/config/state environment for managed services, lacks real macOS LaunchAgent proof, and overlaps active Gateway watchdog design work.

## What This Changes

Adds an optional macOS LaunchAgent watchdog runbook to Gateway docs and links it from the update rollback docs.

## Best Possible Solution

Revise the runbook to match supported health and managed-service contracts, or defer the macOS docs until the canonical Gateway watchdog/recovery direction in https://github.com/openclaw/openclaw/pull/40791 and https://github.com/openclaw/openclaw/pull/53716 is settled.

## Reproduction Assessment

Yes, for the PR defects by source inspection: the sample gates on `openclaw health --json` even though current main treats reachable Gateway RPC as success, and the LaunchAgent example omits managed profile/config/state environment.

## Solution Assessment

No, not as written. The PATH hardening is useful, but the maintainable docs path must describe a reachability watchdog accurately, carry service environment, or wait for the broader watchdog direction.
