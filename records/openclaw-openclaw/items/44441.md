## Summary

Keep open. The docs idea is plausible, but the current branch still has source-proven contract mismatches, no observable macOS LaunchAgent proof, a dirty/conflicting merge state, and unresolved product overlap with https://github.com/openclaw/openclaw/pull/40791.

## What This Changes

The PR adds a macOS LaunchAgent watchdog recovery runbook to the Gateway docs and links it from the update rollback docs.

## Best Possible Solution

Revise the runbook to match supported health and managed-service contracts, or defer the macOS docs until the canonical rescue-watchdog design in https://github.com/openclaw/openclaw/pull/40791 is settled.

## Reproduction Assessment

Yes, for the PR defects by source inspection. The sample gates on `openclaw health --json` even though current main treats reachable Gateway RPC as success, and the LaunchAgent example omits managed profile/config/state environment.

## Solution Assessment

No, not as written. The PATH hardening is useful, but the maintainable docs path must either describe a reachability watchdog accurately and carry service environment, or wait for the broader rescue-watchdog direction.
