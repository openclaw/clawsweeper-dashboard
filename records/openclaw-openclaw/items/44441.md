## Summary

Keep this PR open: current main does not already contain the runbook, but the PR still has two source-backed correctness issues and lacks real behavior proof for the documented macOS LaunchAgent flow.

## What This Changes

This PR adds optional macOS LaunchAgent watchdog recovery documentation and links it from the update rollback docs.

## Best Possible Solution

Revise the runbook to match supported contracts, or defer it until the canonical external rescue-watchdog direction is settled.

## Reproduction Assessment

Yes, for the PR defects by source inspection. The sample gates on `openclaw health --json` even though current health treats reachable Gateway RPC as success, and the LaunchAgent example omits managed service env that current main preserves.

## Solution Assessment

No, not as written. The PATH hardening is useful, but the maintainable docs path must either describe a reachability watchdog accurately and carry service env, or wait for the broader rescue-watchdog design.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Align the watchdog gate with the promised failure mode:** `docs/platforms/mac/bundled-gateway.md:107-109`
  - body: The section promises recovery for process-alive-but-unhealthy cases, but this gate exits whenever `openclaw health --json` returns 0. Current main treats reachable Gateway RPC as success and reports channel issues as nonfatal, so degraded-but-responsive Gateways skip remediation.
  - confidence: 0.9
- **[P2] Carry managed service environment into the watchdog:** `docs/platforms/mac/bundled-gateway.md:133-137`
  - body: The LaunchAgent sample only forwards PATH, while managed Gateway services preserve profile, state-dir, and config-path env. For profile-specific installs, this can make the watchdog probe or restart the default Gateway instead of the intended service.
  - confidence: 0.84
