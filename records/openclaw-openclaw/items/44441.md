## Summary

Keep open: current main does not contain this runbook, and the proposed docs still publish an unreliable recovery recipe because the health gate is reachability-based and the LaunchAgent omits managed-service profile/config/state environment.

## What This Changes

This PR adds optional macOS LaunchAgent watchdog recovery documentation and links it from the updating guide.

## Best Possible Solution

Revise the runbook to match supported health and service-environment contracts, or defer it to the canonical rescue-watchdog work so published docs do not promise recovery the sample cannot perform.

## Reproduction Assessment

Yes, for the PR defects: static inspection of PR head plus current main shows the sample exits on `openclaw health --json` success while current `healthCommand` treats channel issues as nonfatal, and the sample plist only forwards PATH while managed services preserve profile/config/state variables.

## Solution Assessment

No, not as currently written; the PATH fix is useful, but the docs should either describe a reachability-only watchdog with the correct environment propagation or wait for the broader rescue-watchdog work.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Align the watchdog predicate with the documented failure mode:** `docs/platforms/mac/bundled-gateway.md:107-108`
  - body: The new section promises recovery for process-alive-but-unhealthy cases, but this gate exits as soon as `openclaw health --json` can reach the Gateway RPC. Current `healthCommand` reports channel issues without making them fatal, so a degraded but responsive gateway skips the documented remediation; please either frame this as reachability-only or use a predicate that fails for the intended unhealthy state.
  - confidence: 0.9
- **[P2] Propagate the managed service environment:** `docs/platforms/mac/bundled-gateway.md:133-137`
  - body: The LaunchAgent example only forwards PATH. Managed Gateway services also preserve `OPENCLAW_PROFILE`, `OPENCLAW_STATE_DIR`, and `OPENCLAW_CONFIG_PATH`, so profile-specific installs can be probed or restarted through the default config/state path unless the example carries those variables too.
  - confidence: 0.86
