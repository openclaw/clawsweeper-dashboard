## Summary

Keep open. Current main and v2026.4.29 still make omitted `killProcessTree` options default to Unix group-kill, and existing tests deliberately preserve that behavior even though the reporter and prior PR review identified it as unsafe for non-group-leader children.

## What This Changes

Guard Unix `killProcessTree` so process-group kill is used only when the target PID is actually safe to treat as a process-group leader.

## Best Possible Solution

Move the safety invariant into `src/process/kill-tree.ts`: use Unix process-group kill only when the target PID is verified or explicitly known to be its own process-group leader, and cover omitted-option callers with tests.

## Reproduction Assessment

Yes. A focused unit reproduction is current-main `killProcessTree(pid)` with omitted `detached`, which calls `process.kill(-pid, 'SIGTERM')`; the full macOS Telegram path is additionally supported by the reporter's live trace but was not replayed in this read-only Linux review.

## Solution Assessment

Yes. A kill-side guard is the narrowest maintainable fix because it protects existing and future callers that cannot reliably prove spawn detachment, while preserving group kill for verified detached process groups.

## Repair Work Prompt

Fix the macOS/POSIX process-group regression reported in https://github.com/openclaw/openclaw/issues/76259. Current `src/process/kill-tree.ts` treats omitted `opts.detached` as safe for Unix `process.kill(-pid, ...)`; make `killProcessTree` enforce the invariant on the kill side so group-kill is used only when the target PID is actually safe to treat as a process-group leader. Prefer a bounded Unix helper such as `ps -p <pid> -o pgid=` or an equivalent existing seam; if leadership cannot be verified, fall back to direct-pid SIGTERM/SIGKILL. Preserve Windows behavior and preserve explicit caller opt-out `{ detached: false }`. Add focused unit coverage in `src/process/kill-tree.test.ts` for omitted options on a non-group-leader, verified group leader, failed/timeout pgid lookup, and existing `{ detached: false }`. Update affected tests that currently assert group-kill by default. Do not mix in the separate `restart-stale-pids` work from #76254. Add a single-line user-facing `CHANGELOG.md` entry under Unreleased Fixes. Validate with targeted tests and changed gate.
