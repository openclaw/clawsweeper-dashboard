## Summary

Keep open. Current `main` and the shipped `v2026.4.29` tag still default omitted `killProcessTree` options to Unix negative-PID group-kill, current tests assert that default, and multiple non-supervisor callers still omit `detached`, so the reported regression remains valid and narrow.

## What This Changes

The issue requests that Unix `killProcessTree` protect omitted-option callers from unsafe negative-PID group kills on macOS/launchd.

## Best Possible Solution

Move the process-group safety invariant into `src/process/kill-tree.ts`, so negative-PID kills are used only for verified or explicitly known process-group leaders, with omitted-option callers covered by regression tests.

## Reproduction Assessment

Yes. A focused reproduction is `killProcessTree(pid)` or `killProcessTree(pid, { graceMs })` on Unix: current tests show it calls `process.kill(-pid, "SIGTERM")`, and the reporter supplied matching macOS launchd/Telegram traces.

## Solution Assessment

Yes. A kill-side guard is the narrowest maintainable fix because caller cooperation leaves existing and future omitted-option call sites unsafe while preserving `detached: false` as an explicit direct-kill path.
