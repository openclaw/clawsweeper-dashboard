## Summary

Keep open. Current main and the latest inspected release still default omitted `detached` to Unix process-group termination, and the unmerged follow-up PRs that targeted this issue were closed without landing.

## What This Changes

The issue asks to prevent `killProcessTree` from process-group killing non-detached Unix children when callers omit `detached`.

## Best Possible Solution

Make Unix group kill opt-in or PGID-verified in `killProcessTree`, carry actual spawn detachment facts into call sites, and lock omitted `detached` to direct-pid termination with regression tests.

## Reproduction Assessment

Yes. On current main, `killProcessTree(pid)` and `killProcessTree(pid, { graceMs })` route to `process.kill(-pid, "SIGTERM")` on Unix, and the current test suite asserts that default behavior.

## Solution Assessment

Yes. The right boundary is the kill-tree contract plus a caller audit; broad gateway, Telegram, or provider-specific handling would leave other omitted-option callers exposed.
