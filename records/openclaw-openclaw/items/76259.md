## Summary

Keep open: current main and the latest release still preserve Unix group-kill when `detached` is omitted, and several current callers still omit the actual detached fact at kill time. The related fix attempts are closed unmerged, so this issue remains the canonical remaining work.

## What This Changes

The issue reports that Unix process-tree cleanup can SIGTERM the macOS launchd-managed gateway when callers omit `{ detached: false }` and the child shares the gateway process group.

## Best Possible Solution

Land a narrow process-cleanup fix that carries the actual spawn-time detached/process-group fact into every relevant kill path, with tests proving attached children use direct-pid signaling and detached children keep group cleanup.

## Reproduction Assessment

Yes from source, but not from a live local run: current main still has detached-spawn callers that invoke process-tree kill helpers without passing the detached fact while the helper defaults omissions to Unix group-kill.

## Solution Assessment

Yes: carrying the actual spawn-time detached fact into cleanup is the narrowest maintainable fix because it avoids signaling the gateway group for attached children while preserving group cleanup for isolated children.
