## Summary

Keep open. Current main and the latest v2026.5.2 release still preserve the omitted-option Unix group-kill behavior from the #71662 follow-up, and open PR #76398 is already the closing implementation candidate.

## What This Changes

Request that Unix `killProcessTree` stop treating omitted `detached` as permission for process-group killing, preventing gateway SIGTERM under launchd/systemd.

## Best Possible Solution

Review and land or replace #76398 with a clean patch that defaults to direct-pid termination and uses explicit `detached: true` only for truly detached process groups.

## Reproduction Assessment

Yes. Source inspection on current main shows `killProcessTree(pid)` and `killProcessTree(pid, { graceMs })` still reach `process.kill(-pid, "SIGTERM")`, and the current test suite asserts that default behavior.

## Solution Assessment

Yes in direction. A safe kill-side default with explicit `detached: true` for group-kill callers is the narrow maintainable fix; the open PR branch should still be reviewed because it currently carries unrelated changes.
