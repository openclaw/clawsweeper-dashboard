## Summary

Keep open. Current main still defaults omitted `detached` to Unix process-group termination, the latest inspected release tag keeps the same behavior, and the follow-up fix PRs were closed without merge.

## What This Changes

The issue reports that `killProcessTree` still group-kills by default when callers omit `detached`, which can SIGTERM a macOS launchd-managed gateway.

## Best Possible Solution

Make Unix group-kill safe by default in `killProcessTree`, keep direct-pid termination for omitted or false `detached`, and update only verified detached callers to opt into group termination.

## Reproduction Assessment

Yes, source-reproducible. On current main, `killProcessTree(pid)` and `killProcessTree(pid, { graceMs })` still route to `process.kill(-pid, "SIGTERM")` on Unix, and the existing test suite asserts that behavior.

## Solution Assessment

Yes. The narrow maintainable direction is a kill-tree default/guard change plus a caller audit, not a broader gateway or Telegram change.
