## Summary

Keep open. Current main and the latest `v2026.5.2` release still preserve the omitted-option Unix group-kill behavior, and open PR #76398 explicitly references this issue as the fix candidate, so the issue should stay open until that PR lands or is rejected.

## What This Changes

The issue reports that the prior macOS process-tree fix only works when callers pass `{ detached: false }`, leaving omitted-option callers able to group-kill the gateway process group.

## Best Possible Solution

Review and land or reject PR #76398; the desired end state is direct-pid termination by default and process-group termination only for explicitly detached process groups.

## Reproduction Assessment

Yes. Current source and tests show that `killProcessTree(pid)` or `killProcessTree(pid, { graceMs })` on Unix still sends `SIGTERM` to `-pid`, matching the reporter's launchd gateway SIGTERM path.

## Solution Assessment

Yes. A kill-side safe default is the narrow maintainable fix because it protects current and future omitted-option callers while preserving explicit group-kill for detached children.
