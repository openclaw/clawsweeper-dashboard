## Summary

Keep open. Current main and the latest v2026.5.2 release still default omitted Unix `detached` options to process-group termination, while open PR #76398 is the active unmerged fix candidate for this issue.

## What This Changes

The issue asks for `killProcessTree` to stop defaulting to Unix process-group kills when callers omit `detached`, because non-detached launchd/systemd children can SIGTERM the gateway process group.

## Best Possible Solution

Review and land or reject PR #76398; the desired end state is direct-pid termination by default and process-group termination only for explicitly detached process groups.

## Reproduction Assessment

Yes. Source inspection on current main shows `killProcessTree(pid)` and `killProcessTree(pid, { graceMs })` still enable Unix group-kill, and current tests assert the negative-PID SIGTERM path for omitted `detached`.

## Solution Assessment

Yes. A kill-side safe default is the narrow maintainable fix because it protects existing and future omitted-option callers while preserving explicit group-kill for detached children.
