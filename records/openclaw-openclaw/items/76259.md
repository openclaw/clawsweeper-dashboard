## What This Changes

The issue reports that macOS managed gateways can receive SIGTERM because Unix `killProcessTree` still group-kills when callers omit `detached: false`.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/94697 with a process cleanup fix that preserves detached descendant cleanup while preventing attached gateway process-group signaling.

## Reproduction Assessment

Yes, source-reproducible: current main and v2026.6.10 still compute Unix group-kill from `opts?.detached !== false`, and current tests assert omitted `detached` uses group kill. I did not rerun the live macOS launchd gateway scenario.

## Solution Assessment
