## Best Possible Solution

Move the process-group safety invariant into `src/process/kill-tree.ts` so only verified or explicitly known process-group leaders use `process.kill(-pid, ...)`, with omitted-option callers covered by regression tests.

## Repair Work Prompt

Fix the macOS gateway SIGTERM regression reported in https://github.com/openclaw/openclaw/issues/76259. Current `src/process/kill-tree.ts` treats omitted `detached` as permission to call `process.kill(-pid, ...)`, which is unsafe when the target PID is not its own process-group leader. Move the safety invariant into the kill helper: use Unix group-kill only when the caller explicitly opts in with known detached state or when the helper verifies the target PID is its own process-group leader; otherwise fall back to direct PID SIGTERM/SIGKILL. Preserve Windows `taskkill` behavior and preserve process-tree cleanup for real detached process groups. Add/update regression tests for omitted `detached`, `detached:false`, verified group-leader behavior, and at least the current omitted caller mocks. Do not fold in the separate macOS ancestor-walk issue from #76254. Add a single-line user-facing `CHANGELOG.md` fix entry for the active release section, without bot attribution.
