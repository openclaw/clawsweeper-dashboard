## Summary

Keep open. Current `main` and the shipped `v2026.4.29` tag still default omitted `detached` options to Unix process-group kill, current tests assert that behavior, and live caller sites still omit the option, so the reported regression remains valid and narrow.

## What This Changes

The issue reports that Unix `killProcessTree` still defaults omitted `detached` options to process-group termination, which can SIGTERM a launchd-managed macOS gateway when the target pid is not a process-group leader.

## Best Possible Solution

Move the process-group safety invariant into `src/process/kill-tree.ts` so negative-PID group kills are used only for verified or explicitly known process-group leaders, with regression tests for omitted-option callers.

## Reproduction Assessment

Yes. A focused reproduction is `killProcessTree(pid)` or `killProcessTree(pid, { graceMs })` on Unix, which current tests show calls `process.kill(-pid, "SIGTERM")`; the reporter also supplied macOS launchd/Telegram trace evidence for the production path.

## Solution Assessment

Yes. A kill-side guard is the narrowest maintainable fix because it protects existing and future omitted-option callers while preserving group-kill for true detached process groups.

## Repair Work Prompt

Fix the macOS/Unix `killProcessTree` regression tracked by https://github.com/openclaw/openclaw/issues/76259. Current `src/process/kill-tree.ts` defaults omitted `detached` to process-group kill, so `killProcessTree(pid)` and `killProcessTree(pid, { graceMs })` can call `process.kill(-pid, ...)` for a non-group-leader child and SIGTERM the gateway process group. Move the safety invariant into `src/process/kill-tree.ts`: only use negative-PID group kill when the target is known or verified to be its own process-group leader, while preserving explicit `{ detached: false }` as a direct-PID path and preserving group cleanup for true detached children. Add/update focused tests in `src/process/kill-tree.test.ts` and caller tests that currently assert omitted calls (`mcp-stdio-transport`, LSP runtime, bash process/stop, PTY adapter, supervisor child). Add a single-line user-facing `CHANGELOG.md` fix entry. Do not change `src/infra/restart-stale-pids.ts` or attempt to solve #76254 in this PR.
