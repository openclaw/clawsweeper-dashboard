## Summary

Keep open. Current main still detach-spawns and exits the tracked gateway process for non-container Linux/macOS restarts with no recognized supervisor markers, so the reported custom-supervisor orphan/EADDRINUSE failure remains valid despite Windows and container mitigations.

## What This Changes

The request is to make SIGUSR1/config.patch gateway restarts safe under custom supervisors and to clarify the OPENCLAW_NO_RESPAWN guidance.

## Best Possible Solution

Make generic non-update gateway restarts use in-process restart when no known supervisor is detected, keep update-specific fresh-process replacement intact, and update doctor/docs/tests/changelog accordingly.

## Reproduction Assessment

Yes. On current main, unset OPENCLAW_NO_RESPAWN, run on non-container Linux/macOS without launchd/systemd/schtasks markers, and trigger a non-update restart via SIGUSR1 or restart-needed config.patch; the source path returns spawned and exits the original PID.

## Solution Assessment

No. Current main only disables generic detached respawn for unmanaged Windows and detected containers; the narrower maintainable fix is to change the generic non-update no-supervisor path while leaving update and known-supervisor behavior explicit.
