## Summary

Keep open: current main still lacks the requested per-agent description metadata, and open PR #75225 already tracks the implementation with closing syntax, so the issue should remain open until that PR is reviewed, merged, or explicitly rejected.

## What This Changes

Add an optional per-agent `description` config field and include it in `agents_list` so orchestrators can discover sub-agent capabilities.

## Best Possible Solution

Review and land or close PR #75225; if accepted, the field should flow through config types, strict schema, generated metadata, `agents_list`, docs, changelog, and regression tests.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives a high-confidence verification path because current main lacks `agents.list[].description` in both the config contract and `agents_list` response.

## Solution Assessment

Yes, if maintainers accept the public API addition. Explicit metadata in `agents.list[]` surfaced through `agents_list` is narrower and more maintainable than requiring orchestrators to scrape agent files or hardcode catalogs.
