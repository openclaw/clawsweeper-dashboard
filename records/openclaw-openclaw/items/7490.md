## Summary

Keep open: current main still lacks optional per-agent description metadata in the agent config contract and `agents_list`, and open PR #75225 already references this issue with closing syntax, so this should wait for PR review/merge or explicit maintainer rejection.

## What This Changes

Add an optional `agents.list[].description` field and include it in `agents_list` so orchestrators can discover agent specializations.

## Best Possible Solution

Review and land or close PR #75225; if accepted, the field should flow through config types, strict schema, generated metadata, `agents_list`, docs, changelog, and regression tests.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives a high-confidence verification path because current main lacks `agents.list[].description` in the config contract and `agents_list` response.

## Solution Assessment

Yes, if maintainers accept the public API addition. Explicit metadata in `agents.list[]` and `agents_list` is a narrower maintainable path than requiring orchestrators to scrape agent files or hardcode catalogs.
