## Summary

Keep this issue open: current main and the latest release accept `agents.list[].description` in config, but `agents_list` still does not return descriptions, and the live closing pull request is open rather than merged.

## What This Changes

The issue requests an optional per-agent `description` config field surfaced through `agents_list` so orchestrator agents can discover sub-agent capabilities dynamically.

## Best Possible Solution

Land or explicitly reject the linked implementation path after maintainers decide the model-visible metadata boundary, keeping config schema, docs, prompt snapshots, and focused tests aligned.

## Reproduction Assessment

Yes. Current-main source shows `agents.list[].description` is accepted by config types/schema, but the allowlist-aware `agents_list` mapper never reads or returns it.

## Solution Assessment

Yes in direction, but not complete yet. Extending the existing allowlist-aware `agents_list` tool is narrower than prompt injection or static catalogs; the remaining decision is whether and how to expose free-form descriptions.
