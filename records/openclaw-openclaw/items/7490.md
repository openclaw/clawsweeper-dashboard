## Summary

Keep open: current main still lacks the requested per-agent description metadata and `agents_list` output, and the open linked implementation PR #75225 uses closing syntax for this issue, so the issue should stay open until that PR is reviewed and merged or closed.

## What This Changes

Request adding optional `agents.list[].description` metadata and returning it from `agents_list` so orchestrator agents can choose sub-agents dynamically.

## Best Possible Solution

Review and land or close PR #75225; once a complete implementation merges, this issue can close as implemented.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection on current main gives a high-confidence verification path: the config type/schema/generated metadata and `agents_list` output do not include `agents.list[].description`.

## Solution Assessment

Yes, if maintainers accept the public API addition. The narrow maintainable solution is explicit `agents.list[]` metadata flowing through the strict schema, generated docs, resolver/list path, and `agents_list`, rather than asking orchestrators to scrape workspace files.
