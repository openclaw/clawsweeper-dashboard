## Summary

Keep open: current main still does not provide per-agent description metadata in config or `agents_list`, and the linked open implementation PR should be reviewed before this issue closes.

## What This Changes

The issue requests an optional per-agent `description` config field surfaced by `agents_list` so orchestrators can discover sub-agent capabilities at runtime.

## Best Possible Solution

Review and land or close PR #75225; if accepted, the field should flow through the config type, strict schema, generated metadata, `agents_list`, docs, changelog, and regression tests.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives a high-confidence verification path because current main lacks `agents.list[].description` in the config contract and `agents_list` response.

## Solution Assessment

Yes, if maintainers accept the public API addition. Explicit metadata in `agents.list[]` and `agents_list` is a narrower maintainable path than requiring orchestrators to scrape agent files or hardcode catalogs.
