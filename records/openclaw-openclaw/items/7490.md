## What This Changes

The issue requests an optional `agents.list[].description` config field and inclusion of that metadata in `agents_list` for dynamic sub-agent discovery.

## Best Possible Solution

Review, rebase, and either land or reject https://github.com/openclaw/openclaw/pull/75225, keeping config types, strict schema, docs, `agents_list` output, snapshots, and focused tests aligned if the contract is accepted.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an additive config and tool-output feature. Source inspection gives a high-confidence current-main state check: config accepts `agents.list[].description`, but `agents_list` omits it.

## Solution Assessment

Yes, if maintainers accept the new public contract: optional per-agent metadata surfaced through `agents_list` is the narrowest maintainable shape for the requested discovery behavior. The linked implementation PR is the right place to finish or reject that work.
