## Summary

Keep open: current main and the latest release still have the source-level Feishu SecretRef activation bug, and two open closing PRs are already candidate fixes. This issue should remain open until maintainers land one Feishu-scoped fix or close the candidate PRs.

## What This Changes

Resolve top-level Feishu `appSecret` SecretRefs for the implicit default account even when all named accounts override `appSecret`.

## Best Possible Solution

Land one Feishu-scoped fix that mirrors Feishu implicit-default account resolution for top-level `appSecret`, then let the merged closing PR close this issue.

## Reproduction Assessment

Yes, at source level: current main creates an implicit Feishu default account from top-level credentials, but the shared secret collector can mark that same top-level `appSecret` inactive when every named account overrides it. I did not run a live gateway startup in this read-only review.

## Solution Assessment

Yes: the best fix is a narrow Feishu-owned activation rule that recognizes the implicit default account while preserving inactive-surface filtering for sibling channels. The broad shared-helper approach was already closed as superseded by the Feishu-scoped PR path.
