## Summary

Keep open: current main and the latest release still have the source-backed Feishu SecretRef activation bug where the implicit top-level default account can start, but the shared secret collector marks its top-level appSecret inactive when every named account overrides appSecret.

## What This Changes

The issue reports that a top-level Feishu channel appSecret SecretRef remains unresolved when named Feishu accounts define their own appSecret values.

## Best Possible Solution

Align Feishu SecretRef activation with Feishu account listing so the implicit top-level default account keeps channels.feishu.appSecret active, without adding config or weakening inactive-surface filtering for other channels.

## Reproduction Assessment

Yes, at source level: current Feishu account listing preserves an implicit default account while the shared secret collector can mark the same top-level appSecret inactive when every explicit account overrides appSecret. I did not run a live gateway startup in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is to make Feishu top-level appSecret activation recognize the implicit default account, while preserving strict runtime SecretRef handling and inactive-surface filtering for other channels.
