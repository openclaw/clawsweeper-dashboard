## Summary

Keep open: current main and v2026.6.10 still have the source-backed Feishu SecretRef activation bug, and two open closing PRs are candidate fixes that should be reviewed rather than closing this issue early.

## What This Changes

Top-level Feishu appSecret SecretRefs should stay active for the implicit default account when named Feishu accounts override appSecret.

## Best Possible Solution

Land one Feishu-scoped fix that keeps the implicit default account’s top-level appSecret active while preserving inactive-surface filtering for explicit-default overrides and sibling channels.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes, at source level: current main combines Feishu implicit default account creation with shared top-level appSecret inactive filtering when every named account overrides appSecret.

## Solution Assessment

Is this the best way to solve the issue? Yes: the narrow maintainable path is Feishu-scoped active-surface logic, not changing the shared simple collector for every account-scoped channel.
