## Summary

Keep open: the patch has a blocking lifecycle-state bug because an expired stale running claim is recorded as successful `done` work with passed proof from title text, which can unblock dependent cards without worker evidence. The PR also needs real after-fix behavior proof before merge.

## What This Changes

The PR changes Workboard dispatch so expired stale running claims are completed as `done` with a title-derived verdict/proof, while max-runtime-only running cards remain blocked, and updates the store test expectations.

## Best Possible Solution

Keep stale running recovery from treating absence of heartbeat as success: block or reclaim stale claims unless a real worker completion/proof path exists, or add an explicit maintainer-approved reaped lifecycle contract with docs and tests.
