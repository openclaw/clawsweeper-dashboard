Keep open: the PR is not merge-ready because it turns expired heartbeat-stale Workboard claims into successful `done` cards with passed proof, which conflicts with the current documented block/reclaim lifecycle and can incorrectly unblock dependent work.

## What This Changes

The branch changes Workboard dispatch so expired stale running claims are completed with title-derived `stale_claim_reaped` proof, while updating the focused store test expectations.

## Best Possible Solution

Keep stale claim recovery non-successful by blocking or reclaiming abandoned claims unless there is real worker completion evidence; only add a reaped-success lifecycle if maintainers explicitly approve and document that contract.

## Solution Assessment

No; deriving a successful completion verdict from card title text is not the best fix for a missing worker heartbeat. The safer maintainable path is blocked/reclaimed stale recovery unless a real worker completion signal or explicit maintainer-approved lifecycle contract exists.
