## Summary

Keep open: current main and v2026.6.6 still build and dispatch subagent completion announces without a parent-last-assistant fingerprint short-circuit. The related open PR https://github.com/openclaw/openclaw/pull/81184 covers repeated delivery-boundary dispatches, but it does not yet land this transcript-state guard, and https://github.com/openclaw/openclaw/issues/90944 shows adjacent current-main completion delivery races remain active.

## What This Changes

The issue requests a runtime-side subagent announce dedup guard that compares the parent session's most recent assistant message with the child findings and skips duplicate completion announces.

## Best Possible Solution

Make the parent/requester completion handoff authoritative by adding a conservative transcript-state duplicate guard before announce delivery, while coordinating it with the direct-text fallback and the open completion-delivery cluster.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection does show current main still has no requester-last-assistant fingerprint short-circuit before announce delivery, and related reports provide detailed production-shaped evidence for the same completion-delivery family.

## Solution Assessment

Unclear as proposed: a runtime transcript-state guard is likely the right boundary, but the exact fingerprint rule should be designed with the parent/requester handoff and `:text-direct` fallback so it does not suppress valid completion updates.
