## Summary

Keep open: current main still lacks the requested runtime-side parent latest-assistant transcript fingerprint check, and the related open PRs cover adjacent prompt-token and same-idempotency-key dispatch dedupe rather than parent-already-delivered detection.

## What This Changes

The issue requests a runtime-side parent transcript fingerprint check that skips subagent completion announcements when the parent already delivered the child output.

## Best Possible Solution

Add a conservative, fail-open parent latest-assistant transcript check before building or delivering the announce, with regression coverage for duplicate, unrelated, stale, and transcript-read-failure cases.

## Reproduction Assessment

Yes, source-level. Configure runSubagentAnnounceFlow with child findings X and a requester session whose latest assistant reply after the child ended already contains X; current main still builds and queues the announce without checking requester history.

## Solution Assessment

Yes. A pre-delivery, fail-open parent transcript fingerprint check is the narrow structural fix; prompt guards and same-key dispatch dedupe reduce adjacent failure modes but do not observe whether the parent already delivered the result.
