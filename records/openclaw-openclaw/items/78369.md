## Summary

Keep open: current main still builds and delivers the subagent announce without a runtime-side parent transcript fingerprint check, and the related open PRs only cover adjacent prompt parity or same-idempotency-key delivery dedupe.

## What This Changes

Add a runtime-side dedupe check that compares a child completion's findings with the parent session's latest assistant message and skips duplicate subagent announces.

## Best Possible Solution

Add a conservative, fail-open parent-assistant transcript check before announce delivery, with regression coverage for duplicate, unrelated, stale, and transcript-read-failure cases.

## Reproduction Assessment

Yes, source-level: configure runSubagentAnnounceFlow with child result X and a requester session whose latest assistant reply after endedAt contains X; current main still proceeds to build and deliver the announce without checking parent history.

## Solution Assessment

Yes. A pre-delivery, fail-open parent transcript fingerprint check is the narrow structural fix; prompt guards and same-key dispatch dedupe reduce adjacent failure modes but do not observe whether the parent already delivered the result.

In `src/agents/subagent-announce.ts`, after the child findings are known and before `buildAnnounceSteerMessage` / `deliverSubagentAnnouncement`, inspect the requester/parent session's latest assistant message, confirm it was produced after the child ended when timestamp data is available, and compare it against the child findings with a conservative normalized fingerprint/window match. If it matches, return true so cleanup can proceed and no completion-delivery prompt is injected. Fail open on unreadable history, missing timestamps, missing text, or non-matching content.
