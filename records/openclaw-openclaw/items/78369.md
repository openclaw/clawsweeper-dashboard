## Summary

Keep open: current main still builds and delivers the subagent completion announce without inspecting the parent requester transcript, and the adjacent open PRs cover prompt parity or same-idempotency-key delivery dedupe rather than the requested parent-message fingerprint short-circuit.

## What This Changes

The issue requests a runtime-side subagent announce dedupe guard that compares the requester parent's latest assistant message with the child findings before sending a completion announce.

## Best Possible Solution

Add a conservative, fail-open parent-assistant transcript check before announce delivery, with regression coverage for duplicate, unrelated, stale, and transcript-read-failure cases.

Yes, source-level: configure runSubagentAnnounceFlow with child result X, params.endedAt, and a requester history whose latest assistant reply after endedAt contains X. Current main still proceeds to buildAnnounceSteerMessage and deliverSubagentAnnouncement because it never reads the parent requester history.

## Solution Assessment

Yes. A pre-delivery, fail-open parent transcript fingerprint check is the narrow structural fix; prompt guards and same-key dispatch dedupe reduce adjacent failure modes but do not observe whether the parent already delivered the same content.
