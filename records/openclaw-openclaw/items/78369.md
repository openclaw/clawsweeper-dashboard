## Summary

Keep open: current main and the latest release still lack the requested parent-last-assistant fingerprint short-circuit. The existing duplicate credit only recognizes exact `delivery-mirror` records, and the related open PR is conflicting and covers a different process-local dispatch dedupe layer rather than superseding this transcript-state guard.

## What This Changes

The issue requests a runtime-side deduplication guard that skips subagent completion announce delivery when the parent’s latest assistant message already covered the child findings.

## Best Possible Solution

Add a conservative runtime-owned duplicate check at the announce/delivery boundary that suppresses only proven parent-delivered duplicates, fails open, and is tested against direct-text fallback and unrelated parent replies.

## Reproduction Assessment

No high-confidence live current-main reproduction was run. Source inspection does show the requested parent-last-assistant fingerprint guard is absent, and related reports provide production-shaped evidence for the same completion-delivery family.

## Solution Assessment

Unclear as proposed: a runtime transcript-state guard is likely the right boundary, but the exact fingerprint rule and its interaction with direct-text fallback need maintainer judgment to avoid suppressing legitimate updates.
