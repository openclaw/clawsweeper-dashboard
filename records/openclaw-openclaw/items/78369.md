## Summary

Keep open: current main and v2026.6.10 still build and deliver the subagent completion announce without a requester latest-assistant fingerprint short-circuit; the related open PRs and issues cover adjacent retry, text-direct, or broader delivery-contract paths rather than superseding this transcript-state guard.

## What This Changes

The issue asks the runtime to compare a subagent completion result with the requester session's latest assistant reply and skip duplicate completion announces before prompting or direct delivery.

## Best Possible Solution

Adopt a maintainer-approved conservative duplicate guard at the announce/delivery boundary, with failure-open behavior and tests for parent-already-delivered, unrelated parent reply, text-direct fallback, and yielded-parent cases.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection shows the requested parent-last-assistant fingerprint guard is absent, and related reports provide production-shaped evidence for the same subagent completion delivery family.

## Solution Assessment

Unclear as proposed. The runtime announce boundary is likely the right layer, but the exact fingerprint rule needs maintainer judgment to avoid suppressing legitimate completion updates.
