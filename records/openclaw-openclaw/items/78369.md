## Summary

Keep open: current main still relies on prompt-level `NO_REPLY` suppression and child-output checks, and I did not find a parent-session latest-assistant fingerprint short-circuit before announce delivery.

## What This Changes

The issue requests runtime-side duplicate suppression for subagent announcements by matching child findings against the parent session's latest assistant message before injecting the announce prompt.

## Best Possible Solution

Add a conservative, fail-open parent-assistant transcript fingerprint check before announce delivery, with regression coverage for duplicate and unrelated parent replies.

## Reproduction Assessment

Yes, source-level: create a harness where the child result is `X`, the parent session already has a latest assistant reply containing `X` after the child `endedAt`, then call `runSubagentAnnounceFlow`; current main has no parent-history check and proceeds to announce delivery.

## Solution Assessment

Yes, the requested structural check is the narrowest maintainable direction because current main already documents/prompt-enforces no duplicate delivery but leaves the decision to the model.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/78369. Implement conservative runtime-side duplicate suppression in `runSubagentAnnounceFlow`: after `findings` is known and the final target requester session key is resolved, but before `buildAnnounceSteerMessage` / `deliverSubagentAnnouncement`, read the target requester session's latest assistant message from transcript or `chat.history`. If the latest assistant message timestamp is at or after the child `endedAt` and a conservative normalized fingerprint/window match shows the parent already delivered the child findings, return `true` so cleanup can proceed silently. Fail open if parent history cannot be read or timestamps/text are missing. Preserve existing `NO_REPLY`, `ANNOUNCE_SKIP`, idempotency, direct fallback, and nested-subagent delivery behavior; do not add a config option. Add regression coverage for: duplicate parent assistant text suppresses announce, unrelated parent assistant text still announces, and missing/unreadable parent history still announces.
