## Summary

Keep open: current main and v2026.6.8 still lack the requested parent-last-assistant fingerprint short-circuit, and the related open fixes cover adjacent dispatch or text-direct fallback layers rather than superseding this transcript-state guard.

## What This Changes

The issue requests runtime-side subagent announce dedupe that compares the requester session's latest assistant message against child completion findings and skips duplicate announces.

## Best Possible Solution

Add a conservative runtime-owned duplicate check at the announce/delivery boundary that suppresses only proven parent-delivered duplicates, fails open, and is tested against text-direct fallback, unrelated parent replies, and yielded-parent cases.

## Reproduction Assessment

No high-confidence live current-main reproduction was run. Source inspection shows the requested parent-last-assistant fingerprint guard is absent, and related issues provide production-shaped evidence for the same subagent completion delivery family.

## Solution Assessment

Unclear as proposed: the runtime announce boundary is likely the right layer, but the exact fingerprint rule needs maintainer judgment to avoid suppressing valid follow-up updates.
