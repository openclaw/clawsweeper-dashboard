## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/78369. In `runSubagentAnnounceFlow`, before building the announce steer message or calling `deliverSubagentAnnouncement`, add a conservative fail-open check against the requester session's latest assistant reply. Reuse or extend the existing `chat.history`/`readLatestAssistantReplySnapshot` path so the helper can return assistant text and timestamp. If `params.endedAt` is present, the latest parent assistant message timestamp is at or after it, and a normalized conservative fingerprint match holds between that assistant text and `findings` (for example a 60+ character contiguous window in either direction), return `true` so cleanup can continue without injecting another announce. Fail open on missing history, missing timestamp, read errors, short texts, or unrelated content. Add regression coverage for duplicate parent text suppressing delivery and unrelated parent text still announcing. Do not add a config flag or broaden subagent delivery policy. Add a one-line changelog fix entry because this affects user-visible duplicate messages.
