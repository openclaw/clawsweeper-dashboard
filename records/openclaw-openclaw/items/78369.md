## Summary

Keep open: current main still has no parent-session latest-assistant fingerprint short-circuit before subagent announce delivery. The linked open PRs cover adjacent prompt-parity and same-idempotency-key dispatch dedupe paths, but neither implements the requested parent-transcript duplicate check.

## What This Changes

The issue requests a runtime-side duplicate suppression check for subagent announces when the parent has already posted matching child output.

## Best Possible Solution

Add a conservative, fail-open parent-assistant transcript fingerprint check before announce delivery, with regression coverage for duplicate, unrelated, stale, and transcript-read-failure cases.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/78369. In `runSubagentAnnounceFlow`, add a conservative parent-transcript duplicate short-circuit after `targetRequesterSessionKey` and `findings` are resolved but before `buildAnnounceSteerMessage` / `deliverSubagentAnnouncement`. Read the latest assistant message from the resolved requester session; only return true without announcing when `params.endedAt` is present, the assistant message timestamp is at or after `endedAt`, and normalized text has a strong 60+ character contiguous-window match with `findings`. Fail open on missing timestamps, short text, unrelated text, transcript read errors, or unsupported message shapes. Do not add config, do not implement the same-key delivery cache from https://github.com/openclaw/openclaw/pull/81184, and preserve existing child output selection semantics. Add focused regression coverage for duplicate parent reply suppresses delivery, unrelated parent reply still delivers, stale parent reply before `endedAt` still delivers, and transcript read failure still delivers. Account for the adjacent prompt-parity PR at https://github.com/openclaw/openclaw/pull/78425 when editing tests or prompt assertions.
