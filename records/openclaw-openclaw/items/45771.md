## Summary

Keep open. Current main has provider usage snapshots and reactive cooldown/fallback handling, but it still lacks the proposed pace projection, normal Anthropic response-header capture, persistent pace state, prompt/session_status pace guidance, and autonomous/subagent throttling policy.

## What This Changes

The issue requests built-in Anthropic pace tracking, status/session exposure, persistent pace state, and autonomous throttling policy for long-running agents.

## Best Possible Solution

Define an opt-in provider/agent pace contract that extends usage snapshots with freshness and projection data, exposes it in status/session_status and prompt context, and only then adds carefully scoped throttle behavior.
