## Summary

Keep open: current main has provider usage-window visibility plus reactive auth-profile cooldowns and static subagent fan-out guards, but it does not implement pace projection, Anthropic response-header capture, `agents.defaults.pace`, agent prompt/session policy, notifications, or autonomous throttling. The related open trackers overlap with rate limiting, usage-aware routing, and failover, but none supersedes this pace-aware autonomous-agent design request.

## What This Changes

Add pace-aware rate-limit tracking for autonomous agents, including Anthropic usage capture, projected 5h/7d pace tiers, status/session visibility, and policy-based throttling.

## Best Possible Solution

Keep this as a maintainer design tracker; a good first slice is provider-owned structured usage source/freshness plus status/session visibility, with enforcement split out after calibration and UX decisions.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source/docs inspection on current main, which shows quota-window reporting and static/reactive guards but not the requested pace policy surfaces.

## Solution Assessment

Unclear for the full proposal. Usage snapshot/header capture plus status/session visibility is likely the narrow maintainable first step, while automatic throttling needs separate design for provider variance and safe autonomous-agent behavior.

## Risks / Open Questions
