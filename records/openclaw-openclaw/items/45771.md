## Summary

Keep open because current main has provider usage-window visibility and static subagent fan-out guards, but not pace-aware autonomous-agent control. Related rate-limit, usage-aware routing, and failover trackers overlap, but they do not supersede the requested 5h/7d projection plus agent policy work.

## What This Changes

Request built-in pace-aware rate-limit visibility and policy for autonomous agents, including Anthropic 5h/7d usage capture, projected pace tiers, config, status/session context, and subagent throttling.

## Best Possible Solution

Keep this as a maintainer design tracker; a good first slice would add provider-owned structured usage source/freshness plus status/session visibility, then defer enforcement until per-profile persistence, cost calibration, and autonomous-agent UX are settled.
