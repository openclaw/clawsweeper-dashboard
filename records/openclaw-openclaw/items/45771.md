## Summary

Keep open. Current main and the latest release provide Anthropic/provider usage snapshots and status visibility, but the requested pace projection, response-header capture, persistent pace state, prompt context injection, and autonomous/subagent throttle policy are still missing; the discussion also raises real product concerns about utilization-only throttling.

## What This Changes

Add built-in pace-aware Anthropic rate-limit tracking and agent throttling for autonomous OpenClaw sessions.

## Best Possible Solution

Define a core provider/agent pace contract that extends usage snapshots with freshness and projection metadata, exposes that consistently in status/session_status, and only then adds clearly scoped opt-in throttle behavior.

## Reproduction Assessment

Not applicable; this is a feature proposal rather than broken existing behavior. Source inspection gives high confidence that current main lacks the requested header capture, pace tiers, persistent state, and autonomous throttle policy.

## Solution Assessment

Unclear as a single implementation patch. The direction likely belongs in core/provider policy, but maintainers should first settle freshness, projection, prompt context, and throttle semantics.
