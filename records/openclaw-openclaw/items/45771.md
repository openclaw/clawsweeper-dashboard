## Summary

Keep open: current main and the latest release expose provider usage windows in status/session surfaces, but they do not implement pace tiers, projected quota-window policy, persisted pace state, prompt/context injection, or autonomous/subagent throttling; the live labels and discussion point to a maintainer product/API decision rather than conservative cleanup.

## What This Changes

Request built-in provider pace awareness for autonomous agents: capture usage signals, calculate quota-window tiers, show them in status/session_status, and optionally throttle subagents or non-interactive work.

## Best Possible Solution

Design a narrow provider-usage/pace contract first, expose read-only status/session context, and only add opt-in enforcement after maintainers approve provider calibration, persistence, and upgrade-safe config semantics.
