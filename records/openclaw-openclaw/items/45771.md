## Summary

Keep open. Current main and the latest release provide Anthropic/provider usage snapshots and status visibility, but they still do not implement pace projection, response-header capture, persistent pace state, agent prompt context, or autonomous/subagent throttle policy; related trackers are adjacent rather than a complete replacement.

## What This Changes

The issue requests built-in pace-aware Anthropic usage tracking, 5h/7d projection tiers, status/session visibility, and autonomous throttling for agents.

## Best Possible Solution

Define a core provider/agent pace contract that extends usage snapshots with freshness and projection metadata, exposes it consistently in status/session_status, and only then adds clearly scoped opt-in throttle behavior.

## Reproduction Assessment

Not applicable; this is a feature proposal rather than broken existing behavior. Source inspection gives high confidence that current main lacks the requested header capture, pace tiers, persistent state, and throttle policy.

## Solution Assessment

Unclear as a single implementation patch. Usage visibility is already partly solved, but maintainers still need to settle freshness, projection, prompt exposure, and autonomous throttle semantics before this becomes a safe focused change.
