## Summary

## Best Possible Solution

Define a provider-aware pace snapshot contract with source, freshness, reset, and observed-cost inputs, surface it read-only in status and session_status first, then add conservative opt-in throttling after the default policy is agreed.

## Risks / Open Questions

- Provider-specific utilization headers may be absent or transformed by OpenRouter, proxies, and non-Anthropic providers, so a core contract needs source and freshness semantics rather than assuming one header format.
- Automatic throttling can interrupt autonomous work or subagent spawning in ways that affect session continuity, so defaults and opt-in behavior need maintainer product decisions.
- Pace projection based only on utilization percentage may be misleading because request cost varies by model, cache behavior, and context size.
