## Summary

Keep open because current main has provider usage-window visibility and static subagent fan-out guards, but not built-in pace projection, persisted pace state, prompt/status/session integration, or autonomous throttling policy. The discussion adds useful constraints around provider-aware degradation and cost-per-request calibration, so this should stay as a maintainer design tracker rather than be swept closed.

## What This Changes

The issue requests built-in Anthropic usage capture, pace projection, autonomous-agent throttling policy, and status/session visibility for rate-limit-aware agent loops.

## Best Possible Solution

Keep this as a maintainer design tracker; a good first slice would add provider-owned usage freshness/pace snapshots plus status/session visibility, then defer enforcement until calibration, persistence, and resume-safe autonomous-agent UX are settled.
