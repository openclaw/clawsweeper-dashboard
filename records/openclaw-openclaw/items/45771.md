## Summary

Keep open. Current main and v2026.6.10 provide provider usage visibility in status, session_status, and usage.status, but they do not implement pace projection, persisted pace state, tiered throttling, or the proposed agents.defaults.pace config.

## What This Changes

Add built-in pace-aware Anthropic/provider quota tracking that projects 5h and weekly usage, exposes pace in status/session_status, and optionally throttles autonomous or subagent work through new agent config.

## Best Possible Solution

Build on the existing provider-usage/status primitives, then have maintainers scope the narrow core policy for pace projection, persistence, prompt context, and any opt-in throttling before implementation.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature proposal, and source inspection gives high-confidence evidence that visibility exists while pace projection/config/throttling do not.

## Solution Assessment

Unclear until maintainers scope it; existing ProviderUsageSnapshot and status/session_status are the right base, but automatic throttling needs an owner-approved UX, admission policy, and model/context-cost calibration.
