## Risks / Open Questions

- Provider headers and usage endpoints differ by provider/auth path, so the pace contract needs explicit source and freshness semantics before core can enforce it safely.
- Utilization percentage alone may throttle too early or too late unless maintainers decide how to account for observed request cost, model choice, cache behavior, and context size.
- Automatic blocking of subagents or non-interactive work can affect long-running session continuity, so fail-open/fail-closed defaults and user-visible interruption behavior need product scoping.
