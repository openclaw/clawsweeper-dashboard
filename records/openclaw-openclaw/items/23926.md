## Summary

Keep open. Current main still has only partial install policy coverage: gateway-managed skill dependency installs can run the blocking `before_install` hook, but native ClawHub skill install/update, CLI, and UI paths still copy downloaded skill folders without that policy point and there is no post-install observer payload.

## What This Changes

Request adding blocking pre-install and observational post-install lifecycle hooks for skill installation, especially ClawHub skill installs and sync/update paths.

## Best Possible Solution

Add a documented, backward-compatible lifecycle API that covers native ClawHub install/update before file copy, reuses it from CLI/gateway/UI paths, and emits a post-install observer with stable source, target, timestamp, result, and audit metadata.

## Reproduction Assessment

Yes, from source inspection. The reproducible path is `openclaw skills install` or UI/gateway `skills.install` with `source: "clawhub"`, which reaches `installSkillFromClawHub` and copies via `installPackageDir` without `before_install` or a post-install observer.

## Solution Assessment

Unclear. The requested lifecycle hook direction is valid, but the best maintainable fix needs maintainer agreement on whether to extend `before_install`, add new skill-specific hook names, and standardize the post-install audit payload.
