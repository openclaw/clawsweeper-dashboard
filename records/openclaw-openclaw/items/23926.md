## Summary

Keep open. Current main still only partially covers the requested lifecycle policy: gateway-backed skill dependency installs can use the blocking `before_install` hook, but native ClawHub install/update and the UI ClawHub path still bypass that scan/policy hook, and there is still no post-install audit hook.

## What This Changes

The issue requests blocking pre-install and observational post-install hook events for skill installation.

## Best Possible Solution

Define a documented, backward-compatible skill install lifecycle API that covers native ClawHub install/update, gateway/UI ClawHub installs, and a stable post-install observer with source, target, timestamp, result, and audit metadata.

## Reproduction Assessment

Yes for the source-verifiable gap, not as a runtime bug: native ClawHub install/update reaches `installExtractedSkillRoot` with `scan: false`, while the hook surface has only `before_install` and lacks post-install observer metadata.

## Solution Assessment

Unclear. The lifecycle-hook direction is valid, but maintainers need to choose whether to extend `before_install` or add skill-specific events, plus blocking semantics, audit payload shape, UI behavior, and ClawHub scope.
