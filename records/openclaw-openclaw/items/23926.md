## Summary

Keep open. Current main still only partially covers the requested lifecycle policy: gateway-managed skill dependency installers run the generic blocking `before_install` hook, but native ClawHub skill install/update and the UI ClawHub install path still copy downloaded skill folders without a pre-copy policy hook, and there is no post-install observational hook carrying audit/outcome metadata. The recent public Codex review comment reaches the same conclusion, and the code still matches that assessment at `542821c`.

## What This Changes

The issue requests blocking pre-install and observational post-install hook events for skill installation across ClawHub install/sync, the Skills UI, plugin-shipped skills, and gateway install paths.

## Best Possible Solution

Keep this issue open as the concrete tracker for skill install lifecycle policy. The best path is to extend the existing `before_install` contract or add documented skill lifecycle hooks so native ClawHub install/update, gateway `skills.install`, and Skills UI ClawHub installs run a blocking policy point before copying files, then add a post-install observational hook with `installedAt`, `targetDir`, source, outcome, and audit metadata.
