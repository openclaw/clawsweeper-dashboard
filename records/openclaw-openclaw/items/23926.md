## Summary

Keep open. Current main partially addresses the policy gap with the generic blocking `before_install` hook for gateway-managed skill dependency installs, but the requested lifecycle coverage is still missing for native ClawHub skill install/update and the UI ClawHub install path, and there is still no post-install observational hook carrying audit/outcome metadata. A maintainer comment already identified this issue as the concrete tracker for that remaining work.

## What This Changes

Request to add blocking pre-install and observational post-install skill lifecycle hook events so plugins can scan, warn, block, and audit skill installs before they land on disk.

## Best Possible Solution

Add a documented, backward-compatible plugin lifecycle policy point that runs before native ClawHub skill install/update copies files into the workspace, reuse it from the gateway and UI ClawHub paths, and add a post-install observer with stable source, target, timestamp, result, and audit metadata. Treat standalone `clawhub` CLI support as a coordinated ClawHub follow-up unless that code lives in this repository.

## Reproduction Assessment

Yes. The observable gap is reproducible by tracing current main: gateway-managed skill dependency installs call `scanSkillInstallSource`, but `openclaw skills install`, gateway/UI ClawHub installs, and ClawHub updates route through `installSkillFromClawHub` and copy with `installPackageDir` without a pre-copy hook or any post-install observer.

## Solution Assessment

Unclear. The requested security lifecycle hook is a valid direction, but the best maintainable implementation needs maintainer agreement on whether to extend the existing `before_install` contract or introduce dedicated skill lifecycle hook names, plus the post-install audit payload shape.
