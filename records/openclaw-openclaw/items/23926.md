## Summary

Keep open. Current main still has only partial skill-install policy coverage: gateway-managed skill dependency installs can run the blocking `before_install` hook, but native ClawHub install/update and UI/gateway ClawHub installs still copy skill folders without that policy point, and there is no post-install observer payload.

## What This Changes

Add blocking `skill:pre-install` and observational `skill:post-install` hook events for skill installation flows such as ClawHub sync/install, UI installs, and plugin-shipped skills.

## Best Possible Solution

Add a documented, backward-compatible lifecycle API that covers native ClawHub install/update before file copy, reuses it from CLI/gateway/UI paths, and emits a post-install observer with stable source, target, timestamp, result, and audit metadata.

## Reproduction Assessment

Yes, from source inspection. The reproducible path is `openclaw skills install` or UI/gateway `skills.install` with `source: "clawhub"`, which reaches `installSkillFromClawHub` and copies via `installPackageDir` without `before_install` or a post-install observer.

## Solution Assessment

Unclear. The requested lifecycle hook direction is valid, but the best maintainable fix needs maintainer agreement on hook naming, compatibility with `before_install`, and the public audit payload contract.
