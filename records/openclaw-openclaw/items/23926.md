## Label Justifications

- `P1`: Third-party skill installation is a live supply-chain security workflow, and the missing lifecycle/audit observer blocks standard scanner and enterprise review integrations.
- `impact:security`: The issue is about scanning, blocking, and auditing untrusted skills before and after they are accepted onto a user's machine.

## Summary

Keep open: current main covers the blocking pre-install side through `security.installPolicy` and `before_install`, but the requested plugin-visible post-install/audit observer is still absent and needs maintainer product/security review.

## What This Changes

Request adds blocking pre-install and observational post-install skill lifecycle hook events so security plugins can scan, block, warn, and audit skill installs from ClawHub, UI, and update flows.

## Best Possible Solution

Keep `security.installPolicy` and `before_install` as the blocking seams, then add a documented non-blocking skill install/update completion hook with minimal redacted metadata and explicit observer-failure semantics.

## Reproduction Assessment

Yes, at source level. Current main's hook registry contains `before_install` only, and ClawHub install success writes `installedAt`/origin/lock metadata without invoking any post-install observer.

## Solution Assessment

Partly. Reusing `security.installPolicy` and `before_install` is the narrowest blocking solution, but the remaining audit capability should be a carefully specified non-blocking completion hook rather than accepting the original event names unchanged.
