## Label Justifications

- `P1`: Third-party skill installation is a live supply-chain security workflow, and the missing lifecycle/audit hook blocks standard scanner and enterprise review integrations.
- `impact:security`: The issue concerns scanning, blocking, and auditing untrusted skill installation before user machines accept third-party code and instructions.

## Summary

Keep open: current main now covers much of the blocking pre-install side through `before_install` and `security.installPolicy`, including ClawHub archive installs before copying, but it still lacks the requested post-install/audit lifecycle observer and needs product/security decisions before a public hook contract is added.

## What This Changes

The issue requests blocking pre-install and observational post-install lifecycle hooks for skill installation so security scanners can warn, block, and audit skill installs.

## Best Possible Solution

Keep `before_install` and `security.installPolicy` as the blocking policy seams, then add a documented non-blocking skill install/update completion hook with minimal redacted metadata, explicit observer-failure semantics, and a clear boundary for ClawHub-owned CLI behavior.

## Reproduction Assessment

Yes, at source level. Current main shows `before_install` and `security.installPolicy` cover pre-copy policy, while the hook registry and ClawHub install/update code still have no post-install observer event.

## Solution Assessment

Partly. Reusing `before_install` is the narrowest maintainable answer for blocking, but the remaining audit capability should be a carefully specified non-blocking completion hook rather than adding the originally proposed parallel event family unchanged.
