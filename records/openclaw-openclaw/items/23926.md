## Summary

Keep open. Current main only partially covers the requested lifecycle policy: gateway-managed skill dependency installs can run the blocking `before_install` hook, but native ClawHub install/update and the UI ClawHub path still copy downloaded skill folders without a pre-copy policy hook and there is no post-install observer contract.

## What This Changes

Request adds blocking skill pre-install and observational post-install hook events for third-party skill install scanning, blocking, warning, auditing, and compliance.

## Best Possible Solution

Define a documented, backward-compatible skill install lifecycle API that runs a blocking policy point before native ClawHub file copy across CLI, gateway, and UI paths, then emits a stable post-install observer with source, target, timestamp, result, and audit metadata.

## Reproduction Assessment

Yes, from source inspection. `openclaw skills install` and gateway/UI `skills.install` with `source: "clawhub"` reach `installSkillFromClawHub`, which calls `installPackageDir` without `scanSkillInstallSource` or a post-install observer.

## Solution Assessment

Unclear. The lifecycle-hook direction is valid, but the narrowest maintainable API needs maintainer agreement on whether to extend `before_install` or add skill-specific events, plus stable blocking and audit semantics.
