## Label Justifications

- `P1`: The remaining gap prevents security and compliance plugins from consistently auditing completed installations of untrusted third-party skills.
- `impact:security`: The requested lifecycle contract governs install-time blocking and audit visibility at the boundary where third-party skill content becomes trusted local code.

## Summary

Keep open. Current main now provides broad blocking pre-install policy coverage, but the distinct observational completion/audit hook remains absent; related scanner and skill-authored setup issues do not own that remaining plugin contract.

## What This Changes

Add blocking pre-install and observational post-install plugin hook events for skill installation and update workflows.

## Best Possible Solution

Extend the generic install-policy lifecycle with an observational event emitted after durable install metadata is committed, carrying target type, mode, source, target directory, timestamp, and outcome across OpenClaw-owned skill and plugin paths; coordinate standalone ClawHub CLI support separately.

## Reproduction Assessment

Yes, at source level. Current main clearly exercises blocking policy before supported installs, while the typed registry and post-copy code establish that no install-completion observer is emitted.

## Solution Assessment

Partly. Reusing the existing generic `before_install` contract is the best answer for blocking, but the remaining audit requirement should use a generic committed-outcome event rather than duplicate skill-specific hook names.
