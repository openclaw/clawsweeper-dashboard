## Label Justifications

- `P1`: Third-party ClawHub skill installs and updates can currently bypass enforceable local policy hooks before untrusted skill content lands on disk.
- `impact:security`: The issue concerns install-time security boundaries, scanner enforcement, and auditability for untrusted third-party skills.

## Summary

Keep open: current main still only partially covers install-time skill policy. The generic `before_install` hook can block gateway-managed skill dependency installs and uploaded archive installs, but native ClawHub install/update still opts out of scanning and there is no post-install observer with audit/outcome metadata.

## What This Changes

The issue requests blocking `skill:pre-install` and observational `skill:post-install` hook events for skill installation paths so security scanners and audit tools can enforce policy before skills are written and record completed installs.

## Best Possible Solution

Define a backward-compatible skill install lifecycle API that covers native ClawHub install/update and gateway/UI installs with a blocking pre-copy policy point plus a post-install audit observer carrying source, target, timestamp, result, and outcome metadata.

## Reproduction Assessment

Yes. This is source-reproducible as a feature gap: current main shows the ClawHub installer passing `scan: false`, while the hook surface exposes only `before_install` and no post-install outcome observer.

## Solution Assessment

Unclear. Extending `before_install` is plausible, but maintainers need to choose the public lifecycle API, blocking semantics, audit metadata, UI behavior, and OpenClaw-versus-ClawHub ownership boundary before implementation.
