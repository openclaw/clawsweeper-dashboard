## Summary

Keep open. Current main still only partially covers the requested lifecycle policy: installer/upload/dependency paths can use the generic blocking `before_install` hook, but native ClawHub install/update and the UI ClawHub path still route through `installSkillFromClawHub` with scanning disabled and no post-install observer.

## What This Changes

Request new blocking `skill:pre-install` and observational `skill:post-install` hook events for ClawHub/UI skill installation.

## Best Possible Solution

Define a documented, backward-compatible skill install lifecycle API that covers native ClawHub install/update, gateway/UI ClawHub installs, and a stable post-install observer with source, target, timestamp, result, and audit metadata.

## Reproduction Assessment

Yes, from source inspection. `openclaw skills install` and gateway/UI `skills.install` with `source: "clawhub"` reach `installSkillFromClawHub`, which calls `installExtractedSkillRoot` with `scan: false` and has no post-install observer.

## Solution Assessment

Unclear. The lifecycle-hook direction is valid, but the narrowest maintainable fix needs maintainer agreement on whether to extend `before_install` or add skill-specific events, plus stable blocking and audit semantics.
