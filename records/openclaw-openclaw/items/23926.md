## Label Justifications

- `P1`: Third-party skill installation is a live supply-chain security boundary, and the missing audit lifecycle hook blocks standard enforcement and enterprise review tooling.
- `impact:security`: The issue concerns scanning, blocking, and auditing untrusted skill installation before user machines accept third-party code and instructions.

## Summary

Keep open: current main covers much of the blocking side through the generic `before_install` policy hook, including ClawHub archive installs before copying, but the requested post-install/audit lifecycle event is still missing and the public hook contract needs maintainer product and security review.

## What This Changes

Requested blocking `skill:pre-install` and observational `skill:post-install` hook events for skill install and update paths.

## Best Possible Solution

Keep `before_install` as the blocking policy seam, then add a documented non-blocking install/update completion event with minimal redacted metadata, explicit observer-failure semantics, and a clear boundary for any ClawHub-owned CLI behavior.

## Reproduction Assessment

Yes, at source level. Current main shows pre-install policy coverage through `before_install`, while the typed hook registry and committed ClawHub install/update lifecycle still have no post-install observer event.

## Solution Assessment

Partly. Reusing `before_install` is the narrowest maintainable answer for blocking, but the remaining audit capability should be a carefully specified non-blocking completion hook rather than adding the originally proposed parallel event family unchanged.
