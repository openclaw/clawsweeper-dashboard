## Summary

Keep open. Current main only partially covers the requested lifecycle policy: gateway-managed skill dependency installs run the generic blocking `before_install` hook, but native ClawHub skill install/update, CLI, and UI ClawHub paths still copy downloaded skill folders without a pre-copy policy hook, and there is no post-install observational hook. A maintainer comment already identified this as the concrete tracker for the remaining work.

## What This Changes

The issue requests blocking skill pre-install and observational post-install hook events for ClawHub/native/UI skill installs so scanners can block or audit third-party skills.

## Best Possible Solution

Add a documented, backward-compatible lifecycle API that covers native ClawHub install/update before file copy, reuses it from gateway and UI paths, and emits a post-install observer with stable source, target, timestamp, result, and audit metadata.

## Reproduction Assessment

Yes, from source inspection. The reproducible path is `openclaw skills install` or UI/gateway `skills.install` with `source: "clawhub"`, which reaches `installSkillFromClawHub` and copies via `installPackageDir` without the current `before_install` scanner hook or a post-install observer.

## Solution Assessment

Unclear. The requested lifecycle hook is a valid direction, but the best maintainable fix needs maintainer agreement on hook naming, whether to extend `before_install`, and the post-install audit payload shape.
