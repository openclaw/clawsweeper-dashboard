## Summary

Keep open. Current main still only partially covers the requested lifecycle policy: scanner-backed dependency and upload installs can run `before_install`, but native ClawHub install/update still passes through an unscanned copy path and the public hook surface still lacks a post-install audit observer.

## What This Changes

Add blocking pre-install and observational post-install skill lifecycle hook events for ClawHub/UI skill installs, sync/update flows, and plugin-shipped skills.

## Best Possible Solution

Define a documented, backward-compatible skill install lifecycle API that covers native ClawHub install/update, gateway/UI ClawHub installs, a blocking pre-copy policy point, and a post-install audit observer with source, target, timestamp, result, and outcome metadata.

## Reproduction Assessment

Yes, for a source-verifiable feature gap rather than a runtime bug: current main shows native ClawHub install/update calling the archive installer with scanning disabled, while the hook surface exposes only `before_install` and no post-install outcome observer.

## Solution Assessment

Unclear. The lifecycle-hook direction is valid, but maintainers need to choose the public API shape, blocking semantics, audit payload, UI behavior, and OpenClaw-versus-ClawHub ownership boundary before implementation.
