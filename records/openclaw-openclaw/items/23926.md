## Summary

Keep open. Current main still only partially covers the requested lifecycle policy: generic `before_install` exists for installer paths that call the scan runtime, but native ClawHub install/update and the UI ClawHub path still bypass that policy point, and there is still no post-install audit hook.

## What This Changes

The issue requests blocking pre-install and observational post-install lifecycle hooks for skill installation.

## Best Possible Solution

Define a documented, backward-compatible skill install lifecycle API that covers native ClawHub install/update, gateway/UI ClawHub installs, and a stable post-install observer with source, target, timestamp, result, and audit metadata.

## Reproduction Assessment

Yes for the source-verifiable gap, not as a runtime bug: `openclaw skills install` and gateway/UI `skills.install` with `source: "clawhub"` reach `installSkillFromClawHub`, which calls `installExtractedSkillRoot` with `scan: false`. The hook surface also lacks a post-install observer payload.

## Solution Assessment

Unclear. The lifecycle-hook direction is valid, but maintainers need to decide whether to extend `before_install` or add skill-specific events, plus the stable blocking, audit, UI, and ClawHub scope semantics.
