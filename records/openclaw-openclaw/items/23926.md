## Summary

Keep open. Current main still only partially covers the requested skill-install lifecycle policy: gateway-managed skill dependency installs can run the blocking `before_install` hook, but native ClawHub install/update paths still copy skill folders without that policy hook and there is no post-install observer contract.

## What This Changes

Add blocking pre-install and observational post-install hook events for skill installation paths, including native ClawHub, gateway/UI, and sync-style installs.

## Best Possible Solution

Add a documented, backward-compatible lifecycle API that covers native ClawHub install/update before file copy, reuses it from CLI/gateway/UI paths, and emits a post-install observer with stable source, target, timestamp, result, and audit metadata.

## Reproduction Assessment

Yes, from source inspection. The reproducible path is `openclaw skills install` or gateway/UI `skills.install` with `source: "clawhub"`, which reaches `installSkillFromClawHub` and copies via `installPackageDir` without the existing policy hook or a post-install observer.

## Solution Assessment

Unclear. The lifecycle-hook direction is valid, but the narrowest maintainable API needs maintainer agreement on hook naming, compatibility with `before_install`, post-install payload shape, and ClawHub CLI ownership.
