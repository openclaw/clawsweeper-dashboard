## Summary

Keep open. Current main still only partially covers the requested lifecycle policy: scanner-backed dependency installs can run `before_install`, but native ClawHub install/update still bypasses that scan path and there is no post-install observer carrying audit/outcome metadata.

## What This Changes

The issue requests blocking pre-install and observational post-install lifecycle hooks for skill installation, including ClawHub, sync/update, UI installs, and plugin-shipped skills.

## Best Possible Solution

Define a documented, backward-compatible skill install lifecycle API that covers native ClawHub install/update, gateway/UI ClawHub installs, a blocking pre-copy policy point, and a post-install audit observer with source, target, timestamp, result, and outcome metadata.

## Reproduction Assessment

Yes for a source-verifiable feature gap, not a runtime bug: current main shows ClawHub install/update calling the archive installer with `scan: false`, while the public hook surface only exposes `before_install` and lacks post-install outcome metadata.

## Solution Assessment

Unclear. The lifecycle-hook direction is valid, but maintainers need to choose whether to extend `before_install` or add skill-specific events, plus the blocking semantics, audit payload, UI behavior, and ClawHub ownership boundary.
