## Summary

Keep open. Current main still only partially covers the requested lifecycle policy: generic `before_install` exists for some skill install paths, but native ClawHub install/update still bypasses that policy point and there is still no post-install audit observer.

## What This Changes

Request to add blocking pre-install and observational post-install skill lifecycle hooks for ClawHub and UI skill installation flows.

## Best Possible Solution

Define a documented skill install lifecycle API that covers native ClawHub install/update, gateway/UI ClawHub installs, a blocking pre-copy policy point, and a post-install audit observer with source, target, timestamp, result, and outcome metadata.

## Reproduction Assessment

Yes for a source-verifiable feature gap, not a runtime bug: current main shows native ClawHub install/update passing `scan: false`, while the public hook surface only exposes `before_install` and lacks post-install outcome metadata.

## Solution Assessment

Unclear. The lifecycle-hook direction is valid, but maintainers need to choose whether to extend `before_install` or add new events, plus the blocking semantics, audit payload, UI behavior, and OpenClaw-versus-ClawHub ownership boundary.
