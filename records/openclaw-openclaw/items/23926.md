## Summary

Keep open. Current main still only partially covers the requested skill lifecycle policy: some installer paths use the generic blocking `before_install` hook, but native ClawHub installs and updates still pass `scan: false`, and there is no post-install observer with audit/outcome metadata.

## What This Changes

Add blocking pre-install and observational post-install hook events for skill installation paths.

## Best Possible Solution

Define a documented, backward-compatible skill install lifecycle API that covers native ClawHub install/update, gateway/UI ClawHub installs, and a stable post-install observer with source, target, timestamp, result, and audit metadata.

## Reproduction Assessment

Yes, from source inspection. `openclaw skills install` and gateway/UI `skills.install` with `source: "clawhub"` reach `installSkillFromClawHub`, which passes `scan: false` into `installExtractedSkillRoot`, while post-install hook types are absent from the current hook catalog.

## Solution Assessment

Unclear. The lifecycle-hook direction is valid, but the narrowest maintainable fix needs maintainer agreement on whether to extend `before_install` or add skill-specific events, plus stable blocking, audit, and cross-owner ClawHub semantics.
