## Summary

Keep open. Current main still only partially covers the requested skill lifecycle policy: `before_install` exists for scanner-backed install paths, but native ClawHub install/update still passes `scan: false`, and the public hook surface still lacks a post-install observer for audit metadata.

## What This Changes

The issue requests blocking `skill:pre-install` and observational `skill:post-install` hook events for security scanning and audit during skill installation.

## Best Possible Solution

Define a documented, backward-compatible skill install lifecycle API that covers native ClawHub install/update, gateway/UI ClawHub installs, a blocking pre-copy policy point, and a stable post-install audit observer with source, target, timestamp, result, and outcome metadata.

## Reproduction Assessment

Yes for a source-verifiable feature gap, not as a runtime bug: current main shows ClawHub install/update calling the archive installer with `scan: false`, while the hook surface only exposes `before_install` and lacks post-install observer metadata.

## Solution Assessment

Unclear. The lifecycle-hook direction is valid, but maintainers need to choose whether to extend `before_install` or add skill-specific events, plus the blocking semantics, audit payload, UI behavior, and ClawHub ownership boundary.
