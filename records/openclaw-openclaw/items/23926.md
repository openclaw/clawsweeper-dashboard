## Summary

Keep open. Current main still only partially covers the requested install lifecycle policy: scanner-backed dependency and upload installs can run `before_install`, but native ClawHub install/update still bypasses that policy point and the hook surface has no post-install observer with audit/outcome metadata.

## What This Changes

Request adds blocking pre-install and observational post-install hook events for skill installation paths so policy plugins can scan, block, warn, and audit skill installs.

## Best Possible Solution

Define a documented, backward-compatible skill install lifecycle API that covers native ClawHub install/update, gateway/UI ClawHub installs, a blocking pre-copy policy point, and a post-install audit observer with source, target, timestamp, result, and outcome metadata.

## Reproduction Assessment

Yes for a source-verifiable feature gap, not a runtime bug: current main shows ClawHub install/update calling the archive installer with `scan: false`, while the public hook surface only exposes `before_install` and lacks post-install outcome metadata.

## Solution Assessment

Unclear. The lifecycle-hook direction is valid, but maintainers need to choose the public API shape, blocking semantics, audit payload, UI behavior, and OpenClaw-versus-ClawHub ownership boundary.
