## Label Justifications

- `P1`: Third-party skill installation is a live supply-chain security workflow, and the missing completion/audit hook blocks standard scanner and enterprise review integrations.
- `impact:security`: The issue is about scanning, blocking, and auditing untrusted skills before or after they are accepted onto a user's machine.

## Summary

Keep open: current main and the latest release cover the blocking pre-install side through `security.installPolicy` and `before_install`, but the requested plugin-visible post-install/audit observer is still absent and the remaining public security contract needs maintainer product/security review.

## What This Changes

Requested blocking pre-install and observational post-install lifecycle hook events for ClawHub and UI skill installation paths so scanner plugins can block, warn, and audit untrusted skills.

## Best Possible Solution

Keep the existing blocking seams, then add a documented non-blocking skill install/update completion hook with minimal redacted metadata, explicit ordering and failure semantics, and focused coverage for ClawHub, gateway, UI, and update paths.

## Reproduction Assessment

Yes, at source level. Current main exposes `before_install` only, and the ClawHub success path writes `installedAt` and origin/lock metadata without dispatching any post-install observer.

## Solution Assessment

Partly. Reusing `security.installPolicy` and `before_install` is the narrowest blocking solution, but the remaining audit capability should be specified as a non-blocking completion hook rather than accepting the original event names and payload shape unchanged.
