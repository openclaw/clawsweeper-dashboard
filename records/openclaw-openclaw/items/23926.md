## Summary

Keep open. Current main still only partially covers the requested lifecycle policy: generic `before_install` exists for paths that call the install scanner, but native ClawHub install/update and the UI ClawHub path still bypass that scan and there is no post-install audit hook.

## What This Changes

The issue requests blocking pre-install and observational post-install skill lifecycle hooks for ClawHub/UI/native skill installs so scanner plugins can enforce or audit skill installation.

## Best Possible Solution

Define a documented, backward-compatible skill install lifecycle API that covers native ClawHub install/update, gateway/UI ClawHub installs, and a stable post-install observer with source, target, timestamp, result, and audit metadata.

## Reproduction Assessment

Yes, from source inspection. `openclaw skills install` and gateway/UI `skills.install` with `source: "clawhub"` reach `installSkillFromClawHub`, which calls `installExtractedSkillRoot` with `scan: false`, while hook execution only occurs when scan parameters are supplied.

## Solution Assessment

Unclear. The lifecycle-hook direction is valid, but the narrowest maintainable fix needs maintainer agreement on whether to extend `before_install` or add skill-specific events, plus stable blocking and audit semantics.
