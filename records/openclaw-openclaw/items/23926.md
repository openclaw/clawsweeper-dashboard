## Label Justifications

- `P1`: Third-party skill installation is a live supply-chain security workflow, and the missing completion/audit hook blocks standard scanner and enterprise review integrations.
- `impact:security`: The issue concerns scanning, blocking, and auditing untrusted skill installation before user machines accept third-party code and instructions.

## Summary

Keep open: current main covers the blocking pre-install side through `security.installPolicy` and plugin `before_install`, including ClawHub-backed skill installs before copying, but the requested post-install audit/observer lifecycle hook is still missing and needs maintainer product/security design.

## What This Changes

Request to add blocking `skill:pre-install` and observational `skill:post-install` lifecycle hooks for skill install/update security scanning and audit integrations.

## Best Possible Solution

Keep `security.installPolicy` and `before_install` as the blocking policy seams, then add a documented non-blocking skill install/update completion hook with minimal redacted metadata, explicit observer-failure semantics, and a clear ClawHub CLI ownership boundary.

## Reproduction Assessment

Yes, at source level. Current main exposes blocking pre-install policy through `security.installPolicy` and `before_install`, while the hook registry and ClawHub completion path still have no post-install observer event.

## Solution Assessment

Partly. Reusing the existing blocking policy seams is the best fit for `skill:pre-install`, but the remaining audit capability should be a carefully specified completion observer rather than adding the originally proposed parallel event family unchanged.
