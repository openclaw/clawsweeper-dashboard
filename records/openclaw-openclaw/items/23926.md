## Summary

Keep open. Current main now covers the blocking pre-copy policy point better than the prior review because ClawHub/archive skill installs flow through the generic `before_install` policy path, but the requested post-install observational/audit hook and final public lifecycle contract are still missing.

## What This Changes

The issue requests blocking `skill:pre-install` and observational `skill:post-install` hook events for scanning, blocking, warning, and auditing skill installs from ClawHub, UI, sync/update, and plugin-shipped paths.

## Best Possible Solution

Keep this as the canonical OpenClaw tracker for the skill install lifecycle contract: preserve the current generic blocking `before_install` coverage, then add or explicitly reject a documented post-install audit observer for skill install/update outcomes.

## Reproduction Assessment

Not applicable as a bug reproduction, but yes for a source-level feature gap: current main exposes `before_install` and routes ClawHub/archive installs through it, while no post-install outcome hook exists in the hook catalog or docs.

## Solution Assessment

Unclear. Extending the existing `before_install` contract is the cleanest pre-install path, but the post-install observer needs a maintainer decision on public API shape, audit fields, failure semantics, and ClawHub ownership boundaries.
