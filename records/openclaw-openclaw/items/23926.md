## Label Justifications

- `P1`: Third-party skill installation is a live supply-chain security boundary, and the missing audit lifecycle contract blocks standard enforcement and enterprise review tooling.
- `impact:security`: The issue concerns scanning, blocking, and auditing untrusted skill installation before third-party code and instructions are accepted on user machines.

## Summary

Keep open: current main covers the blocking pre-copy side through `security.installPolicy` and the plugin `before_install` hook, including ClawHub archive installs, but it still lacks the requested post-install/audit lifecycle event and needs maintainer security/product decisions for the public hook contract.

## What This Changes

Request to add skill install lifecycle hook events so plugins can block unsafe skill installs before copy and observe completed installs for audit logging.

## Best Possible Solution

Keep `before_install` as the blocking policy seam, then design a documented non-blocking install/update completion hook with minimal redacted metadata, explicit observer failure semantics, and a clear boundary for ClawHub-owned CLI behavior.

## Reproduction Assessment

Yes, at source level. Current main shows pre-copy policy coverage through `before_install`, while the typed hook registry and ClawHub install/update lifecycle still have no post-install observer event.

## Solution Assessment

Partly. Reusing `before_install` is the narrow maintainable answer for blocking, but the remaining audit capability should be a carefully specified non-blocking completion hook rather than the originally proposed parallel event family unchanged.
