## Label Justifications

## Summary

Keep open. Current main and the latest release cover the blocking pre-install side with the generic `before_install` policy hook, but the requested post-install observational/audit lifecycle contract is still missing and needs maintainer security/product approval.

## What This Changes

Request to add blocking pre-install and observational post-install lifecycle hooks for skill installation.

## Best Possible Solution

Keep `before_install` as the blocking policy point and add a maintainer-approved generic install-completion observer for OpenClaw-owned install paths, with standalone ClawHub CLI support coordinated separately.

## Reproduction Assessment

Yes, at source level. Current hook types and install completion code show blocking pre-install coverage exists, but no committed install-completion observer is emitted.

## Solution Assessment

Partly. Reusing `before_install` is the best blocking path, but the remaining audit requirement should use a generic post-commit install lifecycle contract rather than duplicate skill-specific hook names.
