## Summary

Keep this PR open. Current main does not have the requested bundled-key/tools-root collision guard, but the proposed patch has a blocking compatibility issue around trusted managed/extra skill sources and still needs real behavior proof beyond unit tests.

## What This Changes

The PR adds skill-install guards, shared skill-key resolution, bundled skill-key tracking, and tests for non-bundled skills that claim bundled keys or share tools roots.

## Best Possible Solution

Align the collision gate with the existing trusted-source contract, keep the bundled-key/tools-root hardening for untrusted sources, and require real install-path proof before merge.

## Reproduction Assessment

No high-confidence live reproduction was run in this read-only review. Source inspection shows current main lacks the key/root collision check and the PR's unit tests exercise the intended block with mocks.

## Solution Assessment

No, not as written. The hardening direction is narrow, but the implementation should preserve the existing trusted managed/extra source contract or explicitly change that policy with docs and tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Exempt trusted skill sources from the collision gate:** `src/agents/skills-install.ts:107-108`
