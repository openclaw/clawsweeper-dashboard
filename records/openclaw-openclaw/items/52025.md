## Summary

Keep this PR open. Current main still does not render direct `{ type: "image", data, mimeType }` tool-result blocks inline, and the related bug #50779 remains open; the PR is a focused candidate but is stale against the current Control UI media path and still misses live streamed tool-result images plus visible nested captions.

## What This Changes

The PR updates Control UI chat image extraction, tool-card text extraction, and chat view tests so direct, image-only, and nested tool-result image blocks render inline outside collapsed tool output.

## Best Possible Solution

Keep this PR as a focused candidate for #50779, but land it only after adapting the fix to current main: direct and nested tool-result image blocks should render visibly in the default transcript, accompanying text should remain visible when images are pulled out, live stream and history reload paths should preserve structured image content, and existing Control UI media safety constraints should remain intact.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a `toolResult` message using the current `imageResult()` shape `{ type: "image", data, mimeType }`: current main produces that shape but `extractImages` does not read it, and tool-message images that are found render only inside collapsed output by default.

## Solution Assessment

No, not as-is. The PR points in the right direction, but the best fix must be rebased onto current main and cover the live stream projection plus visible text/caption behavior, rather than only static `messages` fixtures.
