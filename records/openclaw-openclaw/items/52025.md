## Summary

Keep open. The PR targets a real Control UI bug and fixes useful stored-message cases, but it remains unmerged/conflicting, lacks required real behavior proof, and still misses the live tool-event path that can discard structured image blocks before rendering.

## What This Changes

The PR updates Control UI grouped chat rendering and tool-card text extraction so direct and nested tool-result image blocks render outside collapsed tool output, with chat-view tests for direct, image-only, and nested cases.

## Best Possible Solution

Rebase or replace this with a current-main fix that preserves structured tool-result image content through live tool events, renders stored direct/nested results inline outside collapsed output, and adds regression coverage plus real Control UI proof.

## Reproduction Assessment

Yes, as a source-level reproduction: imageResult and the read tool emit direct type=image data/mimeType blocks, current extractImages ignores that direct shape, and live tool events stringify result content before grouped rendering. No live Control UI session was run in this read-only review.

## Solution Assessment

No, not as-is. The stored-message renderer changes are useful, but the maintainable fix must also preserve structured images through the live app-tool-stream projection and include real behavior proof.
