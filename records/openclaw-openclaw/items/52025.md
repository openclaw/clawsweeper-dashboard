## Summary

Keep open. This PR is a useful partial Control UI fix, but it still leaves live tool-result image blocks flattened to text, lacks after-fix real behavior proof from the external contributor, and needs rework against current main.

## What This Changes

The branch changes Control UI grouped chat rendering, tool-card text extraction, and chat-view tests so tool-result image blocks can render inline rather than only inside collapsed tool output.

## Best Possible Solution

Land a focused current-main fix that preserves structured tool-result image blocks through stored history and live streaming, renders associated text visibly, and adds regression coverage for grouped rendering and app-tool-stream projection.

## Reproduction Assessment

Yes, by source inspection. The tool helpers emit direct image blocks with data and mimeType, while current main ignores that shape in extractImages and converts live tool result data to text-only stream messages.

## Solution Assessment

No, not as-is. The stored-message renderer changes are useful, but the maintainable fix must also preserve structured image blocks in the live tool stream and include contributor-supplied real behavior proof before merge.
