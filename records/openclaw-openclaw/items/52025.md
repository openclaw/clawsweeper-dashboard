## Summary

Keep this PR open, but do not merge it as-is. Current main still misses the direct `{ type: "image", data, mimeType }` tool-result shape, and the PR addresses a real bug, but it remains incomplete for live streamed tool results and nested image-plus-text output.

## What This Changes

The PR updates Control UI chat image extraction, tool-card text extraction, and chat view tests so direct, image-only, and nested tool-result image blocks render inline outside collapsed tool output.

## Best Possible Solution

Land a current-main implementation that handles direct and nested image blocks from tool results in both completed history and live stream projections, keeps associated text visible with the extracted image, preserves existing media safety constraints, and adds focused tests plus the required changelog entry.

## Solution Assessment

No, not as-is. The PR points in the right direction, but the maintainable fix needs to be rebased onto current main and cover the stream projection plus visible nested text behavior, not only static completed-message fixtures.
