## What This Changes

The PR updates Control UI grouped chat rendering, tool-card text extraction, and chat tests so direct and nested tool-result image blocks can render inline outside collapsed tool output.

## Best Possible Solution

Land or replace this with a current-main patch that preserves structured tool-result images through stored history and live tool-stream messages, keeps images visible outside collapsed tool details, and covers both paths.

Yes, by source-level reproduction. The tool helper emits direct data/mimeType image blocks, while current Control UI extraction ignores that shape and live tool streaming reduces result content to text-only output.

## Solution Assessment

No, not as-is. The grouped-render changes are useful for stored messages, but the maintainable fix must also preserve structured image blocks through live app-tool-stream projection and be rebased onto current main.
