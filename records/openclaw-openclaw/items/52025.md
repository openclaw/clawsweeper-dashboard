## Summary

Keep open: the underlying Control UI bug is still valid and this PR is a plausible implementation path, but it should not merge yet because live tool streams still flatten structured image output and the external PR has no after-fix real behavior proof.

## What This Changes

The branch changes Control UI grouped chat rendering, tool-card text extraction, and chat-view tests so stored tool-result image blocks can render inline outside collapsed tool output.

## Best Possible Solution

Land or replace a focused current-main fix that carries structured tool-result image blocks through stored history and live app-tool-stream projection, renders associated text, and adds regression coverage for both paths.

## Reproduction Assessment

Yes, by source-level reproduction. Tool helpers emit direct image blocks with data and mimeType, while current main ignores that shape in grouped rendering and converts live tool results to text-only stream messages.

## Solution Assessment

No, not as-is. The stored-message rendering changes are useful, but the maintainable fix must also preserve structured image blocks in the live tool stream and include contributor-supplied real behavior proof before merge.
