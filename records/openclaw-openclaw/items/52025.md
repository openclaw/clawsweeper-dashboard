## Summary

Keep open. This is a valid Control UI bug-fix PR, but the patch is not sufficient as-is because live tool streams still flatten structured image results to text, and the external contributor has not supplied real after-fix behavior proof.

## What This Changes

The PR changes Control UI grouped chat rendering, tool-card text extraction, and chat tests so tool-result image blocks with `data` and `mimeType` can render inline instead of staying hidden in collapsed tool output.

## Best Possible Solution

Land a current-main-focused fix that preserves structured tool-result image content through stored history and live app-tool-stream messages, renders associated text and images visibly, and adds regression coverage for both paths.

## Reproduction Assessment

Yes, by source-level reproduction. Tool helpers emit direct `data`/`mimeType` image blocks, while Control UI extraction ignores that shape and live tool-stream code reduces result content to text-only entries.

## Solution Assessment

No, not as-is. The grouped-render changes are useful for stored messages, but the maintainable fix must also preserve structured image blocks through live tool-stream projection and include real behavior proof before merge.
