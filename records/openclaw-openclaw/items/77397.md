## Summary

Keep open: current `main` still rejects Bedrock image/media-understanding execution when `auth.mode` is `aws-sdk` and no static `apiKey` is present. The linked open PR #77479 is the right active implementation path, but it has not merged yet.

## What This Changes

Allow Bedrock image/media-understanding execution to use `auth.mode: "aws-sdk"` without a static `apiKey`, matching the text runtime path.

## Best Possible Solution

Land a narrow media auth fix that allows Bedrock `aws-sdk` no-key auth through image/media-understanding runtime setup, preserves strict key checks for key-based providers, and adds focused regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: return `{ mode: "aws-sdk", apiKey: undefined }` for an image-capable `amazon-bedrock` model, then the image or media runner path reaches `requireApiKey` and throws before AWS SDK credential resolution.

## Solution Assessment

Yes. The narrow maintainable solution is to mirror the text runtime's no-key AWS SDK allowance in the media auth boundary, preferably through a small shared helper, then land the linked PR or an equivalent patch with regression coverage.
