## Summary

Keep open: current main still has the reported Bedrock AWS SDK auth mismatch, and #77479 is an open closing PR for the same bug, so the issue should remain open until that PR lands or is closed.

## What This Changes

Allow Bedrock image and media-understanding execution to use `auth.mode: "aws-sdk"` without a static `apiKey`, matching the text/runtime auth behavior.

## Best Possible Solution

Review and land or close #77479, keeping the final implementation scoped to allowing missing API keys only for AWS SDK Bedrock media auth paths with focused regression coverage.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to have `getApiKeyForModel` or `resolveApiKeyForProvider` return `{ mode: "aws-sdk", apiKey: undefined }` for an image-capable `amazon-bedrock` model; current main then reaches `requireApiKey` and throws before AWS SDK credentials can resolve.

## Solution Assessment

Yes. The narrow maintainable fix is to make the image/media runtime use the same `aws-sdk` no-key allowance as the text path, with regression tests; because #77479 is already open with closing syntax, the issue should wait for that PR outcome.
