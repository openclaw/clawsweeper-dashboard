## Summary

Keep open: current main still has the reported Bedrock AWS SDK auth mismatch. Bedrock auth can resolve to aws-sdk with no static apiKey, while the image and media-understanding paths still call requireApiKey before the AWS SDK credential chain can run.

## What This Changes

The issue requests that Bedrock image-tool authentication honor aws-sdk credential-chain mode without requiring a pasted apiKey.

## Best Possible Solution

Allow aws-sdk/no-static-key auth through the image and media-understanding runtime boundaries, preserve strict key enforcement for other auth modes, and add focused Bedrock regression coverage.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to return { mode: "aws-sdk", apiKey: undefined } from getApiKeyForModel or resolveApiKeyForProvider for an image-capable amazon-bedrock model and call the image/media-understanding path; current main reaches requireApiKey and throws.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the text-path missing-key allowance for aws-sdk in media/image auth preparation, ideally through a small shared helper or explicit allowMissingApiKeyModes seam rather than weakening all API-key checks.
