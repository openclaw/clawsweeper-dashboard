## Summary

Keep open: current `main` still has the reported Bedrock AWS SDK auth mismatch, and an open PR already uses closing syntax for this issue, so the issue should stay open until that PR is reviewed and merged or closed.

## What This Changes

The issue requests that Bedrock image/media execution allow `auth.mode: "aws-sdk"` without a static `apiKey`, matching the text path and documented AWS SDK credential-chain behavior.

## Best Possible Solution

Land or revise the linked media auth PR so image and media-runner Bedrock execution accept `aws-sdk` keyless auth while preserving strict key requirements for normal API-key providers.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure `amazon-bedrock` with `auth: "aws-sdk"` and no `apiKey`, call the image/media runtime, and current `main` reaches `requireApiKey`, which throws before the AWS SDK credential chain can run.

## Solution Assessment

Yes. The narrow maintainable fix is to apply the existing `aws-sdk` missing-key allowance to the image/media runtime boundary, with focused regression tests and no unrelated provider-auth behavior changes.
