## Summary

Keep open: current `main` and `v2026.6.6` still initialize omitted embedded-run provider/model values from OpenAI defaults, while the SDK docs and gateway startup path point to configured default-model resolution. I did not find a merged or viable canonical item that owns this exact omitted-params bug.

## What This Changes

The issue reports that `runEmbeddedAgent` ignores `agents.defaults.model` and falls back to `openai/gpt-5.5` when plugin callers omit `provider` and `model`.

## Best Possible Solution

Resolve the embedded runner's initial provider/model through the existing configured default-model path, preserve explicit per-call overrides, and add regression coverage for a non-OpenAI default with omitted `provider` and `model`.

## Reproduction Assessment

Yes, from source inspection rather than a live plugin run: omitted `runEmbeddedAgent` provider/model values still flow to `openai/gpt-5.5`, while configured-default helpers and docs show the expected path.

## Solution Assessment

Yes. The narrowest maintainable fix is to reuse the existing configured model-resolution helper inside the embedded runner instead of adding a new config option or requiring every plugin to pass provider/model explicitly.
