## What This Changes

The issue reports that object-shaped skill config requirements are stringified as `[object Object]` and then fail skill eligibility checks.

## Best Possible Solution

Land a narrow reviewed fix that normalizes object-shaped config entries by `path`, preserves safe diagnostics, and updates tests plus docs/types to match the supported contract.

## Reproduction Assessment
