## Summary

This should stay open as a product feature request. Current main has reset lifecycle, transcript archiving, session-memory files, and a startup-context prelude, but it does not automatically summarize or inject the previous conversation when an implicit daily reset rotates a session after the user sends the next message.

## What This Changes

Request to carry recent conversation context or a generated summary across daily session resets, likely via a new `session.reset.carryContext` configuration.

## Best Possible Solution

Define a core session-continuity design that reuses existing startup-context or session-memory surfaces where possible, then add a documented config contract and focused coverage for implicit daily rollover.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and tests show the current implicit daily reset path creates a new session and archives the prior transcript without automatically injecting a summary into the next user turn.

## Solution Assessment

Unclear whether the proposed `session.reset.carryContext` shape is the best final API. The direction is plausible, but maintainers should first decide whether this belongs in core startup context, session-memory hooks, plugin next-turn injections, or a new session reset config surface.
