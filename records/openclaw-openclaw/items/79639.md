## Summary

Keep open. The fresh report matches a current source-level agent-id gap in the message-tool outbound media path, and the overlapping fix PR at https://github.com/openclaw/openclaw/pull/39034 is still open rather than merged.

## What This Changes

Agents should be able to send workspace-local images and PDFs through the message tool or outbound MEDIA attachments after the reported 2026.4.29-2026.5.4 regression.

## Best Possible Solution

Reproduce the current-main direct-agent send path, then thread the effective agent identity into message-tool media access or stage the file into managed media without changing the global allowlist model.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes at source level: an agent toolset can have requesterAgentIdOverride without agentSessionKey, after which the message tool passes agentId undefined and workspace-* paths fall back to the rejecting default-root path. I did not establish live WSL2/provider proof in this read-only review.

## Solution Assessment

Is this the best way to solve the issue? The likely best fix is narrow: pass the effective agent id into createMessageTool/runMessageAction and add regression coverage, while leaving trustedLocalFileRoots and default local-media validation unchanged. The existing open PR at https://github.com/openclaw/openclaw/pull/39034 should be evaluated as source material before creating a replacement.
