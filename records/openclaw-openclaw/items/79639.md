## Summary

Current main has the merged gateway message-action media-root fix, and the same source/test behavior is present in the v2026.5.10-beta.2 release tag. The previously affected gateway path now passes agent-scoped workspace roots into plugin dispatch, so workspace-local attachments no longer fall back to default-only media roots.

## What This Changes

Agents should be able to send images and PDFs from their own workspace directory through outbound message actions and MEDIA attachment handling.

## Best Possible Solution

Keep the merged gateway dispatch fix and direct affected users to v2026.5.10-beta.2, current main, or the next stable release containing it.

## Reproduction Assessment

Yes. Source comparison shows v2026.5.7’s gateway message.action handler forwarded agentId but not mediaLocalRoots, while the current regression test exercises a Telegram workspace media path and asserts the agent workspace root is passed.

## Solution Assessment

Yes. Passing getAgentScopedMediaLocalRoots at the gateway dispatch boundary is the narrowest maintainable fix because it uses the existing agent-scoped media contract and leaves plugin action handling unchanged.
