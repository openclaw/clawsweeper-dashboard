## Summary

Keep open. Current main still sends Feishu streaming-card replies as an initial card-reference message followed by Card Kit content/settings updates, so mention-bearing final text is not delivered through a notification-bearing message path.

## What This Changes

Request that Feishu streaming-card replies send push notifications when an agent writes Feishu `<at user_id="...">...</at>` mention tags in response text.

## Best Possible Solution

Keep this as a Feishu plugin-owned bug and implement an API-verified notification strategy for mention-bearing streaming replies, with a documented fallback when Card Kit updates cannot notify.

## Reproduction Assessment

Yes, at source level. Enable Feishu streaming in `card` mode or markdown-triggered `auto`, have the agent emit `<at user_id="OPEN_ID">name</at>`, and current main places that text into Card Kit updates after the initial card-reference message; live push delivery remains the final acceptance proof.

## Solution Assessment

Unclear. Adding a `mentions` parameter to `close()` is not enough by itself because `close()` mutates an existing Card Kit card instead of sending a new notification-bearing message.
