## Summary

Keep open. Current main still delivers Feishu streaming-card reply text through Card Kit card updates after the initial card-reference message, with no mention metadata or notification-bearing send after the final text.

## What This Changes

The issue requests Feishu streaming-card replies to notify mentioned users when agent text contains explicit `<at>` mention tags.

## Best Possible Solution

Keep this as a Feishu plugin-owned bug and implement an API-verified notification strategy for explicit mention-bearing streaming replies, with a documented fallback if Card Kit updates cannot notify.

## Reproduction Assessment

Yes, at source level: enable Feishu streaming in `card` mode or markdown-triggered `auto`, have the agent emit `<at user_id="OPEN_ID">name</at>`, and current main places that text into Card Kit updates after the initial card-reference message. Live Feishu push delivery remains the missing acceptance proof.

## Solution Assessment

Unclear. Adding a `mentions` parameter to `close()` is not sufficient by itself because current `close()` mutates an existing Card Kit card instead of sending a new notification-bearing Feishu message.
