## Summary

Keep open. Current main still routes Feishu streaming-card final text through Card Kit updates after sending the initial card-reference message, so agent-authored mention tags do not have a notification-bearing send path; open #60103 is adjacent mention syntax normalization, not the same streaming notification behavior.

## What This Changes

Ensure Feishu streaming-card replies that include agent-authored `<at user_id="...">...</at>` tags notify the mentioned user like non-streaming Feishu text replies.

## Best Possible Solution

Keep this in the Feishu plugin and implement an API-verified notification strategy for mention-bearing streaming replies, preserving streaming where possible and documenting a fallback when Card Kit updates cannot notify.

## Reproduction Assessment

Yes, at source level. With Feishu streaming enabled in card mode, or auto mode for card-capable content, current main sends only a card reference first and later writes final mention-bearing text through Card Kit update/close calls.

## Solution Assessment

Unclear. Adding a `mentions` parameter to `close()` alone is not enough because `close()` mutates an existing Card Kit card instead of sending a new notification-bearing message; maintainers should choose an API-supported fallback or companion notification pattern.
