## Summary

Keep open. Current main still routes Feishu streaming-card final text through Card Kit updates after the initial card-reference message, so raw mention tags are not delivered on a notification-bearing message path.

## What This Changes

Request that Feishu streaming-card replies notify mentioned users when agent output contains `<at user_id="...">name</at>` tags.

## Best Possible Solution

Keep this as Feishu plugin-owned work and implement an API-verified notification strategy for mention-bearing streaming replies, with a documented fallback when Card Kit updates cannot notify.

## Reproduction Assessment

Yes, at source level. With Feishu streaming enabled in `card` mode or markdown-triggered `auto`, a final response containing `<at user_id="...">` is written through Card Kit update/close paths rather than a notification-bearing message create/reply path; live Feishu push delivery is still the acceptance proof.

## Solution Assessment

Unclear. Adding a `mentions` parameter to `close()` is not enough by itself because `close()` mutates an existing Card Kit card, so maintainers need to choose a Feishu-supported notification path or fallback behavior.
