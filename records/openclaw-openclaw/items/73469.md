## Summary

Keep open. Current main still sends Feishu streaming replies as a card-reference message followed by Card Kit content/settings mutations, so agent-authored raw mentions are not delivered through a fresh notification-bearing message path; related #60103 covers syntax normalization but does not settle the streaming notification behavior.

## What This Changes

The issue asks Feishu streaming cards to trigger push notifications when agent-authored `<at user_id="...">...</at>` mentions appear in streamed reply text.

## Best Possible Solution

Implement this in the Feishu plugin as an API-verified mention-notification strategy for streaming replies, with a documented fallback when Card Kit updates cannot notify mentioned users.

## Reproduction Assessment

Yes. Enable Feishu streaming in `card` mode or markdown-triggered `auto`, have the agent emit `<at user_id="OPEN_ID">name</at>`, and current main will put that text into Card Kit updates after the initial card-reference message; the final proof is a live group push-notification check.

## Solution Assessment

Unclear. Adding a `mentions` parameter to `close()` alone is unlikely to solve the user-visible notification gap because `close()` currently mutates an existing card instead of sending a notification-bearing message.
