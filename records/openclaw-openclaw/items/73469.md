## Summary

Keep open. Current main still routes Feishu streaming-card replies through an initial card-reference message followed by Card Kit updates, so mention-bearing final text is not delivered through a notification-bearing message path.

## What This Changes

Make Feishu streaming-card replies send push notifications when agent-authored `<at user_id="...">name</at>` mentions appear in the response text.

## Best Possible Solution

Keep this as Feishu plugin-owned behavior and implement an API-verified notification strategy for mention-bearing streaming replies, with a documented fallback if Card Kit updates cannot notify.

## Reproduction Assessment

Yes, at source level. Enable Feishu streaming in `card` mode or markdown-triggered `auto`, have the agent emit `<at user_id="OPEN_ID">name</at>`, and current main places that text into Card Kit updates after the initial card-reference message; live push delivery remains the acceptance proof.

## Solution Assessment

Unclear. Adding a `mentions` parameter to `close()` is not enough by itself because `close()` mutates an existing Card Kit card instead of sending a new notification-bearing message.
