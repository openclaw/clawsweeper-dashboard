## Summary

Keep open. Current main still sends Feishu streaming replies as an initial card-reference message followed by Card Kit content/settings updates; agent-authored `<at user_id="...">...</at>` text is not converted into a notification-bearing message path, so the reported gap remains distinct from #60103.

## What This Changes

Issue requests Feishu streaming cards to trigger push notifications when agent-authored text contains `<at user_id="...">...</at>` mentions.

## Best Possible Solution

Keep this as a Feishu plugin-owned bug and implement an API-verified mention-notification strategy for streaming replies, with a documented fallback or controlled final notification-bearing send when Card Kit updates cannot notify.

## Reproduction Assessment

Yes. Enable Feishu streaming in `card` mode or markdown-triggered `auto`, have the agent emit raw `<at user_id="...">...</at>` text, and current main puts that text into Card Kit updates after the initial card-reference message; live Feishu notification confirmation remains the acceptance check.

## Solution Assessment

Unclear. Adding a mentions parameter to `close()` is not sufficient by itself because `close()` currently performs Card Kit updates/settings patches rather than sending a new notification-bearing message.
