## Summary

Keep open. Current main still routes Feishu streaming-card final text through Card Kit updates after the notification-time card-reference message, so agent-authored `<at user_id="...">...</at>` text does not get a notification-bearing send path.

## What This Changes

Feishu streaming-card replies should notify mentioned users when final agent text contains Feishu `<at user_id>` mention tags.

## Best Possible Solution

Keep this as a Feishu plugin-owned bug and choose an API-verified strategy for mention-bearing streaming replies, such as a documented non-streaming fallback or a controlled final notification-bearing send when Card Kit updates cannot notify.

## Reproduction Assessment

Yes. Enable Feishu streaming in `card` or markdown-triggered `auto`, have the agent emit raw `<at user_id="...">...</at>` text, and current main sends the final text through Card Kit updates after the initial card-reference message; live Feishu notification confirmation is still the acceptance check.

## Solution Assessment

Unclear. A `close()` mentions parameter alone is unlikely to be sufficient in the current design because `close()` updates Card Kit content/settings instead of sending a new notification-bearing Feishu message.
