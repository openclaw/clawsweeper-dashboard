## Summary

Keep open. Current main still routes Feishu streaming-card final text through Card Kit card updates after the initial card-reference message, and there is no notification-bearing delivery path for agent-authored raw `<at user_id="...">...</at>` tags.

## What This Changes

Request that Feishu streaming-card replies send push notifications when an agent writes Feishu `<at user_id="...">...</at>` mention tags in response text.

## Best Possible Solution

Keep this as a Feishu plugin-owned bug and implement an API-verified notification strategy for mention-bearing streaming replies, with a documented fallback when Card Kit updates cannot notify.

## Reproduction Assessment

Yes. Enable Feishu streaming in `card` mode or markdown-triggered `auto`, have the agent emit `<at user_id="OPEN_ID">name</at>`, and current main places that text into Card Kit updates after the initial card-reference message; live Feishu group push delivery is the final acceptance proof.

## Solution Assessment

Unclear. Adding a `mentions` parameter to `close()` is not sufficient by itself because `close()` mutates an existing Card Kit card instead of sending a new notification-bearing message.
