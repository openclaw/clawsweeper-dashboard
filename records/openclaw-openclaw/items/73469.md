## Summary

Keep open. Current main still sends Feishu streaming-card replies as an initial card-reference message followed by Card Kit element updates, so mention-bearing final text is not delivered through a notification-bearing message path.

## What This Changes

Request Feishu streaming-card replies to send push notifications when agent-authored text includes explicit `<at user_id="...">name</at>` mentions.

## Best Possible Solution

Keep this as a Feishu plugin bug and implement an API-verified notification strategy for explicit mention-bearing streaming replies, with a documented fallback if Card Kit updates cannot notify.

## Reproduction Assessment

Yes, at source level: enable Feishu streaming in card mode or markdown-triggered auto mode, have the agent emit `<at user_id="OPEN_ID">name</at>`, and current main places that text into Card Kit updates after the initial card-reference message. Live Feishu push delivery remains the missing acceptance proof.

## Solution Assessment

Unclear. Adding a `mentions` parameter to `close()` is not sufficient by itself because current `close()` mutates an existing Card Kit card instead of sending a new notification-bearing Feishu message.
