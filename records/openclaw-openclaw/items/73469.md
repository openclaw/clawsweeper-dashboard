## Summary

Keep open. Current main still routes Feishu streaming-card final text through Card Kit card updates after the initial card-reference message, so agent-authored mention tags do not go through a notification-bearing Feishu message path.

## What This Changes

Feishu streaming-card replies should notify mentioned users when final agent text contains explicit `<at user_id="...">name</at>` tags.

## Best Possible Solution

Keep this as a Feishu plugin-owned bug and implement an API-verified notification strategy for explicit mention-bearing streaming replies, with a documented fallback when Card Kit updates cannot notify.

## Reproduction Assessment

Yes, at source level: enable Feishu streaming in `card` mode or markdown-triggered `auto`, have the agent emit `<at user_id="OPEN_ID">name</at>`, and current main places that text into Card Kit updates after the initial card-reference message. Live Feishu push delivery remains the final acceptance proof.

## Solution Assessment

Unclear. Adding a `mentions` parameter to `close()` is not sufficient by itself because current `close()` mutates an existing Card Kit card instead of sending a new notification-bearing Feishu message.
