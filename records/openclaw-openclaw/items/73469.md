## Summary

Keep open. Current main still delivers mention-bearing Feishu streaming finals by mutating an existing Card Kit card after the initial card-reference message, so it has not solved push notifications for agent-authored `<at>` tags; #60103 is adjacent normalization work but does not close this notification gap.

## What This Changes

The issue asks Feishu streaming-card replies containing agent-authored `<at user_id="...">` tags to send push notifications like non-streaming Feishu replies.

## Best Possible Solution

Keep this in the Feishu plugin and implement a Feishu-supported notification strategy for mention-bearing streaming replies, with a documented fallback when Card Kit updates cannot notify.

## Reproduction Assessment

Yes, at source level. Enable Feishu streaming in `card` mode or markdown-triggered `auto`, have the agent emit `<at user_id="OPEN_ID">name</at>`, and current main places the final text into Card Kit updates after the initial card-reference message; live push delivery remains the acceptance proof.

## Solution Assessment

Unclear. Adding a `mentions` parameter to `close()` alone is not obviously sufficient because `close()` currently mutates an existing Card Kit card rather than sending a new notification-bearing message.
