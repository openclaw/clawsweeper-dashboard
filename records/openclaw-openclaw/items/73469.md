## Summary

Keep open. Current main still routes Feishu streaming-card final text through Card Kit updates after the initial card-reference message, so mention-bearing agent text is not delivered through the normal notification-bearing message path.

## What This Changes

Request that Feishu streaming-card replies notify users when agent-authored `<at user_id="...">` mentions appear in final text, matching non-streaming mention behavior.

## Best Possible Solution

Keep the fix in the Feishu plugin: choose an API-verified mention-notification strategy for streaming replies and document the fallback when Card Kit updates cannot notify.

## Reproduction Assessment

Yes, at source level. Enable Feishu streaming in `card` mode or `auto` with card-triggering markdown, have the agent emit a `<at user_id="OPEN_ID">name</at>` tag, and current main puts the final text into Card Kit updates after the initial card-reference message.

## Solution Assessment

Unclear. Adding a `mentions` parameter to `close()` alone is unlikely to solve this because `close()` mutates an existing Card Kit card instead of sending a new notification-bearing message.
