## Summary

Keep open because current main still delivers Feishu streaming-card final text through Card Kit updates, so agent-authored mention text is not sent on a notification-bearing message path.

## What This Changes

Feishu streaming-card replies should notify mentioned users when the agent emits `<at user_id="...">name</at>` in the final response text.

## Best Possible Solution

Keep this as a Feishu plugin-owned bug and implement an API-verified notification strategy that preserves streaming when possible, with a documented fallback for mention-bearing replies when Card Kit updates cannot notify.

## Reproduction Assessment

Yes, at source level. Enable Feishu streaming in `card` mode or markdown-triggered `auto`, have the agent emit `<at user_id="OPEN_ID">name</at>`, and current main places that text into Card Kit updates after the initial card-reference message; live push delivery remains the final acceptance proof.

## Solution Assessment

Unclear. Adding a `mentions` parameter to `close()` is unlikely to be sufficient because `close()` mutates an existing Card Kit card instead of sending a new notification-bearing message; maintainers should pick an API-verified fallback or notification pattern.
