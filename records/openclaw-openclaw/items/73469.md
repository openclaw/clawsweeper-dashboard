## Summary

Keep open. Current main still delivers Feishu streaming replies through a Card Kit card-reference message followed by card content/settings updates, so agent-authored `<at user_id="...">...</at>` text is not sent through a notification-bearing message path.

## What This Changes

The issue requests Feishu streaming-card replies with agent-authored `<at user_id="...">...</at>` mentions to notify mentioned users without disabling streaming.

## Best Possible Solution

Keep this as a Feishu plugin-owned bug and implement an API-verified notification strategy for mention-bearing streaming replies, with a documented fallback when Card Kit updates cannot notify.

## Reproduction Assessment

Yes. Enable Feishu streaming in `card` mode or markdown-triggered `auto`, have the agent emit `<at user_id="OPEN_ID">name</at>`, and current main places that text into Card Kit updates after the initial card-reference message; a live Feishu group notification check is the acceptance proof.

## Solution Assessment

Unclear. Adding a mentions parameter to `close()` is not enough by itself because `close()` currently updates an existing Card Kit card rather than sending a new notification-bearing message.
