## Summary

Keep open. Current main still routes Feishu streaming-card finals through Card Kit content updates after the initial card-reference message, so the reported mention-notification gap remains plausible and distinct from the broader open syntax-normalization issue #60103.

## What This Changes

The issue requests Feishu streaming-card replies that contain agent-authored `<at user_id="...">...</at>` tags to notify mentioned users like non-streaming text replies.

## Best Possible Solution

Keep this as a Feishu plugin-owned bug and implement an API-verified strategy for mention-bearing streaming replies, such as a documented fallback to non-streaming or a controlled final notification-bearing message when Card Kit updates cannot notify.

## Reproduction Assessment

Yes. Enable Feishu streaming in `card` mode or markdown-triggered `auto`, have the agent emit raw `<at user_id="...">...</at>` text, and current main sends that final text through Card Kit updates after the notification-time card-reference message; live Feishu push confirmation is the acceptance check.

## Solution Assessment

Unclear. A `close()` mentions parameter alone is unlikely to be sufficient because `close()` currently updates Card Kit content/settings rather than sending a message that can carry notification metadata.
