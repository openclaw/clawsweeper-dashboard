## Summary

Keep open. Current main still routes arbitrary unknown CLI roots through the plugin allowlist fallback, and the issue-linked fix PR at https://github.com/openclaw/openclaw/pull/80123 is open and unmerged, so this issue should not be closed before a fix lands.

## What This Changes

The issue requests that truly unknown CLI subcommands fall through to normal unknown-command handling instead of suggesting edits to `plugins.allow`.

## Best Possible Solution

Land or replace the linked fix so only real plugin IDs and known plugin-owned aliases get allowlist remediation, while unmatched CLI roots use the normal unknown-command path.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: `resolveMissingPluginCommandMessage("totally-unknown", { plugins: { allow: ["quietchat"] } }, { registry: ... })` is currently expected to return the misleading allowlist message, and `runCli` can throw it before Commander parsing.

## Solution Assessment

Yes. The narrow maintainable solution is a CLI primary-token classification or known-plugin check before the fallback; the linked PR attempts that direction but must merge before this issue is resolved.
