## Summary

Keep open. This PR is still the canonical branch for the TUI reconnect UX work that current main lacks, but the current head has blocking integration and security issues against main and lacks external real behavior proof.

## What This Changes

The branch adds Ctrl+R/manual reconnect and disconnect guidance to the TUI, reconnect timer handling in the gateway client path, macOS remote-gateway probe UI, and a diffs cleanup test adjustment.

## Best Possible Solution

Rebase onto current main, model manual reconnect as a backend capability or gateway-only path, preserve strict macOS SSH host-key options, and add redacted real behavior proof before merge.

## Reproduction Assessment

Yes. Source inspection reproduces the blockers: the PR head calls client.reconnect() where current main uses a TuiBackend without reconnect, and the PR head uses accept-new SSH host-key checking where current main requires strict known-host verification.

## Solution Assessment

No. The UX direction is reasonable, but this implementation is not the best merge path until it fits the current backend contract and preserves the macOS SSH security policy.
