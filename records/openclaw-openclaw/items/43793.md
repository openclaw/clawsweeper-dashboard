## Summary

Keep open; the central TUI reconnect UX is still not on current main, but this branch is not mergeable as-is because it predates the TuiBackend abstraction, weakens macOS SSH host-key policy, and lacks real behavior proof.

## What This Changes

The branch adds TUI disconnect guidance and Ctrl+R reconnect plumbing, gateway reconnect timer handling, macOS remote gateway probe UI, and a diff artifact cleanup test adjustment.

## Best Possible Solution

Rebase the useful TUI reconnect UX onto current main via an explicit TuiBackend reconnect capability, preserve the shared strict SSH helper for macOS probes, and add redacted real behavior proof before merge.

## Reproduction Assessment

Yes. Source inspection reproduces the blockers: the PR head calls client.reconnect() where current main's TuiBackend has no reconnect method, and the PR head uses StrictHostKeyChecking=accept-new where current main requires strict trusted host keys.

## Solution Assessment

No. The UX direction is useful, but this implementation needs a current-main rebase through the backend contract and must preserve the macOS strict SSH host-key policy.
