## Summary

Keep this PR open: current main still lacks the central TUI reconnect guidance, but the branch is not merge-ready because it conflicts with the current TUI backend contract, weakens the macOS SSH host-key policy, and lacks contributor real behavior proof.

## What This Changes

The branch adds TUI generic disconnect guidance and Ctrl+R reconnect plumbing, gateway reconnect timer handling, macOS remote gateway probe UI, and a diff artifact cleanup test adjustment.

## Best Possible Solution

Rebase the useful TUI reconnect UX onto current main with a backend-aware reconnect capability, keep macOS remote probes on the shared strict SSH host-key helper, and require redacted real behavior proof before merge.

## Reproduction Assessment

Yes. Source inspection reproduces the blockers: the PR head calls client.reconnect() where current main's TuiBackend has no reconnect method, and the PR head uses StrictHostKeyChecking=accept-new where current main requires strict trusted host keys.

## Solution Assessment

No. The UX direction is useful, but this implementation needs a current-main rebase through the backend contract and must preserve the macOS strict SSH host-key policy.
