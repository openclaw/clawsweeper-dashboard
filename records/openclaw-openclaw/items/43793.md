## Summary

Keep open: current main still lacks the central TUI Ctrl+R/manual reconnect guidance, but this branch is not safe to merge as-is because it does not fit the current TUI backend contract and weakens macOS SSH host-key verification.

## What This Changes

The branch adds gateway reconnect plumbing, TUI disconnect guidance and Ctrl+R retry handling, macOS remote-gateway probe UI, and a diffs cleanup test adjustment.

## Best Possible Solution

Rebase and narrow the branch onto current main, model manual reconnect as an optional TUI backend capability or gateway-only path, preserve the shared strict SSH helper, and add the required user-facing changelog entry.

## Reproduction Assessment

Yes. Static inspection against current main reproduces the blockers: `runTui` now uses `TuiBackend` without `reconnect`, while the PR head calls `client.reconnect()`, and the PR head emits `StrictHostKeyChecking=accept-new` where current main uses strict SSH options.

## Solution Assessment

No. The UX direction is reasonable, but the implementation needs to fit the current backend abstraction and must not weaken SSH host-key verification.
