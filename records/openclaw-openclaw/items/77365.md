## What This Changes

The PR changes task terminal delivery to queue ACP completions for group/channel owner sessions, adds a task-registry regression test, and updates task delivery docs.

## Best Possible Solution

Update this PR to classify both documented group and channel owner sessions through the parent completion path while preserving direct delivery for direct chats.

## Reproduction Assessment

Yes, source-level. Current main direct-sends terminal ACP task text when requesterOrigin is deliverable, and the latest PR still leaves documented :group: owner sessions on that same direct-delivery path.

## Solution Assessment

No. Parent-session routing is the right direction, but the current helper only checks channel-shaped sessions; it should use the existing chat-type classification or otherwise cover both group and channel keys with focused tests.
