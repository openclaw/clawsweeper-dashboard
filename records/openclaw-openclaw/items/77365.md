The PR changes terminal task delivery to queue group/channel ACP completions through the requester session, adds a task-registry regression test, and updates task delivery docs.

## Best Possible Solution

Update the PR to classify both documented group and channel owner sessions through the parent completion path, preserve direct delivery for direct chats, add a `:group:` regression, and provide redacted real behavior proof.

## Reproduction Assessment

Yes, source-level. Current main direct-sends terminal ACP task text when `requesterOrigin` is deliverable, and the PR's new helper still leaves documented `:group:` owner sessions on that same direct-delivery path.

## Solution Assessment

No. Parent-session routing is the right fix direction, but the current implementation should use the existing chat-type classification or otherwise cover both `group` and `channel` session keys before merge.
