## Summary

Keep open: this PR targets the valid paired bug in #77251, but the current diff still misses documented `:group:` session keys and adds unrelated changelog bullets, so it should not merge as-is.

## What This Changes

The PR changes ACP task completion routing for channel-shaped owner sessions, adds a task-registry regression test, updates task delivery docs, and appends changelog entries.

## Best Possible Solution

Classify owner sessions using the documented group/channel semantics, cover both shapes in focused tests, and keep docs and changelog scoped to ACP task completion routing.

## Reproduction Assessment

Yes. Source inspection shows current main direct-sends terminal ACP task updates when `requesterOrigin` is deliverable, and the PR's helper still leaves documented `agent:<agentId>:<channel>:group:<id>` owner sessions on that path.

## Solution Assessment

No. Parent-session routing is the right direction, but the implementation should classify both group and channel owner sessions and add a true `:group:` regression case before merge.
