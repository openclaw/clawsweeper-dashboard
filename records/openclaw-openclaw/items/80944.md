## Summary

Keep open: the PR is a useful Control UI feature with sufficient visual proof, but the patch still misses the session-switch case that the linked request explicitly needs, so a tab can keep showing the previous agent until the async identity refresh completes or fails.

## What This Changes

The PR adds dynamic Control UI browser-tab titles based on the active assistant identity, an Appearance toggle to restore the static title, persisted UI settings support, tests, i18n baseline updates, and a changelog entry.

## Best Possible Solution

Keep the dynamic-title feature, but make session changes an immediate title input by deriving a fallback from the active `sessionKey` and preserving the static-title opt-out behavior.

## Reproduction Assessment

Yes for the review finding: source inspection shows `switchChatSession()` mutates `sessionKey` immediately and starts `loadAssistantIdentity()` asynchronously, while the PR title sync does not observe `sessionKey`. The feature request itself is not a bug report against current main.

## Solution Assessment

No, not quite. The direction is maintainable, but the implementation should include session key changes in the title derivation so the tab does not temporarily or permanently advertise the previous agent.
