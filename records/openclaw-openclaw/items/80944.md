## Summary

Keep open: the PR addresses a real Control UI tab-disambiguation gap and has useful screenshot proof, but source review found a blocking fallback bug for unnamed agents because current identity normalization supplies the placeholder name `Assistant`, so the promised agent-id fallback will not disambiguate those tabs.

## What This Changes

The PR adds dynamic Control UI `document.title` updates from the active agent identity, a persisted Quick Settings opt-out, focused tests, i18n baseline entries, and a changelog note.

## Best Possible Solution

Fix the fallback to prefer a real agent id over the default `Assistant` placeholder, keep the opt-out, and then land only if maintainers accept the dynamic-title default and future static-title precedence plan.

## Reproduction Assessment

Not applicable for the original feature request as a bug repro, but the review finding is source-reproducible: current identity normalization can provide `assistantName: "Assistant"` plus a real `assistantAgentId`, and the PR helper chooses the placeholder first.

## Solution Assessment

No, not yet: the implementation is narrow, but it must honor the documented agent-id fallback before it solves the multi-agent tab-disambiguation case for unnamed agents.
