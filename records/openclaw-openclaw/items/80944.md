## Summary

Keep open: current main still has only the static Control UI title, and this recent PR is the active implementation for the linked feature. The branch is close, but source review found a blocking fallback issue where the real app's default `Assistant` placeholder bypasses the intended agent-id/Control fallback for unnamed or not-yet-loaded identities.

## What This Changes

This PR adds dynamic Control UI `document.title` syncing from the active assistant identity, a persisted Quick Settings toggle, storage/render wiring, tests, i18n baseline updates, and a changelog entry.

## Best Possible Solution

Fix the placeholder-name fallback, then land only after maintainers accept the dynamic-title default and the deferred static-title precedence plan.

## Reproduction Assessment

Yes for the review finding by source inspection: the app default normalizes an absent assistant name to `Assistant`, so the PR helper's documented agent-id/Control fallback is bypassed. The linked request itself is a feature, not a broken-current-behavior bug.

## Solution Assessment

No: the lifecycle hook is a reasonable narrow location, but it must distinguish real names from the default placeholder before using `assistantName` as the title source. Static configured-title precedence also still needs maintainer UX agreement.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Use the real agent id fallback when the name is only a placeholder:** `ui/src/ui/app-lifecycle.ts:166-168`
  - body: `assistantName` is not empty in the real host before identity loads: `OpenClawApp` initializes it through `normalizeAssistantIdentity({})`, which returns `Assistant`, and normalized identity responses without a name do the same. Because this helper prefers any non-empty name, unnamed agents and first paint before bootstrap become `Assistant · OpenClaw` instead of using the agent id or `Control`, leaving default multi-agent tabs indistinguishable. Treat the placeholder as unknown or carry an explicit loaded/name source before writing the title.
  - confidence: 0.86
