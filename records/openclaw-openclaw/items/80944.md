## Summary

Keep open: this is a useful Control UI feature PR, but it is not merge-ready. The PR is missing after-fix real browser proof, CI lint fails on the new TODO comment, and the fallback logic does not actually reach the agent id for unnamed agents because current identity normalization fills missing names with `Assistant`.

## What This Changes

The PR changes the Control UI HTML title and adds a lifecycle helper plus tests to set `document.title` from the active assistant name or agent id.

## Best Possible Solution

Implement title sync through a tested resolver that treats placeholder names differently from real agent names, falls back to the active agent id, and composes with any accepted static title config without lint-blocking TODO comments.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature PR. Source and CI inspection do give high-confidence patch evidence for the current blockers: the TODO fails lint, and current identity defaults prevent the intended agent-id fallback.

## Solution Assessment

No. The UI lifecycle hook is a plausible place for the feature, but the implementation needs the fallback and lint blockers fixed and should include real browser proof before merge.
