## Best Possible Solution

Keep the PR open, make the shared hint source-aware so CLI and env override failures get non-conflicting guidance, then require redacted real CLI/TUI proof before merge.

## Reproduction Assessment

Yes by source inspection: a CLI URL override without explicit auth, or `OPENCLAW_GATEWAY_URL` without env token/password, reaches `ensureExplicitGatewayAuth` before opening the gateway request. I did not execute the path because this review is read-only.

## Solution Assessment

No, not as currently written: `ensureExplicitGatewayAuth` is the right shared layer, but the shared `callGatewayWithScopes` hint must not tell env override users to use `--url`.
