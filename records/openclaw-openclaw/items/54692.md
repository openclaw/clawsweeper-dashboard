## What This Changes

## Best Possible Solution

Land a rebased minimal copy/test patch that preserves the fail-closed auth guard, keeps CLI and env remediation source-specific, and includes redacted after-fix CLI/TUI output proof.

## Reproduction Assessment

Yes. Source inspection on current main shows the generic shared auth error and focused tests still assert generic wording; a focused check is a CLI URL override without explicit auth and an OPENCLAW_GATEWAY_URL override without env credentials.

## Solution Assessment

Mostly yes. The shared auth guard is the narrow maintainable place for the added hint, but the caller-provided hint should stay neutral or branch by urlOverrideSource so env users do not first see CLI-only --url guidance.
