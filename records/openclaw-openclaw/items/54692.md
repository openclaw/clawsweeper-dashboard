## Summary

Keep this PR open. Current main still rejects CLI and env URL overrides with only generic auth text, while docs confirm the fail-closed contract; the PR remains a focused messaging and test improvement, with one non-blocking wording issue where the shared caller hint is CLI-biased for env override failures.

## What This Changes

The PR adds source-specific gateway URL override auth follow-up hints in `src/gateway/call.ts` and updates gateway and TUI tests to assert the new CLI and env guidance.

## Best Possible Solution

Merge a rebased patch that keeps gateway URL overrides fail-closed, adds source-specific CLI and env follow-up text at the shared auth guard, and preserves targeted gateway/TUI regression coverage without changing credential precedence or fallback behavior.

## Reproduction Assessment

Yes. The messaging gap can be reproduced through the existing current-main URL override tests in `src/gateway/call.test.ts` and `src/tui/gateway-chat.test.ts`, which cover CLI and env override failures but still assert only the generic `explicit credentials` error.

## Solution Assessment

Mostly yes. The shared `ensureExplicitGatewayAuth` guard is the narrow maintainable place for source-specific follow-up text, but the caller hint added by this PR should remain neutral or branch by source so env override users are not pointed first at a CLI-only `--url` fix.
