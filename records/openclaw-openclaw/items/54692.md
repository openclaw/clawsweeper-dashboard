## Summary

Keep open: current main still emits generic gateway URL override auth guidance, and this PR remains a useful narrow UX cleanup, but it should keep env guidance source-aware and add the required changelog entry before landing.

## What This Changes

The PR adds CLI- and environment-specific follow-up hints when gateway URL overrides are rejected without explicit credentials and updates gateway/TUI tests to assert those hints.

## Best Possible Solution

Land a rebased patch that preserves the fail-closed auth guard, adds source-specific CLI/env follow-up text at the shared auth check, updates focused regression coverage, and records the user-facing fix in `CHANGELOG.md`.

## Reproduction Assessment

Yes. Source inspection on current main shows the shared gateway auth check and gateway/TUI tests still use generic `explicit credentials` wording; the failing behavior is the absence of the PR's source-specific recovery hints.

## Solution Assessment

Mostly yes. The shared auth check is the narrow maintainable place for the improved hint, but the caller hint should remain neutral or branch by `urlOverrideSource` so env override users are not first pointed at CLI-only `--url` guidance.
