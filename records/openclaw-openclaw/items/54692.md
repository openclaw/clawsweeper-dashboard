## What This Changes

## Best Possible Solution

Land a rebased minimal copy/test patch that preserves the fail-closed auth guard, keeps CLI and env remediation source-specific, and includes redacted after-fix CLI/TUI output proof.

## Reproduction Assessment

Yes. Source inspection on current main shows CLI URL overrides and OPENCLAW_GATEWAY_URL overrides without accepted explicit auth still flow through ensureExplicitGatewayAuth and current tests assert only the generic error; I did not run live commands in this read-only review.

## Solution Assessment

Mostly yes. The shared auth guard is the narrow maintainable place for source-specific hints, but the caller-provided generic hint should stay neutral or branch by urlOverrideSource so env users do not receive CLI-only --url guidance first.
