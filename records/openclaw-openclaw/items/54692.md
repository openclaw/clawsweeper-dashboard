## Summary

## What This Changes

The PR adds source-specific gateway URL override auth error hints and updates gateway/TUI tests to assert the new CLI and env remediation text.

## Best Possible Solution

Land a rebased minimal copy/test patch that preserves the fail-closed auth guard and keeps CLI and env remediation source-specific.

## Reproduction Assessment

Yes. Source inspection on current main shows CLI and env URL overrides without accepted explicit auth still flow through ensureExplicitGatewayAuth, and the current tests still assert only the generic error text.

## Solution Assessment

Mostly yes. The shared auth guard is the narrow maintainable place for source-specific hints, but the caller-provided generic hint should stay neutral or branch by urlOverrideSource so env users do not receive CLI-only --url guidance first.
