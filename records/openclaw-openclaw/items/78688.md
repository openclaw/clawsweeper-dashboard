## Summary

Keep open: the consolidation direction is useful and the PR now has real terminal proof, but the current patch removes all `openclaw gateway install --force` guidance for the reported source-checkout plus entrypoint-mismatch path when repair is declined or noninteractive; the expected outcome is one consolidated panel with one actionable hint, not zero hints.

## What This Changes

The PR defers the source-checkout Gateway service config warning, folds service audit findings into one note, suppresses duplicate force-hint output, and adds a regression test for source-checkout entrypoint drift with audit findings.

## Best Possible Solution

Land one implementation that emits exactly one `Gateway service config` panel for source-checkout plus audit-mismatch cases, with the mismatch reported once and a single actionable reinstall/force hint preserved inside that same panel when repair is skipped.

## Reproduction Assessment

Yes. Source inspection plus the paired report gives a high-confidence path: a source-checkout entrypoint, an entrypoint-mismatch audit issue, and a declined or noninteractive repair reach the duplicated current-main notes and the PR's no-hint branch.

## Solution Assessment

No. Consolidating the notes is the right direction, but gating the declined-repair hint on `sourceCheckoutWarning === null` removes the only force guidance in the reported path instead of keeping one hint in the consolidated note.
