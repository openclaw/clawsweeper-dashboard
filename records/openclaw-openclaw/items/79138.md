## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Close relay sessions after appendAudio failures:** `ui/src/ui/chat/realtime-talk-gateway-relay.ts:145`
  - body: When `talk.session.appendAudio` rejects, this path calls `stopLocal()`, which marks the transport closed without sending `talk.session.close`. A transient RPC failure can leave the gateway relay session running until TTL because later `stop()` sees `closed` and skips the close RPC; issue the close best-effort before or during this shutdown.
  - confidence: 0.86
- **[P3] Build launch options from explicit fields:** `src/gateway/server-methods/talk-shared.ts:239-242`
  - body: This returns the full defaults object when no override exists, and callers pass `buildTalkRealtimeConfig()`, which includes `provider`, `providers`, `transport`, and `brain`. `talk.client.create` then spreads that object into provider `createBrowserSession`, so providers can receive non-launch config fields outside the request contract; return a fresh object containing only supported launch options.
  - confidence: 0.78
