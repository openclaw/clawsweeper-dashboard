Keep open. Current main still has the gateway session-action metadata crash surface, and this draft is not merge-ready because its method-scope helper still reads plugin-owned `requiredScopes` arrays unsafely before dispatch can return the new fail-closed response.

## What This Changes

The PR adds guarded plugin session-action registry traversal in gateway method-scope authorization and dispatch, plus contract tests for unreadable metadata and handler receiver behavior.

## Best Possible Solution

Land one canonical gateway session-action lookup helper that guards plugin-owned registration and `requiredScopes` reads for both authorization and dispatch, then rebase or close the overlapping drafts.

## Reproduction Assessment

Yes. Source inspection shows current main and the PR auth path read plugin-owned session-action metadata before dispatch, and a throwing Array proxy can still escape through the PR helper; I did not run repo tests because this review is read-only.

## Solution Assessment

No. The dispatch-side guard is a plausible mitigation, but the best fix is a single shared guarded lookup/scope reader used by both method-scope authorization and dispatch, with Array-proxy regressions.
