## Best Possible Solution

Land one canonical guarded session-action registration and scope reader shared by authorization and dispatch, with array-proxy regressions, then rebase or close the overlapping drafts.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: use a plugin-owned Array proxy for `requiredScopes` that throws on `length` or `filter`, then call `plugins.sessionAction` through authorization. A Node probe confirmed that such a proxy passes `Array.isArray` while those reads throw.

## Solution Assessment

No. The dispatch-side guard is useful, but the best fix is a single guarded reader used by both method-scope authorization and dispatch, plus regressions for throwing array metadata.
