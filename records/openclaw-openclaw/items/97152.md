## Summary

Keep open: current main still has trusted policy and prompt-based plugin/native approval flows, but not the requested capability-scoped, exclusive, proof-bound resolver across Codex, ACP, native tools, MCP, messages, and delegated work.

## Best Possible Solution

Keep this as a maintainer-owned security/API design item, settle capability taxonomy, resolver authority, proof/audit binding, identity context, and fail-closed semantics, then implement a focused core seam while leaving vendor providers in plugins.
