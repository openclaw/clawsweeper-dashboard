## Summary

Keep open: current main and v2026.6.10 still have trusted-policy hooks plus ordinary exec/plugin/Codex approval paths, not the requested capability-level, exclusive, proof-bound resolver across harnesses. The AgentPass prototype and the AgentKit PR are useful validation points, but the remaining work is a security-sensitive API/product design.

## What This Changes

Requests a new core approval-resolver API that is capability-scoped, exclusive, proof-bound, and shared across Codex, ACP, native tools, MCP, messages, and sub-agents.

## Best Possible Solution

Keep this as the canonical maintainer-owned security/API design item, use AgentPass and AgentKit as validation inputs, then land a focused resolver seam only after capability taxonomy, exclusivity, proof binding, and failure semantics are agreed.

## Reproduction Assessment

## Solution Assessment

Unclear pending maintainer design. The proposed seam targets the right core/plugin boundary for a capability that plugins cannot own alone, but the capability taxonomy, resolver authority, proof record, and fail-closed semantics need security/product agreement first.
