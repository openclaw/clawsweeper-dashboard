## Summary

Keep open: current main still has prompt-style exec/plugin approval paths and Codex permission relays, not the requested capability-scoped, exclusive, proof-bound approval resolver across harnesses.

## What This Changes

Request a core approval-resolver API that can exclusively gate side-effecting tool capabilities across harnesses and bind decisions to exact requests with verifier evidence.

## Best Possible Solution

Keep this as a maintainer-owned security/API design item until capability taxonomy, resolver authority, proof binding, and cross-harness failure semantics are agreed, then land a focused implementation with live upgrade proof.

## Solution Assessment

Unclear pending maintainer design. The proposed seam targets a real gap, but capability taxonomy, resolver authority, proof recording, and fail-closed behavior need security/product agreement before implementation.
