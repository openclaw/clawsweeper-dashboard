## Summary

Keep open: current main and v2026.6.10 include the structured doctor health-check foundation and `plugin-sdk/health`, but `doctor` still avoids generic plugin loading and only bridges bundled Policy checks, so the requested upgrade-safe external doctor extension path remains unfinished.

## What This Changes

Request to expose an upgrade-safe doctor extension API so plugins or custom checks can contribute diagnostics and optional repairs without editing OpenClaw core.

## Best Possible Solution

Define a narrow, security-reviewed external plugin doctor registration contract with explicit activation, source attribution, selection, and repair-consent gates, using the current structured health-check contract as the base.
