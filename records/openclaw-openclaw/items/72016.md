## Summary

Keep open. Current main has the structured health-check foundation and a bundled policy-plugin example, but it still does not provide an end-to-end, upgrade-safe path for arbitrary non-bundled plugins to register doctor checks and repairs; the remaining work is a plugin loading and repair-trust policy decision.

## What This Changes

Request an upgrade-safe extension point for custom `openclaw doctor` checks and optional fixes, especially for environment-specific multi-gateway validation.

## Best Possible Solution

Define an additive doctor health extension contract for opted-in plugin checks with source attribution, read-only lint behavior, gated repair consent, docs, and focused contract tests without making `doctor` eagerly load every plugin by default.

## Reproduction Assessment

Not applicable as a runtime bug, but yes for the source-level feature gap. Current main does not load general plugin runtime for `openclaw doctor` and only registers core plus bundled policy checks.

## Solution Assessment

Unclear until maintainers choose the plugin loading and repair trust policy. The structured registry is the right base, but the final API should make attribution, opt-in loading, and repair consent explicit.
