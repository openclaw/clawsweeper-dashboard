## Summary

Keep open. Current main has structured HealthCheck/repair plumbing and a bundled Policy example, but `openclaw doctor` still does not provide an upgrade-safe path for arbitrary non-bundled plugins to register checks and repairs; the remaining work is an API, plugin-loading, and repair-trust policy decision.

## What This Changes

Request to let plugins add upgrade-safe custom `openclaw doctor` checks and optional repairs without patching OpenClaw core source.

## Best Possible Solution

Define an additive doctor health extension contract for opted-in plugin checks with source attribution, read-only lint behavior, gated repair consent, docs, and focused contract tests without making `doctor` eagerly load every plugin by default.

## Reproduction Assessment

Not applicable as a runtime bug, but yes for the source-level feature gap. Current main keeps `openclaw doctor` on `loadPlugins: "never"` and registers only core plus bundled health checks.

## Solution Assessment

Unclear until maintainers choose the plugin-loading and repair trust policy. The existing HealthCheck registry is the right base, but the final API should make attribution, opt-in loading, and repair consent explicit.
