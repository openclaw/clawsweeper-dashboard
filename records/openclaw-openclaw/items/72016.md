## Summary

Keep open. Current main and the latest release have a structured health-check registry plus a narrow bundled Policy plugin bridge, but they still do not provide an upgrade-safe path for arbitrary non-bundled plugins to contribute doctor checks and repair-capable fixes.

## What This Changes

The issue requests a public doctor extensibility surface so custom plugin or local checks and fixes can survive OpenClaw npm upgrades.

## Best Possible Solution

Define an additive, opt-in doctor health extension contract for plugin checks with source attribution, bounded loading, explicit repair consent or unattended-mode policy, public docs, and focused contract tests.

## Reproduction Assessment

Yes for the source-level feature gap, though this is not a runtime bug. Current main keeps doctor from loading general plugins and only registers core plus narrow bundled Policy health checks.

## Solution Assessment

Unclear until maintainers choose the plugin loading and repair trust policy. The structured health registry is the right base, but the public API still needs attribution, opt-in loading, and repair consent semantics.
