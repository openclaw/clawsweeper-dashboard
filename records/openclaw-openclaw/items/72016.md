## Summary

Keep open. Current main still lacks an upgrade-safe public doctor-check registrar or plugin-owned fix contract; the issue discussion correctly identifies a real feature gap and unresolved API/trust-policy questions.

## What This Changes

Request an upgrade-safe plugin/SDK surface for custom `openclaw doctor` checks and fixes for environment-specific diagnostics.

## Best Possible Solution

Design an additive public doctor-check SDK contract with plugin attribution, deterministic ordering, duplicate handling, focused docs/tests, and separately gated repair support.

## Reproduction Assessment

Not applicable as a runtime bug. Source inspection gives a high-confidence feature gap: current main has a static core doctor contribution list and no public doctor-check registrar.

## Solution Assessment

Unclear until maintainers settle the API and trust policy. A narrow additive SDK seam is the maintainable direction; local source edits or ad hoc plugin side commands would not provide the integrated upgrade-safe workflow requested.
