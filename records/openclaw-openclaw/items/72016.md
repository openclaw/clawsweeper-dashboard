## Summary

Keep open. Current main still has static, core-owned doctor health contributions plus limited plugin doctor compatibility hooks, but no public plugin-SDK API for registering custom doctor checks or governing plugin-owned fixes; the discussion has narrowed this to a real SDK/trust-policy design item rather than a stale or ClawHub-only request.

## What This Changes

The issue requests an upgrade-safe plugin or extension API for adding custom `openclaw doctor` checks and optional fixes.

## Best Possible Solution

Treat this as a public plugin-SDK design item: add a narrow additive doctor contribution contract with plugin attribution, deterministic ordering, duplicate handling, focused contract tests, and docs; start with read-only diagnostics or clearly gated repair support rather than local source edits.

## Reproduction Assessment

Not applicable as a runtime reproduction because this is an API/feature request. The verification path is high-confidence source inspection: current main has no public doctor-check registration API or equivalent plugin contribution registry for the main doctor health flow.

## Solution Assessment

Unclear until maintainers settle the API and trust policy. A narrow additive SDK seam appears maintainable; plugin-owned repair should be opt-in and consent-gated rather than hidden behind the existing generic `--fix` behavior.
