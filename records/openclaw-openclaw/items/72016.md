## Summary

Keep open. Current main still lacks a public doctor-check registration API: the main health pass is a static core-only contribution list, while the existing plugin doctor surfaces cover compatibility/config repair rather than custom environment checks.

## What This Changes

Request an upgrade-safe public doctor extensibility API so plugins or local custom code can add environment-specific checks and optional repairs.

## Best Possible Solution

Design an additive public doctor-check SDK contract with plugin attribution, deterministic ordering, duplicate handling, tests, docs, and separately gated repair support with explicit consent semantics.

## Reproduction Assessment

Not applicable as a runtime reproduction because this is an API feature request. The feature gap is high-confidence from source inspection: current main has no public doctor-check registrar and runs a static core contribution list.

## Solution Assessment

Unclear until maintainers settle the public API and trust policy. A narrow additive SDK seam is the maintainable direction; local source edits or sidecar scripts would not provide the integrated upgrade-safe doctor workflow requested.
