## Summary

Keep open. Current main still lacks an upgrade-safe public doctor-check registrar and plugin-owned fix flow; the issue discussion matches the remaining SDK and trust-policy gap, and the adjacent draft at https://github.com/openclaw/openclaw/pull/80055 is not merged and explicitly treats its health contract as an internal/structured-diagnostics step rather than the requested public repair API.

## What This Changes

The issue asks for an upgrade-safe doctor extensibility API so custom environment checks and optional fixes can be registered without editing OpenClaw source.

## Best Possible Solution

Design an additive public doctor-check SDK contract with plugin attribution, deterministic ordering, duplicate handling, focused docs/tests, and separately gated repair support.

## Reproduction Assessment

Not applicable as a runtime reproduction because this is an API feature request. The feature gap is high-confidence from source inspection: current main has no public doctor-check registrar and runs a static core contribution list.

## Solution Assessment

Unclear until maintainers settle the public API and trust policy. A narrow additive SDK seam is the maintainable direction; local source edits or plugin-owned side commands would not provide the integrated upgrade-safe doctor workflow requested.
