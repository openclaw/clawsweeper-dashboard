## Summary

Keep open. Current main still lacks an upgrade-safe public doctor-check registrar or plugin-owned fix contract; the issue is a real feature gap with unresolved API and trust-policy questions, not a cleanup-close candidate.

## What This Changes

Request an upgrade-safe doctor extension API so plugins or local checks can add custom diagnostics and optional fixes without being overwritten by npm upgrades.

## Best Possible Solution

Design a maintainer-approved additive doctor or health SDK contract with plugin attribution and read-only checks first, then gate plugin-owned repairs behind explicit consent, audit logging, docs, and focused tests.

## Reproduction Assessment

Not applicable as a runtime bug. Source inspection gives high confidence that current main has only static core doctor contributions and no public registrar for upgrade-safe custom doctor checks or fixes.

## Solution Assessment

Unclear until maintainers choose the API and trust policy. A narrow additive SDK seam is the maintainable direction; local source edits or ad hoc plugin side commands would not provide the integrated upgrade-safe workflow requested.
