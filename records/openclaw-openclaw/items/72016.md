## Summary

Keep open. Current main still lacks a public doctor-check registration API for custom environment diagnostics and fixes; the existing plugin doctor contract only covers legacy config rules/compat migrations, and the discussion correctly identifies unresolved SDK and repair-trust decisions.

## What This Changes

The issue asks for an upgrade-safe doctor extensibility API so plugins or local checks can add environment-specific diagnostics and optional `--fix` repairs.

## Best Possible Solution

Design a narrow additive public doctor-check SDK contract with plugin attribution, deterministic ordering, duplicate handling, focused tests, docs, and separately gated repair support.

## Reproduction Assessment

Not applicable as a runtime reproduction because this is an API feature request. The feature gap is high-confidence by source inspection: current main has no public doctor-check registrar and runs a static core contribution list.

## Solution Assessment

Unclear until maintainers settle the public API and trust policy. A narrow SDK seam is likely maintainable; local source edits or separate plugin CLI commands would not provide the integrated upgrade-safe doctor workflow requested.
