## Summary

Keep open. Current main has a structured health-check contract and a bundled policy-plugin example, but it still does not provide a general, upgrade-safe external plugin doctor-check activation path or a settled trust policy for plugin-owned repairs.

## What This Changes

Request a supported plugin/SDK extension point for custom OpenClaw doctor checks and optional repairs that survive npm upgrades.

## Best Possible Solution

Design an additive public doctor health registration and activation contract for external plugins, with read-only checks first and plugin repairs gated by explicit consent, source attribution, audit output, and focused tests.

## Reproduction Assessment

Not applicable as a runtime bug. Source inspection gives high confidence that current main only partially implements the requested doctor extensibility and still lacks a general external plugin activation path for custom checks.

## Solution Assessment

Unclear until maintainers choose the API and trust policy. The maintainable path is a narrow SDK/registry seam with explicit repair consent, not local source edits or a policy-specific bundled-loader special case.
