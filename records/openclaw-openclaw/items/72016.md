## Summary

Keep open. Current main and the latest release have the structured doctor health registry and a narrow bundled Policy plugin bridge, but they still do not provide an upgrade-safe public path for arbitrary non-bundled plugins to contribute doctor checks or repair-capable fixes; the remaining work needs maintainer product and security direction.

## What This Changes

Request a public, plugin-safe doctor extension API so custom diagnostic checks and optional fixes can survive OpenClaw upgrades.

## Best Possible Solution

Define an additive, opt-in doctor health extension contract with source attribution, bounded plugin loading, explicit repair consent/unattended policy, docs, and focused tests.

## Reproduction Assessment

Yes for the source-level feature gap, not for a runtime bug. Current main and v2026.6.8 show doctor registering core plus narrow bundled Policy health checks without an arbitrary installed-plugin doctor extension path.

## Solution Assessment

Unclear until maintainers choose the product and security model. The existing structured health registry is the right base, but the public API, loading boundary, and repair consent policy need an explicit decision before implementation.
