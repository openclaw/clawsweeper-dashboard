## Summary

Keep open. Current main still lacks an upgrade-safe public doctor-check registrar or plugin-owned fix contract; the discussion identifies a real plugin API gap with unresolved security and product-policy questions rather than a cleanup-close candidate.

## What This Changes

The issue requests an upgrade-safe doctor extensibility API so custom plugins or local checks can add diagnostics and optional fixes to `openclaw doctor`.

## Best Possible Solution

Design a maintainer-approved additive doctor or health SDK contract with plugin attribution and read-only checks first, then add plugin-owned repairs only behind explicit consent, audit logging, docs, and focused tests.

## Reproduction Assessment

Not applicable as a runtime bug. Source inspection on current main gives high confidence that doctor checks are still core-only and there is no public registrar or fix contract for upgrade-safe custom checks.

## Solution Assessment

Unclear until maintainers choose the API and trust policy. A narrow additive SDK or registry seam is the maintainable direction; local source edits or ad hoc plugin side commands would not provide the integrated upgrade-safe workflow requested.
