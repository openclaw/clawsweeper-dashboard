## Summary

Keep open. Current main has a structured health-check SDK and a bundled policy example, but ordinary `openclaw doctor` still does not discover arbitrary external plugin checks or settle the repair trust policy requested here.

## What This Changes

Request a plugin-extensible `openclaw doctor` check and repair API so custom environment checks survive upgrades and can participate in `--fix`.

## Best Possible Solution

Add a maintainer-approved doctor health extension path that core doctor explicitly discovers for enabled plugins, with source attribution, selection/listing, repair consent, logging, and regression tests.

## Reproduction Assessment

Not applicable as a runtime bug. Source inspection gives high confidence that only a partial health SDK exists and arbitrary enabled plugin checks are not wired into the built-in doctor path.

## Solution Assessment

Unclear until maintainers decide the API and trust policy. The maintainable direction is an explicit doctor/health extension contract, not local source edits or incidental plugin CLI side effects.
