## Summary

Keep open. Current main and v2026.6.6 have the structured health-check registry, repair hooks, and a bundled Policy plugin example, but they still do not provide an end-to-end, upgrade-safe path for arbitrary non-bundled plugins to contribute doctor checks and repairs.

## What This Changes

Request an upgrade-safe doctor extension API so custom plugin or local checks can add diagnostics and optional fixes to `openclaw doctor`.

## Best Possible Solution

Define an additive doctor health extension contract for opted-in plugin checks with source attribution, read-only lint behavior, explicit repair consent or unattended-mode policy, public docs, and focused contract tests without eagerly loading every plugin by default.

## Reproduction Assessment

Not applicable as a runtime bug, but yes for the source-level feature gap. Current main keeps `openclaw doctor` from loading general plugins and only registers core plus bundled Policy health checks.

## Solution Assessment

Unclear until maintainers choose the plugin loading and repair trust policy. The structured health registry is the right base, but the public API still needs attribution, opt-in loading, and repair consent semantics.
