## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve cadence after the final retry fails:** `src/cron/service/timer.ts:498`
  - body: When no further retry is scheduled, this branch can choose the generic error-backoff floor over the preserved natural slot. For short recurring schedules, the last failed retry can skip intended cadence slots, contradicting the no-drift behavior promised by this PR.
  - confidence: 0.9
- **[P2] Recompute failed jobs when retry settings change:** `src/cron/service/jobs.ts:596-600`
  - body: applyJobPatch() stores retryCount and retryDelayMs, but retry-only updates do not recompute nextRunAtMs. Editing a failed recurring job to add retry settings can leave the existing later natural run in place, so the new policy does not affect the active failure cycle.
  - confidence: 0.86
- **[P2] Reject incomplete retry policies:** `src/gateway/protocol/schema/cron.ts:293-294`
  - body: The schema accepts retryCount and retryDelayMs independently, while the runtime only retries when both are positive. API callers can create or update a job that validates successfully but never retries, so the pair should be all-or-nothing or have documented defaults.
  - confidence: 0.88
- **[P2] Reject retry flags for one-shot cron add:** `src/cli/cron-cli/register.cron-add.ts:274-275`
  - body: cron add forwards retryCount and retryDelayMs for every schedule kind, but the scheduler only reads these fields in the recurring error branch. A one-shot job can appear to accept --retry-* while runtime behavior remains governed by the existing global one-shot retry config.
  - confidence: 0.82
- **[P2] Reject retry edits on one-shot cron jobs:** `src/cli/cron-cli/register.cron-edit.ts:140-153`
  - body: cron edit accepts --retry-* and stores the fields without checking the existing job schedule, but one-shot error handling never reads the per-job retry policy. The edit flow should reject these flags for one-shot jobs or implement matching semantics.
  - confidence: 0.8
