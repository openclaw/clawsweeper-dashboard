Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Rebase protocol constants onto v4:** `src/gateway/protocol/version.ts:1`
  - body: This branch adds `PROTOCOL_VERSION = 3`, but current main and the generated native clients already advertise protocol/min protocol 4. Merging this stale constant would make Gateway/CLI/native clients negotiate the wrong protocol level and break compatibility with current protocol checks.
  - confidence: 0.93
- **[P1] Return to preserved cadence after retry exhaustion:** `src/cron/service/timer.ts:498`
  - body: When the last retry fails, this falls back to `Math.max(effectiveNaturalNextRunAtMs, fallbackNextRunAtMs)`. For a short recurring job, the fallback backoff can be later than the preserved natural slot, so the job skips scheduled occurrences instead of preserving cadence after retry exhaustion.
  - confidence: 0.88
- **[P2] Reject incomplete retry policies:** `src/gateway/protocol/schema/cron.ts:293-294`
  - body: `retryCount` and `retryDelayMs` are accepted independently here, but the timer only retries when both are positive. A caller can successfully create or update a job with only one field and get no retry behavior, so this should be validated as an all-or-nothing pair or given explicit defaults.
  - confidence: 0.9
- **[P2] Reject retry flags on one-shot cron jobs:** `src/cli/cron-cli/register.cron-add.ts:90-91`
  - body: These flags are available for every `cron add`, including `--at` jobs, but the one-shot timer branch never reads the per-job retry fields. That lets `openclaw cron add --at ... --retry-*` appear to work while runtime behavior remains unchanged.
  - confidence: 0.82
