## Summary

Keep open: current main partially overlaps by exposing session usage rows with agent, channel, chat type, origin, and model fields, but it does not provide the requested stable cron job/session-kind/channel-target/runtime attribution contract for dashboards.

## What This Changes

Requested enrichment of official usage/status JSON rows with stable agent, session kind, cron job, channel, model, and runtime attribution fields.

## Best Possible Solution

Define an additive, privacy-aware usage attribution contract in the existing gateway/shared usage surfaces, with explicit cron job and channel labels only where the source data is stable, then document it.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection does confirm that current main lacks several requested stable fields, including cron job identity and session kind, on the usage/status JSON surfaces.

## Solution Assessment

Unclear until maintainers choose the public JSON shape. The narrowest maintainable path appears to be extending `sessions.usage` and its shared types/docs rather than creating a parallel dashboard adapter surface.
