## Summary

Keep open: current main partially covers session usage attribution, but it does not expose the requested stable cron identity, session-kind, channel-target-kind, or runtime labels across official status/usage JSON surfaces.

## What This Changes

Request to add stable optional agent, session kind, cron job, channel, model, and runtime attribution fields to usage/status JSON exports.

## Best Possible Solution

Design an additive attribution contract shared by `sessions.usage` and any status JSON usage surface, deriving agent/channel/model from session metadata and cron identity from cron state with privacy-preserving target labels, typed schema updates, docs, and focused tests.

## Reproduction Assessment

Not applicable: this is a feature request rather than a broken existing behavior. Source review confirms current main has partial session attribution but lacks several requested stable JSON fields.

## Solution Assessment

Yes in direction, but not as an automatic bug fix: the maintainable path is an additive, typed usage/status attribution contract with explicit privacy rules for channel target metadata.
