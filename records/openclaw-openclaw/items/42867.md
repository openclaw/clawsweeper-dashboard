## Summary

Keep open. Current main has the heartbeat-aware daily/idle freshness behavior, but it still lacks normal-session max-turn TTL config, schema support, durable counting, and reuse-gate enforcement; the remaining TTL work overlaps the broader open #45390 design rather than being safely closeable.

## What This Changes

Request adds configurable turn-count session rotation and heartbeat-aware reset freshness so maintenance traffic cannot keep sessions alive indefinitely.

## Best Possible Solution

Define one core session TTL contract across this issue and #45390, then add schema-validated, documented max-turn rotation that counts real conversation turns, excludes maintenance/system-event traffic, and can be enforced before session reuse.

## Reproduction Assessment

Yes. Static inspection on current main gives a high-confidence reproduction path: the proposed config key is absent and strict-rejected, the freshness evaluator has no turn-count input, and persisted session metadata has no durable turn counter.

## Solution Assessment

Unclear. The shipped heartbeat freshness implementation is the right narrow solution for the second half, but the remaining max-turn behavior should be unified with #45390 before choosing config location, counter semantics, and rotation persistence.
