## Summary

Keep this open: current main still has the installer/docs UX gap. The Node >=22.14 runtime floor is intentional, but `install.sh` still treats all Darwin hosts as generic macOS and proceeds toward Homebrew `node@24` without a Big Sur/macOS 11 preflight or public install-doc warning.

## What This Changes

The issue asks OpenClaw to support Big Sur-compatible Node, fall back gracefully, or clearly document/fail unsupported macOS hosts before the installer attempts Node/Homebrew work.

## Best Possible Solution

Preserve the Node >=22.14 runtime contract, add a targeted macOS preflight or friendly failure path for unsupported old macOS before Homebrew Node installation, and update public install/Node docs with the Big Sur limitation.

## Reproduction Assessment

Yes for the source-level UX gap: current main shows the Darwin path proceeds to Homebrew Node 24 without checking macOS product version, and install docs do not document Big Sur/macOS 11 limits. Live Big Sur logs would only refine the exact upstream failure wording.

## Solution Assessment

Yes: a narrow installer/docs fix is the best way to solve this issue. Lowering the runtime floor to Node 18 would conflict with the current package engine and runtime guard, while a preflight/docs warning directly addresses the user-visible failure.
