## Summary

Keep open: current main still preserves the Node >=22.16 runtime floor, but the macOS installer and install docs still lack a Big Sur/macOS 11 preflight or clear guidance before the Homebrew Node path.

## What This Changes

The issue requests that OpenClaw handle macOS Big Sur installer failure by supporting a compatible runtime, falling back gracefully, or documenting the unsupported macOS/Node requirement.

## Best Possible Solution

Keep the Node >=22.16 runtime contract, add a macOS version-aware installer preflight or friendly failure before the Homebrew node@24 path, and update public install docs with Big Sur/macOS 11 guidance.

## Reproduction Assessment

Yes for a source-level reproduction: current main maps Darwin to generic macOS, then reaches the Homebrew node@24 installation path without a Big Sur/macOS 11 check, and the install docs omit that limitation. I did not live-run on Big Sur, so this is source_reproducible rather than reproduced.

## Solution Assessment

Yes: a narrow installer/docs fix is the best way to solve the issue while preserving the current Node >=22.16 runtime floor. Adding Node 18 support would conflict with the current package engine and runtime guard.
