## Summary

Keep open: current main still preserves the Node 22.16+ runtime floor, but the macOS installer treats all Darwin hosts as generic macOS, reaches the Homebrew node@24 path, and the install docs still do not warn Big Sur users before that path fails.

## What This Changes

Request clearer installer support, fallback, or documentation for macOS Big Sur systems that cannot use the Homebrew Node version OpenClaw bootstraps.

## Best Possible Solution

Keep the Node 22.16+ runtime contract, add a macOS version-aware installer preflight or friendly failure before the Homebrew Node bootstrap path, and update install docs with Big Sur/macOS 11 guidance.

## Reproduction Assessment

Yes for source-level reproduction: on a Darwin host with no supported Node, current main detects only generic macOS and then attempts Homebrew node@24 without checking the macOS product version. I did not live-run on Big Sur, so this is source_reproducible rather than reproduced.

## Solution Assessment

Yes: a narrow installer/docs fix is the best path while preserving the current Node 22.16+ floor. Adding Node 18 support would conflict with current package metadata and the runtime guard.
