## Summary

Keep open. Current main and v2026.5.20 still default to AGENTS.md section reinjection after compaction, still parse H2/H3 heading strings, and still inject the extracted text into post-compaction model context; resolving the reported bloat/magic-heading concern would intentionally change a documented shipped recovery policy.

## What This Changes

The issue asks OpenClaw to stop or redesign AGENTS.md post-compaction section reinjection because it duplicates retained system-prompt content and makes session behavior depend on user-editable heading strings.

## Best Possible Solution

Choose the supported post-compaction recovery policy explicitly, then align runtime defaults, AGENTS.md parsing, config help/docs, and regression tests so users either get deliberate opt-in reinjection or no duplicate retained prompt content by default.

## Reproduction Assessment

Yes, at source level. With default config and matching H2/H3 AGENTS.md sections, current main and v2026.5.20 extract those sections and inject them after compaction through summary, extraSystemPrompt, and trusted system-event paths.

## Solution Assessment

Unclear until maintainers choose the policy. Removing or making reinjection opt-in would address the bloat and magic-heading concern, but it intentionally changes a documented shipped recovery default; the existing [] opt-out is not the same as settling the default contract.
