## Summary

Keep open. Current main still defaults to AGENTS.md section extraction, appends or enqueues post-compaction refresh content, and appends workspace-critical rules to compaction summaries while AGENTS.md also remains normal Project Context; changing that default would reverse documented recovery behavior and needs maintainer policy judgment.

## What This Changes

Request to remove or rework default post-compaction AGENTS.md section parsing and reinjection because it duplicates retained bootstrap context and makes behavior depend on magic headings.

## Best Possible Solution

Choose a maintainer-approved post-compaction recovery policy, likely making AGENTS.md section reinjection opt-in or skipping it when normal Project Context is retained, then align runtime prompts, summary suffixing, config docs, and focused tests together.

## Reproduction Assessment

Yes, at source level. On current main, a default config plus AGENTS.md with matching H2/H3 sections causes readPostCompactionContext to produce refresh content that is appended or enqueued after compaction while AGENTS.md remains injected as Project Context.

## Solution Assessment

Unclear. Removing default parsing matches the report, but it would reverse intentionally documented recovery behavior, so the maintainable solution depends on a maintainer policy decision rather than an autonomous cleanup patch.
