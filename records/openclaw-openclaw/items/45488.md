## Summary

Keep open. Current main and the latest release still default to AGENTS.md heading-based post-compaction reinjection, and https://github.com/openclaw/openclaw/pull/85960 is an open linked fix path rather than a reason to close this issue before merge.

## What This Changes

The issue reports that the behavior added by https://github.com/openclaw/openclaw/pull/18049 duplicates retained prompt context after compaction and makes session behavior depend on editable AGENTS.md headings.

## Best Possible Solution

Review and either land or explicitly reject https://github.com/openclaw/openclaw/pull/85960, then close this issue only after the selected post-compaction reinjection policy is merged or documented as not planned.

## Reproduction Assessment

Yes, at source level. With default config and matching AGENTS.md H2/H3 sections, current main and v2026.5.22 extract those sections and inject them after compaction through summary, extraSystemPrompt, and trusted system-event paths.

## Solution Assessment

Unclear until maintainers choose the policy. Making reinjection opt-in addresses the bloat and magic-heading concern, but it changes a documented shipped recovery default; the existing [] opt-out is not the same as settling the default contract.
