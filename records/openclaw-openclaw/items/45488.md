## Summary

Keep open. Current main still performs the default AGENTS.md section extraction, summary suffixing, and post-compaction refresh while AGENTS.md remains normal Project Context, so the reported bloat/policy concern is still real but changing the default would reverse intentionally documented recovery behavior and needs maintainer judgment.

## What This Changes

The issue asks OpenClaw to stop default post-compaction AGENTS.md section parsing/reinjection because it duplicates retained system prompt content and makes behavior depend on editable section headings.

## Best Possible Solution

Choose a maintainer-approved post-compaction recovery policy, likely making AGENTS.md section reinjection opt-in or skipping it when normal Project Context is retained, then align the refresh prompt, summary suffix, config docs, and focused tests together.

## Reproduction Assessment

Yes, at source level. On current main, a default config plus an AGENTS.md with matching H2/H3 sections causes readPostCompactionContext to produce a refresh payload that is appended or enqueued after compaction while AGENTS.md also remains injected as Project Context.

## Solution Assessment

Unclear. Removing default parsing matches the report, but it reverses intentionally added and documented recovery behavior, so the narrow maintainable solution depends on a maintainer policy decision rather than an autonomous cleanup patch.
