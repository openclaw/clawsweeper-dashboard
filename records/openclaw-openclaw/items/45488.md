## Summary

Keep open. Current main and the latest release still extract AGENTS.md sections by default and inject them after compaction; the remaining question is the supported recovery policy, because removing or making this opt-in would intentionally change a documented shipped default.

## What This Changes

Report that post-compaction AGENTS.md section reinjection duplicates retained prompt context and makes session behavior depend on configured or legacy section headings.

## Best Possible Solution

Choose a supported post-compaction recovery policy, likely making AGENTS.md reinjection explicit or skipping it when retained context already carries the same content, then align runtime behavior, docs/config help, and focused regression tests.

## Reproduction Assessment

Yes, at source level. With default config and matching H2/H3 AGENTS.md sections, current main extracts those sections and injects them after compaction through summary, extraSystemPrompt, and trusted system-event paths.

## Solution Assessment

Unclear. Removing the parser would address the bloat and magic-heading concern, but it would intentionally change a documented shipped default and opt-out model, so maintainers should decide the supported recovery policy first.
