## Summary

Keep open. Current main and v2026.5.20 still default to AGENTS.md heading-based post-compaction reinjection through the summary, system-event, and extraSystemPrompt paths, and changing that behavior is a shipped recovery-policy decision rather than conservative cleanup.

## What This Changes

The issue asks OpenClaw to stop default post-compaction AGENTS.md section parsing/reinjection because it duplicates retained prompt content and makes session behavior depend on editable heading strings.

## Best Possible Solution

Choose the supported post-compaction recovery policy explicitly, then align runtime defaults, AGENTS.md parsing, config help/docs, and regression tests so users either get deliberate opt-in reinjection or no duplicate retained prompt content by default.

## Reproduction Assessment

Yes, at source level. With default config and matching AGENTS.md H2/H3 sections, current main and v2026.5.20 extract those sections and inject them after compaction through summary, extraSystemPrompt, and trusted system-event paths.

## Solution Assessment

Unclear until maintainers choose the policy. Disabling or making reinjection opt-in would address the bloat and magic-heading concern, but it intentionally changes a documented shipped recovery default; the existing [] opt-out is not the same as settling the default contract.
