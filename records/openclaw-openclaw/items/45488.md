## Summary

Keep open. Current main and the latest release still perform default heading-based AGENTS.md post-compaction reinjection, and the open linked PR that closes this issue is unmerged and currently conflicting, so the issue remains the canonical tracker.

## What This Changes

The issue reports that post-compaction AGENTS.md section reinjection duplicates already-retained system prompt content and makes session behavior depend on user-editable markdown headings.

## Best Possible Solution

Review and complete https://github.com/openclaw/openclaw/pull/85960, or close it with an explicit maintainer policy decision; the final fix should apply one post-compaction reinjection contract consistently across refresh prompt, extraSystemPrompt, summary suffix, docs, config help, and regression tests.

## Reproduction Assessment

Yes, at source level. With unset config and matching AGENTS.md H2/H3 sections, current main extracts those sections and injects them through post-compaction refresh/system-event paths and the compaction summary suffix.

## Solution Assessment

Unclear until maintainers settle the default and upgrade policy. Making reinjection opt-in addresses the bloat and magic-heading concern, but it changes shipped behavior and needs consistent runtime, docs, config, and test coverage.
