## Summary

Keep open. Current main still has the default AGENTS.md section extraction, summary append, and system-event refresh paths active while AGENTS.md remains injected as Project Context; because that behavior was intentionally introduced for post-compaction recovery and is now documented/configurable, changing the default needs maintainer policy instead of conservative cleanup.

## What This Changes

The issue asks OpenClaw to stop default post-compaction AGENTS.md section parsing/reinjection because AGENTS.md already remains in context and the parsing is workspace-dependent.

## Best Possible Solution

Choose a maintainer-approved post-compaction recovery policy, likely making AGENTS.md section reinjection opt-in or skipping it when normal Project Context is retained, then update the refresh prompt, summary suffix, config docs, and focused tests together.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: on current main, a default config plus AGENTS.md with matching sections causes readPostCompactionContext to produce a refresh payload that is appended/enqueued after compaction while AGENTS.md remains injected as Project Context.

## Solution Assessment

Unclear. Removing the default parsing matches the report, but it reverses intentionally added recovery behavior and should be settled as a maintainer policy decision before code automation changes it.
