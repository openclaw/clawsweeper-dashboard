## Summary

Keep open. Current main still performs the reported default AGENTS.md section parsing and post-compaction reinjection while AGENTS.md remains normal Project Context, so this is not implemented or stale; changing the default needs maintainer policy because the behavior was intentionally added for post-compaction recovery.

## What This Changes

The issue requests removing or replacing post-compaction AGENTS.md section parsing/reinjection because the system prompt already retains Project Context and the parsing creates prompt bloat plus workspace-dependent behavior.

## Best Possible Solution

Set a maintainer-approved post-compaction recovery policy, then update the refresh prompt, trusted system-event path, summary suffix, config docs, and focused tests together; a conservative end state would make AGENTS.md section reinjection opt-in or skip it when normal Project Context is retained.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: with default config and an AGENTS.md containing Session Startup or Red Lines, readPostCompactionContext returns a refresh payload that is appended or enqueued after compaction while AGENTS.md remains injected as Project Context.

## Solution Assessment

Unclear. Removing the magic parsing matches the report, but it changes deliberately merged recovery behavior, so the narrow maintainable fix needs a maintainer decision on the intended default before automation changes it.
