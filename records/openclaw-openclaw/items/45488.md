## Summary

Keep open. Current main and the latest release still default to heading-based AGENTS.md post-compaction reinjection through the refresh prompt and compaction-summary suffix paths, and the open linked fix PR is unmerged and currently conflicting, so it owns the implementation candidate but has not solved this issue yet.

## What This Changes

The issue asks OpenClaw to stop duplicating retained AGENTS.md/system-prompt guidance after compaction and stop making session behavior depend on magic AGENTS.md headings by default.

## Best Possible Solution

Review and complete https://github.com/openclaw/openclaw/pull/85960, or close it with an explicit maintainer policy decision; the final fix should apply one post-compaction reinjection contract consistently across refresh prompt, extraSystemPrompt, summary suffix, docs, config help, and regression tests.

## Reproduction Assessment

Yes, at source level. With unset config and matching AGENTS.md H2/H3 sections, current main extracts those sections and injects them through the post-compaction refresh prompt/system event paths and the compaction summary suffix.

## Solution Assessment

Unclear until maintainers settle the policy. Making reinjection opt-in addresses the bloat and magic-heading concern, but the change must be applied consistently and handled as a shipped default/upgrade decision.
