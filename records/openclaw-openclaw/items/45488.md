## Summary

Current main still has the reported behavior: AGENTS.md sections are extracted by default and injected after compaction while AGENTS.md remains normal Project Context. Because that reinjection is now documented, configurable, and covered by tests, changing the default needs maintainer policy rather than conservative cleanup.

## What This Changes

The issue asks OpenClaw to remove or change default AGENTS.md post-compaction section parsing because it duplicates retained system prompt content and makes behavior depend on magic headings in user-editable AGENTS.md files.

## Best Possible Solution

Choose a maintainer-approved post-compaction recovery policy, likely making AGENTS.md section reinjection opt-in or skipping it when normal Project Context is retained, then update the refresh prompt, summary suffix, config docs, and focused tests together.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: with default config and an AGENTS.md containing the default sections, current main returns a post-compaction refresh payload, appends/enqueues it after compaction, and also keeps AGENTS.md in Project Context.

## Solution Assessment

Unclear. Removing default parsing matches the report, but it changes intentionally documented recovery behavior, so the narrowest maintainable fix depends on maintainer policy.
