## Summary

Keep open. Current main still has the reported default behavior: AGENTS.md sections are parsed and injected after compaction while bootstrap Project Context can still include AGENTS.md. This is not stale or implemented cleanup; changing it would alter intentionally merged post-compaction recovery behavior, so it needs maintainer product judgment before a fix PR changes the default.

## What This Changes

The issue requests removing or changing the post-compaction AGENTS.md section parsing and reinjection introduced by #18049 because it duplicates prompt content and makes behavior depend on magic section headings in user-editable workspace files.

## Best Possible Solution

Set a maintainer-approved post-compaction recovery policy, then update the refresh prompt, trusted system-event path, summary suffix, config docs, and focused tests together. A conservative end state would make AGENTS.md section reinjection opt-in or skip it when normal Project Context is retained.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: with default config and an AGENTS.md containing Session Startup or Red Lines, readPostCompactionContext returns a refresh payload, follow-up or auto-compaction injects it, and the summary safeguard can append the same workspace-critical-rules while AGENTS.md remains a normal Project Context file.

## Solution Assessment

Unclear. Removing magic parsing aligns with the report, but the current behavior was deliberately added for post-compaction recovery, so the best fix needs a maintainer decision on the intended default and must update all injection paths together.
