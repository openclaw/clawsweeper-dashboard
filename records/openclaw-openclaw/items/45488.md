## Summary

Keep open. Current main and the latest release still default to extracting configured AGENTS.md sections and injecting them after compaction, while AGENTS.md is also part of normal Project Context; the source-level concern is still real, but changing the shipped default needs maintainer product judgment.

## What This Changes

Stop post-compaction recovery from duplicating or magic-parsing AGENTS.md sections that are already present in normal system prompt context.

## Best Possible Solution

Choose a maintainer-approved post-compaction recovery policy, likely making AGENTS.md reinjection explicit or skipping it when retained Project Context already carries the same content, then align runtime, docs/config help, and focused tests.

## Reproduction Assessment

Yes, at source level. With default config and matching H2/H3 AGENTS.md sections, current main extracts those sections and appends or enqueues a post-compaction refresh while AGENTS.md can also be present in normal Project Context.

## Solution Assessment

Unclear. Removing or disabling default parsing would address the bloat and magic-heading concern, but current releases document the behavior and provide a [] opt-out, so maintainers need to decide the supported recovery policy.
