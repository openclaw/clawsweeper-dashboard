## Summary

Keep open. Current main still has the reported AGENTS.md section extraction and reinjection paths, and the linked discussion identifies a real source-level duplication risk; however, the behavior is now documented as post-compaction recovery, so changing the default needs maintainer policy rather than conservative cleanup closure.

## What This Changes

The issue asks to remove or replace post-compaction AGENTS.md section reinjection because system prompt context is already retained and section-name parsing can add duplicated, workspace-dependent behavior.

## Best Possible Solution

Choose a maintainer-approved post-compaction recovery policy, likely making AGENTS.md section reinjection opt-in or skipping it when normal Project Context is retained, then align runtime prompts, summary suffixing, config docs, and focused tests together.

## Reproduction Assessment

Yes, at source level. On current main, default config plus an AGENTS.md with matching H2/H3 sections makes readPostCompactionContext produce refresh content that is appended or enqueued after compaction while AGENTS.md remains normal Project Context.

## Solution Assessment

Unclear. Removing default parsing matches the report, but it would reverse documented recovery behavior from the merged post-compaction work, so the maintainable fix depends on a maintainer policy decision.
