## Summary

Keep open. Current main and the latest release still extract AGENTS.md sections by default and inject them after compaction; deciding whether to remove, make opt-in, or preserve that shipped recovery behavior is a maintainer policy/default decision.

## What This Changes

The issue asks OpenClaw to stop post-compaction AGENTS.md section reinjection from duplicating retained system-prompt content and from depending on magic Markdown headings.

## Best Possible Solution

Choose a supported post-compaction recovery policy, likely skipping default AGENTS.md reinjection when retained context already carries the same content or making reinjection explicit, then align runtime behavior, docs/config help, and focused regression tests.

## Reproduction Assessment

Yes, at source level. With default config and matching H2/H3 AGENTS.md sections, current main extracts those sections and injects them after compaction through summary, extraSystemPrompt, and trusted system-event paths.

## Solution Assessment

Unclear. Removing the parser would address the bloat and magic-heading concern, but it would intentionally change a documented shipped default and opt-out model, so maintainers should decide the supported recovery policy first.
