## Summary

Keep open. Current main and the latest release still extract configured AGENTS.md sections and inject them after compaction, while the remaining fix is a maintainer policy decision about whether that shipped recovery behavior should stay default, become explicit, or be skipped when retained context already carries the same content.

## What This Changes

The issue asks OpenClaw to stop post-compaction logic from duplicating AGENTS.md/system-prompt guidance and relying on magic heading parsing in user-editable AGENTS.md files.

## Best Possible Solution

Choose a supported post-compaction recovery policy, likely skipping AGENTS.md reinjection when retained context already carries the same content or making recovery injection explicit, then align runtime, docs/config help, and focused tests.

## Reproduction Assessment

Yes, at source level. With default config and matching H2/H3 AGENTS.md sections, current main extracts those sections and injects them after compaction through the extraSystemPrompt and trusted-system-event paths.

## Solution Assessment

Unclear. Removing default parsing would address the bloat and magic-heading concern, but current releases document reinjection and provide a [] opt-out, so maintainers need to decide the supported recovery policy before code changes.
