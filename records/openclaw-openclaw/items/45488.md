## Summary

Keep open. Current main and the latest release still extract AGENTS.md sections and inject them after compaction, so the reported session-state behavior remains real and needs a maintainer policy decision before changing shipped defaults.

## What This Changes

The issue asks OpenClaw to stop post-compaction AGENTS.md heading parsing and reinjection that can duplicate retained prompt context and make behavior depend on user-editable section names.

## Best Possible Solution

Choose the supported post-compaction recovery policy, likely skipping AGENTS.md reinjection when retained context already carries the same content or making recovery injection explicit, then align runtime behavior, docs/config help, and focused tests.

## Reproduction Assessment

Yes, at source level. With default config and matching H2/H3 AGENTS.md sections, current main extracts those sections and injects them after compaction through extraSystemPrompt, trusted system events, and the compaction summary path.

## Solution Assessment

Unclear. Removing default parsing would address the bloat and magic-heading concern, but current releases document reinjection and provide an explicit [] opt-out, so maintainers need to decide the supported recovery policy before code changes.
