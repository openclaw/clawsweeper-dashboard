## Summary

Keep open. Current main still performs AGENTS.md post-compaction section extraction and reinjection in the runtime paths described by the report, but that behavior is now documented and configurable, so changing the default needs maintainer policy rather than conservative cleanup closure.

## What This Changes

This issue requests removing or changing the post-compaction AGENTS.md section parsing/reinjection introduced by https://github.com/openclaw/openclaw/pull/18049 to avoid duplicated session context and workspace-dependent magic strings.

## Best Possible Solution

Choose a maintainer-approved post-compaction recovery policy, likely making AGENTS.md section reinjection opt-in or skipping it when normal Project Context is retained, then align runtime behavior, docs, config help, and focused tests together.

## Reproduction Assessment

Yes, at source level. With default config and an AGENTS.md containing matching H2/H3 sections, current main produces post-compaction refresh content that is appended or enqueued after compaction while AGENTS.md is also part of normal Project Context.

## Solution Assessment

Unclear. Removing default parsing addresses the reported bloat and magic-string concerns, but it would reverse documented recovery behavior, so the maintainable fix depends on a maintainer-approved default policy.
