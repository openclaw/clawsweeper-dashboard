## Summary

Keep open. Current main still performs the reported AGENTS.md section extraction and post-compaction reinjection, and the behavior is now documented/configurable, so changing the default needs maintainer policy rather than cleanup closure.

## What This Changes

The issue asks to stop or replace post-compaction AGENTS.md section parsing/reinjection because it duplicates retained context and depends on user-editable heading strings.

## Best Possible Solution

Choose a maintainer-approved post-compaction recovery policy, likely making AGENTS.md section reinjection opt-in or skipping it when normal Project Context is retained, then align runtime behavior, docs, config help, and focused tests.

## Reproduction Assessment

Yes, at source level. With default config and an AGENTS.md containing matching H2/H3 sections, current main appends or enqueues post-compaction refresh content while the same workspace file can also be part of normal Project Context.

## Solution Assessment

Unclear. Removing default parsing addresses the reported bloat and magic-string concerns, but it would reverse documented recovery behavior, so the maintainable fix depends on a maintainer-approved default policy.
