## Best Possible Solution

## Reproduction Assessment

Yes. Source inspection shows current main defaults group/channel delivery to message-tool-only, while the current doctor compatibility normalizer has no path that persists `messages.groupChat.visibleReplies` for older configured-channel configs.

## Solution Assessment

No, not quite. The doctor-only migration is the right maintainable surface and the generated schema is now updated, but the new configured-channel check should ignore `channels.modelByChannel` so doctor does not mutate configs without real channel sections.
