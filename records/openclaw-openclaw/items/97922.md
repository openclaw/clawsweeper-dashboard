## Summary

Keep open: the PR targets a real subagent completion-loss path, but it is not merge-ready because the new escalation guard is broader than the current successful-completion invariant and real behavior proof is absent.

## What This Changes

The PR preserves expected subagent completion payloads on retry/expiry give-up by reusing suspended final delivery, adding an `escalated` delivery flag, and adding lifecycle tests.

## Best Possible Solution

Narrow escalation to successful expected completions that ended COMPLETE/ok, add captured-timeout or failure regression coverage, and require redacted real behavior proof for the delivery-give-up path before merge.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main has a terminal failed-cleanup fallback, while the PR helper can requeue any expected completion with captured text. I did not run tests because this review is read-only.

## Solution Assessment

No, this is a plausible fix but not the best merge shape yet. The narrow maintainable fix should preserve successful expected completions only and leave timeout/error/killed cleanup terminal unless maintainers explicitly choose broader escalation semantics.
