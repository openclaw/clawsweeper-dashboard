## Summary

Keep open: the branch is a useful slice, but it is not merge-ready because the new give-up helper escalates any expected completion with captured text, including timeout or other terminal outcomes, and the PR still lacks real behavior proof from a real OpenClaw setup.

## What This Changes

The PR preserves selected expected subagent completion payloads on retry/expiry give-up by marking delivery suspended with a new `escalated` flag and adding lifecycle tests.

## Best Possible Solution

Gate escalation to successful COMPLETE/ok expected completions, add captured-timeout or failure regression coverage, and require redacted real-run proof before merge.

## Reproduction Assessment

Yes, at source level: current main has a terminal failed-cleanup fallback for non-suspendable give-up, while the PR helper can requeue captured timeout/failure text because timeout outcomes still freeze captured replies. I did not run tests in this read-only review.

## Solution Assessment

No, this is a plausible fix but not the best merge shape yet. The narrow maintainable fix should preserve successful expected completions only and leave timeout/error/killed cleanup terminal unless maintainers explicitly choose broader escalation semantics.
