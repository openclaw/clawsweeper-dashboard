## Best Possible Solution

Yes, at source level. Current main drops delete-mode expected completions on give-up, and PR head's helper is source-reproducibly over-broad because it suspends any expected captured result without the COMPLETE/ok guard.

No, this is a plausible mitigation but not the best merge shape yet. The narrow maintainable fix should preserve successful expected completions only and leave timeout/error/killed cleanup terminal unless maintainers explicitly choose broader escalation semantics.
