## Summary

Keep open: current main still lacks graph-aware `sessions_send` cycle tracking, and the only focused implementation PRs were closed unmerged. The gap is source-reproducible and bounded, but it is a new safety feature rather than a documented bug in existing `maxPingPongTurns` behavior.

## What This Changes

The issue requests provenance-carried agent-chain tracking so `sessions_send` can block A-to-B-to-C-to-A style cycles that bypass pairwise ping-pong limits.

## Best Possible Solution

Add typed, normalized A2A chain provenance and reject explicit-agent target revisits in `sessions_send`, while preserving existing pairwise `maxPingPongTurns` and non-agent session behavior.

## Reproduction Assessment

Yes, by source inspection. A-to-B-to-C-to-A has no retained path today because each `sessions_send` creates fresh inter-session provenance and the current limiter alternates only the requester and target sessions.

## Solution Assessment

Yes in direction. Provenance-carried chain tracking is the narrow maintainable fix, but it should be implemented against the current schema/tool construction path and normalize agent IDs rather than reviving the closed PR verbatim.
