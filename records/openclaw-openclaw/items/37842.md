## Label Justifications

- `impact:message-loss`: Runaway circular sends can flood coordination traffic and obscure or delay useful messages.

## Summary

Current main still bounds only direct requester↔target exchanges and records only immediate-source provenance, so a separately initiated A→B→C→A cascade remains possible. The linked fix PR was closed unmerged and had a concrete canonicalization bypass concern, so it is useful design context but not a safe superseding implementation.

## What This Changes

Add graph-aware cycle detection so multi-agent `sessions_send` chains cannot revisit an agent and evade the pairwise `maxPingPongTurns` limit.

## Best Possible Solution

Extend canonical inter-session provenance with a bounded chain of normalized agent IDs, reset it at genuine external/system roots, and reject delivery when the canonical target already appears in the active chain.

## Reproduction Assessment

Yes, at source level. Current provenance retains only the immediate sender and `maxPingPongTurns` alternates one requester-target pair, leaving no state that can identify A→B→C→A across separately started runs.

## Solution Assessment

Yes, with one correction: graph ancestry belongs in the canonical inter-session provenance path, but all agent IDs must be normalized and the chain lifetime must end at a true new root; the closed PR should not be revived unchanged.
