Keep open: current main still lacks graph-aware `sessions_send` cycle provenance, and the focused implementation PR was closed unmerged. The gap is bounded and source-reproducible, but it is a new safety feature rather than a documented `maxPingPongTurns` bug.

## What This Changes

Request graph-aware loop detection so multi-agent `sessions_send` cascades such as A-to-B-to-C-to-A cannot bypass pairwise `maxPingPongTurns`.

## Best Possible Solution

Add typed, normalized A2A chain provenance through the current provenance schema and tool-construction path, reject explicit-agent target revisits in `sessions_send`, and preserve existing pairwise and non-agent behavior.

## Reproduction Assessment

Yes, by source inspection. A-to-B-to-C-to-A has no retained path today because each `sessions_send` creates fresh inter-session provenance and the current limiter alternates only requester and target sessions.

## Solution Assessment

Yes in direction. Provenance-carried chain tracking is the narrow maintainable solution, but it should normalize agent IDs and be implemented against the current schema/tool path rather than reviving the closed PR verbatim.
