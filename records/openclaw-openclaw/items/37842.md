## Summary

Keep open: current main still lacks graph-aware `sessions_send` cycle provenance, and the only focused implementation PR was closed unmerged. The remaining work is a bounded new safety feature rather than a documented `maxPingPongTurns` bug, so it is suitable for a maintainer-promoted repair lane but not an auto-close.

## What This Changes

The issue requests chain-aware `sessions_send` loop detection so A2A cascades like A to B to C to A reject revisiting an agent instead of relying only on pairwise `maxPingPongTurns`.

## Best Possible Solution

Add typed, normalized A2A chain provenance through the current provenance schema and tool-construction path, reject explicit-agent target revisits in `sessions_send`, and preserve existing pairwise `maxPingPongTurns` plus non-agent session behavior.

## Reproduction Assessment

Yes, by source inspection. A-to-B-to-C-to-A has no retained path today because each `sessions_send` creates fresh inter-session provenance and the current limiter alternates only requester and target sessions.

## Solution Assessment

Yes in direction. Provenance-carried chain tracking is the narrow maintainable solution, but it should normalize agent IDs and be implemented against the current schema/tool path rather than reviving the closed PR verbatim.
