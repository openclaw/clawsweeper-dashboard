## Summary

Keep open: current main still lacks graph-aware chain propagation or a cycle guard for multi-agent `sessions_send`, and the linked fix PR was closed unmerged rather than shipped.

## What This Changes

The issue requests graph-aware cycle detection for multi-agent `sessions_send` chains so 3+ agent loops cannot bypass `maxPingPongTurns`.

## Best Possible Solution

Add typed inter-session chain tracking, propagate it through runner/tool creation into `sessions_send`, reject explicit-agent target revisits, and cover the 3-agent cycle plus legacy-key non-cycle cases with regression tests.

## Reproduction Assessment

Yes, by source inspection. Current main creates fresh `sessions_send` provenance and only applies the pairwise requester/target `maxPingPongTurns` loop, so a three-agent A→B→C→A chain has no stored path to detect the revisit.

## Solution Assessment

Yes in direction. The narrow maintainable fix is a provenance-carried agent chain with a pre-send target revisit check, updated for the current protocol schema and covered by focused tests.
