## Summary

Keep open: current main and the latest release still lack graph-aware `sessions_send` chain provenance, and the only focused implementation PR was closed unmerged. The source still models A2A turns as an immediate requester/target pair, so triangular or polygonal cascades remain source-reproducible.

## What This Changes

Request graph-aware cycle detection for multi-agent `sessions_send` cascades so A->B->C->A style loops are blocked instead of bypassing `maxPingPongTurns`.

## Best Possible Solution

Add typed, normalized A2A chain provenance through the current provenance schema and tool-construction path, reject explicit-agent target revisits in `sessions_send`, and preserve existing pairwise and non-agent behavior.

## Reproduction Assessment

Yes, by source inspection. Current main keeps only immediate inter-session provenance and the A2A limiter alternates one requester/target pair, so an A->B->C->A cascade has no path-aware state to trip.

## Solution Assessment

Yes in direction. Provenance-carried chain tracking is the narrow maintainable solution, but a replacement should normalize agent IDs and fit the current schema/tool path rather than reviving the closed PR verbatim.
