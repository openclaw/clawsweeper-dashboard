## Summary

Keep open: current main and the latest release still lack graph-aware `sessions_send` chain tracking, and the focused implementation PRs were closed unmerged. The gap is source-reproducible and bounded, but it is a new safety feature rather than a documented bug in the existing pairwise `maxPingPongTurns` contract.

## What This Changes

The issue requests graph-aware `sessions_send` cycle detection so multi-agent A->B->C->A cascades cannot bypass the existing pairwise `maxPingPongTurns` guard.

## Best Possible Solution

Add typed, normalized A2A chain provenance and reject explicit-agent target revisits in `sessions_send`, while preserving the existing pairwise limiter and non-agent session behavior.

## Reproduction Assessment

Yes, by source inspection. A-to-B-to-C-to-A has no retained path today because each `sessions_send` creates fresh inter-session provenance and the current limiter alternates only requester and target sessions.

## Solution Assessment

Yes in direction. Provenance-carried chain tracking is the narrow maintainable fix, but it should be implemented against the current schema/tool construction path and canonicalize agent IDs rather than reviving the closed PR verbatim.
