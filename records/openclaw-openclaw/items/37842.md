## Summary

Keep open: current main and the latest release still lack chain/visited-agent provenance for `sessions_send`, the focused fix PR is closed unmerged, and source inspection still shows polygonal A2A cascades can bypass the pairwise limiter.

## What This Changes

Request graph-aware `sessions_send` loop detection that tracks inter-agent message chains and blocks delivery when a target agent is revisited.

## Best Possible Solution

Add typed, normalized A2A chain provenance through `InputProvenance`, gateway schema, and tool construction, then reject explicit-agent target revisits while preserving existing pairwise and non-agent behavior.

## Reproduction Assessment

Yes, by source inspection. Current main keeps only immediate inter-session provenance and alternates one requester/target pair in the A2A limiter, so an A->B->C->A cascade has no path-aware state to trip.

## Solution Assessment

Yes in direction: provenance-carried chain tracking is the narrow maintainable solution. A replacement should not revive the closed PR verbatim because it needs normalized agent IDs and the current protocol schema/tool path updated cleanly.
