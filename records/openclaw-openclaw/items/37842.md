## Summary

Keep open: current main still carries immediate-only inter-session provenance and pairwise A2A ping-pong limiting, while the linked graph-aware fix PR is closed unmerged and the latest release has the same gap.

## What This Changes

The issue requests graph-aware cycle detection for multi-agent `sessions_send` chains so triangular or polygonal A2A cascades cannot bypass `maxPingPongTurns`.

## Best Possible Solution

Add normalized A2A chain provenance through `InputProvenance`, Gateway schema validation, and nested tool construction, then reject canonical agent revisits while preserving the existing pairwise `maxPingPongTurns` behavior.

## Reproduction Assessment

Yes, at source level. Current main carries only immediate inter-session provenance and alternates a requester/target pair, so an A-to-B-to-C-to-A cascade has no path-aware state to trip; I did not run a live multi-agent replay.

## Solution Assessment

Yes in direction. Provenance-carried chain tracking is the narrow maintainable solution, provided it covers canonical agent ID normalization, Gateway schema validation, nested tool construction, and regression tests.
