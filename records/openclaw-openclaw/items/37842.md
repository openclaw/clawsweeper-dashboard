## Summary

Keep open: current main still only has pairwise A2A ping-pong handling and fresh inter-session provenance, so the graph-aware `sessions_send` cycle guard requested here is not implemented; the linked PR #37945 was closed unmerged.

## What This Changes

Request graph-aware `sessions_send` loop detection that carries a chain through inter-session sends and rejects triangular or polygonal agent cycles.

## Best Possible Solution

Add typed provenance chain tracking across runner/tool creation and reject explicit-agent target revisits while preserving existing pairwise `maxPingPongTurns` behavior.

## Reproduction Assessment

Yes, by source inspection. `sessions_send` creates a new inter-session provenance object on each send and the only loop limiter is the requester/target `maxPingPongTurns` A2A flow, so A->B->C->A has no retained path to detect the revisit.

## Solution Assessment

Yes in direction. The provenance-carried chain approach is the narrow maintainable fix, but it should be implemented against the current schema and tool-factory path rather than resurrecting the closed PR verbatim.
