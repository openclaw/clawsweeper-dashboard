## Summary

Keep open: current main still has immediate-only inter-session provenance plus a pairwise A2A ping-pong limiter, while both graph-aware implementation PRs are closed unmerged and no open canonical PR owns this exact chain-provenance fix.

## What This Changes

The issue requests graph-aware inter-session provenance so `sessions_send` can block multi-agent A-to-B-to-C-to-A cascades that bypass pairwise ping-pong limits.

## Best Possible Solution

Add normalized A2A chain provenance through `InputProvenance`, the gateway schema, and nested tool construction, then reject canonical agent revisits while preserving existing pairwise `maxPingPongTurns` and non-agent session behavior.

## Reproduction Assessment

Yes, at source level. Current main carries only immediate inter-session provenance and alternates a requester/target pair, so an A-to-B-to-C-to-A cascade has no path-aware state to trip; I did not run a live multi-agent replay.

## Solution Assessment

Yes in direction. Provenance-carried chain tracking is the narrow maintainable fix, with canonical agent-id normalization and coverage for the gateway schema plus nested tool construction path.
