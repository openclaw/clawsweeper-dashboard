## Summary

Keep open: current main still has only pairwise A2A ping-pong limiting plus source provenance, while the requested visited-agent chain is not implemented. The issue is now paired with an open draft fix PR, so it should stay open until that PR lands or maintainers choose a different fix path.

## What This Changes

The issue requests graph-aware loop detection for multi-agent `sessions_send` chains so triangular or polygonal A2A cascades cannot bypass `maxPingPongTurns`.

## Best Possible Solution

Review and land or replace the linked cyclic-route guard so `sessions_send` carries a bounded visited-agent chain through provenance and blocks revisiting an agent before starting another run.

## Reproduction Assessment

Yes from source inspection: current main has no propagated visited-agent chain, and the existing A2A helper alternates only one requester-target pair. I did not run a live 17-agent overnight reproduction in this read-only review.

## Solution Assessment

Yes: carrying normalized route provenance and blocking revisits before delivery is the narrow maintainable fix for this safety gap. The best path now is to finish review of https://github.com/openclaw/openclaw/pull/94594 rather than start a second implementation.
