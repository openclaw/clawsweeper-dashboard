## What This Changes

The issue requests graph-aware cycle detection for multi-agent `sessions_send` chains so A-to-B-to-C-to-A cascades cannot bypass the existing pairwise ping-pong limiter.

## Best Possible Solution

Add normalized A2A chain provenance through `InputProvenance`, the gateway schema, and nested tool construction, then reject canonical agent revisits while preserving existing pairwise `maxPingPongTurns` and non-agent session behavior.

## Reproduction Assessment

Yes, at source level. Current main carries only immediate provenance and alternates a requester/target pair, so an A-to-B-to-C-to-A cascade has no path-aware state to trip; I did not run a live multi-agent replay.

## Solution Assessment

Yes in direction. Provenance-carried chain tracking is the narrow maintainable fix, with canonical agent-id normalization and coverage for the gateway schema plus nested tool construction path.
