## Summary

Keep open: current main and the latest release still only carry pairwise `sessions_send` provenance and do not detect 3+ agent cycles, while the exact companion fix PR was closed unmerged.

## What This Changes

The issue requests graph-aware cycle detection so `sessions_send` can block triangular or polygonal inter-agent cascades that bypass `maxPingPongTurns`.

## Best Possible Solution

Add normalized inter-session ancestry to core provenance, enforce cycle blocking in `sessions_send`, update the protocol schema, and cover 3-agent loops without adding a new config knob or changelog entry.

## Reproduction Assessment

Yes, at source level: current `sessions_send` provenance is pairwise source metadata only, and the A2A loop counter alternates two session keys rather than tracking a 3+ agent chain. I did not run a live 17-agent reproduction in this read-only review.

## Solution Assessment

Yes as a fix direction: normalized ancestry in existing inter-session provenance is narrower than disabling A2A broadly and directly addresses multi-agent cycles. The prior closed PR is useful design context, but a new implementation must canonicalize IDs and update the protocol schema.
