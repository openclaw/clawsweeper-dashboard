## Summary

Keep open: current main still has only immediate `inter_session` provenance and pairwise A2A ping-pong limiting, while the focused graph-aware fix PR was closed unmerged. The A->B->C->A cascade remains source-reproducible and is narrow enough for a ClawSweeper repair lane.

## What This Changes

The issue requests graph-aware `sessions_send` chain provenance so multi-agent cycles such as A->B->C->A cannot bypass `maxPingPongTurns`.

## Best Possible Solution

Add typed, normalized A2A chain provenance through the existing provenance schema and tool-construction path, then reject explicit-agent target revisits in `sessions_send` while preserving current pairwise and non-agent behavior.

## Reproduction Assessment

Yes, by source inspection. Current main keeps only immediate inter-session provenance and alternates one requester/target pair in the A2A limiter, so an A->B->C->A cascade has no path-aware state to trip.

## Solution Assessment

Yes in direction. Provenance-carried chain tracking is the narrow maintainable solution, but the replacement should normalize agent IDs and fit the current schema/tool path instead of reviving the closed PR verbatim.
