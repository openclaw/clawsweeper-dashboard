The issue requests chain-aware `sessions_send` provenance so triangular or polygonal agent cascades are detected instead of bypassing `maxPingPongTurns`.

## Best Possible Solution

Add normalized A2A chain provenance through `InputProvenance`, gateway schema validation, and nested tool construction, then reject canonical agent revisits while preserving existing pairwise and non-agent behavior.

## Solution Assessment

Yes in direction: provenance-carried chain tracking is the narrow maintainable fix. A replacement should not copy the closed PR verbatim until agent IDs and incoming chain entries are canonicalized and the gateway protocol schema is updated.
