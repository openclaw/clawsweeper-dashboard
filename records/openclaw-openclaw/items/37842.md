## Summary

Keep open: current main still has immediate-only inter-session provenance and a requester/target ping-pong limiter, while both known related PRs are closed unmerged and no open replacement owns graph-aware cycle detection.

## What This Changes

The issue requests graph-aware `sessions_send` loop detection so multi-agent cycles such as A to B to C to A cannot bypass the pairwise `maxPingPongTurns` guard.

## Best Possible Solution

Add normalized A2A chain provenance through `InputProvenance`, the gateway schema, and tool construction, then reject canonical agent revisits while preserving current pairwise and non-agent behavior.

## Reproduction Assessment

Yes, at source level. Current main carries only immediate provenance and alternates one requester/target pair, so an A to B to C to A cascade has no path-aware state to trip.

## Solution Assessment

Yes in direction: provenance-carried chain tracking is the narrow maintainable fix. A replacement should not copy the closed PR verbatim until agent IDs and incoming chain entries are canonicalized and the protocol schema is updated.
