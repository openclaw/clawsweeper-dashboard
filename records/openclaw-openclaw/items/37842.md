## Summary

Keep open: current main and v2026.5.19 still carry immediate-only inter-session provenance plus a pairwise requester/target ping-pong limiter, while the linked graph-aware implementation PR is closed unmerged and no open `chainAgentIds` replacement owns this exact fix.

## What This Changes

The issue requests graph-aware cycle detection for multi-agent `sessions_send` chains so A-to-B-to-C-to-A cascades cannot bypass the existing pairwise `maxPingPongTurns` guard.

## Best Possible Solution

Add normalized A2A chain provenance through `InputProvenance`, gateway schema validation, and nested tool construction, then reject canonical agent revisits while preserving existing pairwise and non-agent behavior.

## Reproduction Assessment

Yes, at source level. Current main carries only immediate provenance and alternates a requester/target pair, so an A-to-B-to-C-to-A cascade has no path-aware state to trip; I did not run a live 17-agent replay.

## Solution Assessment

Yes in direction: provenance-carried chain tracking is the narrow maintainable fix. A replacement should rebuild the closed PR shape with canonical agent ID normalization and gateway protocol schema coverage.
