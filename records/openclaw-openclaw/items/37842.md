## Summary

Keep open: current main still lacks graph-aware `sessions_send` chain provenance, and the focused implementation at https://github.com/openclaw/openclaw/pull/37945 was closed unmerged. The current code tracks immediate inter-session source metadata and a pairwise A2A reply-back loop, so the reported A->B->C->A cascade remains source-reproducible.

## What This Changes

The issue requests chain-aware `sessions_send` provenance so multi-agent cycles can be blocked instead of only limiting direct pairwise ping-pong turns.

## Best Possible Solution

Add typed, normalized A2A chain provenance through the existing provenance schema and tool-construction path, then reject explicit-agent target revisits in `sessions_send` while preserving current pairwise and non-agent behavior.

## Reproduction Assessment

Yes, by source inspection. Current main keeps only immediate inter-session provenance and the A2A limiter alternates one requester/target pair, so an A->B->C->A cascade has no path-aware state to trip.

## Solution Assessment

Yes in direction. Provenance-carried chain tracking is the narrow maintainable solution, but a replacement should normalize agent IDs and fit the current schema/tool path rather than reviving the closed PR verbatim.
