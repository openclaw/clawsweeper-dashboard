## Summary

Keep open: current main still lacks graph-aware chain or visited-agent provenance for `sessions_send`, and the only focused implementation attempt is closed unmerged. Source inspection still shows immediate-only provenance plus a pairwise A2A limiter, so triangular or polygonal cascades remain plausible on current main.

## What This Changes

The issue requests graph-aware cycle detection for multi-agent `sessions_send` chains so A->B->C->A cascades cannot bypass the existing pairwise `maxPingPongTurns` guard.

## Best Possible Solution

Add typed, normalized A2A chain provenance through `InputProvenance`, gateway schema, and tool construction, then reject explicit-agent target revisits while preserving existing pairwise and non-agent behavior.

## Reproduction Assessment

Yes, by source inspection. Current main keeps only immediate inter-session provenance and the A2A limiter alternates one requester/target pair, so an A->B->C->A cascade has no path-aware state to trip.

## Solution Assessment

Yes in direction: provenance-carried chain tracking is the narrow maintainable solution. A replacement should normalize agent IDs and update the current protocol/tool-construction path rather than revive the closed pull request verbatim.
