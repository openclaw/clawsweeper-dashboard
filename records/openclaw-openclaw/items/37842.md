## Summary

Keep open: current main and the latest release still carry only immediate inter-session provenance, so the existing pairwise A2A limiter cannot detect three-or-more-agent cycles; the only graph-aware implementation found is closed unmerged.

## What This Changes

Request graph-aware sessions_send loop detection so triangular or polygonal multi-agent cascades cannot bypass maxPingPongTurns.

## Best Possible Solution

Add normalized A2A chain provenance through InputProvenance, the gateway schema, and nested tool construction, then reject canonical agent revisits while preserving existing pairwise maxPingPongTurns behavior.

## Reproduction Assessment

Yes, at source level. Current main carries only immediate inter-session provenance and alternates a requester/target pair, so an A-to-B-to-C-to-A cascade has no path-aware state to trip; I did not run a live multi-agent replay.

## Solution Assessment

Yes in direction. Provenance-carried chain tracking is the narrow maintainable solution, provided the implementation normalizes agent IDs and covers both gateway schema validation and nested tool construction.
