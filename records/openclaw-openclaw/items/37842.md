## Summary

Keep open: current main and v2026.6.6 still have pairwise A2A ping-pong handling and inter-session provenance, but no propagated agent-chain ancestry or cycle check for triangular/polygonal `sessions_send` cascades. The linked implementation attempts were closed unmerged, and the open related A2A work only partially overlaps.

## What This Changes

Requests graph-aware `sessions_send` cycle detection so multi-agent A2A chains that revisit an agent are blocked instead of relying only on pairwise `maxPingPongTurns`.

## Best Possible Solution

Add normalized internal A2A ancestry to `sessions_send` provenance and block delivery when the resolved target agent already appears in the chain, with focused tests for direct, triangular, longer-cycle, and acyclic handoff cases.

## Reproduction Assessment

Yes, source-level: current main has no chain field, no target-in-chain check, and only pairwise requester/target `maxPingPongTurns` handling. I did not run a live multi-agent cascade in this read-only review.

## Solution Assessment

Yes as a fix direction: normalized ancestry in provenance is narrower than broad A2A shutdown and directly addresses multi-agent cycles. The prior closed PR is useful design context, but a new implementation must canonicalize IDs and update the closed protocol schema.
