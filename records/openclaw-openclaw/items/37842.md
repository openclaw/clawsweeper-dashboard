## Summary

Keep open: current main and the latest release still carry immediate-only inter-session provenance plus the existing pairwise A2A ping-pong limiter, while the linked graph-aware implementation at https://github.com/openclaw/openclaw/pull/37945 is closed unmerged and no canonical replacement currently owns the chain-provenance fix.

## What This Changes

Request graph-aware `sessions_send` loop detection that propagates inter-agent chain provenance and blocks cycles across three or more agents.

## Best Possible Solution

Add normalized A2A chain provenance through `InputProvenance`, gateway schema validation, and nested tool construction, then reject canonical agent revisits while preserving existing pairwise `maxPingPongTurns` behavior and non-agent session behavior.

## Reproduction Assessment

Yes, at source level. Current main carries only immediate provenance and alternates a requester/target pair, so an A-to-B-to-C-to-A cascade has no path-aware state to trip; I did not run a live multi-agent replay.

## Solution Assessment

Yes in direction. Provenance-carried chain tracking is the narrow maintainable fix, but the replacement should normalize agent IDs and cover the gateway schema plus nested tool construction path.
