## Summary

Keep open: current main still lacks graph-aware `sessions_send` cycle tracking; provenance and protocol state carry no agent chain, `sessions_send` starts fresh inter-session provenance, and the only limiter remains the requester/target ping-pong flow. The prior focused PR was closed unmerged and no open PR currently references this issue.

## What This Changes

Request graph-aware loop detection for `sessions_send` so multi-agent cycles such as A->B->C->A are detected instead of bypassing `maxPingPongTurns`.

## Best Possible Solution

Add typed, normalized A2A chain provenance and reject explicit-agent target revisits in `sessions_send`, while preserving existing pairwise `maxPingPongTurns` and non-agent session behavior.

## Reproduction Assessment

Yes, by source inspection. A->B->C->A has no retained path because each `sessions_send` creates fresh inter-session provenance and the current limiter only alternates requester and target.

## Solution Assessment

Yes in direction. Provenance-carried chain tracking is the narrow maintainable fix, but it should be implemented against current schema/tool construction and normalize agent IDs rather than reviving the closed PR verbatim.
