## Summary

Keep open: current main and the latest release still lack graph-aware `sessions_send` chain state, so multi-agent A->B->C->A cascades remain a source-backed core A2A safety gap rather than an implemented cleanup target.

## What This Changes

Request graph-aware cycle detection for `sessions_send` so multi-agent cascades cannot bypass the current pairwise `maxPingPongTurns` guard.

## Best Possible Solution

Add a narrow provenance-backed graph-cycle guard at the `sessions_send` delivery boundary, with canonical agent IDs, protocol/schema support, and focused A->B->C->A regression coverage.

## Reproduction Assessment

Yes, at source level: current `sessions_send` carries no visited-agent chain and the A2A helper only alternates requester and target sessions. I did not run a live multi-agent storm reproduction in this read-only review.

## Solution Assessment

Yes: a narrow provenance/protocol chain guard at the `sessions_send` delivery boundary is the maintainable fix. The implementation should normalize agent IDs and update gateway protocol tests rather than adding a new config knob or prompt-only rule.
