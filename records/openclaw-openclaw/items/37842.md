## Summary

Keep open: current main still has only pairwise `maxPingPongTurns` handling for `sessions_send`, and neither current main nor the latest release has a propagated agent-chain/trace field. The linked implementation attempts are closed unmerged, so no canonical fix currently owns the remaining work.

## What This Changes

Request graph-aware cycle detection for `sessions_send` so multi-agent A->B->C->A cascades are blocked instead of bypassing `maxPingPongTurns`.

## Best Possible Solution

Add a normalized inter-session agent-chain through the provenance and gateway schema, then block revisiting an agent before `sessions_send` delivery while preserving the existing pairwise `maxPingPongTurns` behavior.

## Reproduction Assessment

Yes, at source level: current `sessions_send` carries no chain state and the A2A helper only alternates requester/target sessions. I did not run a live 3-agent deployment reproduction in this read-only review.

## Solution Assessment

Yes: a narrow normalized chain in `InputProvenance` plus a pre-delivery `sessions_send` cycle guard is the maintainable fix. The earlier closed PR shape was close, but a replacement should normalize agent IDs and update the gateway schema explicitly.
