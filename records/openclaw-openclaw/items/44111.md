Full review comments:

- **[P1] Reject symlinked snapshot store directories:** `src/backup/snapshot-store/provider-folder.ts:67-85`
  - body: The folder store creates `targetDir/snapshots/<installationId>` with recursive `mkdir` and then writes/renames through it. If a synced backup target already contains `snapshots` or the installation directory as a symlink, `backup run` can write payloads outside `backup.target`; validate every existing directory segment before mkdir/write/rename.
  - confidence: 0.92
- **[P1] Validate workspace backup roots before mkdir:** `src/commands/workspace-backup.ts:350-352`
  - body: `workspaceBackupRunCommand` creates `backup.target/workspace/mirrors` and `.staging` before rejecting target-side symlinks. A shared backup folder can redirect those mkdir/cp/rename/rm operations through a symlinked `workspace` or `mirrors` path, so the target tree needs no-symlink validation before any write.
  - confidence: 0.91
- **[P1] Check every restore path segment:** `src/commands/backup-restore.ts:330-337`
  - body: `ensureDirectoryPathWithoutSymlinks` lstat-checks the final existing probe, but `lstat('/a/link/dir')` still follows an intermediate `link`. Walk from the filesystem root and check each existing segment before restore can create temp paths or rename into the target.
  - confidence: 0.9
