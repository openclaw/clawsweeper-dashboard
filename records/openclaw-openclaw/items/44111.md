## Review Findings

Full review comments:

- **[P1] Reject snapshot target symlinks before writing:** `src/backup/snapshot-store/provider-folder.ts:63`
  - body: `writeFileAtomic` and `copyFileAtomic` create and write under `backup.target/snapshots/...` without rejecting symlinked intermediate directories such as `snapshots` or an installation id. In a shared or cloud-synced backup folder, that lets a symlink redirect payload/envelope writes outside the configured backup target; add target-side no-symlink directory creation before the writes.
  - confidence: 0.9
- **[P1] Check workspace target directories before mkdir:** `src/commands/workspace-backup.ts:298-299`
  - body: `workspaceBackupRunCommand` creates the mirror and staging directories before effective target-side symlink validation, and the containment helper follows existing symlinks. A symlinked `backup.target/workspace` can redirect mirror writes/removals outside the backup folder, so validate or create every target-side segment as a real directory before `mkdir`, `cp`, `rename`, or stale cleanup.
  - confidence: 0.88
- **[P2] Remove retention config until pruning exists:** `src/config/types.backup.ts:19`
  - body: The PR accepts typed `backup.retention` settings, but the backup run/list/provider code does not prune snapshots or use these values. Persisting a no-op retention policy creates a misleading config contract, so remove it or implement tested pruning before exposing the key.
  - confidence: 0.86
