## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Validate snapshot store directory segments:** `src/backup/snapshot-store/provider-folder.ts:67`
  - body: `writeFileAtomic` creates the destination parent with recursive `mkdir` and then writes/renames through that path. If a synced backup target already contains `snapshots` or an installation directory as a symlink, snapshot upload can place encrypted payloads/envelopes outside `backup.target`; reject symlinked target-side directory segments before creating or writing files.
  - confidence: 0.93
- **[P1] Reject target-side symlinks before mirroring:** `src/commands/workspace-backup.ts:350-351`
  - body: `workspaceBackupRunCommand` creates `backup.target/workspace/mirrors` and `.staging` before proving those target-side segments are real directories. Because the later containment check canonicalizes through the same symlink, a planted `backup.target/workspace -> $HOME` can redirect mirror writes, replacements, stale cleanup, and status writes outside the backup directory.
  - confidence: 0.94
- **[P2] Bound archive path depth during verification:** `src/commands/backup-verify.ts:337-346`
  - body: `collectNestedArchiveParents` expands every parent directory for every normalized archive entry. The PR only caps total path bytes, so a single deeply nested `a/a/a/...` path can still generate a huge set of parent strings and make `backup verify` or restore verification burn CPU/memory on a crafted archive; add a per-entry depth/length cap or avoid materializing all parents.
  - confidence: 0.86
- **[P2] Remove or implement retention config:** `src/config/zod-schema.backup.ts:25`
  - body: The schema accepts `backup.retention`, but the inspected setup/run/list/status/upload paths do not enforce any snapshot pruning. Users can configure `keepDaily`, `keepWeekly`, `keepMonthly`, or `maxSnapshots` and still accumulate snapshots indefinitely, so this public config surface should be removed or implemented before merge.
  - confidence: 0.9
