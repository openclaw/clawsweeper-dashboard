## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Reject symlinked snapshot directories before writes:** `src/backup/snapshot-store/provider-folder.ts:67`
  - body: `uploadSnapshot` writes under `backup.target/snapshots/<installationId>` by first calling recursive `mkdir` and then writing/renaming files. If a cloud-synced or otherwise writable target already contains `snapshots` or the installation directory as a symlink, these writes follow it outside `backup.target`; validate/create each directory segment without following symlinks before copy/rename/list/read.
  - confidence: 0.95
- **[P1] Reject symlinked workspace target directories before mkdir:** `src/commands/workspace-backup.ts:350-351`
  - body: `workspaceBackupRunCommand` creates `backup.target/workspace/mirrors` and `.staging` before any target-side symlink rejection, and the later containment check canonicalizes both paths through `realpath`. A preexisting `backup.target/workspace` symlink can therefore redirect mirror writes or stale mirror deletion outside the backup directory.
  - confidence: 0.95
- **[P2] Remove or implement the retention config:** `src/config/zod-schema.backup.ts:25`
  - body: The PR accepts `backup.retention.keepDaily/keepWeekly/keepMonthly/maxSnapshots` in the public config schema, but the snapshot store and backup commands never read those fields or prune snapshots. Users can set a valid config that silently does nothing, so either remove the key from this PR or implement and test the pruning semantics.
  - confidence: 0.93
- **[P2] Recheck archive entry types while packing:** `src/infra/backup-create.ts:406-420`
  - body: `createBackupArchive` pre-scans sources for symlinks and special files, then hands the same paths to `tar.c()` without a filter or entry-type guard. A writable source tree can change after the pre-scan and still produce an archive that `backup verify`/`restore` later reject, so enforce the same file/directory-only policy during tar packing.
  - confidence: 0.82
