Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Reject symlinked snapshot store directories:** `src/backup/snapshot-store/provider-folder.ts:67-85`
  - body: `uploadSnapshot` writes below `targetDir/snapshots/<installationId>` after recursive `mkdir` and atomic rename without checking intermediate target-side segments. In a synced or shared backup target, a planted `snapshots` or installation symlink can redirect encrypted payload/envelope writes outside `backup.target`; validate every segment before `mkdir`/rename or use a no-follow traversal helper.
  - confidence: 0.91
- **[P1] Validate workspace backup roots before mkdir:** `src/commands/workspace-backup.ts:350-351`
  - body: `workspaceBackupRunCommand` creates `backup.target/workspace/mirrors` and `.staging` before proving the target tree has no symlink segments, and the later containment checks canonicalize after the write. A symlinked `backup.target/workspace` can redirect mirror creation, copy, rename, or stale cleanup into another directory; reject target-side symlink segments before any write.
  - confidence: 0.9
- **[P1] Validate restore targets before rollback:** `src/commands/backup-restore.ts:442-443`
  - body: `applyRestoreOperations` moves an existing target into a rollback directory before `copySourceToTarget` validates the parent path, and that validator only checks the first existing ancestor. If a restore target parent contains a symlink, restore can move or copy through it before failing; validate every target parent segment before `access`, `mkdtemp`, or `rename`.
  - confidence: 0.88
