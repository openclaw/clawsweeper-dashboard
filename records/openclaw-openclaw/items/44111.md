## Summary

Keep open. Current main still lacks the encrypted snapshot backup flow, so this PR is not obsolete, but the current PR head still has full-host restore gaps for in-state config and OAuth paths and the branch is not mergeable against main.

## What This Changes

This PR adds folder-backed encrypted snapshot backups with `backup setup/run/status/list/restore`, workspace mirroring, backup config/schema/docs, alias updates, and restore/snapshot hardening tests.

## Best Possible Solution

If native encrypted snapshots remain desired, rebase or replace this branch, fix the source-vs-target relative-path restore gaps, then land only after focused backup tests plus the broad changed gate pass. If maintainers decide folder-backed snapshots should stay outside core, close this PR with that product decision and keep the existing local archive flow unchanged.

## Reproduction Assessment

Yes. The current-main absence is reproducible by inspecting the CLI registration, docs, and config schema; the PR defects are reproducible from the branch logic with a full-host restore where the source and target config or OAuth directories are both inside state but at different relative paths.

## Solution Assessment

No for the current PR head as-is. The direction is plausible, but the restore coverage checks should compare source and target relative paths before relying on the state asset, and the branch needs conflict resolution plus fresh validation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Restore config when in-state paths differ:** `src/commands/backup-restore.ts:715-716`
  - body: This condition suppresses the config restore whenever both the source and target config paths are inside state. If the archive used one in-state config relative path and the restore host uses another, replacing state restores the old path and leaves the active `OPENCLAW_CONFIG_PATH` missing or stale. Compare source and target relative paths, or enqueue the config operation when they differ.
  - confidence: 0.88
- **[P2] Restore OAuth creds when in-state paths differ:** `src/commands/backup-restore.ts:741-742`
  - body: The credentials branch has the same coverage assumption: any target OAuth dir inside `stateDir` is treated as covered by the state restore. When the backup's OAuth dir lived at a different relative path inside state, full-host restore completes without populating the active OAuth directory, so provider auth data is not recovered.
  - confidence: 0.86
