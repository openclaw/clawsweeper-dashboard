Full review comments:

- **[P1] Reject symlinked workspace backup targets before writing:** `src/commands/workspace-backup.ts:350-351`
  - body: `backup.target` is treated as untrusted because it may be a synced/shared folder, but `workspaceBackupRunCommand` creates `target/workspace/mirrors` and `.staging` before rejecting symlinked target-side segments. If `target/workspace` or `target/workspace/mirrors` is a symlink, these mkdir/cp/rename/rm operations can write or delete outside the configured backup target. Please lstat/walk the target-side path segments and fail before any write.
  - confidence: 0.9
- **[P1] Reject symlinked snapshot store directories before writing:** `src/backup/snapshot-store/provider-folder.ts:67`
  - body: `writeFileAtomic` and `copyFileAtomic` create parent directories and then write through `targetDir/snapshots/<installationId>` without checking whether `snapshots` or the installation directory is a symlink. A symlink planted inside the backup folder can redirect encrypted payload/envelope writes outside `backup.target`, so the folder store needs the same no-symlink target path invariant before writes and reads.
  - confidence: 0.88
