## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Reject symlinked snapshot directories before writes:** `src/backup/snapshot-store/provider-folder.ts:67`
  - body: `writeFileAtomic`/`copyFileAtomic` still create and write through `backup.target/snapshots/<installationId>` without checking that `snapshots` and the installation directory are real directories. In a synced backup folder, a planted symlink can redirect encrypted snapshot writes outside `backup.target`, so uploads must ensure each target-side directory segment is non-symlink before any mkdir/write/rename.
  - confidence: 0.91
- **[P1] Reject symlinked workspace target dirs before mirroring:** `src/commands/workspace-backup.ts:350-351`
  - body: `workspaceBackupRunCommand` creates `mirrorRoot` and `.staging` before proving those target-side paths are not symlinks, and the later containment helper follows existing realpaths. A symlinked `backup.target/workspace` or `mirrors` directory can make mirror copies, renames, or stale cleanup operate outside the backup target.
  - confidence: 0.9
- **[P2] Remove or implement backup retention config:** `src/config/zod-schema.backup.ts:25`
  - body: The schema accepts `backup.retention` and its keys, but the PR does not read or enforce them anywhere else. Users can configure a retention policy that will not prune snapshots, creating a public config contract that does nothing and will be hard to change later.
  - confidence: 0.86
