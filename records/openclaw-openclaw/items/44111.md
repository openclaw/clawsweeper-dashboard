- **[P1] Reject target symlinks before mirror writes:** `src/commands/workspace-backup.ts:350-352`
  - body: `workspaceBackupRunCommand` creates `backup.target/workspace/mirrors` and `.staging` before any no-symlink validation, and the later containment check canonicalizes through symlinks. In a synced backup target, a preexisting `workspace`/`mirrors`/`.staging` symlink can redirect `cp`, `rename`, and stale `rm` outside `backup.target`; reject target-side symlink segments before mkdir/write/delete and add a regression.
  - confidence: 0.92

  - body: `uploadSnapshot` reaches `writeFileAtomic`/`copyFileAtomic`, which `mkdir` and write under `targetDir/snapshots/<installationId>` without checking intermediate directory segments. A shared backup folder can symlink `snapshots` or the installation id directory so encrypted payload/envelope writes land outside the configured target; validate no-symlink directory segments before write/rename.
  - confidence: 0.9
