## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Validate snapshot target directories before writes:** `src/backup/snapshot-store/provider-folder.ts:67`
  - body: `writeFileAtomic` and the sibling payload copy path create destination parents under `backup.target` without rejecting symlinked directory segments. If a shared/cloud backup folder contains `snapshots` or an installation directory as a symlink, the payload/envelope rename can write outside the selected backup target; validate or create each target segment without following symlinks before writing.
  - confidence: 0.91
- **[P1] Reject backup workspace symlinks before mkdir:** `src/commands/workspace-backup.ts:350-351`
  - body: `workspaceBackupRunCommand` creates `backup.target/workspace/mirrors` and `.staging` before any target-side symlink rejection, and the later realpath containment check will pass through a symlinked `workspace` root. A cloud-synced/shared target can redirect mirror writes or stale-mirror deletes outside the backup folder, so reject symlinked target segments before these mkdir/rm/rename operations.
  - confidence: 0.93
- **[P2] Remove retention config until pruning exists:** `src/config/zod-schema.backup.ts:25`
  - body: The schema accepts `backup.retention` values, but the backup run/list/status/store code does not read them or prune snapshots. Accepting a persistent config key without behavior creates a misleading public contract; remove the key for now or implement retention with tests and docs.
  - confidence: 0.86
