## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Reject symlinked workspace backup targets:** `src/commands/workspace-backup.ts:350-353`
  - body: `workspaceBackupRunCommand` creates `backup.target/workspace/mirrors` and `.staging` before rejecting target-side symlinks, and the later containment checks follow `realpath()` for both the child and root. In a shared/cloud backup folder, a planted `workspace` symlink can redirect mirror writes and stale-mirror removals outside `backup.target`, so validate non-symlink target segments before any mkdir/write/remove.
  - confidence: 0.91
- **[P1] Reject symlinked snapshot-store directories:** `src/backup/snapshot-store/provider-folder.ts:67-85`
  - body: `uploadSnapshot` reaches `writeFileAtomic`/`copyFileAtomic`, which create `targetDir/snapshots/<installationId>` and rename files there without checking whether `snapshots` or the installation directory is a symlink. A shared backup target can redirect encrypted payload/envelope writes outside the configured backup directory, so validate every target-side directory segment before writing or reading snapshots.
  - confidence: 0.9
- **[P1] Check every restore parent path segment:** `src/commands/backup-restore.ts:337-339`
  - body: `ensureDirectoryPathWithoutSymlinks` only `lstat`s the deepest existing path, but `lstat('/a/link/dir')` follows the intermediate `link` segment when `dir` exists. A config/OAuth/workspace restore target with a symlinked parent can still redirect the restore copy/rename despite the intended symlink guard, so walk and validate each existing segment from the root down.
  - confidence: 0.86
- **[P2] Restore moved in-state config and OAuth paths:** `src/commands/backup-restore.ts:715-741`
  - body: Full-host restore skips the standalone config or credentials copy whenever the source path and current target path are both inside the state directory. Supported `OPENCLAW_CONFIG_PATH` or `OPENCLAW_OAUTH_DIR` overrides can point to a different relative path under state, so the state restore can succeed while the active config or credentials directory remains missing/stale; compare source and target relative paths before treating state as covering them.
  - confidence: 0.84
- **[P2] Bound nested archive parent expansion:** `src/commands/backup-verify.ts:337-354`
  - body: Verification builds a `Set` of every parent directory for every normalized archive entry. A crafted archive can stay within the total path-byte cap while using an extremely deep single path, forcing excessive parent-string allocation before restore proceeds; add per-entry depth/length limits or replace the parent set with bounded prefix checks.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The diff is security-sensitive and current head still has concrete symlink traversal and restore-placement defects before merge.

- **[high] Target-side symlink traversal in backup output:** `src/commands/workspace-backup.ts:350`
  - body: Workspace mirroring and snapshot upload write into directories below a user-selected backup target without rejecting symlinked target-side segments, allowing a shared backup folder to redirect writes outside the target.
