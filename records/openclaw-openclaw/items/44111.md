Full review comments:

- **[P1] Reject symlinked workspace target roots before writing:** `src/commands/workspace-backup.ts:350-351`
  - body: `workspaceBackupRunCommand` creates `backup.target/workspace/mirrors` and `.staging` before checking the target-side tree. If a synced/shared backup target already has `workspace` or `mirrors` as a symlink, the mkdir/cp/rename/rm path follows it and can write or remove data outside the configured backup target. Validate each existing target path segment with `lstat` before any mkdir/write/remove under the backup target.
  - confidence: 0.91
- **[P1] Validate snapshot store directory segments before writes:** `src/backup/snapshot-store/provider-folder.ts:67`
  - body: The snapshot store creates destination parents with recursive `mkdir` and then writes/renames payload and envelope files through those paths. A symlink planted under `backup.target/snapshots` or `backup.target/snapshots/<installationId>` can redirect snapshot writes outside the backup target. Add a no-symlink directory creation/traversal helper before `writeFileAtomic` and `copyFileAtomic`.
  - confidence: 0.9
