Full review comments:

- **[P1] Reject symlinked snapshot store directories:** `src/backup/snapshot-store/provider-folder.ts:67`
  - body: `writeFileAtomic` and `copyFileAtomic` create `${backup.target}/snapshots/...` with recursive mkdir and then write/rename through that path. If a synced backup target contains `snapshots` or an installation directory as a symlink, snapshot payloads can be written outside the configured backup target; validate or create each directory segment without following symlinks before these writes.
  - confidence: 0.91
- **[P1] Validate workspace backup roots before mkdir:** `src/commands/workspace-backup.ts:350-351`
  - body: `workspaceBackupRunCommand` creates `mirrorRoot` and `stagingRoot` before proving the target-side workspace path is not a symlink, and the later containment check canonicalizes through `realpath`. A symlinked `backup.target/workspace` can redirect mirror writes and stale-mirror deletes outside the backup folder, so reject symlinked target segments before any mkdir/cp/rename/rm.
  - confidence: 0.9
- **[P1] Validate restore parents before rollback:** `src/commands/backup-restore.ts:442-443`
  - body: `applyRestoreOperations` renames an existing target into a rollback directory before `copySourceToTarget` validates the target parent with `ensureDirectoryPathWithoutSymlinks`. If the parent path traverses a symlink, restore can move user data outside the intended restore root before the later validation runs; validate the parent path before access, mkdtemp, or rename.
  - confidence: 0.88
