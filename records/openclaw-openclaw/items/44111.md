## Review Findings

Overall confidence: 0.86

Full review comments:

- **[P1] Reject symlinked workspace backup roots before mirroring:** `src/commands/workspace-backup.ts:350-351`
  - body: `workspaceBackupRunCommand` creates `backup.target/workspace/mirrors` and `.staging` before proving that `backup.target/workspace` and its child segments are real directories. If a synced/shared backup target contains `workspace -> <other directory>`, these mkdir/cp/rename/rm operations can run outside the configured backup target while the later canonical containment checks still pass because they follow the same symlink.
  - confidence: 0.91
- **[P1] Validate snapshot store directories before writing:** `src/backup/snapshot-store/provider-folder.ts:67`
  - body: `writeFileAtomic` and `copyFileAtomic` recursively create and then write under `targetDir/snapshots/<installationId>` without checking target-side directory segments for symlinks. A symlink planted at `snapshots` or the installation directory can redirect the encrypted payload/envelope writes outside `backup.target`; add no-symlink directory validation before mkdir/write/copy.
  - confidence: 0.9
- **[P1] Validate restore parents before rollback moves:** `src/commands/backup-restore.ts:442-443`
  - body: `applyRestoreOperations` moves an existing target into a rollback directory before `copySourceToTarget` validates that the target parent has no symlink segments. A symlinked parent can redirect the rollback `mkdtemp`/`rename` before the later copy path rejects it, so validate the target parent before `fs.access`/`moveTargetToRollbackPath` as well.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff introduces concrete filesystem security concerns around symlinkable backup targets and restore rollback paths.

Concerns:

- **[high] Workspace mirror can follow target-side symlinks:** `src/commands/workspace-backup.ts:350`
  - body: The mirror and staging directories are created before target-side symlink validation, so an attacker-controlled synced backup folder can redirect mirroring or stale cleanup outside the intended backup root.
  - confidence: 0.91
- **[high] Snapshot store can write through symlinked directories:** `src/backup/snapshot-store/provider-folder.ts:67`
  - body: Recursive mkdir and atomic rename/copy under `targetDir/snapshots` do not reject symlinked intermediate directories, allowing payload and envelope files to be written outside the configured backup target.
  - confidence: 0.9
- **[medium] Restore rollback move happens before parent validation:** `src/commands/backup-restore.ts:442`
  - body: Existing restore targets are renamed to rollback paths before the parent path is checked for symlink segments, which can disturb live files through a symlinked parent before restore fails.
  - confidence: 0.82
