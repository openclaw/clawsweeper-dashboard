## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Compare config paths before skipping restore:** `src/commands/backup-restore.ts:714-716`
  - body: This skips the config restore whenever the archived config and current OPENCLAW_CONFIG_PATH are both inside state, even if their relative paths differ. A full-host restore from state/openclaw.json onto a host using state/custom.json restores the old file but leaves the active config path stale or missing while reporting success.
  - confidence: 0.91
- **[P1] Compare OAuth paths before skipping restore:** `src/commands/backup-restore.ts:740-742`
  - body: The OAuth branch has the same state-contained shortcut, so a backup whose OAuth dir lived at one state-relative path will not populate a different current OPENCLAW_OAUTH_DIR under state. That can leave provider credentials missing after a successful full-host restore.
  - confidence: 0.9
- **[P1] Reject symlinked workspace target dirs before writes:** `src/commands/workspace-backup.ts:350-352`
  - body: The run path creates mirror and staging directories before proving backup.target/workspace is a real directory tree. If a synced backup target already contains workspace or mirrors as a symlink, mkdir/cp/rename/rm operate through that link while the later realpath containment checks still see the same canonical root.
  - confidence: 0.86
- **[P2] Reject symlinked snapshot store parents before writes:** `src/backup/snapshot-store/provider-folder.ts:66-85`
  - body: writeFileAtomic and copyFileAtomic mkdir the destination parent and then write/rename under it without checking intermediate target directories. A symlink planted under backup.target/snapshots can redirect encrypted payload or envelope writes outside the configured backup directory.
  - confidence: 0.84
- **[P2] Bound parent expansion during archive verification:** `src/commands/backup-verify.ts:337-346`
  - body: collectNestedArchiveParents expands every parent segment for each archive entry. The entry-count and total-path-byte caps still allow a single very deep path to allocate and process a huge chain of parent strings before manifest checks finish, so crafted archives can make verify/restore spend excessive CPU and memory.
  - confidence: 0.8
