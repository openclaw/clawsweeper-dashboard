## Summary

Keep open: current main only has the local archive create/verify flow, while this PR still carries useful native snapshot-backup work, but the PR head remains unsafe to merge because target-side symlink escapes and restore coverage gaps are source-reproducible.

## What This Changes

The PR adds `openclaw backup setup/run/status/list/restore`, folder-backed encrypted snapshots, workspace mirroring, backup config schema/docs, deprecated aliases, and backup/export hardening tests.

## Best Possible Solution

Refresh the branch and land a native backup implementation only after target-side no-symlink enforcement, source-vs-target restore coverage checks, the changelog entry, and focused backup validation are complete.

## Reproduction Assessment

Yes. Source inspection of the PR head shows the exact mkdir/write paths that can follow target-side symlinks and the full-host restore conditions that skip config/OAuth restoration for mismatched in-state relative paths.

## Solution Assessment

No for the PR as currently written. The native snapshot direction is plausible, but the implementation needs pre-write target path hardening, corrected restore coverage checks, and maintainer/security review before it is the narrow maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Reject symlinked workspace target directories before writes:** `src/commands/workspace-backup.ts:350-351`
  - body: `workspaceBackupRunCommand` creates `backup.target/workspace/mirrors` and `.staging` before rejecting symlinked target-side directories, and the later containment check resolves both paths through the same symlink. A synced or attacker-writable backup target can point `workspace` outside the backup directory, causing mirror writes and stale-mirror deletes to land outside `backup.target`.
  - confidence: 0.91
- **[P2] Reject symlinked snapshot store path segments:** `src/backup/snapshot-store/provider-folder.ts:66-85`
  - body: The folder store creates parent directories and writes through `targetDir/snapshots/<installationId>` without checking those intermediate directories for symlinks. If a synced backup target contains `snapshots` or an installation directory as a symlink, snapshot payload/envelope writes escape the configured target directory.
  - confidence: 0.88
- **[P2] Restore config when in-state paths differ:** `src/commands/backup-restore.ts:715-716`
  - body: This condition treats any target config path under the current `stateDir` as covered by the state asset, but it does not verify that the source and target in-state config locations are the same relative path. When they differ, full-host restore skips the config operation and leaves the active config missing or stale.
  - confidence: 0.88
- **[P2] Restore OAuth creds when in-state paths differ:** `src/commands/backup-restore.ts:741-742`
  - body: The credentials branch has the same coverage assumption: any target OAuth dir inside `stateDir` is treated as restored by the state asset. If the backup used a different in-state OAuth relative path, full-host restore completes without populating the active credentials directory.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `docs/cli/backup.md:14-28`
  - body: This PR adds user-facing backup commands and changes the preferred archive command from `create` to `export`, but the PR head does not include a `CHANGELOG.md` entry. Repo policy requires user-facing feature/fix changes to appear in the active changelog section.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: Security review needs attention because the PR still permits target-side symlink escapes in secret-bearing backup write paths.

Concerns:

- **[high] Workspace backup target symlink escape:** `src/commands/workspace-backup.ts:350`
  - body: `backup.target/workspace` can be replaced with a symlink before `backup run`; recursive mkdir/cp/rename/rm then operate through that symlink and can write or delete outside the selected backup directory.
  - confidence: 0.91
- **[medium] Snapshot store path segment symlink escape:** `src/backup/snapshot-store/provider-folder.ts:85`
  - body: Snapshot upload writes under `targetDir/snapshots/<installationId>` without rejecting symlinked intermediate directories, so a tampered backup folder can redirect encrypted payload and envelope writes outside the configured target.
  - confidence: 0.88
- **[medium] OAuth credential restore can be skipped:** `src/commands/backup-restore.ts:741`
  - body: Full-host restore may skip the credentials operation when both source and target OAuth directories are inside state but at different relative paths, leaving active provider credentials absent after a reported successful restore.
  - confidence: 0.86
