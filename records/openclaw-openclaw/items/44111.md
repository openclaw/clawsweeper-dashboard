Full review comments:

- **[P1] Reject symlinked workspace backup roots before writes:** `src/commands/workspace-backup.ts:350-351`
  - body: `workspaceBackupRunCommand` creates `backup.target/workspace/mirrors` and `.staging` before rejecting target-side symlink segments, and the later containment check canonicalizes through the same symlinked root. In a synced/shared backup target, `backup.target/workspace` can point outside the backup folder, causing mirror writes, replaces, or stale cleanup to operate outside the configured target.
  - confidence: 0.94
- **[P1] Validate snapshot store directories before writes:** `src/backup/snapshot-store/provider-folder.ts:67-85`
  - body: `writeFileAtomic` and `copyFileAtomic` create and rename under `targetDir/snapshots/<installationId>` without checking intermediate segments for symlinks. A symlink planted at `snapshots` or the installation directory can redirect payload/envelope writes outside `backup.target`, so the store needs a no-symlink directory creation helper and regression coverage.
  - confidence: 0.93
