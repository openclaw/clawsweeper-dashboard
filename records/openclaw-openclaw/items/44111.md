## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Reject symlinked snapshot directories before writes:** `src/backup/snapshot-store/provider-folder.ts:67`
  - body: `uploadSnapshot` reaches `copyFileAtomic`/`writeFileAtomic`, which first creates `targetDir/snapshots/<installationId>` with recursive mkdir and then writes/renames through that path. A synced or shared backup target can pre-place `snapshots` or the installation directory as a symlink, redirecting encrypted payload or envelope writes outside backup.target, so the store needs segment-by-segment no-symlink directory creation/validation before any write.
  - confidence: 0.9
- **[P1] Reject symlinked workspace target directories before writes:** `src/commands/workspace-backup.ts:350-351`
  - body: `workspaceBackupRunCommand` creates `mirrorRoot` and `stagingRoot` before proving those target-side segments are real directories. If `backup.target/workspace` is a symlink, the later realpath-based containment checks can still pass while mirror writes or stale mirror deletion happen in the symlink target, outside the selected backup directory.
  - confidence: 0.9
- **[P2] Remove or implement backup retention config:** `src/config/zod-schema.backup.ts:25`
  - body: The schema accepts `backup.retention.keepDaily`, `keepWeekly`, `keepMonthly`, and `maxSnapshots`, but the branch does not prune snapshots or otherwise consume those values. That creates a public config source of truth that silently does nothing, which is a compatibility trap for a backup feature.
  - confidence: 0.87
- **[P2] Restore config when in-state target paths differ:** `src/commands/backup-restore.ts:715`
  - body: This condition skips a standalone config restore whenever the source config was covered by the state asset and the current config path is somewhere inside `stateDir`. If the backup used one in-state config path and the restore host uses a different in-state `OPENCLAW_CONFIG_PATH`, full-host restore reports success but leaves the active config path stale or missing.
  - confidence: 0.82
- **[P2] Restore OAuth when in-state target paths differ:** `src/commands/backup-restore.ts:741`
  - body: The credentials branch has the same source-covered-by-state shortcut: it suppresses the credentials restore for any current OAuth directory inside `stateDir` without confirming the source and target relative paths match. A restore host with a different in-state `OPENCLAW_OAUTH_DIR` can come up without the expected active credentials.
  - confidence: 0.82
- **[P2] Recheck archive entry types during tar creation:** `src/infra/backup-create.ts:406`
  - body: The export path pre-scans sources for symlinks and special files, then calls `tar.c` without a filter or entry-type guard. A source tree that changes after the pre-scan can still produce an archive with symlink/FIFO/device entries that OpenClaw later rejects or that unsafe external extractors may mishandle, so the pack step should enforce the same policy as it writes each entry.
  - confidence: 0.78
