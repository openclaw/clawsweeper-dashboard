  - confidence: 0.92

  - body: `workspaceBackupRunCommand` creates `backup.target/workspace/mirrors` and `.staging` before rejecting target-side symlinks, and the later containment checks canonicalize through symlinks. A symlinked `backup.target/workspace` can make the mirror writes and stale cleanup operate outside the backup target, so validate the target directory segments before any mkdir/cp/rm.
  - confidence: 0.91

  - body: The schema accepts `backup.retention` (`keepDaily`, `keepWeekly`, `keepMonthly`, `maxSnapshots`), but the PR does not read those fields or prune snapshots.
