## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Reject snapshot-store symlinked directories before writes:** `src/backup/snapshot-store/provider-folder.ts:67`
  - body: `writeFileAtomic` creates the destination directory and writes/renames through it without verifying that `targetDir/snapshots` and the installation directory are real directories. In a synced or shared backup target, a symlinked intermediate directory can redirect encrypted payload or envelope writes outside `backup.target`, so the store needs a no-symlink directory walk before every write/copy/read path.
  - confidence: 0.9
- **[P1] Reject workspace target symlinks before mirroring:** `src/commands/workspace-backup.ts:350-351`
  - body: `workspaceBackupRunCommand` creates `backup.target/workspace/mirrors` and `.staging` before rejecting target-side symlink segments, and the later realpath-based containment checks can follow the same symlinked root. A user-controlled backup target can therefore redirect mirror writes or stale mirror deletion outside the configured backup folder.
  - confidence: 0.9
- **[P2] Remove or implement backup retention config:** `src/config/zod-schema.backup.ts:25`
  - body: The schema accepts `backup.retention` keys, but the PR does not implement snapshot pruning or read these values in the backup run/list/status/restore paths. Validating a public config block that is silently ignored creates a compatibility contract users can rely on without getting the promised retention behavior.
  - confidence: 0.86
