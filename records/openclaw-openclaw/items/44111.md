Full review comments:

- **[P1] Reject symlinked snapshot-store directories before writes:** `src/backup/snapshot-store/provider-folder.ts:67`
  - body: `writeFileAtomic` and `copyFileAtomic` create and write under `backup.target/snapshots/...` without proving intermediate target directories are real directories. A shared or cloud-synced backup target can plant `snapshots` or an installation directory as a symlink and redirect payload/envelope writes outside the configured backup target.
  - confidence: 0.93
- **[P1] Reject symlinked workspace target directories before writes:** `src/commands/workspace-backup.ts:350-351`
  - body: `workspaceBackupRunCommand` creates the mirror and staging roots before target-side no-symlink validation, and the later containment check follows realpaths. A symlinked `backup.target/workspace` can redirect mirror writes and stale-mirror removal outside the backup folder.
  - confidence: 0.93
- **[P1] Remove or implement backup retention config:** `src/config/zod-schema.backup.ts:25`
  - body: The public schema accepts `backup.retention.*`, but the backup/snapshot command paths never prune snapshots or apply those settings. Accepting ignored retention settings creates a user-visible config contract without the promised storage behavior.
  - confidence: 0.88
- **[P2] Bound archive parent expansion during verification:** `src/commands/backup-verify.ts:337-344`
  - body: `collectNestedArchiveParents` expands every parent directory for every archive entry. A crafted tar path with many tiny segments can stay under the total path-byte cap while forcing excessive Set growth during `backup verify` and `backup restore`.
  - confidence: 0.86
- **[P2] Recheck archive entry types while packing:** `src/infra/backup-create.ts:365`
  - body: The new pre-scan rejects symlinks and special files only before `tar.c` walks the source tree. A source path swapped after the scan can still be packed as a blocked tar entry, producing a snapshot that later fails verification or is unsafe with external extractors.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The diff adds security-sensitive backup/restore writes and still has concrete target-side symlink traversal and archive-processing concerns.

Concerns:

- **[high] Snapshot store follows target-side symlink directories:** `src/backup/snapshot-store/provider-folder.ts:67`
  - body: The folder store writes payload and envelope files after recursive mkdir without checking intermediate target directories for symlinks, so an attacker with write access to the backup target can redirect writes outside backup.target.
  - confidence: 0.93
- **[high] Workspace mirror follows target-side symlink directories:** `src/commands/workspace-backup.ts:350`
  - body: Workspace mirroring creates target directories before symlink rejection and relies on realpath containment afterward, which can accept a symlinked workspace backup root and write or remove files outside the intended backup tree.
  - confidence: 0.93
- **[medium] Archive verification can expand deep path parents without a depth cap:** `src/commands/backup-verify.ts:337`
  - body: A crafted backup archive can use very deep relative paths to force large parent-set expansion during verify/restore despite the total path-byte cap.
  - confidence: 0.86
- **[medium] Archive creation relies on a pre-scan for mutable source trees:** `src/infra/backup-create.ts:365`
  - body: The added symlink/special-file checks run before tar packing, so a concurrently changed source tree can still produce an archive containing entries that the new verify/restore path rejects.
  - confidence: 0.78
