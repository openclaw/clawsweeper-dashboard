## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Validate snapshot directory segments before writing:** `src/backup/snapshot-store/provider-folder.ts:67`
  - body: `resolveSnapshotStoreConfig` rejects the top-level target, but `uploadSnapshot` still writes through `targetDir/snapshots/<installationId>` after `fs.mkdir(..., { recursive: true })`. In a synced/shared backup folder, an attacker can replace `snapshots` or the installation directory with a symlink and make the payload/envelope write land outside the configured backup target, so the store needs no-symlink segment validation before mkdir/rename/copy.
  - confidence: 0.91
- **[P1] Reject symlinked workspace target roots before writes:** `src/commands/workspace-backup.ts:350-351`
  - body: The workspace mirror path creates `mirrorRoot` and `stagingRoot` before checking containment, and the containment helper follows symlinks. If `backup.target/workspace`, `mirrors`, or `.staging` is swapped to a symlink inside a cloud-synced target, the later copy/rename/rm operations can write or delete outside the backup directory.
  - confidence: 0.9
