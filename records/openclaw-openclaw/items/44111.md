  - body: `copyFileAtomic` creates the destination parent with recursive `mkdir` and then writes/renames under it without rejecting symlinked path segments inside `backup.target`.   - confidence: 0.93
- **[P1] Reject symlinked workspace target dirs before writes:** `src/commands/workspace-backup.ts:350-351`
  - body: `workspaceBackupRunCommand` creates `backup.target/workspace/mirrors` and `.staging` before the second safety checks, and the later containment check canonicalizes through symlinks. A symlinked target-side workspace directory can redirect mirror writes or stale mirror deletion outside the selected backup target.
  - confidence: 0.9
- **[P1] Validate restore parents before rollback renames:** `src/commands/backup-restore.ts:443`
  - body: Existing targets are moved to rollback before `copySourceToTarget` validates the target parent with `ensureDirectoryPathWithoutSymlinks`. If a restore target parent contains a symlink segment, the rollback `mkdtemp`/`rename` can already operate outside the intended restore boundary before the later guard runs.
  - confidence: 0.86
- **[P1] Remove or implement backup retention config:** `src/config/zod-schema.backup.ts:25`
  - body: The PR accepts `backup.retention.keepDaily/keepWeekly/keepMonthly/maxSnapshots`, but the diff does not implement snapshot pruning. Users can set a validated retention policy that is silently ignored, which is especially risky for encrypted backups in synced folders where retention may be a data-handling expectation.
  - confidence: 0.91
