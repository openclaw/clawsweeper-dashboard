  - body: `writeFileAtomic` and `copyFileAtomic` create and rename through `backup.target/snapshots/<installationId>` without rejecting symlinked directory segments.   - body: `workspaceBackupRunCommand` creates `mirrorRoot` and `.staging` before target-side symlink checks, and the containment helper follows `realpath`. If `backup.target/workspace` is a symlink, the later copy, rename, and stale-mirror removal can operate outside the selected backup folder.
  - confidence: 0.9

  - body: The schema accepts `backup.retention`, but the PR diff has no code path that reads it to prune or cap snapshots. Users can configure `maxSnapshots` or daily/weekly/monthly retention and still accumulate every snapshot, so this config surface should be removed or implemented with tests before merge.
  - confidence: 0.95
- **[P2] Re-check tar entry types during archive creation:** `src/infra/backup-create.ts:406-420`
  - body: The new pre-scan rejects symlinks and special files before `tar.c`, but a writable source tree can swap an entry after the scan and before tar writes it.   - confidence: 0.82
