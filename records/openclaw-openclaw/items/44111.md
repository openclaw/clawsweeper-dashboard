  - body: `writeFileAtomic` and `copyFileAtomic` create parent directories and then write through `backup.target/snapshots/...` without checking that `snapshots` or the installation directory is a real directory.   - confidence: 0.9

  - body: `workspaceBackupRunCommand` creates `backup.target/workspace/mirrors` and `.staging` before rejecting symlinked target-side directories. Because the later containment checks canonicalize through the same symlink, `fs.mkdir`/`fs.cp`/`fs.rename` can operate outside the configured backup folder.
  - confidence: 0.9
