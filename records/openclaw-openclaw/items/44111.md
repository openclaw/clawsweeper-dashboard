## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Validate snapshot target directories before writing:** `src/backup/snapshot-store/provider-folder.ts:85`
  - body: `copyFileAtomic` creates the destination parent and writes a temp file under `backup.target` without rejecting symlinked directory segments such as `snapshots` or the installation id. In a shared or cloud-synced backup folder, a planted symlink can redirect payload/envelope writes outside the configured backup target, so upload/download paths need a target-side no-symlink directory walk or fd-relative writes before any mkdir/write/rename.
  - confidence: 0.9
- **[P1] Reject target-side symlinks before mirroring:** `src/commands/workspace-backup.ts:350-351`
  - body: `workspaceBackupRunCommand` creates `mirrorRoot` and `.staging` before proving the target tree is free of symlinked segments, and the later containment check resolves through symlinks. If `backup.target/workspace` already points elsewhere, the mirror update can create, rename, or remove directories outside the backup folder.
  - confidence: 0.91
- **[P1] Remove or implement retention config:** `src/config/zod-schema.backup.ts:25`
  - body: The schema accepts `backup.retention` keys, but no runtime path reads them or prunes snapshots. Users who configure `maxSnapshots` or daily/weekly/monthly retention would get successful config validation with no retention behavior, so either implement pruning with tests or leave the config surface out of this PR.
  - confidence: 0.86
