Full review comments:

- **[P1] Reject target symlinks before mirror writes:** `src/commands/workspace-backup.ts:350-353`
  - body: `backup.target/workspace` can already be a symlink before this code runs. Because the code creates `mirrors` and `.staging` first and then canonicalizes containment through that symlink, subsequent copy/rename/stale-delete operations can write or delete outside the configured backup target. Validate every existing target-side segment as a real directory before any write under it.
  - confidence: 0.94
- **[P1] Validate snapshot directories before writing:** `src/backup/snapshot-store/provider-folder.ts:66-85`
  - body: `writeFileAtomic` and `copyFileAtomic` create `path.dirname(...)` recursively and then rename/copy into it without checking whether `targetDir/snapshots` or the installation directory is a symlink. A synced or shared backup target can redirect encrypted payload/envelope writes outside `backup.target`; ensure each directory segment inside the snapshot store is a non-symlink directory before writing or copying files.
  - confidence: 0.93
