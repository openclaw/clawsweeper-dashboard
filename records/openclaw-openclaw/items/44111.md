## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Reject symlinked snapshot store directories:** `src/backup/snapshot-store/provider-folder.ts:67-85`
  - body: `uploadSnapshot` writes below `targetDir/snapshots/<installationId>` after `fs.mkdir(..., { recursive: true })` without rejecting symlinked intermediate directories. In a synced or shared backup target, an attacker can plant `snapshots` or the installation directory as a symlink and make OpenClaw write payload/envelope files outside the configured backup target.
  - confidence: 0.88
- **[P1] Validate workspace backup roots before mkdir:** `src/commands/workspace-backup.ts:350-352`
  - body: `workspaceBackupRunCommand` creates `backup.target/workspace/mirrors` and `.staging` before validating the target-side tree, and the later containment check follows symlinks. A preexisting `backup.target/workspace` symlink can redirect mirror/staging writes and stale cleanup outside the backup folder.
  - confidence: 0.86
- **[P1] Validate restore targets before rollback:** `src/commands/backup-restore.ts:442-443`
  - body: Existing restore targets are probed and renamed before `copySourceToTarget` checks for symlinked parent segments. If a configured state/config/OAuth/workspace target contains a symlinked parent, rollback staging can move or replace files through that symlink before the safety check runs.
  - confidence: 0.8
