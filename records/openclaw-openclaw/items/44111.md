## Summary

Keep open. Current main still exposes only local backup archive creation/verification, so this PR is not obsolete, but the current head is not merge-ready because full-host restore can skip active config/OAuth recovery when source and target in-state relative paths differ.

## What This Changes

The PR adds folder-backed encrypted snapshot backups with setup/run/status/list/restore commands, workspace mirroring, backup config/schema/docs, alias updates, and restore/snapshot hardening coverage.

## Best Possible Solution

If native encrypted snapshots remain desired, refresh the branch, fix the relative-path restore coverage, add the missing changelog entry, and land only after focused backup tests plus the changed gate pass.

## Reproduction Assessment

Yes. The current-main absence is reproducible by inspecting CLI registration/docs/schema, and the PR defects are reproducible from a full-host restore where source and target config or OAuth directories are both inside state but at different relative paths.

## Solution Assessment

No for the current PR head as-is. The direction is plausible, but the restore coverage checks should compare source and target relative paths before treating the state asset as covering active config/OAuth paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Restore config when in-state paths differ:** `src/commands/backup-restore.ts:715-716`
  - body: This condition suppresses the config restore whenever the source config was covered by state and the current config path is also inside state. If the backup used one in-state config relative path and the restore host uses another, replacing state restores the old path and leaves the active `OPENCLAW_CONFIG_PATH` missing or stale. Compare source and target relative paths, or enqueue the config operation when they differ.
  - confidence: 0.88
- **[P2] Restore OAuth creds when in-state paths differ:** `src/commands/backup-restore.ts:741-742`
  - body: The credentials branch has the same coverage assumption: any target OAuth dir inside `stateDir` is treated as covered by the state restore. When the backup's OAuth dir lived at a different relative path inside state, full-host restore completes without populating the active OAuth directory, so provider auth data is not recovered.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `docs/cli/backup.md:18-28`
  - body: This PR adds user-facing backup commands and changes the preferred archive command from `create` to `export`, but it does not include a `CHANGELOG.md` entry. Repo policy requires user-facing feature/fix changes to appear in the active changelog section, so release notes would otherwise omit the new backup flow.
  - confidence: 0.82
