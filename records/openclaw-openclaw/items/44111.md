Full review comments:

- **[P1] Reject symlinked snapshot store directories:** `src/backup/snapshot-store/provider-folder.ts:67-85`
  - body: `writeFileAtomic` and `copyFileAtomic` create the snapshot parent directory and then write/rename through `backup.target/snapshots/...` without checking that `snapshots` or the installation directory is a real directory. A symlink planted in a synced backup target can redirect snapshot payload or envelope writes outside the configured backup directory.
  - confidence: 0.92
- **[P1] Validate workspace backup roots before mkdir:** `src/commands/workspace-backup.ts:350-352`
  - body: `workspaceBackupRunCommand` creates `backup.target/workspace/mirrors` and `.staging` before rejecting a symlinked `backup.target/workspace`. Since `fs.mkdir` follows that symlink, the command can create directories outside the backup target before validation runs.
  - confidence: 0.91
- **[P1] Walk every existing restore path segment:** `src/commands/backup-restore.ts:337-344`
  - body: `ensureDirectoryPathWithoutSymlinks` stops as soon as `lstat(probe)` succeeds on the full target parent, so an existing parent like `backup-link/restored` can pass even when `backup-link` is a symlink. Restore then creates temp targets and renames through that symlinked path, defeating the intended restore containment guard.
  - confidence: 0.84
