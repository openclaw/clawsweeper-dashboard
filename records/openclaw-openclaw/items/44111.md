Full review comments:

- **[P1] Reject symlinked snapshot-store directories before writing:** `src/backup/snapshot-store/provider-folder.ts:67`
  - body: `writeFileAtomic` and `copyFileAtomic` create the destination parent with recursive `mkdir` and then write/rename through it. Because only the top-level `backup.target` is rejected as a symlink, a synced target can contain `snapshots` or an installation directory as a symlink and redirect payload/envelope writes outside the configured backup directory.
  - confidence: 0.91
- **[P1] Validate workspace backup target directories before mkdir:** `src/commands/workspace-backup.ts:350-351`
  - body: `workspaceBackupRunCommand` creates `backup.target/workspace/mirrors` and `.staging` before any target-side no-symlink validation, and later containment checks canonicalize through `realpath`. A pre-existing symlinked `workspace` subtree can therefore redirect mirror writes/removes outside `backup.target`; walk and create the target-side segments with `lstat` checks before these writes.
  - confidence: 0.9
- **[P2] Remove or implement backup retention config:** `src/config/zod-schema.backup.ts:25`
  - body: The PR accepts `backup.retention.*` in the public config schema, but the branch never reads those values to prune snapshots. Users can configure retention and assume old encrypted snapshots are removed while they accumulate, so either remove the schema until pruning exists or implement and test pruning behavior.
  - confidence: 0.86
