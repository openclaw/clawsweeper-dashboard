Full review comments:

- **[P1] Reject symlinked snapshot directories before writing:** `src/backup/snapshot-store/provider-folder.ts:67`
  - body: `resolveValidatedTargetDir` only rejects a symlink at the top-level target, but `uploadSnapshot` then creates and writes under `targetDir/snapshots/<installationId>` without checking those directory segments. A synced/shared backup folder can pre-create `snapshots` or the installation directory as a symlink, causing the payload/envelope rename to write outside `backup.target`.
  - confidence: 0.92
- **[P1] Reject workspace target symlinks before mkdir:** `src/commands/workspace-backup.ts:350-351`
  - body: `workspaceBackupRunCommand` creates `backup.target/workspace/mirrors` and `.staging` before rejecting target-side symlinks, and the later containment check follows `realpath` for both root and child. If `backup.target/workspace` is a symlink, the mirror/staging writes and stale cleanup can happen outside the backup directory while still passing containment.
  - confidence: 0.94
- **[P2] Remove or implement backup retention config:** `src/config/zod-schema.backup.ts:25`
  - body: The schema accepts `backup.retention` values, but the diff contains no pruning/list/delete behavior that reads them. That makes a valid config silently promise retention controls that do not run, which is unsafe for an operator-facing backup feature.
  - confidence: 0.88
