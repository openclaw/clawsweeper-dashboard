Full review comments:

- **[P1] Reject symlinked snapshot directories before writes:** `src/backup/snapshot-store/provider-folder.ts:67`
  - body: `writeFileAtomic` and `copyFileAtomic` create and then write through `backup.target/snapshots/<installationId>` without checking each existing target-side segment with `lstat`. In a synced or shared backup folder, a planted `snapshots` or installation-id symlink can redirect encrypted payload/envelope writes outside the configured backup target.
  - confidence: 0.92
- **[P1] Reject symlinked workspace target dirs before mirroring:** `src/commands/workspace-backup.ts:350-351`
  - body: `workspaceBackupRunCommand` creates `mirrorRoot` and `.staging` before rejecting target-side symlinks, and the later containment checks canonicalize through symlinks. A symlinked `backup.target/workspace` can therefore redirect mirror writes, renames, and stale cleanup outside the selected backup target.
  - confidence: 0.9
- **[P2] Remove or implement backup retention config:** `src/config/zod-schema.backup.ts:25`
  - body: The schema accepts `backup.retention.keepDaily/keepWeekly/keepMonthly/maxSnapshots`, but the diff only adds those names to types/schema and no runtime path reads them. Users can configure retention and believe snapshots are pruned when the backup store never enforces it.
  - confidence: 0.89
