## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Reject symlinked snapshot-store directories before writes:** `src/backup/snapshot-store/provider-folder.ts:67`
  - body: `backup.target` is a user-selected folder and may be cloud-synced or otherwise writable by another process. `writeFileAtomic` and `copyFileAtomic` create and write through `targetDir/snapshots/<installationId>` without walking those target-side directory segments with `lstat`, so a symlink at `snapshots` or the installation directory can redirect encrypted payload/envelope writes outside the backup target.
  - confidence: 0.9
- **[P1] Reject symlinked workspace mirror directories before writes:** `src/commands/workspace-backup.ts:350-351`
  - body: This creates `backup.target/workspace/mirrors` and `.staging` before rejecting symlinked target-side segments, and the later containment checks canonicalize through symlinks. A symlinked `workspace` directory in the backup target can redirect mirror writes or cleanup outside backup.target, so validate the target directory chain before any mkdir/cp/rename/rm.
  - confidence: 0.88
- **[P2] Remove or implement retention config:** `src/config/zod-schema.backup.ts:25`
  - body: The schema accepts `backup.retention` keys, but the snapshot store, run, list, status, and setup paths do not read them. Users can configure retention and assume snapshot pruning is active when it is not, so either implement pruning with focused tests or leave these keys out until behavior exists.
  - confidence: 0.86
