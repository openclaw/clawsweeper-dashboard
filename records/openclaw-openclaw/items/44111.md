## Summary

Keep open: the native snapshot direction is useful, but source inspection at the PR head still finds unsafe backup target writes, restore mapping gaps, and an archive-time race, and the PR body lacks after-fix real CLI proof.

## What This Changes

The PR adds folder-backed encrypted backup snapshots, workspace mirroring, backup setup/run/status/list/restore commands, backup config schema/docs, aliases, and hardening tests.

## Best Possible Solution

Land the native encrypted snapshot flow only after target path guards, restore mapping, archive-time validation, and redacted real CLI smoke proof are in place; keep off-machine provider backup as complementary ClawHub/plugin work.

## Reproduction Assessment

Yes for the review findings: the unsafe write, restore mapping, and archive race paths are visible in the PR head source. I did not execute the branch because this review is read-only, and the real end-to-end CLI flow still lacks proof.

## Solution Assessment

No. The feature direction is reasonable, but this is not the best mergeable implementation until it rejects symlinked target segments before writes, restores changed in-state config/OAuth paths, guards tar entries at archive time, and includes real behavior proof.

## Review Findings

Overall correctness: not a patch

Overall confidence: 0.87

Full review comments:

- **[P1] Reject symlinked snapshot store directories:** `src/backup/snapshot-store/provider-folder.ts:67-85`
  - body: `writeFileAtomic` and `copyFileAtomic` create parent directories and then rename/copy through `backup.target/snapshots/...` without checking that `snapshots` or the installation directory is a real directory. A symlink planted inside a synced backup target can redirect snapshot payload/envelope writes outside the configured backup directory.
  - confidence: 0.88
- **[P1] Validate workspace backup roots before mkdir:** `src/commands/workspace-backup.ts:350-351`
  - body: `workspaceBackupRunCommand` creates `backup.target/workspace/mirrors` and `.staging` before the later safety checks can reject a symlinked `backup.target/workspace`. `fs.mkdir` follows that symlink, so the command can create or update directories outside the backup target before it fails.
  - confidence: 0.86
- **[P2] Restore configs when in-state paths differ:** `src/commands/backup-restore.ts:714-716`
  - body: Full-host restore skips the config operation whenever the target config path is inside the current state dir and the source config was covered by the old state dir. If the archive had `state/openclaw.json` but the restore host uses `state/custom.json`, the active config path is left missing or stale after a reported full-host restore.
  - confidence: 0.84
- **[P2] Restore OAuth dirs when in-state paths differ:** `src/commands/backup-restore.ts:740-742`
  - body: The credentials branch makes the same covered-by-state assumption without comparing source and target relative paths. Restoring from one in-state OAuth location onto a host configured for another in-state OAuth location leaves the active credentials directory missing or stale.
  - confidence: 0.82
- **[P2] Validate tar entries while archiving:** `src/infra/backup-create.ts:406-420`
  - body: The export path scans source trees before calling `tar.c`, but the actual tar write has no filter or entry-type check. A source tree can be swapped after the pre-scan so `backup export` emits symlink or special-file entries that restore later rejects, or that are unsafe for external extractors.
  - confidence: 0.76

## Security Review

Status: not_applicable

Summary: Needs attention: the PR adds backup write/restore surfaces and still has symlink traversal and archive-race concerns.

Concerns:

- **[high] Snapshot store follows symlinked target segments:** `src/backup/snapshot-store/provider-folder.ts:67`
  - body: Snapshot uploads create parent directories and then copy/rename files under `targetDir/snapshots/...` without validating intermediate path segments, so a symlink planted inside a synced backup target can redirect writes outside that target.
  - confidence: 0.88
- **[high] Workspace mirror writes before target symlink rejection:** `src/commands/workspace-backup.ts:350`
  - body: The workspace mirror path is created before the safety check revalidates the backup workspace root, allowing a symlinked `backup.target/workspace` to be followed before failure.
  - confidence: 0.86
- **[medium] Archive creation relies on a pre-scan before tar writes:** `src/infra/backup-create.ts:406`
  - body: The export path scans sources for symlinks/special files before invoking `tar.c`, but the tar write itself lacks an entry-type guard, leaving a race where a source tree change can produce unsafe or unrestorable archive entries.
  - confidence: 0.76
