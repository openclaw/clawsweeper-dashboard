## Summary

Keep open. Current main still only ships local archive backup creation and verification, so this PR is not obsolete; however, the branch is an XL restore/encryption implementation with concrete unresolved restore edge cases and needs maintainer/security review before merge.

## What This Changes

This PR adds folder-backed encrypted snapshot backups with `backup setup/run/status/list/restore`, workspace mirroring, backup config/schema/docs, alias updates, and restore/snapshot hardening tests.

## Best Possible Solution

If native encrypted snapshots are still desired, rebase or replace this branch, fix the remaining relative-path restore coverage gaps, and land it only after focused backup tests plus the broad changed gate pass. If maintainers decide snapshots belong outside core, close with that product decision and keep the existing local archive flow unchanged.

## Reproduction Assessment

Yes. Current-main absence is reproducible by inspecting the CLI registration/docs/config schema; the PR restore findings are reproducible from the branch diff with a full-host restore where source and target config or OAuth directories are both inside state but at different relative paths.

## Solution Assessment

No for the current PR head as-is. The direction is plausible, but the restore coverage logic should compare source-vs-target relative paths before relying on the state asset, and the branch needs fresh validation after rebase.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Restore config when in-state paths differ:** `src/commands/backup-restore.ts:663-665`
  - body: This condition suppresses the config restore whenever the current config path is inside `stateDir`. If the archive's config was also inside state but at a different relative path, replacing the state asset restores the old path and leaves the active `OPENCLAW_CONFIG_PATH` missing or stale. Compare the source and target relative paths, or enqueue the config operation when they differ.
  - confidence: 0.88
- **[P2] Restore OAuth creds when in-state paths differ:** `src/commands/backup-restore.ts:688-690`
  - body: The credentials branch has the same coverage assumption: any in-state target OAuth dir is treated as covered by the state restore. When the backup's OAuth dir lived at a different relative path inside state, full-host restore completes without populating the active OAuth directory, so provider auth data is not recovered.
  - confidence: 0.86
