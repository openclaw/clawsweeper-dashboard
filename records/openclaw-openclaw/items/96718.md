## Summary

The branch filters top-level null entries before `runtime.callTool(...)` in bundle-MCP materialization and carries a small test config-path cleanup.

## Best Possible Solution

Preserve explicit JSON null through validation and MCP dispatch by default, or add a schema-aware omission rule only for optional default-null fields with regression coverage and redacted real MCP proof.

## Solution Assessment

No. Unconditional post-validation null stripping is a plausible workaround, but the safer fix is to preserve explicit JSON null or make omission narrowly schema-aware after proving where null becomes an empty string.
