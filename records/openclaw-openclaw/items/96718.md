## Summary

Keep open: the PR targets a plausible MCP bug, but the current fix strips every top-level null argument without consulting the tool schema, which can break existing MCP tools that intentionally accept or require null, and the PR has no real after-fix MCP proof.

## What This Changes

The PR changes bundled MCP tool execution to remove top-level null-valued entries from tool input before calling the session MCP runtime.

## Best Possible Solution

Make MCP null normalization schema-aware: omit null only for optional fields where absence is equivalent, preserve required or semantically nullable fields, add regression coverage, and include redacted real MCP proof.

## Reproduction Assessment

No high-confidence live reproduction was established in this review; the linked issue includes detailed logs, but the raw MCP wire payload was not captured and I did not run a real server. The PR-introduced regression is source-reproducible because the diff drops all top-level nulls regardless of schema.

## Solution Assessment

No. The materialization boundary is a plausible place to normalize arguments, but the current implementation is too broad; the safer fix is schema-aware normalization or a narrower repair at the exact coercion point that turns null into the server-observed behavior.
