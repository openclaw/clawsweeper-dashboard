## What This Changes

The PR adds an own-line `NO_REPLY` helper, applies it across reply normalization, directive parsing, cron delivery/subagent followup, gateway chat streaming/history filters, and adds unit coverage.

## Best Possible Solution

## Solution Assessment

No, not as-is. The own-line suppression direction is plausible, but the branch needs the inline test fixed, channelData/media semantics preserved, and the chosen policy reconciled with the related open alternatives before merge.
