- **[P2] Preserve channelData before marking directives silent:** `src/auto-reply/reply/reply-directives.ts:34`
  - body: Marking own-line NO_REPLY as silent here lets createOutboundPayloadPlan treat the payload as silent when there is no media, and that planner drops silent entries before considering channelData.   - confidence: 0.88

  - body: This regex has no `i` flag, while the existing exact-token helper and docs accept `no_reply` as equivalent to `NO_REPLY`. After this change, `text\nno_reply` would still leak through the new own-line paths even though clean lowercase silent tokens are suppressed elsewhere.
  - confidence: 0.86
- **[P2] Fix the inline NO_REPLY parser test mismatch:** `src/auto-reply/reply/reply-directives.test.ts:33-35`
  - body: The new test expects parseReplyDirectives to remove inline `NO_REPLY`, but the implementation only clears text for exact or own-line silent tokens and never calls stripSilentToken for inline content. As written, the test should fail with the original `😄 NO_REPLY` text still present.
