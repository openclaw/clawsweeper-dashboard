## Summary

Current main still has the reported Dashboard/CLI mismatch: the Gateway treats a managed-service update handoff as a successful control-plane start, but the Control UI renders the same response as a skipped update. This should stay open as a narrow Dashboard update-state bug.

## What This Changes

Issue reports that Dashboard update from OpenClaw 2026.5.22 skipped with managed-service-handoff-started while the CLI update succeeded to 2026.5.27.

## Best Possible Solution

Keep the issue open and make the Dashboard treat a started managed-service handoff as a pending update, while preserving real skipped/error reasons as actionable failures.

## Reproduction Assessment

Yes, source-reproducible: current main returns ok=true with result.status=skipped for a managed-service handoff, and the Dashboard code renders skipped whenever result.status is skipped. I did not run a live update because this review is read-only.

## Solution Assessment

Yes, the best fix is narrow: align Control UI state handling with the Gateway contract for started handoffs and keep the existing post-restart status verification path. Changing CLI updater semantics is not necessary for the reported mismatch.
