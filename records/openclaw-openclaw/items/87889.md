## Repair Work Prompt

Fix the Dashboard update flow for package-manager `update.run` managed-service handoff responses from https://github.com/openclaw/openclaw/issues/87889. The Gateway and docs define `ok: true`, `result.status: "skipped"`, `result.reason: "managed-service-handoff-started"`, and `handoff.status: "started"` as successful handoff initiation; the UI should not display `Update skipped` for that response. Update `ui/src/ui/controllers/config.ts` and focused tests so the Dashboard enters a pending handoff/update verification state, uses `update.status` after reconnect to surface the final result, and still shows actionable errors for unavailable/failed handoffs and version mismatches. Do not change the Gateway control-plane response contract without maintainer approval. Do not edit `CHANGELOG.md`; include release-note context in the PR body if needed.
