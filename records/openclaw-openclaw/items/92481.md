## Summary

This is a valid missing capability, not a duplicate of the linked cron-delivery or heartbeat bugs. Telegram topic delivery and a generic cron outbound pipeline already exist, but cron cannot currently feed static text into that pipeline without an agent turn; maintainers should first approve the permanent payload semantics.

## What This Changes

Add direct static-text cron delivery to a configured Telegram topic without running an agent turn.

## Best Possible Solution

Add a dedicated channel-neutral static-notification payload that bypasses agent initialization and reuses the existing cron completion-delivery pipeline, including `threadId`, webhook, idempotency, retry, failure-reporting, and run-log behavior.

## Reproduction Assessment

Not applicable as a bug reproduction: source and shipped-release inspection confirm a missing payload capability rather than a failure of documented behavior.

## Solution Assessment

No. Direct delivery is the right outcome, but a dedicated closed payload kind is clearer and more maintainable than overloading the session-oriented `systemEvent` contract.
