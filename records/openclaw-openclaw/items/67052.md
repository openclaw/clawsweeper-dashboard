## Summary

Keep open. Current `main` and latest shipped TUI still use watchdog and stale-terminal recovery mitigations, but they do not provide the requested prompt content-complete or finalizing transition after visible assistant content stops.

## What This Changes

The issue asks for `openclaw tui` to stop showing `streaming` promptly after visible assistant content finishes instead of waiting on delayed final lifecycle/chat completion.

## Best Possible Solution

Add a backward-compatible content-completion or finalizing transition across the gateway/TUI boundary, keep the watchdog as recovery, and cover delayed lifecycle finals plus concurrent, hidden-output, tool-only, and reconnect cases.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: after a `delta`, visible text can stop while lifecycle/chat final is delayed by flush or post-run work, and current TUI stays `streaming` until `final` handling or the watchdog. I did not run a live TUI session during this read-only review.

## Solution Assessment

Yes, the best fix is a narrow additive completion/finalizing path rather than treating the existing 30s watchdog as the solution. A purely TUI-side shortcut should only be used if it preserves active-run, tool-only, hidden-output, and reconnect behavior.
