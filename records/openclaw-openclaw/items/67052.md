## Summary

Keep open. Current main still keeps the TUI in `streaming` after the last visible assistant delta until a terminal chat/lifecycle event arrives or the watchdog timeout fires, so the requested prompt content-completion transition is not implemented.

## What This Changes

The issue asks for the TUI to stop showing `streaming` promptly after visible assistant content finishes instead of waiting for full chat/lifecycle finalization.

## Best Possible Solution

Add a backward-compatible content-completion or finalizing path at the TUI/gateway boundary, keep the watchdog as recovery, and cover delayed lifecycle finals, tool-only output, concurrent runs, and reconnect behavior.

## Reproduction Assessment

Yes. A source-level reproduction is an active TUI run that receives assistant `delta` events, then visible text stops while lifecycle/chat finalization is delayed; current main remains `streaming` until terminal handling or the watchdog. No live TUI run was executed in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is a TUI/gateway completion-state change; a broad immediate-idle shortcut would risk hiding active runs, tool-only work, reconnect recovery, or late terminal errors.
