## Summary

Current main and the latest release now separate model-content completion from terminal cleanup: the agent emits a lifecycle `finishing` phase and the TUI changes the visible status to `finishing context` instead of leaving it on `streaming`.

## What This Changes

The issue requests that `openclaw tui` stop showing `streaming` promptly after visible assistant content finishes, rather than waiting on final cleanup before updating the status line.

## Best Possible Solution

Keep the shipped lifecycle `finishing` to TUI `finishing context` path, and leave the separate delta-only projection work to https://github.com/openclaw/openclaw/issues/82988.

## Reproduction Assessment

Yes. The original failure is source-reproducible from the old coupling between `streaming` and terminal chat/lifecycle finalization; current source and tests now show a separate lifecycle `finishing` path before terminal cleanup.

## Solution Assessment

Yes. The shipped solution is the maintainable fix because the runtime emits an explicit lifecycle signal when model output is done, so the TUI no longer has to guess from delta silence while final cleanup continues.
