## Summary

Keep open. Current main has watchdog and stale-terminal mitigations, but it still lacks the requested prompt content-complete or finalizing path; the TUI keeps visible assistant output in `streaming` until a terminal lifecycle/chat event or watchdog recovery, and merged PR #73749 explicitly left this issue as remaining UX timing work.

## What This Changes

The issue requests that `openclaw tui` stop showing active `streaming` promptly after visible assistant content finishes, even while backend finalization continues.

## Best Possible Solution

Add a backward-compatible content-completion or finalizing path that lets the TUI stop presenting visible assistant answer generation as active `streaming` while still tracking final/error/abort cleanup; update protocol/docs if a new event or state is added.

## Reproduction Assessment

Yes. A source-level reproduction is an active TUI run that receives assistant `delta` events, then stops producing visible text while terminal lifecycle/chat final is delayed; current main keeps the status `streaming` until lifecycle end/final or the watchdog timeout.

## Solution Assessment

Yes. A narrow TUI/gateway state-machine change is the maintainable direction; a broad immediate idle shortcut would risk hiding active runs, tool-only work, or delayed terminal errors.
