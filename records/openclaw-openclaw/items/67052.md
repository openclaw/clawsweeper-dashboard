## Summary

Keep open: current main still treats visible chat deltas as `streaming` until a later terminal chat/lifecycle path or the watchdog, and the related open PRs address adjacent watchdog behavior rather than this content-completion transition.

## What This Changes

The issue requests a TUI completion/finalizing state so the status bar stops showing active streaming promptly after visible assistant content finishes.

## Best Possible Solution

Add a backward-compatible content-completion or finalizing path between the gateway and TUI, while keeping the watchdog as recovery and covering delayed lifecycle finals plus concurrent/tool-only/reconnect cases.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a chat `delta` sets the TUI to `streaming`, visible text can then stop while lifecycle final waits on post-run flush work, and the TUI has no typed content-complete state before `final` or the watchdog.

## Solution Assessment

No current implementation is not the best solution. The shipped watchdog and adjacent PRs are mitigations; the maintainable fix needs an explicit content-completion/finalizing design or a carefully bounded TUI heuristic.
