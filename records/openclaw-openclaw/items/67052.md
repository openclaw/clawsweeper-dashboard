## Summary

Keep open. Current main has watchdog and stale-terminal mitigations, but the TUI still has no prompt content-complete or finalizing path, so visible assistant output can stop while the footer remains in `streaming` until lifecycle/chat finalization or the watchdog fires.

## What This Changes

The issue asks for `openclaw tui` to stop showing active `streaming` promptly after visible assistant content finishes, even while backend cleanup continues.

## Best Possible Solution

Add a backward-compatible content-completion or finalizing path shared by the gateway/TUI boundary, keep the watchdog as recovery, and cover delayed lifecycle finals plus concurrent/tool-only/reconnect cases.

## Reproduction Assessment

Yes. A source-level reproduction is an active TUI run that receives assistant `delta` events, then visible text stops while lifecycle/chat final is delayed; current main remains `streaming` until terminal handling or the watchdog. No live TUI run was executed in this read-only review.

## Solution Assessment

Yes. A narrow TUI/gateway completion-state change is the maintainable direction; a broad immediate idle shortcut would risk hiding active runs, late terminal errors, or tool-only work.
