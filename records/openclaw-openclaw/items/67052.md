## Summary

Current main and v2026.6.6 now solve the reported TUI UX gap with an explicit lifecycle `finishing` phase that moves the visible status from `streaming` to `finishing context` before terminal finalization completes.

## What This Changes

The issue asks for the TUI streaming indicator to stop implying active content generation after the visible assistant content has finished.

## Best Possible Solution

Keep the shipped lifecycle `finishing` to TUI `finishing context` path, and ask for a fresh report with v2026.6.6-or-newer evidence if the status still stays on `streaming` after visible content completes.

## Reproduction Assessment

Yes. The original failure was source-reproducible from the older coupling between visible streaming status and terminal finalization, and current source/tests now show a separate finishing lifecycle path.

## Solution Assessment

Yes. The implemented path is the best maintainable shape because the agent runtime explicitly emits a model-turn-ended lifecycle signal instead of making the TUI guess from delta silence.
