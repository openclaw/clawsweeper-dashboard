## Summary

Keep open: current main and the latest release still keep the TUI streaming state tied to chat deltas, terminal chat/lifecycle events, or the 30s watchdog, without a prompt visible-content-complete or finalizing transition.

## What This Changes

The issue asks OpenClaw TUI to stop showing active streaming promptly after visible assistant content finishes instead of waiting on delayed final lifecycle delivery.

## Best Possible Solution

Add a backward-compatible content-completion or finalizing path between the gateway and TUI, while keeping watchdog recovery and covering delayed lifecycle finals, hidden or tool-only output, concurrent runs, and reconnect fallback behavior.

## Reproduction Assessment

Yes, at source level. A chat delta sets the TUI to streaming, lifecycle final can wait on async flush work, and the protocol has no content-complete/finalizing state before final or watchdog recovery.

## Solution Assessment

Unclear. The issue is real, but maintainers need to choose between an additive gateway protocol signal and a bounded TUI heuristic before implementation.
