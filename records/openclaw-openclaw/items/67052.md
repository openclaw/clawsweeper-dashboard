## Summary

Current main and v2026.6.6 now solve the central UX problem: the agent emits a lifecycle `finishing` phase after visible model output completes, and the TUI switches from `streaming` to `finishing context` until terminal cleanup reaches `idle`.

## What This Changes

The issue asks for the TUI streaming indicator to stop implying active token streaming after visible assistant content has finished.

## Best Possible Solution

Keep the shipped lifecycle `finishing` to TUI `finishing context` path, and ask for a fresh v2026.6.6-or-newer repro if `streaming` still persists after visible content completes.

## Reproduction Assessment

Yes. The original failure is source-reproducible from the old coupling between visible `streaming` status and terminal finalization; current source and tests show a separate lifecycle `finishing` transition before final cleanup.

## Solution Assessment

Yes. The shipped solution is the maintainable fix because it uses an explicit runtime lifecycle signal instead of making the TUI guess from delta silence, while preserving terminal cleanup and final idle behavior.
