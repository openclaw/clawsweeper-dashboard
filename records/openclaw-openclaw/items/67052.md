## Summary

Keep open. Current main and the latest shipped TUI still have watchdog and stale-terminal mitigations, but no prompt content-complete or finalizing path; the footer can still remain in `streaming` until lifecycle/chat finalization or the watchdog fires.

## What This Changes

The issue asks for the TUI to stop showing `streaming` promptly after the last visible assistant content token instead of waiting on full run finalization.

## Best Possible Solution

Add a backward-compatible completion/finalizing transition across the gateway/TUI boundary, keep the watchdog as recovery, and cover delayed lifecycle finals plus concurrent, tool-only, hidden-output, and reconnect cases.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: an active TUI run receives a `delta`, visible text stops, and lifecycle/chat final is delayed by post-run work. Current main remains `streaming` until terminal handling or the 30s watchdog; no live TUI run was executed.

## Solution Assessment

Yes, the maintainable fix is a narrow additive content-completion or finalizing path rather than a broad immediate idle shortcut. A TUI-only silence heuristic would need safeguards for tools, hidden output, reconnects, and concurrent runs.
