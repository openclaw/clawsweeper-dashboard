Keep open. Current main still has a credible WebChat path where heartbeat/tool-final events reload `chat.history` and replace visible chat state, while the merge helper only preserves local optimistic tail messages; the issue also has a reporter correction and a second user confirmation.

## What This Changes

The issue reports WebChat assistant responses disappearing after a later heartbeat/tool-call turn, especially with MiMo V2 Pro, even though earlier responses were initially visible.

## Best Possible Solution

Land or fold a generic WebChat history reconciliation fix into the chosen reliability PR, with regression coverage proving heartbeat/tool-final reloads preserve already-visible assistant responses.

## Reproduction Assessment

No; I did not establish a high-confidence live reproduction against current main. Source inspection plus the issue comments show a credible current-main path through heartbeat/tool-final history reloads and narrow tail replacement.

## Solution Assessment

Unclear as an implementation choice because maintainers need to choose between the overlapping WebChat reliability PRs. The best technical direction is a generic history merge/projection regression fix, not a MiMo-specific workaround or heartbeat config change.
