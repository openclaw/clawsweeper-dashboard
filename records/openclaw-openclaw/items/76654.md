Keep open. Current main still has the WebChat heartbeat/tool-final history reload path and the existing merge helper only preserves local optimistic tails, while the overlapping WebChat reliability PRs are still open rather than merged fixes for this issue.

## What This Changes

The issue reports that WebChat loses previously visible assistant responses after heartbeat/tool-call history reloads for MiMo V2 Pro, while the Gateway transcript still contains the messages.

## Best Possible Solution

Land one generic WebChat history reconciliation fix with regression coverage proving heartbeat/tool-final reloads preserve already visible assistant responses, without adding a MiMo-specific workaround.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection does show a credible current-main path through heartbeat/tool-final reloads into an optimistic-only history merge that can replace visible assistant state.

## Solution Assessment
