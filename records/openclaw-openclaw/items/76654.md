## Summary

Keep open. Current main still has a credible WebChat path where heartbeat or silent final tool events can trigger a full `chat.history` reload and replace visible chat state, while the existing guard is aimed at local optimistic tails rather than all persisted assistant messages; the newer refresh-does-not-restore reports are not proven fixed by the older closed WebChat issues.

## What This Changes

The issue reports that WebChat assistant replies disappear after heartbeat tool-call turns for MiMo V2 Pro, while user turns remain and refresh no longer restores the missing assistant messages.

## Best Possible Solution

Keep the issue open until a focused WebChat history reconciliation fix, or one canonical WebChat reliability PR, ensures heartbeat/silent-final tool reloads cannot remove persisted assistant replies and adds regression coverage for the refresh path.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection shows a credible current-main path through heartbeat/silent-final tool events into `chat.history` replacement, and the comments provide two user reports, but the MiMo V2 Pro workflow was not run here.

## Solution Assessment

Unclear for the current open work, because no patch is attached to this issue. The maintainable direction is a narrow Control UI history-merge or reload guard, coordinated with #70391, #74733, and #75776, rather than a MiMo-specific workaround.
