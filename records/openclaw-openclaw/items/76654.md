## Summary

Keep open. Current main still has a credible WebChat heartbeat/tool-final history-reload path that can replace visible chat state, and the overlapping WebChat reliability PRs are open rather than merged or shipped.

## What This Changes

WebChat should keep previously rendered assistant responses visible after heartbeat-triggered tool calls and history reloads.

## Best Possible Solution

Fix WebChat history reconciliation so heartbeat or silent/tool-final reloads cannot remove already visible assistant messages, with regression coverage for persisted messages as well as optimistic local tails.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not live; source inspection shows the current heartbeat/silent-final and tool-final reload paths can still replace visible state, but I did not run the MiMo V2 Pro browser workflow.

## Solution Assessment

Is this the best way to solve the issue? The best direction is a narrow WebChat history-merge or reload guard, not a model-specific workaround; the remaining question is how to coordinate it with the open WebChat reliability PRs.
