## Summary

Keep open. Current main still has a credible WebChat history-reload path where heartbeat/tool-final events can replace visible state, and the overlapping WebChat reliability PRs are still open rather than merged.

## What This Changes

The issue reports WebChat assistant responses disappearing after automatic heartbeat tool-call turns, especially with MiMo V2 Pro.

## Best Possible Solution

Fix WebChat history reconciliation, or fold equivalent regression coverage into an active WebChat reliability PR, so heartbeat/tool-final reloads cannot remove already visible assistant messages.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No live MiMo V2 Pro reproduction was run; source inspection on current main shows a credible path through heartbeat/suppressed final events into `chat.history` replacement, so this is source-reproducible but not live-reproduced.

## Solution Assessment

Is this the best way to solve the issue? The best implementation direction is a narrow Control UI history-merge/reload guard with regression coverage; the only unclear part is whether it should land standalone or be folded into #70391, #74733, or #75776.
