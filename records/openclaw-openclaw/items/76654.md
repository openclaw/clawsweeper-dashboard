## Summary

Keep open: current main still has a source-visible WebChat path where tool-final or silent-final history reloads can replace already visible assistant messages, and the reporter correction plus a second user confirmation make this a credible remaining UI bug.

## What This Changes

The issue asks WebChat to keep already-rendered assistant responses visible when later heartbeat or tool-call events trigger history reloads.

## Best Possible Solution

Land one generic WebChat history reconciliation fix that preserves already-visible assistant tail messages through tool-final and silent-final reloads, with focused regression coverage at the reload/projection boundary.

## Reproduction Assessment

No. I did not establish a live current-main MiMo/heartbeat run; source inspection shows the same class of failure when a tool-final or silent-final event calls loadChatHistory and a stale chat.history snapshot excludes the already-visible assistant tail.

## Solution Assessment

Unclear. A narrow generic reconciliation fix is technically the right direction, but #70391, #74733, and #75776 overlap this area, so maintainers should choose whether to fold the fix into one of those branches.
