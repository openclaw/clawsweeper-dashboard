## Summary

Keep open. Current main still has a credible WebChat heartbeat/tool-final history reload path that can replace visible chat state, and the related reliability PRs are still open rather than merged into main.

## What This Changes

The issue reports WebChat assistant responses disappearing after an automatic heartbeat turn with tool calls, especially with the MiMo V2 Pro model.

## Best Possible Solution

Land a focused WebChat reconciliation fix, or fold it into the active reliability PRs, so heartbeat/tool-final history reloads cannot remove already visible assistant messages and regression coverage locks that in.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection shows a credible current-main path through heartbeat/tool-final `loadChatHistory` replacement, but the MiMo V2 Pro workflow was not run here.

## Solution Assessment

Yes, the maintainable direction is a narrow Control UI history-merge or reload guard with regression coverage. A MiMo-specific workaround or heartbeat configuration change would not address the UI reconciliation bug.
