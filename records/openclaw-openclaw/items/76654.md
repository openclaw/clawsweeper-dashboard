## Summary

Keep open. Current main still has a credible WebChat path where heartbeat or tool-final events reload `chat.history` and replace visible state, while the preservation guard is scoped to optimistic local tail messages rather than already-persisted assistant bubbles.

## What This Changes

The issue reports WebChat assistant responses disappearing after heartbeat-driven tool calls, especially with MiMo V2 Pro.

## Best Possible Solution

Fix WebChat history reconciliation so heartbeat and tool-final reloads cannot remove already visible assistant messages, then land one canonical focused fix or fold it into the appropriate overlapping WebChat reliability PR with regression coverage.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection shows a credible current-main path through heartbeat/tool-final history reloads into state replacement, but the exact MiMo V2 Pro workflow was not run.

## Solution Assessment

Yes, the best fix appears to be a narrow Control UI history merge or reload guard with regression coverage. A model-specific workaround or heartbeat configuration change would not address the underlying WebChat state reconciliation risk.
