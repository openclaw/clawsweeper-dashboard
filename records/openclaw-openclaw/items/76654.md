## Summary

Keep open. Current main still has a credible WebChat path where heartbeat/tool-final events trigger history reloads, and the merge helper preserves only local optimistic tail messages rather than any already-visible persisted assistant tail that a stale or projected `chat.history` response can drop.

WebChat should preserve previously visible assistant responses when later heartbeat or tool-call turns trigger history reloads.

## Best Possible Solution

Land a generic WebChat history reconciliation fix with regression coverage proving heartbeat/tool-final reloads preserve already visible assistant responses, without adding a MiMo-specific workaround.
