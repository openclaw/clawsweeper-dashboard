## Summary

Keep open. Current main still has a plausible WebChat history-reload path where heartbeat/tool or silent-final events call `chat.history` and replace visible chat state, while the reconciliation helper only preserves local optimistic tails; the follow-up comments also report the same disappearing assistant-message behavior and say refresh no longer recovers it. Related WebChat reliability PRs are open, not merged.

## What This Changes

WebChat should keep prior assistant replies visible when a later heartbeat turn with tool calls reloads or reconciles chat history.

## Best Possible Solution

Fix WebChat history reconciliation so heartbeat/tool-final reloads merge safely with already-visible assistant messages, with focused regression coverage for persisted as well as optimistic messages.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not live: source inspection shows a credible current-main path through heartbeat/tool or silent-final events into `chat.history` replacement, but the exact MiMo V2 Pro heartbeat workflow was not executed.

## Solution Assessment

Is this the best way to solve the issue? The best fix appears to be a narrow Control UI history-merge/reload guard with regression tests, coordinated with the overlapping WebChat reliability PRs rather than a model-specific workaround.
