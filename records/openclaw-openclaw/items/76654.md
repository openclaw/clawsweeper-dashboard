## Summary

Keep open. Current main has protections for stale optimistic WebChat history, but the source still has a heartbeat/silent-final path that can reload `chat.history` after tool activity and replace visible state, so the newer heartbeat-after-response displacement is not proven fixed by the older closed WebChat issues.

## What This Changes

The issue reports WebChat assistant responses disappearing after an automatic heartbeat turn with tool calls, with messages still preserved in the Gateway transcript.

## Best Possible Solution

Fix WebChat history reconciliation so heartbeat or silent/tool-final reloads cannot remove already visible assistant messages, with regression coverage for persisted messages as well as optimistic ones.
