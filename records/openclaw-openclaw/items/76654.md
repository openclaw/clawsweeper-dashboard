## Summary

Keep open. Current main still has a credible WebChat path where heartbeat/tool-final events reload `chat.history` and replace visible `chatMessages`, while the current preservation guard is limited to local optimistic tails rather than persisted assistant messages.

## What This Changes

WebChat should keep earlier visible assistant responses when automatic heartbeat/tool-call turns reload or reconcile chat history.

## Best Possible Solution

Fix WebChat history reconciliation so heartbeat or silent/tool-final reloads cannot remove already visible assistant messages, preferably by landing or narrowing the overlapping WebChat reliability work with regression coverage for persisted assistant tails.

## Reproduction Assessment

No live reproduction was established in this read-only review. Source inspection provides a credible current-main path through heartbeat/tool-final history reloads into visible message replacement, but the MiMo V2 Pro workflow was not rerun.

## Solution Assessment

Unclear as a standalone fix because three overlapping WebChat reliability PRs are already open. The maintainable direction is a generic Control UI history merge/reload guard, not a model-specific MiMo workaround.
