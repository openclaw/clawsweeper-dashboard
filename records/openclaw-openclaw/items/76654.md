## Summary

Keep open. Current main still has WebChat history reload paths after tool/silent final events, and the history merge guard only preserves metadata-free optimistic tail messages, so the heartbeat-after-response displacement is not proven fixed by the older closed WebChat issues or current source.

## What This Changes

The issue asks WebChat to keep previously visible agent responses rendered after heartbeat tool-call turns run.

## Best Possible Solution

Fix WebChat history reconciliation so heartbeat or silent/tool-final reloads cannot remove already visible assistant messages, with regression coverage for persisted messages as well as optimistic ones.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection shows a credible current-main path through heartbeat/tool-final events into chat.history replacement, but the exact MiMo V2 Pro workflow was not run here.

## Solution Assessment

Yes, the narrow maintainable fix is in Control UI history reconciliation or reload guarding, not a MiMo-specific workaround or heartbeat configuration change. The main coordination question is whether to fold it into the existing WebChat reliability PRs or land a smaller regression fix.
