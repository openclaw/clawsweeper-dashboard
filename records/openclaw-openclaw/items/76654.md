## Summary

Keep open. Current main still has a plausible WebChat path where heartbeat/tool final events reload `chat.history` and replace visible chat state, while the existing preservation guard only protects optimistic local tail messages; the issue is also confirmed by a later user comment and is not proven fixed by the older closed WebChat reports.

## What This Changes

Reports WebChat losing previously rendered assistant responses after automatic heartbeat tool calls, especially with MiMo V2 Pro, while user messages remain visible.

## Best Possible Solution
