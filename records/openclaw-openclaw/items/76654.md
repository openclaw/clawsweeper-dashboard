Keep open. Current main still has a credible WebChat history-reload path that can replace visible chat state after heartbeat/tool-final events, and the issue discussion adds a correction that refresh does not recover the missing replies plus a second user confirmation.

## What This Changes

The issue reports WebChat assistant replies disappearing after later heartbeat/tool-call turns, especially with Xiaomi MiMo V2 Pro heartbeat/tool behavior.

## Best Possible Solution

Fold this into the chosen WebChat reliability PR, or land a narrow history reconciliation fix that preserves already-visible assistant messages across heartbeat/tool-final history reloads with regression coverage.
