## Summary

Keep open. Current main still has source-level paths where heartbeat or tool-final events can trigger a WebChat `chat.history` reload that replaces visible state, and the related closed WebChat disappearance issues do not prove this heartbeat-after-response case is fixed.

## What This Changes

The issue reports that WebChat agent responses initially render but disappear after automatic heartbeat tool calls, especially with MiMo V2 Pro.

## Best Possible Solution

Fix WebChat history reconciliation and/or projection so heartbeat and tool-final reloads cannot remove already visible assistant replies, with focused regression coverage coordinated against the open WebChat reliability PRs.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No live reproduction was established; source inspection shows a credible current-main path through heartbeat/silent final events into `chat.history` replacement, but the MiMo V2 Pro heartbeat workflow was not run.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose between a narrow regression fix and folding it into the overlapping WebChat reliability PRs; the maintainable direction is a generic history merge/reload guard, not a model-specific workaround.
