## Summary

Keep open. Current main still has the lifecycle race: delayed exec approval follow-ups resume the child session by session key while run-mode subagent cleanup can retire that same session's bundle-MCP runtime, and disposal is not gated by pending follow-ups or active leases.

## What This Changes

The issue asks OpenClaw to prevent delayed exec approval follow-ups from resuming after a run-mode subagent's bundle-MCP runtime has been retired and surfacing `UNAVAILABLE`.

## Best Possible Solution

Add a session lifecycle guard for pending exec approval follow-ups so run-mode subagent runtime retirement either waits for the approved continuation to drain or cancels it with explicit user-facing delivery before disposal.

## Reproduction Assessment

Yes. The source-level path is to start a run-mode subagent with bundle-MCP enabled, trigger an exec approval, let the subagent complete and retire its session runtime, then approve so the delayed follow-up resumes the same child session.

## Solution Assessment

Unclear until the lifecycle policy is chosen. A pending-follow-up-aware runtime retirement guard is more maintainable than only changing the `UNAVAILABLE` fallback because it preserves or explicitly cancels the approved continuation.
