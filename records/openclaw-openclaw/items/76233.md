## Summary

Keep open. Current main still has the reported lifecycle race: delayed exec approval follow-ups resume by session key after approval resolves, while run-mode subagent cleanup can retire the same bundle-MCP runtime without coordinating with pending or active follow-up work.

## What This Changes

The issue asks OpenClaw to prevent delayed exec approval follow-ups from failing after a run-mode subagent has completed and its bundle-MCP runtime has been retired.

## Best Possible Solution

Add session lifecycle coordination so pending or active exec approval follow-ups either pin/drain the bundle-MCP runtime or are cancelled with a clear direct notification before run-mode subagent cleanup retires it.

## Reproduction Assessment

Yes. The source-level reproduction is to trigger an exec approval in a run-mode subagent, let the subagent complete and retire its bundle-MCP runtime, then approve so the follow-up resumes the child session with bundle-MCP enabled.

## Solution Assessment

Unclear. The requested lifecycle hardening is the right area, but maintainers should choose the contract: pin/drain is more robust, while cancellation/direct notification is simpler and changes the user-visible outcome for late approvals.
