## Summary

Keep open. Current main still allows exec approval follow-up delivery to resume by session key after run-mode subagent cleanup retires that session's bundle-MCP runtime, so the reported disposed-runtime `UNAVAILABLE` path remains valid; adjacent #64316, #73792, and #76199 do not close this lifecycle gap.

## What This Changes

Prevent delayed exec approval follow-ups from resuming a run-mode subagent after its bundle-MCP runtime has been retired.

## Best Possible Solution

Add a per-session lifecycle guard so pending exec approval follow-ups and bundle-MCP retirement share one policy for pinning, draining, retrying, or cancelling before disposal.

## Reproduction Assessment

Yes, at source level: trigger a run-mode subagent exec approval, let the child complete so cleanup retires its bundle-MCP runtime, then approve so the follow-up resumes that child session. I did not run a live gateway repro in this read-only pass.

## Solution Assessment

Unclear until the lifecycle policy is chosen. A pending-follow-up-aware runtime guard is the narrow maintainable direction; only changing the surfaced `UNAVAILABLE` error would still lose the approved continuation.
