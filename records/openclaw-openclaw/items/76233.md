## Label Justifications

- `P1`: A real approval/subagent workflow can return UNAVAILABLE and lose the approved follow-up, with a second user confirming the symptom.
- `impact:session-state`: The bug is a session/runtime lifecycle race where pending approval work can outlive the subagent runtime it targets.
- `impact:message-loss`: The approved command follow-up can fail before the result is delivered or converted into a clear terminal message.

## Summary

Keep open: current main still allows an exec approval follow-up to resume by session key while run-mode subagent cleanup can retire the same bundle-MCP runtime without draining or terminalizing pending approvals. The issue comments match the current source path, another user reported the same disposed-runtime symptom, and related GitHub searches did not find a merged or open canonical fix for this exact approval-lifecycle race.

## What This Changes

The issue requests that exec approval follow-ups cannot run against a disposed subagent bundle-MCP runtime and instead keep a valid runtime or end with an explicit terminal outcome.

## Best Possible Solution

Add one approval-lifecycle invariant: a pending exec approval either holds or reacquires a valid runtime lease until the follow-up drains, or cleanup atomically transitions it to a terminal user-visible state before bundle-MCP disposal can make it consumable.

## Reproduction Assessment

Yes at source level: spawn a run-mode subagent, trigger an exec approval, let the subagent finish so cleanup retires its bundle-MCP runtime, then approve the request. I did not execute the live scenario in this read-only pass, but the current source still contains the same unguarded lifetime path.

## Solution Assessment

Unclear as a final implementation: the requested invariant is right, but maintainers should choose whether pending approvals block retirement or are made terminal on retirement. A retry/fallback-only patch would be too narrow because it would not bind approval consumption to a live or retired execution context.
