## Summary

Keep open: current main still lets async exec approval follow-up resume by session key while run-mode subagent cleanup can retire the same bundle-MCP runtime, and the related open/closed PRs do not close this lifecycle gap.

## What This Changes

The issue reports a race where delayed exec approval follow-up resumes a subagent session after its bundle-MCP runtime has been retired, surfacing a disposed-runtime `UNAVAILABLE` error.

## Best Possible Solution

Add a per-session lifecycle guard so exec approval follow-ups and bundle-MCP retirement share one contract for pending follow-ups: either pin/drain before disposal or cancel with explicit recovery.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: trigger a run-mode subagent exec approval, let the child finish and retire its bundle-MCP runtime, then approve so the follow-up resumes the retired child session. I did not run a live gateway repro in this read-only review.

## Solution Assessment

Unclear until the lifecycle policy is chosen. A lease-aware or pending-follow-up-aware retirement guard is the narrow maintainable direction; changing only the surfaced `UNAVAILABLE` error would leave the approved continuation vulnerable to being lost.
