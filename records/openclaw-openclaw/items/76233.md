- `impact:session-state`: The approval followup remains bound to a session whose runtime lifecycle has already advanced to disposed.

## Summary

Keep open: current main still has the reported lifecycle race. The source path shows exec approval followups can be scheduled after approval, while subagent run cleanup can retire the same session-scoped bundle-MCP runtime without draining or terminally cancelling pending followups.

## What This Changes

The issue reports that delayed exec approval followups for run-mode subagents can resume after the session bundle-MCP runtime has been retired and surface as `UNAVAILABLE`, losing the approval result.

## Best Possible Solution

Add one approval-lifecycle invariant for subagent run cleanup: pending exec followups must either hold a valid runtime lease until drained or become terminally cancelled/retired before runtime disposal can complete.

## Reproduction Assessment

Yes for source-level reproduction: a run-mode subagent can request exec approval, finish, trigger bundle-MCP retirement, and then a later approval followup resumes against the same disposed runtime. I did not execute a live gateway repro in this read-only pass.

## Solution Assessment

Unclear until maintainers choose the lifecycle contract. A retry-only or nicer `UNAVAILABLE` path would be too narrow; the better fix is to make approval followups own a live runtime lease or become explicit terminal work before retirement.
