## Summary

Keep open. Current main still has the lifecycle race described in the report: delayed exec approval follow-up resumes by session key while run-mode subagent cleanup can retire the same session's bundle-MCP runtime, and explicit retirement does not wait for active materialization leases or pending follow-ups.

## What This Changes

The issue asks OpenClaw to prevent delayed exec approval follow-up work from resuming against a subagent session after its bundle-MCP runtime has been retired.

## Best Possible Solution

Add a per-session lifecycle guard so bundle-MCP retirement and exec approval follow-up delivery agree on whether pending follow-ups pin the runtime, drain before disposal, or are explicitly cancelled with user-visible recovery.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: trigger a run-mode subagent exec approval, let the child complete and retire its bundle-MCP runtime, then approve so the follow-up resumes the retired child session. I did not run a live gateway repro in this read-only review.

## Solution Assessment

Unclear until the lifecycle policy is chosen. A lease-aware or pending-follow-up-aware retirement guard is the narrow maintainable direction; changing only the surfaced `UNAVAILABLE` error would leave the approved continuation vulnerable to being lost.
