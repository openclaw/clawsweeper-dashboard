## Summary

Keep open. Current main still has an unguarded lifecycle edge where a delayed exec approval follow-up can resume a run-mode subagent session after cleanup has retired that session's bundle-MCP runtime, and the related open PRs described in the discussion do not cover this explicit-retire race.

## What This Changes

The issue asks OpenClaw to prevent delayed exec approval follow-ups for completed run-mode subagents from failing against a disposed bundle-MCP runtime.

## Best Possible Solution

Add a lifecycle gate so run-mode subagent bundle-MCP retirement is coordinated with pending or active exec approval follow-ups for the same session, with explicit cancellation or retry semantics for follow-ups that arrive after retirement begins.

## Reproduction Assessment

Yes. The source-level reproduction is to start a run-mode subagent, trigger a gateway exec approval, let the subagent complete and retire its bundle-MCP runtime, then approve so the follow-up resumes that child session.

## Solution Assessment

Unclear. A pending-follow-up-aware runtime retirement contract looks like the maintainable direction, while merely catching and rewording UNAVAILABLE would still risk losing the approved continuation.
