## Summary

Keep open. Current main still lets delayed exec approval continuations resume by session key after run-mode subagent cleanup can retire the same bundle-MCP runtime, and there is no guard that pins, cancels, or retries pending follow-ups.

## What This Changes

The issue requests fixing a race where a delayed exec approval follow-up resumes a completed subagent after its bundle-MCP runtime has been retired and surfaces `UNAVAILABLE`.

## Best Possible Solution

Add a per-session lifecycle guard that coordinates pending exec approval follow-ups with bundle-MCP runtime retirement, with regression coverage for a subagent that finishes before the user approves.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: trigger a run-mode subagent exec approval, let the child complete and retire its bundle-MCP runtime, then approve so the follow-up resumes the retired child session.

## Solution Assessment

Unclear until the lifecycle policy is chosen. A lease-aware or pending-follow-up-aware retirement guard is the narrow maintainable direction; changing only the surfaced `UNAVAILABLE` error would still lose the approved continuation.
