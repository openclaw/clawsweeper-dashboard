The issue requests durable paired-device creation and last-seen timestamps so operators can identify stale paired clients.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/81189; the desired end state is to refresh existing paired-device last-seen metadata after successful auth or reconnect without changing pairing policy, scope checks, or field names.

Yes. Source inspection gives a high-confidence path: pair a device, authenticate with an accepted device token or reconnect as a paired client, and current main persists token/access metadata without refreshing device-level last-seen fields.

## Solution Assessment
