## Summary

Keep open: current main still writes paired-device creation time but accepted device-token auth and paired WebSocket reconnects do not refresh durable device-level last-seen metadata. The open fixing PR at https://github.com/openclaw/openclaw/pull/81189 uses closing syntax for this issue, so this issue should stay open until that PR lands or is closed.

## What This Changes

The issue requests durable paired-device creation and last-seen timestamps so operators can identify stale paired clients.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81189 if it stays correct and green; otherwise apply the same focused metadata refresh with regression coverage while preserving current schema and auth gates.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: pair a device, authenticate with an accepted device token or reconnect over the paired WebSocket path, and current main updates token/use or access metadata without setting device-level `lastSeenAtMs`.

## Solution Assessment

Yes. The narrow maintainable fix is to wire successful paired auth/reconnect into the existing `updatePairedDeviceMetadata` last-seen fields with focused regression coverage, not to add a new config option or schema migration.
