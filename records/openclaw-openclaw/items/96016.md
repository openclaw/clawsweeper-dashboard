## Summary

Keep open: current main and the latest release still upload the named OpenShell staging directory, while OpenShell now preserves named directory basenames. Two open closing PRs already own the fix path, so this issue should remain open until one lands.

## What This Changes

Issue reports recursive upload-id directory nesting in OpenShell mirror-mode sandbox sync after OpenShell started preserving uploaded directory basenames.

## Best Possible Solution

Review and land one of the linked PRs, preferably the one with full mirror-sync regression coverage, so `uploadPathToRemote` uploads `.` from the staged directory and keeps mirror workspaces flat.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main stages the workspace into a named temp directory and uploads that path, while OpenShell >= 0.0.37 preserves named directory basenames; I did not run a live gateway repro in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to keep the symlink-free staged snapshot but invoke OpenShell flat extraction by uploading `.` with cwd set to the staged directory; this issue should stay open until one candidate PR merges.
