## Summary

This should stay open: current main and the latest release still upload the staged OpenShell mirror directory by its generated basename, and two open PRs already use closing syntax for the same root cause. ## What This Changes

The issue reports recursive previous-workspace directory nesting in OpenShell mirror mode after OpenShell changed named directory uploads to preserve the source basename.

## Best Possible Solution

Land one narrow OpenShell plugin fix that uploads staged mirror contents flat, ideally as `.` with `cwd` set to the staging directory, and include regression coverage for repeated mirror sync cycles.

## Reproduction Assessment

Yes, there is a high-confidence source reproduction path: current main uploads a named staging directory before exec and mirrors the remote workspace back after exec, while the merged upstream OpenShell upload changes preserve named directory basenames. I did not run a live OpenShell backend in this read-only review.

## Solution Assessment

Yes, the narrowest maintainable fix appears to be changing the single OpenShell upload chokepoint to request flat extraction and adding a repeated-sync regression test. The issue itself should remain open until one linked fixing PR merges.
