## Summary

Keep open: current main and the latest release still upload a named OpenShell staging directory into the sandbox workspace, while current OpenShell preserves named directory basenames, matching the reported nested `openclaw-openshell-upload-*` directories.

## What This Changes

The issue reports recursive previous-workspace directories when the OpenShell sandbox plugin uses mirror mode with OpenShell 0.0.37 or newer.

## Best Possible Solution

Change the OpenShell mirror upload path so staged workspace contents are uploaded flat under the configured remote workspace, then add regression coverage that staging directory names never become sandbox workspace entries after repeated syncs.

## Reproduction Assessment

Yes at source level, not by live execution: current main uploads a named temp directory, and OpenShell’s current upload contract preserves named directory basenames under the destination. That matches the reported `openclaw-openshell-upload-*` directory appearing in the sandbox.

## Solution Assessment

Yes. The narrowest maintainable fix is in the OpenShell plugin upload/mirror boundary, not a new configuration option or core sandbox policy change.
