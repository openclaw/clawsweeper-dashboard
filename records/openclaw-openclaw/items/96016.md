## Summary

Keep open: current main and latest release still upload the named OpenShell staging directory, while OpenShell now preserves named directory basenames. Two open PRs already use closing references and appear to own the fix path, so this issue should remain open until one lands.

## What This Changes

Mirror mode with the OpenShell sandbox plugin should stop creating recursively nested upload-id directories after repeated syncs against OpenShell >= 0.0.37.

## Best Possible Solution

Review and land one of the linked PRs, preferably the one with full mirror-sync regression coverage, so `uploadPathToRemote` uploads `.` from the staged directory and keeps mirror workspaces flat.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main stages the workspace into a named temp directory and uploads that path, while OpenShell >= 0.0.37 preserves named directory basenames; I did not run a live gateway repro in this read-only review.

## Solution Assessment

Yes, the linked fix direction is the narrow maintainable solution: keep the symlink-free staging snapshot, but invoke OpenShell flat extraction by uploading `.` with cwd set to the staged directory. The issue itself should stay open until one candidate PR merges.
