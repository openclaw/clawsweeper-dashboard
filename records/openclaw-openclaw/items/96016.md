## What This Changes

The issue reports that OpenShell mirror mode against OpenShell >= 0.0.37 repeatedly nests previous workspace snapshots under upload-id directories in the sandbox.

## Best Possible Solution

Review and land one canonical fix PR that keeps symlink-free staging, uploads `.` from the staged cwd at `uploadPathToRemote`, includes regression coverage, and has redacted live OpenShell proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main stages the workspace into a named temp directory and uploads that path, while OpenShell preserves named directory basenames; I did not run a live gateway repro.

## Solution Assessment

Yes. The narrow maintainable fix is to keep the staged symlink-free snapshot but invoke OpenShell flat extraction by uploading `.` with cwd set to the staged directory; the issue should stay open until a linked PR merges.
