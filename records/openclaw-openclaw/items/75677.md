## Summary

Keep open: the PR is a plausible localization contribution, but the current patch is not safe to merge because it hardcodes zh-CN as the only TUI language and changes internal activity-state strings that current recovery logic and tests treat as semantic keys.

## What This Changes

The PR replaces TUI status, activity-state, disconnect, Ctrl+C/tool-toggle strings, and default waiting phrases in `src/tui` with Simplified Chinese literals.

## Best Possible Solution

Add a real TUI i18n boundary: keep activity state as typed semantic keys, translate only at render/copy boundaries, preserve English as the default, add an explicit locale selection or documented auto-detection rule, cover zh-CN display labels plus state-machine behavior in focused TUI tests, and include the required user-facing changelog entry when the feature is ready.

## Reproduction Assessment

Yes. Applying the PR and running focused TUI tests such as `pnpm test src/tui/tui-command-handlers.test.ts src/tui/tui-event-handlers.test.ts src/tui/tui.test.ts` should exercise the changed `setActivityStatus` literals; static inspection already shows the mismatch with current `"waiting"` and `"streaming"` comparisons.

## Solution Assessment

No. Hardcoding zh-CN over the current strings is not the narrowest maintainable solution because it removes the English default and conflates display labels with state-machine keys; a render-time translation layer with locale selection is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve semantic activity state keys:** `src/tui/tui-event-handlers.ts:377`
  - body: `setActivityStatus` updates shared TUI state, and current recovery logic still compares that state to semantic literals such as `"streaming"`. After this change a chat delta stores `流式输出中`, so stale-streaming cleanup can miss the streaming state and skip idle recovery. Keep stored state typed/semantic and translate only when rendering text.
  - confidence: 0.9
- **[P2] Add locale selection instead of replacing the default:** `src/tui/tui.ts:329-330`
  - body: This initializes every TUI session with Chinese status text regardless of user locale or config. Current docs and tests describe English status labels, and this patch adds no TUI locale setting, so English users lose the default rather than gaining zh-CN support. Make the translation opt-in or locale-aware.
  - confidence: 0.86
