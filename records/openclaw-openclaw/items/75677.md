## Summary

Keep open. The PR is useful in intent, but the current patch hardcodes zh-CN over the English TUI default and mixes translated display strings into activity-state fields that current main treats as semantic control state.

## What This Changes

The PR replaces TUI status, disconnect, Ctrl+C/tool messages, and default waiting phrases with hardcoded Simplified Chinese strings across four `src/tui` files.

## Best Possible Solution

Add a TUI localization layer that keeps internal activity states typed and semantic, translates only at render/message boundaries after explicit locale selection or detection, preserves English fallback, and updates tests, docs, and changelog together.

## Reproduction Assessment

Yes. Static diff inspection is enough: the PR writes translated values into `setActivityStatus` while current main compares `activityStatus` to `streaming` and `waiting`, and focused tests assert those semantic keys.

## Solution Assessment

No. Hardcoding zh-CN over the existing literals is not the best maintainable solution; the safer path is a locale-aware copy boundary with semantic state kept separate from display text.
