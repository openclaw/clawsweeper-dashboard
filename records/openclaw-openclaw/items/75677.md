## Summary

Keep open. The PR addresses a useful accessibility goal, but the current diff hard-replaces the default TUI language and localizes semantic activity state values that current main still uses for status rendering, waiting animation, and stale-streaming recovery.

## What This Changes

The PR replaces TUI status, connection, prompt, and waiting-phrase literals with Simplified Chinese strings across four `src/tui` files.

## Best Possible Solution

Introduce a locale-aware TUI copy layer that keeps activity state as typed semantic keys, translates only at render/message boundaries, preserves English as the default, adds explicit locale selection or documented detection, and updates tests, docs, and changelog together.

## Reproduction Assessment

Yes. Applying the PR and running the focused TUI command/event-handler tests would exercise the changed activity states; static inspection already shows the localized values no longer match current-main `streaming` and `waiting` comparisons.

## Solution Assessment

No. Hardcoding zh-CN over the existing literals is not the narrowest maintainable solution because it removes the English default and mixes display labels with state-machine keys; a render-time translation boundary is safer.
