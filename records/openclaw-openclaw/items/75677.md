## Summary

Keep open. The locale request is useful, but the existing ClawSweeper blockers still hold on current main: this PR replaces internal TUI state keys with translated display strings and adds no TUI locale selector or English fallback.

## What This Changes

This PR replaces TUI status, disconnect, Ctrl+C/tool messages, and default waiting phrases with hardcoded Simplified Chinese strings across four `src/tui` files.

## Best Possible Solution

Introduce a TUI localization layer that keeps activity state typed and semantic, translates only at render/message boundaries after explicit locale resolution, preserves English fallback, and updates tests, docs, and changelog together.

## Reproduction Assessment

Yes. Static source diff is a high-confidence reproduction path: the PR writes translated text into `setActivityStatus`, while current main compares `activityStatus` to semantic keys such as `streaming` and `waiting` and tests assert those keys.

## Solution Assessment

No. Hardcoding zh-CN over current literals is not the best maintainable solution; the safer path is a locale-aware display boundary with semantic state kept separate from rendered copy.
