## Summary

Keep open, but the patch is not mergeable as-is because it conflates TUI control-state keys with display copy and changes the default TUI language globally without a locale contract.

## What This Changes

The PR replaces English TUI status messages, activity keys, disconnect/Ctrl+C/tool text, and default waiting phrases with hardcoded Simplified Chinese strings across four `src/tui` files.

## Best Possible Solution

Add TUI localization at the render/message boundary, keep internal activity states typed and semantic, preserve English fallback until explicit locale selection or detection, and update focused tests/docs/changelog with the chosen contract.

## Reproduction Assessment

Yes for the blocking regression. Static source inspection is enough: the PR stores translated strings where current main branches and tests depend on semantic `activityStatus` keys such as `waiting` and `streaming`.

## Solution Assessment

No. Hardcoding zh-CN over existing literals is not the best solution; a maintainable fix needs a locale-aware display layer that leaves control state unchanged and keeps English as the default fallback.

## Review Findings

Overall correctness: patch is incorrect
