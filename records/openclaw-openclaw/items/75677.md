## Summary

Keep open. The TUI localization goal is useful, but this PR is not mergeable as written because it hardcodes zh-CN over English defaults and changes semantic activity-state keys that current main uses for control flow and tests.

## What This Changes

The PR replaces TUI status, disconnect, Ctrl+C/tool messages, and default waiting phrases across four `src/tui` files with hardcoded Simplified Chinese strings.

## Best Possible Solution

Add a TUI localization layer that keeps internal activity states typed and semantic, translates only at render/message boundaries after explicit locale selection or documented detection, preserves English fallback, and updates tests, docs, and changelog together.

## Reproduction Assessment

Yes. Static diff inspection against current main is enough: the PR writes translated values into `setActivityStatus` while current main compares `activityStatus` to `streaming` and `waiting`, and focused tests assert those semantic keys.

## Solution Assessment

No. Hardcoding zh-CN over existing literals is not the best maintainable solution; the safer path is a locale-aware copy boundary with semantic state kept separate from display text.
