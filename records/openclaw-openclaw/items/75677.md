## Summary

Keep open for maintainer review. The branch is a useful zh-CN localization proposal, but the current diff hardcodes Chinese as the default TUI language and changes semantic activity-state keys that the renderer, watchdog recovery, docs, and tests currently rely on.

## What This Changes

The PR replaces English TUI status, connection, prompt, and waiting-phrase literals with Simplified Chinese strings in four `src/tui` files.

## Best Possible Solution

Introduce locale-aware TUI copy at render/message boundaries while keeping `activityStatus` as typed semantic state, preserving English as the default until a locale is selected, and updating tests, docs, and changelog together.

## Reproduction Assessment

Yes. Applying the PR would make `setActivityStatus` store translated display strings, while current-main code and focused TUI tests compare against semantic values such as `waiting` and `streaming`.

## Solution Assessment

No. Hardcoding zh-CN over existing literals is not the best fix because it changes the default language for all users and mixes display text with state-machine keys; a locale-aware translation boundary is safer.
