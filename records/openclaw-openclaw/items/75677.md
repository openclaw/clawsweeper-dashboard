## Summary

Keep open, but the PR should not merge as-is. Current main treats TUI activity values as semantic state keys, while this patch replaces those keys and the default English copy with zh-CN literals without adding a locale boundary; the existing ClawSweeper comment identified the same blockers and the current source/tests/docs confirm them.

## What This Changes

The PR replaces English TUI status, connection, prompt, event-gap, and waiting-phrase literals with Simplified Chinese strings across four `src/tui` files.

## Best Possible Solution

Add a locale-aware TUI copy layer that keeps activity state as typed semantic keys, preserves English as the default, adds explicit locale selection or documented detection, and updates tests, docs, and changelog together.

## Reproduction Assessment

Yes. Applying this PR and running the focused TUI command/event/waiting tests would exercise the changed status paths; static inspection already shows translated values no longer match current-main `streaming` and `waiting` comparisons.

## Solution Assessment

No. Hardcoding zh-CN over source literals is not the best fix because it changes default behavior for all users and mixes display text with state-machine values; translate only at the render/message boundary.
