## Summary

Keep open: the PR is aimed at a real current-main doctor gap, and the update addressed the earlier dependency-key traversal review, but the new peer-link repair still needs one more path-boundary guard before merge.

## What This Changes

The PR adds doctor detection and repair for missing or stale managed npm plugin `node_modules/openclaw` peer links, plus repair sequencing, docs, changelog, and regression tests.

## Best Possible Solution

Keep the doctor repair approach, but validate or create the `node_modules` link parent as a real in-package directory before any remove or symlink operation, with a regression test for a symlinked parent.

## Reproduction Assessment

## Solution Assessment

No. Doctor is the right repair boundary, but the implementation should guard the resolved link parent before mutating the filesystem.
