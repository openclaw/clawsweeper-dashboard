If the parent is a symlink or resolves outside, skip repair and emit a clear Plugin registry warning. Add a regression test in `src/commands/doctor-plugin-registry.test.ts` with a managed plugin declaring `peerDependencies.openclaw` whose `node_modules` is a symlink to an outside directory; `doctor --fix` must not create or remove the outside `openclaw` path. Preserve the existing docs/changelog unless wording must change, and do not change unrelated plugin install behavior.
