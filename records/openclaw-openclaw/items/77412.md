## Best Possible Solution

Yes, the direction is the narrow maintainable fix: doctor owns legacy/drift repair and the PR does not add startup migrations or package-manager work. The only safer adjustment is to warn when the host package root cannot be resolved instead of silently treating the peer-link check as clean.
