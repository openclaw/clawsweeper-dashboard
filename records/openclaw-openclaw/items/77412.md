## Summary

Keep open: the latest PR head addressed the dependency-key escape case, but the new repair path still does not validate that `packageDir/node_modules` is a real in-package directory before mutating `node_modules/openclaw`.

## What This Changes

The PR adds doctor detection and repair for missing or stale managed npm plugin `openclaw` peer links, updates repair sequencing, and documents the repair with tests and changelog coverage.

## Best Possible Solution

Keep the doctor repair boundary, but validate or create the `node_modules` parent as a real in-package directory before any remove or symlink operation, with a regression test for a symlinked parent.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a managed npm plugin with `peerDependencies.openclaw` and `packageDir/node_modules` symlinked outside the package passes the current scan and reaches the repair writes.

## Solution Assessment

No as written. Doctor is the right place to repair stale peer links, but this implementation needs a parent-directory realpath/lstat guard before it is the narrow maintainable fix.

## Review Findings
