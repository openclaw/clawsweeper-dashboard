## Summary

Keep open. Rechecking current `main` and the latest release shows the requested health-version field and HTTP admin update endpoint are still absent, while the adjacent `update.run` RPC is security-sensitive enough that this needs maintainer API design rather than automated cleanup.

## What This Changes

Request adds a `version` field to `GET /health` and an authenticated `POST /admin/update` endpoint with optional `target_version` for fleet updates.

## Best Possible Solution

Design a public gateway fleet-management API that reuses the existing runtime-version and update-runner surfaces while explicitly specifying admin auth, rate limiting, audit, target-version, restart, failure, and redaction semantics.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection on current `main` plus `v2026.5.12` shows `/health` returns only `{ ok, status }` while no `/admin/update` or `target_version` gateway HTTP contract exists.

## Solution Assessment

No, not as a narrow automated endpoint as written. The maintainable path is a maintainer-designed gateway HTTP/protocol boundary that reuses existing update/version surfaces without weakening admin safeguards.
