## Summary

Keep open. Current main still only partially overlaps the request: runtime version exists on the status surface and admin HTTP RPC exists, but HTTP `/health` remains a shallow liveness response and there is no HTTP update trigger or `target_version` contract; the update endpoint is a privileged API design, not conservative cleanup.

## What This Changes

Request adding a `version` field to `GET /health` and a gateway-authenticated `POST /admin/update` endpoint for fleet update orchestration.

## Best Possible Solution

Design an additive Gateway fleet/admin API around the existing status, admin HTTP RPC, and update primitives with an intentional security model before exposing update triggering to fleet tools.

## Reproduction Assessment

Not applicable: this is a new Gateway API feature request, not broken existing behavior. Source inspection on current main shows the requested HTTP `/health` version field and HTTP update trigger remain absent.

## Solution Assessment

No, not as a one-off `POST /admin/update` route exactly as requested. The safer path is a maintainer-designed admin/fleet API that reuses existing status and update plumbing without widening privileged access accidentally.

## Security Review

Status: needs_attention

Summary: The requested HTTP update trigger is security-sensitive because it would expose package-manager and restart work through a network admin surface.

Concerns:

- **[medium] Design the update-trigger HTTP boundary first:** `src/gateway/server-methods/update.ts:182`
  - body: `update.run` can invoke package-manager update work or a managed-service handoff and schedule restarts, so HTTP exposure needs defined admin auth scope, audit/rate-limit behavior, restart coalescing, and target-version validation.
  - confidence: 0.92
- **[medium] Keep the admin HTTP RPC allowlist intentional:** `extensions/admin-http-rpc/src/methods.ts:49`
  - body: The current admin HTTP RPC plugin exposes `update.status` but not `update.run`; adding update triggering would expand a privileged network control-plane surface.
  - confidence: 0.9
- **[medium] Shape update results for HTTP callers:** `src/gateway/server-methods/update.ts:245`
  - body: The existing update response can include handoff command details, restart metadata, sentinel data, and step output, so a public HTTP contract needs deliberate response shaping and redaction.
  - confidence: 0.84
