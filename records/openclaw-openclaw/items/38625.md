## Summary

Keep open. Current main still lacks both requested HTTP API changes: `/health` still returns only the liveness shape, and there is no `POST /admin/update` or `target_version` gateway HTTP contract; the existing `update.run` RPC is adjacent but exposing host package-manager updates over HTTP needs maintainer product and security design.

## What This Changes

Request to add a `version` field to `GET /health` and an authenticated `POST /admin/update` endpoint with optional `target_version` for fleet update orchestration.

## Best Possible Solution

Design a public fleet-management API that reuses the existing runtime-version source and update runner while explicitly defining admin auth, rate limiting, audit, target-version, restart, failure, and redaction semantics.

## Reproduction Assessment

## Solution Assessment
