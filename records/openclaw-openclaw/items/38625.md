## Summary

Keep open. Current main still lacks both requested API additions: `/health` remains a shallow liveness probe without a version field, and update orchestration is still exposed through privileged Gateway RPC rather than a designed `POST /admin/update` HTTP contract.

## What This Changes

Request to add a Gateway version field to `GET /health` and a token-authenticated `POST /admin/update` endpoint with optional target-version update orchestration.

## Best Possible Solution

Design a Gateway fleet-management API that reuses existing runtime-version and updater surfaces while specifying admin auth, rate limiting, audit, target-version, restart, failure, and redaction semantics.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Source inspection on current main shows `/health` still returns only `{ ok, status }` and no `/admin/update` or `target_version` HTTP contract exists.

## Solution Assessment

No, not as a one-off endpoint exactly as written. The maintainable path is an owner-designed Gateway admin/fleet API, likely coordinated with the admin HTTP RPC and managed-service update work, that reuses existing update/version surfaces without weakening admin safeguards.
