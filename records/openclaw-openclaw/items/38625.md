## Summary

Keep open. Current main still lacks both requested Crab-facing HTTP additions: the shallow health probe does not expose a version field, and gateway updates remain available through the admin-scoped WebSocket RPC surface rather than `POST /admin/update`.

## What This Changes

The issue requests adding a `version` field to `GET /health` and adding an authenticated `POST /admin/update` endpoint for fleet update orchestration.

## Best Possible Solution

Expose the gateway version on the shallow health probe if maintainers accept that compatibility change, then decide whether Crab should use documented WS `update.run`/`update.status` or a REST wrapper with equivalent admin, audit, rate-limit, and validation safeguards.

## Reproduction Assessment

Yes. Source inspection on current main shows `/health` and `/healthz` serialize only `{ ok, status }`, there is no `POST /admin/update` route, and the current update schema rejects a `target_version` field.

## Solution Assessment

Unclear. The health version field is a narrow additive change, but the update endpoint is not clearly the best API until maintainers choose between documenting the existing WS RPC for Crab or adding an HTTP wrapper with matching safeguards.
