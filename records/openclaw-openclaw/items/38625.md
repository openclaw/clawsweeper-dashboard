## Summary

Keep open. Current main still only partially overlaps the request: runtime version is available through status surfaces and the admin HTTP RPC plugin exists, but `/health` still returns only liveness status and there is no HTTP update trigger or `target_version` contract.

## What This Changes

The issue requests adding a Gateway health response version field and an authenticated HTTP admin update endpoint for fleet tooling.

## Best Possible Solution

Design a Gateway fleet-management/admin HTTP contract around the existing version, status, and update primitives before exposing update triggering to fleet tools.

## Reproduction Assessment

Not applicable: this is a new Gateway API feature request, not broken existing behavior. Source inspection on current main shows the requested `/health` version field and HTTP update trigger are still absent.

## Solution Assessment

No, not as a one-off `POST /admin/update` route exactly as requested. The safer path is a maintainer-designed admin/fleet API that reuses existing update and status plumbing without accidentally widening privileged access.

## Security Review

Status: needs_attention

Summary: The requested update trigger is security-sensitive because it would expose package-manager and restart work through a network admin surface.

Concerns:

- **[medium] Design the update-trigger HTTP boundary first:** `src/gateway/server-methods/update.ts:182`
  - body: `update.run` can invoke package-manager update work and schedule restarts, so HTTP exposure needs defined admin auth scope, rate limiting, audit metadata, restart coalescing, and target-version validation.
  - confidence: 0.92
- **[medium] Keep the admin HTTP RPC allowlist intentional:** `extensions/admin-http-rpc/src/methods.ts:49`
  - body: The current admin HTTP RPC plugin exposes `update.status` but not `update.run`; adding update triggering would expand a privileged network surface.
  - confidence: 0.9
- **[medium] Shape and redact update results for HTTP callers:** `src/gateway/server-methods/update.ts:245`
  - body: The current update response can include handoff command details, restart metadata, sentinel data, and step output, so an HTTP contract needs deliberate response shaping.
  - confidence: 0.84
