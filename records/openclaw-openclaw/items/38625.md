## Summary

Keep open. Current main still lacks both requested Crab-facing HTTP additions: shallow gateway health probes return only ok/status, while update orchestration remains on the admin-scoped WebSocket RPC surface rather than a POST /admin/update route.

## What This Changes

The issue requests adding a version field to the gateway health response and a token-authenticated POST /admin/update HTTP endpoint for Crab fleet monitoring.

## Best Possible Solution

Split the request: add health version metadata as an additive probe contract if maintainers approve it, and separately decide whether Crab should use documented update.run/update.status or a REST wrapper with equivalent admin scoping and safety controls.

## Reproduction Assessment

Yes, for source-level verification. Current main shows /health and /healthz serialize only ok/status, lacks a POST /admin/update route, and rejects target_version in the existing update RPC schema.

## Solution Assessment

Unclear. The version field is narrow and additive, but the HTTP update endpoint is security-sensitive and may not be the best path while the supported update contract is admin-scoped WebSocket RPC.
