## Summary

Keep open. Current main still only partially overlaps the request: runtime version is available through status/WS surfaces and admin HTTP RPC exists, but HTTP `/health` remains a shallow liveness response and there is no `POST /admin/update` or `target_version` contract; the update trigger is also a privileged network-control-plane design, not a safe cleanup close.

## What This Changes

Request to add a `version` field to `GET /health` and an authenticated HTTP update endpoint for fleet tools to trigger and inspect OpenClaw gateway updates.

## Best Possible Solution

Design an additive Gateway fleet/admin HTTP contract around the existing version, status, and update primitives, with an intentional security model before exposing update triggering to fleet tools.

## Reproduction Assessment

Not applicable: this is a new Gateway API feature request, not broken existing behavior. Source inspection on current main shows the requested HTTP `/health` version field and HTTP update trigger remain absent.

## Solution Assessment

No, not as a one-off `POST /admin/update` route exactly as requested. The safer path is a maintainer-designed admin/fleet API that reuses existing status and update plumbing without widening privileged access accidentally.

## Security Review

Status: needs_attention

Summary: The requested update trigger is security-sensitive because it would expose package-manager and restart work through a network admin surface.

Concerns:

- **[medium] Design the update-trigger HTTP boundary first:** `src/gateway/server-methods/update.ts:182`
  - body: `update.run` can invoke package-manager update work and schedule restarts, so HTTP exposure needs defined admin auth scope, rate limiting, audit metadata, restart coalescing, and target-version validation.
  - confidence: 0.92

  - body: The current admin HTTP RPC plugin exposes `update.status` but not `update.run`; adding update triggering would expand a privileged network surface.
  - confidence: 0.9
- **[medium] Shape and redact update results for HTTP callers:** `src/gateway/server-methods/update.ts:245`
  - body: The current update response can include handoff command details, restart metadata, sentinel data, and step output, so an HTTP contract needs deliberate response shaping.
