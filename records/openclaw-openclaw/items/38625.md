## Summary

Keep open. Current main has adjacent runtime-version surfaces and the existing admin-scoped `update.run` / `update.status` RPCs, but it still does not implement a `version` field on `GET /health` or a `POST /admin/update` HTTP API, and the update endpoint request needs maintainer product/security design.

## What This Changes

Add a gateway version field to the HTTP health response and an authenticated HTTP admin update endpoint for fleet tooling.

## Best Possible Solution

Design one stable fleet-management contract that reuses `hello-ok.server.version` and `update.run` / `update.status` while preserving admin scope, rate limits, audit metadata, restart behavior, target-version semantics, and log redaction.

## Reproduction Assessment

Not applicable. This is a new gateway API feature request, and source inspection verifies the requested HTTP health/admin contract is absent on current main.

## Solution Assessment

No, not as a direct automated change. The maintainable path is a maintainer-designed HTTP/control-plane API boundary that reuses existing runtime-version and update RPC behavior rather than adding an ad hoc host-update route.
