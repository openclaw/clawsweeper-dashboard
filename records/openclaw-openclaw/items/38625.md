## Summary

Keep open. Current main still lacks both requested API additions: `/health` returns only the shallow liveness payload, and the updater remains exposed through privileged gateway RPC rather than a `POST /admin/update` HTTP contract. The adjacent admin HTTP RPC PR is relevant but not a replacement because it is still open, uses a different route, and does not allow `update.run`.

## What This Changes

Request to add a `version` field to `GET /health` and a token-authenticated `POST /admin/update` endpoint for fleet update orchestration.

## Best Possible Solution

Design a public gateway fleet-management API that reuses existing runtime-version and update-runner surfaces while specifying admin auth, rate limiting, audit, target-version, restart, failure, and redaction semantics.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Source and test inspection on current main show `/health` still returns only `{ ok, status }` and no `/admin/update` or `target_version` HTTP contract exists.

## Solution Assessment

No, not as a one-off endpoint exactly as written. The maintainable path is an owner-designed gateway admin/fleet API, likely coordinated with the admin HTTP RPC work, that reuses existing update/version surfaces without weakening admin safeguards.
