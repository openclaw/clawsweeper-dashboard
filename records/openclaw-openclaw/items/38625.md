## Summary

Keep open: current main only partially overlaps the Crab fleet-tool request. The Gateway has authenticated WS `update.run` and an opt-in admin HTTP RPC plugin, but `/healthz` still has no version field and the admin HTTP RPC allowlist exposes `update.status`, not `update.run`, so the requested HTTP update orchestration remains a security/product decision.

## What This Changes

Request to expose the running OpenClaw version in gateway health output and add a gateway-token-authenticated HTTP update endpoint for fleet tooling.

## Best Possible Solution

Reuse the existing Gateway version resolver and canonical `update.run` machinery, but only expose them through the chosen public/version endpoint and opt-in admin HTTP RPC path after an explicit security review; avoid a parallel package-manager endpoint.

## Reproduction Assessment

Yes, at source level. Current main's `/healthz` response is pinned to `{ ok, status }`, and the opt-in admin HTTP RPC allowlist omits `update.run`, so the requested behavior is absent without needing a live gateway.

## Solution Assessment

No, not as a literal standalone `/admin/update` package-manager endpoint. The maintainable path is to extend the existing version and Gateway RPC surfaces only after maintainers approve the HTTP/security contract.
