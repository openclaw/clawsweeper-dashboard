## Best Possible Solution

Keep this open for the combined security/product decision: let https://github.com/openclaw/openclaw/issues/91795 carry the narrow version-only contract if maintainers prefer, and separately decide whether Admin HTTP RPC may expose the existing update.run path with explicit security tests and docs.

## Solution Assessment

No, not as one combined automatic change. Version reporting is a narrow additive contract, but HTTP-triggered updates should reuse update.run only after maintainers approve the security boundary and public API shape.
