## Summary

Keep open. Current main has adjacent pieces (`hello-ok.server.version` and admin-scoped `update.run` / `update.status`), but it does not implement the requested `version` field on the HTTP probe or a `POST /admin/update` HTTP API, and that new host-update surface needs maintainer product/security design.

## What This Changes

The issue requests adding a gateway version field to `GET /health` and a token-authenticated `POST /admin/update` endpoint for package-manager-driven upgrades.

## Best Possible Solution

Design one stable fleet-management contract that reuses the existing runtime version source and `update.run` implementation while preserving admin scope, rate limiting, audit logging, restart-sentinel behavior, and clear target-version semantics.

## Reproduction Assessment

Not applicable. This is a new gateway API feature request, and source inspection is enough to verify the exact requested HTTP health/admin contract is not present on current main.

## Solution Assessment

No for automated implementation as written. The maintainable path is a maintainer-designed API boundary, likely reusing `hello-ok.server.version` and `update.run`, rather than adding an independent host-update endpoint ad hoc.
