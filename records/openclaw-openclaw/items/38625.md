## Summary

Keep open. Current main only partially overlaps through the opt-in admin HTTP RPC plugin: the unauthenticated HTTP health probe still has no version field, and no HTTP surface exposes update triggering or the requested target-version contract.

## What This Changes

Request to expose the gateway runtime version in `GET /health` and add an authenticated HTTP admin update endpoint for Crab fleet orchestration.

## Best Possible Solution

Design a Gateway fleet-management API around the default-off admin HTTP RPC or another explicit admin surface, specifying version reporting, update status, update triggering, target-version policy, audit, restart, and redaction semantics together.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source inspection on current main shows the requested `/health` version field and HTTP update trigger are still absent.

## Solution Assessment

No, not as a one-off `POST /admin/update` route exactly as written. The safer path is a maintainer-designed admin/fleet API that reuses existing updater logic and the default-off admin HTTP RPC security model without widening privileged access accidentally.
