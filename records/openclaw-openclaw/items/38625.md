## Summary

Keep open. Current main still lacks the requested HTTP health `version` field and `POST /admin/update` endpoint; adjacent version and update RPC pieces exist, but exposing host package-manager updates over HTTP needs maintainer product/security design.

## What This Changes

Request adding a `version` field to `GET /health` and an authenticated `POST /admin/update` endpoint with optional `target_version` for fleet updates.

## Best Possible Solution

Design a public fleet-management contract that reuses existing runtime-version sources and `update.run` while explicitly defining auth scope, rate limits, audit, target-version, restart, and redaction semantics.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection on current main shows `/health` returns only `{ ok, status }` while no `/admin/update` route exists.

## Solution Assessment

No for an automated direct endpoint as written. The maintainable path is a maintainer-designed gateway protocol/HTTP API boundary that reuses existing update/version surfaces without bypassing admin safeguards.
