## Summary

Keep open: current main and v2026.6.10 still expose shallow `/health`/`/healthz` without version metadata, and the existing Admin HTTP RPC plugin exposes `update.status` but not `update.run`. The version-only half overlaps a narrower open issue, but the HTTP update trigger remains a security-sensitive product/API decision.

## What This Changes

Request adds runtime version metadata to Gateway health responses and an authenticated HTTP admin update trigger for fleet tooling.

## Best Possible Solution

Keep this open for the combined maintainer decision: let https://github.com/openclaw/openclaw/issues/91795 carry the narrow version-only contract, and separately decide whether Admin HTTP RPC may expose the existing `update.run` path or a dedicated update endpoint with explicit security docs and tests.

## Solution Assessment

No, not as one automatic combined change. Version reporting is a narrow additive API contract, but HTTP-triggered updates should reuse or wrap `update.run` only after maintainers approve the security boundary and public API shape.
