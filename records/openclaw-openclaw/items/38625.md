## Summary

Keep open. Current main has added a default-off admin HTTP RPC plugin, but it still does not add a `version` field to `GET /health` and it deliberately does not expose `update.run` or the requested `target_version` update contract over HTTP.

## What This Changes

Request to add Gateway fleet-management API support by exposing the runtime version in health responses and adding an authenticated HTTP update endpoint.

## Best Possible Solution

Design a Gateway fleet-management API around the default-off admin HTTP RPC or another explicit admin surface, with version reporting, update status, update triggering, target-version policy, audit, restart, and redaction semantics specified together.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source inspection on current main shows the requested `GET /health` version field and HTTP update trigger are still absent.

## Solution Assessment

No, not as a one-off `POST /admin/update` route exactly as written. The safer path is a maintainer-designed admin/fleet API that reuses the existing updater and the default-off admin HTTP RPC security model without widening privileged update access accidentally.
