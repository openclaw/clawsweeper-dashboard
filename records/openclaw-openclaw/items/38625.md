## Summary

Keep open. Current main has adjacent runtime-version and update RPC pieces, but it still does not implement the requested HTTP health `version` field or `POST /admin/update`; exposing host package-manager updates over HTTP needs maintainer product and security design.

## What This Changes

Request to add a `version` field to the HTTP health response and an authenticated `POST /admin/update` endpoint with optional `target_version`.

## Best Possible Solution

Design a stable fleet-management contract that reuses the existing runtime version source and `update.run` implementation while preserving admin scope, rate limiting, audit logging, restart-sentinel behavior, and clear target-version semantics.

## Reproduction Assessment

Not applicable. This is a new gateway API feature request, and source inspection verifies that the exact requested HTTP health/admin contract is not present on current main.

## Solution Assessment

No for automated implementation as written. The maintainable path is a maintainer-designed API boundary, likely reusing `hello-ok.server.version` or status `runtimeVersion` plus `update.run`, rather than adding an ad hoc host-update HTTP endpoint.
