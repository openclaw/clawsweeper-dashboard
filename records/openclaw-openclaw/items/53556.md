The issue requests backward-compatible Feishu channel configuration handling for renamed fields and legacy enum values after OpenClaw upgrades.

## Best Possible Solution

Add Feishu-owned compatibility for legacy top-level and per-account `botId`, with current `appId` taking precedence, an idempotent `openclaw doctor --fix` migration, documentation, and focused regression coverage.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Feishu plugin and its doctor compatibility contract; a broad startup-time migration system would conflict with the repository policy for legacy config repair.
