Full review comments:

- **[P2] Restore the global process stub after the ARM test:** `src/infra/platform-profile.test.ts:34`
  - body: `vi.stubGlobal("process", ...)` is not undone by `vi.restoreAllMocks()`, so later tests in the same worker can keep seeing the mocked ARM process. Add `vi.unstubAllGlobals()` in teardown or avoid replacing the whole process object.
  - confidence: 0.93
- **[P2] Clamp low-profile resize grids before the first encode:** `src/media/image-ops.ts:30`
  - body: The low-profile grid still starts with `sideStart`, so large inputs perform the expensive original-size encode before trying reduced sizes. Clamp the first low-profile entry, for example with `Math.min(sideStart, 1200)`, so the optimization starts lower.
  - confidence: 0.82
- **[P3] Validate systemd memory limit values before rendering:** `src/daemon/systemd-unit.ts:55-58`
  - body: The new `resourceLimits` strings are appended to the systemd unit without the CR/LF checks used for other rendered fields. Keep these values as a closed set or validate them before rendering so future callers cannot inject extra unit lines.
  - confidence: 0.74
