## Summary

Keep open. Current main still has a source-backed regression path where Anthropic CLI auth setup writes only legacy whole-agent runtime metadata, while runtime selection now requires provider/model-scoped policy to route canonical Anthropic models through `claude-cli`.

## What This Changes

Preserve Claude CLI execution for gateway-mediated Telegram and Dashboard turns after upgrading from 2026.5.7 to 2026.5.12 when Anthropic CLI auth is configured.

## Best Possible Solution

Update Anthropic CLI auth setup and migration to write provider/model-scoped or provider-wildcard `claude-cli` runtime policy for canonical Anthropic refs, with regression coverage that keeps whole-agent runtime keys ignored.

## Reproduction Assessment

Yes, by source inspection. Claude CLI auth setup can produce canonical Anthropic defaults without model-scoped `claude-cli` runtime policy, and current runtime routing then falls through to the non-CLI provider path.

## Solution Assessment

Yes. The narrow maintainable fix is to make auth migration emit the existing provider/model runtime-policy contract, not to restore ignored whole-agent runtime keys or broaden into unrelated helper-path work.
