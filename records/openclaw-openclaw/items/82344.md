## Summary

Keep open. The report matches a current-main setup regression: Anthropic Claude CLI auth setup still writes legacy whole-agent runtime policy, while current runtime selection ignores that policy and expects provider/model-scoped `agentRuntime`, so gateway-mediated turns can fall back to embedded/API behavior instead of Claude CLI.

## What This Changes

The issue asks OpenClaw to preserve Claude CLI-backed Telegram and Dashboard/UI gateway behavior after upgrading from 2026.5.7 to 2026.5.12.

## Best Possible Solution

Update Anthropic CLI auth setup and repair paths so they write provider/model-scoped Claude CLI runtime policy for selected Anthropic models, then add regression coverage proving the configured gateway route resolves to `claude-cli`.

## Reproduction Assessment

Yes, at source level. The Anthropic CLI setup path writes `agents.defaults.agentRuntime`, while current runtime selection and docs ignore that key and only honor provider/model-scoped policy.

## Solution Assessment

Yes. The narrow maintainable fix is to align the Anthropic CLI setup/doctor path with the existing provider/model runtime contract, not to re-enable whole-agent runtime pins.
