Keep open. Current main still has a source-backed regression path: Claude CLI auth setup can write canonical Anthropic model refs plus only legacy whole-agent runtime metadata, while gateway execution now selects CLI backends only from provider/model runtime policy.

## What This Changes

The issue reports a regression where gateway/UI/Telegram Claude CLI runs fall back to Anthropic third-party billing behavior after upgrading to 2026.5.12, while direct CLI use still works.

## Best Possible Solution

Preserve Claude CLI execution for canonical Anthropic refs by writing or repairing provider/model runtime policy during Anthropic CLI auth setup and legacy migration, without reviving ignored whole-agent runtime pins.

## Reproduction Assessment

Yes for a high-confidence source reproduction path: the migration can produce canonical Anthropic model entries plus only `agents.defaults.agentRuntime`, and current runtime selection ignores that key. I did not run a live gateway or Telegram Claude CLI session.
