## Label Justifications

- `impact:message-loss`: The requested route decides whether the intended specialist receives and handles an inbound message rather than requiring an indirect relay.

## Summary

Current main still lacks content-sensitive bindings or a merged same-turn pre-route hook, so the feature remains necessary; maintainers must choose between a first-class binding contract and a generic plugin routing API.

## What This Changes

Add content-pattern matching and optional matched-prefix stripping to agent bindings.

## Best Possible Solution

Define one generic same-turn pre-route contract that receives normalized inbound text, can select the agent/session, and may explicitly transform only the agent-facing body; add binding syntax only if maintainers want content routing as a stable core configuration feature.

## Reproduction Assessment

Not applicable; this requests a new routing mode rather than reporting broken documented behavior. Current source provides high-confidence confirmation that the capability is absent.

## Solution Assessment

No, the proposed binding fields are not yet proven to be the best maintainable boundary. A generic pre-route plugin API better matches repository direction if it can cover same-turn agent selection and narrowly scoped body transformation.
