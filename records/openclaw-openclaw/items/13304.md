## Summary

Keep open. Current main still only supports static binding selectors, and content/prefix routing would add a new validated config and routing capability with unresolved precedence, regex safety, cache, normalization, and body-rewrite semantics.

## What This Changes

Add optional content pattern matching to agent bindings so inbound messages can route to different agents by prefix or regex.

## Best Possible Solution

Define a first-class validated content matcher only after maintainers settle the API contract, then plumb normalized inbound text through representative channel/plugin routing paths and document the behavior with regression coverage.

## Reproduction Assessment

Yes for the current limitation. Source inspection shows strict config validation rejects match.content, and resolveAgentRoute cannot receive message text or return a stripped agent-facing body.

## Solution Assessment

Unclear. A binding-level matcher is plausible, but the proposed exact shape is not yet clearly the narrowest maintainable solution without decisions on precedence, regex safety, caching, normalization, and body rewriting.
