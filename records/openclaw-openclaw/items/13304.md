## Summary

Keep open: current main still lacks declarative content matchers in bindings, and the related pre-route hook and Discord mention-routing threads are adjacent rather than superseding this feature. The request needs maintainer API/product design before implementation.

## What This Changes

The issue requests routing inbound messages to different agents from binding rules based on message content patterns such as prefixes or regexes.

## Best Possible Solution

Define a validated binding content-matcher contract with deterministic precedence, safe pattern handling, content-aware cache behavior or bypass, normalized inbound-text propagation, documented stripped-body semantics, and focused config/resolver/channel coverage.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new routing/config capability request, not broken behavior under an existing contract. Source inspection on current main confirms the requested content matcher and rewritten body path are absent.

## Solution Assessment

Unclear: a binding-level matcher is plausible, but maintainers still need to settle precedence, regex safety, route-cache behavior, normalized content input, and strip semantics versus the adjacent pre-route hook design.
