## Label Justifications

- `P2`: This is a concrete multi-agent routing improvement with clear workflow value, but it is not a regression or broken documented behavior.
- `impact:message-loss`: Without content-based routing, messages intended for a specialist agent can be routed through the wrong handler or require manual relay.
- `impact:session-state`: The requested matcher decides which agent/session owns an inbound turn, directly affecting session scoping and conversation state.

## Summary

Keep open. Current main and v2026.6.5 still only expose static binding selectors and static route resolution, while the related pre-route hook work is open/adjacent rather than a merged declarative binding solution.

## What This Changes

Requests optional binding content matchers, such as prefix or regex rules with optional stripping, so inbound messages can route to different agents based on text.

## Best Possible Solution

Settle the routing API direction first, then implement either a validated binding content matcher or a documented pre-route plugin path with deterministic precedence, safe matching, cache behavior, body rewrite semantics, docs, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new routing/config capability request, not broken behavior under an existing contract. Source inspection gives high confidence that match.content, content-aware route input, and rewritten body output are absent on current main and v2026.6.5.

## Solution Assessment

Unclear for the exact match.content proposal. The need is valid, but maintainers should decide whether declarative binding content rules or a broader pre-route plugin seam is the narrowest maintainable solution.

## Risks / Open Questions

- The exact API shape is unsettled: declarative match.content, a pre-route plugin hook, or both could become competing routing mechanisms if maintainers do not choose the boundary first.
- Regex or prefix routing needs explicit normalization, safe matching, length limits, and precedence rules to avoid accidental broad routing or ReDoS-style hazards.
- stripMatch needs a product decision on whether it rewrites only the agent-facing text or also raw body, command parsing, reply context, hook payloads, grouped history, and logs.
- Content-dependent routing conflicts with the current static route cache unless the cache key includes normalized content, content-rule routes bypass caching, or caching is split by matcher type.
