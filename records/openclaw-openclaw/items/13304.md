## Label Justifications

- `P2`: This is a normal-priority multi-agent routing enhancement with meaningful but bounded impact and no existing-behavior regression.
- `impact:message-loss`: The missing capability causes messages intended for a specialist agent to be routed elsewhere rather than delivered to that target.

## Summary

Keep open for a maintainer product/API decision: current main still cannot select an agent from inbound message content, and the related generic pre-routing work has neither merged nor established a canonical replacement. The user goal is valid, but the proposed core regex and body-stripping configuration conflicts with the repository’s preference for plugin-owned optional policy unless maintainers first approve that binding contract.

## What This Changes

Allow bindings to route inbound messages to agents by message-content pattern, optionally stripping the matched prefix before agent processing.

## Best Possible Solution

Approve one canonical same-turn pre-routing contract that exposes normalized inbound text and permits a typed agent-route decision; keep optional pattern policy in a plugin unless maintainers explicitly choose content matching as a first-class binding feature, and define deterministic precedence, safe matching, cache behavior, and agent-facing body transformation before implementation.

## Reproduction Assessment

Yes at the source-contract level: current main’s strict binding schema and route input have no content or body field, so the proposed configuration cannot be accepted or evaluated. This demonstrates a missing capability rather than a broken existing contract.

## Solution Assessment

No, not yet: adding regex and stripMatch directly to core bindings is not proven to be the narrowest maintainable solution. A typed generic pre-routing plugin seam better matches current architecture, but maintainers must first define its authority and safety boundaries.

## Risks / Open Questions

- A core regex matcher would create unresolved precedence between static bindings, parent-peer routing, mention behavior, and default routing.
- Regex compilation and execution need explicit validation and denial-of-service safeguards before patterns can process untrusted inbound text.
- Content-dependent decisions invalidate the existing route-cache assumption unless content bindings bypass caching or use a carefully bounded key strategy.
- stripMatch has undefined effects on command parsing, raw versus normalized bodies, quotations, attachments, reply context, transcripts, and channel-specific preprocessing.
- A broad plugin hook that can redirect or suppress delivery would expand a public compatibility and security boundary beyond this narrower use case.
