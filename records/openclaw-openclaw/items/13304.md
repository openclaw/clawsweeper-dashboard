## Summary

Keep open. Current main still supports only static binding selectors, and this remains a new routing/config capability whose precedence, regex safety, cache behavior, normalization, and body-rewrite semantics need maintainer design first.

## What This Changes

The issue requests optional binding matchers that route agents by message-content patterns and optionally strip the matched prefix before forwarding.

## Best Possible Solution

Define a first-class validated binding content matcher only after maintainers settle precedence, regex safety, cache/body contracts, and representative channel/plugin plumbing, then document it with regression coverage.

## Reproduction Assessment

Yes for the current limitation. Source inspection shows strict config validation rejects match.content and the route resolver cannot receive message text or return a stripped body.

## Solution Assessment

Unclear. A binding-level matcher is plausible, but the proposed exact shape is not yet clearly the safest path until maintainers decide precedence, regex safety, cache behavior, normalization, and body rewriting.
