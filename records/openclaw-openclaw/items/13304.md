## Label Justifications

- `P2`: This is a meaningful multi-agent routing improvement with bounded impact, but it is neither an urgent regression nor a core availability failure.
- `impact:message-loss`: A wrong content-routing decision can misroute the message and effectively suppress the response from the intended specialist agent.
- `impact:session-state`: The routing decision selects the agent session and conversation context that receive and persist the inbound turn.

## Summary

Current main still supports only static binding selectors and has no pre-routing hook that can consume message text, select another agent for the same turn, or return a rewritten body. The closed hook PR is unmerged and conflicting, while the broader open hook issue requires a larger public API decision, so neither implements nor safely supersedes this distinct feature request.

## What This Changes

Add content or prefix matching to agent bindings so inbound messages can select specialist agents and optionally remove the matched prefix before processing.

## Best Possible Solution

Define one transport-independent pre-routing decision contract that receives normalized inbound text, applies deterministic precedence, avoids invalid static caching, returns a typed agent/session decision, and treats any agent-facing body rewrite as an explicit separate result.

## Reproduction Assessment

Yes at the source-contract level: current-main strict binding validation and route inputs conclusively show that message-content selection and prefix stripping are unavailable. This is a missing feature rather than broken documented behavior.

## Solution Assessment

No, the proposed binding-only object is not yet established as the best fix because it embeds content policy and body mutation directly into core routing; a generic typed pre-routing seam is more reusable, but maintainers must first settle its scope and compatibility contract.
