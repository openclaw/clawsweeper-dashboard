## Label Justifications

- `P2`: This is a useful but cross-cutting multi-agent routing enhancement with limited urgency and significant API-design work.
- `impact:message-loss`: A content-routing decision can misroute, suppress, or deliver an inbound message to the wrong agent if precedence or matching is incorrect.
- `impact:session-state`: Selecting a different agent changes which canonical session and conversation state receive the inbound message.

## Summary

Current main still supports only static binding selectors, while content-sensitive routing would require new config, routing, cache, channel-input, and message-rewrite contracts. The broader pre-routing hook remains an open product/API discussion, so it neither implements nor safely supersedes this distinct declarative request.

## What This Changes

Add regex or prefix-based message-content matching, with optional matched-prefix stripping, to agent bindings.

## Best Possible Solution

Define one first-class pre-routing decision contract with normalized inbound text and deterministic precedence, then either expose validated declarative content rules through it or let plugins implement optional classifiers without duplicating channel-specific routing logic.

## Reproduction Assessment

Not applicable as a failing bug reproduction; this is a new capability request. Source inspection does establish the limitation with high confidence because current bindings, validation, resolver inputs, cache keys, and routing tiers contain no message-content contract.

## Solution Assessment

Unclear. Declarative bindings would be ergonomic, but maintainers should first choose the permanent core-config versus generic plugin-hook boundary and specify precedence, caching, regex safety, and body-rewrite semantics.
