## Label Justifications

- `P2`: This is a normal-priority routing improvement with real multi-agent workflow impact, but it is a feature/product decision rather than an urgent regression.
- `impact:message-loss`: Without content-aware routing, messages intended for a specialist agent can be misrouted to the default agent or require manual forwarding through a middleman.
- `impact:session-state`: The routing decision determines which agent session receives and persists the inbound turn.

## Summary

Keep open. Current main and the latest release still expose static binding selectors only, and the overlapping pre-routing hook work in https://github.com/openclaw/openclaw/issues/81061 / https://github.com/openclaw/openclaw/pull/81198 is not a merged or product-approved substitute for a binding-level content matcher with stripping semantics.

## What This Changes

Request to add optional content pattern matching and stripMatch behavior to agent bindings so inbound messages can route to different agents by text prefix or regex.

## Best Possible Solution

Define one maintainer-approved routing extension with deterministic precedence, normalized text, safe matching, cache semantics, body-stripping behavior, docs, and focused routing/channel tests.

## Reproduction Assessment

Yes, at source level. Current main shows static binding types/schema, static route inputs/cache keys, and representative channel callers that do not carry message text or return stripped body text.

## Solution Assessment

Unclear. A binding content matcher is plausible, but OpenClaw's plugin-boundary guidance and the active pre-routing hook discussion mean maintainers should settle the API/product shape before implementation.
