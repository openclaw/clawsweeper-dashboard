## Summary

Keep open. Current main still exposes only static binding selectors, and the related mention/pre_route work is open or narrower, so this needs maintainer routing/API design rather than conservative cleanup.

## What This Changes

Request adding optional content, pattern, and stripMatch matchers to bindings so inbound message text can route group-chat messages to different agents.

## Best Possible Solution

Define a first-class validated content-routing contract only after maintainers settle precedence, normalization, safe matching, cache behavior, and body-rewrite semantics, then document it with focused routing/config/channel tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new routing capability, not broken documented behavior. Source inspection does confirm the current limitation because binding types/schema and resolveAgentRoute do not accept message text or return a stripped body.

## Solution Assessment

Unclear. A binding-level content matcher is plausible, but a pre_route hook or mention-specific override may be the better boundary for some use cases, so maintainers need to choose the public contract first.
