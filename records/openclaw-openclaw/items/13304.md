## Summary

Keep open. Current main and v2026.5.12 still expose only static binding selectors, and the related hook work is either closed unmerged or tracks a different plugin-level interception API rather than declarative `bindings[].match.content` routing with body stripping.

## What This Changes

The issue requests optional content pattern matching in route bindings, including regex/prefix matching and an optional `stripMatch` rewrite before dispatching to the selected agent.

## Best Possible Solution

Define a first-class validated binding content matcher with deterministic precedence, safe matching rules, content-aware cache behavior, representative channel text plumbing, optional body-stripping semantics, docs, and focused config/routing tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new routing/config capability request. Source, docs, and latest-release inspection give high confidence that current OpenClaw lacks `match.content`, content-aware route input, and body-rewrite output.

## Solution Assessment

Unclear. Declarative `match.content` may be a maintainable direction, but precedence, normalization, regex safety, cache behavior, and `stripMatch` semantics need maintainer API/product decisions before implementation.
