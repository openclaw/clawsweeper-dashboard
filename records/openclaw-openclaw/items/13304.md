Keep open. Current main and the latest shipped release still expose only static binding selectors, and the related mention/pre-routing work does not provide the requested declarative `match.content` matcher or `stripMatch` body rewrite semantics.

## What This Changes

Request to add content-pattern matching and optional matched-prefix stripping to agent route bindings.

## Best Possible Solution

Define a first-class validated binding content matcher with deterministic precedence, safe matching rules, content-aware cache behavior, representative channel text plumbing, optional body-stripping semantics, docs, and focused config/routing tests.

## Solution Assessment

Unclear. Declarative `match.content` may be a maintainable direction, but precedence, normalization, regex safety, cache behavior, and `stripMatch` semantics need maintainer API/product decisions first.
