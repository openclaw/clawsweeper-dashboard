## Summary

Keep open: current main still only supports static binding selectors, and the active pre-route hook and Discord mention-routing threads do not supersede this binding-level content matcher request.

## What This Changes

The issue requests declarative content or prefix matching in agent bindings so inbound messages can route directly to different agents based on text.

## Best Possible Solution

Define a first-class validated binding content-matcher contract with deterministic precedence, safe pattern handling, content-aware cache behavior, normalized inbound-text propagation, documented stripped-body semantics, and focused config/resolver/channel coverage.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new routing/config capability rather than broken behavior under an existing contract. Source inspection on current main confirms the binding schema and resolver still lack the requested content matcher and body-rewrite path.

## Solution Assessment

Unclear. A binding-level content matcher is plausible, but maintainers should first decide precedence, regex safety, cache behavior, plugin-hook overlap, and stripMatch semantics.
