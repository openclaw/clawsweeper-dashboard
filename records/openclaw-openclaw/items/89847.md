## What This Changes

The issue reports that cron failure alerts are not delivered or are recorded as `unknown` when isolated runtime-plugin setup stalls before execution starts.

## Best Possible Solution

Keep this issue as the canonical runtime-stall failure-alert delivery/status tracker, then land or revise narrower fixes for actual alert-result persistence, topic routing, and runtime-stall diagnostics after maintainers decide the public status and best-effort semantics.

## Solution Assessment

No, not as one bundled fix. The durable solution should decouple and persist actual failure-alert delivery outcomes while keeping thread routing, explicit status vocabulary, and runtime-stall diagnostics coordinated but separable maintainer decisions.
