## Summary

Keep open. Current main still has no `session.maintenance` byte or line cap for active transcript `.jsonl` files; the existing `agents.defaults.compaction.maxActiveTranscriptBytes` path is useful but compaction-scoped and does not provide the requested maintenance cleanup control.

## What This Changes

The issue requests a `session.maintenance` mechanism to cap, rotate, or tail-trim individual active transcript `.jsonl` files so long-running chat sessions cannot grow without bound.

## Best Possible Solution

Add an explicit, documented transcript maintenance path for active `.jsonl` files, disabled by default, with `warn`/`enforce` semantics, cleanup dry-run visibility, and focused regression coverage.

## Reproduction Assessment

Yes. The issue gives concrete oversized transcript examples, and current-main source inspection shows no configurable maintenance field or runtime path that can cap an active referenced transcript.

## Solution Assessment

Yes. A focused `session.maintenance` transcript cap/rotation feature is the narrowest maintainable fix for this report; the existing compaction byte guard should remain separate because it is semantic compaction, not general maintenance cleanup.
