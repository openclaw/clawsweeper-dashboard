## Summary

Keep open: current main and the latest release still lack a per-transcript raw JSONL cap; compaction and disk-budget cleanup reduce adjacent pressure but do not close the raw transcript maintenance gap, and recent comments ask for a maintainer product decision.

## What This Changes

Requests a source-level policy to cap, rotate, summarize, or otherwise bound individual session transcript JSONL files under session maintenance.

## Best Possible Solution

Make the transcript policy explicit: either implement a maintainer-approved disabled-by-default raw transcript cap with schema, docs, and oversized-fixture regression coverage, or document compaction as the supported limit and close after that decision.

## Reproduction Assessment

Yes for the missing cap from source inspection: current main exposes no per-transcript raw JSONL cap and the append path keeps writing to the active file. No live oversized-file CPU reproduction was run in this read-only review.

## Solution Assessment

Unclear. A scoped disabled-by-default raw transcript policy could solve the reported failure, but maintainers still need to decide whether raw caps belong in session.maintenance or whether compaction-only behavior is the intended product contract.
