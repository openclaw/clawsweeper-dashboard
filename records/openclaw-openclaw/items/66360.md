## Summary

Keep open: current main and the latest release still lack a `session.maintenance` control that caps or rotates one active transcript JSONL file, while the existing disk-budget and compaction paths only cover adjacent cleanup or semantic compaction cases.

## What This Changes

Request a bounded session-maintenance policy that caps, rotates, or summarizes active transcript JSONL files so long-running sessions cannot grow without bound.

## Best Possible Solution

Choose one maintainer-approved active-transcript bounding policy, then align config schema, docs, cleanup behavior, locking/concurrency, and focused oversized-transcript regression tests around that policy.

## Reproduction Assessment

## Solution Assessment

Unclear. A disabled-by-default raw transcript cap is plausible, but maintainers first need to decide whether raw archival, semantic compaction, disk-budget cleanup, or documentation is the supported policy.
