## Summary

Keep open: current main still lacks a `session.maintenance` control that caps or rotates a single active transcript JSONL file, and the remaining work needs a maintainer decision because it changes live transcript retention semantics.

## What This Changes

The issue requests a bounded raw transcript maintenance policy so long-running `.jsonl` session transcripts cannot grow without limit and stall the gateway.

## Best Possible Solution

Choose one maintainer-approved active-transcript bounding policy, then align schema, docs, cleanup behavior, concurrency locking, and regression tests around that policy.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: current main has no `session.maintenance` field or runtime path that caps an individual active transcript JSONL file; I did not live-reproduce the reporter's exact CPU 100% stall.

## Solution Assessment

Unclear. A disabled-by-default `session.maintenance` raw transcript cap is plausible, but maintainers first need to decide whether raw archival, semantic compaction, disk-budget cleanup, or documentation is the supported policy.
