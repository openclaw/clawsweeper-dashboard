## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Resolve the thread root before plugin dispatch:** `extensions/mattermost/src/channel.ts:203`
  - body: `handleAction` only uses `implicitReplyToId` when `readMattermostReplyToId(params)` is empty. On current main, `handleSendAction` calls the shared reply resolver before plugin dispatch, and this PR's tool context still returns `currentMessageId`, so `params.replyTo` can already be the latest child post instead of the Mattermost thread root. Move the Mattermost thread-root choice into the shared resolver, or mark the context as thread-root-required so `currentThreadTs` wins before dispatch.
  - confidence: 0.87
- **[P2] Use the configured default Mattermost account:** `extensions/mattermost/src/channel.ts:474`
  - body: `buildToolContext` resolves `accountId ?? "default"`, bypassing `channels.mattermost.defaultAccount`. In configs where the selected default account has a different `replyToMode`, the message tool can inherit the wrong mode and route sends incorrectly. Use the same default-account resolution as the existing scoped reply mode path.
  - confidence: 0.84
