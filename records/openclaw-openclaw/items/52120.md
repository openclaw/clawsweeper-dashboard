## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Move implicit thread resolution before outbound routing:** `extensions/mattermost/src/channel.ts:201-203`
  - body: This fallback only assigns `replyToId` inside the Mattermost handler, after the shared runner has already prepared the outbound mirror/session route. A tool send can therefore land in the Mattermost thread while the mirrored transcript and later session lookup stay on the channel-root route; feed the inherited Mattermost thread id into the shared route-time path instead.
  - confidence: 0.9
- **[P2] Derive off-mode promotion from the actual thread root:** `extensions/mattermost/src/channel.ts:484-496`
  - body: `threadTs` comes only from `MessageThreadId` or `ReplyToId`, but Mattermost currently sets those from `effectiveReplyToId`, which is undefined for `replyToMode: off` even when the inbound post has a thread root. The advertised off-to-all promotion is therefore unreachable for real off-mode threaded messages; use the actual Mattermost root/thread fact instead.
  - confidence: 0.82
