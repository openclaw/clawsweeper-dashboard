## Best Possible Solution

## Reproduction Assessment

Yes, at source level. Current main lacks Mattermost `buildToolContext`/`resolveAutoThreadId`, and the canonical issue plus a later user comment describe the same `replyToMode: "all"` thread escape in real Mattermost use.

## Solution Assessment

No. The direction is right, but the patch resolves the implicit thread inside plugin dispatch instead of the shared pre-dispatch route path, so the safer solution is a current-main rework that keeps delivery and route state aligned.
