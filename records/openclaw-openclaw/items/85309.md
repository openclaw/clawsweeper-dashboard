## Summary

Keep this PR open for maintainer handling: it appears to fix the source-reproducible QQ Bot media-root bug and has strong live proof, but it is protected by the `maintainer` label and intentionally changes which local media root is trusted when `OPENCLAW_HOME` is set.

## What This Changes

The PR changes QQ Bot media path helpers and regression tests so media roots honor `OPENCLAW_HOME` while QQ Bot persisted data paths stay under the OS home.

## Best Possible Solution

Land the narrow QQ Bot media-root fix after a maintainer explicitly accepts the `OPENCLAW_HOME` media-root compatibility behavior, or request a transitional legacy-root path if preserving old OS-home media references is required.

## Solution Assessment

Yes, with maintainer sign-off on compatibility. Moving only the media root to the effective OpenClaw home is a narrow owner-boundary fix, but the old OS-home media-root rejection is a deliberate upgrade behavior choice.
