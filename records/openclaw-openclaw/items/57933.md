## Summary

Keep this issue open: current main still has coarse session lifecycle and chat/tool streaming pieces, but not the requested per-session heartbeat/status model or Control UI liveness badges.

## What This Changes

Requests per-session heartbeat and live health indicators for active Gateway sessions, including turn-phase state, stall detection, and Control UI status badges.

## Best Possible Solution

Add an authenticated, additive turn-health contract in the Gateway and render compact live status badges in the Control UI sessions list, reusing existing session/chat/tool event streams where possible.

## Reproduction Assessment

Not applicable as a feature request, but the absence is reproducible by source inspection: current schemas, handlers, UI rows, and docs do not expose the proposed heartbeat/status states.

## Solution Assessment

No, current main is not the best solution yet; the maintainable direction is a typed Gateway session-health model with scoped WebSocket updates and a session-list UI projection rather than log parsing or model calls.
