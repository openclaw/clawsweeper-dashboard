## What This Changes

The PR adds a shared Swift inline-command parser, routes macOS allowlist resolution and system-run validation through it, and expands Swift tests for combined POSIX `c` flags and startup-wrapper handling.

## Best Possible Solution

## Reproduction Assessment

Yes. From source inspection, current main only recognizes exact POSIX inline flags in `ExecShellWrapperParser`, while the Swift validator and TypeScript inline parser already handle combined `c` forms; I did not run the Swift lane in this read-only review.

## Solution Assessment

No, not as currently proposed. Sharing the parser is a maintainable direction, but this head regresses canonical `/bin/sh -lc` allowlist matching and still lacks real behavior proof, a green macOS Swift check, and a changelog entry.
