## What This Changes

The PR filters unreadable command prompt tool names, returns the filtered tool list from `resolveCommandsSystemPromptBundle`, and adds a regression for one unreadable tool name plus one healthy sibling.

## Best Possible Solution

Keep the unreadable-name quarantine, but return original healthy tool instances and carry the separately resolved name list into prompt generation, with a regression covering a class-backed healthy tool beside an unreadable-name sibling.

## Reproduction Assessment

Yes from source. Current main can crash when `tool.name` throws, and the PR’s clone-based fix can break supported class-backed tool accessors that downstream report/export paths still read.

## Solution Assessment

No. The right boundary is the command prompt bundle, but the best fix should cache or collect readable names while preserving original healthy tool instances.
