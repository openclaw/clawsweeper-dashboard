## What This Changes

The PR adds filtering for unreadable tool names in the auto-reply command system-prompt bundle and regression coverage for skipping a malformed descriptor while preserving a healthy sibling.

## Best Possible Solution

Keep the unreadable-name quarantine, carry resolved names separately for prompt generation, return original healthy `AgentTool` instances, and add class/proxy-backed regression coverage before rerunning the focused checks.

Yes. Current main and v2026.6.9 read `tool.name` outside the tool-construction guard, and source inspection shows the PR's clone-based fix can break later `tool.parameters` reads in `/context` and `/export-session`.

## Solution Assessment

No. Guarding unreadable names at the command system-prompt bundle is the right boundary, but the best fix should preserve original healthy tool instances instead of cloning them.
