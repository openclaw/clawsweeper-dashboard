## Summary

Keep open: this PR targets a valid source-reproducible opencode-go stream-liveness bug, but the submitted predicate change can also count a pre-delta `toolcall_start` as progress and shorten the protected first-event window. ## What This Changes

The branch adds four block-boundary event types to the opencode-go stalled-stream liveness predicate.

## Best Possible Solution

## Solution Assessment

No, not as submitted. The correct layer is the provider-owned opencode-go wrapper, but the implementation should preserve the first-event contract by accepting boundary liveness only after real provider progress begins.
