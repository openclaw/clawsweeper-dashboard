## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Reject computed module specifiers before execution:** `src/plugins/import-policy.ts:270-274`
  - body: The policy only records quoted literal `import`/`require` forms, so a plugin can hide the restricted request behind a variable or string expression and still execute it through the module loader. For a claimed import boundary, non-literal specifiers need to be rejected or enforced at runtime before plugin code runs.
  - confidence: 0.94
- **[P1] Reject private OpenClaw imports before allowlists:** `src/plugins/import-policy.ts:371-375`
  - body: `allowedPackages` and the plugin package name return success before the private `openclaw/*` guard runs. A plugin that declares `openclaw` can import `openclaw/runtime` and bypass the exact boundary this PR is trying to enforce.
  - confidence: 0.97
- **[P1] Scan plugin-local dependency entrypoints:** `src/plugins/import-policy.ts:439-455`
  - body: After a declared bare dependency passes the allowlist check, validation immediately continues without resolving or scanning the dependency's entrypoint. A plugin can therefore ship a package under its own `node_modules` that imports private OpenClaw internals after the entry file passes policy.
  - confidence: 0.9
- **[P1] Add an upgrade-safe enforcement path:** `src/plugins/loader.ts:1150-1160`
  - body: This turns import-policy violations into immediate plugin load failures for all affected plugins, but the docs keep existing external plugins as a compatibility baseline and the PR does not add migration, warning, doctor, or real third-party-plugin upgrade proof. Maintainers need an explicit fail-closed policy or a staged compatibility path before merge.
  - confidence: 0.88
- **[P2] Decode file URLs before root checks:** `src/plugins/import-policy.ts:330-334`
  - body: The file URL path checks use `new URL(...).pathname`, which keeps percent escapes and can disagree with Node's file URL to filesystem path conversion. Use `fileURLToPath()` before normalization so encoded dot segments and platform path rules cannot bypass or false-fail the plugin-root check.
  - confidence: 0.82
- **[P2] Resolve overlay forwards before allowing them:** `src/plugins/import-policy.ts:63-75`
  - body: The runtime-overlay exception allows any raw specifier starting with `../../../dist/` before resolving the rest of the path. A suffix containing additional `..` segments can escape the intended dist root, so this allowlist needs to check the resolved real path instead of the raw string prefix.
  - confidence: 0.84
- **[P2] Match package directory resolution for local imports:** `src/plugins/import-policy.ts:318-324`
  - body: Local imports only try direct file names and `index.*`, but package directories can resolve through package metadata. Valid in-plugin imports like `require('./dir')` can be rejected even though the runtime loader would resolve them, creating avoidable compatibility failures.
  - confidence: 0.78
- **[P3] Remove the release-owned changelog edit:** `CHANGELOG.md:55`
  - body: This PR adds a normal feature entry directly to `CHANGELOG.md`, but repository policy says changelog generation is release-owned. Keep the release-note context in the PR body or squash message instead of merging this file change.
  - confidence: 0.86
