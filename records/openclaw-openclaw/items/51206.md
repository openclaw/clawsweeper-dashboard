Overall confidence: 0.9

Full review comments:

- **[P1] Reject private imports before dependency allowlist:** `src/plugins/import-policy.ts:371-372`
  - body: `resolveBarePackageName("openclaw/runtime")` returns `openclaw`, so a plugin that declares `openclaw` in its package.json returns at `allowedPackages` before the private-path guard. That lets the exact private import this policy is meant to block pass validation.
  - confidence: 0.94
- **[P1] Deny computed module specifiers:** `src/plugins/import-policy.ts:269-274`
  - body: The policy only records regex-matched literal specifiers; `require("openclaw/" + "runtime")` or `import(name)` is not collected, and the loader then evaluates the module normally. Either reject non-literal module loads or enforce the boundary in runtime resolution.
  - confidence: 0.88
- **[P1] Validate overlay imports after resolving the target:** `src/plugins/import-policy.ts:395-396`
  - body: `isRuntimeOverlayForwardImport()` accepts any raw specifier starting with `../../../dist/`, and `classifyLocalSpecifierViolation()` returns `null` before computing `rawResolved` or checking `canonicalDistRoot`. A later `..` segment can escape the intended dist root while still passing policy.
  - confidence: 0.82
- **[P2] Support package.json directory resolution for local imports:** `src/plugins/import-policy.ts:318-324`
  - body: Node resolves `require("./dir")` through `./dir/package.json` before falling back to index files, but this resolver only tries direct files and `index.*`. Valid in-package helper directories can therefore be rejected even though runtime module resolution would load them.
  - confidence: 0.86
