## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Reject private OpenClaw imports before allowlists:** `src/plugins/import-policy.ts:371-372`
  - body: `allowedPackages.has(barePackage)` returns before the private `openclaw/*` guard, so a plugin can declare `openclaw` in its package manifest and still import `openclaw/runtime`. That defeats the central boundary this PR is trying to enforce.
  - confidence: 0.95
- **[P1] Reject non-literal specifiers before evaluation:** `src/plugins/import-policy.ts:273-274`
  - body: The scanner only records literal `require()` / `import()` specifiers, but the loader still evaluates the plugin afterward. A computed specifier such as `require("openclaw/" + "runtime")` is not reported and can bypass the policy.
  - confidence: 0.9
- **[P2] Resolve overlay imports before accepting them:** `src/plugins/import-policy.ts:75`
  - body: The runtime-overlay exemption returns success from a raw `../../../dist/` prefix before any resolved-path check. A specifier with traversal after that prefix can skip the root/canonical-dist checks the policy relies on.
  - confidence: 0.86
- **[P2] Convert file URLs with fileURLToPath:** `src/plugins/import-policy.ts:333-334`
  - body: Using `new URL(specifier).pathname` for filesystem checks leaves percent-encoding in the path, while Node's file URL resolution decodes it. That mismatch can make the policy approve a different path than runtime loads.
  - confidence: 0.83
- **[P2] Support package directory entry resolution:** `src/plugins/import-policy.ts:318-324`
  - body: Local resolution tries direct files and `index.*`, but Node also honors directory package entries such as `package.json` `main` and `exports`. Valid in-plugin imports like `require("./dir")` can be rejected even though runtime resolution would load them.
  - confidence: 0.82
- **[P3] Remove the release-owned changelog entry:** `CHANGELOG.md:55`
  - body: Normal PRs should not edit `CHANGELOG.md`; release generation owns that file. Keep the release-note context in the PR body or squash message instead.
  - confidence: 0.8
