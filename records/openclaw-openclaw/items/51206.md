## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Reject computed module specifiers before execution:** `src/plugins/import-policy.ts:256-279`
  - body: The policy only sees static string-literal imports through these regexes. A plugin can build `require("openclaw/" + "runtime")` or `import(name)`, produce no violation, and then be executed by the loader, which defeats the stated pre-execution boundary.
  - confidence: 0.92
- **[P1] Reject private OpenClaw imports before allowlists:** `src/plugins/import-policy.ts:371-372`
  - body: `allowedPackages.has(barePackage)` returns before the private `openclaw/*` check. A plugin that declares `openclaw` in its package manifest can still import `openclaw/runtime`, so the main boundary this PR adds is bypassable.
  - confidence: 0.96
- **[P2] Resolve overlay forwards before allowing them:** `src/plugins/import-policy.ts:389-396`
  - body: The runtime-overlay branch returns success from a raw `../../../dist/` prefix before resolving or normalizing the target. A specifier with additional `..` segments can skip the plugin-root and canonical-dist checks, so the exception should verify the resolved path before returning.
  - confidence: 0.86
- **[P3] Remove the release-owned changelog entry:** `CHANGELOG.md:55`
  - body: `CHANGELOG.md` is generated for releases, and normal PRs should put release-note context in the PR body or commit metadata. This entry should be removed from the branch before merge.
  - confidence: 0.88
