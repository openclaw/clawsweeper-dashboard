## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Reject private OpenClaw imports before allowlists:** `src/plugins/import-policy.ts:371-375`
  - body: `allowedPackages.has(barePackage)` returns before the private `openclaw/*` guard, so a plugin can declare `openclaw` and still import `openclaw/runtime`. Move the private OpenClaw rejection ahead of self/dependency allowlists and add a bypass test.
  - confidence: 0.95
- **[P1] Reject computed module specifiers before execution:** `src/plugins/import-policy.ts:256-279`
  - body: The scanner only collects static string-literal forms, so `require("openclaw/" + "runtime")` or `import(getSpecifier())` is invisible to the policy and still executes. Either reject non-literal module specifiers during parsing or enforce the boundary at module resolution time.
  - confidence: 0.9
- **[P1] Scan plugin-local dependency entrypoints:** `src/plugins/import-policy.ts:439-455`
  - body: Bare imports stop after the allowlist check, so a declared dependency inside the plugin root can import forbidden OpenClaw internals without being scanned. Resolve allowed bare specifiers from the importer and traverse plugin-local dependency code, or block forbidden resolution at runtime.
  - confidence: 0.88
- **[P1] Resolve overlay imports before allowing them:** `src/plugins/import-policy.ts:389-396`
  - body: The runtime-overlay exception allows any raw specifier starting with `../../../dist/`, so traversal after that prefix can bypass the normal root checks. Resolve and realpath the target before allowing the overlay import.
  - confidence: 0.86
- **[P1] Convert file URLs with fileURLToPath:** `src/plugins/import-policy.ts:329-334`
  - body: Using `new URL(specifier).pathname` for filesystem policy decisions leaves percent-encoded path segments undecoded, which can diverge from Node's file URL loading behavior. Convert file URLs with `fileURLToPath()` before normalizing and checking plugin-root containment.
  - confidence: 0.82
- **[P1] Add an upgrade-safe enforcement path:** `src/plugins/loader.ts:1154-1160`
  - body: This turns import-policy violations into pre-evaluation load failures for every plugin, but the PR only says migration is maybe needed. Existing plugins can stop loading unless maintainers approve a staged or default-safe rollout.
  - confidence: 0.88
- **[P2] Resolve local package directories like Node:** `src/plugins/import-policy.ts:318-324`
  - body: Local path resolution only tries direct files and `index.*`, but Node also resolves `require("./dir")` through `dir/package.json` `main`/exports before falling back. Valid in-package directory imports can be rejected even though runtime loading would succeed.
  - confidence: 0.78
