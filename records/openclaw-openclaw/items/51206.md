## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Reject private OpenClaw imports before package allowlists:** `src/plugins/import-policy.ts:371-375`
  - body: `allowedPackages` is checked before the private `openclaw`/`openclaw/*` rejection, so a plugin can declare `openclaw` in dependencies and then import `openclaw/runtime` while passing this policy. That defeats the core boundary the PR is adding, so the private OpenClaw guard needs to run before dependency allowlists, with regression coverage for a declared `openclaw` dependency.
  - confidence: 0.96
- **[P1] Reject non-literal module specifiers before plugin execution:** `src/plugins/import-policy.ts:256-279`
  - body: The scanner only collects quoted literal `import`, dynamic `import()`, `require`, and `require.resolve` specifiers, so computed values such as `require('openclaw/' + 'runtime')` are never validated before the loader executes the plugin. If this PR is enforcing a boundary rather than linting common cases, non-literal specifiers must be denied or module resolution must be enforced at runtime.
  - confidence: 0.91
- **[P1] Resolve overlay imports before accepting the dist shortcut:** `src/plugins/import-policy.ts:389-396`
  - body: `classifyLocalSpecifierViolation` returns success for any runtime overlay specifier starting with `../../../dist/` before resolving the path. A specifier with traversal after that prefix can escape the intended dist root while bypassing the later inside-root checks, so the allowlist needs to be based on the normalized resolved path.
  - confidence: 0.88
- **[P2] Convert file URLs before applying root checks:** `src/plugins/import-policy.ts:329-334`
  - body: The policy uses `new URL(specifier).pathname` for `file:` specifiers, but Node filesystem conversion decodes URL escapes. Percent-encoded dot segments can make the policy and runtime disagree about the real path, so file URLs should be converted with Node's filesystem URL helper before root checks.
  - confidence: 0.84
- **[P2] Support package.json directory resolution for local imports:** `src/plugins/import-policy.ts:318-324`
  - body: Local path resolution only tries direct files plus `index.*`, but Node also resolves `require('./dir')` through `dir/package.json` `main` or exports. Valid in-package plugin imports using that standard pattern will be rejected even though runtime resolution stays inside the plugin root.
  - confidence: 0.86
- **[P3] Remove the release-owned changelog entry:** `CHANGELOG.md:55`
  - body: Normal PRs should not edit `CHANGELOG.md`; release generation owns that file. Keep the release-note context in the PR body or squash message instead of merging a manual changelog line.
  - confidence: 0.95
