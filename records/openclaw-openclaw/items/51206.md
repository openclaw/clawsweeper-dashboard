## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Reject private OpenClaw imports before dependency allowlists:** `src/plugins/import-policy.ts:371-375`
  - body: `validateBareSpecifier` returns `null` for `allowedPackages.has(barePackage)` before checking `openclaw/*`. A plugin can declare `openclaw` as a dependency and still import `openclaw/runtime`, defeating the private-import boundary this PR adds.
  - confidence: 0.97
- **[P1] Reject computed module specifiers before execution:** `src/plugins/import-policy.ts:273-275`
  - body: The scanner only records string-literal import/require calls. Code such as `require("openclaw/" + "runtime")` is invisible to this policy and still runs when the loader evaluates the plugin module, so the intended boundary remains bypassable.
  - confidence: 0.94
- **[P1] Validate plugin-local dependency entrypoints:** `src/plugins/import-policy.ts:440-455`
  - body: After a declared bare package passes the allowlist check, validation immediately continues without resolving or scanning that package. A plugin-local dependency can therefore import private OpenClaw internals or escape the plugin root while the top-level plugin passes policy validation.
  - confidence: 0.92
- **[P1] Constrain runtime overlay imports after resolution:** `src/plugins/import-policy.ts:389-396`
  - body: The runtime overlay exception returns success for any raw specifier starting with `../../../dist/` before resolving the path. A suffix such as `../../outside.js` after that prefix bypasses the later root/canonical-dist checks.
  - confidence: 0.9
- **[P1] Add an upgrade-safe enforcement path:** `src/plugins/loader.ts:1150-1160`
  - body: This changes policy violations into load errors for every plugin with no advisory, doctor check, compatibility mode, or documented rollout. Existing installed plugins that rely on private imports or undeclared dependency reach-through can stop loading immediately after upgrade.
  - confidence: 0.88
