Full review comments:

- **[P1] Reject private OpenClaw imports before allowlists:** `src/plugins/import-policy.ts:371-372`
  - body: `validateBareSpecifier` returns success for a package-name match or declared dependency before it checks `openclaw/*`. A plugin can declare `openclaw` and then import `openclaw/runtime`, bypassing the private import boundary this PR is adding.
  - confidence: 0.96
- **[P1] Reject computed module specifiers or enforce at runtime:** `src/plugins/import-policy.ts:256-279`
  - body: The policy only records literal specifiers matched by these regexes, while Node accepts computed `require()` and `import()` specifiers at runtime. A plugin can build `openclaw/runtime` from a variable and pass validation before the loader evaluates it.
  - confidence: 0.9
- **[P1] Scan plugin-local dependency entrypoints:** `src/plugins/import-policy.ts:439-455`
  - body: After allowing a declared bare dependency, validation immediately continues and never inspects the resolved dependency code under the plugin root. That lets a plugin hide forbidden private imports or root escapes inside a declared local dependency.
  - confidence: 0.88
- **[P1] Add an upgrade-safe enforcement path:** `src/plugins/loader.ts:1154-1160`
  - body: This turns any policy violation into a plugin load error before module evaluation. Existing external plugins are a documented compatibility baseline, so fail-closed startup behavior needs a staged rollout, migration/doctor path, or explicit maintainer acceptance with upgrade proof.
  - confidence: 0.86
- **[P3] Remove the normal PR changelog edit:** `CHANGELOG.md:55`
  - body: `CHANGELOG.md` is release-owned in this repo, and normal PRs should keep release-note context in the PR body or commit message. This line should be dropped before merge.
