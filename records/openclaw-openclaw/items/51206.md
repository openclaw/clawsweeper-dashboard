## Summary

Keep open. Current `main` does not contain the PR's loader-time plugin import-boundary enforcement: there is no `src/plugins/import-policy.ts`, the loader still proceeds from entry-path boundary checks straight into `getJiti(...)` evaluation for both runtime and CLI metadata loads, and the supporting tests/changelog entry from the PR are not present on `main`. This is not strong evidence for any close reason.
