Full review comments:

- **[P1] Reject computed module specifiers before execution:** `src/plugins/import-policy.ts:256-280`
  - body: The policy only collects literal specifiers with regexes, but plugin modules still execute runtime `require` and `import` calls. A plugin can compute `openclaw/runtime` or an escaping relative path and avoid this scan unless non-literal module loads are rejected or the boundary is enforced during resolution.
  - confidence: 0.92
- **[P1] Reject private OpenClaw imports before allowlists:** `src/plugins/import-policy.ts:366-375`
  - body: `allowedPackages` and the plugin package name are checked before the private `openclaw` rejection. A plugin can declare `openclaw` and then import `openclaw/runtime`, skipping the guard this PR is meant to add.
  - confidence: 0.96
- **[P1] Scan plugin-local dependency entrypoints before execution:** `src/plugins/import-policy.ts:440-455`
  - body: Declared bare dependencies are allowed and then skipped, so a plugin-local package can import private OpenClaw internals or escape the root during execution. The policy needs to resolve and inspect plugin-local dependency entrypoints or enforce the boundary at runtime.
  - confidence: 0.9
- **[P1] Add an upgrade-safe enforcement path:** `src/plugins/loader.ts:1154-1160`
  - body: This gate unconditionally rejects plugin loads before execution, but the PR says migration may be needed. Existing plugins that relied on private imports, undeclared dependencies, or tolerated resolution quirks can stop loading on upgrade without a warning, doctor repair, compatibility window, or maintainer-approved fail-closed policy.
  - confidence: 0.88
- **[P2] Resolve overlay forwards before allowing them:** `src/plugins/import-policy.ts:389-396`
  - body: The runtime-overlay special case returns success based only on the raw `../../../dist/` prefix. A suffix containing traversal segments can escape the intended canonical dist root because the resolved-path containment check is skipped.
  - confidence: 0.84
- **[P2] Match Node package directory resolution:** `src/plugins/import-policy.ts:318-324`
  - body: Local resolution only checks direct files and `index.*`, but Node also resolves directory entries through package metadata. Valid in-plugin imports like `require('./dir')` can be rejected even though runtime module resolution would load them.
  - confidence: 0.82
- **[P3] Remove the normal PR changelog edit:** `CHANGELOG.md:55`
