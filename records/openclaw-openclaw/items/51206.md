## Review Findings

Overall confidence: 0.94

Full review comments:

- **[P1] Reject private OpenClaw imports before dependency allowlists:** `src/plugins/import-policy.ts:371-372`
  - body: `validateBareSpecifier` returns `null` for any declared dependency before it checks the private `openclaw/*` guard. A plugin can declare `openclaw` in package.json and still import `openclaw/runtime`, defeating the boundary this PR is meant to enforce.
  - confidence: 0.97
- **[P1] Reject non-literal module specifiers before execution:** `src/plugins/import-policy.ts:273-274`
  - body: The scanner only records literal `require(...)` and `import(...)` regex matches, so computed specifiers such as `require("openclaw/" + "runtime")` are never validated before `loadPluginModule` evaluates the plugin. Either reject non-literal module loads or enforce this at runtime resolution.
  - confidence: 0.93
- **[P2] Constrain overlay forwards after resolving the target:** `src/plugins/import-policy.ts:75`
  - body: The dist-runtime overlay shortcut allows any raw specifier beginning with `../../../dist/` before resolving the final path. A specifier like `../../../dist/../../outside.js` passes this early return even though the normalized target is outside the intended dist root.
  - confidence: 0.88
- **[P3] Remove the release-owned changelog entry:** `CHANGELOG.md:55`
  - body: This normal PR edits `CHANGELOG.md`, but repository policy keeps the changelog release-owned. Move the release-note context to the PR body or squash message and leave changelog generation to the release flow.
  - confidence: 0.95
