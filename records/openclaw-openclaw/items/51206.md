## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Reject private OpenClaw imports before dependency allowlists:** `src/plugins/import-policy.ts:371-372`
  - body: This returns success for any declared dependency before checking the private OpenClaw guard. A plugin can declare `openclaw` in package.json and still import `openclaw/runtime`, defeating the boundary this PR is adding.
  - confidence: 0.95
- **[P1] Reject dynamic specifiers before plugin evaluation:** `src/plugins/import-policy.ts:273-278`
  - body: The policy only collects literal module specifiers with regexes, so `require("openclaw/" + "runtime")` or `import(getPath())` is never reported and the loader proceeds to evaluate the plugin. A security boundary cannot depend only on literal pre-scan matches.
  - confidence: 0.92
- **[P2] Decode file URLs before root-boundary checks:** `src/plugins/import-policy.ts:330-334`
  - body: Using `new URL(specifier).pathname` leaves percent-encoded path segments in the policy decision, while Node converts file URLs to decoded filesystem paths at runtime. Use `fileURLToPath()` before normalization so traversal checks inspect the same path that module loading uses.
  - confidence: 0.86
- **[P2] Resolve overlay imports before allowlisting them:** `src/plugins/import-policy.ts:389-396`
  - body: The dist-runtime exception returns success from a raw `../../../dist/` prefix before resolving the path. A specifier such as `../../../dist/../../...` can pass this branch without proving it stays inside the canonical dist root.
  - confidence: 0.88
