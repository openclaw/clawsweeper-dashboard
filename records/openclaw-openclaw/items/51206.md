## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Reject private OpenClaw imports before dependency allowlists:** `src/plugins/import-policy.ts:371-372`
  - body: The dependency allowlist returns before the private openclaw/* check, so a plugin can declare openclaw in package.json and still import openclaw/runtime. Move the private OpenClaw guard above package/self dependency acceptance and add a regression test for that package.json bypass.
  - confidence: 0.95
- **[P1] Reject dynamic module specifiers before plugin evaluation:** `src/plugins/import-policy.ts:282-284`
  - body: The policy only records literal import/require specifiers, so require("openclaw/" + "runtime") or import(getSpecifier()) is not reported before the loader evaluates the plugin. A pre-execution boundary needs to reject non-literal module loads or enforce the rule in runtime resolution.
  - confidence: 0.86
- **[P2] Decode file URLs before root-boundary checks:** `src/plugins/import-policy.ts:333-334`
  - body: Using URL.pathname for file: imports preserves percent-encoded path segments while Node's file URL conversion decodes them, so the policy and runtime can disagree about whether a path escapes the plugin root. Use fileURLToPath for all file: specifier boundary decisions.
  - confidence: 0.78
- **[P2] Resolve directory package.json entries for local imports:** `src/plugins/import-policy.ts:318-324`
  - body: Local resolution only checks direct files and index.* fallbacks, but Node also resolves require("./dir") through the directory package.json main/exports entry. This can reject valid in-package imports as policy violations even though runtime module resolution would succeed.
  - confidence: 0.82
