## Summary

Keep open: this maintainer-authored, protected PR is still a live security-boundary proposal, but the current branch is not merge-ready because the static import scanner has concrete bypasses and the branch predates current main's shared plugin module-load paths.

## What This Changes

The PR adds loader-time plugin import-policy enforcement that allows SDK seams, declared dependencies, Node builtins, and in-package imports while rejecting private OpenClaw imports, undeclared bare packages, and plugin-root escapes.

## Best Possible Solution

Rebase onto current main, enforce the boundary at a shared module-load or module-resolution layer that covers runtime and setup-runtime paths, close the bypasses, and include an explicit compatibility rollout for existing plugins.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Reject private OpenClaw imports before allowlists:** `src/plugins/import-policy.ts:371-372`
  - body: `allowedPackages.has(barePackage)` returns before the private `openclaw/*` rejection, so a plugin can declare `openclaw` and import private host paths while passing the new policy. Move the private OpenClaw rejection ahead of self/dependency allowlists and add a bypass regression test.
  - confidence: 0.96
- **[P1] Reject non-literal module specifiers:** `src/plugins/import-policy.ts:256-279`
  - body: The policy only records literal import/require strings, so computed specifiers such as concatenated `require(...)` calls are invisible to validation and still run through the module loader. A security-boundary gate needs to reject or runtime-enforce non-literal specifiers before plugin evaluation.
  - confidence: 0.9
- **[P1] Scan plugin-local dependency entrypoints:** `src/plugins/import-policy.ts:440-455`
  - body: Declared bare imports are allowed and then skipped, so a plugin can declare a local dependency whose entrypoint imports private OpenClaw internals or escapes the package root. Resolve declared dependencies from the importer and validate plugin-root dependency code before treating the graph as safe.
  - confidence: 0.89
- **[P1] Convert file URLs with fileURLToPath:** `src/plugins/import-policy.ts:399-400`
  - body: Using `new URL(specifier).pathname` leaves percent-encoding in the path used for policy decisions, while Node file URL loading decodes it. Use `fileURLToPath()` before normalization so encoded dot segments cannot disagree with runtime resolution.
  - confidence: 0.86
- **[P1] Add an upgrade-safe enforcement path:** `src/plugins/loader.ts:1150-1153`
  - body: This call makes the policy fail closed for every plugin before module load, but the PR does not include a staged rollout, compatibility mode, docs/contract update, or proof across existing bundled and representative external plugins. Existing plugins that currently load can become hard failures on upgrade.
  - confidence: 0.88
- **[P2] Resolve overlay imports before allowlisting:** `src/plugins/import-policy.ts:390-396`
  - body: The runtime-overlay exception accepts any raw `../../../dist/` prefix before resolving the rest of the path, so traversal after that prefix can escape the intended canonical dist root. Resolve and verify the target stays inside the canonical dist root instead of allowlisting by string prefix.
  - confidence: 0.82
