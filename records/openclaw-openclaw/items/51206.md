Full review comments:

- **[P1] Reject private OpenClaw imports before dependency allowlists:** `src/plugins/import-policy.ts:371-376`
  - body: The allowedPackages return runs before the private openclaw guard, so a plugin can declare openclaw and still import openclaw/runtime. Move the private-path rejection ahead of packageName/allowedPackages and add a regression test.
  - confidence: 0.96
- **[P1] Reject dynamic module specifiers before evaluation:** `src/plugins/import-policy.ts:256-279`
  - body: The scanner only collects literal import/require specifiers, but jiti still evaluates computed require/import calls. A plugin can build openclaw/runtime dynamically and bypass the policy unless non-literal specifiers are denied or resolution is enforced at runtime.
  - confidence: 0.9
- **[P2] Mirror package.json directory resolution for local imports:** `src/plugins/import-policy.ts:318-324`
  - body: Local path resolution tries files and index files but not a directory package.json main/exports entry. Valid in-package imports such as require("./dir") can be rejected even though runtime resolution succeeds.
  - confidence: 0.86
- **[P2] Convert file URLs before applying root checks:** `src/plugins/import-policy.ts:330-334`
  - body: Using URL.pathname for file: specifiers leaves percent-encoding in the path comparison, which can disagree with runtime filesystem conversion. Use fileURLToPath before normalization and boundary checks.
  - confidence: 0.84
- **[P2] Resolve overlay imports before accepting the dist shortcut:** `src/plugins/import-policy.ts:63-75`
  - body: The dist-runtime overlay shortcut allows any specifier starting with ../../../dist/ before proving the resolved path stays under the canonical dist root. Resolve and normalize the path before bypassing normal plugin-root checks.
  - confidence: 0.85
- **[P3] Remove the release-owned changelog entry:** `CHANGELOG.md:55`
