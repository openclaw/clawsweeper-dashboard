## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Reject private OpenClaw imports before package allowlists:** `src/plugins/import-policy.ts:371-375`
  - body: `allowedPackages.has(barePackage)` returns before the private `openclaw`/`openclaw/*` guard, so a plugin can declare `openclaw` in its package metadata and still import `openclaw/runtime`. Move the private OpenClaw rejection ahead of package-name and dependency allowlists, keeping the explicit `openclaw/plugin-sdk/*` allowance first, and add a regression test for that package.json bypass.
  - confidence: 0.94
- **[P1] Reject non-literal module specifiers before loading plugins:** `src/plugins/import-policy.ts:259-279`
  - body: The policy only records literal forms like `require("...")` and `import("...")`; computed calls such as `require("openclaw/" + "runtime")` collect no violation and the loader proceeds to evaluate the plugin. For a security boundary, parse and deny non-literal import/require/require.resolve forms or enforce the restriction at module resolution time.
  - confidence: 0.88
- **[P2] Convert file URLs before applying root checks:** `src/plugins/import-policy.ts:399-401`
  - body: For `file:` imports, the policy compares `new URL(specifier).pathname` rather than the filesystem path Node will use. Use `fileURLToPath()` before normalization/root checks so encoded path segments cannot make validation and runtime resolution disagree.
  - confidence: 0.83
- **[P2] Resolve overlay imports before accepting the dist shortcut:** `src/plugins/import-policy.ts:75`
  - body: `isRuntimeOverlayForwardImport` allows any specifier with the raw `../../../dist/` prefix before checking where the path resolves, so `..` segments after that prefix can expand the allowlist beyond the intended canonical dist root. Resolve and realpath the target first, then allow only paths inside the canonical dist root.
  - confidence: 0.86
- **[P3] Remove the release-owned changelog entry:** `CHANGELOG.md:55`
  - body: OpenClaw's policy says `CHANGELOG.md` is release-owned for normal PRs. Keep the release-note context in the PR body or squash message and remove this direct changelog edit from the branch.
  - confidence: 0.96
