## Summary

Keep open: this maintainer-labeled security-boundary PR is not implemented on current main, and the proposed loader-time policy still has source-proven bypasses plus an upgrade-sensitive fail-closed rollout problem.

## What This Changes

The PR adds a loader-time plugin import-policy validator, wires it into plugin loading, adds loader tests for allowed and blocked imports, and edits `CHANGELOG.md`.

## Best Possible Solution

Rebase onto current main, replace the regex pre-scan with an AST-backed or runtime-backed resolver policy that covers computed specifiers, dependency entrypoints, and normalized paths, then land only with regression tests plus an explicit plugin compatibility rollout.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Reject private OpenClaw imports before dependency allowlists:** `src/plugins/import-policy.ts:371-372`
  - body: `allowedPackages` is checked before the private `openclaw` guard, so a plugin can declare `openclaw` in its dependencies and then import `openclaw/runtime` while the validator returns `null`. Move the private OpenClaw rejection ahead of package allowlists and add a regression test for a declared `openclaw` dependency.
  - confidence: 0.96
- **[P1] Reject computed module specifiers before execution:** `src/plugins/import-policy.ts:273-274`
  - body: The validator only extracts literal `import`/`require` strings with regexes. A plugin can build `require("openclaw/" + "runtime")` or `import(getSpecifier())`, produce no violation, and then execute through the loader, defeating the boundary this PR adds.
  - confidence: 0.94
- **[P1] Validate plugin-local dependency entrypoints:** `src/plugins/import-policy.ts:455`
  - body: For bare specifiers, validation stops after the package allowlist check and never resolves or scans the dependency code. A plugin-local declared dependency can import private OpenClaw internals or escape the package root and still run when the plugin module loads.
  - confidence: 0.93
- **[P1] Constrain runtime overlay imports after path resolution:** `src/plugins/import-policy.ts:75`
  - body: The overlay exception trusts any raw specifier starting with `../../../dist/` before checking the normalized resolved path. A specifier such as `../../../dist/../../outside.js` can pass the policy even though it escapes the intended canonical dist root.
  - confidence: 0.9
- **[P1] Add an upgrade-safe enforcement path:** `src/plugins/loader.ts:1150-1153`
  - body: This new gate runs before every plugin module load and converts private/undeclared imports into load errors. Existing plugin authors get no advisory, doctor check, compatibility mode, or maintainer-approved rollout path, so valid current installations can stop loading at startup after upgrade.
  - confidence: 0.88
- **[P3] Remove the release-owned changelog edit:** `CHANGELOG.md:55`
  - body: `CHANGELOG.md` is release-owned in this repo; normal PRs should carry release-note context in the PR body or commit message instead of editing the active changelog directly.
  - confidence: 0.86
