## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Reject computed module specifiers before execution:** `src/plugins/import-policy.ts:256-279`
  - body: The scanner only records literal import/require forms, but the loader executes the module immediately after validation. A plugin can build `openclaw/runtime` or an escaping relative path at runtime and bypass the policy entirely, so the hardening needs AST coverage for non-literals or runtime resolution enforcement.
  - confidence: 0.93
- **[P1] Reject private OpenClaw imports before allowlists:** `src/plugins/import-policy.ts:371-375`
  - body: The dependency/package allowlist returns before the private `openclaw/*` rejection. Since documented plugins declare `openclaw` for SDK imports, a plugin can declare that peer/dependency and then import private host paths that this PR is meant to block.
  - confidence: 0.96
- **[P1] Validate plugin-local dependency entrypoints:** `src/plugins/import-policy.ts:440-455`
  - body: Declared bare packages are accepted and skipped instead of being resolved and scanned. A plugin-local dependency can then import private OpenClaw internals or escaping files during module execution while still satisfying the declared-dependency check.
  - confidence: 0.91
- **[P1] Add an upgrade-safe enforcement path:** `src/plugins/loader.ts:1150-1160`
  - body: This unconditional check runs before every plugin module load and turns violations into load errors. The PR body already notes existing plugins may rely on private or undeclared imports, but the patch has no warning, doctor, migration, or compatibility rollout, so existing setups can start failing at startup.
  - confidence: 0.91
- **[P2] Decode file URLs before boundary checks:** `src/plugins/import-policy.ts:330-334`
  - body: `new URL(specifier).pathname` leaves percent encoding in the string used for path checks. Use Node's file URL-to-path conversion before normalization, otherwise encoded dot segments can be checked differently from the path runtime resolution uses.
  - confidence: 0.82
- **[P2] Resolve overlay forwards before allowing them:** `src/plugins/import-policy.ts:389-396`
  - body: The dist-runtime overlay exemption allows any specifier with the raw `../../../dist/` prefix before the resolved target is checked. Resolve and verify the final path stays inside the canonical dist root so traversal after that prefix cannot bypass the plugin boundary.
  - confidence: 0.86
- **[P2] Support package.json directory imports:** `src/plugins/import-policy.ts:318-324`
  - body: Local path resolution tries direct files plus `index.*`, but Node can resolve `require("./dir")` through that directory's package.json main/exports. Valid in-package imports using that standard pattern would now fail policy before runtime gets a chance to resolve them.
  - confidence: 0.82
- **[P3] Remove the release-owned changelog entry:** `CHANGELOG.md:55`
  - body: CHANGELOG.md is release-owned in this repository. Keep the release-note context in the PR body or squash message and let release generation add the changelog entry.
  - confidence: 0.88
