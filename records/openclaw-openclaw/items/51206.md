## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Reject private imports before dependency allowlists:** `src/plugins/import-policy.ts:371-372`
  - body: `resolveBarePackageName("openclaw/runtime")` returns `openclaw`, so a plugin that declares `openclaw` in package.json returns at the dependency allowlist before the private-path guard. That lets the exact private import this policy is meant to block pass validation.
  - confidence: 0.94
- **[P1] Deny computed module specifiers:** `src/plugins/import-policy.ts:269-274`
  - body: The policy only records regex-matched literal specifiers; `require("openclaw/" + "runtime")` or `import(name)` is not collected, and the loader then evaluates the module normally. Either reject non-literal module loads or enforce the boundary during runtime resolution.
  - confidence: 0.88
- **[P1] Decode file URLs before boundary checks:** `src/plugins/import-policy.ts:330-334`
  - body: `file:` imports are converted with `new URL(...).pathname`, but Node resolves file URLs through decoded filesystem paths. Percent-encoded dot segments can therefore be judged differently by the policy than by runtime resolution; use `fileURLToPath()` before normalization and boundary checks.
  - confidence: 0.82
- **[P1] Reject traversal inside bare specifiers:** `src/plugins/import-policy.ts:355-359`
  - body: The allowlist accepts any specifier starting with `openclaw/plugin-sdk/` before checking for `.` or `..` path segments. Requests such as `openclaw/plugin-sdk/../../runtime` can preserve the allowed prefix in policy while resolving outside the intended SDK subpath.
  - confidence: 0.82
- **[P1] Validate overlay imports after resolving target:** `src/plugins/import-policy.ts:389-396`
  - body: `isRuntimeOverlayForwardImport()` accepts any raw specifier starting with `../../../dist/`, and `classifyLocalSpecifierViolation()` returns `null` before computing the resolved target or checking `canonicalDistRoot`. A later `..` segment can escape the intended dist root while still passing policy.
  - confidence: 0.84
- **[P2] Support package.json directory entries for local imports:** `src/plugins/import-policy.ts:318-324`
  - body: Node resolves `require("./dir")` through `./dir/package.json` `main`/`exports` before index files, but this resolver only tries direct files and index fallbacks. Valid in-package helper directories can be rejected even though runtime module resolution would load them.
  - confidence: 0.86
