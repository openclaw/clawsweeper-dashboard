## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Add an upgrade-safe enforcement path:** `src/plugins/loader.ts:1150-1153`
  - body: This new check runs unconditionally before every plugin module load and turns policy mismatches into load errors. Because the PR itself says migration may be needed, existing plugins can stop loading on upgrade unless the branch adds a maintainer-approved rollout path, doctor/advisory mode, or upgrade proof for affected plugin patterns.
  - confidence: 0.91
- **[P1] Reject private OpenClaw imports before allowlists:** `src/plugins/import-policy.ts:371-372`
  - body: `allowedPackages.has(barePackage)` returns before the private `openclaw/*` guard, so a plugin that declares `openclaw` can still import `openclaw/runtime` and pass validation. Move the private OpenClaw rejection ahead of package allowlists and add a regression fixture.
  - confidence: 0.96
- **[P1] Reject computed module specifiers before execution:** `src/plugins/import-policy.ts:256-280`
  - body: The policy only records literal `import`/`require` strings, so computed calls like `require("openclaw/" + "runtime")` are invisible to the scanner and still execute when the loader evaluates the module. A security-boundary policy needs to reject non-literal specifiers or enforce at runtime resolution.
  - confidence: 0.9
- **[P1] Scan plugin-local dependency entrypoints:** `src/plugins/import-policy.ts:440-455`
  - body: Declared bare packages are allowed and then skipped, but plugin-local dependencies can be part of the plugin package and can import forbidden OpenClaw internals or escape their package after validation. Resolve and traverse local dependency entrypoints, or enforce the same boundary during module resolution.
  - confidence: 0.88
- **[P1] Constrain overlay imports after resolving targets:** `src/plugins/import-policy.ts:75`
  - body: The runtime overlay exception accepts any specifier starting with `../../../dist/` and skips the later inside-root checks. A path like `../../../dist/../../...` can pass the raw prefix test, so the exception needs to resolve and verify the target stays inside the canonical dist root.
  - confidence: 0.9
- **[P1] Convert file URLs with fileURLToPath:** `src/plugins/import-policy.ts:330-334`
  - body: The policy uses `new URL(specifier).pathname` for filesystem decisions, but Node file URL loading decodes percent-encoded path segments. Convert with `fileURLToPath()` before normalization so encoded traversal cannot disagree with the loader's boundary check.
  - confidence: 0.87
