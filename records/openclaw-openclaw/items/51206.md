## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Reject private OpenClaw imports before package allowlists:** `src/plugins/import-policy.ts:371-372`
  - body: `validateBareSpecifier` returns `null` for allowed package names before checking private `openclaw/*` paths. Because `openclaw/runtime` resolves to the bare package `openclaw`, a plugin can declare `openclaw` and bypass the boundary; move the private-path rejection ahead of package allowlists and add coverage for that case.
  - confidence: 0.98
- **[P1] Reject non-literal module specifiers before evaluation:** `src/plugins/import-policy.ts:256-280`
  - body: The policy only collects literal regex matches, so computed calls like `require("openclaw/" + "runtime")` or `import(getSpecifier())` are not reported before the loader evaluates the plugin. A security boundary needs to fail closed for non-literal specifiers or enforce the rule in the runtime resolver.
  - confidence: 0.95
- **[P2] Convert file URLs before applying root checks:** `src/plugins/import-policy.ts:329-334`
  - body: For `file:` imports the code uses `new URL(specifier).pathname`, which can leave percent-encoded path segments that do not match Node's filesystem path conversion. Use `fileURLToPath()` before normalization and root checks so policy decisions match the path the runtime loads.
  - confidence: 0.86
- **[P2] Resolve overlay imports before accepting the dist shortcut:** `src/plugins/import-policy.ts:389-396`
  - body: The runtime-overlay exception returns `null` from a raw `../../../dist/` prefix before normalizing the rest of the specifier. Resolve and realpath-check the target against the canonical dist root before allowing this path, otherwise traversal segments can skip the normal boundary checks.
  - confidence: 0.9
- **[P2] Support valid package-local directory entry imports:** `src/plugins/import-policy.ts:318-324`
  - body: Local resolution only checks direct files and `index.*`, but plugins can validly import a local directory whose `package.json` defines `main` or `exports`. As written those imports fail policy even though runtime resolution can load them inside the plugin root, creating upgrade false positives for normal package-local code.
  - confidence: 0.84
- **[P3] Remove the release-owned changelog hunk:** `CHANGELOG.md:55`
  - body: This repo treats `CHANGELOG.md` as release-owned for normal PRs. Keep the release-note context in the PR body or squash message and drop this direct changelog edit before merge.
  - confidence: 0.93
