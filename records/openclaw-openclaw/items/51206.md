- **[P1] Reject private OpenClaw imports before allowlists:** `src/plugins/import-policy.ts:371-372`
  - body: Because `allowedPackages.has(barePackage)` returns before the private `openclaw/*` guard, a plugin can declare `openclaw` and still import `openclaw/runtime`, defeating the boundary this PR is meant to enforce.
  - confidence: 0.95
- **[P1] Reject computed specifiers or enforce at runtime:** `src/plugins/import-policy.ts:273-275`
  - body: The scanner only collects string-literal `require`/`import` forms, so `require("openclaw/" + "runtime")` or another computed specifier can pass validation and then execute through the normal module loader.
  - confidence: 0.88
- **[P2] Resolve overlay paths before accepting dist imports:** `src/plugins/import-policy.ts:75`
  - body: This raw `../../../dist/` prefix check returns true before resolving the rest of the path, so an overlay import with later `..` segments can skip the root checks instead of being constrained to the canonical dist root.
  - confidence: 0.82
- **[P3] Remove the release-owned changelog entry:** `CHANGELOG.md:55`
  - body: Normal PRs should not edit `CHANGELOG.md`; release-note context belongs in the PR body or squash message so release generation remains the single owner of changelog entries.
