## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Reject private OpenClaw imports before dependency allowlists:** `src/plugins/import-policy.ts:371-372`
  - body: `allowedPackages.has(barePackage)` returns `null` before the private `openclaw` guard runs. A plugin can declare `openclaw` in its dependencies and still import `openclaw/runtime`, defeating the boundary this PR is trying to enforce.
  - confidence: 0.98
- **[P1] Reject computed specifiers before plugin execution:** `src/plugins/import-policy.ts:259-279`
  - body: The policy only finds literal specifiers with regexes, so `require("openclaw/" + "runtime")` or `import(getSpecifier())` is not recorded and the plugin still executes. Boundary enforcement should reject dynamic module specifiers or enforce at runtime.
  - confidence: 0.94
- **[P1] Scan plugin-local dependency entrypoints:** `src/plugins/import-policy.ts:439-455`
  - body: After an allowed bare specifier is accepted, validation immediately continues without resolving or scanning the dependency under the plugin root. A declared plugin-local package can import private OpenClaw internals while the entry file passes policy validation.
  - confidence: 0.95
- **[P1] Resolve overlay imports before allowing them:** `src/plugins/import-policy.ts:75`
  - body: The runtime overlay exception allows any specifier with the raw `../../../dist/` prefix before resolving and checking the final canonical path. A suffix with traversal segments can expand the allowlist beyond the intended dist root.
  - confidence: 0.9
- **[P1] Add an upgrade-safe enforcement path:** `src/plugins/loader.ts:1150-1160`
  - body: This new loader gate turns import-policy violations into pre-execution plugin load errors for every plugin, but the PR body says existing plugins may rely on private imports or undeclared dependencies and provides no compatibility mode, migration, or maintainer-approved rollout.
  - confidence: 0.88
- **[P3] Remove the release-owned changelog edit:** `CHANGELOG.md:52`
  - body: Normal PRs should carry release-note context in the PR body or commit message; `CHANGELOG.md` is release-owned in this repository. This line should be removed from the branch before it lands.
  - confidence: 0.9
