## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Reject private OpenClaw imports before allowlists:** `src/plugins/import-policy.ts:371-372`
  - body: `validateBareSpecifier` allows the plugin package name and declared dependencies before it blocks `openclaw/*`. A plugin can declare `openclaw` and still import `openclaw/runtime`, defeating the private-import boundary this PR adds.
  - confidence: 0.97
- **[P1] Reject computed specifiers before plugin execution:** `src/plugins/import-policy.ts:269-278`
  - body: The scanner only records string-literal import and require calls. Code such as `require("openclaw/" + "runtime")` is invisible to the policy and still runs when the loader evaluates the plugin module.
  - confidence: 0.94
- **[P1] Scan plugin-local dependency entrypoints:** `src/plugins/import-policy.ts:440-455`
  - body: After a declared bare package passes the allowlist check, validation immediately continues without resolving or scanning that dependency. A plugin-local dependency can import private internals or escape the root while the top-level plugin passes.
  - confidence: 0.92
- **[P1] Constrain overlay imports after resolution:** `src/plugins/import-policy.ts:389-396`
  - body: The runtime overlay exception returns success for any raw specifier starting with `../../../dist/` before resolving the suffix. Traversal after that prefix bypasses the later root and canonical-dist checks.
  - confidence: 0.9
- **[P1] Reject traversal in bare specifier subpaths:** `src/plugins/import-policy.ts:355-359`
  - body: The `openclaw/plugin-sdk/` prefix allowlist accepts raw subpaths before normalizing or rejecting `.` and `..` segments. A traversal subpath can escape the intended SDK prefix while still passing this check.
  - confidence: 0.86
- **[P2] Use fileURLToPath for file URL checks:** `src/plugins/import-policy.ts:399-400`
  - body: The policy uses `new URL(specifier).pathname` for filesystem decisions, which does not match Node's file URL path conversion for encoded segments. Use `fileURLToPath()` before containment checks so policy and runtime resolution agree.
  - confidence: 0.82
- **[P1] Add an upgrade-safe enforcement path:** `src/plugins/loader.ts:1154-1160`
  - body: This turns policy violations into load errors for every plugin with no advisory, doctor check, compatibility mode, or documented rollout. Existing installed plugins using private imports or undeclared dependency reach-through can stop loading after upgrade.
  - confidence: 0.88
- **[P2] Support package.json directory resolution:** `src/plugins/import-policy.ts:318-324`
  - body: Local path resolution tries direct files and `index.*` fallbacks, but Node also resolves `require("./dir")` through `dir/package.json` main or exports. The new fail-closed policy can reject valid in-package imports that runtime resolution would load.
  - confidence: 0.82
