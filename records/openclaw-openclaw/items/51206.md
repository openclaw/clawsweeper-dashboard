Full review comments:

- **[P1] Reject private OpenClaw imports before allowlists:** `src/plugins/import-policy.ts:371-372`
  - body: `allowedPackages.has(barePackage)` returns before the private `openclaw/*` guard, so a plugin can declare `openclaw` in its manifest dependencies and still import `openclaw/runtime`, defeating the stated boundary.
  - confidence: 0.94
- **[P1] Reject computed module specifiers before execution:** `src/plugins/import-policy.ts:273-274`
  - body: The policy only records regex-matched string-literal imports and requires. A plugin can build `openclaw/runtime` or `../outside.cjs` dynamically and the loader will still execute it, so this is not a reliable security boundary.
  - confidence: 0.91
- **[P1] Scan plugin-local dependency entrypoints:** `src/plugins/import-policy.ts:455`
  - body: Declared bare dependencies are allowed and then skipped without resolving their entrypoints. A plugin-local dependency under `node_modules` can import forbidden OpenClaw internals before the policy ever inspects that file.
  - confidence: 0.88
- **[P1] Constrain overlay imports after resolving targets:** `src/plugins/import-policy.ts:389-396`
  - body: The runtime overlay special-case returns success from a raw `../../../dist/` string prefix before proving the resolved path stays inside the canonical dist root, so traversal after that prefix can bypass the boundary.
  - confidence: 0.86
- **[P1] Convert file URLs with fileURLToPath:** `src/plugins/import-policy.ts:330-334`
  - body: Using `new URL(specifier).pathname` preserves percent encoding, while Node decodes file URLs for module loading. That mismatch can make the policy check a different path than runtime resolution.
  - confidence: 0.84
- **[P1] Retarget enforcement to current plugin load paths:** `src/plugins/loader.ts:1150-1153`
  - body: The new gate is inserted before the old direct `getJiti` call, but current main evaluates plugins through shared `loadPluginModule` paths for full runtime, setup-runtime, and CLI metadata. The branch needs rebase and coverage for the current loader shape before it can enforce the intended invariant.
  - confidence: 0.86
- **[P1] Add an upgrade-safe enforcement path:** `src/plugins/loader.ts:1154-1160`
  - body: This changes previously allowed plugin imports into hard load errors without a default-compatible rollout, diagnostics-only phase, or proof across bundled and external plugins. Existing plugin setups can fail immediately on upgrade.
  - confidence: 0.82
- **[P3] Remove the normal PR changelog edit:** `CHANGELOG.md:55`
  - body: OpenClaw release generation owns `CHANGELOG.md`; normal feature and fix PRs should keep release-note context in the PR body or squash message instead of editing the release changelog directly.
  - confidence: 0.93

## Security Review

Status: needs_attention

Summary: This security-hardening PR is not ready as a security boundary because static scanning and coverage gaps allow bypasses while still making plugins fail closed.

Concerns:

- **[high] Private import guard can be bypassed by declaring openclaw:** `src/plugins/import-policy.ts:371`
  - body: The allowlist check runs before the private `openclaw/*` rejection, so declaring `openclaw` as a dependency can allow private host imports.
  - confidence: 0.94
- **[high] Computed specifiers evade the static scan:** `src/plugins/import-policy.ts:273`
  - body: Regex collection only sees literal specifiers, leaving dynamic `require` or `import()` values to execute outside the proposed policy.
  - confidence: 0.91
- **[high] Declared dependencies are not traversed:** `src/plugins/import-policy.ts:455`
  - body: Plugin-local dependency entrypoints can contain forbidden imports because allowed bare packages are not resolved and scanned before module execution.
  - confidence: 0.88
- **[medium] File URL decoding differs from runtime resolution:** `src/plugins/import-policy.ts:330`
  - body: Using URL pathname instead of `fileURLToPath` can make percent-encoded traversal segments pass policy checks that Node decodes during loading.
  - confidence: 0.84
- **[medium] Runtime overlay prefix skips resolved containment:** `src/plugins/import-policy.ts:395`
  - body: The `../../../dist/` overlay allowlist returns before resolved-path containment, allowing traversal-shaped specifiers to escape the intended dist root.
  - confidence: 0.86
