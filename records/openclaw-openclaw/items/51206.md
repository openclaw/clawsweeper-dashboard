Full review comments:

- **[P1] Reject private OpenClaw imports before allowlists:** `src/plugins/import-policy.ts:371-375`
  - body: `allowedPackages` is checked before the private `openclaw/*` rejection, so a plugin can declare `openclaw` in its package dependencies and then import `openclaw/runtime` without a policy violation. Move the private OpenClaw guard before package/self allowlists and add a regression test for a declared `openclaw` dependency.
  - confidence: 0.96
- **[P1] Reject computed module specifiers before execution:** `src/plugins/import-policy.ts:256-279`
  - body: The policy only collects literal specifiers matched by these regexes, so `require("openclaw/" + "runtime")`, variable-based `require()`, or computed `import()` calls pass validation and then execute in the plugin loader. A boundary check needs to reject dynamic specifiers or enforce resolution at runtime.
  - confidence: 0.9
- **[P1] Validate plugin-local dependency entrypoints:** `src/plugins/import-policy.ts:440-455`
  - body: For bare imports, the validator only checks that the package name is declared and then skips traversal, so a plugin-local `node_modules` dependency can import private OpenClaw internals or escape paths before Jiti executes it. Resolve declared dependencies from the importer and validate plugin-local dependency code, or enforce the same deny rules at runtime.
  - confidence: 0.86
- **[P1] Add an upgrade-safe enforcement path:** `src/plugins/loader.ts:1150-1160`
  - body: This new check runs for every plugin load and turns policy violations into load errors, but current docs preserve external plugin compatibility and the PR body only says migration is maybe needed. Before fail-closing existing plugins, add a maintainer-approved warning/doctor/migration/compatibility rollout or narrow the policy to an explicitly strict mode.
  - confidence: 0.88
- **[P2] Support package.json directory imports:** `src/plugins/import-policy.ts:318-324`
  - body: Local resolution tries direct files and `index.*` only, but Node also resolves directory imports through `package.json` `main` or `exports`. Valid in-package imports such as `require("./dir")` can be rejected even though runtime module resolution would succeed.
  - confidence: 0.8
- **[P3] Remove the release-owned changelog edit:** `CHANGELOG.md:55`
