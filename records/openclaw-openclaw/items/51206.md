## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Reject private OpenClaw imports before dependency allowlists:** `src/plugins/import-policy.ts:371-372`
  - body: `openclaw/runtime` resolves to the bare package name `openclaw`, so a plugin can declare `openclaw` in `dependencies` and return from the allowlist check before the private `openclaw/*` guard runs. Move the private-path rejection ahead of package allowlists and add a regression test for a declared `openclaw` dependency.
  - confidence: 0.98
- **[P1] Reject computed module specifiers before evaluation:** `src/plugins/import-policy.ts:256-280`
  - body: The policy only scans literal regex matches, so `require("openclaw/" + "runtime")` or `import(getSpecifier())` is not reported before `loadPluginModule` evaluates the plugin. For a security boundary, non-literal module specifiers need to fail closed or be enforced by the runtime resolver.
  - confidence: 0.95
- **[P2] Resolve runtime overlay imports before allowlisting:** `src/plugins/import-policy.ts:389-396`
  - body: The overlay exception returns `null` from a raw `../../../dist/` prefix before normalizing the rest of the specifier. A path like `../../../dist/../../...` can skip the normal plugin-root/canonical-dist checks, so resolve and realpath-check the target before allowing this path.
  - confidence: 0.9
- **[P3] Remove the release-owned changelog edit:** `CHANGELOG.md:55`
  - body: `CHANGELOG.md` is release-owned in this repo; normal PRs should keep release-note context in the PR body or squash message. Please drop this hunk before merge.
  - confidence: 0.93
