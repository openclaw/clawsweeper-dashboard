Full review comments:

- **[P1] Reject computed module specifiers before execution:** `src/plugins/import-policy.ts:256-279`
  - body: The scanner only records literal regex matches, but the loader executes the plugin immediately afterward. A plugin can build `openclaw/runtime` or an escaping path dynamically and bypass the policy, so non-literal `require`, `require.resolve`, and `import()` calls need to be rejected or enforced at module resolution time.
  - confidence: 0.93
- **[P1] Reject private OpenClaw imports before allowlists:** `src/plugins/import-policy.ts:371-372`
  - body: `allowedPackages.has(barePackage)` returns before the private `openclaw/*` block. A plugin that declares `openclaw` as a dependency can import `openclaw/runtime` and pass validation, defeating the main boundary this PR adds.
  - confidence: 0.96
- **[P1] Validate plugin-local dependency entrypoints:** `src/plugins/import-policy.ts:439-455`
  - body: Allowed bare specifiers are not resolved or traversed, so a declared dependency under the plugin root can import private OpenClaw internals or escaping paths and still be executed by the loader. Resolve plugin-local dependencies and scan or enforce their module graph before execution.
  - confidence: 0.9
- **[P2] Resolve overlay forwards before allowing them:** `src/plugins/import-policy.ts:389-396`
  - body: The runtime-overlay exception returns success based only on a raw `../../../dist/` prefix. Specifiers with traversal after that prefix can skip the later root checks, so the resolved real path must be proven inside the intended canonical dist root before allowing it.
  - confidence: 0.85
- **[P1] Add an upgrade-safe enforcement path:** `src/plugins/loader.ts:1154-1160`
  - body: This turns import-policy violations into unconditional plugin load errors for every affected plugin. Because existing external plugins are the compatibility baseline and the PR body acknowledges migration may be needed, this needs an approved warning/doctor/opt-in rollout or explicit fail-closed decision with real-plugin proof before merge.
  - confidence: 0.88
- **[P3] Remove the release-owned changelog entry:** `CHANGELOG.md:55`
  - body: Normal PRs should not edit `CHANGELOG.md`; release generation owns this file. Move the release-note context into the PR body or squash message instead.
