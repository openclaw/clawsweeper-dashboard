## Summary

Keep open: this PR targets a real v2026.6.9 memory migration regression, but the proposed fix puts legacy sidecar import into steady-state runtime schema initialization, which can rehydrate stale memory state from an old sidecar whenever the canonical per-agent DB is empty.

## What This Changes

The PR imports legacy memory sidecar SQLite rows into an empty per-agent memory index during schema initialization and adds a regression test.

## Best Possible Solution

Handle sidecar import as a one-shot doctor/update migration, or explicitly approve a bounded startup migration with skip markers and tests for intentionally empty canonical databases.

## Solution Assessment

No. The row copy targets the right data, but wiring legacy sidecar import into steady-state runtime startup is not the best migration owner without explicit maintainer approval and one-shot safeguards.
