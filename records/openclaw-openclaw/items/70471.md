## Summary

Keep open. Current main still lacks the Feishu comment-delivery hardening this PR targets, and the related #66427 is closed unmerged, so this remains a useful implementation PR rather than a cleanup close. ## What This Changes

The PR updates Feishu Drive comment handling to gate notices by recipient, suppress duplicate final replies after visible tool delivery, escape angle brackets in comment writes, scope typing reactions by account, refresh bot identity on startup/reload, and add targeted regression tests.

## Best Possible Solution

Land this PR or an equivalent Feishu-plugin patch after maintainer review, current-main rebase, targeted Feishu regression validation, and changelog credit. The fix should remain in the Feishu plugin because the affected behavior is Feishu-owned comment routing, tool delivery, typing reaction, and bot identity handling.
