## Summary

Keep open. Current main still has the heartbeat transcript marker and Telegram's delivered-text visibility helper still treats every non-final block as visible without checking for that marker; open PR #75167 explicitly closes this issue but remains open and unmerged in the provided GitHub context, so the issue should stay open until that PR lands or is replaced.

## What This Changes

Request to keep the internal heartbeat poll prompt out of Telegram DM-visible messages while continuing to suppress OK-only heartbeat acknowledgments by default.

## Best Possible Solution

Review and either land or replace #75167 with a narrow fix that prevents `HEARTBEAT_TRANSCRIPT_PROMPT` from reaching channel-visible delivery for Telegram and any other affected chat surfaces, while preserving meaningful heartbeat alert delivery and configured `showOk` behavior with regression coverage.

## Reproduction Assessment

Yes. The issue provides a concrete Telegram DM heartbeat configuration, and current main still has the source path that creates the marker and a Telegram visibility helper that does not inspect or drop it.

## Solution Assessment

Unclear until #75167 is reviewed. A shared outbound guard or well-covered channel helper that drops only the internal heartbeat marker before visible delivery appears to be the narrow maintainable direction.
