## Summary

Keep open. Current main still routes the heartbeat transcript marker through the heartbeat reply path, and Telegram's visibility hook still treats non-final delivered text as visible without filtering the marker. The GitHub context also shows open unmerged PR #75167 explicitly closes this issue, so the issue should remain open until that PR lands or is replaced.

## What This Changes

The issue requests that Telegram DM delivery suppress the internal `[OpenClaw heartbeat poll]` prompt while continuing to suppress OK-only heartbeat acknowledgments by default.

## Best Possible Solution

Review and either land or replace PR #75167 with a narrow fix that prevents the exact heartbeat transcript marker from reaching channel-visible delivery, with Telegram regression coverage and no change to meaningful heartbeat alert delivery or configured `showOk` behavior.

## Reproduction Assessment

Yes. The issue provides concrete Telegram DM heartbeat steps and config, and current main still has the marker-producing heartbeat path plus a Telegram visible-text helper that does not inspect or suppress the marker.

## Solution Assessment

Unclear until PR #75167 is reviewed. A narrow guard that drops only the internal heartbeat marker before channel-visible delivery, backed by Telegram regression coverage, appears to be the maintainable direction.
