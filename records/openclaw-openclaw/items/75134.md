## Summary

Keep open. Current main still defines and routes the heartbeat transcript marker, Telegram's visibility hook still ignores the delivered text, and linked PR #75167 is open and unmerged as the implementation candidate.

## What This Changes

Telegram heartbeat poll prompts are reported as visible DM messages and should be hidden from user-facing channel output while preserving real heartbeat responses.

## Best Possible Solution

Review and land or replace #75167 with a narrow suppression of only `HEARTBEAT_TRANSCRIPT_PROMPT` before external channel delivery, while preserving real heartbeat alerts and transcript auditability.

## Reproduction Assessment

Yes. The issue gives concrete Telegram DM heartbeat steps and config, and current main still has the marker plus a Telegram visibility hook that cannot suppress that exact text.

## Solution Assessment

Yes on direction, unclear on the exact implementation until #75167 is reviewed. The maintainable boundary is a focused external-channel suppression for the heartbeat transcript prompt, not a broader suppression of heartbeat alert output.
