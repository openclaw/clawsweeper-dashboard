## Summary

Keep open. Current main still has the internal heartbeat transcript marker wired into heartbeat runs, and Telegram's visible-text helper does not filter that marker before channel-visible delivery. Open PR #75167 remains unmerged and explicitly targets this issue, so the maintainer path is to review, land, or replace that PR rather than close the issue now.

## What This Changes

Suppress the internal heartbeat transcript marker from appearing as visible Telegram DM text while preserving default suppression of OK-only heartbeat acknowledgments.

## Best Possible Solution

Keep this issue open until #75167 or a narrow replacement lands. The end state should drop only `HEARTBEAT_TRANSCRIPT_PROMPT` from channel-visible delivery, add Telegram DM regression coverage, and preserve meaningful heartbeat alert delivery plus configured `showOk` behavior.

## Reproduction Assessment

Yes for the underlying Telegram-routed leak path: current main still puts the marker into the heartbeat transcript path, and Telegram does not inspect the delivered text before marking non-final blocks visible. The exact report detail of no explicit heartbeat target has a current-main caveat because `target` defaults to `none`, so a focused repro should use `target: "last"` or explicit Telegram delivery unless version-specific default behavior is confirmed.

## Solution Assessment

Unclear whether the open PR is the best patch until its diff is reviewed. The maintainable direction is a narrow visibility guard that drops only the internal heartbeat transcript marker before external channel delivery, preferably with coverage proving real alerts still send.
