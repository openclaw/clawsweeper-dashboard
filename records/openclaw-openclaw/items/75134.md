## Summary

Current main no longer has the reported no-target Telegram heartbeat leak: heartbeat delivery defaults to `none`, no-target runs return before outbound sends, and heartbeat prompt artifacts are classified as internal transcript artifacts; the safer default first shipped in `v2026.2.24`.

## What This Changes

The issue reports that Telegram DM users can see the internal `[OpenClaw heartbeat poll]` prompt and asks for heartbeat prompts to stay hidden on external channel surfaces.

## Best Possible Solution

Keep the shipped safer heartbeat default and central heartbeat artifact filtering; any new report should include a current-version config that explicitly enables heartbeat delivery or shows a remaining channel-specific path.

## Reproduction Assessment

No. Against current main, an unset heartbeat target resolves to `target-none` and the runner returns before outbound channel delivery, so the reported default-config Telegram DM path is not present in source; I did not run a live Telegram heartbeat cycle.

## Solution Assessment

Yes. The maintainable solution is the shipped centralized default of no external heartbeat delivery unless configured, plus central heartbeat artifact filtering, rather than a per-channel literal string filter.
