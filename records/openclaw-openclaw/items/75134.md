## Summary

Keep open. Current main still has a marker-producing heartbeat transcript path and Telegram's visible-delivery helper does not inspect or suppress the marker text; #75167 is an open, unmerged PR that explicitly targets this issue, so repository policy keeps this issue open until that PR lands or is replaced.

## What This Changes

Ensure Telegram DM conversations never show the internal `[OpenClaw heartbeat poll]` prompt and continue suppressing default `HEARTBEAT_OK` acknowledgments.

## Best Possible Solution

Land or replace a narrow fix that drops only the internal heartbeat transcript marker before channel-visible delivery, with Telegram DM regression coverage and without suppressing real heartbeat alerts or opt-in OK acknowledgments.

## Reproduction Assessment

Unclear for the exact posted config because current main defaults an unset heartbeat target to `none`. The underlying leak path is source-reproducible once a heartbeat resolves to Telegram delivery: the marker is produced and no Telegram/outbound guard currently drops it.

## Solution Assessment

Unclear until #75167 is reviewed. A shared outbound guard or a channel helper that suppresses only the exact internal marker looks like the narrow maintainable direction; a broad text filter would be riskier.
