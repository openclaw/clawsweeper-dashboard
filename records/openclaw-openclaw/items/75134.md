## Summary

Keep open. Current main still defines the internal heartbeat transcript marker, uses it for heartbeat reply transcript bodies, and Telegram's delivered-text visibility helper ignores the text before treating non-final deliveries as visible. The linked PR #75167 explicitly closes this issue but remains open and unmerged in the provided GitHub context, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Suppress the internal heartbeat poll transcript marker from Telegram DM delivery while keeping HEARTBEAT_OK acknowledgments suppressed by default.

## Best Possible Solution

Land or replace PR #75167 with a narrow suppression of only HEARTBEAT_TRANSCRIPT_PROMPT before it can become user-visible channel output, while preserving real heartbeat alerts, configured showOk behavior, and the underlying audit transcript.

## Reproduction Assessment

Yes. The issue gives concrete Telegram DM heartbeat steps and config, and current-main source inspection shows the marker-producing heartbeat path plus a Telegram helper that cannot filter the marker because it ignores the delivered text.

## Solution Assessment

Unclear until PR #75167 is reviewed. The safest direction is a small shared outbound or channel-helper guard for the exact internal marker, not a broader change to heartbeat prompt generation or heartbeat visibility semantics.
