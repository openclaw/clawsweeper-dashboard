## Summary

Keep open. Current main still defines and uses the internal heartbeat transcript marker, and Telegram's visible-delivery helper still ignores the delivered text before treating non-final blocks as visible. The linked PR #75167 explicitly closes this issue, but it remains open and unmerged, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Suppress the internal `[OpenClaw heartbeat poll]` prompt from Telegram DM-visible delivery while preserving normal heartbeat alerts and default `HEARTBEAT_OK` suppression.

## Best Possible Solution

Review and land or replace #75167 with a narrow marker-specific suppression before user-visible channel delivery, backed by Telegram regression coverage for the streamed/block path and without changing meaningful heartbeat alert delivery or configured `showOk` behavior.

## Reproduction Assessment

Yes. The issue gives concrete Telegram DM heartbeat steps and config, and current-main source inspection shows the marker is still produced as the heartbeat transcript prompt while Telegram does not filter that text in its visibility hook.

## Solution Assessment

Unclear until #75167 is reviewed. The safest implementation direction is a focused guard that drops only `HEARTBEAT_TRANSCRIPT_PROMPT` from user-visible channel delivery, with targeted Telegram coverage and no broad heartbeat behavior rewrite.
