## Summary

Keep open. Current main still defines and routes the heartbeat transcript marker through the heartbeat reply path, and Telegram's visible-delivery helper still ignores the delivered text instead of filtering the marker. The provided GitHub context also shows open PR #75167 explicitly closes this issue but is not merged, so the issue should remain open until that PR lands or is replaced.

## What This Changes

The issue requests that Telegram DM delivery hide the internal heartbeat poll prompt while continuing to suppress OK-only heartbeat acknowledgments by default.

## Best Possible Solution

Review and either land or replace the linked PR with a narrow fix that prevents the exact heartbeat transcript marker from reaching channel-visible delivery, with Telegram regression coverage and no change to meaningful heartbeat alert delivery or configured `showOk` behavior.

## Reproduction Assessment

Yes. The issue provides concrete Telegram DM heartbeat steps and config, and current main still has the marker-producing heartbeat path plus a Telegram visible-text helper that does not inspect or suppress the marker.

## Solution Assessment

Unclear until the linked PR is reviewed. A shared outbound guard, or a well-covered channel helper that drops only the internal heartbeat marker before visible delivery, appears to be the narrow maintainable direction.
