## Summary

Keep open. Current main still leaves the heartbeat transcript marker on the delivery path and Telegram's visibility override ignores the delivered text, while open PR #75167 is already the linked implementation candidate and has not merged in the provided context.

## What This Changes

Hide the internal `[OpenClaw heartbeat poll]` prompt from Telegram DM chat while continuing to suppress `HEARTBEAT_OK` acknowledgments by default.

## Best Possible Solution

Land or replace #75167 with a narrow suppression of the internal heartbeat prompt before channel-visible delivery, while preserving real heartbeat alerts and the audit transcript.

## Reproduction Assessment

Yes. The issue gives concrete Telegram DM heartbeat steps and config, and current-main source inspection shows the marker-producing heartbeat path plus a Telegram visibility helper that cannot filter the marker because it ignores the text.

## Solution Assessment

Yes. Keeping the issue open is the right path until #75167 or an equivalent replacement lands; the fix should be marker-specific channel delivery suppression, not a broad change to heartbeat transcript semantics.
