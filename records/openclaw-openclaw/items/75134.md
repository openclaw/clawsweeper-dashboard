## Summary

Keep open. Current main still defines and uses the `[OpenClaw heartbeat poll]` transcript marker, and Telegram's visibility helper ignores the outgoing text before treating non-final deliveries as visible. The linked PR #75167 explicitly closes this issue but remains open and unmerged in the provided context, so this should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that Telegram DM heartbeat poll prompts remain internal, with only real heartbeat responses delivered and default `HEARTBEAT_OK` acknowledgments suppressed.

## Best Possible Solution

Land or replace PR #75167 with a narrow exact-marker suppression before external channel delivery, preserving real heartbeat alerts, configured `showOk` behavior, and the audit transcript.

## Reproduction Assessment

Yes. The issue gives concrete Telegram DM heartbeat steps and config, and current-main source inspection shows the marker-producing heartbeat path plus a Telegram visibility helper that cannot filter the marker because it ignores the delivered text.

## Solution Assessment

Unclear until PR #75167 is reviewed or repaired. The maintainable direction is to suppress only the exact internal heartbeat transcript marker at a shared outbound or channel-helper boundary, not to change heartbeat scheduling or transcript audit semantics.
