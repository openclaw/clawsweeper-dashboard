## Summary

Keep open. Current main still defines and uses the internal heartbeat transcript marker, while Telegram's delivered-text visibility hook ignores the text and treats non-final delivered blocks as visible; the linked fix PR is still open and unmerged, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Suppress the internal heartbeat poll marker from Telegram DM delivery so users never see `[OpenClaw heartbeat poll]`, while keeping default `HEARTBEAT_OK` acknowledgment suppression intact.

## Best Possible Solution

Land or replace #75167 with a narrow, tested suppression of only the internal heartbeat transcript marker before it reaches user-visible channel delivery, preserving meaningful heartbeat alerts and configured `showOk` behavior.

## Reproduction Assessment

Yes. The issue provides concrete Telegram DM heartbeat configuration and steps, and current-main source inspection shows the marker-producing heartbeat path plus a Telegram visibility helper that does not filter that marker.

## Solution Assessment

Unclear until #75167 is reviewed. The safest direction is a small shared outbound or channel-helper guard that drops only `HEARTBEAT_TRANSCRIPT_PROMPT` from visible delivery without changing heartbeat run semantics.
