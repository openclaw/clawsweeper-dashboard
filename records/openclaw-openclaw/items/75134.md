## Summary

Keep open. Current main still keeps the internal heartbeat transcript marker on the heartbeat run path, Telegram still treats non-final delivered text as visible without checking that marker, and the open linked PR #75167 explicitly closes this issue but has not merged.

## What This Changes

The report asks Telegram DM heartbeat delivery to hide the internal `[OpenClaw heartbeat poll]` prompt while preserving normal heartbeat alert delivery and default `HEARTBEAT_OK` suppression.

## Best Possible Solution

Land or replace the linked PR with a narrow, tested suppression of only `HEARTBEAT_TRANSCRIPT_PROMPT` before user-visible channel delivery, while preserving meaningful heartbeat alerts, configured `showOk` behavior, and the underlying transcript audit trail.

## Reproduction Assessment

Yes. The issue provides concrete Telegram DM heartbeat steps and config, and current-main source inspection still shows the marker-producing heartbeat path plus a Telegram visibility helper that does not filter the marker text.

## Solution Assessment

Unclear. No patch is being reviewed here; the safest solution is a narrow shared outbound or channel-helper guard that drops only `HEARTBEAT_TRANSCRIPT_PROMPT` from visible delivery with Telegram regression coverage.
