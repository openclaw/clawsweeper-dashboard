## Summary

Keep open. Current main still contains the heartbeat marker path and Telegram's visibility hook still ignores the outbound text before treating non-final deliveries as visible; PR #75167 explicitly closes this issue but is still open and unmerged.

## What This Changes

The issue requests that Telegram and other channel surfaces suppress the internal `[OpenClaw heartbeat poll]` heartbeat prompt from user-visible chat output.

## Best Possible Solution

Suppress only `HEARTBEAT_TRANSCRIPT_PROMPT` before it can become user-visible channel output, while preserving audit transcripts, real heartbeat alerts, and configured `HEARTBEAT_OK` visibility behavior.

## Reproduction Assessment

Yes. The issue gives concrete Telegram DM heartbeat steps and config, and source inspection shows the reported marker still flows into heartbeat transcript handling while Telegram has no text-based guard for it.

## Solution Assessment

Unclear whether PR #75167 is the best patch until it is reviewed. The maintainable direction is a narrow shared or channel visibility guard for the exact internal marker, not a broad rewrite of heartbeat generation semantics.
