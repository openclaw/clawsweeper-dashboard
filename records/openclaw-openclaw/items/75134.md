## Summary

Keep open. The report is concrete, current main still shows the heartbeat transcript marker flowing through the heartbeat transcript path without a Telegram-facing marker filter, and open PR #75167 explicitly closes this issue but remains unmerged.

## What This Changes

The issue requests that Telegram and other external channel surfaces suppress the internal heartbeat transcript prompt `[OpenClaw heartbeat poll]` while continuing to suppress default `HEARTBEAT_OK` acknowledgments.

## Best Possible Solution

Land PR #75167 or an equivalent narrow fix that suppresses `HEARTBEAT_TRANSCRIPT_PROMPT` before any external channel delivery, with Telegram regression coverage and preservation of meaningful heartbeat alerts plus optional `showOk` behavior.

## Reproduction Assessment

Yes. The issue gives a concrete Telegram DM heartbeat configuration, and current main exposes a focused reproduction path: heartbeat runs use the internal transcript marker while the Telegram/shared delivery paths inspected here do not filter that marker.

## Solution Assessment

Unclear until PR #75167 is reviewed. The safest maintainable direction appears to be a shared pre-delivery guard or tightly covered channel filter that prevents the internal marker from reaching external chat surfaces without suppressing real heartbeat alert content.
