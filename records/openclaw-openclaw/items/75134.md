## Summary

Keep open. Current main still routes the internal heartbeat transcript marker through the heartbeat reply path, and Telegram's visibility helper does not inspect the delivered text before treating non-final blocks as visible. The linked PR #75167 explicitly closes this issue but remains open and unmerged, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Prevent the raw `[OpenClaw heartbeat poll]` internal marker from appearing as a visible Telegram DM message while continuing to suppress default `HEARTBEAT_OK` acknowledgments.

## Best Possible Solution

Land or replace the linked PR with a narrow, tested suppression of only the internal heartbeat transcript marker before it reaches user-visible channel delivery, while preserving meaningful heartbeat alerts and configured `showOk` behavior.

## Reproduction Assessment

Yes. The issue provides concrete Telegram DM heartbeat steps and config, and current-main source inspection shows the marker-producing heartbeat path plus a Telegram visible-text helper that does not filter the marker.

## Solution Assessment

Unclear until the linked PR is reviewed. The safest direction is a narrow shared outbound or channel-helper guard that drops only `HEARTBEAT_TRANSCRIPT_PROMPT` from visible delivery, with Telegram regression coverage and no broader heartbeat delivery changes.
