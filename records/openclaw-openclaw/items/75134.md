## Summary

Keep open. Current main still carries the exact heartbeat transcript marker and Telegram's visible-delivery helper does not inspect outgoing text, while PR #75167 explicitly closes this issue but remains open and unmerged.

## What This Changes

The issue asks for Telegram heartbeat poll prompts to stay internal so `[OpenClaw heartbeat poll]` never appears in user-visible DM chat.

## Best Possible Solution

Land or replace PR #75167 with a narrow suppression of only the internal heartbeat transcript marker before it can become user-visible channel output, while preserving heartbeat alert delivery and audit transcripts.

## Reproduction Assessment

Yes. The issue provides concrete Telegram DM heartbeat steps and config, and current-main source inspection shows the marker-producing heartbeat path plus a Telegram visibility helper that cannot filter the marker text.

## Solution Assessment

Unclear until PR #75167 is reviewed. The safest direction is a small shared outbound or channel-helper guard for the exact internal marker, not a broader change to heartbeat prompt generation or heartbeat visibility semantics.
