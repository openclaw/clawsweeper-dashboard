## Summary

Keep open: the security docs idea is useful, but the proposed templates still contain a fail-open staged-secret hook and moving or unverifiable third-party execution guidance, and current main does not already provide these pages.

## What This Changes

Adds two docs/security files: a GitHub Actions security scan workflow template using Trivy, KICS, and TruffleHog, plus a TruffleHog pre-commit hook guide.

## Best Possible Solution

Publish these docs only after the hook streams staged paths fail-closed, third-party action/runtime/install provenance is pinned or explicitly verified, and the pages are discoverable with secops-approved guidance.

## Reproduction Assessment

Yes for the PR-patch defects: source inspection and a focused Bash check show command substitution strips NUL bytes before the `read -d ''` loop. This is not a current-main product bug because the docs are unmerged.

## Solution Assessment

No. The topic is useful, but the current patch is not the best mergeable form until the fail-open hook logic, supply-chain pinning, docs metadata/navigation, and KICS provenance issues are fixed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Stream staged paths directly:** `docs/security/pre-commit-secrets.md:41-58`
  - body: `git diff --cached --name-only -z` emits NUL bytes, but command substitution strips them before `read -d ''`. The loop can copy no staged blobs, leaving TruffleHog to scan an empty directory while staged secrets pass.
  - confidence: 0.96
- **[P2] Pin upload-artifact to an immutable SHA:** `docs/security/scan-workflow-template.yml:72`
  - body: The template says action refs are SHA-pinned, but this step still uses the mutable `actions/upload-artifact@v4` tag. Use a full commit SHA and keep the tag only as a comment.
  - confidence: 0.94
- **[P2] Set a fixed TruffleHog runtime version:** `docs/security/scan-workflow-template.yml:98`
  - body: The pinned TruffleHog action defaults its scanner image to `latest` unless the `version` input is set, so copied workflows would still execute a moving runtime.
  - confidence: 0.9
- **[P2] Use pinned TruffleHog install artifacts:** `docs/security/pre-commit-secrets.md:13-21`
  - body: The Linux install path downloads a script from upstream `main`, and the Go alternative uses `@latest`. A security guide should recommend pinned release artifacts and expected checksums instead of moving upstream code.
  - confidence: 0.88
- **[P3] Add docs front matter and navigation:** `docs/security/pre-commit-secrets.md:1`
  - body: The new Markdown guide lacks the Mintlify front matter used by current docs/security pages, and neither new page is listed in `docs/docs.json`, so the docs will not publish like the rest of the security section.
  - confidence: 0.83
- **[P3] Verify or remove the KICS version label:** `docs/security/scan-workflow-template.yml:60-61`
  - body: The KICS action is pinned to a SHA, but the comment labels it as `Checkmarx/kics-github-action@v2.1.0` and a tag lookup returned no matching tag. Use a verified tag/SHA pair or remove the version claim.
  - confidence: 0.82
