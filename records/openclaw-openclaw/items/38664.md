## What This Changes

The PR adds two docs/security files: a reusable GitHub Actions template for Trivy, KICS, and TruffleHog, plus a TruffleHog pre-commit hook guide.

## Best Possible Solution

Publish these security template docs only after the hook is fail-closed, third-party action/runtime/installer references are immutable or verifiable, docs navigation and front matter are complete, and security/docs owners accept the recommended scanner posture.

## Solution Assessment

  - confidence: 0.94

  - body: The pinned TruffleHog action defaults its scanner runtime to `latest` unless the `version` input is set. Copied workflows would still execute a moving scanner image despite the SHA-pinning guidance.
  - confidence: 0.9

  - body: The Linux install path downloads `install.sh` from upstream `main` and runs it after only printing a checksum command. For security guidance, use a pinned release artifact with an expected digest or another verifiable install method.
  - confidence: 0.9
