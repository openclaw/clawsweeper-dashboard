## Summary

Keep open. Current main does not include an equivalent Trivy/KICS/TruffleHog template or TruffleHog staged-content hook, but this PR is not ready to merge because the latest patch still contains a fail-open hook defect and security-sensitive third-party execution/pinning problems in user-copyable guidance.

## What This Changes

This PR adds two docs/security files: a TruffleHog pre-commit secret-scanning guide and a GitHub Actions security scan workflow template for Trivy, KICS, and TruffleHog.

## Best Possible Solution

If this belongs in core docs, land it as a secops-reviewed docs update integrated into the existing Security section, with front matter/navigation, streamed staged-path parsing, immutable action refs, verifiable third-party install instructions, and corrected action version provenance. If generic scanner templates are out of scope, close it with that product rationale rather than merging partial security guidance.

## Reproduction Assessment

Yes for the blocking hook defect: Bash strips NUL bytes from command substitution, and a focused shell check reproduces the loss before the read -d '' loop. Reproduction is otherwise not applicable to the broader docs/product-scope decision.

## Solution Assessment

No. The proposed direction may be useful, but the current patch is not the safest solution because it teaches a fail-open staged secret scan and includes mutable or unverifiable third-party execution guidance.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Stream staged paths directly:** `docs/security/pre-commit-secrets.md:41`
  - body: `git diff --cached --name-only -z` emits NUL bytes, but command substitution strips them before assigning `STAGED_FILES`. The later `read -d ''` loop can then skip copying staged blobs, so TruffleHog scans an empty temp directory and staged secrets pass unchecked. Pipe or process-substitute the git output directly into the loop.
  - confidence: 0.96
- **[P2] Pin upload-artifact to a commit SHA:** `docs/security/scan-workflow-template.yml:72`
  - body: The template says action refs are pinned to commit SHAs, but this step still uses `actions/upload-artifact@v4`. Because this is security workflow guidance, replace the mutable tag with the full commit SHA and keep the tag only as a comment.
  - confidence: 0.94
- **[P2] Use a verifiable TruffleHog install path:** `docs/security/pre-commit-secrets.md:14-18`
  - body: The Linux install section calls this a verified binary path, but it downloads `install.sh` from the mutable upstream `main` branch and runs it after only printing a checksum. Use a pinned release artifact with an expected digest or another verifiable install method.
  - confidence: 0.9
- **[P3] Add docs metadata and navigation:** `docs/security/pre-commit-secrets.md:1`
  - body: The new Markdown page has no Mintlify front matter and is not added to `docs/docs.json`, while the existing Security section lists pages explicitly. As-is, this guide may not publish or appear as a normal docs page.
  - confidence: 0.83
- **[P3] Verify the KICS action version label:** `docs/security/scan-workflow-template.yml:60-61`
  - body: The pinned KICS SHA may be intentional, but the comment labels it as `Checkmarx/kics-github-action@v2.1.0` even though that tag does not resolve via `git ls-remote`. Use an existing tag/SHA pair or remove the version claim so update instructions are accurate.
  - confidence: 0.78
