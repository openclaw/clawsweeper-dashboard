  - body: `STAGED_FILES=$(git diff --cached --name-only -z ...)` strips NUL bytes during Bash command substitution, so the later `read -d ''` loop never copies staged blobs into `TH_TMPDIR`. That leaves TruffleHog scanning an empty directory and lets staged secrets pass the documented hook.

  - body: The Linux install path downloads `install.sh` from `main` and the Go fallback uses `@latest`, so readers copying this security guide still execute mutable, unpinned code. Use a fixed release artifact with an expected checksum or a package-manager path that provides provenance.
  - confidence: 0.9
- **[P2] Pin upload-artifact to a SHA:** `docs/security/scan-workflow-template.yml:72`
  - body: This workflow states that action refs are SHA-pinned, but the KICS artifact step still uses `actions/upload-artifact@v4`. Pin this action to the resolved commit SHA with the tag in a comment so the security template is reproducible.
  - confidence: 0.93

  - body: Pinning the TruffleHog action commit does not pin the scanner runtime: upstream `action.yml` defaults the `version` input to `latest`. Add an explicit fixed `version` input, or otherwise verify the scanner image, before presenting this as pinned security guidance.
