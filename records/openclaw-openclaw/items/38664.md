## Summary

This PR should stay open: the security-docs direction is plausible, but the proposed copy-paste templates still have concrete fail-open and supply-chain problems that should be resolved before publication.

## What This Changes

The PR adds two security documentation files: a GitHub Actions workflow template for Trivy, KICS, and TruffleHog, plus a TruffleHog pre-commit hook guide for staged secret scanning.

## Best Possible Solution

Publish the security scanning docs only after the hook is fail-closed, third-party action and runtime provenance is immutable or clearly verified, and the pages are wired into the normal docs navigation with secops-approved guidance.

## Reproduction Assessment

Yes for the PR-patch defects: source inspection plus a focused Bash check reproduce the staged-path parsing failure, and upstream metadata shows the moving TruffleHog runtime. This is not a current-main product bug because the patch is not merged.

## Solution Assessment

No. The documentation direction is reasonable, but this patch is not yet the best mergeable form until the fail-open hook logic, supply-chain pins, docs publishing gaps, and secops posture questions are resolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Stream staged paths directly:** `docs/security/pre-commit-secrets.md:41-58`
  - body: `git diff --cached --name-only -z` emits NUL bytes, but command substitution strips them before the here-string reaches `read -d ''`. The loop can copy zero staged blobs, so TruffleHog scans an empty directory and staged secrets pass unchecked.
  - confidence: 0.96
- **[P2] Pin upload-artifact to an immutable SHA:** `docs/security/scan-workflow-template.yml:72`
  - body: The template says action refs are SHA-pinned, but this step still uses the mutable `actions/upload-artifact@v4` tag. Replace it with a full commit SHA and keep the tag only as a comment.
  - confidence: 0.94
- **[P2] Set a fixed TruffleHog runtime version:** `docs/security/scan-workflow-template.yml:98`
  - body: The pinned TruffleHog action defaults its scanner image to `latest` unless the `version` input is set. Copied workflows would still execute a moving runtime despite the SHA-pinning guidance.
  - confidence: 0.9
- **[P2] Use pinned TruffleHog install artifacts:** `docs/security/pre-commit-secrets.md:13-21`
  - body: The Linux install path downloads a script from upstream `main` and the Go alternative uses `@latest`, so users copying this security guide execute moving upstream content without a pinned artifact/version and expected checksum.
  - confidence: 0.88
- **[P3] Add docs front matter and navigation:** `docs/security/pre-commit-secrets.md:1`
  - body: The new Markdown guide lacks Mintlify front matter and is not listed in `docs/docs.json`, unlike current `docs/security` pages. Add metadata and navigation if this page should publish as normal docs.
  - confidence: 0.83
- **[P3] Verify or remove the KICS version label:** `docs/security/scan-workflow-template.yml:60-61`
  - body: The KICS action is pinned to a commit SHA, but the comment labels it as `Checkmarx/kics-github-action@v2.1.0` and that tag lookup returned 404. Use a verified tag/SHA pair or remove the version claim.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The docs-only patch is security-sensitive and still contains fail-open hook behavior plus mutable or unverifiable third-party execution guidance.

Concerns:

  - body: The hook stores NUL-delimited staged paths in a Bash variable, which strips delimiters before the `read -d ''` loop and can leave the temp directory empty before TruffleHog runs.

  - body: `actions/upload-artifact@v4` remains a mutable tag in a workflow template that claims SHA-pinned action refs for supply-chain hardening.
  - confidence: 0.94

  - body: The pinned TruffleHog action still defaults to a moving scanner runtime unless the template sets a fixed supported `version` input.
  - confidence: 0.9
- **[medium] Unverified TruffleHog install paths:** `docs/security/pre-commit-secrets.md:13`
  - body: The guide downloads an install script from upstream `main` and offers `go install ...@latest`, so users copying it execute moving upstream content without a pinned artifact or expected checksum.
  - confidence: 0.88
