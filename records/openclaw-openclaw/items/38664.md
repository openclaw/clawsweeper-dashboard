## What This Changes

The PR adds two `docs/security` files: a GitHub Actions security scan template using Trivy, KICS, and TruffleHog, plus a TruffleHog pre-commit hook guide.

## Best Possible Solution

Publish the security template docs only after the hook is fail-closed, third-party action/runtime/installer references are immutable or verifiable, docs front matter/navigation is complete, real behavior proof is posted, and security/docs owners accept the scanner posture.

## Solution Assessment

No. The docs direction is plausible, but this patch is not the safest maintainable solution until the fail-open hook, mutable or unverifiable third-party references, docs publishing gaps, and missing proof are corrected.

  - body: `git diff --cached --name-only -z` emits NUL bytes, but command substitution strips them before `read -d ''` runs. The loop can copy no staged blobs, leaving TruffleHog to scan an empty temp directory while staged secrets pass unchecked.

  - confidence: 0.94

  - body: The pinned TruffleHog action defaults its scanner runtime to `latest` unless the `version` input is set. Copied workflows would still execute a moving scanner image despite the SHA-pinning guidance.
  - confidence: 0.9

  - body: The Linux install path downloads `install.sh` from upstream `main` and runs it after only printing a checksum command. For security guidance, use a pinned release artifact with an expected digest or another verifiable install method.
  - confidence: 0.9
