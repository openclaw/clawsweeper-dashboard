## Summary

Keep open: the docs direction is useful, but the copy-paste templates are not merge-ready because the staged-file hook can scan an empty directory and the security workflow still has mutable/unpinned execution points.

## What This Changes

The PR adds docs/security templates for a Trivy/KICS/TruffleHog GitHub Actions scan and a TruffleHog-based pre-commit secret-scanning hook.

## Best Possible Solution

Repair the two templates so staged blobs are actually scanned and every security-sensitive action, runtime image, and install path is immutably pinned or checksum-verified, then route through secops docs review.

## Reproduction Assessment

Yes for the blocking PR findings: source inspection plus a shell probe show Bash strips NUL bytes from the `git diff -z` command substitution, and upstream action metadata shows TruffleHog defaults to `latest`. This is not a current-main bug because the affected files only exist on the PR branch.

## Solution Assessment

No. Adding security scan docs is a reasonable direction, but this patch should first stream NUL-delimited paths directly and remove the remaining mutable action/runtime/download paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Stream staged paths instead of storing NUL output:** `docs/security/pre-commit-secrets.md:41`
  - body: `git diff -z` emits NUL-delimited names, but command substitution cannot preserve NUL bytes; Bash drops them before the here-string reaches `read -d ''`. The loop therefore never copies staged blobs into `TH_TMPDIR`, so TruffleHog scans an empty directory and commits with secrets can pass. Pipe or process-substitute the `git diff -z` output directly into the loop.
  - confidence: 0.97
- **[P2] Pin the upload-artifact action:** `docs/security/scan-workflow-template.yml:72`
  - body: This template says action references are SHA-pinned, but the KICS upload step still uses the mutable `actions/upload-artifact@v4` tag. Since this is a security workflow template, users copying it would still execute a moving third-party action; replace it with a full commit SHA and keep the tag only in a comment.
  - confidence: 0.95
- **[P2] Pin the TruffleHog CLI image version:** `docs/security/scan-workflow-template.yml:98`
  - body: The action code is SHA-pinned, but upstream `action.yml` defaults `version` to `latest`, which drives `ghcr.io/trufflesecurity/trufflehog:${VERSION}`. Without `with: version: v3.88.0` or another reviewed fixed version, the template still runs a moving scanner image despite claiming exact pinning.
  - confidence: 0.9
- **[P2] Use a checksum-verified Linux install path:** `docs/security/pre-commit-secrets.md:14-18`
  - body: The Linux install block downloads the installer from the mutable `main` branch and then executes it after printing only a local hash. There is no expected checksum or signature comparison, so users copying this security guide still run unverified remote code; use release artifacts with published checksums or remove the verified-binary claim.
  - confidence: 0.86
