## Summary

Keep this PR open: the security-scanning docs are not on current main, but the proposed templates still have actionable security and docs-publishing defects that should block merge until secops/docs review approves the posture.

## What This Changes

Adds two docs/security pages: a copy-paste GitHub Actions security-scan workflow and a TruffleHog pre-commit hook guide.

## Best Possible Solution

Publish these docs only after the hook streams staged paths fail-closed, all third-party action/runtime/install provenance is pinned or explicitly verified, and the pages are wired into Mintlify with secops-approved guidance.

## Reproduction Assessment

Yes for the PR-patch defects: source inspection plus a focused Bash check reproduces the staged-path parsing failure, and upstream action metadata shows the TruffleHog runtime remains moving. This is not a current-main product bug because the docs are unmerged.

## Solution Assessment

No. The documentation direction is useful, but this patch is not yet the best mergeable form until the fail-open hook logic, supply-chain provenance gaps, docs navigation/front matter, and secops posture questions are resolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Stream staged paths directly:** `docs/security/pre-commit-secrets.md:41-58`
  - body: `git diff --cached --name-only -z` emits NUL bytes, but command substitution strips them before `read -d ''`. The loop can copy no staged blobs, leaving TruffleHog to scan an empty directory while staged secrets pass.
  - confidence: 0.96
- **[P2] Pin upload-artifact to an immutable SHA:** `docs/security/scan-workflow-template.yml:72`
  - body: The template says action refs are SHA-pinned, but this step still uses the mutable `actions/upload-artifact@v4` tag. Use a full commit SHA and keep the tag only as a comment.
  - confidence: 0.94
- **[P2] Set a fixed TruffleHog runtime version:** `docs/security/scan-workflow-template.yml:98`
  - body: The pinned TruffleHog action defaults its scanner image to `latest` unless the `version` input is set, so copied workflows would still execute a moving runtime.
  - confidence: 0.9
- **[P2] Use pinned TruffleHog install artifacts:** `docs/security/pre-commit-secrets.md:14-21`
  - body: The Linux install path downloads a script from upstream `main`, and the Go alternative uses `@latest`. A security guide should recommend pinned release artifacts and expected checksums instead of moving upstream code.
  - confidence: 0.88
- **[P3] Add docs front matter and navigation:** `docs/security/pre-commit-secrets.md:1`
  - body: The new Markdown guide lacks the Mintlify front matter used by current docs/security pages and neither new page is listed in `docs/docs.json`, so the docs will not publish like the rest of the security section.
  - confidence: 0.83
- **[P3] Verify or remove the KICS version label:** `docs/security/scan-workflow-template.yml:60`
  - body: The KICS action is pinned to a SHA, but the comment labels it as `Checkmarx/kics-github-action@v2.1.0` and a tag lookup returned no matching tag. Use a verified tag/SHA pair or remove the version claim.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The docs-only patch is security-sensitive and still contains fail-open hook behavior plus mutable or unverifiable third-party execution guidance.

Concerns:

- **[high] Fail-open staged secret scan:** `docs/security/pre-commit-secrets.md:41`
  - body: The hook stores NUL-delimited staged paths in a Bash variable, which strips delimiters before the `read -d ''` loop and can leave the temp directory empty before TruffleHog runs.
  - confidence: 0.96
- **[medium] Mutable GitHub Action tag:** `docs/security/scan-workflow-template.yml:72`
  - body: `actions/upload-artifact@v4` remains a mutable tag in a workflow template that claims SHA-pinned action refs for supply-chain hardening.
  - confidence: 0.94
- **[medium] Mutable TruffleHog runtime image:** `docs/security/scan-workflow-template.yml:98`
  - body: The pinned TruffleHog action still defaults to a moving scanner runtime unless the template sets a fixed supported `version` input.
  - confidence: 0.9
- **[medium] Unverified TruffleHog install paths:** `docs/security/pre-commit-secrets.md:14`
  - body: The guide downloads an install script from upstream `main` and offers `go install ...@latest`, so users copying it execute moving upstream content without a pinned artifact or expected checksum.
  - confidence: 0.88
- **[low] Unverified KICS release provenance:** `docs/security/scan-workflow-template.yml:60`
  - body: The KICS SHA exists in the template, but the documented `v2.1.0` label did not resolve and should be verified before maintainers recommend the template.
  - confidence: 0.82
