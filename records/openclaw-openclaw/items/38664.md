## Summary

Keep open: current main does not already contain these security docs, but the PR still has blocking security-doc and supply-chain guidance defects that would make the copied templates fail open or execute moving third-party code.

## What This Changes

The PR adds a Trivy/KICS/TruffleHog GitHub Actions workflow template and a TruffleHog pre-commit hook guide under docs/security.

## Best Possible Solution

Publish this guidance only after secops-approved, fail-closed shell logic, immutable or version-pinned third-party actions/runtimes/install artifacts, verified KICS provenance, and Mintlify metadata/navigation are in place.

## Reproduction Assessment

Yes. The PR-patch defects are reproducible from source and focused checks: Bash command substitution strips NUL-delimited staged paths, upstream TruffleHog action metadata defaults the scanner runtime to latest, and the mutable or unresolved action refs are visible from the PR source and tag lookups.

## Solution Assessment

No. The docs direction is plausible, but this patch is not the narrowest maintainable solution until the fail-open hook logic, mutable or unverified execution paths, docs publishing gaps, and secops posture review are addressed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Stream staged paths directly:** `docs/security/pre-commit-secrets.md:41-58`
  - body: `git diff --cached --name-only -z` emits NUL bytes, but Bash command substitution strips them before the here-string reaches `read -d ''`. The loop can copy zero staged blobs, so TruffleHog scans an empty directory and staged secrets pass unchecked.
  - confidence: 0.96
- **[P2] Pin upload-artifact to a commit SHA:** `docs/security/scan-workflow-template.yml:72`
  - body: The template says action refs are SHA-pinned, but this step still uses the mutable `actions/upload-artifact@v4` tag. Replace it with a full commit SHA and keep the tag only as a comment.
  - confidence: 0.94
- **[P2] Set a fixed TruffleHog runtime version:** `docs/security/scan-workflow-template.yml:98`
  - body: The pinned TruffleHog action defaults its scanner image to `latest` unless the `version` input is set. Copied workflows would still execute a moving runtime despite the SHA-pinning guidance.
  - confidence: 0.9
- **[P2] Use pinned TruffleHog install artifacts:** `docs/security/pre-commit-secrets.md:13-21`
  - body: The Linux install paths download an install script from upstream `main` and offer `go install ...@latest`, so users copying this security guide execute moving upstream content without a pinned artifact/version and expected checksum.
  - confidence: 0.88
- **[P3] Add docs front matter and navigation:** `docs/security/pre-commit-secrets.md:1`
  - body: The new Markdown guide lacks Mintlify front matter and is not listed in `docs/docs.json`, unlike current `docs/security` pages. Add metadata and navigation if this page should publish as normal docs.
  - confidence: 0.83
- **[P3] Verify the KICS tag label:** `docs/security/scan-workflow-template.yml:60-61`
  - body: The KICS action is pinned to a commit SHA, but the comment labels it as `Checkmarx/kics-github-action@v2.1.0` and that tag lookup did not resolve. Use a verified tag/SHA pair or remove the version claim.
  - confidence: 0.82
