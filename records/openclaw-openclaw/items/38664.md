## What This Changes

Adds two docs/security files: a copy-paste GitHub Actions security scan workflow and a TruffleHog pre-commit secret scanning guide.

## Best Possible Solution

Publish the security scanning docs only after the staged-file hook is fail-closed, third-party action/runtime/install provenance is immutable or explicitly verified, and the pages are wired into Mintlify with owner-approved guidance.

## Reproduction Assessment

Yes for the PR-patch defects: source inspection plus a focused Bash check shows NUL-delimited staged paths are lost in command substitution, and upstream action metadata shows the TruffleHog runtime remains latest. This is not a current-main product bug because the docs are not merged.

## Solution Assessment

No. The documentation direction may be useful, but this patch is not yet the best mergeable form until the fail-open hook logic, supply-chain provenance gaps, docs publishing gaps, and secops posture questions are resolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Stream staged paths directly:** `docs/security/pre-commit-secrets.md:41-58`
  - body: `git diff --cached --name-only -z` emits NUL bytes, but storing it in `STAGED_FILES=$(...)` strips those delimiters before `read -d ''`. The loop can copy no staged blobs, leaving TruffleHog to scan an empty directory while staged secrets pass.
  - confidence: 0.96
- **[P2] Pin upload-artifact to an immutable SHA:** `docs/security/scan-workflow-template.yml:72`
  - body: The template says action refs are SHA-pinned, but this step still uses the mutable `actions/upload-artifact@v4` tag. Use a full commit SHA and keep the tag only as a comment.
  - confidence: 0.94
- **[P2] Set a fixed TruffleHog runtime version:** `docs/security/scan-workflow-template.yml:98`
  - body: The pinned TruffleHog action defaults its scanner image to `latest` unless the `version` input is set, so copied workflows would still execute a moving runtime.
  - confidence: 0.9
- **[P2] Use pinned TruffleHog install artifacts:** `docs/security/pre-commit-secrets.md:14-21`
  - body: The Linux install path downloads a script from upstream `main`, and the Go alternative uses `@latest`. A security guide should recommend pinned release artifacts and expected checksums instead of moving upstream code.
  - confidence: 0.88
- **[P3] Add docs front matter and navigation:** `docs/security/pre-commit-secrets.md:1`
  - body: The new Markdown guide lacks the Mintlify front matter used by current docs/security pages and is not listed in `docs/docs.json`, so it will not publish like the rest of the docs set.
  - confidence: 0.83
- **[P3] Verify or remove the KICS version label:** `docs/security/scan-workflow-template.yml:60`
  - body: The KICS action is pinned to a SHA, but the comment labels it as `Checkmarx/kics-github-action@v2.1.0` and that tag lookup returned 404. Use a verified tag/SHA pair or remove the version claim.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The docs-only patch is security-sensitive and still contains fail-open hook behavior plus mutable or unverifiable third-party execution guidance.

  - confidence: 0.94

  - confidence: 0.9

  - body: The guide downloads an install script from upstream `main` and offers `go install ...@latest`, so users copying it execute moving upstream content without a pinned artifact or expected checksum.
  - confidence: 0.88
