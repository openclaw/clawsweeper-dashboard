Keep open: current main and v2026.6.9 still leave a real native Codex gpt-5.5 context contract mismatch. OpenClaw can resolve configured 1M/950k context metadata for the OpenAI/Codex route, while upstream Codex clamps `model_context_window` to the model catalog maximum; the direct fix PR was closed unmerged and no open PR currently owns this exact issue.

## What This Changes

The issue asks OpenClaw to make native Codex OAuth gpt-5.5 context reporting and runtime behavior agree when user config advertises 1M/950k but Codex runs at 258400 effective tokens.

## Best Possible Solution

Define one native Codex effective-context contract, then align runtime context accounting, model/status output, docs, and regression tests around that contract while documenting any supported larger-context API/OpenClaw route separately.
