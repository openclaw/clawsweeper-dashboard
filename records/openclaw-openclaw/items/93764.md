The issue asks OpenClaw to reconcile configured 1M OpenAI gpt-5.5 context metadata with the native Codex OAuth runtime's 272k catalog and 258400 effective cap.

## Best Possible Solution

Define one native Codex effective-context contract, then align runtime context accounting, models list/status output, docs, and regression tests around that contract while documenting any supported larger-context OpenClaw/API route separately.
