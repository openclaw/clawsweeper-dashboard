## Summary

Keep open: current main and v2026.6.9 still preserve the configured override path for native Codex runs, while upstream Codex clamps above its max window. The direct fix PR is closed unmerged, so the remaining work is a maintainer/product contract decision rather than conservative cleanup.

## What This Changes

The issue asks OpenClaw to stop implying a 1M/950k context budget for OpenAI Codex OAuth gpt-5.5 native turns when Codex reports and uses a 272k catalog window with a 258400 effective runtime window.

## Best Possible Solution

Define one runtime-specific effective context contract for native Codex OAuth, align models/status/session accounting/docs/tests to that contract, and require live Codex OAuth proof for the chosen path.

## Reproduction Assessment

Yes, at source level: current main can accept configured OpenAI provider context metadata for native Codex runs while Codex source clamps model_context_window to max_context_window. I did not execute a live OAuth run in this read-only review.

## Solution Assessment

Unclear: capping/status-warning/runtime-switch options are all plausible, but maintainers need to choose the canonical native Codex context contract before implementation.
