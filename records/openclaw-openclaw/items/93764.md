The issue asks OpenClaw to stop implying a 950k/1M budget for OAuth/native Codex gpt-5.5 when the native Codex turn uses the smaller runtime cap, or provide a supported OAuth-preserving path that really uses the larger window.

## Best Possible Solution

Keep the issue open and define one runtime-aware contract: native Codex status, model listing, session budgets, and docs should report the Codex effective cap while keeping the API 1M path distinct.

## Reproduction Assessment

Yes, source-level: configure openai/gpt-5.5 with 1M/950k metadata and use the native Codex runtime; current OpenClaw can preserve configured metadata while upstream Codex clamps model_context_window to its 272k max. I did not run a live OAuth Codex turn in this read-only review.

## Solution Assessment
