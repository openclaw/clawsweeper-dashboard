## Summary

Keep open: current main and v2026.6.9 still define strict exec approval behavior, the two closing-syntax PRs are closed unmerged, and the remaining request is a security/product decision about task-scoped trust rather than conservative cleanup.

## What This Changes

The issue requests task-scoped autonomous approval behavior so explicitly assigned agent tasks do not repeatedly ask for manual approval.

## Best Possible Solution

Preserve strict ask behavior by default and, if maintainers want task-scoped autonomy, design an explicit audited opt-in policy that composes with existing auto/full modes and heartbeat safety work.
