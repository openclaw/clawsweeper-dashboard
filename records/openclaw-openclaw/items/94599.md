## Summary

Keep open, but not as a strict regression: current main and the latest release already expose no-prompt and auto-review exec modes, while the remaining request would change the security/product model for task-scoped approvals and needs concrete policy/config evidence.

## What This Changes

The issue requests fewer repeated manual approval prompts for actions that are within an explicitly assigned autonomous agent task.

## Best Possible Solution

Keep the existing safe defaults and documented full/auto modes, then decide whether an explicit opt-in task/session-scoped approval policy is worth adding after the reporter supplies effective policy and approval-route evidence.

## Reproduction Assessment

Partly. Source and docs show that when approvals are required, allow-once is per-command and stricter host policy can keep prompting; there is no high-confidence reproduction of a latest-release regression because the issue omits effective policy, command, route, and logs.

## Solution Assessment

No as a bug-fix path. Existing full/off and auto modes are the narrow supported mechanisms today; treating task assignment itself as reusable approval would need an explicit security/product decision and probably a bounded opt-in policy surface.
