## Summary

Keep open: current main and the latest release intentionally keep `ask="always"` as a prompt-every-command mode, while the existing supported no-prompt path is `security="full"` plus `ask="off"`; changing assigned-task approval semantics is a product/security decision, not conservative cleanup.

## What This Changes

Request that explicitly assigned or cron-like agent tasks execute without repeated manual approvals for actions within the assigned task scope.

## Best Possible Solution

Keep the documented no-approval knobs for trusted local automation, and only add task-scoped trust if maintainers approve an explicit audited opt-in policy with tests and docs.

## Reproduction Assessment

Yes at source level: current main preserves `ask="always"` as approval-required for every command, and only cron gets the no-route headless inline fallback. No live agent run was executed in this read-only review, and the issue does not provide the exact policy or logs.

## Solution Assessment

No as a direct implicit-policy change: current OpenClaw already supports no-prompt trusted automation through `security="full"` and `ask="off"`. A safer solution would be an explicit maintainer-approved task-scoped trust policy, not silently downgrading `ask="always"`.
