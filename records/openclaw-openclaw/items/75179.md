## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep split-line offsets contiguous:** `packages/memory-host-sdk/src/host/internal.ts:505-515`
  - body: For long source lines split across multiple chunks, the first flush advances `charOffset` as if a newline existed between adjacent split segments, and later chunks subtract two from `currentRawChars` even when they contain only a non-first segment. That makes the next chunk start one character late and gives the stored offset range the wrong length, so long session messages get incorrect offset metadata.
  - confidence: 0.88
- **[P2] Regenerate the Plugin SDK API baseline:** `src/plugin-sdk/memory-core-host-engine-storage.ts:7`
  - body: This public SDK subpath now exports `chunkMarkdownWithOffset` and `ChunkWithOffset`, but the patch does not update `docs/.generated/plugin-sdk-api-baseline.sha256`. Intentional SDK surface changes need `pnpm plugin-sdk:api:gen` committed or `pnpm plugin-sdk:api:check` will report drift.
  - confidence: 0.92
