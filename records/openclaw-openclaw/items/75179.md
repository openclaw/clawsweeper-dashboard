## Summary

Keep open. The linked session-memory performance bug is still real on current main, and this PR is a plausible replacement for the closed earlier implementation, but the latest branch still has merge-blocking offset, Plugin SDK baseline, and changelog issues.

## What This Changes

The PR adds hash-based incremental indexing for changed session memory files, chunk offset metadata and SQLite columns, public memory-host SDK exports, and memory-core/host-sdk regression and benchmark tests.

## Best Possible Solution

Land the incremental session-sync approach only after the offset-aware chunking contract is correct, the Plugin SDK API baseline is regenerated, the changelog is updated, and focused memory-core plus memory-host SDK validation passes; otherwise replace it with a narrower branch that preserves the same owner boundary.

## Reproduction Assessment

Yes. Current main can be verified from source: selected changed sessions still call indexFile and writeChunks clears all rows for that file. The PR-specific offset defect is reproducible by calling chunkMarkdownWithOffset on a single source line longer than the chunk budget and observing the second split segment's startOffset drift.

## Solution Assessment

No. The design direction is reasonable for the linked performance bug, but this exact patch is not the best merge candidate until the offset metadata, generated SDK baseline, and changelog contract are repaired.
