## Best Possible Solution

Land a corrected incremental session sync that preserves unchanged embeddings, keeps offsets accurate through split lines and overlap, updates generated SDK contract state, and keeps MEMORY.md on the existing full-reindex path.

Yes, at source level. On current main, dirty session transcripts are rebuilt by buildSessionEntry and routed through indexFile/writeChunks, whose first step clears all chunks for the session path before rewriting them.
