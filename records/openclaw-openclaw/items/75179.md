## Summary

Keep open. The linked session-memory performance problem is still present on current main, and this PR is a useful implementation candidate, but the latest patch is not merge-ready because it still has a concrete offset-accounting bug plus required Plugin SDK API baseline and changelog follow-through.

## What This Changes

This PR adds hash-based incremental indexing for changed session memory files, chunk offset metadata and SQLite columns, public memory-host SDK exports, and memory-core/host-sdk regression and benchmark tests.

## Best Possible Solution

Land a corrected incremental session-sync implementation in memory-core that preserves existing chunks for changed session files, stores accurate offsets for long-line and overlap cases, keeps the public SDK baseline aligned, and includes the required user-facing changelog entry.

## Reproduction Assessment

Yes. Current main can be verified statically: a changed session file reaches syncSessionFiles, calls indexFile, and writeChunks clears all chunks for that path/source before reinsert. The PR blockers are also reproducible by inspecting the latest patch and, for offsets, by chunking a single line longer than the chunk budget.

## Solution Assessment

No. The incremental-session-sync direction is the right maintainable fix, but this exact patch should not merge until offset metadata is correct and the generated Plugin SDK/changelog contracts are updated.
