## Summary

Keep open. Current main still full-reindexes changed session memory files, and this PR is the right direction for the linked performance bug, but the current head is not merge-ready because the new offset chunking and public SDK surface have concrete mechanical blockers.

## What This Changes

The PR adds hash-based incremental indexing for changed session files, SQLite chunk offset metadata, memory host/Plugin SDK exports for offset-aware chunking, memory-core regression and benchmark tests, and an unreleased changelog entry.

## Best Possible Solution

Land a corrected incremental session-sync implementation that preserves unchanged session embeddings, stores accurate contiguous offsets across long-line and overlap cases, keeps the public Plugin SDK baseline aligned, and retains focused regression coverage plus the existing changelog entry.

## Reproduction Assessment

Yes. Current main can be verified statically: changed session files reach indexFile, and writeChunks clears all chunks for that path/source before reinserting. The PR blockers are also statically reproducible from the latest diff: split-line offset math is non-contiguous, the overload order loses offset typing, and the SDK API baseline is absent.

## Solution Assessment

No. The incremental-session-sync direction is a maintainable solution for #40919, but this exact patch should not merge until the offset math, overload typing, and Plugin SDK baseline drift are fixed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Keep split-line offsets contiguous:** `packages/memory-host-sdk/src/host/internal.ts:505-515`
  - body: For long source lines split across multiple chunks, the first flushed segment advances `charOffset` as if a newline existed before the next segment, while `endOffset` subtracts as if every chunk text was newline-joined. A 400-character first segment makes the next segment start at 401 instead of 400, so stored offset ranges become shifted and non-contiguous.
  - confidence: 0.88
- **[P2] Resolve offset chunks to the offset overload:** `packages/memory-host-sdk/src/host/embedding-chunk-limits.ts:8-17`
  - body: `ChunkWithOffset[]` is assignable to `MemoryChunk[]`, so placing the generic overload first lets calls with offset chunks resolve to a `MemoryChunk[]` return. The incremental path assigns that result to `ChunkWithOffset[]`, so typecheck can fail and callers lose the offset contract.
  - confidence: 0.84
- **[P2] Regenerate the Plugin SDK API baseline:** `src/plugin-sdk/memory-core-host-engine-storage.ts:7`
  - body: This public SDK subpath now exports `chunkMarkdownWithOffset` and `ChunkWithOffset`, but the patch does not update `docs/.generated/plugin-sdk-api-baseline.sha256`. Intentional SDK surface changes need `pnpm plugin-sdk:api:gen` committed or `pnpm plugin-sdk:api:check` will report drift.
  - confidence: 0.92
