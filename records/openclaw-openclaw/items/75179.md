## Summary

Keep open. The updated head addresses the earlier force-reindex regression, but it is still not merge-ready because the public Plugin SDK export needs the generated API baseline updated and the new offset-aware chunker records wrong offsets after long source-line splits.

## What This Changes

The PR adds hash-based incremental indexing for session memory files, chunk start/end offset storage, offset-aware memory host SDK exports, schema updates, and memory-core tests/benchmarks.

## Best Possible Solution

Keep the incremental session-sync direction, but land it only after the offset metadata is correct for normal and long-line chunks, the public SDK API baseline and changelog are updated, the branch is rebased onto current main, and the memory-core/plugin-SDK checks pass.

## Solution Assessment
