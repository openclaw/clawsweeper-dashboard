## Summary

Keep this PR open. Current main still rebuilds selected session transcripts through the full indexing path, so the incremental session-sync work is not implemented yet; the latest PR head also appears to have addressed the earlier force-reindex regression by preserving `indexFile` when `needsFullReindex` is true. The PR adds hash-based incremental indexing for session memory files, offset-aware chunk metadata/schema/SDK exports, and memory-core tests/benchmarks to avoid re-embedding unchanged session chunks.

## Best Possible Solution

Keep the incremental session-sync direction in memory-core, preserve full `--force` repair semantics, fix offset accounting against actual source positions, add the required changelog entry, then land after targeted memory-host/memory-core tests and the changed gate pass.

## Solution Assessment
