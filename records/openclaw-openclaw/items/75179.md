## Summary

Keep open: current main still uses the full session reindex path, and this PR is the active implementation candidate for the open performance issue, but the patch has two concrete merge blockers plus a dirty head state.

## What This Changes

The PR adds hash-based incremental indexing for memory-core session files, offset metadata for chunks, Plugin SDK storage exports, benchmark/regression tests, and a changelog entry.

## Best Possible Solution

Land a corrected session-only incremental writer that preserves unchanged embeddings, keeps offsets consistent through long-line splits and overlap, updates SDK baselines, and leaves MEMORY.md on the full-reindex path.

## Reproduction Assessment

Yes, at source level. On current main, dirty session files are rebuilt by buildSessionEntry and sent through indexFile/writeChunks, whose first write step clears existing rows before inserting regenerated chunks.

## Solution Assessment

No for this exact patch. The incremental direction is appropriate, but offset accounting, generated SDK API baseline state, and merge conflicts need to be repaired before this is the narrow maintainable solution.
