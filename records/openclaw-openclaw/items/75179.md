## Summary

Keep open. Current main still does not implement incremental session indexing, and this PR is a useful successor to the closed unmerged #59577 for the still-open #40919 report, but the latest head is not merge-ready because it still has a concrete offset-accounting bug plus missing public SDK baseline and changelog updates.

## What This Changes

The PR adds hash-based incremental indexing for session memory files, chunk offset metadata, offset-aware memory host SDK exports, SQLite schema changes, and memory-core benchmark/integration coverage.

## Best Possible Solution

Land incremental session sync in memory-core after the offset contract is correct for normal and long-line chunks, the public SDK API baseline and changelog are updated, force reindex semantics stay intact, and focused memory/plugin-SDK validation passes.

## Reproduction Assessment

Yes. Current main can be traced from a changed session file through syncSessionFiles to indexFile and writeChunks, which clears and reinserts rows; the PR offset bug can be reproduced by calling chunkMarkdownWithOffset on a single source line longer than the chunk limit and checking the startOffset after the first split chunk.

## Solution Assessment

No. The incremental-session direction is the right narrow solution for #40919, but the current PR should not merge until offset metadata, generated SDK baseline state, and changelog policy are fixed.
