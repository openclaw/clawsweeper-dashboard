The branch adds hash-based incremental indexing for memory-core session files, offset metadata in memory chunks, SDK exports, and session sync benchmark/regression tests.

## Best Possible Solution

Land a corrected incremental session sync that preserves unchanged embeddings, keeps offset metadata accurate through split lines and overlap, updates generated SDK contract state, and keeps MEMORY.md on the existing full-reindex path.

## Reproduction Assessment

Yes, at source level. On current main, dirty growing sessions are rebuilt by buildSessionEntry, routed to indexFile, and writeChunks clears rows before rewriting all chunks.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep offsets contiguous across split source lines:** `packages/memory-host-sdk/src/host/internal.ts:582`
  - body: For a long original line split into multiple segments, the first segment contributes a synthetic newline to currentRawChars. If that chunk flushes before the next segment, charOffset advances one character past the real source boundary, so persisted offsets for later chunks drift.
  - confidence: 0.9

  - confidence: 0.88
