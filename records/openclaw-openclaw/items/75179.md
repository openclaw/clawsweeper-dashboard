## Summary

Keep open. Current main still full-reindexes changed session files, and this PR is a useful implementation candidate, but it is not merge-ready because the new offset metadata is wrong for split long lines and the public SDK baseline plus required changelog entry are missing.

## What This Changes

The PR adds hash-based incremental indexing for session memory files, offset-aware chunk metadata and schema columns, public memory-host SDK exports, and memory-core session sync tests/benchmarks.

## Best Possible Solution

Keep the incremental session-sync design, but land it only after offset accounting follows actual source positions, the SDK API baseline is regenerated, the changelog is updated, and focused memory-core plus SDK validation passes on the exact branch head.

## Reproduction Assessment

Yes. Current-main behavior is reproducible from source by following changed session sync to indexFile and writeChunks, which clears all chunks for the file. The PR offset defect is reproducible with chunkMarkdownWithOffset on a single source line long enough to split into multiple segments.

## Solution Assessment

No. The hash-based incremental approach is the right maintainable direction for session growth, but this exact patch should not merge until the offset-aware chunking contract and required repo metadata are repaired.
