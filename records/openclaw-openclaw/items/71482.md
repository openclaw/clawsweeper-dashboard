## Summary

Keep open: the data-loss concern is source-backed, but this branch trades it for a cleanup regression and still lacks after-fix real behavior proof.

## What This Changes

The branch adds an early memory-core dreaming cleanup check that logs and skips orphan archival when `loadSessionStore()` returns an empty store.

## Best Possible Solution

Make the cleanup failure-aware: skip archival only when the session-store read is untrusted, preserve orphan cleanup for valid empty stores, and cover both paths with focused regression tests.

## Reproduction Assessment

Yes, at source level. A missing or invalid `sessions.json` loads as `{}`, and current cleanup can then treat stale dreaming transcripts as unreferenced; the PR regression is also source-reproducible with a valid `{}` store plus a stale orphan transcript.

## Solution Assessment

No. Preventing false archival is the right direction, but an unconditional empty-store skip is not the narrowest maintainable fix because valid empty stores should still run orphan scanning.
