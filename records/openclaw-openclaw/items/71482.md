## Summary

Keep open: the PR targets a plausible session data-loss path, but the unconditional empty-store skip conflates failed loads with valid empty stores, blocks legitimate orphan cleanup, lacks real behavior proof, and is currently conflicting with main.

## What This Changes

The branch adds a guard in memory-core dreaming cleanup that logs and skips orphan transcript archival for an agent whenever `loadSessionStore()` returns `{}`.

## Best Possible Solution

Make the scrubber failure-aware: skip archival only when the store read is untrusted, preserve orphan cleanup for trusted empty stores, and cover both paths with focused regression tests.

## Reproduction Assessment

Yes, at source level. Current main can load a failed or invalid session store as `{}` and then treat stale dreaming transcripts as unreferenced; the PR regression is also source-reproducible with a valid `{}` store plus a stale orphan transcript.

## Solution Assessment

No. Preventing false archival is the right direction, but checking only `Object.keys(store).length` is too broad because valid empty stores should still run orphan scanning.
