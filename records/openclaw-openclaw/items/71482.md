## Summary

The PR is a plausible fix for a real memory-core data-loss path, but it should stay open because the branch adds an over-broad empty-store skip that regresses orphan transcript cleanup for valid empty stores and still has no after-fix real behavior proof.

## What This Changes

The PR adds a memory-core dreaming scrubber guard that logs a warning and skips orphan archival whenever an agent's loaded session store is empty.

## Best Possible Solution

Make the cleanup failure-aware: skip archival only when the session-store read is untrusted, preserve orphan cleanup for a valid empty store, and cover both paths with focused tests.

## Reproduction Assessment

Yes, at source level. A missing or invalid `sessions.json` loads as `{}`, and current cleanup can then treat stale dreaming transcripts as unreferenced; the PR regression is also source-reproducible with a valid `{}` store plus a stale orphan transcript.

## Solution Assessment

No. Preventing false archival is the right direction, but an unconditional empty-store skip is not the narrowest maintainable fix because valid empty stores should still run orphan scanning.
