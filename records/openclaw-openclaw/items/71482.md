## Summary

Keep open: the PR targets a plausible session data-loss path, but the unconditional empty-store skip is not merge-ready because it also bypasses cleanup for valid empty stores and the external PR still lacks real behavior proof.

## What This Changes

The PR adds a guard in memory-core dreaming cleanup to skip orphan transcript archival whenever an agent's session store loads as an empty object.

## Best Possible Solution

Make the scrubber failure-aware: skip archival only when the session-store read is untrusted, preserve orphan cleanup for trusted empty stores, and cover both paths with focused regression tests.

## Reproduction Assessment

Yes, at source level. A missing or invalid `sessions.json` loads as `{}`, and current cleanup can treat stale dreaming transcripts as unreferenced; the PR regression is also source-reproducible with a valid `{}` store plus stale orphan transcript.

## Solution Assessment

No. Preventing false archival is the right direction, but an unconditional empty-store skip is not the narrowest maintainable fix because valid empty stores should still run orphan scanning.
