## Summary

Keep open: the PR targets a plausible memory-core data-loss path, but the current diff uses empty store as a failure signal even though `{}` is also a valid store state, so it would regress orphan cleanup; it also lacks after-fix real behavior proof.

## What This Changes

The branch adds an early memory-core dreaming cleanup guard that warns and skips orphan transcript archival whenever an agent session store loads as `{}`.

## Best Possible Solution

Make the cleanup failure-aware: skip archival only when the session-store read is untrusted, preserve orphan cleanup for a valid empty store, and cover corrupt/missing-store protection plus valid-empty-store archival with focused tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a missing or invalid `sessions.json` loads as `{}`, and the current scrubber can treat every stale dreaming transcript as unreferenced; the PR regression is also source-reproducible with a valid `{}` store plus a stale orphan transcript.

## Solution Assessment

No. Preventing false archival is the right direction, but `Object.keys(store).length === 0` is not the narrowest maintainable fix because valid empty stores should still run the orphan scan.
