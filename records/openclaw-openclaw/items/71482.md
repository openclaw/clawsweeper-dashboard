## Summary

Keep open: the reported false-archival path is source-plausible, but this PR's unconditional empty-store guard regresses valid empty-store cleanup and the external PR still lacks after-fix real behavior proof.

## What This Changes

The branch modifies `extensions/memory-core/src/dreaming-narrative.ts` to skip dreaming orphan archival whenever `loadSessionStore()` returns an empty object.

## Best Possible Solution

Make the cleanup failure-aware: skip archival only when the session-store read is untrusted, preserve orphan cleanup for trusted empty stores, and cover both paths with focused regression tests.

## Reproduction Assessment

Yes, at source level. A missing or invalid `sessions.json` loads as `{}`, and current cleanup can treat stale dreaming transcripts as unreferenced; the PR regression is also source-reproducible with a valid `{}` store plus stale orphan transcript.

## Solution Assessment

No. Preventing false archival is the right direction, but an unconditional empty-store skip is not the narrowest maintainable fix because valid empty stores should still run orphan scanning.
