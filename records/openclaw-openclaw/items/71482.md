## Summary

Keep open: the PR targets a real memory-core data-loss risk, but the proposed empty-store guard is over-broad, would skip cleanup for valid empty stores, and still lacks contributor-provided real behavior proof.

## What This Changes

The PR adds a memory-core dreaming cleanup guard that logs and skips orphan transcript archival whenever an agent's loaded session store is empty.

## Best Possible Solution

Make the cleanup failure-aware: skip archival only when the session-store read is untrusted, preserve orphan cleanup for valid empty stores, and cover both paths with focused tests.

## Reproduction Assessment

Yes, at source level. Current `loadSessionStore()` can return `{}` for missing or invalid stores, and the PR regression is also source-reproducible because a valid `{}` store plus a stale unreferenced dreaming transcript would now bypass orphan scanning entirely.

## Solution Assessment

No. Preventing false archival is the right direction, but an unconditional empty-store skip is not the narrowest maintainable fix because valid empty stores should still run orphan cleanup.
