## What This Changes

The PR adds an early empty-session-store guard in memory-core dreaming cleanup to avoid false orphan archival.

## Best Possible Solution

Make the scrubber failure-aware: skip archival only when the session-store read is untrusted, preserve orphan cleanup for trusted empty stores, and cover both paths with focused regression tests.

## Reproduction Assessment

Yes, source-level reproduction is clear. Current `loadSessionStore()` returns `{}` for missing or invalid stores, and current cleanup treats an empty referenced set as permission to archive stale unreferenced dreaming transcripts; the PR regression is also source-reproducible with a valid `{}` store plus stale orphan transcript.

## Solution Assessment

No. Preventing false archival is the right direction, but an unconditional empty-store skip is not the narrowest maintainable fix because valid empty stores should still run orphan scanning.
