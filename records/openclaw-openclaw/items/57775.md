## Summary

Keep this issue open: current main and v2026.6.10 still omit `system.execApprovals.get/set` from the gateway desktop node command policy, while https://github.com/openclaw/openclaw/pull/88296 remains the open closing PR for the exact fix.

## What This Changes

Advertise `system.execApprovals.get/set` for Windows headless node hosts so `nodes describe` and the Control UI recognize exec-approval-capable nodes.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/88296 if the security boundary remains intact, then close this issue as implemented while leaving broader Windows command-surface reports separate.

## Reproduction Assessment

Yes, source-reproducible. Current main's node host advertises `system.execApprovals.get/set`, but the gateway policy preserves approved desktop commands only through `DESKTOP_HOST_COMMANDS`, which omits those IDs.

## Solution Assessment

Yes, the linked PR is the best current fix path. It narrows the change to the gateway command policy and regression tests while preserving the dedicated `exec.approvals.node.*` RPC boundary.
