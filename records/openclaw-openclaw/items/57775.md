## What This Changes

Windows headless node hosts advertise working exec approval RPCs to the CLI, but `nodes describe` and Control UI do not show them as exec-approval-capable targets.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/88296 if the security boundary remains intact, then close this issue as implemented while leaving broader Windows command-surface reports separate.

## Reproduction Assessment

Yes, source-reproducible: current main's node host advertises `system.execApprovals.get/set`, while the gateway policy only preserves approved live desktop commands present in `DESKTOP_HOST_COMMANDS`, which omits those command IDs.

## Solution Assessment

Yes, the linked PR is the best current fix path because it narrows the change to the gateway command policy and regression tests while preserving the dedicated `exec.approvals.node.*` boundary.
