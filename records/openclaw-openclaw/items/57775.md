## Summary

Keep open: current main and v2026.6.6 still omit the exec-approval command IDs from the gateway desktop node command policy, and the linked draft fix at https://github.com/openclaw/openclaw/pull/88296 is open and unmerged.

## What This Changes

The issue reports that Windows headless node hosts can serve exec approvals through CLI RPCs but do not advertise `system.execApprovals.get/set` to `nodes describe` or Control UI.

## Best Possible Solution

## Reproduction Assessment

Yes. Source inspection shows the node host declares and implements `system.execApprovals.get/set`, while current main's gateway policy omits those IDs before describe/UI consume the advertised command list.

## Solution Assessment
