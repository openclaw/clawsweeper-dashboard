## Summary

Keep open. Current main still has the reported no-progress overflow retry shape: aggregate transcript overflow routes to `compact_only`, the run loop treats `compacted: true` as enough to retry, and regression coverage still expects three compactions and four failed prompt attempts before returning a context-overflow error.

## What This Changes

The issue asks OpenClaw to avoid retrying an unchanged oversized embedded/Paperclip prompt after auto-compaction and either reduce the prompt, rotate/summarize/truncate history, or fail once with a clear error.

## Best Possible Solution

Add a focused post-compaction progress gate for overflow recovery that stops retrying when compaction cannot prove the next prompt fits or materially shrank, while leaving broader session-retention and startup fallback policy to separate work.

## Reproduction Assessment

Yes, source-level rather than live. The issue includes concrete repeated diagnostics, and current main's overflow-loop harness already models repeated over-budget failures after `compact_only` compaction without needing the private session file.

## Solution Assessment

Yes. The narrow maintainable fix is a post-compaction budget or shrink check with a clear no-progress overflow error; the adjacent startup session-size guard is useful but does not solve this in-run retry loop.
