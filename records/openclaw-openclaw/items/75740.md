## Summary

Keep open. Current main still allows embedded-agent overflow recovery to retry after a successful compaction call without proving that the prompt now fits or materially shrank, which matches the reported repeated 873k-token precheck diagnostics.

## What This Changes

The issue asks for embedded-agent auto-compaction to reduce oversized accumulated session history or fail once with a clear no-progress error instead of retrying identical over-budget prompts.

## Best Possible Solution

Add a focused overflow-recovery progress gate that stops retrying after compaction unless the next prompt fits the budget or has materially shrunk, while leaving broader session retention policy separate.

## Reproduction Assessment

Yes. The issue includes concrete repeated diagnostics, and the existing overflow-loop harness can model repeated over-budget precheck failures after compact_only compaction without needing the private session file.

## Solution Assessment

No. Current main treats compacted=true as enough progress; the narrower maintainable fix is a post-compaction budget or shrink check with a clear no-progress overflow error.
