## Summary

Keep open. Current main still retries overflow recovery after a compaction response without proving that the selected prompt shrank or fits the model budget, which matches the reporter's repeated identical `compact_only` diagnostics.

## What This Changes

The issue reports that long-running embedded/Paperclip sessions can hit context overflow and auto-compaction retries without reducing the selected prompt window.

## Best Possible Solution

Add a focused overflow-recovery progress gate that fails with a clear actionable overflow error when compaction does not materially shrink or fit the selected prompt, while leaving broader transcript-retention policy for separate product work.

## Reproduction Assessment

Yes. The issue provides concrete repeated diagnostics, and the existing overflow-loop unit harness can model `compact_only` plus `compacted: true` with repeated over-budget precheck failures.

## Solution Assessment

No. Current main is not the best solution because it treats `compacted: true` as enough progress; the narrower maintainable fix is to require budget or shrink evidence before retrying.
