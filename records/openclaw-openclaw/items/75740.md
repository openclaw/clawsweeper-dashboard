## Summary

Keep open. Current main still has the reported no-progress overflow retry shape: aggregate transcript overflow routes to compaction, and the retry loop treats `compacted: true` as enough to retry without proving the next prompt fits or materially shrank.

## What This Changes

Make embedded/Paperclip agent overflow auto-compaction reduce oversized aggregate session history or fail once with an actionable error instead of retrying identical over-budget prompts.

## Best Possible Solution

Add a focused overflow-recovery progress gate: after compaction, retry only if the selected prompt is below budget or materially smaller; otherwise return a clear no-progress overflow error and leave broader retention policy separate.

## Reproduction Assessment

Yes, by source and harness inspection. Repeated precheck context-overflow results with `compactResult.compacted === true` still drive three compaction calls and four prompt attempts before failure.

## Solution Assessment

Yes. The narrow maintainable fix is a post-compaction budget or shrink check with a clear no-progress overflow error, not a new session-retention policy or config knob.
