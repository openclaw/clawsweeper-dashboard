## Summary

Keep open. The prior ClawSweeper concern still checks out on current `main`: aggregate prompt pressure can route to compaction, and the retry loop treats a successful compaction call as enough progress without proving the next selected prompt fits or materially shrank.

## What This Changes

The issue asks for embedded/Paperclip auto-compaction to reduce oversized aggregate session history, rotate or truncate it by policy, or fail once with a clear no-progress overflow error instead of retrying identical oversized prompts.

## Best Possible Solution

Add a focused overflow-recovery progress gate that stops retrying after compaction unless the retry prompt is below budget or has materially shrunk, while keeping broader session-retention policy separate.

## Reproduction Assessment

Yes. The issue includes concrete repeated diagnostics, and the existing overflow-loop unit harness can model repeated over-budget precheck failures after `compact_only` compaction without needing the reporter's private session file.

## Solution Assessment

No. Current `main` treats `compacted: true` as sufficient progress; the safer maintainable fix is a post-compaction budget or shrink check with a clear no-progress overflow error.
