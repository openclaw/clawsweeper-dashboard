## Repair Work Prompt

Fix the embedded runner overflow-compaction no-progress loop reported in https://github.com/openclaw/openclaw/issues/75740. On current main, aggregate precheck overflow can route to `compact_only`; after `contextEngine.compact()` reports `compacted`, `runEmbeddedPiAgent` retries without proving the next prompt fits or materially shrank. Add a focused progress gate for overflow-recovery compaction only: after compaction, require evidence that the retry prompt is under budget or that the prompt estimate/token count materially decreased; otherwise stop retrying and return a clear `context_overflow` error explaining that compaction made no progress. Preserve legitimate multi-step retries when each compaction makes measurable progress, preserve tool-result truncation behavior for `compact_then_truncate`, and do not add a config knob or broad session-retention/archiving policy. Add or update regression coverage in the overflow-compaction loop/preemptive-compaction tests. Add a changelog entry because this is user-visible reliability behavior.
