- **[P1] Add the cap to the runtime schema:** `src/config/types.agent-defaults.ts:266-272`
  - body: Adding the TypeScript field alone does not make the example config loadable: current `HeartbeatSchema` is strict and this PR does not add `maxCostPerRun`, so runtime config validation can reject the new key before the runner sees it.
  - confidence: 0.95

  - confidence: 0.94
- **[P1] Enforce the cap at the shared embedded-run boundary:** `src/infra/heartbeat-runner.ts:719-724`
  - body: The PR says the cap covers heartbeat, cron, and future triggers, but the check is inserted inside `runHeartbeatOnce`; isolated cron agent turns dispatch through the cron runner and remain uncapped. This would close the cost-cap request while leaving a central trigger class unprotected.
  - confidence: 0.9
- **[P1] Price the model that will actually run:** `src/infra/heartbeat-runner.ts:724-728`
  - body: When `heartbeat.model` is unset, the estimate uses a local default-model helper, but the real run can use per-agent defaults, channel/session overrides, and fallback-selected models resolved later. That mismatch can under-price an expensive run or over-skip a cheap one.
  - confidence: 0.88
