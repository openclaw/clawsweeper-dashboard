Full review comments:

- **[P2] Add maxCostPerRun to the runtime schema:** `src/config/types.agent-defaults.ts:266-272`
  - body: Adding only the TypeScript field leaves the public config invalid. `HeartbeatSchema` is strict and the schema/inventory surfaces do not include `maxCostPerRun`, so normal config validation can reject the example key instead of honoring it.
  - confidence: 0.94
- **[P1] Estimate the full prompt context before skipping:** `src/infra/heartbeat-runner.ts:724-728`
  - body: The guard prices only `ctx.Body`, but non-isolated heartbeat runs can include bootstrap/system context, tool context, and persisted session messages at the LLM boundary. A long transcript can pass this cap and still dispatch the expensive call the feature is meant to prevent.
  - confidence: 0.94
- **[P2] Price the effective model that will dispatch:** `src/infra/heartbeat-runner.ts:724-728`
  - body: When `heartbeat.model` is unset, this precheck uses `resolveDefaultModelId(cfg)` instead of the per-agent/default/session override path used by `getReplyFromConfig`. Multi-agent setups can therefore allow or skip a run based on a different model than the one actually called.
  - confidence: 0.91
- **[P2] Move the cap to the shared embedded-run boundary:** `src/infra/heartbeat-runner.ts:719-724`
  - body: The PR says the dispatch-level guard covers heartbeat, cron, and future triggers, but the new check is inside `runHeartbeatOnce`. Isolated cron turns still call `runEmbeddedPiAgent` through their own executor with no cost-cap path.
  - confidence: 0.89
- **[P2] Keep core tests on public plugin seams:** `src/infra/heartbeat-runner.cost-cap.test.ts:2-5`
  - body: This new core infra test deep-imports Telegram and WhatsApp implementation files from `extensions/*/src`. That crosses the repo's core/plugin test boundary; use public plugin fixtures, SDK facades, or a generic registry seam instead.
  - confidence: 0.84
- **[P3] Cover legacy Claude pricing before the fallback:** `src/infra/heartbeat-runner.ts:1219-1221`
  - body: The hardcoded fallback table has `claude-opus-4`, `claude-sonnet-4`, and `claude-haiku`, but common `claude-3-*` IDs do not match those prefixes. When catalog pricing is unavailable, those models fall to the expensive default and can be skipped far earlier than intended.
  - confidence: 0.78
