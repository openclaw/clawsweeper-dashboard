## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Add the cap to the runtime schema:** `src/config/types.agent-defaults.ts:266-272`
  - body: Adding the TypeScript field alone does not make the example config loadable: `HeartbeatSchema` is strict and the PR does not add `maxCostPerRun` there, so runtime config validation can reject the new key before the runner sees it.
  - confidence: 0.95
- **[P1] Estimate the full prompt context before skipping:** `src/infra/heartbeat-runner.ts:724-728`
  - body: The guard prices only `ctx.Body`, but default heartbeat turns also carry system/bootstrap/tool context and the existing session transcript into the embedded run. A large shared session can pass this check and still dispatch the expensive call the cap is meant to prevent.
  - confidence: 0.94
- **[P2] Price the model that will actually dispatch:** `src/infra/heartbeat-runner.ts:724-728`
  - body: When `heartbeat.model` is unset, this estimates against `resolveDefaultModelId(cfg)`, while `getReplyFromConfig` later resolves the agent-specific default and session state. Multi-agent or overridden sessions can be allowed or skipped against a different model than the one actually called.
  - confidence: 0.91
- **[P2] Move the cap to the embedded-run boundary:** `src/infra/heartbeat-runner.ts:719-724`
  - body: The PR says the cap covers heartbeat, cron, and future triggers, but the new check only runs inside `runHeartbeatOnce`. Isolated cron jobs still dispatch through `runEmbeddedPiAgent` with no cost-cap path.
  - confidence: 0.89
- **[P2] Keep core tests on public plugin seams:** `src/infra/heartbeat-runner.cost-cap.test.ts:2-5`
  - body: This core infra test imports Telegram and WhatsApp private implementation/runtime files from `extensions/*/src`. That crosses the plugin boundary; use public plugin fixtures, SDK facades, or a generic registry seam instead.
  - confidence: 0.84
- **[P3] Cover legacy Claude pricing before the fallback:** `src/infra/heartbeat-runner.ts:1219-1221`
  - body: The fallback table covers Claude 4 and `claude-haiku`, but common `claude-3-*` IDs do not match those prefixes. Without catalog pricing, those models fall to the expensive default and can be skipped far earlier than intended.
  - confidence: 0.78
