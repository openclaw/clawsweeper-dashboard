The branch adds an OpenAI Chat Completions payload role guard, updates blank-prompt replay classification, and adds focused OpenAI transport, prompt-helper, and compaction regression tests.

## Best Possible Solution

Have maintainers choose the compatibility boundary, then land this focused guard and tests if the global invariant is accepted or scope the guard through provider compatibility if preserving system/tool-only sends matters.

## Reproduction Assessment

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers decide the compatibility policy; the stream-time guard is narrow and well tested, but global fail-closed behavior may be too broad if any provider-compatible path legitimately accepts system/tool-only sends.
