## Summary

Keep this PR open: it addresses a real OpenAI-compatible compaction failure path, but the current patch is not merge-ready because it changes every OpenAI-compatible provider to fail closed on system/tool-only payloads without a provider-scoped compatibility decision.

## What This Changes

The PR adds a shared OpenAI-compatible conversation-turn detector, rejects chat-completions payloads without a non-empty user or assistant turn, updates blank-prompt classification, and adds focused regression tests.

## Best Possible Solution

Keep the malformed-payload protection, but scope it through provider compatibility policy or explicit maintainer approval so valid existing OpenAI-compatible setups are not blocked unexpectedly.

## Reproduction Assessment

Yes for the malformed transport shape: current main can build and send an OpenAI-compatible request with only system/tool payload turns. The exact reporter Qwen compaction capture is not reproduced in this review.

## Solution Assessment

No, not as currently scoped. The guard is a reasonable backstop, but applying it to every OpenAI-compatible provider needs provider-scoped compatibility policy or explicit maintainer acceptance.
