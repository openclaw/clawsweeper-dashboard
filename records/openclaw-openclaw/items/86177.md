## Summary

Keep open: this is a focused bug-fix PR with sufficient loopback runtime proof and no blocking code-review finding, but it still needs maintainer approval because the guard changes all OpenAI-compatible chat transports to fail closed locally for system/tool-only payloads.

## What This Changes

Adds an OpenAI-compatible chat payload guard, adjusts blank-prompt replay classification to require user/assistant history, and adds transport, prompt-helper, and compaction regressions.

## Best Possible Solution

Land this after maintainer confirmation that the user/assistant payload invariant should be global, or scope the guard to strict/Qwen-style OpenAI-compatible providers with the same regression coverage.

## Reproduction Assessment

Not fully: current main clearly has no final user/assistant role guard before sending OpenAI-compatible chat payloads, and the linked report gives a concrete Qwen reproduction pattern. I did not run a current-main live Qwen failure; the PR proof is after-fix loopback proof.

## Solution Assessment

Unclear pending maintainer policy: the patch is a narrow maintainable guard for the reported malformed payload, but the global fail-closed scope needs approval. A provider-compat-scoped guard is safer if permissive custom endpoints must keep working.
