## Summary

The PR is a focused bugfix with strong after-fix proof for the Qwen/OpenAI-compatible failure, but the new guard is unconditional across all `openai-completions` routes and can reject payload shapes that current OpenClaw and the OpenAI SDK contract still allow. Keep it open for maintainer compatibility review or a provider/compat-scoped rework.

## What This Changes

Adds a stream-time OpenAI chat-completions guard for payloads without user/assistant turns, changes blank prompt replay classification, and adds regression tests for the guard and compaction branch recovery.

## Best Possible Solution

Preserve the useful Qwen protection, but make the invariant provider/compat-scoped or explicitly maintainer-approved with upgrade proof for native OpenAI and plugin-runtime system-only call paths.

No independent high-confidence current-main live reproduction was run in this read-only pass. Source shows current main can send system-only OpenAI-completions payloads, and the PR body supplies loopback runtime proof that the proposed guard blocks that shape before network send.

## Solution Assessment
