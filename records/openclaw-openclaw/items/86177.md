## Summary

Keep this PR open for maintainer review. The patch is focused and proof-backed with no blocking code findings, but it deliberately makes all `openai-completions` chat payloads fail locally when no user or assistant turn remains, which is a compatibility-sensitive provider policy decision.

## What This Changes

Adds a pre-send OpenAI Chat Completions guard for payloads with no user/assistant messages, adjusts blank-prompt skip classification, and adds focused transport and compaction regressions.

## Best Possible Solution

Land the guard only after maintainers confirm the global OpenAI-compatible payload invariant, or rework it into a provider/compat-scoped guard with the same regression coverage.

## Reproduction Assessment

## Solution Assessment

Unclear pending maintainer policy. The implementation is narrow for the reported Qwen failure, but the best permanent shape depends on whether OpenClaw wants this as a global `openai-completions` invariant or a provider compatibility rule.
