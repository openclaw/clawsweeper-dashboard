## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep review-triggered approvals token-capable:** `.github/workflows/plugin-sdk-impact-gate.yml:6-7`
  - body: This trigger is meant to re-evaluate the gate when a maintainer submits or dismisses a review, but approval only passes when `GH_APP_TOKEN` is available. The PR body says fork `pull_request_review` runs cannot mint that app token, and the added test codifies that approval fails without it, so an external Plugin SDK PR can remain blocked immediately after approval. Use a token-capable trusted path or remove this trigger until the head check can be updated safely.
  - confidence: 0.9
- **[P1] Attach comment reruns to the PR head check:** `.github/workflows/plugin-sdk-impact-gate.yml:8-9`
  - body: `issue_comment` events can fetch the PR and run the policy script, but the added script never creates or updates a check/status for `pullRequest.head.sha`. A maintainer or ClawSweeper comment can therefore re-evaluate policy while leaving the required gate missing or stale on the head SHA. Publish an explicit head result from this path or drop the comment trigger from the required gate.
  - confidence: 0.88
- **[P2] Scope comment concurrency by PR number:** `.github/workflows/plugin-sdk-impact-gate.yml:14-16`
  - body: For `issue_comment` events this expression has no `github.event.pull_request.number`, so it falls back to `github.ref`, which is the default branch for all PR comments. Comment-triggered gate runs on unrelated PRs can cancel each other, making the rerun path flaky even after the head-check issue is fixed. Include `github.event.issue.number` for comment events.
  - confidence: 0.82
