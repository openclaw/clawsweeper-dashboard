- **[P1] Keep review-triggered approvals token-capable:** `.github/workflows/plugin-sdk-impact-gate.yml:6-7`
  - body: This trigger is meant to re-evaluate after review submission or dismissal, but approval only passes when `GH_APP_TOKEN` is present and the PR body says fork `pull_request_review` runs cannot mint that app token. An external Plugin SDK PR needing approval can therefore still fail immediately after approval; use a trusted token-capable path or remove this trigger until it can update the head check safely.
  - confidence: 0.9
- **[P1] Attach comment reruns to the PR head check:** `.github/workflows/plugin-sdk-impact-gate.yml:8-9`
  - body: GitHub documents `issue_comment` as running on the default branch context, and this workflow only runs the policy script; it never creates or updates a check/status for `pullRequest.head.sha`. A maintainer or ClawSweeper comment can re-evaluate the policy while the required Plugin SDK impact gate remains missing or stale on the PR head. ([docs.github.com](https://docs.github.com/actions/using-workflows/events-that-trigger-workflows?utm_source=openai))
  - confidence: 0.88
- **[P2] Scope comment concurrency by PR number:** `.github/workflows/plugin-sdk-impact-gate.yml:14-16`
  - body: For `issue_comment` events, `github.event.pull_request.number` is absent and `github.ref` points at the default branch, so unrelated PR comments share one concurrency group and can cancel each other. Include `github.event.issue.number` for comment events before relying on this rerun path.
