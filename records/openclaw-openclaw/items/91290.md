- **[P1] Keep review-triggered approvals token-capable:** `.github/workflows/plugin-sdk-impact-gate.yml:6-7`

  - confidence: 0.9
- **[P1] Attach comment reruns to the PR head check:** `.github/workflows/plugin-sdk-impact-gate.yml:8-9`
  - body: `issue_comment` reruns can fetch the PR and evaluate policy, but this workflow never creates or updates a check/status for `pullRequest.head.sha`. A maintainer or ClawSweeper comment can therefore re-evaluate the policy while leaving the required Plugin SDK impact gate missing or stale on the head SHA.
  - confidence: 0.88
- **[P2] Scope comment concurrency by PR number:** `.github/workflows/plugin-sdk-impact-gate.yml:15`
  - body: For `issue_comment` events this expression has no `github.event.pull_request.number`, so it falls back to the default-branch ref for all PR comments. Comment-triggered gate runs on unrelated PRs can cancel each other, making the rerun path flaky even after the head-check issue is fixed.
  - confidence: 0.82
