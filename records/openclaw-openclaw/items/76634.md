## Summary

Keep this PR open because current main still sends raw Mattermost `replyToId` values as `root_id`, but the patch is not merge-ready: it lacks redacted real Mattermost proof and the catch-all lookup fallback can turn intended threaded replies into top-level posts.

## What This Changes

The PR adds a Mattermost send-time post lookup that resolves `replyToId` to the thread root before posting, plus mocked send tests for child, root, cross-channel, and lookup-failure cases.

## Best Possible Solution

Resolve Mattermost reply roots immediately before send, but only drop or retarget the thread when the reference is proven invalid or cross-channel, and require redacted live Mattermost proof before merge.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction shape: current main passes a child `replyToId` directly as Mattermost `root_id`, and Mattermost rejects child posts as roots. I did not run a live Mattermost instance in this read-only review.

## Solution Assessment

No, not as-is. Send-time root resolution is the right boundary, but the catch-all lookup fallback should be narrowed or made explicit before this is the best fix.
