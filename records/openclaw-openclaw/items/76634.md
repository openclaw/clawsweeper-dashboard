## Summary

Keep open: the PR addresses a real Mattermost thread-root gap that current main does not fully solve, but it is not merge-ready because indeterminate root lookup failures become top-level sends and no real Mattermost proof is attached.

## What This Changes

The PR changes Mattermost sends to fetch a referenced post before posting, use that post's thread root as `root_id`, drop cross-channel or unresolved reply targets, and adds send tests for those cases.

## Best Possible Solution

Keep the Mattermost-owned root-resolution fix direction, but preserve valid thread targets on inconclusive lookup failures and require redacted live Mattermost proof before merge.

## Reproduction Assessment

Yes for source-level review: current main passes `replyToId` directly as Mattermost `root_id`, while Mattermost rejects nested roots in create-post validation. I did not run a live Mattermost workspace.

## Solution Assessment

No. Resolving child posts to their thread root is the right layer, but dropping the reply target on every lookup failure is not the safest delivery behavior.
