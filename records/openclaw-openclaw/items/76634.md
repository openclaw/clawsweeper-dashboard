## Summary

Keep open. The PR addresses a real current Mattermost threading bug: current main can pass a child reply post as Mattermost `root_id`, and Mattermost rejects non-root posts as thread roots. The implementation is narrow and covered by focused send tests, but the user-facing fix still needs the required changelog entry before merge.

## What This Changes

The PR changes Mattermost outbound sends to resolve a reply target post to its real thread root, drop unresolved or cross-channel reply targets, and add Mattermost send tests.

## Best Possible Solution

Land the Mattermost root-resolution change after adding a single-line Unreleased/Fixes changelog entry and keeping the focused send regression tests green.
