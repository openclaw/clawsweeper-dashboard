## Summary

Keep open: current main and the latest release still pass a raw Mattermost `replyToId` into `root_id` at the send sink, and no merged or proof-positive replacement fully covers this focused root-resolution change. I found no definite code defect in the patch, but real Mattermost proof and maintainer acceptance of the fallback delivery behavior are still required before merge.

## What This Changes

The PR adds a Mattermost send-time lookup that resolves `replyToId` to the thread root before posting, with focused send tests for child, root, cross-channel, and lookup-failure cases.

## Best Possible Solution

Land the narrow send-time root resolution after redacted live Mattermost proof shows child replies thread under the root and fallback cases do not misroute messages.

## Reproduction Assessment

Yes, from source inspection: current main can pass a child post ID as Mattermost `root_id`, and upstream Mattermost rejects child posts as roots. I did not run a live Mattermost instance in this read-only review.

## Solution Assessment

Yes, resolving the referenced post to its actual root immediately before the Mattermost send is the narrowest sink-level fix for direct and sibling callers. It still needs real transport proof because it changes visible message delivery behavior.
