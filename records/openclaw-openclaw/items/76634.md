## Summary

Keep open: the PR is a focused and still-useful Mattermost outbound threading fix, but it is not merge-ready because the returned receipt can still report the original reply target after the send uses a resolved or omitted root, and the external PR still lacks real Mattermost behavior proof.

## What This Changes

The PR changes Mattermost outbound sends to resolve a referenced post into the actual thread root before calling the post API and adds focused send tests for child, top-level, cross-channel, and lookup-failure cases.

## Best Possible Solution

Keep the send-sink root resolution, align receipt metadata with the resolved root or top-level fallback, and require redacted real Mattermost proof before merge.

## Reproduction Assessment

Yes, source-level. Current main and `v2026.6.8` pass raw `opts.replyToId` into Mattermost `root_id`, while the Mattermost API and thread model require a comment root rather than an arbitrary child post id.

## Solution Assessment

Partially. Resolving the referenced post at the Mattermost send sink is the right owner-boundary repair, but the patch should also align receipt metadata with the delivered root or fallback and needs real Mattermost proof.
