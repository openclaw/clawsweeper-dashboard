The issue asks for Control UI Markdown Preview to stop turning local absolute artifact paths into misleading Gateway-origin links.

## Best Possible Solution

Make obvious host-local filesystem Markdown hrefs non-navigable or route only already-allowlisted artifacts through the existing authenticated assistant-media path.

## Solution Assessment

Yes. The narrow maintainable fix is to make local filesystem Markdown links inert or intentionally map already-allowed artifacts to assistant-media, not to add broad local file serving.
