## Summary

Keep open: current main and the latest shipped release still let an explicit `/Users/...` Markdown href survive as a same-origin Gateway link in Control UI Markdown Preview, while the safe assistant-media path is separate and allowlist-protected.

## What This Changes

The issue requests reliable Control UI handling for generated local artifact links in Markdown Preview instead of clickable same-origin `/Users/...` Gateway routes.

## Best Possible Solution

Make obvious host-local filesystem Markdown hrefs non-navigable or explicitly hinted, and only use assistant-media URLs for sources already covered by the existing authenticated allowlist path.

## Reproduction Assessment

Yes. Source inspection shows Markdown Preview calls `toSanitizedMarkdownHtml`, and the sanitizer resolves `/Users/...` against the Gateway origin as an allowed HTTPS link; I did not run a live browser click in this read-only pass.

## Solution Assessment

Yes. The narrow maintainable fix is to make local filesystem Markdown hrefs inert or route only already-allowlisted artifact references through the existing assistant-media path, not to add broad local-file serving.
