## Summary

Keep open: current main and the latest release still let Markdown Preview treat an explicit `/Users/...` link as a same-origin Gateway URL, and the linked open fix PR should remain the closure path until it merges.

## What This Changes

The issue asks for Control UI Markdown Preview to stop rendering host-local absolute artifact paths as misleading clickable Gateway links.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84505, or an equivalent narrow sanitizer fix, while keeping local file downloads behind the existing assistant-media authorization and local-root checks.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Markdown Preview calls `toSanitizedMarkdownHtml`, and the sanitizer resolves `/Users/...` against the Gateway origin while allowing the resulting HTTPS URL.

## Solution Assessment

Yes. The narrow maintainable fix is to make obvious host-local filesystem hrefs non-navigable in the Markdown rendering layer, not to add broad local file serving.
