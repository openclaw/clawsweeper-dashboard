## Summary

Keep open: current main still lets Markdown Preview preserve `/Users/...` style Markdown links as same-origin Gateway anchors, and the linked open PR is the active fix path rather than a reason to close this issue before merge.

## What This Changes

Control UI Markdown Preview should stop treating local absolute filesystem Markdown hrefs as same-origin Gateway routes and handle generated artifacts intentionally.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84505, or an equivalent narrow sanitizer fix, while leaving broader artifact download/open support behind the existing assistant-media boundary.

## Reproduction Assessment

Yes. Source inspection and a direct URL parse show that `[example.docx](/Users/...)` is resolved against the Gateway origin and kept as an allowed HTTPS href; I did not run a live browser click in this read-only review.

## Solution Assessment

Yes. The narrow maintainable solution is to make recognizable local absolute Markdown hrefs non-navigable, or route only already-allowlisted artifacts through assistant-media, rather than adding broad local file serving.
