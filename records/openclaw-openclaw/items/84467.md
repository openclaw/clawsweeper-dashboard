## Summary

## What This Changes

The issue asks Control UI Markdown Preview to stop rewriting host-local artifact links into misleading same-origin Gateway routes.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/84505 with a narrow Markdown rendering fix that makes obvious host-local filesystem hrefs inert, while leaving authenticated assistant-media as the only local-file serving path.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: render `[example.docx](/Users/...)` in Markdown Preview and the sanitizer resolves that href against the Gateway origin as an allowed HTTPS link.

## Solution Assessment

Yes. The maintainable fix is to make local filesystem Markdown hrefs non-navigable, or intentionally map only already-allowlisted artifacts through assistant-media, rather than broadening local file serving.
