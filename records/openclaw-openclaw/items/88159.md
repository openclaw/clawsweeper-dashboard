## What This Changes

The PR removes the `openclaw logs --follow` journal latch in `src/cli/logs-cli.ts` and adds regression tests for retrying `logs.tail` after journal fallback while preserving the journal cursor.

## Best Possible Solution

Retry Gateway logs after temporary journal fallback, but emit source-transition metadata/notices whenever output moves between journal and Gateway/file logs so operators and JSON consumers can trust provenance.

## Reproduction Assessment

Yes, source-level: current main sets `forceJournal` after a journal payload and the next `fetchLogs` reads journal before attempting `logs.tail`. I did not run a live Linux Gateway restart/systemd session in this read-only review.

## Solution Assessment

No, not yet: removing the permanent journal latch is the right direction, but the best fix also reports source transitions and includes live proof or an explicit maintainer proof override.
