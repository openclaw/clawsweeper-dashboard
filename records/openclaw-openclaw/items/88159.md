## Summary

Keep open for maintainer review. Current main can latch `openclaw logs --follow` onto journal fallback after a transient local Gateway outage, and this PR is a focused fix with sufficient terminal proof, but it intentionally changes the observable `--follow --json` metadata stream and needs maintainer acceptance before merge.

## What This Changes

The PR removes the logs-follow journal latch, preserves separate Gateway and journal cursors, emits source-transition metadata, adds logs CLI regression tests, and documents JSON follow source transitions.

## Best Possible Solution

Merge the latch-removal fix once maintainers accept the multi-source JSON follow stream; otherwise keep the retry behavior and adjust JSON output to the accepted compatibility shape.

## Reproduction Assessment

Yes at source level. Current main checks `cursors.forceJournal` before `logs.tail` and sets `forceJournal` after journal payloads, so recovered Gateway RPC is not probed on the next follow loop.

## Solution Assessment

Yes for the implementation shape: removing the CLI follow-loop latch is narrower than changing Gateway logging or the journal reader. Maintainers still need to accept the documented multi-source JSON metadata compatibility change.
