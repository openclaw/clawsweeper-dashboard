## Summary

Keep open. Current main and latest release still normalize malformed Anthropic SSE into a stable sentinel, but that sentinel is not classified as a retryable failover signal, so the reported `failoverReason=null` recovery failure remains actionable.

## What This Changes

The issue reports recurring Anthropic streaming JSON parse failures that surface in Telegram and do not consistently trigger model failover.

## Best Possible Solution

Classify the existing malformed-streaming-fragment sentinel as a retryable transport-style failover signal, with regression coverage, while preserving the current narrow handling of arbitrary raw JSON.parse text.

## Reproduction Assessment

Yes, at source level: current tests synthesize malformed Anthropic SSE into the stable sentinel, and current source inspection shows that sentinel still falls through `classifyFailoverReason` without a failover reason. I did not run a live Anthropic/Telegram reproduction in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to classify the provider-boundary malformed-streaming-fragment sentinel for retry/failover, not to add broad raw JSON.parse string matching.
