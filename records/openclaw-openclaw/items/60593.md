## Summary

Keep open. Current main still only partially mitigates the report: malformed Anthropic SSE JSON is converted to a stable sentinel and sanitized for users, but the failover classifier still does not classify that sentinel, leaving the reported non-timeout recovery path incomplete.

## What This Changes

The issue asks OpenClaw to retry or fail over consistently on Anthropic streaming JSON parse failures and avoid exposing raw JSON.parse errors to Telegram users.

## Best Possible Solution

Add a narrow failover classification for `OpenClaw transport error: malformed_streaming_fragment` as a retryable transport-style failure, keep the existing friendly user-facing copy, and lock it with focused classifier and runner policy tests.

## Reproduction Assessment

Yes. Current test fixtures can synthesize malformed Anthropic SSE data that produces the sentinel, and source inspection shows that sentinel currently falls through the failover classifier to null.

## Solution Assessment

Yes. The narrow maintainable fix is to classify the existing transport sentinel for failover; broad regex handling for all JSON.parse text would risk hiding valid provider diagnostics.
