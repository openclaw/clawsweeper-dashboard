Keep open. Current main and the latest release still normalize malformed Anthropic SSE JSON into a stable transport sentinel, but that sentinel is not classified as a retryable failover signal, so the reported failover recovery gap remains.

## What This Changes

The issue requests consistent retry/failover for Anthropic streaming JSON parse failures and suppression of raw parser errors from Telegram users.

## Best Possible Solution

Classify the normalized malformed streaming fragment sentinel as a retryable transport-style failover signal, with focused classifier and embedded-runner coverage, while keeping arbitrary raw JSON.parse diagnostics non-retryable.

## Solution Assessment

Yes. The narrow maintainable fix is to classify the normalized provider-boundary sentinel for retry/failover, not to broadly treat every raw JSON.parse diagnostic as retryable.
