## Summary

Keep open. Current main partially mitigates the report by converting malformed Anthropic SSE JSON into a stable sentinel and sanitizing that sentinel for users, but the failover classifier still does not classify that sentinel, so the non-timeout recovery path remains incomplete.

## What This Changes

The issue asks OpenClaw to retry or fail over on Anthropic streaming JSON parse failures and prevent raw JSON.parse errors from reaching Telegram users.

## Best Possible Solution

Add a narrow failover classification for `OpenClaw transport error: malformed_streaming_fragment` as a retryable transport-style failure, keep the existing friendly user-facing copy, and lock the behavior with focused classifier and runner policy tests.

## Reproduction Assessment

Yes. The current Anthropic transport test can produce the malformed-streaming-fragment sentinel from invalid SSE JSON, and source inspection shows that sentinel currently falls through the failover classifier to null.

## Solution Assessment

No. Current main is only a partial fix; the maintainable path is to classify the existing transport sentinel for failover rather than adding broad regex handling for all JSON.parse messages.
