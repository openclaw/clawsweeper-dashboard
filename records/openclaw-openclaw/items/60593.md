## Summary

Keep open. Current main still normalizes malformed Anthropic SSE JSON into the stable malformed-streaming-fragment sentinel and sanitizes that text for users, but the failover classifier does not treat that sentinel as retryable, so the reported `failoverReason=null` recovery gap remains actionable.

## What This Changes

The issue asks OpenClaw to retry or fail over consistently on Anthropic streaming JSON parse failures and avoid exposing raw parse errors to Telegram users.

## Best Possible Solution

Classify the normalized malformed-streaming-fragment sentinel as a retryable transport-style failover signal with regression coverage, while leaving arbitrary raw JSON.parse diagnostics unclassified unless a provider boundary normalizes them first.

## Reproduction Assessment

Yes, at source level: current tests synthesize malformed Anthropic SSE into the stable sentinel, and current classifier inspection shows that sentinel still has no failover reason. I did not run a live Anthropic/Telegram reproduction in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to classify the normalized provider-boundary sentinel for retry/failover, not to broadly treat every raw JSON.parse diagnostic as retryable.
