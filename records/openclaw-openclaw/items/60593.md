## Summary

Keep open. Current main and the latest release still normalize malformed Anthropic SSE JSON into a stable transport sentinel, but that sentinel is not classified as a retryable failover signal, so the reported failover recovery gap remains.

## What This Changes

The issue requests consistent retry/failover for recurring Anthropic malformed streaming JSON and no raw JSON.parse errors in Telegram replies.

## Best Possible Solution

Classify the normalized malformed-streaming-fragment sentinel as a retryable transport-style failover signal, with focused classifier and embedded-runner regression coverage, while keeping broad raw JSON.parse diagnostics excluded.
