## Summary

Keep open. Current main partially mitigates the report by converting malformed Anthropic SSE JSON into a stable sentinel and sanitizing that sentinel for users, but the failover classifier still does not classify the sentinel, so the reported non-timeout recovery path remains incomplete.

## What This Changes

The issue requests consistent retry/failover and user-safe handling for recurring Anthropic streaming JSON parse failures in Telegram.

## Best Possible Solution

Classify the existing malformed-streaming-fragment sentinel as a retryable transport-style failover signal while keeping broad raw JSON.parse diagnostics out of failover unless they are normalized upstream.

## Reproduction Assessment

Yes, at source level. Current fixtures synthesize malformed Anthropic SSE data into the stable sentinel, and source inspection shows that sentinel still falls through `classifyFailoverReason` to null.

## Solution Assessment

Yes. The narrow maintainable fix is to classify the existing transport sentinel for failover; broad regex handling for every raw JSON.parse message would risk hiding provider request-validation diagnostics.
