## Summary

Keep open. Current main partially mitigates the report by normalizing malformed Anthropic SSE JSON into a stable sentinel and sanitizing that sentinel for users, but that sentinel still is not classified as a retryable failover signal, so the reported non-timeout recovery gap remains.

## What This Changes

The issue reports recurring Anthropic streaming JSON parse failures that leak raw errors to Telegram and often do not trigger model failover.

## Best Possible Solution

Classify the existing malformed-streaming-fragment sentinel as a retryable transport-style failover signal while keeping arbitrary raw JSON.parse diagnostics out of failover unless they are normalized at the provider boundary.

## Reproduction Assessment

Yes, at source level. Current fixtures synthesize malformed Anthropic SSE data into the stable sentinel, and current source inspection shows that sentinel still falls through `classifyFailoverReason` without a failover reason.

## Solution Assessment

Yes. The narrow maintainable fix is to classify the existing transport sentinel for retry/failover; broad regex handling for every raw JSON.parse message would risk hiding provider request-validation diagnostics.
