## Summary

Keep this open: the report is recent and concrete, and current main still uses the same compositor-heavy macOS WebChat rendering path with no source, test, docs, changelog, or release evidence of a WindowServer-specific fix.

## What This Changes

The issue reports WindowServer CPU spiking when the macOS WebChat window is visible on an Intel Retina iMac, with CPU dropping after hiding OpenClaw.

## Best Possible Solution

Profile current main on affected Intel Retina hardware, A/B test the transparent window/material/shadow/background/animation pieces, and ship the narrowest measured low-compositor fallback.

## Reproduction Assessment

Unclear. The issue gives concrete visible-vs-hidden steps and CPU measurements, but this review did not reproduce the failure on affected Intel Retina macOS hardware against current main.

## Solution Assessment

Not applicable; there is no proposed patch. A source-only change is not the best path until profiling identifies which rendering feature drives WindowServer CPU and what visual fallback is acceptable.
