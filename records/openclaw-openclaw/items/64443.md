## Summary

Keep this open. The report is recent, concrete, and still maps to the first-party macOS native WebChat rendering path; current main and the latest release continue to use transparent/non-opaque windows, active visual-effect material, material/gradient backgrounds, panel shadow, and visible scroll animations with no source, docs, tests, changelog, or release evidence of a measured WindowServer CPU fix.

## What This Changes

Investigate and reduce very high WindowServer CPU when the macOS native OpenClaw chat interface is visible on an Intel Retina iMac.

## Best Possible Solution

Treat this as a macOS WebChat rendering-performance investigation: profile current main on affected Intel Retina hardware, A/B test opaque windowing, removing NSVisualEffectView/materials, disabling panel shadow, and suppressing scroll/typing animations, then add the narrowest measured low-compositor fallback if the regression is confirmed.
