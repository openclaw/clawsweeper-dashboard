## Summary

Keep this open. The report is recent, concrete, and still maps to first-party macOS native WebChat rendering code; current main and the latest release still use transparent/non-opaque windows, active visual-effect material, material/gradient backgrounds, panel shadow, and animated scroll updates with no source, test, docs, changelog, or release evidence of a measured WindowServer CPU fix.

## What This Changes

The issue reports very high WindowServer CPU while the native macOS OpenClaw chat window is visible on an Intel Retina iMac, with CPU dropping sharply when the app is hidden.

## Best Possible Solution

Treat this as a macOS WebChat rendering-performance investigation: profile current main on affected Intel Retina hardware, A/B test opaque windowing, removing NSVisualEffectView/materials, disabling panel shadow, simplifying material/gradient backgrounds, and suppressing scroll/typing animations, then add the narrowest measured low-compositor fallback if the regression is confirmed.

## Reproduction Assessment

Unclear. The issue body gives a high-quality reproduction path and measurements, but high-confidence reproduction against current main requires affected Intel Retina macOS hardware and WindowServer profiling that was not available in this read-only review.

## Solution Assessment

Not applicable; there is no proposed patch. A source-only automated change is not the best next step until maintainers measure which rendering feature drives WindowServer CPU and choose the least disruptive fallback.
