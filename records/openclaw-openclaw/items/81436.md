## Summary

Keep open: this maintainer-labeled draft PR adds useful oc-path YAML coverage, but the patch has blocking write-path bugs, including a redaction-sentinel guard bypass, and it still lacks real behavior proof.

## What This Changes

Adds YAML/YML/.lobster support to the bundled oc-path plugin, wires YAML into resolve/find/set/emit and docs/tests, and adds `$first` positional addressing.

## Best Possible Solution

Keep YAML support in oc-path only after YAML writes preserve the existing sentinel safety contract, empty YAML sequences append correctly, and maintainers accept the feature scope.

## Reproduction Assessment

Yes for the review findings: source inspection at PR head shows the YAML set/insert path can return sentinel-bearing raw bytes and that empty sequences are misclassified by item-shape inference. I did not run the PR branch in this read-only review.

## Solution Assessment

No, not as-is. The narrow fix is to guard YAML caller-provided values like the existing kinds do and branch on yaml's isMap/isSeq helpers instead of inferring collection type from items.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Guard YAML writes before returning edited raw:** `extensions/oc-path/src/oc-path/universal.ts:599-605`
  - body: `setYamlLeaf` and `setYamlInsertion` can accept a string containing `__OPENCLAW_REDACTED__`, store rendered bytes in `ast.raw`, and then the CLI emits that raw text through `emitYaml`'s default round-trip path. That bypasses the existing oc-path write-time sentinel contract; guard the YAML value tree before returning the edited AST or force edited YAML through a render path that scans leaves.
  - confidence: 0.94
- **[P2] Handle empty YAML sequences as sequences:** `extensions/oc-path/src/oc-path/yaml/edit.ts:83`
  - body: For `items: []`, `items.every((p) => "key" in p)` is true, so the code treats the parent as a map and rejects `oc://x.yaml/items/+` instead of appending to an empty sequence. Use the yaml collection type guards (`isMap`/`isSeq`) so empty sequences and empty maps keep their real kind.
  - confidence: 0.9
