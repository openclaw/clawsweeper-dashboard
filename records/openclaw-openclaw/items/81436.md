## Summary

Keep open: this draft maintainer-labeled feature PR now includes real CLI proof and fixes the earlier keyed-marker sentinel issue, but the latest head still leaves a YAML object-key sentinel bypass and the restored YAML/.lobster plus `$first` scope needs maintainer approval.

## What This Changes

The PR adds YAML/YML/.lobster support to the oc-path plugin across parse, resolve, find, set, emit, tests, docs, changelog, `$first` positional addressing, and a plugin-local `yaml` dependency.

## Best Possible Solution

Accept the YAML restoration only after recursive sentinel guarding covers YAML keys as well as values, focused regression coverage is added, and maintainers approve the restored file-kind and positional-token scope.

## Reproduction Assessment

Yes for the review finding by source inspection: YAML insertion parses JSON object values, `guardYamlSentinel` skips object keys, and default `emitYaml` returns edited `ast.raw`. I did not execute the PR branch in this read-only review.

## Solution Assessment

No, not as-is. The narrow repair is to guard YAML object keys recursively or scan the edited YAML bytes before storing `ast.raw`, then keep maintainer review for the broader YAML/.lobster and `$first` scope.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Guard YAML object keys before emitting:** `extensions/oc-path/src/oc-path/yaml/edit.ts:192-193`
  - body: `guardYamlSentinel` walks `Object.entries()` but only checks each child value. For YAML insertion values parsed from JSON, a value like `{"__OPENCLAW_REDACTED__":2}` can therefore be written as a map key, and the default YAML emitter returns the edited raw bytes. Guard keys recursively, or scan the edited YAML bytes before returning the AST.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The latest diff fixes explicit YAML insertion-marker sentinel keys, but still permits the redaction sentinel in YAML object keys supplied as inserted values.

Concerns:

- **[medium] YAML object-key sentinel bypass:** `extensions/oc-path/src/oc-path/yaml/edit.ts:192`
  - body: A JSON object used as a YAML inserted value can contain `__OPENCLAW_REDACTED__` as a property key; the recursive guard checks only child values, and the default YAML emit path can persist that key to disk.
  - confidence: 0.88
