## Summary

Keep open. Current `main` and the reported `v2026.5.7` release still have channel-presence helpers that ignore a channel config whose only key is `enabled`, and the issue is already paired with an open closing PR at https://github.com/openclaw/openclaw/pull/81331.

## What This Changes

The issue asks OpenClaw to treat `channels.openclaw-weixin.enabled: true` as enough channel configuration to avoid an invalid `channels.start` failure, or to report the missing config more clearly.

## Best Possible Solution

Land the linked PR or an equivalent narrow fix after aligning all channel configuration presence helpers, preserving `enabled: false` as explicit disablement and adding regression coverage for the startup/plugin-load path.

## Reproduction Assessment

Yes, at source level. Current main returns false for enabled-only channel config in the helpers that decide whether a channel is configured, matching the reported minimal config failure without needing new product behavior.

## Solution Assessment

Partly. Treating `enabled: true` as a meaningful signal is the right narrow direction, but the best fix should also cover the startup presence helper and its tests, not only `hasMeaningfulChannelConfigShallow`.
