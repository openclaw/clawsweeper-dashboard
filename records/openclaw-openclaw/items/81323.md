## What This Changes

The issue requests that `channels.openclaw-weixin.enabled=true` alone be enough to load the channel, or that the resulting startup/login error explain the missing configuration.

## Best Possible Solution

Land a narrow regression fix that treats `enabled: true` as explicit configured channel state in the startup presence path while preserving `enabled: false` as disabled, with coverage for channel presence and gateway startup planning.
