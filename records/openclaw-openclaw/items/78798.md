## Summary

Keep open: current main and the latest release already cover command registration, argument parsing, help/catalog exposure, and `continueAgent`, but they still do not expose a typed same-turn pass-through result that replaces the LLM input with the remaining command text.

## What This Changes

The issue requests a plugin slash-command API that can parse command arguments, run plugin logic, optionally block handling, and pass the remaining text through as the normal LLM user message.

## Best Possible Solution

Extend the existing plugin command result contract with one typed continuation/pass-through shape that can replace the same-turn agent input body, while preserving today’s no-continuation default and updating SDK docs plus text/native command coverage.

## Reproduction Assessment

Not applicable: this is a feature/API request rather than a broken existing behavior report. Source inspection confirms the current API has parsed `args` and `continueAgent`, but no typed pass-through body result.

## Solution Assessment

Yes on the owner boundary, unclear on the exact field shape. The maintainable direction is to extend the existing plugin command contract rather than add another raw-message hook or require plugins to keep parsing slash input themselves.
