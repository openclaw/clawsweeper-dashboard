## Summary

Keep open: current main supports plugin slash-command registration, argument parsing, command listing, and `continueAgent`, but it still does not expose a typed result that rewrites the same-turn user message to the remaining command text.

## What This Changes

Request a Plugin SDK slash-command API that lets plugin handlers parse command arguments, optionally handle the command, and pass the remaining text back into the same LLM turn.

## Best Possible Solution

Add a backward-compatible plugin command continuation result that can replace the current turn body, wire it through text and native command paths, document it in the Plugin SDK, and cover the rewrite behavior with focused regression tests.

## Reproduction Assessment

Not applicable: this is a feature request for a new public Plugin SDK behavior, not a broken existing contract. Source inspection confirms the adjacent registration, args, list, and `continueAgent` behavior exists while the requested pass-through body rewrite does not.

## Solution Assessment

No: manual parsing through message hooks is a workable workaround but not the best long-term API. The maintainable fix is an additive command-result/body-rewrite contract that preserves existing authorization and command-list behavior.
