## Summary

Keep open: current main has a substantial plugin command API, including registration, argument parsing, `continueAgent`, and command catalog/docs integration, but it still does not provide the requested typed way for a plugin handler to replace the same-turn LLM input with the remaining command text.

## What This Changes

The issue requests a plugin API for slash-command registration where a plugin can intercept a command, inspect parsed arguments, optionally handle it, and pass a replacement user message through to the LLM.

## Best Possible Solution

Extend the existing plugin command result contract with one typed continuation/pass-through shape that can replace the same-turn agent input body, then cover text and native command paths plus SDK docs.

## Reproduction Assessment

Not applicable: this is a feature/API request, not a broken existing behavior report. Source inspection confirms the current API has `args` and `continueAgent` but no typed pass-through body result.

## Solution Assessment

Yes on the owner boundary, unclear on the exact field shape. The maintainable direction is to extend the existing plugin command contract rather than add another hook or ask plugins to keep parsing raw messages.
