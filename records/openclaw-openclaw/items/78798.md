## Summary

Current main already provides the requested plugin command surface: plugins can register slash commands, receive parsed args, choose whether command handling stops or the agent continues, and appear in command/help discovery; the feature shipped through the merged generic host-hook SDK work.

## What This Changes

The issue requests a plugin API for slash-command registration with parsed arguments, optional plugin handling, optional LLM continuation, and help output integration.

## Best Possible Solution

Keep the shipped `registerCommand` plus host-hook continuation contract and point plugin authors to the documented SDK surface instead of adding a second pass-through-shaped command API.

## Reproduction Assessment

Not applicable: this is a feature request rather than a broken existing behavior. Source, docs, tests, PR provenance, and release notes show the requested capability is already present.

## Solution Assessment

Yes. The shipped additive SDK contract covers registration, parsed args, blocking vs continuation, command policy, and help discovery while preserving plugin/core boundaries.
