## Summary

Keep open: current main and v2026.6.10 already provide plugin command registration, parsed args, help/catalog exposure, `continueAgent`, and skill-command prompt rewriting, but code-plugin command results still cannot replace the same-turn LLM input with the remaining command text. This is a useful plugin SDK feature request, reinforced by a second user comment, but it needs maintainer API direction before implementation.

## What This Changes

The issue requests a plugin SDK slash-command API that lets a command handler run custom logic, optionally block, or pass the remaining command text into the normal LLM turn.

## Best Possible Solution

Add a maintainer-sponsored, additive command continuation result shape to the existing plugin command contract, with docs and regression coverage for text and native command paths.

## Reproduction Assessment

Not applicable; this is a feature/API request rather than a broken existing behavior report. Source inspection confirms current main has parsed args and continueAgent but no typed same-turn replacement input result.

## Solution Assessment

Yes on the owner boundary, unclear on the exact field shape. The maintainable path is to extend the existing plugin command result/routing contract rather than ask plugins to keep using raw-message hooks and manual parsing.
