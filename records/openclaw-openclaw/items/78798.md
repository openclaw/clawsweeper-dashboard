## Summary

Keep open: current main and latest release have plugin command registration, argument parsing, help/catalog exposure, and `continueAgent`, but they still do not expose a typed way for a plugin command to replace the same-turn LLM input with the remaining command text. The +1 user comment reinforces that this is still useful, and the current labels/product policy make it a maintainer API decision rather than a safe cleanup close.

## What This Changes

The issue requests a plugin API for registering slash commands that can run plugin logic and pass the remaining command text through to the LLM as the same user message.

## Best Possible Solution

Decide the public command continuation shape, then extend the existing plugin command result/routing contract with focused SDK docs and regression coverage for text and native command paths.

## Reproduction Assessment

Not applicable: this is a feature/API request, not a broken existing behavior report. Source inspection confirms the current API has parsed `args` and `continueAgent` but no typed pass-through body result.

## Solution Assessment

Yes on the owner boundary, unclear on the exact field shape. The maintainable path is to extend the existing plugin command contract rather than ask plugins to keep using raw-message hooks and manual parsing.
