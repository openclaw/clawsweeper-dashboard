## Summary

Keep open: current main covers plugin command registration, argument parsing, command listing, and `continueAgent`, but it still does not expose a typed pass-through result that rewrites the same-turn LLM input. This is a public Plugin SDK design gap rather than a stale, duplicate, or ClawHub-only item.

## What This Changes

Request a Plugin SDK slash-command API that lets plugins parse command arguments, optionally handle the command, and pass a rewritten same-turn message back to the LLM.

## Best Possible Solution

Add a backward-compatible plugin command continuation result that can explicitly replace the same-turn user body for agent continuation, then document and test it across text and native command paths.

## Reproduction Assessment

Not applicable: this is a new public Plugin SDK feature request, not a broken documented behavior. Source inspection shows existing registration, args, help/list integration, and `continueAgent`, but no typed body rewrite/pass-through field.

## Solution Assessment

Unclear until maintainer design: the capability belongs in the Plugin SDK, but the exact result shape and body-rewrite semantics should be chosen before code changes.
