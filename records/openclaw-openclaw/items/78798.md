## Summary

Keep open: current main has plugin command registration, argument parsing, command listing, and `continueAgent`, but it still lacks a typed plugin command result that rewrites the same-turn user message to the remaining command text before the LLM turn. This remains a public Plugin SDK design request, not a closeable cleanup item.

## What This Changes

The issue requests a Plugin SDK slash-command API that lets plugins parse command arguments, run custom logic, and pass the remaining text through to the LLM as the same-turn user message.

## Best Possible Solution

Add a backward-compatible plugin command continuation contract that can explicitly replace the same-turn user body, document it in the Plugin SDK and slash-command docs, and cover both text and native command paths with regression tests.

## Reproduction Assessment

Not applicable: this is a feature request for new public Plugin SDK behavior, not a bug with an existing expected result. Source inspection confirms the adjacent plugin command API exists, but the requested pass-through result shape is still absent on current main.

## Solution Assessment

Unclear: the requested direction is reasonable, but maintainers should choose the exact result contract so it aligns with `continueAgent`, skill-command body rewriting, command authorization, and transcript semantics.
