## Summary

Keep open: current main has plugin command registration, argument parsing, command listing, and `continueAgent`, but it still lacks the requested typed pass-through/body-rewrite result that would send the remaining command text to the LLM in the same turn.

## What This Changes

Request a Plugin SDK slash-command API that lets plugin handlers run custom logic, optionally block, and pass the remaining command text through to the LLM with help integration.

## Best Possible Solution

Add a backward-compatible plugin command result contract for replacing the same-turn user body, wire it through text and native command paths, and document/test the new SDK behavior.

## Reproduction Assessment

Not applicable: this is a feature request for new public Plugin SDK behavior, not a bug with an existing expected result. Source inspection confirms adjacent command registration and continuation exist, but a pass-through/body-rewrite result does not.

## Solution Assessment

Unclear until maintainers choose the SDK contract: the feature direction is valid, but the proposed `passThrough` shape needs API design. The safer path is an additive typed continuation/body-replacement contract rather than asking plugins to parse hooks manually.
