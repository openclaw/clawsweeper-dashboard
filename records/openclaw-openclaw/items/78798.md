## Summary

Keep open: current main has plugin command registration, argument parsing, command listing/help integration, `continueAgent`, and next-turn injection APIs, but it still lacks the requested typed command result that rewrites the same-turn user message to the remaining command text. This is a public Plugin SDK/API design request, not an already-fixed bug.

## What This Changes

Request a public Plugin SDK slash-command API that lets handlers intercept a command, run custom logic, optionally block, and pass the remaining command text to the LLM as the same-turn user message.

## Best Possible Solution

Add a backward-compatible plugin command result contract that can explicitly replace the same-turn user body, document it in the Plugin SDK and slash-command docs, and cover text plus native command paths with focused tests.

## Reproduction Assessment

Not applicable: this is a feature request for new public Plugin SDK behavior, not a bug with an existing expected result. Source inspection confirms adjacent command registration and `continueAgent` behavior exist, but the requested same-turn pass-through result shape does not.

## Solution Assessment

Unclear until maintainers choose the API shape. The maintainable direction is an additive command-result/body-rewrite contract rather than asking plugins to keep using hooks plus manual string parsing.
