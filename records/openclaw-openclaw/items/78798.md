## Summary

Keep open: current main already has plugin command registration, argument parsing, command listing, and `continueAgent`, but it still lacks the requested typed command result that rewrites the same-turn user message to the remaining command text. This is a public Plugin SDK design request, not a cleanup-close candidate.

## What This Changes

Add a Plugin SDK API that lets registered slash commands run custom logic and pass rewritten remaining text into the same-turn LLM prompt.

## Best Possible Solution

Add a backward-compatible plugin command result contract that can explicitly replace the same-turn agent message, then document it and cover text plus native command paths with regression tests.

## Reproduction Assessment

Not applicable: this is a feature request for a new public Plugin SDK behavior, not a bug with an existing expected result. Source inspection confirms the adjacent command registration and continuation behavior exists, but the requested body replacement result does not.

## Solution Assessment

Unclear until maintainers choose the public contract. The maintainable direction is an additive command-result/body-rewrite API rather than asking plugins to keep parsing raw messages through hooks.
