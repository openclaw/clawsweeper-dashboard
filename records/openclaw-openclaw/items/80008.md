## Summary

Keep open: this is a useful plugin runtime feature, but the current patch is not mergeable because ACP authority is global, plugin ACP prompts do not preserve explicit-turn policy semantics, and the PR head fails the architecture cycle check.

## What This Changes

The branch adds api.runtime.acp.spawn() and api.runtime.acp.prompt(), wires a new plugins.allowAcpSpawn config/schema flag, updates runtime types/mocks, and adds ACP runtime tests.

## Best Possible Solution

Land only a maintainer-approved ACP runtime API with caller-scoped authorization, explicit-turn gateway parameters, leaf contract types that keep import-cycle checks green, and docs/release follow-through.

## Reproduction Assessment

Not applicable as a user bug reproduction; this is a feature PR. The review blockers are high-confidence from source and CI: the latest head still uses a global gate, omits acpTurnSource on prompts, and fails the Madge architecture check.

## Solution Assessment

No, not yet. Routing through the gateway delivery path is the right direction, but the implementation needs per-plugin policy, explicit ACP turn semantics, and cycle-free runtime contract types before it is the maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Scope ACP permission to each calling plugin:** `src/config/types.plugins.ts:70`
  - body: The new plugins.allowAcpSpawn flag is global, so enabling ACP dispatch for one trusted plugin grants every loaded plugin access to spawn or prompt host-side ACP agent sessions. Current runtime helpers already carry plugin identity and sensitive policies live under plugins.entries.<id>; gate this surface per caller before exposing it.
  - confidence: 0.91
- **[P2] Mark ACP prompts as explicit manual turns:** `src/plugins/runtime/runtime-acp.runtime.ts:52-67`
  - body: promptPluginAcp sends the gateway agent request without acpTurnSource: "manual_spawn". With acp.dispatch.enabled=false, these follow-up prompts are treated like ordinary ACP dispatch and are blocked, while native ACP spawn sets the marker so manual spawned turns remain allowed.
  - confidence: 0.88
- **[P2] Move ACP runtime types out of the agent implementation:** `src/plugins/runtime/types-core.ts:1`
  - body: Importing SpawnAcpParams/Context/Result from ../../agents/acp-spawn.js into plugin runtime types pulls the agent implementation graph into the plugin runtime contract and the PR head fails check:architecture with a Madge circular dependency. Extract a leaf contract type for this API instead of depending on acp-spawn.ts.
  - confidence: 0.86

## Security Review

Status: needs_attention
