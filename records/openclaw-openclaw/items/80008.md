## Summary

Keep open: this is a plausible feature PR with useful real-world logs, but the latest head still exposes privileged ACP dispatch through a global plugin flag and sends plugin ACP prompts without the explicit-turn marker current ACP policy expects.

## What This Changes

The branch adds `api.runtime.acp.spawn()` and `api.runtime.acp.prompt()`, a new `plugins.allowAcpSpawn` flag, runtime wiring/types/mocks, and plugin runtime tests.

## Best Possible Solution

Accept this seam only with caller-scoped plugin authorization under `plugins.entries.<id>`, explicit ACP turn semantics, docs/config help, and focused runtime/policy tests.

## Reproduction Assessment

Yes for the review blockers: source inspection of the latest PR head shows a top-level ACP gate and a gateway prompt request without `acpTurnSource`. I did not run PR-head tests because this review is read-only.

## Solution Assessment

No, not yet. Routing plugin ACP prompts through the gateway delivery path is the right direction, but the privilege model and ACP policy marker need to be fixed before this is the maintainable core API.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Scope ACP permission to the calling plugin:** `src/config/types.plugins.ts:70`
  - body: `plugins.allowAcpSpawn` is a top-level flag, so enabling ACP for one trusted plugin enables every loaded plugin to spawn or prompt ACP agent turns. Existing sensitive runtime policies are caller-scoped under `plugins.entries.<id>` and the runtime proxy already carries `pluginId` for `llm` and `subagent`; this ACP authority should use the same per-plugin boundary.
  - confidence: 0.88
- **[P2] Mark plugin ACP prompts as explicit turns:** `src/plugins/runtime/runtime-acp.runtime.ts:52-67`
  - body: `promptPluginAcp()` sends the gateway `agent` request without `acpTurnSource: "manual_spawn"`, so prompts into existing ACP sessions are treated as automatic dispatch and are blocked when `acp.dispatch.enabled=false`. Native ACP spawn sets this marker, and current policy tests show explicit manual turns should remain allowed while dispatch is paused.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff introduces a privileged plugin-to-agent dispatch surface, and the current gate grants that authority globally rather than per caller plugin.

Concerns:

- **[medium] Global ACP spawn gate grants authority to all plugins:** `src/config/types.plugins.ts:70`
  - body: Once `plugins.allowAcpSpawn` is enabled, every loaded plugin can dispatch ACP agent turns with channel delivery. This should be scoped to the calling plugin, likely under `plugins.entries.<id>`, before exposing agent dispatch to third-party plugin code.
  - confidence: 0.88
