## Summary

Keep open: this is a plausible plugin-runtime feature, but the current PR head still exposes privileged ACP dispatch behind a global plugin flag and omits the explicit ACP turn marker that current main uses for manual ACP turns.

## What This Changes

The PR adds `api.runtime.acp.spawn()` and `api.runtime.acp.prompt()`, a new `plugins.allowAcpSpawn` config gate, tests, and plugin runtime type/mock updates.

## Best Possible Solution

Land the ACP runtime seam only after the API is explicitly accepted, caller-scoped plugin authorization is enforced, plugin ACP prompts use explicit ACP turn semantics, and the public plugin/config docs are updated.

## Reproduction Assessment

Yes for the review blockers: source inspection of the PR head shows a top-level `plugins.allowAcpSpawn` gate and no `acpTurnSource`, while current main shows per-plugin policy plumbing and explicit ACP turn handling.

## Solution Assessment

No, not yet. Routing plugin prompts through the gateway delivery path is the right primitive, but the privilege model and ACP policy marker need to be fixed before this is the maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Scope ACP permission to the calling plugin:** `src/config/types.plugins.ts:70`
  - body: The new `plugins.allowAcpSpawn` flag is top-level, so enabling ACP for one trusted plugin gives every loaded third-party plugin the same agent-dispatch surface. Current sensitive runtime policies use `plugins.entries.<id>` with runtime `pluginId` scope; gate this per calling plugin before exposing ACP spawn/prompt.
  - confidence: 0.9
- **[P2] Mark plugin ACP prompts as explicit turns:** `src/plugins/runtime/runtime-acp.runtime.ts:54-67`
  - body: `promptPluginAcp()` sends the gateway `agent` request without `acpTurnSource: "manual_spawn"`. Existing ACP session prompts will be treated as automatic dispatch and fail when `acp.dispatch.enabled=false`, while native ACP spawn sets this marker and current tests preserve that explicit-turn exception.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: The patch exposes privileged plugin-to-agent ACP dispatch behind a global flag rather than caller-scoped authorization.

Concerns:

- **[medium] Global ACP spawn gate grants authority to all plugins:** `src/config/types.plugins.ts:70`
  - body: `plugins.allowAcpSpawn` is top-level plugin config, so once enabled any loaded plugin can dispatch ACP agent turns with channel delivery; the safer boundary is caller-scoped policy under `plugins.entries.<id>` plus runtime `pluginId` scoping.
  - confidence: 0.9
