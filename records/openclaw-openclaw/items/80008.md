## Summary

Keep this PR open for maintainer review: the ACP delivery gap is real and the PR includes useful live logs, but the current patch still exposes a privileged agent-dispatch surface behind a global plugin flag and misses the explicit ACP turn marker used by current main policy.

## What This Changes

The PR adds `api.runtime.acp.spawn()` and `api.runtime.acp.prompt()` to plugin runtime, plus a new `plugins.allowAcpSpawn` config key, tests, and runtime mocks.

## Best Possible Solution

Keep the ACP runtime seam only if maintainers want this core API, with caller-scoped plugin authorization, explicit ACP turn semantics, docs, and focused runtime coverage.

## Reproduction Assessment

Yes for the review blockers: source inspection shows the gate is global rather than caller-scoped, and the prompt path lacks the ACP explicit-turn marker that current main policy relies on.

## Solution Assessment

No, not yet. Routing prompts through the gateway delivery path is the right direction, but the privilege model and ACP policy marker need to be fixed before this is the maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Scope ACP permission to the calling plugin:** `src/config/types.plugins.ts:67-72`
  - body: The new top-level `plugins.allowAcpSpawn` flag is checked without plugin identity, so enabling ACP for one trusted plugin enables every loaded third-party plugin to spawn or prompt ACP agent turns. Existing sensitive runtime policies are under `plugins.entries.<id>` and current runtime scope already carries `pluginId`; gate this per calling plugin before exposing agent dispatch.
  - confidence: 0.88
- **[P2] Mark plugin ACP prompts as explicit turns:** `src/plugins/runtime/runtime-acp.runtime.ts:50-65`
  - body: `promptPluginAcp()` sends the gateway `agent` request without `acpTurnSource: "manual_spawn"`, so existing ACP session prompts are treated as automatic dispatch and fail when operators set `acp.dispatch.enabled=false`. Native ACP spawn sets this marker, and the current policy tests show explicit turns should remain allowed when dispatch is disabled.
  - confidence: 0.86
