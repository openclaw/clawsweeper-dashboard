## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Make the startup script executable before switching users:** `Dockerfile:292`
  - body: `USER node` is already active when this added `RUN chmod` executes. Because `COPY` creates `/home/node/start.sh` as root-owned by default, the `node` user cannot chmod it, so the Docker build fails before the new runtime path can start.
  - confidence: 0.92
- **[P2] Write config where OpenClaw will read it:** `start.sh:3`
  - body: The script writes `/home/node/.openclaw/config.json`, but OpenClaw's default config path is `/home/node/.openclaw/openclaw.json` unless `OPENCLAW_CONFIG_PATH` is set. The gateway will start with `--allow-unconfigured` and ignore the generated Discord/DeepSeek config.
  - confidence: 0.93
- **[P2] Include the wildcard required by open Discord DMs:** `start.sh:8`
  - body: The generated Discord config sets `dmPolicy` to `open` without adding `allowFrom: ["*"]`. The Discord schema rejects that shape, so once the config is written to the active file it will fail validation instead of enabling Discord.
  - confidence: 0.9
- **[P2] Preserve the Docker pnpm cache contract:** `Dockerfile:65`
  - body: The repo has a cache-layout test requiring the dependency install mount to use `id=openclaw-pnpm-store`. Replacing it with a generated path-like ID breaks that contract and loses the stable shared cache key.
  - confidence: 0.88
- **[P3] Keep plugin source edits out of the install layer:** `Dockerfile:33`
  - body: The ext-deps stage is designed to expose the bundled plugin tree only long enough to copy selected `package.json` files. Copying the whole plugin directory before `pnpm install` makes unrelated plugin source edits invalidate the dependency layer.
  - confidence: 0.82
