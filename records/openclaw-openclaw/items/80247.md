Full review comments:

- **[P1] Stop printing generated secrets:** `start.sh:21`
  - body: `cat /home/node/.openclaw/config.json` prints the env-expanded Discord token and DeepSeek API key to container logs on every startup, which would expose credentials in Railway or Docker logs.
  - confidence: 0.95
- **[P2] Write the generated config to the active config path:** `start.sh:3`
  - body: The script writes `/home/node/.openclaw/config.json`, but OpenClaw reads `/home/node/.openclaw/openclaw.json` by default unless `OPENCLAW_CONFIG_PATH` is set, so the gateway will ignore the generated Discord/DeepSeek config.
  - confidence: 0.94
- **[P2] Include the wildcard required for open Discord DMs:** `start.sh:8`
  - body: Discord config with `dmPolicy: "open"` is rejected unless the effective `allowFrom` includes `"*"`; this generated config omits that allowlist and will fail validation once written to the active file.
  - confidence: 0.92
- **[P2] Preserve the shared pnpm cache id:** `Dockerfile:66`
  - body: Replacing `id=openclaw-pnpm-store` breaks the repository's Docker cache-layout contract and loses the stable shared dependency cache key that current tests require.
  - confidence: 0.89
- **[P3] Keep plugin sources out of the dependency layer:** `Dockerfile:34`
  - body: Copying the whole bundled plugin directory before `pnpm install` makes unrelated plugin source edits invalidate the dependency layer; the current bind-mount stage only exposes selected package manifests for that reason.
  - confidence: 0.82
