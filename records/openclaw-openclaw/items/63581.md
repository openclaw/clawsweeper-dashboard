## What This Changes

The PR updates iMessage echo-cache text normalization to strip leading NUL corruption markers while preserving interior NUL bytes, with focused tests and a changelog entry.

## Best Possible Solution

Land the narrow PR after normal merge gates, keeping cleanup limited to known leading corruption markers and preserving interior-text distinctions.

## Solution Assessment

Yes. Adding U+0000 to the existing leading-only attributedBody corruption cleanup is the narrowest maintainable fix and avoids broader control-character or whitespace normalization changes.
