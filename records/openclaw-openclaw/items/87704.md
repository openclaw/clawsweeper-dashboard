## Summary

Keep open: current main and latest release v2026.6.6 still contain the progress-mode suppression path and the delayed progress-draft gate, so the reported Discord visibility gap is not fixed or superseded. The root cause is source-reproducible, but the repair should wait for a maintainer UX decision because current tests and docs also encode standalone progress suppression during active preview drafts.

## What This Changes

The issue requests restoring visible Discord per-tool progress when progress-mode draft previews stay empty for fast tool calls.

## Best Possible Solution

Choose and test a fallback that preserves Discord progress visibility without adding config, such as suppressing standalone tool messages only after the progress draft has actually appeared or starting the progress draft on the first visible tool event.

## Reproduction Assessment

Yes for source reproduction: current main suppresses standalone tool messages in progress mode while the progress draft gate can defer the first work event past a fast tool call, but no live Discord repro was run here.

## Solution Assessment

The requested visibility is valid, but the reporter's blanket unsuppression patch is only one plausible fix; maintainers should choose the intended shared progress UX and update Discord/shared tests and docs together.
