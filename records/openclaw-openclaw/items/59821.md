## Summary

This PR should stay open because it targets a real, source-reproducible bug in the bundled skills allowlist, but it is not merge-ready: the added regression test is environment-dependent and the external PR still lacks after-fix real behavior proof.

## What This Changes

The PR preserves empty `skills.allowBundled` arrays through normalization, treats `undefined` as unrestricted and `[]` as block-all for bundled skills, and adds a prompt regression test.

## Best Possible Solution

Land a rebased narrow fix that preserves `undefined` as unrestricted and `[]` as block-all, with isolated regression coverage and contributor-provided after-fix runtime proof.

## Reproduction Assessment

Yes. Source inspection on current main shows `skills.allowBundled: []` is normalized to `undefined`, then treated as unrestricted even though the docs define a set allowlist as limiting bundled eligibility.

## Solution Assessment

Partially. The production change is the narrow maintainable fix, but the test should isolate HOME/state and clean up temp files, and the external PR needs after-fix real behavior proof before merge.
