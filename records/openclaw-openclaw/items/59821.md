## Summary

Keep open: current main still has the documented empty-allowlist bug and the production fix is useful, but this PR is not merge-ready because the new prompt test is not isolated from ambient user skill roots and the external after-fix real behavior proof is missing.

## What This Changes

The PR preserves explicit empty `skills.allowBundled` arrays, treats them as block-all for bundled skills while leaving non-bundled skills eligible, and adds a prompt regression test.

## Best Possible Solution

Land a repaired and rebased version that preserves the undefined-vs-empty allowlist distinction, isolates the regression test from ambient home/OpenClaw roots, and lets maintainers handle the release note during landing.

## Reproduction Assessment

Yes. Source inspection on current main shows `allowBundled: []` normalizes to `undefined` and the bundled gate then treats it as unrestricted, while the docs define set `allowBundled` values as a bundled-skill allowlist.

## Solution Assessment

Partially. The production change is the narrowest maintainable fix, but the submitted test needs environment isolation and cleanup, and the PR needs after-fix real behavior proof before merge.
