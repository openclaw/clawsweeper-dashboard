## Summary

Keep open. Current main still violates the documented `skills.allowBundled` semantics, and the PR's production fix is narrow, but the submitted patch needs test isolation/cleanup and a user-facing changelog entry before merge.

## What This Changes

The PR preserves explicit empty `skills.allowBundled` arrays, treats them as block-all for bundled skills while leaving non-bundled skills eligible, and adds an empty-array prompt regression test.

## Best Possible Solution

Land a repaired version that preserves the documented undefined-vs-empty allowlist distinction, isolates the regression test from ambient skill roots, and records the fix under Unreleased/Fixes.

## Reproduction Assessment

Yes. The current-main source path deterministically reproduces the bug: `allowBundled: []` normalizes to `undefined`, then the bundled-skill gate treats it as unrestricted despite the docs saying a set allowlist limits bundled skills.

## Solution Assessment

Partially. The production-code change is the narrowest maintainable fix, but the submitted patch is not merge-ready until the new test is isolated/cleaned up and the changelog entry is added.
