## Summary

Keep open: current main still violates the documented empty `skills.allowBundled` contract, and this PR has the right production fix shape, but it is not merge-ready because it lacks after-fix real behavior proof and introduces a non-isolated regression test.

## What This Changes

The PR changes skills allowlist normalization and bundled-skill eligibility so `skills.allowBundled: []` blocks bundled skills, and adds a prompt regression test.

## Best Possible Solution

Land a rebased narrow fix that preserves `undefined` as unrestricted and `[]` as block-all, with isolated regression coverage and contributor-provided after-fix runtime proof.

## Reproduction Assessment

Yes. Source inspection on current main shows `skills.allowBundled: []` is normalized to `undefined`, then treated as unrestricted even though the docs say a set allowlist limits eligible bundled skills.

## Solution Assessment

Partially. The production change is the narrow maintainable fix, but the test should isolate HOME/state and clean up temp files, and the external PR needs after-fix real behavior proof before merge.
