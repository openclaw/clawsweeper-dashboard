## Summary

Keep open: current main still has the documented empty-allowlist bug, and the production part of this PR is the right narrow direction. It is not merge-ready because the external PR still lacks after-fix real behavior proof and the added prompt test is not isolated from ambient personal skills.

## What This Changes

The PR preserves an explicit empty `skills.allowBundled` array as a block-all bundled-skill allowlist and adds prompt regression coverage.

## Best Possible Solution

Land a rebased narrow fix that preserves `undefined` as unrestricted and `[]` as block-all for bundled skills, with isolated regression coverage and contributor-provided after-fix runtime proof.

## Reproduction Assessment

Yes. Source inspection on current main shows `skills.allowBundled: []` is normalized to `undefined` and then treated as unrestricted, while the docs define a set allowlist as limiting bundled eligibility.

## Solution Assessment

Partially. The production logic change is the narrow maintainable fix, but the proposed test needs HOME/state isolation and cleanup, and the external PR needs after-fix real behavior proof before merge.
