## Summary

Keep open. The PR targets a real documented bug in current main, but it still needs contributor-provided after-fix real behavior proof and the added regression test has an isolation flaw that can fail on machines with a personal `healthcheck` skill.

## What This Changes

The PR preserves explicit empty `skills.allowBundled` arrays so they block bundled skills, and adds a workspace prompt regression test for that behavior.

## Best Possible Solution

Land a rebased narrow fix that preserves `undefined` as unrestricted and `[]` as block-all, with isolated regression coverage and contributor-provided after-fix runtime proof.

## Reproduction Assessment

Yes. Source inspection on current main shows `skills.allowBundled: []` is normalized to `undefined`, then treated as unrestricted despite the documented allowlist contract.

## Solution Assessment

Partially. The production change is the narrow maintainable fix, but the test should isolate HOME/state and clean up temp files, and the external PR needs after-fix real behavior proof before merge.
