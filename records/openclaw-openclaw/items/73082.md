## Summary

Keep open: current main still only has agent-level exec approvals and autoAllowSkills, not a trusted skill-scoped readonly command approval contract; adding that contract changes host-exec trust semantics and needs maintainer/security design first.

## What This Changes

Request a skill metadata and approval-policy mechanism that lets readonly healthcheck/canary skills auto-approve bounded OpenClaw status commands without per-command prompts.

## Best Possible Solution

Design an additive, versioned skill-scoped exec approval contract in core, bound to trusted skill identity and strict argv rules, then update metadata parsing, protocol/schema, runtime evaluation, docs/UI, and security regression tests together.

## Reproduction Assessment

Not applicable as a feature request. Static source and docs inspection is high confidence that current main lacks a skill-scoped readonly exec approval contract.

## Solution Assessment

Unclear. The proposed metadata shape is plausible, but the narrow maintainable solution needs maintainer-approved trust semantics for skill identity, argv matching, and fail-closed side-effect exclusions first.
