## Summary

Keep open: current main still has agent-level exec approvals and `autoAllowSkills`, but no skill-scoped readonly approval contract. The request would add a metadata-driven host-exec trust boundary, so it needs maintainer/security design rather than cleanup closure or an automated fix.

## What This Changes

The issue requests a skill-level readonly exec approval scope for canary or healthcheck skills, with bounded command allowlists instead of per-command hash approvals.

## Best Possible Solution

Design an additive skill-scoped exec approval contract tied to trusted skill identity and explicit argv rules, then update metadata parsing, protocol/schema, runtime evaluation, docs, UI, and security regression coverage together.
