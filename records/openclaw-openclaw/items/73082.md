## Summary

Keep open: current main still lacks a skill-scoped readonly exec approval contract, and the request would create a security-sensitive host-exec trust boundary that needs maintainer/security design before implementation.

## What This Changes

Request a `SKILL.md` metadata field for readonly skills to auto-approve bounded read-only `system.run` commands without per-invocation prompts.

## Best Possible Solution

Design an additive approval contract bound to trusted skill identity plus explicit argv rules, then update metadata parsing, protocol/schema, approval evaluation, docs/UI, and security tests together.

## Reproduction Assessment

Not applicable as a feature request. Static verification is high confidence: the requested metadata names are absent and current main routes approvals through agent-level allowlists plus `autoAllowSkills`.

## Solution Assessment

Unclear. The proposed metadata direction is plausible, but maintainers should settle the trust boundary first because it would bypass per-invocation approval for host exec commands.
