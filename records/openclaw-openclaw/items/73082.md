## Summary

Keep open: current main still lacks a skill-scoped readonly exec approval contract, and the request changes host-exec trust semantics enough to need maintainer/security design before implementation.

## What This Changes

Add a skill-level readonly exec approval scope and per-skill command allowlist so canary and healthcheck skills can run approved introspection commands without repeated prompts.

## Best Possible Solution

Design an additive skill-scoped exec approval contract, bind it to trusted skill identity and explicit argv rules, then update metadata parsing, protocol/schema, runtime evaluation, docs/UI, and security regression tests together.

## Reproduction Assessment

Not applicable as a feature request. Static source and docs inspection is high confidence that current main has agent-level exec approvals plus autoAllowSkills, but no skill-scoped readonly exec approval contract.

## Solution Assessment

Unclear. The proposed metadata shape is plausible, but the narrow maintainable solution needs maintainer-approved trust semantics for skill identity, argv matching, and fail-closed side-effect exclusions first.
