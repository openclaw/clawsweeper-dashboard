## Summary

Keep open: current main still lacks a skill-scoped readonly exec approval contract, and the proposal changes host-exec trust semantics, so it needs maintainer/security design rather than conservative cleanup.

## What This Changes

Add skill-level metadata that lets readonly healthcheck/canary skills auto-approve bounded read-only host exec commands without per-command prompts.

## Best Possible Solution

Define an additive approval contract that binds readonly auto-approval to trusted skill identity plus explicit argv rules, then update metadata parsing, protocol/schema, runtime enforcement, docs, UI, and security tests together.

## Reproduction Assessment

Not applicable as a feature request. Static inspection provides a high-confidence verification path: current main has no requested metadata symbols and still relies on agent-level allowlists plus broad autoAllowSkills.

## Solution Assessment

Unclear. The metadata direction is plausible, but the exact trust model should be settled by maintainers because it would create a new metadata-driven host-exec auto-approval path.
