## Summary

Keep open: current main still lacks a skill-scoped readonly exec approval contract, and the request would change host-exec trust semantics rather than fix broken existing behavior.

## What This Changes

The issue requests a skill-level readonly exec approval scope for healthcheck/canary skills so bounded introspection commands can run without repeated per-command approval prompts.

## Best Possible Solution

Design an additive approval contract bound to trusted skill identity and explicit argv rules, then update metadata parsing, protocol/schema, runtime evaluation, UI/docs, and security regression tests together.

## Reproduction Assessment

Not applicable as a feature request. Static verification is high confidence: current main has agent-level exec approvals and broad skill-bin auto-allow, but no skill-scoped readonly metadata contract.

## Solution Assessment

Unclear. The proposed metadata direction is plausible, but maintainers should settle the host-exec trust model before accepting skill-authored readonly auto-approval.
