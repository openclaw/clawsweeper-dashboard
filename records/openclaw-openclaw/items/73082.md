## Summary

Keep open: current main still has agent-level exec approval policy plus broad skill-bin auto-allow, but no skill-scoped readonly exec approval contract. This is a real feature request, and it changes host-exec trust semantics enough to need maintainer/security design before implementation.

## What This Changes

Request a skill-level readonly exec auto-approval scope for canary and healthcheck skills so bounded introspection commands can run without repeated per-command prompts.

## Best Possible Solution

Design an additive approval contract around trusted skill identity and explicit argv rules, then update metadata parsing, protocol/schema, runtime evaluation, UI/docs, and security regression tests together.

## Reproduction Assessment

Not applicable as a feature request. Static verification is high confidence: current main lacks the requested metadata fields and still routes exec approval through agent-level allowlists plus autoAllowSkills.

## Solution Assessment

Unclear. The proposed metadata shape is plausible, but the narrow maintainable solution requires a maintainer-approved host-exec trust model before implementation.
