## Summary

Keep open: current main still has only agent-level exec approvals plus broad skill-CLI auto-allow, not a skill-scoped readonly exec approval contract. This is a useful feature request, but it changes host-exec trust semantics and needs maintainer/security design before implementation.

## What This Changes

Request a skill-scoped readonly exec approval metadata and allowlist contract so healthcheck/canary skills can run bounded introspection commands without per-command prompts.

## Best Possible Solution

Design an additive approval contract bound to trusted skill identity and explicit argv rules, then update metadata parsing, protocol/schema, runtime evaluation, UI/docs, and security regression tests together.

## Reproduction Assessment

Not applicable as a feature request. Static verification is high confidence: the requested metadata names are absent and current main routes exec approval through agent-level allowlists and autoAllowSkills.

## Solution Assessment

Unclear. The proposed metadata direction is plausible, but maintainers should settle the host-exec trust model before accepting skill-authored readonly auto-approval.
