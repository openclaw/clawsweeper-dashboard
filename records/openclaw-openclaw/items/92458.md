## Summary

## What This Changes

The issue reports that the 2026.6 OpenAI provider unification migration rewrote legacy Codex OAuth profiles to `openai:*` while the Codex app-server auth path still rejected that profile shape.

## Best Possible Solution

Keep the v2026.6.6 unified OpenAI auth behavior and direct affected v2026.6.5 users to upgrade, then run `openclaw doctor --fix` if legacy profile ids remain.

## Reproduction Assessment

No current-main failing reproduction remains. Source and release-tag checks show current main and v2026.6.6 select `auth.order.openai` and accept `provider: "openai"` OAuth credentials for Codex app-server login.

## Solution Assessment

Yes. Aligning the migration and validator on canonical `openai` auth profiles is the maintainable fix, and it matches the repository direction to keep `openai-codex` as legacy migration input only.
