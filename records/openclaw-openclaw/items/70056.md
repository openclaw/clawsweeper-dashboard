## Summary

Keep open: this active gateway bug-fix PR still has a blocking stale-task race and lacks real behavior proof. Current main has since merged a narrower health-monitor recovery-stop fix in https://github.com/openclaw/openclaw/pull/77686, but it does not implement this PR's manual stop-timeout cleanup and the PR needs revision before it can safely land.

## What This Changes

The PR changes gateway stop-timeout handling to clear stored task/abort entries, mark the account stopped with lastStopAt, and update the restart-after-timeout test.

## Best Possible Solution

Revise the gateway lifecycle fix so releasing or replacing a timed-out task is protected by task identity or generation guards around every old-task-owned state write and restart step, then prove the delayed-old-task-settles-after-replacement path.

## Reproduction Assessment

Yes, source-level: current main still has a deterministic test shape where a never-settling account task hits the 5 second stop timeout and the task-map guard controls whether a later start can proceed. The PR-specific race is reproducible by letting the old task settle after a replacement start.

## Solution Assessment

No. The PR addresses the stale map-entry symptom, but freeing store.tasks before guarding the old trackedPromise's callbacks is not the safest complete fix; identity or generation guards are needed before replacement starts can be trusted.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Guard old task callbacks before freeing the slot:** `src/gateway/server-channels.ts:576-577`
  - body: Clearing `store.tasks` here releases the duplicate-start guard while the old `trackedPromise` is still alive. If that old task settles after a replacement starts, its `.catch`, `.finally`, and restart chain can still write runtime state or enqueue restart bookkeeping for the same account, so the replacement can be marked stopped or restarted incorrectly. Guard task-owned callbacks with a task identity/generation check and cover delayed old-task settlement after replacement start.
  - confidence: 0.9

## Security Review

Status: needs_attention
