## Summary

Keep open. Current main has low-level thinking-block stripping and provider-gated replay sanitation, but it still lacks the requested user-visible retention config and current Claude 4.5+/future-model policy intentionally preserves prior thinking blocks for prompt-cache behavior.

## What This Changes

Add a configurable policy for retaining or stripping assistant thinking blocks in replay history to reduce token usage.

## Best Possible Solution

Add a documented opt-in, provider-aware retention policy that composes with ProviderReplayPolicy and keeps current provider-safe defaults unless maintainers explicitly choose a new default.
