## Summary

Keep open. Current main has low-level thinking-block stripping and provider-gated replay sanitation, but it still lacks the requested user-visible history-retention config and current Claude 4.5+/future-model policy intentionally preserves prior thinking blocks for prompt-cache behavior.

## What This Changes

Request a configurable policy to strip `thinking` and `redacted_thinking` blocks from assistant history using `last-only`, `strip`, or `keep` modes for token savings and fewer replay errors.

## Best Possible Solution

Add a documented opt-in, provider-aware retention policy that composes with `ProviderReplayPolicy` and keeps current provider-safe defaults unless maintainers choose a new default.

## Reproduction Assessment

Not applicable as a feature/config request. Source inspection gives high confidence that the requested history-retention knob is absent and current thinking stripping is only provider-policy driven.

## Solution Assessment

Unclear. The requested `last-only` mode is plausible, but making it the default is not clearly safest because current Claude 4.5+/future-model handling deliberately preserves prior thinking blocks for prompt-cache prefix matching.
