## What This Changes

The issue requests a configurable policy for stripping assistant thinking/redacted_thinking blocks from conversation history before API replay, usually keeping only the latest assistant thinking blocks for token savings.

## Best Possible Solution

Track the remaining design in https://github.com/openclaw/openclaw/issues/54496 so replay pruning, durable transcript persistence, memory-facing history, and provider-required signed reasoning state are specified as one contract.
