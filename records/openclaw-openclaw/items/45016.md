## Summary

Keep open. Current main has low-level thinking-block stripping and provider-gated replay sanitation, but it still lacks the requested user-visible `thinkingHistory` retention policy, and the current Claude 4.x policy intentionally preserves prior thinking blocks for prompt-cache behavior.

## What This Changes

Request adding a configurable retention policy for assistant `thinking` and `redacted_thinking` blocks in conversation history to reduce replay token use.

## Best Possible Solution

Add a documented opt-in, provider-aware retention policy that composes with existing replay policy rather than silently changing provider-safe defaults.

## Reproduction Assessment

Yes. Source inspection and targeted search show the requested config key is absent, while existing stripping only occurs through provider replay policy; a live token measurement is not needed to prove the implementation gap.

## Solution Assessment

Unclear. The requested `last-only` behavior is useful, but making it the default is not clearly safest because current Claude 4.x handling deliberately preserves prior thinking blocks for cache stability.
