Full review comments:

- **[P2] Read the reactor from `reaction.key`:** `extensions/whatsapp/src/inbound/monitor.ts:453-455`
  - body: Baileys 7.0.0-rc11 emits `messages.reaction` as `{ key, reaction }`, with the incoming reaction message key stored on `reaction.key`; there is no top-level `userJid`. As written, group reactions without that invented field fall through to `reactorJid = undefined` and are dropped on the next line, so derive reactor identity from `reaction.key` and keep `key` as the reacted-message target.
  - confidence: 0.91
- **[P2] Clear dedupe state when reactions are removed:** `extensions/whatsapp/src/inbound/monitor.ts:443`
  - body: The empty-emoji removal path returns before the composite dedupe key can be cleared. If the same user removes and reapplies the same emoji within the inbound TTL, the previous add remains cached and the second real add is silently dropped.
  - confidence: 0.86
