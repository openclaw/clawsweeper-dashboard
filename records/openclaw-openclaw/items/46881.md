## Summary

Keep open: the feature direction is useful, but the branch still lacks real WhatsApp proof, is stale/conflicting against current main, and has source-backed correctness blockers for reaction actor identity and removal/reapply dedupe.

## What This Changes

The PR adds WhatsApp inbound reaction fields and a Baileys `messages.reaction` listener that forwards non-empty reactions to the agent through `onMessage`.

## Best Possible Solution

Define one WhatsApp-owned reaction ingress design that uses Baileys `reaction.key`, handles add and removal semantics, preserves access/privacy gates, and decides whether reactions become agent turns, plugin hooks, or queryable state before merging.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not as a broken existing-behavior bug; this is a feature PR. Source inspection and the pinned Baileys contract give high confidence for the two PR defects, but there is no real WhatsApp proof artifact.

## Solution Assessment

Is this the best way to solve the issue? No. The direction is useful, but the patch should not merge until it uses `reaction.key` for actor identity, handles removal/reapply semantics, rebases onto current main, and aligns the public reaction surface with the open hook/query trackers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Read the reactor from reaction.key:** `extensions/whatsapp/src/inbound/monitor.ts:453`
  - body: Baileys emits `messages.reaction` as `{ key, reaction }`; there is no top-level `userJid`, and the reaction event key is on `reaction.key`. This handler skips group reactions when `userJid` is absent, so valid reaction events can be dropped before access control.
  - confidence: 0.95
- **[P2] Clear dedupe state when a reaction is removed:** `extensions/whatsapp/src/inbound/monitor.ts:443-459`
  - body: Empty reaction-removal events return before the dedupe key is computed or cleared, while add events cache the same account/chat/reactor/message/emoji key for 20 minutes. Removing and reapplying the same emoji within that TTL can drop the second real reaction.
  - confidence: 0.88
