## Summary

Keep open: this is a useful WhatsApp reaction-ingress feature PR, but it still has two source-backed correctness defects, no after-fix proof from a real WhatsApp setup, and overlaps open reaction-hook/query design work that needs maintainer alignment before merge.

## What This Changes

The PR adds `reactionEmoji`/`reactionMessageId` fields and a WhatsApp `messages.reaction` listener that forwards non-empty reactions to the agent through `onMessage`.

## Best Possible Solution

Define one WhatsApp-owned reaction ingress design that uses Baileys `reaction.key`, handles add/removal semantics, preserves access and privacy gates, and decides whether reactions become agent turns, plugin hooks, queryable state, or a combination before merging.

## Reproduction Assessment

No, not as a broken existing-behavior bug; this is a feature PR. The PR defects are high-confidence from source inspection against the pinned Baileys contract, but there is no real WhatsApp proof artifact.

## Solution Assessment

No. The direction is useful, but the patch should not merge until it reads actor identity from `reaction.key`, handles removal/reapply semantics, rebases onto current main, and aligns the public reaction surface with the open hook/query trackers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Read the reactor from reaction.key:** `extensions/whatsapp/src/inbound/monitor.ts:453-455`
  - body: Baileys emits `messages.reaction` as `{ key, reaction }`; there is no top-level `userJid`, and the reaction event key is on `reaction.key`. This handler leaves `reactorJid` undefined for valid group reaction events and drops them at the group guard before access control.
  - confidence: 0.95
- **[P2] Clear dedupe state when a reaction is removed:** `extensions/whatsapp/src/inbound/monitor.ts:443-459`
  - body: Empty reaction-removal events return before the dedupe key is computed or cleared, while add events cache the same account/chat/reactor/message/emoji key for 20 minutes. Removing and reapplying the same emoji within that TTL can drop the second real reaction.
  - confidence: 0.88
