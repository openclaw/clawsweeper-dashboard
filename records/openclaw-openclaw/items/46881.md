Full review comments:

- **[P2] Read reactor identity from `reaction.key`:** `extensions/whatsapp/src/inbound/monitor.ts:453-455`
  - body: Baileys emits `messages.reaction` entries as `{ key, reaction }`, with the actor represented by the reaction key semantics; there is no top-level `userJid` in the pinned contract. In groups this makes `reactorJid` undefined and the handler skips valid reactions, while DMs can misclassify self/contact reactions.
  - confidence: 0.9
- **[P2] Evict dedupe state when a reaction is removed:** `extensions/whatsapp/src/inbound/monitor.ts:443`
  - body: An empty emoji means the reaction was removed, but this path continues before computing or clearing the dedupe key. If the same user reapplies the same emoji to the same message within the 20-minute TTL, the new reaction is treated as a duplicate and never reaches the agent.
  - confidence: 0.84
