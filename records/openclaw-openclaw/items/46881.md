## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Gate reaction events before starting agent turns:** `extensions/whatsapp/src/inbound/monitor.ts:542`
  - body: Current turn semantics treat reactions as non-turn events, but this handler sends every accepted reaction through `onMessage`, which can create unexpected agent runs and replies for existing WhatsApp users. Gate this behind an explicit product choice or route it through a passive reaction hook/ingress path before merging.
  - confidence: 0.86
- **[P2] Read the reactor from the Baileys reaction payload:** `extensions/whatsapp/src/inbound/monitor.ts:453-455`
  - body: Baileys 7.0.0-rc12 emits `messages.reaction` as `{ key, reaction }`; there is no top-level `userJid`. As written, group reactions fall through to `reactorJid = undefined` and are dropped, so derive the actor from the actual reaction payload instead of an invented field.
  - confidence: 0.93
- **[P2] Clear dedupe state when reactions are removed:** `extensions/whatsapp/src/inbound/monitor.ts:443`
  - body: The empty-emoji removal path returns before the composite dedupe key can be cleared. If the same user removes and reapplies the same emoji within the inbound TTL, the previous add remains cached and the second real add is silently dropped.
  - confidence: 0.87
