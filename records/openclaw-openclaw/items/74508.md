## Summary

Keep open. The Telegram overflow bug is real on current main and the PR moves in the right direction, but the current head still does not fix the claimed fork-session prompt behavior for file-backed CLI backends because the execute path never creates a prompt file when `useResume` is true.

## What This Changes

The PR changes Telegram draft streaming to chain overflowing reasoning previews across messages with rate-limit/backoff handling, suppresses partial reasoning-tag answer flicker, and adjusts CLI helper argument construction for fork-session resume system prompts.

## Best Possible Solution

Keep the Telegram overflow chaining work, but finish the fork-resume prompt fix in the runtime execute path for file-backed backends, rebase onto current main, and add focused tests plus the required changelog entry before merge.

## Solution Assessment
