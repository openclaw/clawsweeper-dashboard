## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Make 429 backoff apply to every stream on the account:** `extensions/telegram/src/draft-stream.ts:479-486`
  - body: `onTelegramRateLimit` updates module-global adaptive state, but no send path reads `getAdaptiveThrottleMs()`; only this stream's `rateLimitedUntilMs` gates retries. A 429 from one Telegram stream therefore does not pause other streams using the same bot token, so the account-wide retry_after behavior described by the PR can still burst from concurrent sessions.
  - confidence: 0.87
- **[P2] Drop the stale config baseline hash changes:** `docs/.generated/config-baseline.sha256:1-4`
  - body: This PR changes generated config baseline hashes without changing the config schema or baseline generator. Since `docs/.generated/README.md` says these hashes are generated drift artifacts, keeping these branch-local values will leave config docs checks or release checks fighting stale metadata after rebase.
  - confidence: 0.8
