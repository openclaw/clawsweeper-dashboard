Yes. Current main has source-visible paths for the overflow stop, independent Telegram throttlers, CLI fallback bypass, and fork-resume prompt omission, and the PR regression is source-reproducible from the gate being acquired before no-op checks.

## Solution Assessment

No, not as patched. The overall direction is maintainable, but the chat gate should be moved to the actual Telegram API calls and the stale config baseline hash change should be rebased or dropped.
