## Summary

Keep open. Current main has the session/subagent lineage data needed for a hierarchy, but the inspected CLI, slash-command, docs, task, and Control UI surfaces still present flat lists rather than the requested active-agent tree.

## What This Changes

The issue requests a new `openclaw tree` inspection command with text/JSON output, optional watch mode, session details, and resource usage for active agents and subagents.

## Best Possible Solution

Add a stable core hierarchy inspection surface that renders existing session/subagent lineage with text and `--json` output first, backed by focused CLI/docs/tests; treat live watch mode and per-process resource metrics as follow-up scope unless maintainers define those contracts upfront.

## Reproduction Assessment

Not applicable as a feature request. The absence is high-confidence from current-main CLI registration, generated CLI docs, sessions command output, slash-command behavior, and Control UI session rendering.

## Solution Assessment

Unclear. A tree view is a reasonable product direction, but the exact top-level `openclaw tree` shape plus watch/resource flags is not yet clearly the narrowest maintainable first step.
