## Summary

Keep open. Current main has the session and subagent lineage data needed to build this, but the CLI, docs, slash-command, and user-facing list surfaces still do not provide an active-agent tree view.

## What This Changes

The request is to add an `openclaw tree` command with text, JSON, watch, session detail, and resource-usage views for active agents and parent-child session relationships.

## Best Possible Solution

Add a core inspection surface that renders existing session/subagent lineage as a hierarchy with stable text and `--json` output first, then document and test that contract. Treat live watch mode and per-process resource metrics as follow-up work unless maintainers explicitly define those scopes upfront.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The absence is high-confidence from current-main command registration, sessions output code, subagent list rendering, and docs.

## Solution Assessment

Unclear. A hierarchy view is a reasonable product direction, but the exact command namespace and the watch/resource-metrics scope need maintainer direction before an automated implementation is safe.
