## Summary

Keep open. Current main still has the session/subagent lineage data needed for a hierarchy, but the CLI, docs, slash-command, and UI surfaces remain list-oriented and do not provide the requested active-agent tree view.

## What This Changes

Request a tree-style inspection command or view that shows active agents, subagent parent-child relationships, session status, optional resource usage, JSON output, and live watch mode.

## Best Possible Solution

Add a stable core inspection surface that renders existing session/subagent lineage as a hierarchy with text and `--json` output first, with focused CLI/docs/tests; defer live watch mode and per-process resource metrics unless maintainers define those contracts upfront.

## Reproduction Assessment

Not applicable as a feature request. The absence is high-confidence from current-main command registration, docs, sessions output code, subagent list rendering, and UI session controls.

## Solution Assessment

Unclear. A tree view is a reasonable product direction, but the exact command namespace and whether watch/resource metrics belong in the MVP require maintainer direction; the safest first step is a hierarchy renderer over existing session rows.
