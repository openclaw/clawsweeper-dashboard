## Summary

Keep open. Current main has useful session/subagent lineage, status, timing, token, and cost data, plus flat `/subagents` and `openclaw sessions` inspection paths, but it still does not provide the requested CLI/TUI hierarchy or active-agent tree surface.

## What This Changes

The issue requests an `openclaw tree`-style command that visualizes active agents and subagents with hierarchy, JSON output, watch mode, session details, and resource usage.

## Best Possible Solution

Add a core inspection surface that renders the existing session/subagent lineage as stable text and `--json` hierarchy first, with focused CLI docs and tests. Treat live watch mode and per-process resource metrics as follow-up work unless maintainers define those contracts up front.
