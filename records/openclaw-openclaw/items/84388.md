## Summary

## What This Changes

The branch changes the bundled Codex plugin to disable Codex-native execution under active OpenClaw sandboxing by default, expose sandbox-backed shell dynamic tools, and add an experimental sandbox exec-server opt-in with docs and tests.

## Best Possible Solution

Land only after maintainer/security approval accepts the fail-closed default and the experimental exec-server bridge scope; keep full Codex app-server containment tracked separately if that remains required.

## Solution Assessment

Mostly yes: the fail-closed default is the conservative security mitigation and the exec-server bridge is explicit and experimental. The remaining question is whether maintainers accept that scope or require the entire Codex app-server/orchestrator to run inside the agent sandbox.
