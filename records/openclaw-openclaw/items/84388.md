## Summary

## What This Changes

The PR disables Codex app-server native Code Mode, user MCP, and app-backed plugin execution during active OpenClaw sandboxes, exposes `sandbox_exec`/`sandbox_process`, updates Codex sandbox/context tests and docs, and omits an update compatibility env var when no candidate version is available.

## Best Possible Solution

Land a maintainer-approved Codex sandbox contract: active OpenClaw sandboxes should either run native Codex execution inside the sandbox or fail closed to OpenClaw-owned sandbox tools with clear docs and upgrade expectations.

## Solution Assessment

Yes, if maintainers choose fail-closed behavior: the patch disables host-native Codex surfaces under active OpenClaw sandboxing and routes shell access through sandbox-backed dynamic tools. The main alternative is a larger app-server-in-sandbox design.
