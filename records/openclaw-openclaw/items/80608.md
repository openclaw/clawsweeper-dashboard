## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Restore the skill-workshop apply import:** `extensions/skill-workshop/src/tool.ts:8`
  - body: This import was changed to `applyProposalToWorkshop`, but `skills.ts` still exports `applyProposalToWorkspace` and this file still calls `applyProposalToWorkspace` below. The modified plugin will fail to typecheck before any curator behavior can run.
  - confidence: 0.96
- **[P1] Use the actual lifecycle and scheduling contracts:** `extensions/skill-curator/src/index.ts:16-32`
  - body: The current SDK does not accept `activate`/`deactivate` lifecycle callbacks or `schedule`/`run` scheduler callbacks here; lifecycle registrations need an `id`, and session scheduler jobs are cleanup metadata, not hourly runners. This both fails the contract and means the advertised automatic curator tick will not execute.
  - confidence: 0.94
- **[P1] Constrain usage skill names before moving directories:** `extensions/skill-curator/src/run.ts:86-90`
  - body: `determineAllTransitions` returns names from the workspace `.usage.json`, but `moveSkillToArchive` joins that unvalidated name directly into paths. A malformed sidecar key such as `../../x` can make a curator run rename paths outside `skills/`; validate the key and resolve within the skills root before moving anything.
  - confidence: 0.88
- **[P2] Do not write snapshots inside the archived tree:** `extensions/skill-curator/src/snapshot.ts:67`
  - body: The snapshot tarball is created under `skills/.curator_backups` while the command archives the whole `skills` directory. That makes the backup include prior backups and the file being written, which can make tar fail or produce rapidly growing archives before any mutation is applied.
  - confidence: 0.9
- **[P2] Wire the reviewer into curator runs:** `extensions/skill-curator/src/run.ts:189-212`
  - body: The PR adds reviewer parsing and prompt code, but `curatorRun` only computes age-based transitions and applies them; it never builds a review manifest, calls an auxiliary model, or applies parsed review decisions. The advertised LLM patch/consolidate/archive pass is currently a no-op.
  - confidence: 0.84
