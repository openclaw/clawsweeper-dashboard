Full review comments:

- **[P1] Register curator commands through the CLI contract:** `extensions/skill-curator/src/cli.ts:98-106`
  - body: `api.registerCommand` only accepts a flat command definition with a `handler`; it does not support `subcommands`, `run`, `options`, `display`, or `ctx.options`. The advertised curator shell subcommands will not register under the current plugin command contract.
  - confidence: 0.96
- **[P1] Use a real runtime lifecycle registration:** `extensions/skill-curator/src/index.ts:19-26`
  - body: `registerRuntimeLifecycle` requires an `id` and optional `cleanup` callback. `activate` and `deactivate` are not current lifecycle callbacks, so this registration is outside the host contract.
  - confidence: 0.94
- **[P1] Constrain skill names before moving directories:** `extensions/skill-curator/src/run.ts:86-100`
  - body: `moveSkillToArchive` joins the sidecar or CLI skill name directly into source and destination paths before `fs.rename`. A crafted key such as `../../outside` can move directories outside `skills/`; normalize the name and verify resolved paths stay under the skills root.
  - confidence: 0.9
- **[P1] Validate rollback snapshot ids before extracting:** `extensions/skill-curator/src/cli.ts:285`
  - body: The rollback id is joined into the backup path and the resulting tarball is extracted after the current skills tree is cleaned. Reject traversal-like ids and require the resolved archive path to remain inside `.curator_backups`.
  - confidence: 0.88
- **[P2] Keep skill-workshop off curator private internals:** `extensions/skill-workshop/src/tool.ts:24-30`
  - body: This lazy import reaches from `skill-workshop` into another plugin's private `src/telemetry.js`, which breaks the bundled-plugin boundary. Expose a public API seam or keep the integration inside curator.
  - confidence: 0.87
- **[P2] Register long-lived hooks with live-config guardrails:** `extensions/skill-curator/src/index.ts:30-35`
  - body: Adding a new bundled `agent_end` handler needs the hook allowlist and live-config guard patterns. Reading only startup `api.pluginConfig` means runtime config changes may not stop automatic curator mutations.
  - confidence: 0.83
