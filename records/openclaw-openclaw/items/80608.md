## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Register curator through the command contract:** `extensions/skill-curator/src/cli.ts:98-106`
  - body: `registerCommand` requires a flat command definition with a `handler`; it does not support `subcommands`, `run`, `options`, `display`, or `ctx.options`. As written, the advertised curator subcommands will not satisfy the current plugin command contract.
  - confidence: 0.96
- **[P1] Use the real runtime lifecycle shape:** `extensions/skill-curator/src/index.ts:19-26`
  - body: `registerRuntimeLifecycle` requires an `id` and only registers cleanup metadata. `activate` and `deactivate` are not lifecycle callbacks here, so the host will diagnose a missing id and ignore the intended callbacks.
  - confidence: 0.94
- **[P1] Constrain skill paths before moving directories:** `extensions/skill-curator/src/run.ts:88-100`
  - body: `moveSkillToArchive` joins the sidecar skill key directly into source and destination paths before `fs.rename`. A crafted `.usage.json` key or CLI-provided name containing traversal can move directories outside `skills/`; normalize names and verify resolved paths stay inside the skills root.
  - confidence: 0.9
- **[P2] Keep skill-workshop off curator private internals:** `extensions/skill-workshop/src/tool.ts:24-30`
  - body: This lazy import reaches from `skill-workshop` into another plugin's private `src/telemetry.js`, which violates the extension boundary and couples two bundled plugins by private files. Expose a public API seam or keep the integration inside the owning plugin.
  - confidence: 0.87
- **[P2] Resolve live config before automatic mutations:** `extensions/skill-curator/src/index.ts:34-35`
  - body: The long-lived `agent_end` handler reads startup `api.pluginConfig`, so disabling or changing the plugin config at runtime may not stop automatic curator mutations. Follow the live-config pattern used by skill-workshop before running the curator.
  - confidence: 0.83
- **[P2] Constrain rollback snapshot ids:** `extensions/skill-curator/src/cli.ts:285-313`
  - body: Rollback builds `archivePath` from `ctx.options.id` with `path.join` and then extracts it. A traversal id can point outside `.curator_backups`; resolve the requested snapshot under the backup root and reject anything outside it before access or extraction.
  - confidence: 0.74
