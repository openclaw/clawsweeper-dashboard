## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Register curator commands through the CLI contract:** `extensions/skill-curator/src/cli.ts:98-106`
  - body: `api.registerCommand` only accepts a flat command with a `handler`; it does not support `subcommands`, `run`, `options`, `display`, or `ctx.options`. This will fail typechecking and the advertised `openclaw curator ...` commands will not be registered; use `api.registerCli` for shell subcommands or implement one chat command that parses `ctx.args`.
  - confidence: 0.96
- **[P1] Use a real runtime lifecycle registration:** `extensions/skill-curator/src/index.ts:19-26`
  - body: `registerRuntimeLifecycle` requires an `id` and only registers cleanup metadata; `activate` and `deactivate` are not lifecycle callbacks in the current SDK. This object is outside the host contract and will not typecheck or register the intended lifecycle behavior.
  - confidence: 0.94
- **[P1] Constrain sidecar skill names before moving directories:** `extensions/skill-curator/src/run.ts:86-100`
  - body: `determineAllTransitions` returns names from workspace `.usage.json`, but `moveSkillToArchive` joins that untrusted key directly into `srcDir` and `dstDir`. A malformed sidecar key such as `../../outside` can make a curator run rename paths outside `skills/`; validate the normalized name and resolve within the skills root before any filesystem move.
  - confidence: 0.9
- **[P2] Keep skill-workshop off curator private internals:** `extensions/skill-workshop/src/tool.ts:24-30`
  - body: This lazy import reaches from `skill-workshop` into another extension's private `src/telemetry.js`, which breaks the bundled plugin boundary and makes workshop behavior depend on curator internals. Expose a public plugin/API seam or keep this integration inside curator instead of coupling two private extension trees.
  - confidence: 0.87
- **[P2] Register long-lived hooks with live-config guardrails:** `extensions/skill-curator/src/index.ts:30-35`
  - body: Adding a new bundled `api.on("agent_end")` handler needs the hook allowlist/update path and should resolve live plugin config inside the handler. As written it reads startup `api.pluginConfig`, so disabling or changing the plugin config at runtime would not stop the automatic curator mutations.
  - confidence: 0.83
