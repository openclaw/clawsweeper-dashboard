Overall confidence: 0.93

Full review comments:

- **[P1] Gate skill deletion behind workshop approval:** `extensions/skill-workshop/src/tool.ts:322`
  - body: This adds a model-callable `delete` action that removes a workspace skill directory, but the existing trusted tool policy only asks for approval when `event.params.action === "apply"`. Users with the default pending approval policy would still let the agent delete skills without the same operator confirmation that protects writes, so deletion needs its own approval gate or should stay out of `skill_workshop`.
  - confidence: 0.94
- **[P1] Keep workshop integration behind a public seam:** `extensions/skill-workshop/src/tool.ts:27`
  - body: `skill-workshop` now imports `../../skill-curator/src/telemetry.js`, which is another plugin's private source file. Bundled and external plugin packaging relies on manifest/SDK contracts rather than cross-plugin `src/**` imports, so this should be replaced by a public SDK/workshop seam or kept entirely inside the curator plugin.
  - confidence: 0.91
- **[P2] Honor or remove the idle-hours option:** `extensions/skill-curator/openclaw.plugin.json:44-48`
  - body: The manifest exposes `min_idle_hours`, but the runtime decision path only checks pause state, first run, and interval; `agent_end` calls `curatorRun` immediately after turns without using this option. Operators who configure the idle guard would still get curator work as soon as the interval is met, so the option needs implementation or removal before shipping.
  - confidence: 0.88
- **[P2] Register the declared curator tool:** `extensions/skill-curator/openclaw.plugin.json:9`
  - body: The manifest declares a `skill_curator` agent tool, but the plugin entry only registers CLI commands and lifecycle hooks. Tool discovery uses `contracts.tools` as the ownership contract, so declaring a tool that never registers creates a stale tool surface; either register the tool or remove the contract entry.
  - confidence: 0.84
