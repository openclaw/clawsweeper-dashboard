## What This Changes

The PR adds an `openclaw models custom-test` CLI command that probes configured custom model providers and exports onboarding verification helpers for reuse.

## Best Possible Solution

Extend the maintained runtime probe path to cover custom-provider model verification, with redacted parseable JSON, correct exit codes, API-resolution parity, docs/tests, and contributor real-run proof.

Yes. Source inspection of the PR head gives high-confidence reproduction paths: run with `--json`, a failed probe, fractional `--concurrency`, a misspelled `--provider`, Responses/per-model API config, or a sensitive `baseUrl`.

## Solution Assessment

No. A custom-provider verification command is useful, but this implementation duplicates and bypasses the maintained runtime probe path; the safer solution is to reuse or extend `models status --probe` semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Redact provider URLs before JSON output:** `src/commands/models/custom-test.ts:176`
  - body: Each result stores the raw `baseUrl`, and JSON mode writes the raw `results` array. Provider URLs can include embedded credentials, token query parameters, or private endpoint topology, so emit a redacted display value or omit the URL from machine output.
  - confidence: 0.94
- **[P1] Probe through the runtime transport:** `src/commands/models/custom-test.ts:168-172`
  - body: The command calls onboarding fetch helpers with only `baseUrl`, `apiKey`, and `modelId`, bypassing configured headers, auth overrides, local service startup, proxy/TLS settings, and private-network policy. This can mark working runtime configurations as broken or give false passes for configurations that fail through the real model path.
  - confidence: 0.91
- **[P2] Keep JSON output parseable:** `src/commands/models/custom-test.ts:152-154`
  - body: `--json` still logs the progress line before writing the JSON payload, so stdout is no longer valid JSON for callers piping the command to `jq` or CI parsers. Suppress human logs or route them away from stdout in JSON mode.
  - confidence: 0.96
- **[P2] Exit non-zero for failed JSON probes:** `src/commands/models/custom-test.ts:216-218`
  - body: The JSON branch returns before `failCount` is computed and before `runtime.exit(1)` can run, so failed probes exit successfully in JSON mode. Automation using this health check would report a false pass.
  - confidence: 0.96
- **[P2] Use each model's effective API:** `src/commands/models/custom-test.ts:123-124`
  - body: Target collection only checks provider-level `api`, but model entries can override `api` and the current config supports Responses-family APIs. Build targets from `model.api ?? providerConfig.api` so mixed or Responses-backed custom providers are not skipped or probed with the wrong protocol.
  - confidence: 0.94
- **[P2] Reject fractional concurrency values:** `src/commands/models/custom-test.ts:100-102`
  - body: Values like `--concurrency 0.5` pass validation, but `Array.from({ length: Math.min(concurrency, targets.length) })` creates no workers, skips all probes, and returns success. Require a positive integer.
  - confidence: 0.9
- **[P2] Fail explicit provider filters that verify nothing:** `src/commands/models/custom-test.ts:142-148`
  - body: When `--provider` is misspelled or selects a provider with no collected targets, the command reports no providers and exits successfully. An explicit filter that tests nothing should fail so scripts do not get a false pass.
  - confidence: 0.91
