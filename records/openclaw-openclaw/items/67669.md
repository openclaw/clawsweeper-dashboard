Full review comments:

- **[P1] Rebase this command onto the lazy model CLI:** `src/cli/models-cli.ts:2-27`
  - body: Current main moved model command loading into `models-cli.runtime.ts` and no longer has the `src/commands/models.ts` barrel this branch edits. Rebase and register the command through the current lazy loader so the merge result does not reintroduce stale eager imports or fail on a removed barrel.
  - confidence: 0.91
- **[P1] Collect targets with each model's effective API:** `src/commands/models/custom-test.ts:123-124`
  - body: This filter only accepts provider-level `openai-completions`, `anthropic-messages`, and `azure-openai-responses`, but current config supports `openai-responses` and runtime catalog rows use `model.api ?? provider.api`. Responses-compatible custom providers and mixed-API providers will be skipped or probed with the wrong protocol.
  - confidence: 0.94
- **[P1] Route probes through the runtime model path:** `src/commands/models/custom-test.ts:45-56`
  - body: Calling the onboarding probe helpers directly ignores provider/model headers, `authHeader`, request overrides, proxy/TLS/private-network policy, and other runtime request semantics. That can make this diagnostic report false failures or false passes for valid configured providers.
  - confidence: 0.9
- **[P1] Keep JSON stdout parseable:** `src/commands/models/custom-test.ts:151-154`
  - body: The command writes the human progress line before checking `opts.json`, so `openclaw models custom-test --json` does not produce a clean JSON stdout stream. Existing model command docs reserve stdout for the JSON payload so scripts can pipe it directly to tools like `jq`.
  - confidence: 0.93
- **[P1] Return failure status in JSON mode:** `src/commands/models/custom-test.ts:216-218`
  - body: The JSON branch writes results and returns before computing `failCount` or calling `runtime.exit(1)`, so failed verifications still exit successfully. Automation should not need to parse the payload to recover the command's failure semantics.
  - confidence: 0.96
- **[P2] Redact provider base URLs from JSON results:** `src/commands/models/custom-test.ts:174-177`
  - body: Each result includes raw `baseUrl`, and this repo marks `models.providers.*.baseUrl` as sensitive URL configuration because it can expose internal hosts or embedded credentials. Omit or redact this field before users put JSON output in CI logs or support artifacts.
  - confidence: 0.9
- **[P2] Require integer concurrency values:** `src/commands/models/custom-test.ts:100-102`
  - body: `Number('0.5')` passes validation, but `Array.from({ length: 0.5 })` creates no workers, so the command can skip every target and return an empty success result. Parse this like the existing probe options and reject non-integer concurrency values.
  - confidence: 0.86
- **[P2] Fail when an explicit provider filter tests nothing:** `src/commands/models/custom-test.ts:142-148`
  - body: When `--provider` is misspelled or matches no collected targets, this branch reports no providers and exits successfully. A verification command should return nonzero when an explicit filter verifies zero models so CI cannot silently pass a typo.
  - confidence: 0.84
