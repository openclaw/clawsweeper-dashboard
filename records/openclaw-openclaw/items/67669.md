## Summary

Keep open. Current main still lacks the requested custom-provider test subcommand, and the PR branch has blocking correctness and security issues in JSON behavior, target selection, runtime probe fidelity, empty-filter/fractional-concurrency handling, and raw endpoint output.

## What This Changes

This PR adds an `openclaw models custom-test` CLI subcommand, exports custom onboarding verification helpers, and probes configured custom-provider models with provider filtering, concurrency, table output, and JSON output.

## Best Possible Solution

Keep the PR open and rework the feature around the maintained runtime probe path or a shared probe helper, preserving script-safe JSON, failure exits, effective API resolution, sensitive-field redaction, docs, changelog, and focused command tests.

## Reproduction Assessment

Yes. The blocking issues are reproducible by static inspection of the PR head: `--json` logs before writing JSON and returns before failure exits, `--concurrency 0.5` creates zero workers, and supported API shapes are skipped before any live network call is needed.

## Solution Assessment

No. A custom-provider verification command is a useful direction, but this implementation is not the narrowest maintainable solution because it bypasses the runtime probe semantics that already handle config, auth, transport, redaction, and failure classification.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Keep JSON output parseable:** `src/commands/models/custom-test.ts:152-154`
  - body: `--json` still emits the progress line before the JSON payload, so callers piping stdout to `jq` or CI parsers receive non-JSON text. Suppress human logs or route them away from stdout when JSON output is requested.
  - confidence: 0.96
- **[P2] Return non-zero from JSON failures:** `src/commands/models/custom-test.ts:216-218`
  - body: The JSON branch writes results and returns before `failCount` is computed or `runtime.exit(1)` runs. Failed probes therefore exit successfully in JSON mode, which makes automation report a false pass.
  - confidence: 0.96
- **[P2] Build targets from effective model APIs:** `src/commands/models/custom-test.ts:123-138`
  - body: Target collection only checks `providerConfig.api` and the allowlist omits supported Responses-family APIs. Custom providers using `openai-responses`, `openai-codex-responses`, or per-model `model.api` overrides are skipped or probed with the wrong protocol.
  - confidence: 0.94
- **[P2] Use the runtime probe transport:** `src/commands/models/custom-test.ts:168-173`
  - body: The probe calls onboarding helpers with only `baseUrl`, `apiKey`, and `modelId`, bypassing configured headers, auth overrides, request transport settings, and runtime network policy. That can report false health for providers that work through the normal runtime path.
  - confidence: 0.9
- **[P2] Fail empty explicit provider filters:** `src/commands/models/custom-test.ts:142-148`
  - body: When `--provider` is misspelled or selects a provider with no collected targets, the command logs an empty result and exits success. An explicit filter that verifies nothing should fail so scripts do not get a false pass.
  - confidence: 0.91
- **[P2] Reject fractional concurrency values:** `src/commands/models/custom-test.ts:100-103`
  - body: Values such as `--concurrency 0.5` pass validation, then `Array.from({ length: 0.5 })` starts no workers and the command can return success without probing any target. Require a positive integer.
  - confidence: 0.89
- **[P1] Redact provider URLs from JSON results:** `src/commands/models/custom-test.ts:176`
  - body: Each result includes raw `baseUrl`, and `--json` writes those results directly. Provider base URLs are treated as sensitive URL fields in this repo, so JSON output should omit or redact them.
  - confidence: 0.92
- **[P3] Document the new command before shipping:** `src/cli/models-cli.ts:299-303`
  - body: This adds a user-facing `openclaw models` subcommand, but the PR does not update the models CLI docs or changelog. Add the public docs and release note required for new CLI behavior.
  - confidence: 0.84
