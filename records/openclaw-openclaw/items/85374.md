## Summary

Keep open: current main includes the related stale-typing cleanup from https://github.com/openclaw/openclaw/pull/84565, but the source still allows session-derived terminal reconciliation to set the composer status to `done` and clear `chatStream` before the deferred history reload that can render the final assistant message completes.

## What This Changes

The issue asks WebChat to delay the done/run-complete indicator until the final assistant reply is visible in the conversation.

## Best Possible Solution

Defer terminal `chatRunStatus` publication for session-derived terminal reconciliation until the final chat event appends a message or the deferred `loadChatHistory` completes, while still clearing stale local run state safely.

## Reproduction Assessment

Yes, at source level: a terminal session patch for the active run can clear `chatRunId`, clear `chatStream`, and set `chatRunStatus.phase` to `done` before the async `loadChatHistory` path finishes rendering the final message. I did not run a live browser/Windows reproduction in this read-only pass.

## Solution Assessment

No, the current implementation is not the best end state because it couples clearing stale run state with immediately publishing the done indicator. The narrower maintainable fix is to separate cleanup from user-visible terminal status until final content is appended or history reload completes.
