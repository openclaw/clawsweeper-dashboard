## Summary

Keep open: current main still has a source-visible WebChat ordering path where terminal session reconciliation can show the done status before the final assistant message is appended/rendered. The merged stale-spinner fix is related useful context, but it does not prove this inverse early-done symptom is fixed.

## What This Changes

The issue requests that WebChat delay the run-complete/done indicator until the final assistant reply text is visible in the conversation.

## Best Possible Solution

Keep the terminal session reconciliation from the merged stale-spinner fix, but split local run cleanup from the visible done toast so WebChat only shows done after a final assistant payload or history reload has rendered the reply.

## Reproduction Assessment

No live high-confidence reproduction was established in this read-only review. The source-reproducible path is a terminal sessions.changed event clearing the active run and setting done before a later chat final event appends the assistant message.

## Solution Assessment

Yes, the maintainable fix is to reuse the existing reconciliation path but defer the visible done status until content render/history reload completion. A timeout or broad watchdog would be a less precise fix for this event-ordering bug.
