## Summary

Keep open: the one-line scoped probe is the right direction, but this branch leaves the SSH tunnel fallback matching only raw EADDRINUSE even though the callee wraps that condition in PortInUseError, so an occupied preferred port can now fail before the intended ephemeral-port fallback.

## What This Changes

The PR changes the SSH tunnel preferred-port preflight to call ensurePortAvailable with the IPv4 loopback host.

## Best Possible Solution

Merge one canonical SSH tunnel fix that scopes the preflight to 127.0.0.1, routes PortInUseError into the ephemeral fallback, and adds focused regression coverage.

## Reproduction Assessment

Yes. Source inspection shows the patched caller would receive PortInUseError from ensurePortAvailable while the unchanged fallback only handles raw EADDRINUSE; the PR body also supplies real Node output for the hostless versus scoped probe behavior.

## Solution Assessment

No as submitted. Scoping the probe is the right owner-boundary fix, but the complete fix also needs to route PortInUseError through the existing fallback and cover that path in tests.
