## Summary

Keep open: the host-scoped probe is the right direction, but this exact one-line patch can bypass the existing ephemeral-port fallback because `ensurePortAvailable` now throws `PortInUseError` rather than a raw `EADDRINUSE` errno.

## What This Changes

The branch changes `src/infra/ssh-tunnel.ts` so `startSshPortForward` calls `ensurePortAvailable(localPort, "127.0.0.1")` instead of a hostless probe.

## Best Possible Solution

Keep the scoped `127.0.0.1` probe, handle `PortInUseError` in the preferred-port catch path, and add regression coverage for falling back to an ephemeral `-L` port.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with the new scoped probe, `ensurePortAvailable` throws `PortInUseError`, while this PR's catch still only handles raw errno `EADDRINUSE`.

## Solution Assessment

No. Passing `127.0.0.1` is necessary, but the best fix also treats `PortInUseError` as a busy port so the existing ephemeral fallback still works.
