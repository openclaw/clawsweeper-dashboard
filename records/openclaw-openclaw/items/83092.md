## Summary

Keep open: the PR still misses the linked failure because the new resolver runs only inside the workspace-only guard, while the normal read tool bypasses that guard. In workspace-only mode, the resolved media-store path is still checked only against workspace/additional roots, so managed inbound media remains rejected.

## What This Changes

The PR imports the media-reference resolver and resolves `media://` path arguments inside `wrapToolWorkspaceRootGuardWithOptions` before sandbox validation.

## Best Possible Solution

Resolve supported `media://inbound/...` references inside the actual OpenClaw read execution path, preserve the managed inbound-media allowance, and add focused regression coverage for default and workspace-only reads.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main can surface `media://inbound/...` notes, and the default read tool sends that string to the base read implementation without media-reference resolution.

## Solution Assessment

No. Resolving inside the workspace guard is not the best fix because default reads bypass it and workspace-only reads still lack the inbound-media allowance; the maintainable fix belongs in the read path with policy-aware media handling.
