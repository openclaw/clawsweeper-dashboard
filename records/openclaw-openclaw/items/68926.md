## Summary

Keep this PR open, but it is not safe to merge as-is. It addresses a real Control UI gap related to #68959, and the earlier Greptile slash-menu and disabled-guard comments appear addressed in the latest diff, but the current head still has two blocking regressions in agent-session key synthesis and thinking-option resolution.

## What This Changes

The PR adds desktop and mobile Control UI chat agent selection, moves model and thinking selectors into the shared chat controls row, and adds CSS sizing for the new agent selector.

## Best Possible Solution

Rebase or rewrite the PR against current `main` so the agent selector lives with the shared chat session-control code, derives new agent session keys from a plain main key rather than a full `agent:*` key, preserves current model/thinking behavior, and adds focused coverage for non-agent legacy sessions plus session-advertised thinking levels.

## Reproduction Assessment

Yes. A static reproduction is enough: with a non-agent active key and `sessionDefaults.mainSessionKey` set to a full key, changing the PR's agent selector produces a nested `agent:*` key; with session/default thinking levels such as `adaptive`, `xhigh`, or `max`, the PR's thinking selector drops those options.

## Solution Assessment

No. A dedicated agent selector is a reasonable direction for #68959, but this implementation should reuse and extend the current shared session-control helper rather than duplicating older model/thinking logic and bypassing the current thinking metadata contract.
