## What This Changes

The PR routes `session.tool` gateway frames through the existing UI tool-stream handler and adds a regression for rendering externally started session-run tool cards without reloading chat history.

## Best Possible Solution

Land this narrow UI routing fix with the regression, leaving the existing session-key filtering and gateway `session.tool` contract intact.

## Reproduction Assessment

Yes, at source level: current main emits `session.tool` frames for subscribed sessions, while the UI gateway switch only routes `agent` frames into the tool-stream handler. I did not run local tests because this review is read-only.

## Solution Assessment

Yes. Reusing `handleAgentEvent` is the narrowest maintainable fix because it preserves the existing session-key filter, tool-call dedupe, and tool-stream rendering path rather than adding a parallel session-tool implementation.
