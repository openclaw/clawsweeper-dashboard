## Summary

Keep open for normal merge review: current main emits `session.tool` frames for subscribed operator UIs, but the Control UI only routes `agent` frames into the tool-stream handler. The proposed patch is focused, reuses the existing session-filtered tool handler, includes a regression, and has sufficient terminal real-behavior proof; I found no blocking correctness or security findings.

## What This Changes

The PR routes `session.tool` gateway events into the existing Control UI tool-stream handler and adds a regression for externally started session runs rendering a live tool card without a history reload.

## Best Possible Solution

Land the focused UI routing fix with its regression once normal PR checks are green, then close the linked issue after merge.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main can receive `session.tool` frames through `sessions.subscribe`, but the Control UI event dispatcher does not route that event into the tool-stream state handler. I did not execute the repro because this review is read-only.

## Solution Assessment

Yes. Reusing `handleAgentEvent` is the narrow maintainable fix because that handler already owns tool-stream state and session filtering; adding a separate `session.tool` parser would duplicate behavior.
