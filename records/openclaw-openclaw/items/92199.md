## Summary

Current main and the latest release still flush and reset ACP `final_only` output at each intermediate completion, matching the reported message-loss mechanism. The issue should remain open while the overlapping open fix PR is corrected, rebased, and proven through a real ACP transport path.

## What This Changes

Preserve intermediate ACP reply text across tool-call boundaries so WeChat users do not lose progress messages.

## Best Possible Solution

Land one canonical ACP projector fix that preserves all turn-scoped `final_only` state through intermediate invocation boundaries, emits one consolidated completion, and includes focused regression coverage plus redacted real ACP-to-channel proof.

## Reproduction Assessment

Yes at source level: send visible text across multiple ACP completions in `final_only` mode and current main force-flushes and resets each segment independently. This review did not independently run a live WeChat reproduction.

## Solution Assessment

Yes in direction: keeping turn-scoped projector state until the dispatcher's outer completion is the narrowest maintainable fix. The canonical patch must also preserve tool/status delivery and output-budget invariants and prove the actual transport result.
