## Summary

Keep this PR open for normal review and merge gating. The diff is a narrow Discord plugin fix for a current-main regression path, and I did not find functional or security blockers in the proposed change.

## What This Changes

The PR changes Discord message processing so configured ack reactions can send on guild/channel `message_tool_only` routes while keeping status lifecycle reactions suppressed, updates focused coverage, and adds a changelog entry.

## Best Possible Solution

Land the narrow Discord plugin change after exact-head CI and mergeability are green, preserving the focused regression test and changelog entry for #74922.

## Reproduction Assessment

Yes. Current main has a focused Discord process test path where an always-on guild/channel route with `ackReactionScope: "all"` resolves to `message_tool_only` and asserts that no reaction is sent.

## Solution Assessment

Yes. Decoupling the initial ack gate from tool-only source delivery while leaving `!sourceRepliesAreToolOnly` on `statusReactionsEnabled` is the narrow maintainable fix because the existing helper already supports plain acks when status reactions are disabled.
