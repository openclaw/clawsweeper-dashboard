This PR decouples Discord’s configured ack reaction from tool-only source-reply suppression while keeping lifecycle/status reactions disabled for tool-only guild routes and updates the focused process test.

## Best Possible Solution

Yes. Current main has a focused source/test reproduction: an always-on Discord guild route with `ackReactionScope: "all"` resolves to `message_tool_only` and currently asserts no reaction call.

## Solution Assessment

Yes, with one process fix. Decoupling the initial configured ack from `sourceRepliesAreToolOnly` while keeping `statusReactionsEnabled` gated on `!sourceRepliesAreToolOnly` is the narrowest maintainable code change for the reported behavior.
