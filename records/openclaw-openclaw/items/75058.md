## Summary

Keep open. Current main still relies on bounded identifier extraction, file-operation suffixes, and model-facing preservation instructions rather than the deterministic exact path-literal safeguard described by the linked open PR, which uses closing syntax for this issue.

## What This Changes

Preserve slash-separated file paths exactly through context compaction so later tool calls do not receive corrupted path strings.

## Best Possible Solution

Review and land PR #75093, or an equivalent narrow safeguard, so bounded path-like literals are preserved exactly without exposing hidden tool-result details.
