## Summary

Keep open. Current main still depends on model/provider instructions, bounded identifier auditing, and file-operation suffixes rather than a deterministic exact path-literal safeguard, and open PR #75093 already uses closing syntax for this issue.

## What This Changes

The issue reports that context compaction can corrupt slash-separated file paths by inserting spaces, causing later tool calls to receive invalid paths.

## Best Possible Solution

Review and land PR #75093, or an equivalent narrow safeguard, so bounded path-like literals are preserved exactly without exposing hidden tool-result details.

## Reproduction Assessment

No. The issue has concrete failed tool-call paths and current-main source makes the failure plausible, but a high-confidence reproduction would need the original long transcript or a deterministic MiniMax compaction replay.

## Solution Assessment

Yes. Deterministic path-literal preservation in the compaction safeguard is the narrow maintainable direction, and it is safer than relying on provider prompt adherence alone.
