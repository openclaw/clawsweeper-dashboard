## Summary

Keep open. The reporter logs show a plausible compaction corruption path, current main still relies on bounded identifier extraction and prompt/audit behavior rather than deterministic exact path-literal preservation, and PR #75093 is the direct open implementation candidate with closing syntax.

## What This Changes

The issue reports that conversation compaction can corrupt slash-separated file paths by inserting spaces, causing later tool calls to use nonexistent paths.

## Best Possible Solution

Review and land PR #75093, or an equivalent narrow compaction safeguard, so exact path-like literals survive compaction with regression coverage.

## Reproduction Assessment

No. The reporter logs are concrete, but a high-confidence reproduction would need the original long transcript or a deterministic MiniMax compaction replay against current main.

## Solution Assessment

Yes. Deterministic path-literal preservation in the compaction safeguard is the narrow maintainable direction, and it is safer than relying on provider prompt adherence alone.
