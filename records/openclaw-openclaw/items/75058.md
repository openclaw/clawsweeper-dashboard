The issue reports that compaction can corrupt slash-separated file paths by inserting spaces, causing subsequent file-tool calls to fail with ENOENT.

## Best Possible Solution

## Solution Assessment

Yes for the current direction. A bounded deterministic exact-path suffix in the existing compaction safeguard is narrower than a provider-specific workaround or relying only on summarizer prompt adherence.
