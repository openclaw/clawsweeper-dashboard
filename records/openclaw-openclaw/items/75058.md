## Summary

Keep open. The report is fresh and credible, current main still lacks deterministic exact path-literal preservation through compaction, and open PR #75093 is already the direct implementation candidate with closing syntax, so this issue should remain open until that PR lands or is closed.

## What This Changes

The issue requests that context compaction preserve slash-separated file paths exactly instead of corrupting `/` into spaces in later tool-call parameters.

## Best Possible Solution

Review and land PR #75093, or an equivalent narrow compaction safeguard change, so path-like literals needed for future tool calls are collected from model-facing compaction input and appended deterministically with regression coverage proving slash-separated paths survive unchanged.

## Reproduction Assessment

No. A high-confidence end-to-end reproduction would need the original long transcript or a deterministic MiniMax compaction replay; the attached raw tool logs and current source review make the failure credible but not locally replayable.

## Solution Assessment

Yes. Deterministic path-literal preservation in the compaction/safeguard pipeline is the narrow maintainable fix; it is safer than relying on provider prompt adherence or broad provider-specific long-context behavior changes.
