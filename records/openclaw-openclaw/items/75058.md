## Summary

Keep open. The report is fresh and credible, current main still relies on prompt/quality-guard heuristics rather than deterministic exact path-literal preservation, and PR #75093 is the open linked implementation candidate for this issue.

## What This Changes

Preserve exact slash-separated file paths through conversation compaction so later tool calls do not receive paths with inserted spaces.

## Best Possible Solution

Review and land PR #75093, or an equivalent narrow compaction safeguard change, so model-facing compaction input preserves bounded exact path-like literals with regression coverage for slash-separated paths.

## Reproduction Assessment

No. The reporter logs are concrete, but a high-confidence end-to-end reproduction would need the original long transcript or a deterministic MiniMax compaction replay; current source review explains why the failure remains plausible.

## Solution Assessment

Yes. Deterministic path-literal preservation in the compaction safeguard is the narrow maintainable direction, and it is safer than relying on provider prompt adherence or provider-specific long-context behavior changes.
