## Summary

Keep open. Current main still relies on compaction instructions, identifier quality checks, and bounded suffix metadata rather than deterministic exact path-literal preservation, and PR #75093 is the open implementation candidate that directly references this issue with closing syntax.

## What This Changes

The issue asks compaction to preserve slash-containing file paths exactly so post-compaction tool calls do not receive paths with inserted spaces.

## Best Possible Solution

Review and land PR #75093, or an equivalent narrow compaction safeguard change, so model-facing compaction input preserves bounded exact path-like literals with regression coverage for slash-separated paths.
