## Summary

Keep open. Current main still relies on instruction- and quality-guard-based identifier preservation rather than a deterministic exact path-literal safeguard, and PR #75093 is already the open implementation candidate with closing syntax for this issue.

## What This Changes

The issue reports that context compression can insert spaces into slash-separated file paths, causing later tool calls to fail on corrupted paths.

## Best Possible Solution

Review and land PR #75093, or an equivalent narrow compaction safeguard change, so bounded path-like literals are preserved exactly without exposing hidden tool-result details.

## Reproduction Assessment

No. The reporter logs give concrete failed paths, but a high-confidence reproduction would need the original long transcript or a deterministic MiniMax compaction replay; source review confirms the failure remains plausible.

## Solution Assessment

Yes. Deterministic path-literal preservation in the compaction safeguard is the narrow maintainable direction, and it is safer than relying on provider prompt adherence alone.
