## Summary

Keep open. The issue has concrete failed tool-call paths, current main still lacks a deterministic exact path-literal preservation step during compaction, and the open implementation candidate at https://github.com/openclaw/openclaw/pull/75093 uses closing syntax for this issue.

## What This Changes

Preserve slash-separated file paths exactly across conversation compaction so later tool calls do not receive corrupted path strings.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/75093, or an equivalent narrow safeguard, so bounded path-like literals are preserved exactly without broad provider-specific behavior changes.

## Reproduction Assessment

No. The issue has concrete failed tool-call paths and current source makes the failure plausible, but a high-confidence reproduction would need the original long transcript or a deterministic MiniMax compaction replay.

## Solution Assessment

Yes for the proposed direction. A bounded deterministic exact-path suffix in the compaction safeguard is narrower and more maintainable than relying only on provider prompt adherence.
