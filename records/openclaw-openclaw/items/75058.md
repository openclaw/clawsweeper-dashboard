## Summary

Keep open. Current main still relies on prompt-guided and bounded identifier preservation rather than a deterministic exact path-literal safeguard, and open PR #75093 already directly targets this issue with closing syntax.

## What This Changes

The issue requests that compaction preserve slash-separated file paths exactly instead of returning corrupted paths with inserted spaces.

## Best Possible Solution

Review and land PR #75093, or an equivalent narrow safeguard, so bounded path-like literals are preserved exactly during compaction without exposing hidden tool-result details.

## Reproduction Assessment

No. The issue has concrete failed raw_params paths and current-main source makes the failure plausible, but a high-confidence reproduction would need the original long transcript or a deterministic MiniMax compaction replay.

## Solution Assessment

Yes. Deterministic exact path-literal preservation in the compaction safeguard is the narrow maintainable direction, and keeping this issue open while the linked PR is reviewed matches the repository policy for open closing PRs.
