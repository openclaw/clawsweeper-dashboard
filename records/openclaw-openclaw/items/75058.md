## Summary

Keep open. The report is fresh, credible, and still matches current main: compaction has identifier-preservation guidance and bounded suffix metadata, but it does not yet provide deterministic exact path-literal replay. ## What This Changes

The issue requests exact preservation of slash-separated file paths through context compaction so later tool calls do not receive corrupted path parameters.

## Best Possible Solution

Review and land PR #75093, or an equivalent narrow compaction safeguard change, so path-like literals are collected from model-facing compaction input and appended deterministically with regression coverage proving slash-separated paths survive unchanged.

## Reproduction Assessment

No. The logs provide credible evidence of corrupted tool parameters, but a high-confidence local reproduction would need the original long transcript or a deterministic MiniMax compaction replay.

## Solution Assessment

Yes. A deterministic path-literal preservation step in the compaction/safeguard pipeline is the narrow maintainable fix, and it is safer than relying on provider prompt adherence or broad provider-specific long-context changes.
