## Summary

Keep open. Current main still lacks deterministic exact path-literal preservation through compaction, the reporter logs show credible corrupted `raw_params.path` values, and open PR #75093 is already the direct implementation candidate with closing syntax, so this issue should stay open until that PR lands or is closed.

## What This Changes

The issue reports that context compression inserts spaces into slash-separated file paths, causing later file tool calls to use corrupted paths and fail.

## Best Possible Solution

Review and land PR #75093, or an equivalent narrow compaction safeguard change, so path-like literals needed for future tool calls are collected from model-facing compaction input and appended deterministically with regression coverage proving slash-separated paths survive unchanged.

## Reproduction Assessment

No. A high-confidence end-to-end reproduction would need the original long transcript or deterministic MiniMax compaction replay; the attached raw tool logs and current source review make the failure credible but not locally replayable.

## Solution Assessment

Yes for the proposed direction. Deterministic path-literal preservation in the compaction/safeguard pipeline is the narrow maintainable fix, and it is safer than relying on provider prompt adherence or provider-specific long-context behavior changes.
