## Summary

Keep open. Current main still relies on prompt-guided identifier preservation, a bounded recent-message identifier audit, and tracked file-operation suffixes rather than a deterministic exact path-literal safeguard; open PR #75093 already uses closing syntax for this issue and is the direct implementation candidate.

## What This Changes

The issue reports that long-session compaction can corrupt slash-separated file paths by inserting spaces into paths later used by tool calls.

## Best Possible Solution

Review and land PR #75093, or an equivalent narrow safeguard, so bounded path-like literals are preserved exactly during compaction without exposing hidden tool-result details.

## Reproduction Assessment

No high-confidence live reproduction was established. The issue has concrete failed tool-call paths and current-main source makes the failure plausible, but a deterministic MiniMax long-context replay or original transcript would be needed for high-confidence reproduction.

## Solution Assessment

Yes, the proposed direction is the best narrow fix. Deterministic path-literal preservation in the compaction safeguard is more maintainable than relying only on provider prompt adherence.
