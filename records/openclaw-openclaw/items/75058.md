## Summary

Keep open. Current main still lacks deterministic exact path-literal preservation for compaction, and PR #75093 is the open implementation candidate that directly targets this issue with closing syntax.

## What This Changes

The issue asks for compaction to preserve slash-separated file paths exactly so later tool calls do not receive corrupted paths with inserted spaces.

## Best Possible Solution

Review and land PR #75093, or an equivalent narrow compaction safeguard change, so model-facing compaction input preserves bounded exact path-like literals with regression coverage for slash-separated paths.

## Reproduction Assessment

No. The reporter logs are concrete, but a high-confidence reproduction would need the original long transcript or a deterministic MiniMax compaction replay; source review confirms the failure remains plausible on current main.

## Solution Assessment

Yes. Deterministic path-literal preservation in the compaction safeguard is the narrow maintainable direction, and it is safer than relying on provider prompt adherence or provider-specific long-context behavior changes.
