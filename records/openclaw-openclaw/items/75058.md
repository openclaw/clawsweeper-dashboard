The report has concrete tool-call failure logs, current main still uses heuristic identifier-preservation instructions plus bounded identifier/file-operation metadata rather than deterministic exact path-literal preservation, and open PR #75093 directly references this issue as the implementation candidate.

## What This Changes

The issue requests that compaction preserve slash-separated file paths exactly so later tool calls do not receive paths with inserted spaces.

## Best Possible Solution

Review and land PR #75093, or an equivalent narrow compaction safeguard change, so model-facing compaction input preserves bounded exact path-like literals with regression coverage for slash-separated paths.
