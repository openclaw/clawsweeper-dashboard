## Summary

Keep open. Current main still relies on model-facing identifier-preservation instructions, bounded identifier auditing, and file-operation suffixes rather than a deterministic exact path-literal safeguard, and open PR #75093 is already the direct closing implementation candidate.

## What This Changes

The issue asks OpenClaw to preserve slash-separated file paths exactly when long-session compaction summarizes context.

## Best Possible Solution

Review and land PR #75093, or an equivalent narrow safeguard, so bounded path-like literals are preserved exactly without exposing hidden tool-result details.

## Reproduction Assessment

No. The reporter provided concrete failed tool-call paths, but a high-confidence reproduction would need the original long transcript or a deterministic MiniMax compaction replay; source review shows the failure remains plausible on current main.

## Solution Assessment

Yes. Deterministic path-literal preservation in the compaction safeguard is the narrow maintainable direction, and it is safer than relying only on provider prompt adherence.
