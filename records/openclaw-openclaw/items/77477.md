## Summary

Keep open: this PR addresses a real current-main doctor/plugin discovery gap and has sufficient real behavior proof, but the latest head still has a P2 fallback-id bug that can rerun plugin doctor migrations for already-handled bundled channels.

## What This Changes

The PR threads active config into plugin doctor contract discovery and extends doctor warning/fix paths for load-path plugins, with focused tests and a changelog entry.

## Best Possible Solution

Keep the active-config pass-through, but compute fallback plugin ids from unresolved channels plus configured non-channel plugin entries, with regression coverage for handled bundled channels.

## Reproduction Assessment

Yes, at source level. Current main omits active config when resolving plugin doctor contracts even though manifest discovery reads config.plugins.load.paths and the docs require compatibility checks to include those discovery roots.

## Solution Assessment

No for the current PR head. Passing active config through doctor discovery is the narrow right direction, but the fallback helper should exclude bundled channels already handled by channel doctor normalizers.
