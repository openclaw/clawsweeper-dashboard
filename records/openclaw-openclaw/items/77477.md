## Summary

Keep this PR open: the main direction is correct and the latest head added the changelog entry, but the fallback migration helper still reintroduces bundled channels that were already handled by the bundled doctor normalizer.

## What This Changes

The PR threads active config into plugin doctor contract discovery and doctor warning/fix paths, adds load-path plugin coverage, and adds a changelog entry.

## Best Possible Solution

Keep the active-config pass-through, but compute plugin fallback ids as unresolved channels plus configured non-channel plugin entries, with coverage proving handled bundled channels are not run through fallback sidecars.

## Reproduction Assessment

Yes, at source level. Current main omits active config when resolving plugin doctor contracts even though manifest discovery uses config.plugins.load.paths and the docs require compatibility checks to include those discovery roots.

## Solution Assessment

No for the current PR head. Passing active config through doctor discovery is the narrow right direction, but the fallback id helper should exclude bundled channels already handled by channel doctor normalizers.
