## Summary

Keep this PR open: the intended doctor fix is valid, but the current head is not merge-ready because the new migration helper can run plugin fallback normalizers for bundled channels that were already handled, and the user-facing fix still lacks a changelog entry.

## What This Changes

The PR threads active config into plugin doctor contract discovery and legacy migration paths so `plugins.load.paths` plugins can contribute doctor warnings and fixes.

## Best Possible Solution

Keep the config pass-through, but build the migration fallback id set from unresolved channels plus configured non-channel plugin entries only, add regression coverage for a handled bundled channel, and add the required changelog entry.

## Reproduction Assessment

Yes. Current main is source-reproducible: doctor contract discovery omits active config while docs require compatibility checks to include `plugins.load.paths`; I did not run a live failing test because the review is read-only.

## Solution Assessment

No for the current PR head. Passing config through doctor discovery is the right narrow direction, but the migration helper should not re-run plugin fallback normalizers for bundled channels already handled by the bundled contract path.
