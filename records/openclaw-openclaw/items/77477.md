## Summary

Keep this PR open: it targets a valid current-main doctor/plugin discovery bug, but the latest head still has a blocking fallback-id bug and is missing the required changelog entry for a user-facing doctor fix.

## What This Changes

The PR threads active config into plugin doctor contract discovery and extends doctor warning/fix paths plus tests for `plugins.load.paths` and configured plugin entries.

## Best Possible Solution

Keep the active-config pass-through, but compute fallback ids as unresolved channels plus configured non-channel plugin entries, add coverage for a handled bundled channel, and add an Unreleased changelog fix entry.

## Reproduction Assessment

Yes, at source level. Current main omits active config when resolving plugin doctor contracts even though docs require compatibility checks to include `plugins.load.paths`; I did not run a live failing command because this sweep is read-only.

## Solution Assessment

No for the current PR head. Passing config through doctor discovery is the narrow right direction, but the fallback id helper should avoid re-running plugin fallback normalizers for bundled channels already handled by the bundled path.
