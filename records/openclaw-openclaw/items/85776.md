## Summary

No blocking findings. This PR remains necessary because current main and the latest release still send OpenAI image output_compression for PNG/default-PNG request paths, while the proposed patch narrowly gates that field to JPEG/WebP and covers both direct Images API and Codex Responses image-tool requests.

## What This Changes

The PR changes the OpenAI image provider to omit output_compression for PNG outputs, adds direct/Codex PNG regression tests, and documents that PNG ignores openai.outputCompression.

## Best Possible Solution

Land the narrow OpenAI provider guard after ordinary maintainer review so JPEG/WebP compression remains supported and PNG/default-PNG payloads omit output_compression.

## Reproduction Assessment

Yes. Current main and v2026.5.22 source paths unconditionally attach output_compression when openai.outputCompression is set, while the official OpenAI image contract scopes that parameter to JPEG/WebP outputs.

## Solution Assessment

Yes. Centralizing the output-format gate and reusing it at both OpenAI request construction sites is the narrowest maintainable fix; no new config, public API, or product decision is needed.
