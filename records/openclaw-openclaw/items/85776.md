## What This Changes

The PR gates OpenAI `output_compression` to JPEG/WebP image outputs, adds direct and Codex PNG omission tests, and documents that PNG ignores `openai.outputCompression`.

## Best Possible Solution

Land the narrow OpenAI provider guard after ordinary maintainer review so JPEG/WebP compression remains supported and PNG/default-PNG payloads omit `output_compression`.

## Reproduction Assessment

Yes, source-reproducible. Current main and `v2026.5.22` unconditionally forward `output_compression` on PNG/default-PNG request paths, while the official OpenAI contract scopes that parameter to JPEG/WebP outputs.

## Solution Assessment

Yes. Centralizing the format gate and reusing it at both OpenAI request construction sites is the narrowest maintainable fix; no new config or public API direction is needed.
