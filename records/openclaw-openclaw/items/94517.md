- **[P2] Preserve labels when rebuilding paired devices:** `src/infra/device-pairing.ts:95`
  - body: Adding `operatorLabel` to the persisted record is not enough: `buildApprovedPairedDevice` rebuilds records during repair/reapproval from pending plus selected existing fields and does not copy `existing.operatorLabel`. - **[P2] Sanitize display names before gateway logging:** `src/gateway/server-methods/devices.ts:684`
  - body: `displayName` comes directly from RPC params and may contain newlines or terminal escapes. Logging `label=${trimmed}` raw can forge gateway log lines; use the existing log sanitizer or omit the label from the structured log message.
  - confidence: 0.9
- **[P2] Render operator labels in the Control UI path:** `src/cli/devices-cli.runtime.ts:758`
  - body: This only changes CLI display precedence. `device.pair.list` will return the new `operatorLabel`, but Control UI's paired-device type/render path still uses `displayName || deviceId`, so the linked web UI naming request remains visibly broken after a CLI rename. Add the field/precedence to Control UI or stop this PR from closing the broader issue.
  - confidence: 0.86
