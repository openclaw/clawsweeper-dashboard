## What This Changes

The branch adds an `openclaw devices rename` CLI command, a `device.pair.rename` Gateway RPC with protocol schema/types, and regenerated Swift protocol bindings.

## Best Possible Solution

Revise this into an operator-owned paired-device label workflow with explicit label-over-client-displayName-over-deviceId precedence, existing device-management authorization checks, Gateway/CLI/Control UI/docs coverage, and real redacted runtime proof.

## Reproduction Assessment

## Solution Assessment

No. The maintainable fix is a separate operator-owned label field with explicit display precedence and the existing paired-device authorization boundary, not a direct write into client-controlled `displayName`.
