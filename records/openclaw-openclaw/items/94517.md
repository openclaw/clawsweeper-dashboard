## Summary

## What This Changes

This PR adds `openclaw devices rename`, a `device.pair.rename` Gateway RPC, protocol schema/type exports, and a Swift `DevicePairRenameParams` binding.

## Best Possible Solution

Revise this into an additive operator-owned paired-device label workflow, separate from client `displayName`, guarded by the existing device-management authorization boundary, documented for CLI/Gateway users, and proven with a real paired-device rename/list run.

## Reproduction Assessment

## Solution Assessment

No. The PR is a plausible starting point, but the maintainable fix is a separate operator-owned label field with explicit display precedence, existing device-management authorization, docs, tests, and real paired-device proof.
