  - body: Adding the core descriptor advertises `device.pair.rename`, but the PR leaves the lazy device handler list at list/approve/reject/remove and token methods only. Because core descriptor creation skips specs with no handler, the new CLI path can fail as an unknown Gateway method.
  - confidence: 0.96
- **[P1] Enforce device ownership before renaming:** `src/gateway/server-methods/devices.ts:615`
  - body: The new handler receives only `params` and `respond`, then mutates paired-device state for any target `deviceId`. Sibling device-management mutations resolve caller authz from `client` and deny cross-device/non-admin management before mutating, so rename needs the same boundary.
  - confidence: 0.95
- **[P1] Store friendly labels outside displayName:** `src/gateway/server-methods/devices.ts:638`
  - body: This writes the operator’s friendly name through `updatePairedDeviceMetadata`, but reconnect handling refreshes `displayName` from the connecting client through the same metadata path. A renamed device can therefore lose the user label on reconnect; add a separate operator-owned label field and display precedence instead.
  - confidence: 0.94
