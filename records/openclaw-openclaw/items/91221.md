## Summary

Keep open: the PR addresses a real Linux systemd gateway restart cascade with strong live proof, but the patch can retire the user-scope unit whenever a same-name system unit file merely exists, even if that system unit is inactive or disabled. That service-ownership rule needs a fix or explicit maintainer acceptance before merge.

## What This Changes

The PR adds Linux systemd user/system gateway-unit conflict detection, doctor cleanup of the redundant user unit, service-mode diagnostics, and regression tests.

## Best Possible Solution

Merge a version that resolves same-name user/system systemd conflicts only when the system unit is actually authoritative, such as active/enabled or explicitly operator-owned, while preserving intentional multi-gateway setups and prompted doctor repair.

## Reproduction Assessment

Yes. Current main has a source-level reproduction: the detector returns the user unit before checking the system unit, and the linked issue provides concrete Linux/WSL2 restart-loop steps; I did not run live systemd in this read-only review.

## Solution Assessment

No, not as written. The doctor/systemd repair boundary is plausible, but the best fix needs a live/enabled system-owner guard or an explicit maintainer policy accepting file-presence authority before retiring the user unit.
