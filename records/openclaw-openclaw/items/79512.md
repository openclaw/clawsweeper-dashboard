## Summary

Keep open: this is useful Codex/task-registry work, but current head still has a blocking private SDK subpath resolution problem and a security-sensitive Testbox secret change that needs maintainer handling.

## What This Changes

The PR mirrors Codex app-server native subagent lifecycle events into Task Registry with Codex mirror code, protocol/test updates, a private task-runtime subpath, build/changelog changes, and Testbox Codex auth hydration.

## Best Possible Solution

Keep the Codex-owned Task Registry mirror, but route detached task helpers through a normal bundled-Codex-resolvable private seam or approved public SDK subpath, and move Codex auth hydration to a trusted Testbox path before merge.

## Reproduction Assessment

Not applicable as an end-user bug report; this is a feature PR. The blocking resolver issue is source-reproducible from the PR's static private subpath import plus current main's QA-only private alias gates.

## Solution Assessment

No. The mirror placement is plausible, but the current private runtime route is not mergeable until normal bundled Codex loading can resolve it, and the Codex auth CI wiring should be restricted to a trusted path or explicitly approved.
