## Summary

Keep this PR open: it is draft and has the protected `maintainer` label, and the private Plugin SDK-adjacent seam needs explicit owner review. The latest diff fixes the earlier resolver blocker, but it now appears to expose the private Codex task-runtime helper through the general plugin loader alias path rather than only to bundled Codex.

## What This Changes

The PR mirrors Codex app-server native subagent lifecycle events into Task Registry rows and adds a private local-only Plugin SDK subpath for the bundled Codex plugin to mutate detached task runs.

## Best Possible Solution

Keep the Codex Task Registry mirror, but scope the detached task mutation helper to bundled Codex or promote an explicitly approved narrow SDK seam instead of making the private helper broadly resolvable by plugin loaders.

## Reproduction Assessment

Not applicable as an end-user bug report; this is a feature PR. The review finding is source-reproducible by following the new non-QA private subpath allowlist into the general plugin loader alias map.

## Solution Assessment

No. The mirror placement is plausible, but the private task mutation helper should be scoped to bundled Codex or explicitly approved as a supported seam before merge.

## Review Findings

Overall correctness: patch is incorrect
