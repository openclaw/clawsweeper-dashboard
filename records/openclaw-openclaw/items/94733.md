## Best Possible Solution

## Reproduction Assessment

Yes at source level: current main skips unaddressed group media before media download and hook dispatch, while the sibling text path already consults `ingest`. I did not establish a live Telegram reproduction in this read-only review.

## Solution Assessment

No as submitted: the ingest precedence check is in the right helper, but the patch also lets unaddressed passive-ingest media hit existing visible fetch-failure reply paths. The safer fix preserves the bypass for successful media while keeping unaddressed `requireMention` failures silent.
