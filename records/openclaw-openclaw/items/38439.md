## Summary

Keep this open. Current main still has a source-reproducible avatar containment failure for mixed Windows namespace and drive-letter paths, and open PR #38923 still references this issue with closing syntax but has not merged.

## What This Changes

The issue reports that the Control UI/webchat `/avatar/{agentId}` endpoint returns 404 for a valid workspace-relative `IDENTITY.md` avatar on Windows.

## Best Possible Solution

Normalize Windows namespace and drive-letter path comparisons in the avatar resolver/policy, preserve workspace, symlink, and size protections, and cover `IDENTITY.md`, `?meta=1`, and image-route cases.

## Reproduction Assessment

Yes, at source level. A Windows namespaced workspace root plus a normal drive-letter avatar path under that workspace makes the current containment predicate reject a valid in-workspace avatar without needing a live Windows UI run.

## Solution Assessment

Yes for keeping the issue open. The narrow maintainable fix belongs in avatar path normalization plus route-level regression coverage, either by landing #38923 or extracting a focused replacement if that PR cannot be merged cleanly.
