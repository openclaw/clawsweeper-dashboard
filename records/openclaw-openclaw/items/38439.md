## What This Changes

The issue reports that Control UI/webchat returns 404 for `/avatar/{agentId}` when a workspace-relative avatar is declared in `IDENTITY.md`.

## Best Possible Solution

Keep the issue open until maintainers decide whether this remains its own local-avatar diagnostics bug or should be consolidated into the broader Control UI avatar contract issue, then land a narrow fix that preserves workspace containment, size limits, CSP/privacy policy, and clear UI feedback.

## Reproduction Assessment

Yes, at source level for the remaining failure class: current code rejects oversized local avatars as `too_large`, direct `/avatar/{agentId}` serving returns 404 for non-local resolutions, and some upload paths silently ignore over-limit files. I did not establish a live Windows reproduction in this read-only pass.

No, a resolver-only patch is no longer the best complete solution.
