## Summary

Keep open: current main and v2026.6.8 still have the Gateway/browser managed-image path, but native SwiftUI WebChat does not preserve or act on managed-image `url`/`openUrl` fields for Open/Download.

## What This Changes

The issue reports that native macOS WebChat can display MEDIA-delivered images but its Open/Download context-menu actions do nothing while browser Control UI works.

## Best Possible Solution

Implement native WebChat managed-image support in shared Chat UI and macOS transport with authenticated open/download handling for `/api/chat/media/outgoing/...` URLs, then prove it in a real macOS app flow.

## Reproduction Assessment

Yes at source level: Gateway emits managed image blocks with relative `url`/`openUrl`, while native SwiftUI Chat UI currently does not preserve or act on those fields. A live macOS reproduction is still missing.

## Solution Assessment

Unclear on final shape: the fix likely belongs in shared native Chat UI plus macOS authenticated media fetch/open/download plumbing, but maintainers need to choose the native action contract.
