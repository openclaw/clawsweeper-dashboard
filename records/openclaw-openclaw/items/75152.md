## What This Changes

The issue reports that the `edit` tool rejects `edits` payloads that should be accepted, making file edits unusable in the reported DeepSeek/OpenAI-compatible route.

## Best Possible Solution

No live DeepSeek/XML-RPC run was performed, so the exact native-array transport wrapping still needs real-route proof.

## Solution Assessment

Unclear as the complete solution. Forwarding `prepareArguments` is the narrowest maintainable fix for the JSON-string path, but the paired PR should prove the exact reported DeepSeek/native-array failure mode before this issue is closed.
