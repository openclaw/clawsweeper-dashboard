## What This Changes

The issue reports that the `edit` tool rejects both stringified and native `edits` payloads in a DeepSeek/OpenAI-compatible run, making file edits unusable from that environment.

## Best Possible Solution

Capture the raw tool-call arguments received by OpenClaw for the failing DeepSeek run, add a focused regression for that exact boundary, then fix only the layer that creates or receives the malformed `edits` shape while preserving the public canonical `path` plus `edits: [{ oldText, newText }]` contract.
