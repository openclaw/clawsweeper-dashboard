## Summary

Keep open. Current main still proves the canonical in-process `edit` payload, but I did not find evidence that the reported DeepSeek/OpenAI-compatible path now preserves the native `edits` array shape through transport and validation. The issue reports that the `edit` tool rejects both JSON-stringified and native-array `edits` payloads on a DeepSeek route, making the tool unusable in that environment.
