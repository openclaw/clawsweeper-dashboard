## Summary

Keep open. The underlying config-restore bug is still valid and this PR touches the right recovery path, but the branch is conflicting, lacks real behavior proof, and the proposed detector misses supported sensitive config locations and some actual `maskApiKey` outputs.

## What This Changes

The PR adds a parsed-backup redacted-secret detector, calls it before async and sync `.bak` config restores, and adds unit/integration tests for the refusal path.

## Best Possible Solution

Land a current-main guard before both direct backup copy paths that uses schema/path-sensitive placeholder detection, accepts actual `maskApiKey` outputs, and leaves backup creation scanning, rotation policy, and CLI validation as separate follow-up scope.

## Reproduction Assessment

Yes. Source inspection on current main shows that an update-channel-only live config plus a valid-looking `.bak` with `gateway.mode` and a redacted sensitive value reaches `copyFile` or `copyFileSync` without inspecting `backupParsed`.

## Solution Assessment

No as-is. Refusing polluted backups before copying is the right fix shape, but the detector should follow current sensitive schema/path semantics and actual `maskApiKey` output before this lands.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Scan schema-sensitive paths before restoring backups:** `src/config/backup-pollution.ts:12-20`
  - body: `findRedactedSecretSites` only checks the leaf key against this suffix list, but supported sensitive config values can live under leaves such as `value`, `key`, `passphrase`, or arbitrary request header names. A polluted backup at one of those paths would still be treated as clean and restored over the live config.
  - confidence: 0.91
- **[P2] Accept every maskApiKey ellipsis output:** `src/config/backup-pollution.ts:24`
  - body: `maskApiKey` preserves whatever leading and trailing characters the original secret had, but this regex only allows a fixed punctuation subset around the ellipsis. Masked values containing other punctuation can be missed, so the restore guard can still copy a polluted backup.
  - confidence: 0.84
