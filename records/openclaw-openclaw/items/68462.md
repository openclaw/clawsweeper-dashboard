## Summary

Keep open. This PR targets a real current-main config recovery bug, but the proposed detector still misses supported sensitive config paths and some actual masked secret outputs; it also lacks external after-fix real behavior proof and is currently conflicting.

## What This Changes

The PR adds a parsed-config redacted-secret detector, calls it before async and sync `.bak` recovery restores, and adds unit/integration tests for the refusal path.

## Best Possible Solution

Land a rebased restore guard that reuses schema/path-sensitive placeholder detection, accepts actual `maskApiKey` outputs, preserves current audit and copy-failure handling, and leaves backup creation, rotation policy, and CLI scanning as separate follow-up scope.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: use an update-channel-only live config plus a valid-looking `.bak` with `gateway.mode` and a redacted sensitive value, and current main reaches async or sync restore copy without scanning the parsed backup for placeholders.

## Solution Assessment

No as-is. Refusing polluted backups before copying is the right fix shape, but the detector should follow current sensitive schema/path semantics and actual `maskApiKey` output, and the branch must be rebased against current main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Scan schema-sensitive paths before restoring backups:** `src/config/backup-pollution.ts:13-21`
  - body: `findRedactedSecretSites` only checks the leaf key against this suffix list, so supported sensitive paths whose leaf is `value`, `key`, `passphrase`, or an arbitrary request header name can still be treated as clean. A polluted backup at one of those paths would still be restored, leaving the guard bypassable.
  - confidence: 0.91
- **[P2] Accept every maskApiKey ellipsis output:** `src/config/backup-pollution.ts:25`
  - body: `maskApiKey` preserves whatever leading and trailing characters the original secret had, but this regex only allows a fixed punctuation subset around the ellipsis. Masked values containing other punctuation can be missed and restored from a polluted backup.
  - confidence: 0.84
