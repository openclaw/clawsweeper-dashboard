## Summary

Keep open. The PR targets a valid current-main config recovery bug, but its detector misses supported sensitive config paths and some real `maskApiKey` placeholder shapes; it also lacks external real-behavior proof and is currently conflicting with main.

## What This Changes

The PR adds a redacted-secret backup scanner and calls it from async and sync config `.bak` recovery before restoring, with unit and integration tests.

## Best Possible Solution

Rebase or replace the PR with a main-ready restore guard that uses schema/path-sensitive placeholder detection, accepts actual masked placeholder shapes, preserves current main’s copy-failure audit handling, and keeps broader backup creation or CLI scanning as follow-up scope.

## Reproduction Assessment

Yes. A source-level path is clear: use an update-channel-only live config plus a valid-looking `.bak` with `gateway.mode` and a redacted sensitive value, and current main reaches the async or sync `copyFile` restore path without inspecting the parsed backup for placeholders.

## Solution Assessment

No as-is. The guard belongs before the `.bak` copy, but the detector should follow existing sensitive schema/path semantics and actual `maskApiKey` output instead of a small leaf-name suffix list and fixed punctuation regex.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Scan schema-sensitive secret paths:** `src/config/backup-pollution.ts:13-21`
  - body: `findRedactedSecretSites` only checks the current leaf key against `SECRET_FIELD_SUFFIXES`, so supported sensitive paths whose leaf is `value`, `key`, `passphrase`, or an arbitrary `request.headers.*` name can still be treated as clean. A polluted backup at one of those paths would still be restored, leaving the recovery guard bypassable.
  - confidence: 0.91
- **[P2] Match actual maskApiKey placeholders:** `src/config/backup-pollution.ts:25`
  - body: `maskApiKey` preserves whatever characters appear at the start and end of the original secret, but this regex only allows a fixed subset around the ellipsis. Masked values containing other punctuation can be missed and restored from a polluted backup.
  - confidence: 0.84
