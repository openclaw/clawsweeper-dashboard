Full review comments:

- **[P2] Scan supported secret paths before restoring backups:** `src/config/backup-pollution.ts:13-21`
  - body: The detector only treats selected terminal field names as secret-like. Supported config can store secrets under TLS `key`/`passphrase`, request header records, header-auth `value`, and local-service env entries, so a `.bak` polluted at those paths would still be copied over the live config. Reuse the current sensitive-path semantics or cover those paths before merge.
  - confidence: 0.9
- **[P2] Accept every maskApiKey edge character:** `src/config/backup-pollution.ts:25`
  - body: `maskApiKey` preserves the original leading and trailing characters around `...`, but this regex only accepts a limited character class. A masked secret whose edge characters fall outside that class is treated as clean, allowing a polluted backup to restore for that secret.
  - confidence: 0.86
