## Summary

Keep open. Current main still has the direct `.bak` restore bug and this PR targets the right code path, but the patch is not merge-ready because its detector misses schema-sensitive secret fields and some real `maskApiKey` outputs, and the external PR still lacks real behavior proof.

## What This Changes

The PR adds a redacted-secret backup scanner and calls it before async and sync config `.bak` recovery restores.

## Best Possible Solution

Land a current-main restore guard that reuses schema/path-sensitive placeholder detection, accepts actual mask output shapes, and leaves backup creation, rotation depth, and CLI scanning as explicit follow-up scope.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: use an update-channel-only live config plus a valid-looking `.bak` with `gateway.mode` and a redacted sensitive value, and current main reaches the async or sync copy path without inspecting the parsed backup for placeholders.

## Solution Assessment

No as-is. The guard belongs before the direct `.bak` copy, but the detector should follow current sensitive schema/path semantics and actual `maskApiKey` output, and the rebase should preserve current copy-failure audit handling.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Scan schema-sensitive paths before restoring backups:** `src/config/backup-pollution.ts:13-21`
  - body: `findRedactedSecretSites` only checks the leaf key against this suffix list, so supported sensitive paths whose leaf is `value`, `key`, `passphrase`, or an arbitrary request header name can still be treated as clean. A polluted backup at one of those paths would still be restored, leaving the guard bypassable.
  - confidence: 0.91
- **[P2] Accept all actual maskApiKey ellipsis outputs:** `src/config/backup-pollution.ts:25`
  - body: `maskApiKey` preserves whatever leading and trailing characters the original secret had, but this regex only allows a fixed punctuation subset around the ellipsis. Masked values containing other punctuation can be missed and restored from a polluted backup.
  - confidence: 0.84
