## Summary

Keep open. Current main and latest release still expose only global memory-core dreaming configuration, the resolver still fans out across configured agents without an agent-level opt-out, and https://github.com/openclaw/openclaw/pull/67420 remains open with closing syntax for this issue.

## What This Changes

Add per-agent memory-core dreaming controls so agents can opt out of dreaming and potentially use custom schedules instead of sharing one global sweep.

## Best Possible Solution

Define a documented and schema-backed agents.list[].dreaming contract at least for enabled, wire it through workspace resolution and tests, and split or explicitly design per-agent scheduling before adding multi-cron behavior.

## Reproduction Assessment

Yes for the current feature gap. Source inspection shows agents.list[].dreaming is absent from AgentConfig and AgentEntrySchema, while resolver/runtime code still processes all resolved agent workspaces; I did not run tests because this review is read-only and those tests create artifacts.

## Solution Assessment

Yes for keeping this issue open. The maintainable path is to review or replace the linked implementation PR for the enable/disable contract, while treating per-agent schedules as an explicit cron/product design decision rather than closing the issue early.
