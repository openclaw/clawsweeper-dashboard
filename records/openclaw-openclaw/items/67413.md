## Summary

Keep open. Current `origin/main` still exposes only global memory-core dreaming configuration, still resolves every configured agent workspace without an agent-level opt-out, and the same-author implementation PR that uses closing syntax for this issue remains open and conflicting.

## What This Changes

Request per-agent memory-core dreaming controls, including agent-level enable/disable, custom schedules, and precedence over the global memory-core dreaming defaults.

## Best Possible Solution

Keep this issue open while the linked PR or a successor adds a documented, schema-backed `agents.list[].dreaming` contract for enable/disable, with resolver/runtime/tests/docs coverage; split per-agent schedule support into an explicit design if it should not land in the same change.

## Reproduction Assessment

Yes, for the source-level feature gap. Current `origin/main` lacks `agents.list[].dreaming` in the type and strict schema, and the dreaming resolver/runtime still process the configured workspace list without an agent-level opt-out.

## Solution Assessment

Yes for keeping this issue open. The maintainable path is to review, update, land, or replace https://github.com/openclaw/openclaw/pull/67420 for the enable/disable contract, while making a separate maintainer decision on the requested per-agent scheduling semantics.
