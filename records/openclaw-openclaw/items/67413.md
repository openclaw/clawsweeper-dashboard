## Summary

Keep open. Current main and v2026.6.8 still expose global-only memory-core dreaming configuration and sweep all configured agent workspaces, while the same-author linked fix PR is open, conflicting, and still needs maintainer product/API review.

## What This Changes

The issue requests per-agent memory-core dreaming controls so operators can disable or schedule dreaming for individual agents instead of using one global sweep across all configured workspaces.

## Best Possible Solution

Keep this issue open while maintainers choose the durable schema-backed scoping API, then land a refreshed implementation or successor PR with resolver, schema, docs, and regression coverage; split per-agent schedules if they are not part of the first accepted fix.

I did not run a live multi-agent OOM scenario, so this is source-reproducible rather than fully reproduced.

## Solution Assessment

Unclear. A per-agent opt-out or equivalent allowlist is a plausible solution, but the exact config location, fallback behavior, and whether custom schedules belong in the same change need maintainer product/API review.
