Keep open: current main still drops the MCP SDK request AbortSignal before it reaches plugin tool execution, and the issue is paired with the open closing PR at https://github.com/openclaw/openclaw/pull/82426. ## What This Changes

The issue reports that the standalone MCP plugin-tools server does not forward the host cancellation AbortSignal into plugin tool execution.

## Best Possible Solution

Review and land the canonical MCP cancellation fix in https://github.com/openclaw/openclaw/pull/82426 with focused regression coverage, then close this issue through that merge.

## Reproduction Assessment

Yes. The reporter supplied a concrete MCP stdio cancellation probe, and current source plus the locked MCP SDK contract show the request handler extra signal cannot reach tool.execute on main; I did not execute the probe in this read-only review.

## Solution Assessment

Yes. Forwarding the existing MCP SDK RequestHandlerExtra.signal through createToolsMcpServer, createPluginToolsMcpHandlers.callTool, and the existing AnyAgentTool.execute signal parameter is the narrowest maintainable fix.
