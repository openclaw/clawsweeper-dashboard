## Summary

Keep open: current main has documentation and some adjacent doctor checks for heartbeat/session state, but it still lacks the requested cross-field warnings for short global timeouts, aggressive stuck-session aborts, and heartbeat agent-selection combinations. The related merged work covers nearby timeout and heartbeat behavior, not this diagnostic policy.

## What This Changes

Request doctor/config diagnostics for risky heartbeat, timeout, and stuck-session configuration combinations that can make recovery failures look like agent ghosting.

## Best Possible Solution

Add a scoped doctor/config warning collector over existing config fields, with tests and wording that points operators toward worker agents, lighter heartbeat context, and correct timeout ceilings without changing runtime behavior.

## Solution Assessment
