## Summary

Keep open: current main still exposes the bundled Gemini CLI OAuth path and the local flow still opens the browser before the localhost callback listener is started, while token/project failures are still collapsed into the generic Gemini CLI access help shown in the report.

## What This Changes

The issue reports that Google Gemini CLI OAuth fails during onboarding/token exchange on macOS/Homebrew despite the account having Gemini access.

## Best Possible Solution

Keep the issue open for the Google plugin/auth owner to live-reproduce on current main or v2026.5.2, make the localhost callback listener ready before browser handoff, and preserve clearer token-exchange/project-discovery diagnostics.

## Solution Assessment
