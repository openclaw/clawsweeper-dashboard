## Summary

Keep open: current main still has source-visible Gemini CLI OAuth failure surfaces matching the report and follow-up comments, and the related proxy PR remains open, broad, and unmerged.

## What This Changes

The issue reports Gemini CLI OAuth login failing after browser authorization/token exchange with `AbortError` on macOS despite the account having Gemini access.

## Best Possible Solution

Land a focused Google auth/runtime fix that starts the callback listener before browser handoff, routes known Google OAuth calls through target-aware trusted env proxy behavior, preserves manual fallback, and adds regression coverage plus clearer diagnostics.

## Reproduction Assessment

No, not with high-confidence live reproduction: I did not run real Google OAuth. Source inspection gives a medium-confidence reproduction path for proxy-required token exchange failures and the localhost callback listener race.

## Solution Assessment

Yes, the narrow Google plugin/auth-runtime repair path is the best direction. Current main is not enough, and the broad related PR should be split or replaced rather than merged as-is.
