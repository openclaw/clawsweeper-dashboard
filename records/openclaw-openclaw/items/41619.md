## Summary

Keep open: current main and the latest shipped tag still expose the bundled Gemini CLI OAuth path, still open the browser before the localhost callback listener is started, and still collapse OAuth/token failures into the generic Gemini CLI access help seen in the report.

## What This Changes

The issue reports that Google Gemini CLI OAuth fails during onboarding/model auth with `AbortError` despite the account having Gemini access.

## Best Possible Solution

Keep this open for the Google plugin/auth owner to reproduce on current main or `v2026.5.2`, then make the callback listener ready before browser handoff and improve `AbortError`/token-exchange diagnostics in the provider-owned OAuth flow.

## Reproduction Assessment

Unclear for the exact live account-side failure because this review did not exercise Google OAuth. Source inspection gives a concrete current-main target: the browser opens before the localhost listener is started, and the token exchange still goes through a timeout path that can surface `AbortError`.

## Solution Assessment

Yes for the likely fix direction: the narrowest maintainable solution belongs in `extensions/google` and the shared OAuth callback helper, not a broad core migration. The existing current-main flow is not enough to close the report.
