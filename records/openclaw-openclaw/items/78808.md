Request to move Gateway/channel polling off the main Node.js event loop using worker threads or process isolation so multi-agent/channel deployments do not starve agent work.

## Best Possible Solution

Keep the Telegram isolated-ingress worker on main, and use the related open performance trackers for any remaining measurable Telegram backpressure or Gateway/agent responsiveness work.
