## Summary

Keep open. Current main and the latest shipped tag still hard-cap clobber snapshots and only warn once per process/path, so the reported forensic gap remains, but the correct retention and visibility policy needs maintainer choice.

## What This Changes

The issue asks for `openclaw doctor` config-clobber snapshot handling to preserve or loudly surface later events after the `.clobbered.*` cap is reached.

## Best Possible Solution

Choose a retention policy, then update the shared snapshot helper, recovery and doctor tests, and troubleshooting docs so operators either always retain the latest event or get a durable health warning when snapshots are skipped.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create at least 32 `.clobbered.*` siblings for one config path, then the next snapshot attempt warns once and returns `null`, leaving recovery without a new event-specific snapshot.

## Solution Assessment

Unclear. Rotation, a louder health signal, or both could address the gap, but choosing whether OpenClaw should delete older forensic files or keep a hard cap with durable warnings is a maintainer policy decision.
