## Summary

Keep open: current main still lacks an explicit OpenClaw User-Agent on the Z.AI endpoint detection probe, and the reported failure matches the documented onboarding contract for automatic endpoint selection.

## What This Changes

The issue reports that Z.AI API-key onboarding fails to auto-detect Coding Plan endpoints because the detection probe does not send an explicit User-Agent.

## Best Possible Solution

Add an explicit OpenClaw User-Agent to Z.AI endpoint detection probes in the Z.AI plugin, with regression coverage and live Coding Plan proof when credentials are available.

## Reproduction Assessment

Yes for source-level reproduction: current `extensions/zai/detect.ts` does not set an explicit User-Agent on detection probes, and current tests already model 429 as a null detection. I did not reproduce the z.ai 429 live because no Coding Plan API key is available in this review environment.

## Solution Assessment

Yes, adding an explicit OpenClaw User-Agent inside the Z.AI plugin probe is the narrowest owner-boundary fix. Moving provider-specific detection into core or adding a config option would be broader than needed.
