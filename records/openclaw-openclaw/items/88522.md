## Summary

## What This Changes

This PR adds an account-scoped Telegram active-run ingress key for plain-text follow-ups, updates Telegram docs, adds a reply-fence lane query, and covers same-topic steering with a regression test.

## Best Possible Solution

Limit active-run ingress to actual plain-text message updates, keep callbacks and media on serialized lanes, consider sharing the helper with isolated polling if this branch remains broad, and require redacted Telegram proof before merge.

## Reproduction Assessment

Yes, source-reproducible for the review blocker: the PR head derives active-run candidates from callback_query.message, and grammY serializes only matching constraint strings. I did not run live Telegram because this is a read-only review and the PR has no posted live proof.

## Solution Assessment

No, not yet. The sequentialization boundary is a plausible fix location, but the best version must exclude callbacks from text ingress and prove the visible Telegram behavior before merge.
