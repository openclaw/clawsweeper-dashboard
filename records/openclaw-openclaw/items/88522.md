## Summary

Keep this PR open for revision: the active-run ingress direction is useful, but the diff currently lets Telegram callback updates be treated like plain-text follow-ups, and the external PR still lacks real Telegram proof for the visible behavior.

## What This Changes

This PR adds a Telegram sequentializer wrapper for plain-text active-run ingress, updates the Telegram runtime docs, and adds a regression test for same-topic mid-run steering.

## Best Possible Solution

Limit active-run ingress to real text message updates for the same account/chat/thread, keep callbacks and media on their existing serialized lanes, and require redacted live Telegram proof before merge.

## Solution Assessment
