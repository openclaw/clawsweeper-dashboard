## Summary

Keep open. The PR still adds a useful WhatsApp composing-debounce feature, but current main does not already implement it and the branch still has merge-blocking correctness issues plus no real WhatsApp behavior proof.

## What This Changes

The PR adds an opt-in WhatsApp `composingDebounceMs` setting, subscribes to Baileys composing presence updates, and extends the shared inbound debouncer with touch helpers.

## Best Possible Solution

Rebase and keep this as a WhatsApp-owned opt-in feature only if it wires schema/types/account/docs/tests, uses matching direct/group debounce keys, bounds presence subscriptions and caches, and includes redacted live WhatsApp proof.

## Reproduction Assessment

Not applicable for the feature request itself. The blocking review findings are source-reproducible from the PR diff: the strict WhatsApp runtime schema lacks the new key, and group presence touches do not match the current group debounce key shape.

## Solution Assessment

No. The direction is plausible, but this PR is not the best mergeable solution until the schema, group keying, subscription/cache bounds, tests, rebase, and real WhatsApp proof are addressed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Wire the setting into the runtime schema:** `src/config/types.whatsapp.ts:85`
  - body: The new option is added to TypeScript types and account plumbing, but the strict WhatsApp Zod schemas still only accept `debounceMs`. A user who sets `channels.whatsapp.composingDebounceMs` or an account override will hit an unknown-key validation error before the feature can run.
  - confidence: 0.94
- **[P2] Use keys that match group debounce buffers:** `extensions/whatsapp/src/inbound/monitor.ts:490`
  - body: For group messages the buffer key prefers `senderJid`, but this presence handler touches a prefix ending in E.164. Normal group composing events therefore will not extend the pending sender buffer.
  - confidence: 0.89
- **[P2] Deduplicate presence subscriptions per chat:** `extensions/whatsapp/src/inbound/monitor.ts:402-404`
  - body: With composing debounce enabled, every accepted inbound message calls `presenceSubscribe(remoteJid)`. Active chats can generate redundant WhatsApp subscribe requests; track subscribed JIDs and retry only after failure.
  - confidence: 0.82
- **[P3] Bound the LID mapping cache:** `extensions/whatsapp/src/inbound/monitor.ts:78`
  - body: `jidToE164Cache` is never evicted even though it only needs to bridge the short window between a message and a presence update. Long-running WhatsApp sessions with many distinct senders can retain stale entries indefinitely.
  - confidence: 0.78
