## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Wire composingDebounceMs into strict schemas:** `src/config/types.whatsapp.ts:82-85`
  - body: The new config field is added to the TypeScript type and resolver path, but `WhatsAppConfigSchema` and `WhatsAppAccountSchema` remain strict and still only allow `debounceMs`. A user who sets this option at root or account scope gets an unknown-key validation failure, so the feature cannot be enabled from real config.
  - confidence: 0.97
- **[P2] Use debounce-compatible keys for group composing touches:** `extensions/whatsapp/src/inbound/monitor.ts:489-490`
  - body: Group buffers are keyed with `senderJid` before E.164, but the presence handler touches only an E.164 prefix. In normal group traffic with `participantJid`, composing events will not extend the sender's active debounce buffer.
  - confidence: 0.91
- **[P2] Deduplicate and gate presence subscriptions:** `extensions/whatsapp/src/inbound/monitor.ts:397-404`
  - body: This calls `presenceSubscribe(remoteJid)` on every inbound message whenever `composingDebounceMs` is set, even if debounce is disabled and no presence listener was registered. Track subscribed chats and only subscribe when composing debounce can actually touch a buffer, retrying after failures.
  - confidence: 0.89
- **[P2] Bound the LID-to-E164 cache:** `extensions/whatsapp/src/inbound/monitor.ts:78`
  - body: `jidToE164Cache` grows for every mapped JID and never expires entries. Since it only needs to bridge nearby composing events, use a short TTL or size cap to avoid stale contact growth in long-running WhatsApp listeners.
  - confidence: 0.84
