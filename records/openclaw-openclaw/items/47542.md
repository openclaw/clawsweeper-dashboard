## Summary

Keep open. Current main does not implement composing-aware WhatsApp debounce, but this PR is not merge-ready because the new config key is not accepted by the strict runtime schema, group presence updates can miss active debounce buffers, repeated presence subscriptions/cache growth need tightening, and contributor real-behavior proof is missing.

## What This Changes

The PR adds an opt-in WhatsApp `composingDebounceMs` setting, Baileys presence handling, and debouncer touch helpers to delay inbound flushes while a sender is typing.

## Best Possible Solution

Rebase this feature onto current main, add the public config key to schema/types/docs/generated metadata, make presence subscription and JID mapping bounded, align group key matching with the existing debounce contract, add focused fake-timer coverage, and require redacted real WhatsApp proof before merge.

## Reproduction Assessment

No for the live WhatsApp typing behavior: the PR does not provide runtime proof and I did not exercise a real WhatsApp setup. Yes for the main blockers: the schema rejection and group debounce-key mismatch are reproducible by source inspection against current main and the PR diff.

## Solution Assessment

No. The feature direction is plausible and belongs in the WhatsApp plugin runtime, but this implementation is not the best merge candidate until schema wiring, key matching, bounded presence handling, current-main rebase, and real behavior proof are addressed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Wire the option into the WhatsApp schema:** `src/config/types.whatsapp.ts:82`
  - body: Adding `composingDebounceMs` only to the TypeScript config type leaves the strict runtime WhatsApp schemas unaware of the key. Users who set either the root or per-account option will have config validation reject it as an unknown property, so the feature cannot be enabled from normal config.
  - confidence: 0.93
- **[P2] Match group presence updates to group debounce keys:** `extensions/whatsapp/src/inbound/monitor.ts:486`
  - body: For group chats, this prefix always ends with the E.164 value, but group buffers are keyed by the primary sender identity or `senderJid` before falling back to E.164. In normal group messages where the sender JID is present, `touchByPrefix` will miss the active buffer and composing updates will not extend the debounce window.
  - confidence: 0.87
- **[P2] Deduplicate presence subscriptions per chat:** `extensions/whatsapp/src/inbound/monitor.ts:400-402`
  - body: This calls `presenceSubscribe(remoteJid)` on every inbound message when composing debounce is enabled. Active chats can produce repeated subscribe requests for the same JID, adding avoidable network traffic and increasing the chance of provider throttling; keep a per-monitor subscribed set and retry only after failures.
  - confidence: 0.78
- **[P2] Bound the JID mapping cache:** `extensions/whatsapp/src/inbound/monitor.ts:78`
  - body: `jidToE164Cache` stores one entry per distinct internal JID and never expires them. The mapping is only needed around the next presence event, so long-running accounts that see many contacts can retain stale JIDs indefinitely unless this gets a TTL or size cap.
  - confidence: 0.76
