## Summary

Keep open. Current main does not implement composing-aware WhatsApp inbound debounce, and this external feature PR still has merge-blocking patch defects plus missing real WhatsApp behavior proof.

## What This Changes

The PR adds an opt-in WhatsApp `composingDebounceMs` setting, Baileys presence handling, account plumbing, and shared debouncer touch helpers to extend inbound debounce while a sender is composing.

## Best Possible Solution

Land a rebased WhatsApp-owned opt-in implementation after schema/config/docs/test support, direct and group key matching, bounded and deduplicated presence handling, accurate network disclosure, and redacted real WhatsApp proof.

## Reproduction Assessment

Yes for the PR defects: source and diff inspection show the strict schema gap, group key mismatch, repeated subscription path, and unbounded cache. No live WhatsApp behavior is proven because the Human Verification section is blank.

## Solution Assessment

No. Presence-aware debounce is a plausible WhatsApp-owned direction, but this patch is not the narrowest maintainable merge path until schema support, key matching, bounded presence handling, security disclosure, tests, and real behavior proof are fixed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Wire composingDebounceMs into strict schemas:** `src/config/types.whatsapp.ts:82-85`
  - body: The new config field is added only to the WhatsApp TypeScript type. Because the root and per-account WhatsApp schemas are strict and do not include `composingDebounceMs`, user config that enables the feature is rejected as an unknown key.
  - confidence: 0.97
- **[P2] Use debounce-compatible keys for group composing touches:** `extensions/whatsapp/src/inbound/monitor.ts:489-490`
  - body: For group messages, active debounce buffers prefer `senderJid` before E.164, but the presence handler touches only an E.164 prefix. Normal group composing updates therefore do not extend the matching sender's debounce buffer.
  - confidence: 0.91
- **[P2] Deduplicate presence subscriptions:** `extensions/whatsapp/src/inbound/monitor.ts:397-404`
  - body: The inbound path calls `presenceSubscribe(remoteJid)` whenever composing debounce is configured, including repeated messages from the same chat and cases where no debounce listener is registered. Guard it by active debounce state and per-chat subscription tracking with retry after failure.
  - confidence: 0.89
- **[P2] Bound the LID-to-E164 cache:** `extensions/whatsapp/src/inbound/monitor.ts:78`
  - body: `jidToE164Cache` grows one entry per distinct mapped JID and never expires entries. Since the mapping only needs to bridge nearby composing events, use a short TTL or size cap to avoid stale contact growth in long-running listeners.
  - confidence: 0.84
