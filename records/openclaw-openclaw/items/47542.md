## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Wire composingDebounceMs into strict schemas:** `src/config/types.whatsapp.ts:82-85`
  - body: The new config field is added only to the WhatsApp TypeScript type. Because the root and per-account WhatsApp schemas are strict and do not include `composingDebounceMs`, user config that enables the feature is rejected as an unknown key.
  - confidence: 0.97
- **[P2] Match group composing touches to debounce keys:** `extensions/whatsapp/src/inbound/monitor.ts:489-490`
  - body: For group messages, active debounce buffers prefer primary identity or `senderJid` before E.164, but the presence handler touches only an E.164 prefix. Normal group composing updates therefore do not extend the matching sender's debounce buffer.
  - confidence: 0.91
- **[P2] Guard and deduplicate presence subscriptions:** `extensions/whatsapp/src/inbound/monitor.ts:397-404`
  - body: The inbound path calls `presenceSubscribe(remoteJid)` whenever composing debounce is configured, including repeated messages from the same chat and cases where no debounce listener is registered. This can create unnecessary WhatsApp network traffic and rate-limit noise; guard it by active debounce state and per-chat subscription tracking with retry after failure.
  - confidence: 0.89
- **[P2] Bound the JID mapping cache:** `extensions/whatsapp/src/inbound/monitor.ts:78`
  - body: `jidToE164Cache` grows one entry per distinct mapped JID and never expires entries. Since the mapping only needs to bridge nearby composing events, use a short TTL or size cap to avoid stale contact growth in long-running listeners.
  - confidence: 0.84
