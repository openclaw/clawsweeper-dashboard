## Summary

Keep open: the main bug is real and this PR mostly fixes the hardcoded hint, but PR head still resolves primary model metadata before considering a smaller fallback/runtime model, so a fallback overflow can recommend the wrong reserveTokensFloor.

## What This Changes

The PR replaces the hardcoded context-overflow reserveTokensFloor hint with tiered recommendations based on model context window, session fallback data, and agent contextTokens caps.

## Best Possible Solution

Compute the hint from the effective provider/model and capped context budget that produced the overflow, then fall back to primary/session metadata only when the runtime route is unknown.

## Reproduction Assessment

Yes, source-reproducible: current main hardcodes 20000, and PR head still chooses primary model metadata before the active session fallback when primary metadata resolves. I did not run tests because this review kept the checkout read-only.

## Solution Assessment

No, not yet: the tier helper and cap handling are a good direction, but the hint must use the effective runtime/fallback model that produced the overflow before merge.
