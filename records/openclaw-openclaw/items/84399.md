## Summary

Keep this PR open: the dynamic reserveTokensFloor hint is the right direction, but the latest head still derives the hint from primary model metadata before considering the runtime fallback model, so fallback-route overflows can recommend the wrong tier.

## What This Changes

The PR replaces the hardcoded context-overflow reserveTokensFloor hint with tiered recommendations derived from model context windows, session fallback data, and agent context-token caps.

## Best Possible Solution

Compute the hint from the same effective runtime provider/model and capped context budget that produced the overflow, then keep the tiered guidance with regression coverage for primary, capped, and fallback-model cases.

## Reproduction Assessment

Yes, source-reproducible: current main hardcodes 20000, and PR head still computes the new hint from primary model metadata before session fallback when primary metadata exists. I did not run tests because this was a read-only checkout review.

## Solution Assessment

No, not yet: the tiered helper and cap handling are maintainable, but the hint must use the effective runtime/fallback model before merge.
