## Summary

Keep this PR open: the context-aware hint is the right direction for the linked bug, but the patch currently computes the recommendation without the effective agent context cap that runtime compaction uses, so capped sessions can receive an unsafe value.

## What This Changes

The PR replaces the hardcoded `20000` context-overflow `reserveTokensFloor` hint with tiered values derived from resolved context tokens and adds focused unit coverage.

## Best Possible Solution

Compute the hint from the same effective context-token budget used by runtime compaction, including agent/default caps, then keep the tiered guidance only after capped-window tests cover it.

## Reproduction Assessment

Yes, by source inspection: current `main` hardcodes `20000`, and the PR's new helper omits the effective `agentCfgContextTokens` cap that runtime compaction already honors. I did not run tests because this was a read-only checkout review.

## Solution Assessment

No, not yet: the tiered message is a maintainable direction, but it must use the same effective context-window contract as compaction before it is safe to merge.
