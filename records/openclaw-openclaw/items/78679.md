## Summary

Keep open: the PR is a plausible direction but is not merge-ready because it recognizes registered PI provider stream functions, while the normal embedded session path captures PI's auth-wrapped default stream function.

## What This Changes

The PR updates embedded stream resolution to treat registered PI API provider `streamSimple` functions as default streams and adds unit coverage for `openai-completions` provider streams.

## Best Possible Solution

Route marked embedded-session default wrappers through OpenClaw's boundary-aware transport while preserving explicit custom stream functions, then add focused tests, a single-line changelog entry, and redacted live or loopback proof.

## Reproduction Assessment

Yes, source-level: current main returns any non-`streamSimple` current stream unchanged, while the pinned PI session factory installs an async auth wrapper around `streamSimple`. Passing that wrapper as the cached embedded base stream for an `openai-completions` model reproduces the bypass path.

## Solution Assessment

No, this PR is not the best fix as written because it recognizes registered PI provider streams but not the actual PI session default wrapper. A safer solution should distinguish session-managed defaults from explicit custom stream functions and route only those defaults through OpenClaw transport.
