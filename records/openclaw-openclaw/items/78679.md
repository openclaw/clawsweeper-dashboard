## Summary

Keep open: the branch targets the right regression area, but it does not cover the actual PI coding-agent default stream on current main, so the reported embedded `openai-completions` bypass can still happen. It also still lacks contributor real behavior proof and a changelog entry.

## What This Changes

The PR adds an `isPiNativeDefaultStream` helper, uses it in embedded stream strategy/resolution, and adds regression tests for `getApiProvider("openai-completions")?.streamSimple`.

## Best Possible Solution

Fix the resolver for the `createAgentSession`-installed auth wrapper while preserving explicit custom session streams, then add focused regression coverage, a single-line changelog entry, and redacted live/proxy or terminal proof.

## Reproduction Assessment

Yes, source-level: `@mariozechner/pi-coding-agent` creates the session stream as an auth wrapper around `streamSimple`, while current main only recognizes undefined or module-level `streamSimple` before falling back to the current stream. A focused regression path is to pass a createAgentSession-style wrapper into `resolveEmbeddedAgentStreamFn` for an `openai-completions` model and observe it bypass boundary-aware transport.

## Solution Assessment

No, this PR is not the best fix as written because it recognizes registered PI provider streams but not the actual auth-wrapped session stream installed by PI coding-agent. The safer direction is to route only session-managed defaults for supported transport APIs through OpenClaw transport without swallowing genuinely custom stream functions.
