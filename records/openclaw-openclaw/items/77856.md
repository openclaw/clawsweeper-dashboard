## Summary

Keep open: the reported Telegram mention bug is valid and the PR now has sufficient live proof, but the current patch should not merge because it treats any `tg://user` text link as a bot mention and can bypass `requireMention`.

## What This Changes

The PR updates Telegram `hasBotMention` to accept `text_link` entities for tapped bot mentions and adds helper tests for those cases.

## Best Possible Solution

Validate `text_link` identity against the current bot id or the displayed `@botUsername`, then add positive and negative regressions plus the required changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a `requireMention` Telegram group message whose only bot signal is a `text_link` entity with non-`@botUsername` display text is skipped on current main, and the PR body adds live before/after logs for the same behavior.

## Solution Assessment

No. Accepting the `text_link` shape is the right direction, but URL-prefix matching alone is not the narrowest safe fix because it does not verify that the linked Telegram user is this bot.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Verify the linked Telegram user before accepting text_link:** `extensions/telegram/src/bot/body-helpers.ts:164-165`
  - body: This branch returns `true` for any `tg://user` text link, so a group message linking some other Telegram user would satisfy `requireMention` and wake the bot. Please tie the entity to this bot, such as by comparing the linked user id with `primaryCtx.me.id` or only accepting the fallback when the displayed link text matches `@botUsername`.
  - confidence: 0.9
