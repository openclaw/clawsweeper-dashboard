## Summary

Keep this PR open for repair: it addresses a plausible Telegram `text_link` mention gap, but the current patch broadens mention gating by accepting any `tg://user` link as a bot mention.

## What This Changes

The PR updates Telegram `hasBotMention` to accept `text_link` entities with `tg://user` URLs or matching link text and adds helper tests for those cases.

## Best Possible Solution

Validate `text_link` identity against the current bot id or the displayed `@botUsername`, then add positive and negative regressions plus the required changelog entry.

## Reproduction Assessment

Yes, source-level reproduction is clear for a Telegram message whose only bot signal is a `text_link` entity with non-`@botUsername` display text; current main would skip it under `requireMention`. I did not run a live Telegram scenario in this read-only review, but the PR body now includes live logs and copied output.

## Solution Assessment

No, this is not the best fix yet because URL-scheme matching alone does not prove the linked user is the bot. The safer implementation should compare the `tg://user` id to `ctx.me.id` or require the link text to match `@botUsername`.

## Review Findings
