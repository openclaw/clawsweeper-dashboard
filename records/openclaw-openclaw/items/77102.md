## Summary

Close as not actionable in this repo: the reported immediate-resolve failure is in the external `openclaw-feishu@0.1.4` package, while current OpenClaw’s official Feishu plugin is `@openclaw/feishu` and keeps the channel task pending for the WebSocket lifecycle.

## What This Changes

Report asks OpenClaw to address an external `openclaw-feishu@0.1.4` channel plugin whose `startAccount` promise resolves before the Lark WebSocket lifecycle is established.

## Best Possible Solution

Keep the current gateway lifecycle contract and official `@openclaw/feishu` implementation; users of `openclaw-feishu@0.1.4` should switch to the bundled/current plugin or patch the external package to keep `startAccount` pending until abort/error.

## Reproduction Assessment

No high-confidence current-main reproduction of the external package bug is available from this checkout. Source inspection does confirm the gateway side: any plugin whose `startAccount` promise resolves will be treated as an exited channel.

## Solution Assessment

Yes. Closing as not actionable in this repository is the narrowest maintainable outcome because the official in-repo Feishu plugin already follows the lifecycle pattern, and changing gateway semantics would hide plugin lifecycle contract violations.
