## Summary

Keep open: current main and the latest release still extract every assistant transcript row into session-memory output, and several open linked PRs are competing to fix the same bug. This issue should close only after one canonical fix PR lands.

## What This Changes

The issue asks OpenClaw to stop session-memory files from duplicating assistant replies when raw thinking messages and cleaned transcript-only assistant rows both exist.

## Best Possible Solution

Land one focused session-memory consumer-layer fix using the shared transcript-only assistant predicate or precise parent lineage, with regression coverage for raw thinking plus cleaned assistant rows.

## Reproduction Assessment

Yes. Source inspection shows that a JSONL transcript containing a raw assistant thinking+text row followed by a cleaned text-only assistant row will emit both as identical assistant lines in getRecentSessionContent.

## Solution Assessment

No for the issue's suggested storage-layer removal as the default path; transcript-only rows are used by other session surfaces, so a focused session-memory filter is the narrower maintainable fix.
