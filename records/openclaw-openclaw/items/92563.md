## Summary

Current main and the latest release still read every assistant transcript entry without deduping identical cleaned/raw thinking pairs, and two open linked PRs already claim the fix. Keep this issue open until one PR lands or maintainers choose a storage-layer fix instead.

## What This Changes

Reports duplicated assistant lines in session-memory files when a raw thinking assistant transcript entry and a cleaned text-only assistant entry both exist.

## Best Possible Solution

Land one focused fix that makes session-memory include each assistant response once, with regression coverage for raw thinking plus cleaned text transcript entries and no duplicate storage workaround unless maintainers choose that boundary.

## Reproduction Assessment

Yes at source level, but not from a live thinking-model run in this read-only review. A JSONL file with adjacent assistant entries that extract the same first text block will be emitted twice by current `getRecentSessionContent()`.

## Solution Assessment

Unclear until the linked PRs are reviewed. A session-memory dedupe is a narrow mitigation, but a storage-layer canonicalization may be better if duplicate persistence itself is confirmed as the durable bug.
