## Summary

Keep open: current main and the latest release still include every extracted assistant text line in session-memory output, and several open fix PRs explicitly target this issue. This should remain the canonical bug report until one reviewed fix lands.

## What This Changes

The issue reports duplicate session-memory assistant lines when thinking-enabled transcripts contain both raw thinking-plus-text and cleaned text-only assistant entries.

## Best Possible Solution

Land one focused fix in the session-memory transcript extraction path, preferably using the existing transcript-only helper or lineage-aware filtering with regression coverage, then close duplicate fix PRs.

## Reproduction Assessment

Yes from source inspection: a transcript with a raw assistant thinking-plus-text row followed by a cleaned text-only assistant row will produce two identical `assistant:` lines because both rows pass the current extraction loop. I did not run a live DeepSeek session in this read-only review.

## Solution Assessment

No final merged solution exists yet. The best path is a consumer-side session-memory filter for transcript-only or lineage-duplicate assistant rows, with tests preserving legitimate repeated assistant replies.
