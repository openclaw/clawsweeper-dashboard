## Summary

Keep this open: current main and the latest release still append every assistant text row in the session-memory transcript reader, and several open linked PRs already own candidate fixes for this exact bug.

## What This Changes

The issue reports duplicate assistant lines in session-memory output when raw thinking messages and cleaned text-only assistant entries are both read from session JSONL.

## Best Possible Solution

Land one canonical consumer-layer session-memory filter using the shared transcript-only assistant predicate plus focused regression coverage, then close this issue after the PR merges.

## Reproduction Assessment

Yes at source level. Current main reads both raw assistant thinking rows and cleaned transcript-only assistant rows as ordinary assistant text, so the reported JSONL shape will produce duplicate memory lines; I did not run a live provider flow in this read-only review.

## Solution Assessment

Yes, the best path is a narrow consumer-layer filter for generated session-memory output, not a storage-layer removal that could affect replay/history consumers. The open linked PRs should converge on the shared transcript-only helper and focused tests.
