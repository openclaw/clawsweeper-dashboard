## Summary

Keep open: current main and the latest release still read both raw and transcript-only assistant rows into session-memory output, and the open closing PRs are candidates rather than merged fixes.

## What This Changes

The issue requests stopping session-memory from writing duplicate assistant lines when thinking-enabled transcripts include both raw and stripped assistant rows.

## Best Possible Solution

Land one canonical consumer-layer session-memory filter that skips only actual transcript mirror duplicates while preserving legitimate repeated or standalone gateway-injected assistant rows, with regression coverage, then close this issue after merge.

## Reproduction Assessment

Yes, at source level. Current main extracts and pushes text from every assistant message row, so a raw thinking row followed by a stripped delivery-mirror copy yields duplicate assistant lines.

## Solution Assessment

No, the issue's suggested storage-layer fix is not the safest default. A consumer-layer semantic filter is narrower because replay and history surfaces intentionally keep persisted transcript bookkeeping rows.
