## Summary

Keep open: current main and the latest release still let the session-memory transcript reader emit both raw and transcript-only assistant rows, and multiple open closing PRs are still candidates rather than merged fixes.

## What This Changes

The issue reports duplicate assistant lines in session-memory files when thinking-enabled sessions include both raw assistant content and a stripped transcript-only copy.

## Best Possible Solution

Land a focused session-memory filter using the shared transcript-only assistant helper with regression coverage, then close this issue after the PR merges.

## Reproduction Assessment

Yes at source level. A JSONL transcript containing a raw assistant text block followed by a transcript-only assistant copy will be read twice by current `getRecentSessionContent()` because it extracts and pushes text from every assistant row.

## Solution Assessment

Yes, the best maintainable direction is a consumer-layer filter for transcript-only OpenClaw assistant rows. Changing storage would risk replay and history consumers that intentionally keep those persisted rows.
