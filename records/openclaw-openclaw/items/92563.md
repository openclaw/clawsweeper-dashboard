## Summary

Keep open: current main and the latest release still let the session-memory reader emit a persisted raw-plus-mirror assistant pair, and three open closing PRs already own candidate fixes. This issue should remain the canonical report until maintainers land one fix path.

## What This Changes

The issue reports duplicate assistant lines in session-memory output when thinking-enabled sessions persist both raw assistant text and stripped transcript copies.

## Best Possible Solution

Land one canonical session-memory fix that filters only duplicate transcript mirror rows while preserving legitimate assistant content, then close this issue after the PR merges.

## Reproduction Assessment

Yes. A JSONL transcript containing a raw assistant text block followed by a cleaned or transcript-only assistant copy is read twice by current `getRecentSessionContent()` because it extracts and pushes text from every assistant row.

## Solution Assessment

No, the issue’s storage-only suggested fix is not clearly the best remaining path. Current storage already dedupes some mirrors, so the safer fix is a narrow session-memory consumer filter or lineage-aware duplicate filter with regression coverage.
