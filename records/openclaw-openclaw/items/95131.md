## Summary

Keep open: current main still auto-ensures and repairs the Codex plugin for official `openai/*` model selection using only the model ref and base URL, while runtime execution, diagnostics, and docs already recognize explicit `agentRuntime.id: "openclaw"` or disabled Codex as an opt-out boundary.

## What This Changes

The issue asks OpenAI model-selection/setup to stop auto-installing or re-enabling Codex when configuration explicitly disables or opts out of the Codex runtime.

## Best Possible Solution

Make model-selection install/repair reuse the same resolved provider/model runtime policy used by execution and diagnostics, then align setup tests and docs with the maintainer-approved OpenAI/Codex opt-out contract.

## Reproduction Assessment

Yes from source inspection: current main and `v2026.6.8` both run official `openai/*` model selection through a Codex install/repair predicate that ignores explicit runtime opt-out and disabled Codex config.

## Solution Assessment

Unclear pending maintainer product decision: the narrow technical fix is to make install/repair policy-aware, but maintainers must confirm the intended OpenAI/Codex default and opt-out contract before changing setup behavior.
