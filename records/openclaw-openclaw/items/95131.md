## Summary

Keep open: current main still has a source-reproducible model-selection path that treats official `openai/*` as sufficient to ensure Codex, while the documented and tested runtime path says explicit `agentRuntime.id: "openclaw"` is a real opt-out. This needs maintainer policy confirmation rather than cleanup close.

## What This Changes

The issue asks OpenClaw to stop or clarify official OpenAI model selection auto-installing or re-enabling the Codex plugin when a user disabled Codex or selected the OpenClaw runtime.

## Best Possible Solution

Define and implement one consistent contract: official OpenAI may default to Codex, but explicit OpenClaw runtime policy and disabled Codex intent should be honored by setup, doctor, model selection, and docs.

## Reproduction Assessment

Yes, at source level. Current main still lets official `openai/*` model selection trigger the Codex ensure helper, and that helper can repair/enable or install Codex without checking explicit OpenClaw runtime policy; I did not run a live setup flow.

## Solution Assessment

Unclear pending maintainer policy. The narrow technical fix appears to be making the model-selection ensure predicate use effective runtime policy and disabled-plugin intent, while preserving the documented implicit Codex default for plain official OpenAI refs.
