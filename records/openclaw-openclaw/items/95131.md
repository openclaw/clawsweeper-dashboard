## Summary

Keep open: current main and the latest release still contain the model-selection install path that treats official `openai/*` as requiring Codex without first resolving explicit provider/model `agentRuntime.id: "openclaw"`. Adjacent doctor/config paths now respect that opt-out, so this is a real remaining boundary mismatch rather than a duplicate of the earlier closed cleanup.

## What This Changes

The issue asks whether official `openai/*` model selection should auto-install or re-enable the Codex plugin when the operator intends to keep Codex disabled or use the OpenClaw runtime.

## Best Possible Solution

Keep Codex as the default official OpenAI harness, but make model-selection install/repair use resolved runtime policy so explicit provider/model `agentRuntime.id: "openclaw"` does not install or re-enable Codex.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the central issue: the model-selection predicate requires Codex for official `openai/*` without resolving explicit provider/model runtime policy, and the helper then enables or installs the Codex plugin.

## Solution Assessment

Unclear pending maintainer confirmation of the permanent product contract. If the current docs are authoritative, the narrow maintainable fix is to reuse resolved runtime policy instead of adding a new opt-out config surface.
