## Summary

Keep open: current main and latest release still let official `openai/*` model-selection setup require the Codex plugin without checking explicit provider/model `agentRuntime.id: "openclaw"`, while execution, diagnostics, and docs already treat that runtime policy as an opt-out boundary.

## What This Changes

The issue asks OpenAI model selection/setup to respect an explicit OpenClaw runtime opt-out instead of auto-installing or re-enabling the Codex plugin for official `openai/*` models.

## Best Possible Solution

Make model-selection install and repair reuse the same resolved provider/model runtime policy as execution and doctor, so explicit OpenClaw routes do not install or re-enable Codex while default or explicit Codex routes still do, with aligned docs and regression tests.

## Reproduction Assessment

Yes. Source inspection shows selecting an official `openai/*` route reaches a Codex install/repair helper whose predicate does not consult explicit OpenClaw runtime policy.

## Solution Assessment

Unclear until maintainers confirm the product contract. The narrowest maintainable fix appears to be reusing the existing resolved-runtime boundary in model-selection setup rather than adding a new config surface.
