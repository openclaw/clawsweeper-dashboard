## What This Changes

The issue asks for the Control UI webchat composer to clear and stay clear after a message is sent, preventing accidental duplicate sends.

Land exactly one canonical composer/input-boundary stale replay fix with focused regression and browser proof, then close this issue as implemented and retire duplicate fix attempts.
