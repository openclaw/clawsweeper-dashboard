## Summary

Keep open for normal PR review and merge: current main still lacks this CLI-backend limitation note, and source inspection supports the docs claim that CLI backends use fixed argv while `/status` renders stored thinking/reasoning levels. No blocking review findings were found for https://github.com/openclaw/openclaw/pull/77370.

## What This Changes

This PR adds CLI-backend limitation docs, a thinking-docs cross-link, and an Unreleased changelog entry explaining that `/think` and `/reasoning` do not rewrite fixed CLI backend argv.

## Best Possible Solution

Merge the docs clarification if maintainers agree with documenting the current contract, while keeping dynamic CLI argv or SDK support tracked separately in #77303.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction basis: `/status` reads stored thinking/reasoning levels, while CLI backend execution builds argv from fixed backend config without a thinking or reasoning flag mapping.

## Solution Assessment

Yes for this PR. The docs-only change is the narrowest maintainable way to warn users about current behavior without adding a new plugin-SDK argv seam; a code-level mapping belongs in the related issue.
