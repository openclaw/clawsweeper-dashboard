## Summary

Keep open: current main does not yet include this docs clarification, and source inspection supports the PR’s contract for CLI backends. The proposed docs-only path is consistent with related #77303’s option to mark the toggles unsupported rather than add a new Plugin SDK argv-mapping seam.

## What This Changes

This PR adds CLI-backend limitation docs, cross-links the thinking-level docs, and adds an Unreleased changelog note that `/think` and `/reasoning` are inert for CLI backends.

## Best Possible Solution

Merge the docs/changelog clarification after normal docs formatting and review, while leaving any real argv-mapping support for a separate Plugin SDK design tied to #77303.

## Reproduction Assessment

Yes. Source inspection on current main shows CLI backend args are static and the runner does not consult `thinkingLevel` or `reasoningLevel` when constructing argv, while status rendering still reads those stored levels.

## Solution Assessment

Yes for this PR’s scope. Documenting the current limitation is the narrowest maintainable fix for the user surprise; wiring `thinkingLevel` into backend-specific flags would be separate SDK/API owner work.
