## Summary

Keep open: current main does not yet include this docs clarification, and the PR accurately documents the current CLI-backend behavior with no blocking review findings. Related issue #77303 remains the right place to track any future dynamic CLI argv or SDK-seam work.

## What This Changes

This PR adds CLI-backend limitation docs, a thinking-docs cross-link, and an Unreleased changelog note explaining that `/think` and `/reasoning` do not rewrite fixed CLI backend argv.

## Best Possible Solution

Merge the docs clarification if maintainers agree, while leaving dynamic CLI argv or SDK support tracked in #77303.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction basis: status stores and displays thinking/reasoning levels, while the CLI backend path builds argv from fixed backend config without a thinking/reasoning flag mapping or reasoning payload lane.

## Solution Assessment

Yes. For this PR's stated scope, documenting the current contract is the narrowest maintainable change; mapping levels to CLI-specific flags should remain separate owner work because it would add a plugin-SDK argv seam.
