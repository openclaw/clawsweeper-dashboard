## Summary

Keep open: current main does not yet include this clarification, and source inspection supports the docs claim that CLI backends use fixed argv while status still shows stored thinking/reasoning state. The docs/changelog patch has no blocking review findings.

## What This Changes

This PR adds CLI-backend limitation docs, a thinking-docs cross-link, and an Unreleased changelog note explaining that `/think` and `/reasoning` do not rewrite fixed CLI backend argv.

## Best Possible Solution

Merge the docs clarification if maintainers agree, and continue tracking any future dynamic CLI argv/SDK support in #77303.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction basis: `/status` reads stored thinking/reasoning levels while the CLI backend path builds argv from fixed backend config without a thinking/reasoning flag mapping. I did not need a live Claude CLI run to validate the docs-only claim.

## Solution Assessment

Yes, for this PR's stated scope this is the narrowest maintainable change: it documents the current contract without adding a new plugin-SDK argv seam. A code fix for mapping levels to CLI flags should remain separate owner work under #77303.
