## Summary

Keep this PR open for maintainer review: the patch appears correct and the real-behavior proof is sufficient, but it changes an SDK-exported markdown chunking helper so maintainers should explicitly accept positive Infinity as the no-cap sentinel before merge.

## What This Changes

The PR changes markdown-core chunk rendering so `Number.POSITIVE_INFINITY` returns one unbounded rendered chunk and adds a regression test for that behavior.

## Best Possible Solution

Merge the narrow markdown-core fix after maintainer acceptance of positive Infinity as the shared no-cap sentinel, preserving NaN fallback and finite-limit chunking behavior.

## Reproduction Assessment

Yes. Source inspection shows current main collapses positive Infinity to a one-character limit, and the Signal media sender passes Infinity then keeps only chunk zero; the PR body also provides copied live Signal send output after the patch.

## Solution Assessment

Yes, this is the right fix layer: the bug is in the shared render-aware chunker receiving the no-cap sentinel, so markdown-core fixes Signal without adding a Signal-only workaround. The remaining decision is SDK compatibility acceptance, not a code-shape blocker.
