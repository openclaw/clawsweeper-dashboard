## Summary

Keep open for maintainer review: this PR addresses a source-reproducible Signal caption truncation path with a narrow shared markdown-core fix and now supplies real behavior proof, but the SDK-exported helper semantic change still needs maintainer compatibility acceptance before merge.

## What This Changes

This PR makes `renderMarkdownIRChunksWithinLimit` return the whole Markdown IR as one rendered chunk when `limit` is `Number.POSITIVE_INFINITY` and adds a regression test.

## Best Possible Solution

Merge the narrow markdown-core fix after maintainer acceptance of Infinity as the shared no-cap sentinel, preserving the existing NaN fallback and finite-limit behavior.

## Reproduction Assessment

Yes for source-level reproduction: current main normalizes `Number.POSITIVE_INFINITY` to `1`, and the Signal media path passes Infinity then keeps only chunk zero. I did not run a live Signal repro in this read-only review.

## Solution Assessment

Yes, this is the best fix layer: the bug sits in the shared render-aware chunker receiving the no-cap sentinel, so fixing markdown-core covers Signal without a Signal-only workaround while preserving invalid-limit fallback behavior.
