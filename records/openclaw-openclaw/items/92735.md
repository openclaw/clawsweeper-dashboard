## Summary

Keep open: the patch appears to fix the reported Signal caption truncation at the shared markdown-core layer, but this external PR still lacks after-fix real behavior proof and intentionally changes an SDK-exported helper's Infinity semantics.

## What This Changes

The PR makes `renderMarkdownIRChunksWithinLimit` treat `Number.POSITIVE_INFINITY` as an unbounded chunk limit and adds a regression test for that sentinel.

## Best Possible Solution

Merge the narrow markdown-core fix after the contributor adds redacted after-fix Signal/runtime proof and maintainers accept Infinity as the SDK helper's no-cap sentinel.

## Reproduction Assessment

Yes for source-level reproduction: current main normalizes `Number.POSITIVE_INFINITY` to `1`, and the Signal media path then sends only the first chunk as the caption. I did not run a live Signal send in this read-only review.

## Solution Assessment

Yes, this is the best fix shape: the bug is in the shared render-aware chunker that receives the no-cap sentinel, and fixing it there covers Signal without adding a Signal-only workaround while leaving NaN fallback behavior intact.
