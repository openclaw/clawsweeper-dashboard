## Summary

Keep open. Current main still uses the hardcoded OpenAI-compatible generated session prefix, but this PR is not merge-ready because it derives the provider prefix before backend model override resolution and omits the required changelog entry for a user-visible, backward-incompatible session-key change.

## What This Changes

The PR changes OpenAI-compatible chat completions to derive generated session-key prefixes from the selected agent's default provider, threads a pre-resolved agent id into request context, and adds provider-prefix tests.

## Best Possible Solution

Maintainers should choose the intended generated-key namespace; if provider-specific keys are desired, derive the prefix from the final accepted backend model selection, keep explicit `x-openclaw-session-key` authoritative, cover default and override paths, and document the change in `CHANGELOG.md`.

## Reproduction Assessment

Yes. Static inspection reproduces the current-main behavior because `/v1/chat/completions` still passes `sessionPrefix: "openai"`; the PR defect is reproducible with an Anthropic-default agent and `x-openclaw-model: openai/gpt-5.4`, which would run an OpenAI override under an Anthropic-prefixed generated key.

## Solution Assessment

No. The PR is focused, but it derives the prefix from the agent default before backend model override resolution and lacks the required changelog note for the user-visible namespace change.
