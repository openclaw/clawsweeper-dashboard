## Summary

Keep open. Current main does not implement the PR's intended OpenAI-compatible HTTP session-prefix change, and the proposed patch still has a correctness defect for request-level model overrides plus a missing required changelog entry.

## What This Changes

The PR adds provider-prefix resolution for OpenAI-compatible HTTP session keys, threads a pre-resolved agent id through gateway request context, and adds gateway tests for provider-specific generated session keys.

## Best Possible Solution

Maintainers should first confirm the intended namespace for generated OpenAI-compatible HTTP sessions. If provider-prefixed keys are desired, derive the prefix from the accepted backend model selection after validating `x-openclaw-model`, preserve explicit `x-openclaw-session-key`, cover default and override paths in tests, and add the required changelog note.

## Reproduction Assessment

Yes. Static inspection is enough for the main-branch behavior: `/v1/chat/completions` still calls request-context resolution with `sessionPrefix: "openai"`. The PR defect can be reproduced conceptually with an Anthropic-default agent plus `x-openclaw-model: openai/gpt-5.4`, which would dispatch an OpenAI override under an Anthropic-prefixed generated session key.

## Solution Assessment

No. The PR is a plausible focused fix, but it derives the prefix before model-override resolution and omits the required changelog entry for a user-visible, backward-incompatible session-key change.
