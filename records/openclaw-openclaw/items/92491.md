## Summary

The real CLI proof now resolves the previous evidence blocker, but this branch puts a shared configuration invariant only in two MCP helper writes, leaving generic config writers inconsistent and the reported warning flood intact for existing or externally written records. A broader active PR covers the schema, helper, documentation, and warning-cadence boundaries, but it is not merged, so this PR should remain open pending a maintainer choice rather than close as superseded.

## What This Changes

The PR exports the MCP stdio environment-key safety predicate, rejects blocked keys during MCP set and update helper writes, and adds helper-level tests for blocked, clean, and URL-based configurations.

## Best Possible Solution

Choose one canonical implementation that centralizes transport-aware blocked-key detection in the shared MCP configuration contract, applies it consistently to supported writers, retains runtime filtering for noncanonical records, and emits each legacy warning at most once per server and key.

## Reproduction Assessment

Yes. Current-main source shows the MCP writer accepts blocked keys while stdio runtime resolution drops and warns about them, and the updated PR supplies real patched-CLI output for the rejection and safe-key control cases.

## Solution Assessment

No, not in its current boundary. Early rejection is appropriate, but the narrow helper-only implementation leaves the shared schema and generic editors inconsistent and does not resolve repeated warnings for records outside those helpers.
