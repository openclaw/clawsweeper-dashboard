## Summary

Keep open and do not merge as-is. Current main does not have the requested `includedWorkDirs` support, so the PR is relevant, but the branch still has a blocking filesystem-containment issue and two concrete runtime regressions around media-tool root propagation and canonical edit behavior.

## What This Changes

The PR adds `agents.list[].includedWorkDirs` config/schema support and threads extra agent work roots through agent scope, path guards, `apply_patch`, PI file tools, media tools, tests, and some unrelated CI/QA files.

## Best Possible Solution

Land the feature only after included roots reject filesystem roots, all root-aware tool entrypoints derive the same agent-scoped roots from config/session context, canonical edit semantics are preserved, and unrelated CI/QA edits are split or explicitly approved.

## Reproduction Assessment

Yes. On the PR head, configure `tools.fs.workspaceOnly=true` with `includedWorkDirs: ["/"]`; the resolver returns `/`, and the allowed-root matcher then treats arbitrary absolute host paths as in bounds.

## Solution Assessment

No. A per-agent allowlist is a reasonable core design, but this implementation is not the safest maintainable version until containment, entrypoint consistency, and edit semantics are fixed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Reject filesystem-root included work dirs:** `src/agents/agent-scope.ts:191-197`
  - body: `resolveAgentIncludedWorkDirs` accepts `/` or a drive root and forwards it into the allowed-root list. In `tools.fs.workspaceOnly` mode, that makes arbitrary absolute host paths fall inside the agent boundary for read/write/edit/apply_patch, so reject filesystem roots before returning included roots.
  - confidence: 0.9
- **[P2] Derive included roots in OpenClaw tool factories:** `src/agents/openclaw-tools.ts:148`
  - body: `createOpenClawTools` only forwards included roots when callers pass `options.includedWorkDirs`. Gateway and inline-command paths already pass config plus session/agent context but not this new option, so configured included roots still do not work for image/PDF tools outside the coding-tools path.
  - confidence: 0.86
- **[P2] Preserve whitespace-only edit targets:** `src/agents/pi-tools.read.ts:667`
  - body: The canonical edit adapter drops replacements whose `oldText` is whitespace-only. Valid edits that target indentation or blank-line blocks are silently discarded, and all-whitespace edit batches fall back to the legacy tool with the wrong payload shape.
  - confidence: 0.88
