## Summary

Keep open; current main still lacks the requested capability, but this PR is not ready to merge because the branch still has blocking filesystem-containment and tool-entrypoint gaps. Prior bot review context is useful here: the remaining blockers are centered on root allowlist rejection, OpenClaw tool factory propagation, and canonical edit semantics.

## What This Changes

The PR adds `agents.list[].includedWorkDirs` to config/schema and threads explicit extra agent work roots through path-guarded coding/media tools, tests, and related CI/QA adjustments.

## Best Possible Solution

Land a narrowed implementation that follows the existing workspace-root containment precedent, covers every path-guarded entrypoint consistently, preserves edit semantics, and includes the required changelog/docs coverage.

## Reproduction Assessment

Yes. On the PR head, configure `tools.fs.workspaceOnly=true` with `agents.list[].includedWorkDirs: ["/"]`; the resolver forwards `/`, and the multi-root matcher then treats arbitrary absolute paths as inside the allowed roots.

## Solution Assessment

No. A per-agent allowlist is a reasonable core design, but this implementation should reject filesystem roots, derive configured roots in `createOpenClawTools`, preserve whitespace-only edit targets, and separate or justify the unrelated CI/QA edits before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Reject filesystem-root included work dirs:** `src/agents/agent-scope.ts:191-194`
  - body: `resolveAgentIncludedWorkDirs` accepts `/` or a drive root and forwards it into the allowed-root list. In `tools.fs.workspaceOnly` mode, that makes arbitrary absolute host paths fall inside the agent boundary for read/write/edit/apply_patch. Reject filesystem roots before returning included roots, matching the existing workspace normalization precedent.
  - confidence: 0.9
- **[P2] Derive included roots in OpenClaw tool factories:** `src/agents/openclaw-tools.ts:145`
  - body: `createOpenClawTools` only forwards included roots when callers pass `options.includedWorkDirs`. Gateway and inline-command paths already pass config plus session/agent context but not this new option, so configured included roots still do not work for image/PDF tools outside the coding-tools path. Derive the agent roots here when the option is omitted.
  - confidence: 0.86
- **[P2] Preserve whitespace-only edit targets:** `src/agents/pi-tools.read.ts:665-668`
  - body: The canonical edit adapter drops replacements whose `oldText` is whitespace-only. Valid edits that target indentation or blank-line blocks are silently discarded, and all-whitespace edit batches fall back to the legacy tool with the wrong payload shape. Accept whitespace-only strings as valid edit targets.
  - confidence: 0.88
- **[P3] Add the required changelog entry:** `src/config/types.agents.ts:75`
  - body: This adds a public config/runtime capability, but the PR diff does not update `CHANGELOG.md`. Add a single-line user-facing entry for `agents.list[].includedWorkDirs` before merge.
  - confidence: 0.82
