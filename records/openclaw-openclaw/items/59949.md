## What This Changes

The PR adds `agents.list[].includedWorkDirs` config support and threads extra per-agent allowed roots through path policy, `apply_patch`, PI file tools, image/PDF guards, schema/help, and targeted tests.

## Best Possible Solution

Land a narrowed branch that restores private QA and Windows path behavior, keeps the per-agent allowlist explicit and documented, and includes redacted real proof of included-root allow plus outside-root rejection.

## Reproduction Assessment

Yes for the review findings by source inspection: PR head still bypasses single-root Windows sandbox input normalization and lets private QA loading fall through to runtime imports. The `includedWorkDirs` request itself is a new feature, not a current-main bug reproduction.

## Solution Assessment

No. A per-agent allowlist is a plausible core direction, but this branch should not merge until the unrelated private QA fallback is corrected, Windows single-root path normalization is preserved, and real included-root behavior is shown in a real setup.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve Windows sandbox input normalization:** `src/agents/path-policy.ts:71-74`
  - body: The win32 branch now resolves `params.candidate` directly instead of first using `resolveSandboxInputPath`, so single-root callers of `toRelativeWorkspacePath`/`toRelativeSandboxPath` lose the existing `~` and sandbox input normalization. Inputs like `~/file.txt` can become `<cwd>\~\file.txt` rather than the normalized target; keep the resolved-input step before the platform branch.
  - confidence: 0.84
- **[P2] Keep private QA runtime imports behind the opt-in:** `src/cli/program/private-qa-cli.ts:78-86`
  - body: When source resolution returns null and `loadPrivateQaCliModule()` is called without explicit params, this now tries runtime `qa-lab` imports instead of preserving the existing env/source-checkout error. A built runtime module can then load the private command without `OPENCLAW_ENABLE_PRIVATE_QA_CLI=1`, so keep runtime imports behind the same opt-in gate.
  - confidence: 0.86
- **[P2] Only ignore missing candidate modules during QA fallback:** `src/cli/program/private-qa-cli.ts:88-90`
  - body: This catches every `ERR_MODULE_NOT_FOUND` while importing a candidate, but Node uses that same code for dependencies missing inside `qa-lab`. A broken private QA module can be hidden while the loop tries another specifier; continue only when the missing module is the candidate itself.
  - confidence: 0.78
