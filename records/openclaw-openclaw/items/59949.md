## Summary

Keep this PR open: current main still lacks the requested agent-level extra-root config, but the change is a new filesystem-access feature with overlapping open designs, insufficient real behavior proof, and one remaining correctness finding in the private QA loader rebase change.

## What This Changes

Adds `agents.list[].includedWorkDirs` per-agent extra work roots and threads them through config/schema, path guards, `apply_patch`, PI file tools, and image/PDF media path handling, with targeted tests and a private QA loader rebase change.

## Best Possible Solution

Choose one maintainer-approved multi-root filesystem contract that composes with existing hardened path guards, keep private QA loader semantics intact, and land with real allow/reject runtime proof.

## Reproduction Assessment

No for the central request as a bug: this is a new filesystem-access capability, and current main has no `includedWorkDirs` contract to reproduce against. Yes for the review blocker by source inspection: the PR-head private QA loader continues on every `ERR_MODULE_NOT_FOUND`.

## Solution Assessment

Unclear. The per-agent allowlist may be viable, but it is not clearly the best solution until maintainers align it with `tools.fs.allowedRoots` and multi-workspace proposals, and the unrelated private QA loader fallback is fixed or removed.
