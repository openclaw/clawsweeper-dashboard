## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve Windows sandbox path normalization:** `src/agents/path-policy.ts:72-74`
  - body: This win32 branch now resolves the raw candidate directly with `path.win32.resolve`, bypassing `resolveSandboxInputPath()` used by current main. Existing callers of `toRelativeWorkspacePath` and `toRelativeSandboxPath` can therefore interpret inputs like `~/file.txt` as a literal `~` path under the cwd instead of the normalized sandbox path.
  - confidence: 0.86
- **[P2] Keep private QA loading behind the source-checkout gate:** `src/cli/program/private-qa-cli.ts:82-86`
  - body: When source-checkout resolution returns null and no explicit test params are passed, this now tries runtime `qa-lab` imports instead of failing closed. That changes the private QA CLI contract so availability can depend on package layout rather than the existing source-checkout gate.
  - confidence: 0.84
- **[P2] Only ignore missing candidate QA modules:** `src/cli/program/private-qa-cli.ts:88-90`
  - body: This catches every `ERR_MODULE_NOT_FOUND` from importing a candidate, but Node uses the same code when a dependency inside `qa-lab` is missing. A broken private QA module can be hidden while the loader tries another specifier, so continue only when the missing module is the candidate itself.
  - confidence: 0.78
