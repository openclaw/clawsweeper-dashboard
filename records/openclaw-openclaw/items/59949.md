## Summary

Keep open: the requested feature is not on current main, and the PR still has source-backed correctness/security blockers plus insufficient real behavior proof for a filesystem access expansion.

## What This Changes

Adds `agents.list[].includedWorkDirs` as a per-agent extra filesystem-root allowlist and threads it through config/schema, path policy, `apply_patch`, PI file tools, and image/PDF media guards.

## Best Possible Solution

Land a narrowed, maintainer-approved extra-root design after resolving the overlap with https://github.com/openclaw/openclaw/pull/43565, fixing the path and QA regressions, and adding redacted real tool proof for included-root allow plus outside-root rejection.

## Reproduction Assessment

Yes for the review findings by source inspection: PR head still bypasses single-root Windows sandbox input normalization and changes private QA fallback behavior. The requested `includedWorkDirs` support itself is a new feature, not a current-main bug reproduction.

## Solution Assessment

No. A per-agent allowlist may be a reasonable design, but this branch should not merge until the Windows and private-QA regressions are fixed, the overlapping allowed-roots direction is settled, and real behavior proof is supplied.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve sandbox input normalization on Windows:** `src/agents/path-policy.ts:71-74`
  - body: PR head moves `resolveSandboxInputPath` out of the win32 path for `toRelativeWorkspacePath`/`toRelativeSandboxPath`. Single-root callers now resolve inputs like `~/file.txt` as `<cwd>\~\file.txt` instead of using sandbox input normalization; keep that normalization before the platform-specific boundary comparison.
  - confidence: 0.84
- **[P2] Keep private QA runtime imports behind the opt-in:** `src/cli/program/private-qa-cli.ts:78-86`
  - body: When the source-checkout specifier is unavailable and the caller supplies no params, this falls through to runtime specifier imports even if `OPENCLAW_ENABLE_PRIVATE_QA_CLI` is unset. That weakens the existing env-gated private QA contract and makes availability depend on local file layout.
  - confidence: 0.86
- **[P2] Only ignore candidate-module misses in QA fallback:** `src/cli/program/private-qa-cli.ts:88-90`
  - body: This catches every `ERR_MODULE_NOT_FOUND` from importing a candidate, but Node uses the same code for missing dependencies inside `qa-lab`. A broken private QA module can be masked while the loop loads another specifier; continue only when the missing module is the candidate itself.
  - confidence: 0.78
