## Summary

Keep this PR open, but it is not merge-ready: the feature is not on current main, the branch still has source-visible regressions in Windows path normalization and private QA loading, and its proof does not show a real OpenClaw tool allowing an included root while rejecting an outside root.

## What This Changes

Adds `agents.list[].includedWorkDirs` as a per-agent extra work-root allowlist across config, path policy, `apply_patch`, PI file tools, image/PDF media guards, and also changes private QA loader behavior.

## Best Possible Solution

Land a narrowed, maintainer-approved extra-root design that aligns with or supersedes https://github.com/openclaw/openclaw/pull/43565, preserves existing Windows/path and private-QA contracts, and includes redacted real tool allow/reject proof.

## Reproduction Assessment

Yes for the review findings by source inspection: PR head still changes the single-root Windows normalization path and private QA fallback behavior. The `includedWorkDirs` request itself is a new feature, not a current-main bug reproduction.

## Solution Assessment

No. A per-agent extra-root allowlist may be a reasonable design, but this branch should not merge until the Windows/private-QA regressions are fixed, the overlap with the allowed-roots PR is settled, and real behavior proof is supplied.
