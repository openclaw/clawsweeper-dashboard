## Summary

Keep this PR open: current main still lacks the requested `includedWorkDirs` feature, but the latest branch has source-inspection blockers in Windows path normalization, private QA loading, and insufficient real behavior proof for a new filesystem-access surface.

## What This Changes

The PR adds `agents.list[].includedWorkDirs` and threads extra per-agent filesystem roots through config normalization/schema, agent scope, `apply_patch`, PI file tools, and image/PDF path guards.

## Best Possible Solution

Land a narrowed branch that preserves existing path normalization and private-QA gates, keeps included roots explicit per agent, and includes redacted real behavior proof of allowed-root access plus outside-root rejection.

## Reproduction Assessment

Yes for the review findings by source inspection: PR head bypasses `resolveSandboxInputPath` in the single-root Windows branch and adds private QA runtime fallback behavior absent from current main. The `includedWorkDirs` request itself is a new feature, not a current-main bug reproduction.

## Solution Assessment

No. A per-agent allowlist is a plausible core direction, but this branch should not merge until it restores existing path/private-QA invariants and supplies real OpenClaw behavior proof.

  - body: The Windows branch now resolves the raw candidate with `path.win32.resolve` instead of first using `resolveSandboxInputPath`, so single-root callers of `toRelativeWorkspacePath`/`toRelativeSandboxPath` no longer get the existing `~` and sandbox input normalization.   - body: When source resolution returns null and this helper is called without explicit params, it now tries runtime `qa-lab` imports instead of preserving the existing source/env gate. A present runtime facade can therefore load private QA without `OPENCLAW_ENABLE_PRIVATE_QA_CLI=1`, making command availability depend on file layout rather than the explicit opt-in.
  - confidence: 0.84
- **[P2] Only ignore missing candidate modules during QA fallback:** `src/cli/program/private-qa-cli.ts:87-90`
  - body: This catches every `ERR_MODULE_NOT_FOUND` from importing a candidate, but Node also uses that code for dependencies missing inside `qa-lab`. A broken private QA module can be hidden while the loop tries another specifier, so only continue when the missing module is the candidate itself.
  - confidence: 0.78
