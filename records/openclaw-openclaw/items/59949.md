## Summary

Keep open. Current main still lacks `includedWorkDirs`, and the PR is useful source work, but it changes a security-sensitive filesystem boundary and still has merge-blocking containment and tool-consistency issues.

## What This Changes

The PR adds `agents.list[].includedWorkDirs` to config/schema and threads extra allowed roots through agent scope, path guards, `apply_patch`, PI file tools, image/PDF handling, tests, and some QA/workflow adjustments.

## Best Possible Solution

Land the capability only after included roots reject filesystem roots, all tool factories derive the agent's included roots consistently, edit semantics are preserved, unrelated QA/workflow changes are split or justified, and focused docs/changelog plus regression tests cover the public config field.

## Reproduction Assessment

Yes. Configure `tools.fs.workspaceOnly=true` with `includedWorkDirs: ["/"]`; the PR resolver accepts the root and the boundary matcher then treats arbitrary absolute paths as inside the allowed roots.

## Solution Assessment

No. A per-agent allowlist is a reasonable core direction, but this implementation is not merge-ready until the filesystem boundary and inconsistent entrypoint handling are fixed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Reject filesystem-root included work dirs:** `src/agents/agent-scope.ts:191-197`
  - body: `resolveAgentIncludedWorkDirs` accepts `/` or a drive root and forwards it into the allowed-root list. With that value configured, every absolute host path is inside the boundary, so `tools.fs.workspaceOnly` no longer contains read/write/edit/apply_patch for the agent. Reject filesystem roots before adding them to the allowlist.
  - confidence: 0.9
- **[P2] Derive included roots in OpenClaw tool factories:** `src/agents/openclaw-tools.ts:145`
  - body: `createOpenClawTools` forwards included roots only when callers pass `options.includedWorkDirs`. Gateway and inline-command paths already pass config plus session/agent context but not this new option, so configured included roots still do not work for image/PDF tools outside `createOpenClawCodingTools`. Resolve the agent's included roots here when the option is omitted.
  - confidence: 0.86
- **[P2] Preserve whitespace-only edit targets:** `src/agents/pi-tools.read.ts:665-668`
  - body: The canonical edit adapter drops replacements whose `oldText` is whitespace-only, which silently discards valid edits targeting indentation or blank-line blocks. Accept whitespace-only strings as valid `oldText` values instead of filtering them out.
  - confidence: 0.88
