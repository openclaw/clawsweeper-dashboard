## Summary

Keep open. Current main does not have an `includedWorkDirs` agent contract, so this PR is not obsolete, but it needs maintainer filesystem-contract/security review, stronger real behavior proof, and fixes for remaining source-level regressions before merge.

## What This Changes

The PR adds `agents.list[].includedWorkDirs` and threads explicit extra roots through agent config/schema, workspace path guards, `apply_patch`, PI read/write/edit tools, and image/PDF media handling.

## Best Possible Solution

Pick one maintainer-approved multi-root workspace contract, keep existing hardened path normalization and private QA loader semantics intact, and land only with real runtime proof that included roots are allowed while outside roots still fail.

## Reproduction Assessment

No for the central request as a current-main bug: this is a new config capability, and current main has no `includedWorkDirs` contract. Yes by source inspection for the review findings: PR head bypasses `resolveSandboxInputPath` in the win32 helper and catches every `ERR_MODULE_NOT_FOUND` in the private QA fallback.

## Solution Assessment

Unclear. The per-agent allowlist may be viable, but it is not clearly the best OpenClaw contract until maintainers choose how it coexists with https://github.com/openclaw/openclaw/pull/48681 and the prior allowed-roots designs.
