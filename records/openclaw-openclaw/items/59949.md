## Summary

Keep open: current main still lacks the requested per-agent extra-workdir support, but this branch is not merge-ready because it regresses current sandbox read-mount behavior, leaves path/private-QA review blockers, and does not yet provide real OpenClaw behavior proof.

## What This Changes

Adds `agents.list[].includedWorkDirs` config support and threads per-agent extra filesystem roots through agent scope, path policy, `apply_patch`, PI file tools, image/PDF guards, schema/help, and tests.

## Best Possible Solution

Land a narrowed filesystem-root contract after maintainers decide how this should relate to `tools.fs.allowedRoots`, preserving existing sandbox mount and private QA gates and adding redacted real behavior proof.

## Reproduction Assessment

Yes for the review findings by source inspection against PR head and current main: the read-only mount mapping is dropped, Windows single-root normalization is bypassed, and private QA fallback behavior is visible in the diff. The requested `includedWorkDirs` capability itself is a new feature, not a current-main bug reproduction.

## Solution Assessment

No. A per-agent allowlist may be a reasonable direction, but this branch should preserve current mount/path/QA behavior, align with the overlapping allowed-roots PR, and prove the real tool behavior before merge.

The production subcommand loader calls this helper without params, so a present runtime `qa-lab` module can load without the existing `OPENCLAW_ENABLE_PRIVATE_QA_CLI=1` source-checkout gate.
  - confidence: 0.86

  - body: This catches every `ERR_MODULE_NOT_FOUND` while importing a candidate, including dependencies missing inside `qa-lab`. That can hide a broken private QA module and continue to another specifier; only continue when the missing module is the candidate itself.
