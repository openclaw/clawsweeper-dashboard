## Summary

Keep open, but do not merge as-is. Current main still lacks `includedWorkDirs`, and the PR is useful source work, but the latest diff still has a security-sensitive filesystem containment hole plus inconsistent media-tool entrypoint handling and an edit-schema regression.

## What This Changes

The PR adds `agents.list[].includedWorkDirs` and threads extra workspace roots through agent scope, path policy, `apply_patch`, PI file tools, image/PDF guards, config schema/help, tests, and related workflow/QA adjustments.

## Best Possible Solution

Land a revised, focused implementation that rejects filesystem roots, derives included roots at every runtime tool entrypoint, preserves edit semantics, splits or justifies unrelated workflow/QA changes, and adds the required docs/changelog plus focused containment tests.

## Reproduction Assessment

Yes. Configure `tools.fs.workspaceOnly=true` with `includedWorkDirs: ["/"]`; the PR resolver accepts the root and `resolvePathWithinRoots` then treats arbitrary absolute paths as inside the allowed roots.

## Solution Assessment

No. A per-agent allowlist is a reasonable core direction, but this patch is not the best merge shape until it matches the existing workspace root guard, keeps all tool entrypoints consistent, and preserves valid edit behavior.
