## Summary

This remains a valid but low-priority feature request: current main and v2026.6.5 intentionally omit session identifiers from OTel export, while Langfuse currently supports `langfuse.session.id`/`session.id` for grouping. The diagnostics plugin already receives session identity internally, so no new core seam appears necessary, but prior unmerged attempts and the existing redaction contract make the identifier’s privacy, scope, stability, and rotation an explicit maintainer product/security decision.

## What This Changes

Add an opt-in diagnostics-otel configuration that exports a privacy-preserving session correlation attribute for Langfuse grouping.

## Best Possible Solution

Keep the behavior opt-in and owned by diagnostics-otel, but first approve a precise correlation contract: emit `langfuse.session.id` on the root run span using a scoped opaque identifier, preferably keyed rather than a plain hash, with documented stability/rotation and no raw session-key export; then cover config, docs, redaction invariants, and a live Langfuse grouping check.

## Reproduction Assessment

Not applicable: this is a new capability request, and source plus shipped documentation confirm that session identity is intentionally omitted today rather than malfunctioning.

## Solution Assessment

Unclear as proposed: diagnostics-otel is the correct owner boundary and opt-in is appropriate, but a boolean with plain hashing or an ad hoc composite does not adequately define privacy, stability, rotation, or collision semantics.
