## Summary

Keep this issue open. Current main and the latest release intentionally suppress session identifiers, while Langfuse supports recognized session attributes and no canonical OpenClaw item owns this missing capability; maintainer agreement is still needed on the privacy and configuration contract.

## What This Changes

Add opt-in, privacy-preserving Langfuse session grouping to diagnostics OTel spans.

## Best Possible Solution

Define a plugin-owned opt-in correlation policy that emits one standard opaque session or conversation attribute across every span in a trace, with documented stability and rotation semantics, default omission, and no raw or reversibly derived session key.

## Reproduction Assessment

Yes, at source level. Current main carries session identity in trusted diagnostic run events but deliberately removes every Langfuse-recognized session or conversation attribute from OTel export.

## Solution Assessment

No, not fully as proposed. An opt-in opaque attribute is a plausible direction, but root-span-only emission and unspecified hash or composite choices do not establish the required propagation, stability, rotation, and privacy contract.

## Risks / Open Questions

- The requested root-span-only behavior may be incomplete because Langfuse recommends propagating trace-level grouping attributes to every span for reliable filtering and aggregation.
- The proposed option does not define identifier scope, stability, rotation, collision behavior, or its relationship to existing diagnostics privacy controls.
