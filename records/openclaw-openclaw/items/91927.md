## What This Changes

The issue requests an opt-in diagnostics OTel setting that emits a privacy-preserving `langfuse.session.id` so Langfuse can group traces by session.

## Best Possible Solution

Have a maintainer sponsor a narrow diagnostics-otel design that keeps raw session-key redaction, chooses root-only versus all-span propagation, adds default-off privacy docs/config metadata, and then lands focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection is high-confidence that current main and `v2026.6.8` lack the requested config key and `langfuse.session.id` emission path.

## Solution Assessment

Unclear as a product decision. The default-off hashed identifier is plausible, but maintainers should decide the privacy model and propagation scope before another implementation PR is queued.
