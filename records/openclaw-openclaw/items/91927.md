## What This Changes

Request to add an opt-in diagnostics-otel setting that exports a non-raw `langfuse.session.id` on OTel spans for Langfuse session grouping.

## Best Possible Solution

Have a maintainer sponsor a narrow diagnostics-otel design that keeps raw session-key redaction, decides root-only versus all-span propagation, documents the default-off privacy model, and lands focused config/service tests.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection is high-confidence that current main and `v2026.6.8` lack both the requested config key and any `langfuse.session.id` emission path.

## Solution Assessment

Unclear as a product decision. A default-off hashed identifier is plausible, but maintainers should decide the privacy model and whether session attributes belong only on root spans or on all spans before implementation.
