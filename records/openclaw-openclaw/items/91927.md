## Summary

Keep open for maintainer/product review: current main intentionally omits session identifiers from diagnostics-otel spans, and adding `langfuse.session.id` would be a new default-off telemetry/config surface with privacy implications rather than an already-implemented bug fix.

## What This Changes

Request an opt-in diagnostics-otel setting that exports a hashed or otherwise non-raw session identifier as `langfuse.session.id` for Langfuse session grouping.

## Best Possible Solution

Define an explicit default-off session-correlation policy for diagnostics-otel, choose the safe identifier shape, then update config schema/help/docs and focused exporter tests together.

## Reproduction Assessment

Yes from source inspection for the current behavior: diagnostic events carry `sessionKey`/`sessionId`, but diagnostics-otel drops session attributes and never emits `langfuse.session.id` or `session.id`. A live Langfuse repro would still be useful before implementation proof.

## Solution Assessment

The proposed opt-in direction is plausible, but maintainers should first decide the privacy-safe identifier contract and whether a new config key is warranted under the repo's high config-surface bar.
