## Summary

Keep open: current main and the latest release still do not provide a `diagnostics.otel` opt-in for Langfuse session grouping, and the requested behavior is a security-sensitive feature/config decision rather than stale cleanup.

## What This Changes

The issue requests a default-off `diagnostics-otel` option that exports a non-raw stable session identifier as `langfuse.session.id` for Langfuse session grouping.

## Best Possible Solution

Add a plugin-owned, default-off, non-raw stable session grouping attribute in `diagnostics-otel` with focused tests and docs, or explicitly decide that OpenClaw will not expose session grouping identifiers to OTLP backends.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes from source inspection: current main drops session identifiers, omits session identity from run span attributes, and has no `sessionAttribute`/`langfuse.session.id` config or exporter path.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose the privacy contract; a default-off plugin-owned option is a plausible direction, but the exact identifier derivation and config shape need product/security approval.
