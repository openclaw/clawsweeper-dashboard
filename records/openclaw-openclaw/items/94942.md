## Summary

Keep open: this is a viable candidate fix for the open Matrix E2EE fake-indexeddb OOM issue, and the previous metadata-store blocker from the closed Matrix-local PR is addressed. I found no blocking patch defect, but the workaround intentionally owns fake-indexeddb private transaction state, so maintainers should treat that as the main merge-risk decision before landing.

## What This Changes

The PR installs a Matrix-local fake-indexeddb transaction pruner from the Matrix crypto runtime and adds tests for both Matrix rust-crypto database suffixes, unrelated databases, and active/inactive transactions.

## Best Possible Solution

Land the Matrix-owned pruner once normal CI is green, while keeping it scoped to the two rust-crypto database suffixes and documenting/owning the pinned fake-indexeddb private contract in future dependency upgrades.

## Reproduction Assessment

Yes, source-level. Current main routes Matrix E2EE through fake-indexeddb 6.2.5, and upstream fake-indexeddb source shows finished transactions remain in the backing array; I did not run the long-lived live Matrix OOM loop in this read-only review.

## Solution Assessment

Yes. The Matrix-local wrapper is narrower than the root dependency-patch approach, reaches the published plugin runtime path, and now covers both upstream rust-crypto database suffixes; an upstream fake-indexeddb fix would be cleaner long-term but is not required for this OpenClaw mitigation.
