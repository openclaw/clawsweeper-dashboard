## Summary

Keep open. Current main already provides a bundled managed browser plugin with headless launch, persistent OpenClaw profiles, Docker opt-in Chromium baking, and Chrome launch flags, but the issue still asks for product and security decisions around always-available browser binary distribution, first-class proxy/user-agent policy, and reliability boundaries that are not fully implemented.

## What This Changes

Request to bundle a first-class headless Chromium/browser tool that works without user Chrome or third-party APIs and supports JS rendering, sessions, proxies, and user-agent control.

## Best Possible Solution

Keep the shipped managed browser plugin, then split the remaining work into explicit browser distribution/update policy, proxy/user-agent config, profile isolation, and SSRF/credential security follow-ups.

## Reproduction Assessment

Not applicable: this is a feature/product request rather than a broken current contract. Source and docs inspection show the managed browser baseline exists, while always-bundled Chromium and first-class proxy/user-agent policy remain partial or absent.

## Solution Assessment

Yes, keeping this open for maintainer scoping is the best current path. The remaining work is not a narrow fix because it changes browser binary distribution, proxy routing semantics, user-agent policy, credential handling, and SSRF boundaries.
