## Summary

Keep open. Current main already ships a bundled browser plugin with managed headless launch, persistent profiles, Playwright-backed actions, Docker opt-in Chromium baking, and Chrome launch flag escape hatches, but this issue still asks for unresolved product/security scope around always-available browser binaries plus first-class proxy/user-agent policy across container and server deployments.

## What This Changes

The issue requests a first-class built-in headless Chromium browser for JS-rendered, login-required, and containerized web access without relying on the user's Chrome or third-party APIs.

## Best Possible Solution

Keep the shipped managed browser plugin as the baseline, then split the remaining work into explicit browser distribution/update policy, proxy/user-agent config, profile isolation, and SSRF/credential security follow-ups.
