Keep open. ## What This Changes

Request to make a built-in headless Chromium/browser capability reliable for JS-rendered and login-required web access without depending on a user's Chrome or third-party APIs.

## Best Possible Solution

Keep the shipped managed and sandbox browser baseline, use the narrower open issues for proxy/isolation and browser/search evaluation, and split always-bundled Chromium plus proxy/user-agent/security contracts into staged maintainer-approved design work.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high-confidence current behavior: managed/sandbox browser support and global extraArgs exist, while always-bundled Chromium and first-class per-profile proxy/agent binding are not complete.

## Solution Assessment

No, not as one unconditional built-in-browser patch. The maintainable path is a staged product and security design for distribution, proxy/user-agent config, credentials, SSRF policy, profile authority, and session boundaries.
