## Summary

Keep open. Current main and the latest release already provide a bundled browser plugin with an isolated OpenClaw-managed browser profile, headless launch support, remote CDP, and a documented `browser.extraArgs` proxy-flag workaround, but the central request still includes browser binary distribution plus first-class proxy, user-agent, credential, and session-boundary policy that needs maintainer product/security scoping.

## What This Changes

The issue requests a bundled first-class headless Chromium/browser capability for JavaScript-rendered, login-required, proxy-aware web access without relying on the user's Chrome, third-party APIs, or fragile relay paths.

## Best Possible Solution

Keep the shipped managed browser baseline, use the narrower open issues for proxy/isolation/subagent ownership and browser/search evaluation, and split always-bundled Chromium plus first-class proxy/user-agent/security contracts into staged maintainer-approved design work.

## Reproduction Assessment

Yes for the missing feature surface, not for a strict bug: source/docs show the current managed browser support and also show that always-bundled Chromium, first-class per-profile proxy/user-agent policy, and agent-bound session ownership are not implemented.

## Solution Assessment

No as one immediate built-in-browser patch; the maintainable path is a staged product/security design for distribution, proxy/user-agent config, credentials, SSRF policy, profile authority, and session boundaries.
