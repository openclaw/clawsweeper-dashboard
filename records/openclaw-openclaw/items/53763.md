## Summary

Keep open: current main and v2026.5.20 already have a managed Playwright-backed browser plugin with headless/profile support and optional Docker Chromium install, but always-bundled Chromium, first-class proxy/user-agent controls, and session/security boundaries remain product and security design work.

## What This Changes

The issue requests a built-in headless Chromium browser tool with JS execution, persistent sessions, proxy support, user-agent controls, and no dependency on a user-run Chrome or third-party web APIs.

## Best Possible Solution

Keep the shipped managed browser plugin baseline, let https://github.com/openclaw/openclaw/issues/37487 and https://github.com/openclaw/openclaw/issues/60035 own the narrower proxy/isolation slices, and split always-bundled Chromium plus browser egress/session contracts into maintainer-approved design work.

No, not as one unconditional built-in-browser patch. The maintainable path is staged product and security design for distribution, proxy/user-agent config, credentials, SSRF policy, and profile/session boundaries.
