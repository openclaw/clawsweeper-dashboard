## Summary

Keep open. Current main has a default bundled browser plugin with managed headless launches, isolated profiles, Docker/browser install documentation, remote CDP options, and the `browser.extraArgs` proxy/user-agent escape hatch, but the requested always-available Chromium distribution and first-class browser proxy/session/security policy still require maintainer product and security decisions.

## What This Changes

The issue requests a first-class bundled headless Chromium/browser tool with JS execution, persisted sessions, proxy/user-agent support, and no dependency on a user's local Chrome or third-party APIs.

## Best Possible Solution

Keep the shipped browser plugin baseline, then split the remaining work into explicit Chromium distribution policy, SecretRef-safe proxy/user-agent config, profile isolation, SSRF/egress boundaries, and Docker proof.
