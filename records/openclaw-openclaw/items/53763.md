## Summary

Keep open. Current main and v2026.6.8 already ship a substantial managed browser plugin and sandbox-browser option, but they do not provide the requested zero-external-dependency bundled browser-backed web-access layer or settle extraction, proxy, login-state, and security policy.

## What This Changes

The issue requests a built-in bundled headless Chromium/web-access layer so agents can access JS-rendered, login-required, and server/headless web pages without relying on user Chrome, Tavily, or external browser infrastructure.

## Best Possible Solution

Keep the current managed browser as the baseline, then split the remaining work into maintainer-approved browser runtime packaging, extraction fallback behavior, proxy/session isolation, and security boundary design.

## Reproduction Assessment

Yes for the missing capability at source level: current main exposes managed browser/web_fetch surfaces, but web_fetch still cannot execute JavaScript and the browser layer is not a universal bundled Chromium runtime independent of host or configured remote browser availability. I did not run live site benchmarks against zsxq.com, Xiaohongshu, or WeChat articles.

## Solution Assessment

No as a one-shot implementation request. The maintainable path is a staged maintainer decision around packaged runtime ownership, extraction fallback behavior, proxy support, login/session persistence, and browser security boundaries.
