## Summary

Keep open. Current main already ships a default browser plugin with Playwright-backed managed profiles, headless launch paths, persisted profile state, extra Chrome args, and an optional Docker Chromium install, but the remaining request still needs product and security decisions around always-bundled Chromium, first-class proxy/user-agent policy, credential handling, browser egress, and session/profile boundaries.

## What This Changes

Add a first-class built-in headless browser with bundled Chromium, persistent sessions, proxy/user-agent controls, and reliable web access without depending on the user's Chrome or third-party web APIs.

## Best Possible Solution

Keep the shipped browser plugin baseline, use https://github.com/openclaw/openclaw/issues/37487 for the per-profile proxy/isolation slice, and split the broader Chromium distribution plus proxy/user-agent/security contract into staged maintainer-approved design work.
