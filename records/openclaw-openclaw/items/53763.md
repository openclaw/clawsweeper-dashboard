## Summary

Keep open. Current main and the latest shipped release already provide a default managed browser plugin with Playwright-backed CDP control, headless launch support, persisted profiles, `extraArgs`, and optional Docker Chromium installation, but the remaining request still needs maintainer product and security decisions around always-bundled Chromium, first-class proxy/user-agent policy, credential handling, browser egress, and profile/session boundaries; the narrower per-profile proxy/isolation slice is still tracked at https://github.com/openclaw/openclaw/issues/37487.

## What This Changes

The issue requests bundling a headless Chromium browser as a first-class OpenClaw web access tool with JS execution, persistent sessions, proxy support, user-agent rotation, and headless/server support.

## Best Possible Solution

Keep the shipped managed browser baseline, use https://github.com/openclaw/openclaw/issues/37487 for the per-profile proxy/isolation slice, and split always-bundled Chromium plus proxy/user-agent/security contracts into staged maintainer-approved design work.
