## Summary

Keep open: current main still has the install-doc messaging gap, and the linked docs PR that addressed it was closed without merging. The issue is narrow and still actionable as a docs cleanup.

## What This Changes

The issue asks OpenClaw to clarify why `install.sh` and the general docs steer users toward Node 24 while local-prefix `install-cli.sh` manages Node 22.22.0 by default.

## Best Possible Solution

Document the local-prefix managed Node default prominently, explain that it may differ from the shell/global Node 24 recommendation, and warn users to rebuild OpenClaw-adjacent native tools with the runtime that will load them.

## Reproduction Assessment

Yes for the docs mismatch. Source inspection on current main shows Node 24 as the general recommendation/default and `install-cli.sh` still using a local managed Node 22.22.0 default without the requested explanation.

## Solution Assessment

Yes, a docs-only clarification is the narrowest maintainable solution. Changing the managed Node default should remain a separate maintainer product and release decision.
