## What This Changes

Clarify that local-prefix `install-cli.sh` uses a managed Node 22.22.0 runtime that can differ from the Node 24 recommendation and shell/global runtime.

## Best Possible Solution

Land or revise the open docs/script-copy PR that documents the managed-runtime distinction and installer output, leaving any default runtime change to a separate release decision.

Yes. Source inspection on current main shows `install.sh` defaulting to Node 24, `docs/install/node.md` recommending Node 24, and `install-cli.sh` defaulting to a managed Node 22.22.0 runtime with a reuse path that currently has no human clarification.

## Solution Assessment
