## Summary

Keep open: current main still shows the Node 24 recommendation alongside the local-prefix installer’s managed Node 22.22.0 default, and https://github.com/openclaw/openclaw/pull/81278 is an open closing PR for this exact clarification.

## What This Changes

Clarify that local-prefix `install-cli.sh` uses a managed Node 22.22.0 runtime that can differ from the main installer’s recommended/default Node 24 path.

## Best Possible Solution

Yes. Source inspection on current main shows the main installer and Node docs pointing at Node 24 while `install-cli.sh` defaults its managed local-prefix runtime to Node 22.22.0 and can silently reuse an existing managed Node 22+.

## Solution Assessment

Yes. The open PR’s docs/output clarification is the narrow maintainable fix for the reported confusion; changing the managed runtime default should remain a separate release decision.
