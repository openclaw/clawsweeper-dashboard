## Summary

The report is valid on current main and should stay open because the docs/source and the served local-prefix installer currently communicate different Node defaults, while choosing whether to document the split or change the local-prefix default is a maintainer policy call.

## What This Changes

Clarify why the main installer defaults/recommends Node 24 while the local-prefix CLI installer defaults to managed Node 22.22.0 and reuses existing managed Node 22+ runtimes.

## Best Possible Solution

Keep the supported Node 22/Node 24 contract explicit by documenting the local-prefix managed runtime separately and, if the Node 22 pin is intentional, add installer output that tells users which Node will run OpenClaw and how to opt into Node 24.

## Reproduction Assessment

Yes, from source inspection: current main docs/source and the served `install-cli.sh` show the mismatch and managed-Node reuse behavior. No live install was executed because this read-only review did not need to mutate a local prefix.

## Solution Assessment

Unclear for implementation choice: documentation plus installer messaging is the narrowest low-risk path, while changing the local-prefix default to Node 24 needs maintainer policy approval.
