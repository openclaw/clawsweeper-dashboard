## Summary

## What This Changes

The issue asks OpenClaw to clarify or align Node defaults between `install.sh` and local-prefix `install-cli.sh` so users do not confuse shell/global Node with the managed runtime used by OpenClaw.

## Best Possible Solution

Refresh and review https://github.com/openclaw/openclaw/pull/81278, then close this issue only after the chosen docs/install-output clarification or default-alignment policy lands.

## Reproduction Assessment

Yes. Current source reproduces the reported mismatch: `install.sh` defaults to Node 24, docs recommend Node 24, and local-prefix `install-cli.sh` still defaults to managed Node 22.22.0 without the requested explanatory warning.

## Solution Assessment
