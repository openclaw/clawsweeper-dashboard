## Summary

Keep open: current main still has the install-doc mismatch between the general Node 24 recommendation and the local-prefix managed Node 22.22.0 runtime. The closed follow-up at https://github.com/openclaw/openclaw/pull/79586 proposed the right shape, but it was not merged, so the repository still needs the clarification.

## What This Changes

Clarify that `install-cli.sh` local-prefix installs use a managed Node 22.22.0 runtime by default even though general installs recommend Node 24, including native-module implications and the Node 24 opt-in path.

## Best Possible Solution

Add a prominent local-prefix installer warning that explains the managed Node 22.22.0 default, why it can differ from the recommended Node 24 path, how to opt into Node 24, and why native add-ons should be built with the runtime that loads them.

## Reproduction Assessment

Yes. Source inspection on current main shows the Node 24 recommendation, the local-prefix 22.22.0 default, and the existing managed-node skip behavior; I did not run a local-prefix install because the mismatch is directly visible in source and docs.

## Solution Assessment

Yes. The narrow maintainable fix is documentation and possibly human-facing installer copy, while leaving Node defaults unchanged unless maintainers explicitly choose a separate release-level default change.
