## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Require current-source delivery evidence:** `src/agents/embedded-agent-runner/run.ts:3007`
  - body: This guard treats any successful message-tool delivery as proof that the current source conversation was answered. `hasMessagingToolDeliveryEvidence` also returns true for explicit/cross-session sends via `messagingToolSentTargets`, so a model can send elsewhere, write the current-source answer only in private final text, and this recovery path will skip both the retry and the visible error. Gate this on current-source/default-route evidence such as `messagingToolSourceReplyPayloads` or equivalent non-dry-run source reply proof.
  - confidence: 0.91
- **[P2] Preserve planning-only retries first:** `src/agents/embedded-agent-runner/run.ts:3002`
  - body: The new message-tool-only recovery block runs before the existing planning-only retry calculation. A replay-safe plan-only turn like "I'll inspect the files now" in `message_tool_only` mode now matches `hasPrivateFinalReplyText` and gets the `message(action=send)` retry instead of the established act-now planning retry, which can make the agent send a plan rather than take the promised tool action.
  - confidence: 0.86
- **[P2] Keep NO_REPLY silent:** `src/agents/embedded-agent-runner/run.ts:241-245`
  - body: `SILENT_REPLY_TOKEN` is `NO_REPLY`, but `hasPrivateFinalReplyText` currently treats every nonempty assistant/final text as visible private-final content. A deliberate silent turn in `message_tool_only` mode would now retry once and then surface an error, so the classifier should exclude silent reply tokens before triggering recovery.
  - confidence: 0.88
