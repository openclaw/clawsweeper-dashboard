## What This Changes

The issue reports that `openclaw memory index --force` can write a memory index identity that later disables runtime `memory_search` for configured embedding providers, with an additional scopeHash mismatch from multimodal defaults.

## Best Possible Solution

Keep this issue open as the canonical memory index identity bug until the providerKey fix lands and the scopeHash/default handling is either fixed in the same area or moved to a linked narrower follow-up.

## Reproduction Assessment

## Solution Assessment
