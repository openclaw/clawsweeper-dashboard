## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Wait for `flushed` before committing flushed text:** `extensions/gradium/realtime-transcription-provider.ts:343-346`
  - body: After a semantic `step`, this sends `flush` but commits as soon as `end_text` arrives. Gradium documents that pending `text` results are returned before the later `flushed` confirmation, so Voice Call can dispatch an incomplete user turn and then treat remaining text as a separate turn. Keep accumulating until `flushed` or stream end. ([docs.gradium.ai](https://docs.gradium.ai/guides/speech-to-text?utm_source=openai))
  - confidence: 0.86
- **[P2] Send `json_config` as a setup object:** `extensions/gradium/realtime-transcription-provider.ts:223-225`
  - body: The new docs expose `language`, `temp`, `paddingBonus`, and `delayInFrames`, but this stringifies `json_config` before sending the setup frame. Gradium's setup examples put `json_config` on the setup message as an object, so these options can be ignored or rejected instead of applied. ([gradium.ai](https://gradium.ai/content/how-to-use-json-config-gradium-tts-stt))
  - confidence: 0.82
- **[P3] Accept Gradium's documented format aliases:** `extensions/gradium/realtime-transcription-provider.ts:76-82`
  - body: `VALID_INPUT_FORMATS` rejects documented inputs such as `pcm_8000` and the `mulaw_8000` alias for `ulaw_8000`, so a config copied from Gradium's STT docs fails locally even though Gradium accepts it. Add the documented aliases or make the OpenClaw docs explicitly state the narrower supported subset. ([docs.gradium.ai](https://docs.gradium.ai/guides/speech-to-text?utm_source=openai))
  - confidence: 0.77
