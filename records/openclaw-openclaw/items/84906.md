- **[P1] Require explicit STT opt-in before reusing the TTS key:** `extensions/gradium/realtime-transcription-provider.ts:440-441`
  - body: Because Voice Call auto-selects the first configured realtime STT provider when `streaming.provider` is unset, this env fallback makes a TTS-only `GRADIUM_API_KEY` enough to activate inbound Gradium transcription.   - confidence: 0.86
- **[P2] Send `json_config` as a WebSocket config object:** `extensions/gradium/realtime-transcription-provider.ts:244`
  - body: Gradium's realtime WebSocket setup docs show `json_config` as the nested config object, but this sends a JSON string. That means configured language, temperature, padding, and delay options may be ignored or rejected by the ASR endpoint; send the object and update the payload test accordingly.
  - confidence: 0.83
- **[P3] Align the `end_text` docs with the flush wait path:** `docs/providers/gradium.md:194`
  - body: The provider now returns on `end_text` while a flush is pending and commits on `flushed`, `done`, or `end_of_stream`, but the docs still say `end_text` commits after a semantic flush request. Update the table so operators debug the actual finalization lifecycle.
  - confidence: 0.9
