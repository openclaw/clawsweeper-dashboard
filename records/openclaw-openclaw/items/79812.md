## Summary

Keep open: current main still maps normal no-useful-memory Active Memory runs to `status=empty`, and multiple open PRs reference this issue with closing syntax, so it should close only after one canonical fix lands.

## What This Changes

Active Memory should distinguish successful no-relevant-memory recall from failure, unavailable, and truly empty states instead of surfacing normal no-useful-memory results as `status=empty`.

## Best Possible Solution

Land one canonical Active Memory fix that updates the status classification, cache eligibility, regression tests, and any user-facing docs/changelog needed, then close this issue as implemented.

## Reproduction Assessment

Yes, from source inspection: make the embedded recall subagent return `NONE` and inspect the persisted Active Memory status line; current tests already assert `status=empty` for that path and for zero-hit search.

## Solution Assessment

Yes, separating normal no-relevant-memory recall from error/empty states in the Active Memory plugin is the narrow maintainable fix; the remaining choice is the exact public status label in the open PR that lands.
