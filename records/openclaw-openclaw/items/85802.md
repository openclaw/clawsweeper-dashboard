## Summary

Keep open: this is a valid fix direction for a source-reproducible OPENCLAW_HOME bug, but the PR has a protected maintainer label and the diff changes installer defaults without enough upgrade handling or real behavior proof for existing setups.

## What This Changes

The PR updates the update CLI, install scripts, docs, changelog, and regression tests so OPENCLAW_HOME drives dev checkout, installer prefix, onboarding workspace, and config-check defaults when explicit path overrides are absent.

## Best Possible Solution

Honor OPENCLAW_HOME for internal state, git, and onboarding defaults, but preserve or migrate existing HOME-created state and avoid moving the CLI install prefix without explicit upgrade handling and live proof.

## Reproduction Assessment

Yes, at source level. Current main still derives update-cli and installer defaults from os.homedir/HOME when OPENCLAW_HOME is set and the more specific overrides are absent.

## Solution Assessment

No, not yet. The shared TypeScript update direction is good, but the installer changes need upgrade-safe handling for existing HOME-created config/state and a maintainer decision on whether OPENCLAW_HOME should relocate the CLI prefix.
