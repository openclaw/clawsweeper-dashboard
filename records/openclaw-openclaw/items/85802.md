## Summary

Keep open. The PR targets a real OPENCLAW_HOME bug, but it is not merge-ready because it relocates install-cli’s default prefix without an upgrade path, misses HOME-created config detection for existing affected users, and has only test/check proof rather than a live installer/update/onboarding run.

## What This Changes

The PR updates update-cli path resolution, Unix installer scripts, docs, changelog, and regression tests so OPENCLAW_HOME affects default git checkout and onboarding paths, plus the install-cli prefix, when explicit overrides are absent.

## Best Possible Solution

Honor OPENCLAW_HOME for internal state, git checkout, and onboarding defaults while keeping OPENCLAW_PREFIX as the install-location control, and add tested upgrade detection or migration for HOME-created state before onboarding.

## Reproduction Assessment

Yes, at source level. Current main still derives update-cli and installer defaults from os.homedir/HOME when OPENCLAW_HOME is set and the more specific overrides are absent.

## Solution Assessment

No, not in this form. The TypeScript git-dir fix follows the existing home-dir helper, but the shell changes need upgrade-safe config detection and should not move the install-cli prefix without an explicit maintainer decision.
