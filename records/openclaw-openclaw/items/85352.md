## Summary

Keep open: current main and the reported v2026.5.20 release still have the macOS dashboard reload plus disconnected Control UI render path that can flash the credentials gate before the gateway hello arrives.

## What This Changes

The issue asks for an already-authenticated macOS menu bar dashboard open to show the Control UI directly instead of briefly rendering the credentials gate.

## Best Possible Solution

Preserve authenticated macOS dashboard content across reopen, or render an authenticated connecting shell, while reserving the full credentials gate for missing credentials and explicit auth failures.

## Reproduction Assessment

Yes, at source level. Current main reloads the macOS dashboard web UI on show, starts the Control UI disconnected, and renders the credentials gate for every disconnected state before hello.

## Solution Assessment

No patch is proposed here. The best fix should extend the shared reconnect-state work with the macOS reload boundary or avoid reloading the same authenticated dashboard window, without hiding real auth failures.
