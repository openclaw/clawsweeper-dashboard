## Label Justifications

- `P2`: This is a visible authenticated-session regression in a normal macOS dashboard workflow, with limited blast radius and no crash or data loss.
- `impact:session-state`: The issue is about losing the authenticated UI state during a transient macOS dashboard open/startup path and briefly rendering the wrong session state.

## Summary

Keep open: current main and the latest release still have the source-level path that can render the credentials gate during the macOS dashboard reload before the gateway hello arrives, and the related open web UI PR does not yet cover this macOS every-open reload path.

## What This Changes

The issue asks OpenClaw to stop the macOS menu bar dashboard from briefly showing the credentials gate before the Control UI for already-authenticated users.

## Best Possible Solution

Preserve the authenticated dashboard across macOS reopens, or show an authenticated connecting shell during native reload, while reserving the full credentials gate for missing credentials or explicit auth failure.

## Reproduction Assessment

Yes, at source level: current main reloads the macOS dashboard on show, starts the Control UI with `connected = false`, and renders the credentials gate for any disconnected state. I did not run the live macOS app to capture the visual flicker timing.

## Solution Assessment

No patch is proposed here, and the related open web UI PR is not sufficient until it covers the macOS dashboard reload path. The maintainable fix should distinguish authenticated reconnect/startup from missing or failed auth without hiding real auth failures.
