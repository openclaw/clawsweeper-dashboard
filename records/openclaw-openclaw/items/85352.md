- `impact:session-state`: The issue is about briefly losing the authenticated UI state and rendering the wrong session state during macOS dashboard startup.

## Summary

Keep open: current main still has the source-level path that reloads the macOS dashboard and renders the credentials gate before the gateway hello arrives. The related open Control UI PR may help the shared reconnect case, but it does not currently close this macOS every-open reload path.

## What This Changes

The issue requests that the authenticated macOS menu bar dashboard open directly to the Control UI without briefly flashing the credentials/login gate.

## Best Possible Solution

Preserve the authenticated macOS dashboard across reopens, or render an authenticated connecting shell during native reload, while reserving the full credentials gate for missing credentials or explicit auth failure.

## Reproduction Assessment

Yes, at source level: current main reloads the macOS dashboard on show, initializes the Control UI with connected = false, and renders the credentials gate for any disconnected state. I did not run a live macOS app capture for the 200-500ms visual flash.

## Solution Assessment

No patch is proposed here. The maintainable fix should distinguish authenticated reconnect/startup from missing or failed auth, and the existing open web UI PR is not sufficient unless it also covers the macOS fresh-load path.
