## Summary

Keep open. The macOS report is source-reproducible on current main and is not fully covered by the existing webchat reconnect issue because the native dashboard path also reloads the WKWebView on repeated opens, which can show the shared disconnected login gate before native auth reconnects.

## What This Changes

The issue asks OpenClaw to stop flashing the credentials/login gate when an already-authenticated macOS menu bar user opens the Control UI.

## Best Possible Solution

Preserve the existing macOS dashboard WKWebView across repeated status-item opens when the endpoint and credentials have not changed, while landing the shared Control UI reconnect fix for transient WebSocket closes separately or as a coordinated patch.

## Reproduction Assessment

Yes, at source level. The macOS status-item dashboard path reloads the WKWebView through DashboardWindowController.show(url:auth), and the shared Control UI renders the credentials gate while connected is false before the hello handshake completes.

## Solution Assessment

No, closing this as only the webchat duplicate would miss the native reload trigger. The best fix should avoid unnecessary macOS dashboard reloads and then verify the shared reconnect behavior still handles real auth failures correctly.
