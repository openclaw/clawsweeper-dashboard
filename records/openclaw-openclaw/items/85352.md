## Best Possible Solution

## Solution Assessment

Yes directionally: the maintainable fix is to distinguish authenticated reconnect/startup from missing or failed auth. The open related PR is only sufficient if it also covers this native dashboard reload path or is replaced by an equivalent fix that does.
