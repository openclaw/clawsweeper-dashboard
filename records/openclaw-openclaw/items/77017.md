Keep open. The latest head appears to fix the earlier record-before-auth issue and includes useful generated-image proof, but two merge blockers remain: managed image device-token reads bypass the device-token rate limiter before verification, and the new public thumbnail config key is still missing from public docs.

## What This Changes

The branch adds Control UI generated-image frames and actions, managed full/thumbnail media routes and thumbnail config, Codex media timeout/transcript changes, changelog text, and focused tests.

## Best Possible Solution

Land the generated-image UX after the managed-image route uses the same device-token rate-limit semantics as other Control UI read routes and public docs describe imageThumbnailMaxSide.

## Reproduction Assessment

Yes. Source inspection of PR head shows the direct device-token verifier runs before any device-token limiter check, and docs search after pnpm docs:list shows no public imageThumbnailMaxSide entry.

## Solution Assessment

No. The feature direction is reasonable, but the maintainable path keeps device-token verification behind the existing auth-rate-limit contract and documents the new public config surface before merge.
