## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve media captions during transcript replacement:** `src/gateway/server-methods/chat.ts:181-184`
  - body: This helper clears `payload.text` for every media-bearing payload before the replacement content is built. In the agent-run raw `MEDIA:` replacement path, a reply like `Here is the image` plus a media URL can be rewritten as synthetic media text such as `Image reply`, losing the assistant's caption in Control UI/history. Carry the original caption into the replacement while still suppressing duplicate tool-summary text.
  - confidence: 0.9
- **[P2] Restrict MEDIA replacement to the current turn:** `src/gateway/server-methods/chat.ts:1571-1573`
  - body: The replacement helper searches the whole transcript branch in reverse for any assistant message whose `MEDIA:` path matches the current payload. If a path is reused or an older raw media turn remains, this can rewrite an older assistant message and suppress appending the current reply, corrupting chat history. Constrain the match to the latest/current assistant turn or the current run's persisted raw entry.
  - confidence: 0.86
- **[P2] Avoid duplicate transcript updates after rewrites:** `src/gateway/server-methods/chat.ts:2772-2778`
  - body: `rewriteTranscriptEntriesInSessionFile` already emits a transcript update whenever it changes the active branch. Emitting a second message-scoped update here can make clients process two updates for one replacement and duplicate or reorder the visible chat event. Let the rewrite update drive refresh, or change the rewrite helper to emit the precise replacement once.
  - confidence: 0.78
- **[P2] Keep generic Codex tool timeouts bounded:** `extensions/codex/src/app-server/run-attempt.ts:167-169`
  - body: These constants make every dynamic tool without an explicit or configured timeout wait up to 20 minutes, because `resolveDynamicToolCallTimeoutMs` falls back to `CODEX_DYNAMIC_TOOL_TIMEOUT_MS`. Current main already supports per-call/configured image timeouts capped at 10 minutes, so this broad default change can leave ordinary hung tools pinning Codex turns far longer than before. Keep the longer timeout scoped to the image paths that need it.
  - confidence: 0.84
- **[P2] Remove the stale release section from the changelog:** `CHANGELOG.md:1374`
  - body: This inserts a full `## 2026.5.9` release section and many unrelated release notes into the current changelog flow, then existing Unreleased bullets continue below it. That moves current entries under an old release heading and would publish unrelated history with this PR. Keep only the PR's relevant Unreleased entries.
  - confidence: 0.94
