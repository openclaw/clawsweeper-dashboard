## Summary

Keep this PR open: the generated-image UI work is useful, but the current PR head still introduces a managed-image auth bypass and leaves the Open action dependent on unauthenticated navigation.

## What This Changes

The PR adds Control UI generated-image frames/actions, thumbnail routes/config, managed-media preview handling, Codex dynamic-tool timeout changes, and related media transcript tests/changelog.

## Best Possible Solution

Keep the image actions and thumbnail split, but preserve managed-media authentication by fetching full-size managed images through an auth-preserving blob or signed media route.

## Reproduction Assessment

Yes. The auth bypass is source-reproducible from PR head: the route accepts the requester-session header before gateway auth, and that same session key is already present in the URL.

## Solution Assessment

No. The current PR shape is not the best fix because it trades the Open action problem for a server-side auth bypass; the narrower solution is to keep gateway auth first and make UI actions use authenticated fetch/blob or a ticketed URL.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep managed image reads behind gateway auth:** `src/gateway/managed-image-attachments.ts:1034-1041`
  - body: This early requester-session branch runs before authorizeGatewayHttpRequestOrReply. Because requesterOwnsManagedImageSession accepts requesterSessionKey === targetSessionKey and the target session key is already in the URL, an unauthenticated client that knows a managed-image URL can fetch the image by echoing that path value in x-openclaw-requester-session-key. Keep the auth check before this shortcut and add a no-auth regression.
  - confidence: 0.94
- **[P2] Open managed images through an authenticated blob:** `ui/src/ui/chat/grouped-render.ts:1166`
  - body: The Open image button navigates directly to the managed full-image URL with window.open, which cannot include the bearer token or requester-session header expected by the managed-image endpoint. In token-auth Control UI sessions this fails unless the server auth bypass remains, so reuse the authenticated fetch path and open an object URL or another auth-preserving URL.
  - confidence: 0.86
