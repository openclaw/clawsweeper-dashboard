## Summary

Keep open: the feature direction is useful, but the current PR head still weakens managed-image authentication and the Open action still uses a browser navigation path that cannot carry the auth headers the route expects.

## What This Changes

The PR adds Control UI generated-image open/download/copy actions, managed image thumbnail serving and config, Codex media/timeout changes, tests, and changelog entries.

## Best Possible Solution

Keep the image actions and thumbnail split, but keep managed media behind normal Gateway auth and make Open/click use an auth-preserving blob URL or signed media URL.

## Reproduction Assessment

Yes. The auth bypass is source-reproducible from the PR head: a no-bearer request can echo the session key from the URL in x-openclaw-requester-session-key, and the new test expects the route to serve bytes before auth.

## Solution Assessment

No. The maintainable solution is to keep server-side auth first and adjust browser image actions to use authenticated fetch/blob handling or a short-lived ticketed URL for full-size managed images.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Keep managed image reads behind gateway auth:** `src/gateway/managed-image-attachments.ts:1034-1044`
  - body: This requester-session shortcut runs before authorizeGatewayHttpRequestOrReply. Because requesterOwnsManagedImageSession accepts requesterSessionKey === targetSessionKey and that target key is already in the URL, a client that knows a managed-image URL can echo the path value in x-openclaw-requester-session-key and fetch the image without gateway auth. Keep auth before this shortcut and add a regression that a no-auth request with that header is rejected.
  - confidence: 0.94
- **[P2] Open managed images through an authenticated blob:** `ui/src/ui/chat/grouped-render.ts:1166`
  - body: The Open image button passes the managed full-image URL to window.open, but that navigation cannot include the bearer token or requester-session header used by the managed-image endpoint. In token-auth Control UI sessions, Open will still fail once the server route stays authenticated; reuse the authenticated fetch path and open an object URL or another auth-preserving URL instead.
  - confidence: 0.86
