## Summary

Keep this PR open: the feature direction and supplied real-behavior proof are solid, but the latest PR head still has a device-token rate-limit bypass on the managed-image read path and the new public config key is not in the public Gateway config docs.

## What This Changes

The PR adds generated-image open/download/copy actions, managed thumbnail/full-image routing, thumbnail sizing config, Codex media handling adjustments, tests, and changelog entries.

## Best Possible Solution

Land the generated-image UX after the managed-image route uses the same device-token limiter semantics as existing Control UI read routes and the public Gateway config reference documents the new thumbnail setting.

## Reproduction Assessment

Yes. Source inspection of PR head shows device-token verification happens before the device-token limiter, and the public config docs search shows no imageThumbnailMaxSide entry.

## Solution Assessment

No. The feature approach is reasonable, but the mergeable version should reuse the existing device-token auth-rate-limit contract and keep the schema/help/docs config contract aligned.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9
