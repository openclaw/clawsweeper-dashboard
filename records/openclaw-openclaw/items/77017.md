## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve captions when replacing MEDIA replies:** `src/gateway/server-methods/chat.ts:181-183`
  - body: This strips `payload.text` for every media-bearing payload before building the replacement transcript content. A final payload such as `{ text: "Here is the image.", mediaUrl }` can be persisted with synthetic media text instead, so generated image replies lose the assistant's caption.
  - confidence: 0.9
- **[P2] Only replace the current MEDIA reply:** `src/gateway/server-methods/chat.ts:1571-1573`
  - body: The replacement search walks the whole transcript branch backward and picks any matching assistant `MEDIA:` message. If a later turn references the same image path or URL, this can mutate an older assistant message and skip appending the current reply, corrupting the session transcript.
  - confidence: 0.8
- **[P2] Keep Codex dynamic timeouts scoped:** `extensions/codex/src/app-server/run-attempt.ts:167-169`
  - body: Raising the generic dynamic-tool default, max, image, and app-server request timeouts to 20 minutes changes the current contract where generic tools fail closed quickly and only per-call/configured slow image calls get a larger budget. Unrelated stuck tools can now hold Codex turns much longer than intended.
  - confidence: 0.86
- **[P2] Remove the stale release section:** `CHANGELOG.md:1374-1376`
  - body: This inserts the old `2026.5.9` release notes into the current changelog instead of only adding this PR's entry. Leaving that section in the diff would re-publish unrelated release notes and make the changelog chronology incorrect.
  - confidence: 0.95
