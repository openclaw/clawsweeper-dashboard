## Summary

Keep open: the latest head still introduces a pre-auth managed-image read path, and the Open action still depends on direct navigation that cannot carry Gateway auth once that bypass is removed.

## What This Changes

The PR adds Control UI generated-image frames and actions, managed-media thumbnail routes/config, Codex media timeout/normalization changes, tests, and changelog entries.

## Best Possible Solution

Keep the image actions and thumbnail split, but require normal Gateway auth before any managed-image bytes are served and use an auth-preserving blob or signed media URL for Open.

## Reproduction Assessment

Yes. The auth bypass is source-reproducible at PR head 68e4e62: the route can serve bytes when the requester header echoes the session key from the URL before Gateway auth runs, and the new test expects that behavior.

## Solution Assessment

No. The maintainable fix is to keep Gateway HTTP auth first and make full-size Open/Download/Copy actions use authenticated fetch/blob handling or a ticketed URL rather than treating the route session key as a credential.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep managed image reads behind gateway auth:** `src/gateway/managed-image-attachments.ts:1034-1044`
  - body: This new requester-session branch runs before authorizeGatewayHttpRequestOrReply. Since requesterOwnsManagedImageSession accepts a requester key equal to the target session key, and that session key is already in the URL, an unauthenticated client with a managed-image URL can echo the path value in x-openclaw-requester-session-key and receive the image. Keep the auth check before this shortcut and add a regression that no-auth plus that header is rejected.
  - confidence: 0.95
- **[P2] Open managed images through authenticated blobs:** `ui/src/ui/chat/grouped-render.ts:1166`
  - body: The Open button and image click still call openExternalUrlSafe on the managed full-image URL, which uses window.open and cannot include the bearer token or requester-session header. After the server-side auth bypass is removed, token-auth Control UI sessions will fail to open managed images; route Open through the authenticated fetch/blob path or another auth-preserving URL.
  - confidence: 0.87
