## What This Changes

The PR adds generated-image chat open/download/copy actions, thumbnail preview serving, managed media geometry handling, a Control UI thumbnail-size config key, Codex media transcript normalization, and longer Codex dynamic-tool timeouts.

## Best Possible Solution

Repair the auth ordering and config baseline first, then have maintainers decide whether to keep the bundled Codex/timeout changes here or split them behind the related inline-media PR.

## Reproduction Assessment

Yes for the blocking PR regression: source inspection and the added PR test show same-session requester image reads returning before gateway auth, while current main rejects denied auth before serving bytes. The central generated-image action request is a feature, not a bug reproduction case.

## Solution Assessment

No. The maintainable path is to keep gateway auth and chat.history scope checks before requester-session ownership, update the config baseline for the new public key, and reconcile the overlapping Codex inline-media work with #75727.
