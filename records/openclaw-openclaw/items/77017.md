## Summary

This PR should stay open: the image-action direction is useful, but current head still introduces an unauthenticated managed-image read path and the Open action still uses a navigation path that cannot carry Gateway auth headers.

## What This Changes

The PR adds generated-image chat frames/actions, managed thumbnail/full-image routes and config, Codex media timeout/raw MEDIA handling changes, tests, and changelog entries.

## Best Possible Solution

Keep the image actions and thumbnail/full-image split, but preserve managed-media authentication and reconcile the overlapping Codex MEDIA work before final merge.

## Reproduction Assessment

Yes. Source inspection of PR head 2104863 shows a requester-session header can be echoed from the URL path and served before Gateway auth, and the PR adds a test that expects this unauthenticated path to return 200.

## Solution Assessment

No. The maintainable approach is to keep server-side Gateway auth before requester ownership checks and make Open use an authenticated blob or signed media URL rather than direct navigation to the protected route.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep managed image reads behind gateway auth:** `src/gateway/managed-image-attachments.ts:1034-1041`
  - body: This early requester-session branch runs before `authorizeGatewayHttpRequestOrReply`. Because `requesterOwnsManagedImageSession` accepts `requesterSessionKey === targetSessionKey` and the target session key is already in the URL path, an unauthenticated client that knows a managed-image URL can fetch it by echoing the path value in `x-openclaw-requester-session-key`. Keep the auth check first and add a regression that this no-auth request is rejected.
  - confidence: 0.95
- **[P2] Open managed images through an authenticated blob:** `ui/src/ui/chat/grouped-render.ts:1166`
  - body: The Open button still passes the full managed-image URL to `openExternalUrlSafe`, which uses `window.open` and cannot attach the bearer token or requester-session header. In token-auth Control UI sessions, opening a full managed image will fail once the server route stays authenticated; fetch the image with auth and open an object URL or another auth-preserving URL instead.
  - confidence: 0.88
