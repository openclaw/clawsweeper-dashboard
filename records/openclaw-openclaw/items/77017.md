Keep this PR open: the generated-image UI work is useful, but the current head still introduces a managed-media auth bypass and misses the required config-doc baseline for the new public setting.

## What This Changes

The PR adds generated-image chat actions, thumbnail preview routing/configuration, managed image frame handling, Codex media normalization, and longer Codex dynamic-tool timeouts.

## Best Possible Solution

Repair the auth ordering and config baseline first, then let maintainers decide whether the generated-image UX, Codex timeout, and overlapping #75727 media-normalization changes should land together or be split.

## Reproduction Assessment

Yes for the blocking PR regression: source inspection and the added PR test show same-session requester media reads returning before gateway auth. The central generated-image action request is a feature, so current-main bug reproduction is not applicable to that part.

## Solution Assessment

No. The maintainable path is to keep gateway auth and chat.history scope checks before requester-session ownership, commit the generated config-doc baseline, and reconcile the overlapping Codex inline-media changes with #75727.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Require gateway auth before requester media reads:** `src/gateway/managed-image-attachments.ts:1034-1043`
  - body: The new early requester-session path serves the managed image before `authorizeGatewayHttpRequestOrReply` and the `chat.history` scope check run. Because the session key is already embedded in the media URL, a caller with the URL can echo it in `x-openclaw-requester-session-key` and fetch bytes without the configured gateway credentials; keep ownership as a post-auth constraint or use a signed media capability.
  - confidence: 0.93
- **[P2] Commit the config docs baseline hash:** `src/config/zod-schema.ts:801`
  - body: This adds `gateway.controlUi.imageThumbnailMaxSide` as a public config key, but the PR does not include `docs/.generated/config-baseline.sha256`. The repo's `config:docs:check` gate tracks config-surface changes through that generated baseline, so this will drift until the baseline is regenerated and committed.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The PR serves managed generated-image bytes on a requester-session header before gateway authentication and chat.history scope checks.

Concerns:

- **[high] Managed image fetch can bypass gateway auth:** `src/gateway/managed-image-attachments.ts:1034`
  - body: The new requester-session branch returns image bytes before the existing gateway auth path runs. Since the route URL already contains the session key, URL possession is enough to construct the header and fetch media without the configured credentials or operator scope checks.
  - confidence: 0.93
