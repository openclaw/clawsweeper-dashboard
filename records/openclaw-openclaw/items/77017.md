## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Restrict MEDIA replacement to the current turn:** `src/gateway/server-methods/chat.ts:1571-1573`
  - body: `replaceLatestAssistantMediaDirectiveTranscriptMessage` walks backward through the whole transcript and replaces the first assistant entry whose MEDIA path matches the outgoing payload. If the current response reuses a generated image path from an earlier assistant turn, this can mutate that older message and leave the current raw MEDIA turn untouched, corrupting session history. Limit the replacement to the latest/current assistant entry or pass the expected message id.
  - confidence: 0.86
- **[P2] Avoid duplicate transcript updates after rewrites:** `src/gateway/server-methods/chat.ts:2772-2778`
  - body: `rewriteTranscriptEntriesInSessionFile` already emits a transcript update when it persists a changed active branch, so emitting another message-specific update here can deliver duplicate history refreshes for the same rewrite. Remove the extra emit or make the rewrite helper provide the exact event shape needed by subscribers.
  - confidence: 0.82
- **[P2] Keep generic dynamic tools on the short timeout:** `extensions/codex/src/app-server/run-attempt.ts:167-169`
  - body: The PR raises the default timeout for every Codex dynamic tool from 30 seconds to 20 minutes. That solves long image generation by making unrelated dynamic tools hang much longer before failing, which can stall normal turns. Keep the longer default on the image-specific paths or require an explicit configured/request timeout for long-running tools.
  - confidence: 0.8
- **[P2] Remove the stale release block from the changelog:** `CHANGELOG.md:1374`
  - body: This branch inserts the old `2026.5.9` release section into the current changelog instead of a focused entry for this PR. Merging that would duplicate/misplace historical release notes and bury the actual generated-image change; drop the stale block and let maintainers handle the final landing entry.
  - confidence: 0.9
