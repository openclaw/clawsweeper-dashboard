## Review Findings

Overall confidence: 0.89

Full review comments:

- **[P1] Keep gateway auth before requester checks:** `src/gateway/managed-image-attachments.ts:1040-1043`
  - body: The new early requester-session branch can serve the managed image before `authorizeGatewayHttpRequestOrReply` runs. The requester session key is embedded in the media URL path, so a caller with the URL can repeat it in `x-openclaw-requester-session-key` and fetch the image without the gateway token or scopes; keep the normal auth/scope gate before serving bytes or use a signed media ticket.
  - confidence: 0.91
- **[P1] Preserve signed media tickets for local images:** `ui/src/ui/chat/grouped-render.ts:730-735`
  - body: Current main probes the assistant-media meta endpoint and renders local files with a scoped `mediaTicket`; this PR head directly builds image and thumbnail URLs from `opts.authToken` as a query `token`. That exposes the gateway token in DOM/query strings and drops the current unavailable-file gating, so the rebase should keep `resolveAssistantAttachmentAvailability` and use `availability.mediaTicket` for both full and thumbnail URLs.
  - confidence: 0.86
