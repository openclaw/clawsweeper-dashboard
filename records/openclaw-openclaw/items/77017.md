## Summary

Keep this PR open: the generated-image action work is useful, but the current diff introduces a blocking managed-image auth regression and the Open action still cannot authenticate token-mode managed image URLs.

## What This Changes

The PR adds Control UI generated-image open/download/copy actions, thumbnail routes/config, managed media preview handling, Codex media timeout changes, and related tests/changelog entries.

## Best Possible Solution

Keep the image actions and thumbnail split, but preserve managed-media authentication by using an auth-preserving fetch/blob or signed media route instead of a requester-header bypass.

## Reproduction Assessment

Yes. The auth bypass is source-reproducible from the PR diff: a request can echo the session key from the URL in `x-openclaw-requester-session-key` and reach serving before gateway auth.

## Solution Assessment

No. The maintainable fix is to keep server-side gateway auth first and make UI actions use authenticated fetch/blob or a ticketed URL for full-size managed images.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep managed image reads behind gateway auth:** `src/gateway/managed-image-attachments.ts:1034-1041`
  - body: This new early requester-session branch runs before `authorizeGatewayHttpRequestOrReply`. Because `requesterOwnsManagedImageSession` returns true when the requester header equals the target session key, and that session key is already in the URL, an unauthenticated client that knows a managed-image URL can fetch the image by echoing the path value in `x-openclaw-requester-session-key`. Keep the auth check before this ownership shortcut and add a regression that a no-auth request with that header is rejected.
  - confidence: 0.94
- **[P2] Open managed images through an authenticated blob:** `ui/src/ui/chat/grouped-render.ts:1166`
  - body: The Open image button navigates directly to the managed full-image URL with `window.open`, but that navigation cannot include the bearer token or requester-session header used by the managed-image endpoint. In token-auth Control UI sessions, Open will still hit the route unauthenticated and fail; reuse the authenticated fetch path and open an object URL or another auth-preserving URL instead.
  - confidence: 0.86
