## Label Justifications

## Summary

Keep open. Current main and the latest release still filter Telegram reply/quote supplemental context by the reply-target sender without a bot-self exception, and the linked closing PR remains open rather than merged.

## What This Changes

The issue asks for Telegram replies to bot-sent messages to preserve the quoted bot message context under group allowlist visibility.

## Best Possible Solution

Refresh and land https://github.com/openclaw/openclaw/pull/82079, or a narrow replacement, with proof that bot-self reply targets survive allowlist visibility while non-bot redaction remains intact.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: in a Telegram group with contextVisibility allowlist and the user allowed but the bot account not in effectiveGroupAllow, a reply to a bot-sent message can have the bot reply target filtered before the agent turn is built; issue and PR comments also report live shipped repros.

## Solution Assessment

Yes. The narrow maintainable fix is to treat Telegram bot/self reply targets as allowed supplemental quote context before the allowlist gate while leaving ordinary non-bot and forwarded-origin filtering unchanged; the broader durable-context work remains separate at https://github.com/openclaw/openclaw/issues/88032.
