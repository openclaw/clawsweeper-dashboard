## Summary

Keep open. Current main and the latest release still filter Telegram bot/self reply targets through the group allowlist, and the linked fixing PR is open rather than merged.

## What This Changes

The issue reports that Telegram replies to bot-sent messages can lose agent-visible reply context when group allowlist visibility filters the bot/self reply target.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/82079, then close this issue after the bot/self allowlist fix is on main; keep broader reply-context architecture in the related follow-up items.

## Reproduction Assessment

Yes. Source inspection shows a bot-authored `reply_to_message` whose bot sender is not in `effectiveGroupAllow` gets `senderAllowed=false`, and `contextVisibility: "allowlist"` then blocks the quote context; linked comments add live before-fix evidence, but I did not run a live Telegram scenario.

## Solution Assessment

Yes for the narrow bug. The best fix is at the Telegram supplemental-context gate, aligning bot/self reply-target visibility with the existing reply-to-bot implicit mention behavior while preserving non-bot allowlist redaction.
