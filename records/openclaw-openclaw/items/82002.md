## Summary

Keep open. Current main and the latest stable release still gate Telegram supplemental reply context only through the group allowlist, and the open paired fix PR already references this issue with closing syntax.

## What This Changes

The issue reports that Telegram group replies to OpenClaw bot messages under allowlist visibility can drop the quoted bot message before the agent turn.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/82079 or a replacement that adds the narrow bot/self sender exception before the allowlist gate, with live Telegram proof and a control that non-bot redaction remains intact.

## Reproduction Assessment

Yes for source reproduction: configure Telegram contextVisibility to allowlist with a group allowlist that admits the user but not the bot, then reply to a bot-sent message. Current source checks the bot sender only against effectiveGroupAllow and the visibility helper blocks false senderAllowed values.

## Solution Assessment

Yes, the narrow maintainable fix is to treat Telegram bot/self reply targets as allowed supplemental quote context before the allowlist gate. The merged cache work helps related reply-chain hydration, but the open paired PR is still the right place for this specific allowlist fix.
