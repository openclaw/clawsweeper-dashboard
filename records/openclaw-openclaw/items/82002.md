## Summary

Keep open. Current main still applies the Telegram group allowlist to reply targets without a bot/self exception, and the linked open fix PR still owns the narrow remaining work for this issue.

## What This Changes

The issue asks OpenClaw to preserve Telegram bot-sent reply/quote context in allowlisted groups when a user replies to the bot message.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82079, or an equivalent replacement, after live Telegram proof shows bot-sent quote context is preserved while non-bot redaction remains intact.

## Reproduction Assessment

Yes for source reproduction: configure Telegram `contextVisibility: "allowlist"` with a group allowlist that includes the user but not the bot, then reply to a bot-sent message. Current main can drop that reply target because the bot sender is checked only against `effectiveGroupAllow`; I did not run a live Telegram session.

## Solution Assessment

Yes. The narrow maintainable fix is to treat Telegram bot/self reply targets as allowed supplemental quote context before the allowlist visibility gate, without relaxing forwarded-origin redaction or general group authorization.
