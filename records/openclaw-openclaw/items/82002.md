## Summary

Keep open. Current main still filters Telegram reply targets by group allowlist without a bot/self exception, and an open fix PR already references the issue with closing syntax, so the issue should remain open until that PR lands or is replaced.

## What This Changes

Telegram group replies to bot-sent messages should preserve reply/quote context under allowlist visibility.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82079, or a replacement with the same narrow bot/self reply-target fix, once real Telegram proof shows the quoted bot body is preserved and non-bot redaction remains intact.

## Reproduction Assessment

Yes for source reproduction: configure Telegram `contextVisibility: "allowlist"` with a group allowlist that admits the user but not the bot, then reply to a bot-sent message. Current main can drop the reply target because the quote sender is checked only against `effectiveGroupAllow`; I did not run a live Telegram session, so this is source-reproducible rather than fully reproduced.

## Solution Assessment

Yes, the narrow maintainable solution is to treat Telegram bot/self reply targets as allowed supplemental quote context before the allowlist gate. The fix should not relax forwarded-origin redaction or group trigger authorization, and the existing open PR is the right place to validate that direction.
