## Summary

This should stay open. The report is source-reproducible on current main: Telegram already treats replies to the bot as implicit mentions, but the supplemental-context filter can still drop a bot-sent reply target when `contextVisibility: "allowlist"` checks the bot sender against the group allowlist.

## What This Changes

Preserve Telegram bot/self reply target context when a group allowlist is active so agent turns receive the quoted bot message.

## Best Possible Solution

Update the Telegram inbound context builder to treat bot/self reply targets as allowed quote context under allowlist visibility, add focused regression coverage, and evaluate outbound cache persistence separately only if live proof shows it is needed.

## Reproduction Assessment

Yes for source reproduction: configure Telegram `contextVisibility: "allowlist"` with a group allowlist that admits the user but not the bot, then reply to a bot-sent message. Current source checks the reply target sender against the allowlist and can filter the quote before the agent turn is built.

## Solution Assessment

Yes, the narrow maintainable fix is to mark Telegram bot/self reply targets as allowed for supplemental quote context before allowlist filtering. The fix should not relax forwarded-origin redaction or group trigger authorization.
