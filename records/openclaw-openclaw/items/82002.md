## Summary

Keep open. Current main and the latest release still apply Telegram group supplemental-context visibility to the reply-target sender without a bot-self exception, while the direct closing PR remains open and conflicting rather than merged proof.

## What This Changes

Issue requests preserving Telegram reply/quote context when group allowlist visibility would otherwise filter a bot-self reply target.

## Best Possible Solution

Refresh and land https://github.com/openclaw/openclaw/pull/82079, or a narrow replacement, with coverage and live Telegram proof that bot-self reply targets survive allowlist visibility while non-bot sender redaction remains intact.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: use Telegram group allowlist visibility with the user allowed and the bot account not in effectiveGroupAllow, then reply to a bot-sent message; live comments on the issue and linked PR report the same failure on shipped builds.

## Solution Assessment

Yes. The best narrow fix is to treat Telegram bot/self reply targets as allowed supplemental quote context before the allowlist gate, while leaving ordinary non-bot and forwarded-origin filtering unchanged; the broader durable-context issue can remain separate.
