## Summary

Keep open. Current main and the latest release still lack the Telegram bot/self exception in the supplemental-context allowlist gate, while the linked fixing PR remains open and conflicting rather than merged.

## What This Changes

The issue reports that Telegram replies to bot-sent messages can lose agent-visible reply context when group allowlist visibility is active.

## Best Possible Solution

Refresh and land https://github.com/openclaw/openclaw/pull/82079, or an equivalent narrow replacement, while keeping broader durable reply-context contract work separate at https://github.com/openclaw/openclaw/issues/88032.

## Reproduction Assessment

Yes for source-level reproduction: current main checks the reply-target sender only against the group allowlist, so a bot-authored reply target can be excluded under `contextVisibility: "allowlist"` when the bot is not allowlisted. I did not run a live Telegram repro in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to treat Telegram bot/self reply targets as allowed supplemental quote context before the allowlist gate while leaving ordinary non-bot and forwarded-origin filtering unchanged.
