## Summary

Keep open. Current main still gates Telegram supplemental reply context on the reply target sender's group allowlist membership without a bot/self exception, and an open fix PR already references this issue with closing syntax, so the issue should stay paired with that PR until it lands or is replaced.

## What This Changes

The issue asks for Telegram group replies to bot messages to preserve quoted bot reply context under group allowlist visibility.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/82079 or an equivalent narrow fix that treats bot/self reply targets as allowed supplemental context while preserving non-bot allowlist redaction.

## Reproduction Assessment

Yes for source-level reproduction: configure Telegram contextVisibility "allowlist" with a group allowlist that admits the user but not the bot, then reply to a bot-sent message. Current main will compute senderAllowed from the bot sender against effectiveGroupAllow and can drop the quote before the agent turn is built.

## Solution Assessment

Yes, the best fix direction is the narrow bot/self sender exception in the supplemental-context gate, not a broader allowlist relaxation. The fix should keep forwarded-origin and non-bot reply-target redaction intact.
