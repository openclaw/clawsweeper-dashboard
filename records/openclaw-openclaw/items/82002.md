## Summary

Keep open. Current main still has the source-reproducible Telegram allowlist/context path, and an open linked fix PR should be reviewed and landed before this issue is closed.

## What This Changes

The issue reports that Telegram group replies to OpenClaw bot-sent messages can lose reply-target context when supplemental context uses allowlist visibility.

## Best Possible Solution

Review and land the linked Telegram fix after live Telegram proof shows bot-self reply targets are preserved under allowlist visibility without relaxing non-bot redaction.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure Telegram `contextVisibility: "allowlist"` with a group allowlist that admits the user but not the bot, then reply to a bot-sent message; current main checks the reply target sender only against `effectiveGroupAllow`.

## Solution Assessment

Yes. The narrow fix is to treat Telegram bot/self reply targets as allowed supplemental quote context before the allowlist visibility gate, while leaving ordinary non-bot sender filtering and forwarded-origin redaction intact.
