## Label Justifications

- `P2`: This is a normal-priority user-facing Telegram bug with limited channel-specific blast radius but real conversation impact.
- `impact:message-loss`: The quoted bot message can be filtered out before the agent sees it, effectively dropping the reply target context.
- `impact:session-state`: Missing reply-chain context can make the agent answer from stale or unrelated session context.

## Summary

Keep open. Current main and the latest release still route Telegram group supplemental reply context through the quoted sender allowlist without a bot/self exception, and the linked fixing PR is still open rather than merged.

## What This Changes

Request to preserve Telegram bot/self reply target context for group allowlist conversations so quoted bot messages reach the agent.

## Best Possible Solution

Merge a narrow Telegram fix that treats bot/self reply targets as allowed supplemental quote context while preserving the existing non-bot allowlist redaction behavior.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: a group reply target whose sender is the bot but not in `effectiveGroupAllow` gets `senderAllowed=false`, and `contextVisibility: "allowlist"` then blocks the quote context. I did not run a live Telegram bot scenario in this read-only review.

## Solution Assessment

Yes for the narrow bug: the best fix is the linked bot/self exception at the Telegram supplemental-context gate plus regression coverage. The optional outbound cache/durable reply-contract work should stay separate unless maintainers decide to broaden the scope.
