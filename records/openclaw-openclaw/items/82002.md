## Summary

Keep open. Current main and the latest release still apply Telegram group supplemental-context visibility to the reply-target sender without a bot-self exception, and https://github.com/openclaw/openclaw/pull/82079 remains open rather than merged.

## What This Changes

The issue requests preserving Telegram bot-sent reply target context for group allowlist users instead of dropping the quoted bot message from the agent-visible turn.

## Best Possible Solution

Refresh and land https://github.com/openclaw/openclaw/pull/82079, or a narrow replacement, with live Telegram proof that bot-self reply targets survive allowlist visibility while non-bot redaction remains intact.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: use Telegram group allowlist visibility with the user allowed and the bot account not in effectiveGroupAllow, then reply to a bot-sent message; live comments report the same failure on shipped builds.

## Solution Assessment

Yes. The best narrow fix is to treat Telegram bot/self reply targets as allowed supplemental quote context before the allowlist gate while leaving ordinary non-bot and forwarded-origin filtering unchanged.
