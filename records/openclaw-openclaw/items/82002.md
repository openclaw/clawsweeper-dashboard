## Summary

Keep open: current main still drops Telegram bot-authored reply targets when `contextVisibility: "allowlist"` is active and the bot id is not in the group sender allowlist, even though Telegram reply metadata and bot-reply implicit mentions are otherwise supported.

## What This Changes

Ensure Telegram replies to bot-authored messages in allowlisted groups retain reply/quote context instead of being filtered out by sender allowlist checks.

## Best Possible Solution

Patch the Telegram supplemental-context gate to treat the current bot account as allowed only for reply/quote targets it authored, while preserving sender allowlist enforcement for users, other bots, forwarded origins, and group trigger authorization.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path: a Telegram group reply whose `reply_to_message.from.id` equals the bot id is still checked against `effectiveGroupAllow`, so `contextVisibility: "allowlist"` can drop the quote before the turn is built.

## Solution Assessment

Yes. The narrow maintainable fix is to mark only bot-self reply/quote targets as allowed for supplemental quote visibility, then add a focused regression test for allowlisted Telegram groups; outbound cache persistence should remain secondary unless live proof shows it is required.
