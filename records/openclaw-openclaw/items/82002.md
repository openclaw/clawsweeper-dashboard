## Summary

Keep open. Current main and the latest release still lack the Telegram bot/self exception in the supplemental-context allowlist gate, the issue has source-level and live-comment repro evidence, and the linked fixing PR remains open rather than merged.

## What This Changes

The issue asks OpenClaw to preserve Telegram bot-sent message reply context when an allowlisted group user replies to the bot's own message.

## Best Possible Solution

Refresh and land https://github.com/openclaw/openclaw/pull/82079, or an equivalent narrow replacement, while leaving broader durable reply-context architecture to https://github.com/openclaw/openclaw/issues/88032.

## Reproduction Assessment

Yes for source-level reproduction: current main checks bot-authored reply targets only against the group allowlist before including quoted context. Live issue comments also report the same failure on Telegram deployments, though I did not run a live Telegram session in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to treat Telegram bot/self reply targets as allowed supplemental quote context before the allowlist gate while leaving ordinary non-bot and forwarded-origin filtering unchanged.
