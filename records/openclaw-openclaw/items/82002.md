## Label Justifications

- `P2`: The bug affects Telegram reply-context accuracy for an allowlisted group workflow, with limited blast radius to that channel path.
- `impact:session-state`: The reported failure drops quoted reply context before the agent turn, causing transcript/context drift for the response.

## Summary

Keep open. Current `main` and the latest stable release still filter Telegram supplemental reply targets through the group allowlist without a bot/self exception, and the issue is paired with an open closing PR that should be reviewed or replaced rather than bypassed.

## What This Changes

The issue reports that Telegram group replies to bot-sent messages can drop the replied-to bot message from supplemental context when group allowlist visibility is enabled.

## Best Possible Solution

Keep this issue open until the linked PR, or a replacement with the same narrow bot/self reply-target allowance, lands with proof that quoted bot context is preserved while non-bot redaction remains intact.

## Reproduction Assessment

Yes for source reproduction: configure Telegram `contextVisibility: "allowlist"` with a group allowlist that admits the human sender but not the bot, then reply to a bot-sent message. Current source still computes supplemental quote visibility from `effectiveGroupAllow` without a bot/self exception.

## Solution Assessment

Yes. The narrow maintainable solution is to treat Telegram bot/self reply targets as allowed supplemental quote context before the allowlist gate, without relaxing ordinary non-bot sender filtering or forwarded-origin redaction.
