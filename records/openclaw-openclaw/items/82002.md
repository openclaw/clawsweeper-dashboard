## Summary

Keep open. Current main still has a source-reproducible Telegram reply-context bug under group allowlist visibility, and https://github.com/openclaw/openclaw/pull/82079 is an open linked fix PR with closing syntax, so this issue should stay paired with that PR until it lands or is replaced.

## What This Changes

Request to preserve Telegram bot-sent reply target context in group allowlist mode, with optional outbound bot-message cache persistence if needed for hydration.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82079, or an equivalent narrow fix, while keeping non-bot supplemental-context filtering intact and treating cache persistence as separate unless proven necessary.

## Reproduction Assessment

Yes for source reproduction: configure a Telegram group with `contextVisibility: "allowlist"` and an allowlist that admits the user but not the bot, then reply to a bot-sent message. Current main can drop the reply target because the quote sender is checked only against `effectiveGroupAllow`; I did not run live Telegram proof.

## Solution Assessment

Yes. The narrow maintainable fix is to treat Telegram bot/self reply targets as allowed supplemental context before the allowlist gate while preserving ordinary non-bot redaction; the open linked PR is the right venue to validate that direction.
