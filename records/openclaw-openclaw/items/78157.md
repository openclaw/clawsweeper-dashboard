## What This Changes

This branch updates Gateway `chat.history` display projection to emit text-only assistant messages for mixed text/tool-use transcript turns, adds projection and JSONL-backed Gateway regression tests, and records the Control UI/chat fix in the changelog.

## Best Possible Solution

Land this narrow projection fix after maintainer review and exact-head checks are green, then close the linked bug only if no provider-specific disappearing-history shape remains.

## Reproduction Assessment

Yes by source inspection: current main drops assistant turns resolved as `commentary` and the phased sanitizer keeps only `final_answer` text, so a commentary-signed text block in a tool-call assistant turn can disappear from `chat.history`. I did not run a live UI scenario in this read-only pass.

## Solution Assessment

Yes for the mixed tool-use transcript shape: preserving non-empty text blocks from assistant turns that include tool history keeps durable progress visible while pure commentary-only turns stay hidden. A separate fix would be needed for a provider-specific `reasoning_content` variant if that is still reproducible.
