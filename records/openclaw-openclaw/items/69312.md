Full review comments:

- **[P2] Preserve code indentation after audio tag removal:** `src/media/parse.ts:747`
  - body: The branch computes a code-aware leading-normalization regex before `parseAudioTag`, but after removing `[[audio_as_voice]]` it applies the non-code regex unconditionally. For `[[audio_as_voice]]\n\n\tMEDIA:/tmp/code.png`, the tab is stripped from the first surviving code line, so the text can be displayed or reparsed as a top-level directive; reuse the code-aware first-line classification after tag removal.
  - confidence: 0.9
- **[P2] Keep streaming buffering in sync with indented code:** `src/media/parse.ts:563`
  - body: This new guard makes blank-preceded 4-space/tab `MEDIA:` lines plain text in final parsing, but `splitTrailingDirective` still buffers any optional-whitespace `MEDIA:` last line and existing tests assert that old behavior. A streamed indented code line at a chunk boundary can be held as a directive tail and delayed or dropped on reset; update the streaming classifier and tests with the same indented-code rule.
  - confidence: 0.88
- **[P2] Correct the mixed lowercase-prefix expectation:** `src/media/parse.test.ts:425`
  - body: With the restored case-insensitive regex, `media: note MEDIA:/tmp/a.png` is parsed as one greedy directive payload, and that payload is not valid media, so `mediaUrls` remains undefined. This new assertion expects a defined list and is inconsistent with the parser contract; assert the exact intended behavior or use a valid lowercase directive fixture.
  - confidence: 0.84
