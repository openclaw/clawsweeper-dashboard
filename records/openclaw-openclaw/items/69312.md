## Summary

This PR should stay open because it targets a real current-main media parser bug, but the latest head still regresses established lowercase media directive behavior and loses tab indentation after audio-tag cleanup.

## What This Changes

The PR updates `splitMediaFromOutput` and `parseFenceSpans` to skip `MEDIA:` extraction in indented code and malformed fences, removes case-insensitive media parsing, and adds parser/fence tests.

## Best Possible Solution

Land a focused parser repair that skips `MEDIA:` extraction inside indented code blocks and invalidly closed fences while preserving lowercase directive compatibility, code-aware audio cleanup, regression coverage, and a changelog entry.

## Reproduction Assessment

Yes. Current main's source path is clear: `splitMediaFromOutput` only excludes fenced spans, then trim-starts and case-normalizes each line before media extraction, so indented `MEDIA:` code examples enter the directive parser.

## Solution Assessment

No, not as submitted. The indented-code and stricter-fence guards are the right narrow direction, but the PR should preserve lowercase media directives and reuse code-aware leading normalization after audio-tag removal.
