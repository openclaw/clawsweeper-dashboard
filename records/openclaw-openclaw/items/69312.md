## Summary

Keep open. The PR targets a real current-main parser bug, but the proposed patch still conflicts with existing lowercase MEDIA behavior, leaves streaming/audio cleanup rules misaligned, and lacks after-fix real behavior proof.

## What This Changes

The PR changes media and fence parsing to skip MEDIA extraction from indented code blocks, preserve code indentation during cleanup, tighten fence closing validation, and add parser/fence tests.

## Best Possible Solution

Land a focused parser fix that skips indented code blocks while preserving the existing lowercase directive contract and aligning streaming/audio cleanup behavior across the same parser rules.

## Reproduction Assessment

Yes. Source inspection shows `splitMediaFromOutput()` on current main skips only fenced spans before the trimmed `MEDIA:` gate, so a blank-preceded four-space/tab `MEDIA:` line is a clear current-main false-extraction path; I did not execute it because this review is read-only.

## Solution Assessment

No. The parser is the right owner boundary, but this exact patch should not merge until lowercase MEDIA compatibility, streaming buffering, and audio-tag cleanup follow the same rules and real behavior proof is posted.
