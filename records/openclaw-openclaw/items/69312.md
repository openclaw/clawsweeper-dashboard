Full review comments:

- **[P2] Keep streaming MEDIA classification in sync:** `src/media/parse.ts:562`
  - body: The final parser now treats a 4-space/tab `MEDIA:` line after a blank as code, but `splitTrailingDirective` still buffers any trailing line matching `/^\s*MEDIA:/i`. In streaming paths, `Example:\n\n    MEDIA:/tmp/code.png` can be held in `pendingTail`; if a stream boundary resets before final flush, that visible code line is never emitted. Share the indented-code decision with the streaming accumulator and add a streamed regression.
  - confidence: 0.9
- **[P2] Reuse code-aware cleanup after audio tags:** `src/media/parse.ts:743`
  - body: After `parseAudioTag` removes `[[audio_as_voice]]`, this cleanup uses a non-code-aware leading-normalization regex. If the first surviving line is tab- or 4-space-indented code, it strips leading indentation, so a tab-indented `MEDIA:` example can become top-level text on a later parse. Reuse the computed code-aware normalization after audio tag removal.
  - confidence: 0.88
- **[P2] Fix the mixed lowercase MEDIA expectation:** `src/media/parse.test.ts:424`
  - body: This new assertion is backwards for the current parser: `MEDIA_TOKEN_RE` greedily consumes from the first lowercase `media:` through the end of the line, making `note MEDIA:/tmp/a.png` one invalid payload rather than two extracted directives. Either assert no media for this prose case or change the parser and cover the exact `mediaUrls`.
  - confidence: 0.86
