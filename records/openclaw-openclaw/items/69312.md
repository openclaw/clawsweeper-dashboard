## Summary

## What This Changes

The branch adds an indented-code guard to `splitMediaFromOutput`, tightens `parseFenceSpans` closing validation, adjusts cleanup around kept code lines, removes case-insensitive MEDIA matching, and adds parser/fence tests.

## Best Possible Solution

Land a focused parser/fence fix after preserving lowercase MEDIA compatibility, aligning streaming and audio cleanup with the same code-aware rules, and adding redacted real behavior proof plus focused regressions.

## Reproduction Assessment

Yes, source-level. Current main skips fenced spans only and then applies the trimmed, case-insensitive MEDIA gate, so a blank-preceded 4-space or tab-indented MEDIA line reaches extraction; this read-only review did not execute tests.

## Solution Assessment

No. The parser is the right place to fix the bug, but this branch should not merge until lowercase MEDIA semantics, streaming buffering, and audio-tag cleanup follow the same directive/code rules.
