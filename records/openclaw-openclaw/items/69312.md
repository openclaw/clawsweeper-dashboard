## Best Possible Solution

Use one shared indented-code decision across MEDIA parsing, streaming tail buffering, and audio-tag cleanup, then require focused regression tests and redacted real OpenClaw proof before merge.

## Reproduction Assessment

Yes, from source inspection: current main only protects fenced code and then trims indentation before checking for MEDIA:, so a blank-line-preceded 4-space or tab-indented MEDIA line follows the directive path. I did not run tests because this review kept the checkout read-only.

## Solution Assessment

No, not yet: the parser-level direction is maintainable, but the same code-line semantics need to be applied to streaming buffering and post-audio cleanup before this is the best fix.
