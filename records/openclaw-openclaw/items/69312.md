## What This Changes

The PR changes media parsing to skip `MEDIA:` directives in indented code blocks, tightens markdown fence closing validation, and adds parser regression tests.

## Best Possible Solution

Land a focused parser fix that reuses code-aware leading normalization after audio-tag stripping, preserves the lowercase media directive contract, and includes redacted real CLI or Telegram proof.

## Solution Assessment

No, not yet. The indented-code guard is the right direction, but the audio-tag cleanup must reuse the code-aware leading normalization before this is merge-ready.
