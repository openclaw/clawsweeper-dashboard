## What This Changes

The PR adds indented-code MEDIA skipping, code-aware media cleanup, stricter markdown fence closing validation, and media/fence regression tests.

## Best Possible Solution

Land a focused parser fix that preserves indented-code text without regressing lowercase MEDIA directives, keeps streaming/audio cleanup semantics aligned, and includes redacted real CLI or Telegram proof.

## Reproduction Assessment

Yes, source-level. Current main skips only fenced spans and then trims indentation before case-insensitive `MEDIA:` matching, so blank-preceded 4-space or tab-indented `MEDIA:` code lines can be extracted.

## Solution Assessment

No, not as submitted. The indented-code guard is the right direction, but the patch should preserve lowercase directive support and align streaming plus audio cleanup with the new parser behavior.
