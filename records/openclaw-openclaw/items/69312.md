## What This Changes

The PR updates media directive parsing to skip 4-space/tab indented code blocks, preserve retained code indentation, tighten markdown fence close validation, and add media/fence regression tests.

## Best Possible Solution

Keep this as the canonical fix path for https://github.com/openclaw/openclaw/issues/69311, reuse code-aware leading normalization after audio tag stripping, add the tab-indented audio regression, and land only with redacted real behavior proof.

## Reproduction Assessment

Yes. Current main trims indentation before the `MEDIA:` check and only skips fenced spans, so the issue-body input with a blank line followed by a 4-space or tab-indented `MEDIA:` line has a clear source-level failing path.

## Solution Assessment

Mostly yes, but not as currently written. The indented-code guard is the right narrow parser fix, while the post-audio-tag cleanup should reuse the same code-aware normalization decision and the PR still needs real behavior proof.
