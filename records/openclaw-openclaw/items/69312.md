## Summary

## What This Changes

The PR changes media and fence parsing to preserve indented-code MEDIA lines, protect retained indentation during media cleanup, validate fence closing lines, and add media/fence regression tests.

## Best Possible Solution

Land a focused parser fix that skips indented-code MEDIA extraction while preserving existing lowercase MEDIA and streaming contracts, reuses code-aware normalization after audio tags, adds the required changelog entry, and includes real setup proof.

## Reproduction Assessment

Yes, by source inspection. Current main only skips fenced spans and then checks `line.trimStart().toUpperCase().startsWith("MEDIA:")`, so blank-preceded 4-space or tab-indented code examples can enter extraction.

## Solution Assessment

No, not as submitted. The indented-code guard is the right direction, but the patch should preserve current lowercase MEDIA semantics, align streaming behavior, reuse code-aware audio cleanup, add changelog coverage, and provide real behavior proof.
