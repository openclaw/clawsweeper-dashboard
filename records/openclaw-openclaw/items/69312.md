## What This Changes

The PR updates media parsing to preserve indented-code `MEDIA:` lines as text, protects retained code indentation during cleanup, tightens markdown fence parsing, and adds parser regression tests.

## Best Possible Solution

Land the indented-code protection after removing or fully aligning the case-sensitivity change across media parsing, block replies, streaming buffering, and tests, with focused regression coverage and real after-fix proof.

## Reproduction Assessment

Yes from source inspection: current main trims indentation before testing for `MEDIA:` and only checks fenced spans, so an indented code-block line can be extracted as media. I did not run tests because this review is read-only.

## Solution Assessment

No. The indented-code guard is the right direction, but changing MEDIA case semantics in only this parser is not the narrowest maintainable fix because current reply and streaming paths still define lowercase support.
