## Summary

Keep open: the reported parser bug is valid on current main, but this PR is not merge-ready because it breaks established lowercase `media:` directive support, leaves streaming buffering out of sync with the new indented-code rule, and still lacks required after-fix real behavior proof.

## What This Changes

The branch changes media and markdown fence parsing plus parser tests to avoid extracting `MEDIA:` directives from indented code blocks and malformed fence bodies.

## Best Possible Solution

Keep the focused parser/fence fix, but preserve current lowercase directive behavior, update streaming buffering to mirror the parser’s directive/code rules, make post-audio cleanup code-aware, and require redacted real behavior proof before merge.

## Reproduction Assessment

Yes, source-level. Current main skips only fenced spans before a case-insensitive `MEDIA:` line guard, so a blank-preceded 4-space or tab-indented code line can be extracted as media.

## Solution Assessment

No. The parser/fence area is the right fix boundary, but this branch is not the best merge candidate until lowercase directive support, streaming buffering, and audio-tag cleanup are aligned with existing contracts.
