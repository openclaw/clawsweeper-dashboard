The branch adds indented-code detection and code-aware cleanup to media parsing, tightens markdown fence closing validation, and adds media/fence parser tests.

## Reproduction Assessment

Yes, source-level. Current main trims indentation before a case-insensitive `MEDIA:` gate and only skips fenced spans, so indented code-block directives can be extracted; the PR regression is also source-reproducible against the existing lowercase block-reply test.

## Solution Assessment

  - body: Current main explicitly parses `media: ./report.pdf` in block replies, and commit f350bb4dfc62 introduced that contract. Removing the regex `i` flag makes lowercase directives visible text instead of attachment metadata, so the existing block-reply regression will fail.

  - confidence: 0.86

  - confidence: 0.86
