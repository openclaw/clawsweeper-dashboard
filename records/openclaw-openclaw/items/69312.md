## Summary

Keep open: current main still has the indented-code MEDIA false-positive, but this PR should not merge yet because it regresses the existing lowercase media-directive contract, leaves streaming buffering inconsistent with final parsing, and lacks real behavior proof.

## What This Changes

The PR updates MEDIA and fence parsing plus tests to skip MEDIA directives in indented code blocks, preserve retained code indentation, validate fence closing lines, and change MEDIA matching case behavior.

## Best Possible Solution

Land a focused parser fix that skips indented-code MEDIA lines while preserving existing lowercase directive and streaming contracts, reuses code-aware cleanup after audio tags, adds the required changelog entry, and includes real setup proof.

## Reproduction Assessment

Yes, by source inspection. Current main skips only fenced spans and then case-normalizes trim-started MEDIA lines, so a blank-preceded 4-space or tab-indented code example can be parsed as media.

## Solution Assessment

No, not as submitted. The indented-code guard is the right direction, but the patch should preserve existing lowercase directive behavior, align streaming buffering, reuse code-aware audio cleanup, add changelog coverage, and provide real behavior proof.
