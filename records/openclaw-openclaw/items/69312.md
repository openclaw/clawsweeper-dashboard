## Summary

Keep open: the bug is real and the indented-code-block direction is useful, but the latest PR head still regresses the supported lowercase `media:` directive contract and lacks required real behavior proof.

## What This Changes

The PR changes media and fence parsing so indented code-block `MEDIA:` lines are skipped, code indentation is partly preserved after extraction, malformed fence closers stay open, and parser tests cover those cases.

## Best Possible Solution

Keep the indented-code-block protection, but preserve the tested lowercase media directive contract across parser and streaming paths unless maintainers explicitly approve changing it.

## Reproduction Assessment

Yes, source-level. Current main only guards fenced spans before a case-insensitive `MEDIA:` line check, so 4-space or tab-indented code-block lines remain extractable as media directives.

## Solution Assessment

No, not yet. The indented-code heuristic is a reasonable narrow fix, but the case-sensitivity cleanup should be removed or handled as an explicit contract change because lowercase media directives are already tested behavior.
