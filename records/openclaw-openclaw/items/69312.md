The PR updates the media parser and Markdown fence helper to skip `MEDIA:` extraction inside indented code blocks and invalidly closed fences, adjusts cleanup normalization, and adds parser/fence tests.

## Best Possible Solution

Land a focused parser repair that skips `MEDIA:` extraction inside indented code blocks and invalidly closed fences while preserving lowercase directive compatibility, sharing code-aware cleanup after audio-tag removal, adding regression coverage, and documenting the fix in the changelog.

Yes. Current main's source path is clear: `splitMediaFromOutput` skips only fenced spans, then trim-starts and case-normalizes each line before MEDIA extraction, so indented `MEDIA:` code examples enter the directive parser.

## Solution Assessment
