## Summary

Keep open: the reported indented-code MEDIA bug is valid, but the PR still regresses the merged lowercase MEDIA contract and has an audio-tag cleanup path that strips preserved code indentation; it also lacks required real behavior proof.

## What This Changes

The PR changes media and markdown fence parsing to skip MEDIA extraction inside indented code blocks, preserve retained code indentation during cleanup, make MEDIA matching case-sensitive, and add media/fence regressions.

## Best Possible Solution

Land a focused parser fix that protects indented code blocks and retained code indentation without regressing lowercase MEDIA support or audio-tag normalization, backed by parser/block-reply/streaming regressions and real behavior proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current main skips fenced code only, then trims indentation before matching `MEDIA:`, so 4-space/tab indented MEDIA lines outside fences can be extracted. I did not execute tests because this was a read-only review.

## Solution Assessment

No, not as currently written. The indented-code direction is the right narrow fix, but the PR should preserve the existing lowercase MEDIA contract and apply code-aware normalization after audio tag stripping before it is mergeable.
