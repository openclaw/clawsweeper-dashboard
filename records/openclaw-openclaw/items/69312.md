## What This Changes

The PR changes media parsing to skip `MEDIA:` directives in indented code blocks, tightens fence close validation, removes case-insensitive MEDIA matching, and adds media/fence parser tests.

## Best Possible Solution

Land a repaired parser/fence patch that preserves lowercase media compatibility, applies code-aware cleanup after audio tag stripping, records the fix in the changelog, and includes real OpenClaw behavior proof.

## Reproduction Assessment

Yes, source-reproducible. Current main only excludes fenced spans and then uses `line.trimStart().toUpperCase().startsWith("MEDIA:")`, so an indented code-block line can be parsed as a media directive.

## Solution Assessment

No, not as submitted. The indented-code and fence-close direction is narrow, but latest head should preserve lowercase media behavior and reuse the code-aware cleanup path after audio tag stripping.
