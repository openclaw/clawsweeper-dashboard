## What This Changes

The PR changes media parsing to preserve indented-code `MEDIA:` lines, tightens markdown fence close validation, removes case-insensitive MEDIA matching, and adds parser/fence regression tests.

## Best Possible Solution

Land a repaired parser/fence patch that preserves lowercase MEDIA support, aligns streaming with final parsing, reuses code-aware cleanup after audio tags, includes a changelog entry, and includes redacted real behavior proof.

## Reproduction Assessment

Yes from source inspection: current main only skips fenced spans and then tests `line.trimStart().toUpperCase().startsWith("MEDIA:")`, so blank-preceded 4-space or tab-indented `MEDIA:` lines can be extracted.

## Solution Assessment

No, not as submitted. The indented-code direction is maintainable, but the patch should preserve current case semantics, keep streaming and final parsing aligned, and apply the code-aware cleanup after audio tag stripping.
