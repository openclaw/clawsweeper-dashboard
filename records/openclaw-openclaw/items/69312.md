Full review comments:

- **[P1] Keep paragraph-indented MEDIA lines buffered:** `src/auto-reply/reply/streaming-directives.ts:83`
  - body: This check treats any 4-space/tab `MEDIA:` last line after a newline as indented code because `text.slice(0, lastNewline + 1)` always ends at that newline, making `/\n[ \t]*$/` true for ordinary preceding paragraphs. That contradicts the final parser case added in this PR (`some text\n    MEDIA: /tmp/real.png` remains a directive), so a streamed path split like `Preview:\n    MEDIA:/tmp/cov` + `er.png` will no longer be held until the full attachment path arrives.
