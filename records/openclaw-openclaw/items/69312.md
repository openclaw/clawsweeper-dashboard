Full review comments:

- **[P2] Skip indented-code MEDIA tails in streaming buffering:** `src/media/parse.ts:562-563`
  - body: Adding the indented-code guard here means a blank-line 4-space/tab `MEDIA:` line should remain visible text, but `splitTrailingDirective()` still buffers any `/^\s*MEDIA:/i` last line and `reset()` drops `pendingTail`. In block streaming, a chunk ending with `\n\n    MEDIA:/tmp/code` can still lose the code line before final flush. Update the streaming guard/tests to share this indented-code rule.
  - confidence: 0.84
- **[P2] Reuse code-aware leading normalization after audio tags:** `src/media/parse.ts:745-748`
  - body: The main cleanup protects a first surviving code line, but after `parseAudioTag()` this branch always applies the non-code leading normalizer. With `[[audio_as_voice]]\n\n\tMEDIA:/tmp/code.png`, the tab-indented code line becomes first content and the regex removes its tab, changing visible code text and making it look like a directive. Reuse the `firstLineIsCode`/`leadingNormRegex` path and add the tab audio regression.
  - confidence: 0.87
