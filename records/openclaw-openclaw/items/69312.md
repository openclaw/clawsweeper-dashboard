## Summary

Keep open. The reported parser bug is real and this PR is the canonical fix path, but the current head still regresses existing lowercase `media:` behavior, leaves streaming buffering and audio-tag cleanup inconsistent with the new indented-code rules, and lacks after-fix real behavior proof.

## What This Changes

The PR changes media/fence parsing to skip indented-code `MEDIA:` lines, tighten fence closing validation, adjust whitespace cleanup for retained code lines, remove case-insensitive MEDIA matching, and add parser tests.

## Best Possible Solution

Land a focused parser fix that preserves indented-code `MEDIA:` text while keeping lowercase media semantics and aligning batch parsing, streaming buffering, and audio-tag cleanup rules.

## Reproduction Assessment

Yes, source-level. Current main skips fenced spans only and then trims indentation before the `MEDIA:` gate, so an indented-code `MEDIA:` line can be extracted without a new product decision.

## Solution Assessment

No, not as submitted. The indented-code direction is narrow and appropriate, but the branch should preserve the existing lowercase contract and apply the same code-aware rules to streaming and audio cleanup before merge.

  - confidence: 0.9

  - confidence: 0.86
