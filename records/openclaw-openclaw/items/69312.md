## Review Findings

Overall confidence: 0.88

Full review comments:

- **[P2] Align streaming tail buffering with code lines:** `src/media/parse.ts:562-563`
  - body: Now that the parser preserves blank-line 4-space/tab `MEDIA:` lines as visible code, the streaming accumulator also has to stop buffering those same lines as directive tails. `splitTrailingDirective` still buffers `/^\s*MEDIA:/i`, so a streamed chunk ending with an indented code example can remain in `pendingTail` and be lost on `reset()` instead of being delivered as text.
  - confidence: 0.88
- **[P2] Reuse the code-aware normalizer after audio tags:** `src/media/parse.ts:745-748`
  - body: This post-`parseAudioTag` cleanup always applies the non-code leading normalizer. If `[[audio_as_voice]]` is removed before a tab or 4-space indented code line, it strips leading whitespace from the first code line, so the code text is still corrupted by the path this PR is trying to preserve.
  - confidence: 0.87
