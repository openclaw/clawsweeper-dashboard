## Summary

Keep open. The indented-code false positive is real by source inspection, but the latest patch still changes the existing lowercase `media:` directive contract and lacks after-fix real OpenClaw behavior proof, so it should not merge as submitted.

## What This Changes

The PR modifies media and markdown fence parsing to skip `MEDIA:` extraction from indented code blocks, tighten fence closing validation, change MEDIA case matching, and add parser tests.

## Best Possible Solution

Land the indented-code-block parser guard and fence coverage while preserving lowercase MEDIA semantics across `splitMediaFromOutput`, reply delivery, and streaming.

## Reproduction Assessment

Yes, source-level. Current main trims indentation before the `MEDIA:` line gate and only excludes fenced spans, so a 4-space or tab-indented code line can be parsed and stripped as media outside fences.

## Solution Assessment

No, not as submitted. The indented-code guard is a reasonable narrow fix, but changing lowercase directive semantics is not the best solution without preserving the existing contract or making an explicit, fully aligned contract change.
