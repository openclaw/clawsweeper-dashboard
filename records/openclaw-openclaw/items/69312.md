## Summary

Keep this PR open. The bug fix direction is valid, but the current patch still has two blocking parser regressions and the external contributor has not posted real behavior proof beyond test-plan claims.

## What This Changes

The branch changes media and markdown fence parsing to skip `MEDIA:` extraction inside indented code blocks, preserve code indentation during media cleanup, tighten fence closing detection, and add focused parser tests.

## Best Possible Solution

Fix the two parser regressions, keep the focused indented-code behavior with regression coverage, compare against the broader indentation PR, and require redacted real-run proof before merge.

## Reproduction Assessment

Yes, from source inspection: current main trims line starts and only excludes fenced spans, so a blank-line-prefixed four-space or tab-indented `MEDIA:` line reaches extraction. I did not run tests because this review was read-only.

## Solution Assessment

No, not yet. The chosen direction is maintainable, but this PR should first align parser and streaming case semantics and preserve tab-indented code after audio tag stripping.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Keep streaming MEDIA case handling aligned:** `src/media/parse.ts:17`
  - body: Removing the `i` flag makes `splitMediaFromOutput` treat lowercase `media:` as plain text, but the PR head still buffers streaming directive tails case-insensitively in `src/auto-reply/reply/streaming-directives.ts`. A chunk ending in `media:/tmp/x` can be held in `pendingTail` and then dropped on a reset/final boundary instead of being emitted as text, so the streaming and non-streaming paths disagree.
  - confidence: 0.9
- **[P2] Preserve tab code indentation after audio tags:** `src/media/parse.ts:747`
  - body: This second normalization pass runs after `parseAudioTag` strips `[[audio_as_voice]]`, but it no longer knows whether the first surviving line was recorded as code. For `[[audio_as_voice]]\n\n\tMEDIA:/tmp/code.png`, the leading tab is stripped, turning a tab-indented code example into a top-level-looking `MEDIA:` line and undoing the indentation protection added above.
  - confidence: 0.86
