## Review Findings

Overall confidence: 0.86

Full review comments:

- **[P2] Align streaming buffering with indented code parsing:** `src/media/parse.ts:563-569`
  - body: This new indented-code skip changes the final parser so `\n\n    MEDIA:/tmp/example.png` is preserved as text, but `splitTrailingDirective` still buffers any last line matching optional whitespace plus `MEDIA:`. In a streamed reply, that code line can be held in `pendingTail` and dropped if the stream item resets before a final flush, exactly the loss mode the streaming comment warns about. Please update the streaming guard and add a regression for blank-preceded 4-space/tab `MEDIA:` code lines.
  - confidence: 0.88
- **[P2] Preserve tab code indentation after audio tag stripping:** `src/media/parse.ts:745-748`
  - body: The code-aware leading normalization only runs before `parseAudioTag`. If the reply starts with `[[audio_as_voice]]` followed by a blank line and a tab-indented code line, `parseAudioTag` removes the tag and this cleanup path can strip the leading tab from the first surviving code line. Reuse the `firstLineIsCode` normalization after audio-tag stripping, and add a tab-indented regression beside the 4-space case.
  - confidence: 0.84
