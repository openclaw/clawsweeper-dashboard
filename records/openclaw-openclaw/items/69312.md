## Summary

Keep open: the reported indented-code-block bug is valid and the main heuristic is plausible, but the current patch regresses the existing lowercase `media:` directive contract and still has only test claims rather than real behavior proof.

## What This Changes

The PR updates media and fence parsing to skip `MEDIA:` extraction from indented code blocks, preserve some code indentation during cleanup, tighten fence closing validation, and add parser regression tests.

## Best Possible Solution

Land the indented-code protection only after preserving the existing lowercase media directive contract, keeping streaming buffering aligned, and adding redacted real behavior proof for the fixed path.

## Reproduction Assessment

Yes, source-level. Current main only skips fenced code spans before media extraction and then trims indentation before the `MEDIA:` prefix check, so a four-space or tab-indented code-block line can still be treated as a real media directive.

## Solution Assessment

No, not yet. The indented-code heuristic is a reasonable narrow direction, but the parser case-sensitivity cleanup reverts a tested current behavior and should be removed or handled as a broader contract change.
