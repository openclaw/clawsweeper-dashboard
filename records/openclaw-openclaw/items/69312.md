## Best Possible Solution

Yes. Current main’s source path is clear: fenced spans are skipped, but indented code is not, and the MEDIA gate uses trimStart().toUpperCase().startsWith("MEDIA:"), so a blank-preceded 4-space or tab-indented MEDIA line can be extracted.

## Solution Assessment

No, not as submitted. The indented-code and stricter fence-close direction is maintainable, but the patch should preserve current lowercase semantics, keep streaming/final parsing aligned, and apply code-aware cleanup consistently after audio tag stripping.
