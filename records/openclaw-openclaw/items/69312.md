## Summary

Keep open: the reported parser bug is source-reproducible on current main, but this PR still introduces a lowercase MEDIA semantics regression and lacks required after-fix real behavior proof.

## What This Changes

The PR changes media and fence parsing to skip MEDIA directives inside indented code blocks, preserve retained code indentation, tighten fence closing validation, and add parser tests.

## Best Possible Solution

Land a focused parser fix that preserves indented-code `MEDIA:` lines while keeping MEDIA case semantics aligned across `splitMediaFromOutput`, block reply parsing, and streaming buffering, with redacted real behavior proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path: `splitMediaFromOutput` skips fenced spans only, then trims indentation before a case-insensitive `MEDIA:` gate, so indented code lines can be extracted as media.

## Solution Assessment

No, not as patched. The indented-code guard is in the right area, but the case-sensitivity change is not the narrowest maintainable fix because it conflicts with block reply and streaming contracts, and proof is still missing.
