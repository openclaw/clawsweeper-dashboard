Current main still has the reported indented-code false-positive, but the latest PR head is not merge-ready because it regresses established lowercase `media:` compatibility and leaves a post-audio tab-indentation hole; it also needs the required changelog entry and downstream fence-parser validation.

## What This Changes

The PR changes media parsing and fence parsing to skip indented-code `MEDIA:` lines, preserve retained code indentation during cleanup, reject invalid fence closers with trailing text, make MEDIA matching uppercase-only, and add media/fence parser tests.

## Best Possible Solution

Land a focused version that preserves indented-code `MEDIA:` examples without changing established lowercase directive compatibility, applies the same code-aware leading cleanup after audio-tag stripping, keeps the fence tightening only with downstream validation, and adds an active-version changelog Fixes entry.

Yes. ## Solution Assessment

No, not as submitted. The indented-code heuristic is the right narrow direction, but the latest diff should preserve lowercase MEDIA semantics across parser/block/streaming surfaces and reuse code-aware cleanup after audio-tag removal before merge.
