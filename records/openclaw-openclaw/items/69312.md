## Summary

Keep open: current main still has the reported false-positive path and this PR fixes the central parser behavior, but the latest PR head still has two P2 regressions plus missing real behavior proof, so it is not merge-ready or closeable.

## What This Changes

This PR changes MEDIA and markdown fence parsing to skip MEDIA extraction from indented code blocks, preserve retained code indentation, make MEDIA matching case-sensitive, and add parser regression tests.

## Best Possible Solution

Keep the focused indented-code fix, repair the streaming case and audio-tag tab normalization gaps, reconcile overlapping indentation behavior, then merge after redacted real-run proof.

## Reproduction Assessment

Yes. From source inspection, current main trims line starts and only excludes fenced spans, so a blank-line-prefixed 4-space or tab-indented `MEDIA:` line reaches extraction.

## Solution Assessment

No, not yet. The core approach is narrow and maintainable, but the branch should first align streaming case semantics and make post-audio-tag leading normalization code-line-aware.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Keep streaming MEDIA case handling aligned:** `src/media/parse.ts:17`
  - body: Removing the `i` flag makes `splitMediaFromOutput` treat lowercase `media:` as text, but the streaming accumulator still buffers `MEDIA:` tails and partial prefixes case-insensitively. A chunk like `media:/tmp/x` can be held in `pendingTail` and dropped on reset instead of being emitted, so streaming and non-streaming paths disagree.
  - confidence: 0.9
- **[P2] Preserve tab code indentation after audio tags:** `src/media/parse.ts:747`
  - body: After `parseAudioTag` strips `[[audio_as_voice]]`, this second normalization pass no longer knows whether the first surviving line was recorded as code. For `[[audio_as_voice]]\n\n\tMEDIA:/tmp/code.png`, the leading tab can be stripped, changing visible code formatting and making the line look like a top-level directive.
  - confidence: 0.87
