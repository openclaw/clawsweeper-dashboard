## Summary

## What This Changes

The PR updates MEDIA parsing and markdown fence parsing to skip indented-code MEDIA tokens, preserve retained code indentation during cleanup, tighten invalid fence closing, and add parser/fence regression tests.

## Best Possible Solution

Land a focused parser fix that protects CommonMark indented code from MEDIA extraction while preserving existing lowercase MEDIA and audio-tag indentation contracts, with regression coverage and redacted real behavior proof.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: splitMediaFromOutput only excludes fenced spans, then trim-starts indented lines before a case-insensitive MEDIA gate.

## Solution Assessment

No. The indented-code guard is in the right area, but the current PR also changes established lowercase MEDIA behavior and can strip tab indentation after audio tag removal; the safer fix preserves those contracts.
