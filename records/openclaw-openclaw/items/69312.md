## Summary

## What This Changes

The PR adds an indented-code heuristic and tests to skip MEDIA extraction from 4-space/tab code blocks, tightens markdown fence closing validation, and changes MEDIA token case handling.

## Best Possible Solution

Land a focused parser fix that skips `MEDIA:` extraction inside indented code blocks while preserving established lowercase and streaming semantics, sharing one code-aware cleanup path, documenting the user-facing fix in the changelog, and coordinating with the overlapping indentation PR.

## Reproduction Assessment

Yes, by source inspection. Current main only guards fenced code spans before trimming leading whitespace and matching `MEDIA:` case-insensitively, so an indented code-block line can be extracted as media.

## Solution Assessment

No, not as submitted. The indented-code guard is the right direction, but the patch should preserve existing lowercase/streaming MEDIA contracts, reuse the code-aware cleanup after audio tag stripping, and include changelog plus real setup proof before merge.
