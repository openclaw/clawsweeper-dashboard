Full review comments:

- **[P2] Fix the lowercase MEDIA regression test:** `src/media/parse.test.ts:421-425`
  - body: This new test expects `media: note MEDIA:/tmp/a.png` to produce `mediaUrls`, but the restored case-insensitive `MEDIA_TOKEN_RE` greedily matches from the lowercase prefix through the end of the line as one payload. That payload is not a valid media path, so the assertion does not match the parser behavior and should fail unless the parser contract is intentionally changed.
  - confidence: 0.93
- **[P2] Keep streaming buffering in sync with code blocks:** `src/media/parse.ts:562-563`
  - body: The parser now preserves blank-line 4-space/tab `MEDIA:` lines as visible code, but the streaming accumulator still buffers any whitespace-indented `MEDIA:` last line. A streamed chunk ending with an indented code example can sit in `pendingTail` and then be dropped by `reset()` before final flush, leaving the same user-visible text loss in streamed replies.
  - confidence: 0.87
- **[P2] Reuse code-aware cleanup after audio tags:** `src/media/parse.ts:745-748`
  - body: After `parseAudioTag()`, this branch always applies the non-code leading normalizer. For `[[audio_as_voice]]` followed by a preserved tab or 4-space indented code line, tag removal makes the code line first visible content and this regex can strip its indentation, changing the code text the fix is meant to preserve.
  - confidence: 0.88
