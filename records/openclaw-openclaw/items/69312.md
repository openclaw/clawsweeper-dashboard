## Summary

Keep open: this PR targets a real source-reproducible media parser bug, but the current diff regresses the existing lowercase media directive contract and leaves streaming directive buffering aligned with the old indented-media behavior. The external PR also still needs real behavior proof before merge.

## What This Changes

The branch adds an indented-code-block skip in `splitMediaFromOutput`, tightens `parseFenceSpans` closing fence validation, removes case-insensitive MEDIA parsing, and adds parser/fence tests.

## Best Possible Solution

Land the indented-code-block and fence parsing fixes after preserving lowercase media directives, aligning streaming directive buffering with parser semantics, and adding real behavior proof.

## Reproduction Assessment

Yes, source-level: current main skips fenced spans only, then a 4-space/tab-indented `MEDIA:` line can pass through `trimStart().toUpperCase().startsWith("MEDIA:")` and be extracted. I did not run tests because this is a read-only review.

## Solution Assessment

No, not as submitted. The indented-code guard is a reasonable narrow fix, but the branch changes established lowercase directive behavior and should also update streaming tail detection to match the new indented-code semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve lowercase media directive support:** `src/media/parse.ts:17`
  - body: Current main treats lowercase `media:` as a valid reply media directive, with regression coverage in `reply-delivery.test.ts` and case-insensitive streaming buffering. Removing the parser's `i` flag and making the line gate uppercase-only makes those directives visible text instead of media, leaving the parser contract out of sync with existing consumers.
  - confidence: 0.94
- **[P2] Align streaming buffering with indented code lines:** `src/media/parse.ts:565-566`
  - body: This parser now preserves a 4-space/tab `MEDIA:` line after a blank as code text, but `createStreamingDirectiveAccumulator` still buffers any last line matching optional whitespace plus `MEDIA:`. In block streaming, a chunk ending after such an indented code line can move visible code into `pendingTail` and a reset can drop it, so the streaming guard needs the same indented-code exclusion and coverage.
  - confidence: 0.86
