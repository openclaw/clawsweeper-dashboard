The branch changes MEDIA parsing to preserve blank-preceded 4-space/tab indented MEDIA lines as code text, preserves code indentation during cleanup, tightens fence closing detection, updates streaming tail buffering, and adds parser/fence tests.

## Best Possible Solution

Land a focused parser fix only after the streaming tail guard mirrors the final parser's indented-code boundaries, focused regressions cover that path, and redacted live OpenClaw or Telegram proof shows code text is preserved without an attachment.
