The branch changes `splitMediaFromOutput` to skip `MEDIA:` extraction from indented code blocks, tightens markdown fence-close parsing, removes case-insensitive media matching, and adds parser tests.

## Best Possible Solution

Land a repaired parser/fence patch that preserves lowercase media compatibility, keeps audio-tag cleanup code-aware, records the user-facing fix in the changelog, and includes real OpenClaw behavior proof.

Yes, source-reproducible. Current main excludes fenced spans but not indented code blocks before applying a trim-start `MEDIA:` gate, so an indented code-block directive can be parsed as media.

## Solution Assessment

No, not as submitted. The indented-code and fence-close approach is narrow, but the PR should preserve existing lowercase media behavior and reuse code-aware cleanup after audio tag stripping before merge.
