## Summary

Keep open: this PR addresses a real media-parser bug, but the patch should not merge as-is because it changes lowercase `media:` handling while current outbound and streaming paths still treat lowercase media directives as supported, and the external PR still lacks real behavior proof.

## What This Changes

The PR updates media parsing to skip `MEDIA:` extraction from indented code blocks, preserve retained code indentation during cleanup, tighten fenced-code closing detection, and add media/fence regression tests.

## Best Possible Solution

Land a narrow parser fix that skips CommonMark-style indented code directives while preserving the existing lowercase media contract, or update all callers/tests together if maintainers intentionally retire that compatibility.

## Reproduction Assessment

Yes, via source inspection: current main trims indentation before matching `MEDIA:`, while `parseFenceSpans` only covers fenced blocks. I did not run tests because this review was required to keep the checkout read-only.

## Solution Assessment

No, not as currently patched. The indented-code guard is in the right area, but the case-sensitivity change needs to be aligned with outbound and streaming directive handling, and the contributor still needs to add real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

- **[P2] Keep lowercase media handling aligned:** `src/media/parse.ts:573`
