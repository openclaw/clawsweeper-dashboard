## Summary

Keep open: the core indented-code-block fix is valid, but the current PR still changes MEDIA parsing to case-sensitive without aligning the streaming directive accumulator, and the external PR lacks after-fix real behavior proof.

## What This Changes

The PR updates media parsing to skip `MEDIA:` directives inside indented code blocks, preserve code indentation during cleanup, validate markdown fence closers, and add parser/fence regression tests.

## Best Possible Solution

Land a version that preserves indented-code `MEDIA:` text while keeping parser and streaming directive case rules aligned, then add real setup proof before merge.

## Reproduction Assessment

Yes. Source inspection of current main shows the reported false positive path: indented lines are trimmed before `MEDIA:` matching and only fenced spans are protected; I did not execute tests because this review is read-only.

## Solution Assessment

No, not as currently proposed. The indented-code heuristic is a reasonable narrow fix, but the case-sensitivity cleanup must either be dropped or also update `src/auto-reply/reply/streaming-directives.ts`; real behavior proof is still required before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Align streaming directive case handling:** `src/media/parse.ts:17`
  - body: This makes lowercase `media:` plain text by removing case-insensitive MEDIA parsing, but the streaming accumulator still buffers `media:` and partial lowercase prefixes with `/i`. A streamed chunk like `media:/tmp/x` can stay in `pendingTail` and be lost on `reset()` instead of being emitted as text, so keep parser case-insensitive or make the streaming guards case-sensitive too.
