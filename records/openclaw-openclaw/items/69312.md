## What This Changes

The PR adds indented-code-aware MEDIA parsing, code-aware cleanup, stricter fence closing validation, case-sensitive MEDIA matching, and media/fence regression tests.

## Best Possible Solution

## Reproduction Assessment

Yes. Current main can be traced directly through `splitMediaFromOutput`: indented lines are only protected when they are inside fenced spans, then `trimStart().toUpperCase().startsWith("MEDIA:")` can treat indented code as a directive.

## Solution Assessment

No, not as submitted. The indented-code heuristic is the right direction, but the case-semantics change and audio-tag cleanup path need repair before this is the narrow maintainable fix.
