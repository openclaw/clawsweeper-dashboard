## Summary

Keep open: the indented-code false-positive is a real source-level bug and the PR has useful parser work, but the latest diff still regresses the existing lowercase `media:` directive contract and lacks required after-fix real behavior proof.

## What This Changes

The PR adds indented-code detection to `splitMediaFromOutput`, changes markdown fence closing validation, removes case-insensitive MEDIA matching, and adds media/fence parser tests.

## Best Possible Solution

Land a focused parser fix that skips indented code blocks and preserves code indentation while keeping lowercase MEDIA support consistent across parser, block-reply, streaming, and tests.

## Reproduction Assessment

Yes, source-level. Current main trims indentation before the `MEDIA:` line gate and only checks fenced spans, so a 4-space or tab-indented code line can be extracted as media outside fences.

## Solution Assessment

No, not as submitted. The indented-code guard is a reasonable narrow fix, but changing lowercase directive semantics is not the best solution without an explicit contract change and aligned streaming/reply updates.
