Keep this PR open but block merge for now. The reported MEDIA false-positive is real on current main, and the indented-code/fence direction is useful, but the latest head still changes the existing lowercase MEDIA contract and leaves the post-audio-tag cleanup path able to strip tab-indented code.

The PR adds indented-code guards for MEDIA extraction, stricter markdown fence close validation, code-aware cleanup, a MEDIA regex case-sensitivity change, and parser/fence regression tests.

## Best Possible Solution

Land a focused parser fix that skips MEDIA extraction inside indented code blocks and invalidly closed fences while preserving lowercase MEDIA compatibility, reusing one code-aware cleanup path after audio-tag stripping, and adding the required changelog entry.
