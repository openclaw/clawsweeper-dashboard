## Summary

Keep open: current main still has the indented-code MEDIA false-positive, but this PR head introduces parser compatibility regressions and still lacks required real behavior proof.

## What This Changes

The PR updates MEDIA and fence parsing to skip MEDIA extraction from indented code blocks, preserve retained code indentation, tighten closing-fence validation, change MEDIA matching to case-sensitive, and add parser/fence tests.

## Best Possible Solution

Land a focused parser fix that skips indented-code MEDIA extraction while preserving existing lowercase/streaming semantics and code-aware cleanup after audio tags, with targeted regression tests and redacted real behavior proof.

## Reproduction Assessment

Yes. Source inspection on current main shows `splitMediaFromOutput` excludes fenced spans only, then uses a case-insensitive `trimStart()` MEDIA prefix check, giving a concrete path for indented `MEDIA:` false extraction.

## Solution Assessment

No. The indented-code guard belongs in `splitMediaFromOutput`, but this PR is not yet the best fix because it changes established lowercase MEDIA behavior and leaves streaming/audio cleanup paths misaligned.

## Review Findings

- **[P2] Keep lowercase MEDIA directives working:** `src/media/parse.ts:17`
  - body: Current main has block-reply coverage expecting `media: ./report.pdf` to populate `mediaUrl` and `mediaUrls`, and the streaming accumulator at this head still treats MEDIA tails case-insensitively.   - confidence: 0.93

  - body: The PR computes `firstLineIsCode` before `parseAudioTag`, but the audio-tag cleanup always applies the non-code leading-normalization regex. For `[[audio_as_voice]]` followed by a tab-indented code line, the leading tab is stripped and the code sample is reformatted as top-level text.
  - confidence: 0.88
- **[P2] Align streaming buffering with indented-code parsing:** `src/media/parse.ts:563`
  - body: After this change, `splitMediaFromOutput` preserves 4-space/tab indented MEDIA lines as text, but `splitTrailingDirective` still buffers any whitespace-prefixed `MEDIA:` tail and `reset()` drops `pendingTail`. A streamed indented code example ending at a chunk boundary can therefore disappear even though the parser no longer considers it a directive.
  - confidence: 0.82
