## Summary

Keep open: the PR targets a real current-main media parsing bug, but the latest head still introduces parser compatibility regressions and lacks after-fix real behavior proof.

## What This Changes

The PR changes media and fence parsing to ignore `MEDIA:` directives in indented code blocks, tighten closing fence validation, alter MEDIA case matching, and add parser tests.

## Best Possible Solution

Land a focused parser update that preserves existing lowercase semantics or migrates all callers/tests deliberately, mirrors the indented-code rule in streaming buffering, preserves tab indentation after audio tag stripping, and includes redacted real behavior proof.

## Reproduction Assessment

Yes. Source inspection on current main shows `splitMediaFromOutput` excludes fenced spans only, then trims indentation before matching `MEDIA:`, which gives a high-confidence path for false extraction from indented code blocks.

## Solution Assessment

No. The indented-code guard is the right general area, but this PR is not yet the best fix because it changes lowercase directive compatibility, leaves streaming buffering on the old rules, and mishandles tab-indented code after audio tag stripping.
