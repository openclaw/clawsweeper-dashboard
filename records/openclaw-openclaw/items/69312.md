## Summary

## What This Changes

The PR updates media directive parsing to skip 4-space/tab indented code blocks, preserve retained code indentation, tighten markdown fence close validation, and add parser/fence regression tests.

## Best Possible Solution

Land a focused parser fix that preserves code-aware normalization after audio tag stripping, keeps lowercase media compatibility, and includes redacted real OpenClaw behavior proof.

## Reproduction Assessment

Yes. Source inspection on current main shows `splitMediaFromOutput` only skips fenced spans before `line.trimStart().toUpperCase().startsWith("MEDIA:")`, so a blank-line-separated 4-space or tab-indented `MEDIA:` line has a clear failing path.

## Solution Assessment

Mostly yes, but not as currently written. The indented-code guard is the right narrow fix, while the audio-tag cleanup needs the same code-aware leading-normalization decision and the proof gate still needs real runtime evidence.
