## Summary

Keep open: current main still has the indented-code MEDIA false-positive, but the latest PR head is not merge-ready because it breaks existing lowercase MEDIA compatibility and still strips indentation after audio-tag cleanup.

## What This Changes

The PR updates media and fence parsing plus tests to skip indented-code MEDIA lines, preserve retained code indentation during media cleanup, tighten fence closer validation, and make MEDIA matching case-sensitive.

## Best Possible Solution

Land a focused parser fix that preserves indented-code MEDIA examples while retaining lowercase MEDIA compatibility across block and streaming paths, reuses code-aware cleanup after audio-tag stripping, and includes an active-version changelog Fixes entry.

## Reproduction Assessment

Yes. Current main can be reproduced by tracing `splitMediaFromOutput` on an output containing a blank line followed by a 4-space or tab-indented `MEDIA:` line; current tests also give a high-confidence lowercase compatibility check.

## Solution Assessment

No, not as submitted. The indented-code guard is the right direction, but the case-semantics and post-audio cleanup regressions should be fixed before merge.
