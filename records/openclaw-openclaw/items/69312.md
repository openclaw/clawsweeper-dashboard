- **[P2] Keep streaming MEDIA classification in sync:** `src/media/parse.ts:563`
  - body: `splitTrailingDirective` still buffers any whitespace-prefixed `MEDIA:` line, but this parser now treats blank/fence-preceded 4-space or tab-indented MEDIA lines as code text.   - confidence: 0.86

That changes visible code formatting and can expose the line as a top-level directive if text is parsed again, so the post-audio normalization should be code-aware too.
  - confidence: 0.82
