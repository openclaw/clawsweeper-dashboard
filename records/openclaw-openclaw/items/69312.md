## Summary

Keep open. The bug is valid and the PR is directionally useful, but the latest head still has merge-blocking correctness issues around lowercase MEDIA directives, streaming buffering, and audio-tag cleanup, and the external PR still lacks after-fix real behavior proof.

## What This Changes

The PR updates media and markdown fence parsing to skip MEDIA extraction in indented code blocks, preserve code indentation during cleanup, validate fence closers more strictly, and add parser/fence tests.

## Best Possible Solution

Land a focused parser fix that protects indented code blocks without changing the lowercase directive contract, keeps streaming and audio cleanup aligned with the same rules, and includes redacted real CLI or Telegram proof.

## Reproduction Assessment

Yes, source-level. Current main trims indentation before a case-insensitive MEDIA gate and only skips fenced spans, so indented code-block directives can be extracted; the PR regressions are also visible from the latest diff and existing tests.

## Solution Assessment

No, not as submitted. The indented-code guard is the right direction, but the patch should preserve lowercase directive support and reuse the same code-aware rules in streaming and audio-tag cleanup.
