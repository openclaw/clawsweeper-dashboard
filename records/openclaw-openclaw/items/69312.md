## Summary

Keep this PR open. The reported false-positive is valid, but the current head still conflicts with existing lowercase MEDIA behavior, can strip code indentation after audio tag removal, and lacks contributor real behavior proof.

## What This Changes

This PR changes media and fence parsing to preserve MEDIA tokens inside indented code blocks or invalid fence closures, removes lowercase MEDIA extraction, and adds parser/fence tests.

## Best Possible Solution

Land a focused parser fix that protects CommonMark indented code from MEDIA extraction while preserving the current lowercase MEDIA and streaming behavior, with regression tests and redacted real behavior proof.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: splitMediaFromOutput only excludes fenced spans and then trim-starts indented lines before a case-insensitive MEDIA gate, so indented code can be extracted.

## Solution Assessment

No. The indented-code guard is in the right area, but the current PR also changes established lowercase MEDIA behavior and can strip code indentation after audio tag removal; the safer fix preserves those contracts.
