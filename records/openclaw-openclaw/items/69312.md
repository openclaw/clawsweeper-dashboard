## Summary

Keep open. The central indented-code fix is valid, but the current patch regresses the existing lowercase media-directive contract and still lacks required real behavior proof before merge.

## What This Changes

The PR updates shared media and fence parsing to skip 4-space/tab indented `MEDIA:` code examples, preserve retained code indentation during cleanup, reject invalid fence closers with trailing text, make `MEDIA:` matching case-sensitive, and add media/fence tests.

## Best Possible Solution

Land a focused parser fix that skips indented-code `MEDIA:` examples while preserving existing lowercase directive compatibility across `splitMediaFromOutput`, block-reply normalization, and streaming buffering, with after-fix real behavior proof.

## Reproduction Assessment

Yes. Source inspection shows current main only protects fenced spans, then trims indentation before matching `MEDIA:`, so the linked indented-code false positive is a high-confidence source reproduction; I did not run tests because this review is read-only.

## Solution Assessment

No. The indented-code protection is the right direction, but the case-sensitivity cleanup is not the narrowest maintainable fix because it breaks current lowercase media behavior and leaves adjacent streaming/block-reply contracts inconsistent.
