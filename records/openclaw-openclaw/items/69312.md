## Summary

Keep open: this is an active external bug-fix PR for a real media parser bug, but the latest head still conflicts with established lowercase media parsing, leaves streaming/audio cleanup mismatched with the new parser rules, and lacks required real behavior proof.

## What This Changes

The branch adds indented-code detection to media parsing, code-aware cleanup, stricter markdown fence closing validation, and parser tests.

## Best Possible Solution

Land a focused parser fix that preserves indented-code MEDIA text while maintaining lowercase MEDIA compatibility and sharing one code-aware cleanup contract across streaming and audio paths.

## Reproduction Assessment

Yes, source-level. Current main only skips fenced spans, then trims indentation before a case-insensitive MEDIA gate, so an indented code-block MEDIA line can be extracted as media instead of preserved as text.

## Solution Assessment

No, not as submitted. The indented-code guard is the right direction, but the branch changes existing lowercase MEDIA behavior and does not align streaming buffering or audio-tag cleanup with the new code-line rules.
