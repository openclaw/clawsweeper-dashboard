- **[P2] Fix the lowercase MEDIA regression test:** `src/media/parse.test.ts:421-425`
  - body: This new assertion expects `media: note MEDIA:/tmp/a.png` to extract media, but the restored `/gi` regex greedily matches from the lowercase prefix through the end of the line as one payload (`note MEDIA:/tmp/a.png`). That payload is not a valid media path, so `mediaUrls` stays undefined and the added test should fail unless the parser behavior is intentionally changed.
  - confidence: 0.93
- **[P2] Keep streaming buffering in sync with code blocks:** `src/media/parse.ts:562-563`
  - body: This new code-block skip makes blank-line 4-space/tab `MEDIA:` lines visible text, but the streaming accumulator still buffers any `/^\s*MEDIA:/i` last line. A chunk ending with `\n\n    MEDIA:/tmp/code` can sit in `pendingTail` and then be dropped by `reset()` before a final flush, so the same code example can still disappear in streamed replies.
  - confidence: 0.86
- **[P2] Reuse code-aware cleanup after audio tags:** `src/media/parse.ts:745-748`
  - body: After `parseAudioTag()`, this branch always applies the non-code leading normalizer. For `[[audio_as_voice]]\n\n\tMEDIA:/tmp/code.png`, the tab-indented line was retained as code, but tag removal makes it the first visible line and this regex strips the tab, changing the visible code text.
  - confidence: 0.88
