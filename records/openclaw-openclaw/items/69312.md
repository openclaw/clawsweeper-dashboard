## Summary

## What This Changes

The branch updates shared MEDIA parsing and fence parsing to skip indented-code MEDIA-looking lines, preserve code indentation during cleanup, reject malformed fence closers, and add parser regression tests.

## Best Possible Solution

Land a focused parser fix that preserves the existing lowercase MEDIA contract, aligns streaming and audio cleanup with the new indented-code rules, validates shared fence consumers, and includes redacted CLI or Telegram proof.

## Reproduction Assessment

Yes, source-level. Current main only skips fenced spans before the case-insensitive MEDIA line gate, so an indented MEDIA-looking code line can be parsed as an attachment without needing a new feature or config option.

## Solution Assessment

No, not as submitted. The indented-code guard is the right narrow direction, but the branch should preserve the lowercase MEDIA contract and keep streaming plus post-audio cleanup paths in sync before merge.
