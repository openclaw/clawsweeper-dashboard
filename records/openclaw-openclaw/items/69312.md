## Summary

## What This Changes

The PR changes media directive parsing to preserve 4-space/tab indented code, adds code-aware cleanup, tightens fence close parsing, and adds media/fence regression tests.

## Best Possible Solution

Finish the parser fix in this PR by sharing the code-aware normalization after audio-tag stripping, aligning streaming directive buffering with the new indented-code contract, and landing with focused regression coverage plus redacted real behavior proof.

## Reproduction Assessment

Yes. By source inspection, current main trims indentation before the `MEDIA:` check and only protects fenced spans, so the linked blank-line plus 4-space/tab-indented `MEDIA:` input has a clear failing path.

## Solution Assessment

No, not as currently written. The indented-code guard is the right narrow fix, but the PR should reuse code-aware cleanup after `[[audio_as_voice]]`, align streaming buffering, and provide real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Reuse code-aware normalization after audio tags:** `src/media/parse.ts:747-748`
  - body: The main cleanup path preserves a first surviving code line, but the `parseAudioTag` branch still runs the non-code leading-normalization regex unconditionally. With `[[audio_as_voice]]\n\n\tMEDIA:/tmp/code.png`, the loop keeps the tab-indented line as code, then this cleanup strips the tab and returns it as unindented visible text. Reuse `leadingNormRegex` here or add an equivalent code-aware branch, plus a tab-indented audio-tag regression test.
  - confidence: 0.86
- **[P2] Align streaming buffering with the new code guard:** `src/media/parse.ts:562-563`
  - body: This PR makes blank-line-separated 4-space/tab `MEDIA:` lines plain text, but the streaming accumulator still buffers any trailing line matching optional whitespace plus `MEDIA:`. A streamed indented code example that ends a chunk can still be held in `pendingTail` and lost on a reset without final flush, which is the loss case the existing comment calls out. Update the streaming guard and tests to match the new indented-code classification.
  - confidence: 0.75
