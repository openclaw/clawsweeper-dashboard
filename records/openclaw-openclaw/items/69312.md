This PR changes media and fence parsing to preserve indented-code `MEDIA:` examples, protect retained code indentation, tighten fence closing validation, and add parser tests.

## Best Possible Solution

Land a focused version that keeps indented-code MEDIA examples intact, preserves or deliberately migrates lowercase MEDIA semantics across all consumers, reuses code-aware cleanup after audio tag stripping, adds the active changelog entry, and validates shared fence consumers.

## Solution Assessment
