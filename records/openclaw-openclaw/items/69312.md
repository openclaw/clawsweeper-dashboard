## Summary

Keep open: the underlying false-positive MEDIA extraction bug is valid, but this PR still regresses established lowercase MEDIA directive behavior, leaves streaming/audio cleanup paths inconsistent with the new parser rules, and lacks required after-fix real behavior proof.

## What This Changes

The PR updates shared MEDIA/fence parsing to skip indented code-block MEDIA-looking lines, tighten fence close detection, adjust cleanup, and add parser/fence tests.

## Best Possible Solution

Land a focused parser fix that preserves lowercase MEDIA behavior, aligns streaming and audio cleanup with the code-aware parser rules, and includes redacted CLI or Telegram proof.

## Reproduction Assessment

Yes, source-level. Current main only excludes fenced spans before a case-insensitive MEDIA line gate, so a blank-preceded four-space or tab-indented MEDIA-looking code line can be parsed as an attachment.

## Solution Assessment

No, not as submitted. The indented-code guard is the right direction, but the patch should preserve lowercase directive behavior and keep streaming/audio cleanup semantics aligned with the parser.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve lowercase MEDIA directive support:** `src/media/parse.ts:17`
  - body: Current main has block-reply coverage for `media: ./report.pdf`, and commit f350bb4dfc62 introduced case-insensitive MEDIA parsing. Removing the `i` flag and switching the line gate to `startsWith("MEDIA:")` makes lowercase directives visible text instead of attachment metadata.
  - confidence: 0.94

  - confidence: 0.88
- **[P2] Reuse code-aware cleanup after audio tags:** `src/media/parse.ts:745-748`
  - body: The main cleanup picks a code-aware leading regex, but after `parseAudioTag` this branch always applies the non-code regex. With `[[audio_as_voice]]` before a tab-indented code line, the first code line can lose its tab indentation even though the parser kept it as code.
  - confidence: 0.86
