## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Keep streaming indented-code state across lines:** `src/auto-reply/reply/streaming-directives.ts:81-83`
  - body: The new guard only skips buffering when the indented MEDIA-looking last line is at start-of-text or after a blank line. In a multi-line indented code block such as `text\n\n    example\n    MEDIA:/tmp/code.png`, the final parser treats the MEDIA line as code because the previous line was indented code, but streaming still buffers it; if the stream resets before final flush, that visible code line is dropped. Track the previous indented-code line too, or share the parser state enough for this guard to match final parsing.
  - confidence: 0.88
