## Summary

Keep open: the reported parser bug is valid on current main, but the PR still regresses existing lowercase `media:` directive compatibility and lacks contributor real behavior proof.

## What This Changes

The PR updates media and fence parsing so `MEDIA:` lines in indented code blocks are preserved instead of extracted, adds fence-closing validation, and adds media/fence regression tests.

## Best Possible Solution

Keep the indented-code protection, but preserve existing lowercase `media:` behavior consistently across the media parser, block-reply auto detection, streaming buffering, and tests before merging.

## Reproduction Assessment

Yes, by source inspection: current main skips only fenced spans, then strips indentation before checking for a `MEDIA:` prefix, so an indented code-block line can be extracted as media. I did not run tests because this review is read-only and must not create artifacts.

## Solution Assessment

No. The indented-code guard is the right surface, but the case-sensitivity change is not the narrowest maintainable fix because current tests and streaming/block-reply code still define lowercase `media:` as supported.
