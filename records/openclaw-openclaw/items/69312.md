## Summary

Keep open: the reported indented-code MEDIA false positive is valid, but the latest PR still regresses the existing lowercase MEDIA directive contract and has no after-fix real behavior proof from the contributor.

## What This Changes

This PR adds indented-code handling to `splitMediaFromOutput`, tightens markdown fence closing validation, removes case-insensitive MEDIA parsing, and adds media/fence parser tests.

## Best Possible Solution

Land a focused parser fix that skips indented code blocks and preserves indentation while keeping lowercase MEDIA support consistent across parser, reply delivery, streaming, and tests.

## Reproduction Assessment

Yes, source-level. Current main trims indentation before the MEDIA line gate and only checks fenced spans, so a 4-space or tab-indented code line can enter extraction outside fences.

## Solution Assessment

No, not as submitted. The indented-code guard is a reasonable narrow fix, but removing lowercase MEDIA support conflicts with current parser, reply-delivery, and streaming behavior.
