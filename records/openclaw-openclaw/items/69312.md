## Summary

Keep open: this PR targets a real source-reproducible media parser bug, but the latest diff still regresses existing lowercase media directive support and lacks after-fix real behavior proof.

## What This Changes

This PR adds indented-code detection to `splitMediaFromOutput`, tightens markdown fence closing validation, changes MEDIA matching to case-sensitive, and adds media/fence parser tests.

## Best Possible Solution

Land a focused parser fix that skips MEDIA extraction for indented code blocks while preserving current lowercase MEDIA support and validating shared fence consumers if fence parsing changes remain.

## Reproduction Assessment

Yes, source-level. Current main trims indentation before the `MEDIA:` gate and only checks fenced spans, so an indented code-block `MEDIA:` line can be parsed and stripped as media.

## Solution Assessment

No, not as submitted. The indented-code guard is a reasonable narrow fix, but changing lowercase MEDIA semantics conflicts with existing parser/reply/streaming behavior.
