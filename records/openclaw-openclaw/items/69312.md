## Summary

Keep open: this PR targets a real MEDIA parser bug, but the latest diff still changes established lowercase MEDIA behavior, leaves final parsing misaligned with streaming/audio cleanup paths, and lacks required real behavior proof before merge.

## What This Changes

The PR changes MEDIA and fence parsing to skip MEDIA directives in indented code blocks, validate fence closing lines more strictly, adjust MEDIA case matching, and add parser/fence regression tests.

## Best Possible Solution

Land a focused parser fix that preserves existing directive contracts, or updates them deliberately across final and streaming paths, with code-aware cleanup, changelog coverage, targeted tests, and real setup proof.

## Reproduction Assessment

Yes, by source inspection. Current main skips only fenced spans and then treats trim-started, case-normalized MEDIA lines as directives, so blank-preceded 4-space or tab-indented code examples can be parsed as media.

## Solution Assessment

No, not as submitted. The indented-code guard is the right direction, but the patch should keep lowercase MEDIA behavior consistent, align streaming buffering with final parsing, reuse code-aware cleanup after audio tags, and add the required changelog/proof.
