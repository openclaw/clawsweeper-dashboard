## Summary

Keep open: the reported bug is source-reproducible on current main, but this PR still has blocking parser/streaming compatibility regressions and lacks required after-fix real behavior proof.

## What This Changes

The PR changes media and fence parsing to skip MEDIA extraction from indented code blocks, validate fence closers more strictly, alter MEDIA case matching, and add parser tests.

## Best Possible Solution

Land a focused parser update that preserves lowercase directive compatibility, aligns streaming tail buffering with the same indented-code rules, reuses code-aware cleanup after audio tags, and includes tests plus redacted real behavior proof.

## Reproduction Assessment

Yes. Source inspection shows current main only skips fenced spans, then case-insensitively treats indented MEDIA-prefixed lines as directives, so the false-positive extraction path is clear without running tests.

## Solution Assessment

No. The indented-code guard is the right area, but this patch is not yet the best solution because it regresses lowercase MEDIA parsing, leaves streaming buffering on the old directive rule, and strips tab indentation after audio tag removal.

  - confidence: 0.86
