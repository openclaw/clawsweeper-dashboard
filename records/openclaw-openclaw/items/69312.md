## Summary

Keep open: the underlying parser bug is valid on current main, but the patch still regresses existing lowercase MEDIA directive behavior and lacks external real behavior proof before merge.

## What This Changes

The PR changes media and markdown fence parsing to skip MEDIA extraction from indented code blocks, preserve retained code indentation during cleanup, make MEDIA matching case-sensitive, and add parser tests.

## Best Possible Solution

Land a focused version that prevents MEDIA extraction from indented code blocks while preserving existing lowercase MEDIA semantics and validating shared fence parser consumers.

## Reproduction Assessment

Yes, source-level. Current main only excludes fenced spans before checking `line.trimStart()` for `MEDIA:`, so a 4-space or tab-indented line can be stripped as media outside a fenced block.

## Solution Assessment

No, not as submitted. The indented-code guard is a reasonable narrow fix, but changing lowercase MEDIA semantics is not the best solution unless all parser, reply-delivery, streaming, and tests are deliberately aligned.
