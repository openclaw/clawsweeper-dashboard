## Summary

Keep open: the reported parser bug is real and the parser is the right surface, but the latest diff still regresses existing lowercase MEDIA handling, leaves streaming buffering out of sync with code-aware parsing, can strip tab indentation after audio tags, and lacks after-fix real behavior proof.

## What This Changes

The PR changes MEDIA/fence parsing to skip MEDIA extraction from indented code blocks, tighten fence closing validation, adjust cleanup around code lines/audio tags, and add parser/fence tests.

## Best Possible Solution

Land a focused parser fix that skips CommonMark indented code blocks while preserving existing lowercase, streaming, and audio cleanup semantics, with focused regressions and real behavior proof.

## Reproduction Assessment

Yes, source-level: current main excludes fenced spans but not CommonMark indented code blocks before the trimmed `MEDIA:` gate, so the reported false extraction path is clear. I did not run tests because this review is read-only.

## Solution Assessment

No. The parser is the narrow maintainable place to fix this, but this branch should preserve current lowercase directive behavior and keep streaming/audio cleanup aligned with the new code-aware parsing rules before merge.
