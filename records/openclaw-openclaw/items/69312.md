## What This Changes

The PR changes media directive parsing to skip indented-code `MEDIA:` lines, tightens markdown fence close detection, preserves some code indentation during cleanup, and adds media/fence parser tests.

## Best Possible Solution

Land a repaired parser/fence patch that protects indented code and strict fences while preserving existing media directive compatibility and reply formatting.

## Reproduction Assessment

Yes, source-reproducible. Current main trims indentation before its case-insensitive `MEDIA:` line gate and only excludes fenced spans, so an indented code-block directive can still be parsed as media.

## Solution Assessment

No, not as submitted. The indented-code and strict-fence direction is narrow, but the patch should preserve lowercase directive compatibility, reuse code-aware cleanup after audio tag stripping, and add changelog coverage.
