## Summary

Keep open: the reported ellipses match current secret-redaction behavior for signed URL query parameters and Alibaba `LTAI` access-key IDs, but changing that behavior requires a maintainer security/product decision rather than conservative cleanup.

## What This Changes

The issue asks OpenClaw to display full JSON URL strings in WebUI and TUI instead of replacing omitted parts with ellipses.

## Best Possible Solution

Decide whether signed media URLs should stay redacted with clearer guidance, be represented as managed image attachments, or get a deliberately scoped trusted-local reveal/copy path that does not weaken transcript redaction broadly.

## Reproduction Assessment

Yes, at source level: a URL containing `OSSAccessKeyId=LTAI...` and `Signature=...` will match the default redaction patterns, and persisted transcript text is redacted before WebUI/TUI history renders it. I did not run a live Gateway/WebUI/TUI reproduction.

## Solution Assessment

Unclear: showing the raw URL everywhere is not clearly the best fix because these signed URLs contain credential-like material. A safer solution would preserve image usability through managed media or a tightly scoped reveal/copy design after maintainer security review.
