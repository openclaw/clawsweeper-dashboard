## Summary

Keep open: current main still allows isolated agentTurn cron jobs to omit both lightContext and toolsAllow, which leaves the costly/full-capability path available; the right fix needs maintainer direction because changing defaults, adding workspace config, or requiring allowlists is compatibility- and security-sensitive.

## What This Changes

The issue requests safer isolated cron defaults, workspace/UI controls for lightContext and toolsAllow, warnings for expensive runs, and documentation of the current cost behavior.

## Best Possible Solution

Define a maintainer-approved safe isolated-cron policy that makes costly full-context/all-tools runs explicit, adds UI/API/docs warnings or controls, and preserves a clear path for jobs that deliberately need full capability.

## Reproduction Assessment

No live high-confidence token-cost reproduction was run in this read-only review. Source inspection does reproduce the important default path: omitted lightContext and toolsAllow remain omitted through UI/CLI/API creation and reach runtime as full-context or unrestricted behavior.

## Solution Assessment

Unclear as a final fix because changing platform defaults, requiring toolsAllow, adding workspace defaults, and adding warnings are all viable but have different upgrade consequences. The narrowest maintainable path likely starts with maintainer-selected defaults/guardrails plus docs and Control UI alignment.
