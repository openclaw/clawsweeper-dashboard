## Summary

Keep open: the capability-derived Telegram DM topic behavior is useful, but this branch turns it into an auto-only migration that deletes explicit overrides and changes startup bot-info caching, so it needs maintainer product review and stronger live Telegram proof before merge.

## Best Possible Solution

Land a version that uses `getMe.has_topics_enabled` as the default auto behavior while preserving explicit overrides and cached bot-info fallback, with live Telegram DM-topic proof before merge.
