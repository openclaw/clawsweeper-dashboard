## Summary

Keep open. The protected `beta-blocker` label requires explicit maintainer handling, and current `main` still contains the reported Matrix fallback: setup calls `ensureMatrixSdkInstalled`, which derives a package-manager cwd by walking two parents from `import.meta.url` and then runs `npm` or `pnpm` there. Related work such as https://github.com/openclaw/openclaw/issues/80401 and https://github.com/openclaw/openclaw/pull/77988 addresses adjacent Matrix packaging/externalization problems, but it does not remove this residual runtime installer path.

## What This Changes

The issue requests that Matrix runtime dependency repair stop running package-manager installs from parent directories derived from the compiled file path.

## Best Possible Solution

Remove Matrix runtime package-manager mutation, or route any retained repair through the existing explicit plugin install/update/doctor path with a validated OpenClaw-owned npm root and regression coverage for the external `@openclaw/matrix` layout.

## Reproduction Assessment

Yes, at source level: current `main` still reaches a Matrix setup fallback that computes cwd by fixed parent traversal and passes it to a package-manager install. I did not run the destructive npm mutation path during this read-only review.

## Solution Assessment

Yes, the requested direction is the narrow maintainable fix: runtime/setup should not run npm from a path-derived parent directory, and any repair should use the central managed plugin install or doctor flow.
