## Summary

Keep open. Current main still lets unset IRC `blockStreaming` fall through to the global block-streaming default, while the PR is a narrow IRC-owned fix with focused regression coverage and no blocking review or security findings found.

## What This Changes

The PR changes IRC inbound reply options so unset or false IRC `blockStreaming` disables block streaming while explicit true opts in, and adds policy tests for those config states.

## Best Possible Solution

Land or port the IRC-owned default helper and regression tests, preserving explicit per-account opt-in with `blockStreaming: true` and avoiding shared resolver changes for other channels unless separately scoped.

## Reproduction Assessment

Yes. With `agents.defaults.blockStreamingDefault` set to `on` and IRC `blockStreaming` unset, current main passes `disableBlockStreaming: undefined`, so the shared resolver enables block streaming; #42121 provides the user-visible IRC cut-off symptom.

## Solution Assessment

Yes. The PR keeps the behavior change inside the IRC plugin boundary, preserves the existing true opt-in behavior, and adds focused coverage for the three relevant config states.
