## Summary

## What This Changes

The PR changes IRC inbound reply options so block streaming is disabled unless the IRC account explicitly opts in with `blockStreaming: true`, and adds regression coverage for the default.

## Best Possible Solution

Refresh the IRC-local resolver onto current main, put regression coverage in `extensions/irc/src/inbound.behavior.test.ts`, and merge after redacted real IRC or terminal/log proof confirms full final replies.

## Reproduction Assessment

Yes, at source level. Current main passes `disableBlockStreaming: undefined` for unset IRC config, and the shared resolver then inherits `agents.defaults.blockStreamingDefault` despite the docs requiring explicit channel opt-in.

## Solution Assessment

Yes functionally. A channel-local resolver that sends a concrete disable boolean unless IRC explicitly sets `blockStreaming: true` is the narrow maintainable fix; the branch still needs refresh and real behavior proof.
