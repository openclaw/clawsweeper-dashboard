## Summary

Keep open. Current main and the latest release still have the LiteLLM provider-docs gap: the manual setup docs remain unpinned and there is no incident/remediation text, while the plugin code confirms OpenClaw talks to a user-hosted LiteLLM proxy and does not bundle the PyPI package. Because this is third-party supply-chain remediation guidance, it needs maintainer/security review before publishing a pin or upgrade recommendation.

## What This Changes

Request a LiteLLM provider documentation advisory that pins or qualifies the proxy install command and adds remediation guidance for reported malicious LiteLLM PyPI versions 1.82.7 and 1.82.8.

## Best Possible Solution

Land a maintainer/security-reviewed docs-only update to the LiteLLM provider page that replaces or qualifies the unpinned proxy install example and adds a dated incident notice with affected-version checks, credential-rotation guidance, reinstall/downgrade/upgrade advice, and host-audit steps. Keep OpenClaw runtime code unchanged unless that review finds a separate dependency or proxy-safety issue.

## Reproduction Assessment

Not applicable as a runtime bug; this is a docs/security guidance request. The docs gap is high-confidence and directly reproducible by reading `docs/providers/litellm.md:50` on current main and searching for the affected-version/advisory terms.

## Solution Assessment

No current solution on main. The narrow maintainable solution is a provider-doc advisory after maintainer/security confirmation of the current LiteLLM/PyPI remediation; a core dependency fix is not indicated because the current plugin is network/proxy based and does not bundle LiteLLM.
