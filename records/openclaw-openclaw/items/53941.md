## What This Changes

The issue asks OpenClaw to update the LiteLLM provider docs with a supply-chain advisory, safer install guidance, and remediation steps for users who ran compromised LiteLLM proxy versions.

## Best Possible Solution

Publish a docs-only, maintainer/security-reviewed LiteLLM provider guide update that warns about 1.82.7/1.82.8, links authoritative upstream advisory material, gives key-rotation/audit remediation for affected proxy hosts, and uses approved current safe-install wording without changing runtime integration behavior.

## Reproduction Assessment

## Solution Assessment
