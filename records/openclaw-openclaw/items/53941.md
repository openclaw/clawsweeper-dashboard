## Summary

Keep open. Current main and the latest release still have the LiteLLM provider docs gap: the manual setup page recommends an unpinned PyPI install and has no advisory/remediation text, while the OpenClaw plugin itself remains a proxy/network integration rather than a bundled Python dependency. The issue is valid, but the exact public remediation wording should be confirmed by maintainer/secops because upstream guidance has moved beyond the original v1.82.6-only recommendation.

## What This Changes

The issue requests a LiteLLM provider docs update to pin or qualify the manual PyPI install and add remediation guidance for users affected by compromised LiteLLM versions 1.82.7 and 1.82.8.

## Best Possible Solution

Land a docs-only, maintainer/secops-reviewed update to `docs/providers/litellm.md` that replaces or qualifies the unpinned PyPI install example, links the upstream OSV/LiteLLM/PyPI incident sources, and gives dated affected-version checks, credential rotation, host audit, cleanup, and safe install guidance. Keep runtime code unchanged unless that review finds a separate OpenClaw dependency or proxy-safety issue.
