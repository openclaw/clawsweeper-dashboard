## Summary

Keep this open. Current main still has the reported LiteLLM docs gap: the manual setup example installs the LiteLLM proxy without a version constraint, and the searched docs/source surfaces still do not contain advisory or remediation text for the reported compromised LiteLLM versions. The current bundled integration is proxy/API-key based rather than a bundled PyPI dependency, so this remains a docs/security guidance follow-up rather than a runtime dependency fix.

## What This Changes

Update the LiteLLM provider documentation to avoid or qualify an unpinned proxy install and add remediation guidance for users potentially affected by compromised LiteLLM v1.82.7/v1.82.8 packages.

## Best Possible Solution

Land a maintainer/security-reviewed docs-only update to the public LiteLLM provider page that replaces or qualifies the unpinned install example and adds a dated incident notice with affected-version checks, cleanup/reinstall guidance, credential rotation, and host-audit steps. Keep the runtime/plugin implementation unchanged unless that review identifies a separate dependency or proxy-safety issue.
