## Summary

Keep this PR open, but do not merge as-is: it fixes the reported agent-side rejection by expanding the agent-facing Gateway tool allowlist to a raw-conversation plugin trust gate, which conflicts with the current model-to-operator config mutation boundary.

## What This Changes

The PR adds `plugins.entries.*.hooks.allowConversationAccess` to the agent Gateway tool config mutation allowlist and adds guard-coverage tests for setting and toggling that field through `config.patch`.

## Best Possible Solution

Preserve or restore local plugin hook policy through the upgrade/doctor/config-management path, or require explicit maintainer approval before making this security gate agent-mutable.
