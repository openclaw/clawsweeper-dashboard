## Summary

Keep open. The reported symptom is real, but this patch changes the agent-facing Gateway config mutation boundary for a raw-conversation plugin hook gate, so it needs maintainer security/product judgment or a safer upgrade/config-preservation fix.

## What This Changes

The PR adds `plugins.entries.*.hooks.allowConversationAccess` to the agent Gateway config mutation allowlist and adds guard tests for setting and toggling it through `config.patch`.

## Best Possible Solution

Preserve or restore local plugin hook policy through upgrade, doctor, or explicit human-operated config management, and avoid making raw conversation access agent-mutable unless maintainers approve that trust boundary.

## Reproduction Assessment

Yes. By source inspection, current main lacks this path in the agent Gateway allowlist, so a `config.patch` changing `plugins.entries.<id>.hooks.allowConversationAccess` reaches the protected-path rejection before the RPC write is forwarded.

## Solution Assessment

No. The PR addresses one symptom, but it changes a model-to-operator security boundary; the safer direction is to fix config preservation or require explicit human approval for this trust gate.
