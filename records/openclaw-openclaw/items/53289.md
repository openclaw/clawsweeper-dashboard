## Summary

Keep open. The requested clarification is still useful because current main does not explain that runtime tool identifiers are not shell commands, but this PR should not merge as-is: it documents an invalid `openclaw sessions list` command and repeats the built-in-tools intro sentence.

## What This Changes

This PR adds a `docs/tools/index.md` callout distinguishing OpenClaw runtime tool identifiers from terminal commands, with sessions and agents examples plus help-discovery guidance.

## Best Possible Solution

Land a narrow docs update in `docs/tools/index.md` that preserves the useful tool-vs-shell clarification, uses `openclaw sessions` and `openclaw agents list` as examples, points to `openclaw --help` and per-command help, and keeps the built-in-tools table intro only once.

## Reproduction Assessment

Yes. The review can be reproduced by comparing the PR's new examples with current main: `docs/cli/sessions.md` and `register.status-health-sessions.ts` show `openclaw sessions` is the listing command, while `build-program.ts` already treats `openclaw sessions list` as an error-path example.

## Solution Assessment

No. The docs direction is right, but this exact patch is not the best solution until the sessions example is corrected and the duplicated intro sentence is removed.
