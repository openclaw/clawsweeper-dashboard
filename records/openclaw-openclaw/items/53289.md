## Summary

Current main and shipped docs already clarify that CLI commands are invoked through the `openclaw` entry point, while this PR branch adds duplicate prose and an invalid `openclaw sessions list` example.

## What This Changes

This PR adds a `docs/tools/index.md` callout explaining that tool identifiers are not shell commands and gives `openclaw sessions list` and `openclaw agents list` examples.

## Best Possible Solution

Keep the shipped CLI reference as the canonical clarification; any future tools-page note should use `openclaw sessions` and avoid duplicating the built-in-tools introduction.

## Reproduction Assessment

Yes for the review findings. Source inspection shows current docs and CLI registration use `openclaw sessions`, while the PR documents an unregistered `openclaw sessions list` form and repeats existing prose.

## Solution Assessment

No for merging this branch. The current shipped CLI docs are the narrower and more accurate solution, and this PR would regress docs quality if merged as-is.
