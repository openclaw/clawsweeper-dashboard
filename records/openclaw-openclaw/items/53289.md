## What This Changes

## Best Possible Solution

Keep the shipped CLI reference as the canonical fix; any future tools-page note should be a fresh docs edit that uses `openclaw sessions` and `openclaw agents list`.

## Reproduction Assessment

Yes. Source inspection shows the PR documents `openclaw sessions list`, while current CLI docs and registration expose `openclaw sessions` as the session listing command and `cleanup` as the nested sessions command.

## Solution Assessment
