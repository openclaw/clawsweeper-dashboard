## Summary

Keep open: this is an external, security-sensitive PR with the protected `maintainer` label; current main still has source-visible persistence gaps, but the PR head still needs maintainer review, real behavior proof, and fixes for auth-storage migration/key-derivation concerns.

## What This Changes

The PR adds structured secret redaction to persisted audit, log, trajectory, and transcript sinks, expands token-shape patterns, moves OpenAI Codex OAuth tokens into encrypted sidecar storage, and includes daemon plus oc-path churn.

## Best Possible Solution

Keep the persistence-boundary redaction work, move one-way OAuth sidecar migration to an explicit save/doctor or owner-approved path with hardened key handling, trim unrelated churn, and merge only after maintainer review plus real proof.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence. Current main appends config audit records and transcript messages directly and trajectory runtime lacks the PR's broader token-shape redaction; I did not run a live gateway scenario in this read-only review.

## Solution Assessment

No, not yet. Shared persistence-boundary redaction is the right direction, but normal auth-store loads should not rewrite credential storage and the sidecar key derivation/security model needs owner-approved hardening.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.8

Full review comments:

- **[P2] Move auth-store rewriting out of normal loads:** `src/agents/auth-profiles/store.ts:521-523`
  - body: Normal non-read-only auth-store loads now pass `rewriteInlineOAuthSecrets`, so startup/status/runtime reads can create key material, sidecar files, and rewrite `auth-profiles.json`. Keep save-time redaction, but run this one-way migration from an explicit save/doctor path or after auth-owner approval.
  - confidence: 0.82
- **[P2] Harden OAuth sidecar key derivation:** `src/agents/auth-profiles/persisted.ts:371`
  - body: The AES-GCM key is a fast SHA-256 digest of `OPENCLAW_AUTH_PROFILE_SECRET_KEY`/keychain/file seed. If that seed is passphrase-like, the encrypted sidecar can be brute-forced cheaply; require random key material or use an approved password KDF before storing tokens this way.
  - confidence: 0.68
