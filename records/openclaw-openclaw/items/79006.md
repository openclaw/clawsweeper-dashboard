## Summary

Keep this PR open. The shared persistence-boundary redaction is a plausible hardening direction, but the PR is protected by the `maintainer` label, still lacks external real behavior proof, and still changes ordinary auth-store read paths so they can rewrite credential storage.

## What This Changes

The PR adds structured secret redaction across persisted logs, config audit JSONL, trajectory events, and session transcripts, while moving OpenAI Codex OAuth token material out of `auth-profiles.json` into sidecar secret files.

## Best Possible Solution

Keep the shared persistence-boundary redaction, move OAuth token sidecar migration to an explicit save/doctor repair path or get owner approval, resolve the CodeQL auth-secret concern, then merge only after real proof and exact-head checks.

## Reproduction Assessment

Yes, source-reproducible. Current main serializes config audit records and transcript messages directly, and trajectory sanitization lacks the added token-shape redaction pass; I did not run a live gateway scenario in this read-only review.

## Solution Assessment

No, not yet. Shared redaction at persistence boundaries is the right narrow direction, but the load-time auth-store rewrite and unresolved security/proof gates need resolution before this is the best mergeable fix.
