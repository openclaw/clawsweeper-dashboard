## What This Changes

The PR adds a stricter `hasRealPollIntent` helper, switches `message(action="send")` to use it, and updates focused poll/send tests for metadata-only versus real poll payloads.

## Best Possible Solution

## Reproduction Assessment

Yes. Source inspection on current main shows `message(action="send")` still reaches `hasPollCreationParams(params)`, and current tests still expect metadata-only poll values to throw the poll-action error.

## Solution Assessment

Yes. A separate real-poll-intent helper is the narrowest maintainable direction because it avoids weakening poll creation validation while stopping send from treating metadata-only fields as poll content.
