## Summary

Keep open: current main and v2026.6.6 only partially address the poll-send false positive via the merged shared-modifier fix, while this PR still targets the remaining channel-extra metadata case shown in the Discord production comments. ## What This Changes

This PR adds a narrower poll-intent helper for `message.send` and updates outbound tests so metadata-only poll fields no longer reject normal sends while non-empty poll questions or options still do.

## Best Possible Solution

Rebase onto current main, make the channel-extra poll metadata policy explicit, preserve real poll validation, add coverage for the Discord payload shape, and provide real channel proof; keep the broader schema cleanup tracked in https://github.com/openclaw/openclaw/issues/43015.
