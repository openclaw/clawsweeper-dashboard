## What This Changes

The PR narrows message send poll-intent detection so metadata-only poll fields no longer make `action="send"` fail, while preserving rejection for non-empty poll question or options.

## Best Possible Solution

Land a rebased consolidated fix that treats metadata-only/default poll params as absent for plain sends, keeps real poll payloads on the documented `poll` action path, retains focused regression coverage, and includes real behavior proof.

## Reproduction Assessment

## Solution Assessment

Mostly yes as a code direction. Narrowing send rejection to actual poll question/options preserves the documented action contract better than stripping all poll fields, but the landing branch should be rebased, proven in a real setup, and consolidated with overlapping PRs.
