## Summary

Keep open: current main still has the reported source-reproducible send/poll false positive, and this PR is a plausible narrow fix, but it still needs contributor real behavior proof and maintainer consolidation with overlapping poll-send contract PRs before merge.

## What This Changes

The PR adds `hasRealPollIntent`, switches `message(action="send")` to reject only non-empty poll question/options, and updates focused poll/send tests to allow metadata-only poll params.

## Best Possible Solution

Land one consolidated, rebased normalization that treats model/default poll metadata as absent for plain sends, keeps strict validation for explicit `poll` actions, and preserves or rejects real poll payloads according to the chosen contract.

## Reproduction Assessment

Yes. Source inspection shows current main still routes `message(action="send")` through `hasPollCreationParams`, and focused tests still assert rejection for metadata-only duration/public and negative-duration poll params on send.

## Solution Assessment

Unclear as a merge-ready solution. The helper is a maintainable narrow direction for metadata-only poll params, but maintainers still need to consolidate it with the strip-all/default-normalization proposals and require real behavior proof.
