## Summary

Keep open. Current main and the latest release still have source-reproducible poll and Discord component failures for default-filled message-tool params, while the workspace media-root part remains a security-sensitive policy question rather than a cleanup close.

## What This Changes

The issue reports that GPT-5.4/5.5-style default-filled message-tool parameters make plain sends fail through poll validation, Discord component parsing, and local media path policy.

## Best Possible Solution

Land one consolidated normalization contract that treats model-filled no-op poll and Discord component defaults as absent for plain sends, with regression coverage, and handle workspace media-root access as a separate scoped policy decision.

## Reproduction Assessment

Yes for the poll and Discord failures: current source shows `pollDurationHours: 24` reaches the send poll guard, and empty Discord modal defaults reach the parser throw. The media-root portion is not a high-confidence current-main runtime repro without the exact agent/session media-root context.

## Solution Assessment

No as one bundled fix. Default-parameter normalization is the right direction for poll and Discord components, but the workspace media-root relaxation changes local file access policy and should be reviewed separately.
