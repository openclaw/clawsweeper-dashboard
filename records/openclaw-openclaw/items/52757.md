## Summary

Keep open: current main and v2026.5.20 still have source-reproducible poll and Discord component failures for default-filled message-tool params, and the workspace media-root part remains a security-sensitive policy decision with overlapping open PRs still unresolved.

## What This Changes

The issue asks OpenClaw to tolerate GPT-5.4/5.5 default-filled `message.send` poll/component parameters and clarify or relax workspace-local media attachment roots.

## Best Possible Solution

Land one consolidated normalization contract that treats model-filled no-op poll and Discord component defaults as absent for plain sends, with regression coverage, and handle workspace media-root access as a separate scoped policy/security decision.

## Reproduction Assessment

Yes for the poll and Discord failures: current source shows `pollDurationHours: 24` reaches the send poll guard, and empty Discord modal defaults reach the parser throw. The media-root portion is context-dependent on agent-scoped local roots and needs an exact runtime path/config to reproduce fully.

## Solution Assessment

No as one bundled fix. Default-parameter normalization is the right direction for poll and Discord components, but workspace media-root relaxation changes local file access policy and should be reviewed separately.
