## Summary

Keep open. Current main still has source-reproducible poll and Discord component failures for default-filled message-tool parameters, and the open fix PR https://github.com/openclaw/openclaw/pull/75652 is unmerged and intentionally does not cover the media-path portion.

## What This Changes

The issue asks OpenClaw to treat GPT-5.4 default-filled message-tool params as omitted for poll detection, Discord component routing, and local media paths.

## Best Possible Solution

Land a narrow poll and Discord default-normalization fix, then track any still-reproducible workspace media-root behavior as a separate media-policy review.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `message.send` with `pollDurationHours: 24` still reaches the poll-only guard, and Discord `components.modal.fields: []` still reaches the component parser and throws.

## Solution Assessment

Yes for the narrow poll and Discord normalization direction; no for bundling broad workspace media-root relaxation into the same fix. The safer solution is semantic no-op normalization plus a separate media-policy review if the path issue still reproduces.
