## Summary

Keep open. Current main still has the poll and Discord component default-filled parameter failures, and the linked fix PR is open and unmerged; the media-path part should stay separate from the narrow normalization fix unless it is reproduced under the current agent-scoped media policy.

## What This Changes

The issue requests message-tool normalization so GPT-5.4 default-filled optional parameters do not trigger poll handling, Discord component parsing, or local media-path failures on ordinary sends.

## Best Possible Solution

Land a narrow poll and Discord default-normalization fix after review, then handle any remaining workspace media-root behavior as a separate security-sensitive media policy issue if it still reproduces on current main.

## Reproduction Assessment

Yes. A source-level reproduction is clear: `message.send` with `pollDurationHours: 24` and empty poll content still reaches the poll-only guard, and a Discord `components.modal.fields: []` object still reaches the component parser and throws.

## Solution Assessment

Yes for the narrow poll and Discord normalization direction in #75652; no for bundling broad media-root relaxation into the same fix. The safer end state is semantic no-op normalization plus a separate current-main media-policy repro if needed.
