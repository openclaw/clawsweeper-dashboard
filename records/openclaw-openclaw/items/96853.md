## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Preserve terminal timeout and killed statuses:** `src/agents/session-running-reconciliation.ts:14`
  - body: `timeout` and `killed` are valid terminal session statuses, not stale running states. Because both reconciliation functions use this set and then call `markEntryTerminal(... newStatus: "failed")`, startup recovery or no-active-run abort can rewrite an already timed-out/killed session to failed and update its lifecycle timestamps. Restrict this to actual in-progress stale statuses or preserve the existing terminal status.
  - confidence: 0.9
- **[P2] Reserve every persisted core session field:** `src/agents/session-running-reconciliation.ts:86-87`
  - body: This helper persists `staleRunningRecoveredAt` and `safeFallbackDelivered` as top-level session fields, but the PR only types and reserves the two other recovery fields. Plugin `sessionEntrySlotKey` validation can still accept those names, so plugin state can collide with core-written metadata. Either add these fields to `SessionEntry` and the reserved slot list or keep them out of the top-level persisted entry.
  - confidence: 0.82
