## Summary

Keep open. Current main still lacks Feishu-local silent-token suppression, so the reported bug is real, but this PR is incomplete: it suppresses only some text-only paths, can leave Feishu streaming-card placeholders behind, and overlaps with the still-open #56167 consolidation PR.

## What This Changes

This PR adds Feishu `NO_REPLY` checks that early-return in `feishuOutbound.sendText` and in the text-only reply-dispatcher delivery path before Feishu API sends.

## Best Possible Solution

Consolidate one Feishu-owned fix that suppresses exact silent tokens at every Feishu text API boundary, preserves media delivery and streaming-card cleanup, adds focused outbound plus reply-dispatcher regression tests, and includes the required user-facing changelog entry.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists without live Feishu: direct `sendText({ text: "NO_REPLY" })`, `sendMedia({ text: "NO_REPLY", mediaUrl })`, `sendMedia({ text: "NO_REPLY" })`, and dispatcher `deliver({ text: "NO_REPLY", mediaUrl })` can route the sentinel into Feishu send helpers on current main or through gaps left by this patch.

## Solution Assessment

No. The direction is valid, but this patch is not the best complete fix because it misses `sendMedia` caption/fallback paths, suppresses only no-media dispatcher payloads, can bypass streaming-card finalization behavior, and lacks the required changelog entry.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve streaming cleanup for silent finals:** `extensions/feishu/src/reply-dispatcher.ts:259-261`
  - body: In card streaming mode, `onReplyStart` may already create a streaming card before the final payload arrives. Returning here for text-only `NO_REPLY` skips the final-text path that updates stream state, so idle close can leave the placeholder card content visible.
  - confidence: 0.86
- **[P2] Suppress silent text when media is present:** `extensions/feishu/src/reply-dispatcher.ts:259-261`
  - body: The `!hasMedia` condition means a payload with media plus `text === "NO_REPLY"` bypasses this guard and still enters the text-send block before media delivery. Treat the silent token as non-deliverable text while still allowing the media path to run.
  - confidence: 0.91
- **[P2] Guard sendMedia captions and fallback text:** `extensions/feishu/src/outbound.ts:83-87`
  - body: Adding the guard only in `sendText` leaves `sendMedia` paths that call `sendOutboundText` directly for captions and text-only fallback. A direct `sendMedia` call with `text: "NO_REPLY"` can still send the token to Feishu.
  - confidence: 0.9
- **[P3] Return the normal Feishu suppressed result shape:** `extensions/feishu/src/outbound.ts:85`
  - body: Normal Feishu sends return chat-scoped metadata such as `chatId`, but this suppressed result returns `channelId: to`. That can make downstream result formatting treat a suppressed Feishu send differently from successful Feishu sends.
  - confidence: 0.72
- **[P3] Add the required changelog entry:** `extensions/feishu/src/outbound.ts:83-87`
  - body: This is a user-visible Feishu delivery fix, and the PR only changes source files. OpenClaw policy requires a `CHANGELOG.md` entry for user-facing fixes before merge.
  - confidence: 0.82
