## Summary

Keep open: the Feishu silent-token bug is real, but this patch is not mergeable on current main because it imports helpers from the wrong SDK surface and leaves outbound media plus dispatcher mixed-media/streaming paths incomplete; #56167 overlaps but does not fully supersede the dispatcher work.

## What This Changes

This PR adds Feishu NO_REPLY suppression to outbound sendText and the reply dispatcher before Feishu text API calls.

## Best Possible Solution

Land one Feishu-owned fix that uses an exported SDK subpath, suppresses exact silent tokens at all Feishu text boundaries, preserves media and streaming cleanup, and includes focused tests plus changelog.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: exact NO_REPLY can pass through Feishu sendText, sendMedia caption/fallback, and dispatcher mixed-media text delivery without a Feishu-local guard.

## Solution Assessment

No. The direction is right, but the patch imports from a non-exporting SDK entrypoint, suppresses only part of the Feishu surface, mishandles media plus silent text, and can skip streaming cleanup.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Use an exported SDK subpath for silent helpers:** `extensions/feishu/src/outbound.ts:3`
  - body: The root `openclaw/plugin-sdk` entrypoint on current main does not export `isSilentReplyText` or `SILENT_REPLY_TOKEN`; those helpers are on reply-runtime/reply-chunking. As written, the new value import in both changed files will fail before the Feishu fix can run.
  - confidence: 0.93
- **[P1] Preserve streaming cleanup for silent finals:** `extensions/feishu/src/reply-dispatcher.ts:259-261`
  - body: In card streaming mode, `onReplyStart` can create a streaming card before the final payload arrives. Returning here for a text-only silent token skips the final-text path, so idle close can leave the initial thinking card content instead of cleaning up the silent reply.
  - confidence: 0.86
- **[P2] Suppress silent text when media is present:** `extensions/feishu/src/reply-dispatcher.ts:259-261`
  - body: The `!hasMedia` condition means a payload with media plus `text === "NO_REPLY"` bypasses this guard and still enters the text-send block before media delivery. Treat the silent token as non-deliverable text while still sending the media.
  - confidence: 0.91
- **[P2] Guard sendMedia captions and fallback text:** `extensions/feishu/src/outbound.ts:83-87`
  - body: Adding the guard only in `sendText` leaves `sendMedia` paths that call the text helper directly for captions and text-only fallback. A direct `sendMedia` call with `text: "NO_REPLY"` can still send the token to Feishu.
  - confidence: 0.9
- **[P3] Add the required changelog entry:** `extensions/feishu/src/outbound.ts:83-87`
  - body: This is a user-facing Feishu delivery fix, but the PR only changes source files. OpenClaw policy requires a `CHANGELOG.md` entry for user-facing fixes before merge.
  - confidence: 0.82
