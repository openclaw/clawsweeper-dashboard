## Best Possible Solution

Use #56167 or a replacement branch as the single Feishu-owned fix covering all text API boundaries, media handling, streaming cleanup, focused tests, and the required changelog entry.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Close streaming before suppressing silent finals:** `extensions/feishu/src/reply-dispatcher.ts:259-261`
  - body: When card streaming has already opened a Feishu card, returning for a final text-only NO_REPLY before the normal final-text path leaves the placeholder content to be closed later with empty text. Finalize or clear the streaming session before suppressing the silent final.
  - confidence: 0.86
- **[P2] Suppress silent text even when media is present:** `extensions/feishu/src/reply-dispatcher.ts:259-261`
  - body: The `!hasMedia` condition means a payload with media plus `text === "NO_REPLY"` bypasses this guard and still enters the text-send path before media delivery. Treat the silent token as non-deliverable text while preserving the media path.
  - confidence: 0.91
- **[P2] Guard sendMedia caption and fallback text:** `extensions/feishu/src/outbound.ts:83-87`
  - body: The new guard only wraps `sendText`, but `sendMedia` still calls `sendOutboundText` directly for captions and the no-media fallback. A direct `sendMedia({ text: "NO_REPLY" })` can still leak the token to Feishu.
  - confidence: 0.9
- **[P3] Return Feishu chat metadata for suppressed sends:** `extensions/feishu/src/outbound.ts:85`
  - body: The suppressed result returns `channelId: to`, while normal Feishu sends return chat-scoped metadata such as `chatId`. Use the normal Feishu result shape so downstream formatting does not treat suppressed sends differently.
  - confidence: 0.72
- **[P3] Add the required changelog entry:** `extensions/feishu/src/outbound.ts:83-87`
  - body: This is a user-facing Feishu delivery fix, but the PR only changes source files. OpenClaw policy requires a `CHANGELOG.md` entry for user-facing fixes before merge.
  - confidence: 0.82
