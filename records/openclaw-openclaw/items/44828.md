## Summary

Keep open. Current main still has the Feishu silent-token leak, but this PR is incomplete and currently dirty against main; it misses Feishu media and mixed-payload dispatcher paths, and the early return can interfere with streaming-card finalization.

## What This Changes

This PR adds Feishu `NO_REPLY` checks to outbound `sendText` and the no-media reply-dispatcher delivery path before Feishu API sends.

## Best Possible Solution

Consolidate one Feishu-owned fix that suppresses exact silent tokens at every Feishu text API boundary, preserves media delivery and streaming cleanup, adds focused regression tests, and includes the required changelog entry.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists on current main: direct `sendText`, `sendMedia` caption/fallback, and dispatcher mixed-media paths can route exact `NO_REPLY` into Feishu text send helpers without a Feishu-local guard.

## Solution Assessment

No. The direction is right, but this patch is not the best complete fix because it misses `sendMedia` caption/fallback paths, suppresses only no-media dispatcher payloads, can bypass streaming-card cleanup, and lacks changelog coverage.

## Review Findings

Full review comments:

- **[P1] Preserve streaming cleanup for silent finals:** `extensions/feishu/src/reply-dispatcher.ts:259-261`
  - body: In card streaming mode, `onReplyStart` may already create a streaming card before the final payload arrives. Returning here for a final text-only `NO_REPLY` skips the final-text path that updates and closes stream state, which can leave placeholder card content visible instead of completing the silent reply cleanly.
  - confidence: 0.86
- **[P2] Suppress silent text when media is present:** `extensions/feishu/src/reply-dispatcher.ts:259-261`
  - body: The `!hasMedia` condition means a payload with media plus `text === "NO_REPLY"` bypasses this guard and still enters the text-send block before media delivery. Treat the silent token as non-deliverable text while still allowing the media path to run.
  - confidence: 0.91
- **[P2] Guard sendMedia captions and fallback text:** `extensions/feishu/src/outbound.ts:83-87`
  - body: Adding the guard only in `sendText` leaves `sendMedia` paths that call the text helper directly for captions and text-only fallback. A direct `sendMedia` call with `text: "NO_REPLY"` can still send the token to Feishu.
  - confidence: 0.9
- **[P3] Return the normal Feishu suppressed result shape:** `extensions/feishu/src/outbound.ts:85`
  - body: The suppression path returns `{ channelId: to }`, while normal Feishu sends return chat-scoped metadata such as `chatId`. Returning `chatId` keeps downstream result formatting consistent with successful Feishu sends.
  - confidence: 0.72
- **[P3] Add the required changelog entry:** `extensions/feishu/src/outbound.ts:83-87`
  - body: This is a user-facing Feishu delivery fix, but the PR only changes source files. OpenClaw policy requires a `CHANGELOG.md` entry for user-facing fixes before merge.
  - confidence: 0.82
