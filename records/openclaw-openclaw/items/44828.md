## Summary

Keep open: current main still has the reported Feishu NO_REPLY leak, but this PR is incomplete and would introduce build/control-flow regressions before it can merge.

## What This Changes

The PR adds Feishu NO_REPLY suppression in outbound `sendText` and the reply dispatcher by importing silent-reply helpers and returning before text delivery.

## Best Possible Solution

Land one Feishu-owned fix that imports silent helpers from an exported reply SDK subpath, suppresses exact silent tokens at every Feishu text boundary, preserves media and streaming cleanup, adds focused tests, and includes a user-facing changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main shows exact `NO_REPLY` can reach Feishu through `sendText`, `sendMedia` caption/fallback text, and dispatcher text delivery when upstream normalization does not drop it.

## Solution Assessment

No. The direction is correct, but the patch uses a non-exporting SDK import, covers only part of the Feishu surface, mishandles media plus silent text, and can skip streaming cleanup.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Use an exported SDK subpath:** `extensions/feishu/src/outbound.ts:3`
  - body: The root `openclaw/plugin-sdk` entrypoint on current main does not export `isSilentReplyText` or `SILENT_REPLY_TOKEN`, so this new value import fails before the Feishu fix can run. Import from an exported reply SDK subpath instead, or intentionally add and test a public facade.
  - confidence: 0.93
- **[P1] Preserve streaming cleanup for silent finals:** `extensions/feishu/src/reply-dispatcher.ts:259-261`
  - body: Returning here for a text-only silent token skips the normal final-text path. In card streaming mode, `onReplyStart` may already have opened a streaming card, so this can leave the initial thinking card instead of finalizing the silent reply cleanly.
  - confidence: 0.86
- **[P2] Suppress silent text when media is present:** `extensions/feishu/src/reply-dispatcher.ts:259-261`
  - body: The `!hasMedia` condition means a payload with media plus `text === "NO_REPLY"` bypasses this guard and still enters the text-send block before media delivery. Treat the silent token as non-deliverable text while still sending the media.
  - confidence: 0.91
- **[P2] Guard sendMedia captions and fallback text:** `extensions/feishu/src/outbound.ts:83-87`
  - body: Adding the guard only in `sendText` leaves `sendMedia` paths that call the text helper directly for captions and text-only fallback. A direct `sendMedia` call with `text: "NO_REPLY"` can still send the token to Feishu.
  - confidence: 0.9
- **[P3] Add the required changelog entry:** `extensions/feishu/src/outbound.ts:83-87`
  - body: This is a user-facing Feishu delivery fix, but the PR only changes source files. OpenClaw policy requires a `CHANGELOG.md` entry for user-facing fixes before merge.
  - confidence: 0.82
