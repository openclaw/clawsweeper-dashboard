## Summary

Keep open. Current main still matches the reported stale managed web_search path: active runtime snapshots resolve Gemini plugin-scoped SecretRefs for matching source-config callers, but an existing web_search tool wrapper can keep passing an older/raw caller config that the selector preserves, and Gemini then strictly rejects the raw SecretRef during execution.

## What This Changes

The issue reports that Gemini managed web_search can receive an unresolved plugin-scoped SecretRef from stale/raw runtime config after upgrading to OpenClaw 2026.4.29.

## Best Possible Solution

Managed web_search execution should late-bind the active resolved runtime config for gateway/session-owned tool calls while preserving explicit custom config behavior for SDK and plugin callers. Add regression coverage for an existing wrapper that keeps a stale raw Gemini plugin SecretRef after an active resolved snapshot is installed or refreshed.

## Reproduction Assessment

Yes. The issue gives concrete config and error output, and current source gives a high-confidence static reproduction path: call managed web_search from an existing wrapper with stale raw config after a resolved runtime snapshot is active, then Gemini reads the raw plugin-scoped apiKey.

## Solution Assessment

Unclear. The requested resolved-snapshot direction is correct, but applying it globally in selectApplicableRuntimeConfig is risky; the narrower maintainable fix likely belongs at the managed web_search wrapper/runtime boundary.
