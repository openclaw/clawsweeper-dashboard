## Summary

Keep open. Current main still has the reported source-config leak shape: managed web_search tools can close over caller config and runtime metadata, runWebSearch preserves nonmatching input config instead of forcing the active resolved runtime snapshot, and the Gemini provider then reads plugin-scoped credentials from ctx.config through the strict SecretRef reader.

## What This Changes

The issue reports that Gemini managed web_search can receive a raw plugin-scoped SecretRef at runtime after upgrading to OpenClaw 2026.4.29, especially from existing sessions after secrets reload.

## Best Possible Solution

Managed gateway/session-owned web_search execution should late-bind the active resolved runtime config and current runtime web-tool metadata, while explicit SDK/plugin custom config continues to use the caller-supplied config. Add a regression test for an existing managed wrapper holding a raw Gemini plugin SecretRef after a resolved runtime snapshot is active or refreshed.

## Reproduction Assessment

Yes. The issue gives concrete config and error output, and current main gives a static reproduction path: execute managed web_search from a wrapper holding stale raw config that no longer matches the active source snapshot, then Gemini reads the raw plugin-scoped apiKey and the strict SecretRef normalizer throws.

## Solution Assessment

Unclear. The requested resolved-snapshot direction is correct, but the best maintainable fix should distinguish managed agent wrappers from explicit SDK/custom config callers instead of globally replacing every nonmatching caller config with the active runtime snapshot.
