  - body: `planSkillUninstall()` only derives the tracked owner from `.clawhub/lock.json`.   - confidence: 0.91
- **[P2] Stop before lockfile removal on owner mismatch:** `src/skills/lifecycle/clawhub.ts:1611-1614`
  - body: `executeSkillUninstall()` records an owner-mismatch warning but then continues into the lockfile removal block. The CLI currently pre-checks this, but the exported lifecycle helper should enforce the refusal itself so another caller cannot untrack a mismatched skill.
  - confidence: 0.9
- **[P2] Accept legacy tracked slugs when they already exist:** `src/skills/lifecycle/clawhub.ts:1550-1552`
  - body: For bare input, the planner immediately calls `validateRequestedSkillSlug()`. Current update behavior first resolves existing tracked slugs with `normalizeTrackedSkillSlug()`, and existing tests cover legacy Unicode tracked slugs, so this command can reject already-installed tracked skills instead of cleaning them up.
  - confidence: 0.88
- **[P2] Document the new destructive skills command:** `src/cli/skills-cli.ts:382-389`
  - body: This adds a public command with destructive behavior and five flags, but the public skills CLI reference still omits uninstall, `--dry-run`, `--yes`, JSON output, and ClawHub provenance refusal behavior. Add the docs alongside the command so users see the supported safe-removal path.
