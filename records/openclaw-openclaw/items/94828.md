## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Execute uninstall before writing JSON output:** `src/cli/skills-cli.ts:402-404`
  - body: With `--json`, the command writes the plan and returns before it checks installation state, prompts, or calls `executeSkillUninstall`. A scripted `openclaw skills uninstall foo --yes --json` therefore exits successfully while leaving both the skill directory and lockfile entry in place; make JSON an output format for the dry-run or executed result, not an implicit no-op.
  - confidence: 0.9
- **[P1] Refuse untracked skill folders before deleting:** `src/skills/lifecycle/clawhub.ts:1572-1574`
  - body: `executeSkillUninstall` removes `plan.skillDir` whenever the directory exists, even when there is no ClawHub origin metadata and no lockfile entry. Current `openclaw skills install git:...` and local installs use the same workspace `skills/<slug>` root, so this ClawHub-scoped command can delete non-ClawHub skills with `--yes`; gate deletion on ClawHub provenance or make broad skill removal an explicit documented contract.
  - confidence: 0.89
- **[P2] Export the new lifecycle functions from the test mock:** `src/cli/skills-cli.ts:18-24`
  - body: The existing `skills-cli.commands.test.ts` mock for `../skills/lifecycle/clawhub.js` does not provide `planSkillUninstall` or `executeSkillUninstall`. Because this file now imports those names, the current command test module will fail to load under Vitest until the mock and focused command coverage are updated.
  - confidence: 0.84
