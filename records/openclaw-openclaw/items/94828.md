## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Make the uninstall path non-null before removing:** `src/skills/lifecycle/clawhub.ts:1574`
  - body: `SkillUninstallPlan.skillDir` is declared as `string | null`, but `executeSkillUninstall` passes it directly to `fs.rm`. Under the repo's strict TypeScript settings this should fail type checking; make the plan path non-null or narrow it before removal.
  - confidence: 0.92
- **[P1] Refuse untracked skill folders:** `src/skills/lifecycle/clawhub.ts:1572-1574`
  - body: `plan.skillDirExists` is enough to delete `skills/<slug>` even when there is no lockfile entry or `.clawhub/origin.json`. Git/local skill installs share this directory and explicitly clear ClawHub metadata, so a ClawHub-scoped uninstall can delete manually managed skills; gate directory removal on ClawHub provenance and treat untracked folders as not installed unless maintainers choose broader semantics.
  - confidence: 0.92
- **[P2] Do not make `--json` an implicit dry run:** `src/cli/skills-cli.ts:402-404`
  - body: The `--json` branch writes `{ plan }` and returns before confirmation or `executeSkillUninstall`, so even `openclaw skills uninstall <slug> --yes --json` leaves the directory and lockfile untouched. If JSON is only for preview, tie it to `--dry-run`; otherwise emit the actual uninstall result after execution.
  - confidence: 0.82
