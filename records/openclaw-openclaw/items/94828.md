## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Require origin slug match before deleting:** `src/skills/lifecycle/clawhub.ts:1585`
  - body: `planSkillUninstall` treats any normalized origin metadata under the requested directory as ClawHub provenance, but it never checks that `origin.slug` normalizes to `resolvedSlug`. A copied or attacker-written valid-looking `.clawhub/origin.json` for another skill would set `isClawHubInstall` and let `executeSkillUninstall` recursively remove this directory, despite the docs promise that non-ClawHub directories are left untouched. Please require the origin slug, owner, and lock data to match the requested install before setting this true, and add a mismatched-origin regression test.
  - confidence: 0.91
- **[P3] Report owner-qualified misses as not installed:** `src/skills/lifecycle/clawhub.ts:1592`
  - body: For an untracked request like `@owner/missing`, `ownerRequiredButMissing` becomes true solely because a requested owner exists and `trackedOwnerHandle` is undefined. The CLI checks that before the not-installed branch, so a missing skill reports an owner mismatch instead of the documented not-installed outcome. Gate this flag on an actually tracked install, or check the no-dir/no-lock case first.
  - confidence: 0.86
