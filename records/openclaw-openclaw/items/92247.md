## Summary

The PR addresses a real model-listing latency regression with convincing live proof, but it is not merge-ready because the new global auth shortcut can omit catalog models for valid resolver-only provider credentials.

## What This Changes

The PR bounds model-catalog loading, reduces authentication discovery, and caches shared model-provider data for CLI, Telegram, and Discord model-selection flows.

## Best Possible Solution

Keep the bounded catalog and shared-cache performance improvements, but make the no-auth optimization account for resolver-only credentials—or remove that shortcut—and add row-level regression coverage before rebasing.

## Reproduction Assessment

Yes at source level. Build an auth index with only `GOOGLE_CLOUD_API_KEY` and enter `appendAuthenticatedCatalogRows`; the global predicate is false before lazy `google-vertex` authentication can be resolved, so catalog loading is skipped.

## Solution Assessment

No, not as written. Bounding catalog work and caching repeated picker builds are appropriate, but the global auth shortcut is not equivalent to the established per-provider auth contract and must be repaired or removed.
