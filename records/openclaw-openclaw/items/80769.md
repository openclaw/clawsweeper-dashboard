## Summary

Keep open: current main still has static exec policy, allowlist evaluation, and human/channel approval routing, but no verifier-agent approval mode. The request would add a security-sensitive approval evaluator and config contract, so it needs maintainer product/security design rather than cleanup closure.

## What This Changes

The issue requests a new exec approval mode where an independent verifier agent evaluates raw commands against policy before host execution.

## Best Possible Solution

Design a conservative fail-closed verifier approval contract in core, then implement schema, gateway routing, docs, and regression tests after maintainer security review.
