## What This Changes

Request adding an agent-backed exec approval mode that sends raw commands to a verifier agent with policy, timeout, fallback, and denial-notification config.

## Best Possible Solution

Build the verifier as a ClawHub code plugin using `before_tool_call` to inspect raw command params and fail closed; open a narrow SDK/API issue only if that implementation finds a missing hook contract.

## Reproduction Assessment

Not applicable: this is a feature/security-design request, not a bug. Source inspection confirms current core config has no `approvals.exec.mode: "agent"`, while plugin hooks already provide the extension path.

## Solution Assessment

No for core: adding another core exec approval mode is not the best fit after the guardrail-provider scope decision. The maintainable path is a ClawHub plugin on `before_tool_call` unless a concrete API gap appears.
