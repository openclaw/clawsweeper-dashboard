## Summary

Keep open: current main still has deterministic exec approval policy, allowlists, safe-bin handling, forwarding to humans/chat surfaces, and Codex-plugin-specific native approval review, but no generic verifier-agent approval mode. The request changes the host exec trust boundary and config contract, so it needs maintainer product/security design rather than cleanup closure.

## What This Changes

The issue requests an agent-as-approver exec mode where a separate verifier agent receives only the raw command and approves or denies it against a safety policy.

## Best Possible Solution

Define a fail-closed core verifier contract for exec approvals, with explicit schema, audit events, routing/fallback rules, docs, and tests after security/product review.

## Reproduction Assessment

Not applicable: this is a feature and security-design request, not a report of broken existing behavior. Source and docs inspection give high-confidence evidence that current main lacks the requested generic verifier-agent mode.

## Solution Assessment

Unclear until maintainers decide the trust model. The proposed direction is plausible, but it should be shaped as a core approval-policy design with fail-closed defaults instead of a narrow automated fix.
