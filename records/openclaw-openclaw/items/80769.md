## Summary

Keep open: current main supports static exec policy, allowlists, and human/chat approval resolution, but not a verifier-agent approval mode. This is a new security-sensitive product capability and config contract, so it needs maintainer design review rather than cleanup closure.

## What This Changes

Request an agent-backed exec approval mode that sends raw commands to a separate verifier agent under policy before allowing execution.

## Best Possible Solution

Design a conservative, fail-closed verifier contract for exec approvals, then implement it with schema/docs/tests after maintainer security review.

## Reproduction Assessment

Not applicable: this is a feature and security-design request, not a report of broken existing behavior. Source inspection shows current main lacks the requested verifier-agent mode.

## Solution Assessment

Unclear until maintainers decide the trust model. The proposed direction is plausible, but it should be shaped as a core approval-policy design with fail-closed defaults rather than implemented as a narrow bug fix.
