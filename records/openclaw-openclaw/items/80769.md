## Summary

Keep open: current main still supports static exec policy, allowlists, durable approvals, and human/channel approval resolution, but not an isolated verifier-agent mode. The request is a new security-sensitive feature and config contract rather than cleanup that can be closed.

## What This Changes

The issue requests a new exec approval mode where an isolated verifier agent evaluates raw shell commands against a safety policy before approval.

## Best Possible Solution

Define a fail-closed verifier-agent approval contract in core after product/security review, including payload isolation, verifier identity/model constraints, policy format, timeout/fallback semantics, audit/notification behavior, docs, and regression coverage.

## Solution Assessment
