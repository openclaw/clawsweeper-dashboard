## Summary

Keep this PR open, but do not merge the current head: the success path is coherent, while the candidate lifecycle can falsely report a selected answer after a failed or interrupted Codex attempt, and the supplied proof does not exercise the complete real event path.

## What This Changes

The PR emits Codex final-answer candidate lifecycle events, suppresses them from channel progress, and renders candidate, superseded, and selected states as localized Control UI activity entries.

## Best Possible Solution

Derive `selected` from the canonical successful terminal outcome, leave candidates non-selected for failed or interrupted attempts, add focused regression coverage for those outcomes, and prove the real Codex-to-Gateway-to-Control-UI path with redacted diagnostics.

## Reproduction Assessment

Yes, at source level. Codex emits failed and interrupted statuses through `turn/completed`, while the patch marks the final candidate selected from unconditional result construction regardless of that terminal status.

## Solution Assessment

No, not in its current form. Surfacing candidate history is a reasonable design, but selection should be derived from the existing canonical terminal outcome rather than merely from the presence of final-answer text.
