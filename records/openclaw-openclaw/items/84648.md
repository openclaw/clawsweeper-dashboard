## Label Justifications

- `P1`: The PR touches the core exec tool and can make exec unusable for existing installs if merged as written.
- `merge-risk: 🚨 compatibility`: The new mandatory adapter check changes default exec behavior for users without SafeOps environment variables.
- `merge-risk: 🚨 availability`: An unavailable or unconfigured adapter makes every exec command fail closed before existing execution paths run.
- `merge-risk: 🚨 security-boundary`: The diff introduces an external command-preflight decision point that receives command metadata before OpenClaw's existing approval and allowlist contracts.

## Summary

Keep open; this PR is not merge-ready because it hard-codes a SafeOps network gate into `exec`, duplicates the existing generic `before_tool_call` plugin hook path, appears incompatible with strict TypeScript, and can block default exec installs when SafeOps is not configured.

## What This Changes

Adds a SafeOps adapter client and calls it from the `exec` tool before node, gateway, or direct execution, returning a blocked result when the adapter does not allow the command.

## Best Possible Solution

Keep default exec behavior unchanged and implement SafeOps through the existing plugin `before_tool_call` or trusted-policy surface unless maintainers explicitly approve a generic core preflight contract with typed config, tests, and upgrade proof.

## Reproduction Assessment

Yes for the PR-introduced defects: source inspection shows `exec` always calls SafeOps and the adapter denies without a token, while strict TypeScript makes the new untyped files suspect without needing a live run.

## Solution Assessment

No. The proposed hard-coded exec adapter duplicates the existing generic plugin hook and approval surface; the maintainable path is a plugin/trusted-policy integration or an owner-approved generic core contract.
