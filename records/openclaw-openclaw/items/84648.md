## Summary

Close as ClawHub/plugin work: SafeOps is an optional exec preflight integration, current main already exposes a plugin hook and approval seam for this behavior, and the PR hard-codes an external adapter into core exec with default-install, security-boundary, and availability regressions.

## What This Changes

The PR adds a new `src/safeops` adapter and calls it from the `exec` tool before node, gateway, or local command execution.

## Best Possible Solution

Publish SafeOps as a code plugin through ClawHub using `before_tool_call` and `requireApproval`; only open a narrow core PR if that plugin path lacks required generic exec metadata.

## Reproduction Assessment

Yes for the main PR regression by source inspection: `exec` always calls SafeOps, and the adapter denies when `SAFEOPS_ADAPTER_TOKEN` is absent. I did not run the PR branch because this review was read-only.

## Solution Assessment

No. The maintainable solution is the existing plugin `before_tool_call` and approval API, with any missing exec metadata handled as a narrow generic extension point rather than a hard-coded SafeOps core adapter.
