## What This Changes

The PR normalizes OpenClaw CalVer correction runtime versions before ClawHub plugin API range checks and adds regression tests plus a changelog entry.

## Best Possible Solution

Land the narrow plugin API normalization so stable correction builds keep their real version elsewhere while satisfying base plugin API ranges and preserving beta prerelease rejection.

## Reproduction Assessment

Yes. Current main sends 2026.5.3-1 straight into the generic semver comparator, which treats the numeric suffix as a prerelease below 2026.5.3 and makes a >=2026.5.3 plugin API range fail.

## Solution Assessment

Yes. Normalizing only the plugin API host input is the narrow maintainable fix because release docs define YYYY.M.D-N as a stable correction, while the PR leaves other version surfaces and nonnumeric beta prereleases unchanged.
