## Summary

Keep open. Current main partially filters generated session noise, but the core request is still unresolved: ingestion scores remain fixed constants, repeated content can survive timestamp/line-scoped dedupe, and session files scan in deterministic capped order without a fair rotation cursor.

## What This Changes

The issue asks memory-core dreaming to filter noisy session-corpus entries, use content-aware scoring, and avoid session-corpus scan starvation as the corpus grows.

## Best Possible Solution

Keep this as a focused bundled memory-core repair: preserve the existing generated-noise filters, add content-level session dedupe, derive bounded ingestion scores from source/content/recency signals, and persist fair progress across session sources under the sweep cap.

## Reproduction Assessment

Yes for the remaining source-level defects: fixed ingestion scores, timestamp/line-scoped dedupe, and capped sorted session scanning are directly visible on current main. The exact reported April 9 lastLightAt freeze is unclear without the reporter's local .dreams state.
