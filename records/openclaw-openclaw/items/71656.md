## Summary

Keep open. Current main has partial generated-noise filtering, but fixed session/daily ingestion scores, timestamp/line-scoped session dedupe, and deterministic capped session-file scanning still leave the reported memory dreaming quality and scan-stall problem unresolved.

## What This Changes

Improve memory-core dreaming by pre-filtering and deduping session corpus noise, scoring ingested signals by content quality and recency, and rotating scans fairly across corpus sources.

## Best Possible Solution

Keep this as a focused memory-core repair: preserve existing generated-noise filters, add content-level session dedupe, derive bounded ingestion scores from source/content/recency signals, and persist fair progress across session sources while leaving the broader session-corpus promotion boundary to https://github.com/openclaw/openclaw/issues/77831 and https://github.com/openclaw/openclaw/issues/75842.

## Reproduction Assessment

Yes for the remaining source-level defects: fixed ingestion scores, timestamp/line-scoped dedupe, and capped sorted session scanning are directly visible on current main. The exact reported April 9 freeze is not reproducible without the reporter's local memory/.dreams state.

## Solution Assessment

Yes. A narrow memory-core and memory-host SDK repair is the maintainable path; it should improve filtering, scoring, and scan fairness without adding user config or deciding the broader session-corpus promotion policy tracked elsewhere.
