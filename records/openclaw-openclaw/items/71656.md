## Summary

Keep open. Current main has partial session-noise filtering and promotion-time weighted ranking, but the central request is still incomplete: ingestion still emits fixed scores, repeated content can still bypass dedupe when timestamp or line changes, and the session scan loop still has a deterministic global cap that can stop before every file advances.

## What This Changes

Request memory-core dreaming to pre-filter and dedupe noisy session corpus entries, weight ingestion scoring by content/source/recency, and rotate session scans fairly as the corpus grows.

## Best Possible Solution

Finish this in memory-core and the session export layer: keep the existing cron/heartbeat/sentinel filters, add deterministic content-level dedupe for repeated snippets, compute bounded ingestion scores from content/source/recency before recording recall signals, and add a many-file scan-budget regression test that proves later transcript files still make progress.

## Reproduction Assessment

Yes for the remaining source-level defects: current main directly shows fixed ingestion scores, timestamp/line-scoped dedupe, and a deterministic global scan cap. The exact April 9 stall remains unclear without the reporter's local state files.

## Solution Assessment

Yes, a narrow memory-core/session export repair is the best direction. It should not require a core plugin API change, should avoid changing Dream Diary artifact semantics, and should treat related artifact-write issue #72546 as separate unless tests prove a shared root cause.
