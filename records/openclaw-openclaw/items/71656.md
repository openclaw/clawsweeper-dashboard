## Summary

Keep open. Current main partially filters transcript noise, but the broad Dreaming quality problem is still present: session and daily ingestion still use fixed source scores, session batches are recorded as generic daily signals, and light-phase selection can still prefer freshly touched entries before older session-corpus entries. The open related PR is an implementation candidate, not a merged fix or closing reference.

## What This Changes

The issue asks memory-core Dreaming to denoise session corpus ingestion, score material by content importance and recency, and rotate scan budget so session-corpus entries keep advancing.

## Best Possible Solution

Keep this as the umbrella tracker while maintainers define the Dreaming scoring and rotation contract, then land bounded fixes for conservative denoise, source-fair light selection, and content-aware ranking coordinated with related memory-core issues.

## Reproduction Assessment
