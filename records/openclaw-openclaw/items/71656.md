## Summary

Keep open. Current main has partial corpus-noise filtering, but the main requested fixes are still missing: ingestion scores remain fixed by source, repeated normalized session content can still be tracked separately when timestamp or line metadata changes, and light dreaming still has no source-aware rotation that would reliably revisit older session-corpus entries under a tight limit.

## What This Changes

Request to improve memory-core dreaming by filtering noisy session corpus entries, weighting ingestion scores by content and recency, and rotating scan budget across corpus sources.

## Best Possible Solution

Finish a narrow memory-core repair that preserves the existing generated-noise filters, adds deterministic content-level dedupe for repeated session snippets, computes bounded content/source/recency-aware ingestion scores before recall recording, and adds a persisted fair-rotation or prioritization mechanism so session-corpus entries continue to make light-phase progress under tight limits.

## Reproduction Assessment

Yes for the remaining source-level defects: current main directly shows fixed ingestion scores, timestamp/line-scoped session dedupe, and recency-first light selection without source rotation. The exact April 9 production stall is unclear without the reporter's local phase-signals and corpus files.

## Solution Assessment

Yes for the requested direction: the maintainable fix belongs in memory-core/session export and short-term promotion, without a plugin API change unless implementation proves a shared seam is missing. The current partial filtering is useful but not sufficient to close this issue.
