## Summary

Keep open. Current main filters some generated transcript noise, but the core reported problems remain visible in source: daily/session ingestion still use fixed scores, duplicate content can survive timestamp/line-scoped hashing, and capped sorted transcript scanning has no persistent rotation cursor.

## What This Changes

The issue asks memory-core dreaming to pre-filter and dedupe session-corpus noise, use content-aware and recency-aware ingestion scoring, and rotate session scanning fairly under the sweep cap.

## Best Possible Solution
