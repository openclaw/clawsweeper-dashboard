## Summary

Keep open. Current main has partial transcript-noise filtering, but the central Dreaming quality problem remains: session and daily ingestion still use fixed source scores, session batches are recorded as generic daily signals, and light-phase selection can keep favoring freshly re-ingested daily entries over older session-corpus entries. The related open items cover narrower slices, not this full filtering, scoring, and scan-fairness tracker.

## What This Changes

Request source-aware pre-filtering, content-weighted scoring/recency, and fair session-corpus scan rotation for memory-core Dreaming.

## Best Possible Solution

Keep this as the broad Dreaming corpus-quality tracker, decide the session-corpus promotion and scoring contract, then split narrow fixes for fair phase selection, conservative denoise, and content-aware ranking across the linked memory-core issues.

## Reproduction Assessment

Yes at source level for the fixed-score, generic session-signal, and freshness-biased light selection paths. The exact April 9 stall was not live-replayed because it depends on the reporter's private `memory/.dreams` corpus state.

## Solution Assessment

Unclear. The requested direction is plausible, but the safest implementation needs maintainer agreement on whether session-corpus lines are promotion evidence and which content signals should affect Dreaming ranking.
