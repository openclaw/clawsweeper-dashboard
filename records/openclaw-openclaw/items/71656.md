## Summary

Keep open. Current main has partial session-corpus noise filtering, but fixed ingestion scores, metadata-scoped dedupe, and capped deterministic session scanning still leave the reported memory dreaming quality and scan-stall problems unresolved.

## What This Changes

The issue requests memory-core dreaming improvements for session-corpus noise filtering, content-aware scoring, recency bias, and fair scan rotation across corpus sources.

## Best Possible Solution

Keep this as a focused memory-core repair: preserve the existing generated-noise filters, add content-level session dedupe, derive bounded ingestion scores from source/content/recency signals, and persist fair progress across session sources without deciding the broader session-corpus promotion boundary.

## Solution Assessment
