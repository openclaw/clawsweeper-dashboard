## Summary

Keep open. Current main has partial session-corpus noise filtering, but the core requests remain: ingestion scores are still source-constant, repeated session snippets are still keyed by position/timestamp, and the session scan still uses a deterministic sorted pass with a global cap rather than a fair many-file rotation.

## What This Changes

The issue requests memory-core dreaming improvements for session corpus noise filtering/deduplication, content-aware scoring with recency, and fair session corpus scan progress as the corpus grows.

## Best Possible Solution

Finish this in memory-core and session export: keep the existing generated-noise filters, add deterministic content-level dedupe for repeated session snippets, compute bounded content/source/recency-aware ingestion scores before recall recording, and add a many-session-file regression proving fair progress under a tight sweep cap.
