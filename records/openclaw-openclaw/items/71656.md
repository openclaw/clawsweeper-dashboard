## Summary

Keep open. Current main has partial transcript-noise filtering, but the broader Dreaming corpus-quality problem remains: session and daily ingestion still use fixed source scores, session corpus batches are recorded as generic daily signals, and light-phase selection can still favor freshly touched entries over older session-corpus material.

## What This Changes

Requests memory-core Dreaming to pre-filter noisy session corpus content, add content-aware and recency-aware scoring, and rotate scan budget so session-corpus entries continue advancing.

## Best Possible Solution

Keep this as the broad Dreaming corpus-quality tracker, decide the session-corpus promotion and scoring contract, then split narrow fixes for fair phase selection, conservative denoise, and content-aware ranking across the linked memory-core issues.

## Reproduction Assessment

Yes at source level: fixed session/daily source scores, generic session daily signals, and freshness-first light selection are present on current main and v2026.5.18. The exact April 9 stall was not live-replayed because it depends on the reporter's private memory/.dreams corpus state.

## Solution Assessment

Unclear. The requested direction is plausible, but the safest implementation needs maintainer agreement on whether session-corpus lines are promotion evidence and which content signals should affect Dreaming ranking.
