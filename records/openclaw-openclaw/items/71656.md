## Summary

Keep open. Current main filters some generated session noise, but the remaining memory-core dreaming concerns are still present: daily/session ingestion scores are fixed constants, session dedupe is scoped by timestamp or line metadata, and session files are scanned in sorted order under a global cap without persisted fair rotation.

## What This Changes

Improve memory-core dreaming by filtering and deduping session-corpus noise, weighting ingestion scores by content and recency, and rotating session scans fairly under the sweep cap.

## Best Possible Solution

Finish a narrow memory-core and memory-host repair that preserves existing generated-noise filters, adds content-level session dedupe, derives bounded ingestion scores from content/source/recency signals, and proves fair progress across many session files under a constrained sweep cap.

## Reproduction Assessment

Yes for the remaining source-level defects: fixed ingestion scores, metadata-scoped dedupe, and capped sorted session scanning are directly visible on current main. The exact April 9 production stall is unclear without the reporter's local memory artifacts.

## Solution Assessment
