## Summary

## What This Changes

The branch changes builtin memory status to infer vector-store readiness from persisted `vectorDims` metadata and adds memory-core regressions for indexed, chunks-only, and disabled-vector cases.

## Best Possible Solution

Land this as the canonical fix after maintainer acceptance of metadata-backed plain status semantics and normal CI completion, keeping `--deep` as the live vector-store probe.

## Reproduction Assessment

Yes by source inspection. Plain `memory status` uses a status-purpose manager, skips vector probing unless `--deep` or `--index` is set, and current main renders undefined builtin vector-store availability as `unknown`.

## Solution Assessment

Yes. Persisted `vectorDims` is a narrower signal than chunk rows, the fallback now honors `vector.enabled`, and the PR keeps live extension probing in the existing deep status path.
