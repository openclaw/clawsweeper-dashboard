## What This Changes

## Best Possible Solution

Land one canonical memory-core fix that uses a vector-specific persisted readiness signal, preserves the fast status path, covers FTS-only/degraded cases, and includes redacted real CLI proof.

## Reproduction Assessment

Yes by source inspection. Plain `memory status` skips vector probing, current main renders undefined store availability as `unknown`, and the PR's fallback can be checked against the existing FTS-only chunk write path.

## Solution Assessment

No. The status fallback should use a vector-specific persisted signal, not any row in `chunks`, because chunks are also written for FTS-only and degraded indexes.
