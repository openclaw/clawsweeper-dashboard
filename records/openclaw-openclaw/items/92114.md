## What This Changes

The PR changes memory-core builtin status to report vector-store readiness from persisted `vectorDims` metadata and adds regression tests for indexed, chunks-only, and disabled-vector states.

## Best Possible Solution

Land this as the canonical memory-core fix if maintainers accept metadata-backed fast status, then let the linked issue close from the PR.

## Reproduction Assessment

Yes by source inspection: plain `memory status` uses a status-purpose manager, skips vector probing unless deep/index is requested, and current main renders undefined `storeAvailable` as `Vector store: unknown`. I did not run a live failing current-main CLI repro in this read-only review.

## Solution Assessment

Yes, this is the best current fix shape if maintainers accept the semantics: `vectorDims` is a vector-specific persisted signal, and the branch rejects chunks-only and disabled-vector false positives. The older chunks-row fallback is weaker because FTS-only chunks can exist without a usable vector index.
