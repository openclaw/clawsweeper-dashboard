## Summary

Keep open. Current main still suppresses ordinary group-chat text tool summaries through a hard-coded dispatch gate, and the channel schemas do not expose a typed opt-in config; the adjacent open PR only covers `/verbose` behavior, not the requested configuration policy.

## What This Changes

Requests a channel/group configuration option to opt shared chats into receiving verbose tool-call result summaries while keeping suppression disabled by default.

## Best Possible Solution

Add a typed, fail-closed group tool-result visibility policy in channel defaults plus channel/group/topic overrides, feed it through normal dispatch and ACP projection, preserve media and exec-approval exceptions, and document the trusted-room model.

## Reproduction Assessment

Yes for the current behavior: source inspection and existing regression coverage show ordinary group text tool summaries are dropped while media-only and exec-approval payloads can still pass. I did not run tests because this was a read-only review, but the current-main code path is direct.

## Solution Assessment

Unclear. The requested config direction is plausible, but maintainers still need to choose naming, precedence, and interaction with `/verbose` and #44027 before implementation.
