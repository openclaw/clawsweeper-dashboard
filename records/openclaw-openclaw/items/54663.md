## Summary

Keep open. Current main still suppresses ordinary group-chat text tool summaries with a hard-coded dispatch gate, has no schema-backed channel-default or Telegram group/topic opt-in, and the adjacent open PR #44027 addresses `/verbose` behavior rather than the requested typed configuration policy.

## What This Changes

The issue requests an opt-in configuration policy that allows verbose tool-call results to be delivered in group chats while keeping group delivery disabled by default.

## Best Possible Solution

Add a narrow, typed group tool-result delivery policy that defaults to false, supports channel defaults plus channel/group/topic overrides, feeds the resolved policy into normal dispatch and ACP projection, preserves media and exec-approval exceptions, covers ordinary groups and forum topics with regression tests, and documents the trusted-group disclosure model.
