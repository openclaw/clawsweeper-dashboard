## Summary

Keep open. Current main still lacks a typed config-backed opt-in for ordinary group-chat tool-result delivery: the dispatch path suppresses text-only tool summaries for non-forum groups, the config schemas expose no equivalent option, and open #44027 only covers adjacent `/verbose` behavior rather than the requested configuration policy.

## What This Changes

Request a fail-closed configuration option that allows trusted group chats to receive verbose tool-result summaries while preserving the current suppressed default.

## Best Possible Solution

Add a narrow, typed, fail-closed group tool-result delivery policy across channel defaults and channel/group/topic overrides, wire it through normal dispatch and ACP projection, preserve media and exec-approval exceptions, and document the trusted-room model.

## Reproduction Assessment

Yes. Source inspection shows ordinary group sessions set `shouldSendToolSummaries` false and drop text-only tool results, and existing tests assert that suppression while preserving media-only and forum-topic exceptions.

## Solution Assessment

Unclear. The requested config direction is plausible, but the option name, precedence rules, `/verbose` interaction, and shared-room disclosure policy need maintainer judgment before implementation.
