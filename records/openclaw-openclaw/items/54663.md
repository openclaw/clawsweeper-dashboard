## Summary

Keep open. Current main still hard-codes ordinary group chats to suppress text tool-result summaries, has no schema-backed channel default or Telegram group/topic override for the requested opt-in, and the adjacent open PR #44027 addresses `/verbose` behavior rather than the requested typed configuration policy.

## What This Changes

The issue requests a fail-closed config option that lets trusted group chats opt in to receiving verbose tool-call results and summaries.

## Best Possible Solution

Add a narrow, typed group tool-result delivery policy that defaults to false, supports channel defaults plus channel/group/topic overrides, feeds the resolved policy into normal dispatch and ACP projection, preserves media and exec-approval exceptions, includes ordinary group and forum-topic regression tests, and documents the trusted-room security model.

## Reproduction Assessment

Yes. A focused dispatch unit path can reproduce it by invoking `onToolResult` for a Telegram `ChatType: "group"` context and observing that text-only payloads are not delivered; the current test at `src/auto-reply/reply/dispatch-from-config.test.ts:1323` already codifies that behavior.

## Solution Assessment

Unclear. The requested config direction is plausible, but the best fix needs maintainer judgment on naming, precedence, and interaction with `/verbose` and PR #44027 before an automated repair should choose a policy.
