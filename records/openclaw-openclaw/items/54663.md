## Summary

Keep open. Current main still suppresses text-only tool summaries in ordinary group chats and exposes no typed channel-default, Telegram group, or topic config option for opting back in; because this changes shared-chat disclosure policy and overlaps adjacent #44027, it needs maintainer product and security judgment before implementation.

## What This Changes

Request adding a config option to opt selected group chats into verbose tool-result delivery while keeping group suppression as the default.

## Best Possible Solution

Add a typed, fail-closed group tool-result visibility policy with channel defaults plus group/topic overrides, wire it through dispatch and ACP, preserve media and exec-approval exceptions, and document the trusted-room model.

## Reproduction Assessment

Yes, for the current behavior. Source inspection and existing regression coverage show an ordinary Telegram group text-only tool result is dropped while media-only payloads still deliver; the requested config opt-in does not exist on current main.

## Solution Assessment

Unclear. A typed config option is a plausible direction, but the exact option name, precedence, channel/topic scope, and relationship to `/verbose` and #44027 need maintainer policy judgment first.
