## Summary

Keep open. Current main still suppresses ordinary group-chat tool-result text and has no typed config option for channel defaults or Telegram group/topic overrides; the adjacent open PR only covers `/verbose` behavior and does not settle the requested configuration policy.

## What This Changes

Request a fail-closed configuration option that allows verbose tool-result delivery in group chats while preserving the current default suppression.

## Best Possible Solution

Add a typed, fail-closed group tool-result delivery policy with channel defaults plus group/topic overrides, wire it through dispatch and ACP, preserve media and exec-approval exceptions, and document the trusted-room model.

## Reproduction Assessment

Yes. Source inspection and existing regression coverage show ordinary group text tool results are suppressed, media-only results still deliver, and no current config surface can opt groups back in.

## Solution Assessment

Unclear. The requested config direction is plausible, but maintainers need to choose the disclosure policy, option name, precedence, and relationship to `/verbose` before implementation.
