## Summary

Keep open. Current `main` still has no schema-backed config policy for opt-in group-chat tool-result delivery: ordinary group text summaries are suppressed by a hard-coded gate, while related #44027 only proposes `/verbose` behavior and does not add the requested typed configuration surface.

## What This Changes

Add a channel default and Telegram group/topic configuration option that lets trusted group chats opt in to verbose tool-result delivery while preserving the current default suppression.

## Best Possible Solution

Add a narrow, typed, fail-closed group tool-result delivery policy with channel defaults plus Telegram group/topic overrides, wire it through normal dispatch and ACP projection, preserve media and exec-approval exceptions, and document the trusted-room model.

## Reproduction Assessment

Yes. Source inspection and existing regression coverage show that a Telegram `ChatType: "group"` text-only tool result is dropped while media-only payloads still deliver; no live run was needed for that code-path proof.

## Solution Assessment

Unclear. The requested config direction is plausible, but the exact option name, precedence rules, and relationship to `/verbose` and #44027 need maintainer judgment before implementation.
