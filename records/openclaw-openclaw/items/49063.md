## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Require explicit auth when access groups are off:** `extensions/telegram/src/bot-native-commands.ts:344`
  - body: For channel posts, this branch builds an empty authorizer list when `useAccessGroups` is false. `resolveCommandAuthorizedFromAuthorizers` treats no configured authorizers as allow under `modeWhenAccessGroupsOff: "configured"`, so any channel the bot can read can execute slash commands when `commands.allowFrom` is unset, contradicting the explicit-chat boundary.
  - confidence: 0.9
- **[P1] Populate sender_chat before commands.allowFrom auth:** `extensions/telegram/src/bot-native-commands.ts:599`
  - body: Using raw `ctx.msg` sends the original channel post into `resolveTelegramCommandAuth`, which still derives `senderId` only from `msg.from`. Telegram channel posts usually identify the channel via `sender_chat`, and the normal channel_post pipeline synthesizes that into `from`; without the same normalization, `commands.allowFrom` and owner command allowlists can reject the intended channel identity.
  - confidence: 0.86
- **[P1] Honor per-channel policy before allowing channel commands:** `extensions/telegram/src/bot-native-commands.ts:284`
  - body: The changed group-like policy call still uses the fallback-only policy path and the later channel authorizer accepts `channelPolicy?.groupConfig` as enough. A channel entry with its own disabled or allowlist policy can therefore execute native/plugin commands even though the normal inbound channel_post path applies per-channel groupPolicy and sender checks.
  - confidence: 0.8
- **[P1] Route channel callback follow-ups through channel sessions:** `extensions/telegram/src/bot-native-commands.ts:599`
  - body: Once `/models` or another interactive native command can start from a channel post, the callback handler still treats `chat.type === "channel"` as non-group and resolves authorization/session state with direct-style rules. Button follow-ups can be dropped for users who are not separately DM-authorized or routed to the clicking user's direct session instead of the channel session.
  - confidence: 0.82
