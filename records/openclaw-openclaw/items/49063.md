  - body: For `channel_post` with `useAccessGroups=false` and no `commands.allowFrom`, this branch passes an empty authorizer list into configured-mode command gating.   - confidence: 0.92

Telegram channel posts usually identify the posting channel through `sender_chat`, so `commands.allowFrom` and owner-style command allowlists can reject intended channels or push operators toward broader policy.
  - confidence: 0.88

  - body: The new channel-post authorizer treats any explicit `groupConfig` as enough to authorize the command after policy access, while policy evaluation ignores topic/group overrides. A configured channel whose own policy is disabled or narrowed can therefore still execute native/plugin commands after this PR enables the surface.
  - confidence: 0.84
- **[P2] Route channel command callbacks through channel sessions:** `extensions/telegram/src/bot-native-commands.ts:602`
  - body: Enabling channel native commands also enables inline-button commands such as `/models`, but the existing callback path still treats channel chats as non-group and resolves authorization/state through direct-session rules. Button follow-ups can be dropped or applied to the wrong session.
  - confidence: 0.82
