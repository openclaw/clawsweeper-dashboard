## Summary

Keep open: the PR addresses a real Telegram channel_post native-command gap, but the current branch is conflicting and still has security-sensitive channel authorization and callback/session blockers plus no inspected live Telegram proof.

## What This Changes

The PR changes Telegram native and plugin command handlers to use grammY ctx.msg, treats channel_post messages as group-like command contexts, updates sequential/update keys for msg, and adds unit coverage for allowed and denied channel_post command authorization.

## Best Possible Solution

Rebase and keep the ctx.msg entry only after channel_post commands use the same chosen channel identity, allowFrom policy, callback authorization, and session routing as Telegram channel posts, with focused tests and redacted live Telegram proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current main and v2026.6.8 read ctx.message for native commands, while channel_post command contexts require ctx.msg. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

No, this is not the best fix as-is. The ctx.msg entry is the right direction, but the maintainable fix must also preserve the selected channel_post sender/allowFrom rule and channel callback/session behavior before merge.
