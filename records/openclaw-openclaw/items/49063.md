## Summary

Keep open. The direction is useful, but the PR expands Telegram channel command execution before matching the existing channel_post authorization and callback-routing boundaries, and it still lacks required live Telegram proof.

## What This Changes

The PR changes Telegram native command handlers and tests so channel_post slash commands can run from explicitly allowed Telegram channel chats.

## Best Possible Solution

Rework native channel commands to reuse the existing channel_post normalization, explicit channel config checks, groupPolicy/sender authorization, and callback session routing, then attach live Telegram proof.

## Reproduction Assessment

Yes, source-reproducible. Current main's native command handlers read ctx.message, while channel_post updates go through a separate normalized message pipeline; the PR changes the native path to ctx.msg and adds channel-specific authorization.

## Solution Assessment

No. The current patch enables the right surface but should not diverge from the existing channel_post authorization, sender_chat normalization, groupPolicy, and callback-routing semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Require explicit channel config for channel posts:** `extensions/telegram/src/bot-native-commands.ts:333-344`
  - body: When useAccessGroups is false and commands.allowFrom is unset, channel_post commands build an empty authorizer list; resolveCommandAuthorizedFromAuthorizers(..., "configured") treats that as allow. Current channel_post messages require a configured group before processing, so this expands native/plugin command execution to unconfigured channels where the bot is present.
  - confidence: 0.88
- **[P2] Populate sender_chat before commands.allowFrom checks:** `extensions/telegram/src/bot-native-commands.ts:241-245`
  - body: Channel-post contexts carry the posting identity in sender_chat, but this auth path derives SenderId only from msg.from and marks the context as group-like. command-auth will not fall back to From for group contexts, so an allowed commands.allowFrom.telegram entry for the channel sender cannot match.
  - confidence: 0.84
- **[P2] Route channel command callbacks through channel auth:** `extensions/telegram/src/bot-native-commands.ts:602`
  - body: Using ctx.msg makes interactive native commands like /models reachable from channel posts, but callback_query handling still treats chat.type === "channel" as non-group and authorizes/routes the follow-up through DM rules and session keys. Button clicks from channel commands can be dropped or applied to the wrong session.
  - confidence: 0.78
