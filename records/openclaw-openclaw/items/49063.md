## Summary

Keep open: the branch addresses a real Telegram channel_post command gap that is still present on current main and v2026.6.8, but it is not merge-ready because the PR widens a command execution surface while leaving channel sender authorization, callback/session routing, branch conflicts, and real Telegram proof unresolved.

## What This Changes

The branch switches Telegram native/plugin command handlers to use grammY ctx.msg for channel_post updates, treats channel posts as group-like command contexts, and adds unit coverage for channel_post authorization plus sequential/update keys.

## Best Possible Solution

Rebase the branch, preserve explicit channel config plus sender_chat/allowFrom behavior, route channel callbacks through channel-scoped auth/session state, and require redacted live Telegram proof before merge.

Yes, from source: current main and v2026.6.8 read ctx.message in grammY command handlers even though grammY command matching includes channel_post and exposes those updates through ctx.msg. I did not run a live Telegram reproduction in this read-only review.
