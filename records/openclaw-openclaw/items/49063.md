  - confidence: 0.9

  - body: The handlers now pass `ctx.msg`, so channel posts reach `resolveTelegramCommandAuth`, but auth still reads identity from `msg.from`. Channel posts usually carry the posting channel in `sender_chat`; without the same normalization used by the normal channel_post pipeline, `commands.allowFrom` and owner command allowlists reject or check the wrong sender for intended channel commands.
  - confidence: 0.88

  - body: Once channel_post commands can emit inline keyboards, callback_query follow-ups still classify `chat.type === "channel"` as non-group in the current runtime. That makes `/models` and argument-menu buttons use direct-message authorization/session routing, so channel button clicks can be dropped or mutate the wrong session.
