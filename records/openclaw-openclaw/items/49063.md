  - body: This branch adds a separate `commandAuthorizers` path for channel posts. Current main routes native commands through `resolveTelegramCommandIngressAuthorization`, which carries owner, `commands.allowFrom`, access-group, and group allowlist behavior; this fork can allow or deny a different sender set than the canonical gate.
  - confidence: 0.87
- **[P1] Populate sender_chat before commands.allowFrom:** `extensions/telegram/src/bot-native-commands.ts:241-246`
  - body: For channel posts, Telegram may leave `from` empty and provide the channel identity in `sender_chat`. This branch sets a group command context but leaves `SenderId` empty, and command auth does not fall back to `From` for non-direct chats, so an explicitly allowed channel can still fail whenever `commands.allowFrom.telegram` is configured.
  - confidence: 0.86
- **[P1] Keep channel callbacks in channel context:** `extensions/telegram/src/bot-native-commands.ts:602`
  - body: Switching native commands to `ctx.msg` makes interactive commands such as model menus reachable from channel posts, but the existing callback handler still treats `chat.type === "channel"` as non-group and resolves auth/session from the clicking user. Those follow-ups can be dropped or mutate the wrong direct session.
  - confidence: 0.82
