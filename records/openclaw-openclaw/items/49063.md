## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Require explicit channel authorization when access groups are off:** `extensions/telegram/src/bot-native-commands.ts:333-344`
  - body: For channel posts, this branch sets `commandAuthorizers` to `[]` whenever `useAccessGroups` is false. `resolveCommandAuthorizedFromAuthorizers(..., modeWhenAccessGroupsOff: "configured")` treats an empty authorizer list as allow, so any `/...` post in any channel the bot can read becomes authorized, while the existing channel_post message path requires a configured channel entry. Preserve the explicit-chat requirement before enabling native commands on this surface.
  - confidence: 0.87
- **[P2] Populate sender_chat before commands.allowFrom checks:** `extensions/telegram/src/bot-native-commands.ts:241-246`
  - body: Raw channel_post updates usually carry the channel actor in `sender_chat`, but this auth context still gets `SenderId` only from `msg.from` and marks the chat as group-like. The command auth helper does not fall back to `From` for non-direct chats, so `commands.allowFrom.telegram` or owner allowlists can reject every channel-post command even when the channel identity is the intended allowed sender. Normalize `sender_chat` like the regular channel_post handler before resolving command auth.
  - confidence: 0.82
- **[P1] Route channel command callbacks through channel sessions:** `extensions/telegram/src/bot-native-commands.ts:602`
  - body: Switching native command handlers to `ctx.msg` makes channel_post commands with inline buttons reachable, such as model or argument menus. The callback handler still treats channel chats as non-groups and resolves model/plugin callback authorization and session state through the clicking user's direct path, which can drop the follow-up or mutate the wrong session. Update callback routing for channel chats before enabling interactive channel native commands.
  - confidence: 0.78
