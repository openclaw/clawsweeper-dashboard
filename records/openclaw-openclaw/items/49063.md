  - confidence: 0.87

  - body: The channel-post authorizer treats an explicit `groupConfig` as sufficient and ignores `effectiveGroupAllow`/`groups[chatId].allowFrom`. A configured channel that narrows commands to specific sender IDs can therefore authorize every channel post once the chat itself is listed, unlike the current channel_post message pipeline that enforces sender allowlists.
  - confidence: 0.86
- **[P1] Route channel command callbacks through channel context:** `extensions/telegram/src/bot-native-commands.ts:860`
  - body: Enabling plugin commands from channel posts also makes commands with inline buttons reachable, but the callback handler still classifies `chat.type === "channel"` as non-group and resolves auth/session state through the direct callback path. Button follow-ups can be dropped for channel users or mutate the clicking admin's direct session instead of the channel session.
