## Summary

## What This Changes

The PR changes Telegram native command handling to read ctx.msg, authorize channel_post slash commands through channel/group policy, and adds channel-post command and sequential-key tests.

## Best Possible Solution

Rework the Telegram channel_post command path so channel commands require the same explicit channel boundary as normal channel_post ingestion, preserve sender_chat-based command allowlists, and update callback/session handling or block interactive commands until they are channel-safe.
