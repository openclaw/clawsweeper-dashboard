## Summary

Keep open: the branch targets a real source-visible Telegram gap, but it still changes command authorization without live proof and has blocking channel-post auth bugs.

## What This Changes

The PR switches Telegram native/plugin command handlers to `ctx.msg`, treats channel posts as group-like command contexts, and adds channel-post auth/update-key/sequential-key tests.

## Best Possible Solution

Rebase onto current main's Telegram ingress/auth helpers, keep channel commands limited to the intended explicit channel policy, honor configured sender allowlists, add focused auth coverage, and attach live Telegram proof.

## Reproduction Assessment

Yes, source-level. grammY command handlers can receive `channel_post` and `ctx.msg` includes it, while current main reads `ctx.message` and exits before authorization; I did not run a live Telegram session.

## Solution Assessment

No. Using `ctx.msg` is the right dependency direction, but this branch should be rebuilt on current ingress/auth helpers and must preserve explicit channel config plus `commands.allowFrom` and per-channel allow semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Require explicit chat authorization for channel posts:** `extensions/telegram/src/bot-native-commands.ts:341-344`
  - body: In the channel-post branch, `useAccessGroups=false` leaves `commandAuthorizers` empty, and the configured-mode resolver treats that as allow. Because this direct command path bypasses the existing `channel_post` handler's configured-group guard, `/...` commands can run from channels that were not explicitly allowed.
  - confidence: 0.88
- **[P2] Use sender_chat for channel command allowlists:** `extensions/telegram/src/bot-native-commands.ts:352-353`
  - body: When `commands.allowFrom` is configured, authorization now depends on `commandsAllowFromAccess`, but the context still derives sender identity only from `msg.from`. Channel posts can carry the channel identity in `sender_chat`, and command auth does not fall back to `From` for group contexts, so allowed channel posts are rejected.
  - confidence: 0.86
