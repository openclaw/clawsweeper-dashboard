## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Require explicit channel auth when access groups are off:** `extensions/telegram/src/bot-native-commands.ts:333-344`
  - body: With `useAccessGroups` false and no `commands.allowFrom`, channel posts get an empty `commandAuthorizers` list. The shared configured-mode gate treats that as allowed, so this enables slash commands from any readable Telegram channel instead of only explicitly allowed chats.
  - confidence: 0.87
- **[P1] Keep channel command callbacks on the channel path:** `extensions/telegram/src/bot-native-commands.ts:602`
  - body: Switching command handlers to `ctx.msg` makes channel-post commands that emit inline buttons reachable, but the existing callback handler still treats `chat.type === "channel"` as non-group and routes authorization/session state through the clicker's direct path. Interactive commands such as `/models` can be blocked for the wrong reason or mutate the wrong session.
  - confidence: 0.82
- **[P2] Honor per-channel groupPolicy for channel commands:** `extensions/telegram/src/bot-native-commands.ts:299-302`
  - body: The channel-post path disables sender allowlist enforcement and then allows based on `channelPolicy?.groupConfig`, but the policy call does not apply topic/group overrides. A channel entry with its own `groupPolicy: disabled` or stricter allowlist can still become command-capable once the chat has any groupConfig.
  - confidence: 0.74
