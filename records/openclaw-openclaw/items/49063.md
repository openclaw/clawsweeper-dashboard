## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Use channel sender identity for commands.allowFrom:** `extensions/telegram/src/bot-native-commands.ts:351-352`
  - body: This branch makes configured `commands.allowFrom` the sole auth source, but channel posts usually expose the posting channel as `sender_chat` while `msg.from` may be empty. Because command auth does not fall back to group `From` for non-direct chats, `commands.allowFrom.telegram: [<channel id>]` can still reject an explicitly allowed channel. Populate a channel sender identity or define/test a fail-closed policy.
  - confidence: 0.88
- **[P1] Keep channel menu callbacks on the channel session path:** `extensions/telegram/src/bot-native-commands.ts:333-344`
  - body: Authorizing channel posts here also makes native commands with inline menus reachable from channels. The existing callback path classifies channel callback messages as non-group, so `/models` or arg-menu button clicks can be dropped by DM auth or mutate the wrong session. Update callback authorization and session routing for channel-originated native menus before enabling this surface.
  - confidence: 0.82
