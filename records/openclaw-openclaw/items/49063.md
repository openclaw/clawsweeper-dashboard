## Summary

## What This Changes

The PR changes Telegram native and plugin slash-command handlers to read grammY `ctx.msg`, handle `channel_post` inputs, add channel-post command authorization logic, and add focused auth/sequential-key tests.

## Best Possible Solution

Rebase the PR and make native/plugin channel-post commands reuse the existing channel_post authorization, sender identity, group-policy, callback, and session semantics, then show redacted live Telegram proof for allowed and denied paths.

## Reproduction Assessment

Yes at source level: grammY command middleware includes channel posts and current main uses `ctx.message`, so channel_post command contexts return before auth. The live Telegram before/after artifact is still missing.

## Solution Assessment

No. Switching to `ctx.msg` is the right entry-point direction, but this patch is not the best complete fix until channel_post auth, sender identity, callbacks, and session routing are aligned with the existing pipeline.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Require explicit channel auth when access groups are off:** `extensions/telegram/src/bot-native-commands.ts:333-344`
  - body: For channel posts with `useAccessGroups` off and no `commands.allowFrom`, this branch passes an empty authorizer list into `configured` mode, which resolves to allow. That means any channel the bot can read can execute native/plugin slash commands, contradicting the PR's explicitly-allowed-chat boundary.
  - confidence: 0.9
- **[P1] Use sender_chat before command allowlists:** `extensions/telegram/src/bot-native-commands.ts:227-247`
  - body: Channel posts normally identify the posting channel through `sender_chat`, but this auth path only derives `senderId` from `msg.from`. As a result, `commands.allowFrom`, owner allowlists, and per-chat sender allowlists can reject the intended allowed channel or fail to enforce the configured sender identity.
  - confidence: 0.87
- **[P1] Route channel command callbacks through channel state:** `extensions/telegram/src/bot-native-commands.ts:602`
  - body: Enabling `ctx.msg` here makes interactive commands such as model menus reachable from channel posts, but the existing callback handler still treats `chat.type === "channel"` as non-group and resolves direct-session auth/state. Follow-up button clicks can be dropped for channel users or mutate the wrong session.
  - confidence: 0.84
