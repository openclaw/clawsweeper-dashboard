  - body: For channel posts with access groups off and no `commands.allowFrom`, this branch builds an empty authorizer list; `resolveCommandAuthorizedFromAuthorizers(..., "configured")` treats that as allowed, so slash commands can run from any channel the bot can read instead of only explicitly allowed chats.
  - confidence: 0.9

  - body: When `commands.allowFrom.telegram` or owner command allowlists are configured, authorization still uses `senderId` from `msg.from`; channel posts usually carry the channel identity in `sender_chat`, so intended allowed channel senders will be rejected.
  - confidence: 0.88
- **[P1] Route channel callbacks through the channel session:** `extensions/telegram/src/bot-native-commands.ts:341`
  - body: Enabling channel_post native commands also exposes inline-button commands like `/models`, but callback handling still treats channel chats as direct chats, so button clicks can be dropped unless the user is DM-authorized or can mutate the wrong direct session.
  - confidence: 0.84
