## What This Changes

The PR changes Telegram native/plugin command handlers to read grammY `ctx.msg`, adds channel_post command authorization branches and tests, and includes `ctx.msg` in Telegram update/sequential keys.

## Best Possible Solution

Refresh on current main, reuse the shared Telegram channel_post ingress/authorization normalization for native commands, settle the sender-scoped policy with the open related issue, and prove allowed/denied plus callback behavior in a real Telegram session.

## Reproduction Assessment

Yes, source-level. Current main's native command handlers require `ctx.message`, while grammY exposes channel posts through `ctx.msg`; I did not run a live Telegram session in this read-only review.

## Solution Assessment

No, not as written. The `ctx.msg` direction is plausible, but the maintainable fix should reuse the existing channel_post normalization and shared ingress authorization path and update callback/session handling before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Require explicit channel auth when access groups are off:** `extensions/telegram/src/bot-native-commands.ts:333-344`
  - body: For channel posts with `useAccessGroups=false`, this branch builds an empty authorizer list and then calls `resolveCommandAuthorizedFromAuthorizers` in `configured` mode, which treats no configured authorizer as allowed. That lets any readable Telegram channel run native/plugin commands when `commands.allowFrom` is unset, contradicting the explicitly allowed-chat boundary.
  - confidence: 0.89
- **[P1] Normalize sender_chat before channel command auth:** `extensions/telegram/src/bot-native-commands.ts:183-186`
  - body: After channel posts enter this auth path, sender identity still comes only from `msg.from`; the regular channel_post pipeline synthesizes `from` from `sender_chat` before authorization. Deployments that use `commands.allowFrom` or per-chat `allowFrom` for channel relay identity can reject allowed channels or evaluate the wrong sender policy.
  - confidence: 0.86
- **[P1] Route channel command callbacks through the channel session:** `extensions/telegram/src/bot-native-commands.ts:602`
  - body: Switching native commands to `ctx.msg` makes interactive channel commands such as `/models` reachable, but callback handling still treats `chat.type === "channel"` as non-group and resolves auth/session through direct-chat rules. Button clicks can be dropped or mutate the clicker's direct session instead of the channel conversation.
  - confidence: 0.82
