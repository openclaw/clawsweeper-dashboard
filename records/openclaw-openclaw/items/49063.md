## Summary

Keep this PR open, but it is not merge-ready: the central ctx.msg entry point is useful, while the branch still expands Telegram channel_post command execution without matching current channel_post sender, group-policy, callback/session, and real-proof requirements.

## What This Changes

The PR changes Telegram native and plugin slash-command handlers to read grammY ctx.msg, authorize channel_post commands via explicit chat checks, and adds focused auth/sequential-key tests.

## Best Possible Solution

Rebase and implement channel_post native commands through the same sender_chat, group-policy, and callback/session semantics as the current inbound channel_post pipeline, then add redacted live Telegram proof.

## Reproduction Assessment

Yes for the source-level behavior: grammY command middleware reaches channel posts, current main native handlers read ctx.message, and the PR switches that path to ctx.msg. No live Telegram after-fix artifact is available for the allowed and denied paths.

## Solution Assessment

No. Using ctx.msg is the right entry point, but the best fix should reuse or mirror the current channel_post sender, group-policy, and callback/session pipeline instead of adding a parallel authorization path.

Full review comments:

- **[P1] Use sender_chat before command allowlists:** `extensions/telegram/src/bot-native-commands.ts:227-228`
  - body: Channel posts usually do not carry the posting identity in msg.from; current main's channel_post pipeline synthesizes from/senderId from sender_chat before auth.   - confidence: 0.91
- **[P1] Keep channel group policy in the auth path:** `extensions/telegram/src/bot-native-commands.ts:299`
  - body: This disables sender allowlist enforcement for channel posts and later authorizes on groupConfig alone. That diverges from the current inbound channel_post path, which applies group/topic policy overrides before dispatch, so a channel entry with disabled or allowlist policy can still run native/plugin commands.
  - confidence: 0.88
- **[P1] Route channel command callbacks through channel state:** `extensions/telegram/src/bot-native-commands.ts:602`
  - body: Switching command handlers to ctx.msg makes channel-post commands with inline buttons reachable, but the callback path still classifies chat.type === "channel" as non-group/direct. Buttons from commands like model or arg menus can then be authorized and applied against the clicking admin's direct session instead of the channel conversation.
  - confidence: 0.85
