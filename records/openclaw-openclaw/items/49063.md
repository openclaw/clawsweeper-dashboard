  - body: For channel posts with `useAccessGroups` disabled, this branch produces `[]` authorizers; `resolveCommandAuthorizedFromAuthorizers(..., modeWhenAccessGroupsOff: "configured")` treats that as allowed.   - confidence: 0.9
- **[P1] Populate channel sender identity before allowFrom checks:** `extensions/telegram/src/bot-native-commands.ts:180-182`
  - body: Channel posts usually carry the posting identity in `sender_chat`, but this path still authorizes `commands.allowFrom` with `msg.from` only. Because channel posts are treated as group-like, the command auth code will not use `From` as a sender fallback, so non-wildcard command or owner allowlists reject even explicitly allowed channel senders.
  - confidence: 0.86
- **[P1] Route channel command callbacks through channel state:** `extensions/telegram/src/bot-native-commands.ts:580`
  - body: Switching native commands to `ctx.msg` makes channel `/models` and other button-producing commands reachable, but callback handling still classifies `chat.type === "channel"` as non-group and resolves model/menu state through the clicking user's direct session. That can drop follow-up clicks or mutate the wrong session.
  - confidence: 0.83
