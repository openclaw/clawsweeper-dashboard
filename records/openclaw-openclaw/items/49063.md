## Summary

Keep open: current main and v2026.6.8 still do not implement the central ctx.msg/channel_post native-command behavior, but this PR is not merge-ready because it expands Telegram command execution without resolving channel_post sender authorization, callback/session routing, or real Telegram proof.

## What This Changes

The PR changes Telegram native command authorization and update/session key selection so channel_post contexts can use ctx.msg and explicitly allowed Telegram channel chats can run native and plugin slash commands.

## Best Possible Solution

Keep the useful ctx.msg/channel_post direction, but land it only after maintainers choose the channel_post authorization policy and the PR covers sender_chat or senderless allowFrom behavior, channel callbacks/session routing, focused tests, docs if needed, and live allowed/denied Telegram proof.
