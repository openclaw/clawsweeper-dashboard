Full review comments:

- **[P1] Populate channel sender identity for command allowlists:** `extensions/telegram/src/bot-native-commands.ts:234-242`
  - body: For channel_post updates, Telegram can leave msg.from empty and put the channel identity in sender_chat. This path marks channel posts as group-like, but command auth still passes only msg.from as SenderId/SenderUsername, and the shared auth helper does not fall back to From for group chats. With commands.allowFrom.telegram configured, an explicitly allowed channel identity has no sender candidate and the newly enabled command is rejected instead of authorized.
  - confidence: 0.88
- **[P1] Keep channel callbacks on the channel session:** `extensions/telegram/src/bot-native-commands.ts:580`
  - body: Changing the handlers to ctx.msg enables button-producing commands such as /models from channel_post, but callback_query handling still treats channel chats as non-group/direct and resolves authorization/session state from the clicking admin. Those follow-ups will be dropped unless the admin is separately DM-authorized, or they can update the wrong direct session instead of the channel session.
  - confidence: 0.84
