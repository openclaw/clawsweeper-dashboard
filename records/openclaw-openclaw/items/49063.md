## Summary

Keep this PR open: current main still lacks native Telegram `channel_post` command support, but the proposed patch has auth/session regressions around raw channel messages and interactive callbacks that should be fixed before merge.

## What This Changes

The PR changes Telegram native and plugin slash command handlers to read `ctx.msg`, treats `channel_post` messages as group-like for command authorization, adds `ctx.msg` update-key support, and adds channel-post command auth tests.

## Best Possible Solution

Rework the PR on current main so native channel commands reuse the existing channel_post normalization and channel-like callback/session routing, while leaving the broader sender-scoped channel policy decision to #49062.

## Reproduction Assessment

Yes. Source inspection shows current main registers native commands from `ctx.message` only, while Telegram channel posts arrive through `ctx.channelPost`/`ctx.msg`; the normal channel_post text path confirms the missing normalization path.

## Solution Assessment

No. Switching to raw `ctx.msg` is the right entry point direction, but the patch should also synthesize `sender_chat` for command authorization and route channel callbacks through the channel/group session path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Populate channel-post sender before auth:** `extensions/telegram/src/bot-native-commands.ts:580-581`
  - body: Using `ctx.msg` lets raw `channel_post` messages reach command auth, but this path still derives `senderId` only from `msg.from`. The normal channel_post handler synthesizes `from` from `sender_chat`; without doing the same here, configured `commands.allowFrom.telegram` or owner allowlists cannot positively match the channel identity, so authorized channel commands are rejected.
  - confidence: 0.88
- **[P2] Keep channel callbacks on the channel session:** `extensions/telegram/src/bot-native-commands.ts:329-340`
  - body: After this authorizes channel-post commands, commands that emit inline keyboards become reachable in channels, but callback handling still treats `chat.type === "channel"` as non-group. Buttons such as `/models` follow-ups are then dropped by group-only inline scope or resolved against the clicking user's direct session instead of the channel session.
  - confidence: 0.82
