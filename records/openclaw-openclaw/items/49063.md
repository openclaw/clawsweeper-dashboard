- **[P1] Require explicit channel auth when access groups are off:** `extensions/telegram/src/bot-native-commands.ts:344`
  - body: For channel posts with `useAccessGroups: false` and no `commands.allowFrom`, this branch gives `resolveCommandAuthorizedFromAuthorizers` an empty authorizer list.   - body: When `commands.allowFrom.telegram` is configured, this branch makes it the sole auth source, but channel posts usually carry the actor in `sender_chat` while `senderId` is still read only from `msg.from`. Because command auth does not use the group `From` value as a sender fallback, an allowed channel sender cannot pass this check.
  - confidence: 0.87

  - body: This authorizes channel commands that can send inline keyboards, but the callback handler still treats `chat.type === "channel"` as non-group and then authorizes/routes callbacks with direct-session rules. Channel button follow-ups can be dropped or applied to the clicking user's direct session instead of the channel command session.
  - confidence: 0.82
