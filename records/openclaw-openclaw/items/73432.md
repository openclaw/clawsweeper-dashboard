## Summary

Keep open. Current main still prevents QMD embedding maintenance in the reporter's default lexical `search` mode, and open PR #73512 already references this issue with closing syntax and targets the same scheduler behavior, so this should wait for maintainer review of that PR rather than be closed or queued for duplicate repair.

## What This Changes

The issue asks OpenClaw to trigger QMD embedding automatically according to `memory.qmd.update.interval` and `memory.qmd.update.embedInterval` when QMD-backed memory files change.

## Best Possible Solution

Resolve the product contract in PR #73512: either merge a narrow scheduler/test/docs change that treats explicit `embedInterval` as an embedding opt-in, or keep lexical mode BM25-only and improve config/status guidance so users know to choose `vsearch` or `query` for automatic embedding.

## Reproduction Assessment

Yes. A high-confidence static and unit-level reproduction exists on current main: configure QMD with default or explicit `searchMode: "search"` plus `embedInterval`, advance the manager timer, and the current test expects no `update` or `embed` subprocess calls.

## Solution Assessment

Unclear. PR #73512 is a plausible narrow code fix for the observable problem, but it changes the documented BM25-only contract for lexical QMD mode and needs maintainer judgment before the issue should be closed.
