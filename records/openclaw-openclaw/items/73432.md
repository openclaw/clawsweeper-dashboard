## Summary

Keep open. Current main and the latest release still skip QMD embedding maintenance in lexical `search` mode before `embedInterval` can matter, and the remaining decision is whether an explicit existing `update.embedInterval` should override the documented BM25-only contract.

## What This Changes

Request that QMD automatically run embedding on the configured `memory.qmd.update.embedInterval` cadence when documents change, even when `searchMode` is omitted and defaults to lexical search.

## Best Possible Solution

Choose and document one QMD contract: keep lexical `search` embedding-free and direct users to `query` or `vsearch`, or carry explicit `update.embedInterval` intent through resolved config and update runtime, tests, and docs together.

## Reproduction Assessment

Yes. Source inspection reproduces the path: omit `memory.qmd.searchMode`, configure QMD update/embed intervals, and current main resolves lexical `search` then returns from embed execution and timer scheduling before the interval can matter.

## Solution Assessment

Unclear. A maintainable repair, if maintainers want it, should preserve explicit raw `embedInterval` intent and align docs/tests; otherwise the documented workaround is to use `query` or `vsearch` for automatic embedding maintenance.
