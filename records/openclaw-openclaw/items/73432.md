## What This Changes

The issue requests automatic QMD embedding maintenance from `memory.qmd.update.interval` or `memory.qmd.update.embedInterval` when the reporter leaves QMD `searchMode` at its default.

## Best Possible Solution

Resolve the contract in PR #73512: either land a narrow scheduler/config/test/docs change that treats an explicitly configured `embedInterval` as an embedding opt-in, or keep lexical mode BM25-only and update the issue/PR with guidance to set `searchMode` to `vsearch` or `query` for automatic embeddings.
