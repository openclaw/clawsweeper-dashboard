## Summary

Keep open. Current main still checks QMD vector search mode before considering `embedInterval`, so the reporter's omitted/default `searchMode` resolves to lexical `search` and prevents automatic embedding; open PR #73512 remains the unmerged closing implementation candidate.

## What This Changes

The issue requests automatic QMD embedding on `memory.qmd.update.embedInterval` when `searchMode` is omitted or defaults to `search`.

## Best Possible Solution

Resolve PR #73512 or an equivalent narrow scheduler change, with regression coverage and docs/config wording aligned to the chosen `embedInterval` semantics.

## Reproduction Assessment

Yes. Configure QMD with omitted/default `searchMode` or explicit `search` plus `embedInterval`, then advance the manager timer; current main's unit test asserts no `update` or `embed` subprocess calls occur.

## Solution Assessment

Unclear. The linked PR is the narrow implementation candidate for the observable bug, but the repository should first settle the conflict with the documented lexical-mode contract.
