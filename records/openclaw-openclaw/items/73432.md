## Summary

Keep open. Current main still gates QMD embedding maintenance behind vector search modes, while the reporter's omitted `searchMode` resolves to lexical `search`; open PR #73512 is the active unmerged fix candidate with closing syntax, so the issue should wait for that PR to be reviewed, landed, or closed.

## What This Changes

The issue asks OpenClaw to run QMD embedding automatically according to `memory.qmd.update.embedInterval` even when `searchMode` is omitted and defaults to `search`.

## Best Possible Solution

Resolve the scheduler contract through PR #73512 or an equivalent narrow change that updates QMD manager behavior, regression tests, and any docs/config help needed to make the `embedInterval` semantics clear.

## Reproduction Assessment

Yes. Configure QMD with omitted/default `searchMode` or explicit `search` plus `embedInterval`, then advance the manager timer; current main's unit test already asserts that no `update` or `embed` subprocess calls occur in that case.

## Solution Assessment

Unclear. PR #73512 is the right narrow implementation candidate for the observable bug, but the product contract needs maintainer confirmation because current docs explicitly say lexical `search` skips embedding maintenance.
