## Summary

Keep this issue open: current `main` and the latest release still skip `qmd embed` whenever QMD is in lexical `search` mode, and the directly linked fix remains open and unmerged at https://github.com/openclaw/openclaw/pull/73512.

## What This Changes

The issue asks OpenClaw to honor `memory.qmd.update.embedInterval` by automatically running QMD embedding even when `memory.qmd.searchMode` is omitted or lexical.

## Best Possible Solution

Maintain one explicit QMD contract: either land or rework https://github.com/openclaw/openclaw/pull/73512 and align docs/tests, or keep lexical `search` mode embedding-free and close this issue after that decision is documented.

## Reproduction Assessment

Yes, from source inspection: omit `memory.qmd.searchMode` or use `search`, configure `embedInterval`, and current code returns before scheduling or running `qmd embed`. I did not run a live QMD instance in this read-only review.

## Solution Assessment

Unclear: the linked PR is a plausible narrow code fix, but current docs and tests now encode lexical `search` mode as embedding-free. Maintainers should decide that contract before landing or rejecting the PR.
