## What This Changes

The issue reports that QMD automatic embedding is not triggered by `memory.qmd.update.interval` or `embedInterval` after OpenClaw updates memory documents.

## Best Possible Solution

Review and land PR #73512, or an equivalent narrow QMD scheduler change, with tests and docs aligned on the explicit `embedInterval` contract.

## Reproduction Assessment

Yes. Configure QMD with omitted/default `searchMode` or explicit `search` plus `embedInterval`, then advance the manager timer; current main's lexical-mode unit test already asserts that no `update` or `embed` subprocess calls occur.

## Solution Assessment

Unclear. PR #73512 is the narrow implementation candidate for the observable bug, but maintainers should reconcile it with the documented lexical-mode contract before resolving this issue.
