## What This Changes

The issue reports that QMD automatic embedding is not triggered when memory.qmd.update.interval or embedInterval is configured, leaving documents pending embedding until manual qmd embed is run.

## Best Possible Solution

Resolve the product contract in PR #73512: either merge a narrow scheduler, test, and docs update that treats explicit embedInterval as an embedding opt-in, or keep lexical mode BM25-only and improve config/status guidance so users know to choose vsearch or query for automatic embedding.
