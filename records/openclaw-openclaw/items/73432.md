## What This Changes

The issue requests automatic QMD embedding according to `memory.qmd.update.embedInterval` when `searchMode` is omitted or lexical `search`.

## Best Possible Solution

Review and either land https://github.com/openclaw/openclaw/pull/73512 or an equivalent explicit-intent fix, or reject that behavior and align docs/config help around using `query` or `vsearch` for embedding maintenance.

## Reproduction Assessment

Yes, source-reproducible. Omitted `searchMode` resolves to `search`, and current main returns from both embed execution and embed timer scheduling before a positive `embedIntervalMs` can matter.

## Solution Assessment

Unclear. The linked PR targets the right scheduler gap, but the maintainable fix needs a maintainer decision on lexical-mode semantics and should preserve whether `embedInterval` was explicitly configured.
