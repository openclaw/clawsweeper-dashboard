## Summary

Keep open. Current main still suppresses QMD embedding in the reporter's omitted/default `searchMode` path, and PR #73512 is the open, unmerged implementation candidate for the same scheduler behavior.

## What This Changes

The issue asks OpenClaw to automatically run `qmd embed` according to `memory.qmd.update.interval` or `embedInterval` after QMD-managed documents change.

## Best Possible Solution

Review and land PR #73512, or an equivalent narrow QMD manager/test/docs change, so explicit `embedInterval` semantics and the documented search-mode contract agree.

## Reproduction Assessment

Yes. Configure QMD with omitted/default `searchMode` or explicit `search` plus `embedInterval`, then advance the manager timer; current main's unit test already asserts that no `update` or `embed` subprocess calls occur.

## Solution Assessment

Unclear. PR #73512 is the narrow implementation candidate for the observable bug, but maintainers should reconcile it with the documented lexical-mode behavior before resolving the issue.
