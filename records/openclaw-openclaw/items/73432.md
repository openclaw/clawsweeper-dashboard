## What This Changes

The issue asks OpenClaw to automatically run QMD embedding according to `memory.qmd.update.embedInterval` when QMD-managed documents change.

## Best Possible Solution

Resolve the scheduler contract through PR #73512 or an equivalent narrow change that updates QMD manager behavior, regression tests, and docs/config help together.

## Reproduction Assessment

Yes. Configure QMD with omitted/default `searchMode` or explicit `search` plus `embedInterval`, then advance the manager timer; current main's code path and existing test show no embed subprocess is scheduled.

## Solution Assessment

Unclear. PR #73512 appears to target the right scheduler guard, but the docs currently define lexical `search` as skipping embedding maintenance, so the final behavior needs maintainer confirmation.
