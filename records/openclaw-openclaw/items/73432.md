## Summary

Keep this issue open: current main and v2026.6.6 still skip QMD embedding maintenance in lexical `search` mode even when `embedInterval` is set, the linked fix PR is now closed unmerged, and the remaining question conflicts with current docs/tests enough to need maintainer product judgment.

## What This Changes

The issue asks OpenClaw to automatically run `qmd embed` on the configured QMD update/embed cadence when the QMD backend is used without explicitly setting a vector-capable `searchMode`.

## Best Possible Solution

Maintainers should decide the QMD contract: either keep lexical `search` embedding-free and close this with the documented `query`/`vsearch` workaround, or add an explicit raw-config opt-in signal for `embedInterval` and update source, docs, and tests together.

## Reproduction Assessment

Yes. Source inspection reproduces the behavior: default or explicit `searchMode: "search"` makes both QMD embed execution and scheduling return before the configured `embedInterval` can trigger embedding.

## Solution Assessment

Unclear. The requested behavior is plausible for explicit `embedInterval`, but the maintainable fix needs a maintainer-approved contract and an explicit-config signal rather than bypassing the current BM25-only default.
