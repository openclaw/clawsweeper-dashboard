## Summary

Keep open. Current main still gates QMD embedding maintenance out when the effective search mode is lexical `search`, and an open implementation PR already references this issue with closing syntax, so the issue should stay open until that PR is reviewed, landed, rejected, or replaced.

## What This Changes

The issue asks OpenClaw to run QMD embedding automatically according to `memory.qmd.update.embedInterval` when users configure QMD without an explicit `searchMode`.

## Best Possible Solution

Review and either land https://github.com/openclaw/openclaw/pull/73512 or an equivalent explicit-intent fix, or reject that behavior and align docs/config help so users know to set `query` or `vsearch` for embedding maintenance.

## Reproduction Assessment

Yes. Source inspection shows omitted `searchMode` resolves to `search`, and current main returns from both embed execution and timer scheduling before the configured positive `embedInterval` can matter.

## Solution Assessment

Unclear. The linked PR targets the scheduler gap, but the durable fix needs a maintainer decision on whether explicit `embedInterval` overrides lexical-mode docs and should preserve explicitness through resolved config.
