## Summary

Keep open. Current main still blocks QMD embed scheduling in the reporter's omitted/default `searchMode` path, and open PR #73512 is the unmerged closing implementation candidate for the same scheduler behavior.

## What This Changes

The issue reports that QMD documents remain pending embedding because OpenClaw does not run `qmd embed` on the configured `memory.qmd.update.embedInterval` when `searchMode` is omitted/defaulted.

## Best Possible Solution

Review and land PR #73512, or an equivalent narrow change, with QMD scheduler tests and docs/config help aligned on the explicit `embedInterval` semantics.

## Reproduction Assessment

Yes. Source inspection shows omitted `searchMode` resolves to `search`, and the current manager/test path then prevents `qmd embed` from being scheduled or run even with a positive `embedInterval`.

## Solution Assessment

Unclear. PR #73512 is the narrow implementation candidate for the observed scheduler gap, but maintainers should first reconcile it with the documented lexical-mode contract.
