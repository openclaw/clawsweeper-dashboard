## Summary

Keep open. Current main and v2026.6.10 still skip QMD embedding maintenance in lexical `search` mode before `embedInterval` can matter, and https://github.com/openclaw/openclaw/pull/96092 is the active candidate fix but still needs maintainer product/compatibility judgment.

## What This Changes

The issue requests that QMD automatically run embedding on the configured `memory.qmd.update.embedInterval` after OpenClaw changes memory files.

## Best Possible Solution

Have memory/QMD owners choose the lexical-mode contract; if accepted, land a fix that preserves explicit interval intent, rejects malformed opt-ins, updates docs/tests, and includes redacted real QMD proof.

## Reproduction Assessment

Yes. Source inspection shows current main resolves `embedInterval`, but lexical `search` mode returns before the interval can schedule or run embedding; I did not run a live QMD process in this read-only review.

## Solution Assessment

Unclear as a product decision. Preserving explicit interval intent in resolved config is the likely narrow fix if maintainers accept the override; otherwise the supported path remains `query` or `vsearch` for automatic embedding maintenance.
