## Summary

Keep open. Current main still blocks QMD embed execution and scheduling in lexical `search` mode, and the linked implementation candidate at https://github.com/openclaw/openclaw/pull/73512 remains open with closing syntax, so this issue should stay open until that PR is landed, rejected, or replaced.

## What This Changes

The issue asks for QMD `embedInterval` to trigger periodic `qmd embed` after memory changes even when `searchMode` is omitted and resolves to lexical `search`.

## Best Possible Solution

Review and either land https://github.com/openclaw/openclaw/pull/73512 or an equivalent explicit-intent fix, or reject that behavior and align docs/config help so users know to use `query` or `vsearch` when they want embedding maintenance.

## Reproduction Assessment

Yes, source-reproducible. Omitted `searchMode` resolves to `search`, and current main returns from both embed run and timer scheduling before a positive `embedInterval` can matter.

## Solution Assessment

Unclear. The linked PR targets the right scheduler gap, but the durable solution needs a maintainer decision on whether explicit `embedInterval` overrides lexical-mode docs and, if so, should carry explicitness through resolved config.
