## Summary

Keep open. Current main and the latest release still skip QMD embedding maintenance in lexical `search` mode before `embedInterval` can matter, and no merged or open viable replacement owns the remaining contract decision.

## What This Changes

The issue asks OpenClaw to automatically trigger QMD embedding according to `memory.qmd.update.interval` or `memory.qmd.update.embedInterval` after memory documents change.

## Best Possible Solution

Keep one canonical maintainer-owned decision for the QMD lexical-mode contract and only change runtime behavior if maintainers decide explicit `embedInterval` should override `searchMode: "search"`.

## Reproduction Assessment

Yes by source inspection: current main parses `update.embedInterval`, but the QMD manager skips both scheduling and execution before checking it when `searchMode` is lexical `search`. I did not run a live QMD process in this read-only review.

## Solution Assessment

Unclear. A maintainable repair, if maintainers want it, should preserve explicit raw `embedInterval` intent and align runtime, docs, and tests; otherwise the supported path is to use `query` or `vsearch` for automatic embedding maintenance.
