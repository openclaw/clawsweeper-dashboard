## Summary

Keep open. Current main still has the source/test path that prevents QMD embedding in the omitted/default `searchMode` case, and https://github.com/openclaw/openclaw/pull/73512 is an open closing implementation candidate rather than a merged fix.

## What This Changes

The issue asks OpenClaw to run QMD embedding automatically according to `memory.qmd.update.embedInterval` when documents change.

## Best Possible Solution

Settle the lexical-mode QMD contract, then either land an adjusted fix that preserves explicit `embedInterval` intent or document that users must choose `vsearch` or `query` for embedding maintenance.

## Reproduction Assessment

Yes, source-level. Omitted `searchMode` resolves to `search`, `qmdUsesVectors()` rejects that mode, and the timer/run guards return before the configured `embedIntervalMs` can schedule or execute `qmd embed`.

## Solution Assessment

Unclear. The open PR is the right narrow implementation candidate to review, but its current approach needs maintainer product judgment because it conflicts with the documented BM25-only behavior and cannot distinguish explicit from default embed cadence.
