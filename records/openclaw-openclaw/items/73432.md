## Summary

Keep open. Current main still gates QMD embedding execution and embed-timer scheduling on vector-capable search modes, so default lexical `search` mode plus `embedInterval` does not run automatic embedding. Open PR #73512 is the active unmerged implementation candidate with closing syntax, so this issue should wait for that PR to land or be closed.

## What This Changes

The issue requests that QMD automatically trigger embedding according to `memory.qmd.update.embedInterval` when QMD is configured with the default or omitted `searchMode`.

## Best Possible Solution

Settle the QMD scheduler contract through PR #73512 or an equivalent narrow change: either explicit `embedInterval` schedules `qmd embed` regardless of `searchMode` with matching tests/docs, or lexical `search` remains no-embedding and the config/status guidance makes that requirement clear.

## Reproduction Assessment

Yes. A high-confidence static and unit-level reproduction exists: configure QMD with omitted/default `searchMode: "search"` and `embedInterval`, advance the manager timer, and current main's test expects no `update` or `embed` subprocess calls.

## Solution Assessment

Unclear. PR #73512 is the narrow implementation candidate for the observable bug, but it changes behavior that current docs and tests describe intentionally, so maintainer review should decide the contract before this issue is closed.
