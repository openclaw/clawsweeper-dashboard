## Summary

Keep open. Current main still resolves an omitted QMD search mode to lexical `search` and both embed execution and embed timer creation return before considering `embedInterval`; PR #73512 is still open, unmerged, and references this issue with closing syntax, so the issue should wait for that PR or a maintainer contract decision.

## What This Changes

The issue asks OpenClaw to run `qmd embed` automatically according to `memory.qmd.update.embedInterval` when QMD memory files change, even when `searchMode` is omitted and defaults to `search`.

## Best Possible Solution

Decide the QMD lexical-mode contract in PR #73512. If an explicit `embedInterval` should opt lexical mode into embedding, carry explicit config intent through the resolved QMD update config and update the scheduler, tests, docs, schema help, and changelog; otherwise keep the current BM25-only contract and document `query`/`vsearch` or manual `qmd embed` as the supported path.

## Reproduction Assessment

Yes. A high-confidence static and unit-level reproduction exists on current main: use QMD with omitted/default `searchMode: "search"` and configured `embedInterval`, then observe that the current guards and existing timer test prevent `qmd embed` from running.

## Solution Assessment

Unclear. PR #73512 is the right active place to resolve the issue, but the safest fix needs maintainer approval on whether explicit `embedInterval` overrides lexical mode and should avoid treating the resolved default interval as explicit user intent.
