## Summary

Keep open. Current main and the latest release still skip QMD embedding maintenance in lexical `search` mode before `embedInterval` can schedule or run, while the active candidate PR still needs memory/QMD owner direction and real runtime proof.

## What This Changes

The issue asks OpenClaw to automatically run QMD embedding on the configured update/embed cadence when memory changes leave QMD documents pending embedding.

## Best Possible Solution

Have memory/QMD owners choose the lexical-mode contract; if accepted, land a fix that preserves explicit interval intent, rejects malformed opt-ins, updates docs/tests, and includes redacted real QMD proof.

## Reproduction Assessment

Yes, source-level. Current main resolves `embedInterval`, but lexical `search` mode returns before the interval can schedule or run embedding; I did not run a live QMD process in this read-only review.

## Solution Assessment

Unclear as a product decision. Preserving explicit interval intent in resolved config is the likely narrow fix if maintainers accept the override; otherwise the supported path remains `query` or `vsearch` for automatic embedding maintenance.
