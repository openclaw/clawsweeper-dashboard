## Summary

Keep open. Current main still blocks QMD embed scheduling in the reporter's omitted/default `searchMode` path, and open PR #73512 is the unmerged closing implementation candidate for the same scheduler behavior.

## What This Changes

The issue asks OpenClaw to automatically run `qmd embed` according to `memory.qmd.update.embedInterval` when QMD-managed memory files change.

## Best Possible Solution

Land PR #73512 or an equivalent narrow fix that aligns QMD scheduler behavior, regression tests, and docs/config help around explicit `embedInterval` semantics.

## Reproduction Assessment

Yes. Source inspection on current main shows omitted `memory.qmd.searchMode` resolves to `search`, and both the timer and run paths return before considering a positive `embedIntervalMs`.

## Solution Assessment

Unclear. PR #73512 is the narrow implementation candidate for the observed scheduler gap, but maintainers should reconcile it with the documented lexical-mode contract before landing.
