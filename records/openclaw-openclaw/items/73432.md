## Summary

Keep open. Current main still matches the reported behavior for the reporter's omitted `searchMode` configuration, and open PR #73512 already references this issue with closing syntax and targets the same QMD embed scheduler guard.

## What This Changes

The issue requests that QMD automatic embedding run according to `memory.qmd.update.interval` or `embedInterval` after OpenClaw changes memory files.

## Best Possible Solution

Resolve PR #73512 by deciding the QMD product contract: either let explicit `embedInterval` opt into embedding even in lexical mode and update tests/docs, or preserve BM25-only `search` mode and improve config/status guidance so users choose `vsearch` or `query` for automatic embedding.

## Reproduction Assessment

Yes. A high-confidence static and unit-level reproduction exists: configure QMD with omitted or explicit `searchMode: "search"` plus `embedInterval`, advance the manager timer, and current main skips both update/embed subprocess calls under the existing lexical-mode test.

## Solution Assessment

Unclear. PR #73512 is a plausible narrow fix for the observable bug, but it changes the documented lexical-mode contract, so maintainer review should settle code, test, and docs behavior together.
