## Summary

Keep open. Current main still prevents QMD embed scheduling in the default omitted-`searchMode` path, and the linked open PR #73512 is the unmerged closing implementation candidate for this scheduler behavior.

## What This Changes

The issue asks OpenClaw to automatically run `qmd embed` according to `memory.qmd.update.embedInterval` when QMD files change, including the default omitted-`searchMode` configuration.

## Best Possible Solution

Review and land PR #73512, or an equivalent narrow change, after aligning docs/config help on whether explicit `embedInterval` should override lexical-mode embedding suppression.

## Reproduction Assessment

Yes. Source inspection shows omitted `searchMode` resolves to `search`, and both `shouldRunEmbed()` and `shouldScheduleEmbedTimer()` return before considering `embedIntervalMs`; the current test suite also asserts no embed subprocess calls for `search` plus `embedInterval`.

## Solution Assessment

Unclear. PR #73512 is the narrow implementation candidate for the observed scheduler gap, but maintainers should first settle the documented lexical-mode contract versus explicit `embedInterval` semantics.
