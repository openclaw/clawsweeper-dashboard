## Summary

Keep open. Current main still blocks QMD embedding in the reporter's omitted/default `searchMode` path, and the linked closing PR remains open rather than merged.

## What This Changes

The issue asks OpenClaw to automatically run `qmd embed` according to configured `memory.qmd.update.embedInterval` when QMD documents change.

## Best Possible Solution

Decide the lexical-mode contract, then land https://github.com/openclaw/openclaw/pull/73512 or an equivalent narrow fix that tracks explicit `embedInterval` intent and updates tests/docs together.

## Reproduction Assessment

Yes. Source inspection shows omitted `searchMode` resolves to `search`, and current `qmd-manager` returns before scheduling or running `qmd embed` even when `embedInterval` is positive; an existing test asserts that no embed command runs in that setup.

## Solution Assessment

Unclear. The linked PR targets the right scheduler gap, but maintainers still need to settle whether explicit `embedInterval` should override the documented BM25-only mode and preserve explicitness in resolved config.
