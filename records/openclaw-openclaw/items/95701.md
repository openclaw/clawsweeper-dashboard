## What This Changes

The issue reports that completed video generation stays marked running until completion delivery returns, delaying sequential video_generate workflows in the same session.

## Best Possible Solution

Land a focused fix that makes generated-but-delivering media tasks non-blocking for distinct follow-up video requests while preserving delivery-confirmation terminal handling and same-request duplicate protection.

## Solution Assessment

Yes. The maintainable fix is a typed non-blocking generated/delivering phase or equivalent state; moving `completeTaskRun` before delivery confirmation would violate the existing delivery-safety contract.
