## Summary

Keep open. Current main has adjacent manifest metadata checks, config validation, startup SecretRef preflight, and Gateway lifecycle hooks, but it still lacks the requested reusable boot-time plugin preflight contract for plugin-owned env/config/state integrity checks.

## What This Changes

Request a shared plugin preflight framework with plugin-registered boot-time checks for env vars, config files, and state integrity.

## Best Possible Solution

Design an additive plugin preflight contract that composes with manifest/setup metadata, Gateway startup preflight, SecretRef handling, and doctor repair while defining redacted structured results and no-mutation failure semantics.

## Solution Assessment
