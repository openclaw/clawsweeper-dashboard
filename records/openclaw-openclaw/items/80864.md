## Summary

Keep open. Current main has adjacent manifest validation, setup/auth metadata, SecretRef startup preflight, install hooks, and Gateway lifecycle hooks, but it still lacks the requested reusable boot-time plugin preflight/result contract for plugin-owned environment, config-file, and state-integrity checks.

## What This Changes

The issue requests a reusable boot-time preflight framework for all plugins to register environment, config, and state-integrity checks with consistent startup reporting and failure behavior.

## Best Possible Solution

Design an additive plugin preflight contract that composes with manifest/setup metadata, Gateway startup, SecretRef handling, doctor repair, and the open Gateway/config preflight items while defining redacted structured results and no-mutation failure semantics.

## Reproduction Assessment

Not applicable: this is a new plugin framework/API request, not a broken existing behavior report. Source and docs inspection give a high-confidence absence check for a reusable boot-time plugin preflight contract on current main.

## Solution Assessment

Unclear: the requested direction is plausible, but an ad hoc helper or repurposed lifecycle hook would be the wrong shape; the maintainable path is an explicit API/result contract with startup ordering and redaction semantics.
