## Summary

Keep open. Current main has adjacent manifest/config validation, SecretRef preflights, doctor state checks, and gateway/plugin startup hooks, but it still does not provide a reusable plugin boot-time preflight contract for env, config-file, and state-file integrity checks.

## What This Changes

Request a shared preflight framework that lets OpenClaw plugins register boot-time env, config, and state integrity checks with consistent errors and failure behavior.

## Best Possible Solution

Design an additive plugin preflight contract that composes with manifest validation, setupEntry, SecretRef activation, doctor repair, and the separate gateway no-bind preflight work.

## Reproduction Assessment

Not applicable: this is a new plugin framework/API request, not a broken existing behavior report. Source inspection and GitHub searches give high confidence that the requested reusable boot-time plugin preflight contract is absent on current main.

## Solution Assessment

Unclear: a generic plugin preflight framework may be the right direction, but the best implementation needs maintainer decisions on SDK shape, gateway startup ordering, no-mutation boundaries, redaction, and failure behavior.
