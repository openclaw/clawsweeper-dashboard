## Summary

Keep open: current main has adjacent manifest/config validation, doctor lint/state checks, SecretRef startup preflight, security audit collectors, and lifecycle hooks, but it still lacks a reusable boot-time plugin preflight/result contract for plugin-owned env, config, and state integrity checks.

## What This Changes

The issue requests a reusable boot-time preflight validation framework for plugins to register environment, config, and state integrity checks with consistent reporting and failure behavior.

## Best Possible Solution

Design an additive plugin preflight contract that composes with manifest/setup metadata, Gateway startup, SecretRef handling, doctor lint, security audit collectors, and the open Gateway/config preflight work while defining redacted structured results and no-mutation failure semantics.
