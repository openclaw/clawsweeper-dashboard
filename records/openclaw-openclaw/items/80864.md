## Summary

Keep open. Current main has adjacent manifest/config validation, startup SecretRef preflight, doctor state checks, setupEntry, activation metadata, and gateway lifecycle hooks, but it still does not provide the requested reusable plugin preflight contract for boot-time environment, config-file, and state-integrity checks.

## What This Changes

The issue requests a reusable plugin preflight validation framework for boot-time environment, configuration-file, and state-file integrity checks.

## Best Possible Solution

Design an additive plugin preflight contract that composes with manifest validation, setupEntry, SecretRef activation, doctor diagnostics, and gateway no-bind preflight before adding plugin-owned env/config/state checks.

## Reproduction Assessment

Not applicable: this is a new plugin SDK/gateway framework request, not a broken existing behavior report. Source inspection gives high confidence that the requested reusable boot-time plugin preflight contract is absent on current main.

## Solution Assessment

Unclear until maintainers settle the API and startup semantics. A generic plugin preflight surface is plausible, but the maintainable path should coordinate with gateway preflight, config pre-apply, SecretRef, and doctor boundaries rather than adding an ad hoc startup hook.
