## Summary

Keep open. Current main has manifest-based plugin config validation, startup SecretRef checks, and warn-only Gateway lifecycle hooks, but it does not provide the requested reusable plugin preflight contract for boot-time env/config/state checks; the related Gateway and config preflight issues are adjacent open work rather than a canonical replacement.

## What This Changes

The issue requests a reusable plugin preflight validation framework for environment, config, and state checks at boot time.

## Best Possible Solution

Design a narrow additive plugin preflight API that composes with manifest validation, Gateway preflight, config mutation validation, doctor repair, and plugin-owned state boundaries before adding concrete checks.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable: this is a new plugin SDK/startup framework request, not a broken current-main behavior report. Source inspection gives high confidence that the requested generic preflight contract is absent.

## Solution Assessment

Is this the best way to solve the issue? Unclear: a reusable plugin preflight surface is plausible, but the maintainable solution needs explicit SDK, startup, redaction, and failure-policy design rather than repurposing warn-only lifecycle hooks.
