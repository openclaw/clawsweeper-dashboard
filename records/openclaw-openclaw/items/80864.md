## Summary

Keep open. Current main has adjacent manifest config validation, SecretRef/startup checks, safe restart preflight, and lifecycle hooks, but it still lacks the requested reusable boot-time plugin preflight contract with plugin-registered env/config/state checks and defined fail/continue reporting semantics.

## What This Changes

Request to add a reusable plugin preflight framework for boot-time environment, configuration, and state validation with consistent reporting and failure behavior.

## Best Possible Solution

Define a narrow additive plugin preflight contract aligned with manifest validation, setup/onboarding metadata, doctor repair, and gateway preflight before adding plugin-specific checks.
