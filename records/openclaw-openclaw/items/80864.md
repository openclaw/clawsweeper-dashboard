## Summary

Keep open. Current main has adjacent manifest validation, config validation, service/config preflights, gateway lifecycle hooks, and open related preflight threads, but it does not yet provide the requested reusable plugin preflight contract for boot-time env/config/state integrity checks.

## What This Changes

The issue requests a reusable OpenClaw plugin preflight framework with plugin registration hooks and consistent boot-time validation of environment variables, config files, and state integrity.

## Best Possible Solution

Design an additive plugin preflight contract that composes with manifest validation, gateway preflight, setupEntry, SecretRef handling, and doctor repair before adding plugin-specific env/config/state checks.
