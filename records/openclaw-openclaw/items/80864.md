## Summary

Keep open. Current main has partial plugin config, setup, doctor, startup-maintenance, and per-message preflight surfaces, but it does not define the requested all-plugin boot-time preflight framework or a shared failure/reporting policy.

## What This Changes

Request a reusable plugin preflight validation framework for environment variables, config files, and state-file integrity at boot time.

## Best Possible Solution

Define an additive, manifest-first plugin preflight contract that reuses existing configSchema, setup metadata, doctor, and startup-maintenance surfaces, with explicit reporting and failure semantics before implementation.

## Solution Assessment

Unclear; the requested direction is plausible, but the best solution needs maintainer API and boot-policy design first. A narrow additive SDK/manifest seam is safer than adding ad hoc plugin runtime checks during startup.
