## Summary

Keep open. Current main has manifest/config validation, startup SecretRef preflight, and gateway lifecycle hooks, but it still lacks the requested reusable plugin preflight contract for plugin-registered env/config/state integrity checks; the related gateway preflight issue overlaps but does not fully cover this SDK/control-plane design.

## What This Changes

Request a standardized reusable plugin preflight framework for boot-time environment, configuration, and state integrity checks with consistent reporting and failure behavior.

## Best Possible Solution

Define an additive plugin preflight contract aligned with manifest validation, setupEntry/auth metadata, doctor repair, and gateway preflight before implementing plugin-owned env/config/state checks.
