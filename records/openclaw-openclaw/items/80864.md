## Summary

Keep open. Current main has adjacent manifest/config validation, doctor lint/state checks, SecretRef startup/write preflight, and lifecycle hooks, but it still lacks a reusable boot-time plugin preflight/result contract for plugin-owned env, config, and state integrity checks.

## What This Changes

Request a reusable plugin preflight framework for boot-time environment, config-file, and state-integrity checks with consistent reporting and failure behavior.

## Best Possible Solution

Design an additive plugin preflight contract that composes with manifest/setup metadata, Gateway startup, SecretRef handling, doctor lint, and the open Gateway preflight command while defining redacted structured results and no-mutation failure semantics.

## Reproduction Assessment

Not applicable: this is a new plugin framework/API request, not a broken current behavior report. Source and docs inspection give high confidence that current main lacks a reusable boot-time plugin preflight contract.

## Solution Assessment

Unclear: the direction is plausible, but the best solution needs maintainer design across plugin SDK hooks, manifest metadata, Gateway startup ordering, redaction, and failure policy rather than an ad hoc lifecycle hook.
