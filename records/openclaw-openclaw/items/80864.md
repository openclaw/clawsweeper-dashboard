## Summary

Keep open. Current main has adjacent manifest metadata checks, config validation, startup SecretRef preflight, and gateway lifecycle hooks, but it still does not provide the requested reusable boot-time plugin preflight contract for plugin-owned env/config/state integrity checks.

## What This Changes

The issue requests a reusable boot-time preflight framework where plugins can register environment, config-file, and state-file integrity checks with consistent startup reporting and failure behavior.

## Best Possible Solution

Design an additive plugin preflight contract that composes with manifest/setup metadata, Gateway startup preflight, SecretRef handling, and doctor repair while defining redacted structured results and no-mutation failure semantics.
