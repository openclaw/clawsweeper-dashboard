## Summary

Keep open. Current main has manifest schema validation, config preflight pieces, and Gateway startup hooks, but it still lacks a reusable boot-time plugin preflight registration/result contract; the related Gateway and config preflight items are open adjacent design threads rather than superseding this one.

## What This Changes

The issue requests a reusable plugin preflight validation framework for boot-time environment, config, and state integrity checks.

## Best Possible Solution

Define a narrow additive plugin preflight contract aligned with manifest validation, setupEntry/control-plane loading, Gateway no-bind preflight, and doctor repair before adding plugin-specific checks.
