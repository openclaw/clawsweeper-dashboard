## Summary

Keep open. Current main still has adjacent manifest/config metadata, startup SecretRef preflight, safe-restart preflight, and Gateway lifecycle hooks, but it does not provide the requested reusable boot-time plugin preflight/result contract for plugin-owned env/config/state integrity checks.

## What This Changes

The issue requests a reusable boot-time preflight validation framework for plugins to check environment variables, configuration files, and state integrity with consistent failure reporting.

## Best Possible Solution

Design an additive plugin preflight contract that composes with manifest/setup metadata, Gateway startup, SecretRef handling, doctor repair, and the open Gateway preflight command while defining redacted structured results and no-mutation failure semantics.

## Solution Assessment
