## Summary

Keep open. Current main has adjacent static manifest/config validation, Gateway SecretRef startup preflight, setup/status metadata, and lifecycle hooks, but it still lacks the requested reusable plugin-owned boot-time preflight framework and structured result/failure contract.

## What This Changes

Request to add a reusable plugin preflight framework so plugins can run boot-time environment, config, and state-integrity checks with consistent reporting and failure behavior.

## Best Possible Solution

Design an additive plugin preflight contract that composes with manifest/setup metadata, Gateway startup, SecretRef handling, doctor repair, and the open Gateway preflight command while defining redacted structured results and no-mutation failure semantics.

## Reproduction Assessment

Not applicable: this is a new plugin framework/API request, not a broken current-main behavior. Source and docs inspection give a high-confidence absence check for a reusable boot-time plugin preflight contract.

## Solution Assessment

Unclear: the direction is plausible, but the best solution needs maintainer design across plugin SDK hooks, Gateway startup ordering, redaction, no-mutation behavior, and failure policy rather than an ad hoc lifecycle hook.
