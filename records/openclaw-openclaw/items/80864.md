## Summary

Keep open. Current main has adjacent static manifest/config validation, setup/env metadata, SecretRef startup/write preflight, and safe-restart checks, but it still does not provide the requested reusable plugin-owned boot-time preflight hook/result/failure contract.

## What This Changes

Request a reusable boot-time preflight framework for plugins to validate environment variables, config files, and state integrity with consistent reporting and failure behavior.

## Best Possible Solution

Design an additive plugin preflight contract that composes with manifest/setup metadata, Gateway startup, SecretRef handling, doctor repair, and the related Gateway/config preflight work while producing redacted structured results with no-mutation failure semantics.

## Reproduction Assessment

Not applicable: this is a new plugin framework/API request, not a broken existing behavior report. Source and docs inspection give high confidence that current main lacks the requested reusable plugin boot-time preflight contract.

## Solution Assessment

Unclear until maintainers settle the API and failure policy. The direction is plausible, but a narrow implementation should be designed around shared startup/preflight semantics rather than an ad hoc lifecycle hook.
