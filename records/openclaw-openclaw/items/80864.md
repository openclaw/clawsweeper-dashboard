## Summary

Keep this open. Current main and v2026.6.8 have adjacent config guards, doctor health checks, startup SecretRef preflight, plugin install probes, and gateway lifecycle hooks, but they still do not provide the requested standardized boot-time plugin preflight framework for env/config/state integrity with consistent failure semantics.

## What This Changes

The issue requests a reusable boot-time preflight validation framework for plugins covering environment variables, configuration files, and state integrity at startup.

## Best Possible Solution

Define a maintainer-approved plugin/startup validation seam that composes with doctor health checks and real gateway startup planning, with explicit no-mutation, redaction, timeout, and failure semantics.

## Reproduction Assessment

Not applicable as a bug reproduction, but source inspection gives high confidence for the missing requested surface: current main has adjacent doctor/config/startup checks but no standardized boot-time plugin preflight framework.

## Solution Assessment

Unclear. A shared validation framework is directionally aligned, but maintainers need to choose the durable boundary among doctor health checks, gateway preflight, plugin SDK hooks, and config-write interception before implementation.
