## Summary

Keep open. Current main has adjacent manifest/config validation, config startup guards, and gateway/plugin bootstrap logic, but it still lacks the requested reusable plugin preflight contract for boot-time env/config/state checks with consistent reporting and failure semantics.

## What This Changes

Request a standardized boot-time preflight validation framework that plugins can use to register environment, config-file, and state-integrity checks.

## Best Possible Solution

Define a narrow additive plugin preflight contract aligned with manifest validation, setupEntry/runtime loading, doctor repair, and gateway preflight before adding plugin-specific checks.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable: this is a new plugin framework/API request, not broken existing behavior. Source and docs inspection give high confidence that current main does not provide the requested boot-time plugin preflight contract.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers define the API and startup policy. A shared preflight surface is plausible, but an ad hoc startup hook would be risky without no-mutation, failure, timeout, and reporting semantics.
