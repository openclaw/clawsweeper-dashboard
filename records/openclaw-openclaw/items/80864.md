## Summary

Keep open. Current main has adjacent doctor health checks, config validation, SecretRef preflight, and gateway startup bootstrap, but it still lacks the requested standardized boot-time plugin preflight framework with plugin-registered env/config/state checks and consistent startup failure semantics.

## What This Changes

The issue requests a reusable plugin preflight validation framework for boot-time environment, configuration, and state integrity checks.

## Best Possible Solution

Define a maintainer-approved plugin/startup validation seam that composes with doctor health checks and gateway startup planning, with explicit no-mutation, redaction, timeout, and failure semantics.
