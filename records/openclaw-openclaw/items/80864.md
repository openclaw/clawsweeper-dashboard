## Summary

Keep open. Current main has adjacent static manifest/config validation, cheap setup/auth metadata, SecretRef startup preflight, safe-restart preflight, and Gateway lifecycle hooks, but it still does not provide the requested reusable boot-time plugin preflight/result contract for plugin-owned env/config/state integrity checks.

## What This Changes

Request a reusable boot-time preflight framework for plugins to register env/config/state integrity checks with consistent startup reporting and failure behavior.

## Best Possible Solution

Design an additive plugin preflight contract that composes with manifest/setup metadata, Gateway startup, SecretRef handling, doctor repair, and the open Gateway preflight command while defining redacted structured results and no-mutation failure semantics.

## Reproduction Assessment

Not applicable: this is a new plugin framework/API request, not a broken existing behavior report. Source and docs inspection give a high-confidence absence check for a reusable boot-time plugin preflight contract on current main.

## Solution Assessment

Unclear: the direction is plausible, but the best solution needs maintainer design across plugin SDK hooks, manifest metadata, Gateway startup ordering, redaction, and failure policy rather than an ad hoc lifecycle hook.
