## Summary

Keep open. Current main has adjacent startup/config pieces, but it does not have the requested reusable plugin preflight contract; this would change plugin SDK/control-plane behavior and boot-time failure semantics, so it needs maintainer design rather than cleanup closure.

## What This Changes

The issue requests a reusable plugin preflight validation framework for environment, config, and state checks at boot time.

## Best Possible Solution

Define a narrow additive plugin preflight contract aligned with manifest validation, setupEntry, doctor repair, and gateway preflight before implementing plugin-specific checks.
