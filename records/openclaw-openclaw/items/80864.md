## Summary

Keep this open. Current main and the latest release have adjacent doctor health checks, post-upgrade plugin probes, config/startup safeguards, and plugin lifecycle hooks, but they still do not provide the requested standardized boot-time plugin preflight framework for env/config/state integrity with consistent failure reporting.

The issue requests a reusable boot-time preflight validation framework for plugins to verify environment variables, configuration files, and state integrity before plugin startup.
