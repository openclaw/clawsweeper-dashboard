## Summary

Keep open. Current main still routes Codex app-server dynamic tool lifecycle through `item`, `codex_app_server.*`, trajectory, or verbose text paths, while Control UI/WebChat live cards consume only the standard `stream: "tool"` contract.

## What This Changes

Project Codex harness live tool-call start, progress, result, and failure events into Control UI/WebChat tool cards.

## Best Possible Solution

Add Codex-owned projection from OpenClaw-owned dynamic tool lifecycle into the existing runtime-neutral `stream: "tool"` contract, while keeping the existing debug/item events.

## Reproduction Assessment

Yes, at source level. Route a Codex dynamic tool call through the app-server bridge/projector and the emitted live events are not `stream: "tool"` with `toolCallId`, so `handleAgentEvent` cannot render a tool card.

## Solution Assessment

Yes. The narrow maintainable fix is to project Codex dynamic tool lifecycle into the existing Gateway/Control UI tool-event contract, not add a Codex-only renderer or config flag.
