## Summary

Keep open. Current main still has the reported Codex harness projection gap: Codex app-server tool activity is projected as generic item/vendor events or verbose text callbacks, while Control UI/WebChat live tool cards only consume the standard `stream: "tool"` contract.

## What This Changes

Request Codex harness live tool lifecycle and progress projection into Control UI/WebChat tool cards.

## Best Possible Solution

Implement a Codex-owned projection layer that maps stable dynamic tool lifecycle data onto the existing runtime-neutral `stream: "tool"` contract while preserving current item events and transcript delivery.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to feed a Codex dynamic tool lifecycle through the current projector/bridge and observe that WebChat receives `item` or `codex_app_server.*` events while `handleAgentEvent` ignores them because they are not `stream: "tool"` with `toolCallId`.

## Solution Assessment

Yes. The narrow maintainable solution is to emit the existing Gateway/Control UI tool-event contract from the Codex harness path, rather than adding a parallel Codex-only renderer in Control UI.
