## Summary

Keep open. Current main still does not project Codex app-server dynamic tool lifecycle into the standard live `tool` event stream that Gateway and Control UI/WebChat use for tool cards, and the related issues are broader, closed for a different symptom, or channel-specific.

## What This Changes

Request live Control UI/WebChat tool-call cards for Codex harness dynamic tool execution, matching native OpenClaw agent tool-event rendering.

## Best Possible Solution

Add a Codex-owned projection layer that maps stable Codex dynamic/app-server tool lifecycle data into the runtime-neutral `stream: "tool"` contract with stable IDs, sanitized compact args/output, and terminal success/failure phases.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to pass a Codex dynamic app-server tool/item lifecycle through the current projector and observe that WebChat receives `item` or `codex_app_server.*` events; `handleAgentEvent` ignores those for tool cards, leaving `chatToolMessages` unchanged.

## Solution Assessment

Yes. The narrow maintainable fix is to emit the existing standard live tool-card event contract from the Codex harness path instead of adding a parallel Codex-only renderer in Control UI.
