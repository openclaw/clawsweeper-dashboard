Current main implements only a narrow immediate post-compaction identical-tool-outcome guard, while this issue asks for a broader saturated-session feature with config, transcript-tail detection, scoring, guarded instruction augmentation, post-compaction validation, reset recommendation, an incident fixture, and operator docs; same-author stacked PRs are still open for that work.

Add a loop-aware saturated-session compaction guard that detects transcript-tail degradation, guides compaction, validates the result, and recommends reset only as a last resort.
