## Repair Work Prompt

Fix openclaw/openclaw#76616. On current main, optional plugin tools such as `llm-task` and `lobster` can be listed by `tools.catalog` because catalog calls `ensureStandalonePluginToolRegistryLoaded`, but `createOpenClawTools` / `resolveOpenClawPluginToolsForOptions` / gateway effective+invoke paths call `resolvePluginTools` without the same preload. Add a shared or call-site preload before execution-surface plugin tool resolution, preserving `disablePluginTools`, scoped `onlyPluginIds`, `allowGatewaySubagentBinding`, runtime config selection, and lazy-loading constraints. Add focused regression coverage for optional plugin tools through `createOpenClawTools`, `tools.effective`, and `tools.invoke`. Align Lobster/llm-task docs so optional plugin enablement/allowlisting guidance matches supported behavior; do not introduce new `alsoAllow`/`full` semantics unless maintainers explicitly choose that broader product change. Add a user-facing `CHANGELOG.md` fix entry and avoid broad plugin loading or plugin-install policy changes.
