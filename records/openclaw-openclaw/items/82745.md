## Summary

Keep open: the PR fixes a real `sessions cleanup --fix-missing` crash path, but the current diff also prunes every loaded row without a `sessionId`, which can remove legitimate metadata-only session entries that current main intentionally preserves.

## What This Changes

The PR catches transcript path resolution failures during `sessions cleanup --fix-missing`, treats unresolved rows as missing, and adds an integration regression test for malformed key-shaped session IDs.

## Best Possible Solution

Land the cleanup-specific recovery only after it preserves legitimate transcriptless session metadata and adds regression coverage for that preservation path.

## Reproduction Assessment

Yes, at source level. Current main can pass a key-shaped stored `sessionId` with invalid or unresolvable `sessionFile` through `resolveSessionFilePath`, where fallback validation rejects the colon-containing value before cleanup can prune it.

## Solution Assessment

No, not as written. Catching resolver failures inside the explicit `--fix-missing` repair path is the right direction, but the implementation must not prune legitimate loaded entries that have no transcript id after normalization.
