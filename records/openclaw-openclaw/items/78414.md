## Summary

Keep open: the PR now has sufficient live Gateway log proof, but the patch still auto-restarts in `gateway.reload.mode="hot"` before the existing manual-restart warning path, which conflicts with current code and docs.

## What This Changes

The PR adds a Gateway config-reload guard, regression tests, and a changelog entry for restarting when a live config edit enables an unloaded channel plugin.

## Best Possible Solution

Land a version that makes unloaded channel activation explicit without silently wedging, while either preserving hot-mode as warn/manual or deliberately updating the reload-mode contract if maintainers choose auto-restart there.

## Reproduction Assessment

Yes, at source level: current main has a clear hot-mode path that warns and returns for restart-required external config changes, and the linked production report supplies the WhatsApp activation wedge evidence. I did not run a live current-main repro because this review is read-only.

## Solution Assessment

No as written. Restart-on-activation is a plausible narrow repair, but bypassing the documented hot-mode manual restart behavior needs either a code adjustment or an explicit maintainer contract change.
