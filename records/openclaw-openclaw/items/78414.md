## Summary

Keep open: the PR is a focused bug-fix candidate with sufficient runtime proof, but as written it bypasses the existing `hot` reload-mode warning path and turns a manual-restart mode into an automatic restart for this activation transition.

## What This Changes

The PR adds a Gateway config-reload guard, regression tests, and a changelog entry to restart when a live config edit enables a channel plugin that is not loaded in the running registry.

## Best Possible Solution

Land a revision that makes unloaded channel activation observable and recoverable while preserving the documented hot/manual behavior, unless maintainers explicitly update the reload-mode contract.

## Reproduction Assessment

Yes, at source level: current main classifies `channels.whatsapp` edits through the WhatsApp no-op reload rule, and the linked issue supplies production wedge evidence. I did not run a live current-main repro in this read-only review.

## Solution Assessment

No as written. Restart-on-activation is a plausible narrow repair, but this implementation queues restart before the hot-mode guard and therefore changes an existing documented reload-mode contract.

## Review Findings
