## Summary

Keep open: the fix direction and supplied logs are useful, but the diff currently auto-restarts even under `gateway.reload.mode="hot"`, which conflicts with the existing warning-only reload-mode contract unless maintainers explicitly choose that exception.

## What This Changes

The PR adds Gateway config-reload detection, tests, and changelog text for enabling an unloaded channel plugin so the Gateway restarts and can register the plugin after reboot.

## Best Possible Solution

Route unloaded-channel activation through the existing reload-mode policy, or explicitly update the public reload-mode contract and tests if maintainers choose a hot-mode exception.

## Reproduction Assessment

Yes for the review finding: PR head’s added hot-mode case expects `onRestart`, while current main and docs define hot mode as warning-only for restart-required changes. I did not run live Gateway commands in this read-only review.

## Solution Assessment

No as written. Restart-on-activation is a plausible narrow repair for the linked wedge, but the PR needs either to preserve the existing hot-mode contract or get an explicit maintainer decision plus docs/tests for the exception.
