Fix https://github.com/openclaw/openclaw/issues/78145. Current main can relay a bare `[max turns exceeded]` from a harness/backend or child session without actionable OpenClaw context. Add a small helper that recognizes this exact max-turn sentinel and formats a user-facing diagnostic with available run/session context: child session key/session id, run id or task/label when available, terminal status, and recovery suggestions such as inspect `/subagents info`/logs/trajectory or retry with a narrower task/higher applicable limit. Apply it at the Codex app-server failed-turn boundary (`extensions/codex/src/app-server/event-projector.ts` or adjacent run-attempt path) and the generic subagent completion capture/announce path (`src/agents/subagent-announce-output.ts` / `src/agents/subagent-announce.ts`) so both errors and captured assistant text are covered. Do not add a config option, change default turn limits, or expose internal-only metadata unnecessarily. Add focused regression tests for a child output or Codex failed turn whose text is exactly `[max turns exceeded]`, plus a control case proving ordinary output is unchanged. Add a user-facing changelog entry.
