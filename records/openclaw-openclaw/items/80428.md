## Summary

Keep open: current main still has the Discord native-command route-state aliases and nullable unions called out in the report, and the broad `pnpm check` contract includes the lint stage. The issue looks narrow and repairable, but this read-only review did not execute the lint gate.

## What This Changes

The issue asks for the Discord native-command route state types to stop collapsing to `any` so the lint gate no longer flags nullable route-state unions.

## Best Possible Solution

Keep the existing Discord routing behavior, but make the route-state type aliases concrete at the SDK or local Discord boundary so nullable state remains meaningful to oxlint.

## Reproduction Assessment

No high-confidence executed repro from this review; the issue provides the `pnpm check` lint output and current main still has the reported aliases at the same lines. A writable repair branch should rerun the focused lint path to confirm.

## Solution Assessment

Yes, the requested direction is narrow: use exported concrete SDK types or an explicit local non-`any` wrapper for the Discord route-state fields. Avoid changing runtime routing semantics or broad plugin SDK packaging unless focused lint proof shows the collapse originates there.
