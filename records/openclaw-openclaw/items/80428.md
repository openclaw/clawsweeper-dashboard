## What This Changes

The issue reports that `pnpm check` fails oxlint on Discord native-command route state types because `ReturnType`-derived aliases collapse to `any` and make nullable unions redundant.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/80473, or an equivalent narrow type-boundary fix, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes, source-based. The issue gives exact `pnpm check` oxlint diagnostics, and current main still has the same nullable `ReturnType` and `Awaited<ReturnType>` aliases at the reported lines.

## Solution Assessment

Yes. The narrow fix is to use the existing public SDK route result type and spell out the readiness union, which is exactly what the open fixing PR does without changing runtime behavior.
