## Summary

Keep open. Current main still routes runtime plugin text transforms through the indirect active-runtime helper, and the CLI/provider prompt paths still depend on that resolver. The related contributor fix PR remains open and unmerged, so this issue should stay open for maintainer review of that PR or a replacement fix.

## What This Changes

Plugin-registered `textTransforms` should be resolved from the active main-process plugin registry and applied to CLI/provider prompts.

## Best Possible Solution

Fix runtime text-transform resolution so bundled main-process code reads the active plugin registry directly or through a documented generic seam, then add focused regression coverage for the resolver and at least one CLI/provider prompt application path.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: set an active registry with `textTransforms`, then the current resolver still delegates through the require-probing helper before CLI/provider prompt paths consume the result.

## Solution Assessment

Yes as a direction, but #65331 should be adapted to the current `active-runtime-registry` layout and backed by regression tests before landing.
