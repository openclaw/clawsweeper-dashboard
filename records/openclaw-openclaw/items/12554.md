## Summary

Keep open. Current main and the latest release still lack the requested public session-scoped Gateway/RPC/CLI surface to list and terminate exec child processes; lower-level scoped process controls exist, but exposing them is a product/security decision.

## What This Changes

The issue requests a session-scoped operator command or abort enhancement that can list and terminate child exec processes spawned by a session.

## Best Possible Solution

Define one additive, authorization-checked Gateway and CLI exec-control contract that resolves the canonical session scope, lists supervisor-owned exec runs, cancels matching active runs with per-run outcomes, and documents isolation and partial-failure behavior without silently changing sessions.abort or sessions.delete defaults.

## Reproduction Assessment

Yes, at source level: current main exposes session abort/delete and scoped process primitives, but no public operation that lists or terminates all exec activity for an authorized session. I did not run a live orphan-process reproduction during this read-only review.

## Solution Assessment

No, directly making sessions.abort or sessions.delete kill exec children by default would be too broad without an explicit public contract. The maintainable path is an additive, authorization-checked operator surface that composes existing supervisor scope primitives with clear status and failure semantics.
