## Summary

Keep open: current main and the latest release still lack the requested operator surface to list and terminate exec child processes by session, and the remaining work is a security-sensitive Gateway/RPC/CLI design decision rather than a completed implementation.

## What This Changes

Request an operator-facing sessions kill or exec-list capability that can terminate all running exec processes associated with a stored session.

## Best Possible Solution

Land a maintainer-approved additive Gateway protocol and CLI flow that resolves canonical session keys, lists scoped managed exec records, terminates matching active runs with safe fallback, returns per-run status, and documents authorization and isolation rules.

## Reproduction Assessment

Yes, at source level. Current main and v2026.5.22 have no sessions.kill or exec-list method, abort kill flag, or CLI equivalent, while existing controls are model-facing process tools, active chat aborts, or subagent-run controls.

## Solution Assessment

Yes, but only as an additive operator contract after maintainer design. Reusing supervisor scope primitives is reasonable; silently changing sessions.abort or the HTTP subagent kill route would be riskier because authorization and isolation semantics need a clear owner decision.
