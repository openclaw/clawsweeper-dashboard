## Summary

Keep open: current main and the latest release still lack a public session-scoped RPC/CLI operator to list and terminate exec child processes; existing abort/delete/admin kill paths only cover active chat runs, session records, or subagent control, while lower-level process cancellation remains internal or model-facing.

## What This Changes

The issue requests a sessions_kill or exec-list style operator that can list and terminate all exec processes spawned by a session.

## Best Possible Solution

Define one additive, authorization-checked session exec-control contract that resolves the canonical session scope, lists supervisor-owned active exec runs, cancels matching scopes with per-run outcomes, and documents isolation and partial-failure behavior without silently changing sessions.abort or sessions.delete defaults.

## Reproduction Assessment

Yes, at source level: current main exposes session abort/delete and internal process-scope cancellation, but no public operation that lists or terminates all exec activity for an authorized session. I did not run a live orphan-process reproduction during this read-only review.

## Solution Assessment

Unclear until maintainers choose the public API shape. The narrow technical path is to compose existing supervisor scope cancellation, but authorization, visibility, process-tree fallback, and partial-failure semantics are product/security decisions.
