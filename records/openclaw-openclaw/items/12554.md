## Summary

Keep open. Current main has same-agent `process` tool controls and lower-level supervisor cancellation primitives, but it still lacks the requested operator-facing Gateway/RPC/CLI surface to list and terminate active exec child processes for an arbitrary stored session; related #38721 is overlapping and still open, not a canonical replacement.

## What This Changes

Add a session-level command/API to list and terminate active exec child processes associated with a stored session.

## Best Possible Solution

Add an explicit additive Gateway protocol and CLI surface that resolves a canonical session key, lists active supervised exec/process records for that session scope, terminates matching runs through supervisor cancellation with process-tree fallback where appropriate, returns per-run status, and documents authorization and isolation rules without changing default `sessions.abort` semantics.

## Reproduction Assessment

Yes for the missing operator surface: current-main source and docs show no session exec-list/kill command or abort flag, while tests assert background exec survives a tool abort. I did not run a live OS-process scenario because this review is read-only and the source-level absence is enough to keep the feature open.

## Solution Assessment

Unclear as an implementation-ready path: the requested list/kill capability is the right end state, but overloading `sessions.abort` would blur behavior and authorization. A new additive operator API with explicit scopes and per-run results is the safer direction.
