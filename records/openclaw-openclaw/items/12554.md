## Summary

Keep open: current main still has only lower-level scoped process supervision and same-agent process-tool controls, not the requested public session-wide Gateway/SDK/CLI operator to list and terminate all exec children for a stored session.

## What This Changes

The issue requests a session-level operator to list and terminate child exec processes spawned by a session.

## Best Possible Solution

Ship an additive Gateway/SDK/CLI session-process operator that resolves the canonical session scope, lists supervised exec records, cancels matching runs with per-run status, and documents authorization and isolation limits without silently changing sessions.abort semantics.

## Reproduction Assessment

Yes, at source level: current main has scoped supervisor/process primitives but lacks a public session-wide exec list/kill contract in Gateway methods, abort schema, SDK, CLI, and docs. I did not run a live macOS process-kill scenario during this read-only review.

## Solution Assessment

Unclear as an implementation-ready patch: the requested capability is the right end state, but overloading sessions.abort would blur current semantics and authorization. An explicit additive operator with scoped results is the safer maintainable direction.
