## Summary

Keep open: current main still lacks a public Gateway/RPC/CLI operator to list and terminate all exec children for an arbitrary stored session, even though scoped process-supervisor primitives exist underneath.

## What This Changes

Request a session-wide operator and CLI surface to list and kill exec child processes spawned by a session.

## Best Possible Solution

Add an explicit additive Gateway protocol and CLI operator that resolves a session key, lists supervised exec records for that session scope, terminates selected or all matching active runs with per-run status, and documents authorization/isolation rules.

## Reproduction Assessment

Yes, at source level for the feature gap: current main has scoped process-tool and supervisor primitives, but the registered Gateway methods, abort schema, HTTP kill route, CLI, and docs lack a session-wide exec list/kill operator.

## Solution Assessment

Unclear as an implementation-ready patch: the requested list/kill capability is the right end state, but overloading sessions.abort would blur current semantics and authorization; a new explicit operator API with scoped results is safer.
