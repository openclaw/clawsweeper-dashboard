## Summary

Keep open: current main and the latest release still lack the requested session-level Gateway/RPC/CLI surface to list and terminate exec children for a stored session; only lower-level process primitives, model-facing process tools, and subagent-run controls exist.

## What This Changes

The issue requests a session-level operator command/API to list and terminate active exec child processes spawned by a stored session.

## Best Possible Solution

Add an explicit additive Gateway protocol and CLI operator surface that resolves canonical session keys, lists scoped managed exec records, terminates matching active runs through supervisor cancellation with safe process-tree fallback, returns per-run status, and documents authorization and isolation rules.

## Reproduction Assessment

Yes, at source level. Current main has no sessions.kill/exec-list method, abort kill flag, or CLI equivalent, while existing controls are model-facing process tools or subagent-run controls rather than session-wide operator cleanup.

## Solution Assessment

Yes, but only as an additive operator contract. Reusing supervisor scope primitives is reasonable; silently changing sessions.abort would be riskier because process-control authorization and isolation need maintainer design.
