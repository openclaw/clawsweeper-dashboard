## Summary

Keep open: current main still has the lower-level supervised exec/process primitives, but it does not expose the requested session-wide operator surface to list and terminate all exec children for a stored session.

## What This Changes

Request a session-level list/kill capability for exec processes spawned by a sub-agent session, including process-group-style cleanup and operator-visible status.

## Best Possible Solution

Ship an additive Gateway/SDK/CLI session-process operator that resolves the canonical session scope, lists supervised exec records, cancels selected or all matching runs with per-run status, and documents authorization and isolation limits.

## Reproduction Assessment

Yes, at source level for the feature gap: current main has supervised exec scope primitives, but the registered Gateway methods, abort schema, SDK/CLI surfaces, HTTP kill route, and docs lack a session-wide exec list/kill operator. I did not run a live macOS process-kill scenario in this read-only review.

## Solution Assessment

Unclear as an implementation-ready patch: the requested capability is the right end state, but overloading sessions.abort would blur current semantics and authorization. An explicit additive operator with scoped results is the safer maintainable direction.
