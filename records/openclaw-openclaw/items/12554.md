## Summary

Keep open: current main and v2026.5.22 still lack the requested Gateway/RPC/CLI operator surface for listing and terminating exec children by session. The lower-level supervisor and scoped process-tool pieces exist, but they are not exposed as a session-wide operator contract.

## What This Changes

Add a session-level operator command/API that can list and terminate running exec processes spawned under a session.

## Best Possible Solution

Add an explicit additive Gateway protocol and CLI operator surface that resolves canonical session keys, lists scoped managed exec records, terminates matching active runs through supervisor cancellation with safe process-tree fallback, returns per-run status, and documents authorization and isolation rules.

## Reproduction Assessment

Yes, at source level. Current main and v2026.5.22 have no sessions.kill/exec-list method, abort kill flag, or CLI equivalent, while existing controls are model-facing process tools, active chat aborts, or subagent-run controls rather than session-wide operator cleanup.

## Solution Assessment

Yes, but only as an additive operator contract. Reusing supervisor scope primitives is reasonable; silently changing sessions.abort or the HTTP subagent kill route would be riskier because authorization and isolation semantics need maintainer design.
