## Summary

Keep open: current main and the latest release still do not expose the requested session-scoped operator surface to list and terminate child exec processes, and the remaining work is security/product-sensitive rather than stale cleanup.

## What This Changes

Add an authorized session-level operator surface to list and terminate child exec processes spawned by a session.

## Best Possible Solution

Define one additive, authorization-checked session exec-control contract that resolves the canonical session scope, lists supervisor-owned active exec runs, cancels matching scopes with per-run outcomes, and documents isolation and partial-failure behavior without changing sessions.abort or sessions.delete defaults.

## Reproduction Assessment

Yes, at source level: current main exposes session abort/delete and internal process-scope cancellation, but no public operation that lists or terminates all exec activity for an authorized session. I did not run a live orphan-process reproduction during this read-only review.

## Solution Assessment

Unclear until maintainers choose the public API shape. The narrow technical path is to compose existing supervisor scope cancellation, but authorization, visibility, process-tree fallback, and partial-failure semantics are product/security decisions.
