## Summary

Keep open: current main and the latest release still lack a public session-wide Gateway/RPC/CLI operator to list and terminate exec children for a stored session. The process supervisor and subagent kill plumbing provide lower-level pieces, but the remaining work is a new process-control contract with authorization and isolation semantics.

## What This Changes

Request a public session-wide command/RPC/CLI surface to list and terminate exec processes spawned by a stored session.

## Best Possible Solution

Ship an additive session-process operator that resolves the canonical session scope, lists supervised exec records, cancels matching active runs with per-run status, and documents authorization and isolation limits without silently changing sessions.abort semantics.

## Reproduction Assessment

Yes, at source level: current main and v2026.5.12 expose scoped process supervision and subagent kill pieces, but no public session-wide exec list/kill method, CLI command, or abort flag. I did not run a live macOS process-kill scenario during this read-only review.

## Solution Assessment

Unclear as an implementation-ready patch: the requested capability is directionally right, but overloading sessions.abort would blur current semantics and authorization. An explicit additive operator with scoped results is the safer maintainable path.
