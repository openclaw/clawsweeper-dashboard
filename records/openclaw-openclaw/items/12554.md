## Summary

Keep open. Current main still has internal supervisor and same-agent process-tool primitives, but no public Gateway/RPC/CLI operator that lists and terminates all exec children for an arbitrary stored session.

## What This Changes

Request a session-level exec process list/kill surface so operators can terminate running child exec processes spawned by a session.

## Best Possible Solution

Add an explicit additive Gateway protocol and CLI operator surface that resolves a canonical session key, lists supervised exec records for that scope, terminates selected or all matching runs with per-run status, and documents authorization/isolation rules without changing default `sessions.abort` semantics.

## Reproduction Assessment

Yes for the feature gap: current source exposes lower-level scoped process primitives but no session-wide exec list/kill Gateway/RPC/CLI surface. I did not run a live OS-process scenario because the missing public contract is clear from source inspection.

## Solution Assessment

Unclear as an implementation-ready path: the requested list/kill capability is the right end state, but overloading `sessions.abort` would blur current behavior and authorization. A new explicit operator API with scoped results is the safer maintainable direction.
