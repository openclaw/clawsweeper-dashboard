## Summary

Keep open: current main still has scoped supervisor/process-tool primitives, but no public session-wide Gateway/SDK/CLI operator that lets an operator list and terminate all exec children for a stored session. The prior issue discussion correctly identified this as a remaining API and authorization design gap, and current main still matches that assessment.

## What This Changes

Request an operator-facing sessions command/RPC to list and terminate all running exec/process children for a stored session scope.

## Best Possible Solution

Ship an additive Gateway/SDK/CLI session-process operator that resolves the canonical session scope, lists supervised exec records, cancels matching active runs with per-run status, and documents authorization and isolation limits without silently changing sessions.abort semantics.

## Reproduction Assessment

Yes, at source level: current main has scoped supervisor/process primitives but lacks a public session-wide exec list/kill contract in Gateway methods, abort schema, SDK, CLI, and docs. I did not run a live macOS process-kill scenario during this read-only review.

## Solution Assessment

Unclear as an implementation-ready patch: the requested capability is the right end state, but overloading sessions.abort would blur current semantics and authorization. An explicit additive operator with scoped results is the safer maintainable direction.
