## Summary

Keep open. Current main still has scoped supervisor and same-agent process-tool primitives, but no public Gateway/RPC/CLI operator that lists or terminates all exec children for an arbitrary stored session.

## What This Changes

The issue asks for a session-level exec list/kill capability so operators can terminate all child exec processes spawned by a session.

## Best Possible Solution

Add an explicit additive Gateway protocol and CLI operator surface that resolves canonical session identity, lists supervised exec records for that scope, terminates selected or all matching runs with per-run status, and documents authorization/isolation rules without changing default sessions.abort semantics.

## Reproduction Assessment

Yes for the feature gap: current source has scoped supervisor primitives but no public session-wide exec list/kill RPC or CLI surface. I did not run a live OS-process scenario because the missing contract is clear from source inspection.

## Solution Assessment

Unclear as implementation-ready: the requested list/kill capability is the right end state, but overloading sessions.abort would blur current behavior and authorization. A new explicit operator API with scoped results is the safer maintainable direction.
