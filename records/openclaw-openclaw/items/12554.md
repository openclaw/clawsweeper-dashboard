## Summary

Keep open: current main still lacks a public session-wide Gateway/SDK/CLI surface to list and terminate exec children for a stored session, even though lower-level supervisor and same-agent process-tool controls exist.

## What This Changes

Request a session-wide operator that can list and terminate all active exec child processes associated with a session.

## Best Possible Solution

Ship an additive session-process operator that resolves the canonical session scope, lists supervised exec records, cancels matching active runs with per-run status, and documents authorization and isolation limits without silently changing sessions.abort semantics.

## Reproduction Assessment

Yes, at source level: current main has supervised process primitives and same-agent process-tool controls, but no public Gateway/SDK/CLI session-wide exec list or kill contract. I did not run a live macOS process-kill scenario during this read-only review.

## Solution Assessment

Unclear as an implementation-ready patch: the requested capability is useful, but the exact operator shape, authorization model, and isolation semantics need maintainer design. An explicit additive operator is safer than extending sessions.abort implicitly.
