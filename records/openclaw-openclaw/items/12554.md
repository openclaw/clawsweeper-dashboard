## Summary

Keep open: current main and v2026.5.12 still lack a session-wide Gateway/RPC/CLI operator surface to list and terminate exec children, while the existing controls are either chat-run aborts or scoped model-facing process tools.

## What This Changes

Request an operator-facing sessions kill/list capability that can terminate all active exec child processes associated with a stored session.

## Best Possible Solution

Add an explicit, additive Gateway protocol and CLI operator surface that resolves a canonical session key, lists scoped managed exec/process records, terminates matching active runs through supervisor cancellation with safe process-tree fallback, returns per-run status, and documents authorization/isolation rules without silently changing default sessions.abort behavior.

## Reproduction Assessment

Yes, at source level. Current main shows no sessions.kill/exec-list method or abort kill flag, while existing process controls are scoped model tools or subagent-run controls rather than session-wide operator cleanup.

## Solution Assessment

Yes, but only as an additive operator contract. Reusing supervisor scope primitives is reasonable, but maintainers need to choose the authorization model and process-isolation semantics before implementation.
