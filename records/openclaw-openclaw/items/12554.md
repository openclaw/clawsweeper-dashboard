## Summary

Keep open: current main and the latest release still have lower-level process supervision and subagent kill pieces, but they do not expose the requested session-wide Gateway/CLI operator to list and terminate exec children for a stored session. The remaining work is real, but it is a new process-control API with authorization and isolation decisions, so it should stay in maintainer review rather than be auto-closed or queued as a narrow repair.

## What This Changes

The issue requests a session-level operator to list and terminate all active exec child processes associated with a stored OpenClaw session.

## Best Possible Solution

Ship an additive session-process operator that resolves the canonical session scope, lists supervised exec records, cancels matching active runs with per-run status, and documents authorization and isolation limits without silently changing sessions.abort semantics.

## Reproduction Assessment

Yes, at source level: current main has scoped supervisor/process primitives but lacks a public session-wide exec list/kill contract in Gateway methods, abort schema, CLI, and docs. I did not run a live macOS process-kill scenario during this read-only review.

## Solution Assessment

Unclear as an implementation-ready patch: the requested capability is the right end state, but overloading sessions.abort would blur current semantics and authorization. An explicit additive operator with scoped results is the safer maintainable direction.
