## Summary

Keep open: current main still lacks the requested session-scoped Gateway/RPC/CLI operator surface to list and terminate child exec processes, while the remaining design is security-sensitive rather than a stale cleanup candidate.

## What This Changes

The issue requests session-scoped commands that can list and terminate child exec processes started by an OpenClaw session.

## Best Possible Solution

Define an additive, authorization-checked Gateway/CLI session exec-control contract that resolves the canonical session scope, lists supervisor-owned active exec runs, cancels matching scopes with per-run outcomes, and documents isolation and partial-failure behavior without changing sessions.abort or sessions.delete defaults.

## Reproduction Assessment

Yes, source-level: current main exposes session abort/delete plus internal process-control primitives, but no public session-wide exec process list/kill operation. I did not run a live orphan-process reproduction in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public API shape and authorization model. The internal supervisor and process tool primitives are useful building blocks, but the requested operator contract is a product/security decision.
