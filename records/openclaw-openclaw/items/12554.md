## Summary

Keep open. Current main still has scoped process-tool kill/list primitives and subagent-run kill control, but not the requested session-wide Gateway/RPC/CLI operator surface for listing and terminating all exec children for an arbitrary stored session.

## What This Changes

Request a session-level operator command/API that can list and kill all still-running exec processes spawned by a session.

## Best Possible Solution

Add an explicit additive Gateway protocol and CLI operator surface that resolves a canonical session key, lists active supervised exec/process records for that scope, terminates selected or all matching runs with per-run status, and documents authorization/isolation rules without silently changing default `sessions.abort` behavior.

## Reproduction Assessment

Yes for the feature gap: source and docs show no `sessions.kill`, `exec-list`, `--kill-exec`, or `--terminate-children` surface, while background-exec abort tests preserve running background sessions after a tool abort. I did not run a live OS-process scenario because read-only source inspection is enough to confirm the missing operator path.

## Solution Assessment

Unclear as an implementation-ready path: the requested list/kill capability is the right end state, but overloading `sessions.abort` would blur authorization and behavior. A new additive operator API with explicit scopes and per-run results is the safer direction.
