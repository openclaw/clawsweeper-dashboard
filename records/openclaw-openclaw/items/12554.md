## Summary

Keep open. Current main still has scoped process-tool and supervisor primitives, but no session-wide Gateway/RPC/CLI operator to list and terminate exec children for an arbitrary stored session.

## What This Changes

The issue requests a Gateway/RPC/CLI surface to list and terminate child exec processes spawned by a session.

## Best Possible Solution

Add an explicit additive operator API and CLI surface that resolves a canonical session key, lists active supervised exec records for that session scope, terminates selected or all matching runs with per-run status, and documents authorization/isolation rules without changing default `sessions.abort` behavior.

## Reproduction Assessment

Yes for the feature gap: current source and docs show no session-wide exec list/kill operator, while scoped process-tool and supervisor primitives remain internal or same-agent surfaces. I did not run a live OS-process scenario because the missing public surface is clear from source inspection.

## Solution Assessment

Unclear as an implementation-ready path: the requested list/kill capability is the right end state, but overloading `sessions.abort` would blur current behavior and authorization. A new explicit operator API with scoped results is the safer maintainable direction.
