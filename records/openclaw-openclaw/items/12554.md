## Summary

Current main and the latest release still lack a public session-scoped operation to enumerate and terminate all exec children; internal scope-cancellation primitives exist, but the authorization and hard-termination contract remains a maintainer product/security decision. Related issues cover shutdown cleanup and broader delete/hard-kill semantics, but neither fully owns this issue’s process-list and child-exec termination request.

## What This Changes

Add a session-level operator API and CLI surface to list and terminate all active exec processes owned by a session.

## Best Possible Solution

Define one additive, authorization-checked session termination contract that resolves the canonical session scope, lists supervisor-owned active runs, composes existing run/subagent/runtime cleanup, cancels matching process scopes, and returns per-component outcomes without silently changing sessions.abort or sessions.delete defaults.

## Reproduction Assessment

Yes at source level: current main exposes session abort/delete and internal supervisor scope cancellation, but no public operation that lists or terminates all exec activity associated with an arbitrary authorized session. A live orphan-process reproduction was not run during this read-only review.

## Solution Assessment

No, adding all three proposed command variants would create overlapping lifecycle controls. The best path is one canonical hard-termination operation, with a separate status/list view only if maintainers decide observability should be independently authorized.
