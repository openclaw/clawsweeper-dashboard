## Summary

Keep open: current main has supervised exec/process primitives, but it still lacks a Gateway/SDK/CLI operator that can list and cancel all exec children for a stored session.

## What This Changes

The issue requests a session-wide command or abort option that lists and terminates running exec processes spawned by that session.

## Best Possible Solution

Ship an additive Gateway/SDK/CLI session-process operator that resolves the canonical session scope, lists supervised exec records, cancels selected or all matching runs with per-run status, and documents authorization and isolation limits.

## Reproduction Assessment

Yes, at source level: current main has scoped supervisor/process primitives but lacks a public session-wide exec list/kill contract in Gateway methods, abort schema, SDK, CLI, and docs. I did not run a live macOS process-kill scenario during this read-only review.

## Solution Assessment

Unclear as an implementation-ready patch: the requested capability is the right end state, but overloading sessions.abort would blur current semantics and authorization. An explicit additive operator with scoped results is the safer maintainable direction.
