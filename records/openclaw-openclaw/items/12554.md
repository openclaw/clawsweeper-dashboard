## Summary

Keep open: current main still lacks a public session-wide Gateway/CLI operator to list and terminate exec children for a stored session. The remaining work is a new process-control API with security and product semantics, not a narrow bug-fix cleanup item.

## What This Changes

Request a `sessions_kill` or equivalent session abort/listing surface that can terminate all running exec child processes associated with a session.

## Best Possible Solution

Ship an additive session-process operator that resolves the canonical session scope, lists supervised exec records, cancels matching active runs with per-run status, and documents authorization and isolation limits without changing `sessions.abort` semantics silently.

## Reproduction Assessment

Yes, at source level: current main and v2026.5.12 expose scoped process supervision and subagent kill pieces but no public session-wide exec list/kill contract in Gateway methods, abort schema, CLI, or docs. I did not run a live macOS process-kill scenario during this read-only review.

## Solution Assessment

Unclear as an implementation-ready patch: the requested capability is directionally right, but overloading `sessions.abort` would blur current semantics and authorization. An explicit additive operator with scoped results is the safer maintainable path.
