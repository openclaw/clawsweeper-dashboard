## Summary

Keep open. Current main and latest release v2026.6.6 still lack a public session-scoped Gateway/RPC/CLI operation to list and terminate exec child processes; lower-level process cancellation exists, but exposing it needs maintainer product and security design.

## What This Changes

Request to add a session-scoped operator surface that lists and terminates exec child processes spawned by a session.

## Best Possible Solution

Define one additive authorization-checked session termination contract that resolves the canonical session scope, lists supervisor-owned active exec runs, cancels matching process scopes, composes existing runtime cleanup, and returns per-component outcomes without silently changing sessions.abort or sessions.delete defaults.

## Reproduction Assessment

Yes at source level: current main exposes abort/delete and internal supervisor cancellation, but no public operation that lists or terminates all exec activity for an authorized session. I did not run a live orphan-process reproduction during this read-only review.

## Solution Assessment

Unclear until maintainers choose the public API shape. The narrow technical path is to compose existing supervisor scope cancellation, but authorization, visibility, and partial-failure semantics are product/security decisions.
