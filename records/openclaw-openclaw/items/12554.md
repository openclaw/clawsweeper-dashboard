## Summary

Keep open. Current main still has lower-level scoped process and supervisor primitives, but no public Gateway/RPC/CLI operator to list and terminate all exec children for an arbitrary stored session; the related open shutdown issue is adjacent and does not own this operator surface.

## What This Changes

The issue requests a session-level command/RPC to list and terminate exec processes spawned by a session.

## Best Possible Solution

Add an explicit additive Gateway protocol and CLI operator that resolves a session key, lists supervised exec records for that session scope, terminates selected or all matching active runs with per-run status, and documents authorization/isolation rules.

## Reproduction Assessment

Yes, at source level for the feature gap: current main has scoped process-tool and supervisor primitives, but the registered Gateway methods, abort schema, HTTP kill route, CLI, and docs lack a session-wide exec list/kill operator. I did not run a live OS-process scenario because the missing public contract is directly visible in source.

## Solution Assessment

Unclear as an implementation-ready patch: the requested list/kill capability is the right end state, but overloading sessions.abort would blur current behavior and authorization. A new explicit operator API with scoped results is the safer maintainable direction.
