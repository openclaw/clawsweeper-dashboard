## Summary

Keep open: current main still has scoped process-supervisor and model-facing process-tool controls, but not the requested operator-facing Gateway/RPC/CLI surface to list and terminate exec children for an arbitrary stored session.

## What This Changes

The issue requests a session-scoped operator command/API to list and terminate child exec processes spawned by that session.

## Best Possible Solution

Add an explicit additive Gateway protocol and CLI operator surface that resolves canonical session keys, lists scoped managed exec records, terminates matching active runs through supervisor cancellation with safe process-tree fallback, returns per-run status, and documents authorization/isolation rules.

## Reproduction Assessment

Yes, at source level. Current main has no sessions.kill/exec-list method or abort kill flag, while existing controls are either model-facing process tools or subagent-run controls rather than session-wide operator cleanup.

## Solution Assessment

Yes, but only as an additive operator contract. Reusing supervisor scope primitives is reasonable; silently changing sessions.abort would be riskier because process-control authorization and isolation need maintainer design.
