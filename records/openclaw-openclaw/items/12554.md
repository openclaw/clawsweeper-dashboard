## Summary

Keep open: current main has scoped process supervision and same-agent process-tool kill controls, but still lacks the requested public session-wide Gateway/SDK/CLI operator to list and terminate all exec children for a stored session.

## What This Changes

The issue requests a session-level operator command/API to list and terminate exec child processes spawned by a session.

## Best Possible Solution

Ship an additive Gateway/SDK/CLI session-process operator that resolves the canonical session scope, lists supervised exec records, cancels matching active runs with per-run status, and documents authorization and isolation limits without silently changing sessions.abort semantics.

## Reproduction Assessment

Yes, at source level: current main has supervised process primitives and same-agent process-tool controls, but no public Gateway/SDK/CLI session-wide exec list or kill contract. I did not run a live macOS process-kill scenario during this read-only review.

## Solution Assessment

Unclear as an implementation-ready patch: the requested capability is useful, but the exact operator shape, authorization model, and isolation semantics need maintainer design. An explicit additive operator is safer than extending sessions.abort implicitly.
