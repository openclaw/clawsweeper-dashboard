## Summary

Keep open: current main still lacks a public Gateway/RPC/CLI operator to list and terminate all exec children for a stored session, while the lower-level supervisor only exposes internal scoped cancellation primitives.

## What This Changes

The issue requests a session-level command/API to list and terminate running exec child processes spawned by that session.

## Best Possible Solution

Ship an additive session-process operator contract that resolves the canonical session scope, returns supervised exec records, cancels selected or all matching runs with per-run status, and documents limits.

## Reproduction Assessment

Yes, at source level for the feature gap: current main has supervised exec scope primitives, but the registered Gateway methods, abort schema, SDK/CLI surfaces, HTTP kill route, and docs lack a session-wide exec list/kill operator.

## Solution Assessment

Unclear as an implementation-ready patch: the requested capability is the right end state, but overloading sessions.abort would blur current semantics and authorization; an explicit additive operator with scoped results is safer.
