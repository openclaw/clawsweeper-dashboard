## Summary

Keep open. Current main still has agent-scoped process listing and kill primitives, but it does not expose the requested session-wide Gateway/RPC/CLI operator surface for listing and terminating all exec children attached to an arbitrary stored session.

## What This Changes

Add a session-level operator command or abort enhancement that can list and terminate all running exec child processes for a session.

## Best Possible Solution

Add an explicit additive operator API and CLI surface that resolves a canonical session key, lists active supervised exec records for that session scope, terminates selected or all matching runs with per-run status, and documents authorization/isolation rules without silently changing default `sessions.abort` behavior.

## Reproduction Assessment

Yes for the feature gap: current source and docs show no session-wide exec list/kill operator, and existing tests intentionally preserve background exec after a tool abort. I did not run a live OS-process scenario because source inspection is enough to confirm the missing surface.

## Solution Assessment

Unclear as an implementation-ready path: the requested list/kill capability is the right end state, but overloading `sessions.abort` would blur current behavior and authorization. A new explicit operator API with scoped results is the safer maintainable direction.
