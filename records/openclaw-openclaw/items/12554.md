## Label Justifications

- `impact:session-state`: Exec child processes can outlive the session/run state that launched them, creating session lifecycle drift.

## Summary

Keep open: current main and v2026.6.8 still lack a public session-scoped Gateway/RPC/CLI operation to list and terminate exec child processes, while the existing lower-level process controls remain internal or model-facing.

## What This Changes

The issue requests a public session-scoped exec-control surface that can list and kill child exec processes spawned by a session.

## Best Possible Solution

Define one additive, authorization-checked session exec-control contract that resolves the canonical session scope, lists supervisor-owned active exec runs, cancels matching scopes with per-run outcomes, and documents isolation and partial-failure behavior without silently changing sessions.abort or sessions.delete defaults.

## Reproduction Assessment

Yes, at source level: current main exposes session abort/delete and internal process-scope cancellation, but no public operation that lists or terminates all exec activity for an authorized session. I did not run a live orphan-process reproduction during this read-only review.

## Solution Assessment

Unclear until maintainers choose the public API shape. The narrow technical path is to compose existing supervisor scope cancellation, but authorization, visibility, and partial-failure semantics are product/security decisions.
