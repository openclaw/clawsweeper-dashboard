## Summary

Keep open. Current main has same-agent process-tool controls and lower-level supervisor cancellation, but it still lacks the requested session-wide Gateway/RPC/CLI operator surface to list and kill all exec children for an arbitrary stored session.

## What This Changes

Request an operator-facing session kill/list capability that can enumerate and terminate running exec or system.run child processes for a stored session.

## Best Possible Solution

Add an explicit additive operator API and CLI that resolves a canonical session key, lists active supervised exec/process records for that session scope, terminates them with per-run status, and documents authorization/isolation rules without changing default `sessions.abort` behavior.

## Reproduction Assessment

Yes for the missing operator surface: current source and docs show no `sessions.kill`, `exec-list`, `--kill-exec`, or `--terminate-children` path, while tests assert background exec survives a tool abort. I did not run a live OS-process scenario because this read-only source check is enough to confirm the feature gap.

## Solution Assessment

Unclear as an implementation-ready path: the requested list/kill capability is the right end state, but overloading `sessions.abort` would blur authorization and behavior. A new additive operator API with explicit scopes and per-run results is the safer direction.
