## Summary

Keep open. Current main still has scoped process-tool and supervisor primitives, but no public Gateway/RPC/CLI operator that lists and terminates all exec children for an arbitrary stored session; related open issues track adjacent shutdown, delete, and PID-correlation gaps rather than fully owning this operator surface.

## What This Changes

Request a session-level operator and CLI surface to list and terminate running exec/process children spawned under a session.

## Best Possible Solution

Add an explicit additive Gateway protocol and CLI operator that resolves a session key, lists supervised exec records for that session scope, terminates selected or all matching active runs with per-run status, and documents authorization/isolation rules.

## Reproduction Assessment

Yes for the feature gap: current source has supervised and process-tool scoped primitives, but the registered Gateway methods, abort schema, HTTP kill route, CLI, and docs lack a session-wide exec list/kill operator. I did not run a live OS-process scenario because this is a missing public contract rather than a failing existing command.

## Solution Assessment

Unclear as an implementation-ready path: the requested list/kill capability is the right end state, but overloading `sessions.abort` would blur current behavior and authorization. A new explicit operator API with scoped results is the safer maintainable direction.
