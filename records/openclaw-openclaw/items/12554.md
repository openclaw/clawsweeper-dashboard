## Summary

Keep open: current main and the latest release still lack the requested session-wide exec list/kill operator. Lower-level scoped process primitives exist, but exposing them through Gateway/RPC/CLI remains a process-control and authorization design decision.

## What This Changes

Requested a session-level operator surface that can list and terminate running exec child processes associated with a session.

## Best Possible Solution

Add an additive Gateway protocol and CLI surface that resolves the canonical session scope, lists active supervised exec runs, cancels matching runs through supervisor-owned termination, returns per-run status, and documents authorization/isolation rules without changing default sessions.abort semantics.

## Reproduction Assessment

Yes, at source level: current main exposes scoped process primitives, but the session RPC schema, Gateway method descriptors, abort handler, HTTP kill route, CLI, and docs do not provide the requested session-wide exec list/kill path. I did not run a live macOS child-process reproduction.

## Solution Assessment

Unclear as an implementation path: the requested capability belongs in core session/process control, but maintainers still need to choose the exact RPC/CLI contract, authorization scope, and termination semantics.
