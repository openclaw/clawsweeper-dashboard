## Summary

Keep open. Current main and the latest release still lack the requested public session-scoped operation to list and terminate exec/process children; only lower-level process-supervisor scope cancellation and per-tool process controls exist, and exposing this as an operator API needs maintainer product/security design.

## What This Changes

The issue requests a session-scoped operator command to list and terminate child exec processes spawned by a session.

## Best Possible Solution

Define one additive, authorization-checked session termination contract that resolves the canonical session scope, lists supervisor-owned active runs, cancels matching process scopes, composes existing run/subagent/runtime cleanup, and returns per-component outcomes without silently changing sessions.abort or sessions.delete defaults.

## Reproduction Assessment

Yes at source level: current main exposes abort/delete and internal supervisor scope cancellation, but no public operation that lists or terminates all exec activity for an authorized session. I did not run a live orphan-process reproduction during this read-only review.

## Solution Assessment

No if implemented as several parallel command variants; the maintainable fix is one canonical Gateway/RPC/CLI hard-termination and status contract with explicit authorization and partial-failure semantics.
