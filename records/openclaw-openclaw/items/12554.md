## Summary

Keep open: current main and v2026.6.5 still lack a session-level operator surface to list or terminate exec child processes; existing pieces are scoped process-tool/supervisor primitives, not the requested Gateway/RPC/CLI contract.

## What This Changes

Request a session-level operator command/API to list and terminate all exec child processes associated with a session.

## Best Possible Solution

Land a maintainer-approved additive Gateway protocol and CLI flow that resolves canonical session keys, lists scoped managed exec records, terminates matching active runs with safe fallback, returns per-run status, and documents authorization/isolation rules without changing default sessions.abort behavior.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.5 have no sessions.kill or exec-list method, abort kill flag, or CLI equivalent, while existing controls are model-facing process tools, active chat aborts, or subagent-run controls.

## Solution Assessment

Yes, but only as an additive operator contract after maintainer design. Reusing supervisor scope primitives is reasonable; silently changing sessions.abort or the HTTP subagent kill route would conflate chat/subagent control with security-sensitive exec cleanup.
