## Summary

Keep open. Current main still has per-agent process-tool list/kill and lower-level supervisor cancellation primitives, but it does not expose the requested session-wide Gateway/RPC/CLI operator surface to list and terminate all exec children for an arbitrary stored session.

## What This Changes

Request a `sessions_kill` or equivalent session abort enhancement that can list and terminate all running exec processes associated with a session.

## Best Possible Solution

Add an explicit additive Gateway protocol and CLI operator surface that resolves a canonical session key, lists active supervised exec records for that scope, terminates selected or all matching runs with per-run status, and documents authorization/isolation rules without changing default `sessions.abort` semantics.

## Reproduction Assessment

Yes for the feature gap: current source and docs show no `sessions.kill`, `exec-list`, `--kill-exec`, or `--terminate-children` surface, while tests intentionally preserve background exec after a tool abort. I did not run a live OS-process scenario because source inspection is enough to confirm the missing operator path.

## Solution Assessment

Unclear as an implementation-ready path: the requested list/kill capability is the right end state, but overloading `sessions.abort` would blur current behavior and authorization. A new explicit operator API with scoped results is the safer maintainable direction.
