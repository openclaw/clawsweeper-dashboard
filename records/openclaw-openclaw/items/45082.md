- `impact:message-loss`: Messages are delivered to the wrong Mattermost location, such as the channel root or a DM instead of the intended thread.

## Summary

Keep open. Current main still has source-level gaps: Mattermost action sends only use explicit reply fields, the Mattermost threading adapter lacks tool-context/auto-thread hooks, and non-direct inbound Mattermost routes still skip last-route persistence; open linked PRs target the behavior but have not landed.

## What This Changes

The issue requests that Mattermost proactive message-tool sends and cron/heartbeat wake replies inherit the active thread-scoped session context instead of posting to the channel root or a DM.

## Best Possible Solution

Land or rework a current-main Mattermost fix that supplies plugin-owned tool thread context/resolution and safely persists active non-direct delivery routes, using the open linked PRs as input rather than closing this issue early.

## Reproduction Assessment

Yes, at source level. Current main carries Mattermost thread ids in inbound context but does not expose a Mattermost tool-context/auto-thread hook for proactive sends and skips route persistence for non-direct inbound sessions.

## Solution Assessment

Yes in direction, but it is not fixed on main. The maintainable solution belongs in Mattermost-owned threading hooks plus safe active-session route persistence, with the linked PRs reviewed, consolidated, or replaced.
