## Summary

Keep open. Source inspection shows current main and v2026.5.20 still lack Mattermost-owned tool-thread context for proactive sends, and the linked fix PRs are open, conflicting, or incomplete rather than merged.

## What This Changes

Mattermost thread-scoped sessions should make proactive message-tool sends and cron/heartbeat wake replies inherit the active thread when no explicit reply target is supplied.

## Best Possible Solution

Consolidate the linked Mattermost fixes into a current-main PR that adds Mattermost tool-context auto-threading and safe non-direct route persistence, with focused tests and redacted live Mattermost proof.

## Reproduction Assessment

Yes, at source level. Current main carries Mattermost thread ids in inbound context but lacks the Mattermost tool-context hooks for proactive sends and still skips explicit non-direct last-route persistence.

## Solution Assessment

Yes in direction, but no complete fix is on main. The maintainable fix belongs in Mattermost-owned thread hooks plus safe shared or active-session route persistence, with the linked PRs reviewed, consolidated, or replaced.
