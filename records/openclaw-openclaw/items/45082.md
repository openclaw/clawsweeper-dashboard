## Summary

Keep open. Current main and the latest release still show Mattermost send actions using only explicit reply ids, while non-direct inbound sessions still skip the route update needed by cron and heartbeat thread replies; the linked closing PRs remain open and unmerged.

## What This Changes

Mattermost proactive message sends and cron/heartbeat wake replies should inherit the current thread context when the active session is thread-scoped.

## Best Possible Solution

Implement Mattermost-owned tool context/auto-thread resolution plus safe non-direct session route persistence, with focused tests and redacted real Mattermost proof.

## Reproduction Assessment

Yes, at source level. Current main carries Mattermost thread ids in inbound context, but Mattermost lacks the tool-context/auto-thread hooks for proactive sends and still skips non-direct last-route persistence.

## Solution Assessment

Yes in direction, but no complete fix is on main. The maintainable solution belongs in Mattermost-owned threading hooks and safe session route persistence, with the linked PRs reviewed, consolidated, or replaced.
