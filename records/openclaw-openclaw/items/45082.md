- `impact:message-loss`: Messages can be delivered to the wrong Mattermost location, such as the channel root or a DM instead of the intended thread.

## Summary

Keep open. Current main still has the Mattermost source-level gaps reported here, and the open closing PRs are not merged, so the issue should remain as the canonical tracking item until that work is consolidated or replaced.

## What This Changes

Mattermost proactive message-tool sends and cron/heartbeat wake replies should inherit the active thread delivery context instead of posting to the channel root or DM.

## Best Possible Solution

Keep this issue as the canonical tracker until a current-main PR supplies Mattermost tool thread context/auto-thread resolution and safe non-direct route persistence with real Mattermost proof.

## Reproduction Assessment

Yes, at source level. Current main carries Mattermost thread IDs in inbound context, but Mattermost does not expose buildToolContext/resolveAutoThreadId for proactive sends and still skips last-route persistence for non-direct inbound sessions.

## Solution Assessment

Yes in direction, but no complete fix is on main. The maintainable solution belongs in Mattermost-owned threading hooks plus safe active-session route persistence, with the linked PRs reviewed, consolidated, or replaced.
