## Summary

Keep open: the Cron zh-CN localization work is useful, but the current PR head still blocks merge because relative-time localization collapses zh-TW into Simplified Chinese and the external real-behavior proof gate is still missing.

## What This Changes

The PR extracts Cron Control UI copy into i18n keys, updates locale bundles/generated metadata, and passes the active UI locale into Cron schedule and relative timestamp formatting.

## Best Possible Solution

Keep the Cron i18n extraction, but preserve the zh-CN/zh-TW split in relative-time formatting, add focused locale coverage, and require inspectable proof from a real zh-CN Cron tab run.

## Reproduction Assessment

Yes. Current main source shows Cron form/action strings still hardcoded in English while zh-CN is a supported Control UI locale; I did not run a browser reproduction in this read-only review.

## Solution Assessment

No, not as written. The i18n extraction is the right direction, but the formatter should gate Simplified Chinese output on zh-CN or add a Traditional Chinese branch before wiring the active UI locale through it.
