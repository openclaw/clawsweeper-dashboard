## Summary

This PR should stay open: it targets a real Cron Control UI localization gap, but the latest head still has a blocking locale regression and lacks inspectable after-fix real behavior proof.

## What This Changes

The branch extracts Cron Control UI labels and schedule text into i18n keys, adds Simplified Chinese translations and generated locale metadata, and threads the active UI locale into relative timestamp formatting.

## Best Possible Solution

Keep the Cron i18n extraction direction, but preserve the zh-CN/zh-TW locale boundary in relative-time formatting and require inspectable proof from a real zh-CN Cron tab run before merge.

## Reproduction Assessment

Yes. Current main source still contains hardcoded English Cron labels while zh-CN is a supported Control UI locale; I did not run a browser session in this read-only review.

## Solution Assessment

No, not as written. Extracting Cron strings into i18n keys is the right direction, but the relative-time implementation should target zh-CN specifically or add a separate Traditional Chinese branch before wiring it to all active UI locales.
