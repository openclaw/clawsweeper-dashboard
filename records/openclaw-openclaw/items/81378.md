## Summary

Keep open: the PR addresses a real Cron UI localization gap, but current head is not merge-ready because several new keys are still unwired, explicit session targets are mislabeled, and zh-TW would receive Simplified Chinese relative-time strings.

## What This Changes

The PR adds Cron Control UI i18n keys and locale sync output, localizes some Cron list/presenter text, and passes the active UI locale into relative timestamp formatting.

## Best Possible Solution

Keep the Cron localization work, but wire all added keys in the view, preserve exact session target values, separate zh-CN from zh-TW relative-time output, add focused regression coverage, and refresh real UI proof.

## Reproduction Assessment

Yes. Source inspection of the latest PR head shows the remaining hardcoded strings, the session-target label regression, and the zh-CN/zh-TW locale conflation without needing a live browser run.

## Solution Assessment

No. The i18n extraction is the right direction, but this implementation is incomplete and changes existing display semantics for supported session targets and Traditional Chinese locale behavior.
