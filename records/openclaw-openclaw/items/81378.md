## Summary

Keep open. The cron localization work is useful, but the patch introduces a supported-locale regression for Traditional Chinese relative timestamps and still needs concrete real behavior proof before merge.

## What This Changes

This PR extracts Cron page UI strings into i18n keys, adds Simplified Chinese translations, localizes Cron schedule and relative-time labels, and regenerates Control UI locale output.

## Best Possible Solution

Keep the i18n extraction, but make relative-time localization respect `zh-CN` and `zh-TW` separately, preferably with focused tests for both locales before merge.

## Reproduction Assessment

Yes for the blocking review finding. Source inspection shows the UI now passes the selected locale into the formatter and `zh-TW` is a supported locale, so `startsWith("zh")` will render Simplified Chinese relative-time strings under Traditional Chinese.

## Solution Assessment

No. The Cron string extraction is directionally sound, but relative-time formatting should use per-locale text or an i18n/Intl-backed path instead of treating every Chinese locale as Simplified Chinese.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep zh-TW out of Simplified relative strings:** `src/infra/format-time/format-relative.ts:84`
  - body: `ui/src/ui/format.ts` now passes the selected UI locale into this helper, and `registry.ts` supports `zh-TW` as a separate Traditional Chinese locale. Please restrict this branch to `zh-CN` or add separate `zh-TW` output before merging.
