## Summary

Keep open: the PR addresses a real Cron Control UI localization gap and includes sufficient after-fix screenshot proof, but the current patch introduces two source-reproducible UI regressions that should block merge.

## What This Changes

The PR extracts Cron Control UI strings into i18n keys, adds zh-CN and generated locale updates, localizes schedule and relative-time display, and updates a Cron view expectation.

## Best Possible Solution

Keep the Cron i18n extraction, but preserve custom session target labels and make relative-time localization respect the zh-CN/zh-TW boundary with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection of PR head shows custom Cron session targets collapse to the isolated label, and `zh-TW` flows into a Simplified Chinese relative-time branch through the active-locale wrapper.

## Solution Assessment

No, not yet. The extraction is the right direction, but the patch needs narrower session-target handling and locale-specific relative-time output before it is mergeable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve custom cron session target labels:** `ui/src/ui/views/cron.ts:1537`
  - body: This now renders every non-`main` cron job as the localized isolated label. `CronSessionTarget` still includes `current` and `session:<id>`, so jobs created through CLI or plugins with persistent session targets will appear as isolated even though they are pinned to an existing session. Preserve those specific labels or fall back to the raw target when it is not `main` or `isolated`.
  - confidence: 0.92
- **[P2] Keep zh-TW out of Simplified relative-time output:** `src/infra/format-time/format-relative.ts:84`
  - body: The UI wrapper now passes the active Control UI locale into this formatter, so selecting `zh-TW` also enters this Simplified Chinese branch. Since `zh-CN` and `zh-TW` are separate supported bundles, Traditional Chinese users will see Simplified strings such as `刚刚` and `分钟前`. Gate this on `zh-CN` or add a Traditional Chinese or translation-backed branch before merging.
  - confidence: 0.9
