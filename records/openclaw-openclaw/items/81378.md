## Summary

Keep open: the PR is useful, but it still has two source-reproducible merge blockers in the Cron session-target chip and zh-TW relative-time handling.

## What This Changes

The PR extracts Cron Control UI copy into i18n keys, adds zh-CN/generated locale updates, localizes Cron schedule prefixes, and passes the active UI locale into relative timestamp formatting.

## Best Possible Solution

Keep the Cron i18n extraction, but preserve explicit session-target semantics and keep zh-CN and zh-TW relative-time output distinct.

## Reproduction Assessment

Yes. Source inspection of the PR head shows the non-main session-target collapse and the `startsWith("zh")` locale branch that sends zh-TW through Simplified Chinese output.

## Solution Assessment

No, not yet. The extraction is the right direction, but the branch should preserve current Cron target semantics and locale boundaries before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Preserve custom cron session target labels:** `ui/src/ui/views/cron.ts:1537`
  - body: This renders every non-`main` cron job as the localized isolated label. `CronSessionTarget` still includes `current` and `session:<id>`, and the CLI docs describe those as active or persistent session targets, so existing jobs created through CLI/plugins will be mislabeled as isolated in the list. Preserve those specific labels or fall back to the raw target when it is not `main`/`isolated`.
  - confidence: 0.93
- **[P2] Preserve the zh-TW relative-time locale:** `src/infra/format-time/format-relative.ts:84`
  - body: The UI wrapper now passes the active Control UI locale into this formatter, so selecting `zh-TW` also enters this Simplified Chinese branch. Since `zh-CN` and `zh-TW` are separate supported bundles, Traditional Chinese users will see Simplified strings such as `刚刚` and `分钟前`. Gate this on `zh-CN` or add Traditional/translation-backed handling before merging.
  - confidence: 0.9
