## Summary

This PR should stay open because the Cron i18n extraction is useful, but the current patch still introduces a zh-TW regression and lacks inspectable real behavior proof from the contributor’s environment.

## What This Changes

This PR extracts Cron Control UI strings into i18n keys, adds zh-CN translations and generated locale metadata, and passes the active UI locale into Cron schedule and relative-time formatting.

## Best Possible Solution

Keep the Cron i18n extraction, but preserve the zh-CN/zh-TW split for relative-time formatting, add focused locale coverage, and require inspectable proof from a real zh-CN Cron tab run.

## Reproduction Assessment

Yes. Current main source shows Cron form/action strings still hardcoded in English while zh-CN is a supported Control UI locale; I did not run a browser reproduction in this read-only review.

## Solution Assessment

No, not as written. The i18n extraction is the right direction, but the relative-time change should gate on zh-CN specifically or provide locale-aware zh-TW output before wiring it through the UI.
