## Summary

Keep open: this is useful Cron zh-CN localization work with inspectable screenshot proof now attached, but the patch still regresses zh-TW relative-time output by treating every `zh*` locale as Simplified Chinese.

## What This Changes

The PR extracts Cron Control UI strings into i18n keys, adds zh-CN translations and generated locale metadata, and passes the active UI locale into schedule and relative-time formatting.

## Best Possible Solution

Keep the Cron i18n extraction, but make relative-time localization respect the exact locale boundary and cover zh-CN plus zh-TW with focused tests before merge.

## Reproduction Assessment

Yes. Current main source still has hardcoded English Cron advanced labels under a supported zh-CN locale, and the PR's before/after screenshots show the same UI path before and after localization.

## Solution Assessment

No, not yet. Extracting the strings is the right direction, but the relative-time formatter should gate Simplified Chinese on zh-CN or add a Traditional Chinese branch instead of collapsing all Chinese locales.
