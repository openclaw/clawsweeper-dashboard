## Summary

Keep open: the Cron zh-CN localization is useful and the supplied screenshot proves the main UI path, but the PR still regresses zh-TW relative timestamps by routing every `zh*` locale through Simplified Chinese strings.

## What This Changes

The PR extracts Cron Control UI copy into i18n keys, adds zh-CN translations, localizes schedule and relative-time display, and syncs generated locale metadata.

## Best Possible Solution

Keep the Cron i18n extraction, but limit Simplified relative-time strings to `zh-CN` or add distinct `zh-TW` strings with focused regression coverage.

## Reproduction Assessment

Yes. Current main still has hardcoded Cron strings under zh-CN, and PR-head source shows zh-TW now passes through the broad `startsWith("zh")` Simplified branch via the UI formatter.

## Solution Assessment

No, not yet. The string extraction is the right shape, but the formatter should preserve the supported zh-CN/zh-TW boundary instead of classifying all Chinese locales as Simplified.
