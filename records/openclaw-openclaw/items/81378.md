## Summary

Keep open: the latest PR head still has two source-reproducible regressions in existing Cron UI behavior, even though the zh-CN localization proof is sufficient and the security pass is clean.

## What This Changes

The PR extracts Cron Control UI copy into i18n keys, adds Simplified Chinese translations and generated locale updates, localizes Cron schedule labels, and passes the active UI locale into relative timestamp formatting.

## Best Possible Solution

Keep the Cron localization extraction, but preserve custom session target labels and maintain a clear zh-CN versus zh-TW relative-time boundary with focused regression coverage.

## Reproduction Assessment

Yes. A job with `sessionTarget: "session:<id>"` or `current` would render as isolated on PR head, and selecting `zh-TW` would take the Simplified Chinese relative-time branch by source inspection.

## Solution Assessment

No, not yet. The i18n extraction is the right direction, but the current patch loses supported session target specificity and conflates the supported Chinese locale bundles.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve custom cron session target labels:** `ui/src/ui/views/cron.ts:1537`
  - body: This renders every non-`main` cron job as the localized isolated label. `CronSessionTarget` still includes `current` and `session:<id>`, so jobs created through CLI/plugins with a persistent session target will appear isolated even though they are pinned to an existing session. Preserve those specific labels or fall back to the raw target when it is not `main`/`isolated`.
  - confidence: 0.92
- **[P2] Preserve the zh-TW relative-time locale:** `src/infra/format-time/format-relative.ts:84-88`
  - body: The UI wrapper now passes the active Control UI locale into this formatter, so selecting `zh-TW` also enters this Simplified Chinese branch. Since `zh-CN` and `zh-TW` are separate supported bundles, Traditional Chinese users will see Simplified strings such as `刚刚` and `分钟前`. Gate this on `zh-CN` or add a Traditional Chinese/translation-backed branch before merging.
  - confidence: 0.9
