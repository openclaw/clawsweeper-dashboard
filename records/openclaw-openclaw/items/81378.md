## Summary

Keep open: this addresses a real Cron Control UI localization gap and has useful visual proof, but the latest head still has source-reproducible localization regressions that should block merge.

## What This Changes

The PR extracts Cron Control UI copy into i18n keys, adds zh-CN/generated locale updates, localizes Cron schedule labels, and passes the active UI locale into relative-time formatting.

## Best Possible Solution

Keep the Cron i18n extraction, but preserve exact non-main session target labels, avoid Simplified Chinese output for zh-TW, and use the new localized key for the remaining action button.

## Reproduction Assessment

Yes. Source inspection of the latest PR head shows `current` and `session:<id>` collapse to the isolated chip, `zh-TW` enters the Simplified Chinese relative-time branch, and the Cron action button still has hardcoded English source text despite a zh-CN key.

## Solution Assessment

No. The extraction is the right direction, but the implementation needs narrower locale and target handling before it is the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Preserve custom cron session target labels:** `ui/src/ui/views/cron.ts:1612-1614`
  - body: This renders every non-`main` cron job as the localized isolated label. `CronSessionTarget` still includes `current` and `session:<id>`, and the CLI docs preserve those persistent-session targets, so jobs pinned to an existing session would appear isolated. Preserve those values or fall back to the raw target when it is not exactly `main` or `isolated`.
  - confidence: 0.92
- **[P2] Preserve the zh-TW relative-time locale:** `src/infra/format-time/format-relative.ts:84`
  - body: The UI wrapper passes the active Control UI locale into this formatter, and this branch treats `zh-TW` as Simplified Chinese. Since `zh-CN` and `zh-TW` are separate supported bundles, Traditional Chinese users would see Simplified strings such as `刚刚` and `分钟前`; gate this on `zh-CN` or add Traditional Chinese output.
  - confidence: 0.9
- **[P2] Wire the Run if due translation:** `ui/src/ui/views/cron.ts:1668`
  - body: The PR adds `cron.runIfDue` with a zh-CN translation, but this button still has hardcoded English source text in the Cron job actions. Simplified Chinese users can still see `Run if due`, so wire the new key here and cover it in the existing cron view test.
  - confidence: 0.86
