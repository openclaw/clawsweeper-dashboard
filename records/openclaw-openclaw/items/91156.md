## Summary

## What This Changes

The PR changes `resolveTelegramAccount` to pre-resolve omitted Telegram account IDs through the configured default account and adds focused resolver tests for omitted and explicit account selection.

## Best Possible Solution

Land the narrow resolver fix after redacted live multi-bot Telegram proof covers omitted-account default routing and explicit account routing in an upgrade-shaped setup with a top-level/default token present.

Yes, by source inspection: current main resolves omitted Telegram account IDs as the implicit `default` first and returns it when that account has credentials, bypassing the configured default. I did not run a live Telegram repro in this read-only review.

## Solution Assessment

Yes for the code shape: pre-resolving the configured default before the shared fallback helper matches the existing Telegram selector and the Nextcloud Talk sibling resolver. It is not merge-ready until the live Telegram proof gap is closed or consciously waived by maintainers.
