## Summary

Keep open: this PR fixes part of Telegram default-account resolution, but it is not merge-ready because the Telegram message-action send path still has an omitted-account token gate and the PR has only unit-test proof, not real Telegram proof.

## What This Changes

The PR changes Telegram account resolution so omitted account IDs pre-resolve the configured default account, and adds resolver unit tests for omitted and explicit accounts.

## Best Possible Solution

Make Telegram default-account resolution canonical across both the resolver and message-action runtime, add focused regression coverage, then require redacted live multi-bot Telegram proof before merge.

## Reproduction Assessment

Yes for source-level reproduction: current main resolves omitted Telegram accounts through the implicit default first, and the action-runtime send path still checks an omitted-account token before durable delivery. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

No as submitted: the resolver change is a good part of the fix, but the stated CLI/message-action send surface still has a token gate that bypasses the configured default account. The maintainable fix is to make both paths share the same account resolution and prove it live.
