## Label Justifications

- `P1`: The PR changes a real Telegram outbound channel workflow where wrong bot selection can break delivery or bot identity.
- `merge-risk: 🚨 compatibility`: Existing setups with both an implicit default token and configured defaultAccount may see no-account sends switch bot identity after merge.
- `merge-risk: 🚨 message-delivery`: If the configured default bot lacks access to a chat that the implicit default bot could send to, the same no-account send could fail or target a different sender.

## Summary

Keep open: current main and the latest release still use the old Telegram resolver, and this PR is a focused fix for the remaining default-account send path. The code shape looks sound, but the visible multi-bot Telegram behavior still needs real transport proof and maintainer acceptance of the upgrade-routing change before merge.

## What This Changes

The PR pre-resolves omitted Telegram account IDs through the configured default account before calling the shared fallback helper, and adds resolver tests for omitted and explicit account selection.

## Best Possible Solution

Land the narrow resolver fix after redacted live multi-bot Telegram proof covers omitted-account default routing and explicit-account routing in an upgrade-shaped setup with an implicit default token present.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes by source inspection and the added unit-test scenario: current main resolves an omitted account as implicit default first and returns it when that account has credentials, bypassing the configured default. I did not run a live Telegram reproduction.

## Solution Assessment

Is this the best way to solve the issue? Yes for the code shape: the token decision belongs in resolveTelegramAccount, and pre-resolving the configured default matches the existing Telegram selector and the Nextcloud Talk sibling resolver. It is not merge-ready until live Telegram proof or an explicit maintainer waiver covers the visible behavior change.
