## What This Changes

The issue requests schema or lookup metadata that tells agents whether a config field is hot-reloadable, restart-required, or a no-op.

## Best Possible Solution

Land an additive `reloadKind` lookup field that reuses the existing reload planner as the source of truth, then close this issue from the merged PR.

## Reproduction Assessment

## Solution Assessment

Yes, the best path is an additive lookup-level field backed by the existing reload matcher, not a second rules table. The issue should stay open while maintainers review and land or reject the linked PR.
