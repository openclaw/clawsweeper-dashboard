## Summary

This should stay open because the reported schema/API gap still exists on current main and is already paired with an open closing PR. Current source shows `config.schema.lookup` returns schema and UI hints without reload metadata, while the reload planner already classifies the same paths as hot, none, or restart-required.

## What This Changes

The issue requests reload-policy metadata for config schema consumers so tools can tell whether a config path hot-reloads, requires a Gateway restart, or is a no-op before applying changes.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/81612; if it lands, close this issue as implemented by that PR.

## Reproduction Assessment

Yes. Source inspection on current main shows `config.schema.lookup` cannot return reload metadata, and the issue discussion includes Docker/live reproduction evidence for the same field-level gap.

## Solution Assessment

Yes, with maintainer API acceptance. Reusing the existing reload-rule matcher for optional lookup metadata is the narrowest maintainable direction because it avoids a second rule table and leaves config patch behavior unchanged.
