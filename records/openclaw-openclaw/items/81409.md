## Summary

Keep open: current main still lacks reload metadata in the config schema lookup path, and the report is valid. An open closing PR already carries the proposed fix, so the issue should remain open until that PR or an equivalent maintainer-approved API change lands.

## What This Changes

Expose config reload impact metadata so agents and humans can see whether a config path hot-reloads, requires restart, or is a no-op before editing it.

## Best Possible Solution

Land or rebase https://github.com/openclaw/openclaw/pull/81612, or an equivalent additive lookup metadata change that reuses the existing reload planner as the single source of truth.

## Reproduction Assessment

Yes. Current-main source shows lookup responses cannot include reload metadata while the reload planner can classify the same paths, and the issue comments include a concrete Docker reproduction on OpenClaw 2026.5.7.

## Solution Assessment

Yes, with maintainer API approval. The linked PR's approach of reusing the existing reload-rule matcher for optional lookup metadata is narrower than duplicating a second rules table or changing reload behavior.
