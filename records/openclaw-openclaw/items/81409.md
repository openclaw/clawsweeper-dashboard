## Summary

Keep open: current main still lacks reload metadata in config schema lookup, and the active linked implementation PR is open and unmerged, so this issue should wait for that PR to land or be rejected.

## What This Changes

The issue asks OpenClaw to expose whether config schema fields are hot-reloadable, restart-required, or no-op so tools can plan config changes without trial-and-error.

## Best Possible Solution

Review and land the linked additive lookup metadata PR if maintainers accept the protocol surface; otherwise keep this issue as the canonical API decision until a replacement fix is chosen.

## Reproduction Assessment

Yes. Source inspection against current main shows lookup cannot return reload metadata while the reload planner already classifies the same paths; the issue discussion also includes a Docker reproduction on 2026.5.7.

## Solution Assessment

Yes, subject to maintainer API approval. Reusing the existing reload planner for an optional config.schema.lookup field is narrower than duplicating reload rules or adding a separate table-only command.
