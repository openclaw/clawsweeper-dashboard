## Summary

Current main still lacks reload/restart metadata on the config schema lookup surface, and an open linked PR is already the implementation candidate. This issue should stay open until that PR lands or maintainers choose a different additive contract.

## What This Changes

Expose config reload behavior metadata so schema/RPC consumers can tell whether a config path hot-reloads, requires restart, or is no-op before applying a patch.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81612 if its additive lookup contract is acceptable, then close this issue as implemented after merge.

## Reproduction Assessment

Yes. Source inspection shows config.schema.lookup currently returns schema/UI hint data only while reload behavior is computed separately, and the issue comments include Docker/current-main checks asserting reloadKind is absent.

## Solution Assessment

Yes. Adding optional reloadKind metadata to config.schema.lookup and its child summaries is the narrowest maintainable path because it reuses the existing reload planner without changing patch semantics or dumping a second rule table.
