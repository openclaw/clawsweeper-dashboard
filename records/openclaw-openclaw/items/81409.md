## Summary

Keep open: the schema metadata gap is valid on current main, and a live linked pull request explicitly closes it, so this issue should stay open until that PR is merged or rejected.

## What This Changes

Expose per-config-path reload behavior so schema lookup consumers can distinguish hot reload, restart-required, and no-op config changes before patching config.

## Best Possible Solution

Review and land the additive `config.schema.lookup` reload metadata in the linked PR if maintainers accept that protocol surface, keeping reload behavior sourced from the existing reload planner.

## Reproduction Assessment

Yes. Current source shows lookup cannot return reload metadata while the reload planner can classify the same paths, and the issue discussion includes an isolated Docker reproduction on `openclaw@2026.5.7`.

## Solution Assessment

Yes, subject to maintainer API approval. Reusing the existing reload-rule matcher for optional lookup metadata is narrower than duplicating a rule table or changing config patch semantics.
