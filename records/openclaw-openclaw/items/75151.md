## Summary

Keep open. Current main still has the reset durability gap: the reset path publishes a successor sessionFile before a successor JSONL is guaranteed to exist, and replay can return without creating that target file, leaving history lookup to return empty when no transcript candidate exists.

## What This Changes

Ensure context-overflow reset/rotation never persists a sessionFile pointing at a nonexistent transcript and preserves a valid continuity transcript.

## Best Possible Solution

Enforce a reset-time durability invariant: create a header-only successor transcript before publishing the new store entry, or keep the previous valid mapping if successor creation or replay fails.

## Reproduction Assessment

Yes. A focused unit test can call resetReplyRunSession with a previous entry whose source transcript is missing or assistant-only and assert that the persisted successor sessionFile exists; current replay coverage already proves one branch leaves the target absent.

## Solution Assessment

Yes. The issue's write-path guard is the narrow maintainable fix; read-path cleanup remains useful repair tooling but is too late to preserve active continuity.
