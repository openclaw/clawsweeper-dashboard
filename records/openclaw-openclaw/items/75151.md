## Summary

Keep open. Current main still publishes the reset session entry before the successor transcript is guaranteed to exist, and the replay helper has early-return paths that leave the published target missing.

## What This Changes

The issue asks OpenClaw to prevent context-overflow reset/rotation from persisting a `sessionFile` that points to a nonexistent transcript and orphaning the previous history.

## Best Possible Solution

Enforce a reset-time durability invariant: create a header-only successor transcript before publishing the new store entry, or keep the previous valid mapping if successor creation or replay fails.

## Reproduction Assessment

Yes, from source inspection. A focused regression test can call `resetReplyRunSession` with a missing or assistant-only previous transcript and assert the persisted successor `sessionFile` exists.

## Solution Assessment

Yes. The narrowest maintainable fix is a write-path guard in reset/replay durability; cleanup and read-path repair are useful follow-ups but too late to preserve active continuity.
