## Summary

Keep open. Current main still has a source-reproducible reset/replay path that can persist a new `sessionFile` before the successor transcript is guaranteed to exist, and the read path can then return empty history.

## What This Changes

The issue reports that context-overflow reset can update a session mapping to a new transcript path that does not exist, making history appear empty while the previous transcript remains on disk.

## Best Possible Solution

Enforce a reset-time durability invariant: create a header-only successor transcript before publishing the new store entry, or keep the previous valid mapping if successor creation or replay fails.

## Reproduction Assessment

Yes, from source inspection. A focused regression can call `resetReplyRunSession` with a missing or assistant-only previous transcript and assert the persisted successor `sessionFile` exists; the current replay test already proves one target-absent branch.

## Solution Assessment

Yes. The narrow maintainable fix is a write-path durability guard in reset/replay; read-path cleanup and `--fix-missing` remain useful recovery tools but are too late to preserve active continuity.
