## Summary

Keep open. The current reset-after-compaction-failure path can still publish a new active `sessionFile` before that JSONL is durable, and the replay helper still has early-return branches that leave the successor transcript absent, matching the reported empty-history failure.

## What This Changes

The issue asks OpenClaw to make context-overflow reset/rotation durable so `sessions.json` does not point an active session key at a nonexistent transcript while prior session history remains orphaned.

## Best Possible Solution

Enforce a reset-time durability invariant: create or prewarm the successor JSONL before publishing the new active store entry, make replay unable to early-return without a durable target once reset will persist the successor, and keep cleanup/doctor as repair for already-corrupt stores rather than the continuity mechanism.

## Reproduction Assessment

Yes. A focused unit reproduction can call `resetReplyRunSession` with a previous entry whose source transcript is missing or assistant-only, then assert that the persisted successor `sessionFile` exists; current source and tests show that target creation is skipped on those branches.

## Solution Assessment

Yes. The narrowest maintainable fix is at the reset/replay durability boundary; read-path cleanup is too late for active continuity, and the adjacent rollover/header PR does not cover this specific reset path unless it is explicitly reused here.
