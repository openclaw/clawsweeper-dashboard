## Summary

Keep open. Current main still has the reset durability gap: context-overflow recovery can persist a new `sessionFile` before the successor transcript is durable, and replay has branches that return without creating the target JSONL.

## What This Changes

The issue requests that context-overflow reset/rotation never persist a `sessionFile` pointing at a nonexistent transcript or orphan the prior session history.

## Best Possible Solution

Enforce a reset-time durability invariant by creating a header-only successor transcript before publishing the new store entry, or retaining the previous valid mapping if successor creation fails.

## Reproduction Assessment

Yes. A focused unit test can call `resetReplyRunSession` with a previous entry whose source transcript is missing or assistant-only, then assert the persisted successor `sessionFile` exists; current replay coverage already encodes one branch where the target remains absent.

## Solution Assessment

Yes. The narrowest maintainable fix belongs in the reset/replay path before the store entry is published; cleanup and read-path fallback are useful repair tools but too late to preserve active continuity.
