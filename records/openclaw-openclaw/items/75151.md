## Summary

Keep open. Current main still has the data-integrity gap described in the report: the reset path can persist a successor `sessionFile` before that JSONL is durable, the replay helper can no-op without creating the target transcript, and the history reader returns an empty list when no mapped transcript candidate exists.

## What This Changes

Prevent context-overflow session reset or rotation from persisting a `sessionFile` that points to a nonexistent transcript and orphaning the previous valid session history.

## Best Possible Solution

Enforce a generic embedded/direct session reset invariant: create or prewarm the successor transcript before switching the active store, retain the previous valid mapping if successor durability fails, refresh queued follow-up state only after that point, and add focused regression coverage for missing, empty, assistant-only, and normal replay cases.

## Reproduction Assessment

Yes. A focused unit reproduction can drive `resetReplyRunSession` with a prior mapping whose replay source is missing or has no user-starting tail, then assert the persisted successor `sessionFile` exists and `readSessionMessages` does not silently return empty because the mapped JSONL is absent.

## Solution Assessment

Yes. The narrow maintainable fix is to enforce transcript durability at the reset boundary; cleanup/doctor repairs are useful diagnostics but happen too late for the active-session invariant.
