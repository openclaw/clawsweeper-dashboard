## Summary

Keep open. Current `main` still has the reset durability gap: the overflow reset path persists a new `sessionFile` before the successor JSONL is guaranteed to exist, and replay still has branches that return without creating the target file.

## What This Changes

The issue requests a durable context-overflow reset path so `sessions.json` never maps an active Telegram direct session to a nonexistent transcript while the prior transcript remains orphaned.

## Best Possible Solution

Enforce a reset-time durability invariant: create a header-only successor transcript before publishing the new store entry, or keep the previous valid mapping if successor creation or replay fails.

## Reproduction Assessment

Yes. A focused unit test can call `resetReplyRunSession` with a previous entry whose source transcript is missing or assistant-only and assert that the persisted successor `sessionFile` exists; existing replay coverage already proves one branch leaves the target absent.

## Solution Assessment

Yes. The issue’s suggested write-path guard is the narrow maintainable fix; read-path cleanup is useful repair tooling but too late to preserve active continuity.
