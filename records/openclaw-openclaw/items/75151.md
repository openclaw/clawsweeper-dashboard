## Summary

Keep open. Current main still has a source-reproducible reset durability gap: `resetReplyRunSession` publishes a new `sessionFile` before the successor transcript is guaranteed to exist, and replay can still return without creating that target.

## What This Changes

Ensure context-overflow/session-reset rollover never persists a `sessionFile` that points to a missing transcript and preserves recoverable session history.

## Best Possible Solution

Enforce a reset-time durability invariant: create a header-only successor transcript before publishing the new store entry, or keep the previous valid mapping if successor creation or replay fails.

## Reproduction Assessment

Yes, source-level. A focused test can call `resetReplyRunSession` with a missing or assistant-only previous transcript and then assert that the persisted successor `sessionFile` exists instead of allowing history to resolve to `[]`.

## Solution Assessment

Yes for keeping this open and fixing the write path. Cleanup and read-path repair are useful diagnostics, but the narrow maintainable fix is to make reset publication durable before the store points at the successor transcript.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/75151 on current main. The reset path currently publishes `nextEntry.sessionFile` in `src/auto-reply/reply/agent-runner-session-reset.ts` before any code guarantees that the successor JSONL exists, and `replayRecentUserAssistantMessages` can return without creating the target. Enforce a reset-time durability invariant: create a header-only successor transcript with non-clobbering/exclusive semantics before `updateSessionStore`, or keep the previous valid mapping if successor creation fails. Do not duplicate the separate `initSessionState` work in https://github.com/openclaw/openclaw/pull/55817 or the compaction-failure UX work in https://github.com/openclaw/openclaw/pull/70479; rebase/coordinate with them if touching shared helpers. Add focused regression coverage for missing-source and assistant-only previous transcripts, asserting the persisted successor `sessionFile` exists and history does not go empty solely because the path is absent. Add a `CHANGELOG.md` fix entry because this is user-visible session continuity behavior.
