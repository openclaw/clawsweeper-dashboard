## Summary

Keep open. Current main still allows the context-overflow reset path to persist a successor `sessionFile` before the successor JSONL is durable, and the replay helper can return without creating the target transcript, matching the reported empty-history failure mode.

## What This Changes

The issue asks OpenClaw to make context-overflow/session-reset rotation durable so `sessions.json` never points an active Telegram direct session at a nonexistent transcript while prior history remains orphaned.

## Best Possible Solution

Enforce a reset-time invariant for embedded/direct sessions: create or prewarm the successor JSONL before updating the active store, keep the previous valid mapping if creation fails, make replay unable to early-exit without a durable target when the store has already moved, and keep cleanup/doctor as a separate repair path for already-corrupt stores.

## Reproduction Assessment

Yes. A focused unit reproduction can call `resetReplyRunSession` with a previous entry whose source transcript is missing or assistant-only, then assert that the persisted successor `sessionFile` exists; current source and tests show that branch can leave the target absent.

## Solution Assessment

Yes. The narrowest maintainable fix is at the reset/rotation durability boundary; cleanup is too late for active continuity, and the related `initSessionState` helper work does not cover this reset path unless it is explicitly reused here.
