## Summary

Keep open. Current main still has a reset path that publishes a successor `sessionFile` before any successor transcript is guaranteed to exist, and the replay helper can return without creating that target file.

## What This Changes

The issue asks OpenClaw to prevent context-overflow/reset rollover from updating `sessions.json` to a nonexistent transcript and orphaning the prior valid session history.

## Best Possible Solution

Enforce a reset-time durability invariant: create a header-only successor transcript before publishing the new store entry, or keep the previous valid mapping if successor creation or replay fails.

## Reproduction Assessment

Yes, from source inspection. A focused regression can call `resetReplyRunSession` with a missing or assistant-only previous transcript and then assert the persisted successor `sessionFile` exists; current replay code already has an absent-target branch.

## Solution Assessment

Yes. The narrow maintainable fix is a write-path durability guard in reset/replay; cleanup and read-path repair are useful diagnostics but happen too late to preserve active continuity.
