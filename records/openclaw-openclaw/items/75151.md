## Summary

Keep open. Current main still has the reset durability gap: the overflow/compaction reset path can publish a new `sessionFile` before the successor JSONL is guaranteed to exist, and replay still has early-return branches that leave the target absent.

## What This Changes

The issue asks overflow reset/rotation to keep `sessions.json` mapped to an existing durable transcript or the previous valid transcript so history lookup does not go empty.

## Best Possible Solution

Enforce a reset-time durability invariant: create a header-only successor transcript before publishing the new store entry, or keep the previous valid mapping if successor creation or replay fails.

## Reproduction Assessment

Yes. A focused source-level reproduction can call `resetReplyRunSession` with a previous entry whose source transcript is missing or assistant-only and then assert the persisted successor `sessionFile` does not exist; existing replay coverage already proves the assistant-only target remains absent.

## Solution Assessment

Yes. The narrow maintainable fix is a write-path durability guard or prewarm before store publication; read-path cleanup and `--fix-missing` are useful recovery tools but are too late to preserve active continuity.

## Repair Work Prompt

Fix the reset durability bug from https://github.com/openclaw/openclaw/issues/75151. Current main lets `resetReplyRunSession` persist a new `sessionFile` before the successor JSONL is guaranteed to exist, while `replayRecentUserAssistantMessages` can return before creating the target when the source transcript is missing, has no kept user/assistant records, or has an assistant-only retained tail. Implement a narrow write-path invariant: before publishing the reset store entry, durably create a header-only successor transcript, or keep the previous valid mapping if successor creation fails. Reuse or align with any header helper introduced by the related open PR #55817; do not add a new config option or broad transcript-rotation feature. Add regression coverage for missing-source and assistant-only replay/reset cases, preserve existing replay behavior for valid alternating tails, and add a single-line user-facing changelog entry for the bug fix.
