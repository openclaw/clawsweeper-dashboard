## Summary

Keep open: current `main` still appears to compute `models.small_params` web/browser exposure without `tools.byProvider`, while runtime policy resolution does honor provider/model-scoped policy. The remaining question is security-policy semantics, because current tests intentionally keep small models critical unless sandboxing is also enabled.

## What This Changes

The issue asks the security audit to apply resolved provider/model `tools.byProvider` policy when reporting small-model web/browser exposure.

## Best Possible Solution

Have the audit compute per-model web/browser exposure through the same provider/model-scoped effective policy path as runtime, then update tests and docs to state whether denied web tools alone downgrade the finding or sandboxing remains required.

## Reproduction Assessment

Yes, source-level: the current collector omits `tools.byProvider`, while runtime resolution includes provider/model-scoped policy. I did not run the packaged CLI because this review was read-only, but the code path is direct.

## Solution Assessment

Unclear for the full requested downgrade: reusing runtime policy resolution is the narrow fix for the inaccurate `web=[...]` detail, but changing critical eligibility needs maintainer/security policy confirmation.
