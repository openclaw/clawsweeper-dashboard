## Summary

Current main already fixes the reported google-vertex unknown-model failure by adding `gemini-3.5-flash` to the bundled Google catalog used by Vertex and by covering the Gemini 3.5 Flash forward-compat resolver path; the fix is main-only after `v2026.6.10`.

## What This Changes

The issue reports `google-vertex/gemini-3.5-flash` failing as an unknown model after upgrading to `v2026.6.10` with Vertex ADC configured.

## Best Possible Solution

Keep the provider-owned current-main fix and let affected `v2026.6.10` users retest a build after `4d636335dbd1d4de078ee729039627709c7049ec` or the next stable release.

## Reproduction Assessment

Yes for the shipped release: the `v2026.6.10` source lacks the `gemini-3.5-flash` catalog row, matching the unknown-model error. Current main no longer reproduces that lookup miss because the row is present for the Vertex catalog.

## Solution Assessment

Yes. The best fix is the provider-owned Google catalog and forward-compat resolver update that landed on main, not a user config workaround or a core runtime shim.
