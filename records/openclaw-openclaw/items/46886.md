## Summary

Keep open. Current main and the latest shipped tag have gained typed `sessionKey` wake plumbing, but direct `cron` tool wake calls still send only `mode` and `text`, and the cron wake contract still has no `agentId` target. The related open PR at https://github.com/openclaw/openclaw/pull/57199 covers part of the wake target surface but needs a coordinated refresh and does not update the agent `cron` tool path.

## What This Changes

The issue requests optional direct cron wake targeting so wake calls can address a non-default agent or session instead of always falling back to the default agent.

## Best Possible Solution

Refresh or replace the related wake-target PR with one additive contract update that adds `agentId` end to end, threads target metadata through the `cron` agent tool, preserves current `sessionKey` semantics, and updates docs/generated clients/tests together.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the missing support: direct `cron` wake calls still drop target metadata at the tool boundary, and the service wake contract still has no `agentId` field.

## Solution Assessment

Yes, the best direction is an additive target-plumbing update through the existing wake contract. The concrete implementation should be coordinated with https://github.com/openclaw/openclaw/pull/57199 and must preserve the already-merged `sessionKey` behavior.
