## Summary

Keep open: the linked privacy problem is still source-reproducible on current main, and this PR addresses the right boundary, but the branch is not merge-ready because it lacks real behavior proof, is conflicting, and introduces non-media error formatting regressions plus an unsafe recursive cause walk.

## What This Changes

The branch adds a MediaFetchError safe-message helper, sanitizes media fetch failures at attachment hydration, broadcast result, and message_sent boundaries, and adds focused sanitizer tests.

## Best Possible Solution

Land a rebased media-specific sanitizer that maps MediaFetchError to one stable channel-safe message, preserves formatErrorMessage for non-media failures, uses cycle-safe cause traversal, and includes focused tests plus redacted real channel proof.

## Reproduction Assessment

Yes, from source inspection. Current main creates detailed MediaFetchError strings, forwards loadWebMedia failures from attachment hydration, and formats broadcast/message_sent failures without a media-safe substitution.

## Solution Assessment

No, not as proposed. The direction is right, but the patch should keep the shared formatter/redaction fallback for non-media failures, make cause traversal cycle-safe, rebase, and add real behavior proof.
