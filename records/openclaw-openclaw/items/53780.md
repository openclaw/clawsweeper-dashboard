## Summary

Keep open: current main still lacks the requested media-specific sanitizer, but this PR needs rebase, real behavior proof, and targeted fixes before it can merge. The branch is useful and directly tied to the privacy report, yet it currently bypasses shared error formatting for unrelated failures and has an unsafe recursive cause-chain helper.

## What This Changes

The PR adds a MediaFetchError safe message/helper, redacts media fetch failures in outbound action and delivery paths, and adds focused sanitizer tests.

## Best Possible Solution

Land a rebased media-specific sanitizer that maps MediaFetchError to one stable channel-safe message, preserves formatErrorMessage for all other failures, uses cycle-safe cause traversal, and includes focused tests plus real channel proof.

## Reproduction Assessment

Yes, from source inspection. Current main creates detailed MediaFetchError text, forwards loadWebMedia failures from attachment hydration, and formats broadcast/message_sent failures without a media-safe substitution.

## Solution Assessment

No, not as proposed. The direction is right, but the patch should preserve the shared formatter/redaction for every non-media failure, make cause traversal cycle-safe, and be rebased with real behavior proof.
