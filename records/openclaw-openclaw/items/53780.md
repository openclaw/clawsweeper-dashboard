## Summary

Keep open. Current main still lacks the media-specific outbound sanitizer, and this PR is useful source work, but the branch has concrete blocking regressions in non-media error handling plus an unsafe recursive cause-chain walk. Reviewed item: https://github.com/openclaw/openclaw/pull/53780

## What This Changes

The PR adds a safe MediaFetchError message/helper, redacts media fetch failures at attachment hydration, broadcast result, and message_sent failure boundaries, and adds focused sanitizer tests.

## Best Possible Solution

Land a rebased media-specific sanitizer that maps MediaFetchError to one stable channel-safe message at outbound action and hook boundaries, keeps formatErrorMessage as the fallback for every non-media failure, uses cycle-safe cause traversal, and covers attachment hydration, broadcast results, and message_sent failures in tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: failed loadWebMedia produces detailed MediaFetchError text, attachment hydration forwards it, and current broadcast/message_sent paths format that error instead of substituting a stable media-safe message.

## Solution Assessment

No, not as proposed. The direction is right, but the patch should preserve the shared non-media error formatter and make the MediaFetchError detector cycle-safe before merge.
