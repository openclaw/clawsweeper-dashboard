## Summary

Keep open: this is a plausible privacy fix for a source-reproducible media error leak, but the current branch has blocking correctness/security regressions and still lacks required after-fix real behavior proof.

## What This Changes

The branch adds a safe MediaFetchError message/helper, replaces media fetch errors at attachment hydration, broadcast result, and message_sent boundaries, and adds focused sanitizer tests.

## Best Possible Solution

Land a rebased media-specific sanitizer that maps MediaFetchError to one stable channel-safe message, preserves formatErrorMessage for non-media failures, uses cycle-safe cause traversal, and includes focused tests plus redacted real channel/runtime proof.

## Reproduction Assessment

Yes, from source inspection. Current main creates detailed MediaFetchError strings, forwards loadWebMedia failures from attachment hydration, and formats broadcast/message_sent failures without a media-safe substitution.

## Solution Assessment

No, not as currently proposed. The direction is right, but the patch should keep the shared formatter/redaction fallback for non-media failures, make cause traversal cycle-safe, rebase, and add real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve formatted non-media message_sent errors:** `src/infra/outbound/deliver.ts:759-760`
  - body: For non-MediaFetchError failures, this uses err.message or "unknown error" instead of the shared formatter on current main. That drops redaction, non-Error handling, and cause-chain handling for hook-visible message_sent failures; keep the media replacement but use formatErrorMessage(err) as the fallback.
  - confidence: 0.91

  - body: The broadcast result path has the same regression: unrelated plugin or adapter failures lose shared redaction, non-Error handling, and cause-chain text. Only replace the text when the caught error is a media fetch failure; all other errors should keep formatErrorMessage(err).
  - confidence: 0.89
- **[P2] Make media error cause traversal cycle-safe:** `src/media/fetch.ts:34`
  - body: isMediaFetchError is called on arbitrary caught outbound errors, but it recursively follows Error.cause without tracking seen values. A cyclic cause chain from a plugin or adapter error would overflow before the original failure can be handled.
  - confidence: 0.84
