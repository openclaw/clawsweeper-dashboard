## Summary

Keep this PR open: current main still lacks the media-fetch-specific outbound sanitizer, but this branch has blocking regressions that should be fixed before merge.

## What This Changes

The PR adds a safe MediaFetchError message/helper, redacts media fetch failures at attachment hydration, broadcast result, and message_sent failure boundaries, and adds focused sanitizer tests.

## Best Possible Solution

Land a rebased media-specific sanitizer that maps MediaFetchError to one stable channel-safe message at outbound action and hook boundaries, preserves formatErrorMessage for all non-media failures, uses cycle-safe cause traversal, and includes the required changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: loadWebMedia can throw detailed MediaFetchError text, hydrateAttachmentPayload forwards it, and current broadcast/message_sent paths format that error without a media-safe substitution.

## Solution Assessment

No, not as proposed. The direction is right, but the patch should keep formatErrorMessage as the fallback for non-media errors and make cause-chain detection cycle-safe before merge.
