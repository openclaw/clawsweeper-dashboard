## Summary

Keep open: current main still lacks a media-specific safe error path, but this privacy-sensitive PR has blocking correctness/security findings and still lacks after-fix real behavior proof.

## What This Changes

The PR adds a safe MediaFetchError message/helper, sanitizes outbound media fetch failures at attachment hydration, broadcast result, and message_sent boundaries, and adds focused tests.

## Best Possible Solution

Land a rebased media-specific sanitizer that maps MediaFetchError to one stable channel-safe message, preserves formatErrorMessage for all other failures, uses cycle-safe cause traversal, and includes focused tests plus real channel proof.

## Reproduction Assessment

Yes, from source inspection. Current main creates detailed MediaFetchError strings, forwards loadWebMedia failures from attachment hydration, and formats broadcast/message_sent failures without a media-safe substitution.

## Solution Assessment

No, not as proposed. The direction is right, but the patch should keep the shared formatter/redaction fallback for non-media failures, make cause traversal cycle-safe, rebase, and add real behavior proof.
