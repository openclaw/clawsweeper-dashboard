## Summary

Keep open: this is an active privacy fix for a source-reproducible media error leak, but the current patch regresses non-media error formatting/redaction, has an unsafe recursive cause walk, lacks contributor real behavior proof, and is currently conflicting with main.

## What This Changes

The branch adds a safe MediaFetchError message/helper, sanitizes media fetch failures at attachment hydration, broadcast result, and message_sent delivery boundaries, and adds focused tests.

## Best Possible Solution

Land a rebased media-specific sanitizer that maps MediaFetchError to one stable channel-safe message at outbound action/hook boundaries, preserves formatErrorMessage for other failures, uses cycle-safe detection, and includes focused tests plus redacted real runtime proof.

## Reproduction Assessment

Yes, from source inspection: current main creates detailed MediaFetchError strings and forwards loadWebMedia, broadcast, and message_sent failures without a media-specific safe substitution at the targeted boundaries. I did not run a live channel reproduction.

## Solution Assessment

No, not as currently proposed. The direction is right, but the patch should preserve the shared formatter for non-media failures, make cause traversal cycle-safe, rebase, and include real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

  - confidence: 0.91

  - body: The broadcast failure path has the same regression: unrelated plugin or adapter failures lose shared redaction, non-Error handling, and cause-chain diagnostics. Only substitute SAFE_MEDIA_FETCH_ERROR_MESSAGE for MediaFetchError and leave other errors on formatErrorMessage(err).
  - confidence: 0.89
- **[P2] Guard cause traversal against cycles:** `src/media/fetch.ts:34-41`
  - body: isMediaFetchError follows Error.cause recursively on arbitrary outbound/plugin errors without tracking seen values. A cyclic cause chain would overflow before the error can be handled; use an iterative walk or a seen set like formatErrorMessage.
  - confidence: 0.84
