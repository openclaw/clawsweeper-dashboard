## Summary

Keep open: current main still lacks the media-specific sanitizer, but this PR cannot merge as-is because it regresses non-media outbound diagnostics/redaction and its cause-chain helper is not cycle-safe; after-fix real behavior proof is also missing.

## What This Changes

The PR adds a safe MediaFetchError message/helper, applies it at attachment hydration, broadcast-result, and message_sent failure boundaries, and adds sanitizer tests.

## Best Possible Solution

Land a rebased media-specific sanitizer that maps MediaFetchError to one stable channel-safe message, preserves formatErrorMessage for all non-media failures, uses cycle-safe cause traversal, and adds focused tests plus real channel proof.

## Reproduction Assessment

Yes, from source inspection. A failed loadWebMedia can raise detailed MediaFetchError text, hydrateAttachmentPayload forwards it, and broadcast/message_sent paths currently copy formatErrorMessage into returned results or hooks without a media-safe substitution.

## Solution Assessment

No, not as proposed. The direction is right, but the patch should preserve existing formatter/redaction behavior for all non-media failures and make MediaFetchError cause-chain detection cycle-safe before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve non-media message_sent diagnostics:** `src/infra/outbound/deliver.ts:758-760`
  - body: The new fallback uses `err.message` or `"unknown error"` before emitting `message_sent`, bypassing the existing shared formatter for non-media failures.   - confidence: 0.91

  - body: The broadcast failure path has the same regression: non-media thrown strings or objects become `"unknown error"` in per-target results. Only replace the text for MediaFetchError; all other failures should keep the current formatted diagnostic path.
  - confidence: 0.89
- **[P2] Guard cause-chain traversal against cycles:** `src/media/fetch.ts:40-41`
  - body: `isMediaFetchError` recursively follows `Error.cause` on arbitrary plugin or adapter errors. A cyclic cause chain would overflow before the outbound failure can be handled; traverse iteratively with a seen set like `formatErrorMessage`.
  - confidence: 0.83
