## Summary

## What This Changes

The PR adds a safe MediaFetchError message/helper, redacts media fetch failures in attachment hydration, broadcast results, and message_sent delivery failures, and adds sanitizer tests.

## Best Possible Solution

Land a rebased media-specific sanitizer that maps MediaFetchError to one stable channel-safe message, preserves formatErrorMessage for non-media failures, uses cycle-safe cause traversal, and includes focused tests plus redacted real channel proof.

## Reproduction Assessment

Yes, from source inspection. Current main creates detailed MediaFetchError strings, forwards loadWebMedia failures during attachment hydration, and formats broadcast/message_sent failures without a media-safe substitution.

## Solution Assessment

No, not as currently proposed. The direction is right, but the patch should preserve the shared non-media formatter fallback, make cause traversal cycle-safe, rebase, and include real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve formatted non-media message_sent errors:** `src/infra/outbound/deliver.ts:759-760`
  - body: For non-MediaFetchError failures, this switches message_sent failure output from formatErrorMessage(err) to err.message or "unknown error".   - confidence: 0.91
- **[P2] Preserve formatted non-media broadcast errors:** `src/infra/outbound/message-action-runner.ts:348-353`
  - body: The broadcast result path has the same regression: non-media plugin or adapter failures lose shared formatting and redaction and non-Error values become "unknown error". Only replace the text for MediaFetchError; all other errors should keep formatErrorMessage(err).
  - confidence: 0.89
- **[P2] Guard cause-chain traversal against cycles:** `src/media/fetch.ts:40-41`
  - body: isMediaFetchError recursively follows Error.cause on arbitrary caught outbound errors without tracking seen values. A cyclic cause chain from a plugin or adapter error would overflow before the original failure can be handled; use an iterative traversal with a seen set.
  - confidence: 0.84
