## Summary

Keep open: current main still has the reported WebUI `/new` usability gap. The Control UI creates dashboard-scoped sessions without a label, Gateway only persists an explicit `label`, and the local `openclaw sessions` projection/table cannot show labels even when stored.

## What This Changes

Automatically assign friendly labels to WebUI-created dashboard sessions and surface those labels in `openclaw sessions`.

## Best Possible Solution

Use the existing session `label` contract to assign a collision-safe default label for generated dashboard sessions, then expose stored labels additively in CLI JSON and text output while keeping internal keys available.

## Reproduction Assessment

Yes. Source inspection traces typed `/new` to `createChatSession`, which calls `sessions.create` without a label; Gateway persists only explicit labels, and CLI display rows omit labels.

## Solution Assessment

Yes for a narrow first pass. Reusing `sessions.create`/`sessions.patch` label plumbing is the maintainable direction; automatic title derivation from the first message should remain follow-up scope.
