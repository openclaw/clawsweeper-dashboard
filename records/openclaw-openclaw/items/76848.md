## Summary

Keep open: current main still creates generated dashboard sessions without a default friendly label, and an open linked PR is already the implementation vehicle for this issue.

## What This Changes

Request automatic friendly labels for WebUI `/new` dashboard sessions and better CLI/UI display of those labels.

## Best Possible Solution

Use the existing session `label` contract for generated dashboard sessions, preserve explicit labels, and surface stored labels in session list output where users need to distinguish WebUI chats.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: WebUI `/new` sends no label, the Gateway only stores an explicit `p.label`, and the CLI display projection omits labels.

## Solution Assessment

Yes, the narrow maintainable direction is to reuse the existing session label plumbing rather than changing session keys. The open linked PR should be reviewed against the full user-visible goal, especially CLI label display.
