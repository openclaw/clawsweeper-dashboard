## Summary

Keep open: current main still creates dashboard sessions from the Control UI `/new` path without a friendly label, and #77037 is an open closing PR rather than a merged fix.

## What This Changes

The issue requests automatic friendly labels for WebUI `/new` dashboard sessions and friendlier CLI/UI session-list output.

## Best Possible Solution

Review and land or revise #77037, using the existing session `label` contract for generated dashboard sessions and deciding the CLI/UI display behavior deliberately.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Control UI `/new` sends no label, Gateway stores only explicit labels, and CLI display rows omit stored labels.

## Solution Assessment

Yes for the broad direction: reusing the existing session `label` contract is the narrow maintainable solution. The active PR should be reviewed for whether a Gateway default label alone is sufficient or whether CLI label display should be included too.
