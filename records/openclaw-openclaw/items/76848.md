## Summary

Keep open: current main still creates unlabeled dashboard sessions and the CLI row projection still cannot show stored labels; the open implementation PR https://github.com/openclaw/openclaw/pull/77037 is the active candidate and has not merged.

## What This Changes

The issue asks for WebUI `/new` dashboard sessions to receive friendly labels automatically, with session listings able to show those labels instead of only UUID-like internal keys.

## Best Possible Solution

Use the existing session `label` contract to assign a default label for generated dashboard sessions and expose stored labels in CLI JSON/text output without changing session keys.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Control UI `/new` sends no label, Gateway stores only explicit labels, and the CLI display projection omits labels.

## Solution Assessment

No, current main is not the solution yet. The maintainable path is to reuse the existing label plumbing, but the open PR should be reviewed for both default label generation and the requested CLI visibility before this issue closes.
