## Summary

Keep open: current main still has the WebUI `/new` auto-label gap. The Control UI creates dashboard sessions without a label, Gateway persists only an explicit `label`, and local `openclaw sessions` output still cannot show stored labels.

## What This Changes

The issue asks for WebUI `/new` dashboard sessions to receive friendly labels automatically and for those labels to be visible in session listings.

## Best Possible Solution

Add a narrow default label for generated dashboard sessions, surface stored labels in local CLI JSON/text output while preserving keys, and leave first-message-derived titles for a later enhancement.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Control UI `/new` sends no label, Gateway stores only explicit labels, and the local CLI row projection omits labels.

## Solution Assessment

Yes. Reusing the existing `sessions.create`/`sessions.patch` label contract is the narrow maintainable solution; derived titles should stay out of the initial fix.
