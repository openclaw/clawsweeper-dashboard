## Summary

Keep open: current main still creates WebUI dashboard sessions without an automatic friendly label and the CLI session listing path cannot surface stored labels, so the reported usability gap remains.

## What This Changes

The issue requests automatic friendly labels for dashboard sessions created by the WebUI `/new` or New Chat flow and label-aware `openclaw sessions` output.

## Best Possible Solution

Use the existing session `label` contract to assign a safe default label for generated dashboard sessions and expose stored labels in CLI JSON/text output without changing session keys or migrating old sessions.

## Reproduction Assessment

Yes, by source inspection. The Control UI `/new` path sends no label, Gateway only stores explicit labels, and the CLI projection omits labels before JSON/text rendering.

## Solution Assessment

Yes. Reusing `sessions.create` label plumbing is the narrow maintainable fix; derived first-message titles can remain a later enhancement.
