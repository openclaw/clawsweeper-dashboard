## Summary

Keep open: current main still has session label plumbing, but the WebUI `/new` path does not send a label, Gateway only stores an explicit label, and CLI session rows drop stored labels before text or JSON output.

## What This Changes

The issue asks the WebUI `/new`/New Chat dashboard session creation path to assign a friendly label automatically and expose that label in session listings.

## Best Possible Solution

Use the existing session `label` plumbing to generate a friendly default for unlabeled dashboard sessions, keep the internal key available, and surface the label consistently in CLI text/JSON plus focused docs and tests.

## Reproduction Assessment

Yes. Source inspection on current main shows Control UI `/new` creates a dashboard session without `label`, Gateway only stores an explicit `p.label`, and CLI rows omit stored labels; no live WebUI run is needed to establish that path.

## Solution Assessment

Yes. The narrow maintainable path is to reuse the existing label field for a generated dashboard label now, while leaving first-message title derivation or broader session management for later work.
