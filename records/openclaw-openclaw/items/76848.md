## Summary

Keep open: current main still creates dashboard sessions without a default label and the CLI session listing path cannot display stored labels, while open PR #77037 is the active closing implementation candidate.

## What This Changes

WebUI `/new` should automatically assign readable labels to generated dashboard sessions and make session listings easier to identify than raw dashboard UUID keys.

## Best Possible Solution

Land a reviewed fix that reuses the existing session `label` contract for generated dashboard sessions and exposes that label in session list output without changing stable session keys.

## Reproduction Assessment

Yes. Source inspection on current main shows the Control UI `/new` path sends no label, Gateway persists only supplied labels, and the CLI projection/table omit labels.

## Solution Assessment

Yes. Reusing the existing `sessions.create`/session-store label field is the narrowest maintainable direction; the remaining choice is the default text and how CLI rows should present it.
