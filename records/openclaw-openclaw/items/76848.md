## Summary

Keep open: current main still creates unlabeled dashboard sessions from the Control UI `/new` path and the CLI session table cannot surface stored labels. Open PR #77037 references this issue with closing syntax, so this issue should stay open until that PR is reviewed, merged, or superseded.

## What This Changes

The issue requests automatic friendly labels for WebUI `/new` dashboard sessions and session-list display that can surface those labels.

## Best Possible Solution

Review #77037 and land or supersede a complete fix that uses the existing session `label` contract for generated dashboard sessions and surfaces labels where session lists render them.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction path: Control UI `/new` creates a dashboard session without a label, Gateway stores only explicit labels, and CLI row projection omits labels.

## Solution Assessment

Yes for the main direction: the existing `sessions.create`/session-store `label` field is the narrow maintainable seam. The linked PR should be checked for completeness against the CLI display part before this issue is considered solved.
