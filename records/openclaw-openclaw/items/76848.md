## Summary

Keep open: current main still creates WebUI dashboard sessions without a friendly label and `openclaw sessions` cannot surface stored labels from the offline session-store projection. This is a narrow UX feature, not a broken existing contract.

## What This Changes

WebUI `/new` and New Chat dashboard sessions should get friendly automatic labels, and session listings should expose those labels.

## Best Possible Solution

Use the existing session `label` contract to assign collision-safe default labels for generated Control UI dashboard sessions, then include stored labels in CLI JSON and text output without changing session keys.

## Reproduction Assessment

Yes. Source inspection on current main shows `/new` reaches `createChatSession`, which sends no label; Gateway only stores `p.label`; and the CLI row projection omits labels.

## Solution Assessment

Yes. Reusing the existing `sessions.create`/session-store label plumbing is the narrow maintainable solution; first-message derived titles can remain a later enhancement.
