## Summary

Keep open: current main still sends WebUI `/new` dashboard session creation without a label and the CLI display projection omits stored labels, while #77037 is an open closing PR that should land or be adjusted before this issue is closed.

## What This Changes

Request that WebUI `/new` dashboard sessions receive automatic friendly labels and that session lists can show those labels instead of only internal dashboard UUID keys.

## Best Possible Solution

Land an equivalent fix that uses existing session label storage for generated dashboard sessions and updates session list surfaces to display the label while preserving stable internal keys.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: typed `/new` dispatches to New Chat session creation, the WebUI create params omit `label`, the gateway only stores explicit labels, and the CLI row projection omits label fields.

## Solution Assessment

Yes. Reusing the existing `sessions.create`/session-store `label` contract is the narrow maintainable direction; #77037 or an adjusted equivalent should be reviewed before closing this issue.
