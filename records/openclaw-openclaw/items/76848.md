## Summary

Keep open: current main still routes Control UI `/new` and New Chat through dashboard session creation without supplying a label, while the CLI session projection/table cannot display stored labels. This is a narrow user-facing UX enhancement, not a shipped-behavior bug.

## What This Changes

Request that WebUI-created dashboard sessions receive automatic friendly labels and that session lists can surface those labels.

## Best Possible Solution

Reuse the existing session `label` contract to assign a deterministic default label to generated dashboard sessions and include stored labels in CLI JSON/text output without changing session keys.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Control UI `/new` sends no label, Gateway only persists explicit labels, and the CLI row projection/table drops labels.

## Solution Assessment

Yes. The narrowest maintainable fix is to reuse `sessions.create`/`sessions.patch` label plumbing and keep derived first-message titles as a later enhancement if desired.
