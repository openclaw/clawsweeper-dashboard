## Summary

Keep open: current main has session label plumbing, but the Control UI `/new` path does not send a label and the CLI session rows do not retain stored labels, so the reported usability gap remains.

## What This Changes

Request automatic friendly labels for Control UI `/new` dashboard sessions and label-aware `openclaw sessions` output.

## Best Possible Solution

Add a collision-safe default label for unlabeled auto-generated dashboard sessions, surface `label` in CLI text/JSON output, and document the behavior.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: Control UI `/new` creates a dashboard session without `label`, Gateway only stores an explicit `p.label`, and CLI rows omit stored labels. I did not need a live WebUI run to establish the current-main path.

## Solution Assessment

Yes: using the existing session `label` plumbing is the narrow maintainable direction, with a simple generated dashboard label now and any first-message title derivation left as a later enhancement.
