## Summary

Keep open: current main still creates WebUI dashboard sessions without a friendly label and the CLI session projection still cannot display stored labels, so the reported usability gap remains. This is a narrow UX enhancement built on an existing session label contract, not a broken documented behavior.

## What This Changes

Request that WebUI `/new` and New Chat dashboard sessions receive generated friendly labels and that session lists surface those labels instead of only UUID-backed internal keys.

## Best Possible Solution

Use the existing session `label` field to assign a stable default label for generated dashboard sessions and surface stored labels in CLI JSON/text output without changing session keys.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: typed `/new` reaches the New Chat creation flow, that flow sends no label, Gateway only stores explicit labels, and the CLI projection omits labels.

## Solution Assessment

Yes. Reusing the existing `sessions.create`/`sessions.patch` label plumbing is the narrow maintainable direction; first-message title generation can remain a later enhancement.
