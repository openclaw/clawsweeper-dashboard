## Summary

Keep open: current main still has the unlabeled WebUI dashboard-session path, and #77037 is an open unmerged PR with closing syntax for this issue, so the issue should remain open until that PR is reviewed, merged, or closed.

## What This Changes

The issue requests automatic friendly labels for dashboard-scoped WebUI `/new` sessions and session-list display support for those labels.

## Best Possible Solution

Review and land or revise #77037 so generated dashboard sessions receive a friendly label and session-list output surfaces stored labels without changing session keys.

## Reproduction Assessment

Yes. Source inspection on current main shows WebUI `/new` sends no label, gateway persists only `p.label`, and the CLI projection/table omit labels, so the reported no-label readability gap is source-reproducible without running the UI.

## Solution Assessment

Yes, using the existing `sessions.create` label contract is the narrow maintainable direction. PR review should decide the default label text and whether CLI label display is part of the same change.
