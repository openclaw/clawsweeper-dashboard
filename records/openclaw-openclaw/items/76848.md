## Summary

Keep open: current main still lacks automatic labels for Control UI `/new` dashboard sessions and the CLI projection cannot display stored labels. The linked closing PR is still open, so the issue should stay open until that PR is reviewed, expanded if needed, and merged or closed.

## What This Changes

Request automatic friendly labels for WebUI `/new` dashboard sessions, with CLI/UI session lists able to show those labels instead of only internal UUID keys.

## Best Possible Solution

Review the linked closing PR against the full UX goal, then land a change that uses the existing session `label` contract for generated dashboard sessions and surfaces labels in CLI/UI lists as maintainers decide.

## Reproduction Assessment

Yes. Source inspection on current main shows `/new` sends no label, Gateway only stores an explicit `p.label`, and the CLI session-row projection omits labels, so the reported unlabeled dashboard-session path is reproducible from code.

## Solution Assessment

Yes on direction, with scope still to confirm. Reusing the existing session `label` plumbing is the narrow maintainable fix, but maintainers should decide whether the linked PR also needs CLI JSON/text display changes before it closes the user problem.
