## Summary

Keep open: current main still creates unlabeled dashboard sessions from the Control UI path and the CLI session table cannot surface stored labels, so the reported usability gap remains. This is a narrow UX enhancement rather than a strict regression.

## What This Changes

Request automatic friendly labels for WebUI `/new` dashboard sessions and label-aware `openclaw sessions` output.

## Best Possible Solution

Use the existing session `label` contract to assign a collision-safe default label for generated dashboard sessions and surface stored labels in CLI JSON/text output without changing session keys.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Control UI `/new` sends no label, Gateway only stores explicit labels, and CLI row projection omits labels.

## Solution Assessment

Yes. Reusing the existing `sessions.create`/`sessions.patch` label plumbing is the narrow maintainable direction; derived titles can be a later enhancement.
