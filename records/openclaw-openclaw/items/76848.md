## Summary

Keep open: current main still creates unlabeled dashboard sessions from the Control UI `/new` path, and an open linked PR targets the issue but has not merged. The remaining work is a narrow UX enhancement with a pending implementation candidate, not a cleanup close.

## What This Changes

Request that dashboard-scoped sessions created by WebUI `/new` receive friendly labels automatically and become easier to read in `openclaw sessions` output.

## Best Possible Solution

Land a maintainer-approved fix that uses the existing session `label` contract for generated dashboard sessions and surfaces stored labels in session lists where maintainers want them displayed.
