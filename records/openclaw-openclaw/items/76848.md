## Summary

Keep open: current main still creates unlabeled dashboard sessions from the Control UI `/new` path and the CLI session listing cannot surface stored labels, so the requested usability improvement remains valid.

## What This Changes

Request that WebUI `/new`/New Chat dashboard sessions get automatic friendly labels and that `openclaw sessions` can expose those labels.

## Best Possible Solution

Use the existing session `label` contract to assign a stable default label for generated dashboard sessions and include stored labels in CLI JSON/text output, leaving derived first-message titles for a later enhancement.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: Control UI `/new` sends no label, Gateway only stores explicit `p.label`, and the CLI row projection omits labels.

## Solution Assessment

Yes. Reusing the existing `sessions.create`/`sessions.patch` label plumbing is the narrow maintainable path; changing session keys or adding a config option would be unnecessary.
