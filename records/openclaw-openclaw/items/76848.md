## Summary

Keep open: current main still sends WebUI `/new` dashboard session creation through `sessions.create` without a label, and the linked open PR is the right place to resolve the implementation.

## What This Changes

Request automatic friendly labels for WebUI `/new` dashboard sessions and session-list output that can show or prefer those labels.

## Best Possible Solution

Review and land or revise the linked PR, using a collision-safe default label and deciding whether CLI label rendering belongs in the same fix or a narrow follow-up.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: WebUI `/new` creates a dashboard session without a label, Gateway persists only explicit labels, and CLI session display rows omit labels.

## Solution Assessment

Unclear for the open PR until reviewed. Reusing the existing session `label` contract is the right boundary, but the final fix should avoid duplicate-label rejection and settle the CLI display scope.
