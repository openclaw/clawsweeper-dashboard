## Summary

Keep open. Current main and v2026.5.22 still lack a documented ACPX per-agent default model/thinking config surface; only explicit spawn or slash-command runtime options are wired today, so this remains a real feature/config contract decision rather than cleanup.

## What This Changes

Request to add ACPX per-agent default model and reasoning configuration so spawned ACP sessions can start with harness-specific defaults without manual `/acp` commands.

## Best Possible Solution

Choose one documented ACP defaults contract, then feed it through the existing `AcpSessionRuntimeOptions` and advertised-key mapping before the first ACP turn, with explicit spawn and slash-command overrides taking precedence.

## Reproduction Assessment

Yes, as a source-level feature gap: the strict ACPX schema, manifest-backed plugin config, resolved config, and spawn path do not expose or consume per-agent model/thinking defaults. No live run is needed to prove the missing config surface.

## Solution Assessment

Unclear until maintainers choose the public config contract. The narrow maintainable implementation should reuse canonical ACP runtime options and advertised-key mapping rather than adding ACPX-only backend key handling.
