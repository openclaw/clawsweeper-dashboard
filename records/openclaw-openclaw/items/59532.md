## Summary

Keep open. Current main has explicit ACP model/thinking controls and backend key mapping, but it still has no config surface that supplies per-harness ACPX default model or reasoning/thinking when spawn-time inputs are omitted.

## What This Changes

Add ACPX/ACP configuration that supplies per-agent default model and reasoning/thinking options for spawned ACP sessions.

## Best Possible Solution

Choose one public ACP default surface, let explicit `/acp spawn` and `sessions_spawn` inputs override it, then apply resolved defaults through the existing ACP runtime-options/control path with docs and focused tests.

## Reproduction Assessment

Yes, source-level. The strict ACPX schema rejects the proposed default keys, and current spawn/runtime code only carries model or thinking when they are explicitly supplied.

## Solution Assessment

Unclear until maintainers choose the config contract. The maintainable implementation should be narrow and reuse existing runtime-option plumbing, but the proposed ACPX `agentDefaults` shape overlaps with existing ACP runtime defaults.
