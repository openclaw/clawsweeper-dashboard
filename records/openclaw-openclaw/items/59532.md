## Summary

Keep open. Current main still lacks a public ACPX per-agent default model/thinking configuration surface for omitted spawn-time inputs, so the requested capability remains real and is not superseded by the related runtime-option mapping work.

## What This Changes

The issue requests ACPX plugin configuration for per-agent default model and reasoning/thinking options applied when ACP sessions are spawned without explicit runtime inputs.

## Best Possible Solution

Choose one public ACP default surface, let explicit spawn/runtime inputs override it, and apply resolved defaults through the existing ACP runtime-option mapping path with schema, docs, and focused tests.

## Reproduction Assessment

Yes, source-level. The strict ACPX schema has no proposed default keys, resolved ACPX config only carries command overrides, and the spawn path only forwards model/thinking when the caller supplies them.

## Solution Assessment

Unclear until maintainers choose the public config contract. The narrow implementation should reuse the existing ACP runtime-option mapping path rather than adding ACPX-specific backend key handling.
