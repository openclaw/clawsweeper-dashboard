## Summary

Keep open. Current main has explicit ACP spawn model/thinking controls and ACP runtime control plumbing, but it still has no config surface that supplies per-harness ACPX default model or reasoning effort when those spawn-time inputs are omitted.

## What This Changes

Request a per-agent ACPX plugin config default for ACP model and reasoning/config options used when new ACP sessions are spawned.

## Best Possible Solution

Add a bounded ACP default surface that resolves by harness, lets explicit `/acp spawn` or `sessions_spawn` runtime options win, and applies the result through the existing ACP runtime-options/control path with docs and tests.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: adding the proposed config keys would be rejected or dropped by the strict ACPX schema/resolved config, and spawning without explicit model/thinking currently produces no runtime options.

## Solution Assessment

Yes. The narrow maintainable direction is to extend ACPX/default-runtime configuration and reuse existing `session/set_config_option` plumbing; a broad provider model-selection rewrite is not needed.
