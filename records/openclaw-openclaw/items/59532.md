## Summary

Keep open. Current main has explicit ACP spawn model/thinking overrides and runtime control plumbing, but it still has no ACPX plugin config surface for per-harness default model or reasoning effort, so the requested config-level default remains valid.

## What This Changes

Request ACPX plugin configuration that can set default model and reasoning/config options per ACP harness agent.

## Best Possible Solution

Add a bounded ACPX-owned default surface that resolves per harness agent, lets explicit `/acp spawn` or `sessions_spawn` model/thinking inputs win, and feeds the existing ACP runtime-options or `setConfigOption` path without changing the acpx dependency or core provider model selection.

## Reproduction Assessment

Yes. A source-level reproduction is enough: the proposed `agentDefaults` key is absent from the strict ACPX schema, and an ACP spawn without explicit model/thinking produces no runtime options to apply.

## Solution Assessment

Yes, the narrow maintainable fix is to add validated ACPX defaults and tests around config parsing plus runtime initialization/control application. A broad core migration is unnecessary unless maintainers want the public surface under `agents.list[].runtime.acp` instead.
