## Summary

Keep open. Current main still lacks a public ACPX per-agent default model/thinking configuration surface, while the existing runtime-option mapping only helps once explicit runtime options are already supplied.

## What This Changes

Request to add ACPX plugin configuration for per-agent default model and reasoning/thinking options when spawning ACP sessions.

## Best Possible Solution

Define one ACPX default contract, apply it before the first turn through the existing canonical ACP runtime-options path, and document explicit spawn/runtime inputs as higher precedence.

## Reproduction Assessment

Yes, source-level: the strict ACPX schema and manifest lack default model/thinking keys, resolved config only carries command overrides, and the spawn path only forwards explicit runtime inputs. No live ACP session is needed to prove the missing configuration surface.

## Solution Assessment

Unclear until maintainers choose the public config contract. The maintainable implementation should reuse `AcpSessionRuntimeOptions` and advertised-key mapping rather than adding ACPX-only backend key handling.
