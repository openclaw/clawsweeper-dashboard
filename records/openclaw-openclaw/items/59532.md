## Summary

Keep open. Current main still lacks an ACPX per-agent default model/thinking configuration surface; the existing model/thinking paths only apply when explicit runtime options are already supplied, and the public config precedence needs maintainer approval.

## What This Changes

Requests ACPX plugin config support for per-agent default model and reasoning-effort values when spawning ACP sessions.

## Best Possible Solution

Define one documented ACPX default contract that feeds the existing ACP runtime-options path before the first turn, with explicit spawn and slash-command runtime inputs taking precedence.

## Reproduction Assessment

Yes, source-level: the strict ACPX schema, manifest, resolved config, and agent ACP runtime defaults lack model/thinking default fields, while the spawn path only forwards explicit runtime inputs.

## Solution Assessment

Unclear until maintainers choose the public config contract. The maintainable path should reuse AcpSessionRuntimeOptions and advertised-key mapping rather than adding ACPX-only backend key handling.
