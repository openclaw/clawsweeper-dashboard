## Summary

Keep open. Current main has explicit ACP model/thinking runtime controls, but still lacks a config surface that supplies per-agent ACPX defaults when `/acp spawn` or `sessions_spawn` omit those fields.

## What This Changes

Add per-agent ACPX/ACP defaults for model and reasoning effort so new ACP sessions can start with harness-specific runtime options.

## Best Possible Solution

Expose model/thinking defaults on the chosen ACP agent default contract, merge them before ACP session initialization, and feed the result through the existing runtime-options/control pipeline with explicit spawn and slash-command values taking precedence.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: adding the proposed default keys has no accepted schema or resolved-config slot, and spawning without explicit `model` or `thinking` produces no runtime options.

## Solution Assessment

Unclear. Reusing the existing ACP runtime-options and `session/set_config_option` path is the narrow implementation direction, but maintainers should first choose the public config location to avoid overlapping ACPX-plugin and `agents.list[].runtime.acp` defaults.
