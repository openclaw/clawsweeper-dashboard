## Summary

Keep open: current main and v2026.6.6 still only migrate per-agent legacy auth files, while runtime auth uses the SQLite store; the open closing PR is the active repair path and this issue should close only after that PR lands.

## What This Changes

Repair the v2026.6.6 auth SQLite migration so default-agent credentials stored only in global `auth.profiles` are imported into the main agent SQLite auth store.

## Best Possible Solution

Land or explicitly reject https://github.com/openclaw/openclaw/pull/93156 after maintainer review; the durable fix should stay in doctor migration code rather than adding steady-state runtime fallback.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.6 skip config-only default-agent auth during doctor migration, then runtime loads an empty SQLite-backed auth store. I did not run the full macOS upgrade path locally.

## Solution Assessment

Yes, the best direction is a doctor-owned migration that imports shipped legacy default-agent config credentials into SQLite. A runtime fallback from config would be a broader compatibility path and conflicts with the repository's canonical SQLite runtime-state direction.
