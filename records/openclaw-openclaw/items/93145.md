## Summary

Keep open: current main still skips the reported config-only default-agent auth migration, and the linked open fix PR https://github.com/openclaw/openclaw/pull/93156 is the canonical path to resolve it.

## What This Changes

The issue reports that upgrading to v2026.6.6 can leave the default agent's SQLite auth store empty when legacy credentials existed only in global `auth.profiles`.

## Best Possible Solution

Land a doctor-owned migration that imports shipped legacy default-agent config credentials into the main agent SQLite store, strips config secrets after verified import, and keeps runtime on canonical SQLite.

## Reproduction Assessment

Yes, there is a high-confidence source reproduction path: current main skips doctor migration when no per-agent legacy auth files exist, while runtime auth reads the SQLite store. I did not execute a live upgrade in this read-only review.

## Solution Assessment

Yes, the best fix direction is the linked doctor migration, not a runtime fallback. That keeps config `auth.profiles` metadata-only after verified import and preserves SQLite as the canonical runtime store.
