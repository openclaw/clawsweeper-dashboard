The issue reports an upgrade regression where the main agent's SQLite auth store is created empty when no per-agent legacy auth JSON exists, leaving configured provider auth unavailable.

## Best Possible Solution

Add a doctor-owned migration/repair that imports or reconstructs shipped legacy default-agent auth config into the main agent SQLite store without adding runtime JSON/config fallback.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection shows the reported path is plausible because doctor migration is file-detection driven and runtime auth no longer reads legacy/global credential sources directly.

## Solution Assessment

Yes, the best direction is a narrow doctor migration or repair. Runtime should continue reading the canonical SQLite auth store after doctor has migrated shipped legacy state.
