The issue reports a v2026.6.6 upgrade regression where the default agent loses model-provider auth when legacy credentials existed only in global `auth.profiles`.

## Best Possible Solution

Review and land or deliberately reject https://github.com/openclaw/openclaw/pull/93156, keeping the repair in doctor migration code and leaving runtime auth on the canonical SQLite store.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: current main skips migration when no per-agent legacy auth files exist, while runtime reads the per-agent SQLite auth store. I did not perform a live release-to-release upgrade run in this review.

## Solution Assessment

Yes, the best fix direction is the linked doctor-owned migration that imports shipped legacy config credentials into SQLite. A runtime fallback from global config would conflict with the repository's canonical SQLite auth-store direction.
