## Summary

Keep open: current main still has the reported doctor/runtime auth-store gap, and https://github.com/openclaw/openclaw/pull/93156 is the open, mergeable closing PR for this issue.

## What This Changes

Default-agent upgrades can lose model auth when legacy credentials exist only in global `auth.profiles` and no per-agent auth JSON is available to migrate into SQLite.

## Best Possible Solution

Review and land or explicitly reject the doctor-owned migration fix that imports default-agent config credentials into SQLite while keeping runtime on the canonical auth store.

## Reproduction Assessment

Yes from source inspection: a default-agent install with only `openclaw.json auth.profiles` and no per-agent auth JSON makes the migration return before import, and runtime profile eligibility then sees no stored credential. I did not run a live v2026.6.1-to-v2026.6.6 upgrade, so this is source-reproducible rather than live reproduced.

## Solution Assessment

Yes, the linked doctor-migration approach is the best direction because repository policy keeps runtime on canonical SQLite auth state and repairs shipped legacy auth/config shapes in `openclaw doctor --fix`.
