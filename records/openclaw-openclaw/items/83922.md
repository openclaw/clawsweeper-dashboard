## What This Changes

The issue asks for the `codex-cli` metadata path to avoid calling `process.exit(0)` immediately after writing JSON to stdout.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84239, or an equivalent focused fix, then close this issue with the merge commit and pipe-parse verification.

## Reproduction Assessment

Yes. Current main has the exact `process.stdout.write(...)` followed by `process.exit(0)` pattern on the `codex-cli` path, and Node documents that this can truncate stdout before pending writes complete.

## Solution Assessment

Yes. The maintainable fix is to avoid synchronous `process.exit(0)` after the write and let the process exit naturally, while ensuring the branch does not fall through and emit a second JSON payload.
