## What This Changes

The issue asks for the codex-cli early-exit branch of the live CLI backend metadata script to avoid calling process.exit immediately after writing JSON to stdout.

## Best Possible Solution

Land or close the linked PR after review; the durable fix is a drain-safe codex-cli branch with focused regression proof that the piped JSON output parses once and completely.

## Reproduction Assessment

Yes, source-reproducible: current main and v2026.5.19 both write JSON to stdout and immediately call process.exit(0) on the codex-cli path, matching Node's documented stdout truncation hazard. I did not run a live pipe reproduction because this review was constrained to read-only inspection.

## Solution Assessment

Yes, a drain-safe early return is the right fix shape, and the linked PR follows it by using process.exitCode plus an else guard. The issue should remain open until that PR is merged or replaced.
