## Summary

Current main and the latest stable tag still contain the reported early stdout-write plus synchronous-exit path, so this should stay open as a narrow CLI/live-test script bug.

## What This Changes

The issue asks to prevent JSON truncation from `scripts/print-cli-backend-live-metadata.ts` when the unsupported `codex-cli` path writes to piped stdout and exits immediately.

## Best Possible Solution

Refactor the unsupported-provider branch so it writes the JSON exactly once, returns from the script path, and lets stdout drain before the process exits.

## Reproduction Assessment

Yes from source inspection: current main and `v2026.5.18` still write JSON to stdout and immediately call `process.exit(0)` on the unsupported `codex-cli` path. I did not establish a live failing pipe run, so this is source-reproducible rather than directly reproduced.

## Solution Assessment

Yes, a drain-safe early return is the right fix, but a bare replacement with `process.exitCode = 0` at the current top level would fall through and print a second payload unless the rest of the script is skipped.
