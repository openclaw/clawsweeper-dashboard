## What This Changes

Report that the codex-cli unsupported-provider metadata branch can truncate JSON output on piped stdout because it calls `process.exit(0)` immediately after `process.stdout.write`.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/84239; close this issue only after a fix is merged with commit proof.

## Reproduction Assessment

Yes from source inspection: current main and `v2026.5.18` still write JSON to stdout and immediately call `process.exit(0)` on the unsupported `codex-cli` path. I did not run a live slow-pipe reproduction, so this is source-reproducible rather than directly reproduced.

## Solution Assessment

Yes: a drain-safe early branch that emits exactly one JSON document and lets the process exit naturally is the narrow maintainable fix. The linked open PR appears to follow that direction, but the issue should stay open until that PR merges or maintainers choose another fix.
