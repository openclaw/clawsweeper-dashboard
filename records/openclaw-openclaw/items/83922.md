## Summary

Keep open: current main still has the reported stdout-write followed by synchronous `process.exit(0)` path, and the linked fixing PR at https://github.com/openclaw/openclaw/pull/84239 is still open rather than merged.

## What This Changes

The issue reports that `scripts/print-cli-backend-live-metadata.ts` may truncate piped JSON output because the `codex-cli` branch writes to stdout and immediately calls `process.exit(0)`.

## Best Possible Solution

Review and land or close the linked fix PR at https://github.com/openclaw/openclaw/pull/84239; the final fix should let stdout drain, emit one JSON document, and add focused regression coverage if maintainers want CI proof.

## Reproduction Assessment

Yes from source inspection: current main writes JSON to stdout and immediately calls `process.exit(0)` in the `codex-cli` branch. I did not establish a live failing pipe run because the truncation is timing-dependent.

## Solution Assessment

Yes: replacing the synchronous exit with natural exit via `process.exitCode` is the right direction, but the control flow must also skip the normal metadata body so stdout receives only one JSON document.
