## Summary

Keep open. Current main still sends the best-effort multiple-gateway advisory through the CLI error channel even though the docs describe it as warning-style guidance and allow intentional isolated multi-gateway setups.

## What This Changes

The issue requests downgrading the multiple-gateway detection advisory from error-level output to warn or info output.

## Best Possible Solution

Downgrade only the extraServices advisory output in printDaemonStatus to warning/info-style human output while preserving error output for real service, port, RPC, probe, and config failures.

## Reproduction Assessment

Yes. A deterministic source-level reproduction is to call printDaemonStatus with json: false and a non-empty extraServices array; current main routes that advisory through defaultRuntime.error, which maps to console.error.

## Solution Assessment

Yes. The narrowest maintainable fix is to change only the advisory output branch, likely to defaultRuntime.log(warnText(...)), and add a focused regression test for the output channel; detected-service-specific cleanup hints remain a separate issue at https://github.com/openclaw/openclaw/issues/52510.
