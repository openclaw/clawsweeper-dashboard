## Summary

Keep open. Current main and the latest release still route the best-effort extra-gateway advisory through the error channel, and an open linked PR now specifically fixes this issue, so the issue should remain open until that PR lands.

## What This Changes

Change the multiple-gateway best-effort advisory from error-level output to warning/info-style human output.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82922 if it remains correct, then close this issue as implemented after merge.

## Reproduction Assessment

Yes, source-level. Calling `printDaemonStatus` with `json: false` and a non-empty `extraServices` array deterministically executes the `defaultRuntime.error` branch, which delegates to `console.error`.

## Solution Assessment

Yes. The narrow maintainable solution is to move only the extra-gateway advisory to warning/info-style human output and keep real service, RPC, port, and config failures on the error channel.
