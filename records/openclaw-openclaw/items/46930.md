## Summary

Keep open. Current main and the latest release still route the best-effort multiple-gateway advisory through the CLI error channel even though the docs present it as warning/advisory guidance and support intentional isolated multi-gateway setups.

## What This Changes

Request downgrade the multiple-gateway status advisory from error-level output to warn/info-style human output.

## Best Possible Solution

Change only the extraServices advisory branch in printDaemonStatus to warning/info-style human output, preserve error output for real service/RPC/port/config failures, and add a focused regression test for non-empty extraServices.

## Reproduction Assessment

Yes, source-level. Calling printDaemonStatus with json: false and a non-empty extraServices array deterministically routes the advisory through defaultRuntime.error, which maps to console.error.

## Solution Assessment

Yes. The narrowest maintainable fix is to change the extraServices advisory output channel, likely to defaultRuntime.log(warnText(...)), and leave detected-service cleanup-hint correctness to https://github.com/openclaw/openclaw/issues/52510.
