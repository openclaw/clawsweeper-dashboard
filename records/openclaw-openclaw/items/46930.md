## Summary

Keep open. Current main still sends the best-effort multiple-gateway advisory through the CLI error channel, while the docs allow intentional isolated multi-gateway setups and describe this path as warning-style diagnostic output.

## What This Changes

The issue requests downgrading the multiple-gateway detection advisory from error-level output to warn or info-level output.

## Best Possible Solution

Downgrade only the `extraServices` advisory output in `printDaemonStatus` to warning/info-style human output while preserving error output for real service, port, RPC, probe, and config failures.

## Reproduction Assessment

Yes. A deterministic source-level reproduction is to call `printDaemonStatus` with `json: false` and a non-empty `extraServices` array; current main routes that advisory through `defaultRuntime.error`.

## Solution Assessment

Yes. The narrowest maintainable fix is to change only the advisory output path, likely to `defaultRuntime.log(warnText(...))`, and add a focused regression test for the output channel.
