## Summary

Keep open. Current main and the latest release still send the best-effort multiple-gateway advisory through `defaultRuntime.error`, which maps to `console.error`, while the docs frame this path as warning/advisory guidance and explicitly support intentional isolated multi-gateway setups.

## What This Changes

The issue asks to downgrade the multiple-gateway best-effort advisory from error-level output to warning/info-style output.

## Best Possible Solution

Change only the `extraServices` advisory branch to non-error human output, keep real service/RPC/port/config failures on the error channel, and add focused regression coverage.

## Reproduction Assessment

Yes, source-level. Calling `printDaemonStatus` with `json: false` and a non-empty `extraServices` array deterministically executes `defaultRuntime.error`, which delegates to `console.error`.

## Solution Assessment

Yes.
