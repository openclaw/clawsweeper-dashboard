## Summary

Keep open. Current main and the latest release still send the best-effort multiple-gateway advisory through the CLI error channel, even though the surrounding docs describe the path as warning/advisory guidance and support intentional isolated multi-gateway setups.

## What This Changes

The issue asks to change the multiple-gateway advisory emitted by `gateway status --deep` from error-level output to warn/info-style output.

## Best Possible Solution

Change only the `extraServices` advisory branch to warning/info-style human output, keep real service/RPC/port/config failures on the error channel, and add a focused non-empty `extraServices` regression test.

## Reproduction Assessment

Yes, source-level. Calling `printDaemonStatus` with `json: false` and a non-empty `extraServices` array deterministically executes `defaultRuntime.error`, which maps to `console.error`.

## Solution Assessment

Yes. The narrowest maintainable fix is to move this advisory to warning/info-style human output and leave cleanup-target correctness to https://github.com/openclaw/openclaw/issues/52510 and https://github.com/openclaw/openclaw/pull/52515.
