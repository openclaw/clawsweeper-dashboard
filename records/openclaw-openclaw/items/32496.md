Keep open. Current main still lacks first-class `frequency_penalty` and `presence_penalty` forwarding for OpenAI-compatible model params; `params.extra_body` is only an escape hatch, and the linked implementation PRs were closed unmerged.

## Best Possible Solution

Add finite-number passthrough for `frequency_penalty` and `presence_penalty` on OpenAI-compatible completions payloads while preserving `extra_body` for vendor-specific fields.

Yes, source-level. Configure `frequency_penalty` or `presence_penalty` under accepted model params; current main preserves those keys in config/merge paths but never maps them into stream options or the OpenAI-compatible payload unless they are nested under `params.extra_body`.

## Solution Assessment

Yes for the narrow request. A finite numeric mapping for the two OpenAI-compatible penalty fields is more maintainable than broad unvalidated passthrough of every generation knob.
