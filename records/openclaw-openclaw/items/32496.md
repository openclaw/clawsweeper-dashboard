## Summary

Keep open. Current main still accepts arbitrary model params in config, but the OpenAI/OpenAI-compatible transport paths do not normalize or forward direct frequency/presence penalty params. The documented `extra_body` workaround only covers `openai-completions` proxy payloads, and the linked fix PRs are closed unmerged while the broader generation-parameter issues do not replace this narrow request.

## What This Changes

Request support for `frequency_penalty` and `presence_penalty` model params for OpenAI-compatible providers so configured penalties reach provider request payloads.

## Best Possible Solution

Add a narrow transport-level implementation that accepts finite numeric snake_case and camelCase penalty params from the existing merged params sources, forwards them only through compatible OpenAI-family payload builders, covers HTTP Responses, HTTP Completions, and WebSocket request creation, and documents the supported scope separately from broader per-agent generation-parameter work.
