The issue requests support for OpenAI-compatible `frequency_penalty` and `presence_penalty` parameters in configured model params and gateway/provider request payloads.

## Best Possible Solution

Review and land or revise https://github.com/openclaw/openclaw/pull/84094 so standard OpenAI-compatible penalty fields flow through the existing stream-param pipeline, while leaving broader provider-specific knobs to separate scoped work.

## Reproduction Assessment

Yes, source-level. ## Solution Assessment

Yes for the narrow request. A finite mapping for standard OpenAI-compatible penalty fields through the existing stream-param and transport pipeline is more maintainable than broad passthrough of every generation knob.
