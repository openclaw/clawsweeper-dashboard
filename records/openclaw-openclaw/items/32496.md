## Summary

Keep open. Current main still has no direct `frequency_penalty` or `presence_penalty` path for OpenAI-compatible model params; users can only use `params.extra_body` as an escape hatch, and prior implementation PRs were closed unmerged.

## What This Changes

The issue requests direct passthrough of OpenAI-compatible frequency and presence penalty model parameters from OpenClaw config to provider requests.

## Best Possible Solution

Add finite-number passthrough for `frequency_penalty` and `presence_penalty` on OpenAI-compatible completions payloads while preserving `extra_body` as the vendor-specific escape hatch.

## Reproduction Assessment

Yes, source-level. Configure `frequency_penalty` or `presence_penalty` under accepted model params; current main preserves those keys in config but never maps them into stream options or the OpenAI-compatible payload unless they are nested under `params.extra_body`.

## Solution Assessment

Yes for the narrow request. A finite numeric mapping for the two OpenAI-compatible penalty fields is more maintainable than a broad unvalidated generation-parameter passthrough.
