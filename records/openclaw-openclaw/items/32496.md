## Summary

Keep open. Current main still preserves arbitrary model `params` but does not forward top-level `frequency_penalty` or `presence_penalty` through the stream-param and OpenAI-compatible payload path; the current implementation candidate is the open PR at https://github.com/openclaw/openclaw/pull/84094, so the issue should stay open until that PR lands or maintainers choose a different policy.

## What This Changes

The issue requests first-class forwarding for OpenAI-compatible `frequency_penalty` and `presence_penalty` model parameters configured in OpenClaw model params.

## Best Possible Solution

Review and land the narrow OpenAI-compatible forwarding path from https://github.com/openclaw/openclaw/pull/84094, preserving `extra_body` for vendor-specific knobs and avoiding broader provider-specific sampling fields without explicit gating.

## Reproduction Assessment

Yes, source-level. Configure `frequency_penalty` or `presence_penalty` under accepted model params; current main preserves those keys but never maps them into stream options or the OpenAI-compatible payload unless nested under `params.extra_body`.

## Solution Assessment

Yes for the narrow request. A finite mapping for `frequency_penalty` and `presence_penalty` through the existing stream-param pipeline is more maintainable than broad passthrough of every generation knob.
