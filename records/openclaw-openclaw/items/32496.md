## Summary

Keep open. Current main still does not first-class forward `frequency_penalty` or `presence_penalty`, and the remaining narrow implementation is already represented by the open linked PR at https://github.com/openclaw/openclaw/pull/84094.

## What This Changes

Request first-class support for OpenAI-compatible `frequency_penalty` and `presence_penalty` model parameters so configured values reach upstream provider requests.

## Best Possible Solution

Land or revise https://github.com/openclaw/openclaw/pull/84094 so standard OpenAI-compatible penalty fields flow through the existing stream-param and transport pipeline, while leaving broader provider-specific knobs to separate scoped work.

## Reproduction Assessment

Yes, source-level. Configure `frequency_penalty` or `presence_penalty` under accepted model params or send them through the OpenAI-compatible gateway; current main preserves adjacent params but does not map these fields into stream options or OpenAI-compatible payloads outside the `extra_body` escape hatch.

## Solution Assessment

Yes for the narrow request. A finite mapping for standard OpenAI-compatible penalty fields through the existing stream-param and transport pipeline is more maintainable than broad passthrough of every generation knob.
