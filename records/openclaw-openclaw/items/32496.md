## What This Changes

Add first-class passthrough for `frequency_penalty` and `presence_penalty` on OpenAI-compatible provider/model requests.

## Best Possible Solution

Review and land or revise https://github.com/openclaw/openclaw/pull/84094 so the standard OpenAI-compatible penalty fields flow through the existing stream-param pipeline, while keeping broader provider-specific knobs separately scoped.

## Reproduction Assessment

Yes, source-level. Configure `frequency_penalty` or `presence_penalty` under accepted model params or send them through the OpenAI-compatible gateway; current main accepts adjacent params but does not map these fields into stream options or the OpenAI-compatible payload outside the `extra_body` escape hatch.

## Solution Assessment

Yes for the narrow request. A finite-number mapping for standard OpenAI-compatible penalty fields through the existing stream-param and transport pipeline is more maintainable than broad passthrough of every generation knob.
