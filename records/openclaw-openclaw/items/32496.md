## Summary

Keep open. Current main still accepts arbitrary model `params`, but the embedded stream wrapper and OpenAI-compatible request builder do not map `frequency_penalty` or `presence_penalty` unless the user puts them under the documented `extra_body` escape hatch.

## What This Changes

The issue requests first-class forwarding of `frequency_penalty` and `presence_penalty` from model params to OpenAI-compatible provider requests.

## Best Possible Solution

Add finite-number passthrough for `frequency_penalty` and `presence_penalty` on OpenAI-compatible completions payloads while preserving `extra_body` for vendor-specific fields.

## Reproduction Assessment

Yes, source-level. Configure `frequency_penalty` or `presence_penalty` under accepted model params; current main preserves those keys in config but never maps them into stream options or the OpenAI-compatible payload unless nested under `params.extra_body`.

## Solution Assessment

Yes for the narrow request. A finite mapping for the two OpenAI-compatible penalty fields is more maintainable than broad passthrough of every generation knob.
