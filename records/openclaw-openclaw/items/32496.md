## What This Changes

The issue requests support for OpenAI-compatible frequency and presence penalty parameters in model params and gateway/provider request forwarding.

## Best Possible Solution

Review, rebase, and land https://github.com/openclaw/openclaw/pull/84094 or an equivalent narrow patch for standard OpenAI-compatible penalty fields, while keeping broader provider-specific sampling knobs separate.

## Reproduction Assessment

Yes, source-level. Configure `frequency_penalty` or `presence_penalty` under model params or send them through the OpenAI-compatible gateway; current main preserves adjacent params but does not map these fields into stream options or provider payloads outside the `extra_body` workaround.

## Solution Assessment

Yes for the narrow request. A finite mapping for standard OpenAI-compatible penalty fields through the existing stream-param and transport pipeline is more maintainable than broad passthrough of every generation knob, with the linked PR as the current implementation candidate.
