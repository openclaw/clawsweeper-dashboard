## Summary

Keep open: current main does not implement this feature, but this PR is not merge-ready because it changes scheduled reset semantics and still has source-proven delivery, session-key, and prompt-boundary blockers plus no real reset proof.

## What This Changes

The branch stores the last assistant transcript text during scheduled daily/idle rollovers and injects it as a one-shot hint into the next reply.

## Best Possible Solution

Revise the feature so scheduled-rollover context is explicitly approved, safely delimited, delivered through both immediate and queued prompt paths, reserved in the SessionEntry key contract, and proven in a real daily/idle reset setup.

## Reproduction Assessment

Not applicable as a current-main bug reproduction because daily/idle rollover is documented current behavior. The PR blockers are source-reproducible from the branch queue path, SessionEntry slot-key contract, and raw prompt injection.

## Solution Assessment

No, not as written. The safer path is to carry the hint through the shared prompt envelope used by immediate and queued runs, reserve the field, delimit transcript-derived text, and clear it only after actual delivery.
