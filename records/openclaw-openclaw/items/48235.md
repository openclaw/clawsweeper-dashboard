## Summary

Keep open: current main does not implement the scheduled-reset continuity hint, but this PR is not merge-ready because it changes reset semantics, is conflicting, lacks real behavior proof, and has source-visible delivery/session-contract/security blockers.

## What This Changes

The PR adds a SessionEntry field, a transcript tail reader for the last assistant message, scheduled-reset capture, and one-shot prompt hint injection on the next reply.

## Best Possible Solution

Revise the feature as an explicitly approved scheduled-rollover continuity design that reserves the session field, safely delimits prior transcript text, delivers through immediate and queued prompt paths, and is proven in a real reset setup.

## Reproduction Assessment

Not applicable as a current-main bug reproduction because this is a new continuity feature. The PR blockers are source-reproducible from the prompt queue path, SessionEntry slot-key contract, and raw prompt interpolation.

## Solution Assessment

No, not as written. The safer path is to route the hint through the shared prompt envelope, reserve the field, delimit transcript-derived text as untrusted context, clear only after actual delivery, and get product/security approval for the reset semantic change.
