## Summary

Keep open: current main still does not implement scheduled-reset continuity, but this PR should not merge as-is because the hint can be dropped in queued prompt paths, transcript-derived text is injected into prompt instructions without safe delimiting, and after-fix real behavior proof is missing.

## What This Changes

The branch adds a session-store field, transcript-tail helper, scheduled reset capture, and one-shot prompt hint to carry the last assistant message after daily/idle rollovers.

## Best Possible Solution

Revise or replace this PR with a focused implementation that carries bounded scheduled-rollover context through immediate and queued prompt paths, treats transcript text as untrusted quoted context, clears only after actual delivery, and preserves explicit resets as blank starts.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this PR introduces new continuity behavior. Source inspection does show the current daily/idle rollover path creates a new session without carrying prior assistant output.

## Solution Assessment

No, not as written. The feature direction is plausible, but the implementation needs queued prompt delivery, safe delimiting of transcript-derived text, robust one-shot clearing, and real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep reset hints in queued prompt paths:** `src/auto-reply/reply/body.ts:48`
  - body: `applySessionHints` only changes `prefixedBodyBase`, but current queued runs build `queuedBody` from `effectiveBaseBody`. Because this block also clears the one-shot field, followup/collect/steer-backlog runs can drop the scheduled-reset hint before the model sees it.
  - confidence: 0.88
- **[P2] Delimit prior assistant text before injecting it:** `src/auto-reply/reply/body.ts:47`
  - body: `resetHintMsg` comes from a transcript and can include multiline or instruction-shaped text reflected from earlier turns. Embedding it directly inside the wrapper makes prior output indistinguishable from the surrounding hint instructions; normalize it or place it behind explicit quoted-context delimiters.
  - confidence: 0.84
- **[P3] Clear the in-memory hint without store context:** `src/auto-reply/reply/body.ts:49-50`
  - body: The hint fires whenever `params.sessionEntry` carries `lastAssistantMessageBeforeReset`, but clearing is gated on `sessionStore` and `sessionKey`. A caller that supplies only the entry can inject the same one-shot hint repeatedly.
  - confidence: 0.72
