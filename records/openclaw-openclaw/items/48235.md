## Summary

Keep this PR open, but it is not merge-ready: the feature is still useful and canonical for scheduled-reset continuity, yet the current patch drops hints on queued prompt paths, embeds prior transcript text without safe delimiting, and lacks after-fix real behavior proof.

## What This Changes

The branch adds a session entry field, transcript-tail helper, scheduled reset capture, and one-shot prompt hint to carry the last assistant message into the first turn after daily/idle rollover.

## Best Possible Solution

Revise this feature to carry bounded, clearly delimited scheduled-rollover context through both immediate and queued prompt paths, clear it only after actual delivery, and preserve explicit `/new` and `/reset` as blank starts.

## Reproduction Assessment

Not applicable as a strict bug reproduction: current docs and tests define daily/idle expiry as starting a new session. Source inspection does show current main has no scheduled-reset continuity hint and creates a fresh session on stale daily/idle rollover.

## Solution Assessment

No, not as written. The feature direction is plausible, but the patch should use a safer one-shot prompt path that reaches queued runs, delimit transcript-derived text as untrusted context, and include real scheduled/idle reset proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep reset hints in queued prompt paths:** `src/auto-reply/reply/body.ts:48`
  - body: `applySessionHints` only mutates the prefixed command body, while queued followup/collect/steer paths build `queuedBody` from `effectiveBaseBody`. Because this block also clears the one-shot field, those queued runs can drop the scheduled-reset hint before the model sees it.
  - confidence: 0.88
- **[P2] Delimit prior assistant text before injecting it:** `src/auto-reply/reply/body.ts:47`
  - body: `resetHintMsg` comes from a transcript and may contain multiline or instruction-shaped text reflected from earlier turns. Embedding it directly in the wrapper makes prior output hard to distinguish from the surrounding hint instructions; normalize it or place it behind explicit quoted-context delimiters.
  - confidence: 0.84
- **[P3] Clear the in-memory hint without store context:** `src/auto-reply/reply/body.ts:49-50`
  - body: The hint fires whenever `params.sessionEntry` carries `lastAssistantMessageBeforeReset`, but clearing is gated on `sessionStore` and `sessionKey`. A caller that supplies only the entry can inject the same one-shot hint repeatedly.
  - confidence: 0.72
