## Summary

Keep open: current main still lacks scheduled daily/idle reset continuity, and this PR targets that real gap, but the proposed prompt-injection and one-shot clearing path has blocking queued-run and prompt-safety issues before merge.

## What This Changes

The PR adds a session-store field and transcript-tail helper to carry the last assistant message from scheduled daily/idle rollovers into the next prompt.

## Best Possible Solution

Revise this PR or replace it with a focused implementation that safely captures bounded assistant context before scheduled rollover archival, delivers it through both immediate and queued prompt paths, clears only after actual delivery, preserves explicit /new and /reset as blank starts, and adds regression tests plus a changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a stale daily/idle session creates a new session with resetTriggered=false, archives the old transcript, and current main has no continuity field or prompt hint for the prior assistant message.

## Solution Assessment

No, not as written. The feature direction is plausible, but the patch should treat carried transcript text as untrusted prior output, cover queued prompt delivery, clear state robustly after delivery, and include tests/changelog before merge.

  - body: applySessionHints only changes the prefixed prompt body, but followup/collect/steer queued runs use queuedBody built from effectiveBaseBody. Because this block also clears the session field, queued paths can drop the scheduled-reset hint before the model sees it.
