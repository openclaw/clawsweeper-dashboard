## Summary

Keep open: current main still lacks the scheduled daily/idle reset continuity hint, and the later merged continuity work only covers silent in-reply rotations. This PR remains useful source work, but the patch has prompt-delivery, clearing, security, changelog, and stale-head issues before it can merge.

## What This Changes

The PR adds a session-store field, transcript-tail helper, scheduled reset capture, and one-shot prompt hint to carry the last assistant message after daily/idle rollovers.

## Best Possible Solution

Revise or replace this PR with a focused implementation that safely carries bounded context through scheduled rollover, delivers it through immediate and queued prompt paths, clears only after actual delivery, preserves explicit resets as blank starts, and adds regression coverage plus a changelog entry.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a new feature. Source inspection does show the current-main behavior clearly: daily/idle rollover creates a new session without carrying prior assistant output, and applySessionHints only handles aborted runs.

## Solution Assessment

No, not as written. The feature direction is plausible, but this patch should handle queued prompt delivery, treat carried transcript text as untrusted, clear state robustly after delivery, and include tests/changelog before merge.
