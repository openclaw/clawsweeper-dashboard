## Summary

## What This Changes

The PR adds a SessionEntry field, a transcript tail reader, scheduled-reset capture of the previous assistant message, and one-shot prompt hint injection on the next reply.

## Best Possible Solution

Revise the feature as an explicitly approved scheduled-rollover continuity design that safely delimits prior transcript text, delivers through immediate and queued prompt paths, reserves the SessionEntry contract, and is proven in a real daily/idle reset setup.

## Reproduction Assessment

Not applicable as a current-main bug reproduction because this is a new continuity feature. The PR blockers are source-reproducible from the PR head queue path, the current-main SessionEntry key contract, and raw prompt interpolation.

## Solution Assessment

No, not as written. The safer path is to route the hint through the shared prompt envelope used by immediate and queued runs, reserve the field, delimit transcript-derived text as untrusted context, and clear it only after actual delivery.
