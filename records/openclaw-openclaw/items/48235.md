## Review Findings

Overall confidence: 0.91

Full review comments:

- **[P1] Delimit reset transcript text before injecting it:** `src/auto-reply/reply/body.ts:47`
  - body: `resetHintMsg` comes from the prior transcript and is interpolated directly into a model-visible note. A previous assistant message can contain newlines or instruction-shaped text, so this blurs old transcript content into new instructions unless it is normalized and wrapped as clearly untrusted quoted context.
  - confidence: 0.88
- **[P2] Keep continuity hints in queued prompts:** `src/auto-reply/reply/body.ts:50`
  - body: The hint is cleared as soon as `applySessionHints` runs, but current queued followup/collect/steer prompts are built from `effectiveBaseBody` via `queueBodyBase`, not the prefixed body. If the first post-reset turn is queued behind an active run, the one-shot hint can be removed from session state without ever reaching the model.
  - confidence: 0.86
- **[P3] Clear the hint before persistence guards:** `src/auto-reply/reply/body.ts:49`
  - body: The in-memory clear is inside the `sessionEntry && sessionStore && sessionKey` guard. Since the function contract makes store and key optional, a caller can inject the hint but leave `lastAssistantMessageBeforeReset` set on the entry, repeating the one-shot hint on later calls.
  - confidence: 0.78
