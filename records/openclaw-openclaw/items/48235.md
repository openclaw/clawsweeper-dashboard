## Summary

Keep open: current main does not implement scheduled-reset continuity, but this PR is not mergeable as written because it misses the SessionEntry reserved-key contract, can clear the hint before queued runs receive it, injects transcript-derived text without clear delimiting, and lacks real behavior proof.

## What This Changes

The PR adds a stored last-assistant-message continuity hint for daily/idle session resets, reads the transcript tail during rollover, and prepends the hint to the next reply.

## Best Possible Solution

Revise the feature so bounded, clearly delimited rollover context reaches immediate and queued prompt paths, reserves the SessionEntry field, clears only after delivery, and validates the intended daily/idle reset semantics.

## Reproduction Assessment

Not applicable as an existing bug reproduction because current docs define daily and idle expiry as new sessions. The PR defects are source-reproducible from the current prompt queue path and SessionEntry slot-key contract.

## Solution Assessment

No, not as written. The maintainable path is to carry the hint through the prompt envelope used by both immediate and queued runs, delimit transcript-derived text as untrusted context, reserve the new field, and clear it only after actual delivery.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Reserve the new session field:** `src/config/sessions/types.ts:95`
  - body: Adding `lastAssistantMessageBeforeReset` to `SessionEntry` also needs the key in `SESSION_ENTRY_RESERVED_SLOT_KEY_LIST`; that list has a type-level assertion covering every SessionEntry key, so the new core field otherwise breaks the plugin slot-key contract and type check.
  - confidence: 0.9
- **[P2] Carry reset hints into queued runs:** `src/auto-reply/reply/body.ts:47`
  - body: `applySessionHints` prepends and clears the reset hint before prompt bodies are rebuilt, but deferred followup/collect/steer paths send `queuedBody`, which is built from `effectiveBaseBody` rather than the prefixed body. If the first post-reset message is queued behind an active run, the hint is cleared without reaching the model.
  - confidence: 0.88
- **[P2] Delimit prior assistant text before injecting it:** `src/auto-reply/reply/body.ts:46-47`
  - body: `resetHintMsg` comes from the prior transcript and can contain newlines or instruction-shaped text reflected from earlier turns. Embedding it directly in the wrapper makes prior output hard to distinguish from the surrounding reset instructions.
  - confidence: 0.84
- **[P3] Clear the in-memory hint without store context:** `src/auto-reply/reply/body.ts:49`
  - body: The hint fires whenever `sessionEntry` carries `lastAssistantMessageBeforeReset`, but clearing is gated on `sessionStore` and `sessionKey`. A caller that supplies only the entry can inject the same one-shot hint repeatedly.
  - confidence: 0.72
