## Summary

Keep open: current main does not implement the requested continuity hint, and the PR still has source-level prompt delivery and prompt-safety blockers plus missing real behavior proof.

## What This Changes

The PR adds a session-entry continuity field, a transcript-tail assistant reader, scheduled reset capture, and a one-shot prompt hint after daily/idle rollover.

## Best Possible Solution

Revise the feature to carry bounded, clearly delimited scheduled-rollover context through both immediate and queued prompt paths, clear it after actual delivery, and preserve explicit /new and /reset as blank starts.

## Reproduction Assessment

Not applicable as a strict bug reproduction: current docs define daily/idle expiry as starting a new session. Source inspection does show current main has no scheduled-reset continuity hint and creates a fresh session on stale daily/idle rollover.

## Solution Assessment

No, not as written. The feature direction is plausible, but the patch should use a safer one-shot prompt path that reaches queued runs, delimit transcript-derived text as untrusted context, and include real scheduled/idle reset proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep reset hints in queued prompt paths:** `src/auto-reply/reply/body.ts:48`
  - body: `applySessionHints` only prepends the reset hint to the prefixed command body, while queued followup/collect/steer-backlog runs use `queuedBody` built from `effectiveBaseBody`. Because this block also clears the one-shot field, queued runs can drop the scheduled-reset hint before the model sees it.
  - confidence: 0.88
- **[P2] Delimit prior assistant text before injecting it:** `src/auto-reply/reply/body.ts:47`
  - body: `resetHintMsg` comes from a transcript and may contain multiline or instruction-shaped text reflected from earlier turns. Embedding it directly in the wrapper makes prior output hard to distinguish from the surrounding hint instructions; normalize it or place it behind explicit quoted-context delimiters.
  - confidence: 0.84
- **[P3] Clear the in-memory hint without store context:** `src/auto-reply/reply/body.ts:49-50`
  - body: The hint fires whenever `params.sessionEntry` carries `lastAssistantMessageBeforeReset`, but clearing is gated on `sessionStore` and `sessionKey`. A caller that supplies only the entry can inject the same one-shot hint repeatedly.
  - confidence: 0.72
