## Summary

Keep this PR open. Current main still lacks scheduled daily/idle reset continuity, but the proposed patch is not merge-ready because the hint can be cleared before queued prompts deliver it and the prior assistant text is injected into prompt instructions without strong delimiting.

## What This Changes

The PR adds a session-store field, an assistant-only transcript-tail helper, scheduled rollover capture in session initialization, and one-shot prompt injection/clearing for the last assistant message after daily/idle session reset.

## Best Possible Solution

Keep this PR or a replacement open until OpenClaw has a rebased implementation that captures a bounded assistant-only tail before scheduled rollover archival, treats the carried text as untrusted prior output, delivers it through both immediate and queued prompt paths, clears only after actual delivery, preserves explicit /new and /reset as fresh starts, and adds focused tests plus a changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: create a stale daily/idle session with a prior assistant transcript line, let initSessionState rotate it, and current main creates a new session/archive with resetTriggered=false and no assistant-tail hint field or prompt injection.

## Solution Assessment

No, not as written. The direction is plausible, but the patch should route the hint through the prompt body used by queued runs, delimit or normalize prior assistant text, clear the field robustly after delivery, and add regression coverage before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep reset hints in queued prompts before clearing:** `src/auto-reply/reply/body.ts:48-50`
  - body: applySessionHints adds the continuity hint only to the prefixed prompt path, while queued modes build queuedBody from effectiveBaseBody. Since this block also clears the session field, followup/collect/steer queued runs can lose the scheduled-reset hint before the model sees it.
  - confidence: 0.89
- **[P2] Delimit prior assistant text before injecting it:** `src/auto-reply/reply/body.ts:47`
  - body: The previous assistant message is inserted directly into the wrapper text. If it contains newlines or instruction-shaped text, it can escape the intended quote and read like part of the new instruction; treat it as untrusted prior output with explicit delimiters or normalization.
  - confidence: 0.84
- **[P3] Clear the in-memory hint when persistence is unavailable:** `src/auto-reply/reply/body.ts:49-50`
  - body: The hint fires whenever sessionEntry has lastAssistantMessageBeforeReset, but it is only cleared inside the sessionStore/sessionKey persistence guard. A caller with a sessionEntry but no store context will keep prepending the same hint on repeated calls.
  - confidence: 0.72
- **[P3] Add the required changelog entry:** `src/config/sessions/types.ts:90`
  - body: This is a user-facing feat, but the PR does not touch CHANGELOG.md. Repo policy requires a changelog entry with eligible contributor credit before merge.
  - confidence: 0.78
