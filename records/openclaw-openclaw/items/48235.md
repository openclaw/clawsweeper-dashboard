## Summary

Keep this PR open, but it should not merge as-is: current main still lacks the proposed scheduled-reset continuity behavior, while the branch has prompt-delivery, prompt-safety, and proof blockers that need author or maintainer follow-up.

## What This Changes

The PR adds a session-entry field, transcript-tail assistant reader, scheduled daily/idle reset capture, and one-shot prompt hint for continuity after automatic session rollover.

## Best Possible Solution

Revise the feature so bounded, clearly delimited scheduled-rollover context reaches both immediate and queued prompt paths, clears only after actual delivery, and keeps explicit `/new` and `/reset` as blank starts.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this PR proposes new continuity behavior, while current docs define daily/idle expiry as a new session. The PR defects are source-reproducible from the diff and current prompt queue path.

## Solution Assessment

No, not as written. The safer path is to put the hint in a prompt-envelope input used by both command and queued runs, delimit transcript-derived text as untrusted context, and clear the field only after the prompt that includes it is actually delivered.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep reset hints in queued prompt paths:** `src/auto-reply/reply/body.ts:48`
  - body: `applySessionHints` prepends and clears the reset hint on the prefixed command body, but queued followup/collect/steer runs send `queuedBody` built from `effectiveBaseBody`. When the first post-reset message is queued behind an active run, the hint can be cleared before the model receives it.
  - confidence: 0.88
- **[P2] Delimit prior assistant text before injecting it:** `src/auto-reply/reply/body.ts:47`
  - body: `resetHintMsg` comes from a transcript and may contain multiline or instruction-shaped text reflected from earlier turns. Embedding it directly in the wrapper makes prior output hard to distinguish from the surrounding hint instructions.
  - confidence: 0.84
- **[P3] Clear the in-memory hint without store context:** `src/auto-reply/reply/body.ts:49-50`
  - body: The hint fires whenever `params.sessionEntry` carries `lastAssistantMessageBeforeReset`, but clearing is gated on `sessionStore` and `sessionKey`. A caller that supplies only the entry can inject the same one-shot hint repeatedly.
  - confidence: 0.72
