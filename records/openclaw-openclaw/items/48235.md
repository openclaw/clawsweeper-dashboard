## Summary

Keep this PR open, but it is not merge-ready: current main does not implement the requested continuity hint, and the branch still has source-reproducible queued-delivery, session key contract, prompt-delimiting, real-proof, and product/security approval blockers.

## What This Changes

The PR adds a SessionEntry field, transcript tail reader, scheduled-reset capture, and one-shot prompt hint that carries the previous assistant message into the next reply.

## Best Possible Solution

Revise this behind an explicit scheduled-rollover continuity design: reserve the session field, route the hint through immediate and queued prompt envelopes, delimit prior transcript text as untrusted context, clear only after delivery, and prove it in a real reset setup.

## Reproduction Assessment

Not applicable as a current-main bug reproduction because this PR proposes a new continuity feature. The merge blockers are source-reproducible from the PR diff and current queued prompt/session key contracts.

## Solution Assessment

No, not as written. The safer solution needs explicit product/security approval plus a prompt-envelope implementation that works for queued runs and treats prior transcript text as untrusted context.
