## Summary

Keep open: current main and the latest release still have the silent-fallback approval delivery ambiguity, and this PR is a focused fix with sufficient terminal proof; the remaining blocker is maintainer acceptance of the compatibility-sensitive fail-closed routing semantics.

## What This Changes

The PR carries attempted/delivered approval delivery state through exec/plugin gateway handling, the approval forwarder, iOS push delivery, and regression tests so explicit failed or suppressed approval routes resolve as no-route.

## Best Possible Solution

Land the focused routing fix after maintainers explicitly accept strict configured-route failure semantics while preserving the no-explicit-route fallback and the new regression coverage.

## Reproduction Assessment

Yes. Source inspection of current main and v2026.6.6 shows configured delivery failures still collapse to boolean false and can take the same turn-source fallback path as no explicit delivery attempt; I did not run a live transport repro in this read-only review.

## Solution Assessment

Yes, with maintainer acceptance of the compatibility tradeoff. The PR fixes the shared approval routing and concrete delivery boundaries rather than adding channel-specific workarounds; preserving old fallback behavior would require an explicit compatibility policy.
