## Summary

Keep this open because current main still lacks oMLX-specific memorySearch documentation, and PR #74761 is an open same-author implementation candidate with closing syntax for the docs portion. The runtime already supports the reported oMLX shape through the existing OpenAI-compatible embedding path, so the remaining work is docs review rather than a core provider fix.

## What This Changes

This issue requests oMLX-specific memorySearch documentation, and optionally a first-class `omlx` embedding provider, for Apple Silicon users migrating from Ollama.

## Best Possible Solution

Review and land PR #74761, or an equivalent docs-only recipe, while leaving any native `omlx` adapter as a separate provider UX decision.

## Reproduction Assessment

Not applicable as a docs/feature request. The docs gap is high-confidence from current-main search and docs inspection, and the working runtime path is source-backed by the existing OpenAI-compatible remote embedding client.

## Solution Assessment

Yes for the docs portion: documenting `provider: "openai"` with `memorySearch.remote.baseUrl` is the narrowest maintainable solution. A native `omlx` adapter would be a separate UX/product decision, not required to solve the reported dead-end.
