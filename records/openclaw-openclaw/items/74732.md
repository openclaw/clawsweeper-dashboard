Keep this open because current main still lacks oMLX-specific memorySearch documentation, and PR #74761 is already the open, unmerged implementation candidate for the docs portion. ## What This Changes

The issue requests oMLX-specific memorySearch documentation and optionally a first-class `omlx` embedding provider for Apple Silicon users.

## Best Possible Solution

Review and land PR #74761 if maintainers are satisfied with the oMLX recipe, keeping the runtime on the existing OpenAI-compatible embedding adapter and treating a native `omlx` adapter as a separate product follow-up.
