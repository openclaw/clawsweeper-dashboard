## Summary

Keep this issue open. Current main already supports the reported oMLX route through the generic OpenAI-compatible embedding path, but current docs still do not mention oMLX or the Apple Silicon MLX recipe; the linked PR #74761 is open and unmerged, so the issue should stay open until that PR lands or maintainers choose a different docs/product path.

## What This Changes

Request to document, and optionally add first-class support for, oMLX as a memorySearch embedding provider via its OpenAI-compatible local endpoint.

## Best Possible Solution

Land or revise PR #74761 for the focused docs gap while keeping oMLX on the existing OpenAI-compatible embedding adapter. If maintainers want friendlier defaults or oMLX-specific diagnostics, track that as a separate native-provider enhancement after the docs recipe is settled.
