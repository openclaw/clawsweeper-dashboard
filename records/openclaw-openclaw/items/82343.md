## Repair Work Prompt

Fix the source-backed delivery bug from https://github.com/openclaw/openclaw/issues/82343. Current main can let a Discord final-send path receive a runtime config snapshot with unresolved SecretRefs after a Codex embedded run has completed, causing final response delivery to fail or time out. Keep the fix narrow: resolve scoped channel/account SecretRefs at the final outbound delivery boundary used by Discord durable sends and queued followup routing, and avoid broad Codex app-server watchdog changes except where directly required for this delivery path. Be careful that `getAgentRuntimeCommandSecretTargetIds()` excludes channel targets unless `includeChannelTargets` is true; prefer the existing scoped channel target helper when an originating channel/account is known. Add regression coverage that a Discord account token SecretRef is resolved before `withDiscordDeliveryRetry`/send-time account resolution, and that queued followup delivery preserves the execution-resolved config instead of reverting to a stale runtime snapshot. Do not change secret storage semantics or make unresolved SecretRefs silently fall back to env. Add a `CHANGELOG.md` fix entry because this is user-visible channel delivery behavior.
