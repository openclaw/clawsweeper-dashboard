## Summary

Keep open: the report matches the current Codex app-server embedded-run path, where a raw assistant `rawResponseItem/completed` is captured as text but still does not release the session without `turn/completed` or an assistant `item/completed`, so the channel delivery never gets a final result.

## What This Changes

The issue reports Codex app-server backed agent runs completing model output but timing out before Discord or Telegram delivery.

## Best Possible Solution

Teach the Codex app-server harness to release and deliver after a final raw assistant response item when the turn is otherwise idle, while preserving normal `turn/completed` handling and adding regression coverage for the reported timeout shape.

## Reproduction Assessment

Yes, by source: current tests already model a raw assistant `rawResponseItem/completed` without `turn/completed` and assert the run times out instead of delivering. I did not run a live channel repro because this review is read-only and would require real Codex/channel credentials.

## Solution Assessment

Yes: the narrow fix should be in the Codex app-server embedded-run completion logic, not Discord or Telegram, because both channels fail after the same app-server notification state. The safer shape is to extend the existing idle-release fallback with raw assistant final-response safeguards and focused tests.
