## Summary

Keep open: current main still captures raw assistant text from `rawResponseItem/completed` but does not treat that notification as a completion signal, and existing coverage asserts the reported timeout behavior instead of delivery.

## What This Changes

The issue asks OpenClaw to deliver completed Codex app-server responses to Discord and Telegram when the model finishes with only a raw assistant response item and no `turn/completed`.

## Best Possible Solution

Add a narrow Codex app-server completion fallback that can release and deliver a final raw assistant response item when the turn is otherwise idle, while preserving normal `turn/completed` handling and active-tool safeguards.

## Reproduction Assessment

Yes, source-reproducible: current tests model a raw assistant `rawResponseItem/completed` without `turn/completed` and assert that the run times out instead of delivering. I did not run a live channel repro because this review is read-only and would require real Codex and channel credentials.

## Solution Assessment

Yes: the narrowest maintainable fix is in the Codex app-server embedded-run completion logic, not Discord or Telegram. The implementation should add guarded raw-assistant final-response release behavior and regression tests for both positive and negative cases.

## Repair Work Prompt

Fix the Codex app-server embedded-run path for https://github.com/openclaw/openclaw/issues/82343. Current main stores assistant text from `rawResponseItem/completed` but keeps waiting for `turn/completed`, so Discord/Telegram sessions can time out after the model has produced a final raw assistant response. Add a narrow fallback in `extensions/codex/src/app-server/run-attempt.ts` that releases and delivers after a final raw assistant response item only when it is for the current turn and the turn is otherwise idle: do not release commentary/progress/tool-output echoes, user/developer abort markers, unrelated turn notifications, or while active native/tool items or turn-scoped requests are still open. Preserve existing `turn/completed` and `item/completed` behavior. Update `extensions/codex/src/app-server/run-attempt.test.ts`, including changing the current raw-assistant timeout case into a delivery case and adding negative coverage for active items/progress-like raw items. Avoid channel-layer changes and avoid adding a config option. Check related timeout context in https://github.com/openclaw/openclaw/issues/81114, but keep this PR scoped to raw assistant final-response delivery.
