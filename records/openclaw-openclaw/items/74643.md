## Summary

Keep open. This PR is the active implementation for #73680 and fixes the schema/docs/reply-directional parts, but it still leaves direct `openclaw agent --agent <id>` runs using only global verbose defaults, has directive tests with impossible verbose enum values, and lacks the required changelog entry.

## What This Changes

The branch adds per-agent `elevatedDefault` schema/type/help/generated metadata/docs, documents per-agent `verboseDefault`, resolves reply/directive defaults from the agent entry before global defaults, and adds config, agent-scope, and directive tests.

## Best Possible Solution

Land one aligned fix that keeps the schema/docs/generated updates, resolves selected-agent defaults consistently in reply and direct agent command paths, uses valid enum values in tests, and adds the required changelog entry.

## Reproduction Assessment

Yes. The original schema rejection is source-reproducible from current main's strict `AgentEntrySchema`, and the remaining PR defect is source-reproducible by following direct agent execution from global-only `agentCfg` to `resolvedVerboseLevel`.

## Solution Assessment

No. The schema/docs/reply-direction changes are the right direction, but the PR is not complete until direct agent command runs consume the selected agent default and the tests/changelog are cleaned up.
