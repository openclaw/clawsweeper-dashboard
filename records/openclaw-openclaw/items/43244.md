## What This Changes

The PR serializes gateway `config.set`, `config.patch`, and `config.apply` mutations through a dedicated command lane and adds mocked race tests for stale concurrent `config.patch` and `config.apply` requests.

## Best Possible Solution

Land a current-main gateway fix that performs read, base-hash validation, merge/validation, SecretRef handling, commit, response shaping, and restart planning inside one serialized mutation path while preserving current auth and no-op semantics.

## Reproduction Assessment

Yes, at source level: current main still computes gateway config write payloads from a pre-commit snapshot, and the commit helper writes the caller-computed config. I did not run a live gateway race repro in this read-only review.

## Solution Assessment

No, not as-is. Serializing the full mutation is the right direction, but this stale branch needs a current-main rework, structured lane-rejection handling, and real gateway/control RPC proof before merge.
