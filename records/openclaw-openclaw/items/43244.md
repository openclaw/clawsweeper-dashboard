## Summary

Keep this PR open because it targets a real gateway config-write race, but the current branch is stale against main and would regress auth-sensitive config commit safeguards while also missing real behavior proof.

## What This Changes

The PR serializes gateway `config.set`, `config.patch`, and `config.apply` through a `config-write` command lane and adds mocked stale-write tests for `config.patch` and `config.apply`.

## Best Possible Solution

Rebase serialization around the current `commitGatewayConfigWrite()` path, preserving SecretRef, no-op, shared-auth, restart, and persisted-result behavior, then add race coverage for all three handlers plus redacted real gateway proof.

## Reproduction Assessment

Yes, source-backed: current main reads and validates a config snapshot before computing each gateway RPC config mutation and does not serialize the read/compute/write sequence. I did not run a live gateway race in this read-only review.

## Solution Assessment

No, not as written. A dedicated gateway config write lane is a plausible fix shape, but this branch must wrap the current shared commit flow instead of older direct `writeConfigFile()` bodies.
