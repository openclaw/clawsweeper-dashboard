## Summary

Keep open: the gateway config write race is still real on current main, but this branch is stale against current auth-sensitive config write behavior and would bypass newer SecretRef/shared-auth safeguards. It also lacks after-fix real gateway runtime proof.

## What This Changes

The branch wraps gateway `config.set`, `config.patch`, and `config.apply` in a `config-write` lane and adds mocked stale concurrent write tests for `config.patch` and `config.apply`.

## Best Possible Solution

Rebase the serialization around the current `commitGatewayConfigWrite` flow, preserve SecretRef/no-op/shared-auth/restart behavior, add race coverage for all write handlers, and provide redacted real gateway proof.

## Reproduction Assessment

Yes, by source inspection: current main still checks `baseHash` before computing each gateway config mutation but does not serialize the read/compute/write sequence. I did not run a live gateway race in this read-only review.

## Solution Assessment

No, not as written. A dedicated write lane is the right shape, but the branch must queue the current shared commit path rather than older direct `writeConfigFile()` bodies.
