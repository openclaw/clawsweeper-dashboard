Full review comments:

- **[P2] Preserve primary Slack message text whitespace:** `extensions/slack/src/monitor/thread.ts:93`
  - body: This applies `cleanSlackTextCandidate` to existing `message.text`, and that helper collapses every whitespace run to a single space before returning thread starter/history text. On main, populated Slack text is returned unchanged after a trim-only emptiness check, so multiline alerts, logs, or code snippets in thread context would be flattened even when no attachment fallback is needed. Keep the original trimmed message text for the primary path and reserve whitespace collapsing for fallback fields if needed.
  - confidence: 0.91
- **[P2] Read standard attachment blocks:** `extensions/slack/src/monitor/thread.ts:82`
  - body: The fallback loop only checks `attachment.message_blocks`, but the current Slack SDK `MessageAttachment` contract exposes Block Kit content on `attachments[].blocks`. An alert whose visible attachment body is in standard attachment blocks and has empty top-level text/fallback/text/title still gets skipped, so this does not cover the Block Kit attachment case it is meant to fix. Add `blocks` to `SlackAttachment`, read it alongside any legacy field, and cover it with a regression test.
  - confidence: 0.88
