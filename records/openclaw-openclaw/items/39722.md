Keep open: current main and v2026.6.10 still let transcript-derived user text win in the TUI session picker without stripping gateway-injected inbound metadata, and the direct same-repository fix PR was closed unmerged. The open adjacent title-precedence PR only partially overlaps, so this issue remains the canonical tracker for the concrete Gateway/TUI bug.

## What This Changes

The issue reports that gateway-injected inbound metadata is being used as the TUI session title instead of the gateway display name or sanitized user message.

## Best Possible Solution

Land one narrow gateway/session-title fix that strips inbound metadata before derived-title extraction and threads the computed row displayName through both sync and async title hydration paths, with focused regression coverage and real behavior proof.

## Reproduction Assessment

Yes, at source level: the TUI asks for derived titles and renders `derivedTitle` before `displayName`, while the transcript parser returns first-user-message text without using the existing inbound metadata sanitizer.

## Solution Assessment

Yes: the narrow maintainable fix belongs at gateway session-title extraction and row title precedence, not broad persistence stripping or unrelated session-title quality work.
