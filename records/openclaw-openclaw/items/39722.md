## Label Justifications

- `P2`: The defect consistently impairs session identification for gateway-backed sessions but does not make the agent or gateway runtime unusable.
- `impact:session-state`: Injected transcript metadata corrupts the human-readable title associated with persisted session state.

## Summary

Current main still allows the gateway-injected metadata envelope to become `firstUserMessage` and then the derived session title, while the target-repo fix PR was closed unmerged and its same-head fork PR remains open. The issue is therefore still necessary and should stay open until the canonical implementation fixes both transcript sanitization and title-field precedence.

## What This Changes

Strip inbound gateway metadata from transcript-derived session titles and prefer populated session display fields before falling back to user-message text.

## Best Possible Solution

Land one canonical gateway change that sanitizes the first transcript user text with the existing inbound-metadata helper, threads the correct projected display fields into title precedence, and covers synchronous, asynchronous, TUI, and sessions-tool consumers without adding a TUI-only workaround.

## Reproduction Assessment

Yes, at source level with high confidence: a user transcript beginning with the injected metadata envelope flows through `extractTextFromContent` into `deriveSessionTitle`, which normalizes and truncates it as the title. A live TUI run was not performed in this read-only review.

## Solution Assessment

No single related proposal is sufficient as currently evidenced. The narrow maintainable solution is to combine canonical transcript sanitization with correct display-field precedence and focused regression coverage, rather than only shortening or filtering the TUI rendering.
