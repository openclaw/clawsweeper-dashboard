## Summary

Keep open: current main still has the reported source-level title path, and the linked fix PR remains open and unmerged. The issue should stay open until that PR or an equivalent replacement lands.

## What This Changes

The issue reports that TUI session titles are derived from gateway-injected inbound metadata instead of the usable session display name or sanitized user text.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/39790 or an equivalent rebased replacement that strips inbound metadata during title extraction, threads computed row display names into sync and async title hydration, and adds focused regression coverage.

## Reproduction Assessment

Yes, by source inspection. Current main requests derived titles in the TUI, displays them before displayName, extracts first user transcript text without inbound metadata stripping, and ignores computed row display names in title hydration.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing inbound metadata sanitizer and pass the computed session-row display name into the current sync and async derived-title paths before falling back to transcript text.
