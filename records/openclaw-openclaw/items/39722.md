## What This Changes

The issue reports TUI `/sessions` showing gateway-injected inbound metadata as the session title instead of a display name or sanitized user-message text.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/39790 or an equivalent replacement that strips inbound metadata before title extraction, passes computed row display names through sync and async title hydration, and keeps focused regression coverage.

## Reproduction Assessment

Yes. Source inspection shows current main requests derived titles in the TUI, renders them before `displayName`, extracts first-user-message text without inbound metadata stripping, and omits computed row display names from title hydration.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing inbound metadata sanitizer for title extraction and thread computed session-row display names into both sync and async `deriveSessionTitle` call paths.
