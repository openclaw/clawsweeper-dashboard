## Summary

Keep open: current main still has the reported source-level title path, and open PR #39790 explicitly targets this issue but has not merged.

## What This Changes

The issue reports that TUI `/sessions` titles use gateway-injected inbound metadata instead of a usable display name or sanitized user-message excerpt.

## Best Possible Solution

Land #39790 or an equivalent maintainer-reviewed fix that strips inbound metadata during transcript title extraction, prefers the row-level display name before transcript fallbacks, and adds focused regression coverage.

## Reproduction Assessment

Yes. Source inspection shows a high-confidence path: `/sessions` asks for derived titles and displays them before `displayName`, while title extraction returns first user text after only directive-tag stripping.

## Solution Assessment

Yes. The narrow maintainable direction is to reuse the existing inbound-metadata sanitizer in transcript title extraction and pass or prefer the computed row display name before raw transcript text.
