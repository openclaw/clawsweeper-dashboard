## Best Possible Solution

Land a fresh narrow target-repo fix that strips inbound metadata before title extraction, passes the computed row display name through sync and async derived-title hydration, and adds focused regression coverage.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing inbound metadata sanitizer for title extraction and thread computed session-row display names into both sync and async `deriveSessionTitle` call paths.
