## What This Changes

Fix TUI `/sessions` titles so gateway-injected inbound metadata is stripped and computed session display names are preferred before transcript-derived title text.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/39790 or an equivalent replacement that strips inbound metadata during title extraction, threads computed row display names through sync and async derived-title paths, and adds focused regression coverage.

## Reproduction Assessment

Yes, by source inspection: current main requests derived titles in the TUI, renders them before `displayName`, extracts transcript text without inbound metadata stripping, and omits computed row display names from title hydration.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing inbound metadata sanitizer for title extraction and pass computed session-row display names into both sync and async `deriveSessionTitle` call paths.
