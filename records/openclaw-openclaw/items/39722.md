## What This Changes

The issue requests that TUI `/sessions` titles stop using gateway-injected inbound metadata and instead prefer available display names or sanitized user-message text.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/39790 or an equivalent replacement that strips inbound metadata before title extraction, threads computed row display names through sync and async title hydration, and keeps focused regression coverage.

## Reproduction Assessment

Yes, by source inspection: current main requests derived titles in the TUI, renders them ahead of `displayName`, extracts first-user-message text without inbound metadata stripping, and omits computed row display names from title hydration.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing inbound metadata sanitizer for title extraction and pass computed session-row display names into both sync and async `deriveSessionTitle` call paths.
