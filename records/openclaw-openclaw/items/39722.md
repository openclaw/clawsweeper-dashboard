## Summary

Keep open: current main and the latest release still let transcript-derived inbound metadata win as the TUI session picker title, and the remaining fix has not landed.

## What This Changes

The issue requests that TUI/Gateway session-list titles prefer the computed session display name and strip gateway-injected inbound metadata before falling back to transcript-derived user text.

## Best Possible Solution

Land one narrow current-main fix that reuses stripInboundMetadata for title/preview transcript text and threads computed row displayName through all Gateway/TUI title-hydration paths, with focused regression coverage and a real sessions.list or TUI proof.

## Reproduction Assessment

Yes for source-level reproduction: a user transcript whose first message starts with the inbound metadata block flows through extractTextFromContent without the inbound sanitizer, and the TUI picker prefers that derived title over displayName. I did not run a live TUI/Gateway session in this read-only review.

## Solution Assessment

Yes, the reported direction is the narrow maintainable fix: reuse the existing inbound metadata sanitizer and pass the computed row displayName into title derivation. The current open fix branch needs maintainer handling because it is conflicting and stale against current main.
