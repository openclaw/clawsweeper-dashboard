## Summary

Keep this issue open. Current main still has the reported path: the TUI asks for derived session titles and displays them before displayName, gateway title derivation still sees only the inner session entry plus transcript-derived first user message, and transcript title extraction still does not strip the existing inbound metadata envelope. The provided context also shows #39790 as an open, unmerged implementation candidate that explicitly targets this issue, so the tracker should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that TUI session titles prefer the user-visible session display name or sanitized user text instead of gateway-injected inbound metadata blocks.

## Best Possible Solution

Land #39790 or an equivalent maintainer-reviewed fix that keeps row displayName precedence ahead of transcript-derived text, strips inbound metadata before deriving title and preview text, and adds focused regression coverage for external displayName precedence plus fenced Sender metadata stripping.

## Reproduction Assessment

Yes. The current-main code path is enough to reproduce statically: a gateway transcript whose first user content starts with a fenced Sender metadata block can still become fields.firstUserMessage, deriveSessionTitle can turn it into derivedTitle, and the TUI displays derivedTitle before displayName.

## Solution Assessment

Yes, the linked fix direction is the narrow maintainable path: reuse the existing stripInboundMetadata helper in transcript display extraction and pass or otherwise honor the computed outer displayName before raw transcript fallbacks. The issue should not close until that fix lands.
