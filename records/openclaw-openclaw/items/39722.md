## Summary

Keep open: current main still contains the reported title derivation path, and https://github.com/openclaw/openclaw/pull/39790 explicitly closes this issue but remains open and unmerged.

## What This Changes

The issue reports that TUI session titles are derived from gateway-injected inbound metadata instead of the session display name or sanitized user message.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/39790 or an equivalent rebased fix that strips inbound metadata during title extraction, prefers computed row display names in both sync and async session-list paths, and adds focused regression coverage.

## Reproduction Assessment

Yes, by source inspection. Current main requests derived titles in the TUI, displays them before `displayName`, returns first user transcript text after only directive-tag stripping, and ignores computed row display names in both title hydration paths.

## Solution Assessment

Yes for the direction, but the existing linked PR needs current-main follow-through. The narrow maintainable solution is to reuse `stripInboundMetadata` and thread the computed row display name into both sync and async derived-title paths before falling back to transcript text.
