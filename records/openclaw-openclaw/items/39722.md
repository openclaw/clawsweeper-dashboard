## Summary

## What This Changes

The issue reports that the TUI `/sessions` list derives titles from gateway-injected inbound metadata instead of a usable display name or sanitized user message.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/39790 or an equivalent replacement that strips inbound metadata before title extraction, passes computed row display names through sync and async title hydration, and keeps focused regression coverage.

## Reproduction Assessment

Yes. By source inspection, current main requests derived titles in the TUI, renders them ahead of `displayName`, extracts first-user-message text without inbound metadata stripping, and omits computed row display names from title hydration.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing inbound metadata sanitizer for title extraction and thread computed session-row display names into both sync and async `deriveSessionTitle` call paths.
