## Summary

Keep open: current main still has the reported source-level title path, and the linked fix PR is open, unmerged, and conflicting. This issue should stay open until that PR or an equivalent replacement lands.

## What This Changes

The issue reports that TUI `/sessions` shows gateway-injected `Sender (untrusted metadata)` text as the session title instead of a display name or sanitized user-message excerpt.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/39790 or an equivalent rebased replacement that strips inbound metadata during title extraction, routes computed row display names into sync and async title hydration, and adds focused regression coverage.

## Reproduction Assessment

Yes, by source inspection. Current main requests derived titles in the TUI, displays them before `displayName`, extracts first user transcript text without inbound metadata stripping, and ignores computed row display names in title hydration.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing inbound metadata sanitizer and pass the computed session-row display name into both sync and async derived-title paths before falling back to transcript text.
