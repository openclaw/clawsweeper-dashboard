## What This Changes

The issue asks for TUI `/sessions` titles to prefer populated session display names and strip gateway-injected inbound metadata before deriving transcript-based titles and previews.

## Best Possible Solution

Land #39790 or an equivalent maintainer-reviewed fix that honors row displayName before transcript fallbacks, strips inbound metadata during transcript title/preview extraction, and adds focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the TUI requests derived titles and displays them before displayName, while gateway derivation still uses only the inner entry plus unstripped transcript text.

## Solution Assessment

Yes. The linked fix direction is the narrow maintainable solution: reuse the existing inbound-metadata sanitizer at transcript display extraction and honor the computed outer displayName before raw transcript text.
