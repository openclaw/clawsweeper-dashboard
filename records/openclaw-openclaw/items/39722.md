## Summary

Keep open: current main still has the reported source-level title path, and https://github.com/openclaw/openclaw/pull/39790 explicitly closes this issue but remains open, dirty, and unmerged.

## What This Changes

The issue reports that TUI session lists derive titles from gateway-injected inbound metadata instead of the real user message or row display name.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/39790 or an equivalent rebased fix that strips inbound metadata during title extraction, prefers computed row display names in sync and async session-list paths, and adds focused regression coverage.

## Reproduction Assessment

Yes, by source inspection. Current main requests derived titles in the TUI, displays them before `displayName`, returns first user transcript text after only directive-tag stripping, and ignores computed row display names in both title hydration paths.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing inbound metadata sanitizer and thread the computed session-row display name into the current sync and async derived-title paths before falling back to transcript text.
