## What This Changes

The issue requests fixing Gateway/TUI session title derivation so injected inbound metadata is not used as the displayed session title and computed row display names are honored before transcript text.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/39790 or an equivalent replacement that strips inbound metadata during title extraction, passes computed row display names into both title hydration paths, and adds focused regression coverage.

## Reproduction Assessment

Yes, by source inspection. Current main requests derived titles in the TUI, displays them before `displayName`, extracts first user transcript text without inbound metadata stripping, and ignores computed row display names in title hydration.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing inbound metadata sanitizer and pass the computed session-row display name into both sync and async derived-title paths before falling back to transcript text.
