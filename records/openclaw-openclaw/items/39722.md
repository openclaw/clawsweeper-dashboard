## Summary

Keep open: current main still has the reported source-level title path, and https://github.com/openclaw/openclaw/pull/39790 explicitly closes this issue but remains open and unmerged.

## What This Changes

TUI `/sessions` should avoid using gateway-injected inbound metadata as the session title and should prefer usable session display names before transcript text.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/39790 or an equivalent rebased fix that strips inbound metadata during title extraction, threads computed row display names into both sync and async title hydration, and adds focused regression coverage.

## Reproduction Assessment

Yes, by source inspection. Current main requests derived titles in the TUI, displays them before `displayName`, returns first user transcript text after only directive-tag stripping, and ignores computed row display names in both title hydration paths.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing inbound metadata sanitizer and pass the computed session-row display name into both current derived-title paths before falling back to transcript text.
