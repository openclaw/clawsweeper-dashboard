## Summary

Keep open: current main still exposes the reported source-level title path, and https://github.com/openclaw/openclaw/pull/39790 is an open, unmerged fix PR that closes this issue.

## What This Changes

Fix the TUI sessions list so Gateway-injected inbound metadata is not used as a session title and computed display names are preferred before transcript fallback.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/39790 or an equivalent replacement that strips inbound metadata during title extraction, routes computed row display names into sync and async title hydration, and adds focused regression coverage.

## Reproduction Assessment

Yes, by source inspection. Current main requests derived titles in the TUI, displays them before `displayName`, extracts first user transcript text without inbound metadata stripping, and ignores computed row display names in title hydration.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing inbound metadata sanitizer and pass the computed session-row display name into both sync and async derived-title paths before falling back to transcript text.
