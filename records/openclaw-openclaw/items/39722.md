## Summary

Keep open: current main still has the reported source-level title path, and https://github.com/openclaw/openclaw/pull/39790 explicitly closes this issue but remains open and unmerged.

## What This Changes

The issue reports that TUI `/sessions` derives titles from gateway-injected inbound metadata instead of preferring usable session display metadata or sanitized user text.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/39790 or an equivalent rebased fix that strips inbound metadata during title extraction, prefers computed row display names in sync and async session-list paths, and adds focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the TUI asks for derived titles and displays them before `displayName`, while title extraction returns first user text after only directive-tag stripping and both current row-title paths ignore the computed row display name.

## Solution Assessment

Yes. The narrow maintainable direction is to reuse the existing inbound metadata sanitizer and pass or prefer the computed session row display name before falling back to transcript text.
