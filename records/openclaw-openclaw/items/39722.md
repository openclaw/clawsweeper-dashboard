## What This Changes

The issue reports that TUI session rows use a gateway-injected inbound metadata envelope as the derived session title instead of a usable display name or sanitized user message.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/39790 or an equivalent rebased replacement that strips inbound metadata before title extraction, threads computed row display names into sync and async title hydration, and adds focused regression coverage.

## Reproduction Assessment

Yes for a high-confidence source-level reproduction: current main requests derived titles, renders them before displayName, extracts first user-message text without inbound metadata stripping, and omits computed row displayName from title hydration. I did not run a live TUI/Gateway session in this read-only review.

## Solution Assessment

Yes at the fix direction level: the maintainable solution is to reuse the existing inbound metadata sanitizer for title extraction and pass the computed session-row display name into both sync and async `deriveSessionTitle` call paths. The current linked PR may need rebase or replacement before it can land.
