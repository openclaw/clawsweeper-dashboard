## What This Changes

The issue reports that the TUI `/sessions` list uses the gateway-injected inbound metadata envelope as the derived session title instead of the usable display name or sanitized user text.

## Best Possible Solution

Land a focused Gateway/TUI fix that strips inbound metadata before title extraction, threads the computed row display name through both sync and async derived-title hydration, and adds regression coverage.

## Reproduction Assessment

Yes, source-level: current main requests derived titles in the TUI, renders them before `displayName`, extracts first-user-message text without inbound metadata stripping, and omits computed row display names from both title hydration paths.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing inbound metadata sanitizer for title extraction and pass the computed session-row display name into both sync and async `deriveSessionTitle` call paths.
