## Summary

Keep open: current main still lacks `tools.fs.allowedRoots`, and this PR remains useful but not mergeable because the latest head has filesystem-boundary regressions and generated config/changelog drift.

## What This Changes

This PR adds `tools.fs.allowedRoots` and threads additional filesystem roots through sandbox path checks, host read/write/edit/apply_patch paths, media loading, config metadata, and regression tests.

## Best Possible Solution

Land a rebased implementation that preserves pinned root-contained mutation helpers, resolves sandbox bind mounts before allowed-root matching, keeps default `apply_patch` enablement, updates generated config artifacts and changelog, and covers the final contract with focused tests.

## Reproduction Assessment

Yes. Source inspection of PR head 41207898 is enough to reproduce the review failures: `apply_patch` deletes after validation with `fs.rm`, allowed-root checks run before sandbox bridge mount resolution, and `apply_patch` changes from default-enabled to explicit-enabled.

## Solution Assessment

No. The feature direction is useful, but this branch is not the best merge path until the filesystem boundary regressions, generated config drift, and changelog gap are corrected.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Restore pinned deletion for apply_patch deletes:** `src/agents/apply-patch.ts:311`
  - body: The workspace-only delete path validates with `assertSandboxPath()` and then calls `fs.rm()` on `sandboxResult.resolved`. Current main uses `removePathWithinRoot()`, so this reintroduces a check-then-use mutation path that can be redirected outside the matched root if a parent is swapped between validation and deletion.
  - confidence: 0.9
- **[P2] Resolve mounted sandbox paths before allowed-root checks:** `src/agents/pi-tools.read.ts:601`
  - body: This wrapper compares the container path against host-configured `additionalRoots` before `SandboxFsBridge.resolvePath` maps bind mounts. A bind such as `/host/shared:/shared` with `allowedRoots: ["/host/shared"]` still rejects `/shared/file`, so the new setting fails for Docker bind mounts.
  - confidence: 0.87
- **[P2] Preserve default apply_patch enablement:** `src/agents/pi-tools.ts:360-361`
  - body: This branch requires `tools.exec.applyPatch.enabled` to be explicitly true before creating `apply_patch`. Current main keeps `apply_patch` available for OpenAI models unless it is explicitly disabled, so merging this would silently remove the tool from normal coding profiles with no `applyPatch` config.
  - confidence: 0.86
- **[P2] Regenerate the public config schema:** `src/config/zod-schema.agent-runtime.ts:449`
  - body: Adding `tools.fs.allowedRoots` to the runtime schema also changes the generated base config schema and docs metadata. The PR does not update those generated artifacts, so config schema/docs checks will be stale until they are regenerated.
  - confidence: 0.82

  - confidence: 0.78
