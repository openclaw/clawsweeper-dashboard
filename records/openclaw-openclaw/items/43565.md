## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Restore root-scoped deletes for apply_patch:** `src/agents/apply-patch.ts:311`
  - body: At PR head, workspace-only deletes validate with `assertSandboxPath` and then call raw `fs.rm`. Current main routes apply_patch workspace mutations through a root-scoped helper, so this separates validation from mutation and can be redirected by a parent/symlink swap outside the matched root.
  - confidence: 0.91
- **[P2] Map sandbox mounts before checking allowedRoots:** `src/agents/pi-tools.read.ts:601`
  - body: The wrapper checks `additionalRoots` against `sandboxPath` before the sandbox bridge resolves non-workdir bind mounts. A Docker bind like host `/host/shared` mounted at `/shared` with `allowedRoots: ["/host/shared"]` still rejects `/shared/file`, so the advertised sandbox use case fails.
  - confidence: 0.87
- **[P2] Preserve the current media-root helper contract:** `src/agents/tools/media-tool-shared.ts:77-78`
  - body: This helper signature predates current main, where image, PDF, video, and music reference loaders call `resolveMediaToolLocalRoots` with media-source context. Rebase the allowedRoots work through those callers; otherwise the branch will either fail typecheck or leave newer media tools outside the new policy.
  - confidence: 0.78
