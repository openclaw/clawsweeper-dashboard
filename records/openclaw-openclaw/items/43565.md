## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Restore root-scoped apply_patch deletes:** `src/agents/apply-patch.ts:311`
  - body: Workspace-only deletes validate with `assertSandboxPath` and then call raw `fs.rm` on the resolved absolute path. Current main routes these mutations through the root-scoped helper, so this separates validation from mutation and can be redirected by a concurrent parent or symlink swap outside the matched root.
  - confidence: 0.91
- **[P2] Resolve sandbox mounts before allowedRoots checks:** `src/agents/pi-tools.read.ts:601`
  - body: The wrapper checks `additionalRoots` after only the container workdir mapping. A Docker bind mount such as container `/shared` backed by host `/host/shared` will still compare `/shared/file` to host allowlist roots before mount resolution, so the advertised sandbox use case rejects valid bind-mount paths.
  - confidence: 0.87
- **[P2] Preserve the media local-roots call signature:** `src/agents/tools/media-tool-shared.ts:77-78`
  - body: Current main callers pass media-source context into `resolveMediaToolLocalRoots`, but this patch changes the helper to a two-argument signature and updates only part of the caller set. Rebase the allowed-roots threading through the current media contract so image, PDF, music, and video callers keep compiling and enforcing the same policy.
  - confidence: 0.78
