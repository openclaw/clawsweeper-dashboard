## Summary

## What This Changes

The PR adds a `tools.fs.allowedRoots` config field and threads extra filesystem roots through sandbox checks, file tools, `apply_patch`, media loading, config schema/help/types, and tests.

## Best Possible Solution

Settle one maintainer-approved filesystem-root contract that composes with current hardened path guards, then land it with real allow/reject runtime proof and reconciliation with https://github.com/openclaw/openclaw/pull/59949.

## Reproduction Assessment

Yes for the review blockers by source inspection: PR head validates then uses raw `fs.rm` for deletes and drops current-main guard/media contracts. The requested `allowedRoots` behavior itself is a new feature, not a current-main bug reproduction.

## Solution Assessment

No. The direction is plausible, but this branch is not the best mergeable solution until it rebases through current main, preserves root-scoped operations and mount/media contracts, supplies real behavior proof, and is aligned with the overlapping includedWorkDirs proposal.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Restore root-scoped apply_patch deletes:** `src/agents/apply-patch.ts:311`
  - body: Workspace-only deletes validate with `assertSandboxPath` and then call raw `fs.rm` on the resolved absolute path. Current main routes these mutations through the root-scoped helper, so this separates validation from mutation and can be redirected by a concurrent parent or symlink swap outside the matched root.
  - confidence: 0.91
- **[P2] Preserve mounted-root guard options:** `src/agents/pi-tools.read.ts:581-584`
  - body: This wrapper drops current main's `additionalContainerMounts`, `pathParamKeys`, and `normalizeGuardedPathParams` options and then compares only the container-workdir-mapped path against `additionalRoots`. Sandbox bind-mount paths and non-`path` parameters that current main guards can be rejected, skipped, or forwarded incorrectly after this change.
  - confidence: 0.87
- **[P2] Preserve the media local-roots call signature:** `src/agents/tools/media-tool-shared.ts:75-78`
  - body: Current main callers pass media-source context as a third argument to `resolveMediaToolLocalRoots`, but this patch changes the helper to a two-argument signature. Rebase the allowed-roots threading through the current media contract so image, PDF, music, and video callers keep compiling and enforcing the same policy.
  - confidence: 0.78
