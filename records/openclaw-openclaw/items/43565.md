  - body: At PR head, workspace-only deletes validate with `assertSandboxPath` and then call raw `fs.rm`.   - confidence: 0.91
- **[P2] Map sandbox mounts before checking allowedRoots:** `src/agents/pi-tools.read.ts:601`
  - body: The wrapper checks `additionalRoots` against `sandboxPath` before the sandbox bridge resolves non-workdir bind mounts. A Docker bind like host `/host/shared` mounted at `/shared` with `allowedRoots: ["/host/shared"]` still rejects `/shared/file`, so the advertised sandbox use case fails.
  - confidence: 0.87
- **[P2] Preserve current media-root callers:** `src/agents/tools/media-tool-shared.ts:77-78`
  - body: Current main routes image, PDF, music, and video inputs through `resolveMediaToolLocalRoots` with media-source context. Rebase this helper and the allowed-roots threading through those current callers; otherwise newer media tools will keep diverging from the new filesystem policy.
  - confidence: 0.75
