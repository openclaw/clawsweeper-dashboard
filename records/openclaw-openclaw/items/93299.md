Full review comments:

- **[P2] Capture the baseline before running the task:** `src/daemon/schtasks.ts:1126`
  - body: `runScheduledTaskOrThrow` calls this helper only after `schtasks /Run`, so the new baseline can already include a fast-starting Scheduled Task process. That process is then treated as pre-existing and ignored, which can cause an unnecessary duplicate fallback launch; capture the process baseline before `/Run` and pass it into the launch decision.
  - confidence: 0.88
- **[P2] Do not accept another listener as fallback ownership:** `src/daemon/schtasks.ts:532`
  - body: When the verified listener list does not include the fallback child PID, this branch still returns success.   - confidence: 0.92

  - body: `check-test-types` fails because this object adds `pid` after the mock implementation was inferred as returning only `{ unref }`. Give the mock a consistent ChildProcess-like return type or type the mock so the new PID-bearing return value is valid.
  - confidence: 0.96
