## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Pin continuation replies outside Discord:** `src/plugins/host-hook-scheduled-turns.ts:384`
  - body: `/goal` requests announced continuation turns, but this scheduler only derives an explicit target for Discord session keys and otherwise falls back to `channel: "last"`. Telegram, WhatsApp, Slack, and other chat sessions that can run plugin slash commands can therefore send the follow-up to a stored/configured last route instead of the chat that started the goal. Please derive a pinned originating target for all supported session-key shapes or fail closed/no-announce when it cannot be pinned.
  - confidence: 0.84
- **[P2] Check cleanup before storing stopped state:** `extensions/goal/src/workflow.ts:38-41`
  - body: `clearSessionContinuationLease` returns `{ removed, failed }`, but the stop path ignores `failed` before writing a paused/done/blocked/waiting_approval state. If Cron cleanup fails, the UI says continuation stopped while the pending job can still run. Surface the cleanup failure or keep the prior state recoverable instead of claiming the lease was cleared.
  - confidence: 0.9
- **[P2] Check cleanup before deleting goal state:** `extensions/goal/src/command.ts:155-159`
  - body: `/goal clear` deletes the visible state after calling lease cleanup without checking whether cleanup failed. A failed clear can leave a scheduled follow-up turn behind with no visible goal to inspect or clear, so the command should verify `failed === 0` or report the failure and keep the goal state.
  - confidence: 0.89
