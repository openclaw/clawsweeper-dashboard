## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep continuation replies in the originating chat:** `src/plugins/host-hook-scheduled-turns.ts:101-102`
  - body: `/goal` promises same-session continuation, but this helper returns no explicit target for Telegram/WhatsApp/Signal-shaped session keys, so the lease is scheduled with `delivery.channel="last"`. The cron runner resolves `last` from `job.sessionKey`, while this job only sets `sessionTarget`, so non-Discord continuations can announce in the gateway's last/main chat instead of the chat that started the goal. Preserve the trusted session target for all supported channels before merging.
  - confidence: 0.88
- **[P2] Check lease cleanup before confirming stopped state:** `extensions/goal/src/workflow.ts:38-41`
  - body: `clearSessionContinuationLease` returns `{ failed }` when cron cleanup cannot remove a queued lease, but this path ignores the result and writes `paused`, `done`, or `waiting_approval` anyway. If cleanup fails, a pending continuation can still fire after the user stopped the goal, so the command/tool should surface the failure or keep state consistent instead of confirming a stop.
  - confidence: 0.9
- **[P2] Check lease cleanup before deleting goal state:** `extensions/goal/src/command.ts:155-159`
  - body: `/goal clear` also ignores the clear result before deleting the visible goal state. A failed scheduler cleanup would leave an orphaned continuation with no visible goal state left for the user to inspect or clear again, so only delete after successful cleanup or return a visible failure state.
  - confidence: 0.88
