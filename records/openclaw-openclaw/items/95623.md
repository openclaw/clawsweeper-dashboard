## Summary

Keep open: current main still has a credible Anthropic replay gap where strict tool-call ID sanitization is wired, but the sanitizer rewrites only normalized camel-case assistant tool-call blocks while adjacent transcript repair already recognizes raw `tool_use` and `function_call` replay shapes.

## What This Changes

The issue requests sanitizing OpenAI Responses composite tool-call IDs introduced by provider failover before those IDs are replayed to Anthropic with matching tool results.

## Best Possible Solution

Extend the canonical replay ID sanitizer to cover the raw replay block and ID field shapes already handled by transcript repair, then add an Anthropic-target regression for an OpenAI Responses `call_...|fc_...` failover transcript.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Source-level yes, live-run no: construct an Anthropic-target replay history containing an OpenAI Responses `call_abc|fc_123` assistant/tool-result pair in a raw replay shape and verify current main's sanitizer does not rewrite the raw assistant block.

## Solution Assessment

Is this the best way to solve the issue? Yes: the maintainable fix is to broaden the existing replay sanitizer's recognized ID shapes and add focused regression coverage, not to add a config knob or change Anthropic policy globally.
