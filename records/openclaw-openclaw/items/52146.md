## Summary

Keep this open. Current main still does not inject or expose a requester-specific sibling-agent roster to the model: prompt assembly only passes tool/runtime metadata, `agents_list` is still scoped to `sessions_spawn` allowlists, and A2A reachability is enforced separately by session tools rather than returned as roster data.

## What This Changes

Expose configured sibling agents, their identity name/theme, and requester-specific agent-to-agent reachability to agent sessions, either in the system prompt or a discoverable tool.

## Best Possible Solution

Add a focused live roster surface, preferably by extending `agents_list` with backward-compatible fields for `identity.name`, `identity.theme`, and a requester-specific `agentToAgentAllowed` value computed from current session visibility, sandbox context, and `tools.agentToAgent`. Update prompt guidance to tell agents to call the live roster when choosing collaborators, then document and test the payload and policy filtering.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists by inspecting current main: the prompt builder has no peer-agent input, the prompt wrapper only passes tool names/runtime metadata, and `agents_list` tests and implementation only cover spawn-target discovery.

## Solution Assessment

Yes, if implemented as a live tool surface rather than a static full prompt injection. That is the narrowest maintainable path because it avoids stale roster data while reusing existing config, identity, and session-policy helpers.

## Risks / Open Questions

- A roster implementation must not leak agent metadata beyond what the requester can use; `subagents.allowAgents`, `tools.sessions.visibility`, sandbox clamping, and `tools.agentToAgent` are separate gates today.
- Static prompt injection would add prompt-cache churn and can go stale during long sessions; a live discoverable tool plus prompt guidance is likely safer.
- Extending `agents_list` should preserve its existing `sessions_spawn` discovery contract and add fields in a backwards-compatible way.
