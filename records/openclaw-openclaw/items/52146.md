## Summary

Keep open. Current main still scopes `agents_list` to subagent spawn-target discovery and does not expose a policy-filtered sibling roster with `identity.theme` or A2A reachability, while the adjacent open issues cover narrower description and access-control pieces rather than this exact prompt/tool context contract.

## What This Changes

Request to expose a minimal sibling agent roster, either in the session system prompt or via a discoverable tool, with id, name, theme/role, and agent-to-agent reachability.

## Best Possible Solution

Define one policy-filtered agent-roster contract, likely by extending on-demand discovery before considering prompt injection, and document how identity metadata and A2A reachability are filtered.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection on current main confirms the requested sibling roster is absent from both the system prompt and existing discovery tool.

## Solution Assessment

Unclear. A discoverable, policy-filtered tool response is likely safer than unconditional prompt injection for privacy and prompt-cache stability, but maintainers need to decide the public roster surface and filtering semantics.
