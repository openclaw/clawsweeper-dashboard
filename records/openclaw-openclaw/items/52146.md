## Summary

Keep open: current main still lacks a model-facing sibling-agent roster. The prompt and `agents_list` surfaces expose tool/spawn metadata, while agent identity and A2A policy are available only on separate code paths.

## What This Changes

Expose configured sibling agents to sessions, including identity name/theme and whether the requester can use A2A messaging with each peer.

## Best Possible Solution

Add a backward-compatible live roster surface, preferably through `agents_list`, with identity name/theme and requester-specific A2A reachability plus docs, tests, and changelog.

## Reproduction Assessment

Yes. Source inspection on current main shows the prompt builder has no peer-agent input and `agents_list` exposes spawn metadata only, while identity and A2A policy are separate.

## Solution Assessment

Yes for an additive live `agents_list` extension; no for unconditional static prompt injection. The live-tool path avoids stale prompt state and reuses existing config and policy helpers.
