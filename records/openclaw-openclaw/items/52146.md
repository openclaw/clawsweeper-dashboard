## Summary

Keep open: current main still has the reported feature gap. The model-facing prompt and `agents_list` tool expose tool/runtime and subagent-spawn metadata, but not a requester-specific sibling roster with `identity.name`, `identity.theme`, or A2A reachability.

## What This Changes

Expose a minimal sibling-agent summary to an agent session, including peer id, identity name/theme, and whether agent-to-agent communication is allowed.

## Best Possible Solution

Add a backward-compatible live roster surface, preferably by extending `agents_list` with `identity.name`, `identity.theme`, and requester-specific `agentToAgentAllowed`, then add prompt guidance, docs, tests, and a changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main shows the prompt builder has no peer-agent input and `agents_list` only exposes subagent-spawn discovery metadata, while identity and A2A policy are available on separate code paths.

## Solution Assessment

Yes for the live-tool direction; no for an unconditional static prompt roster. Extending `agents_list` additively is the narrowest maintainable path because it avoids stale prompt data and reuses existing config, identity, and A2A policy helpers.

## Repair Work Prompt

Implement issue #52146 as a focused live roster enhancement, not a static dump of all configured agents into every prompt. Extend `agents_list` backward-compatibly so each visible entry can include `identity.name`, `identity.theme`, and a requester-specific `agentToAgentAllowed` value computed from current session visibility, sandbox constraints, and `tools.agentToAgent`. Reuse existing config/identity/session-policy helpers; do not add core knowledge of bundled plugin ids, do not expose unfiltered agent metadata, and keep existing `agents_list` fields stable. Add prompt guidance that tells agents to call the live roster when choosing collaborators, update `docs/tools/subagents.md`, and add a user-facing `CHANGELOG.md` entry. Include focused regression coverage for identity fields, A2A allowed/denied cases, and prompt guidance.
