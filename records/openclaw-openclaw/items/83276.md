## Summary

Keep open: current main has plugin approval gates, OpenResponses client-tool handoff, queue abort/steer behavior, Lobster workflow resume tokens, and a Codex-specific user-input bridge, but not the requested generic plugin/Gateway tool-call pause primitive that resumes with arbitrary human input as the tool result.

## What This Changes

Request a generic HITL runtime primitive that can pause a tool call, ask a human for input, and resume the agent turn with that input as the tool result.

## Best Possible Solution

Run a maintainer design pass and either add a durable SDK/Gateway HITL pause-resume contract or document Lobster, Codex user input, and plugin approvals as the intended scoped alternatives.

## Reproduction Assessment

Not applicable; this is a feature request rather than broken existing behavior. Source inspection confirms adjacent approval, abort, Lobster, and Codex user-input paths exist, but not the requested generic human-input tool-result path.

## Solution Assessment

Unclear; extending `TrustedToolPolicy` with a pause action is plausible, but it needs maintainer design against existing allow/deny approvals, OpenResponses handoff, Codex user input, queue abort semantics, and Lobster resume tokens.
