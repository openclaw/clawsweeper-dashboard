## Summary

Keep open: current main has plugin approval gates, OpenResponses client-tool handoff, queue abort/steer behavior, and workflow-level Lobster approvals, but none provides the requested generic tool-call pause primitive that resumes with arbitrary human input as the tool result. The adjacent open items cover narrower HITL plugin API gaps, exec approvals, or abort/steer semantics, so they do not supersede this request.

## What This Changes

Request a generic runtime primitive for pausing tool-call execution for human input and resuming with that answer as the tool result.

## Best Possible Solution

Define a durable, additive SDK/Gateway HITL pause-resume contract, or explicitly document Lobster/Task Flow plus plugin approval patterns as the intended boundary for this workflow class.

## Reproduction Assessment

Not applicable; this is a feature request rather than broken existing behavior. Source inspection confirms adjacent approval, client-tool handoff, and workflow approval surfaces exist, but not the requested generic human-input tool-result path.

## Solution Assessment

Unclear; extending TrustedToolPolicy with a pause action is plausible, but maintainers need to choose the API boundary against existing requireApproval, OpenResponses function_call_output continuation, exec approvals, Task Flow/Lobster, and transcript semantics.
