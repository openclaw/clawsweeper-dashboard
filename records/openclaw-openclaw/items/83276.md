## Summary

Keep open: current main and v2026.6.8 have useful plugin approval, trusted-policy, and next-turn injection seams, but not the requested generic pause/resume primitive that returns arbitrary human input as a tool result.

## What This Changes

The issue requests a generic HITL runtime primitive that can pause a tool call, ask a human a question, and resume the agent run with that response.

## Best Possible Solution

Design a narrow plugin/runtime contract for HITL tool-call suspension that composes with plugin approvals and next-turn injection, preserves exact run/tool-call binding, and avoids adding a broad orchestration layer to core.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.8 expose decision-only plugin approvals and trusted-policy hooks, but no typed pause/question primitive that resumes a tool call with arbitrary human input.

## Solution Assessment

Unclear: extending TrustedToolPolicy directly is plausible, but the best fix should first decide whether to extend existing plugin approval and next-turn injection seams instead of adding a parallel client-tool-style interrupt path.
