## Summary

Keep open: current main and v2026.6.8 have approval gates, client-tool handoff, sessions_yield, and Lobster workflow resume tokens, but not the requested generic tool-call pause primitive that asks a human for arbitrary input and resumes with that value as the tool result.

## What This Changes

The issue requests a generic runtime primitive for pausing a tool call, asking a human for input, and resuming the agent turn with that response.

## Best Possible Solution

Run a maintainer design pass for a narrow SDK/Gateway HITL pause-resume contract, or explicitly document that arbitrary pipeline HITL should stay in workflow plugins while only smaller approval API gaps move into core.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main lacks the requested arbitrary human-input pause/resume primitive.

## Solution Assessment

Unclear. Extending TrustedToolPolicy is plausible, but the best solution needs maintainer API and trust-model design against existing plugin approvals, OpenResponses client tools, sessions_yield, Lobster, and related open approval issues.
