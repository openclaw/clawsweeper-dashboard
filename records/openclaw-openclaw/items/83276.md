## Summary

Keep open: current main has plugin approval gates, OpenResponses client-tool handoff, next-turn injections, and harness-specific user-input bridges, but it still lacks a generic tool-call pause primitive that asks a human for arbitrary input and resumes with that value as the tool result. The closest durable interrupt PR was closed unmerged as a product decision, so this remains a maintainer design item rather than implemented or duplicate cleanup.

## What This Changes

The issue requests a generic HITL interrupt primitive that can pause a tool call, ask a human for input, and resume the tool pipeline with that response.

## Best Possible Solution

Run a maintainer design pass for a narrow SDK/Gateway HITL pause-resume contract, or explicitly document that arbitrary pipeline HITL should stay in workflow plugins while narrower approval gaps remain tracked in #23451 and #78308.

## Reproduction Assessment

Yes at source level for the feature gap: current main's policy, hook, and gateway approval contracts are decision-only or harness-specific, so they cannot return arbitrary human input as a generic tool result.

## Solution Assessment

Unclear until maintainers choose the product and security model; a generic SDK/Gateway pause-resume contract is plausible, but it must be reconciled with existing plugin approvals, next-turn injection, harness-specific user-input bridges, #23451, and the earlier closed #19181 design.
