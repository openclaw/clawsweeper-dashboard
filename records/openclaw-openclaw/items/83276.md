## Summary

Keep open: current main has plugin approval gates, OpenResponses client-tool handoff, queue abort/steer behavior, and Lobster approval resume tokens, but none provides the requested generic tool-call pause primitive that resumes with arbitrary human input as the tool result. The related https://github.com/openclaw/openclaw/pull/55289 context is useful, but that PR is closed unmerged and its body explicitly scoped out a long-lived pause/resume mechanism.

## What This Changes

Request a generic HITL runtime primitive that can pause a tool call, ask a human question, and resume the agent with the human response as the tool result.

## Best Possible Solution

Run a maintainer design pass and either add a durable SDK/Gateway HITL pause-resume contract or document Lobster/Task Flow plus plugin patterns as the intended boundary.

## Reproduction Assessment

Not applicable; this is a feature request rather than broken existing behavior. Source inspection confirms adjacent approval, abort, client-tool handoff, and Lobster resume surfaces exist, but not the requested generic human-input tool-result path.

## Solution Assessment

Unclear; extending `TrustedToolPolicy` with a pause action is plausible, but it needs maintainer design against existing `requireApproval`, OpenResponses client tools, queue abort/steer semantics, and Lobster resume tokens before implementation.
