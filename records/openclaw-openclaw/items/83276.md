## Summary

Keep open: this is a real feature/API request, and current main has adjacent approval, workflow-resume, client-tool, and abort/steer primitives but no generic tool-call pause that resumes with arbitrary human input as a tool result.

## What This Changes

The issue requests a generic runtime primitive that pauses tool execution for human input and resumes the agent turn with that response.

## Best Possible Solution

Run a maintainer design pass that either defines a durable SDK/Gateway HITL pause-resume contract or documents Lobster/Task Flow plus plugin approvals as the intended boundary for Xstack-style workflows.

## Solution Assessment

Unclear; extending `TrustedToolPolicy` with a pause action is plausible, but it needs to be reconciled with `requireApproval`, Lobster/Task Flow, abort/steer semantics, and session persistence before implementation.
