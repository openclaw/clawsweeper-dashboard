## Summary

Keep open: current main still has approval gates, OpenResponses client-tool handoff, queue abort/steer behavior, and Lobster-style resume surfaces, but no generic runtime primitive that pauses a tool call and resumes it with arbitrary human input as the tool result; the request needs maintainer API/product design rather than conservative cleanup closure.

## What This Changes

The issue requests a generic Gateway/SDK HITL primitive that can pause agent execution during a tool call, notify a human, and resume with the human response as the tool result.

## Best Possible Solution

Run a maintainer design pass and either add a durable Gateway/SDK HITL pause-resume contract with session-safe resume semantics, or document existing Lobster/task-flow plus plugin approval patterns as the intended boundary.

## Reproduction Assessment

Not applicable; this is a feature request rather than broken existing behavior. Source inspection confirms adjacent approval, abort, client-tool handoff, and resume surfaces exist, but not the requested generic human-input tool-result path.

## Solution Assessment

Unclear; extending TrustedToolPolicy with a pause action is plausible, but the right shape needs maintainer design against existing requireApproval, OpenResponses client tools, queue abort/steer semantics, and durable session resume requirements.
