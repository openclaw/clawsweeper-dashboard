## Summary

Keep open: current main and v2026.6.8 have plugin approval gates, OpenResponses client-tool handoff, in-channel aborts, and Lobster resume tokens, but they do not provide the requested generic tool-call pause primitive that asks a human for arbitrary input and resumes with that value as the tool result.

## What This Changes

The issue requests a generic HITL runtime primitive that can pause a tool call, collect human input, and resume agent execution with that response.

## Best Possible Solution

Run a maintainer design pass for a narrow SDK/Gateway HITL pause-resume contract, or explicitly document that arbitrary pipeline HITL should stay in workflow plugins while smaller approval API gaps move into core.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main and v2026.6.8 lack the requested arbitrary human-input pause/resume primitive.

## Solution Assessment

Unclear. Extending TrustedToolPolicy is plausible, but the best solution needs maintainer API and trust-model design against existing plugin approvals, OpenResponses client tools, Lobster resume tokens, and adjacent approval issues.
