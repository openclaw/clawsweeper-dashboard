## Summary

Keep open: current main has adjacent plugin approval gates, OpenResponses client-tool continuation, active-run interrupt/steer behavior, and Lobster workflow approval tokens, but not the requested generic tool-call pause that resumes with arbitrary human input as the tool result. The related open API-gap issue at https://github.com/openclaw/openclaw/issues/82336 is relevant, but it does not fully subsume this broader HITL pause/resume primitive.

## What This Changes

The issue requests a generic runtime primitive for pausing a tool call for human input and resuming the agent turn with that input as the tool result.

## Best Possible Solution

Run a maintainer design pass that either adds an additive SDK/Gateway HITL pause-resume contract with explicit authorization and transcript semantics, or scopes this use case to Lobster/Task Flow plus narrower plugin approval APIs such as https://github.com/openclaw/openclaw/issues/82336.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing behavior report. Source inspection gives high confidence that current main has adjacent approval, interrupt, and workflow-resume surfaces, but not the requested generic human-input tool-result path.

## Solution Assessment

Unclear; extending `TrustedToolPolicy` with a pause action is plausible, but current approval and OpenResponses contracts only cover allow/deny decisions or follow-up `function_call_output`. A design-first SDK/Gateway contract is safer than a narrow implementation patch.
