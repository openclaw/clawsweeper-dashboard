## Summary

Keep open: current main has plugin approval gates, OpenResponses client-tool handoff, queue/steer behavior, and Lobster/Task Flow workflow alternatives, but it still lacks the requested generic tool-call HITL pause/resume contract that resumes with arbitrary human input as the tool result.

## What This Changes

The issue requests a generic runtime primitive that can pause a tool call, ask a human a question, and resume the agent with the human response as the tool result.

## Best Possible Solution

Run a maintainer design pass and either add a durable SDK/Gateway HITL pause-resume contract or document Lobster/Task Flow plus plugin approval patterns as the intended boundary.

## Reproduction Assessment

Not applicable; this is a feature request rather than broken existing behavior. Source inspection shows the requested free-form human-input tool-result path is absent from current main.

## Solution Assessment

Unclear; extending TrustedToolPolicy with a pause action is plausible, but the maintainable solution needs a maintainer-owned contract across SDK, Gateway session state, approval auth, transcript/tool-result semantics, and existing Lobster/Task Flow boundaries.
