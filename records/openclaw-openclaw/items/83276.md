## Summary

Keep open: current main and v2026.6.8 still provide approval gates, client-tool handoff, in-channel aborts, and Lobster workflow resume tokens, but not the requested generic tool-call pause primitive that asks a human for arbitrary input and resumes with that input as the tool result.

## What This Changes

The issue requests a generic HITL runtime primitive that lets tool policy pause an agent turn, ask a human a question, and resume the tool call with the human response.

## Best Possible Solution

Run a maintainer design pass for a narrow SDK/Gateway HITL pause-resume contract, or explicitly document that arbitrary pipeline HITL should stay in workflow plugins such as Lobster while only smaller approval API gaps move into core.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main lacks the requested arbitrary human-input pause/resume primitive.

## Solution Assessment

Unclear. Extending TrustedToolPolicy is plausible, but the best solution needs maintainer API and trust-model design against existing plugin approvals, OpenResponses client tools, Lobster resume tokens, and adjacent approval issues.
