## Summary

Keep open: current main and v2026.6.6 have approval gates, OpenResponses client-tool handoff, queue interruption, and Lobster approval resume tokens, but no generic tool-call pause primitive that accepts arbitrary human input and resumes with that input as the tool result.

## What This Changes

Request a generic runtime primitive that lets trusted tool policies pause a tool call, ask a human, and resume the agent with the human response as the tool result.

## Best Possible Solution

Run a maintainer design pass and choose either a durable SDK/Gateway HITL pause-resume contract or an explicit documented boundary using plugin approvals, Lobster, and narrower approval API extensions.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives high confidence that current main lacks the requested free-form HITL tool-result pause/resume primitive.

## Solution Assessment

Unclear. Extending `TrustedToolPolicy` with a pause action is plausible, but the best solution needs maintainer API design against existing plugin approvals, OpenResponses client tools, Lobster resume tokens, and adjacent HITL plugin API work.
