## Summary

Keep open: current main has tool approval gates, Task Flow/Lobster approval resume, and client-tool handoff, but none provide a generic tool-call pause primitive that resumes with arbitrary human input; this needs product/API design rather than cleanup closure.

## What This Changes

Request a generic runtime primitive that pauses a tool call for human input and resumes the agent with that response for HITL pipeline workflows.

## Best Possible Solution

Run a maintainer design pass and either add a durable SDK/Gateway HITL pause-resume contract or document Task Flow/Lobster plus an Xstack-style plugin pattern as the intended boundary.

## Solution Assessment
