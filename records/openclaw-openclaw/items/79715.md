## Summary

Keep open: this member-authored, maintainer-labeled PR addresses a real current-main schema collision, but the patch should not merge until recognized invalid identity files stop yielding fresh unsaved device IDs.

## What This Changes

This PR updates the TypeScript and Swift device identity loaders to accept and migrate each other's `identity/device.json` schemas, preserves recognized invalid identity files, adjusts macOS protocol call sites, adds regression tests, and updates the changelog.

## Best Possible Solution

Keep the dual-schema compatibility direction, but change recognized-invalid handling so invalid OpenClaw identity files are preserved without presenting a fresh unsaved device identity; maintainer should choose fail-closed or explicit repair/quarantine semantics.

## Reproduction Assessment

Yes. Current main has a source-reproducible collision: TypeScript writes PEM identity fields under `OPENCLAW_STATE_DIR` while Swift reads raw-key fields from the same path, and https://github.com/openclaw/openclaw/issues/76815 includes concrete relaunch and pairing churn evidence.

## Solution Assessment

No. Supporting both schemas with key-pair validation is the right direction, but returning unsaved generated identities for recognized invalid files reintroduces unstable device IDs in an error path this PR explicitly tries to make safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.81

Full review comments:

- **[P2] Do not return ephemeral IDs for invalid TypeScript identities:** `src/infra/device-identity.ts:239`
  - body: When the TypeScript loader recognizes an invalid OpenClaw identity file, this branch preserves the file but returns a freshly generated identity without saving it. The next process launch hits the same invalid file and generates a different device ID, which can recreate the pairing churn this fix is meant to stop; surface a repair/failure path or otherwise provide a stable identity while preserving evidence.
  - confidence: 0.82
- **[P2] Do not return ephemeral IDs for invalid Swift identities:** `apps/shared/OpenClawKit/Sources/OpenClawKit/DeviceIdentity.swift:60`
  - body: The Swift loader has the same preserved-invalid branch: it leaves the invalid identity file in place and returns a newly generated, unsaved identity. A relaunch will generate a different device ID again, so macOS can still appear as a new device repeatedly for this recognized-invalid case.
  - confidence: 0.78
