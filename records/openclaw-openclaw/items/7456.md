## What This Changes

The issue requests in-session Back navigation in the onboarding wizard, with prior answers preserved when moving backward and forward.

## Best Possible Solution

Add a shared typed Back/previous contract and setup-flow history model across WizardPrompter, Clack prompts, WizardSession, Gateway schema/handlers, generated clients, and app clients.

## Reproduction Assessment

Yes, at source level: current main exposes only forward answer/next and cancel/status paths, and the setup flow is still a sequential async flow. I did not run the live wizard because this read-only review should not write local config or wizard state.

## Solution Assessment

Yes directionally, but the exact API/product shape needs maintainer approval. The maintainable path is a shared Back/previous contract rather than a one-off local prompt option.
