## Summary

Keep open; current main and the latest release still expose the onboarding wizard as a forward/cancel flow, and the only Back-navigation implementation PR is closed unmerged. The issue remains the canonical tracker for in-session previous-step navigation with preserved answers.

## What This Changes

The issue requests in-session Back navigation for `openclaw onboard`, with previous selections preserved when moving backward and forward.

## Best Possible Solution

Add an additive typed Back/previous contract across `WizardPrompter`, Clack prompts, `WizardSession`, Gateway wizard schema/handlers, generated clients, and onboarding state so previous answers are preserved consistently.

## Reproduction Assessment

Yes for current behavior: source inspection shows the prompt, session, and Gateway wizard surfaces only advance or cancel, with no Back/previous path. I did not run a live TTY wizard because this was a read-only review, but the missing contract is clear from source and docs.

## Solution Assessment

Yes, the best direction is a shared typed navigation contract rather than a Clack-only menu item. A terminal-only Back option would leave Gateway clients and preserved-answer state inconsistent.
