## Summary

Keep open. Current main still lacks shared in-session Back navigation for `openclaw onboard`, and the directly related implementation PR was closed unmerged, so this remains valid feature/API work rather than conservative cleanup.

## What This Changes

The issue requests a Back option during the onboarding wizard that returns to prior steps while preserving already-entered selections.

## Best Possible Solution

Add an additive typed Back/previous contract across the wizard prompt, session, Gateway, generated-client, and onboarding client surfaces, backed by a history model that safely preserves answers.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Source inspection on current main shows forward answer, cancel, and status paths but no previous-step path.

## Solution Assessment

Unclear until maintainers settle the API and UX contract across CLI and Gateway clients. The maintainable direction is a typed Back/previous flow with preserved answers; a CLI-only prompt option would leave the shared wizard protocol incomplete.
