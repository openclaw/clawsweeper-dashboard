## Summary

Keep open. Current main still lacks shared in-session Back navigation for `openclaw onboard`, and the directly related implementation PR was closed unmerged, so this remains valid feature/API work rather than conservative cleanup.

## What This Changes

The issue requests Back navigation in the `openclaw onboard` wizard with prior answers preserved when moving backward and forward.

## Best Possible Solution

Add an additive typed Back/previous contract across `WizardPrompter`, Clack prompts, `WizardSession`, Gateway schema/handlers, generated clients, and onboarding clients, backed by a history model that preserves safe answers.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Source inspection on current main shows forward answer, cancel, and status paths but no previous-step path.

## Solution Assessment

Unclear until maintainers settle the cross-surface API and UX contract. The maintainable direction is a typed Back/previous flow with preserved answers; only adding a local CLI prompt option would not cover Gateway/client wizard surfaces.
