## Summary

Keep open. Current main still has a forward/cancel onboarding wizard with no shared Back/previous contract, and the direct Back-navigation PR was closed unmerged while related resume/dialog work covers different problems.

## What This Changes

Add in-session Back navigation to `openclaw onboard` so users can return to earlier onboarding steps with prior answers preserved.

## Best Possible Solution

Ship an additive typed Back/previous wizard contract with answer preservation across CLI prompts, `WizardSession`, Gateway RPC/schema, generated clients, docs, and onboarding clients.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: advance through `openclaw onboard` or the shared wizard RPC flow and there is no Back option, previous-step result, or `wizard.previous` method to invoke.

## Solution Assessment

Unclear until maintainers settle the API/UX contract. The maintainable solution is a coordinated typed Back contract and history model; a Clack-only or prompt-only patch would be incomplete.
