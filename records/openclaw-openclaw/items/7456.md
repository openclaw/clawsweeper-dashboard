## Summary

Keep open: current main and the latest release still expose the onboarding wizard as forward/cancel only, and the direct Back-navigation PR was closed unmerged without a shipped replacement.

## What This Changes

Add in-session Back navigation to the onboarding wizard, with prior answers preserved when moving backward and forward.

## Best Possible Solution

Introduce a shared typed Back/previous contract and setup-flow history model across WizardPrompter, Clack prompts, WizardSession, Gateway schema/handlers, generated clients, and UI clients, with preserved-answer regression coverage.

## Reproduction Assessment

Yes, at source level: current main exposes only forward answer/next and cancel/status paths for onboarding, and the setup flow remains sequential. I did not run the live wizard because this read-only review should not write local config or wizard state.

## Solution Assessment

Yes directionally, but the exact API/product shape needs maintainer approval. The maintainable path is a shared Back/previous contract rather than a one-off local prompt option.
