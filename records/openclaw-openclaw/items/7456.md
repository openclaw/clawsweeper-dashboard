## Summary

Keep open: current main and the latest release still expose `openclaw onboard` as a forward/cancel wizard, while the related Back-navigation and dialog PRs are closed unmerged and do not ship preserved previous-step navigation.

## What This Changes

The issue requests Back/previous-step navigation in `openclaw onboard` with prior answers preserved.

## Best Possible Solution

Add an additive typed previous/back contract through the shared wizard prompter, session, Gateway protocol, generated clients, and UI clients, with answer preservation across backward and forward navigation.

## Reproduction Assessment

Yes, at source level: current `openclaw onboard` uses a sequential prompt flow and the shared session/protocol surfaces only support next, answer, cancel, and status. I did not run the live wizard because onboarding can write config/state in this read-only review.

## Solution Assessment

Yes directionally, but it needs maintainer product approval and a cross-surface implementation. The safer solution is a shared typed Back/previous contract rather than a local `[Back]` option in only part of the wizard.
