## Summary

Keep open. Current main still exposes onboarding as a forward/cancel flow, and the related implementation attempts are closed or cover different workflows rather than preserved in-session Back navigation.

## What This Changes

The issue requests a Back option in `openclaw onboard` that returns to previous steps while preserving prior answers.

## Best Possible Solution

Add a version-safe Back/previous contract through `WizardPrompter`, `WizardSession`, Gateway schema/handlers, generated clients, and onboarding clients, backed by a state/history model that preserves answers.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main shows the prompt, session, Gateway, docs, and macOS client surfaces expose only forward/cancel/status behavior.

## Solution Assessment

Unclear until maintainers settle the cross-surface API and UX contract. The maintainable direction is a typed Back/previous flow with preserved answers; adding only a prompt-level Back option would be incomplete.
