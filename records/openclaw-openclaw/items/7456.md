## Summary

Keep open: current `main` and `v2026.5.12` still expose onboarding as a forward/cancel wizard, the direct Back-navigation PR was closed unmerged, and related resume/dialog/macOS onboarding work does not ship preserved in-session previous-step navigation for `openclaw onboard`.

## What This Changes

The issue requests in-session Back navigation for `openclaw onboard`, preserving prior answers when moving backward and forward.

## Best Possible Solution

Add an additive typed previous/back contract through `WizardPrompter`, the Clack adapter, `WizardSession` history, Gateway wizard RPC, generated clients, and UI clients, preserving answers across backward and forward navigation.

## Reproduction Assessment

Yes, at source level: current `openclaw onboard` uses a sequential prompt flow and the session/protocol surfaces only support next, answer, cancel, and status. I did not run the live wizard because this review is read-only and onboarding can write config/state.

## Solution Assessment

Yes directionally, but it needs maintainer product approval and a cross-surface implementation. The safer solution is a shared typed Back/previous contract rather than a local `[Back]` option in only part of the wizard.
