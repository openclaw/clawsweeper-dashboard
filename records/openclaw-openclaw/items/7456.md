## Summary

Keep open. Current main still exposes onboarding as forward answer/cancel flow, and the related Back-navigation branch plus adjacent resume/dialog attempts are closed unmerged, so preserved in-session Back navigation remains unimplemented.

## What This Changes

Request to add Back navigation to `openclaw onboard` so users can return to earlier wizard steps with prior selections preserved.

## Best Possible Solution

Land an additive typed previous-step contract across wizard prompt/session/gateway/client surfaces, then refactor onboarding around preserved answer history for backward and forward movement.

## Reproduction Assessment

Yes. Source inspection on current main shows only forward answers and cancellation, with no Back sentinel, previous-step API, or preserved-answer history in the prompt, session, setup, or gateway protocol contracts.

## Solution Assessment

No shipped solution currently solves it. The maintainable path is a typed cross-surface previous-step contract rather than a local prompt-only patch that would leave gateway and UI wizard clients behind.
