## Summary

Current main and v2026.6.5 still provide only forward answer/next plus cancel/status onboarding flows. The useful direct implementation attempt at https://github.com/openclaw/openclaw/pull/7762 closed unmerged, no open canonical replacement was found, and a shared previous-step design still needs maintainer judgment.

## What This Changes

Add backward navigation to the onboarding wizard while preserving prior selections for forward restoration.

## Best Possible Solution

Approve one typed wizard history model with explicit downstream-answer invalidation, then implement it consistently across CLI prompts, setup flow, WizardSession, Gateway protocol, generated clients, app UIs, docs, and regression tests.

## Reproduction Assessment

Yes. Current main and v2026.6.5 consistently expose only answer-and-advance, cancel, and status behavior across the CLI, session, Gateway protocol, and macOS client surfaces.

## Solution Assessment

No, the suggested per-prompt `[b] Back` convention alone is not the best solution; the maintainable fix needs a shared navigation contract and defined handling for downstream answers that become stale.
