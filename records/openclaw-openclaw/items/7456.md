## Summary

Keep this issue open: current main still exposes the onboarding wizard as a forward/cancel flow, and the related implementation attempts are closed/unmerged or adjacent rather than shipped Back navigation.

## What This Changes

Add in-session Back navigation to the onboarding wizard with prior answers preserved when moving backward and forward.

## Best Possible Solution

Add an additive typed Back/previous contract across the wizard prompt API, session runner, Gateway protocol, and clients, then refactor setup into a history-aware flow that preserves answers.

## Reproduction Assessment

Yes, at source level: current main has only forward answer submission and cancellation paths for the onboarding wizard. I did not run the interactive wizard because this review is read-only, but the missing Back contract is visible across the prompt, session, gateway, and client code.

## Solution Assessment

Unclear until maintainers choose the product contract. The maintainable direction is an additive Back/previous protocol and a history-aware setup flow, not a local menu-only shortcut in one prompt adapter.
