## Summary

Keep open: current main and the latest release still expose the onboarding wizard as a forward/cancel flow, while the related Back-navigation PR was closed unmerged and no open canonical implementation replaces it.

## What This Changes

The issue requests in-session Back navigation for `openclaw onboard`, with prior selections preserved when moving backward and forward.

## Best Possible Solution

Add an additive typed Back/previous contract across `WizardPrompter`, Clack prompts, `WizardSession`, Gateway schema/handlers, and clients, then model setup flow state so answers are preserved safely across backward and forward navigation.

## Reproduction Assessment

Yes. Source inspection shows current `openclaw onboard` prompts and Gateway wizard sessions support forward answers or cancellation only, with no previous-step API or preserved-answer history.

## Solution Assessment

No, a local Clack-only Back menu would not be the best solution. The maintainable path is a typed Back/previous contract shared by CLI prompts, Gateway wizard RPC, and remote clients.
