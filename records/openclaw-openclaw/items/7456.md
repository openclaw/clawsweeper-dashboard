## Summary

Keep open: current main and v2026.5.22 still expose the shared onboarding wizard as forward/answer plus cancel/status only, and the direct Back-navigation PR was closed unmerged. The related resume and dialog PRs do not ship in-session previous-step navigation with preserved answers for `openclaw onboard`.

## What This Changes

The issue requests in-session Back navigation for `openclaw onboard`, preserving prior answers when users move backward and forward through the wizard.

## Best Possible Solution

Add a shared typed Back/previous contract and setup-flow history model across `WizardPrompter`, Clack prompts, `WizardSession`, Gateway schema/handlers, generated clients, and app clients after maintainers approve the API shape.

## Reproduction Assessment

Yes, at source level: current main exposes only forward `answer`/`next` and `cancel`/`status` paths, and the setup flow is still sequential. I did not run the live wizard because this read-only review should not write local config or wizard state.

## Solution Assessment

Yes directionally, but the exact API shape needs maintainer approval. The maintainable path is a shared Back/previous contract rather than a one-off local prompt option.
