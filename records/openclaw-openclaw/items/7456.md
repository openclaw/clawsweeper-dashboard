## Summary

Keep open: current main still exposes `openclaw onboard` as a forward/cancel wizard, and the attempted Back-navigation PR plus related resume/dialog work are closed unmerged and do not ship preserved in-session previous-step navigation.

## What This Changes

Add a Back/previous-step affordance to the onboarding wizard while preserving earlier answers when moving backward and forward.

## Best Possible Solution

Add an additive typed previous/back contract through the shared wizard prompter, session, Gateway protocol, generated clients, and UI clients, with answer preservation across backward and forward navigation.

## Reproduction Assessment

Yes, at source level: current `openclaw onboard` uses a sequential prompt flow and the session/protocol surfaces only support next, answer, cancel, and status. I did not run the live wizard because onboarding can write config/state in this read-only review.

## Solution Assessment

Yes directionally, but it needs maintainer product approval and a cross-surface implementation. The safer solution is a shared typed Back/previous contract rather than a local `[Back]` option in only part of the wizard.
