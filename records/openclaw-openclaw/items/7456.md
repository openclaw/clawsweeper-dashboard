## Summary

Keep open. Current main still exposes onboarding as a forward/cancel flow, and the direct Back-navigation PR was closed unmerged while the other related PRs cover different interaction models.

## What This Changes

Add in-session Back navigation to the onboarding wizard with previous answers preserved when moving backward and forward.

## Best Possible Solution

Ship an additive typed Back/previous wizard contract with answer preservation across CLI prompts, WizardSession, Gateway RPC/schema, generated clients, docs, and onboarding clients.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: advance through openclaw onboard or the shared wizard RPC flow and there is no Back option, previous-step result, or wizard.previous call to invoke.

## Solution Assessment

No current implementation is ready. The maintainable solution is a coordinated typed Back contract and history model; reviving #7762 may help, but a Clack-only or prompt-only patch would be incomplete.
