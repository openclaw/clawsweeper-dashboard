## Summary

Keep open: current main and the latest release still expose onboarding as a forward/cancel wizard with no shared Back or previous-step contract, while the direct implementation attempt is closed unmerged.

## What This Changes

The issue requests in-session Back navigation for `openclaw onboard`, preserving prior answers while moving backward and forward through the wizard.

## Best Possible Solution

Add an explicit typed Back/previous-step contract across the shared wizard interface, terminal adapter, Gateway RPC, clients, and setup state model after maintainer UX/API approval.

## Reproduction Assessment

Yes. Source inspection on current main and the latest release shows the prompt, session, setup flow, Gateway schema, method table, and docs only support forward advancement plus cancel/status.

## Solution Assessment

Yes, a shared typed Back/previous contract is the maintainable direction. A Clack-only Back option would be too narrow because Gateway and UI wizard clients need the same answer-preservation semantics.
