## Summary

Keep this issue open: current main still exposes the onboarding wizard as a forward/cancel flow, and the directly related Back-navigation PR is closed unmerged. The request is aligned with setup reliability and first-run UX, but it needs a deliberate wizard API/product design across CLI, Gateway RPC, and clients rather than an autonomous narrow bug fix.

## What This Changes

The issue requests in-session Back navigation for the `openclaw onboard` wizard with preserved previous answers.

## Best Possible Solution

Add an additive typed Back/previous contract through the wizard prompt/session and Gateway wizard surfaces, then make setup history-aware so preserved answers are restored consistently across CLI and remote clients.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: current `openclaw onboard` and Gateway wizard clients can advance or cancel, but the prompt/session/protocol contracts do not expose any Back operation.

## Solution Assessment

Yes for the direction, but not as a local-only Clack menu. The maintainable fix is a typed Back/previous contract shared by the CLI wizard, `WizardSession`, Gateway schema/handlers, generated clients, and UI clients so answer preservation is consistent.
