## Summary

Keep open. Current main still exposes onboarding as a forward/cancel wizard with no shared Back or previous-step contract, and the direct Back-navigation PR was closed unmerged while related resume/dialog work covers different workflows.

## What This Changes

The issue requests Back navigation in the onboarding wizard with prior selections preserved.

## Best Possible Solution

Ship an additive typed Back/previous contract and history model across wizard prompts, `WizardSession`, Gateway schema/handlers, docs, generated clients, and onboarding clients, preserving answers when moving backward and forward.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: the wizard prompt and RPC contracts have no Back result or previous method, so onboarding users can advance or cancel but cannot rewind with preserved answers.

## Solution Assessment

Unclear until maintainers settle the API and UX contract. A coordinated typed Back/previous contract plus state/history model is the maintainable direction; adding only a prompt-level `[b] Back` option would be incomplete.
