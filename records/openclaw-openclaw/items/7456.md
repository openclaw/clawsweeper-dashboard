## Summary

Keep open. Current main still exposes onboarding as a forward/cancel wizard with no shared Back/previous-step contract, and the direct implementation PR #7762 was closed unmerged while #5880 and #18286 cover different workflows.

## What This Changes

Add preserved Back/previous-step navigation to the `openclaw onboard` wizard so users can revise earlier onboarding choices without restarting.

## Best Possible Solution

Ship an additive typed Back/previous contract and history model across wizard prompts, `WizardSession`, Gateway schema/handlers, generated clients, docs, and onboarding clients, preserving answers when moving backward and forward.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: the wizard prompt and RPC contracts have no Back result or previous method, so an `openclaw onboard` or wizard RPC user can only advance or cancel.

## Solution Assessment

Unclear until maintainers settle the API and UX contract. A coordinated typed contract plus state-machine/history model is the maintainable direction; a prompt-only `[b] Back` option would be incomplete.
