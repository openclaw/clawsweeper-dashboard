## Summary

Keep open. Current main still exposes onboarding as a forward/cancel wizard with no shared previous-step contract, and the direct implementation attempt is closed unmerged, so this remains valid feature/API work rather than conservative cleanup.

## What This Changes

Add in-session Back navigation to `openclaw onboard` with preserved prior selections when moving backward and forward.

## Best Possible Solution

Add an additive typed Back/previous contract across prompt, session, Gateway protocol/handlers, generated clients, and onboarding clients, backed by a history model that preserves safe answers.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Source inspection on current main shows forward answer, cancel, and status paths but no previous-step path.

## Solution Assessment

Unclear until maintainers settle the cross-surface API and UX contract. The maintainable direction is a typed Back/previous flow with preserved answers; adding a local CLI-only prompt option would not cover Gateway/client wizard surfaces.
