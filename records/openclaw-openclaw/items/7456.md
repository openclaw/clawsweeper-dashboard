## Summary

Keep open: current main and the latest release still expose onboarding as a forward/cancel flow, the direct back-navigation implementation attempt is closed unmerged, and the related resume/dialog work does not provide preserved in-session previous-step navigation.

## What This Changes

The issue requests in-session Back navigation for `openclaw onboard`, preserving prior answers when users move backward and forward.

## Best Possible Solution

Add an additive typed previous/back contract through `WizardPrompter`, the Clack adapter, `WizardSession` history, Gateway wizard RPC, generated clients, and UI clients, with preserved answers across backward and forward navigation.

## Reproduction Assessment

Yes, at source level: current `openclaw onboard` is implemented as a sequential prompt flow and the session/protocol surface only supports next, answer, cancel, and status. A live CLI run was not used because this read-only review must not write onboarding state.

## Solution Assessment

Yes directionally, but it needs maintainer product approval and a cross-surface implementation. The safer solution is an additive shared back/previous contract, not a one-off `[Back]` menu entry inside only part of the wizard.
