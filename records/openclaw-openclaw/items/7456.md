## Summary

Keep open. Current main still exposes the onboarding wizard as forward/cancel only: the prompt API, Clack adapter, WizardSession, setup flow, and Gateway wizard protocol do not provide an in-session Back/previous contract with preserved answers. The related implementation attempt is closed unmerged, and the linked resume/dialog work does not ship this requested behavior.

## What This Changes

Add in-session Back navigation to `openclaw onboard` so users can return to earlier wizard steps with prior selections preserved.

## Best Possible Solution

Land an additive typed Back/previous contract through WizardPrompter, Clack prompts, WizardSession, Gateway schema/handlers, generated clients, and UI clients, then refactor runSetupWizard around preserved answer history.

## Reproduction Assessment

Yes. Source inspection on current main shows forward answers and cancellation only, with no previous-step API, Back sentinel, or preserved-answer history in the prompt/session/protocol contracts.

## Solution Assessment

No shipped solution currently solves this issue. The closed Back-navigation branch is useful prior art, but the maintainable path is a typed cross-surface previous-step contract rather than a local prompt-only patch.
