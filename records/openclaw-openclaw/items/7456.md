## Summary

Keep this open. Current main still has only forward answers and cancellation for the onboarding wizard; the linked implementation PR was closed unmerged, and the other related PRs do not ship preserved in-session previous-step navigation.

## What This Changes

The issue requests a Back option during `openclaw onboard` so users can return to earlier wizard steps while preserving prior selections.

## Best Possible Solution

Add an additive, typed previous-step contract across the prompt API, Clack adapter, `WizardSession`, Gateway wizard schema/handlers, generated clients, and UI clients, backed by a setup-flow history model that preserves answers while moving backward and forward.

## Reproduction Assessment

Yes, at source level: the current `openclaw onboard` path has no Back sentinel, no `previous()` session API, no gateway method for previous navigation, and no setup-flow state machine to preserve answers. I did not run the interactive wizard because this review is read-only, but the missing contract is clear from current main.

## Solution Assessment

Yes for the requested direction, but it needs maintainer product/API design before implementation. The best fix is not a small local prompt menu; it should be a typed cross-surface Back/previous contract with preserved answer state and client follow-through.
