## Summary

Keep open: current main and the latest release still expose the onboarding wizard as forward/cancel only, and the direct Back-navigation PR is closed unmerged with no superseding shipped implementation.

## What This Changes

The issue requests in-session Back navigation for `openclaw onboard`, with previous answers preserved when moving backward and forward.

## Best Possible Solution

Add an additive typed previous/back contract through the shared wizard prompter, session, Gateway protocol, generated clients, and UI clients, with answer preservation across backward and forward navigation.

## Reproduction Assessment

Yes, at source level: current `openclaw onboard` uses a sequential prompt flow and the shared session/protocol surfaces only support next, answer, cancel, and status. I did not run the live wizard because onboarding can write config/state in this read-only review.

## Solution Assessment

Yes directionally, but it needs maintainer product approval and a cross-surface implementation. The safer solution is a shared typed Back/previous contract rather than a one-off `[Back]` option in only part of the wizard.
