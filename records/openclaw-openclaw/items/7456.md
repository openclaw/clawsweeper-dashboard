## Summary

Keep open: current main and the latest shipped tag still expose onboarding wizard navigation as forward/cancel/status only, and the direct Back-navigation implementation was closed unmerged with no superseding shipped fix.

## What This Changes

Request to add Back navigation to `openclaw onboard` so users can revisit previous onboarding steps while preserving prior answers.

## Best Possible Solution

Add an additive typed previous/back contract through `WizardPrompter`, Clack prompts, `WizardSession`, Gateway schema/handlers, generated clients, and UI clients, with answer preservation across backward and forward navigation.

## Reproduction Assessment

Yes, at source level: current `openclaw onboard` uses a sequential prompt flow and the shared session/protocol/client surfaces only support next, answer, cancel, and status. I did not run the live wizard because onboarding can write config/state in this read-only review.

## Solution Assessment

Yes directionally, but it needs maintainer product approval and a cross-surface implementation. The safer solution is a shared typed Back/previous contract rather than a one-off `[Back]` option in only part of the wizard.
