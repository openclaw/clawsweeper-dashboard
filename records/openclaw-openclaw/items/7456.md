## Summary

Keep open: current main and the latest release still expose onboarding as a forward/cancel wizard with no shared Back or previous-step contract, and the direct Back-navigation implementation attempt is closed unmerged.

## What This Changes

The issue requests in-session Back navigation for `openclaw onboard`, preserving prior answers when users move backward and forward.

## Best Possible Solution

Keep this issue open and implement a typed Back/previous contract across `WizardPrompter`, Clack prompts, `WizardSession`, Gateway wizard schema/handlers, generated clients, and onboarding state preservation after maintainer API/UX approval.

## Reproduction Assessment

Yes. Source inspection on current main shows the prompt, session, setup flow, Gateway method list, and docs only support forward advancement or cancel/status, with no Back or previous-step path.

## Solution Assessment

Yes, a shared typed Back/previous contract is the right direction. A Clack-only Back option would be too narrow because Gateway clients and preserved-answer state need the same semantics.
