## Summary

Keep open. Current main still lacks an in-session Back/previous contract for onboarding, and the linked implementation attempt is closed unmerged while related resume/dialog work does not provide preserved Back navigation for `openclaw onboard`.

## What This Changes

The issue requests in-session Back navigation for the `openclaw onboard` wizard with prior answers preserved when moving backward and forward.

## Best Possible Solution

Add a version-safe Back/previous contract through `WizardPrompter`, `WizardSession`, Gateway schema/handlers, generated clients, and onboarding clients, backed by a state/history model that preserves answers.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main shows the prompt, session, Gateway, docs, and macOS client surfaces expose only forward/cancel/status behavior.

## Solution Assessment

Unclear until maintainers settle the cross-surface API and UX contract. The maintainable direction is a typed Back/previous flow with preserved answers; adding only a local prompt option would be incomplete.
