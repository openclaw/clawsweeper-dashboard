## Summary

Keep open. Current main still does not provide in-session Back navigation for `openclaw onboard`; the CLI prompt contract, session model, setup flow, Gateway protocol, and advertised methods remain forward-answer/cancel oriented, and the direct implementation attempt was closed unmerged.

## What This Changes

The issue requests a Back option in the onboarding wizard that preserves prior selections while moving backward and forward through setup steps.

## Best Possible Solution

Land an additive typed previous-step contract across WizardPrompter, Clack, WizardSession, Gateway protocol/handlers, generated clients, and UI clients, with preserved answer history and regression proof.

## Reproduction Assessment

Yes. Source inspection on current main shows only forward answers and cancellation, with no Back sentinel, previous-step API, or preserved-answer history in the prompt, session, setup, Gateway, or advertised method contracts.

## Solution Assessment

Yes on the requested user outcome, but the maintainable solution is a typed cross-surface wizard contract rather than an isolated Clack menu option that leaves Gateway and app clients behind.
