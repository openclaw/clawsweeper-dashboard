## Summary

Keep open. Current main still exposes onboarding as a forward-answer/cancel flow, and the related Back-navigation branch plus the resume/dialog attempts are closed unmerged, so the requested in-session Back navigation is not implemented.

## What This Changes

Add Back navigation to `openclaw onboard` so users can return to previous wizard steps with prior answers preserved.

## Best Possible Solution

Land an additive typed Back/previous contract across `WizardPrompter`, Clack prompts, `WizardSession`, Gateway schema/handlers, generated clients, and UI clients, then refactor setup around preserved answer history.

## Reproduction Assessment

Yes. Source inspection on current main shows forward answers and cancellation only, with no Back sentinel, previous-step API, or preserved-answer history in prompt/session/protocol contracts.

## Solution Assessment

No shipped solution currently solves it. The maintainable path is a typed cross-surface previous-step contract rather than a local prompt-only patch.
