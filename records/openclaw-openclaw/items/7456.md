## Summary

Keep this open. Current main still exposes the onboarding wizard as a forward/cancel flow, and the remaining work is a cross-surface feature/product decision rather than a narrow bug fix.

## What This Changes

Add a Back option to the onboarding wizard so users can return to previous steps while preserving earlier selections.

## Best Possible Solution

Define an additive typed previous-step contract across `WizardPrompter`, the Clack adapter, `WizardSession`, gateway wizard schema/handlers, generated clients, and the setup flow state model so CLI and clients preserve and revisit answers consistently.

## Reproduction Assessment

Yes, from source inspection: current main has only forward answer plus cancel/status paths, and the setup flow remains sequential without a previous-step state path. I did not run the interactive wizard because this read-only review avoided commands that could write checkout or user-state artifacts.

## Solution Assessment

Unclear until maintainers choose the UX/protocol contract. The best direction is a typed Back/previous contract with preserved answers, not a local-only Clack menu item.
