## Summary

Keep open: current main still lacks shared in-session Back navigation for `openclaw onboard`, and the related implementation attempt is closed unmerged while adjacent resume/dialog/macOS onboarding work does not provide preserved CLI wizard rewind behavior.

## What This Changes

The issue requests a Back option in the onboarding wizard that returns to earlier steps while preserving previously entered selections.

## Best Possible Solution

Add an additive typed previous/back contract through `WizardPrompter`, the Clack adapter, `WizardSession` history, Gateway wizard RPC, generated clients, and UI clients, with answer preservation across backward and forward navigation.

## Reproduction Assessment

Yes, at source level: current `openclaw onboard` is implemented as a sequential prompt flow and the session/protocol surface only supports next, answer, cancel, and status. A live CLI run was not used because onboarding can write config/state in this read-only review.

## Solution Assessment

Yes directionally, but the best implementation is broader than adding one menu option. The maintainable path is a shared typed previous/back contract with preserved answers across CLI and Gateway clients.
