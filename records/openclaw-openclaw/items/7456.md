## Summary

Keep open. Current `main` still has a forward/cancel onboarding wizard: the prompt contract has no Back result, the Clack adapter only maps cancellation, `WizardSession` has no previous/history API, and the Gateway wizard contract exposes only start/next/cancel/status. The related implementation attempt at https://github.com/openclaw/openclaw/pull/7762 is closed unmerged, and the resume/dialog related PRs do not ship preserved in-session Back navigation.

## What This Changes

The issue requests in-session Back navigation for `openclaw onboard`, preserving prior answers while moving backward and forward through wizard steps.

## Best Possible Solution

Land an additive typed Back/previous contract through `WizardPrompter`, Clack prompts, `WizardSession`, Gateway wizard schema/handlers, generated clients, and UI clients, then refactor `runSetupWizard` around a history model that preserves answers when moving backward and forward.

## Reproduction Assessment

Yes, from source inspection: current `main` exposes forward answers and cancellation only, with no previous-step API or preserved-answer history. I did not run an interactive terminal repro because this read-only review already has decisive contract evidence.

## Solution Assessment

No shipped solution currently solves the request. The best direction is not to reopen the closed branch as-is, but to use it as prior art while designing the typed cross-surface previous-step contract and state/history flow.
