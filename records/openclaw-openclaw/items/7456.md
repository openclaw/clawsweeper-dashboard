## Summary

Keep this issue open: current main and the latest release still expose onboarding as forward-or-cancel, and the closed related PRs did not land the requested in-session Back navigation.

## What This Changes

The issue requests in-session Back navigation for `openclaw onboard`, preserving prior wizard answers when moving backward and forward.

## Best Possible Solution

Add an additive typed Back/previous contract through the prompt API, Clack adapter, WizardSession, Gateway schema/handlers, clients, and setup-state model so navigation consistently preserves answers.

## Reproduction Assessment

Yes. Source inspection is enough to reproduce the missing capability: current prompt, session, and Gateway contracts support forward answers and cancellation, but not previous-step navigation.

## Solution Assessment

Yes. The best fix is a shared typed Back/previous contract across CLI and Gateway wizard surfaces; PR https://github.com/openclaw/openclaw/pull/7762 is useful prior art but closed unmerged and not sufficient as shipped evidence.
