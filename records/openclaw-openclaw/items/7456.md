## Summary

Keep open: current main and the latest release still expose onboarding as a forward/cancel wizard with no typed Back or previous-step contract, and the direct Back-navigation PR was closed unmerged.

## What This Changes

Request to add in-session Back navigation to the onboarding wizard while preserving prior answers.

## Best Possible Solution

Add an additive typed Back/previous contract across CLI prompts, WizardSession, Gateway schema/handlers, and clients, then model setup flow history so answers are preserved safely across backward and forward navigation.

## Reproduction Assessment

Yes. Source inspection shows current openclaw onboard prompt, setup, session, and Gateway wizard contracts support forward answers or cancellation only, with no previous-step API or preserved-answer history.

## Solution Assessment

Unclear until maintainers choose the product/API contract, but the best code shape is a shared typed Back/previous contract rather than a local Clack-only menu.
