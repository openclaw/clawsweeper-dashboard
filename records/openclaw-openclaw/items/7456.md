## Summary

Keep this issue open. Current main still exposes the classic onboarding wizard as a forward/cancel flow, and the related Back-navigation implementation PR is closed unmerged rather than shipped.

## What This Changes

The issue requests in-session Back navigation for `openclaw onboard`, preserving prior answers when users move backward and forward through onboarding steps.

## Best Possible Solution

Add an additive, typed Back/previous contract through `WizardPrompter`, the Clack adapter, `WizardSession`, Gateway wizard schemas/handlers, generated clients, and onboarding clients, then refactor the setup flow around a preserved-answer history model with focused regression coverage.

## Reproduction Assessment

Yes. A source-backed reproduction path is high-confidence: start the classic onboarding flow or inspect the Gateway wizard RPC path, advance past a prompt, and there is no Back option or `wizard.previous`/rewind method to call.

## Solution Assessment

No shipped solution exists yet. The maintainable path is a coordinated typed contract and history model; reviving #7762 may help, but it should be revised rather than treated as the current best fix as-is.
