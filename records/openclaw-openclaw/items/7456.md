## Summary

Keep this issue open. Current `main` still has a forward/cancel onboarding wizard, the prior Back-navigation implementation was closed unmerged, and a proper fix needs a maintainer-approved CLI/Gateway wizard contract rather than a local prompt-only patch.

## What This Changes

The issue asks for in-session Back navigation in `openclaw onboard` that preserves and restores prior wizard answers.

## Best Possible Solution

Add an additive typed previous-step contract across `WizardPrompter`, Clack prompts, `WizardSession`, Gateway schema/handlers, generated protocol clients, and setup-flow state so answers survive backward and forward navigation.

## Reproduction Assessment

Yes, at source level: current `main` exposes sequential prompt calls plus `next`/`answer`/`cancel` wizard session APIs, with no Back or previous-step path. I did not run a live interactive CLI session because the missing contract is directly visible in current source and docs.

## Solution Assessment

Partly. A Back UX is the right user-facing direction, but the best fix is a shared typed previous-step contract across CLI and Gateway wizard clients, not only adding a local Clack menu item.
