## Summary

Keep this issue open: current main and the latest release still expose onboarding as a forward/cancel wizard, and the closed related attempts did not ship the requested preserved-answer Back navigation.

## What This Changes

The issue requests in-session Back navigation for `openclaw onboard`, with previous selections preserved when moving backward and forward.

## Best Possible Solution

Add an additive typed Back/previous contract through `WizardPrompter`, Clack prompts, `WizardSession`, Gateway schema/handlers, generated clients, and UI clients, then refactor setup into a history-aware flow that preserves answers.

## Reproduction Assessment

Yes, at source level: current main has no Back/previous prompt result, no WizardSession rewind API, and no Gateway wizard back method. This is a missing feature rather than a broken existing contract.

## Solution Assessment

Yes, the maintainable solution is a shared typed Back/previous contract across CLI and Gateway wizard surfaces, not a local Clack-only menu item. Maintainers still need to confirm the public UX/API shape before implementation.
