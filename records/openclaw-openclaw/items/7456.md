## Summary

Keep open. Current main still exposes onboarding as a forward/cancel wizard, and the related implementation attempts are closed or cover different workflows.

## What This Changes

Request adding in-session Back navigation to `openclaw onboard` with prior selections preserved when moving between steps.

## Best Possible Solution

Add a version-safe Back/previous flow through `WizardPrompter`, `WizardSession`, Gateway schema/handlers, generated clients, and onboarding clients, with regression coverage for preserved answers.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main confirms the requested Back/previous path is absent from the prompt, session, Gateway, and docs contracts.

## Solution Assessment

Unclear until maintainers settle the API and UX contract. A coordinated typed Back/previous contract plus state/history model is the maintainable direction; adding only a prompt-level Back escape hatch would be incomplete.
