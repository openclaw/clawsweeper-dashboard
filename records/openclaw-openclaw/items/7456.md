## Summary

Keep open. Current main still lacks preserved in-session Back navigation for `openclaw onboard`; the prompt, session, setup, and Gateway wizard contracts are still forward-answer/cancel oriented, and the attempted implementation in https://github.com/openclaw/openclaw/pull/7762 was closed unmerged.

## What This Changes

Add Back/previous-step navigation to the onboarding wizard while preserving prior selections.

## Best Possible Solution

Land an additive typed previous-step contract across WizardPrompter, Clack, WizardSession, Gateway protocol/handlers, generated clients, and UI clients, with preserved answer history and regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows only forward answers and cancellation, with no Back sentinel, previous-step API, or preserved-answer history in the prompt, session, setup, or gateway protocol contracts.

## Solution Assessment

Yes on the requested user outcome, but it should be solved as a typed cross-surface wizard contract rather than an isolated Clack menu option that leaves Gateway and app clients behind.
