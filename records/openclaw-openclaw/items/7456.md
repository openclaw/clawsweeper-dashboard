## Summary

Keep this open. Current main and the latest release still expose onboarding as forward/cancel only, and the direct implementation attempt was closed unmerged, so the central user problem remains valid.

## What This Changes

The issue requests in-session Back navigation for `openclaw onboard`, preserving prior answers when moving backward and forward.

## Best Possible Solution

Add one typed Back/previous contract across the wizard prompter, Clack adapter, `WizardSession`, Gateway schema/handlers, generated protocol clients, and setup-flow state model, with preserved answers and explicit downstream invalidation rules.

## Reproduction Assessment

Yes for the missing capability: current source shows forward answer submission and cancellation only, with no shared Back/previous API in prompt, session, Gateway schema, or docs.

## Solution Assessment

Yes on the requested direction, but the best implementation needs maintainer decisions about rewind scope and side effects before a fix PR should be delegated.
