## What This Changes

Request to add in-session Back navigation to the onboarding wizard while preserving prior selections.

## Best Possible Solution

Add an additive typed Back/previous contract across `WizardPrompter`, Clack prompts, `WizardSession`, Gateway schema/handlers, clients, and setup-flow state so backward and forward navigation preserve answers consistently.

## Reproduction Assessment

Yes. Source inspection shows the prompt contract, Clack prompter, `WizardSession`, and Gateway wizard methods expose forward answers plus cancel/status only; I did not run the interactive wizard because this review is read-only.

## Solution Assessment

No current implementation solves it yet. The best path is a cross-surface wizard navigation contract, not a prompt-local Back menu that would leave Gateway and GUI clients behind.
