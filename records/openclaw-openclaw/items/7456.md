## Summary

Keep open. Current main still lacks an in-session Back/previous contract for `openclaw onboard`, and the related implementation attempt is closed unmerged while resume/dialog-related work does not ship preserved Back navigation for the onboarding wizard.

## What This Changes

Add in-session Back navigation to `openclaw onboard`, preserving earlier selections and restoring them when moving forward again.

## Best Possible Solution

Add a version-safe Back/previous contract through the wizard prompt types, Clack adapter, `WizardSession`, Gateway schema/handlers, generated clients, and onboarding clients, backed by a state/history model that preserves answers.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Source inspection on current main shows only forward/cancel/status wizard paths, which confirms the requested capability is still absent.

## Solution Assessment

Unclear until maintainers settle the cross-surface API and UX contract. The maintainable direction is a typed Back/previous flow with preserved answers; adding only a local prompt option would not solve the Gateway/client surfaces.
