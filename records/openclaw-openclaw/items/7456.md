## Summary

Keep open; current main and the latest release still expose onboarding as a forward/cancel wizard, and the closed Back-navigation PR remains useful prior art rather than a shipped fix.

## What This Changes

The issue asks for an in-session Back option in `openclaw onboard` that preserves prior answers while users move backward and forward.

## Best Possible Solution

Add a maintainer-approved typed previous-step contract across `WizardPrompter`, Clack prompts, `WizardSession`, Gateway wizard RPC/schema, clients, and regression coverage.

## Reproduction Assessment

Yes at source level: current main has no Back sentinel, previous-step API, or wizard RPC for rewinding, so the reported limitation is clear without running the interactive CLI. I did not run live onboarding because this review is read-only.

## Solution Assessment

Unclear as an implementation until maintainers approve the cross-surface UX contract. The requested direction is aligned, but a narrow local prompt hack would not be the best solution because Gateway clients share the same wizard flow.
