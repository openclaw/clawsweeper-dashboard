## Summary

Keep open: current main and the latest release still expose onboarding as forward answer/next plus cancel/status only. The direct Back-navigation attempt at https://github.com/openclaw/openclaw/pull/7762 was closed unmerged, while the related resume and dialog efforts do not provide in-session previous-step navigation with preserved answers for `openclaw onboard`.

## What This Changes

Request adding Back navigation to the `openclaw onboard` wizard with preserved prior selections and forward restoration.

## Best Possible Solution

Add a shared typed Back/previous contract and setup-flow history model across `WizardPrompter`, Clack prompts, `WizardSession`, Gateway schema/handlers, generated clients, and app clients after maintainer API approval.

## Reproduction Assessment

Yes, at source level: current main exposes only forward `answer`/`next` and `cancel`/`status` paths, and the setup flow remains sequential. I did not run the live wizard because this read-only review should not write local config or wizard state.

## Solution Assessment

Yes directionally, but the exact API shape needs maintainer approval. The maintainable solution is a shared Back/previous contract rather than a one-off prompt option in only the CLI path.
