## What This Changes

Requests a Back/previous-step navigation mode in `openclaw onboard` that preserves earlier wizard answers.

## Best Possible Solution

Implement an additive typed Back/previous contract across `WizardPrompter`, Clack prompts, `WizardSession`, Gateway wizard schema/handlers, generated clients, and onboarding state preservation after maintainer UX/API approval.

## Reproduction Assessment

Yes. Source inspection on current main shows the prompt, Clack adapter, WizardSession, Gateway method list, and docs only support forward advancement or cancel/status, with no Back or previous-step path.

## Solution Assessment

Yes, if implemented as a shared typed Back/previous contract. A Clack-only menu option would be too narrow because Gateway and app clients need the same state semantics.
