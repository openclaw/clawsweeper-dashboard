## Summary

Keep open. Current main still exposes the onboarding wizard as a forward/cancel flow, while the related Back-navigation implementation was closed unmerged and the related resume/dialog work does not provide preserved in-session previous-step navigation.

## What This Changes

The issue requests Back navigation in `openclaw onboard` so users can return to earlier onboarding steps with prior answers preserved.

## Best Possible Solution

Add a version-safe Back/previous contract through wizard prompt types, the Clack adapter, `WizardSession`, Gateway schema/handlers, generated clients, and onboarding clients, backed by a state/history model that preserves answers.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Source inspection on current main gives high confidence that the wizard exposes only forward answer, cancel, and status paths.

## Solution Assessment

Unclear until maintainers settle the cross-surface API and UX contract. The maintainable direction is a typed Back/previous flow with preserved answers; adding only a local prompt option would not solve the Gateway/client surfaces.
