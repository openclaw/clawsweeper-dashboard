## Summary

Keep open. Current main still exposes classic onboarding as a forward/cancel wizard with no shared Back or previous-step contract, and the direct implementation attempt was closed unmerged.

## What This Changes

Request to add in-wizard Back navigation to `openclaw onboard`, preserving prior answers when users move backward and forward.

## Best Possible Solution

Ship an additive typed Back/previous contract plus a history model across wizard prompts, `WizardSession`, Gateway schema/handlers, docs, generated clients, and onboarding clients, preserving answers across rewind and replay.

## Reproduction Assessment

Yes. Source inspection on current main shows the wizard prompt and RPC contracts have no Back result or previous method, so users can advance or cancel but cannot rewind with preserved answers.

## Solution Assessment

Unclear until maintainers settle the API and UX contract. The maintainable direction is a coordinated typed Back/previous contract plus state/history model; adding only a local Clack menu option would be incomplete.
