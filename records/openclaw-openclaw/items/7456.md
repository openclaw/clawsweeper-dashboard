## Summary

Keep open: current main and the latest release still expose the onboarding wizard as forward-only answer/next plus cancel/status. The direct Back-navigation PR was closed unmerged, and related resume, dialog, and macOS onboarding work does not implement shared previous-step navigation with preserved answers for `openclaw onboard`.

## What This Changes

Request adding in-session Back navigation to `openclaw onboard`, preserving prior answers when users move backward and forward through the wizard.

## Best Possible Solution

Define an additive typed Back/previous contract across the prompter, session, Gateway schema/handlers, generated clients, and app clients, then refactor setup into a history-preserving state flow with regression coverage.

## Reproduction Assessment

Yes, at source level: current main exposes only forward `answer`/`next` and `cancel`/`status` paths, and the setup flow remains sequential. I did not run the live wizard because this read-only review should not write local config or wizard state.

## Solution Assessment

Yes directionally, but the exact API shape needs maintainer approval. The maintainable solution is a shared Back/previous contract rather than a one-off prompt option in only the CLI path.
