Request to add in-session Back navigation to `openclaw onboard` with previously entered answers preserved when moving backward and forward.

Introduce a shared typed Back/previous contract and setup-flow history model across the wizard prompter, session, Gateway protocol, generated clients, and UI clients, with answer preservation covered by tests.

## Reproduction Assessment

Yes, at source level: current main exposes only forward answer/next and cancel/status paths for onboarding, and the setup flow is still sequential. I did not run the live wizard because this read-only review should not write local config or wizard state.

## Solution Assessment

Yes directionally, but it needs maintainer product/protocol approval before implementation. The maintainable path is a shared Back/previous contract rather than a one-off local prompt option.
