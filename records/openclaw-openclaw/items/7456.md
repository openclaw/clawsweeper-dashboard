## Summary

Current main and v2026.6.5 remain forward/cancel-only, while the closed implementation attempts do not provide a viable canonical replacement. This issue should remain open for a maintainer product/architecture decision because the requested first-run UX is vision-aligned but spans preserved state, reversible step boundaries, Gateway protocol, and client behavior.

## What This Changes

Add backward navigation to the onboarding wizard so users can revisit earlier steps with previous selections preserved.

## Best Possible Solution

Keep one canonical typed wizard navigation model that preserves draft answers in memory, permits Back only across explicitly reversible stages, invalidates dependent later answers when necessary, and coordinates any Gateway protocol and client changes without replaying committed side effects.

## Reproduction Assessment

Yes. Current main and v2026.6.5 expose only forward advancement and cancellation at the prompt, session, and Gateway boundaries, so the requested Back flow is source-reproducibly unavailable.

## Solution Assessment

The user outcome is appropriate, but the suggested universal Back option is not yet a complete design; maintainers should first define reversible stages, dependent-answer invalidation, secret retention, and whether the contract is CLI-only or shared with Gateway clients.
