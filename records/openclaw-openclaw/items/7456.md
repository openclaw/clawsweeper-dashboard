## Summary

Keep open. Current main still lacks preserved in-session Back navigation for `openclaw onboard`; the wizard prompt, session, setup, Gateway protocol, and macOS client paths are still forward-answer/cancel oriented, and the direct implementation attempt at https://github.com/openclaw/openclaw/pull/7762 was closed unmerged.

## What This Changes

Add Back navigation to `openclaw onboard` so users can return to previous wizard steps with prior selections preserved.

## Best Possible Solution

Land an additive typed previous-step contract across WizardPrompter, Clack, WizardSession, Gateway protocol/handlers, generated clients, and UI clients with preserved answer history and regression proof.

## Reproduction Assessment

Yes. Source inspection on current main shows only forward answers and cancellation, with no Back sentinel, previous-step API, or preserved-answer history in the prompt, session, setup, Gateway, or client contracts.

## Solution Assessment

Yes on the requested user outcome, but the maintainable solution is a typed cross-surface wizard contract rather than an isolated Clack menu option that leaves Gateway and app clients behind.
