## Summary

Keep open: current main and the latest release still expose the classic onboarding wizard as a forward/cancel flow, while the direct Back-navigation implementation was closed unmerged and no related item now owns the exact in-session rewind-with-preserved-answers behavior.

## What This Changes

The issue requests in-session Back navigation for the classic `openclaw onboard` wizard with prior answers preserved.

## Best Possible Solution

Add a typed, additive previous-step contract through `WizardPrompter`, Clack prompts, `WizardSession`, Gateway schema/handlers, and clients, with answer preservation and regression coverage.

## Reproduction Assessment

Yes. Source inspection shows the current prompt contract, Clack adapter, `WizardSession`, and Gateway wizard protocol only support forward answers, status, and cancel, with no previous/back operation.

## Solution Assessment

Unclear until maintainers choose the API shape. The narrow maintainable direction is an additive shared Back/previous contract across CLI and Gateway, not a local menu-only workaround.
