## Summary

Keep open: current main and v2026.5.22 still expose onboarding as forward/cancel only, the direct Back-navigation implementation at https://github.com/openclaw/openclaw/pull/7762 was closed unmerged, and the related resume/dialog/macOS work does not add the shared core Back/previous contract.

## What This Changes

The issue requests in-session Back navigation for `openclaw onboard`, with prior selections preserved when moving backward and forward.

## Best Possible Solution

Add a shared typed Back/previous contract and setup-flow history model across `WizardPrompter`, Clack prompts, `WizardSession`, Gateway schema/handlers, generated clients, and UI clients.

## Reproduction Assessment

Yes, at source level: current main exposes only forward answer/next and cancel/status paths, and the setup flow remains sequential. I did not run the live wizard because this read-only review should not write local config or wizard state.

## Solution Assessment

Yes directionally, but the exact API/product shape needs maintainer approval. The maintainable path is a shared Back/previous contract rather than a one-off local prompt option.
