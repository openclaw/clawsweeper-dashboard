## Summary

Keep open: current main and the latest tagged release still expose onboarding as forward answer/next plus cancel/status only, while the direct Back-navigation PR was closed unmerged.

## What This Changes

The issue requests in-session Back navigation for `openclaw onboard`, preserving prior answers when moving backward and forward.

## Best Possible Solution

Add a shared typed Back/previous contract and setup-flow history model across the prompter, Clack adapter, WizardSession, Gateway schema/handlers, generated clients, and app clients after maintainer API approval.

## Reproduction Assessment

Yes, at source level: current main exposes only forward `answer`/`next` and `cancel`/`status` paths, and the setup flow remains sequential. I did not run the live wizard because this read-only review should not write local config or wizard state.

## Solution Assessment

Yes directionally, but the exact API shape needs maintainer approval. The maintainable solution is a shared Back/previous contract rather than a one-off prompt option in only the CLI path.
