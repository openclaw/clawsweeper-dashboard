## Summary

The requested in-session backward navigation is still absent on current main and in the latest release. The closed related pull requests are useful prior art but neither merged nor provides a viable canonical implementation, while VISION.md explicitly prioritizes setup reliability and first-run UX; the issue should remain open for a maintainer product decision on rewind scope and side-effect semantics.

## What This Changes

Add backward navigation to onboarding so users can revisit prior steps with earlier answers preserved.

## Best Possible Solution

Introduce one typed navigation/history model that separates reversible answer collection from committed side effects, defines downstream-answer invalidation, and exposes consistent previous-step semantics through CLI prompts, WizardSession, gateway schema/handlers, generated clients, and app clients.

## Reproduction Assessment

Yes at source level: current main has no Back sentinel, previous operation, answer-history store, or back RPC across the local and remote onboarding surfaces. A live interactive run was unnecessary to establish this missing contract in the read-only review.

## Solution Assessment

No, adding `[b] Back` independently to individual prompts is not the best solution because it cannot safely rewind the sequential orchestration or remote session. The narrowest maintainable direction is a shared typed navigation model with explicit answer invalidation and side-effect boundaries.
