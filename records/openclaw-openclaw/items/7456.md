## Summary

Keep open. Current main still has forward/cancel-only CLI and Gateway onboarding wizard flows, while the related Back-navigation implementation PR was closed unmerged and the separate iOS Back button does not cover `openclaw onboard`.

## What This Changes

The issue requests in-session Back navigation for the onboarding wizard, preserving prior answers when moving backward and forward.

## Best Possible Solution

Ship an additive Back/previous wizard contract with preserved-answer history across CLI prompts, Gateway RPC, generated clients, onboarding clients, docs, and regression tests.

## Reproduction Assessment

Yes. A high-confidence source-backed path exists: advance through `openclaw onboard` or the shared `wizard.*` RPC flow and there is no Back option, previous-step result, or `wizard.previous` call to invoke.

## Solution Assessment

No shipped solution exists yet. The maintainable path is a coordinated typed Back contract and history model; reviving #7762 may help, but it should be revised rather than treated as ready as-is.
