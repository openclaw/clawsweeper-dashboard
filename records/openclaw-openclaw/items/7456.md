This issue requests in-session Back navigation for `openclaw onboard`, with previous answers preserved when users move backward and forward.

Introduce a shared typed Back/previous contract and setup-flow history model across the wizard prompter, session, Gateway protocol, and clients, with answer preservation as part of the contract.

## Solution Assessment

Yes directionally, but it needs maintainer product/protocol approval before implementation. The maintainable solution is a shared Back/previous contract rather than a one-off local menu option.
