## Summary

Keep open. Current main still lacks a shared Back/previous contract for onboarding, and the related implementation attempts are closed or cover different workflows.

## What This Changes

Add in-session Back navigation to the onboarding wizard with prior answers preserved when moving backward and forward.

## Best Possible Solution

Add a version-safe Back/previous flow through WizardPrompter, WizardSession, Gateway schema/handlers, generated clients, and onboarding clients with regression coverage for preserved answers.

## Reproduction Assessment

## Solution Assessment

Unclear until maintainers settle the API and UX contract. A coordinated typed Back/previous contract plus state/history model is the maintainable direction; adding only a prompt-level Back escape hatch would be incomplete.
