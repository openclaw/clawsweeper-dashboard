## Summary

Keep open: current main still lacks a shared Back/previous-step contract for `openclaw onboard`, and the related Back-navigation PR was closed unmerged while resume/dialog work covers different workflows.

## What This Changes

The issue requests in-session Back navigation for the onboarding wizard with prior answers preserved when moving backward and forward.

## Best Possible Solution

Add an additive typed previous/back contract through the shared wizard prompt, session, Gateway protocol, generated client, and UI surfaces, then refactor onboarding around preserved step state before exposing it in CLI and app clients.

## Reproduction Assessment

Yes, at source level. Current main shows a sequential onboarding flow and shared wizard session/protocol surfaces that support next, answer, cancel, and status, but not previous-step navigation with answer preservation.

## Solution Assessment

Yes directionally, but it needs maintainer product approval and a cross-surface implementation. The safer solution is a shared typed Back/previous contract rather than a local `[Back]` option in only part of the wizard.
