## What This Changes

The issue requests in-session Back navigation for `openclaw onboard` with previous answers preserved when moving backward and forward.

## Best Possible Solution

Add a maintainer-approved typed previous-step contract across `WizardPrompter`, Clack prompts, `WizardSession`, Gateway wizard RPC/schema, clients, and regression coverage, backed by a setup-flow state/history model.

## Reproduction Assessment

Yes at source level: current main has no Back sentinel, previous-step API, or wizard RPC for rewinding. I did not run live onboarding because this review is read-only.

## Solution Assessment

Unclear as an implementation until maintainers approve the cross-surface UX/protocol contract. The direction is aligned, but a narrow local prompt hack would not be the best solution because Gateway and app clients share the wizard flow.
