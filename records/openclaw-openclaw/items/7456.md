## Summary

Keep open. Current main still exposes onboarding as a forward/cancel wizard, the prior Back-navigation PR was closed unmerged, and the remaining work is a cross-surface feature/API design rather than cleanup that can be safely closed.

## What This Changes

Add in-session Back navigation to the `openclaw onboard` wizard while preserving prior answers when moving backward and forward.

## Best Possible Solution

Add an additive, typed Back/previous contract through the wizard prompt API, Clack adapter, WizardSession, Gateway schema/handlers, generated clients, and onboarding clients, backed by a history model that preserves answers.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Source inspection on current main shows the wizard has forward answer, cancel, and status paths but no previous-step path.

## Solution Assessment

Unclear until maintainers settle the cross-surface API and UX contract. The maintainable direction is a typed Back/previous flow with preserved answers; only adding a local `[b] Back` prompt option would not cover Gateway/client wizard surfaces.
