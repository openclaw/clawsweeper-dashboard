## Summary

Keep this issue open: current main still exposes onboarding as a forward/cancel wizard, and the direct Back-navigation implementation attempt is closed unmerged while adjacent resume/dialog work does not solve in-session previous-step navigation with preserved answers.

## What This Changes

The issue requests a Back option in `openclaw onboard` so users can return to previous onboarding steps while preserving prior selections.

## Best Possible Solution

Add an additive typed Back/previous contract across the wizard prompt API, session runner, Gateway protocol, and clients, then refactor onboarding into a history-aware flow that preserves answers when moving backward and forward.

## Reproduction Assessment

Yes, at source level: current main has only forward answer submission and cancellation paths for onboarding. I did not run the interactive wizard in this read-only review, but the missing Back contract is visible across prompts, session handling, Gateway protocol, docs, and tests.

## Solution Assessment

Unclear until maintainers choose the product contract. The maintainable direction is an additive Back/previous protocol and history-aware setup flow, not a local menu-only shortcut in one prompt adapter.
