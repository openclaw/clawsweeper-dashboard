## Summary

Keep open: current main still has the fast main-session `systemEvent` `next-heartbeat` handoff without the requested ghost-run warning or threshold config, and the same-author linked implementation PR is still open.

## What This Changes

The issue asks for a scoped warning/run-log annotation and configurable threshold for very fast successful main-session cron `systemEvent` handoffs that may not have produced an agent turn.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/72677, or an equivalent maintainer-approved change, so fast main-session `next-heartbeat` handoffs get a scoped warning/run-log marker and documented threshold config.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: a `main` `systemEvent` job with `wakeMode: "next-heartbeat"` follows the enqueue-and-request-heartbeat path and records success without a warning check.

## Solution Assessment

Yes for the narrow observability problem: a scoped warning and persisted annotation is a smaller change than making cron wait for confirmed agent processing, while the threshold config handles fast legitimate handoffs.
