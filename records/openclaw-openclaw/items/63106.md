## Summary

Keep open: current main still has the fast `main`/`systemEvent`/`next-heartbeat` handoff and no ghost-run warning or configurable threshold, while same-author PR #72677 remains the active linked implementation candidate.

## What This Changes

Add a configurable warning and run-log signal for suspiciously fast successful main-session `systemEvent` cron runs using `wakeMode: "next-heartbeat"`.

## Best Possible Solution

Review and land #72677, or an equivalent maintainer-approved change, so the fast main-session handoff gets a scoped structured warning/run-log annotation plus documented config/schema support.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: a `main` `systemEvent` job with `wakeMode: "next-heartbeat"` follows the enqueue-and-request-heartbeat path and records a successful finished event without any threshold or warning check.

## Solution Assessment

Yes for the narrow observability problem. A scoped warning and annotation is the smallest maintainable fix; making cron wait for confirmed agent processing would be a larger behavior change.
