## Summary

Keep open. Current main still has the reported fast-return `main`/`systemEvent`/`next-heartbeat` cron path without a ghost-run warning or configurable threshold, and the same-author implementation PR that uses closing syntax is still open, so this issue should remain open until that PR lands or maintainers choose a different fix.

## What This Changes

Add a scoped, configurable warning or run-log annotation for main-session `systemEvent` cron jobs using `wakeMode: "next-heartbeat"` that finish suspiciously fast with `status: "ok"`.

## Best Possible Solution

Review and land #72677, or an equivalent maintainer-approved change, so the fast main-session handoff gets a scoped structured warning/run-log annotation plus a documented config/schema threshold without changing cron into a confirmed-agent-turn awaiter.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: a `main` `systemEvent` job with `wakeMode: "next-heartbeat"` follows the fast handoff in `executeMainSessionCronJob`, then the gateway records the finished event and run log without any threshold or warning check.

## Solution Assessment

Yes for the narrow observability problem. A scoped warning and annotation is the smallest maintainable fix; making cron wait for confirmed agent processing would be a larger behavioral change and should be a separate product decision.
