## What This Changes

The issue requests preventing gateway/status restart paths from creating or showing an unconfigured Feishu `default` account unless the user explicitly configured it.

## Best Possible Solution

Review, land, or replace the linked PR with a fix that distinguishes synthetic fallback ids from authored accounts, preserves diagnostics for explicit unresolved configs, and adds Feishu-focused restart/status regression proof.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection on current main shows Feishu can inherit a synthetic `default` account id and the status paths can render or materialize that id without a real configured account.

## Solution Assessment

No, closing this issue now is not the best way to solve it. The maintainable path is to fix the account/status boundary in https://github.com/openclaw/openclaw/pull/64712 or a replacement while preserving diagnostics for genuinely configured but unresolved accounts.
