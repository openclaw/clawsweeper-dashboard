## Summary

Keep open: the Discord status-probe performance problem remains on current main, but this branch is not merge-ready because it regresses current voice/stage audit semantics, its new audit test timed out in CI, and it still lacks required real behavior proof.

## What This Changes

The PR changes Discord permission audits to run configured channel checks through a bounded parallel worker pool, adds a 30-second bot/guild/member permission-context cache, and adds regression tests for ordering and cache reuse.

## Best Possible Solution

Refresh the patch on current main so Discord status audits use bounded ordered concurrency and short per-token/guild context reuse while preserving channel-type permission semantics, then verify with deterministic tests and redacted real status-probe output.

## Reproduction Assessment

Yes at source level, but not as a live timeout. Current main visibly routes status probes into serial Discord channel audits and refetches bot/guild/member context per channel; the timeout claim still needs redacted real Discord proof.

## Solution Assessment

No, not as-is. Bounded ordered concurrency plus short permission-context reuse is the right repair direction, but this branch must preserve current voice/stage requirements, replace the timer-dependent test, refresh against main, and add real behavior proof.
