- **[P1] Preserve voice permission requirements:** `extensions/discord/src/audit-core.ts:100-121`
  - body: Current main selects required permissions from `perms.channelType`, including Connect/Speak/ReadMessageHistory for voice and stage channels.   - confidence: 0.88
- **[P1] Keep the timeout signal through cached fetches:** `extensions/discord/src/send.permissions.ts:251-269`
  - body: Current main passes `timeoutMs` and an abort `signal` into Discord permission diagnostics and `fetchChannelPermissionsDiscord` checks that signal around REST calls. This branch's cached path is based on the older client shape and would need to carry the current timeout/signal through the cached bot/guild/member loads, otherwise the audit can reintroduce unbounded REST waits.
  - confidence: 0.87
- **[P2] Avoid raw bot tokens as cache keys:** `extensions/discord/src/send.permissions.ts:76-77`
  - body: `getDiscordGuildPermissionCacheKey` puts the raw Discord bot token into a module-global Map key, and expired cache entries are never purged unless the same key is touched again. That retains rotated or one-off tokens in process memory for the lifetime of the process; use a non-secret scoped key and delete expired entries.
  - confidence: 0.83
