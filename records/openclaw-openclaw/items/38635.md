## Summary

Keep open: the intended fix is valid, but current main’s user-facing hamburger is `.topbar-nav-toggle`, while the branch and regression target `.nav-collapse-toggle`, so the visible broken path would remain unfixed.

## What This Changes

The PR changes the Control UI sidebar collapse click handler to clear user chat focus mode and adds a browser test for clicking a hamburger while focus mode is active.

## Best Possible Solution

Update the current `.topbar-nav-toggle` handler to exit user-enabled chat focus mode and open the drawer, preserve onboarding-controlled focus semantics, add coverage in `ui/src/ui/navigation.browser.test.ts`, and include a changelog fix entry.

## Reproduction Assessment

Yes, by source inspection on current main. On `/chat`, user-enabled focus mode makes `chatFocus` true, which suppresses drawer visibility while `.topbar-nav-toggle` only flips raw `state.navDrawerOpen`.

## Solution Assessment

No. The intended behavior is right, but the PR applies it to the sidebar collapse control and a stale test surface; the narrow maintainable fix is to update the current topbar drawer handler and navigation browser coverage.
