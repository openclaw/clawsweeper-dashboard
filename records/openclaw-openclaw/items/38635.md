Keep open: the branch addresses a real Control UI focus-mode UX bug, but it edits the sidebar collapse button instead of the visible topbar hamburger on current main, and the external PR still lacks after-fix real behavior proof.

## What This Changes

The branch changes the Control UI sidebar collapse button to clear user-enabled chat focus mode and adds a browser regression test that clicks `button.nav-collapse-toggle`.

## Best Possible Solution

Either close this PR as obsolete if maintainers accept the existing focus-exit affordance, or update the visible `.topbar-nav-toggle` handler to leave user focus mode and add coverage in `ui/src/ui/navigation.browser.test.ts`.

Yes, by source inspection on current main. On `/chat`, user-enabled focus mode makes `chatFocus` true, derived drawer visibility stays false, and the visible `.topbar-nav-toggle` only flips raw drawer state.

## Solution Assessment

No. The branch edits and tests `.nav-collapse-toggle`, but the narrow maintainable fix would update and test `.topbar-nav-toggle`, or be closed as obsolete if the current focus-exit affordance is the accepted UX.
