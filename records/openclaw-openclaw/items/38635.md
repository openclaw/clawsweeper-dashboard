## Summary

Keep open: the PR is a plausible bug fix, but current-main source review and the existing review thread show it patches the sidebar collapse button while the reported visible hamburger path remains unchanged, and the external PR still lacks real behavior proof.

## What This Changes

The PR changes Control UI sidebar collapse handling to clear user-enabled chat focus mode and adds a browser regression test that clicks `button.nav-collapse-toggle`.

## Best Possible Solution

Update `.topbar-nav-toggle` to exit user-enabled focus mode and open the drawer while preserving onboarding semantics, then cover that path in `ui/src/ui/navigation.browser.test.ts`; if maintainers accept the linked issue as already solved by the current focus-exit affordance, close this PR as obsolete instead of merging this branch.

## Reproduction Assessment

Yes, by source inspection. On `/chat`, user-enabled focus mode makes `chatFocus` true, and the visible `.topbar-nav-toggle` only flips raw drawer state while the derived `shell--nav-drawer-open` class remains suppressed.

## Solution Assessment

No. The intended behavior is reasonable, but this branch applies it to `.nav-collapse-toggle` and tests that hidden/sidebar trigger instead of the current topbar hamburger path.
