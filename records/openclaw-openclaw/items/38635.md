## Summary

Keep open: the underlying Control UI focus-mode hamburger path is still source-reproducible on current main, but this PR updates the sidebar collapse control instead of the currently visible topbar hamburger and its regression covers the wrong trigger.

## What This Changes

The PR changes the Control UI nav-collapse click handler to clear user-enabled chat focus mode and adds a browser regression for clicking a hamburger while focus mode is active.

## Best Possible Solution

Update the current `.topbar-nav-toggle` handler to exit user-enabled chat focus mode and open the nav drawer, preserve onboarding-controlled focus semantics, add coverage in `ui/src/ui/navigation.browser.test.ts`, and include the required changelog entry.

## Reproduction Assessment

Yes, by source inspection. On `/chat`, user-enabled focus mode makes `chatFocus` true, and clicking `.topbar-nav-toggle` only flips raw drawer state while the derived `shell--nav-drawer-open` class remains suppressed.

## Solution Assessment

No. The PR applies the desired behavior to `.nav-collapse-toggle`; the narrow maintainable fix is to update the current topbar hamburger path and current navigation browser test surface.
