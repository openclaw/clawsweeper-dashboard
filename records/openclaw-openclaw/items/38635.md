## Summary

Keep this PR open: the Control UI focus-mode hamburger bug still exists on current main, but the branch is stale and updates the sidebar collapse button rather than the current visible topbar hamburger path.

## What This Changes

The PR changes the Control UI sidebar collapse click handler to clear user-initiated chat focus mode and adds a browser regression test that clicks `button.nav-collapse-toggle` after enabling focus mode.

## Best Possible Solution

Adapt this work to current main by updating the `.topbar-nav-toggle` click handler so user-initiated chat focus mode clears before opening the drawer/nav, then add the regression to `ui/src/ui/navigation.browser.test.ts` against `.topbar-nav-toggle`. The sidebar `.nav-collapse-toggle` can remain a normal collapse control unless maintainers want the desktop sidebar control to share the same escape behavior.
