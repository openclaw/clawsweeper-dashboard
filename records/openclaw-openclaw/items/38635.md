## Summary

Keep open: the PR targets a real Control UI focus-mode bug, but the branch still patches the sidebar header collapse button instead of the visible topbar hamburger and lacks real after-fix behavior proof.

## What This Changes

The PR changes the Control UI sidebar header collapse button to clear user-enabled chat focus mode and adds a browser test that clicks `button.nav-collapse-toggle`.

## Best Possible Solution

Update the `.topbar-nav-toggle` handler to exit user-enabled chat focus mode and open the drawer while preserving onboarding-controlled focus semantics, then cover that path in `ui/src/ui/navigation.browser.test.ts`.

## Reproduction Assessment

Yes. By source inspection on current main, user-enabled chat focus makes `chatFocus` true, which suppresses the derived drawer-open class while the visible `.topbar-nav-toggle` only flips raw `state.navDrawerOpen`.

## Solution Assessment

No. The intended behavior is narrow and maintainable, but this branch applies it to `.nav-collapse-toggle`; the fix needs to live on the current topbar hamburger path with a regression against that trigger.
