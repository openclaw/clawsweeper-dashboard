## Summary

Keep this PR open: the focus-mode hamburger bug is still source-reproducible on current main, but this branch changes the sidebar collapse control instead of the visible topbar hamburger and it does not include after-fix real behavior proof.

## What This Changes

The PR changes the Control UI sidebar collapse button to clear user-enabled chat focus mode and adds a browser test that clicks `button.nav-collapse-toggle`.

## Best Possible Solution

Update the `.topbar-nav-toggle` handler to exit user-enabled chat focus mode and open the drawer while preserving onboarding-controlled focus semantics, then cover that path in `ui/src/ui/navigation.browser.test.ts`.

## Reproduction Assessment

Yes, by source inspection. On `/chat`, user-enabled focus mode makes `chatFocus` true, and the visible `.topbar-nav-toggle` only flips raw drawer state while the derived `shell--nav-drawer-open` class remains suppressed.

## Solution Assessment

No. The intended behavior is right, but the branch applies it to `.nav-collapse-toggle`; the narrow maintainable fix is the current topbar hamburger path plus a regression against that trigger.
