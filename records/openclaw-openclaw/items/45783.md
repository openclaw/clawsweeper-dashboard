## Risks / Open Questions

- Merging this branch as-is would enable a provider-bound replacement/restoration layer by default for existing users, changing prompts, tool arguments, output text, and local storage behavior on upgrade.
- The PR is currently merge-conflicting with current main, and current main has refactored relevant TTS/runtime surfaces since the PR base.

- Tool-result content in non-text text block shapes can bypass outbound replacement, leaving a gap in the advertised all-message-role filtering guarantee.
- The reversible mapping store and inbound restoration path can affect session continuity, placeholder visibility, and tool argument fidelity if mappings are missing, stale, or corrupted.
