## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep filtering active when the mapping lock is busy:** `src/privacy/mapping-store.ts:257`
  - body: `withWriteLock()` throws whenever another process owns the active lock, and setup calls `loadSession()` before installing the wrapper. The runner then catches that setup error and continues without privacy filtering, so normal concurrent access can send raw prompts to the provider even though privacy is enabled.
  - confidence: 0.88
- **[P2] Do not send raw TTS summaries after privacy setup fails:** `src/tts/tts-core.ts:478-480`
  - body: If privacy context setup or filtering throws, this catch restores `summaryPromptText` to the original transcript and still calls the summary model. A busy or unwritable mapping store can therefore leak the unfiltered text through the TTS summary path.
  - confidence: 0.82
