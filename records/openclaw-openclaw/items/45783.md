## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Fail closed when TTS privacy filtering fails:** `src/tts/tts-core.ts:478-480`
  - body: If `createPrivacyFilterContext` or `filterText` throws, this catch restores the original transcript into `summaryPromptText`, and the later `completeSimple` call still sends it to the model provider. With privacy enabled, this path should skip the provider call or otherwise fail closed instead of leaking the transcript in the degraded case.
  - confidence: 0.86
- **[P2] Filter text-like tool result blocks before replay:** `src/privacy/stream-wrapper.ts:535-536`
  - body: `filterToolResultMessage()` only rewrites blocks with `type === "text"`, while current session/gateway utilities also treat `input_text` and `output_text` blocks as text. A secret returned in one of those text-like tool result blocks bypasses the new outbound privacy wrapper and can be replayed to the model unfiltered.
  - confidence: 0.74
