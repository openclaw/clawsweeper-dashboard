## Summary

## What This Changes

The branch adds a default-on privacy config, detector/replacer/mapping store, outbound stream scrubbing with inbound restoration, log redaction integration, BTW/TTS filtering, docs, and unit tests.

## Best Possible Solution

Define the supported core privacy contract, then either fix this branch's remaining bypasses or consolidate it with the canonical privacy implementation before merge.

## Solution Assessment

No as-is. The direction addresses a real privacy gap, but the implementation needs the remaining egress/fail-open defects fixed and maintainer approval of the default-on privacy contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Filter OpenAI tool-result text blocks:** `src/privacy/stream-wrapper.ts:535-536`
  - body: `filterToolResultMessage` only rewrites blocks with `type: "text"`, but current OpenAI replay also converts `input_text` and `output_text` tool-result blocks into provider text. A tool-returned secret in those block shapes will be sent upstream raw on the next turn.
  - confidence: 0.89
- **[P2] Keep TTS summaries masked on privacy setup failure:** `src/tts/tts-core.ts:478-480`
  - body: When privacy context setup or filtering throws, this catch restores `summaryPromptText` to the original text and still calls the summary model. A busy or unwritable mapping store can therefore silently send the unfiltered transcript even though privacy filtering is enabled.
  - confidence: 0.78
