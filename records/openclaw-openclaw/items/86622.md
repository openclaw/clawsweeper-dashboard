## Summary

Keep open, but this PR is not merge-ready: current main already sends Tavily credentials as a Bearer header, the current Tavily docs and JS SDK support that header shape, and the patch would duplicate the same API key into the JSON body without real valid-key proof.

## What This Changes

The PR changes `postTavilyJson` so every Tavily search and extract request body includes `api_key` while still passing `apiKey` to the shared POST helper.

## Best Possible Solution

Use exactly one current vendor-backed Tavily credential transport, with redacted valid-key proof and focused coverage for both search and extract before merge.

## Solution Assessment
