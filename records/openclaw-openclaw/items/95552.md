## Summary

Keep this PR open: the Mattermost thread-participation direction is useful, but the current patch still misses the default draft-preview finalization path, so a visible in-thread reply can fail to record participation and the next unmentioned follow-up can still be dropped. The protected maintainer label and always-on mention-gate behavior change also make this a maintainer-reviewed merge, not cleanup-close material.

## What This Changes

This PR adds a Mattermost thread-participation cache backed by plugin state, feeds it into the Mattermost mention gate, records visible thread replies, and adds cache/gate tests.

## Best Possible Solution

Keep the Mattermost-local cache and gate shape, but record participation from a visible-final result path that covers normal sends and preview-finalized edits, document the thread exception, and then prove the restart follow-up behavior.

## Reproduction Assessment

Yes, by source inspection. Mattermost defaults to draft-preview streaming, same-thread text finals can complete by editing the preview post, and this PR records participation only in the normal deliverPayload callback that the preview-finalized path skips.

## Solution Assessment

No. The Mattermost-local cache and mention-gate boundary is reasonable, but the patch must record all visible final delivery outcomes and document the new default behavior before it is the best fix.
