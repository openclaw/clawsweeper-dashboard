## Summary

Keep open. Current main still exposes `message_sending` as a delivery-level hook without `stopReason`, `hasToolCalls`, or a typed agent completion context, while agent run metadata already carries stop/tool-call signals; related PR #39761 remains open as the active API-shape candidate.

## What This Changes

Expose a structural stop/tool-call signal to `message_sending` plugin hooks so plugins can suppress non-final reasoning preambles.

## Best Possible Solution

Add a backward-compatible typed agent completion context derived from `EmbeddedPiRunMeta`, thread it through agent command and reply delivery into `message_sending`, and document the hook semantics with focused contract tests.

## Reproduction Assessment

Yes. Source inspection on current main shows the hook event type and hook payload omit stop/tool-call metadata while upstream agent metadata already carries `stopReason` and pending tool-call data.

## Solution Assessment

Unclear as a final API shape. The structural signal is the right class of fix, but maintainers should decide whether to land direct metadata fields or the broader `agentContext` contract proposed in PR #39761.
