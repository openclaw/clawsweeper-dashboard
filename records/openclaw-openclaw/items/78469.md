## Summary

Close as ClawHub: this PR adds a new optional runtime plugin, and current OpenClaw policy directs optional plugin capabilities to external distribution when the existing plugin API already supports them.

## What This Changes

The PR adds a bundled `extensions/temporal-context` runtime plugin, tests, README, manifest, package metadata, and lockfile importer for injecting date/time and per-session elapsed-turn context into system-prompt space.

## Best Possible Solution

Publish and maintain this temporal grounding capability as a ClawHub/native plugin using the existing hook API, while keeping core's cache-stable time behavior unchanged unless a future plugin API gap is identified.

## Reproduction Assessment

Not applicable; this PR adds an optional plugin rather than reporting broken current behavior. Source and docs inspection show the required runtime injection hook already exists.

## Solution Assessment

No; bundling this optional temporal prompt injection in core is not the narrowest maintainable path. The safer direction is an external ClawHub plugin because the current `before_prompt_build` and `prependSystemContext` APIs already cover the needed seam.
