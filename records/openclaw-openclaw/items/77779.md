## Summary

Close as outside this source repo: current OpenClaw main and the v2026.5.4 tag inject `api.runtime`, while the observed timeout is in the external Tencent WeChat plugin’s runtime handoff code.

## What This Changes

The issue asks OpenClaw to address a WeChat external plugin startup regression where `api.runtime` is reported missing during plugin registration.

## Best Possible Solution

Keep the OpenClaw SDK/runtime contract as-is and have the external WeChat plugin publish a fix that uses the `ChannelGatewayContext.channelRuntime` path or an SDK runtime store instead of waiting on a module-global runtime.

## Reproduction Assessment

No high-confidence live reproduction was established because no WeChat account was run. Source inspection does show the timeout path in the external plugin, while current OpenClaw source shows `api.runtime` is injected.

## Solution Assessment

No repo-local code change is the best fix for the reported timeout. The narrow maintainable fix belongs in the external WeChat plugin’s startup/monitor handoff; a new OpenClaw issue should use a minimal plugin if core registration still appears to pass an undefined runtime.
