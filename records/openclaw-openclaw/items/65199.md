## Summary

Close as superseded: the linked bug report was closed as not reproducible on current main, the safe Cron Jobs formatter work is now tracked by the narrower open PR at https://github.com/openclaw/openclaw/pull/65382, and this PR's Dreaming change conflicts with the documented single managed sweep model.

## What This Changes

The PR adds a cron-expression humanizer, uses it in Cron Jobs schedule text, and changes Dreaming phase badges to show humanized cron text plus next-run time.

## Best Possible Solution

Let the narrower Cron Jobs formatter PR carry the remaining UI enhancement, and keep Dreaming phase status tied to the documented single managed sweep unless a real per-phase schedule API is added.

## Reproduction Assessment

Yes for the PR-introduced Dreaming defect: source shows one dreaming frequency and one managed sweep status copied to all phases, and the screenshot shows identical phase labels. No high-confidence current-main reproduction exists for the original linked `*/6` rendering as `3:00`; that report was closed not reproducible.

## Solution Assessment

No. A shared Cron Jobs formatter is a reasonable narrow direction, but changing Dreaming phase badges to read the shared sweep cron as per-phase schedules is not the best fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep the sweep cron out of phase labels:** `ui/src/ui/views/dreaming.ts:451`
  - body: `phase.cron` and `phase.nextRunAtMs` both come from the single managed dreaming sweep, so this renders light, deep, and REM as if they have independent schedules when they do not. Keep the Dreaming label tied to sweep status, or add a real per-phase scheduling contract first.
  - confidence: 0.9
- **[P3] Guard fixed-minute hourly cron labels:** `ui/src/ui/format.ts:162-164`
  - body: The `hour === "*"` branch accepts any numeric minute, so malformed persisted cron like `60 * * * *` would display as `Every hour at :60` instead of falling back to raw cron text. Add the same 0-59 minute guard used by the other branches.
  - confidence: 0.78
