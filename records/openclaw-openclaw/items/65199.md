## Summary

Keep open. Current main has not implemented cron humanization and this PR still carries useful UI work, but it should stay human-reviewed because the linked bug was closed as not reproducible, the Dreaming copy touches the one-sweep schedule model, and after-fix real behavior proof remains insufficient.

## What This Changes

The PR adds a shared cron-expression humanizer, uses it for Cron Jobs and Dreaming phase schedule text, and adds formatter tests.

## Best Possible Solution

Keep a shared cron-label formatter only if maintainers want the Control UI enhancement, align Dreaming wording with the one managed-sweep model, tighten the invalid-minute fallback, and require after-fix proof covering both Dreaming and Cron Jobs.

## Reproduction Assessment

No for the original reported `*/6` to `3:00` bug: the linked issue was closed as not reproducible, and current main's Cron Jobs path renders raw cron text. Source inspection does confirm the enhancement target: Cron Jobs labels are raw and Dreaming phase badges only show next-run timestamps.

## Solution Assessment

Unclear. A shared formatter is a narrow path for Cron Jobs, but Dreaming phase schedule text needs maintainer product/copy judgment because one managed full-sweep cron status is shared across light, deep, and rem.
