## Summary

Keep open: current main still has the source-reproducible false positive, and https://github.com/openclaw/openclaw/pull/81778 remains an open closing PR for the same issue.

## What This Changes

The issue requests that `gateway status` stop reporting a missing gateway subcommand when a macOS LaunchAgent runs OpenClaw through a `/bin/zsh -lc` wrapper.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81778 or an equivalent narrow service-audit tokenization fix, then close this issue after the merge reaches main.

## Reproduction Assessment

Yes. Current main passes LaunchAgent `ProgramArguments` into the service audit and only recognizes an exact `gateway` argv element, so the reported `/bin/zsh -lc ... gateway --port ...` wrapper is source-reproducible as a false `gateway-command-missing` warning.

## Solution Assessment

Yes. A narrow service-audit command tokenization fix with regression coverage is the maintainable path, and the open closing PR is already taking that route.
