## Summary

Current main and v2026.6.8 still use exact top-level argv parsing for the gateway service audit, so the reported `/bin/zsh -lc "... gateway --port ..."` LaunchAgent command can still produce the false `gateway-command-missing` warning. This issue should stay open because the linked open pull request owns the narrow fix path and uses closing syntax.

## What This Changes

The issue requests that gateway status recognize shell-wrapped macOS LaunchAgent commands containing `gateway --port <port>` instead of reporting a missing gateway subcommand.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81778, or an equivalent narrow service-audit parser and regression-test fix, then close this issue after merge.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to pass ProgramArguments shaped as `/bin/zsh`, `-lc`, and an inline `... gateway --port 18890` command into `auditGatewayServiceConfig`; current main only recognizes exact top-level `gateway` and `--port` tokens.

## Solution Assessment

Yes. The narrowest maintainable fix is to teach the service-audit parser to inspect supported shell `-c` payloads and add the LaunchAgent wrapper regression coverage without changing service generation or gateway runtime behavior.
