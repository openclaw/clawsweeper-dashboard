## Summary

Keep open: current main still has a source-reproducible false positive in the gateway service audit for shell-wrapped LaunchAgent commands, and two open PRs already reference this issue with closing syntax. The issue should remain open until one fix is reviewed and merged.

## What This Changes

The issue reports that `openclaw --profile enterprise gateway status` flags shell-wrapped macOS LaunchAgent gateway commands as missing the `gateway` subcommand.

## Best Possible Solution

Review and land one consolidated fix that normalizes supported shell-wrapped LaunchAgent commands before the gateway subcommand and port audit, with focused regression coverage, then close this issue after merge.

## Reproduction Assessment

Yes from source inspection: a `/bin/zsh -lc` LaunchAgent wrapper keeps `gateway --port` inside one shell payload string, while current main only checks for an argv element exactly equal to `gateway`. I did not run a live macOS LaunchAgent in this read-only review.

## Solution Assessment

Yes: a narrow service-audit command normalization fix with regression tests is the right path. The issue should stay open while the linked fix PRs are reviewed, because both are implementation candidates rather than merged fixes.
