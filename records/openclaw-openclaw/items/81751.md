## What This Changes

The issue asks gateway status to recognize shell-wrapped macOS LaunchAgent commands that execute `openclaw ... gateway --port <port>` without warning that the gateway subcommand is missing.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81778 or an equivalent narrow service-audit tokenization fix, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes. Current main reads LaunchAgent `ProgramArguments` into the status audit and only checks exact argv elements for `gateway`, so the reported `/bin/zsh -lc ... gateway --port ...` wrapper is source-reproducible as a false `gateway-command-missing` warning.

## Solution Assessment

Yes. A narrow service-audit command-token normalization fix with regression tests for the shell wrapper and near-match cases is the maintainable solution, and the active linked PR is already taking that path.
