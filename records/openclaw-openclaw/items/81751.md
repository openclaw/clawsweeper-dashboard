## Summary

Keep open: current main still audits service commands by exact argv tokens, so shell-wrapped LaunchAgent commands can still produce the false missing-gateway warning; the same-author open PR https://github.com/openclaw/openclaw/pull/81778 is the active fix path and should land or be closed before this issue closes.

## What This Changes

The issue asks gateway status service audit to recognize shell-wrapped macOS LaunchAgent commands that run the OpenClaw gateway with a port.

## Best Possible Solution

Land or otherwise adopt https://github.com/openclaw/openclaw/pull/81778, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes. From source inspection, a ProgramArguments array shaped like /bin/zsh, -lc, and a shell payload containing openclaw gateway --port reaches auditGatewayServiceConfig but hasGatewaySubcommand only checks exact top-level elements.

## Solution Assessment

Yes. The narrow maintainable fix is to tokenize or otherwise inspect shell command fragments inside service-audit and add regression coverage for the positive wrapper case plus near-match negatives, which is what the linked open PR does.
