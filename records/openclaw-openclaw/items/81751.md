## What This Changes

The issue asks `gateway status` to recognize shell-wrapped macOS LaunchAgent commands that ultimately run the OpenClaw `gateway --port <port>` subcommand.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81778 or an equivalent narrow service-audit tokenization fix, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes. Source inspection shows a `/bin/zsh -lc` LaunchAgent wrapper keeps `gateway --port` inside one shell payload string, while current main only accepts an argv element exactly equal to `gateway`.

## Solution Assessment

Yes. A narrow normalization/tokenization fix in `src/daemon/service-audit.ts`, with regression coverage for the wrapper and near-match cases, is the maintainable path; the open canonical PR follows that direction.
