## Summary

Keep open: current `main` still has the source-reproducible false positive, and https://github.com/openclaw/openclaw/pull/81778 is an open focused PR with closing syntax for this issue, so closing before merge would be premature.

## What This Changes

`gateway status` should recognize shell-wrapped macOS LaunchAgent commands that execute `openclaw gateway --port <port>` and avoid a misleading missing-subcommand warning.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81778 or an equivalent narrow service-audit tokenization fix, then close this issue after the merge reaches `main`.

## Reproduction Assessment

Yes. Current `main` reads LaunchAgent `ProgramArguments` into the service audit and only checks exact argv elements for `gateway`, so the reported `/bin/zsh -lc ... gateway --port ...` wrapper is source-reproducible as a false `gateway-command-missing` warning.

## Solution Assessment

Yes. A narrow command-token normalization fix with regression coverage for the shell wrapper and near matches is the maintainable solution, and the active linked PR is already taking that path.
