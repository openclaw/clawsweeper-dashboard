## What This Changes

The issue asks `gateway status` service audit to recognize shell-wrapped macOS LaunchAgent commands that execute `openclaw ... gateway --port <port>`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81778, or an equivalent narrow service-audit fix with regression coverage, then close this issue after merge.

## Reproduction Assessment

Yes. Current main reads LaunchAgent `ProgramArguments` into the status audit unchanged, and the audit only checks exact argv elements for `gateway`, making the reported `/bin/zsh -lc ... gateway --port ...` wrapper a high-confidence source reproduction.

## Solution Assessment

Yes. A narrow command-token normalization fix in `src/daemon/service-audit.ts`, with regression tests for the reported wrapper and near matches, is the maintainable path; the open linked PR follows that shape.
